新しいITインフラストラクチャアクセスソリューション：Veeam Availability Orchestrator

ほぼ毎日、別の会社でITインフラストラクチャに重大な誤動作が発生し、それが深刻な財務および評判の損失につながっていることがわかりました。 システム障害は、ITサービスの運用と組織全体のリスクを計算するときに最初に頭に浮かぶものです。 ただし、多くの場合、他の理由があります：人的要因、機器の停止、停電、マルウェア、データ破損-これらすべてがシステムとアプリケーションの完全な停止につながる可能性があります。



スイッチの手順と計画に十分な注意を払わず、適切なタイミングで更新せず、適切なタイミングでテストしない場合、計画または緊急のバックアップサイトへのフェールオーバーが成功する可能性は低くなります。 マシンおよびアプリケーションの誤って指定された依存関係、文書化されていない変更、誤ったプロセス、アプリケーションおよびサービスの正しい動作のチェックされていない、レプリカからのマシンの誤った起動シーケンスは、1つのアプリケーションまたはデータセンター全体の正常な切り替えを妨げる可能性がある問題の一部です。



組織がこれらの問題に対処できるように、Veeamは最近、バックアップサイトへの切り替え時にVMレプリカの管理を自動化する新しいVeeam Availability Orchestratorをリリースしました。 （ソリューションの最初のバージョンは、VMware vSphereに基づく仮想マシンをサポートします。）catの下にある新製品の詳細をご覧ください。







ご存じのとおり、バックアップサイトへの切り替え計画は、災害および災害復旧計画（DR計画）の不可欠な部分です。 したがって、仮想マシンが相互に依存する重要なアプリケーションを実行している場合、事故の場合にシステムを維持するには、これらのVMのレプリカに特定の順序で正しく切り替える必要があります（最初にドメインコントローラー、次にExchange、およびその逆） 。 このプロセスを自動化するために、Veeam Backup＆Replicationはフェイルオーバー計画を作成します。



  • VM切り替えシーケンス
  • 切り替え間の一時停止の長さ（DNSサーバーなどのメインVMが完全に動作する前に依存VMを誤って起動しないように一時停止が必要です）


フェイルオーバー計画は、障害発生時に迅速に開始できるように、事前に準備する必要があります。 切り替えの開始時に、VMのレプリカに切り替える時点（状態）を選択できます。 次に、リストから各VMのVeeam Backup＆Replicationがそのレプリカを見つけ、計画設定に従ってレプリカを含めます。









手順の最終段階を完了することは必須です。これは、バックアップサイトでの最終的な固定またはリバーススイッチング（たとえば、メインサイトでのメンテナンス/修復の完了後）のいずれかです。 このすべてについては、 こちら（ロシア語）をご覧ください 。



深刻な組織にフェイルオーバー計画がないと想像するのは困難です。 では、なぜそのような規則性で障害が発生するのでしょうか？ 計画を立てるのが難しいのはなぜですか？



徹底した文書化の必要性



計画を成功させるための基礎は、完全で最新のドキュメントです。 ITインフラストラクチャはより大きく複雑になり、常に変化しています。 手動のIT作業のすべての重要な側面を文書化することはあまり効果的ではありません。ここで、計画の失敗の考えられる理由の1つに直面しています。



テストの必要性



定期的な包括的なテストは、フェイルオーバーおよび災害復旧計画を計画し、慎重に計画するのに役立ちます。 これは、現代のインフラストラクチャ、特に複数のサイトに基づくインフラストラクチャの構築が複雑であるため、特に重要です。 これに、エンドユーザーの潜在的な結果の重要性を追加します。 しかし、計画の実際の有効性は、しばしばデューデリジェンスに合格しません。



規制要件



計画と災害復旧は、データの整合性、信頼性、および可用性の要件への準拠を管理する規制と密接に関連しています。 要件と規範は業界によって異なりますが、満たす義務によって結ばれています。 そうしないと、高い評判と経済的リスクが伴います。



Veeam Availability Orchestratorが役立つ理由



Veeam Availability Orchestratorは、フェイルオーバー計画の準備とテストのコストを削減します。 このソリューションは、フェイルオーバーおよびリカバリ計画の作成、文書化、テストのプロセスを自動化するように設計されており、すぐに署名のために管理者に送信し、後で監査人または監査人に提示できます。



インフラストラクチャの文書化の支援



新しいソリューションは、監査および規制要件に従ってインフラストラクチャを文書化するのに役立ちます。 製品は以下を提供します：



  • ガイドに署名する準備ができた便利な形式のテンプレートに基づく4つのカスタムレポート
  • 定期的に更新および公開する機能
  • 行われたすべての変更を自動的に文書化する


テスト支援



Veeam Availability Orchestratorは、スイッチプランのテスト中に製造インフラストラクチャの負担を取り除きます。 さらに、手動の手順を節約し、新製品の機能を使用することにより、コストを削減できます。



  • オンデマンドまたはスケジュールされたテスト
  • リアルタイムの計画ステータスレポートを受け取る
  • 実動システムとそのユーザーを関与させない、計画の準備状況の定期的なテスト


計画支援



新しいソリューションは、Veeam Backup＆Replicationを使用してVMware vSphere仮想マシンのレプリケーションを管理するのに役立ちます。 この機能の一部として、次のものがサポートされています。



  • 指定された起動シーケンスに従って、切り替え中の仮想マシン、アプリケーション、およびサービス（Microsoft SQL Server、Exchange、IISなど）の検証
  • オープンAPIを使用した他のツールとの統合
  • 関連するリソースへのアクセスの委任と制御


中身は何ですか



Veeam Availability Orchestratorには以下が含まれます。



  • Veeam Availability Orchestrator
  • Veeam Backup＆Replication Enterprise Plus（統合）
  • Veeam ONE（ビルトイン）
  • Microsoft SQL Server Express


重要！ ビルトイン製品エディションは、Veeam Availability Orchestrator（VAO）をサポートするためにのみライセンスされており、本番環境で完全に機能することを意図したものではありません。 すでにVeeam Backup＆Replicationを展開している場合、新しいソリューションを使用するには、そのバージョンが9.5 Update 3（以上）であり、Veeam Availability Orchestratorとの統合に必要なAPIが提示されているため、Enterprise Plusエディションでなければならないことに注意してくださいこのエディションのみ。 Veeam Backup＆ReplicationまたはVeeam ONEがすでに実行されている同じマシンに製品をインストールすることはできません。



コンポーネントの相互作用図を次の図に示します。







すべての設定は、VAOサーバーのWebコンソールから行われます。 デフォルトのプロトコルはHTTPSおよびポート9898です。



ロールと通知を設定した後、管理者はVAOサーバーで次の手順を実行する必要があります。



  1. VMwareおよびVeeam Backup＆Replicationサーバーを追加します（インストール中に追加されない場合）。
  2. vCenterタグを使用するなど、切り替え計画が実行されるVMを分類およびグループ化します。 分類の詳細については、 ここで説明します （英語）。
  3. 結果のVMグループをアクティブ化します。
  4. 各サイトについて、計画に含める必要があるステップを示します。
  5. テスト「サンドボックス」（およびVeeam Backup＆Replicationで動作する使い慣れた「仮想ラボ」）を使用する予定の場合は、VAOコンソールでもアクティブにする必要があります。
  6. また、バックアップサイトにあるVeeam Backup＆Replicationサーバーでは、バックアップタスクを構成して、切り替え中に使用されるレプリカを保護できます。 このようなタスクの場合、説明にはテキスト[VAO TEMPLATE]が含まれている必要があり、タスク自体にはVMが含まれていてはなりません（空のフォルダーを作成し、[ 仮想マシン]ステップで追加できます）。


その後、スイッチプランを作成し、それらに仮想マシングループを追加できます。







作成された計画は、自動的にチェックされ、スケジュールされたスイッチのスケジュールを設定し、実行履歴を表示することができます。 さらに、ダッシュボードとカスタムレポートを使用して、すべての計画のステータスを監視できます。







ライセンスと費用について



Veeam Availability Orchestratorライセンスは、1年間のサブスクリプションとして、または2〜5年間の割引価格で購入できます。



ライセンスは、ディザスタリカバリプランを必要とするレプリケートされたVMにのみ必要であり（最小注文数量は10 VM）、Veeam Availability SuiteまたはVeeam Backup＆Replicationで保護されているすべてのVMには必要ありません。
