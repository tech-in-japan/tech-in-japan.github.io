Hellopad for Androidを作成し、メモ帳で起動します。 Java 8およびAndroid N







2年半前に、メモ帳でAndroid用HelloWorldを作成、収集、起動する記事を公開しました。 彼女は非常に人気があり、約80,000回の視聴を獲得しました。 Jack ToolChainなどの新しいツールの出現により、記事を再発行および更新する必要が生じました。



Androidの学習を始めたとき、IDEを使用せずに、Androidアプリケーションを完全に手動で作成およびコンパイルしたかったのです。 しかし、この作業は簡単ではなく、かなり時間がかかりました。 しかし、結局のところ、このアプローチは非常に有用であり、IDEを隠す多くの微妙な点を明確にしました。



メモ帳のみを使用して、非常に小さな教育用Androidアプリケーションを作成します。 そして、それをコンパイルし、アセンブルして、デバイス上で実行します。これはすべてコマンドラインを使用して行います。 に興味がありますか？ お願いします。



エントリー



テンプレートアプリケーションがAndroid Studioでどれほど複雑で混乱しているかに驚かされました。 単にリソースが山積みされています。 そして、それほどではありませんが、コードとスクリプトを使用します。 やらなければならないことは、HelloWorldを表示することだけです！ さらに、私が調べた本とマニュアルは、ダイアログボックスを使用してIDEAまたはEclipse HelloWorldを作成する方法を説明します-そして、さらにナレーションがそこから出ています。 そして、「フードの下」で何が起こるか-推測しかできません。



教育目的に理想的な独自のテンプレートプロジェクトを作成します。 余分なものはなく、最も必要なものだけがあります。 そして、それをどのように組み立ててAndroidデバイスで実行するかを詳細に分析します。 記事の最後に、最終プロジェクトとともにアーカイブをダウンロードするためのリンクがあります-質問がある場合は、それを確認できます。



したがって、プロジェクトの構成とアセンブリのプロセスを100％認識し、理解することになります。 このテストプロジェクトは、少し改良されたトレーニングを目的としていますが、実際のプロジェクトの強固な基盤として使用できます。



準備する



開始するには、コマンドラインツールをダウンロードしてインストールする必要があります。 それらをダウンロードするためのリンクは、Android Studioページ（ https://developer.android.com/studio/index.html ）の下部にあります 。









Android SDK 24はAndroid N （ N ougat / 7）にすぎません。 条件を受け入れ、インストーラーをダウンロードして実行します。 デフォルトのままにします。 C：\ Users \ kciray \ AppData \ Local \ Android \ android-sdkという形式のディレクトリにインストールされます。 この方法を覚えておいてください、私たちの主なツールはそこにあります。



次に、（ android-sdkフォルダーから）SDKマネージャーを起動し、デフォルトセットを設定します。 新しいJackコンパイラなど、必要なものはすべて揃っています。 JDK 8も必要です。



この記事を読む前の主な要件は、インストールされているソフトウェアに加えて、EclipseまたはAndroid Studioに付属のHelloworldを既にデバイスで実行できることです。 つまり usbドライバーを構成し、デバイスでusbデバッグを有効にする必要があります...または、エミュレーターを作成して構成します。 これらは非常に基本的なものであり、その検討はこの記事の範囲を超えています-ネットワークに関する十分な情報があります。 ところで、本からいくつかの章を読むことも不必要ではありません-マニフェスト、リソース、そして実際にはJava言語の基本がどのように配置されているかを理解してください。 そして、この記事では、どの本が沈黙しているかについて説明します。



プロジェクト執筆



開始するには、プロジェクトのあるフォルダーを作成します。 testappと呼びましょう 。 その中に、さらに3つのフォルダーを作成します-bin 、 res 、 src 。



testappで空のテキストファイルを作成し、その名前をAndroidManifest.xmlに変更します。



以下を追加します。 



<manifest xmlns:android="http://schemas.android.com/apk/res/android" package="com.example.testapp"> <uses-sdk android:targetSdkVersion="24" /> <application android:label="TestApp"> <activity android:name=".MainActivity"> <intent-filter> <action android:name="android.intent.action.MAIN" /> <category android:name="android.intent.category.LAUNCHER" /> </intent-filter> </activity> </application> </manifest>


ここではすべてが簡単です。 TestAppというアプリケーションを作成し、起動時にMainActivityクラスを開始します。 この小さなクラスを記述するだけで、アプリケーションの準備は完了です。 必要に応じて、 uses-sdk タグのandroid：targetSdkVersionプロパティを編集し、 使用しているバージョンを入力します。



次に、最も単純なリソース（文字列Hello test app）を作成します。 実際、この行をJavaコードに直接挿入することにより、リソースなしで実行できます。 ただし、いくつかのアセンブリ手順はリソースで機能します。興味深い点を見るために、引き続きそれらを使用します。



resフォルダーにvaluesフォルダーを作成しましょう。 すべてのリソースはフォルダーに分割する必要があります。 次に-空のstrings.xmlファイルを作成し、その中に書きます： 



 <resources> <string name="hello">Hello test app!</string> </resources>


それが私たちが必要とするすべてのリソースです。 シンプルでしょ？ 次に、 src内にcomフォルダーを作成し、その中にサンプルフォルダーを作成し、さらに階層の下位にあるtestappフォルダーを作成します。そして、 MainActivity.javaクラスがあります。 そこにコードを追加します： 



 package com.example.testapp; import android.app.Activity; import android.app.AlertDialog; import android.os.Bundle; import android.widget.Button; import android.widget.Toast; public class MainActivity extends Activity { @Override public void onCreate(Bundle savedInstanceState) { super.onCreate(savedInstanceState); Button button = new Button(this); button.setText("Big button"); button.setOnClickListener(v -> { new AlertDialog.Builder(MainActivity.this) .setTitle("From lambda") .setMessage(getString(R.string.hello)) .show(); }); setContentView(button); } }


これは最もシンプルなアクティビティで、全画面に1つのボタンが含まれます。 このボタンをクリックすると、リソースの行を示すダイアログボックスが表示されます。 ラムダに注意してください（構築v-> {...} ）。 Jack ToolChainを使用すると、AndroidでJava 8の多くの機能を使用できます。 developer.android.comおよびsource.android.comで詳細を読むことができます。

コンパイル準備



そもそも、不思議なクラスRがどのように機能するのか疑問に思ったことはありますか？ 実際、彼は最初、彼の超能力のせいで私を当惑させました。 コンパイル段階で、クラスのフィールドを介して他のディレクトリのXMLファイルにアクセスするにはどうすればよいですか？ プリコンパイラがここで動作していると仮定しました-判明したように。



実際には、特別なユーティリティAAPTがあります-リソースのディレクトリを調べて 、同じR.javaを作成します。 すべてが非常にシンプルであることがわかりました-それは単なるクラスであり、整数定数を持つ他の静的なネストされたクラスで構成されています。 それだけです！ 次のようになります。



R.java 
 /* AUTO-GENERATED FILE. DO NOT MODIFY. * * This class was automatically generated by the * aapt tool from the resource data it found. It * should not be modified by hand. */ package com.example.testapp; public final class R { public static final class attr { } public static final class string { public static final int hello=0x7f020000; } }


それではあなたと一緒に作成しましょう。 これを行うには、次のコマンドを使用します。 



 call %AAPT_PATH% package -f -m -S %DEV_HOME%\res -J %DEV_HOME%\src -M %DEV_HOME%\AndroidManifest.xml -I %ANDROID_JAR%


何が何であるかを理解しましょう。 AAPT-Android Asset Packaging Tool-文字通り「Androidプロパティのパッカー」。 そのオプション：



  • パッケージ -リソースをパックする必要があると言います （追加も削除もしません）
  • -f-既存のR.javaを上書きします（存在する場合）
  • -m-R.javaを-Jで指定されたパスのルートではなく、適切なパッケージに配置します
  • -S-このオプションの後、リソースを含むディレクトリを指定します
  • -J-このオプションの後に、結果のR.javaを保存する場所を指定します
  • -I-このオプションの後、接続されたライブラリへのパスを示します-android.jarをオンにします


実行後、同じR.javaファイルがsrcディレクトリに表示されます。 ご覧ください。



プロジェクトに魔法はありませんが、Java内では完全に構文的に正しいです。 そして今、楽しい部分です。 javacを使用して古典的なJavaプログラムをコンパイルする方法を覚えていますか？ 以前は、Androidアプリケーションのビルドシーケンスの一部でもありました。 ソース（* .java）を取得し、そこからJVMバイトコード（* .class）を受け取り、それからのみDalvicのバイトコード（* .dex）を受け取りました 。 Jack ToolChainの登場により、ビルドシーケンスが1ステップ削減されました。 ソース（* .java）から、すぐにDalvic （* .dex）のバイトコードを取得します。



ジャックはどこで入手できますか？ これは、 jack.jarの形式でbuild-toolsフォルダーにあり、通常のJavaアーカイブとして実行されます。 



 %JAVAVM% -jar %JACK_JAR% --output-dex "%DEV_HOME%\bin" -cp %ANDROID_JAR% -D jack.java.source.version=1.8 "%DEV_HOME%\src\com\example\testapp\R.java" "%DEV_HOME%\src\com\example\testapp\MainActivity.java"


引数は次のとおりです。



  • -jar -JVMの標準オプション。Javaアーカイブを実行することを示します。 ジャックとは関係ありません
  • --output-dex-結果のdexファイルを配置するフォルダー。 ビンに入れてください
  • -D jack.java.source.version = 1.8- 「D」は、プロパティを設定していることを示します。 jack.java.source.versionにより、Java 8を使用していることを示すことができます。これがないと、ラムダは機能せず、エラーが発生します。 コマンド％JAVAVM％-jar％JACK_JAR％--help-propertiesを使用して、プロパティの完全なリストを表示できます。
  • [* .javaファイルのリスト] -ソース。 R.javaとMainActivity.javaの 2つのファイルしかない


コマンド％JAVAVM％-jar％JACK_JAR％--helpを使用して、Jackのオプションの完全なリストを表示できます。



classes.dexがbinフォルダーにあることを確認します。 これで、リソースと一緒にAPKファイルにパックするだけです。 それをやってみましょう： 



 call %AAPT_PATH% package -f -M %DEV_HOME%/AndroidManifest.xml -S %DEV_HOME%/res -I %ANDROID_JAR% -F %DEV_HOME%/bin/AndroidTest.unsigned.apk %DEV_HOME%/bin


ここで、オプションはR.javaを作成するときに使用したものと似ています。



  • パッケージ -リソースをパックする必要があると言います （追加も削除もしません）
  • -f-既存のAndroidTest.unsigned.apkを上書きします（存在する場合）
  • -M-このオプションの後、マニフェストファイルへのパスを指定します
  • -S-このオプションの後、リソースを含むディレクトリを指定します
  • -I-このオプションの後、接続されたライブラリへのパスを示します-android.jarをオンにします
  • -F-このオプションの後に、結果のAndroidTest.unsigned.apkを保存する場所を示します
  • 最後の引数は、dexファイルがあるフォルダーへのパスです


AndroidTest.unsigned.apkがbinフォルダーに表示されます。 そして、理由のためにそれを呼び出しました！ 彼にはデジタル署名がありません。 Androidは、署名のないアプリケーションのインストールと実行を禁止しています。 しかし、それを作成することは一見すると思えるほど難しくありません 



 call %JAVA_HOME%/bin/keytool -genkey -validity 10000 -dname "CN=AndroidDebug, O=Android, C=US" -keystore %DEV_HOME%/AndroidTest.keystore -storepass android -keypass android -alias androiddebugkey -keyalg RSA -v -keysize 2048 call %JAVA_HOME%/bin/jarsigner -sigalg SHA1withRSA -digestalg SHA1 -keystore %DEV_HOME%/AndroidTest.keystore -storepass android -keypass android -signedjar %DEV_HOME%/bin/AndroidTest.signed.apk %DEV_HOME%/bin/AndroidTest.unsigned.apk androiddebugkey


実際、これらの行はAndroid SDKとは関係のない2つのJavaユーティリティを実行しますが、これらは必須です。 最初はAndroidTest.keystoreファイルを作成し（存在を確認）、2番目はこのファイルをAndroidTest.unsigned.apkに接続します。 AndroidTest.signed.apkファイルが判明します。 これがそのようなワイルドクラフトファイルです。 ただし、batスクリプトを作成したら、それを実行します。自動モードでこれをすべて実行します。



この記事では、これらのユーティリティのオプションの詳細な分析に時間を費やすべきではないと思います。 ポイントをキャッチするだけです-AndroidTest.unsigned.apkを取得し、 AndroidTest.keystoreファイルで署名して、 AndroidTest.signed.apkに保存します。 必要に応じて、 ドキュメントを読むことができます 。



「 警告：-tsaまたは-tsacertが提供されておらず、 このjar ... 」という警告が表示される可能性が高くなりますが、注意は必要ありません 。



打ち上げ



apkファイルがようやくコンパイルされたので、実行できます。 USB経由でデバイスを接続するか、エミュレーターを実行します。 それから 



 call %ADB% uninstall %PACKAGE% call %ADB% install %DEV_HOME%/bin/AndroidTest.signed.apk call %ADB% shell am start %PACKAGE%/%PACKAGE%.%MAIN_CLASS%


実際、プログラムが既に存在する場合、最初の行はプログラムを削除します。 繰り返し起動する場合に便利です。 2番目-デバイスまたはエミュレーターにAPKをインストールします。 3番目のものが起動します。 彼女の議論を詳しく見てみましょう。

  • シェル -デバイスでいくつかのコマンドを実行したい
  • am-アクティビティマネージャーコマンドの実行に使用
  • start-アクティビティを開始したい
  • 開始するパッケージの名前とクラスの完全な名前（パッケージを含む）


注意-インストール中に、確認ダイアログボックスがデバイスに表示される場合があります。 時間内に承認しない場合、インストールは失敗します[INSTALL_FAILED_USER_RESTRICTED] 。 また、質問があるかもしれません-なぜinstall -rの代わりにuninstall / installを実行してください 。 実験の純度のためにそうしました。 このスクリプトは、アクティビティのすべての製品を完全に削除し、開始するたびにゼロから作成します。 鍵さえ。 install -rを使用できますが、キーの再作成を担当するコードを削除する必要があります。 そうしないと、エラー[INSTALL_FAILED_UPDATE_INCOMPATIBLE]が発生します。
