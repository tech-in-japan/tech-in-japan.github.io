LinuxでIntertelecom用にZTE AD3700 CDMAモデムを構成する

そのようなモデムのセットアップを検討してください。 パスポートによると、ZTE AD3700と呼ばれています。

















Intertelecomプロバイダーは、一時的にこのような廃止されたモデムのKAMAZ全体を購入したようです。 同様のCDMAモデムは、PeopleNetなどの他のプロバイダーによって使用されます。 はい、実際には、Qualcomm gobiチップセットに基づく多くのデバイスで、このガイドが機能するはずです。









ここにこのアーカイブが必要です：







  • 鏡1


モデムのファームウェア、2つのバージョンが含まれています。 適切なファームウェアのバージョンは、モデムのリビジョンによって異なります。 しかし、後でこれに戻ります。









19/10/19のユーザーpalexxxによる更新：





実際には、モデムはデュアル標準です。 しかし、保存の観点から、ベンダーはメモリを保存し、モデムがDLポートとして識別され、ファームウェアが投入されるたびに保存します。 ファームウェアに応じて-機能が異なります。 この記事には2つのオプションがありますが、これは「異なるデバイスリビジョン」用ではありません。 そして、あなたの1つはCDMAオプションであり、2つ目のオプションは奇妙ですが、GSMではありません。 wvdial設定では、ダイヤラー番号を変更するだけで、ZTE AD3700の2つのファームウェアオプションへのリンクdrive.google.com/file/d/1-fMmKFJZjMmWoivPfqi3ph7PTg1pIe3S/view?usp=sharing





まず、wvdialをインストールする必要があります。 













apt-get install wvdial




次に、ルートの下のエディターで、ファイル「/etc/wvdial.conf」を開き、Intertelecomのコンテンツを開きます。 









 [Dialer Defaults] Init1 = ATZ Modem Type = USB Modem ISDN = 0 New PPPD = yes Stupid Mode = 1 Init2 = ATQ0 V1 E1 S0=0 &C1 &D2 +FCLASS=0 Modem Type = Analog Modem Modem = /dev/ttyUSB0 Baud = 460800 Phone = #777 Username = IT Password = IT


/ dev / ttyUSB0の代わりに、カーネルがモデムに発行したデバイスの名前を書きます。 ログインとパスワード「IT」はすべてのユーザーで同じです。 電話も。 別のプロバイダーがある場合は、設定を自分のものに変更します。 次に、udevルールを追加して、モデムの電源を入れたときに正しいファームウェアがロードされるようにする必要があります。 これを行うには、上記のアーカイブから「udev」フォルダーに移動し、ルートの下からコマンドを入力する必要があります。 













 make install




ここで、すべてがエラーなく機能した場合、「/ lib / firmware / gobi」という名前のディレクトリを作成します。 アーカイブには「variant1」と「variant2」の2つのディレクトリがあります。これらは、モデムごとに異なるバージョンのファームウェアです。 モデムに適したものを見つけるには、次が必要です。









  1. 「variant1」ディレクトリから「/ lib / firmware / gobi」ディレクトリにすべてのファイルをコピーします。
  2. モデムをUSBポートに差し込みます。 既にスタックしている場合は、もう一度引っ張ってスティックします。
  3. 彼がコマンドに応答するまで約10秒待ちます。
  4. 「/etc/wvdial.conf」で指定したデバイス名でモデムが決定したことを確認してください。そうでない場合は、構成を修正してください。
  5. ルート「wvdial」の下から実行します。


ファームウェアが正しく選択されていない場合、しばらくするとwvdialがクラッシュし、「モデムが応答していません」というメッセージが表示されます。 この場合、5つのポイントをすべて繰り返し、「variant2」ディレクトリから「/ lib / firmware / gobi」にファイルをコピーするだけです。









動作中のファームウェアの場合、「wvdial」はコンソールで受信したDNSサーバーのアドレスを吐き出し、pppdという単語もそこに表示されます。 理論的には、あなたは今インターネットを持っています。









この接続でnetwork-managerを正しく動作させることができませんでした。 また、「Ctrl-C」を使用して接続を終了すると、モデムをUSBポートに接続しないと再接続できなくなります。 これはまた、中断するたびに「/etc/wvdial.conf」ファイルを編集して、/ dev / ttyUSB0の代わりに正しいモデムファイルが表示されるようにする必要があることを意味します。









時々、「wvdial」がさまざまな間隔であらゆる種類のナンセンスを書き始めることがあります-これはモデムがフリーズしていることを意味し、それを反転する必要があります。 貼り付けた後、毎回10-15秒待つことを忘れないでください。 何かをダウンロードすると、モデムが非常に熱くなることがありますが、パフォーマンスには影響しません。
