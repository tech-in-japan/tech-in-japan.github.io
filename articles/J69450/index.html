アルバム： Wiki-Os



Wiki-OSに関する最近の情報に感銘を受け、それをテストしたかった。

動揺は、Chromeで起動しないことです。



IEとFFでは、すべて問題ありません。 フォーラムでWiki-OSをChromeに接続する方法（en）の推奨事項を見つけました-ここでは、Chromeを適応させる方法を書いています（ アーカイブをダウンロードしてCに解凍します ：\ Documents and Settings \ Username \ Local Settings \ Application Data \ Google \ Chrome \ ApplicationディレクトリXPの場合）。



Vistaでは、C：\ Users \ Valery \ AppData \ Local \ Google \ Chrome \ Application \に解凍できました。



重要！

アーカイブには1つの問題があります-plds4.dllファイルがありません（開始ページwww.wiki-os.orgをロードすると、Chromeにこのメッセージが表示されます）。



C：\ Program Files \ Mozilla Firefoxで見つけ、Cの他のライブラリの隣にコピーしました：\ Users \ Valery \ AppData \ Local \ Google \ Chrome \ Application \

すべてがすぐに機能しました。



比較感覚-Chromeは誰よりも速く動作し、フルスクリーン（F11による）では、単純化されたVS C＃とVBAインターフェースの混合物として環境を完全に認識します。

一般的に、印象はまだ良いです。 これはおそらく、WinAPIを長年踏み込んだ後の最高のMSソリューションの1つです。



将来、他のクライアント（Linux、MacOS）との相互作用が可能になることを願っています。
