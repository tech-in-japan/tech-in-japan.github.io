Veeam Availability Suite 9.5のリリース-新機能

Veeam Availability Suite 9.5の新しいバージョンがリリースされました-障害発生後のデータのバックアップと復元のための完全なソリューション。 本日、このバージョンのVeeam Backup＆Replicationの新機能の概要をご紹介します。



  • Windows Server 2016、Hyper-V 2016、およびこのプラットフォームでの最新のサーバーアプリケーションのサポート
  • Veeam Backup＆Replicationパフォーマンスの改善
  • データバックアップの改善
  • データ復旧の改善
  • 磁気テープの作業分野の改善
  • ストレージの機能強化
  • クラウドインフラストラクチャの機能強化
  • アプリケーションサポートの新機能


これらの新製品をより詳細に検討してください。catを歓迎します。









Windows Server 2016、Hyper-V 2016、およびこのプラットフォームでの最新のサーバーアプリケーションのサポート



もちろん、これはWindows Server 2016の包括的なサポートです 。VeeamBackup＆ReplicationソリューションのすべてのコンポーネントはWindows Server 2016で実行されます。 既存のアプリケーションを考慮して、ファイルシステムのインデックス作成とVM処理を使用して、このゲストOSから仮想マシンをバックアップできます。 同時に、Hyper-V 2016プラットフォーム上の仮想マシンでは、ネットワーク接続（ネットワーク接続）を必要とせずに、そのような処理とインデックス作成、およびファイルレベルの回復（FLR）をゲストOSへの直接アクセスによって実行できます。 PowerShell Directがこれに使用されます。 したがって、Hyper-VプラットフォームでVMを操作するには、VMwareほど効果的な方法はありません。

また、ブロックの高速クローン作成のためのReFSファイルシステムとテクノロジーもサポートされています（詳細については、Habréのこの投稿を参照してください）。







Microsoft Hyper-V 2016のサポート-Veeamは、このプラットフォームの新しいバージョン（VMハードウェア7.1を含む）を完全にサポートします。 これまで、バックアップは、変更されたブロックを追跡するための「ネイティブ」機能なしで存続する必要がありました。その役割は、特別に開発されたVeeamドライバーによって果たされました。 そして最後に、Microsoftが実装したMicrosoft Resilient Change Tracking（RCT）がこの重要な問題に取り組んでいます。これは朗報です。 特に、サードパーティのSMB3リポジトリにある仮想マシンを処理できるようになりました。 Nano-ServerをHyper-Vホストとして使用している場合、Veeam Backup＆Replication 9.5はそのような仮想マシンをバックアップできることに注意してください。 新しいバージョンは、記憶域スペースダイレクトもサポートしています。これは、Hyper-V 2016の展開に推奨されるアーキテクチャです。



人気のあるサーバーアプリケーションの最新バージョンのサポート -当然、Veeam Backup＆Replication 9.5。 2016年のバージョンを更新した、広く使用されているすべてのサーバーアプリケーションをバックアップします。これらは、Microsoft Active Directory、Exchange、SharePoint、およびSQL Serverです。 これらのアプリケーションのオブジェクトのきめ細かい復元は、通常どおり、Veeam Explorerのツールによって実行されます。



重要！ これらの機能はすべて、無料のVeeam Backup Free Editionを含む、 Veeam Backup＆Replicationのすべてのエディションで使用できます。





Veeam Backup＆Replicationパフォーマンスの改善





このリリースでは、バックアップとリカバリ中にリソースを使用する生産性と効率を改善することに多くの注意が払われており、その結果があなたを楽しませてくれることを願っています。 これは、多くの革新のおかげで可能になりました。

  • 構成ベースの最適化 -結果として、クエリははるかに高速に処理されますが、SQLサーバー自体の負荷は軽減されます。

    重要！ Veeam Backup＆Replication 9.5以降、構成データベースにはSQL Server 2008以降が必要です。 この目的でのSQL Server 2005の使用はサポートされません。
  • バックアップメタデータの処理の高速化 -多数のサーバーアプリケーションがインストールされた仮想マシン、およびアプリケーションの非常に技術的な性質（Microsoft SharePointなど）に基づいて、かなりの量のメタデータと共に保存する必要があるサーバーアプリケーションでのタスクの処理をより効率的に実行できます。
  • UIの最適化 -構成ベースの負荷が削減されるだけでなく、特に、バックアップコピーのリストを表示するときのUI応答時間、および詳細な復元中のアプリケーションオブジェクトの階層も削減されます。

    さらに、新しいバージョンでは、プログラム設定メニューで[ 配色]項目を選択することにより、さまざまな配色を使用できます。 設計例の1つは、投稿の冒頭の写真に示されています。
  • 優先プロキシサーバーの目的 -リポジトリ設定で、使用するプロキシサーバーの指定（データの書き込みと読み取り）を指定できるようになりました。 これは、たとえば、異なるブランチでリポジトリを使用するシナリオに便利です。リストにはローカルリポジトリのみを含めることができ、リモートリポジトリは使用できません。 このオプションは、スケーラブルなSOBRリポジトリ（スケールアウトバックアップリポジトリ）の場合にも適用できます。複数の標準リポジトリを単一のバックアップストレージ構造に結合します。この場合、このリストは標準リポジトリごとに設定されます。 優先サーバーが使用できない場合、プロキシの選択はジョブ設定に従って行われます。

    注：この機能は、EnterpriseおよびEnterprise Plusエディションでサポートされています。










データをバックアップするには



  • リソースを大量に消費する読み取り/書き込み操作の高速化-ハイエンドストレージシステムを使用すると、仮想ディスクのバックアッププロセスが最大2倍に加速され、バックアップタスクを実行する際の読み取り/書き込み操作の数が減少するため、運用ストレージシステムの負荷も減少します。 この機能は、次の操作でVMwareプラットフォームでサポートされます。ハードウェアスナップショットからのバックアップ、仮想アプライアンスでのバックアップ（ホットアド）、Direct NFSモード。
  • vSphereインフラストラクチャデータキャッシング-vSphereインフラストラクチャトポロジ情報がRAMに保存されるようになりました。 vCenterサーバーからインフラストラクチャ変更イベントを受信すると、リアルタイムで更新されます。 新しいバックアップサーバーコンポーネントであるVeeam Broker Serviceは、インフラストラクチャデータの収集と保存を行います。 その結果、2つの利点があります。バックアップジョブを開始するときとUIで仮想インフラストラクチャのビューを表示するときのVMリストの作成が加速され、vCenter Serverから負荷の一部が削除されます。
  • トポロジの構築およびインフラストラクチャデータの更新のためのvSphereへのクエリの最適化 -クエリの実行速度が大幅に高速化され、必要に応じてVMwareトポロジ全体を構築します（たとえば、Veeam Backupサーバーの再起動時またはVeeam Broker Serviceの再起動時）、vCenter Serverの負荷は最小です。




データ復旧用



  • インスタントリカバリ物理マシンのインスタントリカバリ -Veeam Backup＆Replication 9.5を使用すると、仮想サーバーだけでなく、物理サーバーまたはユーザーコンピューターもHyper-Vプラットフォームに基づいた仮想マシンに復元できます。 これにより、たとえば、修復作業の進行中に、故障した物理マシンをバックアップから迅速に「ピックアップ」できます。 この機能は、無料のFree Editionを含む Veeam Backup＆Replicationのすべてのエディションに含まれており、Veeam Endpoint Backup 1.5以降で作成された物理マシンのバックアップでサポートされています。 Veeam Agent for Windows 2.0によって作成されたバックアップの最適なパフォーマンスが期待されます（年末にリリースが予定されています）。

    重要！ Microsoft Hyper-Vが必要です（バックアップサーバーでこの役割を有効にできます）。
  • VM全体のリカバリ中のディスクの並列処理 -ディスクストレージシステムに保存されているすべてのリポジトリでデフォルトで有効になっています（重複排除機能が組み込まれたData Domainストレージシステムを除く）。
  • インスタントVMリカバリによる生産性の向上 -シナリオに応じて、最大3倍に成長できます。「VMごと」チェーンからリカバリするときに最高のパフォーマンスが得られます（バックアップファイルがVMによってグループ化される場合）。




磁気テープを使用するには



  • テープからインフラストラクチャに直接復元する -新しいバージョンでは、バックアップテープからのバックアップは以前は中間であったため、このために補助リポジトリは必要ないため、リポジトリを選択するときに容量を追加する必要はありません。 もちろん、オプションも可能です-たとえば、1つのバックアップから多数のVMを復元する必要がある場合。 この場合、補助リポジトリを使用することは依然として合理的です。

    新しいアプローチでは、プロセス中にバックアップへのいくつかの呼び出しが実行されたと簡単に言えます。

    1. 最初に、バックアップメタデータが読み取られ、テープサーバーに保存されます。
    2. 次に、VMデータブロックのマップ（構成ファイルとディスクデータを含む）が構築され、バックアップへの次の呼び出し中に構成データが最初に復元され、次にVMディスクが復元されます。








  • File to Tapeタスク（テープへのファイルのアーカイブ）の生産性の大幅な向上 -多くの小さなファイルをアーカイブすると、生産性が以前のバージョンに比べて数十倍になることがテストで示されています。 最大値が興味深い場合、テスト中のファイルの数は1つのタスクで最大2000万に達したと言えます。
  • バックアップジョブの開始の遅延 -アーカイブジョブが対応するバックアップの処理をまだ完了していない場合、バックアップジョブの開始の遅延を構成できます。 （以前は、この場合、バックアップタスクが優先され、ファイルがテープに転送される前にバックアップタスクが中断されていました。）
  • 筆記用具の自動クリーニング - 筆記用具の状態は私たちの監督下にあり、クリーニングの時間が来るとすぐに、Veeamはもちろんクリーニングカートリッジでこれを行います。






ストレージの新機能



  • ニンブルストレージのサポート -Veeamのハードウェアスナップショット機能はすべて、ニンブルAFシリーズまたはCSシリーズストレージシステムを使用しているユーザーが利用できるようになりました。







    従来のサポート：

    -ハードウェアイメージ（プライマリストレージ）と複製コピー（セカンダリストレージ）のバックアップ

    -ハードウェアスナップショットから回復するためのVeeam Explorerツール

    -ハードウェアイメージに基づくテスト環境オンデマンドサンドボックスの作成

  • メインストレージに接続するためのプロトコルの選択 -ストレージが複数のプロトコル（FC / iSCSI / NFS）をサポートしている場合、バックアップに使用できるプロトコルを選択できます。
  • Data Domainのバックアップとリカバリの高速化 -Data Domainストレージでの新しいバックアップ（v9.5で作成）の最高のパフォーマンスインジケーターを実現するために、データ処理メカニズムが更新されました。
  • NetApp Data ONTAP 9.0のサポート、および大規模クラスターを使用する場合の複数のLIF（論理インターフェイス）間の自動負荷分散。




クラウドインフラストラクチャを使用する場合



Microsoft Azureでの回復



Veeamの新しいバージョンでは、Microsoft AzureでWindowsまたはLinuxを実行しているマシンを回復できます。 バックアップがVeeamバックアップリポジトリに保存されている任意のマシン、つまり、Veeam Backup＆ReplicationまたはVeeam Backup FREEによってバックアップされた仮想マシン、またはVeeam Endpoint Backup FREEによってバックアップされた物理マシンを復元できます。

新しい機能は、Azure Marketplaceから予備のマシンを事前にデプロイする必要がないという点で便利ですが、Veeamインターフェイスからクラウドインフラストラクチャに直接、仲介なしで復元できます。

この方法は、マシンの移行と、たとえばクラウドインフラストラクチャに基づいたテスト環境全体の作成の両方に使用できます。

年の前半にリリースされたテクノロジープレビューに関して、次の新機能が登場しました。

  • ディスクの並列リカバリ -VMに複数のディスクがある場合、プロセスが大幅に高速化されます。
  • クラウドゲートウェイ （オプションコンポーネント）-VPNを使用せずに、リカバリ中のクラウドへのデータ転送を高速化できます。 補助VMは、Veeamコンソールからクラウドに直接展開し、使用後に自動的に切断して、多額の支払いを行わないようにすることができます。
  • Windows VM用のAzure Virtual Machine Agentの自動インストール、リモートマシン用のWindowsファイアウォールルールのセットアップ、およびクラシック（クラシック）およびAzureリソースマネージャーによるさまざまな展開モデルのサポート。









この機能は、Veeam Backup Free Editionを含むすべてのエディションでサポートされています。



Veeam Cloud Connectの新しい機能



  • ユーザー側での並列データ処理 -複数のVM（または同じマシンの複数のディスク）のバックアップとレプリケーションが並列に実行されるようになり、そのようなタスクの数はプロバイダー側​​で設定された最大値によって制限されます。

    注：この機能は、直接転送モードを使用する場合に使用されますが、WANアクセラレーターを使用している場合、処理は連続して続行されます。
  • クラウドリポジトリのバックアップからのレプリケーション -ユーザーはクラウドリポジトリに保存されたバックアップからレプリケートできます。
  • プロバイダー側​​でのスケーラブルなリポジトリのサポート -クラウドリポジトリを単一のスケーラブルなリポジトリ構造に結合できるようになり、管理が大幅に簡素化されます。 新しいタイプのリポジトリに切り替える手順については、後ほど詳しく説明しますVeeam Cloud＆Service Providers Forumのニュースをフォローしてください。
  • VMでグループ化されたバックアップチェーンのストレージ -クラウドリポジトリを操作するときに一般的なストレージオプションが利用できるようになりました。これは、組み込みの重複排除を備えたストレージシステムを使用する場合に特に便利です。


サービスプロバイダー向けのVMware vCloud Directorサポートの強化



VMware vCloudインフラストラクチャからの仮想マシンのバックアップと復元はバージョン8.0に実装され、VMware vCloud Directorのサポートは新しいレベルに達し、プライベートクラウドとハイブリッドクラウドのプロバイダーとユーザーに高度な機能を提供します。

  • 「ネイティブ」のvCloud認証メカニズムを使用すると、vCloud Directorアカウントを持つユーザーがセルフサービスポータルに接続できるようになり、組織に含まれるVMを個別にバックアップおよび復元できます。
  • セルフサービスポータルを介したバックアップ -ユーザーは、最小限の構成を必要とする組み込みテンプレートを使用して、バックアップタスクを作成および編集する便利な方法を提供されます。 ユーザーは、ゲストOSへのアクセス用のアカウント、およびバックアップの保存と通知のセットアップのポリシーを設定するだけで、プロバイダーは残りをその側でセットアップします。
  • セルフサービスポータルを介した復旧 -ユーザーは、Veeam Backup Enterprise Managerでサポートされている復旧操作を許可できます -これには、VM、vApp、ゲストOSファイル、およびアプリケーションオブジェクト（現在はSQL ServerとOracleデータベース）の復元が含まれます。

    重要！ VMware vCloudインフラストラクチャユーザー向けのセルフサービスポータルは、Enterprise Plusで利用できます。




サーバーアプリケーションサポートの新機能



Windowsプラットフォームでの2016年のサーバーアプリケーションの操作を考慮したVMバックアップのサポートに加えて、SUSE Linux Enterprise Server（SLES）プラットフォーム11および12のOracleサーバーに対して同様のサポートが実装されるようになりました。 VMのバックアップとリカバリのみ。VeeamExplorerを使用したきめ細かいリカバリは不要です。
