「Habr」の候補の新しい検索の仕組み



「Habr」の検索では、彼らが住んでいる都市の候補者を検索できます。 彼らが働いている会社によって ; 同様に、そして何よりも、それらが関連付けられているハブ 。







「マイサークル」の「Habr」のユーザーの検索ページ：ハブ「Java」と「Development for Android」を同時に検索します。



複数のハブを同時に指定する場合、これは、探している候補者が指定されたすべてのハブに関連付けられている必要があることを意味します。 ハブとの通信には2つのタイプがあります。

1.ユーザーがハブに貢献している

2.ユーザーはハブを購読しています



ユーザーがハブに貢献しているのは、このハブでパブリケーションを作成し、これらのパブリケーションが他のユーザーから肯定的な評価を受けている場合のみです。



各ハブのページの「Habré」には、このハブに最も貢献した100人の著者が表示されます。 たとえば、 Perlの著者トップ100は次のとおりです。 また、各ユーザーのプロファイルには、最大10のハブが表示され、そこで最も貢献しました。







Habréのプロファイルにユーザーの貢献があるダイの例：それぞれが独自のハブのセットを持っています。



1.「マイサークル」で検索する場合、ハブの作成者の数に制限がないように、ユーザーの投稿があるハブの数に制限はありません。投稿があるすべてのユーザー投稿に対して検索が行われます。



2.検索結果で「PHP」などの 1つのハブを検索すると、最初にこのハブに投稿した人が投稿の降順で表示され、次にこのハブにのみ登録した人が表示されます。



3.複数のハブ（ たとえば、「PHP + Laravel」）で検索すると、ユーザーは、貢献したハブまたは購読したハブの中から目的のハブを見つけることができます。 検索結果は、両方のハブの合計貢献度が最も高い人に最初に表示されます。 これは、1つのハブにのみ貢献した人が、すべてに貢献した人よりも高い可能性があることを意味します。 これは、この1つのハブに対する最初のハブの合計貢献度が、すべてに対する2番目のハブの合計貢献度よりも高くなるためです。



4.「コントリビューションがあるハブで検索」チェックボックスを有効にすると、目的のハブへのコントリビューションを持つユーザーのみに検索が制限されます。 たとえば、「PHP + Laravel + AngulatJS（入力のみ） 」。



5.ハブの前にある星印は、ユーザーがこのハブの最高の作成者であることを意味します。星にカーソルを合わせると、その意味を正確に読み取ることができます。 これまでのところ、この評価は「マイサークル」でのみ考慮されています。 過去の資料の星についてもっと読む 。
