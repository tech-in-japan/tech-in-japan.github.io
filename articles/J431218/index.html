ゲームについて



私は少し描いて、描いてみて、このゲームはそのような絵から始まりました。 ヨット、荒れ狂う海、地平線上の神秘的な島を覗き込むヒーローとのマイクロコミック。 Lovecraftian精神の装飾（タコ、触手など）。 また、ホラージャンルの別の有名なマスターへの参照。 誰がどちらを言うのでしょうか？







そして、私は思った：しかし、コミックブックスタイルで設計されたゲームは、クールで、キャッチーで、珍しい！ そのようなゲームはほとんどありません。 3つ覚えています。 もちろん、これは伝説的な「コミックスゾーン」、シューティングゲーム「XIII」（13番目）であり、ゲーム全体ではなく、「What Remains of Edith Finch」のエピソードです。 しかし、独立した開発者-許されない贅沢-単独で/プラットフォーマーまたはFPSシューティングゲームと一緒に行うには、アニメーションが多すぎ、複雑すぎ、時間がかかりすぎて、ゲームがリリースに対応していない可能性があります。 いいえ、8年間プラットフォームゲームを開発することができた人々（Owlboy）がいると同時に、やる気を失わないことを知っています。 私はそのような人々を心から賞賛しen望していますが、これは私の場合ではないのではないかと心配しています。 しかし、パズル/ポイントアンドクリックゲームはまったく別の問題です！ さらに、2Dパズルを作成した経験があります。機械ボックスパズルシリーズの著者として私を覚えている人もいます。 それで決定しました！ 設定-G.Fが機能します。 ラブクラフト、ジャンル-ポイントアンドクリック、ビジュアルスタイル-コミック。



しかし、インタラクティブな漫画本や視覚的な短編小説だけをやりたくはありませんでした。 このプロジェクトを文学よりもゲームにしたかったのです。 そして、別の趣味が私を助けてくれました-ビデオゲームの歴史。 私は最近、テキストベースのコンピューターゲームについて読んだことを思い出しました。



このゲームは「ハンプザワンプス」と呼ばれていました。 これは、1972年にグレゴリーヨブによって書かれた古典的なテキストゲームです。 ウィキペディアが言うように、これは史上初のテキストベースのゲームであり、前世紀の70-80年代に非常に人気があった最初のアドベンチャーコンピューターゲームです。







要するに、ゲームの本質は次のとおりでした。プレイヤーは迷路に入ります。迷路は20の番号の部屋で構成され、各部屋は廊下で他の3人とつながっています。 つまり、そのような迷路の地図は閉じたグラフです。 ただし、あらゆる種類の用語を詳しく調べないでください。実際、すべてが非常に単純です。 危険なモンスターがこの迷路に住んでいます-ヴァンパイス、迷路を探索するプレイヤーの仕事、このモンスターを見つけて撃ちます。



ゲームの開始時に、プレイヤーはランダムな迷宮の部屋にいることに気づきます。 1つの動きで、彼は3つの隣の部屋の1つに行くか、彼女を撃つことができます。 部屋では、プレイヤーはドラフト、嫌な悪臭を感じたり、奇妙な音を聞くことがあります。 ドラフトは、隣の部屋の1つ-底なしの穴-に落ちて、あなたが失うと言います。 巨大なコウモリは音を立てます。部屋に入ると、プレイヤーを捕まえて、ランダムに選択された別の迷宮の部屋に移します。 さて、悪臭は、3つの隣接する部屋の1つにヴァンパスがあることを意味します。 プレイヤーが彼と同じ部屋にいる場合、Vampusは彼を食べます。



したがって、迷路を旅して近くの危険の兆候を思い出すと、その場所を把握し、ヴァンパスを撃ち、彼のいる場所を正確に知ることができます。 実際、同様のメカニズムを持つより一般的な別のゲームを例として挙げることができます。 これは有名な掃海艇です。 「Vampus Hunt」は、一般に、いくつかのタイプの「地雷」を持つ別のオプションです。







Lovecraftに戻ると、私たちのヒーローは海を渡ってニューイングランドに行きますが、明らかに、彼の計画は実現することはありませんでした。

旅行の17日目に、まるでどこからでもないかのように恐ろしい嵐が、運命の船をさらに未知の海に連れていきました。 フリゲートがcrash落し、主人公は難攻不落の崖に囲まれた湾の砂の上で生き返ります。 湾からの唯一の出口から、崖のすぐ下で切り取られて、それは並外れた古代で吹きます...
これは、Lovecraftの最高の伝統で、主人公は狂気と死（6つの異なるエンディングで！）しか期待していない、名前のないカルトの寺院のダンジョンへの入り口です。 ダンジョンはゲーム「ハンプザワンプス」の迷宮ですが、ヴァンパスの代わりに、ディープウォーターの守護者であるダゴンが迷宮に住んでいます。 彼をすぐに殺すことはできませんが、彼を傷つけることしかできません-結局のところ、クトゥルフ神話のパンテオンの小神の一つはあなたに冗談ではありません！ そして、ここでの巨大なコウモリの役割で、古代人の古代人-Shoggot-千の目、触手、および他の想像を絶する手足を持つ原形質の形のない塊。 Shoggotから逃げることができます。 エスケープは、3つのドアのいずれかを選択して、しばらくの間単純なパズルを解く必要があるミニゲームとして実装されます。 QTE（Quick Time Event）のトピックに関する私の即興は、ビデオゲームでかなり一般的なテクニックです。特定のアクションをすばやく実行する必要がある場合、特定のキーを押します。



マインスイーパのテーマのこのようなバリエーションに加えて、もちろん、メタゲームとしてパズルや他のミニゲームがあるはずです-すべては古典的なクエストに固有のものです。 私は何も発明しなかったので、「スポット」、「回転要素」、「組み合わせを選ぶ」など、古くから実績のあるメカニズムを使用することにしました。 当然、すべてのものをLovecraftianの設定にきちんと含める必要があります。 以下は、いくつかの例を含むGIFです。







開発



Unityの最初のプロトタイプは、約1週間で非常に迅速にまとめました。 そして、彼らに私にスリッパを投げさせましたが、私は1つの簡単なことに気付きました-Unityは私に合っていません。 はい、便利なビジュアルエディター、はい、エンジンの人気により多くの情報があります。 しかし... ...私はほとんどの時間を、既存のキューブから必要なものを組み立てる方法を見つけようと試みました。 「望ましい形状」のキューブを作成して使用する代わりに。 まあ、いくつかのことは奇妙に思えます（習慣から、私は推測します）。 たとえば、開始座標はアニメーションに「配線」されます。アニメーション内でオブジェクトが位置（0,0）から移動を開始した場合、このアニメーションがオブジェクトに適用されると、現在の位置に関係なく（0,0）にジャンプします。 あなたは言う：それで、アセットストアから<プラグイン名>をダウンロードすることも必要でした！ たぶん...



一般的に、私はより身近で理解しやすいフレームワークであるLibGDXを選択しました。 これは、クロスプラットフォームの無料のJavaエンジンです。 どこかで、Unityは高速設計に関するものであり、LibGDXは美しいアーキテクチャ、コードに関するものだという意見を聞きました。 つまり、LibGDXはプログラマーに近く、反対にUnityはコードを減らして「アクション」を必要とする人に適しています。 LibGDXの利点のうち、軽さと速度に注目する価値があります。開発の全期間にわたって、どこかで何かが遅くなるという問題に遭遇したことはありません。 まあ、これが一部の人にとってJavaであるという事実は決定的な要因かもしれません。

転記



これはJavaであるため、異なるプラットフォーム向けのビルドは、必要なJRE（Javaランタイム環境）プラットフォームで同じコードを提供することに本質的に要約されます。 私はすぐに、すべての主要なOS（Windows、Mac、Linux、Android、iOS）向けにゲームをリリースすることにしました。 モバイルストアについては詳しく説明しませんが、Steamについて詳しく説明します。 さらに、これは、Valveストアにゲームを置く最初の経験です。



最近、誰でもSteamでゲームを販売でき、登録手続きを経て100ドルの料金を支払うことができます（ゲームごとに）。 登録には数日かかり、Steamは納税申告書W-8BENに記入されたデータを検証すると言われています。 なぜ、それは非常に明確ではありません。 他のケースでは、このフォームに記入する必要があったときに、コピーだけで十分でした。



いずれにせよ、Steamworksにアクセスし、その中のボタン、メニュー、アイテム、およびサブアイテムの数から最初のショックを受けたら、ゲームページを作成します。 ここではすべてが標準です：スクリーンショット、ビデオ、一連のプロモーションアート、異なる言語での説明（もちろんローカライズを行った場合）。 ページはモデレートする必要があります。 私の場合、すべてが素早くスムーズに進みました。 ページが最初に拒否されたときに、プロモーション画像にはゲームのわかりやすい名前を付ける必要があることを説明し、このコメントを修正したときにページが受け入れられました。







承認後、ページはすべてのユーザーに表示され、ゲームは「近日公開」カテゴリに分類されます。 ビルドを収集してダウンロードできます。 LibGDXの場合にこれがどのように起こるかについてもう少し説明します。 まず、ゲームを全画面で実行する必要があります。そのために、次のコードをDesktopLauncherに追加します。 



LwjglApplicationConfiguration config = new LwjglApplicationConfiguration(); config.vSyncEnabled = true; config.width = LwjglApplicationConfiguration.getDesktopDisplayMode().width; config.height = LwjglApplicationConfiguration.getDesktopDisplayMode().height; config.fullscreen = true;


次に、実行可能ファイル（スターター）、実行されるjarファイル、および目的のプラットフォームのJVMからバンドルを収集する必要があります。 これには多くのツールがあります。 私は2つに焦点を当てます：それはおなじみのユーティリティ、 Launch4Jです。 そしてLibGDXチームによって開発されたPackr。 使用するもの、好みの問題、Packrは個人的に私に近いです。 構成ファイルを作成し、ビルドを1行で収集します。たとえば、Linuxの場合（少なくとも自動的に）：



Packrの構成例 
 { "platform": "linux64", "jdk": "/Users/me/Documents/builds/jdk1.8.0_191-linux-x64.zip", "executable": "My-Game", "classpath": [ "desktop.jar" ], "output": "EFIBB-ZIY32-8Q569", "mainclass": "com.puzzle.comix.desktop.DesktopLauncher", "vmargs": [ "Xmx1G" ] }




次に、Steamにアセンブリをダウンロードする必要があります。Windows、Mac、Linuxの3つがあります。 このプロセスの説明はドキュメント（ロシア語）にあります。そのため、キーポイントを簡単にリストします。



  1. Steamworksで、各OSの「リポジトリ」を作成します。 「Windowsコンテンツ」、「Macコンテンツ」、「Linuxコンテンツ」を作成しました。
  2. 同じ場所で、これらの「リポジトリ」を「キット」に含めます。
  3. Steamworks SDKをダウンロードします-Steamにダウンロードするために使用されます。
  4. ダウンロードスクリプトの例では、アプリケーションIDとストレージIDを独自のものに置き換えます。 ゲームファイルへのローカルパスを指定します。
  5. ブートローダーを起動します。 私の場合（Mac OS）、コマンドは次のようになります。 


 bash ./steamcmd.sh +login "user" "password" +run_app_build ../scripts/app_build_xxxxxx.vdf +quit


実際、それがすべてです。 Steamcmdはサーバーエラー500で3回クラッシュし、4回目はすべてが起動しました。 その後、モデレートのためにゲーム自体のビルドを送信します。 さらに4〜5日後、大切な緑色のボタン「公開」をクリックできます。



代替サイト



今、雨の後のキノコのように、代替のSteamストアが表示されます：Discord、GOG、Green Man Gamingなど。 Epic Gamesが独自のプラットフォームを発表することについて活発に議論しています（Steamと競争できるようになることを期待しています）。 そして、itch.ioについて少し言いたいです。 あなたが私に尋ねた場合、開発者に最適なストアをどのように見ますか？ 私はためらうことなく答えます-itch.io。 開発者にとってもプレイヤーにとっても完璧なストアとは少し違うことを理解しています。 一般的に、かゆみはユートピアです。 しかし、ここではすべてがどれだけクールでシンプルなのでしょう！ 非常に便利なゲームページエディター、すべてが非常に思慮深く、高速でモダンです。 ボタンやアクションを重ねることなく、数回クリックするだけで、世界中でゲームを利用できます！



問題は、すべての代替当事者の問題と同じ-少数の聴衆です。 メインページで紹介されたのは、単純にitch.ioの作成者にメールで書くことです（そうです、可能です。難しくはありません）。 これを念頭に置いて、このゲームの1日あたりのページビューはわずか200に過ぎず、数件の遅延販売がありました。







しかし、ここにある小さなコミュニティはとてもフレンドリーです！ かゆみにはこの機能があります。ゲームの最低「必須」価格を設定し、購入者が開発者をサポートするために自由に値を上げることができます。 そして、そのような販売は、設定価格を超えて、かなり一般的です！ 毒性やコメントが「15ルーブルで買えばよかった」スチームとは違います。



はい、ゲームは小さいです。 ほとんどの場合、すぐに物事の底に到達し、パズルに引っかかっていない場合は、1時間でそれを通過できます。 私はこのゲームには深刻なお金を稼ぐ機会がないことを理解しています。 したがって、Steamでは、99ルーブルの最も正直で予算の価格を設定しました。 おそらく、誰にとっても、この量は重要ではないでしょう。 それにもかかわらず、彼らは50％の割引を要求し、約15ルーブルを書いて、一般に「開発者の名前を宣伝するために無料で配る」（それは意味しない）。 しかし、気が散りました。 それはただの「痛い場所」です。 誰もがそうであるように、この条件付きの時間のゲームプレイにどれだけの労力が費やされたかは知っていますが、それは私を痛めています。



マーケティング



何らかの形でゲームを発表するために行ったいくつかの活動について説明します。 まず、キーメーラー。 このサービスはよく知られていますが、それがどのように機能するかを伝えるために場違いではありません。 ゲームを登録し、ブロガーに配布するために特定の数のSteamキーをサービスにアップロードします。 一方、ブロガーはサービスに登録し、ゲームを確認するためのキーを要求する場合があります。 理論的には、それはいいですね。 しかし、これは理論上のみです。 Keymailerに関する予備的なレビューをGoogleで検索しなかったのは残念です。



このサービスは開発者によって収益化されていますが、私はそれに反対するものではありません。 無料のサブスクリプションでは、ほとんどすべての機能が切断されます。 私は広告に50ドルを投じました-ゲームは1週間、どこかの有名な場所Keymailerに表示されます。 はい、「ブロガー」からのキーのリクエストがはるかに多くあります。 しかし、考えてみれば、このサービスの考えは悪意があるように思えます。 多かれ少なかれ大きなチャネルは、Keymailerを介してキーリクエストを処理する必要がなく、十分なコンテンツがあり、新製品を購入しても問題はありません。 たぶん私は間違っているかもしれませんが、Keymailerは主に不正なユーザーが無料のキーを自分自身のために、または再販のために誘うために使用されているように思えました。



購読者（少なくとも5〜1万人）がいるチャンネルからのリクエストのみを受け入れ、Keymailerから「認定」を受けました。 65個のキーを配りましたが、そのうちの47個がアクティブになり、別の18個がまだどこかで販売されています（たとえば、plati.ruでゲームを見つけました）。 その結果、6（6）のYouTubeビデオと1つのツイートを得ました。 数十のビューで...私の間違いを繰り返さないでください。



しかし、すべてがそんなに悲しいわけではありません。 YouTubeでの古い種類の手紙の手動での郵送は非常に良いことが判明しました。 夕方を過ごして「私はロボットではない」と何度もGoogleに証明した後、私はYouTubeでロシア語を話す適切なレッツプレーヤーのメールを約30件収集しました。 選択基準は簡単です：私はインディーゲームをレビューしましたが、多くのサブスクライバーではなく（ただし、10万から選んだものは少なくありません）、AAAゲームに特化せず、特定のゲーム（Minecraft、Tanks）に特化していません。







次に、正しい文字が必要です。 記憶が正しければ、セルゲイガレンキンのこれらのヒントを読みます（ゲーム業界の有名人であるリンクは提供しません）。 私の無料のリテールでは、彼らはこのように聞こえます：



  1. ゲームの最も簡潔な説明は、2文以内です。「ハワードフィリップスラブクラフトの作品を舞台にしたゲームで、完全にインタラクティブな漫画本の形になっています。」
  2. 何が欲しい？ 「チャンネルで私のゲームをプレイしてくれたらいいですね。」
  3. 手紙の本文に直接最大2 MBの面白いGIF（私はより高い胸を挿入しました）。
  4. Steamリンクとキー。 一般に、主なことは簡潔で要点です。


そして、私はその反応が自分自身に非常に合っていると言いたいです！ 最大4つのチャンネルがゲームに関するビデオを公開し、1人の「億万長者」も手紙に返信しました。 これ自体が成果です！ このゲームはストリーミングにはあまり適していません。 確かに、英語を話すブロガーに関する同様のニュースレターは、残念ながら失敗しました。 不完全な英語が原因であるか、適切なターゲットチャネルを選択できませんでした。
