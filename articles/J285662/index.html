ITのファミリービジネス。 お金を稼ぐ時間





まえがき



2012年の終わりに、セールスマネージャーが私たちを去りました。 2013年1月にプログラマーは終了し、3月にレイアウトデザイナーになりました。 すべてが崩壊したように見えた。 しかし、どのように？ 結局のところ、チームを作成して作業をセットアップしただけです。 現在の状況は憂鬱でした...



私は自分自身を欺き、無責任と無礼のせいで元同僚を非難することができました。 しかし、現実を直視してみましょう。6か月以内に4人の従業員のうち3人が退職した場合、問題は明らかに会社とそのリーダーにあります。



どうやって働きましたか



たぶん、初心者のために、2012年に会社の仕事がどのように構築されたかについて、もっと詳しく話すべきです。 夏までに、マネージャー、デザイナー、レイアウトデザイナー、プログラマーの4人の従業員からなるスタッフを編成しました。



主な活動は、サイトの開発でした。 珍しいオンラインストア、ユーザーの個人アカウントのプログラミング、視差効果のある美しいパノラマの作成、オンライン家具デザイナーなど、複雑で興味深い注文を受け付けました。 難しいほど良い。



締め切りに関する問題







チームは徐々に形成され、そしてもちろん、十分な手がありませんでした。 サーシャはしばしば植字とプログラミングを行わなければならず、経理とSEOに加えて、TKとコンテンツの開発が私にかかっていました。



通常、2〜3つのプロジェクトを同時に実施しました。理想的には、1つのサイトの開発に1.5か月以上かかることはありません。 契約を締結する段階で、作業にかかる時間を推定しました。 さらに、タイミングは非常にリアルに思えました。 しかし、常に何かが間違っていました。 誰かが病気になり、クライアントは時間通りに設計に同意しませんでした。彼らはコードに予期しないエラーを出しました。 一般的に、締め切りは絶えず中断され、クライアントの前で赤面しなければなりませんでした。



私は名刺に「プロジェクトマネージャー」を置いていました。 したがって、私は締め切りに対処することにしました。 プロジェクトの過程で、ボトルネックを事前に確認し、問題を回避しようとしました。 例えば、クライアントがデザイナーが時間内に必要とする写真を提供しないかもしれないと予想して、私は彼がクライアントを蹴るようにマネージャーを蹴りました。 そして、デザイナーがアイコンのレンダリングにぶら下がっている場合、タイプセッターがデザインレイアウトを待っているときに、デザイナーを蹴らなければなりませんでした。 そしてもちろん、完璧主義者のプログラマーが最もキックを獲得しました。 それらを時間内に停止することが重要でした。さもなければ、最もエレガントなソリューションの検索には許されないほど長い時間がかかりました。



徐々に状況は落ち着き始め、締め切りに間に合うように1週間か2週間で済むようになりました。 かつて誰からも（夫でも）密かに、契約のプロジェクトの完了期限を延長しました。 誰もが15日に作業を渡す必要があることを知っていましたが、26日に到達することが可能であることを私だけが理解していました。 だから、初めて締め切りに間に合いました。



その後、作業が調整され、それぞれがその機能を果たし、計画からの逸脱はますます少なくなりました。 ただし、万が一に備えて、プロジェクトの完了期限は、その期限よりも1週間早く設定されています。 そして、不測の事態が発生した場合でも、契約の条件は常に遵守されていました。



価格



契約を締結する段階でプロジェクトの実際の価値を判断することは非常に困難です。 私たちは価格に目を向けていたので、クライアントにとって高すぎず、許容できるものではありませんでした。 その結果、サイトを設置した後で、作業に少なくとも2倍のコストがかかることに気付きました。



改善によってさらに困難になりました。顧客は、1年前にサイトを作成した場合、そのメンテナンスのためにライフタイムサブスクリプションを持っているとしばしば考えました。 そして、私たちがあらゆる小さなことのためにお金を取るのは、なんとなく恥ずかしかったです。



時間が経つにつれて、一見取るに足りないものでさえ、仕事を支払うべきであるという理解が生まれました。 それでも、私たちは主なタスクから気をそらされ、時間を費やしています。



そして、時間コストに基づいてコストを決定する必要があります。 したがって、給与、税金、家賃、管理職の割合、および利益を考慮して、専門家の1時間の労働の平均費用を計算しました。 800ルーブルになりました。 この数値は最小単位になりました。



-当社のサイトに製品写真を追加してください！

-良い、800ルーブル。



-ああ、クライアントがフィードバックフォームでミドルネームを示すことは可能ですか？

-問題ありません、1600ルーブル。



-パートナーがおり、サイトのすべてのページにロゴ付きのバナーを追加したい。

-すごい！ 4800ルーブル。



あなたが支払う必要のある小さなものごとに、顧客が私たちを何もせずに引っ張る可能性が低いことに気付きます。 一部の人は、月額のサービス料で契約を結ぶ方が収益性が高いと理解していました。 または、ウェブサイトのプロモーションのために、クライアントが無料でコンテンツサポートを提供したため、ウェブサイトのプロモーションのために。



明確であり、サイトの開発コストを決定しました。 Mashaは、デザインレイアウトをどの程度開発する必要がありますか？ どこか5営業日。 作業明細書は、1日、レイアウトのAndreiで4日間、Kostaで制御システムへの転送で3日間、充填のためにさらに2日間、予備で2日間で作成します。 合計17日間、つまり 108800ルーブルのコスト。



顧客を選ぶ







仕事は本格的で、複雑なプロジェクトの開発だけでなく、複雑なクライアントとの仕事でも素晴らしい経験を得ました。 ただし、アプローチを見つける必要があるのは難しいクライアントです。 また、一緒に仕事をしようとしないクライアントもいます。 そしてここでは、振り向いて去ることが重要です。さもなければ、頭痛と無駄な時間が保証されます。



かつて、噴水の建設に携わっている会社からアプローチを受けました。 彼らは、会社の栄光のすべてのスキルを示すポートフォリオサイトを必要としていました。 クライアントの代表者は、HTMLの基本に少し精通しており、自分自身をWebデザインの第一人者と考えていました。 彼らは専門家を探していなかった、彼らは既製のフラッシュ効果の選択を行うパフォーマーを必要としていた、そして彼らはこれらの効果のどれとサイトで適用するかを言うだろう。



以前は、タブレットやスマートフォンではフラッシュがサポートされておらず、ウェブデザインの本質は効果や明るい色ではないことを説明し、最後まで戦いました。 しかし、その時までには、「いいえ」と言った方が良い顧客がいることにすでに気付いていました。



クライアントのタイプを認識し、彼と協力しないことは、興味のある顧客と契約を結ぶことと同じくらい重要です。 潜在的なクライアントとの仕事を拒否することは難しい決断です。 しかし、それは価値があります。 そして、顧客に「いいえ」と伝えることは確かに学ぶべきです！ そうすれば、決定が下されたことで、無駄な神経と時間を避けることができたことに気付くのは楽しいことです。



では、なぜチームは逃げたのですか？







退職した3人の従業員全員が、私たちと一緒に働くことは素晴らしいと言っていました。 「君たちはとてもクールで、フレンドリーで快適な雰囲気が好きですが、



...私たち自身のビジネスを始めるために、私の友人たちと私はすでにすべてを考え出しました！」

... Ruby on Railsとフリーランスに特化しています。

...いくつかの新しい分野で試してみますが、それ以外は1年間休息し、まったく機能しません。」



さて、3つすべてが新しいことに挑戦することを決めたようです。 また、彼らは自由を望んでいました。 結局のところ、あなた自身のビジネスとフリーランス、そして1年間の休息は、リーダーがいないことを意味します。



経験豊富な心理学者は悪の根源をすぐに見つけて、すべてをその場所に置くと思います。 知りません 夫も私も、残念ながら（または幸いなことに）心理学に強いわけではありません。 また、人事管理に関する多数の書籍を読んでも、状況は明確になりませんでした。 しかし、私には理論があります。



まず、3つすべてについて、これが最初の仕事でした。 そして最初から、彼らはそれを一時的なものとみなしました。 スキルを高めたり、経験を積んだり、履歴書にエントリーしたりできる場所として。 そしてこれは公平です。 数年しか市場に出ていないウェブスタジオで働くことに他にどのように関係しますか？



さらに、彼らは私たちがサメのビジネスから遠く離れていることを見ました。 実際、明日、このカップルは起業家に飽きず、このビジネス全体を分散させないという保証はどこにありますか？



スタッフは私たちを信じていなかったと思います。 そして、最初は最も近い人でさえ私たちを信じていなかったので、彼らはこれを非難することはできません。 私たちは真剣な会社になる時でした。 「深刻な」という言葉は、1000人のオフィスとスタッフの厳密な住所コードではなく、市場の安定性、評判、名声を意味します。



まあ、タスクは明確でした。 そして、目標を達成するための1つの方法だけを見ました。会社を発展させ続けることです。



新しい仕事のスキーム



数字について話してください。 2012年、私たちの収入は約210万ルーブル、利益-20万強を超えました。もちろん、主な支出項目は給与です。 同時に、夫と私はすでに自分自身に給料を支払う余裕がありました。 従業員よりも少ないが、そのお金で生きることはすでに可能であった。



収益の約40％は、サイトのプロモーションとコンテンツサポートによるものです。 さらに、6人の従業員全員がサイトの開発に関与しました。 そして、SEOとメンテナンスに対処したのは2人だけでした。私（コンテンツマネージャーの役​​割）とマネージャーです。



明らかに、SEOの方がはるかに有益でした。 さらに、私たちが宣伝したほとんどすべてのサイトを開発しました。 その頃には、インターネットユーザーの心に小さな革命が起こりました。 または、この革命は私たちの頭の中でのみ起こりました...一般に、時間の経過とともに、複雑なサイトの流行は過ぎ去っていることを理解し始めました。 サイトはシンプルになり、主な焦点はコンテンツとその適切なプレゼンテーションにありました。 そして、そのようなサイトはクライアントに最大の利益をもたらしました。



そのため、私たちはいくつかの重要な決定を下し、そのおかげで家族のフリーランスがビジネスになりました。



1.複雑なサイトの開発を放棄します。個人アカウント、独自のモジュール、複雑な計算機は不要です。 もちろん、そのようなプロジェクトを実装することは興味深いものでした。 しかし、私たちはそれでお金を稼ぐ方法を学びませんでした。 したがって、単純なサイトのみに焦点を合わせることにしました。



このアイデアを現実に変換するには、可能な限りシンプルなCMSが必要でした。これにより、新しい従業員を訓練する必要がなくなりました。 しかし、ユニークなデザインのサイトを作成できることが非常に重要でした。 いいえ、いいえ、私たちは独自のcmsを開発しませんでした。 クラウドソリューションの研究を開始しました。



その時までに、多くのいわゆるサイトデザイナーがすでに立ち上げられていました。 しかし、それらはすべて非常に原始的であり、そのようなサイトを顧客に販売することは残念でした。



そして、サーシャはSquarespaceを見つけました。 このクラウドベースのCMSにより、システムのすべての組み込み機能を使用して、独自の設計を作成できました。 人気のあるランディングページの場合-ソリューションは完璧です。



このスキームにより、プロセスが大幅に簡素化されました。 サイトの立ち上げはわずか1週間で完了しました。 さらに、Mashaとほぼすべての作業を行い、Sashaはレイアウトを完成させてテンプレートをデザインに合わせるために数時間を費やすこともありました。 ウェブサイトの開発期間を短縮したため、利益を上げるプロセスを加速しました。



2. SEOに焦点を当てます。 クライアントがプロモーション契約を締結する準備ができている場合にのみ、ウェブサイト開発の注文を受け付けます。



3.ウェブサイトのプロモーション契約のスキームを変更します。

以前は、4か月間、サイトをTOPに移動し、作業を停止しました。 現在、TOPへの退会期間は6か月に延長されています。 この後、プロセスを停止することはできません。そうしないと、2〜3か月でサイトは達成されたすべてのポジションを失います。 したがって、プロモーション契約は無制限になりました。



4. seoshnikを使用します。

単純なサイトの開発への移行によりSashaが解放された場合、新しい従業員は私から非常に大きな負荷を取り除きました。



5.新しいオフィスに移動します。

そして、これは労働条件を改善するために必要でした。 前のオフィスが不快だったわけではありませんが、十分なスペースがありませんでした。 今、私たちはリラクゼーションのための追加の部屋を持っています。



結果



第一に、私の夫はほぼ100％彼の会社の従業員を辞め、私も多くのことを捨てました。 私たちは週末を過ごしましたが、就業日には長引くのをやめ、夜は家で働きませんでした。



第二に、2013年には350万ルーブルを獲得しました。 昨年と比較して、利益は5倍に増加し、120万に達しました。

私たちは最初の百万を作りました、それはめちゃくちゃクールでした！



第三に、市場で目立つようになりました。 かつて、クライアントがサーシャと呼ばれ、面白い話をしました。

彼らはバーで友人と座って、差し迫った問題について議論しました。 クライアントがタブレットを取り出して、自分のサイトを友人に見せました。 それに応じて、友人が自分のサイトを見せ始めました。

一般的に、彼らは座って、お互いに自慢し、そのサイトがより涼しいと主張しました。 そして、我々は両方のサイトが私たちによって作られたことを発見しました。 いいですね 私はこの物語が大好きです！



お金は遊ばない



前回の記事で、私は最初の黄金律について話しました。お金は遊ばせてはいけません。

さて、今はお金がありました。 そして、過去の過ちを思い出して、私たちはこのお金を機能させることにしました。 多くのオプションとアイデアがありました。 どこで止めたの？ 1週間で見つけてください！ そして、すぐに警告します。次の記事で私はPRになります！
