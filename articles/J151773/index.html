SQL Databaseを使用してPHPでクラウドサイトを作成し、Gitでデプロイする

このガイドでは、Windows Azure SQLデータベースを使用してWindows Azure PHP Webサイトを構築し、Gitを使用して展開する方法について説明します。 お使いのコンピューターには、 PHP 、 SQL Server Express 、 SQL Server for PHP用のMicrosoftドライバー 、Webサーバー、およびGitがインストールされていることが前提です。 すべての手順を完了すると、Windows Azureプラットフォームで実行されているSQLデータベースを含むPHP Webサイトが表示されます。



ご注意 Microsoft Web Platform Installerインストーラーを使用して、 PHP、SQL Server Express、SQL Server for PHPのMicrosoftドライバー、およびIISをインストールおよび構成できます。



このガイドで説明する内容：



  • 管理ポータル（プレビュー）を使用してWindows Azure WebサイトとSQLデータベースを作成する方法。 PHPテクノロジはWindows Azure Webサイトでデフォルトで有効になっているため、PHPコードを実行するために特別なアクションは必要ありません。
  • Gitを使用してWindows Azureプラットフォームでアプリケーションを公開および再公開する方法。


このガイドの指示に従って、登録用の簡単なPHP Webアプリケーションを作成します。 このアプリケーションは、Windows Azure Webサイトでホストされます。 完成したアプリケーションのスクリーンショットを以下に示します。







Windows Azure Webサイトを構築し、Gitの公開を構成します。 Windows Azureアカウントを作成する



Webブラウザーを開き、 http：//www.windowsazure.comにアクセスします 。 無料アカウントを開始するには、 無料試用版の右上隅をクリックして、次の手順に従ってください。 本人確認を行うには、クレジットカードまたは携帯電話番号の提供が必要になる場合があります。 請求書は発行されません。







Windows Azure Webサイトを有効にする



https://account.windowsazure.com/にアクセスし、Windows Azureアカウントでサインインします。 プレビュー機能要素をクリックして、利用可能なプレビューオプションを確認します。







[ Webサイト ]までスクロールし、[ 今すぐ試す ]をクリックします。







サブスクリプションを選択して、チェックボックスをオンにします。







Windows Azure WebサイトとSQLデータベースを作成するには、次の手順を実行します。



Windows Azureポータル（プレビュー）にサインインします。 ポータルの左下にある[ +新規 ]アイコンをクリックします。







[ Webサイト]をクリックし、[ データベースで作成]をクリックします。







[ URL]フィールドに値を入力し、[ DATABASE]ドロップダウンリストから[ Create a New SQL Database]を選択し、[ REGION]ドロップダウンリストでWebサイトのデータセンターを指定します。 ダイアログボックスの下部にある矢印をクリックします。







NAMEフィールドにデータベースの値を入力し、 EDITION （WEBまたはBUSINESS）の値を選択し、データベースのMAX SIZEの値とCOLLATIONの値を設定し 、 NEW SQL Database serverを選択します 。 ダイアログボックスの下部にある矢印をクリックします。







管理者名とパスワードを入力し（パスワードを確認）、SQLデータベースサーバーを作成する地域を選択し、 Allow Windows Azure Services to access the server



チェックボックスをオンにします。







Webサイトの作成後、Webサイト '[SITE_NAME]'の作成が正常に完了しましたというメッセージが表示されます。 これで、Git公開を有効にできます。



Webサイトのリストに表示されているWebサイトの名前をクリックして、Webサイトのクイックスタートパネルを開きます。







[クイックスタート]ページの下部にある[ Git公開のセットアップ ]をクリックします。







Gitの公開を有効にするには、ユーザー名とパスワードを提供する必要があります。 作成したユーザー名とパスワードを覚えておいてください。 （Gitリポジトリーが以前に構成されていた場合、このアクションはスキップされます。）







リポジトリのセットアップには数秒かかります。







リポジトリの準備が整うと、アプリケーションファイルをリポジトリに配置する手順が表示されます。 これらの指示は後で必要になるため、覚えておいてください。







SQLデータベース接続情報の取得



Windows Azure Webサイトで実行されているSQLデータベースインスタンスに接続するには、接続情報が必要です。 SQLデータベースへの接続については、次の手順を実行します。



管理ポータル（プレビュー）で、[ リンクされたリソース ]をクリックし、データベース名をクリックします。







[ 接続文字列を表示]をクリックします 。







開いたダイアログボックスのPHPセクションで、 SERVER



、 DATABASE



、およびUSERNAME



値を覚えておいてください。



アプリケーションのローカルでの作成とテスト



登録アプリケーションは、ユーザーの名前とメールアドレスを入力してイベント参加者を登録するためのシンプルなPHPアプリケーションです。 以前の登録者に関する情報が表に表示されます。 登録情報は、SQLデータベースのインスタンスに保存されます。 このアプリケーションは2つのファイルで構成されています（コピーと貼り付けのコードを以下に示します）。



  • index.php 登録フォームおよび登録された参加者に関する情報を含む表を表示します。
  • createtable.php アプリケーションのSQLデータベーステーブルを作成します。 このファイルは一度だけ使用されます。


アプリケーションをローカルで実行するには、次の手順を実行します。 ローカルコンピューターにPHP、SQL Server Express、およびWebサーバーがインストールされており、 SQL ServerのPDO拡張機能が有効になっていることを前提としています。



registration



というSQL Serverデータベースを作成します。 これは、次のコマンドを使用してsqlcmd



コマンドラインから実行できます。



> sqlcmd -S localhost \ sqlexpress -U <ローカルユーザー名> -P <ローカルパスワード>

テキストエディタまたはIDEでcreatetable.php

アプリケーションを公開



アプリケーションをローカルでテストすることにより、Gitを使用してWindows Azure Webサイトに公開できます。 ただし、最初にアプリケーションのデータベース接続情報を更新する必要があります。 以前に取得したデータベース接続情報（ 「SQLデータベース接続情報の取得」を参照 ）を使用して、 両方のファイルの適切な値で次の情報を更新します。 



 // DB connection info $host = "tcp:<value of SERVER>"; $user = "<value of USERNAME>@<server ID>"; $pwd = "<your password>"; $db = "<value of DATABASE>";


ご注意 SERVERの値を追加する必要があります。値は、USERNAME、 '@'、およびサーバー識別子の値を結合します。 サーバー識別子は、SERVER値の最初の10文字で構成されます。



これで、Gitの公開を設定し、アプリケーションを公開する準備ができました。



ご注意 「Windows Azure Webサイトの作成」および「Git発行の構成」セクションの最後に記載されているのと同じ手順に従います。



GitBash（またはGitがある場合はターミナル）を開き、ディレクトリをアプリケーションのルートディレクトリに変更し、次のコマンドを実行します。



git init

git add。

git commit -m "初期コミット"
