議題



キューは、パフォーマンスの問題のために開発されている式変換アルゴリズムのようなアルゴリズムの不可欠な部分です。 大量の変換要求により、アプリケーションが非常に迅速に停止する可能性があります。 最も論理的で簡単な方法は、キューに入れることです。

キューを整理するために、Node.jsのライブラリがいくつかあることに注意してください。 アジェンダを選んだ理由は次のとおりです。

  • MongoDBにタスクを保存することが可能です
  • ライブラリを使用すると、通常の意味でキューを整理できるだけでなく、スケジュールに従ってスケジュールを設定することができます。これは非常に楽しい追加機能です。




ジョブをキューに追加 

 service.agenda.now( 'process new formula', {fid: fid} );


ジョブハンドラーの登録 

 service.agenda.define('process new formula', (job, done): void => { var data = job.attrs.data; log.debug( "process new formula: " + data.fid ); AgendaService.processRendering( data.fid, done ); });
