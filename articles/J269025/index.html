アマゾンウェブサービスインフラストラクチャ パート2





Amazon、またはそのクラウド部門では、AWSは最も急速に成長し、展開され、最も高度なインフラストラクチャの1つです。 すでに書いたように 、少し前に、会社はこのインフラストラクチャの作業を整理する原則に関するいくつかの秘密を明らかにしました。 2番目の部分には、データセンターの場所、これらすべてを1つのシステムに接続する方法、およびその仕組みに関する情報が含まれています。



Amazon Web Servicesには少なくとも30のデータセンターがあり、管理計画ではさらに10または15の新しいデータセンターを構築する予定です。 ほとんどのデータセンターはバージニア州北部にあり、AWSは約20のデータセンターを担当し、総容量は約500 MWです。 ただし、AWSデータセンターのインフラストラクチャの地理は、バージニア州だけに限定されません。 現在、オハイオ州には3つの大規模なDCキャンパスが建設されており、さらにクラウドデータセンターはアイルランド、ブラジル、中国、日本、オーストラリア、シンガポールで稼働しています。







米国については、ここで同社のデータセンターは2つのグループに統合されています。1つは西側に、もう1つは東海岸にあります。 これにより、最小限の遅延で、大陸全体でデータを均一に送信できます。 さらに、米国のデータセンターのハブは、他の地域でもカスタマーサポートを提供しています。 たとえば、東海岸のハブはヨーロッパにサービスを提供し、西はアジアにサービスを提供しています。 また、このネットワーク構成により、AWSに問題なく、一時的にアイドル状態のDC（たとえば、災害による）を複製できます。



同様の動作原理が、オレゴンとノースカロライナに独自のサーバーファームを持つGoogleとFacebookでも使用されています。 Amazonにとって、インフラストラクチャ全体の主要なコアはノースバージニアです。



ノースバージニア



会社のネットワークインフラストラクチャは世界中に広がっているため、多くの地域に会社のDCがあります。 しかし、財団はラウドン郡とウィリアム王子郡の23のデータセンターです。 なぜここに多くの企業データセンターがあるのですか？ 事実、Equinixハブを含むほとんどすべての重要なネットワークがここに接続されています。



同社のDCの9つはスターリングに、5つはオズボーンに、6つはマナッサスに、2つはシャンティリーにあります。 このデータセンターの配置により、アベイラビリティーゾーン間で非常に低いレイテンシのデータ転送が提供されます。企業のお客様は、さまざまな地域のittanを使用して、予期しない状況（自然災害など）の場合にビジネス継続性を確保できます。



その結果、米国東海岸には最大数のAmazon Web Servicesデータセンターがあり、同社はシステムにデータセンターを追加することでインフラストラクチャを拡大し続けています。 ここで、DCの総容量は約500 MWに達します。







オレゴンおよびカリフォルニア



2009年以来、Amazonはカリフォルニア州北部にデータセンターを構築しています。 確かに、ここでは電気が非常に高価であるため、同社はオレゴンに重点を置き、コロンビア川沿いの小さな町にデータセンターを再構築します。



オレゴンでは、アマゾンだけでなく、他の多くの企業が建設されています。 ここにはいくつかの理由があります。



  • 外気を使用してシステムを冷却できる涼しい気候。
  • ビジネスに優しい税制。
  • 川にある水力発電所からのエネルギー。


後者の要因は、Amazonに別の利点をもたらします-グリーン企業のリストでのランキングを上げます。 はい、水力発電所は環境にダメージを与えますが、同じ火力発電所ほど重大ではありません。



この地域に構築された3つのデータセンターは、本質的に実験的なものであり、従来のデータセンターとモジュラーデータセンターの機能を組み合わせています。 これにより、DCの機能を迅速に拡張し、オレゴンでのDCの存在感を高めることができます。 現在、別のデータセンターがここに建設されています。



同社は、オレゴン州にさらに15のデータセンターを建設することを望んでいることをすでに発表しています。



オハイオ



同社のリーダーシップは、ダブリン、ヒリアード、ニューアルバニーの3つの場所に一度に約11億ドルを投資してデータセンターキャンパスを設立する意向を発表しました。 各キャンパスには、それぞれ2.5 MWの容量を持つ18台のスタンバイジェネレーターを備えた最大5つのデータセンターがあります。



この情報により、各キャンパスの総容量（約32 MW）を判断できます。



AWSは、CloudFrontネットワークをサポートするために世界中で約40のエッジノードを管理しています。
