伝説の復活：OSをLibretto 50 CTに搭載

90年代にラップトップがどのように作られたかについての友人kurtkrutの記事に触発され、ちょうど同じ機器が国内に横たわっていることを思い出し、すぐにそれを復活させ、使用を開始することさえ考えました。 この東芝Libretto 50 CTは、友人からずっと前に私に届いたもので、主に学校でレポートを書いたり、Turbo Pascalでプログラミングしたりするために使用しました。 私が電源を失った後、ラップトップは地獄への死んだバッテリーで放棄されました。





そこで、Librettoを見つけて、Linuxを使用して復活させることにしました。



はじめに



そのため、このユニットを起動するには、電源を見つけて、在庫の機器と鉄のすべての制限を考慮する必要がありましたが、それらのいくつかはありません。 まず、メーカーのPCMCIAフロッピーしか持っていませんでしたが、CD-ROMやZIPも入手できませんでした。 次に、ラップトップのHDDは810MBであり、Linuxディストリビューションの本格的なイメージを書き込むことはできません。 kurtkrutからの電源が必要であることがわかり 、15Vブロックで店舗に行きました。 すでに店で私は15Vであったものを取りました、それは最初のステップでした。





次に、ディストリビューションと、それらを弱いコンピューターにインストールする可能性について検討しました。 3つのディストリビューションを試すことにしました。

  • DSL
  • Black Cat Linux
  • Deli Linux




試行番号1



DSLをインストールするには、最新のインストールディスクをダウンロードし、ブートフロッピーのセットを準備する必要がありました。 ハードディスクは、インストールディスクを記録するためにラップトップから取り外されました。 仕事のために、外部コントローラーが使用されました



ネジはすぐに3つのセクションに分割する必要がありました。

  • FAT16の120 MB DSLパーティション
  • 32MBスワップスワップパーティション
  • 使用するセクション


新しく壊れたディスクへのイメージの書き込みはすぐに成功し、フロッピーディスクとブートフロッピーの作成も簡単でした（理論的には3つ必要です）。

ネジを元の位置に戻し、プロセスを開始しました。メモリをチェックした後、ラップトップはインストールオプションを含むGRUBプロンプトを表示し、dsl fromhd = / dev / hda mem = 16Mオプションを入力しました。 カーネルは数分間起動し、ルートディスケットが必要でしたが、フロッピーの音が奇妙に響き、ログを調べた後、東芝の標準フロッピーでは判断できないことが明らかになりました...私はそれについて何もできませんでした。

おわりに



フロッピードライブがサポートされていないため、DSLをこのように配置することはできませんでした。



試行番号2



ネジを抜いたので、私は同じ方法でDeli Linuxをインストールすることに決めましたが、tormsbt linuxはフロッピーからの起動に使用されるようになりました。 ネジに必要なデータを書き留め、フロッピーディスクを作成しました。 最初のケースとまったく同じ画像を読み込んで表示します。

おわりに



すべてのフロッピーディストリビューションが同じパッケージに基づいており、東芝フロッピーをサポートしていないことが明らかになりました。 ごめんなさい...



試行番号3



ネットワーク上のこのトピックで見つけたすべてを読み直した後、仮想マシンからインストールするというアイデアが気に入りました。 仮想マシンに物理的なネジを接続する機能を見たことがあり、VMwareがこの仮想マシンでしたが、新しいゲストシステムを作成した後、SCSIデバイスとしてのみ物理的なネジをフックすることに失望しました。 そこで、IDEとして仮想ネジを作成し、それにDeliを配置しました。 次のステップでは、仮想マシンのLiveCD Ubuntuから起動し、仮想ハードドライブと物理ハードドライブを接続しました。 ddユーティリティを使用して、すべてをコピーしました



#dd if = / dev / hda of = / dev / sda



だから私はインストールされたシステムとわずかに間違ったパーティションテーブルを備えたHDDを手に入れました。 fdliskユーティリティを使用して、パーティションテーブルを調整した後、ディスクをラップトップのネイティブコネクタに戻しました。

LILOブートローダーとデリの選択を見たとき、私の熱意を想像することはできません。 システム項目をクリックすることにより、私は再び失望しました：カーネルが起動してinitに到達しましたが、fsck.ext2でパーティションをチェックし始めたので、カーネルダンプが見られました...私の欲求不満に制限はありませんでした...

おわりに



このように、上記で提案した3つのシステムすべてを配置しようとしましたが、結果はカーネルダンプに異なるアドレスのみを持つものになりました。



おわりに



したがって、追加の処理とオーバークロックなしでこのマシンにLinuxをインストールすることは不可能であり、 kurtkrutによってすべてが同じLibretto 50CTでテストされ、結果は同じであると結論付けました 。

3つのアプローチはすべて、ネットワークとCDROMを使用せず、フロッピーだけを手に入れてシステムをインストールする方法を学ぶことができました。 カーネルが正常に起動した場合、私は幸せになるでしょうが、主なことは経験です！



PS



最初のパンケーキはゴツゴツしていますが、私はやめず、DOSとWIN95で動作するようにします;）
