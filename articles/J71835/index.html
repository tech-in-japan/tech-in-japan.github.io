複数の背景画像



誇大宣伝にもかかわらず、Mozillaが最初に-moz-border-radiusを導入したとき-単純なCSSプロパティで丸い角を作ることができます-私は個人的に興奮しませんでした。



それどころか、背景画像の多様性の概念が約束する機会に常にショックを受けました-洗練されたレイアウトに固執することなく 、レイアウト上のいくつかの単純な壮大なトリックによって実行されました。



基本的な構文は非常に簡単です。 background-imageプロパティで1つの画像のみを指定する代わりに、最大4つの画像をコンマで区切ってリストします。



background-repeatおよびbackground-positionに対応する最大4つの値を指定することもできます。 



  #multi { 


  background-image：url（paper.png）、url（paper2.jpg）; 


 バックグラウンドリピート：繰り返しなし、繰り返しなし; 


 バックグラウンド位置：左上、右下。 


  }




やや不自然なソート順を発見したと言わざるを得ません。 2番目のイメージは最初のイメージの下にリストされ、以降も同様です。 しかし、あなたが感じて覚えるとすぐに、問題はありません。







さて、どのブラウザがこのプロパティをサポートしていますか？


現時点では、複数の背景画像は、Safari 4.0、Chrome 3.0、およびFirefox 3.6 alpha（Minefield）でのみ完全にサポートされています。 次のURLでこれらのブラウザのいずれかにアクセスすると、実際の私の例を見ることができます。



http://www.sitepoint.com/examples/css3/multi.html



さて、今このプロパティを使用する価値はありますか？


異なるブラウザでコードの表示をチェックするときに最初に遭遇した問題の1つは、リストの最初の画像のみを拒否し、残りを無視することでした。 これらのブラウザで宣言全体を削除するには、コンマで十分です。背景は空のままです。



ただし、背景画像をすべての標準に最初に宣言すると、Opera、FirefoxおよびSafariの古いバージョンは、コード内でさらに進む可能性のある新しいCSS広告を簡単に無視することがわかりました。



Internet Explorerがこのルールに従うことを拒否するという事実は、IEで実行される単純な単一画像コードを強制する条件でコメントを自由に使用できるため、問題ではありません。 



  <！-[IEの場合]> 


  <style type = "text / css"> 


  #multi { 


  background-image：url（old-paper2.jpg）; 


  background-repeat：繰り返しなし; 


 背景の位置：左上; 


  } 


  </ style> 


  <！[endif]->




一般に、このプロパティは実用的であり、今日でも結果が得られると思いますが、優雅に使用するには非常に注意する必要があります。



デザインが古いブラウザで正常に機能することを確認してください（最終的には、あなたを悪化させることはありません）が、新しいブラウザのユーザーが増え続ける中で、デザインにさらに豪華なタッチを追加します。 それでは、実験の時間です。



背景画像のサイズを設定する



タイトルから推測できるように、background-sizeを使用すると、CSSを使用して追加された画像を背景に拡大縮小できます。







実際、今ではbackground-sizeプロパティはネイティブCSS3形式を除くすべての場所でサポートされていませんが、Opera、Chrome / Safari、Firefox 3.6 alphaはサポートしていますが、それぞれ独自のマークアップがあります。



マークアップは次のとおりです。 



  body { 


  background-image：url（background_image.jpg）; 


 背景サイズ：50％50％;  / * w3c仕様* / 


  -moz-background-size：50％50％;  / * firefox css * / 


  -webkit-background-size：50％50％;  / *サファリ/クロム* / 


  -o-background-size：50％50％;  / *オペラcss * / 


  }

CSSグラデーション



CSSバックグラウンドグラデーションは、現時点ではFirefoxアルファ、Safari 4、Chrome 3でのみサポートされていますが、最も革新的なイノベーションのように思える機能です。

WebKitとMozillaはどちらも放射状グラデーションと線形グラデーションの両方を表示できますが、それらの方法は異なります。 Mozillaが2つの異なるプロパティ値（-moz-linear-gradient（）や-moz-radial-gradient（）など）を使用することを好む場合、WebKitは単一の値を使用しますが、特別なパラメーター（たとえば、-webkit-gradient（radial 、...））

CSSグラデーションを使用することには3つの利点があるように思えます。

  1. 画像とは異なり、CSSグラデーションはそれらを含む要素に比例して拡大縮小するため、ページ上のコンテンツの動作に応じて拡大および縮小できます。
  2. CSSグラデーションは非常に効果的で、わずか数行のコードで必要なトーンの大きく柔軟な構造化領域を生成できます。 これは、ユーザーのページの読み込み速度に大きく影響します。
  3. セキュリティ上の理由やインターネットチャネルの帯域幅の問題のために、ユーザーが画像をブロックできる状況は数多くあります。 CSSグラデーションは、画像では表現できない複雑な色を与える可能性が高くなります-たとえば、iPhoneのSafari。
