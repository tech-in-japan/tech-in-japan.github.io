Ubuntuにjiraをインストールする









要するに、jiraはプロジェクト管理システムですが、そのビジネスを確立することは傑出したものとはほど遠いものです。 長い英語のマニュアルを読むのは非常に骨が折れるため、インターネット上のロシア語の記事は今のところあまり修正しません。







後でJiraに関連付けられるように事前構成されたSVN。



最初に行うことは、javaのインストールです。 これは難しいことではなく、apt-get installだけで完了です。 必要なJavaが存在しない場合は、マルチバースリポジトリを/etc/apt/sources.listに追加すると、すべてが泣き出します。



次に、Tomcatをインストールする必要があります。 同じapt-getインストールですべてインストールすることも、 サイトから取得することもできます 。 これは次のように実行できます。

wget mirrors.axint.net/apache/tomcat/tomcat-6/v6.0.26/bin/apache-tomcat-6.0.26-deployer.tar.gz





開梱：

tar zxvf apache-tomcat-6.0.26-deployer.tar.gz







構造について少し：

  • / bin /フォルダーには、実行可能ファイル（起動、サーバーの強制終了など）があります。
  • フォルダー/ conf /にサーバーの構成ファイルがあります
  • もちろん、ログ/フォルダー、つまりlogs / catalina.outファイルを忘れないでください-これがメインログファイルのある場所です
  • / lib /-ライブラリ、追加のライブラリをそこにコピーする必要があります


打ち上げ

sh /bin/statup.sh





少なくとも問題なく開始しました。 localhostの確認：8080。 ページにキティが表示されている場合-何も表示されていない場合はすべて問題ありません-Tomcatのマニュアルを参照してください。



jiraをダウンロードしていきましょう。 戦争バージョンが必要であり、ダウンロードするためのリンクを表示するには、 [すべて表示]をクリックする必要があることに注意してください（正直なところ、私は長い間検索しました）。



ダウンロードjira：

wget www.atlassian.com/software/jira/downloads/binary/atlassian-jira-enterprise-4.1.tar.gz





解凍：

tar zxvf atlassian-jira-enterprise-4.1.tar.gz







ファイルatlassian-jira-enterprise-4.1 / edit-webapp / WEB-INF / classes / entityengine.xmlを編集します。 mysqlにjiraをインストールします。



ファイルを開くと、最後に次のように表示されます。

<datasource name="defaultDS" field-type-name="hsql"

schema-name="PUBLIC"

helper-class="org.ofbiz.core.entity.GenericHelperDAO"

check-on-start="true"

use-foreign-keys="false"

use-foreign-key-indices="false"

check-fks-on-start="false"

check-fk-indices-on-start="false"

add-missing-on-start="true"

check-indices-on-start="true">

<jndi-jdbc jndi-server-name="default" jndi-name="java:comp/env/jdbc/JiraDS"/>













field-type-name = "hsql"をfield-type-name = "mysql"に変更します

行schema-name = "PUBLIC"を削除します

このファイルのすべてを他のものに触れる必要はありません。



ここで、jiraが存在するディレクトリを確立する必要があります。 atlassian-jira-enterprise-4.1 / edit-webapp / WEB-INF / classes / jira-application.propertiesを開き、jira_home = / path / to / folderを設定します（このフォルダーを作成して正しい権限を設定することを忘れないでください）。



ファイルを実行する

sh atlassian-jira-enterprise-4.1/build.sh





jiraが構築されるのを待ちます。 約2分でビルドします。



次に、Tomcatライブラリを更新する必要があります。 ここで取得できます が、Tomcatの/ lib /ディレクトリに展開する必要があります。 そこで、 ここで取得できるmysql-connector-java-5.1.12-bin.jarファイルをコピーする必要があります。



ビルド後、フォルダーatlassian-jira-enterprise-4.1 / dist-tomcatが表示されます。 そこに行き、必要なTomcatのバージョンを見つけて、 jira.xmlファイルをTomcat / conf / Catalina / localhostフォルダーにコピーします。 / conf / Catalina / localhostでこのファイルを開き、編集します。 このセクションが必要です：

<Resource name="jdbc/JiraDS" auth="Container" type="javax.sql.DataSource"

username="sa"

password=""

driverClassName="org.hsqldb.jdbcDriver"

url="jdbc:hsqldb:/home/max.zloy/atlassian-jira-enterprise-4.1/database/jiradb"

minEvictableIdleTimeMillis="4000"

timeBetweenEvictionRunsMillis="5000"/>







username = "sa" -データベースのパスワードを指定します

パスワード= "" -ログインを指定

driverClassName -put-com.mysql.jdbc.Driver

url -jdbc：mysql：// localhost / jiradb？useUnicode = true＆characterEncoding = UTF8（「＆」はタイプミスではないことに注意してください、そうする必要があります）

行を削除します。

minEvictableIdleTimeMillis="4000"

timeBetweenEvictionRunsMillis="5000"





これらのオプションはhsqlでのみ動作します

そして、それらの代わりに追加します：

maxActive="20"

validationQuery="select 1"





以上で、このファイルの構成は完了です。 近い保存。



jiradbという名前のデータベースを作成し、必ずUTF-8エンコーディングに設定してください。 それだけではありません。 jiraは非常に貪欲なので、Tomcatにより多くのメモリを配置する必要があります。



Tomcat bin / setenv.shディレクトリでファイルを開き、そうでない場合は作成して、そこに行を貼り付けます：



保存します。



これですべてです。

/bin/startup.shおよび/bin/shutdown.shを使用してTomcatを再起動する

localhostを開きます：8080 / jira（jiraがインストールされるまで少し待つ必要があります）。

export CATALINA_OPTS="$CATALINA_OPTS -Dorg.apache.jasper.runtime.BodyContentImpl.LIMIT_BUFFER=true -Dmail.mime.decodeparameters=true -Xms256m -Xmx256m -XX:MaxPermSize=256m"
