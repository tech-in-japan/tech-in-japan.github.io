データストレージシステムの分野での輸入代替：ロシアのソリューションを探すには？

一部のロシア企業にとっては、為替レートの高さや機器の供給の中断や技術へのアクセスの終了を脅かす可能性がある困難な地政学的状況を背景に、顧客はインフラストラクチャをアップグレードする際のリスクを最小限に抑える方法を探しています。 インポートを置き換えることができるソリューションが前面に出てきます。



CROCのデータストレージシステム責任者、ウラジミールコルガノフは次のように述べています。

「私たちの側では、市場を模索し、代替の可能性がある作業オプションを選択しています。 そのため、過去1年だけで、ロシアと東部のさまざまなメーカーの約6つのソリューションが、当社の研究室でテストされています。」







ウラジミール・コルガノフによると、クラウドインフラストラクチャを実装するための最も有望なソリューションの1つは、ARSIENTEKソフトウェアとハ​​ードウェアの複合体であるResilient Cloud Storageシステムです。 CROCはプロジェクトでこのシステムを使用して、ロシアの企業に高度なITソリューションを提供し、この分野のインポートされたソフトウェアとハ​​ードウェアを置き換えます。 そのユニークな点は何ですか？また、なぜこのシステムは最大のシステムインテグレーターから非常に高く評価されているのですか？







エンタープライズクラスのデータストレージシステムであるResilient Cloud Storageは、特に輸入代替のコースに照らして、高度で完全にロシアの開発であり、機能的で関連性があります。



RCSは、ボリュームの増加に伴う生産性の直線的な増加、自己修復機能を備えた水平方向に数百ペタバイトまで拡張できる弾性システムであり、S3やRBDなどの最新のプロトコルもサポートします。大量のデータで。



Resilient Cloud Storageの主な利点は、単一のストレージシステム内で数百ペタバイトまで水平に拡張できることです。 これは、ソリューションのユニークなアーキテクチャにより実現されます。このアーキテクチャでは、常にパフォーマンスが制限される専用のストレージコントローラーがなく、サポートされるディスクシェルフの数と外部と通信するためのインターフェイスがあり、データを操作するときに「ボトルネック」効果を生み出します。 Resilient Cloud Storageでは、コントローラーが各ディスクモジュールに統合されているため、システム全体のパフォーマンスを損なうことなく、膨大な量のデータを扱うことができます。 新しいディスクモジュールの追加により、ストレージ容量とパフォーマンスの両方が直線的に増加しています。



システムの同様に重要な利点は、その信頼性です。 データは3回バックアップされ、RCSクライアントはストレージモジュールと直接やり取りし、優先度の高いアプリケーションには専用のリソースグループが提供されます。 さらに、RCSはデータ回復および自動データバランシング機能を実装しています。 ディスクまたはモジュールに障害が発生した場合、システムは自動的に回復し、パフォーマンスの低下を最小限に抑えて、空き領域に冗長性を再度作成します。 新しいモジュールを追加すると、データバランシングが自動的に行われます。



ARSIENTEKのゼネラルディレクターであるDenis Nestunは、「彼の頭脳」について語っています。

「Resilient Cloud StorageはARCIENTECの主力製品であり、大企業、問題を解決するために実際のクラウドスケーラビリティを必要とするが、データセンターで独自のクラウド内で最善の方法でそれを提供する必要があるプライベートおよびパブリックプロジェクトに焦点を当てています。 CROCはロシアのIT市場の主要なプレーヤーであり、もちろん、国内の大企業でデータストレージインフラストラクチャを構築するなど、幅広い経験を持っています。 CROCはプロジェクトに最適なソリューションのみを選択し、パートナーになったことを誇りに思っています。 当社のシステムは、CROCが顧客に提供する製品ラインで正当な位置を占めると確信しています。」
