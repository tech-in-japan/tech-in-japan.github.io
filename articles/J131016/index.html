テキストのコピーを隠さない4つの理由

最近、コンテンツの盗難が非常に一般的になりました。 「サイトから素材をコピーする場合、ソースへのクリック可能なインデックス付きリンクは必須である必要があります。」、キャプション付きで保存されているものもあります。

テキストのコピーを非表示にしない4つの理由を見てみましょう。







1. RSSフィード





サイトのスクリプトがテキストをカバーできる場合、RSSフィードは常にそうとは限りません。 テープからテキストをコピーするには、そこに行く必要があります。 たとえば、site.ruなどのサイトにいる場合は、名前/フィードに追加します。一緒に、site.ru / feedのようになります。 次に、必要なテキストの断片をコピーします。



2.ソースコード





繰り返しますが、以前のバージョンでスクリプトがコピーに影響する場合がありました。 この場合、何も閉じられません。

  1. テキストのコピー元のWebサイトを開きます。
  2. Ctrl + Uを押してソースコードを確認します。
  3. Crtl + Fを使用して検索を開き、閉じたテキストのフラグメントを入力します。
  4. 必要がある排水溝をコピーします。




3. JavaScript





テキストのコピーは右クリックとCtrl + Cで行われます。別のオプションは同じです。キーボードを使用する代わりに、マウスの左ボタンを押してコピーします。 このプロセスの本質は、プレスごとにプロセスが発生することです。 そして、これらのプロセスの標準操作を抑制する最も一般的な方法は、JavaScript関数です。 最も興味深いのは、JSがクライアント側で処理されるという事実です。 また、ブラウザでその処理を簡単に無効にすることができます。

ブラウザの設定で「JavaScriptを有効にする」オプションを見つけて、チェックボックスをオフにします。 次に、テキストのコピーが禁止されているサイトに移動し、必要なものをすべて用意します。



4.画像からテキスト





コンテンツを盗むこの方法は最長ですが、最も効果的です。 サイトでスクリプトが使用される非常にまれなケースがあります。素材を書き込んだ後、自動的に画像に処理するプログラムです。 実際、そのような動きは私には少し分かりにくいです。 結局のところ、サイトにはテキストがまったくないことがわかります。検索エンジンは何を言うでしょうか？ 彼らが言うように：「私自身はあえてしませんし、私はそれを他の人に与えません。」 まあ、大丈夫、あなたがそのようなサイトに着いたのと同じなら、最初の3つの方法はすぐに捨てることができます。 4番目に進みます。

  1. まず、目的のサイトに移動する必要があります。
  2. PrintScreenキーを押してスクリーンショットを撮ります。
  3. スクリーンショットからテキストを慎重に切り取って保存します。
  4. 次に、画像からテキストを認識できるプログラムを使用する必要があります。 現在、人気のあるプログラムの1つにFineReaderがあり、これをオプションとして使用できます。 また、画像からテキストを非常にすばやく変換できるオンラインサービスも多数あります（たとえば、www.onlineocr.ru）。
  5. 変換後、あらゆるニーズに使用できるテキストを取得します。


おそらくこのリストは完全ではありませんが、4つの方法すべてを知っているので、システムを簡単にバイパスでき、テキストのコピーを防ぐことができます。
