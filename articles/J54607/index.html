個人的な経験：アパート内のホームネットワークの構成

おそらく、ここにいる人々の多くはすでに人生のこの段階を過ぎています。 私自身が遭遇したことを共有します。 誰かがこの情報が役立つことを願っています...;）





開始する



かつては、快適な仕事のために、自宅のPCに加えてラップトップを購入しました（2年前）。 特定のラップトップモデルを選択することは、別の長い話です。

この購入により、作業資料を常に手元に置いておき、必要に応じて自宅で仕事をする機会が与えられました。 さらに、機動性のおかげで、アパートのどこにいてもコンピューターを使用できるようになりました-都合の良い場所ならどこでも。



当時の私たちの都市のインターネットは多かれ少なかれ発展し始め、良い関税が現れ、都市の内部ネットワーク（プロバイダーの加入者の間）は急速に成長し始めました。 私の家でインターネットに接続するのは、ホームPCに延長されたネットワークケーブルを介して行われ、そのPCはプロバイダーのADSL回線に接続されました。



Wifi



しばらくすると、ケーブルでラップトップをPCにバインドするとモビリティが無効になるため、さらに快適になりたいという要望がありました。



Wi-FiをサポートするADSLルーターを購入することにしました。 選択はAsus WL-600Gで決着しました。



残念ながら、その時点で私のプロバイダーはPPTPを介したVPN接続を使用してのみインターネットに接続できました。 このルーターはこの機能をサポートしていないため、自宅のPCをプロキシサーバーとして使用する必要がありました。 それでも、その結果、必要なもの、つまりモビリティを手に入れました。 最後に、私はアパート内のどこにいても、同時にグローバルネットワークと連絡を取ることができました。 テクノロジーを長生きさせましょう！ :)



ネットワークの外観



1年後、ようやくADSL回線からイーサネット回線に切り替えることができました。 プロバイダーのイントラネットリソースに100メガビットでアクセスできる実際の専用ネットワークがありました。 ADSLと比較して、それは何かでした！ この時点で、無制限の関税がすでに私たちの街に現れ始めています。 イーサネットへの切り替えにより、ADSLルーターとしてのWL-600gの主な機能はその関連性を失いました。 通常のWi-Fiルーターとして使用されるようになりました（そして私はゆっくりと販売するためにそれを設置しました）。



さらに6か月後、新しいラップトップ、つまり妻のラップトップがアパートに現れます。 Windows XPを実行している自宅のPCと、Windows XPおよびWindows Vistaを実行している2台のラップトップから、内部のアパートネットワークが既に形成され始めています。



Linuxへの切り替え



この時点で、ラップトップには問題がなかったため、自宅のPCをLinuxに変換することを検討し始めました。私たちに完全に合ったライセンス版のオペレーティングシステムが使用されています。 私は、その瞬間に、サーバー上に海賊版のOSがあったと告白します。



さまざまなLinuxディストリビューションのバージョンから選択するときが来ました。 そして、私は彼らの多様性についても考えませんでした。 多くのフォーラムやトピックを読んだ後、選択できるのはUbuntu 8.10ディストリビューションに限定されていました。 当時のプロバイダーがYandexとのローカルネットワークにあったという事実のおかげで、 Linuxディストリビューションの素晴らしいミラーからシステムの最新のDVDディストリビューションをダウンロードし、自宅のPCにインストールすることができました。 今では完全なサーバーになりました。



PPTPを上げて、 squidとsambaをインストールした初心者はどれだけ苦しみましたか。 すでにLinuxに切り替えているプロバイダーのフォーラムのユーザーのヒントは本当に役に立ちましたが、素晴らしいリソースに言及することはできません。

  • Ubuntu Linuxのロシア語コミュニティ
  • Ubuntu初心者/ Ubuntuの新機能
  • Stepanoffホームネットワーク


新しい-忘れられた古い



ほんの数か月前、ロシアのかなり大きな国のプロバイダーが、以前のようになっていたのではなく、ようやくユーザーに向き合うようになりました。 手ごろな価格の適正な関税が登場しました。



ルーターを販売したことがないのは便利で、新しいプロバイダーがADSLを介してサービスを提供しています。 同時に、最大2 Mbit / sの無制限の速度が得られ、以前のプロバイダーからの以前の無制限の256 Kb / sよりも100ルーブルだけ多くなります。



結果



現時点では、私のアパートには次の相互作用スキームがあります。

  • Ubuntu 8.10のおかげでライセンスの心配がなくなったホームサーバー。
  • サーバーは、ネットワークカードを介して以前のプロバイダーのイーサネットネットワークに接続されています（開発された内部ネットワークのために残され、アクセスには月額150ルーブル）-主にファイルをダウンロードして交換します。
  • サーバーはイーサネット経由でルーターに接続されます。
  • ルーターは、ADSLを介してPPoE接続を自動的に上げることで新しいプロバイダーに接続されます（最終的にインターネットへの接続プロセスを自動化することが判明しました）。
  • ラップトップはWi-Fiを介してルーター（DHCP）に接続され、サーバーと同じネットワーク上にあり、相互に接続されています。
  • ラップトップは、ルーターを介してインターネットにシームレスにアクセスし、ホームサーバーを介して以前のプロバイダーの内部ネットワークにアクセスします。


もちろん、暗号化はWi-Fi用に構成され、NATはサーバー上で構成され（ソリューションを見つけるのにも時間がかかりました）、ルーティングはルーターでラップトップが以前のプロバイダーのネットワークに自由にアクセスできるように構成されます。 さらに、サーバーはインターネットにアクセスできるようにルーティングを構成しました。



その結果、現時点では、最大2 Mbpsの速度で無制限のインターネットを利用でき、アパートの隅から出ることができます。さらに、最大100 Mbpsの速度で以前のプロバイダーのネットワーク内でファイルを交換できます。



はい、私は欲しいものを手に入れました。 そして、これで多くのメリットはUbuntuです 。これは、Windowsで実装した場合、無料で実装できたものに追加の投資が必要になるためです（追加の必要なプログラムのインストールと「ライセンス」の要件を考慮）。
