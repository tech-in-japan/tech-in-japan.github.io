Visual Studio 2015 CTP6リリース

開発チームは、更新のペースについていくために、 Visual Studio 2015 CTP 6の6回目の予備リリースをご紹介します。 これに加えて、 Team Foundation Server 2015 CTPの更新も準備しました。 両方のアップデートを今すぐダウンロードできます 。 製品の予備バージョンをインストールする機会がなく、新しい製品に慣れたい場合は、Azureクラウドに既製の仮想マシンも用意しました 。 これらの予備リリースに含まれるニュースの詳細については、 Team Foundation Server 2015 CTPリリースノートおよびVisual Studio 2015 CTP 6リリースノートの出版物をご覧ください。





Visual Studio 2015 CTP 6



CTP 5のリリース後、組み込みのSend-a-Smileフィードバックメカニズムによる否定的なレビューが急増しました。 これは主に、多くのユーザーに影響を与えたPDBファイルのブロックなど、いくつかの本当に苦痛な問題が原因でした。

真実は、私たちのチームが私たちの仕事の結果をできるだけ早くあなたに提供したいということです、そして、あなたはあなたのフィードバックを与えることができます。 主に更新のリリース速度、チームの生産性、新機能の数、品質など、いくつかの要因の適切なバランスを見つけます。 これに基づいて、更新プログラムにある程度の品質を持たせることを計画していますが、残念ながら、CTP 5で予想されるよりも低いことが判明しました。

CTP 5を試してフィードバックをくれた方々に感謝します。 CTP 6の更新では、すべての要望を考慮に入れており、より良い安定したリリースが表示されます。

CTP 6では、多くの新機能も導入しました。これらの新機能をご紹介します。



シングル サイン イン 。 ユーザーから絶えず受け取っているレビューの1つは、Azure、Windowsストア、MSDNサブスクリプション、Visual Studio Onlineなど、仕事で多くのMicrosoftサービスを使用する開発者が、資格情報の頻繁な入力に頼らざるを得ないという事実に関連しています。 Visual Studioの機能のこの部分で作業を続けており、CTP 6ではこれらの機能が新しいレベルに引き上げられています。 これで、資格情報エントリの数が大幅に削減されました。最初のエントリの後、同じアカウントに登録されている他のサービスと連携するために資格情報が使用される可能性が高くなります。



新しい ASP 機能 。 NET Webテクノロジー開発チームは、ASP.NET 5の改善を続けています。CTP6では、ASP.NETランタイムカーネルのパフォーマンスの改善、New Web Projectダイアログの設定の変更、JSON構成ファイルのIntelliSence更新、およびbower、NPMのパッケージマネージャー機能、 NuGet。 これらの機能の詳細については、この更新プログラムに含まれるASP.NET更新プログラムに関するメモをご覧ください 。



Xamarin 統合の 改善 。 この更新には、XamarinプラットフォームとVisual Studioを統合するための新機能が含まれています。 特に、Androidプロジェクト用のC ++でライブラリを開発できるようになりました。 これらの機能の詳細については、 ネイティブXamarin Androidアプリケーションの開発方法に関するメモをご覧ください。 これに加えて、Visual Studioをインストールすると、Xamarinコンポーネントの構成が簡素化されました。



Apache Cordova 用の Visual Studioツール 。 過去数回のリリースで、Apache Cordovaアプリケーションのデバッグが可能なデバイスの数に関するレビューを受け取りました。 Android 4.4、Android 4.3、jsHybugger、iOS 6,7,8、Windows Store 8.1のサポートに加えて、CTP 6にはWindows Phone 8.1のサポートも含まれていました。



エミュレーターまたは実際のデバイスのWindows Phone 8.1プラットフォームで、ブレークポイントの設定、変数値の表示、コンソールを使用した式の評価、およびその他のデバッグ操作を実行できます。 詳細については、 Apache CordovaのVisual Studioツールを参照してください。



CodeLens 。 CodeLensテクノロジーのおかげで、アプリケーションのコードの編集という主要なタスクに焦点を合わせながら、簡潔なメトリックとメッセージの形でコードに関する詳細を見つけることができます。 この更新に伴い、C ++、SQL、およびJavaScriptコードのGitリポジトリにあるファイルバージョンに関する情報を表示できるようになりました。 TFSを使用する場合、コードのセクションとタスクおよびエラーとの関連付けに関する情報を表示することもできます。





建築ツール 。 コードマップツールキットの以前の実装のレビューでは、使用の複雑さが指摘されました。 新しいリリースでは、コードマップツールキットインターフェイスの応答性が向上し、アーキテクチャメニューが簡素化され、フィルタリングツールの操作性が向上しました。 プロジェクトのコードマップの初期構築にかかる時間が大幅に短縮され、アセンブリに基づくビルドモードに加えて、プロジェクト内にあるフォルダーに基づいて情報をグループ化できるようになりました。 このツールキットの詳細については、CTP 5のコードマップの改善されたメモをご覧ください。





NuGet CTP6とともに、開発者はパッケージおよびコンポーネント管理ツールのアップデートを受け取ります。 これらの改善は、主にパッケージ管理のインターフェース部分に影響を与え、適切なパッケージを検索して更新しました。

XAML UI デバッグ 。 Visual Studioには、いくつかのリリースのHTMLプロジェクトのDOMツリーを詳細に表示する機能があり、このツールはHTMLインターフェイスの開発における強力なツールです。 ユーザーの多くは、XAMLの同様のツールを求めてきました。 このような機会が現れたので、XAMLツリーを視覚化し、XAMLノードのプロパティと属性の値を確認し、XAMLインターフェイスを完全にデバッグできます。 DependencyPropertyオブジェクトを表示および変更することもできます。 これらの機能の詳細については、XAMLの新しいUIデバッグツールの記事をご覧ください。





CTP 6では、XAMLツリービューツールキットはデスクトップWPFプロジェクトでのみ機能します。 残念ながら、Windowsストアのアプリケーションはまだサポートされていませんが、Visual Studio開発チームはすでにこの機能に取り組んでいます。

デバッグ NET コード CTP 6は、さまざまな面でデバッグエクスペリエンスを向上させます。 まず、デバッグ中に多くの例外的な状況を生成する3番目のライブラリーを呼び出す場合のデバッガーのパフォーマンスを改善しました。 Visual Studio 2015で.NETコードをデバッグする際のパフォーマンスの改善から、これらの改善の詳細を学ぶことができます 。 編集と続行モードの改善も行われました。特に、以前のリリースでは利用できなかった新しい変更操作が追加されました。特に、反復子と非同期待機操作ではそうでした。 また、デバッグ中に、デリゲート、インターフェイス、列挙などの一部のデータ型を変更する「オンザフライ」機能を追加しました。 これらの機能の詳細については、 C＃編集および続行を参照してください。



また、CTP 6では、以前のリリースではしばらくの間欠落していたMake Object IDコマンドが再表示されました。 オブジェクトIDの作成メカニズムにより、デバッグセッション中にオブジェクトの特定のインスタンスにマーカー（タグ）を割り当て、このオブジェクトのスコープがない別のコンテキストに切り替えた場合でも、デバッグ中にその状態を追跡できます。 これにより、オブジェクト変数の値と、これらの値がこのインスタンスの存続期間を通じてどのように変化するかを表示できます。 これで、再び$ id構文を使用して、ウォッチとイミディエイトウィンドウでこれらのオブジェクトを参照し、条件付きブレークポイントを設定できます。

それに加えて、おなじみの例外設定ツールのインターフェイスは、今ではDebug-Windows-.Exception Settingsメニューで呼び出され、更新されました。 おそらく既に理解しているように、これはモードレスウィンドウであり、デバッグに関連する他のウィンドウと一緒に配置できます。 詳細については、例外設定ツールウィンドウの使用方法を参照してください。





Android用の Visual Studioエミュレータ 。 多くの開発者にとって朗報は、Visual Studio 2015がHyper-Vテクノロジーに基づいた高性能Androidエミュレーターを統合したことです。 CTP6アップデートには、Lollipop SDKで構築されたアプリケーションのデバッグ（APIレベル21）、OpenGL ES、マルチタッチシミュレーション（マルチタッチ）、カメラシミュレーションのサポートなど、多くの新機能が含まれています。 これらの機能の詳細については、Visual Studio Emulator for Androidの新機能をご覧ください。







フィードバック 。 通常どおり、 UserVoice WebサイトでVisual Studioに関するフィードバックやアイデアを受け取ることを本当に期待しています。これは、製品に実装されるものに直接影響を与えるユニークな機会です。 Visual Studio 2015自体に組み込まれている最も単純なフィードバックツール（ Send-a-SmileボタンとSend-a-Frowunボタン）も使用することを忘れないでください。 深刻な問題を見つけて、それを詳細に説明する時間があれば、 Visual Studio Connect Webサイトを使用してください。
