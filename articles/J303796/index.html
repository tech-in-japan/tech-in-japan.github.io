Dell Storage SC9000：効率的なデータセンター向けのインテリジェントストレージ

ストレージパフォーマンス要件の増加により、ベンダーは最適なストレージアーキテクチャを作成し、従来のドライブとともにフラッシュメモリを使用する新しいアプローチを模索しています。 サーバーとストレージシステムの開発は、同社が多額の投資を行っているデルの優先事項の1つです。 現在、さまざまなタイプの企業向けに新しいバックアップおよびストレージシステムを提供しています。 Dell SC9000ストレージシステムを詳しく見てみましょう。











ハードウェアプラットフォーム



新しいDell SC9000ディスクアレイは2015年10月にリリースされました。 このフラッグシップソリューションは、すでに世界中の主要なデータセンターで見られます。 しかし、このシステムは一見の価値があり、中規模の企業です。 マルチペタバイトの容量とStorage Center 7.0オペレーティングシステムを備えたSC9000は、IT部門に新しい機会を提供します。









SCシリーズのアレイの中で最も古いSC9000ハードウェアプラットフォームは、最高のパフォーマンスとスケーラビリティを特徴としています-アレイあたり最大960のSAS SSDおよび/またはHDDドライブ、およびアレイをグループにまとめることができます（フェデレーション）。



すでに信頼性が証明されているプラ​​ットフォームには、新しいIntel 8コアプロセッサが搭載され、システムメモリ容量は4倍になり、12 Gb / s SASインターフェイスがバックエンドとして使用されます。 システムは、SANプロトコル（FC、iSCSI、FCoE）、およびオプションのFS8600デバイスを介したNASをサポートします。









システムパフォーマンスは、以前のSC製品と比較して40％向上しました。 内部テストにより、SC9000アレイのパフォーマンスは1ミリ秒未満の遅延で36万IOPSを超えることが示されました。 さらに、スループットが2倍になりました。これは、負荷が増加してもボトルネックがなくなることを意味します。









Dell SC9000の中核は、3.2 GHzのクロック周波数と最大512 GBのRAMを備えた4つの8コアIntel Xeonプロセッサを備えた新しい第13世代Dell PowerEdgeサーバープラットフォームです。 システムは、フラッシュアレイまたはハイブリッドストレージとして構成できます。









冗長なホットプラグコンポーネントを備えたSC9000は、フェイルセーフで展開しやすい高可用性ソリューションです。 拡張シェルフSC400 / 420、SC200 / 220またはSC280では、容量を増やすことができます。 システムは、SC8000、SC4020、SCv2000アレイによる統合管理をサポートします。













フラッシュ容量は1Uあたり最大46 TB（データ圧縮なし）です。 この密度は、ストレージシステムでSamsung TLC 3D NAND 3レベルメモリを使用したことで実現しました。 SC9000システムは3 Pバイト以上の正味容量を提供し、ワークロードに大量のストレージが必要な場合はアレイをグループ化できます。









SC280拡張シェルフ（およびこれは96個の3.5インチHDD）は、1Uあたり100 TBを超える容量を提供します。 これは、メディアアーカイブまたは大きなファイルを扱うアプリケーションで使用できます。



ソフトウェアツール



ハードウェアプラットフォームの印象的な特徴にもかかわらず、SC9000はその品質の多くをソフトウェアに負っています。 自動化機能を備えたStorage Center仮想化ソフトウェア環境は、システムをより経済的で幅広いタスクに適したものにします。

フラッシュメモリを操作するための最適化の改善


フラッシュメモリの高コストがこのテクノロジーを採用する上での大きな障壁であることを考慮して、Dellはコストを削減するためにドライブ効率の改善に注力しています。 多くのベンダーは最適化を主張していますが、多くの場合、すべてのタイプのフラッシュメモリが同じように動作するストレージの1つのレベルで、単純化されたアプローチが使用されます。



デルの最新の仮想化テクノロジーにより、同じアレイ内の異なるタイプのフラッシュメモリを使用して、異なるワークロードと異なるアプリケーションシナリオを実行できます。 これにより、同社は完全にフラッシュメモリとハイブリッドバージョンの両方で、高速で経済的な一連のアレイをリリースできました。



デルの自動階層化では、さまざまなタイプのSSDのさまざまな価格/パフォーマンスパラメータが考慮されます。 つまり、IOPSで高いパフォーマンスを達成するために、法外な金額を支払う必要はありません。 そのため、Dellフラッシュアレイのギガバイトあたりのコストはわずか0.65セントです。 ハイブリッドストレージシステムでは、この数値はさらに低くなります。



フラッシュメモリに限定されないディープティアリング


3D NANDメモリの耐用年数を延ばすために、ティアリングメカニズム-ストレージ階層間でのデータの移動が必要です。









最初のレベル（Tier 1）は、多数の書き換えサイクルに耐えることができるSLCまたはeMLCフラッシュメモリで表され、2番目（Tier 2）は、比較的まれに使用されるデータが圧縮形式で書き込まれる3D NANDメモリで表されます。 一部のモデルには、3番目のレベル（Tier 3）-HDDがあります。









システム構成（さまざまなタイプのフラッシュメモリとハードドライブの容量の比率）は、負荷の性質を考慮して選択できます。これには対応するユーティリティがあります。 WI-集中的な書き込み、RI-集中的な読み取り



デルの階層化方法は、業界で最も包括的な方法の1つです。 SCアレイを使用すると、異機種システムからSANおよび/またはNASデータ用のユニファイドストレージプールを作成し、さまざまなタイプのメディア（Tier 0-サーバー側キャッシュを含む）からボリュームを作成し、柔軟なRAID構成を設定できます。 適切なレベルのストレージにリアルタイムでデータが自動的に配置されるため、データの「ウォームアップ」と「冷却」が加速されます。 しかし、主なことは、管理が簡素化され、パフォーマンスの観点からドライブからすべてを絞り込めることです。









すべてのレベルでのインテリジェントなデータ配置とストレージの最適化。



自動マルチレベルデータ配置の特許技術は、さまざまなタイプのドライブの主な利点を使用して、ライフサイクル全体を通してデータを最適化します。



新技術の迅速な導入


同じ仮想化により、SCシリーズアレイが価格とパフォーマンスの利点を実証できるようになり、新しいソリューションを導入できます。 良い例がTLC 3D NANDです。 昨夏、メインストリームリードインテンシブドライブを発売したことにより、デルは耐久性やパフォーマンスを犠牲にすることなく、ティア2フラッシュメモリのコストを半減しました。 TLCメモリは、ハイエンドアレイで正常に使用されていますが、競合他社は依然として下位クラスのシングルレベルシステムのアーキテクチャを変更しようとしています。



ストレージのニーズとパフォーマンスを満たす高速の垂直および水平スケーリング。


垂直および水平の同時スケーラビリティにより、SCアレイは十分に拡張可能です。



柔軟な自己最適化SC9000システムにより、3 Pバイト以上の正味容量を取得し、アレイをグループに結合して、大規模なワークロードに必要な容量を提供できます。 SC9000は、ストレージエリアネットワーク（SAN）およびネットワーク接続ストレージ（NAS）をサポートする高速モジュール拡張を提供します。 また、単一の制御で大規模な連合システム内の他のSCシリーズアレイに接続することもできます。



SCOSソフトウェアには、オプションの「ストレージハイパーバイザー」が含まれており、アレイの「フェデレーション」間でデータを簡単に移動できます。 また、1つのディスクアレイを構築する場合でも、統合管理により大規模な分散システムの容量とパフォーマンスを得ることができます。









Dell Volume Advisorは、事前定義されたルールに基づいて予防的な負荷分散の推奨事項を提供します。 彼はアレイの連携を監視し、データを配置するのに最適な場所をアドバイスします。



容量の効率的な使用


SCOS 7.0ソフトウェアを搭載したSCシリーズアレイは、フラッシュドライブとHDDでのインテリジェントなデータ圧縮をサポートしています。 これにより、容量が大幅に節約されます。 圧縮機能はティアリングと統合されているため、圧縮してもパフォーマンスは低下しません。 同時に、動的容量配分などの機能がアクティブになります。









データ圧縮により、ストレージ容量が大幅に節約されます。これは、フラッシュメモリにとって特に重要です。



Microsoft SQLデータベースの場合は93％に達するデータ圧縮により、ソリューションはさらに経済的になります。フラッシュアレイは、HDD SAS 15kベースのディスクアレイに匹敵する価格です。









圧縮は、データブロックがより低いレベルに移動されたときに発生し、それによりシステムパフォーマンスに対するデータ圧縮の影響が軽減されます。 アクティブなブロックを上位レベルに移動すると、それらは圧縮されたままになります。



継続的な可用性


Live Volume Auto-Failoverと呼ばれる新しいSCOS 7.0機能は、計画外のダウンタイムを防ぎます。 障害が発生した場合、Storage Centerは別のアレイのプライマリボリュームと同期されたバックアップボリュームに負荷をリダイレクトし、復旧手順を開始します。 管理者の介入は不要です。









ホストの再割り当てのおかげで、ユーザーとアプリケーションの操作に影響を与えることなくストレージを変更できます。









フェデレーションクラスター内のアレイ間でのサーバー透過的なボリュームの移動により、負荷が増大してもキャッシュと容量の使用率が向上します。



もう1つの注目すべきSC9000テクノロジーは、故障したLive Volumeアレイを自動的に交換する機能です。これにより、ダウンタイムなしで作業できます。 ディスクアレイ自体の手段を使用して、プラットフォーム間でボリュームを「ストレッチ」することができます。 その結果、データストレージアーキテクチャの再構築を必要としない、安価で災害に強いソリューションが作成されます。









自動フェイルオーバーを備えたLive Volume機能は、停電中もシステムの継続的な動作を保証し、アレイの電源を入れたときに自動的に高可用性環境を復元します。 追加のハードウェアまたはソフトウェアは必要ありません。



SCシリーズアレイのLive Volume機能は、クラウド環境に最適です。 フェイルオーバーを提供し、別のSCシリーズアレイ上の完全に同期されたボリュームに透過的に自動的に切り替えます。

アプリケーションレベルでは、スナップショットはOracle環境、Microsoft Azureプラットフォームに基づくクラウド環境、VMware Metro Stretch Clustersでサポートされています。 Azure Site Recovery機能は、Microsoftのクラウドインフラストラクチャにデータをバックアップします。 また、Oracle、Microsoft、およびVMware環境の統合保護も提供します。



Application Protection Managerソフトウェアスイートは、Oracle、VMware、およびMSデータベース環境のサーバーでデータの整合性を提供します。 オプションのFIPS認定のソリッドステートドライブと自己暗号化ハードドライブは、データの盗難、紛失、または不正アクセスから保護します。 また、組み込みの高度な機能には、システムリソースを管理する包括的な「微妙な」方法が含まれています。
