Windows 7のチューニング：単一アプリケーションのUACテストをバイパスする方法

UAC（ユーザーアカウント制御）は、Windows Vistaで最初に登場したMicrosoft Windowsコンポーネントです。 このコンポーネントは、コンピューターの不正使用から保護するために、管理者権限を必要とするアクションの確認を求めます。

実際には、セキュリティリクエストのポップアップウィンドウが非常に速く表示されるため、ユーザーは警告を理解せずにリクエストの承認を開始し、このコンポーネントの有効性全体がゼロになります。 次の論理段階は完全なシャットダウンであり、それに応じてマシンのセキュリティが低下します。 残念ながら、Windows 7の開発にもかかわらず、UACルール作成メカニズムは実装されていませんでしたが、必要に応じて手動で例外を作成できます。







そのため、Mirandaにはお気に入りのアセンブリがあり、起動時に常にフルアクセスの確認を求めます。 私は彼女の行動が合法であることを100％確信しているので、デフォルトで彼女に完全な権利を与えます。 これは次のように行われます。

  1. Windowsタスクスケジューラを起動します。

    これを行うには、[スタート]メニューの検索バー、コンピューターのコントロール、コントロールパネルの[管理]セクションのショートカット、または[実行]-> "taskschd.msc"を使用します。
  2. 新しいタスクを作成します。

    開いたウィンドウで、タスクの名前を指定し、「最高の権限で実行」の前にDawを置きます。 この状況での他のパラメーターは重要ではありません。
  3. 次に、「アクション」タブに移動して、「作成」をクリックします。
  4. アクションの種類-「プログラムを起動する」を選択し、実行可能ファイルへのフルパスを指定します。この起動はUACで確認しません。
  5. 「OK」をクリックして、アクションが追加されたことを確認します。
  6. この状況では、他のタブは私たちにとって重要ではなく、スキップしますが、私はオプションのチェックを外しました。これにより、3日以上かかるタスクが停止します。
  7. [OK]をクリックして、スケジューラでタスクが正常に作成されたことを確認します。
  8. 次に、タスクを開始するためのショートカットを作成する必要があります。
  9. オブジェクトの場所で、次のことを示します。

    C：\ Windows \ System32 \ schtasks.exe / run / tn "タスク"


    ここで、「名前」（引用符で囲む必要があります）は、新しく作成したタスクの名前です。
  10. 「次へ」をクリックして、新しいショートカットの名前を設定し、作成を完了します。


出来上がり！ このショートカットでアプリケーションを起動すると、UACは不要な質問をしなくなります。
