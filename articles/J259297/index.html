Vivaldi 1.0.190.2のアセンブリのインターフェイス設定など

みなさんこんにちは！



まず第一に、先週アセンブリの欠如をおaびします。 その理由は、開発者のメインチームが働くヨーロッパとアメリカでの休暇と、アメリカのオフィスでのチーム全体の集まりでした。今週は、ほぼ全員が集まって一緒に仕事をしました。 理由はありますが、特定の目的のために集まったものであり、それについては後日学習します。



それまでの間、Vivaldi 1.0.190.2ブラウザの次の週刊ビルドに新しいものを追加しました。 そして、おそらく最も顕著な変更（少なくとも外部的に）は、ブラウザーの開始ページ（または、Expressパネル）の外観をカスタマイズする機能が追加されたことです。 これで、ブラウザウィンドウの背景色とそれを置き換える画像の両方を選択または割り当てることができます。 さらに、配色のようなものが登場しました-選択した画像または背景で快適に表示されるように、開始ページの要素に暗いまたは明るいデザインを選択できます。







これらの設定、特に背景画像を安全に試すことができます-一部のオフィスでは、アニメーションの背景に抵抗して設定できませんでした。



インターフェイスに関する別の革新-このバージョンでは、実際のWebページのスケールに影響を与えずにスケールできます。







インターフェイス要素を任意の規模で定性的に見せるために、SVG形式のアイコンを使用するように切り替えました。 また、内部アセンブリにHiDPIサポートを既に追加していると言うこともできますが、いくつかの欠点を修正するときに、少し後に公開アセンブリに到達します。



次のニュースは、ブラウザウィンドウの左または右にタブを配置することに慣れている人を喜ばせます。 このような配置のタブでページのサムネイルをオフにする機能を追加しました。 ただし、タブを上に戻してからタブバーを開いてページのサムネイルを表示すると、この設定は強制的に失われ、タブを左右に移動すると再びサムネイルが表示されることに注意してください。 これがバグか機能かはまだわかりませんが、あなたと一緒にこの質問に答えることができます。



また、Macユーザー向けのもう1つの小さなお知らせとして、ブラウザーの更新を手動で確認できるメニュー項目がバージョンに表示されます。



今のところすべてです。 以下のリンクを使用して、新しいアセンブリをダウンロードできます。



  • Windows 32ビット
  • Windows 64ビット （実験的）
  • Mac
  • Linux RPM 32ビット
  • Linux RPM 64ビット
  • Linux DEB 32ビット
  • Linux DEB 64ビット
