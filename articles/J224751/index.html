良い一日、親愛なるハブラフチアン。 最近、私はHabrと共有したいいくつかの興味深い有用なツール/ライブラリ/イベントを見ました。



ダット





Dat-データパッケージ管理。 これは、ソースファイルを使用したgitと同様に、コラボレーションワークフローを構築するために大きなデータセットを共有できるツールです。 プロジェクトの現在のステータスはプレアルファであることに言及することが重要です。 



npm install dat -g




ジェスチャーキット





それほど興味深いUI / UXタスクにとっても非常に興味深いものです。 サービスインターフェイスを使用すると、任意のジェスチャを描画できます。その後、GestureKitはジェスチャのキーを生成し、それを使用してイベントハンドラにアクセスできます。 さらに、iOSまたはAndroidだけでなく、 ジェスチャーキット.jsのおかげでWebでも利用できます。 つまり、一般的に、これは異常なイベントの一種のプロバイダーです。



発言



Remark.jsでは、Markdownマークアップを使用してスライドショーを作成できます。 幅広い構文の色分けをサポートします。 出力は、解像度に適応し、さまざまな「タッチ」デバイスと互換性のあるレスポンシブスライドです。

Velocity.js





Velocity.jsは、JavaScript DOMアニメーションの概念を根本的に変えるjQueryプラグインであり、最終的には従来の$ .animate（）の代わりになるはずです。 問題は、Velocityの方がずっと速いということです。 これは、パフォーマンスだけでなく、人がアニメーションを視覚的に認識しているためです。 DOMアニメーションのほぼ全体が十分に滑らかではなく、少しがらくたであることを認めなければなりません。 この問題は、DOMの同期（ 「レイアウトのスラッシング」を防ぐ ）とクエリキャッシングの2つのトリックで解決され、DOMの呼び出し回数を最小限に抑えます。



また、David Walshのこのトピックに関する記事「CSS vs. JSアニメーション：どちらが速いですか？」



クイル





Quillは、有名なSalesforce.comの最新のリッチで互換性のある拡張可能なテキストエディターです。 WYSIWYGの多くの問題により、人々はより頻繁にMarkdownを使用し始めましたが、開発者のペンの言葉はWYSIWYGの古典的なアイデアを取り除くために設計されました。 クイルは実際には非常に柔軟で非常にモジュール化されています。 APIのドキュメントをご覧になると、プロジェクトがスマートな方法で記述されていることがすぐに明らかになります。
