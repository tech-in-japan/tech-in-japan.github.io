皆さん、こんにちは！ jQueryライブラリーの最初のリリースからほぼ10年が経過しました。私たちは、何世紀にもわたって古典からほこりを取り除くことにしました。 私たちは、このライブラリに関するフッサールバラードの第3版のリリースを考えています。









NodeとES6の時代に私たちを惹きつけるものを説明するために（私たちは在庫がたくさんあります）、前述の第3版の直後に発行されたCody Lindleyの記事を知ってください。



jQueryの役に立たないことについての話 は 長い間 終わっ ていないので、私はjQueryの基本的な価値を忘れてしまったという考えを文字通り取り除くことができません。 彼女に思い出させる時間。



この記事では、jQueryが何であるかを皆さんにもう一度伝えたいと思います。今日はjQueryが登場時よりも重要ではないからです。 その重要性の問題は、ブラウザのバグや欠落している機能ではなく、このソリューション全体の最初の目的（つまり、jQuery API自体）と相関している必要があります。 他の何かから始める場合、 絶対に必要ではないかもしれませんが、それでも強力で便利な抽象化が拒否される立場を取る危険があります。



jQueryの名誉を熱心に擁護する前に、その起源に戻って、jQueryとは何か、なぜjQueryが必要なのかをみんなが理解できるようにしましょう。



jQueryとは何ですか？



jQueryは、ブラウザーでHTML要素ノードを使用するとき、またはグラフィカルインターフェイスなしでブラウザーで作業するときに、スクリプトを抽象化、整列、修正、および簡素化するように設計されたJavaScriptライブラリ（つまり、 JavaScriptで記述されます）です。



だから：



  • ドキュメントオブジェクトモデルインターフェイス（別名DOM API）は抽象化されています。
  • ブラウザー間で存在するDOM 実装間のすべての違いは調整されます。
  • CSSおよびDOMに関連する既知のブラウザーのバグは修正されています。


これはすべて、ネイティブDOM APIよりもシンプルで不格好なAPIに変わります-ここにjQueryライブラリがあります。



ここで、「HTML要素をスクリプト化する」という意味を説明しましょう。 jQueryを使用すると、「。htmlドキュメント内の2番目のh2



要素を視覚的に隠す」などのタスクが完全に解決されます。 このようなタスクのjQueryコードは次のようになります。 



 jQuery('h2:eq(1)').hide();


この行をjQueryコードで見てみましょう。 最初に、 jQuery()



関数が呼び出され、それにjQueryライブラリーに特別な CSSセレクターを渡します。jQueryライブラリーは、HTMLドキュメント内の2番目のh2



要素を選択します。 次に、jQuery .hide()



メソッドが呼び出され、 h2



要素が非表示になります。 ここに、jQueryコードがどれほどシンプルで意味的に表現力があるかを示します。



次に、jQueryを使用していなかった場合に記述する必要があるネイティブDOMコードと比較します。 



 document.querySelectorAll('h2')[1].style.setProperty('display','none');


書く方が便利なオプションはどれですか？ 読み取りとデバッグ？ また、上記のネイティブDOMコードは、すべてのブラウザーがここで使用されるDOMメソッドをサポートしていることを前提としていることに注意してください。 ただし、一部の古いブラウザーはquerySelectorAll()



またはsetProperty()



サポートしていないことが判明しています。 したがって、上記のjQueryコードはIE8で正常に動作し、ネイティブDOMコードはJavaScriptエラーをスローします。 しかし、それでも、両方のコード行がどこでも機能したとしても、どちらが読みやすく、書きやすいでしょうか？



jQueryを扱う場合、どのブラウザが何をサポートしているのか、どのブラウザのどのDOM APIがクラッシュするのかを心配する必要はありません。 jQueryは多くの問題を抽象化するため、jQueryを使用すると、よりシンプルなコードでタスクを解決するための作業を高速化でき、心配する必要はありません。



jQuery = JavaScript？



jQueryはどこにでもあるので、JavaScriptがどこで終了してjQueryが始まるのか、おそらくわからないでしょう。 多くのWebデザイナーや初心者のHTML / CSS開発者にとって、jQueryライブラリーはJavaScriptプログラミング言語との最初の接触です。 したがって、jQueryはJavaScriptと混同されることがあります。



まず、JavaScriptがjQueryではなく、DOM APIそのものでもないことを予約しましょう。 jQueryはJavaScriptで記述されたサードパーティの無料ライブラリであり、開発者のコ​​ミュニティ全体でサポートされています。 さらに、jQueryは、HTML、CSS、またはDOM仕様を作成する組織（W3Cなど）の標準ではありません。



jQueryは主に砂糖として機能し、DOM APIの上で使用されることに注意してください。 この砂糖は、その複雑さと豊富なバグで有名なDOMインターフェースと連携するのに役立ちます。



jQueryは、HTML要素のスクリプトを記述するときに使用できる便利なライブラリです。 実際には、ほとんどの開発者はDOMスクリプトでこれに頼っています。なぜなら、そのAPIにより、より少ないコードでより多くの問題を解決できるからです。



jQueryライブラリとそのプラグインは開発者によって非常に広く使用されているため、このようなコードは多くの場合、Web全体で最も人気のあるスクリプトとして賞賛されています。



jQueryの2つの基礎



jQueryのベースとなる2つの基本概念は、「検索して実行」と「書き込みを少なくして、より多くを実行」です。

これらの2つの概念は、次のステートメントとして拡張および再定式化できます。jQueryの主なタスクは、選択肢を整理（つまり、検索）するか、実用的な問題を解決するためのHTML要素を作成することです。 jQueryがなければ、これにはより多くのコードとDOMを処理するより多くのスキルが必要になります。 要素h2



して上記で検討した例を思い出すだけで十分です。



jQuery機能の範囲はこれに限定されないことに注意してください。 ネイティブDOMインタラクションを抽象化するだけでなく、 XMLHttpRequestオブジェクトを使用して非同期HTTPリクエスト（いわゆるAJAX ）も抽象化します。 さらに、 多くのサポートJavaScriptソリューションと小さなツールもあります 。 しかし、jQueryの主な利点は、HTMLスクリプティングが正確に単純化されることと、jQueryで作業できることです。



また、jQueryの利点はブラウザーのバグを正常に除去することではないことも付け加えます。 基礎はjQueryのこれらの側面とはまったく関係ありません。 長い目で見れば、jQueryの最大の強みは、そのAPIがDOMを抽象化することです。 そして、この値はどこにも行きません。



jQueryと最新のWeb開発の融合方法



jQueryライブラリーはすでに10年前です。 Web開発の時代のために作成されたもので、私たちはすでに間違いなく合格しています。 jQueryは、DOMの操作や非同期HTTP要求の作成に不可欠なテクノロジーではありません。 jQueryでできることのほとんどは、jQueryなしでもできます 。 また、1つまたは2つの最新のブラウザーでDOMとの小さな簡単な対話に興味がある場合は、jQueryよりもネイティブのDOMメソッドを使用した方がよい場合があります 。



ただし、表面的な相互作用だけでなく、BOM（ ドキュメントブラウザーモデル ）またはDOMに関連する開発では、jQueryを使用する必要があります。 それ以外の場合は、ホイール（jQuery抽象化の要素）を再発明し、あらゆる種類のトラック（モバイルブラウザーやPCブラウザー）でそれを体験します。



経験豊富な開発者は、「巨人の肩に立つ」ことの意味と、過度の複雑さを回避するタイミングを知っています。 ほとんどの場合、HTMLやDOMに関連する重要な作業を短時間で実行する必要がある場合でも、jQueryなしでは実行できません。



さらに、たとえjQueryがDOMまたはDOM仕様のさまざまなブラウザー実装に関する単一の問題を解決しなかったとしても、HTMLスクリプティングにとって非常に便利であるため、API自体の重要性は減りません。



さらに、jQueryは改善されており、その助けによりプログラマーはよりインテリジェントに、より速く作業できます。 これは今日の状況であり、図書館の作成時でした。 「jQueryは必要ありません」と言うことは、「lo-dashまたはunderscore.jsがなくてもできる」と言うことと同じです。 もちろん、それらなしでも実行できます。 しかし、その価値はこれによって判断されません。

それらの価値はAPIにあります。 複雑すぎるため、開発が遅くなる場合があります。 そのため、lo-dashやjQueryのようなものが好きです-それらはすべて単純化されています。 また、jQueryを使用すると、複雑なタスク（HTML用のスクリプトの作成など）を簡単に実行できるため、時代遅れになることはありません。



jQueryが最新のWeb開発に必要かどうかまだ疑問がある場合は、ライブラリの開発者の1人による次のプレゼンテーションを見ることをお勧めします。彼は、 最新のブラウザの機能に関係なく、その必要性を正当化します 。



付録-jQueryについての重要な事実



最後に、jQueryについてのいくつかの重要な事実をリストします。



  • jQueryライブラリーはJohn Resigによって作成され、2006年8月26日にリリースされました。Johnは、「JavaScriptとHTMLの相互作用を革命化する」ためにこのコードを作成したことを認めました。
  • jQueryは、最も人気があり、人気のある最新のJavaScriptライブラリと見なされています。
  • jQueryは、MITの下でライセンス供与されるフリーソフトウェアです 。
  • jQueryには2つのバージョンがあります。 バージョン1.xはInternet Explorer 6、7、および8をサポートし、2.xはIE9 +のみをサポートします。 IE8のサポートが必要な場合は、バージョン1.xで作業する必要があります。 しかし、これは正常であり、両方のバージョンがまだ積極的に開発されています。
  • jQuery 2.xの最小バージョンは82kbです。 Gzipアーカイブ-約28k 。
  • jQuery 1.xの最小バージョンのサイズは96kbです。 Gzipアーカイブ-約32k。
  • JQueryソースコードはGithubで入手できます。
  • Githubのソースコードに基づいて、 独自のバージョンのjQueryを作成できます 。
  • jQueryはbowerまたはnpmパッケージマネージャーを使用してインストールできます（つまり、 $ bower install jquery or npm install jquery



    ）。
  • JQueryには、公式のCDNネットワークがあり、さまざまなバージョンのjQueryを入手できます。
  • jQueryにはシンプルなプラグインベースのアーキテクチャがあるため、誰でも独自のメソッドを追加できます。
  • jQueryには広範なプラグインのセットがあります。 エンタープライズ開発用の高品質のプラグイン（Kendo UIなど）を購入するか、クールな無料のプラグイン（Bootstrapなど）を使用できます。
  • jQueryは分類できます（APIドキュメントの内訳に従って）。
