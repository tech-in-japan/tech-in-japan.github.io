仮想インフラストラクチャの開発方法：4つの主な傾向

RightScaleのスペシャリストは昨年、調査を実施しました。その間に、さまざまなプロファイルおよび規模の企業のITスペシャリスト数千人に、クラウドテクノロジーの実装に関する組織の取り組みについてインタビューしました。



本日の資料では、レポートから最も重要なIaaSの傾向を示し、昨年クラウドエコシステムで場所を見つけた他の傾向について説明します。





/ Flickr / ステファノ・メネゲッティ / cc



IaaSはクラウド市場で最も急成長しているセグメントです



調査会社のGartnerによると、2017年にIaaS市場の価値は36.6％急増し、350億ドルに達しました。これにより、このセグメントはすべてのクラウドサービスの中で最も急成長しました。 同時に、この傾向は継続すると予想され、この数値は2020年には560億に増加します。



ロイターは 、OrbisResearchの推定によると、2017年末までに、グローバル企業の40％が能力をクラウドに移行すべきだったと指摘しています。 さらに、この方向で最も急成長している地域は、ラテンアメリカ、ブラジル、および中東およびアフリカの国でした。



ロシアでは、IaaS開発の割合が高いことも確認されています。 CNewsランキングの結果は、2016年の結果によると、ロシアの大手プロバイダー10社の収益が76％増加し、38億ルーブルから67億ルーブルに増加したことを示しています。 1クラウドもトップ10に含まれており、2015年から2016年の期間で最高の収益成長率である259％を記録しています。



ハイブリッドインフラストラクチャの上昇



RightScaleの調査によると、調査した企業の95％はクラウドテクノロジーを使用しています。 それらのほぼ90％がパブリッククラウドを使用し、72％がプライベートクラウドを使用しています。 同時に、2017年には、プライベートクラウドへの関心がわずかに減少しました-最大72％（2016年のレポートでは77％）。





/ RightScale 2017 / CC



同時に、市場の42％を占めるVMware vSphereは、プライベートクラウドを整理するためのプラットフォームのリーダーであり続けています。 次に、OpenStack（20％）およびVMware vCloud Suite（19％）が続きます。





/ RightScale 2017 / CC



プライベートクラウドへの金融投資の量を削減したため、企業はこれらの資金を異なる方向、つまりハイブリッドクラウドに送りました。 2016年、ハイブリッドソリューションを実装する企業の数は58から71％に増加しました（WiseGuyReports機関による）。



2017年に関しては、マルチクラウドインフラストラクチャの実装を希望する大企業の数は85％でした。 さらに、これらの組織の半数以上がハイブリッドインフラストラクチャを構築しようとしています。 2016年の23％と比較して、プライベートクラウドのみを使用し続けている企業はわずか9％です。



クラウド内のワークロードの分布を見ると、アプリケーションの41％はパブリッククラウド環境にあり、38％はプライベート環境にあります。





/ RightScale 2017 / CC



スケーラビリティはクラウドの重要な利点です



RightScaleの回答者は、クラウドドライバーをスケーラビリティ-61％-および可用性-56％と名付けました 。 ITスタッフの効率性やインフラストラクチャコストの低下などの要因はわずかに重要性が低くなり、クラウドの利点の中で、回答者の35％がそれを指摘しています。



「VM構成をいつでも変更し、新しいマシンを追加し、古いマシンを削除する機能により、ユーザーはビジネスの現在のニーズに合わせて仮想インフラストラクチャのコストを最適化するほぼ無限の機会を得ることができます」





/ RightScale 2017 / CC



企業へのクラウドテクノロジーの導入を妨げる「主な恐怖」については、クラウドインフラストラクチャのセキュリティの低さやコストの予測の難しさなどの一般的な神話（IaaS領域の神話に関する別の一連の資料を準備しました-最初の部分はここで入手可能です ）は依然として問題でした回答者のわずか25パーセントでしたが、1年前にはこれらの数値はより高く、それぞれ29パーセントと26パーセントでした。



DevOpsの人気が高まっています



DevOps方法論は広く普及しており、クラウドアプリケーションの開発における一種の標準となっています。 一般に、過去1年間で、企業の78％がDevOpsメソッドに注目していました。 大企業では、この数値は84％に達しました。



大企業は徐々にDevOpsを実装し、個々の開発チームから始めて部門全体に移行しています。 ただし、その年には、企業全体で方法論の普及率が21から30パーセントに急上昇しました。



DevOps方法論を習得する際、企業は多くの場合、サーバーの展開と構成プロセスを自動化および標準化するツールを実装することを決定します。 これらには、構成管理アプリケーション（PuppetやAnsibleなど）、コンテナーテクノロジー（Docker）、およびオーケストレーションシステム（Kubernetes、Swarm、Mesosphere）が含まれます。



Dockerは最も人気のあるDevOpsツールになりました。 35％の企業がそれを実装しており、32％がそれを計画しています。 Dockerの最も近い「競合他社」はChefとPuppetで、それぞれ28％の「マーケット」を持っています。



過去1年間で、Kubernetesの有病率も増加していることに注意してください。 実装の数は7パーセントから14パーセントに増加しました。





/ RightScale 2017 / CC



さまざまな規模の企業におけるDockerの使用率を推定すると、大規模な組織の40％がこのツールを使用し 、30％がこのツールを実装する予定です。 中小企業については、ここではドッカーの30％の「シェア」があります。 このセグメントの企業の33％が彼との仕事を開始する予定です。



中小企業の中でも、Kubernetesは5〜12％の良好な成長を示しました。 Googleの回答者の21％がこのオープンなソリューションを使用することを計画しています。 同時に、ますます多くの企業がKubernetesとの連携をサポートするツールを作成し始めています。最近リリースされたユーティリティのVMware Dispatchは、Habrのブログで書きました。



DockerとKubernetesが人気を博している間、ChefとPuppetは視聴者を失いました。 最初の企業は7％を失い、21％の企業がそれを使用し、2番目の企業は6％を失いました（2017年は19％）。



一般的な結論



2017年の調査では、ハイブリッドインフラストラクチャが企業にとって最も好ましいクラウド移行フォーマットであることが示されています。 そして、組織はプライベートクラウドにますます注意を払っていません。



クラウド企業に移行する主な動機は、インフラストラクチャの維持コストとスケーリングの容易さの削減を検討することです。 移行を制限する主な要因については、経験とセキュリティの不足が残っていますが、回答者の25％のみがそれらを示しました（2016年のそれぞれ32％と29％と比較）。



DevOpsに関しては、大企業全体に適切なツールの導入が徐々に行われています。 2017年、Dockerがリーダーになりました。 また、Kubernetesは強力な成長を示しました。このようなフレームワークは、Google、Red Hat、IBM、Microsoftなどの大規模な組織で既に実装されているため、このテクノロジーの人気のさらなる成長が期待できます。
