企業データベース-2016：ホットテン

組織でデータを保存および処理する必要がある場合、そのサイズに関係なく、クラウドまたはローカルDBMSなしでは実行できません。 今日は、企業データベース市場の主要な代表者、2016年に注目する価値のある開発についてお話します。



企業のDBMS市場は数十年にわたって存在していました。 特定のシステムを評価するとき、その歴史を考慮することは、とりわけ有用だと考えています。 しかし、市場の成熟は、今日が静かな場所であることを意味するものではありません。 ここでの競争のレベルは非常に高いです。



ビジネス要件は絶えず増大しており、ITインフラストラクチャを作成するアプローチは変化しています。 DBMS開発者は、市場に必要なものを提供するか、周辺にいることに気付きます。 そして、データベースユーザーは、この選択のおかげで、この状況の恩恵を受けるだけです。



これは、企業のDBMSを選択するのが簡単だと言うことではありませんが、これから説明する主要なシステムの中から、興味のあるシステムを見つけることができると確信しています。



Oracleデータベース









Oracleは、1979年に最初のリレーショナルデータベースをリリースしました。 同社が市場に存在する長年にわたって、「Oracle」という言葉は「企業DBMS」および「データを使用した信頼性の高い作業」の概念と同義語になりました。 ちなみに、オラクルは開発中のフォーチュン500リストに含まれていますが、オラクルの主要な活動分野は強力でかなり複雑なデータベースソリューションです。



会社のDBMSの現在のバージョンはOracle 12cと呼ばれます。 文字cはクラウドを意味します。 これは、クラウドへのOracleの動きを反映しており、組織がデータベースをクラウドサービスとして統合および管理できるようにします。 Oracle DBMSの機能には、マルチテナントアーキテクチャ、ソリューションの高速展開、メモリ内のデータを処理する機能があります。



Microsoft SQL Server









Microsoftについては何でも言うことができますが、Microsoftは世界で最も収益性の高いテクノロジー企業の1つです。 そのデスクトップオペレーティングシステムは文字通りどこにでもありますが、Microsoftの成功にはSQL Serverが重要な役割を果たしました。 SQL ServerなしでMicrosoftのOSを実行しているサーバーを想像するのは困難です。



SQL Serverの使いやすさ、可用性、およびWindowsファミリーのOSとの緊密な統合により、Microsoft製品を企業目的で使用する企業にとってSQL Serverは明らかな選択肢となります。 Microsoftは、オンプレミスおよびクラウドデータベース、およびビジネスインテリジェンスシステムのプラットフォームとしてのSQL Server 2016の最新リリースについて語っています。



さらに、Microsoftは、組織がミッションクリティカルなオンライントランザクション処理（OLTP）アプリケーションの構築を支援できるソリューションとしてSQL Server 2016を推進しています。 このようなソリューションの特徴は、高性能、メモリ内のデータを処理し、保存および移動中にデータを保護する機能です。 SQL Server 2016に基づいて、大容量ストレージ、データ分析システムを作成できます。



MS SQL Serverにはさまざまなレンタルオプションがあります 。 1人のユーザーのライセンスを購入でき、ユーザーの数を制限せずにサーバーコアのライセンスを取得できます。



SQL Server 2016 Express の無料のエントリレベルエディションをダウンロードすることもできます 。これは、小規模なデータベースを運用環境に展開し、低コストのVPSにインストールするのに最適です。これは、最大10 GBのディスク領域を必要とするデータ処理用の小さなサーバー側アプリケーションを作成するのに十分です。

今年の4月に、DBMSの最新バージョンであるDB2 11.1がリリースされました。 多くのシステム、特にLinux、Unix、Windows、IBM z Systemsメインフレームで動作し、Power 8プロセッサーでハードウェアアクセラレーションをサポートします。



SAP ASE









Sybase DBMSの最初のリリースは、1987年にSQL Serverという名前でリリースされました。 彼は、Adaptive Server Enterpriseで育ちました。これは、同社の最も有名で成功した製品です。 2010年、SybaseはSAPに吸収されました。その結果、今日のデータベースはSAP Adaptive Server Enterprise （SAP ASE）と呼ばれています 。 Sybaseは別の会社の一部になりましたが、そのDBMSは依然として企業市場の主要なプレーヤーの1つです。 さらに、Sybaseは、モバイルエンタープライズソリューションに多大な努力を注いでおり、通常は有望な製品の開発者を買収しています。 どうやら、SAPはこの伝統を続けています。



DBMSの最新リリースであるSAP ASE 16は、パーティションレベルのロックテクノロジー、SAP HANAおよびSAP Business Suiteとの統合をサポートしています。 このデータベースは、費用対効果、スケーラビリティ、およびシステム監査機能で注目に値し、動的なスレッド割り当て、スター接続によるクエリプランの最適化、およびその他の多くの機能をサポートしています 。



PostgreSQL









PostgreSQLは、1986年に開発が開始されたPostgresの後継であり、無料のオープンソースのオブジェクトリレーショナルデータベース管理システムです。 オンラインカジノ、データセンターオートメーションシステム、ドメインレジストリなどの非常に興味深い場所でアプリケーションを見つけます。 さらに、高負荷のタスクYahoo!で使用されます。 とスカイプ。 PostgreSQLは、pr索好きな目から隠されている多くのシステムで見られるため、「Secret Corporate Database」と呼ばれることもあります。



PostgreSQLの現在の安定リリースは9.6です。 今年9月29日にリリースされました。



PostgreSQLは多くのオペレーティングシステムをサポートしています。 その中には、Linux、Windows、FreeBSD、Solarisがあります。 PostgreSQLは、Mac OS X Lion以降、AppleのOSで標準データベースとして使用されています。 このDBMSの機能は、OracleおよびIBMの機能に匹敵します。 特に、トランザクションの信頼性に関するACID要件への完全な準拠が特徴であり、高い並列負荷をサポートできます。



MariaDB Enterprise









MariaDB Enterpriseは、GPL、LGPL、またはBSDライセンスの下でリリースされた完全にオープンソースのデータベース管理システムです。 MariaDBの道は2009年に始まりました。 このシステムはMySQLのフォークであり、開発者のコ​​ミュニティはMySQLの作成者の指導の下で作業しました。 彼らは、MySQLを買収したOracleのライセンスポリシーを心配して、新しいプロジェクトを開始しました。



MariaDBはMySQLのおかげで人気を博しました。 特に、MariaDBが有名なLinuxディストリビューションでMySQLに取って代わりました。 そのため、2013年にのみ、Red Hat Enterprise Linuxの開発者はMariaDBを支持してMySQLを放棄しました。Fedora19でも同じことが行われ、openSUSEとSlackware LinuxはMariaDBに切り替わりました。 さらに、MariaDBはWikipediaプロジェクトのサーバーデータベースとして使用されるようになりました。



MariaDBをMySQLよりも優れたものにしたもう1つの主な要因は、高度なクエリオプティマイザーとその他の速度の改善です。



MariaDB Enterprise Serverの最新リリースは、MariaDB Enterprise Spring 2016としても知られる10.1です。このリリースは、アプリケーションおよびネットワークレベルでの攻撃からのデータ保護を改善し、新しい高性能アプリケーションの開発に役立ちます。



MySQL









MySQLは開発者向けの高度に専門化されたソリューションとして始まりましたが、企業のDBMS市場の主要なプレーヤーの1つに成長しました。 最初は2008年にSun Microsystemsに売却され、2009年にはOracle帝国の一部となりました。 長年、MySQLは単なるニッチソリューションではありません。 数十万の商用WebサイトがMySQLで動作し、膨大な数の社内企業アプリケーションのサーバーデータベースとして機能します。



現在、MySQLはWebソリューションの非常に人気のあるオプションであり、LAMPスタックの中心的なコンポーネントとして機能し続けています。 同時に、MySQLはOracleの買収により、ユーザーや独立した開発者からのサポートが弱くなっています。



MySQLの人気の低下により、他のオープンソースDBMSと、 Perconaや前述のMariaDBのような、完全にオープンソースのコードを備えた独自のフォークの適応を加速することが可能になりました。これには、MySQL Enterprise Editionの新しいバージョンで導入されたクローズドモジュールはありません。



このDBMSの最新リリースであるMySQL Community Server 5.7.15は、2016年8月上旬にリリースされました。



Teradataデータベース









Teradataについて聞いたことがありますか？ 大規模なデータウェアハウスを作成している場合は、おそらく聞いたことがあるでしょう。 Teradataは、1970年代後半から物語をリードしてきました。 その後、彼女は後に「データウェアハウス」として知られるようになったソリューションに取り組み始めました。 1992年、TeradataはWal-Martの最初のテラバイトデータベースを構築しました。 それ以来、企業のストレージシステムの専門家による会話では、「Teradata」という言葉が絶えず言及されてきました。



Teradata Databaseの機能は非常に大規模なデータベースです。 このシステムは、ビッグデータの調査、ビジネスインテリジェンス、モノのインターネットなどの最近のトレンドをサポートするのに最適です。 Teradataは、2015年初頭にリレーショナルデータベース管理システムのバージョン15.10をリリースしました。



IBM Informix











ご覧のように、リストにはIBMの別の開発があります。 同社は、 Informixオブジェクトリレーショナルデータベース管理システムに幅広いオプションを提供しています。 複数のプラットフォームに展開できるインテリジェントなデータベースとして宣伝されています。



多くの場合、教育機関に関連して、Informixは企業の世界に入り、顧客満足度で1位になりました。 このDBMSのユーザーは通常、その低コスト、少量のメンテナンスと高い信頼性の必要性を表明するために非常にお世辞です。



アングル









Ingresリレーショナルデータベース管理システムは、企業市場に非常に関連性があり、さらに、データベースの世界では古くからの1つです。 その作業は1970年代初頭に始まりました。



Ingresには魅力的な価格設定モデルがあり、これはシステムの総所有コストの削減を意味する場合があります。 このシステムは、より高価なDBMSからシステムに切り替えるための高度な機能を誇っています。 さらに、HIPAAおよびSarbanes-Oxleyに準拠するために必要な高度な安全機能を備えています。

Ingresの最新の企業バージョンは10.2です。 彼女は2015年に登場し、地理空間データ型、リモートGCAクライアント、UTF-8音訳、DBMS認証およびその他の新機能のサポートを導入しました。



2016年6月、Ingres 11 Technical Previewがテスト用にリリースされました。



Amazon SimpleDB









これはリストの11番目の項目です。 メイン10品のデザートと考えることができます。

一見すると、「Amazon」と「DBMS」の概念は完全に互換性がないように見えるかもしれませんが、実際、特に今日のクラウドコンピューティングの時代にはそうではありません。 SimpleDB （Simple Database Service）は、組織に従来のDBMSに代わるシンプルで柔軟かつ手頃な価格の代替手段を提供します。



SimpleDBを使用すると、ユーザーはデータを保存し、Webサービスのリクエストを通じてデータを操作できます。 スケーラビリティ、高速、最小限のメンテナンス要件、および他のAmazonサービスとの統合を特長としています。 SimpleDBを無料で始めましょう。
