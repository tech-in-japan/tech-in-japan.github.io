「非クラウド」価格の「クラウド」サービスを探して：Oversun-Scalaxyをテストします

Habréでは、さまざまな「クラウドホスティング」が何度か議論されましたが（ 最近はclodoについて書いています ）、 Oversan-Skalaksiへの言及は見つかりませんでした 。 信頼性が高く高速なホスティングプロバイダーの永遠の検索で、私はLinode.comに決めました 。 このホスティングは非常に信頼性が高く、高速で、価格も優れています。 ただし、ロンドンのDCはかなりの距離にあり、pingは常に満足されるわけではありません（結局、すべてを「光の速さ」にしたい）、それはスケーリングに最も便利な方法ではありません。



私たちのプロジェクトのいくつかはOversan Mercuryでホストされています。 McHostの問題にもかかわらず、Oversan-Mercuryは非常に信頼性が高いことが判明しました。半年間の作業で問題が発生したのは2件でした（そのうちの1件はここで、2件目は夜でした。 水星、作業速度、ping、価格はすべて素晴らしいです...一般的に、価格の小さなプロジェクトに適している可能性は低く、個人で動作するかどうかはわかりません。



ごく最近、ネットワークの広がりを歩き回って、Oversan-Skalaksiからクラウドのホスティングに出会いました。 かつて彼らのウェブサイトにアクセスしたことがありますが、特定の価格情報を見つけることができなかったため、しばらく忘れていました。 そして、突然あなたと価格、そして登録、そしてテストする機会が現れました。 考え直すことなく、私は登録しました。 正直なところ、コントロールパネルにはうれしい驚きがありました。



それでは、Oversan-Skalaxiが提供するものを見てみましょう。 メインページで最初に目に付くのは、「スライダー」です。これにより、ホスティングコストを大まかに計算できます。







電力はスロットで計算され、1つのスロットには次の特性があります。



プロセッサー-Intel Xeon 5550（16コア）

RAM-512 MB

スワップ-1 GB

ネットワーク速度（外部）-5 Mbps



メインディスクのサイズを5 GBにすると、1か月に約516ルーブルになります。これは非常に良いことです。 比較のために、Linode 512の料金（RAM 512MB、ディスク上の16GB）は19.95ドルです。 Skalaxiでは、同じ特性（ディスクを16 GBに増やす）のサーバーのコストは603.99ルーブルです。



登録後、アカウント全体にコントロールパネルと250ルーブルがあり、このすべてをテストできます。 このお金がすぐになくなるのは、あなたとあなたのニーズだけです。

したがって、「 Schema 」コントロールパネルの最初のセクションは、最初は次のようになります。







[サーバーの追加]ボタンをクリックする前に、IPアドレスを追加する必要があります。 これは、[ IP＆DNS ]タブで簡単に実行できます。







ところで、同じセクションで逆ゾーンとドメインを追加できます。 以下のDNSマネージャーについて。 それまでの間、「 スキーマ 」セクションに戻って、新しいサーバーを追加します。







デフォルトでは、スライダーは16スロットにありますが、1スロットをテストする必要があったため、選択しました。 第2段階では、 OSを選択します 。







現時点では、3つのOSのみが利用可能です（ Debian 5.0.6 、 Ubuntu 10.04 LTS 、 Centos 5.5 ）。これは、私がDebianサポーターなので絶対に私に合っています。 サーバーを作成すると、「 スキーマ 」セクションは次のようになります。







サーバーアイコンの状態に応じて、サーバーアイコンの配色に満足しました。







サーバーの電源を入れた後、SSH経由でサーバーに接続し、必要なことは何でもできます。 私はほとんど忘れていましたが、これはクラウドでホスティングされているため、簡単にスケーリングできるはずです。 そうです。 「 スキーム 」セクションからアクセスできる「 スケーリング 」セクションには、「 自動スケーリング 」と「 時間スケーリング 」の2つのスケーリングオプションがあります。 「自動スケーリング」では、リソースの増減に応じてルールを作成できます。 そこで、スケーリングが発生するステップを設定することもできます。



「 ステップが必要なものの説明 」はおかしいように見えます。 ステップの目的の説明 」、みんな、どうやらすべてが整頓されているわけではありませんが、なぜ必要なのかはわかっています=）







残念ながら、RAMで何が起こるかに応じてのみスケーリングできます。 しかし、ディスク容量がなくなったらどうなりますか？ 一般的に、疑わしい間。 サーバーの設定は次のとおりです。







そこで、サーバーの電源を変更し、必要に応じて新しいディスクを追加し、サーバーをオフにして再起動します。



ディスクを操作するときに問題が発生しました。 つまり、作業ディスクのサイズを増やす方法はありません。 つまり / var / wwwフォルダーに接続されているドライブ/ dev / sda3（たとえば10GB）があり、そのスペースが不足している場合、ディスク容量を増やすことはできません。 より大きなボリューム（20 GBなど）で新しいディスクを作成し、すべてのデータをコピーして、新しいディスクを/ var / wwwフォルダーに再バインドする必要があります。 行くぞ あまり便利ではありませんが、機能します。



技術サポートはまた、「将来」彼らが間違いなくそれを修正することを約束しました。 「未来」がすぐに来ることを望むことしかできません。 技術サポートといえば。 彼女は24時間、2〜3時間以内に十分に迅速に応答します。

さらに進みましょう。 コントロールパネルには、「 チャート 」などの他のタブがあります。







サーバーは完全に新しいため、興味深い情報はありませんでした。 「 IP＆DNS 」タブについては、もう少し上に書きましたが、組み込みのDNSマネージャーを見るほうが面白いです（はい、これは提供されています）：







一般的に、DNSマネージャーは見た目は良いのですが、それほど印象的ではありませんでした。 レコードの種類はグループ化されていますが、それでも、少し厄介に見えます。 さらに、サブドメインの名前は完全に記述されており、値のみを見ることに慣れています（たとえば、 @ .test.ruではなく@ ）。 さらに、サブドメインの値は、レコードのタイプに応じて、異なる場所、左側のどこか、中央のどこかにあります。 何か間違ったことをしたかもしれませんが、 primary-ns1.scalaxy.ruからドメインに関するデータを取得できませんでした。 たとえば、2番目のもの-ns3.scalaxy.ruおよびns4.scalaxy.ruは、nslookup要求に正しく応答しました。







さて、DNSマネージャーはそのままにしておきましょう。とにかく使用することはないでしょう。 次のタブ（「ユーザー」）で、他のユーザーにさまざまな権限を付与できます。







「 金融 」タブも同様におもしろいです。 ここには、グラフ、GBウォッチ、およびスロットウォッチがあります。







「 サーバー 」、「 ディスク 」、「 IPアドレス 」のセクションが開き、そこに何がどのくらい償却されたかを詳細に見ることができます。 数字を見ないで、サーバーとディスクを作成し、それらを削除し、2つのIPアドレスを購入してから1つ削除したので、そこに1つのIPが表示され、コストは50ルーブル（2など）です。 一般に、テスト済み。 ところで、支払いは法人、個人の起業家、または個人が行うことができます。 支払いは（個人用の）ロボカッサを通じて行われるため、問題はないはずです。 外国のホスティングの支払いが問題である人にとって、これは大きなプラスのように見えるかもしれません。



さて、最後のタブは「 サポート 」であり、これは時々非常に必要な要素です。 上で書いたように、私は技術サポートについて肯定的な印象を持っていましたが、これまでのところ複雑な技術的な問題はなかったため、この場合のサポートのしくみを言うことはまだできません。 時間はわかります。







プラスはおそらく理解できるので、すぐにマイナスになります：



-最も便利なDNSマネージャーではない（主観的に）

-ディスクを動的に拡張する方法はありません（私にとってこれは非常に重要です）

-ディスク容量に応じて自動スケーリングルールを設定することはできません

-追加ディスクの1GB時間の価格は、メインディスクのコストよりも高くなっています（ PPSを参照）。



実際、2日間のテストの結果に基づいて言うことができます。 Skalaxiは喜んでいますが、取り組むべきことがあります。 正直なところ、新しいホスティングプロバイダーが登場していることを非常に嬉しく思います。 これが競争につながり、競争が価格の低下、安定性と信頼性の向上（結局、ホスティング事業者自身がこれに関心を持っている）、そしてサービス品質の向上につながることを願っています。
