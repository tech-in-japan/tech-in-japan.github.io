True J​​ava Elastic Cloud @ JavaOne-クローズドベータテスト

Java言語のすべてのロシア語開発者と同様に、4月12日にモスクワで開催されるJavaOne会議を楽しみにしています。 私たちにとって、このイベントは重要です。 この日のために、Java用の弾性クラウドソリューションであるJelasticのベータ版を準備しました。

テストのこの段階で誰がジェラティックなのか：ワンクリックでエラスティッククラウド環境にアプリケーションをデプロイしたいTomcatとMySQLを使用するJava開発者向け



競合他社と比較したJelasticの重要な特徴は、垂直スケーリングの可用性です。



クローズドベータテストの主な目的は、実際の作業プロジェクトで垂直スケーリングのパフォーマンスをテストすることです。

垂直方向のスケーリングと水平方向の違いのいくつかをリストします。

  • アプリケーションロジックを変更する必要はありません
  • ノード間でデータを複製および同期する必要はありません
  • ノード間の要求/負荷のバランスを取る必要はありません
  • 水平スケーリングよりも安価（オーバーヘッドが少なく、バランシング用の特別なノードがない）
  • 高速スケーリング時間（水平スケーリングよりも大幅に高速）
  • かなり大きな潜在的なスケーラビリティ係数x136（256mから32Gbまで）
  • より細かいスケーラビリティステップ（64Mb）
  • OutOfMemmoryErrorの確率ははるかに低い


同時に、いつものように、多くの欠点があります。

  • スケーラビリティの制限の可用性（32Gb）
  • フェイルオーバー-回復力の低いアーキテクチャ（単一ノードがクラッシュすると、アプリケーション全体にアクセスできなくなります）


垂直スケーリングに加えて、次のような他のアルゴリズムの動作をテストします。

  • アプリケーションの便利な展開メカニズム（戦争展開）、バージョン化された展開
  • QA（品質保証）-アプリケーションの環境のさまざまなバージョンの管理（開発、テスト、実稼働など）
  • タンブラーメカニズム-Webサーバーが重大なエラーでクラッシュすると、システムによって自動的に発生します
  • その他


情報ヘルプ



現在、クラウドへのアプリケーションの100％の移植性を提供する3つの主要なJava PaaSクラウドソリューションがあります。

  • AWS Beanstalk -Amazonソリューション（Amazon EC2を使用）
  • CloudFoundry.com -VMWareのソリューション（Amazon EC2を使用）
  • CloudBees.com -400万ドルの投資があるスタートアップ（Amazon EC2を使用）


ご覧のとおり、すべての競合他社はAmazon EC2に何らかの形で強く結び付いています。

Jelastic-特定のデータセンターのIaaSに結び付けられていないため、深刻なデータセンターにソリューションを展開できます。



もう1つの重要な特徴的な機能は、オペレーティングシステムレベルの仮想化ソリューションであるParallels Containersの使用によるリソースのより良い利用です。 Parallels Containersは、Platform as a Serviceの実装に最適であり、XenやVMWareと比較して高密度でオーバーヘッドが低くなっています。



アクセス方法
