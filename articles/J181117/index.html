Atomのスマートフォン。 成果と展望



Intelがモバイルデバイス向けプロセッサの市場に「再来」してから約1年が経過しました。この期間は、広範囲にわたる結論を定式化するには明らかに十分ではありませんが、すでに理解できます。 この投稿では、今年達成したこと、インテルのモバイル製品がどの方向に開発されているか、そしてそれらが近い将来にどうなるかを見つけようとします。



肉眼でのプラットフォームの成功の最も目に見える尺度は、そのベースでリリースされた市販製品の数です。 私たちの場合には何がありますか？ 現在、Medfieldプラットフォームに基づいたスマートフォンは約12個あり、CloverTrailプラットフォームに基づいたWindowsタブレットは約20個あります。 それはたくさんですか、それとも少しですか？ 市場シェアの観点から-もちろん、悲惨。 そして、これらのモデルをモバイルデバイスの新しいエコシステムを作成するための最初のステップと考える場合、それで十分です。 主なものは、技術が紙とプロトタイプだけに存在するように残っていなかったということです。それは「生きている」鉄で具体化されます、そして、それを実行してその間違いを修正する何かがあります。 当然、プロジェクトが商業的に成功するには、最初、2、3、20代が最初のモデルに従う必要があります。これは、消費者にとってより高度で魅力的です。 モバイル戦線でのIntelの攻撃計画については、次のようにグラフィカルに見えます。







生産的なAndroidスマートフォンとタブレットのMedfieldプラットフォームは、2013年にCloverTrail +に置き換えられます。 予算のスマートフォンは、レキシントンに基づいてリリースされます。 これは、いわば、目先の見通しです。 より遠く-22nmプロセステクノロジーへの移行と、新世代のプロセッサ（より正確にはSoC）のメリフィールド/ベイトレイルのコア数の増加。 強力なタブレットとハイブリッドトランスデバイスは、Intel Core i3 / i5 / i7プロセッサの恩恵を受け、消費電力を削減する予定です。

次に、Medfieldの現在のスマートフォンの特性を、Clover Trail +のフォロワーと比較します。

  • 改良されたカメラ-上記の表に示されているものに加えて、オートフォーカス、撮影時の遅延なし、顔検出などがあります。


Intelベースのスマートフォンの幼年期の病気の1つは、主にマルチメディアなどの多くのAndroidアプリケーションとの不完全な互換性でした。 今、これはどういう状況ですか？ 例外なく、DalvikのすべてのアプリケーションはAtomで「現状のまま」動作します。リリースを準備する各システムは特別にテストされています。 ネイティブAndroidアプリケーション（NDK）もIntelで正常に動作するはずですが、x86で単純に再コンパイルすると、速度が著しく向上します。 この問題の多くは、開発者自身に依存しています。 たとえば、アプリケーションが特定のARM関数を使用する場合、x86で正しく実行するには、SSEセットから同等の関数をそれらに追加する必要があります。 この問題で開発者を支援するために、IntelにはARM NEONからIntel SIMDへの特別な自動トランスレータがあります。

Androidマーケットのトップエンドアプリケーションに関しては、Intelはプラットフォームで作業を個別にテストする必要があります。 これはナビゲーションアプリケーションにも当てはまります。特に注意が必要です。 開発者は、Intelを利用してプログラムの最適化を支援することもできます。



開発ツールについて話す場合、特定の進歩もあります。 x86プラットフォームのサポートがAndroid SDKに追加されました-SDKマネージャーにx86システムイメージをインストールするだけです。 さらに、Intelの仮想化テクノロジーVTプロセッサーを使用するIntel Hardware Accelerated Execution Managerを使用しているため、エミュレーターでのAndroidアプリケーションの実行は大幅（最大4回）です。 ところで、VTサポートはLinuxでも利用できるようになりました。 最後に、電話でアプリケーションをテストするために、インテルはプログラムのパフォーマンスと消費電力を最適化する無料ツール-Intel GPA System Analyzerを提供しています。 リアルタイムユーティリティは、CPUメトリックス、グラフィックスAPI、グラフィックスコア、および電力のグラフを描画します。これにより、プログラムを適切にプロファイルできます。

プラットフォームに合わせてAndroidを最適化するインテルの取り組みは、数字と事実で簡単に測定できます。 たとえば、Intelは外部参加者の間でAndroid 4.2コードに最大数の変更を加えました。

または別の結果：同社はWebKit Webエンジン用に750以上のパッチを準備しました（Googleに続く2番目の結果）。 現在、本格的なのは、新しいクロスプラットフォームツールと開発ツールの作成です。
