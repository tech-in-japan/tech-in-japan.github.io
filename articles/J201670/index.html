SassyStudio-Visual StudioでのSCSS / SASS / Compassのサポート

まず、SassyStudioはMITの下でライセンスされており、プロジェクトコードはGitHubでホストされています 。 ほぼ毎週、拡張機能が更新され、新しい機能で更新されます。 拡張機能は、Visual Studioツールギャラリーにあります。



現在の機能



  • 構文の強調表示
  • 文書内の領域の構造化および折りたたみ/展開、およびフォーマットのサポート
  • 部分的なIntelliSenseサポート（デフォルトでは無効になっており、拡張設定に含める必要があります）
  • CSS生成




CSS生成について


SassyStudioは現在、CSSを生成する2つの方法をサポートしています。PCで特に設定する必要がない内部メカニズムと、Rubyを使用して注意を引く基本的なCompassサポートの外部メカニズムです。



内部のメカニズムはクイックNSassで 、IronRubyの形でフードの下に脂肪はありません（ Mindscape Web Workbenchで行われます ）。 Rubyツールキットを使用するには、設定を見て、Rubyディストリビューションがインストールされているフォルダーへのパスを指定する必要があります（sassおよびcompassパッケージをインストールする必要があります）。 そこでは、設定にIntelliSenseの実験的サポートが含まれていますが、これはデフォルトでは無効になっています。







バッハ、そしてコンパスのサポートを設定してください！







組み込みコンパイラを使用する場合、すべての処理は、処理中のファイルのディレクトリで行われ、そこで準備が整ったCSSファイルが表示されます。 デフォルト設定では、標準と最小化の2つのファイルがそこに表示されます。

解説


テーマのあるフォルダーがあるASP.NET MVCに基づくプロジェクトがあります。 各テーマには、config.rbおよびsass-watch.cmdファイルを含む独自のフォルダーがあります。 SCSSファイルコンパイラの観点から見ると、実際にはプロジェクト内のプロジェクトです。 したがって、テーマを開発するときに、sass-watch.cmdファイルを実行しました。 いくつかのテーマの並行開発により、これらのファイルのいくつかを実行しましたが、これは作業環境をかなり混乱させ、時には混乱を引き起こしました。



外部コンパイラでSassyStudioを使用すると、sass-watch.cmdファイルおよび同様のバッチファイルの必要性がなくなりました。 作業中、config.rbファイルは自動的に互いに独立して取得されます。 編集および保存のためにメインスタイルシートを開きます-コンパイルは自動的に行われます。 また、 インポートで指定されたファイルが変更されると 、自動コンパイルが発生します 。

合計



SassyStudioは、Mindscape Web Workbench（有料版）よりも機能が劣っています。 一方、SassyStudioの主要なビジネスは100％完了しています。これは、コンパイル、構文の強調表示による編集、操作の速度（Mindscape Web Workbenchはこの点では非常に反応しません）、部分的ではありますがIntelliSenseです。 いずれにせよ、SCSS / SASS + Compassバンドルのスタイルライブラリのレベルで効果的に作業するには、今のところWebStormを選択することをお勧めします。 そして、SassyStudioの開発に参加して 、このツールを誰にとっても完璧なものにすることができます！
