典型的なWindowsロールの調整。 パート1：ファイルと印刷







Windowsサーバーとその典型的な役割のパフォーマンスのチューニングに関する短いシリーズの記事を始めます。 この資料は、（赤でペイントすることに加えて）古いサーバーを最大限に活用しようとする場合、および上位サーバーを購入せずに新しい高負荷システムを計画する場合（インテグレーターの助言による）の両方に役立ちます。







一般的な鉄の推奨事項



プロセッサはサーバーの心臓部のようなものであるため、パフォーマンスの点で多くのプロセッサに依存しています。 マーケターのおかげで、コアとメガヘルツが多いほど良いことを知っています。 実際、これはまったく真実ではありません。







  1. 64ビットプロセッサを選択します。 最新のサーバーWindowsは32ビットプロセッサをサポートしていないため、より多くのメモリに対応できます。







  2. コアの数はそれほど重要ではありません。 すべてのアプリケーションとサービスが複数のコアを使用できるわけではありません。一般に、頻度の高い1つのコアは、小さいコアの2つのコアよりも効果的です。







  3. ハイパースレッディング-ハイパースレッディング-1つの物理プロセッサコアが2つの論理的なものとして定義されている場合。 プロセッサ機能により、同じコアで2つの異なるスレッドを処理できるため、一般的に生産性が向上します。 ただし、プロセッサコアキャッシュが1つしかないため、逆にパフォーマンスが低下することがあります。







  4. プロセッサキャッシュ。 ここではすべてが単純です。サイズが大きいほど優れており、多くの場合、キャッシュが大きいほどプロセッサの周波数よりも高いパフォーマンスが得られます。







  5. 異なる世代やメーカーのプロセッサを周波数で比較する必要はありません。データ処理の速度は、キャッシュやバスの周波数など、他の多くの要因にも依存します。







  6. Hyper-Vの場合、プロセッサがSLAT（Second Level Address Translation）をサポートすることが重要です。 Intelの用語では、この機能はAMD-Nested Page Tables（NPT）の拡張ページテーブル（EPT）と呼ばれます。 systeminfo.exeユーティリティを使用して、このプロセッサ機能の存在を確認できます。








Hyper-V要件のプロセッサ検証。







RAMを使用すると、すべてが非常に単純になります。つまり、より大きく、より高速であるほど優れています。 十分なRAMがなく、システムがページファイルを使用する必要がある場合は、少し面白くなります。 ここでは、次の推奨事項に制限できます。







  • スワップファイルは別の物理ディスクに配置する必要があります。 速いほど良い。 これが不可能な場合は、ファイル呼び出しが少ないディスクに配置することをお勧めします。







  • RAID1のようなフォールトトレラントアレイにスワップファイルを保存しない方がよいでしょう。1つのドライブの速度が低下します。 RAID0にスワップファイルを配置するか、異なる物理ディスクに複数のスワップファイルを配置することをお勧めします。








システムドライブにページファイルを配置することは適切なオプションではありません。







次に、ネットワークアダプタについて説明します。 興味深い機能には次のものがあります。







  1. 64ビットシステムをサポートするアダプタのみが、ネットワーク経由でメモリに直接アクセスするためのテクノロジであるDMA（Direct Memory Access）を備えています。 クラスタノード間に非常に高速なネットワークが必要な場合は、これに注意する必要があります。







  2. マルチポートアダプタは、負荷分散とフォールトトレランスに便利です。 ただし、1つのポートを持つ2つのアダプターは、2つのポートを持つ1つのアダプターよりも高速です。 また、サーバーのコアよりも多くのネットワークアダプターをインストールすると、パフォーマンスが低下することに留意する必要があります。








HPE ProLiant DL360 Gen7には 、4つのネットワークポートが付属しています。







これで小さな紹介が終わり、サーバーの役割の最適化に直接進みます。 最も単純なものから始めましょう-ファイルサーバーで。







ファイルサーバー



通常、ファイルサーバーをインストールして実行する場合、パフォーマンスは問題になりません。 ただし、データベース、巨大なExcelファイルなどの「興味深い」ものが通常のファイルウォッシュで起動されている場合に限ります。 SMBのパフォーマンスを向上または低下させる可能性のあるパラメーターについて説明します。







別に、ローカルネットワーク内のクライアントだけでなく、たとえばVPNを介したリモートクライアントも処理するサーバーパフォーマンスの問題に注意します。 個人的には、Windows XP \ 2003のネットワークにWindows 7 \ 2008のコンピューターが表示され始めるという状況に遭遇しました。 それから、新しいコンピューターのネットワークの速度が、古いOSと通信するときに望まれるものを多く残しているという事実に直面しました。 インターネットを読んで、新しいマシンで次のスクリプトを実行しました。 







netsh int tcp set global autotuning=disabled netsh int tcp set global autotuninglevel=disabled netsh int tcp set global rss=disabled chimney=disabled


ネットワークは機能し、スクリプトは展開可能なシステムイメージに追加されました。







また、ネットワークスピードに対する需要が高まり、リモートセグメントがネットワーク上に現れるまで、すべてが正常でした。 ファイルは2 Mb / sより速くVPN経由で転送されませんでした。 問題はローカライズされていました。新しいオペレーティングシステムのLAN \ WANネットワークでの作業専用にオートチューニング機能が追加されたことが判明しました。 これにより、システムは接続速度を決定し、最適なパフォーマンスのためにTCPフレームのサイズを一致させます。 VPNが迅速に機能し、WINDOWS 2003にアクセスするときにサーバーの速度が低下しないようにするには、 autotuningを無効にせずに、コマンドに制限するだけで十分です。 







 netsh int tcp set global autotuninglevel=highlyrestricted


しかし、さらに専門的なパラメーターに移りましょう。







レジストリ設定とサービス設定を変更すると、何にでもつながる可能性があることに注意してください。 したがって、私たちはすべてを注意深く行います。

ファイルサーバーについて直接話すと、いくつかの一般的な推奨事項があります。







  1. パフォーマンスを向上させるには、ファイルシステムのミニフィルター、IPSec、NTFS暗号化と圧縮、SMB暗号化などの不要な機能を使用しないでください。 ウイルス対策を有効にすると、パフォーマンスが大幅に低下する可能性があります。ネットワーク境界が保護されている場合は、インストールしない方が良いでしょう。







  2. ドライバー、特にネットワークカードの関連性を定期的に確認する必要があります。 曲がったドライバーのために、ネットワークカードが100メガビットの強制設定でのみ安定して動作する状況がありました。 そして、新しいドライバーがリリースされて初めて、完全なギガビットを引き出すことができました。







  3. ファイルのコピーは、ファイルサーバーのかなり通常の操作です。 ネットワーク経由でコピーする場合は、robocopy.exeユーティリティと/ mtスイッチを使用することをお勧めします。これにより、多数の小さなファイルのマルチスレッド化が可能になります。 コンソールへの出力がないため、robocopyの場合は/ logスイッチ、xcopyの場合は/ qのパフォーマンスが少し向上します。
