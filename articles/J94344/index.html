Exchange Server 2010に基づくメールホスティング-無料？！

Infoboxによる新しいプロジェクトを発表したいと思います。



しばらくすると、Infoboxはすべてのクライアントメールボックス（100,000以上）を新しいメールプラットフォームに移行し始めます。



発表の中で、どのように、なぜ、なぜ伝えるのか。



メールは、世界中のすべてのホスティング会社にとって、ホスティングサービスに不可欠な属性です。 したがって、何千ものクライアントのメールボックスがあります。

原則として、ホスティング事業者は無料のソフトウェアを使用してメールサービスを提供します。 たとえば、メールシステムとして、さまざまなメーカー、特にPostfix、Courier、Round Cube、Squirelの製品を使用しています。 Postfixはsmtpを介したメールの受信/転送/送信を担当し、受信メールをメールボックスに配置し、courierはpop3とIMAPを介して作業を提供し、最後の2つはメールにアクセスするためのWebクライアントです。



残念ながら、私たちのクライアントは、他のホスティング会社のクライアントと同様に、電子メールの操作に関連する問題について定期的に不満を言っています。 問題の中には、レターの配信の遅延、スパム対策およびウイルス対策の非効率的な動作、1つのフォルダーに数千のメッセージがある場合の動作が遅いこと、電子メールWebクライアントが常に正しく動作しないことがあります-ロシア語、添付ファイル、連絡先、大きな文字など 現在、2つのWebクライアントを使用していますが、それぞれに独自の欠点があるため、理想的なソリューションを見つけることができませんでした。



同時に、顧客は多くの場合、メールホスティング業者と無料サービス（Yandex、Mail、GMailなど）の仕事を比較し、有料メールが無料より悪い理由を尋ねます。 「無料」メール「シェアウェア」、およびホスティング事業者の「有料」メール「シェアウェア」と見なす予約をすぐに行いたいと思います。 なんで？ 無料のサービスが広告で稼ぐのは秘密ではありませんが、これらの収益は、非常に高価でそれに応じて安定したインフラストラクチャを展開および回収できるほどです。 実際、ホスティング事業者にとって、メールはサイトの価格に含まれており、その支払いは非常に条件付きです。 これは一種の追加サービスであり、おそらく関税の価格で考慮されますが、非常に条件付きです。 ホスティング事業者が通常提供する無制限の数のボックスを配置する場合、1ボックスのコストについて話すことは困難です。 もちろん、これらはホスティング事業者の「問題」です。 しかし、いずれにせよ、共有ホスティングに150〜300ルーブルを支払うことに慣れているのは顧客であり、500ルーブルを支払う理由はこれ以上ないでしょう。 そして、欧米のホスティング事業者にとって、共有ホスティングでのメールは中断することなく機能するとは言いません。 ）））

そして、ホスティング事業者が本当に安定したメールサービスを提供しているが、たとえばExchange Hostingのように有料の場合、ユーザーは1箱あたり370ルーブルは高すぎると言います。 そして、それは悪循環に変わります。 メールの支払いはなく、無料サービスのような広告もありません-高価なインフラストラクチャはなく、インフラストラクチャもありません-高品質はありません。 ただし、30を超えるメールストアを含む約100のメールサーバーがあるため、インフラストラクチャが弱いとは言えません。



その結果、メールの問題がおそらくホストを変更する主な理由です。 クライアントは、最高のメールを探してホスティング業者間を急ぎます。 そして実際、彼はそれをうまく行っていない。



聖cra式の質問「何をすべきか」を尋ねる時です。

何かを根本的に変える必要があります。 まず何が欲しいですか？

たとえば、次の方法でメールシステムの速度を上げるには：



個々のレターを個別のファイルに保存しないメールサーバーを使用します。 100文字からヘッダーを読み取ると、ハードディスクで100〜200回検索されます。

メールサーバーは、頻繁な接続が手紙のリストなどの基本情報を常に読み取らないように、ユーザーデータをメモリにキャッシュできる必要があります。 この場合、ファイルシステムレベルでのキャッシュは効果的ではありません。 新しい文字を読み書きすると、キャッシュが常に上書きされます。

また、メールを操作するための真に信頼性の高いバグのある多機能Webインターフェイスをお客様に提供したいと考えています。



問題のほとんどを一挙に解決するソリューションが見つかりました-これはMS Exchange Server 2010です。私たちにとって、このソリューションは表面にありました。 2年間、私たちはこの製品のスムーズな動作を検証し、シックなWebインターフェイスやその他の利点を評価することに成功しました。 ところで、Infoboxは3年前からExchange Serverを企業のメールシステムとして使用しています。



すべてのお客様にビジネスクラスのメールを使用してほしいため、すべてのメールをこのプラットフォームに転送することにしました。

しかし、主な問題は、特にそのような規模で、商用ソフトウェアのライセンスを支払うコストが下がることです。 私たちは、お客様が以前に支払った以上にサイトの配置とメールの代金を支払いたくないことを理解しました。 したがって、これらの費用を自分で負担することが決定されました。

何のため？ 計算によると、クライアントメールを提供するサーバーの数は減少し、これらのサーバーのホスティングと保守のコストを削減できます。 なぜなら 独自のデータセンターを使用しているため、サーバーの数を減らすとスペースが解放され、スペースと電力のコストがさらに削減されます。

もちろん、これはすべての新しいコストを回収するわけではないという事実にもかかわらず、次のような追加投資の準備ができています。 より良い郵便サービスは、既存の顧客の忠誠心を高め、新しい顧客を引き付けると確信しています。



もちろん、これは370ルーブルのビジネス顧客向けの既存のサービスの完全な類似物ではありません。 これは、有料ボックスに比べて機能が多少制限されたソリューションになります。 MAPIプロトコルの機能は無効になります。 次の機能はありません-ActiveSync、BlackBerry、無料のデスクトップOutlook、共有フォルダー、共有の連絡先、その他のコラボレーション機能。 しかし、すべて同じように、ユーザーは以前のホスティングメールとまったく同じ機能を利用できます。これは、IMAP、POP3、SMTP経由で機能し、多機能OWA Webインターフェイス（Outlook Web App）にアクセスする機能です。 この場合、メールは非常に安定したExchange Serverプラットフォームでホストされ、クライアントに余分な費用はかかりません。



お客様にとってのもう1つのプラスは、ActiveSyncサポート、無料のOutlookプログラム、メールボックスへの音声アクセス、および同じプラットフォーム上にある他の機能を備えたフル機能のメールボックスへのホスティング内の透過的な移行の可能性です。



技術的な観点から、これはロシアのために、さらには世界の実践のために実施するためのユニークで非常に興味深いプロジェクトです。 単一のホスティングプラットフォーム内でExchange Serverに100,000を超えるメールボックスを展開しているプロバイダーは世界には多くありません。 また、新しいExchange Server 2010での非常に多くのメールボックスの作業についての先例はほとんどありません。
