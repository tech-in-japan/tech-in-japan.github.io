データの暗号化と圧縮



すべてのエディションは、投稿で説明されているように、ソース側でのバックアップ暗号化をサポートしています 。 さらに、Veeam Backup＆Replicationのように、 圧縮レベルとブロックサイズを調整できます（ ストレージ最適化 ）。



これらのオプションはすべて、[ 詳細設定 ]ダイアログの[ ストレージ ]タブで利用できます。タスクウィザードのステップで[ 宛先]の横にある[ 詳細 ]ボタンを押すと開きます。







これらの設定を使用すると、パフォーマンスとリソース消費の適切なバランスを実現できます。



圧縮レベル



動作の概要：Veeamは、ソースからブロックを読み取り、選択したレベルに従ってブロックに圧縮アルゴリズムを適用します。 ブロックサイズは、ストレージ最適化の設定によって決まります。 すでに圧縮されたブロックがリポジトリに書き込まれます。



可能なオプション：



  • なし （圧縮なし ）-たとえば、組み込みの圧縮および重複排除を使用してバックアップをストレージシステムに保存する場合に使用できます。
  • 重複排除に対応（ストレージの重複排除に最適化）-プロセッサーの負荷を軽減します。 大きな処理能力を必要としない単純なRLEアルゴリズムを使用します。 リポジトリとして組み込みの重複排除を備えたストレージシステムでの作業に最適です。
  • 最適 （推奨）がデフォルト値です。 LZ4圧縮アルゴリズムがアクティブになります。 出力では、約半分圧縮されたブロックが得られますが、プロセッサーの負荷はわずかに上昇します。 出力のバックアップファイルのサイズと、圧縮および解凍中の処理に費やされる時間との最適なバランスが保証されます（データ処理速度は、1コアで150 MB /秒からです）。
  • 高 （高）-圧縮率が10％増加します（最適レベルと比較して）が、プロセッサーの負荷が増加する場合があります。 まだ最大ではないプロセッサー負荷を考慮して、zlib圧縮アルゴリズムが適用されます。 データ処理速度-1コアで最大100 MB \秒。
  • 極端 （非常に高い）-出力は可能な限り小さいサイズのバックアップを生成します（圧縮率にさらに3-5％を追加します）が、パフォーマンスは非常に大幅に低下します（処理速度-1コアで約50 MB /秒）。 プロセッサの非常に高い負荷を考慮して、zlibアルゴリズムが「ねじれた」状態で使用されます。 このオプションを使用する場合は、強力なプロセッサ（6コアの種類）を使用することを強くお勧めします。 一般に、極端な圧縮はめったに使用されません-原則として、非常に低い帯域幅を持つチャネルがリモートリポジトリに行く場合。


Habréのこの記事で圧縮率について詳しく読むことができます。

ストレージに基づいたブロックサイズの最適化



Veeam Agent for Microsoft Windowsでは、リポジトリを編成するストレージシステムに応じて、処理するデータブロックのサイズを構成できます。 以下にいくつかのオプションを示します。



  • ローカルターゲット（16 TB +バックアップファイル） -ローカルストレージと大きなバックアップファイル（16 TBから ）の場合に推奨。 ブロックサイズは4096 KBに設定されます。 ブロックが大きいほど、その数は少なくなり、それに応じてメタデータテーブルも小さくなり、処理に必要なストレージスペースとコンピューティングリソースが少なくなります。 ただし、これにより増分バックアップファイルのサイズが大きくなることに注意してください。
  • ローカルターゲット -SAN、DAS、またはローカルストレージにバックアップする場合に推奨されます。 ブロックサイズは1024 KBに設定されます。 最高のバックアップ速度を提供します。
  • LANターゲット -NASおよびローカルサイトのリポジトリにバックアップする場合に推奨（オンサイトバックアップ）。 ブロックサイズは512 KBに設定されます。 増分バックアップファイルのサイズを縮小できます。
  • WANターゲット -リモートサイトへのバックアップにWANを使用する場合に推奨。 ブロックサイズは256 KBに設定されます。 WAN経由のトラフィックはわずかです。


注：データ圧縮設定はいつでも編集できます-それらはバックアップタスクの次の開始時に適用されます。

そのため、（サーバーまたはワークステーションエディションの）合成フルバックアップの定期的な作成を構成できます。







この場合、バックアップジョブログに[fast clone]というラベルの行が表示されます。







さらに、Veeam Backup＆Replication 9.5でVeeam Agentを使用するように構成されている場合、新しいバージョンのエージェントはスケーラブルリポジトリ（スケールアウトリポジトリ）、Microsoft Hyper-Vプラットフォームに基づく仮想マシンに即座に復元する機能をサポートすることに留意してください、およびMicrosoft Azureでの回復。

Veeam Cloud Connectリポジトリへのバックアップ



Veeam Cloud Connectは、Veeam Cloud＆Service Provider（VCSP）側のクラウドにバックアップを保存するためのリポジトリを作成する機能です。 このようなリポジトリは、バックアップ用の「3-2-1」ルールの実装に役立ち、レプリカとバックアップをオフサイトサイトに保存できます。 通常どおり、Veeam Backup＆Replicationコンソールを使用して、このようなリポジトリを管理できます。 当初、Veeam Cloud Connectテクノロジーは仮想マシンで機能し、その後物理マシンで実装されました。 そのため、Veeam Agent for Microsoft Windows 2.0以降、リモートオフィスサーバーとユーザーのラップトップの両方をそのようなクラウドリポジトリにバックアップできます。



注：クラウドリポジトリへのバックアップは、WorkstationおよびServerエディションで利用できます。 こちらの編集上の比較表をご覧ください 。



もちろんこれを行うには、バックアップタスクでVeeam Cloud Connectリポジトリをターゲットバックアップストレージとして設定する必要があります。 タスクウィザードの[ 宛先]ステップに進み、 Veeam Cloud Connectリポジトリを選択します 。







次に、サービスプロバイダーService Providerの設定を指定するステップに進みます 。 その際に行う必要のあるすべての詳細な説明は、 ここまでです （これまでは英語）。

要するに、クラウドゲートウェイの名前またはアドレス、およびプロバイダーから受信した接続用のポートを設定する必要があります。また、TLS証明書の正当性を確認し、クラウドにアクセスするためのアカウントを指定してから、[ リソースのバックアップ]ステップで目的のリポジトリを選択します。







重要！ Veeam Cloud Connectリポジトリは、オンデマンドでフルバックアップを実行できません（スタンドアロンフルバックアップ）。 また、このようなリポジトリへのデータベーストランザクションログのバックアップはサポートされていません-データベースサーバーのバックアップジョブ設定の対応するオプションは使用できません。



Veeam Cloud Connectリポジトリから復旧する



サーバーおよびワークステーションエディションは、Veeam Cloud Connectクラウドリポジトリからのリカバリオプションもサポートしています。



  • バックアップ全体をクラウドからローカルリソースにコピーすることなく、リポジトリから直接ボリュームおよびファイルレベルでリカバリ
  • 高度なVeeam Recovery Mediaを使用したベアメタルリカバリ-この操作は、クラウドリソースに接続し、リポジトリから目的のリカバリポイントを選択することでも直接実行できます。
