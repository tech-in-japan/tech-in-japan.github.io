高密度サーバーについて知っていること





いくつかのhabrの投稿に対するコメントは、人々が高密度サーバーとその機能を理解しているかどうか疑問に思いました。 この投稿を書く目的は、この問題に関する確実性を紹介することです。 この投稿は、 HPC （ 高性能コンピューティング ）のトピックに関する一連の記事の最初になることも計画されています。



高密度サーバーは、クラスタータイプのスーパーコンピューター、仮想化およびクラウド組織システム、ストレージシステムへの並列アクセスシステム、分析計算システム、検索エンジンなどを構築するためのテクノロジーで最も需要があります。 。 ソリューション、その長所と短所を検討してください。



ブレードサーバー



欧米では、データセンターのサーバーの場所が長い間欠けていました。 したがって、高密度サーバーが最初に登場したのは驚くことではありません。 パイオニアはRLX Technologiesで、2000年に3U 24ブレードに適合するシステムを開発しました。 これらの最初のBladeサーバーの主な顧客は、戦士とNASAでした。 さらに、このスタートアップはHPによって購入されました。 しかし、最も重要なことは行われました-高密度サーバーが作成されました。



先駆者に続いて、巨人：Intel、IBM、HPが登場しました。 次-DELL、SUN、Supermicro、Fujitsu、Cisco、HDSなど



高密度に加えて、ブレードシステムとRACKサーバーの主な違いは、サーバーと関連インフラストラクチャ（ネットワーク、監視、管理、冷却、電源）との統合です。 これらはすべて1つのボックスに収められており、可能であれば、フォールトトレランスの要素があります。 統合要素はBackPlaneです。マザーボードは通常パッシブです。 ブレードシステムのすべての要素が接続されています。 キャビネットが占有するスペースは、3Uから10Uまでさまざまです。 最も高密度のソリューションは、HP BladeおよびDELL PowerEdge-1Uあたり3.2サーバーです。 ほとんどすべてのメーカーは、x86 / x64ファミリプロセッサでのみサーバーを製造しています。 しかし、RISC、MIPS、ARMプロセッサにもソリューションがあります。



ちなみに、RLX Technologiesソリューションでは、サーバーの密度が高くなっています。 これは、シングルコアのCeleronプロセッサを使用していたためです。これは現在、主にデスクトップシンクライアントにのみ使用されています。 最新のプロセッサの熱放散がはるかに高いことは明らかであり、最新のソリューションの密度を上げることができないのはまさにこれです。



ブレードサーバーの利点は何ですか？ 主なポイントを強調しましょう：

  1. すべてが1つのシャーシにあります。
  2. 監視および制御システムには、RACKサーバーと比較して高度な機能があります。
  3. 各ブレードサーバーに複数の種類のネットワークが存在する。 イーサネット（100Mb / s、1Gb / s、10Gb / s）、ファイバーチャネル（2Gb / s、4Gb / s、8Gb / s、16Gb / s）、InfiniBand（SDR、DDR、QDR、FDR）が可能です。
  4. 組み込みの冷却および電力要素には、耐障害性要素があります。
  5. ホットスワップ可能な交換部品。
  6. インストールされているすべてのブレードサーバーの統合ディスクストレージシステムを編成する機能。
  7. キャビネット密度。


弱点は何ですか？ 私が重要だと思う主な欠点 ：

  1. 不完全なキットの高価格。 約70％の充填量に達した場合にのみ、RACK'ovymiアナログで適正価格が得られます。
  2. ブレードサーバー構成の拡張に関する制限。
  3. サーバーブレードを独立したユニットとして拒否できない。
  4. 同時にネットワークインターフェイスの使用が制限されます。
  5. 場合によっては、ブレードサーバーと外部との間にノンブロッキングネットワークを編成できないこと。
  6. サーマルパッケージによるコンポーネントの使用の制限（たとえば、過熱のためにほとんどのトップエンドプロセッサをインストールすることはできません）。
  7. 独自の技術。 あるメーカーから機器を購入した場合、あなたは彼からのみ購入します。
  8. エンジニアリングインフラストラクチャ（電源と冷却）の要件の増加。


デルのソリューションの例を使用して、ブレードシステムの構造を考えてみましょう。 これがDell PowerEdge M1000eです。







ブレードサーバーには2〜4個のプロセッサを搭載できます。 プロセッサの数と種類に応じて、8〜32台のサーバーブレードを1つのシャーシに取り付けることができます。 各サーバーブレードには、1GbE、10GbE、8Gb / s FC、IB DSR、DDR、QDR、FDRインターフェイスを搭載できます。 基本的に、1GbEポートがあります。



ブレードのサイズに応じて、取り付けられているメザニンインターフェイスモジュールの数は1つまたは2つです。 各メザニンモジュールには、4つの1GbEポートまたは他のインターフェイスの2つのポートを含めることができます。



シャーシ内でフォールトトレラント回路を構成するために、スイッチはペアで設置されます。 3組のスイッチを設置することが可能です。 各ペアは同じスイッチで構成する必要があります。 したがって、さまざまな組み合わせがあります。

  • 最初のペア（A）1GbE;
  • 2番目のペア（B）1GbE、10GbE、8Gb / s FC、IB（DSR、DDR、QDR、FDR）;
  • 3番目のペア（C）1GbE、10GbE、8Gb / s FC、IB（DSR、DDR、QDR、FDR）。


また、フォールトトレランスのために、2つのリモート監視および制御モジュールがインストールされています。 これらのモジュールにより、任意のブレードをリモートで制御できます。 BIOS設定の有効化、ブートソースの選択、内部メディアとローカル管理者メディアの両方からのOSのインストールから、KVMへの完全なリモートアクセスの提供まで。



ダウンロードオプションの1つは、SDカードからのダウンロードです。 このようなカードをブレードに2枚取り付けて、どのカードからでも起動できます。 それらをミラーに結合することもできます。



冗長性を持たない唯一のモジュールはKVMモジュールです。 ただし、このモジュールに障害が発生しても、ネットワークを介して接続および制御する機能は無効になりません。



M420ブレードを使用する場合、1Uあたりのサーバー密度は3.2サーバーです。







TWINサーバー



既存のブレードシステムの密度の代替品は、弟のTWINです。 この技術はインテルによって開発され、2006年にSupermicroに移行して市場を促進しました。 最初のTWINサーバーは2007年に登場しました。 それは、1つの電源を備えた1Uの2サーバー設計で、すべてのスイッチングコネクタがサーバーの背面に接続されていました。







過去6年間のこのレイアウトは認知され、その範囲は大幅に拡大しました。 2〜8個のデュアルソケットサーバーをインストールできる1U、2U、4U TWINサーバーが利用可能になりました。 一部のメーカーには、1つの4ソケットの2つの2ソケットサーバーの代わりに配置するオプションがあります。 主な長所と短所を以下に示します。



TWINサーバーの長所：

  1. すべてが1つのシャーシにあります。
  2. 各サーバーに複数の種類のネットワークが存在する。 イーサネット（100Mb / s、1Gb / s、10Gb / s）、InfiniBand（SDR、DDR、QDR、FDR）を使用できます。
  3. 一部のモデルの内蔵冷却および電力要素には、フェイルセーフ要素があります。
  4. 多数のTWINサーバーは、すべてのプラグインコンポーネントに対してホットスワップ可能です。
  5. 標準のPCI-e拡張カードを使用します。
  6. 統合ディスクストレージシステムを編成する機能。
  7. キャビネット密度。
  8. 価格は、Blade and RACKサーバーよりも低くなっています。


短所：

  1. 外部ネットワークスイッチが必要です。
  2. サーバーブレードを独立したユニットとして拒否できない。
  3. 場合によっては、サーマルパッケージによるコンポーネントの使用の制限（たとえば、過熱のために最上位のプロセッサを配置することはできません）。
  4. TWINサーバーがキャビネットを完全にブロックすると、エンジニアリングインフラストラクチャ（電源と冷却）の要件が増加します。
  5. サーバー密度はブレードよりも低くなっています。


長所と短所からわかるように、TWINサーバーとブレードサーバーは競合他社ではなく、相互に有機的な補完物である可能性が高くなります。



TWINサーバーの著名な代表者の1つは、Dell C6000シリーズサーバーです。 これらは、2つの電源と2つ、3つ、または4つのサーバーモジュールをインストールできる2Uの構造です。 各サーバーにPCI-eインターフェースを備えた2枚または3枚の拡張カードを取り付けることができます。







マイクロサーバー



データセンターのサーバー設計の最新動向について話さないと、話は完全ではありません。 マイクロサーバーについてです。 これらは、サイズと消費電力を最小化したシングルソケットサーバーです。 深刻なパフォーマンス特性に頼るべきではありません。 このタイプのサーバーの代表的なものの1つは、図に示すSupermicroサーバーです。







図からわかるように、このソリューションの密度はすでに1Uあたり4サーバーです。 このクラスのサーバーの出現は、クライアントが使用するほとんどのアプリケーションのサーバー要件が低いことによって決まります。 マイクロサーバーは、仮想化の代替として適用できる場合があります。 何らかの理由でアプリケーションが仮想化に推奨されない場合。 小型のマイクロサーバーは、一般的な低負荷のオフィスタスクにも適しています。



おわりに



各メーカーの詳細を掘り下げないようにしました。 これらの詳細は、これらのメーカーのWebサイトで直接調べることができます。



上記のソリューションのどれが特定の顧客に最も適しているかについて明確な結論を出すことは困難です。 たとえば、ブレードサーバーの密度はTWINサーバーの密度よりもはるかに高いため、最小限のスペースでより多くのサーバーを配置できます。 一方、ブレードバスケットの10GbEモジュールのレイテンシは、TWINサーバーの10GbE PCIeカードのレイテンシよりも大きくなる可能性があります。 または、高密度ブレードサーバーのプロセッサのピークパフォーマンスは、TWINソリューションのプロセッサのパフォーマンスよりも低くなります。 しかし、ある高密度ソリューションが別の高密度ソリューションよりも優れている点について議論することは、特定のタスクにのみ基づいて行うことができます。 特定のタスクを考慮して、関心のある人と意見を共有する準備ができています。
