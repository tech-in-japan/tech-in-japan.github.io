家庭および家族向けのDLNAサーバー

どういうわけか、 DLNAサーバータグは、回答よりも質問の投稿に多く見られます。 また、ホームWindows-PCへのインストールの問題でユーザーエクスペリエンスが蓄積されている場合、ホームサーバー/ NAS-a /メディアセンター用のソフトウェアの選択を決定することは困難であることが判明しました。 サンドボックスに分散すると、人々は各プログラムの問題を個別に解決します。 そして、私は個人的に、どれが価値があり、それが必要かどうかを理解できませんでした。



そして今、私はウィキペディアからのリンクで武装していますWindows、Mac OS XまたはLinux用のDLNAサーバーを選択する方法 -私はLinuxの下にあるほとんどすべてを試しました。





DLNA-ホームネットワークでの位置



おそらく大多数はすでにこのようになっています-新しいメディアデバイスが家に登場し、メディアコンテンツを消費する家族の数が増え、映画、音楽、写真の基盤が膨張しています。

同時に、誰もが独自の方法でメディアインフラストラクチャを構築します。 誰かがTV-OUTを備えたコンピューターを持ち、誰かがネットブックまたはHTPCを持っています。 NFSを介してNASとフォルダーを共有する人、HDDをメディアプレーヤーに接続する人、Sony PSの機能を使用する人...

保管場所、フォーマット、コーデックなどに混乱があります。

だから、私の場合、それは判明した：

  • リビングルームのプラズマは、FullHDよりも720pの方が適しています
  • 15Mbit / s以上のWi-Fiを介して接続された子供部屋のソニーテレビは、フィードしない方が良い、H.264彼は最大プロファイル4.1を必要とし、ネットワーク接続ストレージ上のファイルはまったく見えません
  • Openboxサテライトレシーバーは、DTSでサウンドトラックを再生する方法を知りません、SMBでは30Mbit / sを超えない、NFSではキリル文字を表示しません
  • Nokia LumiaとiPhoneは一般的にどこでも制限されています


一般に、この全体の動物園をフレンドリーにする唯一のオプションは、DLNAサーバーを使用することです。

理論的には、さまざまな場所のコンテンツを単一のフォルダーツリーに取り込み、解像度を調整し、コーデックの互換性を確認し、必要に応じてトランスコードします。 美しさ...



ベットするもの



通常、 ホームコンピューターはWindowsマシンです。 誰かがプレイしているときにトランスコーディングを使用して映画を見る-うまくいかない; 年中無休のモードではありません。 役割が合わない。 したがって、私はプログラムのWindowsバージョンさえ考慮しませんでした。

ルータがDD-WRT / OpenWRTでフラッシュ -これらのファームウェア用のパッケージがあり、動作します。 ハードウェアリソースは非常に限られています-トランスコーディングは非現実的であり、USBポートおよびネットワークフォルダへのアクセス速度は非常に制限されています。 それにもかかわらず-多くの人にとって、これは適切です。

NASまたはLinuxホームサーバーは、最も用途の広いソリューションです。 多くのプログラムがあります。 プロトコルの制限はありません。 5年前のx86プロセッサ（私の場合はAthlon X2-6000）のパフォーマンスは、任意の形式の2つのフィルムの同時オンザフライトランスコーディングに十分です。



個人的な経験、プログラム評価





MiniDLNA、uShareおよびxupnpd


これらは異なるプロジェクトですが、本質的には同じものです。 軽量かつ高速。 彼らが実行する唯一の機能は、DLNA / UPnPプロトコルを使用してメディアファイルへのネットワークアクセスを提供することです。 プレーヤーやテレビが自分でできることは他にもあります。 DLNAは完全にはサポートされておらず、多くのデバイスは単にこれらのサーバーを見ることができません。

非常に大規模です。 彼らはどこでも動作します。 パッケージは、あらゆる種類のLinuxディストリビューション、ファームウェアDD-WRT / OpenWRTを備えたルーター、およびNAS-sに存在します。 インストール手順は簡単で、プロファイルフォーラムによく描かれています。 GUIはありません

あなたとあなただけの場合（そうでなければ家族の苦情は避けられません）：

  • 1つの雑多なデバイス（Samsung TVなど）のみを視聴する
  • デバイスがネットワークフォルダーから直接ファイルを取得できないか、処理が遅すぎる（Sony Bravia TVなど）
  • USBドライブを備えた安価なNASまたはルーターに映画/音楽を保存する
  • 目的の形式でコンテンツを事前に選択するか、手動で再コーディングするのが面倒ではない
  • メディアファイルは既にフォルダで適切にソートされているか、まったく気になりません


-次に、そのようなDLNAサーバーに自分自身を制限できます。 鉄のパッケージに含まれているものを選択し、レンダリングで表示されます。



Mediatomb


開発者は「これはDLNAサーバーではなく、その機能の一部のみをサポートしています」と明確に書いています。 また、互換性の問題があるかもしれませんが、これは非常に価値のあるプロジェクトです。

組み込みのhttp-serverを介して動作するGUIが既にあります-その中にコンテンツのあるフォルダーを管理できます。 新しいものの追加、既存のショートカットへのショートカットの作成、スキャン間隔の設定など。

十分に配布されています-Linuxディストリビューションにインストールするのに問題はありません。重い依存関係を引き出しません。 単一のテキストファイルで構成されます。 操作が非常に安定しており、リソースを集中的に使用しません。

トランスコーディングを使用できますが、最も単純な形式で実装されます。 したがって、プロファイルのバインドは、出力デバイスではなく、入力ファイルの形式に対して実行されます。 サーバーは、接続したユーザーをまったく区別しません。 したがって、デバイス用の既成のプロファイルはありません。すべてを手動で構成する必要があります。

  • トランスコードする形式と直接送信する形式
  • ffmpeg、vlc、mplayerなどの状況に応じてエンコーダーを設定します
  • ビットレート、オーディオチャンネル数、H.264プロファイル、その他のエンコードオプションを設定する
  • 切り抜きや切り抜きなどのビデオフィルターをオーバーレイする


私の場合、オーディオトラックの形式（DTSまたはAC3）に基づいてffmpeg引数を設定するためのスクリプトを別途作成する必要がありました。

同時に、トランスコードされたストリームをテレビで見るとき、オーディオトラックの巻き戻しと選択は機能しません。 字幕を台無しにすることも別の歌です。

一般的に-MediaTombは真のLinuxoidにとって非常に残忍なツールです。 ユニバーサルと呼ぶことはできませんが、自分とハードウェアに合わせて完全に構成することはできます。



ライゲル


ミサ、ほとんどどこにでもあります。 しかし、それは非常に曲がってデーモンモードになります（initスクリプトを自分で記述し、フォルダーとユーザーを作成する必要があります）。 これはgnomeプロジェクトの一部であり、gstreamer（またはpulseaudio）も一緒にプルします。 誰かが自分のコンピューターなどでKDEを使用している場合、まったく不便です。 ビデオのアイコンは、XFCEプロジェクトのタンブラー、さらには微妙なニュアンスによって生成されます。 定期的に落ちる。

私の手は、トランスコーディングの機能の研究には至りませんでした。 私の評決は「 不適当 」です。



PS3 Media Server


非常に古いプロジェクト。 また、最初のアプリケーション「映画や音楽を自宅のWindows PCからSony PSに配信する」からは始まったものの、このプロジェクトは大いに発展しました。 しかし、JAVAコードとXサーバーを実行するための要件により、Xサーバーから離れました。



XBMC


DLNAサーバーがあります。 しかし、たった1つのモジュールのためにこの組み合わせを維持するのは愚かなことです。

さらに、Plexメディアサーバーの別のプロジェクトが生まれたのはXBMCからでした。



ユニバーサルメディアサーバー


彼はある種の曲線です。 彼は、選択されたフォルダの代わりにディレクトリツリー全体を見せてくれました。 ほんの数個のファイルを開き、定期的にコンソールにエラーを送りました。 トランスコードがまったく理解できませんでした。

JAVAコードのために非常に重い。 実行中のXサーバーは間違いなく必要です。悪魔と共にインストールすることはできません。 「 良くない。 」



GMediaServer


プロジェクトは中止されました。 注意する価値はありません。



LXiMedia


DLNAサーバーのシンプルで便利な実装。 ファイル、ハードセットトランスコーディングパラメーター、オーディオトラック、字幕のあるフォルダーを選択するだけで十分です。 プロファイルも設定もありません。 また、これはGUIアプリケーションであり、デーモンを置かないでください。

私の意見では、それはこの形式では役に立たない-多くのうちの1つであり、最高のものからは程遠い。



セルビオ


おそらく今最も急速に成長しているプロジェクトです。 定期的に更新されます。 モバイルOS用のプラグイン、プロファイル、アプリケーションで積極的に成長しています。

これは、デーモンを実行するJAVAアプリケーションです。 DLNAプロトコルのサポートが完了しました。 かなり巧みにコンテンツをフォルダーとカテゴリーに分散します。 映画のプレビューを作成できます。 オンライン放送のさまざまなチャンネルを簡単に接続できます。

ffmpegを使用したトランスコード。 デバイスプロファイルのベースは既に非常に大きく（そして成長を続けています）、サーバーは任意のホームグランドのエンコードパラメーターを正しく選択します。 認識できなかったこと-プロファイルを選択するか、独自のプロファイルを作成することは難しくありません。

ffmpegが使用されているという事実（mediatombと同じ問題）により、トランスコードされたストリームに巻き戻しはなく、オーディオトラックは変更されません。

これはすべて、Javaクライアント、PHPで記述されたWeb銃口、またはスマートフォン（AndroidおよびWP用のアプリケーションがあります）から制御されます。

残念ながら、.debまたは.rpmパッケージとしては入手できません。 手動で設定する必要があります：フォルダーの作成、ユーザー、解凍、サポートフォーラムで初期化スクリプトの取得、自動実行への登録。「Synology NASには既製のパッケージがあります。

一般に、自宅のメディアコンテンツへの単一のアクセスポイントになる可能性があります。 必要なことはすべて-彼はその方法を知っています。



スキフタ


Linuxリポジトリには含まれていないため、まれです。 しかし、グラフィカルなインストーラーがあり、jreが付属しているため、ほとんどすべてのシステムに組み込まれます。 2段階で起動するという点で不便です-最初はSystemTrayのユーティリティ、次に-メディアサーバー自体。 Xとデーモンがなければ動作しません。 非常にシンプルで簡潔ですが、システムに大きな負荷をかけます（JAVAコード）。



TVMOBiLi


費用は30ドル（または月額1.5ドルの購読料）です。 .debおよび.rpmパッケージとして入手可能。 Ubuntu / Fedoraにインストールすることは難しくありません-組み込みのvlcとffmpegは、おそらく見つからないライブラリを見つけて動作を拒否します。 GentooでvlcのUSEフラグを取得することができませんでした。 まったく機能しませんでした。プロファイルをやり直す必要があったため、ffmpegのみが使用されました。

一般的に、トランスコーディングプロファイルはその長所です。 任意のロジックとパラメーターを設定できます。

かわいいWEB銃口で管理。 トランスコードプロファイルの豊富なデータベースがあります。 組み込みのhttpサーバーを介した再生ファイルがあります。 サムネイルは生成できません。

それは迅速かつ美しく機能します。 良い候補者。



TwonkyServer


tar.gzを解凍してインストールされます。 またはインストーラーを介して。 Linuxリポジトリではそうではありません。 非常に高速なコード、即時起動、便利なWebベースのインターフェイス。 コンテンツを適切かつ適切にフォルダーにスローします。 既製の初期化スクリプトがあります。 ファイルからメタデータを抽出し、サムネイルを生成できます。 私はすべてが好きだった。 1つの不幸は19.95ドルです。

そしてまだ：「私はそれを推薦する 。」



プレックス


開発者が書いているように、「完全なメディアソリューション」。 確認します。

このモンスターはXBMCから成長しており、あらゆることができます。 映画の場合、ポスター、評価、および一連のデータが検索されます。 シリーズもシーズンとエピソードごとに配信されます。 音楽コレクションは、任意のソースビューから注文されます。

トランスコーディングにより、すべてがすべてのものになります。 パラメータは、テレビの画面上のメニューから直接制御できます。 LG Smart TVやApple TVなどのスマートデバイスの場合、サポートはDLNAプロトコルをはるかに超えています。

独自のクラウドサービスがサポートされており、ホームビデオをリモートで視聴できます。サーバーは、デバイスの機能と通信チャネルに応じて、送信されたビデオのパラメーターを調整します。

モバイルOSおよびWindows 8タイル用に個別に記述されたクライアントがあります。

パッケージは、Ubuntu、Fedora、CentOS、マスNASモデルで利用できます。 他のすべての場合-単純な解凍によってインストールされます。 依存関係のうち、Avahi-daemonのみが必要です。

残念ながら、トランスコーディングメカニズムでは作業の干渉は許可されません。プロファイルを修正できますが、可能性は非常に限られています。



トピックのフレームワーク内の問題を解決できますが、無料で安定していて、奇妙なことに高速です。
