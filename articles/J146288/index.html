Opera 12：アドレスバーメモリ

Operaの新しいバージョンごとに、最高の結果を期待しています。 そして毎回-失望：古い実績は削減され、新しい制限と不具合が追加されます。 アドレスバーは別のトピックに値します。







ドロップダウンリストが長かったとき、URLが完全に表示されたとき、ローカルアドレスを入力できてGoogleに表示されなかったときを思い出してください。



Opera開発者各位...



  • 最初に、 ドロップダウンリストの長さを選択しました。 何らかの理由で、設計者は、提案されたURLの6（！）がすべての人にとって十分であり、長いドロップダウンリストがすべてのユーザーを恐怖に陥れると判断しました。 あなたは推測しませんでした。 より多くの文字を入力するために、「もっと表示する」を時々突く-これはリスト内の項目を選択するほど便利ではありません。 こんにちはOpera 12- 「もっと見る」をクリックしてもリストは画面の半分しか開きません。



  • 次に、ページヘッダーをアドレスバーに追加しました。 URLはあまり気にせず、URLを覚えていますが、解決策は論理的です。 ただし、最初はヘッダーがセカンダリ（リンクの下に灰色で表示）で、次にリンクがセカンダリになりました。 次に、リンクから 「http：//example.com/pa ...」の哀れなビットを残しました 。 Opera 12では 、少し考えを変えて、「最も重要な」リンクを表示することにしました。 有意性による有意性。ただし、リンク構造は表示されません。 これらのリンクは「誰も必要」ではないので（そう、はい、アドレスバーは検索語を入力するためのものですよね？）、その後、リンクを淡い水色でペイントしました。 眼鏡をかけた兄弟の皆さん、こんにちは。



  • 第三に、 文章の「スマートな」ソートを行いました 。 引用符で囲まれた「スマート」-それは心の匂いがしないので。 ずっと前に、何ヶ月も前に入力したリンクは、最初の行に引き続き表示されます。 1日に10回入力したリンクは、30番目のポイントの近くに簡単に配置できます。 以前は、リンクを入力したときに、次に最初にリンクが提供されました。 それは馬鹿だったが、うまくいった。 手動で入力されたリンクには優先順位がありません。 はい、私は缶詰が大好きです。リンクを手動で入力し、毎回グーグルで検索するわけではありません。 Opera 12で何が起こったのですか？ これで、「スマート」ドロップダウンリストのオファーだけでなく、アドレスバー（入力時に表示される強調表示されたテキスト）のオファーも表示されます。 突然、ブロックされたサイトがオファーで浮上しました（！）。そして、「h」ではHabrではなく、http：// wifi /が提供され始めました。



  • 4番目に、アドレスバーの候補にブックマークを追加しました 。 一度、通常のソリューション。 通常-設定で、必要のないセクションを無効にでき、その後は最小化されて表示されるため。 完璧ではありません（私は隠していたでしょう）が、あなたは生きることができます。 しかし、ここにOpera 12が登場します...そして設定は不思議なことに消えます。 ブックマークは常に表示され、そもそも表示されますが、私たちの意見はあなたには興味がありません。



  • 5番目に、http：// homenetwork /のようなローカルアドレスの入力が複雑になります 。 homenetworkにアクセスすると、Googleにドロップされ、「入力したとおりの意味ではないのですか？」と丁寧に尋ねられます。HOSTSファイルが詰まった開発者として、私は不承認を表明します。 確かに、回避策があります：コンピューター名の後に「/」を入力します。 動作しますが、文書化されていません。 文書化されていないものはいつでも選択できます。 アドレスバーと検索フィールドに検索語を入力する機会を与えてください-検索語（奇妙な欲望、そうでしょう？）。 Opera 12には何がありますか？ なし。 興味深いことに、以前のサイト：example.comのテキストクエリは、サポートされていないプロトコルを誓っています。 必要な場合、Googleは表示されません。



  • 第6に、ユーザーは完全なバカであり、アドレスバーに完全なリクエストを表示する価値がないと判断したため、デフォルトではプロトコルとリクエストを非表示にし、表示された場合は灰色に塗りつぶしました 。 「私はバカではありません」というオプションを追加していません。 グレー色はとても便利で読みやすいです。 眼鏡をかけた兄弟に再びこんにちは。



  • 7番目に、リンクの中央から単語の検索を追加しました 。 素晴らしいアイデア。 しかし、一つだけありますが、うまくいきません。 「athari」と入力すると、habrahabr、github、または訪問の歴史にある他の何百ものリソースも表示されません。 私はあなたが再び彼らに行く必要があると思ったが、これは助けにはならなかった。 うまくいきません。



さて、過去からのいくつかのポイント：



  • 11バージョンのどこかで、文章内のリンクはパーセントエンコードされた判読不能な外観を取りました。 英語以外のサイトを訪問しているユーザーを心配して見ることができます。
  • 11バージョンのどこかで、「ブックマークに追加」ボタンがアドレスバーに表示され、ブラウザが破壊されました。 ただし、ドロップダウンリストボタンは非常に小さく、起爆装置ボタンから数ピクセル離れています。慎重にクリックしてください。
  • 11バージョンのどこかで、「g text」など、検索エンジンに最近入力されたクエリの表示が削除されました。


結論：多くのバージョンで、アドレスバーはますます悪化しています。 Operaのブログが、アドレスバーを修正するために多大な努力をしていることを人々に聞いたことを発表したとき、私はうれしかったです。 私はそれが良くなることを望んでいた。 消えていません。 いくつかの明白な欠点を修正し、電気テープで何かを貼り付けてから、新しい欠点で希釈しました。



そして今、簡単に思い出してください...



  • Opera 12の主な機能はハードウェアアクセラレーションであると考えられていましたが、最終的には薄れてオフになりました。
  • ほとんどのサイトは、Googleを含め、これまでのところOperaのサポートに従属しています。
  • トンボは率直に言って、他のブラウザの背景に対して輝きません。
  • プラグインが登場しましたが、機能の点で欠陥があります。
  • Operaは使用前にフラッシュをクリックする必要がある唯一のブラウザです（いいえ、要求に応じてプラグインをダウンロードすることではなく、パテントトロールのために「クリックしてこの要素をアクティブにして使用する」ことを話します）。
  • Operaはフォーマットされたテキストをコピーできない唯一のブラウザです。
  • ウィンドウ間でのタブのドラッグは機能しません（Alt + Tabを使用）。
  • Ctrlによる画像の保存が壊れました。
  • 読みにくいグレー色が増えています（現在はCtrl + Tabでも）...


愛は過ぎ去り、トマトはしおれた。 時の光に対する信仰は枯渇しました。



Firefoxの「コンストラクター」を構成しました。 「Operaのように」何十ものアドオンをインストールする必要があります。個人のユーザースクリプトを書き直し、ユーザースタイルを調整する必要があります...多くの作業があります。 しかし、それを行う必要があります。 他の方法はありません。
