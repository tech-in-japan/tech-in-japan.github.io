一般に、この記事はSQL Liteのインストールおよび操作の指示により類似していることが判明しました。

スタジオの準備。 拡張機能をインストールします。



SQLiteを有効にする最も簡単な方法は、カスタム拡張機能をインストールすることです。 これを行うには、[ツール]メニューの[拡張機能と更新プログラム]を開きます。





拡張機能をインストールした後、スタジオを再起動する必要があります。これで、プロジェクトのSQLiteライブラリへのリンクをプロジェクトに追加できるようになります。



プロジェクトへのSQLiteの追加



最初に、SQLiteライブラリへのリンクを追加する必要があります（プロジェクトを右クリック-参照を追加）：





Microsoft Visual C ++ Runtime Packageリンクをチェックしてください。 このコンポーネントは、SQLiteライブラリによって使用されます。 追加しない場合、プロジェクトは開始されて動作しますが、リンクを提供しない場合、プロジェクトはWindows App Certification Toolkitに合格しないため、追加する必要があります。



デフォルトでは、プロジェクトはコンパイルされません。



プロジェクトをコンパイルするには、Configuration Manager（ビルドメニューの最後の項目）でx86、x64またはArmプラットフォームを選択する必要があります







残念ながら、アプリケーションをレイアウトするときは、プラットフォームごとにプロジェクトを個別にコンパイルする必要がありますが、純粋なC＃にSQL Liteポートが表示されるまで、Any CPUオプションでコンパイルすることはできません。



この場合、プロジェクトプロパティで、プラットフォームおよびプラットフォームターゲットオプションの任意のCPUを返すことができます。



Linqプロバイダーを追加する





.NETプロジェクトにTim Heuer sqlite-netラッパーを追加して、LinqToSqlプロバイダーと同じ方法でSQLiteを操作することにより、SQLiteの操作を大幅に簡素化できます。 このラッパーは、SQLではなくlinqを好むユーザーに役立ち、Windows Phoneからのアプリケーションの移植も簡素化します。



パッケージマネージャーに精通している場合は、次のコマンドでラッパーをインストールできます。 



Install-Package sqlite-net


ビジュアルパッケージマネージャーでの作業を希望する場合、パッケージは、NuGetパッケージの管理メニュー（プロジェクトのコンテキストメニュー（プロジェクトを右クリック）で使用可能）またはスタジオメニューの[ツール]-> [ライブラリパッケージマネージャー]-> [ソリューションのNuGetパッケージの管理]からインストールできます...







キリル名の問題を解決します。 （ロシア語のユーザー名）


ユーザー名はアプリケーションの隔離されたストレージのパスの一部であり、小さなバグのため、sqlite-netはキリルパスを使用できません。 キリル文字で名前が指定されているユーザーに対して、アプリケーションが機能しない場合があります。



Sqlite-netをインストールすると、2つのファイルがプロジェクトに追加されます



SQLite.cs

SQLiteAsync.cs



エラーを修正するには、パブリックSQLiteConnectionクラスのコンストラクター内のsqlite.netファイル（string databasePath、bool storeDateTimeAsTicks = false）で、行を変更します 



 var r = SQLite3.Open(DatabasePath, out handle);


に 

 var r = SQLite3.Open16(DatabasePath, out handle);


これで、データベースを操作する準備がすべて整いました。



次のコードを使用して、データベースに接続できます。 



 using (var db = new SQLiteConnection(dbPath)) { //db code }


ここで、dbPathはファイルシステム内のデータベースへのフルパスです。 



 dbPath = Path.Combine(ApplicationData.Current.LocalFolder.Path, "databasename.db");




sqlite-netを使用すると、Linqプロバイダーを使用して、ADO.NETのスタイルで作業できます。 両方の点をもう少し検討してください。



sqlite-netプロバイダーの追加





たとえば、データベースに製品のリストを保存する必要があります。 まず、対応するモデルを準備する必要があります。



たとえば、自動インクリメントと250文字のテキストフィールドを持つ主キーを示す単純な製品モデルを作成します。 



 public class Product { [PrimaryKey, AutoIncrement] public int ProductId { get; set; } [MaxLength(250)] public string Name { get; set; } }


データベースを操作するための別個のDataLayerクラスがあるとします。 クラスコンストラクターで、データベース接続文字列を作成します。 



 private readonly string dbPath; public DataLayer() { dbPath = Path.Combine(ApplicationData.Current.LocalFolder.Path, "products.db"); }


このメソッドの実行時にデータベースとProductテーブルを作成するクラスにInitメソッドを追加します。 



 public void Init() { using (var db = new SQLiteConnection(dbPath)) { db.CreateTable<Product>(); } }


このメソッドが実行されると、products.dbデータベースがローカルフォルダーに作成されます。 これは、他のプラットフォームで使用できる通常のsqliteデータベースになります。 任意のユーティリティを使用してデータベースを表示することもできます。 たとえば、無料のSQLite Expert Personalを使用してこのデータベースを開くと、表の図を見ることができます。





典型的なシナリオでの作業の例をいくつか考えてみましょう。 たとえば、新しい製品を追加するためのコード： 



 public Product AddProduct(string name) { var product = new Product() {Name = name}; using (var db = new SQLiteConnection(dbPath)) { db.Insert(product); } return product; }


このメソッドが実行されると、ProductIdプロパティによってデータベースから識別子がProductオブジェクトに自動的に割り当てられます。



次のコードセクションは、選択した識別子で製品を受け取ります。 



 public Product GetProductById(int productId) { using (var db = new SQLiteConnection(dbPath)) { return db.Table<Product>().FirstOrDefault(i => i.ProductId == productId); } }


直接SQLクエリを使用してレコードを取得することもできます。 



 public Product GetProductById(int productId) { using (var db = new SQLiteConnection(dbPath)) { return db.Query<Product>("select * from product where ProductId=?", productId).FirstOrDefault(); } }


したがって、同じ方法ですべてのレコードを取得できます。 



 public List<Product> GetAllProducts() { using (var db = new SQLiteConnection(dbPath)) { return db.Table<Product>().ToList(); } }


ラッパーのもう1つの優れた機能は、識別子によって製品を削除する機能です（LinqToSQLにはありません）。 



 public void DeleteProduct(int id) { using (var db = new SQLiteConnection(dbPath)) { db.Delete<Product>(id); } }

SQLクエリ（ADO.NETのスタイル）を介してデータベースを操作する





Linqプロバイダーの利便性にもかかわらず、場合によっては、たとえば既存のコードを移植するとき、SQLクエリを使用してデータベースを直接操作する方が便利な場合があります。 sqlite-netを使用すると、データベースを完全に操作して、データを返さない単純なクエリ、または複雑なデータ型を返すデータを実行できます。



次の例では、単純なクエリ、パラメータを使用したクエリ、および単純なデータ型とテーブルを返すクエリを実行する方法を見ていきます。



例として、6.3で検討したものと同様のものを検討します。 コンストラクターに接続文字列を使用して、データを操作するためのDataLayerクラス： 



 private readonly string dbPath; public DataLayer() { dbPath = Path.Combine(ApplicationData.Current.LocalFolder.Path, "products.db"); }


同様のテーブル作成方法を検討してください。 



 public void Init() { using (var db = new SQLiteConnection(dbPath)) { db.CreateCommand( @"CREATE TABLE ""Product""( ""ProductId"" integer primary key autoincrement not null , ""Name"" varchar(250) );" ).ExecuteNonQuery(); } }


場合によっては、入力引数付きの要求が必要です。 この場合、各パラメーターのsqlクエリで「？」記号を使用して、無制限の数のパラメーターを追加できます。 たとえば、製品を名前で追加する方法の実装を考えてみましょう。 



 public void AddProduct(string name) { using(var db=new SQLiteConnection(dbPath)) { db.CreateCommand("INSERT INTO Product (Name) VALUES (?)", name).ExecuteNonQuery(); } }


場合によっては、クエリの結果として単純なデータ型を返す必要があります。 したがって、データベース内の行数を返すクエリは次のようになります。 



 public int GetCount() { using (var db = new SQLiteConnection(dbPath)) { return db.CreateCommand("SELECT COUNT(ProductId) FROM Product").ExecuteScalar<int>(); } }


テーブルを返す必要がある場合、レコードを取得するための適切なモデルが必要です。 この表では、モデルは次のようになります。 



 public class Product { public int ProductId { get; set; } public string Name { get; set; } }


これで、SQLリクエストを介してデータベースからすべてのレコードを取得できます。 



 public List<Product> GetAllProducts() { using(var db=new SQLiteConnection(dbPath)) { return db.CreateCommand("select * from product").ExecuteQuery<Product>(); } }
