Web開発者の短い辞書

全般

Webサイト -インターネットリソース、インターネット上のプロジェクト（フォーラム、ブログ、ストアなど）

ブラウザ -サイトを表示、変更、インターネットにデータを送信するためのプログラム

設計 -サイトを設計するプロセス。 利便性、効率性、美しさの原則を組み合わせた芸術とデザインの活動の一種。 （c）実際には、ほとんどの場合、既製のデザインは一連のレイアウトと、これらのレイアウトを使用するサイトのロジックの説明です。

参照条件、TOR-新しいプロジェクト、そのすべてのロジック、構造、依存関係、および移行を説明するドキュメント



レイアウト -デザイナーが描いたページの外観（psd、jpg、tiff ...）。

テンプレート-1 . html + css + jsレイアウトデザイナーが行っていたhtmlページ; 2.エンジンによって動的データをロードするための、変数とコードを含むブロックに分割されたエンジンによって処理されたテンプレート。

エンジンは、ページのコンテンツ部分と視覚部分を接続するために必要なサイトのソフトウェア部分であり、そのコアが必要です。 すべてのロジックを担当し、サイトの構造を形成します。

cms-コンテンツ管理システム、サイトの構造とコンテンツを管理するために必要なツールを含む既製のエンジン。

コンテンツ -サイトのWebページのコンテンツ。 デザインに直接関係しないすべてのテキスト、すべての写真、写真、ビデオ音楽、およびその他の情報。

スクリプト -特定のアクションを担当するプログラムコードの一部

サーバー 、 ホスト -サイトが物理的に配置されている場所。ホストはほとんどの場合、インターネット経由でブラウザからドメイン名でアクセスするか、ftp ssh経由でファイルに直接アクセスします。

ドメイン -サイトの名前、ユーザーがブラウザの「アドレス」行に書き込んでサイトにアクセスするもの

管理者、バックオフィス -サイト、その設定、コンテンツが管理されるサイトの特別なセクション。 管理パネルから、情報の編集と追加、ユーザーの管理、サイトのセクション

カウンター-Webに適用される場合、通常、サイトの統計情報を保持する特別なソフトウェアモジュールを意味します。 たとえば、訪問数、滞在時間、ユーザーのブラウザなどに関する情報を収集します

ftp （ssh） -ファイル共有（ftp ssh）およびリモートサーバー管理（ssh）用に設計されたネットワークデータ転送プロトコル。サーバー上のファイルと設定を管理するために使用されます。



レイアウト

html- 一般的な用語 -ブラウザーがページとして認識するファイル形式。サイトのレイアウト全体を記述します

css-スタイルシート。ページ全体のデザインを担当するスタイルを持つクラスを記述します

ブロックはページの独立した部分であり、ページ全体はさまざまな程度のネストのブロックで構成されます

タグ -html要素は山括弧で囲まれたテキストです。 これは、含まれている情報の表示を変更するアクティブな要素です。

ヘッダー、ヘッダー -上部、多くの場合静的なサイトの一部。通常、サイトに関する基本情報-ヘッダー、メインコントロールメニュー、ロゴ、検索、地図へのリンク、ユーザーの個人アカウントが含まれます。

フッター、地下 -サイトの下部、静的な部分。通常、著作権と連絡先を示し、メインメニューの複製、ロゴ、またはその他の情報を示します。



設計

レイヤー -レイアウトの個別の部分、各レイヤー-レイアウトの基本部分。他の部分とは別に変更できます

psd-フォトショップ、レイアウト設計の「標準」形式には、レイヤー、フォント、色、フィルター、透明度、その他の重要な情報に関する情報が含まれています。 グラフィックを圧縮しません。 1つの要素の複数の状態を含めることができます

swf、フラッシュ -マルチメディア形式。 サイトでのベクターグラフィックスの表示、ActionScriptを使用したWebアプリケーションの作成、マルチメディアコンテンツ（音楽、ビデオ、ベクターアニメーション）の表示に使用



プログラミング

言語 -（php、python、perl、asp ...）エンジンが開発されているプログラミング言語。言語は構文と機能が異なります

データベース、 DB-（mysql、mysqli、postgres、oracle）物理システムまたは仮想システムの特性を記述する構造化されたデータのセット。 （c）すべてのコンテンツはデータベースに保存され、サイトの構造とリンクに関するすべての公式情報

モジュールは、他のプログラムでの使用を目的とした、メインコードから分離された機能的に完全なフラグメントです。 モジュールを使用すると、複雑なタスクを小さなタスクに分割できます。

xmlは拡張可能なマークアップ形式です。 構造化データを保存するためのテキスト形式。 データベースとサイトエンジン間のレイヤーとして機能するか、データベースを置き換えます。

xsl 、 xslt -XMLドキュメント翻訳言語。 XSLを使用すると、XMLドキュメントをHTML、WML、RTF、PDF、SQL、SWFなどの任意のフォームに変換できます

JSONは単純なデータ交換形式であり、人とコンピューターの両方による読み取りと書き込みに便利です。

AJAX-いくつかの組み合わせ、ページを再読み込みせずにデータを更新できるようにする技術

log-すべてのイベントの記録を時系列で記録するファイル。 作業品質、トラブルシューティング、エラーの追跡に使用
