マイクロタッチ

はい、iTouchは素晴らしいです。 しかし、そのような「クローズド」デバイスが必要なのは誰ですか？ おそらく、無料のハードウェアとソフトウェアを使用して、独自のタッチスクリーンガジェットを作成し、独自のプログラムを作成する方が良いでしょうか？ はい、このガジェットはMP3を再生できない可能性がありますが、320x240ピクセルのカラータッチ（抵抗）TFTスクリーン、8ビットAtmega32u4マイクロコントローラー、リチウムポリマーバッテリー用充電器、バックライト制御、microSDスロット、3軸加速度計を備えています。 特に自分の手で何かをするのが好きな人にとっては、好奇心の小さなことですが、どう思いますか？









もちろん、1つのスキームと幸運の願いだけでは十分ではありません。 したがって、マイクロタッチの作成者（コード名 " Rossum "で知られています）は、 オペレーティングシステムといくつかのデモアプリケーションを作成しました。たとえば、

  • 画像ビューアー -画像、スライドショー、またはアニメーションを含むmicroSDを挿入できます。これらは「ミニプログラム」として表示されます
  • 調整 -タッチスクリーンの調整
  • 運命 -三次元の迷路
  • 加速 -加速度計を使用して画面の中央にボールを保存する必要があります
  • ペイント -画面をクリアする機能のないフィンガーペインティング
  • フリップ -リバースゲーム
  • マインスイーパ-Windowsをインストールする必要のないマインスイーパ
  • 3D 20面体-加速度計によって制御される3次元の20面体
  • パックマン -スプライトアニメーションのデモ
  • Lattice -3Dメッシュのデモ




このプロジェクトは、カリフォルニアとニューヨークで開発されたロッサムとレディーダのコラボレーションの結果です。



ハードウェア設計





マイクロコントローラー




Microtouchの頭脳は、TQFPパッケージのAtmega32u4です。 このチップには、USBサポートと多くの連絡先が組み込まれています。



リチウムポリマー電池と充電器






ボードには、3.7ボルトのリチウムポリマー電池用の充電器があります（MCP73812）。 バッテリーは2ピンJST-PHに接続する必要があります。接続する前に極性を確認してください。 充電器チップは非常にシンプルですが、通常はバッテリーを充電しますが、充電電流は約250 mAであるため、電力を追加せずにハブに投入しないでください。 ほとんどの場合、どのコンピューターからでも請求されます。 残念ながら、バッテリーが充電されているという通知（LEDなど）はありませんが、マイクロコントローラーのADCを介してバッテリーの状態を判別できます。 R17の抵抗を変更することにより、充電電流を変更できます。 MCP73812のデータシートを読んで、目的の充電電流の抵抗を見つけてください。



電力（3.3ボルト）




バッテリー（またはUSB）リードはLP2985 3.3v LDOに接続されます。 このレギュレータは非常に良質で、約1％の誤差と最大150ミリアンペアの電流を提供します。 マイクロタッチ回路では、4つのバックライトLEDが最も多くの電流を消費します（合計80 mAまで）。 さらに、マイクロコントローラーは10〜20 mAを消費し、約20 mAはTFTを消費し、さらに2 mAは赤色LEDを消費します。 加速度計は実質的にエネルギーを消費しません。



LEDインジケータ




このLEDはPC7に接続され、ブートローダーのアクティビティを示すために使用されます。 好きなように使用できます。



クォーツ




水晶発振器の周波数は16 MHzです。 これは小さなAtmelオーバークロックです（3.3Vでは、12 MHzの周波数を使用することをお勧めします）。 送信する前に、マイクロ回路の機能をテストします。 オーバークロックの理由は、迅速なグラフィカルインターフェイスを作成できることです。 クォーツの近くには、2つの安定化コンデンサがあります。



電源ボタン（またはこの機能はどのようにオンになりますか？）




Microtouchは、バッテリー電源が入っているときに電源を切ることができるトリックを使用しています。 LDOには、共通に短絡された「Enable」出力があります（つまり、LDOはデフォルトで無効になっています）。 長方形の電源ボタンを押すと、LDOがオンになり、AVRがオンになります。 AVRが最初に行うことは、「On」（PF0に接続されている）に高論理レベルを出力することです。 電源ボタンを離すと、このピンはLDOの動作を維持します。 これは、Microtouch用のコードを作成している場合、まずPF0が出力モードであることを確認し、高論理レベルをそれに渡すことを意味します。



したがって、コントローラは自身をオフにするために、PF0を低レベルに下げることができます。 ユーザーは、マイクロタッチを再度オンにするには、電源ボタンをもう一度押す必要があります。 ブートローダーがフリーズする可能性があるため、USBでデバイスに電力が供給されている場合は、これを行わないでください。 「リセット」を押すと、デバイスがフリーズ状態から復帰します。



リセットボタン




リセットボタンがAVRリセットピンに接続されています。 ブートローダーの起動に使用できます。 ブートローダーを上書きすると、単にデバイスが再起動されます。



ISPコネクタ




AVRは、標準のAVRプログラマー、avrdude、または同様のものでリフラッシュできます。 ファームウェアの実行中は、電源ボタンを押したままにするか 、デバイスをminiUSBに接続するか 、プログラマーから電源（3.3ボルト）を供給することを忘れないでください。 電源が実際に 3.3ボルトであることを確認してください。 たとえば、USBtinyISPは5ボルトを出力し、そのような電圧は画面を損傷する可能性があります。



スクリーンバックライト




画面のバックライトとして、4つの白色LEDが使用され、並列に接続されています。 各カソードには22〜68オームの抵抗（画像では68、値は画面のモデルによって異なる場合があります）を使用します。 カソードはピンPC6に接続されています。 出力の電圧を変更することにより、バックライトを変更できます。 バックライトがオフの場合、画面には何も表示されません。



microSD /トランスフラッシュ




MicroSDおよびmicroSDHCフラッシュカードがサポートされています。



三軸加速度計




マイクロタッチボードには、動きデータを取得するための加速度計MMA7455があります。 加速度計は、動きと揺れを検出できます。 + -2g（高精度のため）から+ -8gまで設定できます。 加速度計をテストするためにMicrotouchを落とす必要はありません！ これにはAccelerateアプリを使用することをお勧めします。



USB




USBコネクターを使用して、新しいファームウェアをマイクロタッチにダウンロードできます。 ブートローダーは4 KBのメモリを占有します。 ブートローダー自体は、AVR ISPプログラマーを使用して書き換えることができます。



USBポートを使用してバッテリーを充電することもできます。 MicrotouchはUSBから実行できますが、充電器はLDOに直接接続されているため、USBポートは必要な100〜150 mAを提供できません。 その結果、コアは電圧が低すぎることを検出し、バックライトの輝度を下げます。 そのため、画面はバッテリー駆動のときほど明るくありません。



ブートローダー



マイクロタッチを使用するには、ブートローダーが既にマイクロタッチでフラッシュされているため、AVRプログラマーは必要ありません。 ブートローダーは、USBが接続されているときに「リセット/ブート」ボタンを押すとアクティブになります。 緑色のLED「Boot」を点滅させることで、ブートローダーのアクティビティを確認できます。 その後、ボードはCOMポートとして定義され、ファームウェアにavrdudeを使用できるようになります。 プログラマー名はavr109です。たとえば、ブートローダーを確認するには、avrdude -p m32u4 -P COM3 -c avr109を実行する必要があります。



ブートローダーはすぐにタイムアウトを発行します（約5秒後）。 USB /シリアルコンバーターを使用しないため、ファームウェアは非常に高速です。 コントローラー全体を2秒未満でフラッシュできます！

ブートローダーはFLASHメモリの上位4 KBを占有するため、32 KBではなく28 KBしか使用できないことに注意してください。これはそれほど怖くないわけではなく、28 KBはそれほど小さくないと考えています。 より多くの空きスペースが必要な場合は、いつでもISPコネクタとAVRプログラマを使用できます（ブートローダが消去されます）。

Windowsを使用する場合は、infファイルが必要です。 AppleおよびLinuxユーザーは、デバイスが/ dev / ttyUSB *または/ dev / cu *（ dmesgで確認できます）として定義されるため、何も心配する必要はありません。



プログラム！



プログラムを終了するには、画面の画像の下にある画面の暗い部分をクリックする必要があります。



メインメニュー




メイン画面には、マイクロタッチでフラッシュされたプログラムが表示されます。 プログラムは、目的の円に指で触れることで選択されます。



画像ビューアー（ビルトイン画像ビューアー）




Microtouchを使用して、画像をim2形式に変換することで画像を表示できます（画面に描画する方が便利です）。 コンバーターはgithubリポジトリにあります。 microSDカードのルートフォルダーに画像を配置し、カードをmicrotouchに挿入します。 デバイスの電源を入れると、画像のアイコンが表示されます。 円をクリックして表示します。 指で画像を「ドラッグ」できます。

<

カードが2GB以上の場合、USBフラッシュドライブ上のファイルのアイコンがロード後に表示されないことがあります。 画像アイコンが表示されない場合は、HWTestアプリケーションを起動し、USBフラッシュドライブで問題がないことを確認します。 HWTestを終了すると、アイコンが表示されます。



オフ


Offアプリケーション（推測することはありません）は、デバイスの電源をオフにします。 USBに接続されている場合、デバイスはオフになりません。



Hwテスト




このアプリケーションを使用すると、バッテリー電圧、バックライト、タッチスクリーン、加速度計、SDカードを確認できます。 バックライトの明るさも変更できます。



運命の




運命は単純な3次元エンジンです。 画面のその部分を移動したい方向にタッチすることで、マップ内を移動できます。



アプリの調整




このアプリは、タッチスクリーンを調整するために使用されます。 これはデバイスの寿命の間1回だけ行う必要があります。キャリブレーションはEEPROMに永続的に保存されます。 円の中心を指またはスタイラスでタッチして、キャリブレーションポイントを設定します。

このアプリケーションは、タッチスクリーンを調整するために使用されます。指またはスタイラスで円の中心に触れる必要があります。 キャリブレーションは、デバイスの全寿命期間中に1回のみ実行できます。キャリブレーションデータは直接EEPROMに保存されます。



アプリを加速




このアプリケーションは、3軸加速度計を使用してボールを画面上で移動させ、読み取り中のデータの「心電図」も表示します。 デバイスを振るとボールが動きます。



パックマンアプリ




パックマンのスプライトアニメーションをデモする小さなかわいいプログラム（まだパックマンをプレイすることはできません）。



二十面体アプリ




このプログラムは、画面上に3次元の20面体を描画します。 フィギュアは加速度計に反応し、デバイスを振ることで回転できます。



ラティスアプリ




別の3次元アプリケーション。
