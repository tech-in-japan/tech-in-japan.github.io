A.タイプミス



Michaelは普通のインターネットユーザーです。 MichaelがYandexを使用してインターネット上で何かを検索すると、検索クエリでタイプミスが許可されることがあります。 もちろん、ほとんどの場合、YandexはMichaelが考えていたものを推測し、リクエストを修正できます。 たとえば、Michaelが「deoxyribonucleic」を封印して書いた場合（これは驚くことではありません）、Yandexは「deoxyribonucleic」のリクエストを修正します。



このタスクでは、ユーザーのリクエストのタイプミスを修正するプログラムを作成する必要があると考えたのでしょう。 これはそうではありません。このタスクに加えて、さらに5つのエキサイティングなタスクが提供され、競争の終わりまであまり時間がありません。 マイケルの次のリクエストにタイプミスが含まれているかどうかを確認してください。



入力データ

入力ファイルの最初の行には、Michaelのリクエストが含まれています。1〜5語で、小文字のラテン文字で構成され、単一のスペースで区切られています。 2行目には整数N（1≤N≤100）が含まれます：Yandex辞書に含まれる単語の数（心配しないでください、実際にはYandex辞書に含まれる単語の数は100以上です）。 3行目には、辞書の単語が含まれています。 各単語は、小文字のラテン文字で構成される文字列です。 単語は単一のスペースで区切られます。 クエリからの単語と辞書からの単語の両方が20文字を超えないことが保証されます。



インプリント

出力ファイルの唯一の行に、Michaelのクエリのすべての単語が既存の辞書に含まれている場合は「Correct」 （引用符なし）を、少なくとも1つの単語が辞書にない場合は「Misspell」 （引用符なし）を印刷します。



問題解決

タスクは、最初のセットのすべての単語が2番目のセットに属するかどうかを判断することでした。

この問題に制限がある場合、 O（N・M）で簡単な単純なソリューションで十分です。ここで、 NとMはそれぞれ最初と2番目のセットのサイズです。 それを実装するには、最初のセットの各単語が2番目のセットにあることを直接確認する必要があります。



かなり簡単な実装がPythonで取得されます。 



import sys words = sys.stdin.readline().split() _ = sys.stdin.readline() d = sys.stdin.readline().split() print "Correct" if all(word in d for word in words) else "Misspell"




B.最適なプレイリスト



アンナは通常のインターネットユーザーです。 アンナは音楽を聴いて、Yandex.Musicサービスを使用するのが大好きです。 ある日、Yandex.Musicの曲をランダムシャッフルモードで聴いているときに、アンナは、Grigory Lepsがお気に入りのグループEpidemicの後に突然演奏し始めたことに非常に腹を立てました。 アンナはinりに燃えて、サービスの技術サポートに怒った手紙を書き、その後、Yandex.Music開発チームは、再生する次の曲の選択を改善することを決定しました。



合計で、このサービスにはN曲あり、それぞれが少なくとも1回は聞きたいと思っています。 同時に、アンナは歌に再び耳を傾けることに反対しませんが、彼女は本当にジャンルの急激な変化に反対します。 音楽自体に加えて、Yandex.Musicデータベースには、一部の曲がどのようにジャンルが似ているかに関する情報が含まれています。 各エントリの形式は（a、b、U）であり、番号aの曲の直後に番号bの曲を含めると、ユーザーはU単位の不満を感じることになります。



アンナを含むすべてのユーザーが最も失敗した曲の切り替えに注意を払うことが知られているため、連続した曲の切り替えによるすべての苦情に対して、プレイリストを聴くことに対する最大の不満を最大と呼びます。



そのようなデータがあるため、Yandex.Musicは次の条件を満たすユーザーに最適なプレイリストを作成する必要があります。



  • 各曲は少なくとも1回は含まれています。
  • 隣接する曲の各ペアには、データベース内に対応するエントリがあります（これがどうなるかを知らずに曲を切り替えないほうが良いです）。
  • プレイリストを聞くことに対する不満はできるだけ小さくする必要があります。


これらの条件を満たすプレイリストを作成できますか？



入力データ

入力ファイルの最初の行には、スペースで区切られた2つの整数NおよびM （1≤N≤100,000、1≤M≤200,000）が含まれています。それぞれ、アンナが聴きたい曲の数と、これらの曲の間の遷移に関するメモの数です。 次のM行には、スペースで区切られた3つの整数ai 、 bi 、 Ui （1≤ai、bi≤N、ai≠bi、1≤Ui≤1 000 000 000）が含まれています。番号aiの曲の直後のbiは、ユーザーがUi単位の不満を経験します。 録音中に1つの曲から別の曲への移行が2回発生しないことが保証されています。



インプリント

単一の数字を印刷します。最適に構成されたプレイリストを聴いているときにアンナが経験する最大の不満、またはプレイリストを構成できない場合は-1。



問題解決

追加のパラメーターUiなしで問題を解決することを学びます。

N個の頂点とM個のアークを持つ有向グラフを考えると、2つの隣接する頂点がアークで接続され、グラフのN個の異なる頂点すべてがこのシーケンスに含まれるように、一連の頂点を構築できるかどうかを判断する必要があります。

このグラフの強く接続されたコンポーネントは、1つの頂点に置き換えることができます。 これができるのは 強力に接続されたグラフの場合、グラフのすべての頂点が少なくとも1回出現する一連の頂点を作成できます。 この声明を証明しましょう。 頂点xで終わるK個の異なる頂点を持つシーケンスがあるとします。 まだ表現されていないy頂点を選択します。 グラフが強く接続されている場合、 xからyにつながる弧のチェーンが存在し、このチェーンがシーケンスに通過する頂点を追加します。 得られた新しいシーケンスでは、少なくともK + 1個の異なる頂点。

グラフの強く接続されたコンポーネントを新しい頂点に絞り込んだ後、方向付けられたグラフは循環せずに残ります。 頂点の初期シーケンスを構築できるようにするには、このグラフが頂点をトポロジカルに順序付ける唯一の方法を持っている必要があり、トポロジカルな収縮の任意の2つの隣接する頂点間にアークが存在する必要があることがわかります。

さらに、あるグラフで必要な頂点のシーケンスを作成できる場合、追加のアークを追加した後、このプロパティが保持されることに注意してください。

最適に構成されたプレイリストを聴くプロセスでアンナが経験する最大の不満について、バイナリ検索で問題への答えを探します。



したがって、問題を解決するための次のアルゴリズムを取得します。

バイナリ検索を使用して、Ui≤Uのアークのみをグラフに残す値Uを選択します。

結果のグラフの強く接続されたコンポーネントを別々の頂点に引き出します。

得られたグラフで、頂点のトポロジカルな順序を見つけ、隣接する頂点のすべてのペア間にアークがあることを確認します。

Uの特定の値に必要なシーケンスを見つけることができたかどうかに応じて、バイナリ検索をどの方向に移動するかを決定します。
