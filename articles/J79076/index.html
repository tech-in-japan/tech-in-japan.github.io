Mac OS X用のアプリケーションの移植（コンソールエミュレーター）



最後のトピックでは、 Mac OS XでWindowsアプリケーションを移植する方法の1つの例について説明しましたが、なぜコンソールゲームを起動しないのですか...



これを行うには、zeroPSX Hybrid、zeroDS、DeSmuME、zeroN64、Sharkが必要です



zeroPSXとは何ですか？

zeroPSXは、Mac OS X用のPSXゲームを実行するために設計されたePSXeシェルです。



現在利用可能なのは、Mac OS X 10.4、10.5、および10.6向けのzeroPSX Hybrid v2.0です（ロシア語がサポートされています）

プログラムの基本は次のとおりです。

  • CrosOver Chromium 8.0
  • ePSXe 1.6
  • ePSXe 1.7


実行方法

1. zeroPSX Hybrid v2.0をダウンロードし、イメージを解凍してマウントし、zeroPSX Hybridを/ Applicaitonsに転送します

2.アプリケーション内に移動し、「Emulation Engine Selector」を実行して、目的のエミュレーターを選択します。

3. zeroPSX Hybridを起動し、ファイルメニューでPSXゲームダウンロードタイプを選択し、イメージ自体または実行可能ファイルを選択します

4.完了！



zeroDSとは何ですか？

zeroDSは、Mac OS X用のDCゲームを実行するために設計されたNO $ GMBのシェルです。



ZeroDS v1.0は現在、Mac OS X 10.4、10.5、および10.6で使用可能です（ロシア語がサポートされています）

プログラムの基本は次のとおりです。

  • CrosOver Chromium 8.0
  • いいえ$ GMB 2.5


実行方法

1. zeroDS v1.0をダウンロードし、イメージを解凍してマウントし、zeroDSを/ Applicaitonsに転送します

2. zeroDSを実行し、開いたダイアログでDCゲームを選択します

3.完了！

PS。 DCゲームを起動するための単純なDeSmuMEアプリケーションもあります。



zeroN64とは何ですか？

zeroN64は、Mac OS X用のNintendo 64ゲームを実行するために作成されたProject64のシェルです。



Mac OS X 10.4、10.5、および10.6で現在利用可能なzeroN64 v1.0（ロシア語はサポートされていません）

プログラムの基本は次のとおりです。

  • CrosOver Chromium 8.0
  • Project64 v1.6


実行方法

1. zeroN64 v1.0をダウンロードし、イメージを解凍してマウントし、zeroN64を/ Applicaitonsに転送します

2. zeroN64を実行し、ファイルメニューでNintendo 64ゲームROMを選択します

3.完了！

注 ：ウィンドウの種類を選択できます。これを行うには、アプリケーションに移動して「ウィンドウモードセレクター」を起動します。ここでは、目的の種類を選択します。ゲームおよび保存用のフォルダーへのリンクもあります。



サメとは何ですか？

SharkはDolphinのシェルで、Mac OS X用の任天堂ゲームキューブ、Wii、およびTriforceゲームを起動するために作成されました。



Shark v1.0は現在Mac OS X 10.4、10.5および10.6で利用可能です（ロシア語がサポートされています）

プログラムの基本は次のとおりです。

  • CrosOver Chromium 8.0
  • イルカSVN R4479M


実行方法

1. Shark v1.0をダウンロードし、イメージを解凍してマウントし、zeroDSを/ Applicaitonsに転送します

2. Sharkを起動し、ファイルメニューでtento GameCube、Wii、またはTriforceゲームを選択します

3.完了！

注 ：ウィンドウの種類を選択できます。これには、アプリケーション内で「ウィンドウモードセレクター」を起動します。ここでは、目的の種類を選択します。ゲームや保存用のフォルダーへのリンクもあります。



エミュレータのすべてのラッパーとユーティリティは、より安定した動作のために絶えず開発されており、無料でダウンロードできます。
