SharePoint 2013用のSQL Server 2012をインストールする

このガイドは、SharePoint 2013で使用されるSQL Server 2012の英語版をインストールするためのロシア語の簡単なガイドを探している初心者を対象としています。

この記事は専門家向けではありません。



すべての作業は3つの段階に分かれています。

  • SQL Server 2012をインストールする
  • 最大並列度サーバー構成パラメーターの構成
  • SharePoint 2013をインストールするように設計されたアカウントのアクセス許可を構成する


また、Microsoft Windows Server 2012 R2 StandartでのMicrosoft .NET Framework 3.5のインストールプロセスについても説明します。



注意：カットの下にたくさんの写真！



SQL Server 2012をインストールする



1.インストールする前に、ハードドライブに十分な空き容量があることを確認します（私の場合、2.7 GBかかりました）。

配布を開始した後、左側のメニューで[ インストール ]を選択し、[ 新規SQL Serverスタンドアロン ]をクリックするか、既存のインストールに機能を追加します ：





2.インストールウィザードが起動します。 彼がチェックします。 [詳細を表示]ボタンをクリックして、詳細レポートを表示できます。





3.詳細レポート。 [OK]ボタンを押します。





4.プロダクトキーを入力し、[次へ]ボタンをクリックします。





5.ライセンス契約の条項に同意します。

これを行うには、「 ライセンス条項に同意します 」 をチェックします。 次に、「次へ」ボタンをクリックします。





6.「ロールのセットアップ」ステップで、最初の項目「 SQL Server機能のインストール 」を選択します。 [次へ]ボタンをクリックします。





7. [機能の選択]ステップで、[ データベースエンジンサービス ]、[ 管理ツール-基本 ]、[ 管理ツール-完全]をチェックします 。 次に、「次へ」ボタンをクリックします。





8.その後、インストーラーはもう1つのチェックを実行します。 [詳細を表示]ボタンをクリックして、詳細レポートを表示できます。





9.詳細レポート。 （この段階で、「Microsoft .NET Framework 3.5がインストールされています...」というルールにエラーがありました。これについては以下を参照してください）。 [次へ]ボタンをクリックします。





10.「インスタンス構成」ステップで、SQL Serverサービスのインスタンスを構成する必要があります。

この記事は初心者を対象としています。 したがって、SQL Serverが以前にサーバーにインストールされていないことを前提とします。つまり、すべての既定の設定のままにします。 [次へ]ボタンをクリックします。





11.この手順で、インストールウィザードにディスク容量の要件が表示されます。 [次へ]ボタンをクリックします。





12. [サーバーの構成]ステップで、 SQL Serverデータベースエンジンサービスのドメインアカウントを指定する必要があります。 「アカウント名」フィールドと「パスワード」フィールドに入力したら、「次へ」ボタンをクリックします。





13.「データベースエンジンの構成」手順で、現在のユーザーをSQLサーバー管理者に追加します。 これを行うには、「現在のユーザーを追加」ボタンをクリックしてから、「次へ」ボタンをクリックします。





14.次のステップで、「次へ」ボタンをクリックします。





15.次に、インストールウィザードは結果を再度確認して表示します。 [次へ]ボタンをクリックします。





16.「インストールの準備完了」ステップで、ウィザードに情報の要約が表示されます。 ここで、「インストール」ボタンをクリックする必要があります。





17.インストールが完了すると、実行された操作に関する情報が表示されます。





18.この段階でコンピューターを再起動することを強くお勧めします。 場合によっては（たとえば、Microsoft .NET Framework 3.5のインストール中）、インストールウィザードにコンピューターの再起動を求めるウィンドウが表示されます。 拒否しないでください。



最大並列度サーバー構成パラメーターの構成



デフォルトでは、「最大並列度」パラメーターの値は0です。

SharePoint 2013では、このパラメーターが1である必要があります。

これは簡単に修正できます！



1. Microsoft SQL Server Management Studioを起動します （スタート-すべてのプログラム-Microsoft SQL Server 2012-SQL Server Management Studio）。



2.サーバー接続画面で、「接続」ボタンをクリックします。



3. [ オブジェクトエクスプローラー ]ウィンドウでサーバーを右クリックし、[ プロパティ]を選択します 。





4.開いたサーバープロパティウィンドウで、左側のメニューの[ 詳細 ]ページを選択し、プロパティのリストを画面の一番下までスクロールします。 「 最大並列度 」パラメーターを1に設定し、「OK」をクリックします。





5. SQL Server Management Studioを閉じないでください。これはまだ役に立ちます。



SharePoint 2013をインストールするように設計されたアカウントのアクセス許可を構成する



SharePoint 2013のインストールを実行するアカウントには、SQLサーバーでの昇格された特権が必要です。

このアカウントには次の役割が推奨されます。

  • dbcreator
  • securityadmin
  • 公開


1. SQL Server Management Studioの[ オブジェクトエクスプローラー ]ウィンドウで、[ セキュリティ ]項目を展開します。 次に、「 ログイン 」項目を右クリックして、「 新規ログイン 」を選択します。





2. [ログイン名]フィールドに、SharePoint 2013をインストールおよび構成する予定のアカウントのドメイン名を入力します。



3.左側のメニューで、「 サーバーロール 」ページを選択し、「dbcreator」ロールと「securityadmin」ロールを確認し、「public」ロールがすでにマークされていることを確認します。 次に、「OK」をクリックします。





これで、SQLサーバーでSharePoint 2013をインストールする準備ができました。



MS Windows Server 2012 R2 StandartにMicrosoft .NET Framework 3.5をインストールする



「 SQL Server 2012のインストール 」項目のステップ9でエラーが発生しました。.NETFramework 3.5はインストールされませんでした。

この問題を解決するには、次の手順を実行する必要があります。



1.「 サーバーマネージャー 」 コンソールを開く必要があります 。



2.左側のメニューで、アイテム「ダッシュボード」を選択します。



3.ウィンドウの中央で、[役割と機能の追加]アイテムをクリックします。



4.開いたウィザードで、「始める前に」の手順をスキップします。



5. [インストールの種類]ステップで、[ 役割ベースまたは機能ベースのインストール ]項目を選択します。 「次へ」ボタンをクリックします。



6.次のステップでは、すべてをデフォルトのままにして、「次へ」ボタンをクリックします。



7.「次へ」ボタンをクリックして、「サーバーの役割」の手順をスキップします。



8. [機能]ステップで、[。NET Framework 3.5機能]ボックスをオンにします。 「次へ」ボタンをクリックします。



9.インストールプロセスが完了したら、役割と機能の追加ウィザードを閉じることができます。



10.完了！
