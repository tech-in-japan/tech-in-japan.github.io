自動アクセラレーション：Windowsマーケットプレイス、アプリ内課金など、1年で学んだこと

この出版物では、どれくらいゆっくりと話をします-しかし、確かに、1年前に出版されたプロジェクトは、趣味のように見えましたが、現在（まだ）有望です。

私はすでにこことここでプロジェクトについて書いた 。 一言で言えば、これは0..100の加速時間を測定するための自動車向けモバイルアプリケーションであり、データをダウンロードしたり、統計情報を表示したりできるWebサイトです。



プロジェクトの生涯にわたって、私は読者と共有したい一定の経験を蓄積してきました。

すぐに共有を開始します...



上記の記事では、アプリケーション自体とそのライフパスの始まり（勝利と敗北、予期しない成功と期待を裏切る）の両方について説明しています。 今日は、次に何が起こったのかを説明します。 しかし、最初に、アプリケーションが公開される前に作成された計画、つまり私たちがフォローしている計画を提供します。 そして、私たちは彼が良いかどうかを理解しようとしています。 ここに：



計画について



  1. Androidアプリケーションのロシア語を話す視聴者向けの開発と公開、サイトの公開（ロシア語のruゾーン）。
  2. さまざまな方法でのアプリケーションのプロモーション、有効性の比較分析。
  3. iOS、Windows Phoneでのアプリケーションのロシア語を話す視聴者向けの開発と公開。
  4. さまざまな収益化方法のテスト。
  5. プロのデザイナーに新しいサイトのデザインとアプリケーションを注文し、サイトとアプリケーションの外観を変更します-真面目に見えるようにし、集合的な農場工芸品ではありません。
  6. 投資家、出版社、パートナーを引き付ける可能性の分析-世界市場での申請の適格な撤回のため。 機会があれば-私たちは引き付けます。
  7. サイトのバイリンガル（ロシア語および英語）バージョンの開発、個別の英語（または現在の言語の選択）バージョンのアプリケーションの開発、全世界向けのプロジェクトの公開（条項2および4で得られた経験を考慮して）
  8. 利益。




ポイント1と2については多くのことが書かれています。ロシアでのAndroidバージョンの人気は私の期待を上回りました。前進するための最善の方法は、ドライバーのプロフィールフォーラムでトピックを作成することでした。



計画よりも少し先で、したがって失敗しましたが、投資家やビジネスエンジェルとのコミュニケーションが試みられました。 実際、このプロジェクトはそのような組織にはほとんど関心がありません-収益性が高い場合、この利益の規模は数人のチームにとって重要かもしれませんが、大企業にとってはバケツの低下です。 私が書いたほとんどの組織は親切に答えました。このプロジェクトは興味深いものですが、私たちにとってはそうではありません。 さらに、ベンチャーオフィスの代表が私に連絡し、Auto Accelerationに興味を示したとき、私は驚きました。 私たちは1か月間飲酒し、ビジネスプランを作成し、可能な利益、必要なコストを計算しました（もちろん、かなりの量の楽観的な見方を使って）。 「内部的な理由」のために投資されませんでした。 それでも、このイベントは参加者のインスピレーションの度合いを高め、さらなる発展にプラスの影響を与えました。 結局のところ、プロジェクトの外観、プラットフォームの範囲の広さ、収益化の方法の検証という点でプロジェクトがすでに頭に浮かび上がっていたなら、すべてが違っていたかもしれません。 プロトタイプが完成するまで投資家を探し始めないでください。



Windowsマーケットプレイスの拡大



異なるOS上のスマートフォンの所有者間のホリバーは、チームのジョークにとって無尽蔵のトピックです。 Android vs iOSは楽しいです。 「スマート」プレフィックスのない旧式の携帯電話を使用している人は、両方向にユーモアをかけることができるため、特に有利な立場にいます。 友人の1人が、Windows Phoneを搭載したNokia Lumiaを何らかの理由で購入するまでそうしました。 確かに、主に笑顔の理由はオペレーティングシステム自体ではなく、破れたジーンズでした。 装置の非常に「人間工学的な」形状のため、シャベルのサイズで、鋭い角があります。 ポケットに入れればパンツは一週間で破れます。

WPバージョンの将来の開発者の頭（私たちは彼をAlexeyと呼びます）で、WPのモバイルアプリケーションの市場を征服する計画がすでに浮上してきました。 もちろん、私はすぐにプロジェクトに参加することを申し出ました。長い間オートアクセラレーションに取り組んできたチームメンバーと同じ条件で（つまり、完全に無料で）。 しかし、Alekseyは、天気予報を表示するためのアプリケーションを作成するという、成功する運命にあると思われる別のビジネスアイデアを生み出しました。



天気予報の状況は次のとおりです。情報プロバイダー（RP5など）は、ある程度詳細に完全に無料で提供します。 さらに-すでにお金のため。 Alexeyのアイデアの天才は、拒否できないRP5に申し出をすることでした。WPのアプリケーションを完全に無料で作成することを申し出れば、RP5はこのアプリケーションの有料予測に無料でアクセスできます。 予測の供給者へのこの手紙について書かれたもの。 マネージャーの「私の経験の高さ」から、私はかなりの悲観論を表明し、この特別な有利な提案がRP5に興味を持たない理由を説明しました。 しかし、アレックスは頑固で、最高のものを信じて、手紙が送られました。 回答はチーフボスに代わって拒否されましたが、これも素晴らしいことです。



失望に対処して、アレックスは私たちのランクに加わり、WPの自動加速を非常に迅速に書きました。 これまでのところ、Androidアプリケーションからデザインを取得しました-まだやり直します。 経験豊富なWPフォロワーのレビューによると、それはかなり惨めに見えますが、かなりうまく機能します。



アプリケーションは最初はモデレートされませんでした。ジオデータの受信をオフにする機会をユーザーに提供していないことがわかりました。 プログラムがそれらなしでは機能しえず、それが書かれているという事実は助けにはなりませんでした。 GPSを無効にするには無意味なチェックマークを付ける必要があり、数日後にAutoAccelerationが公開されました。



私たちは通常の実証済みの方法で、専門のフォーラムを通じてそれを宣伝し始めました。 しかし、すぐに目立つ：関心は何倍も低い。 最も一般的な質問は、「Windows Phoneとは何ですか？」です（冗談ですが、いくらか真実があります）。 Windowsマーケットプレイスのアプリケーションは、Google Playよりもはるかに小さく、すぐに新しいアプリケーションのリストに追加されました。 ダウンロードが急増しました：







ところで、本当にたくさんあります。



iOSについて



私たちのプロジェクトとApple製品の関係の歴史は、昨年9月に始まりました。そのとき、私は開発者のサブスクリプションに100ドルを支払い、次に何をすべきかを考えました。 何かをするには、デバイスが必要です。 開発者の話を読んだ後、私は結論に達しました：適切なコンピューターを持っている方が良いです。 それほど安くはありません。他の条件と同等の条件でプロジェクトに参加する準備ができているこの優れたプログラマーはいません。



キックスターターのカウンターパートで資金を集めようとしました。 無駄に。 最終的に、私はiPadを購入し、Mac OS仮想マシンに痛々しく置いて、プログラミングを始めました。ObjectiveCの優雅さと柔軟性から定期的に失望に陥りました。私は徐々に慣れてきたので、今ではこの言語の感情はそれほど憂鬱ではありません。



開発プロセス自体、フレームワーク、Androidをいじくり回さなければならなかったいくつかのことを単純に行う方法-私は言わなければならない：iOSは本当に何もない。 Objective-Cの単純なもののかさばりは、複雑なものの簡潔さによって部分的に相殺されます。 もちろん、思慮深いライブラリのおかげです。 そして、家庭でのワイドスクリーンモニターの出現により、IDEでさえ非常に便利になりました。 特に素晴らしいMercurial SourceTreeクライアントに注目したいのですが、Windows（Gitのみ）向けではないのは残念です。



一般的に、彼は袖をまくり、実際にiOS用の測定アプリケーションを作成しましたが、それ以上の時間はありませんでした。 GUIプログラミングに対する長年の憎しみが感じられ、iOSプロジェクトは失速しました。 フォーラムでは、人々はしばしばiPhoneのバージョンを求めました。



そして今、最近、サンクトペテルブルクの全く知らない人が私に連絡し、他の開発者と同じ条件でプロジェクトに参加することを申し出ました（私たちの場合、これは無料であることを思い出します）！ このために彼からお金を取らないことにすぐに同意し、何をどのように伝え、リポジトリを開き、タスクを設定しました。 セルゲイとの実際の経験の不足に少し恥ずかしがりましたが、結局のところ、無駄でした。 人が何かに情熱を傾けている場合、彼はそれを理解し、すべてを行うことができます。 現時点では、アプリケーションは機能面でほぼ準備ができているので、デザイナーを待っています。 これは「I PR」のHabréに関する別の出版物になると思うので、iOSについては終わりです。



収益化のサンプルペン



アプリケーションでの広告をすぐに拒否しました。 adMobなどの埋め込み広告をクリックするためには、ユーザーがアプリケーションで長い時間を費やす必要があります。 私たちの場合、これはそうではありません。 自動加速の使用は、一時的な、ほぼ1回限りの特性です。 他のプロジェクトですでに見たように、AirPushなどの広告は、優れたアプリケーションでさえ評価を下げるのに最適な方法です。 サイトに広告を掲載する-理由はありませんが、トラフィックが必要です。 現在、1日あたり100人を超える人々がこのサイトを訪れています。 これはごくわずかです。 ただし、新しいデザインは既にGoogleなどからの広告を表示できるように設計されています。



モバイルソフトウェアの世界における現在の傾向は、アプリの課金です。 そう思うなら、アプリ内購入。 これがどのように機能するかを確認してください。Androidバージョンで決定されました。 今年の5月20日に、50ルーブルの「プレミアムアカウント」を購入し、上限速度の設定を利用可能にし、GPSを介して最高速度を保存およびリセットするなどの機会があります。 それ以来、1日約1人がそのような購入をしています。 1日あたりのインストール数は約80です。アンインストールは約50回です。したがって、約30人のユーザーがアプリケーションをインストールしてすぐに削除せず、そのうちの1人が購入します：新規アクティブユーザーの3.3％が収入源です。 購入から30ルーブルの収入（Googleマージンを差し引いた）で、1つのアクティブなインストールから1ルーブルの収入があります。 これは非常に大まかな計算ですが、このプログラムでどれだけお金を稼ぐことができるかを理解するのは興味深いです。 簡単な計算によると、昨年の夏には10,000を超えるアクティブなインストールがありました。 したがって、アプリケーション内ですぐに購入を導入した場合、約10,000ルーブルを獲得できます。 それはデザイナーの現在の費用さえ返済しません。 ところで、興味深い点：3か月前と現在のアクティブなインストールの数を取得すると、1か月あたり350ユーザー（1日あたり約11〜12）の増加になります。 前に依存していた1日あたりのアクティブなインストールと削除の差は30で、これは2.5倍です。 この指標を使用すると、潜在的な収入はすでに25,000ルーブルにもなります。 この計算が非常に近似していることは明らかですが、世界の舞台に入る前に順序と絵を評価することができます。 おそらく、プレミアムアカウントの値を高くする必要があります！



これが収益統計の写真です。







Windows Phoneバージョンでは、ニュアンスが浮上します：7.xはアプリの請求でサポートされておらず、現在の既存のデバイスの多く（Windowsを含む）はWindows Phone 8をサポートしていません。このプラットフォームの現在の配布を考えると、無料のアプリケーションを作成する必要があります。わかりません。 無料で有料の（Pro）バージョンを作成することにしました。 これはわずかに異なるモデルです。どのように機能するかを見てみましょう。レイアウト、観察します。 自慢することは何もありませんが、需要はありません。



思考の過程で、iOSの場合、彼らは独占的な有料版で実験を行っていました：結局のところ、AppleデバイスのユーザーはAndroidの所有者よりもプログラムを購入する意思があり、Windows Phoneの所有者よりもはるかに多くあります。 彼らが言うように、我々は表示されます。 iOSバージョンのリリースについては別途お知らせし、この方法で収益化の初期結果についてお話しします。



このセクションで紹介するものはすべて新しいものであり、研究が不十分です。 出版物へのコメントで、私はアドバイスを聞いて、これらの問題における読者の経験について学びたいです。



設計について



モバイルアプリケーション市場について多くのことが書かれており、興味深いことがあります。 Androidアプリケーションが既に「落ち着いた」後、私は他のプラットフォームを勉強し始めました。 Windows Phoneバージョンのリリースにより、最初のレビューが登場しました。 これらすべてに基づいて、私は結論を下しました。Androidユーザーは、アプリケーション設計に対する要求が最も少ないということです。 AppStoreには、外部からの美しさなしに何もキャッチするものはありません。また、Windows Phoneデバイスの所有者は、システムの従来のタイルが表示されないことに悩まされます。

そのため、AppStoreでは、現在の状態ではまったく使用せず、Win Phoneアプリケーションが最初からやり直されます。



一般に、デザインはこのプロジェクトで最も高価なコストアイテムです。優れたデザイナーマニアは見つかりませんでした。安価なデザイナーフリーランサーはサービスのコストに応じて仕事をしたため、完全にやり直しました。 設計を保存しないでください。スマートフォン用アプリケーションのすべての作成者は、これを原則として使用する必要があります。



結論



特定の例に基づいて、この間に学んだことと理解したことをすべて要約しようと思います。

  1. おもしろい無料のアプリケーションがあり、それに少なくともいくつかの面白いものを結び付けているが、他のタスク、有料機能を解決していない場合、ユーザー数からの購入の約3％を当てにすることができます。
  2. おもしろい無料のアプリケーションがあり、有料のアプリケーションを作成する場合は、まだそれを宣伝して販売する必要があります。この面では、アプリの購入が望ましいようです。
  3. 服に会います。 十分に開発された設計により、単にそれが好きではなかったが、機能を理解していなかったユーザーからの否定的なフィードバックを避けることができます。
  4. Windows Phoneは、参入しきい値の面で興味深い市場です。 あまりプロモーションのないアプリケーションは「新しい」セクションに分類され、「トップフリー」の4ページ目（「ロシア連邦の交通ルール！」のすぐ隣）にあり、最初の数週間のダウンロード数はGoogle Playの数を上回っています。 しかし、Androidバージョンについては、可能な限り多くのことを書きました。
  5. 投資家、出版社との対話を急ぐな。 そのようなプロジェクトの場合、これはすでに適切なときに実行する必要があります。




そして最も重要なこと-あなたが興味を持っていることをしてください。 私はついに邪悪な懐疑論者の質問に対する良い答えを思いつきました。「なぜこのアプリケーションが必要なのか！？」：そのような質問を持っていないユーザーのために。 存在する場合、これはすでに使用する必要がある良い見込みです。
