企業でアップグレードし、ソフトウェアライセンスを失わない方法



Microsoftに次のように電話をかけました。

「組織の名前を変更した場合、購入したライセンスをどうしますか？」

-「コンピューターのアップグレードをサポートする、またはそれに縛られないWindows 7製品を選択するにはどうすればよいですか？」







などの製品：

Windows Server 2xxx

Microsoft Forefront

問題ない、なぜなら ライセンスを転送するための特別な手順があります

「Microsoft Open License Transfer Instructions」



したがって、原則として、最初の質問はすぐに解決されます。





組織には約20台のマシンがあり、購入時にWindows XP SP2 OEMがインストールされており、まさにここで問題が発生します。

これはOEM製品であるため、別のコンピューターに転送したり、アップグレードしたりすることはできません！ 謙虚に！



それでは、将来どのようにしてライセンス製品を入手し、コンピューターパークを近代化できるようになるのでしょうか？

2つの方法！



1）BOXのバージョンを購入します。 ライセンス（ステッカー）をシステムユニットにバインドしない権利が与えられます。コンピューターを完全に変更またはアップグレードでき、検証中に問題は発生しません。 ただし、システムユニット自体ではなく、ボックス自体（Microsoftライセンスサービスからの情報）にある可能性のある購入の際の書類、箱、ステッカーが必要です。



しかし、この方法は組織ではあまり便利ではありません。なぜなら、 システムをインストールするときに、各システムユニットには独自のシリアル番号があります。

これはまさにマイクロソフトが提供する方法です。



会話中に、誤ってGGK製品を思い出しました。 3台のマシンにインストールしましたが、ここから楽しい瞬間が始まります。







2）「インストールされたオペレーティングシステムWindows 7 Professionalのライセンスソリューション（Get Genuine Windows Agreement （GGWA）） 」を購入する



これは、私たちが興味を持っているソリューションです。 これは、1つのシリアル番号と必要な数の「ステッカー」を含む1つのメディアで提供され、Certificate of Authenticity（SOA）と呼ばれます。 この製品は、BOXバージョンとは異なり、ライセンスを別のコンピューターに転送することはできませんが、OEMバージョンでは禁止されているため、アップグレードを禁止しません。



それでは、組織にとってこの製品の利点は何ですか？

-この製品を購入することにより、BOXバージョンのように追加のn番目の金額を使うことはありません。

-初期インストールが非オリジナルメディアから行われた場合は、OSを再インストールすることをお勧めします（必須ではありません）。

-OS展開の簡素化。

-このパッケージは既にインストールされている状態で使用できます：

  • Windows Vista Business
  • Windows XP Professional
  • Windows 2000 Professional
  • Windows NT Workstation 4.0または3.51
  • Windows 98
  • Windows 95


次に、以下で説明する方法を使用してコンピューターをアップグレードします。

-近代化の可能性。



したがって、この製品を選択する場合、システム本体にステッカーを貼る必要があります。 移動の際に偶発的に破壊されることはありません（MSライセンスサービスはこれを禁止していません。主なことは、ケースにステッカーがあることです）。

一連のコンピューターをアップグレードまたは交換する場合、このシステムユニットにステッカーを貼っておく必要があります（これを行わないと、ライセンスが失われます。OEMバージョンの使用機能を参照してください）。 システムユニットの外観は、操作中に特に影響を受けることはなく、原則として、テーブルの下では、たとえばDVDドライブのプラグの紛失など、目を覆うことはありません。

つまり ケース自体の購入には特別な注意を払ってアプローチする必要があります。4〜6年間続く必要があり、2番目のライセンスの購入に追加の資金を費やす必要があるかどうかに依存するためです。

オープンライセンス

Open Licenseプログラムは、1つ以上のマイクロソフトソフトウェア製品の複数のコピー（ライセンス）を購入する必要がある組織を対象としています。 すでに5ライセンスを購入すると、クライアントは同じ製品を箱の形で取得するコストと比較して大幅な割引を受けます 。 最初の注文を出してから2年以内に、顧客は次の任意の小さな注文を出すときに受け取った法人割引を利用できるようになります。







OEMライセンスの不快な瞬間。

特に注目したい（私にとっては正直な発見でした）。 保証期間の終了時にマザーボードがすべて切れた場合、すべてのライセンスが無効になります！



OEMバージョンを使用する機能



OEMバージョンの主な特徴は、それらが最初にインストールされたコンピューターに「結び付けられ」、交換用コンピューターまたは他のPCに転送できないことです。

「パーソナルコンピュータ」という用語は、少なくとも中央処理装置、マザーボード、ハードドライブ、電源、シャーシを含む完全に組み立てられたコンピュータシステムを意味します。 マザーボード以外のコンピューターのコンポーネントを交換する場合、ユーザーはOEMライセンスを使用する権利を保持します。 マザーボードを交換する場合、保証修理中の類似または同一のマザーボードと交換する場合を除き、新しいライセンスが必要です。

ユーザーは、エンドユーザーとシステムビルダーの間で締結されたエンドユーザーライセンス契約 （ EULA ）に基づいて、OEMバージョンを使用する権利を受け取ります 。 特に、この契約は、システムビルダーが提供しなければならない製品の技術サポートをエンドユーザーに与える権利です。







つまり 実際、あなたの「ライセンス」は1年になるかもしれません。 システム装置の1年間の保証は、コンピューターを購入した組織から提供されます。 しかし、この場合、マザーボードが3か月後に保証修理の対象にならないような状況もあるかもしれません-それはそうです、あなたはすでに非ライセンスOSを使用しています!!!



20ライセンスを購入するときの金銭的条件を少し比較してみましょう （価格は概算で、提供されているオンラインストアから取得しましたが、差はそれほど大きくありません+-300ルーブル）。

  • BOX-6667ルーブル。 * 20 = 133340こする。
  • GGWA-5750こする。 * 20 = 115,000こする。
