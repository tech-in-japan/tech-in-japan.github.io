Lispマシンの簡単な歴史

こんにちは、Habr！



いくつかの記事で、著者はLispマシンの歴史について話すことを約束したが、そうではなかったことに気づいた。 おそらく、この短い記事はITの歴史愛好家にとって興味深いものになるでしょう。



はい、世界では、ラムダ計算のハードウェアサポートを備えた車はほとんどありませんでした-1988年の時点で、世界で約7000台です。 もちろん、今でははるかに少なくなっています。 実際、Lispマシンは情報技術の歴史において非常に強力な役割を果たしました。 これらの奇妙なコンピューターのおかげで、世界はレーザー印刷やマルチウィンドウGUIなどのテクノロジーを受け取りました。







背景





1930年代、一部の数学者は計算可能性の概念を認識して分析しようとしました。 そのような試みの1つは、アロンゾ教会によって行われ、λ計算を作成しました。 ここで彼については書きません-トピックが大きすぎます。 少なくともこの単語を知っていることを前提としています:)。



微積分による微積分ですが、適用する必要があります。 人工知能の分野で仕事が始まったとき、この分野の問題を適切に解決するプログラミング言語の必要性が生じました。 このとき、ジョン・マッカーシーはMITで働き、1958年から1963年にかけて、数学の論理の定理を自動的に導出できるAIの実装を目的としたIPLを基礎として、Lispを開発しました。 Lispインタープリターの最初のバージョンはIBM 704で実行されました。1960年、LispはPDP-1に移植されました。 一般に、Lispは配布されました。 1つだけ問題がありました-Lispは、古典的な（つまり、このコンテキストの標準アーキテクチャ）コンピューターで非常にゆっくりと動作しました。 Lispマシンの歴史に移る前に、そのアーキテクチャを表面的に見ることをお勧めします。



Lispマシン理論





Lispマシンの球形実現を見て、それを見てみましょう。 すでに知っているように、彼らはそれをLispでプログラムします。 もちろん、プロセッサはLispコードを直接実行しませんが、マイクロコードとスタックマシンはコード実行用に最適化されています。 これにより、有理数と複素数の両方で冷静に作業することができました。 OSは仮想メモリを使用して十分に大きなアドレススペースを提供し、ガベージコレクターは不要なアイテムをメモリから自動的に削除し、新しいもののためにスペースを解放します（ご存じのとおり、ガベージコレクターは実装されています）。 マシンのアーキテクチャはタグ付きアーチに属します。ロシア語では情報がほとんどないため、そのアーキテクチャについて簡単に説明します。



タグ付きアーキテクチャー（タグ付き？）時々ゴミ。 理論的には、ハードウェアに移りましょう。



λ計算のハードウェア実装





1973年が来ました。 マサチューセッツ研究所のAI研究所は、LISPをハードウェアでサポートするコンピューターアーキテクチャを作成することを真剣に考えていました。当時、標準的なアーキテクチャコンピューターでLISPプログラムを実行するのは気弱だったからです。 単一の変数の入力は、コンパイルではなくプログラムの実行中に実行されるため、数分かかる場合があります。 「奇跡のアーキテクチャ」を作成するタスクは、GreenblattとKnightという2人のプログラマに委ねられました。



「大まかな」24ビットバージョンのアーキテクチャは、リスト設計演算子にちなんで命名され、CONSと呼ばれていました。 ナイトが博士論文を擁護したのは、このマシン上ででした。 将来、このマシンのアーキテクチャは改善され、CADRと名付けられました（lisp関数cadrに敬意を表して）。 しかし、アーキテクチャを開発することとコンピューターを作成することは別のことです。 そして、Symbolics、Inc.がアリーナに入ります。



Symbolics、Inc.





Symbolics、Inc. マサチューセッツ州のラッセル（ラッセルノフツカー）によって設立され、その後カリフォルニアに移りました。 同社はLISPマシンの生産と、それらのためのソフトウェアの作成に従事していました。 奇妙なコンピューターを製造しているのは彼女だけではなく、Lisp Machines、Incも同じことをしていましたが、Symbolicsには専門家と欲望がありました。 ところで、symbolics.comはおそらく.comゾーンの最初のインターネットドメインでした。



1981年に、この会社は最初と最後の32ビットCADRアーキテクチャマシンである1MBのRAMを搭載したCADR LM-2を作成しました。 OSとそのアプリケーションは、奇妙なことに、Lisp自体、またはより正確には、方言Lisp Machine Lispで書かれています。 1982年、Open Generaオペレーティングシステムが作成され、Emacs（Zmacs）に移植されました。LISPマシンで作業し、哀れな小さな人々にラムダ計算の全力を発揮することが可能になりました。



Open GeneraとLisp Machine OSで、ガベージコレクター、仮想メモリマネージャー、マルチタスクGUI、デバッガー、Chaosnetサポートなど、たくさんのおいしいパンが登場しました。



ライセンスに関するMITとSymbolicsの小競り合いの結果として（すでにご存知のように、Richard Stallmanもここを訪れました）、彼らの協力は1983年頃に終わりました。 そして、リチャードは、何も起こらなかったように、主にMITの人々で構成されるLISPハッカーのギャングを組織しました。



1983年、Symbolicsは、LISPマシンの新しいラインの最初のモデル-Symbolics 3600を導入しました。これは、CADRアーキテクチャに基づく家庭用冷蔵庫のサイズです。 もちろん、イノベーションがないわけではありません。 「革新的な」新しい設計に加えて、マシンはガベージコレクタと仮想メモリのハードウェアサポートを受けました。 プロセッサは36ビットになり、メモリは44ビットワードで動作し、そのうち8ビットはエラー訂正（ECC）に使用されました。 3600アーキテクチャには4096個のレジスタ（CISC、こんにちは！）が含まれており、そのうち半分はコールスタックに使用され、残りの半分はマイクロコードと時間を愛するプログラム、およびドライバーに使用されました（ちなみに、ドライバーの概念は作成中に導入されましたLispマシン）。



3600はTTL集積回路上に構築され、周波​​数は約5 MHzでした。 それほど多くはありませんが、当時のLISPのハードウェアサポートを考慮すると、非常に高密度でした。 Chaosnetのサポートは廃止され、今後はSymbolicsのみがイーサネットをサポートしました。 わずか3年で、36xxラインの5つの代表者-3600、3670、3640、3675、および3645がリリースされました。後続のモデルは、メガヘルツおよびその他の数のみが異なり、アーキテクチャに特別な変更はありませんでした。



3600ラインの代表者は、人工知能研究の分野で非常に人気がありました。 ちなみに、3600シリーズの車はHDTV品質のビデオを処理できました。これが日本での人気の理由でした。



1988年、同じ会社がアイボリーラインを世界に紹介しました。その最初の代表はXL400でした。 これらのコンピューターは既に40ビットであり、16 GBのメモリをアドレス指定できました。 IvoryアーキテクチャチップはVLSIとHewlett Packardによって生産され、Symbolics自体に加えて、SunとAppleでも使用されました（そう、LispプロセッサはMacintoshでも使用されましたが、メインとしては使用されませんでした）。



アイボリープロセッサのアセンブラソースを見てみましょう。 



ENTRY: 2 REQUIRED, 0 OPTIONAL PUSH 0 PUSH FP|3 PUSH NIL BRANCH 15 SET-TO-CDR-PUSH-CAR FP|5 SET-SP-TO-ADDRESS-SAVE-TOS SP|-1 START-CALL FP|2 PUSH FP|6 FINISH-CALL-1-VALUE BRANCH-FALSE 15 INCREMENT FP|4 ENDP FP|5 BRANCH-FALSE 6 SET-SP-TO-ADDRESS SP|-2 RETURN-SINGLE-STACK




これは、次のLispコードと同等です。 



 (defun example-count (predicate list) (let ((count 0)) (dolist (i list count) (when (funcall predicate i) (incf count)))))




悪くないでしょ？ ご覧のとおり、ニーモニックは非常に長く、通常の意味ではアセンブラーと呼ぶことすら困難です。 アイボリーアセンブラーやその他のマシンの本質に興味がある場合は、bitsavers.orgのドキュメントをご覧ください。 他にも多くのSymbolicsマニュアルがあります。



Ivory-Sunstoneに基づいたRISCプロセッサを作成するという同社の試みに言及する価値があります。 しかし、プロジェクトは縮小されました。 BBNとXeroxもLISPマシンを使用していました。 BBNの作成は、InerlispでプログラムされたJerichoマシンでした。 この作品は販売されたことはありません。 1980年代にXerox ParcはXerox Lisp Machineを作成しました。これは、完全な開発環境であるInterLisp-Dと最初のHypertextアプリケーション（NoteCards）で知られています。



日本のエンジニアによるLISPマシンの作成に成功した例は知られており、そのうち大阪大学で開発されたTAKITAC-7およびEVLISを呼び出すことができます。 残念ながら、現時点ではそれらについては何も言えません。



終わり




しかし、古典的なコンピューターのますます増大する力を止めることはできませんでした。 ラムダ計算のハードウェアサポートは不要になりました。 シンボリックは最終的に破産し、1995年までにLISPマシンの時代は終わりました。 しかし、私たちはそれらを忘れませんよね？ :)



結論として、LispマシンはX言語のハードウェアサポートを備えたコンピューターを作成する唯一の試みではなく、PrologおよびJavaマシンの実装がありましたが、そのことについては改めて説明します。 読んでくれてありがとう:)。
