みなさんこんにちは！



それで、今年の7月10日に、私はモスクワでこの技術の奇跡を買って、進歩を喜ばせました！ 1か月が経過し、彼らが言うように、蓄積されました。 メガネの実際の使用に関するレビューを書くことにしました。 同時に、ARメガネEpson Moverio BT-200の機能の概要のようなもの。 つまり、装飾やその他のマーケティングなしで。 実際、この問題に私を押し付けたのは、柔軟で非常に本格的なAndroid OSの存在でした。 デバイスの技術的特性に関するドライデータは、インターネットで簡単に見つけることができます。







まあ、すぐに。 特別な警告！ 64 Gbの容量のMicroSDメモリカードをデバイスに挿入しないでください。特に、データがカードに記録されている場合は挿入しないでください。 Moverioは、メモリカードの論理構造を台無しにします。 そのようなカードはポイントによって決定されません。 もちろん、損傷したカードのファイルは、デスクトップコンピューターに復元できますが、ファイル名はありません。



実際、64 GBのメモリカードは通常、exFATファイルシステム用にフォーマットされています。 また、Windows 7の標準的な手段により、このようなカードはNTFS用にフォーマットできます。 もちろん、MS DOSコンソールエミュレーターをかわすことができます（最近のWindowsでは、cmd.exeプログラムを使用してこれを行います-「スタート」、「実行」、「cmd」とEnterを入力）。FAT32でそのようなカードをフォーマットしますが、高度な。 それでも、このような大きなメモリカードをどのようにフォーマットしても、Moverioはそれを「拒否」します。 回避策が見つかっており、2014年8月30日の付録の記事の最後に説明されています。



通常、最大32 Gbのカードが認識されます。 これは、スマートフォンを購入する際に、「32 Gbまでのメモリカードをサポートする」という指示が表示されている場合です。指示の情報は古いと思われます。 長い間、大量のカードが市場に出回っていました。 その後、購入して挿入し、...カードは通常受け入れられ、すべてが機能します。 しかし、Moverio BT-200では、このようなトリックは機能しません。 それだけです！ 警告した。







Androidについて話すと、素人にとっては、Googleとその製品にすぐに関連付けられます。その1つはPlayマーケットです。 しかし！ メガネでは、このプログラムはGoogleの他のプログラムのようにインストールされていません。

しかし、私たちは困難を恐れていません。 したがって、奇跡のデバイスの製造業者のN日間目を減らすと、メガネにプレイマーケットのアプリケーションをインストールできないように見える問題に対するトラブルのない解決策が見つかりました。 このためには、2番目のAndroidデバイス（タブレット、スマートフォン、その他のデバイス）が必要です。 インストールして問題なく動作するはずです：



-Google Playマーケット。

-ラッキーパッチャー。

-WiFiファイルエクスプローラー。



たとえば、タブレットなど、プレイマーケットから目的のプログラムを配置します。 次に、Lucky Patcherで必要なプログラムを選択し、[ツール]、[バックアップコピーの作成]の順にクリックします。

タブレットでWiFi File Explorerを起動します。 フォルダでは、どのディレクトリが表示されるかを決定します。 共有し、スタートボタンを押します。

次に、メガネをかけてブラウザを起動し、WiFi File Explorerがタブレットで提供したhttpアドレスをアドレスバーに入力します。

そして今、適切なプログラムを見つける必要があります。これは、Lucky Patcherの助けを借りて、メガネにインストールするための余分なノイズやほこりなしで取得されました。







一般に、問題を解決するための回避策が常にあるのに、メガネを装着したAndroidからGoogle Playmarketを削除する必要がある理由はまだわかりませんでした。 これについて私が訪問した考えの1つは、プログラムが医学的な理由でチェックされるためにGoogleをノックしないということです。 たとえば、元のAcapela音声エンジンをインストールした後、エンジンは音声をダウンロードしようとすると、まずプレイマーケットでアカウント情報を確認しようとします。 まあ、これは問題にはなりませんでした、なぜなら 忘れられない側面を振り返るという有害な習慣から引き離されたエンジン自体は、適切な声とともにウェブ上で安全に見つけることができます。 また、ロシア語の音声付きSVOXは問題なくインストールされました。

ちなみに、メガネのTalkBack機能もメーカーによって無視されています。 そのため、システムの要素の音声ダビングをテストするために自分でそれを置かなければなりませんでした。 かなり面白いが、すぐにうんざり。



もちろん、Google Playマーケットに代わるものもあります。 同じYandex.Store、ブラックマーケットのモボマーケット。 示された店舗のうち、品揃えの完全性の観点から、おそらく私はモボマーケットを選び出しますが、メガネが付いていても事件なしにはできませんでした。 実際には、プログラムは画面のポートレートモードのみにリンクされています。 そのため、眼鏡をかけ始めた後、横になっている貧しいモボが表示されます。 彼を立たせるには、方向指示器と画面ロックの形の別の「松葉杖」が必要です。 私はこれらの目的のために画面回転コントロールを使用しています。







2D / 3Dモード。 そうそう！ 3D VRリアリティのボリュームと深度を評価できます。 たとえば、Oculus Riftのメガネに記録されたゲームプロセスのビデオデモは感動的です。 ここでは、2Dモードでは、メガネのこめかみにある2台のプロジェクターのそれぞれで画像が複製されることを示す必要があります。 また、3Dモードでは、1つの画像が各プロジェクターで半分に分割されます。 そして、ここで私はMoverioメガネ開発者に、これらの画像の半分を交換し、これらの画像を2つの画像に対して同期的に、またそれぞれに対して選択的に90度単位で左右に回転させる機能を追加するよう強く要請しています。 なんで？ これについては後で詳しく説明します。 たとえば、デモでは、さまざまな碑文が時々非常に奇妙に見えます。 この効果は、原則として「左」と「右」が書かれているステレオヘッドフォンと比較できますが、何らかの非標準のソフトウェアまたはハードウェアを扱っている場合、誰もそれらを耳に交換する必要はありません。







快適な着用。 時間が経つにつれて硬いプラスチック製の腕が耳で感じられるようになり、ゴム製のノズルはメガネを固定するためだけでなく便利になりました。 ゴム入りの硬い鼻用キャリアでは、鼻梁の痛みが現れるまでこれらのメガネをより長く着用できるように、柔らかい布からカバーを引っ張る必要がありました。 少し後に、私はさらに快適な解決策を見つけました。 スカベンジャーの間に耳栓を挿入するだけです。 したがって、より大きな接触面積とより少ない鼻への圧力が得られます。 メガネは単に鼻の上に横たわり、足場の上に立っている間は押さないでください。 このオプションでは、ガジェットを削除しないように、何時間も違和感なく既にできます。

ガジェットからサウンドを抽出するには、ワイヤレスBluetoothヘッドフォンを使用することをお勧めします。 さもなければ、普通のヘッドフォンのワイヤーはメガネのワイヤーと混同され、ワイヤーのグレーターは鼓膜に不快な音を与えます。



Bluetoothについて。 常にトラックパッドをポケットから引き出さないようにするには、ワイヤレスマウスを接続します。 ただし、Androidに欠陥がありました。 実際、トラックパッドには、電源を入れ、デバイスをスリープ状態にし、タッチパッドでボタンをロックする3つの位置のスライダーがあります。 したがって、スライダーを右に動かしてロックすると、Bluetoothマウスも使用できなくなります。 カーソルをワイヤレスで制御することはできません。 OTGを使用して有線マウスを接続することも、ボタンがロックされている場合は役に立ちません。 Bluetoothキーボードでも同じ話をします...

メガネの開発者にとってもう1つの希望は、ボタンをロックしながら外部マウスでカーソルを制御することです。















ビデオをご覧ください。 このために、SpbTVプログラムが使用されました。 ビデオを見るとき、トラックパッドは非常に熱くなっています。 ビデオを長時間見ると、ロックを解除した後、タッチパッドが機能しなくなります。 タッチパッド上で指をどのように動かしても、意味がありません。 この場合、コースは通常のマウスでのみ制御できます。



OTGは動作しますが、フラッシュドライブは認識されません。 マウスとキーボードは通常、OTGとBluetoothの両方で認識されます。



AR / VR。 ARは拡張現実です。 VRは仮想現実です。 これらの現象は、私たちが見ている現実という、あるものの反対の極です。 たとえば、Google GlassはARでのみ機能し、Oculus RiftはVRでのみ機能します（ARの一部のサポートは、カメラを介してのみ要求されます）。 Epson Moverio BT-200はARでうまく動作し、必要な情報を目の前にある透明なガラスに表示します。 また、プロジェクターから送信された画像によってレビューが完全にブロックされないため、完全に没入することはできませんが、VRを使用することもできます。 大まかに言って、視力はこれらのメガネの目に見える画面によって3分の1（水平方向のみをカウントする場合）または1/9（水平方向と垂直方向の両方をカウントする場合）だけブロックされます。











VRゲームおよびプログラム。 Google Playmarketには、上記の方法でメガネにインストールできるDivegamesのアプリケーションがあります。 しかし、仮想現実の機能を十分に理解することは機能しません。 3Dモードでは、メガネが水平線に垂直になるように頭を左肩に傾けることによってのみ、通常の向きの画像を取得できます。 実際には、Durovisプロジェクトのアプリケーションは画面のポートレートモードに焦点を合わせており、加速度計の位置によっては画面の半分の画像を回転させることができません。 画面内の画像全体のみを回転できます。 したがって、イメージセパレーターは予想どおり垂直方向ではなく水平方向に配置されますが、これは完全に間違っています。



問題の今後の解決策は、デバイス側またはサードパーティアプリケーション側にあります。 デバイスには、異なる軸でプラスまたはマイナス90度の偏差で加速度計センサーからアプリケーションデータを転送する設定があります。 または、VRアプリケーションの起動時に、デバイスが空間内の非論理的な位置にあるときにセンサーから必要なパラメーターを最初に記録し、次にVRアプリケーションを起動したときに次回の調整でこのデータをデバイスの論理的な位置に転送します。

彼は私たちのスマートグラスの眼科的特徴について話すことを約束しました。 Android OSで実行されているスマートフォンまたはタブレットを使用している各ユーザーは、少なくとも大体自分を方向づけて、MoverioまたはOculus Riftメガネでどのように動作するかについて少し感じることができます。 これを行うには、Google PlayマーケットからAutoReader 3Dを無料でインストールします。 プログラムの説明のスクリーンショットの1つは、通常、毎日近くのオブジェクトに焦点を合わせていることを明確に示しており、オフィスについて話す場合は、タブレットの仕事以外では常にモニターに焦点を合わせています。



これらのプログラムとメガネの動作原理はまったく逆です-視力は何に集中する必要はありません。 それどころか、目の筋肉はリラックスする必要があります。 タブレットやスマートフォン上のプログラムが努力と、焦点を合わせ、視覚の焦点をぼかすさらなる習慣の形成を必要とする場合、メガネではこの瞬間は自然な方法で通過します。



プログラムの説明からの関連抜粋：「...正しいアプローチで、視覚軸

平行に向けられ（水平線を見ているとき）、外部の目の筋肉をリラックスさせ、両眼の焦点を合わせるのに役立つステレオペアに焦点を合わせます。ただ疲れを感じる）...」。
