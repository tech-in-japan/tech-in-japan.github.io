マイクロソフトについて

数え切れないほどのアプリケーション「LinuxはWindowsよりも収益性が高い」は、グローバルに役に立たないメモを書くことを余儀なくされています。



私はLinuxが好きです。 Xにはマウスカーソルの心地よい慣性があり、Gnomeには少しのMacユーザーを感じる機会があります。 とにかく、これは別の次元に入る方法です-一方、Linuxユーザーはサンドイッチにバターを塗る！



しかし、同時にMicrosoftにはMSDNがあり、すべての人と他の人にオッズを与えるでしょう。 Visual Studio 2005もあります。これは、Linux用のIDEよりも何倍も優れています。 そして、誰も持っていない.NETがあります-IBM WebSphere、QT、および他の誰。 .NETの規模は驚くべきものです。 別に、MS製品ごとに、フリーソフトウェアまたは他の開発者からの回答がありますが、MSにはすべてを統合する力があります。 正確に何か、あまりそうではないが、誰もそのような統合を持っていない。

PDA、デスクトップ、サーバー-これらはすべてすべての共通腺のためのものであり、これに加えて、すべてが相互に統合されています。 他のソフトウェアメーカーはそのようなことを自慢できません。



我慢できないMS製品は1つだけです。それはMS SQL Serverです。 何か他のものがあるかもしれませんが、私はこの種の何かに出会ったことがありません。 Visual Studioは、7番目のバージョンから、正しいものの私の理解に完全に適合する論理的な製品でした。 しかし、SQL Serverは、一見取るに足りないように見える標準のマージンを超えているだけです。その後、独自の方法でトリガーを実装し、次にDDLなどを実装します。 Oracleは高価な（MS SQLよりも高価な）ダンプです。 残りからPostgreSQLを選択します。



一般的に、Linuxは次の場合にのみ使用します。

a）通常の開発ツール。

b）通常のフレームワーク。

c）美しくて正しいオフィスツール。

d）責任者。

しかし、これはMicrosoftのような人が現れることを意味し、一般的に彼らはお互いに違いはありません。 ちなみに、SuSE Linuxライセンスの価格は100ドル以上で、これはWindowsXP Pro OEMのコストとほぼ同じです。MSオペレーティングシステムのライフサイクルのみが3年以上のSuSE Linuxライセンスです。



これらはパイです-企業ユーザーと開発者はWindowsを選択します。 マーケティングは重要ですが、そのような規模ではありません。
