データの可視化：Oracle BIからのデータ検出

最近まで、Oracle BIビジネスインテリジェンスソリューションは、高レベルのデータセキュリティとアクセス権の集中管理を備えた単一のプラットフォームを特徴とする、非常に古典的なアーキテクチャでした。 Classic BIでは、複雑なデータモデル、ETLツール、および多くのデータソースを使用することもできます。



しかし、2015年の終わりに、Oracle BI 12cのリリースにより、このソリューションのアーキテクチャは、データ検出クラスシステムの機能によって補完されました（ データ検出の詳細 ）。 つまり、新しい視覚化コンポーネントであるData Visualizationが開発されました。このコンポーネントでは、データの視覚化が質的に改善されただけでなく、セルフサービス機能も登場しました。 メモリ内処理プロセスも最適化されており、リソースの使用を最大化し、より多くのローカルデータ、部門レベルのデータ、企業データ、およびBigDataを分析します。 また、管理の簡素化と更新プロセスの最適化に加えて、プラットフォームの新しいバージョンにより、企業は実装のTCO（総所有コスト）およびROI（投資収益率）インジケータを大幅に改善できます。ITコストが削減され、新しいレポートの作成時間が短縮されます。



Oracle Data Visualization （以降DV と呼ぶ ）は、 Oracle BIEE 12cの重要な革新です。 DVは、BIツールの最も関連する要件を満たします-セルフサービス（ビジネスユーザーがIT専門家の関与なしに新しいデータソースを接続し、新しい視覚化を作成できるようにする）、直感的なグラフィック、数秒でレポートやチャートを作成できる便利なインターフェイスがあります。









DVは、Oracleの従来のサーバー側Webバージョン（BIEEの一部としてのVisual Analyzer）と、自律的に動作できるデスクトップバージョンの両方で提供されます。 Desktop DVを使用する場合、ユーザーはソースを追加します。様々な形式のデータベース、Oracle BIサーバー・アプリケーションの事前構成済レポート、そしてもちろんExcelファイルです。 サーバーバージョンでは、事前に構成されたBIのサブジェクトエリアに接続することもできます。 サポートされているソースタイプのリストは、デスクトップDVインターフェイスのスクリーンショットに示されています。









DVインターフェイスは非常に簡単に習得でき、Excelでのグラフ作成の経験があるユーザーは簡単に理解できます。









ビジュアライゼーションを構築するには、ドラッグアンドドロップを使用します。 ユーザーは、画面の左側で必要な属性とインジケータ（Oracle BIサブジェクト領域の要素または外部ソース表のフィールド）にマークを付け、構築領域にスローします。 この場合、DVは視覚化のタイプを自動的に選択し、ダイアグラムの構築を実行します。 次に、ユーザーは編集を行うことができます。チャートの種類の変更、選択した属性、必要に応じて新しい計算フィールドの追加、フィルターの適用、チャート要素の書式設定と配色の設定。 現在の視覚化を使用する場合、モデルの任意のインジケーターをドリルダウンして、必要な詳細を取得できます。 メインの視覚化フォームのインターフェースにより、複数のブックマークを作成し、パネル間をすばやく移動できます。 同時に、すべてのビジュアライゼーションはプロジェクトとして保存され、外部のソースやデータ（ファイルが使用されている場合）の設定なしで、またはそれらを使用して、構築用のモデルとして（デスクトップバージョンの場合）同僚にアップロードおよび転送できます。



DVには便利な機能「分析履歴」があり、誤って何かを「壊した」場合、ビューの最新の変更を取り消すことができます。 現在の分析のスナップショットを作成し、それにコメントを追加することができます。分析のそのような一連のスナップショットを作成する場合、プレゼンテーションで後で再生することができ、設定パラメーターのルーチンの繰り返しからユーザーを救います。



DVの標準セットには20種類以上の図が含まれており、熱プラグ、サンキー、ツリー構造、販売ファネル、ガント図、独自のマップに接続した地理的表現（これは地域のマップだけでなく、および部屋図またはメカニズム要素）。 カードの独自のレイヤーを接続するために、DVはGeoJSON形式をサポートしています。









DVを使用すると、データモデルから用語の検索用語のオンザフライレポートを作成できます。 たとえば、検索バーに「利益」と入力すると、インクリメンタル検索がトリガーされ、インジケーターの名前が置換され、DVは指定された期間の合計利益を表示します。 次に、「グループ」に入ります-必要な詳細のレポートを取得するまで、列の視覚化が製品グループなどによって構築されます。 同時に、DV自体は、特定のパラメーターセットに基づいてグラフの種類を選択します。もちろん、この方法で作成されたレポートは、ユーザーが標準のドラッグアンドドロップで作成したかのように、すぐに変更できます。









分析されたデータの品質を評価するために、DVはプロファイリング機能を提供します。各属性のサマリーチャートです。









組み込み関数の中に、予測分析の拡張機能があります。 迅速に予測を立て、傾向や排出量を特定できます。 もちろん、組み込み関数を外部関数で補完したり、独自の関数を（R言語で）開発したりできます。









それとは別に、便利なデータフロー機能に注目したいと思います。 例えば、元のデータセットからいくつかのフィールドを選択し、別のデータセットと組み合わせ、計算されたインジケーターを追加し、フィルターを適用し、最後にデータをグループ化します。 出力は、視覚化の準備が整ったデータセットです。 また、この関数を使用して、クライアントベースをセグメント化できます。









それとは別に、DVの適切な価格設定ポリシー-1300ドルから1ライセンスのコストに注意してください。



まとめると。 独立した分析ツールとしてのOracle Data Visualizationは間違いなく有用ですが、この機能はおそらく中小企業に関連します。 DVの完全な機能は、外部データの事前構成されたサブジェクト領域への接続とサーバーのコンピューティング能力の使用により、従来のOracle BIと組み合わせて使用​​すると明らかになります。 デスクトップバージョンのマイナス面の中で、現時点ではOS Xバージョンがなく、大きなExcelファイル（3,000行と80列でテスト済み）の読み込みが遅れていることに注意してください。 前者の場合、問題はWebバージョンによって補われます。後者の場合、大きな外部ファイルを使用する場合、データを事前に準備する必要があります。不要な分析を削除し、行をグループ化し、最後にテーブルをいくつかの異なるレポートグループに分割します。 このため、DVにはすでにデータフロー機能があります。
