検索エンジンLive Searchの公式ローンチ

Microsoft は 、次世代の検索エンジンLive Search を正式に開始しました。 以前はMSN Searchとして知られており、2006年3月からオープンベータ版です。



公式リリースでは、ベータ版と比較して多くの改善はありません。 少なくとも、開発者は検索の関連性を改善することについては何も述べていませんが、独立した専門家の意見では、検索結果の品質は向上しています。



公式リリースでの主な変更は、ユーザーインターフェイスに関連しています。 これで、 Ask.com検索エンジンと同様に、関連クエリが検索結果（右側の[関連検索]ボタン）に表示されます。



画像を検索すると、縮小された画像のコピーが多数表示され、それらのサイズはリアルタイムで変化します（上部にスライダーがあります）。 画面の右側に特別な「スクラッチパッド」が開き、マウスを使用して検索結果から任意の画像をドラッグアンドドロップできます。 それらはそこに保存され、ユーザーの個人コレクションを形成します。 アイデアはGoogleノートブックプログラムと同じですが、プラグインをインストールする必要なく、より有能に実装されています。



検索エンジンの公式発表により、マイクロソフトは衛星写真のコレクションを拡大しました。 ローカル検索システムは、25のアメリカの都市の画像を更新または追加したため、現在、アメリカの人口の約30％が住んでいる地域を衛星画像でカバーしています。



上記のすべてに加えて、画像検索および従来のWeb検索で無限スクロールが実装されています。 このような出力では、検索結果はページ分割されませんが、ユーザーがPgDnボタンを押すと徐々に読み込まれます。 追加のプロジェクトへのリンクが検索エンジンのメインページに表示されました。 独自の個人検索エンジンを作成するサービスLive Search Macrosが含まれます。



Live Searchの主な欠点は、その利点が続くことです。 これは非常に時間がかかる作業です。これは、明らかに、グラフィカルインターフェイス要素の過度の複雑さによって説明されます。 ライブ検索システムの画面に検索結果を表示する速度は、GoogleやYahooの速度よりも数倍遅いです。 問題は、グラフィカルインターフェイスの複雑さだけでなく、サーバー自体の速度にもある可能性が非常に高いです。 これは、サーバーの負荷が大幅に増加する重大な欠点になる可能性があります。



さらに、Live Searchの結果にはまだ多くのスパムと無関係なドキュメントがありますが、この問題は徐々に解決されています。
