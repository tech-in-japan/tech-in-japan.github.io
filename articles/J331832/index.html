エスケープシーケンスで個々の文字を動的に生成する

私の場合、シンボリックオブジェクト（制御シーケンス、エンティティ、エスケープシーケンス、ニーモニックなど）を操作するとき、タイプ "\ xnn"のエスケープシーケンスでは、動的な形成が必要でした。 このようなタスクは、たとえば、攻撃されたサイトの悪意のあるコードを解析するときに発生する可能性があります。







実際に問題-次のコードは機能しません： 



$inx = '50'; $str = "\x".$inx; $str .= "\x51"; var_dump($str); //  \x50Q




解析すると、このようなコードが機能することがわかりました。 



 $str = "\x50"; $str .= "\x51"; var_dump($str); //  PQ


また、単一引用符で囲まれたコードはまったく機能しません。 シーケンスはまったく解釈されません。 



 $str = '\x50'; $str .= '\x51'; var_dump($str); //  \x50\x51


問題の声明



問題は次のように定式化できます。文字がフラグメントで構成されているタイプ「\ xnn」の動的に形成されたシーケンスを含む文字列の正しい解釈が必要です。



この場合、シンボリックオブジェクトの連結と解釈という2つの操作が実行されます。 そして、それらは厳密な順序で実行されます。 まず、シンボリックオブジェクトの解釈が実行され、次に連結が実行されますが、その逆は実行されません。



したがって、最初の例では、シーケンスの暗号化された文字の1つがフラグメントに分割されます。



このような行は、PHPインタープリターの変換コマンドではなくなりました。







解決策



最初にphpに文字を接着させてから解釈に失敗したため、文字列をそのまま取得し、非解釈シーケンスを既製の文字に置き換えようとします。



タイプ "\ xnn"のシーケンスを文字に変換するための組み込み関数または既製関数を検索しても、明確な結果は得られませんでした。 「彼は見た目が悪いと言っている」（V. Tsoi）。 その結果、この問題はstrtr（）関数を使用して解決されました。 ここでは、解釈不能なシーケンス「\ x50」は、変換可能な「\ x50」に置き換えられます。 違いは引用符のみです。
