小規模企業やオフィスでNASサーバーを使用する

IT開発の現在のレベルでは、より低いサーバープラットフォームとプロセッサでさえ、一部の種類のタスクや小規模企業にとっては冗長すぎます。 サーバーのコンピューティングリソースをよりインテリジェントに使用するために、仮想化テクノロジーが積極的に使用されていますが、次の記事では例としてHuaweiのソフトウェアを使用してそれらについて説明します。 今日は、中小企業やリモートオフィスでのNASサーバーの使用について説明します。 もちろん、このタイプのデバイスは、本格的なサーバーよりも控えめな機能を提供しますが、そのコストは重要な議論の1つです。











NASサーバーの積極的な開発は、製造元がLinuxプラットフォームに基づいたデバイスのファームウェアを作成し始めたときに受けられました。 これ以前は、その機能は非常に控えめで、通常のデータストレージを超えることはめったにありませんでした。 しかし、Linuxシステムの出現により、状況は劇的に変化しました。 今日の記事は、選択の問題、つまり小規模オフィスのニーズを満たすことができる予算デバイスの選択に焦点を当てています。 Synologyは、SMB市場向けの古いモデルであるSynology RS2414 +をこの記事に提供してくれました。 デバイス自体のレビューについては詳しく説明しません。Synology社はそれについて詳しく説明しました。一般的にNASサーバーのソフトウェアコンポーネントについて説明し、これらのデバイスを使用する可能性について話したいと思います。

ただし、このタイプのデバイスのハードウェアから始めたいと思います（すぐに予約し、ラックマウントモデルについて説明します）。 フォールトトレランスのトピックについて言いたいだけです。 多くの人々は、このタイプのデバイスではフォールトトレランスについて話すことはできないと合理的に正当化されますが、これはローエンドの予算モデルにのみ適用されます。 Synology RS2414 + NASサーバーは電源を1つしかインストールできませんが、その兄Synology RS2414RP +はすでに2つの電源をインストールできますが、これには追加で〜30000rがかかり、これはNAS自体のコストのほぼ1/3です。 気づくのは安くない。 コントローラーとディスクについて：NASにインストールできるディスクの数が少ないほど、それに対応して少ないコントローラーがインストールされます。通常、12個のディスクのNAS'eでは、それぞれ4つのディスクに1つのコントローラーがあります。 1、RAID 5、RAID 6、RAID 10）。 ホットスワップ可能なハードドライブ、2〜4のネットワークポート。







プロセッサーは、予算モデルの最も単純なIntel Atomプロセッサーとして使用でき、それぞれ最上位モデルのIntel Xeonまで使用できます。 通常、ほとんどのニーズを満たす「平均」は、周波数が2〜3 GHzのデュアルコアプロセッサです。 ディスクの数は、NASモデルとニーズによって異なります。 最小（ラックモデルの場合）は4ディスクで、最大は最大12ディスクですが、多くの場合、追加のシェルフを接続することでボリュームを増やすこともできます。 3.5「および2.5」ドライブの両方がサポートされています。 RAMの量も512Mbから8Tbまで変化します。 多くのモデルの技術的特性が予算サーバーと競合する可能性があることをすでに理解できていますが、主な違いは何ですか？ 次に、デバイスとサービスの制御に使用されるOSについて説明します。

まず、オフィスで使用できるNAS機能のリストをすぐにリストする必要があります。

  • ACLサポート
  • iSCSIサポート

  • DNSサーバー
  • RADIUSサーバー
  • FTP、WebDAVおよびCalDAV
  • WEBサーバー
  • MySQLサーバー
  • メールサーバー
  • IPテレフォニーサーバー（アスタリスク）
  • ビデオ監視サーバー
  • プリントサーバー
  • アンチウイルス
  • クラウド同期
  • バックアップ




もちろん、リポジトリからの追加のソフトウェアのために、このリストは継続できますが、これは私たちが興味を持つ機能の「メイン」リストと呼ぶことができます。

インターフェースの説明と豊富なスクリーンショットについては説明しませんが、YouTubeビデオ（この場合）ほど良いものはないと思います。

競合他社のインターフェース、 QNAP QTS 4 、 ThecusOS 6もご覧ください 。



それでは、NASが提供するサービスについてお話しましょう。これは、事務作業で役立つでしょう。 おそらく、主なものの1つはActiveDirectoryとの統合です。 これは実際に私たちに何を与えますか？ 結局のところ、ほとんどの読者はADが何であり、どのように機能するかを知っていると思うので、これに気を取られないようにしましょう。 それで、プロは何ですか？ ドメインのグループとユーザーの操作をサポートするファイルサーバーを取得します。 実際、私たちのデータサービスは、Windows ServerやLinux上のSAMBAのもとでは決して劣っていません。管理インターフェイスもWindowsに非常によく似ています。 ただし、Windows Serverの場合もOSのライセンスを購入する必要があり、ドメイン統合を使用したLinuxのSAMBA構成がすべてのシステム管理者に与えられるわけではないことを忘れないでください。 したがって、この機能は、セットアップと操作の容易さの両方の面でプラスに安全に帰することができます。











ただし、何らかの理由でActiveDirectoryドメインがない場合、NASは従​​来のACLもサポートします。これは自分で作成して使用できます。



iSCSIサポート 。 Storage Managerを使用すると、あまり問題なくLUN番号とiSCSIターゲットを作成、削除、編集できます。 直感的なインターフェースを使用すると、数回クリックするだけで設定を変更して適用できます。 LUN番号と目標に関する情報を同時に表示することもできます。

  • LUN番号をいつでも作成、削除、編集します
  • 容量拡張サポート
  • iSNS LUNは、いくつかの目標での作業を大幅に促進します
  • ターゲットマスクには、読み取り専用やアクセスなしなどのアクセスオプションがあります
  • ワンクリックで、セッションごとにマルチパス入力/出力と複数の接続をアクティブにして、接続された負荷とフェイルオーバーのバランスをとることができます








さて、今私たちは興味のあるすべてのサービスを少し速く実行します：

VPNサーバー 。 SynologyのVPNソリューションを使用すると、従業員はセキュリティプロトコルを壊さずにリソースに簡単にリモートアクセスできます。 Synology VPN Serverは、Open VPN、PPTP、およびL2TP over IPSecプロトコルをサポートしています。







DHCPサーバー -コンピューターがTCP / IPネットワークでの作業に必要なIPアドレスおよびその他のパラメーターを自動的に取得できるようにするネットワークプロトコル。 Synology DSM 5に基づくデバイスは、小規模オフィスのニーズに対応するための本格的なDHCPサーバーにすることができます。



DNSサーバーは、ドメイン情報を取得するための分散コンピューターシステムです。 ドメイン内のプロトコルのメールルーティングとサービスノードに関する情報を取得するために、ホスト名でIPアドレスを取得するために最もよく使用されます。 便利なWebインターフェイスを備えた本格的なDNSサーバーは多くの人にアピールすると思います。



RADIUSサーバー 。 RADIUSは、ネットワーク内の接続を認証および承認する主要な手段です。 各ユーザーの認証セキュリティ用に独自のサーバーを構築します。



FTP、WebDAVおよびCalDAV 。 インターネット経由で大きなファイルを転送する必要がある場合は、FTPが最適なソリューションです。 帯域幅制御やSSL / TLS接続などの追加設定の可能性があり、機密ファイルの交換の信頼性が向上します。 さらに、DSMはHTTPSをサポートするWebDAVおよびCalDAVをサポートします。



WEBサーバーとMySQLサーバー 。 通常、これら2つのサービスは、密接に接続されているサーバーで機能しますが、ここでも例外ではありません。 小規模なサイトやブログ向けに本格的なWebサーバーを編成できます。Drupal、Joomla、WordPressなどを自動的にデプロイできます。 サーバーは、PHPで記述されたアプリケーションとPython / Python3 / Perlスクリプトの両方を実行できます。



メールサーバー 。 Synology Mail Stationアプリを使用すると、独自のメールドメインを使用してメールを送受信できます。 多くの微調整オプションを使用して、企業の電子メールアルゴリズムを作成します。







Asteriskは、DiskStationを機能豊富な音声通信サーバーとして使用できるようにするオープンソースソフトウェアです。 会議通話だけでなく、着信通話の転送と処理、発信通話と転送の作成などの機能により、Asteriskでは音声アプリケーションをすばやく作成できます。



ビデオ監視サーバー 。 Surveillance Stationは、IPカメラを管理してオフィスの監視をセットアップできるWebベースのアプリケーションです。 Surveillance Stationを使用すると、ビデオをリアルタイムで表示および録画し、スケジュール録画を設定し、リモート監視用のWebブラウザーまたはモバイルデバイスを使用して録画イベントを再生できます。 重要なイベントが発生したときに通知を設定することもできます。







プリントサーバー 。 プリンターをDiskStationサーバーに接続するだけで十分です。ドキュメントを印刷、スキャン、または任意のコンピューターからFAXで送信できます。



アンチウイルス 。 Synology Antivirus Essentialの無料のウイルス対策保護をインストールするか、業界のリーダーであるMcAfeeからウイルス対策を購入して、安心と法的要件の順守を図ってください。



Cloud Syncは 、Googleドライブ、Dropbox、およびBaidu上のファイルをDiskStationサーバー上のファイルと同期します。 さまざまな専用ストレージサイトにストレージボリュームがあるユーザーの場合、Cloud Syncはすべてのファイルの統合レコードを作成することにより、さらに便利になります。 すべてのファイルのコピーはDiskStationサーバーに保存されるため、サードパーティのストレージに対する潜在的な制限の影響を除いて、デジタルリソースの保護画面にもなります。



一般に、Synologyリポジトリからインストールできる追加ソフトウェアのトピックについて話す場合、「標準」リポジトリでもリストは非常に広範囲ですが、Synology DSMのアプリケーションを見つけることができるサードパーティのリソースもあります。 これらはさまざまな電子販売システム、CRM、SVN、Git、DLNAなどであり、通常のオフィスのニーズをはるかに超えています。







バックアップ Synology DSM 5は、作成したすべてを人の干渉や機械的エラーから保護するのに役立つさまざまなバックアップソリューションを提供します。 現代のダイナミックなビジネス環境では、ビジネスの継続性を維持することは、小さな会社でも機能するための重要な要件であることがよくあります。

  • iSCSI LUN番号をバックアップします。 従来、1つのストレージデバイスから別のストレージデバイスにLUN情報をバックアップすると、サーバーの処理能力のかなりの部分が失われます。 DiskStationは、データストレージ用の信頼性の高い効率的なサーバーであるため、追加の資金を集めてインフラストラクチャを作成することなく、ストレージレベルでバックアップタスクを実行できます。これにより、貴重なデータをオンサイトまたは遠隔で保護できます。
  • Hyper-VおよびVMware用のスナップショットマネージャー。 データの一貫性は、IT管理者が日常業務で直面する課題の1つです。 Synology Snapshot Managerは、アプリケーション整合性のあるスナップショットを提供して、データの整合性を確保します。 VMware vSphereまたはWindowsでより効率的なサーバーサポートを提供するITマネージャーは、仮想マシンの正常性とデータの整合性の保証を期待できます。 サーバーが不完全な操作を「コールドストレージ」にリセットし、アイドル状態になった後にのみ、アプリケーションと合意したスナップショットが作成されます。
  • Synology High Availability（SHA）パッケージ。 SHAは、ビジネスの復元力のために設計された、費用効果が高く、インストールが簡単で強力なソリューションです。 2台のDiskStationサーバーを接続することにより、サーバーの1つに障害が発生した場合でも、すべてのユーザーが常にデータとサービスにアクセスできます。
  • サーバーに保存されたビジネスの重要な情報には、オフィスでの負のイベントの結果としての損失を防ぐために、外部ストレージへの定期的なバックアップも必要です。 Synology Backup＆Replicationを使用すると、同じDiskStationサーバー上と異なるサーバー間、および他のrsync互換のAmazon S3 / Glacierサーバーとストレージの両方でフォルダー間をコピーできます。 または、異なるDiskStationサーバーの共有フォルダーを単一のホストサーバーに同期できます。 これは、本社と地域のオフィス間でドキュメントを交換するのに理想的です-何が起きても、常にオリジナルを手に入れることができます。 複数のバージョンを保存すると、特定の時点でサーバーを復元できると同時に、データを保存するためのボリュームの消費が最小限に抑えられ、保存の間に変更された特定のデータブロックのみを保存することで処理時間が最小限に抑えられます。 データの暗号化と圧縮により、追加の保護が提供されます。 ウィザードインターフェイスを使用すると、バックアップがはるかに簡単になります。 指定した時間に実行が開始されるように、または設定したスケジュールに従ってタスクが繰り返されるようにタスクを設定するだけです。 主なことは、次のことを覚えておくことです。システム構成も保存できます。 ユーザーアカウント、共有フォルダー、ファイル共有、セキュリティ設定...各コピーに保存する内容を決定するだけで十分です。大規模なシステム移行が必要な場合は、完全または選択的なリカバリを実行できます。




ご覧のとおり、NASサーバーの機能は十分に広いため、それらを使用して非常に幅広いタスクを実行できます。 同時に、その技術的特性は小規模オフィスにサービスを提供するには十分です。 もちろん、一部のモデルではフォールトトレランスの問題は非常に深刻ですが、これらのデバイスの重要な利点の1つである価格を忘れてはなりません。 このソリューションは、コストの面で予算サーバーモデルと競合することができ、保証サービスの品質は一部のベンダーの品質よりも優れています。 機器がモスクワで購入されていない場合、デバイスの保証は2〜3年です（モデルによって異なります）。最初に、交換用の機器が送信され、次にSCに送信されます（これらはすべて、保証期間中はクライアントに完全に無料です）。

もちろん、多くの人がフォールトトレランスの問題に関心を持っています。 古いNASモデルをペアで使用し、それらのクラスターを作成すると、かなり予算がかかり、同時に信頼性の高いデータストレージが得られ、オフィスの技術サポートに対する「不必要な義務」を引き受けることができます。

個人的に、Synologyデバイスとの会議後、 RS814 +モデルをインストールして、古いビデオ監視サーバーを置き換えることにしました。
