System Center Virtual Machine Managerテクニカルプレビューの最初の検討

すべての人に良い日と火の主からの暖かい炎！



おそらく、客観的な状況のために、Habréに関する私の最後の聖書から約100年が過ぎました...しかし、彼らは過ぎ去りました、そして、私はここに戻ってきました！

はい、十分な興味深いトピックがあります： Windows 10 Technical Previewの予備バージョンの外観、および新しいWindows Server Technical Preview ...そしてもちろん、System Centerも更新を待っています-今日、私のお気に入りのVMM Technical Previewについて説明します。 それほどではありませんが、簡単に-プレビュー-アフリカでもプレビューします。これまでのところ、あまり語るべきことはありませんが、それでも...









何を着ますか？



まず、研究とテストを行う前に、この柔道の奇跡が何に積み重なるかを考えましょう。

要件は次のように発生します。



-VMM管理サーバー（VMM管理サーバー） -記事の冒頭にあるリンクであるWindows Serverテクニカルプレビューの上にインストールする必要があります。 VMMをインストールするには、 Windows Assessment and Deployment Kit 8.1以降も必要です。



-VMM用のデータベースサーバー -ここでは、 SQL Server 2014のインストールに必要な例外的なバージョンであり、Standard Editionよりも低くはありません。 VMMサーバー自体と同じサーバー、またはリモートの専用サーバーの両方にインストールできます。



-VMMコンソール -まあ、少なくともVMMサーバーと同じサーバーにインストールする必要があり、Windows Server Technical PreviewまたはWindows 10 Technical Previewに基づいたシステムでの利便性のために-クライアントAxisについて話している場合。



-VMMライブラリサーバー -実際、これは同じWS TPに基づくサーバー上のネットワークファイルボールです。



-Hyper-V仮想化ホスト -すべてWindows Server Technical Previewにも基づいている必要があります。



-WSUS更新サーバー -ストーリーは少し異なります。このインフラストラクチャコンポーネントを追加する場合は、Windows Server Update Servicesの役割がインストールされたWindows Server 2012 R2に基づいている必要があります。



-監視およびレポートサーバーは、本質的にWS TPに基づくOperations Manager TPです。 データベースサーバーは、レポートサービス機能を備えたSQL Server 2012 SP2またはSQL Server 2014である必要があります。



VMM自体とSystem Center Technical Previewは、ここから取得できます （登録プロセスを経た後）。



また、仮想インフラストラクチャまたはクラウドのすべてのコンポーネントを管理するために、ドメインを展開する必要があることもお知らせします。

そして、すべての簡単なジェスチャーの後、VMMコンソールにアクセスするための詳細を入力するための通常の（まだ）ウィンドウが表示されます。







彼らはもはや私たちと一緒ではありません



たぶん私は誇張して悲惨すぎる-しかし、私たちはもはやVMMでは見られないものから始めることを提案します。

ネットワーク



多数の改善点と機能もここに登場しました。

それらを詳しく見てみましょう：



-仮想ネットワークアダプターの統合された命名 -これは主にゲストOSに適用されます。 以前は、VM内に複数のアダプターがあり、そのうちの1つが外部ネットワークを調べ、もう1つが内部ネットワークを調べる場合、この目標を達成するには、タンバリン（こんにちはVBスクリプトまたはPowerShell）で踊らなければなりませんでした。 現在、 第2世代の仮想マシンを展開するときに、展開プロセス中とVMテンプレートの両方でアダプターの名前を指定できるため、ネットワークの識別が大幅に容易になります。



-サービス拡張機能によるVMM機能の拡張 -これらのコンポーネントはVMMコンソールから直接追加できます。



-論理スイッチ（論理スイッチ）の作成と適用 -これにより、Windows Server Technical Previewで仮想スイッチテンプレートとポートプロファイルおよび分類を作成し、WS TPに基づく仮想化ホストに大規模に適用できます。



-仮想マシンのネットワーク構成 -この問題を解決するために、論理ネットワーク、VMネットワーク、MACアドレスプール、およびIPアドレスプールを個別に、またはVMの異なるネットワーク設定の組み合わせで使用できます。



保管施設



最後に、ストレージシステムとその化身で何が起こっているのかを説明します。



ストレージアクセスのブロックタイプについて：



-iSCSIおよびファイバーチャネルメカニズムを使用して、ブロックレベルでストレージを提供できます。



-FCのコンテキストでは、SANネットワークの工場とゾーンを管理できます。



ファイルレベルに関しては、ここで状況は次のとおりです。



-WS TPに基づくスケーラブルなファイルサーバーであるSOFSに基づいて、リソースとストレージプールをホストするためのファイルボールの両方を作成できます（ただし、後者は依然としてブロックアクセスレベルです）。



-VMMは、SOFSを自己展開するか、サードパーティのNASデバイスに基づいてリソースを追加することにより、ストレージリソースを提供することもできます。
