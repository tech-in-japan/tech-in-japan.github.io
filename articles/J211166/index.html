データウェアハウスの値下げ



Windows Azureクラウドキャパシティの価格は、価格を低く抑えるか競争力を維持するという Microsoftの約束の一環として、絶えず調整されています。



コンピューティングパワーの価格が 22％に引き下げられた後、Windows Azureストレージの価格修正の次の発表が行われました。 BLOBストレージの価格はすべての地域で20％引き下げられ 、Windows Azureに情報を保存する価格はAmazonと同等になり、多くの地域でさらに安くなります。 取引価格は、価格の50％の大幅な引き下げを受けました。



たとえば、Windows Azureでは、仮想マシンドライブ用の信頼できる地理的に分散したストレージのコストは、1か月あたりGBあたり0.095ドルです。 Amazonで同じレベルの信頼性を保証するために、顧客は標準ボリューム（月額0.05ドル/ GB）およびS3へのEBSスナップショット（月額0.095ドル/ GB）を支払う必要があります。これはWindows Azureの価格よりも34％高くなります。



独立したNasuni 2013クラウドストレージレポートによると、価格の低下に加えて、Windows Azureストレージサービスは、パフォーマンス、スケーラビリティ、および安定性の点でAmazonよりも優れています。 5つの大規模クラウドIaaSプロバイダーの比較分析による別の独立した調査では、Windows Azureは価格/パフォーマンスの点でAmazonよりも3倍高速であることが示されています。



InfiniBandをサポートする新しい仮想マシン



Windows Azureプラットフォームの場合、新しい仮想マシンインスタンスは、特に高性能シナリオとHPC向けに設計されています。 新しいA8（8コア、56 GB RAM）およびA9（16コア、112 GB RAM）インスタンスの特徴は、 40Gbit / sのデータ転送速度とリモートダイレクトメモリアクセス（RDMA）テクノロジーを備えた追加のInfiniBandネットワークの存在です。



この機能と高速プロセッサ、高速データ交換により、並列MPIアプリケーションの効率が向上し、モデリング、シミュレーション、分析、ビデオ処理、HPCクラスターの構築といった産業上の問題を解決できます。



現在、新しいインスタンスは、Windows Azure PaaSの機能、Web、および作業の役割でのみ使用できます。 新しいコピーは、米国中北部およびヨーロッパ西部の地域に配置できます（地域のリストは今後拡張されます）。



新しいコピーの価格は次のとおりです。



  • A8-2.45 ドル （〜$ 1,823 /月）
  • A9-4.90 ドル （〜3,646ドル/月）


詳細は、公式ポータルページで確認できます 。



Hyper-V Recovery Managerの一般公開



Windows Azure Hyper-V回復マネージャー （HRM）ローカルインフラストラクチャの保護と回復サービスが商用利用できるようになりました。 この新しいサービスにより、プライベートクラウドのディザスタリカバリを整理し、必要な機能と追加の構成オプション、自動保護、バイタルサインの継続的な監視、および管理されたリカバリを提供できます。







新しいサービスは、Windows Server 2012で導入されたHyper-Vレプリカテクノロジーを使用して、1つのローカルインフラストラクチャポイントから別のローカルインフラストラクチャポイントへのHyper-V仮想マシンの保護と復旧を整理します。 同時に、仮想マシンに関する必要なメタ情報のみが暗号化されたチャネルを介してクラウドに送信され、すべてのデータ交換はインフラストラクチャ内のポイント間で行われます。



2013年10月のパブリックプレビューバージョンのリリース以来、このサービスには次の革新と改善が加えられています。



  • フェールオーバーサポートの改善。
  • Kerberos認証のサポート
  • VMM 2012 SP1からVMM 2012 R2へのアップグレードのサポート。
  • サービスのスケーリングの改善。


このチュートリアルでは、新しい可能性を探ることができます。 追加のドキュメントはMSDNで入手できます 。



新しいWindows Azure Webサイト



サービスWindows Azure Webサイトは、更新と新機能の次のバッチを受け取りました。



ステージング環境。 WebサイトWeb開発者は、アプリケーションのプレリリースバージョンをホストするためのアプリケーションのステージング環境を含めることができるようになりました。 作業環境およびステージング環境は、サイトの新しいバージョンを公開またはロールバックすることにより、いつでも交換できます。







ステージング環境には独自のURLがあり、必要に応じて任意のDNSアドレスに設定できます。 さらに、環境のステージングに対してSSLを有効にすることができます。



ステージング環境の使用に関する詳細は、 この発表に記載されています。



Always On機能をサポートします。 Always Onという新しいWebサイト機能を使用すると、ホストされているWebアプリケーションをアクティブな実行状態に保つことができます。







この機能を有効にすると、Windows Azureは定期的にサイトにpingを実行し、オフになったりスリープ状態になったりせず、常に「ウォームアップ」されます。



新しいWebジョブ機能。 Webジョブと呼ばれる新しいWebサイト機能は、Webサイトベースのアプリケーション、サービス、またはバックグラウンドタスクを実行する最も簡単な方法を提供します。 exe、.cmd、.batファイルまたは.sh、.php、.py、.jsスクリプトをWebサイトにアップロードして実行できます。







ダウンロードパッケージは、スクリプトの動作に必要な追加ファイルを含む実行可能ファイルまたはスクリプトを含むzipアーカイブである必要があります。 パケットサイズは200メガバイトを超えてはなりません。



スクリプトを1回実行したり、連続して実行するように設定したり、定期的な実行を柔軟にスケジュールしたりできます。







Webジョブをより柔軟に処理するために、 Windows Azure WebJobs SDKが提供されています。これにより、サイトでのバックグラウンドコード実行の多くのタスクのソリューションを簡素化できます。 SDKには、簡単に監視できるタスクステータスバーがあります。

モバイルサービスの更新



SenchaTouchのサポート 。

Windows Azureモバイルサービスは、 SenchaのサードパーティSenchaTouchフレームワークと統合されています。 SenchaTouchは、クロスプラットフォームモバイルアプリケーションとHTML5 / JSベースのWebサイトを構築するためのツールです。 新しい統合により、SenchaTouchベースのアプリケーションでWindows Azureモバイルサービスを使用できます。

新しいWindows Azureストレージ



Windows Azureストレージクラウドストレージサービスは、多数の新しいライブラリと更新されたライブラリを受け取りました。



リポジトリのコピーへの読み取りアクセス 。 ご存じのように、地理分散ストレージモードが有効になっている場合、1つのデータセンター内にユーザーデータのコピーを3つ、さらに別のデータセンターにデータのコピーを3つ保存することにより、Windows Azureストレージのフォールトトレランスが保証されます。



ただし、通常の状況では、2番目のデータセンターのこれらのデータコピーへのアクセスは閉じられ、ユーザーはバックアップコピーを使用できませんでした。 最近のプラットフォームの更新により、このような読み取りアクセスGeo冗長ストレージ（RA-GRS）のコピーの読み取りアクセス機能がユーザーに利用可能になりました。







この革新により、プライマリデータの場所で一時的な障害が発生した場合に、データに即座にアクセスできます。 また、データの複製をテストおよび追跡することもできます。 データにアクセスするには、 myaccountname - secondary 。<service> .core.windows.netという形式の対応するURLを使用できます。



詳細については、 この記事を参照してください 。

新しい監視機能



さまざまなWindows Azureクラウドサービスのパフォーマンス監視機能は常に進化しています。 このアップデートでは、次の新機能が導入されました。



毎分モニタリング 。 Windows Azureの統計エンジンは重要な更新プログラムを受け取り、ユーザーにサービスの動作を1分間隔で監視できるようになりました。







プレミアムSQLデータベースメトリック。 Windows Azure SQL Databaseクラウドデータベースのプレミアムエディションでは、他のユーザーのデータベースから隔離されているため、ユーザーはデータベースのパフォーマンスレベルを保証できます。 監視サービスの更新では、CPUとI / OアクティビティメトリックがPremiumエディションデータベースに追加されました。



さらに、PremiumとStandardの両方のSQL Databaseエディションのリポジトリメトリックを追跡するために、新しいメトリックが更新に追加されました。



統計Webサイトを保存するためのコンテナ 。 以前は、Webサイトの統計を収集するメカニズムを構成するには、ストレージBLOBの既存のコンテナーを選択する必要がありました。







更新では、新しいものを作成し、サイトのログと統計に関する情報用に別のコンテナを設定することが可能になりました。



モバイルサービス運用履歴 。 操作ログのメカニズムにより、Windows Azureアカウントの管理者とユーザーは、このアカウント内のオブジェクトで実行された操作を追跡できます。







この更新により、Windows Azureモバイルサービスの操作に関連する20を超える新しい操作が追加されました。これは、操作の履歴に詳細な監査のために記載されています。



サービスのステータスに関する通知。 通知機能（Windows Azureアラート）を使用すると、Windows Azureアカウントの管理者とユーザーは、サブスクリプションの一部として開始されたクラウドサービスの特定のパフォーマンスを超えたという電子メール通知を受信できます。 これにより、負荷の増加または減少、過剰な容量制限などについて事前に通知することができます。







この更新プログラムは、Windows Azure SQLデータベースサービスのステータスに関する通知のサポート、およびWindows Azure Webサイトの操作に関する通知の大幅に拡張された一連のメトリックを追加しました。







たとえば、CPU時間とデータ出力のメトリックは、Windows Azure Webサイトの無料コピーのクォータの枯渇について事前に通知します。



この更新プログラムは、SQLデータベース通知のサポートも提供します。







標準データベースレベルの場合、現在のストレージ使用量に関する情報を取得できます。 プレミアムレベルのデータベースの場合、IOPSおよびCPUコアメトリックも利用できます。



Windows Azure PCI DSS認定
