Backup Exec 15を正しく選択して購入する方法

Symantec Backup Exec 15は、仮想環境、物理環境、クラウド環境向けの使いやすいバックアップおよびリカバリソフトウェアであり、最新バージョンのVMwareおよびHyper-Vソフトウェアと統合されて、仮想マシン、アプリケーション、およびそれらのオブジェクト、データベース、ファイルを迅速に復元します。 しかし、冗長な機能を過剰に支払わずにそのような製品を購入するのはどれほど簡単ですか？







今年、シマンテックは、バックアップ、アーカイブ、データ復旧のための主要なソフトウェアソリューションの新しいバージョンをリリースしました。 Backup Exec 15は、ITインフラストラクチャ全体を保護し、物理的環境、仮想環境、クラウド環境の情報を保護する総合的なアプローチを実装し、異種ITインフラストラクチャ全体に単一のバックアップポリシーを実装できるという点で興味深い。 この製品は、VMwareの強化されたサポート、高いスケーラビリティとパフォーマンス、簡素化されたアップグレード、仮想化プラットフォームVMware vSphere（5.1 / 5.5 / 6.0）およびMicrosoft Hyper-V（2008/2012 / 2012R2）を含むクラウドストレージ、オペレーティングシステムおよびアプリケーションのサポートを提供します。



仮想環境と物理環境用の単一のソリューションと1つの管理コンソールにより、コストを削減し、IT環境の保護を簡素化できます。また、データバックアップシステムの展開とメンテナンスを一元化する機能、BE 2010 R3以降のBackup Execのバージョンから新しいソフトウェアバージョンへの簡単なアップグレード、ストレージコスト、さらにコストを削減できます。



Backup Exec 15は、新しいバージョンのVMware vSphere 6をサポートしています。ESXi6.0、vCenter 6.0、Virtual SAN 6、Virtual Volumes（VVOL）。 2 TBを超える容量のディスクを持つ仮想マシンを選択的に復元する機能。 アマゾンウェブサービスの組み込みクラウド接続-Storage Gateway VTL（クラウドストレージ）; Oracle 12cの新しいバージョンの詳細（選択的）リカバリのサポート。 vSANおよびSANを介したバックアップとリカバリ。 1つのタスクでの複数のサーバーとアプリケーションの同時バックアップ。 ディスクストレージ上のデータバックアップの自動ライフサイクル管理と、新しいデータを記録するためのディスクスペースの解放。 異なるデータタイプの同じジョブテンプレートで完全、増分、差分バックアップを組み合わせる機能、およびデータボリュームとCapacity Liteエディションの新しい簡易ライセンスモデル：十分な基本機能を備えたユーザーが使用できます。



Backup Execライセンス15



Backup Execは使いやすいだけでなく、ライセンスも簡単です。 ライセンスのコストは、必要なデータ保護レベルとソリューションの機能に依存します。



管理サーバーなどの必須コンポーネントに加えて、Backup Execには追加のコンポーネント（エージェントとオプション）を含めることができます。 エージェントは、保護されたデバイス、アプリケーション、データベース、オプションを接続して、製品の基本機能を拡張するように設計されています。 Backup Execの以前のバージョンと同様に、それらは個別に購入できます。

ライセンスの数は、NASシステムの数によって異なります。



BE 15は、簡素化されたライセンスモデルを提供します。ライセンスのコストは、保護するデータの量に依存し、既に選択されたオプションを備えた製品の2つのエディションを提供します







表からわかるように、Backup Exec 15 Capacity Edition LiteとBackup Exec 15 Capacity Editionのエディションは、サポートされる機能が異なります。「軽量」バージョンでは、集中管理、重複排除、ディスク使用時の高度なバックアップ、NDMPプロトコルのサポート、ファイルアーカイブおよびメールメッセージはありません、1つの物理または仮想テープライブラリのみがサポートされ、Macプラットフォームはサポートされていません。 このエディションは、1〜3台のサーバーを持つ小規模企業を対象としています。



どちらの場合のライセンスのコストも、バックアップされるデータの量に依存します。 サポートされるオプションとエージェントの観点から見ると、違いは次のとおりです。







テラバイト数に応じて支払われるBackup Exec Capacity Editionの各ライセンスには、Backup Execバックアップサーバー、およびBackup Execのすべてのエージェントとコンポーネントのライセンスが提供されます。 同時に、バックアップデータの量（圧縮および重複排除なし）は、ライセンスで指定された値を超えてはなりません。 シマンテックでは、1、2〜5、6〜15、16〜25、26 TBを超える「バックアップレベル」を提供しています。



いくつかの例を見てみましょう。



  1. 3 TBのデータのバックアップが必要です（ボリュームが増加する可能性があるため）。

    この場合、2〜5 TBのライセンスを持つ3 x Backup Exec Capacity Editionが必要です。



  2. 災害対策のために、7 TBのデータを予約し、それらをリモートサイトに複製する必要があります。

    このオプションでは、6〜15 TBのBackup Exec Capacity Editionを7個購入する必要があります。



  3. プロバイダーのリモートサイトに2テラバイトのデータを複製する必要があります。

    2 x 2-5 TBのBackup Exec Capacity Editionが必要です。



  4. 1 TBのデータを予約する必要があります。 特別なバックアップ要件はありません。

    このオプションでは、1 TBあたり1 x Backup Exec Capacity Edition Liteで十分です。


一般に、1年間予約する必要があるテラバイトのデータのライセンスを購入する必要があります。 データ量の増加を予測するのが難しい場合は、20％の成長に集中することをお勧めします。 参考のため、次の表を使用することもできます。左側はテラバイト単位の現在のデータ量で、右側はライセンスの購入量です。









しかし、5 TBのライセンスを購入した後、さらに1 TBを購入する必要がある場合はどうでしょうか？ そのような機会があり、シマンテックは追加容量のライセンスを割引価格で提供します。 ただし、追加オプションを購入しても機能しません。 たとえば、Capacity Edition Liteの重複排除オプションは購入できないため、Capacity Editionパッケージ全体にアップグレードする必要があります。 「テラバイト単位」のライセンスを他のライセンスモデルと組み合わせることもできません。 複数のサイトがある場合は、それぞれに同じエディション（Capacity EditionまたはCapacity Edition Lite）を購入する必要があります。



ただし、Capacity Edition LiteからCapacity Editionに移行することは可能です。 この場合、割引が適用されます。 また、他の既存のライセンスの所有者は、容量によってライセンスに切り替えることができます。



可能なオプション



BE 15の容量と新しいLiteエディションのライセンスに加えて、以前のライセンスオプション（V-Ray Edition、Small Business Edition）も利用できます。









このシナリオを検討してください。バックアップ用のサーバー（仮想または物理）、仮想Exchangeサーバー、仮想SQLサーバー、仮想Active Directoryサーバー、Hyper-Vホスト、1 TB未満のデータがあります。 この状況でBE 15のどの購入オプションを選択できますか？ それらの4つがあります。1. 1 x Capacity Liteエディション。 2.個々のコンポーネントの購入-Backup Execサーバーx 1、VMware＆Hyper-V用エージェントx 1、アプリケーションおよびデータベース用エージェントx 2。 2 x 4コアの3.4 x V-Rayエディション。 4. 1 x Small Business Editionおよび2 x Small Business Edition用エージェント。 各オプションには価格がありますが、新しいライセンス方法では70％以上に削減できます。



別の例を見てみましょう。バックアップ用の3つのサーバー（リモートブランチ内）、1つの仮想Exchangeサーバー、3つのSQLサーバー、1つの仮想ファイルサーバーがあります。 1つのVMwareホストと3 TB未満のデータ。 これには以下が必要です。1. 3 x Capacity Liteエディション。 2.個々のコンポーネントの購入-Backup Execサーバーx 2、VMware＆Hyper-V用エージェントx 1、アプリケーション＆データベース用エージェントx 3、重複排除オプションx 1、エンタープライズサーバーオプションx 1、重複除去サーバーx 1、BEサーバーx 1 ; 2〜4コアのV-Rayエディションx 3.4、アプリケーション＆データベース用エージェントx 3、重複排除サーバーオプションx 1。 さらに、災害復旧のためのクラウドへのバックアップ。



有効なサポート契約のある古いバージョンのBEのユーザーは無料でBE 15にアップグレードでき、有効期限が切れたユーザーには割引が適用されます。 BE 2010 R3 SP4、2012 SP4、または2014からBE 15に移行すると、構成されたすべてのバックアップポリシーが保存されます。
