PC / Win用のWeb開発用のソフトウェアセット

Webプロジェクトに関連するすべてをレイアウト/編集および作成するために毎日使用しているプログラムを記述し、記述したいという奇妙な夜の欲求がありました。 標準リストに加えて、これまたはその「1」が何をするのか、そしてなぜそれを使用するのかを簡単に説明したいと思いました。 おそらく誰かが役に立つでしょう。



Adobe Photoshop CS3：

さて、ここではすべてが明確です。 Photoshopは、Webサイトのグラフィックコンポーネントを準備する分野を含め、事実上の標準になっています。 それを追加できるのは、Web用に何かを準備するためだけです。最新バージョンを使用することは必須ではありません。CS2も優れており、95％の機能で十分です。



Zend Studio 5.5

Eclipseにはバージョンがありますが、古いバージョンを使用する方が便利です。 プラスの点-私は主にUTF-8で作業していますが、プロジェクトでマイナスのマイナスのデフォルトエンコーディングを設定できると便利です-Javaで書かれたすべてのもののように遅すぎる。 Zend.com Downloadsを知っている人はいないでしょうが、 念のために 。



TopStyle CSS Editor 3.5 Pro

私が見た中で最も便利なCSSエディターの1つ。 プラスは、編集の容易さ、タグの自動補完、およびそれらを最適化する手段です。 お気に入りのプラス点は、ウィンドウ内の要素の色を決定できる「ドロッパー」の存在です; FireFoxに追加のソフトウェアやプラグインを配置する必要はありません。 マイナスの点-多くの場合、コメントはWordPressの同じスタイルで必要であり、常に英語であるとは限りません。しかし、UTF-8ではTopStyleは機能せず、近い将来に機能します。 情報を知っている限り、そうするつもりはありません。 トップスタイルホーム 。



Dk LabsによるDenwer

PHPプログラマーのコミュニティには「デンバーで自分を殺す」というスローガンがありますが、私は個人的に、サイトのローカルデバッグのために使用するのが便利であるという事実のみを述べることができます。 プラスの点は、Apacheに仮想ホストを追加/削除するための柔軟な構造です。マイナスの点-Denwerを「手動で」設定することは最も簡単なタスクではなく、「ネイティブ」のApache参照は役に立たない場合があります。 オフサイト 。



Mozilla Firefox 2

さて、ここではすべてが明確です。 2つのブラウザーを開いたままにしないように（レイアウトの過程でIE / FFのみを開いたままにして、後でSafari / Operaの有効性を確認します）、IEViewがハングします。HTMLValidatorのHTML検証のために、Firebugがここに追加されます。 サイトはwww.mozilla.ruで 、誰もがそれを知っていると思います。 :)
