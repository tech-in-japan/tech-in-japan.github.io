RDPを介してWindows、UbuntuまたはDebian（Linux）、Mac OSに接続する方法、およびAndroidスマートフォンとiPhoneから接続する方法

VPS.houseに仮想サーバーを作成すると、完全にすぐに使用できるWindows Serverオペレーティングシステムが得られ、RDPプロトコルを使用して外部デバイスからネットワーク経由ですぐに接続できます。



RDP（リモートデスクトッププロトコル）は、Windowsのリモート制御用にMicrosoftが開発した特別なプロトコル（リモートデスクトッププロトコル ）です。 もちろん、最初はWindowsでの多くのことと同様に、他の人がこのプロトコルを開発しましたが、現在ではMicrosoftをサポートおよび開発しています。



ライセンス条件に従って、Windows Server OSはデフォルトでサーバーへの同時RDP接続を2つまで許可します。さらに必要な場合は、ターミナルサーバーが認識され、ユーザーの数または接続されたデバイスの数がライセンスされます。 ただし、RDPを介してサーバーに接続すると、リモートデスクトップ管理に便利なソリューション（TeamViewer、RAdminなど）を配置することもできます。



この記事では、多くの一般的なオペレーティングシステムからRDPを介してWindowsサーバーに接続するプロセスについて説明します。 これを考慮して、かなり長いことが判明しました。 このメニューで必要なセクションにすぐにジャンプできます。









デスクトップWindowsからVPSサーバーに接続する



このマニュアルはテスト済みで、パソコン用のWindowsのすべての一般的なバージョン（XP、Vista、Windows 7、8、8.1、10）で正常に動作します。



各Windowsオペレーティングシステムには、RDP経由で接続するための組み込みアプリケーションがあります。これは、プログラム「リモートデスクトップ接続」（英語OSのリモートデスクトップ接続）です。

開いたウィンドウで、仮想サーバーのIPアドレスを入力し、「接続」ボタンをクリックします。 [マイサーバー]セクションで、個人アカウントのサーバーのIPアドレスを確認できます。











2つ以上のIPアドレスを持つサーバーを注文した場合、それらのいずれかを使用してサーバーに接続できます-これは何にも影響しません。



リモートデスクトップの作成に初めて直面する人は、「なぜこれらのIPアドレスが必要なのか、どれだけ必要なのか」と自問し、IPアドレスの数をデスクトップまたはアカウントの数と混同するか、またはサーバーを1つのアドレスで開き、ブラウザを開くと、ブラウザが使用するのはこのアドレスになります。 実際、これはサーバーに接続できる外部アドレスのリストに過ぎず、サーバー自体の作業は接続後も変更されません。



ほとんどの場合、接続時のデフォルト設定はクリップボードからのデータの自動転送であり、サーバーにも接続して、接続元のデバイスのローカルディスクで表示されるようになります。 [接続]ボタンをすぐにクリックすると、リモートコンピューターが自分とリモートの両方に損害を与える可能性があることに関する通知が表示されます。 これは、ウィルスに感染している可能性がある他の誰かのサーバーに接続する場合、または他の誰かの感染したPCからサーバーに接続する場合に非常に可能です。







新しく作成されたクリーンなサーバーに接続することにより、以降の通知を安全にオフにして、接続プロセスを続行できます。



次に、認証データを入力するウィンドウが表示されます。







新しいサーバーを作成するたびに、VPS.houseシステムはそのサーバー用の新しい一意のパスワードを自動的に生成しますが、すべてのWindowsサーバーでは、デフォルトのユーザー名はAdministratorです。 サーバーのパスワードは、サーバーの近くのアカウントの[マイサーバー]ページにも表示されます。







セキュリティ上の理由から、デフォルトでは、サーバーからのパスワードは非表示になり、「パスワードを表示」リンクをクリックするだけで表示され、サイトはVPS.houseのアカウントのパスワードを入力するように求めます。



パスワードをコピーして貼り付けた場合にWindows 7に接続しようとすると、システムが誤って入力された資格情報に関するエラーを表示することが非常に多いことに注意してください。他のバージョンでも同じことが起こります。 これが発生した場合は、パスワードを手動で入力し、大文字と小文字を区別します（大文字を大文字で、小文字を小文字で、厳密に英語キーボードレイアウトで入力します）。



承認に成功した後、セッションを開始する前に、リモートコンピューターの認証の失敗に関する通知を受け取ります 。 この通知の理由は、サーバー側のセッションは、サーバーが生成したSSL証明書を使用してデフォルトで暗号化されるためであり、認証された認証局によって発行される可能性があるためではありません。







この通知はセキュリティの問題を示すものではなく、「このコンピューターへの接続要求を再度表示しない」チェックボックスをオンにすることで、同じサーバーへの今後の接続に対して安全にオフにできます。



操作にWindows XPを使用している場合、接続時に「リモートコンピューターはこのコンピューターがサポートしていないネットワークレベルで認証が必要です」というテキストのエラーが表示されます。これは、PCが非常に古く、小さなアップデートをインストールする必要があることを意味します。インストール方法については、こちらで詳しく説明しています 。



リモートデスクトップに接続するための組み込みプログラムを使用すると、ファイルを簡単にサーバーに転送し、PCにコピーして、既にサーバー自体にあるフォルダーに貼り付けることができます。 さらに、事実上すべてのUSBデバイス、ネットワークプリンター、スキャナーのサーバーへの転送を有効にすることができます。



これらの機能を有効にするには、サーバーに接続する前に、「パラメーターを表示」ボタンをクリックします。







これにより、VDSサーバーに接続するための詳細設定モードが開きます。 [ローカルリソース]タブに移動し、必要なデバイスをチェックオフします。











もう1つの非常に便利で人気のある機能は、必要なすべての設定と認証データが既にサーバーに保存されている既製の接続ファイル（いわゆる「ショートカット」）を作成する機能です。 これは、経験のあるPCユーザーではない従業員にアクセスを許可する場合に特に便利です。







これを行うには、[全般]タブの詳細接続設定でもユーザー名を入力し（管理者のVPS.houseサーバーでは常にデフォルトです）、[資格情報の保存を許可する]チェックボックスをオンにして、毎回、入力完了時にパスワードを入力しないようにします必要な他のすべての接続パラメーター（存在する場合）、[名前を付けて保存]ボタンをクリックします。







その結果、同僚に送信できる接続を備えた既製のファイルを受け取り、その同僚はマウスで2回クリックするだけでサーバーに接続します。



セキュリティ上の理由から、デフォルトでは、アカウントのパスワードはこのファイルに追加されず、最初の接続時に要求されます。







Ubuntuを使用している場合、RDP経由でサーバーに接続する方法



Windows Dedicated Desktop Connection Protocol（RDP）はMicrosoft独自のプロトコルです.MicrosoftはLinuxオペレーティングシステムの公式サービスをリリースしていませんが、とにかく、Ubuntuの最新版の元のアセンブリにさえ含まれている安定したソリューションがしばらくの間ありました。

Remminaというクライアントについてです



デフォルトでは、アセンブリにRemminaが含まれている場合、「リモートデスクトップクライアント」の検索で見つけることができます。含まれていない場合は、ターミナルで次のコマンドを使用してインストールします。







  1. Remminaパッケージをインストールする
    sudo apt-add-repository ppa:remmina-ppa-team/remmina-next



  2. 更新プログラムをインストールする
    sudo apt-get update



  3. RDPプロトコルプラグインをインストールする
    sudo apt-get install remmina remmina-plugin-rdp libfreerdp-plugins-standard



  4. 以前にRemminaのバージョンをインストールしていた場合、またはインストール前に実行していた場合は、再起動する必要があります。 これを行うには、コンピューターを再起動するか、次のコマンドを使用するのが最適です。
    udo killall remmina



    実行後に「プロセスが見つかりません」などのエラーメッセージを受け取った場合、すべてが正常であり、コマンドを実行できませんでした。




検索メニューで、「リモートデスクトップ」または「remmina」と入力して、インストールされているアプリケーションを見つけます。







Remminaでは、接続リストを追加して保存し、サーバーにすばやくアクセスできます。 新しい意識については、画像に示すように「+」をクリックします。







個人アカウントに示されている認証データをフィールドに入力します 。







「詳細」タブでは、サーバーへの接続の品質と画像送信時の詳細を指定することもできます。







データを保存すると、Remmina接続リストでサーバーをいつでも見つけることができます。 サーバーに接続するには、サーバー行をダブルクリックします。







このアプリケーションから初めてサーバーに接続する場合、セキュリティ証明書にエラーに関する情報が表示される場合があります。 これが発生する理由は、デスクトップWindowsからサーバーに接続するブロックで前述したとおりです。

[OK]をクリックするだけで、プログラムは引き続きサーバーに接続します。















Debianで作業している場合、RDP経由でサーバーに接続する方法



RDP（リモートデスクトッププロトコル）は、閉じられたMicrosoftリモートデスクトッププロトコルです。 残念ながら、LinuxベースのオペレーティングシステムからWindowsサーバーに接続するための公式クライアントはリリースしていません。 しかし、かなり長い間、安定した実用的なソリューションがあります。

最も人気のあるものの1つは、すべての種類のリモートRemminaのクライアントです。VPS.houseサーバーまたはその他の実行中のWindowsへの接続に使用することをお勧めします。







  1. インストールファイルへのパスを指定する
    echo 'deb http://ftp.debian.org/debian stretch-backports main' | sudo tee --append /etc/apt/sources.list.d/stretch-backports.list >> /dev/null



  2. インストールプロセスを開始します
    sudo apt update



  3. RDPプロトコルプラグインをインストールする
    sudo apt install -t stretch-backports remmina remmina-plugin-rdp remmina-plugin-secret libfreerdp-plugins-standard





インストール直後に、プログラムを検索することでアプリケーションを見つけることができます。







標準のWindows OS RDPクライアントとは異なり、Remminaを使用すると、さまざまなサーバーへのすべての接続をリストに保存して、すぐにアクセスできます。 最初のサーバーをリストに追加するには、左上隅の「+」をクリックします。







開いたウィンドウで、サーバーに接続するためのパラメーターを設定し、覚えやすいサーバー名を入力します。 接続に必要なすべてのデータは、個人アカウントに示されています 。







[詳細設定]タブでは、サーバーからPCに音声を送信するためのパラメーターと、接続時の送信画像の品質を設定できます。







I設定を保存すると、接続のリストに新しい行が表示されます。 サーバーの操作を開始するには、マウスでサーバーをダブルクリックします。







サーバーへの最初の接続時に、Remminaは信頼できない暗号化証明書を含む通知を表示します。 これはバグやセキュリティの問題ではありません。 このメッセージの理由は上で説明されます 。 「OK」をクリックするだけで、サーバーのデスクトップが表示されます。もちろん、接続のすべてのデータが以前に正しく入力されていたことが条件です。















Mac OSからRDPを介してサーバーに接続する



Mac OSのリモートデスクトップに接続するために、Microsoftは公式のRDPクライアントを開発およびサポートしています。 Windows OSのどのバージョンでも安定して動作します。

ダウンロードするには、iTunes Webサイト（ https://itunes.apple.com/gb/app/id715768417）にアクセスしてください。



このプログラムには直感的なインターフェイスがあり、サーバーのリストを作成して、さらにすぐに接続できます。

リストに新しいサーバーを追加するには、スクリーンショットに示すように「新規」をクリックします。







開いたウィンドウで、個人アカウント （サーバーIPアドレス、管理者ログイン、およびパスワード）に示されている承認データを指定し、新しい接続の任意の名前（接続名）を指定します。



エントリが完了したら、ボタンをクリックしてウィンドウを閉じます-すべてのデータが保存され、接続リストにサーバーの行が表示されます。







この行をダブルクリックすると、サーバーに接続されます。



Windows Server 2008以降で実行されているVPSサーバーに接続しようとすると、暗号化証明書を検証できなかったという通知が表示されます。 これはセキュリティの問題ではありませんが、証明書が認定センターによって発行されたのではなく、サーバー自体によって生成されたことを示しています。



このメッセージが今後このサーバーに表示されないようにするには、[証明書を表示]をクリックします。







[常に信頼する...]チェックボックスをオンにして、[続行]をクリックします。







接続設定ですべてのパラメーターがエラーなしで入力された場合（IPアドレス、ログイン、パスワード）、Windowsサーバーのデスクトップがすぐに表示されます。











iOS上のスマートフォンまたはタブレットからVDSサーバーに接続する（iPhoneまたはiPadから）



iOSは、リモートデスクトップへの快適で安定した接続のための本格的な公式RDPクライアントをリリースします。 アプリケーションはMicrosoftリモートデスクトップまたはリモートデスクトップと呼ばれます ： https : //itunes.apple.com/en/app/id714464092







Microsoftリモートデスクトップを使用すると、頻繁に接続するために使用するサーバーのリストを事前に構成できます。



彼の仕事の詳細は、アカウントに関する情報が作成され、サーバーのリストとは別に保存されることです。 したがって、まず、管理者ユーザーアカウントを追加する必要があります。 これを行うには、設定ボタン（左上隅の歯車アイコン）をクリックし、「アカウント」を選択します。







開いたウィンドウで、「ユーザーアカウントの追加」をクリックし、個人アカウントに表示される管理者のログインとパスワードを入力します 。







完了すると、管理者アカウントがリストに追加されていることがわかります。[完了]ボタンをクリックすると、スタート画面に戻ります。



次のステップは、サーバー自体に関する情報を直接追加することです。 アプリケーションの右上隅にある「+」ボタンを押し、開いたメニューで「デスクトップ」を選択します。







新しい接続を追加するウィンドウで、サーバーのIPアドレスを指定し、最後の手順で追加した管理者アカウントを選択します。







最初の接続時に通知が表示され、信頼できないセキュリティ証明書に関する通知が表示されます。 これの理由は上で説明されます 。 [このPCを再度要求しない]を強調表示し、[同意する]を押します。







IPアドレスと自動回転データがエラーなしで指定されている場合、仮想サーバーに正常に接続します。











Android上のスマートフォンまたはタブレットからVPSサーバーに接続する方法



Androidデバイスの場合、Microsoftは本格的なリモートデスクトップアプリケーションであるMicrosoft Remote Desktopをリリースしています 。 Google Playの他のアプリと同じようにダウンロードします 。







上記のPCに接続するためのすべてのアプリケーションとは異なり、Microsoftリモートデスクトップモバイルアプリケーションは、アカウントと接続リスト自体を分離します。 そのため、最初に管理者ユーザーアカウントをパスワードとともにリストに追加する必要があります。パスワードは個人アカウントに表示されます 。











アカウントを追加したら、アプリケーションのメイン画面に戻り、接続自体（VDSサーバーのアドレス）を追加します。







開いたウィンドウで、サーバーのIPアドレス（ アカウントで指定）を指定し、以前に追加した管理者アカウントを選択して[保存]をクリックします。







サーバーがメイン画面の接続リストに表示されます。サーバーをクリックするだけで、プログラムが接続を開始します。 このアプリケーションからサーバーに初めて接続する場合は、アプリケーションが暗号化証明書を検証できなかったことを示す情報メッセージが表示されます。 これが起こる理由は、記事の冒頭で説明されています 。







[このPCへの接続を再度要求しない]をオンにすると、この通知は表示されなくなります。 次に、「接続」をクリックし、すべての自動回転データとサーバーIPアドレスがエラーなしで指定されている場合、正常に接続します。









PS：VPSホスティングのクライアントは、しばしば1つまたは別のデバイスからサーバーに接続する方法を自問します。 検索エンジンでは、写真やビデオを含むこれらすべての指示を見つけることができます。 この記事が、最も人気のあるすべてのデバイスとオペレーティングシステムから接続するための指示を1か所に収めて、ここでも役立つことを願っています。
