ECS（Elastic Cloud Storage）-Dell EMCクラウドストレージプラットフォーム

非構造化データの量は指数関数的に増加しており、このプロセスが遅くなったようには見えません。 一部の組織は、従来のストレージインフラストラクチャを使用して管理を続けていますが、高価であり、データ量の増加に応じたスケーリングをサポートしていません。



しかし、これらの問題は唯一のものからはほど遠い。 IT予算は同じレベルにとどまるか、わずかに増加し（年間約5％）、ほとんどの組織の資本支出は毎年ほぼ2倍になります。 さらに、企業はサービスのレベルを維持および改善し、これに費やすリソースを少なくする必要があります。









データ量（およびそれに対応するストレージインフラストラクチャの要件）の増加は決して一時的な傾向ではなく、企業が時間通りにビジネスを変革しないと、非常に深刻な問題が発生します。 しかし、テクノロジーの開発により、必要なソリューションを見つけることができます。 世界最大のストレージプロバイダーであるDell EMCは、費用対効果の高いECSクラウドプラットフォームを作成しました。



ECSとは何ですか？



まず、ユーザーが自分のデータの保存場所を知らない場合、ECSはパブリッククラウドストレージではないことに注意してください。 それどころか、 Dell EMC（Elastic Cloud Storage ）は物理ストレージアレイであり（ソフトウェア形式のバージョンはありますが）、クライアントはそれを配置する場所を選択します。 また、システムをこの目的に使用できるため、ECSはク​​ラウドストレージと呼ばれます。



ECSソフトウェアを使用すると、標準コンポーネントの大規模インフラストラクチャに非構造化データを保存、管理、分析できます。 これは、包括的なマルチユーザークラウドストレージプラットフォームであり、リソース消費を考慮して、個別に割り当てることができます。



ECSは、完全なハードウェアおよびソフトウェアシステム（Dell EMC ECSアプライアンス）またはソフトウェア製品（ECSソフトウェア）として購入できます。ECSソフトウェアは、お客様のサーバーおよび一般的なDell EMC認定ドライブにインストールされます。 ECSには、オブジェクト管理とHDFSサービス、およびファイルサービスのサポートをサポートする非構造化データストレージモジュールが含まれています。



技術的には、このシステムは次のとおりです。



  • JBODシェルフが接続されたIntel x86サーバー（サーバーあたり最大60台のディスク）、

  • ノードを接続するための10GbEネットワークアダプター。



システム構成には4/8ノードが含まれます。 ソリューションの容量は、ラックあたり3840 TBに達します。



ECSは、第三世代のオブジェクトストレージシステムです。 次のことができます。



  • コアストレージリソースを統合し、投資収益率を向上

  • 従来のアプリケーションとレガシーアプリケーションをアップグレードして、ストレージをより効率的に使用する

  • 新しいビジネス機能でクラウドベースのアプリケーションを高速化します。



オブジェクトストレージの利点：



  • 施設へのアクセスは、専用のプロトコルによって提供されます。 ブロックやファイルとは異なり、HTTP上で動作し、RESTに基づいています：S3、Swift、Atmos。

  • 非構造化情報（クライアント側の静的または動的コンテンツであるファイル）の保存に焦点を当てます。

  • 各オブジェクトには一意のアドレスがあります。

  • グローバルオブジェクト名前空間：任意のオブジェクトが任意のインフラストラクチャセグメントからいつでも利用可能です。



ECSには何が便利ですか？



  1. 収益性。 ECSは、多くの使用シナリオを提供し、Dell EMCストレージインフラストラクチャへの投資を保護する機能豊富なプラットフォームです。 管理を簡素化し、柔軟性を高め、最も重要なこととして、コストを削減します。 スケーラビリティの観点から、ECSは市場で最も費用対効果の高いソリューションの1つです。 Enterprise Strategy Group（ESG）の調査によると、ECSにファイルを保存することは、他のクラウドサービスよりも少なくとも60％高い利益をもたらします。



  2. 既存のストレージ環境との統合。 ECSはク​​ラウドをプライマリストレージに「拡張」し、Dell EMCクラウドコンピューティングソリューション（CloudPools、CloudBoost、CloudArrayなど）でストレージインフラストラクチャを解放します。 お客様は、非アクティブなデータをプライマリストレージ（Isilon、VMAXシリーズ、VPLEX、VNXシリーズ、Vxシリーズ、Data Domain、Data Protection Suiteなど）からECSに移動することでクラウドにプッシュできます。 このリソースの統合により、追加の高価なプラットフォームを購入する必要がなくなり、既存のストレージ環境の効率が大幅に向上します。



  3. オンラインおよびユビキタスなデータアクセス。 ECS Object Platformは、従来のNASと比較して、応答時間を大幅に改善し、データアクセス速度を向上させることができます。 データはエンコードによって保護され、システムのすべてのノードに分散され、どこからでも読み取り/書き込みに瞬時にアクセスできます。 同時に、データの最新のコピーが利用できるため、アプリケーション開発が簡素化されます。 また、ジオキャッシングのサポートにより、操作パターンのインテリジェントなシステム認識のおかげでアクセスが高速化されます。WANトラフィックを最小限に抑え、遅延を改善します。



  4. アプリケーション開発の容易さ。 単一のグローバル名前空間では、開発者が複雑なNFSファイルシステムにアクセスする必要はありません。 運用の詳細ではなく、アプリケーションの構築に集中できます。 オブジェクトストレージ内の従来のアプリケーションをアップグレードすることにより、ユーザーは追加のリソース、インターネット経由のコンテンツへのアクセス、および単一のネームスペースを介したグローバルなアクセシビリティを取得します。 さらに、データセンターのストレージリソースを最適に使用できます。 ネイティブクラウドアプリケーションは、クラウドインフラストラクチャを最大限に活用します。







  5. ユーザーは、ソフトウェア定義ストレージのすべての利点にアクセスできます。 ECSプラットフォームは、ソフトウェア定義のアーキテクチャを最大限に活用しており、コンピューティングリソースとストレージリソースを個別にスケーリングできます。 ECSのすべてのデータはコンテナー化されています。 ECSは複数のプロトコルをサポートしているため、ハードウェアに依存せず、アプリケーションをトランスコードまたは再構成する必要がありません。 これにより、開発者はより迅速に革新を行い、アプリケーションを市場に投入できます。



デジタル変換は簡単です



企業およびクラウドサービスプロバイダーは、ECSを「デジタルトランスフォーメーション」の基盤として使用できます。 従来のビジネスアプリケーションはクラウドアプリケーションに置き換えられ、ECSは「2番目」と「3番目」のプラットフォームのアプリケーション間のギャップを埋め、単一のストレージシステムを使用できるようにします。 ECSは、大量の非構造化データに対応するだけでなく、多目的プラットフォームとしてさまざまな負荷がかかり、あらゆる課題に対応できるようになります。









ECSは、HDFS、OpenStack Swift、Amazon S3、CenteraおよびAtmos APIと互換性があります。 お客様の選択により、ラックあたり3.8 PBまで拡張可能な完全統合ソリューションとして、または認定機器にデータを保存するためのソフトウェアソリューションとして展開できます。 さらに、ECSは使用ごとに利用できます。 これにより、企業の顧客やサービスプロバイダーの間で人気が高まっています。









製造業者は、必要な場合にECSを配置します。



  • 分散アーキテクチャ

  • グローバル名前空間（アクティブ/アクティブノード、ジオプロテクション、マルチプロトコルアクセス、メタデータ検索を備えたすべてのデータセンター）。



ECSの使用シナリオ



ECSプラットフォームは次のように使用されます。



  1. グローバルコンテンツリポジトリ。 この比較的簡単に実装できるソリューションは、グローバルファイルアクセスとエンタープライズレベルのデータ保護を提供します。

  2. ビッグデータ分析の大規模な実装。 このシステムは、地理的に分散したアーカイブからのデータのグローバル分析を提供します。 これは、Hadoop向けの経済的な高密度ストレージソリューションです。

  3. アプリケーション用の最新のプラットフォーム。 ECSは多くのAPIとプロトコルをサポートし、そのアクティブ/アクティブアーキテクチャはデータストレージとアクセス用に最適化されています。

  4. 非アクティブなデータコールドアーカイブシステム。 将来のアプリケーションをサポートする準備ができたアーカイブを作成し、既存のストレージ環境を最適化できます。

  5. モノのインターネット（IoT）の管理プラットフォームを展開します。 IoTアーキテクチャは、大量の非構造化データの処理に効果的です。 そして、最近のDell EMC World 2017フォーラムで、IoTストリーミングがECSで可能になったことが発表されました。









他のDell EMCストレージシステムでのECSの使用例を次に示します。



例1. ECSとDell EMCクラウドアレイの組み合わせ。 ストレージアレイをリモートサイトに展開せずに、リモートオフィス用のファイルサービスとストレージ。









例2. ECSとIsilon（クラウドプール）の組み合わせ。 指定されたルールに従って低コストのストレージに自動ファイル転送するIsilon（水平スケーリングNAS）。









例3.従来のアーカイブ。 ECSはアーカイブとして使用されます。



例4. Dell EMC Cloud Boostとの組み合わせ。 長期的なデータストレージ。









例5.オブジェクトストレージを使用した開発。 モバイルアプリケーションは非常に開発されたため、Webサーバーなどの「ゲートウェイ」なしでは、よく知られているストレージプロトコル（FC、NAS）と関連付けることは困難です。 ECSのおかげで、モバイルアプリケーションはHTTP / HTTPSを使用してリポジトリに直接アクセスし、必要なオブジェクトを受信または保存できます。



ECSプラットフォームは2015年6月にリリースされました。 現在では250を超えるクライアントで使用されており、それらの多くはインストール容量がペタバイトを超えています。 ECSは、オブジェクト、NFS、およびHadoop分散ファイルシステム（HDFS）をサポートしており、クラウドアプリケーション用に数十億のファイルを保存できます。
