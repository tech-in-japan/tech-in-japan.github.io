システム全体のホットキー



「3本指の組み合わせ」を知っています：Ctrl + Alt + Del。 Windowsでは、プロセスウィンドウの呼び出しに使用されていました。 Ctrl + Alt + Delをもう一度押すと、コンピューターが再起動します。 Linuxでは、この組み合わせを押したときの応答は、/ etc / inittabファイルで指定されます。

shutdownコマンドの-r nowパラメーターは、システムをリブート（-r）する（今）必要があることを意味します。 別のコマンド、たとえば再起動が必要な場合や、Ctrl + Alt + Delへの反応が無効になっている場合があります。



次のシステム全体のキーの組み合わせは、Ctrl + Alt + Backspaceです。 このキーの組み合わせを押すと、実行中のX Windowサーバーが再起動します。



コンソールで作業するためのキーボードショートカット



Linuxには複数の仮想コンソールがあります（通常6）。 Alt + Fnキーの組み合わせにより、それらを切り替えることができます。nはコンソール番号です。 ただし、このキーの組み合わせは、現在コンソールで作業している場合にのみ機能し、KDEや他のグラフィカル環境では機能しません。 たとえば、現在コンソール1で作業しており、コンソール4に切り替える必要があります。その後、Alt + F4のキーの組み合わせを押し、元に戻すにはAlt + F1を押します。



通常、Alt + F7の組み合わせを押すと、グラフィックモード（Xウィンドウシステム）に切り替わります。



コンソールのTabキー（またはX Window Systemのターミナルシステム）は、コマンドラインのオートコンプリートを実装しています。 必要なプログラムの起動に使用するコマンドの正確な名前を忘れたとします。 コマンドの最初の文字を入力してTabキーを押します-システム自体がコマンドラインを補完します。 Tabを押した後にビープ音が聞こえる場合、たとえば、daを入力し、システムが「da」で始まる2つ（またはそれ以上）のコマンドを見つけたため、どのコマンドに「入力」する必要があるかがわかりません。コマンドライン この場合、Tabをもう一度押すと、「da」で始まるコマンドのリスト全体が表示されます。



必要なコマンドを見つけて入力すると、プログラムは画面に情報を表示しました。 しかし、プログラムの出力は非常に大きく、画面に収まりませんでした。 もちろん、出力をファイル以下のプログラムにリダイレクトできますが、今後この出力を使用しない場合は、これを行う必要はありません。 キーボードショートカットShift + PgUpおよびShift + PgDnを使用してスクロールします。



X-Windowキーボードショートカット



これで、X Windowでの作業をお楽しみいただけます。 ただし、グラフィック端末に飽きたため、コンソールに切り替える必要がある場合があります。 これは、キーボードショートカットCtrl + Alt + Fnを使用して実行できます。nはコンソール番号（1..6）です。 キーボードショートカットAlt + F7を使用して元に戻すことができます。



モニターの解像度を変更するには、コンフィギュレーターを実行する必要はありません。 Ctrl +“ +”（テンキー上の）を押して解像度を（最大許容値まで）上げるか、Ctrl +“-”（テンキー上の）を押して下げるだけです。



簡単なヒント：弱いボード（RivaTNT2 Vanta 8MBなど）があり、お気に入りのおもちゃ（もちろんtuxracer）が「遅くなる」場合は、解像度を640x480に下げます。



古典的なウィンドウマネージャを備えた「クリーンな」Xウィンドウシステムでは、実際にはだれも動作しないため、追加のキーの組み合わせを検討する理由はありません。 KDEに直行しましょう。



KDEキーボードショートカット



Microsoftは、スタートキーを押してWindowsでの作業を開始することを提案しています。 KDEは、このためにKというラベルの付いたボタンを使用します（より正確には、KDE自体のロゴ付き）。 大型モニター（19インチ以上）があり、マウス（きれいにするのを忘れた）を右上から左下隅にドラッグするのが面倒な場合は、Alt + F1を押すと、マウスポインターの現在の位置にKDEメニューが表示されます。



1つのコマンドを実行する必要がある場合、端末を実行する必要はほとんどありません（呼び出されたプログラムがグラフィカルである場合）、Alt + F2を押してプログラム起動ウィンドウを開きます-コマンドを入力するか、リストから選択して実行をクリックします。





図 1.プログラム起動ウィンドウ



Windowsと同様に、キーボードショートカットAlt + Tabを使用して、実行中のプログラムを切り替えることができます。 デスクトップに混乱が生じた場合は、別のデスクトップに切り替えましょう。 Ctrl + TabまたはCtrl + Fnのキーの組み合わせを使用してデスクトップを切り替えることができます。nは必要なデスクトップの番号です。 キーボードショートカットのAlt + Tabは、現在のデスクトップ内でのみ機能することに注意してください。 1つ目のデスクトップではなく、1つ目のデスクトップで数十のプログラムを実行している場合、2つ目のデスクトップでAlt + Tabキーを押すと、「タスクなし」というメッセージが表示されます。



Alt + F5を押すと、タスクのリスト全体を表示できます。 特定のプロセスが気に入らなかった場合は、タスクマネージャー（Ctrl + Escキーの組み合わせ）を呼び出してプロセスを強制終了できます。



キーボードレイアウトは通常の方法で切り替わりませんか？ CtrlもCtrl + Shiftも助けになりませんか？ 次に、Ctrl + Alt + Kというキーの組み合わせを押します。この組み合わせは、KDEで常に機能します。



クリップボードを操作するためのキーボードショートカットが最も一般的です-Ctrl + C（コピー）、Ctrl + X（カット）、Ctrl + V（貼り付け）。 これら3つの組み合わせに加えて、KDEはユーザーにさらに2つを提供します。



Ctrl + Alt + V-クリップボードメニューを表示

Ctrl + Alt + R-「手動」でクリップボードを操作



しかし、Windowsでの作業はどうですか Windowsの場合のように、Alt + F4の組み合わせはウィンドウを閉じ、Alt + F3-ウィンドウでのアクションのメニューを表示します。



xkillプログラムを呼び出すCtrl + Alt + Escの組み合わせは非常に便利です。 呼び出した後、マウスポインターは骨付きの頭蓋骨に変わります。 あるウィンドウの「頭蓋骨」をクリックすると、プログラムが「ハング」した場合でも破壊されます。 ウィンドウを破壊せずにxkillを終了するには、Escを押します。



Windowsでは、Print Screenキーを押すことで画面の写真を撮ることができます。 Linuxでは、これも実行できますが、デスクトップ全体を撮影するには、Ctrl + Print Screenを押して、アクティブウィンドウAlt + Print Screenを押します。 その後、受け取った画像はクリップボードに置かれ、グラフィックをサポートするKDEアプリケーションに貼り付けることができます。 ただし、ksnapshotなどの特別なプログラムでスクリーンショットを撮ることをお勧めします。 このプログラムを使用して取得した画像は、上記で提案した方法で取得した画像よりもはるかに優れており、ディスクスペースをほとんど占有しません（ksnapshotプログラムはPNG形式を使用するため）。
