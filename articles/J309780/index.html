iOSでブロックを使用します。 パート1

Objective-CとiOS開発を学ぶ過程で、ブロックがどのように機能するかを理解できませんでした。 メソッドにパラメーターとして渡すことができるのは混乱していました。 ブロックだけでなく、アプリケーション開発プロセスも考慮されたため、非常に興味深いと思われる記事に出会いました 。 投稿はxCode 7.3.1に適合しています。



まえがき



ブロックはC / Objective-Cへの非常に強力な追加機能です。これにより、コードの断片を個別のユニットに「ラップ」し、オブジェクトとして操作できます。 より多くのAPIがiOSでブロックを使用する必要があるため、それらを理解することは非常に重要です。 ただし、それらの構文といくつかの微妙な側面は、初心者にとってしばしば混乱を招きます。 しかし、恐れることはありません-このレッスンは非常に役立ちます。



このチュートリアルの2つのパートでは、「iOS Diner」という小さなiOSプロジェクトを作成します。 アプリケーション自体は簡単です。ユーザーはメニューから料理を選択して、まるで昼食を食べているかのように注文を作成します。 注意：創造の過程で、空腹感がエスカレートします！



最初の部分では、アプリケーションのUIを開発し、同時にストーリーボード （文字通り-ストーリーボード-約Per。）を確認します。これには、JSON形式でメニューを読み込むためのWebサービスの作成と使用に関する短いメモが含まれます。



ご注意 ストーリーボードとインターフェイスビルダーが既に得意な場合は、最初の部分をスキップして、ブロックの使用を開始する2番目の部分に直接進んでください。 このパートでは、 StoryboardとInterface Builderのみに焦点を当てます。



2番目の部分では、アプリケーションロジックのプログラミング、非同期データ処理、バックグラウンドタスク、多くの標準APIの代替としての使用など、ブロックの包括的な使用方法を示します。

IBOutletsおよびIBActionsをインストールする



次に、作成したユーザーインターフェイスとコードの間の関係を確立する必要があります。 これには、 IBOutletsとIBActionsが必要です。 IBはInterface Builderの略で、xCodeでUIを作成するために使用されます。



  • IBOutletは 、基本的に、UI要素（ボタンまたはラベル）とコード内のリンクとの関係です。
  • IBActionは、開発されたインターフェイスのイベント（ボタンクリックなど）に接続できるコード内のアクション（またはより便利なメソッド）です。


IBOutletsとIBActionsを介してUI要素を接続するためのコードを作成しましょう。



[ ユーティリティ ]タブを閉じて、 アシスタントエディタを開きます。 構成方法によっては、スクリーンショットがスクリーンショットと同じに見えない場合があります。 アシスタントエディターの表示を変更する場合は、そのメニューを見てください。







ボタンから始めましょう。 -1ボタンを選択し、 Ctrlキーを押しながらコードにドラッグします。 これにより、ボタンのIBOutletの作成が自動化されます。



このオブジェクトに必要なのは、名前を付けることだけです。 xCodeではオートコンプリートによりすべての要素を簡単に見つけることができるため、「ib」プレフィックスが好きです。 このオブジェクトに「ibRemoveItemButton」という名前を付け、[ 接続 ]をクリックします 。







残りのボタンについても同じことを行います。



















同じ方法でラベルを設定します。















これで、 ViewController.mは次のようになります。 



#import "ViewController.h" @interface ViewController () @property (weak, nonatomic) IBOutlet UIButton *ibRemoveItemButton; @property (weak, nonatomic) IBOutlet UIButton *ibAddItemButton; @property (weak, nonatomic) IBOutlet UIButton *ibPreviousItemButton; @property (weak, nonatomic) IBOutlet UIButton *ibNextItemButton; @property (weak, nonatomic) IBOutlet UIButton *ibTotalOrderButton; @property (weak, nonatomic) IBOutlet UILabel *ibChalkboardLabel; @property (weak, nonatomic) IBOutlet UIImageView *ibCurrentItemImageView; @property (weak, nonatomic) IBOutlet UILabel *ibCurrentItemLabel; @end @implementation ViewController - (void)viewDidLoad { [super viewDidLoad]; // Do any additional setup after loading the view, typically from a nib. } - (void)didReceiveMemoryWarning { [super didReceiveMemoryWarning]; // Dispose of any resources that can be recreated. } @end


ここで、 UIButtonsにIBActionsを追加する必要があります 。 これらは、特定のイベント（Touch Up Inside、Touch Up Outside、Touch Cancelなど）に応答して呼び出されるメソッドです。 ボタンの場合、最も一般的に使用される内部タッチアップ。



ボタン「-1」を選択します。 再度、 Ctrlキーを使用してViewController.mにドラッグします 。 これはIBActionであるため、ibaプレフィックスを使用します。 すべてのボタンについて繰り返します。
