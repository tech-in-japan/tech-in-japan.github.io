2017年にゲームボーイ用のゲームを作成するのはどうですか







誰もが子供時代の夢を見ていた。 個人的には、最初のコンソールであるNintendo Game Boy用のゲームを作成することを夢見ていました。 今日、私の夢が実現しました- ゲームボーイ用の最初のゲームを本物のカートリッジでリリースしました ： Sheep It Up！



この記事では、使用したツールと、プロジェクトを作成するために初心者が克服しなければならなかった障害について説明します。 この記事は非常に長いため、2つの部分に分かれています。



  • パート1：使用済みツール/技術的な問題/グラフィックの問題
  • パート2：サウンドの制限/カートリッジの作成/プレイヤーのレビュー（このパートは執筆中です）


ゲーム



「Sheep It Up！」 -これは、羊が登ってベルクロの留め具にしがみつくアーケードゲームです。 コンセプトはシンプルですが、ゲーム自体は急速に複雑になりつつあります。落下することなくどれだけ高く登れるのでしょうか？



ゲームプレイシープ・イット・アップ！






私は自分でコレクターなので、プロジェクトを古いゲームボーイのゲームの精神で作成したかったのです。 したがって、プリント基板、ROM、シェル、保護ケース、さらにはステッカーまで、すべてがこのゲーム専用に設計されています！ また、誰もがゲームを楽しめるように合理的な価格を維持しようとしました： $ 15（+配送料） 。 Super Game Boyを含む、最初からGBA SPまでのGame Boyモデルで動作します。











ゲームボーイをまだお持ちの場合は、出版社のウェブサイトでカートリッジを購入できます。

ツール：2017年、ありがとう！



90年代には、8ビットコンソール用のゲームを作成することは大きな課題であり、経験豊富な専門家のチーム全体が必要でした。 当時のデザイナー、アーティスト、デベロッパーは真のヒーローだと考えています。彼らはツールを使ってこのような素晴らしいゲームを作成することができました。その能力は現代のものよりもはるかに劣っています。



2017年には、8ビットコンソール用のゲームを作成するために一生懸命働く必要があります。 しかし、愛好家の素晴らしいコミュニティのおかげで、私たちは生活を楽にする多くのツールを持っています！ それらがなければ、私のような新人はゲームボーイだけのためにゲームを作成することはできません。 それでは、これらのツールは何ですか？



最初に、プログラミング言語についてお話します。 当時、すべての特殊なゲーム機器はアセンブラーでプログラムされていました。 まだ可能です（さらには推奨されます）。 しかし、8ビットおよび16ビットコンソール用の多くの開発キットはC言語に基づいているため、これが唯一のオプションではありません。



さらに、グラフィックスとレベルデザインを作成するために、グラフ用紙とハンドスキャナーは不要になりました。 幸いなことに、次の2つの補完ツールを使用できます。



  • ゲームボーイタイルデザイナー （GBTD） スプライトとタイルを描画し、それらをGame Boyが理解できるバイナリ形式にエクスポートできます。
  • ゲームボーイマップビルダー （GBMB） このツールを使用すると、GBTDで描画されたタイルに基づいてレベルと背景画像を構築できます（タイル張りに似ていますが、ゲームボーイ用です） 。








最後になりましたが、ゲームをテストする方法が必要です。 そしてここでも、最新のツールが重要です。 90年代には、開発者は高価なICEキットを使用する必要がありましたが、今日ではどのコンピューターでも実行できる強力なソフトウェアエミュレーターがあります。 独自のゲームをテストするには 、 BGBが最適です。 これは、強力なデバッガーを備えた非常に正確なGame Boyエミュレーターです。動作するゲームを作成するための必須ツールです。









ゲーム自体は右上隅にあり、他のすべての恐ろしいウィンドウは異なるデバッグツールです！



ただし、ゲームのパフォーマンスを確保するには、実際の機器でテストする必要があるという事実に備えてください。 90年代、人々は自分のプログラムをEPROMチップに記録し、特別なカートリッジを使用してこのチップを実際のゲームボーイに挿入しました。 このプロセスは効果的ですが、かなり時間がかかり高価です。 今日、いわゆる「フラッシュカート」があります。これは、ゲームボーイで実行されるROMイメージのSDカードを挿入できるカートリッジです。 考え方は似ていますが、新しいツールはより便利で使いやすいです。 Game BoyにはさまざまなFlashcartsがありますが、私の意見ではKrikzz： Everdrive Game Boyが最高のものを作成します。 今年の夏、より新しく改良されたモデルがリリースされましたが、以前購入した古いバージョンを使用しました。

















90年代および2010年代のFlashcart



2017年にはこれらのすばらしいツールがすべて存在していましたが、1989年のゲームコンソール用のゲームを作成することは依然として課題です。 特に、Unity、Unreal Engine、Godotなどの「最新のツール」の使用に慣れている人向けです。 Sheep It Upの開発で遭遇した主な困難についてお話します。 、ゲームボーイがどのように機能したかに驚いたことなど。



技術的な問題



サイズ制限



明白なことから始めましょう： Sheep It Up！ -これは非常にシンプルなゲームです。 これは、1つの理由によるものです。ゲーム全体の重さがわずか32KBです。 つまり、すべてのコード、画像、さらにはサウンドが小さな32KBのスペースに収まります。 比較のために、32KBは非常に小さな解像度でのWikipediaロゴのサイズであると言えます。









フルゲーム：32KB









160x146ピクセル（PNG-24）：32KB



もちろん、ゲームボーイのすべてのゲームが32KBに収まるわけではありません。 私にとって、これは実際のカートリッジでゲームをリリースするための技術的な制限でした（これについては第2部で詳しく説明します） 。 Game Boyの最高で最も有名なゲームは、実際にはもっと多くの重さがあります。



  • ポケモンレッド/ブルーは1024KB（巨大なゲーム！）
  • ワリオランドとゼルダリンクの目覚め -512KB
  • 星のカービィ -256KB
  • ガーゴイルのクエスト -128KB


実際には、ゲームボーイのゲームはわずか32 KBしか使用していません。 それらは、主にコンソールライフの初期段階で作成されました。 たとえば、 Alleyway （日本でGame Boyと共にリリースされたゲームの1つ）とTetris （米国およびヨーロッパでコンソールとともにリリースされた ）の両方が32KBに収まります。 これらのゲームはどちらも素晴らしいですが、カートリッジのサイズが小さいため、スケールが非常に「制限」されています。1つの画面のみ、少量のグラフィックおよびサウンドリソースなどです。















AlleywayとTetrisは32Kゲームで、Sheep It Up！



プロセッサー：ポータブル電源でプレイ！



ゲームボーイには、コンソール用に特別に設計された4MHzプロセッサが搭載されています（これはZilog Z80とIntel 8080プロセッサの組み合わせです） 。 全体的に、Gameboyの計算能力はNESに匹敵し、画面サイズが小さく色数が多いため、わずかに高くなっています（これについては後ほど説明します） 。 4 MHzの低速にもかかわらず、Game Boyのすべてのゲームは安定した60 fpsを提供しました 。 PS4 ProとXbox One Xを学びましょう！



しかし、大学を卒業したばかりのプログラマーにとって、最大の制限は8ビットプロセッサでした。 おそらくご存知のように、すべてのコンピューター内でゼロと1を使用してデータを処理します。 0または1の値を持つことができる1桁は、「ビット」と呼ばれます（2桁の値しか持てないため、「2進数字」、「2進値」の略） 。 1つの操作で8ビットのデータを処理できる場合、プロセッサは「8ビット」と呼ばれます。



これはビデオゲームの作成にどのように影響しますか？



プロセッサが処理できるビットが少ないほど、ゲームプレイ変数が制限されます。 たとえば、ゲームでは、エネルギー、速度、ポイントなど、さまざまな値を表す多数の整数が使用されます。



8データビットでは、整数は16ビット変数よりも低い値を格納できます。



  • 8ビット整数変数： -127〜128 （または、文字を使用しない0〜255）
  • 16ビット整数変数： -32768〜32767 （または符号が使用されていない場合は0〜65535）


それでも理解できない場合は、 Sheet It Upから具体例を見てみましょう。









図からわかるように、points変数には5桁が含まれており、0から99999まで変化する可能性があります。残念ながら、これは8ビット整数に格納できるものをはるかに超えています。 実際、16ビット変数であっても、このような「大きな」アカウントを保存するには不十分です。



したがって、Sheep It Upのポイント数を追跡​​するために！ 1つではなく、5つの異なる8ビット整数変数を使用する必要がありました。 特に、記録結果を保存する機能を追加したことを考えると、これらすべての変数を処理することが「楽しい」ことを想像してください。 5つの異なる変数に格納された2つの値を比較する必要がありました。 8ビットシステムでプログラミングを行うと、常にこのような複雑なソリューションを探すことができますが、現代のシステムでは32ビットまたは64ビットの変数を使用するだけで、プログラマにこのような問題を引き起こすことはありません。



（経験豊富なゲーム開発者向けのメモ：int8全体として0から9の値を持つ単一ビットを保存するよりも効率的なアプローチを見つけることが可能であることを知っています。しかし、実際には、0から255なぜなら、私はビットカテゴリ自体を保存するのではなく、この番号を画面に表示するために使用されるタイル識別子を保存するからです。



グラフィックスの課題



1つの世界、2つの飛行機



ゲームボーイのすべての画像は、背景レイヤー（BKG）とスプライト（OBJ）と呼ばれるいくつかの移動オブジェクトの2つの要素で構成されています。 画面上に存在できるスプライトは40個までです。 さらに、別の制限があります。ゲームボーイは1行に10個を超えるスプライトを表示できません。



バックグラウンド（BKG）とスプライト（OBJ）






実際、背景とは独立してスクロールできる「ウィンドウ」のレイヤーが追加されているため、すべてが少し複雑になります。 通常、ユーザーインターフェイス（メガネなど）に使用されます。 ただし、このレイヤーは不透明です。その背後にある背景レイヤーのすべてのグラフィックデータを非表示にします。 したがって、簡単にするために、ゲームボーイには「背景」レイヤーが1つあると想定できますが、必要に応じて、その一部を独立してスクロールできます。



ゲームボーイは4色を見ています！



最後に、最も明白なものに慣れてみましょう。元のゲームボーイモデルは4色しか表示できませんでした。









ライトグレー（ライトグレー）とダークグレー（ダークグレー）は実際にはライトグリーン（ライトグリーン）とダークグリーン（ダークグリーン）の可能性が高いと主張する人もいるかもしれませんが、それは問題ではありません。グラフィック、いずれの場合でも、使用できるのは4色のみです。 少なくともこれは背景に関連しています。なぜなら、スプライトの場合、状況は異なるからです！









そして実際、写真に見られるように、羊は3色だけで塗られています。 なぜそうですか？ 実際、私はスプライトを4色で塗りました。 しかし、ゲームボーイは「透明な色」として使用されるため、4番目の色（ここでは黒）は表示されません。 このため、すべてのスプライトが正方形の画像になるわけではなく、それらの背後にある背景を見ることができるさまざまな形状を持つことができます。









これは、スプライトの描画に黒を使用できないことを意味しますか？



もちろんそうではありませんが、以下に見られるように、ベルクロスプライトを例にしています。 このスプライトには、3 + 1色の別のパレットを使用しました。「白」は透明色として使用され、「黒」はスプライト上に表示されます。









パレットで楽しい



ゲームボーイは4色に制限されていますが、実際には3つの異なるパレットを使用して画像を表示します。









ご覧のとおり、背景レイヤー用に4色のパレットが1つと、スプライトで使用できる3色+透明色の2つのパレットがあります（各スプライトは1つまたは他のパレットを使用できます） 。 素晴らしいことは、パレットのどのスロットにも任意の色を自由に割り当てることができることです。 これにより、たとえば、多くのゲームで使用されている画面の明るさの段階的な減少や増加など、非常に興味深い効果を作成できます。









Game Boyにスクリーンシャドウを実装するには、3つのパレットの色を変更するだけです。 最初の段階では、パレットのすべてのスロットが白で塗りつぶされています。 次に、第2段階で、最も暗い色のスロット（黒）が明るい灰色で塗りつぶされ、徐々に画像が現像されます。 第3段階では、4つの色がすべて各パレットに表示されるまで、明るい灰色が暗い灰色になります。



同様の方法を使用して、各フレームのパレットの色を変更することで、文字を「ちらつき」させることができます。 たとえば、この方法は、スーパーマリオランドのマリオがアスタリスクをキャッチし、無敵になったときに使用されます。



タイルから世界を作りましょう！



Game Boyが画像を表示する方法には、もう1つの最後の奇妙な点があります。特に、現在の世代のエンジンを使用するゲーム開発者にとっては珍しいことです。 コンソールには「フレームバッファ」がないため、画面上の各ピクセルの色を個別に設定する必要があります。 すべての画面イメージは、多くの「タイル」から収集されます。 8x8ピクセル。 これは背景とスプライトの両方に適用されます：









グラフィックの表示に必要なビデオメモリの量を減らすように設計されたこの手法は、ゲームボーイ画面に任意の線を描画できないことを意味します。 しかし、これはゲームボーイが4 MHzと8 KBのビデオメモリ（メガバイトやギガバイトではなくキロバイト）のプロセッサでこのような美しいゲームを表示できるようにする秘密の1つでもあります。

ただし、このようなタイルディスプレイの欠点の1つは、羊とベルクロのスプライトが複数のスプライトで構成されていることです。 ヒツジのサイズは16x16ピクセルです。したがって、理論的には、表示するには4つの8x8スプライトが必要です。 幸いなことに、Game Boyには8x16スプライト表示モードもあります。つまり、Sheep It Up！のすべてのオブジェクトを意味します。 4つではなく、2つのスプライトのみで構成されます。









画面ごとに最大40のスプライトに制限されていることを覚えていますか？ さらに、ゲームボーイは1行に10個を超えるスプライトを表示できません 。



私たちの場合、これは、最初に考えたように、1列に5つまでの羊を配置できることを意味します（ハードウェアの観点から、羊は2つのスプライトで構成されているため） 。



全画面タイル



これについてはこれまで言及していませんが、ゲームボーイの画面解像度は160x144ピクセルです。 つまり、画面全体を閉じるには、20x18 = 360タイルが必要です。









残念ながら、ゲームボーイのビデオメモリは制限されており（8KB） 、合計256の異なるスプライトを保存できます。 これは、ソフトウェアのトリックがなければ、全画面画像を表示することが不可能であることを意味します。









秘密はタイルを再利用することです。ご覧のとおり、タイルイメージには多くの「空のスペース」があります。つまり、RAMからの1つの「白いタイル」がイメージ上に数回配置されます。 スクリーンセーバーでは178種類のタイルのみが使用されています。つまり、必要に応じて詳細を追加できます。









90年代に使用されたタイルの再利用方法はわかりませんが、今日ではGame Boy Tile Data Generatorと呼ばれる非常に便利なツールがあります。 彼にPNG画像（4色）を渡すと、実際のゲームボーイに表示する必要があるタイルとタイルマップが自動的に生成されます。 明らかに、同じタイルを自動的に認識して再利用し、できるだけ多くのコンソールビデオメモリを節約します！









参考：80年代と90年代のほとんどのコンソールは同様に機能し、8x8タイルに基づく画面を備えていました： Nes、Master System、PC-Engine、Super Nintendo、Genesis、Game Boy、Game Gear ...フレームバッファーはPlayStationとSaturnであり、このテクノロジーを備えた最初のポータブルコンソールはGame Boy Advance でした（Atari Lynxはシングルピクセルコントロールを備えた最初のポータブルコンソールでした） 。 つまり、ゲームボーイの「タイルの操作」をマスターしたので、将来、他のレトロコンソールでスキルを使用できます。
