Pandora、Hulu、およびその他のリソースにアクセスするためのAsus RT-N56Uルーターの構成

はじめに



ご存知のように、そのようなリソースにアクセスするには、アメリカのIPアドレスが必要です。 他のすべての「ラッキーなもの」は、VPN接続、ブラウザープラグイン（ FirefoxやChromeのMedia Hintなど）を使用するなど、さまざまなトリックに頼らなければなりませんでした。 このプラグインは本当にPandora.comでラジオを聴く機会を与えてくれましたが、Hulu.comを見ることができませんでした。

この出版物をハブで見たので 、Padavan 3.0.3.0-02の代替ファームウェアを使用してAsus RT-N56Uルーターのソリューションを確認することにしました。



設定プロセス



ファームウェア3.0.3.0-026にはDNSフォワーダーDNSMasqが含まれており、特定のWebリソースにアクセスすると、DNSMasq構成ファイルで指定されたDNSサーバーに代わってこのリソースにリクエストを送信します。 実際には、アクセスするリソースの名前を登録し、tunlr.net WebサイトのDNSサーバー（184.82.222.5および199.167.30.144）を介してアクセスする必要があることを示すだけです。



Asus RT-N56Uルーターの場合、sshクライアント（PuTTY）を介して接続し、次の行を/etc/storage/dnsmasq.confファイルに追加します（/etc/dnsmasq.confに別のdnsmasq.confがあることに注意してください） 、私たちはそれに触れず、/ etc / storage / dnsmasq.confだけが必要です！）：



サーバー= / pandora.com / 184.82.222.5

サーバー= / hulu.com / huluim.com / secure.hulu.com / 184.82.222.5



ここで、Huluではhulu.comのみを登録するだけでは不十分であるという事実に注意を喚起したいのですが、スラッシュで区切られたhuluim.comとsecure.hulu.comを追加する必要があることを実験的に見つけることができました（上記の例を参照）。 dnsmasq構成に変更を加えた後、ルーターを再起動します。



このサービスは、機能するサイトの特定のリストのみをサポートします 。 以下は、機能するはずのその他のリソース（dnsmasq.confの既成形式）のリストです（すべてをチェックしませんでした）。



サーバー= / pandora.com / 184.82.222.5

サーバー= / hulu.com / huluim.com / secure.hulu.com / 184.82.222.5

サーバー= / fox.com / 184.82.222.5

サーバー= / cbs.com / 184.82.222.5

サーバー= / netflix.com / 184.82.222.5

サーバー= / abc.com / go.com / 184.82.222.5

サーバー= / tv.com / 184.82.222.5

サーバー= / pbs.com / 184.82.222.5

サーバー= / vevo.com / fwmrm.net / 184.82.222.5

サーバー= / thewb.com / 184.82.222.5

サーバー= / cwtv.com / 184.82.222.5

サーバー= / last.fm / 184.82.222.5

サーバー= / turntable.fm / 184.82.222.5

サーバー= / mog.com / 184.82.222.5

サーバー= / iheart.com / 184.82.222.5



これで、必要なリソースにアクセスできるようになりました。サービスを使用したいときや、ルーターに接続されたデバイスから追加の操作を行う必要はありません。
