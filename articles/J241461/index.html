A / Bテストよりも優れたもの

みなさんこんにちは！ 私の名前はヴァシリーです。 私の主な仕事は、あらゆるITビジネスの重要な指標であるWebサイトのコンバージョンを増やすことです。



統計によると、RuNetのサイトの平均コンバージョン率は1％を超えていないため、膨大な数のインターネット起業家が毎日多くのお金を失っています。 これらの起業家は、精神的に2つの主なタイプに分類できます。1つ目は、最適化されていないリソースをただ座って見て、何もしません。 残念ながら、彼らは技術革新と競合他社によって時間によって押しつぶされる運命にあります。 2番目は、それについて何かをしようとしている人です。 原則として、彼らはランディングページを描き、広告チャネルを試し、A / Bテストを実施します。 A / Bテストは現代のマーケティングに不可欠なツールであり、ほとんどのユーザーにとって最も効果的な販売ページのバージョンを判断できます。 この方法がどれほど効果的かを理解するために、 オバマと6000万ドルについてのセンセーショナルな話を思い出してください。 ただし、この方法には裏返しがあります。すべての人が異なるということです。たとえば、女性が猫を愛していることが統計で示されている場合は、ガールフレンドが猫にアレルギーを持っている必要はありません。 つまり、A / Bテストで特定の着陸オプションがユーザーの60％に理想的であり、広告会社にそれを選択した場合、残りの40％は自動的に失われます。 そのため、A / Bテストは完璧ではなく、緊急に何かをする必要があります。 そして今、地獄で地獄に行くと、リードの40％を失い、これはまったくクールではないからです！







顧客を失わないためには、A / Bテストを実施して結果を表示するだけでなく、訪問者が好むランディングページのバージョンを予測し、この特定のオプションを表示できるシステムが必要です。 残念ながら、これは不可能です。なぜなら、潜在的なクライアントが何を念頭に置いているのか、最後まで誰も知らないからですよね？ かどうか？



もちろん、特定の数の訪問者は、上司が燃えているため、またあなたの祖母があなたのサイトを訪問した場合、おそらくあなたのために書かれた販売テキスト息子。 すぐに膨大な数の試合が私の頭に入ります。 しかし、時間、地理位置情報、ブラウザ、オペレーティングシステム、広告会社のUTMタグなど、さまざまなパラメーターに対してサイト上のさまざまなユーザーの行動の類似性を見つけるアルゴリズムを作成すると、これらの対応を自動的に見つけるとどうなりますか？ 彼はいくつの試合を見つけますか？







このことを行って興味深い結果を得たとすぐに言います。約80％の確率で、アルゴリズムは特定のユーザーがどのバージョンの着陸を好むかを予測できることがわかりました。 はい、ユーザーが朝食に食べたデータはありません。また、彼の犬または妻の名前がわかりません。 ただし、利用可能なデータでさえ、一致を自動的に検索するにはすでに十分でした。 アルゴリズムの動作を改善するために、いわゆる重みシステムをそれに追加しました。 着陸オプションがユーザーにとって快適でなく、ターゲットアクションを実行しない場合、ゆっくりと回転して沈み、その結果、ユーザーにはまったく表示されないか、または好きなグループにのみ表示されます。 繰り返しますが、小さな問題があります。それは、通信の識別とシステムの通常の動作のために、いくらかの時間とそれに対応する出席が必要であるという事実にあります。 ローテーションの着陸オプションが増えるほど、より多くの訪問者が必要になります。 残念なことに、1日200人の出席者と5つの着陸オプションがある場合、アルゴリズムがユーザーをセグメント化するまで待つよりも気が狂う可能性が高くなります。 ただし、2000 +で平等に出席している場合、彼は1日半で学習します。 そして、utmタグがインストールされている広告からのトラフィックがある場合、アルゴリズムが魔法を行います。 確かに、魔法の魔神でさえ願いをかなわないことを忘れないでください。ランプをこすらないと、より簡単に言えば、フッターのユニコーンの数だけが互いに異なる不気味で使用できないページの5つのバリアントを作成した場合、アルゴリズムがあなたを助けることを期待しないでください。



一般的に、このことは興味深いことが判明し、これはマーケティングテクノロジーの未来のように思えます。
