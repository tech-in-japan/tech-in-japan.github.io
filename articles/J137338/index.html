habraのトピック「州下院が議員の収入を新しい形で提示した」を読んで、 議員の収入に関するデータは視覚化する価値があると判断しました。 処理言語のトピック「データの可視化」に慣れると、ツールとして行動したのは彼でした。 以下は最初の写真の一部です。Treemappingとは何か、Processingでどのように調理するかを知ることができます。







私にとってデータの視覚化に関する最高の本の1つは、Ben Fry のVisualizing Dataです。これは、Processingのクリエイターの1人による小さな本です。 この本は、1石で2羽の鳥を殺します。情報の視覚化の原理を詳しく説明し、それに処理言語を使用する方法を説明しています。 Habrでの処理については、 こちらとこちらですでに書いています 。 バージョン2.0a4が利用可能になりました（現時点ではアルファ版ですが）。 データの視覚化の最初の章で、Ben Fryはデータの視覚化の7つの段階について説明します：データの取得、データの解析、過剰なデータのフィルタリング、データの処理（たとえば、最小値と最大値の検索）、データ表現の選択、この表現の改善、最後の項目はインタラクティブな追加です。 彼はまた、このプロセスは必ずしも連続的ではなく、しばしば前の段階に戻らなければならないことを指摘しています。

Treemapは、 Ben Shneidermanによる情報視覚化の第一人者の発明です。 1990年、Ben Schneidermanは、どのファイルがハードディスクのほとんどのスペースを占めるのか疑問に思いました（彼が働いていた研究室では、14人のユーザーが80Mバイトのディスクを使用していました）。 ベンは、フォルダーとファイルのツリー構造のコンパクトな視覚化について考え始めました。そこで、彼は後でTreemapと呼ばれるチャートを思いつきました。 Treemappingの歴史について詳しくは、 こちらをご覧ください 。記事の最後に、このタイプの視覚化の使用例が数多くあります。 そして、データの視覚化の本では、章全体がツリーマップに当てられています。 Processingディストリビューションにはこの本の例が含まれており、Treemap専用の例が2つあります。 どちらの例もTreemapライブラリを使用しています-これは 、Processingで動作するように修正されたJavaライブラリのバージョンです。 このライブラリには、スペースを長方形に分割するためのいくつかのアルゴリズムがあります：スライスとダイス、中間ごとのピボット、サイズごとのピボット、サイズごとのピボット、分割レイアウト。 ここで異なるアルゴリズムの動作を比較できます （Javaプラグインが必要です）。 「正方形のレイアウト」が提供する画像は、私の意見では、最も楽しいものです。 このアルゴリズムは、長方形ができるだけ正方形に近くなるように長方形を作成しようとします。 Squarifiedレイアウトアルゴリズムについては、Jack van Wijkの記事で説明されています。 ライブラリを使い始めるには、ライブラリの3つのクラスSimpleMapItem、MapModel、およびTreemapを理解する必要があります。 SimpleMapItemクラスは単一のセルを表し、MapModelは配列内のセルのリストを格納し、Treemapクラスは視覚化自体を構築し、コンストラクターでMapModel型のオブジェクトとチャートの境界の座標を取得します。 MapLayoutオブジェクトは、杭打ちアルゴリズムの実装を定義するTreemapオブジェクトにも保存されます。 デフォルトでは、Treemapはピボットサイズごとのピボットアルゴリズムを使用しますが、関数setLayout（MapLayoutアルゴリズム）を使用して別のアルゴリズムを指定することを妨げるものはありません。 したがって、一般に、ProcessingでTreemap視覚化を取得するには、SimpleMapItemおよびMapModeインターフェイスをそれぞれ実装するDeputatItemおよびDeputatMapクラスを作成します。 SimpleMapItemには、ツリーマップ内の四角形のサイズを決定するsizeパラメーターがあり、代理人が獲得した金額を割り当てます。



データを読み取り、SimpleMapItemインターフェイスを実装するDeputatItemオブジェクトに保存します。 代理人に関する情報を含む作成されたオブジェクトがDeputatMapオブジェクトに追加されます（MapModelインターフェースの実装）。 



DeputatMap dMap = new DeputatMap(); String[] lines = loadStrings("parliamentV2010.csv"); for (int i = 0; i < lines.length; i++) { //...  id, party, name, money  lines[i] DeputatItem d = new DeputatItem(id, party, name, money); dMap.addDeputat(d); } dMap.finishAdd();
