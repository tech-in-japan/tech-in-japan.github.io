Windows Server 2019：LinuxおよびKubernetesのサポート

Microsoftは3月20日に、ハイブリッドワークロード、Linux、およびハイパーコンバージドインフラストラクチャのサポートを強化したWindows Serverの次の公式リリースを今年リリースすると発表しました。



Windows Server 2019は、2016年10月に正式にリリースされたWindows Server 2016に代わるものです。 マイクロソフトがWindows Serverの更新プログラムに昨年1年に2回切り替えたことを考慮すると、これらの更新プログラムは2〜3年に1回、長期サービスチャネルに表示されます。 このサービスモデルは、システムをあまり頻繁にアップグレードしない管理者向けです。







6か月のチャネルにまだ切り替えていない企業は、Windows ServerのLinuxおよびKubernetesサポートテクノロジをテストできます。これらは現在ベータ版です。



Windows Server 2019は、Azure Backup、Azure File Sync、Disaster RecoveryなどのMicrosoftクラウドサービスとのオンプレミスコラボレーションを合理化します。 このアプローチは、企業のクラウドシステムの境界ノードの役割を果たし、管理者がプライベート環境とパブリック環境の間でワークロードを転送できるようにする、Microsoftのオンプレミスハードウェアの広範なビューの一部です。



特に、Microsoftは、Server Coreコンテナーイメージのサイズを約1.7 GBに削減する予定です。これは、このイメージの現在のサイズである5 GBよりも大幅に小さくなります。 。 イメージのサイズを小さくすると、ワークロードの効率の最適化が簡素化され、より柔軟なイメージの欠如に不満を抱いたユーザーへの第一歩となります。



さらに、Microsoftは、この新しいリリースにより、Windows Server環境で一般的なコンテナーオーケストレーション用のオープンソースソフトウェアであるKubernetesとの連携に関連する機能が改善されると述べました。 現在、Windows Serverでのこのテクノロジーのサポートはベータ版であり、MicrosoftはこのOS上で実行されるクラスターの機能を大幅に改善する予定です。

さらに、このアップデートにより、シールドされた仮想マシンのサポートなど、Linuxによって解決されるタスクの実行にいくつかの改善がもたらされると予想されます。 これらの仮想マシンは、悪意のある管理者またはアカウントがハッキングされた管理者から保護されるように設計された環境を顧客に提供します。 さらに、Microsoftは、ユーザーがLinuxスクリプトをWindows Server 2019に移植できるよう支援しています。これにより、OpenSSH、Curl、およびTarのサポートが改善されます。

Linuxセキュリティアップデートに加えて、MicrosoftはWindows Defender Advanced Threat ProtectionテクノロジーをWindows Server 2019に統合しています。 これにより、ユーザーが大規模な問題になる前にセキュリティホールを検出して修正することができます。



Windows Server 2019を試してみたい場合は、Windows Insider Programを使用して今すぐ試すことができます。 長期サービスチャネルでのこのOSのリリースは、半年ごとのリリースチャネルでの関連アップデートのリリースとともに、今年後半に予定されています。 マイクロソフトは、今後数か月以内にこれらすべての更新プログラムの詳細を明らかにする予定です。
