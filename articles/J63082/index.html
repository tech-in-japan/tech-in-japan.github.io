管理者の喜び。 リモート端末の作成

私は、グラフィカルリモートターミナルを整理する最も簡単な方法として、あなたの注意を喚起したいと思います。できるだけシンプルにしようと思います。



そして、 NX NoMachineを紹介します。

NXを使用すると、Windows、Linux、Mac OS X、Solarisなどの一般的なすべてのプラットフォーム用のクライアントと、LinuxおよびSolaris用のサーバーを使用して、リモートX11セッションを実行できます（WindowsおよびMac OSプラットフォームのサーバーサポートも開発中です）。 この情報に出くわしたとき、私はすぐに、なぜWindows NXサーバーを作るのか、RDPプロトコルも持っているのかと考えましたが、最初のインストール後、NXは問題なくネットワーク接続が遅くても問題なく動作することを確信しました。喜ぶことはできません。 さらに、NXはリモートRDPおよびVNCサーバーに接続することもできます。これは同意するのに便利であり、多数のクライアントを使用する必要がなくなります。





図1-リモートKDEセッション、およびWindows Vista上のクライアント。



NXは有料と無料の両方で提供されます（無料版では、NXサーバーへの接続数が2に制限されます）。 公式ウェブサイトのドキュメントセクションですべてが非常に明確に説明されているため、この製品の設定については特に説明しません。おそらく、あなたの愛するDebianのセットアッププロセスを除きます。



まず、システムにはXサーバーとシェルが必要です。私はgnomeとkde4をテストし、結果に完全に満足しました。



次の段階では、 ダウンロードセクションの会社のページに移動し、管理者の目にとって便利で楽しいものをたくさん見ます。 しかし、今日、私たちは特に無料サーバーであるNX Free EditionとNXクライアント製品に興味を持っています。これがクライアントの部分です。



サーバーから始めましょう。 すべてのDebianベースのディストリビューションでは、Linux DEB用のNX Free Editionを選択し、注意して適切なアーキテクチャを選択してください。



そして、サーバー部分：素晴らしい製品を準備するために、3つの成分nxclient、nxnode、nxserverを使用し、料理が素晴らしいものになることを願っています。 そして今、いくつかのコマンド：

  • wget 64.34.161.181/download/3.3.0/Linux/nxclient_3.3.0-6_i386.deb && dpkg -i nxclient_3.3.0-6_i386.deb
  • wget 64.34.161.181/download/3.3.0/Linux/nxnode_3.3.0-17_i386.deb && dpkg -i nxnode_3.3.0-17_i386.deb
  • wget 64.34.161.181/download/3.3.0/Linux/FE/nxserver_3.3.0-22_i386.deb && dpkg -i nxserver_3.3.0-22_i386.deb


注意、厳密な順序でインストールする必要があります。 成功のための最小要件

  • 400 MHzプロセッサ
  • 128 MBのRAM
  • 35 MBの空きディスク容量
  • ネットワーク（ローカルエリアネットワーク、または33.6 Kbps以上のインターネットチャネル）


すべてを正しく実行した場合、サーバーは既に実行されているはずです。これは、コマンド/ usr / NX / bin / nxserver --statusで簡単に確認できます。正しい答えは次のとおりです。

NX> 900サーバーへの接続...

NX> 110 NXサーバーが実行されています。

NX> 999バイ。


すべてが正しい場合は、クライアントセクションに進み、[ ダウンロード]セクションで目的のプラットフォームを選択してインストールします。

  • wget 64.34.161.181/download/3.3.0/Linux/nxclient_3.3.0-6_i386.deb && sudo dpkg -i nxclient_3.3.0-6_i386.deb


接続ウィザードウィンドウが起動し、表示されます。

www.nomachine.com/screenshot/linux-client.php-Linuxクライアントを接続するスクリーンショットのセット、

www.nomachine.com/screenshot/windows-client.php-Windowsでも同様に見えます。

いくつかの簡単な手順の後、ログインして接続を取得します。 ローカルエリアネットワークで作業している場合、または情報が機密でない場合、暗号化を無効にすることができます。
