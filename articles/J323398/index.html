2月のHP 3PAR StoreServ更新がラインの発売以来最も重要である理由

2017年2月13日からのHPE 3PAR StoreServの新しい機能の発表は、企業ブログの一部の解説者によって、発売以来最も重要であると命名されました

4年前のファミリー3PAR 7000。 これは非常に大きな声明です。 この間、業界では多くの変化があり、フラッシュドライブは販売面でハードドライブを上回り、一部のストレージ企業は独立性を失い、 破産しました 。



興味深いことに、2月13日に、概してプログラムによる改善のみが発表されました。 唯一のハードウェアノベルティは、3PAR 8000ファミリの4ポートアダプターですこれは、File PersonaまたはiSCSIを使用しているが、「最も重要な発表」のタイトルに値しない人にとって重要なことです。 それで、取引は何ですか？







2016年8月に戻って、オールフラッシュアレイのGartner マジッククアドラントを確認します。 HPEは3PARラインのリーダーの1つであるという事実にもかかわらず、多くの人が会社の位置が1位からは程遠いことに気付きました（ただし、売上高から判断すると、HPE 3PARはこのセグメントで1位のアレイです）。 同時に、ガートナーのアナリストは、アレイの長所-パフォーマンス、スケーラビリティ、一連のデータ管理ツール、NANDメモリとの知的作業に注目しました。 レポートは、HPE 3PARがオールフラッシュシステムの象限と従来のアレイの象限の両方に存在する唯一のプラットフォームであるという事実さえ評価しました（ 「既存の顧客とチャネルパートナーにアピールするプラットフォーム全体の親しみやすさ」 ）。 これは、アナリストが会社を最高の地位に置くのに十分ではありませんでした。 警告リストには次のものがあります。



  • 圧縮不足
  • 洗練されたライセンススキーム
  • 競合他社と比較した場合の管理ソフトウェアの不十分な分析機能。


これらのポイントがどれほど重要かは議論の余地があります。 市場の一部のソリューションでの圧縮の実装は、このテクノロジーの大きなメリットを納得のいく形で証明していません（IBMとEMCソリューションとの競合で圧縮なしの3PARが最高のストレージコストを示した昨年のコンテストの結果を参照）。 ライセンス方式は、調達段階での先行販売の懸念事項であり、アレイの効率には影響しません。 分析機能については、クラウド内にある必要がありますか？ これを行うには、システムが統計をクラウドに送信する必要があります。 これは、多くのロシアの顧客にとって悪い考えのようです。 さらに、アナリストがHPE StoreFront Remoteなどの分析ツールの可用性を考慮していなかった可能性もありますが、これについては後で説明します。



それでも、これらの「問題」はすべて2月13日の発表で修正されています。



HPE 3PARアダプティブデータ削減



HPE 3PAR辞書の新しい用語。 これは単なる圧縮ではなく、効率的なデータストレージのための一連の手段です。 3PAR ASICチップがファイバーチャネルストリームを分析し、ゼロを記録しない場合、すべてゼロ検出で始まります。 その後、データは重複排除されます。 3PARオペレーティングシステムの新しいバージョンでは、このプロセスのアルゴリズムが再設計されました。 実際の操作の結果として判明したように、一部のデータは完全に重複排除されます-最大12：1で、残りはまったく重複排除されません。 アレイのパフォーマンスを最適化するために、コードの一部が書き直されました。開発者によると、パフォーマンスの大幅な改善が期待できます。 重複排除がサイズが16 KBの重複ブロックを識別して処理した後、アレイはこれらの一意のブロック内にあるデータを圧縮できるかどうかを判断しようとします。 たとえば、ビデオファイルの圧縮にCPU時間を費やすことは意味がなく、3PARはこれを試行しません。 データが圧縮されている場合、有用な情報は0〜16 KBになります。 SSDに書き込む前に適切にパックして、圧縮効率を失わないようにし、コンピューターがバックグラウンドプロセスに非常に熱心で、何も役に立たない場合にWindows 10ユーザーに馴染みのある状況を回避することが重要です。 競合するアレイでは、定期的な「ガベージコレクション」が必要です。上書きによってデータ圧縮率が変更され、論理ボリュームが削除される可能性があるため、同時に空き領域を使用できません。 HPE 3PAR Data Packingの開発者は、アルゴリズムが効率的であり、プロセッサーに過負荷をかけないと主張しています。



3PARオールインクルーシブライセンス



すべての3PARアレイには、HPE 3PARオペレーティングシステムソフトウェア、バーチャルコピー、動的最適化、適応最適化、優先度最適化、仮想ドメイン、仮想ロック、File Persona、Smart SAN、オンラインインポート、ホストエクスプローラー、システムレポーター、Recovery Manager Central。 2つ以上のアレイの相互作用を整理する場合、HPE 3PAR Peer Motion、HPE 3PAR Remote Copy、HPE 3PAR Peer Persistence、HPE 3PAR Cluster Extensionを含むHPE 3PAR All Inclusive Multi-System Softwareが必要になります。 競合他社が提供するほとんどのオールフラッシュアレイのレプリケーション状況があいまいであることを覚えていますか？ 3PARアレイは、相互作用するために仮想化を必要としませんが、フェデレーテッドストレージまたはフォールトトレラントソリューションの構築にはライセンスが必要です（試行するには、一時的なものを取得するだけです）。



新しいライセンススキームは、ストレージシステムの購入と展開の計画を簡素化し、また、ユーザーが第1世代「1つのアプリケーション-1つのストレージ」のフラッシュアレイを使用する従来のモデルから移行することを奨励します。 3PARはデータ統合において非常に効率的であり、4：1レベルでのHP 3PAR Adaptive Data Reductionの宣言された効率を考慮すると、ストレージコストを大幅に向上させることができます。



他の広告について：



  • 3PAR SSDのフラッシュ摩耗に対する7年間の保証 。 ご存知のように、HPEは3PARにインストールされたすべてのSSDに対して無条件の5年保証を提供します。 この保証には、NAND摩耗も含まれます。 実際、アダプティブスペアリングメカニズムとディスク領域の低レベル仮想化により、3PARのフラッシュドライブの摩耗が最小限に抑えられます。 これにより、最も負荷の高いワークロードで現在利用可能な最も容量の大きいSSDを使用する（ストレージのコストを削減する）だけでなく、「自然な」SSD障害を保証ケースとして考慮することもできます。 7年間の保証延長は、2015年から、以前に販売された3PAR 8000および20,000アレイに延長されます。 有効なサポート契約がある場合、新しい条件は既存のストレージシステムに自動的に適用されます。



  • 3Dキャッシュ。 ストレージクラスメモリとNVMeの使用に関連するIT革命が近づいている中で、HPE 3PARアーキテクチャは他のどのタイプよりも優れており、新しいタイプのメモリを使用し、すでに述べた現在の世代の新しいテクノロジーをサポートしていることを強調することが重要です。



  • 3DCピアの永続性。 3つのデータセンターからのフォールトトレラントソリューションを含む、さまざまなレプリケーションシナリオのサポート-これは、長年にわたってHPE XPアレイで一般的なものでした。 現在、3PARに基づいたアナログが利用可能です。 Peer Persistenceテクノロジーを使用すると、さまざまなサイトでVMware vSphere、Microsoft Hyper-V、SQL、Oracleのクラスターを構築できます。 障害が発生した場合、切り替えは完全に自動化され、アプリケーションに対して透過的です。 3PAR開発者は、地理的に離れた場所に配置して最高レベルのフォールトトレランスを提供できる3つ目のデータセンターのサポートを追加しました。 同様のEMCソリューションのコストを想像してください。 XtremIOに加えて、リカバリポイント、VPLEX、および追加のInfiniBandスイッチが必要になります。 3PARの場合、これはHPE 3PAR All Inclusive Multi-System Softwareライセンスのみです。



  • HPE Peer Copy 。 4PARを単一のストレージプールに結合し、必要に応じてビジネスアプリケーションを中断したり、負荷を分散したり、空きスペースを効果的に使用したりすることなくボリュームを移動できる場合、3PARベースのフェデレーションストレージについて説明する前に 、このテクノロジーについて言及しました。 ソフトウェア定義のHPE StoreVirtualと3PARの間でほぼ同じことを実行できるようになりました。 このためには、RMCバージョン4.0がまだ必要です（これはHPE StoreOnce仮想ライブラリへの直接バックアップを担当する3PAR基本ソフトウェアパッケージの一部です）が、近い将来レプリケーションが登場することを期待しています。 可能なユースケースは、HPE StoreVirtualまたはリモートブランチのハイパーコンバージドシステムが、追加の機器なしで中央オフィスのHPE 3PARにデータを複製することです。 RMCライセンスは、HPE 3PAR All Inclusive Single System Base Packageの一部です。
