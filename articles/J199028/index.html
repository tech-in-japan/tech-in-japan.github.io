Webasystブログエンジンを確認する

Webasystフレームワークに基づいて開発した最初のアプリケーションの1つは、 ブログアプリケーションでした。これにより、個人、団体、および企業（承認により閉鎖）のブログを実施できます。 このアプリケーションの例では、フレームワークにフロントエンド、デザインエディター、プラグイン、ウェブサイトなどの多くの基本概念を実装するのが非常に便利だったため、ブログです。 さらに、それから私たち自身がサイトでブログを開かなければなりませんでした。 アプリケーションは無料（LGPL）で、PHP / MySQL / Smarty / jQueryに基づいて実行されます。







デモ ： フロントエンド 、 バックエンド （デモを入力するにはログイン名とパスワードを入力してください）

ダウンロード ： アーカイブ （3.8 MB。アーカイブには既にWebasystフレームワークが含まれています）またはGitHub （アプリケーション自体はwa-apps / blog /にあります）



猫の下で、アプリケーションの詳細なレビューと多くのスクリーンショット。



設置



システム要件は「クラシック」です。PHP5.2 + 、 MySQL 4.1+ 、拡張子mbstring、iconv、json、curl、gd / imagick-ほとんどすべての共有ホスティングにインストールされます。



このアプリケーションはWebasystフレームワークの一部です。 上記のリンクからアプリケーションをダウンロードすると、フレームワークは「ブログ」アプリケーションと共にすぐにインストールされます。 基本配信に「ブログ」がない場合は、フレームワークの基本構造に常に含まれている組み込みの「インストーラー」アプリケーションを使用して簡単にインストールできます（「インストーラー」を使用すると、更新プログラムと新しいアプリケーションをワンクリックでインストールできます）。



インストールはいくつかのステップで行われます：FTP経由でサーバーにファイルをコピーしてから、Webインターフェース経由でMySQLに接続するための設定を入力します。 異常なことは何もありません。 指示 （スクリプトをインストールしたくない場合は、Webasystクラウドでアプリケーションを試すことができます。）

バックエンド



最初の起動時に、アプリケーションはユーザーに会います。





バックエンドインターフェイスは、投稿のリストを中心に構築されます。







投稿の主な区分はブログです。 ブログは投稿の独立したストリームであり、 オープンにすることができます-投稿はサイト上で自動的に公開され、 クローズします -投稿はバックエンドユーザーのみが利用できます。 ブログへのアクセスは、ユーザーとユーザーグループによって構成されます。 各ブログには色を割り当てることができます。これにより、多数の独立したブログを非常に便利に管理できます。 上のスクリーンショットは、3つのブログの共通フィードの例を示しています。そのうち2つ（黄色と赤）は閉じられています。これらのブログに属する投稿のタイトルの横に小さなロックアイコンが表示されます。







投稿は1つのブログにのみ含めることができます。 さらに、各投稿にタグとカテゴリを割り当てることができます（このためには、無料のプラグインをインストールする必要があります）。 遅延読み込みリストのどこでも。



設計



デフォルトでは、フロントエンドのデフォルトテーマが接続されており、最小限に見えます。





「デフォルト」は、バックエンドのデザインエディターで直接他のデザインテーマに簡単に変更できます。デザインテーマを選択すると、Webasyst Webサイトからダウンロードされ、インストールされます。







基本的なトピックはほとんどありませんが、すべてのトピック-すっきりした有効なレイアウト（デモバージョンバリデーター ）で、ユーザーのデバイス（デスクトップ、タブレット、モバイル）に自動的に適応し、さらにすべての基本的なデザインテーマでSchema.org標準に基づく投稿とコメントのマイクロレイアウトが使用されます。



Smartyテンプレートエンジンが使用されます。 ローカライズはgettextに基づいています。 ショーケースデザインは、アプリケーションバックエンドのブラウザーで直接編集できます。 Webインターフェースを介して、HTML / Smarty、CSS、およびJavaScript（jQuery）のすべてのフロントエンドテンプレートを編集できます。







Blogyアプリケーションまたは他のアプリケーションからデータを受信するためのデザインテンプレートを支援するさまざまなSmartyヘルパーがあります。 たとえば、構造$wa->shop->products('tag/')



を使用すると、Shop-Script 5（オンラインストアを作成するためのアプリケーション）からタグ「bestseller」を持つ製品のリストを取得し、サイドバーに表示できます。ブログ（sidebar.htmlテンプレート）：







同様に、 他のアプリケーションからデータを引き出すことができます ： $wa->photos->photos('tag/')



、 $wa->shop->cart()->total()



$wa->photos->photos('tag/')



$wa->shop->cart()->total()

など サポートされているヘルパーのリストは、常にデザインエディターのチートシートにあります。







フロントエンド



フロントエンドブログの購読はRSSであり、バックエンドのバックログを電子メールで送信します。



フロントエンドレコードのCNCアドレスには、レコードアドレスのみ、または年、月、日と一緒のレコードアドレスのいずれかのオプションを使用できます。







1つのアプリケーションに基づいて、異なるサイトで複数の独立したブログを維持できます。 アプリケーションの各フロントエンド（および異なるドメインで好きなだけ多くの場合があります）に対して、個別のブログを添付できます。 例：フレームワークWebサイトでは、英語（webasyst .com / blog /）およびロシア語（webasyst .ru / blog /）言語で単一のストリームでブログを行っており、投稿は異なるドメインで公開され、外観は独立しています。







フロントエンドでも、遅延ロードはどこにでもあります。 SEOの観点からは、すべてが問題ありません。JavaScriptがサポートされていない場合、遅延読み込みの代わりにページナビゲーターが表示されるため、すべての投稿が正常にインデックス付けされます。



また、アプリケーションはブログのWebサイトマップ（デモのWebサイトマップの例 ）を自動的に作成および更新します。



執筆後



投稿を書くための3つの編集モードがあります。

-WYSIWYG、それなしの場合：elRTEエディターが使用され、

-HTML ：Aceエディターに基づいて、

-Markdown ：無料のプラグインをインストールする必要があります。



投稿の場合、メタタグ、タイトル、タグ、カテゴリ（プラグイン）、公開予定日、および任意のKey-Valueパラメーターを編集できます。これらのパラメーターの値は、デザインテンプレートに転送されます。







アプリケーション設定でオプション'can_use_smarty' => true



を有効にすると、投稿の本文でSmartyを完全に使用することもできます。 これは、タグ、レーティング、アルバムなどのルールに従って写真アプリケーションから写真の動的リストを取得する必要がある場合など、他のアプリケーションのデータを操作するのに便利です。



コメント



このアプリケーションは、ツリーのようなコメント構造を持っています。分岐とコメントへの応答機能は、外観がHabréのコメントに似ています。







すぐに使用できるフレームワークには、人気のあるサービス用のOAuth承認アダプターがあります。そのため、設定（Webインターフェイス）で、Google、Facebook、Vkontakteなどのサービスを介して承認を簡単に接続できます。



しつけ



バックエンドの出版物のカレンダーには、投稿が明確に表示されます。すでに公開されており、将来公開される予定です。 多くの投稿が計画されている場合とそうでない場合を簡単に確認できます。 私たち自身のブログの経験が示すように、カレンダー、公開予定日、およびリマインダーは、アプリケーションの最も実用的な機能の1つであり、より規律あるブログを維持するのに役立ちます。







今後の録画のリマインダーを有効にできます。アプリケーションは、録画が公開または削除されるまで、アプリケーションの作成者を毎日メールで「ノック」します。







遅延投稿もサポートされています。日付と時刻を請求すると、投稿は自動的に公開されます（このためには、cronスケジューラを構成する必要があります）。

WordPress、他のエンジン、なぜ別の自転車



Webasystフレームワークに基づいて、私たちは主にチームワークとビジネス用のアプリケーションを作成し、Webasystの今日の主な焦点はオンラインショッピングです。 ブログアプリケーションを作成してWordPressや他のブログエンジンと競合することはしませんでしたが、第一に、フレームワークの開発に技術的に多くの助けをしてくれたので、第二に、フレームワークに基づいてブログを作成できるようにしたいと思いました、したがって、タスクを設定して、ブログに焦点を絞った便利でバランスのとれたツールを作成します。 一般に、フレームワークはその機能を本格的なユニバーサルCMSに拡張しますが、それは他のアプリケーションとの組み合わせのみです。



「ブログ」アプリケーションは、ブログエンジンにうんざりしている人、ブログに加えてサイトのバックエンドに他の多くのツール（フォトギャラリー、オンラインストア、To Doリスト、ディレクトリなどの一般的ではないCMSツール）を持ちたい人に便利です。連絡先、メール、タスクなど Webasystフレームワークには多くのこのようなツールがあり、常に興味深いアプリケーションをリリースしています。



「ブログ」は「 写真 」アプリケーションと組み合わせて使用​​すると特に便利です。「写真」に写真をアップロードした後、一度に1つずつ、またはパック全体としてブログにエクスポートできます。 また、お客様の多くは「ブログ」を使用して、 Shop-Script 5に基づいてオンラインストアからニュースを公開しています。
