クラウドサービス監視サービスと管理者のアラート

この記事では、開発者と管理者向けのWindows Azureクラウドサービスのアラートとステータスアラートの新しいサービスであるWindows Azureアラートについて説明します 。







Windows Azureアラートアラートサービス



Windows Azureアラートは、Windows Azureに展開されたコンピューティングクラウドリソースの管理および監視タスク用に設計された新しいクラウドプラットフォームサービスです。



Windows Azureの開発者と管理者は、 Webサイト 、モバイルサービス、クラウドサービス、または仮想マシンのメトリックの監視ルールを簡単に設定し、ステータスが変化したときにアラートを受信できます。



Windows Azureアラートの機能







プレビューフェーズでは、Windows Azureアラートは、最大10個のメトリック監視ルールを構成する機能を提供します。



Windows Azureユーザーは、 Webサイト 、モバイルサービス、クラウドサービス、仮想マシンのさまざまなパフォーマンスインジケーターを監視できます。 例：CPU消費、ハードディスクI / O、着信および発信トラフィック値など。



構成されたルールがトリガーされると、つまり、パラメーターの1つが指定された制御値に達すると、Windows Azureアラートサービスはアカウント管理者、共同管理者、または別の任意のアドレスのメールアドレスに通知を送信できます。



特定のルールごとに、管理されたメトリックの変更を追跡できるダッシュボードを使用できます。



Windows Azureアラートを使用する







Windows Azure Alertsアラートサービスの使用を見てみましょう。 管理パネルの「設定」セクションに移動します。 [ルールの追加]ボタンを使用して、新しい監視ルールを追加できます。







新しいルールを作成するには、名前と説明を指定し、必要なサブスクリプションを選択し、サービスのタイプと名前、その他のパラメーターを指定します。







次の手順では、監視用のメトリックと、このメトリックに設定される条件を指定します。 たとえば、CPUを消費するには、それ以上でルールが機能する割合を指定します。



管理パネルでの電子メール通知および監視結果の表示の設定を指定します。







ルールを作成したら、サービスの監視結果に移動して、パラメーターの変更を追跡できます。 これで、制御値を超えると、Windows Azure Alertsサービスから電子メール通知が送信されます。
