N900上のAndroid



要件：

Android RootFS （ ユーザーmaemo.org dj_steve ）

鏡

カーネルイメージ （ maemo.org dj_steveによる ）

鏡

MicroSDカード（最小1Gb）

コンピューター上のLinux

マエモフラッシャー

n900のRootshパッケージ



開始するには、電話でルート権限を取得します

sudo gainroot







そして、コマンドを実行します

sysinfo-tool -g /certs/npc/esn/bt_id





受け取った番号を保存する



フラッシュドライブを2つの部分に分けます。

1.最初に1Gbメーター（少なくとも256MB）で行ったFat32ファイルシステム

2.残りのすべてのスペース、ext3ファイルシステム

GPartedを使用しました





ext3パーティションをフォルダーにマウントします（デバイス名は異なる場合があります）

mkdir temp

mount /dev/sdc2 temp







作成されたフォルダーに移動し、RootFSを抽出します

cd temp

tar xjvvpf ../Downloads/NITDroid-N900-0.0.1-Demo.tbz2





注意：ルートの下からのみ抽出します。



フラッシュドライブのルートにいるため、ファイルシステム/ bin / load_modules.shを編集する必要があります

echo 0cddef0a0e6dという行を見つけて、 0cddef0a0e6dをsysinfo-toolコマンドの実行時に受け取った番号に置き換えます。

これは、Bluetoothモジュールが正しく機能するために必要です。



フラッシュドライブをアンマウントします

umount /dev/sdc2







次に、USBケーブルを取り外して電話の電源を切ります

キーボードの文字uを押して、USBケーブルを挿入します。

電話は、いわゆるフラッシュモードに転送する必要があります



次に、maemo-flasher'omがコピーされたフォルダーに移動しますzImage-NITDroid-0.0.1

そして、コマンドを実行します

flasher-3.5 --set-rd-flags=no-ext-wd,no-lifeguard-reset --enable-rd-mode -f -k zImage-NITDroid-0.0.1 -R







その後、android'aのダウンロードが開始されます。

いくつかの写真：





すべてを元の場所に戻すには、古いコアをフラッシュする必要があります。

コマンド./flasher-3.5 --disable-rd-mode -f -k zImage -R





zImageはオーバークロックされたコアです。

maemo.orgでは、カーネルイメージがない場合にアドバイスし、元のファームウェアをダウンロードし、次のパラメーターでフラッシュします

flasher-3.5 --disable-rd-mode -f -F RX-51_2009SE_3.2010.02-8_PR_COMBINED_MR0_ARM.bin --flash-only=kernel -R





RX-51_2009SE_3.2010.02-8_PR_COMBINED_MR0_ARM.binはファームウェアファイルです
