Web 2.0を理解する

Web 2.0-この用語を解釈しないとすぐに。 「スマート」な言葉で輝きたいマネージャー、潜在的な顧客に製品を「押し付ける」マーケティング担当者、怠け者だけがWeb 2.0についてまだ話していません。







結局のところ、それは何ですか？ このテーマに関する定義と考えは、Web上に膨大な量を蓄積しています。 それを理解してみましょう。 事実から始めましょう。



Web 2.0という用語の出現は通常、2005年9月30日付けの記事「Tim O Reilly-What Is Web 2.0」に関連付けられています。ロシア語で最初に発行されたジャーナルComputerra（No. 37（609）および38（610）の11および18 2005年10月）、Computerra Online Webサイトの「What is Web 2.0」という見出しの下に掲載されています。 この記事では、Tim O Reillyは、いくつかの一般的な原則によって結ばれた多数のサイトの出現を、インターネットコミュニティの発展の一般的な傾向と結び付け、この現象をWeb 2.0と呼びました。 この用語の意味は依然として多くの議論の対象であるという事実にもかかわらず、Web 2.0の存在を認める研究者は、この現象のいくつかの重要な側面を強調しています。




このウィキペディアの引用は、用語自体の出現についてはあいまいであり、Web 2.0の本質を構成する特定の側面を強調しています。 ここにあります：

  • Webサービス
  • アヤックス
  • ウェブシンジケーション
  • マッシュアップ
  • タグ（タグ）
  • 社会化




文字通りの説明はしませんが、 記事で読むことができます。開発者とユーザーの観点から考えてみます。



Webサービス

ありがとう、 sirkorg



Webアプリケーション

私の謙虚な意見では、WebサービスはWeb 2.0の主要な側面の1つです。 なんで？ 現時点では、ブラウザのみを必要とする特定のサービスを提供する、プラットフォームに依存しない独立したサービスがますます増えているためです。 これらは、オンラインの画像/テキスト/コードエディタ、あらゆる種類のバリデータ、アナライザ、テキストジェネレータ、最終的にはオペレーティングシステムです。

実際、そのようなサイトにアクセスするユーザーは、詳細に触れたり、不要な大量の情報をフィルタリングしたりすることなく、興味のあるものをすぐに受け取ります。 快適で快適です。

予測はしませんが、このようなネットワークの組織の背後には未来があるように思えます。 当然、しばらくすると、役に立たなかった、または請求さ​​れていないサービスの一部が忘却に陥りますが、これは自然なプロセスであり、最も強力なものが生き残ります。



アヤックス

AJAX自体は議論の余地のある用語であり、Web 2.0の主要な側面の1つではないように思えますが、リクエストとレスポンスの時間を短縮し、ページを完全にリロードせずにデータを更新するためにこのテクノロジーの組み合わせを使用するという事実、明確なプラスであり、少なくとも、ユーザーの関心事です。



ウェブシンジケーション

一般にRSSフィードとして知られています。 Web 2.0の間違いない部分。 RSSの助けを借りて、更新に関する通知、コンテンツまたはその一部の1つのリソースから別のリソースへの転送が行われます。これは、AJAXと組み合わせて、ユーザーの要求に応じてコンテンツを生成する十分な機会を提供します。 出力は、これらのテクノロジーを使用して統合された相互依存サービスのネットワークです。 それ自体はマッシュアップです。



タグ（タグ）

重要な役割を果たさない側面。これは、誰もが知っているカテゴリを完全または部分的に置き換えます。 実際には-同じ卵、プロファイルでのみ。 無数のサイトで輝いたタグの「クラウド」は、特定のリソースでより頻繁に取り上げられているトピックの一般的なアイデアを提供しますが、それ以上のものはありません。



社会化

何時間も社交について話し、レポートを書き、グラフを作成することができます。 私の意見では、Web全体の社会化は、Web 2.0の基準の1つよりも少しグローバルな側面です。 みなさんは、優れたサイト/コミュニティ/ソーシャルネットワークの例に精通しており、そのほとんどはユーザーによって作成されています。 この傾向は、Web 2.0が誕生した長い期間に始まり、Webでの人間社会の反映を体現しています。



設計

この側面はウィキペディアの記事には記載されていませんが、私の意見では重要です。 一方、「フォーマット」Web 2.0の多くのサイトのデザインには、誰もが好きではないアイコン、濃いグラフィック、明るい色が豊富にあります。 しかし一方で、多くの思慮深いプロジェクトでは、最後のデザインが主な機能を果たすようになりました-コンテンツの強調と強調、知覚と視覚的な分離の支援。

メインセクターはカラースキームで区切られており、大きくてわかりやすいアイコンは、サービスと他のサービスとの接続の主要な「トリック」を示し、大きく対照的なテキストを使用すると、本質をすばやく把握して主要なものに進むことができます。 使いやすいインターフェースは、成功したいサイトの切り札です。



一般的に、Web 2.0という用語の大規模な使用に加えて、Webの多くの側面をレビューおよび開発する傾向は、前向きな目標をもたらし、インターネットのまさに構造を発展させることに注意したいです。 情報へのアクセスが容易になり、ユーザーの能力が向上し、マーケティングが活況を呈し、自然選択が行われています。



長いライブWeb 2.0、私たちを先導するスローガン。
