ステージ2.軌道面の位置合わせ



火星まで正常に加速するためには、私たちの軌道の平面を火星の軌道の平面と組み合わせる必要があります。 そのためには、MFDの「軌道面を結合する」（ Shift-A ）が必要です。 火星を選択しましょう（ Shift-T 、メニューから天体-火星を選択）：







次のノードは昇順です（Ascend Node、AN）。したがって、軌道の平面から「下」の位置を取る必要があります。 これを行うには、NMLモードがあります（ボタン'または画面の下部にあるNMLボタンをクリックします）。 ノードまでの時間（MFDのTn）がノードでの操作時間（Est Thrust T）の約半分になるポイントを待ち、その中のメインエンジンをオンにします。







ノードまでの時間が半分になったのはなぜですか？ 実際には、操作中にノードの行がシフトされるため、ノードでの操作時間に等しい時間で軌道の修正を開始すると、ノードは「離脱」します。 同じオフセットラインノードは、エンジンを停止するための基準です。 ノードの線が緑の線にほぼ垂直である場合、それ以上の補正は意味がありません。







この単純な公式のおかげで、1つの操作からの軌道の平面を実際に組み合わせました。 ただし、プレーンを最大限に整列させるために操作を繰り返す価値があります。

ステージ3.火星への加速



操作を準備するには、移行MFDをオンにします （ Shift-X ）。 私たちの目標は火星です。火星は太陽を中心に回転するため、太陽を中心体として示します（ Shift-R 、メニューで[太陽]を選択します）。 地球の周りを回転するため、元のボディとして地球を示します（ Shift-S 、メニューでEarthを選択します）。 ターゲットとして火星を選択しましょう（ Shift-T 、メニューで火星を選択）。 MFDは次のようになります（明確にするために説明が追加されています）。









機動計画モードをオンにします（MFDのHTOボタン）。 操作の開始点の点線の半径が表示されます。 Shift-<および>を押すと移動できます。 計画されたパルスは、 Shift--および+ボタンによって制御されます。 火星を目指す：









火星のインターセプトポイントは、軌道の反対側にないことに注意してください。 これは、私たちの軌道がゴマノフスカヤではなく、より多くの燃料を消費するが、より速く飛行することを意味します。



MFDは、2.8 km / sに加速する必要があると書いています。 現在の軌道速度は7.6 km / sです。 すべてがシンプルなようです：7.6 + 2.8 = 10.4 km / s。 しかし、これは真実ではありません！ 私たちの船は地球軌道上にあります。 そして、地球は今度は太陽の周りを公転します。 このような興味深い画像が判明しました。







MFDが提供する2.8 km / sは、私たちではなく、地球の軌道速度に追加する必要があります。 次の要因により、状況はさらに複雑になります。



  • 地球の軌道を離れるには、2番目の宇宙速度（11.2 km / s）まで加速する必要があります。
  • 軌道速度は、軌道の高さに非線形に依存します。
  • 地球から離れると、運動エネルギーがポテンシャルに移動するため、速度が低下します。
  • 加速は瞬間的なものではなく、加速中、速度の一部は位置エネルギーへの移行ですでに失われます。


原則として、目的の速度を計算するための特別なMFDがありますが、かなり高い入力しきい値があります。 したがって、このゴーディアンノットを非常に簡単にカットします。第2の宇宙速度まで加速し、「目で」500〜1000 m / sを追加します。すでに地球から遠ざかっているので、軌道修正を行います。 私たちの軌道速度が地球の軌道速度に追加されるように、軌道の夜側で加速を開始します。









右側のMFD "Orbit"で軌道のパラメーターを監視します。







2番目のスペースがあります！ 速度は11.2 km / sではないことに注意してください。 加速しすぎないように、約11.5 km / sの速度で加速を停止します。

ステップ5.入り口の廊下。



入り口の廊下とは何ですか？ 火星に接近する必要があります。一方では、火星に衝突しないようにし、他方では飛行しないようにします。 私たちの目標は、20 kmのペリセンターです。これにより、確実に大気圏に入り、減速することができます。 左のMFDを「軌道」モードに切り替え（ Shift-O ）、回転体として火星を選択します（ Shift-T 、メニューでMarsを選択します）。 火星の重力が感じられるようになるまで時間をスキップします（ミーティングポイントまで20〜30キロ秒）。







アポセンターは負であり、火星に衝突しなければなりません。 震源を20 kmに上げる必要があります。 これを行うには、鼻を惑星に向け、火星を増やします（ Zボタンは視野を狭め、 Xボタンは視野を広げます）。







昼間はブレーキを選択します。 夜間、時間的制約がある状況では、空間内で方向を失う可能性があり、これは潜在的に危険です。







ペリセンター20 km。 ステージ5が完了しました。 必ず保存してください！



ステージ6.大気中での制動



右側のMFDで「表面」モードを設定し、通常の視野に戻し、船を進行方向に向け、道徳的に困難な段階に備えます。







主な制御は、地球からの離陸時にすでに使用したトリマーと、空力ブレーキ（ Ctrl-Bを押すことで解除および削除）です。 トリマーを完全に上げて、ブレーキを解除します。 Hを押してHUDを「表面」モードに切り替え、90°のロールで位置を取ります。







中心に到達したら、ロールを制御して、ほぼゼロの垂直速度を維持し、ブレーキをかけ続けます。







アポセンターの高さが1000 kmを下回ると、エアブレーキを取り外して大気から脱出し始めます。







アポセンターに到達すると、ペリセンターを100 km以上に上げます。







すべて、私たちは火星の周りの安定した軌道にいます。 必ず保存してください。



理論的には、この段階から着陸にすぐに近づくようにアプローチを構築できます。 しかし、これは第一に困難であり、第二に、船舶の操縦と並行して複数のMFDを使用した非常に高速な作業が必要であり、エラーの可能性が高くなります。 もっと長く、よりシンプルで安全な方法を選びました。



歴史的に、火星のデバイスではエアブレーキが使用されていました。 たとえば、空力ブレーキMROは、通常のブレーキに必要な燃料の約半分を節約しました。 もちろん、装置は私たちが行ったように強引に減速しませんでした。 MROはアポセンターの高さを徐々に減らし、上層大気で繰り返しブレーキをかけました。



ステージ7.着陸の準備



私たちの目標はオリンパス基地です。オリンパス基地は、その名前が示すとおり、火星の最高峰で太陽系で2番目に高いオリンパスの近くにあります。







この段階はいくつかの部分で構成されています。



MFDカードの準備。 MFDカード（ Shift-M ）を開いて、ターゲットとして火星基地のオリンパス（ Shift-T 、Spaceports-Olympusを選択）を選択してみましょう。



MFDエアロブレーキの準備。 MFDを開き（ Shift-E ）、Olympusベースをターゲットとして設定し（ Shift-T 、キーボードからOlympusを入力します）、PGおよびPRJボタンを押してMFDをマップモードに切り替えます。







MFD通信の準備。 MFDを開きます （ Shift-C ）。 ビーコンのデータでメニューを開きます（ Ctlr-I ）。 オリンパス基地には3つの着陸地点（alas、滑走路なし）があり、最初の着陸地点には129.70 MHzの周波数で動作するビーコンがあります。 MFD <、<<、>、>>ボタンを押して、MFDでこの周波数を設定します。









次に、着陸を計画する必要があります。 軌道の傾きがゼロでないということは、軌道が表面に沿って移動することを意味します。 また、ベース「オリンパス」は夜側にあります。 したがって、軌道に乗っている間、2つの要因が一致するのを待っています。ベースは昼側にあり、軌道はその近くを通過します。 必要に応じて、軌道の傾きをわずかに変更できます。 私の場合、10ターン待つ必要はありませんでした。









基地に4〜5千キロが残っている場合、ブレーキ操作を実行して、基地に千または2キロの不足分を残し、船首を前方に向けます。

ステージ7.大気圏飛行と着陸



MFDカードは、船上でセンタリングモードにすることができます（MFDのTRKボタン）。 私たちの主なコントロールは、再びトリマーと空力ブレーキです。 それらを管理することにより、Aerobrake MFDのベースエリアへの軌道を維持します。







積極的な管理を継続します。
