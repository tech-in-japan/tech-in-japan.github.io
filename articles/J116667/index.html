QutIM 0.3ベータリリース

親愛なるhabrausersこんにちは！ Alphaの後の長い休憩の後、コードネームSciFiと呼ばれるqutIMのベータ版をリリースしました。 この1年で膨大な作業が行われましたが、以下に新しいものが登場したことについて説明します。







  • JabberプラグインはGlooxからJreenに書き直されました。glooxの開発が停止されたという事実により、ライブラリをGlooxからJreenに変更することが決定されました。 前者とは異なり、Jreenは完全に非同期であり、応答性にプラスの効果があります。
  • ファイルを転送するためのいくつかのオプションがある新しいファイル転送エンジン ：クライアントからクライアントへ直接、narod.ruを介して。 エンジンは現在活発に開発中であるため、不安定になる可能性があります。
  • SymbianおよびMaemo 5のポート ：1年にわたってqutIMはよりモバイルになり、 SymbianおよびMaemo 5のポートが登場しました 。 1つのポールがAndroid用のqutIMポートを開発していますが、残念ながら手が足りないために遅れています。 これまでのところ、他のプラットフォーム（iOS、webOS、MeeGo）については、手が足りないため、夢見ることさえできません:(





























  • QMLテーマ ： QMLのテーマとポップアップのサポートが追加されました。 これはまだ実験的な機能であるため、使用するには強い意志があり、QMLの可能性を探求する準備ができている必要があります。 好きですか？ その後、私たちはあなたを待っています！ フォーラムでチャットトピックの作成に関するドキュメントを見つけることができます。

  • 連絡先リスト ：連絡先リストを使用して、あなたの心が望むものは何でもできます。 リストを作成するか、プロトコルを分類できます。 組織を担当するモデルであるCLを描画するためのデリゲートを変更できます。 確かに、モデルを書くのに十分な手がありません。そのため、「アカウントなし」リストのみが実装されています。つまり、すべての連絡先が同じリストにあります。

  • LibAstral ：FreedesktopのTelepathyフレームワークの初期サポート。

    思考を読み、それらをテレパシーフレームワークを介して頭に保存するためのサポートがあります。 過去を調べ、未来を予測するための実験的サポートを追加しました。

  • タッチスクリーンのサポート ：symbianで開始され、Maemoに移行され、どういうわけかデスクトップバージョンで終了しました:)

  • Win、Mac、KDE、Maemoとの統合：Windows 7、KDE4、GNOME（開始済み）、Mac OS XおよびMaemoとの統合があります。





















  • Libpurple ：libpurpleのプロトコルサポートのおかげで、qutIMはこのライブラリの一部であるすべてのプロトコルをサポートするようになりました。

  • チャットの形式 ： チャットにはいくつかの形式があります。モバイルバージョンでは、ジェスチャーとデスクトップによって制御が実行されます。

  • 新しいWebサイトと新しいロゴ ：この機会に、今までqutIMはロゴに十分な注意を払っていませんでした。 今、あなたが目の前で見る「小さな光」がこの方向での最初の実験である新しいロゴの開発があります。

    また、更新されたか、新しいサイトが誕生しました。

  • DBus APIおよびScript API ：JavascriptおよびDBusへのLibqutimバインディングにより、単純な拡張機能を記述したり、cutimのリモート制御を実行したりできます。
