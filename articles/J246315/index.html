TeamCity 9.0：プロジェクトのインポート、VCSの設定、バックグラウンドクリーンアップ、パンダ

先週、CIサーバーの新しいバージョンであるTeamCity 9.0がリリースされました。 彼女は「あなたの常設ビルドエンジニア」というスローガンの下に出てきました。これは偶然ではありません。 新しい機能は、主にビルド​​エンジニアの作業を促進することを目的としており、メンテナンスを中断することなくビルドサーバーを24時間365日動作させることができます。



もちろん、 すべての改善点のリストは 、いつものように非常に印象的であり、必要に応じて理解することができます。 ただし、ここでは、最新リリースの主要な機能に焦点を当てます。











プロジェクトをインポートする



プロジェクトをインポートすると、サーバー間でプロジェクトを簡単に移動でき、履歴とユーザープロファイル全体が保持されます。 この機能は、多数のCIサーバーを管理し、サーバー間でプロジェクトを転送するタスクに頻繁に直面する場合に特に役立ちます。 転送中に以前の設定のみが保存されていた場合、プロジェクトはビルド、変更、ユーザープロファイルの全履歴と共に移動しています。 これにより、大規模プロジェクトでのTeamCityの展開の初期計画に費やす時間と労力を削減できることを願っています。

VCSの設定



これで、プロジェクト設定をバージョン管理システムに保存し（GitとMercurialがサポートされている間）、ソースコードと同様にそれらを表示および検査できます。 お気に入りのツールを使用して、変更を監査および表示できます。 そして最も重要なことは、致命的なミスを犯すリスクが減ったことです。いつでもすべてをロールバックできるからです。 また、同僚をフォローして、設定で誰が、何を、いつ変更されたかを確認することもできます。

バックグラウンドクリーニング



以前は、サーバーのクリーンアップを毎日オンにする必要があり、そのためサーバーが15分から数時間使用できなかった場合、新しいバージョンでは、クリーンアップはバックグラウンドで実行されます。 これにより、サーバーのダウンタイムが完全に防止され、サーバー時間の100％がビルドのビルドに割り当てられます。 これは、分散したチームや、TeamCityのエージェントとサーバーが多数あるチームに特に役立ちます。



チャートの作成と編集



カスタムダイアグラムの簡素化された管理により、XMLを編集する必要なく、TeamCityインターフェイスから直接編集できます。







お気に入りのビルド



お気に入りのビルド機能も登場しました。これにより、特定のビルドを「お気に入り」としてマークし、ブラウザで多数のタブを開いたままにすることなく、いつでも簡単に追跡できます。







統合



TeamCity 9.0は、重要なツールとの多くの便利な統合（プラグインの形式）も提供します。

  • Microsoft Azureとの統合により、必要に応じて「ビルドファーム」を簡単に拡張できます。 TeamCityは、Azureクラウド内の必要な数のエージェントを起動し、負荷が低下するとエージェントを停止します。
  • より効率的なビルド処理のために、 VMware vSphere仮想マシンにインストールされたリモートTeamCityエージェントも使用できるようになりました。 ビルドキューを分析した後、TeamCity 自体が 、対応するTeamCityエージェントと共に仮想マシンを起動します。
  • Microsoft Visual Studio Onlineとの統合により、VSOを TeamCityのバージョン管理システムとして構成し、 TeamCityビルドとバージョン管理履歴をVSOワークアイテムに関連付け、 Team Roomで直接TeamCityから通知を受け取ることができます。




TeamCity 9.0の最新バージョンは、当社のWebサイトからダウンロードできます。 新しいバージョンに関するコメントや提案は、 バグトラッカーに残すことができます。



また、2015年1月21日に無料のウェビナーにアクセスして、TeamCity 9.0の新機能を詳しくご覧になることをお勧めします 。 登録は既にオープンしています 。



TeamCityは時間と機能に制限なく無料で使用できます。 3つのエージェントと20のビルド構成が突然足りなくなったり、メールでサポートとやり取りしたい場合は、以前60日間の試用版を受け取ったEnterpriseライセンスをいつでも購入できます。
