colinux、Virtual PC、およびデスクトップで複数のオペレーティングシステムを同時に使用する

前文：



最近、私はいくつかのコードを書く必要があり、その作業の結果を異なるプラットフォームでテストする必要があります。 当然、仮想マシンを使用します。

私のほかに、妻も私のラップトップを使用しているため、そのメインOSはwinxpです。 原則として、何を扱うかは気にしませんので、ウィンドウを…



以前、IceWMをウィンドウマネージャーとして使用していたとき、複数の「デスクトップ」を使用して、個別に割り当てられた場所でさまざまなテーマアプリケーションを起動できることが気に入っていました。 窓の下にもこのような機会がありますが、私が試したユーティリティにはいくつかの欠点があったので、遅かれ早かれそれらを放棄しなければなりませんでした...最終的に、私は（sysinternalsから） デスクトップに落ち着きました、それはまた欠陥がないわけではありませんが、その安定性仕事と小さなサイズがすべての「短所」を上回りました。



目的：



達成しようとしていること：
  • 個別のオペレーティングシステム上の各「デスクトップ」で実行する
  • それらをできるだけ便利かつ高速に切り替える








必要なもの：



coLinuxはここからダウンロードできます （WindowsでLinuxカーネルを実行するために必要です）

Virtual PCはここからダウンロードできます （Windows OSでWindows OSを実行するために必要です:)）。 他の仮想マシンを使用できます（自宅ではVirtualBoxを使用し、職場ではVirtual PCを使用）。これは重要ではありませんが、この記事ではVirtual PCについて説明します。

linuxディストリビューション （以下、Slackware 12.2を使用するため、これについて説明します）

こちらからPuTTYをダウンロードして、ssh経由で仮想マシンにアクセスできます。

ここからXmingをダウンロードしてください。X11プロトコルを介してグラフィカルLinuxアプリケーションを実行する必要があります。



インストール自体



デスクトップ


最初に、デスクトップをインストールします。すべてがシンプルで明確です-ダウンロード、解凍、実行







「テーブル」を切り替える最も便利な方法は、win-digitをホットキーとして設定することだと思いました。



仮想PC


インストールプロセス自体も簡単です。ダウンロードしてインストーラーを実行し、目的のオペレーティングシステムをVirtual PCにインストールします。 注意したいニュアンスが1つだけあります... win-digitがゲストシステムによってインターセプトされないようにするには、スクリーンショットに示すようにキーボード設定を構成する必要があります。



Virtual PC Console ...メニュー項目File \ Options







次に、 キーボードセクションを選択します







これらの設定をインストールした後、win-digitでゲストシステムから目的の「デスクトップ」に簡単に切り替えることができます。



coLinuxをインストールする


次に、より複雑なトピック-coLinux内へのLinuxディストリビューションのインストールに進みます。

まず、coLinuxをインストールする必要があります。

coLinuxのインストール中に、目的のイメージを後でダウンロードするため、[ ルートファイルシステムイメージ（ダウンロード）]チェックボックスをオフにする必要があります。 coLinuxはc：\ colinuxにインストールする必要があります。次に、Linux Slackware 12.2ディストリビューションイメージが必要です。 作業中の「ミラー」の1つ（http://slackware.com/getslack/）から取得できます。DVDイメージをダウンロードし、ファイル名slack12.isoのc：\ colinux \ linux \ディレクトリに配置する必要があります。

次に、空のファイルを作成してドライブとして接続する必要があります。

c：\> fsutil.exeファイルcreatenew c：\ colinux \ linux \ hda.img 7000000000

c：\> fsutil.exeファイルcreatenew c：\ colinux \ linux \ hdb.img 3000000000

c：\> fsutil.exeファイルcreatenew c：\ colinux \ linux \ swap.img 300000000



次に、c：\ colinux \ディレクトリにcoLinux起動オプションを含むファイルを作成する必要があります-Slackware install.confを直接インストールするため

カーネル= vmlinux

hda = c：\ coLinux \ linux \ hda.img

hde = c：\ coLinux \ linux \ swap.img

hdd = c：\ coLinux \ linux \ slack12.iso

ルート= / dev / ram load_ramdisk = 1 prompt_ramdisk = 0 ramdisk_size = 10000

initrd = initrd.img

mem = 256



Slackwareのその後のインストールでは、coLinuxの最初の実行時にinitrd.imgファイルが必要になります。これは、ダウンロードされた配布イメージ内にあります。 そこから抽出するには、7zipアーカイバまたはwinrarを使用できます。 c：\ coLinux \ linux \ディレクトリに解凍します



Slackwareをインストールする


すべての準備が整ったら、Linuxのインストールを開始する必要があります。 c：\ coLinux \ディレクトリから、次のコマンドを実行します。

colinux-daemon.exe -t nt install .conf



キーボードレイアウトを選択してユーザー名を入力するプロンプトが表示されます。Enterキーを2回押すことができます。 次に、Linuxディストリビューションディスクを接続します

マウント/ dev / hdd /フロッピー



スワップ部門をつなぐ

mkswap / dev / hde

swapon / dev / hde



Linuxをインストールするためのメインセクションをフォーマットしてマウントします。

mkfs.ext2 -j / dev / hda



とにかく続行する質問に？ （y、n） yと答える必要がある

マウント/ dev / hda / mnt



次に、必要なパッケージをインストールする必要があります

cd /フロッピー/ slackware

installpkg -root / mnt ./a/*.tgz ./ap/*.tgz ./d/*.tgz ./f/*.tgz ./l/*.tgz ./n/*.tgz



すべてのパッケージをインストールした後、ファイル/ mnt / etc / fstabを編集して、その後のダウンロードですべてのパーティションが正しく接続されるようにする必要があります。

このようにファイルを編集できます

nano / mnt / etc / fstab



次の行を挿入する 

  / dev / hda / ext3 ro、noatime 2 2
 / dev / hdb /ホームext3 rw、noatime 2 2
 / dev / hdd / media / dvd iso9660デフォルト0 0
 / dev / hdeスワップスワップデフォルト0 0
 proc / proc procデフォルト0 0
 devpts / dev / pts devpts gid = 5、モード= 620 0 0




ファイルを保存したら、次のコマンドで仮想マシンをオフにします。

パワーオフ





サービスとしてcoLinuxを実行する


ここで、c：\ colinux \ディレクトリに標準のcoLinux起動オプションを使用してファイルを作成する必要があります

slack.conf

カーネル= vmlinux

hda = c：\ coLinux \ linux \ hda.img

hdb = c：\ coLinux \ linux \ hdb.img

hdd = c：\ coLinux \ linux \ slack12.iso

hde = c：\ coLinux \ linux \ swap.img

ルート= / dev / hda

ろ

initrd = initrd.gz

mem = 128

cocon = 120x40

eth0 = tuntap



次に、インストールされたSlackwareを初めて実行します

colinux-daemon.exe -t nt slack .conf



すべてがうまくいき、OSが起動したら、 / etc / profileファイルの最後に次の行を配置する必要があります。

エクスポートLANG = ru_RU.UTF-8



そのため、ロケールとしてUTF-8を使用します。

また、ゲストシステムのIPアドレスを設定する必要があります。ファイル/etc/rc.d/rc.inet1.confで 、IPアドレス、マスク、デフォルトゲートウェイを設定する必要があります。 



 IPADDR [0] = "192.168.0.2"
 NETMASK [0] = "255.255.255.0"
ゲートウェイ= "192.168.0.1"




再起動

再起動



インストールされたSlackwareをWindowsサービスとして実行します

cd c：\ colinux

colinux-daemon.exe --install-service colinux slack .conf

net start colinux



sshを介してゲストマシンに接続できた場合、次の手順に進むことができます...



Xming Xming Serverをインストールします



Xmingのインストールと構成は非常に簡単ですが、私が詳しく説明したくない明らかな点がいくつかあります...

  • 最初のポイントは、Xサーバーを起動するためのオプションです
  • 2番目はフォントを接続することです




さらに...



フォント接続


私のように、ロシア語のテキストを表示して入力したい場合は、キリル文字フォントをインストールする必要があります。 これを行うには、フォントディレクトリで実行するだけで非常に簡単です。

mkfontdir

mkfontscale



上記のユーティリティはcigwinキットから取得できます。または、単にフォントディレクトリ全体をLinuxマシンにコピーして、そこでコマンドを実行することもできます。

フォントディレクトリは、ファイルC：\ Program Files \ Xming \ font-dirsに書き込む必要があります。



Xサーバー接続の作成


使用するキーボードレイアウトとそれらの切り替え方法を指定するには、それらをパラメータとして指定する必要があります。

ロシア語と英語の2つのキーボードレイアウトが必要で、Alt-Shiftで切り替える場合、オプションは次のようになります。

-xkblayout us、ru -xkbvariant basic、winkeys -xkboptions grp：alt_shift_toggle



スクリーンショットは、別の「デスクトップ」でXサーバーを実行する方法を示しています











Xming設定ファイルを保存し、デスクトップに起動するショートカットを作成します。



すべてをまとめると...



4つの「デスクトップ」があります

  • 最初に「ネイティブ」アプリケーションを起動します
  • 2つ目は、ゲスト仮想OSウィンドウを全画面で展開します（あらゆる種類の非人的実験に必要です）
  • Linuxを搭載した3番目のフルスクリーンXサーバー


マシンをロードした後、2番目の「デスクトップ」に切り替え（win-2を押します）、Virtual PCを全画面で起動します。 次に、3番目の「デスクトップ」に切り替え（win-3を押します）、ショートカットを実行してXming経由でlixuxに接続します...

ここで、Windows環境からLinuxに切り替えるには、win-3の組み合わせをクリックするだけです。windowsを備えたゲストマシンにアクセスするには、win-2をクリックし、メインの作業に戻るには、win-1をクリックします。 同時に、すべてが並行して動作し、いつでも目的のOSに戻ることができます。
