OpenStackは「新しいLAMP」になりますか？

habraの各ユーザーはLAMPが何であるかを知っていると思います。 しかし、私にとって、LAMPはLinux + Apache HTTPサーバー+ MariaDB / MySQL + PHP / Python / Perlの単なるバンドル（スタック）ではありません。 私にとって、これはオープンソースソフトウェアがIT業界の革命的な変化の原因となりうる（そしてそうすべきである）シンボルです。 インターネットが現在の形で登場したのは、LAMPのおかげだと確信しています。 これらの4つの文字の後ろに隠されているものがなければ、Google、Facebook、およびHabrahabrはありません。 まあ、少なくとも私たちが知っているようにはなりません。 想像してみてください。 世界最大の資本を持つ会社を作ることができる技術は、わずか20〜25年前に登場しました。




仮想化市場で 「 なぜCitrixが「新しいRed Hat」にならないのか 」 という投稿を書いたとき、オープンソースソフトウェアが重要な役割を果たす別のIT領域としてサーバー側の仮想化を考えました。 残念ながら、これは（まだ？）発生していませんが、 おそらくOpenStackはエラーを回避し 、データセンターの事実上の標準になるでしょう 。 かどうか？ 投稿後のアンケート。



OpenStackとLAMPの類似点は何ですか？

もちろん、OpenStackはLAMPと多くの関係があります。 技術的には、 OpenStackを使用するとLinuxが優先され、OpenStackコードはPythonで記述されます。 さらに...統計情報はありませんが、LAMPスタックを備えた仮想マシンをデプロイするためにOpenStackが特に必要になることはそれほど珍しくないと思います。



しかし、これらのスタック間の主な類似点は、もちろん、使用するソフトウェアに共通部分があるということではありません。 主なものは、同様のアイデアが基本的に-簡単に言えば、「高速スケーリング」と呼ばれることもあるということです（そして、安価な場合があり、これはより重要です）。 LAMPを使用すると、リソースを非常に迅速に増やすことができます-いくつかのWebサイトではなく、たとえば数百のWebサイトへのアクセスを提供する場合... そのような拡張にはどれくらい時間がかかりますか？ ライセンスとソフトウェアの購入に問題なく、最寄りのコンピューター店に行き、そこでハードウェアを購入し、LAMPを「埋める」だけです。 OpenStackでは、必要なリソースを迅速にスケーリングすることもできますが、もちろん、別の「仮想」レベルで行うこともできます。



もう1つの類似点は、LAMPもOpenStackも、いずれかの会社の仕事の結果ではないことです。正確には、これらのプロジェクトには明確なリーダーはいません。 LAMPスタックは一般にかなりarbitrary意的な組み合わせであり、そのコンポーネントはさまざまなプロジェクトの一部として開発されました。OpenStackは元々NASAとRackspaceのプロジェクトでしたが、多数の企業や独立した開発者がすぐに接続しました。



ところで、MySQL ABのCEOであるMarten MickosがSun Microsystemsに買収された後、 Eucalyptus Systemsに異動したことは注目に値します。 少し後に、 HPがEucalyptusを買収し 、Martin MikosがHP Cloudの上級副社長兼ゼネラルマネージャーを引き継ぎました。 2012年以来、 HPはOpenStackを使用してクラウドビジネスを構築しており 、このプラットフォームのクラウドソリューションに10億ドル以上を投資しており、2014年にこのIaaSの最も活発な開発者になりました 。 結果は、あるスタックから別のスタックへの一種の移行でした。



ベットは行われますか？

しかし、これはどれほど深刻ですか？ 実際、LAMPは実績のあるソフトウェアパッケージであり、OpenStackはまだ非常に若いオープンソースプロジェクトと考えられています。 私の最後の投稿へのコメントには、次のような言葉がありました。
「Openstockは誇大広告であり、生き残ることはできますが、そうでない場合があります。 現在、彼は単にそこにお金を注ぐ準備ができている企業を犠牲にして生きています。なぜなら、彼らにはたくさんのお金があるからです。」
これはかなり一般的な意見です。OpenStackが登場してから（ほぼ6年間！） しかし、企業は多くのお金を持っているという理由だけで、どこにもお金を注ぎません。 彼らは、これらの投資の潜在的利益が見込める場所に投資します。 これは、特定の技術の見込みを判断する非常に簡単な方法です。大企業がその技術に投資しているかどうかを確認する必要があります。 結局のところ、彼らが何かに投資するなら、彼らは投資に対するリターンを期待します。



近年、OpenStackに関連するさまざまなプロジェクトや企業への投資は数十億ドルに達しました。 OpenStackに関連する主な金融取引を思い出してください。
  • EMCは、OpenStackプラットフォームに基づくクラウドサービスの主要プロバイダーの1つである5,000万ドルでCloudscalingを購入しました。
  • IBMの買収の目的はBlue Boxクラウドサービスプロバイダーでしたが、さらに、IBMは15の新しいデータセンターの建設に12億ドルを費やすことを計画しました。
  • シスコシステムズは、クラウドビジネスの開発に10億ドルを投資する意向を発表し、Piston CloudおよびMetacloudの企業を買収しました（金額は明らかにされていませんが、この購入は最大のOpenStackコミュニティと呼ばれていました）。
  • Red HatはOpenStack eNovanceを9,500万ドルで買収しました。
  • Mirantisは1億ドルの追加資金を集めました。
  • 私はすでに上記のHPEについて書いています-OpenStackベースのクラウドビジネスで10億ドル以上。


たとえば、前述のHPEとシスコがクラウドインフラストラクチャの生産において主導的な地位を占めているなど、OpenStackへの投資はすでに回復しているとすぐに言わなければなりません。 （シスコは、パブリッククラウドのインフラストラクチャ要素、およびプライベートクラウドのHPEの主要生産者です。）上記の企業に加えて、Oracle、SAP AB、Intel、Huawei、NEC、SUSE、エリクソン、VMware、Dell、NTTは、OpenStackコンピテンシーを積極的に開発しています。 Citrix。 さまざまな企業がプロジェクトに投資しているという事実に注目したいと思います。 IT業界全体でのオープンソースサポートの必要性が確認されました。



要約すると。 OpenStackプロジェクトはLAMPの「軌跡をたどっています」が、OpenStack Foundationのおかげで、より管理しやすく、財政的に持続可能です。 このプロジェクトへの大規模なIT企業の投資は数十億ドルになります。 IaaS OpenStackの最新リリースの開発には、345の異なる企業を代表する2,336人の開発者が参加しました。 その後、OpenStackが生き残ることができるかどうかについて疑問がありますか？



追伸：OpenStackには問題がないという印象をあなたに与えたくありません。 前回の投稿でいくつかのことを書きました。
