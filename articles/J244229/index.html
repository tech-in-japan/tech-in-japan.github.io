AndroidアプリでのGoogleマップのルート

最近、アプリケーションのGoogleマップに2つのポイント間のルートを表示する必要がありました。 Habréには、このテーマに関する出版物がすでにありました。 たとえば、 「AndroidアプリケーションでのGoogleマップのルート」 。 しかし、これらの資料はすでに長い間満足しており、新しい機会を利用していません。 ルートを描画する別の方法を示したいのですが、誰かに役立つかもしれません。



Googleマップをアプリケーションに統合するプロセスは省略します。興味のある方は、 Googleの詳細ガイドですべての情報を見つけることができます。 ルートを表示するプロセス全体は、いくつかの段階で構成されています。



  • ルートに関する情報を取得します。
  • 受信した応答の処理。
  • 地図にルートを描きます。


これらの手順を検討してください。



ルートを取得する



ルート情報を取得するには、Googleルートサービスへのリクエストを完了する必要があります。 クエリと返されたクエリの詳細な説明は、 Googleで入手できます 。 ルートを取得するには、次の形式のクエリを実行する必要があることに注意してください。 



https://maps.googleapis.com/maps/api/directions/output?parameters


出力として、 XMLまたは（この場合は）JSONを選択できます。 必須パラメータには、 出発地と目的地が含まれます。これらは、住所のテキスト表現として、またはコンマで区切られた緯度と経度の値として指定できます。 3番目の必須パラメーターsensorは、要求が位置センサーを備えたデバイスからのものかどうかを示します 。この場合、常にtrueになります 。



リクエストの形式を決定したら、Googleルートサービスへのリクエストを処理して応答を受け取る方法を選択する必要があります。 Retrofitライブラリを使用します。これにより、RESTサービスへのリクエストを数行で作成できます。



Retrofitを使用するには、ライブラリをプロジェクトに接続する必要があります。 gradleを使用して、これはgradleファイルに1行の依存関係を追加することで実行されます。 



 dependencies { compile 'com.squareup.retrofit:retrofit:1.7.1' }




次に、Google Route Services APIについて説明する必要があります。 これを行うには、いくつかのメソッドを作成する新しいJavaインターフェイスを作成し、Retrofitアノテーションを使用してそれらをサーバー上のさまざまなメソッドにマップします。 情報のみを受け取るため、GETリクエストのメソッドを1つだけ記述する必要があります。 



 public interface RouteApi { @GET("/maps/api/directions/json") RouteResponse getRoute( @Query(value = "origin", encodeValue = false) String position, @Query(value = "destination", encodeValue = false) String destination, @Query("sensor") boolean sensor, @Query("language") String language); }




引数としてのGETアノテーションは、クエリを実行するサーバー上のディレクトリを取得し、メソッド自体で既に各パラメーターにQueryアノテーションを付けます。 各注釈の引数として、リクエストに含めるパラメーターの名前。 この場合、 出発地と目的地のパラメーターについて、 encodeValueフラグの値を設定します。これにより、 Retrofitに、リクエスト内の緯度と経度の値を区切るコンマをエンコードしないように指示します。 また、サーバーからの応答がロシア語になるように、もう1つの言語パラメーターを追加します。 RESTメソッドはオブジェクトを返す必要があります。それをRouteResponseと呼びましょう。 後で説明しますが、とりあえずRouteResponseという別のクラスを作成してください。



サービスのAPIを説明した後、リクエストを処理できます。 これを行うには、RestAdapterを作成し、リモートサービスを表すサービスを作成し、そのAPIメソッドを呼び出す必要があります。 



 RestAdapter restAdapter = new RestAdapter.Builder() .setEndpoint("https://maps.googleapis.com") .build(); RouteApi routeService = restAdapter.create(RouteApi.class); RouteResponse routeResponse = routeService.getRoute(position, destination, true, "ru");




Googleのルートサービスからルートを取得するのに必要なことはそれだけです。 RestAdapterコンストラクターに行を追加する 
 .setLogLevel(RestAdapter.LogLevel.FULL)
、リクエストを実行し、ログでサーバーからのレスポンスを確認できます。 しかし、私たちはそこで止まりません。



受信した応答の処理



クエリの結果として、RouteResponseオブジェクトを取得します。 実際、サーバーからJSONを要求したため、サーバーからの応答はJSON形式になります。 サーバーから応答を受信したRetrofitは 、Google GSONのパーサーを使用してJSONの解析を個別に開始し、JSONをRouteResponseオブジェクトに解析します。 必要に応じて、別のパーサー-JacksonまたはInstagramのJSONパーサーを選択できますが、GSONを使用することを好みます。 GSONにはRetrofitが付属しているため、プロジェクトで使用するために追加の依存関係を含める必要はありません。



JSONレスポンスからデータを取得するには、このデータを記述するクラスを作成する必要があります。 RouteResponseクラスは既に作成されていますが、いくつかのコンテンツで埋めることは残っています。 Google Route Serviceサーバーからの応答の一般的な構造は次のとおりです。

ご覧のとおり、答えとして、 ルートセグメントを構成するSteps ステップとセグメントに関する情報で構成されるLegsセグメントの配列を含むRoutesルートの配列を取得します。 前の例では、ルートはセグメントの各ステップに関する情報に基づいて構築されましたが、RouteオブジェクトにはすでにOverview_polylineオブジェクトが含まれています-これは、結果のルートの近似（滑らかな）パスを表すエンコードポイントの配列を持つオブジェクトです。 ほとんどの場合、このスムーズなルートで十分です。 したがって、描画に使用します。



この情報に基づいて、GSONのモデルクラスを記述します。 



 public class RouteResponse { public List<Route> routes; public String getPoints() { return this.routes.get(0).overview_polyline.points; } class Route { OverviewPolyline overview_polyline; } class OverviewPolyline { String points; } }




リクエストを処理してRouteResponseオブジェクトを取得すると、そこからポイントラインを取得できます。 初期状態では、ほとんど何も得られません。 そこから情報を取得するには、解読する必要があります。 ここでは、 Google Maps Android APIユーティリティライブラリのPolyUtilクラスが役立ちます。 それを使用するには、プロジェクトに次の依存関係を含める必要があります。 



 dependencies { compile 'com.google.maps.android:android-maps-utils:0.3+' }




PolyUtilにはdecode（）メソッドが含まれており、これはポイントのストリングを取得し 、ルートのノードであるLatLngオブジェクトのセットを返します。 これで、地図上にルートを描画できます。



地図にルートを描く



古い例では、ルートを描画するためにオーバーレイが使用されましたが、ポリラインクラスで管理します-この場合、オーバーレイから継承された追加のクラスを作成する必要はなく、記述する必要があるコードの量は大幅に削減されます。 ポリラインは、マップ上のポイントとそれらを結ぶ線のリストです。 その後、ポリラインをマップに追加できます。 



 PolylineOptions line = new PolylineOptions(); line.width(4f).color(R.color.indigo_900); LatLngBounds.Builder latLngBuilder = new LatLngBounds.Builder(); for (int i = 0; i < mPoints.size(); i++) { if (i == 0) { MarkerOptions startMarkerOptions = new MarkerOptions() .position(mPoints.get(i)) .icon(BitmapDescriptorFactory.fromResource(R.drawable.ic_marker_a)); mGoogleMap.addMarker(startMarkerOptions); } else if (i == mPoints.size() - 1) { MarkerOptions endMarkerOptions = new MarkerOptions() .position(mPoints.get(i)) .icon(BitmapDescriptorFactory.fromResource(R.drawable.ic_marker_b)); mGoogleMap.addMarker(endMarkerOptions); } line.add(mPoints.get(i)); latLngBuilder.include(mPoints.get(i)); } mGoogleMap.addPolyline(line); int size = getResources().getDisplayMetrics().widthPixels; LatLngBounds latLngBounds = latLngBuilder.build(); CameraUpdate track = CameraUpdateFactory.newLatLngBounds(latLngBounds, size, size, 25); mGoogleMap.moveCamera(track);




まず、 PolylineOptionsクラスのインスタンスを作成し、線の太さと色を設定します。 次に、 LatLngBuilderのインスタンスを取得して、マップのスケーリングに使用される境界ボックスを作成します。 次に、Google Routes APIからの応答をデコードした結果として取得されたLatLngオブジェクトのリストを調べ 、各ポイントをlineおよびLatLngBuilderに追加します。 リストの最初と最後のオブジェクト（それぞれ開始点と終了点の座標）に対して、マーカーを作成し、マップに追加します。 リスト項目の列挙が完了したら、addPolyline（）メソッドを呼び出して、構築された行をマップに追加します。



次に、ルート全体を表示するようにマップをスケーリングする必要があります。 マップ内を移動するには、CameraクラスのmoveCamera（）メソッドを使用します。このメソッドは、入力としてUpdateCameraオブジェクトのカメラ設定を受け取ります。 UpdateCameraFactoryクラスのnewLatLngBoudnsメソッドを呼び出して、CameraUpdateオブジェクトを作成します。 これに、作成したLatLngBoundsオブジェクトを渡します。これには、ルートのすべてのポイントが含まれ、画面の幅が渡され、エッジからインデントが追加されます。 その後、メソッドを呼び出してカメラを移動します。 それだけです、ルートが描かれます。
