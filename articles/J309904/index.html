Veeamを使用したドメインコントローラーのバックアップ

Microsoft Active Directoryは、ユーザー認証と集中管理の業界標準です。 システム管理者がこのテクノロジーなしで作業を管理する方法を想像することはほとんど不可能です。 ただし、Active Directoryを使用することには大きなメリットがあるだけでなく、多大な責任が課せられ、かなりの時間と作業プロセスの理解が必要になります。 したがって、Veeamソリューションを使用してActive Directoryを正常にバックアップおよび復元する方法を説明する記事をいくつか紹介します。 特に、Veeamがドメインコントローラー（DC）または個々のADオブジェクトのコピーを作成し、必要に応じてそれらを復元する方法を説明します。



今日の投稿では、Veeamが提供する物理および仮想ドメインコントローラーのバックアップ機能と、バックアップ中に覚えておく必要があることについて説明します。 詳細-猫の下。







Active Directoryサービスは冗長性を考慮して設計されているため、通常のバックアップルールと戦術をそれに応じて調整する必要があります。 この場合、SQLまたはExchangeサーバーで既に機能しているのと同じバックアップポリシーを使用するのは正しくありません。 Active Directoryのバックアップポリシーの設計に役立つガイドラインを次に示します。



  1. 環境内のどのドメインコントローラーがFlexible Single Master Operations（FSMO）として機能するかを確認します。

    便利：コマンドラインで確認する簡単なコマンド： > netdom query fsmo

    完全なドメイン回復を実行する場合、FSMOの役割が最も多いドメインコントローラーから開始するのが最適です。これは通常、プライマリドメインコントローラー（PDC）のエミュレーターの役割を持つサーバーです。 それ以外の場合は、回復後、適切な役割を手動で再割り当てする必要があります（ntdsutil seizeコマンドを使用）。



  2. 個々のオブジェクトを保護する場合、本番サイトで使用可能なすべてのコントローラーをバックアップする必要はありません。 個々のオブジェクトを復元するには、Active Directoryデータベースのコピー（ntds.ditファイル）で十分です。



  3. ADオブジェクトを誤ってまたは意図的に削除または変更するリスクを減らす機会は常にあります。 管理権限の委任、昇格した権限でのアクセスの制限、および事前定義された遅延を伴うバックアップサイトへの複製を推奨できます。



  4. 通常、ドメインコントローラーを1つずつバックアップし、DFSレプリケーションと重複しないようにすることをお勧めします。 現代のソリューションはこの問題を解決する方法を知っていますが。



  5. 仮想VMware環境を使用している場合、ネットワーク経由でドメインコントローラーにアクセスできない場合があります（たとえば、DMZゾーンにあります）。 この状況では、VeeamはVMware VIXを介して接続に切り替え、このコントローラーを処理できるようになります。




仮想DCがある場合



Active Directoryサービスはシステムリソースのごく一部を消費するため、通常、ドメインコントローラーは仮想化の最初の候補になります。 Veeamで仮想化されたコントローラーを保護するには、Veeam Backup＆Replicationをインストールして構成する必要があります。



重要！ このソリューションは、Windows Server 2003 SP1以降のドメインコントローラーVMで動作します。サポートされる最小のフォレスト機能レベルはWindows 2003です。アカウントにはActive Directory管理者権限が必要です。エンタープライズまたはドメイン管理者のアカウントで作業できます。

Veeam Backup＆Replicationのインストールおよび構成プロセスは、すでに何度か説明されています。たとえば、Veeamシステムエンジニアが作成したビデオで、詳細なしで実行できます。 すべてがセットアップされ、準備ができていると仮定します。 次に、ドメインコントローラーのバックアップタスクを作成する必要があります。 セットアッププロセスは非常に簡単です。



  1. バックアップタスク作成ウィザードを実行します。



  2. ドメインコントローラーを選択します。



  3. バックアップチェーンの保持ポリシーを定義します。



  4. アプリケーションの状態を考慮してデータ処理機能をオンにし（図1）、VM上で実行されているOSおよびアプリケーション（Active DirectoryデータとSYSVOLディレクトリのデータベースを含む）のトランザクションレベルの一貫性を確保します。 これを行うには、[ アプリケーション対応画像処理 （AAIP）を有効にする ]オプションを選択します。



    AAIPは、アプリケーションステータスに基づいてVMバックアップを提供するVeeamテクノロジーです。 ゲストOSアプリケーションを検索し、そのメタデータを収集し、適切なメカニズム（Microsoft VSSライター）を使用して「フリーズ」し、復元されたVMの初回起動時に実行されるアプリケーションのVSSを使用した回復手順を準備し、成功した場合はトランザクションログを切り捨てますバックアップ完了。 AAIPが有効になっていない場合、ドメインコントローラーのゲストOSは、バックアップおよび保護されていることを認識しません。 そのため、しばらくすると、 イベントID 2089サーバーログで内部警告が表示される場合があります：バックアップ遅延間隔 （つまり、バックアップ遅延間隔中にバックアップが実行されなかった）。



  5. タスクのスケジュールを設定するか、手動で実行します。



  6. ジョブが正常に完了したことを確認します。





  7. リポジトリで新しく作成されたバックアップファイルを見つけます-完了！


バックアップは、Veeam Cloud Connectサービスプロバイダー（VCC）を使用してクラウドにバックアップすることもできます。 バックアップアーカイブタスクまたは磁気テープのアーカイブ機能を使用して、別のバックアップリポジトリに転送することもできます。 最も重要なことは、バックアップコピーが安全な場所に保存され、必要なデータをいつでも復元できることです。



物理的なDCがある場合



正直に言って、あなたが時代に遅れずについていくことを願っています、そしてあなたの会社ではドメインコントローラーは長い間仮想化されています。 そうでない場合は、定期的に更新し、比較的最新のバージョンのWindows Server（Windows Server 2008（R2）以降）で動作することを願っています。 （古いシステムでの作業のニュアンスに関する別の記事があります）。



したがって、Windows Server 2008 R2以降で実行されている1つ以上の物理ドメインコントローラーがあり、それらを保護する必要があります。 この場合、物理コンピューターとサーバーのデータを保護するために特別に設計されたソリューションであるVeeam Endpoint Backupが必要です。 Veeam Endpoint Backupは、必要なデータを物理マシンからコピーし、バックアップファイルに保存します。 事故が発生した場合、データを「ベアメタルに」復元するか、論理ドライブレベルでリカバリを実行できます。 さらに、Veeam Explorer for Microsoft Active Directoryを使用して個々のオブジェクトを復元できます。



次のことを行います。



  1. Veeam Endpoint Backup FREEをダウンロードし、インストーラーを目的のサーバーにコピーします。



  2. インストールウィザードを実行し、ライセンス契約に同意してプログラムをインストールします。



    注：インストールを自動的に完了するには、 適切な指示を使用してください。



  3. 目的のモードを選択して、バックアップタスクを作成します。 最も簡単で推奨される方法は、コンピューター全体をバックアップすることです。 ファイルレベルでバックアップモード（ファイルレベルモード）を使用して、オペレーティングシステムをコピーオブジェクトとして選択します。 この場合、プログラムは「ベアメタル」の復元に必要なすべてのファイルをコピーします。 Active DirectoryデータベースとSYSVOLディレクトリも保存されます。 詳細は、たとえばこの投稿で読むことができます。







    注： Veeam Backup＆Replicationが環境に既にインストールされており、既存のVeeamリポジトリを使用して物理マシンのバックアップコピーを保存する場合、Veeam Backup＆Replicationから直接再構成できます。 これを行うには、Ctrlキーを押しながら目的のリポジトリを右クリックし、開いたダイアログで目的のオプションを選択してリポジトリへのアクセスを許可します。 必要に応じて、このリポジトリに保存されているバックアップの暗号化を選択して、そこで暗号化を有効にします。







  4. タスクを実行し、エラーなしで成功したことを確認します。





それだけです。バックアップが完了し、ドメインコントローラーが保護されます。 リポジトリに移動して、目的のバックアップまたはバックアップチェーンを見つけます。







Veeam Backup＆Replicationリポジトリをバックアップのターゲットストレージとして設定した場合、新しく作成されたバックアップは、インフラストラクチャパネルの[バックアップ]> [ディスク]に表示され、[エンドポイントバックアップ]をポイントします。







結論の代わりに



もちろん、成功したバックアップは良いスタートですが、すべてではありません。 明らかに、データを復元できない場合、バックアップは価値がありません。 したがって、次の記事では、ドメインコントローラーの復元、Microsoft独自のツールとVeeam Explorer for Active Directoryを使用した個々の削除および変更されたオブジェクトの復元など、さまざまなActive Directoryの回復シナリオについて説明します。
