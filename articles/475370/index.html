is a timesb+c Therefore, at first it seems that this operation is only useful as a microoptimization: one instruction instead of two. However fma

It has a special property - it rounds off only once.



In the usual a timesb+c first calculated a timesb , and then this value, which in the general case cannot be represented in floating point format, is rounded to the nearest float. Then to this rounded value is added c , and this result is again rounded to the nearest float. The second FMA calculates the difference between the works using the FMA, performing rounding only at the very end. Consequently, it is resistant to catastrophic reduction, but it needs to work with a rounded value. c timesd . The return
