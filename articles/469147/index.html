Zimbra Open-Source Edition and automatic signature in letters

Automatic signature in emails is perhaps one of the most commonly used features in a business. A signature that can be set up once can not only permanently increase the efficiency of employees and increase the level of sales, but in some cases increase the level of information security of the company and even avoid lawsuits. For example, charitable organizations often add information about various ways of making their donation to the automatic signature, which helps to constantly increase the amount collected. In addition, signing an email is a great way to promote a corporate blog or website. You can also place various confidentiality warnings in the signature of an email, and commercial banks, for example, often remind you in letters that they never require account data from their customers. Zimbra Open-Source Edition supports the ability to create different signatures for different domains. This is very convenient for SaaS providers who can host hundreds or even thousands of different domains on the same Zimbra OSE infrastructure. However, in order to create signatures for domains, the administrator must first make sure that Zimbra OSE includes support for global email signatures as such. This is done using the zmprov mcf zimbraDomainMandatoryMailSignatureEnabled TRUE command . Once this is done, you can begin to configure domain signatures. For example, create a simple text signature for the Company.ru domain.



Signature text is written to LDAP using the zimbraAmavisDomainDisclaimerText and zimbraAmavisDomainDisclaimerHTML commands . Thanks to these commands, you can add text and HTML signatures to emails, respectively. For example, using the command zmprov md Company.ru zimbraAmavisDomainDisclaimerText ‚ÄúNever print the text of this message on paper in order to save as many trees as possible and show your concern for the environment‚Äù we will create a simple text signature that will be brief and memorable, and also allows you to once again recall the importance of environmental problems for the company. If you create a signature in HTML format, the administrator has the opportunity to add various decorations and formatting to the signature text.



After the signature is added to LDAP, you can activate it by making changes to the MTA settings. If your server has one MTA, you will need to execute the command ./libexec/zmaltermimeconfig -e Company.ru on it. In the event that your Zimbra OSE infrastructure has several MTA servers, on the first you will need to enter the command ./libexec/zmaltermimeconfig -e Company.ru , and on the other servers limit yourself to entering the command ./libexec/zmaltermimeconfig .

If you want to disable the signature on the domain, you can use the command ./libexec/zmaltermimeconfig -d Company.ru . As in the previous case, you need to run it on the MTA server, and if there are several of them in your infrastructure, you must enter the ./libexec/zmaltermimeconfig command on all the others.



Also, Zimbra OSE administrators often have the task of disabling signatures in internal letters, that is, those that users of the same domain send to each other. This can be achieved by executing the zimbraAmavisOutboundDisclaimersOnly TRUE command. By default, this feature is disabled.



Thus, as we have seen, Zimbra OSE provides the administrator with a flexible and convenient toolkit for creating and managing automatic email signatures.
