現在のKSWは、ビジネスエンジェルとベンチャーファンドの利益がまったく異なる面にあることを示しています。 教育プロジェクト、妊婦向けアプリケーション、イスラム教徒向けサービス、スタートアップヘルプデスク、企業ドライバー向けサービス、配達サービス向けアグリゲータープロジェクトのチームは、月桂樹の花輪の背後にあるステージに上がりました。



偽りの慎みがなければ、投資家の手から切望された小切手を受け取った6つのプロジェクトのうち4つが私たちのビジネスインキュベーターの居住者であると言います。 それらとの良好な隣人関係を使用して、私たちはお金を持っている人の注意を引くアプリケーションのフードの下を見て、浮き沈みといくつかの技術的な詳細でプロジェクトの簡単な概要をまとめました。



最大のジャックポット（80万ドル）を受け取ったスタートアップから始めましょう-Halal Guide 。

Halalguide.meはハラールサービス市場へのガイドで、さまざまなサービスで構成されています：Halaleda.me、HalalSkidka.ru、HalalTrading.ru、HalalWork.ru、Namaz-time.ru。

ハラールガイドのチームの努力により、駐車場の有無、ハラール適合証明書、チェックサイズ、評価、距離などに応じて施設を選択できるフィルターが開発されました。







すべてのHalalGuideプロジェクトサイトの主なトラフィックは、SEO検索からです。

サイトに加えて、iOSおよびAndroid上のアプリケーションがあります。

したがって、プロジェクトはASO + SEOの助けを借りて推進されます。SEOリクエストはサイトへのトラフィックを促進します-ユーザーはアプリからアプリに移動します-アプリからHalalGuideコンプレックスの他のサービスに移動します。

ASO最適化は、Anatoly Sharifullin（AppFollow）と共同で開発されました。

Python、データベースで自己記述されたCMS-PostgreSQL。



次のプロジェクトは、Habr- Hexletのオープンスペースでよく知られています。 これは、実際の環境でプログラミングをリモートで学習するためのプラットフォームです。つまり、シミュレーターはありません。各レッスンで、ユーザーはネットワークに接続された本格的なマシンで作業します。

プラットフォームはAmazon Web Services（AWS）で実行され、当初チームはAWSインフラストラクチャに強く結び付くつもりはありませんでしたが、時間の経過とともにそれを気に入っており、現在ではより多くのサービスを使用しています：バランサー、クラウドフロント-cdn、ses-メール、sqs-キュー、 route53-dns、s3-ファイルストレージ。 それにもかかわらず、彼らのスキームは、他のインフラストラクチャへの移動を非常に簡単にします。 これは、プラットフォーム全体の基盤であるDockerのおかげで可能になりました。

バックエンドのほぼ全体がRailsで記述されています。 RDS（リレーショナルデータベースサービス）では、PostgreSQL（メインベース）とRedisが回転しています。

サービスのスキーム全体については、チームのブログで詳しく説明しています。

さらに、連中はhabrogiksに素敵なボーナスがあります-これはプログラマー向けのゲームです→ habrahabr.ru/company/hexlet/blog/269613



Ruby on Railsの候補リストにあるもう1つのプラットフォームは、オンラインの妊娠サポートサービスであるmaminfo.ruです。 チームはパンフレット「Maternal Passport」で長い間オフラインでの作業に成功してきました。2006年にメンバーはメディアを登録し、2009年に特許番号73059を、2014年に彼らは受け取りました。 は、Maternal Passportの商標の所有者になり、現在はオンラインスペースを征服する予定です。

現在、スタートアップはiOSおよびAndroid用のモバイルアプリケーションを準備しており、サービスのオフラインコンポーネントとオンラインプラットフォームが互いに補完するようにプロジェクトを設計することを計画しています。



次のスタートアップであるTeleportは、無料の宅配便業者を集めて配達業務、レストラン、オンラインストア、小売店からの注文を完了するプラットフォームです。

その作成のアイデアについて簡単に教えてください。

当初、チームはタクシー用の自動化コンプレックスを開発し、フリートがUberの地域類似物を作成するのを手伝い、タクシーサービスを改善するためのコードを書き、巨大なプラットフォームを書きましたが、独自のサービスの開発の波に乗り遅れ、より自由な市場を探し始め、急行宅配便市場を見つけました。

最初は、サーバープログラムはPHPで記述され、phpDaemonフレームワークが使用されていました。 しかし、このスキームは作業中にそれ自体を証明しませんでした-phpDaemonフレームワークは絶えず作業にハングアップしたため、開発者はそれを放棄しました。

プラットフォームの2番目のバージョンはNodeJSで書き直されました。 新しいサーバーはフリーズの問題を修正しましたが、時間がたつにつれてプラットフォームはアプリケーションの機能の拡大に適合しなくなったため、開発者はNodeJSの3番目のバージョンを書き直し、プラットフォームアーキテクチャに注意を払いました。 500万件以上の注文。
