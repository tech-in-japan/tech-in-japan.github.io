Adwordsのダイレクトキャンペーンの最初の実際のコンバーター







広告キャンペーンをYandex.DirectからGoogle Adwordsに移行して変換するという問題は、コンテンツ広告に携わり始めたとき、すぐに起こりました。 そして、実践が示すように-私はそのようなニーズを持っているだけではありません。 なぜダイレクトからアドワーズまで、そしてその逆ではないのですか？ はい、すべてが簡単です-Yandex.DirectのExcelでキャンペーンを作成するのははるかに簡単です。







そのようなタスクのためのツールから現時点で私たちが持っているもの：







  1. AdwordsコミュニティのGoogleドキュメントスプレッドシートコンバーター 。
  2. カスタマイズされたマクロ（以前のマクロと機能が似ています）が含まれるExcelファイルリンク
  3. 代理店からの無料コンバーター
  4. そして、新しいサービスSynapse-transfer.ru



プレゼンテーションを簡単にするために、以下では、リストの最初の3つのツールの名前をそれぞれ「第1」、「第2」、「第3サービス」サービスと名付けます。







Adwordsは最近「Expanded Text Ad」という新しいテキスト広告フォーマットを導入しました。 標準との主な違い：2つのフィールド「タイトル」（それぞれ最大30文字）と1つの長いフィールド「説明」（最大80文字）。 また、2017年1月31日以降、AdWordsは標準のテキスト広告を作成および編集できなくなります。 広告を作成するときは、新しいフォーマットを使用する必要があります-拡張テキスト広告。 これは、「テーブル」コンバータの現在のバージョンでは大きな問題です-彼らはそれをサポートしていません。 つまり、2017年2月から、実際には上記のリストの最初のコンバーターも2番目のコンバーターも使用できなくなります。







しかし、これが彼らの唯一の問題ではありません。 彼らが新しいフォーマットのサポートを追加したとしても、かなり長い間彼らのために達してきた多くの問題があります：







  • コンバーターは、古い広告フォーマットでもうまく機能しません 。30文字を超えるヘッダーを処理せず、手動で書き換えることを提案します。 キーに基づいて生成された一意のヘッダーがある場合、多くの書き換えが必要になります。
  • それらは、キーワードタイプと不十分に一致するか、一致しません。 最良の場合、すべての文字がキーから削除され、前置詞に「+」が残ります。 最悪の場合、彼らはすべてをそのままにします。 また、Yandex.Directの通信の種類は、Adwordsの通信とは異なります。また、すべてを手動で編集する必要があります。
  • キャンペーン構造は失われます。 突然グループ内で複数のアナウンスがある場合は、手動での介入なしでは実行できません。
  • 広告表示オプションは手で携帯する必要があります。 5〜10個のキャンペーンがある場合でも、クイックリンクと絞り込みの追加には時間がかかることがあります。 そして、定期的にキャンペーンのスケジュールを変更した場合、おそらくすでにそれを取得しています。
  • キャンペーン設定は移行されません。 戦略、表示地域、入札単価などを設定するには、転送された各キャンペーンを設定する必要があります。 それは少し時間がかかりますが、それは疲れるルーチンです。
  • 除外キーワードは、どのレベルでも転送または処理されません。 ご存知のように、ネガティブフレーズなしでは、クリック率の高い優れたキャンペーンは得られません。 しかし、何らかの理由で、現在のコンバーター（リストの1〜3）は非常に悪いか、除外キーワードをまったく処理しません。 キャンペーン、広告グループ、クロスマイナスレベルにはマイナスがあります。 Adwordsとは異なり、少なくともキャンペーンレベルの除外キーワードは辞退後に再スケジュールする必要があります。


根拠がないように、最後に作成したキャンペーンの1つを取得し、使用可能なすべてのツールを介してそれを転送しようとしました。 （機密性を維持するために、キャンペーンは削減され、URLが変更され、テキストの一部が（元の文字数を維持しながら）置き換えられました。） キャンペーンファイルをダウンロードします 。







それの由来：



  • お知らせ。 キャンペーンの118個の広告の代わりに、最初と2番目のコンバーターのAd​​words Editorに読み込むと、それぞれ84と116を受け取りました。 どのグループがどの広告を得たか-それを把握してください。 3番目は番号を正しく処理しましたが、ヘッダーの長さに問題があります。 変換するとき、新しいAdwords広告フォーマットを選択し、ペンはすべての広告にユニバーサルな2番目のヘッダーを追加しました（これは完全に正しいわけではありませんが、後で詳しく説明します）。 また、Adwordsのタイトルの最初の部分では、Yandex.Direct広告のタイトルが変更なしで取得されました。 その結果、ヘッダーの一部を書き換える必要があります-それらは30文字より長くなりました。
  • キーワードのマッチタイプ。 最初と3番目のコンバーターは同様に機能しました。 「！」記号は削除されておらず、Adwordsは単語形式固定演算子を認識していないため、エディターはエラーを生成します。 Yandex.Directのキーの正確な対応は、Adwordsのフレーズに変わりました-これは間違っており、重複するキーワードが現れています。 2番目のコンバーターは、348個のうち209個のキーを取得しました（どのグループを失ったのか理解したくありません）。すべてのキーは大まかに対応しており、前置詞は「+」です
  • ネガティブな言葉。 1番目と2番目のコンバーターには除外キーワードはありません。 3番目の、キャンペーンの言葉を引いたものは転送されませんでしたが、クロスマイナス1の外観を維持しようとしました。 しかし、彼は言葉を断りませんでした、そして、これはケース、出産と否定的なキーワードの数のすべてのバリエーションのためのディスプレイです。 さらに、グループごとに2つの除外キーワードの制限があります。
  • 拡張機能はコンバーターを転送しませんでした。
  • キャンペーン設定は、どのコンバーターによっても転送されませんでした。






重要な注意-私はすべてのツールを「現状のまま」使用しましたが、変更せず、該当する場合は指示に従いました。 テーブルコンバーターにいくつかの変更を加えると、そのいくつかの痛みを克服できることを知っています。 ただし、このような変更は、特定のキャンペーンごとに個別に行う必要があります。







ご覧のとおり、結果は理想とはほど遠いものです。 もちろん、すべてのヘッダーが30文字以下であるコンバージョンキャンペーンも成功しています。 しかし、より高度なツールであるSynapse-transferコンバーターを簡単に使用できる場合、このような制限を設定する理由はありません。







私たちが提供するもの。



サービスSynapse-transfer.ru。 このサービスの目標は、キャンペーンの転送と変換のプロセスの自動化を最大化することです。 そして今、これは見つけることができるすべての最も先進的なソリューションです。 APIを介してYandex.DirectおよびAdwordsアカウントと直接連携します。 つまり、転送のどの段階でもテーブル、スクリプト、マクロ、デスクトップアプリケーションは表示されません。







上記のリストのコンバーターをテストするために使用されたのと同じキャンペーンが、Synapse転送サービスによって40秒で、単一のエラーなしで転送されます。 比較のため、Googleドキュメントの「最初の」コンバーターのスクリプトのみが2分以上実行されました。











現在、サービスのベータテストを実施しています。 すべての参加者は、1か月間無料で無制限に使用できます。



ベータテスト期間中の機能：







  1. 一度に1つのキャンペーンが移行されます。
  2. 検索キャンペーンのみが正しく変換されます。
  3. キャンペーン構造は保持されます -広告グループ、広告、キーはYandex.Directと同じ場所にあります。
  4. 広告のヘッダーとテキストは、Adwords形式 （拡張テキスト広告）に自動的に変換されます。
  5. キーワードマッチのタイプは 、Yandex.DirectおよびAdwordsでの作業に対する最大限の対応のロジックに従って変換されます。
  6. 除外キーワードは転送され、キャンペーンレベルと広告グループレベルで自動的に曲げられます 。 Yandex.Directの対応するグループからのマイナスは、Adwordsの各広告グループに転送されます。 サービスは、単語の形式が固定されていないすべての単語を自動的に拒否します（「！」記号付き）。 キャンペーン除外キーワードは、キャンペーンレベルで除外キーワードから作成され、読み込まれたキャンペーンに割り当てられます。
  7. 広告グループのレベルに転送してクロスマイナス1を保存するオプション （1つのグループ1キーの構造がある場合に使用可能）
  8. 説明と説明を含むクイックリンクが引き継がれます。 現在、広告グループレベルで記録されています。 Yandex.Directの各グループのキャンペーンに一意のクイックリンクがある場合でも、Adwordsでは同じ形式のままです。
  9. 転送時にUTMタグの値の一部を変更するオプション： yandex、yan、 yからgoogle、goog、 gへそれぞれ。
  10. キャンペーン設定は部分的に移行されます。


したがって、現在の機能を使用しても、1つのキャンペーンのみを転送する場合に10〜20分、最大数時間節約できます。 時間は、Yandex.Directの元のキャンペーンの複雑さと、手がどれだけいっぱいかによって異なります。 また、Adwordsのクロスマイナスのようなものは、手動で行うのは一般的に不適切です-単調なCtrl + C / Ctrl + Vの数時間自分自身を運命づけます







Synapse転送サービスを使用してキャンペーンの見出し、キーワード、およびその他の要素を変換する際のニュアンスの詳細な説明は、個人アカウントの「指示」セクションで確認できます。







欠陥がないわけではありません。 ただし、これらは単にベータ版のコストです。







  1. 現在、一度に複数のキャンペーンを選択する方法はありません。
  2. サービスは変換中にエラーを表示しません。 しかし、その数を最小限に抑えようとしましたが、現時点では、コンバーターが「落ちる」または正しく機能しないキャンペーンはありません。
  3. AdWordsでのキャンペーンの読み込みを高速化します。 Yandex.Directのクロスマイナスを模倣するときに広告グループに記録される除外キーワードが多数あるため、1つのキャンペーンの読み込み時間は3〜5分になります。 現在、プロセスの高速化に取り組んでおり、時間の経過に伴うキャンペーンのバックグラウンドロードも追加します。 これにより、複数のログインからのキャンペーンの同時転送の問題が解消されます。


YANキャンペーンの転送と変換を追加し、キャンペーン設定を完全に転送し、ダイレクトキャンペーンとアドワーズキャンペーンの同期機能を追加する要求があり、話が早すぎることがたくさんあります。







サービスを使用する前に、最初の主要な質問の1つに回答します。

「 そして、それはどのくらい安全ですか？」 結局のところ、APIによってアカウントへのアクセスを許可することは非常に親密です。 」

この質問は、Google Adwords Russiaコミュニティで私に尋ねられ 、そこで詳細に答えました。 私はここで答えの短いバージョンを提供します：

サービスを通過するキャンペーンのデータは保存されません。 また、Yandex.Directログインから自分で選択したキャンペーンを除き、アカウントには何もアップロードしません。 また、すべての要素は「停止」のステータスで読み込まれ、ユーザーの知らない間に新しいキャンペーンは表示されません。

さらに、AdwordsはそのAPIへのアクセスを許可しません。 取得する前に、マルチレベルテストとGoogleテストを実施しました。

そして、少なくとも1人のユーザーが広告キャンペーンの漏洩や、そこにあるべきではないもののアカウントへのアップロードという形で迷惑行為を行った場合、彼らはこれを知り、サービスの評判は永遠に失われます。 そして、これは私たちにとって決して必要ではありません。







私たちは提案を受け付けています。サービスに関するアイデアや提案があれば、コメントやVKのグループで自由に表現してください。
