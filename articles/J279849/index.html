NetApp AltaVault：クラウド内のデータのバックアップとアーカイブ

2014年10月、 NetAppはRiverbed TechnologyからSteelStore製品ラインを買収しました。 この製品は、そのシンプルさと、ほぼすべての可能なバックアップ製品からバックアップを収集する機能（フォールドバックアップの対象となる機能）、およびこのデータをクラウドに保存する機能により、企業環境で長い間人気がありました。 この製品はNetAppのポートフォリオを完全に補完し、ハイブリッドクラウド、プライベートクラウド、パブリッククラウドの将来に対する同社のビジョンに適合しています。 買収後、この製品には新しい名前のAltaVaultといくつかのイノベーションが追加されました。これらについては以下で説明します。





最初のイノベーションは、製品が「鉄に置き換わった」ことで、現在はFASとして知られるONTAPと同じプラットフォームに移行し、それに応じて同じディスクシェルフが使用されています。 これは非常に論理的なステップでした。同社は最近デバッグを完了し、PCI Express Gen 3に基づいて更新されたFAS8000プラットフォームをリリースしました。その間に多くの作業が行われ、重要なことに、PCIe genアーキテクチャ自体のパフォーマンスに関連するハードウェアバグが発見され、修正されました3.さらに、複数のプラットフォームを作成しない方が経済的に実行可能です。 つまり、FASおよびAVA製品（AltaVault）のプラットフォームは同じであり、混乱しないように、シャーシの右上隅に「AVA」と刻印された小さな紫色のアイコンが表示され、これらの製品の画像にコントローラーが付いています。



NetAppは、長い間、すべての製品を深く統合する会社としての地位を確立しており、AltaVaultプラットフォームも例外ではありません。 2番目の主要な革新は、オブジェクトデータウェアハウスとしてのNetApp StorageGrid Web Scaleのサポートの追加です。 これにより、パブリックだけでなくプライベートクラウドでもバックアップスキームを構築できました。 おそらく近い将来、他のNetApp製品とのより深い統合のために、他の機能が追加されるでしょう。



製品リリースフォーム



この製品は物理デバイスとして利用可能で、AVA400（48-288TBローカルストレージ）およびAVA800（72-576TBローカルストレージ）モデル、および仮想アプライアンスとして提供されるAVA-v仮想マシン（2-32TBローカルストレージ）で表されます。ハイパーバイザーHyper-V（2012 R2）、ESXi（vSphere / ESXi 5.5または6.0：VMFS-5またはRDMボリューム）またはKVM（RedHat Enterprise Linux OS 6.5以降）。 2TB、8TB、16TB、および32TBのバージョンがあります。 AVA-s仮想アプライアンスはAmazon（EC2 Marketplace）およびAzureパブリッククラウドでも利用できます。これは、パブリッククラウドにバックアップサイトを展開する場合に非常に便利です。 ここでのローカルストレージは、圧縮を含まないディスク上の物理スペースです。 つまり バックアップが非常にうまく圧縮される場合、物理ストレージよりもはるかに多くのデータをローカルストレージに格納できます。 公式ウェブサイトの詳細。

AVA400およびAVA800


AVA400およびAVA800には、802.3adモードを使用して集約できる10GBインターフェイスがあります。 VLANタギングがサポートされています。 AVA400は4TBドライブのディスクシェルフのみを接続でき、AVA800は6TBドライブのディスクシェルフのみを接続できます。



作業装置



AltaVaultは、バックアップソフトウェアがバックアップを送信する通常のCIFS / NFSファイル共有です。 NFS v3 / v4、SMB v2 / v3（SMBマルチチャネル接続をサポート）およびVeritas OpenStorage API（Veritas OST：NetBackup 6.5および7.7）がサポートされています。 このシンプルさにより、この製品は、IBM Tivoli、EMC NetWorker、HP Data Protector、Dell vRangerなどの競合ベンダーを含む幅広いバックアップソフトウェアと統合されています。 AVAに到着するデータはオンラインで重複排除されます（可変ブロック長を使用して、データ圧縮の度合いを高めることができます）。 データの最新コピーはすべてローカルに保存され、古いデータはプライベートまたはパブリッククラウドに送信されます。 暗号化前のすべてのデータは、暗号化証明書を使用して暗号化されます。



動作モード：コールド＆ホットバックアップ


操作には、コールドバックアップまたはコールドストレージモード（ローカルキャッシュは機能しません）とホットバックアップ（ローカルキャッシュが有効）の2つのモードがあります。

コールドバックアップモードでは、AVAに入るデータは重複排除され、暗号化され、すぐにクラウドに送信されます。ローカルディスクには、バックアップに関するメタ情報の非常に小さなセットのみが格納されます。したがって、バックアップをクラウドに注ぐには、適切な通信チャネルが必要です。

キャッシュモードでは、AVAが受信したデータは重複排除され、暗号化されてからローカルディスクに転送され、可能な限りディスクからクラウドに送信されます。これにより、最新のバックアップをはるかに高速に復元し、クラウドとの通信チャネルの負荷を軽減できますコールドバックアップモード。 つまり キャッシュモードでは、最新のバックアップがローカルドライブとクラウドの両方に保存されます。 キャッシュからの古いデータは、クラウド内にコピーがあるため、単純に削除される新しいデータによって徐々に置き換えられます。



クラウドバックアップサイズ


原則として、クラウドにアップロードできるアーカイブのサイズは、AVAのディスク上のローカルキャッシュの5倍（バックアップモード）です。 コールドストレージモードでは、ローカルキャッシュはメタ情報の保存にのみ使用されます。これにより、クラウドに保存されているデータ量をローカルキャッシュに対して5倍以上増やすことができます。 詳細



スナップミラー


SnapMirrorは、スナップショットベースのレプリケーションタイプです 。 SteelStore NetAppは、購入後、主力製品であるONTAPとの統合を積極的に取り入れました。 ONTAPから直接AVAにデータを複製する機能は論理的なソリューションになるため、バックアップ中に毎回完全なデータセットを転送するのではなく、データの変更のみを転送するため、バックアップの効率が大幅に向上します。 ONTAPからAVAへの直接のレプリケーションも非常に便利です。これは、ホストからバックアップファイルを転送するために必要だった中間プロキシサーバー（ホストCPUとそのネットワークポートがロードされていない）が不要になるためです。 （AVA、クラウド）。 スナップショットは次のようにレプリケートされます。最初に「基本データセット」が送信され、次に増分がロールされます。 このような各スナップショットは（既に複製された基本データセットと古いスナップショットと共に）完全なデータセットであり、増分とは異なり、何かを復元するために「接着」操作を実行する必要はありません。 この構成では、AltaVaultはOS ONTAPでこれらのレプリカをターゲットにできます。これにより、AltaVaultスペースリソースをより経済的に使用できるようになり、ネットワーク経由で転送するデータ量が少なくなり、FullBackup操作を実行するバックアップストレージがロードされます。 NetApp ONTAPのスナップショットの詳細をご覧ください。 SnapMirrorは、 DataFabricコンセプトの一部として、新しいAVA 4.3ファームウェアで利用可能です。

パブリッククラウドとプライベートクラウド



AltaVaultは、内蔵ドライブでのみバックアップストレージモードで動作することはできません。 システムには、プライベートクラウドまたはデータのオブジェクトストレージをサポートするパブリッククラウドプロバイダーにインストールされた特殊なソフトウェアが必要です。 OpenStack SwiftやEMC Atmos、IBM Cleversafe、Dell NetVaultなどの競合他社を含む、プライベートデータストレージ用の公共プロバイダーとソフトウェアの非常に幅広いリストがサポートされています。 AltaVaultを使用すると、クラウドサービスの1つに接続できます。 また、1つから2番目のクラウドサービスへの移行モードがサポートされています。

暗号化



クラウドに送信されるデータは、AES 256ビットアルゴリズムを使用した暗号化証明書を使用して既に圧縮および暗号化されています。 これにより、クラウドプロバイダーのスタッフがバックアップを作成する場合でも、バックアップの場所を気にする必要がなくなります。 KMIPは、キー交換、ユーザーアクセス用のRBAC、AD / LDAP認証、TACACS、RADIUS、Kerberos付きNFSv4でサポートされています。 AVAはFIPS 140-2 Lavel 1セキュリティレベルに準拠しています。





DRおよび復元設定



AltaVaultを使用すると、新しいシステムで構成全体を復元するために使用できる構成ファイルをアップロードできます。再配置機能により、クラウドから最新のデータが新しいシステムのローカルディスクにプルされます。
