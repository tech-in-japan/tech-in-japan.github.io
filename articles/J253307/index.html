VMware 6の初見

「Habrahabr」の読者に、「 初見：VMware vSphere 6の優位性」という記事の翻訳を提供します。



最近では、VMwareは長い間、サーバー仮想化で決定的なリーダーシップを発揮しており、他の製品とは比較にならないほどのカーネル機能を提供しています。 近年、仮想化の競争は激化しており、競合他社はVMware機能に近づいており、同社は他と差別化する機会があまりありませんでした。



競争はどんどん進む可能性があり、VMwareはそのように主要なラインに参入しないかもしれません。 いくつかの主要な機能が改善されました。つまり、以前は個別にしか利用できなかったバックアップ機能とリカバリ機能を組み合わせることで、vSphere 6はvSphereラインに追加する価値があります。 これは、vMotionのリモート使用など、バージョン6のほとんどの新機能がクライアントを大いに惹きつけ、その結果、vSphereの売上が増加することを示唆しています。



vSphere 6の大きな変更



vSphere 6の大きな変更は、リソース制限の拡大、vMotionの機能、LinuxベースのvCenter Server Applianceモジュールのより完全なバージョン、メモリのアンロード、およびWebクライアントの革新を中心に展開されます。 VMwareはすべてに加えて、vSphere 6に特別なテクノロジーを組み込んでいます。vCenterDirectorコンテンツライブラリは、ISOイメージ、テンプレート、スクリプト、OVFファイル、およびその他の要素を保存し、複数のvCenterサーバーを通じて自動的に配布するために使用されます。 さらに、バックアップとリカバリによる高度なデータ保護機能が含まれていました。



VMware vSphere 6では、プリインストールされているフォールトトレランスが改善されています。 このテクノロジーにより、1つの仮想マシンを複数の物理サーバーに同時に存在させることができます。 アクティブなシステムのイメージの1つが停止すると、2番目のイメージをすぐに起動できます。 このテクノロジーを使用しなくても、仮想マシンは別のホストで自動的に再起動できますが、この場合、システム障害を判別して新しいマシンをロードするには時間がかかります。 フォールトトレランスを使用すると、この手順をスキップできます。



vSphereの以前のバージョンでは、フォールトトレランスは、ホストごとに1つの仮想マシンと4つのフォールトトレラント仮想マシンに対して1つのvCPUのみをサポートしていました。 vSphere 6では、これらの制限は、仮想マシンごとに4つのvCPUと、ホストごとに8つのvCPUまたは4つの仮想マシンに拡張されています。





新しいvSphere Webクライアントのスクリーンショットは、Windowsウィンドウの外観に似ています。 頻繁なタスクの履歴がこのウィンドウの下部にあることに注意してください。



VMotionリモート管理



vMotionの改善は、地理的に分散した多数のデータセンターで作業する人にとって最も適切な改善となります。 仮想マシンの長距離移行に先行するvSphere 6は、いくつかの問題により複雑であり、移行を成功させるには大きな帯域幅と小さな遅延が必要でした。 vSphere 6の新バージョンでは、ネットワーク接続の要件が低くなり、vMotionは遅延が100ミリ秒以下の接続に使用でき、実行されるvMotionごとに250メガビットのチャネルも必要になります。



また、vCenterサーバー間で仮想マシンを移行できます。適切なインフラストラクチャを使用すると、ペアのストレージを使用せずにvMotionを移行できます。 ただし、各ネットワークの仮想マシン間の適切な通信を確保するために、主に各側の正しいネットワーク図の形で、機能の拡張にはいくつかの制限があります。



vSphere 6のESXi 6.0ハイパーバイザーは、クラスターあたり最大64の物理ホストをサポートでき、この数を32から増やします。各イメージは、最大480のCPU、12 TBのRAM、および1000の仮想マシンもサポートできます。 各仮想マシンは128個のvCPUで動作し、4 TB RAMを使用してvNUMAメモリをホットアドすることができます。



vSphere環境管理環境の改善（VMware vCenter Server）



管理側では、vCenter Server ApplianceがWindowsのコピーレベルで完全に機能するようになりました。 以前は、LinuxベースのvCenter Server Applianceを実行してESXiホストを管理する必要がありましたが、WindowsベースのvCenter Serverのより高度な機能の一部（特にアップデートマネージャー）は使用できませんでした。 vSphere 6の場合、アプリケーションはWindowsベースのアプリケーションが実行できるすべての可能なタスクを実行できます。



WindowsでvCenter Serverを実行する人は、インストール手順が大幅に簡略化されていることに気付くかもしれませんが、インストールを完了する前にいくつかの手順を実行する必要があります。 vCenter Serverによって実行されるすべてのインストール手順は、SSO、ライセンス、および証明書管理を含む新しいコントローラーサービスプラットフォームを含む1つのインストールプログラムで形成されるようになりました。 vCenter Serverは、すべてのコンポーネントと共に同じシステムにデプロイすることも、プラットフォームコントローラーを使用して複数のシステムに分散してインストールすることもできます。



vCenter Server for WinowsとvCenter Server Applianceはどちらも、デフォルトでローカルPostgreSQLデータベースを使用しますが、Microsoft SQLおよびOracleデータベースインジェクションもサポートされています。 PostgreSQLへの切り替えは、以前のバージョンのvSphereでローカルデータベースを操作する人にとって非常に重要です。これは、以前のMicrosoftデータベースからの制限がなくなったためです。 したがって、ローカルデータベースは、最大1000台のホストと1万台の仮想マシンをサポートできます。





vSphere Webクライアントを右クリックすると、コンテキストメニューが開きます。このメニューはまだ動作は遅いですが、以前よりも高速です。



Web UIの改善



vSphere Webクライアントの最初のバージョンは遅く、不完全で、Windowsクライアントほど透明ではなく、ほとんどのユーザーはそれを使用することを拒否しました。 vSphere 5.5では、Webクライアントが改善されていますが、これまでのところWindowsクライアントには到達していません。 vSphere 6では、新しい使いやすさと高速なWebインターフェイスにより、よりサポートされているブラウザーとオペレーティングシステムを使用できるという点で、Webクライアントがより興味深いものになっています。 VMへのコンソールアクセスなどの重要なユーティリティを使用できるクライアント統合ツールが、Mac OS Xを含むほとんどのプラットフォームで利用できるようになりました。



Webインターフェースで作業するユーザーは、Windowsのクライアントと非常に似ていることに気付くでしょう。画面の下部にある最近のタスクがインフラストラクチャで実行されたものを示すパネルを含む。 さらに、右クリックでコンテキストメニューにアクセスできます。作成が高速になり、Webクライアントを介したすべてのナビゲーションが、以前のバージョンのWebクライアントよりもうまく機能します。



Webクライアントの成功はVMwareにとって重要です。 企業は、管理用の個別にインストールされたクライアントのサポートが一部のソフトウェア実装で間もなく廃止されることを非常に懸念しており、現在、個別のクライアントを使用するとバージョン5.0のvSphere機能が制限されることに動揺しています。 vSphere 5.5の機能と機能は、Windowsクライアントでは使用できなくなりました。

VMwareの仮想ボリューム



VMwareは、Virtual Volumesと呼ばれるvSphere 6を備えた新しいストレージ統合システムを導入しています。 これは、仮想ディスクストレージ操作を管理するためのSANおよびNASデバイスとの最も緊密な統合です。 仮想ボリュームは、仮想ホストを使用して多数のLUNまたはボリュームをシャットダウンする必要がなく、互換性のあるアレイのストレージ管理に関連する操作を仮想ディスクレベルでパーティショニングでオフロードする必要がないように設計されています。



統合には、vSphereポリシーベースのストレージ管理が含まれます。これは、VMware APIを使用してストレージアレイと通信し、vSphereユーザーインターフェイスを介して仮想マシンとストレージに接続します。 したがって、vCenterを介してポリシーを作成し、仮想マシンに適用できますが、機能は最初はアレイによってサポートされます。



VMwareには、vSphere Essential Plus以上のvSphere 6を備えたvSphereデータ保護が含まれています。これは、以前はvSphere Data Protection Advancedと呼ばれ、別の機能であったバックアップおよび復元ツールです。 このツールは、Microsoft SQL Server、メールレベルまでのMicrosoft Exchange、およびその他の一般的なデータベースとアプリケーションのサポートを含む、プログラム指向の仮想マシンのバックアップとリカバリをサポートするために使用できます。

vSphere 5.5でアップグレードする



vSphere 6以降、VMwareは一連の初期機能を提供するようになりましたが、これらの機能は個別の製品としてより関連するようになりました。 すでにインストールされている多くの機能が表示され、インストールプロセスが合理化されました。 Webクライアントは、最初から個々のクライアントを使用していたユーザーにとっては依然として不満の対象になる可能性がありますが、Webクライアントは以前の実行よりもはるかに優れています。



vMotionおよびその他の双方向機能の改善は、基本的な機能を維持するために接続された多数のデータセンターに大きな帯域幅を使用する必要があるため制限されます。 しかし、VMwareは帯域幅と遅延の要件を削減します。つまり、提供されるシステム機能の実行可能性を意味します。



VMwareがサーバー仮想化で引き続きトップの地位を維持していることは間違いありませんが、将来最高のベンダーを設定することで、コンバージェンスは継続し、競合ソリューションはより完全になり続けます。 これはすべて、近い将来、機能のクロスオーバーとライセンスの簡素化が見られるという事実につながります。 また、vSphere 6は、この社会のクリームの中でそれ自体をサポートしています。
