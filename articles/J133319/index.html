ADD-3カンファレンスはスピーカーを招集

3回目のApplication Developer Daysカンファレンスは、5月11〜12日にモスクワで開催されます。 （延期日）



ADDは、プログラマーのためにプログラマーが作成した会議です。 プロジェクトマネージャー、テスター、スタートアップ、アプリケーション開発者向けではありません。 コードを直接書く人は、アプリケーションのアーキテクチャを熟考し、新しい言語とライブラリを使用して生産性を高めようとします。



過去の会議の参加者は、これらが単なる言葉ではないことを知っています。 レポートは本当に便利で、コミュニケーションはとても楽しいです。 しかし、レポートを作成することによってのみ、会議からすべてを本当に得ることができます。



現在、プログラムはまだ結成されているため、会議で講演する絶好の機会があります。 アブストラクトをプログラム委員会に提出するのが早ければ早いほど、できるだけ時間をかけて作成する必要があります。 プログラム委員会のメンバーのアドバイスは、これにおいて非常に貴重です。



すべてのレポートはビデオに記録され、会議の直後にオンラインで公開されます。 これにより、何千人もの人々が良いパフォーマンスを見ることができます。 これは、優れたレポートを作成するもう1つの理由です。



私たちの会議はプログラマ向けです。 そのため、レポートには、プログラミング言語、アーキテクチャー手法、ライブラリー、開発環境、その他のツールなど、プログラマーが関心を持っているものに関するものを含める必要があります。 最後に、完成したプロジェクトに関する興味深いレポート。



要約を送信するには、 スピーカーページのフォームを使用します。



プログラム委員会には以下が含まれます 。

  • Stas Fomin-私たちの時代の最高のスピーカーの1人であるPCの会長、
  • Dmitry Zavalishin-紹介する必要はありませんが、
  • アンドレイ・アクセノフ-Sphinx検索エンジンの作者、
  • レフ・ヴァルキン-ジャーナル「Functional Programming Practice」の創設者、
  • Evgeny Kirpichevはクラスターコンピューティングの真のエキスパートであり、
  • セルゲイ・クドリャショフ-UXデザインの第一人者、
  • アンドレイ・マヨロフ-自分自身に書くのをためらう:)。
結論として-困難について。 Stas Fominによって書かれたスピーカーへのメモを読んでいない場合、これを行う必要があります。 そこで提示されたアドバイスは、非常に貴重です。
