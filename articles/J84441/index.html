Windows Phone 7インターフェイス：MicrosoftがAppleを打ち負かす





クパチーノ、あなたは確かに私を許しますが、マイクロソフトはあなたを打ち負かしました。 Windows Phone 7は、将来のiPhoneのように見えます。 ユーザーインターフェイス（UI）はAppleのハードウェアデザインと同じくらいシンプルでエレガントですが、iPhoneのインターフェイスは装飾されたPalm Pilot（ 画像Palm 3 ）に似ています。



これは、Apple製品よりもユーザーエクスペリエンスが向上するという意味ではありません。 なぜなら インターフェイスの2つの概念、情報指向と機能指向は、互いに大きく異なります。 そして、それらのそれぞれは、ユーザーが以前に扱っていたものとは非常に異なります。



Appleは、iPhoneを使用して、あらゆる年齢の人々に適した非常にシンプルなインターフェイスを作成し、行動を正しい方向に向けています。「これは、クリックしたアイコンに応じて、さまざまな機能に適応し、正しい情報にアクセスできるデバイスです」



このシンプルなアイデアは信じられないほどの成功です。 したがって、Appleは携帯電話だけでなく、コンピューティングの将来についてもこのモデルに依存しています。 同じ理由で、Android、Palms、Blackberrieもこれに従います。



ゼロから



マイクロソフトにはまったく異なるアプローチがあります。 Windows Phone 7チームは、Androidやcoのような別の「類似の」携帯電話を作成する代わりに、携帯電話の可能性についてのビジョンを思いつきました。 開発プロセス中に、彼らは情報指向がユーザーインタラクションの決定要因となる美しいインターフェイスを作成しました。 特定の機能を持つアプリケーションではなく、それ自体に情報があります。 実際、情報はインターフェースです。



最終フォームでは、情報はユーザーのタスクに焦点を合わせた「センター」（ハブ）と呼ばれる領域に分割されます。 ホーム画面のライブタイル、「I」（ユーザープロファイル）、「人」（連絡先）、「写真とビデオ」（メディア）、「音楽」、「ゲーム」、および常に利用可能な検索を通じて利用可能認識のパノラマストリーム、ハブは文字通り、ローカルおよび「クラウド」（インターネット）からの多くのデータソースを織り込みます。







代わりに、電話帳に移動して番号を選択して電話をかけ、別のアプリケーションを開いて彼のtwitterを読み、別のアプリケーションを開いてVKontakteから更新を読み、別のアプリケーションを開いて彼のメールを読みます。もう1つ、写真を表示するために、Windows Phoneの「人」ハブ（連絡先）は、非常にシンプルかつ論理的な方法で、これらすべてのシームレスなビューを提供します。 iPhoneなどの機能指向のインターフェイスモデルでは、ユーザーが「電話をかけたい」と思うと、目的のアプリケーションを起動して連絡先を選択し、電話をかけることでデバイスを通話モードにします。 ユーザーが「Vanya Petrovの新機能」と思うと、VKontakteクライアント、twitter、メールなどの「モード」に電話をかけます。



一方、Microsoftは、画面の形で「センター」（ハブ）を整理し、単一のパノラマ画面の形でグループ化された情報列でつなぎ合わせます。これは、電話の画面よりも大きくても、指でスクロールできます。 そして、これらすべてをミニマリズムの美的インターフェースで補完し、見た目は魅力的で、オーバーロードではないエレガントなアニメーションを提供します。







他のアプリはどうですか？



直感的に、私たちのデジタルライフの中核を整理するマイクロソフトのアプローチが好きです。人々+ソーシャルサービス+マルチメディア+コミュニケーション-すべてがセンター（ハブ）に分割されています。 Appleのアプローチよりもこれが好きです-「これは電話、これはメールクライアント、これはブラウザ、これはiPodです」。 iPhoneまたはAndoroidのモデルよりも剛性が低く、より便利な認識を提供し、さまざまな視点からの情報をより迅速かつ明確に探索し、提供します。 それはより人道的に見え、これがまさにAppleとそのフォロワーが気にしていることです。



これは、機能指向モデルの方が悪いということですか？ 先ほど言ったように、事実ではありません。 特に、情報指向のパノラマは、同じiPhoneのユーザーが期待するすべての単純なタスクに適しているわけではありません。 そして、私が言うとき、すべての単純なタスク-私はAppleのアプリストアを埋めるプログラムの大群を意味します。 もちろん、マイクロソフトはハブで何でもできますが、新しいデバイスにアプリストアがなければ、Appleとの競争で再び負けてしまいます。



Microsoftにとって幸いなことに、Windows Phoneモデルは情報指向であるだけでなく、機能指向でもあります。 MicrosoftのWindows Phone部門の最高経営責任者であるJoe Belfiore氏によると、アプリケーションは必ずしもハブまたはパノラマインターフェイスに統合されるとは限りません。 開発者ツールが来月登場すると、これは今日iPhoneにすでに存在するアプリケーションの出現に貢献します。 言い換えると、Microsoftは、1つのアプローチ（モデルを構築するインターフェイス）が他のアプローチと同じくらい重要であることを理解しています。



彼らは、ハブがより良く、より素晴らしく、ほとんどの消費者が実際に望んでいるデジタルライフを提供し、貢献するためのより直感的な方法になることを望んでいます。 彼らが今日示したものを見ると、彼らは正しい軌道に乗っていると思います。 ただし、Zune HDと同様に、おそらく少し遅れています。
