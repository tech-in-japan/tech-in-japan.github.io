テザーなしの脱獄iOS 4.2.1。 取扱説明書



iPhone 3Gのユーザー、および古いbootromを搭載したiPod Touch 2GまたはiPhone 3GSのユーザーは、テザーなしの脱獄のために、公式サイトの Redsn0wの最新バージョンを使用することはできません。

今日、Dev-Teamは、ユーティリティRedsn0w 0.9.7b1 の最初のベータバージョンをリリースしました。これは、ファームウェア4.2.1で切り離されていない脱獄iOSデバイスを作成します。 このバージョンはまだ非常に未加工であり、いくつかの副作用があることをすぐに警告します。 第一に、Mac OSユーザーのみがテザーなしの脱獄を行うことができます。第二に、「エスケープ」操作の後、bluetoothが動作しなくなります。 。 特定のリスクを負う準備ができている場合は、猫をお願いします。



必要なもの：Mac OSベースのコンピューター、Redsn0wバージョン0.9.7b1、usbmuxdパッケージ、デバイスに対応するオリジナルファームウェア4.2.1、およびファームウェア4.2ベータ3の開発バージョン（ダウンロード先を確認できます）。 それでは始めましょう。



1.ファームウェア4.2.1がすでにインストールされている必要があります。たとえば、ユーティリティRedsn0w 0.9.6b6を使用して、通常のテザード脱獄を行う必要があります。

2. Cydiaに移動し、4.2b3 SHSHファイルがCydiaサーバーに保存されていることを確認します。 画面上のように、Cydiaのステータスに「4.2b3 SHSH」という碑文が表示されます。



これは、いつでも通常のファームウェア4.2.1にロールバックできるようにするために必要です。

3.これで、Redsn0w 0.9.7b1をダウンロードして 、iPhone / iPad / iPod Touchをコンピューターに接続できます 。

4.以来 これはベータ版です。usbmuxdパッケージをインストールする必要があります 。これは、ここからダウンロードできます 。 python-clientディレクトリでターミナルを実行し、引用符なしで次のコマンドを入力します。「chmod + x tcprelay.py」、「./ tcprelay.py -t 22：2222」

エラーが発生した場合は、元のファームウェアをフラッシュして再試行する必要があります。

5. Redsn0wを実行し、デバイスに適切な元のファームウェア4.2.1ファイルを選択します。





6.次のステップで、4.2.1ファームウェアの脱獄を解き、開発4.2ベータ3に変換する「脱獄モンテ」をマークします





7.最後の手順は、Redsn0wの指示に従ってDFUモードでデバイスを入力することです。 これで、自由に脱獄できます。



これは最初のベータ版であることを思い出してください。 将来のバージョンでは、Windowsサポートと組み込みのusbmuxdパッケージが表示されます。 twitter @ios_uaで私をフォローして、最新の最新情報を入手できます。
