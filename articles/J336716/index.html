PyQtにツールバーを作成し、データをExcelとHTMLにエクスポートします

前のパートでは、 SQLクエリを実行するためのモジュールと、これらのモジュールが起動されるシェルの作成について説明しました。 クエリの短い作業の後、明らかな疑問が発生します-画面の表示方法を除いて、選択の結果をどのように使用するのですか？



これを行うには、追加のエクスポートおよびデータツールのコピーを作成します。 Excel形式でファイルにエクスポートし、HTML形式でシステムバッファーにコピーします。



しかし、最初に、ツールバーをメインウィンドウに貼り付けます。







ツールバー



私たちのアプリケーションはシンプルで、普遍的で、拡張可能であるように設計されていることを思い出させてください。 ツールバーをユニバーサルで拡張可能にするために、構成ファイルでその定義を取り出し、ツールバーモジュールに明示的にインポートされていない外部モジュールに関数を配置します。 したがって、新しいボタンと機能の追加は、構成ファイルにそれらを書き込んで、プログラムディレクトリにモジュールを追加することになります。



Toolbar.pyファイル 
#!/usr/bin/python3 # -*- coding: utf-8 -*- import sys from PyQt5.QtCore import * from PyQt5.QtWidgets import * import importlib class ToolBar(QToolBar): def __init__(self, iniFile, parent=None): super(ToolBar, self).__init__(parent) ini = QSettings(iniFile, QSettings.IniFormat) ini.setIniCodec("utf-8") ini.beginGroup("Tools") for key in sorted(ini.childKeys()): v = ini.value(key) title = v[0] params = v[1:] a = self.addAction(title) a.params = params a.triggered.connect(self.execAction) ini.endGroup() def execAction(self): try: params = self.sender().params module = importlib.import_module(params[0]) if len(params) < 2: func = "run()" else: func = params[1] win = self.focusTaskWindow() exec("module.%s(win)" % func) except: print(str(sys.exc_info()[1])) return def focusTaskWindow(self): try: return QApplication.instance().focusedTaskWindow() except: return None if __name__ == '__main__': app = QApplication(sys.argv) ex = ToolBar("tools.ini") flags = Qt.Tool | Qt.WindowDoesNotAcceptFocus # | ex.windowFlags() ex.setWindowFlags(flags) ex.show() sys.exit(app.exec_())




Qtのツールバーには既製のQToolBarクラスがあり、そこから独自のToolBarを生成します。 これで、1つのツールバーで十分ですが、プログラムにいくつかのパネルを追加する可能性があります。 各パネルには独自のボタンセットを持つ独自の構成ファイルが必要なので、ツールバーを作成するときにファイル名をパラメーターとして渡します。

構成ファイルは、伝統的にIni形式でUTF-8エンコードされます。 



 class ToolBar(QToolBar): def __init__(self, iniFile, parent=None): super(ToolBar, self).__init__(parent) ini = QSettings(iniFile, QSettings.IniFormat) ini.setIniCodec("utf-8")

ボタン関数が1つのパラメーター、つまり現在の子ウィンドウを受け入れると判断します。 モジュールがそれで正確に行うことは、ボタンモジュールのタスクであり、ツールバーのタスクはその呼び出しのみに制限されます。



次のtools.iniファイルを作成します。 



 [Tools] 001=Export to Excel,exportview,"exportToExcel" 002=Copy as HTML,exportview,"copyAsHtml"


Pythonで、Iniファイルから定義を解析します。

すべてのボタンに割り当てられた実行メソッドは、目的のモジュールをインポートし、そこからボタンに割り当てられた機能を呼び出します。 ツールバーのインポートリストに各モジュールを登録しないために、importlibライブラリを使用します。 どのボタンが押され、どのQActionから信号が送信されたかを知るだけです-標準メソッドQObject.sender（）がこれを担当し、そこに格納されたパラメーターを取得し、モジュールで意図されていること（それが何であれ）を実行します。 



  def execAction(self): try: params = self.sender().params module = importlib.import_module(params[0]) func = params[1] win = self.focusTaskWindow() exec("module.%s(win)" % func) except: print(str(sys.exc_info()[1])) return

これらの関数は、クエリ結果を表示するためにモジュールで使用したQTableViewデータテーブルと連携します。 モジュールの独立性を維持するために、現在のウィンドウで現在選択されている（フォーカスされている）QTableViewコンポーネントか、現在のウィンドウの子の中の目的のクラスの最初のコンポーネントのいずれかを「オンザフライ」で決定します。

テーブルから、選択したセルのリストを取得します。 何も選択されていない場合は、すべてを強制的に選択します。 



  indexes = view.selectionModel().selectedIndexes() if len(indexes) == 0: indexes = view.selectAll() indexes = view.selectionModel().selectedIndexes() if len(indexes) == 0: return;


Qtでは、データの配列を直接取得するのではなく、モデルのインデックスを操作することを既にご存じでしょう。 QModelIndexインデックスは単純な構造で、特定のデータ位置（行（）行と列（）列、および階層内の親インデックスの親（））を示します。 インデックスを受け取ったら、data（）メソッドを使用してインデックスを取得できます。



モデル内の選択されたセルのインデックスのリストを取得しましたが、このリストのインデックスは、ユーザーが選択した順序に従い、行と列でソートされません。 リストではなく、辞書（位置→インデックス）および関連する行と列のソートされたリストを使用する方が便利です。 



 def sortedIndexes(indexes): d = ob() # - d.indexes = { (i.row(), i.column()):i for i in indexes } d.rows = sorted(list(set([ i[0] for i in d.indexes ]))) d.columns = sorted(list(set([ i[1] for i in d.indexes ]))) return d


インデックスリストにはほとんどランダムに配置されたセルが存在する可能性があるため、デフォルトではQTableViewで未接続のセルを選択できることを考慮することも価値があります。







したがって、d.rowsにはすべての使用済み行があり、d.columnsにはすべての使用済み列がありますが、それらの組み合わせは必ずしもd.indexesにありません。



より美しくするために、QTableViewに表示される列名のリストも必要です。 headerDataメソッドを使用してモデルからそれらを取得します。 



  headers = { col:model.headerData(col, Qt.Horizontal) for col in d.columns }

Excelにエクスポート



Excelファイルにエクスポートするために、 xlsxwriterパッケージを使用しました 。 通常どおり、pipを介してインストールされます。 



 pip3 install xlsxwriter


パッケージのドキュメントは非常に詳細で理解しやすく、例もあるため、ここでは詳しく説明しません。 一番下の行は、レコードが行と列の番号で指定されたセルを通過することです。 追加の書式設定が必要な場合は、スタイルを定義し、セルを書き込むときにスタイルを指定する必要があります。



ユーザーにエクスポート先のxlsxファイルの名前を尋ねます。Qtにはそのような機能があります。 PyQtでは、関数は選択されたファイル名と使用されたフィルターのリストを返します。 空の行のリストが返される場合、これはユーザーが選択を拒否したことを意味します。 



  fileName = QFileDialog.getSaveFileName(None, 'Save Excel file', title,'Excel files (*.xlsx)') if fileName == ('',''): return

Python、Qt、Excelで日付/時刻の理解が異なるため、QDateTimeのダンスが追加されました-まず、xlsxwriterパッケージはPython datetimeで動作しますが、QtからQDateTimeを使用する方法がわからないため、特別な関数toPyDateTimeで追加で変換する必要があります; 第二に、Excelは1900年1月1日の日付でのみ機能し、Excelのそれ以前のすべては単なる文字列です。

HTML形式でシステムバッファにコピーする



特にデータが少ない場合は特に、選択した個別のファイルが常に必要なわけではありません。特に、表形式でシステムクリップボード（クリップボード）にコピーし、Excel、Word、Webページエディターなどの適切な場所に貼り付ける方が便利です。 。



バッファを介して表形式データをコピーする最も一般的な方法は、通常のHTML形式を使用することです。 Windows、* nixおよびMacOSでは、バッファを操作する方法がまったく異なります（複数あることは言うまでもありません）。そのため、Qtが実装の詳細を隠すのは良いことです。



必要なのは、QMimeDataオブジェクトを作成し、setHtmlメソッドを介してHTMLマークアップを入力し、QApplicationからアクセス可能なシステムクリップボードに渡すことです。 



  mime = QMimeData() mime.setHtml(html) clipboard = QApplication.clipboard() clipboard.setMimeData(mime)
