JavaScriptフレームワークをGoogleでホストできるようになりました

AJAX Search APIブログは、昨日AJAX Libraries APIを発表しました。 肝心なのは、Googleが誰でもサーバーでホストされているJavaScriptフレームワークを使用できるようになったことです。 API自体は、jQuery、prototype、script.aculo.us、MooTools、dojoなど、最も一般的なJSライブラリのHTMLドキュメントに接続するための使いやすいメカニズムです。 リストは拡大する可能性があります。



必要なライブラリを接続するには2つの方法があります。 1つ目は、スクリプトのHTML要素を使用して簡単です。 



  <code> <script src = "http://ajax.googleapis.com/ajax/
 libs / prototype / 1.6.0.2 / prototype.js "> </ script> </ code>

2番目の方法はより柔軟です。 特別なJSメソッドgoogle.load()



を使用すると、ライブラリバージョンを正確にではなく、最初の桁で設定できます。 



  <code> <script src = "http://www.google.com/jsapi"> </ script>
 <スクリプト>
 google.load（ "prototype"、 "1.6"）;
 // ...
 </ script> </ code>




例で設定されている値「1.6」は、1.6ブランチのプロトタイプの最新バージョンが自動的に選択されることを意味します。 したがって、AJAX Libraries APIは、呼び出しスクリプトを変更せずにJSライブラリの自動更新を提供できます（もちろん、このような更新は目的のブランチに限定されます）。



AL APIを使用する主な利点は次のとおりです。



  • Googleにスクリプトをある程度保存すると、独自のWebサーバーの負荷が軽減されます。
  • 一部のリクエストが別のホストに転送され、 gz圧縮がサポートされるため、ページの読み込み速度が向上します。
  • 中央に格納されたフレームワークの大規模な使用により、ブラウザキャッシュがより効率的に使用されます。 あるWebアプリケーション用にダウンロードされたファイルは、同じフレームワークを使用して別のWebアプリケーションを操作するときに再利用できます。 これはサービスの潜在的な利点にすぎませんが、それでも快適です。
  • さらに、Googleのおかげで、JSフレームワークを使用してWebアプリケーションの配布を容易にすることができます。
