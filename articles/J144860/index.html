コンテストの結果「ストーリーを伝え、タブレットをプレゼントとして手に入れる」

同僚！



約束されたように、我々は競争の結果を要約しています。 参加してくれたみんなに感謝します。



主な賞品-NetWrix Active Directory監査Active Directory監査プログラム変更報告者-審査員によると、最高のストーリーで、このストーリーのToSshユーザーを獲得します。



今年、コマーシャル、2005年に起こりました。



むかしむかし、ユーザーフックを備えた300のオフィスがありました。 Windows Server 2003 + AD、ファイルサーバー、ターミナルサーバー、FreeBSDのメールサーバー-すべてが個別のハードウェア上にあります。



むかしむかし私が住んでいたのは、その当時最初のアウトソーシング会社の1つで働いていた若くて素朴なシベリアの若者です。



どうして起こったのかというと、そのオフィスにはドメインコントローラが1つしかありませんでした。 まあ、それは「それが起こらないかもしれない場合のために」お金の管理に同情でした。

ある「美しい」日に、ユーザーはPCにログインできませんでした。 「ドメインコントローラーは使用できません。」

電話に乗ることは私に落ちました。 サーバーの電源がオフになり、起動時に「ハードディスクまたはブートディスケットを挿入してください」と表示されました。 ああ、そうだ、襲撃もなかった、そうだ...そしてNTのバックアップはなかった...この不幸な事実は、オフィスの経営者に報告され、「質素さ」を非難した（若かった、しかし勇敢だった:)）。 「少なくとも1Cが機能する」ように作成し、サーバーをオフィスに持ち込みました。



落下の原因は、動作中に死にかけているハードドライブがひどく割れて振動することでした。 どんな種類のダンスと方法はわかりませんが、ハードドライブは完全にコピーされました。 しかし、次に何をすべきか？

私の上司は私に言った-自分でそれを取る。 そして、私は一度ADを上げ、次に10人のユーザーを上げました。



決定が来ました。

新しいディスクを取得し、その上に新しいWin2003サーバーをインストールしました。 AD、ユーザー資格情報、アクセス許可を復元する方法は？ 私は本当にゼロから始めたくありませんでした。

LiveCDから起動し、新しいディスクのWindows、Program Files、Doc＆Settinフォルダを取得して置き換えたコピーから。



コピーが終了したら、震えている交差した指で指を振って、再起動をクリックしました。 しかし...それはうまくいきました！ サーバーが上昇し、ADが実行されています。 サーバーはすべてのテストに合格しました。 朝、私はすでに自分の場所で働いていたので、眠りについた...



残念ながら、コメントに残された関連するストーリーのいずれも単一の肯定的な投票を持たないため、観客の共感賞-Samsung Galaxy Tabタブレットを授与することはできません。 これが起こったことを非常に残念に思います。



ToSsh様 、ギフトとしてプログラムを受け取るにはPMにご連絡ください。
