UbuntuのVirtualBoxでWindowsのプリインストールコピーを実行する

昨日そのような質問に戸惑った。 したがって、在庫あり：



-Pentium Dual-Core 1.86 GHzと2GBのメモリを搭載したラップトップ

-同じネジUbuntu 9.04とWindows XP SP3の上に立っています。 Windowsは、システムとストレージの2つのセクションに分かれています。

-ubuntaから既存のWindowsを実行したい



なんで？ 実際、適切なソフトウェアのセットを備えたソフトウェアが既に存在する場合、新しいWindowsを仮想ボックスに入れるのは単なるハックでした。





私はすぐにそれがゆっくりと動作することを言わなければなりません（実際に予想されるように）、しかし、ブラウザテストと十分であるべきいくつかのささいなことでさえ。 そして重要 -ライセンスを取得したWindowsでは、機器の大幅な変更により再アクティベーションを要求する必要があります。



グーグルで調べた後、この記事が見つかりました。



手順について簡単に説明します。

sudo vi /etc/group



またはsudo usermod -a -G disk <username>



を介してディスクグループにユーザーを追加しますsudo usermod -a -G disk <username>





-Windowsを再起動します。 再起動します-動作する権限を追加するには、再ログインが必要です。 再起動せずにsshを入力しようとしました-イメージファイルの作成が機能しませんでした（以下を参照）

-Windowsで新しいハードウェアプロファイルを追加し、再起動します

-Ubuntuで実行

VBoxManage internalcommands createrawvmdk -filename /pat/to/file.wmdk -rawdisk /dev/sda -register





次のようなものが得られるはずです。



VirtualBox Command Line Management Interface Version 2.1.4_OSE

(C) 2005-2009 Sun Microsystems, Inc.

All rights reserved.



RAW host disk access VMDK file /home/halien/build/Win/image_sda1.wmdk created successfully.







ここでは、Windowsパーティションではなく、メインブートローダーが配置されているドライブを指定することが重要です。 つまり、ブートローダーから起動し、OSを選択します。

-新しく作成したファイルを画像として選択して、仮想ボックスに新しいレコードを追加します

-メモリが少なすぎないようにしました（800 MBで試しました）

-起動、OSおよびハードウェアプロファイルの選択を忘れない

-Guest AdditionsをプリインストールされたWindowsに配置しないことが重要です-そうでない場合は、a）削除される前にロードを停止し、b）（確認されませんが、それ以上何もありません）



実際、それがすべてです。
