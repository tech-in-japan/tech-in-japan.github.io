EEE-PC 1000HEにWindows 7をインストールします



長い間ネットブックを買うことを考えました。 アイデアは昨年からローミングしていますが、彼の手はたった今届きました。 購入の主な理由は、ネットブックがオートバイの後部トランクに収まることです。これにより、フルサイズのラップトップでバックパックを運ぶ必要がなくなります。 長い市場調査の後、私の選択はASUS EEE 1000HEに落ちました。 Atom N280、Appleキーボード、8時間のバッテリー、キットに含まれるケースと布は、競合他社の議論を上回りました。



ネットブックにはチェコ語版のWindows XPが付属していますが、これは私には受け入れられません。 しかし、私はそれを面白くて便利だと思う人を知っています。 この投稿では、ネットブックにWindows 7をインストールする手順について説明します。 Windowsが動作する必要があることをすぐに強調します。 行きましょう。



  1. 公式Webサイトからiso形式でWindows7ディストリビューションをダウンロードします。 そこに鍵を入手することもできます。 32ビットバージョンをダウンロードしました。 ネットブックに64ビットは必要ありません。



  2. 付属のASUS更新ユーティリティを使用して、ネットブックBIOSを最新バージョンに更新します。



  3. フラッシュドライブ（最低4Gb）を準備します。 できれば空です。



  4. 次に、何らかの方法でisoイメージの内容をフラッシュドライブに転送する必要があります。 私はDaemonToolsを利用しました。これは私が誇りに思っている所有者です。 誰でも、 ここから完全に機能するトレイルをダウンロードできます。 DaemonToolsを使用して、イメージを仮想ドライブにマップし、その内容をフラッシュドライブにコピーしました。 注意！ コピーする前に、Windowsエクスプローラーで隠しファイルとシステムファイルを表示するオプションがオンになっていることを確認する必要があります。 DaemonToolsが気に入らない場合は、Nero Burning Romなどのこれを実行できるプログラムを使用してディスクにイメージを焼き付け、ディスクから既にフラッシュドライブにファイルを転送できます。 残りについては、WinRARを使用できます。WinRARはiso-imagesを開く方法も知っています。



  5. Windows XPがすでにネットブックにインストールされており、Windows 7をインストールする空きパーティションがある場合、起動可能なフラッシュドライブを作成する必要はありません。 ネットブックに挿入してインストーラーを実行するだけです。 この場合、次の項目をスキップできます。 私の状況はまさにそれでした。 EEEにはチェコ語XPが搭載されており、Windows 7をインストールした60 GBの無料パーティションがありました。



  6. Linuxがインストールされている場合、またはディスク上のパーティションをシャッフルする必要がある場合は、起動可能なフラッシュドライブが必要です。 これには、LiveUSBHelperの使用をお勧めします。 ここからダウンロードしてインストールします 。 起動時にプログラムにvb6stkit.dllが必要な場合は、 ここからダウンロードできます。 マイクロソフトのWebサイトからダウンロードできるMicrosoftの一連のactiveXコンポーネントがまだ必要でした。 プログラム自体が場所を教えてくれます。 インターフェイスは直感的で、詳細は説明しません。



  7. Windows 7インストーラーを実行しますこれを行うには、フラッシュドライブから起動するか、XPから実行します。 注意、フラッシュドライブから起動するには、BIOSで起動デバイスの優先順位を変更する必要があります。 ただし、フラッシュドライブからのブートは、デフォルトですでにディスクからのブートよりも上にありました。 インストールに問題はありませんでした。 Windowsの質問が少なくなり、すべてが見やすくなりました。 インストールには約30分40分かかりました。



  8. ほとんどすべてのドライバーがすでにディストリビューションに含まれています。 ハードキーボタンの通常の操作では、ASUSaからACPIドライバーをインストールする必要があります。 そして、ASUS Instant Key Utility。 こちらからすべてダウンロードしてください 。 VistaやWin7のバージョンはありませんが、XPドライバーはうまく機能しました。 1つの機能。 AsusSetup.exeを実行しないでください。 彼はWindowsのバージョンを誓います。 同じディレクトリに別のインストーラーがあります。 ACPIドライバーの場合はSetup.exe、インスタントキーユーティリティの場合はEEEInstantKey_1.08.exeです。 ここでは、互換モードで実行します。 すべてが完璧にインストールされ、私のために働いた。


ワイヤレス設定について少し掘り下げなければなりませんでした。 Windaはすべてのワイヤレスネットワークを見ましたが、何らかの理由で私のネットワークに接続したくありませんでした。 最初は、WPA2に問題があると思いました。 しかし、暗号化をオフにしても、Windaはネットワークへの参加を拒否しました。 判明したように、wi-fiカードのドライバーは少し湿っていて、すべてのルーターで正常に動作しません。 G専用モードのApple Airportベースのネットワークがあります。 モードを「Compatible b / g」に変更すると、問題は解決しました。 この問題がアップデートで解決されることを願っています。 ところで、wi-fiドライバーの更新プログラムは、Windows Updateを介して既に利用可能です。 この問題はすでに修正されている可能性があります。



次に、パフォーマンスについていくつか説明します。 この構成では、Windows 7は正常に動作します。 ウィンドウの透明度も自動的にオンになりました。 JDKおよびEclipseを含む標準のJava開発者キットをインストールしました。 Eclipseは正常にインストールされ、高速です。 サイドパネルを閉じると、通常は多少作業できます。 Androidエミュレーターは少し遅くなりますが、動作するのは不可能ではありませんでした。 主なことは、最初の起動後に閉じないことです。 :)もちろん、重いJ2EEネットブックアプリケーションはプルしません。 それが引っ張ると、開発は地獄に変わります。 しかし、簡単なことですが、非常に普通です。 この場合、移動性のためにサイズと電力を犠牲にすることができます。



インターネットでは、Google Chromeをインストールすることをお勧めします。 Internet Explorer 8よりも高速に動作しますが、おそらくこれは主観的なものです。
