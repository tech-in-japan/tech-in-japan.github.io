自分を救う！ セーブ!!!

私たちは皆、ビジネスの成功はさまざまな基準で判断できることを知っています。 もちろん、主な指標は利益です。 また、顧客の数とブランドの認知度もあります。 しかし、具体的にお話ししたい基準が1つあります。それは、データの量です。 会社が大きくなればなるほど、より多くの情報を毎日処理しなければならないことは論理的です。 毎年、組織のデータ量は増え続けています。 線形、指数関数的または指数関数的？ ここで、平均的な評価を行うことはほとんどお勧めできません。 しかし、EDMSへの大規模な移行の時代には、効率的なデータストレージシステムの必要性が明らかであることをご理解いただいています。



「バトルフィールド」に関して特に興味深いのは、SMBセグメントです。 今日、それは「国境地帯」として機能し、以前は独占的に市場の企業セクターに従事していたベンダーと、PCおよびルーターで名声を得て工業レベルに上がることを決めた「ホーム」レベルのメーカーの利益が収束します。 いつものように、オファーが増えると、買い手が勝ちます。 最終的に、彼は非常に「おいしい」価格で最高の技術を手に入れることができます。 まあ、これはデルの企業ブログなので、提供できるものもあります。



この秋、Dellはロシアで新しいネットワークストレージソリューション-PowerVault NX3300およびNX3200を発表しました。 簡単に説明すると、次のようなものになります。これらは、データストレージとファイル/データブロックの便利な交換を最適化し、効果的に管理するように設計された大容量システムです。 そして、これらすべては、前述のように、中小企業に注目しています。 さて、もっと詳しく見てみましょう。



Dell PowerVault NX3300は、ブロックデータとファイルデータを高可用性で保存、管理、共有、および作成するための完全なハードウェアおよびソフトウェアソリューションです。 これは、Windows Storage Server 2008 R2プラットフォームに基づく特殊なソフトウェアと統合されたフォールトトレラントネットワークストレージデバイス（NASゲートウェイ）で構成されています（キャラクターは北欧、シングル） 。







プラットフォーム自体は、Windows Server 2008 R2の柔軟で信頼性の高いファイルサービスの最適化されたリリースに基づいており、ファイルの重複やiSCSIソフトウェアターゲットを排除するなど、多数の有用なテクノロジーが採用されています。



なぜiSCSIであり、たとえばFC（ファイバーチャネル）ではないのですか？ 秘密はありません-ソリューションのコストのため。 本質的に、iSCSIテクノロジーに必要なのは、イーサネットインターフェイス、またはTCP / IP標準と互換性のあるその他のネットワークのみです。 同時に、高価な機器を取得したり、ハードウェアの分野で専門知識を取得したりする必要はありません。これらの知識がないと、FCストレージネットワークの運用は不可能になります。 しかし、NX3300に戻ります。



このデバイスは1Uラックマウントフォームファクターに実装されており、企業のニーズに応じて、クラスター構成を2台から16台に拡張できます。 SIS（シングルインスタントストレージ）テクノロジーは、シンプルなファイルレベルの重複排除を提供し、ファイルサービスリソースマネージャー（FSRM）は、CIFS / SMBおよびNFSファイルサービスを管理および監視するための単一のインターフェースを提供します。 同様に、Windows Server 2008 R2のファイル分類インフラストラクチャにより、管理者は最も効果的なデータ管理のポリシーの分類と実装を自動化できます。分類の有効期限の設定、タスクの設定、レポートの作成。



PowerVault NX3300の利点は、特定のストレージファミリへのバインドがないことです。つまり、バリエーションが可能です。 たとえば、既存のPowerVault MD、EqualLogic、またはCompellentディスクアレイでPowerVault NX3300 NASゲートウェイを使用します。 Windows Storage Server 2008 R2プラットフォームに基づく第12世代のDell PowerEdgeサーバーのクラスターは、非常に高温でも動作することを忘れないでください。 「グリーンIT」（「グリーンアイドタクシー」と混同しないでください。これは別の技術です）の傾向は、ご存知のように世界中で人気を集めています。 特に、EUはデータセンターの最高温度バーを35ºから45ºに引き上げることを計画しています。

このテーブルですべてが終了した場合、ストーリーは不完全になります。 そのため、別の新参者であるDell PowerVault NX3200に会うことを提案します 。



この高密度ネットワークシステムは、ファイルタイプとブロックタイプの両方のデータを交換するように設計されており、コンパクトな2Uエンクロージャーで12台のディスクを使用して最大36 TBの内部ストレージ容量をサポートします。







なんらかの理由で36 TBで十分でない場合（そして、「誰にとっても十分なRAM」を覚えている場合）、外部アレイDell PowerVault MD1200およびMD1220を使用して容量を増やすことができます。



すべてのファイル共有リソースは、NFSネットワークファイルシステムを使用するさまざまなプラットフォームのユーザー間のコラボレーションのために結合されます。 NFSを使用する場合、ストレージの信頼性は、ボリュームシャドウコピーサービス（VSS）を使用して分散ファイルシステム（DFS）を通じて複製され、バックアップされます。また、ファイルサービスリソースマネージャー（FSRM）は監視とレポートを提供します。



NX3200のもう1つの利点は、600 GB SASドライブまたは3 TBニアラインSASドライブを選択できることです。 実際のところ、Near-Line SASはSATAドライブであり、すべての外部電子機器はSAS標準に従って製造されています。 さらに、これは通常のSATAドライブをSASシステムに挿入できる外部のトリックではなく、ポートの1つで問題が発生した場合のシステムディスクのフォールトトレランスを大幅に向上させる本格的な2ポートSASエレクトロニクスです。



結論として、自尊心のある製造業者は長い間それらの世話をする義務があったので、自慢するのは意味がないいくつかの明白なことを言及します。 それは使いやすさとインストール速度についてです。 PowerVault NX3200パッケージには、ファイルサーバーをゼロから展開するために時間、お金、リソースを費やすことなく、安全で効果的な情報交換に必要なすべてのコンポーネントが含まれています（インストールウィザード、追加クライアント用の無制限のCAL）。 さらに、システムは、既に展開されている管理およびセキュリティツールを使用してWindows環境に完全に統合できるだけでなく、別のOSを使用してクライアントコンピューターのデータを保存することもできます（「Windowsではない」特定のK.O.を要求します）。
