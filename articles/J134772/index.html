初心者Drupal開発者向けの最新ツールの概要

比較的最近、DrupalConf'2011カンファレンスがモスクワで開催され、そこで新しいDrupal開発者向けのツールについて話しました。



半時間のスピーチ、プレゼンテーション、私が言ったことの短い要約が撮影されたビデオをあなたの注意に提示します（要約はいくつかの点で私の詳細な方向での私の口頭の会話と異なる場合があることに注意してください）。



ビデオをすぐに挿入すると、Yandex.Videoにアップロードされるので、ほとんどのユーザーは問題なく視聴できます。 誰がトラフィックを節約し、カットの下に行くか、プレゼンテーションとテキストの要約のみがあります。







これで、 SlideShareの実際のプレゼンテーション（ダウンロード可能）とスライドの番号付き要約が表示されます。



スライド1。
そこで、最初にツールを選択する基準の概要を説明します。 いくつかのオペレーティングシステムは、Microsoft Windows、Apple MacOSX、GNU / Linuxなど、世界で比較的異なる人気で配布されています。 いくつかの状況により、私はいくつかの仕事をしており、仕事でこれらのシステムのそれぞれを使用しています。 したがって、私にとってプログラムを選択するための最も重要な基準の1つは、クロスプラットフォームです。 複数のオペレーティングシステムで動作し、同じように見える能力。



私はこのシステムのいずれかの熱烈な狂信者ではないことに注意する必要があります。 私はそれらのそれぞれに等しく快適に感じています。 それはまさに、すべてのシステムで問題なく動作するアプリケーションを最大限に活用しようとしているという事実でした。 しかし、私自身にとって、Microsoft Windowsは、いわゆる保守派と呼ばれる習慣を変えたくない人のためのものだと考えました。 Apple MacOSXは、システムの定期的な再インストール、頻繁なシステムグリッチ（多くの場合、いわゆるハードウェア要因、不適切な構成、またはデバイスへのドライバーの誤ったインストールが原因）にうんざりしている人向けです-これはMacOXSには当てはまりません。 、また、非常に高品質を組み立てました。 たとえば、先日、ある場所に3年以上置かれていた妻のiMacのほこりを掃除することにしました-ケースを開けた後、プロセッサの上に立つファンブレードのわずかなコーティングを除いて、内部にほこりがほとんどないことがわかりました。

そして最後に、GNU / Linuxは、オープンソースコミュニティにできるだけ近づき（Drupalがオープンソースシステムであることは秘密ではありません）、現実に近い条件で開発を行いたい人々のためのものです。 ほとんどのホスティング会社はLinuxをプライマリサーバーシステムとして使用しています。 私の意見では、初心者の開発者でさえ、彼が作業しているものの少なくとも表面的なビジョンを持っている必要があります。



全体的なトピックの結論として、ツールを選択するための基準について話したことを思い出します。 クロスプラットフォームに加えて、利便性と機能性、作業速度、技術的な要件も重要です。 後者のオプションは、モバイル作業にネットブックを使用しているという事実のために重要です。



スライド2。


私が表明した基準に基づいて、議論できることを見てみましょう。 最初はGeanyです。 誰もがメモ帳++エディターを知っており、非常に便利だと感じています。 しかし、残念なことに、Windowsでネイティブに動作し、MacOSXおよびLinuxでは動作しません。 代わりに、私はGeanyエディターを使用します。これは一般にメモ帳++に比べて機能は劣りませんが、WindowsとLinuxの両方で機能し、プラグインにより拡張し、多くのプログラミング言語とスクリプトの構文をサポートします。 エディターは非常に軽量で高速ですが、最初はおそらく外観に少し慣れる必要があります。 おそらく、彼はマイナスの面を持たないでしょう。ただし、原則としてプロジェクトの開発作業の多かれ少なかれサポートはなく、チーム開発ツールのサポートは最小限です。



さらに、初心者の開発者は2つの開発環境に注意を払うことができます。2つのモンスターはEclipseとNetBeansです。 どちらのIDEも非常に機能的で便利で、多くの利点があります。 どちらを選択するかはあいまいですが、快適な作業にはNetBeansの方が好きです。 大規模なチームで作業する場合は、Eclipseを使用する可能性が高くなります-すべて同じですが、チーム開発ツールのサポートはEclipseでより適切に実装されます。



おそらく、NetBeansでの作業に煩わされるものは2つしかありませんが、気にしません-これは、Eclipseで作成されるワークスペースのサポートの欠如です（ワークスペースは別のフォルダーにあり、起動時に選択されます）どの形式で行末を作成するかを指定できない-WindowsのスタイルまたはUnixのスタイル。 ご存じのとおり、Drupalのコーディング標準では、Unixスタイルのエンディングを使用することが望ましいとされています。 ただし、Windowsスタイルの行末を使用する場合の致命的な間違いはありません。 NetBeansはファイルの先頭を見て、その中にあるスタイルを再現します。 つまり ファイルが元々Windowsのスタイルで終わる場合、これはファイル全体になります。 NetBeansで作成された新しいファイルは、現在実行されているオペレーティングシステムに従って作成されます。



特にPHPやDrupalの開発者を使用するエディターやIDEは他にもたくさんありますが、それらのほとんどがクロスプラットフォームではなく、一部は有料であるという理由で、それらを考慮していません。



スライド3。


ご存知のように、PHP（Drupalはほとんどの部分がPHPで記述されています）のスクリプトはサーバー上で実行されます。 したがって、開発者は、プロジェクトが適切に機能することを保証するソフトウェアを選択することが重要です。 最も一般的なのは、Apache Webサーバー、MySQL DBMS、およびPHPインタープリターのセットです。 nginxまたはlighttpd Webサーバー、PostgresSQL DBMSなどを使用するオプションもありますが、それらはあまり一般的ではなく、さらにDrupalモジュールでは完全にサポートされていません。



経験の少ないPHP / Drupal開発者は、Denwerと呼ばれる広範なビルドを知っています。 これは、Apache + MySQL + PHPの最小限のセットであり、Drupalのサイトで作業できるようにOSにインストールするのに十分です。 しかし、このアセンブリはWindowsでのみ動作することを知っておく必要があります。私の意見では、このアセンブリは単純すぎて、開発者は多くの場合、内部ですべてがどのように動作するか想像していません。 さて、Drupalサイトはデンワーで開始したときに正しく機能しなかったことがありましたが、ホスティング業者には問題なくすべてがありました。 このプラットフォームについては、その弱点を示すためだけに言及しました。



私のプロジェクトでは、ローカルで作業する場合、XAMPPを使用します。このアセンブリはDenwerよりも機能的で、ApacheおよびPHP用の拡張モジュールがはるかに豊富です。 さらに、Windows、Linux、およびMacOSXでより高速に動作し、最も重要なことです。 唯一の欠点は、アドレスがローカルホストと異なるサイトで動作するように設定するのが、Denwerを使用するよりも少し複雑なことです。



Apache + MySQL + PHPの動作を実際に理解するために、必要なすべてのコンポーネントを順番に個別にインストールできます。これらは、ネットワーク上で見つけるための構成ガイドです。 私はこの世界でZend Serverについて言及したいと思います-これは非常に有名な会社Zend Technologiesのアセンブリです。 無料のコミュニティエディションと、スクリプトの暗号化/復号化モジュールを含むいくつかのアドオンを含む有料版の2つのバージョンで配布されます。



MacOSXにはDenwerの類似物があり、それがMAMP Proと呼ばれるという非難を避けるために言及します。 ただし、Proバージョンは有料であり、MAMPの無料バージョンの機能は非常に制限されていることを理解しておく必要があります。



スライド4。


開発者が何に取り組むことができるかを検討しました。開発者が開発プロセスをより正確に構築するのに役立つツールをいくつか提案するときです。



私は特にバージョン管理システムに言及したかったです-これはSubversion、GIT、Mercurialなどです。



それらを使用すると、この職場で働いていたか、別の職場で働いていたかに関係なく、いつでも戻って変更の履歴を見ることができます。 そして、確かにこれらのシステムは、複数の人々がプロジェクトに取り組んでいるときに役立ちます。



初心者の開発者には、まずGITに注意を払うことをお勧めします-まず、このシステムは別のサーバーパーツを必要としないため、使いやすくなります。次に、drupal.orgチームがDrupalコードとそのモジュールを扱うために選択しましたこれは、プロジェクトを公開してdrupal.orgのモジュールのリストに追加する場合にも重要です。



チームに多くの開発者がいる場合、Subversionを使用する可能性が高くなります。Subversionの集中化と普及により、Subversionはより制御されます。 ところで、このトピックに関するレポートの1つは今日になります。

そしてもちろん、Drupalの開発者として、少なくともこのDevelとCoderの開発者向けのモジュールを使用し、使用できるようにする必要があります。 Develモジュールは、サイトのボトルネックを理解するのに役立ちます。Coderは、開発標準の観点から正しく記述した限り、コードを制御します。



Drupalのサイトでの作業の利便性のために、drush makeオプションとともにdrushユーティリティの可能性を調査することも強くお勧めします。これは非常に役立ちます。 特に、最近サイトでdrupal.ruは、drushを使用して開発者の各バージョンとサイトの作業バージョンを同期する方法について話しました。



ちなみに、私はWindowsでDrushを設定しようとしましたが、何かをしましたが、Linuxとは異なり、それは非常に簡単なことでした。



スライド5。


最後の言葉-スライド上に、私がリストしたツールの一部についてのリンクと、Drupal用にそれらを構成する方法のリンクがあります。 私は以前に言及していませんが、EclipseまたはNetbeansをセットアップするとき、いわゆるコードテンプレートを追加することが可能です。 特定の名前を入力して特別なキーを押すことで挿入される既に実行されているコードの断片-結果として、Drupalのフック関数の宣言をはるかに速く、より正確に作成できるようになります。
