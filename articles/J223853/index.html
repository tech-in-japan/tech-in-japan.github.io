ホームcctvシステム

こんにちは、ハブロフチャニ！ 子供の頃、私はすべてを知り、みんなを見るのが大好きだったので、私はあなたと私の小さな弱点を共有することにしました！ この考えから、アパート内に独自のビデオ監視システムを構築したいという思いに至りました。 私の道は少し厄介でしたが、最初に最初にしたことです。



第一段階


私が最初にシステムを構築し始めたのは、ビデオ監視システムの構築の基礎を研究することでした。 実際、ビデオ監視システムは、特定のデジタルデバイスに接続されたビデオカメラであり、カメラからのビデオ信号の処理、画像の記録、アラームメッセージの発行を担当します。 記録は、コンスタントモードで、または何らかのイベント（モーション検出、カメラカバー、カメラのシャットダウン、またはセンサーの動作）の場合にのみ実行できます。 最新のビデオ監視システムはインターネットに密接に接続されているため、カメラからのライブ画像の表示、録画したビデオの表示、世界中からのビデオ監視システムの制御が可能です。



タスクを簡素化するために、私は最新のプラットフォームであるIPカメラにとどまることにしました。 これはおそらく最新のものであるだけでなく、最も高価なオプションでもあります。 実際、各IPカメラは、必要なすべての機能を実装する個別のビデオ監視システムです。



次に、ビデオ情報を保存する方法を見つける必要がありました。 私は長い間、ラップトップを支持して固定PCを放棄してきました。古いウィッシュリストの1つであるホームサーバーを実装することにしました。 HP ProLiant MicroServer G7 N54Lは価格が8,870ルーブルであり、このクラスのハードウェアの一部として非常に興味深いため、 すぐに言われましたが、 3920ルーブルで2 TBのHDD WD Red WD20EFRXを購入しました。 Windows Home Server 2011がインストールされました。 この準備で、IPカメラの外観の準備が整いました。











率直に言って、この全体のアイデアは非常に自発的であり、予算やリソースを事前に計算していませんでした。爆発の方向にすべてを行いました。



第二段階


2番目は、IPカメラの選択です。 選択の主な基準は、アパート全体に配線を引っ張らないように、Wi-Fiの可用性でした。 適切に、問題を研究することなく、 2 080ルーブル用の最も手頃な価格のシンプルなIP D-LINK DCS-931Lカメラを選択しました。 ここで、購入する商品の特徴を慎重に検討する必要があることに気付きました。











実際、彼女ができるのはリアルタイムで写真を表示することだけであり、カメラからの画像を保存する問題に関連することはすべて、より高価なバージョンにのみ存在します。 このカメラのすべてのTXを投稿するのではなく、その主な機能の説明に限定します。







  • D-Linkクラウドサーバーを介したリアルタイムビデオストリーミング。 このサービス用のモバイルアプリケーションもあります。これにより、任意のモバイルデバイスでカメラからの写真をリアルタイムで表示できます（写真のみを記録できません）。
  • Webインターフェイスを介してビデオストリームを表示します。
  • 写真とビデオをFTPに送信します。メインファイルは、ビデオが3 MBを超えてはならないことです。












利用可能な機能に基づいて、通常のWindowsツールを使用してMicroServerのFTPサーバーを上げ、それに応じてカメラを構成し、サーバーへのストリーミング録画を整理しました。 その結果、2、3秒のビデオを含む3 MBのファイルが大量に得られました。 実用的な観点から、視聴と分析に適さないビデオアーカイブを受け取りました。ビデオストリームの圧縮は不十分です（ただし、MJPEGには何が期待できますか）。 将来、私はこのカメラを入り口に移動し、イベントの記録を構成することを決定しました。 フレームに動きがある場合にのみ書き込みます。 突然誰かがそれを緩めたいと思うなら、私はそれがどのように見えたか知っているでしょう。 このカメラの2番目の重大な欠点は、もちろん画質でした.640x480の小さな解像度ではなく、最も重要なことは、 45°の小さな視野角です。これは、コーナーから部屋の領域をキャプチャするために非常に小さく、カメラを通して、領域の3分の1しか表示されません。 そしてセンサーの感度は悪く、良い光の下でしか見えません。ナイトモードはありません。音はひどいです。

カスタマイズ


カメラをホームネットワークに接続した後、すぐにファームウェアをV5.1.2ビルド140116に更新し、構成を進めました。









多くの基本的な設定、または数十個の設定を行った後、カメラは必要に応じてストリーミングを開始しました。 さらに、私のタスクはカメラにリモートサーバーへの記録を強制することでした。 基本的にNASとして機能するWindows Home Server 2011を搭載した同じサーバーを使用します。 そして、タンバリンと踊り始めました。 次に、自分の肌で感じる必要があるチューニングの経験を説明します。 ネットワーク上で情報が見つかりませんでした。







そしてもちろん、最も興味深いのは、両方のカメラの画像を比較することです。





次に、このカメラでビデオストリームをアーカイブに書き込む必要がありました。 判明したように、カメラはいくつかのプロトコルを使用してビデオストリームを記録できます。 設定-詳細設定-ストレージ-NASに移動します。



まず、NFSプロトコルを使用してNASを構成しようとしました。 すべてが単純で、Windowsの下で5分で上昇します。 設定でNFSストレージへのパスを指定した後、カメラはサイレンをオンにしてシャットダウンしません（はい、カメラに内蔵サイレンがあります）。その後、[ストレージ管理]タブのフォーマットボタンを押します。コンテナが作成されます。 フォーマットが完了すると、ステータス「Norm」が設定されますが、文字通り数秒後にリセットされ、カメラは再びサイレンをオンにし始めます。 5回試した後、私はいつも同じことをしました。

次に、SMBプロトコルを使用して接続しようとしましたが、ここではまだ簡単です。設定されたアクセス権を持つサーバー上の通常の共有フォルダーです。











また、「ストレージ管理」タブでフォーマットすると、すべてが機能します。



しかし、未解決の問題がいくつかありました。



1.サイレンを無効にする方法 （カメラがサーバーとの接続を失っても常に機能します）。

2. NASとの接続で通信を回復する方法は？ （ルーターまたはサーバーを再起動すると、カメラの接続が失われ、カメラを再起動するまで接続が復元されません。これにはすべてサイレンが伴います）。

最初に解決した問題は、 設定-詳細設定-イベント-イベントに進みます 。 次に、イベントのタイプ（この場合は「HDDエラー」）を選択し、[オーディオ]メッセージボックスをオフにします。











2番目の質問は自然に落ち、今ではすべてが自然に復元されています。 これですべてがPSUソースに接続されましたが、外部電源が供給されると、UPSはカメラとサーバーの両方にルーターで電力を供給します。 ライトがオフになっている場合、モバイルでSMSを受信し、RDPを介してサーバーに接続し、すべてを正しく出力できます。 電源が復旧すると、SMSも受信します。WakeOnLanを介してサーバーの電源を入れることができます。すべて携帯電話から起動できます。
