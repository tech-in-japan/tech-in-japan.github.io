ネイティブクライアントはChromeに含まれています

2008年以来、Google はオープンソーステクノロジーのNative Clientを 開発してきました 。これにより、JavaScriptのようなセキュリティ制限のあるブラウザーでx86コードを実行できます。 Native Clientのおかげで、CPUリソースに直接アクセスできる新しい世代のブラウザベースのアプリケーションが登場することを説明する必要はありません。 ある意味、これはネイティブアプリケーションとWebアプリケーションの間に欠けていたまさに中間リンクです。



Googleは2年半の間、Native Clientのセキュリティ問題の修正に取り組みました 。ついにそれが起こりました 。昨日、Native ClientはGoogle Chrome 14の最新ベータに含まれていました。



ネイティブクライアントアプリケーションは、 Pepper Programming Interfaces（PPAPI）を使用します。これは、Internet Explorer（ActiveX）を除くほとんどのブラウザーで使用されるテクノロジであるNetscape Plugin API（NPAPI）のアップグレードです。



新しいChromeリリースには、さまざまなオーディオエフェクトを備えたWeb Audio APIのサポートも含まれており、これは特にゲームの作成に役立ちます。
