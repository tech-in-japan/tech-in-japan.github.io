Windows上のDjangoとの共同リモート開発のための完全な環境を展開するための手順。 絶対的な初心者向け。

この記事は普遍的であると主張しているわけではありませんが、この記事は環境をゼロから「コードの最初の行を書く」段階にデプロイするための実行可能な命令です。 コメントでは、指示を改善するためのヒントと推奨事項が受け入れられます。



1. Pythonをインストールして構成します。



Pythonをインストールするには、明らかに、開始に必要です
  • http://python.org/download/からダウンロードし、インストーラーの指示に従います。
オプションですが推奨される手順は、PATH環境変数を設定することです。これは、プロジェクトを管理するときにコマンドラインで余分な制御ワードを入力する必要がないようにするために必要です。
  • そのため、PATH変数は（Windows Vistaの場合）[コンピューター] >> [プロパティ] >> [高度なシステムパラメーター] >> [環境変数]で見つけることができます。[システム変数]ボックスでPATH変数を見つけ、 C：\ Python25; C：\ Python25 \スクリプト; C：\ Python25 \ lib \ site-packages \ django \ bin "、
ここで、「C：\ Python25」は、インストールされているPythonのルートディレクトリです。





2. Djangoをインストールします。



ここで、再び、最初に
  • ページに表示されているいずれかの方法で、http：//www.djangoproject.com/download/からDjangoをダウンロードします。
  • （最初の方法で）公式バージョンをダウンロードすることに決めた場合は、アーカイブを解凍した後、コマンドラインで解凍したDjangoのルートディレクトリに移動し、「setup.py install」コマンドを実行します（最初のステップの環境変数が設定されていない場合）次に「python setup.py install」）。
必要なツールがデプロイされましたが、開発環境次第です。





3. Eclipseをインストールします。



ここでは、一般に、すべてが簡単です。Eclipseはhttp://www.eclipse.org/downloadsからダウンロードできます。インストールはそれほど難しくありません。





4. PyDevをインストールして構成します。



Eclipseを手元にインストールしたら、Pythonで開発するためのプラグインをインストールできます。その名前はPyDevです。
  • Eclipseを起動し、メニューから[ヘルプ] >> [ソフトウェアの更新] >> [検索とインストール...]を選択します（[ヘルプ] >> [ソフトウェアの更新] >> [検索とインストール...]）。


  • 選択：インストールする新しい機能を検索します

    [New Remote Site ...]ボタンをクリックして、ネットワークアドレスhttp://pydev.sourceforge.net/updates/を入力します。


  • 追加したサイトの横にあるチェックボックスをオンにして選択し、[完了]ボタンをクリックします。


  • インストールキットのリストでPyDev拡張機能を選択し、[完了]ボタンをクリックします。 選択したコンポーネントがダウンロードされ、システムにインストールされるのを待ちます。
次に、PyDevを構成する必要があります。
  • メニューから[ウィンドウ] >> [設定]を選択し、左側のツリーで[PyDev >> Interpreter-Python]を指定します。


  • Pythonインタープリターパネルの上部で、[新規...（作成）]をクリックし、インストールされているPython（python.exeまたはpython for Linux）へのフルパスを指定します。
  • 次に、System PYTHONPATHパネルで[New folder]をクリックし、ルートDjangoフォルダーへのパスを指定します。
さて、「OK」をクリックすることを忘れないでください。





以下は、バージョン管理システムをセットアップし、リモート共同開発のためにWEBでリポジトリを作成するためのオプション項目です。





5.リポジトリの作成（Googleプロジェクトのホスティング上）。



これを行うには、すべてのために、http：//code.google.com/hosting/createProjectでアンケートに記入し、Gmailアカウントでログインする必要があります。





6. SubVersionをインストールして構成します。



SubVersionまたはバージョン管理システムを使用するために、Eclipse用のSubEclipseプラグインがあります。
  • Eclipseメニューから、[ヘルプ] >> [ソフトウェア更新] >> [検索とインストール...]（ヘルプ>>ソフトウェア更新>>検索とインストール...）を選択します。


  • 選択：インストールする新しい機能を検索します。


  • [新しいリモートサイト...]ボタンをクリックして、次のネットワークアドレスを入力します。http：//subclipse.tigris.org/update_1.4.x



次に、5番目の手順で作成したリポジトリでシステムを構成する必要があります。これを行うには、
  • SubEclipseパースペクティブ（ウィンドウ>>パースペクティブを開く>>その他... >> SVNリポジトリ探索）を選択して、
  • SVNリポジトリパネルで、コンテキストメニューの[新規... >>リポジトリの場所...]を選択し、URLフィールドにリポジトリアドレスを挿入します。
リポジトリのアドレスは、Googleプロジェクトのホスティングに関するプロジェクトのページの[ソース]タブに表示されます。 これは、「https：// <プロジェクト名> .googlecode.com / svn / trunk /」という形式の行です。
  • 次に、システムはユーザー名とパスワードを要求します。
ここでのログインはGmailアカウントの名前です。「プロンプトが表示されたら、生成されたgooglecode.comパスワードを入力してください」リンクをクリックしてパスワードを生成する必要があります。 プロジェクトの同じページの同じ[ソース]タブにあります。









まあ、それだけです。
