Baruch Sadogurskyとのインタビュー：企業開発に理想的な技術スタック

Baruch Sadogursky-JFrogの開発者支持者。 2011年と2013年にデューク賞、2014年にJavaOne Rock Star



このインタビューは、Javaソフトウェア開発のLuxoft TrainingエキスパートであるVladimir Sonkinが実施しました。







LT： Baruch、今日のエンタープライズプロジェクトのテクノロジースタックをどう見ていますか？



BS：プロジェクトの基礎として-Javaでは、どこでも必要なSpringを使用できますが、それは要件に依存します。 Webベースのインターフェースがある場合は、内部使用または外部用の製品であるかどうか、ユーザーインターフェースであらゆる種類の「ホイッスル」が必要かどうかなど、さまざまなフレームワークを見ることができます。 選択は、私たちが何をしているか、ビジネスタスクが何であるかによって異なります。BigDataの処理なのか、ファイルの操作なのか、その他のことなのか。

組み立てに関しては、Apache Mavenは堅実で信頼できる選択肢であり、Gradleは独自のリスクを伴う、よりスタイリッシュで若々しいツールです。 Git-ソース管理システムとして、CIサーバー-Jenkins。 今日のスタックは非常に単純です。 そして、どこかで、多分端の方で、例えばSpokでテストを書くなど、新鮮なものを試してください。 もちろん、これは新しいものではありませんが、エンタープライズ開発者にとってはかなり新鮮な選択です。 ただし、テストは実稼働に移行しないため、この領域で何かを感じて実験することができます。



LT：しかし、Scalaを見ると、一般的には既に成熟した言語です。 一部の小規模なエンタープライズプロジェクトでは、Scalaを関心の対象から除外しています。 これは理にかなっていると思いますか？



BS： Scalaに関連するライブラリから収益を得ようとするTypeSafeなど、Scalaから遠ざかる傾向があるため、Scalaで何が起こっているのか、少し怖いです。 少し前に、TypeSafeは、Javaの開発の大部分が実際にお金のある場所であるため、ScalaのJavaのはるかに大きな強調から離れることが発表されました。 そのため、Scalaに主に重点が置かれていたため、以前は持っていなかったいくつかのJavaインターフェイスをプルアップする多くの新しい人を雇いました。



また、非常にScala指向のその他のライブラリもあります。 たとえば、Sparkは、新しいリリースでJava APIがScalaレベルにどのようにプルアップされているかを示しており、Scala APIよりも悪くないとしても、Scala APIよりもさらに悪化しないようです。

グローバルな状況では、Javaへの反転が見られると思います。 また、アカデミック言語としてのScalaは非常に素晴らしい作成物であるため、これにはまったく驚きません。強力であると同時に非常に複雑なツールではなく、Scalaの型システムを理解するよりも重要なことがあります。



だからこそ、私はScalaの大ファンではないので、Scalaを離れる傾向は私を驚かせません。 確かにScalaはどこにも行かず、開発を続けますが、はるかにニッチな形式です。



LT： Scalaは彼女の過度の複雑さを殺したのですか？



BS：それほど複雑ではありません。 私は言うだろう-アカデミック。 Scalaは当初から大量開発には適していなかったようです。 そして、Scalaが次のJavaと呼ばれたとき、私は聞くのに十分ワイルドでした。なぜなら、すべてが非常に学術的であり、Javaが置き換えなければならないような大規模な言語にとっては簡単ではないからです



LT ：たとえば、KotlinはScalaより低いエントリしきい値を持っていますか？



BS ：はい、Groovyよりは高いですが、Scalaよりはずっと低いです。 Javaの後に学習するのはとても良いことです。 Kotlinでは、JetBrainsのスタッフが必要な機能を選択しようとしたため、すべてが異なります。 業界での採用は言語自体の品質だけでなく、他の多くの理由にも依存するため、それがさらに進む方法を見てみましょう。 たとえば、ご存知のように、言語の作成者にひげがある場合、その言語は人気があります:)。









セントケントムソンとデニスリッチーのクリエーターは、素晴らしいひげを持つ素晴らしい男です。



LT ：Javaの人気の高まりは、同じScalaから多くのメリットをもたらしたJava SE 8が登場したことによると思いますか？



BS ：もちろん、これは役割を果たしました-Javaの停滞の終わりとして。 これは大きなプラスです。 Java 8の革命的なリリースを見ると、Java 9には多くの興味深いことがあると聞きます。一般に、Javaでのある種の生活が復活しました。 多くの企業でのJavaからの脱出は終わりました。 Javaで記述し、生産性を高めることができます。



LT：あなた自身がEnterpeiseプロジェクトを開始した場合、ビルドフレームワークとしてApache Mavenを選択しますか？



BS：チームの力が強くなればなるほど、ビルドシステムを人々が完全に理解できるようになるという事実に依存するようになり、Apache Mavenから離れることができます。 しかし、現実は異なります。通常、開発者は私のビルドスクリプトのニュアンスに応じていないため、一般に、シンプルで信頼性の高いApache Mavenコレクターが必要です。 したがって、選択は、作業する必要がある素材、およびタイミング、ビルドスクリプトの複雑さを掘り下げる時間があるかどうかによって異なります。



LT：プログラミング言語についても同じように言えば、Javaはエンタープライズ開発の事実上の標準です。 今日最も有望なテクノロジーと言語は何だと思いますか？



BS ：非常に良い比較です。ApacheMavenはJavaと比較できると思います。これは事実上の標準であり、誰もが知っているからです。かなり予測可能で理解できるからです。 私はそれらが単純であるとは言いませんが、あなたは人々がそれらについて知っていることを当てにすることができます。



Gradleは、他のプログラミング言語にますます結び付けられています。スクリプトはGroovyで作成でき、すぐにKotlinで作成できるようになるためです。これにより、Javaと比較して何らかの柔軟性が得られます。

したがって、今日見たい興味深い言語について話している場合、GroovyはJavaに非常に似ているため、Groovyはわずかな投資で視野を広げる絶好の機会です。 さて、Ketlinは非常に有望に見えます。JetBrainsのメンバーが他の言語から最高のものを集めようとしたからです。 そこでScala、Groovy、Javaから多くの優れた点を見ることができます。 したがって、これらすべてを注意深く監視する必要があります。 明日の大規模なエンタープライズプロジェクトでは、Kotlinを使用しませんが、はい、プレイして、突いて、理解します。



LT： 9月にLuxoftで、Apache Maven、Javaパフォーマンス、プロファイリングに関するマスタークラスを開催します。



BS：はい、マスタークラスはApache Maven上にあり、Javaの生産性については、プロファイリングとデファギングがボーナストラックとしての講義になります。



LT：他の多くの人気コレクターがいるにもかかわらず、Mavenが選ばれた理由を教えてください。 マスタークラスに来る価値があるのはなぜですか？



BS： Apache Mavenを使用すると、すべてが非常にシンプルになります-業界で通常行われているように、先進的、革新的、興味深い、現代的で珍しいツールがあり、常に最も人気があるわけではありません。すべての最新かつ最も興味深いものが含まれていますが、必要なことを行う主力製品です。 そして、ビルドシステムにも同様の状況があります：市場にはGradleがあり、私たち全員が見たり掘ったりするのが大好きで、珍しい解決策が湧き出ています。 大規模な大企業は、作業標準として2番目のオプション、つまりApache Mavenを選択します。 そして、ここではすべてが論理的です。長年にわたって理解し、より制御され、テストされています。



したがって、私のマスタークラスは、大企業で働き、アセンブリで何が起こっているのかを理解するために、1つのアセンブリ言語を見る必要がある人々に役立ちます。 しかし、抵抗することはできないと思います。マスタークラスの後半では、Gradleをレビュー形式で見て、その違いについて話します。



LT：基本的なことや、微妙なことや秘密について話してくれますか？



BS：すべてについて話す時間は十分にあります。 Apache Mavenにすでに慣れている人でも、すべてのホワイトスポットを閉じるための基本的なことから始めます。 そしてもちろん、プラグイン、構成プロファイルの作成など、より高度なことについても説明します。 そのため、Apache Mavenにある程度精通している人、Apache Mavenにある程度精通している人、Apache Mavenを使用したことがない人など、すべての人にとって興味深いものになることを願っています。



Baruch Sadogurskyのマスタークラスは、2016年9月8日にモスクワで開催されます。
