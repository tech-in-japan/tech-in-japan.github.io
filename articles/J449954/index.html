CentOS 7にZimbra Open-Source Editionをインストールする

企業でのZimbra実装の設計中に、ITマネージャーはZimbraインフラストラクチャノードが動作するオペレーティングシステムを選択する必要があります。 現在、ほとんどすべてのLinuxディストリビューションはZimbraと互換性があり、国内の赤いOSとROSAまで対応しています。 通常、企業にZimbraをインストールするための選択はUbuntuまたはRHELのいずれかです。商業企業はこれらのディストリビューションを開発しているためです。 ただし、多くの場合、ITマネージャーの選択は、Red Hatの商用RHELディストリビューションのコミュニティ主導型のフォークであるCent OSに依存します。







Zimbraの最小システム要件には、サーバー上の8 GBのRAM、/ optフォルダー内の少なくとも5ギガバイトの空き領域、および完全なドメイン名とMXレコードが含まれます。 原則として、初心者にとって最大の問題は、最後の2つのポイントで正確に発生します。 この場合のCentOS 7の大きな利点は、オペレーティングシステムのインストール段階でサーバードメイン名を設定できることです。 これにより、Zimbra Collaboration Suiteを問題なくインストールできます。これまでLinuxの経験がなかったユーザーでも同様です。



この場合、Zimbraがインストールされるサーバーのドメイン名はmail.company.ruです。 インストールが完了した後、 192.168.0.61 mail.company.ru mailの形式の行を追加するだけです。192.168.0.61の代わりに、サーバーの静的IPアドレスを登録する必要があります。 その後、すべてのパッケージ更新をインストールし、 dig -t A mail.company.ruおよびdig -t MX company.ruコマンドを使用してサーバーにAおよびMXレコードを追加する必要があります 。 したがって、サーバーには本格的なドメイン名が付けられ、Zimbraを簡単にインストールできます。



公式サイトzimbra.comから、Zimbraディストリビューションの最新バージョンのアーカイブをダウンロードできます。 アーカイブを解凍すると、install.shという名前のインストールスクリプトを実行するためだけに残ります。 これに必要なコンソールコマンドのセットは次のとおりです。



mkdir zimbra && cd zimbra

wget files.zimbra.com/downloads/8.8.12_GA/zcs-8.8.12_GA_3794.RHEL7_64.20190329045002.tgz --no-check-certificate

tar zxpvf zcs-8.8.12_GA_3794.RHEL7_64.20190329045002.tgz

cd zcs-8.8.12_GA_3794.RHEL7_64.20190329045002

./install.sh






この直後に、Zimbra Collaboration Suiteインストーラーが起動します。 まず、ZCSのインストールを続行するには、ライセンス契約に同意する必要があります。 次のステップは、インストールするモジュールの選択です。 1つのメールサーバーを作成する場合は、すべてのパッケージを一度にインストールするのが理にかなっています。 スケーラビリティを備えたマルチサーバーインフラストラクチャを作成する場合は、以前の記事のいずれかで説明されているように、インストール用に提案されたパッケージのいくつかのみを選択する必要があります。



インストールが完了すると、Zimbraセットアップメニューがターミナルで直接開きますシングルサーバーインストールを選択した場合は、管理者パスワードを設定するだけです。 これを行うには、最初に項目番号7を選択し、次に項目4を選択して管理者パスワードを設定します。管理者パスワードは少なくとも6文字でなければなりません。 パスワードを設定したら、Rボタンを押して前のメニューに戻り、Aボタンを押して変更を受け入れます。



Zimbraのインストールが完了したら、コマンドfirewall-cmd --permanent --add-port = {25,80,110,143,443,465,587,993,995,5222,5223,9071,7071} / tcpでファイアウォールでの操作に必要なポートを開き、コマンドでファイアウォールを再起動しますfirewall-cmd --reload



ここで、開始するには、 service zimbra startコマンドで Zimbraを開始する必要があります。 company.ru：7071 / zimbraAdmin /にアクセスすると、ブラウザーで管理コンソールにアクセスできます。 メールユーザーのアクセスはmail.company.ruで提供されます。 Zimbraの操作中に問題やエラーが発生した場合は、ログ内で回答を検索する必要があります。 ログは/ opt / zimbra / logフォルダーにあります。



Zimbraのインストールが完了したら、Zextras Suite拡張機能をインストールすることもできます。これにより、ビジネスで要求される機能を追加することにより、Zimbraの使用の信頼性と費用対効果を高めることができます。 これを行うには、Zextras.comから最新バージョンのZextras Suiteを含むアーカイブをダウンロードし、解凍する必要があります。 その後、解凍したフォルダーに移動して、インストールスクリプトを実行します。 コンソール形式のプロセス全体は次のとおりです。



wget download.zextras.com/zextras_suite-latest.tgz

tar xfz zextras_suite-latest.tgz

cd zextras_suite /

./install.sh all






その後、Zimbraでメールストレージのデータのアーカイブと重複排除、セカンダリボリュームのマウント、他のユーザーへの管理権限の委任、Zimbra Webクライアントでのオンラインチャットの使用などを行うことができます。
