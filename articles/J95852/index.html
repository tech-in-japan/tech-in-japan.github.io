N900、新しいファームウェアPR1.2-失敗または成功？

こんにちは、habrachelovek。 Nokia N900に関する一連の記事を続けます。 タイトルが「黄色」を与えることは理解していますが、私は本当に記事の本質を反映したかったのです。

この記事では何を議論しますか？ 私は記事に対処します：

  • 既存のN900ユーザー
  • 電話を選択する人;
  • N900を入手する見込みに注目します。
  • 現在の「N900ではない」をN900に変更しない人。
  • 自身の信念または偶然によりNokia N900の所有者にならない人。
  • MeeGoを搭載したN9のリリースを待っています。
  • iPhone 4Gについて読むのにうんざりしている人:-)。




すなわち、モバイル技術の世界に興味があるすべての人にとって。 なぜ誰もが興味を持つべきなのでしょうか？ はい、この記事では、ノキアが現在および将来、新しいマーケティングの方向性（Maemo / MeeGo）を促進するポリシーをどのように追求しているかについて、マーケティングの観点だけでなくソフトウェアのサポートの観点からも説明しようとします。 最近リリースされたNokia PR 1.2の例について、この実証分析を行います。

ここに記載されているものはすべて主観的な観点であり、専門家による評価ではありません。 このトピックは非常に興味深く、難しいものですが、長期計画（少なくとも「別の観点」として）や単に好奇心を満たすために役立ちます。

そして、N900にMeeGoがあるかどうかという質問に答えます!!!





1.結論



そのため、2010年5月24/25に、N900用の待望の（当然N900ユーザーによる）ファームウェアがリリースされました。 これはどういう意味ですか？ 間違いなく、これはエラーの修正と新しい機能の追加ですが、これは主なことではありません。 このイベントの主な本質は、最大のモバイル関連企業であるノキアのマーケティングおよびソフトウェアビジネスモデルの再編成における重要なステップです。

ただし、一部のユーザーは満足しておらず、ノキアの行動を批判しています。 次に、この「怒り」が正当化されるかどうかを把握してみましょう。

まず、N900の変更点と、このアップデートが新しいソフトウェア機能の点で革新的であると個人的に考える理由について説明しましょう。

次に、このファームウェアがモバイル市場全般を促進するための大きな一歩であり、ノキアの市場での地位を維持する上で非常に重要な役割を果たしている理由を説明します。

最後に、開発者向けの小さなメモを作成します。

3.さて、これはどういう意味ですか？



ここで、これらすべてが非常に重要である理由を説明します。 ここに重要なポイントのリストがありますが、以下ではそれらが非常に重要な理由を理解します。

  1. Qt 4.6ビルトイン
  2. Qt Mobility 1.0.0
  3. Nokia Qt SDK
  4. ビデオスカイプ/ジングル/ SIP
  5. 社会サービスの追加


最初の3つのポイントを組み合わせることができます。 ファームウェアPR 1.2は、デバイスの「そのまま」のQtです。 これにより、Qtアプリケーションが自動的にMaemoプラットフォームにネイティブになります。 Qt Mobility 1.0.0は、電話機能にアクセスするためのクロスプラットフォームAPIを追加するQtのアドオンです。 したがって、QtはN900の公式開発フレームワークになりました。 また、Qt + Qt Mobilityを使用して記述されたすべてのアプリケーションは、Symbian、Maemo、MeeGoの3つのプラットフォームで自動的に動作します。 そして、Nokia Qt SDKを使用して、これらのアプリケーションをクロスプラットフォームで開発できます（ これについては、私の記事で詳しく説明します ）。

2番目の非常に重要なポイントは、ネットワークを介したビデオコールの追加です。 現在、クライアントがサポートしている場合、ユーザーの連絡先で「ビデオコール」項目を使用できます（Skypeの場合、最新バージョンが必要であると言われました（番号は指定されていません）.WindowsとMac OSでSkypeを確認して動作し、Linuxで確認する方法はありません）。



N900のSkype / Gtalk / SIPビデオは、単純な3Gビデオとはそれほど大きな違いはないことを知っています-会話の質がかなり低い小さな写真です。 ほとんどの人が期待するものではありません:-)

以下は、通信セッションのスクリーンショットです。



ビデオディスプレイは「オーバーレイ」に実装されているため（ウィンドウシステムをバイパスしてビデオアダプターを使用して画面に表示されます）、スクリーンショットで写真を取得することはできませんでした。そのため、通信中に対話者の画像が表示される赤い領域にマークを付けました。

しかし、1つの大きな「BUT」があり、徐々にビデオがモバイルデバイスに伝わり、ブレーキは多くの要因に関連付けられました。ビデオをエンコード/デコードするソフトウェア/ハードウェアプラットフォームの準備ではなく、モバイルネットワークがそのようなトラフィックを送信する準備ができているわけではありません（モスクワにはこれに問題はありません-石を投げます:-)、GPRSでSkypeを介してSkypeで普通に話すことができなかった、ビデオについては沈黙しています）、ビデオサービス自体は構成されていませんでした。 もちろん、モバイルデータ伝送の状況は国や地域によって異なりますが、これは一般的なグローバルサービスであり、どこからでもアクセスできることが重要です。

だから、彼らは長い間叫んでいました：「ビデオが来ています。」 テクノロジー、実装などの発表がありましたが、今日の主要なビデオチャットプロバイダーはSkype、Gtalk、MSN、Yahooです。 そして、彼らがビデオをモバイルの世界にどのようにもたらすかはすでに明確です。

そして、ここが最初のti病なステップです-N900でのSkypeビデオ。 すでにGtalkビデオがあったことを考えると、Skype / Gtalk / SIPビデオの非常に強力な束を取得します。 多くの人にとってこれはニュースになりますが、それ以前は、Skypeビデオをサポートしているモバイルデバイスはありません。 N900が最初になりました!!! これは非常に重要なイベントです。 おもしろいことは、3Gビデオ通信の失敗を示唆するかのように、N900は3Gビデオコールをサポートしていないことです:-)。

これが革命の始まりであり、この革命の先駆者はN900です!!!

したがって、2つの革新的なインシデントがあります。モバイルプラットフォームでのクロスプラットフォームQtの到着と、モバイル市場での人気のあるIPビデオチャットの到着です。

UPD：この記事を終えると、iPhone 4Gのプレゼンテーションが行われ、このプレゼンテーションでSteve Jobsがビデオ通信のトピックに関する私の考えを確認しました。 FaceTimeはAppleのソリューションであり、興味深いのは、iPhone 4Gにも3Gビデオコールがないことです。 iPhone 4Gのプレゼンテーションビデオは私のN900よりも一桁優れているように見えますが、いくつかの理由から、N900はこの点で競合他社よりも優れていると思います。

  • FaceTimeはオープン標準を使用していますが、通信標準として普及させるには時間がかかり、Skype / GTalk / SIPはすでに人気があり、すでに使用できます。
  • FaceTimeはiPhone 4Gデバイス間でのみ機能し、N900では、すべてのSkype / GTalk / Jabber / SIPサブスクライバーにビデオコールを発信できます。
  • N900のビデオはどのインターネット接続でも動作します（もちろん、品質は速度に依存しますが、ビデオはトルコのGPRSで動作しますので、コメントに結果を書き留めてください）。 ;
  • N900では、FaceTimeへの呼び出しがサポートされる可能性が最も高く（障害はありません）、iPhoneでのSkypeビデオの外観が問題になります。


ここでは、iPhone 4GとN900の比較分析を行いたくありません。ビデオ通信の開始に関する意見を表明したかっただけです。 私は彼女がすでにあることに気づきたかった！ 私の主な考えは、N900がこの方向のフラッグシップだということです。

4.開発者からのビュー。



ここで、開発者の側から簡単に見てみましょう。 Noka Qt SDKについてはすでに詳細に書いています 。 対処方法、存在する制限、およびそれらを回避する方法についての詳細な指示があります。

基本的に：

  • Nokia Qt SDKは公式の開発ツールであり、テスト中ですが、既に使用できます。 バージョン1.2からのファームウェアのサポート。 （Qt SDKと混同しないでください。QtSDKは、アセンブルされたQtライブラリ、そのソース、およびQt Creatorを含むデスクトップコンピューター上の開発キットです。実際、これはプログラムをすぐにダウンロードおよび作成するためのキットです。 SDKは、Nokiaモバイルプラットフォーム用のアプリケーションを作成するためのツールキットです（今日はSymbianとMaemoです）。
  • Maemo で Qtの公式ドキュメントを入手する必要があります 。
  • Maemo5のQtサンプルはextras-develリポジトリにあります。 パッケージの名前はlibqt4-examples-maemo5です。 詳細はこちら 。


開発者向けのグローバルな変更は、Qt Mobility SDK 1.0.0です。 私がこのSDKの外観を賞賛する理由が本当にわからない場合、または詳細を知りたい場合は、 写真付きの例のドキュメントをお読みください。



5.結論。



一部のユーザーはPR 1.2がもたらしたものに満足していないという事実にもかかわらず、これは本当にノキア向けプラットフォームの開発におけるターニングポイントであり、モバイル市場での大きなイベントです。 市場の発展のダイナミクスは非常に喜ばしく、毎月素晴らしいことが起こります。 そして、楽しいイベントだけが私たち全員を待っています。 モバイルプラットフォーム向けのMeeGoの登場を楽しみにしています。 しかし、今ではMaemo用のアプリケーションの開発を開始でき、QtおよびQt Mobility API以外を使用しない場合、アプリケーションはSymbianおよびMeeGoプラットフォームで自動的に動作します。

ノキアのユーザーと開発者への方向性に非常に満足しています。 新しい、新鮮、良い。 みんなありがとう。

N900にMeeGoがあるかどうかについて。 公式声明によると、公式サポートはありませんが、ほぼ100％の確率で、MeeGoをインストールすることは可能ですが、公式サポートはありません。 これは、Nokiaがデバッグ対象デバイスのプラットフォーム全体を変更せず、MeeGoのリリース後もNokiaがMaemoをサポートし続けることを保証したくないという事実によるものです。プロジェクトでの長年にわたる作業の後、Nokiaがそれをすぐに埋めることは容易ではないことが理解できます。 プロジェクトがすでに道徳的に古くなっている場合、それは一つのことです-もう一つは、現代の競争力のある製品です。 そして、失望した人たちをすぐに安心させることができます。 QtはMeeGoのメイン開発ツールになるため、すべてのアプリケーションはMeeGoとMaemoの両方で実行されます。 そして、次に何が起こるか-将来的に発見するでしょう。 N900は来年も最新のプラットフォームであり続けると自信を持って言えます。



したがって、私は正式に宣言します：「PR 1.2は成功だけでなく、モバイルの世界でも重要なイベントです！」 解説の決闘に同意しない人を招待します。
