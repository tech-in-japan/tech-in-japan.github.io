オフィスをMacに変更します。どのような問題がありますか？

西洋では、中小企業がApple製品を仕事で長く使い続けていることは周知の事実です。 大企業が徐々にMacに移行していることは注目に値します。 例えば、私は最近、HabréでこれがIBMでどのように起こるかを読みました。



ただし、ロシアではPCとWindowsの世界が依然として支配しています。 国内ビジネスの大量スイッチングを妨げるものは何ですか？



この記事は、主に個人的な目的ですでにMacを使用しており、このプラットフォームの魅力と利点を理解しているが、オフィスでの仕事にあえて使用しない人を対象としています。 私は、その活動が貿易やサービスの提供に関連している典型的な中小企業のリーダーを意味します。





問題番号1：適応の負担



私たちはすべてWindowsに精通しています。 今日では、タスクバーとは何か、Microsoft Wordの起動アイコンを探す場所を説明する必要のある従業員を見つけることは非常に困難です。 Mac OSの場合、状況は多少異なります。ほとんどの人がこのオペレーティングシステムについて耳にしましたが、誰もがこのオペレーティングシステムを使用したわけではありません。 ユーザーは、Dockが何であるか、Cmnd + Spaceを使用してRU / ENGレイアウトがオフになることを学習します。 残りを覚えておく必要がないため、Mac OS Xのインターフェースはほぼ完璧に設計されています。 さまざまな業界の企業との協力の経験に基づいて、ほぼすべてのマネージャー（弁護士、秘書など）が1時間以内にMac OS Xの基本ツールを習得していると言えます（紹介ブリーフィングでは5分、残りは自習です）。 しかし、1〜2か月後、Windowsに戻すことは簡単な作業ではありません。 :)もちろん、オフィスにはMacでの作業経験のある人がいるはずです-彼は確かにいくつかの質問に答えなければなりません。 ただし、これにはそれほど時間がかかりません。



結論：各従業員の力によって短時間でMacを習得する。



問題番号2：必要なソフトウェアの不足



国内企業がMacに切り替えられない主な問題は、人気のあるロシアのソフトウェアがWindows専用であることです。 まず、1Cのソフトウェアパッケージ、Garantシステム、および銀行プログラムについて話します。 保証人、特定の銀行ソフトウェア、および1C：会計から逃れることが難しい場合、販売、顧客関係、生産などを管理するシステム それらは、Microsoftの有名な「Access」製品のより強力な類似物であるFileMaker Proを使用して簡単に実装できます。 はい。簿記と法務部門は、PCに落ち着くか、 ParallelsまたはVMWare Fusionを使用してMacで作業する必要があります。 しかし、ほとんどの企業のこれらの部門は、ユーザーの総数の20％以下を占めています。 したがって、残りの80％はMac OS Xで非常に快適に動作します。



一般的な市販のWindows製品に対するFileMaker Proソリューションの利点は次のとおりです。

  • WindowsおよびMacのサポート、およびWebベースのインターフェースを介したアクセシビリティ。
  • アプリケーション開発のシンプルさ-FileMakerで作業するためには、プログラマである必要はまったくありません。 頭脳と基本的なコンピュータースキルがあれば十分です。
  • ソリューションを企業のニーズに適合させる-FileMaker Proに基づく複雑な個々のプロジェクトは、会社の特定のタスクを解決し、余分なものは一切含まれません。
  • システムパフォーマンスの本当に簡単な統合とサポート。
FileMaker Proベースのソリューションは、Webサイト、1Cデータベース、およびその他のソフトウェアと対話できます。 たとえば、FileMaker Proプラットフォームで作成されたアプリケーションを使用して、オンラインストアの品揃えを簡単に更新できます。



結論：ほとんどの場合、ソフトウェアの問題は解決され、ソリューションは多くの標準開発の有効性を上回ります。



問題番号3、どこにいますか？



3番目の問題を見つけようとしましたが、できませんでした。 プライベートスイッチングのすべての問題は、インターネットで100回以上議論されました。私は、再びハッキングされたトピックに夢中になりたくありません。



結論：多くの問題はありません。 お金を節約します。 :)
