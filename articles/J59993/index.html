「サイトの費用はいくらですか？」という質問に答える方法

これはおそらく、潜在的な顧客から寄せられる最も一般的な質問の1つです。



最初の会議では、車のようなサイトは非常に異なることをクライアントに説明します。 そして、価格はそのニーズと財務力に大きく依存します。



しかし、TKがまだ書かれておらず、クライアントが何を望んでいるかが完全に明確でない場合、最終的な金額を指定することは非常に困難です。

契約が最終的に署名されるかどうか確信が持てないので、詳細で優れた作業明細書を書くことは有益ではありません。 時間の無駄だからです。 時は金なり。

それは両刃の剣になります。



この問題の解決方法：

（私たちは平均的なウェブスタジオについて話している）



TK最初-次に契約
  1. クライアントは、サイトの開発に関するかなり詳細なアンケートに記入します。
  2. アンケートと個人的な会話に基づいて、予備的な計算が行われます。
  3. クライアントのおおよその金額が満たされた場合、作業明細書の準備に進みます。 サイトの場合は1日ですべてをペイントすることが可能であり、サイトの場合は顧客と密接に通信するのに2週間かかることを理解しています。 サイズとニーズに応じて。
  4. 作業明細書の準備が整ったら、再計算を行い、何かが削除され、金額がクライアントにとって大きすぎる場合は、契約に署名して作業を開始します。
この回路は顧客にとって理想的です。 すでに署名の段階にある彼は、自分のお金で受け取るものと、自分のサイトがどのようになるかを明確に理解しています。

しかし、請負業者にとってはあまり便利ではありません。条項3以降のすべての人が契約して作業に同意するわけではありません。

そして、TORの起草に費やされた時間は返されません。



最初の契約-TK
  1. 可能な「落とし穴」を最大限に特定するために、彼らはさらに詳細なアンケートを作成しました
  2. アンケートに基づいて、より正確な金額を請求します。
  3. 次の性質の条項を含む契約に署名します。

    -参照条件の準備と署名の後、それは契約の不可欠な部分になります

    -追加の合意により、追加の見積もりを作成できます。 働く
  4. TKをより慎重かつ思慮深く準備します（クライアントは既に「私たち」です）。
スタジオにとっての利点は明らかです。プロジェクトに関するすべての作業は有料です。

しかし、顧客はそのようなスキームを好まない。なぜなら、彼は彼が何を支払うのか、そして彼が最終的に受け取るものを理解しないからである。



技術 プロジェクト-契約-TK
  1. すべて同じプロファイル+個人的な会議。
  2. おおよそのコストを表明しました。 満足したら-ステップ3に進みます
  3. アンケートと会議に基づいて、1〜2日でドキュメントを作成します。「技術プロジェクト」は、 非常に一般的な用語で、将来のサイトの本質を説明します。
  4. 必要に応じて、合計金額を変更して同意します。
  5. 前の契約と同様の契約に署名しますが、次のものが追加されます。

    -技術タスクは、技術プロジェクトの一部として作成され、矛盾することはできません。
  6. さらにTKとフォワード:)
顧客はこのスキームを気に入っていますが、私たちはそれがあまり好きではありません。

とにかく、仕事を始める前にすべてを予測することは困難です。 また、顧客は追加料金と再計算に同意することを非常に嫌います。
