PuPHPetを使用した簡単な仮想マシンアセンブリ



開発環境を作成して管理すると、動揺したり、開発が遅れたり気が散ったりしますか？ ローカル環境と展開時の環境が異なるため、問題が発生していますか？ もしそうなら、私はあなたのための解決策を持っています-PuPHPet ！



はじめに



自信を持って言うことはできませんが、開発中の私のお気に入りのコーンの1つは、環境です。さまざまなニーズのさまざまなプロジェクト用にそれを作成および保守し、開発チーム（特に分散）の相互作用を確保し、開発、テスト、および戦闘環境。 これはすべて、特に手動で行う場合、時間がかかるタスクになる可能性があります。



管理者またはクライアントの要件に応じて、さまざまな選択肢が気のめいることがあります。 それらはすべて異なる方向に引っ張ることができます（そして引っ張る！）、しばしば同時に。



たとえば、MySQL、PostgreSQL、Oracle、またはSQLServerデータベースの選択肢を考えてみましょう。 Cassandra、Hadoop、RedisなどのNoSQLソリューションと同様。 または、ApacheやNginxなどのサーバーの選択。 リストは延々と続く。 そして、常に支援する準備ができているシステム管理者（フリーランサーや非常に小さな開発チームなど）がいない場合、開発環境の維持に費やされる時間が大幅に増加し、DevOps *に変わります。



最終的にこの状況を終わらせ、プロセス全体を可能な限り効率的にするために、 VagrantとPuppetの研究を始めました。 同僚との会話とインターネットでの徹底的な検索から、これら2つのツールが実際に再利用可能な環境を作成するための標準であることが明らかになりました。 これらの優れたツールに慣れていない場合、それぞれのマニュアルからの2つの簡単な抜粋を以下に示します。



放浪者：

軽量、ポータブル、およびポータブルの開発環境を作成および構成するように設計されています。 Vagrantは要件に応じて変更されます。


人形：

冗長性と構成から相互作用とレポート設定まで、ライフサイクル全体でインフラストラクチャ管理を自動化するソフトウェア。 Puppetを使用すると、繰り返し使用するタスクを簡単に自動化できます。たとえば、ミッションクリティカルなアプリケーションをすばやく展開し、変更をアクティブに管理し、クラウド内の独自の機器を使用して10から1000サーバーに迅速に拡張できます。


しかし、現代世界のほとんどの人々と同様に、私はイライラしています。 私は、上記の問題を非常に迅速に解決する方法がなければならないことを知っていましたが、情熱的なファンになることはありませんでした。



PuPHPetの概要



さらにググリングした後、Vagrant用のPuppet設定を簡単に作成できる優れたGUIツールであるPuPHPetに出会い、すぐに使用を開始できます。



当初、PuPHPetは、主にPHP開発者向けの作業環境を作成するための非常にシンプルなツールとして、Juan Treminioによって考案されました。



5つの方向に焦点を当てています。

  • 展開（メモリ、IPアドレス、ポート転送）
  • 基本的なサーバー設定
  • PHP（モジュール、ライブラリ、構成設定）
  • データベース（PostgreSQL、MySQL）
  • Webサーバー（Apache 2、Nginx）


PostgreSQLとNginxを併用したPHP 5.5、またはApache 2とMySQLを使用したPHP 5.4が必要な場合は、必要な構成をすばやく作成して読み込むことができます。



PuPHPetを使用すると、30分で環境の準備が整うので、コマンドを実行するだけで済みます。 

vagrant up


ここにある。 解凍されたPuPHPetのあるフォルダーでは、このコマンドを使用する必要があります。少し待ってください。 信じられない？ 最初から最後まで、プロセス全体をステップごとに見ていきましょう。



終了したら、断固としてあなたに尋ねます-構成の変更、オプションの変更、パッケージの追加と削除、そしてあなたが見つけることができるものすべてを恐れないでください。 きっと感動するでしょう。



必要なソフトウェア



始める前の簡単なメモ。 c PuPHPetに最初に出会ったときに出会った最大の問題の1つは、VagrantとVirtualBoxの適切なバージョンとの互換性の問題です。



現在、 Vagrant 1.3.5とVirtualBox 4.3.2を使用していますが、問題なく動作します。 開始するには、これら2つのパッケージをインストールします。



構成を作成する



それらがインストールされたら、 puphpet.comを開きます 。 最上部には、ローカル（ローカルコンピューター上）、デジタルオーシャン、ラックスペース、およびアマゾンウェブサービスの4つの構成オプションがあります。 [ローカル]を選択します。他のオプションは考慮しません。



[ローカルVMの詳細]セクションで、次のオプションを選択できます。

  • オペレーティングシステム
  • ホスト名
  • IPアドレス
  • RAMサイズ


この例では、VirtualBox 4.3でDebian Wheezy 7.2 x64を使用しているため、PHP 5.4または5.5を使用できます。 残りの3つのオプションはそのままにするか、必要に応じて変更できます。



次は、ローカルVM転送ポートセクションです。 ここでは何も触れませんでしたが、必要に応じて、ローカルマシンから仮想マシンへのポート転送をここで構成できます。







フォルダー共有



ここで、VagrantとPuppetはすべての力と柔軟性を示し始めます。 Box Sync Folder Sourceでは、Box Sync Folder Targetで指定された仮想マシンのディレクトリと同期されるローカルディレクトリを指定する必要があります。 共有フォルダータイプのフォルダーのタイプと同様に、デフォルトでこのアイテムを残しました。



このオプションは、仮想マシンに複数のユーザーディレクトリを作成する場合に便利です。 〜/ Documents / workspace / your-projectにプロジェクト作業フォルダーがあるとします。 このディレクトリを仮想マシンのWebサーバーのルートに直接マップできます。 したがって、仮想マシンのプロジェクトの作業フォルダーに同期または手動でコピーする必要なく、ローカルで開発できます。







基本的なサーバー設定



Git、subversion、Vimなどの特定のパッケージが動作する必要がある場合は、このセクションにリストする必要があります。 唯一覚えておくべきことは、選択したオペレーティングシステムと同じ方法でパッケージ名を指定する必要があるということです。







Webサーバー



ApacheまたはNginx？ 次に、Apacheを選択します。 次に、フィールドに「Apache Modules」と入力して、インストールおよび有効化するモジュールを指定する必要があります。 モジュール名の入力プロセスを容易にし、エラーを回避するために、入力フィールドには使用可能なオプションのポップアップリストが表示されます。



次は仮想ホストの構成です。 サーバー名とエイリアスのフィールドは、必要なドメイン名を示します。 この設定を機能させるには、ローカルコンピューターのホストファイルで、対応するIPを使用してこのドメインのエントリを追加する必要があります。 [ドキュメントルート]フィールドで、Box Sync Folder Targetと同じディレクトリを指定して、同期を簡単かつ美しくします。 特別な必要がない場合は、フィールドにポート80を残します。 デフォルトで。



残りの2つの設定、環境変数とAllowOverrideは、アプリケーションで必要な場合にのみ変更されます。 （レーンに注意-私はこれら2つのオプションの目的をよく理解していませんでした。誰が知っているのかを明確にしてください）。







Php



多くのオプションはありませんが、最も頻繁に必要とされる主要なオプションのみが存在します。 デフォルトでPHP 5.5を終了しました（現在、この下のバージョンを見る価値はほとんどないと思います）。 Composerのインストールにも注意しました。 Composerを使用していない場合、またはComposerが必要かどうかわからない場合は、Composerをよりよく理解することをお勧めします。



[INI設定]セクションで、既に選択されているオプションを残し、allow_url_fopen、allow_url_include、error_log、file_uploads、apc.enabledを追加しました。 同じことを自由に行うか、詳細なリストから参照して選択してください。



PHPタイムゾーンを自分の場所に最も近いヨーロッパ/ベルリンに変更しました。 同じことを行い、必要なタイムゾーンを選択することをお勧めします。 PHPモジュールでは、cli、intl、mcrypt、cgi、curl、memcached、memcache、pspell、tidy、sqliteのリストを使用しました。 リストを参照して、必要なものを追加します。 PEARおよびPeclモジュールについては、すべてをそのままにしておきました。



Xdeprof（デフォルト）をインストールしましたが、Xhprofはインストールしませんでした。 XDebugを使用することも強くお勧めします。







データベース



だから、今慎重に。 この例では、MySQLを選択し、rootのパスワードを設定し、すべてのユーザーに特権を残しました。 SitePointデータベース名、TestUserユーザー名、およびtestpasswordパスワード。 この例にはSQLインストールファイルがないため、最後のフィールドは空のままにします。

さて、大きなボタンをクリックしてください。 ダウンロードが開始されたことがわかります。 完了したら、アーカイブを解凍し、ターミナルのディレクトリに移動します。 そこから、vagrant upを実行します。
