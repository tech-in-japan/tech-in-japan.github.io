SSL Explorer

私は海の愛人になりたかったのです...まあ、つまり、私は自分のホームネットワークをどこにいても管理できるようにしました（もちろん、インターネットアクセスを備えています）。 別のコンピューターのリモート制御は簡単に対処できます-リモート管理には多くのソリューションがあります...

しかし、自宅には3台のコンピューターがあり、面倒や心配なしにいつでも各コンピューターの情報にアクセスしたいと思っています。

だから-VPN。

クライアントアプリケーションがなく、どのシステムの下からでも、それがどれほど難しいかわからないことが望ましいです。

だから-クライアント側の有名なソリューションは一掃されます。

望ましいのは、一般的に使用されるポート（他のポートは簡単ではない場所の管理者が閉じることができる）を介して、暗号化が損なわれないようにすることです。

だから-...

この種の決定の主題についてインターネットで大騒ぎして、私は自分自身の主題を発見しました。

SSL-Explorerです。 3SP Ltdが開発した強力なVPNサーバーは、JavaベースのGNUライセンスで配布され、SSLプロトコルで実行され、Webインターフェースを介して制御されます。キャラクターは北欧です...おっと... :)一般的に、私はここに小さなもの、完璧な私のニーズに！ それは判明しました-あまりにも多くの、私がおそらく必要としないような機能をたくさん持っています...しかし、他のものが役に立つかもしれません。



以下は彼が知っていることと持っていることの一部です。

SSL-Explorerは、 OpenVPN （ IPSecに基づく）などのVPNトンネルを介して単一の仮想ネットワークを編成するように設計されていないことに注意してください。これは、永続的なトンネルを確立するのに適しているため、他のタスクに使用されます。 これは、任意の便利な場所からネットワークリソースをリモート管理するための優れたツールであり、その任務を見事に実行します（ところで、これらの目的で必要な場合は、独自のユーザーデータベースを作成するときに指定し、OS認証ポリシーを使用しないことをお勧めします） ）



プログラムのすべての機能の中で、ユーザー権限の柔軟な管理（友人がグリッドで何かを開き、それぞれに自分自身にアクセスできるようにする）、グリッド内の任意のコンピューターのデスクトップへのリモートアクセス、任意のネットワークコンピューター上の便利なフォルダーボールに興味がありました（リソースショートカットはお気に入りに追加して、簡単にアクセスできます）。 サーバーで電子メールおよびSMS通知を構成できます。 また、プログラムのインストール後に開かれたコントロールパネルインターフェイスは、単に私を魅了しました。 Solid Web 2.0、すべてがよく考えられ、メニューは直感的で、すべてのブロックは状況依存であり、可能な限り情報量が多いため、十分に見ることができません=）1つのマイナス（個人的にはマイナスではない）に気づきました：言語の選択には正式な「ロシア語」オプションがありますが、しかし、私はこの選択で英語のインターフェースとの違いに気づきませんでした。 何も、私たちは誇りに思っている人ではないので、それは降りてくるでしょう）

サーバーのインストールは直観的であるため、段階的にペイントすることはありません。

インストール後、インストールプロセス中に作成されたルートの下でローカルアドレスlocalhost ：443のコントロールパネルに移動し、使用するグループのさまざまなポリシーを作成する必要があります：たとえば、自分自身、あなたの最愛の人、友人、ゲストのために（ルートを無効にすることを忘れないでくださいsecurity）...各ポリシーで、それに属するグループと個々のユーザーに何ができるかを決定できます。 特に、これは最も重要なポイントの1つです。対応するユーザーが使用できるプログラムのショートカットを作成できます（また、作成する必要があります）。 そして、私たちが提供する可能性のあるSSL-Explorerプログラムは非常に多くあります-これらはいわゆる拡張機能であり、xml-configsの助けを借りて管理します。 SSL-Explorerの作成者は、独自のリモートコンピューター管理ツールを開発する代わりに、 PuTTy 、 Linux RDesktop 、 Windows RDP 、 UltraVNC 、 Radminなど、最も身近で便利な方法をユーザーに選択できるようにしました。 もちろん、選択が完了したら、プログラムを事前に構成する必要があります。

必要に応じて、ソースコードから自分で複雑なものを構築することができます。また、独自の拡張機能を構築することもできます：人気のあるTightVNCをねじ込む例の手順 。 ところで、最新バージョンのTightVNCサーバーでは、自分で追加する必要はないようです-利用可能なプログラムのリストにあります！

入力した場所からサーバーに接続したら、お気に入りのツールのショートカットをクリックするだけで、LAN内のコンピューターにアクセスできます。 また、他のユーザーにこのような幅広いアクセスを提供する予定がない場合は、ネットワーク上の映画フォルダーと音楽フォルダーに制限できます。
