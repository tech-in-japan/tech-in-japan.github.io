Kotlin for Androidで書く

約2年前、Androidマーケットには、さまざまなプロジェクトに費やされた時間を追跡するのに便利なアプリケーションがないことを発見しました。 本物のアイドラーのように、必要なものを検索するのではなく、自分で作成することにしました。 当初よりも少し複雑であることが判明し、 私はすべてを緩慢な趣味モードで放棄しました。私の欲望の具現化は、アイデアからアルファ版にゆっくりと移行し、1年以上生きました。



そして、突然、善良な人々は、Kotlin'e上のAndroidでの書き方を教えてくれることを提案しました。 これは良い兆候であるように思えたので、Kotlinでアプリケーションをすばやく書き直し、終了してGoogle Playに掲載しました。



重要な説明：この記事は、Kotlin'eでAndroid向けに作成する方法についてのみ説明しています。 Kotlinを試してみたい人、そしてKotlinを試してみたい理由を理解したい人だけが興味を持ちます。 私が書いたアプリケーションは非常に単純です（それ自体、JetBrainsにはまったく適用されません）。 彼のコードで何かを改善できるという保証があります。それは例としてのみGithubに投稿されており 、驚くべきコードのサンプルとしてではありません。 動作します 。試してみたい場合は、ここから配置できます 。 しかし、繰り返しますが、これはデモであり、その目的は完璧なアプリケーションではなく、Kotlin for Androidの紹介です。



繰り返しますが、何か新しいものです。 そして、なぜ私はそれを知る必要がありますか？







最も興味深いものを遅らせないために、Kotlin'aの最も重要な利点のみに言及します。

  • コンパクトなコード。 すばやく書き込み、読みやすく、エラーが少ない。
  • 組み込みエラー保護、まず第一に、null許容性、つまり、プログラマーが特定の変数がnullになる可能性があることを明示的に示す要件。
  • 開発のしやすさ（私の意見では、初心者のJava開発者でも簡単にKotlinをマスターできます）。


Kotlin'eでは、Jav'eよりもはるかに高速に書くことができ、私の意見では、コードはより美しいです。



ちなみに、github'eには、Kotlin'eのコードを含む200以上のリポジトリが既にあります。モバイルアプリケーションについて言えば、KotlinはTelegramメッセンジャーの開発に使用されました。



それでは、どこから始めますか？



動作するには、IDEが必要です。 明らかな理由から、私はIntelliJ IDEAを好みますが、JetBrainsはEclipse用のKotlinプラグインも作成し、vimの愛好家は別の言語コンパイラを使用できます。



さらに、AndroidでビジネスでKotlinを試してみたい人は誰でもIntelliJ IDEAを使用すると仮定します。 動作させるには、Kotlinプラグインをインストールし、コンピューターにgradleとAndroid SDKがあることを確認する必要があります（これはKotlin'yとは関係ありません。これはAndroidの開発に必要です）。



Javaが最初でした



完成することにしたとき、私はすでに部分的に書かれたアプリケーションを持っていました。 すべてがJavaで書かれています。



アプリケーションの本質：画面に赤いタスクのリストがあり、タスクをクリックすると緑色に変わり、時間が経過した：これはユーザーがタスクに取り組んでいることを意味します。 彼が停止したとき、彼は再びそれをクリックし、タスクが再び赤面し、彼がどれだけの時間を費やしたかを示します。 アプリケーションは、何もチェックせずに盲目的にユーザーを信頼します。これは、oDeskの「兄貴があなたを見ている」ということではありません。



同じプロジェクトでJavaとKotlinの両方のコードを使用できるという事実にもかかわらず、私がJavaで書いたものをKotlinに書き直したかったのです。私の魂はKotlinの100％コードを要求しました。 継承されたコードを使用すると、すべてが単純であることがわかりました。src / main / javaの次に、 src / main / kotlinフォルダーを作成し、その中にクラスを配置します。 Kotlinのファイルは.javaではなく.ktで終わります。 ここで、楽しい瞬間が整ったことが判明しました。Kotlinは、「1つのクラス-1つのファイル」という対応を必要としません。 アプリケーションロジックに従って、好きなだけ多くのクラスを1つのファイルに詰め込むことができます。 私のアプリケーションでは、データベースの操作と画面上のユーザーインターフェイスの2つの論理部分だけがあり、ファイルの数を減らすことができました。



IntelliJ IDEA用のKotlinプラグインは、JavaからKotlinへの適切な変換を実行することにより、 .javaファイルを.ktファイルに変換できます。 これは、ファイルとコンテキストメニューを右クリックするか、Javaコードを.ktファイルにコピーするときに行うことができます（プラグインは変換できるかどうかを尋ねます）。



Kotlin'eにコードを書きます



Kotlinコードはコンパクトです。たとえば、コンストラクターとゲッター/セッターを含むクラスは次のように記述されます。 



class Item (val project: String, var status: String, var secondsSpent: Long, var lastactivated: Long)




ボタンをクリックしてテキストを入力するためのダイアログを作成するコードを見てみましょう。



Java： 

 addButton.setOnClickListener(new OnClickListener() { public void onClick(View v) { AlertDialog.Builder alert = new AlertDialog.Builder (this); alert.setPositiveButton("OK", new DialogInterface.OnClickListener() { public void onClick(DialogInterface dialog, int which) { result = input.getText().toString() } }); } }




コトリン： 

 addButton.setOnClickListener() { var alert = AlertDialog.Builder(this); alert.setPositiveButton("") { dialog, whichButton -> result = input.getText().toString() } }




同意して、Kotlin'eのコードは読みやすいです。 注意深い読者にとっては、これはもちろん、ダイアログを作成するときに[OK]ボタンだけでなく[キャンセル]ボタンも必要なため、これはコードの一部にすぎないことに注意してください。 コードはGithubに完全にアップロードされます。



コードを短くできるのはなぜですか？ たとえば、Kotlinでは、この形式の記述が可能です。引数として、1つの抽象メソッドを持つクラスのインスタンスが必要な場合は、そこにラムダを単純に渡すことができます。これは上記の例に示されているとおりです。



Kotlin'eでは、代入の右側から推測されるため、変数の型を省略できることに注意してください。



Java コトリン
AlertDialog.Builder alert = new AlertDialog.Builder（this）; val alert = AlertDialog.Builder（これ）


Kotlin'eの構文の興味深い機能：パラメーターが関数に渡され、最後のパラメーターが関数である場合、角括弧から外すことができます。 これはまさにフラグメントに見られるもので、ラムダがsetPositiveButtonに渡され、OKボタンを押すことで機能します。 



 alert.setPositiveButton("") { dialog, whichButton -> result = input.getText().toString() }


同じように書くことができます 



 alert.setPositiveButton("", { dialog, whichButton -> result = input.getText().toString() } )


読みやすいオプションを自由に選択できます。



KotlinコードでのJavaライブラリの使用


もう1つの良い点は、KotlinコードからJavaで記述されたライブラリを直接使用できることです。



たとえば、SQLiteで保存するときにプロジェクト名の特殊文字とUnicodeを適切に処理するために、一般的なライブラリのStringEscapeUtils.escapeJavaおよびStringEscapeUtils.unescapeJava関数を使用しました。 これらの関数はimportステートメントでインポートするだけです（プロジェクトプロパティに入力します）。 



 import org.apache.commons.lang3.StringEscapeUtils


戻り値


Kotlinでは、関数リテラルは最後の式で計算された値を返すため、関数リテラル（別名ラムダ関数）でreturnを使用することは禁止されています。 そのため、 setOnTouchListenerの呼び出しでは、呼び出しの最後のパラメーターはbooleanを返す関数であることが期待されます。 この値は、実際にgestureDetector.onTouchEvent（aEvent）を返します。



Java： 

 gestureListener = new View.OnTouchListener() { public boolean onTouch(View v, MotionEvent event) { return gestureDetector.onTouchEvent(event); } };


コトリン： 

 getListView()?.setOnTouchListener() { v, aEvent -> gestureDetector.onTouchEvent(aEvent) }


Kotlin関数リテラルコードでreturn GestureDetector.onTouchEvent（aEvent）を記述することはできません。gestureDetector.onTouchEvent（aEvent）を呼び出した結果はこの方法で返されます。



実際、関数リテラルで非ローカルリターンを使用する必要がある特殊なケースがあります。 興味がある場合、詳細についてはKotlinのドキュメントを参照してください。




「=」記号で定義された関数による値の戻りは、次のように同様に機能します。 



 override fun getCount() = data.size()


文字列パターン


読み取りが難しく、最適でないバイトコードを与える長い文字列を取り除くために、Kotlin'eで文字列テンプレートが使用されます。 



 setMessage(" $projectToDelete?")


ここで、projectToDeleteは文字列変数です。 このようなパターンの構文は、たとえばUNIXで環境変数を扱っている人なら誰でも知っています。 特に便利なのは、テキストと多くの変数の値で構成された文字列用のテンプレートであり、書式設定も可能です。 



 secondsSpent.setText("${format("%02d", hours)}:${format("%02d",minutes)}:${format("%02d", seconds)}")


ちなみに、興味深い話がフォーマットを思い付きました。KotlinのStringのタイプは独自のものであり、 フォーマットメソッドはありません。したがって、 java.lang.String.formatを明示的にインポートし、そのように参照する必要がありました。 驚くべきことに、ところで、JavaまたはKotlin'eには、秒の整数をフォーマット文字列に変換するsecondsToHumanReadableStringメソッドがまだありません。 
  HH：MM：SS。


いつ、と


複数選択の場合 、およびメソッドとプロパティの呼び出しを減らす場合に使用すると非常に便利です。どちらもAndroid用の小さなアプリケーションでも十分です。 私の場合、これは読みやすいコードを提供しました。例えば： 



 with (alert) { setPositiveButton("") { dialog, whichButton -> ...      ... } setNegativeButton("") { dialog, whichButton -> } //  ""      create() show() }


を使用しない場合、 alert.setPositiveButton、alert.setNegativeButton、alert.create、alert.showは判明します。 読みやすさも向上する場合： 



 when (item.status) { "active" -> { item.status = "inactive" ...      ... } "inactive" -> { item.status = "active" ...     ... } }


Gradleでコンパイルする



build.gradleを手で作成したり、IDEなしでgradleでプロジェクトをビルドしたりする場合は、 Kotlinlang.orgで 友人をgradle 、Kotlin、Androidにする方法を読んでください 。



Android Studioユーザーにこのような問題があるかどうかはわかりませんが、Gradleで作業しているIntelliJ IDEAに精通している人にとっては、デバッグではなくリリース用に.apkをアセンブルする方法が問題になる場合があります。



このため、IDEAにはGradleプラグインがあり、右側のタブをクリックすると開きます。







デフォルトでは、IDEAは* -debug-unsigned.apkを収集します 。 Google Playに置くことができないもの。 * -release-signed.apkにするには、署名キーを生成してキーストアに配置し、プロジェクトのbuild.gradleに数行を入力して、Gradleがキーを取得する場所を知る必要があります。 リリースをビルドするには、GradleウィンドウでassembleReleaseタスクをダブルクリックして選択します。



キーを作成してアプリケーションに署名する方法について既に知っているか、stackoverflowで詳細を読むことができます 。
