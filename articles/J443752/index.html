Androidアプリ開発の未来としてのKotlin





バージョン1.0のリリースからわずか数年が経過し、この言語は多くの大企業や開発者に採用されました。 言語の作成者がパスの最初に設定した目標を達成することができたという事実に感謝します-Javaより便利で安全であるが、Scalaより複雑ではないツールを開発するため。



Android開発部門GeekUniversityのプログラムで、最近Kotlinを学習するためのコースを追加しました。 今日は、その機能と、この言語が将来のAndroidアプリケーション開発の柱の1つと見なされる理由について説明することをお勧めします。 ちなみに、3月14日にウェビナー「 Android開発におけるKotlinの展望と可能性 」を開催しました。トピックに興味があれば、それをチェックしてください。 それまでの間、Kotlinの可能性と展望に対処しましょう。



したがって、言語自体は、初心者プログラマーと経験のある開発者の両方にとって便利なツールです。 非常に簡潔で、コンパクトで理解しやすいです（ほとんどの場合-直感的であっても）。 時間が経つにつれて、言語は改善されます。



最新バージョンは1.3で、コルーチンをサポートしているため、ノンブロッキング非同期コードを問題なく読み書きできます。 スケーラビリティも利用可能になり、Kotlin / Nativeのベータ版が登場しました。これにより、Kotlinコードを直接マシンにコンパイルできます。 これにより、たとえば、クロスプラットフォームでビジネスロジックを記述できます。



Kotlinは、上記のすべてのおかげで、まず第一に、今日よりも関連性が高くなっています。 第二に、2017年にGoogleがKotlinを「家族の中に」連れて行き、Androidアプリケーションを開発するための公用語として認めたためです。 GoogleがKotlinを認識した後、元々Javaで書かれた多くのアプリケーションはもちろん、Javaで書かれた多くのアプリケーションがKotlinで書き直されました。



多くの開発者がKotlinにAndroid開発の未来があると考えるのはなぜですか？



簡潔さ



これは、開発者ができるだけ早くKotlinに切り替えることを促進する要因の1つです。 同じJavaは、その汎用性にもかかわらず、この言語に比べて非常に冗長に見えます。 まあ、書く必要が少ないほど、間違いを犯す可能性は低くなります。 また、速度の向上を意味します。 もちろん、依存関係は常に直接的なものではなく、他の要因もありますが、多くの場合、Kotlinでの同じタスクはJavaよりも速く解決されます。



オープンソース



はい、2012年以降、言語は公開されています。 これは、言語の改善を支援する開発者の広範なコミュニティを意味します。 コミュニティはコトリンの従業員とつながりがあり、フィードバックは非常に信頼できます。 これは主に、 Kotlin Slackチームの支援によるものです。 Kotlinの代表者は、サードパーティの開発者の意見に耳を傾け、言語を改善できる場合は、しばしば提案を実施します。



コトリンは長い間「熟した」



新しいプログラミング言語に固有の問題のほとんどが修正されているという意味で。 現在では、すべてのルールに従って記述された、コンパイラまたはアプリケーション自体の奇妙な動作という形でキャッチを期待できない信頼性の高いツールです。 さらに、Kotlinは優れたIDEサポートを備えています。 冗談じゃない、本当に素晴らしい。 IDEプラグインは問題なく機能し、Java固有の便利な機能がいくつか追加されます。



後者の場合、Kottlinに付属のIntelliJを使用することをお勧めします。 ちなみに、IntelliJはすでにHabrで賞賛されていました。これは主に、開発者が非常に有能に作成に近づいたためです。 たとえば、IntelliJは、JavaコードがKotlinファイルに挿入され、それをKotlinにブロードキャストするかどうかを「理解」します。



全体として、Kotlinは非常に信頼性の高いプログラミング言語であり、多くの機能を備えています。 同時に、彼は常に改善しています。







更新可能性と開発



完璧には本当に制限はありません。 JetBrainsは、Kotlinの機能を改善および拡張する非常に良い仕事をしています。 執筆時点の最新バージョン（1.3.21）では、マルチプラットフォームプロジェクトに関連するものを含め、Gradleのプラグインに多くの改善が加えられました。



ちなみに、Kotlinは以前のバージョンのAndroid APIに対して、新しいバージョンのJavaと同様の機能を提供します。 希望から-ktor Webフレームワークがアルファから出て大きなコミュニティに成長し、Kotlinがfullstackツールにさらに近づいたらいいのにと思います。



ヌルセキュリティ



これは以前のすべてのものよりも技術的なポイントですが、言うまでもありません。 事実、Java自体は、String型の変数にnullではなく文字列が含まれていることを保証できないということです。 はい、開発者は長い間適応しており、これは大部分は問題ではありませんが、静的型付けによって提供されるセキュリティはこれを減らします。 その結果、Java開発者はNull Pointer Exceptionsについて準備する必要があります。



Kotlinにはこのような問題はありません。ここでは、タイプは非ヌル（これらのタイプの値をnullにすることはできません）とnull可能（nullにすることができます）に分けられます。



Kotlinに書くのは簡単です



すべてが平等であるため、Kotlinで書くのはJavaやScalaで書くよりも簡単です。 Kotlin Androidアプリケーションに必要なライブラリは小さく、開発者の生活を複雑にすることはありません。



さらに、KotlinとAndroid Studioは、開発プロセスを高速化する素晴らしい組み合わせです。 プロジェクトの構築には数分しかかかりませんが、システムは、欠落しているコンポーネントを再インストールする必要があるという警告を出すことがあります。



いつでも、コード、データベース、コマンド、ユーザーインターフェイス要素、およびその他の場所で検索できます。 多くの場合、これは非常に便利です。 たとえば、ある種のIDEコマンドを忘れた場合。 デバッグも問題なく実行されます。



Kotlinは簡単に習得できます



これは論理的な言語であり、学ぶのは楽しいことです。 さらに、記述されたコードは、アプリケーションの作成者だけでなく、Kotlinに精通していないサードパーティのプログラマーでも読みやすいです。



これはすべて、Kotlinの開発者であるJetBrains社が元々、この言語をJavaに代わるものとして考えていたおかげで実現しました。 Kotlinを使用すると、プログラマーはさまざまな方法で問題を解決できるため、トレーニングだけでなく作業も面白くなります。



開発者にとって、世界最大の企業が徐々にKotlinと協力し始めていることも重要です。 とりわけ、これらはPrezi、Basecamp、Amazon Web Services、Pinterest、Coursera、Netflix、Uberおよび他の多くの企業です。 つまり、Kotlinを知っているプログラマーは、優れた職場と、彼が初日から没頭できる興味深いタスクを期待できます。 また、Kotlinを使用する企業の数が徐々に増えているため、Kotlinスペシャリストの需要が高まっています。



一般に、Kotlinは開発者にとって使いやすい言語であり、非常に幅広いタスクを実行できます。 コミュニティの成長率と、グーグル自身がドキュメントのサンプルを徐々にKotlinに翻訳しているという事実を考慮すると、Androidの開発の観点から言語の見通しについてはまったく疑いの余地はありません。 彼はまた、他の開発分野にも展望を持っています。
