LocoLaser：アプリケーションをGoogleスプレッドシートに翻訳する





実際には、AndroidとiOSで、場合によってはWindows Phoneですぐにアプリケーションをリリースする必要があります。 この場合、一部の開発者はローカライズの問題を事前に解決し、各プラットフォームを個別に翻訳します。 同意する、最高の旅行ではありません。 この問題を解決することを決めた当時、ネットワーク上で、Googleスプレッドシートから文字列リソースをダウンロードするためのユーティリティを見つけることはすでに可能でしたが、それらのほとんどは少なくとも不器用に見えました。 私は絶対にこの状況が好きではありませんでした。 その結果、LocoLaserが誕生しました-使いやすく、しかし非常にスマートなJavaのユーティリティです。 数年の間、私はそれを仕事のプロジェクトでのみ使用していました。 この間、彼女はなんとか豊富な機能を成長させることができ、今では一般に公開する準備ができています。



この記事では、LocoLaserとは何か、それをプロジェクトに統合する方法について説明します。 まず、ユーティリティの機能を見てから、Android用のGradleプラグインやiOS用のBashスクリプトなど、より具体的なものに進みます。 私は時々同じことをするのに十分な怠zyなプログラマーなので、初期セットアップの後、必要なときに必要なタスクを起動するだけで済むようにすべてが行われます。 翻訳者にとって、翻訳は非常にシンプルに見えます。Googleテーブルを行と共有し、適切な列に翻訳を入力します。 言語を追加する必要がある場合は、新しい言語で別の列を追加するだけです。 Googleスプレッドシートを使用すると、表を編集するアクセス権を個別に設定できるため、各翻訳者はそれぞれの言語の列にのみアクセスできます。

資格証明ファイル



Googleスプレッドシートにアクセスするには、OAuth認証を行う必要があります。 パラメーター"source.credential_file"



構成ファイルに設定されていない場合、ローカライザーが起動すると、Google認証ページのあるブラウザーが自動的に開きます。 認証に成功すると、ユーティリティは作業を続行します。 次回、このテーブルにログインする必要はありません。



上記のアプローチはユーザーの介入を必要とするため、ローカライズプロセスが自動的に行われる状況には適していません。 プロセスが常に自動的に実行されるようにするには、 credential_file



指定する必要がありcredential_file



。 資格情報ファイルとしては、Googleサービスアカウントファイルが適しています。 Google Developers Consoleで作成するには、次を実行する必要があります。



  1. アプリケーション用にまだ作成されていない場合は、新しいプロジェクトを作成します。
  2. サービスアカウントのリストを開きます 。 プロジェクトを選択するように求められたら、アプリケーションのプロジェクトを選択します。
  3. [ サービスアカウントの作成]をクリックします 。
  4. [ サービスアカウントの作成]ウィンドウで、新しいサービスアカウントの名前を入力し、[ 新しい秘密キーの作成]フラグを有効にして、JSONキータイプを選択します。 アカウントの役割を選択する必要はありません。 作成をクリックします 。


サービスアカウントが作成され、その後、秘密キーファイルが自動的にダウンロードされます。 このファイルへのパスは、パラメーター"source.credential_file"
