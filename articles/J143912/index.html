WEB 3.0

1年以上の間、私は電子商取引の分野で興味深く議論のあるプロジェクトに取り組んできました。 非常にグローバルなタスクの特定のケースに取り組み、経験豊富な専門家の間でも定期的に生じる結果の誤解により、蓄積されたアイデアの一般化を作成する考えに至りました。



私の一般化では、WEB 3.0テクノロジーの定義も同様に目指しています！ WEB 2.0の原則が成功した後、ネームプレート3.0がさまざまな先見の明によって引き出されなかったのは奇妙です。 しかし、彼らの試みでは、典型的な「Better 2.0」を描きました。 「美しい」、「プロ」などの名言の助けを借りてバージョン番号を増やす試みは、マーケティングの殻に過ぎません。

そのようなナンセンスの例は、Firefoxの現在のバージョン番号です。 私はめったにそれを使用せず、プロジェクトの新しいバージョンのレイアウトを表示するためだけに使用します。 起動するたびに、バージョンが変更され、メニューのいくつかのボタンが場所を変更したことがわかります。



新しいバージョンの認識は、Webサービスの重要な分野である技術的な飛躍、つまり情報（コンテンツ）の作成と処理によってのみ正当化されます。 この飛躍は「より美しく」または「より専門的に」なることはできず、技術的に根本的に異なるだけです。



私の考えでは、WEB 3.0の重要な機能は「コンテンツ変換」であり、これはWebアプリケーションの設計と開発の原則だけでなく、著作権の概念も変更し、既存のサービスのビジネスモデルの変換につながります。



すでに議論したいですか？ その後、猫に気軽に！



未来を見る前に、私たちが持っているものを理解する必要があります。



WEB 1.0







作成した古き良きWebは、クライアントとサーバーの概念に基づいて基盤とすべての用語を作成しました。 情報の消費者-クライアント、情報のプロデューサー-プロのジャーナリスト（作家、監督、プロデューサー、アーカイブ）がいます。 生産者は、消費者の関心を引くものを決定します。 消費者は電子メールでレビューを書くことができ、より多くの場合、彼は自分の足で別のリソースに向かって投票するだけです。

2000年、私はコンピューターサイトのニュースエディターとして働きました。 各ニュースは約1000人の訪問者に読まれましたが、電子メールの返信は珍しく、本当の休日でした！ 古典的な作家が死後初めて認識に頼ることができたのは慰めでした



デジタルライブラリ、ニュースサイト、オンライン新聞-これらは古き良きWEBサービスの典型的な例です。



WEB 2.0



開発と拡張として、世界規模のインターネットネットワークは、ますます多くの新しいユーザーを吸収しました。 教育を受けた専門家、いわゆるクリエイティブクラスは、ネットワークに最初に参入した人物の1人です。 彼らは興味のあるコミュニティ、つまりウェブフォーラムを中心に形成されたコミュニティを作り始めました。 これらのアクティブグループは、長い間パッシブコンテンツコンシューマーの運命に耐えることができず、革命を起こしました-彼らはWEB 2.0を作成しました。







Webの新バージョンの主な違いは、ステルスモデレーターである専制的な編集長ではなく、コンテンツ作成の原則でした。 最高のプロジェクトでは、モデレーターはUFOのように見えなくなりました。



webdwanolプロジェクトの成功例には、LiveJournal、Wikipedia、そしてもちろん、愛されているHabrahabrが含まれます。 ソーシャルネットワークは、コミュニティの典型的なアイデアになりました。



WEB 3.0



現在のグローバルインターネットの成長は、次の2つの要因に支えられています。

  1. ネットワークアクセス用のモバイルデバイス数の伸びを上回る
  2. 文化的に大きな違いがある発展途上国および未開発国の加入者の伸びを上回っています。




最初の傾向は、さまざまなサイトの表示のパーソナライズされたバージョンを作成する必要性を客観的に指示します。 このタスクは、 コンテンツ適応という用語さえ引き起こしました。（ コンテンツ適応は 、 コンテンツを再生装置の技術的能力に適応する方法として定義されます） 。



2番目の傾向は、「文化適応」の出現につながり、その最も明白なバージョンは外国語への翻訳です。 ただし、わいせつなコンテンツのフィルタリングなど、他の適応オプションがあります。 そのような適応の最も有名な例は、 偉大な中国のファイアウォールです。



WEB 3.0の課題に対応するテクノロジーはどのようなものでしょうか？



確かに中国の検閲に言及することに興奮した擬似自由主義者に関する短い解説。 私の解釈では、WEB3.0はソース情報をカットしたりフィルターしたりせず、代わりに、新しいプロジェクションが作成されます-コンテンツが適合した新しいサイトです。 したがって、インターネットを制限または共有する代わりに、別の自由度が作成されます。たとえば、読者が利用できる言語で情報を受信および閲覧する自由と、彼が受け入れられると考える制限があります。 新しい投影法は、元の解釈や他の解釈と正直かつ率直に競合します。

新しいスキームでシミュレートされた現実の複雑さに完全に一致して、中間層が表示されます。これを仲介と呼びます。 この用語は、電気通信および法律の分野で豊富な経験があり、アクションの本質を完全に開示しています。 ソースコンテンツ（テキスト）に基づく仲介サーバーは、現実の異なる投影を形成します。 これは、英語から中国語への雑誌記事の翻訳、またはテーマレベデフのわいせつなブログのロシア語文学への修正、または若い学生がその言語にアクセスできるかのいずれかです。



メディエーションサーバーは、たとえばSummly （89歳の中国の億万長者が投資した15歳の英国の子供の天才によって作成された革新的なスタートアップWebテキスト処理スタートアップ）またはAbby Comprenoを使用して、マシンインテリジェンスの影響を通じて制御できます。



クラウドソーシングとWeb2.0のスコアリングに基づいたメディエーターを使用してメディエーションを形成する代替方法。



脳が沸騰している？ 関係ありません！ ハイテクのスラングを離れ、指でウィキペディア3.0を描いてみてください。



ウィキペディア3.0-それは何でしょうか？



  • 新しいウィキペディアでは、個別の国内サイトを必要としなくなりました。 エンサイクロペディアの統一された構造は、さまざまな程度で他の言語に翻訳されます。 欠落している翻訳は、マシンインテリジェンスによって作成されます。 機械翻訳はネイティブスピーカーによって修正され、その結果、アルゴリズムがトレーニングされます。
  • Lurkはもはや別の百科事典ではなく、内部に投影されています。 また、他の多くの新しい予測。
  • ウィキペディアの投影法は、80文字の画面を持つ携帯電話やtwitterリーダーなど、あらゆるデバイスグループで利用できます。
  • 訪問者は、選択した投影を読むか、投影/記事/段落の群衆編集モードをオンにして、他の人の校正に投票することができます。 または、編集者になり、自分で変更を加えることもできます。
  • 投影は、著者の評価と群衆編集者の投票に基づいて、最も人気のある編集から形成されます。
  • プロジェクションは、異なる期間の編集を補間することでオンザフライで形成されます-最も関連性が高い場合は、昨年に最も承認されます。
  • これ以上の編集者の戦争はありません-それが読まれ、投票された場合、いかなる投影にも生命に対する権利があります。 最も使用頻度の低い原則に従って、古い投影は消えます。
