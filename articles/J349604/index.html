地獄のタイムゾーン



サーバーでPython機能を使用して、Webブラウザーでユーザーに表示される日付と時刻を表示することはお勧めできません。 次の例を考えてみましょう。 私はこれを2017年9月28日16:06に書いています。これを書いている間の私のタイムゾーンはPDT（または必要に応じてUTC-7）です。 Pythonインタープリターでは、次のものが得られます。 







 >>> from datetime import datetime >>> str(datetime.now()) '2017-09-28 16:06:30.439388' >>> str(datetime.utcnow()) '2017-09-28 23:06:51.406499'


datetime.now()



呼び出すと、現在地の正しい時刻が返され、 datetime.now()



呼び出すと、UTCタイムゾーンの時刻が返されます。 世界のさまざまな地域に住む人々にこのコードを同時に実行するように依頼できる場合、 datetime.now()



関数は各人に対して異なる結果を返しますが、 datetime.utcnow()



は常に単一の値を返します同時に、場所に関係なく。 それで、どのオプションが世界中のユーザーが利用できる可能性が高いWebアプリケーションで最もよく使用されると思いますか？







明らかに、サーバーは時間を管理する必要がありますが、これは場所に依存しません。 このアプリケーションが開発され、世界中のさまざまな地域で複数の運用サーバーを使用する必要が生じた場合、各サーバーがさまざまなタイムゾーンに従ってタイムスタンプをデータベースに書き込むことは望ましくありません。 UTCは最もよく使用される統一タイムゾーンであり、 datetime



クラスでサポートされているため、これが必要です。 そして、私はそれを使用します。







ただし、このオプションでは、1つの重要な問題が発生します。 異なるタイムゾーンのユーザーの場合、UTCタイムゾーンの時刻を見ると、実際に記録が行われた時期を判断するのは困難です。 時間はUTCで表示されることを事前に理解する必要があり、時間帯に合わせて時間を精神的に調整できる必要があります。 PDTタイムゾーンで午後3時に何かを送信したユーザーが、メッセージが22:00またはより正確には22:00のUTC時間で表示されることをすぐに確認したとします。 わかりにくいでしょう。







UTCタイムスタンプの標準化はサーバーの観点からは非常に理にかなっていますが、ユーザーにとって使いやすさは難しくなります。 この章の目標は、サーバーで管理されているすべてのタイムスタンプをUTCで保存することにより、この問題を解決することです。







タイムゾーン変換



この問題の明らかな解決策は、すべてのタイムスタンプをUTCから各ユーザーの現地時間に変換することです。 これにより、サーバーは一貫性のためにUTCを使用し続けることができ、ユーザーごとに調整されたオンザフライ変換が利便性の問題を解決します。 このソリューションの複雑さは、各ユーザーの場所を決定することにあります。







多くのWebサイトには、ユーザーがタイムゾーンを指定できる構成ページがあります。 これには、タイムゾーンのリストを含むドロップダウンリストをユーザーに表示するフォームで新しいページを追加する必要があります。 ユーザーは、登録の一部として最初にサイトにアクセスするときに、タイムゾーンを入力するように求められます。







これは問題を解決する適切なソリューションですが、オペレーティングシステムで既に設定されている情報の一部をユーザーに入力してもらうのは少し奇妙です。 ローカルコンピューターからタイムゾーン設定を見つけることができれば、より効率的なようです。







結局のところ、Webブラウザーはユーザーのタイムゾーンを認識し、標準のJavaScript APIを介して提供します。 JavaScriptを介して利用可能なタイムゾーン情報を使用するには、実際には2つの方法があります。







  • 「古い学校」のアプローチは、ユーザーが最初にアプリケーションにログインしたときに、Webブラウザーが何らかの方法でタイムゾーン情報をサーバーに送信することです。 これは、 Ajax呼び出しを使用するか、さらに簡単にメタリフレッシュタグを使用して実行できます。 サーバーがタイムゾーンを定義すると、セッションに保存したり、データベースのユーザーレコードに書き込んだりして、テンプレートを表示するときにすべてのタイムスタンプを構成できます。







  • 「新しい学校」のアプローチは、サーバーの状況を変更せず、JavaScriptを使用してクライアントのUTCからローカルタイムゾーンに変換できるようにすることです。

    両方のオプションは機能していますが、2番目のオプションには大きな利点があります。 タイムゾーンを知っているだけでは、ユーザーが期待する形式で日付と時刻を表示するのに十分ではありません。 ブラウザは、AM / PMおよび24時間形式、DD / MM / YYYY対MM / DD / YYYYおよびその他の多くの文化的または地域的なスタイルなどを定義するシステムロケール構成にもアクセスできます。









これで十分でない場合、「新しい学校」を支持する別の利点があります。 それをすべて行うオープンソースライブラリがあります！







Moment.jsとFlask-Momentの紹介



Moment.jsは、考えられるすべての書式設定オプションを提供するため、日付と時刻をまったく異なるレベルに表示するタスクを実行する小さなオープンソースのJavaScriptライブラリです。 また、少し前に、Flask-Momentを作成しました。これは、アプリケーションにMoment.jsを簡単に組み込むことができる小さなFlask拡張機能です。







それでは、Flask-Momentをインストールすることから始めましょう。 







 (venv) $ pip install flask-moment


この拡張機能は、通常の方法でFlaskアプリケーションに追加されます。







app/__init__.py



のインスタンスを作成します。 


 # ... from flask_moment import Moment app = Flask(__name__) # ... moment = Moment(app)


他の拡張機能とは異なり、Flask-Momentはmoment.jsと連携するため、すべてのアプリケーションテンプレートにこのライブラリを含める必要があります。 このライブラリが常に利用できるように、ベーステンプレートに追加します。 これを行うには2つの方法があります。 最も直接的な方法は、ライブラリをインポートする<script>



タグを明示的に追加することです。 しかし、Flask-Momentは、 <script>



を生成するmoment.include_moment()



関数を提供することにより、さらに簡単なオプションを提供します。







app / templates / base.html：moment.jsをベーステンプレートに追加します。 


 ... {% block scripts %} {{ super() }} {{ moment.include_moment() }} {% endblock %}


ここで追加したscripts



ブロックは、Flask-Bootstrapベーステンプレートによってエクスポートされた別のブロックです。 これは、JavaScriptインポートを含める必要がある場所です。 このブロックは、ベーステンプレートで定義された一部のコンテンツが既に含まれているという点で、前のブロックとは異なります。 私がやりたいのは、基礎となるコンテンツを失うことなく、moment.jsライブラリを追加することです。 これは、基本テンプレートのコンテンツを保存するsuper()



命令を使用して実現されます。 super()



を使用せずにテンプレートでブロックを定義すると、ベーステンプレートでそのブロックに対して定義されたコンテンツはすべて失われます。







Moment.jsを使用する



Moment.jsは、ブラウザーにアクセス可能なmoment



クラスを提供します。 タイムスタンプを作成するための最初のステップは、 ISO 8601形式で必要なタイムスタンプを転送することにより、このクラスのオブジェクトを作成することです 。 以下に例を示します。 







 t = moment('2017-09-28T21:45:23Z')


日付と時刻の標準ISO 8601形式に慣れていない場合、形式は次のとおりです： {{ year }}-{{ month }}-{{ day }}T{{ hour }}:{{ minute }}:{{ second }}{{ timezone }}



。 UTCタイムゾーンでのみ作業することを既に決めているので、最後の部分は常にZ



であり、これはISO 8601標準のUTCです。







moment



オブジェクトは、さまざまな表示オプション用のメソッドをいくつか提供します。 以下は、最も一般的なオプションの一部です。 







 moment('2017-09-28T21:45:23Z').format('L') "09/28/2017" moment('2017-09-28T21:45:23Z').format('LL') "September 28, 2017" moment('2017-09-28T21:45:23Z').format('LLL') "September 28, 2017 2:45 PM" moment('2017-09-28T21:45:23Z').format('LLLL') "Thursday, September 28, 2017 2:45 PM" moment('2017-09-28T21:45:23Z').format('dddd') "Thursday" moment('2017-09-28T21:45:23Z').fromNow() "7 hours ago" moment('2017-09-28T21:45:23Z').calendar() "Today at 2:45 PM"


この例では、2017年9月28日21:45 UTCの行で初期化されたmoment



オブジェクトを作成します。 上記で試したすべてのパラメーターは、コンピューターで構成されたタイムゾーンであるUTC-7で表示されていることがわかります。 上記のコマンドをブラウザーのコンソールに入力して、コンソールを開くページにmoment.jsが含まれていることを確認できます。 上記の変更を行った場合、moment.jsを含めることで、 https： //momentjs.com/でマイクロブログでこれを行うことができます 。







異なるメソッドが異なるビューを作成する方法に注目してください。 format()



、Pythonのstrftime関数に似た形式名文字列を使用して出力形式を制御します。 fromNow()



およびcalendar()



メソッドは、現在の時刻に関連するタイムスタンプを表示するという点で興味深いので、「1分前」や「2時間後」などの出力行を取得します。







JavaScriptで直接作業している場合、上記の呼び出しはタイムスタンプを表示する文字列を返しました。 したがって、このテキストをページの適切な場所に追加するには、JavaScriptを使用してDOMを操作する必要があります 。 Flask-Moment拡張機能によりmoment.jsの使用が大幅に簡素化され、テンプレートに必要なJavaScriptマジックを含むmoment



オブジェクトを含めて、ページに正しい方法で時間を表示できます。







プロファイルページに表示されるタイムスタンプを見てみましょう。 現在のuser.htmlテンプレートにより、Pythonは時間の文字列表現を生成できます。 次のように、Flask-Momentを使用してこのタイムスタンプを作成できます。







app / templates / user.html：moment.jsを使用したタイムスタンプ。 


  {% if user.last_seen %} <p>Last seen on: {{ moment(user.last_seen).format('LLL') }}</p> {% endif %}


ご覧のとおり、Flask-MomentはJavaScriptライブラリの構文に似た構文を使用しますが、わずかな違いがありますが、 moment()



引数はISO 8601文字列ではなくPython datetime



オブジェクトになります。また、DOMの目的の場所にレンダリングされたラベルを挿入するために必要なJavaScriptコードを自動的に生成します。







Flask-Momentとmoment.jsを使用できる2番目の場所は、 _post.html



サブテンプレートです。これは、インデックスとユーザーページから呼び出されます。 テンプレートの現在のバージョンでは、各メッセージの前に「username sayss：」という行があります。 fromnow()



を使用してタイムスタンプを追加できます。







app / templates / _post.html：メッセージサブパターンのタイムスタンプ。
