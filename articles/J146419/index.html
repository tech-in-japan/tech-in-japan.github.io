RHEV 3.0：代替仮想化？ いいえ、代替のITエコシステム



RHEV 3.0とは何ですか？

-RHEV 3.0は、2年間の集中的な開発の後、今年2月に登場しました。

-垂直ソフトウェア統合の一般に受け入れられている慣行の代わりに、オープンソース。

-他の仮想化管理システムよりも価格が著しく低い 。

「これは、VMware、Microsoft、Citrixとは異なり、エコシステムの中心ではありません。」

- 売上高が10億ドルの会社の製品。

-クラウドコンピューティングとDBMSのプラットフォームとしての強固な地位。









Linuxディストリビューションから仮想化まで。

Red Hatは、Red Hat Enterprise Virtualizationの第3バージョンで仮想化管理システムを更新しました。 しかし、製品の利点を改めて語る代わりに、仮想化開発の観点から企業の位置を分析し、Red Hatのアプローチが競合するソリューションとどのように異なるかについて、少し分析することにしました。



過去数週間にわたって、Phil Andrewsから多くの新しい情報を学びました（これは2月のRed Hatセミナーで話したものです-およそTransl。）。



Red Hatは、主にLinuxディストリビューションのメーカーとして知られています。 ディストリビューションの最初のバージョンの1つを自宅のコンピューターにインストールした方法をよく覚えています。 当時、彼らの主な競争相手はSuseでしたが、後にノベルに吸収されました（そしてノベル自体が買収されました ）。



当初から、Linuxはサーバーオペレーティングシステム市場に非常に良い展望を持っていました。 もちろん、LinuxはUnixと密接に関連しており、企業ユーザーにとって重要な多くのプロパティをLinuxから借用しているという事実のために、もちろんそうです。



グラフ「図1」は、2003年から2011年までのx86サーバー市場とオペレーティングシステムの成長を示しています。 Linux、Windows、およびx86サーバー市場が足跡をたどるという事実を示しています。 この期間：Solaris（Unixライン）のシェアは、主にOracleがSun Microsystemsを買収した後 、大幅に減少しました。



多くの企業が無料のLinuxディストリビューション（これは数えられない）を持っているという事実のために、Linux市場の規模は（Windowsに比べて）比較的小さいが、 クラウドコンピューティングとデータベースソリューションの基盤としてオープンソースソフトウェアを検討している組織が増えています 。



ITインフラストラクチャの市場での競争。

エンタープライズソフトウェアにおけるRed Hatの市場シェアの拡大は、VMwareとの激しい競争をもたらしました。 実際、VMwareには長年にわたって競合他社がいなかったため、この会社の仮想化ソリューションはサーバーハードウェアのコストを大幅に削減するのに役立ちました。 しかし、今日、VMwareの焦点は仮想化からインフラストラクチャ管理にシフトしているため、VMwareとRed Hatのソフトウェア機能の多くは相互に複製し始めています。



ESXハイパーバイザーはオペレーティングシステムから比較的独立していますが、VMwareの優れた財務パフォーマンスから明らかなように、Windowsサーバーで最もよく使用されます。 それに比べて、Red HatはLinux市場にはるかに依存しています。 Red Hatによると、Linuxでのサーバー仮想化は要求の厳しいタスクに使用されるため、はるかに後に開始されましたが、Linuxサーバーでの仮想化の使用の増加に伴い、RHEVのシェアが増加します。



Red Hatの代表者によると、RHEVの価格のおかげで今ではチャンスがあります。 さらに、顧客の80％はVMwareを使用しています。 Red Hatの目標は、ユーザーに必要な機能とサービスを低価格で提供することをユーザーに納得させることです。



「図2」グラフは、インフラストラクチャおよび関連サービス（Citrix、VMware、およびRed Hatの総収益を構成していますが、MicrosoftおよびNovellの収益の一部のみ）からの収益の成長を示しています。 Citrixは順調に成長していますが、Xenハイパーバイザーに基づいた仮想化ソフトウェアだけではありません。 Citrixの成長の大部分は、各ソフトウェア会社の買収のおかげで、デスクトップ仮想化に関連しています。 Red HatはNovellを上回っていますが、VMwareにやや遅れています。



KVMは比較的新しいハイパーバイザーです。

当初、KVMハイパーバイザーはHyper-V、ESX、Xenほど注目されていませんでしたが、KVMとRHEVの両方をすぐにサポートした企業はIBM、HP、Intel、Ciscoでした。 プロジェクトに注目が集まっていない理由の1つは、KVMがライバルよりもずっと遅れて市場に参入したことです。 サーバー仮想化による利益の点で、ESX、Xen、Hyper-Vに遅れを取っていると考えています（KVMは主に無料のハイパーバイザーとして使用されています）。



「図3」のグラフは、仮想化市場における利益の分布を示しています。



オープンソースおよびサポートサブスクリプションモデル。

Red Hatは常にオープンソース市場のプレーヤーであり、顧客にとって重要な機能をサポートしてOpenSource開発に追加することで利益を上げています。 Red Hatソフトウェアで使用されるコードのわずか13％がRed Hat開発者自身によって記述されているため、このアプローチには利点があり、問題の解決に集中できます。 しかし同時に、Red Hatは同じオープンソース製品を提供できる他のソフトウェアベンダーと競争することを余儀なくされています。



Red Hatは、テクニカルサポートのサブスクリプションから受け取る主な利益であり、サービスのレベルに満足している場合、企業によって毎年更新されます。 ところで、Sun Microsystemsは、その「独立した」生活の最後の2年間に同様のアプローチを適用しようとしました。 しかし、彼女は、Solaris、Java、およびSunの他のソフトウェアの十分な数のユーザーをサポートするよう説得することができませんでした。



両社の違いは、Red Hatは常にサブスクリプションモデルを使用していたのに対し、Sunは既存のソリューションをこの新しいビジネスモデルに統合しようとしたことです。 これは、ビジネスをライセンス販売からサブスクリプションモデルに移行したいすべてのITプロバイダーにとって重大な問題であることに注意してください。古いスキームからの利益の減少で新しい収入の流れを補うには時間がかかります。



次に並んでいるのはCloudFormsとGlusterです

2012年、Red HatはCloudFormsなどの新製品に集中する予定です（ 今月だけで大きな発表がありました 。 さらに、Red Hatは、 最近買収したGlusterのソフトウェアに基づいて、ストレージ市場向けのソフトウェアを提供する予定です。 ストレージコストを削減するためにRed Hatが提供するものを見るのは興味深いでしょう。



一部の調査結果は、さまざまなタイプのエコシステムです。

RHEVの（パートナーおよびソフトウェアベンダーの）エコシステムがVMwareやMicrosoftのようにRed Hatに依存することはほとんどありません。 オープンソースソフトウェアプロバイダーとして、Red Hatはそれ自体、定義上、多極的なコミュニティの一部です。



Red Hatの利点は、既存のオープンソース開発を強化でき、ゼロからすべてを書くことができないことです。そのため、Red Hat製品のコストは常にプロプライエタリなソフトウェアよりも低くなります。 企業のソフトウェア市場におけるプレーヤーとしての会社の重要性の高まりは、これに関連しています。 特にサーバー仮想化の分野では、KVMハイパーバイザーは多くのXenソリューションよりも実装しやすいと見られています。



そのため、オープンソースの原則を引き続き順守している商業会社がありますが、他のソフトウェアベンダーはより垂直に統合されています。 レッドハットは、現在の景気低迷の中で、低コストで信頼性の高いソリューションのプロバイダーとして優れた地位にあります。
