IE8の自動更新からのブロック

Microsoftは、非常に便利なIE8 Blocker Toolkitをリリースしました。これにより、ローカルネットワーク上のコンピューターに最新のIE8ブラウザーがインストールされるのを完全に防ぐことができます。 「ブロッカー」のリリースは、Microsoftがすぐにすべての自動更新パッケージ（自動更新（AU）、Windows Update（WU）、Microsoft Update（MU）、および優先度の高いIE8）にIE8を含めるという事実によるものです。



ブロッカーをインストールしない場合、IE7ブロッカーを持っている人でも、Windows XP SP2 +、Windows Vista、Windows Server 2008、およびWindows Server 2003 SP2 +のすべてのユーザーのマシンでIE8のインストールが開始されます。







Windows XPのスクリーンショットは、ユーザーが[インストールしない]ボタンをクリックしてインストールを拒否する機会があることを示しています。



ご存知のように、多くの企業では、ネットワーク上のローカルマシン上で厳密に統一されたプログラムのセットが必要です。 彼らは、特に新しいブラウザなどの重要なソフトウェアを使用して、最も厳しい方法で新しいプログラムを不正にインストールすることに苦労しています。



企業間の一般的な慣行は、最初に内部テストを実施することです。 その後、新しいソフトウェアが全員によってすぐにインストールされます。



ロックには使用時間の制限がないため、原則として、将来的にロックを無効にすることはできません。
