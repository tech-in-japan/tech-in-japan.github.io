外観



Compaq Presario 3020は、非常にいCRTモニターと遠くから混同される可能性があります。







モノブロックスタンドには、輝度と音量レベルを調整するためのボタンと、Compaqの必要なソフトウェアがインストールされている場合に、ユーザーに追加機能を提供する機能的なQボタンがあります。 ケースの背面にある電源のボタンでコンピューターの電源をオン/オフにします。 側面には内蔵スピーカーがあります。 スタンドの傾きは調整できませんが、12インチLCDディスプレイの角度を変更できます。







モノブロックの右側には、4枚のCD用のチェンジャーを備えた6速NEC CD-ROMドライブがあります。 はい、あなたは間違っていませんでした。本当に4枚のCDをドライブに挿入できます。 4つのCD-ROMリーダーがオペレーティングシステムで同時に使用可能になりますが、残念ながら、一度に複数のディスクから情報をコピーしようとしませんでした。 ほとんどの場合、これは不可能です（可能であれば、非常に遅くてノイズが多くなります-おそらく1つのレーザーしかありません）。







左側には、3.5インチフロッピーディスク、COMおよびLPTポート、内部ISAモデムの2つのRJ11コネクタ、2つのUSBポート、キーボードおよびマウスインターフェイスPS / 2、および内蔵サウンドカードESS 1888のオーディオコネクタ用のドライブがあります。 3つの拡張（2つのPCIスロットと1つのISA）しかありませんが、S3 Aurora64V +ビデオカードは最初のPCIスロットに既にインストールされており、そのビデオ出力に追加のVGAモニターを接続できます。 ビデオカードは、800x600（追加のモニターに画像を表示する場合は1024x768）の解像度と32ビットカラーレンダリングをサポートします。







ケースの背面には、電源コネクタとコンピューターの電源ボタンがあります。

特異性の欠如にもかかわらず、Compaq Presario 3020には最大200 MHzのクロック周波数とRAMを搭載したPentium Iプロセッサが搭載されていると考えられます。

しかし、なぜコーヒーかすを推測して、コンピューターの電源を入れるだけでBIOSにアクセスしてその特性を確認できるのでしょうか？



だから私はやった。 しかし、最初のトラブルは私を待っていました-2001年以前にコンパックのコンピューターのBIOSにアクセスする標準的な方法であったにもかかわらず、コンピューターはF10を押してBIOSに入ることを望みませんでした。



インターネットを徹底的に読んだ後、このコンピューターのBIOSが「彼のハードに依存している」ことがわかりました。 これは、一部には当てはまります。基本的なI / Oシステムをセットアップするためのグラフィカルインターフェイスは、実際にはハードドライブのサービスセクションにあり、明らかに前の所有者によって上書きされました。







ただし、独自のCompaq Personal Computer Setupユーティリティを含むディスケットから起動することにより、このセクションがなくてもBIOS設定を変更できます。



フロッピーディスクから常に起動して設定を変更しないようにするには、グラフィカルBIOSインターフェイスといくつかの診断ユーティリティを含むサービスセクションに戻る必要があります。 これは2枚のフロッピーディスクを使用して実行できます。最初のディスクにはパーソナルコンピュータセットアップユーティリティが含まれ、2番目のディスクにはパーソナルコンピュータ診断ユーティリティが含まれている必要があります。 これらのフロッピーディスクのイメージは、HPの公式Webサイトからダウンロードできます。



最初のディスケットから起動し、ウィザードのプロンプトに従って、サービスパーティションを復元します。 Compaq Presario 3020コンピューターのハードドライブをパーティションに分割しないでください。パーティションを作成しないと、障害が発生します。



これで、サービスセクションが作成されました！ これで、コンピューターの起動中に画面の右上隅にある点滅する白い長方形を待ってF10を押すことで、BIOSセットアップのグラフィカルシェルに移動できます。



利用可能なBIOSバージョンは1996年に遡り、 Presario Specific ModelsシステムROM更新ディスケットを使用して最新バージョンに更新されました。



技術仕様



Compaq Presario 3020の「内部」の内容を確認できます。







Compaq Presario 3020コンピューターのネイティブハードドライブはゆっくりと、しかし確実に老朽化する（動作中に非常に不快な音を出す）という事実により、20 GBもの容量の大きい仲間がそれを置き換えることがわかりました。 起動電流と動作電流が大きいため、より大きなハードドライブをインストールしませんでした。また、電源の正常性を危険にさらしたくありませんでした。







当然、BIOSは新しいハードドライブの全容量を自動的に認識せず、そのC / H / Sジオメトリをペンで登録する必要がありました。 すぐに2番目の問題が発生しました-BIOSは8 GBを超えるハードドライブをサポートしていません。 さて、BIOSで18383/16/63ジオメトリを公開します（これは、IDEハードドライブが8 GBを超えるように見える方法です）。プライマリブートパーティションは、オペレーティングシステムを正しくインストールするのに十分な8ギガバイト以下であると安全に期待できます。



OSの選択とインストール



Compaq Presario 3020のオペレーティングシステムの選択は、操作の速度、ハードウェアサポート、バージョンの関連性という基準に従って行われました。 しかし、最も重要なパラメーターは、古いDOS以降のゲームを実行できることでした。そのため、* nixオペレーティングシステムは5分以内に検討されました。



元のドライバーディスクが不足していたため、Windows NT 4.0のインストールは完全に失敗しました。ビデオカードやサウンドを含むほとんどのデバイスは正常ではありませんでした。 HPサポートサイトでも必要なドライバーが見つからなかったため、選択はWindows 9xファミリーでした。



Windows 95では、作業に必要なリソースは少なくなりますが、Windows 98と比較すると古すぎます。







選択はWindows 98で行われましたが、単純ではありませんが、Fedorenkoから行われました。 著者のWebサイトで、このアセンブリと通常のWindows 98との違いについて詳しく知ることができます。



プライマリブートパーティションをBIOSが認識する量よりも大きくすると、DOSからWindows 98をインストールするときに、ディスクの最後のセクターで読み取り/書き込みエラーなどのエラーが発生する可能性があることに注意してください。



つまり、Windows 98のインストールは、スキャンディスクディスクをチェックする段階を超えることはありません。 ディスクをチェックせずにOSのインストールを開始しても問題は解決しないため、運命を誘惑することはありません。



さらに、OSのインストール後に残りの領域をマークできます。追加のパーティションの最大サイズは、フォーマットされるファイルシステムの制限によってのみ制限されます。 これは、BIOSの設定と制限にもかかわらず、Windows 98オペレーティングシステムが大きなハードドライブでうまく機能するためです。



しかし、すべてが私が望んだほどバラ色ではありませんでした。 作成されたサービスパーティションのハードドライブがPartitionMagicによって別のコンピューターでパーティション化およびフォーマットされた後、サービスパーティションが破損し、Compaq Presario 3020がそのコンピューターからの読み込みを停止しました。



この憂鬱な事実は、別のコンピューターでハードドライブをフォーマットしようとするのをやめるのに十分でした。 Compaq Presario 3020自体にパーティションを作成することが決定されましたが、RAMが少なく、CD-ROMから起動できないため（Plop Boot Managerでも役に立たなかったため）、最も一般的なFDISKを使用するためにPartitionMagicを放棄する必要がありました。



次の迷惑-FDISKは、ハードドライブの合計容量を504 mbと決定しましたが、BIOSには明示的に8 GBと記載されています。 どうやら、これは許容されるヘッド数に対するMS DOSの制限と、BIOSのInt 13hハンドラーコードでCHS値を変換するためのサポートの欠如によるものです。



Windows 98をインストールするには、この量で十分なので、問題の本質を掘り下げず、サイズが496 MBのアクティブブートパーティションを作成しました（サービスパーティションは8メガバイトを消費しました）。 FAT32ファイルシステムパーティションの最小サイズの制限により、作成されたパーティションをFAT16でフォーマットする必要がありました。 オペレーティングシステムをインストールした後、20 GB未満の追加パーティションが作成されました。



この段階で、オペレーティングシステムのインストールに伴う試練は終了し、ファンが起動しました。
