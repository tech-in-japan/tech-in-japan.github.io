エンティティ参照を抽出し、Textocat APIで検索します

Textocat APIは、クラウドベースのSaaSテキスト分析です。 テキストから有用な情報を高品質に抽出することは困難な作業であり、深刻な専門知識が必要です。 Textocatチームの使命は、ワードプロセッシングを非常に使いやすくして、現代の開発者がそれを武器に含めることができるようにすることです。 Textocat APIを使用すると、テキスト分析アプリケーションのプロトタイプをすばやく作成して、ビジネスに変えることができます。 この出版物では、エンティティ（オブジェクト）の言及を認識し、ロシア語のドキュメントを検索するためのTextocat APIの機能をアプリケーションに簡単に統合できることを示します。



Textocat APIベータ版の機能



4月上旬に、 Textocat APIのベータテストを開始しました。 このバージョンでは、サービスの機能の一部を次の機能とともに無料で使用できるように開発者に提供しています：

  • ロシア語の文書コレクションにおけるエンティティの言及の認識（エンティティ認識）。
  • 処理されたコレクションの保管。
  • 選択したエンティティタイプに基づく全文検索。












エンティティ参照の認識


名前付きエンティティ（名前付きエンティティ認識、NER）を含むエンティティの言及を認識するタスクは、テキストの特定の断片を既知のタイプに分離して分類することです。 たとえば、Textocatの現在のバージョンは7つのタイプをサポートしています。

  • PERSON（人々）、
  • 組織（組織）、
  • GPE（地政学的エンティティ）、
  • LOCATION（地理的特徴）、
  • 施設（インフラ施設）、
  • TIME（時間単位）
  • およびMONEY（通貨単位）。




この問題に対する高品質のソリューションが難しいのはなぜですか？ 結局のところ、アルゴリズム理論の大学コースの辞書やメソッドを使用して、同じ結果を得ることができますか？ 実際、自然言語、特にロシア語にはさまざまなニュアンスが豊富です。

  1. 柔軟性：可能性のある単語形式と語尾を考慮して正確に比較するには、優れた形態素解析ツールが必要です。
  2. 語彙のあいまいさ：銀行「ロシア」（組織）とロシア（GPE）としての参照を区別する方法は？
  3. 名前の曖昧さ：ロシア連邦大統領行政の長であるセルゲイ・イワノフの言及を、彼の完全な同名-観光に関するタタールスタン共和国の国家委員会の議長への言及から区別する方法？


機械学習ベースのアプローチの中核となるのは、クラウドソーシングに基づいて評価者から注釈を付ける独自のデータ収集技術と、これらのタスクに確率的グラフィカルモデルを使用する現在の方法を改善する独自の開発です。 このような決定の精度は、キーワードに基づく単純なアプローチよりも1桁高くなります。 さらに、実装されているすべてのTextocatテクノロジーは、ソースコードを書き換えることなく、マシンの数に応じてスケーリングされます。



インタラクティブデモのページで、プログラミングを開始しなくても、Textocatテクノロジーがさまざまなテキストでこのタスクにどのように機能するかを理解できます。 現在の結果は可能性の限界からはほど遠いことに注意してください。近い将来、認識の質が大幅に改善されるでしょう。 現在、テクノロジーはニューステキストで最高の結果を示しています。



ドキュメントのアップロードと検索


処理後、ドキュメントはアンロードできるようになります（この認証トークンを持つクライアントのみ）。 Textocat APIを使用すると、ユーザーはダウンロードしたすべてのドキュメントを検索し、キーワード（Tinkoff Bankなど）による全文クエリをサポートできます。 さらに、Textocat APIは、キーワードが発生する注釈のタイプに制限を課すクエリの拡張構文を提供します。 たとえば、クエリ「ORGANIZATION：Ford」を使用すると、組織のコンテキストでキーワード「Ford」が記載されているドキュメント（「Ford Motors」）を検索して、有名なアメリカの実業家ヘンリーフォードの名前への参照を分離できます。 サービスのフルバージョンでは、辞書またはクライアントナレッジベース（エンティティリンクまたは名前付きエンティティの曖昧性除去）からの特定のオブジェクトのドキュメントの追跡機能が利用できます。



APIの原則



Textocat APIは、古典的なRESTful APIです 。つまり、サービスを使用した作業は、HTTPプロトコルを使用した標準的な要求を通じて実行されます。 これまでのところ、 JSONは、Textocat APIの入力および出力でサポートされている唯一の形式です。 多くのドキュメントで作業しやすいように、Textocat APIを使用すると、ドキュメントをバッチで収集し、処理のために送信できます。 原則として、これにより、大きなコレクションをパッケージに分割し、それらを並列処理に送信することができます（プレミアム機能）。 パッケージの処理時間は、パッケージのサイズとすべてのユーザーが作成する現在の負荷によって異なります。 したがって、クライアントは、対応する要求を送信して、パッケージのステータスを定期的に確認する必要があります 。 ドキュメントのパッケージが処理されるとすぐに、構造化された形式でシステムに保存され、クライアントのサブスクリプションの全期間保存されます。 この瞬間から、クライアントは処理済みのドキュメントをアップロードできます。 Textocat APIを使用すると、説明した検索ツールを使用して、出力ドキュメントをフィルタリングおよびソートできます。



APIキーを取得



以下に説明する手順を繰り返すには、APIにアクセスするための認証キー（auth_token）が必要です。 キーを無料で取得するには、当社のWebサイトで簡単な登録を行ってください。 サービスの登録または使用に問題がある場合は、ナレッジベースを使用してください。



オンラインドキュメント



インタラクティブなオンラインドキュメントTextocat APIを使用すると、サービスの機能を呼び出すための基本的なコマンドとパラメーターを理解できます。 次に、1つのテストドキュメントで構成される単純なパッケージの処理サイクルについて説明します。これは、オンラインドキュメントで直接繰り返すことができます。

オンラインドキュメントを使用するための詳細な手順
  1. Textocat APIオンラインドキュメントページを開きます 。 作業を開始するには、対応するフィールド（auth_token）に認証トークンを挿入する必要があります。

  2. エンティティの言及を強調するためにパッケージを送信する機能から始めます。 リンク/エンティティ/キューをクリックします 。







  3. 開いたフォームで、 bodyフィールド（JSON形式で追加するドキュメントの配列）を入力する必要があります。 これを行う最も簡単な方法は、「モデルスキーマ」ラベルの右にあるJSONサンプルをクリックして、JSONのテキストフィールドにテストドキュメントのテキストを入力することです。 「モデル」リンクの横をクリックすると、Textocat APIによって処理される主要なドキュメントフィールドの説明が表示されます。







  4. ボタンを押して、リクエスト「Try it out」を送信します。 要求処理が成功した場合、フォームはHTTPステータスコード202と、処理中のパケットに対して生成された識別子を含むメッセージを表示します（応答JSONの「batchId」フィールドを参照）。 この識別子は覚えておく必要があります。 エラーが発生した場合は、エラーコード（たとえば、406）を「応答メッセージ」テーブルの説明と比較し、要求の問題を修正する必要があります。

  5. 前のステップで記憶したパケットの識別子をbatch_idフィールドに入力して、送信されたパケット（ /エンティティ/リクエスト ）のステータスを確認します。 サービスは、パッケージのすべてのドキュメントが処理されアップロードの準備ができている場合、コード200と「ステータス」フィールドにFINISHED応答を返します。パッケージがまだ分析中の場合、IN_PROGRESSを返します。







  6. パッケージが処理されたら 、 batch_idフィールドに同じパッケージ識別子を指定してリクエストを送信することで、アンロード（ / entity / retrieve ）に進むことができます。 すべてのユーザードキュメント（ / entity / search ）を検索するには、text_cat検索クエリの構文に search_queryフィールドを入力します 。







  7. どちらの場合も、サービスは、パッケージメタデータ、ドキュメント、および選択されたエンティティで構成されるモデルのフレームワーク内で、JSONを回答として返します。 応答のフィールドの詳細な説明は、「応答クラス」という碑文の下にある「モデル」リンクをクリックして取得できます。











コマンドラインでTextocat APIを操作する



Unixの標準ユーティリティであるcurlを使用して、コマンドラインから直接サービスを操作する方法を示します。

コマンドラインからTextocat APIを呼び出す手順
  1. 3つの小さなドキュメントで構成される単純なパッケージを使用して、テストドキュメントexample.jsonを準備します。 

    [ { "text": "    « »   —  ,           .   .", "tag": "doc1" }, { "text": "      «»  ", "tag": "doc2" }, { "text": "-       .", "tag": "doc3" } ]


  2. コンソールでコマンドを実行して、Textocat APIのテキストを送信し、エンティティの言及を認識します。 

     curl -X POST http://api.textocat.com/entity/queue?auth_token=<YOUR_AUTH_TOKEN> -H "Content-Type: application/json" --data @example.json


    応答として、このbatchIdパッケージの識別子を含む同様の応答を受信する必要があります。 

     { "batchId": "931da87a-fe98-4639-8cf6-570b5a3fc347", "status": "IN_PROGRESS" }


  3. batchIdをパラメーターとして渡して、パッケージのステータスを確認します。 

     curl http://api.textocat.com/entity/request?auth_token=<YOUR_AUTH_TOKEN>&batch_id=931da87a-fe98-4639-8cf6-570b5a3fc347


    パッケージは非常に小さく、すぐに処理する必要があるため、今回はサーバーの応答は次のようになります。 

     { "batchId": "931da87a-fe98-4639-8cf6-570b5a3fc347", "status": "FINISHED" }


    これで、パッケージをアップロードまたは検索する準備が整いました。

  4. 値「ORGANIZATION：Ford」でsearch_queryパラメーターを渡すことにより、ユーザーのアップロードされたドキュメントで検索を実行します。 

     curl -G --data-urlencode 'search_query=ORGANIZATION:' --data 'auth_token=<YOUR_AUTH_TOKEN>' http://api.textocat.com/entity/search


    サーバーから次の応答が返されます。 

     { "searchQuery": "ORGANIZATION:", "documents": [{ "status": "SUCCESS", "tag": "doc2", "entities": [{ "span": " ", "category": "PERSON", "beginOffset": 14, "endOffset": 25 }, { "span": " «»", "category": "ORGANIZATION", "beginOffset": 28, "endOffset": 43 }] }, { "status": "SUCCESS", "tag": "doc3", "entities": [{ "span": "  ", "category": "ORGANIZATION", "beginOffset": 14, "endOffset": 34 }, { "span": " ", "category": "GPE", "beginOffset": 51, "endOffset": 65 }, { "span": "-   ", "category": "FACILITY", "beginOffset": 0, "endOffset": 34 }] }] }


    したがって、処理されたパッケージから、Textocat APIは、指定されたレキシカル値に「Ford」という単語を含むドキュメントのみを返し（ドキュメント配列を参照）、関連性によってランク付けします。





Java SDK



最後に、公式のTextocat Java SDKを使用してエンティティ参照を強調表示する例を示します 。これにより、JVM互換言語でアプリケーションを実装するときにサービスを操作するロジックが大幅に簡素化されます。 



 final EntityRecognition entityRecognition = TextocatFactory.getEntityRecognitionInstance("<AUTH_TOKEN>"); final FutureCallback<AnnotatedBatch> outputCallback = // a callback for dealing with annotated documents ... FutureCallback<BatchMetadata> inputCallback = new FutureCallback<BatchMetadata>() { public void onSuccess(BatchMetadata batchMetadata) { entityRecognition.retrieve(outputCallback, batchMetadata); } public void onFailure(Throwable throwable) {} }; entityRecognition.submit(new Batch(documents), inputCallback);




EntityRecognitionは、Textocat APIのエンティティ参照認識機能にアクセスするための主要なインターフェイスです。 すべての呼び出しは非同期であるため、EntityRecognitionの応答ハンドラーのクライアントコードは、 Google GuavaライブラリのFutureCallbackでラップする必要があります。 この例では、ドキュメントパッケージ（entityRecognition.submit）が送信され、エンティティ参照の準備ができた注釈付きのドキュメント（entityRecognition.retrieve）がアップロードされます。 後者の代わりに、entityRecognition.searchメソッドを使用して、ロードされたすべてのユーザードキュメントを検索できます。



用途



Textocat APIのこれらの機能は、ビジネスインテリジェンスのさまざまな分野で使用できます。 興味深いアプリケーションの例は、Kazan Federal Universityと協力してKazanで実施したText Analytics HackDayハッカソンプロジェクトにあります。 次の2つのケースについて個別に説明します。



企業のワークフローで検索


最新のBIプラットフォームは、ワークフローシステム、ファイルストレージ、CRM、ERP、テクニカルサポート部門のナレッジベース、コールセンター、企業のメール、フォーラムなど、社内のリソースに検索機能を提供します。 Textocat APIは、テキストから値、オブジェクト、場所、および時間単位を抽出します。 結果は、最新のリポジトリにダウンロードするのに便利な構造化された形式で表示されます。 さらに、抽出されたセマンティクスに基づいて全文検索機能を提供します。 名前とキーワードのカテゴリを考慮すると、検索結果を正確かつ正確にランク付けできます。 途中で、ロシア語の独自の検索ソリューションの開発者がよく知られているオープンソース検索ライブラリ（Apache SolrやElastic Searchなど）のツールを使用することで、高品質のトークン化、文への分割、見出し語化など、多くの低レベルの問題が解決されます。



マーケティングツール


ビジネスでは、外部ソースを分析し、会社、製品、競合他社、請負業者、パートナー、買収オブジェクト、個人などに関する事実、意見、レビューを収集する必要がしばしば生じます。 これにより、マーケティングのタイムリーな決定を行い、販売計画を調整できます。 テキストのソースは、公式プレスリリース、ニュース、ソーシャルネットワークへの投稿、ウェブサイトのレビューとコメントです。 目的のオブジェクトに関するデータ（たとえば、サイトの名前やアドレス）は、CRMまたは外部データベース（たとえば、ロシア連邦税務署の登録簿）からダウンロードできます。 開発者は、Textocat APIの機能を使用して、目的のオブジェクトへの参照の全体像（360度ビュー）を取得できるサービスを作成できます。
