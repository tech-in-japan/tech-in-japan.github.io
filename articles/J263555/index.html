Qucsプロジェクトニュース：SPICEの回路モデリングをサポートするリリースの候補が利用可能です

Qucsはオープンソースの電子回路シミュレータです。 それについて、Habrに関する以前の記事を教えてください。



私たちのプロジェクトは、今後数週間でリリースされる次のリリース0.0.19の準備をしています。 このノートでは、近年最も期待されているイノベーションの概要を説明します。SPICE互換シミュレーターでのQucs回路のモデリングのサポートです。



Qucsは、ゼロから設計されたQucsatorシミュレーションコアを使用します。 このカーネルには多くの利点（Sパラメーターモデリング、高度なポストプロセッサー）がありますが、時間領域モデリング（過渡解析）に関連する多くのバグもあります。 特にこれらのバグにより、Qucsでパワーエレクトロニクスをシミュレートすることはできません。 Qucsでは、たとえば、このようなスキームをシミュレートできます。







NgspiceがサポートされているLinuxおよびWindows用のパッケージをここからダウンロードします。

github.com/ra3xdh/qucs/releases/tag/0.0.19S-rc1



10.21.15からのUPD：RC3が利用可能になりました： github.com/ra3xdh/qucs/releases/tag/0.0.19S-rc3



リリース番号の文字「S」はSPICEを意味します。 どうやら、0.0.19のリリースでは、「S」インデックスと通常の番号付けの2つのパッケージセットがリリースされます。これは、spice4qucsの今後の運命が不明だからです。



spice4qucsのドキュメントはこちら： qucs-help.readthedocs.org/en/spice4qucs



猫の下では、spice4qucsの使用を簡単に検討します。







背景





約10か月間、私とマイクブリンソン（ロンドンメトロポリタン大学）は、外部コンソールシミュレーターNgspice qucs-help.readthedocs.org/en/spice4qucsおよびXyce xyce.sandia.govを使用してQucsスキームをシミュレートできるspice4qucsサブシステムを開発しています。 最も興味深いのはNgspiceです。これは、最高のオープンソース回路シミュレータですが、グラフィカルインターフェイスはありません。 パッチスイートを使用すると、QucsはNgspiceのインターフェイスとして機能できます。



開発の進行状況はここで追跡できます。



github.com/Qucs/qucs/issues/226



github.com/Qucs/qucs/issues/77



github.com/Qucs/qucs/issues/181



目的spice4qucs





Spice4qucsは、外部SPICE互換シミュレーターをQucsに接続できるようにするパッチのセットを持つ松葉杖と小道具のバランスの取れたシステムです 。 ソースコードは、rebase_spice4qucsブランチのメインQucsリポジトリにあります。 それは松葉杖を生むことができるだけでした。



したがって、spice4qucsは次の機能を実行します。



  • QucsスキーマをSPICE-netlistに変換します
  • サードパーティのシミュレーターを起動します
  • シミュレータ出力をQucs XMLデータ形式に変換します




SPICEでモデル化できるスキームの例は、$ QUCSDIR / share / qucs / doc / examples / ngspiceディレクトリにあります。ここで、$ QUCSDIRはQucsインストールのルートです（例：/ usrまたはC：/ Program Files / Qucs）。



設置





このパッケージはまだLinuxディストリビューションに含まれていないため、すべてを手動でコンパイルする必要があります。 

tar xvfz qucs-0.0.19-S1.tar.gz cd qucs-0.0.19-S1 ./configure --prefix=/path_to_install/ make -j_ make install




開発者とGCCの場合はQt4をインストールする必要があります。



Qucsシステムのインストールに損傷を与えないように、-prefixスイッチを使用してconfigureを使用することをお勧めしますが、別のディレクトリ（たとえば、/ opt）のどこかに置きます。 make installの代わりに、ディストリビューション用のパッケージを収集するcheckinstallを使用することをお勧めします。



Windowsへのインストールは簡単です。 インストーラーを実行し、インストールパスを選択して、[次へ]-> [次へ]-> [完了]をクリックする必要があります。



Ngspiceもインストールする必要があります。 アセンブリには含まれていないため、個別にインストールする必要があります。 Linuxでは、このようなパッケージは既に存在し、パッケージマネージャーを使用してインストールできます。例： 

 apt-get install ngspice




使用する





使用を検討してください。 読者は以前にQucsを使用した経験があり、Qucsで回路をモデル化する方法を知っていることを前提としています。 練習の最大の関心はNgspiceです。 Xyceの使用について-ドキュメント（katへのリンク）をお読みください。



例として、振動回路をシミュレートします。 1〜10 MHzの周波数範囲での周波数応答（AC解析）と、7.5 MHzの周波数と0.6 Vの振幅の正弦波効果に対する発振回路の応答を見てみましょう（過渡解析）。



まず、次のスキームを収集します。 回路をRCL.schとして保存この回路は、シミュレーション->シミュレーション（F2）をクリックしてQucsでモデル化でき、すべてが機能することを確認します。







次に、Ngspiceで回路をモデル化します。 [シミュレーション]-> [Spiceでシミュレート]をクリックします。 ダイアログボックスが開きます。







ドロップダウンリスト（1）で、シミュレーターを選択できます。 Ngspiceを選択します。 シミュレーターが非標準の場所にインストールされている場合、シミュレーターの実行可能ファイルへのパスを設定する必要があります。 これを行うには、[設定]をクリックして、次のダイアログボックスに移動します。







この手順は、Windowsに必要です。 上部のフィールド（Ngspice実行可能ファイルの場所）でngspice.exeファイルへのパスを指定する必要があります。 Linuxの場合、通常、デフォルトのパスが適切です。 ここで、前のダイアログに戻り、シミュレートをクリックします。 ダイアログボックスで、シミュレーションが成功したことがわかります。







Save Netlistボタンをクリックして、ダイアグラムをSpice-netlistにエクスポートすることもできます。 回路のネットリストは次のとおりです。 

 * Qucs 0.0.19 /home/vvk/qucs/examples/ngspice/RCL.sch V1 _net0 0 DC 0 SIN(0 0.6 7.5MEG 0 0) AC 0.6 VPr1 _net0 vIn DC 0 AC 0 L1 vIn _net1 10U R1 0 vR 30 C1 _net1 vR 40P .control set filetype=ascii AC LIN 1000 1MEG 10MEG write RCL_ac.txt VPr1#branch v(vIn) v(vR) destroy all TRAN 4.97512e-09 1e-06 0 write RCL_tran.txt VPr1#branch v(vIn) v(vR) destroy all exit .endc .END




[終了]をクリックし、表示ページに移動してグラフを作成します。 デカルト図を配置します。 チャートプロパティダイアログが開きます。







ドロップダウンリスト（1）でRCLを選択し、ドロップダウンリスト（2）でNgspiceを選択します。 次に、グラフを作成する変数を選択します（ダブルクリック）。 ウィンドウの右側のリストに移動します（矢印で表示）。 これでシミュレーションは完了です。周波数応答（仮想電流センサーの電流-ac.i（pr1））と電圧波形（tran.v（vIn）、tran.v（vR））が得られます。 それらは、異なるグラフ上に構築する必要があります。







スキーマノード名はSPICEで受け入れられる表記法であることに注意してください。 プレフィックスは、シミュレーションのタイプ（tran。or ac。）およびシミュレーター（ngspice /）を示します。



特徴





  • ほとんどのQucsコンポーネントはSPICEと互換性があります。 サブサーキット、ライブラリコンポーネントがサポートされています。 シミュレーション中に互換性のないコンポーネントに関するエラーメッセージが表示されます。これらのコンポーネントはSPICEと互換性がありません... Equations Qucsシステムは、SPICEと部分的に互換性があります。 制限についてはドキュメントをお読みください。
  • SPICEと互換性のあるシミュレーションタイプが追加されました。.FOURIER、.NOISE、および.DISTORTION Sパラメーターモデリングは、spice4qucsでは機能しません。
  • Spice4qucsを使用すると、デフォルトのシミュレータではアクセスできない回路をシミュレートできます

    Qucsator。 まず第一に、パワーエレクトロニクス、キー回路、カットオフモードへの移行で動作する半導体デバイスに基づく回路、および多数のコンポーネントを持つ回路です。 たとえば、次のように、高品質のトランジスタサウンドアンプをシミュレートできます。











  • 多くのSPICE互換コンポーネントとモデリングのタイプが追加されました。 これらは、Spiceコンポーネント、Spiceセクション、およびSpiceシミュレーションのグループにあります。 それらの詳細については、ドキュメントをご覧ください。 これらのコンポーネントはすべて、デフォルトのQucsatorシミュレーターでは機能しません。 パラメーター、初期条件、およびNgspiceスタイルのビルダーコマンドの設定を実装します。 また、半導体コンポーネントのモデルを追加し、SPICE形式のモデルの完全な説明を追加しました。 これにより、既存のライブラリからモデルをコピーするだけで、コンバーターをバイパスできます。 このようなモデルを使用した回路の例を次に示します。







  • ライブラリにSPICEネットリストを含めることができるようになりました。 変換をバイパスして、SPICE互換シミュレーターに直接転送されます。 そのようなライブラリの例はこちらです： github.com/ra3xdh/qucs-rus-complib/blob/master/Opamp.lib





さらなる展望





spice4qucsプロジェクトの運命は不明です。 リリース0.0.19とともに、2セットのパッケージがリリースされます-spice4qucsサポートの有無にかかわらず。 このパッチセットをアップストリームに含める問題は未解決のままです。



将来的には、新しいコンポーネントとタイプのモデリング、特にデジタルからアナログへのモデルが追加されます。 Ngpsice、Qucsator、またはXyceの各回路にデフォルトのシミュレーターを割り当て、システムダイアログ（F2）を使用してシミュレーションを実行することができます。



おそらく秋に、Qucsとspice4qucsのソースコードに基づいて、SPICEおよびアナログエレクトロニクスモデリングとの深い統合を目指した、独自のオープンソースプロジェクトの並行開発を開始するでしょう。 デフォルトのシミュレーターQucsatorはカットされ、Ngspiceに置き換えられ、SPICEライブラリーとの互換性が確保され、SPICEと完全に互換性のあるコンポーネントが実装されます。
