c：/ temp / Pys60Cowなど、プロジェクトのどこかにフォルダーを作成し、そこにファイルcow.py、cow.mp3、splash.jpg、my_modules.zipを配置します。

cow.pyの名前をdefault.pyに変更します



次に、インストールされたアプリケーションにファイルを配置することについて話しましょう。

グラフィックなどの必要なファイルを含むスクリプト自体は、x：/ private / UID /フォルダーに分類されます。ここで、xはアプリケーションをインストールするディスク、UIDはアプリケーションのuidです。 このフォルダーは読み取り専用であるため、アプリケーションデータは別のフォルダー、たとえばx：/ data / PyS60Cow /に配置する必要があります。したがって、2つの変数app_pathとdata_pathをプログラムに入力しました。 次に、実際に使用するものに変更する必要があります。



まず、プログラムの有効なUIDを生成します。 これを行うには、ensymbleを実行し、スクリプトディレクトリを選択し、参照をクリックしてフォルダーc：/ temp / Pys60Cowを選択します。 [作成]ボタンからパッキングを開始すると、多くの警告が表示されたウィンドウが表示されます。







そして、これは生成されたUIDです。 default.pyを開き、app_pathとdata_pathを編集します。



app_path = app.full_name（）[0] + u '：\\ private \\ eecc323a \\'

data_path = app.full_name（）[0] + u '：\\ data \\ Pys60Cow \\'



したがって、ensymbleで[詳細]をクリックし、設定リストの[uid]フィールドに0xeecc323aと入力します。 また、アプリケーションのバージョンをX.YY.ZZの形式ですぐに設定します

ここで、データ/ PyS60Cowフォルダーを処理します。このフォルダーは、スマートフォンにインストールし、mu.txtファイルを含める必要があります。 c：/ temp / PyS60Cowでは、他のフォルダーなど、フォルダーを作成します。そのフォルダーには、スマートフォン上にあるべき構造を持つディレクトリが作成されます。 私たちの場合、これらはdata / PyS60Cowのサブフォルダーであり、mu.txtファイルをそこにコピーします。 シンボルの[追加オプション]設定で、オプション--extrasdir = otherを指定します



今度はアイコンの番です。 sisアプリケーションのアプリケーションアイコンはsvg形式である必要があります。 このアプリケーションの既製のアイコンを使用できます。 次に、オプション--icon = path_を追加オプションのアイコンに追加します。たとえば、--icon = c：\ temp \ icon.svg



切望されている[作成]ボタンをクリックし、生成されたsisパッケージをスマートフォンにインストールして実行しようとすると、cgiモジュールの欠如に関するエラーが表示されます。 このモジュールをsisパッケージに入れるのは単に「忘れた」という意味です。 手動で追加し、追加オプション--extra-modules = cgiを追加します。 もう一度入れます。 これでアプリケーションの準備が整いました。好奇心itive盛な心を喜ばせてくれます:)



完成したsisファイルはここからダウンロードできます。パパはパッケージ化の準備ができており、スクリプト、メディアファイルなどが含まれています。
