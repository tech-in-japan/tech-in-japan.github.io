見出しにGoogle Web Fontsを使用することについて何を言うべきですか？



まず、Google Web Fontsコレクションは、一般的に@ font-faceを介してフォントをダウンロードするなど、ヘッダーに最適であることに注意してください。 sIFR や Cufónのように、それに先行する「 クランチ 」も見出しにのみ使用されていました。合理的なサイト作成者は、コマーシャルとレンダリングが遅くなり、マウスでテキストを強調表示することさえできなかったので、ページのメインテキストにそれらを使用しなかったでしょう。 しかし、奇妙なことに、 @ font-faceフォントにも独自の問題があります-これはかなりの量です。



これを確認するには、Google Web Fontsサイトで4つのPT Serifフォントスタイル （「Normal 400」、「Normal 400 Italic」、「Bold 700」、「Blod 700 Italic」）をすべて選択してください。スケールの赤い領域に、 値「 350 」が表示されます。 両方の文字セット（「ラテン語」 と「キリル文字」の 両方）にチェックマークを付けると 、矢印がスケールから外れ、インジケーターに値「 700 」が表示されます 。



正直なところ、これらの数字が何を意味するのか、私には完全な考えがありません。Googleの開発者は、それらにラベルを付けることをまったく気にしておらず、ダウンロード時間またはダウンロードされた情報の量に比例するかどうかも完全にはわかりません。より可能性が高い）。 しかし、私は実験を行うことをheしませんでした。Firefoxにアクセスしたとき、「Normal 400」スタイルのチェックマークのみを残し（その後、「Page Load」インジケータの値は「 90 」 を示しました）、 http：//fonts.googleapisでCSSコードを読み取りました.com / css？family = PT + Serifそして、このコードに含まれるURLからフォントをダウンロードしました。 WOFFファイルは43,880バイトのサイズでダウンロードされたので、ページ読み込みインジケータが特定の量を示している場合、これは確かにWOFF形式のフォントのキロバイト数ではないことを考える理由があります-それはおそらくEOT形式のフォントサイズ（通常、フォントはWOFFの約2倍の重量があります）。 ちなみに、エクスプローラで同じアドレスを開くと、CSSコードに EOTフォントにつながる別のURL があります。そのフォントの重さは87 122バイトであり、これは間接的に私の仮定を確認することを意味します。 （「ページの読み込み」インジケータは丸められる場合があります。）



しかし、これはたくさんあります。 サイトへの最初のアクセス中に数百キロバイトのフォントがインターネットからダウンロードされる場合（4つの異なるスタイルに必要）、無制限の高速インターネット（LTE、WiMAX、GPON、ETTH、FTTB、ADSL）のユーザーはおそらく何も気付かないかもしれません-そして、サイトの他のすべての読者は確かに動揺するでしょう。 そして、彼らはinりの中で正しいでしょう。 特に、トラフィックの支払いがメガバイト単位であるモバイルインターネットのユーザーや、トラフィック量が厳しく制限されている企業インターネットのユーザーにとっては残念です。



これが、慈悲深いウェブサイトの作者（つまり、心を痛めている作者）が一度に4つのフォントスタイルを使用するのにふさわしくない理由です。つまり、珍しい（インターネットからダウンロードした）フォントを設定する場合、作者はヘッダーのみに制限されるべきです：ヘッダー、スタイル原則として、1つだけです（たとえば、ストレートまたはイタリック体のみ）。 同時に、選択の幅が広がりつつあります。誕生からの多くのフォントは単一のスタイル（4つまたは2つではなく）を持ち、それ自体が見出しにのみ適しています。



さらに、すべての見出しがロシア語のアルファベットの文字のみで構成されている場合（これが例になっているCyclopediaの場合も同様です）、フォントでファイルのサイズをさらに節約できます。



Google Web Fonts Webサイトにアクセスして、単一（直接）の文字スタイルを持つCuprumフォントを選択します。 ページ読み込みインジケータには43の 値が表示されます 。 以下に、文字セットの設定に使用される「ラテン」 および「キリル」チェックボックスが表示されます。「ラテン」チェックボックスがチェックされ（グレー：チェックを外すことはできません）、「キリル」チェックボックスは設定されませんが、その後にインジケータを配置できます「ページの読み込み」には値「 86 」が表示されます 。 その後（少なくともFirefoxで）、「ラテン語」チェックマークは灰色でなくなり、削除してインストールできますが、「ページ読み込み」インジケータのステータスは影響しません。



これは重要です。 Googleシステムの動作を見ると、ラテン文字がフォントファイルにまったく無条件に含まれているという結論に達するのは簡単です。つまり、ラテン文字を無効にする機能がまだある場合は、ダウンロードしたフォントのサイズを半分にすることができます。



これを非選択的に実現する方法は、Google Web Fontsのドキュメントの「 フォントリクエストの最適化（ベータ） 」セクションに簡単に記載されています。 ダウンロードされたCSSコードの URLは、必要なすべてのフォント文字を含む「text」パラメーターで補完できることがわかりました。 ダウンロードしたフォントのファイルには、これら以外の文字はありません。



したがって、サイトで見出しを提供する前に、切望されている「 フォントファミリ：Cuprum、 sans-serif; 「、このディレクティブをCSSコードの先頭に配置することが最も適切です 。

この場合にダウンロードされるWOFFファイルのボリュームは、最適化のため、つまり限られた文字セットを選択するために、 20,840バイトに削減されます 。

もちろん、フォントを使用してファイルのサイズをさらに小さくすることも可能です。パラメータ「テキスト」がヘッダーで実際に使用されている大文字のみに制限され、すべてをリストするのではない場合に達成されます。 ただし、フォントが使用されているページの見出しの編集を厳しく制限するため、これは決して便利ではありません。 文字の一般的な節約量は約4キロバイトです。つまり、それだけの価値はありません。
