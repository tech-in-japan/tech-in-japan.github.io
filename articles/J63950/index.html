MS Project ServerまたはRakeへの接続-豪華ではない





今日、MS ProjectをWindowsマシンにインストールしましたが、2つの明らかな問題に遭遇しました。



愚か者は彼の間違いから学び、賢い人は見知らぬ人から学びます-私は読者の中でもっと賢い人がいることを願っています。





MS Project Professional 2007 + MS Project Serverについてです。 Projectが何であるか誰にもわからず、それでも猫に夢中になった場合は、簡単にコースを紹介します。MSProjectは多機能プロジェクト管理ツールです。 MS Exchangeとともに、このツールは大規模な組織で非常に人気があります。 「レーキ」について説明します。これは、サーバー管理者ではなく、通常のユーザーとして私の目の前にいることに注意してください。



レーキは3ピースでした-2つは大きく、1つは小さい。



大きな熊手：



1.エラー「不正な認証局。 （ID = 0x800a152a） "Project 2007からサーバーに接続しようとしたとき。

同時に、同じパスワード名で（またはドメインアカウントからログインしている場合はまったく指定せずに）、e-webインターフェイス（いわゆるPWA-Project Web Access）経由で正常にログインできます。



問題の理由は、署名された証明書がないことにあります。 解決策はいくぶん自明ではありません-証明書はMSIE設定にインストールする必要があります。これは、セッションの暗号化に使用するものだけでなく、署名に使用するものすべてにインストールする必要があります。



2.エラー「間違ったユーザー名/パスワードを入力しました」、同書。

サーバー自体がドメインで実行されている場合、ドメインに入力されていないマシンからMS Projectを介してサーバーにログインできません。



私は問題を解決できませんでしたが、2つの方法のいずれかを回避できます-実際には、ドメインにクライアントを入力するか、次の回避策を適用します：IE経由でPWAに移動し、（新規）->（そこでプロジェクト）をクリックします目的のプロジェクトを開きます。



小さな熊手：



3.ガントテーブルを表示するために必要なActiveXアドオンをインストールできない。

PWAを使用するには、インストール済みのActiveXアドオンが必要です（この理由から、WebインターフェイスはIE以外のブラウザーでは完全に機能しません）。 ただし、ドメインポリシーでActiveXコンポーネントをインストールできない場合があります。



ローカル管理者アカウントを持っている場合、問題を解決できます。その下からWebインターフェイスにアクセスし、アドオンをインストールしてから、ドメインアカウントにログインする必要があります。
