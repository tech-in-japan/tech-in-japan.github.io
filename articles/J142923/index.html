快晴





ネットワークは、クラウドファイルストレージサービスのどれが他のサービスよりも優れているかを喜んで主張していますが（ Habrも例外ではありません ）、すべての主要な製品を一度に使用して、同期とバックアップ用の個人用ソリューションをアップグレードすることになりました。







アイデア





Googleドライブの統合、 Dropboxの普及とパブリックリンク、および大量の空きディスク容量Microsoft SkyDriveのすべてのユーザーニーズを提供するために、各サービスの利点を使用することにしました。 併用の考え方は非常にシンプルであり、導入のオイラー-ベン図で明確に示されています。同期されたフォルダーをボリューム定義の順序で相互に配置します。



状況





1.私はDropboxを1年以上使用してきましたが、この間、このサービスの独創的なシンプルさと柔軟性を完全に評価することができました。 異なるプラットフォームで5つの異なるデバイスを同期しますが、主なものは次のとおりです。

  • Ubuntuのデスクトップコンピューター。
  • ネイティブのWindows 7 Starter上のネットブック。
  • Androidスマートフォン。


あらゆる種類の共有では、Box内のスペースは6.5 GBでしたが、4 GBを超えることはありません。これは、一意の個人ファイルのみを保存するためです。 さらに、公開リンクを非常に積極的に使用しています。 主に、ライブWikiを運用知識ベースとして使用する場合、これは完全にではなく、別の話です。 それにしても、便利なパブリックリンクの機能は私にとって非常に重要です。



2. Google製品を5年以上使用してきましたが、この間、Googleドライブが重要な統合ハブとなるように設計されたエコシステムの不可欠な部分になりました。 既にメールの添付ファイルをすばやく保存し 、サードパーティのChromeアプリケーションのファイルを操作する機能（ LucidchartとPixlr Editorを引き続きサポートしています）、Google +への写真とビデオの直接アップロード、およびGoogleドキュメントにあるすべての重要な利点があります（強力です）検索、柔軟なアクセス制御、そしてもちろん、コラボレーションの可能性）。 Android版ドライブでは、Dropboxよりも楽しく動作するようです。



3.私は半年前にSkyDriveを使い始めました-偶然です。 友人が彼を通してファイルを共有しました。 その後、Windows Live Movie Makerのために「Windows Live 2011 Essentials」パッケージをインストールし、 Live Meshに注目し、Dropboxフォルダーのバックアップを開始しました。 実際、これにより同期サービスを共有するというアイデアが始まりました。 一般に、SkyDriveは、大容量ファイルの同期に使用できる25 GBのスペースを提案しました。



解決策





「曇りを介して」の組織は、ロシアの人形を収集する原則に戻っています。

  1. ストレージ内で利用可能なボリュームが最大のアプリケーションがインストールされます。私の場合はSkyDriveです。 そのフォルダーには、論理ドライブ上の場所が選択されます。 最終ファイルへのパスの長さを最小化するために、Dドライブを直接指定しました（ところで、SkyDriveディレクトリ内のフォルダーへのパスの長さは、ファイル名を含めて255文字を超えてはならないため、サブディレクトリとファイルに名前を付けるときは簡潔にする必要があります）。
  2. より小さいボリュームのアプリケーション、つまりDropboxがインストールされます。 フォルダーは、SkyDriveフォルダーに直接配置することはできません。これは、システム全体と見なされるためです。 そこで、Cloudディレクトリを作成し、そこにDropboxフォルダーを配置しました。 うまくいきました。
  3. 使用可能なボリュームが最小のアプリケーション（Googleドライブ）がインストールされます。 結果のファイルへのパブリックリンクを受信できるように、彼のフォルダーをPublic Dropboxフォルダーに配置しました。 burdakovdからの重要な説明 ： サードパーティがDropboxのユーザーコードを知っている場合（パブリックリンクのいずれかに表示される場合）、サブディレクトリ内のファイル名（リンクに表示される名前も）を反復する機会があります。 したがって、重要なファイルを公開することはお勧めできません。重要なファイルがGoogleドライブから自動的にアップロードされないようにする必要があります 。
  4. さらに、Live Meshを設定して同じパブリックをバックアップします。 これは、25 GBのSkyDriveプライマリストレージから独立した、純粋な5 GBのバックアップです。


既にインストール済みのアプリケーションがある場合は、ほぼ同じ手順で既存の同期フォルダーを正しい順序で移動し、プログラムを新しい場所にリンクできます。



結果





結果は、例として紹介イラストファイルを使用して示すことができます。

  • この画像をGoogle図形描画で作成し、適切な形式のファイルとしてGoogleドライブに自動的に保存しました。
  • DropboxフォルダーのPublicサブディレクトリにあるGoogle DriveフォルダーのPicsサブディレクトリにPNG画像をアップロードし、Habrに画像を追加できるようにファイルへのパブリックリンクを作成する機会を得ました： dl.dropbox.com/u/3442793/Google%20Drive /Pics/CrossCloud.png
  • 過剰なトラフィックのため、 UPD Dropboxは一時停止されています。 すぐに同じファイルをGoogleドライブに公開し、記事の画像のソースを置き換えました。 私の意見では、リスク管理の観点から提案されたモデルの有効性をある程度証明していると思います。
  • ファイルは 、Googleドライブを介してAndroidのスマートフォンで、Dropboxを介してUbuntuのデスクトップコンピューターで、SkyDriveを介してWindowsのネットブックで利用可能です 。
  • ファイルの保存 ：上記のすべてのデバイス、Googleサーバー、Dropboxサーバー、Microsoftサーバー-SkyDriveのメインセクションとLive Meshに保存されます。
