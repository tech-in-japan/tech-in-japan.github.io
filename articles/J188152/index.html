ハッシュ関数Stribogまたは都市で新しい保安官

2012年、情報セキュリティに多少なりとも関係する一般の人々全員が、新しいSHA-3データハッシュ標準の選択を注意深く監視しました。 この重要なイベントはハバーで広く取り上げられました：コンテストの各ラウンドの結果（1、2、3）が公開され、新しい標準が説明され 、新しい標準でさえ、なぜそれがとてもクールだったのか説明されました 。

しかし、この誇大広告の背後に、もう1つの重要なイベントがまったく気付かれていませんでした。2013年1月1日、ハッシュ関数の標準もロシア連邦で変更されました。

だから、会う：新しい標準の完全な説明とC＃での実装。 彼らが言うように、遅くなることはないよりはましだ



番号34.11-2012を受け取った新しい標準として、Stribogと呼ばれるハッシュ関数が採用されました。 stribogは、2つの関数のみを含むハッシュ関数のファミリーです。 256ビットの出力値を持つ関数と512ビットの出力値を持つ関数。 これらの関数は両方とも同じ構造を持ち、初期の内部状態のみが互いに異なります。 両方の関数の入力は512ビットのデータブロックです。 メッセージの長さが512ビットを超える場合、メッセージはブロックに分割されます。 長さが512ビット未満の場合、メッセージは補足されます。



アルゴリズムの説明に進む前に、ハッシュ関数の計算に使用される変換について説明します。

圧縮機能



ハッシュ関数のコア要素は圧縮関数です。 新しい標準で使用される圧縮関数g nをアルゴリズムの形式で説明します。

h 、 N、およびmを512ビットシーケンスとします。 関数g（N、m、h）を計算するには、次の手順を実行する必要があります。

  1. 値K = h⊕Nを計算します。
  2. 値K = S（K）を割り当てます。
  3. 値K = P（K）を割り当てます。
  4. 値K = L（K）を割り当てます。
  5. t = E（K、m）を計算します。
  6. 値t = h⊕tを割り当てます。
  7. 値G = t⊕mを計算します。
  8. 関数g（N、m、h）の計算結果としてGを返します。




おそらく、関数g（N、m、h）の説明では、なじみのない関数E（K、m）が使用され 、次のアクションを実行することに気づいたでしょう。

  1. 値state = K⊕mを計算します。
  2. i = 0から11の場合：

    • 値state = S（状態）を割り当てます。
    • 値state = P（状態）を割り当てます。
    • 値state = L（状態）を割り当てます。
    • Kの計算= KeySchedule（K、i）;
    • 値state = state⊕Kを割り当てます。
  3. 結果として状態を返します。




そしてまた不慣れな概念。 今回は、これがKeySchedule（K、i）関数であり、E（K、m）関数の各ラウンドで一時キーKを生成します。 アルゴリズムの説明でこのギャップを埋めます。 KeySchedule（K、i）関数は、次の計算を実行します。

  1. 値K = K⊕C [i]を割り当てます。
  2. 値K = S（K）を割り当てます。
  3. 値K = P（K）を割り当てます。
  4. 値K = L（K）を割り当てます。
  5. 関数の結果としてKを返します。

ハッシュ関数の計算



次に、ハッシュ値を生成する手順を説明します。 入力メッセージMの場合：

  1. 内部変数に初期値を割り当てます。

    • 512ビットの出力長を持つハッシュ関数の場合：h = iv = 0x00 64 。 出力長が256ビットのハッシュ関数の場合：h = iv = 0x01 64 。
    • N = 0 512
    • Σ= 0 512
  2. 次の条件を確認してください。メッセージ長M <512。 条件が満たされている場合は、手順3に進みます。それ以外の場合は、一連の計算を実行します。

    • m-メッセージMの最後の512ビット
    • h = g（N、m、h）
    • N =（N + 512）mod 2 512
    • Σ=（Σ+ m）mod 2 512
    • 最後の512ビットを削除してMをトリミングする
    • ステップ2に進む
  3. 次の規則に従って、メッセージMを512ビットの長さに補足します。

    m = 0 511- | M | || 1 || M、ここで| M | -ビット単位のメッセージ長M。
  4. h = g（N、m、h）を計算します
  5. N =（N + | M |）mod 2 512を計算する
  6. Σ=（Σ+ m）mod 2 512を計算
  7. h = g（0、h、N）を計算します
  8. h = g（0、h、Σ）を計算する
  9. 出力長が512ビットのハッシュ関数の場合、結果としてhを返します。 出力長が256ビットの関数の場合、MSB 256 （h）を返します。
