サイトのオンラインコンサルタントの比較：選択した小麦粉

この記事は広告ではないことをすぐに警告します。 オンラインコンサルティング用の高品質のツールを選択した経験を共有することのみを目的として書かれたものであり、彼ら自身は検索中に詳細な分析で適切な資料を見つけることができなかったためです。



サイトでオンラインチャットを開始することを決めたとき、どの機能が必要かを明確に理解できませんでした。 私たちがすぐに明確に理解したのは、これはチャットルームではなく、ランダムな訪問者が「生きている人はいますか」と書くことができるということです。 そして、私たちの製品の技術的特徴にあまり詳しくないコンサルタントの女の子が訪問者からのメッセージに応答するようなものではありません。











販売および技術サポート部門のすべての従業員がオンラインコンサルタントのオペレーターになることを明確に決定しました。 マーケティング部門の長であるサポートおよびセールスとの数回の会議およびブレーンストーミングセッションの後、オンラインコンサルティングサービスが満たすべき重要なパラメーターをいくつか特定しました。 すなわち：

  • 自己紹介の必要性 ：オンラインコンサルタントを作成することを決定した各ユーザーは、名前、電子メール、および電話番号を残す必要があります（後者はオプション）。
  • 部門の選択 ：質問の主題に応じて、訪問者は連絡したい部門を選択する必要があります。
  • バイリンガリズムと、複数のサイトを1つのアカウントに接続する機能：複数のサイトがあり、すべてが多言語です。
  • ダイアログを別のコンサルタントにリダイレクトする：完全なオンラインチャットであ​​っても、リクエストがターゲットに正確にヒットすることを100％確信することはできません。
  • Windows / Linux とのオンラインチャットを使用する機能：従業員（50/50）は両方のシステムを使用しており、何も変更したくありませんでした。


これらは、クライアントへのオンラインコンサルティングにとって理想的なサービスがどのように見えるべきかという基本的なアイデアでした。 これらのパラメーターによると、20以上のサービスを分析しましたが、そのうち8つだけが準決勝に達しました（一部のサービスは、システムを視覚的に知り合った直後に落ちました）。



次のサービスが8つのファイナリストに含まれます： Jivosite、p3chat、Livetex、Webim、Webconsultant、Krible、Zopim、Olark。



それぞれを20のパラメーターで推定しました。

その結果、私たちが最も気に入った上位3つのサービスが決勝に達しました。 デモバージョンの有効期間中に3つすべてを実際にチェックしました（この種のすべてのサービスは例外なく、14日間無料でシステムをテストすることを提案しています）。



P3chat：信頼性が高く、機能的で、あまり美しくない



オンラインコンサルタントを従業員のジャバークライアントに結び付けることが最も便利であると判断したため、このチャットから始めました。 また、ダイアログの翻訳、訪問者のブラックリストへの送信などを可能にする多数のコマンドをサポートしているため、p3chatにはこの機能が最適に実装されています。



長所：

  • 特別なチームを持つすべてのジャバークライアントのサポート。
  • 無制限の数のサイトおよび部門;
  • 多言語主義;
  • オペレーターに優先順位を付ける機能。
  • 別のオペレーターへのダイアログ転送。
  • 顧客情報（国、ブラウザ）;
  • サイトを遅くしません。
  • 独自の「チャットを開始」ボタンを描画する機能。


短所：

  • 非現代的なデザイン。
  • 追加のウィンドウでチャットを開く;
  • 自分でデザインをカスタマイズする能力の欠如;
  • 対話のトランスクリプトをメールに送信することはできません。




ジボサイト：ファッショナブルで美しい、それほど深刻ではない



p3chatの後、ユーザーにとってもオペレーターにとっても美しくて便利なものを試すことにしました。 このサービスは、チャットを操作するためのいくつかのオプションを提供します。Webバージョン、コンピューターにインストールされたアプリケーション、またはjabberクライアント（ただし、有用なコマンドのサポートなし）です。



長所：

  • 美しいカスタマイズ可能なインターフェース設計。
  • 部門とサイトの数に制限はありません。
  • ページでチャットする（追加のウィンドウで開かない）;
  • 多言語主義;
  • 他のオペレーターへの対話の転送。
  • チャットを評価する機能。
  • クライアントに関する詳細情報（メッセージの作成者がサイトにアクセスした検索クエリまで）。


短所：

  • jabberクライアントの限定サポート（ヘルプなどのコマンドなし）。
  • アプリケーションはWindows / Mac OSでのみ動作します（Linuxではサポートされていません）。
  • 会話のトランスクリプトをメールに送信することはできません。
  • メッセージ履歴は、対応したオペレーターと管理者のみが利用できます。
  • オペレーターに優先順位を付ける方法はありません。メッセージは全員に届きます。




Zopim：すてきで、スマートで、完全にカスタマイズ可能



本当に良いツールを見つけようと完全に必死になっていたときに、偶然に面白い名前でこのチャットに出会いました。 私たちは、jabberクライアントとインストール済みアプリケーションの存在をサポートするという考えを既に放棄しました。



機能（調査対象のオペレーターとクライアントの両方によって策定された要件）と外観の主な要件を作成したため、Webバージョンでのみ機能する適切なサービスを見つけても動揺しませんでした。



Zopimでは、すでに無料期間を終了しており、サービスの使用をさらに1か月延長しました。 結局のところ、システムのすべての機能、長所、短所を評価するには2週間では不十分です。



時間が経つにつれて、このサービスにはさらに多くの欠点が見つかる可能性があります（私たちの食欲も食物とともに成長します）。 しかし、これまでのところ、これは、私たちが検討した人たちにとって最高のサービスです。



そして、結果として選択したシステムの長所と短所（私たちにとっては重要ではない）についてより具体的に説明します。



長所：

  • 美しく完全にカスタマイズ可能なインターフェイス。
  • 優れたAPIドキュメント
  • 部門とサイトの数に制限はありません。
  • ページでチャットする（追加のウィンドウで開かない）;
  • 多言語主義;
  • 他のオペレーターへの対話の転送。
  • ユーザーに関する詳細な統計。
  • チケットシステムをバインドする機能。
  • ユーザーが自己紹介する前にメッセージを送信できないようにチャットを構成する機能（FacebookおよびGoogle+との統合）、電子メールを残し、目的の部門を選択します。
  • ダイアログ履歴をメールに送信します（この機能はオペレーターとユーザーの両方が利用できます）。
  • チャットを評価する機能。
  • コンサルタントのアカウントのリンクされた翻訳者。
  • すべての従業員が履歴を表示できます。
  • ユーザーに関する情報を入力し、コメントを残す機能。クライアントが再連絡した場合でも、すべてのオペレーターが利用できます。
  • 未処理のアプリケーションは、すべてのコンサルタントに表示されます。
  • サイトの特定のページからチャットを削除する機能（これは私たちにとって重要でした。一部のページは追加のオーバーロードを望まなかったためです）。


短所：

  • オペレーターのWebバージョンのみの入手可能性。
  • コンサルタントに優先順位を設定することは不可能ですが、それほど多くのアプリケーションはありません（メッセージを送信する前の登録という形のスループットシステムのため）。そのため、すぐに答えることができないオペレーターをそらすことができます。
