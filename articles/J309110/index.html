ホスティングロジック：捕まっていない-泥棒ではありません。 反対を証明する方法。 HostTrackerサービスの概要、パート3

監視サービスHostTrackerは 、サイトでの作業と、カジュアルからWebマスターまで、あらゆるレベルの専門家の問題の診断を容易にする一連のツールを作成しました。 特に、これらのツールは、さまざまなホスティングの問題を迅速に診断および修正するのに役立ちます。たとえば、実際のSLAが規定を満たしているかどうかを評価します。 一致しない場合は、より優れたサーバーへの転送を要求するか、より優れたホスティングを見つけてください。 問題の理論と実践は何ですか-カットの下を見てください。







レビューの前の部分（ 最初と2番目 ）で、サービスの基本的な機能と作成の歴史について話しました。 この部分は、 HostTrackerによって収集されたデータの実用的なアプリケーションに専念しています。



ログには何が表示されますか？



HostTrackerの当初の目標の1つは、質の悪いホスティング業者との突き合わせの証拠を提供することです。 近年、彼らはより責任を負うようになりました（そして、この傾向が続くことを願っています）。 それにもかかわらず、現在でも、1人または別のホスティング事業者が明白なことを否定し、その罪悪感を認めない状況が頻繁にあります。 しかし、ここでは、HostTrackerの人物のスーパーヒーローが救助に出ます。ログから、障害が発生したときと、それが正確に何を示したのかがはっきりとわかります。 実際、ログではすべてが簡単です-しかし、便宜上、それらはいくつかのカテゴリに分けられます。 まず、チェックの実際のログ：







デフォルトでは、同じステータスのチェック（サイトが常に実行されている場合、または逆に同じエラーが発生する場合）は、簡単に表示できるように1つのレコードに結合されます。 それでも-そうでなければ、1日に1分の監視間隔で、1440レコードが生成されます。必要なものを見つけてください。 ただし、何かを特に慎重に確認する必要がある場合は、個別のログ記録の可能性があります。 また、検索の便宜上、エラーのリストが個別に作成されます。 さらに、特定のエラーについては、スナップショット（エラー発生時のサイトの状態）を保存して表示することができます。 もちろん、これは、チェックされているサーバーから少なくともいくつかの応答が受信された場合に機能します。 たとえば、キーワード検索は機能しました-重要なものがページから消えました（または表示されました）。



第二に、ダウンロード速度と応答時​​間をチェックするためのグラフがあります。これは、多くのタスクに役立つ場合があります。 特に、HostTrackerサーバーが世界中にあることを考慮してください。 タスクの設定で、チェックする世界の地域を選択することができます。 したがって、世界の特定の地域のユーザーに対するサイトの動作を確認できます。 これは便利です。サイトのターゲットオーディエンスが存在する国から確認できます。オーストラリアからは確認する可能性はありますが、異なるオーストラリアについてはあまり心配する必要はありません。







障害がいつ発生したかをすばやく特定し、稼働時間チャートを使用してサイトの履歴を表示できます。 ちなみに、ウィジェットを使用すると、サイトに簡単に配置できるため、HostTrackerにアクセスするたびにアクセスできなくなります。







もちろん、サーバーのログにアクセスできる場合、サイトの問題のほとんどはログにあります。 しかし、最初に、どこを見るべきかを知る必要があります。 そして第二に、問題は外部のものである可能性があり、内部ログにあまり長く留まらない。 これらのデータを組み合わせることで、問題を診断し、将来を予測できます。



データ：自分自身とホスティング事業者との関係



その結果、次のデータが記録されます。



  • ドメインと証明書のステータス。
  • チェックされたIPアドレス（サイトが複数のアドレスに解決できる場合）。 これは、データ配布システムを使用しているお客様に役立ちます-エラーがどこにあるかをすぐに見つけることができます。
  • 応答時間とダウンロード速度。 検証の地域を制限し、たとえば東ヨーロッパからのサイトの読み込み速度を監視することができます。 または、全体像を作り上げます。
  • 可能であれば、記録されたすべてのエラーのコード-スナップショットはエラー中に作成されます。
  • より複雑なタスク（たとえば、データベースの監視）の場合、さまざまな値、チェックサムなどを保存できます。


この情報へのリンクは公開することができます。また、このデータをPDF / XML / CSV形式でダウンロードし、ホストの不名誉の証拠として親切に提供することもできます。 クライアントが低品質のサービスの補償を求めた場合がありました。 多くの場合、これらのチェックはホスティング事業者を刺激してサービスの品質を向上させました。 少なくともこれらの顧客のために。 さらに、これらのレポートによるホストの体系的なクラッシュは具体的な結果をもたらすことを実践が示しています。サイトをより高速なサーバーに移動する、ホストのサポートの側からサイトに真の注意を払うなどです。 別のホスティングに移行する場合もあります。 または、たとえば、サイトのトラフィックの増加により、より高い料金プランに切り替える必要がある瞬間を捉えることができます。



私たちのサービスは、まず第一に、顧客、希望者、批評家のリクエストにより発展していることを思い出してください。
