Snow LeopardにXcode 4.2をインストールする

どうやら-これはすべて昨年行われたものであり、問​​題は発生しないはずです。 しかし、そこにありました。 今日、バックアップからシステムを復元する必要があり、古いXcodeがあります。 問題はそれほど深刻ではないようです。開発者アカウントは正常です-developer.apple.com/xcodeにアクセスしてダウンロードしてください。 しかし、Xcode 4.3.2があり、ご存じのように、Lionのみに配置されています...



Lionにアップグレードしないことを選択する理由はさまざまです。 誰かが次のバージョンがすぐに来るということを知って節約することに決めました。 誰かがすべてをセットアップしただけで、とにかく動作するものを変更するのが面倒です。 そして、AMDプロセッサを搭載したシステムにhackintoshをインストールするのに苦労し、彼のLionは動作しません。 これは別の議論のトピックです。 システムをLionにアップグレードしたくないことが重要です。 特に今、マウンテンライオンの前に数ヶ月が残ったとき。 必要なのはiOS 5用のSDKだけです。このSDKはXcode 4.2にあり、完全に機能し、完全に問題ありませんでした。



追加のダウンロードで、必要なバージョンが見つかりました。 しかし、ダウンロードした後、私は常に未知のエラーを受け取りました：







エラーの原因は/var/log/install.logで見つかりました。



Failed install preflight: Error Domain=PKInstallErrorDomain Code=102 UserInfo=0x1112aab20 "The package “OpenGLApps.pkg” is untrusted." Underlying Error=(Error Domain=NSOSStatusErrorDomain Code=-2147409654 UserInfo=0x1126642c0 "The operation couldn't be completed. CSSMERR_TP_CERT_EXPIRED")







明らかに、問題は「OpenGLApps.pkg」の証明書にあります。 CSSMERR_TP_CERT_EXPIREDをグーグルで検索すると、 stackoverflowが発生しました 。 解決策は恥ずべきことです-証明書がまだ有効だった日付を愚かに過去に設定します。



実際には、投稿の本質：



1. Snow LeopardのXcode 4.2 はAppleのWebサイトにあります（ダウンロードするには開発者アカウントが必要であり、ログインする必要があります）。

2.すべてを確立するには、日付を変更するだけで十分です（スタックオーバーフローについて-2012年3月26日まで）
