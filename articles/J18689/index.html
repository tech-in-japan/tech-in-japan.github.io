[プラグイン] Ajax Comments 2.09（無料）

長い間、私はプラグインをアップロードしませんでしたが、何とか試してみました。



それで、私は何について話しているのですか？ ああ、おそらく私のバージョンのAjax Commentをダウンロードしてインストールした人が数人いるでしょうが、これは主なことではありませんが、特定の周期性を持つブログを持っているという事実は、コメントを送信するときにエラーが発行されました サーバーは時間通りに応答しませんでした。」

私はこのプラグインの新しいバージョンを探すために登り、「Pay $ 1」というメッセージを喜んでいた。 数か月前にお金を買ってすみませんが、支払いの可能性は登録するのが面倒だったPayPalを介してのみでした。

ここで先日、新しい（2.09）バージョンを登録して購入しましたが、本当に気に入ったので、著者の機能をリストし、最も興味深いものを赤で強調します。

  • テンプレートコードを変更する必要はありません。 アクティベーション後すぐに動作します。
  • WordPressプラグインAPIを使用しているため、他のプラグインと連携して動作します。
  • CAPTCHA検証とフォーム検証は、ページを更新せずにサーバー側で行われます。
  • エラーメッセージは、コメントフォームの上の赤いボックスに表示されます。
  • コメント投稿画像は、どのデザインでも見栄えがします。
  • アーカイブには、ソースを掘り下げたい人のための詳細なドキュメントが含まれています。
  • ローカライズとUnicode文字のサポートにより、あらゆる言語のコメントに対応します。
  • jQueryフェード効果により、読者はより幸せになります。
  • サーバーで60秒のタイムアウトが発生しても、リーダーはハングしません。
  • Firefox、Internet Explorer 6/7、Opera、Netscape、およびSafariの現在のバージョンでテスト済み。


プラグインは、以前は無効にする必要があったWP AJAX Edit Commentsで正常に動作するようになりました。 プラグインがcaptchaで適切に動作を開始するために変更する必要さえなかったので、数行のエラーをロシア語に翻訳するだけでした。

そして最も重要なことは、私の意見では、Prototype + ScriptaculousライブラリのJavaスクリプトの代わりに、より軽い（サイズの）jQueryが使用されていることです。

さて、ここにAjax-Comments 2.09プラグへの参照があります-それ自体（90 kb）、著者は無料で配布することを大したことはないと思います。なぜなら、 GNU GPL v2に加えて行うには、ソースコードに変更を加えました）））



私はすぐに予約します-プラグインはほとんどテストしていませんので、コメントでそれを「苦しめ」、作品に関するフィードバックを残してください。

PyT：プラグインはWP 2.1-2.3.xおよびWP MUで動作します。
