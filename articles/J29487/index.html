Microsoft Crm 4.0 WebServicesエラー

問題：

MS CRM 4.0では、サイトをMS CRMサイトの仮想ディレクトリに配置する問題に遭遇しました。



エラーは次のとおりです。

Microsoft.Crm.WebServices.Crm2007.MultipleOrganizationSoapHeaderAuthenticationProvider、Microsoft.Crm.WebServices、バージョン= 4.0.0.0、カルチャー=ニュートラル、PublicKeyToken = 31bf3856ad364e35「存在しません」。

パラメーター名：Microsoft.Crm.WebServices.Crm2007.MultipleOrganizationSoapHeaderAuthenticationProvider、Microsoft.Crm.WebServices、バージョン= 4.0.0.0、カルチャー=ニュートラル、PublicKeyToken = 31bf3856ad364e35



解決策：

Dynamicsフォーラムで発見されました。

ソリューションは完全に妄想的です（なぜこれを行う必要があるかは明確ではありません）が、サイトを立ち上げることができた唯一の方法です。 GAC'eのMS CRMサーバーにMicrosoft.Crm.WebServices.dllライブラリを登録する必要があります（CRM Webインストールの％パス/ビンにあります）



gacutil / i Microsoft.Crm.WebServices.dll



PS gacutil.exeが見つからなかった場合、エクスプローラーで目的のDLLをWINDOWS \ assemblyフォルダーにドラッグするだけです。
