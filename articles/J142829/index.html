「悪い」サイトはモバイルデバイスのバッテリーを消耗します

モバイルデバイスからインターネットをサーフィンするのが好きで、ウィキペディアにアクセスすると、スマートフォンがHabréの記事を読むときよりもはるかに速く放電することに定期的に気付いた場合、これについて合理的な説明があります。

スタンフォード大学の研究者チームは、モバイルブラウザーでサイトを開いたときのバッテリー放電の依存性を調査する作業を実施しました。

テスト対象のデバイスとして、3Gインターネットを搭載したAndroidデバイスを選択しました。CNN、BBC、Gmail、Amazon、Microsoft、eBay、Yahoo、Facebook、IMDB、Apple、Wikipediaなどの人気サイトがオープンしました。







予想どおり、最もエネルギーを消費するのは、元々モバイルWeb用に最適化されていなかったサイトです。 これは、たとえば、Tumblr、Blogger、WordPressなどのサイト、およびバッテリー放電サイトの中で一般的に勝者となったAppleサイトです。

ただし、ウィキペディア、IMDB、Go.comなどのモバイルバージョンのサイトでさえ、スマートフォンの料金を節約するのにあまり役立ちませんでした。







最も多くのリソースを消費するプロセスは、画像のレンダリング、キャッシングの存在とタイプ、JavascriptとCSS処理、特にアニメーションがある場合は論理的であるため、調査中、基本的にはレイアウトに関することがわかりました。



この記事では、ページを最適化するためのヒントも提供しています。

たとえば、画像に触れた場合、研究者はJPGとGIFを支持してPNG形式を放棄するように促しています。

また、CSSの最適化も重要です。 最適でないスタイルシートは、Appleページが他のどのスマートフォンよりも悪い主な理由の1つです。







JavaScriptの最適化も同様に重要です。 jqueryのみをロードするには5ジュールかかります。 そのようなライブラリがいくつかある場合、私は何を言うことができます。



ただし、この研究では批評家が登場しています。 その中には、レミーシャープも含まれています 。レミーシャープは、記事の要点を説明し、コメントを提供しました 。 特に、彼はGmailの研究に同意していません。Gmailは、偶然、タイトル、最も「グリーン」なモバイルサイト、つまり モバイルデバイス用に最も最適化されたサイト。



この研究の結果は、先週リヨンで開催されたWorld Wide Web 2012カンファレンスで公式に発表されました。 近い将来、このトピックに関する一連の記事やコメントが期待できると思います。 モバイルWebは飛躍的に発展しているため、トピック自体の重要性はますます高まっています。これは、現代のスマートフォンのバッテリーについては言えません。
