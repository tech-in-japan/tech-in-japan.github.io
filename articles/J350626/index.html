サーバーとクライアント側



これまで従ってきた従来のサーバーモデルには、アプリケーションサーバーへのHTTP要求を実行するクライアント（ユーザーが制御するWebブラウザー）があります。 リクエストは、たとえば「 プロフィール 」リンクをクリックしたときにHTMLページをリクエストしたり、プロフィール情報の編集後に「送信」ボタンをクリックしたときにアクションをトリガーしたりできます。 どちらのタイプのリクエストでも、サーバーは新しいWebページを直接またはリダイレクトしてクライアントに送信することにより、リクエストを実行します。 次に、クライアントは現在のページを新しいページに置き換えます。 ユーザーがアプリケーションWebサイトにとどまっている間、このサイクルが繰り返されます。 このモデルでは、サーバーがすべての作業を行い、クライアントは単純にWebページを表示し、ユーザー入力を受け入れます。







クライアントがより積極的な役割を果たす別のモデルがあります。 このモデルでは、クライアントはサーバーにリクエストを発行し、サーバーはWebページで応答しますが、前のケースとは異なり、ページデータのすべてがHTMLであるとは限らず、通常Javascriptで記述されたコードを含むページのセクションもあります。 クライアントはページを受信するとすぐに、HTMLフラグメントを表示し、コードを実行します。 これからは、サーバーと通信せずに独立して動作できるアクティブなクライアントができました。 厳密なクライアントアプリケーションでは、アプリケーション全体が開始ページのリクエストとともにクライアントにダウンロードされ、その後アプリケーションはクライアント上で完全に実行されます。データを受信または保存し、最初で唯一のWebページのみの外観を動的に変更するためにサーバーに時々アクセスするだけです。 このタイプのアプリケーションは、 シングルページアプリケーションまたはSPAと呼ばれます。







ほとんどのアプリケーションは、2つのモデルのハイブリッドであり、両方のテクノロジーを組み合わせています。 私のマイクロブログアプリケーションは基本的にサーバーアプリケーションですが、今日はクライアント側のアクションをいくつか追加します。 メッセージをリアルタイムで翻訳するために、クライアントブラウザーは非同期リクエストをサーバーに送信します。サーバーはページを更新せずにサーバーに応答します。 その後、クライアントは現在のページに動的に翻訳を挿入します。 このメソッドはAjaxと呼ばれ、非同期JavaScriptおよびXMLの略です（ただし、最近ではXMLがJSONに置き換えられることがよくあります）。







リアルタイム翻訳ワークフロー



Flask-Babelのおかげで、このアプリケーションは外国語を十分にサポートしています。Flask-Babelは 、翻訳者を見つけられる限り多くの言語をサポートします。 しかし、もちろん、1つの要素が欠落しています。 ユーザーは母国語でブログ投稿を作成します。 したがって、ユーザーが知らない言語で書かれた投稿に遭遇する可能性があります。 自動翻訳の品質は常に優れているとは限りませんが、ほとんどの場合、別の言語のテキストが何を意味するのかを基本的に理解していれば十分です。







これは、AJAXサービスを実装する理想的な機会です。 インデックスページまたは探索ページには複数の投稿が表示される場合があり、そのうちのいくつかは外国語である場合があります。 従来のサーバー側の方法を使用して翻訳を実装すると、翻訳リクエストにより元のページが新しいページに置き換えられます。 実際には、表示されている多くのブログ投稿の1つを翻訳するリクエストは、ページ全体を更新するのに十分な大きさではないため、翻訳されたテキストがソーステキストの下に動的に挿入され、ページの残りの部分がそのまま残されている場合、この機能は非常にうまく機能します。







自動リアルタイム翻訳を実現するには、いくつかの手順が必要です。 まず、翻訳するテキストのソース言語を決定する方法が必要です。 また、他の言語で書かれたメッセージに対してのみ「翻訳」リンクを表示するため、各ユーザーの優先言語を知る必要があります。 翻訳へのリンクが提供され、ユーザーがクリックすると、サーバーにAJAXリクエストを送信する必要があり、サーバーはサードパーティの翻訳APIに接続します。 サーバーが翻訳されたテキストを含む応答を送信した後、クライアントのjavascriptコードはこのテキストをページに動的に挿入します。 お気づきのように、いくつかの重要な問題があります。 それらを一つ一つ検討してください。

app / translate.py ：テキスト翻訳機能。 


 import json import requests from flask_babel import _ from app import app def translate(text, source_language, dest_language): if 'MS_TRANSLATOR_KEY' not in app.config or \ not app.config['MS_TRANSLATOR_KEY']: return _('Error: the translation service is not configured.') auth = {'Ocp-Apim-Subscription-Key': app.config['MS_TRANSLATOR_KEY']} r = requests.get('https://api.microsofttranslator.com/v2/Ajax.svc' '/Translate?text={}&from={}&to={}'.format( text, source_language, dest_language), headers=auth) if r.status_code != 200: return _('Error: the translation service failed.') return json.loads(r.content.decode('utf-8-sig'))

サーバーからのAjax



サーバー側の実装から始めます。 ユーザーがメッセージの下に表示される「翻訳」リンクをクリックすると、非同期HTTP要求がサーバーに発行されます。 次のセッションでこれを行う方法を示しますので、ここでは、サーバーによるこの要求の処理の実装に焦点を当てます。







非同期（またはAjax）リクエストは、アプリケーションで作成したルートと表示機能に似ていますが、唯一の違いは、HTMLを返すかリダイレクトする代わりに、 XMLまたはより多くの場合JSONとしてフォーマットされたデータを返すだけです。 以下に、Microsoft Translator APIを呼び出し、翻訳されたテキストをJSON形式で返す翻訳表示機能を示します。

クライアントからのAjax
