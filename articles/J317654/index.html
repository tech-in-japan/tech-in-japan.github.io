EUへの移行に関する別の話





読者の皆様、ご挨拶。 最近、私は頻繁にEUへの（そしてロシアへの1つさえ）成功した​​移住についてのHabréの物語を観察します。 このような傾向がありますので、私の話をしたいと思います。 それどころか、私は感情的な要素をすべて省こうとします-役に立つかもしれない最も有用な情報を提供しようとします。



あなたがカットの下に行く前に、2つの発言：



  1. このストーリーは、システムエンジニアに関するものです。 開発者が助けになるかどうかはわかりません。
  2. あなたがキャリアの物語にうんざりしているなら、あなたの時間を無駄にしないでください、Habréに関するより興味深い投稿があります。


親愛なる読者の皆さんと私はお互いを知らないかもしれませんが、私の経験に興味を持ってくださることを嬉しく思います。



私は私の年齢の詳細、ロシア企業の高等教育とキャリアの詳細を省略します-すべてを一口に噛むのは意味がありません。



要するに：



  1. 2010年以降のIT業界では、ITサポートのインターンとしてスタートしました。
  2. 彼はRedHat 6、Oracle 11.2、およびMS SQL Server 2008R2によって公式に訓練されました。 他のすべてのテクノロジーとシステムは、独立してマスターされます。
  3. 彼は古典的なITILとアジャイル/ DevOps環境で働いていました。


また、新しい仕事を探す理由も省きます。 誰もが今日のロシアの危機が何であるかを完全に理解しており、たとえそれが私の妻のためでなくても、私はそれを危険にさらすことさえしません。



今、私は仕事を探すために使用したものに。



見どころ



monster.com 、 Indeed.com 、 linkedin.com 、 glassdoor.com 、さらにはhh.ruは、海外で最も人気のある求人サイトです。



私が最初にしたことは、私の履歴書を英語に翻訳して、Monsterに載せることでした。 奇妙なことに、インタビューの招待状は翌日雨が降りました。 手紙を受け取った時（全員がモスクワに午後7時から8時までに到着した）を見て、私はより活発なリクルーターが主に西半球から来ていることに気付きました。



短い読みと迅速な対応の後、次のことが明らかになりました。



  1. ほとんどの採用担当者は、多数の応募者に同時に手紙を送ります。
  2. これは、要件が明らかにあなたの能力よりも高いポジションの面接への招待を受ける可能性が高いという事実を説明しています。
  3. 招待状のほとんどは、一時的な契約ポジション（6か月、1年）に対するものでした
  4. 企業は、ビザの移動と取得に関して支援を提供しません。
  5. 専門分野と文化の違いを理解しないという標準的な問題（正直なところ、「DevOpsエンジニア」という名前から私を混乱させます）は、西側の雇用主にとって決して異質なものではありません。


DevOpsまたはシステムエンジニアの地位に関するインタビューへの招待状をいくつか受け取ったが、要件は10年の実務経験を示していた（当時は5年だった）が、私は長い間困惑し、その後、欧米の企業を雇った経験のある友人と相談した。 雇用主は最初に要件の可能な限り最高の基準を設定し、正しい候補者を見つけることに成功しなかった場合、経験の階層を徐々に下っていくことがわかります。



しかし、私は簡単な理由で面接に同意しませんでした-潜在的な雇用者の誰もビザ、フライト、リフトの代金を支払う気がありませんでした。 切り下げを考慮に入れて、莫大なお金を節約するために、問題が燃え尽きるという事実ではない疑わしい旅行に行くために...私にとって不当なリスク。 しかし、リスクがあり、最も重要なことには、非常に経験豊富な専門家に公式の証明書を試してみることをお勧めします。 上級レベルの給与は非常に高いです。



検索の踏み台としての私にとって理想的なのは、 stackoverflow.com / jobsです。 一部の雇用主は、ビザのサポートと移転を提供しています（そして、彼らは非常に簡単にフィルタリングできます）。 履歴書をそこに投稿した後、彼は英国、フランス、オランダに事務所があるほぼすべての事務所にフィードバックを送り始めました。 彼らのほとんどは、この技術またはその技術の経験の不足を理由に拒否しました、ある事務所は私をインタビューに招待しましたが、給料はロンドンに住むには低すぎました（私は2週間出張していて、生活費について一定の洞察を持っているので）。 最後に、オランダの番号から電話がかかってきました。



インタビュー



リクルーターと



インタビュアーは、私の回答を読んでおり、私の経験についてもっと知りたいと言った。 教育に関する一般的な質問（私は誰のために、どの程度、どのような卒業証書および平均点）、仕事の経験（どのような会社、私は何をしたか、どのプロジェクト）、私は以前の仕事を辞めたのか、なぜ今仕事を探しているのか、どのくらいお金が欲しいのか、など すべての回答を受け取ったリクルーターは、すべてのメモを採用マネージャーに転送し、すぐにフィードバックを提供し、場合によっては採用マネージャーとのインタビューをスケジュールすると述べました。 びっくりしました（技術についてはどうですか？）、しかし翌日、リクルーターから手紙が届き、リクルーターは会社の2人のエンジニアとの最初の技術面接の日付を提供しました。 彼は、これは暴力ではなく、好奇心（特定の技術と彼が使用したもの）だけが入るテクノインタビューの序論であると述べました。 大丈夫。 glassdoorでレビューと給料を勉強した後、Ansibleの知識を更新し、次の会議の準備をしました。



最初の技術



インタビューはSkypeで行われ、2人のエンジニアが私の向かいに座って自己紹介をし、プロジェクトについて尋ね始めました。 インタビューは完全に単純で、次の形式で構成されていました。エンジニアは履歴書を見て、次のように述べています。 あなたは彼らに何をしましたか？ どのOracle機能を使用しましたか？」 「FreeBSDでKDE2にパッチを適用する方法」や「ユーザーがブラウザを開いてfoo.barを検索に入れるとどうなるか」というスタイルの質問はありません。 もう悪くない。 みんなにどんな質問をされたのか尋ねられた後。 私は彼らが働くテクノロジー、チーム間で働く主な問題は何か、スクラムチームの数、チームにいる人の数を尋ねました。 これで、彼らは休暇を取った、彼らは皆、喜んで、そして今のところそれを好む。 後に、採用担当者がSkypeをノックし、最初の段階が完了し、メールでさらなる手順が期待されると述べました。



2番目の技術。



1週間後、リクルーターから2番目の手紙が届きました。 私は過剰なスキルを持ち（システムエンジニアに対応しました）、Linuxのリードエンジニアの役職に就くよう提案されています。 私は同意し、2回目の技術面接で合意しました。 難しくなると約束されたので、徹底的に準備するのに4日間かかりました。 最初の数日間、私はすぐにコースのメモを調べ、LinuxAcademyで練習し、ほとんどの質問が技術面接で尋ねられたものを探しました。 ITの巨人からのLinuxインタビューに関するTOP 30の質問を見つけました。 （Amazon、Google、Microsoft、RedHatなど）お母さん。



私は嘘をつきません。X日が近づいていたので、私はこのインタビューに非常に神経質でした。私は間違いを犯す権利がありませんでした。



指定された日に、私は再びSkypeコールを受け取りました。他の2人のエンジニアがすでに私の前に座っていました（私が割り当てられることになっていたスクラムチームから）。



彼らはすぐに私の経験を調べ、そのうちの1人はITのどの部分が私にとって特に難しいかを尋ねました。 私は正直に答えました：ネットワーク（はい、はい、私はL3とL4を混同し、TCPハンドシェイクシーケンスを思い出せない人です）とプログラミング（すべての言語からBashとSQLだけが与えられました）。 最初の質問は、現在のRADIUSサーバーをオープンソースに変更する必要がある場合、この問題をどのように解決するかです。 素晴らしい。



私が最初にグーグルをすることを言った。 コミュニティの人気、サポート、規模、活動、サポートされている腺のリストに基づいてソリューションを選択します。 次に、スイッチ、アクセスポイントなどを備えたラボを作成します。 そしてテスト。



その後、疑問が生じ、RADIUS自体の側で何を行う必要があります。 Kerberosを設定する必要があると言いました。 おそらく。



残りの質問は非常に簡単でした。useraddを使用せずにユーザーを作成する方法、PID 1のプロセス、および問題chmod -x / usr / bin / chmodを解決する方法



将来の同僚に質問がなかったとき、最初のインタビューと同じ形式でさらなる会話が行われました。



オフィス訪問、最終面接



1週間後、ヘッドハンターは私が技術的な段階を終えたと厳declaredに宣言し、もし彼らと一緒に仕事をすることに興味があれば、最終段階のために会社のオフィスに招待されました。 会社は、空港から市内までのフライト、ホテル、チケットの代金を支払いました。 私はビザを払うだけでした。 私はオフィスの見学を与えられ、将来のチームのリーダーであるプロダクトオーナーとスクラムマスターとのアポイントメントが行われました。



大きな驚きは次のとおりです。



  1. スクラムマスターは、私が話をしたエンジニアの1人でした。
  2. 採用マネージャーは、彼の決定は大きな役割を果たさず、その主で唯一の仕事は、仕事の面ですべてが私に合っていることを保証し、年次業績評価を実施することだと述べました。
  3. RADIUS氏、私の答えの正しさについて彼に尋ねたとき、一般に私の答えは拒否されました。 私は、技術面接を受けなかったという私の言葉に、彼は、人がストレスの多い状況でどのように振る舞うかが重要だと答えました。ギザギザの本。
  4. 最も重要な選択基準の1つは、いわゆるチームフィットです。特定のチームにどのようにフィットするかを経験的に測定します。 つまり、チームと友達になりそうもないプロと中流階級のコンパニオン平均との間の選択では、チームは2番目のオプションを選択します。


その日の終わりに私は申し出を申し出られ、その日に署名しました。



結論



親愛なる読者 ある日トラクターに座って西ヨーロッパの国々に行くことにした場合、ここに私のヒントがあります：



  1. stackoverflowとHHの空席を参照してください-再配置でポジションを見つける機会が多くあります。
  2. 再配置および/またはビザのサポートでポジションを無視しないでください。 これは、雇用主が外国人を雇うことを決意していることを最初に示唆しています
  3. 番号を呼び出す前に、生活費（家賃、税金、食料、交通費、娯楽など）を慎重に検討してください。
  4. あなたが本当にあなたよりも賢く見えるようにしようとしないでください。 私の場合、知識よりも思考の方が重要でした。
  5. これは、知識が増えればお金が増えるという事実を否定するものではありません。
  6. 契約を注意深くお読みください。 会社があなたの移転にお金を払うなら、あなたに長引くように頼む可能性が高いです。
  7. 英語を学ぶ。
