FacebookがSPDYプロトコルを開始

Facebookは、HTTP / 2.0標準およびSPDYプロトコルに関する意見の収集に関して、将来の標準の潜在的な基盤として公式レビューを公​​開しました。 このような調査は 、IETF HTTPbisワーキンググループによってすべての関係者の間で現在実施されています。



最大規模のサイトの代表者は、HTTP / 2.0の開発に参加したいという強い要望を表明し、「SPDY / v2の実装を開始しました。これは、ブラウザによってサポートされ、実装後に直接利益をもたらすことを約束するためです」と述べました。 現時点では、実稼働環境の実サーバーでSPDYがまだ使用され始めていませんが、「システムはほぼ完成」しており、その大規模な実装はすぐに計画されています。



FacebookはHTTP / 2.0への要望も表明しており、Facebookの速度を上げてセキュリティを向上させるために、リクエストの多重化、トランスポートレイヤー暗号化（TLS）、HTTP / 1.1アップグレード（TLS）ヘッダーを使用しないHTTP / 2.0へのアップグレードをご覧くださいNPN）、要求レベルでフローを制御し、サーバー側をプッシュして長いポーリングを置き換え、あるレベルで「バッファリングなし」フラグをオンにします。



Facebookは、HTTP / 2.0の基礎として現在検討されている3つの提案のそれぞれについて意見を表明しています。これらはSPDY、 HTTP Speed + Mobility （Microsoftから）、およびNetwork-Friendly HTTP Upgradeの進化版です。 前述のように、同社は最初のものを支持して選択を行い、すでに実装を開始しており、HTTP / 2.0標準の開発の基礎としてSPDYを推奨しています。 さらに、これはすべての候補者の中で最も一般的なプロトコルであり、 Firefox 13とChromeブラウザーはすでにサポートしています。 FacebookがSPDYで望んでいるのは、gzipではなく、より軽量なヘッダー圧縮ソリューションです。



したがって、SPDYの導入後、このプロトコルはインターネット上の2つの最大のサイトであるGoogleとFacebookで動作し、Webページの読み込みを高速化します。 以前、TwitterはSPDYのサポートも発表しました。
