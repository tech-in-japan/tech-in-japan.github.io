非FlexPod DC：直接接続ストレージ、ワンストップサポート

私の記事の1つで、FlexPod DCアーキテクチャの概要と構成について説明しました。FlexPodDCの物理コンポーネントには、NetApp FASシリーズストレージシステム、Cisco UCSサーバー、Nexusスイッチが含まれます。 これらの3つの主要コンポーネントで構成される、さまざまなサポートされているFlexPod DCデザインがあります。 1つのソースからの協力的なサポートを利用するには、これらすべてのコンポーネントに対して適切なサポートサービスが必要です。



Cisco SnartNetおよびNetApp Support Edgeサービスがあり、アーキテクチャにNexusスイッチがなく、ストレージがUCSファブリックインターコネクトに直接含まれている場合はどうなりますか？



これは「非FlexPod DC」アーキテクチャについて説明しますが、Cisco Infrastructure Support for Critical Infrastructure（SSCI）プログラムの下で1つのソースからサポートすることもできます。





一般的な直接接続SANの設計



FIデバイスでFCプロトコルを使用するには、FC スイッチングモードとFCゾーニング機能を有効にし、 ストレージに接続されているポートをストレージポートモードに切り替え、 ストレージクラウドセクションでVSAN （VMware vSANと混同しないように）を作成する必要があります 。



FCストレージポートとして構成






FCoEストレージポートとして構成する






FIの場合 、同じポートで、「通常の」イーサネットトラフィック（ CIFS 、 iSCSI 、 NFSなど ）とFCoEの両方を同時に受信できます 。 ストレージにCNAまたはUTA2ポート（すべてのFAS 255xおよびFAS 8000シリーズが搭載されている）があり、 FIなど、1つのリンクでイーサネットとFCoEを実行する機能をサポートしている場合（すべてのNetApp FASとCNA / UTA2サポート）、ポートを切り替えることができますこのようなデバイスは、スイッチなしで直接。



統合ポートを構成する
  • まず、Cisco UCSファームウェア2.1以上のファームウェアバージョンが必要です。
  • 次に、10GB CNAポートを備えたストレージが必要です。


NetAppストレージ側から、ストレージでucadminコマンドを使用して、ポートをCNA状態にする必要があります（ CNAポートは必要ありません、通常のイーサネット1 / 10Gbsポートはこれをサポートしません）（ ストレージを再起動する必要があります ）。 システムは、個別に「仮想」イーサネットポートと「仮想」 FCポートを個別に表示します（ただし、物理ポートは、そのような「仮想」イーサネットと1つの「仮想」 FCに使用されます）。 通常の物理ポートと同様に、ポートは個別に構成されます。



FIでは 、「Equipment」タブのFabric A / B設定で、「Switching mode」状態のFCモードを有効にする必要があります。 この設定では、 FIを再起動する必要があります。



[Equipment]タブでFIを再起動した後、数秒後にポートがオンラインになり、コンバージドポートを[Appliance port]モードに切り替える必要があります。 次に、ポートをFCoEストレージポートモードに再構成します。右側のペインに、ポートのタイプが「 Unified Storage 」と表示されます。 これで、このようなポートにVSANとVLANを選択できるようになります 。 また、 VSANによって以前に作成された重要なポイントでは、ゾーニングを実行するためにFIで 「FCゾーニング」を有効にする必要があります 。



直接接続を使用した構成の逆の側面は、スケーラビリティに乏しく、すべてのタイプのネットワークトラフィックのネットワークリンクのフォールトトレランスと負荷分散を構成できないことです。

イーサネットトラフィックの場合、 直接接続とiSCSIプロトコル、直接接続とFCoEプロトコルの両方、 および直接接続とFCプロトコルの両方で、ネットワークリンク経由の負荷分散とフォールトトレランスは、これらのプロトコルに組み込まれたマルチパスを使用して実現されます。



FlexPod DataCenterの設計では、 Nexusシリーズスイッチは依然として必須のアーキテクチャコンポーネントです。

  • まず、FlexPodは何らかの形で既存のインフラストラクチャに統合され、顧客へのアクセスを提供する必要があるため
  • 次に、Nexusスイッチを介した組み込みにより、将来のアーキテクチャの拡張が可能になります
  • 第三に、 SAN （ FC / FCoE / iSCSI ）との直接接続とは対照的に、 NASのイーサネットネットワークの設計では、ネットワークリンクの負荷分散を実行するためにストレージシステムとUCSドメイン間の中間リンク、および主にフォールトトレランス機能が必要です。
  • 第4に、FlexPodへの外部イーサネットクライアントアクセスはフォールトトレラントでなければなりません
