2019年の予測はありません

先月からHabréや他のサイトで、2019年に私たちを待っているものについての予測が現れ始めました：誰かが彼らの新機能について書き、誰かが市場動向について書きます、多くの結果と論争があるようです次の2019年にどのテクノロジーが遅れるかについて。 もちろん、すべての夢想家や先見の明はAI、VR、BigDataのブレークスルーを待っており、現実主義者はボストンダイナミクスの新しいロボットと規制当局からの驚きを待っています。 ロボットも待っています。彼らはとてもクールです。 そして、私たちは、年々行き、トレンドから消えることのないもの（一部、悲しいかな、消える）にそれらのものと原則に今日の投稿を捧げたいと思います。 したがって、予測不能です。









出所 特にWindowsがここにいるとき、私たちは、すべてのまともなITオフィスと同様に、私たち自身のシャーマン（オプションでチュクチ）を持っているだけです。



当社のRegionSoft Developer Studioは、ビジネス向けソフトウェアの開発と実装に従事しています。 私たちのお気に入り、誇り、旗艦は、 RegionSoft CRM 、 RegionSoft CRM Media 、 RegionSoft Application Server 、そして...です。実際、私たちはみんな大好きで選択できません。 結論として、私たちのチームには、ソースに基づいてコンテンツを生成する準備ができている美しい執筆哲学者、翻訳者がいません。また、MBAのマーケティング担当者もいません。 私たちはほとんどすべてエンジニアであり、他の技術者でもあります。 したがって、傾向を見るのは純粋に主観的なものです。



さあ、トレンドから外れましょう。



人









プロフェッショナリズムは、生産性と結果、優れた製品、収益などを引き出す企業の仕事における最高の価値です。 あなたが何をするかは問題ではありません。ジュニアポジションの開発、テスト、コーディングは、特定の期間ごとに専門家でなければなりません。 キャリアの成長の終わりには専門家はいません-彼らの代わりに専門家がいます：プロはチームリーダーだけでなく、手動テストを実行する研修生でもあります（彼はこの分野を知っている必要があります）。 この流れの中でのみ、プロフェッショナリズムが成長し、利益を得ています。 あなたが座って座って、そして-バム-そしてスーパープロになったということは起こりません。 腐食性のみ（ただし、同僚の脳には影響しません！）、細部への注意と絶え間ない発展は、安定した成長につながります。



社内の関係。 今日、私たちは1世紀の特別なコミュニケーションを開発しました。ほとんどの場合、オンラインでコミュニケーションを行い、しばしば行動規則を無視してから、実際のコミュニケーションゾーンに入り、テンプレートの一部を外部にブロードキャストします。 はい、この調子も好きではありません。特に、調性と関係のベクトルは、世界の古い要因、疲労、個人的な問題、健康などによって影響を受けることを考えると 現実には、慎重に関係を構築する必要があります：残念ながら、あなたが非常に専門的であっても、誰かが不快であると思われる場合（有毒という言葉を避け、嫌です）、あなたは簡単に仕事を失うことができます。 したがって、あなたが下品なものと呼びたい同僚に対してさえ、抑制された敬意のある行動のラインを構築するようにしてください。 最後に、誰もが自分のことをします。



スタッフ募集アプローチ。 一部の地域では、雇用主市場が確立されており、企業は一部の地域-雇用市場で労働者を選択でき、夢の仕事を選択します。 ただし、すべての場合において、古いパターンが機能します：欠員を配置し、履歴書を収集し、一度に1つまたは複数を呼び出し、無意味なテストを整理します：心理テスト、HRマネージャーとのインタビュー、第2段階は職業、そして英語での会話HRでは、外国語でのみ英語を使用し、これらの非常にインタビューで、主な仕事は上司を通り抜けることです。 これはよく知られていますが、非常に主観的な理由で専門家を排除することができます。たとえば、内向的な人事が嫌いなためです。



何かを変える価値があると思われます。例えば、プロのスキルへのインタビューにもっと注意を払い、人事スペシャリストがメンタリングを適応させ、導入することです。 そして最も重要なことは、上級のNASAエンジニアが必要だと思われる職務記述書を書くのではなく、実際には手動テストの専門家です。 もう一つのポイント：あなたが空室のテキストに書いたことを忘れないでください。 そして、「7万ルーブルからの賃金」は、「まあ、個人所得税前の45,000で、少なくとも97％のKPIを持つ」とは異なります。 そして、トレーニングの場合、ホッキョクフクロウが満月のオフィスを飛ぶ場合、70,000は試用期間後です。



トレーニングと教育。 私たちはHabréで2年目にこのトピックに取り組んでおり、同じ傾向が見られます。「ITでは教育は必要ありません」。 いいえ、友人、それは必要です-少なくとも、最高として-最高のプロファイル。 教育は、情報源、批判的思考、適切なコミュニケーション、専門分野における非常に多くのことの基本的な理解を扱うスキルを提供します。 たとえば、1か月のコースの後、単純なサイトを弱体化できますが、アーキテクチャとアルゴリズムの教育と理解を備えた専門家だけがそれを安全にすることができます（もちろん、例外はどこにでもあります）。



誰にも追いつく必要がないという理由だけで、教育がキャリアの成長速度を決定することは明らかです。 さらに、開発者の教育は継続的である必要があります。本や情報源を読み、新しい技術の仕様を理解してください。 これが頂上にとどまる唯一の方法です。

技術









開発スタックを使用します。 Nizhny Novgorod GDG-2018で、サーバー側のSwiftに関する素晴らしいレポートがありました。 それは本当に刺激的で、従業員は会話の参加者になりました。ある開発者は、明日は実稼働でこれが必要であり、このアイデアを確実に販売するだろうと熱心に議論しました。 同時に、みんなはJavaですべてを持っており、それはかなりうまくいきます。 「これはSwiftであり、レポートは火事だ」以外は、彼は厳しい選択の理由を動機づけることができなかった。 実際、新しいテクノロジーは毎月登場しており、年に数回、新しい言語やフレームワークが人気を集めています。 これらすべてをテクノロジースタックに取り込むことは、継続的にマスタリングと再構築を行うことを意味します。 新しいテクノロジーが提供するものを自問してみてください。開発を迅速化し、製品を改善し、製品開発のコストを削減し、最高の市場スペシャリストを選ぶことができますか？



利点をテスト、分析、理解してから、新しいトラックに慎重に進むことが最善です。 テクノロジーをオンザフライで変更することは、たとえ誰かの前で前向きな経験があったとしても、危険な話です。 そして、はい、古い曲がった手の新しい技術は魔法のタブレットにならず、松葉杖やバグを書くための新しいスタックになります。 プロジェクトを改善したいですか？ リファクタリング、最適化、テストという3つの言葉があります。



顧客への配達。 ITの世界は、最も便利なソフトウェア展開ツールによって征服されています。コンテナ、DevOps、DevSecOpsは、会議で大きなテーマになりなくなり、一般的になりました。 それにもかかわらず、恐ろしくて時には愚かなバグは、彼らとの関係を台無しにする非常に大きな会社の顧客からも戻ってきます（例えば、ある会社はソフトウェアを数百万で販売しましたが、ウィンドウを閉じるための右上隅に十字がありませんでした） -テスターはこれを逃し、ブロッキングチケットがクライアントから送信されました。もしそうなら、それは私たちではありません:-)）。 クライアント側の問題の原因は多くあります。不公平なテスト、曲がったアセンブリ、貧弱な技術サポート、顧客インフラストラクチャとベンダーのソフトウェア要件の不一致です。



クライアントで勉強したり、テストバージョンで勉強したり、エンジニアを訓練したり、テスト計画を慎重に行ったりしないでください。 さもなければ、明日、顧客は競合他社に行き、これはあなたの給料です（もちろん、売り手が責任を負うことになります）。



ソフトウェア配信モデル。 クラウドブームは衰退し、市場の誰もがクラウドソフトウェアとデスクトップソフトウェアの両方が存在する権利を持っていることに気付きました。 したがって、クラウド内のクリップアートサーバーを使用した美しいプレゼンテーションではなく、クライアントのインフラストラクチャを操作することに注意を払う必要があります。特に、作成するタスクがある場合は注意が必要です。 とにかく、いずれにせよ：言葉を減らして、仕事を増やしましょう。



カスタマーサポートは専門家でなければなりません。 はい、多くの人々は、言語の知識に基づいて（全世界に販売する場合）、または「学生は安い」という原則に基づいて支援を得ることで罪を犯します。 一般に、これには何も問題はありませんが、技術サポートはトレーニングを受け、監視され、定期的に認証を確認する必要があります。そうでなければ、結果は悲惨なものになります。 何よりも、サポートサービスを開発し、製品、問題、リクエスト、顧客の要件などに関する情報を絶えず更新する人/部署（規模に応じて）を特定できる場合 これは、新鮮な情報なしで6か月後にカボチャに変わるサービスです。

事業内容









常に最初のオプションが優れているわけではありませんが、常に落ち着いています:-)



人はより重要であり、プロセスはより高速です。 悲しいかな、ビジネスは人々を大切にすることを学んでいない-それは非難と非難に焦点を当てており、主な資産のランクに人を上げるよりも。 これは一方でです。 一方、中小企業はビジネスプロセスの自動化に向けて非常にゆっくりと動いており、ほとんど方向性を見ていません。 ここに矛盾はありません。プロセスを自動化するつもりはありませんが、自動化の外で最も価値のある人々に感謝するつもりはありませんか？ したがって、ソリューションはシンプルで信頼性の高いものでなければなりません。ルーチン全体をアルゴリズムに入れて自動化し、複雑でコミュニケーションの問題を解決するために人々をリダイレクトします。 ところで、絶対にどの会社にも自動化を必要とするプロセスがあります。



自動化はプロセスだけでなく、横断的でなければなりません：生産、開発、顧客サービス、経理、マーケティング、すべての部門はタスクを簡素化し、データを収集するためにソフトウェアを必要とします。 なし。 関連部門には、CRMシステム、ERP、ECMなどの共通ソフトウェアノードがあることが望ましいです。 インフラストラクチャ内でこの問題を解決するには、統合ソリューションおよび/または統合を使用できます。



KPI-あなたはそれらを憎むことができ、あなたはそれらを使用することができ、あなたは原則的にそれらを拒否することができますが、それらは必要です。 まず第一に、労働者の仕事と製品の作成と実装に対する全員の貢献を評価するため。 ただし、測定できないものを測定しようとしないでください。たとえば、プログラマーやデザイナーにKPIシステムを提供する前に、3回考えることをお勧めします。 しかし、商業サービスでは、KPIは動機付けの手段および生産性の重要な指標として必須です。



アイデアはどこにでもあります：新しいソフトウェアを開発する必要があります-アイデアを探しています。Habrに投稿を書く時が来ました-アイデア、広告、ソフトウェア内の新機能、開発プロセスの編成、技術サポートがすべてアイデアです。 しかし、それらは危険なものです。あなたは独創的な何かを発明したように思われ、そのようなアプリケーションはAppStoreとGoogle Playでいっぱいです。 したがって、各アイデアについて、式に従います：最初にアイデアとその属性（理由、ユーザー、開発方法、おおよその結果）を定式化し、次に実験（製品または製品のリリース、サービスの提供）を行い、最後に適応します（顧客およびユーザーからのフィードバックに基づいて変更します） 。 そうすれば、アイデアは緑になります。
