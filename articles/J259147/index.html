ネットワークの進化-新しい25/50/100 GbE標準

現代のイーサネットネットワークは長い間私たちの生活の不可欠な部分になり、それらのないビジネスプロセスでさえ完全に考えられません。 そして、そのスループットは、その成長が非常に求められている特性です。 1 Gbit / sの速度は長い間一般的なものとなり、プロバイダーでさえ個人に提供しています。 しかし、問題が深刻な情報の流れの転送に関係するとすぐに、これは明らかに十分ではなく、10G規格は現在、SFP + Base-Tコネクタではなくユニバーサル規格であり、最大7メートルの範囲で銅パッチコードとして使用できると言えます（ラックまたはラックのグループ）、および光通信回線。 さて、負荷の高いデータ回線（集約レベル、高性能コンピューティング用のクラスター相互接続、強力なストレージシステムの接続）には、4つの10G SFP +回線の組み合わせである40G QSFP +が使用されます。



ただし、サーバーのパフォーマンスは向上し、データストリームも増加し、ネットワークの負荷も高くなっています。つまり、高速への切り替えのニーズは長い間高まっています。 長い間、CFP、CFP2、CFP4形式の100G製品がありましたが、それらはサーバーインフラストラクチャで使用するには不十分であり、手頃な価格の製品を作成する必要がありました。 これを行うために、25Gコンソーシアムが作成されました。これは、イーサネットアライアンスとともに、一般的なネットワークの速度を2.5倍に高めるように設計された新しい業界標準の開発に携わりました。 標準の開発が終了すると、大衆に製品を宣伝するために作成された2550100 Allianceが登場します。 彼らの仕事の結果と、市場への大規模な侵入に備えて標準がどの程度準備されているかについてお話します。







それで、新しい標準が私たちにやって来ます。 その中の基本速度は25 GB / sであり、2行の組み合わせは50 GB / s、4-100 GB / sを提供します。



そもそも、新しい標準は革新的な開発というよりも進化的なものです。 ネットワークでの作業の観点から、すべての作業スキーム、プロトコル、テクノロジー、および推奨事項は同じままで、物理レベルのみが変更されます。 このためのすべての基本は長い間定められています。標準はIEEE Std 802.3ba-2010に記載されており、2011年以降、それに対応する機器はトランクルーター、対応するカード、およびトランシーバーの形で製造されています。 4年前は技術のピークであり、非常に多額の費用がかかりましたが、現在、低消費電力のマルチポートラックマウントスイッチ、ネットワークアダプター、および関連するケーブルの形で、データセンターに一般的に使用される機器を作成することについて話しています コネクタに関しては、SFP +をSFP28に置き換え、それに応じてQSFP +をQSFP28に置き換える必要があります。



SFP28およびQSFP28コネクタは、バックボーン機器で使用されるCFPソリューションと比較して、通常のSFP +およびQSFP +とそれほど変わらないのは興味深いことです。 状況はエネルギー消費と同じです。新世代のレーザーは100Gポートあたり3.5 W未満のエネルギー放出を提供するため、大量使用に適したコンパクトなソリューションを作成できます。



後方互換性の観点から、すべてが正常です-現在表示されているすべての機器が10 / 40G速度での動作をサポートしているため、新しい標準への移行を徐々に行うことができ、新しい25 / 100Gスイッチを古い10 / 40G機器に追加して、スムーズに新しい速度に引き上げます必要なネットワークセクションの標準。



大量生産と実装のための新しい標準の準備については、ここで、私たちの意見では、すべてがかなり良いです。 すでに、鉄、テキソライト、シリコンで実装されている必要な機器のフルレンジがあります。例えば：



-スイッチ







-ネットワークカード







トランシーバー







-ケーブル







-ライン試験装置







したがって、すべてが順調であり、コンポーネントメーカーからの標準に対する広範なサポートがあります。Qlogic、Avago、Cavium、Finisar、Broadcom、Ixia、Mellanoxなどの大手企業はすでに製品を発表しています。 また、新しい標準をサポートした企業の一般的なリストは非常に大きいため、ここにリスト全体をリストするよりもリンクを提供する方が簡単です。 確かに、このリストに既に参加している場合（製品について少し後で）皮肉にも気付くことはできません。Intelほど重要でない会社はそうではありません。



冗談は別として、これはやや信じがたいことですが、IntelはX710ソリューションの微調整に重点を置いて25 / 100Gの外観を完全に見逃したようです。 競合他社は明らかに、ネットワークアダプタを大量にリリースすることで市場の一部を食いつぶすことはありません。



大手ネットワーク機器メーカーが新しい標準への切り替えを開始する速さや、競合他社を背景にメラニンシリコンに関する決定を強調する議論を検討することも非常に興味深いです。近い将来、すべてが同じBroadcom Tomahawkマトリックスを中心に構築されるからです。 。



新しい標準の人気に一生懸命取り組むことを約束するもう1つの重要な事実は、そのコストが現在の10 / 40Gソリューションと根本的に変わらないことです。 通常、革新（またはそれに慣れている人にとっては革新）は、不均衡に高い価格から長い間恐れています。その結果、最も生産的な技術に切り替えなければ生きられない人だけが導入の先駆者になります。 今回は、大規模な意思決定に重点を置いて計画されているため、「新規性に対する追加料金」を考慮しても（および完全に取り除くことは不可能です）、価格性能比の観点からの新しいソリューションは、最初に前任者を上回ることを約束します。



このような高帯域幅のネットワークインフラストラクチャが必要なのは誰ですか？ さて、明らかな実装例を見てみましょう。



Web-Scaleインストールで使用



サーバー（40U、1 / 2Uの80ノード）で完全に詰まったラックの形での典型的な大規模インストールを検討する場合、各ノードの2ポート10Gアダプターは同様の25Gアダプターに置き換えられます。 したがって、TORレベルでは、4個の10Gスイッチ、たとえばEos420（48個の10Gポート、6個の40Gポート）を2個の100Gスイッチに置き換えることができ、32個の100Gポート20のうち、ブレークアウトケーブルを使用して、25Gノードへの接続を提供し、残りは12アップリンクで動作します。



このようなソリューションの利点：
  • 150％高いスループット
  • ラック内のネットワークインフラストラクチャ（スイッチ、ケーブル）が2倍少ない-> CAPEX未満
  • TORスイッチが2倍少ない-> OPEXより低い
  • すべての既存の仮想化およびオーバーレイネットワークテクノロジーがサポートされています（VXLAN、NVGRE、SPBなど）


高性能コンピューティングで使用



現在、HPCは最も一般的に56G InfiniBandを使用しています。 したがって、「額」を、32台のサーバーのブロック内で100G接続に置き換えることができます。



利点：
  • 78％高いスループット
  • CAPEX以下
  • 簡単なシステムスケーリング
  • すべてのタイプのトラフィック用の統合ネットワーク


ストレージの接続に使用



現在、イーサネットを介したストレージへのブロックまたはファイルアクセスは、ファイバーチャネルよりも実用的であり、パフォーマンスはInfiniBandに近づいています。 新しいインターフェースは、この状況をさらに悪化させ、他のすべてのインターフェースを実質的に不要にします。



利点：
  • サーバーとストレージの統合ネットワークインフラストラクチャ-> CAPEXおよびOPEX未満
  • 大容量のオールフラッシュアレイとストレージに必要な最高のデータ転送速度
  • RDMAによる低遅延とストレージプロトコルのサポート
  • 簡単なスケーリング
  • 便利な管理
  • 単一のインターフェイスを維持しながら、データストレージの方法（ファイル、ブロック、またはオブジェクト）を選択する機能


最後に、2550100への参加について。







パートでは、32個のQSFP28ポートを備えたEos 720スイッチを導入しました。 このモデルは、既存のEosスイッチファミリの継続であり、お気に入りのBareMetal Switchイデオロギーの一部として作成されました。つまり、さまざまなネットワークオペレーティングシステムのインストールをサポートするONIEインストール環境を備えています。 このスイッチはすでに完全に準備ができており、近い将来に研究所で利用できるようになりますが、OSサポートについては、互換性の問題についてメーカーと緊密に協力しています。











スイッチの主な特徴は次のとおりです。
  • ブレークアウトケーブル付き32ポート100G QSFP28 / 128 25GbE SFP28
  • 3.2Tbps Broadcom Tomahawk BCM56960スイッチマトリックス、64バイト転送レート：2400 Mpps
  • Intel Atom 2538プロセッサー
  • MACアドレステーブル-136Kユニファイドフォワーディングテーブル（UFT）
  • 500 ns未満の遅延（PHYなし）
  • VXLAN / NVGREハードウェア処理
  • データセンターの最適化：802.1Qau、802.1Qaz、802.1Qbb、DCBX、EVB（802.1Qbg）、MLAG、32ウェイECMP
  • ONIEを使用したBMSプラットフォーム：Cumulus Linux（更新後）、Broadcom ICOS（11月）サポート
  • SDNスイッチングのサポート：OpenFlow 1.0、1.2、1.3、Open API
  • Broadview Toolkitによる完全なネットワーク制御のサポート
  • 1 + 1スキームに従ったフェイルセーフ電源


要約すると、新しい標準はすでに市場に参入する準備が整っており、1年以内に商用製品が大量に登場することを期待できます。 インフラストラクチャを今すぐ計画する価値があります。
