最新のデータセンターを保護する方法

ソフトウェア定義（ソフトウェア定義）ソリューションの配布の傾向は、長い間企業のデータセンターに届いています。 今日の「正しい」データセンターは、ソフトウェアで構成可能なデータセンター、または言わばソフトウェア定義（SDDC）です。 分析機関によると、75％を超える企業がすでにソフトウェア定義のデータセンターで作業している一方で、それらの仮想化のレベルは成長を続けています。







データセンターには追加の要件があります。 最も重要なものの1つは、高度なインフラストラクチャセキュリティです。 信頼性の高いデータセンター保護は、データセンターの所有者だけでなく大企業にとっても重要な要素です。





データセンターの変更には、セキュリティポリシーのレビューが必要です。 ISは、最新のITインフラストラクチャと同様に柔軟で動的になり、ITシステムのパフォーマンスに悪影響を与えずに、その変化に適応できる必要があります。



Kaspersky Labの専門家によると、従来の保護は最新のデータセンターにはありません;既存のセキュリティ概念はもはや適用できず、レビューする必要があります。 相互作用の新しいシナリオが登場しました;仮想環境は、インフラストラクチャのコンポーネント間の新しい接続を確立します。





すべての新しいトレンドに伴い、企業のデータセンターインフラストラクチャを構築する原則は、まだかなり伝統的です。 ストレージ、データ処理、ネットワークインフラストラクチャが含まれます。



何をどのように保護しますか（そしてどのように保護しないか）？



最新のデータセンターは以下を組み合わせます：



  • 仮想化プラットフォーム（VMware vSphere、Microsoft Hyper-V、Citrix XenServer、またはKVM）を備えたデータセンターインフラストラクチャ。

  • ストレージインフラストラクチャ-ファイルサーバー、さまざまな種類のストレージシステム。

  • データセンターのコンポーネントと仮想ネットワーク（たとえば、VMware NSXに基づく）の相互作用を可能にするネットワークインフラストラクチャ。



これらのすべてのコンポーネントは、データセンターの運用に不可欠であり、特定のテクノロジーの詳細を考慮して保護する必要があります。 たとえば、仮想環境では、その仕様に適合しない従来のセキュリティソリューションは「リソース集約型」であり、重要なビジネスアプリケーションを抑制し、ユーザーに不便をもたらします。









カスペルスキーによる25か国の企業の調査によると、回答者の44％が仮想環境を保護するために従来のソリューションを使用しています。



仮想マシンは冗長タスクを実行します。 VMは、ウイルス対策データベースをローカルに保存および更新し、悪意のあるコードをスキャンし、ネットワーク攻撃から自身を保護します。 これにより、ホストシステムに大幅な追加負荷が発生し、ITインフラストラクチャとそのユーザーの効率が低下します。これは、「更新ストーム」および「チェックストーム」の間に特に顕著です。



VMが長期間使用されない場合、そのウイルス対策データベースは廃止され、「脆弱性ウィンドウ」が表示され、これらは企業のデータセンター全体のセキュリティに対する脅威となります。 また、従来のアプローチでは、すべてのファイル操作のセキュリティを確保できません。





仮想インフラストラクチャを保護するために従来のソリューションが展開されている場合、それらの影響はマルウェアの侵入よりもほとんど悪化する可能性があります。 ITシステムの動作を大幅に遅くし、正常に機能しなかったり、会社の従業員がビジネスタスクを完了するために問題が発生したりする可能性があります。



IS組織の問題が不十分であるか、まったく解決されない場合、既存の脅威の多くは、最新のデータセンターでも危険になります。たとえば、マイクロセグメンテーションを使用して最高のインフラストラクチャセキュリティが実装されます。 したがって、いずれにせよ、データセンターをサイバー攻撃やマルウェアから保護するには、仮想環境専用に設計された専用ソリューションを使用する必要があります。



ソフトウェア定義のデータセンターは、さまざまな問題の解決を目的とした多数のシステムを組み合わせています。 セキュリティテクノロジーは、インフラストラクチャの複雑さと規模に関係なく、適切な場所で適切なタイミングで機能する必要があります。



ネットワークトラフィックが企業ネットワークとその境界を保護するためにインストールされた物理機器をバイパスする場合、VMの数の無制限な増加、仮想化インフラストラクチャのネットワーク相互作用の問題があります。 その結果、ネットワーク保護デバイスは完全な制御を提供しません。 そして長い間、VMをオフにするとVMの更新が停止し、情報セキュリティの弱点となります。 VDIインフラストラクチャでは、ゴールデンイメージの損傷または感染により、何百もの危険な仮想マシンが発生する可能性があります。



Network Attached Storage（NAS）には、システムパフォーマンスを損なうことなくミッションクリティカルなデータを保護するために特別に設計されたアドオンソリューションも必要です。





カスペルスキーは、最新のデータセンターを保護するための特別なソリューションを提供しています。これは、パフォーマンスとシステム速度への影響を最小限に抑えます。 データセンターで使用される技術ソリューションとの統合および単一のコンソールからの管理を提供します。



企業のデータセンターを保護するためにカスペルスキーが提供するものを見てみましょう。



NSX統合とエージェントレス保護



NSXテクノロジーを備えたVMware vSphereプラットフォームを使用すると、Kas​​persky LabとVMwareの共同ソリューションにより、ネットワークトポロジを迅速に作成または再構成し、セキュリティ戦略を迅速に実装できます。 VMwareテクノロジー上に構築されたソフトウェア定義のデータセンターを保護するために特別に設計されています。





仮想環境向けのKaspersky Securityのエージェントレス保護の機能。



仮想環境向けのKaspersky Securityソリューションでは、保護されたVMに追加のエージェントをインストールする必要はありません。仮想化環境の「特別な」プロセスは、主要なデータセンターシステムのパフォーマンスを損なうことなくファイルおよびネットワークトラフィックのチェックを提供する専用のセキュリティデバイスに転送されます。 各VMは、電源を入れた直後に保護されます。





Kaspersky SecurityはAPIを介してVMwareインフラストラクチャとやり取りします。これにより、各VMを保護し、ネットワークの脅威をブロックし、システムプロセスと深く統合することができます。



自動展開により、担当者の作業が簡素化され、各VMは企業のセキュリティポリシーに従って保護レベルを受け取ります。 データセンターは、ネットワークトポロジを再構成することにより、情報セキュリティインシデントにリアルタイムで対応できます。 企業のデータセンター全体が24時間365日モードで保護されています。





ソリューションのアーキテクチャはもともと、ビジネスに不可欠なサーバーの作業への影響を最小限に抑えるために作成されました。



カスペルスキーが悪意のあるアクティビティを検出した場合、VMware NSXプラットフォームは直ちにその情報を受信し、VMware NSXプラットフォームによってITインフラストラクチャの構成を完全に自動的に再構築できます。



ライトエージェント



一部の仮想環境は、インフラストラクチャ自体とそれを保護するソリューションを統合するAPIを提供していません。 また、たとえば、VDIインフラストラクチャでは、最新の脅威に対する認識レベルとそれらの防止方法に関係なく、信頼できるユーザー保護が必要です。





ライトエージェントを備えたKaspersky Securityは、追加の保護レベルを提供します。



ライトエージェントを備えたKaspersky Securityは、VMware vSphere、Microsoft Hyper-V、Citrix XenServer、KVMなどの最も一般的な仮想化プラットフォームをサポートし、VMware Horizo​​nやCitrix XenDesktopなどのVDI環境でパフォーマンスが維持されます。





軽量エージェントは、プログラムの起動を制御し、仮想デスクトップを保護します。



専用の仮想マシン（SVM、セキュリティ仮想マシン）は、すべてのVMの集中スキャンを実行します;各VMにインストールされた軽量エージェントは、ファイルのスキャンだけでなく、メモリとプロセスのスキャンも許可します。





軽量エージェントをVDI環境に展開すると、アプリケーション、デバイス、URLの起動の制御、電子メールおよびインターネットトラフィックを分析するためのヒューリスティックモジュールなど、高度なセキュリティ機能を使用できます。



アプリケーション起動制御とは、次のことを意味します。



  • 信頼できるアプリケーションのみが実行できます。

  • 新しいアプリケーションは管理者によってチェックされます。

  • リストを動的に更新できます。

  • ブラックリストを除くすべてを実行します。

  • システムのすべてのユーザーに対して機能します。





仮想環境向けのKaspersky Securityのライトエージェントを使用したセキュリティ機能。



アプリケーション特権制御は、個々のVDIマシン内のアプリケーションアクティビティを制御し、ルールとポリシーに基づいてリソースへのアクセスを制限します。 仮想デスクトップは、ランサムウェアウイルスなどの複雑な攻撃からも保護されます。



侵入防止システムは、ネットワーク攻撃ブロッカー（NAB）とファイアウォールを組み合わせています。 最新のネットワーク攻撃を検出およびブロックし、悪意のあるアクティビティのネットワークトラフィックを監視します。 また、デバイス制御は、ストレージデバイス、ネットワークデバイス、印刷デバイス、およびシステムバスへのアクセスを制御し、プライバシーとデータ漏洩に対する保護を提供します。



着信および発信トラフィック（HTTP、FTP）の保護も提供されます。 各リンク（URL）はライトエージェントによってチェックされ、悪意のあるフィッシングではないことを確認します。 POP3、SMTP、IMAP、およびNNTPプロトコル保護は、各VDIユーザーのインシデント通知で各電子メールをインターセプトおよびスキャンします。



ライトエージェントでの保護はどのくらい複雑ですか？正しくインストールする方法は？ このプロセスにはいくつかのステップが含まれます。 必要：



  • KSCにKSVコントロールプラグインをインストールします。

  • セキュリティサーバーをハイパーバイザーに展開します。

  • 保護サーバーのポリシーと設定を定義します。

  • 更新タスクとキー配布タスクを作成します。

  • ライトエージェントインストールパッケージをVMに配布します。

  • KSCで、ライトエージェント作業ポリシーを構成します。



ストレージセキュリティ



ストレージシステム用Kaspersky Securtiyは、ICAPおよびRPCプロトコルを使用して多くのエンタープライズレベルのネットワークストレージシステムと統合し、各ファイル操作を保護し、重要な企業ファイルのマルウェア感染のリスクを排除します。 ウイルス対策エンジンは、起動または変更されたときにすべてのファイルをスキャンします。ヒューリスティック分析では、新しい脅威や未知の脅威も検出できます。



スキャン設定を使用すると、バックアップなどの特定のファイル形式およびプロセスとともに、スキャンから除外できるいわゆる「信頼ゾーン」を作成できます。



おわりに



VM保護用の「重い」ソリューションをインストールすると、高負荷が発生し、ハイパーバイザー上のリソースの使用が大幅に増加し、ソフトウェア定義のデータセンターのすべての利点が失われる可能性があります。 古いアプローチを使用したり、データセンター保護の組織に十分な注意を払わなかったりすると、その障害につながり、ビジネスと評判に深刻な損害を与える可能性があります。 セキュリティソリューションがそのようなインフラストラクチャ専用に設計され、コアテクノロジーと効果的に統合され、システムの運用とパフォーマンスに悪影響を与えないことが重要です。



同様に重要なのは、仮想サーバーを展開する際のクラウドのセキュリティです。 たとえば、商業データセンターの所有者であるプロバイダーは、Windows ServerおよびLinuxを実行している仮想サーバー（VPS / VDS）のウイルス対策保護を顧客に提供します。 軽量エージェントはクライアントの仮想マシンにインストールされ、その展開により高度なセキュリティ機能を有効にできます。 そのため、RUVDSは、 Kaspersky Security for Virtualization 4.0 Light Agent （仮想環境向けの軽量のKaspersky Anti-Virus Agent）を提供しています。 リソース消費のピークは、VPS検証タスクを自動的にグループ化し、優先順位を付けるインテリジェントなタスク分散システムを使用して平滑化されます。



セキュリティシステムは、VM上で実行されているプログラムの動作を制御し、悪意のあるアクションをロールバックし、暗号化プログラムから保護します。 ネットワーク攻撃保護テクノロジーと侵入防止システムは、仮想インフラストラクチャをネットワークの脅威から保護します。 外部URLを制御することにより、セキュリティポリシーに準拠していないか、リスクが高いインターネットリソースから各VMとそのユーザーを保護できます。 プログラムの起動とその権限の制御により、ユーザーのアクティビティが制御されるため、信頼できるアプリケーションのみがVMで起動できます。



メールとWebトラフィックを保護するシステムは、企業ユーザーと「外の世界」との間のデータ交換と通信のセキュリティを保護し、デバイス制御は、仮想化されたUSBデバイスとネットワークプリンターが接続されている場合、各仮想デスクトップを保護します。
