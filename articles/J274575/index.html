IBMはApache Sparkとの連携を継続：企業はSpark-as-a-serviceを開始





IBM Insight 2015カンファレンスで、いくつかの興味深い発表が一度に行われました。 主なものは、Apache Sparkプロジェクトをサポートするというアイデアの開発の継続です。 IBMは、Apache SparkでIBM Analyticsを起動します。Bluemixはクラウドプラットフォームとして機能します。 IBMは6月に、数年間で3億ドル以上のプロジェクトに投資する意向を発表したことを思い出してください。 さらに、Linux用のApache Sparkがz Systemsによってサポートされることが以前に発表されました。



このようなサポートは、メインフレーム分析プロジェクトの一部として提供されます。 これにより、データマイニングの専門家は強力なz SystemsメインフレームでApache Sparkを使用できるようになります。



Apache Sparkは、Bluemixプラットフォーム上のサービスとして機能するだけでなく、Cloudant NoSQLソリューションやSashDBクラウドストレージプラットフォームなど、他のクラウドおよび分析ソリューションとも統合されます。 Bluemixを使用する開発者は、プロジェクトをIBMの分析ソリューションおよびDBMSと統合できます。



IBMはSparkとともに、Insight Cloud Servicesと呼ばれるものも提供しています。 このソリューションを使用すると、「Twitter、The Weather Companyなどのソースから、人、イベント、企業、ビジネスプロジェクトに関する外部データ」を取得できます。 IBMのお客様は、Insight Cloud Servicesを使用して既存の情報を補完および拡張し、Apache Sparkを使用して収集されたデータ複合体の完全な分析を実施できます。



Sparkは機械学習、自然言語認識、画像処理技術をサポートし、他の多くの機能も提供するため、IBMはSparkを本格的なデータ環境と見なしています。 たとえば、Insight Cloud Servicesの一部であるIBM Datacapサービスを使用すると、クライアントは、その形式と構造、テキストおよび数値情報など、ドキュメントのコンテンツを自動的に分類および認識することができます。



同社は、ツールの信頼性が非常に高いと考えているため、15を超えるIBM独自の商用および分析製品がSparkに移行されました。 これにより、たとえば、 DataWorksのコードの行数を4000万行から500万行に減らすことができました。



すでに近い将来、IBMは、Apache Sparkのサポートを、自社ビジネスのすべての分野の分析を超えて拡大します。
