シベリアからのスピーカーの目を通してジョーカー2014 Java会議

最近、Java-Joker 2014カンファレンスがサンクトペテルブルクで開催されましたが、私の意見では、ロシアで最も類似したカンファレンスの中で最も技術的に豊かで大規模なカンファレンスです。



昨年のレポートが気に入ったので、今年の講演者としてのJavaに参加することにしました。 この記事では、イベント、レポートの品質、および同志の意見について意見を述べています。 Java 9リリースに関するジガード。







会議の数ヶ月前に、私はそれでプレゼンテーションをするためのリクエストを送信しました。 数ヶ月の長い選択の後、私は待望の確認と、トラックの1つで最初に演奏するという良いニュースを受け取りました。



アプリケーションを受け入れた後、いつものように、私はパイプに息を吹き込んで何もせずに、プレゼンテーションをゆっくりと彫ることができるようです。 しかし、そこにありました！ 最終報告書を主催者に提出する前に、厳密な事前聴取といくつかのトレーニングを経なければなりませんでした。 このチームのマニアにアプリケーションを送信するときは、このことに留意してください！



リンクを使用して会議のスケジュールに慣れることができます。講演者が技術や彼の会社を一般に「販売」する会議プログラムに実質的に報告がなかったことは素晴らしいことです。 レポートは、技術を自分で作成するか、技術を積極的に使用して、何らかの形で彼らの分野の「教祖」になる専門家、テクノエバンジェリストによって提示されました。







次に、私が最も覚えている、または何らかの形で私の精神に影響を与えたレポートについて説明します。



この日は、ロシアで最も人気のあるソーシャルネットワークの1つにかかる負荷が、エッフェル塔、エジプトのピラミッド、その他の世界の驚異と比較される、オレグアナスタシエフが発表したオドノクラスニキからの未来的な基調講演で始まりました。







それから、朝のコーヒーの散歩道だけでなく、データサイエンティストの主要な作業ツールとしてJavaを使用する方法についての話をして、聴衆を元気づけました。







プレゼンテーションのスライドはこちらから入手できます。



また、私のレポートのビデオをあなたの注意を喚起します。







物語の中で、私はTamtekで関連プロジェクトに取り組んだ経験と私の研究結果の両方を組み合わせて、学生にJava開発者をデータマイニングエンジニアに進化させる方法を提供しようとしました。







ピーターズバーグの人々は、非常に気さくで攻撃的ではないことが判明し、報告後に私を脳破りのケースと質問で満たした。 一般的に、非常にやる気のあるリスナーがキャッチされました。



一方、「Raspberry Piからの心とJava Embeddedの頭脳を使って1人のプログラマーの力に変える発電所」の傍観者に関する議論が展開されました。







昼食後、私は眠りたかったのですが、Christoph EngelbertとPeter Lawreyは、このパッケージの関数を左右に使用し始めた場合にコードで得られる信じられないほどの機会と危険性についてのUnsafe Java Worldの話を聞きました。 スピーカーコンタクト付きスライドはこちらで利用できます 。

しかし、何よりも最初の日は、3人のSpring達人の決闘に触発され、コード、XML、Groovyコードを介してSpring構成のいくつかのタスクをリアルタイムで解決しました-configs。 友情は勝とうとしましたが、精神分野でのBaruch Sadogurskyの説教者であるGroovyの優位性は、私の心の中でGroovyを支持する結果を決定しました。 1か月の使用で、フライトは正常です！



また、このJavaに似た言語の助けを借りて解決された多くの実際的な問題について、上記の著者によるもう1つのレポートをすべての若いGroovyファンに強くお勧めします。 スライドはこちらから入手できます 。
