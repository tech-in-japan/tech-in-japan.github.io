更新されたCisco CCNA認定プログラム

この記事は、Fast Laneと共に作成されました。



数か月前、シスコはCCNA Routing and Switching（Cisco Certified Network Associate Routing and Switching）になるために必要な試験を更新しました。 試験に加えて、CCNAステータスの候補者を準備する公式トレーニング （コース）が更新されました。 フレッシュ試験には多くの新しいトピックが含まれ、更新されたコースにはこれらのトピックに関する資料が含まれています。つまり、現時点では、新しいバージョン3.0のCCNA Routing and Switchingを扱っています。 この記事では、CCNA認定プログラムにどのような変更が加えられたか、そして現在のCCNA R＆Sが今日どのように見えるかを正確に見ていきます。







変更について説明するには、まずCCNAの以前のバージョン（バージョン2.0）がどのようなものであったかを覚えておく必要があります。 ここで、2つの方法でCCNAステータスを取得できることをすぐに言及する価値があります。



  1. 2つの試験ICND1とICND2に合格します（ICND-Interconnecting Cisco Network Devices）。 正確に同じ名前のシスコの公式コースは、これらの試験に対応しています。
  2. ICND1およびICND2試験のトピックを含む1つのCCNA試験に合格します。


どちらを選ぶかは、誰もが自分で決めます。 理論的資料の研究に対するより徹底的なアプローチを持つ候補者は、通常、最初のパスを選択します。 このような候補者は、1年目の理論部分を習得し、1つの試験に合格した後、2年目のプロセスを繰り返します。 彼らの意見では、すでにシスコの機器でかなり多くの経験を持っている人は、1つの試験に合格することを選択します。 さらに、これらのオプションの間にお金の違いは事実上ありません。



すでに述べたように、更新されたCCNAバージョン3.0は、まず試験の新しいトピックとそれに対応するコースを意味します。 たとえば、旧バージョンのICND1コースには次のトピックが含まれていました。



  1. シンプルなネットワークを構築します。 ここでは、パケットネットワークの用語を検討し、トポロジとは何か、7レベルのISO / OSIモデルがどのようなもので、なぜなのか、そしてTCP / IPスタックの一般的な考え方を示しました。 その後、スイッチの論理であるローカルネットワークを構築する原理について説明し、Cisco IOSオペレーティングシステムで動作する基本的なスキルを示しました。
  2. インターネット接続。 コースのこの部分では、第3レベルでデバイスの相互作用を調べ、IPプロトコルの概念を示しました（IPアドレス指定の原理の詳細な説明も行います）。 次に、トランスポート層とTCPおよびUDPプロトコルについて検討しました。 その後、ルーターとは何か、パケットネットワークで使用する必要がある理由、外観、およびロード方法についてのアイデアが与えられました。 ところで、同じトピックで、パケットネットワークを介して送信者から受信者にデータを配信するプロセス（TCP / IPプロトコルスタックの既存の知識を使用）について十分に詳細に説明しました。 次に、静的ルーティング、ACL、動作原理、NAT設定の概念が導入されました。
  3. ネットワークデバイスのセキュリティを管理します。 ここでは、広範なACT資料が提供され、ポートセキュリティテクノロジーが検討され、Ciscoネットワークデバイスの保護に関する一般的な推奨事項（パスワードの設定、ユーザー認証）も提供されました。
  4. 中規模ネットワークの構築。 このセクションでは、VLANテクノロジーとスイッチ間のトランクリンク、異なるVLAN間のトラフィックルーティング、および動的ルーティング方法について説明します（OSPFプロトコルの基本構成が考慮されました）。
  5. IPv6の概要。 ここでは、学生は、IPv6プロトコルと関連するプロトコルおよび技術（ICMPv6、近隣探索）に精通し、動的な方法の使用を含むIPv6トラフィックのルーティングを検討するよう招待されました（OSPFv3についてのアイデアが与えられました）。
  6. コースの最後に、スーパーラボが提供されました。半日のラボ作業で、合格したすべての資料を統合するのに役立ちます。


実際、5日間のトレーニングで、この一連の理論的トピックと関連する実験室作業をマスターすることが提案されました。 合計で、最後に13の実験室と1つのスーパー実験室がコースで計画されています。 ICND1 3.0の新しいバージョンには、次の変更が加えられています。



  1. テーマはモジュールに再配置され、素材の認識がより論理的で一貫したものになりました。
  2. 現在学生に提供されているラボは、ディスカバリーラボとチャレンジラボの2つのカテゴリに分類されます。 ディスカバリーラボには、コース参加者が自分で行うことができる非常に難しいタスクは含まれていないと想定されていますが、公式コースのインストラクターに特定の機能を設定するプロセスを説明するためにこのラボの進捗状況を示すよう依頼することができます これらのラボは、検討中のトピックごとに提供されます。コースには28のラボがあります。 チャレンジラボは、理論資料の特定の部分を統合するように設計されています。 計画どおり、各リスナーが個別に実行する必要があります。 このような実験室作業には、さらに多くのタスクがあります。 ICND1コースには17のチャレンジラボが含まれています...つまり、より実用的なタスクが新しいバージョンに追加され、理論のより完全な統合に貢献していることがわかります。
  3. コース終了時にスーパーラボを削除しました。


古いバージョンと新しいバージョンの違いはそれほど大きくないと思われるかもしれませんが、非常に重要な変更が1つあります。これは、教材と実験室の作業へのアクセスです。 トレーニング資料（いわゆるスチューデントガイド）が電子的に提供されるようになりました。 公式トレーニングICND1 v.3.0に参加している学生は、cisco.comのアカウント内のコース資料にアクセスできます。 つまり、コースに関するペーパーブックは発行されなくなり、インターネットに接続していれば資料にアクセスできます。



変更され、実験室作業の概念。 以前にICND1コースが特定のトポロジに組み立てられた実際のシスコ機器を使用していた場合、仮想ラボが使用されます。 つまり、仮想スイッチとルーターで実践的なトレーニングが行われます。 実際、仮想マシンを使用すると、タスクごとに異なるトポロジを収集できるため、つまり、各ラボ作業のタスクを修正が必要な材料に合わせて調整できるため、これは良いことです。 最も肯定的な点は、公式コースの学生の実験室作業へのアクセスがコース自体の終わりで終了するのではなく、180日間残るということです。 つまり、コースを修了した学生が提案されたプロトコルとテクノロジーのセットアップを練習したい場合、6か月以内にそれを行うことができます。 これは、試験の準備という点で非常に便利です。試験の直前に練習できます。



ICND2コースも大幅に変更されました。 古いバージョンは全体として、さまざまな技術やプロトコルの動作の不具合を検出するプロセス（いわゆるトラブルシューティング）に焦点を当てており、次のトピックが含まれていました。



  1. スケーラブルな中規模ネットワークの構築。 ここでは、VLANテクノロジーとトランクリンクを使用するL2インフラストラクチャのトラブルシューティングテクニックを学ぶことを生徒に奨励しました。 また、STPプロトコルのロジックと、その最適化およびトラブルシューティングのプロセスについても説明しました。 Cisco CatalystスイッチのEtherChannelテクノロジーとその構成ロジックについても説明しました。
  2. パケットネットワークで最も単純な障害を検索します。 クライアントとサーバー間の接続性の欠如の例を使用して、学生は、トラブルシューティングのツールとそのア​​プローチに精通するように招待されました。
  3. EIGRPプロトコルの実装 。 このパートでは、EIGRPルーティングプロトコルの動作の基本的な理論的側面、およびその構成とトラブルシューティングのロジックを開示しました。 また、IPv6を使用するネットワーク用のこのプロトコルの構成についても説明しました。
  4. 複数のエリアでOSPFを使用してスケーラブルなネットワークを展開します。 ICND1コースが1つの領域でOSPFプロトコルの動作を調べた場合、このコースはこのプロトコルに関する高度な知識を提供し、学生はいくつかの領域でOSPFの動作を習得、構成、検証するように求められました。 OSPFの失敗のトラブルシューティングプロセスに関する推奨事項も作成されました。
  5. グローバルネットワーク。 リスナーは、ルーターでのシリアルインターフェイスの使用、およびHDLC、PPP、およびフレームリレープロトコル（後者が多くの注目を集めた）を使用するための構成について詳しく紹介されました。 さらに、このモジュールでは、学生はVPNテクノロジーに精通し、最も単純なGREトンネルの構築と構成のロジックを理解するように招待されました。
  6. ネットワークデバイス管理。 このモジュールでは、学生はネットワークと個々のデバイス（SNMP、NetFlow、Syslog）を管理するためのプロトコルと技術のアイデアを与えられました。 学生は、ルーターの読み込み方法と、このプロセスの管理方法も学びました。 モジュールの最後で、Cisco IOSソフトウェア15.0以降のISR G2シリーズルータ（29xx、39xx）のさまざまな機能のライセンス問題が考慮されました。
  7. コースの最後に、学生はスーパーラボを提供され、両方のコースの材料であるICND1とICND2を統合したタスクを完了しました。 一般に、この実験室の作業は、対象となるすべてのトピックに関するかなり大きなタスクのセットでした。


ICND2 3.0の新しいバージョンには、次の違いがあります。



  1. DTPおよびVTPプロトコルのレビューがコースに追加されました。 以前は、これらのプロトコルはSWITCHコース、つまりプロフェッショナルレベルのコースでのみ検討されていました。
  2. 別のデバイス間の接続モジュールに、より多くのラボが追加されました。
  3. HDLCプロトコルは考慮事項から削除されました。 ただし、マルチリンクPPPおよびPPPoEクライアントの構成は、グローバルネットワーク経由でモジュールに追加されます。 ところで、一部のリスナーは、PPPoEクライアント構成がどのコースでも考慮されていないと不平を言うことがあり、その分析のためにドキュメントを掘り下げる必要があります。 CCIEの準備のために、誰かが実際のクライアントを構成する必要があります。
  4. コースからフレームリレーを削除しました。 多くの学生や試験に合格する候補者にとって、コース内および試験中のフレームリレーの存在の問題はつまずきであるため、これを別のポイントとして強調します。
  5. このコースでは、BGPプロトコルのロジックと、単一のサービスプロバイダーに接続した場合のeBGPセッションのセットアッププロセスに関する基本的な説明を追加します。
  6. TACACS +およびRADIUSプロトコルの詳細情報が、ユーザー認証プロセスにネットワークデバイスを使用するコンテキストでネットワークデバイスを管理するためのモジュールに追加されました。
  7. QoSツールの基本的な知識を追加しました。
  8. SDNに概要モジュールを追加しました。


研究所の作業も変更されました。古いバージョンのICND2に18しかなかった場合、新しいバージョンでは、25のディスカバリーラボと19のチャレンジラボが提供されます（このコースのディスカバリーとチャレンジラボの概念も保持されます）。 つまり、コースの重点は実践的な演習にシフトされ、理論だけでなく提案されたトピックを研究することができます。 さらに、ICND2コースの新しいバージョンとICND1は、教育資料と実験室作業へのアクセスという同じ概念を使用しています。 公式のICND2 v.3.0の学生は、研究室にアクセスしてから6か月間、研究室での作業を続け、提案された技術とプロトコルを設定するスキルを磨くことができます。



CCNAステータスを取得するための試験と明らかに相関する2つの言及されたコースICND1とICND2に加えて、 CCNAX （CCNA Accelerated）と呼ばれるトレーニングもあることも注目に値します。 これはICND1およびICND2コースの要約を含む5日間のコースです。 このコースは通常、パケットネットワークの動作、シスコの機器に基づいた構築の論理、およびこれらのネットワークで機能するプロトコルとテクノロジーの基本的な理解を得たいが、残念ながらトレーニングに費やすことができない人に適しています。最長2週間（ICND1およびICND2コースを完全に聞くには時間がかかります）。 新しいバージョン3.0のCCNAXコースプログラムには、ICND1およびICND2コースでカバーされているすべてのトピックが含まれていますが、理論部分（論理的）に費やされる時間が短縮されます。 実践的なトレーニングの観点から、このコースでは53のディスカバリーラボと15のチャレンジラボを提供しているため、この5日間のコースには十分なプラクティスがあります。



実際に、関連する試験に合格するための候補者を準備する公式のシスココースのトピックが提示されました。 試験自体に関しては、変更は非常に簡単です。 新しい試験の番号は次のとおりです。



  1. ICND1-100-105
  2. ICND2-200〜105
  3. CCNA-200-125


伝統的に、試験の内容は公開されていませんが、それらの主題が言及されたコースに対応すると仮定することは論理的です。 試験の古いバージョンは、8月20日（CCNAおよびICND1）および9月24日（ICND2）まで利用できます。 前と同じように、CCNAステータスを取得するには、2つの試験ICND1 + ICND2、または1つのCCNAに合格する必要があります。 新しい試験の登録は、Pearson VUEから引き続き利用できます。
