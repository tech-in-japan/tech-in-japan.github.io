Mac OS Xの画面ロック

ある晴れた日、mac os xをメインのオペレーティングシステムとして使い始めました。 そして、それ以来、私は以前使っていた機能のいくつかが何らかの形で欠落していることに気付くことがあります。 10年以上にわたってコンピューターを使用していたため、2分間でも離れると、デスクトップをブロックするという強い習慣を身に付けました。



まず、ほとんどすべてを設定できる「ホットキー」で検索することにしましたが、「ロック」はありませんでした。 私は少し検索する必要がありましたが、判明したとおり、いくつかの解決策があります。 従来は、2つの方法に分けることができます。 まず、スクリーンセーバーを無効にするためにパスワード要求をブロックします。 第二に、ユーザーの素早い変更。



スクリーンセーバーロック



スクリーンセーバーを無効にするためのパスワード要求を有効にするには、[システム環境設定]-> [セキュリティ]に移動し、[スリープまたはスクリーンセーバーからコンピューターをウェイクアップするにはパスワードが必要]チェックボックスをオンにします。



これは悪い選択肢ではないように思えますが、オンになるのを待つことはまったく面白くないです。 最も厄介なことは、単純な方法（サードパーティアプリケーションなし）では、キーの組み合わせでスクリーンセーバーをオンにすることに成功しなかったことです。 Google、読んで、Exposeについて覚えておいてください。 カーソルをいずれかの角に移動すると、スクリーンセーバーをアクティブにできます。



非常に長い間、この方法が最もお気に入りでした。



ユーザーの素早い変更



デンマーク王国ではすべてがそれほど悪いわけではないことが判明しました。パスワードを入力するためのデスクトップをブロックするユーザー間の切り替え機能があります。 また、非常に簡単にアクティブ化されます。 これを行うには、[システム環境設定]-> [アカウント]に移動します。 ログインして「高速ユーザー切り替えを有効にする」チェックボックスを有効にするには、ロック画像をクリックします。



これで、ユーザーを切り替えるための美しいアイコンができました。



それをクリックし、「ログインウィンドウ...」項目を選択し、ブロックされたときにクールな効果を観察します。



キーに戻ると、進行状況はゼロです。これは正常なため、ある組み合わせではユーザーの切り替えも機能しないためです。 組み合わせの連鎖を伴う倒錯については説明しません。



クイックシルバー



しばらくして、無料のquicksilverユーティリティとそのトリガー機能を発見しました。これらはホットキーに掛けることができます。 そして、cmd + shift + Lの組み合わせにより、スプラッシュスクリーンを含めるための小さなスクリプトを書くというアイデアに思いつきました。 このために必要なもの：



1.クイックシルバーを入れる


quicksilver.en.softonic.com/macからダウンロードできます。インストールは簡単です。



2.スクリーンセーバーを起動するスクリプトを作成する


記述には、標準アプリケーション「Script Editor.app」を使用します。 以下のテキストを開始して記述します。



tell application “ScreenSaverEngine”



activate



end tell







次のようになります。







3.スクリーンセーバーのアクティベーションスクリプトを保存する


好きな場所に保存します（ホームディレクトリに保存しました）。パスを覚えておいてください。



4.目的のキーの組み合わせにトリガーを追加します


これを行うには、quicksilverの[トリガー]タブを開きます。





QSのトリガー

下のプラス記号をクリックして、ホットキー項目を選択します。 図のように、セルに前の手順で作成したスクリプトを追加します。





ボーナス機能



その後、アイデアが思い浮かびました。画面がロックされたらiTunesを一時停止してみませんか？ これを行うには、スプラッシュスクリーンを起動するスクリプトの最後にそのような行を追加するだけです。



tell application “iTunes”



pause



end tell







新しい機能により、生活が少し楽になります。
