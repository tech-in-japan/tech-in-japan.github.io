Visual Studio 2012で1つのEntity Frameworkモデルに基づいて複数の図を作成する

これは、多くのエンティティでEntity Frameworkビジュアルモデルを使用する開発者や企業が待ち望んでいた待望の機能です。



数百のエンティティを持つデータモデルを想像してください。 少なくとも200のエンティティのモデルダイアグラムで作業するのは非常に困難であり、作業が遅くなる可能性があり、そのようなダイアグラムはあまり管理できません。







しかし、今では、Entity Framework 5.0とVisual Studio 2012の登場により、同じデータモデルの多くの個別のダイアグラムを持つことができます。 そして、これは、異なるエンティティを異なる色で着色する新しい機会よりもさらにクールです。



例として、上記のモデルを見てみましょう。そこで、いくつかの異なるエンティティ（顧客に関連する）を選択し、その後、マウスの右ボタンで「Mode to new diagram」オプションを呼び出します。



ご覧のとおり、小さな図が指先に表示され、顧客に関連するエンティティのみが表示されます。 この図をCRMと呼びました。







単純なCOPY / PASTEを使用して異なる図を作成できるため、データモデルの任意の図に任意のエンティティを表示できます。



両方の図は、基本的に、使用可能なすべてのエンティティを含む同じデータモデルのVIEW表現であることに注意してください。



このモデル（* .edmx）には多くの図を含めることができます。 これは、大規模で複雑なデータモデルのさまざまな領域を視覚的に分離するための非常に便利な方法です。
