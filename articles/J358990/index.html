ハリコフでのmitap RubyRoars＃1の発表





Sync.NET 、 ThinkPHP 、 ThinkJava 、 WordPress Kharkiv 、 iThinkの会議を開催した豊富な経験により、チームは論理的な質問に導かれました。Rubyコミュニティを作成するときが来たのでしょうか。 今こそ、すべてのRuby開発者をRubyRoarsキックオフミーティングに招待する理由です。これは、NIX Solutionsからの新しい大規模なコミュニティの先駆けとなるように設計されています:)。



会議で何を話しますか？



1）ウラジミール・モズギン、「レール-良い規範の哲学」

この講演では、Sオブジェクト指向設計の最初の基本原則である単一の責任で、Railsプログラミングの初心者を紹介します。 責任の領域、プロジェクトのどこでどのような機能を使用するか、何を担当するかに従って、オブジェクトの役割を分析しましょう。 また、シンコントローラーとモデルについても説明します。



2）アントン・トカチョフ、「個人的な専門的価値の成長」

このレポートは、お客様にビジネス面と顧客の期待を明確に理解させます。 参加者は、優れた開発者コードの測定基準が、それを支払う人の測定基準とはかけ離れていることが多い理由を学びます。 また、プログラミングに加えて、どの領域を開発する必要があるか、どの領域に注意を払うべきかを把握します。 デザートについては、個人的なブランディングについて話しましょう。



3）ミハイル・ステファンスキー、「Practical React / TypeScript」 *

反応する理由 TypeScriptを使用する理由 一般的なパターン（コンテナー/プレゼンテーションコンポーネント、大規模アプリケーションでの状態の処理）、モバイルへの移行-Webアプリケーションとネイティブアプリケーション間でコードを再利用するためのパターン（何をいつ再利用するか）。

*レポートは英語になります。



私たちはレポートを提示し、質問に答え、経験を共有し、週の終わりに料金を請求します:)。
