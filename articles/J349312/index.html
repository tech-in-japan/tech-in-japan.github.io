お金の出番：シスコITインフラストラクチャ支出レポート

同社は、Cisco Global Cloud Indexレポートの結果を発表しました。 企業がITインフラストラクチャに費やす金額、およびクラウドプロバイダー市場がどのように発展するかについて話しましょう。



/写真コムスコープ CC BY



Cisco Global Cloud Index（GCI）は、クラウドおよびデータセンターのグローバルIPトラフィックメトリックを測定および予測します。 このレポートは、2016年から2021年までのクラウドコンピューティングとデータセンターの仮想化に関連する傾向を強調しています。



レポートを編集する際、データセンター内およびデータセンター間のトラフィックデータ、およびデータセンターとエンドユーザー間のデータ交換が考慮されました（ 方法論の表1）。



ハイパースケールはいくらですか



最近、ハイパースケールデータセンターの数が増加する傾向について説明しました。 シスコのレポートの主な傾向の1つは、ハイパースケールインフラストラクチャの開発にも言及しています。



シスコの予測によると 、超スケーラブルなデータセンターの数は2021年までに628に達します（現在338になります）。 この時点で、そのようなデータセンターのトラフィック量は4倍に増加し、データセンター内の総トラフィックの55％（現在は39％）になります。



シスコは、非常にスケーラブルなデータセンターの数を計算するために、クラウドプロバイダーの収益を分析しました。 レポートのコンパイラーのロジックは次のとおりでした：IaaS / PaaS / SaaSサービスを提供する企業がこれらの活動分野からの収益について選択された基準を満たしている場合、それはハイパースケーラブルなデータセンターの運用を保証できるインフラストラクチャを持っていることを意味します。 レポートによると、ハイパースケールプロバイダーの年間収入の最小要件は次のとおりです。



  • IaaS、PaaS、ホスティングプロバイダー（Rackspace、Google）に10億ドル。
  • SaaSプロバイダー（Salesforce、ADP、Google）に20億。
  • インターネットプロバイダー、検索エンジン、ソーシャルネットワーク（Facebook、Yahoo、Apple）に40億。
  • 電子商取引および支払い処理サービス（アリババ、eBay）に80億。


これまでのところ、これらの基準に該当するデータセンターは24あります。 ところで、見事な収益にもかかわらず、超スケーラブルなデータセンターは非常に高価なプロジェクトです。 Platformonomicsの分析によると、AWS、Microsoft、Googleは1,000億ドルを費やしてハイパースケールインフラストラクチャを作成しました。



彼らは何にお金を使っていますか



Spiceworks の分析部門のレポートによると、企業の44％が2018年にIT予算を増やす予定であり、43％がIT予算を変更せず、11％がITコストの削減を望んでいます。 ]。 組織の規模に応じて、クラウドインフラストラクチャのコストは総IT予算の約3分の1になります。



このレポートにはもう1つの興味深い点があります。すべてのソフトウェアコストの10％が仮想化に関連するタスクに割り当てられます（仮想インフラストラクチャの例 ）。



これは、シスコのレポートに記載されている傾向と一致しています。 GCIのレポートで、同社は世界中のビジネスのクラウドテクノロジーへの移行のペースを強調しました 。 クラウド内のワークロードの量は、2016年から2021年にかけて2.7倍に増加すると予想されます。 同時に、従来のインフラストラクチャにかかるワークロードの量は減少します。平均的な年間減少率は5％です。



これらの傾向は他の企業でも注目されています。 Gartnerによると 、2020年までにIaaSとPaaSの市場規模は72億と210億ドルに達するとのことです。 さらに、シスコは、スマートカー、都市、および関連アプリケーションなどのIoTソリューションへの投資の増加に注目しています。 2016年の58億ドルに対して、2021年までにこのIoT市場は137億ドルに達すると予想され、追加のクラウドインフラストラクチャリソースが必要になります。

個別のアプローチ



大規模なIT企業（Google、Facebook、Microsoftなど）は、個々のニーズに合わせて設計された機器に積極的に投資しています。 たとえば、カスタムハードウェアを使用して、 Open Compute Project（OCP）が実装されます。 Facebookは（ 彼らによれば）「これまでにないスケーラビリティを最低価格で提供する最もエネルギー効率の高いデータセンター」を作成するためにそれを立ち上げました。



その結果、エンジニアのチームは独自のデータセンターを設計することができました。OCPによると、以前Facebookで使用されていたデータセンターよりもエネルギー効率が38％向上し、保守が24％安くなります。 現在、他の企業がプロジェクトに参加しています-OCPの一部として、「カスタムデータセンター」の作成分野でベストプラクティスを交換しています。



たとえば、OCPに基づいて、Microsoftはソリューションを開始します。 Zettastructureカンファレンスで、ITの巨人はオープンソースハードウェア開発の新しいモデルであるOlympusプロジェクトを導入しました 。 Azureは、Fv2ファミリの仮想マシンを使用した実稼働環境で既に使用しています。 マイクロソフトは、このプロジェクトにより、顧客が財務モデリングとディープラーニングの問題を解決できると主張しています。



さらに、さまざまな負荷に対応したカスタムチップの開発が進んでいます。 これらのプロジェクトには、ディープラーニング用のGoogleのテンソルプロセッサ （TPU）と、Azureシステムを加速するためのマイクロソフトのプログラマブルゲートアレイ （FPGA）があります。 Intelはまた、 OracleやFacebookなどのIT企業の個々のニーズに合わせてプロセッサチップを絶えず開発しています 。 Intelは、オプションで追加のインターフェイスを実装したり、電子機器をCPUコアに直接接続したり、プロセッサ命令の個々のセットを作成したりできます。
