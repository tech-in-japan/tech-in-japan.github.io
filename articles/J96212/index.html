AWS Import / Exportは、大量のデータのS3ストレージへの転送を加速します

AmazonはAWS Import / Exportサービスを発表しました。 彼らによると、メールサービスを使用し、場合によってはインターネットをバイパスして最終的な物理デバイスとの間でデータをダウンロードする方が非常に高速です。





問題の原因は、正しいアメリカのインターネット速度にもかかわらず、非常に大量のデータをS3にアップロードするのに時間がかかりすぎることです（1TBから始まるデータ量を読み取ります）。



Amazonは、APIとジョブエンジンを使用してS3からデータをインポート/エクスポートできる物理デバイスを送信するためにデータを転送するようユーザーに提供します。



デバイスの要件は、ほぼすべてのスペクトルに対応しています（USB、eSATA、3.5および2.5インチSATAデバイス、ファイルシステム：ntfs、ext2、ext3、fat32）。 さらに、アダプターを非標準デバイスに接続できます。



Amazonは、新しいサービスを使用することが理にかなっていることを示す表を引用しました。



インターネット接続速度 1TBデータをネットワークの80％に転送するための理論上の最小日数 AWS Import / Exportを使用する場合
T1（1.544Mbps） 82日間 100GB以上
10Mbps 13日間 600GB以上
T3（44.736Mbps） 3日間 2TB以上
100Mbps 1〜2日 5TB以上
1000Mbps 1日未満 60TB以上


サービスのコストは、デバイスごとに80ドル、データ操作の完全および不完全な時間ごとに2.49ドルです。
