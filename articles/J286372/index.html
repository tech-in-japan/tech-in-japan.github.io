サイトを作成しない方法。 LandingストアのYandex統計へのオープンアクセス

同僚、最初は本質です-私は私のサイトの1つ（着衣衣料品店）のYandexの一般的な統計情報を開きます。 ユーザーの行動の分析に興味があり、少し乾燥した統計を描く人は大歓迎です。



そして今、私はそれがすべてを行うことに決めた理由と理由を説明します。



注：訪問の統計に関するすべての値は、記事の執筆日-03/04/15に記載されています。



少し前に、私は小さな予算を使い始め、自分のビジネスを開こうと決心しました。 いくつかのオプションを試した後、サイトの作成を停止しました。 可能性が限られていたため、本格的なウェブスタジオの問題はありませんでした。 選択は、既製ソリューションのサイトでホストされている既製サイトのテンプレートの作成にかかっていましたBitrix Marketplace（以下、MPと呼びます）。



デザイナー、レイアウトデザイナー、プログラマーがいました。 そして（実際には数ヶ月が経ちましたが、これは全く異なる話です...）私たちの最初の既製の解決策が出てきました-適応性のあるランディング衣料品店「美容」。 MPでの公開に成功し、Yandexにカウンターを置くことにしました。



実際、最初は私の決定を何人の人が見ていたのか不思議でしたが、すべてがもっと面白くなりました。



訪問の統計を分析すると、サイトを開発するために設定されたタスクに応じた仮説と仮定は必ずしも正確ではなく、時には正反対になり、最終的にソリューションの販売またはむしろ不在に直接影響することが判明しました。



それで、あなたはあなたのサイトへの訪問の分析から何を学べましたか？



1.ワイドスクリーンモニターを使用しているユーザーは、思っているよりも多い



サイトを作成するためのデザイナーのタスクを設定して、私は特に詳細には立ち入らず、言った-美しいサイトを作るだけです。 レイアウトでも同じことが起こりました-適応性があることが要求され、それだけです。 当時、電話でレイアウトが正しく表示されている場合、それは他のすべての画面で正しく表示されていることを意味するように思えました。 これは誤解の1つであることが判明しました。 もちろん、HTMLおよびCSSバリデーター（記事の最後にあるソースへのリンク）を正常に渡すための基準は破られましたが、実際、これはサイトの成功にもほとんど影響しませんでした。 そして、そのような表面的なアプローチは最終的に成果を上げ、自然に否定的でした。



1920以上の画面解像度を持つユーザーがこれほど多くいるとは考えられませんでした。 したがって、この解像度では、完成したレイアウトもテストされませんでした。 特別なリソースを通じて、モバイルレイアウトのみがテストされました-それは大きな間違いであることが判明しました。 すべてが正反対に起こりました。



1920以上の解像度を持つユーザーへのアクセス数は、他のすべてのユーザーよりも多くなりました。 80人のユーザーのうち20人。この解像度では、サイトのデザインはテストされていませんでした-これは最初の失敗です。 そのような画面では、サイトはかなり惨めに見えることが判明しました。







テストレイアウトは、 quirktools.com / screenflyサービスを使用して実行されました



サイトのヘッダーが引き伸ばされすぎており、ワークスペースが圧縮されすぎています。



合計：高解像度の訪問の25％のうち、25％が失敗。 さらに、解像度が高いほど、故障率が高くなります。

•1920×1080-23.5％

•1920×1200-33.3％

モバイルデバイスからの訪問-わずか17％。

その結果、サイトをモバイルバージョンに適合させるために多くの努力が注がれましたが、モバイルクライアントはありませんでした。



2.愚かなブラウザは忘れ去られます-これは事実です



レイアウト要件の1つは、すべてのブラウザーでの正しい表示でした。 特別なことは何もないように見えますが、すべてが節度にあるべきです。 サイトをIE 8以下に適合させるのに多くの時間を費やしました。 そして実際に何が起こったのでしょうか？ そして、誰もIEをまったく訪問しないという事実は0％です。 訪問の43％がChromeから、15％がSafariから、14％がOperaから、残りは少しずつ行われました。



結論：愚かなブラウザをサポートするために大量のリソース（一時的およびマテリアル）を費やすべきではありません。 他の部分に投資する方が良い。



3. Cookieは100％含まれています



この段落には説明は必要ないと思います。



4.ユーザーは常に写真をクリックしたい



当サイトのデザインは非常にシンプルです。 何らかの形でのアニメーションは完全に存在しません-アニメーション化されたボタン、メニュー項目、アイコン、写真などはありません。これは、私の意見では、別の省略であることが判明しました。 サイトをクリスマスツリーに変えることについては決して話したいとは思いませんが、それでもいくつかの要素を強調した方が良いでしょう。

クリックマップを分析した結果、訪問者は何らかの理由で、たとえ動画がアニメーション化されていないことが明らかであっても、常に写真をクリックするよう努めていることが判明しました。

5.プラセボ効果



ご存じのように、サイトのロゴは通常、メインページへのリンクになっています。 しかし、私は考えました-なぜこれが着陸に必要なのですか？ 結局のところ、これは1ページであり、メニューはフローティングではないので、ロゴをクリックしても何にもつながりません。 それにもかかわらず、彼らはとにかくロゴをクリックします。 マウスをポイントしても、マウスポインターは変化しません。 これは理解できることです。ロゴはリンクであるという事実に誰もが慣れており、ここには何らかのテンプレートのギャップがあります。



結論：たとえどこにも通じていなくても、常にロゴをリンクではなくリンクにすることをお勧めします。 これはユーザーにとってより身近なものです。



現時点でYandexの統計から何を引き出すことができるかを要約するために、私は純粋に個人的な意見では、サイトを開発する際に犯すべきではない主な間違いを強調したいと思います。



1. 最も重要なことはテストです。 使用しないテストツールが多いほど、受け取る障害が多くなります。

2. ワイドスクリーンモニターを奪わないでください 。 1366x768の解像度はすべての訪問者の90％であるという理論から進む必要はありません。 実際には、訪問の最大の割合は1920以上の解像度であることが判明しました。

3. 愚かなブラウザの開発コストは正当化されません 。 技術は進歩しており、最小の割合がすでにこれを利用しています。

4. モバイルスクリーンの適応人件費を適切に評価します 。 もちろん、これは必要ですが、限られた予算で、他の重要な側面に焦点を合わせ、電話の理想的なレイアウトに焦点を合わせない方が正しいでしょう。 最高は善の敵です。 最初はシンプルにしましょう。プロジェクトから利益が得られたら、そのプロジェクトに取り組むことができます。

5. サイトを新聞に変えないでください 。 ボタンが押されている場合、ホバリングするときに、たとえば背景の邪魔な変更ではなく、何らかの形でこれを表示する必要があります。 小さなテキストブロックがあるアイコンがある場合-わずかに強調表示すると、ユーザーの注意を引き付けることができます。

6. ロゴは常にリンクを作成します。
