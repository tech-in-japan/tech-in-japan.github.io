Microsoft Skype For Businessに基づくテレフォニー構築オプションの分析

すべての良い一日。 企業でテレフォニーを構築する場合、ほとんどの場合、Cisco / Avaya / Siemens（Unify）/ Panasonic / NECなどの従来の電話メーカーから選択できます。 これは主に、人々が電話を選択し、古典的なメーカーを見ているという事実によるものです。 しかし、要件の中でますます多くの企業が、とりわけ、ユニファイドコミュニケーションの可能性を示しています。 そして、なぜこの問題に別のアプローチをし、ユニファイドコミュニケーションシステムを選択し、それに基づいてすでにテレフォニーシステムを構築しませんか？ この記事では、Microsoft Skype For Businessに基づいた本格的なテレフォニーを構築する可能性を検討します。



マイクロソフトは当初、OCS製品、Lyncに統合されたユニファイドコミュニケーションシステムで市場に参入し、現在はSkype For Businessに名前を変更しています。 すぐに、従来のSkypeには、このソリューションにはほとんど何の関係もないことを予約してください。 ユニファイドコミュニケーションのすべての利点（すべてのコミュニケーション/ビデオ会議/ドキュメントコラボレーション/インスタントメッセージング/ Officeとの統合/プレゼンスステータスのプラットフォーム）については詳しく説明しません。 ここでは、Microsoftがこの分野の世界的リーダーであることを既に証明しているとしか言えません。これは、「魔法の」ガートナースクエアを含む多数の独立した研究を裏付けています。 しかし、ここでは電話として、ソリューションはそれほど人気が​​ありません。 Microsoft Skype For Business（以降S4B）を検討する場合、テレフォニーに必要な機能を提供する十分な数の電話機能があります。

  1. 着信転送
  2. 着信転送
  3. 同時通話
  4. 検索グループ
  5. コールピックアップ
  6. その他。






さらに、これらの機能の構成の観点から-これらはすべてS4Bクライアントを介して簡単かつ簡単に構成でき、この機能を非常に便利で直感的に使用できます。 だから、結局のところ、Microsoft S4Bがテレフォニーほど人気が​​出ていないのは、このためのすべてが揃っているからです。 これに影響する主な要因：

  • マイクロソフトは、ソフトウェアクライアントを備えたソフトウェアプラットフォームをリリースしましたが、電話と電話ネットワークに接続するためのソリューションをリリースしませんでした。 電話は20世紀であり、ソフトウェアクライアントを使用する必要があるというマイクロソフトの概念は、SIPを介して電話ネットワークに直接接続するだけで十分であり、通信事業者はMS S4Bのすべての仕様をサポートする必要があります。 この概念は、他の多くの概念と同様に、紙上では優れていますが、実際の生活には適していません。 これまでは、電話が必要であり、マイクロソフトが提供する電話会社だけでなく、長年働いてきた人や、番号を保存して長い間働いているチャンネルと通信する電話会社とのやり取りも必要です。
  • マイクロソフトはユニファイドコミュニケーションソリューションとしてこの市場に参入しました。テレフォニー市場は、テレフォニーがユニファイドコミュニケーションの一部にすぎないことをすぐに受け入れることはできません。 電話事業者のマイクロソフト自体に対する態度は言うまでもありません。


テレフォニーハードウェアの問題を解決するために、Microsoftはパートナーシップソリューションの使用を決定しました。 認定ソリューションに関するすべての情報は、MicrosoftのWebサイトに掲載されています。

パートナーソリューションを使用すると、次の問題を解決できます。

  1. IP電話。 互換性のある電話とは、Microsoft Lync / Skype For Businessで直接動作する電話です。 この場合、Skype for Businessは独自のプロトコルを使用して動作するため、標準のSIP電話機をS4Bに接続しようとしても、動作しません。 このプロトコルの基本はSIPです。 このような電話の利点は次のとおりです（機能の観点から、AudioCodes電話の機能について説明します。他のメーカーの電話については、MicrosoftのWebサイトで確認してください）。

    • 他のアプリケーション、放電、過負荷などでビジーになる可能性がある場合、コンピューターやスマートフォンなどの多機能デバイスからの独立
    • Skype For Businessとの完全なステータス同期、および電話から直接ステータスを管理する機能
    • 電話帳の連絡先をS4B電話帳と同期する
    • 電話機からの着信転送を設定し、着信転送ステータスをSkype For Businessと同期する機能。
    • コール転送/コールピックアップ/グループコール/ヘッドセクレタリーフィルター/プレゼンス表示付きスピードダイヤルMS S4Bのサポート
    • 重要で便利な機能の1つに、Better Togetherドライバーがあります。これにより、IP電話をヘッドセットとして使用できます。 これにより、Skype For Business経由で電話をかけたり、電話からSkype For Businessクライアントにアクティブな通話を転送したり、S4Bクライアントからの通話を管理したりすることができます。


  2. 重要ではない側面は、 Microsoft S4Bと都市ネットワークの接合部です 。 ここで、Microsoftは、音声ゲートウェイとSBC（セッションボーダーコントローラー）のさまざまなデバイスを使用することを提案します。これらのデバイスは、Microsoft Skype For Businessを既存のPBXまたは都市回線に接続する役割を果たします。 さらに、これらのデバイスのラインは非常に大きく、数千のアナログラインから数千の同時接続用に設計された高性能デバイスまで、キャリアクラスのフォールトトレランスを提供します。 ポートオプションは完全に異なる場合があります。CO（FXS / FXO）、E1、BRI、SIP。 SIPプロトコルを検討する場合、いくつかのポイントを強調することは理にかなっています。

    • Microsoftは、SIP over TCPおよび/またはTLSをサポートしています。 SIP over UDP Microsoftはサポートしていません。 しかし、これを超えて、MicrosoftのSIPの実装は非常に独特です。 より正確に言うと、MicrosoftはSIP標準の非常に一般的な拡張機能を使用していませんが、どこかで独自の方法でこれらの標準を解釈します。 すべてのステーションがMicrosoft S4Bと互換性があるわけではなく、互換性がある場合でも、いくつかの制限があります。 これらの制限はすべてTechNetページで説明されています。
    • 標準のSIPを使用して、S4Bと電話交換の統合に関する制限を削除するには、Microsoft S4Bとサードパーティの電話交換との完全な互換性を提供するSBCデバイスを使用し、SIP方言を異なる側の要件に変換します。
    • また、SBCの重要な機能の1つは、パブリックインターネットを介してSIPオペレーターに安全な接続を提供することです。


  3. リモートオフィスの弾力性システム 。 Microsoft S4Bをテレフォニーシステムと見なす場合、重要なポイントはリモートオフィスのフォールトトレランスを確保することです。 この場合、マイクロソフトはSBAソリューションを使用することを提案します。SBAソリューションは、リモートオフィスと中央オフィスの間のチャネルが失われたときに、リモートオフィスのフォールトトレランスを提供します。 AudioCodesは、複数人のオフィスから大規模なリモートオフィスに至るまで、Microsoft S4B向けの独自のリモートオフィス復元ソリューションも多数提供しています。
  4. 会話録音システム。 会社のセキュリティサービスに通話録音の要件がある場合、MicrosoftはこのタスクにAudioCodes SmartTAPなどのアフィリエイトソリューションを使用します。 これにより、ユーザーが気付かないうちにすべての会話を記録し、すべての会話を一元的に保存できます。

  5. 自動応答システム 。 MicrosoftはAuto Attendant-Exchangeサーバーを使用します。 また、これらの問題を解決するために、リモートオフィスで専用の自動秘書を提供するなど、より広く柔軟な機能を提供できるサードパーティの自動秘書を使用できます。 他のSkype for Businessに適合したAutoSecretarの主な違いは、さまざまなシナリオに従って着信コールを配信するときに、システムがMicrosoft Skype For Businessのユーザーステータスを考慮することです。
  6. コンタクトセンター。 ご存知のように、現在、コンタクトセンターは多くの企業の重要なコンポーネントであり、電話交換機に統合されていることが非常に多いです。 コンタクトセンターの機能を実装するために、パートナーソリューションも使用され、顧客のビジネス要件に完全なコンタクトセンターソリューションを提供します。 さらに、コンタクトセンターの認定ソリューションのリストは非常に多く、その中にはコンタクトセンターのマーケットリーダーであるGenesysを含むコンタクトセンターの大手メーカーがあります。
  7. アナログ電話とファックスの接続。 私の個人的な経験に基づいて、このタスクはほぼすべての企業に関連していると言えます。 アナログサブスクライバーとFAXをMicrosoft Lync / S4Bに接続するためのソリューションがあります。さらに、AudioCodesはこの問題の解決に多くの注意を払っています。




このリストからわかるように、Microsoft S4Bを使用すると、UCシステムを社内に実装できるだけでなく、本格的な電話交換機にまで拡張できます。 これは、パートナーシップソリューションを通じて行われます。 さらに、パートナーソリューションのおかげで、FAXサーバー、既存のSIP電話をS4Bに接続、通話品質の管理と監視、秘書プログラムコンソールなど、他のタスクを実現できます。 Microsoft Lync / Skype For Businessの音声サービスの主なパートナーは、AudioCodesです。 Microsoft Lync / Skype For Businessに音声サービスを提供する機器の最大数を提供します。 Lync / S4の音声ソリューションを見る場合、サポートなどの要素を忘れないでください。 マイクロソフトは、認定パートナーソリューションと組み合わせて使用​​する場合、ソリューションのサポートを完全に提供します。 これは重要な瞬間ではありません。Microsoftには多くのパートナーシップソリューションがあり、Microsoftのサポートを失うことなく、クライアントには本当に選択肢が与えられているからです。



また、本当に重要になる可能性のあるいくつかのことにも触れたいと思いますが、誰もがそれに気付くわけではありません。

  1. Microsoftは、SkypeおよびSkype For BusinessソリューションでSILKコーデックを使用しています。 このコーデックはSkypeから取得され、Skype For Businessに移行されました。 なぜこれが重要なのですか？ いくつかの理由があります。

    • a。 SILKは最高の音質のためにHD品質の伝送をサポートしています。
    • b。 SILKは、公衆インターネットを介して音声情報をオンラインで送信するために開発され、他の多くのコーデックよりもSkypeで優れていることが証明されています。 標準のIPテレフォニーについて話す場合、電話は通常、音声トラフィックが優先され、品質の問題がない別のVLANに配置されるため、これは重要な要素ではありません。 しかし、ユニファイドコミュニケーションの技術では、このタスクはほとんどすべてのデバイスから、どこからでも（インターネット、WiFiネットワーク、ソフトウェアクライアントなどから）発信できるため、ほとんど不可能になります。 これらの瞬間に、既存の環境に適応し、最適な品質を提供できるコーデックを持つことが重要です。
  2. マイクロソフトは、DashBoardなどの非常に強力な品質分析ツールを提供します。これにより、ユニファイドコミュニケーションネットワークで何が起こっているかを正しく分析し、ネットワーク上の問題を時間通りに見つけて修正できます。
  3. マイクロソフトは現在、最もマルチベンダーの企業になろうとしていますが、MS Lync / S4Bクライアント用のプラットフォームを優先していません。 したがって、本格的なクライアントは、PC /タブレット/スマートフォン（Windows / MacOS / Adroid / iOS / Windows Phone）のほとんどすべての一般的なOSで利用できます。 Linuxのみを除き、サポートされていません。
  4. Microsoftには、クライアントとサーバーの両方のかなり強力な対話インターフェイスがあります。 したがって、エンドクライアントは、MS Lyncのほとんどすべてのアプリケーションを個別に追加したり、MS Lyncとシステムを統合したりできます。 さらに、MS Lync / S4Bとの1C統合など、ターンキー統合ソリューションを提供する企業があります。
