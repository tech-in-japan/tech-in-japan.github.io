Web 2.0は携帯電話に適合させる必要があります

実際、 Googleマップマッピングサービス、ソーシャルネットワーク、その他多くのWeb 2.0生成アプリケーションは、携帯電話やPDAでの使用に最適です。 しかし、理論的にのみ。 実際には、すべてがそれほど単純ではありません。 専門家は、携帯電話にWeb 2.0を実装するには別のアプローチが必要であると警告しています。



「PCサービスを単純​​にコピーすることに注意してください」とSymbianのエグゼクティブバイスプレジデントであるロンドンのSymbian Smartphone Showで講演しました。



David Woodは、携帯電話と携帯電話ネットワークには「固有の制限」があると警告しています。その理由は、多くのWeb 2.0サービスがブロードバンド通信チャネルで接続されたパーソナルコンピューターで動作するのと同じ方法で携帯電話で動作できないためです。 Web 2.0プログラムは、携帯電話用に特別に適合させる必要があります。



例は、もともとPC用に設計されたアプリケーションであるGoogleマップです。 このプログラムはAjaxで実行され、クライアントのコンピューターへの予備的な「ダウンロード」方法をサポートして、プログラムでのその後の作業を高速化します。 携帯電話の場合、このような操作アルゴリズムはあまり適していません。トラフィックを最小限に抑えることが望ましいです。 そのため、 Googleは最近、トラフィック消費を削減し、リアルタイムで特別なキロバイトカウンターを備えた特別なモバイルバージョンのGoogleマップを導入しました（ GoogleマップのTreoのビデオ ）。



Web 2.0サービスの他の作成者は、携帯電話所有者の10億番目の軍隊に到達したい場合、ほぼ同じことをしなければなりません。



モバイルプラットフォームに切り替えるためのオプションの1つは、ユニバーサルブラウザーを部分的に放棄することです。これに代わって、電話にインストールされて1つまたは別のWeb 2.0サービスと連携する特別なクライアントソフトウェア、 モバイルウィジェットがリリースされます。 たとえば、無料のShoZuプログラムは、携帯電話ユーザーが写真やビデオをオンラインで投稿するのに役立ちます。 このプログラムを使用して、 Flickr Webサイトの写真にコメントを投稿することもできます。 比較のために、モバイルブラウザーでFlickrサイトを開いた場合、コメントの追加には165秒と71.4 KBのトラフィックがかかり、ShoZuプログラムではそれぞれ16秒と3.25 KBしかかかりません。



もちろん、ユーザーが新しいソフトウェアをダウンロードしてインストールしなければならないことが多いため、クライアントプログラムの使用にはいくつかの欠点があります。 開発者は、通信チャネルのスループットの改善により、特殊なプログラムを放棄できることを望んでいます。



専門家によると、近い将来、 Web 2.0サービスの多くの所有者がモバイルユーザーへの関心を呼び起こし、この方向に大きな進歩が見られると考えています。 おそらく、位置情報などのモバイルデバイスを利用したり、24時間ユーザーのそばにデバイスが常に存在することを活用したりする、新しいWeb 2.0サービスも登場するでしょう。 多分それはすでにWeb 3.0でしょうか？
