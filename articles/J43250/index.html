LJの広告は正直に削除します

LiveJournal自体のスタイルでコードを挿入して広告を削除すると、これはまずSOUPによってすぐに削除される可能性が高く、次に、悪名高い利用規約と矛盾します。



幸いなことに、最新のブラウザでは、ユーザーの側で広告をカットできます。



このコードをテキストドキュメントに挿入します。



div.adv {display: none;}

div.ljad {display: none;}

div.ljadleaderboard-bottom {display: none;}

div.ljadleaderboard {display: none;}

div.ljadwrapper-journal-after-post-c {display: none;}

div.ljadwrapper-app-home {display: none;}

div.ljadmedrect {display: none;}







userContent.cssとして保存します。



Mozilla Firefoxの場合：

ファイルの保存先：

Linux： 〜/ .mozilla / firefox / RANDOM CHARACTER_SET.PROFILE_NAME（デフォルト、「デフォルト」）/ chrome

XP： C：\ Documents and Settings \ USERNAME \ Application Data \ Mozilla \ Firefox \ Profiles \ PROFILE NAME \ chrome

Vista： C：\ Users \ USERNAME \ AppData \ Roaming \ Mozilla \ Firefox \ Profiles \ PROFILE NAME \ chrome

Mac OS X：ライブラリ>アプリケーションサポート> Firefox>プロファイル>プロファイル名> chrome

（アプリケーションデータまたはAppDataフォルダーがない場合：[スタート]> [コントロールパネル]>左側の[クラシック表示に切り替える]を選択し、[フォルダーオプション]> [表示]タブを起動し、[隠しファイルとフォルダー]フォルダーに隠しファイルとフォルダーを表示します）。 ブラウザをリロードします。



Internet Explorerの場合：

ファイルはディスク上の任意の場所に保存する必要があり、IEパスでパスを指定する必要があります：ツールメニュー>インターネットオプションメニュー>一般タブ>アクセシビリティボタン>スタイルシートを使用してドキュメントをフォーマットするためのボックスをオンにします>スタイルシートフィールドにファイルへのパスを入力します。

ロシア語版の場合：[ツール]メニュー> [インターネットオプション]メニュー> [一般]タブ> [外観]ボタン> [外観]チェックボックスで、ユーザースタイルを使用してボックスをオンにし、[スタイル]リストボックスにファイルへのパスを入力します。 ブラウザをリロードします。



Operaの場合：

ファイルはディスク上の任意の場所に保存する必要があり、Opera設定でパスを指定する必要があります：[ツール]メニュー> [設定]> [詳細設定]タブ>左側のコンテンツを選択> [スタイルオプション]ボタン...> [表示]タブ> [マイスタイルシート]フィールドに、ファイルへのパスを入力>タブプレゼンテーションモード> 2つの[マイスタイルシート]チェックボックスをオンにします。

ロシア語版の場合：[ツール]メニュー> [設定]メニュー> [詳細設定]タブ> [コンテンツの選択]ボタンスタイル...ボタン> [表示]タブ> [マイスタイルシート]のファイルへのパスを入力> [プレゼンテーションモード]タブ> [マイスタイルシート]の2つのチェックボックスをオンにします。 ブラウザをリロードします。



Safariの場合：

ファイルはディスク上の任意の場所に保存する必要があり、そのパスはSafariの設定で指定する必要があります：編集メニュー>設定メニュー>詳細タブ>スタイルシートドロップダウンリストでその他...を選択...>ファイルを選択します。

ロシア語版の場合：[編集]メニュー> [設定]メニュー> [詳細設定]タブ> [スタイルテンプレート]ドロップダウンリストで、[その他...]を選択し、ファイルを選択します。 ブラウザをリロードします。



このコードは、プロファイルおよびホームページの広告を含むすべての広告を削除する必要があります。
