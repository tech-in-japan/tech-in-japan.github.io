Open Server-プロフェッショナルなWindows Web開発ツール

Windows用のWeb開発用の新しいプロフェッショナルツールを紹介します。



Open Serverは、多機能管理プログラムと多数の接続コンポーネントを備えたポータブルローカルWAMP / WNMPサーバーです。 提示されたソフトウェアパッケージは、「膝の上」で組み立てられた別のアマチュアアセンブリではなく、推奨事項や希望を考慮してWeb開発者向けに特別に作成された最初の本格的なプロフェッショナルツールです。



まだDenwer、Xampp、Vertrigoなどを使用している場合 または、すべてのサーバーコンポーネントを個別にインストールすることをお勧めします-catへようこそ。







コンポーネントとツール



さまざまな環境でスクリプトをデバッグするために、Open Serverは2種類のHTTPサーバー、さまざまなバージョンのPHPおよびDBMSモジュール、およびそれらをすばやく切り替える機能の選択肢を提供します。



HTTPモジュール： Apache 2.2.21およびNginx 1.0.11。



DBMSモジュール： MySQL 5.1.61、MySQL 5.5.20およびPostgreSQL 9.1.1。



PHPモジュール： PHP 5.2.17（IMagick 2.2.1、Zend Optimizer 3.3.3、IonCube Loader 4.0.7、Memcache 2.2.4）およびPHP 5.3.9（IMagick 2.3.0、Xdebug 2.1.3、IonCube Loader 4.0。 10、Memcache 2.2.6）;



優れたツールキット：HeidiSQL、Adminer、PHPMyAdmin、PHPPgAdmin、PgAdmin。

このパッケージには、Perl、FTPサーバー、Sendmail、Memcachedサーバーなどの機能も含まれています。



Open Serverは、Nginxを含む私が知っている唯一のプロジェクトです！ さらに、ドメインのルートにある.nxaccessファイルを介した書き換えルールの便利な接続が実装され、PHPはTrue FastCGI（unkillable）モードで動作します。



すべてのコンポーネントは公式リポジトリから取得され、パッケージが更新されるたびに常に最新バージョンに更新されます。



特徴



まず、Open Serverは完全かつ完全にポータブルなサーバーです。 システムサービスなし、レジストリおよびsystem32にゴミの山。 フラッシュドライブ（できれば高速ドライブ）でいつでも持ち運べ、何かが動作しないのではないかと心配することなく、作業用/家庭用マシンで実行できます。

必要なシステムコンポーネントがコンピューターで利用できない場合、Open Serverはそれらを単独でインストールします。サーバーが初めてコンピューターで起動される場合は、[ツール-初回実行]メニューで選択します。



プログラムの主な機能については説明しません。ほとんどの場合、この種のソフトウェアの標準機能です。 Open Serverを使用すると、サーバーを起動/停止したり、目的のドメインを開いたりできることは言うまでもありません。 Open Serverを特別な、本当に特別なものにする特定の「機能」について学ぶことは、はるかに興味深いでしょう。



-すべてのコンポーネントのログをリアルタイムで詳細に表示。

-HTTP、DBMS、およびPHPモジュールの任意の組み合わせの選択。

-すぐに使用可能なSSLおよびキリル文字ドメインのサポート。

-別の方法でのエイリアスまたはドメイン名のサポート、およびそれらをセットアップするための便利なフォーム（Drupalのマルチサイトのファンにこんにちは！）;

-インターネット上のメインドメインの可視性を失うことなく、ローカルサブドメインを作成します。

-ドメインへのアクセス（ワンクリックで）およびモジュールの構成テンプレートへのクイックアクセス。

-多言語インターフェース（ロシア語、ウクライナ語、ベラルーシ語、英語）;



プログラムは絶えず改善されており、Open Serverユーザーからの適切なリクエストはすべて詳細に調査され、それらのほとんどが実装されています！



ちょっとした歴史



最初は、DevelStudio環境で制御プログラムの開発を主導しました。 この開発環境についての私の記事を読んでくれる人がいると思います。 それ以来、たくさんの水が流れました。このプロジェクトの作業は非常に遅く、Open Serverの機能を開発するという見通しはありませんでした。 私は最初からプログラムを書き直し、バージョン4.0以降、Delphiで開発が行われています。



おわりに



長年にわたって日々の仕事でデンバーを使用し、しばらくしてからVertrigoに切り替えました。 XAMPP、AppServ、WAMPserver、TopServerなど、利用可能な他のほとんどすべてのオプションを試しましたが、上記のアセンブリはどれも私を満足させませんでした。 限られた機能、カスタマイズの難しさ、時には単純な外部のさから、これらの「作成物」を使用することはできませんでした。



これで、Open ServerをすべてのWeb開発者に、例外なくすべてを手動でインストールすることに慣れている保守派にすら、お勧めできます。
