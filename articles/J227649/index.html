Gmail API

Googleは、サードパーティアプリケーション用にGmailの受信トレイのコンテンツを開くことにし、 Gmail APIを導入しました。 Googleによると、これらはIMAPよりも便利で安全です。



Gmail APIは、 Threads



、 Messages



、 Labels



、 Drafts



、 History



などの特定のリソースへのアクセスを提供します。 たとえば、サードパーティのアプリケーションは、手紙の送信のみ（読み取りは不可）、読み取りのみ（送信は不可）、手紙と会話のラベルの変更のみ、または特定の手紙と会話の検索のみへのアクセスを要求できます。



Gmailは、世界で最も人気のあるメールサービスの1つです。 正確なユーザー数は不明ですが、2012年にGoogleはオーディエンスを4億2,500万人と推定しました。一方、Gmailは世界の100の大学のうち66校と500万の商業会社のメールプロバイダーです。



これは、GmailがおそらくFacebookに続くアクティブユーザーの数で2番目のプラットフォームであることを意味し、間違いなくアプリケーション開発者を引き付けるはずです。 もちろん、メールサービスに基づいてアプリケーションを作成するのは奇妙に思えますが、Googleは実験を望んでいます。



メールボックスをスキャンすると、さまざまなアプリケーションでメリットが得られます。 たとえば、どのプログラムでも、ユーザーに確認リンクをクリックさせることなく、電子メールを自動的に確認できます。 対応するテキストの外観については、Inboxフォルダーをスキャンするだけで十分です。 または、たとえば、原価計算プログラムは、領収書をInboxフォルダから自動的にインポートして、経費の項目とオンラインストアで費やした金額を示すことができます。 または、指定された条件に従って、プログラムはユーザーに代わって手紙を送ることができます。



さらに、Gmail APIを使用して、標準のGmail Webインターフェイス用のさまざまな拡張機能を作成できます。 たとえば、受信ボックスフォルダーを30〜60分間「フリーズ」するボタン、または気を散らされたくない日または週。 ここで多くのアイデアを思いつくことができます。 あなたを含めると、あなた自身のブランドを宣伝する拡張機能を作ることができます。 たとえば、事前定義された内容の手紙を受け取ったときにプログラムされたアクションを実行する拡張機能です。



開発者には新しい機会が開かれていますが、ユーザーは、個人情報にアクセスするためにサードパーティのアプリケーションに許可を発行することについて、より注意する必要があります。 ただし、この点で、Gmail APIはサードパーティプログラムの機能を必要最小限に制限できるため、標準のIMAPより安全です。
