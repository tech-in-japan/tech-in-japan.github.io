「ブロックチェーン」の概念に正確に含まれるもの

この記事の著者は、Kaspersky Labのウイルス対策技術開発部門の専門家であるAlexei Malanovです。



ブロックチェーンが何を意味するのかという質問は、アイドル状態ではありません。 ブロックチェーンの専門家でさえ、テクノロジーから遠く離れた人々は言うまでもなく、テクノロジーの異なる定義と異なる境界を提供します。 そして今、ブロックチェーンは政治家や大規模な投資家の間でも議論されていますが、彼らにはどう感じますか？



正しくしましょう。



すぐに警告しますが、この記事は哲学的で、長い論法とデマゴジーを備えています。 詳細が必要な場合は、 別の記事をお読みください。 それを理解したい-3 番目を読んでください。





メラニー・スワンの本のイラスト「 ブロックチェーン。 ニューエコノミーのスキーム ”



マキシマリストの観点から見たブロックチェーン



簡単な観点から始めましょう。 ブロックチェーンは、暗号通貨で使用される、または使用されるすべてのテクノロジーです 。 まだ発明されておらず実装されていないものを含む。



暗号通貨の概念に含まれるものは、背景に消えていきます。 気が散らないように、簡単にするために、暗号通貨はhttps://coinmarketcap.com/currencies/のリストにある通貨であると想定します 。





大文字による上位暗号通貨（ ソース ）



一方で、このような定義は、任意のステートメントを作成できるため便利です。 例：



  • ブロックチェーン：ロシアの新油。
  • 私の意見では、ブロックチェーン技術は新しいインターネットです。 これは、インターネットと同じレベルのアイデアです。
  • それはともかく、ブロックチェーンエコシステムはさまざまな分野で多くの新しい機会を切り開き、分散化の概念を使用して遺伝子配列の大規模な計算を可能にします。
  • ブロックチェーン内の個人の医療データの集約、つまり外部データへのポインターのブロックのチェーンに格納すると、医学研究全体がより効率的になります。
  • そして、他の多くの。


結局のところ、50年後には現金が流通から引き出され、デジタルマネーで支払うことになると主張するのは困難です。 確かに、現在使用されているか、暗号通貨で使用されるいくつかの技術は、このデジタルマネーで役割を果たします。 だから、ブロックチェーンのお金が普通のお金に取って代わることがわかっていますよね？



ブロックチェーンはより良い遺伝学的および医学的研究を可能にしますか？ まあ、それは確かに干渉しません。



一方、そのような最大主義の定義により、ブロックチェーンについて多くを語ることができますが、実質的な会話を行い、特定の何かを肯定することはできません 。 例を見てください：



  • ブロックチェーンソリューションはスケーリングしません。
  • ブロックチェーンソリューションは、集中型ピアよりも低速です。
  • ブロックチェーンソリューションは、参加者間の信頼を必要としません。
  • ブロックチェーンソリューションはオープンで検証済みです。
  • ブロックチェーンはブロックのチェーンです。


それらはすべて偽りになります。 結局のところ、彼らが何を思い付くのか決してわかりませんか？ たとえば、IOTA暗号通貨では、 方向性の非循環グラフが既に発明されており、トランザクションを整理するために使用されています-ブロックのチェーンはありません。 または、たとえば、いつかPlasmaがおそらく動作し、ブロックチェーンがスケーラブルになります。



ミニマリストの観点からのブロックチェーン



同様に単純な視点で続けます。 ブロックチェーンは、単なる増分書き込みデータベースです。 それだけです 末尾に追加するか、末尾からスローして再度書き込むことができる場合は、ブロックチェーンがあります。 論理的にはいつでも捨てることができます。実際のブロックチェーンソリューションは、例外的な場合（たとえば、 920億ビットコインの場合 ）にのみこの機会を使用しようとします。 配布も、コンセンサスも、ブロックも、以前のブロックへの参照もありません。何も必要ありません。



私が知っているすべての暗号通貨がそのようなブロックチェーンで動作するため、この定義は便利です。



しかし、そのような定義は以前の定義よりもさらに不合理で役に立たない。 さらに、このような「ブロックチェーン」は約100年前から知られており、スタックと呼ばれます。



基本装備



退化した定義で終わり、多くの専門家がブロックチェーンと呼ぶことができる最小限の合理的なパッケージを作ります。



ブロックチェーンは次のことを想定しています：



1.データは、これらのブロックが何であれ、ブロックで装飾されます。

2.ブロックは、ジェネシスブロックであるか、前のブロックを指します。

a。 最後にブロックが追加されます。

b。 通常の操作中にブロックを捨てることは禁止されています。

3.ブロックは通信環境で独立して飛ぶ-任意のブロックをダウンロードできます。

4.ブロックチェーンは、少なくとも2人の参加者によって読み取られます。

5.少なくとも1人の参加者がブロックチェーンに書き込みます。

6.すべての作家が合意に達する-ブロックは単一のルールセットに対応します。

a。 明らかな結果は、コンセンサスに達した後、ブロックチェーンがすべての参加者で同じになることです。

b。 ネットワーク上にソフトフォークが存在する場合、「統一」ルールは明らかにより厳格です。





ソフトフォークの概念に関するベビーベッド。 出所



追加するものがありますか？



次に、そのような定義の背後にあるものを見てみましょう。



  1. 「ブロックチェーン」の概念には、コンセンサス構築の原則が含まれていません。つまり、 Proof-of-Work、Proof-of-Stake、Proof-of-Authorityなどです。 しかし、Proof-of-Workでは、ブロックチェーンの創設者であるビットコインの革新の最大のシェアは正確にあります。
  2. 「ブロックチェーン」の概念には、分散化の原則は含まれません 。 縮退した場合、サーバーが存在することが判明し、データベースに書き込みます。 残りの人は読んで確認することも、確認しないこともあります。
  3. 「ブロックチェーン」の概念には、オープン性の原則は含まれていません。広い範囲の人々、特に録音のために読むためにアクセスできる必要はありません。
  4. 「ブロックチェーン」の概念には、信頼の分配の原則は含まれません 。参加者は排他的に信頼できます。
  5. 「ブロックチェーン」の概念には、悪意のある参加者からの保護は含まれていません 。存在しない場合があります。
  6. 「ブロックチェーン」の概念には暗号化は含まれません 。 それなしで実行できるとは考えられませんが、どの段階でどの形式で使用されるかは定義されていません。


私たちの定義から何が続きますか？ したがって、ブロックチェーンソリューションは、集中化され、効率的で、さらにはスケーラブルです。 結局、私たちはすべての「過剰」を捨てました：配布、不信、コンセンサスの遅さ-そして私たちはキャンディを手に入れました。それはまだブロックチェーンです。



ちなみに、そのような甘いものはプライベートブロックチェーン （プライベート/許可）と呼ばれ、一部の企業（さらには政府機関も）が現在賭けているブロックチェーン上にあります。 これは、有名な分散レジストリ （ 分散データベースと混同しないでください）の特殊なケースですが、すべての分散レジストリが明らかにブロックチェーンであるとは限りません。



高度な機器-「本当の」ブロックチェーン



しかし、ブロックチェーンが一般的な利点（および同時に欠点）に起因するためには、新しい条件を定義に追加する必要があります。



まず、「実際の」ブロックチェーンのプロパティを定義します。



  • 分散型 。 これにより、非ブロック性/安定性が得られますが、同時に効率（鉄の単位あたりのスループット）とスケーラビリティ（鉄の量の増加に伴うスループットの増加）がなくなります。
  • 読み取り用に開きます。 これにより有効性が得られますが、同時にプライバシーが失われます（上から順にねじ込むことができます）。
  • 録音用に開きます。 これにより、アクセシビリティが提供されますが、記録も有料になります。 原則として、ボードはシンボリックですが、ライターのリストを大幅に制限するにはこれで十分です。
  • 不変です。 これは、改ざんに対する保護を提供しますが、ゴミにもなります（公共のブロックチェーンには書き込みません）。
  • 通貨で 。 この楽しい特性は、ブロックチェーンソリューションで特に高く評価されています。 それは財政的な機会を提供しますが、代わりにプロジェクトの法的地位に不確実性をもたらします。 同時に、サイバーセキュリティリスクも生じます。
  • 信頼できる 。 参加者の評判を必要としません。


ブロックチェーンソリューションが通貨を使用する必要があるという事実は、もちろんオプションのプロパティです。 しかし、通貨がなければ、参加者にネットワークを提供するように動機付けることはそれほど容易ではありません。つまり、「本当の」ブロックチェーンの不可欠な機能である分散を確保することはそれほど容易ではありません。



説明されているプロパティを取得するために、「実際の」ブロックチェーンに以下を追加します。



  1. 最初のブロックを除く各ブロックには、前のブロックの暗号化ハッシュが含まれており、基本構成のように参照されるだけではありません。
  2. 参加者間のコンセンサスは、少数の人、通常はProof-of-WorkまたはProof-of-Stakeのバリエーションによってデータベースを書き換えるリスクを最小限に抑えるような方法で達成されます。 私の意見では、これが最も重要です。 少数派の概念は、選択した方法によって決まります。
  3. ピアツーピアネットワークを使用するか、全員が正式にマスターノードになる権利を持っています。
  4. ネットワークの円滑な運用（正しいブロックの形成）を確保するための参加には 、何らかの形で報酬があります。 したがって、おそらく、すべてのオープンブロックチェーンには独自の「コイン」があります。
  5. 各参加者はキーのペアを持ち 、何らかの形でブロックチェーンに書き込むことで自分の意図を確認します。


いくつかのプロパティ（要件ではありません）：



  1. データベースの正確性を検証しても、 報酬 はあり ません 。 参加者は、自分の安全のためにこれを行います。
  2. 各参加者は、ネットワークのルールを満たすトランザクションをネットワークに送信し、合理的な支払いで合理的な時間内にトランザクションがブロックチェーンに含まれることを計算する権利を有します。


セクションの冒頭で指定したプロパティを持つブロックチェーンは、多くの場合、 オープンブロックチェーン （open / public / permissionless）と呼ばれます。 暗号通貨を超えて著者にそのようなブロックチェーンを使用する見込みは、現在一般に信じられているよりもはるかに控えめに思えます。 集中化することを妨げるものが何もない場合は、分散化された何かを行う正当な理由がなければなりません。 しかし、これは著者の個人的な意見にすぎません。



追加オプション



このような便利な概念もあります-「ブロックチェーン技術」。 これらは、ブロックチェーンではありませんが、それに直接関連するテクノロジーです-アドオンまたは通常のブロックチェーン（その定義のいずれか）への改良のいずれかです。 それらの半分はブロックチェーンなしで使用できますが、そのような必要性は生じません。



たとえば、 Lightning Networkはおなじみのクリアリングですが、すでに展開されているブロックチェーンを通じて得られた信頼の上にのみ構築されています。



別の例はミキサーです。 きれいなお金と混ぜて汚れたお金を洗うことは長い間知られていましたが、暗号通貨に関しては変容し、異なる実装オプションを持つ独立したテクノロジーになりました。



リングシグニチャまたはディスクロージャーのない証拠は、暗号通貨分野で適用されると「ブロックチェーンテクノロジー」になる有名な暗号プロトコルです。



「ブロックチェーン技術」の概念は、最初のセクションでブロックチェーンを定義するための最大限のアプローチと本質的に同じことを意味します。 しかし、私だけがこのコンセプトが好きです-より正確です。 ブロックチェーン技術が言及されている場合、これはブロックチェーンがあるという意味ではありません。



おわりに



「ブロックチェーン」の概念に含まれているものと含まれていないものを判別しようとしました。 不合理なものから非常に適切なものまで、いくつかのオプションを検討しました。 テクノロジーのさまざまな境界から生じる、または逆にこれらの境界を定義するプロパティについて説明しました。

残っている唯一のことは、各特定のユースケースで何が意味されているかを単純に明確に理解することです。
