ReactNativeで簡単な歩数計アプリを開発する



今日、プログラマーの輪の中で、ほとんど誰もがFacebookライブラリ-Reactについて知っています。







Reactはコンポーネントに基づいています。 これらはブラウザのDOM要素に似ており、HTMLではなくJavaScriptを使用して記述されています。 Facebookによると、コンポーネントを使用すると、インターフェースを1回作成してすべてのデバイスに表示できます。 ブラウザではすべてが明確になっています（これらのコンポーネントはDOM要素に変換されます）が、モバイルアプリケーションについてはどうでしょうか。 ここでも予測可能です。Reactコンポーネントはネイティブコンポーネントに変換されます。







この記事では、簡単な歩数計アプリケーションの開発方法を説明します。 ハイライトを示すコードが表示されます。 プロジェクト全体はGitHubへのリンクから入手できます 。







それでは始めましょう。







必要条件


iOS開発には、Xcodeを使用したOS Xが必要です。 Androidを使用すると、すべてがシンプルになります。Linux、OS X、Windowsから選択できます。 また、Android SDKをインストールする必要があります。 戦闘テストには、iPhoneとLollipopを搭載したAndroidスマートフォンが必要です。







プロジェクト構造の作成


最初に、プロジェクト構造を作成します。 アプリケーション内のデータを操作するために、フラックスの考え方、つまり実装としてReduxを使用します。 ルーターも必要です。 すぐにReduxをサポートするため、react-native-router-fluxをルーターとして選択しました。







Reduxについて一言。 Reduxは、アプリケーションの状態を保存するシンプルなライブラリです。 レンダリングレンダリングなど、状態の変化にイベントハンドラーをアタッチできます。 ビデオチュートリアルでreduxを理解することをお勧めします。







実装に取り​​かかりましょう。 npmを使用してreact-native-cliをインストールします。npmを使用して、プロジェクトでのすべての操作を引き続き実行します。 







npm install -g react-native-cli


次に、プロジェクトを作成します。 







 react-native init AwesomeProject


依存関係をインストールします。 







 npm install


その結果、iosおよびandroidフォルダーはプロジェクトのルートに作成され、それぞれのプラットフォームに「ネイティブ」ファイルがあります。 index.ios.jsおよびindex.android.jsファイルは、アプリケーションのエントリポイントです。

ディレクトリ構造を作成します。

アクションフォルダーには、ストア内のデータに何が起こるかを記述する関数が含まれます。

app / containersフォルダーにapp.jsを作成します。 アプリケーションのルート要素はreduxラッパーです。 すべてのルートは通常のコンポーネントとして登録されます。 初期プロパティは、アプリケーションの初期化時に機能するルートをルーターに指示します。 ルートのコンポーネントプロパティでは、ルートに切り替えるときに表示されるコンポーネントを渡します。

アクション-各ルートにメソッドがあるオブジェクト。 そのようなメソッドの名前は、ルートのnameプロパティから取得されます。

ファイルapp / constants / actionTypes.jsで、可能なカウンターイベントについて説明します。 







 export const INCREMENT = 'INCREMENT'; export const DECREMENT = 'DECREMENT';


app / actionsフォルダーで、内容を含むcounterActions.jsファイルを作成します。 







 app/actions/counterActions.js import * as types from '../constants/actionTypes'; export function increment() { return { type: types.INCREMENT }; } export function decrement() { return { type: types.DECREMENT }; }


増分および減分関数は、レデューサーに何が起こるかを記述します。 アクションに応じて、リデューサーはアプリケーションの状態を変更します。 initialState-リポジトリの初期状態を説明します。 アプリケーションが初期化されると、カウンターは0に設定されます。 







 app/reducers/counter.js import * as types from '../constants/actionTypes'; const initialState = { count: 0 }; export default function counter(state = initialState, action = {}) { switch (action.type) { case types.INCREMENT: return { ...state, count: state.count + 1 }; case types.DECREMENT: return { ...state, count: state.count - 1 }; default: return state; } }


counter.jsファイルには、カウンター値を増減するための2つのボタンが含まれており、現在の値も表示されます。 







 app/components/counter.js const { counter, increment, decrement } = this.props; … <Text>{counter}</Text> <TouchableOpacity onPress={increment} style={styles.button}> <Text>up</Text> </TouchableOpacity> <TouchableOpacity onPress={decrement} style={styles.button}> <Text>down</Text> </TouchableOpacity>


イベントハンドラとカウンタ値自体は、コンテナコンポーネントから渡されます。 以下を検討してください。

チャート


歩数計アプリケーションを開発しているため、それに応じて、測定結果を表示する必要があります。 最良の方法は、私には思えるが、チャートです。 したがって、単純な棒グラフを作成します。Y軸はステップ数を示し、Xは時間を示します。







ReactNativeはそのままではcanvasをサポートしていません。さらに、canvasを使用するにはwebviewが必要です。 したがって、2つのオプションがあります。各プラットフォームのネイティブコンポーネントを記述するか、コンポーネントの標準セットを使用します。 最初のオプションは最も労働集約的ですが、その結果、生産的で柔軟なソリューションが得られます。 2番目のオプションについて説明します。

歩数計


ReactNativeは、ネットワークからデータを取得して表示する単純なアプリケーションを作成するための基本的なツールセットのみを備えたかなり若いプロジェクトです。 ただし、タスクがデバイス自体でデータを生成することである場合、プラットフォーム固有の言語でモジュールを作成する必要があります。







この段階では、歩数計を作成する必要があります。 Objective-CとJava、およびデバイスAPIを知らなければ、これを行うことは困難ですが、可能です。すべては時間に依存します。 幸いなことに、Apache CordovaやAdobe PhoneGapなどのプロジェクトがあります。 彼らはかなり前から市場に出ており、コミュニティは彼らのために多くのモジュールを書いてきました。 これらのモジュールは簡単に移植して対応できます。 すべてのロジックは変更されず、インターフェイス（ブリッジ）を書き換えるだけで済みます。







IOSには、アクティビティデータを取得するための優れたAPI（HealthKit）があります。 Appleには優れたドキュメントがあり、通常の単純なタスクの実装も含まれています。 Androidは別の状況です。 私たちが持っているのはセンサーのセットだけです。 さらに、ドキュメントには、API 19以降、ステップセンサーデータを取得できることが記載されています。 膨大な数のデバイスがAndroidで動作し、真の中国メーカー（有名ブランドを含む）だけでなく、主要なセンサーセット（加速度計、周囲光センサー、近接センサー）のみをインストールしています。 したがって、Android 4.4以降とステップセンサーを備えたデバイス（および古いデバイス）については、コードを個別に記述する必要があります。 これにより、測定の精度が向上します。

最後に、ReactNativeの利点と欠点を強調したいと思います。







利点：


  1. JavaScriptの経験がある開発者は、簡単にアプリケーションを作成できます。
  2. 1つのアプリケーションを開発すると、すぐにAndroidとIOSで実行する機会が得られます。
  3. ReactNativeには、多くの場合、すべての要件をカバーする実装済みコンポーネントのかなり大きなセットがあります。
  4. さまざまなモジュールを迅速に作成しているアクティブなコミュニティ。


短所：


  1. 同じコードが両方のプラットフォームで常にスムーズに機能するとは限りません（多くの場合、表示の問題）
  2. 特定のタスクのために、多くの場合、実装されたモジュールがなく、自分でそれらを記述する必要があります。
  3. パフォーマンス。 PhoneGapやCordovaと比較すると、反応は非常に高速ですが、ネイティブアプリケーションの方が高速です。


ReactNativeを選択するのが適切なのはいつですか？


サーバーからデータを受信して​​表示する単純なアプリケーションを開発する必要がある場合、選択は明らかです。 クールなデザインを実装するタスクに直面している場合、パフォーマンスが重要であるか、市販のコンポーネントを使用して解決するのが難しいタスクがある場合は、検討する価値があります。 ほとんどはプラットフォームのネイティブ言語で作成する必要があるため、これからピラミッドを構築することは間違いなく最良の選択肢ではありません。
