投稿では、次の2つのことについてお話したいと思います。

1）問題への解決策「入り口の部屋の上部の照明のスイッチが、ベッドサイドも欲しい」

2）照明の明るさを調整したい

そして、それは奇妙なことではありません。これらの問題は両方とも 一挙に解決できます。



そして、私は自動化（スマート）ホームのタイプを分類しようとします



叙情的な余談：私は、多くの読者のように、「 スマートホームになれなかったら」と思ったかもしれません。テクノロジー（z-wave、zigbee、homekit、それらの何千も！）を見たかもしれません。その非常に「スマートホーム」のインテリジェンスが不十分なための問題および/または正しい解決策



分類



さまざまなシステムで多数のマニュアルを読んだ後、次のようなシステムのグラデーションが得られました。

1）自動化

2）自動

3）インテリジェント

4）プロアクティブ



最後の2つのシステムのみが多かれ少なかれスマートであり、完全に「スマートホーム」は最後だけと呼ぶことができますが、今日、私が約束したように、スマートホームについては話しません
自動化 -ライトスイッチをリモートコントロールリレーに変更し、1つの一般的なスイッチからライトを制御する機会を与えます-おめでとう、プロセスを自動化しました（ただし、これはスマートホームではありません）



自動 -すでにどこにでもリモートスイッチがあり、友人がトイレに向かっている場合、リモート/アプリを使用してリモートでライトを点灯できますが、何かが欠落しています。 そして、センサーシステムに追加します。 これで、システム自体があなたがこの部屋にいると判断できるようになり、ここでライトをオンにすることをお勧めします- 自動システムを作成したことを祝福します（ただし、これはまだスマートハウスではありません）



インテリジェント -夜間にトイレで必要な場合は、付属のライトで迎えられます。これは、あなたが男性であり、ドアを閉められない可能性があるというデータを考慮に入れていますが、これは光で満足しない理由ではありません。 （システムは、現時点ではあなたが一人でいるのを知っています。つまり、トイレに行くことができるのはあなただけなので、ドアセンサーを無視してライトをオンにする必要があります）- これはインテリジェントシステムですさらに、インテリジェントシステムと通常のシステムの重要な違いの1つはエイリアスのサポートです。 システムは、単純なコマンドではなく、音声コマンドを処理できる必要がありますが、要求のコンテキストを理解する必要があります。 これはあなたの「スマートホーム」にありませんか？ 悪い知らせがあります。



プロアクティブ -家の主要なシステムにアクセスできる単一のハブがあります：光、水、エアコン、暖房。 （家について話す場合：ゲートなど...）すべてのシステムをリモートで、音声およびインターネット経由で制御でき、「I'm going home」ボタンを押すこともできます。システムはすべてを調整します（たとえば、エアコンをオンにするか、地理位置は近くになります）。

スマートホームのように聞こえますか？ いいえ次のシナリオを処理できる場合、システムはプロアクティブになります。

1）簡単なオプション：自宅に到着するシナリオがあり、システム自体が地理位置情報をチェックし、移動ベクトルに基づいて、家に帰るかどうか（過去のデータに基づいている場合）を尋ねることができます（音声、メッセンジャーのテキスト）ルート同様）スクリプトを実行します。 しかし、スクリプトの実行中に突然ルートを変更した場合、システムはこれを理解してスクリプトを停止する必要があります。



2）複雑：システムはスクリプトの手動入力を必要とせず、人間の行動を分析することでそれ自体をコンパイルします。 彼女は、朝は明るい光を好み、夜はくぐもった光を好むことを知っています。 彼女は、寝る前に暖房を少し弱め、可能であれば通気する必要があることを知っていますが 、あなたの女の子は下書きに耐えられません。つまり、女の子が家にいないか、別の部屋/シャワーにいる間に通気する必要があります。





特定の問題



修理中にすべてを計画することは不可能でしたが、私と同じことが判明しました-私はベッドが部屋のどこにあるのか絶対に知りませんでしたので、スイッチの場所を選択できませんでした

（はい、私のベッドはすでに部屋を動き回っていました...）したがって、多くの人々と同様に、部屋の入り口、ドアの隣に、1つのオーバーヘッドライトスイッチがあります。



しかし、時間が経つにつれて、 怠lazが問題を打ち負かしました。ベッドに横たわっていると、電話のオーバーヘッドライト、テーブルランプ、懐中電灯をオフまたはオンにすることは不可能です。 また、夕方には照明を暗くしたいのですが、修理中に調光器が取り付けられておらず、標準の調光器は快適ではありません。





標準調光器



最後に、問題を解決するための要件を次のように策定しました。

部屋の上部の照明をベッドから調節して、明るさを調整し、ホームオートメーションの道を歩き続けたいと思います。



いくつかのシステムオプションを見て、NooLiteが次のように私に合っているという結論に達しました。

1）リレー

2）2つのスイッチ



彼らは私のウィッシュリスト用の既製のキットさえ持っています






設置



そのため、古いスイッチを解体し、ワイヤーを接続し、リレーを開回路に接続し、スイッチを設置する必要がありました。



リレーの設置場所を選択する過程で、シャンデリアに完全に直接収まるだけでなく、さらに冷却されるという結論に達しました。



シャンデリア



次のシャンデリアがあります。





ガラスドームを取り外し、電気をオフにすることを忘れないでください。また、連絡先へのアクセスを閉じるプラットフォームを取り外します。











リレーを見てみましょう。





中央のハードウェアモードスイッチである黒いジャンパー線に注意してください。輝度調整（ディマー）形式で作業する場合は、噛み付きます。



シャンデリアの端子台から天井から元のワイヤーを外し、そこに短い長さの新しいワイヤーを挿入します。







端子台を用意し、電源220をリレーと負荷に接続します（ワイヤを混ぜないでください）









ワイヤが熱くならないように、これらすべてをコンパクトに梱包します。





この段階が完了したら、電源を入れてシステムを確認できます-接続されたリモコンからすでに点灯しているはずです（最初に古いリモコンのライトをオンにすることを忘れないでください）



スイッチ



与えられた：ここにスイッチがあります：





解体する必要があります。



詳細に
ボタンを外し、中央の2本のネジを外し、プラスチックフレームを取り外します





側面のネジを外します（ただし、完全ではありません）





スイッチを取り外し、ワイヤーを外します。

テストの後にライトを消すのを忘れていませんでしたか？





端子台の配線の準備





端子台に乗せます





リモコン



リモコン





上部（または設定方法によっては下部）にコマンド送信の緑色のインジケータがあり、押すとウィンクします。





裏面





ベースを取り外します





コンソールの内部世界





フレームを古いものと同じネジで固定します





2番目のリモートをバインドする







アルゴリズム：

1）すでに接続されているリモコンのサービスボタンを押します

2）既に接続されているリモコンのライトをオンにするをクリックします



ライトが点滅します



2番目のリモコンでも同じ操作を行います

3）新しいリモコンのサービスボタンを押します

4）新しいリモコンのライトをオンにするを押します



ライトがより頻繁に点滅します

最後の2つの手順をもう一度繰り返します。



5）すでに接続されているリモコンのサービスボタンを押します

6）既に接続されているリモコンのライトをオンにするをクリックします



ライトが消灯-リモコンを縛りました



結果



システムの最終組み立て後、シャンデリアではワイヤとモジュール自体は見えません（ライトもオンになっています） 。





明るさは次のように制御できます。スイッチの上端をクリックして、通常どおりシャンデリアをオンにします。 そして今、下をクリックして押したままにします-ライトが減少します。 （同じ方法で追加できます）







休息の過程で、リレーはかすかな緑色の光で点灯します（下の写真よりもさらに弱く、電話は必要以上に明るくなりました）





NooLiteの開発状況



NooLiteは一般的によくやった-リレーからのフィードバックがないことをscった投稿の前に、彼らはこれを行うことができる新しいリレーをリリースする過程にある。 古いモジュールを置き換えるために最近リリースされたこの新しいモジュールのヒント。



説明
MTRF-32トランシーバーモジュールは、Arduinoプラットフォーム、マイクロコントローラー、またはPCのnooLiteおよびnooLite-Fシステムと対話するように設計されています。



構造的には、モジュールは、制御コントローラーとトランシーバーが配置されたプリント回路基板の形で作られています。



モジュールは、シリアルUARTインターフェイスを介して制御されます。

モジュールで低レベルの通信インターフェイスを使用すると、独自の設計で使用できます。



（私は6か月後にraspberri piおよびesp8266 / arduinoと組み合わせて使用​​することに関するマニュアルがあると確信しています）。
