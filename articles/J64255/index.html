BlogEngine.NET 1.5。 テーマと拡張機能を備えたRussifiedアセンブリ

ブログエンジンBlogEngine.NET（バージョン1.5.0.7）のディストリビューションを自分で構築し、数秒でホスティングにデプロイし、必要に応じてRussifiedに準備しました（スラッグから、拡張機能を含む特別なファイルとコード自体の翻訳の多くの改善まで）、さまざまな拡張機能とトピックの大きなパッケージを使用しますが、彼は他の誰かのために役立つかもしれないと思いました。

さらに：BlogEngine.NETが何であり、何を食べているかを知らない人にとっては、おそらくそれについての物語は興味深いでしょう。「主題」にいる人にとっては、エンジンで行われた変更のリストです。



BlogEngine.NETとは何ですか？



BlogEngine.NETは、ASP.NETテクノロジーを使用して記述された軽量で機能的なオープンソースのブログエンジンです。 データベース（Microsoft SQL、Microsoft SQL Express、MySQL）とストレージとしてのXMLファイル（デフォルト）の両方で機能します。 通常のブログサイズでは、パフォーマンスの違いはありません。

標準のテーマを備えたすぐに使えるBlogEngineの例は、 エンジン開発者ブログです。



BlogEngineではないものは何ですか？





BlogEngineはWordpressではありません。 繰り返しますが、これはWordpressではありません。 もちろん、比較することを強要します（一般に、2つの異なるプラットフォームで最高のブログエンジン）。 ただし、BlogEngineは別のクラスのエンジンです。既製のプラグインではそれほど拡張性がありません（ただし、独自のプラグインをすばやく簡単に作成できます）が、できるだけ軽くて高速であり、通常のブロガーが必要とするよりも多くの機能を提供します（彼のブログで遊ぶのではなく、ただそれを書くだけです） 。

関数が必要な場合は使用しないでください。必要なUserControlと拡張機能を作成してASP.NETを本当に学習したい場合を除きます。 ただし、軽量でデプロイが簡単で、簡単にカスタマイズでき、エンドユーザーにとって便利なものが必要な場合（BlogEngineコントロールパネルはWordpressパネルよりもはるかにシンプルで簡単です）-BlogEngineを試してみるのは理にかなっています。 開発者のFAQには、「私の母はそれを使用できますか？」という質問があります:)答えは「はい」です。 ユーザーにとっても開発者にとっても、すべてが非常にシンプルで直感的です。 開発者にとっては、一般に、Visual Studioの公式Webサイトに投稿されたプロジェクトをダウンロードし、美しく書かれたコードを操作する便利さを楽しむだけで十分です。



彼は何を箱から出して何ができますか？

  1. キャッシング（このユーザーを煩わせることなく）、モバイルデバイス（デフォルトでは特別なテーマが与えられます）、トラックバック、ピンバック、さらにはOpenSearch（Yandex、Google、Bingなどに加えて、最新のブラウザーの検索バーにブログ検索を挿入できます） d。）
  2. ツリーコメント、グラバター、メールに関するコメントの通知、pingサービス
  3. 複数著者のブログ、詳細な著者プロファイル
  4. タグクラウド、ツイッター、お気に入りのブログ、メールによる新しい投稿の通知、ドラッグアンドドロップ用のウィジェットはこれらのウィジェットで動作します
  5. アップロードされた画像の自動サイズ変更
  6. コメントでbbcodeを使用して、コメントをプレビューする
  7. フラッシュMP3オーディオプレーヤー
  8. そして、はるかに！
そして最も重要なこと-これはすべて管理が容易であり、ブログの機能はまったく遅くなりません。 即時インストールと最小システム要件も重要な資産です。 インストーラーも不要です。以下を参照してください:)





Windowsホスティングは高価です!!



種類はありません。多くの真面目な企業は、UnixとWindowsのホスティングを同じ価格で提供しています。 さらに、Windowsホスティングには通常、テスト期間があります。 個人的には、 InfoboxとAgaveからのWindowsホスティングに満足していますが、まったく主張していません。 さらに、BlogEngine.NETホスティングには、Wordpressよりもはるかに強力なホスティングが必要です。



誰がこれを必要としますか？



ASP.NETプラットフォームのみに関心がある場合、これは開始に適したコンストラクターです。 BlogEngineのコードは美しく簡単に書かれており、.NETテクノロジーの分野で特別なトレーニングがなくても十分に理解できます。 ASP.NETを徐々に理解しながら、問題なく自由に掘り下げて修正できます。 ASP.NETを使用したことがないが、試してみたい場合-独自のCMSをゼロから作成せずに遊んでみてください-BlogEngineを試してみてください。後悔することはありません。 それに基づいて独自のCMSを作成することは非常に可能です。すべてが通常の非ブログCMSの準備ができています（NBは完了したわけではありません;-)たとえば、コントロールパネルでページのツリーベースの編成がサポートされていますが、実際のサイトでは個別にエンコードする必要があります）。



設置



BlogEngineはデータベースなしで動作しますが（接続することは可能ですが）、インストールするためにホスティングにアップロードし、App_Dataフォルダーの書き込み権限を指定するだけです。 次に、メインページをロードします。 初めてロードされる時間が長くなりますが、その後はすべてがすばやく発生します。



このアセンブリで何が変更されましたか？

  1. ファイル\ App_GlobalResources \ labels.ru.resxの公式翻訳は、真剣に補足および修正されています
  2. 可能な場合-名前はコード内で直接翻訳されます（すべてが個別のファイルに取り出されるわけではなく、エンジンに多くの変更を加えることなくすべてを取り出すことはできないため、いつかこれが変わることを望みます）
  3. \ bin \ BlogEngineCore.dllが再構築されました。 現在、ロシアの見出しはd09ed0b1d0be-d0bcd0bdd0b5 （ BlogEngineの頭痛の種です。URLの書き換えはここでは無効にできないため）の形式のスラッグにはなりませんが、ロシア語の文字（ウィキペディアなど）で直接表示されます。 個人的には、そのように気に入っています:)しかし、投稿からナメクジを音訳するプラグインは、最後の手段としてアセンブリに含まれています（残念ながら、これはカテゴリとタグには適用されないため、解決策は私に完全には合いませんでした...）
  4. 英語の代わりに完全にロシア語版のTinyMCEを追加
  5. 多数の拡張機能が追加され、すべてが翻訳されています。
  6. アセンブリキットには、50のトピックが含まれています。 それらはロシア語化されていませんが、そこではほとんど変更が必要ありません（平均して、Wordpressテーマよりもはるかに少ないです;より単純な構造化コードが影響します）。 将来的には必要に応じて翻訳する予定ですが、これは問題ではないと思います

他に何をする必要があるか

  1. 翻訳するだけでなく、エンジン、特にコントロールパネルを定性的に翻訳します。 用語の不一致を取り除き（悔い改め、仕事の終わりがない...）、いくつかの場所で正しい翻訳のためにコードを修正します。
  2. トピックの翻訳（少なくともいくつか）
  3. BlogEngineを再び改善、改善、改善するには:)一般に、翻訳全体が将来1つのファイルのみを通過するようにしたいと思います-そして、サイトはタンバリンなしでログインしたユーザーの言語に自動的に翻訳されます（これにより、ASP.NETテクノロジーをすぐに実装できます）。 美しさ:)

最後に、なぜ私はこれをしたのですか



簡単です。ASP.NET用のすぐにインストールできるブログエンジンディストリビューションを手元に用意しなければなりませんでした。 テーマと拡張機能が既にあるように。 テーマにUserControlsを入力する必要がない拡張機能。 もちろん、これは難しくありませんが、この場合はそうではありません:)

ゼロからBlogEngineはロシア語圏のオープンスペースで十分な問題を抱えており、さらにすべての問題がスクリプトの編集によって解決されるわけではありません（何らかの理由でカーネルを再構築する必要があり、ここでは誰もがこのためにMicrosoft Visual Studioを開くわけではありません...）、修正-そしてレイアウト。

誰もそれを必要としない場合（最終的には、Webのこのような「軽い」セグメントでASP.NETをほとんど使用せず、テクノロジが非常に優れているのは残念です）、少なくとも必要です。 :)



誰かが興味を持っている場合は、サイトからダウンロードできます 。 ソースへのリンクで使用されるトピックと拡張機能のリストがあります。 何かが正しく動作しない場合（工業的にテストされていない可能性があります:)-書き込み、それを把握します。 :)
