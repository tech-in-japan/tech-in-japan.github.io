知識の進化

このトピックでは、知識の進化などのトピックに触れたいと思います。 情報は抽象的に表示されます。 しかし、これが読者のコメントへのさらなる反省につながることを願っています！



進化は、宇宙のシステムの質的変換の自然なプロセスです。 （c）したがって：知識の進化は、この世界に関する知識とアイデアのシステムの質的な変換の自然なプロセスです。



知識の進化を次のように分割できます。

  1. パーソナル-世界についての私たちのアイデアは、自分自身の中で時間とともにどのように変化するか
  2. 対人-それは、異なる人々の間で交換するときに特定の側面のアイデアがどのように変換されるかです


個人の発達について多くのことが書かれているので、特に触れません。



人間の（遺伝子）進化の場合のように、相互作用の間に知識の定性的な変換が発生します。 次のレベルの情報認識（ 知識ベースのソフトウェアエンジニアリング（Stefanyuk V.L.） ）が区別されます。

  1. プログラミングレベル-簡単に説明すると、「do as I do」。 このレベルでは、人はこれまたはそのことをどれだけ正確に行うべきかを認識します。 ロジックはありません-人は、彼が言われたとおりにすべてをしなければならない（そして、しなければならない）ことだけを理解しています。 これがなぜそうなのかを理解し、そうでなければこのレベルを超えて存在しない。 このレベルでの知覚の典型的な例：テニスやその他のスポーツのトレーニング。
  2. 説明レベル-このレベルの説明には、前のレベルで使用されたすべての要素が含まれる場合があります。 しかし、それに加えて、特定の例が人に与えられ、その人が画像や肖像の同じ分野から他の問題を解決できることを知っています。
  3. 類推のレベル-人が既存の知識やスキルとの類推を見つけることができるレベル。 このレベルの知覚では、人はいくつかの共通の特徴、法律を見つけ、資料を一般化することができます。
  4. 創造性のレベル-このレベルでは、人は以前に遭遇したことのない問題を解決するためのツールを見つけることができます。 このレベルの知覚を使用すると、以前は知られていないか、それほど明白ではない新しい知識や法律がしばしば表示されます。


同じレベルで分割することができ、情報の提示方法。 情報の最も効果的な伝達は、教師（情報のソース）と生徒（情報の受信者）の両方が同じ認知レベルで対話するときに正確に発生します。



さまざまなレベルで情報を送信する簡単な例を考えてみましょう。

ある人がフォーラムで自分の問題を書いた。「助けて！ Linuxでwindows / dev / sdb1にパーティションを接続できません。 どうすればいいですか？」

さまざまな認知レベルからの応答の例を次に示します。

  1. プログラミングレベル：コマンド「mkdir / mnt / windows」を使用して接続用のディレクトリを作成し、/ etc / fstabに次の行「/ dev / sdb1 / mnt / windows auto noatime 0 0」を書き込みます。
  2. 説明のレベル： "fstab" / dev / sdc3 / mnt / myporn auto noatime 0 0 "にこの行があり、すべてがうまくいきます！
  3. 類推レベル：これがどのように行われたか正確には覚えていませんが、デバイスとセクションの内容を表示するディレクトリとの間にマッピングを設定する必要があります。 クラスのフィールド名とデータベース内のフィールドの物理名の間でORMでマッピングを行うようなものです。
  4. 創造性レベル：「vim ./mount/mount.c ./mount/fstab.c」など！


質問：質問者がLinuxの絶対的な初心者であり、それを勉強しない場合、質問者はどの回答をより理解しやすいでしょうか？ しかし、彼が行く場合はどうなりますか？ そして、これが、Hibernateが運命の意志でLinuxに切り替えたことを知っているプログラマーである場合はどうでしょうか。 そして、これがMicrosoftの元Windowsカーネル開発者である場合はどうなりますか？



情報の交換中、教師と生徒の両方が相互作用のテーマに関して自分の認知レベルにあることを別に強調します！ しかし、最も効果的な情報交換のためには、教師と生徒は同じレベルでなければなりません。

  • あなたの目に賢く見えたいなら、フォーラムに行ってください

  • 問題があまりにも多く捕捉されており、その主題分野を完全に理解したい場合は、書籍、レビュー、購読などを探しています。 この件について
  • 問題が単一の場合、または独自の解決策を見つけた場合は、Habrに関するトピック、フォーラム、スタートアップ、このトピックに関する本を作成します




知識の進化の2つの主な方向を区別できます。

  1. 知識の帰納的進化：より一般的なものが特定のものから派生した場合。 たとえば、人々がWindowsパーティションをマウントできない方法についてのフォーラムの投稿を見て、理由、方法、理由を答える1つの良い記事を書くことができます。 帰納的変換は、教師と生徒の相互作用の結果として発生します。後者の場合、認知的認知のレベルがより高くなります。 たとえば、人はテニスを学ぶことができ、このように特定のアクションを実行する必要がある理由を考え、トレーナーが示す物理的な言語にすべてが基づいている「テニスの仕組み」に関する本を書きます。
  2. 知識の演ductive的進化：理論が行動の具体的な指示に変わるとき。 たとえば、「女の子とのデート理論」は、特定のピックアップトレーニングに変わる可能性があります。 演ductive的変換は、教師がより高い認知レベルにあり、「イベント-アクション」という形式の原始的な指示に材料を「単純化」することを余儀なくされるときに、トレーニングの結果として発生します。 たとえば、経験豊富なレースインストラクターがブロンドの女の子に運転方法を教えるのを見たことがありますか？
