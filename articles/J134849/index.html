HPとOracleの対立。 おわりに

私が約束したように、私は前の2つの記事「HPとOracleの間の反対はオープンソースにどのように影響しますか？」と「HPとOracleの間の反対。 継続 " 。 Itaniumの見通しに関連する公式情報が少なくとも一部あったため、遅れが生じました。



残念ながら、この情報はあまり楽しくありません。 それにもかかわらず、Itaniumは、ゆっくりと、しかし確実に、サーバー市場を離れることが明らかになりました。 Oracleに代表される「ソフトウェア」部分を失ったため、Itaniumの運命は事前に決められていました。 顧客の間でパニックはありませんが、長期的にはプラットフォームを離れる必要があることを誰もが理解していると思います。 それでも、状況は壊滅的なものではなく、企業はItaniumサーバーを長期間使用することができ、ほとんどのHPの顧客は可能な限り長く使用します。



どこへ行く？

オラクルの計画によると、ほとんどのお客様がOracleソフトウェアに従ってHP-UX / ItaniumからSolaris / SPARCに切り替える必要があることは間違いありません。 しかし、これは起こりそうにありません。 理由を説明します。



ほとんどの大企業は、RISCサーバーとさまざまなメーカーの垂直スタックからItaniumに切り替えました。 彼らはすでに、すべてのハードウェア、ソフトウェア、およびサービスが1つのメーカーによって提供されている状況にありました。 2000年代初頭には、ほとんどすべてのメーカーがItanumでサーバーを構築する予定だったため、「重いサーバーの標準アーキテクチャ」としてItaniumに切り替えて、ベンダーのロックインを回避しようとしました。 残念ながら、近年、Itaniumサーバーのサプライヤの数が大幅に減少し、HPがこの市場を支配し始め、ほぼ95％を占めています。



現在、オラクルは80年代にそれらを呼び戻し、製品の最適化と統合がいかに優れているかを物語っています。 皮肉なことに、何年も前にOracle DBMSを選択したほとんどの顧客にとって、主な論点は「Oracleはどのプラットフォームでも実行できる」ということでした。 オラクルがどのようにItaniumを放棄するかがわかり、Larry Ellisonがそこで停止するという確実性はありません。



ある調査によると、 HP-UX管理者は、Oracle Itanium DBMSの代替として次のオプションを検討しています。





サンのハードウェアに切り替える準備ができている回答者はわずか6％であることに注意してください。 新しいフォールトトレランス機能と「シングルスタック」に戻りたくないx86プロセッサのパフォーマンスを追加した場合、元のItaniumユーザーは、SPARCまたはIBM POWERではなくx86サーバーに導かれることが明らかになります。



ソフトウェアはどうですか？

OSのHP-UXに最も近い「精神」はLinuxです。 さて、HP-UXからWindowsへのアプリケーションの転送は、どういうわけかそれほど簡単ではありません。実際、アプリケーションを再度書き換える必要があります。 したがって、中期的には、HP-UXからLinuxへのアプリケーションの移行、およびHP-UXシステム管理者の「再トレーニング」を期待する必要があります。



繰り返しになりますが、選択の点では、LinuxはItaniumを含むさまざまなハードウェアプラットフォームで実行されます。 まあ、価格要因も重要です-OSコストをゼロに削減する機会がある場合、これは経済的に混乱した時期のITディレクターにとって良い議論です。



次に、HP-UX / Itaniumの主なユーザーが持っているオプションを見てみましょう。 すでに書いたように 、それらは3つのグループに分けることができます。



1）SAPユーザー

SAPはSybase ASEを持ち、DBMS市場で2位になりたいという願望を持っているため、ほとんどの場合、SAPユーザーは購入に投資したくないでしょう。したがって、新しいハードウェアはHPの既存のハードウェアに残り、SAPでは、顧客がERP競合他社の鉄に切り替えるのを阻止するために可能な限りのことを行います。



2）Oracleユーザー

ユーザーがOracleのDBMSだけではない場合、ほとんどの場合、x86サーバーに、おそらくはSun / Oracleのx86サーバーに切り替えます。 しかし、SPARCではありません。 Itaniumへの移行から全員が回復したわけではないため、SPARCへの移行は考えられません。 ところで、Oracleの最新の財務報告によると、 機器の販売による収益は減少し続けています。



3）独立した開発者からのソフトウェア

HP-UX / Itaniumの独立系ソフトウェアベンダーは、既存のソリューションを可能な限りサポートしますが、ソフトウェアをLinuxに移植し、可能な場合はコンポーネントをオープンソースに置き換えます。



その結果。

この一連の記事の最初の投稿で、特定のs0rr0wがコメントで尋ねました。
そして、OpenSourceがそれとどう関係しているのか、私はまだ理解していません... 2つの企業が前髪をまとめることにしました。 エカは見えない！ オープンソースソフトウェアはそれと何の関係がありますか？
私は答えます、オープンソースソフトウェアはそれと何の関係があるのでしょうか。 「両社は前髪をばらばらにすることを決めた」という事実の結果として、オープンソースに直接関係する2つの掘り起こし会社は明らかに利益を得ています。 1つ目は 、x86サーバー向けエンタープライズLinuxのリーダーとしてのRed Hatです。 ほとんどのHP-UXユーザーは、Linuxの移行計画を検討します。 OpenVMS、HP-UX、NonStop OSを使用してソフトウェアをLinuxに移植する企業の数はそれぞれ増加し、Linuxを使用する企業の数は増加します。 2つ目はEnterpriseDBです。これは、PostgreSQLの開発の背後にあり、Oracle DBMS互換性レイヤーと、 最近発表されたHP-UX / Itaniumのサポートを備えています。
