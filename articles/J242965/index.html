Microsoft Report ViewerおよびOracle Instant Clientレポートを使用した.Netアプリケーションのポータブル配布





多くの場合、インストーラーの作成を拒否し、ファイルフォルダーをターゲットコンピューターにコピーしてアプリケーションを配布することを拒否する必要性または要望があります。 Report Viewerレポートを使用して.Netアプリケーション用のポータブル配布キットを作成する方法、またはOracleデータベースにアクセスするためにクライアントとドライバーをポータブルにコピーする方法に興味がある場合は、catを使用してください。 私はすべてを詳細に説明しようとします。



実用的な部分として、Supermagトレーディングシステム（実際にはOracleデータベースを使用）のレポートを表示するアプリケーションの作成を検討します。



Oracleクライアントのポータブル配布の利点の1つは、その標準インストールが快適な体験ではないことです。 また、レポートビューアーもインストールする場合、複数のマシンでのインストールプロセスは面倒なタスクになるリスクがあります。 もちろん、代替手段としてClickOnceを使用できますが、ClickOnceを使用して配布する場合、この例のようにライブラリフォルダーをコピーすることもできます。

インターフェイスの凍結は非常に悪い習慣なので、データサンプリングは別のストリームで開始します。 データストレージには、ReportDataSMクラスの競合する汎用（汎用）コレクションを使用します。 変数宣言に追加します（クラスの先頭）：

コードのどこかで初期化した後：

インターフェイスをブロックせずにデータを受信することにしたので、メソッドを非同期に呼び出すデリゲートが必要です。 



 delegate void MyGetDataDelegate(string s);


この例では、デリゲートはパラメーターとして1つの文字列値を取ります。

パラメーターは、変更履歴に関するデータを受信するドキュメントの識別子になります。

デリゲート実装であるメソッドは、getDataMethodと呼ばれるメソッドです。

データを取得する方法を実装するための簡略化されたコードを提供します（エラーを処理することなく）。

このメソッドを非同期で呼び出します： 



  MyGetDataDelegate dlgt = new MyGetDataDelegate(this.getDataMethod); IAsyncResult ar = dlgt.BeginInvoke(txtDocN.Text, new AsyncCallback(CompletedCallback), null);


パラメーターとして渡されるCompletedCallbackメソッドに注意してください。

その実装は、呼び出されたメソッドの完了後に他のメソッドを実行するために必要です。 この場合、データを抽出した後、このデータをレポートにバインドし、インターフェースを更新する必要があります。 



  void UpdateUserInterface() { _reportViewer.Reset(); ReportDataSource reportDataSource= new ReportDataSource("DataSet1", list4R); //            _reportViewer.LocalReport.ReportPath = "ReportSM.rdlc"; _reportViewer.LocalReport.DataSources.Add(reportDataSource); _reportViewer.RefreshReport(); }


しかし、アプリケーションストリームからのみインターフェイスを更新できるという点で問題があります。

これを行うには、ディスパッチャスレッドからアプリケーションインターフェイス更新メソッドを呼び出します。 つまり、CompletedCallbackメソッドの内容は次のようになります。 



 void CompletedCallback(IAsyncResult result) { Dispatcher.BeginInvoke(System.Windows.Threading.DispatcherPriority.Normal, new NoArgDelegate(UpdateUserInterface)); }
