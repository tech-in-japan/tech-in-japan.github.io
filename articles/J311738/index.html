Veeam Backup for Microsoft Office 365を使用してクラウドからメールをバックアップおよび復元する

ますます多くのユーザーがMicrosoft Office 365に切り替えており、もちろんクラウドソリューションを使用する場合はデータの安全性を確認したいと考えています。 一見すると、文字がクラウドにある場合、文字はクラッシュや損失から自動的に確実に保護されているように見えますが、実際はそうではありません。 実際、メールサーバーのフォールトトレランスは提供されますが、レターのバックアップは提供されません。 結局のところ、たとえば、ユーザー側で起動されたウイルスがクラウド内の文字を暗号化した場合、その文字を回復することはできません。 そのため、レターのバックアップコピーをローカルに保存できる製品を作成することにしました。 メールボックスの内容を復元するツールは既に利用可能です-これはMicrosoft Exchange用のVeeam Explorerであり、時間のテストに合格しています。



そこで、本日、今年の第4四半期にリリースが予定されている新しいVeeam Backup for Microsoft Office 365をご紹介します。

UPD：リリースは11月29日に行われ、ロシア語の説明を読み、試用版をここからダウンロードします 。



詳細については、猫へようこそ。







データセキュリティのためにクラウドで行われていることOffice 365 Exchange Online



もちろん、管理者はExchange側のポリシー、特にリモートオブジェクトの保持ポリシーを監視します。 削除済みアイテムストアのレターの保存期間がまだ切れていない場合は、そこから取得して復元できます。 既定では、保持期間は30日間で、リモートメールボックスは同じ量で保存されます（同じ既定値を持つ独自のポリシーがあります）。 この値は、Exchange管理センターで変更できます。 MRM（メッセージングレコード管理）ポリシー設定の詳細については、 こちらをご覧ください 。







Microsoftは、Exchange Data Availability Group（DAG）を使用してデータを保護しています。 もちろん、メーカー側での内部使用には、従来のバックアップツールが非常に適用可能です。バックアップは災害復旧の際に役立つはずです。 ただし、ユーザー側で事故が発生することを認める必要があります。この場合の対処方法として、データを保護および復元する方法は完全には明らかではありません。 ユーザー組織側のサービスレベルアグリーメントはメーカーのポリシーに依存しますが、データの回復を保証するものではありません。 これは、Veeamが提供する新しいツールが役立つ場所です-その助けを借りて、あなたはあなたの側でバックアップの作成と、柔軟な回復オプションを確実にすることができます。



Veeam Backup for Microsoft Office 365の仕組み



次の図からわかるように、操作スキームは非常に単純です。







1. Veeam Backup for Microsoft Office 365は、Exchange WebサービスとPowerShellを使用して、Exchange Onlineから階層情報とメールボックスデータを収集します。 データは、Exchangeデータベースと同一の構造でリポジトリ（ローカルフォルダー）に保存されます。 管理者がバックアップタスクを作成するスケジュールでこのプロセスを自動的に開始すると便利です。 このようなタスクが成功するたびに、対応する時点のExchangeデータベースの構造とデータを含む回復ポイントが作成されます。



2、3。Veeam Backup for Microsoft Office 365コンソールでは、メールデータベースからオブジェクトを復元するために必要な状態に応じて、リカバリポイントを選択できます-これは、作成された最後のポイント、または指定された時点。 データベースは、このポイントが作成された時点の状態で、Veeam Explorer for Microsoft Exchangeを使用して（リポジトリから直接）表示するために自動的に開かれます。



4.これで、Exchange Onlineのメールボックス、フォルダー、およびメールボックスオブジェクトの検索、エクスポート、および復元機能を利用できます。



インストールとセットアップ



これに関してはすべて透明です：ZIPアーカイブをダウンロードします。2つのインストーラーがあります。1つは実際にはMicrosoft Office 365用の新しいVeeamバックアップで、2つ目は新しいツールをサポートするために若干変更された使い慣れたMicrosoft Exchange用Veeam Explorerです。



物理マシンと仮想マシンの両方に、一緒にまたは別々にインストールできます。



重要！ Windows 2008 R2以降（64ビット）をサポートします。 別のマシンに配置する場合、Veeam Explorer for ExchangeがインストールされているマシンのOSバージョンが、Veeam Backup for Office 365がインストールされているバージョンよりも古いバージョンであることを確認してください。また、インストールするローカル管理者権限が必要です。



インストール後、新たにデプロイされた2つのアプリケーションがアプリケーションリストに表示されます。 Veeam Backup for Microsoft Office 365を起動します。インストール直後のコンソールは次のようになります。







初期セットアップを実行します。



  1. 左上の[ ホーム ]タブの横にあるアイコンをクリックしてメインメニューを開き、コマンドのリストから[ オプション ]を選択します。







  2. 開いたダイアログで、まずリポジトリ設定を設定する必要があるため、[ リポジトリ ]タブを使用します。







    • 場所-クラウドからのデータがバックアップされるローカルフォルダーへのパスを示します。 デフォルトでは、これはC：\ VeeamRepositoryです。
    • バックアップの保持期間 -このデータをリポジトリに保存する期間を指定します。 最小は1か月、最大は7年、デフォルトでは3年（ 3年 ）です。
    • 保存を適用する -期限切れのデータを確認するタイミング。 希望の周波数を設定できます。 保存期間は、検証日からカウントされます。 期限切れのアイテムは削除されます。


  3. 次に、[ フォルダ ]タブに移動して例外を指定します。つまり、バックアップしないフォルダです。







  4. [ 適用]をクリックして設定を保存し、[ OK ]をクリックしてダイアログを閉じます。


重要！ 行われる設定はグローバルになります。 すべてのバックアップジョブに有効！



すべてが設定にあります。 メインウィンドウの水処理に移ります。ここでは、バックアップするメールの組織であるOffice 365を指定する必要があります。



  1. メニューで[ 組織の追加 ]コマンドを選択すると、対応するウィザード（組織の追加ウィザード）が開始されます。



  2. Office 365組織へのアクセスに使用されるユーザー名とパスワードを指定します。







  3. デフォルトでは、このユーザーに偽装を許可するオプションが選択されています。 [ 次へ ]をクリックすると、ユーザーに必要なロールがあるかどうかが確認されます。 そうでない場合は、PowerShellを使用して割り当てられます。 これらのロールは、 ロール管理 、 表示専用設定 、 表示専用受信者です。









注：原則として、同じPowerShellコマンドレットを使用してこれらの役割を手動で割り当てることは誰も気にしません-この場合、Veeamはそれらの存在をチェックするだけであり、すべてが正常であれば、作業を続行できます。



バックアップ



次に、バックアップタスクを作成する必要があります。



  1. ツリーで組織を選択し、[ バックアップ ]コマンドをクリックしてウィザードを起動します。
  2. 最初のステップで、タスクの名前と、必要に応じて簡単な説明を入力します。
  3. 次に、バックアップするものを選択します。 この組織のすべてのメールボックス （ すべてのメールボックスをバックアップ ）または選択したメールボックスのみ（ 次のメールボックスをバックアップ ）のいずれかです。後者の場合、 選択をクリックして、ロードされたリストから必要なメールボックスを 選択します。







    注：グローバルオプションで構成した除外フォルダー（迷惑メール、削除済みアイテムなど）は、どのタスクでもバックアップされないことに注意してください。



  4. 最後に、スケジュールを設定します-すべてが明確です：指定された頻度で定期的に、作業が長すぎる場合は自動終了（デフォルトでは4時間）、障害が発生した場合は再起動（デフォルトでは10分待ってから3回再起動）



    ウィザードが終了した直後にタスクを開始する場合は、[完了]をクリックしたときにジョブを実行するオプションを選択します。









いくつかの重要なポイント：



  • タスクを編集または完全に削除する場合は、まずタスクを非アクティブ化する必要があります。そのためには、メニューで[ 無効化 ]コマンドを選択する必要があります。
  • 異なるタスクの処理に同じメールボックスを含めないでください。
  • ベータ版では、処理されるメールボックスの数の制限は、MicrosoftのSMB（小規模企業）の制限、つまり最大300のメールボックスに準拠しています。 この製品の独自のライセンススキームは、リリースバージョン用に準備されます。


オブジェクトの回復



復旧するには、Microsoft Exchange用のVeeam Explorerが必要です。 たとえば、次のように実行できます。



  1. Microsoft Office 365のVeeam Backupメニューで、<date_time>の時点で最新の状態を探索コマンド （<date_time>から最後に保存された状態を表示）または特定の時点の状態を探索 （ 一度に状態を表示）を選択します







  2. 特定の瞬間が必要な場合は、開いた[バックアップの探索]ダイアログで、適切な復旧ポイントを選択し、必要に応じてオプションを有効にします。



    • ユーザーが削除したアイテムを表示 - ユーザーが削除したオブジェクトを表示します（選択したリカバリポイントが作成されるまで）
    • ユーザーによって変更されたアイテムのすべてのバージョンを表示 -ユーザーによって削除されたオブジェクトのすべてのバージョン（履歴）を表示






  3. [ 完了 ]をクリックすると、メールストレージ階層がバックアップから直接Veeam Explorer for Microsoft Exchangeに「フック」され、Veeam Explorerウィンドウに表示されます。これは自動的に開始されます。







Veeam Explorerの使い慣れた機能を使用して、オブジェクトを目的の場所にエクスポート、転送、または復元できます。
