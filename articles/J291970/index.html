Salesforceプラットフォームの時間追跡ツールの概要





Salesforce CRMは、業界で最も包括的な顧客関係管理（CRM）ソリューションです。 販売管理とマーケティングおよび顧客サービスを組み合わせて、すべての潜在的な既存の顧客とのすべての取引と関係を効率的に管理できます。



欲しいものを見つける最良の方法は、 https：//appexchange.salesforce.comで既製のアプリケーションを調べることです。 AppExchangeでアプリケーションをインストールするには、いくつかの簡単な操作を行う必要があります。



  1. AppExchange https://appexchange.salesforce.comにログインします （別の方法：組織の[設定]-> [AppExchange Marketplace]をクリックします）。
  2. アプリを見つけます。
  3. 「Get It Now」ボタンを押します。
  4. インストール場所に応じて、[実稼働環境にインストール]または[サンドボックスにインストール]ボタンを押します。
  5. 契約条件への同意を確認します。
  6. パッケージのインストールの詳細を確認してください。
  7. パッケージAPIアクセスを承認します。
  8. セキュリティレベルを選択します。
  9. パッケージをインストールします（「インストール」ボタンを押します）。


お楽しみください！

印象

これがパッケージのすべてのコンテンツです！ 論理はなく、妥当な検証規則さえありません。 合計もレポートもありません。

これらはすべて独立して実装する必要があります。 パッケージが追加するすべてのもの（フィールドとタブを持つオブジェクト）は、1時間以内にパッケージなしで作成できます。







パッケージは、管理されていないパッケージとして配布されます。つまり、自動アップグレードやその他のサポートは提供されません。

Salesforceのパッケージページ

American Data Companyのタイマーアプリケーションの2番目のバージョンは、改善されたインターフェイスを備えた時間追跡に関心のある組織にとって強力なツールです。 組織が時間料金をSalesforce標準またはカスタムオブジェクトに関連付けることができます。



説明

クラスは、別の名前空間プレフィックス：vftimerで設定されます。 これは管理パッケージであるため、クラスのコンテンツは非表示になっています。 単体テストが含まれています。



唯一のタブ「タイマー設定」には、設定のリスト（同じカスタムオブジェクトのレコード）が含まれています。 構成には、4つのフィールド（チェックボックス）を表示するための一連のオプションがあります。



-件名を表示 -件名が必要
-説明の表示 -説明が必要
-仕事の種類を表示 -作業タイプが必要
-請求タイプを表示 -請求タイプが必要








複数のエントリが存在する場合があり、現在の構成には「アクティブ」チェックボックスが付いています（同時に2つのアクティブ構成はできません-EnforceSingleActiveConfigトリガーによってチェックされ、2番目の構成をアクティブとして割り当てようとするとエラーが生成されます）。

[件名が必要]、[説明が必要]、[作業タイプが必要]、[請求タイプが必要]チェックボックスは、対応する[表示]チェックボックスがオンになっている場合にのみチェックできます。

「Timed Item」オブジェクトには、時間償却レコードが含まれています。 フィールド：



-請求タイプ-選択リスト。 選択肢（変更および追加可能）：

  • 請求可能
  • 定価
  • 無償
  • 請求不可




-作業タイプ-選択リスト。 選択肢（変更および追加可能）：



  • ビジネス分析
  • コンサルティング
  • ドキュメンテーション
  • クライアント側の開発
  • サーバー側の開発


-ケース-標準のケースオブジェクトのオプションのルックアップ

-実装日-請求日

-時間-控除される時間

-説明-テキストフィールド。



「Timed Item」オブジェクトには独自のタブがありません（必要に応じて作成できます）。代わりに、別のアプローチが提案されています。 改善が必要です（完了手順については、パッケージに付属の説明書に記載されています）。 簡単に：



  1. 「Timed Item」に、ユーザーが作業しているオブジェクトのルックアップなどのフィールドを追加する必要があり、このオブジェクトを使用する時間を考慮する必要があります。 デフォルトでは、このようなルックアップは既にCaseオブジェクトに存在します。 カスタムオブジェクトまたは標準オブジェクト（たとえば、連絡先）に追加して、会社のクライアントとのコンサルティングに費やした時間を相殺することができます。 多くの関係が存在する可能性があるため、さまざまなタイプのレコードを操作するときに時間を考慮することができます。
  2. 特定の各レコードの時間償却を確認できるように、関連リストを関連オブジェクトのレイアウトに追加する必要があります。
  3. コンポーネントのVisualForceページへのURLを含むカスタムボタンを追加する必要があります。 このボタンをクリックすると、タイマーパッケージが開きます（スクリーンショット「3. Timer window.png」を参照）-このパッケージの主な機能。 このウィンドウでは、オブジェクトの作業の開始時と終了時にタイマーを開始/停止し、以前の償却を表示できます。








したがって、 CRMユーザーには次のアクションアルゴリズムが提供されます。



  1. ユーザーが作業している既存のオブジェクトレコードを作成または開きます。
  2. ボタンを押して、タイマー付きのウィンドウを開きます。
  3. タイマーを開始します。
  4. 実際には、最初のステップで録音を開いて作業します。
  5. タイマーを停止します。 償却時間（ある場合）のフィールドに入力します。 必要に応じて、タイマー値を調整できます。
  6. タイマーウィンドウを閉じます。 ユーザーが作業したレコードを含むウィンドウを閉じるか、次のウィンドウに進みます。








レポートとグラフはパッケージに含まれていません。 ただし、単純なカスタムオブジェクトを使用して時間キャンセルレコードを保存するため、標準のSalesForceツールを使用して自分で作成することは難しくありません。



タイマーページには、American Data Companyのロゴが含まれています。 宣言されたブラウザーサポート：



  • Firefox 3.x（クロス—プラットフォーム）
  • Internet Explorer 8（Windows）
  • Chrome 2.x（Windows）


私は特にテストしませんでしたが、Operaに小さなバグがあることを偶然発見しました。ボタンの署名が重複しているので、生き残ることができます。

ページは管理パッケージに含まれており、編集することはできませんが、コードは開いています。 パッケージページからコンテンツをコピーしてページを作成し、スタイルを調整したり変更したりできます。 ボタンを押してページを開きます。



合計：簡単にカスタマイズでき、カスタマイズ可能性は元々定められていて、非常に使いやすいです。 すべてのデータはSalesforceに保存されるため、標準のSalesforceツールを使用できます。 無料で。
