Citrix無料テストドライブ

Apple iPadでのCitrix Receiverクライアントのテストと新しい開発-Citrix XenClientに関する資料では、このようなソリューションの価格についていくつかの質問がありました。 また、Citrixは仮想化技術の販売で本当に利益を上げていますが、同社のほとんどすべてのソリューションは完全に合法的に無料でインストールして使用できます。 たとえば、特定のテクノロジが実際にどのように機能するかを確認したり、小規模企業に仮想デスクトップへのアクセスを提供したりすることもできます。 このような小規模では、Citrixのロイヤリティを支払う必要はなく、Microsoftライセンスと、もちろんハードウェアのみに費やします。



テスト構成では、わずか数台のラップトップで対応できました。 1つ（強力なプロセッサと大量のRAM）がサーバーの役割を果たしました。 もう1つは、クライアントの役割に最適なコンパクトなLenovo ThinkPad X100eです。 ただし、クライアントは他の任意のデバイス（たとえば、SymbianまたはAndroid、iPhoneまたはiPadベースのスマートフォン）にすることができます。 これにより、サーバー上の仮想マシンに接続したり、「実際の」ハードウェアで実行されているオペレーティングシステムにリモートアクセスできます。 2番目のオプションは、「ライト」クライアントから強力なコンピューターのリソースを取得する場合に関連します。マルチメディアコンテンツやゲームを処理するためです。





まず、問題を定式化します。結果として何を得たいのでしょうか？



•サーバーとして機能するラップトップ。 3つの仮想OSが動作します。クライアントから接続するユーザーOSと、小規模ネットワークのドメインコントローラーとして機能するWindows 2003 Serverです。



•Windows Server 2003を搭載した別の仮想マシン。DesktopDelivery Controllerプログラムを実行し、クライアントデバイスを仮想マシンまたは物理マシンに接続するプロセスを提供します。



•WindowsクライアントOSを搭載したコンピューター。これもローカルネットワークに含まれます。 実際のハードウェアで実行されているこのシステムに、クライアントからのリモートアクセスを手配します。



•コンピューターをネットワークに接続するためのネットワークデバイス。 ギガビットイーサネットポート（サーバーとコンピューターの接続用）およびWiFiアクセスポイント（クライアントラップトップの接続用）を備えた通常のホームWi-Fiルーターを使用しました。



開始するには、Citrix WebサイトからXenDesktop Free Edition配布キットをダウンロードする必要があります。 1.34ギガバイトのXenDesktopディストリビューションは、簡単な登録後にここからダウンロードできます。 アーカイブには、XenServerディストリビューション、仮想OSを管理するXenCenterプログラム、および仮想デスクトップの配信を管理するDesktop Delivery Controllerコンポーネントがあります。 このエディションのXenDesktopは無料で使用でき、最大10人のユーザーが作業できます。 XenDesktopのすべての主な機能と利点をこのビデオに示します（英語のコメント付き）：







次に、XenServerをインストールします。このプロセスでは、少なくとも2、3回オペレーティングシステムを再インストールしたユーザーから質問が寄せられることはほとんどありません。 注意する必要があるのは、正しいネットワーク設定だけです。 XenServerの典型的なインストールプロセスは、CitrixシステムエンジニアのSergey Halyapinによるコメントとともにこのビデオで見ることができます。







XenServerは、コンピューターまたはラップトップにインストールできます。唯一の要件は、ハードウェア仮想化サポートを備えた64ビットプロセッサー-Intel VTまたはAMD-Vです。 XenServerのシステム要件の詳細については、 ドキュメントを参照してください。 次のビデオは、XenServerでの仮想マシンの起動とWindows 2003 Serverのインストールを示しています。 ここではすべてがはるかに簡単です。クライアントデバイスから、XenCenterプログラムを使用して、以前にインストールされたXenServerが既に実行されているサーバーに接続します。 オペレーティングシステムのイメージのライブラリ（ISO形式）を接続し、OSを仮想環境にインストールします。 OSをインストールし、ドメインをセットアップするプロセス全体（若干の縮小を伴う）をこのビデオに示します。



Windows Server 2003がインストールされている2番目の仮想マシンには、Desktop Delivery Controllerコンポーネントをインストールします。これにより、リモートクライアントデバイスを実際のハードウェアで実行されている仮想またはオペレーティングシステムに接続できます。 Desktop Delivery Controllerのセットアップ手順はこちらです。 小さな会社にとって、これは最適なソリューションです。1台の強力なサーバーが、仮想コンピューターパークの管理とクライアント仮想OSのサポートを担当します。 サーバーがそのような負荷を引くことができることを確認してください。



その後、すべてが簡単です。サーバー上に必要な数の仮想マシンを作成し、それらにWindowsクライアントOSをインストールします。 仮想OS自体にCitrix VDAエージェントをインストールし、これらのマシンをDesktop Delivery Controllerに登録します。 「実際の」Windowsを備えたコンピューターに同じエージェントをインストールし、リモートアクセスを提供します。 この段階では、仮想システムとハードウェアベースのどちらのシステムに接続しているのかは重要ではありません。



最後に、 ここからダウンロードして 、Citrix Receiverプログラムをクライアントのラップトップにインストールします。 それを使用して、デスクトップインフラストラクチャに接続し、接続するオペレーティングシステムを選択してリモートで作業します！ 仮想OSに接続するには、WiFiまたはセルラーネットワークを介して接続する必要がありますが、3次元グラフィックスなどの一部の複雑なコンテンツをクライアントマシンに転送するには、高帯域幅のチャネルが必要です。 HDXテクノロジーを使用すると、比較的低速の接続を介して3次元グラフィックスを迅速に転送できますが、これはすでに他の素材のトピックです。



まとめ



もちろん、示されている例では、かなり複雑なシステムを組み立てています。自宅では、RDPを介したデスクトップへのリモートアクセスで十分です。 しかし、異なるオペレーティングシステムでさまざまなクライアントデバイスを使用する従業員向けのドメインコントローラーと仮想デスクトップを備えた小規模企業向けの非常に機能的なシステムを手に入れました。 また、グラフィカルアプリケーションを使用するユーザーは、強力なワークステーションにリモートで接続できます。



Citrixの商用ソリューションは、はるかに柔軟性があります。 たとえば、この場合、各仮想マシンは常にサーバーのハードドライブのスペースを占有します。 XenDesktopの商用バージョンを使用すると、各仮想OSにディスクスペースを割り当てる必要はありません。 この場合、仮想OSのイメージ、プログラム、およびユーザーデータは別々に保存され、クライアントが接続されると、通常の作業環境で「収集」されます。 この場合、サーバーの読み込みがより効率的になり、セキュリティが強化されます。プログラムが正しくインストールされなかったり、ユーザーエラーが発生してもデータが失われることはありません。 また、管理者は各クライアントOSを個別に操作する必要がないため、ソフトウェアを集中的に更新できます。
