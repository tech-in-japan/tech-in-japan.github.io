Say2GO





インスタントメッセージングテクノロジーは、近年大きく変化していません。 現在のAOL IMは、97年のバージョンとそれほど変わりません。 VoIPのオープンとWebカメラの数の増加のおかげで、Skypeが登場しました。Skypeは通信とIM市場の何かを本当に変えました。 ただし、音声通信はリアルタイムで実行され、テキスト通信はほぼリアルな状態で行われるという TechCrunch レポートの 2つは同じままでした。



Say2GOは、両方のタイプの通信を組み合わせることができる新しいほぼリアルタイムのメッセージングシステムです。 このテクノロジーは、IMよりもボイスメールに似ています。 現在非常に限られているこの製品は、ユーザーがオリジナルおよびテキスト（プログラムが音声を認識する）形式で送信される音声メッセージを送受信する機能を提供します。 クライアントはMicrosoft Speech APIを認識に使用するため、Windowsオペレーティングシステムのユーザーのみが使用できます。 さらに、いくつかの追加ライブラリをインストールする必要があります。それらを追加しないと、プログラムはきれいできれいなインターフェイスを備えた通常のIMになります。





このテクノロジーは非常に新しいため、TechCrunchのジャーナリストが非常に残念な結果を述べなければならなかったことは驚くことではありません。 「音声認識プログラムの別のテスト」というフレーズは、Say2GOシステムによって「男はあなたが読んで以来、電話はミッションの犠牲者であると何か他のことを言った」と理解されていました。 2番目のテスターは、「なぜこのプログラムの学習に10分費やしたのか」というフレーズに「プログラムのコーチングに10分費やさない」というフレーズがあり、より幸運でした。 何と言っても、Say2GOのユーモアでは、すべてがうまくいきます。



MSN Messengerには、2005年に登場したMS Voice Clipsと同じ実用的な機能があります。これにより、ユーザーは最大15秒間音声メッセージを友人に送信できます。 また、Say2GOを使用すると、後でメッセージを送信するために、録音を再度繰り返したり、キャンセルしたり、キューに入れたりすることができます。



多くの専門家は、適切なレベルでの音声認識がモバイルプラットフォームの未来であると言っているため、Microsoftはこの技術に真剣に関心を持っているようです。 これは、Yahoo（少なくとも1つの検索を思い出してください）とGoogleの両方によって理解されています-先日、彼らはGOOG-411音声認識システムのテスト開始を発表しました。 彼らはこれとソフトなものを実装しており、Live Search for Windows Mobileは長い間音声クエリを使用できました。 マイクロソフトとSay2GOの関係は非常に友好的であり、最近2番目の会社は、マイクロソフト2008パートナーのISV / Software Solutionsコンテストで47,000社の参加企業を迂回して1位になりました。 現時点では、システムはICQプロトコルをサポートしていますが、開発者はGTalkとJabberの友人と通信する機能を追加することを約束しています。
