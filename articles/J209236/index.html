AWS Test Drive-Amazonのサンドボックス

少し前まで、AWS AmazonはAWS Test Driveと呼ばれるサービスを導入しました。 幸運なことに、Amazonと当社の共同AWSプログラムの一環として、このサービスをよりよく知り、その機能を試すことができました。 そもそも、それが何であり、なぜそれが必要なのかを簡単に説明したいと思います。



これは何ですか


実際、これは、Amazonパートナーが提供する既成の技術ソリューションのセットであり、実験室作業の形で作成されています。

各実験室の作業には、その目的の簡単な説明、テキストベースの段階的なマニュアル、ビデオレッスン、そしてもちろん、クラウド形成テンプレートが含まれており、作業の大部分を行う必要があります。

各実験室作業には、制限時間と開始回数があります。

AWS Test Driveユーザーとして無料でリソースが提供されるため、これは非常に論理的です。

Oracle、Microsoft、SAP、Redhat、Alfresco、Trend micro、Infor、Sophos、F5からテストラボが利用可能になりました。









残念ながら、すべてのテストラボが公開されているわけではありません。 たとえば、RedHatテストラボに登録しようとしたときに、Gmailでの登録は「RedListed」であると言われました。

ただし、 Microsoftラボに簡単に登録して、たとえばSQL Server AlwaysOnソリューションを試すことができます（このテストラボは、2つのデータセンターに展開されたSQL Server 2012の高可用性機能を実証します。さまざまなフェールオーバーシナリオを検証します）

登録とログイン後、およそ次の画像が表示されます。







「今すぐお試しください！」をクリックして、ラボメニュー自体にアクセスします。ここで、環境の作成を直接開始するか、このラボ作業に関するドキュメントをダウンロードできます。









誰がこれを必要としますか？


はい、原則として、開発、テスト、または管理に関連するIT分野の代表者に対して。

これは、業界をリードする開発者からソフトウェア製品と技術ソリューションを無料で試用すると同時に、システム管理とシステムアーキテクチャの分野で新しい知識を得る絶好の機会です。

たとえば、Oracleのラボの一部として、Oracle Database Disaster Recoveryに慣れることができます。



パートナーシッププログラムの一環として、MongoDBをレプリケーションモードで展開するためのラボ作業を準備していました。
