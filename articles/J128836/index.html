上記のリンクをたどると、これらの2つのすばらしい製品について読むことができます。特に、SQLiteの例を使用して、Qtでのデータベースプログラミングを具体的に検討します。 SQLiteは、クライアントサーバーアーキテクチャを持たないという点で、MySQLなどの「通常の」データベースとは多少異なるとしか言えません。 つまり、データベースエンジンは、プログラムが対話する個別に動作するプロセスではありません。 SQLiteは、プログラムがリンクされるライブラリであるため、エンジンはプログラムの不可欠な部分になります。 つまり、プログラムが「遭遇する」すべてのデータを通常のファイルに保存することにしたと想像してください。 ある晴れた日、データをファイルに保存することにしましたが、「リレーショナル」の観点から整理しました。 その後、新しいファイル構造を「特別な方法で認識する」必要があることに気付きました。 このことから、少なくとも、このデータファイルとアプリケーション間のリンクを提供するAPIを提供する必要があります。 一般に、上記のシナリオの論理ステートメントに従って、データベースサーバーと実際にはクライアントを必要としないデータベースシステムを作成します。 「クライアントサーバー」データベースシステムと比較して非常に高速であることが判明し、プログラム自体が簡素化されています。

そして、Qtとデータベース



上記ですでに暗黙的に言及したように、Qtには、データベースを使用するための便利な「サービス」を提供する別のモジュール-QtSqlがあります。 Qtの経験がある場合は、.proファイルについて知っています。そうでなければ、 知り合いになります。 .proファイルに次の行を追加する必要があることを忘れないでください。 

QT += sql


これはQtSqlモジュールを使用し、そのクラスを操作するために、同じタイトルを含める必要があります。 

 #include <QtSql>


Qtブックでは、QtSqlモジュールの3つのレベルについて説明しています。

  1. ドライバーレベル
  2. プログラムレベル
  3. ユーザーインターフェイスレベル
ドライバーレベル


ドライバレベルには、次のような物理レベルでデータを受信するためのクラスが含まれます。

  • QSqlDriver
  • QSqlDriverCreator <T *>
  • QSqlDriverCreatorBase、
  • QSqlDriverPlugin
  • QSqlResult
QSqlDriverは、特定のデータベースにアクセスするために設計された抽象基本クラスです。 クラスを「直接」使用しないようにすることが重要であり、代わりにQSqlDatabaseが必要/使用できます。 ただし、独自のSQLドライバーを作成する場合は、 QSqlDriverから継承して、必要な仮想関数と仮想関数を実装できます。

QSqlDriverCreatorは、特定の種類のドライバーにSQLドライバーファクトリーを提供するテンプレートクラスです。 テンプレートパラメーターはQSqlDriverのサブクラスである必要があります。

QSqlCreatorBaseは、SQLドライバーファクトリーの基本クラスであり、提供するQSqlDriverクラスの特定のサブクラスのインスタンスを返すには、 createObject（）メソッドを「言い換える」必要があります。

QSqlDatabaseは、データベースドライバープラグインの読み込みと管理を担当します。 データベースが追加されると（これはQSqlDatabase :: addDatabase（）関数によって行われます）、必要なドライバープラグインが（ QSqlDriverPluginを使用して）ロードされます。 QSqlDriverPluginは、カスタムQSqlDriverプラグインの抽象基本クラスを提供します。

QSqlResult自体は（すべてのQtクラスのように）それ自体について語り、このクラスは特定のデータベースのデータにアクセスするための抽象インターフェースを提供します。 QSqlQueryはQSqlResultの DB固有の実装のラッパー（「一般化」）を提供するため、実用的な観点から、 QSqlResultの代わりにQSqlQueryを使用します。

したがって、ドライバーのレベルは、独自のドライバーを作成するときに使用することが重要であるため、ドライバーのフレームワークとして使用できる（最も関心のある）コードの例を示します。 



 class XyzResult : public QSqlResult { public: XyzResult(const QSqlDriver *driver) : QSqlResult(driver) {} ~XyzResult() {} protected: QVariant data(int /* index */) { return QVariant(); } bool isNull(int /* index */) { return false; } bool reset(const QString & /* query */) { return false; } bool fetch(int /* index */) { return false; } bool fetchFirst() { return false; } bool fetchLast() { return false; } int size() { return 0; } int numRowsAffected() { return 0; } QSqlRecord record() const { return QSqlRecord(); } }; class XyzDriver : public QSqlDriver { public: XyzDriver() {} ~XyzDriver() {} bool hasFeature(DriverFeature /* feature */) const { return false; } bool open(const QString & /* db */, const QString & /* user */, const QString & /* password */, const QString & /* host */, int /* port */, const QString & /* options */) { return false; } void close() {} QSqlResult *createResult() const { return new XyzResult(this); } };




プログラムレベル


データベースに接続するには、まず静的メソッドQSqlDatabase :: addDatabase（）を使用してドライバーをアクティブにする必要があります。 メソッドは、DBMSドライバの識別子を示す文字列を引数として受け取ります。 QSQLITEが必要になります。 



 QSqlDatabase sdb = QSqlDatabase::addDatabase("QSQLITE"); sdb.setDatabaseName("db_name.sqlite"); if (!sdb.open()) { //.... }


addDatabase静的関数には、ドライバー名ではなくドライバー自体（ QSqlDriver *）を受け取るオーバーロードされた「兄弟」があります。

接続はopen（）メソッドを使用して行われます。 QSqlDatabaseクラスは、データベース接続を表します。 接続は、サポートされているデータベースドライバーを介したデータベースへのアクセスを提供します。 同じデータベースに複数の接続を確立できることが重要です。

接続中にエラーが発生した場合（ open（）メソッド） 、エラーに関する情報はQSqlDatabase :: lastError（）メソッドから取得できます（ QSqlErrorを返します）。 



 if (!sdb.open()) { qDebug() << sdb.lastError().text(); }


Qtを使用してSQLコマンドを実行する方法を検討してください。 QSqlQueryクラスを使用してこれを行うことができます。 このクラスは、 SELECT 、 INSERT 、 UPDATE 、 DELETEなどのDML（データ操作言語）式の実行だけでなく、 CREATE TABLEなどのDDL（データ定義言語）式の実行にも使用できます。 SQL標準ではないDB固有のコマンドを実行できることに注意してください（たとえば、PSQLの場合-「SET DATESTYLE = ISO」）。

正常に実行されたリクエストはリクエストのステータスを「アクティブ」に設定するため、 isActive（）は true を返します 。そうでない場合、状態は非アクティブに設定されます。 クエリは通常の文字列の形式で発行され、コンストラクターまたはQSqlQuery :: exec（）メソッドに渡されます。 最初のケースでは、コンストラクターに渡すときに、コマンドが自動的に起動されます（オブジェクトの構築時）。

非常に興味深いのは、 QSqlQueryが提供するナビゲーション機能です 。 たとえば、 SELECTクエリの後、 next（）、previous（）、first（）、last（）、およびseek（）メソッドを使用して、収集されたデータをナビゲートできます。 



 QSqlQuery query("SELECT country FROM artist"); while (query.next()) { QString country = query.value(0).toString(); do_something(country); }


next（）メソッドを使用すると、データの次の行に移動し、前の行にそれぞれprevious（）を呼び出すことができます。 first（）、last（）は、それぞれ結果から最初のレコードを取得します。 seek（）は整数インデックスを取得し、受信したインデックスの結果からレコードを取得し、抽出されたレコードに「クエリを配置」します。 サイズ（）メソッドを使用して、サイズ、またはデータの行数（結果）を確認できます。 最初のレコードが位置0にあり、クエリがアクティブな状態である必要があり、 isSelect （）がtrue （最後のクエリがSELECTの場合に発生）を返してからseek（）メソッドを呼び出すことに注意してください。 公式ドキュメントの seek （）メソッドについて詳しく読むことをお勧めします。

クエリ文字列をQSqlQueryクラスのコンストラクタに渡すと、オブジェクトの作成時に-構築中にクエリが実行されることを上で述べました。 exec （）メソッドを使用すると、いわば、リクエストの実行時間を監視できます。 建設業 

 QSqlQuery query("SELECT country FROM artist");
次のように表すこともできます。 

 QSqlQuery query; query.exec("SELECT country FROM artist");
そのため、 exec （）は要求をQStringとして受け取ります。 リクエストが満たされると、成功した場合、このメソッドはtrueを返し、状態をアクティブに設定します。それ以外の場合、すべては示された操作と「反対」になります。 もちろん、クエリ文字列は、要求されたデータベース（特にSQL標準）の構文規則に従う必要があることも覚えておく必要があります。

興味深いのは、実行後、要求が無効な（uy？）レコードに配置されることです。つまり、結果を適切に使用するには、たとえばnext （）メソッドを使用する必要があります。

exec （）メソッドには、引数を受け取らないオーバーロードされた代替があります。 このバリアントexec （）の呼び出しは、この前に準備された要求を実行します。 注意してください-「準備済み」。 これを行うには、要求の準備が成功した場合にtrueを返すprepare （）メソッドを使用します。

メソッドの重要性、または言えることは、 bindValue （）を使用して値をバインドするための「プレースホルダー」をリクエストに含めることができることです。 

 QSqlQuery my_query; my_query.prepare("INSERT INTO my_table (number, address, age)" "VALUES (:number, :address, :age);"); my_query.bindValue(":number", "14"); my_query.bindValue(":address", "hello world str."); my_query.bindValue(":age", "37");


匿名パラメーターのオプションを使用することもできます。 



 QSqlQuery my_query; my_query.prepare("INSERT INTO my_table (number, address, age)" "VALUES (?, ?, ?);"); my_query.bindValue("14"); my_query.bindValue("hello world str."); my_query.bindValue("37");


最後に、QStringが提供するインライン引数を単純に使用できます。 



 QSqlQuery my_query; my_query.prepare( QString("INSERT INTO my_table (number, address, age) VALUES (%1, '%2', %3);") .arg("14").arg("hello world str.").arg("37") );


コンパイル済み（コピーして貼り付けて自分のアイデアに）​​の例： 



 #include <QtGui/QApplication> #include <QtSql> int main(int argc, char *argv[]) { QCoreApplication app(argc, argv); QSqlDatabase dbase = QSqlDatabase::addDatabase("QSQLITE"); dbase.setDatabaseName("my_db.sqlite"); if (!dbase.open()) { qDebug() << "-   !"; return -1; } QSqlQuery a_query; // DDL query QString str = "CREATE TABLE my_table (" "number integer PRIMARY KEY NOT NULL, " "address VARCHAR(255), " "age integer" ");"; bool b = a_query.exec(str); if (!b) { qDebug() << "    ,  !"; } // DML QString str_insert = "INSERT INTO my_table(number, address, age) " "VALUES (%1, '%2', %3);"; str = str_insert.arg("14") .arg("hello world str.") .arg("37"); b = a_query.exec(str); if (!b) { qDebug() << "   ,  ,   ?"; } //..... if (!a_query.exec("SELECT * FROM my_table")) { qDebug() << "   ,  ."; return -2; } QSqlRecord rec = a_query.record(); int number = 0, age = 0; QString address = ""; while (a_query.next()) { number = a_query.value(rec.indexOf("number")).toInt(); age = a_query.value(rec.indexOf("age")).toInt(); address = a_query.value(rec.indexOf("address")).toString(); qDebug() << "number is " << number << ". age is " << age << ". address" << address; } return app.exec(); }


クエリ結果を取得するには、 QSqlQuery :: value（）メソッドを呼び出す必要があります。このメソッドでは、例でrecord （）メソッドが使用された列番号を渡す必要があります。 このメソッドは、 SELECTクエリに関連する情報を含むQSqlRecordクラスのオブジェクトを返します。 QSqlRecord :: indexOf（）を呼び出すことにより、列インデックスを取得します。

value（）メソッドはQVariant型（オブジェクトに異なる型の値を含めることができるクラス）の値を返すため、メソッドQVariant :: toInt（）およびQVariant :: toString（）を使用して結果の値を変換しました。



ユーザーインターフェイスレベル


QtSqlモジュールは、プレゼンテーションで使用するための多数のモデルを提供することにより、インタビューの概念をサポートしています。 この概念に慣れるには、 こちらをご覧ください 。

例として、 QSqlTableModelクラスを使用すると、データを表形式および階層形式で表示できます。 文献によると、インタビューは表データを表示する最も簡単な方法であり、ここでは表の行を調べるサイクルは必要ありません。 以下に小さな例を示します。 



 #include <QtGui/QApplication> #include <QtSql> #include <QTableView> int main(int argc, char *argv[]) { QCoreApplication app(argc, argv); QSqlDatabase dbase = QSqlDatabase::addDatabase("QSQLITE"); dbase.setDatabaseName("my_db.sqlite"); if (!dbase.open()) { qDebug() << "-    !"; return -1; } QTableView view; QSqlTableModel model; model.setTable("my_table"); model.select(); model.setEditStrategy(QSqlTableModel::OnFieldChange); view.setModel(&model); view.show(); return app.exec(); }


接続後、 QTableViewテーブルビューオブジェクトとQSqlTableModelテーブルモデルオブジェクトが作成されます。 setTable （）メソッドはモデルの現在のベースを設定し、 select （）呼び出しはデータで満たされます。

QSqlTableModelクラスは、次の編集戦略を提供します（ setEditStrategy （）を使用して設定）：

  • OnRowChange-ユーザーがテーブルの別の行に移動するとすぐにデータを記録します。
  • OnFieldChange-ユーザーがテーブル内の別のセルに移動した後に記録します。
  • OnManualSubmit - submitAll （）スロットへの呼び出しでデータを記録します。 revertAll （）スロットが呼び出されると 、データは元の状態に戻ります。
