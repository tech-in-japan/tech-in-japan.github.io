マルチポイントサービス



MultiPoint Server（MPS）は、基本的なリモートデスクトップ機能を提供するWindows ServerとRDSに基づくテクノロジとソリューションです。 負荷やスケーラビリティに大きな要件がない教室や施設での使用に適しています。 特徴は、ユーザーステーションはモニター、キーボード、マウス（「ゼロ」クライアント）のみで構成され、USBハブ、ビデオケーブル、またはLAN（RDP-over-LAN、クライアントがあれば）を介してMPSサーバーに直接接続できることです。たとえば、ラップトップまたはシンクライアント）。 その結果、エンドユーザーは、ユーザーエンドステーションに最小限のコストでデスクトップ機能を提供する低コストのソリューションを手に入れることができます。



2010年2月にリリースされたMPSの最初のバージョンには、専用のUSBハブとビデオポートを介してのみステーションを接続する機能がありました。



RDPを介した接続機能は、2011年3月にリリースされたMPS 2011の次のバージョンでのみ追加されました。RDP-over-LANに加えて、MPS 2011は次のように更新されました。

  • RemoteFXのサポート
  • 仮想化サポート
  • あるステーションから別のステーションにデスクトップを投影する（たとえば、トレーナーまたは教師のデスクトップがユーザーステーションに複製される）
  • フィルターベースのインターネットアクセスを制限する機能
  • リモートアプリケーションの起動、接続されたステーションの周辺機器ロック（キーボード、マウス）


次に、そして現時点では、MPS 2012の最新バージョンが追加されました。

  • 集中デスク管理用の新しいコンソール
  • 不要な変更からシステムパーティションを保護する
  • タブレットを含むステーションを監視および管理するためのMPSコネクタクライアント

ステーションの主なタイプMultiPoint Services



前述のように、MPSは従来のRDPをサポートするだけでなく、次の方法で「ゼロ」クライアント（ Wyse 1000を例に）を接続することも可能にします。



  • ヘッドエンドビデオカードへの直接接続



    この図では、4つのクライアントステーションがUSBおよびVGAポートなどを介してメインステーションに直接接続されています。 明らかに、このタイプの接続は、ヘッドエンドのハードウェア構成に対応する要件を意味し、一部のシナリオ（スケール、距離、モビリティ）には適用されません







  • USB接続



    以下の図は、プライマリステーション（MPSに直接接続され、シナリオに関係なく初期構成に使用されるステーション）と、USBハブを介して接続された2つの「ゼロ」クライアント（例： Wise 1000 ）の相互作用を示しています。 最初の方法とは異なり、MPSサーバーのビデオサブシステムの構成を追加で計算して、必要な数のビデオ出力を形成する必要はありません。 ただし、ステーションとMPS間の距離が限られているため（Dell Wise 1000〜5メートルの場合）、少数のエンドユーザーがいる小さな部屋で使用することをお勧めします。







  • USB-Over-Ethernetを使用する



    よりスケーラブルなタイプの接続。 USBからUSBへの代わりに、LAN経由のUSB転送が使用されるため、大規模な敷地内にMPSシステムを構築する機会が提供されます（クライアントの例： Wise 1003 ）





そして、Windows Server 2016はどうですか？



上記の機能は、Windows Server 2016（現在はTechnical Preview 4 ）に完全に移植されています。 MultiPoint Serverは、新しいタイプのRDS展開になりました。



2016年には、このような展開スキームのライセンスに関するデータはまだありません。 つまり、MPS 2012のライセンススキームはWS 2016に部分的に移行され、MPSのエディションは廃止されます。













展開プロセス



VDIまたはセッションベースのソリューションの一部としてRDS構成手順にすでに精通している経験豊富なエンジニアまたは管理者にとって、MPSのセットアップと使用のプロセスはより簡単で迅速に思えます。 MPSの対象読者を考えると、これもプラスです。



MultiPoint Servicesをインストールするには、サーバーマネージャー（ロールベース）、Powershell、およびRDSインストールの3つの方法があります。



最初の2つを簡単に確認してから、基本的なMPSセットアッププロセスに進みます。

  1. サーバーマネージャーを使用して役割をインストールし、MultiPoint Servicesを選択し、追加コンポーネントのインストールを受け入れて、次の手順に進みます。







  2. MPSとは何かをもう一度読むことができます。 RDライセンスは、MPSの構成後にアクティブ化する必要があることに注意してください。







  3. メインのMPSサービスと一緒に、印刷およびドキュメントサービスが追加で展開されます。その目的は、すべての人に知られていると思います。 興味深いことは何もありません。







  4. すべてをデフォルトのままにします。



    • プリントサーバー -複数のプリンターの管理に不可欠
    • 分散スキャンサーバー -分散スキャン管理をサポートするスキャナーを管理および共有します
    • インターネット印刷 -インターネット印刷プロトコルを介して印刷するドキュメントを送信する機能を備えたプリンタージョブへのWebアクセス
    • LPDサービス-Line Printer Daemonサービスは、UNIXクライアントがLine Printer Remoteサービスを使用して、利用可能なプリンターに印刷タスクを送信する機能を提供します。








  5. 本格的なRDSは必要ないため、デフォルト値は提案されたままにします。







  6. 確認後、サーバーは再起動し、プライマリステーションを使用して、MPSの最初の起動時に必要な構成を行う必要があります。 起動時に、「B」キーを押してプライマリステーションを識別することが提案されます。その後、サーバーはRDS / MPSサービス構成モードに切り替わります。







    MPSは、 WmsShellアカウントを追加してマルチステーション操作をサポートし、 WmsOperatorsグループを作成してダッシュボードへのアクセスを作成します。





PowerShellでは、6つのポイントすべてを1つのコマンドに「絞る」ことができます。







MPSマネージャーに行きましょう







リモートステーションから、RDP-over-LANを介したMPSへのアクセスを設定したい。 これを行うには、新しいMPSユーザーアカウントを追加します







MPSの観点からは、MPSにアクセスするための標準ユーザー、ユーザーセッションを管理するためのユーザー、および管理者の3種類のユーザーがあります。 実際、これは本格的なRBAC（ロールベースのアクセス制御）の模倣です。







そのため、ユーザーが追加されます。 接続を確認しましょう。 MSTSCおよびRDP機能を使用して、上記のアカウントを使用してMPSサーバーに接続します。 各ユーザーが最初にMPSに接続すると、メッセージが表示されます： 「このコンピューターの使用を支援するために、システム管理者がアクティビティを監視することができます/このコンピューターの使用を支援するためにシステム管理者がシステムを監視します。」







確認後、ユーザーの新しいターミナルセッションが作成され、管理者はMPSダッシュボードを使用してユーザーセッションをオンラインで管理できます。



MPSダッシュボード（別のコンソール）に進みましょう。 コンソールの主要部分は、ユーザーセッションの動的に変化するミニ画面で占められます。 カメラからのビデオ信号を監視するためのセキュリティサービスの画面を思い出させますが、MPSを使用すると、ユーザーセッションで何が起こっているかを観察できるだけでなく、それらを実際に管理および変更することもできます（制御の取得、ステーションのブロック、ログオフの開始、送信選択したユーザーへのIM、USBデバイスのブロック、またはアプリケーションのリモート起動/終了）。







たとえば、各ユーザーステーションとそのセッションでは、次のことができます。



  • ステーションのロックを解除/ロックし、特定のステーションの画面にメッセージを表示します。







  • 許可または禁止されたU​​RLのリストを指定して、Webアクセスを制限します。







  • クライアントステーションにデスクトップを投影します。









MPSマネージャーに戻ると、接続されたステーションが[ステーション]タブに表示され、選択したステーションを追加で管理できます。







MPS自体の設定は、ホームの開始コントリビューションにあります。 たとえば、セッションがプライベートではないという通知をオフにして、ユーザーに追加の質問がないようにすることができます:)







マルチセッションモードにはいくつかのセキュリティリスクが伴うため、不要な変更からシステムドライブを保護することができます。 ディスク保護を有効にするには、ワンクリックで確認します。







クライアント環境（場合によってはスタンドアロン環境）を必要とするアプリケーションがある場合、これは仮想デスクトップを有効にすることでMPSで実現されます。 動作原理は、本格的なVDIのプールされたコレクションに似ています。 各「仮想化ステーション」はテンプレートから作成され、各ユーザーがシステムからログアウトした後に変更をロールバックします。 ご覧のとおり、VDIの完全な機能は実現されていませんが、そのような機会が存在することでMPSの範囲が拡大します。









よくある質問



MPSサービスを正しく削除する方法は？



サーバーマネージャーから役割を削除し、サーバーを再起動してスクリプトを実行します 。









MPS用のPowerShellモジュールはどうですか？



現在、個別のモジュールはありません。 私の意見では、RDSの既存のモジュールは、MPS管理をサポートするために更新される予定です。

MultiPoint Serverの機能をWindows Server 2016に移行することは、かなり興味深い有用なソリューションであり、使用が推奨されるシナリオでMPSの適用性を正確に復活させるはずです。 標準のトレーニングクラスに加えて、パートナーはMPSを使用して、デモスタンドまたはショールーム、独立トレーナー、および学生や顧客に情報を正しく伝えることを目標とするその他の専門家を提供することもできます。
