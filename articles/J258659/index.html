まえがき



ユーザーに通知するために、まだSMSまたは電子メールを使用しています。 通知はすぐに消えて、次に何が起こるかわかりません。 PushAllでは、通知データ収集システムを実装しました。 チャンネル内のユーザーにニュースレターを送信し、通知で何が起こるかを監視できます。

  • 通知を受け取った人の数
  • 通知を開いた人（リンクをクリックした人）
  • リストから通知を削除した人の数


すべては、デバイスで実行されたアクションに関するフィードバックのおかげで機能します。



用途は何ですか？



多くの人がすでに推測していると思いますが、説明します。 YouTubeに複数の発信動画があり、PushAllにチャンネル登録者が1000人いるチャンネルがあるとします。 情報が視聴者に届いたかどうか、どれだけの人が各ビデオにアクセスし、どれだけ面白くないかを評価できます。 そして最も重要なのは、これらの指標を比較して、ダイナミクスで確認できることです。



将来、このデータはAPIをリクエストすることで取得できます。 また、タイムスタンプも記録されます。 つまり、グラフを見ることができます。ユーザーが通知を受信し始めた時間、ユーザーが通知を開いた時間または閉じた時間です。 原則として、コンテンツに興味がない場合は、すぐに閉じます。 コンテンツが彼らにとって興味深いものであるが、今はそれまでではない場合、彼らは通知を「ハング」させて後で開くことができます。 彼が今彼らにとって重要なら、彼らはすぐに彼をクリックします。 このため、このアプローチの2番目の利点があります - 通知または情報が時間内に「腐敗」したか、間違った時間（夜間、職場、早朝）に到着したかどうかを確認します 。



また、通知の配信を決定することにより、通知を別のソースに複製できます。 たとえば、監視に通知を使用する場合、問題が発生した場合にプッシュ通知を送信できます。配信を確認してから10分後に、配信されていない場合はSMS経由で複製できます。 または、たとえば、何らかのブログシステムの場合、プッシュを送信できます。プッシュが届かない場合、または反応がない場合は、コピーをメールに送信します。 その結果、メッセージは確実に届き、ボックスが常に詰まることはありません。



これらの推奨事項は、選択したサービスの競争上の優位性としてだけでなく、アプリケーションのプッシュ通知の利点の例としても引用されています。



ユニキャストAPI



前のトピックで、この機能が実装されていることを書きました。 まだほとんど使用していませんが、非常に便利です。 前述のように、プライベートメッセージまたはコメントへの返信に関する通知を送信できます。 APIは非常に単純です。送信するには、コールバックを介して取得できるユーザーIDを知る必要があります。

すべてが非常に簡単です：







1.編集モードで作成されたチャンネルで、IDを取得するアドレスを入力します。







2.サブスクリプションリンクをコピーします。 このリンクにより、ユーザーにはすぐにチャンネルが表示され、ワンクリックでチャンネル登録できます。



3.スクリプトにpushalluseridパラメーターのGETを実装しますが、ADDRESSとして渡されるようになりました？Pushalluserid = 1、つまり 拡張オプションはサポートされていません。 必要に応じて、柔軟なパラメーター追加のためのリンク定義を作成します。



また、ユーザーがカタログにアクセスし、サイトについて知らない可能性があることにも留意してください。サブスクリプション時に、サイトが新しいタブで開き、ユーザーが登録できます。 この場合、さらに番号をユーザーにバインドするには、セッションでIDを記録する必要があります。



さらに送信は、PHPの例であるcURLを介して行われます。 



curl_setopt_array($ch = curl_init(), array( CURLOPT_URL => "https://pushall.ru/api.php", CURLOPT_POSTFIELDS => array( "type" => "unicast", "id" => " ", "key" => " ", "uid" => "ID " "text" => " ", "title" => "" ), CURLOPT_SAFE_UPLOAD => true, CURLOPT_RETURNTRANSFER => true )); $return=curl_exec($ch); //    curl_close($ch);


すべての種類の通知に適用される追加オプションもあります。

  • url-プッシュへのリンクを追加します
  • hidden-1に等しい場合、送信後にプッシュを履歴から非表示にします
  • エンコード-エンコード、たとえばutf8を使用する場合はcp1251-使用しない




サブスクライバーのリストでユーザーIDを自分で確認することもできます。 また、手動で彼にメッセージを送信します。



オタク誌







最近、 SoHabrが私たちに接続しました。Habrahabr、Giktayms、MegamindのITリソースに関する記事を1本のテープで表示するための便利なインターフェイスを提供します。 Habrahabr、Giktayms、Megamind ITリソースに関する記事をすべて1つのストリームで表示するための便利なインターフェイスを提供します。 そして今、PushAllのおかげで、プッシュ通知をサポートしています。







購読すると、すべての発信記事があなたにまき散らされます（かなり不便です）。 この問題を解決するフィルターがあります。 上記はSoHabrの記事の例です。ここでは、フィルターに必要なハブを1行に1つずつ入力し、それらのハブでのみ通知を受信できます。 さらに、ハブだけでなく、記事タイトル内の単語や説明のテキストも入力できます-すべてが考慮されます。 これにより、このバンドルは単純なハブ通知よりも柔軟になります。



これと並行して、お気に入りのシリーズの音の出力に関する通知をサブスクライブし、サービスの監視を強化できます。 そして、これらすべてを1つのアプリケーションで行います。
