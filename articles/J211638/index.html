マイクロマーキング（またはセマンティックマークアップ）とは、ページに書き込まれた内容を検索ロボットに示すタグ内の追加タグと属性でページをマークすることを意味します。



マイクロマークアップは、辞書と構文で構成されます。



辞書は一種の「言語」であり、クラスとそのプロパティのセットであり、その助けを借りてページ上のコンテンツの本質が示されます。 たとえば、ディクショナリは、名前を示す用語（「名前」、「タイトル」、または「n」）を決定します。



構文は、そのような言語を使用する方法です。 辞書。 Webページなどで、どのタグでエンティティとそのプロパティがどのように示されるかを決定します。



セマンティックマークアップは段階的に開発され、さまざまなイニシアチブグループがコンセプトの開発に着手しました。 そして、最終的に、さまざまな辞書と構文からビネグレットを入手しました-それらは非常に多くあり、それらすべてを最初に処理するのは簡単ではありません。

インターネットで最も一般的な辞書



グラフを開く




Open Graph（OG）は、最も一般的で最も簡単な辞書です。 現在、Open Graphが最もよく使用されているため、サイトから公開されたリンクが拡張され、美しく、理解しやすくなっています。 OGマークアップを使用すると、人気のあるすべてのソーシャルネットワークにリンクが表示されます。



また、Open GraphマークアップはFacebookのアプリケーションで積極的に使用されています。これにより、ユーザーはアプリケーションのアクションをページに反映できます。



OGのおかげで、動画を見たり、記事の簡単な説明を読んだり、無限のニュースフィードを見ながら友人が共有する情報の本質をすばやく理解したりできます。 Open Graphマークアップは、Facebookに加えて、 Vkontakte 、 Google + 、 Twitter 、 LinkedIn 、 Pinterestなどによっても認識されています。



辞書自体は非常に簡単に使用できます。使用を開始するには、4つのプロパティが必要です。



  • og：title-オブジェクトの名前。
  • og：type-オブジェクトのタイプ。たとえば、「video.movi​​e」（映画）。 タイプに応じて、他のプロパティを指定できます。
  • og：image-それを説明する画像のURL。
  • og：urlは、永続IDとして使用されるオブジェクトの正規URLです。


たとえば、人の説明のOpen Graphマークアップは次のようになります。 



<html prefix="og: http://ogp.me/ns# profile: http://ogp.me/ns/profile#"> <head> <meta property="og:title" content=" " /> <meta property="og:type" content="profile" /> <meta property="og:url" content="http://example.com/" /> <meta property="og:image" content="http://example.com/" /> <meta property="profile:first_name" content="" /> <meta property="profile:last_name" content="" /> <meta property="profile:gender" content="male" /> ... </head> ... </html>


ここで、ロボットはページがユーリ・ガガーリンという名前の男性専用であると認識し、彼の写真へのリンクがあります。 ここで、urlなどのプロパティは、ページの正規URLとして示されます。



og：タイプタグでは、「プロファイル」タイプに加えて、さまざまなタイプのエンティティを指定できます（これらにも独自のプロパティがあります）。



  • 音楽（music.song、music.album、music.playlist、music.radio_stationのサブタイプ）-曲の場合、期間、アルバム、アーティスト、アルバムの場合-曲、アーティスト、リリース日を指定できます。
  • ビデオ（video.movi​​e、video.episode、video.tv_show、video.other）-映画には、俳優とその役割、監督、脚本家、長さがあります。
  • 垂直（記事、書籍、プロファイル、ウェブサイト）なし-ここでは、上記のカテゴリに該当しないタイプを示します。 記事では、タグ、著者、公開日を指定できます。 プロフィール-性別、姓、名。


サイトページでそのようなマークアップを使用しない場合、Facebookでリンクを公開するときに、システムはどのような場合でもプレビューを作成しようとします。 しかし、原則として、これはそれほどうまくいくにはほど遠いです-記事の写真の代わりに、サイトのロゴが反映され、タイトルがサイトのカテゴリの名前に置き換えられ、記事の本質を反映しない会社の歴史のテキストが記事の簡単な説明になりますユーザーを喜ばせそうにありません）。

Schema.org






Schema.orgは、2011年に検索エンジンのイニシアチブで登場した辞書です。 Yandex、Google、Bing、Yahoo！でサポートされています。



Schema.orgは、さまざまなエンティティとそのプロパティを記述するクラスのセットも提供します。 しかし、OGとMicroformats.orgにそのようなクラスが多数ある場合、Schema.orgにはすでに数百個あります。 すべてのクラスには、 ツリー階層での位置があります 。



これは活気に満ちた柔軟な辞書です。 追加の前に新しいエンティティが積極的に議論されます。このため、イニシアチブグループのメンバーが毎週会合を開き、スキームの実装、拡張、および使用について議論します。



エンティティの最も一般的なタイプは、サブタイプを持つThingです。 それらのいくつかを検討してください。



  • アクション -特定の人（個人または組織）が実行できるアクションについて説明します。 このアクションは、コミットされた場所、オブジェクト、およびツールをさらに示す場合があります。 他のアクションと同様に、結果、参加者、およびそれが行われた期間を設定できます。 アクションを記述するこのクラスの助けを借りて、 Yandex.Islandsが開発され、 Gmail Actionsプロジェクトが実装されます。
  • CreativeWork-創造的な作品の特徴を説明します。 ビデオ、写真、レシピ、ダイエット-すべてこのタイプを使用して説明できます。 すべての創造的な作品は、著者、ジャンル、短い説明だけでなく、レビュー、聴衆、参考文献などを示すことができます。
  • イベント -他のイベントと同様に、ここで会場、日付、参加者、スピーカーなどを説明できます。
  • 製品は、売買されるものすべてです。 一部の有料サービス（ヘアカットなど）は、製品タイプによって説明することもできます。 製品には、評価、ブランド、色、視聴者、価格、モデルなどがあります。
  • 人 -Schema.orgのドキュメントに示されているように、生きている、架空の、またはすでに死んでいる人、および「アンデッド」（明らかに、作成者はより多くのゾンビを記述する必要があり、適切なタイプはありません）。 人々は、連絡先情報、仕事、家族、人間関係などに関する情報を持つことができます。

Microformats.org






Microformats.org （Microformats）は、愛好家のコミュニティによって2007年に作成されたオープンスタンダードです。 このコミュニティは、既存のテクノロジを使用してサイトのセマンティックマークアップの標準を作成することを本当に望んでいました。 6年前、これは実装が簡単だったため、標準の明確なプラスでしたが、現在はマイクロフォーマットのマークアップを追加するのは簡単ではなく、場合によっては他の辞書よりも複雑です。 OGやSchema.orgと比較して、使用頻度は減っています。



現在、いくつかのサブジェクト領域に約10の一般的なマイクロフォーマット仕様があります。 一部は完了していますが、ほとんどはドラフト段階です。 組織、製品、レビュー、イベント、および他の多くのエンティティに関する情報を公開するためのマイクロフォーマットがあります。 各エンティティには独自のプロパティがあります。



新しいmicroformatsはオープンモードで開発されています;別のmicroformat wikiがあります 。 各マイクロフォーマットを作成するとき、ファウンダーは全員に同意し、妥協点を見つけようとするため、プロセスは非常に長く続き、時には終わらないこともあります。 このため、修正されたmicroformatsは指で数えることができ、ドラフトステータスを持つものが非常に多くあります。
