FIXber-ソフトウェアテストサービス市場

こんにちは、Guards！



私の仕事の性質上、さまざまなプロジェクト（Web、デスクトップ、モバイルなど）をテストするためにテスターを選択する必要があります。

同時に、テスターは宣言されたスキルとプロ意識を常に正当化するわけではありません。



このすべての骨の折れる活動の結果として、ソフトウェアテストサービスの本格的な市場を作成するというアイデアが生まれました（各テスト担当者は彼のスキルのおかげで仕事を得ることができ、顧客はプロジェクトの段階に関係なく、プロジェクトの完全なテストの結果です） 。



100を取得して、興味深いバグに対してのみ支払うことができる場合、「1人」のテスターを雇う理由。









顧客に関するいくつかの事実：
  1. 顧客はもはや自分のプロジェクトに苦労してテスターを選択することはありません。テスターは顧客のプロジェクトを自分で見つけます。
  2. 興味深いバグのみを購入し、
  3. バグを購入する前に、彼はそれについて簡単な説明だけを見ます。
  4. プロジェクトへのアクセスを制限したい-問題なく、各テスターを個人的に承認し、
  5. プロジェクトが特定の場合、特定のテスターのみが作業に招待されます。


テスターに​​関するいくつかの事実：
  1. 各テスターは、自分の知識とスキルに基づいて自分自身のプロジェクト（または複数のプロジェクト）を選択します。
  2. バグを販売する-評価を獲得して引き上げる、
  3. バグの重大度が高いほど、コストが高くなります。
  4. 評価を上げます-より多くの機会を得ます。


PSさて、私はあなたのためにかなり多くの情報をスケッチしました-続きは少し後になります。

システムをテスト用にレイアウトした後、興味のあるすべての人にフルアクセスを提供し、すべてに「触れる」ことができます。
