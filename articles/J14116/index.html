最初の図-ギフトとしてのWEB 3.0アイデアMail.ru

私はゆっくりと目標に到達しようとします。

私の目標は、 Web 3.0イデオロギーポータルの潜在的な機能を実証し、Web 3.0プロジェクトを実際に実装できるチームを編成することです。



Web 3.0の理論は Vkontakteで議論され、ユーザーグループを作成する作業を行うためのこの技術に関する提案がありました。 以下は、このようなシステムの説明例です。



そのため、特定のトピックへの関心に関連するユーザーのオーディエンスを1か所に集めて保持するのが仕事です。 このようなあなたの作品に興味があるのは誰ですか？ -さて、Mail.ruを例に考えてみましょう（例としてVkontakte.ruを挙げますが、これはまだ非営利プロジェクトです）。マイルは彼のページから広告を販売することで稼いでいます。 これで、Mail.ru（以下、マイルと呼びます）が条件を作成する必要があります！



例を挙げましょう：人々に気高いサボテン栽培者がいて、興味のあるユーザーを自分の周りに集めて長い間保持できるとしましょう。 ここで、「そして、彼らにサイトを作らせて、それらでお金を稼がせてください！」 しかし、どのくらいの有名なサボテン栽培者が良いサイトを作ることができますか？ これらのうちどれだけが高トラフィックの技術的問題を解決できますか？ 最後に、サイトの広告の可能性をどれだけ効果的に販売できますか？ -答えは、単位または一般的に「0」です。



マイルはこれらすべての問題を専門的に解決します-そして、これらは彼が作成しなければならない必要条件です。 つまり ポータルのMileは、ハイブリッドサイト-ブログ-フォーラム-ギャラリーを作成し、それに統計システムを固定します（そうすることで、彼らはあなたにお金を正しく請求すると信じられます）。 同時に、主要なサボテン栽培者としてのあなたの仕事は狭くなり、実行可能になります。 あなたはサボテン愛好家のグループを集め、同時にたむろして広告を消費し、コンテンツを生成します。 マイルは訪問者をカウントし、成功に比例して支払います。 また、Web 3.0システムに必須のユーザートレーニングシステムを導入し、たとえば、アクティブなバザーの後にそれらをトレーニングする場合は、主要な正しい考えを簡単に残しておきます。 それからしばらくすると、乾燥した残留物が参考書の本に、またはそこに入力されます-これも製品ですが、内部には再び広告があります:-)。



したがって、マイルは、あらゆるトピックのイデオロギー的リーダーのモードでお金を稼ぐために、ユーザーの無限の輪を提供します。 有能なリーダーは0.1％未満であると仮定しますが、総数はまだ多く、彼らは時々出席を増やすことができます-したがって、広告収入。 マイケルがそのような組織に何を必要としているのか考えてみましょう。 ブログ、ギャラリー、統計システム、電子メールの操作があります。 彼らはお金を稼ぐ方法を知っており、開始プロモーションを組織することができます-明らかに何も必要ありません。



彼らにこの考えを与えることができますか？ たぶん油を投げる？
