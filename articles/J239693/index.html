ITプロフェッショナルの目から見たクラウド：ビジョンと期待

今世紀のほとんどの組織の情報は、非常に重要な要素になりました。 血液が体内を循環しているように、適切なタイミングで適切な場所に配信される情報は、ビジネスの機能と発展に決定的な役割を果たします。 IT企業が現在直面しているだけでなく、その流れが雪崩のように成長したため、データ、ストレージ、および管理を操作する新しい方法を探す必要がありました。 「次は何ですか？」という質問に対する答えは、空中でハングするのに時間がかかりませんでした。 インターネットチャネルの成長とそのユニバーサルな可用性により、この答えが生まれました-「クラウドサービス」。







登場したクラウドには、真に無制限のデータ管理機能があります。 パブリック、プライベート、ハイブリッド（最初の2つの混合）-これらはすべて、ネットワーク管理者の奥の部屋にある古いテラビット「シーゲイト」ではなく、自動バックアップシステムが確立された超近代的なTIER III-IVデータセンターにある情報へのアクセス制御のモデルですインターネットが存在するクマのコーナーからのデータの24時間の可用性。 同時に、クラウドサービスは、たとえば、生産ニーズのために互いにほとんど調整されていないさまざまなソフトウェア製品を使用している企業にとってはあまり効果的ではない場合があります。 この問題の解決策は、会社の事務を単一のサプライヤーから単一のソフトウェアプラットフォームに完全に移行することです。 多くの企業が現在行っているのは、この最適化の方法です。これはトレンドです。 多くの場合、単一のソフトウェア環境内でデータのバックアップ、アーカイブ、リカバリを整理するのは非常に簡単です。



QuinStreet Enterprise Research（QSER）が行った最近の調査では、1〜3つのソフトウェアシステムを使用した実務経験の回答者の35％、56％が4つ以上を使用したことが示されました。 また、この調査では明らかですが、それほど興味深い機能も示されました。回答者の3分の2が、現在職場で直面している多数のソフトウェアから統合データ管理ソフトウェア環境への移行を支持しました。







調査会社QSERは、EWeek、Datamation、CIO Insightなどのオンライン出版物も含むメディアホールディングの不可欠な部分です。 223の大規模IT企業の従業員を含む、会社が実施した調査。 調査の主な目的は、これらの企業の従業員の活動を単一のソフトウェアプラットフォームに移行する可能性についての意見を見つけることでした。 また、クラウドサービスを使用する主な利点と欠点を特定するタスクも調査の範囲内でした。



上記のデータからわかるように、最初の質問は回答者を少し混乱させ、回答には明確なリーダーはいませんでした。 そして質問は、「単一のインターフェースからすべてのデータ管理を使用することの利点」でした。 しかし、回答者の約4分の3が、実行された作業の効率の向上が移行の主な利点であると考えていることが明らかになりました。 それでも、回答者の3分の1にとって、この利点は現在のソリューションからの脱却を保証するほど重要ではありませんでした。 スケジュール2からわかるように、この立場、何かを変更することを嫌がるのは、ソフトウェアメーカーとの既存の契約、および大規模な再編成に関連するコストが原因である可能性があります。 統合ソフトウェアの利点を理解している人にもっと詳しく戻ってきたら、チャート3を見る価値があります。 何らかの形でのデータの安全性は、会社の従業員によって表明された要求の中でリーダーになりました。 約80％は、クラウドベースの統合データ管理システムが、あらゆる種類の予測不可能な損失から情報を保護する役割を担っていると考えています。そのリスクは私たちの生活の不可欠な部分です。 セキュリティのリクエストの直後に、データの複製と重複排除のリクエストがあります。 また、回答者は、保存されたデータを分析する能力の向上と、アクセスしやすさを賞賛しました。



クラウドデータ管理がより高度になっています



多くの組織は、最小限の投資でデータを保存および処理する新しい機会を積極的に探しています。 また、パブリックかプライベートかに関係なく、クラウドサービスに切り替えるほど、このサービスに関連するテクノロジーがより速く改善され、データを保存および処理する理想的な場所になります。 現在の状況は、現在観察しているクラウドへの企業の雪崩のような移行への対応です。



多くの企業にとって、クローズドクラウドオペレーティングモデルは、許容可能なレベルの情報保護と制御を提供します。 その結果、サービス市場全体がここで拡大しています。 図4からわかるように、現在使用されているクラウドサービスのほぼ半分は、それらへのアクセスが制限されたプライベートインフラストラクチャプロジェクトです。 要求された容量の残りの半分は、パブリックサービスとハイブリッド（オープンクラウドとクローズドクラウドの混合）サービスに分割されます。 最も有名なオープンクラウドの例は、Amazon Simple Storage、Service（S3）、Microsoft Azure Storage、およびGoogle Cloud Storageです。
