メッセンジャーのUX：2005-2015。パートI.背景。 2005-2007年

競争の過程で、メッセンジャーは顧客によりアクセスしやすくするために大幅な変更を受けました。 過去10年間の機能、インターフェイス、UXインスタントメッセージングシステムの開発における最も重要なステップを検討することをお勧めします。



2005年：その前後



1996年に発売された最初のインスタントメッセンジャーであるICQの人気の波は減少しました。 ICQの大規模なクライアントベースは、サービスのシンプルさとアクセシビリティによって提供されました。 メッセンジャーの基本的なアイデア-名前、居住地、メール、ニックネーム、性別、年齢、およびユーザーが示すその他の情報で連絡先を検索する機能-は、元の名前にも反映されていました：ICQの略語は、「私はあなたを探しています」（=私が探していますあなた）。 長い間、インスタントメッセージング用のクライアントを選択するとき、最小限の機能と要求の厳しいリソースが決定的な要因でした。



ICQのよく知られた機能は、昇順でユーザーに発行された一意の識別番号でした。 通信を開始する前に、対談者の番号を確認する必要がありました。 同時に、美しく特別な部屋は特に価値があり、かなりの量で転売することができました。



ICQの古いバージョンでは、個別のウィンドウでの着信メッセージと発信メッセージの出力など、現代のユーザーには珍しい機能がありました（これは明らかに、電子メール形式からの移行の段階でした）。 ICQは、他の海外のインスタントメッセンジャーよりも人気が劣っていたにもかかわらず、ロシアで普及しており、一部の人はまだICQを使用していることは注目に値します。



ICQの基盤となるOSCARネットワークプロトコルは、その独自のクローズドコードにもかかわらず、この時点ですでにリバースエンジニアリング手法によって解析され、機能を拡張する膨大な数の非公式クライアント（たとえば、登場したロシアのマルチプロトコルQIPクライアントなど）の開発に使用されていました2005年）。



非公式の顧客の人気は高まりましたが、公式のICQは遅くなり、広告でいっぱいになりました。 2005年に、ICQ 5にスキンのシステムが導入されましたが、その時点で、サービスにはすべての種類の見掛け倒しが既に過剰に搭載されていました。



人気の減少傾向は、アメリカ企業AOLのAIMメッセンジャーにも影響を与えました。 AIMは、特にオフィスや職場で人気がありました。 マイクロソフトでさえ、MSNメッセンジャーでAIMプロトコルのサポートを導入しました。これにより、AIMコードの書き換えが繰り返され、「サイバー戦争」が発生しました。その間、マイクロソフトプログラマーは更新されたAIMコードを何度もクラックしました。 対立の終わりは、Microsoft MSNに悪意のあるコードを導入する脅威によってもたらされました。



AIMメッセンジャーは革新的でした。 ボイスチャットは、Skypeよりもずっと早く導入されました。 メッセンジャーは、ファイルを転送し、携帯電話とSMSを交換することができました。 ただし、無料で配布されたため、開発にはあまり注意が払われませんでした。 メッセンジャーで少なくとも何かをmoneyけようとして、AIMインターフェースにあまり収まらない広告バナーを導入することが決定されました。 同社はサードパーティのプラグインの開発を阻止しました。 AIMの小さな開発チームは、企業経営者の反対にもかかわらず、彼らの革新を擁護しようとしました。



別のニッチはIRCクライアント（たとえば、 mIRC ）によって占有されます。 IRCプロトコルには多くのクライアントがありますが、プロトコルのシンプルさ、コードのオープン性、統合の幅広い可能性、スクリプト言語を使用した拡張性および自動化により、技術的に高度なユーザーに特に人気があります。 多くの場合、プライベートサーバーでオンライン会議を作成するために使用されます。



同様の機能は、XMPPプロトコル（Jabber）に基づいてクライアントによって実行されます。 これらのインスタントメッセージングシステムは現在人気があります。 それらは、特定の応用分野を持ち、その設立以来実質的に変更されていません。 これらのシステムは、サイトに直接埋め込まれたWebクライアントを使用することで、二の風になります。 サイトで表示されるチャットは、これらのプロトコルのいずれかを使用する可能性が非常に高いです。



IRCおよびJabberクライアントの大部分は、単なるテキストチャットです。 フォントをカスタマイズすることもありますが、デフォルトのフォントは「技術的」で、等幅です。 強調表示されたメッセージやニックネームが色付きで表示されることもありますカスタマイズ可能な場合もあります。 誰かがそのような実装を「マルチプレイヤーメモ帳」と巧妙に呼んでいました。



2003年に開始されたSkypeは 、当初はマイクとビデオカメラを介した通信機能を備えたインスタントメッセージングサービスとして位置付けられ、当時最も有望なプロジェクトの1つでした（後に判明）が、Skypeは当時のコンピューターリソースをかなり要求していたため、人気はすぐに彼から遠ざかりました。 主に音声およびビデオ通信の手段として使用するインスタントメッセンジャーのコンテキストでそれを正確に認識した人はほとんどいませんでした。



Microsoftオペレーティングシステムの基本パッケージに含まれているMSN Messenger （後のWindows Live Messenger）は、主に配布方法が理由で人気がありました。 この時点で、ファイル転送や最初のVoIPサービスの1つを介した音声通話などの機能が既に備わっていました。 Windows XPのリリースにより、連絡先のグループ化と音声会話のグループ化の機能がMSNメッセンジャーに追加されました。 MSNの興味深い機能の1つに、Windows Media Playerとの統合プラグインがありました。これは、ユーザーのプロファイルで（User Playing）が聞いていた音楽の行を表示しました。 2005年、フラッシュアニメーションファイルがMSNメッセンジャーに挿入され、「wink」（wink）というメッセージが挿入されました。 それらは、現代のアニメーション絵文字と絵文字の基礎として機能しました。



2005年にGoogleトークが開始されました。その主な機能は、もちろんGoogleアカウントとGmailとの深い統合でした。 Googleトークは、上記のXMPPプロトコル（Jabber）を使用します。



多数のインスタントメッセンジャーとそのプロトコルは、統合と標準化の傾向があり、マルチプロトコルメッセンジャーフォーマットの出現につながっています。 その1つであるQIPはすでに上記で言及されています。 開発者がこれらのプロトコルの作成に関与した独自のプロトコルを備えた古いインスタントメッセンジャーとは異なり、このようなインスタントメッセンジャーでは、インターフェイスと特別な機能に重点が置かれています。 開発の結果、全体の「組み合わせ」が得られました。



たとえば、この形式のメッセンジャー-Trillianを考えてみましょう。 独自のプロトコルAstraに加えて、ICQ、IRC、Jabber、Facebook、MSN（Windows Live）、Yahoo、Facebookなどのプロトコルをサポートしています。 彼はまた、Twitterでの書き方やメールの使い方を知っています。 Trillianは、音声通信中に感情のあるサウンドファイルを再生したり、転送されたファイルをアーカイブにパックしたり、XMLマークアップをサポートして会話やユーザーアクションを記録するための幅広い機能を備えています。 このようなプログラムを使用すると、さまざまなソーシャルネットワークやサードパーティサービスに散らばっている連絡先を「収集」できます。



2006年



Twitterが登場しました。 このサービスは、簡潔な状態と短い公開メッセージについて空中にあるアイデアを実現しました。 ほとんどのマルチプロトコルクライアントはTwitterのサポートを追加しましたが、独自のクライアントを介したスマートフォンでの使用に適していますが、コンピューターではWebインターフェイスを使用する方が簡単です。



2007年



Skype 3.5がリリースされました。ユーザープロファイルでビデオがサポートされ、チャットにビデオを埋め込むことができます。 Skypeは人気が高まっており、ICQのような他のインスタントメッセンジャーを締め出している。
