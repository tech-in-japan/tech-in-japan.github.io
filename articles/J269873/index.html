ダイジェスト：今年の夏に検索を教えたもの

こんにちは、Habr！ ご存じのように、鶏は秋に考慮されます。 秋が本格的です。今年の夏にGoogle検索を教えたことを見てみましょう。







検索開発は2つの方向で行われます。 それらの1つはあなたが見ない-それは内部の変化、7つのアザラシの背後にある謎、神聖な聖なるもの、その他すべてです。 ここで魔法が起こり、 言葉が認識され 、音声クエリが処理され、写真、ビデオ、音楽が検索され、発見されます。



2番目の方向-Google検索サービスと対話するためのユーザーインターフェイスとツールは、個人的に観察できます。 今日、これらの改善についてお話しします。突然、誰が何かを見落としましたか？





Twitterからのモバイル検索と結果



1つの投稿の140文字には、驚くほど多くの情報を含めることができます。 当初の非常にシンプルで最小限のサービスは、機能でいっぱいになり、全世界を結び付けました。最速の「ニュース代理店」の1つであり、最も予期しない状況（緊急事態に関する警告を含む）を支援します。 Google検索を使用すると、このソーシャルネットワークから検索結果をカード形式で直接取得できます。











％interest_object％Twitterのようなリクエストは、対応するアカウントが存在する場合、そのアカウントからの最新の投稿になります。 また、人気のハッシュタグのリクエストにより、WebバージョンのTwitterや対応するアプリケーションにアクセスしなくても、最新の投稿を見ることができます。



5月末に一部の言語で導入されました。 これまでのところ、それらの間にロシア語はありません。



Google NowのContext別名Context上のGoogle Now



そして再び、モバイルプラットフォーム。 小さな対角線上での作業の利便性（および物理キーボードなしで、片手で、外出中、または「静かに座って静かに作業する」から明らかに遠い条件でも）は常にホットな問題でした。 マルチタスクは優れていますが、便利でコンテキストに応じたマルチタスクはさらに優れています。



Google Now on Tapを使用すると、アプリケーションを切り替えることはできませんが、どの画面からでも、ほとんどすべてのタスクの上からモバイル検索を呼び出すことができます。 アプリケーションは一時停止され、入力した情報、現在のステータス、画面上の位置は失われません。











Nowのコンテキストは、スマートフォンにインストールされているアプリケーションと対話できます。 たとえば、友人との通信を「一時停止」し、Now on Tapを使用して、夕方に会いたいレストランを見つけました。 テーブルを予約できるアプリケーションがあり、検索結果がテーブルとやり取りできる場合でも、「メイン」画面を離れることなくそれらを使用できます。











もう1つの例は、Spotifyストリーミングプレーヤーでの作業です。 作曲を行っていて、バンドのボーカリストについてもっと知りたいですか？ Google Nowのコンテキストを使用するだけです。 「誰がバンドのボーカリストですか」と尋ねます。 彼は、残りの情報を起動コンテキストから取得します。この場合、Spotifyプレイヤーから取得します。











スマートフォンをさらにスマートにするために設計された小さな変更。 Android Mのリリースで英語が利用可能になります。



アプリ内でiDeviceを検索する



iOSユーザーのことは忘れませんでした。 iPhoneでNow on Tapを呼び出すことはできませんが、iOSアプリケーションのデータを操作するようGoogleに教えました。 この機能は2年以上Androidデバイスに存在していましたが、Apple OSの8番目のバージョンのリリースでのみiOSに実装されました。



起動時（6月上旬）には、限られた数のアプリケーションでさまざまな理由でアプリケーションコンテンツ内を検索できました。 米国では、Google SearchはEat24、Free Dictionary、Huffington Post、OpenTable、Pinterest、SeatGeek、Slideshare、Tapatalk、Yellow Pages、YouTube、Zillowとやり取りでき、ブラジル、ドイツ、日本、オーストラリアの一部のアプリケーションもサポートされています。 さらに、他の国を含め、他の（最も一般的で便利な）多数のアプリケーションが開発中です。 iOSアプリケーション開発者で、同様の機能を自分で実装したい場合-ブログのこの投稿の最後に、iOSでのGoogle検索との統合に関する簡単な説明があります。



Google音声投稿



Androidのアプリケーション間の相互作用ははるかに自由に実装されているため、Google Now（単純な人間の音声を完全に認識する）に最も人気のあるインスタントメッセンジャーでテキストメッセージを送信するように教えることができました。 誰に何を送信するかを伝えるだけで、残りのGoogleアプリケーションがあなたに代わってくれます：











WhatsApp、Viber、Telegram、NextPlus、WeChatは現在サポートされています。 発売時（7月下旬）に、この機能は英語のみで機能し、現在はVkとGetTaxiがロシア語で100％動作します。 パートナーのリストは徐々に拡大しており、これらの相互作用の可能性について個別に説明します。



PS：さらに以前の春には、クエリレベルで音声検索と対話するようにアプリケーションカードを教えました 。 たとえば、「shazam this song」は、スマートフォンに適切なアプリケーションと最新バージョンのGoogle検索があるかどうかを考慮すると、もちろんShazamサービスを使用して自動音楽認識を開始します* 。



* -正常に動作するには、音声Google検索をメイン言語として英語に切り替える必要があります。 Shazamをお持ちでない場合は、ロシア語をメイン言語として選択し、「Ok Google！」と発声してください。 Google検索自体は、どこかで音楽が聞こえることを認識します。 あとは、右下隅の「メモ」をクリックするだけです。











goo.glとモバイルアプリケーションの短いリンク



Goo.glリンク短縮サービスを更新すると、検索アプリケーションとモバイルアプリケーションを統合するという美しい状況を補完できます。 アプリケーションへのクロスプラットフォーム統合のサポートを追加しました。 たとえば、現在地をGoogleマップと共有したいとします。 「共有」ボタンをクリックすると、特別なAPIが短縮リンクを生成し、受信者がそれをクリックすると、魔法が始まります。 Goo.glサービスは、対応するアプリケーションがデバイスにインストールされているかどうかをユーザープラットフォームを判断し、インストールされている場合は結果を開きます。 ワンクリックでシンプル、便利。



新しいGoogleアプリのデザイン



既にお気づきのように、1999年以降ほとんど変わらないGoogleロゴは、2015年9月1日に完全に更新されました 。 新しい、クリーンでシンプルな視覚スタイルは、マテリアルデザインコンセプトに有機的に適合し、主要な製品であるGoogle検索は、現代の企業スタイルと整合する必要があります。 Google Nowを更新し、精神のマテリアルデザインに対応するだけでなく、Android 5.0および6.0が構築されている検索システムと視覚言語の両方の最新機能を完全に反映しました。



退屈なもの



Google検索のユーザーとして新しい機能に満足している場合は、当社のテクノロジーを使用するアプリケーションであなたを喜ばせる機会を実装する人々、つまり開発者について忘れないでください。 すぐには修正できない複雑なタスクがいくつかあり、新しい標準の準備や現在の作業アルゴリズムの完全な改訂が必要です。 それでも、今日の「退屈な技術」の道は休日です。



新しいトップレベルドメインのサポート



ローカライズされたドメインの開発には、インターネットのさまざまな非Unicode要素の適切な適応が必要であり、Google自体の内部で動作します。 現在、ますます多くの新しいgTLDが導入されており（Eng。Generic Top-Level Domain）、Google検索がそれらに適切に応答するようにしています。



たとえば、新しいgTLDは検索結果のランキングに影響しません。 .com、.orgドメインゾーンのサイトを探しているか、.BRANDなどの新しいサイトを探しているかにかかわらず、違いはありません。 ドメイン自体内のキーワードは、結果と配信順序に影響しません。



また、日本語および中国語の国別アルファベットのサポートも行いました。 これを確認するには、[site：みんな]を検索します。 残念ながら、サイト所有者自身に課せられた制限がいくつかあります（たとえば、検索エンジンがリンクをリンクとして適切に解釈できるように、すべてのリンクをUnicodeで指定する必要があります）。



ちなみに、検索エンジンのリンクの形式（全国的にはUnicodeを使用、またはPunycodeを介してASCIIに変換）に違いはありません。また、リンクの記録形式は結果に影響しません。



他のすべてのgTLDの質問（主に地理的にリンクされたドメイン（例：.london））は、検索結果に影響を与えないか、わずかに影響します。 ドメイン名の情報を使用せず、可能であれば結果をジオターゲットにバインドします。 多言語および多地域のサイトに関する質問については、 ヘルプセンターで説明しています。



サイトを新しいTLDに移行することは、サイトを別のドメインに移動することと同じです。これらの問題については、 ヘルプセンターで詳しく説明しています。 最も重要なことは、そのような結果の処理には時間がかかり、古い電子メールアドレスが誰かに与えられた名刺に載っている可能性があることを忘れないでください。



自動補完APIの変更



Google検索は、オートコンプリートクエリで有名です。 過去にいくつのインターネットミームと面白い写真が、自動置換によって一般に公開された人々からの奇妙な要求を引き起こしましたか？

実際、もちろん、オートコンプリートアルゴリズムは娯楽目的のためではなく、私たちの生活を簡素化するために設計されています。



長年にわたって、この機能は、一部のサードパーティ開発者がリバースエンジニアリングによって開発された非公式ツールを介して使用していた内部APIを閉じていました。 Google検索自体に関係なく、検索クエリの置換を使用できました。



文書化されていないAPIをこのように使用すると、興味深い結果が得られ、サービス間で非常に便利なバンドルを作成できることがありました。 機知に富んだプログラマーがサードパーティアプリケーションと対話するためのいくつかの内部APIを開くためにプッシュした技術を使用する方法の良い例（たとえば、Google Mapsデータと他のソースを接続することで驚くべき結果を見た後、これはGoogle Maps APIで起こりました）。



それでも、「非公式」APIのすべてのユーザーは、警告なしにAPIを変更または無効にできるという事実に起因するリスクを抱えており、現在ではこのようなケースの1つです。



検索要素としてオートコンプリートを作成しましたが、検索クエリとは別にオートコンプリートを使用することを暗示していません。 オートコンプリートデータの主な価値は、それら（データ）が検索クエリと相関しており、単独では存在しないことです。



8月10日以降、内部 API への不正アクセスを制限しています。 オートコンプリートを使用したい開発者やサイト所有者向けに、代替ソリューションを提供する準備ができています。 Googleカスタム検索エンジンサービスを使用すると、サイト検索とともにクエリのオートコンプリートのあらゆる可能性を利用できます。 既にGoogle CSEを使用しているすべての人にとって、変更は発生しません。このテクノロジーに参加したいすべての人がここで幸せになります 。







新しい検索分析機能



ユーザーはサイトではなく情報を探しています。 サイトのユーザーにとって正しい情報が多いほど、検索結果のサイトが高くなります。 情報の関連性、検索結果の関連性-これはGoogleで本当に誇りに思っていることですが、ウェブマスターについては忘れていません。



8月、Search ConsoleでGoogle Search Analyticsを更新し、新しいAPIを介してデータアクセス機能を追加しました。 これにより、ウェブマスターが重要な統計情報を便利に収集できるようになります。 既にGoogle APIを使用している場合、Search Analytics APIを操作するのは難しくありません。 このプランを初めて使用する場合は、特別にあなたのために、PythonでAPIと例を使用するための簡単なガイドを含むページがあります 。



たとえば、Google検索でサイトのデータがどれだけ新しいか、サイトのトップ10ページ、プラットフォームや地域によって制限されたすべてのクエリのトップ10などを確認できます。



この小さな実験から何が出てくるのか、おそらく新しいサービスが生まれるのか、詳細で簡単にアクセスできる統計情報によって作業を最適化した最大のサイトの評価が変わるのか、興味があります。 最も重要なことは、検索エンジンの最適化により、キーワードやジオタグの言及が無意味に増加するのではなく、 ユーザー情報の検索が改善されることを忘れないでください。 結局のところ、エントロピーを増やすためではなく、ユーザーのために努力していますか？ ;）
