C ++ロシア2019：最初のホールの無料放送と、カンファレンスの内容について少し

4月19〜20日、つまり、この週末にはすでにC ++ロシア会議が開催されます。 これらは2日間で、並行性、パフォーマンス、アーキテクチャ、インフラストラクチャソリューションなど、C ++に特化した筋金入りのテクニカルレポートのいくつかのトラックです。これらはすべて、コードの外観を改善し、より多くのことを行います。 このカンファレンスは経験豊富な開発者を対象としており、紹介レポートはありません！







まず、完全に無料のオンラインブロードキャストを視聴できます。 それは最初の日と最初の部屋 +レポート間のオンラインインタビューのみです。 どのようなレポートがカットの下で議論されます。







放送は、午前9時45分（モスクワ時間）、開始の15分前に始まり、夕方7時近くに終了することが重要です。 この間ずっと、スピーカーとのオンラインインタビューのために短い休憩でレポートを聞くことができます。 リンクは終日機能するため、最も重要なレポートでのみ開くことができます。







ビデオとプログラムを含むサイトへのリンク-カットの下。 そこで、放送されないが、ライブで会議に到着した参加者が利用できるいくつかのことについて議論します。

報告書



Nicolai Josuttis基調講演「C ++ 17-最大の "」から始まります。 Nikolaiはまさに The C ++ Standard LibraryとC ++ Templatesの著者であり、ほぼ20年間標準化委員会のメンバーです。 彼の報告書は、おそらく「未来と現代のC ++」のトピックに関する最も話題の1つになるでしょう。 レポートでは、言語の問題、その背後にあるアーキテクチャ、誰が責任を負うべきか、何をすべきかについて説明します。







2番目のレポートは、IntelのRuslan Harutyunyanからです。 Ruslan-自動運転、テクライドの分野で使用するための標準C ++ライブラリの作成に従事しています。 レポート「Modern C ++アロケーター」では、アロケーターの履歴を調べ、設計のエラーを分析し、標準ライブラリが提供する最新のアロケーターを調べ、それらが解決する問題を確認します。







3番目のレポートでは、 「コンパイル時の最適化。 DINSのKirill Tikhonovは、使用されるデータとそのアクションに関する情報をコンパイラーに送信する方法を示します。単一のコンパイル単位から、リンク時間の最適化、静的初期化、constvalおよびテンプレートメタプログラミングまで。







リアクティブC ++は、リアクティブプログラミングのチュートリアルレポートです。 Artec 3DのAlexander Schmidtと一緒に、このアプローチ（RxCppを使用）が、テストを簡素化しながら、複雑な非同期コードの操作を抽象化するのにどのように役立つかを理解します。







そして、有名なスピーカーIvan Chukichによるレポートである「Move-only C ++ design」の放送が完了しました。 Ivanは、プログラミング言語の設計とKDEへの貢献を専門とする、ベオグラード大学の数学部のPhDコンピューターサイエンス部門で働いています。 このレポートでは、コピー不可の移動専用型（「線形型」）のアーキテクチャと、これを可能にするC ++機能を扱います。 たとえば、C ++ 20または&&-修飾関数の概念を見てください。








合計：新しいC ++標準、最適化、および反応性の機能。 素晴らしい金曜日を過ごすために必要なものはすべて揃っています。
