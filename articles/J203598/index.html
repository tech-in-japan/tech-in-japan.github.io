LinuxおよびMPDサウンドステーションを構築する





リファレンスオーディオプレーヤーBryston BDP – 2の心臓部-1分に相当する15万6000ルーブルは、Intel AtomプロセッサとESI Juli @ PCIサウンドカードを搭載した標準マザーボードです。 開発者は、このプレーヤーのOSとしてDebian Linuxを選択しました。 時間があれば、既存のハードウェアから同様のデジタルソースを収集できます。必要なソフトウェアをインストールし、システムパラメーターを微調整することで、この記事が役立ちます。





はじめに



BDP – 2を作成するとき、Brystonのエンジニアはソフトウェア部分の作業にそれほど煩わされなかったと言わなければなりません。 。 実際には、標準カーネルにはかなり大きなサウンド遅延（11〜20ミリ秒）があり、これによりサウンドを専門的に扱うことはできません。 リアルタイムシステムでは、この遅延は約1 msであり、すでに優れた結果と考えられています。 したがって、まず、遅延を最小限に抑えるためにMPDカーネルとサウンドサーバーをRTパッチでコンパイルしてから、高精度のイベントタイマーを設定し、オーディオ送信に関連するデバイスとプロセスの最高の優先順位を設定し、最後にシステムパラメーターの調整を完了します。 一般的に、私たちはハイエンドのメンバーがすべきことをします。



古代のコンピューター（VIA Epia-MSマザーボード、VIA C3パーセント800 MHz、512 MB RAM、ESI Juli @ PCIサウンドカード）の例と、最小限に設定されたDebian 7.1ディストリビューション、つまり、グラフィックスと不要なサービスなしの例を考えます。 外部のUSBカードは一部のマザーボードでは正常に動作しない可能性があるため、別の音が聞こえるかもしれませんが、PCI / PCI-Eのみを推奨します。

RTコアのコンパイル



RTパッチは、カーネルでリアルタイムをサポートするように設計されており、「ソフト」ではなく「ハード」に対応しています。 違いは、ソフトリアルタイムシステムでは目的の実行時間をわずかに超過できることですが、ハードリアルタイムシステムではそのような超過は許容されません。 パッチはバニラコアに重ねられているので、コンパイルに必要なものをすべて置いて、カーネルとパッチをダウンロードします。 



# apt-get install kernel-package libncurses5-dev fakeroot build-essential pkg-config wget gcc autoconf git # mkdir kernel && cd $_ # wget bit.ly/149djQn # wget bit.ly/14rDBvQ


開梱してパッチを適用： 



 # tar xjf linux-3.8.13.tar.bz2 && cd linux-3.8.13 # bzcat ../patch-3.8.13-rt11.patch.bz2 | patch -p1


次に、menuconfig / nconfigで、オプションの[プロセッサの種類と機能]-> [プリエンプションモデル]-> [完全プリエンプティブカーネル（RT）]を有効にし、カーネルをアセンブルします。 



 # CONCURENCY_LEVEL=3 fakeroot make-kpkg --initrd --append-to-version=-rt kernel_image kernel_headers


CONCURENCY_LEVEL = 3の代わりに、プロセッサコアの数+ 1に応じて、数値を入力できます。

インストールと再起動： 



 # dpkg -i ../*.deb # reboot

MPDの準備



そもそも-それは何ですか？ 彼らがたくさんいるのに、なぜ別のプレーヤーが必要なのですか？ 実際、MPDはクライアントサーバーアーキテクチャを備えたデーモンであり、次のような非常に興味深い可能性を開きます。



  • 他の多くのグラフィックプレーヤーとは異なり、Xがなくても十分に機能します。したがって、コンピューターに落ちたり、コンピューターに完全に存在しない場合でも、音楽は再生されます。
  • これはクライアント/サーバーアプリケーションです。必要に応じて、グラフィカルインターフェイスから制御できます。
  • MPDはクライアント/サーバーだけでなく、ネットワークでもあります！ そのため、ヘッドレスサーバーに配置し、ネットブック、別のコンピューター、スマートフォンやタブレット（iOS / Android）からでも静かに操作できます。




MPD自体は、FLAC、ALAC、WAV、MP3、OGG、ストリーミングオーディオ再生、ギャップレス再生（一時停止のない再生）のサポートなど、多くの機能をサポートしています。すべてをリストするわけではありません。

公式Webサイトからソースを取得し（下記の理由により、バージョン0.17.1が必要です）、アセンブリに必要な依存関係を配置します。 



 # wget bit.ly/14wxPtj # tar xjvf mpd-0.17.1.tar.bz2 # cd mpd-0.17.1 # apt-get build-dep mpd # apt-get install libcdio-paranoia-dev


その後、原則として、ディレクトリに移動してautogenコマンドを入力してからmake ...を実行できますが、急いで移動することはありません。そうしないと、MPDが既定のオプションでコンパイルされて、動作しない可能性があります。 また、RTパッチをMPD自体に適用する必要があります。MPDは、ストリームの優先順位を制御することで音質を改善するように設計されています（無駄にRTコアをコンパイルしましたか？）。 プレーヤー自体はパッチよりも速く開発されているため、MPDバージョンは正確に0.17.1である必要があります。 パッチをダウンロードして適用します。 



 # wget bit.ly/10kbsHY -O mpd-rtopt.diff.gz # gunzip -c mpd-rtopt.diff.gz | patch -p1


これで、アセンブリオプションを選択できます。 もちろん、これは好みの問題ですが、次のオプションで収集しました。 



 # ./autogen.sh CFLAGS="-O2 -mtune=`uname -m`" --enable-alsa --enable-rtopt --enable-id3 --enable-sqlite --enable-audiofile --enable-flac --enable-cdio-paranoia --enable-lsr --disable-oss --disable-pulse --disable-jack --disable-ipv6 --disable-inotify # make && make install


いくつかのオプションをさらに詳しく考えてみましょう。



  • --enable-alsa-ALSAサポートを有効にします。
  • --enable-rtopt-実際には、パッチを適用したものに対して。
  • --enable-id3-ID3タグのサポート。
  • --enable-sqlite-内部MPDデータベースのSQLiteサポート。
  • --enable-audiofile-WAVファイルのサポート。
  • --enable-flac-FLACのサポート-品質を損なうことのない圧縮形式。
  • --enable-cdio-paranoia-オーディオCD;
  • --enable-lsr-その場でサンプルレートを変更するためのサポート。
  • --disable-oss-ALSAを使用するため、OSSは不要です。
  • --disable-pulse、-disable-jack-ALSAのラッパーも役に立ちません。
  • --disable-ipv6-IPv6がない場合、なぜ有効にするのですか？
  • --disable-inotify-リソースの消費を抑えます。




そして、アセンブリとインストールが正常に完了したら、オーディオシステムのセットアップに移りましょう。

必要に応じて（および対応するオプションでMPDがコンパイルされた場合）、たとえば、HTTP経由のストリーミングオーディオを有効にできます。後者の場合、次の行を構成に追加します。

Mpd.conf構成ファイル



システムのチューニングも必要です。 最初の手順は、次の内容で/etc/security/limits.d/mpdファイルを作成することです。 



 @audio - rtprio 99 @audio - memlock unlimited @audio - nice -19


このファイルは、デーモンの優先度制限を事実上無効にします。 次に、HPET設定を変更する必要があります（これはクロックジェネレーターであり、多くのかなり古いマザーボードではデフォルトでは使用されません）。 これを行うには、ファイル/ etc / default / grubの行を編集します-同時に、RT-kernelに関連し、将来必要になるthreadirqsオプションを追加します。 



 GRUB_CMDLINE_LINUX_DEFAULT="quiet clocksource=hpet threadirqs"


HPETは、DACへのより正確な（時間内の）信号に必要です。 ビットストリームを適用するときの数マイクロ秒（またはナノ秒）の差はごくわずかであるため、それに注意を払う価値はありませんが、違います。 少し遅れて、2つが先に来ました-ハイエンドステレオシステムでは、人間の耳で違いを感じることができます。 HPETを使用すると、このようなシナリオの可能性が減少します。

また、sysctlを使用していくつかのカーネルパラメーターを構成しましょう。 お気に入りのエディター/etc/sysctl.confで開き、次の行を追加します。 



 #   ,         vm.overcommit_memory = 2 vm.overcommit_ratio = 50 vm.swappiness = 10 #     HPET dev.hpet.max-user-freq = 2048


最後のパラメーターに加えて、rtcについても同じように変更する必要がありますが、sysfsにあるため、rc.localに登録する必要があります。 



 echo 2048 > /sys/class/rtc/rtc0/max_user_freq


rtirq-initパッケージをインストールします。 オーディオ機器に関連するIRQストリームの優先度を上げるスクリプトが含まれています。 



 # apt-get install rtirq-init


インストール後、/ etc / default / rtirqファイル、つまり優先度を上げるIRQストリームのリストを編集する必要がある場合があります。 



 RTIRQ_NAME_LIST="rtc snd usb i8042"


そして最後に-PCIサウンドカードがある場合、レイテンシタイマー（他のカードがバスにアクセスしている場合にカードがバスにかかる時間を設定するレイテンシタイマー）を最大化し、それに応じて増やす必要があります他のPCIデバイス用。 ただし、最初にPCI IDカードを確認する必要があります。 



 # lspci | grep -i audio


私の場合、IDは01：09.0でした。したがって、待ち時間タイマーを増やすには、次のコマンドを入力します。 



 # setpci -v -d *:* latency_timer=b0 # setpci -v -s 01:09.0 latency_timer=ff


これらのコマンドをrc.localに書くこともできます。

ヘルスチェック



一般に、MPDデーモンを実行できるようになりました。 開始する前に、対応するディレクトリに音楽が存在し、プレイリスト用のディレクトリが作成されていることを確認してください。 異なるディレクトリに散在している場合は、シンボリックリンクでそれらを指すことができます。 手動開始のコマンドは次のようになります。 



 # mpd /etc/mpd.conf


しかし、パフォーマンスを確認する方法は？ 実際のところ、libmpdclientライブラリのように、最も単純なコンソールクライアントでさえMPDソースに含まれていないため、それらを自分でコンパイルするか、適切なパッケージをインストールできます。 クライアントにはパッチを適用する必要がないため、コンパイルに煩わされる特別な理由はありません。つまり、パッケージを置くことを意味します。 



 # apt-get install mpc


開始する前に、ミキサー設定を確認することをお勧めします-Debian 7.1では、デフォルトでサウンドがオフになっています。 MPDデータベースを更新し、すべての音楽をプレイリストに追加して、再生を開始します。 



 # mpc update --wait # mpc listall | mpc add # mpc play


すべてが正常であれば、音楽は鳴ります。 ストリーミングの場合、このストリームの動作を確認する必要があります。 構成で指定されたアドレスとポートを指定します。 一部のプレーヤーでは、mpd.oggファイルも指定する必要があります-たとえば、 192.168.1.5 ：8000 / mpd.ogg。





不要なものを削除



さて、今あなたはシステムをきれいにする必要があります。 一般に、これは自分で行うことができますが、いくつかの微妙な点があります。たとえば、apt-get build-depを使用してインストールされたパッケージは、削除するのは非常に簡単ではありません。 そのため、まずMPDに必要なパッケージをマークして、次のコマンドで削除されないようにします。その後、アセンブリに関連するパッケージを削除します。 



 # apt-mark manual libcdio-paranoia1 libavahi-glib1 libcurl3-gnutls libshout3 # apt-get autoremove kernel-package libncurses5-dev fakeroot build-essential pkg-config wget gcc autoconf git # apt-get remove libcdio-paranoia-dev


次に、次の3階建てのコマンドを実行します。 



 # apt-get remove $(apt-cache showsrc "mpd" | grep Build-Depends | perl -p -e 's/(?:[\[(].+?[\])]|Build-Depends:|,|\|)//g')


このコマンドは恐ろしく見えますが、完全に無害です。MPDのビルドに必要なすべてのパッケージを削除します。



MTAも削除します。なぜオーディオステーションでメールが必要なのですか？ 



 # apt-get remove exim


一般に、オーディオステーションにとって役に立たない他のパッケージは自分で削除できます。 ただし、たとえばcronやsyslogが必要かどうかわからない場合は、initスクリプトから単純に無効にすることをお勧めします。



ROモードでのルートFSのマウント



音楽がシステムとは別のドライブにある場合、ダウンロード時にルートファイルシステムを読み取り専用モードでマウントするのが適切です。 これを行う最も簡単な方法は、/ etc / fstabに適切なオプションを記述することです。 ただし、これには、すべての可変部分を他のセクションに配置するか、このデータを生成するプログラムを削除する必要がありますが、これは完全に単純ではない場合があります。

別の方法は、unionfs / aufs / overlayfsを使用することです-これらのファイルシステムでは、同じマウントポイントでROとRWの両方を組み合わせることができます-後者はRAMに配置できます。 この手法は、ライブ配信を作成するために使用されます。 これを行う方法の詳細については、 ページをご覧ください 。



MPDのクライアント



MPDの他のクライアントはありますか？ もちろんあります。 さらに、複数のクライアントを持つことは、このデーモンの機能の1つです。 それらのいくつかをより詳細に説明します。



  • Ncmpcは、かなり単純なncursesベースのMPDクライアントです。 残念ながら、プレイリストを操作することはあまり便利ではありません。
  • そして、ncursesの別のクライアントがあります-この意味でncmpcppははるかに快適です。 タグ検索があります。
  • xfceのクライアントであるxfmpcは、この環境に固有のミニマリズムと不要な機能がないことを特徴としています。
  • 別のグラフィカルクライアントであるSonataでは、とりわけ、アルバムカバーと歌詞をインターネットからダウンロードでき、多数の設定があります。 たとえば、複数のMPDサーバーでの作業をサポートします。 確かに、設定ウィンドウはあまり直感的ではありません。
  • QMPDClientは、その名前が示すとおり、Qtクライアントであり、非常に強力なプレーヤーのように見えますが、特に感銘を受けたわけではありません。 それでも、プレイリストから既に再生された曲を自動的に削除する可能性があることに注意してください。




また、コミュニケーターのクライアントに言及する必要があります。 Androidの場合、Google PlayはMPDroidとDroidMPDを見つけました。 どちらを選ぶかは好みの問題です。 2番目のインターフェイスは少し不器用に見えました。 iPhone / iPodの場合MPoDが一般的ですが、iPadの場合は専用バージョンのMPaDがあります。

最適なBIOS設定



以下に、パフォーマンスを改善するために微調整することが推奨されるBIOS設定をいくつか示します。



  • ハイパースレッディングがある場合は、無効にすることをお勧めします。
  • ビデオBIOSのシャドウもお勧めです。
  • PCIデバイスの場合、PCI遅延トランザクションオプションをオフにすると、待ち時間が長くなります。
  • 不要な組み込みデバイスをすべて取り外します。
