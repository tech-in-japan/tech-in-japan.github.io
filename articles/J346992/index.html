Microsoftストアに公開するための超完全ガイド

こんにちは、Habr！ そのような状況はよく起こります。私は何か複雑なことをしていますが、ある時点でエラーが発生します。 私はそれぞれ、混乱している事柄の問題を探し始め、しばしば必要ではないことをします。 そのため、問題は増大しています。 そして最終的に、問題は単純なものにあったことが判明しました。たとえば、管理者の代わりにプログラムを実行する必要がありました。 私だけですか？ いずれにせよ、ゲームの例を使用してMicrosoft Storeにアプリケーションを公開するための超完全ガイドを以下に共有します。 あなたの時間を節約するために猫の下を見てください！







著者に発言権を与えます。



エントリー



ミニゲームを作成するというアイデアを得た後。 「AAAおよび実生活のグラフィックス」に対する主張はありません。 ミニマリズムとローポリに触発され、古いゲームのようにパッセージの複雑さから、小さなゲームを作りたかった。 その結果、迅速に決定を下す必要のあるゲームが完成し、環境全体がプレイヤーに対して機能します-速度、カメラの配置、地形です。 おそらく既に理解しているように、開発自体に加えて、たとえばゲームのサイトを選択するなど、他の困難に直面する必要がありました。 Microsoft Storeの選択に至った経緯を説明します。



ターゲットプラットフォームとしてPCを選択し、最初のゲームを完全に無料にすることを決定したため（そして、一般的には利益を得ないことにしました）、配置には、 価格+利便性+視聴者+可用性の最適な比率のサイトが必要でした。 詳細に触れない場合は、この特性を持つ多数のサイト（有名なサイトとあまり知られていないサイト）をチェックし、いくつかのインディーズのパブリッシャー（私のような小規模なインディーズプロジェクトではなく、ミニゲームの配布に従事している） 、その後、Microsoft Storeが私に最も適しているという結論に達しました。



登録料は626ルーブル（個々の開発者向け。企業は3000 rを支払う必要がある。ところで、私が知る限り、MSには開発者の権利を取得する無料の方法があります。たとえば、学生は無料で開発者アカウントを取得できます） Windows 8以降、さまざまなデータから判断すると、このサービスは人気を奪われていないため、ステレオタイプを忘れることができます）は非常に魅力的です。 特に初心者向け。 利便性に関しては-主観的にですが、すべてが私に合っています。



ビジネスに取り掛かりましょう -この記事は、Microsoft Storeでゲームをリリースしたい人や、「方法」を参照したい人に役立ちます。 この記事では、ここでエラーが発生することが多いため、出版ガイドを詳しく説明し、非常に注意する必要がある点に焦点を当てて、詳細にステップごとに試していきます。



ツール、材料など：



  • Unity Personal 5.6.1f1
  • Visual Studioコミュニティ2015
  • UWP SDK Universal 10（10.0.10240.0）

  • 「Development Center for Windows」の開発者アカウント
  • 開発センターで予約されている製品名（これは公開時に行うことができます。説明します。事前に行った場合は、10ポ​​イントを自分に追加できます。将来、これらのポイントは不要になります）
  • また、作業中にWindows 10で開発者モードを有効にして、追加機能（オプション->更新とセキュリティ->開発者向け）を使用できるようにすることもお勧めします


はじめに



ステップ0。公開プロセスを開始する前のハイライト。



1.プロジェクトの名前を決めて予約します（以前に予約していない場合）。



製品名を予約するには、開発センターのダッシュボードに移動します。 自動的に概要ページに移動するはずです。 何らかの理由でこれが発生しなかった場合は、サイドメニューを使用する必要があります。



[概要]ページで、[新しいアプリケーションの作成]ボタンをクリックします。







その後、アプリケーションの名前を入力し、その可用性を確認して予約するように求められます。







成功すると、緑色のチェックマークが表示されます：







ただし、名前が使用できない場合（たとえば、誰かが予約した場合）、確認または予約しようとすると、次のように表示されます。







あなたとの予約が成功すると、アプリケーション概要ページにリダイレクトされます：







2.アセンブリとゲームの情報を入力します。 これを行うには、[ファイル]-> [ビルド設定]に移動します。







開いたウィンドウで、アセンブリのシーンを追加する必要があります（このためには、 [開いているシーンを追加]を使用するか、単に[ アセット]から[ ビルド中のシーン]にドラッグします）。次に、プラットフォームをWindowsストアに変更します（プラットフォームリストからWindowsストアを選択し、プラットフォームの切り替えをクリックします）。アセンブリパラメータ（SDK、ターゲットデバイス、UWPビルドタイプなど）を表示します。その後、そのままにするか、必要に応じて変更します。 これが完了したら、[ プレーヤー設定]ボタンをクリックします。







次に、 プレーヤー設定が開きます。ここで、会社とゲームに関する情報を入力する必要があります。







ご注意
執筆時点では、204800バイトのアイコンの重量に制限がありました。 したがって、アイコンがこのサイズを超える場合、Visual Studioでアプリケーションパッケージを作成するときに、次のようになります。









Playerの設定が完了したら、ビルドウィンドウ（[ファイル]-> [ビルド設定]）に戻り、[ビルド]をクリックしてビルドのフォルダーを選択し、ビルドが完了するまで待ちます。



ステージ1. Visual Studioでのアセンブリと認証



Unityでプロジェクトをまとめると、ビルドフォルダーに次のようなものが表示されます。







ここで、Visual Studioで拡張子.sln（Microsoft Visual Studio Solutionとも呼ばれる）のファイルを開く必要があります。



ファイルを開いたら、「ソリューションの構成」と「ソリューションエクスプローラー」に注意する必要があります。



構成にMasterを配置する必要があります。







そしてブラウザで、次の手順を再現する必要があります：（私の場合）RMB on（私の場合） New Unity Project（Universal Windows）-> Store-> Create application packages ：







または： プロジェクト->ストア->アプリケーションパッケージの作成 ：







その後、次のウィンドウが表示されます。







その中で、すべてをそのままにして（つまり、[はい]をクリックして）、[次へ]をクリックして続行する必要があります。



その後、アプリケーションの名前を選択または予約するように求められます。 すでに予約している場合は、選択して「次へ」をクリックします。







そうでない場合は、適切なウィンドウに名前を入力し、「予約」をクリックします。







その後、リストが自動的に更新され、そこにアプリケーションの名前が表示されます。



次に、設定の選択肢が表示されます。 主なことは、各構成にマスターがあることをもう一度確認し、他のすべてをそのままにして「作成」をクリックすることです。 組み立てには時間がかかることがありますので、ご心配なく。







次に、選択を行う必要があります。 または、認定セットを実行して、ゲームがMicrosoft Storeを通過するかどうかを確認します。 または、すぐにMicrosoftストアに送信してみてください。 ただし、ここでの選択は相対的です。なぜなら、ゲームを認証できない場合、公開が許可されないからです。 違いは時間だけです。 認定に数分かかるか、最大3日間（すべてのチェックを含む製品を公開するプロセスには非常に時間がかかります）のいずれかを行うことができます。







それでも証明書セットを実行することに決めた場合は、数秒後に次のウィンドウが表示されます。







[次へ]をクリックするだけで、その後認証プロセスが開始されます。 検証が完了すると、結果が表示されます。 すべてがうまくいけば、美しい緑色の碑文「合格」が表示されます：e







「Not Passed」と表示された場合、落胆しないでください-これは起こります。 「ここをクリックして結果を表示する」をクリックし、問題のあるポイントを見つけてから、問題を自分で解決するか、テーマコミュニティのヘルプを使用してください。



たとえば、この場合、アセンブリ中にマスターではなくリリースを選択したため、検証に合格しなかったため、API検証に合格できませんでした。







ステージ2.ショップ



認証セットによる検証が完了したら、APPXUPLOADファイルに移動します。







このファイルはストアに送信する必要があります。 これを行うには、「概要」ページの開発センターに移動し、「提出を開始」をクリックする必要があります。







その後、送信ページが開きますので、もう一度「 送信を開始 」をクリックする必要があります。







「Sending 1」を形成しました。 ストアに送信するには、最初の5ポイントが「完了」していることを確認する必要があります。







記事をまったく説明しないために、いくつかのポイントを簡単に説明します。



「 価格と利用可能性 」-これは主に市場（ゲームが利用できる国）、価格、公開日、試用版（例：有効期間付きの無料試用版）の選択です。



「 プロパティ 」-ゲームのカテゴリとサブカテゴリ、システム要件、パラメータなど。

「 年齢カテゴリ 」-ゲームの年齢カテゴリを決定するための国際年齢評価連合（IARC）の小さなテストフォーム。 また、Microsoft Storeで公開するゲームが別のサイトで既に公開されており、International Age Rating Coalitionを受け取っている場合は、インポート評価を使用できます。 テスト自体は、「はい」または「いいえ」の選択に要約されるいくつかの質問で構成されています。



テストに合格すると、スコア（ESRB、PEGI、RSVRなど）が表示されます。



「 パッケージ 」-このセクションでは、準備されたAPPXUPLOADファイルを呼び出してロードします。







ダウンロードの最後に、利用可能なプラットフォームが同じページに表示されます。 Windows 10を搭載したPCでのみプロジェクトをリリースすることにしたので、余分なボックスのチェックを外しました。







「 ストア内の説明 」-これは少し調整する必要があります。 このセクションから追加情報を引用します。



「サポートされている各言語で、ストアの説明に表示されるテキストと画像を提供します。 この情報を個別に入力するか、すべての説明データを含むCSVファイルをエクスポートし、それを変更してインポートし、すべての新しい情報を一度に入力できます 。



最初の出版物では、CSVファイルを使用して説明を行うことにしました。 この記事では、この方法を簡単に説明しますが、少し退屈です。 誰かがCSVファイルを使用してやりたい場合は、 この記事をご覧ください。



「 言語の追加/削除 」 をクリックします。







その後、「 追加言語の管理 」をクリックします。







そして、説明する対象の言語を選択します。







その後、「 保存 」をクリックして、「 送信 1」ページに戻ります。 私の場合、「ストアの説明」セクションでは、パッケージのダウンロード段階でシステムが自動的に検出した英語と、私が選択したロシア語の2つの言語が表示されます。 説明の最後に、「未完了」マークが「完了」に変わります。







説明を追加するには、選択した言語をクリックして、ページ上のすべての情報を入力します（ロシア語がロシア語の場合、英語が英語の場合など）。









その結果、次のようになります。







「 認証ノート 」は、テスターが配送を使用して理解するのに役立つテスター向けの情報です。 たとえば、テストアカウントの名前とパスワード、非表示またはブロックされた機能にアクセスするためのアクションなど。 この項目はオプションであり、必須ではありません。



これで、「 ストアに送信 」をクリックするだけで、その後、公開を待つ時間が開始されます。







完了後、申請審査の「 認証プロセス 」のステータス







素敵な「 ストアプレースメント 」に変わります。これは、公開パスがついに終わったことを意味します。







将来、ゲームで何かを変更したい場合は、アプリケーションの概要を送信するのとは反対に「 更新 」をクリックするだけで十分です。



おわりに



結論として、私はすべての開発者に良いゲームを作ってもらい、新しい地平線を実験して開こうとすることを恐れないようにしたいです！
