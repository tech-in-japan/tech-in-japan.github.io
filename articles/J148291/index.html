14番目のキツネを再び飼いならす：正しいデフォルト設定

Firefox 14（Windows）にアップグレードした後、改善の良い印象は戸惑いに置き換えられ、問題の認識：デフォルトのprefs.js設定ファイル\ prefは以前は4番目から13番目までのすべてのバージョンで機能し、ロードが停止しました。 さまざまな実験が設定されましたが、すべて無効でした。フィッシング対策、自動更新、その他の味の悪さの兆候が含まれています。



すべてのファイルで文字列「defaults」を検索しますが、多くのファイルがありますが、最初の注意はxul.dllにあります。 同じものを探して、オリーで開きます。 そして今-興味深い詳細が表示されます 

... 100196C2 MOV ECX,xul.10BE1CF0 ASCII "greprefs.js" 100196D5 PUSH xul.10BE1CFC ASCII "defaults/pref/*.js$" 10019744 PUSH xul.10BB0D28 ASCII "PrfDef" 10019799 PUSH xul.10BE1D10 ASCII "defaults/preferences/*.js$" 10019849 PUSH xul.10BB0EF4 ASCII "PrefDL" ...
そしてまだここに： 
 1073043C MOV EDI,xul.10BE1B94 ASCII "defaults/preferences/*.(J|j)(S|s)$"


デフォルトで設定ディレクトリを作成し、そこにprefs.jsを転送しようとしています。 立ち上げ中...はい！ すべての設定が整っています。 そのため、js設定ファイルはMozilla Firefox \ defaults \ preferencesにあります。 ディストリビューションのdefaults \ prefがまだあるのは不思議ですが、channel-prefs.jsはそこでは動作しないようです。 しかし、ポイントではなく、そのままにしておきました。 次に、Linuxを調べました。すでに10番目のFoxで、defaultsディレクトリに設定（prefsではなく）が含まれていることがわかりました！ Mozillaが（2度目に）馴染みのある定評のある物事の順序を破る理由に対する質問への答えです-異なるプラットフォームでのレイアウトの統一のためです。



そして、よく知られている推奨事項、批判、および私自身の研究に基づいて作成された私の設定ファイルがあります。 そして、はい-広告に対してimglikeoperaプラグインを使用します。対応する設定の一部は彼向けです。
