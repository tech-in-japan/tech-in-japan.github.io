Githubは、Roskomnadzorとの協力により、GNUプロジェクトをホストすることは受け入れられないと宣言しました





Free Software Foundation（FSF）とGNUプロジェクトは、GNUの倫理原則に準拠するためにコードホスティングの4つのサービスをテストした結果を公開しています -まず、これは機密性と情報の自由です。



Savannah 、 Gitlab 、 Github 、 Sourceforgeのサービスを確認しました 。



その結果、各サービスは次のいずれかの評価を受けました。



F-ホスティングに受け入れられません。



C-ホスティングに受け入れられるすべての基本的な基準を満たします。



B-推奨ホスティング。 それはすべての基準を満たし、クライアント側でフリーソフトウェアのみを使用し、広告を含むユーザーを追跡せず、非フリーライセンスの使用を奨励しません。



A-優れたホスティング。 上記のすべては、JavaScriptをオフにして正常に動作し、クライアントおよびサーバー側でフリーソフトウェアのみを使用し、GPL 3+ライセンスの使用を推奨し、「オープンソフトウェア」ではなく「フリーソフトウェア」という正しい用語を使用し、フリーソフトウェアの動きのアイデアを明確に支持します。 GNU / Linuxに関しては、GNUプレフィックスなしのLinuxという用語の使用を避けます。



A-非常に優れたホスティング。 上記のすべてに加えて、許可なしにファイルをダウンロードする許可、訪問者のログなし、オンラインサービスの電子フロンティア基金の推奨事項の順守、WCAG 2.0標準の順守、サーバーで収集されたすべてのデータは機械可読形式でエクスポートされます。



チェックの結果に基づいて、サバンナホスティングのみが評価Aを受け取りました。 GitlabのホスティングはC格付けで許容できる最小のものとして認識されていますが、GithubとSourceforgeはFの恥ずべき評価を受賞しています。



特に、Githubは次の2つの理由でCの評価が高くなりません。



  1. JavaScriptを無効にすると、重要なサイト機能は動作しません。
  2. RoskomnadzorおよびExport Restrictionsセクションに示されているように、特定の国の情報がすべての国で利用できるわけではありません。




2014〜2016年、GithubはRoskomnadzorから、クリミアを訪問するための自殺方法とルールに関するコンテンツの削除を要求する5つの通知を受け取りました。 Githubは自発的にブロックすることを認めます：



「Githubは、ロシア連邦で違法であると認められている特定のコンテンツの検閲について、通信、情報技術、マスメディア（Roskomnadzor）の監督を求める連邦政府からの要求を随時受け取ります。 常にこれに同意するわけではありませんが、ロシアのユーザーの多くが引き続きGithubとその共同作業にアクセスできることを保証し、ソフトウェアを開発するために、リクエストを受け取った場合、特定のコンテンツをブロックする必要がありますRoskomnadzorから。 これを行うたびに、公式リクエストを発行します。これにより、リポジトリがブロックされます。



私たちはそのようなインターネット検閲を懸念しており、そのような政府の要求を社会的に透明にします。」


このポリシーは、Githubの実際的な立場はもちろん理解できるものの、GNUの倫理的完全性について疑問を投げかけています。 Roskomnadzor は 、子供の自殺を促進するために、禁止サイトの登録にGithubを繰り返し追加しています。
