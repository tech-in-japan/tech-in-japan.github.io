よくある質問



ユニキャスト-1台のコンピューター（つまり1つのIPアドレス）へのブロードキャスト：これはユニキャストブロードキャストと呼ばれ、

マルチキャスト -動的に変化するクライアントのグループにブロードキャストします。このグループはいつでもブロードキャストに参加したり、ブロードキャストから切断したりできます（つまり、複数のIPアドレス）。これはマルチキャストブロードキャストと呼ばれます。

VLC -VideoLAN Client（誰もが見ているように、彼はプレイヤーです）

VLS - VideoLAN Server （ほぼ完全にVLCに転送された）、VLC機能に加えて、IPv4およびIpv6で送信されるストリーミングビデオをブロードキャストします。

1.衛星デコーダー（DVB-S）または従来のデジタルテレビデコーダー（DVB-T）から信号を受信する。

2. MPEGでエンコードされたキャプチャカードの信号を受信します。

ライブ動画を放送するだけでなく、

SAP / SDP - SAP Mini ServerはIPv4またはIPv6マルチキャストブロードキャストをアナウンスし、VLCはこれらのアナウンスを受信し、アナウンスされたブロードキャストをプレイリストに自動的に追加します。

VLMa-デジタルまたは衛星で受信したテレビチャンネルのフローを制御するユーティリティ 。 インターフェイスは、Webサイトとして設計され、Javaで記述されています。 オーディオおよびビデオファイルのストリーミングもサポートします。 注目に値する主要な機能の中で：

  • VLCのTelnetインターフェイスを介したVLMインスタンスのシーケンス
  • アクティブストリームモニタリング
  • 動的なストリームタスク（サーバーが利用できない場合のバックアップタスクを含む）
  • グラフと統計の描画
  • メール、ICQ、IRC、MSNによる通知




チャネルインターネット/ LANの要件。



  • MPEG-4（ブロードキャスト）の場合、512Kb / sから4 Mb / s
  • 3-4 Mb / s衛星およびデジタルTVから、またはMPEG-2エンコーディングカードからMPEG-2でビデオをブロードキャストする場合。
  • 6〜9 Mb / sからDVD放送用。
