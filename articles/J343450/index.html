モノのインターネット：Arduinoとクラウド

私たちの世紀では、多くの人が独自のスタートアップを作ることを考えてきました。 しかし、興味深く有用な製品を自分で開発することは、しばしば困難です。 節電のために、クラウドを使用できます。 特に、クラウド内のIoTプロジェクトの場合、十分な数のサービスを見つけることができます。 「モノのインターネット」とは何であり、どのように生産的に使用できますか？ カットの下でこれについてすべて読んでください。







注：Hacker誌の記事の完全版の出版物シリーズを継続します。 著者のスペルと句読点が保存されました。



モノのインターネットは、単なるスマートホームやスマートビジネスではありません。 専門家は、近い将来、数十億のデバイスがさまざまな分野のさまざまな問題を解決すると予測しています。 はい、彼らはすでに多くの問題を解決しています。



たとえば、最も一般的なタスクを分析してみましょう。センサーを搭載したデバイスは、このセンサーから取得したデータをクラウドに送信します。 次の写真に、光センサー（フォトレジスター）からデータを取得するデバイスの最も簡単な例を示します。







このデバイスは、マイクロUSBコネクタから給電できます。 搭載されているGenuino MKR1000は、統合Wi-Fiモジュールです。 料金は約35米ドルです。 通常、あるセンサーを別のセンサーに置き換えても、回路が特に複雑になることはありません。 ご覧のように、ピックアップされる信号を安定させ、さまざまな干渉が影響を与えないようにする抵抗が使用されています。 しかし、これについては非常に多くの専門文献があります。 今、何か他のものについて。



Azureクラウドで動作するデバイスの次の機能図を見てみましょう。これは一般的なものと呼ぶことができます。







その上で、デバイスはIoTハブと対話します。 IoTハブは、モノのインターネットのセンターとしてロシア語に翻訳されています。 原則として、このサービスはデバイスとの対話に使用されるため、この名前はその本質を非常に伝えています。 デバイスからデータを受信し、メッセージ/コマンドを送信できます。



モノのインターネットセンターに加えて、イベントハブと呼ばれる別のサービスがデバイスと対話できます。 これらのサービスの最大の違いは、IoT Hubは数百万台のデバイスで動作し、Event Hubは数千台のデバイスで動作しますが、1秒あたり数百万のメッセージを受信できることです。 2番目の大きな違いは、IoTハブはデバイスにコマンドを送信できるのに対し、イベントハブは送信できないことです。 一般に、IoTハブにはさらに多くのオプションがあります。 ただし、イベントハブは早期に廃止されます。 すでに書いたように、非常に高い帯域幅を持っています。 多くの場合、イベントハブは、デバイスと対話するためではなく、中間サービスとしてプロジェクトで使用されます。



Azure IoT CenterとAzure Event Hubsの詳細な比較については、 こちらをご覧ください 。



IoTハブは、MQTT、WebSocket経由のMQTT、AMQP、WebSocket経由のAMQP、HTTPなどのプロトコルをサポートしています。 イベントハブは、AMQP、WebSocket経由のAMQP、およびHTTPをサポートしています。



プロトコルのルールに従ってメッセージを生成して送信するコードを作成できますが、SDKを使用することをお勧めします。 ArduinoボードからHTTPプロトコルを使用するメッセージの形成はこの記事で既に議論されました。



このコードは機能しますが、理想からはほど遠いです。 同じ記事で、IoTハブの構成とデバイスのペアリングについて説明しました。



現在、新しい機能があり、SDKの使用を検討し、ハブの構成を少し異なる側面で再検討します。 さらに、他のAzure IoTサービスの構成を検討してください。



IoTハブの作成



1つのデバイスでIoTハブを作成します。 単一のデバイスを使用する場合、IoTハブは無料で使用できます。 私たちが思いつくハブの名前。



ヒント：IoTハブには、デバイスからファイルを受信する機能があります。 この機能は合法的なハックとして使用される場合があります。 複数のデバイスからのデータは1つのメインデバイスに転送され、特定のフォーマットのテキストファイルに既に保存されてクラウドに送信されます。







デバイスツインを作成してSASを取得する



次に、実際のデバイスに一致する仮想デバイスをAzureクラウドに作成する必要があります。 英語では、このデバイスはデバイスツインと呼ばれ、ダブルデバイスを意味します。



以前は、Windowsデスクトップユーティリティのデバイスエクスプローラーを使用して、このようなデバイスを作成できました。 次のリンクからダウンロードして、最新のSetupDeviceExplorer.msi SetupDeviceExplorer.msi



見つけるか、iothub-explorerクロスプラットフォームユーティリティを使用できます。これは、NPM（Node.jsパッケージマネージャー）を使用してインストールできます。



気にする必要はありません。ハブ設定でデバイスエクスプローラーを見つけることで、Azureポータルでデバイスを二重に作成できます。







デバイスエクスプローラーで作成したデバイスをクリックすると、デバイス情報を開くことができます。 これらのうち、主キーである接続文字列を取得する必要があります。 Arduinoコードで使用します。



ArduinoボードファームウェアSSL証明書



Azure IoT認証として、ハブは2つのオプションを使用します：SSL + SASまたはSSL + X.509証明書。 しかし、Arduinoのリソースは最初のオプションにのみ十分です。 それでも、このデバイスは低電力であり、十分なコンピューティング能力がありません。



いずれにしても、SSL証明書でボードをフラッシュする必要があることがわかります。



これは、クロスプラットフォームユーティリティWiFi101 Firmware Updaterを使用するか、Arduino IDEに組み込まれている機能を使用して実行できます。



しかし、その前に、Arduino IDEでいくつかの準備をする必要があります。

他のAzureサービスに接続する



たとえば、SQL Serverデータベースにデータを保存しましょう。 Azureは多くのデータ形式をサポートしていますが、SQL Serverが最も一般的な形式だと思います。 ただし、JSONデータを大量に保存するには、おそらくNOSQL形式の方が適しています。 SQL Serverデータベースの価格は、月額5米ドルからです。 つまり、このお金で、完全に機能する2 Gbクラウドベースを取得できます。



データベース作成マニュアルは非常に膨大ですが、特別なニュアンスはありませんので、公式マニュアルへのリンクを残します。AzureポータルでAzure SQLデータベースを作成し、 SQL Server Data Toolsを使用してデータベースにテーブルを作成します 。



多くの場合、Visual Studioが既にインストールされている場合、追加のインストールは必要ありません。



データベースを編集用に開く最も簡単な方法は、Visual Studioメニューから[表示]-[サーバーエクスプローラー]を選択することです。 ここで、Azureサブスクリプションに接続し、SQL Serverオブジェクトエクスプローラーでデータベースを開くことができます。







さて、すでにこのウィンドウでテーブルを作成できます。







言及する必要がある1つのポイント。 既定では、セキュリティ上の理由から、AzureのSQL Serveryへのアクセス、およびそれに応じてそのデータベースへのアクセスはすべてのIPアドレスから閉じられます。 つまり、ユーザー名とパスワードを知っていても、データベースに接続できません。 Azureポータルにアクセスし、現在のIPを許可リストに追加する必要があります。
