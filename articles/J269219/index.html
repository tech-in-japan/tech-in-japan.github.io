Plesk 12.5ホスティングコントロールパネルの概要

最近、人気のあるホスティングコントロールパネルOdin Plesk 12.5が更新されました。 このバージョンではユーザーインターフェイスが大幅に改善され、Server Shield PlusやMagicSpamなどのセキュリティ機能が追加されました。 サーバー管理者は、更新されたインターフェイスデザイン（モバイルデバイスを含む）を高く評価し、ユーザーの操作と回復ツールへのアクセスを簡素化します。



クラシックホスティングを使用していて、十分な機能と速度がない場合、Pleskは、 VPSまたはクラウドでの通常のホスティングインターフェースの取得を支援し、サイトのメンテナンスとセキュリティを自動化するための多くの便利な追加機能を提供します。



Plesk 12.5の違いは何ですか？ 本当に便利で簡単になりました。 以前にPleskを試して別のものを使用した場合でも、このバージョンを確認することを強くお勧めします。 彼らが言うように、違いは明らかです。 新しいPleskをテストできるように、記事の最後にクラウドサーバーの試用版を提供します。







この記事では、Plesk 12.5の新機能について詳しく学び、以前のバージョンからPleskを更新するプロセスと、クリーンなサーバーにPleskをインストールするプロセスを検討します。



新機能



Plesk 12.5の新機能は次のとおりです。



  • 主な機能強化：新しいインターフェイスにより、ユーザーは単一のプラットフォームからWebサイトを簡単に展開および管理できます。 特に、PHPとデータベースの設定プロセスが改善され、増分バックアップが追加されました。 改善された移行ツールにより、古いバージョンからの移行やcPanelからの移行が容易になります。
  • サポートの強化 ：サーバーの設定経験がほとんどないユーザーにとって、Pleskサーバーのサポートが容易になりました。
  • 自己修復ツール ：多くの問題は、テクニカルサポートに連絡することなく自動的に修正されるようになりました。
  • 数十、数百のサイトの管理が簡単です 。組み込みのWordPressツールキットを使用すると、多くのインストールのテーマと拡張機能を同時に管理できます。
  • Sidekick.pro ：サポートに連絡せずに新機能に慣れるのに役立つ拡張機能。 組み込みのマニュアルは、テキスト、音声、および視覚的なキューを使用して、ユーザーにタスクを段階的に教えます。
  • Server Shield Plusを追加する機能を備えたセキュリティの更新 ：CloudFlare Web Application Firewallは、脅威がサーバーに到達する前に阻止し、ネットワークレベルで動作し、インストールや追加のハードウェアまたはソフトウェアを必要としません。 また、MagicSpamは電子メールセキュリティパッケージに含まれています。


サポートされるソフトウェアには、Debian 8、CloudLinux 7、RHEL 7、CentOS7、Apache 2.4（MPMイベント）、MariaDB 10、PostgreSQL 9.4、PHPバージョンのフルセット（5.2-5.6）、phpMyAdmin 4.4.9、ProFTPD 1.3があります。 5a、パッセンジャー5.0.10、Dovecot 2.2.18、Courier 4.16.1、Postfix 2.11.5、Roundcube 1.1.2、Nginx 1.9.2、Fail2ban 0.9.2、MySQL 5.5.41など。



モバイルサポート



モバイルデバイスからPleskにアクセスすると、適応されたコントロールパネルインターフェースが表示されます。 緊急のことは、電話またはタブレットから直接行うことができます。











リサイクルバックアップ



Plesk 12.5では、バックアップが大幅に再設計されました。 最も待望の変更は、増分バックアップのサポートです。 増分バックアップははるかに高速に行われ、占有するディスク容量は少なくなります。







自己回復



理想的な世界では、プログラムはクラッシュせず、データベースは常に一貫性を保ち、ファイルストレージは劣化しませんが、Pleskはこれがすべて起こる現実の世界で機能します。 そのため、pleskには、問題を自動的に検出して修正する自己修復ツールが組み込まれています。 Plesk 12.5では、これらのツールはLinuxとWindowsの両方で「そのまま」使用できます。







コントロールパネルから異なるバージョンのPHPをセットアップする



以前は、PleskはすでにさまざまなバージョンのPHPをサポートしており、キット内のさまざまなバージョンのPHPとともにインストールされていました。 これで、PleskコントロールパネルからこれらのすべてのバージョンのPHPを管理できます。PHPハンドラーの有効化と無効化、php.iniの編集、PHP拡張機能の設定の変更を行います。







PHPInfo


ユーザーの生活を楽にする非常に小さな機能。 おそらくすべてのPHP開発者が単一行のPHPファイルを作成した 



<?php phpinfo(); ?>


、実行してから削除するのを忘れていました。 ホスティング情報を表示する場合は、リンクをクリックしてください。







Pleskは、ファイルが表示された後に削除されることを保証します。







コードエディター



コードエディターを使用すると、サイト上の小さなエラーをすばやく修正できます。







ログをリアルタイムで表示する機能



メッセージをフィルタリングできるログをリアルタイムで表示すると、サイトのエラーを見つけるのに役立ちます。







Plesk 12.5への更新



すでにPleskを使用している場合は、 [ツールと設定] > [ 更新とアップグレードの設定]に移動し、 早期導入者リリース更新チャネルを選択します。







次に、 [ツールと設定] > [ 更新とアップグレード ]に移動し、Plesk 12.5の最新バージョンにアップグレードします。







アップグレード後、Plesk 12.5を最大限に活用できます。



サーバー作成



インストールには、Debian 8のサーバーが必要です。Infobox から InfoboxCloudクラウドまたはVPSを使用できます。 クラウドを使用すると、サーバーで使用可能なリソース（CPU、RAM、ストレージ）の量をいつでも変更し、割り当てられたリソースに対してのみ支払うことができます。 1か月の間にサーバーに必要なリソースの数を正確に把握している場合、VPSはより経済的です。



クラウドサーバーの作成方法
クラウドを注文した後、コントロールパネルに移動して [ 新しいサーバー ]をクリックします。







サーバーのユーザー名として、最初のサイトが配置されるドメインを指定します。 サーバーに必要なリソース量を設定します。 少なくとも1つのパブリックIPを設定することを忘れないでください-サーバーのアドレスと十分なネットワーク速度を設定して、「次へ」をクリックします。







Debian 8 OSを選択し、[次へ]をクリックします。







[完了]をクリックして、選択を確認します。 サーバーにアクセスするためのメールが届きます。



これでVPSまたはクラウドサーバーができたので、SSH経由で接続します。



Plesk 12.5をインストールします



DNS Aレコードで、最初のサイトが配置されるドメインを専用サーバーのIPアドレスに向けます。







サーバーで、OSを更新してcurlをインストールします。 

 apt-key update && apt-get update && apt-get -y upgrade && apt-get install -y curl




通常の設定で不要な質問をせずにPleskをインストールする場合は、次のコマンドを入力します。 

 bash <(curl -s autoinstall.plesk.com/one-click-installer)


Pleskを手動モードでインストールします
次のコマンドでPleskのインストールを開始します。 

 bash <(curl -s autoinstall.plesk.com/plesk-installer)


ライセンス契約を読むように求められます。 続行するには、「N」を入力して「Enter」を押します。







  • Pleskのインストール元を選択するよう求められます。 公式のOdinサーバーからインストールします。 これを行うには、 Nを入力してEnterを押します。
  • 製品が/ root / parallelsにインストールされることが通知されます。 N > 入力します。
  • HTTPプロキシを構成するように求められます。 N > 入力します。
  • 最後に、どの製品をインストールしますかという質問があります。 1 >を入力します。
  • ここで、製品バージョンを選択する必要があります。 N > Enter
  • オーディン製品をより良くしたいですか？ はい > 入力
  • 次に、インストールするPleskコンポーネントを選択するよう求められます。 [完全]を選択して、すべての可能性を探索できます。 3 > 入力します。 N > Enterを押します。








  • Pleskの最大インストールには、2.7 GBのディスク容量が必要です。 これが多すぎる場合は、P> Enterを押して、別のインストールオプションを選択します。 必要に応じて、N> Enterを押します。
  • 最後のステップは、経験豊富なユーザーが完全なセットからインストールするパッケージを選択することです。 必要なものがわからない場合は、 N > Enterを押します。




インストールが完了したら、次の場所でPlesk Webインターフェースを開くことができます。 

 https://_:8443
ここで、domain_nameには、DNSを介してサーバーの専用IPアドレスに送信したドメインの名前を使用します。



ログインウィンドウが表示されます。 最初のログインには、SSH経由でアクセスするために使用するサーバー管理者のログインとパスワードを使用します。







ライセンス契約が表示されます。 「同意する」をクリックして同意します







Pleskの使用方法を選択します。







デフォルトのインターフェイスを選択するように求められます。 独自の顧客サイトまたは独自のサイトをホストしている場合は、「パワーユーザービュー」を選択します。 ユーザーにPleskへのアクセスを許可する場合は、「サービスプロバイダービュー」を選択します。







次に、管理に使用するサーバーのドメイン名を指定する必要があります。 これを行うには、サービスドメインのいずれかで、DNSのサーバーの専用IPアドレスにA –レコードをダイレクトします。 Pleskの設定時にこのドメインを指定します。



デフォルトIPアドレスがサーバーの専用IPアドレスと一致することを確認します。 リストの2番目のアドレスは、内部ネットワークのアドレスです。 また、ログインにサーバーのルートパスワードを使用しないように、管理者パスワードを設定します。 この場合、Pleskを入力するためのユーザー名は「admin」になります。







管理者の連絡先情報を提供します。







Pleskは、ライセンスキーを入力するか、14日間の無料トライアルを注文するように求めます。



Pleskライセンスキーを注文する方法
メインページのInfoboxCloudコントロールパネルで、[新しいサービスを注文]をクリックします。







[ライセンスのレンタル]セクションを選択します。







必要なエディションをバスケットに追加します。 Plesk 12レビューでエディションを比較しました 。







契約条件に同意し、「チェックアウト」をクリックします







注文後、メールのPleskキーを受け取ります。







Pleskトライアルキーを取得する必要がある場合は、このリンクで入手できます。



キーを入力したら、システムユーザーを追加して、FTPおよびSSH経由でWebスペースにアクセスする必要があります。







Pleskが正常にインストールおよび設定されました。 Pleskを使用してサーバーに送信したドメインの最初のサイトに既にアクセスできます。







デフォルトのロシア語をオンにする



[ツールと設定]セクションに移動し、[Plesk Apperance]グループで[言語]を選択します。





ロシア語を選択し、「デフォルトにする」をクリックします。







コントロールパネルを終了して、もう一度入力します。







現在、デフォルトの言語はロシア語です。
