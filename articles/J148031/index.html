Switchvoxホーム-小規模企業向けのIP PBX





Switchvox SMBは、アスタリスクで最も人気のある商用ソリューションです。 あらゆる企業のニーズを満たすことができる統合通信システム。 コールセンターオペレーター向けのWebパネル、PDF形式のFAXの受信、強力なAPI 、fail2banベースの攻撃に対する保護などなど...



しかし、今日はそれについてではありません。 Digiumは、Switchvoxの3つのバリエーションをサポートしています。

  • Switchvox SOHO
  • Switchvox SMB
  • Switchvoxホーム


Switchvox SOHOおよびSMBは、中小企業向けに設計された有料製品です。 これらはわずかに異なる機能を備えており、さまざまなレベルのテクニカルサポートを利用できます。



Home EditionはIP-PBXの無料バージョンです。唯一の制限は、システムに8人までのユーザーしか存在できないことです。



新興のIT企業、つまりスタートアップを考えると、多くの場合、そのスタッフはシステムによって課せられた制限に達しません。 会計士、数人のマネージャーと開発者、リーダー-それだけです。



わずか数時間で独自のテレフォニーサーバーを展開できますが、Linux、アスタリスクの構成、およびsshの一般的な知識さえ必要ありません:)システム管理はWebインターフェイスのみで実行されます。



カットの下にこの製品の概要が表示されます...







私が言ったように、インストールは問題を引き起こしません：



  1. Digium WebサイトからシステムのISOイメージをダウンロードします。
  2. CD-ROMまたはフラッシュドライブに書き込みます。
  3. このSwitchvox / RedHat / Asteriskディストリビューションをインストールし、正しいタイムゾーンを入力することを忘れないでください
  4. インストールが完了すると、システムはDHCPサーバーによって割り当てられたシステムのIPアドレスを報告します。 特別なユーティリティを使用して自分で入力することもできます
  5. ip_address / admin経由でWebインターフェースに入り、Machine Admin> Product Registrationメニューに移動し、必須フィールドに入力します
  6. しばらくすると（ほぼ瞬時に）登録キー付きの手紙が受信トレイに届きます。 製品のアクティベーション後、8人のユーザーと8つの同時会話にアクセスできます。




ユーザーを追加する



システムで最初に必要なことは、ユーザーを取得することです。

拡張機能-拡張機能を管理します。 新しい拡張機能を作成します。

タイプを選択します：SIP電話またはSIPアダプター、[作成]をクリックします。



新しいユーザー登録ウィンドウ（クリック可能な画像）







  • デフォルトでは、システムは3桁の数字を使用します。 101などの内線番号を入力します。
  • 従業員の名前、姓。 英語で、Caller-idの正しい定義について
  • 数値パスワード。 ユーザーアカウントにログインできるデジタルパスワード。
  • 次をクリックして詳細オプションを表示します。 開いたパネルで、パスワードをSIPアカウントに入力します。
  • 残りのアイテムは変更しないでおくことができます。 Switchvoxを使用すると、使用するコーデックを選択したり、ルーティングルールを禁止したり（たとえば、携帯電話番号をダイヤルするときに拒否したり、ボイスメール、個人アカウント、通話転送などの特定のPBX機能の使用を禁止したりできます...




接続プロバイダー



次のステップでは、PBXをセットアップするときに、外部回線を追加する必要があります。

Switchvoxのすべてのバージョンでは、外部SIP / IAXプロバイダーの数に制限はありません。これは例外ではありません。



システム設定に従う-VoIPプロバイダー-新しいSIPプロバイダーを追加する







プロバイダーの名前は何でもかまいません。 ログイン、パスワード、SIPサーバーアドレス、プロバイダーから受信します。

また、CallBack拡張機能を使用する必要があります。 これは、外部コールが着信する内線番号です。 拡張キュー、IVR、または従業員番号。



外部回線（FXO、E1）を接続する別の方法は、アスタリスク用の元のボードを使用することです。 Digiumから。



システムのセットアップ-ハードウェアのセットアップ。

正しい国設定（この場合はロシア）を選択します。



システム設定-チャネル管理者。 スクリーンショットの例に従って、チャネル設定を操作します。







キュー



追加のキューを作成するには、同じキューに移動する必要があります：拡張機能-拡張機能の管理。 新しい拡張機能を作成します。 ドロップダウンリストからのみコールキューを選択します。



キューの名前を指定し、内線番号を割り当てて、呼び出し戦略を選択します。



呼び出しをキューに分散するためのルールがいくつかありますが、それらの多くはアスタリスクでよく知られています。

-順番に

-みんなに電話する

-ランダム

-すべてのコールよりも少ないコールを受けた人へ

-最も長く回答しなかった人に。



次に、キューのメンバーを指定し、追加の要件を選択します。 例：発信者が数分以上待機した場合の、待機中のコール、位置に関する通知、待機時間、別の内線へのコールの転送。 システムは英語の組み込みフレーズを提供しますが、自分のファイルをアップロードする人はいません（OGG形式）







IVR（自動受付）



設定はPBX機能-IVRエディターによって実行されます。



コールをキューまたは特定の加入者に配信する標準操作（例：「1」を押してテクニカルサポートを受けるか、回線に留まる）に加えて、IVRを使用すると、コールルーティングをより柔軟に構成できます。 午後6時以降のすべての通話を携帯電話またはボイスメールにルーティングし、忙しくプレイするなど







通話ルール



システム設定-送信ルール-新しい送信ルールを追加します。



ルールを記述するための構文は誤解を招くものではありません。 番号の一部を追加/切り取り、プロバイダーを選択します。

Switchvoxを使用すると、一部のユーザーへのモバイルへの呼び出しを禁止するなど、ルールを柔軟に構成できます。



また、いくつかの演算子、すなわち 一方（またはマイナスの残高）が利用できない場合、システムは別の通話を転送します







おわりに



PBXをセットアップする際に通過しなければならない主なポイントを調べましたが、Switchvoxはさらに多くのことができます！



  • 通話履歴を表示し、統計を収集します。 グラフ形式のレポート、およびxlsドキュメントへのデータのエクスポート。
  • ワンクリックでのサーバー設定の予約。 （構成を外部FTPサーバーにアップロードする機能を備えている
  • 呼び出しの診断。 （現在、失敗など）
  • ボイスメール 通知を受け取り、自分自身に直接メールを送信する機能
  • 各従業員の個人アカウント。 コールルールを柔軟に構成することが可能です。 たとえば、ビープ音が3回鳴った後、ボイスメールに4回鳴った後、携帯電話番号に電話をかけます




会社に8人以下の従業員がいる場合、Switchvox Homeは間違いなく必須です:) AsteriskNOW、Elastix、または「バギー」FreePBXを備えたTrixboxはありません。
