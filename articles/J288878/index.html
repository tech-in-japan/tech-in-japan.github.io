Hi10Pとは何ですか？

基本的に、Hi10Pはよく知られているH.264規格のプロファイルであり、 色の 再現に 10ビットを使用するため、 画像のセグメンテーション （たとえば、多くの迷惑なステップグラデーション）を減らし、ファイル全体の重量を減らします。 詳細に触れない場合は、このプロファイルの使用が大きな前進であると言えば十分です 。 このテキストは、Hi10Pビデオを正しく再生できるプレーヤーを選択するのに役立つことを目的としています。

元の記事にはいくつかの変更があるため、この翻訳は時代遅れと見なすことができますが、指示は引き続き機能します。



Hi10Pの見方


高品質のHi10P再生に適したソリューションがいくつかあります。

  • SMplayerをダウンロードするか、 コンソールからmplayer2を使用します
  • 最新のCCCPをインストールし、それに含まれるプレーヤーを使用します（CCCPは、表示する前にビデオを8ビットに変換します。さらに、低速のマシンではパフォーマンスが低下する可能性があります）
  • VLC Media Playerの最新ビルドをダウンロードしてインストールします（これを行うことはお勧めしません）
  • madVRの代わりに別のレンダリングを使用する場合、または追加の後処理フィルターを使用する場合*（ 例 ）、最終プレーヤーでLAVフィルターを使用できます（potplayerには字幕の表示に問題がある場合がありますが、MPC-HCまたはpotplayerを使用することをお勧めします）

    *変換しないため、madVRが10ビットに設定されていることを常に確認してください（以下を参照）
  • パフォーマンスの低下が原因でHi10Pの再生に問題がある場合は、 CoreAVC 3を使用できますが、 いくつかの 問題があり ます 。 そして、ソリューションは ffdshowのパフォーマンスを失うようです


選択してください！


Mplayer2（および拡張機能付きのSMplayer）は、ほとんどのユーザー、特にすぐに使用できる実用的なソリューションが必要なユーザーに最適です。 ただし、最高の品質を得るには、以下に示すようにMPC-HC + madVRを使用する必要があります。



Windows以外のオペレーティングシステム、たとえばGNU / Hurd / Linux / Haiku / BSD / VMX / OS / 2 / MS-DOS / Menuet / BareMetalOS / などを使用する場合、すべて同じ-mplayer2またはVLCを取得します。



Mac OS Xユーザーは、 Mplayer OSX Extendedを使用する必要があります。



Hi10Pを視聴しない方法


  • CUDA、DXVA、LAV CUVIDモードのCoreAVC、およびVLCとffdshowの安定バージョンは、10ビットデコードをサポートするビデオカードがないため、執筆時点でHi10Pをサポートしていません。
  • SVNビルド ffdshow- tryoutがHi10Pをサポートするようになりました
  • 注：Hi10PでVSFilter（別名DirectVobSub）を使用するには、8ビットへの変換が必要です。これにより、品質が著しく低下する可能性があります。


Hi10Pの利点


  • 大幅に高い圧縮レベル。これにより、ファイルサイズがはるかに小さくなります。 最初のテストでは、ファイルサイズが20〜40％削減されました。
  • 元の画像からさらに多くの情報が保存され、暗いシーンでのセグメンテーションや詳細の不備などの問題が解消されます。
  • エンコーダーの頭痛が少ない


Hi10Pの欠点


  • より速いエンコードおよびデコード速度
  • 執筆時点でのDXVAまたはCUDAのサポートの欠如
  • PS3でビデオを再生するなど、あいまいで古くなったハードウェアとの非互換性に関するいくつかの問題が...


なぜこれが必要なのですか？


テクノロジーはまだ止まっていないので、 DVD 、 Blu-ray 、 DivXでも同じことがHi10Pでも起こります。 最新のデコードデバイスは、1年前の8ビットデコーダーと比較して10ビットデコードでもはるかに高速であるため、さらに一歩前進する力があります。



Hi10Pでコーディングする方法


既にビデオエンコーディングに精通していると仮定して、「10bit depth」とラベル付けされた対応するx264ビルドを入手してください。 Linuxでは、コンパイルする前に「./configure –bit-depth = 10」を実行するだけです。



特記事項：公式のx264ビルドにはまだバグがあり、レベルを誤って変換します。 修正された非公式のx264ビルド（ JEEBのビルド ）を使用するか、公式のビルドに自分でパッチを適用することができます。



CRFを使用する場合は、少し増やします。 ビットレートを使用する場合は、わずかに減らすことができます。 リリースグループは引き続き、最適な設定を取得するために取り組んでいます。そのため、CRFをどの程度増減する必要があるかは時間だけがわかります。



リリースに「Hi10P」というラベルを付けます。 セクション/ファイルの説明でこの指示へのリンクを自由に指定して、フォーラムで発言する前にプレーヤーをセットアップできるようにすることができます。

Hi10Pおよび10ビット出力に関する誤解


Hi10Pの利点を確認するために、10ビット（つまり、 32ビットの「リアル」カラー ）モニターを使用する必要はまったくありません 。 8ビットに変換した後でも、10ビットでエンコードされたビデオは、8ビットでエンコードされたビデオと比較して、品質が大幅に向上します。 そして、これはファイルサイズの縮小は言うまでもありません。



そのようなモニターを持っている人の同じ0.01％は、10ビットビデオを出力するために不安定なドライバーでひねる必要があります。



MPC-HC + madVRインストール手順



簡単に保存して読むことができるバージョンが必要な場合は、 図の指示を参照できますが、不定期に更新されるため、古い情報がある可能性があることに注意してください。



この方法により、CCCPベータ版に比べて画像とパフォーマンスが大幅に向上します（一般に、ffdshowに基づいたすべてのもの）。したがって、Hi10Pまたは通常の1080pでビデオを視聴する場合は、次のことを試してください。 MadVRは定期的に更新され、最新バージョンのlibavをデコードに使用します。これにより、不条理な古いffdshow-tryoutよりもはるかに高速になり、それに基づいたさらに不条理なCCCPは言うまでもありません。



ステップ0：要件


  1. MPC-HC、CCCP、ffdshow、madVR、Haali、K-lite、CoreAVCなどのすべてのコピーをアンインストールしてください。 この命令にはこれ以上の要件はなく、理想的には将来は必要ないはずです。


ステップ1：ダウンロードしてインストールする


  1. MPC-HCの最新バージョンをダウンロードしてインストールします（インストーラーの.exeバージョンを選択します）。 注：32ビットのみ （x86）を選択してください！ 64ビットバージョン（x64）はmadVRと互換性がなく、動作しません
  2. HaaliのMedia Splitterの最新バージョンを入手してインストールする
  3. madVRの最新バージョンをダウンロードし、目的のディレクトリに抽出します。 抽出ディレクトリにあるinstall.batを実行します。 これ以降、このディレクトリを削除しないでください。madVRはこのフォルダーにあり、他の場所に自分自身をコピーしません。 このディレクトリの内容をインストール前のどこかに移動します （例：C：\ Program Files \ madVR）。

    注：これを行う前に、管理者としてログインしていることを確認してください！ マウスの右ボタンを押して「管理者として実行」を選択し、管理者としてログインして通常どおりに起動しないでください。


ステップ2：セットアップ


  1. MPC-HC設定を開きます。 「内部フィルター」を選択した後、次を無効にします。

  2. 「再生」を選択した後、「字幕の自動読み込み」をオンにします

  3. [字幕]で、[バッファリング時にアニメーションを有効にする]が有効になっており、[テクスチャの最大解像度]が[デスクトップ]に設定されていることを確認します

  4. 「出力」アイテムで、レンダリングとして「madVR」を選択します

  5. madVR設定に入ります（これを行う最も簡単な方法は、ファイルの再生を開始し、トレイアイコンをクリックすることです）。以下の設定を有効にします。

  1. ファイルを再生し、Ctrl + Jを押してデバッグ情報を表示します。 （出力が以下と異なる場合-出力設定を再確認してください）

考えられる不具合


  • 奇妙なブロックを見たり、 飽和に問題があるが、他のすべてが問題ない場合、上記のプログラムを使用していないか、Hi10Pを正しくサポートしていない古いバージョンを使用しています
  • 設定を変更した後は、毎回MPC-HCを必ず再起動してください。 フィルターとレンダリングの再読み込みは、プログラムの起動時にのみ発生します
  • MPC-HC内部字幕レンダラーを使用し、アニメーションまたは他の何かに問題がある場合-[字幕]設定で[バッファリング時にアニメーションを有効にする]（バッファリング時にアニメーションを許可）をオンにします。 注：MPC-HC内部字幕レンダラーは、マルチスレッドを除き、DirectVobSubと同じプログラムコードを使用します。 DirectVobSubで再生できることはすべて、MPC-HCでもできます。 それは軽くたたかれておらず、欠陥もありません。
  • madVRデバッグ情報で「libav」以外の何かを取得した場合は、次のようにフィルターを手動で追加してみてください



  • フィルターリストにmadVR iが表示されない場合は、madVRを解凍したフォルダーで「install.bat」を実行することを忘れないでください。 また、何らかの理由でinstall.batが失敗した場合は、[追加...]ボタンをクリックした後に表示されるウィンドウで[参照]をクリックし、unpackフォルダーにあるmadVR.axという名前のファイルを選択しますmadVR
  • madVRを追加した後、字幕が表示されなくなった場合は、MPC-HCのバージョンを少なくともビルド3520 （まだベータ版です）にアップグレードしてください！ または、フィルターを追加せずに、競合するフィルターの優先度を削除/下げないでください（たとえば、上記のffdshowを使用）
  • それでも字幕が表示されない場合は、64ビットではなく、32ビットバージョンのMPC-HCがインストールされていることを確認してください
  • スクリーンショットを撮るのに問題がある場合-madVRからスクリーンショットを撮る最善の方法は、キーボードのPrintScreenキーを押して、画像をmspaintなどの画像編集プログラムに貼り付けることです。

（ボーナス/オプション）ffdshowの追加


ffdshowを追加すると、オーディオデコーダー（フィルターおよびより柔軟なミキサー用）およびAVC1（H.264）、VC1、MPEG-2以外の形式を使用できるようになります。 特に、古い480pおよびさらに悪いファイル（AVC1でエンコードされる可能性が低い）の場合、フィルターを使用して品質を向上させることができます。

  1. ffdshow-tryoutsをインストールする
  2. ffdshowの設定を開き、「DirectShowコントロール」項目で「メリット」スライダーを「通常」の位置に設定します（設定を取得するには、次のパスに従ってください：スタート->すべてのプログラム-> ffdshow->ビデオデコーダー構成）

  3. 「内部フィルター」の下のMPC-HC設定に戻り、残りのフィルターをすべてオフにします。

  4. もう一度、「ステップ3：確認」を行って、ffdshowがH.264ビデオに触れないようにします
