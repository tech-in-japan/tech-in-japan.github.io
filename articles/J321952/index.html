部分的に曇り：バックアップコストの削減







ハイブリッドクラウドは、中小企業の現代ビジネスを最適化するための重要なソリューションの1つになりました。 独自のサーバーをインストールおよび保守する場合と比較して、それらは拡張が容易であり、ITインフラストラクチャのコストを最適化できます。 同時に、ハイブリッドクラウドは、パブリッククラウドに配置する場合と比較して、アプリケーションとデータのアクセシビリティを維持します。 同時に、ハイブリッドプラットフォームへの移行により、クラウドソリューションプロバイダーとその顧客にとって新たな課題が生まれました。 以前のバックアップおよび災害復旧スキームは、新しいシステムの仕様のために効果がないことが証明されています。 この記事では、Microsoft Azureクラウドプラットフォーム向けに最近導入されたアクロニスのソリューションについて説明します。



Acronis Backup Cloudは、拡張性の高いハイブリッドバックアップサービスです。 単一のセキュリティポリシーの一部として、データ保護を効果的に管理するのに役立ちます。 このサービスは、データソースの場所やバックアップのあるコンテナの場所に関係なく機能するため、あらゆる技術環境と互換性があります。



このサービスは、Acronis Backup Cloud Extension for Azureを使用して使用できます。 Azure Stackの拡張バージョンは、Microsoftによるリリース後まもなく表示されます。



Acronis Backup Cloud Extensionは 、クラウドサービスプロバイダーとその顧客を対象としています。 このソフトウェア製品は、次の主要なタスクを解決します。



-分散ファイルシステムのオブジェクトをバックアップします。

-データとアプリケーションの保証された災害復旧を実行します。

-ハイブリッドクラウドで安全なファイル共有を提供します。

-Azure仮想マシンへの移行を安全にします（WindowsまたはLinuxベースのローカルサーバーをAzureクラウドに転送します）。

-他のクラウド（Amazon WebServicesなど）からAzure仮想マシンにデータとアプリケーションを転送するために使用されます。

-Azure環境の既存および新規の仮想マシン（LinuxまたはWindows）のバックアップを提供します。



Acronis Backup Cloud Extensionは、クラウドサービスプロバイダーにMicrosoft AzureおよびAzure Stackプラットフォームにハイブリッドバックアップスキームを使用する機能を提供します。 これにより、管理者は、どのデバイス、どのデータ、どの物理メディアに書き込むかを長時間計算する必要がありません。 バックアップ計画は一度構成され、より高い抽象化レベル（ハイブリッドクラウド）で表示されます。 このアプローチにより、人的要因の影響が軽減され、スペース不足の問題を忘れて、主要なタスクに集中できます。



提案されたハイブリッドバックアップスキームは、そのロジックにおいて普遍的です。 固定およびモバイルデバイスを使用して、あらゆる規模の物理環境および仮想環境で同等に機能することにより、データ保護の度合いを高めます。 今日、モバイルIT環境の保護には特別な役割が与えられています。 ほとんどすべての現代企業は、ビジネスに不可欠な操作でモバイルデバイスを積極的に使用しています。









Microsoft Azureは単なるクラウドではなく、相互接続されたブランド製品のプラットフォーム全体です。 それらの統合により、独自の隠れた特異性を持つ独自のエコシステムに急速に発展しています。 今日、Microsoft Azureはビジネスアナリティクスで使用されており、「ビッグデータ」の分野での厳しいコンピューティングやさまざまなソリューションに使用されています。 アクロニスは、Microsoftパートナーとして、Active Directory、Exchange、Hyper-V、Office 365、SQL Server、SharePointなど、すべてのコンポーネントのデータ保護方法を開発しています。



Acronis Backup Cloud Extensionは、単一のコンソールを使用してバックアップ操作の集中管理の原則を実装しています。 クラウドプロバイダーは、サブスクリプションを設定し、Azure Stackポータルを介して共有バックアップサービスを展開できます。 Azure PowerShell APIを使用してバックアップエージェントをインストールし、単一のAcronis Backupコンソールからバックアップを管理することもできます。



ハイブリッドクラウドはストレージにカスタムサブスクリプションを使用するため、バックアップストレージボリュームの管理は非常に簡単です。 十分な空き領域がない場合は、ストレージをアップグレードせず、必要な量をクラウドから数回クリックするだけで追加します。 さらに、Acronis Storage Gateway for Microsoft Azureを使用すると、データをMicrosoft Azureコンテナーに直接バックアップできます。

Microsoft Azureクラウドサービスプロバイダーがアクロニスのバックアップソリューションを接続して顧客に提供するのに数分かかります。



結論：



  1. ハイブリッドクラウドは、バックアップの作成と保存を最適化するなど、ITインフラストラクチャコストの削減に役立ちます。
  2. 最新のクラウドプラットフォームは密接に相互接続されたサービスのセットであり、その保護を非常に具体的なタスクにします。
  3. 必要な技術的詳細を明らかにした関連会社のみが、独自のハイブリッドクラウドの効果的なバックアップソリューションを作成できます。
  4. バックアップサービスは、クラウドプロバイダーがわずか数分で有効にできます。
  5. ハイブリッドバックアップスキームは汎用的であり、単一のコンソールからすべての操作を管理できます。
