この記事では、オペレーティングシステムのインストール後にコンピューターをセットアップする技術（前後）について説明します。テスカと、jinsanというニックネームを持つ優れた管理者を探しました。



説明されている手順は、純粋に助言であり、かなり一般化されています。 各段階でどのように進むか、どのソフトウェア製品を使用するかは、誰にとっても個々の問題です。 各ステップは簡潔ですが簡潔にレイアウトされているため、理由と理由は明確です。 アクションのシーケンスは、不要な動きを少なくするように最適化されています（「勤勉な管理者を信じないでください...」©）



この記事は、さらなるアクションのテンプレートとして利用できます。 この設定は、クライアントコンピューター（ほとんどの場合-職場）で使用するように設計されており、アメニティ/アメニティは提供していません。

BIOSセットアップ



  1. ダウンロードの優先度

    外部デバイスからの起動を防ぐには、最初の項目のみを残し、他のすべてを無効にすることをお勧めします。

    1. HDD
    2. 無効またはCD-ROM
    3. 無効またはネットワーク


  2. BIOSパスワード

    BIOSのロードの優先順位の変更を防ぐため。 パスワードを入力することをお勧めします。この場合、開示するのは残念です。 機会がある場合-ホットボタンを使用してインストールし、ブートシーケンスを選択する必要があります（以下を参照）。
  3. ホットボタンによる優先ダウンロード

    ほとんどのBIOSでは、ホットボタンを押すことで起動の優先順位を変更できます。 一部では、F12、F8です。 この機能のパスワードを設定できる場合は、パスワードを残しておくことをお勧めします（将来、実行する必要のあるアクションは少なくなります）。
  4. 不要なデバイスを無効にする

    たとえば、LPTおよびCOMポート、USBキーボードおよびマウス、ゲームポートのサポート。 もちろん、これのいずれかを使用する場合...


HDDの準備



私の練習では、通常、ハードドライブを3つのセクションに分割します。

  • C：（システム） -システム、ブート。 コンピューターの使用目的に応じて-20〜50Gb
  • D：（データ）は、ユーザーデータを含む最大のセクションです。 使用可能なすべてのスペースを占有します（Eを除く:)
  • E：（バックアップ）-C：ドライブのバックアップが保存される最後のパーティション。 サイズ-ドライブCの1.5〜2倍：


システムパーティションにはユーザー情報はなく、オペレーティングシステムのみが存在します。



すべてのファイルシステムはNTFSです。

ドライバー



必要なすべてのドライバーをインストールします。



ドライバーを備えたネイティブディスクを読み取ることができなかった、または他の何かがあるが、ネットワークのドライバーが見つからなかったことが何度かありました（Windowsの役割...）。 このような状況では、同じSlaxを備えたLiveCDが非常に役立ちます。



最初のバックアップ



すべてのドライバーをインストールした後（デバイスマネージャー（devmgmt.msc）に疑問符や感嘆符が表示されないようにしてください）、最初のバックアップの作成に進みます。



最初に、E：ドライブに使用されるすべてのドライバーを、たとえば、ドライバーフォルダーにコピーします。



次に、システムを再起動してHirenから起動します。 優先バックアップウィザード（Norton Ghostを使用）を選択し、最初のパーティション（C :)の完全バックアップをE：ドライブに作成します。 アーカイブに名前を付ける方法はあなた次第で、私はそれをC_Driversと呼び、説明ではインストールされているもの（この場合はドライバー）を示します。



管理者パスワードとユーザー



すぐに、ここで説明するほとんどすべてのアクションは、[スタート]-> [実行]から呼び出されることに注意してください。 それらのほとんどはコントロールパネルからアクセスできますが、これは長い時間であり、まだやることがたくさんあります。



（Start-> Runを介して）lusrmgr.mscを実行し、ユーザーで管理者パスワードをお好みのシークレットパスワードに変更し、ユーザーuserも追加します。 彼に単純なパスワード、たとえばユーザーを設定することをお勧めします。



残り（他のユーザー、権利）-ポリシーの裁量と要件により、一般的なオプションを検討しています。



外観と通知



前の段落の実装中に、セキュリティセンターウィンドウが複数回ポップアップし、すべてが不良であり、何も機能しないことを確信しています。 気にしないように、すぐにオフにしてください。 マウスでクリックし、左側に表示されるウィンドウで、「通知の表示方法を変更する」（またはそのようなもの）をクリックします。 すべてのジャックドーを削除し、これらのウィンドウを閉じます。



次に、可愛さと特殊効果を削除します。[Win +一時停止]、[詳細設定]タブをクリックし、一番上のボタンは[パフォーマンス]です。 表示されるウィンドウの[視覚効果]タブで、[最高のパフォーマンスを保証する]を選択します。



次に、3つの大きなボタンを使用してタスクマネージャーを祖国に戻します。コントロールパネルの[アカウント管理]に移動し、[ウェルカムウィンドウの変更]を選択します。 すべてのジャックドーを削除し、ウィンドウを閉じます。



フォルダの表示モードを変更します。 エクスプローラーを開き、表示モードをテーブルに設定します。 便宜上、最初の列を少し伸ばすことができます。 次に、[ツール]-> [フォルダオプション]-> [表示]を選択します。 簡単にファイルを表示し、各フォルダーの表示設定を記憶し、保護されたシステムファイルを非表示（確認）、登録されたファイルタイプの拡張子を非表示にします。 システムフォルダの内容を表示し、隠しファイルとフォルダを表示し、サムネイルをキャッシュしません。 変更を適用し（[適用]ボタン）、[すべてのフォルダーに適用]をクリックします。



必要な言語を追加し、それらを切り替える便利な方法を選択します



安全性



最初のものはオートランです。 gpedit.mscを実行します。 ユーザー構成->管理用テンプレート（通常、ツリーの最後のアイテム）->システム。 その中に「スタートアップを無効にする」という項目があります。 有効に設定し、すべてのドライブに適用します。 わかった



いくつかの不要なサービスをオフにします：Run-> services.msc。 次のサービスでは、スタートアップモードを「無効」に設定します： Windows Update 、リモートレジストリ、Windowsタイム、Windowsテーマ。 サービスを利用してさらに削減できます（最初は、クライアントコンピューターに13〜15のサービスのみを残しました）。



ドライブCのユーザーへの書き込みを拒否する：ドライブCのプロパティの[セキュリティ]タブで、ユーザーに書き込み権限がないことを確認します。 ご注意 このパラメーターを変更するときは、追加のパラメーターで子フォルダーの権限を置き換えるチェックボックスが不要であることを確認してください。そうしないと、Documents and Settingsフォルダーの設定が満たされ、最も簡単な方法はすべてを再配置することです。



用途



主なアプリケーションをインストールします：アンチウイルス、アーカイバ、ブラウザ。



好みに合わせて追加のアプリケーションをインストールします。



ユーザーのデスクトップ（またはすべてのユーザー-C：\ Documents and Settings \ All users \ Desktop）に必要なショートカットを配置します



ユーザーと彼の文書



次に、ユーザーを設定します。 彼のアカウントでログインし、まずMy Documentsフォルダーへのパスを変更します。MyDocumentsフォルダーを右クリックします-> Properties-> Folder Path：D：\ My documents-> Ok-> Yes-> Yes



繰り返しますが、管理アカウントに戻り、C：\ Documents and Settings \ user \ Desktopフォルダのセキュリティを変更します。 ユーザーがこのフォルダーに書き込むことを禁止します。 もちろん、レジストリを登って、ユーザーのデスクトップフォルダーへのパスを変更することもできますが、私にとってはこれが簡単な方法です。 はい、見た目もきれいなデスクトップは、大量のアイコンやドキュメントよりもはるかに美しく見えます。



ユーザーがマイドキュメントフォルダーへのパスを変更できないようにします：gpedit.msc->ユーザーの構成->デスクトップ->ユーザーがマイドキュメントフォルダーのパスを変更できないようにします。



化粧品の洗浄と検査



貧しいユーザーの作業を簡素化するには、自動ログインを有効にします：Run-> control userpasswords2。 ユーザー名とパスワードを要求するチェックボックスを削除します。 ユーザー名と（2回）ユーザーパスワードを指定します。



バックアップディスクから文字を削除します：Run-> diskmgmt.msc。 バックアップドライブ（E :)を見つけて右クリックし、[ドライブ文字の変更]を選択します。 すべての文字を削除します。



再起動します。



ユーザーアカウントに入り、すべてのアプリケーションが動作することを確認します。 何かがうまくいかない場合は、問題を解決する方法を探しています。 ほとんどの場合、アプリケーションはそのフォルダーへの書き込み権限を必要とします-このため、管理者アカウントから、アプリケーションフォルダーへの権限を変更し、ユーザーに書き込みを許可します（子オブジェクトの権限を置き換えることをお勧めします）。



ユーザーがデスクトップ、ドキュメントのフォルダーなどに書き込むことができるかどうかを確認します。



すべてが機能する場合は、先に進みます。



2回目のバックアップ



長時間のセットアップとシステムの骨の折れる舐めの後、別のバックアップを作成する価値があります。 本質的に、その作成のプロセスは、名前と説明を除き、最初のプロセスと変わりません。



さらなるサービス。



バックアップの利点を過大評価することはできません。 将来、コンピュータに何らかの問題があり、通常の再起動では解決できない場合、または20分以内に、サイレンから起動し、最後のイメージを復元し、ディスクを取り出し、コーヒーを飲みに行くだけで十分です。システムを再起動します。



システム更新。



新しいソフトウェアをインストールするか、設定を変更する場合は、バックアップを更新するのが面倒ではありません。そうしないと、バックアップの魅力がすべて失われ、前の段落で説明した方法がそれほどエレガントではなくなります。各復元後に、不足しているソフトウェアを再インストールするか設定を変更する必要があるためです
