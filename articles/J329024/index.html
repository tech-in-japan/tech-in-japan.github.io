Qt Creatorの診断ツールプラグイン

良い一日。 最近、RAM使用量に関するデータを視覚化し、現在実行中のプロセスによってCPUをロードするように設計された、Qt Creator開発環境用のシンプルだが機能的なプラグインを作成するというアイデアを得ました。 私のインスピレーションとして、Visual Studioに存在する「診断ツール」ツールキットが役立ちました。 以下に、主な開発の詳細について説明します。









Qt Creatorのプラグインを作成するトピックは、たとえばQt Creator Extensions Systemの記事でHaberで既に詳細に説明されているため、一般的な情報については説明せず、プラグインの説明に直接進みます。



実際、オペレーティングシステムで実行されているプロセスに関する情報を取得するには、まずその識別子（PID）を見つける必要があります。 Qt Creatorを介して起動されたプロセスのPIDを取得する問題は、最初に対処しました。 このトピックのグーグル検索に失敗した後、Qt Creatorのソースコードを調査しました。その間に、必要なデータを見つけることは難しくないことがわかりました。 これを行うには、ProjectExplorer :: ProjectExplorerPlugin :: instance（）の静的インスタンスによって発行される次の信号をキャッチする必要があります。

void runControlStarted（ProjectExplorer :: RunControl * rc）;

void runControlFinished（ProjectExplorer :: RunControl * rc）;
