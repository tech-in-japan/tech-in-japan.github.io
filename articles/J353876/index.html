VHDLファイルを操作するSublime Text 3の構成

Sublime Text 3でVHDLを使用する



Sublime Textエディターは、vhdlおよびverilogファイルを操作する際の時間を大幅に節約します。 Sublime Text、Notepad ++などのエディターを使用したことがない人向け これらのエディターの主な便利な機能について説明します。



  • コード行の複数選択/編集（マウスの中央ボタンを押すか、Ctrlキーを押しながら）
  • コードにラベル（ブックマーク）を設定すると、大きなファイルをナビゲートしやすくなります。 （Ctrl + F2またはメニュー項目ジャンプ→ブックマークを使用）
  • ワークスペースを複数のウィンドウに分割する機能（Alt + Shift + 2またはメニューの表示→レイアウトから）
  • 1つのファイルを数回開く（ファイル->ファイルへの新しいビュー）
  • 選択したコード行をコメントアウト（Ctrl + /）
  • 検索と置換（Ctrl + h）
  • 開いているすべてのファイルを検索（Ctrl + Shift + f）
  • スニペット（コードテンプレート）を挿入（キーワード+ Tabキーを記述）
  • Python関数の作成と使用
  • さまざまなアドオンをインストールする機能
  • 柔軟な設定


崇高なテキスト統合



最初に、FPGAとSublimeエディターを操作するためのCADソフトウェアをドッキングします。



  • Sublime TextとザイリンクスISEの統合：

    ISEでメニュー[編集]→[設定]→[エディター]に移動します。テキストエディター→エディター=カスタム

    「コマンドライン構文」ウィンドウに行を挿入します。



    {C:\Program Files\Sublime Text 3\sublime_text.exe} $1



  • Sublime TextとXilinx Vivadoの統合：

    ツール->オプション->テキストエディター->現在のエディター：Sublime

    または

    ツール->オプション->テキストエディター->現在のエディター：カスタムエディター

    エディターへのパスを指定します。次に例を示します。

    C:\Program Files\Sublime Text 3\sublime_text.exe [file name]





プラグイン



さまざまなプラグイン（パッケージ）がエディターの機能を拡張します。 パッケージはできます

オンラインとオフラインの両方をインストールします。



プラグインをオフラインモードでインストールするには、いくつかの簡単な操作を行う必要があります。



  1. GitHubから必要なプラグインをダウンロードします
  2. アーカイブから抽出
  3. フォルダの名前を変更します。たとえば、「Sublime-HDL-master」を「Sublime HDL」に変更します
  4. 結果のフォルダーは、Packagesフォルダーにコピーされます（このフォルダーの場所は、メニュー項目[設定]→[Sublime Textでパッケージを参照]を選択すると簡単に見つかります）


SyncViewScroll-複数のウィンドウで作業するときに垂直および水平スクロールを同期するためのプラグイン。 プラグインを機能させるには、各ウィンドウのメニューで[表示]→[同期スクロール]を選択する必要があります。



テキストペストリーは、自動複数番号付けのプラグインです。 多数の番号付き信号/ポートを使用する場合に役立ちます。



テキストペストリーの使用方法
  1. 行の必要なセクションを選択します
  2. メニューを呼び出すCtrl + Shift + P
  3. アイテム「Text Pasty Command Line」を探しています
  4. 画面の下部に表示されるウィンドウで、次を入力します。

    • 0-0からの番号付け
    • \ i（1,10）-1から10までの番号付け
    • 1終了= 4-1、2、3、4、1、2、3、4などの番号付け
    • 上部の文字ac-A、B、C、A、B、Cなど
    • 大文字AC x3-A、A、A、B、B、B、C、C、Cなど
    • 1 x3-1、1、1、2、2、2、3、3、3など
    • xyz-x、y、z、x、y、z、x、y、zなど






Sublime Verilog -Verilog言語構文のサポート



Verilogガジェット -Verilogファイルを操作するための関数とスニペットのセット。



SmartVHDL -VHDL構文のサポート。 また、コード内の信号またはポートにカーソルを合わせると、この信号/ポートのタイプ（ビット数）に関するヒントを含むウィンドウが表示されます。 信号にカーソルを合わせると、コンテキストメニューに「後藤定義」項目が表示されます。信号のアナウンスの場所に移行します。



VHDLモード -VHDLファイルを操作するための関数とスニペットのセット。 機能の主要部分は、たとえば、キーボードショートカットAtl + K、C、Pによって起動されます。CとPは交互に押します。 主な機能：



  • ポートデータのコピー（ポート名、データ型）
  • 信号宣言としてポートデータを挿入
  • コンポーネント宣言としてポートデータを挿入
  • コピーされたポートデータに基づくテストベンチ生成
  • コードの自動フォーマット（タブの配置など）


Ucfファイルのサポート



デフォルトでは、Sublimeエディターはucfファイルを使用できません。 Ucfマークアップは、tclマークアップと同等です。 これをエディターに説明するだけです。



  • パッケージフォルダーに新しいTcl.sublime-settingsファイルを作成します
  • ファイルに行を入力します

    {"extensions" : ["ucf"]}



  • ファイルを保存


コードテンプレート（スニペット）の作成



コードテンプレートを挿入します。 



 My_proc : Process(clk, rst, data_in) begin if(clk'event and clk = '1') then if(rst = '1') then else then -- rst = 0 end if; -- data_in end if;--clk end process My_proc;


Tabキーを押してテキストを挿入した後、カーソルをMy_proc、clk、rst、data_inに配置して、このデータの値をすばやく変更する必要があります。 これを行うには、新しいスニペットを作成します：ツール→開発者→新しいスニペット。 データの編集： 



 <snippet> <description>process rst</description> <content><![CDATA[ ${1:<PROCESS_NAME>} : Process(${2:clk}, ${3:rst}, ${4:data_in}) begin if($2'event and $2 = '1') then if($3 = '1') then ${5} else then -- $3 = 0 $0 end if; -- $4 end if;--$2 end process $1; ]]></content> <tabTrigger>procrst</tabTrigger> <scope>source.vhdl</scope> </snippet>


このスニペットを保存します。 これで、procrstキーワードを記述するときに、現在のカーソル位置にテンプレートが挿入されます。



テンプレートの作成の詳細については、記事「スニペットを作成する方法」を参照してください。



Pythonでカスタム関数を書く



関数（プラグイン）の作成に関する詳細は、 「 シンプルなプラグインの 作成方法 」 、 「複雑なプラグインの作成方法 」の記事で説明されています 。



もちろん、スニペットの挿入は良いですが、たとえば、入力信号に応じて同じプロセス作成テンプレートが自動的に入力され、また、rstやceなどの信号が存在する場合にプロセスが変更されるようにしたいと思います。 より一般的には、プロセスの後、モジュールの外部ポートには内部信号の値が割り当てられますが、自動的に行われます。



VHDLファイルデータを解析するには、Vhdlモードプラグインの機能を使用します。



アクションの近似アルゴリズム：



  1. すべてのモジュールポートでデータを取得する
  2. タイプ「in」のポートはすべて、プロセスヘッダーに含める必要があります。
  3. ceおよび/またはrstというポートがある場合、対応するif else条件をプロセスに追加します
  4. プロセスの後、出力ポートを内部信号の値に割り当てるための文字列を挿入します（通常、このような信号はポートとも呼ばれ、接頭辞「s_」または「_net」が追加されます）


最初に、新しいスニペットを作成します。 



 <snippet> <tabTrigger>procclk</tabTrigger> <scope>source.vhdl</scope> <content><![CDATA[ ${DATAINPORTS} ${OUTPORTS} ]]></content> <description>process clk</description> </snippet>


ここで、$ {DATAINPORTS}はプロセスの説明が挿入されるラベルです。

$ {OUTPORTS}-内部信号の値の外部出力ポートへの割り当てのラベル。



VHDL Mode / Snippetsフォルダーにあるtest.sublime-snippetなどの名前で保存します。

VHDL Modeフォルダー内の記述された関数を使用します。 私はpythonの基本的な知識を持っているので、プラグインの機能を、既に説明されている機能との類推によって変更します。



vhdl_lang.pyファイルで、Interface（）クラスに新しい関数を作成し、in_portおよびout_portを呼び出します。



機能 
 def in_port(self): """ Generate Process depending on the input ports """ lines = [] bus_index = "" max_data = "" my_ports = "" is_clk = False is_ce = False is_rst = False if self.if_ports: for port in self.if_ports: if port.mode.lower() == 'in': if port.name.lower() == ('clk'): is_clk = True my_ports = port.name else: if port.name.lower() == ('ce'): is_ce = True elif port.name.lower() == ('rst'): is_rst = True my_ports = my_ports + ", " + port.name lines.append("Process("+ my_ports +')' ) lines.append("begin") if is_clk: lines.append(" if(clk'event and clk = '1') then") lines.append("") if is_rst and is_ce: lines.append("if(rst = '1') then") lines.append("") lines.append("elsif (ce = '1') then") lines.append("") lines.append("end if; -- rst") elif is_rst: lines.append("if(rst = '1') then") lines.append("") lines.append("else -- working body ") lines.append("") lines.append("end if; -- rst") elif is_ce: lines.append("if (ce = '1') then") lines.append("") lines.append("end if; -- lines.append(" end if;--clk") lines.append("end process;") # lines.append(str(testind indent_vhdl(lines, 1) return '\n'.join(lines) else: return None def out_port(self): """ Generate data after Process """ lines = [] if self.if_ports: for port in self.if_ports: if port.mode.lower() == 'out': lines.append("{} <= {}_net;".format(port.name, port.name)) indent_vhdl(lines, 1) return '\n'.join(lines) else: return None




out_port関数は、プロセスの後に文字列を挿入します。次に例を示します。

data_out1 <= data_out1_net;

data_out2 <= data_out2_net;



VHDL Modeフォルダーに新しいファイルを作成し、my_func.pyという名前を付けて、テキストを挿入します。 

 import sublime import sublime_plugin from.import vhdl_interface as face class PasteAsProcess(sublime_plugin.TextCommand): def run(self, edit): snippet_clk = "Packages/VHDL Mode/Snippets/test.sublime-snippet" in_port_str = face._interface.in_port() out_port_str = face._interface.out_port() self.view.run_command("insert_snippet", { "name" : snippet_clk, "DATAINPORTS" : in_port_str, "OUTPORTS" : out_port_str }) print('paste_as_process')


ホットキーの割り当ては残ります。 クラスはPasteAsProcessと呼ばれるため、コマンドはpaste_as_processと呼ばれる必要があります（最初の文字を除き、大文字で書かれた文字の前にはアンダースコアを入れる必要があります）。



[設定]→[キービルディング]に移動します 次の行を挿入します。 



 {"keys": ["alt+k", "p", "z"], "command": "paste_as_process", "context": [{"key": "selector", "operand": "source.vhdl"}] },


次に、作業のために、最初にキーの組み合わせ「alt + k」、「p」、「w」（デフォルト）を使用して、ファイルのvhdlポートの値をコピーする必要があります。 次に、キー「alt + k」、「p」、「z」を使用して関数を呼び出します。
