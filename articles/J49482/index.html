Windows Server 2008ライセンスポリシーの変更

良いニュースを共有し続けてください！ ライセンスポリシーを変更しています（より正確には、米国でのリーダーシップ）。 Andrei Beshkovにニュースをありがとう。

2009年1月16日に、ライセンスポリシーの変更が有効になり、次のようになりました。WindowsServer 2008が仮想化のプラットフォームとしてのみ使用される場合、Windows Server 2008 CALは必要ありません。



PS。 CAL-クライアントアクセスライセンス、つまり、ロシア語で言えば、サーバーに接続するためのクライアントライセンス。 Windows Server 2008のライセンスはこちらにあります 。



Windows Server 2008を使用する主なシナリオと、この変更がそれらにどのように影響するかを見てみましょう。

シナリオ1：顧客がHyper-Vの役割なしでWindows Server 2008を展開する

物理環境で展開するためのライセンスの変更はありません。 Windows Server 2008のCALは引き続き必要です。

シナリオ2.顧客がWindows Server 2008 Hyper-Vを展開して、Windows Server 2003ベースのサーバーを統合します。

物理サーバーの場合、CALは必要ありません。 CALは適切なバージョンのWindows Server（この場合はWindows Server 2003）に必要です

シナリオ3。顧客がWindows Server 2008 Hyper-Vを展開して、Windows Server 2003およびWindows Server 2008に基づいてサーバーを統合します。

物理サーバーの場合、CALは必要ありません。 Windows Server 2008ベースの仮想サーバーにはWindows Server 2008 CALが必要ですが、Windows Server 2003サーバーの場合は、Windows Server 2003 CALまたはWindows Server 2008 CALを使用できます。

私が話したすべてを説明し、必要に応じて参照できるマイクロソフトの公式ドキュメントは、 ボリュームライセンスブリーフと呼ばれます。 このドキュメントでは、クライアントアクセスライセンスとエクスターナルコネクタのセクションに注意してください。



ほとんどのMicrosoftサーバーにはクライアントアクセスライセンス（CAL）が必要で、オプションの外部コネクタ（EC）があります。 CALとECの要件は、インスタンスが物理OSEまたは仮想OSEのどちらで実行されていても同じです。 CALは、サーバーソフトウェアのインスタンスに直接または間接的にアクセスする各デバイスまたはユーザーに必要です。

これらのユーザーまたはデバイスがインスタンスのみを管理している限り、最大2つのデバイスまたはユーザーがインスタンスにアクセスするためにCALは必要ありません。 Windows Server 2008の場合のみ、次の目的でのみ使用されている物理OSEで実行されているサーバーソフトウェアのインスタンスにアクセスするためにCALは必要ありません。

I.ハードウェア仮想化ソフトウェアを実行し、

II。 ハードウェア仮想化サービスを提供する、または

III。 ライセンスサーバーのオペレーティングシステム環境を管理およびサービスするソフトウェアを実行します。

ただし、サーバー上の仮想OSEのサーバーソフトウェアのインスタンスにアクセスするには、適切なCALが必要です。

さらに、Windows Server 2008では、認証やその他の方法で個別に識別されることなく、インターネット経由でのみソフトウェアインスタンスにアクセスするユーザーまたはデバイスのCALまたはECは必要ありません。



近い将来、上記の変更は公式のWindows Server 2008ポータルの製品使用権文書および製品価格表に表示される予定です。
