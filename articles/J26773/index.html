そのため、議題： FireFoxのプラグインを迅速かつクリーンに作成します。



私はすぐにあなたに警告したいです ：プラグインの実験がGreaseMonkeyのスクリプトで行われる前は、私はこの問題のプロでもグルでもありません（コメントを簡単に編集するためのギズモであるHabraHackerを見ました）。 ただし、プラグインは書かれているようで、機能します;）



私が必要：

  1. Firefox たとえば、3番目のバージョン（minefield）があります。
  2. Firefox Portable もちろん、それは必要ではないかもしれませんが、私はそれをセットアップしました。2番目のバージョンに従ってプラグインを簡単かつ迅速に実行するためです。
  3. エディター 。 Notepad ++を選択します。 同じことをお勧めします-それは非常に速く、簡単で、便利で、膨大な数の機能を備えており、何でも強調しています。 気に入らない場合は、JavaScriptとXMLを強調表示することをお勧めします。
  4. JavaScriptの知識 。 奇妙なことに、それらも必要になります。
  5. ちょっとした工夫と想像力 。 理由はわかりません。




実際、どこから始めるべきかを考えることから始めました。 Googleでチュートリアルを検索したところ、カップルが見つかりましたが、どういうわけか私はそれらが好きではありませんでした-おそらく午前3時にそのような反対意見を読む時間がないでしょう。）私のインターフェースを見てみると、計画されたものと非常によく似たプラグイン-Khabragolik 彼はまた数字を表示します-それはあなたが必要なものではありませんか？ ;）

特別な道徳的障壁がなければ、彼の配布キットをハードドライブにダウンロードしました。XPI形式のプラグインはすべて、さまざまなファイルを含むZIPアーカイブに過ぎないことに留意して、拡張子をhabraholic-0.2.zipに変更しました 。 どこかに解凍し、インベントリのパラグラフ5を適用して、物のインベントリを作成しました。 

  +クローム
    +コンテンツ
      -panel.xul //プラグインの本体。  XUL *ソケットの説明
      -script.js //これはすでに魂です。 まあ、より正確には心...本能。 要するに、すべてのjavascript実行可能コード
      -prefs.xul //設定ウィンドウの説明
      -habr.ico //大文字のHを含む奇妙な絵は、おそらくhabrahabrを象徴しています
  +デフォルト
    +設定
      -defaults.js //プラグインがブラウザに永久に保存する変数のデフォルト値があります。 通常は設定があります
  -install.rdf //プラグインを説明する奇妙なXMLファイル
  -chrome.manifest //ブラウザのロード時に、どのXULファイルを接続するかをブラウザに伝えます


*-これは、インターフェースを記述するXMLベースの言語です。 XHTMLに似たJSをサポートしています-それだけです。 プラグインに書き込まれます。

次の行はchrome.manifest ファイルです 。 その中で、panel.xul ファイルへのパスのみを変更しました。habraholicを自分の名前に置き換えました 。



/defaults/preferences/defaults.jsを見たところ 、ユーザーが構成可能な変数しかないことがわかりました。 コメントアウトしました（//）。



次に、ファイル/chrome/content/prefs.xulを不要なものとして削除し（設定はもうありませんよね？）、 / chrome / content / habr.icoをアイコンに置き換えました 。



設計とレイアウトの時が来ました： /chrome/content/panel.xul 最初にしたことは、すべての要素IDとラベルを変更することでした。「habraholic」と「Karma」を別の言葉に置き換えました。 設定項目とセパレーター（Menuseparator）をメニュー（要素内で説明）から削除し、単語「karma」を最初の項目のテキストから削除しました。 アイコンとスクリプト（および<script ...>）へのパスを変更し、onclick = ``およびoncommand = ''と呼ばれるjavascript関数の名前を変更しました。



ここで、少し遊びましょう。アイコンをさらに追加し、ラベルやその他の要素を追加します。 しかし、あなたはいつでもいじることができ、プログラムのコード部分/chrome/content/script.jsに進みます。



すべての機能の名前を変更することから始めました（名前からhabraholicを削除するなど）。 それから彼はそれらすべてを見ました：

  • ただのコード -表示されるとすぐに通常どおり実行されます。 その後、彼はブラウザの読み込みにイベントハンドラーのみを割り当てました-そのままにしておき、ハンドラーの名前を変更しました。
  • habraholicLoadはまさにこのハンドラーです。 5分ごとにカルマを更新する機能を起動します。 ハンドラーは変更されず、起動される関数の名前が変更されました。
  • habraholicLeftClick-クリックを処理します。 ユーザー名が設定に入力されると、カルマアップデーターを呼び出します。 そうでない場合は、設定を開く関数を呼び出します。 すべてのIFを削除し、最初のブランチを残しました。現在は更新のみです。
  • habraholicUpdateKarma-アップデーター自体。 設定が構成されている場合-habra-apiへのXMLHttpRequestを作成します。 構成されていない場合は、ラベルの内容を変更して、すべてを構成するときが来たことをユーザーが理解できるようにします。 彼はオブジェクトの作成のみを残し、すべてのチェックを削除し、アクセスされているURLを変更しました。
  • habraholicUpdateKarmaRepeatedly - habraholicLoadによって呼び出され、10分ごとにhabraholicUpdateKarma（）を呼び出す同じ関数。 時間間隔（ミリ秒単位）と固有名（関数名）を変更しました。
  • habraholicSetKarma-そのXMLHttpRequestからのコールバック。 次のように動作します。デフォルト値で変数を作成し、try {}でサーバー応答からkarmaおよびhabrasilaの値を分離し、それらの変数で置き換えようとします。 私はXMLを持っていなかったので、受け取ったページのすべてのHTMLコードを変数に引き込み、定期的に実行しました。

  • habraholicGetUsername、habraholicIsRatingPositionHidden-永続的に保存された設定を取得します。 削除しました。
  • habraholicLoadProfile-ユーザープロファイルを開きます。 独自の目的のために変更されました。
  • habraholicLoadPage-とても便利なもの：必要なアドレスでページ（タブ）を開きます。 ホイールをクリックするだけで素晴らしい友達（「新しいタブで開く」）。 名前が変更され、もう触れられていません。
