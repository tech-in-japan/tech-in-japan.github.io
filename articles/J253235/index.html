背景



自己組織化されたNASには多くのソリューションがありますが、いくつかの理由でFreeNASに立ち寄りました。

  • ZFSのネイティブサポート。
  • AD、LDAP、およびその他のディレクトリサービスとの良好な統合。
  • システムはFreeBSD上に構築されています。つまり、jailとportsがあります。


自宅だけでなく職場でも使用しているので、これまでの決定を後悔していません。



私はCrashplanを長い間使用しており、有料のサブスクリプションを持っています。 クロスアーカイブとクラウドへのアーカイブの両方を使用します。 説明されたイベントの前にLinuxおよびWindowsでのみ使用されます。



したがって、FreeNAS（9.3）を展開したとき、VirtualBoxをインストールした後に2番目にやったことは、Crashplanを展開することでした。 そして、ここでいくつかの驚きが私を待っていました。 Crashplanのインストールの問題に最初に出会ったとき、私は古いFidoの熊手ジョークを思い出しました。

  1. プラグインでキリル文字の正しい表示を設定してみてください。
    捨てた
    プラグインはpbiファイルとして作成されます。このファイルは、展開時に標準のjailディレクトリを使用せずに、独自に作成します。 それを理解する試みは何にもつながりませんでした。私は構造の複雑さにdrれ、継続しないことに決めました。

    延期する
    VirtualBoxでは、FreeNASは32ビットカーネルのみを実行できます。 これは邪魔ではありませんが、どういうわけかディレクトリを仮想マシンにマウントする必要があります。 いくつかのオプションがあります：nfs（ネストされたデータセットの問題）、smb / cifs（最も機能するソリューションですが、smb-ballsを必要とし、これは常に可能であるとは限りません）、VirtualBoxのリソースを共有します（何らかの理由で、常に機能しません）。 私は最後の手段としてこの決定を延期しました。
  4. クリーンなjailをインストールし、Linux BSDエミュレーターでCrashplanを実行します。

準備作業


FreeNASの初期セットアップのすべての瞬間、最初の打ち上げの後、私は省略します、これについてはたくさん書かれています。

  1. ssh経由でFreeNASに接続します。
  2. Linuxのカーネルモジュールがロードされているかどうかを確認します。 

    kldstat | grep linux


    次のようなものがある場合： 

     22 2 0xffffffff81d09000 46855 linux.ko


    その後、すでにダウンロードしていますが、そうでない場合： 

     kldload linux
  3. システムでlinux-compatを有効にします

    Webインターフェースで
    linux-compatを有効にすると、linux.koカーネルモジュールが自動的にロードされるため、linux_loadパラメーターを省略できます。



  4. 刑務所を作成します。

    の作成
    デフォルト設定は変更できません。 VIMAGEパラメーターはオンのままです。







    そして、インストールが完了するのを待っています






    このプロセスには10〜45分かかります。





Crashplanと必要なものすべてを刑務所にインストールする


アクションはssh FreeNASで実行されます（特に指定がない限り）。

  1. 私たちは刑務所のIDを決定します 
     jls


    私たちはそのような答えを得る




    jail ID = 2であることがわかる場所から。

  2. 私たちは刑務所に接続されています。 

     jexec 2 /bin/tcsh


  3. jailのパッケージとポートを更新する 

     pkg update pkg upgrade portsnap fetch extract


    完成を待っています。

  4. nanoエディターをインストールします（vi愛好家はこの項目をスキップします） 

     cd /usr/ports/editors/nano make install clean


  5. fedora-linuxエミュレーターをインストールする 

     cd /usr/ports/emulators/linux_base-f10/ make install clean


  6. procfsエミュレーターをインストールする 

     cd /usr/ports/sysutils/linux-f10-procps/ make install clean


  7. Linux用のJREをインストールする 

     cd /usr/ports/java/linux-sun-jre17 make install clean


    そして得る
    間違い




    これは、Oracleがこのファイルを直接ダウンロードすることを禁じているためです。 したがって、このリンクに従ってファイルjre-7u75-linux-i586.tar.gzをダウンロードしてください （ライセンスに同意することを忘れないでください）。 ダウンロードしたファイルは、ディレクトリ/ usr / ports / distfiles （jailのパス）またはFreeNASのディレクトリ/ <jailへのパス> / <jail name> / usr / ports / distfiles （たとえば、パスがある/ mnt / poolz / jail / crashplan / usr / ports / distfiles ）

    そして繰り返す 

     cd /usr/ports/java/linux-sun-jre17 make install clean


    そして、インストールが完了するのを待っています。

  8. FreeNASでのLinux用procfsのマウント

    刑務所を出る 

     exit


    コマンドラインFreeNASに行きます。 ルートFSを書き込みモードに再マウントします。 

     mount -uw /


    procfsマウントスクリプトを作成します。 

     nano /conf/base/etc/rc.d/mountproc


    スクリプトテキスト（修正後） 
     #!/bin/sh # PROVIDE mountproc # REQUIRE LOGIN # BEFORE: ix-jail . /etc/rc.subr name=mountproc rcvar=mountproc_enable start_cmd="${name}_start" stop_cmd="${name}_stop" mountproc_start () { /sbin/mount -t linprocfs linproc /mnt/poolz/jail/crashplan/compat/linux/proc/ } mountproc_stop() { echo "--- mountproc stopped ---" } load_rc_config $name run_rc_command "$1"




    重要！ mountproc_startが行の最初の位置から開始することを確認してください！

    パス/ mnt / poolz / jail / crashplan / compat / linux / proc /は 、独自のものに変更されます（前の段落と同様）。

    スクリプトを実行可能にします。 

     chmod +x /conf/base/etc/rc.d/mountproc


    自動起動スクリプトをオンにします

    有効にする方法




  9. すべてが正しく行われたかどうかを確認し（主にスクリプトテキスト）、FreeNASをオーバーロードします。

  10. 再起動後、設定が有効になっているかどうかを確認します。 

     kldstat | grep linux mount | grep linprocfs


    2番目のコマンドの後に次のような行が表示されない場合： 

     linprocfs on /mnt/poolz/jail/crashplan/compat/linux/proc (linprocfs, local)


    次に、スクリプトを作成するポイントに戻り、エラーを探します。

    すべてが順調であれば、続行します。

  11. クラッシュプランを置く

    刑務所に行きます（最初の2つのポイントと同様）： 

     jls jexec <jail_id> /bin/tcsh


    / compat / linuxディレクトリに移動し、Crashplanをダウンロードし、解凍してインストールします。 

     cd /compat/linux fetch http://download1.us.code42.com/installs/linux/install/CrashPlan/CrashPlan_3.7.0_Linux.tgz tar -xzf CrashPlan_3.7.0_Linux.tgz cd CrashPlan-install /compat/linux/bin/bash /compat/linux/CrashPlan-install/install.sh


    ライセンス契約を読んで同意します。

    原則として、デフォルトですべてを残すことができます。 何かを変更したい場合は、慎重に変更してください！

    インストール後、 CrashPlan_3.7.0_Linux.tgzファイルとCrashPlan-installディレクトリを/ compat / linuxディレクトリから削除できます。これらは不要です。

  12. run.confクラッシュプランの編集。 

     nano /compat/linux/usr/local/crashplan/bin/run.conf


    各行の最初の文字SRV_JAVA_OPTS = "およびGUI_JAVA_OPTS ="の後に insert

    -Djava.nio.channels.spi.SelectorProvider = sun.nio.ch.PollSelectorProvider 。

    その結果、受信する必要があります： 

     SRV_JAVA_OPTS="-Djava.nio.channels.spi.SelectorProvider=sun.nio.ch.PollSelectorProvider -Dfile.encoding=UTF-8 -Dapp=CrashPlanService -DappBaseName=CrashPlan -Xms20m -Xmx1024m -Djava.net.preferIPv4Stack=true -Dsun.net.inetaddr.ttl=300 -Dnetworkaddress.cache.ttl=300 -Dsun.net.inetaddr.negative.ttl=0 -Dnetworkaddress.cache.negative.ttl=0 -Dc42.native.md5.enabled=false" GUI_JAVA_OPTS="-Djava.nio.channels.spi.SelectorProvider=sun.nio.ch.PollSelectorProvider -Dfile.encoding=UTF-8 -Dapp=CrashPlanDesktop -DappBaseName=CrashPlan -Xms20m -Xmx512m -Djava.net.preferIPv4Stack=true -Dsun.net.inetaddr.ttl=300 -Dnetworkaddress.cache.ttl=300 -Dsun.net.inetaddr.negative.ttl=0 -Dnetworkaddress.cache.negative.ttl=0 -Dc42.native.md5.enabled=false"


  13. javaへのパスを正しいパスに修正します。

    デフォルトでは、Crashplanはパス/ usr / local / crashplan / jre / bin / javaを使用します

    /usr/local/linux-sun-jre1.7.0/bin/javaポートからインストールされたjavaパスに変更します 

     nano /compat/linux/usr/local/crashplan/install.vars


    JAVACOMMON変数で 、パスを/usr/local/linux-sun-jre1.7.0/bin/javaに変更します

    こんな感じ 
     TARGETDIR=/usr/local/crashplan BINSDIR=/usr/local/bin MANIFESTDIR=/usr/local/var/crashplan INITDIR=/etc/init.d RUNLVLDIR=/etc/rc.d INSTALLDATE=20150314 APP_BASENAME=CrashPlan DIR_BASENAME=crashplan JRE_X64_DOWNLOAD_URL=http://download.code42.com/installs/proserver/jre/jre-7u45-linux-x64.tgz JRE_I586_DOWNLOAD_URL=http://download.code42.com/installs/proserver/jre/jre-7u45-linux-i586.tgz JAVACOMMON=/usr/local/linux-sun-jre1.7.0/bin/java




  14. パフォーマンスをチェックします。

    刑務所から 

     /compat/linux/bin/bash /compat/linux/usr/local/crashplan/bin/CrashPlanEngine start /compat/linux/bin/bash /compat/linux/usr/local/crashplan/bin/CrashPlanEngine status


    FreeNASから 

     jexec <jail_id> /compat/linux/bin/bash /compat/linux/usr/local/crashplan/bin/CrashPlanEngine start jexec <jail_id> /compat/linux/bin/bash /compat/linux/usr/local/crashplan/bin/CrashPlanEngine status


    起動時に「OK」になり、ステータスを確認するときに多くのPIDが表示される場合、すべてが機能しています。

  15. Crashplan自動実行のセットアップ

    刑務所に行きます： 

     jexec <jail_id> /bin/tcsh


    自動実行スクリプトを作成します。 

     nano /etc/rc.d/crashplan


    スクリプトテキスト 
     #!/bin/sh # PROVIDE crashplan # REQUIRE LOGIN . /etc/rc.subr name=crashplan rcvar=crashplan_enable start_cmd="${name}_start" stop_cmd="${name}_stop" crashplan_start() { /compat/linux/bin/bash /compat/linux/usr/local/crashplan/bin/CrashPlanEngine start } crashplan_stop() { /compat/linux/bin/bash /compat/linux/usr/local/crashplan/bin/CrashPlanEngine stop } load_rc_config $name run_rc_command "$1"


    crashplan_startおよびcrashplan_stopコマンドが行の最初の位置にあることを確認してください！



    スクリプトを実行可能にします。 

     chmod +x /etc/rc.d/crashplan


    rc.confスクリプトの編集。 

     nano /etc/rc.conf


    行を追加する 

     crashplan_enable="YES"


    ルールに沿って 

     sshd_enable="YES" hostname="crashplan"


    SSHを有効にしてホスト名を尋ねます。

  16. 保管フォルダーを刑務所にマウントします。

    刑務所内のフォルダを接続する方法は、 ドキュメントに詳しく説明されています 。

    Crashplanのファイルシステムのルートはjail（ディレクトリ/ ）のルートではなく、ディレクトリ/ compat / linuxになることに注意してください !!!

    つまり、たとえば、 / mntディレクトリではなく、 / compat / linux / mntディレクトリに接続する必要があります。そうしないと、Crashplanはディレクトリを表示しません。

    また、ネストされたデータセットも登録する必要があることを忘れないでください！

  17. SSHの最後の仕上げ。

    ルートのパスワードを設定します。 

     passwd


    重要！ FreeBSD（およびFreeNAS jail）では、sshを介したルートログインは禁止されています。 ssh経由でログインするには、ユーザーを追加するか（彼をwheelグループに含めてsuにしてください ）、 / etc / ssh / sshd_configファイルを編集する必要があります。 セキュリティの観点から見ると、最初のオプションはより読み書きができます。

    adduserコマンドの使用方法は説明しませんが、2番目のオプションについて説明します。

    sshd_configファイルの編集： 

     nano /etc/ssh/sshd_config


    文字列を探しています 

     #PermitRootLogin no #PasswordAuthentication no


    に変更します 

     PermitRootLogin yes PasswordAuthentication yes


    SSHを保存して実行します。 

     service sshd start


    または再起動： 

     service sshd restart
