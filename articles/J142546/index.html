Scalaの非対称感覚



Scalaについて書いている私のお気に入りの著者の1人であるRunar Bjarnson （ Apocalisp ）は、非対称レンズに関するTony Morrisの記事を作成しました。 非対称レンズは、純粋に機能的な言語ではすべてのオブジェクトが不変であるため、他のオブジェクトのプロパティへのアクセスまたは変更を提供するオブジェクトです。 レンズは、ゲッターとセッターに似た2つのメソッドを定義します。 



case class Lens[R,F] ( get: R => F set: (R,F) => R )


ここで、 Rはプロパティの読み取り元のオブジェクトのタイプであり、 Fはこのプロパティ自体のタイプです。 Runarは、これらのレンズの構成をオブジェクトへの一連の変更として表示し、そのような複雑な修正を転送し、より複雑な操作を定義することができ（製品または「選択」）、他のオブジェクトと操作を助けを借りて表現できると言います。 最終的には、機能的なクリーンさと不変性を維持しながら、プログラムはほぼ必須のスタイルで記述できるようになります。 レンズを操作するために、 レンズ付きコンパイラー用の特別なプラグインもあります。これにより、言語の一部としてレンズをすぐに使用できます。

Scalaはどこに行くのですか？



Martin Oderskyは、Scala 2.10の新機能：先物/約束、Scala自身のリフレクション/具体化、マクロ、多くのコンパイラー最適化について述べました。







焦点は反射/具体化とマクロにありました。 反射はパイのパイの形で構築されます:)各モジュールは悪名高いケーキパターンに基づいています 。 1つのパイはコンパイラのプログラムのモデル、2番目は表示用のミラー、一般的なコードは3番目のパイ（それらの上）に配置され、最後にこれらのパイの上に1つの美しいパイファサードがあり、これは使いやすく、基礎となる実装の機能を隠しますパイ。







マクロを使用すると、 ASTプログラム自体にアクセスできます。 つまり、リフレクションのように、メソッドの本体を1行ずつ調べて、どのコードでどのコードが実行されているかを調べることができます。 このような追加により、（対話者の1人によれば）コンパイラーの90％がこれらのマクロに組み込まれ、カーネルがクリアされ、すべてのツールでの各サポートの費用のかかる開発なしで言語拡張機能を作成できるようになります。
