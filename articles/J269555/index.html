Joker 2015 Java Conferenceレポート

2015年10月16〜17日に、サンクトペテルブルクでジョーカー2015カンファレンスが開催されました。 2日以内に、 Javaプログラミング言語の開発に関連する30以上のレポートが読まれました。 誰が講演者の中にいましたか、どのレポートが興味深いと思われるものを見ることができたか、これはさらに説明されています。







Joker 2015 Conferenceは、毎年サンクトペテルブルクで、そして常に10月に、毎年3回続けて開催されます。 Jokerは、別のJava会議JPointの双子の兄弟です。 JPointも毎年3回開催され、4月にサンクトペテルブルク（ JPoint 2013 ）またはモスクワ（ JPoint 2014 、 JPoint 2015 ）で開催されます。 両方の会議は同じJUG.RUチームによって行われます。



JPoint 2015からJoker 2015までの時間：



  • HabrahabrのAlexei Fedorovは、会議のいくつかの発表を発表しました。
  • シリーズ「Without Slides 」（ Dmitry Zhemerov 、 Andrey Pangin 、およびAlexei Shipilevとの ）からの3つのインタビューが登場しました。
  • 将来のカンファレンススピーカーは、 JUG.RU 、 CodeFreeze 、 jug.msk.ruでなんとか話をすることができました。


私はこの機会を利用して会議をオンラインで見ました。 レポートを選択するとき、彼は主に講演者の性格、次に他のレポートのトピックに導かれました。 したがって、アレクセイ・シピレフ、エフゲニー・ボリソフ、ニコライ・アリメンコフの報告がすぐに選ばれました。 残念ながら、Baruch SadogurskyとSergei Kuksenkoによる報告がなかった。 セルゲイは会議の参加者であり、バルークは完全に成長している段ボールの写真の形で目に見えない存在でした。 セルゲイ・ククセンコとバルーチの段ボールの写真 。



私が選択したレポートは、他のレポートがより悪いことを意味するものではありません。これは会議の期間中だけの私の選択です。 会議中に特定のレポートを選択すると、残りを表示する機会を失うことはありません。 必ず他のレポートのビデオを選択して視聴してください。それらを表示するためのリンクはすでにメールで届いています。

ロードオブザストリングス：2つの洗掘

Alexey Shipilev、Oracle







Alexeyには、レポート用の部屋をすべて集めて、それらについて興味深いことや興味深いことを伝える驚くべき能力があります。



この講演では、 JDK / JVMで近日中に利用可能になるはずの2つの機能（ コンパクト文字 列とIndify文字列連結）の開発について説明しました。 Compact Stringsは、シングルバイト文字ストリングのコンパクトな表現を実装します; Indify String Concatは、ストリング連結をランタイムに転送します。 どちらか（または両方）を実装すると、文字列操作のパフォーマンスが大幅に向上します。

ピンポンプログラミング

アントン・ケクスとアンドレイ・ソルンツェフ、コードボーン







AntonとAndreyは、 Codeborneで積極的に実践されているペアプログラミングのスタイルを示しました。



  1. 最初に、最初のプログラマーが失敗する単体テストを作成します。
  2. 次に、2番目のプログラマーは、テストが以前に作成された機能の実装を作成し、テストに合格します。 彼は、合格しない次の単体テストを作成します。
  3. 最初のプログラマは、2番目の機能の実装を記述します。


アイデアは良いです、最初の15-20分は活発に見えます。 パフォーマンスの終わりに、リスナーは30分間積極的に質問をしました。 最初の30分で提案されたプラクティスの使用を示し、後半の30分で質問に答えれば理想的で、見た目もよくなります。

会議終了



アレクセイ・フェドロフ、イワン・ドルゴフ、アンドレイ・ドミトリエフは会議の終わりに。







アレクセイ・フェドロフがステージに招待した組織チームのすべての参加者とボランティア。







さらに、それらに参加したすべての会議スピーカー。







まとめ



唯一のマイナス点は、昼食に関する繰り返しの問題です。 2日目には、2交代で昼食を用意することが決定されました。



はるかに肯定的な側面があります。



  • 高レベルの会議組織。
  • 技術的および組織的な問題の主催者による迅速な決定;
  • 過去数年間のフィードバックで表明された希望を考慮に入れる;
  • チケットの価格は、特に早期購入の場合、過度に思えません。
  • 会議後、すべてのレポートのビデオへのリンクを送信します。
  • レポートのプレゼンテーションファイルは、会議の直後にサイトで利用可能になります。
  • オンライン放送の視聴を購入できます。


会議に直接参加する機会がない場合は、オンラインブロードキャストが役立ちます。



  • 旅行の時間、飛行機/電車とホテルでお金を節約します。
  • 放送中に何かを聞いたり、朝の初めに眠った場合は、スクロールバックできます。
  • レポートの先頭を聞いて気に入らなかった場合は、別のパラレルレポートに簡単に切り替えることができます。さらに、レポートを開始できます。
  • オンライン表示用のチケットを1枚購入することで、会議室で同僚との共同表示を簡単に整理できます。
  • 昼食に関する誤解の可能性は無関係になります。


残念ながら、参加者や講演者との個人的なコミュニケーションの機会を失います。



すでに素晴らしいイベントを改善するための建設的な提案：



  • JokerおよびJPointカンファレンスでレポートを繰り返さないようにしてください。 両方の会議の聴衆は同じです。 レポートの言語の変更（英語からロシア語、またはその逆）またはレポートの内容の大幅な更新（50％以上）によってのみ、繰り返しが既に受け入れられるように思えます。
  • 会議後に表示するレポートの選択に役立つ助言機能を追加します。 フィードバックによって決定される参加者のフィードバックに従って、レポートの説明のページに直接、レポートの評価を表示することを提案します。 技術的に複雑でない場合は、コメントを追加する機能も追加します。
  • レポート間で、休憩を30分から15分に短縮します。
  • 時間規制をより明確に遵守してください。そうしないと、部屋から部屋へ移動する時間がありません。
  • 聴衆に質問するときは、マイクを使用することを忘れないでください。そうしないと、オンラインブロードキャストで聞こえません。
  • 今後のJPointから始めて、2シフトランチプラクティスを永続的にします。


10月18日、 大学の日は参加者の年齢を制限して開催されました（24歳以下）。 レポートや講演者のトピックから判断すると、興味深いものもありました。 レポートに精通する法的機会がなかったため、少なくともメインのJoker 2015カンファレンスの参加者にレコーディングへのアクセスを提供することは可能でしょう。
