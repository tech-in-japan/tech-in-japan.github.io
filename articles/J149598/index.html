ボード上にAllwinner A10 SoCを搭載したタブレットでLinuxを実行する

背景



SoC Allwinner A10でタブレットを購入しました（ sun4iは、A8コアファミリのCortexアーキテクチャのAllWinner A10 ARMv7-Aプロセッサのコード名です ）。 最初はAndroid 2.xで、次に4.xでプレイしましたが、時間が経つにつれて十分ではなくなりました。 さらに、Linuxを搭載したこのSoCに基づくミニコンピューターが生産を開始しました。 「うーん、私のタブレットより悪いことは何だろう？」と思い、ビジネスに取りかかりました。 判明したように、すべてがすでに私のために行われており、物事を行う必要はありませんでした ：

  1. カーネルはこのプラットフォーム用に最適化されています。
  2. このプラットフォーム用に最適化されたブートローダー
  3. 新しいdebian armhfポートがリリースされました 。


この全体が構成され、組み立てられたままです。



水曜日



システム-ArchLinux x86-64。 アセンブリ用の環境の場所とインストール方法については説明しませんが、インターネット上ではこの情報で十分です。 はい、そしてこれのほとんどが理由ではなく、私は必要なものすべてを集めさせます。

私の「台所」はこれです：

  • qemu-user-static -rootfsでchrootできるようにするために担当者がいなかったため、debianリポジトリからパッケージを取得して収集する必要がありました。 このすべてが、異なるアーキテクチャのマシン（この場合は私のマシン）でarm elf実行可能ファイルを実行するために必要です。
  • gccバージョン4.6.1（ Sourcery CodeBench Lite 2011.09-70）-GCC ARMクロスコンパイラー。 古いものかもしれませんが、まだ私を失望させません。 linaroの新しいクロスコンパイラを使用することをお勧めします。


SoCブートプロセス



  • brom（mmc、nand、usbでboot0と呼ばれるブートローダーを探します（SPL-2番目のプログラムローダー：sun4i-spl.bin）。
  • boot0 （SPL-2番目のプログラムローダー：sun4i-spl.bin）はデバイスを初期化します
  • SPL （boot0）はu-boot（boot1）を起動します
  • u-boot （boot1）は残りのデバイスを初期化し、ハードウェアを操作するための抽象化を作成します
  • initrdのようなu-bootの横にハードウェア構成ファイル（script.bin。Evb.bin、mele.bin sys_config.bin）をロードします。
  • u-bootはカーネルをロードします。


つまり ダウンロードはSDカードから開始され、そこにブートローダーがない場合は、NANDフラッシュが続行されます。

allwinner-ubootブートローダーのドキュメントによると、SDカードの構造は次のようになります。

始まり サイズ 説明
0 8KB 未使用、パーティションテーブルなどに使用可能
8 24KB 初期SPLローダー
32 512KB u-boot
544 128KB 環境
672 352KB 予約済み
1024 パーティションは無料


U-bootアセンブリ


収集する必要はありません。sun4i-spl.binu-boot.binはここからダウンロードできます 。

uboot-allwinnerをビルドします。 


git clone git://github.com/hno/uboot-allwinner cd uboot-allwinner make sun4i CROSS_COMPILE=arm-none-linux-gnueabi- cd ..

microSDドライブを壊します





注意、操作の結果として、MicroSDドライブ上のデータは失われ、必要なものはすべて安全な場所に保存されます。

  • 必ず2つのセクションを作成する必要があります。
  • 最初のセクションは、鉄の構成ファイル（evb.bin）とカーネル（uImage）を含むセクションです。 セクタ2048から始まり、サイズが32MB〜100MBで、FATファイルシステム（FATパーティションタイプ）またはext2 / 3（パーティション83タイプ）が必要です。
  • 2番目のパーティションはrootfsパーティションで、システム全体です。パーティションサイズを1GB〜4GB以内にすることが望ましいです。ファイルシステムは厳密にext4で、パーティションタイプは83です。
  • 必要に応じて、別のスワップセクションを作成できます。
  • 残りのスペースを自分で使用してください。 データ用のセクションを作成しました。/home/にマウントできます


16GBカードを壊した方法は次のとおりです。

上記のポイントに従って、MicroSDドライブを破損します。

Linuxを使用する人がfdiskを使用できることを願っています。 

 fdisk ${SD}






書式設定： 


 mkfs.vfat ${SD}1 -n sun4i-boot mkfs.ext4 ${SD}2 -L sun4i-rootfs


適切な場所にマウントします。 

 mkdir /mnt/sun4i-boot mkdir /mnt/sun4i-rootfs mount ${SD}1 /mnt/sun4i-boot mount ${SD}2 /mnt/sun4i-rootfs


カーネルアセンブリ





カーネルをコンパイルできません。下のモジュールを使用して既製のカーネルを添付しました。 

 #  git clone git://github.com/amery/linux-allwinner.git #  cd linux-allwinner make ARCH=arm CROSS_COMPILE=arm-linux-gnueabi- sun4i_defconfig #  make ARCH=arm CROSS_COMPILE=arm-linux-gnueabi- gconfig


gconfigの代わりに、xconfig（qtインターフェイス）またはmenuconfig（コンソールインターフェイス）のテイストを使用できます。 カーネルに必要なドライバーと必要な機能を含めることを忘れないでください。 オプションのヒントをお読みください。

タッチスクリーンモジュールとfs（fat、ntfs）を忘れないでください。 

 # make ARCH=arm CFLAGS="march=armv7-a -mfloat-abi=hard -mfpu=neon -ftree-vectorize -mvectorize-with-neon-quad -mcpu=cortex-a8 -mtune=cortex-a8 -mthumb -O2 -pipe -fstack-protector --param=ssp-buffer-size=4 -D_FORTIFY_SOURCE=2 -ffast-math" CROSS_COMPILE=arm-none-linux-gnueabi- -j3 uImage modules #  make ARCH=arm CFLAGS="march=armv7-a -mfloat-abi=hard -mfpu=neon -ftree-vectorize -mvectorize-with-neon-quad -mcpu=cortex-a8 -mtune=cortex-a8 -mthumb -O2 -pipe -fstack-protector --param=ssp-buffer-size=4 -D_FORTIFY_SOURCE=2 -ffast-math" CROSS_COMPILE=arm-none-linux-gnueabi- -j3 INSTALL_MOD_PATH=/mnt/sun4i-rootfs #   boot  cp arch/arm/boot/uImage /mnt/sun4i-boot


-j引数は、同時コンパイルスレッドの数を指定します。 プロセッサのコアよりもスレッドが1つ多いことが実験的に確立されました。

最適化フラグはValdikSSの記事から取得されます 。 正直なところ、カーネルはそれらでコンパイルされました。 ただし、SDフラッシュカードにはまだインストールしていません。

rootfsをインストールする



debootstrapを使用して初期rootfsをロードする方法については説明しません。これは別の記事に値するからです。 既製のDebian Wheezy ARMHF rootfsを配置した方が良いでしょう。

u-bootはデフォルトでこのように設定されているため、rootfsは2番目のSDパーティションに保存されている属性で解凍する必要があります（設定についてはこちらを参照してください） 。

Debian Wheezy Rootfsをダウンロードして解凍します。 

 tar -xpf debian-wheezy-armhf-rootfs.img.tar.bz2 -C /mnt/sun4i-boot


ところで、カーネル3.0.36+へのモジュールもあります。 また、ユーザーuser、network-manager、nm-appletの自動ロードを備えたスリムなlxdeがセッションの開始時に開始されます。 タッチスクリーンはタッチパッドモードで動作します。

udevは必要なすべてのモジュールをロードする可能性がありますが、確かに、それらは/ mnt / sun4i-rootfs / etc / modulesに記述できます。 必要なモジュールのリストは、TerminalEmulatorを介してAndroidOSで取得できます。

TerminalEmulator：$ lsmod






ADBを介して： 

 adb shell lsmod


Linuxブートセットアップ


最初のブートパーティションの内容を含むアーカイブをダウンロードします 。

そして、アーカイブから/ mnt / sun4i-bootのevb.bin構成ファイルを抽出します。 

 tar -xf debian-wheezy-armhf-boot.tar.bz2 evb.bin -C /mnt/sun4i-boot


カーネルをビルドしなかった場合、アセンブルされたカーネル3.0.36+も抽出できます。 

 tar -xf debian-wheezy-armhf-boot.tar.bz2 uImage -C /mnt/sun4i-boot


打ち上げ


おそらく、すべてが準備できています。 システムをSDドライブにインストールしました。 microSDカードをタブレットのカードリーダーに挿入し、電源を入れます。 電源を入れてから5秒後に、バックライトが点灯し、画面が黒のままになります。 数分後、システム全体が起動するはずです。

ルートのデフォルトパスワードはrootであり、ユーザーはuserであることに注意してください。 変更することをお勧めします。

USBキーボードがない場合は、rootfsに仮想キーボードがプリインストールされています。

WiFiネットワークに接続している、またはUSBイーサネットアダプターを使用しており、nm-appletを使用してネットワークに接続しています。 ssh経由ですぐに接続できます。

lxterminalを起動し、必要なソフトウェアを提供します。 

 sudo -i apt-get update apt-get install vlc geany


考えられる問題



  1. 電源を入れても何も表示されない場合は、ダウンロードが進行中の可能性があります。5分ほど待ちます。ただし、ハイライトが表示されない場合は、evanda.binをnandaにあるAndroidファームウェアの設定ファイルに置き換える必要があります。

    Terminal Emaulatorまたはadbを介したAndroidのデバイス： 

     adb shell su mkdir /local/data/tmp/nanda mount /dev/block/nanda /local/data/tmp/nanda


    何らかの方法でscript.binを抽出します。これを行いました。

    $ adb pull /local/data/tmp/nanda/script.bin evb.bin 

     cp evb.bin /mnt/sun4i-boot/


  2. バックライトはまだあるが、システムが起動していない場合は、タブレットからMicroSDカードを取り外して/ mnt / sun4i-rootfsにマウントすることにより、システムログを表示できます。 

     cat /mnt/sun4i-rootfs/var/log/dmesg cat /mnt/sun4i-rootfs/var/log/Xorg.0.log


  3. タッチが機能しない場合は、Androidでどのモジュールがタッチの原因であるかを確認する必要があります。

    ADBによる方法： 

     adb shell 'lsmod | grep ts'
