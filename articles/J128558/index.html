Clonezillaサーバー-夜の100台のPCのバックアップはもう問題ではありません

Ubuntuの誤った更新またはWindowsの別のウイルスの悪戯による問題に慣れていない場合、システムをひざから持ち上げる必要があり、それは退屈になります。



バックアップはシステム管理者の保険です。 通常は、1Cデータベース、ファイルとフォルダー、Oracleデータベース、DHCP構成など、回復できないほど失われる可能性のあるデータをバックアップします。 HDDが突然死んだ場合、普通の店員が働いているPCはどうなりますか？ ワールプールが始まります-OS、ロールソフトウェア、会計、プリンターなどを配置します。

しかし、トップディレクター、主任会計士、および他の重要な人物はどうでしょうか。第一に、以前の状態を必要とし、第二に、30〜60分で動作し調整されたコンピューターです。



Clonezilla Server（CE）を使用してください！



CEは、ネットワークの兄弟であるClonezillaの論理的な継続です。

参考までに、Clonezillaは無料のオープンソースソフトウェアであり、ディスクと個々のハードディスクパーティションのクローンを作成し、バックアップと災害復旧システムを作成します。



Clonezilla自体は天の恵みです。 LiveCDをダウンロードし、ブートし、数回クリックするだけで、パーティションまたはHDD全体を別のHDD（外部USBドライブ）にバックアップします。 もちろん、コピーはビット単位のコピーではddではなく、使用されるディスク容量と圧縮された形式で等しくなります。



Clonezillaは、FAT、NTFS、ext2、ext3、ext4、reiserfs、reiser4、XFS、JFS、JFS、VMFS、HFS +などのFSと友達です。



つまり 1回限りのバックアップ操作Clonezilla-以上。

重要なお知らせ：CEはソフトウェアRAIDを区別せず、個別のデバイスに分割します！



CEは次の場合に役立ちます

  1. 同じタイプのPCがOSなしで多数あります。 OSと必要なソフトウェアを最初のPCに配置し、「キャスト」を作成し、ネットワーク上で同時に残りのPCに展開します。
  2. ネットワーク上の1台以上のPCのバックアップ（できれば夜間）。


CEのインストールについては何度 も書かれているので、私はCEのインストールについて何の意味もありません。



夜にPCを起動する方法は？ Wake On Lan！ マザーボードのWOLサポートを確認し、オンにする手順をお読みください。 ところで、私のp5b-deluxeは起動できず、安価なP5KPL-CMには問題がありませんでした。



CEを設定することで、そのことには柔軟性がないと確信できます。 DHCPを設定してステーションをロードし、CEサーバーのIPアドレスと複製されたシステムのIPアドレスが最大3（！）桁一致することを確認します。 不一致がある場合、NFSサーバーのIPアドレスを必要に応じて設定します。



したがって、次の便利な変更点



ネットワーク上で既に発生したDHCPサーバーで、次を構成します。

  1. 066パラメーター-CEサーバーのIPアドレス
  2. 067パラメーター-pxelinux.0


次に、DHCPサーバー名のチェックを削除します。

/opt/drbl/sbin/mknic-nbi -cn







次に、IPチェックを少なくとも2桁目に拡張します。

  1. ファイル/ tftpboot / node_root / sbin / initを編集します
  2. IP_prefix = "$（echo $ IP | cut -d"。 "-f1-3）"をIP_prefix = "$（echo $ IP | cut -d"。 "-f1-2）"に変更します。


バックアップの宛先ディレクトリをローカル/ホーム/ partimagからバックアップサーバーのSMBボールに変更します。このため、SMBボールパラメーターを/ tftpboot / nodes / $ IP / etc / fstabファイルに追加し、テンプレートを作成します。

/ opt / drbl / sbin / drbl-gen-ssi-files $ IP



最後に 、IPでコンピューターを起動してclonerHDD HDDをバックアップするスクリプトを作成しました 。



まだある：

  1. バックアップローテーションを作成して、1台のマシンからのバックアップが2つ以下になるようにします
  2. バックアップステータス、サイズ、マシン名をNagiosに送信する
  3. コマンドでバックアップからの回復を追加してスクリプトを追加します
