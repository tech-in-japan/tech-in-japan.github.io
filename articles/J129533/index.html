工科大学のUbuntu

Linux全般、特に工科大学の教育プロセスでUbuntuを使用した経験についてお話したいと思います-トレーニング分野の学生のためのIvanovo State Textile Academy-「組織および技術システムのモデリングと研究」。



開始する前に、アカデミーネットワークの作業を提供している部門のすべての従業員とYu.A. Kosinovに個別に感謝したいと思います。 サポートとヘルプ。 始める前に、各項目について、理由や設定の詳細な説明を含む数十枚のテキストを書くことができますが、これはレビュー記事の範囲をはるかに超えています。



なぜUbuntu自体なのか？



分布は、いくつかの基準に基づいて選択されました。 最も重要な要因の1つは、配布キットの使いやすさと普及率でした（Fedoraでは、rpm循環依存関係についてはあまり良い経験はありませんでしたが、これは以前のバージョン1または2にありました）。 多数のソフトウェアとインストールの容易さも重要な役割を果たしました。 この記事の残りの部分では、主にこのソフトウェアを確認します。



人文科学は簡単です



一般にオープンシステム、特にUbuntuを使用した経験から、「技術」よりも「人文」を移植する方が簡単であることがわかります（Linuxでネイティブに動作する1Cのリリースではさらに簡単です）。 そして、そのような一見矛盾する理由のリストの最初の場所は、CADシステム、つまりCADにあります。なぜなら、物事はCAM / CAEの方が優れているからです。



技術者が必要なのは何ですか？



専門分野は間接的に情報技術と結びついています-それ自体が目的ではなく、手段です（ただし、新しい教育基準での役割は拡大しています）。 主なタスクは、カウント、描画、書き込みのように説明できます。 さらに、「描画」はモデリングも意味します。 これら3つのタスクに基づいて、アプリケーションソフトウェアが選択されました。



カウントする



伝統的に、大学はMatlab / MatCad / Mapleなどを使用しています。 一般的に、どのライセンスを持っている人が使用されます。 Matlab + Simulink + SimMechanicsのライセンスがあり、伝統的に使用しています。 場合によっては、プログラミング言語を使用して「カウント」します。 言語ではもちろん、Ubuntuですべてがうまくいきます。Python、PyQT / PyGTK、Numpy、matplotlib、Eclipse + Pydevを使用していると言って、それについては詳しく説明しません。 OpenCvにはC ++を使用していますが、Pythonで料理できない場合がありますか？-遅いことがわかります）。

しかし、Matlab + Simulink + SimMechanicsの代替/追加は、個別に言及する価値があります。 もちろん、Linux用のバージョンをインストールできます（必要に応じてオーバーロードしないようにインストールする可能性が最も高い）が、Matlabにはまだ何かがありません。たとえば、シンボリックモジュールは購入されませんでした。 したがって、交換のために、次のキットを使用します（PythonおよびNumpyに加えて）-Scilab + Scios、Maxima、およびごくまれにOctave。 私の意見では、MaximaはMatlabの2つのシンボリックパッケージよりも優れているため、最も頻繁に使用します。 Scilab / OctaveはMatlabを置き換えることができますが、Simulinkの代わりとしてのSciosは非常に怖いです。 0 pのプログラムについては注目に値しますが。 0警官 彼はまともです。 SimMechanicsにお願いしますか？ 非常に具体的な問題を解決するため、一般に論理的な代替物はまだありません。



描くだけでなく



すでに述べたように、移行の複雑さの主な理由はCADです。 伝統的に世界中で彼らはAutocadを使用しています...悪魔、私たちはしていません。 ここでのポイントは、ライセンスとお金だけではありません（ただし、そもそもそれらにはあります）。 コンパスで2D / 3Dを描き始めた人-頭を壊さずにAutoCADに戻ることはありません。 しかし、コンパスのほかに、SolidWorks + CosmosWorks + FloWorksを使用しています。これは、ほとんどの設計およびモデリングタスクをカバーできる1つの大きなソフトウェアパッケージです。 SolidWorksをインストールするための価格を見るだけで、まともなオープンな代替案を忘れることができることはすぐに明らかです。 ライセンスがあるため、Wineの下にCompass 3D LTとSolidWorksをインストールしようとしました。 コンパスがインストールされ、すぐに動作を開始したため、SolidWorksはインストールを拒否したため、代替ソリューションの検索が開始されました。 残念なことに、実験は1年以上前に行われました。たとえば、このレビューを書く準備のためにWineのWebサイトにアクセスしたとき、SolidWorks 2009はゴールデンステータスであることがわかりました。

ほとんどの場合、Linux用のCADシステムを選択または確認する際には、まずプログラムの結果を確認し、その後、もしあればインターフェースを確認する必要があります。 私たちが見た、または使用する予定の短いリスト：



Blender [1]は素晴らしいことです（特にPythonでモジュールを書くことができるので）。 3DsMaxの代替として、完全に適合します。 また、学生は自分のゲームエンジンでいくつかのゲームを作成しましたが、CADとして使用することは非常に困難です（図1）。

図 2 LibreCad

OpenCasCade [3]-LinuxFormat雑誌の1つで、この非常に強力なライブラリとそれに基づくシステムが説明されました。 OpenCasCadeを使用すると、独自のCADシステムを構築できますが、このタスクは我慢できず、意味がありません。 OpenCasCadeをベースに構築されたFreeCad [4]（図3）の非常に有望な意見プロジェクトに参加することをお勧めします（内部のPython。適切な言語を選択できたことを嬉しく思います）。



図 3 FreeCad

BRL-CAD [5]-ターミナルで3Dモデルを作成しますか？ これについて学ぶと、学生はショックを受けます。21世紀にどうなるでしょうか。 ただし、システムには1つの大きな欠点しかありません。 そして奇妙なことに、それは通常の手段を使用してサイズを決定することができないことです。 しかし、モデルのテキストによる記述には、モデルの作成速度、リビジョン管理システムを使用してバックアップを作成する機能、コラボレーションなど、多くの利点があります（図4）。



図 4 BRL-CAD

OpenScad [6] -BRL -CADのイデオロギーに非常によく似ています。小さい方がよりシンプルで便利です。 すべての力も本文にあります。 （図5）

エレクトロニクスおよびマイクロプロセッサ技術



学生にマイクロコントローラの基本を教えるために、有名なArduinoデバッグボードを作成しました。 Ubuntuには、アプリケーションを開発およびダウンロードするためのIDEと、回路を開発するための環境がすぐに使用できます。 追加のモジュールを開発するには、KiCad [9]を使用します（図7）。 もちろん、KiCadでは、PiCadからの脱退者には少し珍しい瞬間があり、Altiumのすべての機能が欠けていますが、これは素晴らしいプログラムであり、その類似物にそれほど劣っていません。 回路をシミュレートするには、これを行う必要はほとんどありませんが、Qucs [10]を使用します（図8）。
