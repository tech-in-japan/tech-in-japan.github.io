シリーズの前のパートでは、新しいRAD Studio XE5の主な機能を確認しました。 今日は、練習に移りましょう。 まず、タスクを決定しましょう。



問題の声明




アカデミックなタスクがいくつかありますが、例としてクックブックを選択しました。 また、開発したアプリケーションに独創性を持たせるために、いくつかの追加機能を使用して、このようなプログラムの通常の機能を拡張しようとします。



必要な製品の量の再計算。


通常、レシピは使用される製品の正確な量を示します。 しかし、サービングの数を変更したい場合は、それに応じて、製品の数を再カウントする必要があります。 料理を準備する過程で、これは必ずしも便利ではありません。



タイマー


料理に慣れていない場合は、特定の操作の時間を明確に示すレシピを使用する必要があります。 また、タブレット、電話、またはネットブックのタイマーは、キッチンタイマーの便利な代替品になります。

多くの場合、複数の操作が並行して実行されます。 各操作には独自の「タイマー」を設定できます。

同時に、アプリケーションの「デスクトップ」とモバイル実装の両方に関心があります。 モバイルアプリケーションは、レシピを視覚化し、準備プロセスに付随するツールとして使用されます。 デスクトップバージョンは、新しいレシピの作成など、より高度な機能で使用できます。




原則として、プログラミングの本のような料理の本は、読者の準備のさまざまなレベルに合わせて設計できることに注意してください。 私たちの場合、「ダミーのレシピ」、つまり製品の正確な数と特定のアクションの正確な時間を示すものについて話しています。



このアプリケーションをWindowsおよびAndroid向けに実装しています。 次に、単一のソースコードベースに基づいて、アプリケーションをMacOSおよびiOSに移植できます。



開発ツールの選択




開発に直接進む前に、ツールを決定しましょう。 もちろん、アプリケーションで使用されるデータは、最も適切にはデータベースに保存されます。



実験の純度のために、 SQLiteをDBMSとして使用します。 このDBMSには、速度、使いやすさ、リソースに関する費用対効果など、多くの利点があります。 単純なタスクに最適であり、さらにAndroidにはSQLiteサポートが組み込まれています。



将来的には、 InterBase DBMSを使用してアプリケーションを移行する方法を検討し、Embarcaderoの「ネイティブ」ソリューションのすべての利点を示します。



アプリケーションとともに、個別に作成された既製のDBMSを配布します。 このアプローチは、タスクの詳細によるものです。



もちろん、 Delphi XE5が必要です 。 Windowsアプリケーションを作成するには、原則として、 Professionalエディションに制限することができます。 しかし、いわゆる モバイル開発では、少なくともEnterpriseエディションの製品が必要です。 または、 Delphi XE5 Professionalのモバイルアドオンパックを使用します。



WindowsでSQLiteを使用するには、公式Webサイトからsqlite-dll-win32-x86-3080100.zipライブラリをダウンロードする必要があります。 このライブラリをシステムフォルダの1つ（たとえば、Windows / SYSTEM32）に配置する最も簡単な方法。

現在、非常に多くの無料のSQLiteデータベース管理ツールがあります。 そのうちの1つを使用することもできます。



FireDACライブラリを使用して、アプリケーションからデータベースにアクセスします。



FireDACは、さまざまなデータベースに接続するアプリケーションを開発するために設計されたユニバーサルデータアクセスライブラリです。 FireDACを使用すると、VCLとFireMonkeyアプリケーションの両方を開発できます。 このライブラリは、 InterBase 、 SQLite 、 MySQL 、 SQL Server 、 Oracle 、 PostgreSQL 、 DB2 、 SQL Anywhere 、 Advantage DB 、 Firebird 、 Access 、 Informixの DBMSをサポートしています。 ただし、この段階では、 FireDACを使用する「直接」（DataSnapテクノロジーを使用しない）モバイルアプリケーションでは、 SQLiteとInterBaseにのみ接続できます。



データベース作成




データベース構造を作成して、開発プロセスを開始します。 その論理モデルを図に示します。





このチャートは、Embarcadero Technologies- ER / Studioの別のツールを使用して取得されました。 この製品のDeveloper Editionは、 RAD Studio Architectのユーザーが利用できます。 この出版物の枠組みでは、 ER / Studioの説明については触れません。 この強力なツールは、別個の素材を使用する価値があります。




データベース構造を作成するとき、論理モデルの各エンティティには物理テーブルがあります。 結果のテーブルの目的を簡単に検討してください。



tblFoodstuffテーブルでは、可能なすべての製品のリストを保持します。 tblUnitテーブルで、測定単位（グラム、スプーン、グラスなど）のリスト。 tblRecipeテーブルには、レシピのリストが含まれています。 テーブルtblIngredientesおよびtblActionには、それぞれ成分（ qtyフィールドの数量を示す）とアクション（時間と説明を示す）のリストが含まれています。



SQLiteデータベースを作成するには、次のスクリプトを使用するだけです。 



CREATE TABLE [tblAction] ( [Id] INTEGER NOT NULL PRIMARY KEY AUTOINCREMENT, [IdRc] INTEGER NOT NULL, [Sec] INTEGER NULL, [DescriptionShort] VARCHAR(128) NOT NULL, [DescriptionExtendent] TEXT NOT NULL ); CREATE TABLE [tblFoodstuff] ( [Id] INTEGER PRIMARY KEY AUTOINCREMENT NULL, [Abbr] VARCHAR(10) NULL, [Title] VARCHAR(50) NULL ); CREATE TABLE [tblIngredientes] ( [Id] INTEGER NOT NULL PRIMARY KEY AUTOINCREMENT, [IdCB] INTEGER NOT NULL, [IdFS] INTEGER NOT NULL, [IdUnit] INTEGER NOT NULL, [qty] INTEGER NULL ); CREATE TABLE [tblRecipe] ( [Id] INTEGER NOT NULL PRIMARY KEY AUTOINCREMENT, [Title] VARCHAR(150) NOT NULL ); CREATE TABLE [tblUnit] ( [Id] INTEGER NOT NULL PRIMARY KEY AUTOINCREMENT, [UnitName] VARCHAR(25) NOT NULL, [Abbr] VARCHAR(7) NOT NULL ); CREATE INDEX [IDX_TBLACTION_Recipe] ON [tblAction]( [IdRc] ASC ); CREATE INDEX [IDX_TBLINGREDIENTES_] ON [tblIngredientes]( [IdCB] ASC ); CREATE INDEX [idxFoodStuff] ON [tblIngredientes]( [IdFS] ASC ); CREATE INDEX [idxRecipe] ON [tblIngredientes]( [IdCB] ASC ); CREATE INDEX [idxUnit] ON [tblIngredientes]( [IdUnit] ASC );




Windowsアプリケーションの作成




アプリケーション開発プロセスを次のように編成します。 まず、Windows用のアプリケーションの基本バージョンを作成し、それをAndroidプラットフォームに移植します。 次に、両方のアプリケーションの機能を並行して拡張します。 MacOSおよびiOSバージョンのアプリケーションが最終段階になります。 既製のWindowsおよびAndroidバージョンができたら、作成を開始します。



Delphi IDEは、いくつかの新しいアプリケーションテンプレートを提供します。 この場合、 FireMonkey Desktop Applicationに関心があります 。







適切なメニュー項目を選択すると、追加のダイアログが画面に表示され、新しいFireMonkeyアプリケーションの種類（HD（High Definition）または3D）を選択するように求められます。







アプリケーションのメインフォームの名前を変更し（オブジェクトインスペクターでNameプロパティを変更します）、新しく作成したプロジェクトを保存します。 プロジェクトに新しいデータモジュールを追加し（Delphiファイルブランチの[ファイル] | [その他]メニュー）、名前を変更して保存します。







したがって、新しいプロジェクトを作成し、開発のために直接準備しました。

この場合、ドライバーIDとしてSQLiteを指定し、Databaseパラメーターの値としてデータベースファイルへのパスも指定します。 [テスト]ボタンを使用して設定を確認できます。

 function TDM.ConnectToDB: Boolean; begin try FDConnection1.Connected := True; except end; Result:= FDConnection1.Connected; end;

データ接続




既に述べたように、FireMonkeyには特別なデータ表示コンポーネントはありません。 代わりに、 LiveBindingバインディングメカニズムが使用されます。 仕組みを見てみましょう。 レシピのリストをメインフォームのTListBoxコンポーネントに接続します。 まず、レシピのリストを操作するためのデータセットを作成する必要があります。 データモジュールでは、 TFDTableコンポーネントを使用します。 プロパティを次のように設定します。





ConnectToDB関数のコードで、データセットを開くメソッドへの呼び出しを追加します。 



 FDTRecipe.Open;




ランタイムモードでデータベースへの接続を確立するには、ConnectToDB関数への呼び出しを追加する必要があります。 データモジュールのOnCreateイベントにバインドできます。 



 procedure TDM.DataModuleCreate(Sender: TObject); begin DM.ConnectToDB; end;




アプリケーション開発の初期段階では、接続中に発生する可能性のある例外を処理しませんが、これは将来行う必要があります。



TRecipeテーブルからのデータの出力をメインフォームのリストに整理します。 これを行うには、まず、プログラムのメインフォームのモジュールのUsesセクションにデータモジュールを追加します。 

 interface uses uDM, System.SysUtils, System.Types, System.UITypes, System.Classes, System.Variants,




フォームエディターで、コンテキストメニューを呼び出し、[視覚的にバインド]アイテムを選択します。 画面の下部で、 LiveBindingデザイナが開きます。 その中に、ダイアグラム要素の形式で、オブジェクトとその基本プロパティが表示されます。 ドラッグアンドドロップするだけで、図に示すように3つのオブジェクトのプロパティを接続します。







実際には、これがレシピテーブル（つまり、レシピ名のリスト）からのデータを表示するために必要なすべてです。 ここでプログラムを実行すると、リストが表示されますが、空であるため、レシピを追加する方法に注意する必要があります。



新しいFireMonkey HDフォームを作成します。 プロパティを次のように設定します。

データモジュールを接続し、TEditコンポーネントと2つのボタンをフォームに配置します。 LiveBindingデザイナで、TEditコンポーネントのTextプロパティをデータセットのTitleフィールドに関連付けます。







新しいフォーラムを保存し、自動作成されたフォームのリストから削除します（[プロジェクト] | [オプション]メニューの[フォーム]タブ）。



アプリケーションのメインフォームの場合、メソッドを作成します。 

 procedure TfMain.RecipeAfterInsert(DataSet: TDataSet); var fAddRecipe: TfAddRecipe; begin try fAddRecipe:= TfAddRecipe.Create(Application); fAddRecipe.ShowModal; if fAddRecipe.ModalResult = mrOk then begin if DataSet.State in [dsInsert, dsEdit] then DataSet.Post; end else begin if DataSet.State in [dsInsert, dsEdit] then DataSet.Cancel; end; finally FreeAndNil(fAddRecipe); end; end;




プログラムが起動したら、レシピデータセットのAfterInsertイベントを定義します。 



 procedure TfMain.FormShow(Sender: TObject); begin DM.FDTRecipe.AfterInsert:= RecipeAfterInsert; end;




ここで、アプリケーションを起動することにより、ナビゲーターを使用して変更できるレシピのリストを取得します。 新しいレシピを追加すると、入力フィールドのあるフォームが画面に表示されます。
