次のコンテンツを含むファイル/www/doodles/_build/resolvers/resolve.tables.phpを作成します。

素晴らしい。 ここではすべてが明確だと思います。 スイッチの構造があり、現在のアクションに応じてタスクを実行できます。 モデルへのパスを示し、 addPackage（）メソッドを呼び出します。これにより、xpdoスキームが追加されます（最初のレッスンから覚えていますか？）。 最後に、 $ modx-> getManager（）を実行し、次に$ manager-> createObjectContainer（ 'Doodle'）を実行します。 このメソッドは、MODXにSQLを実行し、Doodleクラスのデータベースにテーブルを作成するコマンドを提供します。 これで、最初の部分で行ったように、データベーステーブルの存在のチェックを削除できます（リゾルバーを使用する必要はありませんが、便利です）。 そして最後に、すべてがスムーズに進んだことをMODXが認識するようにtrueを返します。



これで、パッケージをインストールするときに、データベースへの追加のテーブルが作成されます。



cangelog、readme、ライセンス、およびインストールオプションファイルの追加





docs /フォルダーに次の内容のreadme.txtファイルを作成しましょう。 



 -------------------- Extra: Doodles -------------------- Version: 1.0 A simple demo extra for creating robust 3rd-Party Components in MODx Revolution.




また、 license.txtファイル（ライセンスの説明を含む）とchangelog.txt （変更ログ）を作成します（まだない場合）。



build.transport.phpスクリプトに戻って、 $ builder-> pack（）の前に次の行を追加しましょう。 



 $modx->log(modX::LOG_LEVEL_INFO,'Adding package attributes and setup options...'); $builder->setPackageAttributes(array( 'license' => file_get_contents($sources['docs'] . 'license.txt'), 'readme' => file_get_contents($sources['docs'] . 'readme.txt'), 'changelog' => file_get_contents($sources['docs'] . 'changelog.txt'), 'setup-options' => array( 'source' => $sources['build'].'setup.options.php', ), ));




ご覧のとおり、 setPackageAttributes（）メソッドが呼び出され 、パッカーに属性が設定されます。 「setup-options」と呼ばれる新しい配列もあります。 この配列には、キー 'source'を持つ要素（PHPファイルへのパス（リゾルバーなど））があります。



次の内容で/www/doodles/_build/setup.options.phpファイルを作成します。 



 <?php $output = ''; switch ($options[xPDOTransport::PACKAGE_ACTION]) { case xPDOTransport::ACTION_INSTALL: $output = '<h2>Doodles Installer</h2> <p>Thanks for installing Doodles! Please review the setup options below before proceeding.</p><br />'; break; case xPDOTransport::ACTION_UPGRADE: case xPDOTransport::ACTION_UNINSTALL: break; } return $output;




おなじみですね。 このコードにより、ユーザーがパッケージをインストールするときに「インストールオプション」を表示できます。 現在、アドオンをインストールした人に「ありがとう」と言うメッセージのみを表示しています。



ここで、パッケージをインストールするときに表示され、インストーラーによってさらに処理されるフォーム要素を追加できます。 Quipコンポーネントで例を見ることができます： github.com/splittingred/Quip/blob/develop/_build/resolvers/setupoptions.resolver.php



それだけです パッカー（http：//localhost/doodles/_build/build.transport.php）を実行すると、トランスポートパッケージファイル「doodles-1.0-rc1.zip」がcore / packages /フォルダーに表示されます。 このファイルは、MODXアドオンリポジトリにダウンロードでき、 「パッケージ管理」からインストールできます。
