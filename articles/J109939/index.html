TeamLab：1つのスタートアッププロジェクトの歴史

今年の7月の7日、午後7時。 TeamLabプロジェクトの新しいバージョン（ www.teamlab.com）のリリースが予定されていたのは、この魔法の日でした。



TeamLabは、プロジェクト管理と社内のコミュニケーションのためのシステムです。 スタートアップの主な目標は、意味のある何かを作成することです。 アイデアが新しいものでなくても、常に改善できるため、世界が少し良くなります。 それで、私たちは新しいものを導入したいと思い、現代のコラボレーションシステムのビジョンを提示しました。



私たちは自分の熱意をどうにかして得たのですか -それはあなた次第です、親愛なるKhabrovites。



背景またはイントラネットポータルを構築する必要があるもの。



2008年、同社は成功し、すでに利益を上げているITプロジェクトと、まだ開発および設計の段階にある新しいプロジェクトに取り組みました。 当然、定期的に新しいアイデアについて議論したり、興味深いWebリソースへのリンクを共有したり、開発プロセスを文書化したりする必要が生じました。 ブログ、フォーラム、世論調査、Wiki、そしてもちろん写真アルバムを作成するための機能を含むシステムを作成することが決定されました（それが無い場合）;）システムはあなたのニーズに合うように書かれているため、従業員の間で非常に迅速に定着しました。 これに続いて、プロジェクトを一般の人々と共有したいという完全に理解できる欲求が続きました。 そして、この欲求が実現しました。 2009年末に、TeamLabというプラットフォームディストリビューションがリリースされました。



社会性は素晴らしいです！ しかし、プロジェクト管理はどうですか？



配布パッケージはダウンロード、インストールされましたが、使用されていません...おそらくブログ、フォーラム、フォトギャラリーは、ほとんどのユーザーが必要とするものではなかったでしょう。 コミュニケーションはコミュニケーションですが、プロジェクトはさらに重要です。 タスクの配布、進捗の監視、ファイルの共有、レポートの作成が必要です。 当社ではプロジェクト管理ツールの必要性が成熟しているため、コミュニティオプションの拒否は新しい機能の開発を加速するだけでした。 そして、この夏、TeamLabの新しいバージョンがリリースされました。 それらが非常に標準的であるため、私は彼のツールのすべてを詳細に説明しません。 タスク、マイルストーン、ディスカッション、ファイル、レポート、オンラインチャット-私たちと同様のサービスのほとんどは、同様の機能を提供します。 唯一の違いは、この機能の実装方法です。インターフェースの利便性と使いやすさです。 彼らが言うように、一度見た方が良い...







Amazon Cloudsへの移行



その頃には、 Amazon Web Servicesにも参加し、企業ポータルをAmazonサーバーに移動していました。 したがって、彼らは私たちのユーザーに配布キットではなく、SaaSバージョンを提供し始めました。 さらに、 Amazon EC2のポータルイメージ（AMI）をリリースしました。 したがって、既にAWSアカウントを持っているユーザーは、この代替を使用して、独自の仮想サーバーでTeamLabを操作できます。 AMIなどのSaaSサービスは 、制限なしで無料で提供されます。



オープンソース：共有すれば、本当に



また、SourceForgeでTeamLabソースコードを公開しました。 はい、はい、私たちは製品のソースコードを開いた開発者です。 これを決めるのはそれほど簡単ではありませんでした。 しかし、他のどのようにして同僚の注意を新しいサービスに引き付けることができますか？



TeamLabコードを使用すると、GPLv3ライセンスのフレームワークに適合するすべてを実行できます。 もちろん、まず、コードをコンパイルして、独自のサーバーにポータルをインストールできます。 必要に応じて、自分でリメイクすることもできます。何かを削除したり、修正したりすることができます。一般に、使用すると便利です。 このリンクをたどってソースをダウンロードする前に、使用されているテクノロジーについて少し説明します。 TeamLabはASP.NETで記述されており、MySQLおよびSQLiteデータベースで動作します。 もちろん、これは一種のマイナスです。なぜなら、大多数がLinuxをホストしているからです。 それはまさにその通りで、私たちは提供に特化しており、私たちにできることは何もありません。 しかし、良いニュースがあります。 来年、プロジェクトをMonoに移植する予定です。



結果は何ですか？



TeamLabの新しいバージョンは、当社の従業員の心だけでなく、他の多くのユーザーにとっても暖かい反応を発見しました。 1,000人目の登録ユーザーを記録したように、サービスの開始後1か月も経たないうちに。 今日、TeamLabは人気のあるオンラインプロジェクト管理サービスとよく比較されます（方法は知っています）。



しかし、真剣に、このサービスは本当に機能的であり、その機能は有料のアナログに決して劣っていません。 TeamLabを実際に試すことで、これを自分で確認できます。



継続するには...



すべてのTeamLabソリューションを無料で提供しています。 可能な質問を予想して、私たちは利他主義者ではないことをすぐに言います。 そしてもちろん、今後製品を収益化する予定です。 それまでの間、次のリリースに向けて準備を進めており、新しい興味深い機能に取り組んでいます。これについては後で説明します。



以来、私たち自身のためだけでなく、ユーザーのためにもサービスを開発しています。それを改善するためのフィードバックや提案に感謝します。 また、twitter @ teamlabdotcomまたはFacebookのファンページでチャットすることもできます 。
