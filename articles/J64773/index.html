ラップトップでSSDを使用するためにWindowsを最適化する

このトピックへの回答として彼のブログに最初に公開されました。 ここに移動する機会を与えてくれた親切なhabravchanin boriskoに感謝します。



この資料は、いくつかのソースからの（可能な限り完全な）編集物です（投稿の下部にあるリンクを参照）。 この資料を準備する過程で私が持っていた質問（テキストにあります）への回答に感謝します。



IBM Lenovo ThinkPad X61s（対角12インチのこのような小さくて快適なラップトップ）を買収したので、私は仕事中に常にバスでそれを使用したいと思っていました。 しかし、バスは非常に揺れているため、ハードドライブにとっては単に怖いものになります。 これに関連して、ソリッドステートディスク（通常のフラッシュドライブとほぼ同じ、成人のみ）がOCZによって購入されました。これは信じられないほど安価で、SSDのすべての魅力を直接試すことができました。 OCZ Solid Series OCZSSD2-1SLD30G 2.5 "30GB SATA II MLC内部ソリッドステートディスク（SSD）のコピーは当時220ドルで購入され（他のすべては400から始まりました）、前述のラップトップに厳重に貼り付けられました。 、いわゆる第1世代のパフォーマンスについては、まだ多くのことが望まれていますが、速度を向上させてディスクの寿命を延ばすために何かを行うことができます。







Vistaの多くの調整はWindows 7でも正常に機能します。ちなみに、SSDに7をすぐにインストールすると、以下の設定の多くがすぐに正しい位置に表示されます。



1. Windows XPではプリフェッチを無効にし、Windows Vistaではプリフェッチとスーパーフェッチの両方を無効にします。 Prefetch / superfecthは、プログラムの実行に必要なファイルに関する情報をキャッシュするためにWindowsによって使用されます。 詳細は、Russinovichの記事をご覧ください 。 最適化は通常のディスクに対して機能しますが、ディスクのさまざまな部分へのアクセス速度は同じであるため、SSDには意味がありません。さらに、プリフェッチはシステムに作業を追加します（プリフェッチオーバーヘッド自体が遅くなります）。



XP：



Hkey_local_machine \ SYSTEM \ CurrentControlSet \ Control \ Session Manager \ Memory Management \ PrefetchParameters ... Enableprefetcherを右クリック...変更を選択し、3を0に変更し、OKで確定



Vista：

プリフェッチ： Hkey_local_machine \ SYSTEM \ CurrentControlSet \ Control \ Session Manager \ Memory Management \ PrefetchParameters ... Enableprefetcherを右クリックします...変更を選択し、3を0に変更し、OKで確定します



superfetch ： Hkey_local_machine \ SYSTEM \ CurrentControlSet \ Control \ Session Manager \ Memory Management \ PrefetchParameters ... EnableSuperfetchを右クリックし、変更を選択し、3を0に変更し、OKで確定します



2. Hkey_local_machine \ SYSTEM \ CurrentControlSet \ Control \ Session Manager \ Memory Management（ XP / Vista ）ブランチのいくつかのパラメーターを「1フロア以上」変更します。



ClearPageFileAtShutdown-0から1に変更します（ Windows XPでは逆の効果があり、シャットダウンが遅くなるため、個人的にこのフラグに触れることはお勧めしません ）

単純なGoogle検索で判明したように、これは悪いアドバイスです。コンピューターのシャットダウンプロセスを遅くし、さらに、ページファイルの非アクティブなページで占有されているセルを上書きすることで余分なパスを作成します。





LargeSystemache-0から1に変更-このパラメーターについては何も見つかりませんでした。引き続き作業します。



SecondLevelDataCache-0から特定のコンピューターの2次キャッシュのサイズに変更します。必ず10進値を変更してください。 それが何であり、MSDNを調べる必要がある理由は、現代のシステムでは関係ないはずです（未知のプロセッサを使用しない限り）。



3. HKEY_LOCAL_MACHINE \ SYSTEM \ CurrentControlSet \ Control \ FileSystem（XP / Vista）：





NtfsDisable8dot3nameCreation-0から1に変更します。そのようなキーがない場合は、作成する必要があります（DWORD）。または、コマンドラインからコマンドを実行できます： fsutil behavior set disable8dot3 1 -FAT16との互換性のために8.3形式のクラシック名の作成を無効にします。 古代のDOSプログラムをまだ実行している場合は、そのままにしておくのが理にかなっています。



NtfsMemoryUsage-値を2に変更します。そのようなキーがない場合は、作成する必要があります（DWORD）。 この設定により、ファイルの読み取り/書き込み操作のキャッシュに使用できるページのプールが増加します。 特に、ファンがすぐにdofigaアプリケーションを起動したり、ブラウザで多数のタブを開いたりするのに役立ちます。



（XPのみ）：HKEY_LOCAL_MACHINE \ SOFTWARE \ Microsoft \ Dfrg \ BootOptimizeFunction-YをNに変更します。この設定により、システムはディスク上のブート領域をさらに最適化する（ファイルを相互にドラッグアンドドロップする）ようになります。



3. [管理ツール]-> [サービス]を使用して、ディスクのインデックス作成を無効にします。 Liveデスクトップが既にインストールされている場合（Microsoftのローカルマシンで検索）-哀conの意ですが、取り壊そうとします。 Googleデスクトップに似ています。 一般に、メモリにハングアップしてディスクに何かを書き込むプログラムはすべて無効にすることをお勧めします。 特に勇敢な人は、常駐アンチウイルスを無効にしたり、インストールしないこともできます。



4.ハードウェアまたはデバイスマネージャーからディスクへの書き込みのキャッシュを有効にします->ディスクドライブ-> SSD->プロパティ->ボックスでディスクの書き込みキャッシュを有効にし、高度なパフォーマンスを有効にします（このフィールドがある場合）。



SSDでは書き込みが最も遅く、読み取りが最も速いので、書き込みキャッシュのすべての可能なオプションを有効にし、読み取りキャッシュを無効にすることは理にかなっています。 システムを直接SSDにインストールした場合、この設定はデフォルトで設定されます。



5.スワップファイルを無効にします。 これは、[システムのプロパティ]-> [詳細設定]タブ-> [パフォーマンス]フレームの[設定]で実行できます。 開いた[パフォーマンスオプション]ウィンドウで、[詳細]タブを選択し、一番下の[仮想メモリ]フレームで[変更]をクリックし、[ページングファイルなし]を選択し、[設定]をクリックし、[OK]をクリックしてウィンドウの迷路から出てマシンを再起動します。



この点は、最初は最も物議をかもしているように思えました。 ただし、Windows XPとWindows 7の両方でテストした結果、3ギガバイトのメモリを搭載したラップトップでは、スワップの不足から落ちる唯一のプログラムはGIMPであると確信できます。 Visual Studio 2010 for Windows 7は、メモリ不足を誓いますが、ゆっくりではありますが、それでも動作します。



遅延-パフォーマンスオプションの同じセクションで、最初のタブ（視覚効果）でパフォーマンスの最適化を選択できます。 システム自体がハードウェアに適応しようとします。 個人的には、同じ設定を手動で調整することで、同じ（またはより良い）効果を達成しています。 しかし、私はITで、とても気に入っています:)



6.休止状態を無効にします。 切断の主な理由は、ディスクリソースを保存することです。休止状態ファイルは同じ物理的な場所にあるため、ストレージに使用されるセルはリソースをより速く生成します。 予防のため、ファイルを月に1回削除してデフラグし（SSDディスクの場合は2〜3か月に1回以上行うことはお勧めしません）、ファイルを再作成します。 ほぼ間違いなく、彼は新しい場所に登場します。



7. Windows XPの所有者には、使用が強く推奨されるユーティリティRAMDiskがあります。 RAMディスクの下にある3ギガバイトのうち1つが割り当てられ（XPがラップトップにまだインストールされていたとき）、その上でスワップを768 MBに作成し、そこでInternet ExplorerとFirefoxブラウザーのキャッシュをリダイレクトしました。 したがって、ほぼすべての一時ファイルは仮想ディスク上にあり、再起動するたびに安全に応援されました。



残念ながら、現時点ではプログラムのあるサイトは利用できないため、コメントにある類似の（そしてできれば検証済みの）製品へのリンクを喜んでいます。



いずれにせよ、仮想ディスクを作成するためのユーティリティをダウンロードできた場合、次のようにブラウザキャッシュをリダイレクトできます。



Internet Explorer 7 ：ツール->フレームブラウジング設定->設定->フォルダーの移動。



Firefox ：空白のタブを開き、「about」と入力し、configを押してEnterキーを押し、browser.cache.disk.parent_directoryキーを見つけて（ない場合は作成します）、ff_cacheが値として配置される親フォルダーへのパスを指定します。 ところで、Firefoxには別の微調整があり、上記の喜びと同時にブラウザーの速度も加速します。 [ツール]-> [オプション]-> [プライバシー]に移動し、このメニューでほとんど使用されない項目を無効にする必要があります。 Firefoxは（たとえば、..の履歴を保持するために）、一度に複数のSQLiteテーブルでアクセスしたページごとに個別のエントリを作成します。 eeePCオーナーフォーラムの情報によると、これらのオプションを無効にすると、インターネットサーフィンのプロセスが大幅に高速化されます。



Opera ：opera：config＃DiskCache | CacheDocsまたはUser Prefsセクションのoperadef6.iniファイルでCache Directory4 = X：\ OperaCache



同様に、Windows自体のTEMPフォルダーを構成することは理にかなっています。 [マイコンピューター]を右クリックし、[プロパティ]-> [詳細設定]-> [環境変数]を選択し、[ユーザー変数]セクションで、必要なフォルダーの仮想ディスクへのパスを設定します。



8.差分バックアップに特殊なプログラムを使用しない場合は、最終アクセスタイムスタンプ（ファイルが最後に開かれたときを示す特別なマーク）を無効にできます。 これは、 fsutil behavior set disablelastaccess 1コマンドを使用してコマンドラインから実行できます（同じコマンドを実行してユニットをゼロに変更することで、コマンドを返すことができます）。



9. NTFSのログを無効にします。 コマンドラインからこれを行うかどうかを決めるのはあなた次第です： fsutil usn deletejournal / d



10. dskcacheというMicrosoftのユーティリティを使用します。 この機能の詳細と必要な理由については、 http ： //support.microsoft.com/kb/Q811392をご覧ください。



11.この機能を使用しない場合は、Windowsシステムの復元を無効にします。



12.メールクライアント（Outlook、Thunderbird）を使用している場合-メールファイルを含むファイルまたはフォルダーを非SSDドライブに転送してみてください。 システムにディスクが1つしかない場合は、スワップの場合と同様に、定期的にファイルをディスク上の新しい場所に転送することをお勧めします。



13. Vistaのみの実際-Windows Vista / Windows 7では、デフラグがデフォルトで有効になっています（私の経験から-SSDにインストールする場合でも）。 原則として、SSDドライブはまったく最適化できません（ドライブの異なる部分からの読み取り速度は同じであるため）。 ただし、それほど単純ではありません。 一部のコントローラーは、すべてのセルに負荷を分散するために意図的にファイルを断片化しますが、そうしないものもあります。そのため、明確なアドバイスを与えるのは困難です。ディスクのメーカーとコントローラーのメーカーによって異なります。 「病院平均」では、ディスクの空き容量が40％未満で、60％を超えるファイルがファイルシステムによって何らかの方法で上書きされている場合（つまり、60がある場合）、ディスクを2か月に1回だけ最適化することをお勧めします占有ボリュームの％-常に更新されるログファイル）。 これにより、ファイルの変更によって常に占有されているセルでの「過負荷」を回避できます。



14.さまざまなソースからの情報（AnandTechの記事、下のリンクを参照）によると、SSDの実際の最適化は低レベルのフォーマットです。 フォーマット処理により、すべてのセルが元の状態に戻り、新しいディスクと同じレベルのパフォーマンスを実現できます。 情報は検証されていません（ディスクを低レベルでフォーマットすることに熱心な人は多くありません）ので、自己責任で行ってください。
