A / Bテストのための100のアイデア。 パート2

A / Bテストは、サイトでのコンバージョンを増やす最も効果的な方法の1つです。 初心者にとって、テストプロセスはほとんどの場合、問題と失望のみを引き起こします。 良いアイデアを思いつくのはそれほど簡単ではありません。 したがって、A / Bテストのアイデアの最大のリストを支援し、編集することにしました。









適切なA / Bテストは非常に複雑なプロセスであることをすぐに警告したいと思います。 全体の難しさは、実験を開始しないことです。 これが最も簡単です。 全体の難しさは、仮説の定式化、テストのアイデアの検索にあります。



専門家が提唱するルールに従って、A / Bテストの潜在的に成功するアイデアを見つけるには、Web分析を使用し、ユーザーの行動を分析し、ターゲットオーディエンスとコミュニケーションを取り、調査を実施する必要があります。



そして、これらのルールは絶対に論理的です。 それらを観察する必要があります。 しかし、小さな会社、小さなチームがあり、必要な専門家がいない場合、上記のすべてを行うことは非常に困難です。



この場合、100のアイデアのリストが編集されました。 前の記事では、行動を促すフレーズ、フォーム、テキスト、視覚要素をテストするための51のアイデアを既に検討しました。



この記事では、A / Bテストの残りの49のアイデアについて説明します。 サイトで最も関連性の高いアイデアを研究、採用、テストします。



ソーシャルボタンをテストするA / B





近年、ソーシャルネットワークは、ターゲットオーディエンスとの相互作用の最も効果的なチャネルの1つになりました。 それらを通して、あなたはターゲットオーディエンスを引き付け、それとコミュニケーションを取り、さらには販売をすることができます。 各サイトには、ソーシャルボタンがあります。 グループを使用してより多くの視聴者をグループに引き付けるために使用する人もいれば、コンテンツを共有するために使用する人もいます。 いずれにしても、ボタンはほぼどこにでもあります。 また、テストの対象になる場合があります。

54.ボタンのデザインを試してください 。 クリエイティブなものを使用している場合は、標準設計を試してください。 クリエイティブはもちろん良いことです。 しかし、長年にわたって人々は特定の基準に慣れてきており、その違反は多くの場合否定的な結果につながります。 私の意見では、ソーシャルボタンには標準的なデザインが必要です。これは、最も人気のあるサイトで使用されています。

56.ソーシャルネットワークでアクティブな場合は、グループ/公開ページへのリンクを含むウィジェットを追加します。 これにより、購読者の数を増やすことができます。

A / Bテストページと価格









顧客が購入を決定するとき、価格は最も基本的な要因の1つです。 結論は？ これは、販売目標到達プロセスの重要なページの1つです。 したがって、サイトのこの特定の要素のA / Bテストは、コンバージョンと販売を増やすための実際の方法の1つです。



テストのアイデア：




66.いくつかの関税計画がある場合、テスト版ではそのうちの1つを「最も人気のある」ものとして強調表示します 。 サイズ、色、追加のテキストなどで区別できます。 たとえば、その横に「最も人気のある」または「最も収益性の高い料金プラン」という碑文を挿入できます。 最低価格または最高価格を選択しないでください-平均的なものを選択することをお勧めします



67.価格自体を試してください。 確かに、100ルーブルの代わりに99を書き込むとき、あなたはそのようなトリックに精通しています。そして、このテクニックはほとんどどこでも使われています。 おそらく動作します。 A / Bテストでこれをテストしてみませんか。 以下は、このトピックに関する非常に素晴らしい記事です。 みんなにお読みになることをお勧めします！

68.この製品または料金プランの今日の割引、プロモーション、特別オファーのみが有効であるという情報を追加し、これが販売に与える影響を分析します。



69.期間限定で製品を無料でテストする機会を追加します。 ほとんどの場合、顧客は支払いを決定する前に製品に「触れる」必要があります。 そして、テスト期間が終了すると、有料ユーザーに変換するのがはるかに簡単になります。 そのため、このような関数を追加して、これが有料ユーザーの成長と将来の販売量にどのように影響するかを分析してみてください。



70.無料試用版（限られた期間のフルアクセス）と比較して、フリーミアムプラン（限られた機能でのアクセス）の有効性をテストします。 多くの場合、すべての機能をテストする能力が不足しているため、製品を拒否して代金を支払いたくないことがあります。 この場合、完全な機能を備えた1週間または2週間のトライアルは、クライアントベースの増加に非常に有益な影響を与える可能性があります。



71.無料試用期間をテストします。 より多くは良くありません。 ある実験では、KISSMetricsは、14日間のトライアルが30日間のトライアルよりも有料の顧客への変換に良い影響を与えることを発見しました。 ユーザーにこのような長い無料期間が提供されると、多くのユーザーは後日テストを延期し、その後は製品について完全に忘れます。 この場合、次の週のテストを延期するには14日では短すぎます。



72.訪問者が製品を初めて知ったページに価格ブロックを配置します。 一部のサイトでは、価格を見つけることが非常に難しい場合があります。 実際、なぜ誰かが最後の瞬間までコストを隠そうとしているのかは明らかではありません。 しかし、多くの顧客にとって、これは決定の基礎となる基本情報です。 したがって、彼らがそれを見つけなければ、彼らはあなたに個人的に連絡したいと思わない。 サイトを離れ、競合他社のサービスを利用する方が簡単です。 Web分析を使用して、潜在的な顧客にとって価格設定情報がどれほど重要かを判断できます。 たとえば、Googleアナリティクスでは、訪問者が最も頻繁にアクセスするページを確認できます。 価格ページがそれらの間でリードする場合、この情報は彼らにとって非常に重要です。



73.関税計画のページで、1種類ではなく、2種類の支払い（月払いと年払い）をテストします。 また、顧客がより快適で、より良い支払いを受けている方法を確認してください。 結局のところ、その年の支払いはあなたにとってはるかに有益です。クライアントはすぐにあなたに執着します。 そして来年、彼のビジネスプロセスがあなたのサービスに大きく依存している場合、彼があなたを拒否することはさらに難しくなります。



74.製品の信頼性の保証、取引のセキュリティに関する情報を追加します。 そして、これが売上成長にどのように影響するかを確認してください。 また、価格と行動を促すフレーズの横にレビューを配置することもできます。 これにより、優柔不断なクライアントに対する信頼がさらに高まります。



75. 30日間の保証の追加をテストする：顧客が何かに満足していない場合、あなたは彼にお金を返します。 この小さなメモが、クライアントに購入/購読を促すトリガーになることがあります。 その後は、あなた次第です-クライアントは処理を開始し、彼があなたと一緒にいることを望みます



76.何かを無料で提供する場合は、現金相当のメモを追加します。 例：「無料コース：30日間でコンバージョンを2倍にする方法（コスト300ドル）」 人は、同様の為替レートが金銭的にどれくらいかかるかを知るとすぐに、この申し出をよりよく理解し始めます。 これにより、彼は特定のアクションを実行するインセンティブが増えました。電子メールを残す、再投稿するなどです。



77.顧客が特定の金額で購入するという条件で、無料配送を提供する場合、この要素は常に見えるはずです。 第一に、送料無料は購入の動機の1つです。 第二に、顧客が少額の注文をして、配達のために追加料金を支払わなければならないことに気付いた場合、これは彼が何かを買うように刺激します。 より多くの購入-より多くの販売-より多くの利益。 それでテストしてください！



A / Bセールスファンネルテスト









セールスファンネルは、サイトとの最初のコンタクトから最終的なターゲットアクション（製品またはサービスの販売、登録、サブスクライブ、ファイルのダウンロード）まで潜在的な顧客を促進するプロセスです。



例として、オンラインストアを検討できます。 最初の段階でユーザーはメインページに移動し、検索を通じて適切な製品を見つけてバスケットに追加し、支払いページに移動して、最終的に購入します。 この場合、目標到達プロセスは次のようになります。サイトのメインページ>製品検索>製品ページ>カートに追加>支払い。



明らかに、販売目標到達プロセスの各段階でコンバージョンを増やすことができれば、利益が増加します。 したがって、ここでA / Bテストを実施できます。



テストのアイデア：




78.ユーザーが支払いデータを入力し、他の主要なアクションを実行するページの注意をそらす情報（他の製品、プロモーションなど）をすべて削除します。 Amazon.comでの支払いプロセスを見ると、気を散らす要素はほとんどないことに気付くでしょう。 もしそうであれば、支払いページを離れる可能性が高くなるため、多くの場合、これはクライアントを失うことに等しい。



79.セールスファネルが多数のステージで構成されている場合は、削減するようにしてください。 たとえば、意思決定に重要な役割を果たさない場合は、一部のページを結合したり、販売ファネルから削除することもできます。 これにより、ユーザーのアクションの数が減ります。 結局のところ、誰もがほとんどの場合、小さくシンプルであるほど良いことを理解しています。



80.支払いと個人データの入力のプロセスがいくつかの段階で構成されている場合、プログレスバーを試してください。 ユーザーが自分が5段階の3番目の段階にいることを確認すると、これが無意識のうちに影響します。 これにより、彼が仕事を終える可能性が高まります。



81.個人データと支払い情報を入力する段階で、見込み顧客を販売目標到達プロセスから誘導する他のページへのリンクを削除します。 ユーザーが配信情報を知る必要がある場合は、販売ファネルのページの1つに追加しますが、決して支払いに戻らないと思われるページに送信します。
