WiFiネットワークでのローミング-802.11i / r / k / v / OKC、本当に必要なものと認識方法

ローミングに関しては、通常、2つの異なるプロセスがこの概念の下に隠されています。 以前に登場したセルラーネットワークの世界では、ローミングとは「外部」ネットワークで動作する能力を意味し、基地局間のシームレスな移行（ハンドオーバー）を意味しません。 セルラーネットワークのBS間の感知できない動きは非常に自然であるため、ほとんど覚えられていません。



WiFiの世界では物事は異なり、ローミングは通常、同じネットワークのアクセスポイント間でユーザーの目に見えない動きを意味します-近い将来にSMS認証が広く導入されると、オペレーターはセルラーインフラストラクチャのスタイルで他のWiFiネットワーク間のローミングの標準を導入するよう促されるはずです彼女の識別に基づいています。



以下は、不慣れな機器での既存のローミングテクノロジーとその検出方法の説明です。読者はWiFiの基本原理に精通していることが前提となります。



WiFiネットワークのスイッチングローミング（ハンドオーバー）をセルラーネットワークの観点から評価する場合、最も正確な説明はこれになります。標準では提供されておらず、長年にわたって状況は変わっていません。 セルラーネットワークでは、サブスクライバーを別のBSに切り替えると、クライアントからの情報メッセージに基づいてネットワークコントローラーが開始され、近隣の拠点からのクライアントの信号を評価します.WiFiでは、クライアントは常に切り替えることを決定します-拠点はそれをより速く行う方法しか伝えられません。 しかし、WiFiには、アクセスポイントを50ミリ秒に変更し、IP経由で加入者に音声通話を保存するプロセスを成功させることができる多くの標準化された「クランチ」があります。 （ユビキタスゼロハンドオフは、松葉杖が以前より悪化した例です）。 ここでは、著者に簡単に石を投げることができますが、802.11r / vについてはどうでしょう-しかし、それらはWiFiのフレームワーク内で必須ではなく、すべてのデバイスでサポートされているわけではなく、帯域幅予約による強制変換のようなものを意味するものではありません。 どこで、いつ切り替えるかを選択します-すべて同じことがクライアントに残ります。 さらに、802.11rを含めると、次のように、古いクライアントのネットワークに接続できなくなります。 これは802.11フレームの必須オプションです！ 場合によっては、不要なものではなく、有害なもの（古いドライバー、プリンター、スキャナーなどのデバイス）があります。



理論



一般的な入門ノートがあるので、ローミングのためにWiFiで何がなぜ役立つのかを簡単に説明する価値があります（これを呼び出します）。



802.11i



2007年に標準で導入された2004年の修正では、セキュリティに焦点を当て、認証と暗号化（WPA2）について説明しています。 興味があるのは、 キー交換手順と外部リソース（RADIUS）との相互作用により、AP間のクライアントスイッチングが大幅に遅くなります。 クイック再接続の最初の原則について説明します-PMKキーを保存しますが、クライアントがすでに完全な手順を1回完了している場合のみです。 ネットワークにすばやく戻ります。



OKC（日和見キーキャッシング）



802.1x認証のプロセスで最初に知られている松葉杖は、アクセスポイントが各クライアントのペアワイズマスターキー（PMK）を保存します。アイデアは、このキーをコントローラーを介して隣接ポイントに送信することでした。これにより、RADIUSへの新しい呼び出しがなくなり、交換が大幅に削減されます新しいポイントに切り替える時間。 これは標準の一部ではないため、すべての結果が発生しますが、WiFi-ironのすべての深刻なメーカーと一部の顧客によってサポートされています。 クライアントからのサポートがなければ、この機能は役に立ちませんが、WPA2-PSKでも同様です。 一部のベンダーは、クライアントがRequestで要求しなかった場合でも、保存されたキーを見て、強制的にメソッドを使用しようとします。



802.11k



無線リソース管理 、標準、オプションの2012年以降の2008年の修正。 フラグの付いたアクセスポイントは、オプションのビーコンサポートを示し、クライアントから要求された場合、隣接ポイントのリストを送信します。クライアントは、使用可能なすべてのチャネルのスキャンに時間を浪費せず、すぐに目的のチャネルに切り替えて、新しいポイントを選択します。 バッテリーは節約され、高負荷はエーテルの全体的な状態も改善します。 802.11vと併用することで、クライアントの生活を他のテクノロジーについて考えないように十分快適にすることができます（結局、クライアントは候補ポイントを自分で選択します）-もちろんVoIPと50 msマジックがWPA2-Enterpriseにとって重要でない限り。 カスタマーサポートがなければ、役に立ちません。



802.11v



2011年および2012年に公開されたワイヤレスネットワーク管理（WNM）の修正は、多数のオプションである標準になりました。 主な目的は、無線環境の効果的な管理（ステーション間の環境データの交換、クライアントの省エネ、ローミングとバランシングプロセスの改善）です。メッセージは適切なAPでクライアントに送信され、ポイント過負荷（ロードバランシング）と弱い信号でクライアントが「スタック」する問題に対処します。その他の機能。 アシスト省電力は、頻繁なキープアライブメッセージを必要とせずにクライアントの最大タイムアウトを設定します。DirectMilticast Serviceでは、セル速度ではなくクライアントの接続速度でマルチキャストフレームを受信できます。これにより、エーテルが解放され、バッテリーが節約されます（ローミング時、これらの機能適用しません）。 しかし、BSS移行は非常に関連性が高く、フレームワーク内には3つのタイプのメッセージがあり、これは適切なポイントを示すクライアントからの要求と、ポイントからの2つのメッセージです-ポイントが過負荷の場合の負荷分散要求RSSIおよびデータレートパラメータがAPの最小要件を満たしていない場合。 これらは助言メッセージであり、アクションはクライアントの裁量に任されていることに注意することが重要です。 強制切断は、独自のバンド/ロードステアリング/バランシングテクノロジーのフレームワーク内でのみ可能であり、クライアントによって誤って解決されるか、完全に無視される可能性があります（フレームの関連付け解除によって無効にされます）。



802.11k / vを一緒に使用すると良い結果が得られ、ほとんどの場合、さまざまなデバイスの動作に問題を引き起こすことなく、ホームネットワークと小規模オフィスネットワークで十分です。 次に重砲が登場します-これは主な問題を根本的に解決しますが、副作用を引き起こす可能性があります-これは802.11rです。



802.11r / ft



高速ローミング/高速BSS移行 -クライアントがポイントで使用される場合、802.11rは必須です。 サポートしていない人は接続できません-これは制御フレームのフラグであり、キー交換メカニズムが変更されています。加入者が古く、その存在を知らない場合、問題があります（新しいデバイスでは、機能のサポートがなくても、このフラグの理解を追加することがありますが、標準に従って、プロトコルを完全に実装する必要があります）。 また、古いクライアントアダプターの誤ったドライバーアダプターをダウンさせる可能性があります-共有キーを初めて配布するときに4ウェイハンドシェイクを使用するときは、標準で次のように記述されます：「STAは、FTを使用するとき、FT認証アルゴリズム以外の認証アルゴリズムを使用してはなりませんプロトコル。」



Fast BSS Transitionは、RSNA（Robust Security Network Association-WPA2）ネットワークおよび完全にオープンなネットワークで動作します。 WPA2-PSKの場合、高速ローミングの意味は失われます。 クライアントとポイントはまだ4パケットを交換しますが、ここで高速化するものはありません。 計算では、適切なポイントを検索する時間を考慮していません.5 GHz帯域ではかなりの可能性があります-16チャネルをスキャンして適切なAPを見つける必要があるため、一般的な戦略はk / vおよびrプロトコルを一緒に使用することです。



認証にRADIUSを使用し、非常に高速なローミングが必要な場合は、802.11rのみを選択する必要があります。



802.11rでのローミングに加えて、クライアントに必要なリソースの可用性についてポイントをポーリングし、それらを予約する（QoS）可能性があります。 したがって、プロトコルにはFTプロトコルとFTリソースリクエストプロトコルの2つの亜種があります。 クライアントとポイント間の通信は、直接無線経由（Over-the-Air）と使用済みポイントとコントローラー経由（Over-the-DS）の両方で発生します。2番目の方法は少し長くなります。 クライアント上のポイントからのQoS要求はまだ実装されておらず、実際にどこでも使用されています。
