HostCMSの新しいバージョン、6-それは何ですか？

多くは、HostCMSの優れたコンテンツ管理システムに精通しています。



利点：





  • 使いやすさ -管理部分のインターフェイスは実装されていますが、少し複雑ですが、すべてが原則的に論理的であり、ほとんどの要求はAJAXによって行われます。 したがって、インターフェースとの速度は高速です。
  • 機能 -Freebieの基本バージョンには、非常に多くの機能が含まれています。 情報システムモジュールを使用すると、サイトでほとんどすべての機能を実行できます。 実際、システムの古い（有料）エディションに追加されたモジュールは、単に便利ですぐに使用できる実装です。 ただし、Freebieの編集オフィスですべてを行うことができます。
  • マルチサイト -チップ。このバージョンでは、おそらく無料で利用可能なCMSでは利用できません。




短所：





  • ホスティング要件はメインのZend Optimizerであり、残念ながらバージョンPHP => 5.3では問題があるため、PHP <5.3




物議を醸す：





  • おそらく議論の余地があるのは、XSLシステムでのテンプレートと変換の使用です。 その理由は、入場の閾値が高いことですが、座って整理した後、使用はますます便利で高速になっています
。



まあ、ポイントまで





まもなく、 HostCMS 6の新しいバージョンがリリースされ 、その欠点はなくなります。



バージョンはアップデートのみに利用可能で、ベータ版です。



  • 開発者はZendOptimizerを放棄します。 これで、カーネルとモジュールのソースコードが利用可能になり、変更できます。 それは自然に喜ぶことができます。
  • ORMは開発されました-その助けを借りて、データを操作することがさらに便利になりました。
  • 正直に言うと、プロジェクトのソースを見ると、私はうれしい驚きを覚え、古き良きMVCを見ました。 私の意見では、開発者にとって非常に合理的なソリューションです。
  • タンバリンを使用した小さなダンスの新しいバージョンにアップグレードできます。 （ダンスは長くなく、原則としてすべてが単純に解決されます）
  • 管理パネルでは、新しいセクションが個別のウィンドウで開き、最小化できます。 たとえば、構造から通常の動的ページに絶えず切り替える場合に非常に便利です。




おわりに





  • 少なくともテストサーバーでアップグレードし、自分自身を見て満足していることをみんなに勧めます。
  • 以前にシステムをインストールしたことがない人は、必ずインストールしてください。
  • バージョンの公式リリースを待っています。
  • 次の記事では、新しいメソッドとクラスについて説明します。
  • アップグレード方法は、 hostcms.ru / support / updatesをご覧ください




PS。 この投稿は、HostCMSを使用していて、差し迫ったアップデートを聞いたことがない人、およびまだ自分で適切なツールを選択していない人向けです。
