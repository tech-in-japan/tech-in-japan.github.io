Tiny Core Linux 1.2の発表-最小限の10 Mb Linuxディストリビューション。

ロバート・シングルデッカーを紹介

Tiny Core Linux 1.2の新しいリリース-Linuxミニディストリビューション（サイズは10 MBのみ）、ネットワークアーカイブから追加のアプリケーションをインストールすることで拡張可能





-ウィンドウマネージャーJWM、Tiny X、およびグラフィカルユーザーインターフェイスFLTKを使用します。

-新しいtcz-symlinkerは、ブートとTCZ処理を高速化します。

-高度なudevを使用すると、USBとより高速に対話できます。

-他のデバイスのより良いサポートのために、mnttoolが近代化されました。

-使用されていないファイルを削除しました。

-/ optから欠落しているアプリケーションメニューを追加。

-tclocalのマウント時にsuidが関与します。





ここからダウンロードできます



インストール手順（流な翻訳 ）を望む人のために：



Tiny CoreはLiveCDからダウンロードされ、インターネットにアクセスできるターゲットシステムですでに実行されていると想定されています。

1. appbrowserを使用してcfdiskとGRUBを取得する

wbarアプリアイコンをクリックするか、 アプリブラウザのコンテキストメニューから



[ 接続] > [ TCE]を選択します



利用可能なアプリケーションのリストが左側に表示されます。 リストからcfdisk.tceを選択します。



[ 選択したものをインストール ]ボタンをクリックすると、成功した通知が表示されます。



GRUB-0.97-splash.tceを選択してインストールするプロセスを繰り返します。

2.ルートシェルを開きます



3.マークアップ

次のコマンドは、ドライブとその現在のパーティションを一覧表示します。

fdisk -l

このガイドでは、/ dev / hdaが使用されます。



Cfdiskを使用してハードドライブをパーティション分割します。

cfdisk / dev / hda

キーボードの左右の矢印キーを使用して下部のメニューをナビゲートし、ENTERキーを押してオプションを選択します。

移動して選択（ 新規 ）



（ プライマリ ）を選択します



サイズを選択するか、デフォルト値を受け入れます。

Enterを押して確認します。 デモンストレーションには、100 MBが使用されます。



新しいセクションを配置する場所を選択します



hda1を示すために、新しいセクションBootable を作成します 。



記録の変更



変更を確認（ はい ）

4.フォーマット

新しいパーティションをフォーマットします（ ext3のデモ用 ）

mkfs.ext3 / dev / hda1

5. Tiny Coreシステムファイルをコピーする

新しいセクションをマウントする

マウント/ mnt / hda1

Tiny CoreおよびGRUBファイルのディレクトリを作成します。

mkdir -p / mnt / hda1 / boot / grub

元のブータブルメディアをマウントします。 デモのために、Tiny CoreはIDE CD-ROMからダウンロードされました 。

マウント/ mnt / hdc

ファイルをコピーする

cp -p / mnt / hdc / boot / * / mnt / hda1 / boot /

6. GRUBのインストール

GRUBファイルをコピーする

cp -p / usr / lib / grub / i386-pc / * / mnt / hda1 / boot / grub /

menu.lstの作成：（「LST」には文字「L」が含まれていることに注意してください）

VI / mnt / hda1 / boot / grub / menu.lst

i （入力モード）を押して、次を入力します

デフォルト0

タイムアウト10

タイトルtinycore

カーネル/ブート/ bzImage quiet

initrd /boot/tinycore.gz

必要に応じて、他のオプションを追加できます。

すべてが完了したら（esc）を押し、 （x） （保存して終了）を押します

GRUBの実行：

GRUBは、hdXyとhdXの代わりに、（hdM、n）と（hdM）の形式のHD番号を使用することに注意してください。

Mは、Xの「a」を表す「0」からの数字に対応します。

その後の各文字について、1つずつ追加します。

「B」は「1」、「C」は「2」、「d」は「3」など。

nはyの「1」を表す「0」からの数字に対応します。

yから1を引いてnを取得します。

「1」は「0」、「2」は「1」、「3」は「2」などになります。

デモンストレーションでは、hdXyにhda1を使用します。つまり、（hdM、n）は（hd0、0）

また、hdX（hdM）の原理でhdaを使用すると（hd0）

グラブラインで

ルート（hd0.0）

セットアップ（hd0）

やめる
