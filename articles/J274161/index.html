私たちのブログでは、メーリングリストの作成について、そのレイアウト 、 デザイン 、興味深い統計的事実についてたくさん書いています 。 本日、メールニュースレターや通知を購読するための真に高品質のフォームの作成について、AcquireConvertプロジェクトの創設者であるGiles Thomasによる資料の翻訳をご紹介します。



フォーム



問い合わせフォームは、潜在的な顧客を引き付けるための基盤であり、オンライン販売の成長の鍵でもあります。 フォームは、収益、ビジネスの成長、さらにはその寿命にさえ影響します。



退屈なので、フォームに記入するのを好む人はほとんどいません。 それをどうするか、そしてコンバージョン率を最大化する方法は？



今日は、この質問に答えて、次のトピックを検討します。



  • 最適化できるさまざまな種類の形状。
  • 効果的なユーザーエクスペリエンスを提供するフォームの作成。
  • 顧客開発;
  • 効果的な碑文とテキストの作成。
  • 変換サポート;
  • 高品質の形状を作成する他の方法。


各章では、調査のデータに基づいて特定の例を検討します。 私は自分の意見を表明しないようにし、普遍的な解決策を提供しません。



チュートリアル全体を通して、最適化の対象とその理由に関するヒントを提供します。 すぐに現在のアイデアのテストを開始できますが、もちろんテストケースを使用します。



最適化するタイプ 変換に関しては、主に4つのタイプのフォームに関心があります。



  • リードフォーム（連絡先およびサブスクリプションフォーム）;
  • オンライン販売フォーム（注文フォーム）;
  • ユーザー登録フォーム（登録フォーム）;
  • 運用フォーム（特定のタスクの遂行に向けられたフォーム）。


リードフォームにより、ユーザーは会社の代表者に連絡したり、メーリングリストに参加したりできます。 オンライン販売フォームは、製品の購入に関連しています。



登録フォームを使用して、ユーザーはプロジェクトに参加できます。 操作フォームは、サイトまたはアプリケーションで操作を実行するために必要です。 良い例はインターネットバンキングです。



使用するフォームの種類に関係なく、ユーザーのフローとファンネルデータを分析して、コンバージョン率を高める必要があります。

ほとんどの場合、フォームの外観を最初に想像するときは、フォームに書く言葉については考えません。 しかし、それらは、たとえばデータ入力フィールドの名前において非常に重要です。



ユーザーの行動（フォームに記入するか、サイトを離れる）は、言葉遣いのシンプルさと理解度に依存します。 37Signalsの創設者Jason Fried氏は、次のように述べています。「 インターネット上のテキストのほとんどは、読者ではなくコピーライターのためにコピーライターによって書かれているため、ひどいものです。 読者に書いてください。 それがすべてです。」



適切なコピーライティングはインターフェース開発の重要な部分であり、各単語はアイコンや小さなピクセルと同じ意味を持ちます。



フォームの名前はその目的を反映する必要があります。


まず、ユーザーはタイトル、つまりフォームの名前を読みます。 目的を明確かつ簡潔に説明していることを確認してください。 また、インセンティブの言葉を使用して、ユーザーに行動を促すようにします。

[はい]または[いいえ]ボタンのあるダイアログボックスを使用しないでください。


知っているように、ユーザーはページを一onlyするだけで、すべてを読むわけではないため、Webアプリケーションで必要なタスクを実行する速度は、[アプリケーション]の使用の効果に影響します。 それは、人がクライアントになるかプロジェクトを去るかによって異なり、それがユーザーの流出係数に影響します。



したがって、ユーザーとのやり取りの形式に情報を追加して、詳細に注意する必要があります。 特定のタスク向けのフォームのテキストを作成する際のよくある間違いは、yes / noダイアログの乱用です。



[はい] / [いいえ]または[確認] / [キャンセル]ボタンのあるダイアログボックスでは、多くの場合コンテキストが表示されません。 典型的な例は、Windows 7のファイル削除ダイアログです。



ボタンを見るだけで、ユーザーは有用な情報を抽出しません。







ユーザーがボタン上のテキストのみを読み取るとします。 この場合、ボタンの表記は質問に直接関連している必要があります。







この例では、「ワークシートを保存」ボタンのテキストをフォームのタイトルにリンクして、その有効性を高めています。 これで、ユーザーは自分に何が必要かをよりよく理解し、すぐに決定を下すことができます。



ヒント：最大限の変換を実現するには、ボタンラベルがフォームヘッダーで尋ねられた質問に答えていることを確認してください



プレースホルダーは使いやすさを損ないます


一部のフォームでは、名前が入力フィールドの指示またはプロンプト、つまりプレースホルダーに置き換えられます。 例：







ただし、 IT追跡調査では、空のフィールドがユーザーの注意を引く可能性が高いことが示されています。 人は空でないフィールドの検索により多くの時間を費やし、それらに気付かない場合があり、これはコンバージョンの減少につながります。



さらに、選択したフィールドにカーソルを置くとプレースホルダーが消えるため、ユーザーはフォームの目的を覚えておく必要があります。 入力フィールドのテキストをもう一度見るためには、クライアントはすでに書いたものを削除する必要があることがわかります-これにより、充填時間が遅くなり、ユーザーがイライラしているように感じ、変換が減少します。 大きくて長いフォームに記入するとき、それは本当に不快になります。



ユーザーがTabキーを押して次の入力フィールドに移動した場合、対応するフォーム要素を見るまでに、プレースホルダーはすでに消えています。



ヒント：プレースホルダーを名前として使用せず、入力のヒントまたは例としてのみ使用してください



フィールド名の場所が重要


プレースホルダーだけでフォームの入力が遅くなることはありません。 入力フィールドに関連する名前の場所自体が障害になる場合があります。



調査中、UXmattersオンラインマガジンは、フォームに名前を付けると、フォームの完成速度に影響することを発見しました。



同社は、フィールドヘッダーの場所に関する3つのオプションのIT追跡調査を実施しました。

  • 名前は左揃えで、入力フィールドと同じ行にありました。
  • 名前は右揃えで、入力フィールドと同じ行にありました。
  • 名前は、左側の入力フィールドの上にありました。






最初のケースではフォームに記入するのが最も難しいことがわかりました。要素間の距離のために、ユーザーはある点から別の点に目を移動しなければならなかったからです。







フォーム上の要素の相互配置の2番目のバリアントは、最初のバリアントよりも優れています。







ただし、3番目のオプションが最良のオプションであることが判明しました。このような要素の相互配置により、ユーザーは視線を修正するのが最も簡単です。



ヒント：読み取りプロセスを高速化するには、フィールドの名前をその上に置き、左に揃えます。



入力フィールドの上に名前がある長いフォームは大きくて扱いにくいため、ユーザー間で拒否される可能性があります。 視覚的に形状を縮小するには、入力フィールドの名前をそれらと同じ行に配置します。



フォームの横に入力されたデータの正確性に関するメッセージを表示します

入力したデータを確認するときに、エラーが発生することがよくあります。 エラーメッセージはフォーム上でグループ化される場合があります。







このアプローチでは、ユーザーが誤って入力されたデータを見つけて修正することは困難です。 1つのフィールドを修正した後は、毎回メッセージを読んで次のエラーを見つけるためにスクロールする必要があります。



上記の図は、エラーのあるフィールドの名前が赤で強調表示されていることを示していますが、エラーの原因を示す入力フィールドの横に、誤って入力されたデータに関するメッセージを表示することをお勧めします（下の画像を参照）。







Luke Wroblewski による調査の結果は、入力されたデータをリアルタイムで検証する場合、つまり、ユーザーがフォームに記入した直後にメッセージを表示する場合、変換がさらに大幅に増加することを示しています。







このビデオでは、ユーザーがフィードバックフォームに記入し、入力されたデータの正確性または不正確性に関するメッセージをリアルタイムで受信する方法を示します。



フォーム検証後の検証と比較して 、リアルタイム検証が許可されました：



  • フォームの正常な完了数を22％増やします。
  • 充填中のエラーの数を22％減らします。
  • 満足度を31％向上させます。
  • 充填時間を42％短縮します。
  • 注視翻訳の数を47％削減します。


テストでは、エラーのチェックは、フォームの各要素に入力した後、つまりユーザーが次または前のフィールドに移動した時点で行うのが最適であることが示されています。



さらに、この調査では、ユーザーがエラーメッセージと入力の正確性が時間とともに消えないことを好むことがわかりました。



ヒント：誤って入力されたフィールドの横にエラーメッセージが表示されることを確認してください。 可能であれば、完了したフィールドをリアルタイムで確認し、時間とともに消えない情報メッセージを追加します。

顧客開発


コンバージョン率の最適化に関与する多くの企業は、顧客開発の段階をスキップしています。 しかし、私の経験では、これは作業の中で最も価値があり効果的な部分です。



顧客調査は、顧客を知り、顧客が直面している問題を理解し、製品が市場でどのような位置にあるかを知る最良の方法です。



最初に、調査の顧客または潜在的なユーザーを見つける必要があります-それはそれほど単純ではありません。 または、Launchrock Webサイトを使用して簡単なランディングページを作成し、Facebook広告を起動します。



ソーシャルサービスを使用して、自分の分野で「スピニング」している人々と連絡を取ってください。 ネットワークまたはメールで連絡します。 製品について書いている記者と話してみてください。 また、 設定が簡単なGoogle / Buzzsumoアラートにも注意する価値があります。



別のオプションは、顧客にボーナスを提供することです。 連絡先リストにメッセージを送信して、数分待ちます。



ボーナスに動機付けられた人は調査結果を割り引くことができると考えられていますが、ほとんどの場合、受け取った質の高いレビューの数は利己的な人からのレビューの数よりもはるかに多くなります。



また、お住まいの地域の企業をGoogleで検索して、見つかった番号に電話することもできますが、この方法で悪用しないでください。 同僚の推薦を求める方が良いです。



人々に自分の時間をあなたに捧げるよう説得することは難しいので、最大限にそれを使う必要があります。 インタビューできる友人や連絡先についても質問します。 新しい顧客を見つけることができるかもしれません。
