UbuntuをMacBook pro 11.3（2013）またはhackintoshに配置しました

ラップトップを交換する時が来たので、自分にとって非常に不満なことに、市場に出回っているある果物会社の製品以外に、16x10の画面を備えたラップトップはないことがわかりました。 鉄の新しいファームウェアはそれほど悪くなく、馬の値札にもかかわらず、ヒキガエルはつぶされ、ラップトップが購入されたと言わなければなりません。

この記事は、リンゴの独立と戦う新しい方法ではありません。すべてのレシピはインターネットで収集されます。 私の経験が誰かに役立つことを願っていますが、すぐに予約をします、あなたがすることはすべてこの記事に導かれ、あなたはあなた自身の危険とリスクで行うので...



インターネットの休憩、私はLinuxをポピーにインストールして使用するための3つの基本的な可能性があるという結論に達しました。

  • macサフィックスを持つ特別なディストリビューションを使用します。 EFIブートを無効にするために、BootCampと密かに対話します
  • rEFIt（またはその開発rEFInd）を使用する
  • ネイティブアップルローダー（以降、アップルローダーと呼びます）を使用する


mac IMHOという接尾辞が付いた特別な配布キットを使用するアプローチは、2013ファームウェアには適用されません-インストール後にLinuxを機能させるために、多くのソースがカーネルのさまざまなキーを提供します-nolapicからnosmpまで、これは絶対に良くありません。 消します。

残りの2つのうち、ネイティブローダーに精通しているため、エンティティが少なくなります。実際、rEFIndを使用したロードには生命権があります。 アップルローダーで止めましょう。

最新のファームウェアにはuefiブートローダーが備わっていることに注意してください。しかし、いつものように、それはあまり一般的ではありません-すぐにはすべてをロードしません。 まだいくつかのニュアンスがありますが、今は分解しません。

アップルローダーが心から愛するLinuxをロードするには、後者はosxのふりをするか、Linux自体ではなくefi grubのふりをする必要があります。 これを行うには、次の条件を満たす必要があります。

  • hfsファイルシステムでは、それぞれApple HFSのように分離する必要があります
  • このpartinには、boot.efiファイルとSystemVersion.plistファイルを含むSystem / Library / CoreServicesディレクトリが必要です。これは、特別な形式でパーティションの内容を記述します
  • パーティションのルートには、ファイルmach_kernelが必要です。


これらの条件に応じて、アップルローダーはこのmacosxの偽のコピーをダウンロードすることを検討します。



さあ、行きます。



  1. mirror.yandex.ru/ubuntu-releases/13.10/ubuntu-13.10-desktop-amd64.isoからUnetbootinを使用してインストールフラッシュドライブを準備しています（ Macバージョンではありません!!! ）
  2. osxを起動して、パーティションのサイズを変更します（osxの下に160GBを残しましたが、この方法ではosxを使用せずに実行できますが、私にとっては-そのままにしてください）
  3. 再起動し、起動時にAltを押してダウンロードソースを選択します。 フラッシュドライブには2つのバージョンがあります。efiブートオプションを選択する必要があります。
  4. GRUBメニューで、「ubuntuを試す」を選択します。 USBフラッシュドライブから起動する場合、 libata.force = noncqを追加してカーネルブートパラメーター行を編集するのが理にかなっています。ディスクにアクセスするときに定期的に20〜30秒のシステムフェードが発生するという苦情がインターネット上にあります。
  5. インターネットに接続する-このために、独自のドライバーWiFi システム設定->ソフトウェアと更新->追加ドライバーを設定します 。ここでBroadcom 802.11ドライバーをオンにし、 近隣の WiFiに接続します
  6. gdiskを置く 

    sudo apt-get install gdisk
  7. gdiskを使用して、10メガバイトのパーティションを作成します（ 以降、このパーティションを10MBパーティションと呼びます ）。 osxを使用してパーティションから128メガバイトを退却する必要があります。そうしないと、後者が非常に心配されます。 パルチザン配置パラメーターとして+ 128Mおよび+ 10Mを使用すると便利です。 AF00パーティションのタイプ（Apple HFS / HFS +）を設定することが不可欠です。そうしないと、 Appleローダーはパーティションをロード可能として認識しません。
  8. この時点で、osxを再起動してドライブを確認するのが理にかなっています。 また、Mac OS Extended（ジャーナリングバージョンではないため、Linuxで問題が発生します）でフォーマットし、Ubuntuに名前を変更することもできますが、この手順では、osxを完全に破壊したいので、Linuxですべてを実行します。
  9. フラッシュドライブからaltを再起動し、ubuntuを試してください（libata.force = noncqを追加することを忘れないでください）
  10. インストーラーを実行します（Ubuntu 13.10をインストールします）
  11. インストール用にパーティションをマークアップします-私はルート20Gと他のすべてを作成しました-ホーム。 スワップはしませんでした-いつでもファイルを作成できます。
  12. 次に、ubuntuに1つのコメントを付けます。インストールの最後にリブートしません。 （ コンピューター名には、DMIを正しく読み込めなかったことを示す説明文が含まれていることに注意してください。これは修正されると改善されます。そうでない場合、端末の画面の半分が招待状になります ）
  13. インストールが完了した後、再起動せずに（再起動する場合は問題ありませんが、Linuxはosxのふりをしていないため、再度フラッシュドライブから起動する必要があります）、新しく受け取ったインストールをマウントします 

     mkdir /tmp/u sudo mount /dev/sda4 /tmp/u sudo cp /etc/resolv.conf /tmp/u/etc/resolv.conf sudo mount --bind /dev /tmp/u/dev sudo mount --bind /proc /tmp/u/proc sudo mount --bind /sys /tmp/u/sys


    • resolv.confをコピーしました-これは、chroot環境で名前解決が正常に機能するために必要です。 インストールされたubuntuの開始時に、このファイルはNetwork Managerによって再作成されます。
    • まだ/ dev / ptsをマウントできます


  14. 実際、chrootに移動します。 
     sudo chroot /tmp/u/
  15. hfsおよびgdiskサポートをインストールします 
     sudo apt-get install hfsprogs icnsutils gdisk
    （chrootからこれを行っているため、これらのパッケージは新しいインストールに配信されます）
  16. 作成された10MBパーティションをフォーマットします（私の場合/ dev / sda3） 
     sudo mkfs.hfsplus /dev/sda3 -v Ubuntu
  17. blkidを使用してパーティションのUUID 10MBを見つけます
  18. edit / etc / fstab-mount / boot / efiを前のステップで定義されたUUIDに変更し、ファイルシステムをvfatからautoに変更します
  19. 再マウント/ブート/ efi 
     umount /boot/efi mount boot/efi
  20. ファイル/ usr / sbin / grub-installを修正し、ファイルシステムタイプの検証についてコメントします。 

      if test -n "$efidir"; then efi_fs=`"$grub_probe" --target=fs "--device-map=${device_map}" "${e$ # if test "x$efi_fs" = xfat; then :; else # gettext_printf "%s doesn't look like an EFI partition.\n" "${ef$ # efidir= # fi fi


  21. grub-installを実行します
  22. 偽のosxインストールを作成します。 

    •  touch /boot/efi/mach_kernel

    • 次の内容で/boot/efi/System/Library/CoreServices/SystemVersion.plistファイルを作成します。 

       <?xml version="1.0" encoding="UTF-8"?> <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList- 1.0.dtd"> <plist version="1.0"> <dict> <key>ProductBuildVersion</key> <string></string> <key>ProductName</key> <string>Ubuntu</string> <key>ProductVersion</key> <string>13.10</string> </dict> </plist>


    • grubx64.efiへのリンクを作成します 
       ln /boot/efi/EFI/ubuntu/grubx64.efi /boot/efi/System/Library/CoreServices/boot.efi
  23. / etc / default / grubを編集し、次の行をキャストします：

    GRUB_CMDLINE_LINUX_DEFAULT = "libata.force = noncq"
  24. 演じる 
     update-grub
  25. 過負荷。


この段階で、すでにaltを使用して起動できる実行可能なシステムがあり、osx（ブートディスク）をメインとして選択できます。



いくつかのメモ：



  • 再起動後、UUID 10MBパーティションが変更されました-fstabを再度修正する必要がありました
  • アップルローダーの3番目の「Windows」というアイテムの出現と消滅の理由をまだ特定できていませんが、一般的には干渉しません。
  • たとえば、 ここからカーネル3.12を置くのは理にかなっています 。
  • NVidiaのクローズドドライバーは、バックライトコントロールが故障しても、些細なことではありません。 grub acpi_backlight = vendorを追加することにより、状況はわずかに改善されます。
  • Ubuntuをインストールした後、アップルローダーは時々思慮深い-NVRAMリセットで処理されます-起動時に、alt + cmd + P + Rを同時に押し続けて2番目のpa-paを待ちます！
  • 音はありませんが、このパッチで簡単に処理できます（このパッチはカーネル3.11のネイティブモジュールにも適しています）
  • キーF1..F12の動作を通常のものに再割り当てできます。 fnなし 

     echo options hid_apple fnmode=2 | sudo tee -a /etc/modprobe.d/hid_apple.conf sudo update-initramfs -u -k all
