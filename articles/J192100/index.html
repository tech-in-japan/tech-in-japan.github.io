Navigation Controllerでモーダルセグエを使用する方法



そのため、問題があります。クライアントは標準のNavigation Controllerトランジション（「右へのオーバーラップ」）を好まないため、たとえば「画面を下から回す」ことによって新しい画面を表示したいと考えています。 すぐに問題の解決策がわかります。セグエの表示をモーダルに変更します。 すべては問題ありませんが、画面の階層を持つNavigation Controllerのロジックは壊れています。 その結果、アプリケーションがクラッシュします。 モーダルセグエが必要になりましたが、プッシュセグエ機能が必要です。 この問題を解決する簡単な方法があると思いますが、UIStoryboardSegueのサブクラスを記述することをお勧めします。 変更する必要があるのは、クラスの.mファイルだけです。 より具体的には、 performメソッド：



押して！ 
- (void) perform{ //  ,     UIViewController *src = (UIViewController *) self.sourceViewController; UIViewController *dst = (UIViewController *) self.destinationViewController; //    [UIView transitionFromView:src.view toView:dst.view duration:1 options:UIViewAnimationOptionTransitionFlipFromBottom completion:nil]; //    Navigation Controller'a [UIView transitionFromView:src.navigationItem.titleView toView:dst.navigationItem.titleView duration:1 options:UIViewAnimationOptionTransitionFlipFromBottom completion:nil]; //  Push  Segue [src.navigationController pushViewController:dst animated:NO]; }


UIViewAnimationOptionTransitionFlipFromBottomの代わりに、クライアントの中心に近いトランジションスタイルを配置できます。

以上です！ なんて簡単なのでしょう。 これで、ストーリーボードセグエのカスタムスタイルを指定し、新しいクラスを指定して、遷移タイプでNavigation Controllerセグエを取得できます。



コアグラフィックス



ある時点で、クライアントはボタンの幅を5ピクセルずつ変更する必要があるたびにデザイナーを蹴るのにうんざりしています（結局のところ、ボタンごとにデザイナーが別々の絵を描いた）。 QuartzCore.frameworkを使用して、丸みを帯びた縁と境界線を持つ独自のボタンを作成します。 繰り返しますが、シングルトンの場合と同様に、プロジェクトの作業を高速化するスニペットのようなものです。



実際、このアプローチはUIViewのサブクラスで使用できます（UIButtonはまさにそれです）。 UIViewのawakeFromNibメソッドを書き換えます。 



 - (void)awakeFromNib { [super awakeFromNib]; self.layer.cornerRadius = 5.0f; self.layer.masksToBounds = YES; self.layer.borderColor = [UIColor whiteColor].CGColor; self.layer.borderWidth = 1.0f; }


すべてがシンプルです。 まず、継承では、スーパークラスが作業を完了した後にコードを実行する必要があります。 そのため、同じメソッドをスーパークラスで呼び出します。 次に、この種のレイヤーの角の曲率半径を設定し、レイヤーが指定されたマスクに従うようにします。 第三に、フレームの色（もちろんCGColor）とその厚さを設定します。

グラフィックコンテキストで深い作業を期待している人もいることは知っていますが、だからこそフリーランサーにとっては高速チートシートです。CoreGraphicsの新しいタスクが登場したら、記事を書きます。



UIWebViewおよびScrollViewを使用する



すでにUIWebViewがあります。コンテンツをロードしてみましょう。 



 NSString *htmlString; NSString *cssString; <... ...> htmlString = [NSString stringWithFormat:@"<style>%@</style>%@", cssString, htmlString]; NSURL *url = [[NSURL alloc] initFileURLWithPath:pathToApplicationDirectory]; [self hideOverscrollShadowsForWebView:webView]; [webView loadHTMLString:htmlString baseURL:url];


私たちのスタイルとhtmlのWebページを取得しました。 UIWebViewのオーバークロック中に不快な灰色の影を隠すために、次のメソッドを使用しました。 



 /*! ,         UIWebView \param webView   WebView,    */ - (void)hideOverscrollShadowsForWebView:(UIWebView *)webView { id scrollview = [webView.subviews objectAtIndex:0]; for (UIView *subview in [scrollview subviews]) if ([subview isKindOfClass:[UIImageView class]]) subview.hidden = YES; webView.backgroundColor = [UIColor clearColor]; }


UIWebViewに属するScrollviewからすべての子ビューを非表示にします。 また、透明な背景を設定します。



UIScrollViewについて言うべきことは1つだけです。contentSizeプロパティを設定することを忘れないでください。



カスタムポップアップ



この記事の巨人について話す時が来ました。 今回、クライアントは、参加せずにポップアップの外観をすばやく変更する機能を必要としていました。 まあ、仕事のために！



理論、私たち全員がどのように物事を成し遂げるかについて少し話しましょう。

ストーリーボードでは、本格的なポップアップウィンドウを実装することはできませんが、別のPopupView.xibファイルを作成できます。

モデルとして、3つのクラスPopupView、PopupController、PopupControllerDelegateを作成します。



最初に、スクリーンショットのようにポップアップウィンドウのレイアウトを組み立てます。 この場合、ファイルの所有者はPopupControllerクラスのオブジェクトになり、ビュー自体はPopupViewクラスのオブジェクトになります。 背景ビューは、半透明の明るい灰色のUIViewです。







最初にPopupView.hの実装を見てみましょう。



押して！ 
 #import <UIKit/UIKit.h> @interface PopupView : UIView @property (strong, nonatomic) IBOutlet UIView *backgroundView; @property (strong, nonatomic) IBOutlet UIView *innerPopupView; @property (strong, nonatomic) IBOutlet UILabel *popupTitleLabel; @property (strong, nonatomic) IBOutlet UILabel *popupTextLabel; @property (strong, nonatomic) IBOutlet UIButton *popupButton; @end


すべてのユーザーインターフェイス要素をコードにフックしました。 PopupView.mは変更せず、標準の生成コードを残しました。

ポップアップウィンドウテンプレートがありますが、今では巧みに使用する必要があります。 PopupControllerを作成します。



PopupController.h ：

必要なプロパティと、初期化のためのパブリックメソッドを設定し、簡単なポップアップウィンドウを表示し、すべてのポップアップウィンドウを閉じるだけでした。 また、PopupControllerのポップアップ内のボタンに小さなアクションを追加しました。

初期化中に、すぐにクラスのデリゲートを設定します。

読みやすくするために、 showHelloWorldPopupメソッドのコードを他の3つのメソッドに分割しました。ポップアップウィンドウを初期化し、構成し、表示します。

ユーザーがポップアップウィンドウのボタンに触れると、これについてデリゲートに通知します。

dismissAllPopupsメソッドは、共有配列内のすべてのポップアップを単純に走査して、それぞれを閉じます。

  3. 1つの状態のボタンテキストを設定するだけで、他の状態のテキストは変更されないため、 setTitle：forButton：メソッドはすべての状態のボタンテキストを一度に変更します


同様のポップアップアーキテクチャを作成することにより、クライアントは参加せずにウィンドウの外観を変更できます。 クライアントが要素の色、サイズ、配置を変更できるようにします。コードとの接続に触れない限り、すべてが正常に機能します。
