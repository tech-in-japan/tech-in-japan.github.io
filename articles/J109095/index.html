Skalaxiの更新：ディスクのサイズ変更、新しいOSおよびその他の改善

2週間ごと、場合によってはより頻繁に、クラウド管理システムの更新をリリースします。 通常、2週間で、どの機能を起動または改善する必要があるかを判断し、実装し、テストし、最終的に展開します。



少し前に、サーバーを管理するためのajaxコンソールが登場し、オペレーティングシステムを再インストールする機能が登場しました。 しかし、今日の更新を強調し、どのような変更が含まれているのかについて個別に説明したいと思います。 興味のある人とクライアントの両方から、何回もそれらを実現したいという要望を聞きました。



UPD ：私たちのサイトに登録されている人は、ユーザーサポートサイトの特別なセクションで、 ここに自分のアイデアや機能のリクエストを書くことができます。 そこで、他のユーザーのアイデアに投票できます。 本当にこれまたはその機能が必要な場合、例えばfreebsd-アイデアを書いて投票してください:)プロジェクトへの参加とサポートに感謝します。



そのため、更新には以下が含まれます。

  • ディスクのサイズを変更する機能。 これで、パネルを開いて、サーバーを再起動せずにディスクサイズを増減できます。 これらの操作はAPIでも利用できます。
  • 利用可能な新しいOS：OpenSuSE 11.3。 これで、Debian 5.0.6、Ubuntu 10.04、CentOS 5.5、およびOpenSuSE 11.3を使用できます。


ディスクのサイズ変更には多くの制限があります。

  • ルートパーティションのサイズは、電源がオフのサーバーでのみ変更できます。
  • ディスクのサイズを変更する前に、対応するパーティションをアンマウントする必要があります。
  • ディスクサイズを変更した後、 resize2fs /dev/%partition%



    コマンドを使用して自分でファイルシステムのサイズを変更する必要があります。


すでに当社のサービスを使用しているすべての人に感謝し、それらを単にテストしたい人にようこそ。 サイトに登録した後、しばらくの間無料でサービスをテストできます。



そして、アップデートをフォローし、ツイッターをフォローしている人には、特別な驚きがあります:)



ところで、不足しているオペレーティングシステムは何ですか？ Windows Server？ FreeBSD？ フィードバックをお待ちしております！
