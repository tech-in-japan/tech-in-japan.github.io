はじめに



空気と気象観測所の品質を制御するには、多くのワイヤーとさまざまなセンサーが必要ですが、すべての測定値を良い小さなボードに結合するセンサーが登場してから状況は容易になりました。







追加のコンポーネントなしでRaspberry Piに直接接続するオールインワンセンサーを使用して、温度、湿度、気圧、および空気質を監視することを検討してください。 電子工学の経験は必要ありませんが、はんだごては大歓迎です（ただし、オプションです）。 このプロジェクトは、現在の測定値を表示し、すぐに傾向を確認するためのグラフィカルインターフェイスとともに、履歴測定値を保存するためのデータベースを実装しています。

ソフトウェア



GitHubでbalena-senseプロジェクトを開きました。このプロジェクトには、稼働中のシステムに必要なすべてのソフトウェア、構成、およびコードが含まれています。 このプロジェクトを無料のbalenaCloudアカウントに展開して、Raspberry Piのすべてのソフトウェアを配布し、リモートアクセスを提供します。 したがって、次のものが必要です。



  • balenaEtcherのようなSDフラッシュカードライターツール

  • 無料のbalenaCloudアカウント

  • GitHubからプロジェクトを複製またはダウンロードします


機器の組み立て



このプロジェクトのハードウェアに関する作業はほとんどありません。 目標は、センサーボードをRaspberry PiのGPIOピンに接続することです。



BME680は、I2C（eye-squared-see）と呼ばれるバスを介してRaspberry Piと通信します。これは、2本のワイヤを備えたシリアルバスです。 これらの2本のワイヤは、同期ライン（SCK）およびデータライン（SDA）と呼ばれます。 2本のワイヤに加えて、センサー（3.3Vまたは3V3）に電源を供給し、接地する必要があります。



PimoroniまたはSparkfunからはんだ付けせずに、またはSense HATを使用せずにオプションを選択した場合、光沢のある新しいコンポーネントをRaspberry Piに接続するだけで、すべての接続が自動的に行われます。次のステップに進みます。



Pimoroniまたは別のライザーを使用してセンサーをRaspberry Piに直接接続する場合、主なことは、上記の接点（SDA、SCK、3V3、およびGND）を正しく接続することです。











Raspberry PiでGPIOを操作するための便利なサイトはpinout.xyzです。 ピン1（3V3電源）、3（SDA）、5（SCL）および9（グランド）が必要であることを明確に示しています。 Pi 2以降では、40ピンGPIOが標準です。 Pimoroniボードを使用している場合、コンタクトはボードを接続する正しい順序になっています。 ボード上の表記が上記のピン配置図と一致することに注意してください。







しかし、この方法でセンサーを接続すると、測定値はRaspberry Piプロセッサーからの熱の影響を受けます。 より正確な測定値を収集するには、ボードからセンサーを取り外す必要があります。以下に示すように、 DuPontのマザーツーマザー（またはジャンパージャーキー ） ケーブルで行います。







同様に、ピン配置図が正確に一致しない別のセンサーを使用する場合、同じ方法を使用してこれらのボードをPiに接続できます。 主なものは、3V3、SDA、SCL、およびGNDへの準拠を保証することです。 ボードにSDI、SDO、CSの接点がある場合は、このプロジェクトでは使用しない代替SPIインターフェースに使用されるため、無視できます。



上記の写真では、Raspberry Pi 3A +を使用していますが、Pi Zeroも安価なソリューションとして適しています。

ワークステーションにbalena CLIツールをインストールする



BalenaCloudを使用すると、アプリケーションソフトウェア/コードをRaspberry Piおよびその他のデバイスにリモートで転送できます。 これを可能にするには、ワークステーション（Raspberry Piではなく）がいくつかの最小ソフトウェア要件を満たしている必要があります。 注：このプロジェクトを正常にデプロイするために、SSHまたはbalenaCloudダッシュボードからデバイス自体でコマンドを実行する必要はありません。 すべてがリモートで行われます！



コンピューターにnpmが既にある（または構成できる）場合、これはCLIツールを実行する最も速い方法です。 Windows、macOS、およびLinux用のバイナリもあります。 注：他のプロジェクト用にこれらのツールを既にインストールしている場合、この部分をスキップして次のステップに直接進むことができます。



CLIツールのドキュメントでは、npmパッケージのインストールと構成、およびバイナリについて説明しています。

どのように機能しますか？



このプロジェクトには、3つの主要コンポーネント、3つのサービスがあります。 これらの名前は、balenaCloudダッシュボードにあります。



  • InfluxDB-センサーの測定値を保存するために使用されるデータベース

  • センサー-センサーにアクセスし、読み取り値を取得してデータベースに保存するためのPythonライブラリとコード

  • Grafana-グラフとレポートを含むダッシュボードの作成に使用


ボッシュのBME680センサーは、温度、湿度、圧力、およびガス抵抗を検出します。 温度、湿度、圧力センサーはすぐに使用できますが、ガス抵抗の測定値だけではあまり役に立ちません。 これらは、空気組成の変化の指標です。 センサーが起動すると、基準が設定され、変更が監視されます。 空気の質を示す指標はパーセントで示され、湿度とガス抵抗の影響を考慮に入れています。



ガス抵抗センサーがないSense HATを使用する場合、目標の温度と湿度の値に基づいて空気品質の測定値を概算します。

カスタマイズ



ダッシュボードとレポート用のGrafanaプログラムには、多くの構成パラメーターと設定があります。 4つの各測定オプションのチャートとグラフを表示する基本パネルを作成しました。







タイトルの上にマウスを移動して編集に進むことで、各Grafanaパネルをカスタマイズできます。 設定をいじってみることをお勧めします-ダッシュボードの変更は自動的に保存されないため、間違えた場合はページをリロードするだけですべてが元のフォームに戻ります。 変更を保存する場合は、コントロールパネルの上部にある[ 保存 ]をクリックすると、すべての設定がデバイスに保存されます。







センサーを構成することもできます。 たとえば、ゲージ図では、数値の代わりにいくつかの基本的な範囲とテキスト値を設定します。 このすべてがあなたの好みに変わります！
