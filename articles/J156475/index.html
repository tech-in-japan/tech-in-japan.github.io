Ivideon：ビデオ監視および放送用アプリケーションの更新





Habré に関する記事の 1つで、私たちはあなたの個人アカウントの更新をすぐにリリースすることを約束しました。また、サイトでビデオ監視と放送を整理するための無料のIvideon Serverアプリケーションもリリースします。 同時に、新しいバージョンでは大幅な変更が行われ、プロのシステムにレベルが近づくことに注目しました。 それで、私はこれが起こったこととIvideon Server 3.0がダウンロードできることを発表できてうれしいです。 これまでのところ、WindowsおよびMac OS Xのバージョン用の更新プログラムがリリースされています。近い将来、 Linuxバージョンをレイアウトします。 カットの下で興味がある人のために、変更の歴史といくつかの写真。



Windowsサービスとしてインストールする機能。







おそらく最も頻繁に要求されるオプションの1つです。 ユーザーがログインする前にIvideonサーバーが起動して動作するように。 これはIvideon Server for Windowsの最大の欠陥の1つであると言われるべきです（LinuxのバージョンはすでにMac OS Xのバージョンのようにデーモンのように機能します）。 今、私たちはそれを排除しました。 インストール中に、「サービスとしてインストール」ボックスをチェックします。 Windowsにサービスとしてアプリケーションをインストールするには、多くの権利が必要であることを覚えておいてください。



モーション検知エリアから除外ゾーンを設定します。







明らかに、ビデオ監視システムとしてIvideonを使用する場合、目的のイベントを検索するためにアーカイブされたレコードの何時間も見たくないでしょう。 そのため、フレーム内に動きがあるアーカイブ内のそのビデオのみを記録するソフトウェア検出器を提供しました。 しかし、非常に多くの場合、フレームには、検出器を起動させない「移動」要素が多数あります。 時計、点滅するライト、樹冠、窓の外の車。 Ivideon Serverディテクターを設定するときにフレーム内でそれらを選択することで、それらを検出エリアから簡単に除外できます。



スケジュールされたレコードの実装。







曜日に各カメラの録画モードを柔軟にプログラムできます。 たとえば、カメラの1つで夜間にのみモーション検出器でビデオを録画し、日中は連続録画をオンにします。 同時に、週末に録音をオフにすることができます。



ライン入力マイクをIPカメラからのビデオに接続する機能。







Ivideonがラジオ局やミュージシャンを使用して、サイトでビデオを放送し始めたことは本当に気に入っています。 彼らの何人かは、IPカメラからのビデオとコンピューターのライン入力の音声を結合する機能を追加するように私たちに求めました。 スタジオからビデオをブロードキャストし、オンエアするものをサウンドとして接続するため。 実際にIvideon Server 3.0で可能になりました。



画像を個別のJPEGフレームとして提供するJPEGカメラおよびデバイスのサポート。







D-Link、Trendnet、およびその他のメーカーの最初のモデルは、RTSPまたはMJPEGをサポートしていませんでした。 また、特定のHTTPリクエストに対して1つのJPEGフレームしか提供できませんでした。 Ivideon Serverの新しいバージョンでは、このようなカメラのサポートを追加しました。 それらのいくつかは既にリストにあり、カメラを追加するときに利用できます。 デバイスがリストにない場合は、自分で追加できます。 これを行うには、「別のメーカー」を選択し、リンクを手動で設定します。 JPEGカメラの場合、jpeg：//プレフィックスで始まる必要があることに注意してください。 mjpeg：//のように、モーションjpegカメラには以前にプレフィックスが提供されていました。



AMRおよびAACコーデックでエンコードされたIPカメラからのサウンドサポート。







AMR形式では、D-LinkおよびVivotekカメラの新しいモデルが音声を送信します。 また、AAC形式は、Axisなどのトップエンドカメラによってほとんどの部分がサポートされています。 IPカメラで使用される他のコーデックとは異なり、8〜11 KHzよりも高いサンプリングレートでサウンドをエンコードできるという点で興味深いです。 その結果、本当に優れた音声認識と音声認識が得られます。 ほとんどの場合、これらのカメラは、興行収入、カジノ、その他の出来事のサウンドトラックが非常に重要な場所を監視するためにインストールされます。



IPカメラからのビデオをプレビューします。







残念ながら、以前のバージョンでは、IPカメラからのビデオをIvideonサーバーで直接見ることはできませんでした。 それに対する必要性は大きかった。 もちろん、接続後すぐにカメラが動作することを確認する必要があります。 Ivideon Server 3.0では、カメラツリーでIPカメラをダブルクリックすると、プレビューウィンドウが開きます。



検出結果の表示によるモーション検出器の視覚的調整。







おそらく、私たちの主な革新の1つでもあります。 以前は、検出器の感度はスライダーを移動することによってのみ設定され、その影響が明確ではありませんでしたが、プレビューが設定ウィンドウに表示され、検出器の設定に応じてカメラの視野の動きがどのように検出されるかをリアルタイムで確認できます



音声がしきい値を超えたときに録音を開始するためのオーディオ検出器が追加されました。







これまでのところ、これは最初のバージョンにすぎません。 オーディオストリームでの適応ノイズ低減と音声検索の計画があります。 それまでの間、しきい値を設定すると、フレーム内に人はいないかもしれないが、声は聞こえるビデオが補充されます。



RTSPモジュールの更新と最適化。







それは普通の形式のように聞こえますが、実際にはそうではありません。 RTSPモジュールを大幅に再設計しました。 なぜなら、標準的なものがあるという事実にもかかわらず、すべてのカメラメーカーがそれに従うわけではありません。 これは特に、低コストで非常に魅力的な中国のカメラに当てはまります。 たとえば、「Content-Length5340」のようなContent-Lengthパラメーターを渡すときに報告した人に出くわしました。 しかし、コロンとスペースだけがなければいいと思います。 値自体は、実際には5340ではなく534に対応します。



一部のD-LinkカメラのWAV形式のサウンドサポート。



多くの場合、人々はカメラの特性を見るのではなく、カメラのコストを考慮します。 そのため、多くのD-Link MJPEGカメラが当社のサービスに接続されています。これは、 ストアにある同じMicrodigitalカメラやSarmattカメラとは異なり、最高の写真とはほど遠いものですが、安価です。 しかし、MJPEG規格は音声伝送を提供していません。 MJPEGカメラの各メーカーは、独自の方法で実装しています。 D-Linkの場合、カメラでWAV形式の別のストリームを使用できます。 以前はサポートしていませんでしたが、現在では多くのカメラモデルで使用できます。



新しいサポート対象デバイスの追加。



デバイスのリストには、少なくとも20台のカメラとDVRが補充されています。 カメラでデータを送信してくれたユーザーに感謝します。



バグ修正。







もちろん、これなしでは、もちろんどこにもありません。 各バージョンでのIvideon Serverの安定性が向上し、ユーザーがそれを見ることができることを本当に願っています。 それでも、新しいエラーが発生する可能性があることを理解しています。 あなたが彼らについて教えてくれて、それを修正するのを手伝ってくれたら、とても感謝しています！ 最後まで読んでくれてありがとう。
