コンピューターウォーターゲート

2008年2月下旬のWindows Vista Capableプログラムの試用期間中、裁判所は158ページのMicrosoft内部ドキュメントをリリースしました。 これらのドキュメントの30ページには、John KalkmanというMicrosoft従業員が書いた電子メールのテキストがありました。 Intel Capable Chipsetプログラムの認定情報が含まれていました。 「インテルが四半期ごとの収益を上げるのを助けるために、特に要件を過小評価しました。 したがって、915チップセットを搭載したマザーボードを販売し続けることができました」とジョン・カルクマンは書きました。 しかし、後に、否定的な結果をもたらしたいくつかの市場調査の後、彼は「真のグラフィック要件を人為的に過小評価するのは間違いだった」と指摘しています。



うわー！ 2つの巨大企業が、何百万人ものユーザーを欺き、ハードウェアとソフトウェアの互換性に関する誤ったデータを共同で提供していたことが判明しました。 Windows Vistaに関するレビューが特にお世辞ではなかったことは驚くことではありません：Windows Vista Capableステッカーを搭載したすべてのマシンがオペレーティングシステムで快適に動作しなかっただけでなく、ひどいブレーキが観察されただけでなく、OSのトップバージョンからの多くのプレミアム機能もありました利用できません。 両社からのカルクマンの手紙に関する公式の説明は現在曖昧であり、以下に記載されています。



1）Chuck MulloyはIntelの代表者です。



Chuck Malloy氏は、Intelの従業員ではないJohn Kalkmanが、チップセット、マザーボード、またはその他の製品の社内財務状況や販売予測について知らなかったと述べました。 したがって、彼の声明は個人的な憶測と見なされなければなりません。



ただし、この答えは、カルクマンの手紙に記載されている内容に対する反論ではありません。 そして、これについて発言するために、チャック・マロイは、この種の情報は開示のためではないことを改めて指摘します。 これも反論ではなく、想像力はすでにインテルとマイクロソフトがすべての法律を迂回して密かに同意することができる何千もの選択肢を引き出しています。



2）David Bowermasterはマイクロソフトの代表です。



Microsoftの担当者は 、Windows Vista Capableプログラムでの915チップセットのサポートは、そのようなチップセットを搭載したコンピューターでのWindows Vistaのベータ版のテストの成功に基づいており、市場でのこれらのチップセットの広範な可用性も考慮に入れていることを示しました。 Windows Vista Home Basicは、これらのコンピューターにインストールできます。 ユーザーがより洗練されたバージョンのプレミアム機能を使用できるようにするコンピューターについては、Microsoftは「Premium Ready」サインの使用を許可しています。



Bowermasterは、Kalkmanの書簡の内容の重要性を意図的に軽視している一方、915チップセットおよびWindows Vistaでの使用に関する認定に関するIntelとMicrosoftの間の合意の可能性については触れていません。 マイクロソフトの広報担当者は、原告が公表した文書を「請求自体とは無関係」と呼び、これはWindows Vista Capableを最適に展開する方法に関する活発な議論の一部にすぎないと説明します。



残念なことに、大規模な会話と長い議論の抜粋のみが公開されており、何が起こっているのかを完全に把握することはできません。 これらの書簡は、「HPの意思決定と起こりうる結果」、つまり一部の契約の結果もOEMに関係しているというテーマに触れています。 当然のことながら、IntelとMicrosoftは、たとえそれが実際に起こったとしても、彼らの罪悪感を認めずに、回避的に応答しました。 しかし、どの企業も陰謀の事実を直接公式に否定していないため、ポイントとストーリーの代わりに省略記号が付けられていることに注意する必要があります。



一方、事実の不連続な連鎖があるため、織り交ぜられ、重なり合っているストーリーのいくつかのバージョンを想定できます。これは、この対立の各当事者の真実と見なされます。 「インテルが四半期ごとの利益率を改善するために必要な要件を過小評価してきました」。これは単純すぎて真実ではありません。



原告の話

原告にとっては、話は簡単だろう。 Windows Vista Readyは、認定されたコンピューターがすべてのバージョンのVistaをサポートするため、理にかなっています。 Windows Vista対応-Home Basicの1つのバージョンのみが正常に動作するコンピューターの認証は理解できないため、意味がありません。



この場合の主な証拠は、カルクマンの手紙です：マイクロソフトは、Windows Vista互換性プログラムでIntel 915チップセットの存在を正当化するCapableプログラムを導入しましたが、Home Basicバージョンのみで動作し、他のバージョンでは動作しません。 公開された文書からの他のレポートに基づいて、検察は他のマイクロソフトパートナーがこの詐欺に反対していることを示したでしょう。

検察は、このような状況のMicrosoftとIntelが、忙しくしている互いを明確に保護することに関心があることをさらに明確にするだろう。



物語は良いです。 彼女はju審員を納得させるほど良かった。 しかし、それはこのような複雑な話には簡単すぎるでしょう。



インテルの歴史

この話は、Intelと共謀するという罪悪感を取り除きます。 この話では、IntelはWindows Vistaをサポートする製品を急いでリリースしないことを決定し、Microsoftを困難な立場に追い込みました。



Intelは、Windows Vistaをサポートするかどうかを選択します。 Intelは、チップセットでWindows Vistaをサポートするために必要な変更の価格が不当に高いと判断しています。 そして、Microsoftにとっての物語は次の意味を持っています。Intelがゲームを去るなら、それで終わりです。



その結果、Microsoftは、最も人気のあるチップセットの1つに適応するために、要件を過小評価しています。 たとえば、Windows製品管理担当副社長のマイクナッシュは、915チップセットで2100ドルのラップトップを購入し、すべての機能を使用できないことに驚きました。 Windows Vista そして、マイクナッシュも含めたこのような巨大なユーザーセグメントを失わないために、Windows Vista Capableプログラムの915チップセットを認定する必要がありました。



マイクロソフトの歴史

この話には最初の問題があります。公開された通信文のどこにも、915のチップセット認証の要件が過小評価されているというカルクマンの判断に反論する会社の幹部はいません。 しかし、Kalkmanからのスキャンダラスな手紙の1年前に送信された別のメッセージで、Mike Ybarraは「Intelの下で曲がっています！」



158ページのドキュメントから、3番目のストーリーに追加される一連のイベントを復元できます。 2005年11月9日に、マイクロソフトは1つの認定プログラムのみを計画しました。 David Berrettは、「PC Ready」はプログラムの正式名称であり、Readyの代わりに使用される用語はCapableであると説明しました。 英語の「可能」の意味はより明確であり、プログラムの意味に近いためです。



しかし、2006年1月に、何かが大きく変わりました。 Microsoft幹部は、Windows CapableプログラムからのWindows Display Drive Model（WDDM）標準の削除と、2006年6月の2番目のVista Readyプログラムの立ち上げについて活発に議論しました。さらに、Intelは915チップセットを支持して、Microsoftに圧力をかけ、WDDM標準の削除を要求しました。



その直後、Mike Ibarraは18か月の作業が無駄になったと書面で書き、MicrosoftはWDDM規格の実装とプロモーションにすでに多大な投資を行っていたHPの代替を余儀なくされました。 他のOEMメーカーも、何が起こっているのか不満です。 MicrosoftはIntelに屈することを余儀なくされています。

HPはWDDM標準グラフィックアクセラレータのサポートに投資しましたが、Microsoftはプログラムを変更しました。 当然、HPはこの事実に満足していませんでした。



上記とは一致しないストーリーの別のバージョンがありますが、それは生命に対する権利も持っています。 情報は、2006年初頭にマイクロソフトの担当者からアナリストの1人が受け取りました。 その本質は、Windows Readyがメインプログラムであったことですが、同社は当初、Windows対応の2番目のプログラムも計画していました。 また、Microsoftの広報担当者は、Windows Capableは、Windows Vistaを使用するのに適した低コストのコンピューターを販売できる証明書を望んでいたOEMの要求で作成されたプログラムであると説明しました。



まとめ

しかし、裁判所によって公表された文書は、IntelがMicrosoftに何らかの行動をとらせることをIntelに責めるほど完全ではありませんが、そのような選択肢は事実の連鎖に見ることができます。



興味深い統計プレゼンテーションがMicrosoftによって提示されました。これに対して、すべてのストーリーと事実を表示できます。 2007年の初めに、2006年の第4四半期のグラフィックアクセラレータ市場で次の図が得られました。



ラップトップおよびデスクトップコンピューターのグラフィックアクセラレータの60％が WDDM標準をサポートしていました。

30％には、Intelの915チップセットがありました。

ラップトップの74％とデスクトップコンピューターの58％に埋め込みビデオがありました。

ラップトップの86％は理論的にはWindows Vistaのインストールに適していましたが、Aeroインターフェイスを使用できるのは46％だけでした。



つまり、当時のラップトップの半分未満が、Windows Vistaを完全に機能させるための準備が整っていたということです。



915チップセットのイベントとWDDM規格の削除の後に展開されたストーリーはまだ語られておらず、多くの「what-if」のために正確にそれを伝えることは不可能かもしれません。



公開されたドキュメントを分析すると、ソフトウェアの巨人とそのOEMパートナーは、Windows Vistaのグラフィカルインターフェイスで少なくとも18か月の作業を失っていると自信を持って言えます。 そして、一見すると、2006年1月の突然の変化（WDDM標準を放棄）が低い標準を設定しているように見えるかもしれませんが、それはWindows Vistaのニーズを満たすには不十分であり、完全な機能を備えた栄光を示しています。 これがマイクロソフトとインテルの利益のためだけでなく、深い意味を持ち、ユーザーの便宜のためだけに意図されていることを願っています。 このオプションは私には完全にユートピア的であるように見えますが、事実が公表されたことを考慮してください。
