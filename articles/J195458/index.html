Raspberry Piの分散音楽プレーヤー



長い間、私はこの投稿を書くつもりでしたが、進歩は遅かったです。 この質問はペースを加速するように促しました。そこで私はそのような投稿を書いていると言って不本意だったので、彼は強い関心を呼び起こしましたので、私はこの約束を果たす以外に選択肢がありませんでした、私は質問M03Gの著者に特別な感謝を表しますこの加速ペンデルを手に入れました。 そして、この投稿は金曜日にリリースされるので、かなり自由な（申し訳ありませんが、的な非自発的な）プレゼンテーションを許可しました。 みんなが満足することを願っています。



（はい、投稿は多くの点でOdroid U2の投稿分散オーディオプレーヤーと重複していますが、設定が少し簡単です）





プロローグ



部屋数が1を超えるアパートに住み始めるとすぐに、2003年に同様のシステムを夢見始め、キッチン（そして実際にはアパート全体）での音楽の問題が最高潮に達しました。 状況は、アパートがニキータセルゲイエヴィチフルシチョフの建築の天才に従って建てられ、コンピューターが「奥」の部屋に置かれ、キッチンがまったく聞こえないようになったという事実によって悪化しました。











時間の経過、アパートの変更、さまざまな選択肢の検討と破棄、夢は残り、バックグラウンドを完全に残すか、潜在意識の奥から再び浮かびましたが、カートは動きませんでした。











背景



すばらしいシングルボードのRaspberry Piコンピューターが広く利用可能になるまでに、私は3か所の居住地、同じ数の仕事、居住国を変更しました。そのため、何が起こっていたのか、私の夢は弱い声でも声を出さず、long睡の夢で眠りに落ちましたメモリの裏庭にほこりだらけのクローゼット。 しかし、とにかくマリンカを注文しました。 なぜ-彼はまだ理解していませんでしたが、抵抗することができませんでした：価格/潜在的な比率がすべてを決定しました。 「将来的には、NAS、実験用のWebサーバー、その他多くのこともある」と思いました（実は、NASを引っ張ることはありませんが、そのことを知っていても気にしません止まらなかった）。











しかし、「ラズベリー」が私の手に落ちた後、私は仕事と個人的なプロジェクトで閉塞を整理したので、「ベリー」は棚に置かれ、電気を味わわずに数ヶ月間置かれました。











しかし、すべてが変わったら。



ネクタイ



暖かい夏の夜には何もすることはありませんでしたが、それは数年前からありませんでした。私はこの機会を逃さず、少なくともクローゼットの中に小さなゴミを捨てることに決めました。 そこで私はブラックボックスに出会い、悲しいことに空のスロットのスロットを通して私を見ました。

「どっちでも、これはある！」
-私は考えて、すぐに彼女のシステムのイメージのためにインターネットに乗った。 彼女はテーブルの上に横たわり、音声出力が私の左目をまっすぐに見つめ、愚かな質問をしました。

「さて、私たちは何かをするつもりですか？」
計画はすぐに成熟しました：音楽付きの外部ネジをusbポートに差し込み、スピーカーをオーディオ出力に接続し、MPDをラズベリーに入れて、家のどのコンピューターからでも制御できるアクセス可能な優れた音楽プレーヤーを取得します（顧客の利益はどのOSでも完全です）。 だから、マリンカは最終的に用途を見つけ、多色のLEDで陽気に点滅し、それは良かったです。



良いが、十分ではない。

「はい、Icecast2のようなすばらしいものがあります。これはMPDとうまく連携して、音楽をWebにストリーミングします！」
-私の心を通してフラッシュしました。 そして、ネットワークからオーディオストリームを取得できるプレーヤーであれば、どのデバイスでも聞くことができます。 この考えは非常に大声で響き渡り（私は耳をつんざくような）、忘れられた塵のクローゼットに転がりこみ、絶望的に眠っている夢を目覚めさせ、すぐにブリキのマグカップで壁に打ち込み始めました：

「さらにラズベリーを食べるのを止めている人は、ネットワークからのコマンドで、同じmplayerをオン/オフして特定のURL（Icecast2が最初のラズベリーからブロードキャストするのと同じもの）をオン/オフする簡単なソフトウェアを作成し、最後にここから出させます終わりの??」
しかし、ここで私（と彼女）は最初の不満を待っていました（後で判明-良くなったように）：ラズベリーの力では、その場でストリームを絞ってストリーミングするには不十分でした-音が途切れました。 Icecast2をつまらないように「説得」できるかどうかを調べ始めました。 そして、MPD configを吸っているときに、「pulseaudio」というおなじみの言葉に出会いました。

「それでは、これは、ネットワーク上で潜在的に再生できる種類のがらくたですか？」 あるいは、Icecast2と自作のオン/オフスイッチで庭をフェンスで囲む必要はないのでしょうか？」
そしてそれが起こった...そして、クローゼットは組み立てられていないままでした。



行こう！













行きましょう。 ソース素材：2つ以上のRaspberry Pi（1つはMPDが回転するサーバー、残りはクライアントになります）、サウンドを再生するための同数のスピーカー、Linuxについての少しの知識、そして夜の自由時間。 もちろん、正しいスペルのタグを含む音楽コレクションは、 MPDはそれらと共に動作します。 これらはすべて私の所有物でした。そのため、私は事前に妻に通知し、翌晩そこにいなかった、病気になり、家から姿を消し、バンクーバーに向けて出発し、一般的にこの惑星からナフィグを移住し、明日の午後に戻るとグーグルに没頭。 Googleは失望しませんでした。



準備段階（サーバーとクライアントで同じ）



Raspberry Piにシステムをインストールするプロセスについては詳しく説明しませんが、インターネット上の指示は完了しています。 私は、8年の経験を持つアマチュアDebianとしてRaspbianを使用したことを予約することができるだけなので、すべての指示は彼のために与えられます。 一般的に、画像をダウンロードし、フラッシュし、カードを挿入し、ロードします。 初期設定も標準です。私がやったことをリストします。



  • パスワードを設定します（もちろん）
  • アクティブ化されたsshサーバー
  • タイムゾーンを設定する
  • 読み込みxを削除
  • Xがなくてもビデオメモリを無駄に費やす必要がないため、ビデオメモリの量を最小（16メガバイト）に設定します。


コンソールにあまり自信がない場合は、最後の2つのポイントをスキップして「ラズベリー」GUIから直接構成できますが、ssh経由でログインすることですべてを実行しました。



必須項目：各「Malinka」にルーターの設定で静的アドレスを割り当てます（または、独自のネットワーク設定で違いはありません）-その後、非常に役立ちます。



さあ、始めましょう：



1.最初に、システムを最新の状態に更新し、PulseAudioをインストールします。 一般的に、ルートから多くのことを行うようになりましたので、好きな方法でアカウントにアクセスしてください（ 好みに合わせて選択してください ）： 



  $ sudo -i
 ＃aptitude update && aptitude upgrade && aptitude install pulseaudio




2.なぜなら 読み込み後すぐにすべてが機能することが想定されます。理想的にはXとユーザーログインなしで、システム全体のモードで動作するにはPulseAudioが必要です。 これを行うには、次のようにファイル/ etc / default / pulseaudioを編集します。 



  ...
 PULSEAUDIO_SYSTEM_START = 1
 ...




3.次に、すべてが機能するために、ユーザーpiをパルスアクセスグループに追加する必要があります。 



  ＃usermod -a -G pulse-access pi




4.ここで、ネットワークを介した音声の受信を許可します。 これを行うには、行を/etc/pulse/system.paに追加します 



  ...
 load-module module-native-protocol-tcp auth-ip-acl = 127.0.0.1; 192.168.1.0/24
 ...


ここでは、「自分自身」とアドレス192.168.1のローカルネットワークからのすべてのデバイスの両方を承認します。*（異なるアドレス範囲がある場合は、それに応じて調整します）。 原則として、最初の部分はサーバー（それ自体で再生する）にのみ必要であり、2番目の部分はクライアント（ネットワークから受信するため）に必要ですが、両方を行います。そのため、ラズベリーは最小限の動きで交換可能です。



4.実際に、pulseaudioサービスを開始します 



  ＃サービスpulseaudio start




これで一般的な部分は終わりです。 クライアントとサーバーの設定に移りましょう。



お客様



誰かにとっては奇妙に思えるかもしれませんが、「クライアントの準備ができている」ため、追加の構成は必要ないので、 約50メートルのサーバーに厳密に移動し、サーバーを処理します。



サーバー



まず、サーバーにMPDをインストールすると同時に、コンソールクライアント（初期セットアップ用）をインストールする必要があります。 



  ＃aptitude install mpd mpc




config /etc/mpd.confを編集します。 彼は、Linuxの他の多くの構成と同様によくコメントされているので、基本的な必要な変更のみを概説します。



コレクションへのパスを指定します 

  music_directory "/ mnt / disk1 / music / mp3 /ソート済み"




任意のネットワークインターフェイスへのBindim： 

  bind_to_address "any"




標準ポートへ： 

 ポート「6600」




ファイルを変更するときにコレクションを自動更新する： 

  auto_update "yes"




そして最も重要なことは、オーディオ出力を構成することです。 ここでは、クライアント「ラズベリー」のそれぞれに1つの出口を作成し、さらにサーバー自体に1つの出口を作成します。これは、同等のクライアントとしても機能します。 ここで、各ラズベリーに割り当てた静的IPアドレスが役立ちます。 



  audio_output {
        タイプ「パルス」
        名前「オフィス」
        サーバー "localhost"＃サーバー
 }

 audio_output {
        タイプ「パルス」
        名前「リビングルーム」
        サーバー "192.168.1.4"＃クライアント＃1
 }


（利用可能な顧客の数など）



さて、MPDを再起動します。 これでルートセッションを完了できます。 



  ＃service mpd restart
 ＃終了




コレクションをスキャンしてデータベースを作成します（コレクションのサイズによっては時間がかかります）。 

  $ mpc -w update




実際、それだけです。



ルリム！



残りは何もありません。この全体を管理する方法です。 ここでは複雑なことは何もありません。あらゆる種類のオペレーティングシステム用のワゴンと小さな顧客用トロリーを用意しています。 選択する際には、クライアントが出力チャネル（または、出力デバイスとも呼ばれる）を切り替えることができるかどうかに注意してください-実際、これは特定のクライアントを有効/無効にする機能です。 Macの場合、 テルミン 、 Gmpcの Linuxの場合、Windowsクライアントを使用する必要がなかったため、何もアドバイスできません。
