Web OS：1つの幻想の未来

インターネットは実世界よりも収益性が高いです。 そして精神的な安らぎの面だけではありません。 インターネットでの販売は、店舗での販売よりもはるかに収益性が高くなります。買い手はどこにも行かなくてもよく、売り手はモールで何かを借りる必要はありません。 データを保存する

インターネットは、自宅のコンピューターよりもはるかに便利です。ネットワークにアクセスできれば、どこからでも情報を入手できます（この「if」について詳しく説明します）。 まあ、そして最も重要なこと-インターネットでは、先に行くほど、サービスにお金を払わず、代わりに広告を見ることが可能になります。 それに反応する必要さえありません。



ソフトウェア製品は、デジタルの世界で主なサービスであり続けています。これが、現代のコンピューターが科学者向けのツールから家電に変わった理由です。 80年代前半以来、ソフトウェアはコンピューター市場を形成してきました。価格が安ければ安いほど、理解しやすく、機能的であるほど、それが取り付けられたコンピューターの販売は良くなります。 人々はコンピューターを購入し、ソフトウェアを購入、購入、購入しました。大衆市場では、製品自体よりもアクセサリーやサービスが常により多くの利益をもたらします。



Sweet Vinegar Web 2.0



2004年4月1日までです。 この晴れた日に、GoogleはGMailメールサービスを開始しました。 誰もがすぐに何が起こったのかを理解したわけではありません-多くの人は単に、Googleがオタク向けのおもちゃをリリースしたと判断しましたが、Ajaxのみを使用しており、... 。 すぐに、GMailは通常の電子メールクライアントと同じくらい悪くなく、競合他社からの多数のクローンとピンと張ったバージョンが追加されました。 史上初

オタクは、普通の人と一緒に、Webインターフェースを介してメールを読み、サーバーに保存し始めました。POP3を介してそれを取得しませんでした。



その間、GoogleカレンダーとGoogleドキュメントが登場し、インターネットアプリケーションの分野だけでなく、聖地であるオフィスプログラムにも侵入しました。 彼らはまた、アナログと新しい標準を伴いました。 Webアプリケーションは、デスクトップの兄弟に比べて機能がまだ劣っていますが、誰もがこれが今だけであることを理解しています。 これらは単なるWebアプリケーションではないため、無料のアプリケーションです。



今お話しすることはおもしろそうに思えるかもしれませんが、最初の一歩はマイクロソフト以外の何者でもありません。 失われたWebブラウザー市場を追求して、彼らはNetscapeを打ち負かそうとし、Win-Winの戦術を選択しました。彼らはその時点で最もフル機能のIE4ブラウザーをリリースし、無料で配布し始めました。 そして、この戦術的な決定を下した人々がその戦略的結果を予見したことはありそうにありません。ブラウザ市場を征服したマイクロソフトは、実際、ダウンロード可能なアプリケーションのための無料プラットフォームをリリースしました。



IE4はHTMLだけでなく、プログラミング言語（JavaScriptおよびVBScript）も理解し、ページをリロードすることなくサーバー（およびハードディスク）とデータを交換できますが、警告および制限はありますが、HTMLドキュメントをオブジェクトモデルと見なしました。 Mozilla FirefoxとOperaは非常に迅速に彼を追いかけた後、エキゾチックに追いついた

サファリ 2004年までに、それらはすべて、インターネットからダウンロードされ、サーバーにデータを保存するアプリケーションを作成するための本格的なプラットフォームであり、Microsoftは1998年には予測できなかったが、オペレーティングシステムに依存していなかった（IE4の完全な機能は、クライアントおよびサーバー側から）。



長い間、Webアプリケーションは特定の特定の問題の解決にのみ関係していましたが、すぐにインターネットがより高速かつ安価になり、ブラウザがより強力になりました。2004年以降、疑問が生じました。ブラウザでほとんど何でもできるなら、なぜオペレーティングシステム全般が必要なのですか？ これまでのところ、これはブラウザーとWebアプリケーションの間の余分なレイヤーではありませんが、どのくらいの期間ですか？



何もない、テレビのみ



実践が示すように、ソフトウェアの製造業者と消費者の両方が道徳的にインターネットOSに切り替える準備ができています。 インターネットは現在、一般ユーザーの生活の中で5〜6年前よりもはるかに多くのスペースを占有しています。今日のコンピューター初心者（およびあらゆる年齢層の人々）が質問を聞くことは珍しくありません-新しいアカウントを購入してもMail.ruアカウントは残りますか？ハードウェア、またはコンピュータ上でごみ箱からファイルを復元できる理由、およびフォーラムへのコメントは不可能です。 特に、最も人気のあるOSの開発者はこれに対して多くのことを行ってきたため、ユーザーの意識はローカルデータと外部データを区別しません。



オペレーティングシステムのメーカーには、Webと統合するさまざまな理由があります。まず第一に、コンピューター市場の技術的および心理的状態がWeb OSというコード名のオープンスイミングプロジェクトの立ち上げを可能にする、決定的なスタートの準備ができていないという恐怖です。 主にGoogleのウェブアプリケーションのメーカーは、それ以外にも理由があります。現在、既製の製品ベースを持っているのは誰ですか。 存在しない場合の優位性のための戦い

高速インターネットの世界は本格的ですが、誰も明確な戦略を提案していません。そのため、勝者は依然として深刻な疑問を抱えています。

1つの頭は良いが、まだ



現時点では、最も人気のある3つのユーザーOSと、Webソフトウェアの大手メーカーがあります。 それらはすべてこの方向にいくつかのステップを踏んでおり、それらはすべて明らかに明らかに不十分です。 まず第一に、インターネットはウェブだけではなく、次に何が起こるか誰も知らないからです。 これまでのところ、Webは将来のWeb OSのプロトタイプをテストするための最も人気のあるプラットフォームですが、

テクノロジーが端末のイデオロギーに戻る前に決定的な行動に賭けることは誰もが恐れています。



ちなみに、2000年代前半には、LAN内の端末を介して作業を整理しようとする試みは非常に人気があり、私はそれらの1つを小さなIT組織で見ました。 正直なところ、私は長い間、これらすべての試みが失敗した理由を理解していませんでした。 管理の観点からは、モニターとキーボードのみが接続されている単一のサーバーを介して会社の作業を整理することは、管理の観点から非常に便利です。

労働リソース、および機器コストの面で。 しかし、実際には、すべてがそれほど単純ではないことが判明しました。パフォーマーは常に端末の機能を欠いており、拡張しようとすると、予期しない技術的な制限に直面しました。 理論的にはすべてが機能するはずですが、実際にはより高価であることが判明した状況は、Webを介して労働組織に関係している人にはよく知られています。



重要なビジネスプロセスがオフィスプロバイダーの汚いシステム管理者に直接依存している状況では、モスクワからコストロマへの数十ギガバイトの転送はFTP経由よりも車で高速であり、端末ポリシーの勝利を待つのはばかげています。



ポケット、ハンドバッグ



しかし、それを準備しないことはバカなことです。 いずれにせよ、コンピューターに加えて、通信技術の機能の非常に発達した別の領域があり、そこでは末端原理が決定的で無条件の勝利を勝ち取ったからです。 これはモバイル通信です。 プロトコルの開発にもかかわらず、ユーザー機能自体がかなり弱かったため、長い間無視することが可能でした。 しかし、その後、iPhoneが登場しましたが、これは機能を持たないかもしれませんが、人々はそれを使用し、仮想的に持っていることを知っているだけではありません。 彼の登場は他のメーカーに拍車をかけ、今ではモバイルWebが毎月勢いを増しています。



4頭すべてのクジラはこの戦いに熱心に参加します。この戦いでは、クジラがいなくても世界を支配するのに十分な偽装者がいます。 AppleはモバイルOS XでiPhoneを起動し、ノキアとモトローラはモバイルLinuxを試します（OpenMokoもありますが、Linuxの世界全体に見られるように、多くの機会とユーザーの千分の1がいます）そして彼らは皆、Blackberry OSのアメリカでの成功を期待しています。



同時に、すべての主要なプレーヤーは、彼らにとって珍しい領域で足場を獲得しようとしている-可能なすべてをカバーしようとしています-そして、デスクトップ、およびWeb、および電話。 Apple、Microsoft、およびLinuxコミュニティの両方に、コンピューター、スマートフォン、および独自のWebサービス用の独自のオペレーティングシステムがあります（ただし、Linuxは完全にオタクな方法で実装されていますが）。 Googleにはデスクトップアプリケーションがありますが、デスクトップOS

5つの気の重い質問



Web OSを表示するには何が必要ですか？ 最初の答えは明らかです。データ転送速度を上げる必要があります。 同時に、このデータがどのように送信されるかは誰にもわかりません。新しいケーブルや衛星、WiFi / WiMax、またはまったく新しいワイヤレステクノロジー（そしてカバレッジエリアではどうでしょうか）、または3Gで敗北します（日本ではすでに深刻です） 4Gについて話し合う）。 または、一般的に、上記のいずれも機能せず、インターネットはメーター周波数で送信されます

デジタルテレビに切り替えた後。 超高速通信がどのように実行されるかが明確になるまで、具体的な計画や設計はできません。



第二に、デバイス自体に何が起こるかはあまり明確ではありません。 一方では、振り子がデバイスのサイズを縮小する方向にどのように揺れるかを確認します。 これは、生産性が向上するまで常に発生しますが、サイズ重視のテクノロジーが登場します。 最初のCDプレーヤーがどれほどかさばったかを思い出してください

マイクロカセットレコーダー？ コンピューターの小型化、ラップトップの普及が進んでいますが、ナノテクノロジーの勝利はまだ速くなく、パフォーマンス要件がデバイスのサイズに達する可能性があります。 したがって、何を賭けるべきかはあまり明確ではありません-画面スペースの各領域が役割を果たすマニピュレーター制御を備えた従来のウィンドウインターフェース、または空間制御組織のまったく異なる法則であるタッチパッド？ マイクロソフトは、私たちが知っているように、より強力なアップルほど大きなタッチパッドに引き寄せられますが、逆にアップルは、

彼はiPhoneを超えて、電話とコンピューターを別々に開発しようとはしていません（これが、彼のファンがJobsを長時間待たない理由です。タブレットPCです）。

3番目の質問は、データストレージの問題です。 プライバシーをどうするか？ ボットネットとスパイウェアは今日のインターネットに殺到しており、インターネットから切断せずに安全を感じることは不可能です。 Web OSの概念は、外部サーバーへの完全な信頼を意味します。 情報を個人的に管理するにはどうすればよいですか？ 外部メディアのコピーは常に信頼性が高くなります。なぜなら、あなた自身だけがその安全性と

彼女の確信。 しかし、外部メディアを許可する場合、なぜウェブが必要なのでしょうか？



4番目：はい、Webはブラウザーに優しいです。 しかし、一般にHTTPとハイパーテキストは強力なツールであることを忘れないでください。しかし、普遍的ではなく、すべてのタスクに最適ではありません（どうしてWebDAVがFTPに取って代わらないと思いますか？）1つの大きなブラウザを備えたオペレーティングシステム全体がある場合-なぜですか？それはウェブブラウザですか？



そして最後に、私がこのコラムを始めたのは何ですか-そしてお金はどうですか？ 今日のオペレーティングシステムの開発とサポートには、すでに多くのお金の価値があります。そして、それらに通信料金も追加する場合はどうでしょうか。 アプリケーションを操作するためにユーザーからお金を受け取らないGoogleを引き続き検討する場合はどうすればよいですか？

私たちが世界の終わりを迎えたときのことを覚えていますか？



これらの問題はすべて、私が言及するのを忘れていた問題と同様に、私たちが自分自身に質問する前に解決しなければなりません。「いつの日が来るのか」「テレビはありません」。 明らかに、クライアント-サーバーの概念は徐々にターミナル-コンピューターの概念に成長し、それによってこのすべてのデジタルの歴史が始まりました：振り子は後退します。 しかし、ほとんどGoogle、Microsoft、Apple、または

（ご主人様、ごめんなさい）今日のLinuxコミュニティは、実際にどのように見えるか想像できます。



Microsoftは、.NETと呼ばれるテクノロジを導入し、デスクトップ、電話、およびゲームコンソールの消費者をMS Live Webサービスにリンクすることにより、細長いWinAPIピラミッドを破壊していることがわかります。 Appleは困難ときしみを抱えてライブを視聴していますが、そのばかげた.Mac Webサービスを、より現代的なMobile Meに変換し、すべてのMP3プレーヤー、電話、コンピューターを単一のユーザースペースに統合しています。 Googleは、ウェブおよびモバイルプラットフォームでのGoogleの革新を追っています。 それらはすべて、
