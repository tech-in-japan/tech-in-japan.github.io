登録から店頭へ：モバイルアプリケーションをApp StoreとGoogle Playに配置する方法（パート2）





チュートリアルの最初の部分では、App StoreとGoogle Playの両方のストアに登録しました。 ただし、アプリケーションをダウンロードできるようにするには、まだ多くのアクションを実行する必要があります。 それらを見てみましょう。



アップロード手順に進む前に、アプリケーションが公開アプリケーションの品質とコンテンツの要件（ Google 、 Apple ）を満たしていることを確認してください。 これらの要件を順守しないと、初心者の開発者が公開の拒否や違反の警告を受け取ることに驚かされることがよくあります。



まず第一に、アプリケーションには何らかの機能、つまり利益と価値が必要です。 アプリケーションが少なくともその類似物と多少異なることを確認してください。 店舗で類似品を探していない場合は、怠けすぎないでください。この段階で、多くのプロジェクトの作成が終了します。 これは、「革新的な」アイデアを持つ初心者には特に当てはまります。



誤って記述されたコードのために拒否される多くのオプションもあります。 これは主にアプリケーションのクラッシュに関するものです。 また、多くの開発者は、ベンダーが提供するソフトウェアインターフェイスが不足している場合、内部APIにアクセスしようとします。 しかし、APIが閉じられている場合、理由はありません。 使用したテクノロジに関するドキュメントを読み直してください。何かを見逃していることを確認してください。



適切な許可なしに、アプリケーションの名前またはそのコンテンツに他社の名前、商標、著作権で保護された画像および画像（スクリーンショットとしても）を使用する場合、application責で「ラップ」されるアプリケーションを安全に期待できます。 したがって、このような状況は慎重に避けてください。 著作権で保護された素材を使用または言及する必要がある場合は、適切なフォームを使用してその素材を使用する権利を直ちに証明する必要があります。



複雑なインターフェイスまたは理解できないインターフェイスは、障害のもう1つの一般的な原因です。 アプリケーションを5分間操作してもユーザーの目を傷つけないように、アプリケーション画面が過負荷にならず、読みやすく、色が選択されていることを確認してください。 両社には、インターフェース設計に関する推奨事項がありますので、確認してください（ Google 、 Apple ）。



Appleは、宣言されたすべての機能のパフォーマンスを特に懸念しています。まだ完了していないスタブを配置した遠いメニュー項目がある場合、レビュアーはこれを見つけて、最初にアプリケーションを変更するように強制します。



アプリケーションがユーザーとデバイスに関する情報を収集する場合、このデータの使用方法をアプリケーションでできるだけ明確に説明する必要があります。 説明なしに、あらゆる種類のテレメトリを収集してサーバーに送信することは推奨されません。



組み込みの支払いシステムであるApp StoreとGoogle Playを使用してユーザーからお金を受け取ることを計画している場合、アプリケーションがこれらのシステムでサポートされているカテゴリのいずれかに該当するかどうかを確認します。 たとえば、アプリケーションを介してサービスを販売する場合、他の支払いシステムを「固定」する必要があります。



バックグラウンドモード。 これらは、アプリケーションが最小化/閉じた形式で使用できるモードです。 ただし、アプリケーションには、これらのモードの選択を正当化する機能が含まれている必要があります。 オーディオがあり、オーディオがない、またはVoice over IPがあるが、アプリケーションから電話をかけることができない場合は、拒否を待ちます。



ほとんどの場合、アプリケーションは次の理由でレイアウト時に節度を通過しません。



  1. 付随するすべてのフォームの記入が不完全です。
  2. バグと「クラッシュ」。
  3. アプリケーションは、偽の、詐欺的な、または誤解を招くデータを使用します。
  4. サードパーティアプリケーションのアイコンと同様のアイコンが使用されます。
  5. インターフェイスを開発する際、公式ガイドライン（Apple、Google）からの推奨事項は考慮されませんでした。
  6. 宣言された関数および/またはスクリーンショットと実際のアプリケーションの不一致。


ハッキングのためのアプリケーションの使用、スパム、不正な誘引方法の使用など、あらゆる種類の悪意のあるものについて -あなたは言及することはできません。



アプリケーションが要件を満たしているかどうかを確認するという質問に対して責任あるアプローチを取らないと、開発者アカウントが完全にブロックされる可能性があります。



アプリケーション検証プロセスについての結論として、公式ウェブサイトからの引用：「 私たちは、境界を越えるコンテンツまたは機能を備えたアプリケーションを拒否します。 この特性はどこにありますか？ ある裁判官がかつて言ったように、「私はそれを見ればわかる」。 そして、あなたもあなたがそれを越えるときを理解するだろうと思います 。」



これで、アプリケーションをテストし、全世界と共有する準備ができました。 適切な手順を見てみましょう。



アプリストア



使用ツール：



  • Appleコンピューター（OS Xがインストールされている）。
  • Appleストア-Appleアプリストア。
  • developer.apple.comはApple Developer Centerです。
  • itunesconnect.apple.com-アプリケーションの管理、ストアへのアップロード、統計、財務などの表示を行うサイト
  • Xcodeは統合開発環境です。
  • Application Loader-収集されたアプリケーションアーカイブをiTunes ConnectにダウンロードするためのAppleプログラム。


App Storeにアップロードされた各アプリケーションは、Appleが発行した有効な証明書で署名する必要があります。



アプリケーションに署名するには、次のものが必要です。



  1. Appleに証明書を要求します。
  2. デベロッパーセンターに新しいアプリケーションを追加します。
  3. プロビジョニングプロファイルを作成します。
  4. itunesconnect.apple.comポータルで新しいアプリケーションを作成します（以降-iTunes Connect）。
  5. アプリケーションに関するすべての必要な情報を入力します。
  6. プロビジョニングプロファイルで署名して、Xcodeからアプリケーションアーカイブを収集します。
  7. Application Loader（Appleソフトウェア）を使用して、アーカイブをiTunes Connectにダウンロードします。
  8. アプリケーションの新しいバージョンでダウンロードしたアーカイブを指定し、確認のためにアプリケーションを送信します。




各ステップをより詳細に検討してください。



  1. Appleからの証明書要求。

    1. Xcodeに移動します。
    2. Xcode->環境設定を開きます...
    3. Apple IDを選択します。 リストにない場合は、下部の「+」をクリックし、「Apple IDを追加」をクリックします。



    4. 右側で、アプリケーションをレイアウトするグループを選択し、[詳細の表示...]をクリックします。
    5. 証明書をダウンロードした後、上部の「iOSディストリビューション」のボックスを探し、右側に[作成]ボタンがある場合はクリックします。 証明書が作成されました。



  2. Dev Centerに新しいアプリケーションを追加します。

    1. developer.apple.com/account/ios/identifier/bundleにアクセスします。
    2. 「+」をクリックします。 アプリケーションの名前を入力し（Dev Centerのみ）、バンドルIDはアプリケーションの一意の識別子であり、プロジェクトで指定されたものと一致する必要があります。 通常、com.company.appname。







      必要なサービスを以下に示します。 これらは、プロジェクトで指定したものと一致する必要があります（Xcode-> Target-> Capabilities）。



    3. [完了]をクリックします-アプリケーションエントリが作成されました。
  3. プロビジョニングプロファイルの作成。

    1. developer.apple.com/account/ios/profile/productionにアクセスします。
    2. 「+」をクリックします。 [配布]-> [App Store]を選択します。 続行をクリックします。
    3. 作成した証明書（iOSディストリビューション）を選択します。 続行をクリックします。
    4. 証明書の名前を設定します。 続行をクリックします。
    5. 証明書が作成されました。 それをあなたのコンピューターにダウンロードして開いてください-今、あなたはそれをXcodeで持っています。
  4. iTunes Connectで新しいアプリケーションを作成します。

    1. 「マイアプリケーション」セクションのitunesconnect.apple.comにアクセスします。
    2. 「+」をクリックし、「新規アプリケーション」をクリックします。



    3. アプリケーション名、バージョン（1.0）、SKU-個人会計システムの任意の識別子を入力します。
    4. メイン言語を選択します。 [バンドルID]フィールドに、開発者アカウントのすべてのアプリケーションがロードされます。 前に作成したものを選択します。
    5. 作成をクリックします。 アプリケーションレコードが作成されました。



  5. アプリケーションに関するすべての必要な情報を入力します。

    1. itunesconnect.apple.comの [マイアプリケーション]セクションに移動し、リストからアプリケーションを選択します。
    2. AppStore->アプリケーション情報。 ここに、アプリケーションに関する一般情報があります。
    3. AppStore->価格と可用性。 ここで、アプリケーションのコストとアプリケーションを利用できる国を指定できます。



    4. 以下で「1.0送信の準備」を選択します-これは、App Storeに送信するアプリケーションの最初のバージョンです。 ここでは、ページ全体を閲覧して、アプリケーションに関する最も詳細な情報を指定する必要があります：スクリーンショット（デバイスごとに最大5個、シミュレータで最も便利に行われます）、説明、アプリケーションアイコンなど アプリケーションで認証が提供されている場合は、以下のテストアカウントのデータを入力することを忘れないでください。 Appleの従業員がアプリケーションにアクセスできない場合、すぐに拒否されます。
  6. プロビジョニングプロファイルで署名して、Xcodeからアーカイブを収集します。

    1. Xcodeに移動し、左側のファイルインスペクターでプロジェクト->ターゲットを選択し、アプリケーションを選択します。 ビルド設定セクションには、コード署名セクションがあります。 [プロビジョニングプロファイル]フィールドで、ステップ3でダウンロードしたプロファイルを選択します。すべてが正しい場合、[コード署名ID]フィールドで、ステップ1で作成したiOS配布証明書を選択できます。



    2. 上部のXcodeメニューで、[製品]-> [アーカイブ]を選択します。 アーカイブが完了したら、[アーカイブ]オーガナイザーウィンドウで、作成したアーカイブを選択して[エクスポート]をクリックし、[iOS App Store展開用に保存]を選択して、アーカイブを保存する場所を選択します。 このアクションが完了すると、指定されたパスに.ipa拡張子のアーカイブが作成されます-これがアプリケーションのアーカイブです。
  7. Application Loaderを使用して、アーカイブをiTunes Connectにダウンロードします。

    1. iTunes Connectホームページで[マイアプリ]をクリックします。 ドキュメントのリストとインストール用のダウンロードのページが表示されます。
    2. 「Download Application Loader」リンクをクリックして、自己解凍インストールパッケージをダウンロードします。
    3. インストール後、Apple IDを使用してアプリケーションにログインする必要があります。
    4. ログイン後、「アプリの配信」ボタンをクリックして、手順6で作成したアーカイブファイルを選択します。



  8. ダウンロードしたアプリケーションの新しいバージョンのアーカイブを指定し、検証のためにアプリケーションを送信します。

    1. 「マイアプリケーション」セクションのitunesconnect.apple.comにアクセスします。 前に作成したレイアウト済みアプリケーションを見つけます。 「1.0出荷準備」を選択します。 [アセンブリ]セクションで、ダウンロードしたアプリケーションアセンブリを指定します。



    2. 一番下で、自動または手動で確認した後にアプリケーションを公開するかどうかを選択できます。 手動モードでは、適切と判断したときにアプリケーションを公開できます。



    3. 変更を保存します。 上から「確認のため送信」ボタンを押します。



    4. すべてが正しく行われた場合、「検証待ち」というファイルが表示されます。 エラーがある場合は、修正が必要な情報を含むメッセージが表示されます。 すべてのアプリケーションステータスの変更には、Appleからの手紙が添付されます。 通知を設定するには、Itunes Connectのホームページ->ユーザーとロール->リストからユーザーを選択->「通知」を選択します。 ここで、ユーザーが受信する通知を構成できます。


Google Play



  1. APKファイルに署名します。

    Google Playにアプリケーションを配置するには、特別なキーで署名する必要があります。 キーには2つのタイプがあります。



    • 開発-開発環境からデバイスへのインストール中に絶対的にすべてのアプリケーションが署名されるキー。
    • production-Google Playにアップロードする前にアプリケーションに署名するためのキー。


    2番目のタイプのキーが必要になります。 アプリケーションごとに固有であり、アプリケーションへのすべての更新に署名する必要もあります。 突然それを失った場合、それを復元することは不可能になり、完全に新しいアプリケーションをアップロードする必要があります。



    署名プロセスを検討してください。

    1. [署名済みAPKの生成]をクリックします。



    2. 「次へ」をクリックします。



    3. 次の段階で、キーストア（キーストア）へのパス、そのパスワードを指定する必要があります。また、アプリケーションが署名されるこのリポジトリ内のキー自体も選択する必要があります。 キーストアとキー自体がまだないため、「新規作成」をクリックします。



    4. 表示されるウィンドウで、作成されたキーストアファイルとそのためのパスワードが置かれるパスを選択します。 次に、キー名とパスワードを入力し、キーの寿命（少なくとも25年）を選択します。 「証明書」ブロックでは、「名と姓」フィールドに入力するだけで十分です。残りのフィールドはオプションです。



    5. キーが正常に作成されると、データが入力された前のウィンドウに戻ります。 「次へ」をクリックします。



    6. 表示されるウィンドウで、「完了」ボタンをクリックします。







      次に、アプリケーションのAPKファイルを作成するプロセスが起動します。その最終段階は、新しく作成されたキーを使用した署名です。
  2. 新しいプロジェクトを作成し、APKファイルを開発者コンソールにアップロードします。

    1. 右上の開発者コンソールで、「アプリケーションの追加」ボタンをクリックします。



    2. 表示されるウィンドウで、Google Playストアのアプリケーションのページでデフォルトで使用される言語を選択します。 また、アプリケーションの名前を入力し、[APKのダウンロード]ボタンをクリックします。



    3. 次に、「作業バージョンの最初のAPKをダウンロードしてください」という画面の中央にあるボタンをクリックします。







      ウィンドウで、APKファイルへのパスを指定します。







      ダウンロードプロセスが開始されます。



    4. APKファイルのダウンロードに成功すると、次の画面が表示されます。







      「APKファイル」の行の反対側の左上隅で、灰色のチェックマークが緑色に変わったことに注意してください。 これは、ファイルを正常にダウンロードし、アプリケーションを正常に公開するためのGoogle Playの要件の1つを満たしたことを示しています。 次に、他のすべてのチェックマークを緑色にする、つまり、残りの要件を満たす必要があります。
  3. アプリケーションに関する必要な情報を入力します。

    1. 「Google Playのデータ」に進みましょう。 ここでは、アスタリスク*でマークされたすべてのフィールドに入力し、必要なすべてのスクリーンショット（少なくとも3個）を追加し、アプリケーションアイコンをロードし、アプリケーションのタイプとカテゴリ、年齢制限を示します。











    2. アプリケーションがロシア語圏の視聴者だけを対象に設計されていない場合は、[翻訳]ボタンをクリックして、他の言語で説明とスクリーンショットを追加できます。



    3. すべてのフィールドに入力した後、右上隅の[プロジェクトの保存]ボタンをクリックします。



    4. [年齢制限]タブに移動します。



    5. 最初のステップは、メールアドレスを入力して確認し、アプリケーションのカテゴリを選択することです。 次に、質問の小さなリストに答える必要があり、ページの一番下にある「保存」ボタンをクリックします。



    6. その後、「年齢制限の定義」の横にあるボタンが使用可能になり、クリックします。 質問への回答に基づいてリストが表示されます。 すべてが正しい場合、それを確認します-「年齢制限を設定する」ボタンをクリックします。







      すべてがうまくいった場合、次の画面が表示されます。



    7. 「価格と配布」タブに移動します。







      アプリケーションを無料で配布する場合は、「無料」ボタンをクリックします。 それ以外の場合は、[有料]をクリックします。 有料配布の場合、アプリケーションの価格を指定し、国ごとに個別の価格を選択する必要があります。







      アプリケーションの公開後、アプリケーションの支払いは変更できませんのでご注意ください！



      同じ画面で、アプリケーションをダウンロードできる国を指定する必要があります。



    8. ページを最後までスクロールして、すべての必須フィールドの前にチェックマークを付けます。



    9. 次に、右上隅の[プロジェクトの保存]ボタンをクリックします。
    10. アクションが正しく完了すると、右上隅の[公開]ボタンが使用可能になり、クリックします。



    11. おめでとう、1〜2時間（アプリケーションの公開にかかる平均時間）以内に、Google Playから作成した作品をインストールできるようになります。





したがって、アプリケーションはすべてのチェックに合格し、ストアに表示されました。 最初のダウンロードとユーザーレビューを期待しています。 しかし、他の多くのアプリケーションの中でユーザーの注意を引くことは非常に困難です。 したがって、アプリケーションの公開後、そのプロモーションに細心の注意を払う必要があります。 これを行う方法については、今後の出版物で説明します。
