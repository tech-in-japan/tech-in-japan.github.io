公式のAndroid 2.1がインストールされたHTC Heroの点滅

このトピックでは、FOTAを介してAndroid 2.1に既に公式に更新されたHTC Heroにカスタムファームウェアをインストールした経験を共有したいと思います。



少しの背景



数ヶ月前、MelskでVelcomから購入したウクライナロケールのHTC HeroのAndroid 2.1からの公式アップデートを待つのにうんざりしていましたが、代替ファームウェアのインストール方法を見つけようとしましたか？ しかし、不十分な構造の素材の豊富さ、いくつかの失敗した試み、再フラッシュ電話の使用経験に関する相反するレビューにより、私はこの問題を深く理解することを思いとどまらせました。



その結果、6月に公式の更新を待って更新されました。 更新により、修正されたRecoveryのインストールを可能にする脆弱性が閉じられ、その結果、カスタムファームウェアがインストールされるという警告によって、私は止められませんでした。 しかし、遅かれ早かれスクラップに対する方法はないと確信していましたが、方法は見つかりました。



その結果、2か月間Android 2.1を搭載したHTC Heroの幸せな所有者でしたが、今では同じファームウェア（99％）とルートを持つ同じ電話の所有者でもあります！



なぜルートが必要なのですか？ まず、Market Accessを使用して市場の有料アプリケーションにアクセスするために、インストールにルートが必要な便利なアプリケーションをいくつか集めました。



まず第一に、公式の、しかしルートのファームウェアにアップグレードすることにしました。 私の意見では、まず第一にこれで十分です。 MoDaCo、VillainROM、7Heroなどのカスタムファームウェアが必要かどうかは時間が経てばわかります。



この時点で、以下で説明する方法が個人的に検証され、特定の電話インスタンスで問題なく機能することを警告する必要があります。 ただし、これらの指示の実施の場合、あなたのデバイスに起こりうる結果についてはあなたのみが責任を負います！ 私はAndroidベースのデバイスやその他のデバイスのフラッシュの専門家ではありませんが、個人的な経験のみを共有しています。



また、点滅手順を実行すると、サービスセンターの保証が失われます。



規約



リカバリ -ファームウェア、バックアップの作成、バックアップからの復元などのツールを含むシステムパーティション



ワイプは完全なリセットです（WMへのハードリセットのアナログ）。 これにより、/ dataおよび/ cacheフォルダー、つまり インストールされているすべてのアプリケーションが削除され、すべてのシステム設定がリセットされます。



メス、妹！



フラッシング手順自体は約10分かかり（マニュアルと実験の予備調査を考慮することなく）、いくつかの簡単なステップで構成されていました。

  1. 必要なソフトウェアを準備します。
  2. 変更されたリカバリを使用して電話をダウンロードする。
  3. バックアップを作成します。
  4. 電話の設定をリセットします。
  5. カスタムファームウェアのインストール。


必要なソフトウェアを準備する



  1. HeroGSM21Rootユーティリティをダウンロードします。
  2. Hero_HTC_RUS_3.32.411.1-rooted2配布キットまたはインストールするものをダウンロードします。 ここで彼についての詳細。
  3. 配布キットHero_HTC_GB_3.32.411.1-rooted2をメモリカードのルートにコピーします。


変更された回復を使用した電話のダウンロード



  1. 電話機でUSBデバッグモードをオンにします。 メニュー>設定>アプリケーション>開発> USBデバッグ（USB接続時のデバッグモード） 。
  2. 電話をUSBケーブルでコンピューターに接続し、 充電専用モードを選択します（USBで電話を充電します） 。
  3. HeroGSM21RootをドライブCに解凍します（たとえば、C：/ HeroGSM21Root）
  4. Windowsコンソールを起動します。 スタート>実行> cmd 。
  5. cd C：\ HeroGSM21Rootコマンドを実行して、 HeroGSM21Rootフォルダーのコンソールに移動します。
  6. 変更されたリカバリの起動プロセスを開始します。 コンソールで、 runindos.batコマンドを実行します。
  7. スクリプトの完了を待っています。 完了すると、コンソールに次のメッセージが表示されます。

    "Files copied and permissions set, rebooting HTC Andriod 2.1"

    "Your phone will now reboot into normal mode, then reboot into recovery mode. I

    f it does not reboot the second time, wait 10 minutes and manually reboot and be

    gin again."

    "Your phone is now rebooting in Rooted Recovery mode, do a backup and load your

    ROMs"







  8. 電話機は、OSを完全にロードせずに1回再起動します。 その後、2回目の再起動が行われ、画面にroot権限でAndroidシステムリカバリインターフェイスが表示されます。


バックアップ



  1. [ バックアップ/復元 ]メニュー項目を選択します。
  2. [ Nandroidバックアップの作成 ]メニュー項目を選択して、システム全体のバックアップを作成します。
  3. Nandroidバックアップシステムの作成の質問ですか？ [ ホーム ]ボタンを押して、バックアップを確認します。
  4. システムイメージのコピーがsd card / nandroidに保存されます。
  5. バックアップが正常に完了したことは、バックアップ完了により確認されます！ 。
  6. システムのバックアップを作成した後、[ 戻る ]ボタンをクリックして、 Androidシステム回復のメインメニューに戻ります。


電話をリセットする



原則として、この手順をスキップすると、携帯電話、インストールされているアプリケーションなどのすべての設定が保存されます。 ただし、インストールされたアプリケーションの正しい動作が中断される可能性があります。

  1. [ ワイプ ]メニュー項目を選択します。
  2. システムを工場出荷時設定にリセットするには、[ データのワイプ/工場出荷時リセット ]メニュー項目を選択します。
  3. データとキャッシュシステムのワイプが要求されたら、 [ ホーム ]ボタンをクリックして、システムの工場出荷時設定へのリセットを確認します。
  4. 工場出荷時設定へのシステムリセットが正常に完了したことは、 データワイプ完了メッセージによって確認されます。 。
  5. システムを工場出荷時の設定にリセットした後、[ 戻る ]ボタンをクリックして、 Androidシステム回復メインメニューに戻ります。


カスタムファームウェアをインストールする



  1. sdcardからメニュー項目Flash zipを選択して、インストールされているファームウェアのイメージを選択します。
  2. システムは、以前にUSBフラッシュドライブにアップロードしたすべてのファームウェアの選択肢を提供します。 私の場合、これはHero_HTC_RUS_3.32.411.1-rooted2.zipです。
  3. トラックボールを使用して、目的の画像を選択し、クリックして選択を確認します。
  4. インストールの質問：SDCARD：Hero_HTC_RUS_3.32.411.1-rooted2.zip？ [ ホーム ]ボタンを押して、選択したシステムイメージの電話へのインストールを確認します。
  5. 選択したシステムイメージのインストールが正常に完了したことは、 [SDカードからインストール完了]によって確認されます。
  6. メニュー項目[ 今すぐシステムを再起動]を選択して、電話を再起動します。


説明されたHTC Heroのフラッシュのすべての経験は、フォーラムサイトw3bsit3-dns.comの多数の資料の助けを借りて得られました。
