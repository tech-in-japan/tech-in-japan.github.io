厳格な優先順位付け



Windows 8のアプリケーションの適切な設計（使用シナリオ、フォームファクター、インターフェイスソリューションは異なりますが、同じことがWindows Phoneにも当てはまることを強調します）は、すべての決定に浸透する厳密な優先順位付けに基づいています 。



優先順位付け（または優先順位付け）は、ターゲットオーディエンスの定義から始まり、たとえば、実装された機能とそのユーザーへの可用性の計画で終わる、ほぼすべてのステップに関係します。



この記事では、優先順位を付けなければならない5つの重要な段階を検討します。



  1. 対象者の定義
  2. アプリケーション目標ステートメント
  3. 主なシナリオの選択
  4. ナビゲーション計画
  5. 機能開発




1.ユーザーを知る



すべてはユーザーで始まり、ユーザーで終わります。特定のユーザー向けにアプリケーションを作成し、特定のユーザーがアプリケーションを使用するかどうかを決定します。



設計の観点から、誰があなたのアプリケーションを使用するかを（合理的な確実性をもって）想像することが重要です。 考えるべき（そしてできれば答えを修正する）重要な質問は次のとおりです。



  • あなたのユーザーは誰ですか？
  • なぜ彼はあなたのアプリケーションが必要なのですか？
  • いつ、どこで、どのように使用しますか？

2.アプリケーションが他のアプリケーションより優れているのはなぜですか？



アプリケーションの作成者がわかったので、アプリケーションの一意性を明確に述べることが重要です。 数十万のアプリケーションが存在する市場では、常に競合他社が存在しますが、その中で際立っている必要があります。



また、ストアに配置されたアプリケーションは、間違いなく1つまたは別のカテゴリ（「エンターテイメント」または「健康とフィットネス」など）に分類されることを覚えておく価値があります。



したがって、正しく優先順位を付けるために、アプリケーションがそのカテゴリの他のアプリケーションよりも優れていること、つまり「最高の陳述」を明確に述べることが重要です。 「最高の声明」は、まさにユーザーがあなたに走り、愛とお金をもたらすこと、そしてそれをすべての友人に伝える理由です。



策定、作成、フレーム化：



「私のアプリケーションは、___が何のためにあるのかというカテゴリーで最高です」



または



「私のアプリケーションは、______カテゴリの他のアプリケーションよりも優れています。



1）明確な文章、2）言葉遣い、3）競合他社との違いを明確にする必要があります。 あなたの声明が現実とあなたの意図と完全に一致していることが重要です。



たとえば、「私のアプリケーションは、映画を見るための音楽とビデオのカテゴリーで最高です」と言うことは、あなたを助けることは何もしない冒とくであるか、実際には、例えば、配信と映画を見ています。 ユニークなコーデック、圧縮技術、アダプティブストリーミング、画像品質、またはユーザーを視聴に関与させるユニークなエクスペリエンス（拡張現実）があります。



または別の例：「私のアプリケーションは最大のオンライン映画館です」という文は、もちろん、あなたが他の競合他社に対して多種多様な映画の最大の（数での）コレクションを持っていることを意味しますが、それはユーザーとして私を保証するものではありませんあなたが利用できる最新かつ最も面白い映画の、または、もし私がアニメ好きなら、私は見るためにお金を払う準備ができていてもそこに私の好きな漫画を見つけるでしょう。



後者の場合、ちなみに、より焦点を絞ったアプリケーションだけで成功する可能性が高くなります-ターゲットオーディエンスから複雑なデザインまで、すべての結果が生じます。



差別化焦点の存在は非常に重要です。 あなたのアプリケーションはさらに多くのことを行うことができますが、それは最も重要なものの1つである必要があり、それを使用する理由と理由を教えてくれます。 このようなフォーカスとキーキャラクターがあれば、他のすべての決定を検証できます。特に、必要性を確認し、さまざまなシナリオと実装された機能に優先順位を付けます。



例 。 上記の発想に戻ると、私のアプリケーションの焦点は次のようになります。「エンターテイメントカテゴリで最高のアプリケーションは、友人と映画鑑賞を計画および整理することです」。



このアプリケーションは、私が見たい面白い映画を選択し、都合の良い時間と便利な場所で友人と視聴を調整し、実際にその場で視聴を整理するのに役立ちます（映画のチケットを購入するか、自宅で放送します）。
