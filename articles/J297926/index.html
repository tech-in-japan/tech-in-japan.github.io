タスクマネージャーが生産性を向上させない理由

こんにちは 過去1年間を総括すると、何かが満たされていないことに気付いたかもしれません。 私も同じです。 したがって、私は生産的に仕事をすることを妨げる理由を徹底的に扱うようにタスクを設定しました。



まず、さまざまなタスクマネージャーの欠点を分析し、独自のアプリケーションを作成することにしました。 しかし、別のタスクマネージャーではありません。そのようなアプリケーションは既にたくさんあります。 新しいプログラムは、仕事を適切に計画するのに役立ち、多くのタスクにdrれることはありません。



このアプリケーションはどのように機能しますか？



ソースデータ



個人が問題を計画できるようにするタスクマネージャーを検討します。 チームのみに焦点を当てたマネージャーは考慮されません。



事業計画のプログラムは、2つの大きなグループに分けることができます。タスクのリストを操作することを目的とするものもあれば、カレンダーとその日の計画を使用するものもあります。 最初のグループにはTodoist、Wunderlist、およびTickTickが含まれ、2番目のグループには日の出カレンダー、Googleカレンダー、Yandexカレンダーが含まれます。 一部のプログラムでは、Essential PIM、TimeMaster、LeaderTaskなど、カレンダーとタスクのリストを操作できます。



明確にするために、ユーザーは個人の時間管理システムの作成に関与していないか、これを試みたが、あまり成功していないと想定しています。



1つのアプリケーション-ワンタイム管理システム



1番目と2番目のグループの多くのプログラムの説明では、さまざまな時間管理システムを使用して物事を計画できることが言及されています。 人が自分のシステムを持っているか、完成したものを自分用に研究し、適応させたことは暗黙的に暗示されています。 しかし、これを行うことは、既成のプログラムを使用するよりもはるかに困難です。 これを行うには、1冊以上の本を読んで、それらから主なものを強調し、自分ですべてを試してみる必要があります。 これに時間がありますか？ これには時間がありませんでした。



私のアプリケーションは、ユーザーに時間管理手法を独自に研究することを提供すべきではないと考えています。 私は独自に時間管理へのさまざまなアプローチを分析し、テストし、アプリケーションで機能するものを実装します。 ユーザーは、時間と問題を適切に計画する方法をユーザー自身に伝えるプログラムを入手する必要があります。



ユーザーは、プログラムを正しく使用する方法を知るために、時間管理システムの学習に時間を費やすべきではありません。 プログラム自体は、物事を正しく計画する方法を促すはずです。



通常のタスクリストの欠点



WunderlistやTodoistなど、最初のグループのタスクマネージャーが生産性を向上させない理由を考えてください。 これらのプログラムを使用すると、タスクのリストを作成し、タスクの優先度を示し、リマインダーとコメントを追加できます。 同時に、タスクを作成するには、その名前を書き留めておけば十分です。 私たちの脳は、簡単な方法を模索し、行動を実行する努力を最小限に抑えるように設計されています。 ユーザーが事前にタスクを処理するためのシステムを構築していないか、そのようなシステムを使用しているが、それに準拠していない場合、プログラムには優先順位や期限などが設定されていない多くのタスクが含まれます。 その結果、人は忙しくなりますが、生産的ではありません。 主で最も重要なタスクは完了しないか、緊急時に実行する必要があります。



タスクリストのもう1つの欠点は、タスクリストが常に成長しており、タスクを管理するよりも高速であることです。 これは、タスクの追加が簡単である一方で、タスクの分析にはある程度の努力が必要であり、脳がそれに抵抗できるためです。 最終的に、タスクのリストは非常に長くなるため、タスクを操作したいという欲求はなくなります。 これは何度もありました。



私の意見では、本当の問題はより深いです。時々、忙しい人は生産的であるように思われます。 「彼は常に忙しいですが、私は座って何もしていません」-私たちは、タスクマネージャーを取り、タスクを書き留めて、すぐにそれらを実行し始めます。 しかし、生産的であることは、一生懸命働くことを意味するのではなく、目標を達成することを意味します。 しかし、いくつかの目標を達成するには、一生懸命働く必要があります。



生産的であること=目標を達成する。



インターネットには、目標を設定する理由を説明する多くの本や記事があります。 要するに、目標がなければ、私たちは事業計画において「近視眼的」になります。多くのことをしているように見えますが、実際には時間をマークしています。 したがって、プログラムがタスクのリストのみで機能するだけでは不十分です。目標を達成する機能も必要です。



事業計画プログラムは、目標を達成できる必要があります。



プログラムは目標とどのように連携する必要がありますか？



プログラムが目標をどのように機能するかを正確に明確にしましょう。 目標のリストのみがある場合、これはタスクの通常のリストに相当します。 あなたはもっと簡単で似たような目標に取り組みたいと思うでしょう。



すべての既存の目標と新しい目標は、3〜5年後にあなたの人生がどのように見えるかのビジョンに従属する必要があります。 そうでなければ、異なる目標が異なる方向にあなたを引き寄せます。



目標は、3〜5年後に人生がどのように見えるかというビジョンに従属するものでなければなりません。



したがって、プログラムにはビジョンや戦略目標を記録するセクションが必要です。 この場合、このビジョンの実装の推定日を設定する必要があります。



残念ながら、ビジョンを記録するだけでは十分ではありません。 あなたが時々彼に連絡しなければ、それは忘れられます（あなた自身でテストされます）。 したがって、このビジョンを読み直すために、少なくとも週に1回リマインダーをトリガーする必要があります。



通常、多くの目標があり、何らかの方法でそれらをグループ化する必要があります。 目標は、時間と人生の領域によってグループ化できます。 私は人生の領域から始めます-これはより重要であるように思えます。



仕事、家族、友人との関係など、人生のすべての分野でうまくやっていると、人は自分の人生をコントロールしていると感じます。人生の分野の不均衡は、長い間問題につながります。 たとえば、仕事に多くの時間を費やすと、家族の問題が始まります。 したがって、プログラムは生活の領域を監視し、過度の不均衡を回避するのに役立つはずです。



このプログラムは、生活の領域を監視し、長期にわたる過度の不均衡を警告するのに役立ちます。



時間によって、目標は次のように分類できます。

  1. 長期（1年以上）、
  2. 中期（3か月から1年）、
  3. 短期（2週間から3か月）、
  4. 現在（1日から2週間）。


現在の目標は、短期、中期、中期-長期、長期-3〜5年間のビジョンまたは戦略目標に従属する必要があります。 はい、あなたは時間をかけてこれらの目標を書き留める必要があります-これはあなたの人生とタスクの長いリストを制御する唯一の方法です。 理想的には、目標を紙に設定するのではなく、目標に従うことを思い出させるプログラムに設定する必要があります。



プログラムは、目標の区分を時間内に把握し、定期的に目標を達成し、実装を追跡することを提案する必要があります。



目標を設定するための特定のルールがあります。 SMARTテクニックを知っています。 ウィキペディアに詳細に書かれています。 ここで非常に簡単に説明します。

  1. S-特定。 「私はより良く生きたい」ではなく、「モスクワでアパートを買いたい」。
  2. Mは測定可能です。 「アパートが欲しい」だけでなく、「モスクワの特定の地区にある公園近くの3部屋のアパート、エレベーターで5階の地下鉄から徒歩5〜10分」。
  3. A-達成可能。 アパートを買うのは本当ですが、モスクワの半分はありそうにありません。
  4. R-関連。 アパートを購入することで問題を正確に解決できますか？
  5. T-時間制限（時間制限あり）。 どのくらい目標を達成したいですか？




最後に、目標はあなたの個人的であるべきであり、たとえ他人が善意で行動したとしても、他の人によって課されるべきではありません。 あなたの人生...



プログラムは、各目標のさまざまな基準への準拠を制御できる必要があります。



目標について多くのことを話しました。 小計をまとめましょう。 多くのタスクがある場合は、時間と人生の領域に目標を設定し、それらに従う必要があります。 重要なことは、タスクが少なくならないことです。何を処理するのか、何を待つのか、単純に拒否できるものを確認するだけです。



目標を設定しても、タスクの数は減りません。 あなたはそれらをコントロールできます。まず何をする必要があるか、何を延期できるか、何を放棄できるかを確認します。



特定の日の計画の機能



そのため、目標があり、その日のTo Doリストを作成しました。 次のようになります。

  1. タスク1
  2. タスク2
  3. 問題3（など）


このリストの何が問題になっていますか？ 各タスクに費やす時間の見積もりはありません。 タスクの時間を見積もると、たいていの時間を自分が望むよりも短くすることができます。



タスクの正しいリストには、各タスクの推定時間が含まれています。



実際、その日の計画を取得しました。これは、サンライズカレンダー、Google、Yandexカレンダーなど、2番目のグループのプランナーに実装されています。 これらのスケジューラは通常、タスクの名前とその実行時間を指定できるという事実に制限されています。 最初のグループのタスクマネージャーの場合のように、これは、人が単純で興味深いタスクで時間を埋め、重要なタスクをバックグラウンドにプッシュするという事実につながる可能性があります。 その日の計画自体に関しては、私の経験では2つのことがその使用を妨げています。

  1. その日の計画に従うことはめったに成功せず、
  2. タスクを扱う際に柔軟性が失われていると感じることがあります。




人生は時々予期せぬ動きをするため、計画外のタスクや予期せぬことが起こります。 どういうわけか彼らに反応し、何かをする必要があるので、その日の計画から離れる必要があります。 ここから、その日の計画で作業するときの柔軟性の喪失感が生じます。



ただし、その日の計画から一歩下がっても悲劇はありません。 いつでも何をしたいのかを知るためには、その日の計画が必要です。 これにより、規律が向上し、重要かつ緊急の問題を遅らせる可能性のあるリスクが軽減されます。 その日の計画から離れなければならなかった場合、できるだけ早くそれに戻って、残りの時間を再計画する必要があります。



デイプランを柔軟にする方法



私は一日の計画を柔軟にするいくつかのルールを考え出しました。



まず、どのタスクを実行する必要があるかという質問に答えて、残りのタスクがそれほど重要または不要にならないようにする必要があります。 そして、この仕事は一日中、できれば朝に捧げるべきです。



毎日、最も重要なタスクを1つだけ選択し、メインの時間を割り当てます。



その日の計画で作業するとき、すべてのものは4つの部分に分割されます。



  1. 人生のあらゆる分野で目標を達成するために直接行われる業務。
  2. 間接的に目標を達成するのに役立つケース（たとえば、プログラマーの専門レベルを向上させる）;
  3. 対処せざるを得ないが、目標を達成していない問題（食事、飲酒、睡眠が必要です。突然の故障が発生する場合があり、個別に修正する必要があります）。
  4. 目標の達成か​​らあなたを遠ざけるもの（たとえば、プロジェクトに取り組む代わりに、インターネットでテレビやエンターテイメントサイトを見ること）。


問題の値を1〜4の数値と呼びます。これは、このタスクが上記のどの部分に関連するかを意味します。 したがって、1日は3〜4つの部分に分割され、各時間を割り当てる必要があります。 そして、プログラムは、タスクの値を示すことを許可する必要があります。 1日の大半が価値のあるタスク1と2に費やされる場合、目標の実現に徐々に近づいていきます。



1日のほとんどは、タスク1と2の値に費やす必要があります。



デイプランの柔軟性を高めるには、いくつかのトリックを使用することも役立ちます。 1つ目は、特定のタスクの代わりに、作業の目標またはプロジェクトをその日の計画に書き留めることができることです。 この時が来たら、この目標のタスクを開き、その実装に取り​​組みます。 次の記事「Target Management Command Post」では、1つのプログラムで特定の目標に関する作業を整理する方法を説明します。



2番目のトリックは、時間内に1つのタスクを別のタスクの近くに設定しないことです。 私たちはロボットではなく、すぐに切り替えることはできません。 10〜30分の短い休憩をタスク間に入れます。 この場合、予定より5〜10分遅れてタスクを完了することは恐ろしいことではありません。ほとんど問題はありません。



個人の生産性評価



生産的になりたいので、フィードバックなしではできません。 これを行うには、完了したタスクをチェックするだけでは十分ではありません。事態の全体像を把握する必要があります。 生産性はどのくらいですか？ 弱点はどこですか？ Todoistのカルマを除いて、自分の生産性を測定するソリューションを見たことはありません。 TodoistのKarmaは、タスクを完了するためのポイントです。 これらのポイントは、お金と交換することはできません。割引や無料の特典のために貯めることはできません。 期限切れのタスクの場合、ポイントが差し引かれます。 カルマが高いほど、より多くのタスクが完了します。 しかし、上記では、完了したタスクの数が人間の生産性の指標ではないことを明らかにしました。



次に、個人の生産性を評価する方法は？ 私の意見では、主な指標は、計画されたすべてが計画通りに進んでいるという事柄に対する内部統制の感覚です。 最終的に、人は幸せになり、自分自身を実現したいと考えています。 したがって、たとえば、完了したタスクや目標の数など、魂のない数を追いかけるのは危険です。



これは独自のニュアンスを持つ大きなトピックであるため、フィードバックのトピックについては後で詳しく説明します。



通常のタスクマネージャーの利点について少し



リストの使用は、目標が設定され、その日の計画があるときに発生します。 たとえば、その日の計画は「記事Xを書く」タスクであり、2時間の時間が割り当てられます。 タスクリストでは、記事を書くためのより小さな特定のタスクを書き留めます。これの概要を作成し、これとそれを比較し、そのような瞬間を出力に反映します。 その過程で、新しいタスクが表示されます。 彼らは心に留めておくべきではありません-これには普通のタスクマネージャーがいます。
