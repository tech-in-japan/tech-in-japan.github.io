FlashプラグインはGoogle Chromeで常にクラッシュしますか？ 解決策があります！

前文。



少し前に悲しみが私を追い越した。 Flashコンテンツの集中度が高まり、Flashプラグインに不満を抱いているサイトやページにアクセスした後、Chromeは突然、徹底的かつ容赦なく落ち始めました。 Chromeを再起動しても保存されず、システムを完全に再起動する必要がありました。



私はChromeが大好きで、YouTubeやすべてのフラッシュやjsを含む動的なコンテンツに使用していることに注意してください（明らかな理由のため）。



怠azineな母親はこれを重要視しないように促しました。 しかし、私にとってのすべての再起動はイベント全体です。 数百ではないにしても、数十のタブが開いており、数十のプログラムが実行されています。 私にはそのような生き方があります。 したがって、再起動は非常に望ましくないイベントです。



忍耐が止まった瞬間が来たので、この問題に対処することにしました。 この話を終わらせてください。



ところで、ソリューションは非常にシンプルであると同時に、効果的であることが判明しました。



この解決策は、トップ10のGoogleのリクエスト「Google Chrome Shockwave Flashプラグインのクラッシュ」で見つかりました。



さらに、私の無料の簡潔な翻訳、本質のみ、文字通りの要約。



Chromeには、システムとは別に、Flashプレーヤーのインストールのコピーがあります。 この問題は、原則として、Chromeがプラグインで両方の設定を定義し、それらが互いに競合する場合に発生します。



これに該当するかどうかを確認するには、Chromeのアドレスバーに「about：plugins」（引用符なし）というテキストを入力する必要があります。 プラグインマネージャーが開きます。 インストールされたプラグインに関する情報（これは拡張機能とは異なります）は、圧縮形式で発行されます。



写真でわかるように、Flashプラグインはすでに2個です。 私の場合、3つもありましたが、1つはオフにされていたので、詳細には触れませんでした。



右上隅にある「[+] more」をクリックします。





行う必要があるのは、「追加」プラグインを無効にすることだけです。 この記事の著者は、組み込みのクロムを無効にすることを推奨しています。 その間、組み込みプラグインはChromeが更新されるたびに自動的に更新されます。 しかし、システム1は必要ありません。 つまり 組み込みプラグインを無効にすると、Chromeを使用したFlashプラグインの自動更新が失われます。 著者もこれについて警告しています。



ビルトインプラグインは... \ AppData \ Local \ Chrome \ Application \ ...フォルダーにあり、パスのこの部分だけで認識できます。



無効なプラグインは灰色の背景でマークされます。





その後、Chromeを再起動し、フラッシュの機能を確認する必要があります。 すべてが順調であれば、人生を楽しみ、非落下のフラッシュコンテンツをお楽しみください。



問題が解決しない場合は、外部プラグインをオフにし、ビルトインをオンに戻すなど、他の組み合わせを「試してみること」をお勧めします。 また、次のように、シークレットモードでFlashコンテンツのあるサイトにアクセスすることをお勧めします。 このモードでは、Flashプラグインへのトラックを越える可能性のあるすべての拡張機能が無効になります。



最後に、プラグインを使用した「ゲーム」で状況が解決しない場合は、Chrome自体を再インストールすることをお勧めします。



元の記事へのリンク： www.howtogeek.com/103292/how-to-fix-shockwave-flash-crashes-in-google-chrome



PS：私の場合、プラグインを切断してChromeを再起動するだけでは機能しませんでした。 再起動する必要がありました。 これまでのところ、飛行は正常です。



更新：突然ビデオクリップがクロームで再生されなくなった場合、ビデオの代わりに黒いウィンドウと永遠に回転するローダーがあり、ビデオウィンドウで右マウスボタンを押そうとすると、フラッシュではなくhtml5モードでビデオが再生されることがわかります。 YouTubeが実験中です（校正リンク： www.youtube.com/html5 ）。 実際、すべてがリンクに書かれています。 そこで、以下で、実験を中止することができます。 それはまさに私がやったことです。 ChromeでのYouTube動画の再生が再開されました。
