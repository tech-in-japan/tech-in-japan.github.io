シスコ 最初の問題。 2つのネットワークを接続します。

始めましょう。









シスコのルーターは何から始まりますか？ 右、コンソールケーブルから。

これは、機器に付属のRS-232およびRJ-45コネクタを備えた水色のワイヤです。 コンピューターの空きCOMポートに接続する必要があります。 これで、ハードウェアで何でもできます。 もちろん、コンソール接続用のパスワードが設定されていない限り（ハードウェアで他の人のパスワードを克服する方法については後で説明します）。



そのため、PuTTYなどの端末接続クライアントプログラムを開きます。 プログラムで、接続ポート（COM1）と接続速度（9600）を指定します。

何が見えますか？ 黒い画面、点滅するカーソルなど何もありませんか？ そのため、ほとんどの場合、ルーターの接続速度は異なって設定されています。 原則として、これは115200の速度である可能性があります。それが適合しない場合（そしてこれが希少な場合）、それを拾わなければなりません。 いずれの場合でも、接続が行われているCOMポートが示されているかどうかを確認する価値があります。



わかりました、整理しました！ すべてが正常であり、ルーターからの招待が表示されます。 デフォルトでは、次のようになります。Router>

これは、ユーザーモードになっていることを意味します。 このモードでは、使用できるコマンドはほとんどありません。 これらのコマンドはすべて、ルーターの動作を監視することのみを許可しますが、構成を変更する機会を与えません。 このモードから、たとえばPingコマンドまたはshow ip interfaceを実行できます 。



ルーターの動作設定（読み取り、設定）を変更するには、特権モードに入る必要があります。 特権モードはパスワードで保護できます。 特権モードに入るには、 enableコマンドを入力する必要があります。 その後、コマンドプロンプトはルーター＃に変わります。



ここでは、さらに多くのチームがすでに利用可能です。 このモードでは、動作中の設定を変更し、変更した設定をROMに保存できます。



ただし、ルータの基本設定はグローバル設定モードからです。 configure terminalコマンドを実行すると、特権モードからアクセスできます。 プロンプトがRouter（config）＃に変わります。 既にお気づきのように、コマンドラインプロンプトで現在のモードがわかります。



ポイントに到達します。


はじめに、最も簡単な方法で、ルーターを使用して2つのネットワークを接続します。

内部ネットワークには192.168.10.1/24のアドレス範囲があり、そのルーターのアドレスは192.168.10.254で、ネットワークアダプターはFastEthernet0 / 0です

外部ネットワークのアドレス範囲は10.54.0.0/16で、ルーターアドレスは10.54.0.1 、ネットワークアダプターはFastEthernet0 / 1です。

グローバルコンフィギュレーションモードで、コマンドInterface FastEthernet0 / 0を入力します 。 プロンプトは、 Router（config-if）＃になります。 デフォルトのインターフェースにはアドレスがなく、オフになっています。 最初にIPアドレスを入力します。 これは、コマンドip address 192.168.10.254 255.255.255.0で実行されます 。

インターフェイスがオフになっていることを覚えていますか？ no shutdownコマンドによってオンになります。 すべてが順調であれば、碑文が実行されます。



Router(config-if)#no shutdown



%LINK-5-CHANGED: Interface FastEthernet0/0, changed state to up

%LINEPROTO-5-UPDOWN: Line protocol on Interface FastEthernet0/0, changed state to up







最初の行は、物理層およびデータリンク層の点でネットワークインターフェイスがすべて正常であることを示しています（ネットワークケーブルが接続され、互換性のある機器がもう一方の端で動作しています）。 2行目は、ネットワーク層（IP層）も正常に機能することを示しています。



次に、 FastEthernet0 / 0インターフェイスコンフィギュレーションモードを終了し、 FastEthernet0 / 1インターフェイスに入り、IP設定を構成する必要があります。

これは自分で処理できます。



特権モードに戻り（ exitコマンド ）、 show ip interface briefコマンドを発行することにより、すべてが正しく設定されていることを確認できます 。 ネットワークインターフェイスのステータス情報が表示されます。 コマンドからの出力は次のようになります。



Router#show ip interface brief

Interface IP-Address OK? Method Status Protocol



FastEthernet0/0 192.168.10.254 YES manual up up



FastEthernet0/1 10.54.0.1 YES manual up up







やった！


できた！ ルーターは、ネットワーク間でパケットを転送できます。

ほぼ完了=）行ったすべての変更と設定は、ルーターのRAMにのみ保存されます。 再起動後も構成を保持するには、ROMにコピーする必要があります。 これは、梨を砲撃するのと同じくらい簡単です。 特権モードから、コマンドcopy running-config startup-configが入力されます。 今、再起動は怖くない！



はい、もう1つ小さな詳細です。 構成のないルーターをオンにすると、IOSはウィザードを使用してルーターの基本パラメーターを構成することを提案します。 彼を丁寧に拒否します。 難しいのになぜ簡単なのですか？ =）



合計：


作業には、4つの基本構成モードが必要です。



ユーザーモード： ルーター>

特権モード： ルーター＃

グローバルコンフィギュレーションモード： ルーター（config）＃

オブジェクトの構成モード（インターフェイス、ルーティングプロトコルなど）： ルーター（config-if）＃



以上です。 同意する、あなたはそれが難しいだろうと思った！
