Ubuntuの省電力テスト

挑戦する



一部のオフィスは、 Ubuntuの省エネシステムの効率を測定するために着手しました。 テストマシンとの互換性の問題が原因で削除された、最初のリリースである4.10を除く7.10と以前のリリースの両方が対象でした。



やる気



2.6.22カーネルを搭載する新しいリリース7.10では、設定（ CONFIG_NO_HZ ）が登場しました。 この設定により、「タイマー割り込みオンデマンド」モードを有効にできます。これにより、システムは長時間アイドル状態になり、不必要な（不要な）割り込みがなくなります。 これにより、モバイルデバイスのバッテリー寿命が長くなります。



試験装置



テストでは、次の構成のLenovo ThinkPad R52ラップトップを使用しました。

Intel Pentium M 750（拡張Intel SpeedStepテクノロジー搭載1.86GHz）、2GBのDDR2-533、ATI Radeon X300 64MB、15.4 "1400 x 1050、Intel 802.11g WiFi、および80GB IDE 。



試験構成



パラメータは、主電源からの動作中およびバッテリ動作中に測定されました。 また、ロードされたアイドルシステムで測定が実行されました。

ネットワークから作業する場合、 SeaSonic PowerAngelを使用して電力消費を測定しました。 アイドルモードでの測定中に、 GNOMEがデフォルト設定で起動され、スクリーンセーバーが無効になりました。 負荷テストは、追加で起動されたglxgearsとCPU Burn-Inで構成されていました 。 測定は開始から15分後に行われました。



リフレッシュする



Ubuntuのリリースの構成の詳細：

  • Ubuntu 5.04 （2.6.10カーネル、X.Org 6.8.2）
  • Ubuntu 5.10 （2.6.12-9カーネル、X.Org 6.8.2）
  • Ubuntu 6.06.1 （2.6.15-26カーネル、X.Org 7.0.0）
  • Ubuntu 6.10 （2.6.17-10カーネル、X.Org 7.1）
  • Ubuntu 7.04 （2.6.20-15カーネル、X.Org 7.2.0）
  • Ubuntu 7.10 （2.6.22-12カーネル、X.Org 7.2 / Xサーバー1.3）
