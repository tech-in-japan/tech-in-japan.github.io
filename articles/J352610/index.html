Virtual Boxでホットスポットをしたように

こんにちは、Habrahabrの読者の皆さん。 今日は、ラウンドアバウト交差点（VirtualBoxの形式）でホットスポットをどのように行ったかを説明します。



理論のビット



ホットスポット-Wi-Fiアダプターを備えたデバイスがWi-Fiポイントを作成できるようにするテクノロジー。









Oracle Virtual Box-有名なオペレーティングシステムの特性を備えた仮想マシンを作成できるプログラム。









背景



コンピュータークラスで1ペアに飽きてしまったときに、自宅からUSB Wi-FiアダプターをドラッグしてWi-Fiを作成できる（オフィスにはない）と思いました。 当然、私は管理者の下に座っていないことを忘れていました。数週間後に来て、私は苦い失望に気づきました。 しかし、私はあきらめず、次に何をすべきかを考え始めました。



三方



私は自分が興味のあるビジネスに落ちた場合、問題を解決するか、現時点で問題が解決できないことを100％確信できるまで解決します。 次の3つのオプションがありました。



  • これらすべてを試して、彼らが私たちに言っていることをさらに聞いてください。 上記のため、このオプションはパスしませんでした。
  • 管理者パスワードをリセットし、すべての汚いことをすばやく実行してみてください。 このオプションは、そのようなアクションがあまりにも多くの注意を引き付けて乗ることができるという事実のために適合しませんでした-最初の数字。
  • Virtual Boxを起動し、Win7 / 8/10イメージと既にその中にあるシャーマンをダウンロードします。 これが唯一のオプションであり、この方法を使用して問題を解決し始めました。


本当のヒーローは簡単な方法を探していません



VirtualBoxでは、次のオペレーティングシステムを搭載した複数のマシンが利用できました。

-Ubuntu;

-WinXP;

-Win7。



私が最初に選んだものを推測しますか？ WinXP。 私が知っていただけなら...



まず、2つの（！）ネットワークをセットアップします。 1つは有線インターネット経由、もう1つはWi-Fiアダプター経由です（これはすでに最初のエラーでした）。 仮想マシンにドライバーをインストールし、ワイヤレスネットワークをインストールするためのウィザードを起動し、名前とパスワードを設定します。他のデバイスにネットワークを手動でインストールする方法を選択すると、エラーが発生します。





「コンピューターで実行されているワイヤレスネットワークのソフトウェアによっては、ワイヤレスネットワークウィザードが機能しない場合があります。 詳細については、マイクロソフトサポート技術情報871122を参照してください。
これは私を怒らせ、私は古いXPに唾を吐き、Ubuntuに行きました。そこで彼は実際に私の幸福も見つけられませんでした。彼はアダプターさえも見なかったからです。



Windows XPは死んだ、長生きするWindows 7！



ホットスポットを入手する最後のチャンスはWindows 7でした。その時までに、ネットワークとしてではなくUSBとしてアダプターを接続する必要があることに気づいたので、XPで最初に確認し（状況は変わりません）、7つの拡張を征服しました。 Windows 7は私のWi-Fiアダプターを見ました（ところで、これはAsus WL-167G V3です）。 私は静かにドライバーをダウンロードし、それらのインストールを開始し、それから再び失敗しました！ コード10！







それから私はここで私がキャッチすることは明らかに何もないことに気づきました、そして私は家に帰って家でこれをすべて確認する必要があります。 自宅では、USB 1.0ではなく、2番目のバージョン（拡張パックのプリインストール）を介して起動することを決定しました。 コマンドラインを起動して、管理者からコマンドラインを起動し、最初のコマンドを入力しました。 



netsh wlan set hostednetwork mode=allow ssid="user” key="password" keyUsage=persistent


ここで、ssid = "user"はネットワークの名前、key = "password"はパスワード、mode = allowはネットワーク割り当てモードのアクセス許可、keyUsage = persistentはパスワードを保存することを決定し、起動するたびに指定する必要はありません仮想ネットワーク。



ポイントを立ち上げます：



netsh wlan start hostednetwork







次に、ネットワーク管理センターに移動し、インターネットを受信するネットワークアダプターの設定に移動します。 そこで、「アクセス」タブに移動し、「他のユーザーにアクセスを提供する...」にチェックマークを付けます。 それだけです。Wi-FiポイントはVirtual Boxを介して正常に機能し、無料のWi-Fiを提供できます。



ポイントを止めるには、コマンドラインに次のように記述します。 



 netsh wlan stop hostednetwork


ご覧のとおり、必要なものを取得するための回避策をいつでも見つけることができます。



それでは、システム全体をバイパスするためにどのような手順を実行する必要がありますか？



  1. Win7 / 8/10、Virtual BoxおよびVirtual Box Extension Packのイメージをダウンロードします。 Virtual BoxとVirtual Box Extension packをインストールして、USBフラッシュドライブに転送します（どこで作業/学習/隣人からの権利を取得しますか？）。 車をカスタマイズします。 USBでバージョン2.0をインストールします。 ネットワーク上で、ネットワークブリッジを指定し、インターネットを提供するアダプターを選択します。 VirtualBoxシステムにインストールします。
  2. USB Wi-Fiアダプターをコンピューターに接続します。 仮想マシンで、デバイス-> USB->（USB Wi-Fiアダプターの名前）タブに移動します。
  3. アダプタのfireを仮想マシンにインストールします。
  4. 仮想マシンをフラッシュドライブ/クラウドにドロップします
  5. 上で説明したように、仮想マシンを介して、他の誰かのコンピューターでホットスポットを構成します。
  6. 利益！
