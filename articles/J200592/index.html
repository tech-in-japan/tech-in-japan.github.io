OS X MavericksのWindowsプリンターに印刷する

この記事では、Windowsマシン（実機または仮想機）を使用して、サポートされていないMac OSプリンターに印刷する方法について説明します。 この方法はOS X 10.8では正常に機能しましたが、Mavericksのアップデートで停止しました-プリンターへの接続が単に消え、Macはプリンターに接続できないと報告しました。



まず、プリンターをOS Xに再インストールするだけで、システムに組み込まれているドライバーセットが切断され、プロキシプリンターとして使用されていたApple 16/600ドライバーがなくなっているのを確認しました。 一般的なPostScriptドライバーを使用してみましたが、役に立ちませんでした。 PostScriptをプロキシプリンターとしてサポートするいくつかのプリンターを列挙しても、結果は得られませんでした。 それから私はこれをしました。



まず、WindowsにLDPサーバーをインストールする必要があります。 Windows XPでは、Windowsコンポーネントのインストールの「ネットワーク上のその他のファイルおよびプリンター共有サービス」という見出しの下にあり、「UNIX用印刷サービス」と呼ばれます。





インストール後、TCP / IP Print Serverサービスを開始し、自動開始方法を設定する必要があります。





これで、LDP経由でプリンターに接続できます。 Mac OSでは、新しいプリンターを追加します。 接続タイプは「IP」、プロトコルは「LDP」、キューは上記の記事に従って構成したWindowsのプロキシプリンターのネットワーク名です。 ドライバーとして、「一般的なPostScriptプリンター」を選択します。





プリンターを追加した後、何かを印刷しようとしていますが、成功しました。
