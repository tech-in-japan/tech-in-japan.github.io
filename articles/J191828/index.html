Veeam Backup Free Edition v7-新しいバージョンのレビュー

1年前、 VMwareおよびHyper-V- Veeam Backup Free Editionで仮想マシン（VM）をバックアップおよび管理するための無料ツールを確認しました 。 最近、バージョン7.0がリリースされ、新機能と改善が加えられました。 この投稿では、私にとって最も興味深いと思われる機能を検討したいと思います。











最初に、3つの主要な機能に言及します。









  • VeeamZIP-たとえば、VMとファイルを別のホストに転送するために、実行中のVMをシャットダウンせずにバックアップコピーを作成できます。
  • バックアップからの復元-VM全体、ゲストOSファイル、または個々のアプリケーションオブジェクトを含む、多数の復旧シナリオをカバーします。
  • VMwareのクイックマイグレーション -VMware vMotionおよびvStorage Motionがなくても、クラスターおよび共有ストレージを使用せずに、最小限のダウンタイムでホストまたはストレージ間で実行中のVMを移行できます。




それでは、機能をさらに詳しく見ていきましょう。









1. VeeamZIP



VeeamZIPは、Veeam Backup Free Editionの中核をなすテクノロジーであり、動作中のVMをバックアップできます。 VeeamZIPを使用する場合、ユーザーは、アーカイブするVMと最終ファイルの保存場所のみを指定する必要があります。 実際、これがすべての設定です。 VMのバックアップコピーは圧縮および重複排除され、通常、ストレージシステムで占有するスペースのサイズが大幅に削減されます。













写真1. Veeam Backup Free Edition v7でのVeeamZIPの簡単な構成









VeeamZIPは、次の3つの場合のいずれかに役立ちます。









  • バックアップVMが必要です。 ほとんどの場合、これは、更新を適用する前など、VMに変更を加える前に行われます。 スナップショット上でのVMの長寿命は、その大きなサイズとまったく同じであるため、ほとんどのデータセンターで致命的な罪と見なされているため、VMスナップショットを使用できません。
  • VMのアーカイブコピーが必要です。 すべてが単純です-使用を停止する前に、重複排除および圧縮されたVMバックアップが作成されます。
  • VMをリモートホストまたはテストラボにコピーする必要があります。 VeeamZIPは、必要な場所に結果ファイルを送信する前に、VMの構成をカプセル化します。


バージョン7では、VeeamZIPの動作アルゴリズムが最適化されたため、VeeamZIPがはるかに高速に動作し始めました。1つのVM内に複数の仮想ディスクの並列処理が現れ、空のVMブロックなどを無視しました （詳細は下記）。









2.バックアップから復元する



Veeam Backup Freeは、単一のファイルの復元からVM全体の復元に至るまで、バックアップからデータを復元するためのいくつかの方法を提供します。 完全なリストには以下が含まれます。









  • VM全体の回復。 必要に応じて、VM全体を復元し、元のホストまたは新しいホストで実行できます。
  • VMディスクリカバリ（VMware用）。 この機能は、OS障害の場合のシステムディスクなど、別のVMディスクが破損した場合に役立ちます。 目的のVMディスクを復元して、元のVMまたは新しいVMに接続するだけです。
  • ゲストOSファイルの回復。 Windows OSを搭載したVMおよびFATやNTFSなどのファイルシステム（Veeam Backup FreeがWindows Server 2012にインストールされている場合、OSレベルでのReFSおよび重複排除のサポートを含む）の場所になります。 さらに、追加のウィザードと補助アプライアンスを使用して、別の14個のファイルシステムがサポートされます。 さらに、すべての場合において、Veeam BackupはVM全体を復元するのに十分なほどスマートであり、目的のファイルを抽出して接続するだけです。






写真2. Veeam Backup Free Edition v7でのゲストOSファイルの復元









  • VMファイル（.vmdk、.vmxなど-VMwareの場合、.vhd、.vhdx、.xmlなど-Hyper-Vの場合）の回復。 ゲストOSファイルの回復の場合と同様に、ここでも同様のアプローチが機能します。 必要に応じて、Veeam Backup FreeはVM全体を復元せず、必要なVMファイルのみを復元し、使用済みスペースを節約します。
  • 個々のMicrosoft ExchangeおよびSharePointオブジェクトの回復。 詳細については、以下で説明します。


3. VMwareのクイック移行



前述のように、クイック移行（「クイック移行」と翻訳）を使用すると、クラスターや共有ストレージを使用せずに、ホストやデータストア間で作業中のVMを移行できます。 クイック移行は、VMware vMotionおよびvStorage Motionが動作を拒否する、低速で不安定なデータチャネルでも正常に機能します。













写真3. Veeam Backup Free Edition v7のクイック移行を使用したVM移行









もちろん、v7の新しいバージョンは、既存の機能の更新だけでなく、新しい「トリック」も世界にもたらしました。 以下は、私が最も気に入った7つの製品です。









1.ネイティブテープサポート



磁気テープはしばしば「必要な悪」と見なされているという事実にもかかわらず、多くのデータセンターは、主にデータをアーカイブするためにそれらを積極的に使用し続けています。 磁気テープ上のデータの長期保存は安価で、単純にスケーラブルであり、全体としてディスクよりもはるかに信頼性が高いため、理解できます。









Veeam Backup Free v7では、開発者がユーザーに会いに行き、磁気テープのサポートを実装しました。 バックアップをテープドライブおよびテープライブラリ（物理または仮想）に保存し、そこにVM全体または個別のファイルを送信できるようになりました。 任意の復旧ポイントからデータを復旧できます。









ちなみに、Veeam Backup Freeは、Windows Server 2003以降、Windowsで提供されていないNTBackupユーティリティの優れた代替品です。VeeamBackup Freeは、NTBackupで記録されたテープのフォーマットもサポートします。













写真4. Veeam Backup Free Edition v7のネイティブテープサポート









2. VMware vCloud Directorのサポートの強化



バージョン7には、VMware vCloud Director（vCD）とのより深い統合が含まれています。 Veeam Backup Freeは、vCD VMに固有の追加のメタデータを含め、vCDによって管理されるVMのバックアップ、複製、および復元ができるようになりました。 同時に、vCD構造はVeeam Backup Freeコンソールに直接表示され（作業がより便利です）、すべての属性を持つVMおよびvAppをvCDインフラストラクチャ（元の、または別の）に直接復元することができます。









ちなみに、これまでのところ、VMware vCloud Directorに同様の機能レベルを無料で提供するVeeam Backup Freeの類似物は見つかっていません。













写真5. Veeam Backup Free Edition v7を使用したVMware vCloud Directorのバックアップ









3. Veeam Explorer for Microsoft SharePoint



Veeam Explorer for Microsoft SharePointは、Veeam Backup Freeに組み込まれている無料のツールで、次のことが可能です。









  • SharePointを使用してVMバックアップ内の個々のSharePointオブジェクトを検索します。
  • 見つかったオブジェクトを元の場所に復元します。
  • 見つかったオブジェクトをレターの添付ファイルとしてユーザーに転送します。


これはすべて、最初にバックアップからVM全体を3番目に復元することなく機能します（ストレージ内のスペースを節約します）。バックアップ中に特別なエージェントをインストールする必要はありません。









Veeam Explorer for SharePointの現在のバージョンはMicrosoft SharePoint 2010を完全にサポートしており、次の製品アップデートでSharePoint 2013のサポートが追加されます。













写真6. Veeam Backup Free Edition v7のVeeam Explorer for Microsoft SharePointを使用してMicrosoft SharePointからVMバックアップからファイルを復元する









4. Microsoft Exchange用Veeam Explorer



Veeam Explorer for Microsoft Exchangeは、Veeam Explorer for SharePointに似た、Microsoft Exchangeを備えた無料のVMバックアップツールです。 Veeam Explorer for Exchangeは、VMバックアップ内の個々のExchangeオブジェクト（メッセージ、メモ、タスクなど）を検索したり、電子メールを.msg形式で保存したり、見つかったオブジェクトを添付ファイルとして電子メールで送信したりできます。









Veeam Explorer for SharePointと同様に、Veeam Explorer for ExchangeはVM全体を事前に復元する必要がなく、VMwareとHyper-Vの両方で迅速に動作します。 新しいバージョンでは、Veeam Explorer for ExchangeはMicrosoft Exchange 2013をサポートしています。













写真7. Veeam Backup Free Edition v7のMicrosoft Exchange用Veeam Explorerを使用して、Microsoft ExchangeでVMバックアップからレターを復元する









次の3つの機能により、VeeamZIPの速度が向上します。









5.空のブロックを無視することでVeeamZIPを高速化



新しいバージョンでは、VeeamZIPはバックアッププロセス中にVMwareのVM内の空のブロックを無視するように「学習」し、それにより作業の効率を大幅に向上させました。









6. VM内の複数のディスクの並列処理によるVeeamZIPの高速な作業



バックアップ中、Veeam Backup Freeは1つのVM内の複数のディスクを同時に処理できます。 これにより、特に複数の大きなディスクを持つVMの場合、VeeamZIPの速度が向上します。 VeeamZIPはより高速に動作し、VMスナップショットの寿命を短縮します。これにより、サイズが小さいためにVMスナップショットを後で削除する時間が短縮され、ストレージデバイスの負荷と仮想環境に対するバックアッププロセスの影響が大幅に削減されます。









7.ハードウェア圧縮加速



バージョン7では、新しいデフォルトの圧縮タイプが導入されました。 現在、VeeamZIPはバックアップサーバー「スマート」で利用可能なプロセッサリソース（CPU）を使用し（SSE圧縮アルゴリズムの使用により）、圧縮中のバックアッププロキシの負荷を大幅に削減できます。 このため、場合によっては、サーバーの負荷を最大10倍まで削減できます。 以前のVeeamZIPがCPUに100％の負荷がかかったために単に「オーバークロック」できなかった場合、これはもはや問題ではなく、多くのユーザーがv7への切り替え後にバックアップ速度の大幅な加速を報告しています。









リストされたイノベーションは、Veeam Backup Free Edition v7の最新バージョンで最も興味深いと思われました。 更新プログラムの全リスト（有料版を含む）を表示したい場合は、 ここで （一時的に英語で）見つけることができます。
