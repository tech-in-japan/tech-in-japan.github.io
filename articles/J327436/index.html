棚にあるすべて：RamblerによるWeb分析/トップ100、パート1





パート1では、 Rambler / top-100サービスの再起動と、詳細な階層分析のための新しいツール「サイトセクション」について説明します。



2016年の秋に、Web分析サービスであるRunblerサイトの最も古いディレクトリであるRambler / top-100を再起動しました。 それ以来、お話ししたい機能をいくつか追加しました。 この春、トップ100はロシアのインターネット上のテーマ分析の旗の下で会合を開きました。



一般に、サービスの再起動にはいくつかのタスクがありました。 最初に、サポートの古い高価なコンポーネントを最新のより安定したコンポーネントに置き換えました。 第二に、見た目を一新する時が来ました-古いTop-100の温かいチューブデザインはゼロの始まりを与えました。 第三に、カウンターと分析インターフェイスの機能を修正し、新しい機能を追加する必要がありました。



トップ100の新バージョンの開発に着手し、アナリスト、マーケティング担当者、マネージャーなどの同僚に相談しました。 彼らは、サービスのターゲットオーディエンスとして、提案やコメントとともに、トップ100の有用性と向上を支援しました。 これらの職業の人々は私たちの最もアクティブで要求の厳しいユーザーです。そのため、 MVP （最小実行可能製品）の要件には、2つの重要なポイントが含まれていました：前バージョンのサービスのすべての機能を保持し、RAMBLER＆Co自身の分析タスクに必要なツールを追加します。



サイトセクションレポート



廃止されたバージョンを思い出さずに、第2部から始めます-サイトページの訪問者の行動に関するデータを収集および視覚化できる3つのTop-100分析機能について説明します：サイトセクション、ブロック分析、および訪問パラメーター。 具体的には、この記事では「サイトセクション」レポートについて説明します。



記事には、調査の問題を解決する例とカウンター設定の技術的な詳細、およびコード例があることをすぐに予約してください。 したがって、この記事は、ウェブマスターやフロントエンド開発者だけでなく、マネージャーとマーケティング担当者の両方にとっても興味深いものになると期待しています。 すべての技術的な詳細を知りたい人のために、Top-100カウンターのセットアップに関する公開ドキュメントへのリンクを共有します。 これで十分でない場合は、 トップ100サポートサービスにご連絡ください。カウンターの機能とインターフェースの操作に関するご質問にお答えします。



効果的に-どのくらい？



最初に、サイトが直面している問題と、所有者がトップ100行動レポートまたは同様のツールを使用して解決できるタスクを検討します。 オンラインストアの分析について多くの記事が書かれているので、変更のために、学童と保護者向けのコンテンツプロジェクトの例を考えてみましょう。



大幅に簡素化された場合、教室プロジェクトに加えて、そのようなプロジェクトの主要な製品パフォーマンスインジケータは、表示の深さとサイトで費やされた時間になります。 訪問者がサイトのページに滞在する時間が長くなり、見るページが増えるほど、広告が多く表示され、サイトが獲得するお金が増えます。 スキームはシンプルで明白です。



私たちのサイトの所有者は、最大の利益を得たいと考えており、彼はサイトでのユーザーの行動に関する次の質問を心配しています。

まず、レポートのデータ収集を構成しましょう。 これを行うのは非常に簡単です。トップ100カウンターの初期化コードで、「chapters」パラメーターを追加し、特定のページのチェーン値をそれに渡すだけです。 将来レポートを読みやすくするために、「章」の値をキリル文字で指定できます。



たとえば、「入学/大学の選択/モスクワ州立大学」ページの場合、カウンターコードは次のようになります。 



<!-- Top100 (Kraken) Counter --> // ... var options = { project: PROJECT_ID, chapters: ['', ' ', '  '] }; // ... <!-- END Top100 (Kraken) Counter -->


そして、「中学・高校/ベビーベッドとレシェブニックス/化学、グレード8」のページは次のようになります。 



 <!-- Top100 (Kraken) Counter --> // ... var options = { project: PROJECT_ID, chapters: ['   ', '  ', ', 8 '] }; // ... <!-- END Top100 (Kraken) Counter -->

したがって、サイトに同時に複数の論理構造がある場合、複数の階層を指定できます。レポートには複数のツリーがあります。 それらはいくつあってもかまいませんが、それぞれの深さは3レベルに制限されています。 レポートの最後の（4番目の）レベルは、常にマークアップされたページのURLです。 たとえば、サイトのモバイルバージョンがサブドメインにある場合、デスクトップバージョンとモバイルバージョンのページ間の分布が表示されます。



セクションと見出し自体のページをレイアウトすることを忘れないことが重要です。レポートのこれらのレベルの値は、見出しと見開きページ内のすべてのページを考慮します。



さて、質問に戻って、以前の回答を得たいと思いました。 現在最も人気のあるセクションはどれですか？ サイトのどのコンテンツが訪問者をより惹きつけ、サイトに長く滞在させますか？ どの記事が訪問者にとって最も面白くないですか？ サンプルレポートでわかるように、選択した期間で最も人気のあるのは「入場」セクションであり、「指導者とコース」という見出しが最も訪問されています。 「大学の選択」という見出しの表示深度は著しく高く、「チューターとコース」という見出しのページは直帰率が高いことがわかります。ほとんどの訪問者は訪問ごとに1ページを読んでサイトを離れます。 これは、チューターを探している人、必要なコンテンツをすぐに見つけている人、またはその逆の人が失望することを意味します。



サイト所有者にとって、この情報がコンテンツとナビゲーションの変更の理由である可能性があります。 たとえば、「大学の選択」というルーブリックのページを促進するための推奨事項や、表示深度と平均時間の良い他の関連するルーブリックを含むブロックを追加できます。 これにより、このようなコンテンツは、個別指導およびコースセクションの訪問者にとって興味深いものであり、サイトに長く滞在し、より多くのページを見て、再びサイトに戻る可能性が高いという仮説をテストします。 最も人気のあるページにさらに広告ユニットを追加できます。 直帰率が最も高いページと見出しを再設計するか、必要性をすべて確認する必要があります。
