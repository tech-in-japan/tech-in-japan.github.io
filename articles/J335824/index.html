Djangoの紹介



最近のWebサイトは、実際には多機能アプリケーションであり、デスクトップシステム用の本格的なアプリケーションにかなり近いものです。 Pythonには、Webアプリケーションを構築するための豊富なツールセットが含まれています。 この章では、 Djangoを使用して、特定のトピックに関して受け取った情報を追跡するオンラインジャーナルシステムであるLearning Logプロジェクトを構築する方法を学習します。



このプロジェクトの仕様を作成し、アプリケーションが使用するデータのモデルを定義します。 Django管理システムを使用していくつかの初期データを入力し、Djangoがサイトのページを作成する際に基づいてプレゼンテーションとテンプレートを作成する方法を学習します。



DjangoはWebインフラストラクチャであり、インタラクティブなWebサイトを構築するためのツールのセットです。 Djangoはページ要求に応答でき、データベース、ユーザー管理、およびその他の多くの操作に対する情報の読み取りと書き込みを簡素化します。 第19章と第20章では、学習ログプロジェクトを改良し、サーバーに展開して、ユーザー（および友人）が使用できるようにします。



プロジェクト作成の準備

プロジェクトの作業の開始時に、仕様でプロジェクトを記述する必要があります。 次に、プロジェクトを構築するための仮想環境を作成します。



仕様書

完全な仕様では、プロジェクトの目的、機能、外観およびユーザーインターフェイスについて説明しています。 優れたプロジェクトや事業計画と同様に、仕様は最も重要な側面に焦点を当て、プロジェクトの円滑な開発を保証する必要があります。 ここでは、完全な仕様を記述しませんが、開発プロセスの方向を設定するいくつかの明確な目標を策定します。 仕様は次のようになります。



Learning Logと呼ばれるWebアプリケーションを作成します。これを使用して、ユーザーは関心のあるトピックのジャーナルを保持し、各トピックを学習しながらジャーナルにエントリを作成できます。 学習ログのホームページには、サイトの説明が含まれており、ユーザーに資格情報の登録または入力を促します。 ログインに成功すると、ユーザーは新しいトピックを作成したり、新しいエントリを追加したり、既存のエントリを読んだり編集したりできます。



新しい資料を勉強するとき、学んだことをジャーナルに保存しておくと便利です。記録は、必要な情報を監視して返すのに役立ちます。 適切なアプリケーションを使用すると、このプロセスの効率が向上します。



仮想環境を作成する

Djangoを使用するには、まず作業用の仮想環境を作成する必要があります。 仮想環境とは、他のすべてのPythonパッケージから分離してパッケージをインストールできるシステムのサブセクションです。 あるプロジェクトのライブラリを他のプロジェクトから分離することは、第20章でサーバーに学習ログを展開することで恩恵を受けます。



learning_logという名前でプロジェクトの新しいディレクトリを作成し、ターミナルモードでこのディレクトリに移動して、仮想環境を作成します。 Python 3を使用している場合、次のコマンドを使用して仮想環境を作成できます。 



learning_log$ python -m venv ll_env learning_log$


このコマンドはvenvモジュールを起動し、それを使用してll_envという仮想環境を作成します。 この方法が機能した場合は、「仮想環境の有効化」セクションに進んでください。 382.何かが機能しない場合は、次のセクション「virtualenvのインストール」をお読みください。



virtualenvをインストールする

Pythonの以前のバージョンを使用している場合、またはシステムがvenvモジュールを正しく使用するように構成されていない場合は、virtualenvパッケージをインストールします。 virtualenvのインストールは、次のコマンドで行います。 



 $ pip install --user virtualenv


このコマンドを少し変更したバージョンを使用する必要がある場合があります。 （まだpipを使用していない場合は、「pipを使用したPythonパッケージのインストール」セクション（227ページ）を参照してください。）



注

Linuxを使用しているが、この方法が機能しない場合は、システムのパッケージマネージャーを使用してvirtualenvをインストールします。 たとえば、Ubuntuでは、sudo apt-get install python-virtualenvコマンドを使用してvirtualenvをインストールします。



端末ウィンドウのlearning_logディレクトリに移動し、次のコマンドで仮想環境を作成します。 



 learning_log$ virtualenv ll_env New python executable in ll_env/bin/python Installing setuptools, pip...done. learning_log$


システムに複数のバージョンのPythonがインストールされている場合、virtualenvが使用するバージョンを指定します。 たとえば、virtualenv ll_envコマンド--python = python3は、Python 3を使用する仮想環境を作成します。



仮想化アクティベーション

仮想環境を作成したら、次のコマンドでアクティブ化する必要があります。 



  learning_log$ source ll_env/bin/activate (1) (ll_env)learning_log$


このコマンドは、ll_env / binディレクトリーからアクティブ化スクリプトを実行します。 環境がアクティブになると、その名前が括弧（1）で表示されます。 これで、環境にパッケージをインストールし、以前にインストールしたパッケージを使用できます。 ll_envにインストールされたパッケージは、環境がアクティブな間のみ利用できます。



注

Windowsシステムで作業している場合は、ll_env \ Scripts \ activateコマンド（sourceという単語なし）を使用して仮想環境をアクティブにします。

仮想環境の使用を停止するには、deactivateコマンドを入力します。 



 (ll_env)learning_log$ deactivate learning_log$


環境は、それが機能するターミナルウィンドウを閉じると非アクティブになります。



Djangoのインストール

仮想環境を作成してアクティブにしたら、Djangoをインストールします。 



 (ll_env)learning_log$ pip install Django Installing collected packages: Django Successfully installed Django Cleaning up... (ll_env)learning_log$


仮想環境で作業しているため、このコマンドはすべてのシステムで同じように見えます。 --userフラグを使用する必要も、python -m pip install package_nameのような長いコマンドを使用する必要もありません。



Djangoで作業できるのは、環境がアクティブな間だけであることを忘れないでください。



Djangoでプロジェクトを作成する

アクティブな仮想環境を離れることなく（ll_envは括弧内に表示されます）、次のコマンドを入力して新しいプロジェクトを作成します。 



 (1) (ll_env)learning_log$ django-admin.py startproject learning_log . (2) (ll_env)learning_log$ ls learning_log ll_env manage.py (3) (ll_env)learning_log$ ls learning_log __init__.py settings.py urls.py wsgi.py


チーム（1）はDjangoにlearning_logという新しいプロジェクトを作成するように命令します。 コマンドの最後のポイントは、開発が完了した後、アプリケーションをサーバーに簡単に展開できるディレクトリ構造を持つ新しいプロジェクトを作成します。



注

重要な点を忘れないでください。さもないと、アプリケーションのデプロイ時に構成の問題が発生する可能性があります。 まだ忘れている場合は、作成したファイルとフォルダー（ll_envを除く）を削除して、コマンドを再度実行してください。



lsコマンド（Windowsではdir）（2）は、Djangoがlearning_logという新しいディレクトリを作成することを示しています。 manage.pyファイルも作成されます。これは、コマンドを受け取り、それらを実行のためにDjangoの適切な部分に渡す短いプログラムです。 これらのコマンドを使用して、データベースの操作やサーバーの実行などのタスクを管理します。



learning_logディレクトリには4つのファイル（3）があり、その中で最も重要なのはsettings.py、urls.py、wsgi.pyファイルです。 settings.pyファイルは、Djangoがシステムと対話し、プロジェクトを管理する方法を定義します。 プロジェクトの開発中に、既存の設定の一部を変更し、新しい設定を追加します。 urls.pyファイルは、ブラウザのリクエストに応じてどのページを構築するかをDjangoに伝えます。 wsgi.pyファイルは、Djangoが生成されたファイルを提供するのに役立ちます（ファイル名は「Web Server Gateway Interface」の略です）。



データベース作成 

  (ll_env)learning_log$ python manage.py migrate (1)Operations to perform: Synchronize unmigrated apps: messages, staticfiles Apply all migrations: contenttypes, sessions, auth, admin ... Applying sessions.0001_initial... OK (2)(ll_env)learning_log$ ls db.sqlite3 learning_log ll_env manage.py


各データベースの変更は、移行と呼ばれます。 migrateコマンドを最初に実行すると、Djangoはデータベースがプロジェクトの現在の状態と一致することを確認します。 SQLiteを使用して新しいプロジェクトでこのコマンドを最初に実行するとき（すぐにSQLiteについて詳しく説明します）、Djangoは新しいデータベースを作成します。 ポイント（1）で、Djangoはプロジェクトで使用される情報を保存するために必要なデータベーステーブルの作成を発表し（移行されていないアプリを同期する）、データベース構造が現在のコードと一致することを確認します（すべての移行を適用します）。



lsコマンドを実行すると、Djangoがdb.sqlite3（2）という別のファイルを作成していることがわかります。 SQLite-単一のファイルで動作するデータベース。 データベース管理を特に監視する必要がないため、単純なアプリケーションの作成に最適です。



プロジェクトを見る

プロジェクトが正しく作成されたことを確認してください。 runserverコマンドを入力します。 



  (ll_env)learning_log$ python manage.py runserver Performing system checks... (1) System check identified no issues (0 silenced). July 15, 2015 - 06:23:51 (2) Django version 1.8.4, using settings 'learning_log.settings' (3) Starting development server at http://127.0.0.1:8000/ Quit the server with CONTROL-C.


Djangoはサーバーを起動して、システム上のプロジェクトを表示し、その動作を確認できるようにします。 ブラウザにURLを入力してページをリクエストすると、Djangoサーバーはリクエストに応答します。 これを行うために、彼は対応するページを作成し、ページをブラウザに送信します。



ポイント（1）で、Djangoは作成されたプロジェクトを検証します。 ポイント（2）で、Djangoのバージョンと使用されている設定ファイルの名前が表示されます。 ポイント（3）で、プロジェクトが利用可能なURLが返されます。 URL 127.0.0.1：8000 /は、プロジェクトがローカルホスト（localhost）、つまりコンピューターのポート8000​​でリクエストをリッスンすることを意味します。 「ローカルホスト」という用語は、システムからの要求のみを処理するサーバーを指します。 開発中のページを他の人が閲覧することはできません。



ブラウザを開き、 localhost ：8000 /-または127.0.0.1：8000 / URLを入力します（最初のアドレスが機能しない場合）。 図に似たものが表示されます。 18.1は、これまですべてが正常に機能していることを通知するためにDjangoが作成するページです。 まだサーバーをシャットダウンしないでください（ただし、中断したい場合は、Ctrl + Cを押してシャットダウンできます）。



注

「ポートはすでに使用されています」というエラーメッセージが表示された場合は、Djangoに別のポートを使用するように指示してください。 これを行うには、python manage.py runserver 8001コマンドを入力し、開いているポートが見つかるまでポート番号を昇順に繰り返します。
