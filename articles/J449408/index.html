アドバンテックHMIベースのHMIコントロールパネル



ビデオ：Habrom管理パネル。 カルマ、評価、およびユーザーの禁止を調整できます。



TL; DR：この記事では、Webaccess / HMI Designer産業開発環境とWebOPターミナルを使用して、Habrのコミックコントロールパネルを作成しようとします。



ヒューマンマシンインターフェイス（HMI）は、人間が制御されるマシンと対話するための一連のシステムです。 通常、この用語は、オペレータとコントロールパネルがある産業システムに使用されます。



WebOPは、ヒューマンマシンインターフェイスを作成するためのスタンドアロンの産業用端末です。 生産管理パネル、監視システム、管理室、スマートホームコントローラーなどの作成に使用されます。 産業機器への直接接続をサポートし、SCADAシステムの一部として機能します。





WebOPターミナル-ハードウェア





WebOPターミナルは、ARMプロセッサをベースにした低電力コンピュータで、モニターとタッチスクリーンを備えた単一のケースで、HMI Designerで作成されたグラフィカルインターフェイスでプログラムを実行するように設計されています。 モデルに応じて、ターミナルにはボード上のさまざまな産業用インターフェースがあります：RS-232 / 422/485、車のシステムに接続するためのCANバス、追加の周辺機器を接続するためのUSBホストポート、ターミナルをコンピューターに接続するためのUSBクライアントポート、オーディオ入力およびオーディオ出力、不揮発性メモリおよび転送設定用のMicroSDカードリーダー。



デバイスは、強力なプロセッサと本格的なデスクトップコンピューターのリソースを必要としないタスクのための、モノブロックの予算代替品として位置付けられています。 WebOPは、他のWebOPと組み合わせて、データを管理および入出力するためのスタンドアロン端末として、またはSCADAシステムの一部として動作できます。





WebOPターミナルは産業用デバイスに直接接続できます



受動冷却とIP66保護



発熱量が少ないため、一部のWebOPモデルはアクティブな空冷なしで完全に作られています。 これにより、ノイズレベルにとって重要なオブジェクトにデバイスを取り付けることができ、ケースに落ちるほこりの量を減らすことができます。



フロントパネルには隙間や継ぎ目がなく、IP66保護レベルがあり、圧力下で水と直接接触できます。





WOP-3100Tターミナルの背面パネル



不揮発性メモリ



WebOPのデータ損失を防ぐために、128KBの不揮発性メモリがあり、RAMと同じように作業できます。 メーターの測定値やその他の重要なデータを保存できます。 電源障害が発生した場合、データは再起動後に保存および復元されます。



リモートアップデート



端末で実行されているプログラムは、イーサネットネットワークまたはシリアルRS-232 / 485インターフェイスを介してリモートで更新できます。 これにより、ソフトウェア更新のためにすべての端末をバイパスする必要がなくなるため、メンテナンスが簡素化されます。



WebOPモデル









2000Tシリーズは、HMI RTOSリアルタイムオペレーティングシステム上に構築された最も手頃なデバイスです。 このシリーズは、WebOP- 2040T / 2070T / 2080T / 2100Tで表され、画面の対角線はそれぞれ4.3インチ、7インチ、8インチ、10.1インチです。





3000Tシリーズは、Windows CEオペレーティングシステムに基づくより高度なモデルです。 これらは、多数のハードウェアインターフェイスが2000Tシリーズと異なり、ボード上にCANインターフェイスを備えています。 デバイスは拡張温度範囲（-20〜60°C）で動作し、帯電防止保護（空気：15KV /接触：8KV）を備えています。 このラインはIEC-61000規格の要件を完全に満たしており、静電気放電が問題となる半導体製造でのデバイスの使用を可能にします。 このシリーズは、WebOP- 3070T / 3100T / 3120Tモデルで表され、画面の対角線はそれぞれ7インチ、10.1インチ、12.1インチです。



WebAccess / HMI Designer開発環境



すぐに使用できるWebOPターミナルは、任意のソフトウェアを実行できる低電力ARMコンピューターですが、このソリューションのポイントは、独自のWebAcess / HMI産業用インターフェース開発環境にあります。 システムは2つのコンポーネントで構成されています。



  • HMI Designerは、インターフェースとプログラミングロジックを開発するための環境です。 コンピュータープログラマーのWindowsで動作します。 最終プログラムは、単一のファイルにコンパイルされ、ランタイム実行のために端末に送信されます。 プログラムはロシア語で利用可能です。
  • HMIランタイム -ターミナルでコンパイルされたプログラムを実行するためのランタイム 。 WebOP端末だけでなく、Advantech UNO、MICコンピューター、および通常のデスクトップコンピューターでも機能します。 Linux、Windows、Windows CE用のランタイムのバージョンがあります。






Hello World-プロジェクトの作成



Habrコントロールパネルのテストインターフェイスの作成を始めましょう。 WinCEを実行しているWebOP-3100Tターミナルでプログラムを実行します。 最初に、HMI Designerで新しいプロジェクトを作成します。 WebOPでプログラムを実行するには、正しいモデルを選択することが重要です。最終ファイルの形式はこれに依存します。 この手順では、デスクトップアーキテクチャを選択することもできます。その場合、結果のファイルはX86ランタイムでコンパイルされます。









新しいプロジェクトの作成とアーキテクチャの選択



コンパイルされたプログラムがWebOPにロードされる通信プロトコルの選択。 このステップでは、シリアルインターフェイスを選択するか、端末のIPアドレスを指定できます。









プロジェクト作成インターフェース。 左側には、将来のプログラムのコンポーネントのツリー図があります。 Screens項目のみに関心がありますが、これらはターミナルに表示されるGUI要素を備えた直接的な画面です。











最初に、テキスト「Hello World」とボタンを使用して画面を切り替える機能を持つ2つの画面を作成します。 これを行うには、新しいScreen Screen＃2を追加し、各画面にテキスト要素と、画面を切り替えるための2つのボタン（画面ボタン）を追加します。 各ボタンは、次の画面に切り替えるように構成されています。







画面を切り替えるボタンを設定するためのインターフェース



Hello Worldプログラムの準備ができました。これで、コンパイルして実行できます。 コンパイル段階で、変数またはアドレスが正しく指定されていない場合、エラーが発生する可能性があります。 エラーは致命的とみなされ、エラーがない場合にのみプログラムがコンパイルされます。

この環境は、コンピューターをローカルでプログラムをデバッグできるように、端末をシミュレートする機能を提供します。 シミュレーションには2つのタイプがあります。



  • オンラインシミュレーション -プログラムで指定されたすべての外部データソースが含まれます。 これらは、 USOまたはシリアルインターフェイスまたはModbus TCPを介して接続されたデバイスです。
  • オフラインシミュレーション -外部デバイスを使用しないシミュレーション。


外部データはありませんが、プログラムをコンパイルした後、オフラインシミュレーションを使用します。 最終的なプログラムは、 ProjectName_Program Name.px3という名前でプロジェクトフォルダーに配置されます。









シミュレーションで起動されたプログラムは、WebOPターミナルのタッチスクリーンと同じように、マウスカーソルで制御できます。 すべてが意図したとおりに機能することがわかります。 素晴らしい。

プログラムを物理端末にダウンロードするには、[ダウンロード]ボタンをクリックするだけです。 しかし、ターミナルの開発環境への接続を構成しなかったため、USBフラッシュドライブまたはmicroSDメモリカードを使用してファイルを簡単に転送できます。







プログラムインターフェイスは直感的で、各グラフィックブロックを分解しません。 背景、図形、テキストの作成は、Wordに似たプログラムを使用した人なら誰にでも明らかです。 グラフィカルインターフェイスの作成にはプログラミングスキルは不要で、すべての要素はフォームにドラッグアンドドロップすることで追加されます。



メモリを操作する



グラフィック要素を作成できるようになったので、動的コンテンツとスクリプト言語での作業を学習します。 変数U $ 100のデータを表示する棒グラフを作成します。 チャート設定で、データ型を選択します：16ビット整数、およびチャート値の範囲：0〜10。







このプログラムは、VBScript、JavaScript、および独自の言語の3つの言語でのスクリプトの作成をサポートしています。 3番目のオプションを使用します。これは、ドキュメントに例があり、エディターに自動構文ヘルプがすぐに表示されるためです。



新しいマクロを追加します。







変数のデータを徐々に変更するための簡単なコードを記述します。これは、チャートで追跡できます。 変数に10を追加し、100を超える場合はゼロを追加します。 



$U100=$U100+10 IF $U100>100 $U100=0 ENDIF


スクリプトをループで実行するには、一般的なセットアップ設定でメインマクロとして実行間隔250ミリ秒でインストールします。









シミュレーターでプログラムをコンパイルして実行します。







この段階で、メモリ内のデータを操作し、視覚的な形式で表示する方法を学びました。 これは、外部デバイス（センサー、コントローラー）からデータを受信し、それらをメモリに書き込む簡単な監視システムを作成するのに十分です。 HMI Designerでは、矢印、さまざまなチャート、グラフを備えた円形ダイヤルの形で、さまざまなデータ表示ブロックを使用できます。 JavaScriptスクリプトを使用して、HTTPを介した外部ソースからのデータのロードを実装できます。



Habrコントロールパネル



取得したスキルを使用して、Habrom管理パネルのコミックインターフェースを作成します。











リモコンは次のことができる必要があります。



  • ユーザープロファイルの切り替え
  • カルマとランキングデータを保存する
  • スライダーを使用してカルマと評価値を変更する
  • 禁止ボタンをクリックすると、プロファイルは禁止としてマークされ、プロファイルは取り消し線に変わります


各プロファイルは個別のページに表示されるため、プロファイルごとにページを作成します。 カルマと評価をメモリ内のローカル変数に保存し、プログラムの起動時にセットアップマクロを使用して初期化します。





写真はクリック可能です



カルマと評価の調整



カルマを調整するには、スライダー（スライドスイッチ）を使用します。 レコードアドレスには、セットアップマクロで初期化された変数を指定します。 スライダーの値の範囲を0〜1500に制限します。スライダーが移動すると、新しいデータがメモリに書き込まれます。 この場合、スライダーの初期状態はメモリ内の変数の値に対応します。









カルマと評価の数値を表示するには、数値表示要素を使用します。 操作の原理は、サンプルプログラム「Hello World」の図に似ています。変数のアドレスをMonitor Addressで指定するだけです。



ボタン「禁止」



禁止ボタンは、トグルスイッチ要素を使用して実装されます。 データストレージの原理は、上記の例に似ています。 設定では、ボタンの状態に応じて、異なるテキスト、色、または画像を選択できます。









ボタンを押した瞬間に、アバターを赤で消す必要があります。 これは、画像表示ユニットを使用して簡単に実装できます。 Toggle Switchボタンの状態に関連付けられた複数の画像を指定できます。 このため、ブロックには、ボタンが付いたブロックと同じアドレスと状態の数が与えられます。 アバターの下のネームプレートの写真も同様に構成されています。









おわりに



一般的に、私は製品が好きだった。 以前は、同様のタスクにAndroidタブレットを使用した経験がありましたが、そのためのインターフェースの開発ははるかに複雑であり、ブラウザーAPIでは周辺機器へのフルアクセスが許可されていません。 1つのWebOP端末で、Androidタブレット、コンピューター、コントローラーのバンドルを置き換えることができます。



HMI Designerは、古風なデザインにもかかわらず、かなり高度です。 特別なプログラミングスキルがなくても、機能するインターフェイスをすばやくスケッチできます。 この記事では、アニメーションブロック、シリンダー、グラフ、トグルスイッチなど、多くのグラフィックブロックを考慮していません。 すぐに使用できる多くの一般的な産業用コントローラーをサポートし、データベースコネクタが含まれています。
