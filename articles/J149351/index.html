SteamまたはWindowsストア。 または、今のところSteamを選択する理由

過去数日間にわたって、Steamのイノベーションに関する誇大広告が増えています。 トピックとそれらに関するコメントを読んだ後、私は誰もがこれらのアプリストアに関する信頼できるデータを持っているわけではないことに気付きました。 特に、私はWindowsストアについて、そしてより正確には私がまだSteamに賭けている理由について話したいです。



長い間、私は統合されたアプリケーション配信システムを手に入れたかったのですが、これはWindowsには欠けていました。 アプリケーションの更新を監視し、アプリケーション自体をインストールできるシステムを多数試しました。 それらのどれも私のニーズを満たしていない。



数年前、私は積極的にSteamを使用し始め、最初に無料ゲームに座ってから購入を始めました。 「海賊ではない」ということは非常に簡単だということに気づきました。 私がこれから私を制限した唯一のものは、取得の利便性であることに気づきました。 それ以来、私は自分のために、同様の種類のショッピングの基準をいくつか策定しました。

  1. 買収の利便性;
  2. アカウントへのアプリケーションのバインド。
  3. サードパーティのソースからではなく、同じストアからのインストールと更新。




時間が経ち、RP Windows 8が登場し、戦闘システムとしてインストールし、「サイド」を見て、アプリケーションを公開するためのルールを読み、失望しました。 とにかく期待したものが得られないことに気付きました。 つまり、私の基準を満たしていません：

  1. その中のすべてのソフトウェアをダウンロードして購入することは可能です。 マイクロソフトは、アプリケーションをメトロアプリケーションと「クラシックアプリケーション」に分けています。 従来のアプリケーションの場合、購入できるリンクが提供されます（下のスクリーンショットの例）。
  2. その結果、購入したアプリケーションがアカウントに関連付けられるかどうかはわかりません。 また、ライセンスキーを紙に書き留めておく必要があります。
  3. その結果、必要なすべてのアプリケーションを1か所からインストールして更新できるかどうかはわかりません。








Windowsストアでの公開ルールの詳細については、 こちらをご覧ください 。



今、Steamからの応答を待っています。 結局、購入をアカウントにリンクすることのすばらしさを本当に感謝しました。 ゲームを何度も再インストールした後、それらを探すためにキー/ディスクを保存する必要はありません。 主張されていることから判断すると、Valveはアプリケーションストアをゲームと同じくらい便利にすることが期待できます。

私は尊敬し、私自身のやり方で各OSが大好きです。 Linuxでは、AppStoreが好きなiOSおよびMacOSで、リポジトリを操作する便利さを喜んでいます。 そして、マイクロソフトがそのストアをより便利で快適なものにすることを今でも願っています。



おそらく独占禁止委員会による新しい訴訟を恐れているか、そうするつもりがないために、同様の措置が取られたのでしょう。 しかし、インセンティブにアプリケーションが登場することは、Microsoftのストーリーに関する意見に影響を与え、彼らの決定を再考することを願っています。 実際、競争は常に良好であり、SteamとMicrosoftの両方にとって、そして最も重要なことには、エンドユーザーにとって良いことです。



また、上級ユーザーとは異なり、Steam自体をインストールする可能性が低い一般ユーザーについても忘れないでください。 もしそうなら、彼らがそれを使用するという事実ではありません。
