更新または再インストールしますか？ Windows XPおよびVistaからWindows 7へ

既存のWindows XPまたはWindows VistaからWindows 7にアップグレードできると、通常、多くの質問と困難が生じます。 Microsoftは状況を明確にするために小さなタブレットを作成しましたが（以下を参照）、誤解の大きな波を引き起こしました。







表に示されているインストールには2つのタイプがあります。それらについて詳しく説明します。



インプレースアップグレード -インストール済みのOSを更新します。 この場合、インストールディスクから起動した後、またはオペレーティングシステム内の後者からsetup.exeを実行した後に、アップグレードオプションを選択することができます。 この場合、インストールされたプログラムとその設定が保存されます。



カスタムインストールは非常に曖昧な概念であり、何を意味してもかまいません。 この場合、これはデータのコピー（Easy Transfer Utility経由）、ハードドライブのフォーマット、Windows 7のインストールが必要であることを意味します。その後、保存したデータとプログラムの設定をインポートします。 ところで、これはデータと設定を32ビットWindowsから64ビットに転送する唯一の方法です。 また、この方法は最も時間がかかり、時間がかかります。



事前にフォーマットされたハードドライブにインストールすることを好みます。 利用可能な情報のコピーを作成することをお勧めします。 Windows転送ユーティリティはデータ転送に非常によく対応しますが、神は安全な人を救います。 Windows転送ユーティリティは、インストールされているプログラムの設定とシステム設定（壁紙、DPI、サウンドなど）も転送します。 ほとんどの種類の外部メディアおよびネットワークドライブで動作します。

多数のコンピューターを使用して、オフィス環境では、現代の管理者の有用なユーティリティであるMDTとUSMTが助けになります。 まず、Microsoft Deployment Toolkit 2010では、組織の特定のニーズに基づいてインストールをカスタマイズできます。 無料でベータ版の開発段階にあり（Windows 7での作業が追加されています）、将来的にはWindows Deployment ServicesまたはSystem Center Configuration Managerを使用して、ネットワーク経由でインストールを1つまたは数千のコンピューターに配布できます。 また、変更されたインストールイメージをUser State Migration Toolkit 4.0に統合できます。 後者は一種のWindows転送ツールですが、企業レベルです。 User State Migration Toolkit 4.0の特別な利点は、32ビットOSから64ビットにデータを移行できることです。 Microsoft Deployment Toolkit 2010と組み合わせることで、数百、数千のワークステーションに関して強力な管理ツールを入手できます。
