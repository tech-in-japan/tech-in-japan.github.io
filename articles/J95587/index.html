ロストフの学校プロジェクト

5月22日土曜日、私たち3人のグループ、ユージーン（enpx）、ドミトリー（diskin）、および私（supervisor）-RND-LUGのすべてのメンバーが、学校番号83を再び訪れました。 私たちが到着したのは、この学校に2つのコンピュータークラスがあり、最初の訪問時に1つしかインストールできなかったためです。



2番目のクラス（または、むしろ実験室）で、インストールを開始しました。 今回は、5番目のプラットフォームをインストールしました。 幸いなことに、鉄は優れていた。 2ギガバイトのRAMと強力なプロセッサを搭載した10台の東芝製ラップトップでした。



Linuxでは、2番目のシステムをインストールしました。 この場合、最初のシステムはMS Windows Vistaにインストールされました。 いくつかのおもちゃとウイルスの痕跡を除いて、ラップトップで特別な過剰なソフトウェアは見つかりませんでした。 インストールは迅速で問題ありませんでした。 WiFi、サウンド、ウェブカメラまで、すべての機器が決定されました。 周辺からは、古代のキヤノンPixmaプリンターだけでした。 このモデルのシステムにはネイティブドライバーはありませんでしたが、キヤノンのドライバーは動作しませんでした。 この接続では、プリンター接続は延期されました。 ちなみに、Eugene enpx（彼はそれでもALT Linux Teamのメンバーです）が、5番目のプラットフォーム用のドライバーを後で作成しました。 ドライバーが必要な場合は、 こちらで入手できます 。







もちろん、このインストールフェスティバルを本格的な実装と呼ぶことはできません。 しかし、私たちはそのようなタスクを設定しませんでした。 実際、私たちが選択した主なアイデアは、最初の一歩を踏み出す際の支援です。 これがインストールです。 もちろん、実際の操作中に多くの問題が発生し（そして解決する必要があります）、Linuxがインストールされていない場合にのみ、その操作は開始されません。





教師は、コースでのトレーニングまたはセルフトレーニングの後、実装と運用の運用上の問題を解決する必要があります。また、学校にサービスを提供するITスペシャリストも解決する必要があります。 たぶん、私たちのLUGが何かに役立つでしょう。 トレーニングに関しては、当グループは5日間の教師トレーニングコースも実施しました。 そして、もっと使う準備ができています。



プロセスに対する教師の態度は非常に異なっており、方法によって教師の知識のレベルと非常に相関しています。 別の学校のレビューによると、学童はコンピューターにWindowsがインストールされたときよりも非常に興味があります。 この現象の説明はありません。 学校の校長は、状況を理解することに基づいて態度を構築し、状況は原始的に単純です-予算にはプロプライエタリなソフトウェアの購入は含まれず、資金の目標とする使用を制御できます。



UOにも予算があり、独自のソフトウェアの購入も含まれていません。 まあなど。 そして一番上は予算コードです。 しかし、これは完全に異なるトピックです。
