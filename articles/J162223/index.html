NetWrix Password Managerセルフサービスパスワードリセットマネージャーの概要



困難な状況でエンドユーザーを支援するタスクは、大企業や成長中の企業のIT部門にとって大きな負担です。 いくつかの推定によると、パスワードリセットの各ケースは70ドル（作業時間の損失を考慮に入れると）かかり、そのようなインシデントはテクニカルサポートへの電話の約30％を引き起こします。 銀行部門など、特定の法律が制定されている業界では、コストがさらに高くなる可能性があります。



ユーザーがシステムにアクセスするためにパスワードを忘れるのはなぜですか？

いくつかの理由が考えられます。

  1. 覚えておくべき多くのパスワード。 各ユーザーには、平均して約12個のサービスとアプリケーションがあり、それぞれにパスワードが必要です。 多くの場合、ユーザーはこれらのパスワードで単に紛失し、間違ったパスワードを入力します。
  2. 定期的なパスワード変更。 厳格なパスワードポリシーを持つ組織では、パスワードには一定の有効期限があります。 そして、一定の期間の後、パスワードを変更する必要があります。 ユーザーはパスワードの変更を忘れ、自動的にランダムに設定されます。 または、彼はパスワードを変更しますが、それでも慣れず、間違ったパスワードを入力すると混乱します。
  3. 強力なパスワードポリシーを実装します。 複雑なパスワードは忘れやすいです。




しかし、私たちは専門家が理解できる理由について話しているのではなく、そのような状況で、まず管理者とヘルプデスクサービスの生活を簡素化する方法について話しているのです。 短時間パスワードをリセットします（ 例を参照）が、テクニカルサポートサービスへのそのようなリクエストが多数ある場合、その処理に大量のリソースが費やされます（時間が無駄になります）。 もちろん、ユーザーはシステムにアクセスできないときには作業できません。 全体的なパフォーマンスが低下します。





Active Directory環境では、ユーザーパスワードの管理には、グループポリシー、ヘルプデスクアクティビティ、ユーザーアカウント管理オプションの複数の構成によるパスワードセキュリティ要件の強化など、多くのタスクが含まれます。 多くの場合、これらの操作は分散化されており、アカウント所有者はアカウントの管理から離れています。

ここで、自己リセットパスワードのディスパッチャが助けになります（ osp.ruのレビューを参照 ）。 パスワード管理タスクを技術サポートからユーザー自身に部分的に移行するように設計されています。 dr死を救うことはdr死者の仕事です。

この製品グループの本質については説明しませんが（上記のレビューへのリンクを参照）、AD- NetWrix Password Managerでユーザーパスワードを自己管理するためのソリューションについて説明します。 このプログラムには、基本無料（最大50ユーザー）と拡張商用の2つのバージョンがあります。



会社の従業員の3つのカテゴリの作業を一度に簡素化します。

  1. ユーザーは自分でパスワードを変更でき、テクニカルサポートサービスの順番に達するまで待つことはできません。
  2. ヘルプデスクオペレーターは、このような問題の解決に時間を費やしません（上記で説明したように、時間の30％まで）。 彼らはより重要なことに集中できます。 スタッフの最適化を達成。
  3. 管理者は、より強力なパスワードポリシーを実装し、パスワードの変更に関する詳細なレポートを受け取ることができます。




これを達成するために、3つの役割が区別されます。

  • エンドユーザー
  • ヘルプデスクオペレーター
  • 管理者




これらの役割をグループおよび個人に適用することにより、管理者はパスワード管理へのアクセスを制御できます。



製品の主な機能と説明





NetWrix Password Managerは、ヘルプデスクと管理者の負荷を軽減するのに役立ちます。

  1. ユーザーには、パスワードを使用して基本的なアクションを実行できるWebポータルへのアクセスが許可されます。
  2. ヘルプデスクのスペシャリストは、ユーザーアカウントを管理し、Webインターフェイスを介してステータスのステータスに関するレポートを受信できます。
  3. 管理者は、より厳しいパスワード要件を実施できます。




製品アーキテクチャ





NetWrix Password Managerは、次のコンポーネントで構成されています。

1. Webアプリケーション ：プログラム機能を提供する3つのWebポータルが含まれています。

  • 管理ポータル：ユーザーが使用できる制御ドメイン、質問ポリシー、オプション、およびシステムのユーザーを一括登録するプログラム設定を変更できます。
  • ヘルプデスクポータル：登録（登録済み）アカウントの集中管理およびレポート用に設計されています。
  • ユーザーポータル：パスワードを自己管理するためのWebベースのユーザーインターフェイス。


2. コアパスワードマネージャーサービス ：Webポータルから要求された操作を実行します。

3. パスワードマネージャークライアント （Windowsログイン画面の拡張機能）：標準のログインダイアログの機能を拡張し、ユーザーがパスワードをリセットおよび変更できるだけでなく、アカウントを個別にロック解除できるようにします。 クライアントは登録ウィザードをサポートしています。







製品は、デフォルト設定（パスワードセキュリティ設定、ユーザーが利用できるオプション、検証の問題ポリシーなど）でインストールされます。 管理ポータルから変更できます。



そして、ポータルについてさらに説明します。



すでに上で示したように、プログラムの操作は3つのカテゴリのユーザーによって実行されます。 それぞれに独自のポータルがあります。



管理ポータル





ポータルへのアクセスは、URL（プログラムのセットアップ中に設定）によって任意のコンピューターから取得できます。







プログラム設定は、このポータルを介して設定されます。 プログラムを使用する場合、管理者は次のことができます。

  1. 監視対象ドメインのリストを追加、削除、または変更します。
  2. 以下を構成します。

    • ブランディング（ロゴ、連絡先）
    • ユーザーオプション（ユーザーが利用できるパスワード管理オプション）
    • 質問（検証に使用される一連の質問）。 ユーザーに母親の旧姓ではなく、何か別のものを尋ねたい場合は、この関数を使用します。
    • 質問ポリシー（質問と回答の長さ、検証に必要な質問の最小数）
    • パスワードポリシー（パスワードの長さ）
    • アラート（応答するイベントおよびそれらに関する通知の送信先）


  3. ユーザーとグループにロールを割り当てます（管理者/ヘルプデスクオペレーター/ユーザー）
  4. プログラムでのユーザーの一括登録（バッチ登録）の実行-.csvファイルからのアカウント情報のインポート




ユーザーポータル（セルフサービスポータル）





ユーザーポータルは4つの基本的なアクションを実行します。

  1. NetWrix Password Managerに登録する
  2. パスワードリセット
  3. パスワードを変更する
  4. アカウントのロック解除




NetWrix Password Managerでのユーザー登録は、手動で（ユーザーが自分のコンピューターまたはユーザーポータルで質問に答える）、または管理者が実行するバッチ登録操作で実行できます。



パスワード管理クライアント経由




プログラムのインストール後、パスワード管理クライアントコンポーネントが自動ロードに追加されます。 同時に、ユーザーは次のログイン時に次のウィンドウを表示します。







ユーザーに対する一連のアクションは非常に明白です。質問を選択し、それに答えます。 もちろん、いくつかの質問があるかもしれません。 また、これがポリシーで禁止されていない場合、ユーザーは自分の質問を思い付くことができます（この設定は管理ポータルで行われます）。



ユーザーポータル（セルフサービスポータル）を介して




ポータルを開きます（[スタート]> [すべてのプログラム]> [NetWrix]> [パスワードマネージャー]> [セルフサービスポータル]）。

このポータルはロシア語で利用できることに注意してください。

登録を選択します。







ユーザーは質問を選択し、それらに回答します。



システムに登録されているユーザーは、次のアクションを実行できます。

  • パスワードをリセット
  • パスワードを変更する
  • アカウントのブロックを解除します




パスワードリセット

パスワードリセットは、ユーザーがWindowsログイン画面でユーザーポータルまたは拡張機能にアクセスできない場合に、ヘルプデスクのユーザーとオペレーターによって実行されます。

このプログラムのユニークな機能は、サーバーに接続されていないときにクライアントを介してパスワードをリセットすることです。

上記のレビューからの引用は次のとおりです。

「 NetWrix Password Managerのユニークな機能は、オフラインモードでのパスワードリセットです。 この場合、Windowsの登録画面でGINA拡張機能がアクティブになり、ドメインに接続されていなくても、ユーザーのキャッシュされたパスワードがリセットされます。 多数のモバイルユーザーを抱える企業にとっては非常に便利ですが、GINA拡張機能のローカルインストールが必要です。



ユーザーパスワードのリセット：

パスワードを変更するには、ユーザーはいくつかの手順を実行します。

  1. ポータルを開きます。
  2. [リセット]セクションで[リセット]を選択します。
  3. ユーザー名とドメインを入力します。
  4. 登録手順中に選択された質問に答えます。
  5. 質問に対する正しい回答の後、ユーザーは新しいパスワードとその確認を入力します。








その結果、パスワードが変更されます。

プログラムの助けを借りて、ユーザーは自分のアカウントのロックを解除することもできることに注意してください（たとえば、間違ったパスワードを連続して数回入力すると、ブロックされます）。 この場合も、ヘルプデスクのオペレーターが関与することなく、これらすべてが独立して行われます。



Windowsログイン拡張機能によるパスワードリセット：

  1. ログアウトする
  2. 「ログインできませんか？」をクリックします
  3. ログオン支援ウィザードが表示されます
  4. ドメイン\ユーザー名を入力し、[次へ]をクリックします

  5. 新しいパスワードを入力して確認します

    • アカウントがブロックされている場合は、[アカウントのロック解除]を選択します
    • パスワードの有効期限が切れないようにする場合は、[パスワードを無期限にする]を選択します


  6. 質問への回答を入力します。 パスワードが変更されます。




ヘルプデスクポータルを介したパスワードのリセット（ヘルプデスクオペレーターが実行）



  • ヘルプデスクポータルを開きます（スタート>すべてのプログラム> NetWrix>パスワードマネージャー>ヘルプデスクポータル）
  • パスワードを変更する必要があるユーザーを選択します
  • [ロック解除]または[パスワードのリセット]を選択すると、[ユーザーID確認]ページが表示され、ユーザーが登録時に選択した質問とその回答が表示されます。 この場合、回答の最初と最後の文字のみが表示されます。 ユーザーはオペレーター（最初と最後の文字）にヘルプデスクをオペレーターに知らせ、オペレーターはパスワードをリセットします。
  • ユーザーに新しいパスワードを割り当てます。
  • パスワードがリセットされます。




管理者コントロール





管理者がすべてのパスワード変更を制御できるように、プログラムにはレポートと通知があります。

  1. 次のイベントの通知が含まれる場合があります。
  2. Password Managerでアカウントを登録します。
  3. パスワードのリセット。
  4. アカウントのロックを解除します。
  5. パスワード変更
  6. 検証に失敗しました。




次の2種類のレポートが利用可能です。

  1. ユーザーアクティビティには、一定期間のパスワードに関連するすべてのユーザーアクション（登録、変更、リセット、アカウントのロック解除）が表示されます。 レポートには、結果と時間が表示されます。

  2. ユーザー登録では、監視対象ドメイン内のすべてのユーザーの登録ステータス（NetWrix Password Managerデータベースに登録されているかどうかにかかわらず）が表示されます。
