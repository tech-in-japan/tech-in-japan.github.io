高速化するキャッシュ



過去10年間で、サーバーは他のコンピューター機器と共に、根本的にパワーを追加してきました。 1つのパッケージに数十個のプロセッサコアと数百ギガバイトのRAMがあれば、誰も驚くことはありません。 ただし、複雑なシステムと同様に、サーバーには、プラットフォーム全体を最大限に活用できない弱点があります。 現代のサーバーエコノミーでは、このような弱いリンクがデータストレージサブシステムであることがよくあります。 この問題の解決策の1つは、 Intel Cache Acceleration Softwareを使用してディスク操作を高速化することです。



名前が示すように、Intel Cache Acceleration Software（Intel CAS）は、データキャッシングを管理するソフトウェア製品です。 通常のデスクトップコンピューターには、Intelの同様のソフトウェアが存在します。 スマートレスポンスと呼ばれます。 ただし、サーバーコンポーネントはまったく異なる条件で、異なるハードウェア上で動作します。 それらの要件もはるかに厳しいです。 対応するオペレーティングシステムの機能と組み合わせて、Intel CASは、RAMと高速ソリッドステートドライブ（SSD）の両方を使用するマルチレベルキャッシュシステムを実装します。 これにより、アプリケーションに要件を課さない透過的なキャッシュが作成され、データにアクセスする際の待ち時間が短縮されます。 CASソフトウェア自体は、特定の負荷のデータをキャッシュするための最適な戦略を選択します。その結果、 システムはSSDに完全に配置されている場合よりも高速に動作します 。



メインストレージは、ローカルSATA / SAS RAID、外部またはネットワーク-SAN、iSCSI、ファイバーチャネルなど、何でもかまいません。 さらに、キャッシュされたSSD自体へのネットワークアクセスが十分なデータ転送速度を提供する場合、キャッシュされたSSD自体もリモートに配置できます。



Intel CASは、物理システムと仮想システムの両方、あるいはそれらにインストールされたアプリケーションを高速化するために使用できます。 どれ？ 原則として、最も効果的なものを除き、データベースサーバー、メールサーバー、Webサーバー、ビジネスアプリケーションサーバーなど、ファイルシステムでアクティブに機能するものがすべてです。 Intel CASの機能は、キャッシュプロセスをアプリケーションおよびファイルのレベルまで微調整する機能であり、場合によってはさらに深くなります。たとえば、SQLサーバーの場合、個別のテーブルで操作することもできます。







Intel CASソフトウェアおよびハードウェアの要件：

  • オペレーティングシステム-Microsoft Windows Server 2008 R2、Red Hat Enterprise Linux 5/6、CentOS 5/6、SUSE Linux Enterprise Server 11 SP1。
  • 仮想化環境-Vmware、Windows Hyper-V、KVM、Citrix XenSever。
  • SSD-Intel DC S3700シリーズやIntel SSD 910シリーズなどのデータセンタークラスのSSD。


Intel CASを使用すると、サーバーフリートの機能を再確認できます。サーバーが1つのタスクをより効率的に処理できるようになった場合、パフォーマンスを犠牲にすることなく別のタスクを任せることができるからです。 これにより、ラックユニット、電力、メンテナンスなどが節約されます。これは、Intel Cache Acceleration Softwareがディスク作業を高速化するためです。

Intel Webサイトのフォームに記入して、30日間の評価版を入手できます。
