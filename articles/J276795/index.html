5nine Managerを使用したHyper-Vフェールオーバークラスターの管理

ほとんどの組織は、仮想環境でサービスの提供または企業リソースへのアクセスの継続性を維持する必要があります。 したがって、管理者は、Hyper-Vフェールオーバークラスターの構成に関する問題を簡単かつ迅速に解決する方法を探しています。 現在、この構成は大企業だけでなく、中小企業でさえ、安価な製品である5nine Managerの機能を使用して利用可能です。 この資料は、便利ではあるが高価なVMwareソリューションの代替を探しているvSphere管理者にとっても興味深いかもしれません。



これでポイントになりました！ 5nine Managerはホストを検出し、それらからフェールオーバークラスターを作成します。 ノードを定義し、クラスター検証を起動し、アクセスポイントを構成してからクラスターを作成します。







フェールオーバークラスターの検証は、Microsoftの構成をサポートするための前提条件であるため、展開の重要なタスクです。 5nine Managerを使用すると、構成中にクラスターのステータスを確認し、トラブルシューティングツールとして使用できます。 さまざまなテストから選択して、使い慣れたグラフィカルレポートを実行および表示します。







クラスタ構成のテストに加えて、5nine Managerコンソールから各Hyper-Vホストで一連の高度なベストプラクティステストを実行できます。 その結果、ノードの安定性とパフォーマンスを改善するための推奨事項を受け取ります。







ライブ移行では、クラスターの状態と重要なサービスに影響を与えないように、専用ネットワークを作成することが重要です。 5nine Managerでは、同時ライブマイグレーションとストレージライブマイグレーションの数を決定し、クラスター内の他のノードに設定をコピーできます。







集中情報パネルには、クラスター、そのホスト、およびその仮想マシンの状態に関する情報が表示されます。 ホストまたはVMの問題をすばやく特定し、リソースステータスアラートのリストを表示します。 より明確にするために、情報の詳細をデータセンター、クラスター、ホスト、またはVMのレベルで構成できます。







5nine Managerを使用して、仮想マシンドライブと仮想ネットワークを構成できます。 また、クラスターノードの一時停止や再開など、標準的なクラスターメンテナンスタスクを実行し、それに伴う他のノードとのVMの移行を行うことができます。 アクティブなタスクとクラスター障害のリストもインターフェースに表示されます。





Windows Server、Windows、Linux、UNIX、Windows Server 2016 Nano Serverなど、Hyper-Vでサポートされているあらゆる種類のVMを一元管理できます。 製品はマイクロソフトと共同で開発されているため、OSのすべての関連機能がサポートされています。



完全なグラフィカルインターフェイスは、Windows Server CoreおよびHyper-V Serverを含むWindows Serverのすべてのバージョンで動作します。







各リソースの構成情報と設定を使用して、クラスターの構成とステータスに関するレポートを簡単に作成できます。 このドキュメントは、後で監査するためにエクスポートおよび保存できます。







5nine Managerを使用すると、クラスターノードを動的最適化グループに結合して、VMの負荷分散を実現できます。 プロセッサリソース、RAM、またはディスクアクセスのカスタム値を超えた場合、ホスト間で「ライブ」VM移行を実行します。 このタイプの動的最適化により、ホストが過負荷になることがなくなり、高可用性が実現し、VMの全体的なパフォーマンスが向上します。







クラスター上のすべてのイベントを確認することはかつて困難でした。 5nine Managerは、すべてのクラスター、ホスト、およびVMログを収集して、トラブルシューティングを簡素化します。







5nine Managerは、パフォーマンスダッシュボードを使用して、クラスターノード、個々のホスト、およびVMの現在および過去のリソース使用量に関するデータを表示します。 どのVMがより多くのリソースを消費しているか、最新の通知、CPU、メモリ、ディスク、およびネットワークの消費量をグラフィカルに表示します。 以前のパフォーマンスデータを表示して、最新の問題をローカライズすることもできます。







Hyper-Vレプリカを使用すると、事故が発生した場合の復旧のためにVMディスクのコピーを保存できます。 5nine Managerを使用すると、1つのホストでレプリケーション設定を構成し、それを他のホストに適用できます。







Hyper-Vレプリカブローカーが構成されたクラスターで実行されるVMでレプリケーションを構成することもできます。 レプリカのステータスもコンソールに表示されます。







フェールオーバークラスターは仮想インフラストラクチャの不可欠な部分である必要があり、5nine Managerはクラスター上のすべてのVMを集中管理する機能を提供します。 製品には無料版があります。
