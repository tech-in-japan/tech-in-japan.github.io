Ubuntu 9.04で指紋スキャナーを構成する

このトピックを書くために、この重要なイベントの結果として、ラップトップの交換とベータ版Ubuntu 9.04への移行を促されました。

バージョン7.10以降、この価値のある製品を使用していると言わざるを得ません。 この間、いくつかのマシンが変更され、機器を決定するのに問題がなかったため、選択を後悔することはあまりありませんでしたが、使用しているIBM Thinkpad機器はすべてのLinuxと同様に常に十分にサポートされているという事実によるかもしれませんIBMで行われたこと。

仕事のために、HPの手頃な製品、nc6400ラップトップを手に入れました。 このデバイスには指紋スキャナーが搭載されており、パスワードの混乱の中で生活が少し楽になります。 これまで、スキャナーの使用はMS Windowsに限定されていたため、コミュニティがこのようなあまり人気のないデバイスを扱うことは期待していませんでした。



ラップトップにベータ9.04をインストールした後、「箱から出して」すぐに、モデム（！）を含む、搭載されているすべての機器を認識したことがわかりました。 おまけとして、AES2501指紋スキャナーは、実際にはどこでも使用されていなかったため、USBデバイスのリストで顕著になりました。

グーグルで、このデバイスを接続する方法を見つけましたが、これは非常に簡単でした。

ターミナルでコマンドを入力します。



sudo aptitude install fprint-demo libpam-fprint libfprint0







または、名前にfprintを含むモジュールをシナプスで検索してインストールすることもできます。

これらは、実際の指紋スキャナーのサポートを含み、認証モジュールに接続する2つのライブラリーと、「認識」に参加する指のタイプと数を構成できるグラフィカルアプリケーションです。



その後、コマンドを使用して認証ファイルを変更する必要があります



sudo gedit /etc/pam.d/common-auth







行を追加する



auth sufficient pam_fprint.so







行の前



auth [success=1 default=ignore] pam_unix.so nullok_secure







これにより、何らかの理由で指が認識されない場合に、標準のパスワード入力手順を使用できます。

認識手順は、どの指を適用するかを示すメッセージを表示しますが、一部のアプリケーションではこのメッセージは抑制され、削除する方法を探すのが面倒でした。 したがって、アドバイス：指の1つを特定し、それだけを適用します-混乱しないでください:)。

それだけです。 健康に使用してください。 誰かがこれまで役に立たないデバイスの使用を手伝ってくれたら嬉しいです。
