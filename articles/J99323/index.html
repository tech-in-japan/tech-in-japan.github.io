数独を例にしたAndroidプログラミングの基本

この記事では、Android用のアプリケーションを作成する際の主な問題について説明します。

Androidプログラミングの基本概念が考慮されます。

たとえば、「 Hello、Android-Ed Burnette 」という本から数独ゲームを作成することが説明されています。

多くのスクリーンショットに注意してください。







1.開発の難しさ



Androidはユニークなオペレーティングシステムです。 アプリケーション開発者は、良い結果を得るために、その機能とニュアンスを知っている必要があります。 開発時に考慮すべきいくつかの困難があります（ こちらの説明をご覧ください ）。 それらを簡単にリストします。

1）アプリケーションは、アプリケーションの元のサイズの2倍（または4倍）以上のスペースをインストールする必要があります。

2）内蔵フラッシュドライブ上のファイルの処理速度は、空き領域が少なくなると数十回低下します。

3）各プロセスは、最大16 MB（場合によっては24 MB）のRAMを使用できます。



2. Android用の生産的なアプリケーションの開発の原則



Android用の生産的なアプリケーションを作成するための推奨事項がいくつかあります。 これらは、Effective Java-Joshua Blochおよびプログラミングプラットフォーム上のモバイルデバイスの書籍Net Compact Framework-I. Salmreに基づいて拡張することもできます。

Android向けの生産的なアプリケーションを開発するための基本原則をリストします。



戦略的：

1）リソースを保存する必要があります。

2）アクションに対して即座に反応を示し、ユーザーとのフィードバックを維持する必要があります。

3）アプリケーションのパフォーマンスが主な目標です。 この作業を後で行うことなく、開発プロセスで生産性を常に最適化する必要があります。

4）実行時間を測定し、アプリケーションの進行状況を記録して分析し、コードの狭いセクション、イベントの発生、メモリ割り当て、オブジェクトの寿命を記録する必要があります。 測定されないものは最適化できません。



戦術：

1）不要なオブジェクトを作成しないでください。

2）可能であれば、メソッドを静的にします。

3）プロキシメソッドの代わりに直接フィールドアクセスを使用します。

4）定数には静的ファイナルを使用します。

5）通常の整数変数で十分な場合は、enumを使用しないでください。



3. Androidの主な機能



AndroidはLinuxに基づいています。 アプリケーションとカーネルの間には、ネイティブコードのAPIレイヤーとライブラリレイヤーがあります。 アプリケーションはJava仮想マシン（Dalvik仮想マシン）で実行されます。

Androidで多くのアプリケーションを実行できます。 ただし、そのうちの1つがメインであり、画面を占有します。 現在のアプリケーションから、前のアプリケーションに移動するか、新しいアプリケーションを開始できます。 閲覧履歴のあるブラウザのように見えます。



各ユーザーインターフェイス画面は、コード内のアクティビティクラスで表されます。 さまざまなアクティビティがプロセスに含まれています。 アクティビティはプロセスよりも長く生きることができます。 必要なすべての情報を保存して、アクティビティを一時停止および再開できます。







Androidは、Intentに基づいてアクションを記述するための特別なメカニズムを使用します。 アクションを実行する必要がある場合（電話をかける、手紙を送る、ウィンドウを表示する）、Intentが呼び出されます。



Androidには、バックグラウンドで必要なアクション（音楽の再生など）を実行するためのLinuxのデーモンに似たサービスも含まれています。

コンテンツプロバイダーは、コンテンツプロバイダーを使用してアプリケーション間でデータを交換します。



4.適用例-数独



数独ゲームの完成したドラフトを検討します （ Sudokuv4フォルダ内の完全なコード ）。



動作するにはAndroid SDKとEclipseが必要です。 すべてをインストールして起動する方法はここに書かれています 。



プロジェクトをEclipseにロードするには、次の一連のアクションを実行する必要があります。

1）プロジェクトをEclipseワークスペースの別のフォルダーに解凍します。

2）メニュー項目「ファイル」->「新規」->「Androidプロジェクト」を選択します。

3）[新しいAndroidプロジェクト]ダイアログで、[既存のソースからプロジェクトを作成する]オプションを選択します。

4）[場所]フィールドで、プロジェクトフォルダーへのパスを指定します。 次へをクリックします。



プログラムメニュー



ゲームメニューはres / layout / main.xmlで説明されています。 インターフェイスの説明は、XMLまたは視覚化されたインターフェイスとして編集できます。 切り替えるには、コンテンツ表示領域の下部にあるタブを使用します。











通常、コントロールはコンテナ内に含まれます。この場合、LinearLayoutです。 彼はすべての要素を単一の列に配置します。



資源



すべてのテキストラベル（android：text）はリソースからデータを取得することに注意してください。 たとえば、エントリandroid：text = "@ string / main_title"は、main_title（Android Sudoku）という名前のノードのres / values / string.xmlファイルでテキストを検索する必要があることを示します。 背景色もリソース（android：background = "@ color / background"）に含まれていますが、color.xmlファイル（＃3500ffff）に含まれています。 エディターでリソースファイルを開くと、エラーが発生する場合があります。 ただし、いつでもXMLを表示できます。











コードからアクセスする必要があるコントロールにはIDが必要です。 ボタンにはid（android：id = "@ + id / continue_button"）があるため、クリックハンドラーをボタンにアタッチできます。 プラス記号は、ボタンの場合、ファイル/gen/org.example.sudoku/R.javaに識別子を作成する必要があることを示します（public static final int continue_button = 0x7f0b000b;）。 このファイルは自動的に生成されるため、変更することはお勧めしません。 このファイルにはRクラスが含まれており、このクラスを介して、任意のインターフェイス要素やその他のリソースにアクセスできます。







ウィンドウ作成



プログラムに関する情報を含むウィンドウを作成することを検討してください。 このウィンドウのマークアップは/res/layout/about.xmlファイルにあります。 Activityクラスは、/ src / org.example.sudoku / About.javaファイルに記述されています。 アクティビティは、AndroidManifest.xmlファイルのマークアップに関連付けられています。 このファイルは、エディターまたはXMLとして表示できます。 エディターのさまざまなタブで、ファイルのさまざまなセクションを選択できます。 [アプリケーション]セクションには、アクティビティパラメータがあります。 テーマはandroid ：style / Theme.Dialogに設定されていることに注意してください。 これにより、ウィンドウスタイルはモーダルダイアログに似ています。







プログラムについての情報を含むウィンドウは、Aboutボタンを押すことで数独クラスから呼び出されます。 Sudokuクラスは、Clickイベントを処理するように記述されています（パブリッククラスのSudokuはActivityがOnClickListenerを実装します）。 public void onClick（View v）メソッドは、イベントをトリガーしたボタンを決定し、対応するコードが実行されます。 バージョン情報ウィンドウを表示するには、対応するインテントが呼び出されます。

case R.id.about_button:

Intent i = new Intent(this, About.class);

startActivity(i);

break;







イベントハンドラーは、特定のコントロールにインストールすることもできます。 たとえば、Keypadクラスでは、setListeners（）メソッドでクラスを作成すると、個々のボタンのハンドラーが設定されます。





簡単な対話



ユーザーには、難易度を選択する機会を与える必要があります。 これは、いくつかのオプションのいずれかを選択する必要がある小さなダイアログです。 このために別のIntentを作成する必要はなく、AlertDialogクラスを使用することを非常にうれしく思います。

新しいゲームを開始するプロセスを分析します。 ユーザーは[新しいゲーム]ボタンをクリックします。 クリックハンドラは、SudokuクラスメソッドonClickです。 次に、openNewGameDialogメソッドが呼び出され、難易度選択ダイアログが表示され、選択した難易度でゲームが開始されます。 このダイアログはAlertDialogクラスを使用して構築されます。 



 private void openNewGameDialog() { new AlertDialog.Builder(this) .setTitle(R.string.new_game_title) .setItems(R.array.difficulty, new DialogInterface.OnClickListener() { public void onClick(DialogInterface dialoginterface, int i) { startGame(i); } }) .show(); }




ダイアログの内容（ボタンのセット）は、文字列の配列R.array.difficultyから構築されることに注意してください。 ダイアログのボタンをクリックするためのハンドラーがすぐに割り当てられ、押されたボタンの数に応じて、startGameメソッドを呼び出して、指定されたレベルの複雑さで新しいゲームを起動します。



グラフィックス



ゲームクラスは、ゲームロジックを担当します。 ここでタスクがロードされ、勝ちの条件がチェックされます。 GameクラスはActivityですが、インターフェイスはXMLで記述されていませんが、コードによって作成されます。 onCreateメソッドはビューを作成します。



puzzleView = new PuzzleView(this);

setContentView(puzzleView);

puzzleView.requestFocus();







PazzleViewはViewから派生したクラスで、競技場を描画し、タッチイベント（onTouchEventメソッド）とキーストローク（onKeyDownメソッド）を処理します。



Androidでの描画プロセスを分析しましょう。 描画するには、onDrawメソッドをオーバーロードする必要があります。 メソッドは、描画が実行されるCanvasオブジェクトを受け取ります。 色を設定するには、Paintクラスのオブジェクトを作成します。 色はARGB形式で設定されます。 色はリソース（colors.xmlファイル）として保存するのが最適です。 ペイントは、色情報を保存するためのクラスだけではありません。 たとえば、テキストを描画する場合、テキストの塗り方、フォント、および配置に関する情報が含まれます。



Canvasには、グラフィックスを描画するための一連のメソッド（drawRect、drawLine、drawPath、drawTextなど）が含まれています。



グラフィックスを最適化するには、onDrawメソッド内でオブジェクトと不要な計算を作成しないようにすることをお勧めします（グラフィックス実装の検討例は最適ではありません）。







ミュージック



音楽を再生するには、MediaPlayerクラスを使用します。 ゲームの音楽がリソースに追加されました。 必要なファイルを/ res / rawフォルダー（WAV、AAC、MP3、WMA、AMR、OGG、MIDI形式）にコピーするだけです。

まず、MediaPlayerクラスのインスタンスを作成する必要があります。

mp = MediaPlayer.create（コンテキスト、リソース）;

ここで、コンテキストは通常​​、音楽の起動を開始するクラスであり、リソース-音楽を持つリソースの識別子です。 再生を制御するには、start、stop、およびreleaseメソッドを使用します。



ゲームでは、音楽はメインメニュー（数独クラスから起動）とゲームプロセス（ゲームクラスから起動）で再生されます。 再生を制御するために、Musicクラスが作成されました。 このクラスにはMediaPlayerの静的インスタンスが含まれているため、オーディオリソースの起動ごとに個別のプロジェクトを作成する必要がありません。



SudokuクラスとGameクラスでは、onResumeメソッドとonPauseメソッドが再定義されます。このメソッドでは、アクティビティが開始すると音楽が開始し、非アクティブにすると音楽が停止します。
