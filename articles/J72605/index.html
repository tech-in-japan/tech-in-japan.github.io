Mac OS X Snow Leopardレビュー

Mac OS Xの次のバージョンのリリースから多くの時間が経過したという事実にもかかわらず、私はまだそれについて書くことにしました。 私はこのオペレーティングシステムの比較的最近のユーザーであり、実際、これは私の最初のメジャーアップデートです。 このノートでは、私の意見では、システムの以前のバージョンに関連する長所と短所を強調します。 欠点よりも利点のほうがはるかに多く、実際には驚くことではありません。 だから、ポイントで。



注意！ 注意トラフィック。



  1. スペルチェッカー（スペルチェックを担当するソフトウェア層）と同様。 今、彼はロシア語と英語を同時に使用する方法を知っています。 確かに、作業にはかなりの不器用さがあります。 テキストがキリル文字とラテン文字の混合で構成され、最初の文字が優勢である場合、何らかの理由で英語の単語はチェックされません。 しかし、一般的に、辞書自体はそれと同じくらいお粗末なものであり、 Aspellを載せることは時々恥ずかしいことです。 我慢しながら。
  2. QuickTime Xは、マルチメディアプレーヤーのQuickTime 7に取って代わりました。バージョンの最初の桁の変更に伴い、プログラムのグラフィカルインターフェイス全体が変更されました。 私の意見では、それはより良い方向に変化し、今ではシステムの全体設計とより密接に調和しています。 注意し、速度を上げる必要があります。 含む、高解像度ビデオ（HD）の再生に現れます。 マイナスのうち、設定の完全な欠如は答えられるべきです。 当初、デフォルトでは、開いたファイルの再生を開始せず、自分で「再生」ボタンを押すことを常に強制していました。 幸いなことに、これは非表示の設定を使用して修正できます 。 それを今日追加できるのは、デスクトップでビデオファイルを再生する主な手段であるQuickTime XとPerianコーデックのセットです。
  3. 新しいオペレーティングシステムでは、スタッフファイルマネージャーがゼロから書き直されました。 私は彼について2つのことが好きでした。 まず、以前のように、ディレクトリに入るたびにマルチメディアファイルアイコンのインデックスが作成されなくなりました。 これにより、ファイルシステムの操作中にハードドライブの迷惑なクラッキングを聞くことがなくなりました。 まあ、そして最も重要なこととして、Finderはコンピューター全体だけでなく、現在のディレクトリ内でも検索することを学びました。 デフォルトではなく、真。 これを行うには、設定に移動して適切なモードを設定します。 以前は、 SpotlightとTerminal + Findのみを使用して検索しました。 今、ほとんどの場合、私はグラフィック手段で管理しています。 成長するか、劣化するか？ 冗談。 選択はいつでも歓迎です。
  4. キーボードレイアウト管理も改善されました。 多くの人が待ち望んでいたものが現れました。 たとえば、「ドキュメントごとに異なるレイアウトを許可する」オプションを使用して、単一のウィンドウ内でレイアウトを記憶できるようになりました。 以前は、この機能が必要な人はPunto Switcherをインストールしていました。 もちろん、私は彼らの一人ではありませんでした。 そして今、私はこの機能を使用していません。 これで、小数点を小数点として使用することもできます。 また、Cmd + Spaceキーを約1秒間押し続けると、デスクトップの中央にあるグラフィックインジケーターを確認できます。 なぜ誰がそれを必要としているかはわかりませんが、事実は残っています。 私の意見では、最も重要な問題はまだ解決されていません。 これは、レイアウトの切り替え時の2番目の遅延です。 個人的には、あまり気にしませんが、すばやく盲目的に書く人にとっては、これは重要なしきい値です。 また、別のキーの組み合わせを使用してレイアウトスイッチを割り当てる機会を実際に害することはありません。 GNU / Linuxを使用する場合、私は常にXorgパラメーターを設定してCaps Lockキーのみをオンにします。 とても快適でした。
  5. Image Captureの新しいバージョンは、Leopardにあったものとは大きく異なります。 インターフェイスは完全に書き直されています。 これで、システム内の他のプログラムの設計により近くなりました。 この機能は、さまざまなユーザビリティによって大幅に補完されます。 しかし、私は写真のインポートにApertureを使用しているため、新人のすべての喜びを完全に体験することはできませんでした。
  6. デスクトップ用の少し補足された壁紙のセット。 標準的な画像はあまり変更されていません。 コンセプトは同じで、再設計のみです。 原則として、これも悪くありません。 私が気に入ったいくつかの写真を以下で見ることができます。
  7. Exposéメカニズムに大幅な変更が加えられました。 Mac OS X 10.5で、比率に合わせて画面に収まるようにウィンドウが縮小された場合、10.6では比例関係は解消されませんでした。 Safariウィンドウは、理論的にはiChatロースターよりも小さくすることができます。 これはやや混乱を招き、目的のウィンドウに集中するのに時間がかかります。 私の意見では、グリッドの位置も論理的ではありません。 以前は、Exposéのウィンドウは、デスクトップ上で占めていた画面上のそれらの位置にできるだけ近くに配置されていました。 また、「Snow Leoprad」では、いくつかの固定グリッドに配置されているため、しばしば混乱を招き、適切なウィンドウを探すのにさらに時間を費やすことになります。 このツールの機能に関しては、多少拡張されています。 ここで正確に何が読めるか。 さて、ホバーするときの通常の減光の代わりに、水色のフレームが表示されます。 誰かが好き、誰かが好きではない。 実際、私にとってこれは特に重要ではありません。
  8. Snow Leopardのリリースは、32ビットシステムコンポーネントから64ビットシステムコンポーネントへの移行を示しました。 速度に大きな違いを感じたとは言えませんが、アプリケーションのメモリがより多く消費され始めました。 システムのインストール後、すべてのプロセスにステータス「Intel 64ビット」がマークされました。 2つを除くすべてがkernel_taskとiTunesです。 カーネルが32ビットモードでデフォルトでまだ動作する理由を説明する理由をここで説明します 。 iTunesに関しては、すべてがはるかに簡単です。 Carbonアプリケーション開発環境で書かれています。 ご存じのとおり、Appleはこのフレームワークのアプリケーションに64ビット互換性を提供していません。 代わりに、 Cocoaの使用を提案しています。 さて、iTunesが書き換えられるまで待ちます。
  9. iTunes 9版はMac OS X 10.6に同梱されていませんでしたが、私もそれについて書くことにしました。 まず、これは私のデスクトップ上のメインアプリケーションの1つです。 第二に、彼は「Snow Leopard」のリリース直後に8区を変更し、新しいシステムに固有の設計機能を採用しました。 iTunesにはリリースを完了する時間がなかったようです。 これは、第9ブランチの最初のバージョンで見られました。 マルチディスクアルバムを並べ替えるだけでグリッチの価値はありましたか。 幸いなことに、この更新プログラムは十分に迅速に公開され、現在では非常に実用的な製品です。 8回目と比較したイノベーションの中で、私が最も注目したのは、デザインのニュアンスです。 たとえば、グリッドの灰色の背景を白に変更し、ナビゲーションボタンの「膨らみ」を変更します。 また、ファイルシステム上のライブラリをより詳細に整理できることにも満足しています。 現在、ポッドキャスト、音楽、映画の以前のダンプはありませんが、すべてが自動的にグループ化されます。
  10. 古いアプリケーションとの互換性についても説明する価値があります。 予想どおり、この点ですべてが私たちが望むほどスムーズではありません。 もちろん、ここでの欠点はAppleではなく、アプリケーションをまだ更新できていないサードパーティのプログラム開発者です。 正直なところ、私は自宅のコンピューターでそれほど多くの追加ソフトウェアを使用していませんが、互換性の問題は免れませんでした。 これは主に、 Xmarksアプリケーションに影響し、Safariのブックマークをオンラインサーバーと同期します。 次に、3GモデムHuawei E220の標準ドライバーが機能しませんでした。 確かに、すべてがシンプルで、すぐにわかりました。 現在までに、私がインストールしたほとんどすべてのアプリケーションは、Mac OS X 10.6「Snow Leopard」との完全な互換性を誇っています。
