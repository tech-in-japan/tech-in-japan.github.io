DevExtreme：Visual StudioでクロスプラットフォームHTML5ビジネスアプリケーションをすばやく作成する

前の記事で、モバイル開発のフレームワークであるPhoneJSについて既に書いています。 今日は、モバイルプラットフォームを含む、開発者の生産性を向上させるために設計された別の製品についてお話します。



Visual Studio 2010以降のツールキットであるDevExtremeについてです 。



DevExtremeに含まれるもの：

  1. PhoneJSの上で言及されています。
  2. ChartJSデータを視覚化するためのJavaScriptライブラリ。
  3. Visual Studioのプロジェクトテンプレート。
  4. クロスプラットフォームアプリケーションをすばやく作成するためのウィザード。
  5. Visual Studioに統合されたビジュアルデザイナー。
  6. ブラウザでアプリケーションをデバッグするためのモバイルデバイスのシミュレータ。
  7. QRコードによるデバイスでの迅速な起動の手段
  8. アプリケーションをネイティブコンテナにパッケージ化するためのツール。


通常、ビジネスアプリケーションは、データベースとビジネスロジックのインターフェースです。 多くのWebフレームワーク（Ruby on Rails、Django、Yii、ASP.NET MVCなど）には、既存のデータに基づいてアプリケーションフレームワークを即座に生成するためのscaffoldツールが含まれています。



DevExtremeは、モバイルデバイス（iOS、Android、Windows Phone 8、Tizen、Microsoft Surfaceなど）と通常のWebブラウザーの両方で動作する、いわゆるマルチチャネルアプリケーションを作成する同様の機会を提供します。 これは特別なジェネレーターを使用して行われます。



注文を処理するための簡単なシステムを作成する例を使用して、彼の実際の作業を見てみましょう。



ODataサービスの作成



適切なアプリケーションは、フロントエンドとバックエンドで構成されます。 バックエンドは、シンサーバー（RESTサービスなど）の形でますます作成されています。 ODataプロトコルは .NET開発者の間で人気があるため、ウィザードはODataサービスによって返されたメタデータに基づいてコードを生成できます。



一般に、ウィザードを使用してアプリケーションを生成する場合、 CORSまたはJSONPをサポートする既製のODataサービスを使用できます。 この例では、 DevExtreme 13.1 WCF ODataサービスプロジェクトテンプレートを使用してサービスを作成します。これは、Entity Frameworkモデルより若干改善された標準サービスです（CORSとJSONP、 ImageStreamProvider



サポートを追加しImageStreamProvider



）。



そのため、新しいプロジェクトを作成します。 たとえば、CRM.DataServiceと呼びます。 ソリューション名には、単にCRMを指定します。







「データベースから生成」を選択します。







[新しい接続]をクリックします。







たとえば、よく知られているNorthwindを使用します。



必要なテーブルを選択します：カテゴリ、顧客、従業員、注文の詳細、注文、配送業者、サプライヤー。







できた すでにブラウザでサービスを開き（コンテキストメニューの[ブラウザで表示]）、すべてが正常に機能することを確認できます。







アプリケーションの生成に進みます。



アプリケーション作成



データを備えた既製のサービスができたので、 DevExtreme 13.1 Multi-Chanel Applicationテンプレートを使用して新しいプロジェクトを追加する必要があります。 [ファイル] > [ 追加] > [ 新しいプロジェクト]を選択します。







アプリケーションが動作する必要のあるプラットフォームを選択します。 すべてに注意します。



次に、事前定義されたレイアウトの1つを示します。 SlideOutにします。







次のステップで、ウィザードはデータソースの選択を提案します。 ODataサービスがアプリケーション自体と同じソリューション（当社のものなど）にある場合、データソースは自動的に指定されます。 何らかのリモートサービスを使用する場合は、URLを手動で指定する必要があります。



[検出]をクリックして接続を確認します。 これで、モデルとビューを生成するために必要なテーブルを選択できます。







[ビューの生成]列では、UIで使用する3つのエンティティのみをマークしたことに注意してください。 残りはデータを操作するためにのみ必要です。



「ビュー生成オプション」セクションのスイッチを使用すると、ビューの分離の程度を選択できます。 「生成されたビューをプラットフォームに依存しない共有プロジェクトに追加する」と指定すると、すべてのプラットフォームで共有ビューファイルが使用され、コードの重複が回避されます。



一方、異なるプラットフォームで外観をカスタマイズする場合は、プロジェクトごとに個別にファイルを生成するのが理にかなっています。 ただし、共有プロジェクトからプラットフォームプロジェクトにファイルをコピーすると、一部のビューは一般的で、一部は特定のものになります。



[完了]をクリックして、生成を完了します。



その結果、次のプロジェクトを取得できます。

  • CRM.Mobile
  • CRM.Desktop
  • CRM.Shared
  • CRM.Win8
  • CRM.Win8Phone


それらは同様のフォルダーとファイル構造を持っています。もう少し詳しく見てみましょう。



  • データ -共有プロジェクトにのみ存在し、エンティティ（製品、カテゴリなど）の監視可能なモデルが含まれます。
  • js-ライブラリ（jQuery、Knockout、PhoneJSなど）のファイル、
  • css-プラットフォームに応じてアプリケーションを着色するための組み込みテーマスタイル、
  • レイアウト -事前定義されたレイアウトのセット、
  • view-アプリケーション画面（各画面にマークアップファイルとコードファイル）


ルートには、インデックスファイルと構成ファイルがあります。



CRM.Mobile開始プロジェクトを作成して、 F5を押しましょう。



ブラウザーが起動し、シミュレーターでアプリケーションが開きます。 その中で、製品、サプライヤー、注文に関する情報を表示および編集できます。







結果のアプリケーションで使用される視覚要素（ウィジェット）について少し詳しく説明します。



データのリストを表示するには、 dxListが使用されます。 アイテムのテンプレート、グループ化されたデータの表示、およびスクロールして更新や読み込みを行うためのプルダウンなどのモバイルUIイディオムをサポートしています。



入力フィールドは、次のウィジェットで表されます。

  • dxTextBox-テキストデータの場合、
  • dxNumberBox-数字の場合、
  • dxSwitch-論理値を切り替える（true、false）
  • dxLookup-リストから選択したデータ入力。


ウィジェットの完全なリストはドキュメントにあります 。



デバッグ



特定のプラットフォームでのアプリケーションの動作を確認するために、シミュレーターのあるページの左側にパネルが用意されています。このパネルでは、例えばiOS7や日々予想されるTizenで 、アプリケーションのテーマとシミュレーターのスキンを変更できます。 AndroidおよびWindows Phone 8のテーマも利用できます。



トピック切り替えパネルの下には、クラウドプロキシサーバーを介してモバイルデバイス上のアプリケーションをデバッグするためのQRコードがあり、開発者のマシンへの「ピアシング」アクセスが可能です。



QRコードには、デバイスのブラウザーまたはCourierアプリケーション（ iTunesおよびGoogle Playで利用可能）で開くことができる特別なURLが含まれています。 Courierを介した起動は、機能的にはネイティブパッケージをビルドしてからスマートフォン/タブレットにインストールするのと似ていますが、時間を節約できます。



他のプラットフォーム用のアプリケーションはそれぞれのプロジェクトにあり、単純に起動します。CRM.Desktopはブラウザーを起動し、CRM.Win8はWindowsストアアプリケーションを全画面またはエミュレーターで起動し、CRM.Win8PhoneはWindows Phone 8エミュレーターで起動します（ただし、 SDKがインストールされています）。







そのため、コードを1行も書かずにアプリケーションフレームワークを取得しました。 もちろん、ビジネスアプリケーションの典型的なアプリケーションの外観はかなりシンプルですが、DevExtremeではUIを好みに合わせて変更およびカスタマイズできます。

ネイティブパッケージをビルドする


DevExtremeを使用してモバイルアプリケーションを開発する最後のコードは、ネイティブパッケージのアセンブリとデバイスへのインストールです。



DevExtremeでは、次のプラットフォーム用のパッケージをビルドできます。



  • iOS
  • Android
  • Windows Phone 8
  • ティゼン
  • Phonegapビルド






Androidプラットフォーム用にアプリをパックしましょう。 これを行うには、プロジェクト「CRM.Mobile」で右クリックして、表示されるメニューで「Build native packages ...」を選択する必要があります。 その後、ダイアログが開き、その助けを借りてネイティブパッケージを収集します。



Androidを選択：







次に、Android用の既存の開発者証明書を使用するか、新しい証明書を生成できます。 新しい証明書を作成するには、管理者権限でVisual Studioが実行されている必要があります。







これで、パッケージの構築を開始し、結果のAPKファイルをAndroidデバイスにインストールできます。







次に、「Build another package」をクリックして他のプラットフォーム用にビルドするか、「Finish」をクリックしてIDEに戻ります。
