デバイス構成ファイルの理論



1Cでは、データベースのリスト、プレーンテキスト、utf-8エンコーディングの拡張子が.cfgおよび.v8iの読み取り可能なファイルに関してすべてが整理されています。 そのため、おそらくご想像のとおり、ユーザーを中断することなく、通常のテキストエディターでネットワーク経由でファイルを開いて直接編集することなく、同じ操作を実行できます。



ユーザー側のファイルの場所


ユーザーのコンピューターでは、1C 8.2はデータベースリストのファイルをディレクトリに保存します。



Windows XPの場合：

すべてのユーザーのプロファイル：C：\ Documents and Settings \ All Users \ Application Data \ 1C \ 1CEstart \

ユーザープロファイル：C：\ Documents and Settings \％username％\ Application Data \ 1C \ 1CEstart \



Windows 7の場合：

すべてのユーザーのプロファイル：C：\ ProgramData \ 1C \ 1CEStart \

ユーザープロファイル：C：\ Users \％username％\ AppData \ Roaming \ 1C \ 1CEStart \



ユーザープロファイルの内容は、1CEStart.cfg、ibases.v8iの2つのファイルです。

すべてのユーザーのプロファイルディレクトリの内容は、1CEStart.cfgのみです。



起動時に、1CはユーザーファイルCに表示されるデータベースのリストを取得します。\ Users \％username％\ AppData \ Roaming \ 1C \ 1CEStart \ ibases.v8iが、最初にすべてのユーザーCのプロファイルから設定を読み取りますC：\ ProgramData \ 1C \ 1CEStart \ 1CEStart.cfg、次にユーザープロファイルCから：\ Users \％username％\ AppData \ Roaming \ 1C \ 1CEStart \ 1CEStart.cfg。ネットワーク上の構成データベースへのリンクがある場合は、このファイルに追加します。



1CEStart.cfgファイルの説明


すべてのユーザーのプロファイルで、構成ファイルC：\ ProgramData \ 1C \ 1CEStart \ 1CEStart.cfgの内容は次のとおりです。 

InstalledLocation=C:\Program Files (x86)\1cv82 InstallComponents=DESIGNERALLCLIENTS=1 THINCLIENTFILE=1 THINCLIENT=0 WEBSERVEREXT=0 SERVER=0 CONFREPOSSERVER=0 CONVERTER77=0 SERVERCLIENT=0 LANGUAGES=RU


どこで：

InstalledLocation



-1C：Enterpriseインストールが実行されたディレクトリの表示が含まれます。 デフォルトでは、この値はC：\ Program Files（x86）\ 1Cv82です。

CommonCfgLocation



共通構成ファイルのパスと名前を示します。 このパラメーターを持つ複数の行が許可されます。

CommonInfoBases



共通の情報ベースのリストでパスとファイル名（.v8i）を示します。

DistributiveLocation



自動インストール用の新しいバージョンが検索されるディレクトリの指示が含まれています。

InstallComponents



ローカル構成ファイル（1CEStart.cfg）には、コンポーネントをインストールするかどうかを示す記号が付いたインストール済みコンポーネントのリストが含まれています-1またはそうでない-0。



InstallComponentsパラメーターの次のコンポーネントが可能です。

DESIGNERALLCLIENTS



すべてのクライアントとコンフィギュレーDESIGNERALLCLIENTS



。

THINCLIENT



は、クライアントサーバーバージョンの作業用のシンクライアントです。

THINCLIENTFILE



は、ファイルTHINCLIENTFILE



を操作できるシンクライアントです。

SERVER



-1C：エンタープライズサーバー。 インストーラーがランチャーから起動する場合、サーバーはアプリケーションとしてインストールされます。

WEBSERVEREXT



-Webサーバーの拡張コンポーネント。

CONFREPOSSERVER



-1C：エンタープライズ構成ストレージサーバー。

SERVERCLIENT



-1C：エンタープライズサーバークラスターを管理するためのコンポーネント。

CONVERTER77



バージョン1Cからの情報データベースコンバーター：Enterprise 7.7。

LANGUAGES



インストール用のインターフェース言語のリスト。 複数の言語が指定されている場合、それらは「、」でリストされます。 例： LANGUAGES=RU,UK,BG







ユーザープロファイルでは、構成ファイルC：\ Users \％username％\ AppData \ Roaming \ 1C \ 1CEStart \ 1CEStart.cfgは最初は空です。 ただし、特定のユーザーに対して個別にいくつかの設定を行う必要がある場合は、その設定に書き込みます。ここでは、すべてではなく、設定キーの優先度が高くなります。 これは別の問題であり、頭を散らかすことはありません。



ibases.v8iファイルの説明


ユーザーのプロファイルにある2番目の重要なインフォベースファイルは、C：\ Users \％username％\ AppData \ Roaming \ 1C \ 1CEStart \ ibases.v8iです。 塩基の最終リストはそこに収集されます。 その内容の例： 

 [PhoneBook] Connect=Srvr="1C-server:1551";Ref="phonebook"; ID=34891493-907f-4783-8a37-3cbc092a989a OrderInList=16640 Folder=/ OrderInTree=16640 External=1 ClientConnectionSpeed=Normal App=ThinClient WA=1 Version=8.2 [ITIL] Connect=File="\\Dvc-server\it\Equipment\Rarus\ITIL"; ID=2829c91a-137f-457c-93fe-9e74c8ec5ee4 OrderInList=16384 Folder=/ OrderInTree=256 External=1 ClientConnectionSpeed=Normal App=Auto WA=1 Version=8.2.14.540


どこで：

[phonebook]



-ベース1Cの名前。 ロシア語の文字と英語の両方を使用できます。 これはユーザーに表示されます。 

 Connect=Srvr="1C-server:1551";Ref="phonebook"; -   . | | | | | | | |-     1. | | |-------------  ,    . | |---------------------  . |-----------------------------    : Srvr, File, ws - -.


ID=34891493-907f-4783-8a37-3cbc092a989a



自動生成された一意のデータベースコード。 2つのレコードのIDが同じ場合、これは1つのデータベースです。

OrderInList=16640



ベースがサブディレクトリなしの1つの一般リストで表される場合のベースのリスト内の順序。 ユーザーがすでにibases.v8iでこのデータベースで満たされている場合、ネットワークリストのこのパラメーターは純粋なibases.v8iでのみ同期され、ネットワーク上で変更されてもこのパラメーターは上書きされません。

Folder=/



-ベースリストのビューがツリーによって設定されている場合、ディレクトリツリー内の場所を設定します。 このパラメーターにはユーザー優先順位があり、ネットワーク構成で変更しても変更されません。

OrderInTree=16640



データベースのリストがサブディレクトリの階層として表示されるときのツリー内の順序。 このパラメーターにはユーザー優先度もあり、データベースが最初に追加されたときにのみ同期され、ローカルユーザーのみが変更する必要があります。

External=1



外部プラグイン構成レコードかどうか。 0の場合、ベースはこのファイルにのみ存在します。 この場合、このエントリはネットワークの.v8iデータベースリストファイルからインポートされます。 これは識別子です。これがネットワーク上にあるデータベースのリスト（共有）である場合、このパラメーターは構成ファイルから完全に削除できます。

ClientConnectionSpeed=Normal



クライアント接続速度。 オプションは「Nofmal」と「Low」です。 私の投稿がなくても論理は明確です。 このパラメーターは対話式であり、ネットワーク化されている場合、1Cが起動されるたびに同期されます。

App=Auto



クライアント接続のタイプ。 次の3つのタイプがあります。

-自動-サーバーによって決定されます。

-ThinClient-シンクライアント。

-ThickClientはシッククライアントです。

このパラメーターは対話式であり、1Cが起動されるたびに同期されます。

WA=1



このパラメーターは、システムがWindows認証を使用する必要があることを示します。 このパラメーターは対話式であり、1Cが起動されるたびに同期されます。

Version=8.2



このベースに使用されるバージョン。 使用するプラットフォームを完全に指定すると、たとえば、2番目のレコード-バージョン= 8.2.14.540のように、指定したプラットフォームが正確に使用されます。 このパラメーターは対話式であり、1Cが開始されるたびにネットワーク構成と同期します。

DefaultApp



データベース用に構成されていない場合のデフォルトのクライアント接続タイプ、およびDefaultVersion



データベース用の構成で指定されていない場合に使用されるDefaultVersion



バージョンなどのパラメーターもあります。 このパラメーターはカスタムであり、最初の起動時にクリーンなデータベースファイルに同期されます。 さらに、このパラメーターはローカルユーザーによって制御されます。



ネットワーク上の構成ファイルへのリンク


この全体の作業スキームの利点は、1Cメーカーがネットワーク上にある構成ファイルを見る機会を提供したことです。 これは編集により実装されます：

  • すべてのユーザーの構成ファイルC：\ ProgramData \ 1C \ 1CEStart \ 1CEStart.cfgコンピューターのすべてのユーザーのデータベースを表示する場合。
  • または、特定のユーザーの構成ファイルC：\ Users \％username％\ AppData \ Roaming \ 1C \ 1CEStart \ 1CEStart.cfgデータベースをコンピューター上の特定のユーザーにのみ表示する場合。


また、パラメーターを追加できます。

  • またはCommonCfgLocation=\\server\1C\config\bases.cfg



    共通構成ファイルのパスと名前を示します。 このパラメーターを持つ複数の行が許可されます。 拡張子のみが保持される場合、ファイル名は重要ではありません。
  • またはCommonInfoBases=\\server\1C\config\buh_bases.v8i



    パスとファイル名（.v8i）を一般的な情報ベースのリストとともに示します。 拡張子のみが保持される場合、ファイル名は重要ではありません。
  • または両方の組み合わせで、多くの行で。


ネットワーク上では、拡張子が.cfgおよび.v8iのファイル、および対応するコンテンツを配置する必要があることに注意してください。



機会を使用するという考え



アイデアは、最小限の繰り返しで編集することです。 データベース設定またはサーバー上のその場所が変更された場合、1箇所で一度レコードを修正すると、すべてのコンピューターで関連情報を受け取ります。

これを行うには、ネットワーク上でボールを作成する必要があります： \\server\1C\



。 このボールに少なくとも2つのディレクトリを作成します。

.. \ bases \

ディレクトリリスト： 

 \\server\1C\bases\.. .. base1.v8i base2.v8i base3.v8i base4.v8i


拡張子が.v8iのファイルは、このディレクトリに保存されます。 これらのファイルには、ファイルごとに1つのデータベースの設定のみが含まれています。 さらに、これらのパラメータ、この特定のデータベースにとって重要な設定のみを指定する必要があります。他のすべてはデフォルトで自動的に設定されます。 サンプルファイル： 

 [ ] Connect= ClientConnectionSpeed=Normal App=Auto WA=1 Version=8.2


次のように、 ID



パラメーターを使用しないでください。 1C開発者は、既存のデータベースから新しいコピーアンドペーストデータベースを作成するために使用します。 また、1つのIDを持つベースは競合します。

.. \グループ\

ディレクトリリスト： 

 \\server\1C\bases\.. .. buh.cfg fin.cfg managers.cfg


拡張子が.cfgのファイルは、このディレクトリに保存されます。 これらのファイルの内部には、ディレクトリ.. \ bases \のベースへのリンクがあります。 サンプルファイル： 

 CommonInfoBases=\\server\1C\bases\base1.v8i CommonInfoBases=\\server\1C\bases\base2.v8i


このディレクトリでは、ユーザーグループまたは特定のユーザーのデータベースの個々のリストを作成します。 ユーザーのオペレーティングシステムの構成ファイルで参照するのは、このグループのファイルです。

このスキームでは、非管理者向けにネットワーク上の1Cユーザーデータベースのリストの管理を行います。 1C開発者に構成ファイルを使用してネットワークディレクトリにアクセスさせ、必要に応じてプレイさせます。

データベースの設定を変更する必要がある場合は、\\ server \ 1C \ bases \ base.v8iファイルで一度だけ編集します。これはすべてのユーザーに影響します。 すべてのユーザーは、このファイルのデータベース情報を確認します。



結果



構成ファイルのこの構築の結果：

  • ベースの名前、場所、起動オプションを修正します。すべてを1か所で行うことができます。
  • データベース名の統一-データベースのすべてのユーザーは、1つの共通名で知っています。
  • ユーザー自身が構成設定を破ることはできません。 それらは彼のコンピューターには保存されません。
  • ユーザーのコンピューター上のデータベースのリストを管理するプロセスは、1Cスペシャリストに戻すことができます。
  • ユーザーがタイムゾーンよりもはるかに遅れてスリープしていて、コンピューターが現在オフになっている場合でも、構成ファイルを修正する機能。
