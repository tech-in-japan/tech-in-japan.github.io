CHEFaレシピ：HPE OneViewを使用したビジネスアプリケーション環境の自動展開

情報技術は、新しいITスタイルの製品およびサービスの不可欠な部分になりつつあり、ビジネスでは、新しいアプリケーションに必要なリソースをほぼ瞬時に受け取ることが期待されています。 ITリソースは、最新のアプリケーションの新しい要件のパラダイムで、コンピューティング、ストレージ、およびネットワークのノードの共通プールから自動的に割り当てられ、使用され、返され、再利用される必要があります。









このようなインフラストラクチャには、単一の管理システムの下に抽象化されたさまざまなタイプのハードウェアプラットフォームを含めることができます。 このアプローチは、コードとしてのインフラストラクチャと呼ばれます。 Hewlett Packard Enterpriseのビジョンは、Composable（Composed）Infrastructureに移行することです。これは、HPE OneViewマネージャーの管理下で、テンプレートとAPIを使用して、アプリケーションを展開、保守、制御する柔軟なリソースプールです。



Composable Infrastructureの概念を実装し、ビジネスアプリケーションの環境の展開を自動化するために、HPEはその専門知識を有名なアプリケーション構成マネージャーCHEF Softwareと組み合わせました。



CHEF（Chef）-RubyおよびErlangで記述された構成管理システム。ドメイン固有の言語を使用して構成を記述します。 複数のサーバーの構成とサポートのタスクを簡素化するために使用され、クラウドプラットフォームに統合して、現在の管理を自動化し、新しいサーバーをセットアップするプロセスを自動化できます。



CHEFユーザーは、サーバーアプリケーション（Apache、MySQL、Hadoopなど）とその設定を管理する方法の説明を含む特定の「レシピ」を作成します。



「レシピ」とは、インストールされたパッケージ、実行中のサービス、作成されたファイルなど、特定の時点に配置されるシステムのリソースの状態の説明です。 CHEFは、各システムリソースが正しく構成されていることを確認し、予想されるリソースを満たしていない場合はリソースの状態を修正しようとします。



レシピは、LinuxノードとWindowsの両方の管理に使用されます（最近のリリース）



HPE OneViewのCHEF展開ドライバーを使用すると、IT管理者はCHEFレシピを使用して、サーバー、ネットワーク接続、ストレージ、オペレーティングシステムとアプリケーションの展開など、物理環境の物理展開を自動化できます。 このような統合により、すべての手順を自動化することで、ビジネスの収益速度が向上し、展開の信頼性が向上します。









図1-CHEドライバーとHPE OneViewの相互作用図

高速で効率的なITインフラストラクチャを作成するには、組織は従来の物理システムを物理リソースと仮想リソースの柔軟なプールに変換する必要があります。



この変革により、組織は多くの場合、ベアメタル物理機器の展開を自動化するのが困難になります。 展開は常に複雑なプロセスです。 このタスクに適した機器にハードウェア、接続、OS、およびアプリケーションスタックをインストールして準備する必要があります。 これらのプロセスでは、多くの場合、手動の制御と監視が必要であり、時間がかかり、エラーにつながる可能性があります。



これが、Composableインフラストラクチャのアイデアの出番です。 このようなインフラストラクチャには、コンピューター、ストレージ、およびネットワークの柔軟なプールがあり、ソフトウェア定義のアプローチを使用して、その中のリソースが動的に割り当て、増加、減少、および再利用されます。 Hewlett Packard Enterpriseのソリューションポートフォリオでは、すべてのサーバー、ストレージ、およびデータネットワーク製品が、このようなソフトウェア定義のインフラストラクチャに統合されるように特別に設計されています。



たとえば、各サーバーにはiLOチップが搭載されているため、OSにエージェントをインストールすることなく、すべてのハードウェアコンポーネントの状態を収集し、サーバーとの対話を構成できます。 Hewlett Packard Enterpriseのソリューションポートフォリオのストレージおよびネットワークデバイスは、オープンAPI RESTful標準を理解しています。 ポートフォリオのハードウェアのすべての開発を結合し、データセンターのインフラストラクチャ全体を管理できる本格的なオーケストレーターを作成できる製品を作成することは残っていました。 このような製品は2013年10月にリリースされたOneViewになりました。



HPE OneView









図2-HPE OneView環境

HPE OneViewはフルサイクルのデータセンター管理ソフトウェア製品です：Hewlett Packard Enterprise、Brocade、一部のCiscoスイッチの製品で構築されたネットワーク、サーバー、ストレージシステムを管理するためのコネクターを使用し、既知のすべてのハイパーバイザーに統合し、それらを使用して仮想環境を展開できます。



VmwareまたはHyper-vの仮想マシンイメージの形式で提供され、これらのハイパーバイザーをサポートする任意の機器に展開できます。 HPE OneViewには、グラフィカルコンソールでコンポーネントを追加または削除することで、管理者の特定のグループに適応できる直感的で覚えやすいインターフェイスがあります。 さらに、HPE OneViewは、UPS、PDU（Schneider Electric、Eatonなど）、ラックなどのデータセンターインフラストラクチャと統合できるため、サーバールームで起こっていることの3次元画像を取得し、受信したデータに基づいてアプリケーションや仮想マシンの動きを制御できます。









図3-OneViewのデータセンターの温度チャート

管理者は、OneViewインターフェースでデバイスインタラクションマップを作成できます。









図4-サーバーアーキテクチャ要素の関係

HPE OneViewは、以前の5つのサーバーインフラストラクチャ管理製品（HP SIM、Insight Control、Onboard Administrator、VCM、HPSUM）全体を置き換えます。



バージョン2.0では、HPE OneViewの機能が拡張されました。同じブレードサーバー（同じプロセッサタイプ、メモリ、ネットワークアダプター）、およびLANアダプターやFCアダプターなど、CPUの世代が異なるブレードサーバーの世代が異なるコンパートメント間など。 さらに、このような移行は現在、1つのブレードバスケットのフレームワーク内だけでなく、複数のブレードバスケット間でも可能です。









図5-OneView 2.0の新しいプロファイル移行機能

HPE OneViewを使用すると、管理者はテンプレートとプロファイルを使用して、企業でソフトウェア定義のアーキテクチャまたはインフラストラクチャとしてのインフラストラクチャの概念を実装できます。



OneViewに付属のテンプレートは、さまざまなアプリケーションに合わせて、また各ケースの機器の仕様に基づいて簡単に調整および変更できます。



これらのテンプレートは、開発者のインフラストラクチャ展開時間を短縮し、IT担当者間の不整合を排除し、数分でアプリケーションのテストを開始できるため、DevOpsアプローチでの使用をお勧めします。

図6-OneViewでのプロファイルの作成

このシナリオでは、データセンターの特定の機器モデルに基づいて、プロファイル名が「Webサーバーテンプレート」に設定され、ブレードバスケット、ブレードサーバー、ネットワーク接続のパラメーターが入力されます。



[作成]をクリックして、プロファイルを保存します。 次に、CHEFとOneView統合ドライバーをインストールする必要があります。



OneViewのテンプレートを使用したOS展開

GitHub Webサイトから「chef-settings-Linux®」スクリプトをダウンロードします。 スクリプトで次の値を変更します。 



CHEF_PROXYSERVER: the proxy server address YUM_REPOSITORY_INFO: the yum repository information CHEF_CERT: the SSH certificate information to be copied into the authorization files LOGIN_DISABLED: should be set to YES/yes to disable user login and NO/no if LOGIN using username/password should not be disabled


このスクリプトは、OS展開パッケージ-OSビルドプランに組み込む必要があります。 OSビルドプランの作成の詳細については、 こちらとこちらをご覧ください 。



Cookbook CHEFを使用したアプリケーションスタック構成

図11-Webサーバーは稼働しており、動作する準備ができています！

HPE OneViewの展開エンジンは、5つのステップで構成されています。



1. HPE OneView REST API検索を使用して、HPE OneViewによって管理される使用可能なサーバーのプールから互換性のあるサーバーハードウェアを選択します。



2. HPE OneView REST APIを使用して新しいサーバープロファイルテンプレートを作成し、以下を割り当てます。



A.ファームウェアバージョン、BIOS、ローカルストレージ構成：HPE OneViewは、ビルトインHP Intelligent Provisioning（IP）、Linux環境のPXEブートを使用します。これは、iLOベースボード管理カード（BMC）プロセッサー、SmartStartに既に組み込まれています。



B.ネットワーク接続：HPE OneViewは、仮想接続ネットワークプラットフォームを使用して、ネットワーク接続プールからサーバーのネットワーク接続を動的に提示します。



C. SAN：HPE OneViewは、Brocade、Cisco、HPE NetworkingなどのHPE 3PARストレージおよびファイバーチャネルスイッチのSANの表示を自動化します。



3.物理リソースのプロビジョニングが完了するとすぐに、選択したOSが展開されます。



A.サーバープロファイルは、ネットワーク経由でインストールイメージをダウンロードするように構成されています。



B.サーバーの電源が入り、PXEブートが開始されます。



C. OneViewドライバーは、HPE OneView REST API（Insight Controlサーバープロビジョニング）を使用してOSインストールを起動します。



4.ネットワークパラメータが設定されている



A. IPアドレス、サブネット、ホスト名など さまざまなネットワークアダプターおよび仮想ネットワークアダプター用。複雑なネットワーク構成を作成できます。



5. CHEFは、デプロイされたOSの上にアプリケーションスタックをデプロイします



A. CHEFクライアントをサーバーにダウンロードします。その後、CHEFクライアントはCHEFサーバーに登録します。



B.新しいサーバーでCHEFクライアントを起動します。これにより、CHEFサーバーから必要なクックブックが取り出され、Webサーバーを構成するためのレシピが適用されます。
