OpenSceneGraph：ソースとHello Worldからビルド





はじめに



OpenSceneGraph （以降OSGと呼びます）は、C ++で記述されたオープンクロスプラットフォームフレームワークで、プログラマにOpenGLへのオブジェクトインターフェイスを提供するグラフィックエンジンを表します。 私たちの国では、このエンジンは特に人気がありません。Habréでさえ、 それについて多かれ少なかれまともな出版物を見ただけです。 OSGは海外で多く使用されています。たとえば、無料のFlightGearフライトシミュレータの基盤であり、 OpenMWと呼ばれるMorrowindゲームのオープンな実装があり、その開発もOgreエンジンからOSGに転送されます。 それに関するロシア語のドキュメントは消えていくほど小さく、英語のドキュメントの中で注目に値するのは開発者からの一連の本だけです： OpenSceneGraph 3.0。 初心者向けガイドおよびOpenSceneGraph 3.クックブック 。



それでも、このエンジンは次の理由で非常に興味深いものです。



  1. C ++でのオープンなクロスプラットフォーム実装。

  2. モジュラーアーキテクチャ。

  3. 組み込みのプラグインシステムによる拡張性。

  4. グラフィックデータのマルチスレッド処理とその実装のための組み込みツールの可能性

  5. スマートポインターメカニズムによる動的メモリ管理



Habrの読者にとって、このプロジェクトをより詳しく知ることは興味深いと思います。 ロシア語のOSG知識ベースの補充は不要ではありません。 このトピックに関して私が公開するすべての資料は、 OpenSceneGraph 3.0という本に基づいています。 初心者向けガイドですが、それらは翻訳ではなく、そこに提示されている資料の創造的な改訂版です。 このトピックに興味があるなら、カットをお願いします



マシンでOSGの最新バージョンを入手する唯一の確実な方法は、ソースからライブラリをビルドすることです。 Windows用の既存のバイナリインストーラーは、MS Visual C ++コンパイラーに焦点を当てています。 私にとって、私のプロジェクトでは、GCCコンパイラ、またはQtフレームワーク開発ツールの供給に含まれているMinGW32のバージョンを使用する必要があります。 必要なのは：



  1. MinGW32バージョン5.3コンパイラおよびQtCreator IDEを使用してインストールおよび構成されたQtフレームワーク

  2. Windows用Gitクライアント

  3. Windows用のCmakeユーティリティ



読者は、Qtプロジェクトで使用されるQtCreator IDEおよびqmakeビルドシステムに精通している必要があります。 さらに、読者はGitバージョン管理システムの使用の基本を理解しており、原則としてゼロ以外のプログラミングスキルを持っていることを前提としています。



1. OpenSceneGraphソースの取得



たとえば、パスD：\ OSGに沿って、OSGを構築するハードドライブにディレクトリを作成します。







このディレクトリに移動して、Githubの公式OSGリポジトリからソースを取得します 



D:\OSG> git clone https://github.com/openscenegraph/OpenSceneGraph.git

3.エンジンのデバッグバージョンとリリースバージョンのアセンブリとインストール



次に、cmdシェルを実行します。ショートカットは、スタート->プログラム-> Qt-> Qt 5.11.2-> Qt 5.11.2 for Desktop（MinGW 5.3.0 32bit）にあります。







実行中のコマンドラインセッションは、mingw32ビルドツールが機能するために必要な環境全体をセットアップします。 OSGソースディレクトリに移動します 



 C:\Qt\Qt5.11.2\5.11.2\mingw53_32>D: D:\> cd OSG\build-win32-debug


コマンドを与える 



 D:\OSG\build-win32-debug>cmake -G "MinGW Makefiles" -DCMAKE_INSTALL_PREFIX=E:\Apps\OSG -DCMAKE_BUILD_TYPE=DEBUG ../OpenSceneGraph


パラメーターの意味をさらに詳しく分析してみましょう。



  • -G "MinGW Makefiles"-mingw32-makeユーティリティ用のMakefileを生成する必要があることを示します

  • -DCMAKE_INSTALL_PREFIX = E：\ Apps \ OSG-OSGがインストールされるパスを設定します

  • -DCMAKE_BUILD_TYPE = DEBUG-エンジンのデバッグバージョンをビルドする必要があることを示します。

4.環境変数の設定



インストール後のOSGライブラリの場所は任意です。特定のユーザーの希望と、コンピューターにファイルを配置する能力によって決まります。 同時に、データを使用する特定のプロジェクトを設定する場合、ライブラリは特定の統合を必要とし、ライブラリの特定の場所から抽象化します。



ライブラリ、ヘッダーファイル、OSGプラグインへのパスを示すいくつかのシステム環境変数を作成しましょう。 私の例では、このようになります







スクリーンショットで名前が赤丸で囲まれている変数を作成する必要があります。 変数を作成した後、特に開発ツールで変数を表示するには、QtCreatorは少なくともシステムにログインする（現在のユーザーとしてログアウトしてログオンする）か、場合によってはシステムを再起動する必要があります（これはWindowsです！）



その後、コンピューターにOSGをインストールする手順は完了したと見なすことができます。

プロジェクトが組み立てられるOSに応じて、ライブラリディレクトリとOSGヘッダーファイルへのパスを示す変数を定義します。 ここで、OSG_BIN_PATHおよびOSG_INCLUDE_PATH環境変数が便利になりました。現在、OSGライブラリがインストールされている場所は関係ありません。 自分のコンピューターでこのプロジェクトを使用したい人はだれでも、ビルドスクリプトを編集せずに適切な環境変数をシステムに書き込むだけです。 



 CONFIG(debug, debug|release) { TARGET = $$join(TARGET,,,_d) LIBS += -L$$OSG_LIB_DIRECTORY -losgd LIBS += -L$$OSG_LIB_DIRECTORY -losgViewerd LIBS += -L$$OSG_LIB_DIRECTORY -losgDBd LIBS += -L$$OSG_LIB_DIRECTORY -lOpenThreadsd } else { LIBS += -L$$OSG_LIB_DIRECTORY -losg LIBS += -L$$OSG_LIB_DIRECTORY -losgViewer LIBS += -L$$OSG_LIB_DIRECTORY -losgDB LIBS += -L$$OSG_LIB_DIRECTORY -lOpenThreads }


UNIXライクなOSでビルドするスクリプトを書く 



 unix { CONFIG(debug, debug|release) { TARGET = $$join(TARGET,,,_d) LIBS += -losgd LIBS += -losgViewerd LIBS += -losgDBd LIBS += -lOpenThreadsd } else { LIBS += -losg LIBS += -losgViewer LIBS += -losgDB LIBS += -lOpenThreads } }


ここでは、実行可能ファイルの名前を設定し、さまざまなビルドオプション（デバッグとリリースの両方）に対してプログラムでコンパイルする必要があるライブラリを指定します。 OSGデバッグライブラリには、ファイル名の後に接尾辞「d」が付いています。 デバッグオプションとリリースバージョンを区別するために、プロジェクトの実行可能ファイルに接尾辞「_d」を追加します。 



 INCLUDEPATH += $$OSG_INCLUDE_DIRECTORY INCLUDEPATH += ./include HEADERS += $$files(./include/*.h) SOURCES += $$files(./src/*.cpp)


最後に、ヘッダーファイルとプロジェクトツリーに含まれるファイルの検索パスを決定します。 include /ディレクトリに空のmain.hファイルを作成し、src /ディレクトリにmain.cppファイルを作成します。 このプロジェクトをQtCreatorで開き、スクリーンショットに示すように構成します

 #include "main.h" int main(int argc, char *argv[]) { (void) argc; (void) argv; osg::ref_ptr<osg::Node> root = osgDB::readNodeFile("../data/cessna.osg"); osgViewer::Viewer viewer; viewer.setSceneData(root.get()); return viewer.run(); }

コードの最初の2行

 viewer.setSceneData(root.get());
