ターミナルサーバーでのスキャンは無料です

職場では、数人の従業員をターミナルサーバーに転送するタスクを設定して、ローカルスキャナーからのスキャンを含め、すべてがターミナルサーバーで行われるようにしました。 ユーザーはWindows上で実行されます。 USBを転送することは可能ですが、このスキャナーモデルにはWindows Server用のドライバーがありませんでした。



Scan over RDPでネットワークを検索し、Scan Redirector RDP Edition、TSX Scanなどのプログラムを見つけました。この地域では1ライセンスで数百または2ドルを必要とし、800ドルでさえ見ました。



ライセンスのないソフトウェアは好きではないので、自分で実装することにしました。







リモートスキャナーを管理するためのメインページが表示されます。 私のソリューションのプラスは、ターミナルサーバー（またはスキャンする他のコンピューター）に何かをインストールする必要はなく、ブラウザーのみが必要なことです。



[新しいJPGスキャン]をクリックすると、次のように表示されます。







それから：







写真は右ボタンで保存できます。 [新しいPDFスキャン]をクリックします。







次のページで[PDFをスキャン]を数回クリックします。







写真をクリックすると、全画面で表示されます。







[ページの削除2]をクリックします。

[画像の参照]をクリックします。







ここでは、以前にスキャンされたすべてのものを見ることができます。 どういうわけか動作します。 TWAIN互換スキャナーはすべてサポートされています。 Chromeでテスト済み。



私は開発経験がほとんどないので、コードはおそらくすべての最悪の伝統で書かれています。 コメントをキックします。



セキュリティは最小限です。部外者がスキャナーを使用するのを防ぐために、ファイルはscan57578.phpと呼ばれます（名前は別のものに変更できます）。 そしてindex.phpに「アクセス拒否」と書いてください。



スキャナーを搭載したコンピューターに何をインストールする必要がありますか？



ご想像のとおり、Webサーバーが必要です。 Denwerを使用しましたが、この場合はスズメ大砲ですが、PHPをサポートしている他のものを使用できます。 この場合、デンバーパス（ドライブZ：など）を使用するため、それに応じてコードを調整する必要があります。



他に必要なもの：コマンドラインから実際にスキャンするCmdTwainプログラム、およびコンソールユーティリティjpeg2pfg 。 CmdTwainをインストールし、CmdTwain \ Start from Sourceを実行して、スキャナーを選択します。 デンバーをインストールし、デフォルトですべてに同意します。



次に、ディスクZで、php.iniを見つけてパラメーターmax_execution_time = 130を設定する必要があります。スキャナーでの1回のスキャンにかかる時間を測定して、値を選択します。 次に、Denwerを再起動します。 デスクトップからデンバーの開始ショートカットを取得し、起動時にスローします。 コンピューターに静的IPを設定します。



Z：\ homeフォルダーに、コンピューターのIPと同じ名前のフォルダーを作成します。 私の場合、フォルダの名前は10.74.0.98です。 その中にwwwフォルダーを作成し、その中にスキャンが置かれるフォルダーを作成します。 私の場合、スキャンと呼ばれます。 フルパスは次のようになります：Z：\ home \ 10.74.0.98 \ www \ scans。



Webサーバーが構成されます。 次に、Z：\ jpeg2pdfフォルダーを作成します。 上記のリンクからダウンロードしたjpeg2pdf.exeをコピーします。 また、次の内容のmakepdf.batファイルを作成します。 



@echo off z: cd z:\home\10.74.0.98\www\scans\%1 z:\jpeg2pdf\jpeg2pdf.exe -z none *.jpg -o %1.pdf move *.pdf ..
