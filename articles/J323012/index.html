Kotlin 1.1をご覧ください：JavaScript、コルーチンなど

Kotlinプログラミング言語の新しいバージョンであるKotlin 1.1をご紹介します。









私たちの目標は、最新のアプリケーションのすべてのコンポーネントを効果的に記述できる、表現力豊かな静的型付け言語を作成することです。 本日のリリースでは、この方向に2つの重要なステップを踏みます。









まず、JavaScriptのサポートは実験的なものではなくなりました。 言語のすべての機能が利用可能です。標準ライブラリの重要な部分であり、もちろん、JavaScriptコードと対話する機能もあります。 これで、アプリケーションのクライアント部分もKotlinに移行できるようになり、Reactなどの通常のライブラリとフレームワークを放棄する必要がなくなりました。







第二に、コルチンのサポートがあります。コルチンは、スレッドの軽量類似物であり、アプリケーションを適切にスケーリングし、1台のマシンで大きな負荷をサポートできます。 さらに、コルーチンは非同期コードの記述を大幅に簡素化します。これは、すべてのプラットフォームでのUIの「応答性」にとって重要です。







これらの変更の詳細については、以下で説明します。 他の革新には、 型エイリアス 、 バインドされた呼び出し可能な参照 、 ラムダでの破壊が含まれます。 興味深い変更の完全なリストは、 新機能ページにあります（例を実行することに注意してください！）。







コルーチン（コルーチン）



Kotlinのコルーチンにより、非同期コードを通常の同期コードと同じくらい簡単に作成できます。







非同期コードは大きくなってきており、それを書くのは非常に難しいことがよくあります。 この作業を容易にするために、Kotlin言語レベルのコルーチンサポートを追加しました。これには、1つのプリミティブ（ サスペンド関数）のみが含まれます。 このような関数の実行は、スレッドをブロックせずに一時停止し、後で再開できます。 さらに、実行の中断と再開のロジックに対する制御は、ライブラリの作成者に完全に属し、スレッドの軽量な代替としてコルーチンを使用できます。 現在、費用のかかるフローのブロックの代わりに、コルーチンのほぼ無料のサスペンションを使用できます。







言語の構文にはほとんど追加されていません（1つの修飾子）が、ライブラリにはアクティビティの巨大なスコープが与えられています。 たとえば、 kotlinx.coroutinesプロジェクトでは、Rx、CompletableFuture、NIO、JavaFx、およびSwingのサポートを既に実装しています。 このようなライブラリーは、AndroidおよびJavaScript用に作成できます（予定されています）。 他のプログラミング言語に組み込まれているコンストラクトでさえ、ライブラリのKotlinで実装できます。たとえば、ジェネレーター/ Pythonからの出力、チャネル/ Goからの選択、またはC＃からの非同期/待機： 







// runs the code in the background thread pool fun asyncOverlay() = async(CommonPool) { // start two async operations val original = asyncLoadImage("original") val overlay = asyncLoadImage("overlay") // and then apply overlay to both results applyOverlay(original.await(), overlay.await()) } // launches new coroutine in UI context launch(UI) { // wait for async overlay to complete val image = asyncOverlay().await() // and then show it in UI showImage(image) }


ドキュメントの詳細







重要なお知らせ 。 コルーチンが提供するすべての利点にもかかわらず、その設計はまだ比較的新しいものです。 したがって、すべてを正しく実行したことを確認するために、実際に集中的に検証する必要があります。 したがって、1.1のコルーチンは「実験的」 フラグの下に出てきます。 言語コンストラクトが変更される可能性は低いと想定していますが、Kotlin 1.2では、コルーチン関連のAPIへの変更が可能です。







JavaScriptサポート



上で述べたように、Kotlinのすべての機能は、JVM / AndroidとJavaScriptの両方で同等に利用できるようになりました（リフレクションを除き、現在取り組んでいます）。 これにより、JetBrains内の一部の製品で成功しているKotlinで完全にWebアプリケーションを開発できます。 近い将来、関連するドキュメントと手順を公開する予定です。







JavaScript用のKotlinは、JavaScriptコードと対話するための動的タイプをサポートしています。 人気のあるライブラリへの型付きアクセスの場合、 DefinitelyTypedのヘッダーでts2ktコンバーターを使用できます。

ツールキット



中間リリースでは、言語の構文に関係のない新しい機能をリリースしようとしています。 Kotlin 1.0以降に登場した最も印象的なものは次のとおりです。







  • すべての主要なIDEのプラグイン：IntelliJ IDEA、Android Studio、Eclipse、NetBeans
  • IntelliJ IDEAおよびGradleでのインクリメンタルコンパイル
  • Spring、JPA、Mockitoとの統合を改善するコンパイラプラグイン（デフォルトで拡張可能なクラスと空のコンストラクターの自動生成）
  • kapt-注釈ツール
  • Androidプロジェクトに固有のコードチェック
  • そしてもちろん、多くの多くの新しい診断、チェック、クイックフィックス、リファクタリング、およびオートコンプリートの改善がIDEに追加されました


ただし、バージョン1.1.xのツールキットの改善は既に停止しており、既に準備中です。

次は何ですか



Kotlinを真にユニバーサルなフルスタック言語にするために、複数のプラットフォームの共通コードを一度にコンパイルするための言語とツールのサポートに取り組んでいます。 これにより、クライアント側とサーバー側の両方でコードが使用されるモジュールを作成できます。 また、サードパーティのJavaScriptライブラリのツールとサポートを改善し続けています。 JavaScriptのインクリメンタルコンパイルが進行中です。バージョン1.1.xのいずれかでリリースします。







Java 9のサポートは、このプラットフォームの公式バージョンのリリース（この夏）までに表示されます。







コルーチンに関する多くのフィードバックを期待しており、設計、パフォーマンス、およびライブラリが利用可能になったときにライブラリの改善に努めます。







しかし、概して、次のリリースでは、既存の言語機能のツールの品質、パフォーマンスの向上、インフラストラクチャの拡大、バグの修正に主に焦点を当てます。 Kotlinは徐々に多くのプロジェクトのキーテクノロジーになりつつあり、それに応じて成長に対応しています。







PSすべての主要プラットフォームでのKotlinの配布は、当社にとって戦略的な方向性です。 サーバー、デスクトップ、Androidデバイス、ブラウザーで既に実行できます。 近い将来、「仮想マシンのない世界」をカバーし、iOSや組み込みシステムなどのプラットフォームをサポートする予定です。 現在、Kotlin Nativeの優れたチームがJetBrainsで取り組んでおり、比較的短時間で何か面白いものを見せたいと考えています。







試し方



いつものように、ブラウザで直接Kotlinを試すことができます： try.kotlinlang.org

Maven / Gradle ：1.1.0をコンパイラと標準ライブラリのバージョン番号として使用します。

IntelliJ IDEA ：Kotlin 1.1のサポートは2017.1に既に組み込まれています。以前のバージョンでは、プラグインをバージョン1.1に更新する必要があります。

Android Studio ： プラグインマネージャーを使用してプラグインをインストールまたは更新します。

Eclipse ： マーケットプレイスで新しいプラグインを利用できます。

コマンドラインコンパイラは、Githubのリリースページから入手できます。







互換性 Kotlin 1.1では、言語と標準ライブラリは完全に後方互換性があります （モジュロバグ）。 何かが1.0の下で構築および開始された場合、1.1でも機能し続けます。 大規模なチームの段階的な移行を容易にするために、1.1のすべての新しい言語機能を無効にする特別なコンパイラキーを提供しています。 このドキュメントでは、移行のすべての微妙な点について詳しく説明しました。
