ロシアとアメリカの.NET開発者のスキル、違いは何ですか？





「.NET Developer Skills」という記事は、労働市場におけるスキルの需要に関する初めての分析的研究であり、私の職業の専門家に近いロシア市場のニーズを示しました。 この記事では、外国市場との比較を紹介します。 彼らは、私たちのIT市場全体、特に私たちが使用している技術は西側に数年遅れていると言っています。 「海外」を一緒に見て、これがそうであるかどうか、そして私たちが学ぶべきことがあるかどうかを理解してみましょう。

分析準備



以前の分析では、約300の手動の欠員を処理する必要がありましたが、これは非常に面倒で日常的な作業でした。 したがって、今では新しい研究の結果だけでなく、高度なツールも手にしています！ 空室のテキストをターゲットサイトからダウンロードし、個々の単語やフレーズに解析してから、特別な辞書を使用してスキルに変換するユーティリティを実装しました。 必要に応じて、さまざまなジョブサイトに新しいアダプターを追加し、プログラムをトレーニングして、空席で見つかったスキルを比較できるようにすることで、ユーティリティをさらに開発する予定です。



今回、新しいツールは500を超える求人をマスターするのに役立ちました。そのうちの430の場合、専門の.NET開発者を探していましたが、70は.NETに間接的にのみ関連しています。 合計約15,000語と550種類のスキルが処理されました。



Stack Overflow CareersとMonster.comの人気リソースから外国人の欠員を取得しました。



私をテストして独自の分析を準備する場合は、 OneDrive / USA-net developer.xlsxというExcelファイルで中間の「生の」統計データを見つけることができます。



リハーサル











この研究の結果については、SkillsWikiカンファレンス「ロシアと海外の雇用主の目を通して.NET開発者」で初めて話しました。 次に、計算と結論に関する最初のフィードバックを収集しました。 特に、聴取者はこの研究の結果に興味がありましたが、ロシアとアメリカの比較についてさらに異なるセクションと結論を得たいと考えていました。 ちなみに、私はマネージャーに関する同様の研究についても尋ねられました。これは私の将来の研究のトピックの1つです。



さて、私はあなたを失望させず、あなたの好奇心を満足させないことを望みます、始めましょう！



スキル評価



前と同じように、最初のステップは、労働市場でトップ10の最も人気のあるスキルを知ることです。 ランキングでの認識を容易にするために、すべてのスキルは、テクノロジー、プロフェッショナル（ハード）、および一般的な能力（ソフトスキル）の3つのグループに分けられます。



技術評価











ご覧のとおり、技術スキルの人気の違いはわずかであり、給与については言えません。 ロシアの左側では、給与は1か月あたりのルーブルで示され、米国の右側では、1年あたりのドルで示されます。つまり、職務明細書で示されているとおりです。 米国ASP.NET開発者の収入をルーブルに再計算するために急がないでください。 私はすでにあなたのためにこれをやった：それは現在の為替レートで月に約380,000ルーブルであることが判明した。 しかし真剣に、直接比較はここではあまり適切ではありません。 まず、さまざまな税法：たとえば、米国では、支払いが収入の3分の1を占めることがあります。 第二に、完全な比較のために、一般的な生活水準を考慮する必要があります。これは別の大きなトピックです。 したがって、私は、1つの国の枠組み内でのみ、給与を相互に比較することを提案します。



スキルの需要（人気）の評価に戻りましょう。 それにもかかわらず、わずかな不一致がそこに記録されます。



  • Windows Formsの経験は、米国のトップ10のスキルには含まれていません。そのような要件は、空室の4％にしか見られないためです。 米国市場の統計では、この技術が死にゆくものとして分類されていることが確認されています。
  • ASP.NET MVCの知識は、求人のわずか56％で必要です。 これとWindows Formsの人気の低さを考慮すると、疑問が生じます。米国では、フロントエンド開発者は全体としてそれほど必要ではないのでしょうか、それとも他のテクノロジーに占有されているのでしょうか？ 合計で、ASP.NET、ASP.NET MVC、およびWindows Formsは空席の64％で見つかります。JavaScriptまたはそのフレームワークについては既に76％が言及されています。 残りの24％のテクノロジーのランキングは、それらの中で最も人気のあるスキルが、テストだけでなくTransact-SQLとSQL Serverの知識であることを示しています。 オファーの選択的レビューは、これらが開発チームのリーダー、インフラ開発者、バックエンドの欠員であることを示しています。
  • 米国では、エンティティフレームワークは空室のわずか8.5％でしか見られないため、トップ10には入りませんでした。 他のすべてのデータアクセステクノロジーに遭遇しても、その割合は12.5％にほとんど達しません。 おそらく、米国の空室では、雇用主は何らかの理由で特定のORM実装を明確にすることを重要とは考えていません。 ただし、空室の半分以上でSQL Serverの知識が必要です。
  • また、JavaScriptフレームワークについても言及する必要があります。JavaScriptフレームワークは、勢いを増しており、空席の10分の1ごとに見られるように、すでに米国のトップ10に達しました。 Angular.jsが最も一般的であり（29の空室で言及されています）、backbone.js、knockout.js、node.jsがほぼ同じ人気を誇っています。

スキル評価











プロのスキルの要件では、違いははるかに鮮明です。



まず、ソフトウェアコードだけでなく最終製品の品質を向上させるスキルの要件が豊富であることはすぐに明らかになります。特に、テストスキル（自動化だけでなく手動）、ユニットテスト、TDD、コードレビューなどです。 これらのスキルはすべて、ロシアの開発者に対する雇用主の要件では非常にまれです。 米国全体では、開発とテストははるかに統合されており、これらの統計の例だけでなく、大企業からも見ることができます。 たとえば、これは、開発者自身のアプリケーションの品質に対する責任について話した新しいCEOのSatya Nadellaのイニシアチブで、マイクロソフトのテスターの大幅な削減に関する昨年のニュースです。



第二に、アジャイルの方法論とチームでこれらの原則に取り組む能力を理解することに重要な役割が与えられます。 米国では、アジャイルはトレンドセッターから作業ツールに移行しました。



また、米国では他の人のコードを読むことにあまり注意が払われていないという印象を受けました。



一般的な能力評価











一般的なスキルのランキングでは、違いが最も重要です。 アメリカでは英語の知識を必要としないのは論理的です。 それでも、かなりの違いがあります。



  • 米国の開発者のコ​​ミュニケーションスキルは、空席が8％しかなく、空席の3％ごとに必要であり、4％のケースであなたの考えを明確に表現できるため、主要な場所を占めています。
  • また、アメリカの開発者には、「期限の設定」や「詳細志向」に関連する「問題解決」などの重要なスキルが求められます。 これにより、開発者はコードだけでなく、技術的な方法だけでなく、ビジネス上の問題を解決し、ユーザーとそのニーズに集中できる必要があることを理解できます。


一般に、米国市場は、問題の知識、アーキテクチャを設計する能力、さまざまなツールとアプローチの知識、開発された機能のテストを設計して直接テストする能力を持つ技術専門家だけでなく、技術的専門知識を持つコンサルタントのようなものにも向けられています。 ロシアでは、これは非常にまれです。原則として、これらは2つの異なる職業です。

この評価の目的は、最も給与の高いスキルや人気のあるスキルを特定することではなく、市場で平均して最高の収入を得るスキルを特定することです。 それは主に職業に就いているだけで、すべての需要の最初に取得し、すでに2番目に高い収入を得たい人によって主に導かれるべきです。



たとえば、最初のグラフから、ASP.NETと米国とロシアのWPF開発者の給与はほぼ同じレベルであることがわかりますが、その人気の大きな違いにより、ASP.NETはWPFよりもはるかに流動的な投資になります。



結果を分析すると、スキルの多くは特定の給与範囲内でのみ請求でき、給与の増加に応じて徐々に需要が増加する、またはその逆の仮説が現れました。 各給与オファーのスキルの人気をプロットしてみましょう。 以下、技術的なスキルを使用します。残りのスキルについては統計がはるかに小さく、相関関係を追跡することが難しいためです。











特定の給与オファーには欠員が少なすぎるため、このようなスケジュールを分析することは困難です。そのため、失敗を避けるために給与範囲のデータを集計し、同時に最大および最小給与のスキルの人気の表示をグラフから削除します。











すでにはるかに優れていますが、多くのグラフが空室数のグラフと同様に正規分布に似ていることがわかります。











したがって、最初に別の手順を実行する価値があります。空席数をスキルが発生する空席率に変換します。











110,000の範囲ではWPFとXML / XSLTの使用に顕著なバイアスがあり、SQL ServerとHTMLテクノロジーのシェアが減少していることを除いて、強力なバーストはありませんので、線形トレンドに注意する必要があります。











どんな価格カテゴリーでも要件が安定しているスキルがあることは注目に値します-これらはASP.NETとHTML / JavaScript / CSSです。 給与の増加に伴い需要が減少するスキルグループには、SQL Server / Transact-SQL、JavaScriptフレームワーク、WPFがあります。 そして、人気を集めているスキル：WCF、Webサービス、SOAP、XML / XSLT、IIS、およびソース管理はほとんどありません。



グラフによると、特定の時点でスキルが人気を獲得したり失ったりすることが顕著であるため、各範囲で最も人気のあるスキルも見つける必要があります。

あとがき



2015年8月4日



同僚、親愛なる、もう一度私はすべてを歓迎します！



ここでのあなたのコメント、および他のリソースに関する同様の議論により、私はこの記事に小さな追加を書くよう促しました。



私が見たように、記事を読んだ後、多くはこれがHRからのレビューであり、したがって職業やその他の詳細を理解せずに非常に正式に行われたという結論に達しましたが、研究は客観的であると主張しています。 これらの神話を払拭したい。



  1. 私の友人はレビューの準備を手伝ってくれました。私たちは全員、有名な大規模なIT企業で働いており、それぞれが.NET開発に8年以上直接取り組んでいます。
  2. スキルの需要に関する唯一の大量アクセス可能な情報源である空席を可能な限り効率的に処理しようとしました。 多くのキーワードが1つのスキルにまとめられ、一部は完全に省略され、約13,000の単語が手動で処理され、結果の統計の処理はカウントされませんでした。 私の意見では、記事で提示されたスキルの評価は標準ではありませんが、予期しないことに注意を向けたり、仮定の正確さをもう一度確立することができます。


そして、これは私たちのスキルに関する考えの一つに過ぎません。 たとえば、次のスキルの需要を確認できます。



  1. 専門家と企業。 このようなスキルレビューの例を次に示します。

    • Sijin Josephは開発者のスキルマトリックスを説明しようとしました： sijinjoseph.com/programmer-competency-matrix;
    • インタビューの質問に対するScott Hanselmanの意見： www.hanselman.com/blog/WhatGreatNETDevelopersOughtToKnowMoreNETInterviewQuestions.aspx;
    • 学生向け.NET開発者開発マップ： edu.cbsystematics.com/en/education/roadmap
  2. 州。 すでにNARKについて聞いたことがありますか？ いいえ、これは冗談ではありませんが、 National Qualifications Development Agencyのポータルでは、プログラマ向けのプロフェッショナルな標準を見つけることができます。 ご覧になることをお勧めします。国営企業の開発者には特に近いはずです。
