Windows上のNode.js（パフォーマンステストあり）





この記事では、初心者のWebプログラマーとNode.jsの学習を考えている人々にとって最も重要な質問、つまり以下を検討します。

  • Node.js作業環境をコンピューターにインストールする方法。
  • コードの書き方。
  • デバッグ方法。
  • リモートサーバーで発生したことを展開する方法。
そして、記事の最後にあるパフォーマンステストは、実際にはNode.jsを学習する必要がある理由についての質問への回答を提供するでしょう。



Node.jsは、ネットワークアプリケーションを構築するためのイベント駆動型Javascriptフレームワークです。 主なアイデアは、コードの実行中は何もブロックされないということです。データの転送、ユーザーの入力、接続の確立など、何かを待機する操作はありません。 すべては、同期操作が待機している開始時に発生するイベントに基づいて構築されます。 これにより、古い同期システムに比べて、時には数十倍の重要なパフォーマンス上の利点が得られます。 2011年11月にリリースされたバージョン0.6.0以降、Node.js for Windowsのアセンブリは安定していると宣言されました。





作業環境を設定する





まず、 Web Platform Installerをダウンロードしてインストールし、起動し、[オプション]をクリックし、[追加シナリオを表示]フィールドにHelicon Zoo Feedへのリンクを追加する必要があります ： http : //www.helicontech.com/zoo/feed/









その後、Web Platform InstallerにZooタブが表示されます。











Node.jsをインストールする





Zoo-> Enginesタブには、Node.jsを含む利用可能なすべてのWebエンジンのリストがあります。 ただし、Node.jsパッケージをインストールすることをお勧めします。Node.jsパッケージには、ノード自体に加えて、さらに便利なモジュールがいくつか含まれているため、 Zoo->パッケージ-> Node.js Hosting Packageを選択し 、[追加]、[インストール]をクリックします。







Helicon Zooギャラリーで、現在利用可能なすべてのWebフレームワークとアプリケーションを確認できます。 ライセンス契約に同意すると、IIS（まだインストールされていない場合）、Helicon Zoo Module、およびWindows自体のnode.exeのダウンロードとインストールが開始されます。



システムの重要なコンポーネントはノードパッケージマネージャー（npm）で、追加のモジュールをインストールする必要があります。 残念ながら、Windows上のnpmの現在のバージョンは不安定です。 しかし、対応するものがあります-ryppi.pyユーティリティは、npmと同じ方法で使用できます。 ryppi.pyはPythonで記述されているため、Node.jsホスティングパッケージをインストールするとpythonも描画されます。 将来、Windowsでnpmが安定して動作する場合、ryppi.pyをそれに置き換えます。これにより、Pythonをインストールするために80MBのディスクスペースが節約されます。







WebMatrixテンプレートをインストールする





そのため、Node.jsをインストールしました。このアプリケーションを作成するために、WebMatrixのテンプレートを使用できます。 これらのテンプレートを使用すると、空の空のアプリケーションを作成して、さらなる開発に使用できます。

それらをインストールするには、 Zoo->パッケージ-> WebMatrixテンプレートを選択します。







WebMatrixがインストールされていない場合は、問題ではありません。WebMatrixは、テンプレートのインストール中に自動的にダウンロードおよびインストールされます。 インストール後、WebMatrixを実行し、メインのテンプレートから [ サイト]を選択します。







スクリーンショットを見るとわかるように、WebMatrixテンプレートが利用できるフレームワークはNode.jsだけではありません。



Node.jsサイトを作成した後、指定されたURLに移動するか、「実行」をクリックすると、単純な「Hello、World！」が表示されます。







デフォルトでは、新しく作成されたサイトには、Webアプリケーションを簡単に作成するためのエクスプレスフレームワークが含まれています。 彼とその依存関係は、サイトの下のnode_modulesディレクトリにあります。これは、リモートサーバーにアプリケーションをデプロイするのに便利です。



パブリックディレクトリは、静的ファイルを格納するためのものです。 このディレクトリに配置されたファイルは、Node.jsの呼び出しにつながることなく、IISによって静的ファイルとして直接処理されます。 これは、サーバーで誤ってclient * .jsファイルを実行しないようにするために特に重要です。



web.configファイルには、特に静的ファイルのURL書き換えルールが含まれています。 最初に、パブリックディレクトリにこのような静的ファイルが存在するかどうかの要求がチェックされます。 これは、静的リソースと動的リソースを1つのディレクトリ（通常はルート）に混在させたいWebアプリケーションに必要です。 アプリケーションがこの明白に悪質な慣行の対象でない場合、web.configファイルからMicrosoft URL Rewriteのルールを削除し、パブリックファイルを明示的に指定して静的ファイルを参照します。



web.configファイルには、このサイトでNode.jsおよびHelicon Zoo Moduleを実行するために必要な構成ディレクティブも含まれています。

Node.jsアプリケーションのデバッグ





Node.jsでアプリケーションをデバッグするには、ノードインスペクターという優れたツールがあります。 Node.jsサイトテンプレートにあるnode_modulesディレクトリに既に含まれています。 ノードインスペクターは次のように機能します。

  • デバッグする必要があるアプリケーションがデバッグモードで起動されます（--debugまたは--debug-brkオプションを指定してnode.exeを呼び出します）。
  • 本質的にはWebアプリケーションでもあるノードインスペクターが起動されます。
  • WebSocketプロトコルを使用して、これら2つのアプリケーションは互いに通信してデバッグデータを転送します。
  • デバッグされたアプリケーションを含むページがブラウザーで開きます。
  • Webkit互換ブラウザで、デバッグ情報を含むノードインスペクタインターフェイスが開きます。そのインターフェイスは、Google ChromeまたはSafariのWebインスペクタに似ています。


テンプレートのnode.jsサイトのルートディレクトリには、現在のアプリケーションのデバッグを開始し、デバッグ用にブラウザーでページを開くstart_debug.cmdファイルがあります。

サーバー展開





そこで、Webアプリケーションを作成し、それをネットワークに配置したいと考えています。 これを行うにはサーバーが必要であり、Node.jsで動作するようにWindowsサーバーをセットアップすることほど簡単なことはありません。 作業環境を展開するために行った記事の最初からいくつかの手順を繰り返す必要があります。 つまり、Microsoft Web Platform Installerをインストールし、それにHelicon Zooフィードを追加して、ZooリポジトリーからNode.js Hosting Packageをインストールします。 それだけです-サーバーはアプリケーションを受け入れる準備ができています。 サーバープラットフォームからは、Windows 2008および2008 R2、32ビットおよび64ビットバージョンがサポートされます。



これで、ホストしている場合は、IISマネージャーまたはホスティングパネルを使用してサーバー上に空のWebサイトを作成し、FTPまたはWebDeployを介してアプリケーションをサイトにコピーするだけです。 WebDeployの場合、必要なフォルダー権限も配布されます。 Gitまたは別のバージョン管理システムを使用することもできますが、これはこの記事の範囲外です。



Helicon Zoo Moduleは、もともとホスティングソリューションの構成を考慮して設計されました。 そのため、その下のすべてのアプリケーションは分割され、重複しません。 デフォルト設定のモジュールは自動的に動作し、負荷が小さいときに1つのワーカー（プロセスプロセッサ）を作成するか、コアの数までワーカーを追加して、アプリケーションの負荷が増加した場合に最大のパフォーマンスを提供します。



Helicon Zooは、エンジンとアプリケーションの概念を使用しています。 そのため、エンジンでは、何をどのように、どのプロトコルで、どのポートで、許可される最小および最大ワーカー数とapplicationHost.configファイルでグローバルに設定される同様のグローバル設定によって決定されます。 次に、サイトの下で、特定のエンジンを使用するアプリケーションを作成し、このアプリケーションが機能するために必要なパラメーターを渡すことができます。 これにより、ホスティング管理者の作業をクライアントから分離し、クライアントを互いに分離することができます。

結論





Node.jsは非常に有望なテクノロジーだと思います。 優れたパフォーマンスと柔軟性を備えています。 Node.jsがUnixとWindowsの両方で同等に優れており、これらのオペレーティングシステムのそれぞれに適切な技術的ソリューションを使用していることは特に素晴らしいことです。これはテスト結果から明らかです。



まもなくHelicon ZooにErlangとJavaのサポートを追加する予定です。 これらのテクノロジーのパフォーマンスを比較することは興味深いでしょう。 それまでの間、Node.jsは、サポートされているWebフレームワークの中でスピードの明確なリーダーです。
