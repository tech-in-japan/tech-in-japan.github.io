モスクワの大気質監視システム

この投稿では、キックスタータープロジェクトの形式で作成されたvozduh.msk.ruモスクワ大気質監視システムについて説明します。 最初に、大気を監視することがなぜ興味深いのか、なぜ既存のシステムではこれが不十分なのかについて、いくつかの言葉を紹介します。 次に、プロジェクトの資金調達に関する話があります（数字を使用）。 次に、システムのアーキテクチャ、その特性、長所、および制限に関する話があります。



白chからアルドゥイーノへ







2012年4月26日、モスクワは濃い黄緑色の雲で覆われていました。 雲に加えて、爆発したとされる肥料工場について噂が広まった。 しかし、すぐに陰謀説の場所がないことが明らかになりました。この春、白springの独特な急速な開花が発生しました。 緑の雲は花粉で構成されていました。 アレルギーの友人が家に座って、二重ガラスのサッシをしっかり閉めました。



しかし、残念な白biがなくても、モスクワの空気は通常、山岳地帯の純度ではありません。 休暇から戻った人の多くは、軽度の中毒に似た特徴的な状態を訴えています。



市内の大気汚染に関する2つのデータソースを知っていました。

  1. モセコモニタリング 二酸化硫黄、二酸化窒素、その他のガスの含有量を測定します。 このシステムは、固体粒子（塵、煙、植物の花粉）による大気汚染に関する正確なデータを監視（または公開）しません
  2. 抗アレルギー薬のメーカーのサイト 。 これらのデータは、ほこりや煙を考慮しておらず、運用もされていません。遅延は数日に達します。


mosekomonitoringのデータから判断すると、モスクワの空気中の汚染ガスの不純物濃度は通常、プレミアムレベルです。 つまり （特にAvtozavodskaya地域で）何でも起こりますが、一般に、これらのデータから判断すると、私たちの空気は他の大都市とほぼ同じです。 そのため、人々の健康に大きな影響を与える他の積極的な要因があります。



西の空気の監視方法
AQI （Air Quality Index）という名前の大気質の不可欠な指標は、世界中で広く使用されています。 オゾン、一酸化炭素、二酸化硫黄、二酸化窒素、粒子状物質による大気汚染を考慮しています。 AQIは公開していませんが、私が理解している限り、その計算のための完全なデータはありません。



西部の州大気質監視ポイントのネットワークに加えて、民間の民間監視の実践が広く開発されています。 おそらく、この活動の枠組みの中で最も有名なプロジェクトは、「空気質の卵 」、 空気質の卵です。



「目に見えない」ダストエネルギーが大気質に重要な貢献をするという仮説が立てられています。



キックスタートプロジェクト。



ソリューションを探すことにした「最大1000ドル」の価格帯には、2つの一般的なダストセンサーがあります。12ドルのGP2Y1010AU0Fという名前の覚えやすいシャープの会社と、290ドルのDylosのDC1100 proです。 最初の利点には、低価格が含まれます。 それ以外の場合、Dylosは細かい（<2.5 um）および粗い塵の個々のインジケータを測定できるため、Dylosに劣ります。 これにより、AQIの計算に使用されるユニバーサルインジケータPM2.5にデバイスの読み取り値がつながることが期待されました。 今後は、この希望が正当化されることはなかったと言います。



2012年6月21日に、LiveJournal、フレンドフィード、Facebookで、DC1100ダストメーターの購入資金を集め始めたことを発表しました。 6日間の友人の助けのおかげで、彼らは必要以上に少し高い420ドルを集めることに成功しました。 塵による雨の強い影響がありそうであるため、この過剰のために素晴らしい光学雨センサーが購入されました。 繰り返しますが、先を見て、雨と塵のレベルの強い関係が確認されたと言います。



私にとって、このような国内のオンラインファイナンスの成功は嬉しい驚きでした。 困難な状況でお金を必要とする人々を助けるために募金活動を見ることは習慣です。 したがって、私は民間の大気監視プロジェクト、実際には研究プロジェクト（およびそのように発表された）が資金を受け取ることができるかどうかについてはまったく確信がありませんでした。 その結果、さらに快適になりました。 彼は、問題は関連性があり、多くの人々を心配していると言います。

7月20日にセンサーがモスクワに到達し、8月20日にプロジェクトは半工業モードで動作を開始しました。 現在、プロジェクトのウェブサイトは関連データを公開していますが、サイト自体は積極的に最終化されています。

建築



システムは、メーター、中間サーバー、データ公開サーバーの3つのモジュールで構成されています。

ほこり、雨、温度、および湿度のセンサーからのデータは、arduinoベースのメーターによって読み取られ、jbee形式でXbee-link経由でホームサーバーに送信されます。 そこに記録され、cosm.comおよびvozduh.msk.ruでの公開のために送信されます。



メーター





ストリートからパイプ（2）を介してファン（8）によって駆動される空気は、必要に応じて相対湿度を下げ、測定結果への水蒸気の影響を防ぐために、発熱体（3）を通過します。 空気の流れは矢印で示されています。 湿度センサー（6）のデータを使用するarduino unoプラットフォームに基づく制御回路（5）は、電力リレー（7）を使用してヒーター（3）を制御します。



DC1100（4）と雨センサー（図には示されていません）からのデータは、制御回路（5）によって読み取られ、XBeeカードを介してホームサーバーに送信され、そこで処理されて公開Webサイトで公開されます。

画面が壊れている古いラップトップ上のFreeBSD 9.0。 自分自身、タンバリンと踊らずに、Xbeeモジュールを備えたftdi-cardを見つけました。ルビーデータインポートスクリプトが画面で回転しています。



サーバーを公開





現在、データはcosm.comとvozduh.msk.ruの2つの場所で公開されています。 Vozduhはruby-sinatraスクリプトであり、グラフィックはdygraphsライブラリを使用して描画されます。 チャートは10分間隔で滑らかに表示されます。 「生の」データに興味がある人は、対応するページでそれらを見ることができます 。

最小統計はvozduh-eにねじ込まれています。ダスト測定の分位点と傾向は線形モデルに従って計算されます。 統計は、データベース（SQLite）から直接データを読み書きするR言語の小さなスクリプトを使用して計算されます。





プロジェクト開発の結論、結論、および方向



私の意見では、プロジェクトは成功していると考えることができます。 第一に、これはわが国で大気を監視するための最初の公的資金によるプロジェクトです。 第二に、2つの大気汚染物質である粗粉と細粉の正確なデータが毎分公開されます。 この情報はプロジェクトの前には利用できませんでした。



塵による大気汚染に関するデータは非常に興味深いです。 たとえば、中央アフリカからのダストストームが北アメリカの海岸に到達し、数千キロメートルの距離を移動することが知られています。 私たちの測定では、数百キロメートルの距離を移動しても微細な塵の量はほとんど変化しないことが示されています。 ある日、モスクワの南西、モスクワ環状道路、そしてドゥブナの街で証言を続けました。 証拠は非常に近かった。 さらに、時間の経過とともに、汚染レベルは数時間にわたって急速に変化することがあります。 これにより、このパラメーターの値が解釈に適しています。

残念ながら、ダスト測定をこのビジネスで一般に受け入れられているPM2.5インジケータに変えることはできませんでした。 実際には、DC1100は単位体積の空気あたりのダスト粒子の数を示し、PM2.5は単位体積あたりのダストの質量を示しています。 一方から他方への変換が試みられたという事実にもかかわらず、現時点での私の最善の理解は、正しく実行することは不可能だということです。 [5]に問題に興味のある人に対処できます。



プロジェクト開発のさらなる計画には以下が含まれます。



  • センサーネットワーク ダストがかなりの距離を移動する様子を見るのは非常に興味深いでしょう。 複数の測定ポイントの製造に資金があれば、モスクワや他の都市での塵埃や雨の動きを調べることができます。
  • API これは、すぐにサイトに表示される優先事項の1つです。
  • Twitter 街の空中で何が起こっているかを報告するスケジュールされたフィード。 アスリートやアレルギー患者に役立つことがあります。 そして、多分、例えば夜に雨が降っていたかどうかに興味がある人は誰でも。
  • モバイルクライアント。 最も近いリストからの別のステップ。
