この記事では、アマゾンウェブサービスまたはAWSの概要、その用途を説明し、最初からそれらを構成する方法について順を追って説明します。 この記事では、 Amazon Elastic Compute CloudまたはEC2とAmazon Simple Storage ServiceまたはS3に焦点を当てます。 S3の論理的な拡張機能であるAmazon CloudFrontは、 Mishの記事「 S3で動作するようにCloudFrontを構成する」で詳しく説明されています。 他のAmazonサービス（ Amazon SimpleDB 、 Amazon Simple Queue Service、SQS 、 およびその他のAmazonサービス）を自分で知ることができます 。 この記事は、 1番目と2番目の記事の続きです。 だから：



アマゾンウェブサービスとは何ですか？



これは、個別にまたは一緒に使用できる便利なサービスのセットです。 まず第一に、これらは「クラウドコンピューティング」（英語のElastic Compute Cloud）です。 これはどういう意味ですか？ つまり、必要な構成の任意の数のコンピューターを「クラウド」で、数分で必要なオペレーティングシステムで実行できます。 本当にそうです。 1つのインスタンス（インスタンスはAmazonでは1つの仮想サーバーと呼ばれます）の起動時間は通常2分を超えません。 インスタンスを起動するとすぐに動作を開始します。Windowsがインストールされている場合は、SSHまたはリモートデスクトップ経由でルートにアクセスできます。 それは素晴らしいことではありません！ 起動の瞬間から、インスタンスの支払いは時間通りに行われます。 いつでもインスタンスを停止することができ、その使用のためのお金は撤回されなくなります。 インスタンスの稼働時間に加えて、着信トラフィックと発信トラフィックも別々に支払われます。



Amazonの2番目の優れたサービスは、Amazon Simple Storage ServiceまたはS3です。 これは、信頼できる分散データウェアハウスです。 S3では、数バイトから数ギガバイトの任意のファイルを配置し、参照により通常のブラウザーからアクセスできます。 HTTP経由。 S3にアクセスするには、EC2インスタンスを実行する必要はありません。 たとえば、 http ： //s3.amazonaws.com/moodbox-server-data/channel_logo/1の写真へのリンクを次に示します。 サイト全体をS3に配置できます。静的な写真で構成されている場合は、さらに特定のバスケット（英語のバケット、S3ではすべてのファイルが格納されている）にサイトのドメイン名をバインドできます。ブラウザで通常のサイトにアクセスします。 以下で説明するように、S3のファイルは公開アクセス用に非公開にすることができ、それらのファイルへのアクセスはキーでのみ取得できます。



もちろん、これらはすべてのサービスではありませんが、記事ではこれら2つのサービスをRisovaskaプロジェクトで実際に使用する最も興味深いサービスとして説明します。



AWS登録



Amazonサービスの使用を開始する前に、そこにアカウントを作成し、それにクレジットカードを添付する必要があります。 これを行うには、登録ページに移動し、[新しいユーザーです]オプションを選択して、新しいアカウントを作成します。 2番目のステップは、 支払い情報を設定することです。 実際、ロシアの場合、支払いオプションはクレジットカードのみです。 私は個人的に仮想カードAlfa-Bank of MasterCardシステムを使用しました。 仮想カードを使用すると、インターネット経由でのみ商品やサービスの代金を支払うことができ、通常のATMには適していません。 Amazonは私から普通のデビットカードを受け入れませんでした。おそらく銀行に依存します。



そして最後に必要な手順は、 アクセスキーを取得することです。 EC2とS3を使用するには、 アクセスキーIDとシークレットアクセスキーとX.509証明書の 2種類のキーが必要です。 両方とも指定されたページで取得されます。 重要：同じページの右上にアカウント番号が記載されています（例： 8311-6132-8591） 。 この番号は、画像を作成するときに役立ちます。



上記のすべてのポイントは5〜10分で完了します。 その後、必要なサービスをサブスクライブできます。 購読は非常に簡単です。たとえば、SimpleDBを購読するには、 http： //aws.amazon.com/simpledb/にアクセスして、右側の[Amazon SimpleDBにサインアップ]ボタンをクリックします。 他のサービスでも同様です。 サービス自体へのサブスクリプションについてはお金は請求されませんので、あなたが欲しいものにサインアップしてください。 一般に、Amazonの支払いポリシーは次のとおりです。ユーザーは、使用した分のみに支払い、サービスの提供後に支払います。 支払い後。



それでは、サービスのより詳細な説明に移りましょう。



Amazon EC2またはAmazon Cloud



EC2の主な利点をリストします。

  • 数分以内に任意の数のインスタンスを開始および停止します
  • さまざまなタスクのインスタンスタイプの柔軟な選択
  • 信頼性：サービスレベル契約により、99.95％のインスタンスの可用性が保証されます。 一般的に、通常の専用サーバーはより頻繁に落ちます:)
  • サーバーの場所の2つの地理的ゾーン：アメリカとヨーロッパ
  • エラスティックIP-インスタンスが実行中か停止中かにかかわらず、常に存在するIPアドレス
  • かなり低価格


EC2の説明は、インスタンスの種類の選択、オペレーティングシステムのイメージの選択と構成（ Amazon Machine Image ）、および1つまたは複数のインスタンスの起動と操作の3つの部分に分けることができます。



Amazonは、標準インスタンスと高CPUインスタンスの2つのカテゴリに分かれたいくつかのタイプのインスタンスを提供しています。 最小インスタンスには次の特性があります：スモールインスタンス（デフォルト）1.7 GBのメモリ、1 EC2 Compute Unit（1 EC2 Compute Unitを含む1つの仮想コア）、160 GBのインスタンスストレージ、32ビットプラットフォーム、およびインストール時のコストは1時間あたり0.10ドルUnixシステムとWindowsの場合は0.125ドル。 多くの場合、この「最小限の」インスタンスで作業に十分です。 1か月で74ドルの費用がかかり、1ギガバイトの着信トラフィックあたり0.10ドル、1ギガバイトの発信トラフィックあたり0.17ドルを追加で支払う必要があります。 通常、トラフィックは1か月あたり15ドルを超えません。 詳細については、インスタンスのタイプと価格がEC2ホームページにリストされています。



最近では、1年以上使用することでインスタンスのコストを削減することが可能になりました。 これらはリザーブドインスタンスです。 一番下の行は、年間契約の場合は325ドル、3年契約の場合は500ドルを前払いし、そのようなインスタンスをいつでも開始すると、そのコストは1時間あたり0.03ドルになります。 比較のために、1年間の典型的なインスタンスのコストは846ドルで、リザーブドインスタンスのコストは325ドル+ 259ドル= 584ドルです。 詳細はEC2ホームページで入手できます。



インスタンスを選択した後、そこで動作するオペレーティングシステムのイメージを選択して構成する必要があります。 イメージは、オペレーティングシステム自体と、操作に必要なすべてのプログラム（Apache、MySQL、Memcached、Erlangなど）の両方であり、必要なすべてのデータをイメージに含めることができます：サイトコンテンツ、プログラムソースなど。 .d。 アマゾンや愛好家からの多数の既製の画像があります 。 たとえば、 http： //alestic.com/の非Amazonian Ubuntuイメージを使用しました。

マウント可能なボリュームまたはEBS



EC2の不快な機能は、インスタンスが停止またはフリーズしたときにすべてのデータが失われることです。 一方では、これはシステムを設計する際の良い挑戦です。 一方、今は抜け道があります -これはAmazon Elastic Block Store（EBS）です。 EBSは、1GBから1TBまでのマウントされたボリュームです。 インスタンスを起動した直後に、Elasticfoxの[ ボリュームとスナップショット ]タブを使用してインスタンスにマウントできます。 インスタンス自体では、Linuxで通常どおりこのボリュームをマウントする必要があります。

EBSの速度と信頼性は、組み込みのストレージインスタンスよりも高速です。 インスタンスが停止すると、マウントされたボリュームとそのデータはどこにも失われず、別のインスタンスに再マウントできます。 さらに信頼性を高めるために、S3で定期的にボリュームスナップショットを作成できます。 EBSの唯一の欠点は、追加料金を支払う必要があることですが、実際には月額2ドルしかありません。



Amazon S3または高負荷分散ストレージ



S3の主な利点をリストします。

  • 低いストレージ価格：ギガバイトあたり月額わずか0.15ドル、通常はトラフィックあたり1ドルから10ドル
  • 信頼性：S3に置かれたファイルは、その損失を回避するために複数のサーバーに自動的にコピーされます
  • あらゆる負荷に耐える：S3は、ファイルのコピー数を自動的に増やし、このファイルを受け取りたい人の数を増やすように構築されています
  • HTTPプロトコルによる作業の容易さ
  • S3の各ファイルに最大2kbのサイズで各メタデータを添付する機能。
  • S3を介した各ファイルへのアクセスに関する統計を取得する機能
  • バスケットの2つの地理的位置：アメリカとヨーロッパ
  • Amazon CloudFrontを使用して、S3からファイルを受信する応答時間を短縮し、コストをわずかに増加させる機能


S3を使用する最も簡単な方法は、Forefox S3Foxのプラグインをインストールすることです 。 インストール後、[ツール]メニューからプラグインを開き、[アカウントの管理]ボタンをクリックして、 アカウント名と以前に受け取ったAWSアクセスキーとAWSシークレットアクセスキーを入力します。 これでS3の使用を開始できます。



ファイルシステムが左側に表示され、S3フォルダー、フォルダー、およびファイルが右側に表示されます。 最初のステップは、右上の[ バケット/ディレクトリの作成 ]ボタンをクリックして、最初のバスケット（Eng。Bucket）を作成することです。 これは、ファイルが保存される場所です（S3ではファイルはオブジェクトと呼ばれます）。 バスケットを作成するとき、その地理的位置はアメリカまたはヨーロッパで示されます。 また、バスケットの名前はすべてのS3ユーザー間で一意でなければならないことを理解することも重要です。 その理由は、ファイル内のアクセスパスが「 s3.amazonaws.com/your_bucket/your_file 」または「 your_bucket.s3.amazonaws.com/your_file 」のように形成されているためです。 バスケットを作成したら、通常のファイルマネージャーのように、バスケットにファイルをコピーできます。 ファイルをS3にコピーした後、右上の[ ACLの編集 ]ボタンをクリックして、右側のペインでファイルを選択し、アクセス権を設定できます。 たとえば、ファイルの所有者として、すべてのユーザーにファイルを表示するか、許可されたユーザーのみに表示するか、自分だけに表示するかを指定できます。



Webサイト全体をS3にコピーし、特定のバスケットをドメインにバインドすることもできます。 サイトがJavaScriptのみを使用して記述され、データベースにデータを保存しない場合、1か月あたりギガバイトあたり0.15ドル+ギガバイトあたり0.17ドルのトラフィックでホスティングする非常に安価なWebサイトが得られます。 実際には、費用は通常月額5〜10ドルを超えません。 信頼性と重い負荷に耐えることを考えると、このソリューションは共有ホスティングよりもはるかに優れています。



さらに空想した場合は、JavaScriptからアクセスできるデータベースとしてAmazon SimpleDBを使用できますが、このトピックの専門家ではないので、希望する人のために可能な研究の方向性についてのみ概説します。
