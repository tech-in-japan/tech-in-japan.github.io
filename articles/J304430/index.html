レーキA / Bテスト-サンクトペテルブルクのローマンポボルチムとの出会い

7月7日木曜日午後8時、 オラクルのサンクトペテルブルクオフィスは、Java、検索品質、およびA / Bテストの専門家であるRoman PoborchとのCodeFreezeミーティングを開催します。 会議の主題は、 A / Bテストレーキです。



参加は無料です。 座席数には限りがあり、 リンクによる登録が必要です。







レポートについて



A / Bテスト（Eng。A / Bテスト、スプリットテスト）-調査方法であり、その本質は、要素のコントロールグループが、1つ以上のインジケーターが検出されるために変更されたテストグループのセットと比較されることですどの変更がターゲットを改善します。 例は、サイトコンバージョンに対するインターフェイス要素の配色、場所、サイズの影響の研究です。



ウィキペディア





サービスに対するすべての変更は、ユーザーに対する実験を経て行われます。 ターゲットメトリックの大幅な改善を示すもののみを展開します。 はい、ターゲットメトリックがあります。 はい、あなたはすべて統計的有意性について知っています-まあ、または少なくともあなたはそれについて聞いた。



では、なぜ今日のサービスのターゲットメトリックが1年前とまったく同じレベルなのでしょうか？



1つの可能な答えは、エラーが実験システムに忍び込んだことです。 この小説では、ユーザーの実験を開始するほぼ全員が直面するいくつかの一般的な問題について説明します。 特定の問題に加えて、レポートはインジケーターに触れ、その後、何かが壊れていることがわかります。 プロジェクトにA / Bテストがある場合、ストーリーの結果に応じて、おそらくそれを改善する必要があります。



スピーカーについて







ローマンポボルキーは1998年にサンクトペテルブルク州立大学を卒業しました。 2004年までは、Sun JDKで作業し、1.1.8からJava 5のリリースに参加しました。2004年から2008年まではIntelで働いていました。



2008年から2015年まで、彼はYandexで検索の品質を評価することに従事し、特にA / Bテストの経験を積んだ。



2015年以来、彼は主な職業を変え、現在はITのプレゼンテーションに従事しています。 彼は、 Highload ++ 2015およびRIT ++ 2016の準備に関するプレゼンテーションのコンサルタントとして参加しました。
