こんにちは、Habr！ 私の名前はDmitry Okunevです。フィンテック企業ID Financeのプロダクトマネージャーとして働いています。 これは同社の最初のブログ投稿です。 ここでは、国際的な専門知識を含む経験を共有します。 行こう



ほとんどのオンラインビジネスでは、サイトのコンバージョンは、見込み客を顧客に変えるための基盤です。 ID Financeでは、CRにプラスとマイナスの両方に影響を与える可能性のある膨大な数の変更を行っているため、コンバージョンファンネルの有効性を文字通り毎日継続的に追跡しなければ、問題が生じます。

サイトのコンバージョンを測定するためのさまざまなアプローチがあります。そのような追跡のために、Google Analyticsからのファネルの毎日の自動アップロードに基づく独自のダッシュボード形式を開発しました。 なぜそのような方法ですか？



  • Googleアナリティクスはセッション分析ですが、ほとんどの場合、ユーザーに関心があります。 一意のclientID。 このデータは、GAからユーザーメトリックを照会することで最も簡単に取得できます。
  • この場合のダッシュボードは通常のGoogleスプレッドシートファイルであり、簡単に共有したり、さまざまなユーザーにアクセス許可を設定したりできます。また、それを読むのに分析の操作経験はあまり必要ありません。
  • レポートデータは、利害関係者への定期的なレポートに適しており、簡単に読み取って更新でき、製品の変換の変更をすぐに表示します。
  • 柔軟なレポート形式により、CRの監視に加えて、ユーザーフローの問題領域をすばやく見つけたり、ABテストを測定したり、仮説をテストしたり、データをすばやく計算したりするなど、他の多くのタスクを実行できます。


なぜこれが必要なのですか？



歴史は、最初の反復からの製品が私たちが望むほど効率的に顧客への着信トラフィックの変換を開始する場合をほとんど知っていません。 トラップを設定します。非効率的で複雑なUX、記入する多数のフォーム、バグ、各配信で増加するバグ-これらすべてで作業できます。 しかし、何を扱うべきかを理解するためには、最初に問題を解決する必要があります。a）ローカライズ、b）常に監視。製品とトラフィックの品質の両方が絶えず変化しているため。 現在の状況を表示するツールを手元に用意しておくと、改善に関する意思決定をすばやく行い、問題を追跡し、追加の追跡を実装できます（ユーザーフローに影響する場合）。



現在、当社の各プロジェクト（同社は4つのブランドで8か国で運営されています）には独自のレポートが含まれており、各レポートには、モバイルアプリケーションを含む新規およびリピート顧客のすべての主要ユーザーケースをカバーする10-15のダッシュボードが含まれています



どのように機能しますか？



サイト上のファンネルは、ユーザーアクションのシーケンスとして説明できます。 Googleアナリティクスは、このシーケンスをセグメントとして記述する機会を提供します。

セグメント自体は条件とシーケンスに分割され、スコープ（ユーザー（ユーザー）またはセッション（セッション））によっても定義されます。 目的のユーザーグループを取得するには、交差を指定します（シーケンスに条件を課します）。 たとえば、ABテストの特定のバリエーションの参加者と一緒に、モバイルデバイスのみから顧客の目標到達プロセスの写真を取得する場合。



Google Analyticsは、必要なデータにアクセスするための特別なCore Reporting APIを提供します。 リンクでそれをよく理解できます。 ここではクエリの構文について詳しくは説明しません。Googleのドキュメントはいずれにしても、より包括的なものになります。



以下に、Core Reporting APIでリクエストできるセグメントの例をいくつか示します。



  1. 1）ページ/登録/ステップ1にあったモバイルデバイスからすべてのユーザーのリストを取得します：



    ユーザー::条件:: ga：deviceCategory == mobile; ga：pagePath == / registration / step1
  2. 2）ページ/登録/ステップ1にいて、次にページ/登録/ステップ2にいたユーザーのリストを取得します。



    ユーザー::シーケンス:: ga：pagePath == / registration / step1;->> ga：pagePath == / registration / step2


ページ/登録/ステップ1から/登録/ステップ2へ、そして2番目から3番目などへの遷移の変換を見たい場合、ステップ1、ステップ1、ステップ2へのリクエストを順番に行う必要があると推測するのは簡単ですそして最後まで続きます。 これを行うには、以下で説明する小さなスクリプトを使用します。
