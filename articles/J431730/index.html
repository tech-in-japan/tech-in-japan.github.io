Windows Server 2019

Windows Serverの歴史は25年以上前に遡ります。WindowsNT 3.1 Advanced Serverは1993年7月27日にリリースされました。 以前のメジャーリリースの3年後の2018年10月にWindows Server 2019がリリースされ、Windows Server 2019は以前のリリースに固有の機能を開発および改善します。 これについての詳細-カットの下で！







Windows Server 2016以降、新しいリリースサイクルが採用されました。 現在、2つの流通チャネルがあります。LTSC（長期サービスチャネル）-2年から3年でリリースされ、5年間のプライマリおよび5年の延長サポート、および半年チャネル-6か月ごとにリリースされるリリース、 6か月の基本サポートサイクルと18か月の延長サポートがあります。 これらの2つのチャネルは何のためですか？ Microsoftは、Azureクラウドプラットフォームを積極的に革新しています。 これは、Linux仮想マシン、LinuxおよびWindowsを備えたコンテナー、および他の多くのテクノロジーのサポートです。



クラウドでこれらのテクノロジーを使用しているお客様は、データセンターでも使用したいと考えています。 半期チャネルは、Azureとローカルデータセンターの間のギャップを埋めます。 半年ごとのリリースは、開発が動的で、ビジネスにITサービスを提供するための柔軟なサービスモデルに切り替えた企業を対象としています。 LTSCリリースは、Exchange Server、SharePoint Server、SQL Serverなど、インフラストラクチャの役割、ソフトウェア定義のデータセンター、ハイパーコンバージドインフラストラクチャなど、サポートサイクルが長い定評のあるアプリケーションを使用する企業向けです。



Windows Server 2019は、まさにLTSCチャネルのリリースです。 これには、Windows Server 2016およびその後の半年ごとのリリースからのすべての機能更新が含まれます。



Windows Server 2019の開発者の主な取り組みは、次の4つの重要な分野に焦点を当てていました。



  1. ハイブリッドクラウド -Windows Server 2019と新しいWindows Admin Center管理センターにより、Azureクラウドサービス（Azure Backup、Azure Site Recovery、Azure Update Management、Azure AD Authenticationなど）を簡単に使用できます。
  2. 安全は、お客様にとって最も重要な優先事項の1つです。 Windows Server 2019には組み込み機能があり、サイバー犯罪者がシステムに侵入して足がかりを得ることを困難にします。 これらは、Windows 10で知られているDefender ATPおよびDefender Exploit Guardテクノロジーです。
  3. アプリケーションプラットフォーム -コンテナは、アプリケーションをパッケージ化してさまざまなシステムに配信するための現代のトレンドになりつつあります。 同時に、Windows ServerはネイティブのWindowsアプリケーションだけでなく、Linuxアプリケーションも実行できます。 これを行うために、Windows Server 2019にはLinuxコンテナー、Linux用Windowsサブシステム（WSL）があり、コンテナーイメージのボリュームが大幅に削減されました。
  4. ハイパーコンバージドインフラストラクチャ -単一サーバー内で標準アーキテクチャとコンピューティングおよびストレージを組み合わせることができます。 このアプローチは、優れたパフォーマンスとスケーラビリティを提供しながら、インフラストラクチャのコストを大幅に削減します。


Windows管理センター



Windows Admin Center（WAC）は、新しいサーバー管理ツールです。 インフラストラクチャにローカルにインストールされ、Windows Server、Windows 10コンピューター、クラスター、ハイパーコンバージドインフラストラクチャのオンプレミスおよびクラウドベースのインスタンスを管理できます。



WACは、mmc、サーバーマネージャーコンソールなどの既存の管理ツールを置き換えるのではなく、補完します。 WACへの接続はブラウザーから実行されます。







タスクを完了するには、WinRM、WMIリモートコントロールテクノロジ、およびPowerShellスクリプトが使用されます。



WACを公開し、組織の境界の外側からサーバーを管理できます。 Azure ADの多要素認証およびプロキシアプリケーションサービスは、このアクセスを外部から保護するのに役立ちます。MicrosoftEnterprise Mobility + Security（EMS）を使用すると、デバイスのポリシー、リスク、場所、その他の要素へのコンプライアンスに応じてアクセスを許可または拒否できます。 管理のためにリモートワーカーアクセスの代わりにWebアプリケーションを使用することは、私の意見では、セキュリティのための正しい戦略です。







システム分析



Windows Server 2019はよりスマートになりました。 新しいシステムインサイト機能を使用して、予測分析が実装され、事後対応型からプロアクティブ型のサーバーフリート管理への移行が可能になります。 機械学習モデルでは、パフォーマンスカウンターとイベントを考慮して、ディスクサブシステムの空き領域に関する問題を正確に予測し、プロセッサーコンピューティング、ネットワーク接続、ストレージパフォーマンスの傾向を特定します。







ストレージサブシステムの新機能



ストレージ移行サービス



Windows Server 2019は、古いサーバーから新しいサーバーにデータを移行するための新しいテクノロジーであるストレージ移行サービスを導入しました。



移行はいくつかの段階で行われます。



  • さまざまなサーバー上のデータのインベントリ
  • ソースサーバーからファイル、ネットワークフォルダー、およびセキュリティ構成をすばやく転送する
  • 制御をキャプチャし、サーバーIDとネットワーク設定を古いサーバーから新しいサーバーに置き換えます


移行プロセスは、Windows Admin Centerを使用して管理されます。







Azureファイルの同期



Azure File Syncは、従来のファイルサーバーを変換し、実際のストレージを実質的に到達不可能なボリュームに拡張します。 データはいくつかのレベルに分けられます。ホットキャッシュはファイルサーバーディスクに保存され、ユーザーが最大速度でアクセスできるデータです。 クールになると、データはAzureにシームレスに移動されます。 Azure File Syncは、ファイルにアクセスするための任意のプロトコル（SMB、NFS、FTPS）と組み合わせて使用​​できます。



ストレージレプリカ



この災害保護テクノロジーは、Windows Server 2016で初めて登場しました。バージョン2019では、Windows Server Standardエディションの限定サポートが登場しました。 現在、小規模企業は、企業のインフラストラクチャに2つ目のリモートデータセンターがない場合、Azure仮想マシンのストレージの自動コピー（レプリカ）も作成できます。



ストレージスペースダイレクト



ローカルディスク領域は、ハイパーコンバージドインフラストラクチャとスケーラブルなファイルサーバーを構築するために必要なコンポーネントです。 Windows Server 2019には、不揮発性メモリの組み込みサポート、ReFSファイルシステムを含むボリュームでの重複排除アルゴリズムの改善、クラスターあたり最大4PBのスケーラビリティ、およびパフォーマンスの向上があります。



フェールオーバークラスタリングの変更



クラスタセットが登場し、数百のノードへのスケーラビリティが向上しました。

偶数のノードを持つクラスターでクォーラムを確保するために、特別なリソース（監視ディスク）が使用されます。 Windows Server 2012 R2の時代には、監視ディスク用のストレージにディスクを割り当てる必要があり、Windows Server 2016では、ネットワークフォルダーまたはクラウド監視ディスクを使用できるようになりました。 Windows Server 2019の改善は、小規模企業のインフラストラクチャ要件の削減に関連しています。 監視ディスクは、たとえばルーターに接続されたUSBディスクにすることができます。



ドメイン間のクラスター移行およびフェールオーバークラスタリングサービスの他の改善点が登場しました。



アプリケーションプラットフォームの新機能



これで、同じDocker管理プログラムを使用して、同じコンテナーノードでWindowsおよびLinuxベースのコンテナーを実行できます。 これにより、コンテナノードの異種環境で作業し、開発者にアプリケーションを作成する柔軟性を提供できます。



コンテナのアプリケーション互換性が改善され、Server CoreおよびNano Serverイメージのサイズが大幅に削減され、パフォーマンスが改善されました。



Windows Server 2019には、他にも多くの改善点があります。 公式のdocs.microsoft.comドキュメントWebサイトのWindows Serverセクションで、それらに慣れることができます。



システム要件



このドキュメントでは、Windows Server 2019をインストールするための最小要件について説明しています。ロールとコンポーネント、実行中のアプリケーションによっては、サーバー要件が増加する可能性があることを理解する必要があります。



  • 1.4 GHz 64ビットプロセッサ
  • X64準拠の命令セット
  • NXおよびDEPサポート
  • CMPXCHG16b、LAHF / SAHFおよびPrefetchWのサポート
  • 第2レベルのアドレス変換サポート（EPTまたはNPT）


最小RAM要件：



  • 512MB（「デスクトップ付きサーバー」インストールオプションの場合は2GB）


ストレージ要件：



Windows Server 2019は、ブート、スワップファイル、またはデータドライブのATA / PATA / IDEおよびEIDEをサポートしていません。 最小量は32 GBです。



ネットワークアダプター：



  • 少なくとも1GBの帯域幅を持つイーサネットアダプター。
  • PCI Expressアーキテクチャ仕様に準拠。
  • PXEリモートブートプロトコルをサポートします。


Windows Server 2019 StandardとDatacenterエディションの比較



Windows Server 2019には、StandardとDatacenterの2つのエディションがあります。



データセンターのエディションには高度な機能があります。ハイパーコンバージドインフラストラクチャのサポート、ローカルディスクスペース、仮想化使用時の拡張ライセンス権。
