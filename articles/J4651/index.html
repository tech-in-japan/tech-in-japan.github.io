マイクロソフトとノベル：オープンソースの歴史における新たなマイルストーン？

非常に珍しいコラボレーションの例が、 MicrosoftとNovellによって実証されました 。 両社は 、自社製品の互換性の分野で協力協定のパッケージを締結しました。



企業の代表者は、技術、マーケティング、特許の3つの分野での協力について話します。 「技術」契約によると、Windows環境とSuse Linux環境でのプログラムとWebサービスの相互作用を可能にする新しい技術が開発されます。 マーケティング計画では、企業はパートナー製品のプロモーションを支援します。 特許契約に基づき、MicrosoftはSuseユーザーの特許請求からユーザーを保護します。



マイクロソフトのCEOであるスティーブバルマーは次のように述べています。「この契約のパッケージは、オープンソースと商用ソフトウェアの間に素晴らしい相互作用を生み出すのに役立ちます。」



両当事者の代表者によると 、契約締結の動機は、顧客がソフトウェアおよびサーバー管理を使用する機会を最大限に活用できるようにしたいというものでした。 多くのユーザーが異種システムを使用し、WindowsとLinux用に開発されたプログラムとネットワークサービスを切り替えることが知られています。 「お客様に選択の機会を提供したいと考えています」と、ノベルのCEO、ロナルド・ホブセピアンは述べています。



パートナーがオープンすることを計画している開発センターは、新製品の作成とテストに取り組み、商用クライアントやオープンソースコミュニティと対話します。 仮想化、サーバーサイドWebサービス、およびMS OfficeとOpenOfficeのドキュメント互換性の3つの分野に特に注意が払われます。



現時点では、NovellはXen仮想化テクノロジーを使用しています。これにより、仮想環境を実行し、それらを使用して同じコンピューター上の2つのオペレーティングシステムを使用できます。 マイクロソフトは独自の技術であるVeridianに取り組んでいます。 両社はこれらの技術の最適化に協力します。ノベルはWindows Server Longhorn用に最適化された仮想化を備えたSuse Linux Enterprise Serverを提供し、マイクロソフトはノベルソフトウェア用に最適化されたサーバーソリューションを販売します。



さらに、Microsoftは、WindowsとLinuxを並行して使用したいお客様にSuse Linux Enterpriseを推奨します。 また、Linux Enterprise Serverのテクニカルサポート用の特別なクーポンを配布します。



契約に基づき、開発者は特許の主張を恐れることはありません。Microsoftは、Linux用の非商用ソフトウェアソリューションの独立した開発者、 およびOpenSuseのソリューション開発の注文を遂行する開発者を落胆させません。



当事者は契約の財務上の詳細を開示しませんが、ノベルは近い将来これを行う予定です。これらの詳細は会社の資金調達の問題に関連しており、ノベルは法律で公開する必要があります。



この「和解」にもかかわらず、Novellは2004年に独占禁止法当局に提出された申請を撤回する予定はありません（MicrosoftはWordPerfectに関するMicrosoftの誤った行動について不満を言いました）。 それにもかかわらず、この場合でさえ、私たちの前には2人の相容れないライバルの同盟があり、その闘争は実際に20年続いた。



さらに、この提携は、おそらく、オープンソースコミュニティと営利企業との関係の分野に新しい時代を開きます。 マイクロソフトの動機がスティーブバルマーが主張するほど優れているのか、それともさまざまな国、特にEUの独占禁止当局の前で有利なイメージを作り出そうとするのかを言うのは難しい。 記者会見で握手をしなければならなかったのはスティーブ・バルマーとロン・ホブセピアンではなく、ビル・ゲイツとレイ・ヌルダ（マイクロソフトとの戦争を宣言した元ノベルのチーフ）だった場合、取引が行われたかどうかは言いにくい。 ただし、協力は確かに顧客の目から見たマイクロソフトのイメージを改善し、オープンソースプラットフォームの普及を支援します。これはすべてのユーザーにとって最も重要な結果です。
