サイトのデバッグにおける一括コメントシステム

このレビューでは、Webページへの集合的なコメントのためにいくつかのシステムを分析し、顧客および作業の他の参加者との進行中のサイトのリモートディスカッションおよびデバッグに最適なツールを見つけようとしました。







ウェブ開発者であり、ウェブマスターフリーランサーでもある会社の仕事には、仕事を受け入れる段階があります。 これらは、プロジェクト開発の異なる段階にある複数の段階である場合があります。 もちろん、この瞬間はすべての人に心から愛されています。 自分のビジネスについてそれほど知識を必要とせず、プロジェクト参加者全員のスムーズなやり取りが必要です。



彼はどういう意味ですか？ 何をどのページで変更しますか？ どこに行ってしまったのですか？ このような質問は、プロジェクト管理者またはクライアントに仕事を同僚に転送しようとしたときに、すべてのデザイナー/レイアウト/プログラマー/コンテンツデザイナーによって少なくとも1回尋ねられました。



部分的に、 コメンターなどのシステムはこの問題に対処するのに役立ち、いくつかの簡単なツールで画像にコメントを追加し、画像の特定の部分を指すことができます。 ただし、レイアウトまたは完成したサイトで作業が既に完了している場合はどうなりますか。 官僚主義を軽減し、バグを収集する時間を短縮し、編集は既存のサービスである可能性があります-集合的なコメントシステム。



このレビューは、 Nimaxデザインスタジオでの実装システムの検索に基づいているため、特定の要件があります。

  • 簡単な登録。 クライアントは、プラグインの登録とインストール（存在する場合）に対処する必要があります。
  • ページ全体にコメントする機能、ページの特定のセクション、たとえばテキストのブロックや画像にコメントする機能。
  • 無料です。 もちろん、有料サービスを提供することはできますが、顧客にサービスを提供することはできません。
  • 受け取ったコメントの通知。
考慮されるオプションでは、ソーシャルサービスにあまり注意を払うのではなく、送信前にWebページのデバッグ/コメントに正確に役立つ可能性のある機能に焦点を当てます。



印刷画面が各サービスの概要に添付され、異なるサービスを使用した同じ問題の解決策を示します。テキストの段落にコメントを付けます。



トウィドラ



原則として、解説サービスに最も近い。 彼は、自分のパブリックルームまたはプライベートルームを作成することを提案します。このルームでは、任意の画像をアップロードしたり、任意のサイトを開いて一括コメントを投稿したりできます。 Twiddlaは、ライブチャットだけでなく、描画、サイトのキャンバスにさまざまなオブジェクトを配置するためのツールを提供します。



1つの部屋は1つのキャンバスであるため、このサービスはサイト全体またはページセットにコメントする場合にはあまり便利ではありません。 残念ながら、無料版ではプライベートルームを作成したり、パブリックルームを維持したりすることはできません。そのため、通信するにはすべての参加者が同時にオンラインである必要があります。 しかし、これらの欠点は、1か月あたり14ドル以上で解消できます。







公式ページ： http : //www.twiddla.com



長所

  • サイトの表面に描画;
  • 作業ページ上のオブジェクトの配置。
  • プラグイン、クロスブラウザのインス​​トールは必要ありません。
  • 登録は不要です。
短所

  • 1つの部屋で複数のページを同時に操作できない。
  • 無料版にはいくつかの機能があります。

サイドウィキ



サイトに公開コメントを作成するためのGoogleパネルのサービスアドオン。 GoogleアカウントとGoogleパネル（IEまたはFF用）を使用して、公開Webページにメッセージを残すことができます。 コメントはサイドバーに配置され、ページ全体または特定のテキストブロックに適用されます。 サービスには独自のAPIがあり、新しいメッセージを作成者に通知するスクリプトを作成できます。 サイトの所有者は、色で強調表示され、リストの一番上に配置された独自の特別な管理者コメントを書くこともできます。 残念ながら、約束にもかかわらず、現時点ではサービスにはプライベートコメントの可能性がないため、パブリックアドレスの視覚的なバグトラッカーとして使用することは不可能です。







公式ページ： http : //www.google.com/sidewiki/intl/en/index.html



長所

  • 使いやすさ;
  • Googleパネルを使用した幅広い視聴者;
  • ロシア語のサービスとヘルプ。
短所

  • 画像にコメントできない;
  • サイトの「上」にコメントできない。
  • プライベートコメントモードの欠如;
  • 通知の難しさ。

リフレミット



実装Wikiと同様に、このサービスはパネルのインストールを必要とせず、プラグイン（IE7 +、FF）の形式で非表示にし、Sidewikiと同様のサイドパネルを提供しますが、機能ははるかに豊富です。 ページ全体、選択したテキストまたは画像のブロックにコメントを付けることができます。 メッセージは公開または非公開のままにすることができ、コメントを表示するグループを定義することができます（これらはプロジェクトのワーキンググループにすることができます）。新しいメッセージの出現を監視します。 Reframeit Webサイトでは、特定のグループおよび監視対象ページの新しいメッセージをフォローできます。 コメントのリストについては、パネルはいくつかの表示モードとフィルタリングを提供しますが、一般的に、インターフェースは直感的とは言い難いため、チーム内ではなくクライアントへのサイトの転送に悪影響を与える可能性があります。







公式ページ： http : //www.reframeit.com



長所

  • コメントに応答する機能。
  • グループに区切られたプライベートコメント。
  • OpenIDを介して、登録なしで作業する能力。
  • RSSを含む、新しいコメントを追跡する機能。
  • 内部/閉鎖サイトのページにコメントすることができます。
短所

  • 主観的に、インターフェースはクライアントにとって複雑になる可能性があります。
  • サイトの「上」にコメントできない。
  • 開発者は、IE用プラグインの初期ベータステータスについて警告します。

ディゴ



「Diigo」-IEとFFのパネル、Opera、Safari、Chromeのブックマークレット。 主な目的は集合的なブックマークですが、ページ要素をコメントしたり強調表示（強調）する機能のおかげで、タスクに非常に適しています。 どのページでも、メッセージを残したり、選択を行ったり、選択にコメントを残したりできます。 さらに、コンテキストメニューを使用すると、任意のページに任意の数のステッカーを追加できます。 メモは、ページの任意の要素の上に配置したり、ページ上の特定の選択に添付したりできます。各メモには独自のコメントを付けて、フィールドに直接配置できます。 非アクティブなメモは、混乱を恐れずにページに配置するのに便利な小さなアイコンに折りたたまれます。







公式ページ： http : //www.diigo.com



長所

  • 内部/閉鎖サイトのページにコメントできます。
  • サイトの平面にステッカーを配置できます。
  • 代替ブラウザ用の公式ブックマークレットがあります。
  • RSSを含む、表示されるコメントを追跡する機能。
短所

  • 深刻なものはありません。

追加ツイート



TwitterのOpenID IDと承認を使用してコメントパネルを作成するFF用のプラグイン。 パブリックページと内部ページの両方に別のページにメッセージを残す可能性があるにもかかわらず、テキストまたは画像の特定のブロックを指定することはできないため、詳細については説明しません。 プライベートコメントを作成することもできませんが、Twitterメッセージを複製してTwitterスタイルのコメントに返信することはできます。 このサービスは、ミニマリズムのファンに適しています。







公式ページ： http : //www.addatweet.com



長所

  • Twitterとの統合。
  • Twitterはミニマリズムです。
短所

  • IEのプラグインであるFirefox 3以降のみが後日約束されます。
  • テキストや画像の個々のブロックにコメントする可能性はありません。
  • 非公開コメントはありません。

グージー



メインのrunetブラウザ（Opera、FF、IE）のパネル。 快適なデザインとビジネスへの有能なアプローチを備えたロシアのサービス。 グループ用のステッカーと同様に、プライベートおよびパブリックのステッカーを作成できます。 各メモには独自のコメントを付けることができます。 このサービスには追加の登録は不要で、最初にログインしたときにすべてが非常に詳細に「噛まれ」ています。 ステッカーは自分の場所を覚えており、インターネットからアクセスできないサイトを恐れていません。







公式ページ： http : //www.goozzy.com



長所

  • ロシア語でのサービスとヘルプ。
  • OpenIDを介して、登録せずに作業する能力。
短所

  • アルファステータスのIE用プラグイン。
  • 定期的に、サイトとflash-pluginの読み込みに問題があります。これは、パネルのベータステータスによって正当化されます。
私たちが検討したオプションの中には、それぞれに長所と短所があり、特定の場合により適しています。 その結果、Goozzyサービスは非常にシンプルで便利であり、ロシア語のインターフェイスを備えているため、Goozzyサービスに決めました。 プロジェクトの内部議論のために、私たち自身が英語のサービスを使用することを許可することができますが、ほとんどの場合、クライアントとのコミュニケーションのためではありません。 大規模で複雑なプロジェクトについて議論するために、当社ではTwiidlaを使用しています。Twiidlaを使用すると、議論中のサイトに好きなようにコメントし、「ペイント」できるからです。
