新しいRemoteAppサービス



TechEd NA 2014の注目すべき発表の1つは、Azureクラウドに既存のWindowsクライアントアプリケーションを配置し、RDPクライアント（Microsoft提供）を介して任意のコンピューター、タブレット、ラップトップ、または電話からそれらにアクセスできるようにする公開の予備テストでリリースされたAzure RemoteAppサービスでしたWindows、Mac OS X、iOS、Androidの場合）。







Azure RemoteAppは、電力をオンデマンドでまたは自動的に動的に割り当てまたは削減することにより、リモートユーザーが必要とする計算能力を効率的に拡張する機能を提供します。



このサービスは、アプリケーションでのリモート作業に高度なセキュリティを提供します。 同時に、アプリケーション自体は保存されず、ユーザーデバイスに書き込まれませんが、集中リモートアクセス用に提供されます。



Azure RemoteAppサービスは、ハイブリッドシナリオをサポートし、既存のアプリケーションホスティングインフラストラクチャと統合できます。



サービスの公開テスト期間中、Azure RemoteApp は 20ユーザーが無料で利用できます。 テスト中は、サービスの使用条件に注意してください。

新しいAPI管理サービス



TechEd NA 2014のもう1つの注目すべき発表は、 API Managementサービスの予備テストの開始です。 このサービスは、独自のAPIの開発者に、クラウド内および独自のインフラストラクチャを含むホスティング上の任意の場所にあるAPIを管理、監視、管理するための完全な環境を提供する機会を提供します。







API Managementは、本質的に中間プロキシサービスとして機能し、豊富なセキュリティ機能セットに基づいて独自のAPIへのアクセスを保護するだけでなく、API使用統計を監視し、豊富な分析ツールを使用し、クォータに設定し、APIにアクセスするためのさまざまなサブスクリプションを提供します負荷の下でスケーリングし、ユーザー要求の結果を集中的にキャッシュします。







API Managementの最も興味深い機能の1つは、APIを公開した各開発者に提供されます。APIは、 Orchard CMSに基づいて構築され、独自の設計を詳細にカスタマイズできる特別なポータルです。 このポータルは、ポータルで自動的に生成されるさまざまなプログラミング言語のコードサンプルなど、APIのすべての機能の説明とともに、サービスの外部クライアントにAPIへのアクセスを提供するために使用されます。 次のリンクで、この開発者ポータルのカスタマイズと使用に関する記事を読むことができます。







もう1つの優れた機能は対話型コンソールです。これにより、開発者はポータルでAPIを直接テストし、要求パラメーターを設定し、HTTP要求のヘッダーを管理できます。

新しいRedisキャッシュサービス



TechEd NA 2014のもう1つのすばらしい発表は、テスト用にクラウドでオンデマンドで利用できるRedisベースの分散キャッシュサービス（REmote DIctionary Server）の開始です。 Redisは、メモリ内に分散Key-Valueデータストレージシステムを構築するために設計された一般的なオープンソーステクノロジーです。



Azure Redis Cacheサービスを使用すると、最初の要求時にデータキャッシュタスクに必要なサイズの既製のredisストレージを取得できます。 開発者には、シンプル（基本）と標準（標準）の2つのレベルのサービスが提供されます。 サービスレベルは、単一のRedisストレージノードとして提供されます。 このレベルのサービスは、開発およびテスト中の重要ではないタスクに最適です。 標準層は、信頼性を高めるためにホスト複製をサポートしています。







Azure Redis Cacheサービスインスタンスは、 新しいAzureポータルで作成できます 。 サービスのインスタンスを作成した後、ポータルは作業を監視し、サービス設定にアクセスするための一連の機能を提供します。 Redisストレージには、 多くのプログラミング言語からアクセスできます。 アクセス用のキーはポータルで取得できます。



サービスの価格に関する情報は、公式ウェブサイトで入手できます。

新しい仮想マシン機能



Azure Virtual Machine Infrastructure Platformは、いくつかの興味深い更新プログラムを受け取りました。



InfiniBandを備えた新しい高性能VM A8およびA9



Microsoft Azureのユーザー向けに、高負荷ソリューションでの使用に特化したサイズA8（8コア、56GB RAM）およびA9（16コア、112 GB RAM）の仮想マシンが利用可能になりました 。 以前は、このようなマシンはクラウドサービスの形式でのみAzureで利用可能でした。 これらのタイプの仮想マシンには、InfiniBandに基づいた高性能ネットワークが用意されています。







これらのタイプの仮想マシンを作成するには、コマンドラインコマンドを使用する必要があります。 ポータルを介した作成はまだ利用できません。



PS C：\> New-AzureQuickVM -windows –ServiceName mya8vm -name mya8vm -imagename $ image -instancesize A8 –AdminUserName $ admin –password $ pwd -location "West US"



詳細については、 この記事をご覧ください 。



ネイティブのウイルス対策サポート



仮想マシンのインフラストラクチャMicrosoft Azureには、集中管理などのさまざまな機能を実行するためにマシンに自動的にインストールできるエージェントの機能があります。 エージェントの例は、VMとPuppetサーバーを統合するエージェントです。



プラットフォームの更新では、仮想マシンの新しいエージェントが追加されました。セキュリティ拡張機能、VMの作成および展開時にVMに自動的に追加できるウイルス対策モジュール。 現在、Microsoft Azureは、Microsoft Antimalware、Symantec Endpoint Protection、TrendMicroのDeep Security Agentの3つのエージェントから選択できます。







別のエージェントがエージェントのリストに追加されました-カスタムスクリプト。仮想マシンの作成直後にマシン上で任意のPowerShellスクリプトを実行できるため、VMに環境をデプロイするさまざまな機能を自動化できます。



VMイメージキャプチャのサポート



今年初め、ビルド会議で、VMイメージ自体とそれに接続されたすべての仮想ディスクの両方を含む仮想マシンイメージをキャプチャする機能が導入されました。 提示された機能は、コマンドラインスクリプトを介してのみ使用できます。







最新のプラットフォームアップデートでは、完全なVMイメージを削除する機能がポータルに追加され、ポータルに簡単にアクセスできるようになりました。 この新しい機能により、停止したVMと実行中のVMの両方のイメージを、再起動することなく取得できます。 キャプチャされたイメージは、すぐに再展開できます。



VMイメージギャラリーのJava 8



Oracleとの戦略的パートナーシップを締結した後、JDKアプリケーションを実行するためのデプロイされた環境ですぐに使用できるイメージを含む、多くのOracleサーバー製品が仮想マシンのイメージギャラリーに追加されました。







しばらく前に、オラクルが公式にサポートしているJDK8のイメージがイメージのリストに追加されました。



新しいネットワーク機能



Azureの新しい仮想マシンインフラストラクチャ機能に加えて、仮想ネットワークおよびネットワーク用の興味深い新機能が追加されました。



ExpressRouteサービスリリース



Azure ExpressRouteサービスは、インターネットを中間リンクとしてバイパスして、ユーザーが独自のインフラストラクチャをMicrosoft Azureデータセンターに直接接続する機会を提供します。 この機能により、ネットワーク接続の可用性と品質を保証し、インフラストラクチャを自然に拡張できます。



以前、ExpressRouteは予備テスト中でしたが、サービスの提供に関与するパートナーのネットワークを常に開発および拡大しています。 新しいアップデートにより、サービスは商用運用され、さらに多くのパートナー（AT＆T、Equinix、Verizon、BT、Level3、TelecityGroup、SingTel、Zadara）を通じて利用可能になりました。



現在、Azure ExpressRouteはこれらのパートナーを通じてのみ利用可能ですが、このリストは常に更新されています。 あなたは公式ページでサービスの利用規約を見つけることができます 。



複数のVPNサポート



仮想ネットワークで複数のVPNを接続する非常に一般的な機能が商用運用に導入されました。 この機能により、Azure内の仮想ネットワークをローカルインフラストラクチャ内の複数のネットワークに接続できます。







仮想ネットワークの説明ページの詳細。

IP予約



アプリケーションのパブリックIPアドレスを予約する新しい機能により、単一のサブスクリプション内でアプリケーション用に最大5つのIPアドレスを無料で予約して使用できます。 追加のIPアドレスを有料で購入できます。 予約済みのIPは、選択したVMまたはクラウドサービスに割り当てることができます。



現在、ユーザーはコマンドラインスクリプトを介してIPを予約できます。



＃IP予約



New-AzureReservedIP -ReservedIPName EastUSVIP-ラベル「EastUSの予約済みVIP」-ロケーション「East US」



＃配置時にIPを使用



New-AzureVM -ServiceName "MyApp" -VMs $ web1 -Location "East US" -VNetName VNetUSEast -ReservedIPName EastUSVIP



将来的には、使いやすさのためにIP予約がポータルに追加されます。



この機能の発表により、特定のIPアドレスを仮想マシンに割り当て、それらに直接アクセスできるようになりました。 これにより、FTPサーバーなど、クラウドにインフラストラクチャを展開するための多くのシナリオが簡素化されます。 次のリンクで可能性について詳しく知ることができます。



内部負荷分散



新しい内部ロードバランシング機能により、Azureのお客様はプライベートIPアドレスを使用してネットワーク内のトラフィックのバランスを取ることができます。 内部負荷分散は、たとえばデータベースレベルなどのリンクの一部がインフラストラクチャ内に隠されているが、それでも負荷分散が必要な多層システムに役立ちます。



内部負荷分散機能は、追加費用なしで標準レベルの仮想マシンで利用できます。 この機能は現在、事前テスト中です。新しく作成された仮想マシンでのみ使用できます。



Traffic Manager外部エンドポイントリリース



Traffic Managerサービスを使用すると、異なる条件および異なる目的（負荷分散またはフォールトトレランス）に応じて、Azureデータセンター間でトラフィックをルーティングできます。



この更新により、Traffice Managerに、内部Azureリソースだけでなく外部ユーザーリソースのトラフィックをルーティングする新しい機能が追加されました。 Azureのお客様は、Azureエンドポイント、自社サーバー、または他のクラウドプロバイダー間でトラフィックを集中的にルーティングできるようになりました。



この新機能はプレビューで利用でき、現在はコマンドラインスクリプトでのみ構成できます。

インポート/エクスポートメディアサービスのリリース



Azure Import / Exportサービスは、クラウドプラットフォームユーザーがメディア（HDD）上の大量のデータ（ファイル）をAzureインフラストラクチャと送受信して、クラウドからのデータのロードまたはアンロードを加速する機能を提供します。



このサービスは以前はプレビューとして利用可能でしたが、Azure Import / ExportはTechEdで商品化されました。 この発表により、サービスは多くの新しい機会を受け取りました。データのインポート/エクスポートをサポートする地域のリストが拡大され、サービスが米国、ヨーロッパ、アジアで利用可能になり、サプライヤのアカウント番号を示すだけで配信にDHLとFedExの両方を使用できるようになりました。



サービスの運用に関する質問は、パブリックアドレスwaimportexport@microsoft.comに送信できます。



新しい共有SMBストレージサービス



Azureクラウドのデータストレージに関する最も重要な発表は、新しいAzure SMBファイル共有サービスの発表でした。 このサービスを使用すると、SMBプロトコルを使用してネットワークリソースとしてAzure Storageデータにアクセスできます。これにより、ネットワーク相互作用を通じて仮想マシンからのデータへの通常のアクセスを使用できます。



WindowsまたはLinux仮想マシンは、SMBプロトコルを使用して新しいサービスでファイルの読み取りと書き込みを行うことができ、同時に、保存されたファイルへのアクセスが必要なアプリケーションを構築するためのRESTインターフェイスも提供されます。



Azure SMBファイル共有サービスは、AzureストレージBLOB、テーブル、キューと同じインフラストラクチャ上に構築されているため、データストレージの高い信頼性、可用性、冗長性を意味します。 サービス自体はAzureインフラストラクチャによって管理されているため、顧客は仮想マシンを作成および構成しなくても、既製のSMBファイルストレージインフラストラクチャを取得できます。



サービスに関する追加情報は、 次のリンクから入手できます。



新しい統合機能



導入された新しい統合機能のおかげで、Webサイトやモバイルサービスなどのクラウドリソースを、企業のローカルネットワーク内の独自のインフラストラクチャでホストされているデータベースにリンクできるようになりました。 現在、開発者は、サーバー上で常にホストされるデータを使用して、クラウドWebおよびモバイルアプリケーション、サービス、モバイルバックエンドを簡単に構築できます。







モバイルサービスとWebサイトをローカルインフラストラクチャにリンクするハイブリッドシナリオを提供するために、新しい無料レベルのBizTalkサービスが導入されました。 この革新のおかげで、新しいハイブリッドシナリオでは追加の支払いは必要ありません。



このシナリオの重要な利点は、開発者が企業とクラウドバックエンド間の通信のためにVPNチャネルを開いたり構成したりする必要がないことです。 代わりに、BizTalk Servicesは、追加のネットワークセットアップやセキュリティの低下なしに、ハイブリッドシナリオを提供する機能を提供します。



ハイブリッド接続は、Webサイト（現在は.NET、Java、PHP、Python、Node.js）およびモバイルサービスのバックエンド（.NETおよびNode.js）でサポートされているすべてのフレームワークとプログラミング言語をサポートしています。

GreenButton Cloudサービス



Microsoft AzureのBig ComputeというAzure公式ブログエントリで、 MicrosoftはGreenButtonの買収を発表しました。



GreenButtonは、巨大なコンピューティングリソースを必要とするアプリケーションをクラウドに転送する機能を提供する人気のあるクラウドサービスです。 以下の人気のあるツールが一度にそのようなアプリケーションとサービスになりました：Blender、ブラジル、Genious、LuxRender、Yafaray、Deep Exploration、GreenButton inCus、RenderMan On Demand、Numerix。
