Microsoft CEOが辞任します。 楽観的な記事

今日、スティーブバルマーは、株主総会が新しい後継者を見つけて承認するとすぐに、12か月以内にマイクロソフトを辞任すると発表しました。







このステップは、Microsoft内のグローバルな再編成の一部です。



「時間はそのような変化に理想的ではありませんが、今ではそれらに適しています。 すでに新しい組織戦略の実施に着手しており、上級管理職の優れたチームがあります”
Ballmer RBCを引用します。



▌ さて、実際には、トピックは何のために書かれたのか。 私見と事実のビット。


Ballmerが2000年代にBill Gatesによって作成された帝国を台無しにしたと隅々で叫ぶ一部のユーザーの怒りのコメントを読むのは面白いです。 彼がついに去り、彼が物事が上り坂になるなど、神に感謝します。 しかし、そうですか？



スティーブバルマーはマイクロソフトの創立者の親友の1人であることを思い出してください：ビルゲイツ（辞任後、ゲイツは社内でバルマーと連絡を取り、管理している可能性があります）



彼の在位中、スティーブ・バルマーはマイクロソフト社の進路を根本的に変えることができました。多くの人は市場を驚かすことができない不器用な水泳管理脂肪層マシンと考えていました。 （一度にノキアのように：habrahabr.ru/post/171325/）

しかし、バルマーは市場の動向を感知することができ、会社の再編に青信号を与えました。



私は何について話しているのですか？ あなたは例のために遠くに行くべきではありません。 同じWindows Mobile 6.xを思い出してください。2007年には、その古代のアーキテクチャのために市場の製品と競合できなくなりました。 BallmerとMicrosoftの功績により、彼らが長年取り組んできた失敗した製品をリリースする代わりに、彼らは立ち止まってやり直すことにしました。 このように、世界はWindows Phoneを目の当たりにしました。 システムのマイナス面については長い間議論することができますが、ユーザーとの対話の基盤となるインターフェースにはすでに多くの費用がかかっています。





2010年の発売時には、Metroの禁欲的なミニマリズムが、通常の範囲を超えて、優れたiOSおよびAndroidインターフェイスを備えた、skeomorphic（「skeomorphism」という用語についてはこちらを参照 ）を背景にしています。 これは現在、2013年に業界全体がフラットでミニマリズムに向かっています。 そして、それは危険な動きでした。 他のマネージャーは、Metroを落ち着いて拒否し、他のメーカーのソリューションのように見える製品を、数年後に市場の変化を予測することなくリリースできました。



その後、バルマーの下で、マイクロソフトの完全なブランド変更も行われました。 2000年代の保守的な「老人」から、同社は完全に異なる形式に変身しました。 このビデオはそれを最もよく説明しています：



これに続き、さまざまなMicrosoft部門のすべての製品の統合が始まりました。 彼らが互いに対話できるように。 Windows 8をキックすることはできますが、これはさらなる開発の非常に良い基盤です。 Windows 8が、Microsoftがすべての製品を組み合わせるというコンセプトで想像しているとおりだと言う人はいません。 まだまだ多くの作業がありますが、すべてに時間がかかります。 iPhone用App Storeもすぐには表示されませんでした。 最初は、iPhoneはいくつかのアプリケーションがプリインストールされており、コピー/貼り付け、MMS、3Gなどのない美しい携帯電話に過ぎませんでした。 リソースがあれば、これだけの時間で十分でした 。 しかし、これが会社が目指していることだと言う人はいません。 本格的に動作します。



そのような変容が小さな会社の内部で発生した場合、誰も驚かないでしょう。 しかし、本社に8万人以上の開発者がおり、その製品が何十億人もの人々によって使用されている「ソフトウェアの巨人」に関しては、これは別のケースです。



私の記憶では、市場の急進的な変化に直面して関連性を維持できる企業はほとんどありません。 コダック、ポラロイド、フィリップスを思い出してください。 かつて、これらは業界のリーダー、イノベーターでしたが、明らかなことを見ていなかったリーダーの盲目は、時代を先取りし、変えたくなかったため、これらの企業は崩壊しました。



バルマーが在位中にマイクロソフトの開発にどのように悪影響を与えたかを検討する価値はありますか？ そして、別のシナリオでは、Microsoftを長い間忘れていた可能性はどのくらいありますか、これはKodakでどのように起こりましたか？
