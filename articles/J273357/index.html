DarkMarketに弱点がありました。 イギリスのカーダーJiLsiがこのサイトを使用しました。 彼は同じパスワード「MSR206」を使用しました。これには、MaxがすべてのパスワードにアクセスできるCardersMarketも含まれていました。 MaxがDarkMarketに侵入してホストできるようになりました。



しかし、バウチドは要塞でした。ブラウザに信頼できるデジタル証明書がインストールされていないと、サイトにアクセスすることさえできませんでした。 幸いなことに、JiLsiはここに登録され、モデレーターの権利さえ持っていました。 Maxは、JiLsiの電子メールアカウントの1つで、通常の「MSR206」パスワードで保護された信頼できる証明書のコピーを見つけました。 残ったのはJiLsiとしてVouchedに行くことだけで、データベース全体が利用可能になりました。



Maxは、TalkCashとScandinavianCardingでのサイト検索がSQLインジェクション攻撃に対して脆弱であることを発見しました。 マックスはこの問題の先駆者ではありませんでした。 このような攻撃に対する脆弱性は、サイトにとって一般的なものです。 SQLインジェクションは、構造が複雑なサイトのアーキテクチャを使用します。 ニュースノート、ブログエントリ、株価情報、オンラインストアサイトなどの動的なコンテンツを含むウェブサイトにアクセスすると、サイトのソフトウェア（ソフトウェア）がデータベースから取得した情報を提供します。 このデータベースは通常、コンピューターが接続されているホスト上ではなく、別のコンピューター上にあります。 Webサイトはファサードであり、データサーバーはブロックされています。 理想的には、インターネットからはまったくアクセスできません。 サイトソフトウェアは、SQLでデータを保存するサーバーと通信します（構造化照会言語は構造化照会言語です）。 たとえば、SELECTコマンドは、特定の条件に一致するすべての情報をサーバーに照会します。 INSERTはデータベースに情報を追加します。 めったに使用されないDROP命令は、大量のデータを削除します。 多くの場合、ソフトウェアはサーバーへのSQLコマンドの一部としてサイト訪問者にリクエストを送信する必要があるため、これは危険なツールです。 サイト訪問者が検索バーに書き込む場合：

シナトラ

サイトソフトウェアは、次のようにサーバーに情報を要求します。



music_catalogからタイトルを選択

WHEREアーティスト= 'シナトラ';



SQLインジェクションは、ソフトウェアがコマンドをサーバーに送信する前にユーザーリクエストを適切に処理しない場合に発生します。 句読点はわかりにくい場合があります。 上記の例の場合、サイトの検索バーに書き込みます：



シナトラ '; DROP music_catalog;



サーバーは次の形式のコマンドを受け取るため、アポストロフィとセミコロンに注意してください。



SELECT * FROM music_catalog

WHEREアーティスト= 'シナトラ'; DROP music_catalog; ';



データベースの場合、これらはセミコロンで区切られた2つの連続したコマンドです。 最初はSinatraのアルバムを検索し、2番目は音楽カタログを削除します。



SQLインジェクションは、ハッカーの武器の一般的な武器です。 今日でも、電子商取引サイトや銀行サイトなど、あらゆるレベルのWebサイトにこの方法で侵入しています。 そのため、2005年にTalkCashとScandinavianCardingソフトウェアが攻撃の対象となりました。



TalkCashの脆弱性を利用するために、Maxはログインし、一見無害なメッセージを送信しました。 メッセージの本文には隠されたSQLコマンドがあり、その色が背景色と一致するフォントで記述されているため、目に見えません。 彼は検索クエリを入力し、サイトソフトウェアは隠しコマンドをデータベースに渡して実行しました。 このチームはINSERTで、サイトに別の管理者を追加しました-Max。 彼はScandinavianCardingでも同じことをしました。



8月14日までに、マックスは自分ができることをカーディングする世界を示す準備ができていました。 彼はひそかに細工された穴からWebサイトにアクセスし、偽の管理者アカウントを使用してデータベースをコピーしました。 このような計画はSun Tzuに値するものでした。競合するサイトはいずれも攻撃とキャプチャを期待していませんでした。 ほとんどのカーダーは宣伝を避け、自分自身を展示しませんでした。 敵対的なキャプチャは前例のないものでした。
