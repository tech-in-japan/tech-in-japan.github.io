Stream TVを使用するためのDHCP構成を使用したNetComm NB6Plus4WのZTE531Bの点滅

自宅にZTEモデムがあります。 それ自体で機能し、機能しますが、これらのモデムには「小児病」があります。コンセントから切断すると、すべての設定が忘れられます。

そして今、テーブルの下からすべてのコンピュータワイヤを1か所で収集しようとした後、モデムは別の「メモリ損失」を経験しました。

インターネットで少し読んだ後、ZTE531BモデムはNetComm NB6Plus4Wと同じ容量を持っていることが明らかになったので、ファームウェアは適切であるはずです。

このファームウェアには、より多くの設定、DHCPおよびdyndns.comのサポートがあります







最初に、 公式サイトから新しいファームウェアをダウンロードします

次にtftpサーバー （別のサーバーを使用できますが、これを使用しました）



ファームウェア：



0.設定のスクリーンショットを記憶/記録/撮ります。

1. tftpサーバーをインストールし、その設定でデータの送受信を有効にします「ファイルの送受信」。

2.ファームウェアをtftpサーバーファイルフォルダー（デフォルトではC：\ TFTP-Root）に解凍します。

2.1。 ファームウェアファイルの名前を短い名前に変更すると、さらに名前を書きやすくなります（バージョン番号は363のままにしました）。

2.2。 あらゆる種類のウイルス対策およびファイアウォールを無効にしても問題はありません。電話回線から切断することもできます。

3. telnetを実行します（Vista telnetがある場合は、有効にする必要があります。これを行うには、 msdnを読んでください 。

4.コマンドopen <modem ip address>を作成します（192.168.1.1を開いています）。

4.1。 ユーザー名/パスワードadmin / adminを入力します（以前に変更されていないかのように）。

4.2。 コマンドtftp -g <tftp server ip address> -f <firmware name> -ti（私はtftp -g 192.168.1.2 -f 363 -tiを持っています）;

4.3。 モデムが点滅するのを待っています。

5.モデムをデフォルト設定にリセットします（マシンの背面で、電源ボタンのダイオードが赤く点灯するまで10〜30秒間リセットボタンを押します）。

6.モデムの設定を開きます（ブラウザーで、192.168.1.1と書き込み、ログイン/パスワード：admin / admin）;

7.これでNetComm NB6Plus4Wができました。



重要！ ファームウェアがオフになった後のWi-Fiは、ワイヤを介してフラッシュする必要があります。
