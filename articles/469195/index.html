DevOpsProdigy KubeGraf v1.1.0

Today we have unveiled a new version of our k8s monitoring plugin - grafana.com/grafana/plugins/devopsprodigy-kubegraf-app - which we wrote about earlier .



The key feature of the new version is the ability to authenticate using a bearer token. Now through the manifests (which can be found here ) you can create a user with readonly accesses, get a token (the entire process is described in the readme ) and use it in the configuration window of your cluster.







We also changed the endpoints used to communicate with k8s-api to support kubernetes v1.16 : Deployments, Statefulsets and Daemonsets from this version finally move to / apis / apps / v1 /



Minor minor changes:



  • layout errors in Grafana v6.3.5;
  • automatic connection to the cluster of the default instance of Prometheus;
  • instructions have been added for generating the key and certificate (which is already especially unnecessary, since there is access by token :-));
  • fixed memory formatting in the presence of Mi.


We are waiting for you in our telegram chat , where you can consult the plug-in developers online.



And on our github : stars, ishshui, pull-quests, as always, are welcome!
