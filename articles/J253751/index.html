SAP Solution ManagerとSAP WPBをユーザーとサポートサービスの喜びに





こんにちは、Habr！



今日、ユーザー自身によるユーザー指示の関連性を開発および維持するプロセスをシミュレートします。 このアプローチにより、システムのサポートと開発を担当するサービスの寿命が大幅に簡素化されます。 これを行うために、2つの製品、 SAP Solution ManagerとSAP Workforce Performance Builderを使用します。これは、ITシステムのユーザー向けのインタラクティブな指示を開発するためのツールです。 私は、大多数が前者に精通していると確信していますが、後者に出会った人はほとんどいません。 最初の部分では、これら2つのソリューションを友人にしてプロセスを機能させ、2番目の部分では、SAP WPBで作成された関連する（常に！）





では、なぜ新しいユーザーのサポートとトレーニングのトピックにこだわることにしたのですか？
  • 第一に、ITシステムは、ユーザーがシステムに入る前に設計されたとおりに機能するためです。 これは公理です。
  • 第二に、ITシステムユーザーのトレーニングプロセスは規制されないことが多く、本質的に正式なものであり、これは、「どこで、どこで、どこで、なぜ、なぜ、誰？」というリクエストの形でのサポートサービスだけでなく、有効性にも大きな影響を与えます組織全体。
  • 第三に、多くの場合、ユーザー適応の問題の解決策として、サポートサービスへの追加リソースのリクエストを受け取ります。 現在の経済環境では、これは許されません。


何を提供していますか？

1.教材の更新プロセスに主要なユーザーを巻き込み、

2.参加者にとって資料の更新プロセスをできるだけ便利にします。



そのため、Solution Manager 7.1およびWorkforce Performance Builder 9.3 Enterprise Editionが必要です。 ソリューションの概要は次のとおりです。







パート1. 1つのチェーンに束縛されている

WPBマネージャーポータル（教材の開発を管理するセンター）をSolution Managerにバインドするために、 SPROトランザクションを開始し、 「WPBインターフェイスの構成テーブル」を開きます。







次の3行を入力する必要があるプレートが表示されます。

BASE_URL -WPBサーバーアドレス

CUSTDOCU -Solution Managerによってプロジェクトドキュメントとして認識され、トランザクションSOLAR02の[プロジェクトドキュメント]タブに表示されるWPBドキュメントのタイプ。

Solution Managerのコードを含むWPBドキュメントタイプのリスト：

•標準ドキュメント-標準

•トレーニングドキュメント-トレーニング

•作業文書-作業

•ジョブヘルプ-jobhelp

•テストドキュメント-テストシート

•プロセスガイド-business_script

•HP Quality Centerドキュメント-hpqc

•監査およびコンプライアンス文書-fda

W_AREA_ID-ドキュメントが保存されるWPBワークスペースの識別子。 WPBポータルで見つけることができます







データを入力して保存した後、システムの実装中に作成されたビジネスプロセス構造をWPBポータルに直接アップロードできます。 WPBマネージャーポータルから、プロジェクトのチーム（プロジェクトが進行中の場合）またはシステムが既に稼働している場合は主要ユーザーのいずれかに、インタラクティブな指示の開発/更新のためのタスクを配布します。



この例では、非常に単純な構造を作成しました。それぞれ2つのステップで構成される2つのビジネスプロセスがあります。







この構造をWPBに転送するには、トランザクションSOLAR_WPBを実行します







[ 同期 ]ボタンをクリックした後、トレーニングドキュメントの開発または更新が必要なスクリプト/プロセスを選択し、[ OK ]をクリックします。 同期に成功すると、Solution Managerのプロセス構造がWPBポータルに表示されます。 ここでは、ビジネスプロセスの1ステップが1 WPBプロジェクトに等しいことを明確にする必要があります。

これは次のようなものです。







次に、各ステップのコンテンツ開発のためのタスクを直接実行者に割り当てる必要があります。これらのアクションは、システムで毎日これらのアクションを実行し、すべての微妙な点をよく知っているキーユーザーです。 これは、WPBマネージャーポータルから直接行います。







[ 保存 ]をクリックすると、パフォーマーは特定のタスクのパフォーマーとして任命されたという「選択された」という通知を電子メールで受け取ります。 その後、DariaはWPB Instant Producerを起動し、その助けを借りて、システム内の出張用のアプリケーションを作成するプロセスを記録します。 その後、（WPBマスタードキュメントの形式で）レコードがWPBマネージャーサーバーに送信され、Dariaはタスクが完了したと見なします。



次に、WPBマスタードキュメントからトレーニング資料を作成する必要があります。 これを行うには、WPB Producerで、Dariaから送信されたマスタードキュメントを開き、必要な形式でドキュメントの作成を開始します。







その結果、インタラクティブなシミュレーションのパッケージと、システム内のアクションを説明する通常の「ペーパー」ドキュメントが得られます。



トレーニングパッケージが作成、テスト、および承認されたら、Solution Managerにダウンロードする必要があります。 これを行うには、トランザクションSOLAR_WPBで同期を再度開始します。 その結果、 SPROで行った設定に応じて、構造内のプロセスのすべてのステップが、プロジェクトドキュメントまたはトレーニングドキュメントに関連するドキュメントで「成長」します。







重要事項：Solution Managerはドキュメント自体をロードするのではなく、WPBポータルにつながるドキュメントへのリンクをロードします。 リンクは永続的です。 したがって、ITシステムでのプロセスの変更によりWPBドキュメントを更新/書き換えする場合、Solution Managerは常に現在のバージョンを参照します。



短い要約 ：



Solution ManagerとWPBを組み合わせました。 1つ目はプロセスの構造のソースであり、2つ目はトレーニング資料をこの構造に結び付け、それらの関連性を保証します。 主要なユーザーは、便利で直感的なツールを使用してコンテンツを作成します。サポートサービスは「必要な光沢をもたらす」だけです。



パート2. SAP Educationからのマッピング

次に、ユーザーが作成したトレーニングドキュメントを使用してトレーニングプログラムを作成する方法を示します（ソリューションマネージャーの用語ではトレーニングカード ）。 これを行うには、トランザクションSOLAR_LEARNING_MAPを実行し、F5を押して新しい学習マップを作成します。







トレーニングカードの名前を入力してください







また、たとえば同じプロセスロールアプローチを使用して、その構造を作成します。







その結果、次のようなものが得られました。







次に、学習マップに知識を吹き込みましょう。 これを行うには、マップセクションを選択し、[ 更新 ]ボタンをクリックして、トレーニングマップに挿入できるコンテンツを表示します。







各セクションへのWPB /ドキュメントのトレーニングシミュレーションの割り当ては、目的のドキュメントを選択し、 [ドキュメントの割り当て ]ボタンをクリックして実行されます。



完成したトレーニングカードは、 「ユーザーに送信」ボタンを使用して渡すためにユーザーに直接割り当てることができます。







または、SCORM形式にアップロードして学習管理システム（SAP Enterprise LearningやSuccessFactors Learningなど）にさらにアップロードし、システムの新しいユーザーを準備および認証するプロセスを完全に自動化できます。







短い要約 ：



Solution Managerでは、ITシステムで作業するための新しいユーザーを準備するためのトレーニングプログラムを作成しました。 プログラムのビルディングブロックは、WPBで作成されたトレーニング資料であり、常に関連しています（ソリューションマネージャーにはリンクのみが保存されていることに注意してください）。 Solution Managerを介してトレーニングプログラムを割り当てるか、LMSシステムにエクスポートし、自動コース割り当て、その通過の制御、ユーザーへの通知を設定できます。 トレーニングを正常に完了すると、ユーザーは本稼働システムへのアクセスを許可されます。



トレーニングプログラムの作成という点でソリューションマネージャーの機能があなたに合わない場合は、別の方法があります。トレーニングプログラムはWPB Producerで作成され、SCORM形式へのエクスポートまたはWPB Producerから直接LMSシステムにダウンロードされます（LMSシステムの場合SAP Enterprise Learning Solutionを使用してください）。



WPBで作成されたコンテンツは、たとえばSAPヘルプメニューに統合されるなど、さまざまな方法で使用できます。そのため、ユーザーは、問題が発生した場合に、自分がいるトランザクションやより便利な形式で正確にヘルプを受け取ることができます。
