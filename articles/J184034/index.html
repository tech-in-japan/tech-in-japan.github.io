Glazum-iOSアプリケーションでアプリ内の質問をするためのシンプルで効果的な方法





注意 ：この投稿は、まずモバイルアプリケーションの開発に携わる人にとって、そして次にこれらのアプリケーションを使用する人にとって興味深いものになります。 開発者の特定の質問に対するユーザーの回答の不足の問題について議論し、iOSのソリューションを提供します。





問題の説明



おそらく、開発者の1人は、新しい機能を決定し、それを実装し、アプリケーションをAppStoreに送信し、悪の検閲官によって承認され、アプリケーションを開いて...沈黙するときに、おなじみのストーリーを見つけるでしょう。 ただし、多くのユーザーを持つことができます。 いいえ、もちろん、アプリケーションが時間とともにクラッシュしても、沈黙はありません。 しかし、すべてが機能する場合、実質的に沈黙が保証されます。 たまに、軽い心地よい風のように、この機能に関する言及を含むコメントがすり抜けることがあります。



沈黙は、開発者を混乱させ、やる気をそらすことができます。 また、すべての機能が優れているかどうかを理解するのが難しい場合があります。 もちろん、ユーザーがそれを使用するかどうかについていくつかの結論を出すには、ユーザーのフィードバックがなくても可能です。単純なモバイル分析があれば十分です。 しかし、ユーザーが遭遇した困難を明らかにしますか？ どんな小さなものを磨く必要がありますか？ これらのニュアンスを理解することは非常に困難です。



どうする



次の2つの方向に進むことができます。モバイル分析をより集中的に使用し、さまざまな指標を入力して分析し、ユーザーからのフィードバックと関心のある質問への回答を取得します。 最も人気があり、同時にフィードバックを受け取る簡単な方法は、開発者にメッセージを送信するためのボタンを作成し、ユーザーが必要なときにそれを見つけられるようにすることです。 はい、「見つける」、彼らは通常それを隠します、なぜなら そのような機能のためにUIを乱雑にしたい人はいません。

そして今、ユーザースーツのように感じましょう。 小さなバグがありますが、開発者と共有できないのは何ですか？



  • 中断する必要があります（人々はあなたのアプリケーションを使用し、バグを探しません。彼らはあなたのアプリケーションを読んだり、聞いたり、見たり、遊んだりします）
  • フィードバックを送信する方法を見つけます。

    • 開発者のメールを探し、メールクライアントを起動し、手紙を書きます-たとえ支払ったアプリケーションのクラッシュであっても、誰もがそれを決定するわけではありません。
    • フィードバックボタンをお探しですか？ あなたがそれを見たなら、それは簡単です、あなたはどこを覚えている必要があります。 まだ見ていなかったら、それはどこから来たのかという考えはどこから来たのですか
  • 「開発者はこれを必要としますか？ 彼はおそらくそんな些細なことを必要としないでしょう。」




だから、まだ何をすべきか？



解決策は表面にあるようです-ユーザーに考えを共有する機会を与える必要があります：

  • 主な職業からの注意散漫が最小限になるように
  • 応答を残す/コメントはできるだけシンプルにする必要があります
  • 人は自分の意見が重要であると感じなければならない


答えは今やほとんど明らかであるように思われます-機能と問題について人に尋ねる必要があります。 メールまたは他のユーザーの連絡先がある場合、非常に幸運です。もちろん、少なくともいくつかの機会があります。 別のオプションは、アプリでそのような質問をすることです。 いつ誰に尋ねるかは自分で決める必要がありますが、紹介したいサービス-Glazumを使用してこれを行うことができます。



Glazumの目的は何ですか？



Glazumを使用すると、ユーザーに質問したり、フィードバックを要求したり、以前は利用できなかったメトリック（ Net Promoterなど）を測定したりできます。

質問の表示は完全に開発者の管理下にあるため、アプリケーションコードに特別なマーカーを配置し、いつでもサイトのパネルから質問に添付できます。 いつ、誰に質問を表示するかは、特別な表示基準によって質問設定で示されます。



マーカー


マーカーアプローチを使用すると、質問の制御と柔軟性のバランスをとることができます。 マーカーはいくつでも設定でき、コード内の重要なイベントをマークする必要があります（ボタンを押す、音量を変更する、購入するなど）。 それらを配置することは、モバイル分析でイベントを追跡することほど難しくありません。 



[Glazum setMarker:@"button pressed"];


同時に、ラベルを設定するための別のより高度なオプションがあります 



 [Glazum setMarker:@"button pressed" doBefore: ^(BOOL willShowQuestion) { //do before block } doAfter:^(BOOL questionWasShown) { //do after block }];


このメソッドを使用すると、質問用のアプリケーションを準備し（音楽の再生を停止するなど）、質問ウィンドウを閉じた後にいくつかのアクションを実行（再生を再開）できます。



ユーザー応答タイプ


質問を作成するとき、ユーザー応答のタイプを選択する機会があります。

これは、自由形式の短い詳細な回答、提案されたオプションからの単一および複数の選択、またはネットプロモートスコアの計算です。



さまざまなタイプの回答を含む質問の例






質問基準


Glazumには、多数の異なる質問表示基準があります。 以下に抽象的な例を示します。

  • 質問をランダムに20％のユーザーのみ表示します -多くのユーザーがいる場合、選択を制限する可能性があります
  • ロシア、ウクライナなどからの質問を示します。 -たとえばロシア語など、特定の質問をしたい場合。
  • 7.0以上のiOSバージョンのユーザーにのみ質問を表示する -新しいオペレーティングシステムのユーザーの意見を知るのは興味深い
  • アプリケーションを20回以上実行したユーザーに質問を表示することは、ユーザーの忠誠心の良い基準です
  • 5回以上お気に入りに追加するユーザー -何らかの機能を集中的に使用するユーザーに質問を表示する


たとえば、アプリケーションにログインしているなど、何らかの方法でユーザーを識別することができる場合は、電話をかけることができます 



 [Glazum setUserIdentifier:@"example@example.com"];


これにより、特定のユーザーに質問するなど、この情報に基づいて条件を使用できます。



たとえば、A / Bテスト中にユーザー変数を使用する場合、ユーザーグループごとに異なる質問をする機会があります。 



 [Glazum setCustomVariable:@"true" named:@"ab-test-printing"];


必要なのは、表示条件でab-test-printingがtrueまたはfalseである質問を作成することだけです。



価格



Glazumは現在無料です。 近い将来に価格設定を検討する予定です（無料のプランがあります）が、新しい非常に興味深い機能により製品の価値を高めることに集中したいと考えています。



現在のユーザーについては、最も快適な条件を作成しようとします。 したがって、私たちは皆を今すぐ登録するよう招待します！



まとめ



要約すると、Glazumはどのように役立ちますか？

  • Glazumは製品を改善します。

    • 1つの質問をすることで、ユーザーが最も待ち望んでいる新機能を特定できます。
    • ユーザーが直面する未知の問題や困難について学びます。
  • Glazumは、ユーザーをよりよく理解する機会を提供します。 たとえば、購入を行った理由や特定のステップで停止した理由を確認します。 これを行うには、適切なタイミングで適切な質問をするだけです。
  • 前の段落の結果として、Glazumは収入とユーザーの忠誠心を高めることができます。
