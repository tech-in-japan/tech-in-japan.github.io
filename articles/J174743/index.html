GPOを使用して.EXEインストーラーでアプリケーションをインストールする

組織内でクライアントコンピューターへのアプリケーションのインストールを自動化する（または、正直に言うと、自動化することをお勧めします）ことは誰にとっても秘密ではありません。 当然、たとえばMicrosoft System Center Configuration Managerなどの複雑な製品は、このようなタスクに完全に対応できますが、何らかの理由でそのような製品を取得しなかった組織は常に存在し、存在します。 したがって、Windowsオペレーティングシステムと同じ標準ツールを使用してソフトウェア製品をインストールする必要があり、このタスクではグループポリシーの機能が優先されます。

また、 グループポリシーソフトウェアインストール -GPSIのクライアント側を拡張することにより、アプリケーションをクライアントマシンに展開する方法は2つあります。ユーザーのみに公開するか、ユーザーまたはコンピューターに割り当てることです。

zapファイルの目的は何ですか？



概して、MSIインストーラーと* .zapファイルの両方を使用してアプリケーションを公開できますが、両方のインストールパッケージの処理方法は異なります。 たとえば、MSIソフトウェアインストールパッケージファイルには、アプリケーションのインストールまたはアンインストールに必要なすべての指示を含むデータベースが含まれています。Windowsインストーラーサービスは、*。msiファイルを読み取るためにMsi.dllライブラリを使用するクライアントコンピューターにプログラムをインストールします。 したがって、インストールサービスで発表された情報に基づいて、このサービスはプログラムファイルをハードディスクにコピーするだけでなく、レジストリを変更し、インストーラーで指定された他のタスクを実行します。 ほとんどの場合、msiインストールファイルで指定されたすべての情報は、インストールされたソフトウェア製品の開発者によって設定されます。

ただし、すべてのソフトウェアメーカーがWindowsインストーラーのインストールファイルを使用しているわけではないため、場合によっては、ゼロ管理イニシアチブに基づいて低レベルのアプリケーションパッケージを作成する必要があります（オリジナルでは、これはZAW-Zero Administration for Windowsのように聞こえます）。 、アプリケーションのインストールに必要な手順が記載されたテキストファイルです。 このようなファイルを作成した後、クライアント側の拡張機能がインストール手順に従うことができるように、インストールパッケージフォルダーに配置する必要があります。公開後、アプリケーションは「プログラムと機能」コントロールパネルコンポーネントに表示され、ユーザーがインストールできるようになります。 ただし、MSIインストーラーとは異なり、zapファイルには、インストーラーにインストールパラメーターが含まれていない場合、アプリケーションのインストールプロセスを構成できないなどの重要な制限があります。

この方法は、MSIインストールパッケージを使用してソフトウェアをインストールするほど柔軟ではないことに注意してください。ただし、インストールファイルを変換できず、アプリケーションを集中的に展開する必要がある場合、この方法はちなみに。 既存のexeファイルをインストールするための指示が32ビットアプリケーションと64ビットアプリケーションの両方のテキストファイルに含まれているという事実により、この方法を使用してソフトウェア製品をインストールする可能性には以下の機能が含まれません。

  • 昇格した特権でアプリケーションを実行します。
  • ソフトウェアのインストールプロセスが失敗した場合の変更のロールバック。
  • インストールプロセスの損傷の検出と、その後のこのプロセスの復元。
  • 後者への最初のアクセス時のソフトウェアコンポーネントのインストール。
  • 変換ファイルの使用と実装。


これは、ZAPファイルを使用したソフトウェアの展開中に実行できないことの主要部分にすぎません。 ちなみに、この方法でアプリケーションを展開するには、アプリケーションを公開する必要があります。 上記の数行で述べたように、ユーザーは「プログラムと機能」コントロールパネルコンポーネントの「 ネットワークからプログラムをインストールする 」ページからそのような製品を直接インストールできます 。

そのようなファイルを作成し、テスト目的でのみ使用してIrfan Viewなどのソフトウェア製品を展開する前に、そのようなファイルの構造について少し説明すれば正しいと思います。 このようなファイルでは、最初の2行のみが必須であり、次のようなパラメーターがあります。

  • FriendlyName-つまり、コントロールパネルの「プログラムと機能」のコンポーネントに表示される名前です。
  • SetupCommand-アプリケーションインストールファイルへのパスはここで定義されます。 必然的に、UNCパスを指定し、マップされたドライブを使用できます。 また、追加のパラメーターを使用してアプリケーションをインストールできる場合は、インストールパスの後にこれらのパラメーターを指定する必要があります。


つまり、この方法を使用する場合、最初に各製品のインストールファイルのすべての追加パラメーターを調べる必要があります。

これらの2つのパラメーターに加えて、アプリケーションバージョンやその他の情報をCSE拡張機能の「プログラムのインストール」または「プログラムとコンポーネント」コントロールパネルのコンポーネントに表示するために、このようなファイルの書き込み中にいくつかの追加パラメーターを追加できます。 たとえば、次のオプションパラメータは最も一般的と見なされます。

  • DisplayVersion = 2.13.1.5 ご理解のとおり、このパラメーターは表示されるソフトウェアのバージョンに責任があります。
  • Publisherはアプリの製造元の名前です。 この名前は、コントロールパネルの「プログラムとコンポーネント」のコンポーネント、およびクライアント側の拡張機能「プログラムのインストール」に表示されます。
  • URL-当然、これはアプリケーションに関する詳細情報を目的としたインターネット上のアドレスです。


ZAPファイルの制限



既に説明したように、msiインストールパッケージとは異なり、zapファイルを使用して展開されたアプリケーションをコンピューターまたはユーザーに割り当てることはできません。 また、現在の方法でアプリケーションをインストールすると、標準のインストールプログラムが起動することにも注意してください。 つまり、追加されたパラメーターに加えて、このファイルの追加パラメーターを使用して、ソフトウェアのインストールプロセスを構成することはできません。 もう1つの制限は、管理者アカウントの代わりにアプリケーションをインストールできないという事実であり、場合によっては非常に不便です。 つまり、アプリケーションは常にローカル管理者によってインストールされます。

ZAPファイルを作成するときは、次の点に注意する必要があります（上記のいくつかの行で既に説明しました）。



  • まず、ZAPファイルを実行するだけでインストールプログラムが起動されるため、既に述べたように、このツールはインストールプロセスをMSIパッケージを使用して実装できる昇格された権限で実行できません。 したがって、インストール中にアプリケーションが権限の昇格を要求した場合、そのような許可が明示的に登録されているユーザーのみがインストールできます。
  • 第二に、ZAPファイルはテキストドキュメントであるため、そのようなファイルを保存すると、拡張子.zap.txtのファイルを誤って作成する可能性があります。これは明らかな理由により、正しく機能しません。 つまり、ファイルが正しい拡張子で保存されていることを確認してください。
  • 第三に、ネットワークに64ビットオペレーティングシステムを搭載したコンピューターがある場合、そのようなオペレーティングシステムで32ビットアプリケーションのインストールと動作を確実にテストする必要があります。 ZAPファイルを使用してデプロイされた多くの32ビットアプリケーションはインストール中に失敗する可能性があるため、これは非常に重要な手順です。
  • 4番目に、32ビットソフトウェアでZAPファイルを作成するときにその動作を変更しない場合、64ビットオペレーティングシステムのコントロールパネルのプログラムと機能のコンポーネントには表示されません。


ソフトウェアの公開を目的としたこれらのファイルの目的とアプリケーションに関する情報を既に学習しているので、この方法を使用してアプリケーションの作成と配布を試してみましょう。

展開プロセスは次のとおりです。まず、ソフトウェア製品のインストールプロセスに関する情報がZAPファイルに詳細に書き込まれます。 もちろん、インストーラーは公共の場所にある必要があります。そうしないと何も機能しません。 その後、生成されたファイルをパブリックソフトウェア配布ポイントに配置する必要があります。 これは、作成したGPOがこのファイルをローカライズできるようにするために行われます。 そして最後に、GPOオブジェクト自体が作成され、必要なユニットに接続され、クライアント側の「プログラムのインストール」の拡張により、分散ソフトウェア製品がユーザーに公開されます。

ZAPファイル自体の作成から始めて、これらのプロセスを検討してください。

.zapファイルを作成する



先ほど書いたように、Irfan Viewなどのソフトウェア製品を公開します。したがって、どこかからインストールファイルをダウンロードし、パブリックフォルダーに配置することをお勧めします。

さて、すでに述べたように、ソフトウェアの展開中に実行可能ファイルへのパスを書き込むだけで、そのインストーラーがユーザーに対して起動されます。 後者は、プログラムをインストールする前にウィザードのすべての手順を実行する必要があります。 多くのユーザーにとって、これは大きな問題になります。そのため、インストールファイルへのパスとともに、サイレントインストールのパラメーターを指定する必要があります。 幸いなことに、このソフトウェア製品はそのようなインストールの可能性を提供します。

Irfan Viewインストーラーには、この製品のサイレントインストール中に役立つさまざまなパラメーターの全リストがあります。 それらを見てみましょう：

  • サイレント -サイレントインストール、つまりユーザーの介入なし。
  • フォルダー -宛先フォルダー。 このパラメーターを使用して指定されていない場合、古いIrfanViewフォルダーが使用されます。 存在しない場合、フォルダ「Program Files \ IrfanView」が使用されます。
  • デスクトップ -デスクトップにショートカットを作成します。 0 =いいえ、1 =はい（デフォルト：0）;
  • 親指 -サムネイルモードのショートカットを作成します。 0 =いいえ、1 =はい（デフォルト：0）;
  • グループ -[スタート]メニューにグループを作成します。 0 =いいえ、1 =はい（デフォルト：0）;
  • すべてのユーザー -デスクトップ/ユーザーの[スタート]メニューのショートカット。 0 =現在のユーザーの場合、1 =すべてのユーザー。
  • Assoc-ファイルの関連付けを設定できます。 0 =インストールしない、1 =イメージのみ、2 =すべて選択する（デフォルト：0）;
  • Assocallusers-このオプションを使用すると、すべてのユーザーに対してファイルの関連付けが確立されます（Windows XPでのみ機能します）。
  • Ini-使用する場合、INIファイルのユーザーフォルダーを設定します（システム変数を使用できます）。


したがって、このソフトウェアをインストールするには、次のコマンドを使用します。

iview435_setup.exe / silent / desktop = 1 / thumbs = 1 / allusers = 1 / assoc = 1

このソフトウェア製品のインストールがどのコマンドで実行されるかが正確にわかったので、zapファイル自体の書き込みを開始できます。 したがって、このようなファイルを作成するには、次の手順を実行します。

  1. 同じメモ帳などのテキストエディターを開きます。
  2. 作成されたファイルの最初の行は、角かっこで囲まれたアプリケーションセクションを示しています。 つまり、この行に次のように記述します。 [Application] ;
  3. すでに上で書いたように、ここでは2行、つまりFriendlyName行とSetupCommand行が必要です。 1行目と2行目の両方で、値は引用符で等号の後に示されています。 記号はスペースで均等に囲まれています。 つまり、次の行が表示されます。

    FriendlyName =“ Irfan View”

    SetupCommand =“ \\ SERVER NAME \ Install \ ... 、そしてすべてのパラメーター“ iview435_setup.exe”を含むインストールファイルの名前/ silent / desktop = 1 / thumbs = 1 / allusers = 1 / assoc = 1

    必要に応じて、行の先頭にセミコロンを挿入してコメントを追加することもできます。 たとえば、新しい行に、プログラムのインストール方法を記述するコメントを追加します。

    ; このプログラムはサイレントモードでインストールされ、デスクトップ上のショートカット、サムネイルのショートカット、およびグラフィックイメージのみの関連付けを持つすべてのユーザー用のショートカットが作成されます。
  4. これで、プログラムのバージョン、発行元、サポートURLを決定する追加のパラメーターを処理できます。 つまり、 DisplayVersion、Publisher、およびURLパラメーターが適用されます。

    DisplayVersion = 4.35

    出版社= IrfanView

    URL = www.irfanview.com
  5. 結果のファイルをzap拡張子で保存します。 たとえば、このファイルに「IrfanView.zap」という名前を付けます。
