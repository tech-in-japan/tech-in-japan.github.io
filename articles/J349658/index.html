ケース「3次元視覚化を使用した銀行のビジネスローンポートフォリオの監視」

はじめに



この記事では、非標準の視覚ツールを使用して興味深い分析問題を解決した経験を紹介します。 この記事は、データ分析に携わる人々、および銀行の融資ポートフォリオの監視と分析を専門とする銀行マネージャーにとって興味深いものになります。



実際に以下で説明するアプリケーションは、iDVP（Interactive Data Visualization Platform）プラットフォームに基づいています。







それでは始めましょう！



この記事で解決した問題と、この記事で説明する問題は、次のように定式化されました。

当行は、大規模な法人-借り手に融資を提供しています。 銀行の経営者は、銀行のローンポートフォリオの全体像を見る（監視する）ことができる便利なツールを必要としています。 同時に、ローンポートフォリオ全体から見て、各借り手に関する詳細な情報に移行することが可能であるべきです。


リーダーはどのような状況で、彼は何を必要としますか？



  1. 経営陣は、アプリケーションの操作、視覚情報の解釈、データ分析に最小限の労力を費やしたいと考えています。
  2. 経営陣は、マウスを1回クリックするだけでアプリケーションを開くだけで、ローンポートフォリオのステータスをすぐに確認したいと考えています。
  3. 情報は「可能な限り」表示される必要があります-スクロールの必要なしに、1つの画面に すでに最初の画面で、ユーザーはどの借り手が「問題」であるか、どの「問題」であるか、量的および価値の観点からポートフォリオのシェアを確認する必要があります。
  4. データのフィルタリングとグループ化のためのツールは、便利で直感的でなければなりません。
  5. アプリケーション画面は、それを使用する経営陣が創業者と株主にレポートを「効果的に」提示できるように、「美しく」あるべきです。



銀行のアナリストとBIベンダーは、指定されたすべての要件を満たすソリューションを作成しようとしていますが、すべての要件を完全に実装できるわけではないため、作成されたソフトウェアソリューションは必ずしも経営者に好まれているわけではありません。 私たちは独自の方法で、可能な限り最高の品質ですべての要件を満たすソリューションを設計することにしました。



データ分析タスクへのアプローチについては、以前の記事で既に説明しました 。必要に応じて読むことができます 。



この記事のキーポイント
  1. 顧客を調査するとき、データ分析の助けを借りて解決できる顧客の痛み（問題）を常に特定しようとします。 そして、この問題を完全に解決するようなアプリケーションを作成します。
  2. データ分析には、「通常の」BIレポートではなく、3次元アプリケーションを使用します。 これらのアプリケーションでは、分析情報の視覚化は3Dオブジェクトの形式で実行され、テーマ別のインタラクティブシーン（画面フォーム）に結合され、論理的な遷移によって相互接続されます。
  3. 私たちが作成するソリューションには、その基礎に3つの原則があります。


  • 顧客のビジネス写真の視覚的表現。 アプリケーションの最初の知り合いで、最初の画面で、ユーザーは自分のビジネスの興味のある部分をすべて表示する必要があります。
  • 問題の原因の開示。 問題点を選択すると、ユーザーはドリルダウン機能を使用できるようになります。ドリルダウン機能を使用すると、問題のゾーンをさらに深く掘り下げ、次の画面フォームで問題の原因を確認できます。
  • 技術的な美学。 アプリケーションはすごい効果、つまり 魅力的で直感的で便利なものでなければなりません。


これらの原則は、機能要件とともにソリューション要件の形成において同等の位置を占めるべきであると確信しています。

列挙した最後の記事の論文に従って、ソリューションを作成し始めました。



アプリケーション設計の段階を思い出させてください：



  1. 問題の記述と作業の開始。
  2. 顧客調査とオープンソースの使用。
  3. 分析、要件および文書の作成。
  4. 最終文書「アプリケーションの説明」の作成。


詳細な説明は、これらのステップに構造化されています。



タスクの設定と開始



このフェーズの一部として、銀行の専門家とともに、顧客の主な「苦痛」はローンポートフォリオのステータスを監視することであると判断しましたが、特定の借り手にドリルダウンすることが可能であるべきです。



当然、アプリケーションは、上記の銀行の管理に関する特定の要件をすべて満たす必要があります。



顧客調査とオープンソース



調査中に、銀行のビジネスのこの方向について次の写真が得られました。



ほとんどの銀行の主な収入は、企業や一般に融資を提供することです。



一部の銀行は、人口への融資を専門とし、他の銀行は法人への融資を専門としています。



この銀行では、大規模な借入企業に発行されたローンを監視するタスクが特に深刻でした。 借入企業はさまざまな業界に属しているため、企業と業界の両方でポートフォリオ分析が必要です。



銀行では、借り手ごとにそのプロファイルが編集され、絶えず更新されます。これには、借り手の信頼性とその財務実績に関する情報が含まれています。



銀行のアナリストはまた、借り手のキャッシュフロー（キャッシュフロー）およびその他の指標に関する情報を収集し、キャッシュフローモデルを構築します。 情報は、銀行のいくつかの情報システムから収集されます。



収集された情報の分析に基づいて、借り手には問題が特定され、借り手は銀行が借り手をグループ化するために使用する5つの「問題領域」のいずれかに割り当てられます。



  1. グリーンゾーン-このゾーンには、ローンの返済に影響を及ぼす可能性のある問題を特定していない借り手が含まれます。
  2. 黄色のゾーン-借り手はいくつかの問題を特定しました。
  3. レッドゾーン-借り手は重大な問題を特定しました。
  4. ブラックゾーン-100％に近い確率を持つ借り手はローンを支払いません。
  5. ホワイトゾーン-借り手の問題はまだ計算されていません。


借り手の問題に応じて、銀行は、特別な口座に発生する可能性のある損失に対する引当金を設定する義務があります。その金額は、ローンの金額と借り手の信頼性によって異なります。 この点で、これらの埋蔵量の大きさを制御し、成長を防ぐ必要があります。 準備金は銀行にとって「死んだ」お金であり、使用することはできません。



銀行のアナリストはまた、不良債権（NPL）を分析します。 分析の結果によると、借り手は4つのNPLゾーンのいずれかに割り当てられます。



  1. グリーンゾーン-借り手によるローンの支払いは、延滞または最大4日間延滞しません。
  2. 黄色のゾーン-遅延は5〜29日です。
  3. レッドゾーン-30〜89日。
  4. ブラックゾーン-90日以上。


借り手のすべての指標を検討した結果、銀行はその格付けを計算し、この借り手がどれだけ信頼できるかを示します。



ローンごとに、適時の支払いとローン契約の他の条件の履行が監視されます。



次回の支払いが遅れた場合、銀行は遅れの理由を見つけ、借入会社に関して行動を起こします。 これは罰金または融資契約の条件を厳しくする可能性があります。



ローン契約には「契約」も示されています。これは、借入会社がローンを返済する能力に悪影響を与える行動をとることを禁止する特別な契約条件です。 契約の例としては、借り手が銀行に財務諸表を提出する義務、他の銀行の口座を閉鎖すること、他の銀行からのローンの取得の禁止、ローンの担保の提供などがあります。


分析、要件形成、文書化



このサブジェクト領域の監視を提供するアプリケーションの主な機能は次のとおりでした：貸出量の管理、借り手の信頼性または問題、およびその他の指標。



銀行の金銭的条件で「不良」なローンが多いほど、ローンポートフォリオの質は低下します。 したがって、銀行の経営者は、問題のある借り手の詳細な状況を確認し、それに関連するさらなる行動を決定できるように、「悪い」ローンと「悪い」借り手をすぐに見なければなりません。



結果として、アプリケーションを使用したユーザーマネージャーの作業は、「問題のある借り手を見つけて、彼の問題が何であるかを見つける」というゲームに似ている必要があると判断しました。



また、アプリケーションを銀行の管理に便利にするために、Windows用のデスクトップバージョンだけでなく、Mac OS、iOS、Android用のデスクトップバージョンも作成することにしました。 さらに、これらのアプリケーションを開発するプラットフォームにより、「ワンタッチ」と言われるようにこれを行うことができます。



分析の結果に基づいて、借り手ごとに追跡する必要がある次の指標が特定されました。



  1. 負債額
  2. トラブルゾーン
  3. NPLゾーン
  4. 引当額
  5. 借り手の評価


アプリケーションは、ユーザーに以下を許可する必要があります。



  1. 1つの画面ですべての借り手を表示します。 銀行が同時に最大1000人の大規模な借り手にサービスを提供することを覚えておく必要があります。
  2. 借主を借金でフィルタリングします。
  3. 問題のある分野で借り手をフィルタリングします。
  4. NPLゾーンで借り手をフィルターします。
  5. 借り手にローンを発行した銀行支店で借り手をフィルタリングします。
  6. 業界別に借り手をフィルタリングします。
  7. 借り手に特定された問題で借り手をフィルタリングします。


これらの要件と、記事の冒頭で示した要件を満たすレポート（または複数のレポート）を想像してみてください。 提示？ そして今、私たちのソリューションに慣れることをお勧めします。



前述したように、アプリケーションの利便性と美しさは非常に重要な役割を果たします。 そのため、アナリストだけでなく、3Dデザイナーやユーザビリティリストもアプリケーション画面の作業に関与しています。



その結果、iDVPアプリケーションのメイン画面、銀行、クレジット処理が得られました（下図を参照）。







一見、画面はかなり飽和しているように見えますが、同時にすべての情報がゾーンに分散されているため、認識が容易になります。 結果のゾーンは何ですか？



中央ゾーン







このゾーンでは、銀行の借り手はカラフルな惑星（ボール）として表されます。 惑星の大きさは、この借り手からのローンの負債額に相当します。 惑星の色は借り手の問題領域に対応しています。 同時に、同じ色の借り手がグループ化され、ローンポートフォリオのシェアを（定量的および負債額ごとに）視覚的に評価できるようになります。 したがって、「すべての借り手を1つの画面で見る」という問題を解決しました。



同じゾーンには、惑星のサイズに応じたフィルターがあります（惑星の右側にあるスケールと円に注意してください）。 このフィルターを使用して、表示された借り手の債務の最小額と最大額を指定できます。 たとえば、大きな借り手だけを画面に残すことができます。 「借金の量によって借り手を除外する」というタスクは解決されました。



任意の惑星をクリックすると、「借り手のカード」画面（下の写真を参照）に移動し、この借り手とクレジットの特徴を示すインジケーターの詳細情報を提供します。







「融資ポートフォリオの全体像から特定の借り手に移行する」という状況を分析するタスクは、最小限のクリック数で実行する必要があります。



小さな惑星の画面の初期状態では、クリックするのは必ずしも便利ではありません-マウスで、またはタッチベースのインターフェースの場合は指でそれらにアクセスするのは簡単ではありません。 この困難を補うために、中央ゾーンでは、惑星系の任意の部分をズームインおよびズームアウト（ズームインおよびズームアウト）できます。 これは、マウスホイールを使用するか、タッチスクリーンを使用している場合はアクション「ピンチ」を使用して行われます。



左のフィルター領域







このゾーンには、問題のある借り手のカラーゾーンに応じたフィルターが含まれています。 必要な/不要な問題領域をクリック/スナップできます。 その結果、ユーザーが必要とする色の借りている惑星のみが中央ゾーンに残ります。 「問題領域による借り手のフィルタリング」、「NPLゾーンによる借り手のフィルタリング」というタスクは解決されました。 問題のあるゾーンのみをフィルタリングするため、注意深い読者は、おそらくこのツールを使用してNPLゾーンごとに借り手をフィルタリングする方法を尋ねるでしょう。 簡単です。画面の左上に「RESIDUE DEBT」というテキストがあります。これは実際、借り手の表示モードを選択するためのドロップダウンリストです。 次のモードを選択できます。



  1. BALANCE OF DEBT-このモードでは、惑星のサイズは負債のサイズによって決まり、惑星の色は問題のゾーンによって決まります。
  2. NPL VOLUME-このモードでは、惑星のサイズは期限切れの負債の量によって決まり、惑星の色はNPLゾーンによって決まります。
  3. RESERVE-このモードでは、惑星のサイズは予備のサイズによって決まり、惑星の色は問題のゾーンによって決まります。
  4. RATING-このモードでは、惑星のサイズは評価値によって決まり、惑星の色は問題のあるゾーンによって決まります。


「NPLボリューム」モードでは、左側のフィルターがNPLカラーゾーンによるフィルターになります。



フィルターゾーン右







このゾーンには、3つのフィルターを含むフィルター要素「アコーディオン」が含まれます。



  1. CA + TB（中央事務所+領土銀行）-このフィルターを使用すると、中央事務所（銀行の本社）または領土銀行（支店）によってローンが発行された借り手のみを画面に残すことができます。
  2. INDUSTRIES-特定の業界の借り手をフィルタリングできます。
  3. 問題-このフィルターを使用すると、銀行アナリストが特定の問題を特定した借り手のみを画面に残すことができます。


「アコーディオン」要素の特性は、一度に1つのフィルターのみが展開されることです（「PROBLEMS」フィルターはスケッチ上で展開されます）。 残りのフィルターは折りたたまれた状態です。



「銀行の支店、業界、問題ごとに借り手をフィルタリングする」というタスクは解決されました。



下のグラフエリア







このゾーンには、問題のあるゾーンまたはNPLゾーンの比率の経時変化が表示されるグラフがあります。 これを行うには、グラフタイプ「累積付き折れ線グラフ」を使用します。 グラフの色は、問題領域またはNPL領域に対応しています。



ユーザーはチャート上の任意の日付にスライダーを設定する機会があり、中央ゾーンではその時点で銀行にいた借り手のみが表示されます。 惑星のサイズと色は、債務の額と、各借り手が選択した日に持っていた問題領域に対応します。
