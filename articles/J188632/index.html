Hyper-V 2012 R2の新機能

親愛なる同僚と愛好家の皆さん、こんにちは！



昔、私はHabrrに手紙を書きませんでした。何とかしてやることがたくさんあり、誰もがそれを手に入れることができませんでした。

しかし、製品サイクルのように、時間は止まりません（笑顔）。

夏は屋外で、Windows Server 2012 R2およびSystem Center 2012 R2のプレビューバージョンがリリースされます。

そのため、近い将来、興味深い記事や記事を掲載するのに十分な新機能が追加されました。

Windows ServerとSystem Centerの両方が同時に出てくるという事実に注意を喚起したいと思います。これは、これらの製品の最終RTMバージョンでも計画されています。つまり、最終的に（私も個人的に非常に満足していますが）以前に生じた多くの恥ずかしさから解放されます。







今日、私は、すでにバージョン2012 R2に成長しつつあるハイパーバイザーHyper-Vの分野での革新について注意を払い、話したい（またはむしろ伝えたい）。







そして、あなたのVMのどのバージョン？..





言及する価値のある主な革新の1つは、おそらく第2世代または第2世代VMの仮想マシンの外観です。

しかし、VMの新世代の出現に関して何が新しく、塩は何ですか？



まず、第2世代の仮想マシンは、仮想マシンの設定におけるすべての種類のCOMポート、仮想キーボードおよびマウスドライバーである仮想鉄ごみから完全に免れます。 仮想IDEコントローラーはアーカイズムに起因する場合もあります。これは、自動的に仮想SCSIディスクから起動できることを意味します 。 SCSIコントローラーが唯一のものになりました。







図1.第2世代の仮想マシンとVM作成ウィザードのプロパティ。



WS2012とWindows 8のみが第2世代VMのゲストOSとして適していることに注意する価値があります。ゲストサーバーOS、Windows Serverを自動的にアクティブ化することも可能です。 Windows Server 2012 R2 Datacenter



また、第2世代の仮想マシンは古き良きBIOSをspareしみません...

現在、彼はUEFIに置き換わっています。これは、すべてを意味します— セキュアブートセキュアブート 、合成ネットワークアダプターからのブート（つまり、レガシは忘却に沈んでいます）。







図2.仮想UEFIファームウェアのロードのプロパティ。



パフォーマンスの面では、これらの変更（およびそれらだけでなく） は、VM内へのOSのより高速なインストール （50％高速）とVMのより高速な起動 （20％の速度向上） を伴います 。



また、WS2012 R2を使用するホストのネットワーク仮想化設定を変更するという事実にも注目する価値があります。

以前、Windows Server 2012で、ネットワークアダプターまたはネットワークアダプターのプロパティにアクセスして特別なドライバーを含める必要があった場合、この機能は自動的に仮想スイッチに移動しました。







図3. WS 2012（左）とWS 2012 R2（右）のネットワーク仮想化設定の比較。



住宅問題または仮想マシンの移転





Hyper-V 2012 R2の変更は、仮想マシンの再配置、つまり移行にも適用されます。

それらのための良いニュース。 VMを既存のWS2012からWS2012 R2に移行することをすぐに考えました。

古いサーバーから新しいバージョンへの共有ストレージなしのライブマイグレーションはサポートされていますが、その逆はサポートされていません。

ホストは、写真や保存された状態があっても、VMを使用して大胆にライブにアップグレードできます。

また、転送中のデータ圧縮メカニズムによって、移行の単純な圧縮も補完されました。これにより、転送されるデータの量が減少し、移行時間が短縮されます。

また、 RDMAおよびマルチチャネルSMB（SMB-マルチチャネル）メカニズムは、再配置プロセスの触媒として機能します。

また、仮想マシンのスナップショットを特定のステータスまたは状態のVMに個別にエクスポートできるようになりました。







図4.スナップショットからVMをエクスポートします。



また、メモリの状態とともにVM全体をエクスポートする機能がエクスポートチップに追加されたことも追加する価値があります。 VMがオンになっている場合でも、動作しているVMのクローンを作成する可能性を示唆します。



ペンギンの風邪はいつですか？..





ペンギンに関しては、私はそれが非常に寒いのではないかと疑っています-それは一般的に寒さの中に住むために自然によって作成されます。 さて、「ペンギンカテゴリ」OSのサポートに関しては、革新の束もここに登場しました。



1）2012 R2では、VMを停止せずに動作中のVHDXディスクのサイズを変更できるため、これはWindowsだけでなくLinux VMでも同様です。

2）ゲストLinux OSを搭載したVMの動的メモリが完全にサポートされています。

3）Linuxカーネル用の改善されたビデオドライバー。

4）VM移行中を含む、Linuxを使用したVMのバックアップのサポート。



ゲストクラスター、ホームクラスター...クラスターの予約





仮想ディスクに関する変更を思い出せば、重要な革新があります。

共有仮想ハードドライブを作成できるようになりました（物理的に共有されたストレージ上にあります）-ゲスト仮想クラスターの作成について話すことができますが、SC VMM 2012 R2の観点からはサービステンプレートからクラスターを作成します！ これが自動化の力であり、クラウドの力です！ もちろん、WS 2012 R2 ...



クラスターについて説明している場合は、もちろん、ディスクサブシステムのパフォーマンスが非常に重要な役割を果たします。

ここでは、ディスクサブシステムのレベル、パフォーマンスレベルを作成する機会がありました。 システムには、SATA、SAS、SSDのいずれであるかにかかわらず、ディスクのタイプを判別するための十分なインテリジェンスもあります。



最後に、仮想マシンのレプリケーションの改善。

このメカニズムはWS 2012で初めて登場し、ご想像のとおり、バックアップデータセンターを使用して壊滅的なシナリオまたはバックアップシナリオを解決することを目的としていました。

WS2012 R2では、このメカニズムが進化し、現在では次の機会があります。



1）メインホストとスタンバイ間のVMレプリケーション間隔を設定します-間隔は30秒、5分、15分です。

2）マルチステップ複製の可能性、すなわち ホストAからホストBへ、およびホストBからホストCへのVMレプリケーションを構成します。



ハッピーエンド？





さて、今日、Hyper-V Server 2012 R2の仮想化分野のWS 2012 R2に登場した技術革新に出会いました。
