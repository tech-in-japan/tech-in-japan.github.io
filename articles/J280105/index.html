NetApp FlexClone：テクノロジーが開発を加速する方法

ここでは、データベースフォールトトレランスを確保するためのITインフラストラクチャの従来のアーキテクチャと、このアーキテクチャを使用する開発スキームについて簡単に説明します。 また、テストや開発に必要なデータベースまたはその他の大規模なデータセットを使用するFlexCloneテクノロジを使用して、テスターと開発者の生活を簡素化する方法について詳しく説明します。



従来のアプローチは、プライマリとセカンダリのデータベースクラスタを作成することです。 さらに、このデータベース（通常はスタンバイ側にある）は時々開発者にコピーされます。 また、データベースが大きいほど、コピーおよび削除される頻度が少なくなり、そのような操作からストレージシステムの負荷が高くなります。 一方、開発者とテスターは古いデータベースを取得します。通常は1か月、さらには6か月前です。 デバッグされたコードを本番環境に組み込むときが来ると、データベースが大幅に変更されたことがわかります。新しい追加データ、テーブル、フィールドが現れ、古いものが削除された可能性があります。 これは、古いデータベースのデバッグされたコードが新しいデータベースで機能しないという事実につながり、新しいコードの開発、テスト、実装、および操作を大幅に複雑にします。







FlexCloneは、大規模なデータベースでテスト/開発を実行するためのより効率的な方法です



FlexCloneテクノロジを使用すると、システムのパフォーマンスに影響を与えないNetAppスナップショットに基づいてシンクローンを作成できます。 クローンは、多くの場合非常に頻繁に、稼働日の途中で削除できます。 この単純な現象のおかげで、テスターと開発者の生活が大幅に簡素化され、コードの品質が向上します。 これは、コピーデータ管理（CDM）と呼ばれる人気のあるアプローチに適合し、1つのデータセットを複数のタスクに一度に使用することを目的としています。 このアプローチは、次の事実により便利です。



  • シンクローンでQoSを強制終了できます（これは、 UNMAPを使用する場合のスペース再利用には適用されません）
  • 時間がかからず、変更が必要になるだけです。読みやすいため、多くの変更はありません。
  • FullBackupのコピープロセスでは、ディスクサブシステムはロードされません。
  • すぐに削除され、長時間かかりません
  • テスターはより頻繁に最新のデータを受け取り、関連する（非常に古いデータではなく）コードをより適切にデバッグできます。 これは、コードのデバッグを高速化し、品質を向上させるために非常に重要です。
  • グリーススナップショットソフトウェアを使用して自動化することは難しくありません






これには何が必要ですか？



-FlexCloneのライセンス。

-ProdまたはStend-ByをONTAPファームウェア（ONTAP Select / ONTAP Cloud / NetApp FASプラットフォーム）を備えたストレージに配置します。これにより、FlexCloneテクノロジーを使用してシンデータベースクローニングが実行されます。

-データベースとNetAppのハードウェアアシスタントスナップショットを統合して、クローンの作成元となる一貫性を実現できる製品：



自動化および一貫性ソフトウェア


以下の製品がバックアップされ、それぞれがソフトウェア整合性のあるスナップショットを作成できます。



  • SnapCreatorは無料ですが、データベースをフリーズしてスナップショットを取得するには、スクリプトを使用して変更する必要があります。
  • NetApp SnapCenterは 、クローン作成プロセスを完全に自動化し、便利なGUIを提供する有料製品です
  • Commvault Simpanaは有料のバックアップ製品です。 すべてがSnapCenterと同じですが、カタログ化やテープへのアーカイブなどのトリックがあります。
  • データベースおよびNetApp ONTAPと統合してハードウェアアシスタントスナップショットを削除できるその他のサードパーティ製品（Symantec、 Veeam 、Syncsort、 EMC Networker 、 HP DataProtector 、 IBM Tivoli 、 Acronisなど）。
  • スタンバイからクローンを削除する場合、データベースを短時間停止して（その後、一貫性を確保するためにNetAppスナップショットを削除して）再度実行することができます。データベースと対話するスクリプトを作成せずに、一貫したスナップショットを削除するソフトウェアは不要です


一貫性のあるスナップショットの削除（上記のリストを参照）、クローン作成、およびデータベースへの接続の操作は、無料のワークフローオートメーションの Webベースのインターフェースを使用してテスト/開発部門のサービスとして提供することで簡単に自動化できます。



CodeEasy FlexCloneツールキット



CodeEasyは、スナップショットやFlexCloneなどのNetAppテクノロジーを使用してテストおよび開発時間を短縮する、DevOps用の一連のスクリプトおよび方法論です。 CodeEasy Toolkitスクリプトは、NetApp Manageabilitys SDKを使用して、開発者とテスター用のワークスペースを準備する手順を自動化します。これにより、変更なしで、または最小限の設定で使用できます。



システム要件：



  • ONTAP（cDOT）8.2以降
  • NetApp Manageability Software Development Kit（NMSDK）5.3.x以降


CodeEasy FlexClone Toolkitをダウンロードします 。



NetApp-PowerShellコマンドレット



Windowsマシンの場合、 NetApp PowerShell Toolkitを使用して、NetApp管理スクリプトを作成できます。



NetApp Docker統合



NetApp Dockerボリュームプラグイン（nDVP）は、DockerとNetAppストレージシステム（ONTAP、SolidFire、およびEシリーズ）間のリンクを提供し、コンテナー内に永続ストレージを作成して接続します。 このトピックは特別な注意に値し、 次のいずれかの記事で説明されます 。



バックアップ



FlexCloneは、バックアップ環境でバックアップの状態をテストするためにも頻繁に使用されます。 非常に多くのバックアップソフトウェアがこのプロセスを自動化できます。 たとえば、 Veeam On-​​Demand SandboxはONTAPと統合され 、FlexCloneの使用により、バックアップに影響を与えずに任意のサイズの仮想マシンからクローンを即座に作成し、クローンを実行して、その操作性と回復性をテストできます。



ワークフロー自動化（WFA）



Workflow Automationは、ONTAP管理プロセスを自動化するためのセットまたはタスクバンドルを作成できるWindowsおよびLinux用の無料のグラフィカルユーティリティです。 たとえば、ボリュームまたはムーンのクローンを構成し、それらをテストSVMに転送し、新しいファイルボールまたはiGroupを作成し、クローンされたボリュームと新しいホストイニシエーターをテストサイトから追加し、新しいLIFインターフェイスを上げることができます。 そして、複製関係が一時的に中断した後、データを複製して元に戻す-これはすべて、マウスを1回クリックするだけで行えます。 WFAは、REST APIだけでなく、PowerShellおよびPerlスクリプトを介して管理する機能も提供します。



サーバー仮想化とVDI



NetApp FlexCloneテクノロジーは仮想化環境でも使用されており、主要なグローバルハイパーバイザーメーカーとの統合により、仮想マシンのクローンを迅速に展開できます。 そのため、パフォーマンスの問題（VMware仮想ディスクシングプロビジョニング、VMware COWスナップショット、リンククローンなど）を発生させることなく、数千の仮想マシンを数分でデプロイできます。



性能



FlexCloneテクノロジーは、大規模なデータベースやその他のデータの展開を大幅に加速するだけでなく、ディスク容量の消費を大幅に削減するだけでなく、間接的に作業を加速することに注意することが重要です。 クローニングは、本質的に重複排除と同様の機能を実行します。 占有スペースの量を減らします。 事実、NetApp FASストレージは常にシステムキャッシュにデータを配置し、システムキャッシュとSSDキャッシュは重複排除に対応しています。 すでに存在する重複ブロックをドラッグアウトしないため、両方のキャッシュは物理的にできるよりも多くのデータを論理的に保持します。 これにより、ストレージの操作中、特にキャッシュとのデータのヒット/読み取りが増加するため、ブートストームの瞬間のパフォーマンスが大幅に向上します（a）。



追加のユースケース



テスト/開発に加えて、微妙なクローニングは他のアプリケーションを見つけるかもしれません。 たとえば、データへのアクセスを優先度の異なるタスクに分割することが理にかなっている場合があります。 DRサイトが事故の場合の予備としてだけでなく、開発環境（クローンデータ保護ボリューム）としても使用される場合、クローンテクノロジーはSnapMirrorレプリケーションの機能を完全に補完します。



トランザクションVSレポート


例としては、トランザクションデータベースとして使用されるデータベースがあり、月次または四半期ごとにレポートの作成にも使用されます。 トランザクションは通常、小さなランダムな読み取り/書き込み操作を生成します。このような操作は非常に正常にキャッシュされるため、低い応答速度（通常は10ミリ秒以下）で実行する必要があります。 一方、大規模な順次読み取り操作を生成するレポートがあり、結果を保存するときにレコードがあります（通常はもっと少なく、たとえば10％）。 レポートは月、四半期、または年に1回作成され、1日中収集できます。1日ではなく、たとえば3日間収集された場合、基本的な違いはありません。



したがって、レポートをトランザクションに追加すると、すべてのレポートが1時間または1日遅れることは重要ではありませんが、トランザクションが深刻な損傷を受ける可能性があります。 これにより、リソースが非常に不公平に分配されます。 これは、FlexClone、QoS、およびキャッシュポリシーが役立つ場所です。



シンクローンを作成し、その上でMB / sをスローできます。たとえば、制限なしでレポートを実行した場合の3倍、キャッシュポリシーを変更し、ディスクサブシステムへの影響を3倍、3倍減らすことができます。より長いレポート時間（通常はかなり許容されます）。 また、ストレージシステム上のこれらすべてのプロセスは、無料のNetApp Workflow Automationソフトウェアを使用して自動化できます。



クローンがデータベースクラスターのスタンバイから削除されるとき、すべて同じことをスキームに適用できます。本番トランザクションで直接ロードされませんが、メインデータベースから行われた変更に対応する必要があります。 また、速度が低下しないように、FlexCloneとQoSを再度使用すると便利です。



これらのタスクは、有料または無料のソフトウェアを使用して簡単に自動化できます 。



結論



FlexCloneテクノロジーにより、ストレージリソースの最適な使用（パフォーマンスとスペースの両方）が可能になるだけでなく、企業のビジネスプロセスが大幅に高速化され、新しいコードの開発とデバッグのプロセスが加速および簡素化されます。
