Kaviza VDI-in-a-Boxの展開

DEPO Computersのお客様がデスクトップ仮想化ソリューションの構築を依頼すると、一般的に理解できる問題に遭遇することがよくあります。CitrixXenDesktopやVMware Viewなどの主力製品は、IT予算が限られている小規模な成長企業にとって非常に高価です。



しかし、それほど前ではありませんが、Citrix SystemsはKavizaを買収しました。Kavizaは、中小企業向けのデスクトップ仮想化ソリューションであるKaviza VDI-in-a-Boxを開発しました。 名前が示すように、これは必要なすべてのツールを含むボックス化されたソリューションです（詳細な説明と他のCitrix製品との比較については、最近の発表を参照してください）。



DEPO Computersは長年Citrix Systemsと提携しており、その製品を仮想化ソリューションで使用しています。 当然、テストサイトにKavizaを展開し、それを使用して顧客に提供することがいかに簡単かを確認することにしました。



Kaviza（試用版はkaviza.comからダウンロードできます）を展開するには、XenServerまたはESXiハイパーバイザーが必要です（既に利用可能なベータ5バージョンはHyper-Vもサポートしています）。



Kaviza_VDI_kMGR_XenServer_v4_1_5.xva aplinsをハイパーバイザーにインポートします。







インポート後、仮想マシンKaviza_VDI_kMGR_XenServer_v4_1_5が作成されます。 彼女にIPアドレスが自動的に割り当てられたことを確認してください。 DHCPサーバーが構成されていない場合、コンソールでアドレスを手動で指定します（ログイン：kvm、パスワード：kaviza123）：



sudo vi / etc / network / interfaces

auto eth0 iface

eth0 inet static

アドレス10.0.100.33

ネットマスク255.0.0.0

ゲートウェイ10.0.0.1





DNSを指定するには、resolv.confファイルを修正する必要があります。

$ sudo vi /etc/resolv.conf




Kavizaサーバーを構成するすべての操作は、ブラウザーを介して実行されます。http：// [kMGRアドレス] / admin、ここで（[kMGRアドレス] KavizaサーバーのIPアドレス）。 Kavizaでインフラストラクチャを展開するには、いくつかの手順が必要です。



ステップ1. Kavizaサーバーの構成



管理ページに入るには、デフォルトのログイン（kavizaadmin）とパスワード（kaviza）を使用します。 最初のセットアップウィンドウが開き、ハイパーバイザーのIPアドレス/ DNS名、ユーザー名、およびパスワードをルート権限で入力する必要があります。 ストレージとネットワーク接続を選択し、[保存]をクリックして設定を保存します。



Kavizaは、XenServerサーバープールの一部であるサーバーには展開できないことに注意してください。この制限は、Kavizaサーバーをプールし、HighAvailabilityを提供する技術の実装に関連しています。 ESXiにはそのような制限はありません。







次に、グリッドを設定します。 「新しいKavizaグリッドを作成する」項目を選択し、名前を設定して保存します。 既存のグリッドに参加するには、「既存のグリッドにKavizaサーバーを参加」項目を選択し、構成とテンプレートを複製する必要があるKavizaサーバーのIPアドレス/ DNS名を設定します。 複製が実行されるサーバーにユーザー名とパスワードを設定し、パラメーターを保存します。 残りは自動的に行われます。







サーバーを構成する最後の手順は、ユーザーベースを選択することです。 内部ユーザーデータベースを作成するか、ActiveDirectoryデータベースを使用できます。 保存します。







サーバーのセットアップが完了すると、デスクトップを展開するためのテンプレートを作成するよう求めるウィンドウが表示されます。



ステップ2.「ゴールデン」イメージの作成



したがって、デスクトップをさらに展開するには、「ゴールデン」イメージが必要です。

仮想マシンを作成し、Windows 7、アップデート、XenToolsをインストールして、ドメインに入力します。 ビルトイン管理者アカウントとRemoteDesktopをオンにします。 RDPを介して接続できるActiveDirectoryからユーザーグループを追加します。 ファイアウォールが有効になっている場合は、RemoteDesktopを例外に追加します。 すべてのアクションは、ローカル管理者アカウントで実行することをお勧めします。



ヒューストン、問題があります


また、ここには、公式のKavizaドキュメントには記載されていないニュアンスがありますが、他のソースでは見ていません。 英語版のWindowsを搭載した仮想マシンでは、将来のデスクトップ用のイメージの作成に問題はありませんでした。 ロシア語版を使用する場合、Kavizaはロシア語のローカル管理者アカウントの受け入れを拒否しました。 この問題は、アカウントの名前を変更するだけで解決されます。

テンプレートを作成した後、デスクトップユーザーとして管理者を追加しようとすると、Kavizaは、それが見つからないことを通知します。明らかに、彼のアカウントは秘密の目的で使用されています。

Windows XPのインストールは、インストール中にアクティベーションキーを直接入力する必要があるという点でのみ異なります。



適切なリストで、「ゴールデン」イメージにインポートする構成済みの仮想マシンを選択し、「OK」をクリックします。 表示されるウィンドウで、画像の名前と説明を設定します。



インポートの準備が整っていない仮想マシンは、インポートにアクセスできないセクションに一覧表示されます。たとえば、XenToolsがインストールされていない、RemoteDesktopサービスがオンになっていない、仮想マシンがオフになっているなどです。







画像の作成が開始されます。これは、進行状況バーで追跡できます。 イメージの準備ができたら、RDP経由でイメージに接続し、OSのビット深度に対応するエージェントをインストールする必要があります。 エージェントをインストールすると、仮想マシンが再起動し、[次へ]ボタンがアクティブになります。







次の画像準備ウィンドウで、RDPとHDXプロトコルを使用して接続を確認します。 また、このステップでは、必要なソフトウェアまたは更新をインストールできます。



セキュリティグループ「Direct RDP Access Administrators」に、デスクトップに接続するドメインユーザーグループを追加します。 すべての要件が満たされていることを確認し、チェックボックスに注意してください。 さらに進んでいます。







次のステップは、sysprepユーティリティを実行することです。 フォームデータを入力し、タイムゾーン、コンピューター名のプレフィックス、アクティベーション方法を選択し、ドライバーを再インストールします（既定で推奨）。 [準備]をクリックし、sysprepが完了するのを待ちます。







次に、接続の可用性とインストールされたソフトウェアの保守性についてイメージをテストし、リストを再度確認して、[次へ]をクリックし、「ゴールデン」イメージを保存します。







作成された画像は、公開のカテゴリに入ります。 当然、必要に応じて、「ゴールデンイメージ」を編集できます。ソフトウェアの追加、アップデートのインストールなど。 -手順は同じです。



ステップ3.デスクトップテンプレートを作成する



デスクトップテンプレートを作成するには、[テンプレート]タブに移動し、緑色のプラス記号が付いたボタンをクリックします。 表示されるウィンドウで、テンプレートの名前、説明のフィールドに入力し、テンプレートを作成する画像を選択し、RAMの量を設定します（Windows XPでは512 MB以上を推奨、Windows 7では1536 MBから）、色深度、デスクトップの最大数、preの数デスクトップを実行している場合は、デスクトップ更新モードを選択し、[保存]ボタンをクリックします。







[管理]タブに移動し、[ユーザーの管理]をクリックして、ユーザーまたは対応するIDをActiveDirectoryに追加します（ユーザーグループの場合-グループの管理）。 [デスクトップテンプレートの割り当て]リンクは、利用可能なテンプレートをユーザーに割り当てます。



仮想マシンへのアクセスは、ブラウザ（http：// [kMGRアドレス] /）、Javaクライアント（http：// [kMGRアドレス] / dt / kavizaclient.jnlp）またはCitrixプラグインを介して行われます。 Javaクライアントを使用する場合、デフォルトの接続はHDXプロトコルを介して行われ、利用できない場合はRDPを介します。 モバイルデバイスからも接続できます。



非常に短い結論



それは、一般に、すべてです。 これまで見てきたように、Kavizaは非常にシンプルで使いやすい製品であり、数時間でユーザーへのデスクトップ配信を構成できます。 上記のいくつかのニュアンスを除いて、Kavizaでの作業に問題はありませんでした。
