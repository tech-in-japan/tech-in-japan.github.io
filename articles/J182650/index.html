先史時代のPython：過去のパターン

Lennart Regebro-1994年に彼の最初のWebサイトを作成し、2001年からオープンソースWeb開発にフルタイムで取り組んでいます。彼は本「Porting to Python 3」の著者です。 Lennartは独立した請負業者で、主にPloneまたはDjangoを使用してWebサイトとWebアプリケーションを開発しています。

Pythonの変更によって廃止されたプログラミングパターンを通過しますが、実際の製品コードにはまだあります。 DjangoやPlone / Zopeなどのフレームワークでの時代遅れのコーディング手法の例を取り上げ、それらが何をするのか、なぜこれが良いアイデアではないのか、代わりに何をすべきかを説明します。



チューリップ-非同期コードの新しい標準

Andrey Svetlov-2000年からPythonを使用しています。 彼は、言語とPython Core Developerのような標準ライブラリの開発に従事しています。 チューリップの作成に参加しています。 現在getgoing.comのプログラマーasvetlov.blogspot.comによる投稿



Boost.Python-多数のC ++ / Pythonを使用して離陸

Vladimir KerimovはC ++ / Python開発者です。 彼は、C ++コード、C ++ / Pythonバンドル、およびネットワークプロトコルの最適化を専門としています。 現在、TensorでSBiS 3.xプラットフォームを開発しています。



Leap MotionとPythonを使用した次世代UI

パベル・コロミエツ。 2003年からWebプロジェクトを開発しています。 2010年以来、彼はInsolloでゲームの開発を率いてきました。 彼はPython3（！）、Zeromq、redis、その他の高度なテクノロジーが好きです。



Django ORMはいつ継ぎ目でクラックし始めますか？

ミレンコ・アレクサンダー。 私は2000年からウェブ開発に従事しています。 2009年から現在まで、彼はPythonとDjangoに焦点を当ててきました。
