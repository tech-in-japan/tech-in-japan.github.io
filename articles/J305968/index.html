DotNext 2016 Moscow .NET Conference、12月9日

12月9日に、6回目のDotNext .NET開発者会議が開催されます。これはDotNext 2016モスクワと呼ばれ、ご想像のとおり 、モスクワ（Hotel Slavyanskaya）で開催されます。 ちなみに、DotNextは600人のシニア/ミドルの.NET開発者を集めたロシアで唯一の.NET会議です。







今回の新機能は？ 猫をお願いします。



スピーカー



それぞれのDotNext、およびこれにより、年に2回、スピーカーの構成がより強くなることを思い出してください-強力で評価の高いレポートを持つスピーカーは、スピーチを磨き、新しい脳吹きヒットを準備し、ロシアの同僚と彼らの経験を共有したいと思っている外国の専門家の数は着実に増えています。 乾燥した数値に感謝します。2014年に英語で2つのレポートしかなかった場合、2016年6月にはすでに6つのレポートがありました。





それでは、誰がすでに参加を確認していますか？ もちろん、過去のDotNextの最高のレポートの著者：



  • Sasha Goldshtein 、Sela GroupのCTO、Microsoft C＃MVP、Azure MRS、Pluralsightの作成者、Pro .NET Performanceの作成者、世界最高の.NETパフォーマンスエキスパートの1人。 最高のスピーカーDotNext 2016 Piter;
  • Dino Esposito 、ASP.NETの一連の本の伝説的な著者、CQRSおよびBDDの専門家。
  • Andrey DreamWalker Akinshin 、.NET MVP、博士、ACM ICPCの銀メダリスト、JetBrainsのRider開発者の1人。 彼はマイクロ最適化が好きです。 BenchmarkDotNetプロジェクトのメンテナー。
  • マイクロソフトのテクノロジーエバンジェリストであるDmitry Soshnikov博士は、最新のソフトウェア開発テクノロジーを普及させています。
  • パフォーマンスとフォールトトレランスを専門とする開発者であるAdam Sitnik 、BenchmarkDotNetおよびcorefxlabプロジェクトへのオープンソース貢献者。
  • ReSharperとProject Riderの開発に直接関与した、JetBrainsの.NET製品部門の責任者であるSergey Shkredov 。


ただし、非常に興味深い「新人」がいます。



  • Marco Cecconi 、Stack OverflowのフルスタックWeb開発者。
  • イベントストアLLPの熱心なオープンソース開発者であるJames Nugentは、PackerおよびTerraformとも協力しています。


もちろん、これは決してすべてではありません。プログラム委員会にはまだ多くのアプリケーションがありますので、アップデートに従うことをお勧めします。 ところで、スピーチのために出て行くべき人に関連したアイデアや提案がある場合は、この投稿へのコメントを書いてください！



講演と論文募集



スピーカーを整理しましたが、レポートのトピックはどうですか？ 心配することはありません。DotNextは、経験豊富な開発者にとっては筋金入りの会議です。 これが真実であることを確認するために、プログラム委員会が注意を払っています。





Microsoft C＃MVP、Azure MRS、Pluralsightの作成者、Pro .NET Performanceの作成者、および世界最高の.NETパフォーマンスエキスパートの1人。 .NET MVP、Ph.D.、JetBrainsのRider開発者の1人であるACM ICPCの銀メダリスト。 彼はマイクロ最適化が好きです。 専門分野：静的および動的コード分析、情報セキュリティ、.NET CLR内部の研究。 彼は、ビジネスアプリケーション、分散型およびフォールトトレラントシステムの設計と構築に従事しています。 ES、CQRS、およびDDDで休憩します。


ご覧のように、プログラムに不快な人は、24時間以上CLRの「根性」を掘り下げなければプログラムに参加します。したがって、コンテンツについて心配する必要はありません。頭を痛めるものがあります。 このプログラムは、.NETプラットフォームの現在と未来、パフォーマンスとマルチスレッドの最適化、.NETプラットフォームとCLRの内部構造、および.NETコードのプロファイリングとデバッグに関する20を超えるテクニカルレポートを計画しています。 さらに、製品のマイクロサービス、開発ツール、コード生成についても説明します。



ロシアの.NETコミュニティについて何か技術的なことを話したい場合は、レポートを送ってください ！







すでにどこかで演奏し、主催者と聴衆から温かい歓迎を受けている場合は、ビデオレポートを見せてください。これにより、プログラム委員会の決定が大幅にスピードアップし、話す機会が増えます。 過去のクラスのパフォーマンスは、プログラム委員会が伝統的に指導してきた重要な基準の1つです。



初心者も恐れるべきではありません-私たちは間違いなく皆と話をします。 さらに、トピックが新鮮で興味深い場合は、レポートの内容、プレゼンテーション手法、タイミング、およびプレゼンテーションを磨いて、完璧なレポートを作成できるように最善を尽くします。
