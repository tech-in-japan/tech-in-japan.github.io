急いでFlashを埋めないでください-Flashテクノロジーの応用分野の概要

ネットワークとHabrでの非常に興味深いデバイスとSteve Jobsによる「スキャンダラスな」声明の発表後、特にFlashは時代遅れであり、すぐに有望なHTML5 \ CSS3テクノロジーに置き換えられるという意見が形成され始めました。 Web開発者の観点からは、状況は本当にこのように見えますが、Flashアプリケーションの他の領域、つまりプログラマが通常職業に基づいて毎日触れない領域を思い出したいと思います。



Web







CSS3とHTML5は間違いなく強力なものです。 情報を表示およびフォーマットします。

スターウォーズの歩行ロボットは間違いなく美しいものですが、それはむしろ「ハンマーでネジをハンマーで打つことを学んだことを見てください」という領域のものです。 開発者は通常、アニメーションを実行しません。アニメーターは、すべてがより速く、より明確に実行できる場合、多くのコードを好まないため、書きたくありません。

例に目を向ける必要はありません; FWA：Favourite Website Awardsをご覧ください 。 CSS \ Silverlight \タブレットでマウスまたはペンで描画できる機能が使用可能になるまで、Flashはイメージサイトで長期間存続します。Flashは、ビジネスロジックや使いやすい高速インターフェイスではなく、まず「すごい効果」が必要です。



アニメーション







Masyanya 、 Happy Tree Friends 、 Antimultなどの有名なフラッシュプロジェクトは、おそらくすべてを知っています。 しかし、カンヌ映画祭の勝者である映画「 ワルツとバシル 」では、Flashも使用されていたという事実については知っていますか？ たとえば、「Adobe Flashテクノロジーの使用分野と応用分野」というトピックに関するリンクを探していたときに、今日それについて知りました。 ちなみに、子供がいるだけでなく、テレビやカートゥーンネットワークやニコロデオンチャンネルへのアクセスがある場合は、Scunk Fu！、Foster's Home for Imaginary Friends、Little Einsteinsなどの漫画を見たことがあるはずです。 これもフラッシュです。 彼のおかげで、フィルムを着色したり、中間フレームを描いたりするためにスタッフをたくさんの人から遠ざける必要はありません。 現時点では、私の意見では、生産的な作業に必要なすべてを含むこのようなプログラムはありません（ アニメStudoのようなプログラムはありませんが、私はそれを使用しませんでしたが、アニメーターの意見から判断すると、いくつかの欠点があります）



ゲーム







JavaScriptおよびAJAXのゲームは次のとおりです。 そして非常に美しい。 ただし、CSS3の場合と同様に、実際の製品よりも機能のデモについて説明しています。 JavaScriptでMachinariumを書くことは、自由な時間がたくさんある非常に必死のハッカーに過ぎません。

そして、これはDofus 、 Magic.ru 、 TimeZeroのようなMMOは言うまでもなく 、 それらのいくつかは非常にクロスプラットフォームであり、ブラウザが動作する必要はありません。

C ++などのプログラミングゲーム用の「実際の」言語に関しては、ActionScript3はより単純であり、それほど基本的な知識は必要ありません。



広告



おそらく特別なケースですが、それでも言及します。 ミニバスでは、多くの場合、さまざまな強迫的な広告が表示される運転席の上にモニターが設置されます。

車が起動し、コンピューターがオンになり、Linuxが起動し、広告付きのFlashプレーヤーが全画面で起動します。 ちなみに安くて陽気な。



RIAのテーマ、バナー、スライドショー、その他のプレゼンテーションはここでは影響を受けません。 フラッシュは良い感じですが、それなしでもできます。



一般的に、ここでの教訓は次のとおりです。Flashの差し迫った死に関するすべてのうわさは非常に誇張されており、その適用範囲は、ご覧のとおり、Webとブラウザだけに限定されません。 そして、インターネット上で今後数年でそのシェアが減少する可能性がある場合、アニメーションやゲームではこれはすぐには起こりません。 Flashプログラマーとイラストレーターは、長い間需要があります。
