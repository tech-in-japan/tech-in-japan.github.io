Debian LennyをAndroidにインストールする

まあ、Androidが出てきて、ソースコードさえもレイアウトされています。 そして、最初のAndroidデバイスを入手しました。 私たちが本当に持っていなかったのは、突然電話をコントロールできるようなゲームの変更でした。 昨年Googleが私たちに約束したことを覚えていますか。

現実には、Googleでさえモバイルデバイスの世界に影響を与えることはできません。 すでにG1を受け取り、しばらくG1で遊んでいる人たちは、私たちがまだ自分の携帯電話の普通のユーザーであるという事実に腹を立てました。T-Mobileは、何もしないことを恐れて、rootユーザーの使用を許可しませんでした。



この指示を実際に試してみたい場合は、あなたがすることはすべてあなた自身の危険とリスクで行うことに留意してください。



スーパーユーザー特権の取得（ルート）/ジェイルブレイク



私にとって、この種の制限はデバイスを単なるおもちゃに変えるものです。iPhoneの世界で私の冒険を追いかけてきた人たちは、多くの方法で最新のオペレーティングシステム、大画面を実行する高速ARMプロセッサを搭載した携帯電話を着用するのは恥ずべきことだと思っていることを知っています別の携帯電話として使用するために、制御/入力。



幸いなことに、これは数日前に誰かがAndroidファームウェアに重大な脆弱性を発見したときに変更されました。 特権を取得することは可能でした。 コンソールでtelnetdと入力し、telnetクライアントを使用して接続するだけで、rootになります。



残念ながら、Googleのbusybox-toolboxを置き換えることで価値のあることを行うのは依然として困難です。 必要なのは、本格的なUnix環境です。 この電話は、アプリケーションを直接開発するのに十分なほど強力です。



Debian ARMELのインストール



アンドロイドにbusyboxをインストールしようとする試みはすでにいくつかあります。 私たちはさらに先に進むことができると思います。この手順で、完全なDebianディストリビューションをインストールします。これは、好きなものをインストールできる非常に尊敬されているLinuxディストリビューションの1つです。

これを行うには、G1に関連するいくつかの質問を検討する必要があります。 まず、どこに保管しますか？ 電話機にはフラッシュメモリが組み込まれていますが、オペレーティングシステムやその他のアプリケーション用に128MBしかありません。



そのため、電話のより容量の大きい部分、microSDカードに注目しました。これにより、最大16GBのデータを保存できます。 残念ながら、すべてのカードリーダーで読み取ることができるように、このカードにはFATファイルシステムがあり、Unixのプログラムやデータの保存には役に立ちません。

しかし、ここに私たちが思いついたものがあります-Debianイメージをメモリカードに保存し、それを単にマウントすることができます。



/ proc / filesystemsの内容を見ると、使用できるすべてのタイプのファイルシステムを見ることができます：vfat、yaffs、yaffs2。 T-Mobileでは、Androidの機能に必要なファイルシステムのドライバーの最小部分のみがインストールされます。 これは、カーネルがモジュールをサポートして構築されているため、イメージをマウントできるドライバーをロードする必要があることを意味します。



必要に応じて、G1用のモジュールを組み立てる方法があるため、マニュアルを作成できます。 G1の開発/修正のトピックについて議論することに興味がある人を奨励します。 議論のために、G1-Hackersメーリングリストを購読できます。これはTelesphoreoで管理しています。

また、＃androidチャンネルのirc.saurik.com（電話指向のチャンネルが配置されているサーバーと同じサーバー）で、IRCをサポートしています。



Debianディスクイメージの作成



だから、話をやめて、始めましょう！ まず、Debianファイルシステムイメージが必要です。これを電話のメモリカードに配置します。 作成するには、既存のDebianシステムを使用できます。 幸いなことに、DebianはすでにARM EABIを完全にサポートしており、インストール方法についての有用な指示さえあります。



イメージの作成を台無しにしたくない人のために、すでに750MBの準備ができています。ここから完全なイメージをダウンロードできます（RapidShare）。 このイメージはbzip2を使用してパッケージ化されており、約85MB必要です。



RapidShareはかなり遅いサーバーであるため、 modmyGphone.comのメンバーはサーバーにイメージミラーをアップロードしました。 詳細はこちらをご覧ください 。



自分でイメージを作成したい場合（おそらく750MB以外のDebianの異なるファイルシステムサイズが必要な場合、または単にプロセス全体を制御したい場合）、これは次のコマンドを使用して実行できます。



apt-get install debootstrap

dd if=/dev/zero of=debian.img seek=749999999 bs=1 count=1

mke2fs -F debian.img

mkdir debian

mount -o loop debian.img debian

debootstrap --verbose --arch armel --foreign lenny debian ftp.de.debian.org/debian

umount debian







Debianツールキットを構築する



これで、debianの画像ができました。メモリカードにコピーする必要があります。 また、追加のファイルをコピーする必要があります。 このステップの重要なポイントの1つは、使用しているファームウェアに対応するドライバーをインストールする必要があることです。



* ext2.ko（ RC19 ）（ RC29 / 30 ）-標準Linuxファイルシステムモジュール

* unionfs.ko（ RC19 ）（ RC29 / 30 ）-フォルダを組み合わせることができます（高度）

* busybox-システムユーティリティのセット



このすべて（およびdebian.img）をmicroSDカードのフォルダーにコピーします（USB接続を使用しました。これは、他よりも簡単で高速です）。 完成したイメージをダウンロードした場合は、後で使用しやすいように、名前をdebian.imgに変更するのが理にかなっています。



インストールと取り付け



これらすべてをどこに置けばよいでしょうか？ ツールキット（キット）をmicroSDカードのルートディレクトリにコピーし、他のすべてを/ data / local（通常の書き込みが可能な非常に便利なフォルダー）に配置しました。



常に長いパスを収集しないように、パスをエクスポートします。 また、この命令は意味的に読みやすくなります。



export kit=/sdcard/kit

export bin=/data/local/bin

export mnt=/data/local/mnt







次に、さらにいくつかのパスをエクスポートする必要があります。これらのパスは、今後使用するプログラムで必要になります。 これにより、ext2のドライバーをロードする時間を節約できます。 これは奇妙に見えるかもしれませんが（少なくともHOME変数）、後で質問しないと、大きな問題が発生する可能性があります。



export PATH=$bin:/usr/bin:/usr/sbin:/bin:$PATH

export TERM=linux

export HOME=/root



insmod $kit/ext2.ko







次のステップは、将来の使用のためにbusyboxをどこかにコピーすることです。 G1にはcpコマンドが含まれていないため、これにはcatを使用します。 注：mkdirコマンドには-pオプションがないため（ツールボックスではサポートされていないため）、ネストの大きいディレクトリ（または既存のディレクトリ）を選択した場合、いくつかのエラーを無視するか、別の方法で解決する必要があります。



mkdir $bin #-p

cat $kit/busybox >$bin/busybox

chmod 755 $bin/busybox







busyboxがインストールされました。これを使用して、ループバックドライバーのデバイスノードを作成できます。 G1にはノードを作成するために必要なmknodユーティリティが含まれていないため、busyboxが必要です。 また、busyboxのエイリアス（短い名前）を作成し、コマンドを入力しやすくします。



alias _=busybox

_ mknod /dev/loop0 b 7 0







起動時にエラーが発生した場合：_が見つかりません、リセットしてみてください：



unalias _

alias _=busybox







最後に、ループバックデバイスが作成され、debianイメージを接続できます！ debian.imgの画像名として、異なる場合は画像の名前を使用してください。



_ mkdir -p $mnt

_ mount -o loop $kit/debian.img $mnt







ここでいくつかの基本的なエラーが発生する可能性があります：エラーが/ etc / fstabの場合、明らかに$ kit / debian.imgファイルが存在しないか、$ mnt変数が設定されていない可能性があります。



最終インストール段階



この時点で、debootstrapが実行できないことで終わる必要があります。 まず、いくつかのスクリプトを実行し、DebianリポジトリへのURLを調整する必要があります（何らかの理由で、debootstrapは何らかの理由でそれらを混乱させます）。 完成したイメージを既にダウンロードしている場合、これらの2つのコマンドを実行する必要はありません。

_ chroot $mnt /debootstrap/debootstrap --second-stage

echo 'deb ftp.de.debian.org/debian lenny main' >$mnt/etc/apt/sources.list







これらのコマンドが実行された後（10分近くかかることがあります）、chrootコマンドを使用して環境をdebianに変更できます。 シェルと完全なDebian環境が用意されており、できることはすべて揃っています。 このコマンドは、完成したイメージをダウンロードした人も含め、全員が実行します。



_ chroot $mnt /bin/bash







次に、devpts、proc、sysfsファイルシステムを環境に接続します。



mount -t devpts devpts /dev/pts

mount -t proc proc /proc

mount -t sysfs sysfs /sys







また、rootユーザーのパスワードを設定して、電話にリモート接続できるようにする必要もあります。



passwd root







OpenSSHをインストールする



これは、Debianのフルパワーが発揮される場所です。電話での新しいプログラムの迅速なインストールです。 さて、完全なコンソールを取得するためにSSHサーバーをインストールしましょう。

このために、APT（Debianパッケージ管理システム）を使用します。 最初のコマンドは、リポジトリ内のパッケージ情報を更新し、更新をインストールすることです。 2番目は、sshサーバー自体をインストールします。

apt-get update

apt-get install openssh-server







後でOpenSSHを再び起動する場合（Debianサービスの通常の起動は使用されないため）、次のコマンドを使用してこれを行うことができます（または「start」の代わりに「restart」を使用します）。



/etc/init.d/ssh start









次は？



楽しみましょう！ 私が最初にしたことは、Subversion、gcc、vimをインストールして、開発を開始できるようにすることでした。 繰り返しますが、Androidの内部（カーネルドライバー、ハードウェアへのアクセス、フラッシュプロセス）について詳しく知りたい人は、 G1-Hackersメーリングリストに参加できます 。



電話を再起動した後、すべてではないにしても、これらのコマンドのほとんどを再度実行する必要があります。 私と同じパスを使用した人は、スクリプトを使用して再接続できます（ script ）。



また、画像がマウントされている間は、USBケーブル経由でAndroidを使用できないことに注意してください。 これを行うには、以前にマウントされたすべてをアンマウントする必要があります



_ umount $mnt/dev/pts $mnt/proc $mnt/sys $mnt







Androidにはumountアンマウントユーティリティが付属していますが、これは正常に機能しますが、それでも、USB接続のためにmicroSDカードをブロックし続けるループバックデバイスが残ります。 次のコマンドを使用して、手動で削除できます。



_ losetup -d /dev/loop0







/でDebianを実行する



このプロセスの最大の動揺は、アンドロイドとデビアンのどちらかを選択する必要があることです。 ここでUnionfsが助けになります。



後続のコマンドはすべて、debian環境外で実行する必要があることに注意してください。 このためには、現在のtelnetセッションで入力する必要があります。

exit







または別のセッションを開きます。 すべてのコマンドは、 完成したスクリプトとしてダウンロードできます 。

このスクリプトを実行するには、すべての標準変数を既にエクスポートしている必要があります。 スクリプトを$ kitにコピーした場合、次のコマンドを使用して実行できます。



. $kit/unionfs.sh







wgetを使用してこのスクリプトをインストールできます。



_ chroot $mnt wget -O /tmp/unionfs.sh cache.saurik.com/android/script/unionfs.sh

cat $mnt/tmp/unionfs.sh >$kit/unionfs.sh

rm $mnt/tmp/unionfs.sh









既製のスクリプトを使用しなかった場合：



insmod $kit/unionfs.ko

mount -t unionfs -o dirs=$mnt/etc=rw:/etc=ro unionfs /etc







これで/ etcにはAndroidとDebianのファイルが含まれています。 / etcでファイルを変更/作成すると、すべての変更がDebianセクションに保存されます-これにより、/ etcへのフルアクセスが可能になります！



次の問題は、AndroidとLinuxが動的リンカーに異なる命名規則を使用していることです。 Androidの場合、/ system / bin / linkerがあり、Linuxの場合は/lib/ld-linux.so.3があります。 これは、作業プログラムを開始するときにファイルが見つからないというエラーを受け取ることを意味します。 これは、シンボリックリンクを使用して簡単に修正できます。



_ mount -o remount,rw /

_ ln -s $mnt/lib /







一見、これは危険に見えるかもしれませんが、そうではありません。 電話のルートファイルシステムを変更すると、「rootfs」：特別なLinux ramfs状態（メモリ内ファイルシステム）に変わります。 これは、電話機の再起動後に行った変更がすべて消えることを意味します。



$ mntからプログラムを実行するだけで、chrootを使用せずにほとんどのDebianプログラムを実行できるようになりました。 残念ながら、ほとんどのファイルは本来あるべき場所にないため、すべてが機能するとは限りません。 シンボリックリンクで修正しましょう。



for x in

bin boot home media mnt

opt selinux srv usr var

do

_ ln -s $mnt/$x /

done







修正が必要なディレクトリがまだいくつかあります。 最初：/ルートは空なので、リンクに置き換えることができます。 /のファイルの変更が完了したため、/ルートを読み取り専用として再マウントすることを強くお勧めします。



rmdir /root

_ ln -s $mnt/root /

_ mount -o remount,ro /







さらに2つの/ sbinと/ devが残っています。 / devと大したことはありません。 / dev / ptsはこのフォルダー内にマウントされているため、unionfsを使用してマージすることはできません。 この問題を解決するには、unionfsの上に/ dev / ptsをマウントするだけです：



mount -t unionfs -o dirs=$mnt/sbin=rw:/sbin=ro unionfs /sbin

mount -t unionfs -o dirs=$mnt/dev=rw:/dev=rw unionfs /dev

mount -t devpts devpts /dev/pts







opensshのようなプログラムが正しく動作するようになっても、すべての準備が整っています。サービスを再起動します。



/etc/init.d/ssh restart







結論として



それはすべて良いように見えますが、まだ注目に値するいくつかの問題があります：

まず、ユーザー名と認証情報が保存されている場所はまだ明確ではありません。 Debianの一部であるlsを使用してファイルの所有者を見ると、彼らはそこにいないことに気付くでしょう。



2つ目は、chrootを使用する代わりに/にシンボリックリンクを使用する（これまでのところ、すべてのサービスを透過的にロードできなかった）ことは、/にディレクトリを追加しようとするパッケージが失敗することを意味します。 apt-getを介してプログラムをインストールするには、chrootを使用する必要があります。



そして、最後のRC30アップデートが近日公開され、その後Telnetdを介したジェイルブレイクは無効になります。 そのため、十分にプレイするまではインストールしないでください。
