HeroCraft会社：内部からのモバイルアプリケーションの開発[インタビュー]

「HeroCraft」は、携帯電話向けのゲームやその他のソフトウェア製品の開発と公開に携わっているロシアの企業であり、現在、新しいプロプライエタリなバダプラットフォーム向けのゲームの開発で当社と協力しています。 2002年に設立され、今日では、ロシアのモバイルアプリケーション開発者の間で重要な位置を占めています。



habrakatでは、HeroCraft全体の仕事と、同社に新しい開発の見通しをもたらしたSamsungとの仕事の両方について、そのリーダーの数人とのインタビューを見つけることができます。 あなたが興味を持っているすべての質問はコメントで尋ねることができます、 アレクセイ・サゾノフはそれらに答えようとします。

インタビュー



あなたの会社はどのように設立されましたか？



HeroCraftのCEO兼共同設立者であるPavel Prokonichの回答：



2002年に起こりました。 最小限の資金で（主にコンピューター用に）5人の学生が自分のビジネスを組織することにしました。 要するに、初期資本金20,000ドルのガレージスタートアップのまれな成功例でした。お金は、Andrei Petrov社の創立者が稼いだもので、適切なタイミングでハンドヘルド向けゲームをリリースしました。 オフィスは兵舎に賃貸され、アパートに変換され、怒ったスタッフォードシャーテリアがキッチンに住んでいました。 現在、アンドレイは新しいスタートアップに取り組んでおり、多額の資金を提供していますが、スタッフォードは引き続き彼のオフィスに住み、幸運をもたらします。



今、何を持っていますか？ 成果は何ですか、計画は何ですか？



最新の成果は、AndroidバージョンのFarm Frenzyです。これは、バルセロナで開催されたMobile World Congressで名誉ある賞にノミネートされました。 「モバイルワールドコングレス」は、毎年開催されるモバイルデバイスのトピックに関する最大の会議および展示会です。



このゲームは、トップAndroidマーケット（ リンク ）およびNokia OVIにもあります。



別の成果： 2009年第1四半期の品質チャートのモバイルゲームパブリッシャー＃1-評判の良いPocketGamerリソースによると、総合レビューインデックスで1位になりました。



会社自体の成長については、過去4年間で約40人から80人に成長し、ゲームポートフォリオは30から100に増加し、約20が開発中です。その時点で、ほとんどのゲームはJ2MEでした。 Symbian、Windows Mobile、古いPalm、少しBREW。 もちろん、iPhone用のゲームが追加されました。 私たちはまだJ2MEを放棄していません（多くの企業がこの方向で作業を大幅に削減したか、このプラットフォームでのアプリケーションのサポートを完全に停止しました）が、Androidに強く切り替えました-現在、このプラットフォームで15のゲームがあります。



新しい-バダのための3つのほぼ既成のプロジェクト（そのうちの2つはサムスンの直接参加によるものです-部分的な資金調達プログラムに参加しました）、そして私たちは270万ドルの賞金プールでバダチャレンジで手を試したいので、さらにいくつかのゲームをします:)



その時点で、国際的な販売を確立することが重要です-さまざまな時期に、最大数百のアクティブなアフィリエイトアカウントがありました：コンテンツプロバイダー、ポータルなど。 今では、キノコのように成長するアップショップに重点が移っています。



さらに重要なのは、著作権所有者とライセンスIP（ブランド）との作業を確立したことです。当時は、それなしでは不可能でしたが、今ではそれも重要です。 「Crimsonland」、「Postal：Babes」、「Majesty」、「KranX」および「Alawar」を含む最初のプロジェクトをリリースします。ここでも、AndroidマーケットプレイスとNokia OVI TOPが破損した「ファームフレンジー」に特に注目します。 一般に、当社の製品はNokia OVIのTop Appによって公式に認められています。 そして、これは1つのポートしか販売されていなかったという事実（N97 / 5800グループ）にもかかわらず、他のデバイスが最近追加されました。



彼らはまた、最大のロシア語ライセンスでゲームを作成しました-ここでは、「奇跡のフィールド」と「パパの娘」に注意することができます（PCゲームも作成し、対応するサイトの1位を占めました）。



前述のイベント： badaプラットフォームでの開発者の競争については、すでに書いています 。



私の知る限り、あなたのチームはさまざまな都市に分散しています。 そうですか？ コラボレーションはどのように機能しますか？



サンクトペテルブルク、ドネツク、クラスノダールにユニットがありますが、私たちのチームの大部分はまだカリーニングラードにあり、約40人です。 しかし、たとえば、プロジェクト全体がカリーニングラードにあり、アーティスト（アウトソーシング業者ではなく）がサンクトペテルブルクにいることがあります。



または、たとえば、それは一般的なことです-小さなリモートチームプログラム、描画、およびその後、ローカライズおよびその他の機能を備えた多数のデバイスへの移植がカリーニングラードで行われます。 チームが遠隔地にあり、ある種の相互作用が必要な場合（たとえば、ゲームデザイナーがここにいて、残りがそこにいる場合）、絶え間ない連絡、プロジェクトマネージャーの積極的な作業などがない場合、オーバーレイが発生する可能性があります。 これらはすべて作業の瞬間であり、問​​題が発生したときにそれらを解決しようとします。



たとえば、多くの企業は、テストを外部委託したり、移植とともにテストしたりします。 実際、製品の移植やテストに関して開発者からオファーを受け取らないように、1か月も経っていません。



しかし、パートナーからのバグ報告に迅速に対応できる場合、高品質で成功する作業の鍵は、依然としてあなた自身のポーティング/テスト部門です。

まあ、それはあまりにも大胆な声明でしょう。 はい、私たちはこの市場の主要なプレーヤーですが、それ以上ではありません。



Yuri Keshishev、ポーティング責任者：



私たちのデータベースには、ほぼ3000台の電話に関する情報が含まれています。その中には、サポートしていない電話もありますが、それでも、その存在は知っています。 アクティブなサポートでは、1500台弱の電話が75のグループにグループ化されています。 私たちは常に新しいモデルの出現と販売からのモデルの離脱を監視し、新しいグループの出現と廃止されたグループの削除につながります。



グループを構成する主要なデバイスを購入して、各グループで少なくとも1台の電話が利用できるようにします。 ほとんどの場合、私たちの街で購入しますが、海外で電話を注文する必要がある場合もあります。 その結果、約150台の電話が利用可能になり、最も古い電話は若い世代によってバラバラになりました。 アクティブに使用する場合、約70台の電話があります。



さらに、DeviceAnywhere電話への有料のリモートアクセスのサービスと、Nokiaからの電話への無料のリモートアクセスのサービスを使用します。



プラットフォームはどうですか？



現在、主要なプラットフォームはJ2ME（Blackberryを含む）とAndroidであり、これらからJavaに自然に取って代わることが期待されています。 多くのJ2MEパートナーがAndroidを1つずつサポートしているため、これが現在目撃しているものです。 おそらく、Googleのプラットフォームは、従来のコンテンツプロバイダー、オペレーターポータルなどの死にゆくビジネスを救うでしょう。



さらに、最高のゲームをiPhone（およびiPadで1つのゲームも発表）、bada、Symbian、およびWindows Mobileに移植します。 最後の2つは現在ポイントを失っていますが、市場の拡大により、かなり自然なプロセスです。

いいえ、AndroidはJ2MEと同じ人々です。



Andoridの現在の状況は？



これはかなり膨大な質問です。 現時点では、誰もが第一に、Android Marketplaceで光が合わなかったことを理解しているわけではありません。Androidゲームを実装するためのチャンネルがたくさんありました。 Appleのアプリとは異なり、販売は一切行われません。 Androidゲームは既にJ2MEと同じチャネルで販売されています（つまり、Andoridは徐々にJ2MEに取って代わります）。つまり、独立した開発者がこのプラットフォーム用にゲームを適切に販売できるように、パブリッシャーまたは仲介者なしではできません。 もちろん、あなたはまだ素晴らしいゲームを作り、あらゆる角度でそれを宣伝することができます-そしてTOP10に入る-そしてまた大きなお金がありますが、これは大きな出版社が完全にトップを占めるまでです。 そして、彼ら全員がこのプラットフォームの開発数の増加をすでに発表しています。

また、世界中のゲームやデモのプリロード、ミーティング、セミナーなどの機会は勇気づけられます。これは大歓迎です。会社は本当に開発者を引き付け、お金を稼ぐ機会を与えようとしています。



プラットフォームごとの開発ツールについては、詳細はありますか？ SDKとはどのようなものですか？



回答Yuri Keshishev、ポーティング部長：



同社は今日の成功の大部分を、開発の最初に採用されたツールの選択と技術開発の方向性に関する正しい決定に負うところが大きい。 その結果、同社には現在、公開されているアプリケーションと独自のプログラムの両方を単一のソフトウェアパッケージに統合したツールセットがあります。



ゲームは、Eclipse IDE、ProGuard難読化ツール、および電話メーカーが提供するSDKを使用して開発および移植されます。 Eclipse用の追加のプラグインが開発されており、ゲームおよびアプリケーションの開発と移植が高速化および促進されています。 グラフィックはAdobe Photoshopで作成され、独自のプログラムであるHeroToolは、アニメーション、マップ、およびフォントセットの作成に使用されます。



ソースとリソースのバージョンを管理するために、SVNが使用されます。SVNは最近、MercurialとGitに徐々に置き換えられ始めています。 プロジェクト、パートナー、電話グループに関する情報を設定および保存するために、MS Accessに基づくデータベースであるHeroBaseが使用されます。これにより、プロジェクトに関与するチーム全体がプロジェクトで同時に作業できます：プログラマー、テスター、マネージャー。



Microsoft Word / ExcelおよびOpenOffice Writer / Calcとの統合により、外部の開発チームや顧客と連携するためのさまざまなレポートドキュメントを作成できます。 データベースには、ライフサイクル全体を通してゲームに付随するツールが含まれています-ローカライズを追加するためのゲームテキストのエクスポート/インポート、新しいポートの追加、顧客の要件に応じたカスタマイズ、テスト結果など。



私たちの特別な誇りの主題はJMBuilderプログラムです。これにより、特定のコンテンツプロバイダーの要件に従って構成された、移植されたゲーム（特定のローカリゼーションのゲーム）から商用製品を作成できます。 プログラムには独自のスクリプト言語があり、ループと条件ステートメントを備えたほぼ完全なプログラムを作成でき、使いやすいユーザーインターフェイスと、データベースやスプレッドシートと対話する機能を備えています。 HeroBaseデータベースには、プロジェクト、コンテンツプロバイダー、電話グループを操作する形式でJMBuilderを直接呼び出す機能があり、JMBuilderはHeroBaseデータベースからデータを読み取ってゲームを収集します。



同社はツールの開発に多大な注意を払っているため、リストされているすべてのプログラムとプラグインは引き続き開発され、新しい機能が追加され、新しいプラットフォームがサポートされています。



HeroCraft-UkraineのディレクターAlexander Similetovの回答：



スマートのために、Symbian、Windows Mobile、iPhone、Bada、モバイルLinux（以前はPalm OSがサポートされていました）、Windows XP / Vista / 7で動作する独自のSDKを開発しました。 MicrosoftのXNAには、C ++とC＃の両方のバージョンがあります。



C ++で記述できるすべてのものに転送します。 SDKは、半自動モードでのJavaからC ++へのゲームの転送を大幅に簡素化するように作成されています。 チップのうち、3Dエフェクト付きのサウンド、互換性のあるソフトウェアモードとアクセラレータの両方でのグラフィックス出力。 独自の3Dエンジン、2D、GUIエンジン。
