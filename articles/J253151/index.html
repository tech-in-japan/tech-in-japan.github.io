GroovyプロジェクトはApache Software Foundationに参加する予定です

これは、Groovyの公式プロジェクトマネージャーであるGuillaume Laforgeのブログからのこの出版物の翻訳です。



Groovyチームは、 Apache Software Foundation（ASF）に参加する意向を発表します。



Groovyの通常の開発者への資金提供の終了に関するPivotalからの最近の発表を受けて、チームはプロジェクトが長期的に消滅しないことをGroovyコミュニティに示すことが賢明だと判断しました。 Groovyは、民間組織やスポンサーによる資金調達の有無に関係なく、また開発チームの変更に関係なく開発されます。



さまざまなソフトウェア基金の代表者、特にEclipse Foundation 、 Software Freedom Conservancy 、 ASFと数回（オンラインとオフラインの両方で）話し合いました。



ユーザー、これらの資金の開発者、さまざまなフォーラムの参加者などから寄せられたアドバイスに感謝します。 これらの議論は非常に有益で有益でした。 もちろん、これらの資金はすべて、プロジェクトにとって非常に興味深く、素晴らしいものですが、もちろん、すべてに長所と短所があります。 しかし一般的に、Apache Software Foundationは、私たちの限界と哲学を考えると、まさに私たちが最も適していると考える基盤です。



インキュベーターでプロジェクトを受け入れるための申請プロセスをすぐに開始します（ プロジェクトはASF ソフトウェア開発およびライセンススキームと統合されます。 申請が受理された場合、プロジェクトのインキュベーションの過程で、議論段階で特定された「灰色の領域」に遭遇します。 しかし、GroovyがJVMエコシステムで最大のオープンソースソフトウェアプロジェクトの1つであることを保証するために、メンターと協力して非常に満足しています。
