組織内のLinux ...

多くの場合、Linuxを中小企業にインストールする可能性について議論するとき、すべてはその下で1Cプログラムを実行する問題などに依存します。 ただし、無料ではありませんが、この問題を回避するためのシンプルでエレガントな方法、ターミナルアクセスがあります。 Linuxは、ターミナルアクセスの可能性があるすべてのワークステーションにインストールされます。 1台のコンピューター-Windowsターミナルサーバー。 1Cが必要です-ターミナルに登って作業します。また、ネットワーク経由でDBFを追跡せずに、迅速に作業します。 他のすべて、メール、ドキュメント、インターネット-はLinuxですでに無料です...

もちろん、Microsoft Windows Terminal Serverのコストは非常に高く、Linuxを実装する利点をすべてカバーできますが、オプションがあります...



したがって、Windowsから逃れることはまだできないため、ターミナルアクセスサポートを備えたシステムのコストを最小限に抑えるためだけに留まります。 仕事には、ライセンスを取得したWindows XP Proが必要です。そのために、Windowsのみを必要とするすべてのタスクを設定します。 次に、2つのオプションがあります。

  1. 半合法：Windowsはこのアルゴリズムをハッキングします 。 その後、多くのターミナルで同時に登ることができます。 検査機関がこのハッキングを検出できるかどうか、そしてそれが与えられるものは難しい質問です...しかし、少なくとも、会社のソフトウェアのほとんどは絶対に合法です * ...
  2. 法的な方法： XP Unlimitedは145ユーロで、XP Proを10の競合クライアント用のRDPサーバーに変えます。 455ユーロ-顧客の制限なし。 とにかく、Microsoftの本格的なサーバーよりも安価です。 ちなみに、非商用目的で1台のXP Proコンピューターを3台のクライアント（たとえば、1台はローカル、2台はリモート）で共有できるデモ版があります。 たとえば、古いコンピューターを新しいリソースに接続するために、自宅にインストールできます。




* 完全に合法 -最近、新しい法律に関連して、ロシアのGPLの合法性が問題にされていますが、とにかくこの不条理が修正されることを願っています...
