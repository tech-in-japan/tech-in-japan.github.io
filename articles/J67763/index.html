ScalaのユーザーによるJavaの本の以前のレビューは、過去3年間に市場に参入した「実用的な」本について語っています。 これらの本や他の本について、少し異なる角度からお話したいと思います。本の情報の価値を強調してみてください。

UPD： sseに よるM. Fowlerによる書籍「Enterprise Application Architecture」の説明





本に関する私の意見。

本を区別する最初のことは、著者のプレゼンテーションのスタイルです。 すべてが非常に明確で、楽しい生徒が作成したあらすじに似ています-要点が記録され、瞬間が強調表示され、必要なイラストがあります。 この本は、アプリケーションをレイヤーに分割する、ビジネスロジックを整理する、データベースを操作するための考えられるシナリオ、WebアプリケーションでUIを整理するなど、さまざまな問題を網羅しています（デスクトップアプリケーションのUIは扱っていませんが、著者はその後、この情報を自分のWebサイトに追加しました）。 重要なのは、Javaには現代の開発者向けの共通語があるという例があります。

「しかし、このテーブルをクラスでラップし、それを継承します-なんてこった、私は新しい方法を思いつきました」という考えが好転し始めたときはいつでも、ファウラーの本を読んで読む価値があります-それはおそらく文書化され、述べられ、改善された方法が記述され、説明されています落とし穴とそれらを回避する方法が与えられます。

本は2つの部分に分かれており、これはGoFの本「Design Patterns」の構造に非常に似ています。 最初の部分には、システムの構築に関する一般的な規定と、本書の知識を実際の用途に応用した例が含まれています。 2番目の部分は、アプリケーション層ごとにグループ化されたパターンのディレクトリで、各層に対応するのが一般的です。

結論として、多くの基本パターンは、データ処理およびストレージの分野から最も一般的で普遍的なものとして提示され、ビジネスオブジェクトを構築し、インターフェース部分のヒントを提供します。

この本は安くはありませんが、それに含まれる情報はそれ自体に報います。 Fowler martinfowler.com Webサイトの電子版で入手できます。「選択」して、必要かどうかを確認できます。 私はすぐに必要なものに気づきました:)



UPD2：英語のユーザーカルバートエディションの説明



私の意見では、これまでにJavaについて書かれた最高の本のリストです。 書籍は重要度順に並べられています（最も基本的なものが最初にリストされています）。



0. James Goslingらによる J ava Language Specification第3版は、最も重要な本であり、このリストの本番号0、JLSとも呼ばれます。 第3版では、言語の5番目のバージョンについて説明します。 すべてのJavaプログラマーがJLSを少なくとも1回は読むべきだと思います。 自分で使用したり、他の人が書いたコードを理解したりするには、言語が提供するすべての可能性を知る必要があります。 JLSは、使用するライブラリのJavaDocまたはその他のドキュメントの類似物です。 ただし、通常プログラミング環境からドキュメントが利用できるライブラリクラスとは異なり、この言語はプログラミング環境でコンテキストドキュメントを提供しません。したがって、JLSは少なくとも目次と各項目の一般的なコンテンツで実質的に「最初から最後まで」読む必要があります。 JLSは、Sun Certified Java Programmer（SCJP）試験の準備に必要な資料です。



1. Joshua BlochによるEffective Java 2nd Editionは、Java言語を使用して、他の人にとってわかりやすく理解しやすいプログラムを作成する方法を説明しています。 これは、すべてのJavaプログラマーのレパートリー内にあるべき実用的なヒントとパターンのコレクションです。 彼のベストセラー本の第2版では、BlochがJavaの第5バージョンのすべての機能の効果的な使用について語っています。 長年の経験を持つプログラマーでさえ、新しい何かを発見し、Javaプログラミングに関するすべての基本的な知識の体系化を確実に楽しむでしょう。



2.マルチスレッドJavaアプリケーションを作成するすべての人には、Brian Goetzらによる実践的なJava同時実行性 （適切なマルチスレッドコードを記述したい場合に読む必要があります）が必要です。 単純なアプリケーションを作成する場合は読む必要はありませんが、ビジネスロジックには、プログラマをマルチスレッドから分離するJ2EEまたはその他のアプリケーションフレームワークを使用してください。 ただし、アプリケーションのパフォーマンスまたは反応速度を最適化する必要があるとすぐに、現代の世界ではマルチスレッドから逃れることはできません。 この場合、Goetzの本が最高のアシスタントになります。



3. Java Puzzlers：トラップ、落とし穴、コーナーケースJoshua BlochによるNeal Gafterは、Java言語専用の知的パズルで構成された興味深い本です。 Java言語のすべての境界ケースを明らかにし、コードを理解できるようにプログラミングしない方法を教えます。 この本は読む必要はありませんが、SCJPの準備とあなた自身の喜びのために強くお勧めします。
