兄の物語：Toshiba Portege SS S4 / 275PNHW上のLinux OS

赤ちゃんLibrettoとの3日間の闘争の後、私は右足で立ち上がって仕事に行きました。 地下鉄の車の中に立って、私は自分の失敗について考えていました...そして、それは私に夜明けしました。

職場に着いて、私は自分の考えを経営陣に報告しました。 昼食後、東芝製のPortege SS S4 / 275PNHWを手に入れましたが、何が私を待っているのかわかりませんでした！







説明



半日、私は奇跡を待っていました。2002年のほぼ通常のラップトップにLinuxをインストールしました。 それでは、どんな種類のデバイスか見てみましょう。

このラップトップはPortege 2001シリーズに属し、DynaBookと呼ばれます。 インターネット上のこのユニットの説明はほとんどありません。

このラップトップは軽量で非常に快適な人間工学です。

OSのインストール



ラップトップと一緒にUSBフロッピー（Noname）、USB CD-ROM MiniCD Archosを入手しました。

ディスクを挿入してマシンを起動した後、CD-ROMからのロードが機能しないことに気付きました...しかし、まだフロッピーで試したインストール方法を作成した後、フロッピーが見つからないというおなじみのメッセージを見ました。

しかし、ネットワークブート（PXE）もあります！ これが現時点での唯一の方法です。

それでは、PXEのインストールを始めましょう。

まず、ラップトップをネットワークに接続する必要があります



次に、Linuxマシンで、DHCPサーバーとTFTPサーバーを構成する必要があります。

サーバー構成



Ubuntuでは、最初にインストールする必要があります



apt-get tftpd-hpaのインストール

apt-get install dhcp3-server

mkdir -p / var / www / ubuntu-606-server-i386 /

mount -o loop /dl/ubuntu-6.06-server-i386.iso / var / www / ubuntu-606-server-i386 /

tar xzvf /dl/netboot.tar.gz -C / var / lib / tftpboot





次に、DHCPサーバーを構成する必要があります。

/etc/dhcp3/dhcpd.conf

ddns-update-style none;

オプションドメイン名 "hifichoice.com";

オプションドメイン名サーバー83.146.21.6、212.158.248.5;

ping-check = 1;

デフォルトのリース時間600。

最大リース時間7200;

log-facility local7;

サブネット192.168.95.0ネットマスク255.255.255.0 {

範囲192.168.95.100 192.168.95.150;

オプションルーター192.168.95.1;

filename = "pxelinux.0";

next-server 192.168.95.20;

}



そして、tftp-hpa

/ etc / default / tftpd-hpa

RUN_DAEMON = "yes"

OPTIONS = "-l -s / var / lib / tftpboot"





netinstall.tar.gzをUbuntuアーカイブから/ var / lib / tftpbootフォルダーに解凍します

それだけです！ サーバーが構成されたので、クライアントから接続し、ラップトップのネットワーク経由でダウンロードを設定して開始できます！ インストールのダウンロードを入手してください！ それから私はすべてが明確だと思います;）
