アセンブリの移植性（移植性）はどのように作成されますか？
私は「額」と呼ばれるものの物理的経路からプログラムを「解き明かしました」が、すべてが非常にシンプルで明確です。

アセンブリのルートフォルダーを「www」というフォルダーにしました（つまり、周囲のすべてがその中に存在します）が、既にどこにでも配置できます。

構成ごとに、フルパスが必要な場所では、パスは「？/ Www / path-where-need-relative-to-folder-www」として示されます。

コピー後、新しいパスにコピーすると、特別なファイルが起動します。これは大まかに言って次のことを行います。すべての「初期」構成を取得し、「？」サービスを登録します。



ベースアセンブリの内部には何がありますか？
Apache-起動時にApacheSAサービスを登録します。

MySQL-起動時にMySQLSAサービスを登録します。

PgSQL-起動時にPgSQLSAサービスを登録します。

fast-cgiとして接続されたPHP 5。

mod-wsgiとして接続されたPython 3（または2）



コンポーネントが停止すると、サービスも停止して登録解除されます。



どのバージョンのWindowsで動作しますか？
Windows XP（32/64）、Windows 7（32/64）は正確に機能します。

理論的には、Windows 8で動作するはずですが、残念ながらチェックする方法がありませんでした。



組立構造
次のフォルダーがルートにあります。

  • Apache - Apache Webサーバーファイル
  • MySQL -MySQLファイル
  • NodeJS -NodeJSファイル
  • PgSQL -Postgre SQLファイル
  • PHP -PHPファイル
  • Python -Pythonファイル
  • sys-アセンブリが機能するために必要なファイル
  • tmp-一時ファイルのフォルダー（PHPでのセッション、アップロードされたファイルなど）
  • ツール - ツールのあるフォルダー
  • webroot-サイトのルートフォルダー


そしてファイル：

  • _runner ___。cmd-メインビルドメニューのファイル（Apacheの再起動、再構成、ログなど-すべてはここにあります）
  • _setconf __。cmd-アセンブリ再構成ファイル（ランナーから呼び出すことも、個別に実行することもできます）
  • _showlogs_.cmd -Apacheログを操作するためのファイル-クリーニング、表示（ランナーからまたは手動で起動することもできます）
  • clean.cmd-アセンブリをクリーニングするファイル（たとえば、USBフラッシュドライブに転送する-一時ファイル、ログなどを削除する）
  • phpinfo.cmd -PHP情報を生成し、調査のためにブラウザーで開くためのファイル
  • readme.txt-アセンブリ記述ファイル
  • upgradeSites.txt-アセンブリコンポーネントを更新するためのリンクを含むファイル


Sysフォルダー
内部には、これらすべてのサービスの構成テンプレートを保存するapacheData、mysqlData、pgsqlData、phpData、pythonDataのサブフォルダーがあります。

これらは、アセンブリを新しいフォルダーにバインドするときに、パスが変更された適切な場所にコピーされる同じ構成テンプレートです。



さらに、データベースのマネージャー （ mysqlData / managerおよびpgsqlData / manager ）もそこに保存されます。

マネージャーはアセンブリに含まれていません-誰もが彼が慣れているものを使用できます。

私はEMS MySQLとPgSQLマネージャーを使用します-機能性と利便性の点で最も気に入っています。

最も重要な条件は、これらのフォルダー内のマネージャーの実行可能ファイルはmanager.exeと呼ばれることです。



sysフォルダーのルートには、 services.cmdコマンドラインからアセンブリを操作するための別のファイルがあります。

たとえば、 services.cmd RestartApacheのようにApacheを再起動できます 。

これが人生で必要になることはまずありません-すべてはメニューから行うことができますが、誰かが自分の何かを固定したい場合は、機会があります。



Tmpフォルダー
アセンブリ全体の一時ディレクトリ。

一時ディレクトリを指定する必要があるすべての構成では、ここに示されています。

その内部では、PHPセッションが存在するセッションディレクトリを除くすべてを削除できます。



ツールフォルダー
さまざまなツールのあるフォルダー。

Apache、MySQL、NodeJS、PgSQL、PHP、Pythonのサブフォルダーには、これらのサービスのさまざまなドックとトリックに加えて、追加機能を実行できるサービスファイルが含まれています。

たとえば、データベースの場合、データベースのダンプのインポートとエクスポート、データベースの初期化などのためのバッチファイルがあります。



さらに、このフォルダーにはGIT（同じ名前のフォルダー）、YUIミニマイザー（Minimizerフォルダー）、Image Optimizers（IMGoptimizeフォルダー）、cURL、Inkscapeなどがあります。



常に使用するツールがある場合は、ここに配置できます。



Webrootフォルダー
このフォルダーには、ApacheまたはNodeJS構成でパスが指定されているサイトのあるライブフォルダーです。



ここでの唯一の特別なサブフォルダーは、 Pythonの mod_wsgiのホストハンドラーが存在するPythonサブフォルダーです。また、Apache構成にも登録されており、いつでも好きなものに置き換えることができます。



ビルド更新
更新により、すべてが簡単になります。サイトから目的のコンポーネントをダウンロードし、置換ファイルを使用して目的のフォルダーに解凍します。



ただし、もちろん、いくつかのニュアンスがあります。

  • アセンブリ32ビット-それに応じてファイルをダウンロードする必要があります（ただし、64ビット全体を作成できます-混ぜないでください）
  • PHPはスレッドセーフとしてダウンロードする必要があります
  • Pythonを更新する場合は、適切なバージョンのmod_wsgiをダウンロードし、 sys \ pythonDataに入れて、Apache configに登録する必要があります。


製造元はmsiパッケージとしていくつかのコンポーネントを提供し、 msiexec /コマンド "MSI package name" TARGETDIR = "％CD％\ tmp" / qbを使用してwinでアンパックできます。目的のフォルダーにコピーできます（tmpフォルダーで、注意深いMSIインストーラーがそこに置くパッケージファイルのコピーを削除することを忘れないでください）。



ビルド機能
  • すべてのルートデータベースのユーザー名とパスワード
  • アセンブリ全体は32ビットです
  • タイムゾーンはGMT + 6
  • Pythonには、2.xと3.xの2つのバージョンがあります
  • アセンブリのコンポーネントは127.0.0.1と目的のポートに掛けられており、それに応じてそこから利用できます。
  • 127.0.0.1はwebrootを指します/
  • 127.0.0.2はwebroot / site1を指します
  • 127.0.0.3はwebroot / site2を指します
  • webroot / Pythonのmod_wsgi経由の127.0.0.5
  • Apache configには、＃CL＃という形式の2つのラベルがあり、それらの直後にDocumentRootとRewriteEngine Onがあります。これらは、ランナーからこれらのパラメーターを見つけて変更するために必要です。




アセンブリの開始
最初に行うことは、構成テンプレートに移動し（sysフォルダーに関する情報を参照）、それらを「自分用に」調整します。 webroot内のサイトで必要なフォルダーを指定し、必要なPythonのバージョンを見つけます。



アセンブリをどこかにコピーする操作中、または再構成中に、アセンブリのすべてのコンポーネントを停止する必要があります 。 Apache、MySQL、PgSQLなどを停止する必要があります。

これを実行するには、ランナーから[ すべて停止]を選択します。



次にランナーを実行し、そこでSet configurationを選択するか、 すぐにsetconfを実行してReset to defaultsを選択し、現在のパスをメインパスとして設定します。この後、アセンブリはこのフォルダーで動作するように再構成されます。



覚えておくべき主なことは、この操作の後、現在のすべての構成がテンプレート構成に置き換えられることです！ そして、それらを変更したが、テンプレートを変更しなかった場合、これらの変更は失われます。



再構成後、ランナーから必要なサービスを開始します。これで作業は完了です。 たとえば、127.0.0.1のブラウザに移動し、（設定でこの設定が変更されていない場合）、サイトはwebrootのルートから開く必要があります。



アセンブリを初めて起動した場合-少なくとも1つのデータベースを作成して、MySQL / PgSQLデータベースを初期化することを忘れないでください。
