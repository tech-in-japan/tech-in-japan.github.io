3CX PBX Express：Google Cloudの1年間無料PBX

今日は、15分間でGoogleクラウドで8つの同時通話用の無料の3CXクラウドベースPBXを作成する方法について説明します。 ただし、以前にクラウドでの配置が支払われていた場合、Googleインフラストラクチャを12か月間無料で使用できるようになりました！



必要なのは、Googleアカウントと支払いカードだけです。 カードは、あなたがロボットではないことを確認するためにのみ必要であることにすぐに注意します。 12か月を完了すると、決定するまで資金は引き落とされません。



Google Cloud Platformプロジェクトの作成



Googleの資格情報を使用してGoogle Cloud Platformコンソールにログインし、利用規約に同意します。 次に、個人情報と支払いカードの詳細を入力します。







その後、試用期間が開始されます。







次に、新しいプロジェクトを作成するか、既存のプロジェクトを使用するように求められます。 その名前に注意してください。







プロジェクトを作成したら、Compute Engine-VM Instancesセクションに移動します。 異なる容量の仮想マシンがここに作成されます。 インスタンスを作成しないで、提供されるオプションを見てください。 特に、仮想マシンのパワーとコストに注意してください。 8コールの小型PBXには、「小型マシン」が非常に適しています。 そのコストは配置に依存します。







これで、Google Cloud Consoleの準備が完了しました。 PBX Expressサービスで引き続きPBXを作成します。



PBX ExpressサービスでのPBXアクティベーション



3CX PBX Expressサービスインターフェイスに移動し、作成する3CXサーバーの名前と、それが配置されるドメインゾーンを指定します。







PBX管理者の資格情報を入力します すべてのシステム通知は管理者の電子メールに送信されます。







内部（内線）ユーザー番号の長さを選択します。 注意してください-システムを再インストールせずに将来変更することはできません！ 中小企業の場合、2〜3桁以下の長さをお勧めします。







PBXシステムのボイスメッセージの場所、タイムゾーン、および言語を示します。 国とタイムゾーンに関する情報は、特に、追加のシステムセキュリティとハードウェアIP電話の自動構成に使用されます。







次に、PBXユーザーを追加します。 200人のユーザーを1人作成しましたが、組織内のすべてのユーザーを指定できます。







次のステップで、自分のGoogleアカウントを使用していることを示します。







Google Cloud Platformプロジェクトが作成されたアカウントを選択する承認ウィンドウが表示されます。







その後、ウィザードのインターフェースには、以前に作成されたGoogle Cloud Platformプロジェクトの選択、PBXインスタンスの場所と容量が表示されます。 私は小さな仮想マシンを選択しました。これは、8つの同時呼び出しの負荷に十分対応できます。







最後の1つで、ライセンスが生成される登録データを指定します。 3CXによって検証されるため、あなたと組織に関する実際の情報を提供します。 3CXの従業員がこの情報の信頼性を疑った場合、PBXは自動的に更新する機能を失います。







[完了]ボタンをクリックすると、指定した電子メールの確認コードが届きます。確認コードは、表示されるウィンドウに挿入する必要があります。 このコードは、作成された3CX PBX Express構成をGoogle Cloud Platformプロジェクトにリンクします。





電話交換は約10分で完全に準備が整います。 これは、ウィザードインターフェイスと、接続パラメーター付きの受信メールの両方に表示されます。 追加された各ユーザーは、Windows、Mac、Android、およびiPhone用の3CXソフトウェアクライアントの自動構成ファイル、およびWebクライアントへのアクセスURLを含む電子メールも受信します。







自動的に作成された仮想マシンインスタンスがGoogle Cloud Platformインターフェースに表示されます。







3CXパートナーの場合は、同様に顧客向けに無料のPBXの他のインスタンスを作成できます。 ただし、いくつかの制限があり、以下で説明します。



無料のシステム制限



結論として、もちろん、PBX Expressサービスで作成された無料のPBXの制限に言及する価値があります。



無料のPBXはPBXエディションと呼ばれ、 ここで指定されている機能セットが少し削除されています 。 1年間使用した後、 アップデートのサブスクリプションを更新するか、システムの商用バージョンにアップグレードするように求められます 。



無料のGoogle Cloud Platformアカウント（および仮想マシンインスタンス）は、静的IPアドレスを1つだけ持つことができます。 一方、このようなアカウントでは、最大8個のいわゆるアカウントが使用されます。 「エフェメラル」（動的）パブリックIPアドレス。 したがって、アカウントに3CX PBXの複数のインスタンスを配置する場合は、「一時的な」IPアドレスを使用してください。







また、 Googleの「常に無料」のオファーにも注意を払うことをお勧めします 。 特に、このプログラムは、小容量システムの無制限の無料使用を提供します。



  • 米国にある1つのf1-microインスタンス（バージニア北部を除く）

  • 30 GBのディスクと5 GBのスナップショット

  • すべての地域（中国およびオーストラリアを除く）への1か月あたり1 GBの発信トラフィック



この構成は、小規模オフィスのニーズに非常に適しています。
