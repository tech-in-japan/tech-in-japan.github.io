お互いを数える方法について



こんにちは。



すでに標準的なグレーディングシステムにうんざりしていませんか？ これらすべての± 、 5つ星 など ？ それらは隅々で使用され、その単純さにもかかわらず、多くの問題とあいまいさを抱えています。 最も重要で統一的な問題は、感情を評価できないことであり、これは時には非常に重要です。 結局のところ、トピックは面白いかもしれませんが、同時に、あなたはそれにプラスを入れます-リソースのポリシーによって導かれる方法、どうですか？ あなたの感情に？ ムーンフェイズに？ 明確な答えはなく、明確な解決策もありません。これによりあいまいさが生じます。



「 コンテンツを評価する方法を選択する 」という記事の助けを借りずに、自分のプロジェクトに実装してそれを補完した自明でないシステムについてお話します。



すでにそこにあるものについて



感情によってトピックを解析する必要があるため、これらの感情を最初に強調する必要があります。 感情か何か。 私のプロジェクトは創造的なオリエンテーションであるため、「オリジナリティ」、「芸術」、「テクニック」のカテゴリを使用し、それぞれをストーバルスケールに従って評価することが提案されました。



私は同意します、これは少し不快で混乱している、例えば82または87など、何を置くべきかという質問が常に生じました。 そのため、選択と視覚的なプレゼンテーションを容易にするために、最初に3つのカテゴリすべてで50に設定されたスライダーが作成されました。 そして純粋に心理的には、錯覚は常にランナーの左側、つまり50未満-マイナス、右-プラスを引き起こします（ 前の記事で説明したように、ユーザーに何らかのスキャンダルがあったことを具体的に尋ねました） 。



すべてのユーザーが各評価カテゴリの意味をすぐに理解するとは限らないという事実にも問題が生じました。 何が何であるかについての論争と議論があり、その後、すべてが多少なりとも正常化されました。 繰り返しますが、おそらくプロジェクトは創造的であり、すべての通常の理解が通常の方法でそれに作用するわけではありません。



やりたいこと



創造性、創造性、そして共通のテーマを持つリソースでは、評価カテゴリは異なるはずです。 たとえば、「面白くない」、「役に立たない」、「刺激的/つまらない」などです。 すでにこの3つで、投稿を操作して面白い写真を撮ることができます。 これで、面白い画像を+ 1 / -1 / + 1と評価し、大きなCSSトピックを-1 / + 1 / -1と評価できます（ただし、最後の推定値はストーリーのスタイルと最初のスタイルに依存する可能性が高くなります）。 一般的に悪名高い+ 1 / -1に限定されても価値はありません。 スライダーが停止する数値を表示せずに、各方向に50、または100ポイントのスライダーを実装し、カテゴリ別に並べ替えることがはるかに簡単で便利になります。



次にすること



そして、3つすべてのインジケーターを1つにまとめることは価値があります（私は3つと言いますが、他の整数を意味します！）。



球体を想像してください！ これには3つの座標があり、-100から+100までの評価の範囲があります。 次に、一連の笑顔を取り、それぞれの笑顔に座標を与えます。例：「:)」-+ 100 / + 50 / + 50; ";）"-+ 75 / + 85 / + 20; 「O_O」-0/0 / + 100など。 ここで、各座標に重みが1のポイントを配置する場合、重心の法則に従って、スマイリーの重みの中心は球の中心にあり、考えられるすべての音声と確率のバランスを取る必要があります。



しかし、一般的な公式を導き出す方法は言いません！
