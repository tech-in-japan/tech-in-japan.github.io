1週間でgma3600（intel cedar trail）とLinuxを友達にする方法





Asus x101chは父親への贈り物として購入されました。 一見したところ、特に価格と品質の点で、シックなネットブック。 ユーロセットでは、控えめな7,500ルーブルを与えました（冬でした）。 最も魅力的だったのは、マットディスプレイ、デュアルコア1.6Ghz Atom、およびフルHD（1080p）を視聴できるという主張でした。これは、古いeee pc 900の代替品としての選択に大きな影響を与えました。 6、および上記のすべてについて、私は最も重要なことを忘れました：購入前にLinuxでこのデバイスの正常性の問題を調査し、後で支払いました...







Linuxまたは「Ubuntuを試す」をインストールする最初の試み



ネットブックをパッケージから喜んで解放し、彼はもう一度、すでにおなじみの操作-Ubuntu 12.04 LTSを新しいデバイスにインストールします。 しかし、ライブセッションでXを読み込んだ後、解像度が低く、変更できない（800x600、ネットブックには1024x600がある）ことに失望しました。 すべてがソフトウェアで描かれていると感じられ、vesaドライバーがこのすべての背後にありました。 さらに、マウスの非常に奇妙な振る舞いが観察されました：それは消え、再び現れました-数秒後に非常に迷惑になり始めました。 ディスクにインストールしても状況は変わりませんでした。 すべては、問題がビデオドライバーにあることを示しています。



インターネットをぐるぐる回った後、およそ次のような画像が現れました。

  1. gma3600専用のテーマ 。 42ページ。 ロシア語フォーラムUbuntuで、 グラフィック機器のセクションで3番目に人気のあるもの
  2. 3D自体の欠如（すべてのOSで）。
  3. ハードウェアデコーダー（ビデオアクセラレーションAPI）にアクセスできる独自のドライバーの存在。
  4. gma3600 "sympathize"のあるネットブックのIntelユーザー。 しかし、彼は助けることができません（したくないですか？）。
  5. Cedar Trail用の Linux（MeeGo）の可用性 。 Win8について忘れることができるコメントがあります。
  6. さらに、Linuxのスリープには問題があります。 また、HDMI、VGA、および多数のFnキーも機能しません。
  7. グラフィックスチップ自体はImagination Technologiesの発案であり、 PowerVR SGX545です 。これは、gma3600でgma500の「論理的な」継続です。 後者は、Linuxsoidだけでなく非常に悪い評判を持っています。




当時、このセット全体によって、このようなサポートで鉄を生産するベンダーを購入およびスポンサーすることの実現可能性について既に考えさせられました。 しかし、何もする必要はなく、ネットブックを動作状態にする必要がありました。この状態では、プリインストールされたWin7でさえわいせつでした。 贈与者と授与者の贈り物とそれに伴う肯定的な感情は完全に影を落としていました。 ppaから専用ドライバーをインストールしようとすると、黒い画面が表示されました。 Ubuntu 12.10では、原則として、Xsはインストール後に読み込まれませんでした。 カーネル3.7.1をUbuntu LTSにバックポートすると、正常な画面解像度に加えて、ボーナスとして動作しないwifiドライバーが成功しました。 購入後3日目に、抵抗は無用であることに気づき、MeeGoをインストールすることにしました。



MeeGoとの2日間の戦い


Intel Webサイトからのリンクを使用して、MeeGo Cedar Trailイメージがダウンロードされ、そのままUSBフラッシュドライブに記録されました。 ネットブックは通常の解像度で起動し、MeeGoはデスクトップおよびアプリケーションメニューに遷移アニメーションをすばやく表示しました。 夢と一連のFnを獲得しました。 私は喜びました：全体的な印象は、プリインストールされたスターターの仕事からのものよりもはるかに優れていました。







しかし、幸福は短命でした。 文字通り、インストールの1時間後に、この小さな男は更新プログラムが利用可能になったことを喜んで発表しました。 結果はすぐに来ました... Wi-Fiは落ちました、そしてすぐに、それは非常に憂慮すべきでした。 フラッシュドライブから再起動し、カーネルイメージを比較すると、その更新が理由であることが示唆されました。 私は再配置し、更新をオフにし、「作品-触らない」という原則に導かれ、使用するために父にそれを渡しました。



父がシステムの奇妙な振る舞いについて話すまでに1時間も経っていませんでした.Wi-Fiの更新と同様にmc（真夜中の司令官）をインストールすると、Wi-Fiがブロックされますが、インストール自体はライブラリの更新はもちろん、パッケージに沿ってドラッグしませんでした。 配布キットの再インストールで十分に遊んだ後、私は自分では不可解で致命的で恐ろしいバグを見つけました-任意の（はい）パッケージのインストールは同じ問題を引き起こしました。 パッケージシステムは疑わしいものでしたが、MeeGoではrpmであるため、唯一の論理的な説明は、ディストリビューション開発者のグローバルジャムである可能性があります。 そのような問題を解決する欲求や強さはもはやありませんでした。 翌日は、問題を発見した場合に単一の手がかりをもたらさず、神経はより高価でした。

Xorgドライバーをインストールする



楽器と素材：Ubuntu 12.04.1 i386またはMint 13（最初の子孫の直接の子孫）、atom i386プロセッサとgma3600グラフィックスサブシステムを備えたデバイス。



進捗状況：

最初に必要なのは、ディストリビューション自体です。 最適なオプション：最新の安定版リリースをダウンロードします 。 サイト。



通常どおりインストールします。 ライブセッションとインストール後の両方で、解像度が低くなります。 これは「正常」です。



インストール後、パッケージのリストを更新します。 



sudo apt-get update




すべてのドライバーインストールガイドには、cedarviewカーネルモジュールがpaeカーネルと互換性がないと記載されています。 paeカーネルにドライバーをインストールしないでください。 あなたが持っているものを見つけてください： 



 uname -r




paeカーネルがある場合（これは、出力uname -rの接尾辞「-pae」で示されます）、通常のカーネルとそのヘッダーをインストールします（専用ドライバーのカーネルモジュールをビルドするために必要です）。 



 sudo apt-get install linux-headers-generic linux-image-generic




paeコアを破壊し、 完全に破壊します。そうしないと、黒い画面が表示されます。 



 sudo apt-get purge linux-headers-generic-pae linux-image-generic-pae




カーネルpaeの残りの設定は、apt-get removeで削除されず、1週間にわたって障害になりました。



カーネルをインストール/削除した後、grub2は他のOSと新しいカーネルのパーティションをスキャンする必要があります。 これが発生しない場合、次のコマンドを実行します。 



 sudo update-grub2




同時に、gdmをインストールします（一部のユーザーはlightgdmのxorg設定に問題があります）。 



 sudo apt-get install gdm




インストール中に、使用するディスプレイマネージャー-gdmを選択するという質問が表示されます。



再起動します。



ドライバーのインストール


事は小さいです-ドライバーをインストールするために残っています。 ほとんどの場合、再起動後にjockey（追加ドライバー）が起動し、cedarview-drmのインストールを提供しますが、正しい操作に必要なのはこれだけではありません。 



 sudo apt-get install cedarview-graphics-drivers cedarview-drm




カーネルモジュールがコンパイルされているため（見かけ上1つのスレッドで）、インストールは10〜15分ほど長く続きます。 インストール中に画面が空白になることがあり、画面が色付きのおになります。 主なことは、ネットブックを時期尚早にオフにしないことです。



ドライバーが正常にインストールされると、解像度が正常になり、スリープモードからの終了と輝度調整が正しく機能するはずです。



再起動後にXが起動しなかった場合は、tty1 Ctrl + Alt + F1を開き、ユーザーとしてログインし、grub設定を編集します（x101chではこの手順は不要です）。 



 nano /etc/default/grub




行を見つけます： 



 GRUB_CMDLINE_LINUX=""




引用符の間に追加します。 



 video=LVDS-1:d




grub2を更新して再起動します。 



 update-grub2 reboot




ブート中にeを押して「quiet splash」の後にオプションを追加することにより、カーネルブート構成を1つの実行に変更することもできます。



デザート用のハードウェアアクセラレーションビデオ



gma3600を搭載したネットブックまたはその他のデバイスを購入する際、「Full HD 1080p解像度での再生」という魅力的な広告に気付くでしょう。 すでにネットブックは720pを失う可能性がありますが、デコードはプログラムで実行され、プロセッサの負荷とバッテリーの寿命に悪影響を及ぼします。 たとえば、mplayerは4つのスレッドすべてを使用して、2つのコアそれぞれに最大80％をロードできます。 しかし、なぜですか？ また、グラフィックシステムに組み込まれた特別なマイクロプロセッサがあり、720pを3〜4％に再生するときの負荷を削減し、1080pではプロセッサを10％以内に保ちます。 有効にするには、 vaapiをハードウェアデコーダに実装する追加のライブラリをインストールする必要があります。



インストール： 



 sudo apt-get install libva-cedarview-vaapi-driver




また、mplayer-vaapiの特別なビルドが必要になります。 



 sudo apt-add-repository ppa:sander-vangrieken/vaapi sudo apt-get update sudo apt-get install mplayer-vaapi




さて、gnome-mplayerのグラフィカルシェル： 



 sudo apt-get install gnome-mplayer




最後のタッチ


Gnome Mplayerを開き、[編集]-> [設定]、[Mplayer]タブに移動し、[Mplayerの追加オプション]フィールドに追加します。 



 -vo vaapi -va vaapi




お気に入りの猫を含む1080pビデオがハードウェアアクセラレーションされました！



要約すると、LinuxとWindowsの両方のIntelのサポートは、描かれているほど良いものではないと自信を持って言えます。 検証されていないソースから、Intelは明らかにgma3600 / gma3650に特に穴が開いていることを認識しており、次世代のIntel Atomはi3とi5でグラフィックスを削除することを決定したことがわかっています。 それでも、gma3600の状況は5年前のgma500の状況よりもはるかに良好です。



追伸 mplayer-vaapiはmplayer2と競合し、vaapiで動作するいくつかのパッケージをインストールすることに注意することが重要です。 安定性は劣りますが、グラフィックスサブシステムのリソースを使用できます。



cedarviewソースを含むリポジトリも発見されました。 これは関連性があり、その最後のコミットは今年1月20日にさかのぼります。 コミットから判断すると、カーネル3.5.0および3.7.2のサポートが追加されたと判断できます。 コードベースは、Ubuntu 12.04に1.0.2のみが存在するIntelのcedarview 1.0.3に基づいています。 私自身はそれを収集しようとしませんでしたが、明らかな利点の中には、vaapiのメモリリークの修正があります。 私の場合、ドライバー1.0.2（リポジトリーから）を使用した2時間のFullHDスキャンでは、リークは検出されませんでした。

Ubuntu 13.04に同梱されているカーネルには、変更されたコミュニティドライバーが含まれています。 彼は2Dレンダリングの方がはるかに優れていることを示していますが、vaapiが不足していることに同意する必要があります。
