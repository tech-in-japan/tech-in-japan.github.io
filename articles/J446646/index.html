InterSystems IRIS 2019.1リリース

3月中旬に、InterSystems IRIS 2019データプラットフォームの新しいバージョンがリリースされました。







ロシア語の変更点のリストをご紹介します。 変更点の完全なリストと英語のアップグレードチェックリストは、 こちらにあります 。







InterSystems Cloud Managerの機能強化



InterSystems Cloud Managerは、InterSystems IRISインストールをクラウドに簡単に展開するためのユーティリティです。 2019.1リリースでは、次の機能がICMに登場しました。







  • アベイラビリティーゾーンのサポート。 同じリージョン内のいくつかのゾーンを占有するインストールの作成。 詳細-「 複数のゾーンに展開する 」。
  • 非同期ミラーのサポート。 インストール構成に非同期ミラーノードを含める。 詳細- ミラー化された構成要件 。
  • インストールパッケージから直接コンテナを使用せずにInterSystems IRISをインストールします。 詳細-「 コンテナレス展開 」。
  • サービス検出のサポート。 詳細-ICM展開の共有 。


クライアント言語



このリリースには、InterSystems IRISを操作するための新しいモジュールが含まれています。







  • INET Native API for .NET-.NETアプリケーションからグローバルへの低レベルアクセス。 詳細- ファーストルック：InterSystems IRIS Native API for .NET 。
  • Pythonからのリレーショナルアクセス-pyodbcの使用のサポート。 詳細-「 pyodbc Python ODBCブリッジのサポート 」。
  • 動的Javaゲートウェイ-プロキシクラスを作成せずにInterSystems IRISからJavaプログラムを呼び出します。 詳細- 動的オブジェクトゲートウェイの使用 。
  • Java Gatewayで作業するときに共有メモリを使用する。
  • Hibernate 5.2、5.3と互換性があります。 詳細-「 Hibernateサポート 」。
  • バルクデータロード用の新しいJavaユーティリティ。 詳細-「 シンプルデータ転送ユーティリティ 」。


スケーラビリティと分散クラスター管理の改善



InterSystems IRISの分散クラスターは、複数のサーバー間でデータとキャッシュを共有し、データのクエリと追加に柔軟で低コストのスケーラビリティを提供します。 このリリースには、次の改善が含まれています。







  • SQL用のより多くのスクリプトのサポート。 使用するデータベーススキーマとキーに関係なく、ノードをいつでもクラスターに追加できるようになりました。 ノードを追加した後、データのバランスを取り直すことができます（オフライン）。 詳細-「 追加のシャードデータサーバー間でシャードデータのリバランス 」。
  • 管理ポータルに概要とクラスター構成を含む新しいページが表示されます。
  • 一貫したクラスターバックアップを作成するための新しいAPI。 詳細-「 シャードクラスターの調整されたバックアップと復元 」。
  • 大量のデータをロードするための新しいJavaユーティリティも、クラスターでの作業用に最適化されています。


SQLの機能強化



このリリースには、SQLの速度と使いやすさが大幅に改善されています。







  • 一致するクエリを自動並列化します。 詳細-「 システム全体の並列クエリ処理 」。
  • SQLインターフェースを介してテーブルを調整するための新しいTUNE TABLEコマンド。 詳細-「 TUNE TABLE 」。
  • SQLシェルの改善。現在の領域で定義または使用可能なスキーマ、テーブル、およびビューを表示できるようになりました。 詳細-「 SQLシェルインターフェイスの使用 」。
  • クエリプランを表示すると、並列化およびクラスタークエリのマスタープランのサブプランが表示されるようになりました。
  • これで、クエリテキストにオプションを追加して、このクエリのSQLシステム設定を上書きできます。 詳細-「 コメントオプション 」。
  • InterSystemsには、各リリースのアプリケーションに対するさまざまなSQL非表示拡張機能が含まれています。 2019.1では、特にこのような多くの改善がクエリオプティマイザーとコードジェネレーターに追加されました。 ユーザーは、自動クエリ並列化とともに、InterSystems IRIS SQLを使用するアプリケーションの生産性を大幅に向上させる必要があります。


分析の改善



  • ビジネスインテリジェンスで部分的な日付を設定する機能。 たとえば、年または年と月のみがわかっている日付を指定します。 詳細- 部分的な日付 。
  • MDXクエリ内のSQLでデータをフィルタリングするためのNew％SQLRESTRICTコンストラクト。


統合の改善



このリリースには、製品のセットアップとトラブルシューティングを簡素化する多くの機能強化があります。







  • 製品内でメッセージが通過できるすべてのパスを検索して表示します。 詳細-「 インターフェイスマップの表示 」。
  • 製品コンポーネントが他の製品コンポーネントを参照する場所を検索します。 詳細-「 インターフェイス参照の検索 」。
  • データ変換のテスト。 テストダイアログで、初期化されたオブジェクトを使用して変換が呼び出されたかのように、aux、context、およびprocessオブジェクトの値を設定できるようになりました。 詳細については、変換テストページの使用を参照してください 。
  • DTLエディター。 新しいアクション- スイッチ/ケース 。 アクションをグループ化 し 、変換にコメントを追加する機能 。
  • これで、すべての製品に対してメッセージを実行せずに、ルールにメッセージを送信し、実行の結果を確認できます。 詳細-「 ルーティングルールのテスト 」。
  • メッセージビューアーからローカルコンピューターにメッセージをダウンロードする機能。 詳細-「 メッセージのエクスポート 」。
  • ログイベントをローカルコンピューターにダウンロードする機能。 詳細-「 イベントログページの概要 」。
  • ルールエディターで、ルールにコメントを追加し、編集したルールで使用される変換を開いて編集できるようになりました。
  • キュー待機アラート設定は、製品ラインのメッセージまたはアクティブなメッセージがアラートを生成するまでの時間を示すようになりました。 以前は、このタイムアウトは製品アイテムのキュー内のメッセージにのみ適用されていました。 詳細- キュー待機アラート 。
  • システムデフォルトへのアクセスを制限します。 管理者は、デフォルト設定を編集、表示、または削除するようにユーザーを構成できます。 詳細-「 システムのデフォルト設定のセキュリティ 」。
  • 製品をローカルコンピューターにエクスポートする機能。 詳細-「 プロダクションのエクスポート 」。
  • ローカルコンピューターから製品を展開することが可能です。 詳細-「 ターゲットシステムへのプロダクションの展開 」。
  • 製品設定ページの高度なナビゲーション。 対応する要素を別のウィンドウですばやく開くために、製品設定ページのブックマークにリンクが追加されました。 [キュー]タブで、メッセージ番号をクリックするとトレースが開きます。 [メッセージ]タブで、セッション番号をクリックすると、トレースが開きます。 [プロセス]タブで、メッセージ番号をクリックするとトレースが開き、プロセス番号でプロセスの詳細を示すウィンドウが開きます。
  • ビジネス製品項目の追加ウィザードの新しいオプション。 フィールドが空白のままの場合、ユーザーはシステムのデフォルトを自動的に割り当て、パケットプレフィックスを設定してルーティングルールを生成できるようになりました。 詳細-「 ウィザードオプション 」。


システムのパフォーマンスと機能



  • 特に不均一に分散されたメモリアクセス（NUMA）を備えた大規模システムの場合、スケーラビリティとパフォーマンスが大幅に向上します。 これらの改善には、統計を収集してグローバルバッファを管理するときのスケーラビリティの変更、インデックスレベルでのグローバル表示（サブスクリプトレベルマッピング）のパフォーマンスの改善、およびポインターブロックのバイパスを回避するその他の最適化が含まれます。 これらの改善を可能にするために、 このリリースのチェックリストに記載されているシステム統計とメモリ使用量に変更が加えられました。 これらの改善により、グローバルバッファメタデータに割り当てられるメモリのサイズが、Intelシステムではバッファあたり64バイト、IBM Powerでは128バイト増加します。 たとえば、8キロバイトブロックのバッファーの場合、Intelシステムの成長率は0.75％になります。 また、これらの改善により、ユーティリティおよび管理ポータルの統計情報の表示がわずかに変更されました。
  • キー管理相互運用プロトコル（KMIP）。 このリリース以降、InterSystems IRISは産業用キー管理サーバーのクライアントになります。 OASIS標準であるKMIPは、一元化されたキー管理の力を提供します。 KMIPサーバーのキーを使用して、データベースと個々の要素の両方を暗号化できます。 KMIPサーバーキーは、ログファイルを暗号化するなど、ファイルに保存されているキーと同じ方法で使用できます。 InterSystems IRISは、KMIPサーバーからローカルファイルへのキーのコピーをサポートして、ローカルバックアップを作成します。 続きを読む-「 キー管理相互運用プロトコル（KMIP）によるキーの管理 」
  • グローバルを表示するための設定を変更しながら、あるデータベースから別のデータベースにデータを転送するための新しいDataMoveユーティリティ。 詳細-InterSystems IRISでのDataMoveの使用 。
  • JSONオブジェクトでの3'641'144より長い文字列のサポート。
  • IRIS StudioとCachéおよびEnsembleの接続のサポート。
  • HTTP接続用のSPNEGOプロトコル（Microsoft統合Windows認証）のサポート。 現在、％Net.HttpRequestは、HTTP 1.1を介してWindows認証を使用して、安全なサーバーに接続できます。 ユーザーがアクセスの詳細を提供するか、％Net.HttpRequestが現在のコンテキストを使用しようとします。 サポートされる認証スキームは、ネゴシエート（KerberosとNTLM）、NTLM、および基本です。 詳細- 認証の提供 。
  • ロギングと非同期I / Oパフォーマンスの改善。
