カットの下でのウェビナーのデコードと記録。



今日は、リモートユーザー作業などについてお話します。 Windows Server 2012にはこのようなRDSテクノロジーがあります。今日詳しく説明し、Azure RemoteAppの機能についても検討します。



イベントの計画。 ライセンスについて、与えられたセッションの助けを借りて解決できるビジネスタスクについて、これが構築された基礎、それをすべて管理する方法、RemoteAppとは何かについて話します。 RDSがVDIとどのように異なるのかという質問に答え、これから説明するテクノロジーを実際に見ていきます。



したがって、Windows Server 2012とそのサーバーの役割RDS。 ライセンス-RDSは、標準とデータセンターの2つのエディションに含まれています。 私たちが知っているように、マイクロソフトは機能を完全に同一にし、仮想化での作業が異なります。



RDSには何が必要ですか？ まず、Server StandartまたはDatacenterライセンス。 プロセッサの数が考慮されることも忘れないでください。各ライセンスには2つの物理プロセッサがあります。 ターミナルセッションをターミナルサーバーに接続するためには、RDSライセンスだけでなく、ユーザー用のCallライセンスも必要です。 さらに、ここでは示していませんが、よく質問が寄せられます：ターミナルサーバーに1つのバージョンのオフィスがインストールされるような方法でターミナルサーバーのアーキテクチャを構築する人もいます。 実際、これはそうではなく、これは大きな間違いです。 はい、オフィスは1つのコピーでインストールされますが、ライセンス用に非常に多くのライセンスを購入する必要があるため、多くのユーザーまたはデバイスがサーバーに接続される予定です。 RDS-OSの古いバージョンの新世代のターミナルサーバー。この技術は「ターミナルサーバー」と呼ばれていましたが、現在はRDS-リモートデスクトップサービスと呼ばれています。







ビジネスタスク。 主なアイデアは、組織のすべてのユーザーがデスクトップまたはアプリケーションにアクセスできるプラットフォームを準備することです。 ほとんどの場合、このように見えます。 何度もこのサーバーを異なる組織にインストールする必要がありました-これは銀行や政府機関で最も人気があります。たとえば、アウトバックのどこかに支店を展開すると、ある種の支店が設立されるためです。 それから最大電力の問題はありません。 それは、人々がタイムリーかつ効率的にタスクを遂行することを保証することです。 そして、このためには、実践が示すように、シンクライアントとしてこのようなデバイスがあれば十分です。 このため、ユーザーはRDSに接続し、ターミナルセッションをセットアップし、サーバーにログインしてその能力で作業し、ビジネスタスクを実行します。 同じことは、大都市の外に支店がある州の機関にも当てはまります。また、大容量については話されていないため、この解決策は非常に一般的です。



RDSとは何ですか？ これにより、リソースを集中管理し、ユーザーにアプリケーションとデータを集中的かつ制御的に提供できます。 さらに、RDSテクノロジーは2つの可能な実装を意味します。 設定する例を見て、この役割がどこに置かれるかを示します。



タイプ1-セッションベースのテクノロジー。 ユーザーがこのサーバーに接続すると、リモートデスクトップが開き、ユーザーのセッションの下でログインします。

タイプ2-仮想デスクトップテクノロジー、つまりVDI /テクノロジーの使用



これらは1つのサーバーの2つの部分です。 さらに、RDSは職場への迅速なアクセスを提供できます。 なんで？ デバイスに接続されていません。 ポータルに接続し、サーバーに接続して仕事をすることができます。 ユーザーがどこにいても、オフィスの職場、出張中、休暇中など、ユーザーの継続的な操作を保証できます。



どのような新製品がありますか？ RomoteFXテクノロジーの更新された作業。つまり、RDSサーバー2012および2012R2でのグラフィックスの更新された作業。 ストリーミングデータの最適化に加えて、DirectX 11がサポートされています。 私たちが知っているように、これらはまさにシステムの以前のバージョンに対する否定的なレビューでした。



RDSを使用して、単一のエントリポイントを取得します。 ユーザーはドメインにログインし、すべてのリソースにアクセスできます。 RDSは、USBリダイレクトと、セッションを実行しているデバイスとサーバーの両方にUSBを実装します。



ご存知のとおり、Windows Server 2012および2012R2はSMB 3.0プロトコルで正常に動作します。 これは同様にRDSサーバーに影響を与え、SMBおよびRGCDSunプロトコルでユーザーデータを保存するためのディスクを保存できます。 さらに、このテクノロジーはサーバー2012および2012R2に実装されているため、このサービスは簡単に管理できます。 つまり、マネージャーにブックマークがあり、あなたと私はかつて考えましたが、そのおかげで、RDSサーバーの管理は、winserver 2012で実装されたすべてのツールと同様に直感的になります。



そしてもちろん、このサービスは、ユーザーの作業が1つのサーバーのみの操作に依存しない場合に、高可用性機能を提供します。







RDSは何に基づいていますか？ 複数の役割に基づいて構築され、異なるサーバーの両方で構成でき、インストール時に簡単なインストールを選択し、ほとんどの役割を単一のサーバーにインストールします。



ここで強調できる主な役割は何ですか？

リモートデスクトップ接続ブローカー。 彼は、RDSを構築したテクノロジーに応じて、クライアントデバイスをリモートアプリケーションRemoteAppとセッションベースのデスクトップ、または仮想デスクトップに接続することに取り組んでいます。



さらに、リモートデスクトップへのWebアクセスを提供する役割があります。 その使命は、Webブラウザーを介してリソースを提供することです。 さらに、リモートデスクトップセッションホストがあります。この役割により、サーバー上でリモートアプリケーションまたはセッションベースのデスクトップをホストできます。 リモートデスクトップ仮想化ノードがあります。 このノードでは、これがHyper-Vサーバーであり、VDIテクノロジを使用して、すべてのユーザーがアクセスできるすべての仮想マシンを展開します。



後者はリモートデスクトップゲートウェイであり、外部ネットワークからのクライアントと、内部ネットワークおよびアプリケーション上のセッションのコレクションとの間の仲介者です。 ゲートウェイはセキュリティであり、当社のサービスは完全に安全です。 また、2012年と2012R2のRDSがより柔軟になったことにより、以前に解決されたすべての欠点が完全に解決されました。 現在、このサービスは、大企業向けの巨大で大規模なプロジェクトに簡単に実装できます。 以前は、いくつかの質問がありました。







そのため、RDSサーバーにユーザーとして接続するときに、いくつかの質問がありました。 まず、すべてを管理する方法は？ コントロールを紹介します。ここでは、どのユーザーにとっても非常にシンプルで直感的です。 ここでは問題はありませんが、管理者が主にリソースを節約し、ユーザーの質の高い作業を確保できるようにするパラメーターを選択することは可能です。



切断されたセッションの終了時間を制限、設定できます。 このパラメーターは、サーバーが切断されたセッションを終了するまでの時間を指定します。 ユーザーが何らかの理由で切断した場合、これは通信が不安定な場所に最もよく適用されます。 指定された時間内にユーザーが接続し直した場合、ユーザーは自分のセッションに参加します。つまり、使用するすべてのリソースがアクティブになり、接続して作業を続行します。 時間が経過すると、サーバーはすべての一時ユーザーファイルを削除し、セッション全体をキャンセルします。



さらに、アクティビティ、セッションの期間を制限できます。 なぜこれが必要なのですか？ 現在、ユーザーが長く働きすぎている、「仕事は仕事をしているに違いない」などの事実に企業が苦しんでいるとき、その傾向は広まっています。 この原則が尊重される場合、ルールによって非常に簡単に確立できます。RDSサーバーで最大ユーザー時間を設定します。 ほとんどの場合、今回はもう少し多くを入れます。



アイドルセッションを制限します。 ユーザーがしばらくアクションを実行していない場合は、切断されます。 便利なことは、セッションがハングしていることが非常に多く、ユーザーがセッションを忘れてしまい、サーバーの負荷がかかっており、他のユーザーの能力が不足していることです。



これらの設定をすべて設定する場合、 ユーザーはセッションが中断されたときに気に入らないことを覚えておく必要があります。 したがって、分析、監査、すべての要望を収集し、このセットに基づいてこれらすべてのパラメーターを収集することをお勧めします。



RDSサーバーでは、ユーザーに個別のディスクを提供できるようになりました。 つまり、RDSサーバーの設定のパラメーターで、各ユーザーの仮想ディスクを保存する場所を指定できます。



仮想ディスクは、リモートプロファイル、ローミングプロファイルの古いサービスを排除するように設計された新しい「機能」です。 ここでは、各ユーザーに対して、制限されたVHDDドライブを作成することができ、指定したパスに沿って配置されます。 このディスクは、その設定を使用してユーザーに接続します。 リダイレクトされたプロファイル、フォルダーリダイレクトを使用して、各ユーザーに独自の設定、プロファイルを実装できます。



個人的には、すべてのプロジェクトにITインフラストラクチャシステムを展開するときにファイルサーバーを実装することをお勧めします。 これにより、すべてのユーザーのホームフォルダーを作成でき、ユーザーがグループポリシーを使用してログインするときに、このフォルダーをディスクとして接続できます。 これは、ユーザーにドライブを提供する別のオプションです。



ユーザーがリモートデスクトップに接続する別のセッションでログインし、サーバーを作業マシンとして使用する場合の最初の可能性を検討しました。 デスクトップにあるものは、接続するデバイスとしてのみ使用します。







RDSサーバーの2番目の実装。 リモートアプリケーション。 これは、ユーザーがログインし、ユーザーがアクセスできるアプリケーションを表示するポータルであり、ユーザーはそれを使用して作業できます。







RDSはVDIとどう違うのかという質問をよく耳にします。

この質問は、VDIがRDSに含まれるWindows Server 2012および2012R2が登場するまでは正しかった。 セッションレベルと仮想デスクトップレベルでの接続の違いについて話している場合、最初のケースでは、デバイスをサーバーに直接接続し、リモートユーザーとしてリモートデスクトップにログインし、独自のプロファイルを使用してサーバーで作業します。 危険は何ですか？ ユーザーが管理者権限を取得した場合、誰もこれから安全ではありません;すべてのユーザーがサーバーに感染したり、組織全体に損害を与えたりする可能性があります。



VDIを使用する場合、各ユーザーは自分の仮想マシンで作業します。 したがって、VDIテクノロジーを使用して特定の仮想マシンに対してのみ被害が発生するため、「倒れた」仮想マシンを迅速に発生させることができます。 ただし、組織に害を及ぼすことはなく、すべてのユーザーが引き続き作業を続けます。 RDSは重要なサーバーであるため、その使用の計画については十分に注意する必要があります。 これが実装されている多くの企業、その実装に関するアドバイスを提供できる企業があります。

また、試用版を使用することもできます。 すでにサービスを使用して、購入前であってもその機能を確認できます。 すべての機能を無料で見ることができます。







Microsoft Azure RemoteAppが表示されます。 すべてのアプリケーションへのユニバーサルアクセス、安全なアクセスを提供します。最も興味深いのは、世界中のどこからでもアクセスできることです。

このテクノロジーにより、既存のテンプレートとして使用できます。これらは、ターミナルセッションで最も頻繁に使用するアプリケーションです。オフィスアプリケーション+サーバー上にあるもの、計算機、グラフィカルエディターなどです。
