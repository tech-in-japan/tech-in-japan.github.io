JoliOSオペレーティングシステムのテスト

06/07/2012 VMware仮想マシンに「クラウドオペレーティングシステム」JoliOSをインストールして構成しました。 このオペレーティングシステムが「クラウド」であるという事実に私は確信しました。JoliCloudアカウントがなく、インターネットに常時接続していなければ、このオペレーティングシステムはインストールされません。 このオペレーティングシステムを入手したのは、インストールの最初からインターネットに接続してからで、割り当てられたパラメーターとパスワードに注意を払っていました。 そして、この仮想マシンは通常、独自の時間を決定しませんでした。 しかし、順番に見てみましょう。



まず、オペレーティングシステムとしてのJoliOSはCanonicalのUbuntu Linuxクローンの1つであり、そのすべての「母斑」はこのファミリが好きではありません（これは私見です）。 彼のデスクトップインターフェイスは、TiledインターフェイスとGnomeの中間にあります。 Ubuntuのバージョン自体は、インターフェイスとメニューの写真から判断すると、2009年のいつか最新ではありません。 しかし、Ubuntuの新しいバージョンごとに古いエラーが修正され、新しいエラーが表示されるため、これはさらに良いことです（Linux Formatログを参照）。



このインターフェースの使いやすさを否定することはできません。 LinuxのシンプルさとJoliCloudインターフェースの「タフネス」を備えているため、インターフェースをフレンドリーと呼ぶことができます。 私の意見では（再び私見です！）このディストリビューションキットでLinuxを学び始めるのが最善です。



第二に、この配布の欠点は、ロシアの新しい時刻形式のサポートがないことです（GMTから+4時間、夏時間はありません）。 また、ASUS X58Cシリーズノートブックのサウンドカードと画面解像度は、Ubuntuオペレーティングシステムの「ネイティブ」バージョンと同様に、通常のサポートはありません。 この「確執」は、新しいラップトップを購入した後も続きました。



ところで、配布キットをインストールしてオペレーティングシステムの更新システムを起動すると、問題のほとんどが修正されました。



第三に、インターネットに接続していなくても、このオペレーティングシステムは「問題なく」動作します。 自宅の全員が「ルーター」モードで動作するモデム、ホームルーター、またはイーサネットやWi-Fiチャネルを介したインターネット（州内の接続は依然として非常に高価です）で動作しているわけではないため、これによりこのサービスの普及を防ぐことができます。



このオペレーティングシステムの通常の操作では、ブリッジモードで動作する仮想ネットワークカードを備えた仮想マシンにインストールすることをお勧めします（NATと比較してコンピューターリソースの負荷が少ないため）。 その後、モデムがDHCPプロトコルを使用してIPアドレスを設定できるようにする必要があります。このオペレーティングシステムのインストールとダウンロードに問題はありません。



4番目に、このシステムのsuコマンドは、Ubuntuのように、このオペレーティングシステムでもブロックされます。 ただし、Ubuntuとは異なり、コマンドラインからスーパーユーザー権限でプログラムを実行できる「ルートコンソール」アプリケーションがあります。 このコンソールを呼び出すと、コマンド「sudo bash」、「su root」、およびその他の「crutches」を使用してスーパーユーザー権限を取得できなくなります。 システム内のSSHサーバーの存在を確認しませんでした。



第五に、サウンドカードが正しく判断されませんでした。 デフォルトでは、Realtek HD Definition Audioサウンドカードを使用する必要があります。 代わりに、Realtek AC'97サウンドカードドライバーがインストールされており、いかなる方法でも交換することはできません。



第六に、明らかに、JoliOSプログラムは更新され、インターネットに接続するとすぐに更新されます。 そのため、JoliCloudアカウントに再接続した後、その欠点の一部は「自動的に」修正されました（コマンドラインにロシアのインターフェイスが表示され、作業に必要なモジュールが読み込まれました）。



第7に、このシステムには、ダウンロードおよびインストールされたJoliOSモジュール（アプリケーション）の厳密なセキュリティチェックがあります。 ただし、常にモジュールをインストールする場合、有効期限が切れた証明書と無許可のアプリケーションに問題がありました。 このため、プログラムのインストールは非常に遅くなりました（悪意のあるコードの存在についてテストされた可能性があります）。 そのため、Midnight Commanderモジュールは10分以上インストールされました！ Sambaモジュール（10個のパッケージのうち）、私は同時にインストールしました。 これらのモジュールには証明書が必要です。



仮想マシンのその後のテスト：JoliOSは以下を明らかにしました。



1.原則として、JoliOSはインターネットがなくても動作します（通常のUbuntu Linuxマシンのように）。 確かに、「リモートサービス」は機能しません。

2.このゲストOSの通常の操作では、次をインストールする必要がありました。ファイルマネージャーMidnight Commander 、テキストエディターEmacs 、NFSおよびSambaネットワークをサポートするプログラム。

3.すべてのファイルでホスト名、lmhosts、およびホスト名のマシン名とそのIPアドレスの競合を許可しないことが重要です。そうしないと、「原因不明のエラー」が発生する可能性があります。

4. JoliOSでは、ローカルコンピューターにあるすべてのアプリケーションは、JoliOSデスクトップの最終画面のAppsフォルダーにあります。 すべてのプログラムは、Nautilusファイルマネージャーのタイルに配置されます。 そこから起動できます。



VMwareゲストマシンでJoliOSを使用して「ゲストコンピューター」でSambaサーバーを構成することは、すぐには不可能であり、完全ではありませんでした。 このために私はしなければなりませんでした：



5.仮想マシンの/ etc / hostsファイルを更新し、 ホストマシンの％SystemRoot％\ system32 \ drivers \ etc \ hostsファイルと同じにします 。

6.「ゲスト」およびホストマシン上のlmhostsファイルをディレクトリ/ etc / samba / lmhostsおよび％SystemRoot％\ system32 \ drivers \ etc \ lmhostsから削除します。

7.ファイル/etc/samba/smb.confで、WORKGROUPのサイズで、このネットワークの「ワークグループ」を指定します。 このグループは、ホストコンピューターのファイルに登録されています。



％SystemRoot％\ system32 \ drivers \ etc \ networks ;



その後、ホストコンピューターのハードドライブに（管理者権限で）アクセスしましたが、「共有フォルダー」にアクセスできませんでした。 しかし、すでにこのアクセスは、「作業に便利」にしてこのゲストオペレーティングシステムをテストするのに十分でした。 SMBプロトコルを使用したフォルダーへのアクセスに関しては、ここで追加の調査が必要です。
