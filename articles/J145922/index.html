FAQ： OS Infernoとは何ですか、なぜ必要なのですか？



前の投稿の情報はほぼ4年間古くなっていたため、更新するように求められました。 また、1つのポストでインストールとセットアップを混在させないでください。したがって、インストールのみが行われ、 セットアップは別のポストの別のポストで説明されます 。 更新： Windowsのインストールの説明は2014年6月に更新されました。



そのため、 分散Inferno OSをインストールします。 公式Webサイト にはインストール手順がありますが、完全に正しいわけではなく、少し古くなっています。 Infernoは2つのモードで動作できます- ネイティブ （ベアメタルまたはqemu /など。すべての通常のOSと同様）とホスト （* NIX / Winの下の通常のアプリケーション）。 ネイティブInfernoのインストール手順は 、 ロシアのwikiにあります 。 さらに、他のオプションもあります-たとえば、 AndroidにInfernoをインストールする 。 個人的には、通常のコンピューターでネイティブのInfernoを使用する意味がわからないため、Gentoo、Ubuntu、FreeBSD、MacOSX、WindowsでのホストされたInfernoのインストールについて説明します。

インストール機能



OS Infernoバージョン


理論的には、第4版の最新の公式バージョンは2004年頃にリリースされました。 現在のバージョンはGoogle CodeのMercurialリポジトリにあり、「New Edition」と呼ばれています。 実際には、リポジトリの現在のバージョン以外のものを使用することは意味がありません。完全に安定しており、常に安定しています。 入れます。



1人または複数ユーザーのインストールスタイル


Infernoはシステム全体にインストールすることができ（たとえば/usr/inferno/



）、すべてのユーザーが使用できます。 Infernoは、これに必要なすべてをサポートします-ユーザー権限、個別のホームディレクトリなどでの作業。 一方、インフェルノはホームディレクトリに簡単に配置できます（例： ~/inferno/



）。これはさらに便利です。 両方の方法を同時に説明することで前回の記事を複雑にしましたが、シングルユーザーバージョンのインストールのみを説明する方が簡単だと判断しました。 この記事の読者の1人に複数のインフェルノユーザーがいるサーバーがある場合、インフェルノをインストールするための私の指示はほとんど必要ありません。 ;-)したがって、NIXシステムでは~/inferno/



に、WindowsではC:\inferno\



にインストールします。



32/64ビット


OS Inferno 32ビット。 したがって、64ビットOSにインストールして実行するには、これらのOSでの32ビットアプリケーションのサポートが必要です。 残念ながら、64ビットFreeBSD-9.0では、infernoを起動できませんでした。



Hardened / PaX / SeLinux /など


Infernoは仮想マシンでコードを実行し、さらにJITをサポートするため、Javaなどのさまざまな保護と同じ問題を抱えています。 前回の記事で、このトピックにもっと注意を払いました。質問がある場合は、こちらをご覧ください。



時間と場所


インストールされたインフェルノは約200MBかかります。 ただし、コンパイラのインストールには最大3奇数ギガバイトが必要になる場合があります（XcodeやVisual Studioなど）。 Infernoは、平均的なシステムで文字通り数分でコンパイルします。



場所


ホームディレクトリにインフェルノをインストールするとき、インフェルノはスペシャルが好きではないことに注意してください。 ファイル/ディレクトリの名前に文字が含まれているため、ホームディレクトリへのパスにスペースが含まれている場合は、考慮しなかった問題がある可能性があります。

環境変数を構成する


本当に必要な変数はPATH



だけです。 EMU



は、infernoを実行するためのデフォルトパラメータを設定しますが、これは便宜上のみ必要です。 INFERNO_ROOT



、infernoはそれについてまったく知りません。便宜上この変数が必要です。 現在のセッションで変数を設定することに加えて、開始スクリプトに変数を書き込みます。



... Ubuntu 


 export INFERNO_ROOT=$(pwd) export PATH=$INFERNO_ROOT/Linux/386/bin:$PATH export EMU=-r$INFERNO_ROOT echo "export INFERNO_ROOT=$INFERNO_ROOT" >> ~/.bashrc echo "export PATH=\$INFERNO_ROOT/Linux/386/bin:\$PATH" >> ~/.bashrc echo "export EMU=-r\$INFERNO_ROOT" >> ~/.bashrc


... FreeBSD 


 export INFERNO_ROOT=$(pwd) export PATH=$INFERNO_ROOT/FreeBSD/386/bin:$PATH export EMU=-r$INFERNO_ROOT echo "export INFERNO_ROOT=$INFERNO_ROOT" >> ~/.bash_profile echo "export PATH=\$INFERNO_ROOT/FreeBSD/386/bin:\$PATH" >> ~/.bash_profile echo "export EMU=-r\$INFERNO_ROOT" >> ~/.bash_profile


... Mac OS X

PS



「誰に、なぜこれがすべて必要なのか」という伝統的な質問に事前に答えます。 ホストモードでは、infernoはErlang、Java、またはGoとほぼ同じ方法で使用されます。シンプルで快適な記述環境で実行されるクールなプログラミング言語でアプリケーションを開発し、少なくとも他の言語の同様のアプリケーションと同様に動作します。 Erlangとは異なり、Infernoにはすぐに使用できるものはあまりありません（ただし、必要に応じてハンドルで簡単に実装できます）が、問題を解決するのに適した汎用システムです（通常、低レベルドライバーなどを除く）。 Javaとは異なり、完全な軽量スレッドと、シンプルなマルチスレッドアプリケーションをCSPスタイルで作成する機能があります。 Goとは異なり、仮想化および単純化された環境はどのOSでも同じです。 一般に、インフェルノは、商用プロジェクトでの運用で使用する準備が長い素晴らしいシステムであり（そこで使用しています）、それよりもはるかに人気があります。
