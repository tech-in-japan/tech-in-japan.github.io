行こう



Xbee



XBee / Zigbee / Other-beeはそのようなUART-aワイヤレス延長ケーブルだといつも思っていました。 したがって、それはいつも私には理解できませんでした-なぜそんなに高価なのですか。 しかし、その後、友人が「遊び回る」ために2つのXBeeモジュールをくれたので、それらを家にドラッグし、フラッシュ中にそのうちの1つをすぐに殺し、Digiサポート（メーカーXBee： www.digi.com ）に連絡しました。 、ケースを閉じ、実際には手に「焼けた」鉄片が残ったままで、何とか返さなければなりませんでした。 いつものように、私はテーブルで頭を掘り、理解し、誓い、beatりつけなければなりませんでした。 タンバリンと何日も踊った結果、私は次のことを学びました。

  • Zigbeeはそのようなプロトコルです。 低電力ワイヤレスメッシュネットワークの場合
  • XBeeは、異なるプロトコル（Zigbee、Wi-Fi、BlueToothなど）を使用した通信用のワイヤレスモジュールのブランドです。
  • XBee Pro Millivateモジュールを手に入れました（シリーズ2、ZB機能）
  • XBeeはUART（「透明」または「IPA」モード）をスキップできるだけでなく、GPIO /アナログ入力も備えているため、Arduino digitalReadの用語で（5つなど）足をリモートで動かしたり、足の値を読み取ったりできます（つまり、HIGH / LOW）およびanalogRead（つまり、ADC-脚の電圧）。 つまり 設定された特定のXBEEに電源とセンサー（温度センサーなど）を接続でき、それ自体または要求に応じて電圧値をコーディネーターに送信します。
  • Zigbeeネットワークでは、デバイスは次のいずれかの役割を実行できます。

    • コーディネーター-ネットワーク上には1つしか存在できず、眠ることはありません-メイン病棟
    • ルーター-ネットワーク上に大量に存在し、スリープすることはなく、ノードポイントとして機能し、端末デバイスとして機能することができます
    • ほとんどの場合、端末デバイスには自律的な電源から電力が供給されます。この場合、時間の大部分は枯渇し、定期的に目覚めて所有者（ルーターまたはコーディネーター）にpingを送信し、そのメッセージがあるかどうかを確認します。 前の段落で説明したような小さな作業を行うことができます。 足を引くか、入力をサンプリングします。
  • 最大電圧値を超えない場合、XBeeは本当に殺すのが困難です。 ブートローダーが組み込まれており、何か問題が発生した場合は、3本の指の組み合わせでブートローダーモードに入り、モジュールをデフォルト状態に戻すか、必要なファームウェアをフラッシュできます。


一般に、すべてが正常であり、センサーとパフォーマーのネットワークがあり、それらの間のすべてのメッセージがルーティングされ、ルーターを介して相互に通信し、コーディネーターがその上に座ってすべてを監視します。 問題は、私たち人間にはこのネットワークへの入り口がないということです。 最も単純な解決策：USB-UARTコンバーター、コンピューターの電源を入れ、ターミナルまたは特別なプログラム（たとえば、digi：X-CTUまたはXBee-Operatorなどのその他のプログラム、またはpythonでスクリプトを記述する）を実行すると、digi.comにはプラグインがありますXBeeで動作します）。 悪い点は、いつでもどこでもセンサーの状態に関する情報を消化可能な形式で取得したい場合、実行中のWebサーバーを備えたコンピューターであっても、インターネットに突っ込んだものが最もエレガントなソリューションではないことです。



ルーター



よりエレガントなソリューションは、常にオンであり、インターネットを直接見るルーターを使用することです。 UARTを持ち、コンソールにアクセスできるルーターが適しています。 私は古き良きASUS wl500gPとOlegのファームウェア（wl500g.info）を使用していますが、TP-Link、DLink、および消費電力が1桁少ないその他の小さなボックスを使用するのがよりファッショナブルで、UARTをアンソルダーし、そこでOpen-WRTファームウェアを混在させることができます。 方法-Googleで。 以下に例を示します： wiki.openwrt.org/en/toh/tp-link/tl-mr3420 設置およびシリアルポートの項目に興味があります。



集める



重要 ：XBeeは3.3Vデバイスです。 5Vを印加すると、焼き付きます。

XBeeを取り、アダプターをUSB-UARTに接続し、コンピューターに挿入し、X-CTUを起動し、必要なファームウェアでフラッシュします。 最も簡単なオプションはZB24 Coordinator ATです（はい、この時点で、XBeeのファームウェアがどのように異なるか、デバイスに必要な役割などを知る必要があります。インターネットで読む必要があり、ここからかなりよく始めてください： The Unofficial XBee FAQ（version 1.3） ）またはRobert Faludiの本から-ZigBee、XBee、Arduino、Processingを使用したワイヤレスセンサーネットワークの構築XBeeをアダプターとコンピューターから切断し、ルーターのシリアルポートに接続します。

Xbee ルーター
VDD（ピン1） + 3.3V
VSS（ピン10） GND
DOUT（ピン2） Rx
DIN（ピン3） TX


パフォーマンスをチェックします。 minicomを起動し、XBeeが構成されている速度でXBeeがハングするポートに接続します（-D / dev / ttyS1 -S115200があります）。 ATコマンドモードに入ります。「+」を3回押します（これ以上何もしません！）。 しばらくすると、XBeeは「OK」と答えるはずです。これは、コマンドモードに入ったことを意味します。 ATCN <input>コマンドを入力するか、10秒間待機することにより、モードを終了できます。 原則として、これですべてです。このポートで何でもできます。 たとえば、起動時にser2netを登録し、/ dev / ttyS1をTCPポート（たとえば2000）にマップします。その後、ネットワーク上の任意のマシンから、telnetでルーターに接続するか、リモートTSPポートをローカルシリアルにマップするマシンにソフトウェアを配置できます。



例



ルーターにPythonインタープリターがあります。 1時間に1回、cronはいくつかのスクリプトを取得します。 スクリプトの1つは、XBeeがハングするシリアルポートを開き、リモートデバイス（ライザーのトイレに立っている）にリクエストを送信します：「gv」。 デバイスは次の行で応答します： 'hw25.054cw0.000 \ r \ n'および 'v8.72 \ r \ n'。これは、温水計が25.054立方メートルの水を計ったことを意味します。 、バッテリーの電圧が8.72ボルトであること。 スクリプトはこれらの値を取得し、ファイルに書き込みます。 cronが5分ごとに1回プルする別のスクリプトは、これらの値をhtmlテンプレートにパックし、index.htmlとして/ opt / share / www /フォルダーに保存します。 Lighttpdはルーターでも発生し、ルーターのアドレスにアクセスすると、この静的なページが表示されます。 したがって、妻と私は、いつでもソファから立ち上がることなく水道メーターの測定値を調べることができます。

次は？



さて、最も明らかなことは、センサーの動的な調査を行うことです。 つまり、Pythonをlighttpdのfast-cgiモジュールとして登録し、ブラウザから直接スクリプトにアクセスします

さらに、digiにはiDigi Diaソフトウェアがあります-実際には管理コンソールです。 これを使用して、XBeeコマンドをセンサーに与えたり、ファームウェアをリモートでセンサーにアップロードしたり、リセットしたりできます。

Digiにはクラウドもあります（http://www.idigi.com/）。 つまり、切り離されたセンサーまたはネットワーク全体をクラウドに接続し、制御ソフトウェアがどこからでもスピンしてアクセスできるようにします。

コネクタの束もあります（http://www.idigi.com/idigiconnector/）-頭とスキルを持つ人のための一般的な広がり

サードパーティの開発者が作成したソフトウェアがあります。 たとえば、XIG（xig.googlecode.com）-Xbee Internet Gateway。 これはweb <-> xbeeコンパイラです。 たとえば、センサーの状態に関するデータを受信する場合、このデータの末尾をget-requestに固定し、この要求をリモートサーバーに送信します。 または、XBeeからのリクエストに応じて、特定のサイトから情報をダウンロードします。

一般的に、多くのことが可能です。 ルーターからXBeeに4本の配線を接続することは、モノのインターネットへの最初のステップにすぎません。 しかし、この最初のステップでさえ非常に興味深いものです。
