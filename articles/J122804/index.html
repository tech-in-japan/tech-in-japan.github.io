Eclipse Indigoについて

Eclipse Foundationは、Eclipseプラットフォーム、Java開発者向けツールセット、ソーステキスト管理ツール、GUIのビジュアルビルダーなど、Eclipseの支援の下で開発されたすべてのプロジェクトを統合するEclipse 3.7 （Indigo）の年次リリースのリリースを発表しました 。 49の異なる組織からの402人のコミッターが新しいリリースの準備に参加しました。 Eclipse 3.7の一部としてリリースされたソースコードの総量は、4,600万行のコードです。 リリースに含まれるプロジェクトの数は62に達しました （昨年は39のプロジェクトがありました）。



10個のEclipseビルドがダウンロード用に準備され、Java EE、JavaScript、およびC / C ++を使用する開発者向けオプションを含むさまざまなプラットフォームアプリケーション向けに最適化されています。 残念ながら、今年、メンテナーがいないため、PHP言語の開発者向けの既製のアセンブリは作成されませんでしたが、Eclipse環境に組み込まれたアドオンインストールマネージャーにPHP開発ツール（PDT）をインストールすることで、PHPサポートを有効にできます。

Java開発者向けの主な改善点 ：

  • GitリポジトリーEGit 1.0を操作するためのユーザーインターフェイスの安定リリース。

  • m2eclipse -Mavenシステムとのシームレスな統合。pomファイルの操作、Eclipse作業環境からのビルドの起動など。

  • Jubula -JavaおよびHTML用の自動GUIテストシステム。

  • WindowBuilderは、GUIインターフェイス用のビジュアルデザインシステムです。 WindowBuilderコードは今年Googleによって開かれました。

  • Eclipse Marketplace Clientは 、ドラッグアンドドロップを介してディレクトリから「ソリューション」をインストールするためのサポートを追加しました。

  • Mylyn 3.6は、Eclipse環境から直接Hudsonのビルドを監視する機能を追加します。



モデリングの改善：

  • Xtext 2.0は 、ドメイン固有の言語を作成するプロセスを大幅に簡素化しました。 組み込みのJavaのような式でドメイン固有の言語を作成するためのサポートが追加されました。 Xtendが追加されました。これは、テンプレートを定義するための新しい言語であり、コード生成をEclipseツール環境に緊密に統合できます。 ドメイン固有言語用の新しいリファクタリングフレームワークが追加されました。

  • Acceleo 3.1は、コード生成をAntおよびMavenのアセンブリチェーンに統合し、コードジェネレーターを編集する機能を改善します。

  • CDO Model Repository 4.0は、Objectivity / DB、MongoDB、DB4OなどのいくつかのNoSQLデータベースのサポートを追加します。 キャッシュの最適化およびその他の改善により、数ギガバイトを占有するモデルを操作できるようになりました。

  • EMF 2.7は、プロセスを簡素化し、分散ネットワークのすべてのノードで変更を複製する効率を高めます。 データの全部分を送信する代わりに、クライアントは特定の変更についてサーバーに通知するようになりました。

  • EMF Compare 1.2はUMLのサポートを追加し、SCMとの統合を改善しました。

  • EMF Facetは、既存のメタモデルを変更せずに拡張機能を作成できる新しいプロジェクトです。



その他の改善点 ：

  • EquinoxでのOSGi R4.3仕様の実装。

  • Eclipse Platformインターフェースの多数の改善 。

  • マルチタッチおよびジェスチャ制御のサポート。

  • すべてのプラットフォーム（Windows、Linux、およびMac OS X）のWebKitサポート。

  • Eclipse Communication Framework（ECF）でのOSGi 4.2 Remote ServiceおよびRemote Service Admin標準の実装。

  • Riena 3.0は、Eclipse RAPを使用してWebブラウザーアプリケーションを構築するためのサポートを追加します。
