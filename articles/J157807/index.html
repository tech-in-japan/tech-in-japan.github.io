Windows 8 x64でCisco VPN Clientを使用してVPNに接続する（ほぼ解決済み）

こんにちは



Windows 8互換性センターがCisco VPN Clientと新しいOSとの完全な互換性を主張しているという事実にもかかわらず、このクライアントを非自明なアクションによってのみ動作させることができました。



しかし、情報が有用であり、おそらく集合的な心が問題を最後まで解決するのに役立つことを願っています。



そのため、シスコの機器上に構築されたVPNと、64ビットWindows 8 Professionalで接続する必要があります。 開始するには、Cisco VPN Client 5.0.07.0440の最新バージョンをインストールしてください。 複雑なことなくインストールが行われます。 接続設定でお気に入りの.pcfをインポートし、接続を試みます。 それから問題があります：



問題番号1：「理由442：仮想アダプターを有効にできませんでした」エラー



この問題は、レジストリのキー値を修正することで解決されます。

  1. レジストリエディターを開きます（検索バーに「regedit」と入力し、見つかったアプリケーションを実行します）。
  2. ブランチHKEY_LOCAL_MACHINE \ SYSTEM \ CurrentControlSet \ Services \ CVirtAが見つかりました。
  3. パラメーターDisplayNameを見つけます。
  4. このパラメーターの値には、「oem4.inf、％CVirtA_Desc％; Cisco Systems VPN Adapter for 64-bit Windows」などが含まれます。 「64ビットWindows用のCisco Systems VPN Adapter」のみを残して、この値を変更する必要があります。


レジストリを更新した後、再起動する必要はありません。 これで、仮想アダプターは安全に配置され、共有キーを使用して認証が実行された場合（クライアント証明書は不要）、これで問題は解決されました。



証明書を使用して認証が実行される場合、次のものがあります。



問題番号2：「理由403：セキュリティゲートウェイに接続できません」



証明書（eTokenタイプの別のデバイスに秘密鍵を必要としない）は通常、ユーザー証明書ストア（ユーザーストレージ）にアップロードされることが理解されています。 同時に、クライアントのログに次のメッセージがあります。「ストアMicrosoftユーザー証明書から証明書[証明書の説明]をロードできませんでした。 理由：空のストア。 つまり、ストア内に証明書が存在するにもかかわらず、VPN Clientはそれを認識しません。



この問題を解決するには2つの方法がありました。

  1. 証明書をユーザーストアからローカルコンピューターストアに移動します。
  2. シスコシステムズのサービス設定の変更 「ログオン」タブの「VPNサービス」。ユーザーアカウント（システムに自分で入力して接続しようとしているのと同じアカウント）でサービスを強制的に開始します。


次のレベルに進みます：e-token（Alladin）を使用したキー認証ができました。 キー（eToken PKIクライアント）が付属するプログラムがあり、USBトークンがマシンに接続されると、トークンの証明書がユーザー証明書ストアに自動的に配置されます（これが、2番目の方法を使用して問題番号2を解決した理由です）。 この構成でVPNに接続しようとすると、次のエラーが表示されます。



問題番号3（未解決）：「理由401：VPN接続の確立中に認識できないエラーが発生しました」



クライアントのログには、「署名の生成に失敗しました：署名の生成に失敗しました」というメッセージと、その他の情報の少ない製剤が表示されます。 残念ながら、これは行き詰まりです。ログメッセージは問題の本質を明らかにするものではなく、さらに掘り下げる方向は不明です。



私はこの問題で私一人ではなく、誰かがより賢くて幸運になることを願っています。



UPD：接続の代替オプションとして、Windows 8での実行時に問題のないShrew Soft VPN Clientを使用できます（このプログラムのインストールと構成に関する記事は既にハブでスキップされています）。 プログラムにはマイナスが1つあります-Windows証明書ストアからの証明書の操作方法がわからないため（接続のセットアップ時に証明書をファイルからダウンロードする必要があります）、これもeTokenのキーの場合には適していません。
