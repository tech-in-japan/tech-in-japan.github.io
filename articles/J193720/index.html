ノキアコンテストのアイデア

Lumia 1020のリリースまでに、NokiaはFuture / Captureコンテストを開催しました（ ハブについて書いています）。 41メガピクセルのカメラを最大限に活用するアプリケーションの3つのアイデアをコンテストに提出できます。 私はこの機会を利用しましたが、資格はありませんでした。



私のアイデアは、写真の高解像度にほとんど関係しているため、この形式でそれらを実装する方法はわかりません。 ほこりっぽくしておくのは恥ずかしいことなので、3つのアプリケーションのアイデアをパブリックドメイン（ CC0 ）に渡します。



1. Waldoはどこにありますか？



本やゲームのWhere's Waldoに似たゲームプレイを備えたインタラクティブなアプリケーション。



どのように機能しますか？


  1. コンサート、ビーチ、森など、細部をたくさん撮った写真を撮ります。
  2. 写真からクエストを作成します-関心のある小さな領域（たとえば、あなた、子猫、小さなオブジェクト）をマークします。
  3. 友達とクエストを共有します。
  4. 友達の仕事は、マークしたポイントを見つけることです。 友人同士の競争、時間の検索、評価などがあります。

2. 1枚の写真のストーリー



1つの写真から短いビデオストーリーを作成するためのアプリケーション。



どのように機能しますか？


  1. あなたは素晴らしい写真を撮ります。
  2. 画像の興味深いフラグメントを選択すると、これらが新しいビデオのキーフレームになります。
  3. 各キーフレームにフィルターを追加し、説明を追加し、それらを変換（スケール、回転）します。
  4. キーフレーム間にトランジション（増加、ターン）を適用し、トランジションの長さとキーフレームの期間を設定します。
  5. ビデオを友人と共有する（YouTube、Vine、または同様のサービスを使用）。

3.コラージュ



コラージュを使用して、不足している詳細を写真に追加します。



どのように機能しますか？


  1. 写真を撮ります。
  2. 不足している詳細を個別に撮影します。
  3. このアプリケーションは、元の画像と元の写真を組み合わせて、全体像を取得します。


三人組


あなたが3人で写真を撮っている場合、フレームで行方不明になっている人が常にいます。 第三者の写真を別々に撮り、全員を集めます。







興味深いオブジェクト


美しい写真を撮りましたが、面白い物体（車、雲）に気付いた後です。 新しいディテールで別のショットを撮り、元の写真と組み合わせるだけです。







技術的な詳細


許容できる品質で実装されている場合でも、最も技術的に困難なタスク。 パノラマを接着するときに使用されるアルゴリズムを使用する必要があります。 いくつかのアルゴリズムは詳細に説明されており、いくつかのアルゴリズムはオープンソースソフトウェアで使用されています。 すべては共通の手順に基づいています：

  • 両方の画像で共通点を見つける
  • 2番目の画像の変換行列を見つける
  • 2番目の画像を最初の画像に合うように変換します。
  • 画像を組み合わせる
