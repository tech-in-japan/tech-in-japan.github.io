ドメインのJabber

自分でドメインを登録しました。 もちろん、メール、ジャバー、ブログをねじ込みたいと思っていました... Google Appsはメールで素晴らしい仕事をしています。 すべてのサービスを使用しようとしましたが、多くの人がGoogleトークでよく知っている迷惑に出会いました-メタコンタクト、ブックマーク、帰属するユーザー情報の欠如、そしてもちろん、アカウントから5分ごとにステータスを送信します。 少しグーグルで、共有ホスティングをレンタルするとき、または別のサービスとしてJabberを現在提供しているロシアのホスティング会社をいくつか見つけました。 それとは別に、hc.ruでのみJabberを注文できましたが、無料で将来的には月額50〜100ルーブルを超える可能性は低く、サーバーの地理的位置はGoogleよりも高速な接続を提供します。 カットの下での登録に関する詳細。

1）-まず、 hc.ruで登録する

2）-次に、 hc.ru / ru / services / jabberに移動し、[注文]をクリックします。

3）-次の図が表示されます。



サービスの有効期間を選択し、最初のアカウントの名前を入力して、「次へ」をクリックします。

4）-次のことが判明しました：



[ドメインを選択]をクリックすると、次のように表示されます。



5）-[以前に登録したドメインを使用する]の前にチェックマークを付け、[次へ]をクリックします

6）-ドメインの名前を入力します：



7）-2行に注意を引きます：

_xmpp-server._tcp SRV 0 0 5269jb1.hc.ru。

_xmpp-client._tcp SRV 0 0 5222jb1.hc.ru。

これをドメインの管理パネルのSRVに追加する必要があります（具体的には、 webnames.ruに登録し、そこでドメインのすべての操作を実行します）。 これについてもう少し説明します。



このような何か、SRVレコードを追加する必要があります。つまり、service-_xmpp-client._tcp、priority-0、weight-空のままにします。デフォルトは0、port-5222、target-jb1.hc.ruです。 （はい、最後にドットが付いています）。

8）-[次へ]をクリックして注文を確認します。



「注文」をクリックしてください

さて、実際には、これですべてです:)これで、電子メールでメッセージを受信します。

-注文について

-ユーザー名jabberサーバーがアクティブ化されていること



これで、すべての新しいSRVレコードが更新され、サーバーがアクティブになるまで待つ必要があります。 確実に知るために、jabberクライアントでサーバーを見つけてみてください。 見つかった場合、それはすでに機能していることを意味し、使用を開始できます。 夜、モスクワ時間の2時に登録しましたが、午前中はすべて機能していました:)これまでのところ、このサービスは企業ソリューションには適していません-1ドメインで10 jidの制限があります。 しかし、彼自身と彼の家族にとって、この決定はちょうどいいです。



PS別のサービスとしてjabberを提供している他の会社は見つかりませんでしたが、1gb.ruとspaceweb.ruがあり、共有ホスティングをレンタルすることで、ドメインでjabberサーバーを無料で使用できます。
