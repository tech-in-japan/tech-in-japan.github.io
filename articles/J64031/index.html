WordPress 2.8.1がリリースされました。 このバージョンには、重要なセキュリティ修正が含まれています。

WordPress 2.8.1は多くのバグを修正し、プラグイン管理ページのセキュリティボルトを強化します。 Core Security Technologiesは、一部のプラグインによって追加された管理ページは、適切なアクセス権がなくてもユーザーが表示できるため、情報が漏洩する可能性があることを通知しました。 すべてのプラグインがこの問題の影響を受けるわけではありませんが、安全のためにバージョン2.8.1にアップグレードすることを強くお勧めします。



2.8のリリース以降、他に何が新しくなりましたか？ 以下に、2.8以降の最も重要な変更点の短いリストを示しました。

  • 一部のトピックでは、エラーが発生するような方法でget_categories（）関数を呼び出しました。 2.8.1。 この欠陥を修正します。
  • コントロールパネルのメモリ消費が削減され、一部のユーザーは空きメモリを使い果たし、コントロールパネルの読み込みが不完全になりました。
  • 自動更新により、インストールファイルのクリーニング中および更新の失敗後にファイルが誤って削除されることはなくなりました。
  • 圧縮の問題に関連する問題は解決されました。そのため、テキストエディターが読み込まれませんでした。
  • 明示的なセキュリティチェックに合格しないプラグインからユーザーを保護するために、追加のセキュリティ対策が講じられています。
  • 現在、wp_page_menu（）は、タイトルではなく、メニューでユーザーが選択した順序でデフォルトでソートされます。
  • ダウンロードエラーメッセージが正しく表示されるようになりました。
  • 一部のIEユーザーの自動保存エラーを修正しました。
  • プラグインエディタのスタイルの不具合が修正されました。
  • ブラウザの非互換性により、構文の強調表示が無効になりました。
サイトにはまだロシア語版はありませんが、Russified WordPressはエラーなしで自動的に自動的に更新されます。 素敵なアップデートをしてください。
