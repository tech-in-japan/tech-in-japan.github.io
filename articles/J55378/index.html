LinuxでのIEブラウザーでのWEBアプリケーションのテスト

LinuxでWEBアプリケーションをテストするには、IE6とSafari（SSLなし）を正常に起動するWineを使用します。 Google Chromeは明らかに起動に失敗しました。 ies4linuxを使用したIE7およびIE8については何も言いません。



Googleで短い検索をした後、 試用版のWindowsを使用したVirtual PCで仮想マシンの画像を見つけました。 必要なようですが、残念ながら、Virtual PC for Linuxは存在しません。 幸いなことに、VirtualBoxの最新バージョン（2.x）はVHDイメージで動作します。



IE7でtrial'nika Windowsを開始する手順：

1）Wineをインストールします。

sudo apt-get install wine





2）VirtualBoxをインストールする

sudo apt-get install virtualbox





3）イメージ「IE7-XPSP3.exe」を含むアーカイブをダウンロードし、コマンドを実行します。

wine IE7-XPSP3.exe





ウィザードで、アーカイブを解凍します。

5）VirtualBoxを起動し、ゲストシステムを作成し、OSの種類を選択します-Windows XP、メモリサイズを約300Mb（十分であるはずです）に設定し、既存のイメージ「XP SP3 with IE7 2008-Dec.vhd」をハードディスクとして追加します。

6）仮想マシンをセーフモードで実行します（Windowsブート画面が表示される前にF8を押します）。

7）Runを介してロードした後、次のコマンドを実行します。

sc config processor start= disabled

sc config intelppm start= disabled





8）仮想マシンを再起動します。

9）ゲストOSのソフトウェアをインストールします。

wget download.virtualbox.org/virtualbox/2.0.4/VBoxGuestAdditions_2.0.4.iso





そして、ゲストOSにISOイメージをマウントします。 ドライバーをインストールし、再起動します。

10）人生を楽しむ。



残念ながら、試用版ゲストOSを追加するとエラーが発生します。

A hard disk with UUID {b6202eec-1791-b921-4c0c-4306e34cc487} or with the same properties ('/home/user/XP SP3 with IE7 2008-Dec.vhd') is already registered.





したがって、次のようにディスクイメージを変換する必要があります。

11）qemuをインストールします（sudo apt-get install qemu）。

12）コマンドを実行します：

qemu-img convert -f vpc XP\ SP3\ with\ IE7\ 2008-Dec.vhd -O raw XPIE7.bin





VHDイメージをバイナリに変換します。

13）その後：

vboxmanage convertdd XPIE7.bin XPIE7.vdi





バイナリイメージからVDIイメージを作成するには。

ここで、VHDからVDIへの直接変換により、使い慣れたイメージが作成されることに注意してください。

14）結果のVDIイメージを新しい仮想マシンに追加します。

15）7項を繰り返します。
