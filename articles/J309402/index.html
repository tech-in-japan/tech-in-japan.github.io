Microsoft StorSimple仮想アレイ。 パート2

以前のMicrosoft Azure StorSimpleの投稿：Easy Hybrid Cloud Access。 パート1では 、Microsoft StorSimpleがハイブリッドクラウドストレージを使用し始めたばかりの人に最適なソリューションである理由を説明しました。 Microsoft Azureを使用できるエンタープライズ契約を締結した企業は、3月には早くも仮想バージョンであるMicrosoft StorSimple仮想アレイにアクセスできました。 VMwareおよびHyper-V環境でも使用できます。 この決定に関する一連の質疑応答からの小旅行を提供します。



StorSimple仮想アレイは誰にとって完璧なソリューションですか？



マイクロソフトのStorSimple仮想アレイは、常に柔軟性が必要な分散ITインフラストラクチャで運用している企業に最適なソリューションです。 多くの企業では、IT部門はデータのバックアップと復元のプロセスを制御し、災害復旧を一元的に実行しようとしています。 多くの場合、ハードウェアへの長期投資は最適なソリューションではありません。特に、会社が動的な成長の段階にある場合はそうです。 これが、StorSimple仮想アレイの出番です。



その利点は明らかです。 このソリューションは次のとおりです。

  • 適応的
  • 費用対効果の高い
  • 非常に柔軟
  • 管理が簡単


そしてまた：

  • Microsoft Azureクラウド環境での作業を含む、多くの主要な機能とアプリケーションを提供します。




このソリューションは技術的な観点からどのように見えますか？



最初に、StorSimple仮想アレイは、その兄弟であるStorSimpleハードウェアデバイスとは異なり、iSCSIプロトコルだけでなくSMBプロトコルでも使用できることを覚えておく必要があります。 これは、既存のファイルサービスの操作を簡単に開始できると同時に、以前のファイルサービスとまったく同じ利点を提供できることを意味します。



会社で使用されているDFS手法はすべて、StorSimple仮想アレイと簡単に組み合わせることができます。

Microsoft Azureクラウド環境と組み合わせると、StorSimple仮想アレイは追加の可能性を開きます。



  • 電子バックアップストレージであるAzureに統合されたBackup Vaultは、StorSimple仮想アレイに保存されたデータをバックアップおよび復元します
  • 事故が発生した場合、バックアップアレイへの移行は、単に
  • 必要に応じて、Azure Automationサービスを使用することにより、必要に応じてこの手順を自動的に実行できます。
  • 兄によって実行される次の機能もこの仮想アレイで使用できます。

    • ボリュームのローカルバインディング
    • ログイン
    • 暗号化
  • 接続は「内部から外部へ」のスキームに従って排他的に確立されます。これは間違いなくITセキュリティスタッフに高く評価されます。


必要条件



StorSimple仮想アレイには、次の仮想化プラットフォームが必要です。

  • Hyper-V（2008、2012、2012 R2）または
  • VMware（5.5以降）
さらに、VMは次のパフォーマンス要件を満たしている必要があります。

  • 8 GB RAM
  • 4コア
  • オペレーティングシステムとキャッシュ用に500 GBのローカルディスク領域


どのアーキテクチャを使用できますか？



Azure Blob Storageは、重要なアーキテクチャ要素です。 ボリュームとコンテナーのストレージ内のボリュームを作成するには、ユーザーはAzureストレージアカウントを登録する必要があります。 ストレージアカウントは、BLOBストレージの可用性を決定するために、冗長設定（たとえば、地理冗長ストレージ（GRS）、ローカル冗長ストレージ（LRS））を設定します。 仮想アレイはこのストレージを使用して適切なデータブロックを格納します。



StorSimple管理ツールは、すべてのStorSimpleインストールの管理に使用される主要な管理ツールです。 Webブラウザからのみアクセスできるため、ローカルコントロールをインストールする必要がありません。







Microsoft Azure Cloudは、他の高性能サービスも提供します。 したがって、現在クラウドに保存されているデータは常に利用可能です。 データは、Azureクラウドのいわゆるクラウドイメージに保存され、仮想デバイスを使用してデータにアクセスできます。 この機能は、Hadoopなどの別のアプリケーションでデータを処理する必要がある場合に非常に便利です。



可能なユースケース



StorSimple仮想アレイは、主に現場にITスタッフがいない企業の支店でファイルソリューションとして使用されます。 この環境は、StorSimple管理ツールによって提供される中央管理なしでは機能しません。 仮想化ブランチサーバーが既にインストールされている場合、仮想アレイで追加のVMを使用しない理由はありません。



まとめると



Microsoftの仮想アレイは、非常にシンプルなソリューションです。 さらに、StorSimple仮想アレイは、クラウドの踏み台として奇跡的に役立つハイブリッドソリューションです。
