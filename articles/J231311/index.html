AndroidおよびMikroTik

真新しい小型MikroTik mAP 2nを見越して、AndroidとMikroTikのバンドルの可能性に対処することにしました。 結局のところ、すべてはそれほど難しくありません。









私はオリジナルを持っています：

  • MikroTik RouterBoard 951g-2hnd RouterOS 6.17
  • HighScreen Boost Android 4.1.2


WiFiテザリングを使用してそれらを接続する方が簡単ですが、ここではAndroidですべてが簡単です。アクセスポイントモードをオンにして、MikroTikを接続します。 USBテザリングを介した通信の可能性を検討することはより興味深いです。



一般に、RouterOS 6.7以降およびMikroTikでのUSBの存在が操作に必要です。

6.7の新機能（2013-Nov-29 13:37）：

*）Android USBテザリングインターフェイスをサポート。

...


Androidスマートフォンバージョン2.2以降。



  • AndroidスマートフォンをUSBケーブルでMikroTik機器に接続します。

    MikroTikはデバイスを識別する必要があります。

    /system resource usb print







    私にとってはこのように見えます：

    # DEVICE VENDOR NAME SPEED





    0 1:1 Linux 3.3.5 ehci_hcd RB400 EHCI 480 Mbps





    1 1:5 Android Android 480 Mbps







  • Androidスマートフォンでは、データ転送とUSBモデムをオンにします。

    1. 設定-ワイヤレスネットワーク-その他...-モバイルネットワーク-データ転送=有効
    2. 設定-ワイヤレスネットワーク-その他...-モデムモード-USBモデム=有効


    明らかに






  • MikroTikで：

    1. 表示されたlte1インターフェイスで：デフォルトルートの追加=有効
    2. IP-DHCPクライアントで、lte1インターフェイスのクライアントを追加します
    3. IP-ファイアウォール-NATでは、NATのルールを追加します




コンソール経由：

/interface lte set lte1 add-default-route=yes





/ip dhcp-client add interface=lte1 disabled=no





/ip firewall nat add chain=srcnat action=masquerade out-interface=lte1 disabled=no
