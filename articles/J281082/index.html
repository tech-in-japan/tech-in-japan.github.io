マトリックスKVMスイッチング：ソリューションの比較

マトリックススイッチングの現象は新しいものからはほど遠いものの、比較的需要が高いという事実にもかかわらず、今日利用可能なMatrix KVMソリューションの客観的な比較を見つけることはできませんでしたが、それは義務のために必要でした。 「他の人に教えれば、あなたは自分自身を理解するだろう」という原則に導かれて、私はあなたが今読んでいるものを書きました。 私の経験が誰かに役立ってくれたら嬉しいです。 同様のシステムで作業しなければならなかった経験豊富な人々の専門家のコメントと意見も歓迎します。



しかし、最初に、彼らが何について話しているかをよく理解していない人々のための小さな余談。



今日、私は確信しています：KVMは非常に有望であると同時に、何からでも機能的で柔軟で管理可能なマトリックスネットワークを構築できる唯一のテクノロジーです。



  • UNIX / Windows / Mac / Sunサーバー、
  • ビデオカメラとビデオプレーヤー、
  • あらゆるものの診断と監視のための機器、
  • 電源、照明、温度制御モジュール、
  • 建設用クレーン、オイルポンプ、
  • 洗濯機、最後に（ビデオ出力とHIDインターフェースのみの場合）


つまり、KVMテクノロジーのおかげで、すでに世界を支配できます。 そして、これがIPを介したKVMである場合、ソファから上昇することさえありません。 これまでのところ、エレクトロニクスの世界だけが明らかになっていますが、サイボーグ化はもうすぐです。



それまでの間、私たちは自分の持っているものに満足するでしょう。 上記から何かを制御するために必要なのは、ディスプレイとマウス付きキーボード（まあ、ジョイスティック、タッチパッド、トラックボール、またはペダル付きステアリングホイール-好きな人）です。 もちろん、マトリックススイッチングのソリューションの一部です。



「でも何がそんなに特別なの？」 -お願いします。 - 結局のところ、シンクライアント、GSMモジュール、および組み込みKVMスイッチを備えたIPコンソールがありますが、決して新しいものではありません。 これにより、ソファから立ち上がることなく、機器をリモートで制御することもできます。



「しかし、同じではない 」私は神秘的に答えます。



まず、世界支配は楽しいはずです。 そのため、ソフトウェアの気まぐれさと遅いシンクライアントはすぐに一掃されます（洗濯機でOSを更新した後は、ソファから出るだけでなく、ソファ自体を引っ張る可能性が非常に高くなります）。



第二に、ご存知のように、権力者はしばしば妄想に苦しんでいます。 そして特に妄想的なKVMは、脆弱なTCP / IPまたはGSMプロトコルを介してではなく、切断のみ可能なケーブルを介して直接制御する機能を提供しますが、誰もこれを行うことはできません。リモートで視聴することもできます。



第三に、分割して征服します。 世界を管理するという全体の負担を負う必要はまったくありません。 12人または2人のスマートアシスタントを募集し、それらの間で責任を分散させることができます。 しかし、a）それらを制御する必要があり、b）最も良心的なヘルパーでさえ気絶する可能性があるため、運用上の互換性を提供する必要があります。 さらに、遅かれ早かれ、新しいアシスタントが完全に管理できるようになる前に、新しいアシスタントを訓練する必要があるかもしれません。 これらはすべて、すべてのMatrix KVMソリューションで提供されます。これにより、管理者はユーザーグループまたは各ユーザーのアクセス権を個別に微調整し、ユーザーが管理対象システムを即座に切り替えることができます。



しかし、他のリモート接続方法との最も重要な違いは、 KVMテクノロジーが、1つのソースからの信号を多数のワークステーションへの同時ブロードキャストで配信する機能を提供することです。 つまり、同じシステムからビデオをVasyaとFediのディスプレイに出力し、オーディオをPetyaのヘッドフォンに出力できます。 また、システム全体はGennady Petrovichが管理します。 ほとんどのコマンドポイントとコントロールポイントの動作は、この原則に正確に基づいています。 そして、これはまさにマトリックススイッチングです。



したがって、 マトリックススイッチングは、多くのユーザーが動的に調整された組み合わせで多くのシステムに接続できる場合です 。



  • 1人のユーザー-多くのシステム。
  • 多くのユーザー-1つのシステム。
  • 多くのユーザー-多くのシステム。


比較表で、これがどれだけ多いかという質問に戻ります。一般に、2x2スイッチングシステム（2人のユーザー、それぞれがワークステーションから2つのシステムを制御できます）は既にマトリックスと見なすことができます。 ただし、この場合、本当に印象的なマトリックスに興味があります。 つまり、100を超えるポイント（システムとユーザー）の総数を持つマトリックスです。そして重要なことは、ロシアで利用可能です。



マトリックスネットワークの構築



マトリックスネットワークを構築する場合、2つのグローバルタスクが解決されます。



  1. ビデオ、オーディオ、USB HID、およびさまざまな必要な周辺信号の拡張。
  2. マトリックスの構築。 つまり、柔軟に構成可能で制御されたシステムへのリアルタイムのユーザーアクセスの組織です。


最初のタスクは、「適切な」KVMエクステンダーを選択することで解決されます。 マトリックススイッチは、2番目の問題の解決に成功します。 すべてが単純なようです：1つを選択し、別のものを選択-通勤、設定。 しかし、マトリックススイッチングは複雑なソリューションであり、通常は「ネイティブ」の延長コードしかマトリックススイッチに接続できないため、最適なソリューションを選択することは困難です。



それでは、これら2つの問題を解決する際に、どのような要件を考慮するのでしょうか？



信号拡張



送信信号の規格 。 標準セットは、ビデオ、オーディオ、USB HIDまたはPS / 2です。 USB 2.0が必要な場合がありますが、USB 3.0およびRS-232が必要になることは少なく、他の周辺機器信号が必要になることもあります。



一方、標準セットのビデオとオーディオは、アナログまたはデジタルにすることもできます。 また、デジタルビデオにはDVI、DP、HDMIなどがあります。一般に、これも非常に異なります。



ビデオ品質とデータ転送速度 。 物理学は容赦ありません。ビデオの解像度が高いほど、長距離での信号伝送速度は低下します。 一方、最先端のマトリックススイッチングシステムは、4Kビデオを長距離で送信でき、遅延はミリ秒単位で測定されます。



もう1つは、利用可能なビデオ解像度が増加すると、KVM延長コードの価格が正比例して増加することです。



中および最大データ伝送距離 。 1）光学系（ファイバーMMまたはSM）またはツイストペア（CATx）を介して直接切り替えることができます。 2）IPネットワーク経由。



（送信機と受信機の両方の）各延長コードからマトリックスに直接切り替える場合、別個のワイヤを延長する必要があります。つまり、マトリックスのために、実際には別個のケーブルシステムを構築する必要があります。



IPを切り替える場合、送信機は通常のネットワークスイッチに接続され、受信機はユーザーワークステーションの標準ネットワークソケットに接続されます。 KVM over IPを使用すると、標準の既存のネットワークインフラストラクチャを使用できます。 また、KVM over IPマトリックスを構築するためにマトリックススイッチは必要ありません。



直接接続では、ビデオ信号の伝送距離が長くなるほど、損失と歪みが大きくなります。 KVM over IPの場合、距離は重要ではありません。 図を参照してください：







マトリックス構成



マトリックススイッチが直接関与するパラメーター：



マトリックスサイズ。 スケーラビリティ 。 接続可能なユーザーとシステムの可能な数。 行列には、静的ポート（個別に入力、個別に出力）または動的（I / O）を使用でき、ポートの数が限られているか、モジュール式に拡張可能です。 スケーリングのための最も柔軟で便利なソリューションは、動的ポートを備えたモジュラーアーキテクチャのマトリックススイッチに基づいていることは明らかであるため、そのようなソリューションだけを比較します。



グループの作成とユーザー権利の設定の柔軟性 。 権利管理機能は異なる場合があります。 そのため、たとえば、一部のユーザーはシステムを制御する権利を持ち、他のユーザーは表示する権利のみを持ち、さらに他のユーザーは接続する権利を持たないようにします。さらに、さまざまなマトリックススイッチが独自の制御インターフェイスを提供します。度。



集中接続管理ツール 。 Webインターフェイス、管理コンソールのソフトウェア、リモートOSDメニューなどを通じて。



耐障害性 。 ネットワーク冗長性の可能性、マトリックススイッチコンポーネントのホットスワップ可能性、バックアップ電源



その他のチップ 。 自動バックアップ、集中ファームウェア更新、メニューのロシア化、設定のコピー、ログのアップロードなど。



さて、最後の基準は、マトリックススイッチではなく、ソリューション全体に大きく関連しています。



システム間の切り替えの速度と利便性、マルチシステムマルチモニタージョブを作成する機能 。 システムごとのモニターの最大数、1人のユーザーが制御できるシステムの最大数。 システム間の切り替えの可能性（マウスカーソルの移動、OSDメニュー、コントロールパネルなど）。



実際、これらはすべてKVM信号のマトリックススイッチングの決定において異なる可能性がある主要なパラメーターです。 その間、どんな詳細な決定からも、あなたは非常に詳細な情報を見つけることができます（そして、多くの場合、機器の公式ディストリビューターでさえ、システムのすべての能力について知りません）。



したがって、これまでのところ、いくつかの（基本的な）パラメーターのソリューションを比較できます。
