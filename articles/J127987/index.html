時期尚早のスケーリング-スタートアップの死の主な理由は？

Startup Genomeは、5月のスタートアップ分析レポートに 付録を公​​開しました。 現在、3200社以上の企業に関する情報がすでに収集されており、スタートアップがバラバラになる主な理由は詳細に検討されています。



数千のスタートアップの経験を調査した結果、Startup Genomeは次のように結論付けています。失敗の70％は早期のスケーリングによるものです。



要するに、何を意味するのか。



スタートアップゲノムの専門家は、スタートアップを一種の5D生物と考えています。 成長中に、彼は5つの独立した領域を調和して開発する必要があります。

  • ユーザー
  • 製品
  • チーム
  • 財政
  • ビジネスモデル
大企業になる前のスタートアップ開発の6つの段階のそれぞれで、これら5つの分野の異なる比率は調和していると見なされます。 早すぎるスケーリング-1つまたは複数の方向で開発が速すぎるか遅すぎる。 スタートアップを管理する技術は、それらすべてを均等かつ調和して開発することに正確にあります。そうでなければ、崩壊が発生します。

  • ユーザー
    1. 市場が成熟し、持続可能なスケーラブルなビジネスモデルを開発する前に、ユーザーを引き付けるには多すぎるコストがかかります。
    2. マーケティングおよび報道の努力による市場の利用不能に対する過度の補償。
  • 製品
    1. 適切な問題がない場合の製品（問題の解決策としての製品）の作成。
    2. 市場が成熟する前の製品スケーリングへの投資。
    3. クールな機能を追加します。
  • チーム
    1. 早すぎる従業員の雇用。
    2. 財務管理者、顧客サービス部門の従業員、データベーススペシャリストなど、非常に必要になる前にスペシャリストを雇用する
    3. 従業員の代わりに採用マネージャー（VP、製品マネージャーなど）。
    4. 複数の階層レベルがある。
  • ファイナンス
    1. 「死の谷」を克服するための投資が少なすぎる。
    2. あまりにも多くの投資を集めています。 これは必ずしも悪いことではありませんが、起業家の規律をほとんど常に奪い、他の分野で、例えば製品を過剰に開発したり、余分なスタッフを雇用したりして、時期尚早にスケーリングする機会を与えます。 また、投資が多すぎると、必要に応じて限られた額を割り当てたほうがよい投資家に大きなリスクをもたらします。
  • ビジネスモデル
    1. 早すぎる利益を得るためにあまりにも多くの努力。
    2. 過度の計画、必要なフィードバックなしの計画に従ったアクション。
    3. 変化する市場へのビジネスモデルの適応の欠如。
    4. ビジネスモデルに準拠できず、その結果、規模を拡大する際に、収益を収益よりも低く抑えることはできません。
