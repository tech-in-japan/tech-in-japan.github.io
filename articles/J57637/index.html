VMWareのディスクパフォ​​ーマンス：ホステス

VMWareのパフォーマンスに対する苦痛の結果を読者と共有したいと思います。 最初に結果、次に歌詞：



OS： Windows 2003サーバー

大量のディスク操作（サーバーアプリケーション）を使用するVMWare 6.5（おそらく他のバージョン）は、数時間アクティブに使用した後、激しく減速（速度が10倍低下）し始めます。 そして、これは断片化ではありません。



解決策：

.vmx仮想マシン記述ファイルに、次を追加します。



MemTrimRate = "0"

sched.mem.pshare.enable = "FALSE"

mainMem.useNamedFile = "FALSE"





MemTrimRateは、GUI、[オプション]-> [詳細]-> [メモリページのトリミングを無効にする]からも設定できます。



その後、すべてが期待どおりに動作し始めます（フライ:-)）。







プロセスに関するいくつかの言葉：

サーバー自体-NAT、DNS、DHCP

最初の仮想マシン-WSUSサーバー：Microsoftからの更新プログラムのみをダウンロードして、プロバイダーとの合意によりダウンロードできるインターネットアカウント。

2番目と3番目-他のネットワークへのプロキシサーバー（ネットワークが異なるため、すべてを1つのシステムに配置することはできません）。



最初のWSUSブートの開始から数時間後、パフォーマンスが著しく低下しました;仮想マシンの再起動には最大10〜15分かかりました。



予想していませんでした-ヘッドレス（ビデオカードなし）でVMWareがスローダウンするか、RDCを介してスローダウンし始めると思いました... 全体として、このような問題につながります。



PS。 はい、Windows 2003以降のサーバーを実行している場合、BIOSでエラーチェックがオフになり、サーバーが起動し、ビデオカードなしで正常に動作します。追加の設定は不要です。 RDCを介したGUIへのアクセスは残ります。 しかし、ビデオカードなしでOSをインストールすることはそれほど簡単ではありません:-)
