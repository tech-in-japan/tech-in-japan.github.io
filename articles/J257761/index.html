SAP HANA Cloud Platform：データのダウンロードと保存

前の記事で、クラウドプラットフォームのテストインスタンス（以下HCP- HANAクラウドプラットフォーム ）を作成する方法と、単純なXSアプリケーション（XS-eXtended Services）を使用してEclipse開発環境をクラウドに接続する方法について説明しました。



次に、さらに進んで、外部サービスからデータベースにデータをロードする方法を示します。 同時に、1つの試用システム内でのユーザー分離の要件に関連する、試用版クラウドを使用する際の微妙な点がいくつか示されます。



だから私たちが持っているもの。 1つ目は、 https：//account.hanatrial.ondemand.com/cockpitにあるクラウドのコントロールパネルへのアクセスです。



2番目は、 helloworldtrialと呼ばれるSAP HANA上のインスタンス化されたプラットフォームです。





3番目に、SAP HANA Cloudのプラグインとhelloworldtrialへの作成された接続を備えた、インストールされたEclipse開発環境：





やりたいこと：SAP HANAデータベースにテーブルを作成し、外部ソースからデータをロードします。 ここでは、いくつかの都市の現在の気温に関する情報を保存します。 たとえば、openweathermap.orgからデータを取得します。



新しいプロジェクトを作成することから始めましょう。 Eclipseを起動し、SAP HANA開発のパースペクティブを開きます（ ウィンドウ>パースペクティブを開く>その他... ）：





名前を指定して、新しいプロジェクトXSアプリケーションを作成します（[ファイル]> [新規]> [プロジェクト...]> [SAP HANA Cloud]）。





次に、ワークスペースとパッケージを選択するための標準的な手順に従う必要があります。 注意！ この例では、クラウドインスタンスに一致する、pXXXXXXXXtrial.helloworldtrialという形式のパッケージを選択することが重要です。 pXXXXXXXXtrialはクラウド上のユーザー名です。





そして、すべての作業を行うスクリプトget_weather.xsjsを使用してメインファイルを示します。





パッケージアクセス制御。 .Xsaccessファイル

.xsaccessファイルを使用して、XSアプリケーションのプロパティ（たとえば、スクリプトを実行するために必要な承認など）を指定します。 このファイルの内容を次のものに置き換えます。 

{ "exposed" : true, "default_file" : "get_weather.xsjs" }


公開されたプロパティは、このXSアプリケーションがURLからサーバー上で実行できることを示します。 default_fileプロパティは、アプリケーションのメインの実行可能ファイルを示します。 これにより、ファイル名を追加せずにコントロールパネル（コックピット）から直接起動できます。



ここで、アプリケーションをアクティブにして、この段階ですべての手順が正しく完了したことを確認します。 これを行うには、Project Explorerタブでプロジェクトを右クリックし、コンテキストメニューでTeam> Activateを選択します。



更新（F5）をクリックして、Webブラウザーのコントロールパネルを再起動します。 それ以外の場合、XSアプリケーションはリストに表示されません。





XSアプリケーションのURLをクリックします。





新しいウィンドウが開き、コンテンツ「間違ったコンテンツタイプのリクエストがアプリケーション/ jsonを使用します」が表示されます。 ワークピースが起動することを確認したら、次に進みます。



データベースにテーブルを作成する

データベースにテーブルを作成します。このテーブルには、日付と時刻、居住地の名前、現在の気温の列が含まれます。

システムにテーブルを作成するには、インタラクティブデザイナー、SQLスクリプト、または.hdbtableファイルを使用する方法がいくつかあります。



最速の方法は、スキーマの名前を置き換えてスクリプトを実行することです。 [システム]タブに移動し、クラウドのSQLコンソールを起動します。





NEO_スキームの名前を置き換えて、このようなスクリプトを実行します 。 

 create column table "NEO_<<<>>>"."T_WEATHER"( "ID" INTEGER GENERATED ALWAYS AS IDENTITY (MINVALUE 0 START WITH 0 INCREMENT BY 1), "FORECAST_TMSTMP" TIMESTAMP NOT NULL , "LOCATION" NVARCHAR(50) NOT NULL, "TEMPERATURE" REAL NOT NULL, "TIMESTMP" TIMESTAMP DEFAULT CURRENT_TIMESTAMP NOT NULL, PRIMARY KEY ("ID"));


列ストレージを持つテーブルを作成しているという事実に注目しましょう。 また、暗黙的にいわゆる シーケンス。整数のシーケンスを生成し、テーブルにレコードを作成するときにID属性に格納します。



NEO_ディレクトリを更新すると、テーブルが表示されます。





データベースユーザーの役割。 .Hdbroleファイル

無料試用システムでは、ユーザーは非常に限られた権限を持っています。 これは、特に、ディレクトリ内のオブジェクトを変更する許可とユーザーアカウントの管理に適用されます。 したがって、試用版システムには、「HCP」スキームに一連の手順があり、ユーザーが作成したオブジェクトに対する権限を管理できます。



T_WEATHERテーブルのデータを読み書きするには、新しいロールが必要です。 タイプ.hdbroleのファイルをプロジェクトに追加します（メインメニューの[ファイル ] > [新規 ] > [その他 ]で）。





weather_accessと呼びましょう：





このファイルの内容を次のフラグメントに置き換えてから、ファイルをアクティブにします。 

 role p<<<>>>trial.helloworldtrial.TryWeather::weather_access { catalog sql object "NEO_<<<>>>>"."T_WEATHER": SELECT, UPDATE, INSERT; }


p <<< >>>トライアルをユーザー名に置き換えます。 NEO _ << << >>>>スキームの代わりに、テーブルの作成時に保存したスキームの名前を挿入する必要があります。 そして今、魔法が始まります。これは、試用モードのクラウドが1つのシステムで多くのユーザーを作成するという事実に関連しています。 また、アクセス権を分離するために、これらのユーザーには非常に限られた特権があります。



システムユーザーがweather_accessのロールを取得するには、システム手順を実行する必要があります。

SQLコンソールを実行し、次のスクリプトを実行します： 

 call "HCP"."HCP_GRANT_ROLE_TO_USER"('p<<<>>>trial .helloworldtrial.TryWeather::weather_access', 'p<<<>>>');


試用システムの追加の組み込み手順
試用システムディレクトリのテーブル、ビュー、プロシージャを変更する場合、これらのオブジェクトにアクセスするためのアクセス許可を明示的に更新する必要があることに注意してください。 これは、プロシージャを呼び出すことで実行されます。 

 call "HCP"."HCP_GRANT_SELECT_ON_ACTIVATED_OBJECTS";


.hdbroleファイルを変更してアクティブにした後、以下を行う必要があります。 

 call "HCP"."HCP_GRANT_ACTIVATED_ROLES";




メインスクリプト。 .Xsjsファイル

したがって、データベースには空のXSスクリプトとテーブルがあり、データの書き込みと読み取りのアクセス許可を追加しました。 この表では、現在の天気に関するデータを書き込みます。

天気データはどこで入手できますか
温度を取得するには、openweathermap.orgのAPIを使用します。 このサービスの詳細は、Habréの記事で説明されています 。

モスクワで現在の天気データを受信するためのURLの例： http : //api.openweathermap.org/data/2.5/weather?q= Moscow, ru&units=metric

JSONオブジェクトの形式で受信したデータには、特に、Unixタイムスタンプ形式の気温に関する情報が含まれています。 タグ「dt」： 

 { ... "dt " : 1430318471, "id " : 524901, "name " : "Moscow ", "cod " : 200 }




天気データを抽出し、テーブルに書き込みます。

スクリプトソースコード
重要！ 最初の2行の定数値をユーザー名（-trialで終わる）とNEO_データベーススキーマの名前に置き換えます 。 





 var CONST_ROOT_PACKAGE = "<<<pXXXXXtrial>>>"; var CONST_NEO_SCHEMA = "<<<NEO_XXXXXXXXXXXXXXXX>>>"; function getCurrentWeather(city_name) { var destination_package = "helloworldtrial.TryWeather"; var destination_name = "o_weather"; var destPackagePath = CONST_ROOT_PACKAGE + "." + destination_package; var dest = $.net.http.readDestination(destPackagePath, destination_name); var client = new $.net.http.Client(); var req = new $.web.WebRequest($.net.http.GET, "?q=" + city_name + "&units=metric"); client.request(req, dest); var response = client.getResponse(); var weather_data = JSON.parse(response.body.asString(), function(key, value) { if (key === "dt") { return new Date(value * 1000); } return value; }); return [ weather_data.dt, city_name, weather_data.main.temp ]; } function storeData(data_receiving_time, city, temp) { var conn = $.db.getConnection(); var stmt = conn .prepareStatement("INSERT INTO \"" + CONST_NEO_SCHEMA + "\".T_WEATHER (FORECAST_TMSTMP,LOCATION,TEMPERATURE) VALUES (?,?,?)"); stmt.setTimestamp(1, data_receiving_time);//      OpenWeatherMap stmt.setString(2, city); stmt.setFloat(3, temp); var rs = stmt.executeQuery(); rs.close(); stmt.close(); conn.commit(); conn.close(); } function processRequest() { var cityNames = [ "Moscow,ru", "Saint Petersburg,ru", "Novosibirsk,ru", "Volgograd,ru", "Krasnoyarsk,ru" ]; var body = ""; var cityName, i; var dt; var city; var temp; var out_vals; try { for (i = 0; i < cityNames.length; i++) { cityName = cityNames[i]; out_vals = getCurrentWeather(cityName); dt = out_vals[0]; city = out_vals[1]; temp = out_vals[2]; storeData(dt, city, temp); body += " .\n: " + dt + "\n: " + city + "\n: " + temp + " C\n\n"; } $.response.setBody(body); $.response.contentType = "text/plain"; $.response.status = $.net.http.OK; } catch (e) { $.response.contentType = "text/plain"; $.response.status = $.net.http.INTERNAL_SERVER_ERROR; $.response.setBody(e.toString()); } } processRequest();




このコードは、いわゆる 外部APIを呼び出す宛先 。

また、Javascriptからデータベースへのアクセスは、$ .db名前空間の接続オブジェクトを介して行われます。 

 var conn = $.db.getConnection(); var stmt = conn.prepareStatement("INSERT INTO <<table>> (FORECAST_TMSTMP,LOCATION,TEMPERATURE) VALUES (?,?,?)");




外部URLへのアクセス。 .Xshttpdestファイル

o_weatherという名前の.xshttpdest型の新しいオブジェクトをプロジェクトに追加します。





そして、このファイルの内容を次のものに置き換えます。 

 host = "api.openweathermap.org"; port = 80; description = "current weather"; useSSL = false; pathPrefix = "/data/2.5/weather"; authType = none; useProxy = true; proxyHost = "proxy-trial"; proxyPort = 8080; timeout = 5000; // in milliseconds


このオブジェクトを保存してアクティブ化します。



これで、XSアプリケーションのタブのHCPコントロールパネルの[宛先]パネルに新しい要素が表示されます。





アプリケーションのURLをクリックして、スクリプトを実行し、およそ次の結果を画面に表示します。





これでEclipseで、データがテーブルに保存されたことを確認します。 これを行うには、T_WEATHERテーブルを右クリックして、[データプレビューを開く]を選択します。





そして、データベースに新しいレコードが作成されたことを確認します。





合計

したがって、この記事では、外部ソース（この場合はWebサービス）からデータを受信し、それをHANAデータベースのテーブルに保存できるXSアプリケーションを作成しました。 さらに、ロールの定義（weather_access.hdbrole）と外部Webサービスにアクセスするためのチャネルの定義（o_weather.xshttpdest）の2つの特別なオブジェクトを作成しました。 また、トライアルシステムのデータベーステーブルへのアクセス許可を与えるために、ストアドプロシージャを呼び出しました。
