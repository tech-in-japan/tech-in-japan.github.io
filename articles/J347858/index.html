PostgreSQLニュースダイジェスト。 第2号



最も興味深いPostgreSQLニュースを引き続き紹介します。 最初の問題はほぼ1か月前でした。 2番目の号では、1.5倍の資料を収集し、既存のセクション（Postgresとデータベース技術に関する書籍）に新しい資料を追加しました。

リリースとコミット

  • Cyber​​tecは 、 postgresql.confファイルを編集するためのビジュアルコンフィギュレーターであるpgconfigurator のリリースを発表しました 。 以下をカスタマイズできます。
    • 複製オプション
    • ブレークポイントに関連付けられたパラメーター
    • 入力/出力、並列リクエストの最適化。
    • メモリを構成します。
  • Devart は 、PostgreSQLデータベースとオブジェクトを操作するためのグラフィカル環境であるdbForge Studio for PostgreSQLを 提供しています。
  • JDBCバージョン42.2.0リリース

    新規：
    • JDBC 4.2（Java 8+）でのPostgreSQL 10のSCRAM-SHA-256サポート、
    • Ongres SCRAMライブラリを使用します。
    • SELECT INTOおよびCREATE TABLE ASがクライアントに行数を返すようになりました。
    • SSL接続のサブジェクトの別名サポート
    • PostgreSQL 10のID列のメタデータでのisAutoIncrementのサポート。
    • プリミティブの配列のサポート。
    • 接続でのget / setNetworkTimeout（）のサポートが登場しました。
    • Make GSS JAASでは、ログインはオプションになりました。 追加されたオプション「jaasLogin」
  • 2ndQuadrantは 、PostgreSQLの論理レプリケーションモジュール2.1のマイナーリリースであるpglogical 2.1.1 のリリースを発表しました 。 以前のバージョンのエラーは修正され、ログの記録時に情報が追加されます。
  • PgCodeKeeper 4.2.0がリリースされました 。 pgCodeKeeper-移行スクリプトを生成するためのツール。 現在、PostgreSQL 10をサポートしています。
  • Michel Pacierは、バージョン11に含まれるAlvaro Herreraのパッチが採用されたと書いています。
  • バージョン11のRobert Haasパッチが受け入れられました 。パーティションテーブルでUPDATEを実行すると、変更された行が別のセクションに分類される可能性があります（自由に配布されるpg_pathmanモジュールで、このトリガーが起動されました）。
