Mitap MSK.NETコミュニティ





みなさん、こんにちは！



2015年6月、 SPB.NETコミュニティの最初の会議が開催されました。 皆さんの多くは、そのようなコミュニティについて聞いただけでなく、SPB.NETミーティング（および多くの参加したミーティング）から素晴らしいスピーチを見たことがあると思います。 コミュニティは非常に精力的に発展し始め、モスクワから熱狂的なファンが好調を取り戻し 、 独自のローカルコミュニティ-MSK.NETを作成することにしました。



モスクワの会議は2016年8月から開催されており、すでに多くのファンがいます。 多くの開発者が例外なくすべての会議に参加しています。 もちろん、mitapsの主な目標は、.NET開発の分野での知識の交換です。 サブタスクとして、ファンを獲得したり、ピザを食べたり、クッキーと一緒にコーヒーを飲んだりすることができます。 さて、それらなしでどこに！







すべては 、 ニキータツカノフとミハイル シェルバコフ （SPB.NETのリーダー）が講演した会議から始まりました。

NikitaはAkka.NETの使用について話し、MichaelはWinDbgについて話しました。



これまでに、白熱した議論を含む7つの興味深い会議が既に開催されています（負傷者はいませんでした）。 ここで、すでに議論できたものを紹介します（最初の会議を除く）：



  • .NETアプリケーションのRESTFul API：方法、理由、理由

    Julia Tsisykは、RESTベースのAPIの作成、既存のベストプラクティス、避けるべきこと、設計する際に探すべきことについて説明します。






  • 10行のコードでWebアプリケーションでシングルサインオンを作成する方法

    Vyacheslav Mikhailovが、アプリケーションでの認証および承認メカニズムの選択方法、およびIdentityServerの使用経験について説明しています。






  • Cでのエラー処理＃

    Ilya Fofanovは、例外を使用することの利点と欠点、Resultモナド、および一般的なエラー処理の哲学について説明しています。






  • 例としてMassTransitを使用したメッセージングシステム。

    非同期メッセージングシステムのJacob Cook 。






  • デスクトップアプリケーションを最適化する手法。

    Prefetech、Superfetchなどを使用したさまざまな最適化に関するSergey Sentsov 。






  • .NET開発者にとってダンプの解析に役立つものは何ですか？

    Alexander Rakhmanovは、ダンプの収集プロセス、その分析と修正について、このプロセスをアプリケーション開発ライフサイクルに組み込んでいます。







  • 非ゲームアプリケーション用のHoloLens。

    Egor Bogatovは 、Unityを使用せず、ゲーム開発の経験もなしに、C＃のHoloLensメガネを使用して、C＃の非ゲーム2Dおよび3Dアプリケーション/ホログラムを開発する方法について説明します。






  • .NETパフォーマンスの状態

    .NET CoreおよびC＃7の最新のパフォーマンス変更に関するAdam Sitnik 。







  • .NETベンチマークの理論と実践。 （スライドのみがあります）

    Andrey Akinshinは、sabzhパズルを使った彼自身の企業のハードコアスタイルでのベンチマークの複雑さと知恵について語っています。






  • Autofacの例のIoC / DI。

    このコンテキストでのIoC / DIおよびIDisposableの詳細に関するIlya Efimov 。






  • モジュラーアプリケーションの設計と開発。

    キリル・モーリンは彼自身のオープンソースのマイクロライブラリを使用して、TRUEモジュラーアプリケーションを構築できます






  • 自動エラー検索。

    ウラジミール・コシェレフは、静的解析によるエラーの検索について説明しています。






会議は、 CUSTIS、Yandex、Kaspersky、Microsoft、Sbertech、Odin（Ingram Micro）で開催されました。 したがって、mitapを訪れることは、肝臓を食べ、楽しみ、経験を共有する機会であるだけでなく、誰がIT企業の1つとして生きているかを知る機会でもあります。 たとえば、Odinはオフィスの興味深いツアーを手配しました。これは開発者が本当に気に入ったものです。
