字幕についてのイグロデロ：理論と実践

字幕が便利に作られているかどうか、それは美しいですか？これはプレイヤーの体験に影響を与える可能性があります。 しかし、何らかの理由で、開発者は常に小さな（そして簡単に修正可能な）リストから同じレーキにつまずきます。 この記事では、ゲームと関連業界の両方でテストされた3つの最も重要な原則と10の良いヒントを思い出します。 同時に-ユーザーに関する情報を収集および処理する方法に関するいくつかの考え。



プレイヤーはしばしばスピーチの字幕と重要な音を使用します。 他の種類のメディアの消費者よりも頻繁に。 これには多くの理由があります-難聴だけでなく、貧弱なモバイルスピーカー、周囲の騒音、眠っている子供も。 また、重要なレプリカがゲームノイズで失われるリスク。 最後に、ゲームでは、しばしば字幕のみがローカライズされます。







そのため、CNet字幕アンケートで、 79％のプレイヤーが字幕で遊ぶと答えたのは驚くことではありません。 数字はかなり非科学的ですが、彼女はすでに彼らが字幕で頻繁に遊ぶと言っています。



そして、これにもかかわらず、字幕はしばしば適切な注意を払われていません。 字幕は多くの場合、出口の直前に添付され、インターフェースの不可欠な部分として機能しません。 優れた滴定システムは事前に行うのが最適で、表示されたらテキストで埋めます。



非常に基本的なことは、必要に応じて行う方が簡単です。実行しないと、多くの不満を感じるプレイヤーがいるでしょう。 これは、 最近のNeografの議論で説得力をもって示されています。 コメントを次に示します。

  • 適切なサイズの字幕を作成する人はほとんどいません！
  • ゲームの字幕は安っぽいです。 ただくだらない。
  • ゲームのほとんどの字幕はたわごとです。
  • 良い字幕は必須です。
  • 適切なフォントサイズで問題の半分を解決できます。
  • 字幕-ゴミ箱！
  • フォントとサイズを設定してほしい。
  • 標準化が必要です。
  • 設定は傷つきません。
  • さて、字幕のサイズを制御できれば。
  • 画面上部の字幕が嫌いです。
  • 多くのゲームでは、字幕を改善できます。
  • ひどい、小さすぎる。


基本



Neografからは、それほど多くの欠点がなく、ほとんどの部分が視覚的であることは明らかです。 これは、字幕を目立たなくする方法についての誤解に一部起因しています。



通常、字幕が邪魔に思える場合は、フォントサイズを小さくします。 しかし、これは逆の効果につながります。テキストが読めない場合、読むのに時間がかかり、目が字幕に費やされ、ゲームに費やされなくなり、字幕がより邪魔になります。



車輪を再発明する必要はありません。これらはすべて他のメディアで長い間決定されてきました。 これら3つのポイントは、Neografだけでなく、フォーラムやソーシャルネットワークでも絶えず不満があり、例外なくすべてのゲームで行う必要があります。



1.テキストと背景の適切なコントラスト





アサシンクリード：ブラックフラッグ。 複雑な明るい背景に明るい影付きの白い字幕。





ポータル2.プレートは、あらゆる背景にコントラストを加えます。



少なくとも目立つ黒いストロークが必要です。 文字の半分が背景から分離されないため、単純な影はお勧めしません。 ブラックダイを試すこともできます。 それほど明白ではなかったため、デフォルトでは半透明（50％）にすることができます。 理想的には、この数値はカスタマイズ可能である必要があります（以下の「ヒント」を参照）。



2.テキストの「シート」をダンプしないでください





X-Com。 たくさんのテキストがあり、あなたはそれを読まなければなりません。





証人。 字幕は短い行に分かれています。 これは目の隅からでも読み取ることができます。



テキストの大きなブロックはネタバレになる可能性があり、事前にイベントを提供します。 例：

こんにちはジョン、古い仲間！ * SHOT *なぜ、ジョン？


このフレーズは、次の3つの個別に分割する必要があります。

こんにちはジョン、古い仲間！

*ショット*

なぜ、ジョン？


大きなブロックも読みやすさを妨げます。 テキストが小さいほど、一目で簡単につかんでゲームに戻ることができます。



そのため、論理的な一時停止と句読点でできる限り分割して、テキストを小さな断片で配る方がはるかに優れています。



テレビでは、国とチャンネルの間で基準が設定されています。 国によって若干異なります。HDビデオの場合、1行あたり最大37（オーストラリア）〜40（フィンランド）文字、最大2行（例外的な場合は3）をお勧めします。 1つのフレーズが複数の行にまたがる場合、最初のフレーズが最も長くなりますが、それらの長さがほぼ等しくなるようにしてください。

3.十分な大きさのフォントを作成する





Sleeping Dogs：Definitive Edition。 テキストが小さいため、字幕が読みにくい。 たとえ誰かがそれらを読むことができたとしても、時間と労力がかかります。





バートラム・フィドル 大きくてクリアなテキスト、読みやすい。



Neogafに関する議論からわかるように、小さなテキストは一般的な問題です。 鼻の下の大きなスクリーンで十分に見えるという事実を無視しないでください-誰もがそれほどよく見えず、誰もが大きなスクリーンを持っているわけではなく、誰かが遠くに座っています。 「ソファから」テストしている場合でも、誰もが理想的な距離に移動できるわけではないことを忘れないでください。



他の業界では、解決済みの推奨事項があります。たとえば、「3メートルインターフェース」に関するAmazonの推奨事項は次のとおりです。可能な最小のテキストサイズは14ピクセル独立ピクセル（sp）、または1080pで28ピクセルです。 字幕は限られた時間だけ表示されるため、テキストはより大きくする必要があります。たとえば、チャンネル4（英国）では、すべてのHD字幕が46ピクセル（またはSDでは26）である必要があります。







これらの寸法は、コンソールのメーカーの1つが実施した未発表の研究と一致しています。



もちろん、ゲームには、他のメディアにはない別の制限があります-字幕は、ゲームインターフェースで画面上のスペースを奪い合います。 ただし、行の長さを38文字に制限する場合、これは問題になりません。



良いアドバイス



これらのガイドラインは、BBCおよび米国通信局によって設定された業界標準からまとめられています。 私たちのほとんどはゲームで時々見ますが、常にではなく、すべてを一緒に行うゲームはまだまれです。



そうすれば、字幕は業界に追いつき、その方法の例になります。 特にゲーム内で多くのチャタリングがある場合は、可能な限り、これらの字幕を検討する必要があります。



1.レプリカが言ったら、字幕を再確認する必要があります



時間とテキストのわずかな違いでさえも顕著です。 最新の状態を保つために、レプリカは短縮されます（以下の段落7を参照）。 俳優がギャグを言うこともあります。 したがって、ダビングが終了する前にテキストとローカライズが完了している場合、最終サウンドが到着したらすぐにテキストをもう一度調べてエラーをチェックする必要があります。



2.すべての重要なダイアログにパッチを適用し、必要になる前に字幕をオンにできることを確認します。



一般的な不満は、ゲームのどの部分にも字幕がないことです。 したがって、ビデオの挿入、ゲーム内のスクリプト化されたダイナミックオーディオ、および紹介ビデオを確認する必要があります。



しかし、同時に、導入が始まる前に何らかの形で字幕を含める必要があります。 悪名高い：First Lightは簡単な解決策を思いつきました。ボタンをクリックしてオンにします。



デフォルトで字幕を有効にしておくこともできます。 XboxOne / Android / iOS向けに開発している場合は、引き出してデフォルト値と見なすことができるシステム設定があります。



3.中央下に配置しますが、インターフェース要素にはオーバーレイしません



デフォルトでは、サブタイトルは画面中央下にあり、画面の下端からわずかにインデントされています。



この時点で一時的なインターフェース要素（QTEメッセージなど）が表示される場合、字幕は一時的に高くシフトされて重複しないようにします。要素が大きく、字幕が画面の上部に移動する場合は、最上部に配置します。 この例はテープボタンに適用されますが、QTEメッセージやゲーム内メニューなど、他のことにも同じことが言えます。



ただし、Xbox Oneの通知システムに依存して、メッセージを下部と中央に配置しないでください。 彼らはそこにいるべきではありません、そして私は彼らが将来それを修正することを望みます。



4.複数の字幕を重ねる必要がある場合は、下から新しい字幕を追加します



通常、字幕は1つずつ表示されます。1つは消え、2つ目は表示されます。 しかし、それらは同時に表示することができます-たとえば、頻繁に表示される場合。 この場合、新しいものが下から表示され、古いものが上に移動します。



5.誰が言うのかを知ろう



一部のゲームはキャラクターのポートレートに到達します。 しかし、彼らはこれを行うことはめったになく、通常は異なる色で行います。 誰が話しているのかわからない-群衆から、スピーカーから、電話で、フレームの後ろから-彼の名前を表示します。

ジョン：こんにちは！


スピーカーの色が異なる場合は、暗い輪郭またはダイとのコントラストが良好になるようにスピーカーを明るくします。 赤に特に注意してください;ほとんどの色盲の人には暗褐色に見えます。



6.音がどこから来るかを示します。



音が画面の後ろからである場合、どちら側に矢印を表示します：

←ジョン：こんにちは！


7.字幕を読む時間を与えます



行に38文字がある場合、1語で最短のレプリカでも1秒間画面に表示され、1行あたり2〜2.5秒長いことを確認します。



確かに、これを達成するためには、単語をトリミングする必要がある場合があります。 しかし、不一致は理解を妨げます。特に高速の読者にとってはそうです。 極端な場合にのみ切り捨てることができ、意味は保持され、単語は元の単語に非常に近いままでなければなりません。 だから、レプリカは「いやいやいや！ 今ではありません！」、「いいえ！」にカットできます 今ではありません！」、しかし、「いいえ、後で！」までBBCには、デリケートな状況に対処する方法に関する完全なガイドがあります。



子供にはもっと時間が必要です。 BBCは、13歳未満の子供には1行あたり3〜3.5秒、7歳未満の子供には4〜4.5秒を推奨しています。



8.明確で読みやすいフォントを使用する



ケースは普通です：サンセリフなしの読みやすい文字で「こんにちは」、「こんにちは」、「すべて」または「すべて」ではありません。 バックグラウンドノイズに大文字を指定できます。



字幕はグラフィックスのスタイルと組み合わせる必要がありますが、読みやすさを犠牲にしてはなりません。 読みやすさが損なわれると、字幕は基本的に正しく実行されません。 設定（次の項目）は、すべての利害の対立を適切に解決するのに役立ちます。



9.字幕をカスタマイズ可能にする



字幕の使用にはいくつかのシナリオがあります。 誰かがときどきそれらを見て、目立たないようにしたい。 誰かが完全に字幕に依存しています。 そのため、パラメーターの一部またはすべてが構成可能であることを確認してください。 理想的には、結果をリアルタイムで表示します。



  1. フォント：アーティストの選択/サンセリフ/ディスレクシアのための特別。
  2. フォントサイズ：小/中/大。
  3. 誰が言う：/名前/色を表示しないでください。
  4. エッジ効果：なし/ストローク/シャドウ/両方。
  5. ダイ：オン/オフ
  6. 透明性は死にます。


繰り返しになりますが、XboxOne / iOS / Android向けに開発している場合、これらすべてのシステム設定があり、APIを通じてそれらを拡張できます。これらはデフォルト値になります。



この程度のカスタマイズは落胆する可能性がありますが、既製のセットで簡単に解決できます。「控えめ」、「クリア」、「カスタム」、およびこれらすべての微調整。



これらの設定のいくつかでも価値があります。 それらの中で最も重要なのは、大きな/小さな活字とオン/オフサイコロです。 すでにこれら2つは多くの人を満足させます。



10.字幕ノイズ用に別のオプションを作成します



翻訳者から......
「クローズドキャプション」という用語の適切な翻訳が見つかりませんでした。意味内で翻訳しています。


「字幕」という言葉は通常、スピーチを指しますが、他の重要な音を滴定する価値があります。 ノイズ字幕は他のメディアでは一般的ですが、ゲームではまれです。 Portal 、 Left for Dead 、 Tomb Raider 、 Dragon Ageなど、いくつかの有名なゲームで使用されています。



一部のゲームでは簡単に追加されます。 他には、他のメディアにはない追加の複雑さがあります-ゲームサウンドの予測不可能な性質により、いくつかのノイズが同時に再生される可能性があります。 Dead to Rightsでは、このための優先順位を付けてキュー全体を配置しました。 各サウンドには優先順位があり、このような結果になります。



  1. キャラクター（優先度1）は、字幕が2秒間表示されると言います。
  2. 犬の鳴き声（優先度2）、画面にスペースがない、無視されます。
  3. 文字ごとに2秒（優先度1）が経過し、タイトルを削除します
  4. 突風（優先度2）、キャプションを表示します。


このようなシステムでは、最も重要な字幕のみが正確な時間に表示されます。重要な発言と重要なヒントのみがサウンドポリッジに表示されますが、すべてが落ち着いている場合、小さなノイズも滴定されます。
