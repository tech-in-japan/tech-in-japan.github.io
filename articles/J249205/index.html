SSD SASとSATA。 何を選ぶ？

こんにちは、ハブラリュディ！



HGSTのブログが休憩の後に戻ってきました。 そして今日、SATAドライブに対するSAS SSDの利点についてお話したいと思います。



デバイス間の通信をサポートするSASインターフェースは、企業向けに設計されており、スケーラビリティ、信頼性、高いデータ可用性を提供します。一方、SATAデバイスは、より安価なユーザーアプリケーション向けに最適化されています。



ディスクメーカーは高性能ドライブにSASインターフェースを使用し、クライアントドライブと大容量記憶装置にSATAインターフェースを使用しているため、ソリッドステートドライブ（SSD）メーカーはほぼ同じパーティション分割を使用し続けています。 市場には、高性能を提供するSATAインターフェイスを備えたエンタープライズクラスのSATAドライブもあります。 ただし、よりエラー耐性の高いフラッシュデバイス、コントローラー、ファームウェアを備えたSASインターフェイスを使用すると、オンライントランザクション処理（OLTP）、高性能コンピューティング（HPC）、データベースアクセラレーションなどのエンタープライズレベルのワークロード向けの優れたソリューションが得られますデータストレージ組織/データロギング、仮想化および仮想PCインフラストラクチャ、大量のデータおよびハイパースケールデータの操作、メッセージングおよびコラボレーション、インターフェイス Webサーバ、マルチメディアの伝送ストリームとNASとSANネットワークのためのビデオ・オン・デマンド（VOD）、クラウドコンピューティングおよびデータ記憶装置ティア0を提供します。



SASインターフェース機能と、CellCare、PowerSafe、Data Path Protectionなどの業界をリードするHGSTテクノロジーにより、次の利点が得られます。



•ライフサイクル全体にわたる安定した高性能SSDパフォーマンス

エンタープライズクラスのSASドライブがサポートする必要があるワークロードには、次のものがあります。

•オンライントランザクション処理（OLTP）

•大量のデータとスケーラブルなデータの分析

•メッセージングおよびコラボレーションソフトウェア

•ストリーミングメディアとビデオオンデマンド（VOD）

•クラウドコンピューティング

•SANおよびNASシステム用のTier-0ストレージデバイス



SAS（シリアルSCSI）およびSATA（シリアルATA）は、接続されたデバイス間でデータを転送するための標準プロトコルです。 これらは、コンピューターが外部メモリコントローラーやハードドライブなどの周辺機器と対話できるように設計されています。 両方のインターフェイス（SASとSATA）には長い開発の歴史があります。1980年代に初めてパラレルインターフェイスとして登場し、約10年前にパフォーマンスをさらに向上させるためにシリアルプロトコルに変換されました。 外部メモリコントローラーと併用する場合、SASまたはSATAインターフェイスは、ハードドライブとSSDを接続するための内部インターフェイスだけでなく、外部サーバーインターフェイスとしても使用できます。 コントローラは多くの種類のインターフェイスをサポートできますが、ディスクにはSASまたはSATAの1種類のインターフェイスしかありません。 インターフェイスは、情報ストレージデバイス（フラッシュメモリ、ハードディスクなど）、またはディスク内のコンポーネントまたはファームウェアの品質に依存しません。 この観点から、SASおよびSATAインターフェースは同じように動作します。



ドライブの基本的なパラメーターを見てみましょう



性能

•SCSIプロトコル。 SASインターフェイスで使用されるSCSIプロトコルは、ATAパラレルインターフェイスコマンドセット（SATA）よりも高速で、複数の同時データI / O操作をより効率的に実行します。

•データ転送速度の向上-6 Gb / sから12 Gb / s、そして最大24 Gb / s。 SASインターフェースを使用すると、データ転送速度を6 Gb / sから12 Gb / sに上げることができます。 さらに、最大24 Gb / sの速度をさらに上げるための明確なロードマップがあります。 現在、SATAインターフェースは最大6 Gb / sのデータ転送速度をサポートしていますが、将来速度を上げる具体的な計画はありません。

•タグ付けされたチームのキュー。 ほとんどのSASドライブは、128ビットの深さのコマンドキュー（プロトコル制限65,536）をサポートしています。これにより、待ち時間が短縮され、ワー​​クロードが高い場合のパフォーマンスが向上します。 SATAインターフェイスのコマンドシーケンスのハードウェア設定は、32コマンドのみをサポートします。

•デュアルポートとマルチチャネル入出力。 SASインターフェースを備えたドライブにはデュアルポートが装備されており、ストレージシステム内の多くのイニシエーターをサポートしています。 したがって、マルチチャネルI / Oと負荷分散により生産性が向上します。 SATAインターフェースは複数のイニシエーターをサポートしておらず、ほとんどのSATAドライブにはデュアルポートがありません。

•全二重データ転送。 SASドライブは全二重モード（2方向の同時データ転送）をサポートし、SATAドライブは半二重モード（一方向のデータ転送）で動作します。



拡張性

•複数のドライブを1つのポートに接続できます。 SASインターフェースは、最大255台のデバイス（2層構造）のポートエクスパンダーをサポートしているため、最大65,635台のディスクを1つのイニシエーターポートに接続できます。 SATAインターフェースは、ポイントツーポイント接続のみを使用します。

•延長ケーブルの使用。 SASデバイスを使用すると、最大10 mのパッシブ銅ケーブルと最大100 mの光ケーブルを使用できるため、データセンター（データセンター）を拡張するより便利なプロセスが提供されます。SATAでは、2 mを超えるケーブルを使用できません。

•スケーラブルなパフォーマンス。 RAID構成のソリッドステートSASドライブのパフォーマンスは、SATAドライブよりもスケーラブルです。

•SATAインターフェイスとの互換性。 SASインターフェイスを備えた外部メモリコントローラーは、SATAドライブをサポートします。SATAドライブは、単一アレイ内のSASドライブとSATAドライブの両方を使用して階層型データストレージを提供します。 ただし、SATAはSASドライブをサポートしていません。



高いデータ可用性

•フォールトトレランス用のデュアルポート。 SASはデュアルポートをサポートしていますが、ほとんどのSATAドライブはサポートしていません。

•複数のイニシエーター。 SASインターフェイスを使用すると、複数のコントローラーをストレージシステム内のハードドライブセットに接続できるため、障害発生時に迅速に交換し、別のリソースに移行できます。 SATAインターフェースにはこのような機能はありません。

•「ホット」モードでの接続。 SASおよびSATAドライブは、ホットスワップモードで接続できます。



合理化されたシステムアーキテクチャとの対話

•将来の拡張のためのロードマップ。 SASインターフェースを備えたデバイスのメーカーの計画には、データ転送速度を最大24 Gb / s、おそらくさらに高速にすることが含まれていますが、SATAにはそのようなロードマップはなく、データ転送速度は6 Gb / sの現在の値によって制限されます。 SASを使用することにより、企業は既存のインフラストラクチャで使用されている以前のバージョンとの互換性を維持しながら、将来的にデバイス群をアップグレードし、より高速なドライブにアップグレードできます。

•SCSI。 企業にインストールされているほとんどのドライブはSCSI命令セットを使用しているため、SASインターフェイスはストレージシステムの世代と互換性があります。



HGST SSDは、ドライブの寿命にわたって高いパフォーマンスを実現します。 彼らは革新的な高度なフラッシュ管理とCellCareテクノロジーを使用し、シーケンシャルおよびランダム読み取り/書き込みモードで非常に高速を提供します。 SSDはハードドライブと比較して非常に高速ですが、特にプログラムのインストール/ファイル削除サイクルの増加に伴い、フラッシュメモリセルが摩耗し、速度が低下します。 HGSTの高度なフラッシュ管理テクノロジーは、SSDの寿命、信頼性、パフォーマンスを向上させるために、従来のウェアレベリングアルゴリズム、エラー検出および修正スキーム、損傷ユニットの復旧、データの冗長性の排除を使用します。



HGST CellCareは、フラッシュメモリコントローラーを製造するための特許取得済みの技術であり、フラッシュメモリを備えたデバイス向けに高密度の要素を備えたコスト効率の高いロジックチップを使用して、エンタープライズクラスのデバイスの耐久性、パフォーマンス、信頼性を保証します。 CellCareテクノロジーは、メモリセルパラメーターの消耗に応じて動的に追跡し、予測テクノロジーを使用して、フラッシュメモリとコントローラー間の適応フィードバックを作成することにより、NANDフラッシュメモリチップの消耗を最小限に抑えます。 Cellcareテクノロジーの同様に重要な側面は、フラッシュメモリのエージング効果を制御し、SSDの寿命が長くなるにつれて速度が低下するのを防ぐ機能です。 Cellcareのユニークなテクノロジーのこの機能により、HGST SSDの耐用期間中、トラブルのない操作と高いパフォーマンスが保証されます。
