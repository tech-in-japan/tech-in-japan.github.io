Qt 4.7でMaemo5をアップデートする可能性はありますか？

これまでのところ、これは非常に興味深いと有望に聞こえます。 しかし、説明する前に、ノキアは多くの場合多くのことを約束しましたが、約束された内容を正確に受け取ったわけではありません。 ところで、 記事にはこう書かれています：



「「可能性」は、それが起こるかもしれないし、起こらないかもしれないことを意味し、私たちは、いつ、いつ、手掛かりを持っていない」

「可能性（更新）とは、それが起こる可能性があるかどうかということです。そうなるかどうか、そうなるとしたら、いつになるかはわかりません」（無料翻訳）



更新自体の作業は進行中ですが、上記のように、彼らはそれを可能性と呼びます。 彼らが言うように、多くの特定のバグ修正がありますが、maimoは4.7-fremantle（4.7はkittyのバージョンに関するものであり、fremantleはmaimo5のコード名です）に切り替えるため、新しい機能もあります。 N900はマルチタッチをサポートしていないため、Qt 4.6ではデフォルトでQGestureを無効にしただけでしたが、4.7ではシングルタッチスクリーンのジェスチャサポートを追加しました。たとえば、押してから長押しします（記事から理解したように）両方のアクションは1つと見なされます。それ以外の場合、通常の長押しとプッシュアンドホールドは長い間別々に機能していたため、これには新しいものはありません。 そして彼らが約束するように、それは「友好的なUIをより友好的にする」（=によって意味される））。 また、Qt MobilityのBearer Managementの革新（ TMOで既に表面化されているため、次の更新でQt MobilityをMaemo5に導入する必要があります）の説明によると、MaemoInternetConnectivity :: setAutoConnectEnabled（） 。 それに加えて、Qt 4.7の多くの新機能はN900で動作します。特に（強調しているように）QtDeclarativeは興味深いものです（ ここにいくつかのビデオと英語の説明があります）



実験的なQt 4.7パッケージは、extras-develリポジトリにあります。 Scratchbox SDKユーザーは、 fakerootを実行し、 apt-get install libqt4-experimental-devを実行することでインストールできます。

すべてのバグはbugreports.qt.nokia.comに送信されます。



もちろん、私は更新を待っていて満足していますが、N900の独立したプロジェクトが好きです。たとえば、Androidは毎日より良く機能し、鉄の箱（スピーカー、ヘッドフォン、青い歯など）からサポートが追加されています。 また、N900用のKDE Mobile Plasmaを長い間見つけて、これを行った人と連絡を取ろうとしましたが、どうやら消えたようです。先日、N900のプラズマの作業に関するニュースを読みました（リンクにビデオがあります）。
