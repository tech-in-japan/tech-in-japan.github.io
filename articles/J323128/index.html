Web開発者向けのブロックチェーンの説明

Habrahabrの読者に、 フランソワ・ザニノットによる記事「Web開発者に説明されたブロックチェーン、パート1：理論」の翻訳を提供します。 この記事はMarmelabブログで見つけました。 この記事は、ブロックチェーン技術をゼロから紹介する優れた入門書ですが、すでに知識を持っている人にとっては興味深いものになります。 ブロックチェーンがどのように機能するかだけでなく、その開発の見通し、およびブロックチェーンを使用して独自のプロジェクトを作成する場合の開始点も関係します。



Web開発者向けブロックチェーンの説明、パート1：理論



ブロックチェーンは新しい人気のある技術です。 聞いたことがないとしても、おそらくビットコインについて知っているでしょう。 ブロックチェーンは、ビットコインの基盤となる基本的なテクノロジーの1つです。 専門家は、ブロックチェーンがインターネットがかつて引き起こしたものと同様の革命を引き起こすと言います。 しかし、それは本当に何であり、どのようにそれを使用してアプリケーションを作成できますか？ この記事は、ブロックチェーン現象について語る3回シリーズの最初の記事です。 理論について議論し、実際のコードを示し、実際のプロジェクトに基づいて経験を共有します。



まず、ブロックチェーンとは何かを理解してみましょう。



ブロックチェーンとは、パート1



ブロックチェーンはビットコインをサポートするために作成されたという事実にもかかわらず、ブロックチェーンの概念はビットコインのエコシステムとは独立して決定できます。 文献では通常、ブロックチェーンを次のように定義しています。

ブロックチェーンは、ピアツーピアネットワーク（P2P）内の複数のコンピューターに複製されるファクトジャーナルです。 事実は、金銭取引からコンテンツへの署名まで何でもかまいません。 ネットワークメンバーは、 ノードと呼ばれる匿名の個人です。 ネットワーク内のすべての通信は暗号化を使用して、送信者と受信者を確実に識別します。 ノードがジャーナルにファクトを追加する場合、ネットワーク上でコンセンサスが形成され、このファクトがジャーナルのどこに表示されるかが決定されます。 このコンセンサスはブロックと呼ばれます 。


あなたについては知りませんが、これらの定義を読んだ後、ブロックチェーンを理解するのにまだ問題がありました。 もう少し詳しく見てみましょう。



注文の事実



分散型のピアツーピアネットワークは新しいものではありません。 NapsterとBitTorrentはP2Pネットワークです。 映画を共有する代わりに、ブロックチェーンネットワークの参加者は事実を交換します。 では、ブロックチェーンの本当の特徴は何ですか？



P2Pネットワークは、他の分散システムと同様に、非常に複雑なコンピューターサイエンスの問題を解決する必要があります。つまり、競合の解決または調整です。 リレーショナルデータベースは参照整合性を提供しますが、分散システムにはそのような機能はありません。 互換性のない2つのファクトが同時に到着した場合、システムには、どちらのファクトが正しいと見なされるかを決定するルールが必要です。



たとえば、 二重支出の問題を考えてみましょう。アリスは10ドルで、ボブとチャーリーに2回送ります。 合計で10ドルを持っているのは誰ですか？ この質問に答えるための最善の方法は、事実を合理化することです。 互換性のない2つのファクトがネットワークに表示される場合、最初に記録されるファクトが勝ちます。







P2Pネットワークでは、ほぼ同時に送信された2つのファクトがリモートサイトに異なる順序で到着する可能性があります。 それでは、ネットワーク全体をどのように調整するのでしょうか？ P2Pネットワークの整合性を確保するには、事実を調整する方法が必要です。 コンセンサスシステムが必要です。



分散システムのためのコンセンサスアルゴリズムは研究のための非常に活発な分野です 。 PaxosまたはRaftアルゴリズムについて聞いたことがあるかもしれません。 ブロックチェーンは、ブロックを使用して、作業の証明に基づくコンセンサスという別のアルゴリズムを実装します。



ブロック



ブロックは、信頼できないノードで事実をオンラインで合理化するための巧妙なトリックです。 アイデアは単純です。ファクトはブロックにグループ化され、ネットワーク全体に複製されるブロックのチェーンは1つだけです。 各ブロックは前のブロックを参照します。 つまり、ファクトFがブロック21にあり、ファクトEがブロック22にある場合、ファクトEは次のファクトFとしてネットワーク全体で考慮されます。ブロックに追加される前に、ファクトは考慮中です。 確認されていません。







マイニング



チェーン内の一部のノードは、未確認のファクトを持つ新しいローカルブロックを作成します。 彼らは、サイコロを投げることによって、ローカルブロックがネットワーク全体のチェーンの次のブロックになるかどうかを競います。 ノードが2つの6をスローすると、そのローカルブロックを公開する機会が得られ、このブロック内のすべてのファクトが確認されます。 このブロックは、ネットワーク内のすべてのノードに送信されます。 すべてのノードは、ブロックが正しいことを確認し、回路のコピーに追加して、新しい未確認のファクトで新しいブロックを構築しようとします。







しかし、実際には、ノードはサイコロを投げるだけではありません。 マイナーがブロックチェーンで解決するタスクには、膨大な数のサイコロを投げることが含まれます。 設計上、ブロックを検証するためのランダムキーの発見はほとんどありません 。 これにより、不正行為が防止され、ネットワークが安全になります（攻撃者がネットワーク内のノードの半分以上を制御できない限り）。 その結果、新しいブロックが一定の時間間隔で回路に公開されます。 ビットコインでは、ブロックは平均で10分ごとに公開されます。



ビットコインでは、タスクは未検証のファクトのストリング、前のブロックの識別子、ランダムなストリングのダブルSHA-256ハッシュです。 ハッシュに少なくともn個の先行ゼロが含まれている場合、ノードが勝ちます。



// Bitcoin

787308540121f4afd2ff5179898934291105772495275df35f00cc5e44db42dd

// Bitcoin, n=10

00000000009f766c17c736169f79cb0c65dd6e07244e9468bc60cde9538b551e







数nは、ノードの数が変わっても、ブロックの計算時間を保持するために時々調整されます。 この数は複雑さと呼ばれます 。 他のブロックチェーン実装では、計算にGPUを使用できないようにする特別なハッシュメソッドを使用します（たとえば、大量のメモリの転送が必要です）。




ブロックを見つけるプロセスは、 マイニングと呼ばれます。 問題は、金の採掘（およそTransl。Mining-マイニング）と同様に、ブロックマイニングは経済的な見返り-何らかの形のお金をもたらすということです。 ブロックチェーンにノードを含む人々がマイナーと呼ばれるのはそのためです。



注：デフォルトでは、ノードはマイニングしません。他のノードをマイニングするブロックを受信するだけです。 これは任意のプロセスです。ノードをマイナーノードに変換します。



お金と暗号通貨



毎秒、すべてのマイナーが数千のランダムな文字列をテストして新しいブロックを形成します。 つまり、ブロックチェーンのマイナーの作業には、膨大な量のコンピューターリソース（メモリとCPU）が必要です。 だからこそ、ブロックチェーンに事実を書くためにお金を払わなければなりません 。 一方、ファクトの読み取りは無料です。自分のサイトを開始するだけで、他のサイトで作成されたファクトの完全な履歴を取得できます。 要約すると：



  • 無料でデータを読む
  • 事実の追加には少額の手数料がかかります。
  • ブロックマイニングは、ブロックに含まれるすべての事実に対するコミッションの金額でお金をもたらします


これは本物のお金に関するものではありません。 実際、各ブロックチェーンには独自の（暗号）通貨があります。 ビットコインネットワークではビットコイン（ BTC ）、イーサリアムネットワークではエーテル（ ETH ）などと呼ばれます。 Bitcoinネットワークで支払いを行うには、銀行に手数料を支払う必要があるのと同じように、ビットコインで小額の手数料を支払う必要があります。 しかし、最初のお金はどこから来たのですか？



マイナーは、ネットワークのパフォーマンスとセキュリティを維持することで報酬を受け取ります。 ブロックを正常に形成するたびに、一定量の暗号通貨を受け取ります。 ビットコインでは、報酬は1ブロックあたり25 BTC、イーサリアムでは1ブロックあたり5 ETHです。 したがって、ブロックチェーン自体が独自のお金を生成します 。



暗号通貨はすぐに本物のお金に変換可能になりました。 それらの名目値は需要と供給によってのみ決定されるため、暗号通貨は推測の対象です。 執筆時点では、ビットコインマイニングプロセスで必要なエネルギーと機器のコストは、受け取ったコインを販売して得られるお金よりも少なくなります。 だからこそ、人々は毎日新しい鉱山労働者を追加し、電気をお金に変えたいと思っています。 ただし、BTCのコストの変動により、マイニングの収益性は低下します。







契約



ここまでは、主に事実のリポジトリとしてのブロックチェーンについて説明しましたが、 プログラムを実行することもできます 。 一部のブロックチェーンでは、各ファクトにミニプログラムを含めることができます。 そのようなプログラムはファクトで複製され、各ノードはそれらを実行し、ファクトを受け取ります。 ビットコインでは、これは条件付きでトランザクションを完了するために使用されます 。たとえば、ボブは、今日が2月29日である場合にのみ、アリスから100 BTCを受け取ります。



他のブロックチェーンは、より複雑な契約を可能にします。 たとえば、イーサリアムでは、各コントラクトがミニデータベースを保持し、そのデータを変更する方法を提供します。 コントラクトはすべてのノードにわたって複製されるため、それらのデータベースも複製されます。 ユーザーがコントラクトからメソッドを呼び出し、それに応じてデータを変更するたびに、このコマンドが複製され、ネットワーク全体で繰り返されます。 これにより、約束を果たすための分散コンセンサスが可能になります。



事前にプログラムされた条件を使用してブロックチェーンを実世界とペアリングし、それらをすべてのノードに転送するこのアイデアは、 スマートコントラクトと呼ばれます。 契約は、当事者が合法的にそれを確保するために署名するという約束です。 スマートコントラクトも同じで、統合は「法的に」ではなく「技術的に」行われます。 これにより、公証人または両当事者によって承認されたその他の許可された人物の必要性がなくなります。



50％の前払いで1000ドルで1週間家を借りたいと想像してください。 あなたとテナントは契約書に署名しますが、ほとんどの場合は弁護士によって書かれています。 支払いを受けるには銀行も必要です。 週の初めに、5000ドルのデポジットを要求します。 テナントがチェックを提供します。 週の終わりに、彼は残りの50％の支払いを拒否します。 また、彼がウィンドウを壊し、預金の小切手が空の口座につながることもわかります。 今、あなたはあなたの賃貸契約を法廷に持ち込むために弁護士が必要です。



ブロックチェーンのスマート契約により、銀行、弁護士、弁護士、裁判所なしで行うことができます。 特定の条件の場合に送金する金額を決定するプログラムを作成するだけです。



  • 賃貸の2週間前：テナントから所有者への500ドルの送金
  • 所有者によるキャンセル：所有者からテナントへの500ドルの譲渡。
  • レンタル期間の終了：テナントから所有者への500ドルの譲渡
  • レンタル期間後の機械的損傷の証明：テナントから所有者への5000ドルの転送


このスマートコントラクトをブロックチェーンに追加すれば、問題はもうありません。 契約で指定された時間までに、お金は送金され、所有者が機械的損傷の証拠を提供できる場合、彼は自動的に5000ドルを受け取ります（そして預金の必要はありません）。



おそらく、機械的損傷の証拠をどのように取得するのか疑問に思うでしょう。 これがモノのインターネット （IoT）の出番です。 現実世界と対話するには、ブロックチェーンにセンサーとドライブが必要です。 IoT革命がなければ、ブロックチェーンの革命は不可能です。



スマートコントラクトに基づくこのようなアプリケーションは、 分散アプリケーションまたはDAppと呼ばれます 。



スマートコントラクトは、 スマートプロパティや他の多くのスマートなものに簡単に拡張できます。 覚えておく必要があるのは、「スマート」とは「仲介者なし」または「技術的に実行された」ということです。 ブロックチェーンは、インターネットで音楽を販売するのと同じように、仲介者なしでビジネスを行う新しい方法です。







ブロックチェーンとは何ですか？ パート2



ブロックチェーンを理解する最良の方法は、さまざまな角度からそれを見ることだと思います。



彼は何をしていますか？ ブロックチェーンを使用すると、信頼できないネットワークを介して複数のユーザー間でデータを安全に配布および/または処理できます。 データは何でも構いませんが、データの最も興味深いオプションは、信頼できる第三者を必要とする情報を送信する機能です。 そのような情報の例は、お金（銀行の参加が必要）、財産権（公証人の参加が必要）、ローン契約などです。 本質的に、ブロックチェーンは第三者が参加する必要を排除します。



彼はどのように働いていますか？ 技術的な観点から見ると、ブロックチェーンは、P2Pネットワーク、非対称暗号化、数学的問題の解決に基づく分散コンセンサスという3つの概念に依存する革新です。 これらのアイデア自体は新しいものではありません。 すべてを理解していなくても心配しないでください。ブロックチェーンを開発するのに必要な知識を持っている人はほとんどいません（これは問題です）。 ただし、TCPまたは認証局のスロースタートを知らなくてもWebアプリケーションを作成できるのと同様に、ブロックチェーンを完全に理解していなくても、ブロックチェーンの使用を止めることはできません。



彼はどのように見えますか？ ブロックチェーンは、ネットワーク内のノードと同じ回数だけ複製される（弱い）同期データベース、またはそれに含まれるすべてのCPU / GPUノードの複合体によって形成されるスーパーコンピューターと見なすことができます。 このスーパーコンピューターを使用して、データを保存および処理できます。 また、リモートAPIを使用できます。 唯一の違いは、バックエンドを作成する必要がなく、データがネットワークで適切に保護および処理されることを確認できることです。



実用的な意味



ブロックチェーンに保存されているファクトは失われません。 それらは永遠にそこにとどまり、すべてのノードに複製します。 さらに、ブロックチェーンは最終状態を保存するだけでなく、以前のすべての状態も保存します。 したがって、誰もが最終状態の正しさを確認でき、最初から事実を再集計できます。



ブロックチェーンの事実を信頼できます。なぜなら それらはコンセンサスによって技術的に確認されています。 ネットワークに侵入者がいたとしても、一般的に彼女の判断を信頼できます。



データをブロックチェーンに配置することは、分散コンセンサスの達成を必要とするため、かなり遅い操作です。

ブロックチェーンを作成したくないでしょう



ブロックチェーンが構築されるテクノロジーは、高度な暗号化、ユーザーネットワークプロトコル、パフォーマンス最適化を使用します。 これはすべて、プロジェクトがブロックチェーンを必要とするたびに開発するには複雑すぎます。 幸いなことに、ビットコインに加えて、さらにいくつかのオープンソースのブロックチェーン実装があります。 最も高度なものは次のとおりです。



  • Ethereum ：Ethereum Foundationのオープンソースブロックチェーンプラットフォーム
  • Hyperledger ：別のオープンソース実装で、Linux Foundationのみが提供しています。 最初の実装が最近公開されました 。
  • Eris Industries：主にプライベートネットワークを作成するために、イーサリアム、ビットコイン、または完全に独立したブロックチェーンを使用するためのツール。 彼らの指示とガイドは、ブロックチェーンのレビューの素晴らしい出発点です。


これらの実装の複雑さは大きく異なります。 今すぐアプリケーションを作成する必要がある場合は、以下をお勧めします。

翻訳者のあとがき
この記事と他のいくつかのソースを翻訳して、ブロックチェーンがどのように機能するかを理解しました。 そして、この一連の記事は、おそらくインターネット上のブロックチェーンで見つけた最高のものです。 しかしそれでも、この記事は明白な質問に答えていません：「複雑な数学的問題の解決はブロックの同時発行の可能性を減らしますが、複数のノードが同時にそれを解決したらどうなりますか？」 答えは簡単です。ブロックチェーンは分岐するため、ブロックチェーン内のこのような状況には、「最も長いブランチを選択する」という特別なルールがあります。 複数のノードが同時に1つの問題を解決する可能性がありますが、それらが連続して2回以上それを行う確率は大幅に低下するため、遅かれ早かれ、最長の分岐が表示され、ブロックチェーンに配置されます。 ブロックチェーンはすぐに安定しています。 これとトランザクションセキュリティへの分岐の影響については、既に提供されているビデオから詳しく知ることができます。
