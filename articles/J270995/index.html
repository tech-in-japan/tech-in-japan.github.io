Outlookアドイン、または作業時間の25％がどこに行き、それを戻すことができますか？

発行



いくつかの外国の研究の結果に基づいて、平均的なオフィスワーカーは 1日あたり2時間以上の仕事時間を電子メールで処理していることをご存知でしたか？ もっと正確に言えば、週に13時間 、これはほぼ85営業日です！ これらの数字について考えてください。 この時間はどれくらいできますか？ また、時間をお金に換算すると、組織にどのくらいの費用がかかりますか？ 考えるべき非常に興味深い質問...











スライドはSlideshareのプレゼンテーションから取得されます。



電子メールクライアントの世界では、Outlookは主要な地位の1つになります。 さらに、Officeは10億人以上の人々に使用されており、企業セグメントの主要なツールです。











ほとんどの人が当社でそれを使用しています。つまり、従業員が勤務時間の25％を費やすのはOutlookであるということです。 もう少し掘り下げて、Outlookに戻ってメールをチェックするときの正確な時間を考えてみましょう。



ほんの数人の時間を食べる人を考慮してください：



  1. 私たちは手紙を読みます。 誰かが速く、誰かが遅くなりますが、誰もが読む必要があります。私たちはこれをさらに続けます。
  2. 質問される場合がありますが、質問によっては、カレンダーの空き時間の明確化、ドキュメントの検索、オンラインマップ上で必要な場所の場所の決定、誰かへの電話、会議室の予約などに時間を費やすことができます。 ;
  3. 電子文書管理システムに保存する必要がある文書を送信できます。
  4. リマインダーレターからサードパーティのシステムにアクセスし、実際のリマインダーレターに記入するなど、日々の労働報告書に記入するなどの時間を費やしています。
  5. 手紙に応じます。


貴重な時間を要する無数のタスクをリストできます。 しかし、もっとよく考えてみましょう。 生産性を向上させ、執筆に費やした時間の25％の少なくとも一部を返すことができますか？ Outlookはこれを支援できますか？ 純粋な形で-実際にはそうではありませんが、 Outlookはアドインを使用して拡張できます！ 今日はこれについてお話します。



Outlookのアドインはどのシナリオで生産性を向上できますか？



考えられるすべてのシナリオを3つの主要なグループに分けます。



  1. 「情報を迅速に把握する」

    • 手紙に住所があります-ワンクリックで地図を見たいです。
    • 手紙に外国語があります-ワンクリックで翻訳を見たいです。 など


  2. 「迅速に回答」

    • 私は常に繰り返し文を書かなければなりません-それらをリストに整理し、それらを手紙にすばやく挿入できるようにしたいです。
    • 電子文書管理システムからレターに文書を添付する必要があります-すぐに文書を検索し、Outlookを終了せずに適切な文書を選択します。 など


  3. 「もっと早く」

    • サードパーティのシステムからリマインダーを受け取りました-システムに直接アクセスして入力する必要なく、必要なアクションを実行したい、つまり Outlookインターフェースから直接。
    • ドキュメントを調整する必要があります-ドキュメントを表示し、レターから直接「同意する」をクリックできるようにしたい。
    • 彼らは新しい契約を送った-私はそれを数回クリックするだけで企業の電子文書管理システムに転送したい。
    • 予約する-手紙から直接指定された時間にタクシーを予約したい。 など


私たちのほとんどはこのような「ウィッシュリスト」を常に持っていますが、最近ではOutlook用のJavaScript APIとOffice 365 APIを使用してそれらをすべて実装できます 。



JavaSctipt APIを使用したOutlook用アドイン開発の経験



私は誰もが組織の労働の会計の問題を直接知っていると思います。 ほとんどの場合、ITプロジェクトへの最終投資を決定するのは、開発、テスト、および実装の時間数です。



当社には、従業員の時間費用を会計処理するタスクもあり、独自の製品K-Pointで解決しました。 詳細を説明することなく、 K-Pointは組織の作業プロセスを自動化するためのプラットフォームであり、設計者と同様に、さまざまな機能を収集して組み合わせることができます。 私たちが自分用にセットアップし、積極的に使用しているモジュールの1つにより、従業員の時間コストを追跡できます。



しかし、会社のシステムに関係なく、プロジェクトマネージャーがプロジェクトに費やした時間を提供するために何度も従業員に通知する必要があるとき、従業員は実際に時間を埋める作業に時間がかかりすぎることに満足していないことは誰にとっても痛々しいことです。



最近まで、スクリプトは4つのステージで構成されていました。







  1. 一日の終わりに、人件費を記入するためのリマインダーが従業員に送信されました。
  2. 従業員はOutlookで通知を開き、K-Pointのリンクをたどりました。
  3. 従業員はシステムからログインとパスワードを入力しました。
  4. 従業員は、指定された日の各プロジェクトのワークアウトに記入しました。


これがかなり長い日常的なプロセスであるという事実に加えて、さらにいくつかの問題が見つかりました。 多くの従業員はそのような夕方の通知を「明日のために」残し、後でそれらについて忘れていました。 Outlookに個別のフォルダーを作成し、従業員の空き時間を増やすためにこれらの通知が蓄積されるものもありました。



Outlook用の新しい種類のアドインの開発の可能性について知ったとき、プロセス全体を大幅に簡素化する機会があることに気付きました。 そこで、Outlook用のK-Pointアドインの開発を開始しました。これにより、最終的にシナリオのステップ数を2つに減らすことができました。







  1. 午前中、従業員はK-Pointシステムから通知を受け取ります。通知は、一種の「チケット」と見なされるようになりました。 すべてのアクションをその中で直接実行できます。
  2. 従業員は、日中の都合の良いときにOutlookで「チケット」を開き、第三者のシステムにアクセスしたり、ログインやパスワードを入力したりすることなく、ワークアウトを直接レターに記入します。










このアプローチにより、プロセスが大幅に簡素化され、従業員とマネージャーの両方の時間を節約できます。



  • アドインを使用すると、特定の営業日のデータを入力できます。すべてのデータが入力されたら、リマインダーレターを簡単に削除できます。これは、チケットが解決されたことを意味します。
  • アドインがインストールされると、タスクに対する従業員の態度が変わります-通知は、放棄されたフォルダーにフィルターされる迷惑なスパムとは見なされませんが、タイムリーに完了する必要があるタスクとして、行動を促すフレーズと見なされます;
  • プロジェクトマネージャーは、各従業員が常にリマインダーを使用して実行する必要はありません。


追加の利点は、そのまま使用できるアドインがOutlookのデスクトップバージョンだけでなく、 Outlook Onlineやモバイルデバイスでも機能することです 。 これは、たとえば出張中の場合に特に便利です。 以下のビデオは、 iPhoneでアドインを操作する方法を示すデモの例を示しています。







実装機能



Outlookのアドインについて話すときに強調すべき魅力的な機能の1つは、実装の容易さです。 アドインの特徴は、時間追跡のタスクのみを解決するように設計されていないことです。 アドオンは、K-Pointシステムの論理的な継続であり、ターゲットプロセス用に柔軟に構成できます。 一見、機能はそれほど単純ではありません。 しかし、どうやってこれを達成したかを見てみましょう。



アドイン自体は、K-Pointサーバーから受信するデータのプロセッサとビジュアライザーのみであり、表示されたフォームの形式とデータ入力後に実行する必要があるアクションをカスタマイズできます。 アドインは、 JSON形式のフォームとアクションの「説明」を取得し、ユーザーに必要な情報を表示し、フォームに入力されたデータをK-Pointに送り返すことができます。



K-PointはWebソリューションであり、フォームを視覚化できるJavaScriptエンジンを既に実装しています。 上部構造に移ったのは彼でした。 このようなコードの「再利用」により、開発作業が大幅に削減されました。 Outlookからユーザー認証のロジックを実装し、K-Pointとアドイン間でデータを交換するためのAPIを開発するだけで済みました。 この実装の技術的な詳細は、この記事の範囲外です。 それらについて学ぶことに興味があるなら、この記事へのコメントにそれについて書いてください。



完成したアドインが既にある場合は、 Officeストアに公開するか、特定の組織内に展開できます 。 2番目のオプションは、特定の顧客向けにターゲットを絞ったB2Bソリューションを作成する場合に特に役立ちます。 Officeストアでアドオンを公開するには、 Microsoft Seller Dashboard Webサイトで登録する必要があります 。 登録後、個人アカウントにアクセスし、アプリケーションのマニフェストをダウンロードして、必要な情報をすべて記入できます。



結論



ワークフローにOutlook用のK-Pointアドインを導入したことで、すべての従業員とマネージャーの両方がレポートに費やす時間を半減することができました。 これは確かに良い結果であり、将来的には改善する予定です。



現在、 OutlookのK-Pointセットアップはパブリックアクセスには使用できず、社内で導入されています。 近い将来、Officeストアで公開し、既存のお客様の一部に同様の自動化シナリオを提供する予定です。 これが何をもたらすのか見てみましょう。



私たちの会社の仕事とお客様の仕事の一部として、Outlook用のアドインを使用できるシナリオがたくさんあります。 このようなアプリケーションは、電子文書管理システム、CRM、ERP、その他のタイプのシステムなど、既存のソリューションの機能を拡張するための興味深いオプションであるという事実に別に注意を払う価値があります。 生産性を高めるためのいくつかのアイデアもあなたに現れたと確信しています。したがって、この記事を書くという目標は達成されました。 コメントにあなたのアイデアを書いてください、私たちは一緒に問題の解決策を議論し、探します！
