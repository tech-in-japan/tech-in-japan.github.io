ルア。 ダミーのメタテーブルの簡単な紹介







この記事を書くために、私はLuaのメタテーブルとOOPに関する多数の質問に促されました。これはこの言語の学習者にとって最も困難で問題のあるセクションであるためです。しかし、Luaは初心者およびメタテーブルの助けを借りて、複雑なタスクに対する驚異的で非常にエレガントなソリューションを使用できることを考えると、開発用の資料を「後で」残すことは価値がありません。



この出版物では、Lua 5.1-5.3のバージョン用のすべての標準メタメソッドを例とともに説明します。 始めましょう。



メタテーブル



これは何ですか



実際、メタテーブルは通常のテーブルと違いはありませんが、コントロールテーブルとしてリストされている点が異なります。



概略的には、たとえば次のように想像できます。







メタテーブルは、アクションに対するメインテーブルの反応を説明します。たとえば、テーブルを関数として呼び出したり、任意の値でテーブルを分割したり、特別なキーのおかげで、テーブルからキーを抽出しようとします（特に明記しない限り、Lua 5.1）：



一般的なメタメソッド



  • __index-元のテーブルが存在しない場合にキーを探す関数またはテーブル。
  • __newindex-テーブルに* new *キーを追加する方法の機能は、既存のものに影響しません。
  • __call-テーブルが関数として呼び出されるときに呼び出される関数。
  • __tostring-たとえば、printまたはtostringを使用して、__ concatと組み合わせて、テーブルを文字列に変換しようとするときに呼び出される関数。
  • __concat-テーブルを何かと連結しようとするときに呼び出される関数は、__ tostringと組み合わされます。
  • __metatable-このテーブルからメタテーブルを取得しようとして返される値により、メタテーブルを非表示にできます。
  • __mode-ガベージコレクション中にテーブル内のリンクの「強度」を制御する文字列。これを使用して、弱リンクテーブルまたはエフェメロンを作成できます。
  • __gc-ユーザーデータ（5.1+）またはテーブル（5.2+）がガベージコレクターによって収集されるときに呼び出される関数（5.1で本当に必要な場合）-使用する方法があります。
  • __len-＃（5.2+）演算子を使用してテーブルの長さを計算しようとするときに呼び出される関数。
  • __pairs-関数、特定のテーブル（5.2+）のペア反復子の代替。
  • __ipairs-関数、ipairs（5.2+）の代替;


数学的メタメソッドと比較（関数）



  • __add-（+）追加;
  • __sub-（-）減算;
  • __mul-（*）乗算;
  • __div-（/）部門;
  • __pow-（^）べき乗;
  • __mod-（％）モジュロ除算;
  • __idiv-（//）整数部を削除した除算（5.3+）;
  • __eq-（==）等値比較;
  • __lt-（<）比較「より小さい」、反対方向で自動的に実行され、引数を逆にします。
  • __le-（<=）比較「以下」;


ビット操作（機能、5.3以降のみ）



  • __band-（＆）「そして」;
  • __bor-（|） "OR";
  • __bxor-（〜）排他的な「OR」（a〜b）;
  • __bnot-（〜） "NO"（〜a）;
  • __bshl-（<<）ビット左シフト。
  • __bshr-（>>）ビットを右にシフトします。

索引



最も一般的なメタメソッドの1つであり、最も多くの質問を引き起こしています。

引数（self、key）を持つテーブルまたは関数になります。selfは検索するテーブルで、keyは値を取得するキーです。



このメタメソッドに基づいて、OOP、プロキシテーブル、テーブルのデフォルト値など、多数の機能が構築されます。



このテーブルの正確なキーを取得する必要がある場合は、有害になる可能性があります。そのような場合は、テーブルのデフォルトのアクセス関数である関数値= rawget（テーブル、キー）を使用します（キーで値を取得しようとすると、バイトコードで呼び出されます） 。 



--[[1]] foo = {} foo.key = 'value' --[[2]] mt = {__index = foo} -- setmetatable       --[[3]] bar = setmetatable({}, mt) bar.key2 = 'value2' -- : --[[4]] print('bar.key2', bar.key2) --> 'value2' --[[5]] print('bar.key', bar.key) --> 'value' --[[6]] bar.key = 'foobar' foo.foo = 'snafu' print('bar.key', bar.key) --> 'foobar' print('bar.foo', bar.foo) --> 'snafu' print('bar.foobarsnafu', bar.foobarsnafu) --> nil --[[7]] foo = {key = 'FooBar'} --    bar = setmetatable({}, {__index = foo}) snafu = setmetatable({}, {__index = bar}) print('snafu.key', snafu.key) --> 'FooBar' --[[8]] foo = {} foo.__index = foo setmetatable(foo, foo) -- print('foo.key', foo.key) --> error: loop in gettable print('foo.__index', foo.__index) --> "table: 0x12345678" print('rawget(foo, "key")', rawget(foo, "key")) --> nil --[[9]] foo = {} foo.key = 'value' setmetatable(foo, {__index = function(self, key) return key end}) print('foo.key', foo.key) --> 'value' print('foo.value', foo.value) --> 'value' print('foo.snafu', foo.snafu) --> 'snafu' --[[10]] fibs = { 1, 1 } setmetatable(fibs, { __index = function(self, v) self[v] = self[v - 1] + self[v - 2] return self[v] end })


ここで何が起こっていますか：



1. foo-持っていないキーを探すテーブル。

2. mtは、キー__index = fooを持つテーブルです。 メタテーブルのようなものにアタッチすると、「キーがない場合は、fooでそれらを見つけてください。」と表示されます。

3.メタテーブルmtを空のテーブル（バーになる）にチェーンするプロセスを次に示します。

4.テーブルキーへの直接アクセスの例。 この場合、通常のようにバーテーブルからキーを取得します。

5. __indexのキーへのアクセスの例。 この場合、キー["key"]はbarテーブルにありません。fooテーブルの__indexメタテーブルで検索します。

6.明確化：バーテーブルにキーキーを追加すると、その中に見つかり、値を取得しようとすると、メタメソッドチェーンは呼び出されません。 しかし、["foo"]などのその他の存在しないキーはすべて、メタメソッドチェーンを呼び出し続けます。 キー["foobarsnafu"]は両方のテーブルに存在せず、その意味は通常nilです。

7.インデックスを使用すると、検索チェーンを作成できます。 この場合、キー検索アルゴリズムは次のとおりです。



1. snafuテーブルでキー["key"]を探しています。

2.見つかりません。 snafuメタテーブルには、バーテーブルを指す__indexキーがあります。 私たちはそこを見ています。

3.再び見つかりません。 しかし、fooテーブルを指す__indexキーを持つメタテーブルもあります。 探しています。

4.見つけた！ ここがキーであり、その意味は「FooBar」です！







8.この場合、それ自体に等しい__indexキーを持つテーブルを作成し、それ自体のメタテーブルとして設定します。 不足しているキーの値を取得しようとすると、内部で検索を試行し、__ indexメタテーブルをナビゲートしてさらに検索する再帰サイクルが発生します。 したがって、閉じた検索チェーンを実行しない方が良いです。 rawgetを使用する場合、メタメソッドは呼び出されず、このキーの正確な値が取得されます。

9. __indexキーとして、メタテーブルは2つの引数を持つ関数を持つことができます-self-テーブル自体、key-値を取得したいキー。 関数の戻り値が値になります。 これにより、テーブルに任意のインデックスを作成したり、プロキシを作成したりできます。

10. Wikipediaから例を取り上げます。 この場合、fibsテーブルの__indexは、メモ化を使用してフィボナッチ数の値を自動的に再計算します。 print（fibs [10]）は、10番目のフィボナッチ数を出力します。 欠落したテーブル値の再帰的な計算を介して機能します。 後続の値はテーブルにメモされます。 理解するには少し時間がかかります。fibs[v-1]が存在しない場合、fibs [v]と同じアクションセットが実行されます。



Newindex



あまり一般的なメタメソッドではありませんが、閉じたテーブルの作成、フィルタリングまたはプロキシ処理などに便利なこともあります。



常に引数（self、key、value）を持つ関数のみにすることができます。



有害な場合があるため、このメタメソッドを強制的に使用しない場合、rawset（self、key、value）関数が使用されます。これは、テーブルのデフォルト関数です。

2. __newindexは、存在しないキーにのみ適用されます。



3.数値のみをテーブルに追加できるフィルター関数の例。 同様に、「数字キーのみを追加する」をチェックするか、行テーブル番号などのために事前にいくつかのテーブルを作成し、対応するものに値を追加することができます（分類/バランスなど）。

OOP



最初に頼むのは、OOPを作成する試みです。



いくつかの抽象的な「クラス」を実装する単純な関数を作成してみましょう。 



 function Class() local class = {} --   . local mClass = {__index = class} -- ,   ""  . function class.instance() return setmetatable({}, mClass) end return class end --   . Rectangle = Class() function Rectangle.new(x, y, w, h) local self = Rectangle.instance() self.x = x or 0 self.y = y or 0 self.w = w or 10 self.h = h or 10 return self end function Rectangle.area(self) return self.w * self.h end --  rect = Rectangle.new(10, 20, 30, 40) print('rect.area(rect)', rect.area(rect)) --> 1200 print('rect:area()', rect:area()) --> 1200


ここでは、すでにOOPに似ています。 継承やあらゆる種類のクールなものはありませんが、これは悪くありません。

rect.areaを呼び出すとき、オブジェクトテーブルにはエリアキーがないため、クラステーブルの__indexを介して検索し、それを見つけて、そこで最初の引数に置き換えます。



メタテーブルからの小さな余談：2回目の呼び出しの例は、この記事の最初のコロンの出現です。 コロンは、Lua言語の構文糖衣です。 テーブル内の関数をピリオドではなくコロンで呼び出す場合、テーブル自体がこの関数の最初の引数として置換されるため、そのコードは同等です。



詳細： 



 foo = {x = 10, y = 20} function foo.bar(self, a, b) return (self.x + a) * (self.y + b) end print('foo.bar(foo, 1, 2)', foo.bar(foo, 1, 2)) --> 242 -- ,  self "" . function foo:bar(a, b) return (self.x + a) * (self.y + b) end print('foo:bar(1, 2)', foo:bar(1, 2)) --> 242


このオプションを少し改善してみてください。



まず、オブジェクトを返す関数としてクラスを呼び出す機能を追加します。次に、クラス自体の演算子をオーバーロードする機能を追加します。第三に、継承です。

 function proxy() local real_table = {} local logger = {} local metatable = {} --         function metatable:__index(key) local value = rawget(real_table, key) table.insert(logger, "Get key "..tostring(key)..' is '.. tostring(value)) return value end --     ,    function metatable:__newindex(key, value) table.insert(logger, "Set key "..tostring(key)..' as '..tostring(value)) rawset(real_table, key, value) end return setmetatable({}, metatable), logger end prox, log = proxy() prox.a = 10 prox.a = 20 print('prox.a', prox.a) --> 20 print('log', '\n'..table.concat(log, '\n')) --> Set key a as 10 --> Set key a as 20 --> Get key a, is 20
