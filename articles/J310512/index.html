オラクル、クラウド市場でAmazonを打ち負かすのに役立つ新製品を約束





オラクル、第2世代クラウドインフラストラクチャを発表 Oracle Data Centerをアプリケーションで使用したいサードパーティ開発者向けです。 これは、サンフランシスコで開催されたOpenWorldカンファレンスで、同社のテクニカルディレクターであるラリーエリソンによって語られました。



インフラストラクチャパラメータは次のとおりです。1つの仮想マシンで28.8 TB、512 GB、36コア。 それはすべて1時間あたり5.40ドルかかります。 オラクルの代表者は、これらの数字は、このセグメントの会社の主要な競争相手であるアマゾンウェブサービス（AWS）の機能よりも10倍高いと主張しています。



さらに、Oracleは別の新しいサービスCloud @ Customerを開始しました。 このサービスの主なアイデアは、クラウドコンピューティングに必要な機器をクライアント側にインストールして、機密性とデータセキュリティを保証することです。 開発者によると、Cloud @ Customerサービスは、競合他社の製品よりも強力で安価なツールです。 また、残りのサービスは、よく知られているOracle Cloudツールと同じタスクを実行します。



ブルームバーグの報告によると 、Oracleの理事長であるラリー・エリソン氏は、顧客がハードウェアの費用を支払うことはなく、毎月のサブスクリプションを購入できると述べた。



「Amazonの優位性は終わりました。 将来、彼は深刻な競争に直面するだろう」とアリソンは言った。

ロシアの新しいサービスの発売日に関する情報はありません。
「当社のGeneration2 IaaSソリューションは、2倍のコンピューティングパワー、2倍のRAM、2倍のストレージ、10倍のI / Oリソースを、Amazon Webサービスよりも20％低い価格で提供します。 -ラリー・エリソンは言った。 「IaaS（Infrastructure-As-A-Service）は、急成長中のSaaSおよびPaaSビジネスにクラウドインフラストラクチャを追加する大きな機会です。」



2016年の冬、オラクルはパートナー向けの新しいOracle PartnerNetwork（OPN）クラウドプログラムの開始を発表しました。これは、パートナーがOracle Cloudでクラウドビジネスを拡大し、技術サポートと市場開発支援を提供できるように設計されています。 また、企業のクラウドビジネスの発展を刺激することになっていた。







Amazonは最近、SQLエンジンを介したストリーミングデータのリアルタイム分析処理を可能にする新しいサービスであるKinesis Analytics（「アクティブ分析」）を開始しました。 Amazon Web Services（AWS）には、ストリーミングデータと連携するように設計されたKinesisプラットフォームに2つのサービスが実装されていました（このようなデータをダウンロードするFirehoseと、この情報を処理するアプリケーションを作成するStreams） S3、Redshift、Elasticsearchサービスを自分で管理する必要がありました。 これで、標準のSQLツールを使用できます。



新規取得



2016年9月、オラクルはPalerraスタートアップの買収を発表しました。 この取引は、クラウドサービスを特定および保護するための包括的なツールを顧客に提供することを目的とした戦略の一部です。 購入価格はまだ明らかにされていません。

Palerra-2013年に設立された、クラウドサービスの情報保護に使用されるソフトウェアの開発者。 同社の主要製品はLoricプラットフォームであり、これにより顧客は脅威検出プロセス、インシデント対応、クラウドベースの分析を自動化できます。
トランザクションが完了すると、すべてのスタートアップ従業員がOracleに移動します。 ただし、彼らは単一のチームのままであり、別のユニットで作業します。



Larry Ellison氏は、Palerraソリューションはオラクルのクラウドセキュリティポートフォリオへの重要な追加になると述べました。



「Oracleではセキュリティが最も重要です。 私たちはそれを構築し続け、この仕事をうまくできるサプライヤーを見つけたら、それを買います」とアリソンは言いました。



Oracleは、Oracle Identity Cloud ServiceソリューションとLoricを組み合わせて、「ユーザーのクラウドを安全に実装するためのユーザー、アプリケーション、API、データ、およびインフラストラクチャの包括的な保護を提供する」とOracle Information Securityの上級副社長Peter Barkerが手紙で書いています顧客とパートナーに宛てた。



オラクルはまた、同社はPalerra製品ポートフォリオをレビューしており、後に特定のソリューションの使用に関する推奨事項を顧客に提供すると述べています。 さらに、オラクルは、Palerraユーザーに製品とOracleソフトウェアの効果的な統合を約束しました。



お金の言葉で



アナリストは、クラウドサービスを世界全体、特に個々の地域の両方で最も成長している分野の1つと考えています。 これは、クラウドインフラストラクチャを展開するために大量の機器を購入することで確認されます。







したがって、2016年の第1四半期には、IDCによると、EMEA地域（ヨーロッパ、ロシア、中東、アフリカを含む経済地域）でクラウドサービスを実行およびサービスするために購入したサーバー、ストレージシステム、ルーターの売上は$ 1.3に達しました10億、17.6％の増加。



IDCのアナリストは、2020年までにEMEAクラウドインフラストラクチャ市場は107億ドルに達すると予測しています。これはIT機器支出の46.4％に相当し、従来の市場はせいぜいゼロ成長を示し、最悪の場合は落ちることです。

Oracle Corp.は、2017年度第1四半期のSaaSおよびPaaSクラウドサービスからの収益が7億9,800万ドルに達し、77％（恒常通貨で計算すると79％）増加したと推定しています。



IaaSを含むクラウドサービスへの加入による総収益は9億6,900万ドルに達し、米ドルで計算すると59％（固定通貨で計算すると61％）増加しました。



ソフトウェアの総売上高は68億ドルでした。 この金額には、エンタープライズに直接展開するためのライセンスの販売による収益とクラウド環境のサブスクリプションが含まれます。 前年同期比は、米ドルで計算した場合5％、恒常通貨で計算した場合6％でした。



OracleのCEOであるSafra Katzは、クラウドビジネスと企業で展開するためのソフトウェアの販売という2つの戦略的分野の成長を結び付け、SaaSとPaaSからの総収益が82％増加することを求めています（一定通貨で計算し、GAAP要件を除外した場合）。 この増加は、計画よりも大幅に増加しました。



「SaaSおよびPaaSビジネスの急速な成長に伴い、粗利益率は今四半期の62％から目標の80％に上昇すると予想しています」とSafra Katz氏は予測しています。



オラクルのCEOであるマーク・ハードは、企業の使命はSaaSとPaaSから20億ドル以上の経常収入を生み出すことであると述べました。



「オラクルが他のクラウドプロバイダーよりも多くのSaaSおよびPaaSソリューションを販売するのは、今年で2年目になると考えています」とMark Heard氏は付け加えました。 「第1四半期だけで、344の新しいSaaS Fusionの顧客を含む750を超える新しいSaaSの顧客がいました。ERP-この会社のすべての年のWorkdayより多くのERPクライアントです。」



1つの「しかし」



ただし、オラクルの大規模で収益性の高い従来のソフトウェアビジネスは縮小しています。 過去の財務四半期では、新しいソフトウェアライセンスの実装は11％減の10億3,000万ドルでした。 その結果、同社の総収益はわずか2％増加し、86億ドルに達し、アナリストの予測には達しませんでした。



さらに、Oracleは18億3,000万ドル、1株当たり43セントの純利益を報告しましたが、前年同期は17億5,000万ドルまたは40セントでした。 調整後利益は1株当たり55セントでしたが、トムソン・ロイターが調査した専門家は1株当たり58セントの結果を期待していました。



株式市場はこれに反応してOracleの相場を引き下げました。TheFinancial Timesによると、同社の株式はほぼ3％下落しました。 オラクルは今四半期、1株当たり58〜60セントの調整後利益を予測していますが、ウォール街のコンセンサス予測では65セントを提供しています。







過去9四半期にわたって、オラクルの調整後の収益はアナリストの3倍も悪化しています。



しかし、Safra Katzは、ライバルNetSuiteの買収プロセスは最終段階にあると述べました。 93億ドルの購入が7月に発表されました。 これは、オラクルが中小企業の顧客を引き付けるのに役立つと期待されています。 以前は、会社は主に大規模な顧客を扱っていました。



Safra Katzによると、この取引は米国の規制当局を除くすべての独占禁止当局によってすでに承認されています。 9月末までに決定を発表する必要があります。



バラスト放電



オラクルは、その力を分配し、最も有望な分野にのみ投資します。 これは、2010年に買収されたSunから継承されたプロジェクトの閉鎖を説明しています。

次はNetBeansの番です。 Oracleはこれをサポートする予定はありません。

NetBeansは、オープンIDE、ツールプラットフォーム、およびアプリケーションフレームワークです。 これはJava言語で書かれており、主にJavaプログラムの作成を目的としています。 しかし、JavaScript、PHP、C / C ++などの他の言語もサポートしています。



NetBeansは1995年にリリースされました。 2000年にSunに買収されました。 また、2010年には、Sunの買収の一環としてOracleの財産となりました。
Oracleは体系的にSunソフトウェア製品を取り除きます。 買収がOpenSolarisの開発を中断した直後のLarry Ellison。



2013年、オラクルはサンの仮想化テクノロジーのほとんどのサポートを完了しました。

ただし、NetBeansの現在の状況のほとんどは、Sun自身がOpenOfficeプロジェクトを放棄したときの状況に似ています。



OracleがNetBeansプロジェクトを閉じただけでなく、Apache Software Foundationに開発を継続するように招待したことは良いことです。 オラクルは、NetBeansの下で150万人の開発者がまだ働いていると説明しました（会社の主観的な感覚による）。



すべてのNetBeans関連のソースコード、NetBeansの商標、netbeans.orgドメイン、および一部のインフラストラクチャ要素に対する権利は、Apacheに移行されます。 NetBeansは、OpenOffice.orgの場合のように、GPLv2およびCDDLライセンスとともにコピーレフトのソースコードで配布されるため、プロジェクトは、再ライセンスおよび許可されるApache 2.0ライセンスへの移行を期待しています。



NetBeans Dream Team開発者コミュニティのメンバーであるZoran Sevarachは、提案された取引を承認しています。「これは非常にクールです。これは、NetBeansに明るい未来があることを示しています。 NetBeansコミュニティ全体がこのステップに肯定的です。」



James Goslingはこれに同意し、Facebookに投稿しました。「NetBeansはApacheに移行しています！ オラクルは、私のような人がお気に入りのIDEにコードを簡単に追加できるように、NetBeansをさらに開くことにしました。 既存の最高のIDEがさらに良く、高速になります！」



Apache Foundation運営委員会によるプロジェクトの採用に関して共同決定が行われた後、NetBeansはApacheインキュベーターに配置され、そこでインフラストラクチャが準備され、ライセンスのクリーン度が監査され、Apacheコミュニティで受け入れられている開発原則に準拠する能力がチェックされます。



「Eclipseは何年も前にNetBeansを追い抜いてきた。 プログラマーが実際に使用する言語とツールを追跡するEvans DataのCEOであるJanel Garwin氏は次のように述べています。



長年、NetBeansとEclipseの間で、Java開発のメインIDEのタイトルをめぐる競争がありました。 Eclipseを倒した。



OracleはJavaに依存しなくなりました。 同社は、Java Enterprise Edition（JEE）8の新しいエンタープライズバージョンのリリースを強く遅らせました。これは、2017年に最終的にリリースされる予定です。



「オラクルがEEのリリースを延期しているのはそれほど怖くはありません。Javaコミュニティとの共同作業から離れたのは恐ろしいことです。 まず、Javaを独自のトラックに切り替えてから、ゴキブリトラップのRoach Motelモデルを言語に適用しました。これに応じて、中に入ることはできますが、外に出ることはできません」とJames Gosling氏は述べています。
