openSUSE 12.1でWiMaxインターネットをセットアップし、Wi-Fi経由で他の人に配布する

はじめに



Linuxディストリビューションの最初のインストール時に、インターネットがイーサネット経由で接続されていない場合、インターネットを取得して他のユーザーに配布する際に、ほとんどの場合に問題が発生します。 多くの場合、これにより初心者はさらに知り合いになり、Linuxを使用するようになります。 opensuseのこのトピックに関する完全なHOWTOを探しても、見つかりませんでした。 試行錯誤により、この方法は判明しました。



WiMaxインターネット接続



標準DVDには、WiMaxモデム用のドライバーがありません。 この方法は、Samsung CMC-730チップに基づくデバイスに適しています。 現在、次のデバイスがサポートされています。

  • サムスンSWC-U200
  • サムスンSWC-E100
  • Samsung SWM-S10R（Samsung NC-10ネットブックの一部）
ここから事前にRPMをダウンロードすることをお勧めします-software.opensuse.org、i586を選択-32ビットバージョンのopenSUSE 12.1、x86_64-64ビットバージョンのOSでは、新しくインストールしたOSで後で使用するためにUSBフラッシュドライブにドロップします。 openSUSEのインストール後、ファイルをUSBフラッシュドライブからユーザーのホームディレクトリにコピーします。

将来的には次のことを検討します。

1.ユーザーのホームディレクトリ-/ home / user /

2. WiMaxモデムドライバーファイル-madwimax.rpm

3. wlan0-Wi-Fi接続

4. wimax0-WiMax接続

zypperバッチマネージャーを使用してターミナルにインストールします

sudo zypper in /home/user/madwimax.rpm







ほとんどの場合、インストール中にDNSは通常登録されません。DNSを入力すると同時に、コンソールテキストエディターNanoを配置して、後で必要な構成ファイルを修正できるようにします

sudo zypper in nano

sudo nano /etc/resolv.conf







ネームサーバーを含むファイルに行があるかどうかを確認し、ない場合は追加します（習慣からopenDNSサーバーを使用します）

nameserver 208.67.222.222

nameserver 208.67.220.220







CTRL + Xを押して保存し、上書きを確認します。

これで、WiMaxモデムを接続してインターネットを楽しむことができます。



インターネット配信用のWi-Fiのセットアップ



必要なパッケージをインストールする


zypperを使用して設定する必要があるパッケージをインストールします

sudo zypper in hostapd dhcp-server yast2-dhcp-server







接続とDHCPサーバーを構成する


必要なインターフェイスの可用性を確認します：wlan0およびwimax0

/sbin/ifconfig







すべてが準備できました。先に進みます。 Yastに移動し、[ネットワーク設定]を選択します。 Network Managerを使用している場合は、それを無効にして、[グローバルオプション]タブでifupを使用して管理に移動します。 ネットワークマネージャーは、madwimaxドライバーにまだ馴染みがなく、接続が表示されないため、Network Managerをオフにします。 [概要]タブに移動して、Wi-Fi接続を編集します。 設定を入力してください：

IP adress - 192.168.0.1

Subnet mask - 255.255.255.0







[次へ]をクリックすると、それ以降の設定は変更されません。

YastでDHCPサーバーを開き、サーバーがIPを配布するインターフェイスとしてwlan0を選択し、DCHPのファイアウォールを開くためのチェックボックスを有効にして、グローバル設定を入力します。

Primary Name Server IP - 208.67.222.222

Secondary Name Server IP - 208.67.220.220

Default Gateway (Router) - 192.168.0.1

Dynamic DHCP :

Network - 192.168.0.0

Netmask - 255.255.255.0

Netmask bits - 24

First IP Adress - 192.168.0.2

Last IP Adress - 192.168.0.100







次に、起動時に起動をオンにして、すべてを保存します。



SuSEFirewallの構成


コンソールを開き、ファイアウォール設定ファイルを開きます

sudo nano /etc/sysconfig/SuSEfirewall2







私たちは何も削除しません。突然、将来自分で何かを仕上げたいと思います。 行を見つけて、次のパラメーターを入力します

#

FW_DEV_INT="wlan0"

#

FW_ROUTE="yes"

#

FW_MASQUERADE="yes"

#,

FW_MASQ_NETS="192.168.0.0/24"







ファイアウォールを保存して再起動します

sudo /etc/init.d/SuSEfirewall2_setup restart







ソフトAPセットアップ


元の構成ファイルをコピーし、新しい構成ファイルを開いて編集します

sudo mv /etc/hostapd.conf /etc/hostapdbkp.conf

sudo nano /etc/hostapd.conf





ファイルに次の内容を書き込みます。MYAPとMYPASSはそれぞれネットワークの名前とパスワードに置き換えます

# Wi-Fi

interface=wlan0

#

driver=nl80211

# ,

ssid=MYAP

#

country_code=RU

# (b, g, n)

hw_mode=g

# Wi-Fi. , 1 11

channel=9

#

logger_syslog=-1

logger_syslog_level=3

logger_stdout=-1

logger_stdout_level=2

# (SSID)

ignore_broadcast_ssid=0

#

auth_algs=1

wpa=2

wpa_key_mgmt=WPA-PSK

wpa_passphrase=MYPASS

rsn_pairwise=CCMP







ファイルを保存します。 アクセスポイントを起動する準備がすべて整いました。



最後のステップ。 アクセスポイントの起動


sudo /usr/sbin/hostapd /etc/hostapd.conf







別のデバイスをネットワークに接続し、インターネットの可用性を確認します。

インターネットがない場合は、デバイスが受信したパラメーターを確認します。パラメーターは次のようになります。

IP- - 192.168.0.* // 2 100

- 255.255.255.0

- 192.168.0.1

DNS- 1 - 208.67.222.222

DNS- 2 - 208.67.220.220







リストに何かが欠けている場合は、記事のその瞬間に戻って設定し、実行されたアクションの正確性を確認します。
