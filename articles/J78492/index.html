「ブラジルのシステムに応じた」トレーニング

こんにちは！

最近の投稿（ ウルフの日 、 蝶の効果 、 自由または恐怖。選択をしてください ）で取り上げられたトピックに関する私の考えをコミュニティと共有したいと思います 。

おそらく少し離れたところから始めるべきでしょう。

1997年、私は極東州立大学の生物学部を卒業し、生物学者として働きに行きました。 私は実験室に座って退屈し、9年間、大河を駆け回り、さまざまな興味深い動物を研究していました。 そしてその前に、控えめに言っても、彼らがその仕事にほとんどお金を払っていないという事実を特に気にしなかったことは、私にとって興味深いことでした。





2006年、彼はサンクトペテルブルクに移り、すぐに仕事を探しませんでしたが、2007年に都市研究所で働き、トヴェリの一般計画とアルハンゲリスク地域の開発計画の作成に参加し、2007年11月からロシア南極遠征で生態学者としての仕事を得ました。 私は南極で1年以上過ごし、ペンギンを勉強しました...給与は許容範囲内でしたが、それはもっと大きかったかもしれません。



それで、冬の季節に、「次は何？」というカテゴリから、さまざまな賢い考えが頭に浮かび始めました。もう遠征に行きたくありませんでしたが、研究所の生物学者の退屈な予算給与にこれ以上座りたくありませんでした。

現時点では、友人の助けがなくても、生物学の分野ではすべてが資金調達でとても悲しいので、活動分野全体を変える価値があるかもしれないという考えに立ち会いました。



LinuxディストリビューションのDVDを2枚持っていましたが、そこにはプログラミング言語とドキュメントの両方が含まれていました（ステーションにはインターネットへの無料アクセスはありませんでした）。 そして、Merlezonskyバレエの最初の部分が始まりました-最初にデータベース（実際の作業のために構築できるため-私が収集したフィールド資料を処理するため）、次にこれらのデータベースへのインターフェース。



その結果、冬の終わりの時点で、私は独学者として、MySQLで実際のPostgreSQLデータベースとそのレプリカを作成し、C、Perlでこれらのデータベースを操作するための簡単なインターフェイス（データ入力、変更、削除、あらゆる条件での出力）を作成しました、Python、PHP、およびRuby（純粋なRuby、私はまだレールについては知りませんでした）。



しかし、すでに繰り返し言及されているその同じ恐怖は、公開された空席から同じラインを繰り返し私の内の視線にもたらしました：「それはプログラマーを必要とします... 3年間の実務経験があります...」 10年、プログラミングの経験-いくつかのゴミ...



2009年5月末に戻り、休暇手当を受け取り、すぐに親relativeや友人と次に何をすべきかについて話し合いました。 私の良き友人たちはすぐに、仕事の経験の不足を心配するのではなく、最初にデータベースでの10年の経験を示す履歴書を書くようにアドバイスしました（実際、過去数年間はMS Accessを使用していました）フィールド材料を処理するために）、2番目に、そこに1年のWeb開発経験を追加します（PHPでデータベースへのインターフェイスを行いましたか？仕事の説明でよく見られる言葉は魔法のようなもので、明らかにHTML + CSS + JQuery + AJAXの集まりです。 だから私はやった。 そして、彼はある有名なサイトのヘッドハンターに掲載されているすべての空席に答えました。



すぐに応答が届き始めました。 原則として、追加の質問（「これを行う方法を知っていますか？」、「それでどのように行いますか？」）、またはタスクをテストします。 いずれにせよ-到達できるすべてのものを1日または2日間集中的に読み、私は手紙にあるすべての質問に自信を持って答えることができました。



1か月後、ある事務所から電話でインタビューを受けて、コードを確認しました。 その結果、彼らは私をそこに連れて行かず、マラソンは続きました。 ジョブズ-テスト-スタディ-欠員-テスト-スタディ-万歳！ 就職面接！ -失敗...

5回目のインタビューの後、私は雇われました。 彼らは長い間考え、疑っていましたが、すべては、それを素早く学び、それを行う能力と意欲によって正確に決定されました。

就職活動全体は3ヶ月弱かかりました。 極地休暇は、最初の給与が「背中合わせ」になるまで十分でした。



このようなトレーニングの質は、次の事実によって判断できます。

1）試用期間が正常に経過した。

2）最近、私は再び給料を上げられました。



正直なところ、34歳で突然（生物学からプログラミングに）分野を変えることは簡単ではありません。 しかし、実際には、ご覧のとおりです。



私たちの組織がRORを使用した興味深いプロジェクトを開始して以来、RubyOnRailsが議題になりました。 私たちは先に進み、読み、書き、学びます...



変更することを恐れないでください、新興企業に合格することを恐れないでください、雇用主（投資家、美しい少女など）の失敗を恐れないでください。 やれ！
