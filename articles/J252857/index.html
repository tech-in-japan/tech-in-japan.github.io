仮想化されたエンタープライズ環境でMSAを実装する

仮想化は、現代のITで最も重要なトピックになっています。 VMwareによると 、ロシアのサーバー仮想化は2014年に30％に増加し、3年で倍増しました。 その結果、サーバーの使用効率が向上します。さまざまな推定によると、平均負荷率は50％から80％になりました。これは、物理サーバーの必要性が減少することを意味します。 当然のことながら、仮想化技術は大企業や組織だけでなく、中小企業でもますます使用されています。 まず、サーバーの仮想化について話します。







SMBの仮想化



仮想化は、数百台の仮想マシンがある大規模なデータセンターだけでなく、複数のVMが同じサーバー上で実行されている場合にも大きな利点を提供します。 その中でも-サーバー統合の利点、展開の容易さ、リソースの簡単で便利な割り当て、および機器の標準化、「動物園」システムの排除。 最終的には、これらすべてがITコストの削減と総所有コスト（ TCO ）の削減につながります。



一方、仮想化を実装する際の重要な問題は、ストレージシステムの選択です。 仮想化によりストレージシステムの負荷が増加するため、その実装、計画、ディスクアレイの適切な選択が必要になります。 I / Oパフォーマンスと待機時間の要件が高い仮想環境では、ストレージがボトルネックになることはありません。 サーバー仮想化の展開を成功させるためには、これはプロセッサー能力とサーバーRAM容量よりもさらに重要です。



サーバーの仮想化とストレージ統合タスクを解決する中小企業には、信頼性がありながら、安価で保守が容易なストレージシステムが必要です。 HPは、これらのお客様にMSAシリーズエントリレベルのディスクアレイを提供しています。 IT投資を最大限に活用したい予算が限られている企業向けに設計されています。



MSAは、最も成功したHPディスクアレイの1つです。 MSAファミリの40万以上のアレイがすでに世界で販売されており、これらはファイバチャネルで最高のエントリレベルストレージシステムの1つと考えられています。 これらのストレージシステムは、高性能、信頼性、管理の容易さを提供し、中小企業（SMB）を代表する企業に最適です。 HP MSA 1040およびMSA 2040アレイは、Windows Server 2012内から管理できます。これは、Windowsプラットフォームが小規模な組織でよく使用されるため、さらにプラスです。



MSAは多数の仮想環境をサポートし、異種ITリソースを統合し、仮想化と組み合わせて効率を向上させることができます。 HPの統計によると、インストールの60％で、MSAアレイは仮想環境に展開されています。 これらのSANディスクアレイの前世代と比較して4倍高いMSAパフォーマンスは、システムがより多くのVMをサポートできることを意味し、アプリケーションの応答時間が短縮されます。 ランダム読み取りの場合、MSA 1040のパフォーマンスは29,000 IOPS、MSA 2040は82,000 IOPSであり、スループットはそれぞれ3.1および6.2 GB / sです。



そのパフォーマンスと機能により、HP MSA 1040/2040アレイは、特にHP ProLiantサーバーと組み合わせた場合に、仮想環境の展開を計画する際に注意を払うべきソリューションを提供します 。 モデル1040は最初の仮想化プロジェクトに適したオプションであり、2040アレイを使用して、既に仮想サーバーの使用経験を積んだ企業に高性能仮想環境を展開できます。 HP MSAは、仮想環境の管理、展開、サポートを簡素化し、データを操作するために必要なサービスを提供する機能を提供します。



HP MSAの高度な機能



大企業だけでなく中小企業も、コストを削減する方法を常に模索しています。 過去10年間で、ストレージテクノロジーはストレージの効率と柔軟性を大幅に向上させました。 ただし、最近まで、これらの改善は中級および上級クラスのディスクアレイにのみ実装されていたため、中小企業にはアクセスできませんでした。



2014年の終わりに、HPは高度な仮想化機能を備えた MSAアレイのソフトウェアアップデートを導入しました。新しいソフトウェアはストレージコストを削減し、管理を大幅に簡素化します。 更新には、動的データ最適化機能、SSDフラッシュドライブの使用による生産性の向上、HP MSAハードウェアアーキテクチャの第4世代機能を使用したインテリジェントストレージ自動化が含まれます。





高度なMSA仮想化機能： シンプロビジョニング （容量の動的割り当て）、 SSD読み取りキャッシュ （ソリッドステートドライブ上の読み取りキャッシュ）、 自動階層化 （ストレージ層間でのデータの自動移動）、 書き込み時のリダイレクト（ 書き込み時のリダイレクトを伴うスナップショットデータ）およびワイドストライピング （HDDによる分配を伴う仮想プール）。



シンプロビジョニング

この機能により、タンクを柔軟に使用し、必要に応じて拡張できます。 シンプロビジョニング-最初は小さなスペースを使用し、データが書き込まれると「成長」する論理ボリュームを作成します。 ディスクスペースの動的な割り当てにより、ストレージリソースを仮想サーバーにすばやく割り当て、必要以上の容量の予約を排除できます。 シンプロビジョニングは一般的なサーバー仮想化環境でサポートされており、 VMwareによると 、そのパフォーマンスへの影響は無視できます。



シンプロビジョニングとは、ストレージコストを削減することを意味します。 管理者は、物理容量を超えた場合でも、アプリケーションに大きな仮想ボリュームを割り当てることができます。 実際、VMは現時点で必要なだけのディスク容量を使用しています。 システム自体は、すぐに追加の容量を購入する必要があることを管理者に警告します。 もう1つの便利な機能は、ディスク障害またはRAIDレベルの変更後のRAID再編成の過程で、すでに割り当てられたブロックのみがコンソールに関与している場合のRAIDの「シンリビルド」です。 これにより、この手順の時間が大幅に短縮されます。



また、HP StoreVirtualラインとは異なり、MSAは動的割り当てだけでなく、容量割り当て解除（T10-Unmap）もサポートしていることに注意することも重要です。ファイルシステムからファイル削除コマンドを受信すると、システムはデータブロックを解放します。 シンプロビジョニングを効果的に使用するために重要です。



SSDキャッシング

MSA 2040で使用可能なSSD読み取りキャッシュ機能（MSA 1040はSSDをサポートしていません）は、読み取り時にソリッドステートドライブにデータをキャッシュできることを意味します。 MSAは、読み取りキャッシュとして最大1.6TBの容量の最大2台のSSDドライブのインストールをサポートします。 SSD読み取りキャッシュは、ランダム読み取りのストレージパフォーマンスを向上させます。 ラボでは、読み取り操作中、ディスクシステムの応答時間が80％に向上します。これは、仮想環境の顕著な加速に反映されます。



自動階層ストレージ

この機能により、高価なSSDドライブの使用を最適化することができます-頻繁にアクセスされる「ホット」データ、つまり、読み取り/書き込み操作が最も多いデータを保存します。 自動階層化は、ホットブロック（4 MBページ）を高速SSDに移動し、それらの操作を高速化します。 「アーカイブのティアリング」も同様に機能します。めったに使用されない（「コールド」）データがSASディスクから低速のMDL SASディスクに転送されます。 階層化は、システムのコストとパフォーマンスの最適なバランスを見つけるのに役立ちます。 HPによると、自動階層化により、ギガバイトあたり70〜80％のコストが節約されます。 ティアリングと読み取りキャッシュは、データを操作するアプローチの補完的なテクノロジーです：読み取りキャッシュはほぼ瞬時に機能し、いわゆる「アクティビティのバースト」の期間中に作業をスピードアップし、生成された一時レポートシステムに基づいてブロックを引き裂きます。





階層化ストレージレベルとして、異なるクラスのSASディスク（MSA 1040）またはSASとSSD（MSA 2040）を使用できます。



ワイドストライピング

この機能は、実際にはMSA内のストレージリソースの仮想化を意味します。 既存の物理ディスクをストレージプールに整理し、その容量を使用して仮想ディスク（Vディスク）を作成できます。 この場合、Vディスクはプール全体に分散できます。 仮想ディスクあたり16個の物理ディスクという以前の制限がなくなり、ボリューム容量が増加しました。 仮想ディスクを物理ドライブに分散することにより、パフォーマンスが向上します。 仮想ボリュームは、プールで使用されるすべてのディスクのディスクリソースを使用できます。





ワイドストライピングは、Vディスクボリュームデータをすべてのディスクに分散することにより、ディスク使用率を改善し、応答時間を短縮します。



データスナップショットメカニズムの改善

より効率的なスナップショット機能により、管理が簡素化され、バックアップパフォーマンスが向上し、より複雑な状況での災害復旧が可能になります。 新しいリリースのスナップショットは、リダイレクトオンライトの原則に基づいて動作するようになりました。これにより、従来のコピーオンライトに比べて操作のパフォーマンスが向上します（バックエンドレベルでのI / Oが少なくて済みます）。 スナップショットのセットアップの簡素化。





新しいMSAファームウェアバージョンには、ソフトウェア開発者がバージョンをデバッグするために使用する「スナップショット」を作成する機能があります。



Web UIと高度な管理の改善

管理インターフェイスが更新され、HTML5がプラットフォームとして使用されるようになりました。 新しいインターフェイスは、特にVMの数の急激な増加によって管理が複雑になる可能性のあるサーバー仮想化環境で、管理を簡素化および高速化し、エラーの可能性を減らします。 並べ替え、検索、フィルタリング、およびグループ化機能により、多数のホストとボリュームの管理が容易になります。 VMware vCenterなどの一般的な仮想環境管理システムからMSAアレイを直接管理することもできます-論理ボリューム（LUN）の作成、サーバーへの割り当てなど。

Microsoft Exchangeの例での仮想化



Microsoft Exchange仮想化は何をしますか？ このような環境は、リソース（メモリやプロセッサパワーなど）を仮想マシンに割り当てることで簡単に拡張できます。 機器のより効率的な使用により、電力、冷却、ストレージインフラストラクチャ、ストレージ管理などの運用コストが削減されます。 システムの信頼性の向上：仮想化により、管理者はデータベース可用性グループ（DAG）を作成し、最小限のハードウェアリソースでExchangeデータベースレプリケーションツールを使用できます。 Exchangeアーキテクチャ全体のフォールトトレランスレベルを構成し、システムコンポーネントの障害から保護できます。 さらに、ユーザーをメールサーバーから切断せずにソフトウェアを更新できます。





Exchange仮想環境を展開するためのリファレンスアーキテクチャの例（正面図と背面図）：

1. 2台のHP ProLiant DL380p Gen8サーバー。

2. Exchangeデータベースおよびログ用に割り当てられた8つの2 TB SAS LFFディスクを備えたHP MSA 2040のアレイ。 残りのディスクはバックアップとして使用されます。

3.プライマリおよびセカンダリ接続FC 16 Gb / s。

4.プライマリおよびセカンダリSAS 6 Gb / s。

5. 8個のディスク2 TB SAS LFFを備えたディスクシェルフHP MSA 2040 LFFは、Exchangeデータベースおよびログに割り当てられています。



HP MSA 2040は、ディスクシェルフを接続することで、ユーザー数の増加に合わせて拡張できます。 アクティブ/アクティブモードで動作する2つのシステムコントローラー、デュアルポートドライブ、および冗長コンポーネントにより、ストレージ障害の可能性を最小限に抑えます。 HP MSA 2040では、さまざまな接続オプションを選択して、SAN構成の要件を満たすこともできます。 組み込みツールにより、アレイの展開と管理が簡素化されます。



Exchange 2013仮想化ガイドラインは、 Microsoftで入手できます 。



Microsoft Hyper-VおよびHP MSA 2040アレイを使用してMicrosoft Exchange 2013を仮想化する場合、同じサーバーで追加の仮想化アプリケーションをホストできます。 たとえば、 HP ProLiant DL380p Gen8サーバーは、Exchangeの負荷だけでなく、他の仮想サーバーにも対応できます。 物理サーバーの数を減らすと、管理が簡素化され、サポートのコストが削減されます。



HP ProLiantおよびHP MSAストレージ上のMicrosoft Exchangeは、費用対効果が高く、信頼性が高く、スケーラブルなメッセージングソリューションです。 Microsoft Windows Server 2012 Hyper-V仮想環境では、HP MSA 2040ディスクアレイを備えたProLiant DL380p Gen8サーバーは、約750個のExchange 2013メールボックスをサポートできます。



適切な使用例は、SQLサーバー2012用のMSA 2040アレイの使用です。この場合、MSA 2040アレイは 、シーケンシャルおよびランダムの読み取りおよび書き込みロードで動作することが実証されています。
