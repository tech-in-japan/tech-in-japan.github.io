Microsoft Azure Media Services-主要なプラットフォーム機能の概要

スマートフォンのウェブサイトまたはプレーヤーにアクセスする前に、ストリーミングビデオとオンラインブロードキャストはいくつかの重要な処理ステップを経ます。 写真が正しく表示され、最大数のデバイスとプラットフォームで機能するために必要です。 この処理は、特別なサービスを使用して行われます。









Azure Media Servicesは、クラウドからSaaS（サービスとしてのソフトウェア）およびPaaS（サービスとしてのプラットフォーム）形式で利用可能なMicrosoft Media Platformテクノロジーサービスです。 それらを使用する利点の1つは、コンテンツをエンコード、ブロードキャスト、および配信する独自​​の手段と組み合わせて使用​​するだけでなく、拡張できることです。 このプラットフォームでは、コンテンツのダウンロード、コードの変換と変換、ライブブロードキャストの整理、スケジュールされたブロードキャスト、Webサイトへのビデオの埋め込みも可能です。 また、ユーザーはコンテンツ保護にアクセスして、広告と分析を操作できます。



これらおよび他の多くの便利な機能により、Azure Media Servicesは非常に人気のあるプラットフォームになりました。 たとえば、2016年のリオデジャネイロでのオリンピックの放送に使用され、Azure Mediaは世界で最もタイトルのあるフットボールクラブの1つであるReal（マドリード）を使用しています。



インターフェース



[ブラウザー]タブでAzure Media Servicesのすべての機能を使用します。 アカウントにログインしてサービスを開始するだけです。 インターフェイスは非常にシンプルです-左側のメニューバー、上部のアドレスバー、通知と設定パネル、中央のワークスペース。 たとえば、次のスクリーンショットでは、ビデオのトランスコードと処理のプロセスを確認できます。







マルチメディアを使用する



特定のサイトにビデオが表示される前に、いくつかの準備段階を経ます。 すべては、ソースビデオのサーバーへのダウンロード、処理、変換などから始まります。これらおよびその他の多くのアクションのために、Azure Media Servicesには特別なREST API、.NET、およびJava SDKツールがあります。 Microsoftが開発したクライアントツールも多数あります。 この広範なツールキットを使用すると、広告の巻き戻しや追加など、ビデオの準備と再生のあらゆる側面をカスタマイズおよび制御できます。 これらすべての段階について順番に簡単に説明します。



読み込み中



Webインターフェースを介してさらに処理するために、ファイルをAzureクラウドに送信できます。 これを行うには、[リソース]項目を選択し、[送信]ボタンをクリックします。 次に、コンピューターからファイルを選択し（ビデオ、オーディオ、画像、サムネイルコレクション、テキストチャンネル、クローズドキャプションファイル、およびこれらのファイルのメタデータをダウンロードするため）、クラウドに送信する必要があります。 同時に、ファイルサイズに制限はありませんが、安全なクラウド環境に保存され、その後の変更やストリーミングに使用できます。



.NETおよびRESTツールを使用して、リソースをクラウドに送信することもできます。 この場合、データはすぐに暗号化でき、他の機能も利用できます。 詳細と例は、MicrosoftのWebサイトで見つけることができます。



多くの大きなファイルを送信する必要がある場合は、.NETライブラリの一括アップロード機能を使用することもできます。 別の方法— Asperaなどのサードパーティツール—高速ファイル転送用のソフトウェア。 この製品は、Azure Marketplaceで個別に購入する必要があります。 製品を登録し、ブラウザとクライアント用のモジュールをインストールし、ツールをセットアップした後、大量のファイルをクラウドに簡単に送信できます。 Asperaの詳細については、 こちらをご覧ください 。



Azure StorSimpleを使用してアカウントにファイルを送信することもできます。AzureStorSimpleは、オンプレミスおよびAzureクラウドでタスクを管理する統合ストレージソリューションです。 Azure StorSimpleは、オンプレミスソリューションの拡張機能としてクラウドストレージを使用し、データを自動的にリンクします。 ファイルをクラウドに送信する前に、StorSimpleはそれらを複製して圧縮するため、大量のデータを効率的に転送できます。 Azure StorSimpleの操作の詳細については、 こちらをご覧ください 。



エンコード



Windows Azure Media Servicesには、ファイルをさまざまなマルチメディア形式に変換するためのツールがあります。 システムは、FLV、MXF、GXF、MPEG2、3GP、Windows Media Video、Microsoft Digital Video Recording、AVI、MP4、Matroska / WebM、WAVE / WAV、およびQuickTimeをサポートしています（サポートされているすべてのビデオおよびオーディオコーデックを参照してください ）。



競合他社に対するAzure Media Servicesプラットフォームの主な利点の1つは、動的コンテンツパッケージを使用することです。 クラウド内の各ファイルは1つのバージョンで保存され、各プラットフォーム（Android、iOS、XboXなど）のファイルのコピーを保存する代わりに、エンドデバイスに配信する直前に再コーディングされます。 これにより、クラウドのスペースを大幅に節約でき、その結果、お金を節約できます。



マルチメディア処理速度は手動で調整できます。 このようなタスクには、S1、S2、およびS3という値が割り当てられます。 たとえば、予約ユニットS2のタイプを使用する場合、エンコードタスクはタイプS1が選択されているタスクよりも高速です。 アカウントの種類によって、同時処理タスクの数は異なります。 たとえば、アカウントに5つの予約ユニットがある場合、同時に実行される5つのタスクを実行できます。 残りのタスクは、配信された場合、順番を待って、スレッドの1つが処理されるとすぐに、キューからのタスクが自動的に実行されます。 エンコードの詳細については、MicrosoftのWebサイトを参照してください 。



ターンテーブル



Azure Media Servicesのもう1つの際立った機能は、ブランドプレーヤーの汎用性です。 このサービスは、MacおよびPC、iOSおよびAndroid、HTML5、Flash、Silverlight、XboXなど、ほぼすべての最新のデバイスおよびプラットフォームをサポートします。独自の開発フレームワークを備えたさまざまな組み込みデバイスを使用するには、Smooth Streaming Clientの移植用パッケージ移植キットとMicrosoft PlayReadyデバイス移植キット。 そのようなデバイスの例は、メディアプレーヤー、スマートテレビなどです。







プレーヤーは、アダプティブストリーミングに最適な業界標準のMedia Source Extensions（MSE）およびEncrypted Media Extensions（EME）も使用します。 デバイスまたはブラウザーで使用できない場合、プレーヤーはFlashとSilverlightに切り替わります。 これらのオプションは、プレーヤーの詳細設定で、ドロップダウンリストから適切なアイテムを選択することで設定されます。 さらに、ブロードキャストでは、上記の自動選択機能と特定のテクノロジーの使用の両方を指定できます。







Microsoft Azure Media Servicesは、コンテンツを再生するためのDASH、スムーズストリーミング、およびHLSストリーミング形式をサポートしています。 Azure Media Playerはこれらのさまざまな形式を考慮し、プラットフォームまたはブラウザーの機能に応じて最適なリンクを自動的に再生します。 ただし、特定の形式で作品を指定することもできます。 開発者が選択するテクノロジーとプラットフォームが何であれ、彼はAPIにアクセスするための単一のJavaScriptインターフェースを取得します。 これらすべてにより、Azure Playerは、幅広いデバイスやブラウザーで再生するコンテンツを提供できます。 簡単に言えば、ある種のデバイスではビデオが機能しない、または正しく表示されないという心配はありません。







翻訳者が不正アクセスからリソースを暗号化する場合、PlayReady、Widevine、または128ビットAES暗号化アルゴリズムを使用してこれを行うことができます。 行う必要があるのは、プレーヤーの設定で対応するアイテムを確認し、必要なマーカーを示すことです。 詳細については、Microsoft Webサイトをご覧ください 。





128ビットAES暗号化スキームおよびその他のAzure Media Servicesツール



選択したすべてのオプション、値、およびパラメーターを適用して、[プレーヤーの更新]ボタンを押すように設定します。 その後、リソースに挿入するために指定されたすべてのパラメーターを含むムービーコードを取得できます。







ブランドコネクタ



Azure Media Servicesは、サードパーティのサービスと統合するための便利で使いやすいコネクタを提供します。 Dropbox、Facebook、Twitterなどのサービスやプラットフォームで利用できます。 コネクタを使用すると、さまざまなデバイスでのビデオの準備と配置に伴う問題を心配することなく、アプリケーションとサービスでビデオとマルチメディアの資料を準備して投稿できます。 使用できる場所の例を次に示します。



- ビデオが重要な要素であるアプリケーション 。 たとえば、これは教育用アプリケーションです。

- ビデオをベースにした垂直型エンタープライズソリューション 。 例として、学習管理システムまたはEnterprise Tubeなどのサービス。

- ビデオ付きのWebサイト 。 さまざまなビデオコンテンツでマーケティングWebサイトを作成したい人向け。

- ライブストリーミングソリューション 。 クライアントが幅広い聴衆を獲得したいところならどこでも、会議、調査、ウェビナー用のアプリケーションになります。







結論



Azure Media Servicesは、ビデオやその他のマルチメディアデータを処理してデバイスに配信するためのシンプルで便利なクラウドサービスです。 さまざまなデバイスで公開するために、数回クリックするだけでビデオを準備できます。また、さまざまなオペレーティングシステムのデバイスで、このビデオが正しく再生されることを確認できます。
