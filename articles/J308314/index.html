Windows Server 2012 R2をインストールし、その後200の更新を取得しない方法

Windows Server 2012 R2は2013年10月18日にリリースされました。 それ以来、製品の脆弱性と欠陥を修正し、機能を改善するために、このMicrosoftサーバーオペレーティングシステムで数百の更新がリリースされました。



膨大な数の更新-頭痛の種 。 2014年11月のアップデートが統合されている最新のサーバー配布、いわゆる「Update2」は、絶望的に古くなっています。 それからオペレーティングシステムをインストールすると、2〜4時間後にインストールされる別の200以上の更新プログラムを取得できます。



この短い手順では、すべての累積的な更新パッケージとセキュリティ更新プログラムを統合して、11月の配布を更新します。



配布キットに加えて、インストールメディアの更新方法、各ステップが実行される理由、さまざまな段階でどのようなトリックが待ち受けているかを思い出して、管理者の記憶を更新します。



標準ツールを使用して、できるだけ簡単にそれを行います。



すべての作業は、Windows Server 2012 R2が既に展開されているサーバーで行うのが最適であるため、 DISMユーティリティのバージョンと重複することはありません。 ISOファイルを展開せずに接続することも便利です。



作業ディレクトリを準備します



操作には次のディレクトリが必要です。



ISO-配布ファイルはこのディレクトリにコピーされます。 その中に、イメージをマウントしてからアンマウントした後、配布キットSW_DVD9_Windows_Svr_Std_and_DataCtr_2012_R2_64Bit_English_-4_MLF_X19-82891.ISOの内容をコピーします。









MOUNT-空のディレクトリ。wim-fileのイメージがその中にマウントされます。

CU-このディレクトリに累積的な更新を配置します

SU-このディレクトリには、セキュリティ更新およびその他の更新が含まれます 



mkdir D:\WS2012R2\ISO mkdir D:\WS2012R2\MOUNT mkdir D:\WS2012R2\CU mkdir D:\WS2012R2\SU


累積的な更新をダウンロードする

セキュリティ更新プログラムをダウンロードする



累積的な更新に加えて、WSUSオフライン更新ユーティリティがダウンロードできる更新を統合します。

これを行うには：



  1. プログラムdownload.wsusoffline.netをダウンロードします
  2. Windows Server 2012 R2の更新の選択







  3. ダウンロード後、wsusoffline \ client \ w63-x64 \ glbディレクトリを開き、* .cabファイルをC：\ WS2012R2 \ SUディレクトリにコピーします


注意：KB2966828またはKB2966827がリストにある場合は削除してください。削除しないと、インストール後にNet Framework 3.5コンポーネントを追加できません（ 詳細 ）。



更新の準備ができました。統合を始めましょう。



統合の更新



更新を統合するには、次のものが必要です。

  1. install.wimのいずれかのイメージのコンテンツをマウントします 



     dism /mount-wim /wimfile:{  wim } /Index:{N} /mountdir:{   }


  2. すべての更新をオフラインインストールに統合する 



     dism /image:{   } /add-package /packagepath:{  *.msu  *.cab }


  3. 変更を保存 



     dism /Unmount-WIM /MountDir:{   } /Commit




このプロセスは、次のバッチファイルによって簡単に自動化できます。 



 for /l %%N in (1,1,4) do ( dism /mount-wim /wimfile:"D:\WS2012R2\ISO\sources\install.wim" /Index:%%N /mountdir:D:\WS2012R2\MOUNT for %%f in (D:\WS2012R2\CU\*.*) DO (dism /image:D:\WS2012R2\MOUNT /add-package /packagepath:%%f) for %%f in (D:\WS2012R2\SU\*.*) DO (dism /image:D:\WS2012R2\MOUNT /add-package /packagepath:%%f) dism /unmount-WIM /MountDir:D:\WS2012R2\MOUNT /Commit )




ヒント：バッチファイルを実行する場合、出力をログにリダイレクトします 

 UpdateWIM.cmd >>log.txt 2>>&1




その結果、ファイルDが取得されます。\ WS2012R2 \ ISO \ sources \ install.wimのサイズは6.15Gbです。 削減できますか？ はい、エクスポートを使用すると、サイズが5.85Gbの最適化されたファイルを取得できます。



節約量はわずかですが、さらに、インストール中にオペレーティングシステムを選択するためのダイアログは見栄えがよくないため、次の手順はオプションです。



何をする必要がありますか？
圧縮されたイメージを取得するには、以下を行う必要があります。



  1. 最初の画像を元のwimファイルから新しいファイルにエクスポートします 

     dism /export-image /sourceimagefile:{    wim} /sourceindex:1 /destinationimagefile:{    wim} /compress:max
  2. 元のwimファイルからマウントポイントに次のイメージをマウントします 

     dism /mount-wim /wimfile:"D:\WS2012R2\ISO\sources\install.wim" /Index:2 /mountdir:{   }
  3. 「キャプチャ」メソッドを使用して、新しいファイルに次の画像を追加します 

     dism /append-image /imagefile:{    wim} /captureDir:{   } /name:{ } /description:{ }
  4. 画像をアンマウントし、追加の画像ごとに追加を繰り返します 

     dism /unmount-WIM /MountDir:{   } /Discard


スクリプトを自動化します。

コンパクトなinstall.esdイメージを作成する
D1abloRUSのアドバイスに基づいて 、 DVD5に適合する最小のインストールディスクを取得したい場合は、1つ（1つだけ）のイメージをesdファイルにエクスポートできます。 たとえば、Windows Server 2012 R2 Standardをエクスポートするには、次のコマンドを使用します 

 dism /export-image /sourceimagefile:D:\WS2012R2\ISO\sources\install.wim /sourceindex:2 /destinationimagefile:D:\WS2012R2\ISO\sources\install.esd /compress:recovery


元のinstall.wimは削除できます。

ビルドするには、 Windows ADKの oscdimg.exeユーティリティが必要です。 お持ちでない場合は、 リンクからユーティリティをダウンロードできます（このキットのユーティリティ以外のものは使用しないでください）。

ヒント＆トリック＃4。 install.wimが4700Mbよりも大きい場合、インストーラーはエラー「Windowsは必要なファイルDを開けません：\ sources \ install.wim」でクラッシュします。 エラーコード：0x8007000D "。



人生は戦いであると教えられたので、コマンドを使用して元のinstall.wimを2つに分割しました 



 dism /split-Image /imagefile:D:\WS2012R2\ISO\sources\install.wim /swmfile:D:\WS2012R2\ISO\sources\install.swm /filesize:4096


元のinstall.wimファイルは削除できます。
