OS X Serverで企業のiOSデバイスを管理し、社内でアプリケーションを配布します





遅かれ早かれ、大企業の大部分では、社内のモバイルアプリケーションを開発するという疑問が生じます。 この点で、ITスペシャリストは、従業員の個人用デバイスにアプリケーションをインストールする必要がある場合と、従業員が特定のタスクを実行するために会社の所有物であるデバイスを配布する必要がある場合の2つのシナリオを解決するタスクに直面しています。 この記事では、OS X Serverを介したiOSデバイスの操作について説明します。





はじめに





現在、ご希望であれば、OS X Server、MDMソリューションなどの構成に関する情報を簡単に見つけることができます。 英語では、一般に、構成は複雑さを示しません。 この記事は、最初にこの問題に遭遇し、何が彼らを待ち受けていて、すべてがどれほど複雑で怖いかを理解したいロシア語圏の人々を対象としています。

提示された資料は、アクションの簡単なガイドとしてまとめられ、不要なものとしてゴミ箱に直行しましたが、「空のゴミ箱」をクリックする前に、このトピックに精通していない他の人に役立つと思いました。

したがって、コメンテーターの別のカテゴリーをconめてください






問題の声明





そこで、会社（または顧客）が設定したタスクを定義しましょう。



モバイル方向の2つの独立した開発ベクトルがあります。

  • 個人のデバイス、この場合はiOSデバイスにインストールする、従業員向けの企業モバイルアプリケーション。 たとえば、主な機能はニュースやイベントのカレンダーを読むことです。
  • 特定のカテゴリの従業員、たとえば路上で投票を行う社会学者向けの高度に専門化されたモバイルアプリケーション。 彼らはこれらのタスク専用のiOSデバイスを与えられており、もちろん、管理者はこれらの人々の意識を期待しながら、労働時間とデバイスの両方の誤用の可能性を排除したいと考えています。 簡単に言えば、音楽を聴く、インターネットを使用する、アプリケーションストアからゲームをインストールするなど、すべてを禁止する必要があります。ソーシャルイベントにはアプリケーションを使用する機能のみを残します。 世論調査




教育プログラム





これら2つの分野は異なるという事実にもかかわらず、それらには共通点があります。つまり、企業内でのアプリケーションの配布です。 Apple Developer Enterprise Programの下で。 訓練を受けていない開発者にとっては、これは単に開発会社を法人として登録し、いくつかのエンタープライズガジェットを使用しているように見えるかもしれません。 残念ながら、プログラムの本質は文字通り「Apple AppStore：エンタープライズアプリを配布する場所がない」、つまり このプログラムを入力すると、AppStoreをバイパスしてiOSアプリケーションを配布できますが、公式AppStoreに公開する機能は失われます。



それでは、Appleの目を通してだけでなく、企業のモバイルアプリケーションの普及の問題をもう少し広く見てみましょう。 今日の市場には、Google、Apple、Microsoftの3つの主要なプレーヤーがいます。

そのため、これらの各テクノロジー企業は、ストアを迂回するアプリケーションの配布を見ています。



グーグル

ここではすべてが簡単です。 Androidはオープンなオペレーティングシステムです。 アプリケーションパッケージ（* .apk）をビルドしてから、必要な処理を行います。 Google Playデベロッパープログラムに1回限りの料金を支払うだけです。



マイクロソフト

2つのプログラム。 公式ストアでの公開-少額の1回限りの料金。 ストアをバイパスして公開するには、1年に1回特別な証明書を購入する必要があります。これは、アプリケーションに署名するエンタープライズモバイルコード署名です。 大まかに言えば、スマートフォンのアプリケーションガレージへのアプリケーションのパスのようなものです-「私はVIPです。バックドアからインストールできます」。



りんご

次に、クパチニアンは、モバイルアプリケーションの開発へのアプローチを次のように考えています。

公式AppStoreストアでの公開には、個人と法人の両方が参加できるApple Developer Programがあり、価格は同じです。 社内でアプリケーションを公開する必要がある場合、つまり 皆のためではなく、AppleはApple Developer Enterprise Programを提供していますが、Apple StoreではAppStoreでアプリケーションを公開できなくなります。

しかし、両方のシナリオでアプリケーションを公開する必要がある場合はどうでしょうか？ または、法人を登録する方法はありませんが、プロトタイプの作成を始めたばかりで、個人を登録できます。 1人の開発者。

この場合、従来のApple Developer Programは、開発者のコ​​ンソールに登録されているさまざまなタイプ（iPhone、iPad）の100台のデバイスにアプリケーションを直接アップロードする機能を提供します。 科学的には、これはAdHocと呼ばれます。 テスト目的の配布。

奇妙なことに、このことは初期段階で問題の層全体を解決することが多いので、この記事のフレームワークで説明します。



さて、プロトタイプのモバイルアプリケーションがあり、Apple Developer Programに開発者アカウントがあり、問題を解決したいと強く望んでいます。



それでは、iOSデバイス管理モデルに移りましょう。 以前にiPhoneまたはiPadを使用したことがある場合は、おそらくプロファイルなどについて聞いたことがあるでしょう。 いいえ、SharePointのユーザープロファイルではなく、デバイスの権限を説明するプロファイルです。 たとえば、ベータ版のiOSへのアクセス。 つまり、このことに関しては、アプリケーションのインストール、デバイス機能の制限、デバイスの構成など、すべてがAppleの世界で展開しています。 おそらく既にご存知のとおり、これらのプロファイルを操作します。 誰かと自発的に、誰かと強制的に。





Apple Configurator 2





企業環境でiOSアプリケーションを配布するには、2つのツールが必要です（AppStoreから）：

  • Apple Configurator 2アプリケーション（ 無料 ）;
  • OS X Serverアプリケーション（ 1.490ルーブル ）;


どちらのツールもプロファイルを管理できますが、ユーザーはApple Configurator 2によって設定されたプロファイルのみを削除できません。

つまり すべてが制御されている、すなわち




プロファイルのリモート構成には、その構造に含まれるOS Xサーバー、またはむしろプロファイルマネージャーが必要です。 Apple Configurator 2は「コード上」でのみ機能します。



スキームを明確に理解できるように、プロファイル（基本的にはあちこちにある設定ファイル）を通じて、アプリケーションのリモートインストールとデバイスの設定が実行されます。 Apple Configurator 2は、インストールされたプロファイルがユーザーによって削除されないようにし、OS X Serverのプロファイルマネージャーを使用すると、インストールされたプロファイルをリモートで構成できます。 最初の近似では、画像はこれです。



ほんの一握りで考えを収集したので、不足している要素をさらに2つパズルに追加する必要があります。 プロファイルに加えて、Apple Configurator 2によって提供される監視と呼ばれるメカニズムに別のリンクがあります。これは、デバイスを工場出荷時設定にリセットすること、プロファイルマネージャーを使用するためのハード構成、およびデバイスのハードリセットの禁止にあります。 また、メモリのイメージを取得する機能、つまりバックアップをサポートします。



そして最後に、あなたはすでに考えたと思いますが、それでは普通の従業員はこれらのひどい放電と設定なしで企業アプリケーションをどのように取得できますか？ プロファイルマネージャーは、デフォルトで「MyDevices」と呼ばれるWebサイトを提供します。このWebサイトから、スマートフォン上の必要なアプリケーションを自動的に構成するプロファイルをダウンロードできます。 はい、これは普通の従業員にとって非常に簡単です。

Apple OS Xサーバー





OS X Serverは、2014年までApple Mac miniコンピューターの上位バージョンにプリインストールされたアプリケーションです。 現在、AppStoreを介してスタンドアロンアプリケーションとして配布されており、任意のAppleコンピューターにインストールできます。

これは、Appleのデバイス群にサービスを提供するためのサービスのセットであり、iOSおよびOSXデバイスのリモート管理でもあります。

AppStoreで見る




主な機能：



  • Open Directoryのフレームワーク内でのユーザー管理（Active Directoryとの統合が可能）。
  • カレンダーを同期する
  • 連絡先の同期
  • ファイル共有;
  • メールサーバー;
  • グループユーザーチャット。
  • iOSおよびOSXデバイスのリモート管理用のプロファイルマネージャー。
  • TimeMachine Server（バックアップ）;
  • VPN
  • PHPおよびPythonでのWebサイトホスティング。
  • CMS Wiki
  • 継続的な統合のためのXcodeサーバー。
  • DHCPサーバー。
  • DNSサーバー。
  • FTPサーバー。




OS XサーバーをActive Directoryと同期する





もちろん、私たちは企業セグメントで働いているため、Active Directoryを過去に取得することはできませんでした。 Microsoftと同様に、AppleはディレクトリサービスであるOpen Directoryを使用します。OpenDirectoryは、概念的にはActive Directoryと変わりません。

Active Directoryとの統合には、バインディングメカニズムが使用されます。 非常に簡単に構成されますが、鉄の安定性に違いはありません（Windows ServerとOS X Serverのバージョンの組み合わせが異なると、接続が失われる可能性があります）。

モバイルデバイス管理サーバー



プロファイルマネージャーを使用すると、OS X 10.7以降またはiOS 4以降を実行している登録済みのモバイルデバイスを集中管理および管理できます。



Appleは、経験を活用するために、MDMソリューションのサードパーティ開発者にデバイスのAPI管理を提供していることに注意してください。



通常、これらのモバイルデバイス管理（MDM）システムには、カスタマイズのためのオプションが多くあります。

サーバーが開始する設定のダウンロードを実装するには、ソフトウェアにプッシュサービスが必要です。 プッシュサーバーはOS X Serverの一部です（プッシュサービスはMDMソリューションの一部として実装される場合があります。この場合、Appleはプッシュサービスを必要としません）。 プッシュサーバーは、モバイルデバイスソフトウェアの構成またはインストールに関連するコマンドをロードする機能を実行し、Appleアプリストアと対話します。 これにより、盗まれたデバイスのブロックやデバイスからの情報の削除などの緊急のアクションが実行されます。



Appleデバイスのみを管理する予定で、特別な要件がない場合は、プロファイルマネージャーがコスト/機能の面で最適なソリューションです。 この場合、Apple Push Serverを使用して、エンタープライズソフトウェア設定を集中的にダウンロードします。 また、企業ネットワークの境界を越えて接続されたデバイスにポリシーをインストールする機能は、Apple Push Notification Service（APNS）を提供します。
