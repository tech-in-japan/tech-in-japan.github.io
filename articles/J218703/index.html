2014年4月8日以降のWindows XPでのMicrosoft Security Essentialsの継続







今日、Windows XP上のMicrosoft Security Essentialsウイルス対策の中止に直面しました。 ご存知のとおり、2014年4月8日にWindows XPのサポートが終了したため、Microsoftはこのシステムのウイルス対策も無効にすることを決定しました。 このシステムでMSEが今後1年間機能するというメッセージは実現しませんでした-ディストリビューションとして利用可能なバージョン4.4.304.0は機能しますが、セキュリティアップデートで自動的にインストールされる新しいバージョン4.5.216.0は、ウイルス対策をオフにし、ユーザーに警告しますOSは時代遅れであり、それを取り除く時間です。 しかし、あまりにも多くの人がこのシステムとこの便利なアンチウイルスを使用しているため、アンチウイルスをさらに機能させるソリューションを見つけました。



問題の解決策は、ウイルス対策設定の文書化されていないキーでした。



[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Microsoft Antimalware\Signature Updates]

"ExpiryNotificationIntervalDays"=dword:ffffffff







現時点では、これはインターネット上で最初で唯一のソリューションであり、完全に合法であり、ユーザーがすばやくインストールできます。 4月9日以降、システム管理者は、ウイルス対策が機能しなくなったという多くの呼び出しを期待しており、ウイルス対策を交換せずに迅速な解決策が必要です。 この調整を行うには、レジストリブランチに権限を設定する必要があることに注意してください。デフォルトでは、それへの書き込みは禁止されています。 また、 ここで私のソリューションを使用することもできます。この場合、このキーはアクセス権を変更せずにレジストリに書き込まれ、ディストリビューション自体は任意のアーキテクチャのWindows XP SP2（必要なアップデートが含まれています）以降にインストールされます。
