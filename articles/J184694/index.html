JelasticのワイルドカードSSLとは何ですか？どのようにお金を節約できますか？

少し前まで、REG.RUはJelasticホスティングプラットフォームに基づいてユーザーにクラウドサービスを導入しました。 それを使用することは非常に簡単であるという事実にもかかわらず、REG.RUテクニカルサポートスペシャリストは、多くの場合、Jelastic PaaSプラットフォームでの作業に関する明確な質問に出くわします。 現在、ロシア語でのWeb上の詳細な指示は非常に少ないので、REG.RUブログで顧客が最も関心を持っていることについて話すことがあります。



次に、投稿のトピックに直接移動します。



ワイルドカードSSLとは何ですか？



ワイルドカードSSLは、1つのドメイン名から複数のホストを1つのサーバーに一度に送信できるユニバーサルSSL証明書です。 これにより、サブドメインごとに個別のSSL証明書を購入する必要がなくなります。 そのため、Wildcard SSLは、大規模な組織や商用インターネットプロジェクトのサイトにとって理想的なソリューションです。



ワイルドカードSSLプロ



• コスト 。 ワイルドカードSSL証明書を購入すると、ユーザーはかなりの額を節約でき、従来のいくつかの証明書を購入する必要がなくなります。

• 時間を節約します 。 一度に20の異なるSSL証明書を設定する時間を無駄にする必要がなくなり、それらを更新するたびに設定する必要がなくなります。 1つのSSL証明書のみをインストールしてサポートする必要があります。



ワイルドカードSSL証明書は次のとおりです。



• 便利な管理 。 SSL証明書の設定プロセスは最大限に簡素化されており、特別な知識は必要ありません-すべては数回クリックするだけで実行できます。

• クイックチェック 。 Wildcard SSLのおかげで、サイトはできるだけ早くチェックされ、封印されます。

• 一般に受け入れられているデータ暗号化の標準 。 使いやすさと低価格にもかかわらず、セキュリティを犠牲にする必要はありません。 ウェブサイトの訪問者は常に可能な限り安全に感じます。



ワイルドカードSSLを接続する方法は？



1. REG.RU Webサイト（ https://app.jelastic.regruhosting.ru/）で登録するか、既存のJelasticアカウントにログインします。

2.次に、Jelasticコントロールパネルで、 「環境の作成」をクリックします：







3. 「セットアップウィザードモード」に入り、アプリケーションサーバー（Tomcat 7など）を選択します。

4. [ SSL ]ボタンをクリックして、[ Jelastic SSL] （ワイルドカードSSLの別名）に切り替えます。

5.環境の名前を入力し、[ 作成 ]ボタンをクリックします。







これですべてです。



次に「ブラウザで開く」ボタンをクリックすると、環境との通信がHTTPS経由で行われていることがわかります。









注 ：

•ワイルドカードSSLは、* .jelastic.regruhosting.ruのサイトでのみ使用でき、カスタムURLでは使用できません。

•環境内のノードでIPv4が有効になっている場合、ワイルドカードSSLは無効になります。







JelasticのワイルドカードSSLは、各サブドメインの保護に費やす時間とお金を節約します。



Jelasticプラットフォームに基づいたREG.RUサービスの詳細については、REG.RU Webサイトの「Jelastic Cloud Hosting」セクションを参照してください。
