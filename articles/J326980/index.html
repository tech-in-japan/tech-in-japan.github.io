SAMの公会計ソフトウェアツール

ソフトウェア資産管理マネージャーの役​​割を試してみる人、または単に組織内の既存のライセンスと実際に使用されているライセンスのコンプライアンスを計算しようとする人は、この記事が役立つと思うかもしれません。



ベンダー自身が提供し、ソフトウェアのみの使用を制御するように設計されたツールについてです。 このような制限は、これらのツールの「有用性」を低下させ、確かにマイナスになりますが、明確なプラスもあります-無料で提供されます。



ベンダーが提供するツールとその使用方法を見てみましょう。







IBM License Metrics Tool（ILMT）



IBM License Metrics Tool（ILMT）-PVU（Processor Value Units）、RVU（Resource Value Units）、および他のいくつかのタイプのライセンスを使用して、ライセンスの消費を制御できるツール。 このツールの制限は、AU（許可ユーザー）やCC（同時ユーザー）などのユーザーライセンスタイプの消費を測定できないことです。 この制限は、ILMTの将来のバージョンで削除される可能性があります。 このツールは、製造元のWebサイトからダウンロードできますが、IBMのお客様のみがダウンロードできます。



IBMパスポートアドバンテージ契約を結んでいる組織は、通常、サブキャパシティー方式を使用して、必要なライセンス権の数を計算します。 この方法により、複雑なコンピューティング環境のライセンスコストを最適化できます。 Incomplete Capacity方式を使用する場合、ILMTツールを使用することがIBMとのライセンス契約の前提条件になります。



ILMTを使用するには、別のソフトウェア製品であるIBM BigFixもインストールする必要があります。 ILMTは、IBM BigFixによって収集されたデータを処理します。 ILMTをサポートするためだけにIBM BigFixを使用する場合、IBM BigFixのライセンスを取得する必要はありません。 BigFixエージェントは、情報が収集される各デバイスにインストールされた組織にインストールされたIBMソフトウェアに関するデータを収集するために使用されます。 仮想化構造データは、オプションのVM Manager Toolコネクターを使用してIBM BigFixによって収集されます。



ILMTでは、ソフトウェア製品に対して取得した権利の数を考慮することはできませんが、取得したライセンスのリストを保存することはできます。 したがって、プログラム自体では、既存のライセンス権と使用済みのライセンス権を比較することはできません。



ライセンス権を考慮し、既存および使用済みの権利のコンプライアンスを計算するには、ソフトウェア資産管理（SAM）機能を備えた追加のソフトウェアを使用する必要があります。



MAP（Microsoft）ツールキット



MAP（Microsoft）Toolkit-組織のITインフラストラクチャを分析および評価するための一連のツール。エージェントプログラムを使用せずに動作します。 Software Usage Trackerは、MAP Toolkitでソフトウェアライセンスを管理するために使用されるレポートを生成します。



このユーティリティは、Windows Server、SharePoint Server、Exchange Server、SQL Server、およびSystem Center Configuration ManagerのMicrosoftソフトウェア製品に関するレポートを生成します。 これらの製品は、サーバーごと、プロセッサごと、ユーザーCAL、デバイスCALのメトリックでライセンスを制御します。 したがって、これらのソフトウェア製品について、サーバー、プロセッサ、およびクライアント（CAL）ライセンスのニーズを計算および評価できます。



MAPは、組織のライセンス権を考慮に入れ、使用されている権利と比較する方法を知りません。これには、追加のソフトウェアの使用も必要です。



使用ハブ（HPE）



Usage Hub（HPE）は、ライセンス管理のためにHewlett Packard Enterpriseが作成した特別なツールであり、「無料サービス」として位置付けられています。 Usage Hubは、HPEプログラムで使用可能なライセンス、有効期限、およびこれらのライセンスの使用量に関する情報を収集および表示できます。



Usage Hubは、エージェントプログラムを使用して情報の一部を収集し、一部をエージェントレスで収集します。 ライセンスとその有効期間に関する完全な情報を得るには、使用状況HUBとHPE AutoPassライセンスサーバーとの相互作用を追加で構成する必要があります。 Usage Hubとそのエージェントの両方をWindowsおよびLinuxオペレーティング環境にインストールできます。



このソリューションの紛れもない利点は、確立されたライセンスとその有効期間に関する情報の自動収集のおかげで、現時点と一定の履歴期間の両方で生成されたコンプライアンスレポートを受信できることです。 ただし、ここで予約する価値があります。UsageHubは、製品ドキュメントで明示的に示されているように、組織内のライセンスコンプライアンスの監査を実行することを目的としていません。



また、ソフトウェア資産管理（SAM）を編成する場合は、このアクティビティを自動化する他の手段が必要になります。
