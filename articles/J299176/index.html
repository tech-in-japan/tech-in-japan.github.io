4個人データに関する驚くべき事実

それでは、PDとは何ですか？ 個人データに関する以前の投稿では、かなりの数のコメントが収集されています。 論争のほとんどは、正確に個人データと見なされるものの問題を中心に展開されました。

これを理解し、約束を果たすと約束しました。





事実1.法律には特定のリストは含まれていません。



152-では、は、決定または決定された個人（個人データの対象）に直接または間接的に関連する情報を意味すると理解されています。

確かに、この定義からはほとんど理解されていません。



事実第2。PDには3つのタイプがあります



法律は、PDの3つのカテゴリ、一般、特別、および生体認証を区別しています。

  1. 一般的なカテゴリには、F。I. O.、住所、電話が含まれます。
  2. 法律は、人種、民族、政治的見解、宗教的または哲学的信念、健康状態、親密な生活に関連する特別なカテゴリデータを参照しています 。
  3. 生体認証データには、身体部分、指紋、手のひら、網膜、DNA分析などの構造的特徴が含まれます。


ファクトNo. 3. PDの実践



既存の司法慣行に基づいて、PDに起因する必要があるものを分析しましょう。

  • 姓、名、ミドルネーム、年、月、日、出生地、住所、婚 ital 、社会、財産状況、教育、職業、収入 （2010年5月10日のケース番号A15-2016 / 2009の解決。ロシア連邦最高裁の裁判長、決議ケース番号A36-5713 / 2014、2015年4月29日、19日AAS）。
  • パスポートデータ （たとえば、2014年5月22日のモスクワ市裁判所の控訴裁第33-14709号を参照）。

    パスポートのシリーズと番号が文書の形式を特定しているが個人ではなく、したがって個人データに関連していないことが示されている裁判所の決定があります（詳細については、2012年2月29日のモスクワ市裁判所の定義No. 33-6709を参照してください。 2010年6月21日、ケース番号A56-4788 / 2010の第13仲裁裁判所。

    シリーズとパスポート番号は個人を簡単に識別するため、このような結論に同意することは困難です。

  • 電子メールアドレス （たとえば、2015年5月26日の事件番号12-253 / 2015の判決を参照してください。カリニスキー地方裁判所（サンクトペテルブルク市）。

    PDとしての電子メールの分類に関するRoskomnadzorの慣行と意見はあいまいであることに注意してください。 多くの説明で、Roskomnadzorはフルネームを含む電子メールアドレスがPDとして分類されることを示しており、アドレスが文字（単語）のセットである場合、個人データとは見なされません。
  • 家の技術パスポートのデータ （たとえば、2014年4月28日のプリモルスキー地方裁判所の定義No. 33-3718を参照）。
  • 行政機関の公式ウェブサイトに掲載された法人および個人起業家の検査を実施する計画で示された個々の起業家の住所 （例えば、2014年4月24日のボルゴグラード地方裁判所の控訴裁定第33-4427 / 2014年を参照）。
  • 州境を越えることに関する情報 （例えば、2014年1月4日モスクワ市裁判所の控訴審判決第33-11688号を参照）。
  • 公式の手続きで提供されていない形式で配布された公式の登録住所、彼の収入と財産に関する情報（たとえば、2014年3月31日のサンクトペテルブルク市裁判所の判決33-4198 / 14を参照）。
  • 雇用契約で指定された従業員データ （例えば、サハ共和国（ヤクーチア）の最高裁判所の控訴審判決、2013年10月23日No. 33-4172 / 13を参照）。


データを個人として分類することに関して、さらに2つのポイントを理解することが重要です。



  1. データが特定の個人に関連していることを確認または確認する必要はありません（法律はそのような手順を規定していません）。 したがって、オペレータは実際にはこれが架空のデータであるかどうかを知りませんが、個人データを処理する義務を軽減しません。
  2. 個人データとは、個人を特定できるデータのみです（たとえば、2013年3月26日のサンクトペテルブルク市裁判所の判決第33-3815 / 13号、2014年1月28日のモスクワ市裁判所の控訴判決を参照N 33- 5461/14）。


事実4。より複雑な問題



名前や姓などの「単純な」データに加えて、IPアドレス、ニックネーム、ソーシャルネットワーク上のプロファイルなどの「複雑な」カテゴリと、個人データとしての定義に関して疑問が生じることがよくあります。

裁判所でさえ、これらのカテゴリの個人データの分類に関して異なる視点を持っています。



  • IPアドレスに関して、裁判所の判決の1つには、かなり良い法的分析が含まれています。「...通信事業者によって割り当てられ、サービス契約の締結時にエンドユーザー機器に永続的に割り当てられる静的IPアドレスによって個人データを設定することによる、情報および通信ネットワークインターネットのユーザーの識別インターネットへのアクセス（静的IPアドレスを使用する場合、すべてのユーザー接続は常に通信ネットワークのこのIPアドレスによって識別されます） その法的結果は、このデバイスをインターネットに接続している期間（セッション期間）に電気通信事業者によってユーザー機器に自動的にIPアドレスが割り当てられた場合（動的IPアドレス）と異なることはありません」（ケース番号2-5354 / 2015の決定2015年9月24日。サマラのオクチャブリスキー地方裁判所（サマラ地方））。



    同時に、一部の裁判所はIPアドレスをPDに関連付けません（たとえば、2015年6月1日のケース番号A56-75017 / 2014の決議を参照してください。13thAAS）、反対に、他の裁判所は（ケース番号A76- 29008/2015から2016年2月11日。ACチェリャビンスク地域。）。



    静的IPアドレスは、個人データを使用してユーザーを簡単に識別できるため、個人データにかなり起因しているようです。 しかし、オペレーターは静的IPアドレスを知ることができません。その場合、PDのすべてのIPアドレスを認識する必要があります（すべての消防士について）。
  • （メールまたはソーシャルネットワークからの） ログインとパスワードにより、紛争が少なくなります。 Roskomnadzorは、個人データに帰属させることは不可能であると繰り返し述べています。
