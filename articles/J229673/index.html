ハイリスク投資ツールの私の実装

まず、無料の企業ブログを作成してくれたgabrahabrに感謝します。







この記事では、3年かけて開発したツールについて説明し、2014年春にgithubに最終的に投稿しました。 それにより、お金を稼ぐことも失うこともできます。 ツールのスキームと同様に、すべてのリスクが考慮されます。

サーバーへのソースコードの配置はオプションになりました。ブラウザーのアドレスバーにアドレスを入力するだけで、他の人のノードを使用できます。



はじめに



このツールは、ユーザーのコンピューターに配置されたソースコードのセットであり、それらに設定されたアルゴリズムに従って、他のユーザーのコンピューターと対話します。 Dcoinツールと呼ばれます。 説明の代わりに記事が大きくなりすぎないように、wikiへのリンクを提供します。



Dcoinについて



これは 、Dcoinに関するHaberに関する以前の記事です。

ビットコインのように、各ノードがデータベースの完全なコピーを保存するように、Dcoinには中央サーバーがありません。 ノード間の新しいデータの交換は、 ブロックを使用して発生します 。 各ブロックには、前のブロックのハッシュが含まれています。 ユーザーからの各トランザクションには、そのユーザーからのトランザクションであることを証明する署名が含まれています。





ビットコインとは異なり、Dcoinには1つではなく66種類のさまざまなトランザクションがあります。Dcoinが提示された形で存在できるおかげです。

誰が次のブロックを生成してブロックチェーンに追加するかはどのように決定されますか
すべてのマイナーはレベルに分けられます。 マイナー、レベル0、つまり ブロックを生成し、他の誰かの前にブロックチェーンに追加する権利があります。これは次のように定義されます。 

$ctx = hexdec(substr($hash, 0, 6)); $hi = $ctx / 127773; $lo = $ctx % 127773; $x = 16807 * $lo - 2836 * $hi; if ($x <= 0) $x += 0x7fffffff; $leve_0_miner_id = (($ctx = $x) % ($max_miner_id + 1)); $leve_0_miner_id = ($leve_0_miner_id==0)?1:$leve_0_miner_id;


$ハッシュがsha256（sha256（user_id、block_id、prev_head_hash）））の場合、$ max_miner_idはminersテーブルの最大miner_idです



たとえば、miner_id 12689がレベル0の場合、ID 12690-12691（2）のマイナーはレベル1、12692-126995（4）のレベル2などになります。

0番目のレベルのマイナーがブロックを生成できなかった場合、1番目のレベルのノードが作業に含まれ、そうでない場合は2番目のノードなどが作業に含まれます。 変数sleepで定義されているレベル間に遅延があるはずです。 つまり sleep-> generator [0] = 120の場合、1番目のレベルのマイナーは、120秒が経過するまでブロックチェーンにブロックを追加できません。

同じレベルのマイナー間で、ブロックが最も低いハッシュで検索され、見つかったブロックがブロックチェーンに書き込まれます。



攻撃51％

レベルは、ブロックヘッダーに基づいて決定されます。ブロックヘッダーには、操作できる値はありません。 これは、特定のminer_idが0レベルにあることを確認することが不可能であることを意味します。 攻撃者が鉱夫の秘密鍵の51％を押収すると、51％の攻撃が可能になります。 DCネットワークに100,000人のマイナーがいる場合、攻撃者は50,000を超えるプライベートキーを制御する必要があります。

略語について



削減プロセスはどのように発生するのか、それを把握しました。 次に、このプロセスが開始されたときに何が起こるかを考えます。

DUSDの新しい現金交換リクエストにより、約束額が減少します。 DUSDの量が減ると、DUSDに現金を払いたい人はほとんどいません。

このような統計が表示されると想像してください。

DUSDボリューム 95万
約束金額 90万
履行された交換リクエスト 6％
あなたの残高： 100 DUSD
次の48時間で削減される可能性があります。 ほとんどの場合、着信要求を満たすことはできません。

次に、DUSDを$に交換するために、別のマイナー自体にリクエストを送信するかどうかを決定する必要があります。 さらに、システム全体の統計を見ると、彼がそれを達成する確率はわずか6％です。 そして最も重要なことは、このようなリクエストを送信すると、システム全体の状況が悪化し、削減が継続されるためです。 約束金額の交換に対する未処理のリクエストは、システム内の約束金額の合計を減らします。 DUSDを$に交換するリクエストの送信をほとんど停止すると、削減は停止します。 $をDUSDに変更することを拒否した人は、2週間ごとに約束額を更新でき、システム全体の統計で再び考慮されます。

100ドルを与えることにより、100 DUSDを受け取れないというリスクが残ります。 つまり あなたはリクエストを受け取り、リクエストの送信者に連絡し、Yandexのお金などで口座番号を示し、100ドルを送ったところ、彼はお金を受け取って応答を停止しました。

最も信頼できる方法は、 現金を直接dcoinに交換することです。 このため、Dcoinはワンタイムコードを使用します。 あなたはお金を与えますが、代わりにハッシュが既にあなたに知られているコードを受け取ります。 したがって、会議場所にインターネットがなくても、sha256（sha256（「コード」））を簡単に計算できます。ハッシュと一致すれば、すべてが整い、インターネットが表示されたら、コードを指定するだけです。オフィス。

しかし、金額が少なく、採掘者が否定的なレビューを持っていない場合は、個人的な会議なしでチャンスを取り、取引を行うことができます。
