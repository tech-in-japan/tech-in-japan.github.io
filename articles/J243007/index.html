非プログラマーがモバイルアプリケーションを作成した方法

こんにちは、Habr！



プログラマーではない私が、MS Visual Studio 2013のおかげで（またはそれに反して）最初のモバイルアプリケーションを作成し、それをWindows Phoneストアで公開する方法についてのストーリーを共有したいと思います。 私はこの分野でほとんど何もできない経験の浅い非プログラマーである読者を考慮に入れるようお願いします。そのため、問題を解決できるほどエレガントに問題を解決しなかった場合、特に私をock笑しないでください。



それはすべて、タリン工科大学のサイバーセキュリティコースで勉強しながら、Windows Formsの操作の一環としてVB.NETコースを受講したという事実から始まりました。 その後、MS Visual Studio 2013を使用して、ミステリアスなC＃またはJavaだけでなく、VB.NETでもモバイルアプリケーションを作成できることを誤って読んでしまいました。



「これはチャンスです-私もVB.NETのメガスペックです」と私は思いました。 学生ライセンスを緊急に使用して、MS Visual Studio 2013をダウンロードし、最初のアプリケーションを作成する謎に違反しました。



コースでは、最も重要なことはスキルではなくアイデアであると言われました。 スキルプログラマは多数いますが、成功するアプリケーションはほとんどありません。 同じRovioが数回破産し、Angry Birdsに着くまで何百もの役に立たないゲームをリリースしました。 ここでは、すべてが私にとっては問題ありませんでした-スキルはゼロでした（後で判明しました）が、長い間考えていました。 アイデアは簡単です-ゆるい鳥のように。

Windows Phoneの場合は、構文全体をゼロから学び、頭の中で1つのVB.NETと別のVB.NETを分割する必要がありました。 たとえば、VB.NETには、空間内のオブジェクトの座標を決定/設定するためのコマンドがあります。 Windowsフォームでは、次のようになります。
