自動更新


Operaには、新しいバージョンがリリースされたときに自動的に更新する機能が含まれています。

デフォルトでは、Opera 10.0 Alpha 1は利用可能なアップデートをユーザーに通知するだけです。 ユーザーは、opera：configのすべてのアセンブリをダウンロードするオプションを有効にすることで、利用可能な最新バージョンに自動的にアップグレードすることを選択できます。 「Opera update level。」の設定を変更することにより、設定（[設定]> [詳細]> [セキュリティ]）で更新のチェックまたは自動インストールを無効にすることもできます。



内部スペルチェック。


スペルエラーは、入力フィールド、Opera Mail、Notesで下線付きの赤で強調表示されるようになりました。 置換オプションは、入力フィールドのコンテキストメニューで使用できます。スペルチェックを有効/無効にし、辞書の言語を変更することもできます。 Opera 10.0 Alpha 1には、アメリカ英語（アメリカ英語）の辞書が含まれています。 デフォルトでは、複数行の入力フィールドに対してスペルチェックが有効になっています。

Opera 10.0はHunspell辞書形式を使用しています。

このリリースでは辞書のインストールが簡単になりますが、現在のところ、辞書ファイル（* .affおよび* .dic）をローカル辞書フォルダーにコピーすることにより、追加の辞書をダウンロードしてインストールできます。 Operaは再起動後に追加の辞書を認識します。 各プラットフォームの辞書のフォルダー：

Windows：$ PROFILE \ dictionaries \（オペラの$ PROFILE 'Opera folder'：について）

Linux / UNIX：$ PROFILE / dictionaries /（オペラの$ PROFILE 'Opera folder'：について）

Mac：〜/ライブラリ/アプリケーションサポート/ Opera 10 /辞書/



メール/ニュース




メッセージエディターでのリッチテキストのサポート


Opera Mailは、画像、定型化されたテキスト、リンク、および/または任意のHTMLを追加する機能を含むリッチテキストメッセージを送信できるようになりました。 必要に応じて、転送および転送されたリッチテキストメッセージも送信されます。 デフォルトのリッチテキストでのメッセージの作成は、メールアカウント設定の[送信ボックス]タブでメールアカウントに対して有効にできます-[HTML形式を優先する]このオプションは、編集ウィンドウの1つのメッセージに対して有効にできます。

太字、斜体、下線は、それぞれCtrl-B、Ctrl-I、Ctrl-Uで有効にできます。

リッチテキストはまだ署名でサポートされていません。



X日後に削除（POP3のみ）


X日後に削除は、指定された日数が経過すると、POPサーバーからメッセージを自動的に削除します。これは、メールボックスのサイズが限られているユーザーにとって特に便利です。 このオプションは、「＃日後にサーバーから削除」アカウント（デフォルトでは7日）のPOP設定の[受信トレイ]タブで有効にできます。 また、削除されたメッセージを読み取りおよび/または完全にダウンロードされたものに制限できます。

警告：完全にダウンロードされていないメッセージはサーバーから削除され、完全なメッセージを受信できなくなるため、トラフィック最小化モードを使用する場合、最後のオプションを無効にすると危険です。
