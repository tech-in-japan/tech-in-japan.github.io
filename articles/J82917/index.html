Intel LGA775プラットフォームは死んでいますか？ SLIからIntel X48へ-ほとんど！





ご存じのように、SLIは、グラフィックアプリケーションの生産性を向上させるために、1つのシステムに複数のビデオカードを組み合わせるためのテクノロジーです。 すでにNVIDIA社からの2番目の化身であり、長い間コンピューターゲームの世界に参入しましたが、人気はありませんでした。 ゲーム開発者自身が作成を最適化することを望まないため、また誰もが一度に1台のコンピューターで複数のビデオカードを購入する余裕がないため、この技術は幅広いサポートを受けていませんでした。 。 また、SLIの開発における障害は、チップセットの非常に政治的な要件であり、より具体的には、テクノロジーサポートのライセンスです。 たとえば、NVIDIA自体のチップセットのみがSLIを完全にサポートし、SLIサポートで広く使用された最初のIntelチップセットはIntel X58であり、これは約1年前に登場しました。 また、最初はすべてのX58マザーボードがSLIをサポートしなかったという事実は、NVIDIAがIntelと市場を共有することをためらうことによって再び影響を受けました。 しかし、NVIDIAはチップセットの構築でNVIDIAが技術的な競争に負け、消費者が最終的に選択する権利を得たことを示しています。 しかし、最新世代のIntelプラットフォーム（LGA 1366）でのみ-非常に高価であり、決して曖昧ではありません。1年後でも、Intel X48に基づいたクール溶接システムを変更する必要性は依然として疑問です。

しかし、その後、プロットは予想外に発展しました...







突然、X58チップセットに基づくマザーボードのSLIサポートが「オン」になりましたが、リリース時点ではサポートされていなかったようです（そして、ここにいます！プレスリリース、ドライバーの更新など）。最初にSLIで動作し、他のIntelチップセット（前の世代）で動作するようにビデオカードドライバを変更できることを希望しました。 そして、それは起こりました！ 熱狂者は、LGA 775プラットフォームでSLIを有効にするためのドライバーと手順の更新版を配布し、その後、Intel X48ベースのシステムの多くのユーザーに対して、「あるべきかどうか」という質問を明らかにしました。 -すべての生き物よりも生き生き！

詳細な調査を実施し、Intel X48のチップセットでのSLIテクノロジーのパフォーマンスをテストしました。 このチップセットにはSLIの公式サポートがないことを思い出させてください。



トリッキーなトリック



しかし、まず最初に、元の情報源とSLIを有効にするための実際の手順-xDews.comを紹介し 、IntelチップセットでSLIを有効にした理由について簡単に説明するのは正しいと思います。 NVIDIAビデオカード用ドライバーのIntel X58プラットフォームのリリースにより、このチップセット専用のSLIサポート付きの特別なファイルが登場しました。 愛好家が直面した主なタスクは、ドライバーにマザーボードにIntel X58チップセットが搭載されていると信じさせることでした。 このため、Intel X48チップセットに関する情報を含むファイルは、Intel X58チップセットに関する情報を含むファイルに強制的に置き換えられます。 次に、変更されたドライバーがインストールされます。これにより、操作が正しいと見なされ、エラーなしでインストールされます。 再起動後、システムはインストールされている複数のビデオカードを検出し、NVIDIAコントロールパネルにSLIを有効にするタブが表示されます。 非常に簡単な操作の後、GPU-Z保証システムはSLIテクノロジーがアクティブであると主張します。 最終的にSLIがIntel X48で動作することを確認するために、独立したテストを実施しました。



SLIが機能するために必要なもの



SLIテクノロジを使用するには、SLIサポートのあるマザーボード、またはIntelチップセットで公式のSLIサポートのないマザーボードが必要ですが、同じモデルのPCI-E拡張スロットと2つ以上のNVIDIAビデオカード、できれば1つのメーカーが必要です。 さまざまなブランドのビデオカードを使用することもできますが、ビデオアダプタに接続されているさまざまなBIOSによって問題が発生する可能性があります。 多くのブランドは、コア/シェーダードメイン/ビデオメモリの周波数を高くして、ビデオカードをオーバークロックすることを好みます。 同じモデルのカードの異なる周波数は、SLIモードで問題になる可能性があります。 マザーボードとビデオカードに加えて、SLIシステムの「必須」属性は、ビデオアダプターを相互に接続する柔軟または剛性のSLIブリッジです。 カードはSLIモードでも機能しますが、カード間でデータを転送および交換するためのPCI-Eスロットの帯域幅は十分ではありません。 これは、SLIブリッジが救助に来る場所です。

テスト方法



テストは、次のアルゴリズムに従って実行されました。

1. 1枚のビデオカードを使用したデフォルト設定でのシステムパフォーマンスの調査。

2. 1つのビデオカード、周波数648/1476/1242 MHz（それぞれカーネル/シェーダー/メモリ）でプロセッサを4500 MHzにオーバークロックするときのシステムパフォーマンスの調査。

3. 1つのビデオカード、周波数740/1560/1380 MHz（それぞれコア/シェーダー/メモリ）でプロセッサを4500 MHzにオーバークロックするときのシステムパフォーマンスの調査。

4. SLIモードの2つのビデオカード、周波数740/1560/1380 MHzでプロセッサを4500 MHzにオーバークロックするときのシステムパフォーマンスの研究（それぞれコア/シェーダー/メモリ用）。

結果









3DMark'05は、プロセッサのオーバークロックによる優れたスケーラビリティを示しました。 ビデオカードをオーバークロックすると、パフォーマンスの向上は観察されますが、CPUの周波数の増加ほど大きくありません。 SLIテクノロジーを含めることで、目に見えるパフォーマンスの向上が得られます-2500の3DMarks。 3DMark'05はCPUパフォーマンスに非常に敏感であるため、これは彼にとって非常に重要です。







「Anabiosis」では状況は同じですが、プロセッサーをオーバークロックしても目に見える利益はありません。 しかし、ここでは主にグラフィックスに重点が置かれているため、これをこのゲームの利点に帰することにします。 SLIでは、単一のビデオカードを使用するモードと比較して、生産性が約25パーセント向上します。







グラフィックの品質を向上させると、わずか数フレームでfpsが消費されます。これは、高解像度で快適なゲームを保証するためにグラフィックカードのパフォーマンスに十分なマージンがあることを示しています。







解像度を1680x1050に上げると結果が多少悪くなりますが、それでもインジケーターは高レベルであり、これらのグラフィック設定で問題なくプレイできると自信を持って言えます。 SLI-一般的に競合していません！







グラフィックスの品質を最大に設定すると、ほとんど目に見えない結果の劣化が観察されます。 1枚または2枚のビデオカードを使用することとの違いは、ほぼ20フレームであり、これは間違いなくテクノロジの優れた指標です。







Far Cry 2は、プロセッサのパフォーマンスに大きく依存していることを示しています。50％の加速により、結果がほぼ比例して増加します。 ただし、SLIモードの最適化は行われません。







DirectX 10 API Suiteを含めると、状況は不合理になります。 SLIモードの2つのビデオカードは、オーバークロックされたビデオカードを1つも失います。







解像度を上げてDirectX 9を含めると、状況はよくなり、2枚のビデオカードは高速になりますが、SLIを使用することによる利点は非常に弱くなります。





再びDirectX 10とSLIが再び失敗しました。 ゲーム開発者やNVIDIAドライバー開発者に石を投げますか？







Crysis Warheadは Far Cryと同じ開発チームによって作成されたため、ゲームが再びプロセッサのクロック速度に大きく依存するという事実に驚くことはありません。







驚いたことに、AA 16xQモードを含めるとfpsの増加は非常に小さくなりますが、逆の効果が生じるようです。 解像度が低すぎませんか？







DirectX 10への移行とグラフィックの品質の最大化により、力とフレームの配置はまったく変わりません！





DirectX 10の動作モードでのSLIからのゲインは小さいですが、問題はまだCrysis Warhead自体にあることを意味します。





解像度を上げても、電力のバランスは変わりません。 違いが見えるのは本当にHD解像度だけですか？





テストの結果に基づいて16xQアンチエイリアスを有効にしても、結果はまったく低下しません。 これは少なくとも奇妙に思えます。







快適なゲームに必要なのは、より高いクロック速度と最新のビデオカードを備えたプロセッサです。 ここで彼は成功の鍵です！





Crysis Warheadは、結果に基づいて、SLIテクノロジーがアクティブで使用されていることを確認します。







HAWXは、コンポーネントのオーバークロックによる顕著な増加が見られないことを示しています。 追加のビデオカードをインストールしても、ほとんど目立ちません。3フレームの増加はほとんどありません。



結論



さて、今では、IntelチップセットでSLIモードをアクティブにできます-「グリーンカード」に十分なPCI-Expressスロットがある場合のみ。 問題なく、Intel X48チップセット-Asus Rampage ExtremeをベースにしたマザーボードでSLIモードを有効にすることができました。 これに約10分を費やしましたが、実際には非常に簡単でした。 ウクライナのTiN愛好家に行われた仕事に感謝するのは公平だと思います。多くのコンピューターが、別のビデオカードを購入するだけで現代のゲームのパフォーマンスに追いつくことができます。 それでも、メーカーの指を完全に一周することは不可能であったことを認めなければなりません。SLIを適用する効果は、ハードウェア部品のメーカーだけでなく、ドライバーとゲームプログラムコードの最適化にも依存しますが、ここではそうではありません今日テストされたゲームの多くは、2番目のビデオカードからの増加よりもプロセッサへの依存度が高いことを示しました。 ゲーム開発者は、プログラムコードの最適化に時間とお金を費やすよりも、問題をユーザーに移し、新しい、より生産的なプラットフォームの取得を強いることに関心があります。ビデオカードメーカーとその作成を最適化します。
