開発者ツールキット：SQL Assistant

私はデータベース開発者（MS SQL）として働いており、私の仕事のほとんどはT-SQLコードを書いています。 クエリアナライザーで作業してからManagement Studio（2005、2008、2008R2）で作業していたため、コードの作成、クエリの作成などのプロセスに特化したエディター機能や追加の開発環境機能が本当に不足していました。またはEmbarcadero RAD Studio、Query Analyzerは何を提供できますか？ コードブロックのインデントを変更し、選択したテキストの大文字と小文字を変更し、コードブロックのコメントをコメントアウトしてキャンセルし、テーブルまたはフィールドの名前をエディターウィンドウにドラッグし、選択したオブジェクトのスクリプトを作成します。 しかし、完全なIntelliSenseが欠けていました。 これらすべての便利な自動補完機能、ダイヤルされたコードのインタラクティブなヒント、オブジェクトのリストからの選択など。



MS Management Studio 2005のリリースで、状況は変わりませんでした。 そして、MS Management 2008でのみTransact-SQL IntelliSenseが登場しました。 はい、入力された単語の自動補完、データベースオブジェクトとテーブルフィールドのリスト、構文の強調表示（begin ... end、（））がありました。 しかしその前に、SoftTree Technologiesのサードパーティ製SQL Assistantユーティリティを使い始めました。 SQL Assistantが提供する機会の中で、ネイティブIntelliSenseはまさにオークです。 しかし、それはSQLであり、オブジェクト指向言語ではないため、さらに何かを取得することは可能でしょうか？ できます！ この記事では、SQL Assistantユーティリティ、特にSQLコードを作成するときに使用する関数と手法について説明します。



理論


SQL Assistantは、開発者およびデータベース管理者向けのツールキットです。 小さなデータベースから複雑な企業システムまで、あらゆる複雑度のプロジェクトで作業するときに使用できます。 このソフトウェアは、インストール後すぐに使用できる状態であり、現在の作業プロセスの中断は含まれません。 このツールキットは、SQL開発者の生産性を向上させ、高品質のコードを保証します。 パッケージを含めると、SQL構文の正確性、構文構造の自動補完、プログラムコード内の迅速なナビゲーションなどを確認するための優れたツールが含まれます。 また、ユーザーは統合されたスペルチェッカーを見つけるでしょう。 この製品のその他の利点の中でも、最も一般的なDBMS（Oracle、SQL Server、DB2、MySQL、Sybase ASE、Sybase ASA、MS Access、および対応するSQL方言）のサポートに言及する価値があります。



SQL AssistantはManagement Studioに組み込まれており、そのすべてのコマンドは、メインメニュー項目またはコードエディターのコンテキストメニューから使用できます。







主な機能は次のとおりです。

  • コード入力アシスタント
  • コードナビゲーション（コード構造、ドキュメントマップ）
  • コードのフォーマット
  • パターンとコードジェネレーター
  • リファクタリング関数
  • SQLコンストラクトを入力するための対話型ヘルプシステム
  • オブジェクトスクリプトの取得（テーブル作成スクリプト、プロシージャ、関数など）
  • クイックビューテーブル
  • データのスクリプト作成、インポート、エクスポート
  • 個別のSQL実行
  • SQL実行スケジューラ（Windowsスケジューラを使用）
  • テストデータジェネレーター
  • 単体テストフレームワーク
  • バックグラウンド構文チェック
  • クエリパフォーマンスアナライザー
  • バージョン管理システムとの統合
  • スペルチェック
  • コード比較
  • しおり




基本情報は、公式Webサイトおよびドキュメント（記事の最後にあるリンク）にあります。 そして、このユーティリティの最も重要な機能であるアシスタント自体についてお話します。コードエディタに表示されるコンテキスト依存のポップアップウィンドウです。
