バジェットDlink dir-615ルーター、代替ファームウェアDD-WRTおよびIPTV

インターネットが私たちの街にやって来ました！



過去2年間、ヴォロネジのインターネットプロバイダーは途方もない速度で開発と拡大を続けてきたため、クライアントのインターネット機器を変更およびアップグレードする必要があります。 長い間、ネットワークへのアクセスとしてADSL2 +モデムと安価なWi-Fiルーターを使用していましたが、プロバイダーはイーサネット経由で、またはDomolinkが呼んでいるFttx経由で電話回線へのアクセスを提供することを決めました。 これに伴い、UDPを介したIPTVに過ぎない無料のOpen TVサービスも利用可能になりました。 これらすべてにより、単純なLevel ONE WBR-3408をより機能的なものに置き換えるように促されました。 驚いたことに、機能が常に高い価格と連動しているわけではないことがわかりました。 猫の下でDlink dir-615を選択した方法と理由を確認できます。







ブランドと偏見


すぐにDlinkに来たわけではないことに注意する必要があります。最初は、どの会社を選ぶべきか疑問に思いました。 会社自体が自信を呼び起こし、ネットワークのオープンスペースでさまざまなファームウェアに何度も出会ったことがあります。 ルーターの仕様を深く掘り下げることなく、検索を続けました。 私にとっての主なパラメーターは、Wi-Fi標準NのサポートとIPTVでの正しい操作でした。店に来て売り手に尋ねると、デバイスがDraftNとIPTVをサポートしている兆候があるAsus rt-n13ルーターの箱が届きました。 インターネットとWi-Fiのセットアップは問題を引き起こしませんでしたが、コンピューター上のIPTVは、どちらかが機能するPpoeセッションの上昇に頑なに行きたくありませんでした。 ためらうことなく、私はDomolinkテクニカルサポートに連絡することにしました。このサービスは、Wi-Fiなしのdlink dir-100 / 120ルーターとWi-Fi標準Gのdir300 / 320でのみ動作することを心地よい声で教えてくれました。 、私はどれも見つけられなかったことに驚いた、ルータでの実験はこれで停止し、そしていくらか動揺して、私はIPTVの欠如に我慢した。 数か月後、地元のフォーラムで、非常においしい価格でDlink dir-615ルーターの販売広告を目にしましたが、保証はありませんが、2度考えずにEbayに登り、価格が配送料で30ドルから始まったことを確認しました。 DD-WRT Webサイトでこのモデルのサポートを確認した後、ロットはすぐに注文されました。



Dlink dir-615 revD


地元のロシアの郵便局で小包を受け取るプロセスについては説明しません。梱包に多少のしわが寄っていたが、完全に無傷だったとしか言えません。 家に着くと、私は箱の中に標準セットのマニュアル、パッチコード、設定ディスク、電源、ルーター自体を見つけました。 驚いたことに、アンテナは取り外し不可能であることが判明しました。後で判明したように、このモデルのルーターにはさまざまなリビジョンがあります。完全に黒いケース、固定アンテナ、Ralink 3052プロセッサを備えたリビジョンモデルD2を受け取りました。他のリビジョンは、ケースの色、およびプロセッサとアンテナの設計が異なる場合があります。 すべてをPCに接続し、Dlinkネットワーク機器の標準インターフェイスを確認した後、DD-WRTにフラッシュする前に動作を確認することにしました。インターネットとWI-Fiは正常に動作し、テレビはまだありませんでした。

こんにちはLinux DD-WRT


Dir-615をDD-WRTに再フラッシュするのは可能な限り簡単です。

1. dd-wrt Webサイトでdir-615「ルーターベース」を見つけます（示されているものとルーターのリビジョンを必ず確認してください）。

2. Dir-615 revD2ページのルーターデータベースに移動し、dlink-dir615d-factory-webflash.binファイルをダウンロードします。

3.ルーターに移動し、標準ファームウェアで[ ファームウェアの更新]アイテムを見つけ、ダウンロードしたファイルをポイントして、[更新]をクリックします。


電源が切断されたり、その他の予期せぬ状況が原因でファームウェアが失敗した場合、あまりパニックに陥らないようにしてください。このルーターには、次の手順で使用できる緊急ブートローダーがあります。

1.ルーターからすべてのケーブル（電源とLAN）を引き抜いて、ルーターを取り外します。

2.コンピューターのネットワーク接続設定に静的IPアドレス192.168.0.2、マスク255.255.255.0、ゲートウェイ192.168.0.1、DNS 192.168.0.1を書き込みます

3.ルーターのWANポートを介してコンピューターとルーターを接続します。

4.ルーターの[リセット]を押し​​続けます。

5.ルーターの電源を入れます（電源を差し込みます）。

6. 5秒待ちます。

7.リセットを解除します。

8.ブラウザから192.168.0.1で入力します

9.開いたページで、[参照]をクリックして、ルーターの元のファームウェアファイルを選択します。 （投稿の最後に元のDir-615 revD2ファームウェアへのリンクを投稿します）。

10. [アップロード]をクリックして、指示に従います。


すべてが順調に進んだ場合、しばらくすると、ルーターがLEDを点滅させて再起動します。 これで、 http：//192.168.1.1/のルーターに移動できます 。 このファームウェアのインターフェースは元のDlinkよりもはるかに使いやすく機能的であることに注意する必要があります。ルーター上のLinuxは確かにコンソールであり、多くのマニュアルを読んでいることを恐れてはいけません、90％のケースではすべてが数回のマウスクリックで設定されています。 DD-WRTインターフェースは、 http ： //dd-wrt.com/demoのリンクでテストできます。 英語が怖い人は、 Administration-Managementタブでインターフェースをロシア語に切り替えることができますが、すべての公式ファームウェアが自慢できるわけではありません。



DomolinkおよびIPTV用のDD-WRTの構成


初めてログインするとき、DD-WRTはセキュリティの面倒を見て、新しいユーザー名とパスワードを見つけ出すためにあなたを永続的に送信します。 その後、インターネット接続とIPTVの直接設定に進むことができます。

インターネット接続は、 [セットアップ ]-[ 基本セットアップ ]タブで設定され、Domolinkに接続するのに数回クリックするだけです。

1.接続タイプの選択-PPPoE

2.ユーザー名とパスワードのフィールドに入力します

3.ページの下部にある[設定を保存して適用]をクリックします。






その後、「惑星」LEDの色がオレンジから緑に変わります。これが発生しなかった場合は、入力したデータを再度確認し、すべてが正しい場合は、[ 管理 ]タブに移動してページの下部にある[再起動ルーター]をクリックします。



接続が成功した場合、IPTVの構成に進むことができます。

1. [ セキュリティ ]タブに移動し、SPIファイアウォールを無効にします。 保存をクリックします。



2. [管理]-[コマンド]タブに移動し、次のテキストを挿入します

ifconfig vlan2 1.1.1.1 netmask 255.255.255.0 broadcast 1.1.1.255

cd / tmp

寝る30

wget http://dl.dropbox.com/u/14930004/udpxy

chmod + x udpxy

/ tmp / udpxy -a 192.168.1.1 -p 4022 -m 1.1.1.1 -B 1Mb -M 30


3. [スタートアップの保存]をクリックし、[ 管理-管理 ]タブに移動して、ページの下部にある[ルーターの再起動]ボタンをクリックします。





説明：

vlan2 -WANポートに対応するインターフェイスは、ルーターによって異なる場合があります。* WANポートの割り当て*のセットアップ-ネットワークタブを参照してください。

wgethttp：//dl.dropbox.com/u/14930004/udpxy-udpxyファイルへの直接リンクがあるサイト。ブートごとにルーターはそこからudpxyをダウンロードして起動します。 私はこのリポジトリを使用していますが、このファイルがどのくらい続くかはわかりません。 したがって、個別に以下に掲載します。 DD-WRTを使用するルーターごとに、 UDPXYファイルは異なります。DD-WRTフォーラムから異なるUDPXYをダウンロードして、科学的な突く方法を使用して選択しました。

* / tmp / udpxy -a 192.168.1.1 -p 4022 -m 1.1.1.1 -B 1Mb -M 30 *割り当てられた値はルーターのストリームに割り当てられたキャッシュに対応します。この値を1Mbに減らしました。 より高い値では、画像がばらまかれます。画像にアーティファクトがある場合、この値で遊ぶことができます。


手順が完了したら、UDPXYが開始されたかどうかを確認する必要があります。これは、統計http://192.168.1.1:4022/statusを示す次のリンクで実行できます。
