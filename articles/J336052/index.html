3CXのIP電話の集中VLAN設定

はじめに



VLANトラフィックのタグ付け（VLANをIP電話に割り当てる）により、ケーブルやネットワークポートを追加する必要がなくなります。 音声とデータ用に2つの別個のネットワークを作成しないでください。 PCは、デフォルトVLANの組み込み電話スイッチ（PCポート）または割り当てられたVLANに接続できます。



3CX管理インターフェイスのv15.5 SP1以降、ユーザーのIP電話のWANポートまたはPCポート、あるいはその両方でリアルタイムトラフィック（RTP）の優先度を集中的に設定できます。 つまり、IP電話自体とミニスイッチを介して接続されたPCの両方にVLANを割り当てることができます。







VLANパラメーターは、デバイスに適切なファームウェア機能がある場合、3CXシステムでサポートされる電話でのみ構成されます。 集中型VLAN構成は、レガシー電話（たとえば、Cisco 7900シリーズ）およびDECTデータベースではサポートされていません。



IP電話スイッチでVLANを使用する機能



内蔵スイッチによりケーブルとネットワークポートを節約できるという事実にもかかわらず、その操作は接続されたPCの操作に影響します。 電話を再起動するか電源を切ると、PCはネットワークとの接続を失います。 ネットワークアプリケーションがPCで実行されている場合、それらの操作はエラーで停止する場合があります。 さらに、PCポートに接続されたPCが大量のトラフィック（グラフィックスやビデオの処理など）を生成すると、VoIP通信の品質が著しく低下する可能性があります。 この場合、PCおよびスイッチ上のアプリケーショントラフィックに優先順位を付けることをお勧めします。



AutoConfiguration 3CXでは、電話機のLAN / WANおよびPCポートでのみVLANパラメータを設定できますが、企業のスイッチ、ルーター、およびDHCPサーバーのセットアップはシステム管理者の責任です。 電話機のVLANの自動設定が正しくないと、ネットワークから切断される可能性があります。 この場合、ハードウェアのリセットが必要になります。これは通常、キーボードから直接行われます。 電話機とスイッチのVLAN設定が一致しないと、通信品質が著しく低下し、会話が断続的に途切れることがあります。



IP電話でのVLANの構成



一般に、VLANタギングの実装を成功させるには、高品質のネットワーク機器と、スイッチ/ルーターを構成するシステムエンジニアのスキルが必要です。 事前にテストネットワークセグメント内の電話の小さなグループでVLANの自動設定をテストすることをお勧めします。



VLANを実装する前に、次のことを考慮することが重要です。



  • VLANタギングは必ずしも必要ではありません、なぜなら これは、ネットワークと管理者の作業を複雑にします。
  • これが利用可能な場合は、PCとIP電話に企業スイッチの異なるポートを使用するだけです。
  • VLANの利点は、大規模なインストールでのみ顕著です。
  • テクノロジーを実稼働ネットワークに展開する前に、必ずVLANをテストしてください。
  • ネットワークが異常に混雑している場合、VLANは役に立ちません。


ほとんどの場合、VLANは次のように実装されます。



  1. データネットワーク

    • VLANタギングなし
    • PCにインストールされた3CXおよび3CXPhone Webクライアント
    • 音声ネットワーク上にある3CXサーバーでルーティングを構成
  2. 音声ネットワーク

    • VLANタギング（この例では、Vlan ID55）
    • 音声ネットワーク上にある3CXサーバー


この設定は、IP電話トラフィックがVLANタグ付きであり、PCトラフィックがタグなしで通過することを前提としています。 トポロジの例：



  1. データネットワーク10.10.10.0 / 24

    • IP PC 10.10.10.3 / 24
  2. 音声伝送ネットワーク10.10.20.0 / 24（VLANタグ55）（トラフィックはIP電話のWANポートでタグ付けされます）

    • 3CXサーバーIP 10.10.20.2 / 24
    • IP電話10.10.20.5 / 24


このトポロジには1つの問題があります。ネットワークに接続された新しい電話にはまだVLAN設定が設定されておらず、データネットワークからIPアドレスを受け取ります。 3CXがプラグアンドプレイテクノロジーを使用してIP電話を自動構成できるようにするには、デバイスがサーバーと同じネットワークセグメントに存在する必要があります。 そのため、IP電話の音声ネットワークへの自動切り替えを構成する必要があります（電話によって送信されたマルチキャスト自動構成パケットは、サーバーによってインターセプトおよび処理されます）。



DHCPオプション132



注：スイッチのLLDPはオフにする必要があります。 DHCPサーバーで、オプション132を作成し、Vlan ID 55を指定します（この例の場合）。 通常、PCはこのオプションを無視しますが、IP電話はそれを正しく解釈する必要があります。 オプション132を受信した電話機は、現在のIPアドレスを解放し、VLANを変更し、VLAN 55から新しいIPアドレスを受信します。その後、電話機は3CXによって処理されるマルチキャストPnP自動調整要求を送信します。



LAN / WANポート



VLAN IDは 、電話機の自動構成インターフェースで直接指定されます。 DHCPオプション132に対応する必要があります。この構成には、多くの利点があります。



  • 設定済みのデバイスは、目的のVLAN IDに切り替えるために、最初にデータネットワークからIPアドレスを取得しようとしません。
  • IP PhoneはDHCPオプション132を使用しません






VLAN優先度オプションはデフォルトで有効になっており、範囲は0〜7です。パケットの優先度は、通過するネットワークノードによる処理方法を決定します。 値の範囲は1（低優先度）から6（高優先度）です。 7の使用は推奨されません。 緊急事態のために予約されています。 この値を誤って指定すると、通信の品質が低下することに注意してください。 IP電話からのリアルタイムトラフィックには、他のネットワークトラフィックよりも常に高い優先順位が必要です。



PCポート（オプション）



電話機のPCポートにVLANを定義しない場合、ネットワークのデフォルトVLANで機能します。 データネットワーク内（これは、PCポートが切断されることを意味するものではありません-PCの接続に引き続き使用可能です）。







PCポートに接続されたPCは、デフォルトネットワーク以外の別のVLANに「転送」（つまり、タグ付け）する必要がある場合があります。 この場合、VLANタギングは電話のPCインターフェイスで使用されます、たとえば：



  1. デフォルトのネットワークは10.10.10.0 / 24です

    • DHCPプール10.10.10.3 / 24
  2. 音声伝送ネットワーク10.10.20.0 / 24（VLANタグ55）←VLANポートWAN

    • 3CXサーバーIP 10.10.20.2 / 24
    • IP電話10.10.20.5 / 24
  3. データネットワーク10.10.30.0 / 24（VLANタグ77）←VLANポートPC

    • IP PC 10.10.30.3 / 24


VLANの構成例



音声ネットワークのIP電話でのVLANタギングの設定+ DHCPオプション132



  1. IP電話を接続すると、データネットワーク（10.10.10.5）でIPアドレスを受け取ります。 このネットワーク（10.10.30.2）上のPBXサーバーへのアクセスはなく、電話はPnPテクノロジーを使用して構成できません。
  2. 電話機のIPアドレスにDHCP冗長性を設定し、55というテキスト値でDHCPオプション132を割り当てます。
  3. IP電話を再起動する
  4. 再起動後、IP電話は55のVLANタグを受け取り、音声ネットワーク（10.10.20.5）からIPアドレスを再び受け取ります。
  5. 3CX管理インターフェイスで、[ 電話]セクションに移動し、接続されている新しい電話を選択して、ユーザーに割り当てます（または、この電話の新しいユーザーを作成します）。
  6. [ VLAN構成]セクションの[ 自動調整電話]セクションで、[ WANポートのVLANを有効にする]オプションを有効にします。
  7. [ WAN PORT VLAN ID]フィールドの[ VLAN設定の構成]（数値ベンダーに依存-Snomは空、許可される値は1〜4094）で 、値を55に設定します。
  8. 上部の[ OK ]をクリックして、変更を適用します。
  9. 電話は新しい設定を受け取り、再起動します
  10. 再起動後、電話機のWANポートにVLAN ID 55のタグが付けられます
  11. IP電話は、音声ネットワークのIPアドレスを受け取ります（10.10.20.5）
  12. DHCPオプション132は、自動構成によって取得されたパラメーターによってオーバーライドされます。


音声ネットワークのIP電話でのVLANタギングの構成



  1. IP電話を接続すると、データネットワーク（10.10.10.X）でIPアドレスを受け取ります。
  2. [ VLAN構成]セクションの[ 自動調整電話]セクションで、[ WANポートのVLANを有効にする]オプションを有効にします。
  3. [ WAN PORT VLAN ID]フィールドの[ VLAN設定の構成]（数値ベンダーに依存-Snomは空、許可される値は1〜4094）で 、値を55に設定します。
  4. 上部の[ OK ]をクリックして、変更を適用します。
  5. IP電話は新しい設定を受け取り、再起動します
  6. 再起動後、電話機のWANポートにVLAN ID 55のタグが付けられます
  7. IP電話は、音声ネットワークのIPアドレスを受け取ります（10.10.20.5）


音声ネットワークのIP電話でのVLANタギングの設定+ PCを介してPCをデータネットワークに接続



  1. 電話機のPC IPポートと電話機のWANポートを備えたPCを企業ネットワークに接続します
  2. 接続されると、電話はデータネットワーク（10.10.10.5）でIPアドレスを受け取ります。
  3. 電話機のPCポートは、データネットワークのアドレスプール（10.10.10.3）からIPアドレスも受信します。
  4. [ VLAN構成]セクションの[ 自動調整電話]セクションで、[ WANポートのVLANを有効にする]オプションを有効にします。
  5. [ WAN PORT VLAN ID]フィールドの[ VLAN設定の構成（数値ベンダーに依存-Snomは空にすることができ、許容値は1〜4094）]で 、値を55に設定します。
  6. PCポートのVLANを有効にするを有効にしないでください
  7. 上部の[ OK ]をクリックして、変更を適用します。
  8. IP電話は新しい設定を受け取り、再起動します
  9. 再起動後、電話機のWANポートはVLAN ID 55でタグ付けされ、PCポートはデータネットワーク（10.10.10.3）で引き続き機能します。
  10. IP電話は、音声ネットワークのIPアドレスを受け取ります（10.10.20.5）
