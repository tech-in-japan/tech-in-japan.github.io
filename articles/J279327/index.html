Veeam Self-Service File Restore Webポータルを使用して仮想マシンファイルのリカバリを整理します

Veeam Backup Enterprise Managerは当初、主にWebコンソールを介してVeeam Backup＆Replicationサーバーを集中管理するためのツールとして位置付けられていました。 ただし、Enterprise Managerを使用すると、仮想マシンファイルを復元する権限を委任するのにも非常に便利です。 この機能は、Veeam Backup＆Replication 9.0のすべての商用版 （つまり、Standard、Enterprise、およびEnterprise Plus ）で利用可能です。 その助けを借りて、VMおよび/または個々のファイルの回復を「ランクごと」に実行するユーザーに必要な特権を割り当てることができます。 たとえば、組織に管理者間の責任分担がある場合（誰かがSQLを担当し、誰かがExchangeを担当するなど）、これらのサーバーの復元の責任をさまざまな粒度で委任することは合理的です（これについて話している次回お知らせします）。 Enterprise Plusエディションがインストールされている場合、管理者は同僚にVeeam Self-Service File restoreポータルから大切なファイルを取得するよう送信することで、「ああ、バックアップからPDFを1つしか取得できません」などの同僚の迷惑なリクエストから身を守ることができます 。 詳細については、猫へようこそ。









セルフサービスポータルを介したユーザーの準備





ポータル自体は、Enterprise Managerのセットアップ時に同じサーバーにインストールされます。 ただし、仮想マシンまたはファイル自体を復元したい人に権限を委任する前に、いくつかの手順を実行する必要があります。

  1. Veeam Backup＆Replication Enterprise Plusライセンスがインストールされていることを確認してください-この機能は他のエディションではサポートされていません！
  2. のどが渇いたユーザーが、回復しようとしているマシンのローカル管理者グループのメンバーであり、Enterprise Managerサーバーと同じドメイン（または信頼できるドメイン）にいることを確認してください。 Enterprise Managerサーバーに対して逆DNSルックアップが実行されていることを確認します。



    注： Windowsマシンのみがサポートされています。 信頼されていないドメインのユーザーの場合、セルフサービスポータルを介した作業はサポートされません。

  3. Veeam Backup＆ReplicationサーバーがVeeam Backup Enterprise Managerサーバーに接続されていることを確認します（接続方法については、 こちらを参照してください ）。 これは、Enteprise Managerがバックアップタスクの完了時にデータを受信するために必要です。


次に、ユーザーのマシンのバックアップ中に、ゲストOSファイルのインデックスが作成され、これを行うプロセスがこのユーザーのアカウントで実行されることを確認する必要があります。 すべてが正常に実行されると、バックアップ作成時にローカル管理者だったユーザーがセルフサービスWebポータルを開き、バックアップの内容を表示し、必要なファイルを見つけて復元できるようになります。



重要！ ユーザーは、このマシンのローカル管理者権限を受け取った後、タスクによって作成されたすべてのマシンリカバリポイントを対応する設定で操作できます。 ユーザーが取り消されると、タスクが再度実行されるまで、リカバリポイントからのVM​​ファイルはポータルを介して利用可能になり、新しいリカバリポイントが作成されます（ユーザーはこのVMの管理者ではなくなります）。



そのため、目的のユーザーのアカウントでゲストOSのインデックス作成を有効にします。

  1. Veeam Backup＆Replicationコンソールでジョブ設定を開きます。
  2. ゲスト処理の手順に進み、[ ガストファイルシステムのインデックス作成を有効にする]オプションを選択します。
  3. [ ゲストOS資格情報]セクションで、ユーザーアカウントを指定します。








手動で開始するか、タスクがスケジュールどおりに実行されるまで待ちます。







次に、以下を実行します。

  1. 管理者権限を持つアカウント（ ポータル管理者 ）を使用して、Enterprise Manager Webコンソールを開きます。 通常、これはEnterprise Managerをインストールした人のアカウントです。 管理者、オペレーターなどの詳細 次回お話しするか、 ここ （英語）で自分で読むことができます。
  2. [ 構成 ]（右上）をクリックすると、対応するビュー（管理インターフェイス）が表示されます。
  3. ここで、完了したジョブと作成されたバックアップに関する情報を取得するために、Enterprise Managerに接続されたVeeam Backupサーバーから新しいデータの収集を開始する必要があります。 これを行うには、左ペインで[ バックアップサーバー]を選択し、[ 収集の開始 ]をクリックします。
  4. 次に、左側のペインで「 セッション」を選択し、データ収集セッションが正常に完了したことを確認できます。






これで、ユーザーはセルフサービスポータルに移動し、そこで自分の車のバックアップを見ることができます。



Webポータルを使用したセルフサービスのファイル回復



そのため、ユーザーはインターネットブラウザーを開き（Edgeを含むすべての一般的なブラウザーがサポートされます）、アドレスバーにポータルURLを入力します。

https://enterprise_manager_server:9443/selfrestore





開いたダイアログで、ユーザー名とパスワードを入力し、[ ログイン ]をクリックします。







ポータル自体は極端にEnterprise Managerのインターフェースに似ていますが、パネルが1つしかありません- ファイル 。 デフォルトでは、ローカル管理者としてアクセスしたマシンの最新のバックアップからゲストOSファイルが表示されます。 [以前の日付を選択]リンクを使用して同じマシンの古いバックアップを開くか、このユーザーが管理者でもある別のVMを選択して別のバックアップVMを選択できます。







簡単な検索（[ 参照 ]タブのウィンドウ）を使用するか、[ 検索 ]タブでより複雑な条件を入力して、必要なファイルを検索できます。

見つかったファイルで次のアクションを実行できます。

  1. 復元 -元のマシンに復元します。



    重要！ ソースマシンの電源を入れる必要があります。 さらに、このバックアップの作成以降に移動されていないことを確認してください。



    回復には2つのオプションがあります。

    -上書き-バックアップからのファイルは、ターゲットマシン上のファイルを上書きします。

    -keep-両方のファイルをターゲットVMに保存します。バックアップから取得したファイルにはRestoredプレフィックスが付きます。



    注：デフォルトでは、ターゲットマシン上のゲストOSにアクセスするために、このバックアップを作成したタスクのプロパティで指定されたアカウントが使用されます（上記を参照）。 何らかの理由で、たとえばパスワードが変更されたなど、権利がない場合は、入力ダイアログで必要なデータを指定する必要があります。



  2. ダウンロード -保存します。 デフォルトでは、選択したファイル（または複数のファイル）は、 DownloadsフォルダーのFLR_​​date_time.zipという名前でアーカイブに保存されます。



    重要！ Veeam Backup＆Replicationは、このフォルダーを定期的にクリーニングし、1日以上前のファイルを削除します。
  3. 復元リストに追加- 回復リストに追加します。 このコマンドは、たとえば、異なる復旧ポイントまたは複数のVMから複数のファイルをリストに収集するために便利に使用されます。 選択したファイルが[ 保留中の復元 ]タブに一覧表示されます。

    ここで、ファイルを選択し、[ 復元]または[ ダウンロード ]をクリックして、目的の操作を実行することもできます。 保存されたファイルへのリンクは、日中のセッションログにも保存されます。 ログを開くには、右下の[履歴を表示 ]をクリックしてから、回復操作から必要なログを選択し、 [ログ ]タブに移動して[ ダウンロード ]をクリックします 。







それは一般に、プロセス全体です。

まあ、同僚が回復ポータルにファイルを表示できないと不平を言って駆け寄った場合、この問題を次のように解決することをお勧めします。



-対応するタスクの操作を担当するVeeamバックアップサーバーがEnterprise Managerの下位に追加されているかどうかを確認します。 そうでない場合は、上記のように追加します。

-おそらく、タスクに関する最新のデータがまだVeeamバックアップサーバーから収集されていない可能性があります（デフォルトでは、15分ごとにデータ収集が開始されます）。 前述のように、データ収集を手動で開始できます。

-[ ゲストファイルシステムのインデックス作成を有効にする ]オプションがジョブ設定で選択されていませんでした。 ジョブ設定を開き、このオプションを選択します。

-おそらく、仮想マシンの復旧ポイントを作成した時点では、このユーザーはローカル管理者グループのメンバーではありませんでした。 彼が現在ローカル管理者であることを確認し、記事に記載されている手順に従ってください。

- それでも解決しない場合は、通常のアルゴリズムに従って進めてください（ Veeamフォーラムで質問する、ナレッジベースで検索するなど）。
