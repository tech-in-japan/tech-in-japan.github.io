Eee PC 701 4Gでのメガバイトの戦い

EldoradoからWindows XP Homeがインストールされた最新のAsus Eee PC 701 4Gを開梱して電源を入れた後、4GBのハードドライブには約1.5ギガバイトの空きディスク領域しかないことがわかります。WindowsXP、IE7、.NET Frameworkのアップデートをインストールした後、そのJREは約600〜700メガバイトのままです。 このような状況は私にはまったく合わないと思い込んで、SSDのスペースをできるだけ解放することにしました。



最初の考えは、インストールされたシステムを最大限にクリーンアップすることでした。 MS Work（399 MB）、Windows Liveに関連するすべてのアプリケーションを削除し、Windows Updateで残ったゴミからWindowsフォルダを削除し、ページファイルを無効にします。 その結果、1.9 GBの空き容量を解放することができました。これは、インストールされているJRE（110 MB）とOpera（13 MB）を考慮に入れると、良い結果であると考えていますが、Win XPで約1.5ギガバイトかかるという考えは、私に栄誉を与えませんでした。



Googlingがフォーラムhttp://forum.eeeuser.com/にアクセスし、トピック「 Ultimate XP HOW-TO Guide：XP SP2、537MB install size、Boots in 17sec -Eee PCでWindows XPを作成およびインストールする方法の詳細ガイド」に移動しましたHome Edition SP2のサイズは537 MBで、起動時間は17秒で、最も重要なのはWindows XP FLPおよびGame Editionがありません。



主なアイデアは、非常に便利な無料のユーティリティnLiteを使用することです。 このユーティリティを使用すると、元のWin XP配布キットに基づいて独自の軽量配布を作成し、そこから「不要な」コンポーネント、ドライバー、アプリケーションなどを捨てることができます。 ガイドの著者の推奨で、 nLite 1.4.1をダウンロードしました



ラップトップに付属のディスクからWin XPのネイティブ配布を使用することは、フォルダーの名前のためにうまくいきませんでした。 nLiteの場合、ディストリビューションはI38​​6フォルダーにあり、ASUSドライブでは、このフォルダーはi386XPと呼ばれていました。 幸いなことに、ブナのFutsitsu-Siemensからの私のOEM配布キットは横になっていて、nLiteはそれを非常によく食べました。



nLiteプログラムはロシア語をサポートしていますが、翻訳による混乱を避けるために、英語を使用することをお勧めします。 このマニュアルには、独自の配布キットを作成する各段階でのnLiteの詳細なスクリーンショットが記載されており、英語版で作成されています。



出力の指示とスクリーンショットに続いて、軽量のWinXPディストリビューションを備えたブートディスクのISOイメージを取得し、著者が推奨するすべて（ゲーム、DHCPクライアント、メディアプレーヤーなど）を削除せず、サイズは約150 MBになりました。
