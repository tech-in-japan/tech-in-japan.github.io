AtlassianのJIRA + Confluenceをサポートシステムとして構成する

Atlassian JIRAアプリケーションの登録および処理システムのセットアップの経験と、技術システムとしてのAtlassian Confluenceを共有したいと思います。 ユーザーサポート。 これについてアトラシアンが書いていることは、ここで見つけることができます： http : //confluence.atlassian.com/display/JIRA/JIRA+as+a+Support+System



それらを整理するためのいくつかの概念があります。 サポート。 1つ目は、クライアントごとに個別のプロジェクトを設定することです。 しかし、この概念は、それらの従業員という事実のために合理的ではありません。 サポートは全体像を把握しておらず、多数の顧客がいるとプロジェクトが過剰になります。 したがって、別の概念について説明します-構成されたスコープを持つすべてのクライアントに対して1つのService Deskプロジェクトを使用します。





プロジェクトのアクセス許可スキーム、つまりアクセスを設定することから始めましょう。 これがどのように行われるかについては、 http ： //confluence.atlassian.com/display/JIRA/Managing+Project+Permissionsをご覧くださいファイルを添付し、コメントを残します。



次に、スコープを構成します。 JIRAにはセキュリティレベルのような概念があります。 特定のアクセスレベルにあるリクエストは、このアクセスレベルの表示を許可されたユーザーにのみ表示されます。 管理者であっても、他のユーザーのアクセスレベルが割り当てられているリクエストは表示されません。 ここには2つのオプションもあります。オプションの選択はニーズによって異なります。 アクセスレベルを作成および構成する方法： http : //confluence.atlassian.com/display/JIRA/Configuring+Issue+Level+Security

オプション1：従業員を許可するプライベートアクセスレベルを作成します。 サポートおよびレポーターのタスク。 このレベルをデフォルトにします。 アクセス許可スキームは、ユーザーによるセキュリティレベルの設定を無効にする必要があります（まあ、または有効になっていますが、すべてのユーザーに要求を表示したままにすることができます。これは状況によっても必要です）。 そのため、単純なユーザーには、自分のリクエストとその従業員のみが表示されます。 サポート-すべてのリクエスト。これが必要なものです。

オプション2：このオプションは、顧客グループに多くのレベルのセキュリティを提供します。これは、顧客が多くの従業員を抱える企業であり、同じ企業のすべての従業員に共通の要求である場合に必要です。 したがって、最初に、各クライアントに適切なアクセスレベルを作成する必要があります。 次に、We Engineerプラグイン（ https://plugins.atlassian.com/plugin/details/22874 ）が必要です。 ワークフローの事後機能としてプロジェクトロールに応じてセキュリティレベルを設定できます（アプリケーションライフサイクル、それについて読んで設定することができます： http : //confluence.atlassian.com/display/JIRA/Configuring+Workflow ）。 クライアントに対応する追加のプロジェクトロール（ユーザー、グループ、ロール->プロジェクトロールブラウザ）を作成し、サービスデスクプロジェクトでこのロールを対応するユーザーグループに割り当てます。 次に、ワークフローの設定に進み、最初の移行-プラグインのドキュメントに示されているように、関数の投稿を作成して追加します。 （ 重要な注意：最初のステップ（作成）でこれらの関数を作成するとき、最初にそれらを設定する必要があります。他のステップでは、最後になります。 ）したがって、注文を作成すると、セキュリティレベルが自動的に割り当てられます。



ワークフローの設定については説明しません。必要なことはすべてここで説明されているためです： http : //confluence.atlassian.com/display/JIRA/Configuring+Workflow



また、アプリケーションを開発者に転送する方法についても言わなければなりません。 このためには、Clone Plusプラグイン（ https://plugins.atlassian.com/plugin/details/44043 ）を使用すると便利です。 配置するだけで、[C ++ ++]ボタンが表示され、すべてのコメントと添付ファイルを含むリクエストを別のプロジェクトに複製できます。 クローンとオリジナルは参照によりリンクされます。



一般的に、JIRAをサポートシステムとして使用するのに役立つプラグインはかなりあります。 https://plugins.atlassian.com/search/category/21338にあります 。



Confluenceについて。



主な機能はここにあります： http : //confluence.atlassian.com/display/DOC/Confluence+Documentation+Home



このシステムは異なるアクセスレベルのドキュメントを保存できるという事実に加えて、もう1つの便利な機能があります。ConfluenceでJIRAシステムとの同期を構成する場合、ユーザーを追加できるようにグループ内の一部のユーザーに「管理者権限」を与えることができますグループに追加して、グループから削除します。 つまり、会社自体が従業員とシステムへのアクセスを管理できます。 同時に、登録を開くことはできますが、管理者に登録されるまで、新規登録ユーザーが何もすることを禁止できます。 これを行うには、カスタムスペースユーザー管理プラグインプラグイン（ https://plugins.atlassian.com/plugin/details/133?versionId=373044 ）が必要です。 彼が完全な所有者になる各スペースクライアントに対して「サンドボックス」を作成し、「サンドボックス」のキーとなるプレフィックスを持つグループを作成し、これらのグループにJIRAおよびConfluenceへの適切なアクセス権を付与します。



現時点ではこれですべてです。誰かが興味を持ったら、私はいくつかの瞬間に深く入ることができます。
