Firebird：10年間の成功した開発



Firebirdは、ユニバーサルオープンソースDBMSです。

このレビューは、Firebirdの10周年（7月に行われます）と、Firebird 2.5の最終バージョン（4月下旬-5月上旬）の差し迫ったリリースに捧げられています。

MindTheBird！

www.MindTheBird.com





Firebirdについて考える多くの理由があります...







  • 100万人以上のFirebird開発者
  • Windowsでの2,500以上の毎日のインストール、Linuxインストールはカウントしません
  • Firebirdの何百万ものコピーが、ボックス化アプリケーション（ISV）と共にインストールされます
  • Firebirdは、すべての主要なLinuxディストリビューションとリポジトリに含まれています。




Firebirdは...





マルチバージョンアーキテクチャ





  • ハイブリッドアプリケーション

    マルチバージョンアーキテクチャにより、Firebirdベースのシステムを作成して、オンラインデータ（OLTP）とレポート（OLAP）を同時に処理できます。
  • 自動バージョン管理

    Firebirdは必要に応じて自動的にスイープします（他のDBMSのバキュームとして知られています）


イベント







イベント

  • イベント通知は、トリガー、ストアドプロシージャ、および実行ブロックから送信できます。
  • 柔軟なサブスクリプションメカニズム

    -ON TRANSACTION START（トランザクションを開始するため）

    -ON TRANSACTON COMMIT

    -など



ロギングと同期

  • データベースレベルのトリガーからインスタントデータベースレベルのイベント通知を呼び出す（およびログに記録する）ことができます

    -ON CONNECT（接続ごと）

    -ON DISCONNECT（切断するため）

    -など



強力なSQL機能



  • ANSI SQLとの高い互換性
  • 共通テーブル式（CTE）
  • 柔軟なトランザクション管理
  • 強力なストアドプロシージャ（選択的プロシージャにより、単一のSELECT内でテーブルと結合できます）
  • 複数のデータベースのクエリ


トレースAPI







トレースAPI

  • リアルタイム監視
  • SQLクエリのデバッグ


監査

  • イベント
  • 部分的または完全なSQLロギング
  • リモート接続経由！


安全性







SQLレベルのセキュリティ

  • ユーザーとロール
  • データおよびデータベースオブジェクトを使用したすべての基本操作の権限の設定
  • データベース所有者の概念


Windows信頼できる認証

  • Windowsおよびクライアントプログラムでのシングルサインオン
  • WindowsおよびActive Directoryドメインとの統合


ネットワーク

  • 必要なポートは1つのみです（デフォルトでは3050、構成可能）
  • エイリアスは実際のデータベースファイルパスを隠します


4アーキテクチャ



  • クラシック

    複数のCPUと十分なRAMを備えた強力なシステムのアーキテクチャ
  • スーパーサーバー

    複製ソフトウェアに統合するためのコンパクトで高性能なバージョン
  • 埋め込み

    DLLのシングルユーザーDBMS。 古いバージョンへの透過的な移行！
  • スーパークラシック

    2.5の新しいアーキテクチャ！ SuperServerとClassicの最高の組み合わせ。 仮想化に最適


拡張性



  • 4つのアーキテクチャにより、1MBから1テラバイトまでのベースを作成できます
  • 1ユーザーから1000まで
  • DLLから複数のCPUへ


マルチプラットフォームサブ





  • Windows、Linux、HP-UX、Solaris、Mac OS X、FreeBSDなどのサポート
  • 32および64ビット
  • OS間の100％の互換性とプラットフォーム間の1ステップの移行




開発ツール

  • UIB、FIBPlus * 、IBObjects * （Delphi / C ++ Builder）
  • FreePascal＆Lazarus
  • FirebirdのPHP
