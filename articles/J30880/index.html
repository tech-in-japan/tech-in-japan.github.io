Race for Google（略称）



ジャーナリストのミゲル・ヘルフトは、この競争の主要な段階を検討し、多くのベンチャーキャピタルが検索市場に注がれていることに注目しています。Googleを「打ち負かす」だけでなく、独自の広告パイを手に入れるためです。 ただし、Helftは、多くの企業の努力にもかかわらず、Googleに追いつくことができるかどうかは明確ではないと結論付けています。 GoogleはMicrosoftのような独占企業ではありませんが、現時点ではGoogleを転覆できる会社はありません。

私たちは皆、Googleの優位性についてほとんど同じ気持ちを持っていますが、なぜですか？ 理解するために、Googleの競合他社とそのアプローチを分類しています。 この記事では、Emreの分類を使用して、Googleが引き続き検索の王であると直感的に感じる理由を理解します。

競合技術の分類



新しい会社の発展には、改善の道と安くする道の2つの古典的なアプローチがあります。 検索市場の場合、より安価な方法は不可能です。 したがって、競合企業にとって唯一の方法、つまり改善する方法しかありません。 この記事では、改善のための3つのオプションを検討します。

  • 技術の改善-より良い（関連する）結果。
  • インターフェースの改善-結果がより良く表示されます。
  • 垂直/セマンティック検索-改善は、テクノロジーとインターフェースの両方に関するものです。


技術改善



PageRankは、Googleによって生まれた現象です。 競合他社には、リーダーを「回避する」ための2つのオプションがあります。1）自然言語/ AIと2）人々の力。

AIキャンプでは、 HakiaとPowersetの 2つの会社に言及します。HakiaとPowersetは 、検索する内容を理解することで検索結果を改善する努力をしています。 自然なクエリを理解することの利点は否定できませんが、検索結果の改善への影響は明らかではありません。

これらの企業が何を獲得できるかを犠牲にしては明らかではありません。 まず、通常のクエリを入力する機能は既にGoogleによって実装されており、競争上の優位性はありません。 検索結果を大幅に改善することは競争上の優位性になりますが、想像するのはまだ困難です。 もう少し良い-それは可能ですが、大きな違いを挙げることはできません。

人々の助けを借りて検索することは、コンピューターでは利用できない否定できない利点をもたらすという点で既に興味深いものです-常識。 del.icio.usは検索エンジンとして表されることはあまりありませんが、おそらく人々が検索する最良の例です。 del.icio.usで得られる主なものは、フィルタリングされた高品質の結果です。 欠点は、すでに人々に人気のある結果が得られることです。 実際、これはかなり予測可能ですが、場合によっては重大な制限になる可能性があります。

もう1つのdel.icio.usの問題は、製造可能性です。 たとえば医療トピックなどで何かを検索しているときに、良い結果が得られることはほとんどありません。 ChaChaは別のアプローチを使用しており、これは12月のレビューでRead / Write Webが書きました。 この会社は、文字通り人々を使用して、必要な情報を見つけます。 ただし、少なくとも2つの欠点があります。検索クエリのコストが高いことと、スケーリングに関する問題の可能性です。

del.icio.usに似たサービスが静的サイトディレクトリよりも桁違いに優れていることは否定できませんが、検索アルゴリズムは人々の力で打ち負かすことは困難です。 Google検索の速度と完全性は、人々の手の届かないところにあります。

改善されたインターフェース



Googleのモットーはシンプルです。 これはすべての会社の製品の重要なアイデアであり、厳密にそれを遵守しています。 これが、非常に多くの競合他社がより良い検索結果を提供しようとしている理由です。

SnapおよびMicrosoft Image Searchは、インターフェースを強化するために画像を表示します。 画像は本当に面白くて便利です。 個人的には、Microsoft Image Searchインターフェースがとても気に入っています。これは、検索結果の表示が大幅に改善されています。 残念ながら、両方のテクノロジーは結果の関連性だけでなく、その数も改善するはずです。 さらに、画像検索に適しているのはテキスト検索にはあまり適していません。小さな画像から結果の品質を判断することは困難です。

次に、検索結果のクラスタリングを見てみましょう。これは、本質的には垂直検索ですが、すべての結果が含まれています。 Clustyは、検索結果に対する理解を変えることなく、検索結果に価値を加えることで大きな印象を与えます。 検索結果に加えて、結果を絞り込むクラスタータブの存在により、基本的に2つ以下の手順で回答を得ることができます。

2番目の検索エンジンSearchMeshは、Googleの所有物であり、会社の「遊び場」です。 Ajaxを使用し、いずれかのモードでは、検索結果ページのタイトルのみを表示できます。 この方法は驚くほど便利だと思います。 ブログ、ニュース、または写真を検索して、結果を垂直方向に「切り刻む」こともできます。 どちらのサイト（SearchMashとClusty）も非常に便利で、メインストリームに参加できます。

検索インターフェイスの機能強化の最新のサブカテゴリでは、 QuinturaとKartooが導入され 、革新的な検索視覚化技術が提供されています。 Quinturaは、検索結果のクラスターを提供する「クラウド」を提供しています。 一般に、アイデアはClustyに似ていますが、グラフィック表現は異なります。 Kartooはまた、グラフィカルな表現を使用して結果をクラスター化します。 問題は、検索結果の複雑な表示です。これにより、検索結果が主流になる可能性は低くなります。 グラフィカルな視覚化についてのアイデアはありますが、直観的またはシンプルと呼ぶことのできる検索エンジンはありません。 この化身では、視覚化された検索は広まりません。

これらのインターフェースの機能強化の多くは正しくて興味深いものですが、いずれもGoogleを打ち負かすことはできません。 主な理由は、再生のしやすさです。 もちろん、法的観点からは、正確なクローンは作成されませんが、市場で大きなシェアを獲得している技術がある場合、Googleは必要な改善を簡単に行うことができます。 さらに、SearchMeshを使用した実験では、一部の垂直検索エンジンのマップがすでに混乱している可能性があります。

垂直検索



垂直検索の利点についてはすでに多くのことが書かれていますが、Googleを居住地から移動する試みはありません。 これらのテクノロジーは、検索結果、インターフェースを改善し、特定の業種にも焦点を当てています。 多くのプレイヤーが、ブログ、広告、製品、仕事などの検索市場に参入しています。

簡単な例を考えてみましょう。 Web 2.0でブログを検索する場合、GoogleまたはGoogleブログ検索を使用しますか？ Googleでさえ、ブログや音楽などの一部のトピックの垂直検索の利点を認識しています。 したがって、垂直検索はすでに行われており、シーンを離れることはありません。 重要な問題は、垂直検索を食い物にする「ケーキ」です。

結論



一般的に、この分野での重要な活動にもかかわらず、Googleは近い将来「丘の王」であり続けるでしょう。 検索ケーキを食い止めるには、さまざまなアプローチが多かれ少なかれ成功しますが、大きな歯を作るには時間がかかり、完璧な実装、かなりの量のマーケティング費用、そしてもちろん、新しい技術が必要です。 これは、成功のコンポーネントの単純な組み合わせではありません。

最初の3つのタスクは複雑ではありますが実行可能であるという事実にもかかわらず、テクノロジーの改善には依然として理解が必要です。 一方で、より良い結果をより早く得られるはずであり、他方では、別のプレーヤーが市場に登場する可能性があります- パーソナライズされた検索 。 このテクノロジーでは、結果はPageRankを使用してランク付けされるのではなく、関心に従ってランク付けされます。 Googleと比較しても、垂直検索と検索検索の組み合わせにより、検索結果が根本的に改善される可能性があります。 ただし、すでに上で述べたように、Googleは栄光にとどまるつもりはありません。
