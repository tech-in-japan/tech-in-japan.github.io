Symantec Backup Exec：LinuxにインストールされたOracleのバックアップ

Backup Execは、信頼性の高いエンタープライズレベルのバックアップシステムです。 この製品のインストールと設定はすでにハブで説明されています。問題はなく、すべてがシンプルで直感的です。 このバックアップツールの大きな利点は、このプログラムが企業で使用されるほぼすべてのソフトウェアをバックアップできることです。コピーの主な機能をリストします。

•Windows、Linux、Macintoshをコピーできます。

•VMWareおよびESX

•Active Directoryドメイン

•システム状態

•Microsoft Sharepointファーム

•Microsoft Exchange

•MSSQLおよびOracleデータベース、Oracle Real Application Clusters

•ユーザーリソース

•その他。

これは大きな機会のリストではありません。利点は、複数のユーティリティを1つの製品に置き換えて、すべてのバックアップを1か所で見るか、クラッシュメッセージを受信できることです。

製品管理は直感的ですが、LinuxにインストールされたOracleのバックアップとリカバリのオプションには特に微妙な違いがあります。ここでは、ドキュメントを調べて把握する必要があります。 LinuxにインストールされているOracleのバックアップ中に発生する可能性のある最も基本的な問題とその回避方法を説明しようとします。

最初に、Linuxサーバーにクライアントをインストールする必要があります。 Linuxのメニュー「サービス-他のサーバーへのエージェントのインストール」の通常の方法は、常に機能するとは限りません。 したがって、手動でインストールします。BEディストリビューションには、LinuxUnixMacディレクトリにRALUS_RMALS_RAMS-xxxx.tar.gzアーカイブが含まれています。 それをLinuxサーバーにコピーし、解凍して起動します（インストールと構成はrootとして実行されます）： 



tar –xzvf RALUS_RMALS_RAMS-xxxx.tar.gz cd RALUS_RMALS_RAMS-xxxx ./installralus




すべての質問に肯定で回答し、Backup ExecがインストールされているIPサーバーを示します。 インストール中に、beoperグループを作成するよう求められます。 LinuxでOracleを実行するユーザーをすぐにこのグループに追加します。 その後、Oracleと連携するようにエージェントを構成する必要があります。

BEクライアントをセットアップするときは、Oracleデータベースサーバーが実行されている必要があります。 問題が発生した場合： 



 oracle: error while loading shared libraries: libhasgen10.so: cannot open shared object file: No such file or directory Failed to login.




クライアント構成をセットアップする前に、Oracleの環境変数を設定する必要があります。 このような問題が発生した場合、将来サービスを開始するときに変数を設定する必要があります。 



 --->>> No Oracle instances were discovered. Do you want to enter one? (Y/N):Y Enter the Oracle instance: SM1 --->>> The entry could not be validated. Ensure that the instance is configured correctly.




このようなメッセージは、oratabファイルがない場合に発生し、そのファイルとその内容（SIDとORACLE_HOMEへのパス）に対する権限を確認します。

クライアントのインストールと構成が完了したら、サービスを再起動してサーバー側のBEに移動します。 



 /etc/init.d/VRTSralus.init restart


バックアップを構成するには、Oracleデータベースがアーカイブモードになっている必要があることに注意してください。 



 SQL> select log_mode from sys.v$database; LOG_MODE ------------ NOARCHIVELOG ----    SQL> shu immediate SQL> startup mount SQL> alter database archivelog; SQL> alter database open;




Backup Execのサーバー側で、このようなアクションをすぐに実行します（接続時の多くの問題を回避するのに役立ちます）。 



  >  >  > Oracle >   >  (  ,   Oracle   ,   ).




その後、すべてが正常である、メニュー>ファイル>新規>バックアップタスク。

Oracle Databaseのホットバックアップを構成する必要がある場合、データベース（データベースファイル）をフォルダとしてコピーする必要はありません。


すべてが以前に正しく行われた場合、データベースサーバーの名前がLinux / Unixサーバーセクションに表示されます。[ROOT]の横にある[+]をクリックすると、資格情報が要求されます。[作成]をクリックし、データベースを実行しているLinuxユーザーの資格情報を指定します。 + Oracle Databaseでは、作成ボタンをクリックして、データベースユーザーデータ、つまりクライアントに指定されたsysdba権限を持つsysユーザーを指定する必要があります。 これで、ORACLE_HOMEを保存するディレクトリを選択し（データベースファイルのあるフォルダーのチェックを外し、データベースが停止したときにこれらのファイルをバックアップできます）、「Oracle Database」の横のボックスをチェックします（「ホット」コピーの場合）。 タブを通過して、すべてが十分に明確です。 Oracleタブで、「コピーされたログファイルを削除する」チェックボックスを選択することをお勧めします-これは、アーカイブログをバックアップサーバーにコピーした後、データベースサーバーから削除されることを意味します [デバイスとメディア]タブでディスクまたはテープへのコピーを設定し、スケジュールを設定します-[すぐに実行]をクリックします-設定したパラメーターの一般的なリストが生成され、[OK]をクリックします。[タスクモニター]セクションに進行状況が表示されます。







Oracleデータベースが大きく、頻繁に使用される場合は、追加のタスクを実行してアーカイブログのみをコピーすることをお勧めします。これにより、情報損失の時間が短縮され、データベースのディスク領域がより頻繁にクリアされます。
