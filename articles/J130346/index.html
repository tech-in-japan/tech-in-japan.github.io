マイクロソフトはHadoopをサポートしています

Microsoftは、Windows ServerおよびWindows AzureでHadoopクラスターを作成するユーザーをサポートすることを決定しました。 Hortonworksとの戦略的コラボレーションの一環として、 Hadoopのスタックが開発されます。 マイクロソフトは、オープンライセンスの下でコードを公開し、開発の最後にApacheプロジェクトにコミットすることを約束します。



さらに 、Microsoftは、HDFS、Hive、PigなどのさまざまなHadoop関連テクノロジー用の「ダウンロード、インストール、および構成可能なシンプルバージョン」を作成します。 これにより、企業顧客によるHadoopの使用が促進されます。



Hortonworksは、YahooでHadoopに取り組んだ開発者を集めています。 ご存知のように、Yahooはこのオープンソースプロジェクトの開発に多大な貢献をし、2008年2月に1万台以上のLinuxサーバーからHadoopで最大のクラスターを立ち上げました。



Windows Azure用のHadoopスタックのテストバージョンは、2011年末までに、Windows Server用に-来年リリースされます。 新しいスタックは、Excel、PowerPivot、PowerViewなどのMicrosoftビジネスソフトウェアで動作します。



これは、HadoopをサポートするためのMicrosoftからの最初のステップではありません。 先週、SQL ServerおよびParallel Data Warehouse用のHadoopコネクタの最終バージョンがリリースされました。



同時に、Microsoftは独自のシステムの開発を続けており、HadoopとMapReduceの代替となるはずです。 確かに、コードネームDryad （Hadoopのアナログ）およびDaytona （MapReduce）というコード名のプロジェクトの作業は、まだ完全ではありません。



同時に、Microsoftは、SQL Server 2012（Denali）の次のバージョンが2012年前半にリリースされ、現在はCTP 3ステージにあることを発表しました。
