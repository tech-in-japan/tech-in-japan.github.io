クラウドには新しいものは何もありません

私は「クラウド」という用語の大ファンではないことを認めなければなりません。 彼は1マイル離れたところからマーケティングや広告の匂いがしており、実際、彼には超自然的なものは何もありません。 組織は、長年にわたってIT要素の一部を外部委託していました。 おそらく、これに最もよく使用されるサービスはWebホスティングでした。 社外の誰かの機器にデプロイされた単一のWebサイトを削除するか、サードパーティのデータセンターにサーバーを配置するかのいずれかです。

私たちの議論では、「クラウド」の概念は、ITの一部の要素を指しているだけであり、ITの一部はアウトソーシングされ、基盤となるインフラストラクチャを隠していることに同意します。 たとえば、ホスティング会社のデータセンターにサーバーを配置し続ける場合、通常、ネットワークの内部構造、インターネットへの接続方法、ルーターの種類などに関する詳細情報はありません。この場合のデータセンターは、あなたがお金を払う抽象的な何か。 たとえば、Windows AzureやAmazon Elastic Cloudが提供する最新のクラウドコンピューティングモデルでは、仮想マシンが実行されている物理ホストがわかりません。 物理層の抽象化、およびデータストレージ、ネットワーク通信などの関連要素については、お金を支払います。 「サービスとしてのソフトウェア（SaaS）」サービスを使用する場合、基盤となるインフラストラクチャ全体を抽象化するために料金を支払うため、注文したソフトウェアの作業にどの仮想マシンが関与しているかさえわかりません。




インフラストラクチャの大規模または小規模のサブシステムの数がクラウドサービスプロバイダーの手に渡ったとしても、これらのコンポーネントはビジネスの一部です 。 通常の機能に依存する重要なビジネスアプリケーションとプロセスがまだあります。 リモートインフラストラクチャを制御できなくなり、特定の時点で内部で何が行われているかを理解する能力が低下します。



この場合、特殊なツールは完全に使用できなくなります。 間違いなく、アウトソーシングの一般的な考え方の一部は、現在他の人が取り組んでいるタスクについてあまり心配することではありませんが 、社外に移動したITシステムは、 あなたが個人的に責任を負うビジネスのサポートにまだ関与しています 。 したがって、少なくとも、クラウドに取り込まれた要素の作業が環境の残りの部分にどのように影響するかを確認する必要があります。 さらに、問題の特定の原因で信頼できる「指を突く」能力が必要です。たとえそれがアウトソースされたアイテムであっても、障害の除去に直接影響を与える能力はありません。 これは、統合監視がIT環境において正当な位置を占める場所です。 たとえば、図2.4。 Amazon Web Servicesなどの一部の外部委託コンポーネントを含む、一部のインフラストラクチャコンポーネントの一般的なステータスを示す、非常に単純な「統合モニタリングダッシュボード」が表示されます。
