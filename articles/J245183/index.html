古いWindows PCからシンクライアントを作成する方法

遅かれ早かれ、動作が遅いために1台以上のPCを交換する必要があるという疑問が生じます。



最も簡単な方法は、PCを発明するだけで交換するだけです。



最も簡単な方法は、ターミナルサーバーまたは仮想デスクトップのバージョンで「リモートデスクトップ」の実装を開始することではありません。



HP 、 DELL、またはその他のブランドのシンクライアントのコストは、本格的なPCのコストと比較することができ、古いPCをシンクライアントとして使用すると、十分に長い期間の寿命が延びます。



古くなったPCに対処する方法：

-Windows PCを離れると、ユーザーはリモートデスクトップに接続されます。

-ネットワーク経由でPCをダウンロードします。これは、 thinstationの Linuxオプションの1つです。

-ローカルバージョンのLinux、 options seaをPCにインストールします。



さらに、Windowsのバージョンについて説明します。このようなシンクライアントには、Linuxオプションと比較した場合にいくつかの利点があります。





なぜ私はこれをすべてしたのですか：



-ターミナルサーバーで作業するために転送する必要のあるユーザーがいるリモートオフィスがあります。ドメインでグループポリシーを使用すると、オフィスにいなくても、PCを交換しなくても目的の結果を得ることができます。

-HP、Wyse / DELLのシンクライアントのユーザーインターフェイス用のWindowsとLinuxの両方のオプションは、さまざまな理由で私には向いていません。



Windowsオプションの利点：

-RDP / RemoteFXの完全サポート。

-リムーバブルメディアの完全サポート。

-ローカルプリンターを使用する機能。

-銀行のクライアントにスマートカードを使用する機能。

-Windows Media Playerを使用して、ブレーキやサーバーへの負荷をかけずにビデオ/オーディオ再生をシンクライアントにリダイレクトすると、1080pビデオを視聴できますが、これは別の話です=）。



結果から始める場合：



これは、Windows XPがPCにインストールされている場合、ユーザーのデスクトップを読み込むように見えます。





これは、Windows 7がPCにインストールされている場合、ユーザーのデスクトップを読み込むように見えます。





スライドに表示される結果を得るには、PCとユーザー用のADドメインといくつかのグループポリシーが必要です。



XP SP3を搭載したPCでは、rdpクライアントKB969084およびFixit50588の更新をインストールする必要があります。高度なグループポリシーの場合は、更新KB943729をインストールする必要があります。



主なポリシー：



No. 1-ユーザーはシングルサインオンを有効にする必要があります。このポリシーをドメイン全体に配布しています。





No. 2 -PCの場合、個別のOUを作成し、このOUのグループポリシーを閉じます。





№3-新しいOUで、ユーザーを「wscript c：\ thinPC \ thinPC.vbs / nologo / b」に変更するポリシーを作成します。



3つのファイルをターゲットPCにコピーする必要があります。これには高度なグループポリシーを使用します。

これらのファイルは、グループポリシーの中央リポジトリ\\ドメイン名\ SYSVOL \ドメイン名\ポリシー\に配置することをお勧めします。これにより、ドメインコントローラーのいずれかが使用できない場合のフォールトトレランスが確保されます。

ドメインコントローラーでは、このネットワークリソースはフォルダーCに対応しています。\ Windows \ SYSVOL \ sysvol \ドメイン名\ポリシー



ThinPC.cmdファイルの内容

C：\ BGInfo \ Bginfo.exe "C：\ BGInfo \ config.bgi" / NOLICPROMPT /タイマー：0

mstsc "c：\ thinPC \ thinPC.rdp"

シャットダウン-l
BGInfoは、必要なデータを含む碑文をユーザーのデスクトップに追加します。BGInfoの結果はスライドに表示されます。

mstscはrdpクライアントを起動し、リモートセッションの終了後、shutdown -lコマンドを実行して、シンクライアントでユーザーセッションを終了します。



説明したスクリプトの操作中、ユーザーはウィンドウ内のコマンドの実行を観察し、ウィンドウを非表示にするために、 VBSによってスクリプトが起動されます。



thinPC.vbsファイルの内容

薄暗いシェル

oShell = WScript.CreateObject（ "WSCript.shell"）を設定します

oShell.run "C：\ thinPC \ thinPC.cmd"、0

oShell = Nothingを設定


ThinPC.rdpファイルの内容

これは、必要な接続パラメーターを含むrdpファイルです。

-全画面で作業するときは、接続パネルの表示をオフにする必要があります。

-ローカルドライブの転送を無効にしますが、後で接続されたドライブの転送を許可します。これにより、ユーザーはリモートセッションの開始後に接続されるリムーバブルメディアを操作できます。

-Windows 7の場合、 RemoteFXプロトコルを使用するには、色深度を32ビットに設定し、10メガビット/ LANの接続速度を指定する必要があります。

-サーバー証明書が自己署名の場合、「サーバー認証」セクションで警告を無効にする必要があります。


それだけです。これをユーザーに置き換えると、シンクライアントのブートプロセスが通常のPCのブートプロセスにできるだけ近くなります。



提案されたスクリプトの主な欠点は、ユーザーが自分で画面解像度を選択できないことです。しかし、22〜24インチのモニターを要求し、その上で文字を増やすように要求するとき、私は正直わかりません。

そのような場合、ターゲットPCにVNCサーバーをインストールし、それで解像度を変更します。





マイナーポリシー：



No. 4-このメッセージをオフにするには、ユーザーは通知しないようにボックスをオンにするだけです。



管理手段によってプロセスを自動化するには、レジストリにキーを追加する必要があります。

[HKEY_CURRENT_USER \ Software \ Microsoft \ Terminal Server Client \ LocalDevices]

「サーバーアドレス」= dword：0000000d


No. 5 -RDP接続のキャッシュを増やす必要があります。WindowsXPの場合は重要ですが、Windows 7の場合はキャッシュを増やすことなく実行できます。
[HKEY_CURRENT_USER \ Software \ Microsoft \ Terminal Server Client]

「BitmapCacheSize」= dword：0000ffff
グラフィックの最適化、 ターミナルサービス、およびグラフィック集中アプリケーションの詳細 。



No. 6- 対話型ログオン：CTRL + ALT + DELは不要

パラメーターを無効に設定します。



No. 7- 対話型ログオン：最後のユーザー名を表示しない

Enabledパラメーターを設定します。



No. 8- 対話型ログオン：ログオンしようとするユーザーのメッセージテキストと対話型ログオン：ログオンしようとするユーザーのメッセージタイトル

ユーザー向けのタイトルとテキストを入力します。最も単純なケースでは、サポートセンターの連絡先を指定する必要があります。



No. 9-シンクライアントで視覚効果を無効にするには、レジストリにキーを追加する必要があります。

[HKEY_CURRENT_USER \ Software \ Microsoft \ Windows \ CurrentVersion \ Explorer \ VisualEffects]

「VisualFXSetting」= dword：00000002


No. 10- 電源オプション

高度なグループポリシーを使用して、電源ボタンを押すとThin Clientがオフになる電源プランを作成する必要があります。



No. 11- ソフトウェア制限ポリシー

ソフトウェアの起動制御を設定することをお勧めします。このメカニズムはWindowsXPおよびWindows 7 PROで動作します。

AppLockerはより柔軟ですが、Windows 7 Enterprise以降でのみ機能します。

シンクライアントの場合、柔軟性は必要ではなく、目的はマルウェアを起動する可能性を排除することです。



No. 12- 自動再生をオフにする

リムーバブルメディアの自動起動を無効にするには、すべてのドライブに対して有効を設定する必要があります。



No. 13- このコンピューターからサポートされている他のRemoteFX USBデバイスのRDPリダイレクトを許可する

USBデバイスを転送する場合は、管理者とユーザーのポリシーを有効にします。



No. 14-システムの再起動時に指定された日数よりも古いユーザープロファイルを削除する

パラメーターを180日に設定しました。ポリシーはWindows 7でのみ機能します。



No. 15- ユーザーアカウント制御

UACが私を妨害し、これで切断します。



No. 16-デスクトップの背景を置き換えるには、美しさを犠牲にする必要があります。



Windows XPの場合、このレジストリキーは、ログインおよびパスワード入力画面の壁紙を担当します。

背景付きのファイルはどこにでも配置できますが、bmpファイルである必要があります。

[HKEY_USERS \ .DEFAULT \ Control Panel \ Desktop]

「壁紙」=「C：\\ thinPC \\ rd.bmp」

"WallpaperStyle" = "2"


Windows 7の背景を変更するには、キーをインストールし、background％WindowsDir％\ System32 \ oobe \ info \ backgrounds \ backgrounddefault.jpgでファイルを転送する必要があります。

[HKEY_LOCAL_MACHINE \ SOFTWARE \ Microsoft \ Windows \ CurrentVersion \ Authentication \ LogonUI \ Background]

「OEMBackground」= dword：00000001


また、シンクライアントでユーザーセッションの背景を設定する必要があります。

[HKEY_CURRENT_USER \コントロールパネル\デスクトップ]

「壁紙」=「C：\\ thinPC \\ rd.bmp」

"WallpaperStyle" = "2"
