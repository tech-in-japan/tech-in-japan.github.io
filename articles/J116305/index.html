BBCでF1をインターネットでライブで見たとき

この記事は、ウクライナとベラルーシのF1ファンにとって興味深いものになるでしょう。 残念ながら、ウクライナとベラルーシではF1を放送する権利を購入したチャンネルはないため、レースをライブで観戦するには、さまざまなトリッキーな方法を考え出す必要があります。



私はBBCの放送を見るのが本当に好きです。 他のチャンネルと比較して、彼らはレース、彼ら自身の多くのインタビュー、分析よりもずっと早く放送を開始します。 コメンテーター：デビッドクールハード、エディジョーダン、マーティンブランドル。 私の意見では、誰もが王室の人種についてコメントすることはほとんど不可能です。 かつてBBCを見た人は、他の何も見ることができません。 確かに欠陥があります...英語の知識が必要です 当然、翻訳なし。



私はBBCが素晴らしいiPlayerオンライン放送サービスを持っていることを知っていました。 BBCのすべてのテレビおよびラジオチャンネルの生放送を視聴する機会を提供します。 さらに、放送の1〜2日後に、プログラムを視聴およびダウンロードできます。











唯一の欠点は、これらすべてが英国からしか入手できないことです。 「お住まいの地域ではご利用いただけません」というメッセージが他の全員に表示されます。







まさにこの領域に到達するために、私はトリックに頼らなければなりませんでした。 私たちはいくつかの英語プロジェクトに取り組んでおり、イギリスにサーバーがあります。 まず、サーバー上でプロキシを構成して、BBCを経由して英語のふりをするようにしました。 しかし、その後、小さなグーグルは、他の場合に役立つことができるよりシンプルでエレガントなソリューションを見つけました。



SSHを介したFirefoxトラフィックのトンネリングという記事を見つけました。 この記事では、サーバー上で何も変更せずにSocks Proxyを作成する方法について説明します。 唯一の要件は、サーバーへのSSHアクセスです。



すべてが単純に行われるのではなく、非常に単純に行われます。 プロキシモードでsshを起動するには、パラメーターを指定してコマンドを実行する必要があります



ssh -fND localhost:12345 login@server.net







12345は、使用されていないポートの番号です。 パラメーターの詳細については、sshのマニュアルページを参照してください（例： unixhelp.ed.ac.uk/CGI/man-cgi? ssh +1） 。



その後、ブラウザを構成して、127.0.0.1のSOCKSプロキシおよび選択したポートで動作するようにします。 すべてのアクションについては、記事で詳しく説明しています 。



Windowsで同じことを行うには、PuTTYを使用できます。 セッションを開始するときに、[接続]-> [SSH]-> [トンネル]に移動し、ポートを追加します。 必ず[動的]を選択して[追加]をクリックしてください。 この場合の宛先は省略できます。 その後、サーバーに接続し、プロキシが必要になるまで端末を閉じないでください。







iPlayerはFlashを使用します。 Linuxではどのように動作するのかわかりませんが、Windows FlashではIE設定を使用してインターネットにアクセスします。 したがって、IEがプロキシで動作するように設定する必要があります。 そうしないと、Flashはインターネットに直接アクセスし、iPlayerは「お住まいの地域では利用できません」というエラーを生成します。 IEですべてを設定して実行します。



iPlayerは、Adobe Airで書かれたデスクトップアプリケーションとしても利用できます。 お気に入りのプログラムをコンピューターにダウンロードして、アプリケーションまたはwma形式で表示できます。 このオプションは、コミュニケーションが非常に悪い人に適しています。



土曜日と日曜日にはすべてがうまくいきました（BBCは無料の乗り物も表示します）。 キエフのVolyaプロバイダーの速度（ウクライナ人は理解します）は十分でした。 常に、ビデオが「スローダウン」したのは2回だけでした。 通常の動作では、1 Mb / s程度の速度で十分です。 速度が十分でない場合、iPlayerには「低帯域幅」モードがあります。 画質は悪化していますが、まだ許容範囲です。



私たち全員のためにあなたの視聴と興味深い季節をお楽しみください！ 始まりは興味深かった。



UPD：

コメントでは、彼らはイングランドからではなくBBCを見るためのいくつかのオプションを提案しました

1） 無料のプロキシを使用します。 リストはproxy-list.orgにあります。 そこですぐに見つけたプロキシは正常に機能し、ビデオを高速で見ることができました。 この方法の主な欠点は、誰も何も保証しないことです。 つまり ブロードキャストの途中で、プロキシの速度が低下したり、プロキシが完全にシャットダウンしたりする場合があります。 1つの美徳-景品。

2） VPNを購入します 。 KebrumとIvacyが言及されました 。 確かに他の同様のサービスがあります。 彼らは同じことをします-彼らはあなたにVPNとサーバーを選択する能力を提供します。 サービスは有料です。 デモアクセスでは、英語のサーバーへの接続は許可されません。 Kebrumの最安値は、50 Gbのトラフィックに対して3.90ドル/月です。 月に2〜3レースで十分です。 完全なレース記録は、約3 Gbに加えて資格とテストの重量があります...
