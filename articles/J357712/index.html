CMake 3.10とAndroid Studioをフィーチャーした探偵物語

免責事項：以下に記載されているものはすべて良い習慣ではありません。 このテキストはアクションのガイドとして読まれるべきではありません-その役割はむしろ面白いです。 同じ理由で、著者（私）に言語、ツール、OS、ハードウェア、性別、居住国を変更するよう助言することは意味がありません。



プロジェクトが1つあります。 彼はCMakeを使用してビルドし、また、 Hunterと呼ばれるC ++パッケージマネージャーを使用して、CMakeとうまく統合しています。 このプロジェクトは、いくつかのプラットフォーム用に組み立てる必要があります。そのうちの1つはAndroidです。 HunterはAndroidの依存関係を問題なく収集しますが、CMake> = 3.7のバージョンが必要です。3.7でこのプラットフォームのサポートが追加されたためです。 これは非常に重要な詳細です。



CMakeは、 NVIDIA Nsight Tegraプラグインを使用して、Windows / Visual Studio用のAndroidプロジェクトの生成をサポートします。 残念ながら、このプラグインはVisual Studio 2017にインストールされておらず、1つのプロジェクトのために2015をインストールしたくありませんでした。 一方、VS2017にはAndroidのビルドサポートがありますが、CMakeからそのようなプロジェクトが生成されることはありません。 より正確には存在しますが 、 MicrosoftのCMakeの自作バージョンが必要であり、その開発はバージョン3.4の領域で放棄されました。つまり、私には合いません。



さて、Visual Studioではない場合は、Android Studioです！ CMakeはもちろんサポートしていませんが、すでにバージョン2.xからサポートしています。 build.gradleでCMakeLists.txtへのパスを指定し、同期を開始すると、すべてが機能するはずです。 ただし、機能しませんが、「CMakeがインストールされていません」と表示されます。 理由-しかし、Androidリポジトリからダウンロードする必要があり、バージョン3.6でフリーズした独自のフォークCMakeでも動作するためです。



行き止まりのようです。 ただし、実際にはそうではありません。バージョン3.0以降、Android Studio は 、「Androidサポートの改善」により、3.6より古いCMakeの外部バージョンで動作できるようになりました。 local.propertiesにcmake.dirフィールドを記述するだけで十分で、すべてが機能するはずです。 しかし、それは機能しませんが、言う...うーん、正直言って、私は彼がそこに言ったことをすでに忘れていましたが、ポイントはGoogleで採点されたエラーテキストがバグトラッカーの会話につながったということですAndroid Studioを使用した3.8 CMakeは何らかの方法で誤って通信するため、これらはすべてバージョン3.9で修正する必要があります。



そして、私はCMake 3.8を持っています。 さて、それは問題ではありません。サイトにアクセスして、バージョン3.10の最新のCMakeをダウンロードします（実際、3.10は3.9よりも良いのでしょうか、それとも少なくとも悪くありませんか？）。 インストールすると、すべてが機能するはずです。 ただし、機能しませんが、「CMake Serverとの通信エラー-ログを参照してください」と表示されます。 ログは簡潔に宣言します： 



CMAKE SERVER: [== "CMake Server" ==[ CMAKE SERVER: {"supportedProtocolVersions":[{"isExperimental":true,"major":1,"minor":1}],"type":"hello"} CMAKE SERVER: ]== "CMake Server" ==]


ご覧のとおり、間違いはありません。 ところで、はい、しかしCMake Serverはどうですか？ 初めて聞いた。 ドキュメントにアクセスして、これは外部プログラムがCMakeプロジェクトに関する情報を機械可読形式で受信できるようにする新しいCMake操作モードであり、さまざまなIDEおよび追加ツールとの統合を簡素化することを読みます。



これが探偵の始まりです。 最初の証拠は、サーバー応答の「メジャー」および「マイナー」フィールドです。 プロトコルバージョンがあるため、誰かがそれをチェックでき、「間違った」バージョンでは動作しません。 仮定は非常に可能性が高いが、仮定のみ-確認を取得するのが良いでしょう。



実際、この時点で私は一時停止して警察に報告し、Android Studioのバグトラッカーのバグを取得することにしました。 残念なことに、バグに対する反応はなく、数日後にトラッカーから消えました-WONTFIXでも閉じられませんでした。つまり、トレースなしで消えました。 怒ったので、私は自分の調査を終えることにしました。



おそらく読者の中には、Android Open Source Projectで積極的に作業している人、または少なくともそのデバイスに精通している人がいます。 私のさらなる冒険を彼らに笑わせてください。



CMleとの統合を担当するGradleのAndroidプラグインのソースを見つける必要がありました。 普通のまともな市民として、Googleで「Android gradle plugin source」というフレーズを採点し、それに応じてリポジトリページを取得しました。 ただし、Webインターフェースを使用して簡単に検索すると、masterでの最後のコミットは2年前であることがわかりました。 そして、新鮮な種類はどこですか？ はっきりしない！



同じ検索の2番目のリンクから、 「AOSPソースコードをダウンロードする方法」という指示がありました。 これを行うには、簡単ではない、と言わざるを得ません。Googleは、いつものように、人のようではなく、Gitから特定のプロジェクトをダウンロードすることはできません。 リポジトリと呼ばれる独自のツールを使用して使用する必要があります。

それで十分でした。 Google検索で名前を付けて、すぐにリポジトリで名前を見つけました 。 確かに、これはstudio-2.2-preview3ブランチのファイルでした。つまり、CMakeの外部バージョンとの統合がなかった以前のファイルです。



それから、はるかに早く来るはずだったことがわかりました-2年前、プラグイン開発者はマスターへのコミットを停止し、ブランチですべての開発を開始し、結果をメインにアップロードしませんでした。 私を正当化するのは、このスタイルでGitを使用したことがないということです。一般的に、見たすべてのプロジェクトは、通常、最終的にmasterの変更をマージしました。 ただし、ブランチのリストを詳しく調べる必要があることを知っていたとしても、正しいブランチを見つけるのは事実ではありません。何百ものブランチがあり、作成日が表示されないため、名前がわかりませんでした。

コードを見ると、それが本当にプロトコルバージョンであることがわかりました。 プラグインはそれを明確な通信1.0（getSupportedVersion関数内）と比較し、新しいCMakeは1.1を使用しました（ところで、次のバージョンでは、すでに1.2になります）。 CMakeコードを調査したところ、バージョン1.1で根本的に破壊されるものは何も追加されないことがわかったため、この愚かなチェック以外にAndroid Studioが動作するのに障害はありませんでした。 したがって、ディスク上にgradle-core-3.0.0.jarファイルを見つけ、そのクラスを削除し、Java Bytecode Editorを使用して、すべてが機能するようにチェックを変更しました（ 免責事項：自宅で繰り返さないでください）職場）;トリックは必死で非常に怒っているプログラマーによって行われました ）。
