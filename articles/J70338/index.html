1.はじめに。



ホームサーバーについて説明する必要があるのはなぜですか。 誰もが自分で決める。

DSの要件は次のとおりです。

  • 鉄-安価、小型、ノイズレス。
  • オペレーティングシステム-Ubuntu 9.04デスクトップ。
  • Web開発キット（LAMP）+ SVN;
  • ファミリー内のコンピューターの共有リソース。
  • 友人へのアクセスを提供するには-FTP。


利用可能なすべてのリンクをたどりながら、初心者のubunt-gadfly、または完全な記事でさえ必要な包括的なガイド記事が見つかりませんでした。 そのため、このマニュアルは、いわゆる「ステップバイステップインストール」と呼ばれ、それ自体が部品にまとめられています。 完了後、Ubuntu 9.04のクリーンインストールと構成をゼロから説明することをすべて1つの記事で説明することにしました（そして開発はまもなくブログに掲載されます）。



別のポイント。 ホームサーバーは専用チャネルの存在を意味し、できれば専用IPアドレスを使用しますが、DynDNSは一部の人には非常に適していますが、プロバイダーに10 UAH /月（$ 1.2）を支払うことを選択したため、この記事の質問はDynDNSです。まだ考慮されていません。 良好な速度の存在-世界に4メガビット/秒、ウクライナに100メガビット/秒（UA-IX）、ウクライナ内のサーバーを最大限に使用することが可能になります。



2.鉄。



警告：

ハードウェアは何でも構いませんが、確認する必要があるのはインターネットへのアクセスがあることだけです。そのハードウェアはUbuntuでサポートされています。 つまり ARMに何らかの特別な超変更を加えた場合、何かがうまく機能していないと文句を言わないでください。 私はこのために別のマシンを購入することを個人的に選択しました。



DSのハードウェアの選択時に、別の記事（http://habrahabr.ru/blogs/hardware/68805/）で既に購読を解除しました。 読むのが面倒、これ：

マザーボード -Zotac IONITX-A（Intel Atom N330）;

RAM -2 Gb;

ウィンチェスター -Seagate-Maxtor DiamondMax 23 500Gb（7200rpm、16Mbキャッシュ、SATA II、3.5インチ）;

ケース -CFI A8989;

プロペラ（ブロー用） -Zalman Ultra Quiet 80mm。

追加のうち、CD / DVDドライブで再生するのではなく、フラッシュドライブから直接インストールするために、1GBの別のフラッシュドライブを購入しました 。



PS：これはSeagate-Maxtorのハードドライブであり、WDキャビネットの方が確かに優れていることを指摘しないでください。しかし、残念ながら、私たちの街では購入時に痕跡もなく消えてしまい、少なくとも1つは一時的に使用する必要がありました。



DSでのWi-Fi-すぐに使用できます。 まったく意味がありません。 54Mbpsは100％の信号レベルでわずか6.75MB / sを提供し、チャンネルをトレント+ホームネットワーク（非常に現実的な状況ではサーバーからビデオを視聴し音楽を聴く）に分割する場合、実際の速度は1.5-2 MB / sです。何ですか-受け入れられません。 私の選択は、Zotac IONITX-Aボードが1Gbイーサネットポートを保持しているため、イーサネット経由でルーターに直接接続することです。 Wi-FiルーターはD-link DIR-615 +からD-Link DIR-655に変更されました。後者は1Gb WAN + 4 x 1Gb LANイーサネットポートをサポートしているためです。 一緒に、それは情報転送の最大速度を達成することを可能にしました。 Wi-Fiカバレッジを強化するために、自宅のラップトップとwi-fiデバイスでは、2つのD-Link ANT-24 0070Cアンテナが使用されました（通常の2dbiの代わりに7dbi）。



ルーターはDHCPを介してインターネットを配布するように構成され（静的にだまされたくはありませんが、あまり意味がありません）、サーバーは自然に静的であり、ルーターはポートを内部固定IP 192.168.0.66に転送するように構成されています。

  • SSH： 44000;
  • HTTP / HTTPS： 80/443
  • FTP： 21


PS：私の意見では、Wi-Fi 802.11nをインストールして、すべてのデバイスでアイロンを802.11nに置き換える可能性を忘れたくないので、お金の無駄になります。 時間が経つにつれて、これが必要になりますが、これまでのところ、ホームサーバーにイーサネットを選択しました。 あなたは私が間違っていると思う？ あなたと手にカード。 :)



3.フラッシュドライブを調理します。



個人的には、WindowsでUSBフラッシュドライブを準備しました。 使用したソフトウェア：

  • HP USB Disk Storage Format Tool（検索エンジンで最新バージョンを探しています）;
  • Unetbootin（http://unetbootin.sourceforge.net/）


クイックインストール：

  • HP USB DSFTでUSBフラッシュドライブをフォーマットします。 ファイルシステムはFAT32です（必須！）。
  • Unetbootinを実行します。
  • [配布]-[Ubuntu]-> [9.04 Live]を選択します。
  • ディスクイメージ：ISO->省略記号をクリック-> Ubuntu 9.04ディストリビューションキットでディスクを選択します。
  • タイプ：USBドライブ、ドライブ：フラッシュドライブ名。
  • [OK]をクリックします。
  • コピーが終了するまで、「興味のないもの」をさらに読みます。
  • フラッシュドライブを引き、DSを突きます。


4. Ubuntu 9.04をインストールし、セットアップを開始します



おそらく子供でも自分でUbuntuを置くことができます。 フラッシュドライブからさらに高速で便利になりました。だれでもBIOSでUSBフラッシュドライブからブートをセットアップできると思いますか？ Ubuntuの最大の利点は、ほとんどすべてのドライバーがすぐに使えることです。 私の場合、NVidia GeForce 9300Mドライバーだけではありませんでした。 個人的に使用するいくつかのインストールポイントについて説明します。

1. LiveCDから起動するときのインターネットの可用性。

なぜこれが必要なのですか？ インストール中に、言語-「ロシア語」を選択します（必要に応じて、デフォルトでAlt + Shiftに切り替えます）。 Ubuntuはインストール中に独立して、必要なすべてのローカライズファイルをプルアップします。 私は彼らの検索で将来プレイしようとはしなかったし、わずかな欲求もありませんでした。 したがって、私は事実としてそれを取ります-インターネット接続はインストールする必要があります。

2.ハードドライブの故障。

Linuxでディスクを分割する方法に関するすべてを理解したいので、さらに再インストールしてもメインパーティションのすべての情報が失われないように、この問題に関してどれだけのドキュメントが収集されました。



この決定が下され、500 GBのネジが次のように分割されました。

プライマリパーティション-> RaiserFS-> 300Mb->“ / boot”

プライマリセクション->スワップ-> 4096Mb

プライマリパーティション-> Ext3-> 40000Mb->「/」

プライマリパーティション-> Ext3-> ... 450 GB それ以外はすべて場所です...-> "/ home"。



インストーラー言語の選択や名前とパスワードの選択など、他のすべてのポイントは「宿題」として残されます。 事前に、コンピューター名「 homeserver 」と管理者ログイン「 odmin 」を設定することを書きますが、今後は同じ名前を使用することを想定します。



インストール後のすべて-USBフラッシュドライブを引き出し（10回ロードされないように）、再起動します。

出来上がり！ クリーンなシステムが実行されています。



今、あなたの耳に少し気を失いました。 私はUbuntuのロシア語メニューに慣れていて、システムのロシア語ローカライズを使用したいのですが、「デスクトップ」、「ビデオ」、実際にロシア語のフォルダー名などのフォルダーの名前については猛烈に怒り、「デスクトップ」、「ビデオ」、ドキュメント。」 他の方法でそれを行う方法がわからないので、この問題を取り除くためにどのように行うかを説明します。 システム->管理->ローカリゼーションを踏みます。 彼は、ローカライズパッケージの更新を求めます-私たちは更新しています（繰り返します-これは、PUREインストールをゼロから始めるための段階的なガイドです）。 その後、メニューとログインで言語「English（USA）」を選択します-「OK」ボタンをクリックします。 このプロセスでは、「システムの更新」ウィンドウが背景に対してポップアップする可能性が最も高くなります。段階的に進んでいるので、まだ触れません。 「 CTRL + ALT + F1 」を押します -コンソールに移動します。 ログインして、次のコマンドを入力します。

sudo /etc/init.d/gdm restart



これにより、 Gnome Desktop Manager （ gdm ）も再起動します。 グラフィカルモードでログインします（スローされない場合は、 CTRL + ALT + F7を押します ）。 ロード後、システムはブルジョアローカリゼーションの名前の変更について尋ねるウィンドウを表示します。 「 二度と尋ねない 」というチェックマークを付け、「 名前の更新 」を選択します（これはフォルダーの名前変更です）。 Localizationアイテムを再度踏み（System- > Administration-> Localization ）、メニューに「ロシア語」を返し、ログインします。 その後、 gdmを再起動します。



背景に対して、システムは悲しいことにアップデートを調べようとしています。 今彼女を満足させます。 彼女が物ggingいにうんざりしていてウィンドウが表示されない場合は、 システム->管理->システム更新項目から呼び出します。 [ アップデートの確認]をクリックしてから、 [アップデートのインストール]をクリックします 。 このシステムは喜んでインターネットをうめき、インターネットチャネルのサイズに応じて安全に喫煙できます。 私が持っている4 Mbit / sチャンネルでは、157メガバイトの更新がダウンロードされ、約2〜3分後にパッケージがインストールされ、さらに4分...プラス、マイナス、合計、どこか、たとえば10分です。 平等なもの-トルコ人+タバコで独立して醸造されたコーヒー1杯。 私たちは戻ります-システムが完全に満足し、再起動を要求するという嬉しいメッセージが表示されます。 別の段階を通過しました。



さらに進んでください-そして、まだ「サーバー」があることを認識しておく必要があります。つまり、グラフィック画面に座る価​​値はありませんが、たとえば、別の部屋やラップトップから離れて座る必要があります。 このためにはSSHが必要ですが、 Midnight Commanderを使用しないSSHが必要です。 ターミナルに行きましょう（まったく知らない人のために、 ターミナルはプログラム->アクセサリ->ターミナルのように呼ばれます ）。

＃更新後のクリーニング

sudo apt-get autoremove

＃SSH、MCをインストール

sudo apt-get install ssh mc



SSH構成のいくつかの編集

sudo mcedit / etc / ssh / sshd_config



行「 ポート22 」を「 ポート44000 」に変更し、SSHを再起動します。

sudo /etc/init.d/ssh restart



それだけです 披露すると、ばかげたロボットに対するマイクロプロテクションがあります。



プレインストールが完了しました。 モニターをサーバーから切断し、サーバーをテーブルの下に押し込みます。さらに、サーバーにアクセスできない完全なリモート環境を作成するために、いくつかの書籍をさらに充実させます。



PS：このセクションでは、マザーボード用にNVidia GeForce用のドライバーをインストールしますが、それらのインストールについては後ほど説明します。



5.ホームSAMBA



この時点で、私は複数のタンバリンを壊しました。 私がしなければならなかったのは、一般的なアクセスのためにいくつかのフォルダを共有し、パスワードでアクセスできるいくつかのフォルダ（父と妻の個人用）でした。 これまで、ネットワーク上のすべての人がアクセスを共有する方法を理解しただけでしたが、専門家の助けを借りてこのセクションを補足する予定なので、すぐにパスワードで保護されたプライベートフォルダに関する質問を追加しました。



Windowsの後、ほとんどの場合、ディスクへのパーティション分割ができず、私にとって非常に不便であり、フォルダ内の順序を維持することを好みます。 そのため、共有フォルダー用のこのようなディレクトリ構造を開発しました。



  • /ホーム
  • /ホーム/サンバ
  • / home / samba / public-ごみ箱などの共有フォルダー。
  • / home / samba / private_father-父親のプライベートフォルダ。
  • / home / samba / private_wife-配偶者のプライベートフォルダ。
  • / home / samba / private_my-私用ストレージ;
  • / home / samba / video- ビデオのあるフォルダー。
  • / home / samba / music- 音楽のあるフォルダー。


/ホームフォルダーが巨大であることを考慮すると、すべてがそこに追加されます。 再インストールすると、フォーマットされません。 将来的には、外部ドライブにフォルダーを追加します。



sambaのインストールと構成に戻りましょう。

＃インストール

sudo apt-get install samba samba-tools

＃メインフォルダーを作成します（クリーンシステムを検討していると言いました）

sudo mkdir / home / samba

＃ユーザーを添付します

sudo smbpasswd -a odmin

＃ユーザーをアクティベートします：

＃（すべてのアクションを理解しているわけではなく、それらが機能していることを知っているだけです）

sudo smbpasswd -e odmin

＃ファイル/ etc / samba / smbusersを作成して編集用に開きます

＃（理由もわかりませんが、必要です）

sudo mcedit / etc / samba / smbusers



# /etc/samba/smbusers

#

odmin = Admin







すべての指示は、新しい共有フォルダーを追加および削除するために/etc/samba/smb.confファイルを構成する必要があると言っていますが、タスクには厳格な要件があります-不要なタスク、コマンドなどをできるだけ少なくするため、構成ファイルの編集を拒否しました。 代わりに、私は素晴らしいチームに会いました-「ネット」。 （もちろんすべての構文を知っていれば）コマンドラインからユーザーなどを追加できるので、私にとっては便利です。 何か間違っている場合は、繰り返し、修正してください。一緒にこのマニュアルを補足します。 それまでの間、私が知っているように続けましょう。

＃共有するフォルダーを作成する

sudo mkdir / home / samba /パブリック

＃このフォルダーをsambaに登録します。ここで、

＃ 構文：net usershare share_name / path / to / share [acl] [<guest_ok = [y | n]>]

#Publicは、Windowsのフォルダーが表示される名前です。

＃ / home / samba / publicはフォルダーへのパスです。

＃ 全員：Fは少し怖いACLです。その目的もわかりませんが、

＃しかし、私が理解している限り、それはこの方向にあるということです。

＃特定のユーザーにフォルダーへのアクセスを許可するため。

＃guest_ok = y-これは、すべての「ゲスト」へのアクセスを提供しています。

sudo net usershare add Public / home / samba / public Everyone：F guest_ok = y


同様に、VideoフォルダとMusicフォルダを追加します。

すべての変更を適用するために、sambaコマンドを再起動します。

sudo /etc/init.d/samba restart


これで、sambaについての私の知識はすべて終わりましたが、これはフォルダーの内部共有と掘る方向については十分です。



PS：実際の達人がこれらの決定にセキュリティの問題や他の問題を見つけた場合は謝罪します。 私は管理者ではありません-Web開発者です。テストにはホームシステムが必要です。 共有フォルダーを自動的に構成するスクリプトを作成するためのヘルプを求めています。 誰かが何かを作りたいと思っているなら、私は彼から得たいものを捨てることができます。 または、理由の範囲内でそれを支払うことができますが、私はできます。 いずれにせよ、個人のICQまたはSkype番号に連絡方法を書いてください。この問題は解決できると思います。 当然、スクリプトとあなたの名前はすぐに記事に添付され、再印刷されます。



6. Apache、MySQL、PHP、phpMyAdminをインストールします



さて、ここではすべてが非常に簡単です。

フォルダーの場所に関するすべての便利な要件から、再び選択したディレクトリ構造。



  • /ホーム
  • / home / sites -Web開発に関連するすべてが保存されているフォルダー。
  • / home / sites / www- デフォルトのサイトフォルダー。



    コンソールとゴー：

    ＃サイト用のフォルダーを作成する

    sudo mkdir / home /サイト

    sudo mkdir /ホーム/サイト/ www

    ＃odminアカウントからのアクセスの便宜のためにアクセス権を変更します

    sudo chown -R odmin：www-data / home / sites / www

    ＃MySQLをインストールする

    ＃ダウンロードの最後に、MySQLはルートアクセスパスワードの入力を求めます

    ＃MySQLに、ここに入力します。なぜ後で変更するのですか？

    sudo apt-get install mysql-server

    ＃Apache 2.2.xxおよびPHP 5.2.xをインストールします

    ＃システム自体が公式リポジトリの最新バージョンを選択し、

    ＃他の人はこれまで気にしませんでした

    sudo apt-get install apache2 php5 php5-mysql

    ＃そしてPHPのパッケージングモジュール

    sudo apt-get install php5-curl php5-adodb php5-gd php5-ffmpeg php5-imagick php5-memcache php5-xmlrpc php5-xsl

    ＃phpMyAdminをインストール

    sudo apt-get install phpmyadmin

    ＃個人的には、作業時にImage Magickも使用するため、

    sudo apt-get install imagemagick


    Apacheの再起動時にエラーが発生しないように、/ etc / apache2 / httpd.confファイルを更新します。

    sudo mcedit /etc/apache2/httpd.conf


    # /etc/apache2/httpd.conf

    #

    ServerName localhost





    ファイル/ etc / apache2 / sites-available / defaultを編集してください：

    # /etc/apache2/sites-available/default

    #

    <VirtualHost *:80>

    ServerAdmin webmaster@localhost



    DocumentRoot /home/sites/www

    <Directory />

    Options FollowSymLinks

    AllowOverride All

    </Directory>

    <Directory /home/sites/www/>

    Options -Indexes FollowSymLinks

    ErrorDocument 404 /404.php

    AddDefaultCharset UTF-8

    AllowOverride All

    Order allow,deny

    allow from all

    </Directory>



    ScriptAlias /cgi-bin/ /usr/lib/cgi-bin/

    <Directory "/usr/lib/cgi-bin">

    AllowOverride None

    Options +ExecCGI -MultiViews +SymLinksIfOwnerMatch

    Order allow,deny

    Allow from all

    </Directory>



    ErrorLog /var/log/apache2/error.log



    # Possible values include: debug, info, notice, warn, error, crit,

    # alert, emerg.

    LogLevel warn



    CustomLog /var/log/apache2/access.log combined



    Alias /doc/ "/usr/share/doc/"

    <Directory "/usr/share/doc/">

    Options Indexes MultiViews FollowSymLinks

    AllowOverride None

    Order deny,allow

    Deny from all

    Allow from 127.0.0.0/255.0.0.0 ::1/128

    </Directory>



    </VirtualHost>





    そしてもちろん、Apacheを再起動します。

    sudo /etc/init.d/apache2 restart
