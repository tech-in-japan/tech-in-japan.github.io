鈍い船長またはTor Projectがブラウザのフィンガープリントと戦う方法

特定のサークルで知られるHovav ShacamとKeaton Moweryに よる 2012年の記事では、当時の新しい方法であるJavaScriptと Canvas Fingerprinting から直接ブラウザとシステム識別子を生成する方法について説明しました。 機器を含む区別できる方法...











記事



メソッドの本質は、異なるブラウザーの異なるシステムでは、テキスト（テキストだけでなく）が異なるようにレンダリングされることです。これは、異なるレベルの多くの異なるコンポーネントがこれを担当するためです。



  1. フォントは角質である場合があります（異なる文字セット、わずかに異なるグリフ、異なる合字およびカーニングを使用）。
  2. 異なるブラウザーでライブラリー関数を呼び出すための異なるパラメーター。
  3. libfreetypeおよびその他のレンダリングライブラリの異なるバージョン。
  4. OSの異なる実装と異なるOS設定（たとえば、 ClearTypeの異なるバージョンと異なる画面解像度）。
  5. 異なるグラフィックドライバー。
  6. その他のグラフィックハードウェア。


この記事では、ピクセル単位で画像を返すgetImageData APIを使用するメソッドについて説明します。 WebGLの使用も考慮されます。異なるシステムでは、3Dシーンは異なる方法でレンダリングされます。



この記事では、PCにインストールされているOSコンポーネントやその他のソフトウェア、ノイズ、その他の保護技術を使用せずに、純粋にソフトウェアレンダリングを使用することを推奨しています。 Tor Projectでは、プリントを作成する可能性が修正され、getImageDataの許可を求め、ランダムフォントを代替フォントに置き換えました（したがって、10回測定します）。



設計上の欠陥



情報が最新の他のAPIがあります。 たとえば、テキストディメンションAPI。 テキストのレンダリングが異なる場合、サイズはわずかに異なるはずです。 同じキャンバスのmeasureText APIとDOMのgetBoundingClientRect APIを使用して、この仮説をテストします。



リポジトリの クローンを作成し、コンピュータで最後の（または最後ではない） TBBを実行し、リポジトリ内のHTMLファイルを開き、コメントに結果を投稿するか、PRを実行します。 フィドルまたはフルスクリーンフィドルに移動することもできます 。 原則として、ハッシュを行わないことも可能でした。これにより、データマイニングの余地が大きくなりますが、圧縮形式であっても、各フォントに関する完全な情報セットは多くのスペースを占有します。 したがって、ハッシュを取得しましょう。 さらに深く掘り下げたい場合は、ソースの複数の行のコメントを解除できます。

ランダム化のトピックに関する若干のオフトピック



この記事では、追跡を防ぐためにフィンガープリント可能な値をランダム化する方法を提案していますが、そのようなランダム化がTBBでどれほど成功したかはすでに見ました。 各サイトのランダム化は、常に回避される可能性があるため、役に立たないと思います。 記事で提案されているように、ランダム値キャッシュが使用されている場合（ランダム値が保存され、指紋が再起動されると、ランダム化された値はこのサイトで同じになります）

  • たとえば、別のドメインからeフレームを実装することで回避できますが、同じサービスに属します。 キャンバスの場合、同じ絵を別の絵に描くことができます。ノイズは異なり、分離できます。
  • 識別子として使用できます。
  • DoS攻撃に使用できます。 特に、この記事では、キャンバスにノイズを追加することを提案しています。 このノイズがキャッシュされている場合


すべてのサイトに共通のキャッシュを使用し、新しいセッションでのみ変更することは1つのオプションです。 これにより、ペダル上のセッションを追跡できますが、セッション間で残る識別子を作成することはできません。





結論



  • メソッドの変更では、ハードウェアに関する情報を抽出することはできませんが、一部のフォントにはソフトウェアが付属しているため、OS、ブラウザー、およびフォントセット（およびそれらのソフトウェア）を区別できます。たとえば、Calibri-Microsoft Officeソフトウェアパッケージなど）異なる環境で実行されているTBBおよび同じ環境で実行されているTBBの異なるバージョン。

  • 開発者が実行したい手順の1つは、TBBに付属のフォントを使用することですが、テキストはシステムごとに異なってレンダリングされ、ブラウザーのバージョンによって異なるため、これはおそらく役に立ちません。

  • JavaScriptを無効にする必要があるか、少なくともconfig：

    TBBまたはTAILSのどちらでもデフォルトでどちらも行われなかったため、特にTorプロジェクトの資金源と 代表者の推奨事項を考慮して、特定の考えに至りました。 
     curl https://ooni.torproject.org/install.sh | bash
    （これを行わないでください。悪意のあるスクリプトがこのアドレスに表示される可能性がありますが、HTTPSは忘れてください。CAは新しい証明書を発行する場合があります）。 HTTP公開キーピニングは部分的に役立ちますが（CAを変更することに対して、既存のCAが証明書を再発行することを妨げるものは何もありません）、執筆時点では有効になっていませんでした。



    テールは依然として許されます-指紋は機器に依存せず、システムは誰でも同じです（もちろん、同じバージョンでSSESがありません ）。



    機能の無効化は検出できるため、一度にすべて無効にする必要があります。 Tor Projectバグトラッカーにすべてのアイデアを投稿してください。 登録は必要ありません。サイファーパンクログインとライトコードパスワードを持つ公式のマルチユーザーアカウントがあります。

  • JSを無効にすると、 CSS （ JSコードを1行も使わずにシンプルなゲームを実装できる要素の表示を調整するための高度なテクノロジーもあります！ ）があるため、役に立ちません。コンピューターに関する情報を抽出することもできますが、 CSSを介して利用できる情報はJSを使用するよりも少ないため、収集してサーバーに送信することはより困難です。

    CSSを閉じても、他の抜け穴があります。

    しかし、これは決してオフにする必要がないことを意味します。

  • 匿名ブラウザ標準の緊急の必要性があります。

  • JSを無効にすることは、主に金makingけを目的とした今日のWebと互換性がありません。 ビジネスはビジネスです。利益を得るために（または会社のリーダーシップを変えて森林を伐採しないために）、特別なサービスに屈して、オープンまたは秘密のレジストリに入り、ユーザーを追跡し、ユーザーの行動を分析し、マルウェアを配布する必要がある場合、これが行われます

    一部のサイトはJSなしで意図的に非機能的にされていますが、一部のサイト（たわごとや石鹸など）では、JSなしではサイトを使用できないことが明確に記述されています（ただし、場合によってはバイパスできます）。

    結論：サイト所有者からのJSなしで作業の実装を期待する必要はありません-彼らの利益ではありません。
