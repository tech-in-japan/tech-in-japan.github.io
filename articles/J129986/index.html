Habra-informer v.0.1

ごく最近、私はHabra-inforormの作業を開始しました。これは、サイトの更新を便利に監視し、再び気が散らないようにするためです。

いくつかの作業が実施され、その中で実施されました。

-3つのスレッド（新しい記事、サンドボックス、QAの監視）。

-いくつかのエラーを修正しました。

-これで、スクリプトはすぐにタイトル、コメント数、トピックを残した人を表示します。



ダウンロードする


Windows用のアプリケーションをダウンロードします （依存ライブラリはアーカイブに含まれています）

Linux用アプリケーションのダウンロード （コンパイル済みバイナリ）

MAC OS X用アプリケーションのダウンロード -（ AcidSlayerによりコンパイル ）

OS X 10.6+向けビルド -（ Amka によるコンパイル ）

ARCH LINUX- （ Alexandr Baevが主催）または#yaourt -S habratray

ソースをダウンロード









作業は同じ方法で行われました。AcidSlayerのデザインのおかげで、データベース（sqlite）に基づいて作業を開始し、QMLを使用してフォーマットしました。 しかし、いくつかの理由により（結果は、たとえば宣言型ライブラリに応じてユーザーに不便をもたらしました）、この中間バージョンではこのソリューションを使用する必要はありませんでしたが、作業は停止されず、この方向で実行されています。



将来的には

-QMLでの実装

-また、ユーザーはインフォーマーからサイトへのブロードキャスト（コメントの作成、新しい記事の作成など）を実装するように求められましたが、この機能は計画に残っています。
