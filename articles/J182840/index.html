オフサイトバックアップ転送：Veeam Backup＆Replication v7のWANアクセラレーター



数か月後にリリースされるVeeam Backup＆Replication v7の新しいバージョンには、WAN経由でバックアップを転送するために特別に最適化されたWANアクセラレータが組み込まれています。 オフィスの外でファイルをバックアップすると、データストレージシステムの信頼性が大幅に向上しますが、同時にWANチャネルの負荷が大幅に増加します。これは主にバックアップ時間の大幅な増加に反映されます。 WANアクセラレータを使用すると、次の作業アルゴリズムを使用するため、低速接続でもより効率的なデータ転送が可能になります。

  1. 変更されたデータのみを転送します（すべてのデータはブロックに分割され、変更されたブロックのみがWANを介して送信されます）
  2. データブロックのグローバルキャッシュ（つまり、使用可能なすべての仮想マシンをカバーするキャッシュ）により、ブロック自体からのデータの代わりに、以前に送信されたブロックのハッシュを転送できます。
  3. 可変ブロック長によるネットワークトラフィックの圧縮とデータ重複排除
  4. WANネットワークの長い応答時間特性を持つ接続のデータ転送速度と他のTCP / IP最適化を向上させるマルチスレッド並列TCP / IP接続
  5. 切断された接続の自動再開




Backup＆Replication v7に組み込まれているWANアクセラレーターは、コンテンツの転送用に特別に最適化されたアクセラレーターであり、バックアップコピーです。 彼は常に、以前に転送されたデータの完全なセット（WANの両側のバックアップリポジトリ）を自由に使用できます。その結果、送信されたコンテンツの構造とコンテンツを「トレーニング」する時間は必要ありません（汎用オプティマイザーにはWANが必要です）。 これは、Veeam WANアクセラレータが汎用WANアクセラレータよりも効率的なバックアップ転送ツールであることを示唆しています。



Veeam WANアクセラレータを使用するには、WANアクセラレータの役割を設定する必要があります（プロキシおよびリポジトリの役割と同様）。 WANアクセラレータは、既存のプロキシまたはバックアップリポジトリにインストールできます（インストールはWANの両側で行う必要があります）。 Veeamの推定によると、組み込みWANアクセラレータを使用したデータバックアップの速度は、WANを介した同じデータの通常のコピーよりも最大50倍高速です。



汎用WANオプティマイザーを使用する場合、Veeam Backupトラフィック最適化を除外して、組み込みWANアクセラレーターを最大限に活用できます。
