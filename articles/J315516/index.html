Citrix NetScaler ADC周辺。 パート1

Citrix NetScaler ADCが何であるかを知らない企業ネットワークの専門家を見つけるのは困難です。 NetScaler ADCは数千の企業ネットワークに展開されています。つまり、このソリューションは他のベンダーのネットワークソリューションと常に対話しています。 将来の記事では、シスコおよびマイクロソフトとの統合を検討します。





NetScaler ADCは、最大エディションで3つの重要なネットワーク機能を提供する多機能ソリューションです。ネットワークアプリケーションの負荷分散、アプリケーションとリソースのセキュリティ、企業の企業ネットワークへのリモートアクセスの組織です。 バランサーの全機能の一部のみを必要とするお客様には、お客様が必要とする機能のみを提供する個別の製品があります-ロードバランサー（ADC）としてのNetScaler、Webアプリケーションのトラフィックを保護するNetScaler Web Application Firewall、Webアプリケーション自体およびネットワークリソース、およびNetScaler仮想プライベートネットワークを整理するためのゲートウェイ。 異なる製品オプション、NetScaler MPX、NetScaler VPX、NetScaler SDX、およびNetScaler CPXは、異なるレベルのサービスを提供するように設計されていますが、エンタープライズレベルのNetScalerプラットフォームと同じ機能、同じREST APIコードベースと同じNetScaler ADC操作インターフェースを備えていますが、ズームレベル。 これらすべてにより、NetScaler ADCプラットフォームは、他社のソリューションとの統合に非常に便利です。



CitrixおよびCisco Software Defined Networks



CitrixとCiscoのパートナーシップは古くからあり、2014年にシスコはバランサーの販売完了を発表し、顧客にCitrix NetScaler ADCを推奨しました。 Citrixはこのパートナーシップをどのようにサポートしていますか？



現在、多くの顧客は、ネットワーク制御層がデータ伝送デバイスから分離され、ソフトウェアで実装されるソフトウェア定義ネットワーク（SDN）としてデータ伝送ネットワークを構築するためのこのようなアプローチを検討しています。 簡単に言えば、SDNは単一のコントローラーから制御されるデバイスのセット（多かれ少なかれ「スマート」）です。 この概念のアプリケーションは最前線にあり、ネットワークの構成に関係なく、アプリケーションが機能するための最良の条件を提供する必要があります。



多くのベンダーがSDNの分野で活躍しており、自社のSDNネットワークを開発しているベンダーもあれば、さまざまなメーカーと統合するソリューションを提供しているベンダーもあります。 Citrixは独自のSDNソリューションをリリースしていませんが、さまざまなベンダーのソフトウェア定義ネットワークで使用できる製品を顧客に提供しようとしています-他のメーカーと協力できるのはなぜですか？



たとえば、Netscaler ADCはOpenStackのNeutron SDNコンポーネントと統合し、単一のコントローラーで管理し、サービスとして機能します。リソースを自動的に割り当て、構成し、運用することができます。 他のサプライヤーとの統合が存在します。 シスコのSDNについて話す場合、主なコンポーネントはCisco ACIとの統合です。



Cisco ACIは、特定のモデルの機器、つまりCisco 9000スイッチのラインで動作するテクノロジーです。Cisco9000スイッチは、個別に動作することも、単一のCisco APICコントローラの制御下で動作することもできます。 Netscaler ADCは、このスイッチのネットワークに統合されており、各スイッチはそれぞれに接続されています。 そして、コントローラ自体がすべてのネットワークデバイスに指示します。すべてのネットワークポイントを最適に相互接続するために、何を構成する必要があります。 ネットワークの従来のビューを再作成するためにネットワークを再構築する必要はなく、ソフトウェア定義のネットワークの概念は最大限に機能します。 Citrix Netscaler ADCは、Cisco APICコントローラーと統合され、制御されます。 これを行うには、コントローラーに特別なドライバーをインストールします。その後、コントローラーはCisco APIコマンドをNetscalerが理解できるAPIコマンドに解釈できます。



Netscaler ADC – Cisco ACIバンドルは、実質的にあらゆる帯域幅のスケーラビリティを提供します。 NetScalerのスケーラビリティは、ハードウェアデバイスと仮想デバイスの両方に提供されます。 同じシリーズ内のハードウェアデバイスの場合、ハードウェアを交換することなく、「若い」モデルから「古い」モデルに切り替えることができます。 古いモデルの帯域幅が十分でない場合、複数のCitrix Netscaler ADCコンプレックスをクラスター化して、帯域幅を拡張できます。 このようなクラスターでは、最大32個のノードが可能です。 現在最高の帯域幅を備えたCitrixモデルでは、最大160Gb / sのストリームを処理できます。これに32倍をかけます。

複数のデータセンターを持つ顧客は、Netscaler ADCの機能のおかげで、ユーザーに単一のエントリポイントを提供できます。 Netscalerは、いわゆるGlobal Server Load Balancing（GSLB）のテクノロジーをサポートします。これは、エンドユーザーからの遠隔性など、任意の場所にあるサーバーファームのバランシング問題を解決できます。 このシステムは、いくつかの異なる負荷分散アルゴリズムをサポートし、世界中に散在する最適な顧客サービスを提供できます。 管理者にとって、システムは柔軟なリソース管理ポリシーの形成を可能にします。



このテクノロジーはDNSに基づいており、Netscaler ADCは特定のゾーンのDNSサーバーとして機能しますが、静的（ドメイン名からIPアドレスに応じて）レコードを使用せず、動的なレコードを使用します（主な役割はドメイン名によって行われ、IPアドレスはバランサーによって決定されます）。 したがって、このソリューションでは、エンドユーザーの単一のエントリポイントを指定でき、GSLBテクノロジを使用するNetscaler ADCは、ユーザーリクエストを処理するデータセンターを決定します。 このテクノロジーにより、たとえば、データセンターへの近接度、チャネルの負荷、サーバーの負荷に応じて、データセンターをメインまたはバックアップとして指定するなど、いくつかのルールを規定できます。



上記から、CitrixとCiscoのパートナーシップは相互の推奨事項に限定されないことは明らかです。これは、機器相互作用プロトコルの共同開発および実装との技術的パートナーシップです。 最初に、最初の独自のCisco vPathプロトコルがCitrix Netscaler ADCに登場し、次にRISEプロトコルが登場しました。これにより、CitrixはCisco機器と連携する機能も提供します。 最後に、Netscaler ADCは単なるメンバーではなく、Cisco ACIエコシステムのメンバーになりました。 シスコのコントローラーAPIを介して構成できる機能の量（およびこれらは数十の機能）によって、Citrix Netscaler ADCはリーダーであり、今後も長い間使用されます。
