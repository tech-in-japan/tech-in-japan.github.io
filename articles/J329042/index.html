GPLがどのように緩和されたか

オープンソースライセンスが登場して以来、その耐久性をテストする誘惑がありました。 GNU Public Licenseはそれらの中で最も妥協のないものです;それはユーザーとプログラマーの自由を、他人の仕事の結果の不正流用、オープンで公開された製品の代替、クローズドおよびペイドから保護します。 驚くことではありませんが、中小企業、さらには大企業にさえ違反することが最も多いのはGPLです。

















GPLの実施は、オープンコミュニティにとって戦略的に重要な問題であり、これがなぜなのかを簡単に理解できます。 GPLは、コミュニティの要であり、コミュニティの最も重要な支持者です。 これらのうち、最も有名なのはGNUの創設者でありGPLの作者であるリチャード・ストールマンです。 コミュニティがライセンス違反者とどのように戦っているか、GPLに違反しており、GPLに違反し続けていること、そしてオープンソースの収用者がどのように収用しているのかを調べましょう。







GPLとコピーレフト



GPLは、 Software Freedom Conservancy Bradley M. Kuhn hawksの 1人によれば、Free Software Worldの憲法です。 4つの基本的な規定、ハッカーの4つの自由度を定義しています。







  • 自由にプログラムを実行する自由（自由度0）;
  • プログラムの動作を理解するためにプログラムとそのソースコードを研究する自由（自由度1）。
  • 隣人を助けるためにプログラムを配布する自由（自由度2）。
  • コミュニティがあなたのベストプラクティスを使用できるように、プログラムの変更や改善を広める自由（自由度3）。


ライセンスの最初のバージョンは1989年に登場しました。GPLv1は次の問題を解決しました。







  1. ソフトウェア製品の製造元はコードを提供せず、バイナリのみを提供します。 これを防ぐために、プログラムの製造元の要件の1つは、人間が読み取れるソースコードを配布することです。
  2. ソフトウェア製品のメーカーは、追加の制限を課す場合があります。 これを防ぐために、ライセンステキストには、コードへのすべての変更もパブリックドメインに含める必要があると記載されています。


ライセンスの2番目のバージョンは、1991年の最初のバージョンの直後にリリースされました。GPLv2は、 自由または死をモットーにリリースされました。 新しいライセンスの重要なポイントは、いくつかの制限により、プログラムの製造元がユーザーの自由の権利と原則を適切に提供できない場合に、ソフトウェアの配布を禁止する規定でした。 したがって、不un慎なビジネスマンは、フリーソフトウェアの製造業者から特許控除を剥奪しようとはしません。

理論と実践におけるGPLの仕組み



GPLの理論的根拠は次のとおりです。







  • 著作権-国際的に運営され、彼の作品に対する創作者の権利を定義する民法の機関。
  • 著作権ライセンスを使用して、使用と配布の許可を与えます。
  • 前者の条件は、ユーザーに4つの自由度を提供することです。
  • そのため、著作権はコピーレフトに変わります。
