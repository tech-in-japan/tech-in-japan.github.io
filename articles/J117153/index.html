Aspia 0.2.1

Aspiaの新しいバージョンをご紹介します。 Aspiaは、Windowsのシステム情報を収集および表示するためのプログラムです。 このバージョンの準備には多くの作業が行われ、多くの要望が考慮され、新しい機能が追加され、情報の量と質が改善され、多くのエラーが修正されました。



このリリースの新機能













新しいカテゴリ



  • SPD-コンピューターのRAMに関するデータを受信します。 現在のバージョンは、限られた数のチップセットをサポートしているため、どこでも動作しません。 サポート：Intel ICH7 / ICH8 / ICH9、SIS 962/963/968、VIA 8235/8237 / 8237A / 8251、ATI SB400 / SB600、Nvidia nForce CK804。
  • Windowsデバイス-インストールされているすべてのデバイスと、個別に不明なデバイスを表示する機能。 不明なデバイスを識別するための小さなベースは、プログラムのアーカイブにあります。
  • センサーの読み取り値-ハードドライブとプロセッサの温度に関する情報。 IntelおよびAMDプロセッサのみがサポートされており、すべてのデータが完全に正しいわけではありません。 Intelプロセッサの場合、問題はさまざまなプロセッサの異なる熱保護トリップ温度（Tjmax）にあります。これは、Core i7プロセッサ上のソフトウェアによってのみ決定できます。 AMDプロセッサーでは、別の種類の問題-テスト用のマシンがありません。
  • ライセンス-インストールされた製品のライセンスキーを表示します。 現時点では、Microsoft製品のライセンスのみが決定され、すべてではありません。 徐々に、このセクションは完成します。
  • ファイルを開く-ネットワーク上で開かれたファイルを表示し、それらを開いたユーザーの名前を表示できます。
  • アクティブセッション-コンピューターにログオンしたユーザーのリスト。


新機能



  • 設定ダイアログ-パラメーターを変更して保存する機能。
  • プログラムをさまざまな言語に翻訳する完全な機会。プログラムを再起動せずに言語を切り替えます。
  • プログラムのウクライナ語への翻訳を追加しました。
  • 独自のアイコンテーマを作成し、それらを切り替える機能。
  • センサーの値を表示するセンサーのシステムトレイアイコンに表示する機能（現在はプロセッサーとハードドライブの温度のみ）。
  • キャンセルするオプションを使用してレポートステータスを保存するためのダイアログ。
  • 選択した複数の行をコピーする機能。 行を選択するには、Ctrl + A（すべて選択）を使用するか、Ctrlを押して必要な行をクリックします。
  • Windowsのインストールの日付を表示します。


変更者



  • レポート保存ダイアログの使用がより便利になりました。
  • 「プログラムについて」ダイアログ。
  • データを受信すると、「データを読み込んでいます...」というメッセージは表示されなくなりました。 代わりに、データは受信時に追加されます。

あとがき



このプロジェクトでは、インターフェイスを他の言語に翻訳する必要があります。 英語、ロシア語、ウクライナ語以外の言語を知っていて、翻訳を手伝いたい場合は書きます。

また、言語を切り替える機能を備えた英語版のサイトを作成する必要があります。 エンジンがJoomla 1.6を使用したため。 あなたがこれを行うことができ、あなたがそれをしたいという願望を持っているならば、私はそのような助けを受け入れることを非常にうれしく思います。

また、プログラムの現在のバージョンはリリースではなく、エラーや一部の機能の誤った操作が含まれている可能性があることを思い出させてください。
