この投稿は、独自のオーディオプラグインの作成に関するMartin Finkeのガイドの一連の翻訳の最初です。

この資料の特徴は、プラグイン形式とその使用プラットフォームに依存しないことです。 オーディオプラグインの全体的な構造に注目しています。 次に、アルゴリズムは、 VST 、 VST3 、 AU 、 RTAS 、 AAXの形式で、または別のアプリケーションでアセンブリするための抽象化レイヤーにラップされます。

Martin自身はMac上でより多くの作業を行っていますが、このマニュアルにはWindowsでの開発に必要なすべての手順も含まれています。



はじめに



オーディオプラグインは、ホストにダウンロードされるプログラムです（ Ableton Live 、 Logic、 REAPERなど ）。 オーディオおよび/またはMIDIデータを処理します。 通常、GUI（グラフィカルユーザーインターフェイス）があります。 3つの例（ U-He Zebra 、 Sonalksis FreeG、およびD16 Decimort ）は次のとおりです。







ご覧のとおり、GUIにはコントロールがあり（ほとんどの場合これらはペンです）、プラグインが受信データを処理する方法を変更します。 プリセットのセット（プリセット-スクリーンショットではComboおよびEmulatorと呼ばれます）もあり、ノブやその他のパラメーターの位置が保存されます。



歪みから始めます-これはかなりシンプルなプラグインです。 その後、次のような減算方式のシンセサイザーを作成します。







C ++とWDL-OLライブラリを使用します。 これは、 Cockos WDLライブラリー（ホイットルと発音）に基づいています。 彼女は私たちのために多くの仕事をしています。最も重要な点は次のとおりです。



  • 既製のXcode / Visual Studioプロジェクトが含まれています
  • コードからVST、AudioUnit、VST3、RTAS形式のプラグインを作成します。 形式を選択して、実行をクリックするだけです！
  • 32 \ 64ビットバージョンを作成します
  • プラグインをWindowsまたはMacOS上のスタンドアロンアプリケーションとして起動します
  • オーディオプラグインで使用されるさまざまなGUI要素が含まれています




このライブラリには、オーバーサンプリングなど、一般的に使用されるアルゴリズムもいくつかあります。 このフォーラムスレッドには、WDLを使用して記述されたプラグインのスクリーンショットが多数あります。



異なるプラグイン形式は、ほぼ同じことを行います。 したがって、通常、コードには多くのコピー＆ペーストがあります。 プログラマーとして、自分自身を繰り返さないようにする必要があります。そのため遅かれ早かれ、異なるフォーマットの上に独自の抽象化レイヤーを書くことになります。 ただし、WDLの一部であるIPlugフレームワークとして既に存在します。 これは開発プロセスの最も面倒な部分なので、次のような興味深いことに集中できます。



  • オーディオおよびMIDI処理
  • プラグインの外観
  • ホストとの統合（自動化、プリセットなど）




WDLのもう1つの素晴らしい点は、ライセンスライセンスです。 ライブラリは商用製品でも自由に使用できます（上記のリンクの詳細情報を参照）。



どうやってやるの



「追跡は略奪品よりも優れています」-H. P.バクスター




プログラミングでは、結果自体はすでに報酬ですが、学習する結果への道に沿っています。 このガイドには、完璧なソリューションを含む長いリストはありません。 何が機能するかから始め、徐々にそれを発展させ、さまざまな問題に対処します。 欠点は、後で最適なコードに置き換えるコードを書くことです。 しかし、プログラマーはそれに慣れることができません。 最初のプラス：すべてを一挙に行うのではなく、徐々にソリューションにアプローチするため、圧倒的な喜びはありません。 2番目は、プログラミングに関する貴重な知識を得るということです。



この一連の投稿は、一枚の紙に電卓がなければmatanの問題を解決することについてではありません。 プロデューサーの友人に低音をより豊かにする良いソフトウェアを作成する方法についてです。



何が必要ですか



  • Xcode 4以降を搭載したMac、またはVisual Studio C ++ 2010以降を搭載したWindows
  • C ++のある程度の理解（構文、ポインター、OOPの基本、メモリー機構）。 C ++は重い言語であると聞いたことがあるかもしれませんが、あまり複雑なテクニックは使用しません。
  • [デジタル]サウンドの一般的な知識：振幅と周波数とは何か、フィルターは何をするか、特定のサンプルレートとビット深度でオーディオがどのように表示されるか。 あなたが以前に音楽を作ったことがあるなら、おそらくあなたはすでにこのすべてを知っています。
  • デジタル信号処理とGUI開発への関心：これらは、このガイドの2つの主要なトピックです。 デジタル信号処理の知識がない場合は、開発中にギャップを埋めるために追加の資料を読む必要があります。 追加の読み物へのリンクを提供しようとします。




有用なリソース



役立つリンクがいくつかあります。



  • スタックオーバーフロー ：ここでは、たとえば、コンパイラから受け取る不可解な警告の意味を確認できます。
  • DSPガイド ：非常に優れた無料の本（英語）で、必要以上の資料を網羅しています。 信号処理の概念を理解していない場合は、このトピックを読んでください。




さあ、すべてをセットアップして準備しましょう。



WDL-OLをインストールして構成する



次に、WDL-OLライブラリとその依存関係をインストールします。 次に、すべてを正常に機能させるために、例を実行します。



この資料のほとんどは、オリラーキンのビデオにあります。











次のセクションの資料は、主にMacでのカスタマイズ用です。 Windowsは少し低くなります。



Xcode 4を使用したMac OS Xでのセットアップ



始める前に、いくつかの新しいものをダウンロードしてインストールする必要があります。 これは少し面倒ですが、一度だけ行う必要があります。



事前準備



  1. Xcodeがインストールされていない場合は 、ダウンロードしてインストールします。
  2. VST3 SDKの最新バージョンをダウンロードします 。 ダウンロードするには、アカウントを登録する必要があります。
  3. Xcode用のオーディオツールをダウンロードします 。 Xcodeメニューで、[ 開発者ツールを開く ] → [ その他の開発者ツール... ]をクリックします（またはここをクリックしてください ）。 リストでXcode用のオーディオツールを見つけ、ダウンロードしてインストールします。 OS Xごとに異なるバージョンがあるため、慎重に選択してください。
  4. RtAudioをダウンロードします （ASIO SDKから必要なファイルがパッケージに含まれています）。
  5. gitがない場合は、 インストールする必要があります。

mkdir ~/plugin-development

cd !$

public.sdkフォルダーは手動で作成する必要があることに注意してください。 最終的に、フォルダツリーは次のようになります。

Mac OS X 10.5を使用します。 プロデューサー、特にスタジオの所有者は、OSを更新するのが非常に面倒です。 新しいチップは必要ありませんが、ソフトウェアの更新に関する問題のために数日間使用しないと、大きな経済的損失を招く可能性があります。

実際、 展開ターゲットにはユーザーが必要とするバージョンに関する情報が含まれていますが、メインSDKに同じバージョンを登録することを好みます。

SDK 10.5はXcode 4には含まれていません。すでにインストールされている場合は、「テストプロジェクトの実行」項目に進みます。 そうでない場合は、以下をお読みください。



SDK 10.5をインストールします



それをインストールするには、 Xcode 3.2.6をダウンロードする必要があります。 ダウンロードしたら、 .dmgファイルをマウントし、Terminal.appを開いて入力します



open /Volumes/Xcode\ and\ iOS\ SDK/Packages/







検索ウィンドウが開き、 MacOSX10.5.pkgという名前のファイルがあります。 2回クリックします。 OS Xバージョン10.8以降を使用している場合、不明な開発者のプログラムをインストールできないというメッセージが表示されます。 この場合、ファイルを右クリックし、メニューで[ 開く ]をクリックして、ソフトウェアを本当にインストールすることを確認する必要があります。 インストール時に、 インストール場所の変更を選択し、 インストールするディスクをクリックして、その上のディレクトリを選択できるようにします。 たとえば、ユーザーフォルダーのダウンロードフォルダーに設定します。 インストールが完了すると、選択した場所に新しいMacOSX10.5.sdkフォルダーが表示されます。 Terminal.appに次を入力します。



open /Applications/Xcode.app/Contents/Developer/Platforms/MacOSX.platform/Developer/SDKs







検索エンジンウィンドウが再表示されます。 MacOSX10.5.sdkフォルダーをそこにドラッグアンドドロップします。 Xcodeを再起動すると、Xcode 4で古いSDKが使用可能になります。MacOSX10.5.sdkをどこかにバックアップしてください。Xcodeを新しいバージョンにアップグレードする場合は、これらの手順を繰り返す必要があります。



プロジェクトの立ち上げをテストする



ターミナルで、例のあるフォルダーに移動します。



cd IPlugExamples







dublicateスクリプトを実行して、最初のプラグインを作成します（YourNameを独自のものに置き換えます）。



./duplicate.py IPlugEffect/ MyFirstPlugin YourName







これで、新しいMyFirstPluginフォルダーができました。 それに行き、 MyFirstPlugin.xcodeprojを開きます 。 APPターゲットを選択：







WDL-OLには小さなバグがあります。別のオーディオ入力を選択しない限り、スタンドアロンアプリケーションはサウンドを出力しません。 しかし、これは簡単に修正できます。 Xcodeプロジェクトナビゲータで、「 その他のソース」→「OSX App Wrapper」→「app_main.h」を開きます。 34行目（ #define DEFAULT_INPUT_DEV



）の内容を次のように変更します。



#define DEFAULT_INPUT_DEV "Built-in Microphone"







実行をクリックします 。 いくつかのリンカエラーが表示されます。







簡単に修正できます。 プロジェクトのビルド設定に戻り（他のターゲットではなくプロジェクトを選択していることを確認してください）、 Implicitly Link Objective-C Runtime Supportパラメーターの値をNoに変更します。







もう一度実行をクリックします。 数秒後、プラグインはスタンドアロンアプリケーションとして表示されます。







ロジックを使用してAudioUnitのバージョンを確認する



Logicがない場合は、無料のauval



ツールを使用して検証できます。 AUターゲットのビルドが成功したことを確認し、ターミナルを開いて次を入力します。



auval -a 2> /dev/null | grep MyFirstPlugin







次のようなものが生成されます。



aumu Abcd Efgh - YourName: MyFirstPlugin







LogicまたはauvalがAUを見つけられない場合（およびREAPERのような他のホストがそれを見る）、ビルド設定に移動し、 AUターゲットが左上で強調表示されていることを確認します。 「アクティブアーチ」を検索します。 結果には、 Build Active Architecture Onlyパラメーターの1行のみが含まれ、その値はである必要があります。 いいえに変更します したがって、デバッグモードでも32ビットバージョンと64ビットバージョンがコンパイルされます。 その後、 Product→Clean （Cmd + Shift + K）に移動して、ビルドを再度実行します。 これで動作するはずです。



Visual C ++ 2010 Expressを使用してWindowsにインストールする



開始するには、 Visual C ++ 2010 Express .VS 2012をインストールしても動作するようですが、個人的にはテストしていません。



プラグインの64ビットバージョンを作成するには、コンパイラーの更新が必要です。これには、Windows SDK v 7.1の存在が必要になる場合があります。 VisualStudio 2010 SP1更新プログラムをインストールした後、Windows SDK v 7.1を正しくインストールすることは非常に困難です。 インストール中に、 fatal error



がfatal error



場合があります。

これを修正するには、Windows SDK v 7.1インストーラーを再起動し、Visual C ++コンパイラーとライブラリーのチェックを外します。

正しい構成を確実に行うには、次の製品をアンインストールして、この順序でインストールします。



  1. Visual Studio 2010
  2. Windows SDK 7.1
  3. Visual Studio 2010 SP1
  4. Windows SDK 7.1用のVisual C ++ 2010 SP1コンパイラー更新
