Intel Quick Sync Videoからのビデオの再コーディング-高速化





NVidiaおよびAMDの今日のトップエンドソリューションの4倍の速度。 これは何ですか サンディブリッジの秘密兵器：クイックシンク。 これは、トムのハードウェア記事が昨年呼んだものです。 「なぜQuick SyncがIntelの秘密兵器なのか」-これはすでにテクレーダーの出版物です。 実際、Intel Quick Sync Video（Intel QSV）テクノロジーには秘密はありませんが、残念ながら、それが何であるか、どこにあるか、そして最も重要なことには、それを効果的に使用する方法を完全に明確にすることはできません。 これらの質問に答えるのは、単純なユーザーだけでなく、ほとんどのインテル従業員でさえ、 特に夜に目が覚めた場合は難しいでしょう。 それを修正して、基本から始めましょう。



ロシア語のスペルでは、Intel QSVという略語は「IKSV」のように見えますが、これはさらに不確実性をもたらします-「X」、さらには「in」？ したがって（名前だけでなく）、名前は単に翻訳する方が適切です。

完全な翻訳は「高速ビデオ同期」です。 「ビデオ」とは何ですか-あなたはおそらく自分自身を知っています。 「同期」とは、元の「デスクトップ」からビデオを変換する機能です。 高解像度形式、モバイルデバイスおよびビデオホスティングでサポートされている形式。 ただし、これはQSVの唯一の可能な使用法とはほど遠いものです。

ここでの「高速」という形容詞は、すべての独立したトランスコーディングテストによると、QSVビデオがCPU上のソフトウェアコーディングだけでなく、ハイエンドGPUでも大幅に負ける場合があることを反映しています。



Intel Quick Sync Videoは、H.264 / MPEG-4 AVC、VC-1およびMPEG-2形式のビデオコンテンツのハードウェアデコードおよびエンコードのマーケティング用語です（これまでのところデコードのみ）。Intel統合GPUに実装-ウルトラブックからサーバーまでCore（Sandy Bridge）マイクロアーキテクチャの第2世代から始まります。

つまり、DVDまたはBlu-rayからビデオをトランスコードできます。


組み込みのIntel GPUには、汎用のエグゼクティブデバイス（実行ユニットまたは単にEU）に加えて、ビデオのデコードとエンコードを実装する特別なハードウェアモジュール、マルチフォーマットコーデックエンジン（MFX）があります。





このような固定ハードウェアソリューションは、ビデオ処理を高速化するだけでなく、CPUの負荷を軽減し、システムの電力消費を削減します。

さらに、デコードは前述のハードウェアモジュールで完全に実行されます。

エンコードは2段階で行われます。1つはGPUアクチュエーターで、2つ目はハードウェアで行われます。



Intel Developer Forumのプレゼンテーションのスライドは、コーディング段階がどこで行われるかを示しています。 GPUですべてが発生すること、つまり、 ハードウェア、および「ハイブリッドHW / SWソリューション」は、EUのプログラマブルGPUユニットを使用するという事実のみを意味します。





コーディングのこれらの2つの段階は、適切に伝達されていることに注意してください。 MFXハードウェアは1つのフレームを処理しますが、

このフレームに関する作業の一部を完了したEUは、既に次のフレームを処理してハードウェアに転送しています。

これにより、システム全体のパフォーマンスが自然に向上します。



Ivy Bridgeプロセッサ、つまりQuick Syncの2番目のバージョンでは、Sandy Bridgeハードウェアモジュールが改善されました。エンコード速度と品質が改善され、4Kビデオまでの超高解像度のサポートが追加されました。 このシステムは、複数のQuad HDビデオストリームを同時にデコードすることもできます。

そのため、ビデオをすぐにトランスコードする場合は、次が必要です。



1. Quick Syncをサポートする統合GPUプロセッサの存在。

既に述べたように、CPUは第2世代のCore（Sandy Bridge）マイクロアーキテクチャより古いものであってはなりません。

さらに、統合されたグラフィックスは、 Intel HD Graphics 2000以降の第2世代のものでなければなりません。 Pentiumブランド名で販売されている初期のSandy Bridgeモデルは、Quick SyncをサポートしていないIntelの紛れもないHDグラフィックスを備えているため、これは重要です。

現在、モバイルおよびデスクトップコンピューターのQuick Syncのサポートは、Intel HD Graphics 2000、2500、3000、4000、およびP3000モデルとP4000モデルのサーバーにあります。



2. GPUチップセット（マザーボード）のサポート 。

GPUがマザーボードのサウスブリッジ、つまり最終的にディスプレイポート（DisplayPort）にデータを送信するには、チップセットがフレキシブルディスプレイインターフェイス（FDI）をサポートする必要があります。これはGPUとディスプレイ間の「ブリッジ」です。 FDIをサポートするチップセットとCPUのモデルの現在のリストは、ウィキペディアの対応する記事にあります

ちなみに、この要件はQuick Sync Videoのサポートに特に関連するものではなく、単にIntel HD Graphicsがシステムで動作するための前提条件です。



そしてもちろん、システムがCPUの統合グラフィックスを無効にする外部グラフィックスカードを使用している場合、Quick Syncを使用することは一般的に不可能です。 いくつかの場合のこの問題の解決策：

  • ラップトップの場合-システムグラフィックを外部から内部に切り替える機能を使用します（切り替え可能なグラフィック）
  • デスクトップコンピューターの場合-2番目のモニターを統合グラフィックスポートに接続し、BIOSで使用できるようにしてプライマリにします。
  • LucidLogix Virtuグラフィック仮想化ソリューションを使用する

QSVは現在、Windows 7、Windows 8、およびMac OS Mountain Lionドライバーでサポートされています。

また、適切なWindows 7ドライバーをWindows Server 2008にインストールすることも可能です（デスクトップエクスペリエンスコンポーネントがインストールされている場合）。

しかし、最初に、上記のリストよりも自然のビデオ処理のための桁違いのアプリケーションがあります。 そして、それらのいくつかは、謙虚さからこれを報告することなく、QSVを使用するかもしれません。 第二に、宣言されたQSVのサポートとタスクへの実際の関与は、2つの大きな違いです。 多くのアプリケーションでは、QSVを有効にするには、完全に予期しない場所でチェックボックスをオンにするか、何らかの明白でない設定を変更する必要があります。何らかの理由でQSVが機能しない場合は、探す価値があります。

このすべてが、ビデオの処理時にハードウェアアクセラレーションが実際に使用されたかどうかを確認するというタスクにつながります。



上記のリストのアプリケーション、およびIntel Media SDKの使用が疑われるアプリケーション（証拠-アプリケーションによるlibmfxsw * .dllライブラリの使用）の場合、次のようにこれを行うことができます。

  1. Intel Media SDKをダウンロードしてインストールします
  2. 「<install-folder> \ tools \ mediasdk_tracer \」フォルダで、MSDKプロファイラ-tracer.exeを見つけて実行します

  3. 開始ボタンでロギングを開始し、実験アプリケーションでビデオ処理を開始します。数秒後、停止をクリックしてロギングを停止します。
  4. [出力ファイル]フィールドで指定されたログファイルで、次の行を探します。

    •MFXInit.impl =ハードウェアまたはMFXInit.impl = AUTO

    •MFXInit.status = MFX_ERR_NONE

    それらが見つかった場合-すべてが正常である場合、QSVが使用されます。 それ以外の場合、いいえ、または間違っていたため、アプリケーションはMSDKを使用しません。




後者の場合、鉄によって実行される別のテストは疑念を払拭するのに役立ちます。

  1. Intel Graphics Performance Analyzer（GPA）をダウンロードしてインストールします
  2. GPAモニターを起動し、システムトレイのGPAモニターアイコンを右クリックして、コンテキストメニューを呼び出し、その中のメディアパフォーマンスを選択します。



    開いたウィンドウには、エグゼクティブデバイス（EU）およびハードウェアビデオプロセッサ（MFX）Intel GPUの読み込みが表示されます。

  3. アプリケーションでビデオ処理を開始し、リアルタイムでIntel GPUコンポーネントの使用表を確認します。 または-表示されませんが、キャプチャを押してこのデータをログに保存します。 以下に示すように、表またはログファイルで「デコード」（デコード）および「エンコード」（エンコード）の行が点滅する場合、QSVが使用されます。
