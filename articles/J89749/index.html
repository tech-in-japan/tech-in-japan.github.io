OpenSourceが好きで、Microsoftと仕事をする理由

この投稿に基づきます。 ITスペシャリストの観点から状況を考慮します。

この質問をビジネスの目で見たいです。





あなた自身について少し。

私は30歳です。 6.5年間、私は小さな地方の町（約170,000人）の小さなITビジネスの共犯者でした。 同僚の中で最も専門性が高い：ソフトウェアを販売し、1Cを実装および維持し、システム統合に従事し、ユーザー向けのトレーニングクラスを開催し、ネットワークを引き出し、愚かなユーザーの司祭を一掃するなど。 一般的に、読者、死神、パイプの男。



私からの雌犬についての作家は悪いとすぐに言わなければならない

私は簡潔になります



マイクロソフトのソフトウェアが、その価格にもかかわらず世界の市場で最大のシェアを占めるのはなぜですか？







ビジネス向けMicrosoftソフトウェアの最も重要な利点は、標準化と相互統合です。 これにより、その人員の高い互換性、つまり ITプロフェッショナル。 さらに、販売による利益のおかげで、マイクロソフトは製品サポートだけでなく販売も獲得するアフィリエイトネットワークを常に開発しています。 これは重要な要素です。なぜなら Microsoftソフトウェアは積極的に販売されています。



フリーソフトウェアには、ユーザーの初期費用（読み取り-製品価格）がありません。これが主な利点です。 特にロシアのビジネスでは;）しかし、フリーソフトウェアとその使用を統合する過程で、Microsoftソフトウェアに固有ではない問題が発生します（私自身が遭遇した問題をリストします）。

-資格のあるスペシャリストを見つけることはさらに困難です（Windowsスペシャリストはさらに多くいます：「私はスペシャリストです。自宅でWindowsを2回再インストールしました！」;）。

-各スペシャルは、彼自身のチャーターで修道院に来ます。 誰もがソフトウェアを互いに統合するための独自のスキームと方法を持っています。 実際のところ、優先ソフトウェア。 したがって、「最初からやり直す」という状況を回避するために、次第に凍りついた人をdulわなければならない場合があります。

-一部の開発者は、ディストリビューションのサポートと開発を単に忘れていることもあります。 「最初からやり直す」。

-状況の信じられないほどの組み合わせにより、Windowsやその他のユーザーに慣れているユーザーを再教育する必要があります。

-本当の互換性の問題があります。 多くのビジネスパートナーと顧客は、マイクロソフト製品を何らかの方法で使用しています。



OpenSourceと著作権を組み合わせます。



フリーソフトウェアはどのようにして市場シェアを増やし、一般ユーザーの心をつかむことができますか？



これについての私の考えは次のとおりです。

1. 標準化と統合。 フリーソフトウェア開発者の異なるグループが団結すべきでしょうか？ または、ソフトウェア相互の相互作用、インターフェースなどの統一された標準とスキームを開発および承認する機関を作成します。 または、さまざまなソフトウェアの統合の問題を解決する別の開発者グループ。

2. ウィザードが有名になりますように！ Windows Server 2008でウィザードを見たことがありますか？ 私の意見では、彼らの助けを借りて新入生でもサイドボード上のサービスの束を上げるでしょう。 要するに、あなたは* nixの完全な単純化と、あらゆるものに関するステップバイステップのヘルパーを与えます!!! 複雑さとわかりにくいことで学校のベンチを怖がらせる必要はありません。 グルと好奇心のために-シェルはどこにも行かないでしょう。

3. 互換性！ これはすでに専門家の仕事です-お尻を引き裂きますが、WordとExcelと完全に互換性を持たせます 私の実践では、これが* nixでのワークステーションの展開に対する主な議論です。

4. ビジネスには完全な景品が必要ですか？ フリーソフトウェアは愛好家によって開発されていることを理解しています。 たぶん彼らは熱意を正しい方向に向け、さらにはお金で彼らを温めるべきでしょうか？ Microsoftソフトウェアと特性が類似し、フォーマットで互換性のある機能を備えたソフトウェアは、後者の3倍のコストがかかると想定します。これは、ソフトウェア市場の爆弾になります。 そして、これがフリーソフトウェアのアイデアそのものを殺すと考える必要はありません。 企業はコードを購入するのではなく、最小限のコストで可能な限り効率的にタスクを解決できるターンキーソリューションです。 ビジネスマン（マネージャー）は、これがオープンコードであるかどうかを気にしません。 彼は、製品の購入、実装、保守にかかる費用の合計、およびこの製品がもたらす利益と収入の組み合わせについて心配しています。 ソフトウェアを販売するのではなく、ターンキービジネスソリューションを販売してください。 まあ、ユニークなソリューションは罪ではなく、コピーペーストから保護します。



合計：



オープンソースの紳士は、著作権の大物から良いアイデアを急ぐことをheしないでください！ 同じマイクロソフトが何度も良いアイデアを取り、エンドユーザーの利便性を考慮して処理し、方向付け、そしてこれでお金を稼ぎ、その人気を高めました。 最後のオフィスからツールバーをコピーすることはお勧めしません。 ソフトウェア製品を構築および対話するための一般的なモデルについて説明しています。インターフェイスのシンプルさ、相互統合、構成と管理の容易さ、互換性、特定のタスク用の既製（組み立て済み）ソリューションです。 そして最も重要なことは、標準化です。

また、誰かが「フリーソフトウェア-マネー」接続を嫌う場合は、トレーニングスペシャリストに投資してください。 彼らは非常に不足しています。



PS実際には、トピックの見出しに対する回答（ソフトウェア消費者としてのビジネスの観点から）：

オープンソースソフトウェアは、その「低コスト」に満足しています。これは、最初は中小企業を絞めないために役立ちます。

また、Microsoftのソフトウェア（Apple、Oracle ...-近い方）-理解しやすく、便利で、保守が容易です。



そのようなもの。 私の意見では、それはやや厄介であることが判明しました。 私はオラトリーで成長することを約束します。
