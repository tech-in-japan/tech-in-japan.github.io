Raspberry Pi 3B +およびWebMeetingアップデートでの3CX v16テスト

Raspberry Pi 3B +でのMicro PBXのテスト+



もちろん、企業のIP-PBXを開発する場合、中規模および大規模企業向けに機能を追加することは興味深いことです。 しかし、あなたは認めなければならない、強い筋肉は強い骨格でのみうまく機能する。 製品レポートの品質は、オンラインレポートの色よりも重要です。 そして、この品質は、最小限のハードウェア機能を備えたシステムでのみ明らかになります。



私たちは、3CX v16がどのようにハードウェアに要求のないものであるかを見つけ出し、Raspberry Piで起動することにしました。 システムはさまざまなテストに合格しており、その結果を急いで共有します。







このガイドで説明されている非常にシンプルな上にv16 Raspberry Piをインストールします 。 前述のとおり、次のものが必要です。



  • デバイス-Raspberry Pi 3B +

  • メモリカードは32 + GB Class 10 Micro SDで、アクセス速度は少なくとも10 Mb / sです。

  • ケースはラズベリーパイと互換性があり、ラジエーターとファンの冷却が好ましい。

  • 電源-Raspberry Pi、2.5 A、Micro USBと互換性があります。



同様のキットは、たとえば、Aliexpressで54ドルで提供されています







ファンキットを推奨する理由 特に「重い」コーデックが使用されている場合、適切な負荷でPBXを長期間動作させると、デバイスが大幅にウォームアップされることが判明しました。 したがって、組織でPBXがアクティブに動作すること、またはG729コーデックとOPUSコーデックを使用する多くのモバイルユーザーが期待する場合は、必ずラジエーターやファンをインストールしてください。



最初は、Raspberry Piが多少なりとも深刻なインストールを「プル」しないことを恐れていましたが、3CXは最大約32ユーザーのシステムで優れた結果を示しました。







  • プロセッサの負荷は、非動作時の4％からピーク時の46％までで、平均負荷は約23.7％です。 これは、最適化された3CXコードを示唆しています。

  • メモリ消費-アイドル時の32％から負荷のかかった中程度の38％まで。 コードの優れた品質の別の確認。

  • ネットワーク負荷-Pi 3B +には、有効速度が約224 Mbpsの改良されたギガビットイーサネットコントローラーがあります。 8つの同時呼び出しでの平均ストリームは〜1.5 Mbit / sです。 負荷は1％未満です。 もちろん、多くはネットワークの品質と使用される音声コーデックに依存します。

  • 消費電力-X86-64アーキテクチャ上の平均的なサーバーが1日あたり約20.4 kWhを消費する場合、Raspberry Piは1年あたり同じ量を消費します。

  • 温度モード-ラジエーターまたはファンを使用しない場合、Pi温度センサーはアイドル状態では47.2℃、最大負荷では最大59.5℃を示しました。 平均動作温度は52.5°Cです。



$ 54でPBXを使用できる場所



テストを行った後、適切な冷却を行えば、この赤ちゃんは何年も続くと主張できます。 現在、推奨される使用例は、8つの同時通話と16〜32人のユーザーがいるシステムです。 各ユーザーのIP電話では、平均で最大8つのBLFインジケーターを構成できます（その番号は、3CX SIPスタックと組み込みWebサーバーの負荷に影響します）。 Raspberry Pi +の3CXが便利になる場所を想像してください。



  • 世界中のどこからでも従業員が互いに通信できるだけでなく、家族のメンバーも通信できる小規模企業向けのビジネスクラスPBX。

  • 親会社の電話交換機との接続が失われた場合に、それぞれが本格的なローカル電話交換機を必要とする大企業の支店。

  • 居住者がiOSおよびAndroid用のアプリケーションを使用し、コールパネル（SIP-インターコム）をシステムに接続できる住宅用のATS。

  • 従業員、教師、さらには生徒の学習間のコミュニケーションのための教育機関のATS。

  • ホテルの電話交換の興味深い機能を提供する小さなホテル。



ボーナス そして、何かがうまくいかなかった場合、あなたはすぐにあなたのポケットにPBXを入れて、それを持って行くことができます！

3CX WebMeetingアップデート-50倍高速化



会議の機会の改善



3CX WebMeeting Web会議プラットフォームの最新のアップデートでは、新しい会議機能、音声品質の強化、および便利な追加のMCUサーバーが提供されます。



Conference Flipchart（Whiteboard）は、圧力の度合いに応じて、よりスムーズで正確な描画を提供するようになりました。 この機能は、タッチスクリーンを搭載したタブレットやラップトップのユーザーに高く評価されます。 また、会議参加者のパネルなどでオブジェクトの上にマウスを移動すると、強調表示されます-便利です。







WebMeetingインターフェイスから信号強度が消えました。 通信チャネルの低速がビデオ品質の低下を引き起こす場合に表示される小さな通知に置き換えられます。 通知のおかげで、問題の原因を理解して対処することができます。 接続の品質に関する詳細情報に関心がある場合は、3CX WebMeetingセクションの「設定」>「ネットワーク」に移動して、詳細な統計情報を確認してください。







サービス品質の改善



現在、1か月あたり約100,000のWeb会議を開催し、150,000人が参加しています。 サービスカバレッジを拡大し、信頼性を高めるために、3CX WebMeetingサーバーをGoogleとAmazonのインフラストラクチャに移行しています。



短時間保存される会議データ（記録、統計）は、ヨーロッパのGoogleデータセンターに投稿されます。 これにより、信頼性、安全性、追加の技術的利点が得られます。 Web会議サーバー（MCU）自体は、世界中のAmazonのインフラストラクチャでホストされています。 これにより、エンドユーザーとの近接性と優れたパフォーマンスが保証されます。 MCU WebMeetingは現在、オーストラリア、ブラジル、カナダ、フランス、ドイツ、ロシア、南アフリカ、シンガポール、イギリス、アメリカ、ベトナム、インドにあります。 どこにいても、3CXは自動的に最も近いサーバーを提供しようとします。



音声会議参加者用のOPUSコーデック



WebMeetingの人気の客観的な理由の1つは、その優れた音質です。 最新の更新では、音声のみで会議に参加するユーザー向けに、適応性のある高品質のOPUSコーデックを接続しました。 以前は、G.722コーデックが使用されていましたが、モバイルネットワークではあまり機能しません。 もちろん、参加者がIP電話から接続する場合、他の音声コーデックもサポートされます。



記録とレポートが50倍高速



この更新では、ビデオ会議の記録と最終レポートの生成の処理も高速化しました。 統計によると、オーディオとビデオを使用した4人の参加者による1時間の会議は、完了後約50分でさらに処理されます。 その後、会議の記録（.mp4ファイル）へのリンクが開催者の電子メールに送信されます。 現在、後処理には1〜3分かかります。 インストールされているオプションとビデオ品質に応じて。
