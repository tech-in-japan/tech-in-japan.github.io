新しいWindows Azureリージョナルデータセンター



Windows Azureプラットフォームは、開発者やITプロフェッショナル向けのサービスやツールを絶えず拡大するだけでなく、インフラストラクチャによっても拡大し続けています。 2月25日に、日本の地域に2つの新しいWindows Azureデータセンターが発表されました 。

















クラウドプラットフォームのユーザーは、アプリケーション、サービス、インフラストラクチャをホストするために、プラットフォームの8つの領域を利用できるようになりました。









発表の一部としていくつかの数字が示されました。同社はすでに150億ドル以上をクラウドインフラストラクチャに投資し、世界中の90市場の10億人以上の顧客に200以上のクラウドサービスを提供しています。









新しいクラウドサービス-ExpressRoute



スコットガスリーは、新しいWindows Azure ExpressRouteクラウドサービスを発表しました。これは、顧客データセンターをWindows Azureインフラストラクチャに直接接続するための専用のプライベートな高性能ネットワークチャネルです。

















Windows Azure ExpressRouteを使用すると、クラウドプラットフォームのお客様は、Windows Azureのパワーをインターネットを経由せずにデータセンターに接続できます。これにより、クラウドとの連携がデータセンターに対してより透過的になり、データ交換速度の保証とセキュリティの向上が保証されます。









現在、ExpressRouteはプレビューステータスであり、米国のお客様のみが利用できます。 ご希望の方は、サービスの試用を申し込むことができます 。 サービスの説明とその可用性は、 このページで確認できます 。

Webサイト。 バックアップとサイト回復のサポート



Windows Azure Webサイトでホストされているサイトには新しい機能があります。 開発者と管理者は、サイトのバックアップコピーを作成し、必要に応じて管理ポータルから直接復元することができます。

















Windows Azure Webサイトのバックアップを作成するということは、デプロイされたアプリケーションの現在のバージョンのスナップショットを、サイトで使用されるコードとデータベース（SQLデータベースまたはMySQL）とともに作成することを意味します。 ユーザーは、スケジュールに従って手動バックアップにアクセスしたり、自動バックアップを構成したりできます。









標準ユーザーは追加料金なしで新しい機能を利用できます。









Windows Azureモバイルサービスの新機能



2月に、開発者に人気のあるすべてのモバイルプラットフォーム用の既製の単一バックエンドを迅速に提供するWindows Azureモバイルサービスが、別の更新プログラムを受け取りました。

更新プログラムのリリースにより、開発者は次の利点を利用できるようになりました。









  1. モバイルアプリケーションを開発するために、ASP.NET Web APIとVisual Studioをモバイルサービスとともに使用する機能

  3. ODataコントローラーやBreezeなどのサードパーティフレームワークを含むASP.NET Web APIのすべての機能を使用する
  4. Visual Studioでのモバイルバックエンドサーバーコードのローカルおよびリモートデバッグの可能性
  5. WebAPIコードは、モバイルサービスのおかげで自動的に実行、動作、管理、および拡張されます

新しい仮想マシン機能



2月に、Windows Azureおよび仮想マシンのIaaSプラットフォームに新しいVMエージェント機能が導入されました。









VMエージェントは、各仮想マシン（WindowsまたはLinux）に、仮想マシンとクラウドインフラストラクチャ全体を管理するための高度な機能の実装に役立つ特別なプロセスを自動的かつ透過的に配置する機能です。

















エージェント自体には特別な機能はありませんが、便利な拡張機能をインストールして使用できます。拡張機能は、仮想マシンの機能を拡張し、管理を簡素化するソフトウェアです。 時間の経過とともに、多くのそのような拡張機能が表示されます。 現在、チームはそのようなデモ拡張機能の1つ、 Background Info Extension（BGInfo）を提供しています。

















BGInfoはVMエージェントの拡張機能であり、RDPを介してデスクトップに接続すると、デスクトップ上の仮想マシンに関する有用な情報を表示できます。 この情報により、マシンの設定と構成に関するデータをすばやく取得できます。









HDInsight Service Update（Hadoopクラスターオンデマンド）



HDInsightは、クラウドベースの100％互換性のあるWindows AzureベースのHadoopエコシステムの実装であり、別の更新プログラムを受け取りました。









Windows Azure HDInsightユーザーは、Hadoop 2.2ベースのクラスターをオンデマンドで展開できるようになりました。 新しいバージョンでは、要求に対する応答性が大幅に向上し（最大40倍）、データ圧縮（最大80％）によりストレージ要件が削減され、サポートおよびYARNのすべての利点が得られます 。









HDInsightクラスターの操作を開始するには、ポータルAzureHub.ruまたは次のリンクの公式ドキュメントにアクセスできます。

新しい管理および監視機能



Windows Azureの管理、管理、および監視機能は常に進化および改善されています。 2月に、いくつかの有用な革新が行われました。









ポータルでのプラットフォーム障害の通知 。 残念ながら、Windows Azureプラットフォームではまれですが、一部のサービスは失敗します。 このような状況について理解を深め、タイムリーに顧客に知らせるために、Windows Azureポータルにリアルタイムの障害通知のサポートが追加されました。

















これで、ユーザーはポータルで直接使用するサービスの失敗に関する通知を受け取ります。









  • 劣化と品質の通知
  • サービスの部分的な利用不可
  • 完全なパフォーマンス低下
  • サービスが完全に利用できない
  • セキュリティ通知


リアルタイムの通知とともに、ユーザーにはインシデントに関する詳細情報が提供されます。

















共同管理者の数の制限を増やします。 以前は、Windows Azureアカウント管理者は、サブスクリプションの一部としてクラウド容量を管理する権限が譲渡された最大10人の共同管理者を作成できました。 ただし、共同管理者が10人しかいないという制限は、多くのユーザーには適合しませんでした。









2月の更新では、共同管理者の数の制限が、サブスクリプションごとに10から200に増加しました。 これにより、1つのサブスクリプションの力で、より多くの開発チームがより効率的に作業できるようになります。









新しいPowerShellコマンドレット コマンドラインおよびWindows Azure PowerShellスクリプトを使用した管理ツールは、最新のプラットフォーム機能をサポートするように常に更新されています。









Windows Azure PowerShellの新しいバージョンには、新しいプラットフォーム機能を使用できるようにする多数の新しいコマンドレットが用意されています。









  • ステージング、WebJobsなどをサポートするWebサイトのコマンドレット。
  • 拡張サポートなどを備えた仮想マシン用のコマンドレット。
  • 拡張サポート、静的IP設定などを備えたクラウドサービスコマンドレット。
  • メトリックとロギングなどをサポートするストレージコマンドレット。
  • ExpressRouteコマンドレット


詳細なリストと説明は、このオープンツールキットのGiHubページにあります。









Windows Azure Active Directoryの革新



SaaSソリューションカタログの拡張 。 シングルサインオン（SSO）を提供するためにWindows Azure Active DirectoryでサポートされるクラウドベースのSaaSソリューションのカタログは、600個のアプリケーションに拡張されました。 その中には、Box、Google Aps、Dynamics CRM、Office 365、Citrixサービスなどの人気のあるサービスがあります。

















Windows Azure Active Directoryと提供されているカタログを使用すると、シングルアカウントを介してサービス、Microsoftサービス、サードパーティサービスにシングルサインオンを簡単に統合できます。









プレミアムモードとレポート 。 大規模なエンタープライズクライアント向けに、Windows Azure Active Directoryは新しいプレミアムモードを提供します 。 2月に、プレミアムユーザー向けに追加のレポート機能が導入されました。これにより、サードパーティサービスの消費と、社内でのサードパーティサービスへのアクセスをよりよく理解できます。









プレミアムサブスクライバーのレポートは、[レポート]タブで利用できます。









グループ管理 。 2月、Windows Azure Active Directoryプレミアムサブスクリプションのユーザーに、ユーザーグループの自己管理のための新しい機能が提供されました。

















新しいポータルを使用すると、管理者だけでなく、会社の一般ユーザーもグループを作成、検索、参加、およびグループからの参加を除外できます。









Windows Azure Active Directoryのこれらおよびその他の発表は、開発チームのブログで見つけることができます。









新しいBizTalkサービスクラウドサービス



BizTalkサーバーのいくつかの機能のクラウドベースの実装-Windows Azure BizTalkサービスは更新された機能を受け取りました。









EDIFACTプロトコルのサポート 。 クラウドサービスは、メッセージングの国際標準であるEDIFACTプロトコル（国連/管理、商業、および輸送のための電子データ交換）をネイティブでサポートするようになりました。









Service Busメッセージのサポート 。 BizTalk Servicesは、Windows Azure Service Busクラウドメッセージングサービス、キュー、トピックと簡単に統合できるようになりました。 更新されたBizTalk Services SDKは、そのままの状態でService Busキューからメッセージを取得するためのサポートを提供します。

















BizTalkサービスは、サービスバスのサポートとともに、メッセージキューを操作するためのサポートと共有アクセス署名（SAS）を追加しました。









バックアップとリカバリのサポート 。 2月に、BizTalk Srevicesは、管理ポータルから簡単に構成および使用できるバックアップおよび復元機能を追加しました。

何百万人ものファンのためのWindows Azure 。 Windows Azureプラットフォームのおかげで、何百万人ものファンやゲームのゲストが、ウェブサイトsochi2014.comで必要な情報をリアルタイムですぐに受け取りました。 ピーク負荷時に必要なコンピューティングリソースの可用性を確保しましたが、その量を事前に予測することはほとんど不可能でした。 Windows Azureプラットフォームは、動的で費用対効果の高い、高性能の情報システムを迅速に作成および運用するのに最適です。

















スポーツイベントの全期間を通じて、Windows Azureが提供するsochi2014.comサイトの中断のない運用に関して、以下の統計が収集されました。









  • ソチのオリンピック開催中、公式ウェブサイトは1億回以上訪問されました。
  • ユーザーの最大数は、ロシア、アメリカ、カナダ、ウクライナ、フランスから来ました。
  • フィギュアスケート大会と非公式のメダルの順位が最も注目を集めました。
  • ピーク負荷時に、サイトには同時に175千人以上の訪問者が参加しました。これは、最小消費期間中のユーザー数の15倍でした。 オンライン結果のあるページを調べて、1秒あたり9万件以上のクエリを作成しました。
  • 500万人以上のユーザーが毎日サイトを訪れました。 彼らは3000万ページ以上を閲覧しました。
  • Windows Azureは、あらゆるプラットフォームのモバイルデバイスに情報を提供するために使用されました（今日では、ソチ2014の公式アプリケーションのコピーが600万個以上インストールされています）。
  • 大会中のサイトビュー数は5億を超え、Azure仮想マシンへのリクエスト数は250億を超えました。
  • 公式ウェブサイトの設計と実装が監督されている法人顧客向けのコンサルティングおよび技術サポート部門も技術サポートを提供しました。 23人の専門家からなるチームが、24時間年中無休でゲームの全期間を通して現場で作業しました。
