王-王が長く生きます！





カードゲームがプレイされており、プレイされます。 何千年もの間、ギャンブラーは何百ものゲームを試し、数十を残しました。 何世紀もの間。 文学評論家のようになると、カードゲームは3つのテーマ（ポーカー、好み、ジン）に分類されます。 たとえば、 愚か者は単純化された好み、一種の切り札です。



今はどうなっているのかわかりませんが、 学校や毛皮の仲間で昼と夜を優先して切り刻まれました。 16〜24人の聴衆で講演をすることができた人もいました。 そこは便利です 行-サーカスのように。 多くの人がMCCを知らないからです。

女の子は王様 、女性の好みを好む。 良い家族ゲーム。



アルゴリズム化、プログラミングが簡単です。 デスクトップPCとTurbo Pascalコンパイラを入手するとすぐに、最初にこのゲームのコンピューターバージョンを作成しました。 1991年、カエデの葉が落ちていました。 ライバルがi286をプレイした場合 、クラシックミニマックスは実装できませんでした。 このプログラムは、3移動の深さで最適な答えを正直に計算しました。 ストロークの深さ4で、コンピューターは30分間定期的に思慮深くなりました。これは、最も愚かなライブプレーヤーでさえ余裕がありませんでした。 ゲームの入り口を5桁のパスワードで暗号化しました（当時は流行でした）。 パスワード-人気新聞Komsomolskaya Pravdaの購読インデックスであり、ヒントはパスワード入力画面にありました。

コムソモルスカヤのジャーナリストは、何度か私に電話してゲームについて話してくれましたが、私は拒否しました。 控えめな理由ではなく、当時トイレットペーパーがなかったため、人々は新聞を使いました。 私の記事がどこに行くかは明らかです...







DOSの下のキングI



最初のKingは、MS-DOS 2.0オペレーティング環境で起動されました。 ゲームはどのように作成されましたか？

カードゲームの作成は簡単です。 32枚のカードを引き、画面の解像度を知るだけで十分です。 DOSの時代の画面は640 x 350、カード-64 x 96ピクセルでした。







カードの表紙はサナダムシから引き抜かれ、彼が描いた女性と王の写真は、くまのプーさんからチェブラーシカまで有名なソビエトの漫画のキャラクターの画像を目の前に保持していました。 雑誌から写真を撮りました-明るいカバーのビデオカセットはありませんでした。 数年後、最初のハンドヘルドスキャナーが登場しました。写真の品質は手の精度に依存していました。 強打！ スキャナーをマガジンに通すと完了です。 写真は白黒で小さく、スキャナーは奇跡のようでした。



このゲームは非常に人気があり、以前のユニオン全体にフロッピーディスクで広まりました。 彼女はお金を持ってこなかった、その時はお金がなかった。 フードスタンプと男性用パンツがありました。



Windows 3.1でのKing II





2番目のKingはWindowsで再リリースされ、Visual Studio 2.0でC ++で作成されました。 1997年の新しいパラダイムへの移行は私にとって困難でした。パスカルでは直線的に考えましたが、ここではいくつかのクラスでいくつかのイベントが現れました。 脳の準備ができていませんでした。 それにもかかわらず、640 x 480のWindowsウィンドウのハード解像度でゲームを作成しました。マップのサイズは64 x 96ピクセルのままでした。 アートの質は変わりました-カードを引きませんでしたが、solitaire.dllから引き出しました。 独自のMicrosoftイメージをアプリケーションに挿入しました。



2つのおっぱいとライブの対戦相手を持つゲームのバリアントがありました。 ネットワークゲームは、数学部門（1,500人）の内部秘密構造のために作成されたもので、研究所を超えていません。 リモートプレイでは、MFCソケットを使用しました。



Windows CE 2.0でのKing III





3番目の王は2002年にiPaq 3700用に作られました。 最初のハンドヘルドコンピューター、おとぎ話。 Glitchy、ただしWindowsで。 MS VSからMS Embedded Studioへの開発の移行は、VAZ 2103からVAZ 2101への移行方法でした。非常に自然です。 各方向で半分になったカードのサイズ。 240 x 320の新しい画面サイズは、画像の変更を余儀なくされました。 カードの表面には、意味とスーツだけが残っていました。 残りは迷惑なほど小さく、有益ではありませんでした。 デバイスにアプリケーションをインストールするプロセスは面倒でした。 たくさんの魔法の踊り。 エミュレータは上手く機能しました。 スクリーンを押すことは、スタイラスを使用して実行されました。 それ以来、私はスタイラスが嫌いです。 彼らは手を取り、迷子になります。 もちろん、半年後、ハンドヘルドのすべての所有者は、失われたスタイラスの代わりにつまようじを使用しました。



SymbianのキングIV





4番目の王は、2002年にノキア9210用に作られました。 それは単なるハンドヘルドコンピューターではなく、コミュニケーター、電話でした。 凡例 Epoc32。 同じC ++ですが、出力ビットマップの構成がわずかに異なります。 Symbianの開発は問題を引き起こしませんでした。 画面サイズが640 x 200になり、最初の白黒CGAモードを思い出させたため、カードのサイズは再び元のサイズに戻りました。







エミュレータは起動しましたが、非常にゆっくりでした-各起動には1分間の待機が伴いました。 アッラーを称賛し、インターネットが登場し、どのDLLがコンピューターから取り外され、そのような遅延が発生しないかに関する情報が提供されました。 Nokia PC Suiteをインストールするには、ダンスだけでなく、タンバリンとNa-naグループの歌と一緒に踊る必要がありました。 ところで、電話はまだ作動状態です。



AS3.0下のキングV





5番目のキングは、フラッシュアニメーションの最初のコースを学び、開発用のツールを入手した2007年から2008年に作成されました。 これは、コムソモルスカヤ広場のキオスクで100ルーブルで正式に購入したAdobe FLEXツールです。 チェックは保存されています。

画面サイズとカードサイズは動的に変化し始めましたが、画像の品質は変わりません。 iOS氏が来たため、ゲームを開発することはできませんでした。



iOS 2.0用のKing VI





6番目のキングは、2009年にiPhone用に作られました。 画面サイズは320 x 480、最適なカードサイズは66 x 96です。このサイズはiPhoneのすべての将来のバージョンで保持されているため、お勧めします。 私は、デザインではなくプロセスが重要であると信じて、地図を急いで描きました。 しかし、カードゲームのおかげで、Photoshopで働くことを学びました。 時々ぼろぼろのように、地図の画像をリアルにしました。







ゲームは有料（0.99）で無料で、4年間で229ドルと11,864ダウンロード（図を参照）でもたらされました。







iOS 7.0のKing VII





7番目の王は昨日作られ、自由になりました。 フラットデザイン、アニメーション、スピン、フリップ、リットベルガーなど、時代の精神に従ってゲームを作り直したかったのです。

起こったことを評価してください。







強力なインターネットが私に写真の海を与えてくれたので、私はサウスパークからヒーローを選びました。私の意見では、iOS 7のスタイルでうまくいきました。

ゲームの知能は低いですが、コンピューターをプレイするためのアルゴリズムについて考えていると思います。







ご清聴ありがとうございました。結論として、ゲームへのリンクを無料で公開します。
