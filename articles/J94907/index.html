Web Standards Days-リガ

短いレビュー



すべてがレベルで、非常にスムーズに実行されたという事実から始めたいと思います。 トピックは関連性があり、提示された情報は関連性があり、スピーカーは興味深く、ユーモラスでさえありました。 技術的には、すべてが素晴らしかった-会議室は1か月も経っていなかったので、最新のファッションが装備されています：2つのスクリーン、正面に1つ、側面に1つ、部屋全体に均等に配置されたスピーカー、ワイヤレスリモートコントロール、マイク、およびWiFiやエアコンなどのグッズ。 飲酒やスナックにも問題はありませんでした。ホテルの従業員は、お茶、コーヒー、レモネード、クッキーを聴衆の最後に常に補充していました。



講義のコースはうまく機能し、ほとんどすべてがスケジュール通りであるかのように十分な時間がありました。 はじめに、1日の始めに30〜45分の3つのレポート、15分の休憩、さらに3回の講義の後、ご自身の財布からのおいしいランチ、最後に講師は3分の1分間小さなホリバルキクを配置しました。



概要の詳細



上記の日課から次のように、私たちは最初に自分がどこにいるのかを知らされ、それから正確に6つの講義があり、最後に少し光りました。 以下にすべてを時間と他の棚に入れます。



11:00-ニキータ・セレツキー
発見


Web Technologiesの学部長/教師であるNikita Seletskyは、全員に喜びで挨拶し、会議のアイデアについて話しました（名前は理解には十分ですが）が、自分自身とプレゼンテーションが必要な他の人たちを紹介し、みんなが楽しい一日を過ごすことを望みました。







11:15-ニキータ・セレツキー
ラトビアのWebレイアウト：昨日。 今日は！ 明日？


導入から、Seletskyはスムーズに彼のレポートのプレゼンテーションに進みました。 彼は少し不確かに言ったが、これは行動の印象を損なわなかった。 コンテンツは本質的に統計的でした。 グラフは、特定のDOCTYPEの使用、標準への準拠、HTML5の存在、および同様のものについて提示されました。 会議の参加者のWebサイトから統計が収集され、有効なページの割合に関するデータを提示すると、目が低下しました。 この物語は、情報よりも入門的でした。 最も興味深いものはすべて、Seletskyに不快感を与えることなく続きます。



12:00-ドミトリー・デュレポフ
モバイルデバイス用のWebサイトを作成する機能


ラトビアのWeb開発の先駆者であるDmitry Dulepovは、ガジェットのWebサイト開発における問題と解決策について話しました。 スピーカーは最初、スマートフォンからモバイルインターネットをほとんど使用していることをみんなに安心させたので、気にする価値のあるブラウザはOpera、Safari、Chromeのモバイルバージョンになります。 同様に、これらは最も活発なクライアントであり、デスクトップのクライアントとほぼ同じ機能を備えています。 したがって、HTMLパーツは簡単に実装できます。 モバイルブラウザは、サイト上のフラッシュなどを除外し、指向けのインターフェイスも必要とします。 そして、講師が詳しく説明した多くのフレームワークが私たちの助けになります：iWebKit、WebApp.Net、SproutCore。 彼はまた、Gianduiaについて話し始めましたが、名前以外は何も言いませんでした。 それは非常にリンゴであり、すべてがiPadとiPhoneの周りで回転しました。 ちなみに、6人の講師のうち5人がMacBookから発表したようです。







13:15-アントン・ネムツェフ
HTML5およびCSS3。 Web標準の進化


10年以上の経験を持つ民間人であるAnton Nemtsevは、現在の未来のWeb標準で私たちを待っているグッズについて詳細に全員に語りました。 これはおそらく、最も有益なプレゼンテーションの1つでした。 彼は、HTML5がどれだけ大きく前進したか、なぜ革命よりも進化しているのか、以前のバージョンから残っているか消えているもの、出現したものは新しいもの、DOCTYPEになったもの（そして、そうなった-<！DOCTYPE HTML>）などを段階的に廃止しました。 また、第2部では、Nemtsevはより遠い未来のWeb標準について少し夢を見ました（たとえば、ページにデータをプレーンなCSSで配置するアイデアや、互いに相対的なdivの動作の単純な宣言など）。それらのいくつかはすでにいくつかの場所にあります実装（通常はSafariで）。 3番目の部分では、CSS3の主要な革新とそれらの潜在的な問題を明らかにしました。 すべてには、追加の種類のフォームやアニメーションなど、既存のコンポーネントの動作のデモンストレーションが伴いました。 非常に有益で新しいことが判明しました。



14:00-すべてすべてすべて
食べ物。 よく食べる場所を見つける


会議参加者だけでなく、すべてすべてすべてが、適切なレストランまたはマクダカを求めてリガに分散しました。 私は幸運だった、私は地元に住んでいて、私は思うように食べた。 残りを保証することはできませんが、誰もが幸せに戻ってきたようです。







15:00-ヴァディム・マケエフ
CSS管理 3年後


Opera SoftwareのWebエバンジェリストであるVadim Makeevは、CSSの開発と管理を深刻なプロジェクトの最も重要な部分と呼び、プレゼンテーションを始め、この巨大な仕事の量を減らす方法について話しました。 全体のストーリーはyandex.ruスタイルパーツを作成した経験に基づいていたため、提案された問題の関連性およびソリューションの正確性については質問がありませんでした。 Vadimが最初に通知したのは、.cssファイルのモジュール性です。 必要なものだけがページに添付されていると非常に便利です。サーバーとクライアントの負荷が軽減されます。 各モジュールが個別に要求されたときに遅延が失われないように、開発段階で自動CSS接着を個別のファイルに作成することはさらに良い方法です。 また、サーバーの肩にブラウザの定義とハッキングを配置できないという事実、それが間違っていることが判明する可能性、およびこれらのハックの定義と最終ファイルへの正しい組み込みについても説明しました。 CSSマークアップの良いスタイルについていくつかの言葉がありました-閉じ中括弧のピエロ、要素のツリー構造（親による）、およびコード部門のコメントによるマークアップ。 標準のHTMLタグの一部のプロパティをリセットすることの有用性と、特定の基準（ページの表示順序、作成者の例 ）に従ってCSSファイルのコンテンツをソートする必要性について説明しました。 非常にスムーズかつ明確に提示されました。







16:00-ミハイル・バラノフ
開発者向けツール-どれが美味しいか？


SPbSPUの専門家養成学部のWeb開発コースの著者であるミハイル・バラノフは、HTML + CSS + JavaScriptを活用するためのツールについて話すことにしました。 最初のスライドには1996年という数字が付いていました。これは、著者が2006年（FireBugのリリース年）を意味していました。 これにより、クライアントアプリケーションのデバッグと解析のこれまでの無秩序で恐ろしい光の中で、平和で静かになりました。 このレポートは、タイプセッターやJavaScriptプログラマーにとってかなり明らかなものでいっぱいでしたが、これらのツールの可能性を棚に置いて、それぞれの特別な瞬間を強調しました。 彼らは、ColorZilla、YSlow、IE Developer Toolbar、およびOperaのDragonFlyや新しいIEとChromeのnonameなどの一部のブラウザーのネイティブ機能について言及しました。







17:00-マキシムバーチ
リーンスタートアップ。 アイデアから百万まで


ラトビアのスタートアップであるWeb開発者であるMaxim Berezaがカンファレンスに参加して、自分のプロジェクトを作成した経験を共有しました。 マキシムはうまく話しましたが、どういうわけかぼやけています。 彼がどれだけ稼いだか、どれだけ投資したか、正確にスタートアップとは何なのか、誰も理解していませんでした。 しかし、彼はプロジェクトの技術的な部分を紹介しました-使用されたフレームワーク（Django、jQuery）、サーバー上にあるもの、作成中に保持された原則（主なことは可能な限りユートピアに近い使いやすさ）、コードを保持する（SVNサーバー上）、保存トラフィック（スクリプトとHTMLの圧縮）、スタートアップのバイブル「Getting Real」も紹介しました。







17:40-ほとんどすべてのスピーカー
ラウンドテーブル「Web Standards vs. リッチインターネットアプリケーション»


後に参加したミハイルバラノフを除くほとんどすべてのスピーカーは、ホリバーのaをゆっくりと静かに掘りました。 それは、もはや抽象的なリッチインターネットアプリケーションではなく、その代表的な代表であるFlashについてでした。 フラッシュの明らかな支持者はいませんでしたが、この技術に関する残りの信念を揺るがした人たちがいました。 かなり控えめな状況が発生し、誰もがフラッシュの長所と短所、彼が悩んでいる人、どのように、そして将来貧しい人が彼のために輝くものを話し合った（そして、結局のところ、彼はまだ輝くだろう）。 良い結末、正直に言うと、大笑い。







終わる代わりに



イベントは素晴らしいです。 たぶん、ここで誰かが自分自身のために新しいものを見つけることはないかもしれませんが、いずれにせよ、これは楽しくて面白い人々の間で楽しい時間を過ごすための大きな理由です。 昼食時に、2人の地元の学生に会い、多くの興味深いことを学び、特定のプロジェクトで協力することさえ計画しました。 だからここに。 最高！
