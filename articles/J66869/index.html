Javascript



まず、Opera Uniteサービスは特別な種類のウィジェット（はい、Operaの同じウィジェット）であり、多くの機能を備えているという脳の情報を取り入れます。 これにより、ウィジェットAPI関数が適用可能になります。

Javascriptはサーバーとクライアントに分かれています。 Opera Unitedのすべてのロジックはサーバーに登録されます（PHP / Ruby / Python /の類似物を考慮してください）。クライアントではユーザーが見るサイトのロジックが実装されます。 たとえば、jQueryを使用できます。 UniteのHTMLは、MVCからのビューに似ています（それぞれ、サーバー側のJavaScriptはコントローラーに類似しています）。 ユニットは、スクリプトから渡された変数に応じてHTMLを生成します。

したがって、クライアントスクリプト、CSS、ファイルはデフォルトフォルダーpublic_htmlに配置されます。 サーバースクリプトとHTMLは任意のフォルダーに配置できます。それらへのパスは自分で指定します。 これを行うには、ルートで、将来の構成ファイルの横に、index.htmlファイルが作成され、タイプの構造を示します。

<!DOCTYPE html>

<script src="script/markuper.js"></script>

<script src="script/functions.js"></script>

<script src="script/operafunctions.js"></script>

<script src="script/script.js"></script>

ファイル共有



最も重要な機能。APIにはほとんど説明されておらず、例にはまったく記載されていません。 そのため、サービスのインストール後、config.xmlで次の構成を指定した後、共有するフォルダーをユーザーに確認するように強制します。

<feature name="http://xmlns.opera.com/fileio">

<param name="folderhint" value="home" />

</feature>







homeは、ディスク上のユーザーフォルダーです（Documents and Settings / userなど）。 フォルダーの他の例はAPIにあります。 ただし、フォルダーへのパスを指定することはできません。 クロスプラットフォーム、あなたは知っています。

これが困難の始まりです。 フォルダーは共有されていますが、ファイルはありません（！）。 間接的にユナイテッドにマニュアルから反対が続きますが。 ファイルを共有しなければならないことに加えて、別の問題があります。 サービスが不安定なため、一度共有されたファイルがセッション中に共有されるという事実に依存することは不可能です。 したがって、リクエストが利用可能になったときにファイルへのアクセスを処理することをお勧めします。 ユーザーがファイルを要求しました-共有、ダウンロード-共有のステータスを取得しました。 残念ながら、既存のunshareFile関数、shareFile関数は、既に共有されていないファイルまたは共有ファイルを適宜スリップすると致命的なエラーをスローします。 つまり 2回の共有は機能しないものではなく、機能しないサーバーも受け取ります。

共有フォルダーにはエイリアスが共有されています。 それに加えて、アプリケーション（サービスフォルダー）、ストレージ（サービスデータに割り当てられたフォルダー）があります。 サーバーに共有接続します：

window.onload = function () {

サーバーとサービスをオンにした後、共有フォルダーをマウントします。 上記のOperaの不安定性を考慮すると、サーバーがリクエストを手動で処理することをお勧めします。

次に、ダウンロード機能自体を作成します。

function download(e) {

var req = e.connection.request;

var res = e.connection.response;

var mp = opera.io.filesystem.mountPoints;

var filePath = fullPathWithoutServiceName(req.uri);

var file = mp.resolve(filePath);

if(file.exists) {

if(file.isFile) {

opera.io.webserver.shareFile( file, filePath );

res.closeAndRedispatch();

opera.io.webserver.unshareFile(file);

}

if(file.isDirectory) {

//deep inside... folders, folders, folders. total commander :)

showEntryList(e);

res.closeAndRedispatch();

}

}

else {

res.closeAndRedispatch();

}

res.close();

}







ここで長い間停止することはありません。API関数の名前からすべてがほぼ明らかです。 urlのサービス名の後にパスをマウントします。 fullPathWithoutServiceNameは、自分で作成した関数、URL解析です。 ファイルが実際に存在する場合は、それがフォルダーであるか、ここでハングアウトしているだけかを確認します。 フォルダーの場合、showEntryListハンドラーにアクションを渡します。このハンドラーは、ディレクトリ内のファイルとフォルダーのリストを表示するだけです。 ファイルの場合、ファイルを共有し、手綱をサーバーに転送し、戻ったときにファイルから権利を奪います。 ファイル名を表示する前に、すべての文字がそこでエンコードされていることに注意してください。表示する前に、それらをunescape関数でラップする必要があります。 それ以外の場合、最も単純な場合、ファイル名のスペースの代わりに、ユーザーには％20が表示されます。

closeAndRedispatch関数を使用して、サーバーに関数を返し、作業を続行します。 T.O. マルチレベルのフォルダーとファイルで作業をセットアップします。



バージョンアップ



Firefoxのような、更新に関する同様の通知メカニズムはありません。 または、ユナイテッドのサイトにあるDOAPファイルを使用して、アプリケーションの最新バージョンに関する情報を使用できます。 したがって、たとえば、サービスにアドレスがある場合：

unite.opera.com/service/322

次に、そのDOAPファイルは次の場所にあります。

unite.opera.com/service/doap/322

これはプレーンXMLであるため、解析してバージョンを確認し、ポップアップウィンドウにポップアップすることを提案できます。 処理例を次に示します。

function getLastVersion() {

try {

var req = new XMLHttpRequest();

req.onreadystatechange = function(x) {

if (this.readyState == 4) {

if(this.status == 200) {

var xml = this.responseXML;

var release = xml.getElementsByTagName('release').item(0);

var version = release.getElementsByTagName('Version').item(0);

var revision = version.getElementsByTagName('revision').item(0).firstChild.data;

var lastCheckedVersion = getLastCheckedVersion();

if(revision && lastCheckedVersion && revision!=lastCheckedVersion) {

widget.showNotification("New version "+revision+" is available. Click here to update.", function() {

widget.openURL(globalSettings.url);});

setLastCheckedVersion(revision);

} } } };

req.open('GET', globalSettings.doap);

req.send(null);

}

catch(e) {

}

}



function getLastCheckedVersion() {

return widget.preferenceForKey('last_checked_version');

}



function setLastCheckedVersion(version) {

widget.setPreferenceForKey(version, 'last_checked_version');

}











外部リソースへのアクセスが使用されることに注意してください;前述のように、config.xmlを修正する必要があります。

URL unite.opera.com/service/322は、理論的には、サービスディレクトリをアタッチしてconfig.xmlファイルに移動することで自動的に取得できます。これは、IDの形式でURLを入力する場所です。 しかし、私はこれを掘り下げるために破られました、誰かがコードを書いて開いたら、私はうれしいです:)そして、この関数が書かれた時までに、私はすでにこのURLを知っていました
