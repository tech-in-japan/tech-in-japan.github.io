ブレーンストーミング



原則として、ブレーンストーミングは、Windows Phone用のアプリケーションを設計する（またはこの領域から何かを設計する）最初のステップです。 この段階は完全に自由です。 たとえいくつかのアイデアが他のアイデアよりもクレイジーであっても、それはたくさんのアイデアを思いつくことです。 それは革新的なソリューションを思い付くために枠を超えて考えることです。

ブレーンストーミングに最適なツールはストーリーテリングとメモ用の膨大な数のカラフルなステッカーですが、スケッチやムードボードも役立ちます。 まず、 この記事には、効果的なブレーンストーミングのための10の非常に良いヒントがあります。

基本的に、 ブレーンストーミングを使用して2つのことを特定します。1）アプリケーションの一般的なアイデア、 2）アプリケーションの可能性。

ブレーンストーミングは、ユーザーにユニークな価値を提供し、マーケットプレイスの他のアプリよりも目立つWindows Phoneアプリの興味深いアイデアを思い付くのに役立ちます。 アプリケーションの全体的な目的を特定したら、ブレインストーミングを使用して、アプリケーションの特定のアイデア（機会）を作成および探索できます。

最初に、解決するにはテーマまたはメイン（一般）ユーザースクリプトが必要です。 ここで見られる典型的なものの1つは、「外部からアクセス可能なAPIとWebサービスがあるという理由だけでアプリケーションを作成する」というほぼ直感的なアプローチです。 したがって、最終的には、多数のYelp！、Twitter、またはFoursquareの顧客がいます。 これらのサービスに対してより良いクライアントを作成しようとすることには何の問題もありませんが、革新する本当の機会は、APIやWebサービスではなく、ユーザーエクスペリエンスの観点から考えることで見つかります。 したがって、「 Yelp APIに基づいてアプリケーションを作成します！ 」と言う代わりに、「 ユーザーが食事を楽しめるようにアプリケーションを作成します 」 と言います 。 アプリケーションで必要なユーザーエクスペリエンスは、アクティビティの一部であり、アクティビティ、個人的な好み、趣味、または人々が本当に欲しいものを聞いたことに基づいて決定します。 最も成功しているTwitter（Seesmic）またはFoursquare（4番目と市長）の顧客は、APIの範囲外であると考えている人です。

この段階で一部の人々に見られるもう1つの一般的なアプローチは、最初から自分自身をサポートすることです。 たとえば、「 株価をチェックする 」アプリケーションを作成することを知って 、アイデアの段階を開始します。 したがって、別の株価アプリまたは別のRSSフィードリーダーになります。 アプリケーションの優れた独創的なアイデアを思いつくための鍵は、ユーザーエクスペリエンスの考え方です 。 したがって、「 私は証券市場で取引するためのアプリケーションを作成します 」 と考える代わりに、 「 証券市場で の取引の経験においてユーザーをどのように支援するかを検討します 」と考えます。 これははるかに広範であり、エクスペリエンスに貢献する方法を考え出すための選択肢が増えます。

Windows Phoneアプリケーションのアイデアを生成するこの種のアプローチを実装するためのブレーンストーミングの強力な手法は、「主題+動詞+オブジェクト」手法です。 覚えている、私たちは建築学校でこれをやった、それは楽しかった！ 昨年、ユーザーインターフェイス開発にも使用を開始しました。



サブジェクト+動詞+オブジェクト



Windows Phone Design Dayツアーでは、このグループブレーンストーミング手法を使用しました。 ユーザーインターフェイスに関連付けられた主題、動詞、オブジェクトなど、一連の単語を書く人々のグループで構成され、これらの単語の一部を選択して新しいスクリプトを作成します。 このエクササイズは創造性に火をつけます。なぜなら、自分で書いていない言葉に基づいたスクリプトの解決策を見つけることを余儀なくされるからです。グループの集合意識がなければ、自分で発明したことのないスクリプトです。

-マルチカラーのステッカーとマーカーを用意してください。 できれば、すべてのマーカーは同じ線の太さと同じ色（黒）を持つ必要があります。

-3枚のカラフルなステッカーを取り、1枚目は件名 、2枚目は物 、3枚目は動詞を書きます 。

-これらのステッカーのうち3つを大きな壁に貼り付けます。

-この時点で、Windows Phoneのアプリケーションの特定のテーマが必要です-ランチ、セーリング、開発などのユーザーエクスペリエンスについて考えることを忘れないでください（今回は特定のソリューションなし）。 この例では、 ランチエクスペリエンス（レストランでの食事）を使用します。

-参加者に、「ランチ」のプロセスと経験に関連する1 つの主題 、 1つの動詞 、 1つのオブジェクトを書くよう依頼します 。 個別のステッカーにそれぞれ1語​​ずつ入力し、元々作成した見出しステッカーの下に貼り付けます。

-これは、昼食の経験に関連する主題、動詞、およびオブジェクトの束で壁を満たします（または、あなたやあなたのチームが選択した、ランニング、水泳などのトピック...）





-次のステップは、グループを2人または3人（チームメンバーが少ない場合は1人）の小さなチームに分け、チームが好きな3つの単語（ 1つの主題、1つの動詞、1つのオブジェクト） を選択するように依頼します。 アイデアを生み出し、カスタムシナリオの想像力を盛り込む言葉を選択するように依頼します。 一部のチームは、どの単語を選択するかについて意見を交わします。 ここでの解決策は、チームの各メンバーに選択する単語を1つ与えることで、最終的には1つの主題、動詞、1つのオブジェクトを持ちます。 実際、各チームにランダムな単語のセットを割り当てることができます。これにより、チームの枠を超えて考え、ソリューション用の独自のカスタムアプリケーションスクリプトを作成できます。

-チームが3つの単語を選択すると、ランダムな主題、動詞、オブジェクトを選択することで魔法のように表示されるスクリプトを解決または実行する方法に関するアイデアの生成を開始できます。 例：

-ベジタリアンのガールフレンド+叫び声+ダブルチーズバーガー

-子供+遊び+食べ物

-料理人+料理人（料理）+トマト

-祖父（祖母）+申し出+領収書

-レディー・ガガ+日付+食器



-次に、参加者はこれらの3つの言葉を刺激するスクリプトを考え出す必要があります。たとえば、レディガガ+ミート+ディッシュ...レディガガ専用のWindows Phoneアプリケーションを開発しないことはわかっていますよね？ 私たちが大ファンでなければ、Lady Gagaに特化したアプリケーションを作成するのではなく、Lady Gagaを「Famous Persons」などのより広い概念に変換するとします。 だから、有名人は料理と出会う。 昼食または夕食のためにさまざまな場所を訪れる人々のために追跡アプリケーションを作成したらどうなるでしょうか？ カードや性格が存在する場合があり、ユーザーが有名人を見つけると、その人をマークします。 私はパパラッチのアプリケーションについて話しているのではなく、XまたはYを配置するために夕食または昼食に行くと、ユーザーがお気に入りの女優またはヒップホップラッパーに直面することを想像する機会をユーザーに与えるアプリケーションについて話します。FoursquareおよびYelpの機能を使用します。ユーザーエクスペリエンスをサポートする優れたAPIがいくつかあります。 これは単なるアイデアです。 参加者が文字通り言葉をとるのを恥ずかしがらずに、すべての意味に外挿することを奨励することが重要です。レディガガの場合は、有名な人や「悲鳴を上げる」動詞に拡張し、「完全に動揺する」または「問題を抱えている」ように拡張できます。 私のグループの一部の人々がこれらのステッカーに書いたこれらの言葉がなければ、レストランに登場する人格のこの考えは、私がそれを思い付くことは決してなかったようなものです。

ここに、私の注意をひいたもう1つの3ワードの例があります。 子供+遊び+食べ物です。 私はこのスクリプトを自分で思い付くことはありませんでした-私には子供がいませんが、これらの言葉を書いた3人の人々のおかげで、今私はそれに取り組むスクリプトを持っています。 子供たちはレストランで食事をします-これは本当の問題です。 このシナリオを解決するにはどうすればよいですか？

いくつかの方法は、 Game Assaultの本で説明されており、役に立つかもしれません。 これは不可能なシナリオと呼ばれるブレーンストーミング技術を使用する1つの例です。 これらはゲームであることを忘れないでください。したがって、これを真剣に考えれば、遠くに行かず、アプリケーションの作成方法や必要な内部サーバーアーキテクチャについて考える必要はありません。 ただ楽しんでください。 次の段階では、優先順位を設定し、技術的な能力を調査し、アイデアに対処するのに十分な時間があります。 チームでこのゲームをプレイしましょう。



「子供+遊び+食べ物」の不可能なシナリオ



母と父、おそらく2人か3人の子供と食事をします。これは問題ですが、100人の子供を持つ同じカップルは不可能なシナリオになります。 不可能ですよね？ （私は思う...）食べ物で遊んで、レストランに連れて行く100人の子供の親として何をしますか？ それは悪夢のように聞こえます-彼らの名前、好み、彼らが従うもの、それらのうちのどれが菜食主義者であるか、彼らの年齢層を知っているべきですか？ どれが最高の振る舞いをし、誰が最悪の振る舞いをしますか？ 等 100人の子供の思いやりのある親として、あなたはこれらのことを知っているでしょう、それは単なる不可能なシナリオです。 それを現実に戻しましょう：親が彼女/彼の100人の子供をレストランに連れて行くことはできませんが、100人の子供（異なる親から）を単純なレストランや博物館などの公共の場所に置くことは本当に可能です-マクドナルドまたはチャッキーチーズ（アメリカの子供向けレストランのチェーン- 約トランスレーション ）で見つけてください！ 次に、「学習体験」全体を100人の子供の親として翻訳し、それをレストランマネージャーまたはレストラン自体に使用します。 この100人の子供（またはその両親）がレストランで食事をして、この場所で混乱を起こさないようにするにはどうすればよいでしょうか？ アプリケーションが親の一種のミニソーシャルメディアであるレストランでのソーシャルアクティビティのスポンサーである場合、イベントで子供たちに会って一緒に遊ばせる、または単にこれに参加する他の子供たちに基づいて他の子供たちのサブカルチャーについて子供たちに知らせることができますイベント？







主語+動詞+目的語をチェックするもう1つの方法は、「月のレストランにいるとしたら？」のようなことを言う代替世界です。 子供が投げたミートボールはどうなりますか？ ミートボールがレストラン全体を飛んで、向こう側の不機嫌そうな老人に落ちるのを想像してみてください！ それは私にアイデアを与えてくれました...もしあなたのアプリケーションが本物のゲームになったら、子供たちが望む混乱をどこで作ることができますか？ このレストランの実際の食べ物とは？ 小さなビッグマックやマクナゲットを投げてみてはいかがですか。 Fruit Ninjaが既に存在する場合、メッシーフード（食べ物との混乱、レストランでの騒ぎ）を作ってみませんか。 Angry BirdsがFruit NinjaとCooking Mamaに出会ったらどうなるかを考えさせられます。

別の方法は、 他人の画像です。 ここで、あなたはあなたが誰かまたは食事体験の何かであると想像しなければなりません。 もちろん、あなたはウェイターとして行動し、彼/彼女の観点から忘れられない経験を発見することができます。 これは、他の方法では考えもしなかった心のシナリオを発見するのに役立ちます。 あなたとあなたのチームが完全に異なる視点から考えることができます。 今、あなたが人であると信じる代わりに、あなたはパスタの皿だと思います。 どのようにしてレストランにたどり着いたか、どのような材料から作ったのか、どのように調理され、提供され、食べられたのかを考えてください。 また、以前は考えもしなかったような観点から、忘れられない体験を提供します。 これは、パスタプレートのアプリケーションを作成するということですか？ 必ずしもそうではありません...しかし、パスタのライフサイクルを研究すると、カウンターの後ろ、キッチン、レストランの在庫または倉庫（倉庫）で何が起こっているかがわかります。アプリケーションの新しいアイデアが開かれると確信しています。
