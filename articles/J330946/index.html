「データベースで重大な事故が発生すると、それは常に幾分壮大に発生します」-イリヤ・コスモデミャンスキー

本日のインタビューは、PostgreSQLコンサルティングをリードするData Egretの CEOであり、 PG Day Russiaの共同設立者であるIlya Kosmodemyanskyによって行われました 。 15年間、Ilyaは開発者とDBAから、データベースサポートサービスを提供する彼の会社の責任者になりました。 今日、イリヤはデータ白g開発戦略の形成と実施に従事しており、ロシアおよび国際社会で同社のブランドを促進し、会議のスピーカーを選択する方向を監督しています。



PG Day'17 Russiaで、Ilyaはシステム管理者とDevOpsに集中的なPostgreSQLトレーニングコースを提供します 。



会話の中で、イリヤは、現代のデータベース市場におけるPostgreSQLの現在の位置についてのビジョンを共有し、ロシアの技術会議と西洋の技術会議の主な違いについて話し、彼が作成したマスタークラスの対象者について説明しました。







PG Day：設立した会社はPostgreSQLのサポートを提供しています。 なぜMS SQL ServerやORACLEではなくPostgreSQLなのですか？



イリヤ： 流行になる前に Postgresを始めたので、正直に言って、それは意識的な選択だったと言えます。 今では怠zyな人だけがPostgresのことを話しておらず、当時はそれは良いオープンソースのベースでしたが、それ以上のものではありませんでした。



比較するものがあります。Oracleでかなり多く、MySQLで少し働いたことがあります。 Oracleはクールなベースですが、SQL * Plusにログインするだけで、 多くのことが長い間行われているため、あまり便利ではないことを理解できます。 Oracleの技術的優位性により、Postgresは真剣な機能を備えた完全なデータベースであるという事実にもかかわらず、まさに使いやすさを引き付けました。



一方、それはオープンソースであり、商用製品のサポートとは異なる他の法律に従って少しビジネスを構築します。 あなたはコミュニティに依存しています。 これから誰かが不快になりますが、同時に商業的な製造業者への執着はありません。 突然、オラクルがすべての製品を認定パートナーのみがサポートする必要があると判断した場合、どうしますか？



商用データベース-これは突進中の豚であり、あなたの意志ではなく絶えず変化しているため 、これを制御および予測することはできません。 この意味でのオープンソースははるかに優れています。



オープンソースのサポートは非常に興味深い方向です。 現在、商用データベースの市場では、オペレーティングシステムの市場でかつて発生していたのとほぼ同じことが観察できます。 非常に高価な商用オペレーティングシステム（Novell Netware、HP UX、Solaris）があり、これらは非常に過大評価され、多くの費用がかかりました。 そして今、無料のデスクトップオペレーティングシステムが登場し、Linuxサーバーで競合他社を強く押し付けたとき、データベースでも同じことが起こります。 商用メーカーは、ライセンスの価格において、アプローチを変える必要があります。 私はPostgresが明日世界を支配するとは主張していません。「競争を長生きする」ためにこれが起こったとしても悲しいでしょう。 しかし、彼はこの意味で膨大な数の見通しを持ち、私たちの目の前でこの市場を変えています 。



PG Day：あなたの意見では、PostgreSQLは商用DBMSの技術的能力を置き換えることができますか？



イリヤ ：特にこの言葉遣いで-はい、もちろん、それは積極的に置き換えることができます。 しかし、 これはPostgreSQLがOracleよりも技術的に優れているという意味ではありません 。 すべてのプロジェクトに、ファイルシステムなしでベアメタルで作業するなど、Oracleに備わっているいくつかのスーパー機能が必要なわけではありません。 これらはオラクルに固有のものです。これらは超ハイテクですが、非常に高価なライセンスの購入が必要です。 この技術力はどこからでも必要です。 多くの場合、会社が慣れているという理由だけで、小さなプロジェクトのために非常に高価なOracleのライセンスを購入するという状況に遭遇しました。 「Oracleがあります。安価なライセンスに対処したくありません。優れたOracle EEに慣れています。」プロジェクトが問題なくPostgreSQLで動作できるという事実にもかかわらず。



PostgreSQLはそのようなタスクに十分であり、完全に対応し、操作が簡単で、 ライセンスコストを必要としません 。 これは、Postgresが商用DBMSを実際に置き換えることができる非常に広範な状況です。 同時に、Oracleが技術的に適しているタスクはますます少なくなっています。 ほとんどの場合、ライセンス価格は、これらのプロジェクトに対するOracleの技術的優位性を正当化するものではありません。



PGデイ：CEOは、多くの場合、日常の技術作業から脱却します。 会社の技術的な問題をどの程度解決できますか？



イリヤ：そのようなフレーズがあります：私はスタートアップを成功させ、そこでCEOを雇い、そこでプログラマーとして働きたいです。 残念ながら、このようなスキームは実際には機能しません。 このすべてを前進させるには、技術的なタスクをバックグラウンドに追いやる必要があります。 もう1つの質問は、主に技術サポートとコンサルティングに従事していることです。組織全体が技術的な問題に従属しています。 データベースインシデントの分解に参加せずに、DBAが関与する技術的な詳細やタスクに進むことなく、PostgreSQLデータベースに対する最高の技術サポートを構築することはできません。 私の仕事は、これを行う人々の相互作用を整理することです 。 データベース管理には直接関与していませんが、何が起こっているのか、どの方向に進む必要があるのか​​などを理解するために、これらすべての問題を掘り下げなければなりません。



PG Day：あなたのTwitterプロフィールの碑文をよく説明している最も壮大な事例を教えてください-「DBAは自転車に乗るのと同じくらい簡単で、ただ自転車に乗っているだけで、燃えている、すべてが燃えている、地獄にいる」



Ilya： DBAが多くなることはまずありません。彼らは練習中に最も壮大なエピソードを喜んで語ります。 何でも起こりました。 ある種の重大な事故がデータベースで発生すると、どこでもストローを配置しようとしますが、 それは常にやや壮大に発生します。



彼らは午前2時に電話をかけ、すべてが落ちた、何も機能していない、データベースが閉じられた、彼女がトランザクションIDを使い果たし、何かが設定されていなかったと言います。 同時に、この時点で彼女を監視することになっていたすべての管理者は、午前2時に庭のボードで酔っており、発信者は電話のバッテリーを使い果たしており、何かをする必要があります。 このようなケースは定期的に発生するため、 自転車に関するフレーズは私たちのモットーです。



PG Day：よく会議に行きます。 ロシア、ヨーロッパ、アメリカのコミュニティの違いは何ですか？ ロシアおよび海外での会議で、PostgreSQLコミュニティの存在をどの程度強く感じていますか？



イリヤ：これらは2つの大きな個別の質問です。 海外の多くの会議はコミュニティ主導型であり 、愛好家は直接利益ではなくコミュニティの発展に焦点を当てています（ただし、そうした目標をまったく追求していないと言うのは間違っています）。



あなたの目を引く最初の違いはロシアの会議の主催者がお金を稼ぐことを期待するということです。 イベントへのチケットはより高価であり、すべてがショービジネスのタッチで発生します。 ロシアとは異なり、西側の人々はスポンサーに興味を持たせる方法についてあまり心配していません。 多くのロシアのカンファレンスは、スポンサーが退屈するため、非常に興味深いプログラムを買う余裕がありません。聴衆は彼らにではなく、興味深いレポートに行くからです。 同時に、ロシアの会議は非常によく組織されています。すべてが時計仕掛けのようにデバッグされ、多くのお金が費やされ、専門家が準備しています。 西洋の会議では、多くの場合、フォームよりもコンテンツに焦点が当てられます。



2番目の違い：私たちの市場は客観的な理由で世界市場よりもはるかに小さいです。 ロシアの会議にのみ参加する場合、自分を最も賢い人と考えるのは非常に簡単です。 したがって、 国際会議に参加することをお勧めします。何人の人があなたよりはるかに賢く、はるかに速く考え、学ぶべきことがあるかをすぐに理解できます。 とても助かりました。



そしてもう一つ。 北米では、会議に参加してリラックスしたり、友人とくつろいだりする方法として誰も見ていません。 ほとんどの人、リスナー、スピーカーは、 これを真剣な学習と継続教育と見なしています。 イベントに到着すると、彼らは最高の結果を出します。 午前9時にアメリカの会議でレポートを開始することは許可されています-学生はオフィスルーチンに隠れず、そこで働きに来ます。



PG Day：PG Day Russiaに期待することは何ですか？ 聴衆のための要件-最低限の知識が必要であり、読むことをお勧めしますか？



Ilya：私たちは、Posgresだけでなく、データベース全般にも特化した良いイベントを開催できると期待しています。 私は、データベースを扱うすべての技術者が参加することが面白いといつも心配しています。 プログラムが高レベルであり、興味深い技術レポートで満たされていること。



良い外国人スピーカーだけを連れてショーを見せることは不可能だという考えを持っています。 ロシアのコミュニティを内側から成長させる必要があります。 私たちは、世界コミュニティの有名なスピーカーに電話をかけて、基準を設定します。 私たちは、多くの興味深いことを話すことができる信頼できるロシアの同志を招待し、 以前は一般の人と話したことのない新しいスピーカーを見つけようとします。 最も簡単なアプローチではありませんが、それにもかかわらず、近年では成功しています。



今回はさらに深刻なアプリケーションがあります 。メインのPostgreSQLカーネルに加えて、いくつかの追加スレッドを計画しています。 MySQLやその他のオープンデータベースでストリームを配信するPerconaは、講演者と聴衆を惹きつけるのに役立ちます。 システム管理に関するスレッドもあります。 Postgresの管理者と開発者はこれらのトピックに非常に没頭しているので、境界線のことも聞くことができます。



そして、 2つの異常なフローを整理します 。 現在、多くの心配があります：PostgresはOracleより優れていますか？ PostgresがついにOracleを倒すのはいつですか？ 理解するために、 商用データベースを介してフローを作成し 、「オラクル主義者」と「スグレスト主義者」の両方が他のテクノロジーを見る機会を提供することにしました。 そして、私たちにはコンピューターサイエンスの流れがあります 。 これには、データベースの運用と、この科学で現在起こっている傾向との境界が含まれます。 多くの期待がありますが、最も重要なことは、主催者としての私の関心は、誰もが興味を持って元気になることです。



観客に実際に要件はありません。私たちは誰もが自分にとって興味深いトピックを見つけることを期待しています。 私たちのレポートは多種多様です：すべてをすでに知っている専門家だけではありません。 自分のレベルを平均にもう少し近づける人や初心者向けです。 ワークショップをご覧ください。 たとえば、 MS SQL ServerおよびOracleパフォーマンス分析 に関する非常に興味深い入門コースがあります 。 ポスグレスによると、 私のワークショップは入門であり、 アレクセイ・レソフスキーのワークショップは非常に集中的で、非常に真剣な聴衆を期待しています。 誰もが興味とレベルを見つけることができます。



PG Day：システム管理者とDevOpsのためのPostgreSQL入門コースと言えます。 常に多くのリスナーを魅了します。 このワークショップはなぜそんなに人気があるのですか？



イリヤ：これには驚くべきことは何もありません。 PostgreSQLは人気を集めており、多くの場合、膨大なドキュメントをまとめたコースを聴きやすくなっています 。 そうしないと、より痛みを伴う方法で読んで経験を積むのに長い時間がかかります。



個人的には、ドキュメントのデバイスがあまり好きではありません。 リファレンスマニュアルとして設計されており、情報はセクションに構造化されています。 人間の思考はそのようには機能しません。 人々は、より広範な問題を解決する方法についての回答を求めています。PostgreSQLをインストールして、それを使い始めてください。 ドキュメントはこれを目的としておらず、完全に読む必要があります。 wikiにはマニュアルがありますが、多くの場合、それらは関連していません。 したがって、このようなコースは、実際の経験に基づいており、管理者の実際のタスクに近いため、大きな需要があります。



私は常に、社内のすべてのDBAの経験に基づいて教えようとしています。 Postgresの新しいバージョンのリリースで変化する、クライアントとのいくつかのインシデントの分析で何が起こるかを見ていきます。 いくつかの非常にクールな専門家の実際の経験のこのような一般化が判明しました。 人々はそれを気に入って利益を得るので、勉強に来ます。



PG Day：ワークショップと講義は常に非常に詳細です。 技術情報を最新の状態に保つ方法を教えてください。



イリヤ ：まず第一に、私は常にcommitfestと開発プロセスに従います 。 機能リストが多少落ち着いたら、どのような変更が発生し、どのように機能するかを確認します。 これは単純なプロセスであり、もちろん多くの詳細を追加しません。



私は毎日「現場」で働くDBAの同僚と相談しています。 私は社内のナレッジシステムとカスタマーサポートで何が起こっているのかをフォローしています。 すでにこのような場合に慣れているDBAは、スライドの素材を知っており、「だから、このスライドを更新する必要があります。すべてが既に変更されています。」と言います。 これは骨の折れる作業であり、多くの準備作業であり、私だけでは習得できなかったでしょう。



私の役割は技術的な詳細の検索ではなく、方法論的な形式での体系化とプレゼンテーションであり、人々が素材を理解しやすいようにします。



PG Day：レポートはオンラインで入手できます。 プレゼンテーションをダウンロードするだけなので、マスタークラスに参加する必要がありますか？



Ilya ：プレゼンテーションが必要です-人々はそれらを見て、そこからいくつかの情報を抽出するのが好きです。 率直に言って、私は自給自足のプレゼンテーションをどのように、そしてどのようにしたいかを本当に知りません。 なぜレポートが必要なのか理解できませんか？ たとえば、スライドには何らかの種類の図があり、PostgreSQLまたはLinuxで何かがどのように機能するのか、この図には説明が必要です。 これらの説明をいくつかの隣接するスライドに置いたとしても、何が起こっているのか明確ではありません。 多くのテキストが表示され、プレゼンテーションが過負荷になります。 したがって、私は通常、レポートを声で、ポインタを使って、または他の方法で見せて話します。



トレーニングに関しては、私は常にセミナー形式で働くことのサポーターです 。 あなたはあなたの問題を （このマスタークラスのトピックにある場合） 来て、質問をし、それらに対する答えを得ることができます。 これは、より大きな収益とより大きな効果に貢献します。



PG Day：最初にマスタークラスに参加する専門家は誰ですか？ さまざまなレベルのトレーニングを受けた学生は、自分に役立つ何かを学ぶことができますか？



Ilya ：会議でマスタークラスを行うときに特定の聴衆を当てにするのは非常に困難です。 まず第一に、これらは初心者と実務家の両方である管理者とDBAです。 おそらく、経験豊富な専門家にとっては、私は通常、単純な素材から複雑な素材に移行しようとするので、最初はいくつかのことが明白に思えます。 しかし、 繰り返しは知識を体系化するのに役立つので、私は常に初歩的な素材でさえ、もう一度聞くことを勧めます 。



ほんの2週間前、私はPostgreSQLのパフォーマンスレポートを聞くように頼まれました。 私は聞いていて、私が長い間忘れていたいくつかの瞬間を思い出しました。 Linuxでメモリから異なるページをスワップする機能について説明しましたが、その瞬間、私は注意を払わず、人々がそれを常に理解していないことに気付きました。 したがって、 私自身は頭の中で秩序を回復するのを聞くのが大好きで、他の人に助言します。



PostgreSQLの専門的な管理に関与しておらず、クールなDBAになる予定がない人には、マスタークラスに参加することを特にお勧めします。 この資料は、DevOpsに関与してアプリケーションを開発している人々にとって特に役立ちます。 スタッフに資格のあるDBAを配置する機会がないため、これらすべてを自分で処理する必要があります。 何をすべきか、どこを見るべきかについての完全な指示を与えようとします。 この知識を使用して、学生はPostgreSQLを効果的に使用できるようになります。



PG Day：ありがとう、イリヤ！
