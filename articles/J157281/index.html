OpenCV 2.4.3

こんにちは、Habr！



11月2日に、無料のコンピュータービジョンライブラリであるOpenCVの新しいバージョンがリリースされたことをお知らせします。 この投稿はItseezで働いているライブラリ開発者によって書かれました 。 以前のバージョン以降の主なイノベーションをリストしますが、それらの多くが注目に値します：新しいアルゴリズム、既存のアルゴリズムの高速化、新しいプラットフォームのサポート、開発プロセスの更新、統合とテスト、および更新されたドキュメント。 リリースの準備中に、トラッカー上の210を超えるタスク（ hackathon 、 リリース候補 、 リリース ）が閉じられました：パッチ、バグ修正、既存の機能の拡張。



しかし、まず最初に。





Webリソース



OpenCVはついにインターネット上のopencv.orgに独自のホームを手に入れました。 このサイトでは、ライブラリの存続期間中の最新のイベントを検索し、それに関連する他のすべてのWebリソースへのリンクを見つけることができます。 ニュースを自動的に購読するには、公開RSSフィードまたはtwitterを使用できます。



ライブラリ開発者向けのより専門的な情報は、 ChiliProjectに基づいて作成されたcode.opencv.org Webサイトに集中しています。 ここでは、ライブラリの次のリリースのスケジュールに関する情報、または毎週のミーティング中に開発者が下した決定に関する情報をすべて見つけることができます。 情報はWikiモードで作成および編集されるため、ほとんどの場合最も関連性が高くなります。 また、メインのgit OpenCVリポジトリはここにあります（ただし、githubでミラーを使用することをお勧めします。詳細は以下を参照してください）およびバグトラッカー。 したがって、ライブラリの使用で問題が発生した場合は、このリソースを通じて遠慮なくお知らせください。



また、最近では、ユーザーサポートを目的とした別のサイトを開設しました。 これは、OpenCVとコンピュータービジョン全般の問題を解決するためのStackOverflowスタイルのフォーラムです。 短い歴史にもかかわらず、このリソースについてはすでに1,000以上の質問が議論されています。



最新のリリースバージョンに必要なすべての情報が含まれているライブラリのオンラインドキュメントがあるサイトを忘れないでください。

Opencl



OpenCLテクノロジーを使用してコンピュータービジョンアルゴリズムを実装する実験的なoclモジュールがライブラリに追加されました。 現在、次のアルゴリズムが利用可能であり、これまでのところGPUでのみ実行されています。

  • 算術演算;
  • さまざまなフィルター（Sobel、Scharr、GaussianBlur、boxFilter、bilateralFilter）;
  • さまざまなタイプの幾何学的変換（サイズ変更、再マッピングなど）。
  • カスケード分類器（最初に顔を検出するため）;
  • オプティカルフローアルゴリズム（ピラミッドルーカスカナデ法）。


モジュールのドキュメントはすでに利用可能です。詳細については、お問い合わせください。



CUDA / GPU



今回は、GPUモジュールに多くの便利なアルゴリズムが追加されました。

  • 画像ノイズ低減：

    • バイラテラルフィルター;
    • 非ローカル手段ブルートフォース-実装は高速ではありませんが、元の記事に完全に従います。
    • 高速非局所平均-この実装では、ガウスの重み付けが省略されたため、ランニングサムアプローチを使用して、アルゴリズムを数十倍高速化できました。
  • バックグラウンド選択アルゴリズム： ViBe 、GMG、FGD、MOG（混合ガウス分布）。
  • 長い間CPUに存在していたLBP機能（ローカルバイナリパターン）のカスケードのサポートがカスケード分類に追加されました。
  • ハフ線、ハフ円、および一般化されたハフ変換。
  • 接続コンポーネントのラベリング/一般化されたフローティング範囲FloodFill。


また、このリリースでは、いわゆる「デバイスレイヤー」（CUDAで記述され、nvccコンパイラー向けのテンプレート関数とクラスを含むヘッダーファイルのセット）を公開しました。 これは、CUDAを自分で作成する開発者に役立ちます。 デバイスレイヤーの機能とコードを組み合わせることで、ユーザーはCUDAに新しい画像処理アルゴリズムを実装しやすくなりました。



Android



NVIDIAによってサポートされているOpenCV for Androidチームは、ライブラリを大幅に更新し、ライブラリをより便利にし、より完全なドキュメントを提供しています。 さらに、Tegra 3プラットフォームでは、多くの最適化を使用して、エンドアプリケーションのパフォーマンスを大幅に改善しています。

  • OpenCV for Androidに関する多くの資料が公開されています。 それらの中には：

    • 公式プロジェクトページhttp://opencv.org/androidが作成されました。
    • 欧州の主要コンピュータービジョン会議ECCV2012で 報告します。
    • 初心者開発者向けの大幅に改善されたAndroidチュートリアル 。
    • Java APIライブラリのJavadocドキュメントを更新しました。
  • 新しいバージョンでは、 OpenCVマネージャーはエンドユーザーからのクリックをはるかに少なくし、さらにOpenCVに依存するアプリケーションを再起動する必要がありません。 OpenCV Managerには動的ライブラリが付属しているため、Google Playから追加のバイナリパッケージをインストールする必要がなくなりました。 互換性の理由から、古いパッケージはバージョン2.5までGoogle Playに残りますが、Android開発者はOpenCV 2.4.3に切り替えることをお勧めします。
  • Androidの最新バージョンであるJelly Beanが、サポート対象のリストに追加されました。 ネイティブOSレベルで動作するOpenCVカメラは、より多くのデバイスをサポートし始めています。
  • Javaアプリケーション用の新しいフレームワーク。 新しいアーキテクチャと更新されたアプリケーションの例をよく理解することをお勧めします。 新しいフレームワークは、Javaおよびネイティブカメラ、デバイスの画面で画像を視覚化するコード、OpenCVマネージャーの簡単な初期化、適切なイベント処理（一時停止、続行、回転など）を含むコンピュータービジョンアプリケーションの単一の基盤を提供します。
  • NVIDIA Tegra 3プラットフォームのパフォーマンスを最適化するために多くの作業が行われていますが、Google PlayからOpenCV Tegra Demoアプリケーションをダウンロードすることで、これを自分で確認できます。 最適化の完全なリストは、 リンク 17のドキュメントにあります。Tegra用のOpenCVは、NVIDIA Tegra Android Development Packに含まれているため、 Androidの開発を始めたばかりの場合は、必要なすべてのパッケージの長いインストールから解放されるため、使用することをお勧めします。

  • Google Summer of Code 2012のフレームワークでのEduard Feichoの作業のおかげで、ビデオカメラを操作するためのコードが大幅に変更されました。 現在、これはhighguiモジュールの一部であり、CvVideoCameraクラスとして利用可能です。

  • さらに、AppleモバイルデバイスにOpenCVを使用するための基礎に関するトレーニング資料を追加し、 ECCV2012のフレームワークでiOSのすべてのイノベーションに関する詳細なトレーニングを実施しました 。

Buildbot



OpenCVは、 Buildbotを継続的な統合システムとして使用します 。 Windows、Linux、Mac OS、Android、iOSの32および64ビットプラットフォームで、TBB、SSE、GPUの最適化を使用して、毎日50を超えるさまざまな構成が収集されます。 各構成で約5,000の異なるテストが開始されます。 最も一般的な15の構成が1日に2回収集されます。 Tarball、Windows用インストーラー、iOS用フレームワークが自動的に収集され、コードカバレッジが計算され、バイナリ互換性がチェックされます。 ご覧のとおり、ライブラリの高い安定性を確保するために素晴らしい仕事をしました:)



リポジトリに該当する変更があると、いくつかのクイックビルドが実行されます。 結果が成功した場合、コードがコンパイルされ、他の構成のテストの大部分に合格するというかなり信頼できる仮定が行われます。



OpenCV 2.4.2のリリース後、約束どおり、バージョン管理システムを変更しました。 リポジトリがgitを実行しています。 このステップの論理的な結果は、github上にミラーを作成することでした。これにより、さらに多くのサードパーティ開発者をライブラリの開発に引き付けることができました。 Githubは便利なプルリクエストメカニズムを提供するため、サービスのすべてのユーザーがOpenCVの共著者になれます。 各プルリクエストは自動アセンブリを通過し、成功すると、レビューのためにOpenCVチームから開発者に届きます。 彼はコードをチェックし、著者に必要な質問をし、提案された変更を行うか拒否するかを決定します。



このリリースのリリースの数週間前に、すべての常勤の開発者も、サードパーティの開発者の観点からそのようなプラクティスの長所と短所を評価するために、プルリクエストを介して変更を行いました。 投稿の1つで、OpenCVプロジェクトへの継続的な統合について詳しく説明する予定です。 コメントでこのトピックに関する質問をすることができます。可能な限り詳細に回答するよう努めます。



計画



番号2.5の下の次のリリースは2013年春に予定されています。 2.4.3と比較して、大幅な変更が発生し、ライブラリのアーキテクチャとAPIを最新化するためにバイナリ互換性が損なわれます。

計画されている作業の中で、次の点を強調しています。

  • 機能の「マイクロモジュール」への分解、カスタムマイクロモジュールのサポート。
  • 廃止されたコードの削除（サポートされていない別のモジュールへの転送）。
  • 並列化と最適化に関する作業の継続。
  • Kinect、3Dリカバリ、計算写真、追跡などと連携するための新しいアルゴリズム。
  • OpenCVユースケースのドキュメントとナビゲーションの改善。
