すべての伝送ダウンロードの完了時にシャットダウンする



私は、そのシンプルさと使いやすさからトランスミッションが大好きです。 しかし、私が追加するのは、 すべてのダウンロードが完了したときにコンピューターを自動的にオフにする機能です。 私はUbuntuでこの機能を自分用に作成しましたが、それに興味がある人は別のOSに自由にやり直せると思います。



この問題は、Transmissionから親切に提供されたRPC-JSONサービスによって解決されました。 この機能を有効にするには、[送信]で、[設定]の[Webインターフェイス]タブに移動して、[Webインターフェイスを有効にする]チェックボックスをオンにします。 見知らぬ人からのWebパネルへのアクセスを制限する場合は、パスワードとユーザー名を設定できます。



最初のステップは、python transmissionrpcのモジュールをインストールすることです。 わかりました：

hg clone www.bitbucket.org/blueluna/transmissionrpc





注： hgプログラムはmercurialパッケージに含まれています。 インストールされていない場合は、インストールする必要があります。

sudo apt-get install mercurial





その後、モジュールのソースコードがローカルコンピューターにある場合は、transmissionrpcディレクトリに移動してインストールする必要があります。

cd transmissionrpc

sudo python setup.py install





モジュールがインストールされたら、スクリプト自体を作成できます。 しかし、最終的な目標はコンピューターの電源を切ることであるため、既定では管理者権限なしでは実行できないため、さらにいくつかのジェスチャーを行う必要があります。

  • スクリプト/ usr / bin / shutdownを作成して作成します。

    #!/bin/sh

    sudo /sbin/shutdown $*







    スクリプトのテキストからわかるように、通常の/ sbin / shutdownを使用します。これは、/ usr / bin / shutdownに渡されるすべてのコマンドラインパラメーターを受け取ります。 また、/ sbin / shutdownはsudoコマンドで実行されます。これにより、システム管理者に代わって他のコマンドを実行できます。
  • このスクリプトをシャットダウングループに割り当てます。

    chgrp shutdown /usr/bin/shutdown



  • シャットダウングループのみを開始する権限を設定します。

    chmod g+x /usr/bin/shutdown



  • sudoがパスワードを要求しないように、/ etc / sudoersファイルに次の行を追加します。

    %shutdown ALL= NOPASSWD: /sbin/shutdown





    これは、シャットダウングループ内のすべてのユーザーが、パスワードを入力せずに管理特権で/ sbin / shutdownスクリプトを実行できることを意味します。
  • シャットダウングループを作成する

    sudo groupaddシャットダウン

    groupadd / usermodを使用するか、/ etc / groupファイルを編集して、自分自身を追加します。 私は後者の方法を使用しました、これのために/ etc / groupファイルで次のような行を見つける必要があります

    shutdown:x:1002:





    このグループに自分を追加するには、このエントリを次のように変更します。

    shutdown:x:1002:user1





    ここで、user1はユーザーのユーザー名です。 このグループにさらに複数のユーザーを追加する必要がある場合は、同じ行にコンマで区切って（スペースなしで）リストします。 番号1001（グループID）は異なる場合があります。

    最終的なフォームでは、次のような行を取得しました。

    shutdown:x:1002:skymanphp





これで、パスワードを要求せずにコンピューターの電源をオフにするコマンドを使用する準備が整いました。

おそらく、この決定は誰かに少し歪んでいることが判明するので、あなたの提案をするようお願いします。
