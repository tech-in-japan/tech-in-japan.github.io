Windows、Mac OS、iPhone、iPad、Android、Ubuntu、またはDebian（Linux OS）からRDPを介してサーバーに接続する方法

UltraVDS上のWindowsサーバーによってデフォルトで作成されたすべてのサーバーは、標準のRDP（リモートデスクトッププロトコル）プロトコルを使用した接続にすぐに使用できます。これは、ロシア語版のWindowsの通常の「リモートデスクトップ接続」です。



この記事では、さまざまなデバイス（オペレーティングシステム）からUltraVDS上に新しく作成された仮想サーバーに接続する方法について説明します。 RDP接続が成功した後に他の接続方法（RAdmin、TeamViewerなど）が必要な場合は、他のソフトウェアと同じ方法で構成できます。



この記事では、最も人気のある各システムから接続する方法について詳細に説明しているため、ナビゲーション用のホットリンクが表示されます。







デスクトップバージョンのWindows（XP、7、8、8.1、10）から仮想サーバーに接続する



標準のWindowsオペレーティングシステムを使用する場合は、次のパスを開いてサーバーに接続する必要があります。 

 ->  ->  ->
または、 WinとRのキーの組み合わせを押して、開いたウィンドウでmstscと入力します







表示されるウィンドウで、作成されたVDSサーバーの IPアドレスを指定する必要があります。 複数のIPアドレスを持つサーバーを注文した場合、それらのいずれかを使用して接続できます。

サーバーのIPアドレスは、アカウントの[サーバー]セクションのサーバーの近くに表示されます。







サーバーのIPアドレスを入力した後、「接続」ボタンをクリックすると、認証フィールドのあるウィンドウが表示されます。 ここでは、新しいユーザーとしてログアウトする必要があります。







新しいサーバーを作成するたびに、システムは管理者ユーザーのランダムパスワードを自動的に生成します。 このパスワードは、サーバーの近くのアカウントにも表示されます。







Windowsの一部のエディションでは、奇妙なことがあります：接続する前にサーバーからパスワードを許可フィールドにコピーアンドペーストすると、接続しようとしても何も起こりません。RDPクライアントは、資格情報が間違って入力されたかのように許可ウィンドウを再び表示します。 この場合、 パスワードを手動で入力する必要があります （英語のキーボードレイアウトでは、大文字が大きく、小文字が小さい）。



リモートデスクトップへの接続は非常に便利です。クリップボードのテキストと同じように、ファイルをコピーして貼り付けるだけで、コンピューターからサーバーにファイルを転送できます。 また、コンピューターから、サーバーに含まれているUSBデバイスとネットワークプリンター/スキャナーをサーバーに転送することもできます。 これを行うには、サーバーに初めて接続するときに、左下隅の「設定を表示」を選択します。







開いたウィンドウで、[ローカルリソース]タブに移動し、必要なパラメーターを選択します。







サーバーに接続すると、 信頼できないセキュリティ証明書に関する情報を確認できます 。 これは、サーバーが自動的に生成したSSL証明書で送信データを暗号化するためです。 この通知はセキュリティの問題を示すものではなく、許可されたセンターによって発行されていない証明書を使用して接続が暗号化されていることを警告するだけです。







[このコンピューターへの接続を再度要求しない]の横のボックスをオンにし、[はい]をクリックします。







Mac OSを使用してVDSサーバーに接続する



Mac OSの場合、Microsoftは公式のRDPクライアントをリリースし、Windowsのどのバージョンに接続しても安定して動作します。

次のiTunesからダウンロードできます： https : //itunes.apple.com/en/app/microsoft-remote-desktop-10/id1295203466?mt=12



プログラムのインターフェイスはシンプルで直感的です。 開始する前に、接続設定を構成する必要があります。 これを行うには、新しいものを作成します。





接続設定ウィンドウで、「UltraVDS上のサーバー」、作成したサーバーのIPアドレス、承認用のデータ（自動モードでサーバーに割り当てられた管理者ログインとパスワード）などの任意の名前を指定します 。これらのデータは個人アカウントに表示されます 。



設定ウィンドウを終了すると、すべてが自動的に保存され、接続のリストに新しい接続が作成されます：







それをダブルクリックすると、サーバーに接続します。

サーバーに接続すると、信頼できないセキュリティ証明書に関する情報を見ることができます。 そのような通知がなぜ起こるかとそれが意味することは上で説明されます 。











サーバーの自己署名証明書を信頼できる証明書に追加するか、または単に一度受け入れるだけで、 仮想サーバーの Windowsデスクトップが表示されます 。

サーバーに接続する前に、Apple StoreからMicrosoftリモートデスクトップアプリケーションをダウンロードする必要があります（これはMicrosoftの公式RDPクライアントです）。

https://itunes.apple.com/en/app/microsoft-remote-desktop-10/id1295203466?mt=12







インストール後にアプリケーションを起動し、[新しい接続の追加]をクリックします。







新しい接続を作成するためのウィンドウで、仮想サーバーのIPアドレスと、個人アカウントに表示される認証データを指定する必要があります（ 上記の場所についての詳細 ）。







サーバーに接続すると、信頼できないセキュリティ証明書に関する情報を見ることができます。 これの理由は上で説明されます 。

[今後このPCを要求しない]を選択し、[同意する]をクリックします。





認証データとサーバーIPアドレスがエラーなしで入力された場合、サーバーに正常に接続します。











Androidのスマートフォンまたはタブレットから仮想サーバーに接続する



まず、Google Playからダウンロードして、Microsoftリモートデスクトッププログラムをインストールする必要があります（これはMicrosoftの公式RDPクライアントです）。

https://play.google.com/store/apps/details?id=com.microsoft.rdc.android&hl=en







インストール後にアプリケーションを起動し、[新しい接続を追加]をクリックします







新しい接続を作成するためのウィンドウで、作成したVDSサーバーのIPアドレスと認証データを指定する必要があります（ 取得する場所についてはもう少し詳しく説明します ）。







サーバーに接続すると、信頼できないセキュリティ証明書に関する情報を見ることができます。 これの理由は上で説明されます 。

認証データとサーバーIPアドレスがエラーなしで入力された場合、サーバーに正常に接続します。











UbuntuからRDPを介してサーバーに接続する



RDPはMicrosoft独自のプロトコルですが、Linuxオペレーティングシステム用のRDPクライアントをリリースしません。

ただし、いまだにさまざまなメーカーのさまざまな作業バージョンがあります。

Remminaクライアントの使用をお勧めします。



Ubuntuユーザーには、RemminaやRDPなど、さまざまなアプリケーションパッケージを備えた特別なリポジトリがあります。

ターミナルで順番に入力される3つの簡単なコマンドでインストールが実行されます。



Remminaパッケージをインストールするには 
 sudo apt-add-repository ppa:remmina-ppa-team/remmina-next


更新プログラムをインストールする 
 sudo apt-get update


RDPプロトコルプラグインをインストールする 
 sudo apt-get install remmina remmina-plugin-rdp libfreerdp-plugins-standard


以前にRemminaの既存のバージョンをインストールまたは起動した場合は、再起動する必要があります。 これを行うには、コンピューターを再起動するか、同じ端末で次のコマンドを実行します。 

 sudo killall remmina
プロセスが開始されなかった場合、エラーメッセージが表示されます。プロセスが見つかりませんでした。



検索メニューを開き、新しくインストールされたRemminaパッケージを見つけます







新しい接続の追加をクリックし、サーバーに接続して認証するためのデータをフィールドに入力します（サーバーに接続するためのデータは上記で説明されています ）。







保存後、サーバーは接続リストで常に利用可能になり、すばやくアクセスできます。 接続するには、サーバーの行をダブルクリックします。







サーバーに初めて接続するとき、信頼できないセキュリティ証明書に関する情報を見ることができます。 これの理由は上で説明されます 。 [OK]をクリックすると、サーバーのデスクトップが表示されます。















Debianからのリモートデスクトップ接続（RDP）



RDP（リモートデスクトップ接続）はMicrosoft独自のプロトコルであり、Linuxオペレーティングシステム用のRDPクライアントをリリースしません。

しかし、まださまざまな開発会社からのさまざまな作業バージョンがあります。

RDPクライアントRemminaの使用をお勧めします



RemminaアプリケーションとそのRDPプラグインをインストールするには、パッケージインストールマネージャーを開く必要があります。







ここでは、検索バーに「Remmina」と入力し、インストールするすべての結果を選択します。







インストールには文字通り3〜4秒かかり、その後すぐにアプリケーションを使用できます。

メインメニューで見つけて実行します：







始める前に、将来の便宜のために新しい接続を作成する必要があります。







表示されるウィンドウで、正しいRDP接続パラメーターと承認用のデータ（ UltraVDSアカウントで指定 ）を設定する必要があります。







保存後、サーバーは接続リストで常に利用可能になり、すばやくアクセスできます。 接続するには、サーバーの行をダブルクリックするだけです。







サーバーに初めて接続するとき、信頼できないセキュリティ証明書に関する情報を見ることができます。 これの理由は上で説明されます 。 [OK]または[同意する]をクリックすると、サーバーのデスクトップが表示されます。











LinuxファミリのOSに接続しようとしたときにすぐにエラーが発生した場合はどうすればよいですか？



デフォルトでは、UltraVDSで作成されたすべてのサーバーで、ネットワークレベル認証を実行しているコンピューター（クライアント）からのRDP接続のみが許可されます。 一部のLinux RDPクライアントは、この認証をサポートしていない場合があります。 この場合、RDPを介してサーバーに接続する前に、VDSサーバー自体でこの要件をキャンセルする必要があります。



UltraVDSは、接続できない場合に緊急時にサーバーにアクセスする機能を提供します。これを使用して、必須認証の要件を無効にします。 緊急モードを開くには、[マイサーバー]ページの個人用ページに移動し、必要なサーバーのスクリーンショットをクリックします







開いたウィンドウで、ユーザーを許可する機能をアクティブにする必要があります。 これは、キーの組み合わせCtr + Alt + Delを押すことによって行われますが、そのような組み合わせはWeb経由で送信できないため、ウィンドウのトップパネルに特別なボタンが配置されました。







次に、 管理者パスワードを入力してEnterを押します 。







標準のWindowsデスクトップが表示されます。 ここで、[スタート]ボタン（スタート）をクリックし、そこに[マイコンピュータ]（このPC）を見つけて右クリックします。







メニューから「プロパティ」を選択して、システム情報ウィンドウを開きます







左側のメニューでは、リモートデスクトップの設定を制御するためのボタンを見つける必要があります（リモート設定）。







最後のステップとして、 「...からの接続のみを許可する」パラメータのチェックを外し 、「OK」をクリックします。
