締め切りをスキップする理由と対処方法





「締め切りが大好きです。 私は特に彼らが私を追い越すwhiが好きです。」 - ダグラス・アダムス 。



将来のプロジェクトに関しては、計画に取りつかれています。 しかし、タスクに真剣に取り組み、計画を行動に移す必要があるときは、設定された期限の境界を超えて、最後の最後まで、またはさらに悪化するまで、それらをさらに押し進めます。



それは皆に起こります。 しかし、一部の人々は他の誰よりも締め切りに苦労しています。 これは部分的にしか私たちのせいではないことを考慮する価値があります。人間の脳は隠れた素因に満ちており、それが私たちをこのtrapに導きます。



期限をスキップする理由



シドニーの有名なオペラハウスの建設は1963年に完了する予定でした。プロジェクトの計画予算は700万ドルでした。 終了したことに驚くことはないでしょう。10年後の1973年に建物の小さなバージョンが完成し、建設価格が1億200万ドルに上昇しました。



開発者は計画の誤りの犠牲者です。特定の問題を解決するのにどれだけの時間がかかるかを過小評価するという無意識の傾向に導かれています。 外部要因が期限を逃すことにつながり、効果的に計画を立てることができないと考えるようになります。

私たちが最初に作業を計画するとき、通常、プロジェクトの完了期限はかなり遠く、抽象的であるように見えるため、計算は最初は欺かれます。 締め切りが近づいてくると、「具体的」になり、最終的にプロジェクトの作業を開始します。



私たちが何度も何度も計画を間違えるという事実は、仕事の失敗や障害を考慮に入れていないという事実のせいです。 タスクまたはプロジェクトを完了するのに必要な時間を計算するとき、ワークフローの各段階に最適なシナリオを提示します。 必然的に、私たちは困難に直面し、仕事は膨大な量に膨れ上がり、締め切りに「適合する」タスクをさらに困難にします。



まだ期限が必要なのはなぜですか



迫り来る締め切りで働くことのストレスは紛らわしいかもしれないという事実にもかかわらず、私たちの科学のメリッサ・ダールが正しく指摘したように、それは「しばしば人々が何かを完了する唯一の理由である」締め切りです。



歴史的に、「期限」という言葉には文字通りの意味がありました。 それが、（死者-死者、線-線）が刑務所の周りの境界を呼んだ方法です。「囚人は昼夜を問わず、そうでなければ彼らは射殺されるでしょう」。

今日、締め切りは仕事に関連しており、私たちは罰の脅威をそれほど恐れていません。 しかし、考え方は変わっていません。期限は、生産性、プロジェクト、または仕事の生死を意味する場合があります。



「作業は、その実装に割り当てられたすべての時間を満たします」と、 パーキンソンの法律 。



言い換えれば、仕事はあなたがそれに捧げるのと同じくらいの時間がかかるでしょう。 そのため、短期間でどれだけのことができるかに驚かされることがあります。 また、この理由から、私たちは数日と数週間はありましたが、最後の数分で必死に何かを完了することがよくあります。



パーキンソンの法則を考えると、特定の期限がなければ作業を完了できないと結論付けることができます。 あなたがタスクを完了するために永遠を与えられている場合、それはあなたにとても時間がかかります。







期限変更を克服する方法



では、計画のエラーにどのように対処し、プロジェクトの完了期限を逃さないようにするのでしょうか？ 私たちの潜在意識に組み込まれている傾向を克服することは非常に困難ですが、期限が私たちの行動にどのように影響するかを理解することは、遅れにもかかわらず、より生産的に働くのに役立ちます。



1.週の前半をダウンロードします。



1週間、1か月、または1日間の作業計画を作成するときは、ワークフローの開始を最も忙しくします。 まず、最も難しい、重要な、または緊急のタスクを実行し、後期を無料にします。 あなたの仕事は必然的にあなたが計画したよりも多くの時間を必要とするでしょう、そしてまたあなたをコースからノックする追加の責任があるかもしれません。



1週間または1か月の在庫があるため、ワークフローに問題がある場合に柔軟にワークフローを変更できます。 そうすれば、以前に回避する予定だった新しいタスクに対処する必要がなくなります。



2.バッファーを作成します。



ZapierプロジェクトのマーケターであるMatthew Guayから学んだトリックは次のとおりです。この締め切りの数日前にプロジェクトの納期を設定します。 過去にこれを行ったため、必要以上に早いにもかかわらず、タスクマネージャーで指定した日付に集中することができました。 プロジェクトの実際の完了日が記録されていないというだけです。 バッファーとして自分で何日余分に与えるかについてのルールはありませんでした。 そのため、自分が設定した期限から実際の期限がどれだけ離れているかを100％知らなかったため、タスクマネージャーに記録された日付を見逃すことはできませんでした。



自分で設定した期限を守れない場合、この方法では絶望的な状況に陥ることがあります。



3.締め切りの時間を明らかにします。



「私はこの記事を夕方5時までに終えることを厳soleに誓います。」



黙って何かを約束すると、約束を守るかどうかは誰にもわかりません。 だから声を出して言ったら、責任を感じることができます。



目標を公に宣言することにより、タイムリーな期限を満たすことを名誉の問題とします。仕事を延期したいという願望と、公に失敗する機会との間で選択する必要があります。



Evelyn Waughが「 Return to Brideshead 」の執筆を開始したとき、彼は3か月以内に小説を完成させるという目標を設定しました。 彼は目標を発表し、「新しい小説の計画を作成しました。これには、執筆に約3か月かかります。」



Evelynは、プロジェクトのすべての作業を時間通りに完了しませんでした-デッドラインの遅延を何度か要求しました。 しかし、彼の公式声明のおかげで、彼は他の人が彼が適時に本の仕事を終えることを期待していることを知っていたので、彼は時間通りに収まるように素早く書きました。



4.自動アラームを設定します。



上記のいずれでも予定通りにプロジェクトを提出できない場合は、 Zapierオートメーションツール （ IFTTアナログ-翻訳者注）を使用してリマインダーを設定してみてください。



この戦術を試すには、Googleカレンダーを使用して期限を今後のイベントとして指定し、Zapierを使用してSMSリマインダー、Slackメッセージ、電子メール、またはその他の通信手段を受信します。 したがって、締め切りの24、12、または8時間前に関連する通知を受け取ります。 時間とメッセージはカスタマイズ可能です。 メッセージに動機を与えます。引用符を含めるか、Evelyn Waughのように、それらを公開リマインダーにします。



締め切りは、それらを通過させるように設計されています。 しかし、プロジェクトの期限を設定する必要性とプロジェクトとのやり取りの方法をより深く理解することで、プロジェクトから利益を得ることができます。 締め切り前にプロジェクトを完了できるという自信を持って職場に向かう場合は、ホフスタッターの法律を忘れないでください。
