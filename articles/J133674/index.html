サーバーとクライアントのパーツ。

サーバー部分とクライアント部分を作成するとき、まず質問がありました。 何に書きますか？ 特定の言語を理解して何かを書くことができることは明らかです。 すべてを学ばないでください。 はい。特定の目的に最も適した言語を選択します。 はい



そのため、サーバー側を記述するために、phpとfacebook自体が提供するライブラリを使用しました。 ここから入手できます： developers.facebook.com/docs/reference/php

実際、サーバー側全体がphpで記述されており、データベースのクエリやFacebookを使用したユーザーの承認に使用されます。 APIはユーザーデータへのアクセスを提供し、アプリケーションでユーザー認証を整理することができます。 概して、これで作業には十分です。 そして、これに限定しました。 一般に、承認にはいくつかのレベルがあります。 デフォルトでは、基本的なユーザー情報のみが要求されます：名、姓、生まれた場所、住んでいる場所、性別、年齢、興味など（彼自身について示すもの）、最後の訪問の時間、ユーザーのページのアドレス。 基本情報に加えて、壁へのアクセス、オフラインユーザーデータなどをリクエストできます。 これらはすべて、facebookのドキュメントページで詳細に説明されています： developers.facebook.com/docs/authenticationつまり ユーザーをfbでアプリケーションに取り込むと、彼に関するさまざまな情報を得ることができます。



実際、phpは、ユーザーに関するデータ、彼のポイント、アプリケーションへの最後のアクセス時間、質問のトピック、回答のカテゴリ、およびプロセスで必要になる可能性のあるその他のものを保存するデータベースの操作にも使用されます。



クライアント部分は、jQueryライブラリを使用してJavaスクリプトで完全に記述されています。 しかし、実際には、これはiframeのサイズに合わせたHTMLページであり、アプリケーションのFacebookを提供します。 レイアウトについては以下で説明しますが、先を見据えて、これらの言語（php / js / css / html）の初期知識はありませんでした。 最善の方法ではないことが判明しましたが、Internet Explorerでのみ最も一般的な方法で、いくつかの問題が発生します。 しかし、それについては以下で詳しく説明します。 以前にこれらの言語で少なくとも何かを書いた人は、より速く、より良く、より美しくするチャンスがずっとあります！



Flickr Photo Quizアプリケーションの一般的なアルゴリズムは次のとおりです。 アプリケーションは、データベースから取得した特定のパラメーターを使用して、flickr apiへのjsonリクエストを使用して写真と写真を受信します。 たとえば、モスクワまたはアジア。 そうでなければ、完全にランダムな画像が得られます。 さらに、私たちが要求したものに応じて、5ポイントスケールのポイント数が選択されます。ユーザーは、質問に正しく答えると受信します。 選択は主観的であるため、誰かにとって5点の質問は非常に簡単です。 プレイヤーが間違って答えた場合、答えが正しく選択された場合のポイント数に関係なく、いずれの場合でも2ポイントが奪われます。 回答オプションは、flickrに送信された要求に従ってデータベースから選択されます。 正しい答えは1つだけで、残りの3つは目的のカテゴリ（国、都市、世界の一部）からランダムに選択されます。 応答はランダムに投稿されます。



アプリケーションの別の機能。 ユーザーのポイントが100ポイント未満の場合、不正解の場合、ポイントはユーザーから奪われません。 これは、新規ユーザーにとって一種のボーナスです。 人がプレーし、100ポイント以上を獲得した場合、ペナルティシステムはすでに機能し始めています。 画像はサーバー上のどこかに保存されるのではなく、flickrから直接取得されるため、アプリケーションは優れています。 したがって、ユーザーが1か月後にこのアプリケーションにログインすると、アプリケーションデータベースからの同じ要求で、flickrは既に他の写真を提供し、写真を記憶するだけでは勝てません。

Google Chromeでは、まったく問題はありません。 これは最高のプログラマーによって作成されたスマートなブラウザーです。 OperaとMozilla FireFoxでは事態は少し悪くなっています。 ただし、これらのブラウザには特に問題はありません。



Internet Explorerでアプリケーションを適切に表示させようとすると、最も困難で理解できないタスクが発生します。 これを行う方法に関するルール、方法、アルゴリズムはありません。 すべてを目で、試行錯誤で行わなければなりません。 Internet Explorerでは多くの機能が動作しないため、この1つのブラウザー専用のレイアウトを作成する必要があります。 そしてそれは必要です。 ほとんどの人がまだそれを使用しているため。 残りについては、CSSマークアップの基本について少し学ぶことで、誰もがこれを行うことができます。



プロモーション。

作成されたアプリケーションの成功指標をたくさん考えることができますが、それでも、最も簡単な方法は、測定できるものを管理することです。 したがって、アプリケーションの成功を、それがもたらす金額で測定します。

明らかに、より多くの人々、より多くのお金をもたらすでしょう。



100万人以上が積極的にアプリケーションを使用している場合、アプリケーションは成功したと見なされます。 アプリケーションが存在するためのクリティカルマスは10万ユーザーです。

最も成功したソーシャルネットワーキングアプリケーションには、500万人以上のユーザーがいます。 これは良い安定した収入をもたらします。



ソーシャルネットワークの膨大な数のアプリケーション、開発会社のWebサイトの情報、雑誌の記事を分析した結果、アプリケーションの成功の6つの主要な基準を特定しました。



•「来る、やる、去る」という原則。

この原理は、フェルマータイプのアプリケーションで完全に実証されています。 私は入って、掘ってベッドに水をやり、収穫して出ました。 ユーザーの介入なしで、作物が成長し、熟します。 これはソーシャルゲームの主なゲームプレイです。 この原則が欠如しているため、従来のゲームのリーダーはソーシャルゲームアプリケーションで最高になれませんでした。 従来のゲームは刺激的ですが時間がかかります。 したがって、そのようなゲームはすぐに視聴者を獲得し、その後急速に失いました。



•アプリケーションへのユーザーの定期的な戻り。

開発者の目標は、最大限の利益を得ることです。 多くの新しいユーザーが良いです。 しかし、「古い」散布は悪い。 それらを何度も入れます。 農場は素晴らしい例です。 期限内にアプリケーションに入らないと、作物が腐ったり、劣化したり、盗まれたりします。 ゲームに頻繁に登場するためにボーナスを与えることができます。たとえば、毎日の訪問のためのゲームポーカーでは、カジノでプレイできるチップが追加されます。 したがって、人は常に勝つだけでなく、毎日アプリに参加するだけでなく、大きなスコアを持つことができます。 多くの人が毎日ポイントを求めてそのようなアプリケーションにアクセスし、それが習慣になります。



•友情を最大限に活用する。

「友人を招待する」ボタンが必要です。 さらに、アプリケーションはプレイヤーの友達と直接だけでなく間接的にも動作する必要があります。 たとえば、このアプリケーションは、奇跡のピース、カクテル、かわいい子猫、グリーティングカード、または友人の壁に助けを求めるリクエストを発行します。 一部のゲームは、グループプレイまたは決闘に特化しています。 多くのアプリケーションは、招待された友人にボーナスを提供します。 これはうまく機能します。



•ビッグネームと魅力的なアバター。

友人の壁やニュースに掲載されている美しい写真や面白い写真は、中に隠されているものを見る良い動機となります。 ここでの主なことは、無理をせず、スパムメーリングリストにしないことです。 ご存じのように、ほとんどのアプリケーションはダウンロードする前に、ユーザーに個人情報へのアクセス許可を求めます。 魅力的なアバターの目的は、アプリケーションがロードを開始する前にユーザーがログアウトしないようにすることです。



•評価。

適切に設計された評価システムがなければ、成功はありません。 ユーザーが何かを測定する必要があります。 ポイントの数、強さ、キャラクターのレベル、総合ランキングまたは友人の間の場所。 「誰がもっとクールで、誰がもっとクールか」という単純な原則は、常に完璧に機能します。 人々は常に、残りのどの場所を占めているかを知りたいと思っています。 より多くの異なる尺度と評価、より良い。 格付けシステムのおかげで、競争心、競争心が目覚め、これにより彼は何度も何度もアプリケーションに参加しなければなりません。 この項目は、とりわけ、アプリケーションの収益化の大部分を実現できるという点で重要です。



•ユーザーとの絶え間ないフィードバック。

人は常に、自分が誰であるか、何を達成したか、あといくつかのステップを残していること、近い将来に達成できることを思い出す必要があります。 そのため、時々ユーザーの壁にフィードバックを送信する必要があります。 そして、それがカラフルであればあるほど良いです。 アプリケーションの参加者に、すでに多くのことを達成していることを思い出させて、再びプレイするように誘うだけでなく、これを見て、彼の友人は競争したり、新しいユーザーになろうとするかもしれません。 ところで、一部のアプリケーションは、そのようなバイラル広告のおかげで人気を得ました。
