セイルフィッシュOS 2016サマースクールについて

この記事では、7月27日から30日に発生した素晴らしいイベントについてお話したいと思います。SailfishOSサマースクールは、イノポリス大学の支援を受けて「オープンモバイルプラットフォーム」によって開催されました。 Sailfish OS、スマートフォンでの実際のLinux、またはQML / C ++のアプリケーションの別の分野の開発の見通しに興味がある場合は、面白いだけです-私はhabracatを求めます。















まず第一に、私にとって、Sailfish OSでのこの話は2016年4月1日に始まりました。 冗談ではなく、待ちに待っていたホヤを腕に抱いたのはこの日でした。 OSを掘り下げ、マニュアルを読み、オタクの記事をいくつか公開し、公式ストアで最初のアプリケーションを公開します。



一度、自宅に座ってHabrを読んで、セイルフィッシュOSサマースクールについての告知を見て、すぐにそこに行くことに決めました。 数週間後、私はすでにイノポリスのメインストリートであるユニバーシテツカヤに立ちました。





写真はクリック可能です。



なんて素敵な場所だった！ なんと開発者のコ​​ミュニティ！ 信じられないほどでした！ アエム...さて、感情を脇に置いて、順番に取りましょう。



7月27日



知り合いの日、入門スピーチの日、宿泊の日でした...







快適なダブルルームに参加者を決めました：







部屋には、コーディングの「コア」間の短い休憩に必要なすべてのものがありました：ベッド、シャワー、エアコン、テーブル。 私の部屋での唯一のことは、2脚と0脚（！）の椅子用の4つのナイトスタンドでした。2つの「余分な」ナイトスタンドの使用方法を推測したと思います。



椅子がなかった理由
イノポリス大学の代表が私たちに説明したように、このキャンパスはまだ学生のために完全に準備されていません。 うーん、しかし、私はそのような「未完成」に住んでいたでしょう。


私は隣人と特に幸運でした。尊敬されている全員と一緒に暮らしました。SailfishOS のパッチのマスター 、Andrey Kozhevnikovは、CodeRusというニックネームでよく知られています。 Sailfish OSには、ドキュメントの小さな問題、特にその構造と一部の詳細の不十分なカバレッジがあります。ここで、Sailfish OSのウォーキングナレッジベースに落ち着きました。 :) Andrey、私の質問に対する包括的な答えをありがとう。



午後3時までに、全員が講堂307に集まり、実際に私たちは次の48時間生きました。







紹介のスピーチ、Sailfish OSに関する一般的な言葉、ロシアでの展望、および「Open Mobile Platform」の役割を聞きました。 学長はイノポリスで勉強するように呼びかけました。



その後、パートナーは次のように話しました。



Alexander Kiselev - Sailfish OSへのVipNetソリューションの埋め込み 。 彼らは、企業市場に非常に人気のあるソリューションだと言います。



Georgy Martirosov- 安全な衛星ブラウザ 。 興味深いことに、彼らは検索を行いましたが、最終的にはブラウザを作成しました...まあ、個人的には、標準ブラウザのライバルがSailfish OSに登場するかどうかは気にしません。



Egor Levichev、Alexander Sergeantov - Sailfish OSプラットフォームのActive Sync 。 とても面白い人。 実装の品質に関しては、非常に安定して動作し、少量のバッテリーを使い果たし、すべてがシステムにネイティブに統合され、「Androidユーザーがうらやましい」とユーザーから聞きました。



ピーターバルバニャガ - モバイルペイメントキャッシュターミナル（Sailfish OS）



夕方、私たちはイノポリスの街のツアーに連れて行かれました。











イノポリスについて
記事をやりすぎないように、簡単に答えます。都市はまだ発展しており、現在約2,000人の住民が住んでいますが、快適な生活に必要なものはすべて揃っています。 この「プロジェクト」が将来どのようなことにつながるかを見てみましょうが、幸運を祈り、メイントピックに戻ります。

はい、今日は接頭辞「crypto」が付いたテーマは流行していますが、深刻なことに、多くの人がセキュリティのトピックを心配していました。



私のチームは、ロールプレイングゲームKillerに基づいてゲームを作成することにしました。 実装機能を決定しました：



-被害者はサーバー上で選択されます（チームはAndroid向けのKillerの作成者の1人を含めたため、サーバーは既に準備ができていました）。

-犠牲者の写真と名前を取得するためのVkontakteでの要求と承認。

-Bluetooth経由の「ショット」を使用して「犠牲者」を「殺す」。

-GPSを使用して、プログラムはGoogleマップで被害者を示します。

-プレイヤーの位置を追跡するデーモン。



一般的に、計画は素晴らしかった...



私たちがやったこと、またはハッカソンに勝たなかったが、素晴らしい経験を得た方法

下から上にスワイプすると、「キル」ページに到達しますが、残念ながら、Bluetoothを処理できませんでした。そのため、サーバーが被害者に一意のコードを送信し、「殺人者」が「被害者」画面から直接読み取り、自宅で入力する必要がありました。





コードが正しくない場合、再度入力を試みます。それ以外の場合は、「殺人」を祝福し、被害者選択ページに移動します。



したがって、ハッカソンの終わりまでに、私たちにはする時間がありませんでした。



-Vkontakteでの写真と被害者の名前の問い合わせ。
