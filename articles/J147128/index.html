アンダーワールドの小さなC関数

最近、私の学生と私はC標準の微妙な点を理解しようとしました。このような問題を明確にする最も簡単な方法は、コンパイラ開発者がそれを考慮に入れたかどうかを調べることです。



私はこのような関数を書きました： 

int foo (char x) { char y = x; return ++x > y; }


++x



を1ずつインクリメントするため、ほとんどのx

1つの仮定は、関数が確実に「 0 」を返すことでした。 もう1つは、charが符号付き型であるプラットフォームで++x



定義されていないこと++x

結論を印刷するテストプログラムが作成されました。 

 int main (void) { int i; for (i=CHAR_MIN; i<=CHAR_MAX; i++) { printf ("%d ", foo(i)); if ((i&31)==31) printf ("\n"); } return 0; }


このコードは、charがintより小さいプラットフォームでのみ機能しますが、逆のことが発生したプラットフォームは見ていません。
