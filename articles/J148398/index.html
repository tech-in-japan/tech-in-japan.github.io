光学ドライブを使用せずにMacにWindows 7/8をインストールする

現在、多くのMacコンピューターには光学式ドライブが搭載されていません。MacbookAirおよびMac Miniにバンドルされていないだけでなく、一部のMacBook / iMacでは追加の2.5インチドライブに置き換えられます。



新しいポピーはフラッシュドライブから起動できますが、私は新しいポピーの所有者ではありません（2010）。このガイドは、ポピーのイメージからWindowsをすばやくインストールしたい人にも役立ちます。



私はさまざまな方法を試しました-USBフラッシュドライブ/ディスクからインストールしようとしましたが、すべてが無駄でした（すべてのMacがフラッシュドライブまたは外部ドライブから起動するわけではありません）。 修理も何も与えませんでした-暗い画面がありました。 この方法はただの救いでした。



必要なもの：



  1. 仮想ボックス
  2. WinClone （最新の無料版）
  3. 好きなWindows 7/8 ISOイメージ


ステップ1：Boot Campパーティションを作成する



Boot Campアシスタントを使用してBootcampパーティションを作成します。 操作は非常に簡単です。 10.6では、アシスタントは10.7でコメントなしのセクションを作成します-彼はあなたがドライブを持っていないことを警告します。



ステップ2：Windows 7/8汎用インストールイメージを作成する



ここで「一般化」という言葉は、画像が特定のMacハードウェアに結び付けられていないことを意味します。 （WinCloneを使用して）Boot Campパーティションにアップロードした後、そこから起動すると、Windows 7/8はすべてのハードウェアを自動的に構成し、再起動して作業できます。 ただし、Boot Campキットからインストールできる追加のドライバーが必要になる場合があります。 将来、このイメージを任意のMacで使用して、Windowsを非常に迅速にインストールできます。 ただし、初めて作成する必要がある場合、この手順は簡単ではありません。



ステップ2.1：VirtualBoxにWindows 7/8をインストールする


VirtualBoxで新しい仮想マシンを作成し、 win7という名前を付けます。 VDI仮想マシンには、動的に割り当てられたディスクイメージを使用することが重要です。 10 GBのサイズで十分です。 インストールISOイメージを使用してWindowsをインストールします。 VirtualBoxの補助ソフトウェアを含むソフトウェアをインストールしないでください。 クリーンなシステムでなければなりません。



ステップ2.2：Windows 7/8でシステム予約済みパーティションを削除する


問題は、Windowsが別の追加パーティションを作成し、その中にブートローダー自体が配置されていることです。 ブートローダーをメインパーティションに転送し、この補助パーティションを削除する必要があります。



更新 ：Windows 7/8のインストールを開始するとき、ディスクパーティションを選択するときに、新しいパーティションを作成すると、システムが予約され、C：\ドライブが表示されます。 次に、C：\ドライブを削除し、System Reservedを目的のサイズに拡張してフォーマットします。 その後、Windowsをインストールできます。システム予約パーティションはなくなります。 ありがとうIGHORまたは、この方法を使用します（既にインストールされているWindowsの場合）：



  1. Windowsをダウンロードする
  2. [ ディスクの管理 ]に移動します（エクスプローラーで[コンピューター]を右クリック-> [管理]）
  3. 非表示のパーティションに任意の文字を割り当てます（パーティションを右クリックして、[ ドライブ文字またはドライブパスの変更 ]を選択します）。 たとえば、彼らは文字Eを割り当てました：
  4. 管理者の下でコマンドラインを起動します。 または、 慎重に、次のコマンドを入力します。 

    • reg unload HKLM\BCD00000000
      このレジストリブランチをアンロードします 
    •  robocopy e:\ c:\ bootmgr
      bootmgrファイルをコピーする 
    •  robocopy e:\Boot C:\Boot /s
      ブートフォルダーをコピーする 
    •  bcdedit /store c:\boot\bcd /set {bootmgr} device partition=C:
      セクションCでブートローダーを登録します。
    チームの結論によると、私たちはすべてがうまくいったと確信しています。 これで、コマンドラインを閉じることができます。

  5. ディスクの管理で 、Cパーティションをアクティブにし、既に不要なシステム予約パーティションを削除します。 未割り当ての領域（100 MB）は残りますが、それは問題ではありません。
  6. 再起動すると、システムはセクションCから正常に起動します。




ステップ2.3： sysprepユーティリティを使用してWindowsを一般化する


  1. 管理者の下でコマンドラインを起動します。 以下を入力してください： 

     net stop WMPNetworkSvc
    「なぜ」という質問に対する正確な答えはありません。 ただし、このサービスは停止する必要があります
  2. C：\ Windows \ System32 \ sysprepフォルダーで、次の内容のunattend.xmlファイルを作成します。 

     <?xml version="1.0" encoding="utf-8"?> <unattend xmlns="urn:schemas-microsoft-com:unattend"> <settings pass="generalize"> <component name="Microsoft-Windows-Security-Licensing-SLC" processorArchitecture="x86" publicKeyToken="31bf3856ad364e35" language="neutral" versionScope="nonSxS" xmlns:wcm="http://schemas.microsoft.com/WMIConfig/2002/State" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"> <SkipRearm>1</SkipRearm> </component> </settings> </unattend>
    Windowsはデフォルトで「TXT」：unattend.xml.txtを追加できるため、ファイル拡張子が「XML」であることを確認してください。 これを行うには、sv-vahフォルダー内のすべてのファイルの拡張子を常に表示するようにオプションを設定します。

  3. C：\ Windows \ System32 \ sysprep \ sysprep.exeを起動します。 次のオプションを追加します。







    「OK」をクリックすると、マシンは自動的にオフになります。





ステップ2.4：VDIイメージをWinCloneイメージに変換する


  1. VirtualBoxで、 [ファイル]-> [メディアマネージャー]に移動します。 [Windowsパーティションのコピー]をクリックします。必ず[ 固定サイズ ]オプションを使用してください。 開始コピーはwin7_copyと呼ばれます 。 この操作が完了するのを待っています。
  2. ターミナルに移動し、コピーを保存したフォルダーに移動して、次を入力します。 
     sudo VBoxManage internalcommands converttoraw win7_copy.vdi win7.raw
    再び完了を待っています
  3. ターミナルでもう1つのコマンドを入力します。 

     sudo hdiutil attach -imagekey diskimage-class=CRawDiskImage win7.raw
    その後、 無題のイメージをマウントする必要があります
  4. WinCloneに入り、最初のタブでこのマウントされた無題ディスクのコピーを作成しますImage
  5. 画像の準備ができました！


ステージ3：WinCloneイメージをBoot Campパーティションに塗りつぶします



WinCloneの2番目の[ 復元 ]タブを使用して、前の手順で作成したイメージを既に物理的なBoot Campパーティションにアップロードします。



今後のインストールでは、手順1と3のみが必要になり、2番目の手順は1回だけ実行する必要があります。
