ファイブファイブ-25！

（時間：1秒。メモリ：16 Mb複雑度：8％）

VasyaとPetyaは同じクラスで学校に行きます。 最近、PetyaはVasyaに5で終わる自然数を2乗するトリッキーな方法について話しました。Vasyaは5で終わる2桁（さらには3桁）の数字を簡単に2乗できます。方法は次のとおりです。 5で終わる場合は、最後の5つを次の番号で削除することにより、元から取得した数を乗算するだけで十分です。その後、右側の結果に「25」を割り当てるだけです。 たとえば、125を2乗するには、12を13倍して25を割るだけで十分です。 番号12 * 13 = 156番号25に起因すると、結果15625が得られます。 1252 = 15625。 Vasyaがスキルをテストできるように、5で終わる数字を四角で囲むプログラムを作成します。

入力データ

入力ファイルINPUT.TXTの唯一の行には、4 * 10 ^ 5を超えない、5で終わる正の整数Aが1つ含まれています。

インプリント

出力ファイルOUTPUT.TXTで、1つの正の整数（先行ゼロなしのA2）を出力します。

例：

INPUT.TXT

5

75

4255

OUTPUT.TXT

25

5625

18105025

B.ビーバー

制限時間：テストごとに1秒

メモリ制限：64 MB

ビーバーは、狭い川の水路にダムのカスケードと居心地の良い小屋を建設しようとしています。 川は理想的にまっすぐな道に沿って流れ、川の幅は非常に小さいため、このタスクの枠組みでは無視できます。 ビーバーが建設に使用できる川のほとりに木があります。 科学者は、ビーバーがダムや小屋を建設する場所を、木を輸送するのに必要な最短距離の観点から最適に選択する方法を見つけることにしました。

川の直線区間の始点に対する樹木の与えられた座標に従って、同じ方向の流れの軸を数える場合、樹木が移動しなければならない最小合計距離に対応するオブジェクトの座標を決定するプログラムを書きます。

入力フォーマット：

入力の最初の行には、1つの正の整数1 <= T <= 10が含まれます-次々に進むテストブロックの数。 各テストブロックの最初の行には、2つの正の整数1 <= N <= 1000、0 <= M <= 10、0 <= L <= 100-それぞれ、川の土手に生えている木の数、建設に必要な木の数が含まれています1つのオブジェクトと、組み立てる必要があるオブジェクトの数。 次のN行のそれぞれに、正の実数のみが記録されます。これは、川の直接セクションの開始点（最上流）から対応するツリーが成長する場所までのメートル単位の距離です。 すべてのオブジェクトを構築するのに十分なツリーが保証されていることが知られています（N> = M * L）

出力形式：

テストブロックごとに、個別の行に、単一の数字を表示する必要があります-構築のためにツリーを移動する必要がある総距離が最小になるように、オブジェクトを直立させる場所の座標の合計で、小数点以下の3つの正確な記号を示します。

アルバトロスは、海の広がりを越えて長い距離を乗り越えて、長い飛行を行うことができます。 鳥類学者は、アホウドリが陸地に行かなくても何キロメートル飛ぶことができるかを決定することにしました。 このために、浮遊する研究室の小隊が海中に散らばり、RFIDタグが取り付けられた調査対象者のデータを記録しました。 科学者は、アホウドリを発見した場所の時間と現在の座標を記録します。
