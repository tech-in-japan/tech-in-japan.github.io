NexStreaming NexPlayer SDKを使用したアプリケーション開発

モバイルプレーヤーアプリケーションの開発は簡単ではありません。 ビデオ、サウンド、字幕の多くの形式をサポートする必要があります。 ローカルおよびインターネットに保存されているファイルの再生を実装する必要があります。 これらすべての作業の結果が、リソースを注意して扱う必要がある環境で実行されることを覚えている場合、最適化を行う必要があります。 Android向けの高品質で高速かつ経済的なプレーヤーの作成を開始するにはどうすればよいですか？ NexPlayer SDKは、この質問に答えるのに役立ちます。







予備情報



NexStreamingは、モバイルデバイス用のソフトウェアを開発するグローバル企業です。 NexStreamingの本社は韓国のソウルにあり、支店はスペイン、米国、日本、中国にあります。 NexPlayer SDKは最も人気のある製品です。 これは、プレーヤーを作成するためのソフトウェア開発キットです。 ビデオストリーミングに携わる非常に有名な企業がモバイルアプリケーションで使用しています。 このプレーヤーは、すべての最新のDRMテクノロジーと互換性があります。 さらに、その主な機能を拡張できます。 たとえば、サウンドを改善するツール、広告挿入のサポート、視聴者に関する統計情報を収集する機能を追加します。



NexStreamingのNexPlayer SDKは、ビデオおよびオーディオデータをデコードおよび再生するためのツールを提供します。 アプリケーション開発者はSDKを使用して、独自の製品をすばやく作成できます。 NexPlayer SDKは信頼性が高く、安定しており、国際標準に準拠しています。 この記事では、NexPlayer SDKを使用してx86ベースのプレーヤーを作成する方法を学習します。



主な機能



NexPlayer SDKはx86プラットフォーム向けに最適化されています。つまり、このプラットフォーム上に構築されたデバイスで完全にサポートされています。 SDKの主な機能は次のとおりです。



  • Android 1.6以降をサポート
  • mp4、3gp、avi、asf、piffファイル形式の再生
  • HTTPライブストリーミング5.0および3GPPプログレッシブダウンロードのサポート。 AES128およびHTTPSプロトコル、h.264、AAC、AAC +、eAAC +コーデックのサポート。 SDKを使用すると、ソフトウェアとハ​​ードウェアの両方のコーデックを操作できます。
  • .smi、.srt、.sub、3GPPタイミングテキスト、TTML（PIFF / CFFのみ）、CEA 608およびCEA 708形式の字幕のサポート。WebVideo Text Tracks（WebVTT）テクノロジーのサポート。


NexPlayer SDKを使用してx86プラットフォーム用のプレーヤーを作成する方法



まず、NexStreaming WebサイトからSDKをダウンロードする必要があります。短いフォームに入力し、データが記載された電子メールを待って、資料が配置されているサイトの閉じたセクションに入力します。 特に、ダウンロード可能な資料の中で、Androidデバイスにすぐにインストールできる.apkファイルのデモアプリケーション、IDEにインポートできるサンプルプロジェクト、ライブラリのセットとしてのSDK自体、およびそのドキュメントを見つけることができます。 。 必要なライブラリを決定するには、ドキュメントに精通し、例を確認する必要があります。 NexPlayer SDKを使用したアプリケーション開発は非常にシンプルで簡単であり、サンプルの存在はその迅速な開発に貢献しています。 NexPlayer SDKを使用する作業アプリケーションを作成するには、約1時間かかります。



NexPlayer SDKのx86ベースのAndroidアプリケーションへの統合では、ライブラリとソースコードをSDKからプロジェクトにコピーします。 以下に説明するいくつかの簡単な手順を実行した後、Intelのコンポーネントに基づいて構築されたデバイス上で完全に動作できる基本コンポーネントをアプリケーションに装備します。 したがって、準備段階は次のとおりです。



  • 必要なライブラリをSDK / libsフォルダーから独自のプロジェクトのasset / x86フォルダーにコピーします
  • 必要なライブラリをSDK / libsフォルダーからプロジェクトのlibs / x86フォルダーにコピーします
  • JavaファイルをSDK / srcからプロジェクトのsrc / com / nexstreaming / nexplayerengineフォルダーにコピーします


コピーするライブラリのセットは、アプリケーションのニーズによって異なります。 必要なライブラリを理解するには、デモを見て、以下の推奨事項を使用する価値があります。



コピー後、NexPlayer SDKは変更を検出し、ライブラリを使用してIntelリソースとの効率的な作業を整理できるようになります。 すべてのライブラリが指定されたディレクトリにあるとすぐに、SDKはアプリケーションのARMバージョンとx86バージョン間の切り替えを引き継ぎます。 既存のプロジェクトのNexPlayer SDKライブラリを更新する場合は、それらのファイルを上書きするだけです。



NexPlayer SDKには、DRMサポートを担当するライブラリを含む多数のライブラリが含まれています。 それらはapp / asset / x86フォルダーにあります。 必要なライブラリのセットには、プレーヤーエンジン（エンジン）、デコーダー（デコーダー）、および視覚化ツール（レンダリングレイヤー）が含まれます。 それらのリストは次のとおりです。



  • libnexplayerengine.so

  • libnexalfactory.so
  • libnexadaptation_layer_for_dlsdk.so
  • libnexralbody_audio.so
  • libnexralbody_video_opengl.so
  • libnexral_nw_ics.so
  • libnexral_nw_jb.so
  • libnexcal_oc_ics.so
  • libnexcal_oc_jb.so
  • libnexcralbody_mc_jb.so
  • libnexcal_in_aac_x86.so
  • libnexcal_in_mp3_x86.so
  • libnexcal_in_amr_x86.so


一部のライブラリ名には略語icsが含まれています。 ライブラリがAndroid Ice Cream Sandwich用であることを示しています。 「jb」という略語は、ライブラリがAndroid Jelly Bean用に設計されていることを意味します。 アプリケーションが特定のバージョンのAndroidをサポートするように設計されている場合、サポートされていないバージョンのライブラリは削除できます。



コーデックのサポートを担当するライブラリは次のとおりです。



  • libnexcal_h364_x86.so-H.264ビデオサポートライブラリ
  • libnexcal_aac_x86.so-AAC、AAC-Plus、HE-AAC形式のサウンドサポートライブラリ
  • libnexcal_mp3_x86.so-MP2およびMP3形式のサウンドサポートライブラリ


以下は、字幕をサポートすることを目的とするライブラリです。



  • libnexcal_3gpp_x86.so-3GPPタイミングテキスト形式のサポート
  • libnexcal_closedcaption_x86.so-CEA 608およびCEA 708形式のサポート
  • libnexcal_ttml_x86.so-TTML形式（CFF）のサポート
  • libnexcal_webvtt_x86.so-WebVTT形式のサポート


アプリケーションのサイズを縮小するには、プロジェクトに必要なライブラリのみを含めます。

app / libs / x86フォルダーからライブラリーをダウンロードする必要があります。 これは、対応するクラスのinitManager（）メソッドで行われます。 たとえば、NexHDSampleを使用するには、次のコードを使用して、NexHDManagerクラスの上記のメソッド（app / src / NexHDManager.java）で適切なx86ライブラリをロードする必要があります。

System.loadLibrary（ "NexHTTPDownloaderSample_jni"）;



x86ベースのアプリケーション画面にビデオを表示する方法



NexPlayer SDKを使用する場合、ビデオを表示する方法は2つあります。 これは、NexVideoRendererおよびOpenGLに基づく出力サブシステムの使用です。 NexVideoRendererを正確に使用することをお勧めします。 実際、このサブシステムは、複雑な表面制御操作とビデオ出力タスクを抽象化するレベルを提供します。 彼女は、デバイスのハードウェアとそれにインストールされているオペレーティングシステムに基づいて、ビデオを出力するのに最適な方法を選択します。



NexVideoRendererを使用するには、次を実行する必要があります。



  1. コンテキスト（android.context.Context）をコンストラクターに渡します。
  2. イベントリスナー（NexPlayer.IListenerおよびNexPlayer.IVideoRendererListener）をインストールします。
  3. NexPlayerクラスのオブジェクトのインスタンスを作成します。
  4. NexPlayerの必要な準備を実行します（NexPlayer.setNexAlFactoryおよびNexPlayer.initメソッドを使用します）。
  5. クラスNexVideoRenderer（NexVideoRenderer.init）のオブジェクトのinitメソッドを呼び出します。
  6. NexVideoRendererのインスタンスをインターフェイス要素としてレイアウトに追加します。


NexVideoRendererクラスのコードは、NexPlayerSample / src / com / nexstreaming / nexplayerengine / NexVideoRenderer.javaのSDK使用例の中にあります。



ストリーミング



HTTPライブストリーミングテクノロジーにより、複数のオーディオおよびビデオストリームを操作できます。 SDKはこの機能をサポートしています。setMediaStream（）APIを使用すると、再生するストリームを選択できます。 これは、コンテンツの再生中にユーザーインターフェイスから実行できます。 ビデオとサウンドを操作するには、次のオプションがあります。



  1. さまざまなサウンドトラックオプションでプレイリストを使用します。 このアプローチのビデオとサウンドは、個別に選択できます。



  2. さまざまなビデオオプションでプレイリストを使用する。 再生される各ビデオクリップにはオーディオとビデオの両方が含まれますが、ユーザーは代替ビデオオプションを選択できます。 たとえば、さまざまな角度またはさまざまなポイントからカメラで撮影されたマテリアルです。



  3. 利用可能なオーディオトラックとビデオのリストの組み合わせ。 これは、上記の2つのオプションの混合です。 メインビデオストリームは、異なるビットレートのビデオトラックを提供しますが、同じサウンドで再生されます。 同時に、たとえば異なる言語で追加のオーディオトラックを使用できます。
