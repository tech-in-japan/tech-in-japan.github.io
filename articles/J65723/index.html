GNOME Nautilus：メインパネルの構成

少し前まで、Nautilusのメインパネル（GNOMEデスクトップファイルマネージャー）に新しいタブを作成するためのボタンを追加する必要に直面しました。 設定を検索しても何も得られなかったので、インターネットで調べてみました。



その結果、ユーザーフレンドリーな方法ではありませんが、この問題の解決策を見つけました。





アクションのシーケンスは次のとおりです。



1.ルートの下にある任意のテキストエディターを開きます。

2.ファイル/usr/share/nautilus/ui/nautilus-navigation-window-ui.xmlを開きます

3.ファイルを<toolbar name = "Toolbar">セクションの先頭までスクロールします

4.このセクション内に必要な要素を追加します。

5.セッションを終了して再度ログインし（ログアウト->ログイン）、変更を有効にします。

（ Tormからの注意：セッションを終了できますが、Nautilusを再起動するだけです。これは、たとえばkillall nautilusコマンドを使用して実行できます）



いくつかの説明





このファイルは通常のxmlファイル（拡張子から確認できます）であり、手動で編集できます。 彼はNautilusのグラフィカルナビゲーションを担当しています。



<toolbar name = "Toolbar">セクションのコンテンツは、 ツールバーを埋める役割を果たします（名前が示すように-toolbar）。



エレメント自体の形式は次のとおりです。 <toolitem name = " display name " action = " action to be perform" />



たとえば、新しいタブの場合、次の行を追加しました。



<toolitem name="New Tab" action="New Tab"/>







垂直の分離棒（セパレーター）を追加する場合は、残りの要素の間に<separator />要素を挿入するだけです。



...

<toolitem name="Computer" action="Go to Computer"/>

<separator/>

<toolitem name="Search" action="Search"/>

...









ヒープの前に、これらの要素を追加しました。



<toolitem name="Cut" action="Cut"/>

<toolitem name="Copy" action="Copy"/>

<toolitem name="Paste" action="Paste"/>

<toolitem name="Trash" action="Trash"/>









この小さなトピックがNautilusのパネルを変更したい人に役立つことを願っています。



Post Scriptum：パネル内の要素をグラフィカルにカスタマイズする方法が見つかりませんでした。
