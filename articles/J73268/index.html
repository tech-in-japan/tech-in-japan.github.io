WDS経由でネットワーク経由で展開するためのWindows XP SP3のイメージを作成する

善人fcsergのリクエストに応じて公開されたので、カルマに転送を依頼します



多くの場合、「多くのワークステーションを展開する」などのタスクがあり、これらすべてのステーションの鉄は異なる場合があります。 さらに、これらのマシンの一部にはさまざまなソフトウェアを事前にインストールする必要があります（「企業セット」-ICQ、ブラウザー、pdfリーダー、「大学」セット-Visual Studio、Project Manager）。 以前は、RIS（リモートインストールサービス）がそのような目的で使用されていましたが、現在はMicrosoftがWDS（Windows展開サービス）のキャンペーンを行っています。 後者に焦点を当てます。



WDSは、Microsoft-Vista、Windows Server 2008、Windows 7の新しいシステムのイメージでうまく機能しますが、古き良きWindows XPを使用する必要がある場合（つまり、この状況は頻繁に発生します。 。 この記事では、WDSサーバーと連携する.WIM形式のイメージを作成するためにOSを準備する方法を学習します。

最初のステップ：Windows XP SP3のテンプレートインストールを作成する



仮想マシンにWinXP SP3をインストールします。 私は個人的にVMWareを仮想マシンのプラットフォームとして使用しました。



また、この段階では、必要なすべてのソフトウェアを配置することは理にかなっています。 標準パッケージに含まれています：



  • Winrar
  • K-Liteコーデックパック
  • Microsoft Office
  • アドビリーダー
  • 総司令官
  • IMクライアント（QIP、ICQなど）
  • ブラウザ（Opera、Firefoxなど）




コンピュータークラス（大学、学校、ゲームクラブ）にサービスを提供している場合、この段階で、追加のソフトウェア（開発ツール、グラフィックエディターなど）を配置する機会があります。

第2段階：必要なすべてのドライバーをパッケージ化するなど、イメージを取得するためのシステムを準備します。



これで、結果の仮想OSをSysprepで処理して、インストール後を正しく行う必要があります。 念のため、SysprepはWindows XPのインストールディスク（D：\ Support \ Tools \ deploy.cab）にあります。D：は仮想マシンのCDドライブの文字です。



仮想マシンのC：\ Sysprepなどでdeploy.cabの内容を解凍し、setupmgr.exeを実行します。 必要に応じて応答ファイルを作成するプロセスを実行します。 唯一の注意点は、応答ファイルのタイプを選択するときに、2番目のオプション「Sysprepのインストール」を選択する必要があることです。





その結果、応答ファイルSysprep.infを取得します。 次に、このファイルの内容をわずかに変更する必要があります。



[無人]セクションで、パラメーターを変更します



InstallFilesPath = "C：\ WINDOWS \ Driver Cache \ i386"



追加します：

DriverSigningPolicy =無視

UpdateInstalledDrivers =はい



[SetupMgr]セクションで、パラメーターを変更します



DistFolder = "C：\ WINDOWS \ Driver Cache \ i386"





次に、一連のドライバーを作成する必要があります（これらは、イメージを展開するときにシステムにすべてのデバイスをインストールするために使用されるドライバーです）。 会社にワークステーションをインストールするために使用されるドライバーで構成することも、既製のものを使用することもできます（これらの多くはインターネット上にあります）。 これらのすべてのドライバーをC：\ Windows \ Driver Cache仮想マシンにコピーします。



次のステップは、Sysprep Driver Scannerを使用することです。 ダウンロードして、C：\ Sysprepフォルダーに配置します。 以下の手順を開始し、実行します。



  • C：\ Windows \ Driver Cacheでドライバーを検索するパスを変更します。
  • 「スキャン」をクリックします。
  • 「デフォルト」をクリックします。
  • [保存]をクリックします。
  • 「完了」をクリックします。






Sysprepミニインストールが完了したので、指定したディレクトリでドライバーを探します。



繰り返しますが、Sysprep.infファイルに変更を加える必要があります-ファイルの最後に[SysprepMassStorage]を追加します。 そして、コマンドラインから「sysprep.exe –bmsd」を実行します。 出力は次のようになります。 

 [SysprepMassStorage]
 pci \ ven_1022＆dev_7007 = c：\ windows \ inf \ machine.inf
 pci \ ven_1022＆dev_700d = c：\ windows \ inf \ machine.inf
 pci \ ven_1022＆dev_700f = c：\ windows \ inf \ machine.inf
 pci \ ven_1022＆dev_740b = c：\ windows \ inf \ machine.inf
 pcmcia \ ibm-pcmcia_portable_cd-rom_drive-84e3 = c：\ windows \ inf \ pnpscsi.inf
 pcmcia \ ibm-pcmcia_cd-rom_drive_cd-400-5afa = c：\ windows \ inf \ pnpscsi.inf
 pcmcia \ ibm-pcmcia_portable_cd-rom_drive-84e2 = c：\ windows \ inf \ pnpscsi.inf
 pcmcia \ pcmcia-ide_card-3220 = c：\ windows \ inf \ pnpscsi.inf
 ...




これらは、ドライバーの識別子とその場所です。



次に、イメージのコンピューターカーネル設定の変更を開始します。 このためには、別のユーティリティMySysprepが必要です。 仮想マシンのC：ドライブの同じSysprepフォルダーにダウンロードして解凍します。 解凍したMySysprep.infを調べて、[CPU]セクションからすべてのコメント記号（;）を削除します。 編集したファイルが保存されます。 ここで、デフォルトの「ACPIを備えたシングルプロセッサコンピュータ」を「コンピュータACPI」に変更する必要があります。 これを行うには、「デバイスマネージャー」を開き、デバイス「ACPI Uniprocessor Computer」のドライバーを更新するオプションを選択します。 オファーの検索を拒否し、自分ですべてを見つけると言い、最後のウィンドウ（ドライバー選択ウィンドウを手動で）で「ACPIコンピューター」を選択します。





その後、C：\ Sysprepフォルダーからmysysprep.exeファイルを再起動して実行します。 プログラムウィンドウで、チェックマーク「ミニインストール」を入れて、「再シール」ボタン（「コンピューターのシール」）を押します。



これで、OSはWDSを使用して「キャプチャ」してイメージを作成する準備が整いました。





仮想マシンが起動したら、BIOSに入り、ネットワークブート（PXE）を設定します。 PXE経由でダウンロードする場合、WDSサーバーイメージを選択するメニューに移動し、WDSサーバーで「キャプチャ」するために使用するイメージを選択します。 ブートローダーを実行した後、イメージを保存する場所（WDSサーバーまたはローカルディスク）を選択し、実際にイメージを作成するプロセスが実行されます。





結果のイメージは、指定されたマシンにWindows XPを自動的に展開しますが、すべてのデバイスは正しいドライバーでインストールされます。 また、さまざまなハードウェア構成を「恐れない」ことも得意です。





上記のすべてのアクションは、展開および構成されたWDSサーバーを備えたインフラストラクチャに関連しています。 また、Light-Touch展開イメージを作成するための複合施設では、WDSサーバーイメージの応答ファイルを作成する可能性について話すこともできます。これにより、イメージの展開や1-2キーの押下（ハードディスクの自動パーティション、WDS-サーバーなど）。 このようなインフラストラクチャがない場合は、その作成が次の記事のトピックになる可能性がありますが、この場合、年表の観点からはあまり正しくありません。
