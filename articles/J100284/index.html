Palm Pre Activation Window Bypass

私たちの緯度でPalm Preを使用する際の最初の困難に戸惑うことなく、それでも自分で購入することに決めた場合は、別の問題（新しいデバイスの購入に関する問題）が発生する可能性があります。



すべてのPalm Preを使用する前にアクティブにする必要があります（ www.palm.comで既存のPalmプロファイルからデータを作成または入力する必要があります）。 このアクションがない場合、デバイスは緊急電話をかける機会のみを提供します（ただし、任意の番号に:)。



「問題は何ですか？」と思われますが、問題は、最初はアクティベーション操作がモバイルネットワーク（GPRS / EDGE / 3G）を介してのみ可能であることです。 これはWi-Fi経由ではできません。 デバイスはメニューで起動せず、Wi-Fiをオンにして設定することはできません。



しかし、最も重要なことは、ご存知のように、パームプレはロシア語/ベラルーシ語/ウクライナ語/および他の多くのオペレーターからのモバイルインターネットの設定については知りません。 はい、 前の記事で行ったように設定ファイルを置き換えても機能しません。 開発モードを有効にすることもできません（デバイスは、プロファイルフィールドに入力する以外のアクションに応答しません）。 したがって、アクティベーション画面を超えて進むことはできません。







この状況から抜け出すには2つの方法があります。

-eBayでデバイスを購入する場合、SIMカードの「ネイティブ」を使用して販売する前に販売者に電話を有効にするように依頼することができます。

-ただし、ここでアクティベートされていないPalm Preを購入し、アクティベートできない場合、プロセスはもう少し複雑になります。



2番目のケースでは、簡単に自分の手で、手順は次のようになります。

-最新のファームウェアファイルをダウンロードする必要があります（同時に、電話で最新に更新します）。

-特別なソフトウェアでそれを変更します（ロード時に最初に使用するアプリケーションの起動を無効にします。これにより、電話をアクティブ化するよう求められ、メニューでさらに進むことはできず、デフォルトのファームウェアで開発モードを有効にします）;

-リカバリモードで、この変更されたファームウェアを電話機にフラッシュします。



別のニュアンスは、ファームウェアを変更するためのプログラムがLinuxとMac OSにのみ存在するため、それらをインストール/検索するか、既に変更されたファームウェアを探す必要がありますが、自分で修正する方が信頼性が高いようです。



Ubuntu 10.04を仮想マシンにインストールし、その中ですべての操作を実行しました。

さらに、実際には、 MetaDoctorの作成者からの英語の説明を私のコメントとともに翻訳し、それを使用してファームウェアを変更します。

  1. ターミナルを開きます。
  2. gitをインストールします（インストールされていない場合）：

    sudo apt-get install git-core



  3. MetaDoctorを配置します。

    git clone git://git.webos-internals.org/tools/meta-doctor.git



  4. パッチを置きます：

    sudo apt-get install patch



  5. そうでない場合は、Java 6ランタイムをインストールします（アプリケーション> Ubuntu Software Center）。
  6. Palm Novacomドライバーをダウンロードしてインストールします。
  7. MetaDoctorがインストールされているフォルダーに移動します。

    cd meta-doctor



  8. 「downloads」という名前のフォルダーがあるかどうかを確認します。 そうでない場合は、作成します。

    mkdir downloads



  9. Palm Preの最新のファームウェアバージョンをここからダウンロードし（最新バージョン1.4.5を書いている時点で、O2ドイツ、イギリス、アイルランド/ Movistarスペインのオペレーターのバージョンが必要です）、「ダウンロード」フォルダーに入れます。

    cd downloads

    wget palm.cdnetworks.net/rom/pre/p145r0d06302010/eudep145rod/webosdoctorp100ueu-wr.jar



  10. ダウンロードしたファイルの名前を変更して、バージョン番号が含まれるようにします。

    mv webosdoctorp100ueu-wr.jar webosdoctorp100ueu-wr-1.4.5.jar





MetaDoctorを起動します。

  1. ターミナルで、メタダクターのあるフォルダーに移動します。
  2. ファームウェアを変更するスクリプトを使用してファイルを開きます

    gedit Makefile





    必要なオプションのコメントを外して保存します。 この場合、BYPASS_FIRST_USE_APPおよびENABLE_DEVELOPER_MODE（144、145行目）：





  3. その後、ファームウェアを再構築します。プロセスには最大10分かかります（5〜6分かかりました）。

    make DEVICE=pre all-wr





更新されたファームウェアバージョンは、「meta-doctor / build / webosdoctorp100ueu-wr-1.4.5 / webosdoctorp100ueu-wr-1.4.5.jar」フォルダーに表示されます。 仮想マシンですべてを行ったので、完成したファームウェアファイルをWindowsに転送し、そこでフラッシュを続けた方が良いでしょう。 「通常の」Ubuntですべてを行った場合、この.jarファイルをすぐに実行できます。



更新されたwebosdoctorp100ueu-wr-1.4.5.jarを開始し（Windowsの場合はJavaも必要です）、言語を選択し、条件に同意します。



電話の電源を切り、PCに接続し、音量アップキーを押しながら、電話の電源ボタンを同時に押す必要があります。 デバイスの画面にUSB接続アイコンが表示され、電話は復旧モードに入ります。 次に、PC画面の指示に従います（最後の段階で、プログラムが電話を認識した場合、[次へ]ボタンがアクティブになります）。 [次へ]をクリックすると、デバイスが点滅し始めます（マイクロサーキットの画像が画面に表示されます）。



10分後、電話が再起動し、空のデスクトップが表示されます。







これで、たとえばWi-Fi設定に移動してネットワークに接続し、初回使用アプリケーション（最初の起動までメニューにハングアップします）を起動し、静かに電話を有効にできます。 地区にWi-Fiがない場合は、モバイルインターネットのセットアップを試みることができます（ ここから、または直接電話で：電話->設定->手動設定、ただし、デバイスがそれらを保存することはありません-これも既知のバグです） ）



PSこれらの操作はアクティベーションウィンドウを削除するだけで、ロックされたデバイスのロックは解除しません。

PSあなたはあなた自身の責任でファームウェアにすべての変更を加えます。
