Nokia N900：「選択した場所に十分なメモリがありません」

昨日、出張から帰宅して、Nokia N900のファームウェアをアップデートすることにしました。 私はそこに立派な老人の前に立っていた。



しかし、これを単純にしようとすると、一般的に、ケースは「選択した場所に十分なメモリがありません」というメッセージをポップアップしました。 どのメモリが足りないのですか？ そのような「場所」とは？ 誰が彼を選んだのですか？ 私？ だから私は選択しませんでした。 General Feylurは再び不正にディスクを読み取りますか？



デバイスを再起動しました-助けにはなりませんでした。



私はインターネットにアクセスしましたが、最初の試行から問題を解決する方法についての指示を見つけました。 幸せになるだろうが、私はさらに悲しかった。 理由を理解できるように、すべての指示を与えます。



rootshパッケージをインストールします（まだ実行されていない場合）

根になる

根



/ usrの内容を2ギガバイトのパーティションにコピーします。この手順の所要時間は、これまでにインストールされたパッケージの数によって異なります。 すべてを破壊する方が良い

cp / usr / home -p -R



コピーしたフォルダーを古いポイントにマウントします

mount –bind / home / usr / usr



再起動後にさらに自動マウントするように、fstabマウントファイルが生成および適用されるスクリプトを編集します

vi /etc/event.d/rcS-late



そこにこの断片を探して、この場所に私たちの魔法のチームを挿入します

＃これで安全に起動を続行できます。

ノーマレク、ハァッ？ ファームウェアを簡単にアップグレードするには、多くの興味深いコマンドを作成し、スクリプトを編集し、何かをマウントする必要があります。正直なところ、私はそのような観点からは絶対的なカテゴリーではありません。 私は長い間ファームウェアを更新していますが、このような不自然な前奏曲で...



興味深いことに、Linuxの愛好家は、このマニュアルに関するコメントを喜んでいます。それは、どれほどシンプルで、どれほど便利かということです。



再び、私はこれらの美しいエイリアンのために根本的に異なる染色体セットについて考えました。



幸いなことに、N900を更新する簡単な方法がすぐに見つかりました。 「アプリケーションマネージャーにアクセスし、ターミナルに入り、ルートをオンにして、apt-getアップグレードを実行します。 質問が表示されたら、「y」と答えます。 ファイルのダウンロードが開始されます。 プロセスの最後に、コマンドプロンプトが表示されます。 再起動して、新しいファームウェアをお楽しみください！」



私が試した-すべてがうまくいった。 ノキアの人々がこのスクリプトのグラフィカルインターフェイスを固定せず、目立つ場所に配置しなかった理由は明らかではありませんか？ 結局のところ、N900は非常に酸性であり、「WindowsおよびMacユーザーは使用を厳しく禁じられています」などの警告サインは見つかりませんでした。
