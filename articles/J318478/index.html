.NETをオープンソースに変換すると、Microsoftに具体的なメリットがもたらされました







近年、マイクロソフトはより開発者に優しい企業になろうとしており、オープンソースコミュニティに積極的に関与しています。 今日、MicrosoftはGitHubで最大のリポジトリの 1つを持ち、フリーソフトウェアの開発に専念しています。 さらに、会社の一部のプロプライエタリ製品は、オープンソースソフトウェアのステータスに移行しています。



この分野のテクノロジー大手の成功の1つは、GitHubでのソースコードの公開により、.NETをオープンソース製品のステータスに移行したことです。 これは、2年以上前の2014年11月に起こりました。今日、会社のこのステップが害よりも利益をもたらしたと自信を持って述べることができます。



開発者の注目をめぐる激しい競争の状況では、従来は外部のMicrosoftに閉ざされていましたが、それ自体は非典型的な一歩を踏み出しましたが、もちろんこの恩恵を受けました。 ASP.NET、.NETコンパイラ、.NET Core環境、フレームワーク自体、およびライブラリがオープンアクセスで投稿されました。 したがって、同社はMacおよびLinuxで.NET開発者と協力することを許可しました。



mattwarren.orgが提供する統計によると、Githubでの.NETソースの公開以来、VS 2012+のアクティブな開発者の数は61％増加しています。







さらに、コミュニティはマイクロソフトと一緒に開発に積極的に関与しています。







コミュニティからのプルリクエストの数はそれほど印象的ではありませんが、コミュニティがアクティビティの平均20〜40％を占めるという事実は、会社の部門の開発者の関心を示しています。







「.NET Coreユーザーの40％は新しい開発者です」 と 、Microsoftの.NET部門のディレクターであるScott Hunter 氏は言います。 「新しい人を引き付けたい。」



オープンソースコミュニティでの経験はマイクロソフトにとってプラスですか？ もちろん。 これは、今年11月に同社が作成したVisual Studio for Macプレビューの最近のリリースによって間接的に証明されています。



スコットハンターは、これが同社のIDEがWindows以外のプラットフォームに移植されるのは初めてだと主張しています。 同時に、IDE Visual Studio 2017の開発には、.NET Coreでオープンソースプロジェクトを操作するための構文の簡素化が含まれます。つまり、同社はできるだけ多くの製品をオープンソースコミュニティに近づけ、それに応じて視聴者を最大化しようとしています。
