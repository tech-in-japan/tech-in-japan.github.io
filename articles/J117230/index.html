Google SPDY：Life 1.1のHTTP 1.1

2011年1月19日以降、 SPDYプロトコル （SPeeDY）のサポートがChromeブラウザーに登場し、Googleサーバー上のSSLトラフィックも有効になりました。 したがって、Googleが所有するすべてのサイトは、他のブラウザよりもChromeではるかに高速にロードされます。 4か月間、ChromeユーザーはHTTP経由ではなく、Googleサイトから多くのページを受け取ります。



これは、IISサーバーがApacheよりも速く応答するようにIEブラウザーの標準を「改善」したマイクロソフトの状況を誰かに思い出させるかもしれません。 しかし、ここでは状況は根本的に異なります。 これは、ある種の独占権を取得することではなく、すべての人に高速インターネットを提供することです 。これは、Googleにアプリオリに有利です。



SPDYプロトコルは完全にオープンです。 対応するプロキシは、任意のサーバーの所有者（たとえば、 Apacheのモジュール ）によって提起でき、任意のブラウザーの開発者はクライアントサポートを実装できます。



ここでは、標準を置き換える代わりに、実験的でオープンな拡張機能が提案されています。 これは、一気に置き換えることのできない広範な標準を改善するための最良のアプローチです。 マイクロソフトでさえ、標準への同様の拡張機能をリリースしたときに、このアプローチの有効性を証明しました（たとえば、XMLHTTPRequestの概念は、Microsoft Exchange Server 2000のOutlook Web Accessの開発者によって提案されました）。



HTTP 1.1の現在のバージョンは、Webがはるかに単純だった1996年に開発されたため、HTTPはHTTP 1.2 SPDYで拡張できる「ボトルネック」になりました。 HTTPの改良バージョンは、リクエストの多重化、リクエストの優先順位付け、HTTPヘッダーの圧縮などのさまざまなトリックにより、Webページの読み込みを44〜64％高速化します。



以前は、SPDYはChromeの個別の実験アセンブリにのみ実装されていましたが、プロトコルはようやく準備が整い、ドラフト標準のグローバルな変更が行われたようです。 SPDYはTCPに基づいており、GoogleはWebの整合性に害を及ぼさないと確信しています。
