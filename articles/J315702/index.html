AndroidおよびiOS用の3CXクライアントのベータバージョン、およびWindows 7での3CXアクティベーションの問題の解決策

今週も良いニュースがあります-AndroidおよびiOS用の新しい3CXモバイルクライアントをリリースしました（現在ベータテスト中です）。



そして今、まず最初に！



Android向けBeta 3CXクライアントバージョン



Android向け3CXクライアントのベータ版の紹介。 彼は、更新されたユニファイドコミュニケーションインターフェイス、改良されたダイヤラーインターフェイス、および完全なロシア語ローカリゼーションを受け取りました。 また、アプリケーションは、ユーザーのアクションに対して非常に高速に応答します。









主な改善点は、組み込みの音声コーデックG729です-無料です！











Android向け3CXクライアントのベータ版を入手するには、 Google Playの3CXベータテスタープログラムに登録して 、新しいクライアントをインストールしてください 。 新しいクライアントをインストールするには、最初に既存のクライアントをアンインストールする必要がある場合があります。



iOS用Beta 3CXクライアントバージョン



Android用3CXクライアントのベータ版のリリースに加えて、iOS用クライアントのベータ版もリリースしました。 注意してください-クライアントはiOS v10用に設計されています！



iOSクライアントには、Androidクライアントに登場したのと同じ新機能があります。 さらに、Apple CallKitのサポートのおかげで、通常のGSM通話と同じ方法でVoIP通話を管理できるようになりました！ たとえば、着信GSMコールはVoIPコールを保留にしません。どのコールを優先するかを決定します。











現在、ベータテストプログラムは、ユーザーが登録フォームを介して送信する最初の1000個のApple IDに制限されています。 ただし、 このニュースのコメントでテスト用の招待を求めてみてください。AppleIDの電子メールを指定してください。



Windows 7 64ビットでの3CXアクティベーション問題の解決



残念ながら、最後の段階でWindowsに3CXをインストールすることを決めた多くのユーザーは、奇妙なアクティベーションエラーが発生し、次のようなメッセージがウィンドウに表示されました。 



license_error.HTTP/1.1 400 Bad Request Cache-Control: private Content-Length: 1647 Content-Type: text/html Server: Date: Fri, 11 Nov 2016 16:24:35 GMT






このエラーは通常、ファイアウォールまたはHTTPプロキシがパケットをフィルターしようとしたときに発生します。 この場合、正しいアクティベーションは不可能です。ユーザーがプロキシをバイパスして3CXをアクティベートするか、一時的にDPIを無効にすることをお勧めします。



ただし、この場合、ユーザーはDPIを使用せず、理由は異なります。Windows7でロシア語の文字を使用したコンピューターの名前です。デフォルトでは、ユーザー名に-PCを追加したものです。







ライセンス認証エラーの原因となるのは、コンピューター名のロシア語の文字です！ 現時点では、図に示すように、 名前からキリル文字を削除して、単にコンピューターの名前を変更することをお勧めします。 名前を変更した後、コンピューターを再起動します。 3CXのインストールを完了するには：



  • コンソールから3CXを構成およびアクティブ化する-C：\ Program Files \ 3CX Phone System \ Bin \ PBXWizard \ PbxConfigTool.exeを実行します
  • Webインターフェイスを介して3CXを構成およびアクティブ化します-C：\ Program Files \ 3CX Phone System \ Bin \ PBXWizard \ PbxWebConfigTool.exeを実行します（Chromeブラウザーを使用）。


次のサービスパックはこのエラーを解決します。
