AWSマーケットプレイス



まず、 Amazon AWS Marketplaceについて少しお話したいと思います 。 モバイルアプリケーションがアップロードされるGoogle PlayやApp Storeとは異なり、開発者はAWSにサーバーアプリケーションを投稿します。 ここでは、無料のLAMPスタックからSAPのようなサーバーベースのエンタープライズソリューションまで、多くの興味深いものを見つけることができます 。



市場と連携することの利点は、導入のスピードと利便性にあります。 原則として、必要なアプリケーションサーバーは既に構成され、最適に構成されており、 EC2インスタンス自体 （Amazon EC2仮想サーバー）の起動時に起動されます。 インスタンスの起動はWebインターフェースから行われ、追加の管理者の知識とスキルは必要ありません。



ただし、操作中にLinux管理スキルが必要になる場合があります。 ただし、インストールと構成のすべての段階をスキップし、トラブルシューティングと熊手でタンバリンを歩く時間を1〜2時間節約して、作業を開始することは間違いなく素晴らしいことです。 その結果、必要な機能が事前に設定され、すでに機能しているライブサーバーを取得します。









後で、SSHを介してこのサーバーにログインし、現在のすべての設定、ファイルシステムなどをチェックすることを妨げるものは何もありません。 その結果、サーバーへのフルアクセスが可能になり、貴重な時間を節約できました。



AWS Marketplaceで作業するには、もちろんAmazon Web Servicesのアカウントが必要です。 幸いなことに、多くの開発者とIT専門家はすでにそのようなアカウントを持っています。 そうでない場合は、 AWSウェブサイトでいつでも入手できます。



WebRTCメディアサーバー



ご存知のように、 WebRTCは、ブラウザおよびモバイルデバイスでオーディオおよびビデオデータをキャプチャ、再生、および送信するための技術です。 WebRTCを使用すると、オンラインブロードキャスト、ビデオチャット、ビデオコール、会議、インターネットラジオなど、 RTC自体が必要な多くのプロジェクト（リアルタイムで低遅延のリアルタイム通信）などのアプリケーションを作成できます。









WebRTCメディアサーバーは、WebRTCテクノロジーの機能を拡張します。WebRTCテクノロジーは、このようなサーバーなしではピアツーピアで機能し 、多くの技術的な制限があります。



メディアサーバーは、次のようなタスクに使用できます。



  • サーバーでのWebRTCビデオストリームの記録
  • ビデオストリームリレー
  • ビデオストリームトランスコーディング
  • WebRTCからRTMPまたはHLSへのストリーム形式の変換
  • ビデオストリームをスナップショットにスライスする
  • IPカメラからWebRTCビデオストリームを再生する
  • など


この記事では、2回クリックするだけでAmazon AWSでWebRTCサーバーを起動する方法と、WebRTC経由で受信したビデオの処理に関連するいくつかの例をテストする方法を示します。

開始後すぐに、記録、中継、RTMPへの変換、IPカメラからの再生など、メディアサーバーに固有のいくつかの機能をテストします。

ウェブ開発



Dashboradで実行したすべての例は、単なるオープンソースのJavaScript + HTMLです。



たとえば、Playerのサンプルコードはこちらから入手できます 。



これらすべての例の作業は、3つの単純なJavaScript関数になります 。



1.接続を確立します。



このコードは、Websocketプロトコルを使用してサーバーへの接続を確立します。 ESTABLISHEDステータスを受け取ったら、セッションオブジェクトでcreateStream（）を呼び出して、ビデオストリームまたは再生を送信できます。 



 Flashphoner.createSession({urlServer: "wss://ec2-34-207-147-235.compute-1.amazonaws.com:8443"}).on(Flashphoner.constants.SESSION_STATUS.ESTABLISHED, function (session) { //session connected, start streaming }).on(Flashphoner.constants.SESSION_STATUS.DISCONNECTED, function () { //display status }).on(Flashphoner.constants.SESSION_STATUS.FAILED, function () { //display status });


2.ビデオストリームの送信



ビデオストリームをサーバーに送信するには、 stream.publish（）メソッドを使用します。 コードからわかるように、最初のイベントハンドラーは作成されたStreamでハングし、最後にpublish（）が呼び出されます。 



 session.createStream({ name: "eabcd", display: localVideo, cacheLocalResources: true, receiveVideo: false, receiveAudio: false }).on(Flashphoner.constants.STREAM_STATUS.PUBLISHING, function (publishStream) { //display status }).on(Flashphoner.constants.STREAM_STATUS.UNPUBLISHED, function () { //display status }).on(Flashphoner.constants.STREAM_STATUS.FAILED, function () { //display status }).publish();


3.ビデオストリームを再生する



ストリームを再生するには、 play（）を呼び出す必要があります。 ストリームは、id = remoteVideoのdiv要素で再生されます。 



 session.createStream({ name: "stream222", display: remoteVideo, cacheLocalResources: true, receiveVideo: true, receiveAudio: true }).on(Flashphoner.constants.STREAM_STATUS.PLAYING, function (playStream) { //display status }).on(Flashphoner.constants.STREAM_STATUS.STOPPED, function () { //display status }).on(Flashphoner.constants.STREAM_STATUS.FAILED, function () { //display status }).play();


したがって、サンプルの開発と適応には、JavaScriptとHTMLの強固な知識が必要であり、これらの機能を既存のページコードとスクリプトに埋め込むことができます。



ビデオストリームを再生および送信するためのスクリプトのWeb開発に関する詳細なドキュメントは、Web Call ServerのWeb SDKページにあります 。



運営



AWS Marketplaceで取得されるインスタンスは、標準のSSH管理方法が適用される通常のAmazon EC2インスタンスです。



SSHを使用してサーバーコンソールにログインするには、たとえばPuttyを使用します



1. [ セッション]メニューに EC2サーバーのIPアドレスまたはドメインを入力します 。









2.メニューの接続/データで標準ユーザー名ec2-userを設定します









3.ファイルへのパスを秘密鍵で指定します。









キー管理は、セキュリティ設定のAmazon AWSアカウント内で行われます。 ほとんどの場合、すでにアカウントを持っているので、少なくとも1つのセキュリティグループを作成し、キーをダウンロードしています。 そうでない場合は、 Amazon AWSのドキュメントを参照して、 秘密鍵を作成/ダウンロードしてください。



次に、Linuxサーバーのコンソールにアクセスして管理できます。









たとえば、上で示した例のあるフォルダーに移動して、そこにあるファイルを確認できます。 特に、双方向ストリーミングの例は次のパスに沿って配置されます。 



 /usr/local/FlashphonerWebCallServer/client2/examples/demo/streaming/two_way_streaming


サンプルファイルのツリーは次のようになります。 



 cd /usr/local/FlashphonerWebCallServer tree -L 1 client2








これらのサンプルはパッケージ化して、ApacheやNginxなどのWebサーバーにコピーできます。



たとえば、次のように： 



 cd /usr/local/FlashphonerWebCallServer tar -czf client2.tar.gz client2








HTTPSを使用してすべてをテストするという事実に注意を払うことは不必要ではありません。 Chromeでは、WebRTCは安全でない（HTTP）ページでの作業を拒否します。 したがって、サンプルをApacheまたはNginxに移植するときは、これらのページにHTTPS経由でアクセスできることを確認してください。
