HPEが構成可能なインフラストラクチャを提供

12月上旬にロンドンで開催されたヨーロッパの発見会議で、HPEはSynergyプロジェクトの最初の結果を発表し、従来のサーバーアプリケーション（ERPシステム）として迅速に（数分で）展開できる「構成可能な」ITインフラストラクチャを構築するソリューションを作成しました、データベース、MS Exchangeメールサーバーなど）、およびクラウドでの作業用に特別に作成された新しいアプリケーション。 Synergyプロジェクトは、HPエンタープライズ製品部門が統合システムで数年間使用してきた統合インフラストラクチャの進化における次のステップです。

HPEの構成可能なインフラストラクチャは、3つの重要な要素に基づいています。



-柔軟なリソースプール：コンピューティング、データストレージ、ネットワーク-さまざまなワークロード（物理、仮想、コンテナ）の展開に単一のインフラストラクチャを提供します。



-テンプレートの使用に基づくソフトウェア定義のロジック。



-HPE Link Infrastructureと任意の管理およびクラウドサービスプラットフォームの簡単で便利な統合を提供する統合API。



構成可能なインフラストラクチャの概念は、来年の第2四半期に出荷を開始する予定のHPE Synergyシステムに実装されています。 Synergyベータテストは2016年の第1四半期に開催され、ロンドンのDiscoverの参加者はフォーラムのデモゾーンでこのシステムを見ることができました。





HPE Synergyを搭載したラック。



HPE Synergyシステムの主要な構成要素は、構成可能なインフラストラクチャの柔軟なリソースプールを構成するコンピューティングモジュールとストレージモジュールを収容するFrame 12000 10ユニットラックシャーシです。 1つの標準サーバーラックは、これらのシャーシのうち4つを収容します。 第9世代のProLiantテクノロジーを使用するCompute Module 480 G9および660 G9コンピューティングモジュールは、ハーフおよびフルハイトの2つおよび4つのプロセッサーソケットを備えたブレードです（最大12 480モジュールまたは6つの660 G9モジュールを1つのFrame 12000シャーシにインストールできます）。 Compute Module 620 G9および680 G9の2つおよび4つのソケット用のコンピューティングモジュールは、重要なアプリケーションに対応するように設計されており、それぞれシャーシの1つのフルハイトコンパートメントを占有します。





ストレージモジュールとコンピューティングモジュール480、660、620、680のコンピュートモジュールを備えたシャーシSynergyフレーム12000。



Storage Module D3940は、Frame 12000シャーシの2つのハーフハイトベイを占有し、シャーシ内のコンピューティングモジュール間で共有され、Synergy 12を介して接続された40台の2.5インチホットスワップ可能な12Gb SAS、6GB SATA、またはSSDドライブを保持しますGb SAS接続モジュール。 1つのシャーシに最大5つのD3940モジュールをインストールし、最大100台のドライブをHPE Synergy 480 Gen9コンピューターに接続できます。





拡張ストレージモジュールを備えたシャーシSynergyフレーム12000。



イーサネットネットワークへの接続用。 コンバージェンス、高速HPE Virtual Connectモジュールを使用して、最大5フレーム12000シャーシを単一ドメインに結合できるため、Top-of-the-Rackスイッチが不要になり、サーバー接続の管理が大幅に簡素化されます。



HPE Synergyは最新のすべてのストレージシステムと互換性がありますが、HPE 3PARには高度な統合と自動プロビジョニングが用意されています。 SANに接続するには、Virtual Connectモジュールまたは「クラシック」ファイバーチャネルスイッチを使用します。



HPE Synergyインフラストラクチャは、Frame 12000シャーシに2つの特別なスロットがある専用Composerサーバーによって管理されます。1組のComposer（プライマリモジュールとバックアップモジュール）は、5つのラック（20シャーシ）を単一システムとして管理します。 Fabric Linkモジュールを使用して相互に接続されたシャーシでは、制御ネットワークは実稼働ネットワークから物理的に分離されています。



Composerは、HPE OneViewテクノロジーを使用し、テンプレートベースの管理コンセプトを実装します。計算機の構成、外部および組み込みストレージシステムへの接続の構成、ドライバーとファームウェアの更新の実行を、Webインターフェイスおよび外部アプリケーションからの要求に応じたユニバーサルAPIを介して実行しますおよびプラットフォーム。

Image StreamerモジュールはComposerの機能を補完し、必要なアプリケーションを使用してオペレーティングシステムのイメージをすばやく作成し、すぐにコンピューターにアップロードできるようにします。 このような画像の準備にはわずか15秒しかかかりません。





拡張Composerモジュールを備えたシャーシSynergyフレーム12000 。



Composerを使用してアプリケーションのニーズに合わせてリソースを正確に再構築すると、計算能力とディスク容量をより効率的に使用できるため、アイドル状態またはハードウェア機能の制限によるコストを削減できます。 HPE Synergyは、リソースの検出、検索、アカウンティング、構成、プロビジョニング、更新、診断のための単一のユニバーサルインターフェイスを実装しています。 このAPIは、VMware vCenter、Microsoft System Center、Chef、Schneider Electric、OpenStack、Docker、HPE Helion、およびAristaとすでに統合されており、将来的にEmerson、Eaton、およびF5と統合する予定です。



HPE Synergyの統合インフラストラクチャは、従来のサーバーアプリケーションの使用を簡素化し、クラウドアプリケーションとサービスの展開を容易にします。 この調整可能なインフラストラクチャでは、単一のインターフェイスを使用して、コンピューティングリソース、ストレージリソース、ネットワークリソースを特定のアプリケーションに必要な構成にすばやく計算できます。
