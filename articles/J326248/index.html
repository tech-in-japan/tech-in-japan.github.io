W3Tech：世界のnginxのシェアは3分の1に上昇し、Apacheは半分を下回りました



さまざまな国で最も人気のあるWebサーバー。 出典：W3Tech.com



最近では、2012年にnginx WebサーバーがMicrosoft IISを追い越し、アクティブサイトの中でインターネットで2位になったことを祝ったようです（nginxは12.18％、Microsoftは12.14％）。 過去5年間で、nginxはその地位を大きく強化しました。 2017年3月のNetcraft統計によると 、同じカテゴリでのシェアは19.67％であり、Microsoft IISは8.95％に低下しました。 前者は45.82％のApacheです。



Nginxは、2017年3月にすべてのNetcraftメトリックでシェアが増加した唯一のWebサーバーでした。 これは、シェアが着実に増加している唯一のWebサーバーでした。





アクティブサイト上のWebサーバーの割合に関するNetcraft統計、2017年3月



Netcraftに加えて、Webサーバーには他の統計情報があります。 たとえば、W3Tech.comは昨日、 非常に美しい「ラウンド」番号を発行しました 。 彼らによると、nginxは現在、世界の3つおきのWebサーバーにインストールされています。



7年前、nginxのシェアはわずか3.9％でしたが、現在は33.3％です。









2010年から2017年の同時期に、Apacheのシェアは71.5％から50％をわずかに下回りましたが、Microsoft IISのシェアは20.6％から11.3％に低下しました。



遷移の統計によると、ほとんどの場合、Apacheからnginxに切り替えます。 逆のプロセスも観察されます。









nginxの実装における世界的リーダーはロシアであり、76.8％の指標です。 nginxの作成者はロシアの独学のプログラマーであり、元システム管理者の Igor Sysoevであるため、これは理解できます。 彼は2004年にプログラムを書きました。 ロシアに加えて、nginxは、アジア、アフリカ、南アメリカの特定の国で、ベラルーシ、カザフスタン、ウクライナで絶対的なリーダーシップを発揮しています。



Apacheは、Microsoft IISが首位にある中国、トルコ、エジプト、およびアラブ諸国を除き、ほぼ世界のリーダーです。



nginxのシェアが増加し、Apacheが減少するのはなぜですか？ おそらくこれは、インストールと構成がより簡単であり、最新のプロトコルがサポートされているためです。 新しいサイトの中で、nginxのシェアは36.4％、Apache-47.7％です。



HTTP / 2をサポートするサイトの76.8％はnginxで実行され、Apacheでは2.3％のみです（ApacheのHTTP / 2モジュールのステータスは「実験的」のままです）。 IPv6をサポートしているサイトでも状況は似ています。サイトの63.6％はnginxで動作し、Apacheでは3倍少ないです。 Nginxは、AngularJS、Vue、js、Reactなどの最新のJavaScriptライブラリを使用しているサイトと同様に、Ruby、Python、Scalaで記述されたサイトの中で一番の選択肢です。 すべての最新の作品はnginxで動作しているようです。



オペレーティングシステム間に明確な傾向はありませんが、Red Hat / CentOS上のサーバーは圧倒的にApacheを使用し、Gentoo上のシステムはnginxを好みます。
