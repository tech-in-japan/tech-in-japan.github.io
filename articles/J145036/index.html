Visual Studio 2012 RCの新機能

Visual Studio 2012、Team Foundation Server 2012、および.NET Framework 4.5の候補リリースのリリースを発表したいと思います。 これらの製品は、 Visual Studio Webサイトからダウンロードできます。



Visual Studio RCのリリースに加えて、Windows 8リリースプレビューの新しい予備バージョンも今日利用可能になりました。 Stephen Sinowskiの公式発表を公開しているBuilding Windows 8ブログをご覧ください。 Windows 8リリースプレビューを今すぐダウンロードできます。

Visual Studio 2012 RCの機能の完全なリストは、 MSDNで入手できます。 この投稿では、ベータ版と比較した主な革新について説明します。 現在の製品生産サイクルに基づいて、これらの変更は、既存の機能と生産性のパフォーマンスを向上させることを目的とした最後の仕上げであり、ベータテスト期間中にコミュニティから受け取ったフィードバックに基づく変更です。 これらの変更の一部は、 ビデオ （英語）で実際に実行されています。



ロゴとブランディング



おそらく、ブランディングに変更を加え、「バージョン11」という名前から離れたことにお気づきでしょう。 RCはVisual Studio 2012と呼ばれています。これは、RTMバージョンが今年リリースされることを意味します。

製品ロゴも変更しました。



設置



ほとんどすべての製品で最初に知っているコンポーネントの1つは、インストーラーです。 ベータからRCにアップグレードする機能が追加されたため、同じバージョンの製品（Professional、Ultimateなど）を使用する場合は、アンインストールする必要はありません。 Visual Studio 2012 RCインストーラーを実行するだけで、古いコンポーネントが削除され、新しいコンポーネントに更新されます。 唯一の例外は、Windows 8を実行している場合です。この場合、OS自体をWindows 8 Release PreviewのバージョンにアップグレードしてからVisual Studio 2012 RCをインストールする必要があります。

また、インストールプロセス全体を改善する重要な変更を追加しました。 多くの人はすでにベータ版の新しいインストーラーインターフェースを見ています。RCバージョンでは、製品のインストール時間が20％短縮されています！

これで、コンポーネントを選択するオプションがあります。 これらのイノベーションを楽しんで、時間を節約してください。





性能



ベータ版とRCバージョンの間で、パフォーマンスを改善するために多くの作業が行われ、製品のほぼすべての側面がカバーされました。 XAML（コンパイラ、ドキュメントの読み込み、デザイナ）の操作に改善を追加し、デバッガと、大きなC ++ファイルを操作するためのコードエディタであるTest Lab Managerのパフォーマンスを向上させました。 実際、これは完全なリストではありません。

いくつかの例を挙げるために、いくつかのメモを公開する予定です。最初のメモは、エディタのパフォーマンスと応答性に関するものです。 パフォーマンスを分析するための内部ツール、実行された製品の変更、およびエディターの全体的な応答性に影響を与えた結果について学ぶことができるこの投稿に慣れてください。 次の投稿では、ツールボックスコンポーネントの改善について説明します。



インターフェース



RCには、製品の一般的なインターフェイスに関連するいくつかの改善が含まれており、ベータ版の後に慎重に分析したフィードバックに基づいています。 これらの変更の詳細については、この投稿で読むことができます。この投稿では、インターフェースの革新について詳しく説明しています。





公開後、多くの人が好む暗いVisual Studioのトピックについて多くの要望が寄せられました。 今週初め、私たちはあなたが会える投稿をすでに投稿しました。



IDE





IDEの改善は、ベータテスト期間中に受け取ったフィードバックに焦点を合わせました。

たとえば、検索の操作、現在のコンテキスト、およびクイック検索機能とファイル内検索機能間のオプションに関するフィードバックを受け取りました。 RCに含まれている拡張機能がリリースされ、Find-in-Filesのコンテキストと設定を自動的に変更せずにクイック検索を実行できるようになりました。

ソリューションエクスプローラーは、いくつかの追加の変更を加えてRC用にデバッグした領域の例です。 ソリューションエクスプローラーのツールバーがオーバーロードされていることがわかったため、IDEコンテキストに既に存在するいくつかのコマンド（コンテキストメニュー、キーボードショートカット、メニュー）を削除し、ソリューションのプレゼンテーションやプロジェクト階層に影響を与えませんでした。

多くのプロジェクトで構成される大規模なソリューションの場合、必要なファイルを検索することは非常に困難であったため、RCには追加のフィルタリングとスコープの制限が含まれ、関連性を高め、重要な作業に集中できるようになりました。 ここで、作業中のファイルのみを表示するようにソリューションエクスプローラーを制限する機会があります。 新しいフィルターオプションには、保留中の変更モードとファイルを開くが含まれます。







さらに、クイック起動のパフォーマンスが改善され、ほぼ瞬時に機能するようになりました。



Metroスタイルアプリ



Metroスタイルのアプリケーションを作成する場合は、Windows 8 Release PreviewがインストールされているコンピューターにVisual Studio 2012 RCを必ずインストールしてください。 RCバージョンでは、Metroスタイルアプリを作成するためのいくつかのアドオンがあります。

JavaScriptを使用したMetroスタイルアプリ





Blendには、カスケードスタイルシートを簡単にする新しい機能が追加されています。 たとえば、CSSプロパティパネルのウィニングルールモードの「並べ替え：ソース」では、現在アクティブな要素とその元の場所を簡単に見つけることができます。







グラデーションと2次元変換を編集する機能も追加されました。







最後に、リファクタリングと全体的なユーザビリティを改善する素晴らしい仕事をしました。 たとえば、コンテキストメニューを使用して、CSSクラス名をすばやく変更したり、要素IDまたはクラス名の新しいルールを作成したりできます。 また、CSSプロパティをCSSルール間で簡単に移動できる順序でCSSプロパティを切り取り、コピー、貼り付けするオプションも用意されています。







Windows 8アプリ開発者ブログで、MetroスタイルアプリのRCの革新について詳しく学んでください。



ASP.NET 4.5





ASP.NET Web Formsは、新しいC＃言語機能とasyncおよびawaitキーワードを完全にサポートするように更新されました。 ページおよび要素のイベントを非同期としてマークできるようになり、.NET 4.5プラットフォームの新機能を使用できるようになりました

MVC 4.0には、「移行」モードを有効にしたEntity Framework 5のサポートが含まれ、プログラマーはデータベースとソースコードのデータスキームを同期できます。







これ以降のASP.NET MVCは、デフォルトでHTML 5でページをレンダリングします。たとえば、上記のモデルは、Webページに次のコードを生成しますid =“ BirthDate” name =“ BirthDate” type =“ datetime” value =“ 5/31/2012 12： 00:00 AM»



Web開発者向けのツール





Web開発に携わっている場合は、RCに含まれているこれらの革新に間違いなく気付くでしょう。 まず、これらはテンプレートの変更です。 MVC4空のテンプレートは本当に「空」になりました（古いテンプレートはBasicと呼ばれるようになりました）。 また、新しいプロジェクトを作成するパフォーマンスが向上しました。







複数のブラウザーをサポートするWebアプリケーションを起動するためのIDEスイッチが追加されました。







アプリケーションを公開するためのメカニズムも更新され、データソースの増分更新モードが追加されました。 革新の完全なリストについては、 Web Development Toolsブログを参照してください。



ライトスイッチ





LightSwitchプロジェクトでは、RCに対する多くの変更と追加も行われました。 3月の最後の日に、私たちはLightSwitchプロジェクトの新しいテーマであるCosmopolitan Shell＆Themeを公開し、これらのアプリケーションに最新のコーポレートルックを提供しました。 それ以来、私たちはコミュニティからの多くの問題と希望を修正しました。 一般的なレビューに基づいて、このアプリケーションテーマはデフォルトで使用されるようになりました。







レビューにも基づいている別の革新は、LightSwitchアプリケーションの認証ページにロゴを追加する機能です。 このウィンドウは、アプリケーションが公開された後に表示されます。 さらに、データ操作のパフォーマンスインジケーターが改善され、クエリ、ページング、グリッド/エディターへのデータの読み込みなど、多くの使用シナリオにプラスの影響を与えました。 LightSwitchおよびRCの更新の詳細については、 blogs.msdn.com / b / lightswitchのブログをご覧ください。



チーム基盤サーバー





ベータ版の後にいくつかのTFS更新があります。 1つ目は、既存のTeam Foundation ServerインストールのSQLバックアップを使用して新しいバージョンにアップグレードする機能です。 2番目の機能を使用すると、アップグレード時に古いプロセステンプレートに基づいて既存のプロジェクトに新しい機能を自動的に追加できます。これは、レビューによるとTFSで最も人気のある機能です。

TFSチームは、Webアクセスインターフェイスも改善し、反復を最小限に抑える機能を追加しました。







TFS関連の更新の詳細については、 Brian Harryのブログをご覧ください。



建築家向けのツール



RCには、アプリケーション構造とコンポーネント間の関係を分析するための新しい機能が含まれています。 これで、コンテキストコマンドを使用したり、依存グラフで関心のあるコード要素を選択して、たとえば、他の型から直接または間接的に継承されたすべての型、または現在選択されているメソッドを呼び出すメソッドを選択できます。







また、上位レベルのコンポーネントとスコープの間に存在する関係を確認し、関係に突入して依存関係のソースを発見することもできます。







インターフェイスに触れると、コンパクトな「リストビュー」モードなど、目的のコンポーネントのさまざまなビューを有効にできるようになりました。
