Parallels Plesk Panel 11.5：新しいバージョンに登場したもの

こんにちは オンエアのParallels Plesk Panel 11.5ホスティングサービス向けのコントロールパネルの新しいバージョン。 ここでは以前のバージョンについて何度も説明しましたが、すべてのコメントに注意深く従いました。 その結果、新しいバージョンでは、ユーザーの要望が最大限に考慮されました。 それでは、11.5で何が変わったのでしょうか？



Php



•各サイトについて、サーバーにインストールされ、Pleskに登録されているPHPの任意のバージョン（独自のアセンブリまたはリポジトリからダウンロード）を選択できるようになりました。 ウェブサイトごとに異なるバージョンのPHPが必要になることが多いため、これについて尋ねられました。

•PHPスクリプトの一部をnginxを使用して起動でき、残りはApacheで動作するときに、「ハイブリッドWebサーバー」モードが表示されます。 同時に、ハイブリッド性はサーバーではなくドメインを設定しています。





サイト設定ページのPHP設定



安全性



•安全なFTPサポート-すべてのトラフィック（パスワードを含む！）が暗号化されたプロトコルを使用して送信されます。

•Linuxで追加の管理者を作成できるようになりました（以前はWindowsでのみ可能でした）。 ユーザーだけでなくユーザーにとっても、パスワード生成は機能します。



移行



•以前のバージョンでさえ、移行条件の予備検証のためのメカニズムを作成し、考えられるすべての問題と制限を特定しました。 しかし、これらのボトルネックについて事前にユーザーに通知し、便利なハンドルを提供して、エラーの多くがすぐに修正できるようにします（必要なサービスの有効化、ディスク領域の追加など）。開始から数時間後にプロセスの途中で。





これは見た目です



•ユーザーは、プロセス中に同じタイプのエラーに関する多数の警告で攻撃されなくなりました。エラーの適切な優先順位が表示され、すべての情報を1つのメッセージに「接着」します。 また、各移行に関連するすべてのログは、/ usr / local / psa / PMM / logs / migration-2013-04-10-19-39-22-557 /のように、日付と時刻を含む個別のフォルダーに収集されます。

•リモートデータベースを使用してサイトを移行できます（以前に発生した「データベースが既に存在する」などのエラーのリスクはありません）。 注意してください。古いサイトとそのデータベースを削除しないでください。一時停止することをお勧めします。

•バックアップ時間を短縮するために、データ圧縮中に利用可能なすべてのプロセッサとコアを最大限に活用するマルチスレッドバージョンのgzip-pigz（=「gzipの並列実装」）の使用を開始しました。 その結果、テストによると、バックアップは実サーバーで3.5倍高速に実行されます。 pigzファイルは、gzipで展開することもできます。

•Windowsの場合：遅延ファイル転送が導入されました。これにより、移行時間と必要な追加スペースの両方が削減されます。

•Windowsの場合：メールデータをネイティブメールサーバー形式（IceWarp、Smartmail、MailEnable）に移行できます-遅いIMAPを放棄しました。 MS SQLに組み込まれているバックアップツールも使用されます（データベースの回復速度が2倍になったため）。



見出し「あなたは本当に尋ねた」



古いバージョンのParallels Plesk Panelの多くのユーザーは、システムで十分な制御が与えられなかったために動揺していました。 お願い：

•データベースユーザーは、同じアカウントから複数のデータベースにアクセスできるようになりました。

•ユーザーのパスワードは自動的に生成されます（ワンクリックで）。

•パネルへのアクセスをメールユーザーにすばやく与えることができます。作成するときは、パネルの対応するオプションをチェックするだけです。 システムでユーザー名を変更することを禁止できます（一部はFTPアクセスを使用します）。

•メールキューは、インターフェイスを介して直接クリーニングできます。

•ディスク使用量に関するデータを追加しました。これにより、「10ギグかかったのは何？！」など、以前に発生した質問が削除されます。



インターフェース



Parallels Panelのユーザーは、常にインターフェイスについて多くの質問をしました。 それは過負荷であり、ユーザーはしばしば混乱し、機能が必要な場所を忘れていました。 これで、ユーザーには必要なもののみが表示されます（不要な/人気のない適切なタブで削除しました）。 したがって：

•すべての重要な（ただし、毎日必要ではない）情報（システムデータ、Pleskバージョンなど）はホームページから削除されました。

•ホームページは、すべてのツールを備えた「サイトとドメイン」タブでした。 すぐにすべてのサイトがアクティブリストの形式で表示されます。 現在は次のようになっています。





サイトが空の場合、アクティブリストにサイトがある最小化されたプレートのように見えます





だから-あなたがダイの下部にある矢印をクリックすると



•各サイトのダイのアクティブリストには、最も要求された機能が表示されます。 サイトが空の場合、パネル自体は、Parallels Presence Builderデザイナーを使用してコンテンツを作成し、Webアプリケーション（同じWordpress）をインストールし、ファイルをアップロードし、データベースを作成します。 コンテンツがすでに存在する場合、リンクがプレートに表示されます。





そして、それはサイトの最小化されたプレートのように見え、その中にはファイルとデータベースがあります



•多くのサイトがある場合、アクティブリストモードは不便かもしれません。 その後、テーブルビューに戻ることができます。

•設定の変更は月に1回しか必要ないため、設定は非表示になりました。

•ユーザーのホームページにある12個の不明瞭なシステムフォルダー（古いバージョンで作業していた人たちにとってまさに本物の痛み）の代わりに、明確な目的で必要なのは3つだけです。

•混乱が生じないように、サイトの設定とPHPの設定を分けました。 同時に、Apache、nginx、IISのほとんどのオプションをパネルで直接設定できるようになりました。

•ファイルマネージャーのインターフェイスをより使い慣れたものに変更しました。フォルダーのツリーがあり、波紋はありません。 ファイルは、ドラッグアンドドロップでアップロードしたり、複数のファイルを一度にアップロードしたり、プレビューしたり、アーカイブを解凍したりできます。





新しいファイルマネージャーインターフェイス-フォルダーツリーとより正確な



SEO



SEOを最適化するために、PHPとWebサーバーの設定を管理できるようにしました。

•エイリアスとwwwドメインプレフィックスは、サイトのカスタム正規名を含むSEO要件に従って設定できます-wwwの有無は問いません。

•ドメイン転送でも同じ-2種類のリダイレクト（永続的な301と一時的な302）から選択できます。

•Webサーバーの適切な中断。 サイトが中断されている場合はコード503が発行され、閉じられている場合は200（以前は、いずれにしても、200とデフォルトページ）が発行されます。 トピックに含まれていない人のために、検索ボットがサイトが消えたと判断する前に（デフォルトページ）、検索エンジンはそのサイトに関するすべての蓄積された有用な情報を「失いました」。



アプリケーションとライセンス



•アプリケーションのインストール（現在、無料と商用の両方で200以上）をワンクリックで起動できるようになりました。ユーザーは多くの質問に悩まされなくなり、すべてのパラメーターはデフォルトで設定されます。

•ユーザーは、アプリケーションの更新が表示されたという通知を受け取り、自動更新を有効にできます。 ホスティング事業者は、サーバーが古いバージョンのアプリケーションをハッキングするリスクよりも、アプリケーションがクラッシュするリスクを好む場合、サーバー上のすべてのアプリケーションを強制的に更新する機会があります。

•アプリケーションの検索：無料と有料の両方をフィルタリングして（以前と同様）、人気で並べ替えることができます。カテゴリだけでなく各サブカテゴリでもアプリケーションの数を確認できます。





快適な変更-各サブカテゴリのアプリケーションの数



•アプリケーションの特定のバージョンを自分でインストールできます（たとえば、プラグインの互換性の問題を回避するため）。

•APS Mail Aspectのサポートが追加されました。これにより、一部のメールアプリケーション（Open-Xchangeなど）を統合できます。



郵便



•古いWebメールクライアントの製造元はサポートを停止したため、人気のあるRoundcube（Linuxバージョン）に切り替え、HordeはIMP 5.0に更新されました。 最後に、私たちが知っているように、あなたにとって重要です-これのために、多くはサーバーの生産をプレビューに翻訳しました。



コマンドラインインターフェイスと管理の利便性



長年、私たちはグラフィカルインターフェイスに焦点を当てていましたが、CLIも作業に便利である必要があることに気付きました。 したがって：

•Linux上ですべてのCLIユーティリティを実行して、ツール、ログ、構成、および管理データベースにアクセスできるユニバーサルユーティリティが登場しました。 自動補完をサポートしています。

•特別なcfgmonユーティリティにより、管理者は重要な構成で行われたすべての変更を追跡し、以前のバージョンにロールバックできます。実際、これは一部の構成のバージョン管理システムです。

•「トレランス」およびAtomicやRemiなどの接続されたサードパーティリポジトリのサポート。
