Windows上のiOS向けのコードベースUE4プロジェクトの開発





iOS開発に関する公式のUnreal Engine 4フォーラムで最もよく寄せられる質問の1つは、公式ドキュメントで必要とされるMac OS Xを購入して使用しなくても、Windowsでコードベースのプロジェクトを開発できることです。 解決策はありますが、設計が不十分であり、試行錯誤のために多くの時間がかかる可能性があります。 ロシア語への翻訳の難しさについて話す必要はありません。 それで、このチュートリアルが生まれました。あなたと共有したいと思います。





まず第一に、それほど多くの事実はない。 UE4では、iOS向けのゲームの開発は、それぞれMac OS XとXcodeでのみ公式に許可されています。 特にわが国では、既存のPCと同等の性能の強力なMacを購入することに同意しますが、これはかなりのオーバーヘッドです。 はい、すべての人が新しいOSおよび開発環境の再トレーニングを望んでいるわけではありませんが、決定した場合、以前の効率レベルに達する前にかなりの作業時間を費やします。



少なくとも現在のバージョンのエンジンでは、OS Xを実行するコンピューター（PCまたはMac）がないと完全に機能しないことをすぐに予約します。 ただし、問題に対する解決策はかなり最適で妥協があります。 すでに、GitHubのソースにアクセスできる（または、有名なトレントトラッカーを使用してダウンロードした）エンジンの最新バージョン（4.2.x以降）のすべてのユーザーは、Mac OS Xを実行するコンピューターをサーバーとして構成する文書化されていない機能を持っていますプロジェクトのアセンブリとコンパイル、およびプロジェクトの開発とデバイスでのテストは、Windowsからのみ行われます。 Macを定期的に見て、それがまだ機能するかどうか、またはコードを絶えずコンパイルすることですべてが失われるかどうかを確認するだけです。



これには何が必要ですか？



既に述べたように、Mac OS X（10.8以降）を実行しているコンピューターが必要です。 使用済みのMacまたはMac Bookを購入するか、仮想マシンでOS Xを実行できます。 OS Xがどのように機能するかは、ユーザー次第です。 例として、現在、安価に購入したMac Mini（Late 2009）を使用しています。 彼は自分のタスクに対処しますが、将来的にはマシンを確実に高速化します（最も重要なのはプロセッサとディスクシステムです）。 eBayでコンピューターを探すのは理にかなっています（もちろん、配信で2〜3週間待つ時間がない限り、私たちにはありませんでした）。



したがって、必要なものすべて、つまり、Windowsを実行している稼働中のPC（7 SP1以降）、およびOS Xを実行しているコンピューター（10.8以降）があるとします。 コンピューターは通常のローカルネットワークに接続する必要があります。 理論的には、提案する方法は次のとおりです。 UE4のソースには、UnrealRemoteToolプログラムがあります。 OS Xで起動することにより、リモートコンピューター（コンパイル、パックなど）から実行するコマンドを送信できるようになります。 この場合、コマンドはWindowsを実行しているPCから送信され、UE4で開発が進行中です。コンテンツの編集、プロジェクトコードの作成などが行われます。 ネットでは、正しい構成の後、通常のPCビルドの場合と同様に、UE4エディターからiOS用のプロジェクトを直接コンパイルおよびパッケージ化できます。 そこから、PCに接続されているすべてのiOSデバイスに既製のipaをインストールできます。 以下は、両方のシステムですべてのプログラムと機器を構成する方法の詳細な手順です。 行こう



ステップ1：まず、UE4のソースにアクセスする必要があります。 非公式のソースから既に入手している場合は、この手順をスキップできます。 Unreal Engine 4の公式に登録された開発者であり、アクティブなサブスクリプションをお持ちの場合は、以下を行う必要があります。 アカウントをまだお持ちでない場合は、 GitHub Webサイトにアクセスして登録してください。 その下に入力します。 次に、 Unreal Engine Webサイトにアクセスします-ログインして、アカウント設定（アカウントセクション）で、対応する行にGitHubアカウントを指定します。 変更を保存します。 次に、GitHubのEpic Games GHページにパスがあります： github.com/EpicGames/-リポジトリのリストでUnrealEngineを選択します。 右上の[フォーク]ボタンをクリックし、選択したアカウントで表示されるウィンドウでGitHubアカウントを選択します。 バージョン4.2以前が取得されていることを確認してください。 その後、パーソナルエンジンブランチのページに移動します。 先に進む前にページの説明全体を読むことをお勧めしますが、ここで行う必要がある主なことは、セクション起動と実行にある2つのアーカイブRequired_1of2.zipおよびRequired_2of2.zipをダウンロードすることです。 ソースとのローカル作業のためにGitHubクライアントをダウンロードすることは残っています。 Windowsの場合はこちらから、OS Xの場合はこちらからダウンロードできます。



次に、オペレーティングシステムごとに、独自の一連の手順を個別に実行する必要があります。 Mac OS Xから始めて、PC上で既にセットアップを完了します。その後、iOSプラットフォーム用に自由に開発できます。

ステップ2：まず最初に、システムを少なくともバージョン10.8、できれば最新バージョンのMaverics（10.9.x）に更新する必要があります。 左上隅で、リンゴをクリックし、[このMacについて]-> [ソフトウェアの更新]を選択します。 AppStoreに移動し、更新が利用可能な場合はインストールします。



ステップ3：次にXcodeをインストールします。 無料です 。 こちらからダウンロードできます。



ステップ4： iOS用のプロジェクトを開発するには、Apple Developer Programに登録する必要があります。 プログラムの費用は99ドル/年です。 開発者のステータスを取得するのは非常に簡単になりました。このためにFAXを送信する必要はありません。 まだ登録していない人は、ここから始めることをお勧めします。 すでに登録されている場合は、ゲーム用の証明書とそれに基づいてプロビジョニングプロファイルを作成する必要があります。また、GameCenter、iCloud、IAP、およびプッシュメッセージを完全にテストできるように、AppIDについて検討することをお勧めします。 これらはすべて、 会員センターで行うことができます。 サイト上の説明ですべてを正確に実行すると、プロセスに時間がかかりません。 プロビジョンプロファイルをOS Xコンピューターにインストールするには、ダウンロードしてダブルクリックで開くだけです。 Xcodeが開き、それで終わりです。 追加のメッセージは表示されません-これは正常です。 コンピューターに証明書をインストールすると、すべてが少し複雑になりますが、それでもなお透明度が高くなります（インポート後にキーチェーンアプリケーションで確認できます）。 唯一のことは、証明書（Apple Webサイトからダウンロード可能）はあるが、証明書の生成に使用された秘密鍵と公開鍵を持たない元のUDK開発者が問題を抱えている可能性があることです。 この場合、AppleのWebサイトにアクセスし、再度指示に従って新しい証明書を作成してから、それに基づいて新しいプロビジョニングプロファイルを作成します。 AppIDは、これらのアクション、および既存のすべてのアプリケーションとプロビジョニング/配布プロファイルの影響を受けません。



ステップ5：証明書とプロビジョンプロファイルをシステムにインポートした後、証明書とその秘密キーをエクスポートして戻す必要があります。 これを行うには、[アプリケーション]-> [ユーティリティ]-> [キーチェーンAccess.app]に移動します。 左上のウィンドウで[ ログイン]を選択し、下の[ マイ証明書]を選択して、右側のリストでインポートした証明書を見つけます。 それを右クリックして、「 エクスポート」を選択します。 最初に* .cer拡張子でエクスポートしてから開き、表示されたキーを選択して、* .p12拡張子を選択してエクスポートします。 これらの2つのファイルと、後でPCのプロビジョニングプロファイルが必要になるため、たとえばUSBフラッシュドライブまたはDropboxに保存します。



ステップ6：次に、GitHubからソースを取得する必要があります。 これを行うには、OS Xターミナルを開きます：アプリケーション-> Unitilites-> Terminal.app。 それなしでも実行できますが、最終的にはより高速ではるかに便利になります。 すぐに理由がわかります。



ステップ7：次のコマンドを入力して、ルートディレクトリに移動します。 



cd /


注：パスワードの入力を求められた場合は、運送業者の不動性に注意を払わずにパスワードを入力してください。 そうであるはずです。 盲目的にパスワードを入力し、Enterを押します。



手順8：ルートディレクトリに新しいフォルダーを作成します。 



 sudo mkdir UE4


手順9：このフォルダーにすべてのアクセス許可があることを確認します。 



 sudo chmod 777 UE4


ステップ10：次に、ダウンロードしたGitHubクライアントをMacにインストールします。 実行して、アカウントでログインします。 左の列には、リポジトリをダウンロードできるアカウントのリストが表示されます：アカウントを選択します。 右側に表示されるリポジトリのリストで、* Your_Account * / UnrealEngineを選択し、[コンピューターに複製]をクリックします。 クローニングフォルダーには、ハードドライブのルートディレクトリにある、以前に作成したUE4フォルダーを指定します。



重要：リポジトリのクローン作成、UE4プロジェクトの作成などのために、すべてのパスとディレクトリを明確に守ってください。そうしないと、あちこちにファイルが見つからないため、プロジェクトが単に組み立てられません。 現時点では、システムはエンジンのソースコードとプロジェクト自体の場所の変更に非常に敏感です。



ステップ11：以前にダウンロードしたzipファイルはOS Xで解凍する必要がないことに注意してください。ここではソースコードのみが必要です。



ステップ12：ソースがコンピューターにコピーされたら、コマンドラインに戻り、次のコマンドを実行します。 



 for (( ; ; )) ; do ./UnrealRemoteTool; done


UnrealRemoteToolユーティリティを起動し、 これから Windows PCからのコマンドをリッスンします。 ループで実行したことに注意してください。 たとえば、エラーのためにユーティリティが突然終了した場合、ユーティリティは再び開き、動作を継続します。 したがって、OS Xコンピューターをいつでもオンにしておくことができ、いつでもそれを使用してiOSプロジェクトをパックできます。 Macが完全にオフになっている場合は、緊急シャットダウン後にオンになるように構成できます。 これは次のように行われます：システム環境設定->省エネルギーは、電源障害チェックボックスの後に自動的に起動をオンにします。 ターミナルを毎回手動で起動せず、forコマンドを実行してUnrealRemoteToolを起動しないようにするには、ターミナルウィンドウで次のコマンドセットを実行します（現在のウィンドウはUnrealRemoteToolからのコマンドを待機しているため、新しいウィンドウを開く必要があります）： 



 cd /UE4/UnrealEngine/Engine/Build/IOS sudo chmod 777 /UE4/UnrealEngine/Engine/Build/IOS touch StartUnrealRemoteTool.command sudo chmod 777 StartUnrealRemoteTool.command open /UE4/UnrealEngine/Engine/Build/IOS


IOSディレクトリを含むFinderウィンドウが表示されます。 そこにStartUnrealRemoteTool.commandファイルを見つけます。 それを右クリックして、メニュー項目で開く-> TextEdit.appを選択します



開いたファイルに、次のコマンドを記述します 



 cd /UE4/UnrealEngine/Engine/Build/IOS for (( ; ; )) ; do ./UnrealRemoteTool; done


一番上の[ファイル]-> [保存]をクリックします。 次に、[システム環境設定]-> [ユーザーとグループ]システム設定を開き、左側のウィンドウでアカウントを選択して、左上の[ ログインアイテム ]タブに移動します。 開いたタブのプラス記号をクリックし、ルートに移動してファイルStartUnrealRemoteTool.commandを選択します。 したがって、システムが起動するたびに、UnrealRemoteToolが自動的に起動し、Macはいつでも使用できる状態になります。



ステップ13： OS Xでは、コンピューターの名前を確認し、ファイル共有を構成するだけです。 [システム環境設定]-> [共有]に移動し、 [ファイル共有]の横にあるチェックボックスをオンにします 。 [ すべてのユーザーにアクセスを許可]を選択します 。 ウィンドウの上部に、コンピューターの名前が表示されます。 [ 編集]をクリックし、表示されるウィンドウで名前を変更して、スペースが含まれないようにします。 たとえば、HGSMacMini。 コンピューター名の末尾の.local後置に注意してください。 一枚の紙にどこかに名前と後置記号を書くと、すぐに必要になります。



窓



ステップ14： Windowsでコードベースのプロジェクトを操作し、エンジン自体をソースからコンパイルできるようにするには、Visual Studio 2013（それ以上ではない）をインストールする必要があります。 無料のExpressバージョンで十分です。 こちらからダウンロードできます。



ステップ15：インストールの完了後、VS2013を1回起動します。 テーマを選択します。 好みの環境設定を選択してください ログインの申し出-ログイン（将来TFS Onlineをバージョン管理システムとして使用する場合に必要）



ステップ16：次に、WindowsにGitHubクライアントをインストールします。 インストール後、アカウントでログインすると、2番目と3番目の手順をスキップできます。 次に、クライアント設定（ギア、右上）を開き、[ クローンパス]フィールドにC：\ UE4と入力し、[ 更新 ]ボタンをクリックします。 これで、インターネットからソースをダウンロードできます。 左上隅のプラス記号をクリックして、[ クローン ]タブに移動します。 OS Xで行われた方法と同様に、左側の列と右側のUnrealEngineでアカウントを選択し、ウィンドウの下部にある[ クローン ]をクリックします。



ステップ17：オプション：バージョン管理システムとしてTFSを使用する予定がある場合、この段階でGitHubで選択したプロジェクトを無効にする必要があります。 クライアントのメイン画面で、左の列にUnrealEngineプロジェクトが表示されます。 それを右クリックして[ 削除 ]を選択します。 次に、GitHubを閉じ、C：\ UE4 \ UnrealEngineフォルダーに移動して、非表示の.gitフォルダーを削除します-削除されない場合は、コンピューターを再起動するか、ロック解除プログラムを使用して削除します。



ステップ18：ソースのコピー後、以前にダウンロードしたzipファイルを交互に開きます（標準のWindowsユーティリティではなく、WinRarなどを使用することをお勧めします）： Required_1of2.zipおよびRequired_2of2.zip-コンテンツをフォルダーC：\ UE4 \ UnrealEngineに解凍します交換。



ステップ19：次に、C：\ UE4 \ UnrealEngineフォルダーで、テキストファイルを作成し、その拡張子を次のように変更します：UE4Games.uprojectdirs-Notepad ++または通常のメモ帳で開き、 `。/`を追加します（引用符なしでのみ、ピリオドとスラッシュ）。 ファイルを保存します。



ステップ20：フォルダーC：\ UE4 \ UnrealEngine \ Engine \ Saved \ UnrealBuildToolに移動し、BuildConfiguration.xmlファイルを開きます（エンコードを損傷しないように、このファイルとそれ以降のすべてのファイルをNotepad ++で開くのが最善です。挿入を慎重に使用してください）。



終了タグの最後（バージョン4.3では、このセクションは構成ファイルの先頭に既に存在します-編集する必要があります）、次のxmlコードを挿入します。 



 <RemoteToolChain>    <PotentialServerNames>        <Item>AlternativeServer</Item>    </PotentialServerNames>    <RemoteServerName>HGSMacMini.local</RemoteServerName> </RemoteToolChain>


HGSMacMini.localを、以前に記録したOS Xコンピューターの名前に置き換えます。 ファイルを保存します。



ステップ21：次に、次の2つのファイルを順番に開きます。



C：\ UE4 \ UnrealEngine \ Engine \ Binaries \ DotNET \ IOS \ IPhonePackager.exe.config

C：\ UE4 \ UnrealEngine \ Engine \ Source \ Programs \ IOS \ iPhonePackager \ Properties \ iPhonePackager.exe.config



セクションに次の行が存在することを確認します（存在しない場合は挿入するか、falseからtrueに修正する必要があります）。 



 <loadFromRemoteSources enabled="true"/>


変更が行われた場合にファイルを保存する



ステップ22：フォルダーC：\ UE4 \ UnrealEngine \ Engine \ Build \ IOSに移動し、ファイルUE4Game-Info.plist（現在は正確にNotepad ++にあります）を開き、CFBundleURLNameプロパティを見つけます。 OS Xにインストールされたプロビジョンプロファイルの作成時に指定されたバンドル値で置き換える必要があります。通常は、「com.AwesomeDeveloper.CoolGame」のようになります-引用符なし。 CoolGameの代わりに、この方法でバンドルを構成した場合（複数のゲームをリリースするために）が存在する場合があります。 これは正常です。 次に、次の行を見つけます。 



 <string>com.epicgames.${BUNDLE_IDENTIFIER}</string>


com.epicgamesをバンドルの値に置き換えますが、今回は意図的にゲームの名前を省略しています。 エンコーディングがBOMなしのUTF-8に一致することを確認してからファイルを保存します。 これは、メモ帳++メニューの[エンコード]をクリックして実行できます。



ステップ23：証明書とプロビジョニングプロファイルを記憶します。 フォルダを残さずに、OS Xにインストールされたプロビジョニングプロファイルをコピーし、UE4Game.mobileprovisionに名前を変更します。



ステップ24：証明書をインポートするために残ります。 [スタート]ボタンをクリックし、検索にMMC.exeという名前を入力して、Enterキーを押します。 表示されるウィンドウで、[ファイル]-> [スナップインの追加と削除...]をクリックします。表示されるウィンドウの左側の列で[リスト]から[ 証明書 ] を選択します。 ダイアログが閉じて、メインウィンドウの左側の列に[証明書-現在のユーザー]項目が表示されます。 それを展開し、個人->証明書フォルダーを選択します。 証明書が表示されるウィンドウの中央を右クリックし、[すべてのタスク]-> [インポート]を選択します。 表示されるウィザードウィンドウで、[次へ]をクリックします。 ディスク上のOS Xにエクスポートされた証明書を見つけて選択します。 [次へ]、[完了]の順にクリックします。 その後、すべてのタスク->インポート->次へを再度クリックします。 ただし、今回は、フィルターから最初に選択して、拡張子が* .p12のエクスポートされた秘密キーを見つけます。 次に[次へ]をクリックすると、パスワードを設定するように求められます。 スキップします。それ以外の場合、UE4でプロジェクトを自動的にパックする場合、アクセス拒否が可能です。 また、[次へ]、[完了]の順にクリックします。 すべての準備作業が完了しました。 MMC.exeを閉じます。 続行の申し出で、拒否します。 これは証明書とは関係ありません。



ステップ25：最後に、C：\ UE4 \ UnrealEngine \ GenerateProjectFiles.batファイルを実行して、エンジンプロジェクトと補助プログラムを生成できます。 スクリプトを実行したら、生成されたUE4.slnを初めて実行します（これはエンジン自体の解決策です）。 上部のプロジェクト設定で、このビデオを最初に視聴した後、 開発エディターとWin64を選択します。これにより、Win64を選択する場所がわからなくなります。 次に、ソリューションエクスプローラーでUE4プロジェクトを選択し、右クリックして[ ビルド ]を選択します。 エンジンは15〜40分コンパイルします。 前述のように、すべてはプロセッサとディスクシステムに依存します。 エンジン自体（UE4プロジェクト）のアセンブリを完了した後、ソリューションエクスプローラーでプログラムフィルターを開き、UnrealFrontendプロジェクトをコンパイルして、出力で同じ名前のプログラムを取得します。 コードのプロファイリングに役立ちます。また、エディターの起動ツールを使用してデバイスにプロジェクトをパックおよびインストールする際のエラーメッセージを回避するのにも役立ちます。



ステップ26：アセンブリが完了すると、完成したUE4エディターを開くことができます。このエディターは、C：\ UE4 \ UnrealEngine \ Engine \ Binaries \ Win64 \ UE4Editor.exeにあります。 最初に、プロジェクトを作成するように求められます。 [ Code Base Project]を選択し、[ スターターコンテンツを含める]チェックボックスをオフにして、ディレクトリとしてC：\ UE4 \ UnrealEngine \を選択します。ディレクトリは非常に重要です。 次に[ プロジェクトの作成 ]をクリックします 。



ステップ27：プロジェクト作成ウィンドウが閉じ、Visual Studio 2013が開き、新しいコードベースのプロジェクトのソースコードが表示されます。 必要なのは、ソリューションエクスプローラーでプロジェクトを選択し、右クリックして[ビルド]を選択することだけです。



ステップ28：プロジェクトをコンパイルした後、UE4Editor.exeを再度実行し、リストからプロジェクトを選択します。 開くと、プロジェクトの基本機能（デバイス上でビルドおよびデプロイする機能）のテストに適したデフォルトで作成されたマップが表示されます。 したがって、このマップをプロジェクトのディレクトリに保存します。 まず、プロジェクトを梱包するための開始カードとして機能します。 次に、[編集]-> [プロジェクト設定]-> [マップとモード]に移動し、[ ゲームのデフォルトマップ]フィールドに保存したマップを指定します。 以下では、 Default GameModeフィールドに、作成してコンパイルしたプロジェクトのGameModeが含まれていることを確認してください。 たとえば、IdolGameMode。 選択されていない場合は、リストから選択します。



ステップ29：設定を終了せずに、 パッケージングセクションに移動し、 プロジェクトの上部セクションを展開します。 それ以外の場合は、プロジェクトをビルドするときに[ 完全な再構築]のチェックを外すと、エンジン全体が再コンパイルされます。これは非常に長い時間です。 完全な再構築は、完全な構築が本当に必要なリリースまたはその他の重要な構築を構築する場合にのみ使用してください。



ステップ30：最後のステップは残り、iOS用にプロジェクトを構成します。 設定で、iOSセクションに移動し、構成ボタンをクリックします。 すべて、プロジェクトはデバイスにパックしてダウンロードする準備ができています。



ステップ31：プロジェクトを* .ipaファイルにパックするには、設定を閉じて、メインエディターウィンドウの[ファイル]-> [プロジェクトのパッケージ化]-> [iOS]をクリックします。 完成したipaファイルを配置するディレクトリを選択し、[OK]をクリックします。 プロジェクトのコンパイルとパッケージングが開始されます。 パッケージの最後に、アセンブリが実行されたプラットフォームの名前を持つフォルダー（この場合はiOS）が指定されたディレクトリに表示され、その中にゲームで切望されているipaファイルがあります。 パッケージングは​​非常に長くなる可能性があり、その進行状況はウィンドウウィンドウ->ログの出力で監視できることに注意してください。 OS Xで弱いマシンを使用している場合、このプロセスには数分から数時間かかることがあります。それはすべて、財務能力に依存します。 しかし、最初のコンパイルとエディター自体のプロジェクトコンテンツのみの定期的な変更の後、弱いMac Miniでもパッケージ化に1分もかかりません。



ステップ32：すべてが順調に進み、アセンブリが成功すると、たとえばiTunesを使用して、デバイスにダウンロードできるipaファイルを取得します。 iTunesにipaを追加します（または、ディスク上の他のアプリケーション用のフォルダーに配置します。そうすれば、彼はそれを自分で確認できます）。 デバイスをコンピューターに接続し、[アプリ]タブでリストからアプリケーションを選択し、[インストール]をクリックします。 デバイスを同期するために残り、デバイスにアプリケーションがインストールされます。



ステップ33：デバイスでアプリケーションを起動する別の方法として（複数同時に実行できます）、エディターのメニューの右側で、 起動ボタンの横にある矢印をクリックし、リストからAll_iOS_On_YourPCなどを選択して、 起動ボタンをクリックします。 プロジェクトは、接続されたデバイスへの梱包とインストールを開始します（Macも作業に含まれます）。



おわりに



以上です。 職場では、Windows PCから直接UE4でiOSプロジェクトを作成できます。 結論として、この方法はコードベースのプロジェクトでのみ機能することに注意してください。 ただし、空のコードファイルを追加するだけで、Blueprintプロジェクトをコードベースに簡単に変更できます。エディターで、[ファイル]-> [コードをプロジェクトに追加]をクリックします。 したがって、プロジェクトはコードベースになり、通常は上記の方法を使用してビルドされます。
