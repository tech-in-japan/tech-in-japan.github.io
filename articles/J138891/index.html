Xcode 4.3：コンパイラーを個別にインストールできるようになりました

先日、更新されたXcodeの変更リストを調べたところ、興味深いことに気付きました。「コマンドラインツール」はパッケージに含まれておらず、設定で個別にインストールされています。 TCPアナライザーとGoogleは、これらの同じ「コマンドラインツール」 を Xcode からだけでなく、 Xcodeからも個別にインストールできることを示しました。 これはユーザーにとって何を意味しますか？ さて、macportsを使用したり、Rubyの最新バージョンをビルドしたりするために、数ギガバイトのXcodeをダウンロードする必要はありません。約160メガバイトの「コマンドラインツール」をダウンロードするだけで、完全なgcc、make、その他のツールチェーンを自由に使用できます。 キャットの下で、クリックする場所に関する指示と、 Tkサポート付きの最新のruby 1.9.3-p125 のアセンブルの例







コンパイラは簡単にインストールできます。 developer.apple.com/downloadsにアクセスして、Apple IDでログインし（お持ちでない場合は無料で登録できます）、「コマンドラインツール」をダウンロードしてインストールします。





実際にはすべて。 使用できます。 たとえば、システムRubyを最新の1.9.3-p125に置き換え、さらにTkをサポートしている場合でも、ActiveTclをインストールし、公式サイトからRubyソースをダウンロードし、フォルダーに解凍し、ターミナルを起動し、解凍したフォルダーで： 



./configure --with-arch=x86_64,i386 --enable-pthread --enable-shared --with-gcc=clang --prefix=/usr make sudo make install
