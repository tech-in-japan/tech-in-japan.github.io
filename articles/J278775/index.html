Raspberry Pi用のDでの記述



Dlang、または単にDは、長い歴史を持つ若いプログラミング言語です。 この名前の言語はかなり前に登場したという事実にもかかわらず、現在D2または単にDと呼ばれているものが最近登場し、その前身にほとんど似ていません。 Dでの記述は非常に便利であり、パフォーマンスはC ++に劣りません。したがって、彼がARMとそのAndroidおよびiOSのモバイル担当者になったことは驚くことではありません。 さらに、モノのインターネットとポータブルデバイスのみに関心が高まっています。

この記事では、Raspberry Piのdlangコードをクロスコンパイルするタスクについて説明します。 これは複雑なことではなく、落とし穴もありません。 この出版物は、一般的なさまざまなデバイス、特にRaspberry PiでDを使用するための簡単なマニュアルです。



ハローワールド



以下が必要です。

  • Linuxコンピューター（私の場合、仮想マシン上のUbuntu 14.04）
  • sshアクセスを備えたRaspberry Pi（Raspberry Pi B +で使用）


Raspberry Piでソースを直接収集できるという事実にもかかわらず、別のより強力なコンピューターでこれを行うことをお勧めします。 Raspberry自体のメモリは少なすぎるため、Raspberry Piでのクロスコンパイルは簡単です。 まず、Raspberry Pi用のdlangコンパイラが必要です。 LDC（LLVMベースのDコンパイラ）とGDC（GNU Dコンパイラ）の2つのオプションがあります。 適切なバージョンのGDCを見つけるのは簡単でした。 ARM用のLDCがあり、使用することもできます。GDCを使用するのは偶然です。

ダウンロードセクションのGDC Webサイトにアクセスします 。

ターゲットarm-linux-gnueabihfプラットフォーム用のx86_64アセンブリをダウンロードします（より慎重に、GDC自体はx86とarmhfターゲットで実行されます。デバイスで実行されるarmのGDCと混同しないでください）。

ダウンロードして、コンソールから解凍します。 
wget http://gdcproject.org/downloads/binaries/5.2.0/x86_64-linux-gnu/gdc-5.2.0-arm-linux-gnueabihf+2.066.1.tar.xz


開梱 

 tar -xf gdc-5.2.0-arm-linux-gnueabihf+2.066.1.tar.xz


便宜上、フォルダの名前も変更しました 

 mv arm-unknown-linux-gnueabihf gdc-arm




永続的に使用するために、gdcを使用してフォルダーを/ usr / binにコピーし、エイリアスgdc-armを作成しましたが、記事の例では、フォルダー内のすべてをローカルで実行します。 そして、ショートカットリンクを作成します。 

 ln -s gdc-arm/bin/arm-linux-gnueabihf-gdc gdc


一般的に言えば、これはすでにRaspberry PIでDコードを収集するのに十分です。 Hello、worldの例を確認しましょう。 したがって、ソースを記述します。 

 import std.stdio; void main() { writeln("Hello, World!"); }


そして、hello.dに保存します。 コンパイルします： 

 ./gdc hello.d -o hello


次に、デバイスにコピーし、そこに行って確認します。 私にとってはこのように見えました： 

 scp hello pi@192.168.1.85/~/dlang/hello


そして、デバイス上で： 

 pi@raspberrypi:~/dlang $ ./hello Hello, World!


タスクをより難しくする-ライブラリのリンク



上記のすべては簡単であり、箱から出して動作します。 それでは、もっと複雑なものを集めましょう。 たとえば、「He​​llo、vibe.d」、つまり、 Webフレームワークvibe.dの基本的なアプリケーションです。 通常のHelloWorldとの1つの違いは、ライブラリをリンクする必要があることです。 この問題でのdlangのクロスコンパイルは、C、C ++およびその他のコンパイルされた言語と違いはありません。 したがって、Raspberry Piでのクロスコンパイルには便利なアプローチを使用できます。

ライブラリは特定のアーキテクチャに必要であり、その最適なソースはRaspberryディストリビューションにインストールされたリポジトリです。 これは通常https://www.raspbian.org/RaspbianRepositoryです。 ライブラリを使用する最も簡単な方法は、デバイスから直接行うことです。 ただ収集するのではなく（非常に悪い考えで、スワップで非常にゆっくりと永遠に）、ファイルを使用するだけです。 sshfsを使用することをお勧めします（こちらからご覧ください： wiki.dlang.org/GDC/Cross_Compiler/Existing_Sysroot ）。 このアプローチの主な利点は、絶対的な汎用性と安定性です。 どの配布キットがインストールされていても、完璧なライブラリが使用されます。 配布バージョンまたはライブラリ間に競合または矛盾はありません。

ビルドマシン自体のパッケージのソースとしてRaspbianを選択しないのはなぜですか
残念ながら、このソースを接続してライブラリを使用しようとする試みはすべて失敗しました。 これらに非常に似た多くの苦痛の後、 answers.launchpad.net / ubuntu / + source / build-essential / + question / 250970アイデアは放棄されました。



腕のためにubuntuリポジトリを利用してみませんか
マイナーな接続の問題があります。 後で接続したり、組み立てることもできましたが、起動時に違法な指示を受けました。 armhfのアーキテクチャにはまだ多くの種類とフラグがあり、Raspberry Piに合ったリポジトリを見つけて見つける必要があります。



Raspberry Piファイルシステム全体をマウントするフォルダーを作成し、環境変数に書き込みます。 

 mkdir rpi echo $RPIROOT export RPIROOT=~/test_pi/rpi/ echo $RPIROOT /home/user/test_pi/rpi/


sshfs経由でマウントします。 

 sshfs -o idmap=user,follow_symlinks pi@192.168.1.85:/ $RPIROOT


libのアーキテクチャと検索パスを指定する必要があるため、レコードを短縮するための別のショートカットを作成します。 gdc-rpiファイルを作成し、必要なフラグを使用してGDCを起動するスクリプトを作成します。 

 #!/bin/bash ~/test_pi/gdc -march=armv6j -mfpu=vfp -mfloat-abi=hard --sysroot=$RPIROOT -B$RPIROOT/usr/lib/arm-linux-gnueabihf "$@"


ここでもう少し詳しく：-march = armv6j -mfpu = vfp -mfloat-abi = hard-これらは、Raspberry Piプロセッサのアーキテクチャのフラグです。 sysroot-デバイスのルート、-B-libを検索する場所、usr / libの同じデバイス上にあります

起動権を追加して確認します。 

 user@ubuntu:~/test_pi$ chmod 777 gdc-rpi user@ubuntu:~/test_pi$ ./gdc-rpi gdc: fatal error: no input files compilation terminated.


すべてが動作します。ビルドマシン上にDUBが必要です（これはビルドシステムであり、1つの依存関係マネージャー用です）。 ここで説明されている便利な方法でダウンロードして設定します 。

単純なvibe.dプロジェクトを作成します。 

 dub init -tvibe.d test_vibe_pi


DUBは、最小限のプロジェクトを含むフォルダーを作成します。 次に収集します： 

 cd test_vibe_pi/ dub build --compiler=../gdc-rpi


出力には、test_vibe_piというファイルがあります。 libcurlのようなものがない場合は、デバイスにアクセスしてすべてを取得します。 過去の実験の後、私はすでにすべてを持っていました。

デバイスにコピーして確認します。 

 cp test_vibe_pi $RPIROOT/home/pi/dlang


Raspberry Piの場合： 

 pi@raspberrypi:~/dlang $ ./test_vibe_pi Listening for requests on http://[::1]:8080/ Listening for requests on http://127.0.0.1:8080/ Please open http://127.0.0.1:8080/ in your browser.


すべて、「Hello、World！」を与える最小サイトは127.0.0.1でのリクエストに：8080 /準備ができています。



デバイスで直接プロジェクトを収集したい人向け



このような弱いデバイスでコードを収集することは悪い考えですが、ソースコードの形式でデバイスにプロジェクトを展開する方が便利な場合があります。 Raspberry用GDCの準備ができました。gdcproject.org / downloadsのARMセクションにアクセスして、armhfバージョンをダウンロードできます。 ほとんどの場合、ビルドにDUBが必要になりますが、サイトからの完成したバイナリはRaspberry Pi B +で開始されないため、今では組み立てる必要があります。

https://code.dlang.org/downloadサイトからソースをダウンロードします （githubから入手できます。これがその方法です）。 開梱して、便利な名前のフォルダーに入れます 

 user@ubuntu:~/test_pi$ wget https://github.com/rejectedsoftware/dub/archive/v0.9.24.tar.gz user@ubuntu:~/test_pi$ tar -xf v0.9.24.tar.gz user@ubuntu:~/test_pi$ mv dub-0.9.24/ dub user@ubuntu:~/test_pi$ cd dub


LinuxでGDCを使用してビルドするための別のスクリプト：build-gdc.shがあります。 彼はgdcがシステム上にあるか、GDC環境変数が設定されていることを期待しています。 変数を使用します。 ショートカットスクリプトのパスを指定して実行するだけです。 

 GDC=../gdc-rpi ./build-gdc.sh Generating version file... ./build-gdc.sh: 15: ./build-gdc.sh: git: not found Running ../gdc-rpi... DUB has been built as bin/dub. You may want to run sudo ln -s /home/user/test_pi/dub/bin/dub /usr/local/bin now.


この結論が得られ、リンクエラーがなければ、Raspberry PiのDUBが機能しています。 デバイスにコピーして確認します。 

 user@ubuntu:~/test_pi/dub$ cd .. user@ubuntu:~/test_pi$ cp dub/bin/dub rpi/home/pi/dlang/dub-test/dub


デバイス上： 

 pi@raspberrypi:~/dlang/dub-test $ ./dub Neither a package description file, nor source/app.d was found in /home/pi/dlang/dub-test Please run DUB from the root directory of an existing package, or run "dub init --help" to get information on creating a new package. Failed to find a package named ''.


DUBが起動し、フォルダーにプロジェクトが存在しないことに気付きました。 私たちには本当にプロジェクトがなく、主な目標は達成されました-DUBはRaspberry Piで実行されます。



Rapberry Piの下でDのプロジェクトを収集できます。 たとえば、スマートホームのサーバーを起動できます。 vibe.dのプラグイン形式でのMQTTのサポートと、既存のCライブラリを使用する機能があります。
