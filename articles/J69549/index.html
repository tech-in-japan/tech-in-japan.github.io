Intelとの最初の週

ご挨拶！ 今日、 ヴィクトリア・ジスリナとアレクセイ・ロガチコフが対談者になりました-私たちは彼らにいくつかの質問をしました。その答えは下にあります。







最初の議論のトピックは、 Lynnfieldファミリーの新しいIntelプロセッサー、32 nmテクノロジー、ハイパースレッディング、SIMD命令、マルチコアとプログラムパフォーマンスへの影響、新しいIntelマイクロアーキテクチャーのアプリケーション最適化機能、キャッシュの使用、データ共有の問題などです。 気軽にディスカッションに参加して質問してください-最も興味深い質問（トピックに関する）およびアクティブな参加者は、注意を払う必要はありません。 理解していない人のためのリンク :)



Vika Zhislina 、Intelソフトウェア会社のコンサルティングエンジニア、2000年からIntel。



美しい女性がそのような技術的なことをするのはどうしてわかったのですか？



私は巨大なコンピューティングキャビネットの中で育ちました。これらはEUおよびBESMシリーズのコンピューターでした。 私の父は数学者であり、彼の研究所ではそのような技術があったので、プログラミングに非常に興味があったことがわかりました。 さらに、大学を選んだときに物理学を学びました。レーザー物理学の学位を取得してNSUを卒業しましたが、同時に勉強しながら、プログラミングが私にとってさらに興味深いことに気づきました。 だから大学を卒業したとき、私はほとんど偶然にオフショアプログラミングに携わる小さな会社で働くようになりました。 この会社はすぐにIntelに買収されました。



Intelで何をしていますか？



最初の数年間、私は非常に大規模なプロジェクト「Synthetic MPEG4 Video」に関与していました。 アイデアは、人々の3次元モデルを自動的に作成することです。たとえば、ビデオ会議中に、人物の画像を送信することはできません。普通のウェブカメラは、その人が何をしているかを認識します-笑顔、点滅、または何かを言って-これらのアクションのパラメータのみが別の対話者に送信され、その後、合成3次元モデルが繰り返されます。

あなたは知っている、そのような良いフレーズがあります：「誰がどのように行うかを知っている-誰が、どのように知らない-他の人を教える」。 ある時点から他の人に教えています。 私の役職はシニアアプリケーションエンジニアです。 エンジニアには2つのタイプがあります。一部は、ソフトウェア製品を作成するソフトウェアエンジニアであり、私はインテルの外部クライアントと連携するコンサルティングエンジニアであるアプリケーションエンジニアです。

以前は、Intelは周波数が増加する新しいプロセッサを作成し、開発者はこれらのプロセッサで自動的に高速に動作する新しいプログラムをリリースしましたが、状況は変わりました-プロセッサの周波数を上げる技術はほぼ限界に達しましたが、新しいプロセッサ技術が登場しました-たとえば、マルチコア 現在、開発者はプログラムを新しいIntelプラットフォームに適応させる必要があります-これが私がしていることです。 もちろん、私だけではありません-Intelには私たちがたくさんいます。誰もが自分の専門分野を持っています。 当初、私はXScaleモバイルデバイス用のプロセッサに従事していましたが、2006年にIntelがそれらを販売し、今では他のタスクに集中しました。 まず、グラフィックスの経験が豊富なので、既存の統合されたIntelグラフィックスカードと、Larrabeeというコードネームの新しいディスクリートグラフィックスアダプターを扱います。 第二に、そしておそらく、「主に」、マルチコアおよびマルチスレッドのプログラミングに関連するさまざまな問題に対処します。



勤務日はどのように構築されていますか？



無料のスケジュールに従って構築されています-私はそれにとても満足しています。 私はいわゆるアカウント、つまり私が働いている外部のクライアント企業を持っています。 彼らは現在のタスクについていくつかの質問を持っています-私たちは対応し、電話の橋と個人的な訪問でそれらに同意します、私のタスクはインテルの技術に問題がないようにプログラムをデバッグするのを助けることであり、したがって、私たちの技術はサードパーティの開発者のソフトウェアに問題がありました。 もちろん、私も同僚とやり取りします。通常は電話のブリッジを介して集まり、いくつかの問題について話し合います。



オフィスにはかなりゆったりしたユニフォームがあります。 そして、あなたの周り、おそらく、男性プログラマーはたくさんいますか？



はい、正確に。



そして、あなたは服を着る...



ご存知のように、非常に異なる方法です。 私は一般的に異なり、変更したいです。 ここで、私の夫もIntelで働いていると言わなければなりません。彼はIntelツールの主要な開発者なので、特別な自由を買う余裕はありません。

アレクセイ、どうやってインテルに行きましたか？



Intelに入社する前は、基本的に勉強していました。 概して、私は大学を卒業する前に採用されました。 面白い事実：私はモスクワ出身ですが、ニジニ・ノヴゴロド研究室で私が応募した職位が明らかになりました。 当初、移動のオプションは考慮されていませんでしたが、興味深い話が起こりました。 私はニジニのインタビューに来ましたが、特に仕事に就くことは期待していませんでした。霧のあるアルビオンの2人の代表者からの一連の質問の後、そこでダイニングルームに行きました。 神経は、あなたが知っている、穏やか。 支払いは私の番であり、現金の容疑者の背後にいる少女は私の五百ルーブル紙幣と変わらなかった。私には他のお金はなかった。 その瞬間、私の状況が膠着状態であるのを見て、すぐに彼の職場に駆け込んで戻ってきた男が私に並んで立っていたことに非常に驚きました-結局、私は昼食なしに留まらず、潜在的な同僚の最初の肯定的な印象を受けました。 そのため、2週間後、人事部から電話があり、申し出があったとき、私は長い間ためらいませんでした。



そして、あなたはモスクワに戻りましたか？



そうです-ニジニではサーバーグランドに従事していました。基本的に、顧客が技術的な問題を解決するのを手伝い、1年半後にモスクワオフィスの別の職に移りました。



つまり？



今、私は3つの主要な活動分野を持っています。 1つ目は、地元企業の新しいプラットフォームとテクノロジーへの移行です。 特に、私の仕事は、新しいプラットフォームが登場したとき、その機能、移行がもたらす利点、新製品が会社のラインアップにどのように適合するかを顧客に知らせることです。 顧客は、製品の公式リリースのかなり前にエンジニアリングサンプルをテストできる必要があります。 もちろん、テスト段階にあるほとんどすべての新製品では、解決する必要のある予期しないタスクが発生します。非常に簡単に解決できるものもあれば、工場の同僚の介入が必要なものもあります。 そのため、プラットフォームの立ち上げまでに、インテルが提供するビルディングブロックに基づく既成のソリューションを発表とともにパートナーが提供できるよう、多くの作業が行われています。

2番目のタスクは技術的なPRです。特に、インテルが次世代プロセッサやその他の製品に搭載しているもの、興味深いと思われること、およびその仕組みをより詳細に明らかにするためにジャーナリストと協力します。大企業を批判するのは常に素晴らしいことですが、新製品にどれだけの労力が投資されたか、市場に出て店頭にたどり着くために克服しなければならない困難を理解するために、開発者に対する敬意の態度、経験、知識、仕事、勤勉さが生じます。 また、質問への回答の結果、肯定的な記事またはレビューが表示され、特定の詳細について詳しく説明されている場合、今回は自分のタスクに対処したと想定できます。



3番目の領域は主にエンドカスタマーに関連しており、外国の領土は第三者企業の内政であるため、すべての情報が公開されているわけではありません。 私は時々、競争上の地位を評価し、既存のタスクを背景に正しい構成を選択し、提案​​と正当化を準備することに惹かれます。







アレクセイ、聴衆に何を伝えることができますか？



つい先日、Intelは以前Lynnfieldとコードネームされていた新しいプロセッサの出荷開始を発表し、Intel Core i7 8xxシリーズおよびIntel Core i5のブランド名で販売されます。 これらのプロセッサは比較的最近のトピックです。

そしてもちろん、サーバー、デスクトップ、モバイルプロセッサ、システムロジックセットの現在のラインアップについて質問がある場合は、 Intelテクノロジー、これらのテクノロジーの目的、および実装-これらの質問に答えようとします。



私たちがプロジェクトエリアを利用する場合、一般的な計画を選択します。 政府機関であろうと銀行であろうと、エンドカスタマーで行われるプロジェクトは常に別の話です。 そして、これは私たちの歴史だけでなく、お客様の歴史でもあります。 したがって、機密性を維持しながら、私たちの参加がどのように見えるか、どの段階で、私たちが正確に何を支援できるか、どの部門とやり取りするか、プロジェクトがどのように共有されているか、各当事者が責任を負っています。
