Firefox 5がCSS3アニメーションのサポートを開始する可能性が高いです。 （そして、開始しない場合は、Firefox 6にアニメーションが表示されます。）

WebKitベースのブラウザー（SafariやChromeなど）では、比較的以前（ 2009年の初め ）に、特定のCSSプロパティを周期的に簡単に公開できるドラフト標準「 CSS Animations Module Level 3 」で説明されている機能のサポートが実装されましたJavaScriptに頼らずに、サイトの作成者が設定した規則に従って変更します。



昨日、2011年4月12日（ガガーリンの飛行50周年）のバグ435442のパッチがmozilla-central リポジトリに 「上陸」したため、FirefoxでCSSアニメーションのサポートが登場しました。



どの Firefoxで登場したかを推測するのは少し難しいです。 確かに、昨日と同じ日に、 誰もが知っているように、Firefox 6の作業が始まり、Firefox 5のコードはmozilla-centralからmozilla-auroraブランチに切り替わりました 。 それでは、Firefox 5ではCSS3 Animationsのサポートを期待していますか、それともFirefox 6ではサポートしていますか？



Firefox 5で それを信じる理由があります。



まず、 バグ435442のパッチの「着陸」の時間（ バグの対応するコメントは00:27:05 PDTの時間でマークされます）とFirefox 6の出現の時間（対応する差分は 06:51:00 -0700の時間でマークされます）の手作りの比較に依存しています太平洋夏時間（ PDT）は UTC-7とちょうど一致するため、Firefox 6はCSS3アニメーションと比較して6時間以上遅れて登場したことがわかります。



同じ結論（Firefox 5）は、サイト "Can I use"からも出ました 。



これからは、（WebKitとFirefoxの両方からの）デュアルサポートにより、CSSアニメーション標準の将来のドラフトが以前よりもさらに輝かしくなります。
