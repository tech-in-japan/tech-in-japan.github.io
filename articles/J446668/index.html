PythonがWeb開発に適している理由



Valentin Dombrowski：なぜPythonはWeb開発に適しているのですか？ たとえば、PHPやJavaScriptを使用しないのはなぜですか？



グリゴリー・ペトロフ：だから本当に選択肢はありません。 最新のWebでは、バックエンドなしで（実際にはJavaScriptのフロントエンドテクノロジのみで）単一ページアプリケーションまたはプログレッシブWebアプリケーションをアセンブルすることは事実上可能ですが、それでも複雑で、インデックス付けが不十分で、クールな開発者が必要です。



Webサイトまたはサービスを作成する場合、組み合わせたアプローチを使用します。ロジックを実装し、Webページを作成する何らかの種類のバックエンドがあり、フロントエンドはこれらのWebページをブラウザーに描画します。 そして、何かのためにすべてをすばやくまとめる必要があるとき、あまり選択肢はありません。



オプションを見てみましょう。



  • C＃ Microsoftは本当に素晴らしいです。彼らは.NET Coreを作り、あらゆる方法でそれを推進しています。 しかし、第一に、これは新しいクロスプラットフォームテクノロジーであり、すべてが順調に進んでいません。 第二に、それは本当に高価であり、C＃開発者はほとんどいません-単に人気がないからです。
  • Java これには注意が必要です。 Javaで通常のサイトを作成するのは、Pythonのような10行のコードではありません。 これは多くのコードであり、これらはフレームワークであり、Javaサーバーのセットアップの詳細を知る必要があります。 一般的に、継続的な痛みと苦しみ。
  • PHP 最近のバージョンでは、素晴らしいです。 PHP 7.2はPythonよりも悪くはありません。 しかし、PHP 7.2をそのまま使用することはできません。 通常のトップではない開発者がPHPでサイトを作成する場合、彼は7.2だけを書くわけではありません。まだいくつかの教科書やチュートリアルをどこでも読む必要があります。
  • JavaScriptおよびNode.js。 1つの言語がフロントエンドとバックエンドの両方にある場合、これは素晴らしく非常にモダンです。 あまり安定していません。 Node.jsは良いことですが、クラッシュせずに安定して動作するように、実稼働環境にデプロイするのは問題です。 さらに、高品質のJavaScriptコードを作成する場合は、JavaScriptではなくTypeScriptが必要です。 しかし、TypeScriptは予想外に複雑で、一見すると普通の開発者が立ち上がっています。


Ruby、Haskell、Erlang、その他のニッチなものを省いてみましょう。Pythonがあります。 一貫した構文、統一された標準ライブラリ、最高のドキュメント、人気のある軽量フレームワーク、大人気のDjangoを組み合わせた言語。
最も幅広い選択肢があるにもかかわらず、トップの開発者ではなく普通の開発者がいる場合、私たちは普通のサイトを作りたい普通のビジネスであり、50人の開発部門はなく、Pythonを採用しています。



職業に入るために必要な知識



Zlata Obukhovskaya： 1つのフレームワークをよく知っている必要があると思います-そして、他に何が起こり、いつ使用されるかを知る必要があります。 Tornado、Django、Flask、aiohttpなどがあります。

プロトコルなどが存在することを知っておくと便利です。 特に、httpプロトコルの知識は、Webアプリケーションを構築するための中心です。



それでも、少なくともおよそ、フロントエンドがWebプロジェクトでどのように配置されるかを想像する必要があります。HTML、CSS、JSとは何ですか。



Alexey Shtyrnyaev：ドキュメントがどこにあるかを知ってください。 これが最も重要なことです。



グリゴリー・ペトロフ：ここでは、非常に不安定な土壌を踏んでいます。 幸運でなく、どういうわけか現代のフロントエンドを真剣に研究し始めた場合、Pythonバックエンドよりも約10倍難しくなります。 初心者の開発者は、HTMLの学習を開始するためにフォーカスを制限する必要がありますが、これらのすべてのdiv、span、float、すべてがどのように配置され整列されるかには該当しません。



Alexey Shtyrnyaev：ブートストラップの基本コースが必要です。 そして、HTMLの基本。

最初の年は、JSフレームワークを掘り下げるべきではありません（バックエンドに焦点を合わせている場合）。 基本的なBootstrapコースには既製のモジュールが既にあります。スライダーが必要な場合はスライダーを作成し、フローティングメニューが必要な場合はフローティングメニューを作成します。
Zlata Obukhovskaya：フロントエンドの研究は、特にWebアプリケーションに一般的に静的が与えられる方法に没頭できると思います。 そのため、開発者はスムーズに進み、原則として、Webアプリケーションのアーキテクチャがどのように配置され、本番環境でどのように動作するかを学び始めます。



Grigory Petrov：はい。開発バックエンド言語としてPythonを選択し、たとえばフレームワークとしてDjangoを選択した場合、すぐにお勧めします。DjangoのドキュメントはDjango Bookにあります。彼女は初心者には本当にいいです。



Alexey Shtyrnyaev： Djangoを勉強することが目的であれば、一部のDjango Girlsもクイックスタートに適しています。 これはそのようなチュートリアルであり、1日で2階に移動し、基本事項とフレームワークの能力を理解できます。



Valentin Dombrovsky：ポッドキャストを記録する準備をして、Web開発にPythonプログラマーが必要とするもののリストを作成しました。



Python Web開発の基礎に含まれるもの



  • WebフレームワークDjango、Flask、aiohttp、Tornadoなど（および残りの存在について知っている）。
  • プロトコルとAPI：主にHTTP、JSON-RPC、プロトコルバッファー、gRPC。
  • ORMと移行、リレーショナルデータベース、SQLAlchemy、SQL、PostgreSQL、MySQL。
  • HTML、CSS、Bootstrap、JSフレームワークおよびJQueryの基本。
  • 実動、テスト、単体テスト、自動テスト、バージョン管理システム、gitのアプリケーションの原則。


ジュニアにはアルゴリズムが必要ですか



Zlata Obukhovskaya：最初は、アルゴリズムを知る必要はありません。長い間開発に携わると、アルゴリズム自体が徐々に頭に浮かび上がります。 アルゴリズムの正式な正式なコースを持っていなかった優秀なエンジニアの集団を知っています。



グリゴリー・ペトロフ：火に油を加えたい。 それでは、アルゴリズムへの渇望はどこから来たのでしょうか？



今、私たちはアルゴリズムの基礎的な教育を受けておらず、プログラマーを訓練する方法がわかりません。技術的な基盤はありません。



彼らはそれをしようとしていますが、ここにはホグワーツの物語があります。ウィザードが1つになるまでウィザードの学校を作ることはできません。 したがって、大学は「プログラマーのトレーニングを開始します」と言っても、プログラマーはいません。誰もがMail.ru、Rambler、Yandexで働いているからです。



大学で彼らは見て、言う：

-OKプログラミング。 ある種の関連する知識分野を見つけ、そこから専門家を招待しましょう。 テキストを書くことができるジャーナリスト、電気回路を作成できる電気技術者、アルゴリズムを作成できる数学者を招待しましょう。



最終的に、レンガとセメントが素粒子で構成されているため、建築者に素粒子の物理学を教えるのと同じくらい便利であることがわかります。



同時に、彼らはセメントとレンガ自体について話さない。なぜなら、建築者を訓練しようとしている物理学者は家を建てる方法を知らないからだ。 その結果、「セメントがどのように機能するか」を完全に描くことができるが、彼を見たことがなく、それから何かを作る方法を知っているビルダーを取得します。



アルゴリズムとデータ構造は非常に優れていますが、これは非常に小さなアプリケーション領域です。 たとえば、ゲームエンジン、コンパイラ、ネットワークプロトコルを作成する場合、これらは緊急に必要です。



ほとんどのプログラマーは、アルゴリズムとデータ構造が不要なビジネス上の問題を解決します。




そこで最も難しい数学は、2回折り畳んでから割り算することです。 彼らは完全に異なる知識を必要とします。 ビジネス上の問題を解決するには、基本的な知識ではなく、主に応用された知識が必要です。



初心者の開発者にとっては、ビジネスについてのアイデアと、既製のブロックから必要な構造を正確かつ迅速に組み立てる方法、それらをデバッグする方法、バラバラにならないことを保証する方法、バラバラになった理由、要件が変更されてプログラムが開始されたときに何が起こるかを把握する方が良い雨の後の家のように、「基礎の上に落ち着く」。

これらは応用されたものであり、ソフトウェアの書き方の理解です。 彼は、デバッガーに加えて、プログラムがスローダウンする場所を正確に示すツールのセットを持っていることを知っておく必要があります。



Valentin Dombrovsky：この比較は私に起こりました：これはビジネスの言語からコンピューターと通信できる言語への翻訳です。 つまり、プログラマーは特定の言語学者の一種です。



Grigory Petrov：ビジネスには、言語学者ではなく作家が必要です。 作家は何千年前にこの言葉がこれに変わったのかを知る必要はありません。 彼はこれらの言葉を適用できる必要があります。



開発者として最初の仕事を見つけるために必要なもの



Alexey Shtyrnyaev：おそらく、後輩を準備する必要がある普遍的なレシピはありません。

ある会社に来ても、Django、JSON、およびいくつかのアルゴリズムを知っているという事実にとらわれません。 この会社が今ここで必要とするスキルにおそらくとらわれるでしょう。
多くの企業があり、誰もが異なる要件を持っています。 履歴書をさらに準備し、仕事を見つけるために取得する必要がある知識のような普遍的な量はありません。



グリゴリー・ペトロフ： VoxImplantで複数のジョーンズを探していたとき、テクニカルディレクターは次のように基本的な要件を策定しました。人は問題を解決できなければなりません。 ジューンがこれを常に効率的に行うのではなく、最良の方法ではなく、常に正しく実行するわけではないことは明らかです。 これは、雇用主が主に求めているスキルです。



Zlata Obukhovskaya：仕事を探している人、他の地域から引っ越している人は、すでに何らかの方法で問題を迅速に解決できるため、ビジネスの観点からいくつかの利点があります。 これらはソフトスキルです。私はそれを仕事文化とさえ呼びます。 多くの場合、この仕事文化の大学院生はまだ開発されていません。



しかし、私はまだ初心者のためにいくつかのレシピを与えたいと思います。



初心者開発者向けの最初のステップ



Zlata Obukhovskaya：最初は、履歴書に何かを書いて最小限のポートフォリオを示す必要があるため、まだ独自のプロジェクトの一種です。 これらのプロジェクトが自分のためではなく、フリーランスで-誰かのために作られたほうがクールです。



最初のプロジェクトの後、あなたはすでに履歴書を作成し、6月の職があるすべての会社にそれを送ることができます。 インタビューは、企業が必要とするものの理解を与えます。 遅かれ早かれ、誰かが少なくとも小さな会社に連れて行ってくれるでしょう。 その後、この実務経験により、会社に参加する機会がますます増えます。



Valentin Dombrovsky：ちなみに、私たちは学生が10週間のトレーニングのために独自のプロジェクトを持つようにコースの準備をしています。 さらに、チーム開発のスキルをトレーニングします。 これらは、ズラタが語ったソフトスキルです。



Alexey Shtyrnyaev：経験から、私はあなたが非常に長い間最初の仕事を探すことができると言います。 1か月か2か月を探しているとき-これは正常です。 すべての会社に履歴書を提出する場合は、面接に行ってください。3か月目には間違いなく何かが見つかります。



Valentin Dombrovsky：あなたは自分のプロジェクトをカットしたり、フリーランスで簡単なプロジェクトに参加したり、同時に履歴書を送ったりすることができます。



Python開発者の展望は何ですか



Zlata Obukhovskaya： Python開発者はどこへでも行くことができます。 あなたはテストに行き、上級アーキテクトに開発を続けることができます。 または管理でも。 テクニカルマネージャーは異なり、トップマネジメントに成長できます。 データサイエンス、DevOpsで開発し、自動テストまたは機械学習にアクセスできます。



Valentin Dombrovsky：一般的に、私たちのコースを含む多くのオプション、機会もあります。 入り口ではそれほど多くの知識は必要ありませんが、より広い範囲をカバーすることをお勧めします。
