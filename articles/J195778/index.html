スマートBLFとDigium D40 IP Phoneの比較



これで、アスタリスクがさらに柔軟で機能的になります。 Digium IP Phoneを追加するだけです;）



製造業者は、番号1.4の新しいファームウェアと新しいDPMAモジュールをリリースしました。



アスタリスクとの特別な接続のおかげで（SIPプロトコルだけでなく）、この電話を使用すると、電話で直接同僚のステータスを確認したり、キューに出入りしたり、ボイスメールを聞いたり、独自のアプリケーションを起動したりできます。



今日、Digiumは別のコンセプト「スマートBLF」を導入します。 IP電話D40、D50、およびD70の短縮ダイヤルキーの動作を説明しています。 以前の短縮ダイヤル機能と比較して、Smart BLFはまったく新しい可能性の世界を提供します。



どれに興味がありますか？ そして、あなたはこのユニットが他に何のためにとても良いのか知りたいですか？ habrakatの下に進むようお願いします。

スマートblf



ワンタッチ転送


電話中にどこかに通話を転送する場合はどうなりますか？ これで、短縮ダイヤルキーをワンタッチ翻訳モードに設定し、文字通り「ワンクリック」で行うことができます:)さらに、従業員、キュー、駐車場の場所は関係ありません。 これは、着信に応答していない場合にも機能します。



通話中のDTMF


features.confで構成されている組み込みのアスタリスク機能を使用していますか？ おそらく、既存のものを使用します：会話の録音、駐車、またはブラインド翻訳。 Smart BLFでは、通話中にDTMFを直接送信するように短縮ダイヤルキーを設定できるようになりました。 たとえば、突然、「当社にご連絡いただきありがとうございます」または「サービスの質については、blablabla @ email.comまでご連絡ください」というwavファイルを再生したいと思います。 電話のオペレーターは、ワンクリックでこれを行うことができます。



条件付きダイヤル


IP電話の動作や連絡先に応じて、同じ短縮ダイヤルキーを押して電話に異なる番号をダイヤルさせますか？ 今、あなたはこの機会を持っています。 スマートコール転送を簡単に設定できます。 連絡先がスタンバイモードの場合、短縮ダイヤルボタンにはブラインドトランスレーションの機能があります。 従業員が突然忙しい場合、まったく同じキーを使用して同僚にコールを転送できます。



もちろん、あなたはクレイジーなダイヤルプランでこれを行うことができます。 しかし、なぜそんなに複雑なのでしょうか？



LED制御


スマートBLFでは、緑または赤のインジケータをユーザーに割り当てることができ、さらに、連絡先のステータスの点滅頻度を設定できます。



アプリケーションのダウンロード


1つのボタンで、ユーザーは独自のアプリケーションを起動できます。 任意の短縮ダイヤルキーを割り当てると、アプリケーションがすぐにフォアグラウンドに表示されます。 メニューに移動し、使用可能なすべてのアプリケーションのリストを表示して、必要なものを実行する必要がなくなりました。



関数を組み合わせて、たとえば条件付きのセットと組み合わせると便利です。 転送キーを押しましたが、同僚は忙しいです。 電話では、（CRM会社からの）発信者に関する追加情報がすぐに開くか、忙しい従業員のカレンダーが開くかは関係ありません。 必要な情報を即座にクライアントに通知できます。



スピードダイヤルキーを既存のアプリケーション（駐車、ボイスメール、キューなど）に割り当てることができることも追加する価値があります。



音声表示を設定する


たとえば、同僚に電話がかかってきた場合などに、Digiumの電話で特定のメロディを再生できます。 つまり、特定の音声通知が有効になります...



対談を聞くのが非常にのどが渇いたらどうしますか？ あなたは彼の電話を鳴らす機会があり、これの唯一の停止はあなたへの折り返し電話になります。 面白いエンターテイメント。 :)



スマートBLFは、これらすべてを実行できます。 使用を開始するには、ドキュメントに慣れるだけで、最新のDPMA 1.7と新しいファームウェア1.4をダウンロードできます。

イノベーションに関するほぼすべてのことを、比較に移りましょう。



Digium D40 IP Phoneの比較





私の意見では、IP電話は3つのカテゴリに分類できます。



予算セグメント。 これらは70〜110ドルの価格のデバイスです。 彼らは通常、1本の動的なライン、ハード（安い）プラスチック、非常に低品質のスピーカーフォン（悪いスピーカー？）を備えた電話スタンド、小さな画面を持っていません



最近、「HD」トレンドが登場しましたが、率直に言って、このクラスのデバイスに取り付けられたこれらのマイクでは、実際のHDからはほど遠く、G722を十分に活用しています。



例：Grandstream GXP1160 / GXP1400、Fanvil C56、D-link DPH-150S / F3



中間セグメント ：最も興味深い人気のあるセグメント 。 これらは120ドルから160ドルのIP電話です

このカテゴリでは、より大きな画面、より快適な外観、PoE、リアルHDが表示されます。

そして、これは今日のレビューのヒーローが落ちるところです。

例：Cisco SPA504、Yealink SIP-T22P、Digium D40



上位セグメント ：価格は170ドルから500ドル...



これには、ユニファイドコミュニケーションレベルのさまざまなデバイス、AndroidファミリのIP電話、またはカラー/タッチスクリーンが含まれます。 需要の少ない、非常に特定のデバイス。

例：Snom 821、Polycom CX500 / CX600、Cisco SPA525G2、Digium D70



Digium D40 VS Cisco SPA504




コストがほぼ同じであるこれら2台の電話を比較するには、一般的な機能から始める必要があります。

人間工学に類似した電話：ABSプラスチック、同数のファンクションキー、重い（快適な）ハンドセット。 2つのイーサネットポート、Vlanタギングのサポート、Voice Vlan。 着信および不在着信の表示、アクティブな回線の強調表示。 実際にはすべて。 次に、違いに進みます。



どちらの電話でもヘッドセットを接続できますが、DigiumはEHSに加えて、Ciscoを使用できます。

Digium D40の画面は240 x 120ピクセルで、Ciscoの128 x 64に対して鮮明ですが、LCDディスプレイの物理的なサイズはほぼ同じです。つまり、電話のDigium画面はより有用な情報に適合します。



私の意見では、D40のデザインはより新鮮で快適です。 しかし、Digium製品とCiscoの競合製品との主な違いは、この電話機には独自のAPIがあり、外部システムと統合できることです。 JavaScriptを使用して、たとえばWebサイトやCRM会社と通信するための独自のアプリケーションを作成できます。



Digium D40は、ユニファイドコミュニケーションシステムレベルの電話です。 アスタリスクまたはSwitchvoxユニファイドコミュニケーションプラットフォームに接続すると、オペレーターは通話を受信するためのツールを受け取るだけでなく、たとえば、キュ​​ーへの出入り、休憩、ボイスメールボックスの表示、会社の従業員のリストの表示などができるようになります... この機能は、開発者の介入なしで、すぐに使用できます。



シスコの電話はシンプルで信頼性があります。 数年前から市場に供給されており、不良品率は非常に低いです。

次の要因もDigium電話の保護に起因する可能性があります：販売された1000台の電話のうち、修理のために返されるのは2台のみです。



Digium D40 VS Yealink T22P








IPmatikaは、ロシア市場でYealink電話を非常にうまく宣伝しています。 Yealink SIP-T22Pは機能的でまともなデバイスです。 推奨価格は127ドルなので、これら2つのデバイスを比較する理由がわかります。



利点として、Yealinkは100％のローカライズを作成する必要があります。 携帯電話のインターフェースとウェブインターフェースの両方が翻訳されています。

ただし、Digiumは静止していません。最後のリビジョンでは、非常に珍しい方法でデバイスをローカライズしました。







英語のメッセージ、会議、保留、リダイヤル がピクトグラムに変更されました。

確かに、画面上のインターフェイス（およびWebフェイス）は同じままです。 これは、一般ユーザーにとっては不便です。 （これはすべて習慣の問題ですが、さらに、英語の知識は誰も傷つけません;））



Yealinkは、シスコと同様に、画面のD40よりも劣っています。132x64ピクセルと240x120です。 また、彼はEHSに接続することもできません。 触覚により、T22プラスチックは、予算のファンビルのように滑らかです。 （誰もがヒットBW210を覚えていますか？）。

他のすべての点で、これらのIP電話は似ています：コーデックのセット、2つのイーサネットポート、自動プロビジョニング、PoE、クリアキーストローク、明るいバックライト。



そして、別の興味深い違いについて少し忘れていました。 Digium電話には、ハンドセットキーはありません。 電話は磁石を使用して位置を決定します。 ノウハウ。実際に使用するには不便です。 もちろん、これは習慣の問題かもしれませんが、番号を間違ってダイヤルしたとき、私の手は常にリセットキーに手を伸ばします。 しかし、パイプ全体を配置する必要があります...
