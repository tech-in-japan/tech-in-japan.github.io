ZX Spectrumクローン「レニングラード」のアセンブリに関する記事を書いてから1年以上が経ちました。 この間、その機能は大幅に拡大しました。 続編を書くとは思いませんでしたが、それでもレトロコンピューターに関する記事はそれほど多くないので、決めました。



拡張ポート



何も接続できないコンピューターの機能は非常に限られているため、拡張ポートをはんだ付けすることにしました。 40ピンIDEに加えて、ボードには他に何も収まりませんが、必要な接点の数を計算した後、これで十分であることがわかりました。 キーごとにマイナス1接点、2接点+ 5V、アースへの2接点。











ケース内のキーボード



以前は、外部キーボードを使用していましたが、より本格的なフォームファクターが必要でした。 組み立ての時に、90年代のSpectrumからステッカーが貼ってあります。 Sinclair of joysticksとRESETボタンのポートはそこにねじ込まれていました。 簡単なこととして、ケースの長方形を切り取り、ラックのボードに取り付けます。 背中も少しカットする必要がありました。







ダウンロードなしのゲーム



Spectrumのカートリッジは、ALFゲームコンソールを除き、私たちには人気がありませんでした。 高価なデバイスZX Interface 2を要求したため、ヨーロッパでも人気があるというわけではありません。公式には、リリースされたゲームは10ゲームのみでした。 カートリッジは、一連の連絡先を備えたボード上に事前に記録されたゲームを備えた単なるROMであるため、このデバイスの作成は簡単です。 私は長い間、回路基板のエッチングを試みたいと思っていましたが、今はまさにその場合です。 しかし、最初に、それも分離する必要があります。このため、私はスプリントレイアウトプログラムを使用しました。







Leningradボード自体で、プルアップ抵抗をROMの20番目の接点にはんだ付けし、接点自体を拡張ポートに接続しました。 カートリッジを接続すると、+ 5Vが供給され、コンピューターのROMがオフになります。 まあ、連絡先/ RDROMも拡張ポートに持ってきました。 ボードは離婚しているので、ソビエトのPCBから正方形を切り取ります。







光沢のある雑誌のページをプリンターに入れ、印刷してアイロンをオンにします。







ボードは両面で、ホッチキスを使用してページから封筒を作成します。







このようなことが判明しました。もちろん、マーカーでいくつかの領域をより良くペイントする必要がありました。







エッチングには塩化第二鉄を選びましたが、他の新しい方法がたくさんあることは知っています。







それはまあまあ判明しました。







そして、ボードがエッチングされた後にのみ、気が散って混乱し、印刷時にレイヤーを混同しました。 新しいPCBをカットしたのは怠wasで、必死になってビアを作りました。 たまにはんだごての先端に箔の破片があり、MGTFがなくてできなかったため、トラックをいじくり始めました。 カートリッジおよびレニングラード自体のROMは、EEPROM W27C512を選択しました。 ゲーム（各16 kbの4個）を選択するために、カウンター74HC393を使用して、リセットボタンを押すとコレクションでDANDYに切り替わるようにしました。 しかし、それは悪い考えでした。その結果、私はカウンターを引き出し、2つのディップスイッチを切り替えました。 ピンコンタクトのはんだ付けポイント40の剛性を高めるために、ホットグルーを充填します。











当時のプログラマーはすでに自分で買っていました。 Jetpacを含むUltimate Play The Gameのすべてのヒットを満たします。 もちろん、GyrussやStar Warsなど、決して出てこなかったゲームを含めて、後で試しました。











ドライブを接続します



私はすでに記憶媒体としてテープとカートリッジを持っていますが、もちろん最も穏やかなのはフロッピードライブの使用です。 Black Raven、UFOなど、この媒体の下で最高のゲームが書かれています：敵は不明であり、近い将来、POWER BLADEが登場し、さらにテープのほとんどすべてのおもちゃがそれに移植されます。 ダウンロード速度は、テープレコーダーからのダウンロード、640 kbフォーマットのTR-DOSフロッピーディスクのボリュームよりも15〜20倍速く、利益は明らかです。 ただし、ドライブ自体をスペクトルに接続することはできません。スペクトルの半分（フロッピーディスクドライブのコントローラー、別名KNGMD）を収集する必要があります。 レニングラードの他の多くの改良点と同様に、zxbyte.ruサイトからこのデバイスのスキームを採用することにしました。







私は現実的であると決めたので、おそらく次の世紀までに15個のマイクロサーキット用の回路を作成するので、ブレッドボード上で組み立て始めました。 まあ、私は本当にボードをコンパクトなケースに入れたかった。 国内のドライブコントローラーの心臓部はKR1818VG93チップでした。 彼女の欠点は非常に重大であり、12ボルトを必要とし、信頼性に違いはありませんでした。 したがって、私はチップアナログMB8877を注文しました。 私は知らない、中国人が私にオリジナルか偽物を送ったが、それはうまくいき、素晴らしい。 レニングラード州とバルト州に接続するために作成されたKNGMD S-48回路を選択した後、不快な詳細に遭遇しました。読み取りノードでは、PLLロジックがK556RT4 PROMチップに実装されましたが、これは中国のプログラマーにはわかりませんでした。 以前に使用したW27C512を代わりに使用しようとしましたが、何かを台無しにしたか、EEPROM自体に問題がありましたが、すでにTR-DOSに入ることができましたが、フロッピーは機能しませんでした。 次に、PLLを使用せずにペンタゴン128方式に従って読み取りユニットを再構築しました。その結果、ドライブはフロッピーディスクをフォーマットできました。 ちなみに、標準の高密度ディスケットは、ドライブに倍密度ディスクとして表示されるように、角に穴を開けて密封する必要があります。











ダウンロード後にフロッピーディスクを使用すると、テープ/電話からでも高速化できます。 KNGMDスキームは単純化されており、2つのドライブ（同じペンタゴンと最大4つの他の多くのクローン）でしか動作しませんが、これで十分です。 3.5 "ドライブで十分にプレイしたので、5.25を取得したかったのですが、問題なく動作しました。ところで、私は最初に5インチフロッピーディスクに触れました。 便宜上、BASIC-128の代わりにクイックコマンダーを求めました。BASIC-128は音楽を再生し、画像やあらゆる種類のファイル操作を見ることができます。







ZX Studioプログラムを使用してTR-DOSフロッピーディスクイメージをPCに書き込み、ZX Spectrum Navigatorを使用して独自のイメージを作成します。











デモを見る



当社のデモシーンは、主にペンタゴン-128スペクトラムクローン用に開発されました。回路の仕様により、zxスペクトラムが69,888クロックサイクルであった場合、1フレームを描画するのに71,680クロックサイクル、メモリからの応答が遅延したためにレニングラードは65,184クロックサイクルでした。 ほとんどのゲームや他のプログラムでは、これはそれほど重要ではありませんでしたが、デモでは重要ではありませんでした。 信号/ WAITを遮断し、+ 5Vにします。 9D5は、信号/ BCの代わりにグラウンドに給電します。 しかし、SpectrumマルチカラーはPentagonのタイミングでは機能しない可能性があるため、Spectrum / Pentagonジャンパーを作成します。 CMOSバージョンのプロセッサ（Z84）では、このオプションは機能しませんでしたが、U880には問題はありません。 コンピューターは五角形と100％の互換性はありませんでしたが、五角形用に作成されたデモは遅くなりません。 1998年に開発されたREFRESHがとても気に入りました。







SR / M-80で作業しています



このOSを実行できるクローンはほとんどありません。BYTE-01、Quorum-64、ZS Scorpion 256、Prosを思い出すことができます。 SR / Mを使用するには、コンピューターがシャドウRAMをサポートしている必要があります。 また、ZS Scorpion 256は高速でレニングラードであるため、少し変更するのに大きな困難はありません。 スキームと研究をダウンロードしてください。 ポート＃1FFDのROMビット0およびスペクトラム+3を無効にしました。これは、すぐにSR / Mを実行する機能がありました。 また、ビット4は128kbページを切り替えます。 しかし、私と一緒に、ポート＃7FFDスイッチのビット6は、プログラムと競合する場合がありますが、1つのデモだけで問題が発生しました。 ポート＃1FFDをサポートするために、拡張スキームを再描画する必要がありました。 ペンキでそれをしました、それはこのような何かになりました。







ボードにこれ以上スペースがありません。ATXケースで20 * 30ボードを使用しなかったことを100回後悔しました。 6ビットのポート番号7FFDを使用しなくなったため、1533TM2チップは不要になり、代わりに1533TM9を配置し、ポート番号1FFDの0ビットと4ビットを設定します。 拡張スキームでは最大1533KP12を使用するため、必要なOR要素が解放されます。 スコーピオンキットからCP / Mディスケットに事前に書き込むことで確認します。 うまくいく！







灰色の背景と黒い文字のCP / Mを見るのは少し珍しいですが、それを移植した人は明らかに、ネイティブのBASIC-48との類似性を望んでいました。 デフォルトでは、OSには48kbのRAMと160kbのRAMディスクがあり、1行あたり64文字（幅3ピクセル+スペース）で始まります。これは非常に読みにくいですが、許容範囲です。 モードユーティリティは、ほとんどのプログラムに必要な32、40.48、および80（スペースなしの3ピクセル幅、判読できない乱雑な）文字に変更できます。 一部のクローンには、512x192モードのByte-01など、より高い解像度モードがあり、CP / Mでの作業に非常に適しています。 しかし、ブランド化されたスペクトル+3、スコーピオン256、および他のほとんどのクローンには、そのような幸福がありませんでした。これは、おそらくOSのスペクトルに対する不人気を引き起こしました。 CP / M-80自体、または1979年のバージョン2.2については、別の記事を書くことができます。ソフトウェアについて説明します。
