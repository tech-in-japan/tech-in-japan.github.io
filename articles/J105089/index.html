Node-mysql-libmysqlclient v1.0.0

アナウンスを気にしていないことを望みますが、これからはnode-mysql-libmysqlclientが安定したモジュールと見なされるので、これには価値があります。 Node-mysql-libmysqlclientは、 Node.jsの MySQLの操作のサポートを追加し、データベースへのクエリの同期および非同期実行を提供し、PHP / Perl / Rubyなどの同様のコネクタのAPIに近いAPIを備えています。 例に従って使用された場合、すべての関数はエラーとメモリリークについてテストされます。 fetchAll（）関数のクラッシュが修正されました。 以前に発表されたバージョンと比較して、例とAPIドキュメントの外観を別々に注目する価値があります。





  • 修正：
    • 書き換え関数result.fetchAll（）
    • 書き換えられたプロシージャMysqlResult :: SetFieldValue（）
    • 結節を扱う際のバグを修正
    • 統一されたエラー処理が完了しました
    • MysqlConnectionでのメモリ割り当ての削減:: EscapeSync（）
    • スタイルを含むその他のコードの改善
  • テストの改善：
    • テストは、単純、複雑、遅く、無視されるグループに分けられます
    • Caolanリポジトリを再度使用して、nodeunitを更新しました
    • メモリリークを検出するためのインタラクティブコンソールを追加
  • ドキュメントの改善：
    • README.markdownからindex.htmlの作成を追加
    • マークダウンでフォーマットされたChangeLog
    • CHANGELOG.markdownからchangelog.htmlの作成を追加
    • 最新のzipおよびtarballへのナビゲーションとリンクを追加しました
    • リポジトリから削除されたHTML。「node-waf doc」を使用して生成できます
  • セマンティックバージョンの命名が導入されました
プロジェクトサイト： sannis.github.com/node-mysql-libmysqlclient

ドキュメント： sannis.github.com/node-mysql-libmysqlclient/api.html

使用例： sannis.github.com/node-mysql-libmysqlclient/examples.html



このバージョンの開発中に、現在のNode.js APIを反映するためにNodelintに変更を加えました。

Nodeunitも更新し、標準のassertモジュールからのすべてのチェックをサポートするようになりました。

ドキュメントを生成するために、TJ Holowaychukの拡張Doxスクリプトを使用してC / C ++ファイルを処理しました。

私が使用しているDoxのバージョンはGitHubでも入手できます 。
