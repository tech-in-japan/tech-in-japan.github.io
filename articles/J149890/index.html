Raspberry Pi：スクラッチからTorrentBoxまでの詳細なセットアップ

前戯



HabraHabrでRaspberry Piプロジェクトを長い間見ていましたが、ミニコンピューターを入手することにしました。 事前注文が始まったとき、私はそれをほとんどすぐに使用しましたが、2012年6月17日にのみ、RSComponents.ComからRaspberry Piの注文の可能性についてのメールを受け取りました。 合計で、予約注文から約1か月が経過しました。



同じ日に注文を作成し（ところで、ロシア連邦では「公式に」注文することはすでに可能でした）、「ラズベリー」を待ち始めました。 ニュースレターは最大6週間の出荷を約束しましたが、この期間中に料金を受け取りませんでした。 RSのモスクワ駐在員事務所への電話で、会社のマネージャーは、配達はすぐに行われることを明らかにしましたが、知られていないときでした。



8月17日に、モスクワRSの従業員から、支払いがオフィスに引き渡されて引き取り可能であるというメールを受け取りました（DHLは個人に小包を配達しないため、RSから宅配で注文することは不可能であるため）。 実際、同じ日にRaspberry Piコンピューターを受け取りました！







コンポーネントのすべての必要なセットは事前に私が購入しました（実際、他のデバイスからすべてを借りました）。 私が使用した：

  • トランセンドの4Gb Class6 SDカード
  • NoName 1A MicroUSBケーブル付きUSB充電器
  • HDMIケーブルハマ
  • イーサネットケーブル


キーボードとマウスとして、LogitechのUSBレシーバーUnifyingを使用しました。 HDMI-DVIケーブルを使用してRaspberryをモニターに接続しました。







最初のアクション。 OSインストール。



RaspbianがRaspberryのOSとして選ばれました（フォーラムから理解したように、ほとんどすべてのビルドはDebianに基づいているため、経験の浅い私の意見では、その選択はそれほど豊かではありません）。 このOSはインターネット上で広く説明されており、RPi専用に最適化されています。

OSイメージは、公式ウェブサイト2012-07-15-wheezy-raspbian.zipからダウンロードできます。 画像はZIPでアーカイブされ、IMG拡張子が付いています。 解凍する必要があります。

また、管理者権限で実行する必要があるWin32DiskImagerユーティリティが必要です。



SDカードをカードリーダーにインストールし、エクスプローラでシステムで受信した文字を確認します（別のメディアのデータを誤って消去しないようにするため）。

Win32DiskImagerプログラムで、以前にダウンロードしたRaspbianイメージを選択し、目的のドライブ文字を選択して、[書き込み]をクリックします。 「はい」警告で応答します。

OSをカードに書き込み、セクションに分割するプロセスは次のとおりです。







プロセスは成功メッセージで終了します：







ここで、メモリカードのルートにあるconfig.txtファイルをわずかに変更する必要があります。これは、Raspberry Piのシステム構成ファイルです。 指定した設定は、画面解像度が1920 * 1080のLCDモニターに関連していることに注意してください。



パラメータdisable_overscan = 1はコメント解除する必要があります（RCA出力を使用する予定がない場合）。

また、固定解像度を設定することをお勧めします。このため、行hdmi_groupおよびhdmi_modeのコメントを外します。 hdmi_modeパラメーターの値は、 ここに記載されている表に従って変更する必要があります （また、このリンクには、役に立つかもしれない他のパラメーターがリストされています）。 たとえば、解像度が1920 * 1080のモニターの場合、 hdmi_mode = 16と記述します。



変更を保存し、SDカードを取り外して、Raspberry Piに挿入することを忘れないでください。

電源をボードに接続し、モニターでブートプロセスを確認します（少なくとも今のところ）。 Raspberry Piの電球が楽しく点滅しているのがわかります。







最初の起動時に、 raspi_configシステムセットアッププログラムが自動的に起動します：







  1. expand_rootfsコマンドを実行します 。これにより、ルートパーティションがSDカード全体に拡張されます。
  2. configure_keyboardセクションに進み、キーボードの最適なタイプを設定します。 Logitech Cordless Desktopを選びました
  3. 次に、 change_passセクションに移動し、ユーザーpiの新しいパスワードを設定します（入力した文字は、アスタリスクの形式でも表示されないことに注意してください！）。
  4. change_localeアイテムを使用して追加のロケールをインストールします（これを行わず、デフォルトのロケールであるen_GB UTF8のみを残しました）。
  5. タイムゾーンを設定します（ set_timezone ）。 たとえば、モスクワに住んでいる場合、ヨーロッパのポイントを見つける必要があり、その中に-モスクワ
  6. Memory_splitは、私たち自身の希望に従ってインストールします。グラフィカルインターフェイスを使用する予定がある場合は、ビデオに32Mbを割り当てることをお勧めします。
  7. 必ずsshを有効にしてください！
  8. Raspbianの起動時にグラフィカル環境を自動的に起動する場合は、 boot_behaviourオプションを有効にします。


最後に、[CTRL] + [F]を押して[ 完了 ]を選択し、デバイスの再起動に同意します。



再起動が完了すると、ユーザー名とパスワードの入力を求められます。 ユーザー名はpiで、Raspbianシステムのセットアップ時に設定したパスワードです。



これで、システムのインストールと初期セットアップが完了しました！



次の部分は、Linuxに特に慣れていない人にとっては興味深いものです（私もそうでした）。



2番目のアクション。 Webサーバーをインストールし、Sambaを構成します。



Raspberryがネットワークに接続されている場合、SSHアクセスを使用できることを忘れないでください。これは何倍も便利です。



開始する前に、 apt-getを更新します 。 

sudo apt-get update


Webサーバーのインストール：


MySQLをインストールします。 

 sudo apt-get install mysql-server mysql-client


ルートのパスワードを設定するように求められたら-覚えているパスワードを指定します。



Lighttpdをインストールします。 

 sudo apt-get install lighttpd


ネットワーク上の任意のコンピューターのブラウザーにIPアドレスを入力すると、この時点からRpiはテストページで応答します！



PHP5をインストールします。 

 sudo apt-get install php5-cgi




ここで、サーバー設定でPHPをアクティブにする必要があります。 nanoエディターでファイルを開きます。 

 sudo nano /etc/lighttpd/lighttpd.conf


server_modulesアイテムは次のようになります。 

 server.modules = ( "mod_access", "mod_fastcgi", "mod_alias", "mod_compress", "mod_redirect", "mod_rewrite", )


そして、ファイルの最後に、これを追加します： 

 fastcgi.server = ( ".php" => (( "bin-path" => "/usr/bin/php5-cgi", "socket" => "/tmp/php.socket" )))


[CTRL] + [X]、[Y]、[Enter]を押して保存します。



PHP5構成ファイルを編集するために残ります。 

 sudo nano /etc/php5/cgi/php.ini


文字列cgi.fix_pathinfo = 1を見つけてコメント解除します（文字「;」を削除します）。 ファイルを保存します。



すべて完了したら、次のコマンドを実行してLighttpdを再起動します。 

 sudo /etc/init.d/lighttpd restart




Sambaをインストールして構成する


Sambaをインストールします。 

 sudo apt-get install samba samba-common-bin


Rpiはホームネットワークにあるため、フォルダーにアクセスするためのパスワードを設定せず、ネットワーク全体のパブリック共有を設定することにしました。

これを行うには、smb.confファイルを開きます。 

 sudo nano /etc/samba/smb.conf


利用可能なすべてのコンテンツの代わりに、次のように記述します。 

 [global] workgroup = WORKGROUP guest ok = yes netbios name = Raspberry security = share browseable = yes [www] path = /var/www writeable = yes browseable = yes


保存します。 Sambaの再起動： 

 sudo /etc/init.d/samba restart


これから、新しいRASPBERRYデバイスがネットワークに表示されます。これにはwwwフォルダーがあります。

その中で、ブラウザを使用してネットワーク全体で表示できるファイルを作成できます。



ちなみに！ Webインターフェイスを提供するSWATを使用してファイルとフォルダーの共有を管理する方がはるかに便利です。

インストールはとても簡単です： 

 sudo apt-get install swat


SWATコントロールパネルは、http：// [IPデバイス]：901にあります。

ユーザー名とパスワードがアカウント（SSHに使用するもの）と一致する



3番目のアクション。 ファイルキャリアをマウントします。



キャリアとして、私は通常のフラッシュドライブを使用することにしました。これをRaspberry PiのUSBポートの1つに接続しました。 必要に応じて、外付けハードドライブをRpiに接続できますが、ボードのUSBポートはそのような負荷向けに設計されておらず、最良の場合、ハードドライブは単に「起動」しないため、別の電源を編成する必要があります。 フラッシュドライブのボリューム（16GB）で十分です。



メディアを接続して、コマンドを実行します。 

 sudo fdisk -l


このコマンドは、デバイスに接続されているすべてのデバイスを表示します。 目的のデバイスのボリュームごとのリストを探しています。 たとえば、私の希望する行は次のようになります。 

 Disk /dev/sda: 16.0 GB, 16013852672 bytes


デバイスへの望ましいパスは/ dev / sdaです 、覚えておいてください！



fdiskを実行して、メディアをフォーマットします。 

 sudo fdisk /dev/sda


最初に、 dコマンドで既存のパーティションを削除し（数字で必要なパーティションを選択）、 nコマンドで新しいパーティションを作成し（デフォルトですべての値を受け入れます）、 wコマンドで行った作業を保存します。



メディアにext2ファイルシステムを作成します。 

 sudo mkfs -t ext2 /dev/sda1




マウント： 

 sudo mount -t ext2 /dev/sda1




次に、Raspbianが起動するたびにメディアが自動的にマウントされるようにする必要があります。 これを行うには、フォルダーを作成します。 

 sudo mkdir /mnt/flash


設定ファイルを開きます。 

 sudo nano /etc/fstab


行を追加します： 

 /dev/sda1 /mnt/flash ext2 defaults 0 0


デバイスを保存して再起動します。 ロード時に、メディアは自動的にマウントされるはずです。これは次のコマンドで確認できます。 

 df


マウントされたデバイスのリストとマウントポイントの表示が表示されます。



ところで！ コンソールでファイルを操作するには、Midnight Commanderファイルマネージャーをインストールすることもお勧めします。 

 sudo apt-get install mc


Puttyを使用してSSHを操作する場合、MCが正しく動作するにはセットアップを行う必要があります。 Putty設定で、TranslationセクションのRemote character set値を「UTF-8」に設定します。







4番目のアクション。 Transmissionのインストールとダウンロードの構成



私たちはこのトピックの目標に到達しました-デバイスへのTorrentクライアントのインストールと設定。 私は伝送を選択しました。



伝送をインストールします。 

 sudo apt-get install transmission-daemon


ダウンロード用、不完全なダウンロード用、接続されたメディア上のトレント用のディレクトリを作成し、書き込み権限を付与します。 

 sudo mkdir /mnt/flash/torrent sudo mkdir /mnt/flash/torrentfiles sudo mkdir /mnt/flash/incomplete sudo chmod 777 /mnt/flash/torrent sudo chmod 777 /mnt/flash/torrentfiles sudo chmod 777 /mnt/flash/incomplete


設定の編集： 

 sudo nano /etc/transmission-daemon/settings.json


ここでは、次のパラメーターを指定された値に変更する必要があります。 

 "cache-size-mb": 2; "download-dir": "/mnt/flash/torrent", "incomplete-dir": "/mnt/flash/incomplete", "preallocation": 2, "rpc-password": "    (    )", "rpc-username": "pi", "rpc-whitelist-enabled": false, "speed-limit-down": 3000, "speed-limit-up": 1000,


残念ながら、ダウンロードとアップロードの速度が速いと、Raspberryは非常に遅くなり始めるため、上記の設定で見られる制限は実験的に確認されています。



Transmissionコマンドを再起動します。 

 sudo /etc/init.d/transmission-daemon restart


この瞬間から、動作中のトレントクライアントをインストールしました。そのWebコントロールパネルは、http：// [ip-devices]：9091、ログインpi、設定ファイルで設定したパスワードで利用できます。



ダウンロードしたファイルを他のデバイス（メディアプレーヤーなど）で表示できるように、 / mnt / flash / torrentフォルダーもSamba経由でネットワーク共有に追加することを忘れないでください。





映画の名前は意図的に変更されており、そのような映画は存在しません



私の意見では、Raspberry Piはネットワーク上のファイル転送を完全に処理します-Raspberry Piからコンピューターにファイルをコピーすると、最大速度は7Mb / sに達しました。これは使用されるフラッシュドライブの最大読み取り速度にほぼ対応しています
