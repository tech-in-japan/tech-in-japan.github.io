Mac OSの比較

要するに、これはOS Xで良いことと、WindowsとUbuntuに比べて悪いことの比較リストです。 この比較は、Windowsでの4年間の経験、Ubuntuでの3年間の選択、およびMac OS Xとの4か月間の友情に基づいています。 この記録が生まれたのは、主にGoogleまたはAppleテクノロジーの所有者に、それがより良いものであり、何を使って答えられるかを尋ねたときだ。 セキュリティ、美しいフォント、はかない利便性の悪名高い切り札に加えて、私は何も聞いたり読んだりしませんでした。 したがって、私が「ポピー」の所有者になるとすぐに、気付いたことを書き留め始めました。





背景


G +フィードで、ドイツの私の友人からのメッセージを見ました：

「Macbook Proを購入しました。 私は、その前は自分の人生すべてを苦しめていたことに気付きました。」

この人の意見は、さまざまな理由で私にとって非常に権威があります。 20万ルーブル以上の給与を持つ「シニアJava開発者」の求人を公開するとき、彼らは彼のような人を探していると思います。 彼がWindowsとLinuxの両方を専門的に使用していることは明らかです。 OS Xがかっこいいと彼から聞いたとき、半年後（2011年末）にAmazonでMacbook Pro 15を購入しました。

購入後、私は別の友人の質問にも答えなければなりませんでした。私の友人は、「ポピー」が「Windows」よりも優れているかどうか、そして何について権威がありました。 客観的になりたかったので答えられませんでしたが、3、4か月後に自分の印象を伝えると約束しました。

Mac OSの良い点と悪い点


実際には4か月間のインプレッションのリスト。 読むときは、意見が主観的であることを忘れないでください。



マイナス：




1.プログラムをアンインストールした後、ドックからアイコンを削除できませんでした。 彼女はリブート後に退職しました。



2.フォルダーの1つで、Finderはファイルを表示しませんでした。 彼は、ターミナルでkillall Finderと入力することで回復しました。



3.多くは再学習する必要があります。



4.複数のウィンドウを持つアプリケーションでコマンド+ Tabを頻繁に使用するアプリケーションを切り替えると、妨害が発生します。 簡単に説明することは困難です。 アプリケーションを切り替えると、メインウィンドウの代わりにフォント設定のあるウィンドウが表示されます。



5. Pagesでは、スタイルalt + shift + command + Vなしでテキストを挿入するための組み合わせは、shift + command + Vの組み合わせによって中断され、予期しない動作をします。 すぐにalt + command + Vに変更すると、すべて問題ありません。 しかし、デフォルトの組み合わせとの混乱は依然として不快です。



6.アプリケーションを完全に削除するには、特別なソフトウェアをインストールする必要があります。 事実ではなく、どのプログラムもタスクを適切かつ正しく処理します。 多くの人は、プログラムフォルダーからアプリケーションを削除するだけで十分であるという意見を持っていますが、そうではありません。



7.指揮者を通して隠されたファイルを見るのは、あなたから奪う子供からのお菓子ではありません。



8.ドライバーの問題は最後まで解決されません。 物事はubuntuよりも優れていますが、すべてが順調に進んでいるわけではありません。 多くのプリンターはフォレストによって送信されます。



9.すべてのBluetoothマウスが正常に機能するとは限りません。 マウスを起動した後、再接続する必要があります。 タッチパッドを使用すれば、これは我慢でき、iMacまたはMac miniでどのように解決されるかは完全には明らかではありません。



10. Microsoft Office for Macでは、キーボードショートカットが何らかの形で落ちました。 プログラムを再起動すると、すべてが大きくなりすぎました。



11.望みどおりにウィンドウが展開するとは限りません。 時々、自分でフルスクリーンに拡大する必要があります。



12.長期運用中、firefox、chrome、およびsafariはRAMを食い尽くします。 定期的に再起動することをお勧めします。 または、より多くのRAMを提供します。 他の2つのOSはこの点で優れていません。それは単純であり、ポピーではそのような問題があります。



13.ページでは、すべての文字を小文字に変換する方法はありません。 解決策は見つかりましたが、それは本当の失敗です。



14. 10日20時間の稼働時間の後、Finderが「エンジン」をダブルクリックしてファイル名に従って列幅を選択するのを停止したことに気付きました。 killall Finderは問題を解決しませんでした。 自分で列を伸ばすことができ、これはダブルクリックより長くないため、重要ではありませんが、ダブルクリックしたときに予期されるアクションがないのは不快です。



15. App Storeからの賢明なアプリケーションが少し届きました。 たとえば、Parallels Desktopは含まれていませんが、残念です。 ストアで購入することは、別のアカウントを登録してカードをもう一度輝かせることよりも便利です。



16.この記録をどこに置くべきかさえ、ノートや欠点にさえわかりません。 私は働いて、働いて、突然すべてが凍り、ブルースクリーンが現れ、それからデスクトップ、そして今-私の目の前に承認フォームがあります。 ログアウトしたかのように、何もしませんでした。 パスワードを入力し、ドックを見ました-出発前に起動されたすべてのプログラムが開いているとマークされ、それらの下のドットが焼けました。 ただし、プログラムウィンドウは閉じているかのように表示されません。 ただし、起動時のようにアイコンはジャンプしません。 数秒待ってから、ブラウザウィンドウが表示されます。 私はチェックします-事件の前にあったすべてのタブを持つすべてのウィンドウ。 次に、起動されたすべてのプログラムが異なる順序で開かれました。 すべてのデータは、Microsoft Excelでも適切に配置されていました。もちろん、作業中は保存しませんでした。 私は何が起こったのか理解できませんでしたが、サンダーバードの手紙では、Yandexからのアカウントメールに来始めました。 それ以前は、何らかの理由で接続することはできませんでしたが、現在は突破されています。 私は困惑しています。



17.稼働時間の19日目に、firefoxは起動を拒否しました。 彼は、コピーが既に実行されていると言いますが、これはプロセスでは見つかりませんでした。 おそらく再起動が役立つでしょうが、今のところキツネは実際には必要ありません。
