VLC拡張



2つのモニターで映画を見たいですか？ 質問ではありませんが、デジタル家電店で働いているなら、棚に置いてあるすべての画面で見ることができます。 いいえ、いや、VLCをすでに持っているので、何もインストールする必要はありません。このプログラムを使用すると、81モニターで映画を見ることができますが、幅99行、長さ99で見ることができます。 1モニターはあえてしませんでした=）



少なくとも2人でこれがどのように機能するか、あなたは尋ねます...私は答えます：

-CTRL + E（拡張設定）-ビデオエフェクト-Vbut / Overlay

-Wallにチェックを入れると、ビデオがウィンドウに数量（デフォルト）で表示され、必要なフィールド数に変更し、チェックを外して、再度チェックします。 次に、各ウィンドウを目的の画面に配置します。







なぜ彼らがこれを思いついたのかわかりませんが、おそらくこの楽しい要素はすでにその目的のために繰り返し使用されています。 たとえば、私はおそらく子供たちに論理を教えて、漫画からパズルを組み立てることができるようにします。パズルを完成させれば、通常は見ることができます。 これは、インタラクティブ機能を備えたもう1つの興味深いビデオ効果です-「 パズルゲーム 」。







仕組み：

-CTRL + E（拡張設定）-ビデオエフェクト-Vbut / Overlay



プラグインとフィルターのすべての魅力を深く掘り下げたわけではありません。自分で試してみることができます（ CTRL + E（拡張設定）） 、ブラーモーションフィルターに注意することをお勧めします。 さらに、定義アンプなどがあります。



ポッドキャストを聞いて、ストリームにリダイレクトします。



最近、Podcastsを聴く機会がありましたが、リリース後は明らかに目立った機能ではなく、それについてはほとんど言及されていませんでしたが、それに注意する価値があります：

CTRL + P、設定を表示（すべて）。 プレイリスト-検索サービス-ポッドキャスト 、およびポッドキャストフィードを追加します。

それからただ行く

- メディア-サービスの発見-ポッドキャスト （追加されたテープのロード用）

- メディア-サービスの発見-ポッドキャストの構成 （新しいフィードを編集および追加するには）







これで十分ではなく、1つの受信ポッドキャストストリームを別のコンピューターにリダイレクトするか、ローカルネットワークにブロードキャストする場合は、ポッドキャストブロードキャストをネットワークに設定するだけで十分です（「http」で例を示しますが、これには「UDP」をお勧めします。聞くだけで、「http」がリモート管理インターフェースでした）

1.ポッドキャストまたは曲のリストを追加したら、

2. CTRL + Yを押してプレイリストを保存します。これをさらにストリーミングするためにVLCにフィードします。

3.ストリームのCTRL + S 、表示されるウィンドウで、プレイリストファイルを追加し、[ ストリーム ]をクリックします。 。 次に、必要に応じてMP3形式を選択し、構成します。 その後、「 ストリーム」をクリックします。

4.次に、 [表示]-[インターフェイスの追加]-[Webインターフェイス]に移動して、Webインターフェイスを起動する必要があります。



PS：これらすべてのステップを作成する過程で、[ オプション ]ボタンをクリックして、生成されたコードに注意を払うことができます。生成されたコードは、コマンドラインまたはバッチファイルで使用できます。 確認のために、管理パネルのWebインターフェースから、現在ブロードキャストされているものを表示する画面を表示します。







「同様に、VLCにあらかじめ作成されたプレイリストを供給するか、単にオーディオまたはビデオファイルのスタックを選択することで、独自のラジオステーションを作成できます。」



-オンラインのラジオ局やテレビにアクセスするには、次のことを行う必要があります...

1. [ メディア] -[ サービスディスカバリー]に移動し、 Shoutcast radio （tv） streamingを選択します

2.その後、 表示-プレイリスト（CTRL + L）に移動し、ポッドキャストとラジオおよびビデオストリームがここでライブ-選択して再生します。このトピックに関する最後の投稿を注意深く読むと、簡単に接続できますWebインターフェイス経由で聞いたり、別の部屋やバスルームでiPhoneやPDAから聞いたりします。

2.1。 カバーをロードし、検索クエリでラジオ局をフィルタリングできることに注意してください。

VLCを使用して「ScreenCast」を作成する



画面からビデオを撮影するには、次のようにします。

1. CTRL + C、-キャプチャモード（デスクトップ）

2.撮影するフレームの数を選択します。 次へ2回次へ 。

3.「 新しい宛先」セクションには、画面から撮影するものを固定する多くの可能性があります。

-ファイルにローカルに保存します。

-ネットワーク上でのブロードキャスト（ リモートの従業員または知人のトレーニングに使用することができます ）;

また、電力とチャネルの両方がある場合は、異なるプロトコルでブロードキャストし、ローカルファイルに同時に保存することもできます。

4.次に、コーデックを選択/編集し、「 ストリーム 」をクリックします。



PS：[ ローカルに表示]チェックボックスをオンにすると、この機能はすぐにあなたが何をしているかのビデオを開きます（ 最初は、デスクトップの多くが互いに衝突するのを見たときに怖がっていました= ）、それは誰かに役立つかもしれませんが、追加のCPU負荷。



LAST.FMのプロファイルに音楽データを送信します



ここでも、不必要な追加ソフトウェア、この場合はLAST.FM Scrobblerをすべて破棄する機会があります。



現在、唯一の問題であるVLCはScrobblingに非常にうまく対処しています。記事に追加したように、テストする時間がありませんでした。最大2日間、プロファイルとウィジェットにトラックが表示されることがよくありました。 したがって、VLCで実行する方法を説明し、これを決定します。



CTRL + Pを押します。 次に（ 設定を表示-すべて ）そして/ インターフェース-コントロールインターフェース-Audioscrobblerに移動し 、Last.fmのログインとパスワードを指定して保存します。次にVLCを起動すると、トラックが落書きされます。

1. *** VLCが機能しないか、インターフェースがまったく見えない程度に

1.パラメーターを指定して実行します。

-I qt





2. C：\ Users \\ AppData \ Roaming \ vlc \ vlcrcに移動します

構成、編集、または強制終了が必要です。VLCはそれらを再作成します。



2. Mplayerが好きで、同じ「非GUI」VLCが必要な場合は、パラメーターを指定して実行します。

vlc -Irc





または

-Idummy





また、GUIなしでVLCをサーバーとして起動するには、

-Itelnet





それぞれtelnetまたはhttpの場合

vlc -Ihttp







3.プロキシを使用していて、カバーのダウンロード、ラジオの再生、スクロブリングなど、VLCでオンラインで作業する喜びをすべて享受できない場合は、次のパラメーターでVLCを起動します。

--http-proxy=_:





例えば：

vlc --http-proxy = 192.168.0.2：8080
