北極圏の都市でどのようにコミュニケーションを提供したか

北極圏は気温が低く、永久凍土層が足元にあります。 これらの地域の気候は急激に大陸性で厳しいです。 しかし、ここでは接続が必要です。 たとえば、SalekhardとLabytnangiで。





サレハルドの厳しい冬の気候



これらの都市でモバイル通信機器を近代化し、新しいコアネットワークを展開しました。 モバイルスイッチは、Ob Riverを越えて転送され、新しい機器にテレポートされました。 私はこの北部プロジェクトの組織的および技術的部分を担当しました。



背景



サレハルドは探検遠征の中心であり、同時に先進的な漁業の町でもあります。 市はオブ川に位置し、独自の川の港があります。 「大きな」土地から離れているにもかかわらず、この都市はヤマロネネツ自治管区（ヤマルネネツ自治管区またはヤマル）の地域の中心地です。 冬には、ここの温度は–40度まで下がることがあります。 サレハルドの人口は47.9千人です。 この街の隣、オブ川の反対側、そこからわずか16 kmのところに、別の小さな町ラビトナンギがあります。 これら2つの北部都市の人口の約3分の1は、当社の加入者です。



ヤマル・ネネツ自治区の通信組織は簡単な仕事ではありません。平均年間気温は常にマイナスであり、11月末から2月中旬にかけて、極地の夜がこの地域のかなりの地域を支配します。 今年のこの時点で、太陽は地平線上に短時間昇り、再び沈みますが、これは新しい通信施設の建設には寄与しません。 しかし、これらの場所で、「オンボード」ベースステーションを備えたリモートセルタワーの構築を開始しただけでなく、音声トラフィックスイッチング用のコアモバイルネットワークの近代化を決定しました。



ネットワークの近代化の仕事は、会社の特別なプロジェクトのステータスを受け取りました。 それは、本格的な3G / 2Gメディアゲートウェイの構築でした（それについては、後で詳しく説明します）。これは、ヤマルネネツ自治区の大規模なスイッチングシステムの不可欠な部分です。 ところで、ネットワークの信頼性を確保するために、各地域センターにはモバイルスイッチと呼ばれる個別のスイッチングシステムがあります。 しかし、ヤマルにとっては、より「柔軟な」システムを展開する必要がありました。



最初は、リモートバックボーンネットワークが設計された場所を正確に「見る」ことすら簡単ではありませんでした。各地域のオペレーターの技術的条件はどこでも同じであり、基本アーキテクチャをスケーリングする集中原則に基づいています。 プロジェクトが詳細に「成長」したとき、少し後にヤマルの設計問題を「試飲」しました。 私たちが遭遇した主な特異点は、「リモート」バックボーンネットワークの展開コストは、慎重に慎重に計画する必要があるということでした。この問題は、実施されている近代化の収益性に常に隣接しているためです。





ヤマル・ネネツ自治区、サレハルドとラビトナンギの地理的位置



サレハルドとラビトナンギの加入者にサービスを提供している古い2Gモバイルスイッチでは、新しいサブスクリプションサービスを設定することはできませんでした。これはプラットフォームで許可されていませんでした。 特に、事業者間の携帯電話番号転送サービスは「上昇」せず、これは多くの制限の氷山の一角にすぎませんでした。



さらにもっと。 ヤマルのこれらの都市の2Gトラフィックが切り替えられた古いホールは、近代化の機会を失い、拡大する方法がありませんでした。 問題は、新しい3G / 2Gメディアゲートウェイのラックとその近代的な「ハーネス」が、既存の部屋に置く場所が他にないことでした。 近年、地域のモバイルネットワークは急速に成長し始め、基地局（BS）の構築は、加入者トラフィックを「運ぶ」ことで、機能しやすい「コンベヤ」のレベルに達しました。音声トラフィックを渡すためのネットワーク。 さらに、より大きなサレハルドではなく、ラビトナンギの小さな町に古い機械室があり、さらに拡大するために、地域の中心部に近い川を「渡り」、そこに新しいモバイルスイッチングセンターを作成する必要がありました。 テクニカルを含め、同社の子会社のすべての人員はサレハルドにいました。 この「ゴーディアンノット」をどうにかして切る必要がありました。



用語をさらに明確にするために、キャリアグレードのモバイルスイッチは、加入者通話を接続し、地域でセルラーカバレッジを提供するすべての3Gおよび2G基地局を管理する大型コンピューターであることは注目に値します。 このスイッチは、事業者のモバイルネットワークの「心臓部」であり、地域の「コア」のアーキテクチャの重要な要素です。 2Gネットワ​​ークの場合、スイッチはMSC（モバイルスイッチングセンター）、3Gネットワ​​ークの場合-MSS / MGW（モバイルソフトスイッチ/メディアゲートウェイ）と呼ばれます。 3Gスイッチはユニバーサルネットワークノードであり、3Gと2Gの両方のベースステーションの音声トラフィックを同時に処理できます。 （現在、2G MSCスイッチは、カリーニングラードからカムチャッカまでのすべてのVimpelCom地域ネットワークから廃止され、現在では使用されておらず、時代遅れです。）モバイルスイッチの動作原理は、互いに何万ものモバイル音声通話の自動切り替えに基づいています。 この「スマート」マルチプロセッサマシンは、加入者の音声トラフィックを最適に圧縮して、オペレーターのチャネルを通過させることができます。



新しい3G / 2Gスイッチの重要な機能は、分散アーキテクチャの原則に従って製造されていることです。 この状況により、個別の3G / 2G MGW（つまり、音声メディアゲートウェイ）の地理的な「削除」により、小都市の加入者のローカル音声トラフィックを最大限に「ラップアップ」し、ベースステーションのチェーンを制御するローカルコントローラーに接続できます。 実際、今では、メディアゲートウェイをインストールすることにより、加入者に近い遠隔地にモバイルスイッチの「ピース」を構築し、「ビッグランド」のMSSブレインを使用して音声トラフィックを管理できます。 友人や親relativeに電話をかける加入者の場合、その場で音声通話を「まとめる」ことで、会話中の通話設定時間と通話遅延の両方が数倍短縮され、音声認識の品質が大幅に向上します。 大規模な地域のMSS / MGWスイッチをロードせずに、Salekhardにスイッチコンポーネントのインストールを設計し、その場でローカル音声トラフィックをドッキングできるようにしたのは、新しいスイッチの分散アーキテクチャを実装する可能性でした。





ネットワーク図Labytnangi-Salekhard-近代化前のノヤブリスク



まず、「大地」とは、ヤマロネネツ自治区の大規模なMSS / MGWスイッチボードの「コア」とそれに対応する「バンドル」を意味します。ヤマロネネツ自治区は、ノヤブリスクの大都市にあります（以下、近隣地域の同僚の遠隔支援と呼びます）センター）。 私たちのアイデアによると、現在600km離れたノヤブリスクとサレハルドの間で、アーキテクチャは加入者のトラフィック制御の信号交換のみを送信し、音声トラフィック自体は送信できません。これにより、消費されたリース帯域を削減することにより、ヤマル北部の都市への高価な輸送チャネルのレンタルを節約できます。



特別なプロジェクト自体に戻ると、1つの重要な事実に注意する必要があります。 Yamalのアーキテクチャの理論的検索に関する状況は、SalekhardとLabytnangiの加入者にサービスを提供しているモバイルスイッチのサプライヤー（ベンダー）が2Gスイッチング機器を廃止し、しばらくして技術サポートを完了することを意図したという事実によって最終的に促されました。 そして、そのような場合に通常起こるように、考えられるすべてのリスクが予想外に増加し、決定的な行動をとらなければなりませんでした。 プロジェクト計画フェーズは終了しました。 サレハルドのローカル条件での音声トラフィックの最適な「ラッピング」という構想の実装にすでに費やした時間。



ホールの建設と設備のセットアップ



ワーキンググループの同僚、設計されたすべてのサブシステムの相互作用手順を開発して同意したので、建設の準備を始めました。 私たちが最初にしたことは、メディアゲートウェイ用の新しいコンピュータールームを配置するための場所に関する情報を収集するために、サレハルドに「フィールドに」旅行したことです。



多くの建物をレビューし、多くの時間を費やした後、残念な結論に達しました。 キャリアクラスの機器用の機械室に変換できる部屋はありませんでした。 多かれ少なかれすべての通信を備えた首都サイトのレンタル価格は非常に高く、私たちにとって不当でした。 サイトの価格に間接的に投資された重要なコスト項目は、2つの独立したエネルギー入力と、カバレッジ2Gおよび3G（および将来の4G）の容量のさらなる拡大を考慮した、市内の電力コストです。 そしてもちろん、選択の困難な瞬間は、調査したポイントにすべてのチャネルとコミュニケーションを「引き上げる」ことができないということでした。 これは、プロジェクトの実際の「待ち伏せ」であり、まだ始まっていませんでした。



考えられるすべてのオプションを検討した結果、これらの北部の都市では、オペレーターの状況に応じて必要な時点で「ゼロからのクリーンフィールド」に機械室を構築する方が簡単であることに同意しました。 言うのは簡単ですが、そう簡単ではありません。 結局のところ、それは永久凍土と北部の厳しい気候の問題であり、「クリーンフィールド」は、機械室の実装の基礎が地上の気密容器の形をとらなければならなかったことを意味しました。 最初に述べたように、2つの独立したエネルギー入力がそこにプルアップされ、2番目に、オーバーヘッド無線中継リンク（RRL）を介した通信の複製トランスポートチャネルがスローされました。 基本的に、ローカル3G / 2G MGWメディアゲートウェイについては、あらゆる意味で予約された独自の「ハウス」を構築し、オペレーターワークステーションを装備したままでした。



将来の機械室の図面に必要なすべての機器ラックを準備し、拡張を考慮して、コンテナの適切な寸法を決定し、自分で建設を開始する場所を選択しました。 サイトの建設作業は、論理的に2つの段階に分けられました。 ステージ1-メインコンテナの建設。機械室用の「生命維持」システムを装備し、ビデオ監視システムを接続すると同時に、変電所を別のコンテナに構築します。 ステージ2-3G / 2Gバックボーンネットワークの通信機器のインストールとインストール、MGWの試運転と統合、ノヤブリスクの大型MSS / MGWスイッチの「コア」、およびLabytnangiの古いスイッチ、すべてのサブシステムのマルチベンダーテストモバイルサービスの検証、およびパイロットの商用運用のためのシステム全体の準備。



「ウィッシュリスト」の建設会社と請負業者の選択は、競争を通じて行われました。 永久凍土は冗談ではありません。 その後、請負業者と一緒に、春にウラルの製鉄所の1つにコンテナの製造の注文を送り、生産の終了を待ち始めました。 その印象的な寸法と重量は、基地局用に標準で提供されているボックスでは一般的ではなかったため、製造は「注文どおり」に行われました。





コンテナの杭打ちと基礎準備



痛みを伴う待機の後、コンテナは工場から受け取られ、その場所に届けられました。 私たちの「家」の組み立てはすぐに始まりました。 一般に、北部のハードウェアボックスの建設は、責任があり非常に骨の折れる問題であると言えます。 そして、新しい3G / 2Gメディアゲートウェイ用のコンテナは、これらのパーツのハードウェアボックスを構築するすべての技術段階に従って構築されました。 たとえば、プロジェクトのこの部分の土壌では、不可抗力なしで「家」の建設が完了したなど、乗り越えられない困難はありませんでした。



機械室のコンテナは、熱と防水性が強化され、信頼性が高いことが判明しました。 エアコンと自動暖房システムが設置されました。 コンテナ内の動作温度は常に約22度に維持され、完全な空調制御が行われます。 火災、洪水センサー、完全な監視システムがあり、誤動作のケースを検出できます。 さらに、別のコンテナ内のサイトで、必要な配電装置を備えた独自の変電所が組み立てられました。





変電所に接続された開閉装置



小さな象徴的な「新築」を伴う第1建設段階の結果によれば、機械室のコンテナを稼働させ、すでに現場で「住む」ことが可能でした。 さらに、寒さを引き締め始めたため、第2段階で加速する必要がありました。



コンテナの製造と同時に、コアネットワークに必要な3G / 2Gテレコム機器がすべて購入され、すでに地域の倉庫で私たちを待っていました。 しかし、工場からの「家」の生産と配送の長期化、請負業者による組み立て、さらに現場での独自の変電所の立ち上げにより、晩秋にプロジェクトの第1ステージの利用が延期されました。 短い夏と秋が終わり、北極圏で雪が降り始め、第2段階の実装に「追いつく」ことが重要でした。購入したスイッチング機器の配送とテストです。



北部との協力の経験から、チュメニ市にある倉庫から鉄道でオペレーター機器を送るには、少なくとも1か月前、またはそれ以前に改善する必要がありました。 ラックを備えた列車は、北極圏の私たちの場所にスィクティフカルだけを通って到着し、同時に別の4つの地域を迂回して、ラビトナンギの鉄道駅にのみ到着しました。 これ以上の方法はありませんでした。Ob川は私たちのサイトへの道を塞いでいます。 LabytnangiからSalekhardへの鉄道、橋もまだありません。 このような状況を考慮して、私たちは機器を1か月以上でLabytnangi輸送会社に送り、到着後すぐに鉄道から荷を下ろし、川を渡ってフェリーに向かいました。 「大きな」土地でロジスティクスを計画することを事前に知っていれば、悪化する天気予報に大いに要約されます。



このルートの最終日の5時間だけで、多数のクレートと箱をすべて運ぶことができました。 ところで、人々は川を渡ることができました、オフシーズンには都市のミニバスのようなエアクッションボートがありますが、貨物輸送のための川を通る道路はもはやありませんでした。 川は、リモートサイトに到達しようとするすべての試行を一度に「遮断」します。





Salekhard – Ob川を渡るLabytnangi冬の交差点



これは時々起こります。 私たちは「パッケージ」を必要なすべての「鉄」で両方の北の都市に詰め込み、予備でさえ、航空便でそのような貨物を送ることができなかったので、私たちはそれを鉄道で1ヶ月間運転しました。 そして、Ob川を通る貨物輸送の通過のために安全な道路が再び開くまで、「天気の海」で3週間待つ以外に何もすることはありませんでした。 しかし、これを生き延びました。 ただし、このような正確な日付のずれと北への長期にわたる負の気温の到着が、厳しい気候条件で動作する新しい3G / 2Gメディアゲートウェイの完全な立ち上げに役立つのは、まさにそのようなものだと誰が考えたでしょう。



テレコム「鉄」の必要な箱がすべてマッシュルームに届けられるとすぐに、新しい機器ベンダーの代表者はサイトへのフライトによって「放棄」され、作業は加速して「沸騰し始め」ました。 エンジニアは作業を事前に通知され、指定された曜日と時間に、干渉することなく、お互いを遅らせることなく、Salekhardに1つずつ飛んで、統合の期限を待ちました。これは非常に貴重でした。 技術者の同僚とともに、ベンダーとその請負業者は、コアネットワークのさまざまなサブシステムを互いにドッキングし、経験を交換し、テストと再テストを行い、すぐに作業に弾みがつきました。 私たちの目の前でシステムが一緒に成長し始めました。





新しい機械室への通信機器の設置



新しい3G / 2G MGWメディアゲートウェイのテストは、コンテナ内で組み立てられた2つのテスト基地局3Gおよび2Gで行われましたが、減衰器を介してテストアンテナに接続されました。 基地局の信号レベルに必要なしきい値を設定すると、テストスマートフォンで3G-2G-3Gモバイルトランジションのすべてのチューニングテストに自信を持って合格しました。 同様に、都市間のすべてのMSS-MGWシグナリングテストが実行され、次にモバイルサービステストが実行されました。 その結果、コアネットワークおよびトラフィック制御システムに必要なすべてのネットワーク要素が機械室で「発生」しました。 そして当然、バックアップバッテリーのテストと試運転も行われました。 サイトの2番目の構築フェーズはスムーズに終了し、テストされたバックボーンネットワークは最終的にトラフィックを受信する準備が整いました。





バッテリーバックアップ



ここで注目すべきことは、ヤマルのプロジェクト管理の詳細は、これらの北部の都市では常に技術専門家が不足しているということでした。 会社内の類似のモバイルサブシステムを専門とする近隣地域のエンジニアを探し、同僚に北上して、ベンダーや地元の技術者が大企業のネットワークの一般的な要件に合わせてハードウェアを最適に構成できるように頼む必要がありました。結局のところ、サレハルドの地元の同僚は新しいハードウェアの調整者ではありませんでしたが、共同でスイッチングネットワークを設定する過程で、彼らは多種多様なサブシステムの装備を専門とする真のマルチステーションオペレーターになりました。その結果、プロジェクトは「ホームストレッチ」に展開され、私たちの前に「ライブ」加入者トラフィックが転送されました。



加入者トラフィック転送
