plink：IPアドレスを非表示にし、5分でトラフィックを暗号化します

すべてのインターネットユーザーは、少なくとも一度はWeb上の匿名性について考えました。 この記事では、IPアドレスを確実に隠し、送信された情報を暗号化する独自のプロキシサーバーを構成する最も簡単な方法の1つについて説明します。



私は独創的であるふりをしません、以下に述べるすべてが多くの人に知られていますが、確かにこの情報が役に立つと思う人々がいます。



IPアドレスを非表示にし、トラフィックを暗号化および圧縮するには、plinkユーティリティを使用してSSHトンネルを構成します。 このユーティリティは、 公式サイトからダウンロードできます。



前提条件-SSHサポートのあるホスティングが必要です。 共有ホスティング、VPS / VDS、または専用サーバーにすることができます。 独自のWebサイトを持っている場合（独自のWebサイトを持っていない場合など）、SSHサーバーのユーザー名、パスワード、およびアドレスをテクニカルサポートサービスに確認します（FTPと一致する場合があります）。



plinkユーティリティの構成



plink.batファイルを作成し、その中にテキストを入れます

plink -ssh _SSH_ -C -N -l -pw -D 127.0.0.1:8081

pause





（「login」、「password」、「SSH_server_address」を実際のユーザー名、パスワード、SSHサーバーアドレスに置き換えます）。

plink.exeファイルがある同じフォルダーにplink.batファイルを保存して実行します。 すべてが正しく実行されると、およそ次のテキストを含む黒いウィンドウが表示されます。



「y」（ラテン）キーとEnterキーを押します。



おめでとうございます、SSHトンネルが作成されました。 このウィンドウを閉じることはできません。閉じないと、サーバーへの接続が閉じます。



間違ったユーザー名またはパスワードを入力した場合、対応するエラーが表示されます。





FireFoxでプロキシを構成する



FireFoxが作成されたSSHトンネルを使用するには、[ツール]-> [設定]-> [詳細設定]-> [ネットワーク]-> [構成]メニューに移動します。 [プロキシサービスの手動調整]チェックボックスをオンにし、[SOCKSホスト]列でアドレス127.0.0.1、ポート8081を指定し、[SOCKS5]ボックスをオンにします。 残りの列は空白のままにします。



確認する



確認するには、サイトhttp://2ip.ruを開きます。 すべてが正しく実行されると、IPアドレスの代わりにSSHサーバーのアドレスが表示されます。



IPアドレスは非表示になり、トラフィックは暗号化されて圧縮されます。



Internet Explorerのプロキシ設定



まだInterner Explorerを使用している場合は、[ツール]-> [インターネットオプション]-> [接続]-> [ネットワーク設定]に移動します。 [ローカル接続にプロキシサーバーを使用する]チェックボックスをオンにして、[詳細]ボタンをクリックし、[ソックス]列でアドレス127.0.0.1、ポート8081を指定します。



QIPのプロキシ設定



接続設定メニュー。 [プロキシタイプ]リストで[SOCKS5]を選択し、[プロキシサーバー]列で127.0.0.1、ポート8081を指定します。



その他のプログラム



SOCKSをサポートしないプログラムまたはSSHトンネルを介した直接作業を拒否するプログラムでプロキシを構成するには、 FreeCapプログラムまたは同様の製品を使用できます。 このプログラムの設定については詳しく説明しません。接続パラメーターでは、アドレス127.0.0.1とポート8081を指定する必要があるとしか言えません。



Webサイト（ホスティング）がない場合はどうすればよいですか？



理論的には、SSHトンネルでは、SSHサポート付きの無料ホスティングを使用できますが、高品質の無料ホスティングは知りません。



こんにちはビルゲイツ



説明されている方法は、Windows XPおよびWindows Vistaでテストされています。 Windows 7では、理論的には動作するはずですが、確認する方法はありません。

* nixライクなオペレーティングシステムでは、plinkユーティリティの代わりにsshを使用できます。



サーバーのセットアップ（オプション）



VPS / VDSまたは専用サーバーがある場合は、ssh経由で接続する権限のない特別なユーザーを作成することをお勧めします。



安全なパスワード入力（オプション）



SSHトンネルの開始ごとにパスワードを手動で指定する場合は、スタートアップコードから「–pwパスワード」を削除します。

plink -ssh _SSH_ -C -N -l -D 127.0.0.1:8081





便利ではありませんが、より安全です。

絶対的な匿名性はありません



技術的な詳細は説明しませんが、この方法では絶対的な匿名性は得られません。 彼らがあなたを探す愚かなことをしないでください。
