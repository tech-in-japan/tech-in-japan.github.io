SQL Server Management Studioの便利な拡張機能

読者の皆さん、こんにちは！



この投稿は、SQL Serverで作業するときにSQL Server Management Studio（SSMS）を使用する人にとって興味深いものになります。

SSMS用のSQL Refactor Studioの拡張についてです。

この拡張により、日常の日常業務の時間を大幅に節約できます。



だから、私は私の意見で最も要求される機能を検討し始めます：



  1. グリッド内の選択されたセルの合計。 常に数字を扱う人のためのキラー機能。 以前は、グリッドの内容をExcelにコピーし、それで作業する必要がありました。 グリッド内の必要なセルを選択するだけで十分で、スタジオはそれらをSUM、MIN、MAX、AVGで見つけます。



  2. スクリプトオブジェクトの取得。 クエリエディタで、オブジェクトの名前にカーソルを置き、F2キーを押してスクリプトを新しいウィンドウに表示します（またはCtrl + F2キーを押して同じウィンドウにスクリプトを表示します）。 以前は、ObjectExplorerで必要なオブジェクトを検索し、そのスクリプトを作成する必要がありました。
  3. クリップボードのデータに基づいてSELECTを生成します。 プレートをExcel（Access）からデータベースにすばやくロードする必要があるとします。 もちろん、DTS、bcp、OPENROWSET、およびExcelでのスクリプトの手動作成はキャンセルされていません。 しかし、我々はより速く、より簡単にする必要があります;）。 したがって、ソースからクリップボードにデータをコピーし、クエリエディターでRMBを使用して[SELECT ..を生成]を選択します。 次に、生成された要求をSELECT INTOまたはINSERT INTOに追加します。



  4. 依存関係を検索します。 標準の依存関係ビューアの優れた代替手段。

    プロから：
    • ネイティブビューアーが見つけられない依存関係を見つける
    • 複数のデータベースを検索する
    • オブジェクトスクリプトを表示する機能
    • たくさんのオプションと機能




  5. テキストを検索します。 TODOのメモを残し、それらすべてを検索するプロシージャ/ビューを作成するとします。 ObjectExplorerでデータベースを選択し、PCMでアイテム「SQL Refactor Studio-> Find code」を選択し、検索したいテキストを入力して検索を開始します。

    プロから：
    • 正規表現をサポート
    • 複数のデータベースを検索する
    • ディスク上のファイルを検索する機能
    • たくさんのオプションと機能




  6. オブジェクトの名前を変更します。 すべての依存関係を考慮して、データベースオブジェクトの名前を安全に変更できます。
  7. データ検索。 テーブル間でデータを検索できます。



  8. さまざまなリファクタリング。 CRUDメソッドの追加 、履歴のトリガーの導入 、 ルックアップテーブルの追加 、 列の移動

  9. テーブルデータに基づいてINSERTスクリプトを生成します 。
