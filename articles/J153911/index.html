Windowsジョブのマルチシート



VMPlayerに基づいたマルチユーザーワークステーションを作成する通常とは異なる方法について説明します。



このソリューションは商用運用用に開発されたもので、この問題のソリューションへのメッセージとして、購入した機器の現在の特性はユーザーとユーザーが使用するソフトウェアのニーズを上回ると述べられました。 もちろん、これは主に典型的なオフィスプランクトンと多くの周辺ポストに適用されます。 また、完全に正当化された理由により、管理者がワークステーションの更新要求を拒否した、または拒否する可能性が高い人の問題を解決するのにも役立ちます。 悲しいことに、それは起こります...



ソリューションが適用に値するようになるために、次の要件が提示されました。

  • エンドユーザーのオペレーティングシステムとして、Windowsはそのままにしておく必要があります。
  • 理想的には、2人の従業員のシステムユニットのコストは、その時点で購入した典型的な従業員のコストに匹敵するはずです。
  • ユーザーの作業は、1台のコンピューターが使用されているという事実のために、著しく遅くなったり不快になったりしてはなりません。


さて、実際には、ビジネス要件は明確であるため、ソリューションに移る時が来ました...



技術部



開発は現在控えめなパラメーター（AMD Athlon 64 X2、GeForce GT 440、DDR II 4Gb、HDD 320GB）を備えたワークステーションで実施されました。結果は十分に受け入れられましたが、上記の要件を満たすために技術コンポーネントを選択する際には、以下の推奨事項を考慮する必要があります：

  • 2つのコアを持つプロセッサ -各VMに1つが割り当てられます。もちろん、最新の仮想化テクノロジのサポートが望ましい場合は、「鉄」ワークステーションに近いパフォーマンスを期待できます。
  • RAMは特に高価なコンポーネントではないため、後悔することなく、職場ごとに3Gbを計算し、ホストシステム用に1Gbを計算する方が良いでしょう。 ただし、特定の状況では、各場所で1.5 Gbで処理できる可能性があります。
  • 仮想マシンが配置されるハードディスク。ワークステーションごとに個別であることが望ましい。これにより、I / O操作に独立性が与えられ、仮想マシンの相互干渉が少なくなります。 コスト削減の目標がより高い優先度である場合、この要件は省略できます。
  • マザーボードには 、できれば既に2つのビデオ出力が必要です。これにより、別個のビデオカードを購入する必要がなくなります。 ただし、ビデオカードを使用したい場合や、マザーボードにビデオ出力がない場合は、もちろん外部を使用できます。


周辺機器、つまりモニター、キーボード、マウスに関する注意がいくつかあります。一見奇妙に見えますが、実装は難しくなく、コストもかかりません。これはソリューション要件を満たすために重要です。

  • モニター モニターを選択する場合、DVIおよびVGAビデオ出力を備えたマザーボードはDVI-> VGAアダプターをサポートしないことに注意してください。 したがって、使用可能なモニターとビデオ出力が互いに一致していることを確認する必要があります。
  • キーボードとマウス 。 これはおそらくコメントではなく、これらのHIDデバイスの選択に対する制限です。 まず、すべてのデバイスがUSBでなければなりません。 次に、あるキーボードのVID：PIDが別のキーボードのVID：PIDと一致してはなりません。 同じ条件がマウスに適用されます。 簡単に言えば、キーボードモデルとマウスモデルは異なる必要があります。 この制限は、VIDとPIDを使用して仮想マシン間でデバイスを配布するという事実に関連しており、この条件を満たすことは難しくないため、同じモデルを接続する可能性は検討されていません。


たとえば、デュアルシートの標準として採用した構成を示します。IntelCore i3、MB（DVI + VGA、オーディオ、GLan）、HDD 500Gb x 2 。



したがって、技術的な部分は説明され、正当化されます。 次に、ソフトウェアの詰め込みと作業の原則について説明します。



準備する



さあ、始めましょう。 デュアルシートを作成するには、いくつかの基本的なコンポーネントを準備する必要があります。



プラットフォーム


デュアルシート用のプラットフォームを作成する最初のステップは、Ubuntu Server 12.04 x64のインストールです。 システムをかなり簡単にインストールした後、残りのコンポーネントの機能を保証するパッケージを提供すると同時に、いくつかの有用なユーティリティを提供します。

インストール後、システムの起動がユーザーに透過的であるようにシステムを構成します。ホストシステム自体のグラフィックデザインは目立たず、さらに良いことに、完全に存在しないようにします。

システム環境


次に、システム環境を準備する必要があります。ユーザーといくつかのフォルダーを作成します。

  • vmは、nodmにログインし、そこから仮想マシンを起動するユーザーです。
  • / home / vm / user01および/ home / vm / user02は、ユーザー仮想マシンのフォルダーです。 システムに2台のハードドライブが用意されている場合、2番目のハードドライブをこれらのフォルダーのいずれかにマウントする必要があります。


仮想ワークステーション


仮想マシンの作成とシステムのインストールはあなたに任されます。このプロセスはほとんどの読者になじみがあると思います。 しかし、目標はユーザーの快適な仕事であるため、まだいくつかの推奨事項があります。

  • 非常に非効率的な仮想ディスクサブシステムを取得するため、仮想ディスクはシステムに関係なくSCSIである必要があり、IDEを選択する必要はありません。
  • 未使用の機器 （プリンター、フロッピーディスク、シリアルポートなど）は、仮想マシン構成から削除する必要があります。
  • VMWareツールは、ゲストシステムを効果的に運用するための前提条件です。


以下のスクリプトの重要な規則：

  • 両方のvmxファイルの名前はworkstation.vmxと予想されます。
  • 仮想マシン名は、それぞれuser00とuser01でなければなりません。


仮想ワークステーションが準備され、それぞれの場所に配置された後、チューニングを実行し、再び最大のパフォーマンスを絞り込みます。

すべてのUSBデバイスは2つのカテゴリに分類できます。永続的に接続されているものと、操作中に接続されているものです。

最後のタッチ



ソリューションを完了するには、次のシナリオが機能するはずです。

  1. システムユニットをオンにすると、仮想マシンが自動的に起動します。
  2. 隣接する職場が機能している場合でも、各ワークステーションのオン/オフを切り替えることができます。
  3. すべてのジョブをオフにすると、システム装置がオフになります。


ポイント1および3は、上記のスクリプトを使用して実装されます。 ポイント2は部分的に実装されています。 仮想マシンの1つをオフにしても、2番目の仮想マシンは動作し続けます。 しかし、マシンを再びオフにする方法は？ 質問は非常にエレガントに解決されます。このために、仮想マシンを起動するために電源ボタン（これはシステムユニットにあるボタン）を切ります。 これを行うには、 /home/vm/.fluxbox/keys



ファイルにいくつかの行を/home/vm/.fluxbox/keys



ます。
