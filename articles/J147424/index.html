フォトアルバム

同じことが起こり、私の手は親の写真アルバムに届きました。 少なくとも1,000枚の写真があります。 かつて、私の親はそのようなカメラを持っていました：







素晴らしいことであり、適切なスキルを備えていれば、今でも通常のデジタルカメラよりも悪い結果を出すことはできません。 しかし、その大きなマイナス点は、アナログ写真を撮影することであり、そのような写真はあらゆる点で黄色に変わり、腐敗し、劣化する傾向があります。 したがって、写真のアーカイブを保存する場合は、写真をデジタル化するか、もちろん、アンティークのように保存して、さまざまな遊び心のある小さな手を避けてください。



もちろん、これをスキャンすることは、画像をデジタル化するプロセスにおいて長く非常に重要なビジネスです。 そして、何をすべきか？







さて、最も明白な解決策はPicasaです。 そしてまた？ リモートサービスではなく、個人用アプリケーション、さらにはオープンソースでもありますか？







Ostora.PhotoはWebアプリケーションです。 php + mysqlで書かれています。 Ostora.Photoの中心​​には、同名のファイルマネージャーであるOstoraがあります。 ファイルマネージャーの左側のインターフェイスから、ユーザー権利管理、ファイルとクォータの「トリッキーな」削除。 私の目的にとって、これらは本当に重要な機能です。 また、たとえば、ファイルバージョンの保存はここでは必要ないため、削除されます。 グラフィックファイル形式のみをアップロードできます（形式はconfig.iniに追加できます）。 1回クリックすると、jqueryプラグイン（ ？ ）FancyBoxで写真が開きます。 写真は、キーボードのボタン「左」、「右」、マウスのスクロール、またはパネルのボタンでスクロールできます。







フォトアルバムには他に何が必要ですか？ 私にとって、これらはタグ、メモ、お気に入り、そして写真に写っている人や物をマークする機能です。



ところで、偶然にも、いくつかのタグやタグ付けされた人をすぐに選択できるようになりました。 たとえば、「海」というタグと「クセニア」および「パシャ」のマーク-海でクシュシャとパシャを一緒に表示します。

並べ替えの不要な単語でボタンまたは十字をクリックすると、並べ替えをリセットできます。







ちなみに、「トリッキー」削除とは、イメージを削除済みとしてマークする機能であり、管理者権限を持つユーザーのみが完全に削除できます。 これは、ユーザーの偶発的な突発行為から重要なファイルを保護する方法です。



GitHub： https : //github.com/Zazza/photo



アプリケーションのインストールと構成は、FM Ostora（ http://bitboom.ru/ostora/ ）と大差ありません。



config.ini：



...

[fm]

rgb = 0xFFFFFF //プレビュー用の塗りつぶし色

quality = 100 //圧縮品質のプレビュー

maxUploadSize = "20 MB" //最大アップロードファイルサイズ

pre_width = 210 //生成されたプレビューの幅

pre_height = 170 //生成されたプレビューの高さ

file_types = "* .jpg; * .gif; * .bmp; * .png" //ダウンロード用のファイル形式

file_upload_limit = "100" //一度にロードできるファイルの数



[db]

adapter = "mysql"

host = "localhost"

ユーザー名= ""

パスワード= ""

dbname = ""

...



ユーザー名/パスワード：adminostora / adminostora
