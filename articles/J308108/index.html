HPE Mobile Centerがテストしているもの

HPE Mobile Centerは、本質的に、チームがモバイルアプリのテスト、監視、ライフサイクル管理アクティビティを集中化するのに役立つゲートウェイです。 これは、一連のモバイルデバイス（物理および仮想、クライアントおよびクラウド）を集中的に使用し、HPE ALM製品とオープンソース（Appium / Selenium）の両方と統合することで実現されます。







HPE Mobile Centerは、モバイルアプリケーションとサイトのモバイルバージョンのテストに限定されません。 初期のHPE MCパッケージを購入することにより、ユーザーはAppPulse Mobileクラウドアプリケーション監視サービス（2つのアプリケーションに限定）にアクセスし、Fortify-on-Demandにアクセスしてアプリケーションセキュリティをテストします。 自動および手動テストの機会も最も興味深いものであり、特にHPE UFTの1ライセンス-自動機能テストモジュールも最小のHPE Mobile Centerパッケージに含まれているため、アプリケーションの手動テスト用ツールであるSprinterはHPE MCの所有者には完全に無料です。



名前自体-モバイルセンター-モバイルテストの重要性と特異性を強調しています。 イスラエルと中国のHPEソフトウェア開発センターに部門が設置され、約70人の従業員がモバイルテストツールやその他の関連トピックの開発に携わっています。 そして、理由がないわけではありません。この分野では、問題とその機能は一見しただけでは必ずしも明らかではありません。



HPE Softwareのソフトウェアの開発およびテスト用製品-ツールとテクノロジーの大きなパッケージであり、HPE ADM（HPE Application Delivery Management）と密接に関連しています。 UFT、Sprinter、およびMobile Centerはすでに上記で言及されています。 さらに、HPE ADMには以下が含まれます。ALM Octane-アジャイルおよびDevOps手法を使用した開発および実装プロジェクトの計画、実装、およびサポートのためのALM（アプリケーションライフサイクル管理）バージョン。 開発者（MS Visual Studio、Eclipse、IntelliJ IDEA）が使い慣れた環境でJava、JavaScript、またはC＃で単体テストを作成するためのHPE LeanFTが、独自のHPE UFTテクノロジーを使用しているため、開発者によると、そのようなテストの開発が大幅にスピードアップします; HPE LoadRunner-長い間無料で利用できる（50人以下の仮想ユーザーの負荷に対して）誰もが好む負荷テストツール。 HPE Network Virtualization、ネットワーク仮想化ツール、HPE Service Virtualization統合環境およびサービスのスタブの迅速な開発。



しかし、おなじみの名前とおなじみのツールは、モバイルアプリケーションの世界が特別であるという考えから私たちをそらすべきではありません。



クラウド内のモバイルデバイス



もちろん、その特異性は、モバイルデバイスの悪名高い断片化だけではありません。これは、Androidプラットフォームの批評家にとって一般的な場所になっています。 ここでは、モバイルデバイス、モバイルジェスチャなどのセンサーを使用するための幅広い可能性を強調できます。他のアプローチが必要です。 もちろん、モバイルエミュレーターを使用できます。HPEMobile Centerは、Google SDKとGenymotionの商用エミュレーターの両方をサポートしています。 しかし、このアプローチでは、アプリケーションが市場に存在する何千ものデバイス上のハードウェアプラットフォームとどのように対話するかという根本的な質問に正確に答えることはできません。



2つの方法があります。 まず、テストは依然として「ブラックボックス」ではなく、Sencha Touch、Apache Cordova / PhoneGap、Adobe Air、またはXamarinなどの有名なソフトウェアプラットフォームで開発されたアプリケーションです。 実際、これらのプラットフォームは、製品開発段階ですでに鉄の多くの機能から抽象化することを可能にします。



2番目（または最初の値による）：テストカバレッジ。 フレームワークとコードの分析からではなく、これらのアプリケーションのユーザーに関する分析と知識から行くことは理にかなっています。 アメリカの企業内で使用するアプリケーションをテストしている場合（そしてそのような顧客が多い場合）、アジアの安価なスマートフォンをテストする必要はありません。 逆もまた同様です。中国市場向けのアプリケーションをテストすることで、地元の市場で人気のあるデバイスに精通している上海オフィスの開発者は、よりよく理解できるでしょう。 モバイルセンターを使用すると、モバイルデバイスのすべての地域公園を集中化して、すべてのユーザーにそれらへのアクセスを提供できます。 これらのデバイスは、JenkinsとBambooのプラグインを介して、モバイルセンターのユーザーによる継続的なテストのCI / CDサイクルにも使用できます。



さらに、モバイルビジネスの世界では変化が特に速いため、人生は独自の調整を行います。 マイクロソフトはXamarinを買収しました。つまり、顧客の関心が急上昇する可能性に備えて、これらの技術に努力の一部を集中する価値があるということです。 Adobe Airの人気が衰えた場合、開発者がCordovaなどの現在上昇中のテクノロジーに切り替えるほうがより有益です。 「目を開けて鼻を風に当ててください」と、イスラエルのHPEモバイルセンターの開発センターの責任者であるYevgeny Karasikは言いました。



仮想化の継続：Now Networks



約1年前、HPEはネットワーク仮想化ソフトウェアであるHPE Network Virtualization、NVに特化した会社であるShunra Softwareを買収しました（従業員もイスラエルにいて、長い間開発センターで働いているため）。 NVを他のHPEソフトウェア製品と統合する作業が開始されました。 モバイルセンターには、競合他社から脱却する別の機会があります。NVもモバイルセンターパッケージに含まれています。 彼らがビジネスで言うように、付加価値。



顧客はすぐに新製品に興味を持つようになりましたが、驚くことではありません。その前に、モバイルデバイスが物理的にテストされました。携帯電話のテスターが全国を回り、さまざまなネットワークや地域でのアプリケーションの動作を説明しました。 イスラエルにとっては、問題はおそらく小さなものですが、ロシアではこれは本当の狂気になります。理由として、国内の大手携帯電話会社の1つがこれらの目的でHPEモバイルセンターを長年使用してきました。 もちろん、希望する人は国のさまざまな場所で見つけることができます-テスターのコミュニティは巨大ですが、そのような全国的なテストの結果の信頼性と期間はすべての顧客、特に企業の顧客に適さないかもしれません。



もちろん、真剣な企業顧客はHPE MCユーザーの中核です。 それらのほとんどは、すでにHPE製品やサービスを使用しています。 彼らは、「外の世界」からのアクセスのためだけでなく、モバイルアプリケーションを必要とします。それは、世界銀行、グローバルな小売業者、または自動車産業の巨人であろうと、生活を促進し、自分の従業員の仕事を加速するためです。 内部アプリケーションの場合、企業ネットワーク上のアプリケーションの正常性の予測可能性は、異種で複雑であるため、特に重要です。



ところで、ネットワーク仮想化は、利便性と信頼性だけの問題ではありません。 仮想ネットワークで仮想デバイスを実行することで、分析用の情報を収集し、クライアントに最適化の推奨事項を提供できます。「アプリケーションにはヘッダー圧縮がないため、低速ネットワークでは正しく動作しません」など



仮想的にテストされたネットワークの数が増えるとしたら（そして他の方法は？）、巨大なデバイス群が存在する場合、組み合わせの数は天文学的なものになります。 ここでは、人工知能技術を使用せずに行うことはできません。 幸いなことに、Mobile Centerの開発者はもはやそれらを使用するだけではありませんが、それらに賭けたと言えます。 これは、ビッグデータ、感情分析（感情の機械分析）、機械学習、その他のファッション技術の分野です。 将来的には、分析モジュールの人気が高まることが予想されます。
