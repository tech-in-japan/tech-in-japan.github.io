NooLite Ethernet Gateway PR1132-スマートフォンとタブレットからの光を制御

ハブで繰り返しnooLiteシステムについて書きました 。これにより、無線で照明や電化製品を制御できます。 最近、nooLiteシステムのメーカーであるベラルーシの会社Nootekhnikaが新しいデバイス、nooLite Ethernetゲートウェイを発売しました。これにより、スマートフォンやタブレットからの光を制御できます。

会いましょう！ イーサネットゲートウェイPR1132：







デバイスはローカルネットワークで動作します（たとえば、アパートの周りにインターネットを配布するルーターに接続できます）。 ライトコントロールは、ウェブインターフェース（ブラウザなど）またはHTTP APIを介して行われます。



技術仕様


ゲートウェイには32の制御チャネルがあります。 これは、32個の独立した照明器具グループを制御するために使用できることを意味します。 私見、アパートのためにこれは十分以上です。 カントリーハウスの場合も同様です。 ゲートウェイに従来のnooLiteリモートよりも強力な送信機があるという事実は勇気づけられます。 ゲートウェイの送信機電力は、コントロールパネルで約5 mW-約3 mWです。 このため、電源ユニットの最大範囲は100 mに拡大されます（リモコンを使用すると、最大50 mの距離で電源ユニットを制御できます）。

ゲートウェイには、トランスミッターに加えてnooLiteレシーバーもあり、PT111温度/湿度センサーからデータを受信できます（現在はまだ利用できません）。 最大4つのセンサーをゲートウェイに接続し、Webインターフェイスを介して読み取り値を表示できます。

ゲートウェイには、220 V、50 Hz、消費電力-3.5ワットのネットワークが搭載されています。



外観と機器




イーサネットゲートウェイは、輸送中の損傷を避けるために、非常に厚い段ボールの箱に入っています。 箱にはデバイスの概略図があり、その主な特徴が示されています。







内部は、PR1132ゲートウェイ自体とそのアンテナ、電源、指示書、1 mのパッチコードパッチケーブルです。







このデバイスは珍しいデザインで違いはありませんが、非常にモダンに見えます。 ゲートウェイのサイズは比較的小さい：115x85x35 mm-もう少し石鹸入れ:)







フロントパネルには、モデルの名前、nooLiteロゴ、およびゲートウェイのLEDインジケーターが表示されます。







ゲートウェイが動作しているとき、インジケータは常に点灯しています。 インジケーターはあまり明るくありません。私の場所では、ゲートウェイはクローゼットの寝室にあり、夜間はインジケーターは干渉しません。



背面パネルには、送信アンテナを接続するためのSMAコネクタ、イーサネットケーブルを接続するためのコネクタ、および電源用のコネクタがあります。 アンテナとEtherntetケーブルのコネクタの間にサービスボタンがあり、これを使用してゲートウェイをファームウェア更新モードにするか、設定を標準設定にリセットできます（ハードリセットなど）。

接続


ゲートウェイの接続に移ります-ここではすべてが非常に簡単です。







イーサネットケーブルをルーターに接続し、電源ケーブルをコンセントに接続します。LEDインジケータが点灯します-ゲートウェイは動作準備完了です。 ブラウザ192.168.0.168のアドレスバーに入力すると、以下が表示されます。







デフォルトでは、静的IPアドレス192.168.0.168とマスク255.255.255.0がゲートウェイ設定で設定されています。 ネットワークのパラメーターが異なる場合、互換性のあるパラメーターを使用してゲートウェイをネットワークに接続し、その設定を変更する必要があります。



インターフェース


ゲートウェイのWebインターフェイスは、ライトコントロール、タイマー、設定の3つの大きなセクションに分かれています。 デフォルトでは、照明制御セクションが開きます。 セクション間の遷移は、ページの上部にあるメニューを使用して実行されます（タブと同様）







インターフェイスは、コンピューターとスマートフォンの小さな画面の両方に正しく表示されます。

コンピュータ上では、コントロールが大きすぎるというわずかな感覚がありますが、通常は普通に知覚されます。

また、Windows Phone 8ブラウザー（Nokia Lumia 625を参照）では、ラベルの一部が非常に大きなフォントで表示されている機能にも気付きました。 私の知る限り、これはWindows PhoneのInternet Explorerの機能です。ページのテキストを適切に選択します。 コンテンツのあるサイトを表示する場合、これは便利ですが、ゲートウェイに似たアプリケーションを使用する場合、あまり見栄えがよくありません。 一般的に-重要ではありません。 AndroidとiPhoneでは、すべて問題ありません。



調光部


このセクションはデフォルトで開いています。 これには複数のグループが含まれ、各グループにはコントロールが含まれます。 たとえば、部屋のグループ（「キッチン」、「廊下」、「寝室」）を作成し、照明コントロールを内部に配置できます（たとえば、寝室には「メインライト」と「壁取り付け用燭台」があります）。







各要素について、そのタイプを選択できます：スイッチ（オン/オフボタン）、調光器（ライトの明るさを変更できます）、スクリプトボタン、RGB LEDストリップコントロール。 各コントロールには、コマンドを送信する1つ以上のチャネルを割り当てることができます。







各グループのコントロールに加えて、温度/湿度センサーからの情報を表示するパネルを追加できます。 ゲートウェイには、センサーをバインドするための4つのチャネルがあります。 グループの設定で、グループに表示するセンサーを選択できます。







タイマーセクション


PR1132ゲートウェイにはリアルタイムクロックが組み込まれており、指定した時点が発生したときにコマンドをパワーユニットに送信できます。 たとえば、この方法で、プレゼンスの模倣を整理できます（夜間にアパートに所有者がいない場合、誰かがそこにいるかのようにライトが点灯します）。 または、田舎の家に住んでいる場合、夕方には家の前の照明を自動的にオンにし、全員がすでに寝ている夜には自動的にオフにすることができます。







ゲートウェイでは、最大8つのタイマーを構成できます。 それらのそれぞれについて、応答時間、応答週の曜日（たとえば、アクションが平日にのみ実行されるように）、および実際には、指定された時間に達したときに実行されるアクション（たとえば、電源投入コマンドの送信または照明スクリプトの呼び出し）を指定できます。

インターフェース設計


上で書いたように、ゲートウェイインターフェースは小さな画面サイズに適応します。 私の意見では、それは小さな画面ではるかに良く見える:)私たちはさまざまなブラウザとさまざまなモバイルデバイス（WP、iPhone、Android）でゲートウェイを操作しようとしましたが、問題はありませんでした。

頭に浮かぶ唯一の願いは、より軽いデザインを作ることです-現在のデザインは非常に厳しいようです。 そして、判明したように、この方向への動きはすでに進行中です。 次のようになります。







API


ゲートウェイには、HTTP要求を送信することにより、接続されている電源ブロックを制御する機能があります。 コマンドを送信するには、GETメソッドを使用してゲートウェイに「api.htm」ファイルを要求し、パラメーターにコマンドのチャネル番号、コマンドタイプ、およびデータを渡す必要があります。 コマンドが正常に完了すると、ゲートウェイはこの要求への応答として文字列「OK」を返します。



例： 

 http://192.168.0.168/api.htm?ch=0&cmd=15
 （チャネル番号1のバインディングコマンドを送信します）

 http://192.168.0.168/api.htm?ch=0&cmd=2
 （チャネルNo. 1の「enable」コマンドを送信します）

 http://192.168.0.168/api.htm?ch=2&cmd=6&br=50
 （チャンネルNo. 3の明るさを50％に設定）




要求形式の詳細については、ゲートウェイの手順を参照してください 。



接続されたセンサーから情報を取得するには、ゲートウェイからsens.xmlファイルを要求する必要があります。 ゲートウェイは、次の形式で結果を返します。 

<response> <snst0></snst0> <snsh0></snsh0> <snt0></snt0> ... <snst3></snst3> <snsh3></snsh3> <snt3></snt3> </response>




ファイルには、要素snstX（温度）、snshX（湿度）、sntX（センサーステータス）が含まれます。Xはセンサー番号（0〜3）です。 センサーの数が属性の値ではなく要素の名前で指定されるのは少し不便です（たとえば、サイクル内でセンサーを均一に処理することはできません）が、センサーの数が少ないため、これは特別な問題を引き起こしません。



また、ゲートウェイからグループとその中のコントロールのリストを取得できるようにしたいと思います。 たとえば、ゲートウェイ用のモバイルクライアントを作成する場合に便利です。 現在、唯一のオプションは、要素グループのページをリクエストして解析することです。 情報によると、将来的にはAPIが完成し、ゲートウェイ設定をエクスポートおよびインポートできるようになります。 この機能は、設定（たとえば、以前に保存した設定を復元する）で動作するように設計されていますが、グループのリストを取得できます。



一般に、私の意見では、ゲートウェイAPIは非常に便利です。 これにより、独自のスマートホームシステムでゲートウェイを使用できます。
