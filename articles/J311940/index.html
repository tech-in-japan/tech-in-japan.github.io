既定では、 Azure Service FabricはWebサービスのパブリックインターフェイスを提供しません。 アプリケーションをHTTPクライアントに対して機能させるには、エントリポイントとして機能し、個々のサービスとデータを交換するWebプロジェクトを作成する必要があります。

ASP.NET Coreサービスをアプリケーションに追加する



ASP.NET Coreは、最新のWebユーザーインターフェイスとWeb APIを作成できるシンプルなクロスプラットフォームWeb開発ツールです。 ASP.NET Web APIプロジェクトを既存のアプリケーションに追加しましょう。



1.ソリューションエクスプローラーでアプリケーションプロジェクトを開いた状態で、[ サービス ]を右クリックし、[ 追加]> [新しいService Fabric Service ]を選択します。







2. [ サービスの作成]ページで[ ASP.NET Core]を選択し、このサービスの名前を指定します。







3.次のページは、ASP.NET Coreプロジェクトテンプレートセットを示しています。 これらは、Service Fabricアプリケーションの外部でASP.NET Coreプロジェクトを作成するときに使用されるものと同じパターンであることに注意してください。 このガイドでは、 Web APIを使用します 。 ただし、これらのメソッドを使用して本格的なWebアプリケーションを作成できます。







注：さらに作業を進めるには、 .NET Core 1.0をインストールする必要があります 。



Web APIを作成した後、アプリケーションには2つのサービスがあります。 アプリケーションを改良するときに、同じ方法で新しいサービスを追加できます。 これらの各サービスは、特定のバージョンを含め、個別に更新できます。



注：ASP.NET Coreサービスの作成の詳細については、ASP.NET Coreのドキュメントを参照してください。



アプリケーションの起動



実行した手順をよりよく理解するために、新しいアプリケーションをデプロイし、ASP.NET Core Web APIテンプレートの使用に関連するデフォルトの動作を見てみましょう。

2.配置が完了すると、Visual StudioはASP.NET Web APIサービスのルートディレクトリ（たとえば、 localhost ：33003）でブラウザーを起動します。 ポート番号は任意に割り当てられ、異なる場合があります。 Web APIテンプレートは、ルートディレクトリのデフォルトの動作を提供しないため、ブラウザにエラーメッセージが表示されます。



3.ブラウザのアドレスにパス/api/values



を追加します。 これにより、Web APIテンプレートのValuesControllerオブジェクトのGet



メソッドが呼び出されます。 テンプレートによって提供されるデフォルトの応答、2行を含むJSON配列を返します。







このガイドの手順を完了すると、デフォルト値はステータスカウンターサービスカウンターの最後の値に置き換えられます。



サービス接続

ローカルクラスタでのWebサービスの動作



ローカルクラスターに展開されたService Fabricは、通常、複数のコンピューターを含むクラスターに展開できます。 このアプリは正常に動作します。 これは、ローカルクラスターが同じコンピューター上にある5ノード構成であるためです。



ただし、1つの重要な機能がWebサービスに関連付けられています。 Azure環境で提供されているようにクラスターがロードバランサーに含まれていない場合は、Webサービスが各コンピューターに展開されていることを確認する必要があります。 これは、ロードバランサーがコンピューター間のトラフィックを単純にループするために必要です。 これを行うには、サービスのInstanceCount



パラメーターに特別な値「-1」を指定します。



それどころか、Webサービスをローカルで実行する場合、サービスのインスタンスが1つだけ実行されていることを確認する必要があります。 そうしないと、同じパスとポートでリッスンしている複数のプロセスと競合します。 したがって、ローカル展開では、Webサービスインスタンスカウンターを「1」に設定する必要があります。



さまざまな環境にさまざまな値を設定する方法の詳細については、 記事を参照してください 。
