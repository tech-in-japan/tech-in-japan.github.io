システム管理からAndroid開発に移行した方法







数年前、私は他のAndroid開発者と同じように、この分野で仕事を見つけることにしました。 あなたが初心者のスペシャリストであり、 Androidの学習を始める場所がわからない場合、またはこのトピックに関する知識がすでにあり、すぐに仕事に就く予定がある場合は、この資料が役立ちます。 計画どおり、経験を積むとこの記事に戻ります。現在の段階とさらに学習する段階を確認します。



それがすべて始まった方法



システム管理の分野では、2年以上にわたって成功を収めています。 市場を研究した後、私はさらに進んで何かを変える必要があることに気付きました。 テクノロジーへの関心が高まり、プログラミング、つまりJava言語に定住しました。 Java SEをゼロから学び始めました。 この計画は、 Java SE 、そしてJava EEに完全に没頭することでした。 しかし、 Java SEを学ぶ途中で、私は通常のHello World Androidプロジェクトを立ち上げました。 そして、将来についての私の見解は大きく変わりました。 プログラムへの最初の「没入」から求人までの全行程に1年かかりました。



あるかどうか



Androidの選択は、飛躍的に進歩するため、十分に検討する必要があります。 毎年、多くの革新的な新バージョンが着実にリリースされます。 さらに、新しいバージョンの既存のアプリケーションの特定の割合が起動しない場合もあります。 これらの問題を解決する必要があります。



Android用に開発できる言語



1.公式には、 Googleは2つのオプションを提供しました



  • Android SDK 公用語はJavaとKotlinです。

    既存のプロジェクトのほとんどはJavaで記述されています 。

    KotlinはJavaの孫です。 一部の開発者は、プロジェクトで積極的に使用し始めました。 （しかし、Javaと比較してそれらの多くはありません）

    Android は2017年5 月17日からKotlinの公式サポートを開始しました。言語機能はAndroid Studio 3.0に組み込まれます。


  • Android NDK 公用語はC / C ++です。

    Google自体は、最大のパフォーマンスが必要な場合にNDKを使用することを推奨しています。 （私たちのプロジェクトの1つでは、NDKを使用して音声通信を整理しました）





2. クロスプラットフォームツールもあります 。



Androidの学習を開始する場所



クロスプラットフォームツールまたはKotlinに気づいたかもしれませんが、急いではいけません。



クロスプラットフォームツールを使用すると、 AndroidおよびiOS用にすぐに開発できます。



クロスプラットフォームツールは特定の範囲の問題を解決できますが、柔軟性の観点からネイティブソリューションと比較することはできません。 さらに、このような専門家は労働市場であまり需要がありません。



Kotlinに関しては、今日、Androidプロジェクトがまったく書かれている会社はありません。 コトリンは就職後にのみ研究されるべきだと私には思えます。



Javaから始めることをお勧めします 。 特に、プログラミングの経験がない場合。



Javaを学ぶ



必要な知識
  • データ型。 （原始および参照）
  • ループ（ for / while / each ）
  • Javaのメモリデバイス（ Stack、Heap ）。 Garbarageコレクターがどのように、どのレベルで機能するかを理解することも重要です。
  • メソッド-返される/返されない
  • 配列とコレクション、およびそれらを使用した操作（ソート）
  • OOP（多態性、継承、カプセル化、抽象化）
  • ComparatorおよびComparable （オブジェクトのソート用）
  • スレッドとRunnable。Javaのマルチスレッドがどのように機能するかを理解します。
  • 例外（try / catch / finnaly） -エラーをキャッチするため
  • ジェネリック （ジェネリック型）


この特定のリストについてインタビューを受けました。

私の意見では、これで十分です。



JavaとAndroidを学習するための多くの優れたリソースがネット上にあります 。 私が関わった人だけに触れます。



1. JavaRush



最初の学習リソースはJavaRushでした。 10のレベルで構成される「Java構文」の無料コースを受講するだけで十分だと思われます。









リソース自体は、多くのタスクの不正確さとバリデーターの非常に顕著な欠陥にもかかわらず、明らかに注目に値します。 多くの人がこのサービスに懐疑的であり、時間を無駄にする価値はないと考えています。 私は、このリソースと、私と一緒に始めた現在の同僚を称賛し、強く批判します。 JavaRushは良いスタートを切って、アルゴリズムを上手くポンピングします。 しかし、それに座ることは明らかに価値がありません。



2. Ivan GolovochによるJavaコア



JavaRushと並行して、Ivan Golovochによるビデオ講義も視聴しました。IvanGolovochは、私の意見ではRunetで最高のJavaです。 今日では、優れたプログラミングと説明の両方が可能な人はほとんどいません。 JavaRushで得た知識を強化したのはこれらの講義でした。 Java Coreに関する講義を完全に視聴することをお勧めします。



Androidの学習



必要な知識
  • XML-インターフェースはその上にレイアウトする必要があります
  • アクティビティ/フラグメント-UIの場所の「フォーム」。 それらの違い、ライフサイクル、データ操作（ onActivityresult ）を知る必要があります
  • インテント - アクティビティ間でデータを転送し、サードパーティのアプリケーションを開きます
  • バンドル - フラグメント間でデータを転送し、画面の状態を保存します
  • ViewPager 、 RecyclerView 、 Spinnerの基本アダプター
  • NavigationView-ナビゲーションメニュー「シャッター」
  • Dialog、DialogFragment-ダイアログ（たとえば、テキストとボタンが「はい」、「いいえ」の場合）
  • AsyncTask - Androidマルチスレッド
  • SharedPreferenses-設定と小さなデータを保存するには
  • サービス -バックグラウンドでの長い操作の実行
  • SQLite-データベース。 ほとんどすべての新しいプロジェクトでORMを使用しているため、人気が失われています。 しかし、古いプロジェクトでは間違いなく会うでしょう
  • リソースの操作： 文字列、色、スタイル、洞穴、アニメーション
  • マニフェストファイル、 権限 -アプリケーションコンポーネントと権限の説明

Android Juniorのテクノロジーとツール



TelegramのAndroid開発者コミュニティでは、このトピックに関して質問がありましたが、私の答えは、ジュニア要件の点では高すぎると見なされていました。 （ RxJava 、 Dagger2 、 Realm 、 Picassoと組み合わせてRetrofit2を説明しました）。 ほとんどすべてのスタジオでAndroid Juniorの要件のレベルが異なることに注意してください。 原則としてテクノロジーなしで、誰かが雇われ、 Androidの 初期知識を持っています。 しかし、これはまれであり、私はこれらの幸運なものの中にいませんでした。 知れば知るほど、仕事を見つける可能性が高くなります。



それでは、行きましょう：



1. Git

これはコードバージョン管理システムです。 これにより、チームで積極的に作業し、コードの変更を追跡し、以前のバージョンにロールバックできます。これは非常に便利です。 GeekBrainsから良いコースを取ることをお勧めします。 Gitクライアントとして、 SourceTreeを使用することをお勧めします 。 それは完全に無料で便利であり、スタジオはソフトウェアの節約が非常に好きです。



2.バターナイフ

XMLからビュー要素を宣言するための便利なライブラリ。 （そしてそれだけではありません）。 非常に迅速に学習し、一定量のルーチンコードを削減します。



3. Retrofit2 、 RxJava 、 RxAndroid 、 OkHttpロギングInterCeptor

Retrofit2 - RESTクライアント

RxJava - Javaリアクティブプログラミングモジュール

RxAndroid - Androidのリアクティブプログラミングモジュール

OkHttpロギングInterCeptor - HTTPリクエストをログに記録するためのモジュール（ OkHttpライブラリの一部）

このバンドルでの作業の基本を知るだけで十分です。 （たとえば、データでJSONを解析できるようにする ）



最初からRxJavaを深く掘り下げるべきではありません。 Android開発者の大部分は、Retrofit2と組み合わせてのみ使用します。

5.レルム

データベース。 ある意味では、 SQLiteよりも高速です。 確かに、実動プロジェクトではまだそれほど頻繁に使用されていません。 安定版は比較的最近リリースされました。 しかし、私の意見では、 Androidでのローカルデータストレージの将来はRealmにあります。 抽象レベルで見るだけで十分です-基礎となるトランザクションを解析します。

6.ピカソ

画像を操作するためのライブラリ。 ネットワークから画像をダウンロードするために最もよく使用されます。 可能な限り使いやすい。



7. EventBus

メッセージを送信するためのライブラリ。 勉強する価値はありますが、実際のプロジェクトで使用することはお勧めしません。 EventBusは 、将来維持するのが非常に困難です。

しかし、実際のプロジェクトでは、残念ながら、しばしば会うでしょう。



練習する



原則として、 GitHubに既存のアプリケーションを持っている候補者は、雇用の可能性が高くなります。 （特に、私が説明したすべてのテクノロジーを使用している場合）。



仕事を見つける機会を増やすために、得られた知識を統合するテストアプリケーションを作成することをお勧めします。 それがどうあるべきか、何をすべきかはあなた次第です。 個人的には、他の候補に勝る議論の余地のない利点は、ネットワークからデータをダウンロードし、ユーザーにとって便利な形式で表示するアプリケーションになることです。



同時に、ユーザーがインターネットに接続していない場合、最新の関連データがデータベースからダウンロードされます。 適切なAPIが見つからない場合は、問題ではありません。 このリソースに複数のJSONファイルを作成して使用します。 彼らは、あなたのアプリケーションが世界に役立つことではなく、あなたがどのようにそれを実装したかを見ます。



私のアプリケーションは、ネットワークから会社のマネージャーのリストをダウンロードし、販売計画の実施のスケジュールを示しました。 すでに開発の過程で、私は求人を受け取りました。



このアプリケーションの開発プロセスにおける最大の間違いは、すべてを完璧にしようとしたことです。 標準により、コードを適切に評価する。 MVPの非常に抽象的な知識を所有しているため、プロジェクトにMVPを埋め込み、コードをさらに混乱させようとしました。



どんな方法を試しても、最初のアプリケーションを完璧にしましょう。 自分をさらに混乱させたくない場合は、最初のプロジェクトでMVPを使用しないでください。 （将来、雇用後-間違いなく価値があります）。
