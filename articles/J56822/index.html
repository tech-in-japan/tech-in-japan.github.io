FLASH API VKontakteの12の革新

作成されたアプリケーションは、ユーザーコンテンツへの実用的なフルアクセスを持ち、作成者は収益を得る機会を得ます。



もっと稼ぐことに集中しましょう 、P。Durovが書いているように、「実際、このアプリケーションはあなたのサイトと同じです-あなたはそれを収益化し、あなたに都合の良い方法でお金を受け取りたいのです。」

  • 現在、アプリケーションのページに表示される「投票」に対する支払いを伴うターゲット広告の内部システムからの収益は、アプリケーションの作成者に送金されます。
  • アプリケーションはシステムの内部エコノミーにアクセスし、ユーザーは、たとえばアプリケーションから何かを購入することにより、投票をアプリケーションのアカウントに移すことができます。


システムは、いわゆる「投票」の内部通貨を通じて開発者とのすべての決済を行うため、「投票」を現金化する予定です。 デュロフの言葉から判断すると、ここでの唯一の「車輪にこだわる」はMTS SIM詐欺です。VKontakteはお金を受け取らないという票を受け取ったからです。 しかし、開発者は、MTS加入者からの支払いに制限を導入する必要がある場合でも、キャッシュアウトの可能性が現れると主張しています。







12の革新のリスト：



ソーシャルグラフとメディアへのアクセス。 これで、アプリケーションは友達リストを操作したり、ユーザーページに関する情報にアクセスしたり、ユーザーの写真や音声記録を読み込んで保存したり、アルバムを作成したりできます。 ユーザー自身が個々のアプリケーションへのアクセスを提供する情報を決定します。



通知の配布。 アプリケーションは、任意のテキスト形式でいつでも通知を設定するユーザーに通知を送信できるようになりました。 新しい通知が表示されると、ユーザーは左側にカウンターの更新が表示され（アプリケーション（1））、すべてのアプリケーションニュースが時系列で表示されます。



招待状の配布。 これで、「友達を招待」機能を使用して、お互いをアプリケーションにすばやく招待できます。 通知は、アプリケーションセクションのすべてのユーザーに適用され、高品質のアプリケーションに関する情報を迅速に配信します。



広告収益の100％。 アプリケーションページから広告が完全に消えました。 ただし、人気のあるアプリケーションの作成者は、アプリケーションのページにターゲットを絞った広告を表示し、アプリケーションの残高を即座に補充することで、そこから収入の100％を受け取ることができます。 広告は50秒ごとに更新され、移行時に新しいウィンドウで開きます。



国内経済へのアクセス。 アプリケーションは、VKontakte内部支払いシステムにアクセスできます-投票。 各アプリケーションは、独自のリモートサーバーを使用して、ユーザーアカウントを安全に補充または減少させて、投票でトランザクションを行うことができます。 さらに、作成者は、アプリケーション自体の残高をいつでも個人アカウントに移すことができます。



全画面モード。 ページあたりのアプリケーションの最大サイズが25％増加しました。 これで、作成者がディスプレイでお金を稼ぐことを選択した場合に備えて、ページにサードパーティのバナーを配置できるようになりました。 さらに重要なことは、全画面モードが機能し、どのアプリケーションでも必要に応じて画面全体を使用できることです。



アクティビティテープを介して広がります。 サービス「My News」は、このアプリケーションをインストールした友人を示します。 アプリケーション作成者は、アプリケーションアイコン（16x16）をダウンロードできるようになりました。 ニュースおよび今後のユーザーの個人ページでは、標準のアイコンの代わりに個々のアプリケーションアイコンが使用されます。



出力の順序を変更します。 ユーザーは、アプリケーションリストの出力順序を変更できるようになりました。 出力順序は、個人ページ上のアプリケーションの完全なリストと短いリストの両方に適用されます。 したがって、最適なアプリケーションが上に表示されます。 ブロック「アプリケーション」全体がユーザーのページの上に表示されるようになりました。



詳細な組み込み統計。 開発者は、アプリケーションユーザーのユニークビジター、ビュー、性別、年齢、地理に関する情報を入手できるようになりました。これらはすべて、詳細なフラッシュグラフの形式で提供されます。 開発者は、統計情報へのアクセスレベルを変更して、たとえば広告主に表示することができます。



サブドメインと短いアドレス。 人気のあるアプリケーションは現在、タイプchat.vkontakte.ruのサブドメインを登録する機能を備えています。 vkontakte.ru/app1のように、標準アプリケーションアドレスの外観が短くなりました 。 個々のユーザー向けの特定のアプリケーションインスタンスvkontakte.ru/app49416_1（「Pavel Durovがいた都市」）



無限の数のアプリケーションページ。 これで、アプリケーションはWikiマークアップを使用して独自のページを無限に作成できます。 メインWikiページはアプリケーションの真下に表示され、実際には長さの制限はありません。 多くの制限があった古いテキストの説明は、検索時にアプリケーションに関する基本情報を表示するために使用されます。



個人ページに出力される情報。 アプリケーションは、サードパーティのサーバーからいつでも、接続したユーザーに短いステータスを設定する機会を得ました。 このステータスは、アプリケーションのリストのユーザーの個人ページに表示され、すべてのユーザーに表示されます。 この機会は、たとえば、ゲームでのユーザーの評価やソーシャルアプリケーションでのステータスを表示するために使用できます。
