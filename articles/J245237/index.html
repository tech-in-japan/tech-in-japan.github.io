最新のWebパフォーマンス最適化の課題について



Alexander Astapenko： 2014年はもう終わりです。つまり、この分野のベストプラクティスが体系化されたWebパフォーマンスの最適化について話し始めてから7〜8年が経過しました。 今日のWebパフォーマンスの最適化についてどう思いますか？ これはどの程度関連性があり、Web開発者はどのような課題に直面していますか？ ブラウザが変わり、テクノロジーが変わりました。



Sergei Chernyshev：これまで、Webスタックの透明性は深刻な問題であり、技術的な観点からだけではありません。 現在、これは教育上の問題です。つまり、ツールは既に十分に開発されていますが、Webパフォーマンスの最適化に対処する必要があることを知らないだけです。 私の意見では、これはトレンドを変えるための大きな障壁です。 そして、その傾向は非常に単純です。毎年、Webの速度が低下するだけです。 テクノロジーは進歩しているように見えますが、そうではありません。 人々が速度の問題に注意を払った場合にのみ、状況を変えることができます。 つまり、非技術的な問題が最も重要です。



現在、新しいツールがあります。たとえば、Webサイトの合成テスト用にShow Slowを開発してから、新しい方向が開発されました-Real User Monitoring（RUM）を使用すると、実際のユーザーの速度をリアルタイムで分析し、実際の状況を確認できます。



また、問題は多くの場合、これが合成テストの分析方法の概念間の大きな飛躍であり、楽器がスコアボードにどれだけ表示されたかを確認することです（3秒か5秒ですか？あなたのサイトをご覧ください。 単一の数字はありません。非常に速いものから非常に遅いものまで、数字の全範囲があります。 タスクは、それらをすべてゼロに近づけることです。



次の課題はモバイルWebです。これは関連性のあるものになりました。 私たちが以前抱えていた問題は再び重要になります。彼らは注意を払う必要があります：ネットワーク、完全に異なる伝送速度、ネットワーク遅延、その他の同様の問題。



既に述べたWebスタックの不透明度とそれに含まれる膨大な数のコンポーネントのために解決するのが難しいもう1つの重要なタスクがあります-物事の速度に関するデータの視覚化です。 そして、多くの人々がRUMと合成データの両方でこの問題を解決しようとしています。



実際、ブラウザーはテクノロジーを大幅に進歩させており、Webサイトの高速化を可能にする多くの企業があります。 したがって、技術的な最適化の観点から見ると、状況は非常に良好です。 速度に関する分析および開発戦略は、より問題の多い場所です。



Pavel Pavlov：これらすべての技術的問題に引き続き関わっていますか？ 新しいツールを使用していますか？ たとえば、Show Slowを作成しました。 このプロジェクトは関連性がありますか？



Sergey Chernyshev： Show Slowはまだ使用されており、主に最適化プロセスをサポートするための重要なツールであり、傾向を確認できます。 ただし、RUMの登場により、Show Slowはあまり重要ではなくなりました。 そしてもちろん、プロジェクトは以前ほど活発ではありません。 Patrick Meenanが作成した非常に人気のあるWeb Page Testプロジェクトに彼を統合させました。 これら2つのツールを組み合わせることで、Webページテストを使用してページの分析を詳細に調べたり、これらすべてのメトリックの傾向を確認したりできます。



つまり、スロー表示ツールの重要性です。 おそらくそれは、純粋に技術的および分析的なソリューションではなく、ダッシュボード、またはむしろプロセスをサポートするツールになるでしょう。 他のツールがより良い仕事をするからといって。



私はWeb Page Testを絶えず使用し、開発者としてではなく、現場の人として開発に積極的に参加しています。 そして今、私は個人的に進むべき方向を理解しようとしています：加速ツールを開発する価値があるか、視覚化を使用する価値があります。 私は最近、Velocityカンファレンスで、データスピードを視覚化する興味深いアプローチを持つデザイナー、マークゼマンと話をしました。 そして、おそらく、私たちはこの方向に進むでしょう。



会議とWebパフォーマンスコミュニティについて



アレクサンダーアスタペンコ： Web Performance Optimization Foundation。 私はこのイニシアチブについてほとんど知らないことを正直に認めます。 そして、あなたからいくつかの言葉を聞きたいです。 組織の使命と現在のプロジェクトは何ですか？



Sergey Chernyshev：この組織は、Webパフォーマンスの最適化を促進するために作成されました。 参加者には、Steve Souders、Stoyan Stefanov、Patrick Minen、およびWeb生産性に積極的なその他の人々が含まれます。 現時点での主なプロジェクトは、Web Page Testのサポートと大衆へのプロモーションです。 私たちの目標は、たとえばWebPerfDays Unconferenceなど、速度専用のmitapとさまざまなイベントを作成することです。 組織のタスクは非常に単純です-速度。 人々は速度を改善できること、そして改善する必要があることを知る必要があります。



アレクサンダー・アスタペンコ：会議のトピックについてはすでに触れました。 Velocityについて少し教えてください。 誰が講演者で、誰がこの会議のリスナーであるかを簡単に見つけることができますか？ あなたは、ビジネスはスピードアップに投資することの重要性を理解していないと言いました。 そこにビジネスは来ますか？ または、これは開発者専用の会議ですか？



Sergey Chernyshev：最初に、この会議を作成してくれたO'Reillyに感謝します。 私の意見では、世界のさまざまな場所で年に4回開催されています。 会議はカリフォルニア州サンタクララで始まりました。 Velocityは、実際には2つの部分で構成されています。パフォーマンスとオペレーション-わずかに重なっているがそれほど多くない2つのキャンプです。



会議には、システム管理者、フロントエンド開発者、および企業が参加します。 特にここ数年-膨大な数の企業。 現実には、特にモバイルWebが関連性を持つようになった場合、Webサイトが成長し、速度が低下するだけです。



Velocityは幅広い問題をカバーしています。 そして、ビジネスが活発にやって来ています。 もちろん、これらは主に技術管理者です。 会議は技術的なものであり、ビジネス用の蹄鉄を持っている人は難しいでしょう。 しかし、その一方で、私はそこに米国政府の代表者と、主に電子商取引からのビジネスだけを見ました。 もちろん、彼らはこの問題を最初に取り上げました。なぜなら、彼らは速度とお金の間に類似点を描くことは非常に簡単だからです。 これは他の分野では困難です。 私の意見では、Velocityカンファレンスは素晴らしいカンファレンスです。 もしあなたがヨーロッパかアメリカのどちらかでそれを訪れることができるなら、私はそれをすることを強く勧めます。



Pavel Pavlov： Velocityはアメリカ、ヨーロッパ、中国でも行われています。 実際、世界中で開発されています。 しかし、同時に、トピックが十分に加熱されていない大きなスペースがRunetにあります。 もちろん、Webパフォーマンスに携わっている企業があり、このテーマに関する書籍が出版されています。ブログ、講演者がいます。 しかし、そのようなコミュニティはありません。



Sergey Chernyshev：はい、同意します。 主にスケーラビリティと可用性の問題を検討するHighLoad会議があります。 つい最近です。 何人かのスピーカーがそこに来ます：フィリップ・テリスとバディ・ブリューワーがそこで話しました。



Pavel Pavlov：そして、Stoin Stefanov。



セルゲイ・チェルヌイシェフ：はい、確かに。 同意します。 どういうわけか存在しますが、ごくわずかです。 一般に、ロシアでの会議の開催方法についてはあまり詳しくありません。 しかし、ヨーロッパは、神に感謝して、近いです。 そのため、そこで聴くことは完全に理にかなっています。



Pavel Pavlov： Steve Sodersと会社に、少なくともモスクワで小さなカンファレンスを開催するよう促すことは理にかなっているのでしょうか？



Sergei Chernyshev：主なことは、誰を招待するかではなく、誰がこれを地元でできるかだと思います。 つまり、問題にローカルで興味を持っている人々を見つけることです。 当然、スピーカーを招待できますが、これは難しくありません。 しかし、私の意見では、主催者がコミュニティを集めることが重要です。コミュニティは、正しい方向に向かって話す準備ができているスピーカーを動かします。



大きな会議を待つ必要はありません。 実際、オライリーは、ニューヨークとヨーロッパで会議を開催する前に、地域コミュニティがどのように進化するかを注意深く観察しました。 ロンドンでヨーロッパに非常に人気のあるmitapがあったように、ニューヨークでも最も人気のあるmitapの1つでした。 そしてその後、彼らは会議を作成しました。 O'Reillyのようなビジネスでは、地元の活動の確認が必要です。 モスクワ、キエフのミンスクで会議を開催します。 そして、これが問題を促進する方法です。



Alexander Astapenko： mitaps、Velocityがありますが、WebPerfDaysもあります。 WebPerfDaysの本質と目的について話してください。



Sergey Chernyshev： WebPerfDaysは元々、DevOpsDaysに似たサンフランシスコで作成されました。 大まかに言って、これは1日の会議であり、形式は少し異なります。 もちろん、伝統的なスタイルのプレゼンテーションがいくつかありますが、残りはディスカッションです。つまり、誰もがWebパフォーマンスのトピックについて自分の質問を持ち込むことができます。その後、全員がグループに分かれてしばらくの間この問題について話し合います。



形式の点では、これは非常にシンプルですが、従来のスタイルの会議への重要な追加です。 多くの場合、公式のVelocity Conferenceの後に、WebPerfDays Unconferenceが翌日に行われます。これは、メインの会議に既に参加している人を引き付けたり、ローカルの1日イベントを作成したりするためです。



WebPerfDaysは、自分が何について話しているかを知っている人、つまりWebパフォーマンスなどに従事している人向けです。 これは、Velocityで話す人の多くがWebPerfDaysにも参加しているため、あまり正式ではない設定でスピーカーと話し、問題をより詳細に議論する良い機会です。
