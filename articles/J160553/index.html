ステージ2、修正、実行可能、VirtualBox



サービスをインストールするには、古い小さなものが使用されます。マシンを起動するユーザープロセスを作成するためのWindowsリソースキットの2つのファイルです。



Windows Resource Kitに含まれているSrvany.exeおよびInstsrv.exeファイルは、ここから取りました。 両方のファイルをC：\ Windows \ system32に配置する必要があります



（DEB1は例で、私が持っていたマシンの名前です。）

次に、コンソール（start> cmd（管理者として））を起動し、太字のエントリを将来のサービスの名前に置き換えてエントリを追加します。最初の名前にはスペースを入れず、2番目には好きな説明を含める必要があります。

sc create DEB1 binPath = "％SystemRoot％\ System32 \ srvany.exe" start = auto DisplayName = " VB VM Debian "


すべてがうまくいけば、「正常に」書き込まれます。

プロセスが作成されたことを確認するには、services.mscを実行するか、「サービス」タブを確認する必要があります。



次に、パラメーターをレジストリに追加し、ファイルを作成します

パスとマシン名はシステム上のものです。

DEB1.reg

Windowsレジストリエディターバージョン5.00

[HKEY_LOCAL_MACHINE \ SYSTEM \ CurrentControlSet \ Services \ DEB1 \パラメーター]

"アプリケーション" = "\" D：\\プログラムファイル\\ Oracle \\ VirtualBox \\ VBoxHeadless.exe \ ""

"AppParameters" = "\"-s DEB1 -vrdp = off \ ""

"AppDirectory" = "\" D：\\プログラムファイル\\ Oracle \\ VirtualBox \\ \ ""


次に、コンピューターのシャットダウン時にコンピューターの状態を保持する場合は、gpedit.mscを実行して、コンピューターの構成/ Windowsの設定/スクリプト/シャットダウン（コンピューターの構成\ Windowsの構成\スクリプト（開始\シャットダウン）\シャットダウン\）を見つけます。

スクリプトを作成して名前を付けます
D：\ Program Files \ Oracle \ VirtualBox \ VBoxManage.exe
または、参照ボタンでパスを選択し、パラメータを設定します
vboxmanage controlvm DEB1 savestate
DEB1は、マシンの名前に置き換える必要があります。

.VirtualBoxフォルダーをC：\ Users \ {Your name} \（{Your name}はアカウントの名前）からC：\ Windows \ System32 \ config \ systemprofile \にコピーします。これは、マシンのプロパティを変更するたびに繰り返す必要があります。開始されない場合があります。または、次回、不適切な（無関係な）変更で開始されます。

マシンが起動しているかどうかは、VBoxHeadless.exeプロセスを調べることで確認できます。プロセスが起動している場合は、マシンが実行されています。



ステージ3、VMware Player、サービスとして





仮想マシンの新しいバージョンVMwareでは、Vmxファイルにいくつかのパラメーターを追加するだけで、利用可能な物理メモリを考慮して仮想マシンのRAMメモリを自動的にスケーリングできます。

memsize = "2048"

MemAllowAutoScaleDown = "TRUE"


この場合、仮想マシンは2 GBのRAMを破棄します（存在する場合）。

RAMが2 GB未満のマシンでは、プレーヤーは使用可能なRAMに基づいて使用量を自動的に調整します。

Memsize 2048は制限ではありません。

2 GBを超えるメモリがインストールされている場合、この数を増やすことができます。



サービスを実行するマシンにVMware Playerをインストールします。

VMWare Playerのサービスの作成は、VirtalBoxに似ています。

2つのファイルが必要です：instsrv.exeおよびSrvany.exe、 Windows 2003リソースキットから 。 両方のファイルをC：\ Windows \ system32 \に配置する必要があります



VMware Playerがユーザーインターフェイスによって生成されるすべての「OK」メッセージを抑制し、ユーザーインターフェイスを表示しないように、Vmxファイルにフィールドを追加します。

DEB1r.vmx

... ...

hints.hideAll = "TRUE"

msg.noOk = "TRUE"

...




instsrv.exeを使用して、ターゲットマシンにVirtualServerサービスを作成します。

instsrv VirtualServer "％SYSTEMROOT％\ System32 \ srvany.exe"

Srvany.exeのレジストリキーを追加して、仮想マシンでvmplayer.exeを実行します。

DEB1.reg



Windowsレジストリエディターバージョン5.00

[HKEY_LOCAL_MACHINE \ SYSTEM \ CurrentControlSet \ Services \ DEB1 \パラメーター]

"アプリケーション" = "\" C：\\プログラムファイル\\ VMware \\ VMware Player \\ vmplayer.exe \ ""

"AppParameters" = "\" C：\\ VM \\ DEB1.vmx \ ""

"AppDirectory" = "\" C：\\ VM \\\ ""


ACEなどの別のVmware製品を使用するには、アプリケーションキーの値を変更します。 別の仮想マシンを使用するには、「AppParameters」の値を変更します。



start-vm.batという仮想マシンを起動するバッチファイルを作成します。 このスクリプトは最初に、仮想マシンファイルがインストールされているかどうか、およびサービスが実行されていないかどうかを確認します。 次に、古いVMwareプロセスをすべて強制終了し、仮想マシンの最後の起動からさまざまな状態ファイルを削除します。 最後に-仮想マシンのサービスが開始されます。 

:: :: start-vm.bat :: :: If the service is already running, then skip starting it net start | grep -q -i "DEB1" if %ERRORLEVEL% == 0 goto noServer :: To make sure that lingering processes are dead, force kill the process taskkill /F /IM vmplayer.exe taskkill /F /IM vmware-vmx.exe :: Remove *.lck, *.vmss from the grid directory del "C:\VM\*.lck" del "C:VM\*.vmss" del "C:\VM\*.vmem" :: Start the server service net start DEB1 :noServer




バッチファイルを作成して、stop-vm.batという仮想マシンを停止します。 仮想サービスシャットダウンスクリプトは、実行中の仮想マシンに関連付けられているすべてのVMwareプロセスを強制終了します。 

 :: :: stop-vm.bat :: :: Stop the service net stop DEB1 :: Force kill the processes taskkill /F /IM vmplayer.exe taskkill /F /IM vmware-vmx.exe :noServer




仮想マシンのプロセスを破棄することは、物理コンピューターの壁のコンセントからコードを引いた場合と同じです。 仮想マシンの状態が重要でない場合は、非永続ディスク機能を使用できます。 仮想マシンの実行中はディスクの内容が変更されないため、このようなプロセスの終了は仮想ディスクの状態に影響を与えることはできません。



Windowsの起動時に仮想マシンを起動するには、％SYSTEMROOT％\ System32 \ GroupPolicy \ Machine \ Scripts \ Startupに起動スクリプトを配置する必要があります。 この例では、起動スクリプトはstart-vm.batファイルです。

コンピューター上で、グループポリシーの自動起動のリストに追加することにより、システムの起動時にstart-vm.batを起動する必要があります。 これは、gpedit.msc-グループポリシーエディターを使用して実行できます。 [コンピューターの構成]、[Windowsの設定]、[スクリプト]、[スタートアップのプロパティ]（コンピューターの構成\ Windowsの構成\スクリプト（開始\終了）\開始\）グループポリシーエディターで開きます。 [追加]ボタンをクリックして、仮想マシンの起動スクリプトの名前を入力します（前のスクリプトと同様）。



グループポリシーシャットダウンの場合も実行しますが、シャットダウンスクリプトとしてstop-vm.batを使用します。 シャットダウングループポリシーの追加は厳密には必要ありませんが、仮想マシンプロセスを終了するだけでなく、より多くのコマンドが含まれている場合はstop-vm.batを実行することが重要です。
