LinkedIn Girl Hackathon Experience

LinkedInがマウンテンビューのキャンパスで開催したDevelopHer 2013ハッカソンの後、私は1日間眠りについたことがあります。 私はそれに参加できたことを非常に嬉しく思います。また、第一印象が頭から消えることはありませんが、観察と経験を共有したいと思います。 何よりも、おそらく、質問と回答の形式がここで適しています。 何か面白いものを逃した場合は、記事の最後で質問にお答えします。 だから：







-それは何でしたか？

-ハッカソンとは、多数のプログラマーが自発的に集まり、指定されたトピックの枠組みの中で、単独または小グループで自分のプロジェクトで昼夜を問わず作業することです。 作業の終わりに、プロジェクトは他の参加者や審査員に提出されます。 ハッカソンはテーマ別になっており、特定の技術に専念し、大企業が後援しており、時には友人の間で自然発生することもあります。 DevelopHerにはトピックがありませんでしたが、名前が示すように、若い女性だけがそれに参加しました。



-すべてプログラマーでしたか？

-私の気持ちによると（公式統計はありません）、75％はプログラマー（ほとんどがフロントエンドですが、多くのバックエンドがありました）で、残りの25％はユーザビリティーとデザイナー、そしてマーケターやその他の目的でハッカソンに来た興味深いキャラクターですチームを選んで、アイデアやパズルを実装します。 業界には多くのプロがいました-LinkedIn、eBay、Appleの開発者。 大学の学部生、大学院生（博士課程の学生）、ハックブライトアカデミーの学生がいました。



-参加にはチームが関与しましたか？

-事前に結成されたチームがありました。 その場でお互いを見つけたチームがありました。 「モバイルアプリケーションにはUI / UXの専門家が必要です」というスタイルのアナウンスが大きな掲示板に投稿されました。 私は自分の能力に自信がなく、他の人を「遅く」したくなかったので、事前にプロジェクトのアイデアを準備し、ソロに参加しました。 これらもそうでした。



-そして、本当に、彼らは昼夜を問わずキーボードを叩きましたか？

-ハッカソンは金曜日の午後5時に始まり、土曜日の午後4時に終了しました。 最初の1時間-登録、デート、チームの選択。 午後6時-主催者からの短い挨拶と背景情報。 午後6時30分頃、全員がプロジェクトのあちこちに散らばり、作業が沸騰し始めました。 しばらくの間、お互いにほとんど馴染みのないチームは、役割、ツール、共通のビジョンに同意しながら、新しいプロジェクトについて大声で議論しました。 土曜日の13時30分に終了する必要があり、その後デモと最終審判が始まりました。 したがって、実際のプロジェクトには約19時間が割り当てられました。



-19時間で何が書けますか？

-私はこの問題に非常に興味があったので、チームで何がどのように役割が分配されるかを管理した他の人々のプロジェクトのデモンストレーションを非常に興味深く見ました。 これは最も興味深い瞬間です。おそらく、業界に参入したばかりの場合（または、私の場合のように長い休憩後に戻ってきた場合）、そのようなイベントに参加すると、多数の専門家の集合作業を一度に「スパイ」するのに役立ちます。 これは私に非常にインスピレーションを与えました-多くの参加者がプロジェクトを「仕事中」に取り、すべてのツール（ライブラリ、API、テクノロジー）を使用して時間通りにそれをやったこと。



プロジェクトの大部分は、LinkedIn、FourSquare、Facebook、Google Analytics、Google MapsなどのAPIを使用したソーシャルWebアプリケーションでした。約20パーセントはスマートフォン用のソーシャルアプリでした。 そして、最終的に到達したすべての人々は、原則として働いた-宣言された機能を実証し、同時に非常に使いやすいように見えた。 プロジェクトの大部分（私の見積もりによると、少なくとも40％）は、採用担当者や仕事を探している人向けのサービスでした。 ちょっとしたブーム。 おもちゃと教育プロジェクトがいくつかありました。 「プログラマーのために何かを書くプログラマー」というプロジェクトはほとんどありませんでした。 私のグループ（27のプロジェクトに3つの予備司法委員会があり、ランダムに3つのグループに分けられました）で、私のプロジェクトはこの分野で唯一のものでした。 jQuery UIの機能を変更および拡張する一連のウィジェットを作成しました。



-彼らは主に何を書いたのですか？

-フロントエンド：JavaScript、jQuery / jQuery UI、Bootstrap、Backboneなど、さまざまなAPIを含む。 バックエンド：非常に多様-Scala、Ruby on Rails、Java、すべてのプロジェクトを見る以外にすべてをソートする方法はありません。 ところで、 ここにあるのはすべて （審判に提出されたもの）です。 私の目に留まったのは、広範なツールセットを所有していることです。 多くのチームがその場で形成されたことを考えると、アイデアは1人の参加者に伝わり、他の部分はプログラム部分を実装しました。 そして、どのパズルにも、このパズルに適したツールのセットがあったのです。 サーバープログラマーは、フロントエンドと簡単にドッキングできるようにするために、いくつかのライブラリ（ブートストラップ、jQuery UI）を所有する必要があります。 同じことがUI / UXデザイナーにも当てはまります。 最終選考では、UXデザイナーのソロプロジェクトが1つもあり、Bootstrap / SQLAlchemy / LESS / jQuery /などを使用して完全に独立してアイデアを実現しました。



-プロジェクトまたはツールに制限はありましたか？

-はい。 オープンテクノロジーとオープンデータのみが許可されました。 件名に制限はありませんでした。 プロジェクトはゼロから書かれるはずでした。 しかし、もちろん、少なくともチームが慎重に準備してハッカソンに来た人たちは、自分のプロジェクトを考え、必要なライブラリを選択し、研究したと確信しています。 私の準備は、githubでプロジェクトを作成し、新鮮なjQuery UIで埋め、インターネットからWebインターフェイス用のいくつかの無料スキンをインターネットからパパにドラッグして、ウィジェットを引き付けるものができるようにすることでした。 スキンへの手は届きませんでした。



-組織の問題について詳しく説明してください。 一晩中ピザを噛んでコーヒーを飲みましたか？ そして、午前中にそれが削減された場合、あなたは何をしましたか？

-LinkedInはハッカソンの下で本社の広々としたカフェテリアを取りました。 私たちが自由に使えるのは、コーヒーマシン、あらゆる種類のお茶、スナック入りのアイスクリームヨーグルト、冷蔵庫に入れた既製のサンドイッチ、あらゆる種類の小さなスナックでした。 日中に何度か朝食の昼食を用意しました。 真夜中に、チョコレートで覆われたイチゴと小さなチーズケーキが私たちの上に落ちました。



多くの人がまったく眠らないように思えました。 前日は十分に眠れなかったので、会議室のいずれかで開催されたトイレを喜んで使用しました。 彼らは巨大な枕椅子を投げました。 睡眠について、私はあなたの強さを計算し、疲れから一箇所で滑り始める瞬間に休息/睡眠を取ることが重要であると言わなければなりません。 私はそのように2時間を失いました。4時間の夢から目覚めた後、私は何も書かずにすべてを消し、非生産的な時間ですべてを書き、20分で再び書き直しました。 より早く停止する必要がありました。







この組織について他に興味深いのは、土曜日に、誰かがそれを必要とする場合、幼稚園が子供のために組織されたことです。 ハッカソンの間に、2つのヨガセッションが行われました。1つは夕方遅く、もう1つは朝に行われました。 彼らは、ヨガは女の子ハッカソンの典型的な属性であると言います。 2番目の典型的な属性は、クリーム入りのカップケーキです。 13:30にコンピューターを閉じた直後にも、彼らはそこにいました。 主催者のプロフェッショナルな仕事が本当に好きでした！ 誰もがラップトップや電話をデモモニターに接続することと戦う必要はありませんでした。このため、疑いもなく即座にすべてを正しく行う技術者がいました。



-そして、参加者の年齢は？

-主に28〜35歳。 学生は当然、20〜28歳です。 約50歳の少数の参加者。 興味深いことに、彼らのほとんど全員が予選を通過し、決勝に達しました。



-そして、審査員は誰ですか？

-最後の審判が行われました（今はわかりました...これが眠い脳の意味です！）LinkedInのCEO、Jeff Weiner、LinkedInの企業関係担当副社長Shannon Stabo、および他の2つのIT企業の責任者。 ジェフの背中の後ろで、魅力的な5歳の娘が面白い動物を描いた後、彼の腕に登りながらプレゼンテーションを非常に注意深く聞いた。



-このイベントに参加できるのは何ですか？ 自宅で24時間ロックして、自宅のモニター、冷蔵庫、ベッドの近くで同じことをできないのはなぜですか？

-もちろんできます。 選択した分野で真剣に成長したい人にとって、これを行うことは可能であるだけでなく、時々必要であると思います。 しかし、ハッカソンは追加のボーナスを提供し、専門能力開発のレベルごとに異なります。 私のような初心者は、何を目指して努力するかについて、より明確なアイデアを得ることができます。 プロと意欲的な大学院生が賞を競います（受賞者はMacBook Airで受け取りました）。 一部の人にとって、これは主催会社についてより多くを学び、より自信を持って欠員の申請書を提出するチャンスです。 それぞれに独自の目標と関心があります。



-そのようなクールな専門家がすべている場合、なぜこの性別による差別があるのですか？

-まず、第一に、すべての参加者が成功した専門家ではありません。 ご存じのように、社会であなたに近い人からの援助は、受け入れがはるかに容易であり、その有効性は高くなります。 ロールプレイの例でも同じことが言えます。 したがって、能力に自信を持ちたい場合は、自分に近いものから始めるのが理にかなっています。 さらに、主に男性グループでの作業では、「すべての男性を削除する」場合、私たちがプロとして本当に何であるかを理解することはできません。 繰り返しますが、それぞれに独自の考慮事項があります。 ハッカソンには約100人が参加したため、どのような目標を設定しても、このイベントは需要があったと言えます。



来年はこのハッカソンに参加でき、その他のテーマや一般的なものにも参加できます。 そして一年後には賞品を競いたいと思っています！ ロシアでのこのようなハッカソンに参加できてうれしいです： 私の母国のウリヤノフスクでは、ハッカソン も 人気 を 集めており、年に1回（そう、3月8日でも！）
