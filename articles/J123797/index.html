この記事では、アプリケーションサーバーがさまざまなクライアントとやり取りするものについてのビジョンと、.NETでそれを実現する方法について説明しようとしました。 記事の過負荷を避けるために、実装の詳細には触れないようにしましたが、主な考えはあなたの興味を引くと思います。



それでは、アプリケーションサーバーとは正確には何ですか？



私は自分の言葉でそれを定式化しようとします。

-「アプリケーションサーバー」は、分散N層システムのアーキテクチャに関連する用語の1つであり、中央の場所を占め、データの処理と受信の要求を満たし、このデータを保存するクライアント間に配置されます。

-アプリケーションサーバーは、サーバー側で動作する特別なアーキテクチャとテクノロジーに基づくソフトウェアです。 特異性は、その数とタイプの多様性の両方の点で多くの顧客にサービスを提供することにあります。

-アプリケーションサーバーは、クライアントからの受信とクライアントへの送信の両方で、特定の形式のデータの必要な処理と変換を実行します。

-アプリケーションサーバーはアプリケーションレベルのAPIを提供し、そのメソッドを使用して特定のクライアントアプリケーションを実装できます。



このプロジェクトの背景はこれでした。

Windows用のシック.NETクライアントに加えて、新しいMicrosoft Windows Communication Faundationテクノロジー（以下、単にWCFと呼びます）に基づく次のアプリケーションを何らかの形で開発し、このサービス用のWebインターフェイスを作成したいという要望がありました。 ExtJsはマスタリング用に選択されました。 「AjaxクライアントをWCFサービスに接続する方法」というトピックに関する小さな資料をグーグルで調べ、最初のドラフトが作成されました。 レイアウトは機能しましたが、それでも一見基本的なことを達成する方法に不満が生じ始めました。 一見したところ、あまりにも多くの「不必要なこと」を行う必要がありましたが、実際には単に必要でした。

そのため、クライアントはサーバーに特定のアプリケーションメソッド（RPC）を呼び出し、サーバーから明確な回答を取得できる必要があります。 呼び出されたメソッドは、名前付きの文字列で指定されます。 メソッドにはパラメーターが必要です。 彼はそれらを正しく認識して処理しなければなりません。 柔軟性を高めるために、メソッドはサーバー自体に属するだけでなく、サーバーで使用可能な任意のアセンブリにも属することができますが、セキュリティ上の制限があります。

RESTスタイルは一般的ですが、完全に柔軟ではありません。 これは「テンプレート」スタイルです。 左右にステップすると、汎用性が失われます。 不可欠なQueryStringが残っています。

Silverlightを試してみませんか？

「厚い」.NETクライアントとの違いは、HttpWebRequestおよびHttpWebResponseに同期メトが完全に存在しないことです。 Jsonと連携するために、Silverlight用のJson.NETの実装があります。 残りはデスクトップ.N​​ETアプリケーションの場合とほぼ同じです。

アプリケーションサーバーとの対話を簡素化するために、クエリ文字列を形成して実行する特別なRequestクラスが開発されました。 HttpWebRequestとHttpWebResponseの処理をカプセル化します。 

 RequestParams requestParams = new RequestParams(); requestParams.Method.Name = "GetImage(fileName)"; requestParams.Method.Params.Add("fileName", "DSCN2099"); Request request = new Request("http://localhost/AppService"); request.Execute(requestParams, Action<RequestCompletedEventArgs> onRequestCompleted);


つまり 要求パラメーターを準備したら、サーバーメソッドを呼び出して、コールバックメソッドで結果を処理します。 簡単になることはありません！



次の行はJavaです。

メインクラスはHttpURLConnectionで、getInputStream（）メソッドがあります。 呼び出し-.NETとの2つの違いを見つけます（名前はカウントしません）。 考え方は同じです-ヘルパークラスRequestParamsとRequestが作成されます。 RequestParamsクラスは、呼び出されたメソッドとそのパラメーターの名前であり、RequestはHttpURLConnectionを操作するためのロジックをカプセル化します。 Jsonのシリアル化には、Googleのライブラリ-Gsonが使用されます。Gsonは、Android開発にも使用できます。 Javaに対して拮抗的なプラットフォームに実装されたアプリケーションサーバーからJson形式で提供されるものはすべて、問題なく「消化」されます。 Javaがデフォルトで理解しない唯一のものは、MSからの日付形式です。 しかし、Gsonは拡張可能なライブラリであり、問​​題は簡単に解決されます。 

 public class MSDateJsonSerializer implements JsonSerializer<Date> { public JsonElement serialize(Date date, Type typeOfT, JsonSerializationContext context) { return new JsonPrimitive("/Date(" + date.getTime() + ")/"); } } public class MSDateJsonDeserializer implements JsonDeserializer<Date> { public Date deserialize(JsonElement json, Type typeOfT, JsonDeserializationContext context) throws JsonParseException { String jsonDateToMilliseconds = "\\/(Date\\((.*?)(\\+.*)?\\))\\/"; Pattern pattern = Pattern.compile(jsonDateToMilliseconds); Matcher matcher = pattern.matcher(json.getAsJsonPrimitive().getAsString()); String result = matcher.replaceAll("$2"); return new Date(new Long(result)); } }
