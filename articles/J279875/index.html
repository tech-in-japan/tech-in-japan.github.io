NetApp仮想ストレージアプライアンス：Data ONTAP-v

長い間、どういうわけか、いつの間にか、Hyper-VとAzureがFreeBSDのサポートを開始したというニュースがありました。 古き良きFreeBSDのファンは喜んでおり、他の多くの人々は「なぜ？」と困惑していました。 NetAppの努力のおかげで、FreeBSDはHyper-VとAzureでサポートされるようになりました。 NetAppはこれにどのように興味を持っていますか？ 実際、ONTAP OSはFreeBSDに基づいています。 Cloud ONTAPやData ONTAP Edgeなど、Data ONTAP-vベースの製品を正確に表示するため。 NetAppは、FreeBSDの最新のコードコントリビューターの1つでもあります。







この記事のData ONTAP Edgeについて詳しく説明します。 2012年に、Data ONTAP Edge製品が登場しました-Hyper-VおよびESXiハイパーバイザー用の仮想アプライアンスです。 最初は、DATA ONTAP 7-Mode OS（CBvM100、FDvM100）が付属していましたが、2014年には、Clustered ONTAP（FDvM200）に基づくData ONTAP Edgeがリリースされました。 このようなソリューションは、本格的なFASプラットフォームをインストールまたは購入する方法がない小規模オフィス向けに設計されています。



Data ONTAP Edgeは、スタートポロジを構築するのに理にかなっている多くのローカルオフィスを持つ企業に特に便利です。スタートポロジは、センターに完全なFASシステム（バックアップをレプリケートするため）とブランチにData ONTAP Edgeを備えています。



Data ONTAP-vとFASプラットフォームに基づいたONTAPの違いは、最初のRAIDテクノロジーがまったくないことです。 Data ONTAP-vは、サーバーにインストールされている従来のRAIDコントローラーに完全に依存しています。 FCPのサポートもありません。これは非常に論理的です。 彼は記念碑であり、誰が彼を植えるのでしょうか。 仮想マシンです。 NAS（CIFS、NFS）およびIP SAN（iSCSI）プロトコルをサポートします。









サポートされているハードウェアハードウェア



グローバルパートナーである富士通は、Data ONTAP-vで使用するために、 PRIMERGY BX400ブレードサーバーを備えたVSX960専用ソリューションを発売します。 VSX960ソリューションは、最大10台のHDDをサポートします。

制限事項



このシステムは元々小規模オフィス向けに設計されていたため、FASハードウェアプラットフォームよりも高い制限があります。

ライセンスは、10、25、および50 TBの使用可能なスペースで販売されます（重複排除を除く）。 つまり 重複排除がデータを十分に活用できれば、はるかに適合します。 必要なメモリ量は4/8 GBで、2つのCPUが必要です。 また、前述のように、RAID-DPがないため、キャッシュとバッテリーを備えたサポートされたRAIDコントローラーが必要です。 SnapMirrorおよびSnapVaultレプリケーションプロトコル、重複排除、Thing Provitioning、クローン作成、およびSnapRestoreがサポートされています。 公式ウェブサイトの詳細。 高可用性をサポートしないシングルモードシステムのみがサポートされます。



SnapMirrorレプリケーション



SnapMirrorライセンスにはDataONTAP Edgeが付属しています。 DataONTAP Edgeは、レプリカのソースまたは受信者として機能できます。 SnapMirrorに基づいて、 DR SVMソリューションを構築できます。 ONTAPをファイルストレージとしてのみ使用する場合、複製のためにサードパーティソフトウェアとデータの整合性を提供する必要はありません。 IP SANまたは仮想化アプリケーション、データベースなどが使用される場合、情報の一貫性を確保するためにバックアップソフトウェアが必要です。



未来-SDS



NetAppは、Data ONTAP-vの開発を続けており、これは本格的なSDSになります。 そのため、近い将来、彼はクラスタ化して、高可用性機能とより多くの情報を提供できるようになるでしょう。



結論



Data ONTAP-vを使用した仮想アプリケーションは、ローカルデータを保持し、場合によってはそれらを中央オフィスに複製する必要がある小規模企業やブランチ構造を持つ企業に適しています。 サーバーには、バッテリー付きのサポートされているRAIDコントローラーが必要です。 ONTAPを仮想環境のHyper-VおよびESXiと統合するバックアップ製品は、このようなスキームで強く求められます。 Veeam Backup＆Replicationを使用すると、一貫性のあるバックアップを実行し、SnapMirrorレプリケーションを管理できます。これにより、このインフラストラクチャで他の類似製品を最適なタンデムにできます。 SnapMirror for SVMテクノロジーは、DRソリューションの構築に役立ちます。 それとは別に、ONTAPで一貫したスナップショットをネイティブに提供するVMware vVOLテクノロジーを強調する価値があります。これは、このようなインストールでも非常に興味深いものです。
