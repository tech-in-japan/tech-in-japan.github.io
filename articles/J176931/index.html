Firefoxの高速化

Habréでこのトピックを特別に検索しましたが、既製のポータブルFirefoxのレシピのみが見つかりました。

その欠点は知られています-それは公式ではないため、いくつかの中間バージョンをスキップするまで、遅れて更新されます。

したがって、私は自分の小さな経験について書くことにしました。 すでに自宅、ラップトップ、職場で作られています-完全に機能します。





タスクは、FF自体とプロファイルを最大速度で仮想ディスクに転送することでした。



ホームオプションを最も洗練されたものとして説明します。

マシン（win 7）にはDATARAMからの仮想ディスクがあり、1 GBのサイズで-ちょうどそのような目的のために-高速化し、部分的に暗号化します。

仮想ディスクのイメージは、暗号化された（trueタイプの）ディスク上にあります。

1日1回、仮想ディスクのイメージがバックアップにスローされます。 また、コンピューターの電源を切ると、仮想ディスクが保存されます。



1年前に作成した最初のオプションは、プロファイルを含むフォルダーとfirefox自体を含むフォルダーを仮想ディスクに転送し、ntfsリンクを残しておくことでした。 解決策は5秒で完了しますが、ファイル（たとえば、小さな画像がたくさんあるキャッシュ）を操作するとき、パスの開始元のハードドライブを常にヤンクするという事実に惹かれません。

まあ、常にリンクについて覚えておいてください-それも良くありません。



したがって、最も完全な転送を行います。





実行可能部分の転送：

新しいFirefoxをインストールするとき、すべてが非常に簡単です-すぐに仮想ディスクに配置します。

転送するとき-フォルダ「 x：\ Program Files（x86）\ Mozilla Firefox \ 」を仮想ディスクにコピーし 、そこからfirefox.exeを実行します。 彼はすぐにデフォルトのブラウザになるように私たちに申し出ます。 「はい」と言って、コールショートカットを編集します（デスクトップ、クイック起動バーなど）。

レジストリを調べて、いくつかの場所でパスを編集します。 ほとんどのキーは古いバージョンに属し、最も必要なものはデフォルトのブラウザを再割り当てしたときに更新されたように思えましたが、それでもレジストリを確認することをお勧めします-フルパス「 x：\ Program Files（x86）\ Mozilla Firefox 」を直接検索できます

たとえば、「Mozilla Firefox.old」の古いフォルダーの名前を変更します。実験が成功したら、削除できます。
