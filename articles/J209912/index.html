私自身も長い間この質問に悩まされてきましたが、アプリケーションのサーバーバックエンドをどのように整理できますか。 最初は、WCFサービスを使用して独自の何かを書きました。 次に、 ASP.NET Web APIが登場しました。 しかし、今日は何か他のことについてお話したいと思います。 使いやすいもののファンとして、私はWindows Azureクラウドプラットフォームに登場した比較的新しいサービスをすり抜けることができませんでした。 このサービスの名前は、Windows AzureモバイルサービスカスタムAPIです。



このサービスは、Windows Azureモバイルサービスの他の便利な機能とともに、PaaSソリューションであり、クラウドベースのRESTful APIを迅速に展開する機能を提供します。これには、任意の言語およびプラットフォームのプログラムからアクセスできます。 このソリューションの基盤は、新しいものではなく、むしろ人気のあるNode.jsテクノロジーでした。 カスタムAPIは、すべての結果を伴う完全に機能するNode.jsアプリケーションです。妥協のない完全なソリューションです。 そして、3つの人気のあるモバイルプラットフォームすべてでネイティブSDKを使用するためにネイティブSDKが作成されたという事実を考慮すると、このソリューションはさらに興味深いものになります。

APIの使用



この記事では、Windows Azureモバイルサービスが提供してくれるテストアプリケーションを使用します。 これを行うには、[クイック作成]ページ（これはクラウドとアイコンが点灯しているページ）に戻り、[Windows Phone 8]を選択します（豊富な選択に注意してください）。[新しいWindows Phoneアプリの作成]をクリックします。





必要なプレート（TodoItem）を作成し、[ダウンロード]ボタンを使用してアプリケーションをダウンロードした後、Visual Studioで開きます。



まず、2つのことに興味があります。 ファイルApp.xaml.csには、次のような行があります。 



public static MobileServiceClient MobileService = new MobileServiceClient( "https://mva-test-api.azure-mobile.net/", "___" );




このフィールドを使用して、新しく作成したモバイルサービスと通信します。 あいまいな文字のセットは、サービスの個々のキーであるApplicationKeyです。



アプリケーションを実行して、少なくともその外観を見てみましょう。





まあ、それは新興企業のヒットではありませんが、スタートには悪くありません。 追加のロジックを付与して、APIメソッドを呼び出しましょう。 MainPage.xaml.csファイルに移動します。 彼から、[保存]ボタンをクリックすると、サービスへのアクセスが発生し、その後、結果がデバッグコンソールに単純に書き込まれることを確認したいと思います。 これを行うには、InsertTodoItemメソッドの最後に次のコードを追加します。 



 //   POST var result = await App.MobileService.InvokeApiAsync("CoolAPI", null, HttpMethod.Post, null, null); Debug.WriteLine(result.StatusCode); var stringData = await result.Content.ReadAsStringAsync(); Debug.WriteLine(stringData); //   GET var resultJson = await App.MobileService.InvokeApiAsync("CoolAPI", HttpMethod.Get, null); Debug.WriteLine(resultJson);




このコードのメインメソッドは、InvokeApiAsyncと呼ばれます。 彼はAPIメソッドの呼び出しを担当しています。 このメソッドはオーバーロードされ、異なるパラメーターのセットを備えています。 この例は、POSTの場合、最大5つのパラメーターを渡し、GETの場合、3つだけを渡すことを示しています。これは、POSTのメソッドが通常の文字列になるように設計されているためです（バックエンドでのスクリプトの実装を思い出してください）、 GETを使用したオプション-JSONオブジェクトを操作する（結果はNewtonsoft.Jsonになります ）。

彼らはどうですか



Windows Azureコントロールパネルの[クイック作成]ページでアプリケーションの種類を選択する可能性に注意を払うだけではありません。 実際、モバイルサービスを操作するためのネイティブSDKは、Windowsデバイス用だけでなく作成されています。 それらのライブラリはiOSとAndroidの両方でリリースされているため、これらすべてのプラットフォームで同じものを使用できます。 以下は、例とほぼ同じことを行うObjectiveCコードの例です。 



 [self.client invokeAPI:@"CoolAPI" data:nil HTTPMethod:"POST" parameters:nil headers:nil completion:^(NSData *result, NSHTTPURLResponse *response, NSError *error) { NSLog(@"%i", response.statusCode); NSString *stringData = [[NSSatring alloc] initWithData:result encoding:NSUTF8StringEncoding]; }];




ご覧のとおり、詐欺のない「クリーンな」ObjectiveC。 Android、WinRT、およびWebバージョン（HTMLおよびJavaScript）でも同様です。
