一年として一年

こんにちは、ハブロフチャン。

4月には、 Relational Programming （以下、略してRP）という会社が出現してから1年が経過しました。 この年は波乱に富み、ポーランド共和国で働くすべての人々にとって、まるで一瞬であるかのように過ぎ去りました。

Habréの最初の投稿、そして実際、一般にポーランド共和国の歴史の最初の投稿で、私たちはなぜ、どのように現れたのか、過去1年間で何ができたのかについて話します。 将来的には、当社の製品が何に適しているか、RPアプリケーションを開発する方法、RP開発者およびイデオロギー家が行うこと、RPがすでにRP開発を行っているか、RPサーバーをタスクを完了するための「働き者」。

はい...別の瞬間。 皆様のご支援、批判、質問をお待ちしております。 私たちは本質的にすべてに詳細に答え、可能な限り広告を避けようとします。

ですから、RPと会社の主要製品であるRPサーバープラットフォームについてです。

昨年末から業界メディアでRPサーバーと呼ばれていたビジネスアプリケーションを開発および実行するためのソフトウェアのメインアイデアと最初のバージョンは、15年以上前に登場しました。



最初のアイデアは、システムパーツの開発と、BOSSシリーズのいくつかのソフトウェア製品のアプリケーションコードを分離することでした。 利点-完成したアプリケーション製品の開発時間を短縮し、実装とメンテナンスの段階で時間とお金を節約します。 特定の企業やユーザーのニーズに合わせて、アプリケーションの製品を実質的に無制限の制限内で（サブジェクトエリアのフレームワーク内で）合理的な実装期限で適合させる能力も影響を受けます。

上記の「システム部分」と呼ばれていたのは、RPサーバーです。 企業向けの応用製品の開発および実行環境。 適用された製品の1つの例はBOSS-Kadrovikであり、ロシア連邦の大企業の20％で機能します（エキスパートRAレーティングTOP-100およびTOP-400から、PC Week No. 21、2010年6月8日を参照）。 ちなみに、2011年5月には、BOSS-Kadrovikトーナメントが顧客企業の1つで開催され、勝者は9つのノミネートで選ばれ、ギフトを受け取ります。

約3年前、エンタープライズ開発者がBOSSシリーズアプリケーションツールを使用して、他の主題分野の問題を解決していることに気付きました。 追加の研究が行われ、その結果は興味深いことが判明しました。用語であるシステム部分であるアプリケーションの開発および実行環境は、標準的な企業自動化タスクと非常に具体的なタスクの両方を解決するためによく使用されることがわかりました。 柔軟で便利なため、技術機器や他のシステムなどと統合できます。

2010年の初めに、システム部分を個別の製品に分離することの妥当性を評価し、新しい製品の予備設計を開発し、少し後にRPサーバー（リレーショナルプログラミングサーバー）プラットフォームという名前を思いつきました。

2010年の終わりに、彼らは3つのエディションでRP Serverのリリースを発表しました：Express、Standard、およびEnterprise。 RP Server Expressは製品の完全無料版であり、その主な制限は、ライセンスキーの共同開発と生成を整理するためのツールの欠如です。 StandardおよびEnterpriseのエディションは、配信パッケージに含まれるMicrosoft SQL Serverのバージョン（StandardおよびEnterprise）がそれぞれ異なります。

2011年の第2四半期に、RPサイトにはすでに製品の無料版の最終バージョンがあり、製品版の商用バージョンの購入者向けにセクションが開かれました。
