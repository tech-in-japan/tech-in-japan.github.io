アスタリスク+ FreePBXでのオフィステレフォニー

先史時代


政治的な理由から、私はオフィスの電話をパナソニックKX-TDA200ハイブリッドアナログPBXからSIPに移行するという課題に直面しました。 オフィスにSCSが存在することにより、タスクが簡素化され、コールをすばやく切り替えるためのボタンを備えたパネルのオペレーターによる使用が複雑になりました。



考慮されるオプション：

  • PBXをKX-TDEシリーズにアップグレードする
  • CiscoやAvayaなどの鉄製品を購入する
  • アスタリスクを入れる


フリーソフトウェアの提唱者として、私は上司に3番目の選択肢を受け入れるよう説得しました。 さらに、このタスクに使用できるサーバーを解放しました。



購入と準備


市の電話ネットワークとの通信用に、 OpenVox DE130Eボードが購入されました。 電話はGrandstreamによって購入されました。



私はサーバーに置きます：

  • debianの現在のバージョン
  • dhcp3-server-アドレスとパラメーターを電話に配布するため
  • ntp-電話に現在の時刻を提供する
  • tftp-電話機にファームウェアと構成を提供する
  • mysql、apache、php-FreePBX用
  • build-essential-アスタリスクの構築用

FreePBXが機能するには、Asteriskにマネージャーインターフェイスが必要です。 このモジュールを有効にすることを忘れないでください。FreePBXをインストールする前に、正しいパスワードとパーミッションを/etc/asterisk/manager.confファイルに入れてください。

驚いたことに、熊手は現れませんでした。 厳密に指示に従ってダウンロードし、データベース、ユーザーを作成し、スクリプトを実行し、起動しました。 設定インターフェイスに移動できます。



カスタマイズ




初期設定


「FreePBX Notices」領域の管理インターフェイスの最初の画面で、/ etc /アスタリスクにFreePBXが管理すべき多くの構成ファイルが含まれているというメッセージが大量に落ちました。 これらのファイルを削除しました（コピーを保存しました）。 大きな赤い「構成を適用」ボタンをクリックすると、FreePBXは必要なファイルを自動的に生成しました。



FreePBXモジュールの選択


「管理者-モジュール管理者」に行きました。 Basic、Extendedリポジトリ、および念のためUnsupportedに注意してください。 「オンラインで確認」をクリックして、自分でモジュールを入力する機会を得ました。 デフォルトのものに加えて、私は選択しました：

  • 通話録音-会話の録音用
  • 会議-パスワードなどを使用して会議を開催するため
  • リンググループ-グループコール処理（秘書）
  • PBX End Point Manager-電話構成の作成用
  • ファックス設定-ファックスを設定します




電話設定の準備


電話機のグループ設定は、End Point Managerを使用して非常に便利に行われます。



接続-エンドポイントの構成では、使用する電話のメーカーとモデルを選択する必要があります。 プロジェクトは非常に活発で、常に新しいモデルをリリースするため、初心者の場合は「更新の確認」をクリックする必要があります。



次に、接続性-エンドポイントの詳細設定で、tftpがファイルを配布するディレクトリへのパスを指定する必要があります。 そして今、いくつかのオプションがあります：

  • すべての電話を接続し、ダウンロードを待って、「接続性-エンドポイントデバイスリスト」からネットワークをスキャンできます。
  • デバイスを入力するときに、End Point Managerセクションに入力して、MACアドレスとデバイスモデルを指定できます。
  • デバイスを1つずつ追加できます


モデルごとに1台の電話機を登録し、設定テンプレートを編集してから、各デバイスを入力するときに、MACアドレスを指定してモデルとテンプレートを選択しました。 その後、End Point Managerは正しいcfgMACファイルを生成し、tftpで配布するためにディレクトリに配置します。



DHCPセットアップ


電話機が設定を正しく取得するには、これについて通知する必要があります。 /etc/dhcp/dhcpd.conf



ファイルからの/etc/dhcp/dhcpd.conf



は/etc/dhcp/dhcpd.conf

 subnet 192.168.xxx.0 netmask 255.255.255.0 { range 192.168.xxx.10 192.168.xxx.240; server-name "192.168.xxx.1"; option domain-name "my.domain"; option domain-name-servers 192.168.xxx.1; option domain-search "my.domain"; option routers 192.168.xxx.1; option nntp-server 192.168.xxx.1; option tftp-server-name "192.168.xxx.1"; next-server 192.168.xxx.1; }

設定終了


外界との通信回線を設定するには、「接続性-トランク」画面を使用します。 SIPプロバイダーとの接続はそこに登録され、私の場合はOpenVoxボードを介したフローです。

着信コールの処理方法を説明するには、「接続性-着信ルート」を設定します。 私はそこに、どの着信番号がコールを送信するかを示したストリームがあります（DID番号がどのフォーマットで来るか、オペレーターに確認してください）。

発信通話の処理方法をスケジュールするには、接続-発信ルートを使用します。 ここでは、たとえば、sipnetを介した都市間、またはプレフィックスをダイヤルすることによるオペレーターの選択を構成できます。



追加の特典


FreePBXはすべての構成ファイルを自動的に生成しますが、奇妙なものが必要な場合はどうでしょうか？ 幸いなことに、開発者はそのような要望に応えてきました。 使用されるすべてのコンテキスト、マクロ、またはサブルーチンでは、element -custom



要素が必須で含まれています。

追加のチップを/etc/asterisk/extensions_custom.conf



ファイルに配置する必要があります。 たとえば、私はこれを追加しました： 

 ; Talking clock for Russian [sub-hr24format-custom] exten => ru,1,Playback(at-tone-time-exactly) exten => ru,n,SayUnixTime(${FutureTime},,kM) exten => ru,n,Return() ; Directed FAX [app-fax-custom] exten => 667,1,Set(FAX_FOR_NUM=${CONNECTEDLINE(num)}) exten => 667,n,NoOp(Directed call transfer from ${CALLERID(all)} for ${FAX_FOR_NUM}) exten => 667,n,GoTo(ext-fax,${FAX_FOR_NUM},1)


最初のブロックでは、すべてが明確で、箱から出して、時計のロシア語の設定はまったくありません。 しかし、2番目のブロックは興味深いかもしれません。 デフォルトでは、システムFAXは666にあります（はい、FreePBXの作成者は明らかにFAXが悪であると考えています）。 この番号に接続すると、FAXが受信され、「FAXのシステム受信者」としてシステムで指定された電子メールアドレスに送信されます（Exchangeにこの共有フォルダーがありますが、通常はスパムだけが届きます）。 また、手動ファックスで主任会計士のイヴァン・イワノビッチを受け取りたい場合はどうなりますか？ そしてもちろん、彼はそれを共有フォルダーに入れたくありません。 私の電話設定では、Ivan Ivanovichは次のとおりです。 

 Fax: Enabled Fax Email: Ivan.Ivanov@my.domain


そして今、Ivan Ivanovichが電話を番号667に転送すると、受信したFAXは彼の個人メールに送信されます。 貧しいイヴァン・イワノビッチが苦しみ、tiffを開こうとしないようにGhostScriptを置くことを忘れないでください。

  • OpenVoxボードにはまったく問題はありませんでした。 すぐに離陸しました。
  • Grandstreamの携帯電話は、一方で便利で簡単にカスタマイズできることが判明しました。 一方、ビルドの品質はひどいです。 保証対象のデバイスの10％以上を手渡しました-誰もがハンドセットを持たず、スピーカーを持っている人もいれば、マイクを持っている人もいました。 DECT DP715電話を別にお勧めしません。
