前の記事「Microsoft SQL Server用のツールの完全なリスト」のコメントには、SQL Serverの各ユーティリティの簡単な説明がないことに関連する多くのコメントがありました。

これは、示唆的なコメントの1つです。







<そして、そのような選択のポイントは何ですか？ 読者は、すべてのユーティリティをクリックして、それぞれの機能を理解することになっていますか？ それらの143があり、名前で、神は禁じ、10％。 少なくとも各ユーティリティの簡単な説明を追加してみませんか？







この記事へのこの種のコメントの出現を予想して、すぐに答えようとします。







  1. リストは主に個人用に作成され、SQL Serverを操作するための便利なツールを探す際の出発点としてのみハブに公開されます（個人的には、より詳細で関連性のあるリストを見つけることができませんでした。成功した人は、コメントで報告してください。 ）
  2. 各拡張機能の簡単な説明は英語（ Githubを参照）であり、 はい （上記のコメントへの回答）、読者はそれをクリックして、自分に合ったものだけを選択する必要があります。
  3. リストにない便利なツールがあれば、いつでもgithubでプルリクエストを取得するか、それについてのコメントを書くことができます。
  4. リスト内の拡張機能の場所は、個人用ユーティリティの順に作成されます。 一番上は、作業で常に使用されている拡張機能です。以下は、あまり理解されていない拡張機能です。
  5. すべての拡張機能がダウンロードされ、SQL Server 2012およびSQL Server 2014との互換性がテストされました。


私の主観的な意見で最も有用な拡張は、 SSMSBoostです。

SQL Serverを操作するためのメインツールとしてSSMSを使用し、SSMSBoostを試したことがない場合は、この拡張機能を緊急にインストールし、新しい開発者ごとに必須のリストに追加してください。
