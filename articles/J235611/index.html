Github、Reddit、およびStackExchange Propose Markdown Standard Syntax

Markdown Markup Languageは、2004年にJohn GruberとAaron Schwartzによって開発されました。 Markdownの哲学は、読みやすく、同時に自動的に通常のHTMLに変換されるテキストを書くことです。 著者はすぐにPerlパーサーをリリースし、すぐに大きな人気を得て、Markdownは大衆に行きました。



残念ながら、一般に受け入れられているMarkdown仕様はまだ採用されておらず、 混乱が生じています。 Gruberの標準的な説明では、すべての質問に対する回答や、前述のパーサーのコードが提供されていません。 多くの場合、明らかに悪い結果をもたらします。 一般に、問題は10年間続きます。



この決定は、Berkeley大学のJohn McFarlane（PandocコンバーターmarcdownコンバーターおよびBabelmarkテストの著者）、Meteor、Github、Reddit、StackExchangeおよびDiscourseの代表者を含む活動家グループによって提案されました。 2年間のコラボレーションの中で、彼らはStandard Markdownプロジェクトの「最高の構文仕様」に同意しました（ ） また、各Markdown実装の仕様をテストするための包括的なテストスイートもリリースしました。



少なくとも参加者の構成に基づいて、プロジェクトには良い見通しがあります。 これらすべてのサイトが単一のMarkdown構文に切り替わると、公式の仕様がなくても、事実上事実上の標準になる可能性があります。 最終的に、同じPerlはそれらなしで成功した開発を妨げませんでした。



イニシアチブはすべての賞賛に値するように思われます。 ただし、John Gruber自身は、見知らぬ人が構文の次の実装を「標準」と呼ぶことにsomewhatしている。 構文の実装は24個以上ありますが、なぜ最初から25日目が突然「標準」と呼ばれるのでしょうか？ 一般に、これはITの世界では非常に一般的な古典的な状況です。







さらに、少し前に、W3CコンソーシアムはMarkdown Community Groupを作成しました。これは、Markdownの単一バージョンを開発する取り組みを調整する必要があります。 それらの背後にある「標準」仕様の作業とリリースは、少なくとも、このプロジェクトに時間と労力を費やした人々に対するある種の無礼の指標です。



ただし、Meteor、Github、Reddit、StackExchangeは、Web開発者に大きな影響を与えるサイトです。 また、単一の構文実装について合意した場合、多くのWeb開発者がそれに続く可能性が高くなります。 そして、これは最も重要なことです。 Standard Markdownの作者は、コミュニティとの議論の後、バージョン1.0をリリースする準備ができていると言います。
