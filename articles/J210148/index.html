15分でフリーソフトウェアに基づくオンラインラーニングセンター、またはBigBlueButtonについて再び

この記事は、記事「無料遠隔教育」（ http://habrahabr.ru/post/161783/ ）の続きです。 今回は、Moodleと組み合わせたBigBlueButtonオンライン会議サーバー（以下、BBB）の実用化に最も注意を払っています。



事実は、「無料通信教育」という記事を書いた後、BBBをmoodleとは別に使用する方法、ユーザー認証を整理する方法、同時クライアント数のBBB機能、ローカルタスク用のBBB内の特別なモジュールの開発など、多くの魅力がありました仮想部屋内のビデオデモ）。 これらの質問に答えることはできますが、何らかのFAQセクションを作成する必要があります。原則として、すでに公式Webサイトにあります。 今回は、組織内にオンラインラーニングセンターを作成し、経験を共有し、仮想マシンの既製のイメージを提供するプロセスについて説明します。そうすることで、興味のある読者は、もちろん興味があるなら、そのようなシステムをネットワークに迅速に展開してテストできます

仕事の目的は、無料のソフトウェアに基づいた電話会議だけでなく、遠隔形式で教育イベントや方法論イベントを実施する問題を解決できるオンライン学習センターの組織です。 記事の最後に、既製の仮想マシンへのリンクが用意されており、それらを起動し、ネットワークインターフェイスを構成できます。その結果、オンラインラーニングセンターを運用する準備が整います。



はじめに



今日の遠隔教育の発展の歴史は100年以上あります。 アイザック・ピットマンは祖先と考えられており、1840年に英国の速記を教えるために郵便の助けを借りて始めました。 父の話から初めて遠隔学習について聞いた。 彼はトルクメニスタンで生まれて勉強し、方法論的な指示と割り当てを郵便で受け取り、1980年にロシアの大学への入学試験に合格することができました。 今日、情報技術の積極的な開発と情報ネットワークの帯域幅の増加により、大陸から大陸への膨大な量の知識の転送が可能になり、重要なことに、これはほぼ瞬時に行われます。

遠隔学習には、オンラインモード（インタラクティブレクチャー、ウェビナー）とオフラインモード（レクチャーノート、テスト、教育的および方法的複合体）の2つの主なタイプがあります。 それらの主な違いは、トレーニング資料の入手可能性と教育プロセスの構成です。 私は、遠隔教育が質の面でフルタイムに近づくことは決してないと信じています。 ただし、インタラクティブな要素を使用したオンライン学習であり、全員（学生と教師）が仮想聴衆に同時に集まり、チャットで音声やテキストを尋ねることができる場合、フルタイムの学習プロセスに少なくとも少し近づくことができます。 論理的な質問は次のようになります：なぜ一般的なオンライントレーニング、古典的なフルタイムがあるのですか？ 簡単な例を挙げましょう。「数学の教師」の学位を取得した12人の卒業生のための、物理学および数学の学部の卒業生の就職日には、12人の校長が来ます。



システムの説明



概して、遠隔教育センター、オンライン教育センター、または現在IDO（遠隔教育機関、おそらくすべての大学に既にある）に電話するのが非常に流行しているため、目的のコースを注文したり、既存のコースを購読したりできる小さなサイトです。 その結果、承認が行われるWebポータルと、クラス自体を実施できるWeb会議サーバーがあります。 大学のセンターでは、2つの主要な課題が解決されました。地区の学生を対象とした試験に合格するための遠隔学習コースと、学生向けのオンライン講義の開催です。 作業の最初の部分の必要性に挑戦できない場合、2番目は省の新しい要件の結果です：「大学の標準は、ネットワークモードで実装されたプログラムの少なくとも20％の存在であるべきです」（http://www.rg.ru/2013/10 /29/kontent.html）。 私たちの組織では、多くの人がこの状況をinりながら取りましたが、個人的には正しいと思っていました。 受講者にとっては、トレーニングポータルに接続し、講義、特にクラス内の講義でパーソナルラップトップからプレゼンテーションスライドを直接フォローする方がはるかに便利です。これは講師が引き続きそのようなプレゼンテーションを管理するため重要です。 正当な理由でリモートで接続する学生に関しては、ここの全員がいくつかの利点を理解していると思います。

センターサイトのエンジンの選択（Wordpress、DLE、Joomla-好きなものを選択）ですべてが簡単な場合は、前回の記事でWeb会議サーバーの選択も決定しました。 実際、オープンソースソリューションの選択肢はそれほど大きくありません。BigBlueButtonとOpenmeetingsです。 最初に決めました。

BBBを使用する過程で、長所と短所の両方が明らかになりました。



利点：


1.一般的なCMSとの統合。

2.記録する能力。

3.リモート形式のイベントへのシステムの適応（2013年11月8日の最新バージョンでは、6つの形式の仮想ルームが定められました-ウェビナー、ビデオチャット、アシスタント講師など）。

4.プレゼンテーションの上にあるインタラクティブホワイトボードモード（テキストラベル、マーカー）で作業するための一連のツール。

5.音声にFreeSwitchを使用します。

6.デスクトップをブロードキャストします（誰かがデスクトップのコンテンツをブロードキャストできる仮想カメラなどのサードパーティソフトウェアを使用する方が簡単になる場合があります）。



短所：


1.設定と使用の複雑さ。 BBBは何か（たとえば、Moodle）と組み合わせて使用​​するか、ユーザーの承認と権利の差別化のために独自のモジュールを追加できます。

2.「低速」接続への強い依存。複数のパケットが失われた場合、セッションが終了し、ユーザーは仮想ルームに再度入らなければなりません。

3.ユーザー数の制限。 BBBは小グループでの作業には非常に優れたシステムですが、BBBのベースであるRED 5サーバー自体は、クライアントの最大数に制限があります（RED 5についてはこちらhabrahabr.ru/post/143554 ）。

したがって、BBBを実際に使用した結果、このプラットフォームは小グループ（最大20人）での作業に最適であるという結論に達しました。 確かに、「練習」に関しては、教師がレッスンの各参加者を聞いたり見たりすることが重要な場合、特定の参加者から質問をして回答を得ることができ、プレゼンテーションをダウンロードして次の生徒にレポートを開始する権利を与え、生徒にスライド上で示すように頼みますわかりにくいマーカーなど -BBBは完璧です。 しかし、講義クラスがあります。200〜300人、または500人の人々に公開する必要がある講師がいる場合、彼はプレゼンテーションと一緒に見たり聞いたりすることが重要です。 この場合、何らかの軽いビデオサーバーでメディアストリームを開始したいだけで、テキストチャットによるフィードバックは最小限にすることが望ましいです。 この場合、Dmitry Shatrov（プロジェクト「momentvideo」）が私たちの助けになり、ビデオ放送用のサーバー「Lectorium」がチャットの可能性とともに作成され、Moodleの統合モジュールが開発されました。

同時に、lectoriumサーバーのロードについて主張されている数値に注目する価値があります。

次の表は、Intel Core i7 2600プロセッサを搭載した64ビットシステムで500 Kbpsのビデオストリームを配信したときのビデオサーバーのパフォーマンスを測定した結果を示しています。 必要なRAMの量の下では、ビデオサーバープロセスが占有するメモリの量として理解されます。



10,000は良い数字ですが、私たちにとっては達成できないことに注意してください。 サーバーは1000 Mbit / sの速度で接続され、最小ストリームは640 x 480のビデオストリームで200 kbit / s +音声48 kbit / s、合計250 kbit / sになります。 4人-1メガビットまたはサーバーの最大接続数は4000人です。 私たちのタスクでは、これで十分です。

インストールプロセス



インストールプロセスを可能な限り簡素化するために、2つの仮想マシンに基づいて15分でオンラインラーニングセンターを展開する方法を示す短いビデオを用意しました。 このようなセンターの組織は、ユーザー認証サーバー（たとえば、Moodleオープンソースソフトウェア）とビデオ会議サーバー（BBB）の2つの仮想マシンを作成することから成ります。 便宜上、対応する仮想マシンのイメージがレイアウトされているため、誰でも展開して、環境内でどのように見えるかを確認できます。 （ルートのパスワードは222123であることに注意してください）

おわりに
