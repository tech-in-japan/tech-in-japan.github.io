Cisco 7912G IP PhoneファームウェアをSIPからスキニーに変更する

シスコでのテレフォニーのセットアップに関するトピックの最初の部分の続きを書く過程で、2台の電話をCMEに接続する必要がありましたが、1つの問題に遭遇しました。 電話はSIPの下でフラッシュされましたが、この段階でSkinnyですべてをやりたかったです。 また、SIPのSkinny-phoneファームウェアに特別な問題がない場合（このプロセスは既にHabréで説明されています）、逆の手順は非常に簡単なことが判明しました。



したがって、この問題を解決するには2つの方法があります。1つは、サーバーに展開された本格的なCisco Unified Call Managerの使用です。 彼の手はまだ彼に届いていないので、このオプションは受け入れられませんでした。

2番目の解決策は、CMEルーターを使用したフラッシュです。

まず、ファームウェア自体が必要です。CiscoWebサイトの最新バージョンは8.0（4）で、ダウンロードするファイルの名前はcmterm-7912G-sccp.8-0-4.zipです。 しかし、問題があります。それをダウンロードするには、MTRアカウントの特権が必要です。これは私には持っていません。そのため、別のソースを探す必要がありました。 アーカイブにはファイルCP7912080003SCCP070409A.sbinが含まれています。

次に、TFTPを介してファイルをルーターにアップロードし、電話機のDHCP設定を作成します。

ip dhcp pool PHONES

network 10.13.6.0 255.255.255.0

default-router 10.13.6.10

# 150 TFTP-

option 150 ip 10.13.6.10





その後、必要なdebug debug ip dhcp serverイベントを実行し、 tftpイベントのデバッグをデバッグして 、何が起こるかを確認します 。

CME#

Jun 1 02:41:05.447: DHCPD: Sending notification of ASSIGNMENT:

Jun 1 02:41:05.447: DHCPD: address 10.13.6.22 mask 255.255.255.0

Jun 1 02:41:05.447: DHCPD: htype 1 chaddr 0019.2f9c.554e

Jun 1 02:41:05.447: DHCPD: lease time remaining (secs) = 86400

Jun 1 02:41:05.527: TFTP: Looking for gk00192f9c554e

Jun 1 02:41:05.539: TFTP: Looking for gkdefault.cfg





電話機は通常DHCPアドレス10.13.6.22を受信し、TFTPサーバーとして機能するルーターに要求を正しく送信していることがわかります。 電話機は、それ自身（gkxxxxxxxxxxxx、xxxxは電話機のMACアドレス）専用の設定ファイルまたはデフォルトの設定を要求します。 公式ソースによると、CP7102はffで始まるファイルを要求し、IDにはCP7105を要求します



この段階でのタスクは、必要な設定ファイルを電話に入れて、SCCPに転送できるようにすることです。 このファイルとファイルに含まれるフィールドについて読むことができます。



そのため、gkdefault.txt構成ファイルを作成します



#txt

upgradecode:3,0x601,0x0400,0x0100, 10.13.6.10 ,69,0x 070409A ,CP7912080003SCCP 070409A .sbin





バイナリ形式でダウンロードする必要があるため、 ここで取得できるcfgfmt.exeユーティリティを使用して変換する必要があります 。 ユーティリティには、同じ場所にある作業のためにsip_ptag.datファイルも必要です。

プログラムを実行する

cfgfmt -tsip_ptag.dat gkdefault.txt gkdefault.cfg





結果のファイルもCMEにアップロードされます。



したがって、すべての手順を実行した後、フラッシュに次のものが必要です。



CME#sh flash:



System flash directory:

File Length Name/status

1 18296372 c1700-ipvoicek9-mz.124-25.bin

2 341023 CP7912080003SCCP070409A.sbin

3 62 gkdefault.cfg





次に、必要なファイルをtftpサーバーに追加します。



tftp-server flash:CP7912080003SCCP070409A.sbin

tftp-server flash:gkdefault.cfg





この段階で、電話機の電力を歪めることができます。すべてが正しく設定されている場合、次の内容がデバッグに表示されます。

Jun 1 03:48:33.495: DHCPD: Sending notification of ASSIGNMENT:

Jun 1 03:48:33.495: DHCPD: address 10.13.6.22 mask 255.255.255.0

Jun 1 03:48:33.495: DHCPD: htype 1 chaddr 0019.2f9c.554e

Jun 1 03:48:33.495: DHCPD: lease time remaining (secs) = 86400

Jun 1 03:48:33.575: TFTP: Looking for gk00192f9c554e

Jun 1 03:48:33.587: TFTP: Looking for gkdefault.cfg

Jun 1 03:48:33.587: TFTP: Opened flash:gkdefault.cfg, fd 0, size 62 for process 127

Jun 1 03:48:33.595: TFTP: Finished flash:gkdefault.cfg, time 00:00:00 for process 127

Jun 1 03:48:33.599: TFTP: Looking for CP7912080003SCCP070409A.sbin

Jun 1 03:48:33.603: TFTP: Opened flash:CP7912080003SCCP070409A.sbin, fd 0, size 341023 for process 151

Jun 1 03:48:40.279: TFTP: Finished flash:CP7912080003SCCP070409A.sbin, time 00:00:06 for process 151





その後、 フラッシュを書き込みます...電話画面に表示されるまでお待ちください 、再起動してスキニーフォンになります！



PS

または、 tftpd32.exeを使用してDHCPサーバーを上げ 、その中に150番目のオプションを構成し、ファイルCP7912080003SCCP070409A.sbinおよびgkdefault.cfgをTFTPサーバーのルートフォルダーにアップロードできます。
