Googleで404エラーページを合理的にする

それで、私は私のサブスクリプションに掛かっているブログとサイトを調べて、すべてが悲しいことに気づきました。 404番目のエラーのページを意図的に呼び出すと、考えられるすべての言い訳、空白ページ、またはサーバーが生成するページに遭遇する可能性があります。 さて、または最後の手段として-サイトマップ。 多くの場合、彼らはそこに面白い写真を投稿するのが好きです。



たとえば、次のようなページを見つけることができます。

404-見つかりません

条件に一致する投稿はありません。


見つかりません。

申し訳ありませんが、ここにはないものを探しています。






一部の大規模なサイトでは、404エラーページが妥当であり、アドレスの記述を間違えた可能性があること、代替アドレスが提供されること、および/またはエラーが見つかったことを管理者に通知するように求められることを示します。



小規模なサイトやスタンドアローンのブログ用に同様のメカニズムを作成するのは、費用がかかり、時間がかかり、一般に、誰もが採点しました。 なんで？ 結局、Googleを使用して、404番目のエラーを妥当なものにする信頼できる機会があります。



開始するには、メソッドでよりスマートになった404番目のエラーのページを見てください。これについては少し後で説明します。

http://absolvo.ru/contactf



素晴らしいですか？ 真実はすぐに少しがっかりです。このブログ用の方法は、Googleによって完全にインデックス付けされ、Googleウェブマスターパネルにも追加されている場合、あまり適していません。



したがって、すべてが非常に簡単です。



<p>

<script type = "text / javascript"> var GOOG_FIXURL_LANG = 'ru'; var GOOG_FIXURL_SITE = 'http://absolvo.ru/'; </ script>

<script type = "text / javascript" src = " linkhelp.clients.google.com/tbproxy/lh/wm/fixurl.js "> </ script>

<ul> <li>メインの<a href="/">ページに移動</a> </ li> </ ul>

</ p>




JSスクリプトの404番目のページfixurl.jsに頭脳を追加します。検索エンジンwww.site.ruとsite.ruについて、言語（この場合はru）とサイトのURL（この場合はabsolvo.ru ）を指定する必要があります。簡単に別のサイトのように見えますが、私には致命的ではないようです。



自分の目的に安全に使用できるスタイル（たとえば、ブログにGoogle検索バーを隠しましたが、必要ありません）

＃goog-fixurl

＃goog-wm

.content

.other-things

＃goog-wm-sb


.contentを使用してリストを操作します（ul、li）。



上記で引用したコードには、Googleが代替手段を提供しない場合に汚れに直面させない追加のリストがあります。



私は自分自身を設定し、一般的に、私は満足しています。 見た目が良く、より正確です。



サイトが大きくて厚い場合、このスクリプトはいくつかの選択肢を作成できることに注意してください。
