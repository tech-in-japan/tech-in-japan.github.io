Cowon D2：ミュージカル長肝

スマートフォン市場の新製品に関する最近の会話から、この記事を書くことに触発されました。 知人の一人は、彼が次の人をどのように獲得したか、どれほど素晴らしく、他とは違うかについて話しました。 新しいワイヤレスヘッドフォンで音楽を聴くのがどれほど便利で、どれだけ素晴らしい音がするか。 彼は見て、聞いて、比較して、感銘を受けませんでした。



私自身はガジェットを頻繁に変更しません。 私は過去10年間それらのうちの1つを使用しており、私は彼を休ませるために急いでいない。







実際、さらに徐々に消滅するタイプのマルチメディアプレーヤーの代表についてお話しします。 そして、彼がどのようにして私と長い間付き合ってきたかについて。



写真はコミックで、ガジェット専用のリソースの1つで作成されています。 実際、プレーヤーは次のようになります。







まあ、またはこのように：







デバイスは2007年に買い戻されました。 2ギガバイトのモデル。その内部メモリの量は、それでもばかげているように見えました。 しかし、プレーヤーにはSDカード用のコネクタが装備されていたため、問題が大きく変わりました。



プレーヤーのサイズは小さく、2.5インチのカラースクリーンと抵抗センサーを備えています。パフォーマンスのテストにのみ適したイヤホンとタブレットに加えて、ほとんどの所有者がすぐに遠くの引き出しに投げ込む小さな厄介なスタイラスがありました。







このプレーヤーと当時生産されていた数千のプレーヤーとの違いは何ですか？



  • 自治の大きなマージン。 製造業者は52時間の再生を要求しました。 実際、後壁全体がバッテリーで占められています。
  • mp3、ogg、flac、ape、wmaなどの多数の音楽形式のサポート。
  • 高品質のサウンドとスタジオヘッドフォンを「ロック」する機能。
  • iTunesなどのソフトウェアをサポートする必要はありません。


一般に、私にとっては初めて、それで十分でした。 しかし、最初にのみ。



そうそう、D2はタッチスクリーンを装備した最初のプレーヤーの1つでした。



より大容量のメモリカード-32Gb以上が表示されるようになったとき、メモリが完全にいっぱいになると、プレイヤーが奇妙な動作を開始することに気付き始めました。フリーズしてオフにします。 ファンフォーラムでのかなりの数の議論、メーカーへの機能リクエストがありましたが、...



しかし、製造業者は発見されたバグを処理し、コミュニティの要求を満たすことを急いでいませんでした。 結局のところ、これは目に見える利益をもたらさないでしょう。 「お使いのデバイスは古くなっています。新しいモデルを購入してください」というモデルは、はるかに有益です。



しかし、製造業者の態度を無視したにもかかわらず、デバイスは恋に落ち、かなり広範なファン層を獲得しました。







コミュニティの勢力は多数のデザインテーマを作成し、ファームウェアを反転させ、特定のヘッドフォンのイコライザープリセットを作成しました。また、オープンサークルの広く知られたオープンロックボックスファームウェアのポートは、このすべての活動の神話になりました。 これにより、プレーヤーには非常に欠けていたものが与えられました。







メモリカードの容量の制限を削除します。 最新の128Gbを挿入しますか？ 簡単！







musepack、wv、alac、aiff、mod（xm、it、s3mなど）など、より多くの形式のサポート。 つまり、道路上で同じ追跡された攻撃のアーカイブを聴きたい場合、mikmodでラップトップを乗り越える必要はまったくありません。



もちろん、ソフトウェアによるリサンプリングによる24ビット96KHzトラックのサポートもあります。







キューマーキングを使用した画像のサポートがあり、これによりリスニングの快適性も大幅に向上しました。 アルバムの画像とキューをダウンロードし、変換せずにプレーヤーに投げて行きました。



公式ファームウェアの開発者から長い間要求されてきた、一時停止なしでプレイ。 構成可能なクロスフェード（スムーズなフェードと再開）、ReplayGainのサポート（アルバムまたはロードされたライブラリ全体の音量レベルの正規化）、last.fm scrobblingのサポートさえ（関連する場合）。 多数のプラグインやゲームでさえも私にはあまり興味がありませんでしたが、興味のためにDoomがリリースされました。 また、さまざまなインターフェイスが多数あります。 私は最もシンプルなテキストが好きでした。











マイナスがないわけではありません。 ロックボックスレコーダーモードは使用できません。 また、内部メモリへの書き込み機能の問題と消費電力の増加の問題（ランダム再生で20時間に対して50時間）は解決されていません。



Rockboxのインストール



前述のように、これはマルチメディアプレーヤー用の無料のファームウェアです。 公式ウェブサイト： https : //www.rockbox.org/



ブートローダを変更することにより、標準の隣に配置されます。



まず最初に、githubからソースを取り出します（これはすべて、私の愛するDebian安定版から行います）。 



$git clone 'https://github.com/Rockbox/rockbox'


ツールチェーンの構築に必要な依存関係を配置します。 



 #apt install build-essential texinfo libtool-bin flex bison


ツールチェーンの配置： 



 #tools/rockboxdev.sh Download directory : /tmp/rbdev-dl (set RBDEV_DOWNLOAD or use --download= to change) Install prefix : /usr/local (set RBDEV_PREFIX or use --prefix= to change) Build dir : /tmp/rbdev-build (set RBDEV_BUILD or use --builddir= to change) Make options : (set MAKEFLAGS or use --makeflags= to change) Restart step : (set RBDEV_RESTART or use --restart= to change) Target arch : (set RBDEV_TARGET or use --target to change) Select target arch: s - sh (Archos models) m - m68k (iriver h1x0/h3x0, iaudio m3/m5/x5 and mpio hd200) a - arm (ipods, iriver H10, Sansa, D2, Gigabeat, etc) i - mips (Jz47xx and ATJ-based players) r - arm-app (Samsung ypr0) x - arm-linux (Generic Linux ARM: Samsung ypr0, Linux-based Sony NWZ) y - mips-linux (Generic Linux MIPS: AGPTek Rocker) separate multiple targets with spaces (Example: "sma" will build sh, m68k and arm)


そして、オプション「a」を選択します。 次に、ツールチェーンをポンピングして組み立てるプロセスが続きます。 別のプレフィックスが指定されない限り、これはすべて/ usr / localに配置されます。 ただし、/ opt内の別のディレクトリに移動したり、checkinstallで練習したり、完全なパッケージをコンパイルすることもできます。



しばらくすると、ツールチェーンの準備が整います。 パッチャーを置く： 



 $cd rbutil/mktccboot; make && cd -


出力では、rockbox / rbutil / mktccboot / mktccbootディレクトリに移動し、これを使用してブートローダーをアセンブルします。 



 $mkdir build; cd build; ../toold/configure && make


リストで、項目「33」を選択してから、項目「B」（ブートローダー）を選択します。 ビルドプロセスが進行中です。 出力はファイル「bootloader-cowond2.bin」です



公式ファームウェアを解凍します。 



 $unzip D2_2.59.zip


そしてパッチ： 



 $../rbutil/mktccboot/mktccboot D2_2.59/1/d2N.bin bootloader-cowond2.bin d2n.bin OF entry point: 0x2000006c New entry point: 0x203204fc


出来上がり、d2n.binファイルを取得し、プレーヤーの内部メモリにコピーします。 プレーヤーをusbから切断し、電源を入れます。短いファームウェアプロセスで完了です。



次に、同じものを使用してロックボックス自体を収集できます（「N」を選択して）。 



 $tools/configure && make && make zip


または、サイトから完成したものをダウンロードします。 



 wget 'https://build.rockbox.org/data/rockbox-cowond2.zip'


収集またはダウンロードされたファームウェアのカタログがメモリカードにスローされます（以前に解凍されました）。 私の場合（カードリーダーを使用したい）： 



 cp -r .rockbox /media/sdb1-usb-Mass_Storage_Dev


プレーヤーにメモリカードを挿入し、無料で完全に機能するファームウェアを取得します。
