W3Cは警告しています：HTML 5は実験用です！

HTML 5を取り巻く誇大宣伝にもかかわらず、仕様は互換性の問題のためまだ使用できる状態にありません。 これはW3Cコンソーシアムの公式な立場であり、W3C インタラクションドメインの責任者であるPhilippe LeHégaret が 1週間前にInfoWorldとのインタビューで述べました 。 このグループは、ユーザーインターフェイス（X）HTML、CSS、MathML、SMIL、SVG、XFormsの技術を開発しています。



「現在、HTML 5はさまざまなWebブラウザでさまざまな実装段階にあります。 それらを見ると、最も積極的な実装はベータ版であることがわかります、とLe Egareは言いました。 -たとえばIE9は、来年半ばまで最終リリースを予定していません。 これは、ほとんどの企業が新世代のブラウザへの切り替えを開始する瞬間です。」



同時に、Le EgareはHTML 5が「革新的な技術」（ゲームチェンジャー）であることに同意したため、レンダリングエンジンを制御できる内部ネットワーク上で新世代のWebアプリケーションを作成することを企業に提案します。 オープンWebでの開発はまだ可能ではなく、実験のみです。



Le Egareは、Flashテクノロジーについていくつかの言葉を述べました。 一部のマーケットプレーヤーはHTML 5をFlashの代替品と考えていますが、今後数年のうちにHTML 5を放棄することはできません。 すべてのWebクライアントがHTML 5のサポートを開始するには長い時間がかかります。彼はIE6をすでに10年前の人気クライアントの例として挙げました。



HTML 5仕様の最終承認は2〜3年後に予定されています。 Le Egareは、2011年半ばまでに機能の完全な説明をまとめることができ、その後コメントの収集を開始できると考えています。 ワーキングドラフトの後には、推奨候補、推奨の提案、そしてW3Cの最終的な推奨が続きます。 その過程で、たとえばビデオコーデックに関する特許の問題をさらに解決する必要があります。



権利管理でも何かする必要があります。 つまり、W3Cは仕様にDRMアナログを含めないため、コンテンツ所有者自身が何かをしなければなりません。 Le Egareによれば、テクノロジーをオープンにすると、ハッカーは数日でそれを破るので、意味がありません。



つまり、この技術はまだ完全に未加工です。 しかし、W3Cからのこの意見にもかかわらず、一部の大規模なサイトは既に実稼働環境でHTML 5を使用しており、実験用ではありません。 コンソーシアムの代表者と議論するかのように、昨日Facebookの開発者は、過去数ヶ月にわたってHTML 5で行った機能のリストを公開しました 。 一般に、FacebookプログラマーはW3Cの推奨事項に根本的に同意しません。







最高のHTML 5本の1つの共著者であるRemy Sharpも、同僚との意見の相違を表明し 、各論文について合理的に回答しました。
