誰もが十分なIPアドレスを持つのはいつですか？

IPv4を介してグローバルネットワークで利用可能なデバイスのIPアドレスの数は約43億に達し、IPアドレスの長さが32ビットであるため、すでにほぼ枯渇しています。 IPv6アドレス空間には128ビットが関与するため、アドレス可能なデバイスの数は事実上無限になります。 IPv4とIPv6には、相互に直接かつ単純な互換性はありません。 これは、IPv6への迅速かつ安価な移行のせいです。 怖い いやいや この道を歩む必要があります-遅かれ早かれ、「純粋な」IPv6の時代に入ります。 それは良いですか悪いですか？

IPv6 自体は1998年に登場し、 RFC2460で説明されています;モバイルネットワークでのIPv6のサポートは、3GPP Rel99勧告（2000）に登場しました。 この技術は広く使用されていません。 おそらく、世界のほとんどの人がIPv6が何であるかを知らず、IPv4アドレスの不足はありませんでした。 10年後、緊急の必要性はなかったため、内部テストを実施しましたが、導入しませんでした。



LTEネットワーク用の3GPP Rel8から始まって、デュアルスタックの時代が到来しています。 その利点は明らかです。 デュアルスタックにより、 NAT64やDNS64などのテクノロジーが不要になり 、IPv4ネットワークとの「下位互換性」が維持され、テクノロジー間のスムーズな移行が実現します。



それは良くなり、すでに使用することができました。 残念ながら、接続を確立すると、ネットワークは2つのPDPコンテキストを発生させ、ネットワークリソースを非効率的に使用します。







Dual-Stackのさらなる開発は3GPP Rel9で受けました。 2G / 3Gネットワ​​ークのサポートを導入し、同じPDPコンテキスト（ベアラー）内でIPv4およびIPv6アドレスを発行することも可能になりました。 MTSネットワークで使用されるのはこの技術です。







少し詳しく見ると、3GPP Rel10はDHCPv6 Prefix Delegation（DHCPv6-PD）テクノロジーを説明していると言えます。 現在、モバイルネットワークでの使用の必要性は明らかではありません。 特定の提案がある場合-コメントまたはPMに記入してください。



テクノロジーが機能するために、必要な機能がネットワーク要素HLR / HSS、SGSN / MME、GGSN / PGW、PCRF、OCS、CDRコレクター、そしてもちろんトランスポートネットワークに含まれていました。







HLR / HSS

サービスの接続により、HLRおよびHSSでは、APN設定のプロトコルタイプがIPv4からIPv4v6（またはその両方）に変更されます。 TS 29.272（Rel9）仕様には、次の逐語的な記述があります。



7.3.62 PDNタイプ...
7.3.62 PDNタイプ

PDN-Type AVPは列挙型であり、PDNのアドレスタイプを示します。 以下の値が定義されています：



IPv4（0）

この値は、PDNにのみアクセスできることを示すために使用されます。

IPv4モード。



IPv6（1）

この値は、PDNがIPv6モードでのみアクセスできることを示すために使用されます。



IPv4v6（2）

この値は、IPv4モード、IPv6モード、およびデュアルスタックIPv4v6をサポートするUEの両方からPDNにアクセスできることを示すために使用されます。



IPv4_OR_IPv6（3）

この値は、IPv4モードまたはIPv6モードでPDNにアクセスできることを示すために使用されますが、デュアルスタックIPv4v6をサポートするUEからはアクセスできません。 UEはサポートされているPDNタイプの1つ、つまりIPv4のみ、IPv6のみ、またはIPv4v6（デュアルスタック）のみを使用するため、この値はUEから要求されたPDNタイプとして決して使用されないことに注意してください。 この値は、HSSとMME間の認証メカニズムとして、APNサブスクリプションコンテキストの一部としてのみ使用されます。



ご想像のとおり、オプション2を使用します。



SGSN / MME

これらのデバイスをIPv6をサポートするように構成するのは簡単です。実際には、SGSNとMMEサービス、およびSGSNに接続されたRNC設定で、対応するライセンスのアクティブ化とデュアルアドレスPDPサポートの有効化に制限されます。



PGW / GGSN

すでにそれほど単純ではありません。 アップグレードは次の対象となります。

-トランスポートネットワークへのインターフェース（アドレスファミリipv6が上昇）、およびIPルーティングが登録されている。

-IPプールが登録され、そこからサブスクライバーにアドレスが発行されます。

-APNでのIPv6サポートが有効化されます。

-IPv6 DNSが登録され、接続時にサブスクライバーに発行されます。



PCRF / OCS

PCRFおよびOCSとのPGW交換は、Diameterプロトコル（それぞれGxおよびGyインターフェイス）を使用して実行されます。 IPv6アクティベーションの特性は、AVP Framed-IPv6-Prefixがこの信号交換に表示され、IPv6アドレスを持つもう1つのAVP PDP-Addressが追加されることです。 したがって、PCRFおよびOCSはそれらを受け入れ、説明する（保存する）必要があります。
