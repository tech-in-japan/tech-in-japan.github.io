InfoboxCloudはエンタープライズSSDキャッシングを開始しました。 Amazon EC2およびWindows Azureのベンチマーク

すべてのユーザーがモスクワとアムステルダムのInfoboxCloudにエンタープライズSSDキャッシングを追加したことをお知らせします。 SSDキャッシングテクノロジーを使用すると、任意のデータを高速で記録できます。これは、ロードされたデータベースを操作するユーザーにとって特に重要です。



アムステルダムでは、高負荷タスク用に最適化されたDataCenterレベルのSSDと、データ保護を強化する内蔵バッテリーを使用しています。 モスクワでは、PCI – Express SSDディスクを使用しています。これは、通常のSSDよりも大幅に高いパフォーマンスを備えています。







この記事では、Amazon M3.xlargeおよびAzure D3と比較したInfoboxCloudディスクサブシステムの実際のパフォーマンステストについて説明します。 最後に、すべての費用を計算します。



エンタープライズSSDキャッシングとは何ですか？



InfoboxCloud SSDは、ファイルシステムログの配置だけでなく、書き込みと読み取り、データチェックサムの計算、クラスターの状態の定期的な監視（プロセスの正常な動作とクラスターホストの使用を確認するためにクラスターデータの定期的な読み取りを実行）のボリュームキャッシュとして使用されます。 1つまたは別のSSDドライブまたはホストに障害が発生しても、データが失われることはありません。 Cloud Storageの分散フォールトトレラントファイルシステムでは、すべてのデータがその場で自動的に確実に複製されます。 ディスクサブシステムの負荷は、ホスト間で均等に分散されます。



クラスター全体で、データセンターレベル以上のSSDを使用しています。 一部のクラウドプロバイダーは、安価で低速なSSDをインストールし、主にマーケティング目的でSSDという用語を使用しています。 通常のデスクトップクラスのSSDとエンタープライズSSDの違いは、速度、多数のクライアントを同時に読み書きできること、サーバーの電源を切ったときに記録中にデータを失わないようにする内蔵バッテリーです。



Cloud Storageの分散型フォールトトレラントストレージシステムでは、エンタープライズSSDキャッシュとSAS15Kアレイの組み合わせを選択して、高速データ処理だけでなく、比較的低コストのデータストレージを提供しました。 以前は、ユーザーは安価なストレージを使用するか高速で使用するかを選択する必要がありました。 1つのクラウドで両方の利点を組み合わせようとしました。



実際のパフォーマンステスト



このセクションでは、SSDでクラウドを提供する競合他社と比較して、InfoboxCloudディスクサブシステムをテストしようとしました。 人気のあるテストを行って、異なる仮想マシンで実行しました。 結果はそのまま表示されますが、もちろん異なる場合があります。 ただし、さまざまなクラウドプロバイダーのディスクサブシステムのパフォーマンスの概要を提供します。 以下のテスト結果は、SSDディスクのアレイを使用して高速SSDキャッシュの速度を同等にすることができるという質問に答えることを目的としています。 より手頃な価格で、キャッシュを使用すると、プロジェクトの予算を心配することなく、ユーザーは大量のデータを操作できます。



テストでは、AmazonとWindows Azureは、エンタープライズセクタークラウドの代表として競合他社と見なされます。



テストしたすべてのクラウドの中で、InfoboxCloudのみがCPU、RAM、およびディスクの値を互いに独立して設定できるため、ユーザーは実際に必要なリソースの量を取得し、必要に応じて他のリソースを参照せずに必要なリソースを追加できます。 InfoboxCloudでのテストでは、Azure D3と同じ値が設定されました。

価格比較



テストで使用したマシンがユーザーにどれだけのコストをかけるかを計算してみましょう。 すべてのマシンについて、1か月あたり5000ギガバイトの発信トラフィックを使用すると想定しています。



Amazon m3.xlarge


「アイルランド」の地域の価格を計算します。

クラウドサーバーには、周波数が不明な4つのプロセッサコア、13ギガバイトのRAM、2つの40ギガバイトSSDドライブが含まれており、1時間あたり0.308ドルのコスト+ 1ギガバイトあたりアウトバウンドトラフィックのコストが0.090ドル（ギフトとして最初のギガバイト）。

1か月あたりのサーバーコスト= 0.308 * 24 * 31 * 52.69（2014年5月12日のドル為替レート）= 12074ルーブル。

5ギガバイトのトラフィックのコスト=（5000 * 0.090-0.090）* 52.69 = 23705ルーブル。

1か月あたりのサーバーとトラフィックを使用する総コスト= 35779ルーブル。

ただし、基本的なテクニカルサポートは、請求とサブスクリプション管理についてのみアドバイスします。 その他はすべて有料です。

Azureにはリソースの点でAzure D3に似た既製のインスタンスがなく、他のリソースとは無関係にInfoboxCloudに必要な量のリソースを設定することができないため、比較されるインスタンスのメモリは2GB異なります。 ただし、この違いはテストで目立った役割を果たしませんでした。



Windows Azure D3


クラウドサーバーには、周波数が不明な4つのプロセッサコア、14ギガバイトのRAM、および200ギガバイトのディスク領域が含まれています。 サーバー自体のコスト：Windowsを使用する場合は1時間あたり23.71ルーブル、Linuxを使用する場合は1時間あたり13.03ルーブル。 表示された価格は、2015年1月1日に施行されますが、それよりも前に価格が高くなります。 発信トラフィックのコストは、ギガバイトあたり3.0146ルーブルです（最初の5ギガバイトは無料です）。

1か月あたりのWindowsを搭載したサーバーのコスト= 23.71 * 24 * 31 = 17640ルーブル。

1か月あたりのLinuxを搭載したサーバーのコスト= 13.03 * 24 * 31 = 9694ルーブル。

5000ギガバイトのトラフィックのコスト= 5000 * 3.0146-5 * 3.0146 = 15073-15 = 15058ルーブル。

Windowsと毎月のトラフィックを持つサーバーのコスト= 17640 + 15058 = 32698ルーブル。

Linuxと毎月のトラフィックを備えたサーバーのコスト= 9694 + 15058 = 24752ルーブル。

ただし、基本的なテクニカルサポートは、請求とサブスクリプション管理についてのみアドバイスします。 その他はすべて有料です。



Infoboxcloud


テストの値を設定しました。周波数2.3ギガヘルツの4つのプロセッサコア、14ギガバイトのRAM、および200ギガバイトのディスクスペースです。 リソースのコストはページに示されています 。 実際、手動で何もカウントする必要はありません。選択した設定のクラウドコントロールパネルに、1か月間のサーバーのおおよそのコストが表示されます。 外部への発信トラフィックのコストは0.5ルーブル/ギガバイトで、最初の3テラバイトは無料です。

1か月あたりのサーバーコスト：8154.00ルーブル。

月あたりの交通費：0.5 * 2000 = 1000ルーブル。
