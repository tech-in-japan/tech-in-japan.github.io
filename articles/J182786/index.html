標準インストーラーを使用せずにRed Hat Enterprise Linuxクローンをインストールする

単一の物理サーバーと仮想ファーム全体の展開の両方で、最新のLinuxシステムを1回だけインストールする必要があります。 それにもう少し時間を費やして、必要に応じてすぐにすべてを実行し、レールに沿ってインストーラーから来たものを終了しないでください。 さらに、標準インストーラーの「最小」モードでインストールされるソフトウェアの量は、実際の最小値に対応していません。

一部の人にとっては、手動インストールは時間の無駄に思えるかもしれませんが、すべてのLinuxoidは、それがどのように機能するかをよりよく理解するために、少なくとも一度は実行する必要があります。 fdisk、grub、chrootの言葉を恐れないのであれば、読み進めてください。



オペレーティングシステムのインストールは、4つの異なる段階に分けることができます。

  1. システムストレージの準備
  2. システムコンポーネントのインストールと構成
  3. セルフブートのためのシステムの準備
  4. 作業タスクのシステム設定


最近のほとんどのシステムでは、タスク1〜3はインストーラーによって実行されます。 半自動インストーラーでは、項目1〜3は質問と回答モードで実施されます。 大部分の自動化されたもの-同様に、準備済みの応答ファイルを指定することができます。



インストーラーの作成は簡単な作業ではありません。 これはGentooプロジェクトの人たちには常に知られていましたが、 Gentooプロジェクトではこれまで起こりませんでした。ArchLinuxコミュニティでは最近、より緊急のタスクを優先してインストーラーのサポートを放棄することにしました。 Microsoftでさえこれを理解していました。Vistaから始まるWindowsインストーラーの2番目のステップは、既に組み立てられたイメージを展開することです。 他の問題では、ウィンドウのことではなく、これは別の記事のトピックです。



OSを手動でインストールすることにより、すべての段階で完全に自由に行動できます。 この記事は当然、唯一の正しい指示ではなく、「真空中の球状サーバー」をインストールするためのシナリオです。 特定のタスクと条件は、どのステップにも大きく影響します。 例として、 Microsoft Hyper-Vを実行している仮想マシンにOracle Enterprise Linux 6をインストールします 。 もちろん、最も自由な組み合わせではありませんが、それはポイントではありません。 ところで、このインストール方法では、Oracleへの登録は必要ありません。 Comrade Stallmanの純粋主義者と支持者には、KVMを使用して、ほぼ1対1のR​​HELクローンであるCentOSまたはScientificをインストールすることをお勧めします。したがって、それらのインストールに実質的な違いはありません。 OELには多くのユニークなニュアンスはありません。 カーネルUEKに加えて、 Oracle LinuxはDjango Fettのコアも提供します。DjangoFettには、Hyper-VデバイスとPnPフォワードドライブ用の準仮想fireがあります。



インストールには、原則として、LinuxまたはLiveCDが適しています。多くのプラットフォームでrpmとyumを使用できます。 ただし、RHELを使用するため、同じメジャーバージョンのRHELのようなdistrの新しいLiveCDを使用するのが最も便利です。 間違いなくrpmとyumの互換バージョンがあるため、タスクが大幅に簡素化されます。 CentOS LiveCDを選択しました。これは、 最も近いミラーからダウンロードできます。

この例では、システムストレージはVMkeに接続された10 Gbの仮想ディスクです。 ブートローダー用の小さなセクションを除くすべてのスペースをLVMに提供します。これは、Linuxプラットフォームで最も安定したボリューム管理システムだと考えています。

2.システムコンポーネントのインストールと設定



インストールされているすべてのrpmパッケージに関する情報は、特別なデータベースに保存されます。 rpmとyumの両方に対して、代替ルートfsを指定できます。これにより、最小限のコマンドで任意のディレクトリにRHELをインストールできます。 RPMDBはRHELのスパインです。新しいシステム用に作成してください。



＃rpm --root = / mnt / system –initdb




yumのインストールパッケージのソースはパッケージリポジトリであり、そのリンクは/etc/yum.repos.d/ configsにあります。 このような接続には、インターネットを介してアクセス可能なパブリックリポジトリ、ネットワークストレージからマウントされた独自のリポジトリ、または使用可能なインストールディスクのイメージがある場合はそれを使用できます。 このように：



＃yum install createrepo

＃mkdir -p / mnt / iso / {1,2,3}

＃mount -o loop/mnt/nas/install/oel-6.4-cd{1,2,3►.iso/ mnt / iso / {1,2,3}

＃createrepo / mnt / iso




CentOSをインストールする場合は、既存の.repoファイルで$ releaseverを明示的に指定するだけで十分です。 これらの変数は、新しいシステムにはまだないディストリビューションのディストリビューションパッケージから取得されるため、sedする必要があります



＃sed -i.orig 's / $ releasever / 6 / g' /etc/yum.repos.d/*.repo




OELが必要なので、yumはOracle パブリックリポジトリで構成されます

＃mv /etc/yum.repos.d{,.orig}

＃mkdir /etc/yum.repos.d

＃cd！$

＃wget public-yum.oracle.com/public-yum-ol6.repo




システムのインストールを開始できます。 リリースパッケージ、glibc、bash、およびそれらの依存関係のパッケージの完全に「初歩的な」セットから始めましょう。



＃yum --installroot = / mnt / system oraclelinux-release glibc bashをインストール




このようなシステムは、Linuxからchrootとして実行する準備がすでに整っています。 しかし、他の計画もあるので、すべての依存関係とともにrpmとyumを配置します。



＃yum --installroot = / mnt / system install rpm yum
