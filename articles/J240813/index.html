データセット


データを処理するには、まずマシンのメモリでそれらを表現する方法を見つける必要があります。 たとえば、数字のコレクションを覚えておく必要があります。 



2, 3, 5, 7, 11

幸いなことに、JavaScriptは、数値のシーケンスを保存するためのデータ型を提供します。 これは配列と呼ばれ、カンマで区切られた角括弧内の値のリストとして記述されます。 



 var listOfNumbers = [2, 3, 5, 7, 11]; console.log(listOfNumbers[1]); // → 3 console.log(listOfNumbers[1 - 1]); // → 2




配列から要素を取得するレコードも角括弧を使用します。 内部に別の式を含む式の後の括弧のペアは、最初の式で指定された配列、2番目の式でシリアル番号が与えられた要素を検索します。



最初の要素の数は1ではなくゼロです。 したがって、最初の要素はlistOfNumbers [0]のように取得できます。 以前にプログラムしたことがない場合は、そのような番号付けに慣れる必要があります。 しかし、それは長い伝統があり、常に観察されている間、それは素晴らしい作品です。



プロパティ


前の例では、myString.length（文字列の長さを取得）やMath.max（最大値を取得）などの疑わしい式が多数見られました。 これらの式は数量のプロパティを使用します。 最初のケースでは、myString変数のlengthプロパティにアクセスできます。 2つ目は、Mathオブジェクト（数学に関連する関数と変数のセット）のmaxプロパティへのアクセスです。



JavaScriptのほとんどすべての変数にはプロパティがあります。 例外はヌルで未定義です。 これらの非数量の存在しないプロパティにアクセスしようとすると、エラーが発生します。 



 null.length; // → TypeError: Cannot read property 'length' of null




プロパティにアクセスする2つの主な方法は、ドットと角括弧です。 value.xとvalue [x]はvalueプロパティにアクセスしますが、必ずしも同じものではありません。 違いはxの解釈方法です。 ポイントを使用する場合、ポイントの後のレコードは既存の変数の名前である必要があるため、名前でプロパティを直接呼び出します。 角括弧を使用すると、括弧内の式が評価されてプロパティの名前が取得されます。 value.xは「x」というプロパティを呼び出し、値[x]は式xを評価し、その結果をプロパティの名前として使用します。



関心のあるプロパティが「長さ」と呼ばれることがわかっている場合は、value.lengthを記述します。 変数iからプロパティ名を抽出する場合は、値[i]を記述します。 また、プロパティには任意の名前を付けることができるため、「2」または「Jon Doe」という名前でプロパティにアクセスするには、角括弧を使用する必要があります。値[2]または値[「John Doe」]。 これは、プロパティの正確な名前がわかっている場合でも必要です。「2」または「John Doe」は有効な変数名ではないため、ドット表記を使用してアクセスできないためです。



配列要素はプロパティに保存されます。 これらのプロパティの名前は数値であり、変数の値から名前を取得する必要があることが多いため、角括弧を使用してアクセスする必要があります。 配列のlengthプロパティは、その中にある要素の数を示します。 このプロパティの名前は有効な変数名であり、事前に知っているため、配列["length"]を記述するよりも簡単なので、通常array.lengthを記述します。



方法

 var doh = ""; console.log(typeof doh.toUpperCase); // → function console.log(doh.toUpperCase()); // →




各行にはtoUpperCaseプロパティがあります。 呼び出されると、すべての文字が大文字に置き換えられた文字列のコピーを返します。 toLowerCaseもあります-それが何をするか推測できます。



おもしろいことに、UpperCaseの呼び出しは引数を渡しませんが、関数は何らかの方法で呼び出した「Duc」行にアクセスします。 この仕組みについては、第6章で説明しています。



関数を含むプロパティは、通常、それらが属する変数のメソッドと呼ばれます。 つまり、toUpperCaseは文字列メソッドです。



次の例は、配列で使用可能なメソッドの一部を示しています。

オブジェクト


リスに戻りましょう。 一連のジャーナルエントリは配列として表すことができます。 ただし、レコードは数字や行だけで構成されているわけではありません。それぞれがヒーローの行動のリストと、ジャックがリスに変わったかどうかを示すブール値を保持する必要があります。 理想的には、各レコードを1つの変数にグループ化し、それらを配列に追加します。



タイプオブジェクト（オブジェクト）の変数-任意のプロパティのコレクション。必要に応じてオブジェクトのプロパティを追加および削除できます。 オブジェクトを作成する1つの方法は、中括弧を使用することです。 



 var day1 = { squirrel: false, events: ["", " ", "", "", ""] }; console.log(day1.squirrel); // → false console.log(day1.wolf); // → undefined day1.wolf = false; console.log(day1.wolf); // → false




括弧内に、コンマで区切られたプロパティのリストを指定できます。 各プロパティは名前として記述され、その後にコロンが続き、プロパティの値である式が続きます。 スペースと改行は考慮されません。 オブジェクトプロパティのレコードを複数の行に分割することにより、コードの可読性が向上します。 プロパティ名が有効な変数名でない場合は、引用符で囲む必要があります。 



 var descriptions = { work: "  ", " ": "  " };




JavaScriptの中括弧には2つの意味があることがわかります。 命令の先頭で使用され、命令の新しいブロックを開始します。 他の場所では、オブジェクトを記述します。 通常、オブジェクトの説明から命令を開始することは意味をなさないため、プログラムでは、通常、これら2つのブレースの使用についてあいまいさはありません。



存在しないプロパティの値を読み取ろうとすると、最初にwolfプロパティを読み取ろうとした例のように、未定義になります。



=演算子を使用して、プロパティに値を割り当てることができます。 以前に値があった場合は、置き換えられます。 プロパティが欠落している場合、作成されます。



触手と変数を使用してモデルに戻ると、プロパティもそれらに似ていることがわかります。 値を取得しますが、他の変数とプロパティは同じ値を参照する場合があります。 オブジェクトは、任意の数の触手を持つタコであり、それぞれにプロパティの名前が書き込まれます。





削除オペレーターは、触手を切断します。 これは、プロパティアクセス式に適用される単項演算子です。 これはめったに行われませんが、かなり可能です。 



 var anObject = {left: 1, right: 2}; console.log(anObject.left); // → 1 delete anObject.left; console.log(anObject.left); // → undefined console.log("left" in anObject); // → false console.log("right" in anObject); // → true




バイナリ入力演算子は、文字列とオブジェクトの名前を受け取り、オブジェクトにその名前のプロパティがあるかどうかを示すブール値を返します。 プロパティ値を未定義に設定することと、プロパティを削除することには違いがあります。 最初の場合、プロパティはオブジェクトに保存され、空になります。 2番目では、プロパティはもうないので、inはfalseを返します。



配列は、シーケンスの格納に特化した一種のオブジェクトであることがわかります。 式typeof [1、2]は、「オブジェクト」を返します。 彼らはすべての触手が均等に配置され、数字でラベル付けされた長い平らなタコと考えることができます。

可変性




すぐにプログラミングに取り掛かります。 それまでの間、理論の最後の部分を理解する必要があります。



オブジェクトの値は変更できることがわかりました。 前に調べた値の種類（数値、文字列、ブール値）は不変です。 特定のタイプの既存の値を変更することはできません。 それらを組み合わせて新しい値を派生させることはできますが、文字列値を操作する場合、この値は一定のままです。 行内のテキストは変更できません。 「cat」行へのリンクがある場合は、コード内の文字を変更して「midge」を作成することはできません。



ただし、オブジェクトの場合、プロパティの値を変更することでコンテンツを変更できます。



120と120の2つの数値がある場合、それらが同じ場所のメモリに格納されているかどうかに関係なく、それらを1つの同じと見なすことができます。 しかし、オブジェクトを扱う場合、同じオブジェクトへの2つの参照があるか、同じプロパティを含む2つの異なるオブジェクトがあるかによって違いがあります。 例を考えてみましょう： 



 var object1 = {value: 10}; var object2 = object1; var object3 = {value: 10}; console.log(object1 == object2); // → true console.log(object1 == object3); // → false object1.value = 15; console.log(object2.value); // → 15 console.log(object3.value); // → 10




変数object1とobject2は同じオブジェクトを保持するため、object1を変更するとobject2が変更されます。 変数object3は、最初はobject1と同じプロパティを含む別のオブジェクトを指しますが、それ自体の寿命を持ちます。



オブジェクトを比較するとき、比較されるオブジェクトが同じ変数である場合にのみ==演算子はtrueを返します。 異なるオブジェクトを比較すると、内容が同じであってもfalseが返されます。 オブジェクトの内容を比較する「深い」比較演算子はJavaScriptでは提供されていませんが、独立して実行できます（これは章の最後の演習の1つです）。

マップとしてのオブジェクト


1つの方法は、名前と値のプロパティを持つオブジェクトを使用して、配列に相関を保存することです。 ただし、配列内の相関の検索はかなり面倒です。目的の名前のオブジェクトを見つけるには、配列全体を調べる必要があります。 このプロセスを関数でラップすることもできますが、とにかくコードを記述する必要があり、コンピューターは必要以上の作業をします。



より良い方法は、オブジェクトのプロパティをイベント名とともに使用することです。 角括弧を使用してプロパティを作成および読み取り、in演算子を使用してプロパティの存在を確認できます。 



 var map = {}; function storePhi(event, phi) { map[event] = phi; } storePhi("", 0.069); storePhi(" ", -0.081); console.log("" in map); // → true console.log(map[" "]); // → -0.081




マップは、あるエリアの値（この場合はイベントの名前）を別のエリアの値（この場合は係数ϕ）に関連付ける方法です。



このオブジェクトの使用にはいくつかの問題があります。それらについては第6章で説明しますが、現時点では心配する必要はありません。
