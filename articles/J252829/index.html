Windows Phone 8アプリケーションの効果音





ほとんどのアプリケーションでは効果音を再生する必要はありませんが、効果音なしではできない場合があります。 次に、Windows Phone 8アプリケーションで効果音を再現する方法についての論理的な疑問が生じます。



Windows Phone Silverlightのドキュメントを参照すると、次のWindows Phoneのメディアとサウンド効果の再生に関する記事を参照できます。 ドキュメントの内容に基づいて、アプリケーションでエフェクトを再現する方法はMediaElementまたはXNAを使用する2つの方法しかないという結論に達することができます。 これらの各方法をより詳細に検討してください。



MediaElementを使用して効果音を再生する



サウンドエフェクトを再生する最も簡単で最も「ネイティブ」な方法は、MediaElementコントロールを使用することです。 このコントロールは、オーディオおよびビデオコンテンツを再生するための十分な機会を提供し、目的に使用できます。



コントロールを使用するには、現在のページのビジュアルツリーに追加する必要があります。



これを行うには、XAMLページに次のコードを追加します。 



<MediaElement x:Name="viewMediaElement" AutoPlay="True" />


または、次のコードを使用して、cedebehindページからコントロールを作成します。 



 private MediaElement _mediaElement; private void CreateMediaElement() { _mediaElement = new MediaElement { AutoPlay = true }; LayoutRoot.Children.Add(_mediaElement); }


コードから、コントロールにAutoPlayプロパティが設定されていることがわかります。これにより、コントロールは転送されたコンテンツの即時再生に切り替わります。



サウンドファイルを再生するには、そのアドレスをコントロールのSourceプロパティに渡す必要があります。 



 private void PlaySoundClick(object sender, RoutedEventArgs e) { viewMediaElement.Source = new Uri("/Assets/sound.wav", UriKind.Relative); }


XAMLデータバインディングを使用してSourceプロパティの値を設定すると、分離コードコンテンツが完全に排除されます。



この効果音再生方法を使用する利点：


  • 実装と使用のしやすさ
  • サポートされているオペレーティングシステム形式のファイルコンテンツを再生する


短所：


  • コントロールは、アプリケーションのビジュアルツリーに追加する必要があります。
  • この方法では、一度に1つのサウンドしか再生できません。ドキュメントには、1つのコントロールしか許可されていないことが示されています。
  • サウンドの再生はユーザーインターフェイスに関連付けられており、UIストリームとの同期が必要です。
  • シームレスループ再生を手配できません。


XNAを使用して効果音を再生する



サウンドエフェクトを再生するには、XNAライブラリを使用できます。 このライブラリは、ゲームプロジェクトの開発時にXNAインフラストラクチャのフレームワーク内で動作するように開発されたという事実にもかかわらず、Windows Phoneアプリケーションで使用してサウンドを再生できます。



これを行うには非常に簡単です： 



 private void PlaySoundClick(object sender, RoutedEventArgs e) { var stream = TitleContainer.OpenStream("Assets/sound.wav"); var soundEffect = SoundEffect.FromStream(stream); var instance = soundEffect.CreateInstance(); FrameworkDispatcher.Update(); instance.Play(); }


2つの興味深い点に注意してください。 サウンドエフェクトのCreateInstanceメソッドを呼び出す必要はありません。SoundEffectインスタンスでPlay（）メソッドを呼び出すだけで再生できますが、エフェクトの明示的に作成されたインスタンスのみが、エフェクトを完全に制御して微調整できます。 この点を知らなかったため、WP SLプログラマーは、再生開始後にSoundEffectを制御できないと誤解するようになりました。



XNAライブラリを正常に動作させるために必要なXNAゲームサイクルをエミュレートするには、FrameworkDispatcher.Update（）メソッドの呼び出しが必要です。



XNA SoundEffectを使用してサウンドを再生する利点：


  • 使いやすさ。
  • 音量、速度、繰り返しなどの効果音の高度な設定


短所：


  • マイクロソフトによってサポートされなくなった非推奨のテクノロジー。
  • 大量のメモリを消費するPCM形式のWAVファイルのみを再生します。
  • ゲームサイクルエミュレーションの形での追加操作の必要性。


XAudio2を使用して効果音を再生する



残念ながら、WP 8アプリケーションのサウンドエフェクトをすぐに再生する他の方法はありません。 ただし、ネイティブアプリケーションの場合、低レベルのXAudio2サウンドエフェクトテクノロジが利用可能です。 この技術は、ゲームで効果音を再現する技術として位置付けられていますが、Windows Phone 8アプリケーションで効果音を再生するために使用できます。



WPアプリケーションからXAudio2を直接使用することはできないためです。 WP Silverlightで効果音を再生できるようにするシンプルなWindowsランタイムコンポーネントを開発します。



Windows Phone 8オペレーティングシステム用のWindowsランタイムコンポーネントは、C ++言語のみを使用して開発されています。



新しいWindowsランタイムコンポーネントプロジェクトを追加します（Windows Phone 8.0）











プロジェクトを正しくアセンブリするには、xaudio2.libライブラリを使用する必要があることをリンカーに示す必要があります。 これを行うには、リンカー->入力タブでプロジェクトのプロパティを開き、xaudio2.libライブラリを追加の依存関係リストに追加します











作成されたコンポーネントのクラスを定義します。 



 public ref class SoundEffect sealed { private: std::shared_ptr<WaveData> _waveData; Microsoft::WRL::ComPtr<IXAudio2> _audioEngine; IXAudio2MasteringVoice* _masteringVoice; IXAudio2SourceVoice* _sourceVoice; public: SoundEffect(const Array<byte>^ source); void Play(); void Stop(); };




クラスコンストラクターは、単一のパラメーター、PCMまたはADPCM形式のオーディオファイルの内容を含むバイトの配列を受け入れます。 サウンド効果を制御するには、PlayおよびStopメソッドがあります。 Playメソッドが呼び出されると、エフェクトは最初から再生され、Stopが呼び出されると、エフェクトは再生を停止します。



クラスフィールドは、XAudio2コンポーネントへの参照とWaveDataサウンドファイルの説明を格納するためのものです。 IXAudio2コンポーネントはCOMオブジェクトであるため、Microsoft :: WRL :: ComPrtコンテナーを使用して格納します。これにより、リンクカウントメカニズムが正しく動作し、オブジェクトが解放されます。 XAudio2オブジェクトへの残りのポインターはComPtrを使用せず、その存続期間の参照カウントはIXAudio2オブジェクトによって制御されます。



XAudio2の初期化 



 SoundEffect::SoundEffect(const Array<byte>^ source) { _waveData = std::make_shared<WaveData>(source); auto hr = XAudio2Create(&_audioEngine); if (FAILED(hr)) { throw ref new Platform::Exception(hr); } hr = _audioEngine->CreateMasteringVoice(&_masteringVoice); if (FAILED(hr)) { throw ref new Platform::Exception(hr); } hr = _audioEngine->CreateSourceVoice(&_sourceVoice, _waveData->Format); if (FAILED(hr)) { throw ref new Platform::Exception(hr); } hr = _audioEngine->StartEngine(); if (FAILED(hr)) { throw ref new Platform::Exception(hr); } }


WaveDataクラスについて以下で説明しますが、wav形式の音声ファイルから音声データ形式および音声データに関する情報を抽出することを知っているだけで十分です。



最初の手順は、XAudio2Create関数を呼び出して、IXAudio2インターフェイスを実装するオブジェクトへのリンクを取得することです。



次のステップは、IXAudioMasteringVoiceインターフェイスを実装するオブジェクトを作成することです。このオブジェクトは、元の音声からのデータを単一のオーディオストリームに結合する役割を果たします。 実際、このオブジェクトはサウンドミキサーです。



CreateSourceVoiceメソッドを呼び出して、効果音ファイルからサウンドストリームを表す元の音声を作成します。 元の音声を作成する場合、再生のために音声を転送する音声データの形式を指定する必要があります。



XAudio2 VoicesでXAudio2ボイスの詳細をご覧ください。



XAudio2を初期化する最後の手順は、メソッドStartEngineを呼び出すことです。このメソッドは、ライブラリのコアでサウンドエフェクトの処理を開始します。



エフェクトを開始および終了する方法 



 void SoundEffect::Play() { Stop(); XAUDIO2_BUFFER playBuffer = { 0 }; playBuffer.AudioBytes = _waveData->Length; playBuffer.pAudioData = _waveData->Data; playBuffer.Flags = XAUDIO2_END_OF_STREAM; auto hr = _sourceVoice->SubmitSourceBuffer(&playBuffer); if (SUCCEEDED(hr)) { hr = _sourceVoice->Start(0, XAUDIO2_COMMIT_NOW); } } void SoundEffect::Stop() { auto hr = _sourceVoice->Stop(); if (SUCCEEDED(hr)) { _sourceVoice->FlushSourceBuffers(); } }


再生を開始する前に、XAUDIO2_BUFER構造体にサウンドファイルデータを入力し、元の音声のデータソースとしてSubmitSourceBufferメソッドを使用して設定する必要があります。



最後のステップは、Startメソッドを使用して、再生用の音声を起動することです。



実際、音声データソースを1回設定し、Playメソッドを使用してのみ再生を開始できますが、誤って実行されたStopメソッドの状況に遭遇し、その結果、再生を開始しようとするとエラーが発生しました。 各プレイでソースバッファを設定することで、この問題を解決しました。



オーディオファイルの準備とダウンロード



WP 8のXAudio2ライブラリは、PCMおよびADPCM形式のオーディオデータを再生できます。 ADPCMファイルを準備するには、adpcmencode.exeユーティリティを使用する必要があります;この使用により、受信したファイルとXAudio2ライブラリとの互換性が保証されます。



実際、サードパーティのユーティリティを使用してADPCM形式の単一のファイルを準備することはできませんでした。それらのすべては、要件に正式に準拠しているにもかかわらず、互換性がないことが判明しました。



2つの必須コンポーネントをwavサウンドファイルから抽出する必要があります。オーディオデータ形式と、PCMまたはADPCM形式のオーディオストリームデータです。 これらのコンポーネントを取得するには、wavファイルを解析する必要があります。



wavファイルはRiffコンテナです。詳細については、 WIKI RIFF 、 Resource Interchange File Format Services 、 ADPCM RIFFを参照してください 。



WavDataクラスは、wavファイルからデータを抽出し、その後に保存する役割を果たします。 



 private struct WaveData { private: const Array<byte>^ _buffer; ChunkInfo FindChunk(const Array<byte>^ buffer, const RiffType& chunkType) const; void Read(const Array<byte>^ buffer); public: WaveData(const Array<byte>^ buffer); const WAVEFORMATEX* Format; const byte* Data; uint32 Length; };




ファイルを解析するとき、クラスは以下をチェックします：



  1. ファイルには波データが含まれています
  2. ファイルにfrm（フォーマット）部分が含まれており、そのサイズがWAVEFORMATEX構造体のサイズより小さくないこと
  3. PCMまたはADPCMオーディオデータ形式
  4. ファイルにデータ部分が含まれています


パラグラフ2および4で取得したデータへのポインターは、将来の使用のために保存されます。



生成されたコンポーネントを使用する



作成したコンポーネントをWP Silverlightアプリケーションに接続し、それを使用して効果音を再生します。 



 private async void PlaySoundClick(object sender, RoutedEventArgs e) { var soundEffect = await CreateSoundEffectFromFile("ms-appx:///Assets/sound.wav"); soundEffect.Play(); } private async Task<XAudio2SoundEffect.SoundEffect> CreateSoundEffectFromFile(string fileUri) { if (string.IsNullOrWhiteSpace(fileUri)) { return null; } try { var file = await StorageFile.GetFileFromApplicationUriAsync(new Uri(fileUri)); var buffer = await ReadFile(file); return new XAudio2SoundEffect.SoundEffect(buffer); } catch (Exception ex) { return null; } } private async Task<byte[]> ReadFile(StorageFile file) { using (var stream = await file.OpenReadAsync()) using (var reader = new DataReader(stream)) { var buffer = new byte[stream.Size]; await reader.LoadAsync((uint)stream.Size); reader.ReadBytes(buffer); return buffer; } }

XAudio2ライブラリを使用してサウンドエフェクトを再生するコンポーネントを作成したら、廃止されたXNAライブラリを取り除き、MediaElementに固有の欠点を取り除きました。



作成されたコンポーネントは、最も単純な音響効果制御機能を実装します。 開発者は、ボリュームコントロール、再生速度コントロール、ループ再生、効果音など、XAudio2が提供するすべての機能を利用できます。
