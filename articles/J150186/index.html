クラス/メソッド/変数を宣言するためのルール。


  • ソースファイルは、このファイル内の唯一のパブリッククラスと同じ名前である必要があります。
  • 各ファイルは1つのパッケージにのみ属することができますが、複数のクラスをインポートできます
  • packageキーワードは常にimportキーワードよりも高く、importキーワードは常にクラス宣言よりも高くなります。
  • パッケージとインポートは、アクセスレベル/名前などに関係なく、ファイル内のすべてのクラスに適用されます。
  • 非パブリッククラスは、識別子の作成規則に矛盾しない限り、ファイル名に関連付けられていない任意の名前を持つことができます。




クラスアクセス修飾子。


  • アクセス修飾子には、パブリック、保護、プライベートの3つがあります。 そして、最大4つのアクセスレベル ：パブリック、保護、デフォルト、プライベート。 デフォルトのアクセスレベルは、修飾子がないことで決まります。
  • 外部クラスの場合、使用できるアクセスレベルは、パブリックとデフォルトの2つだけです。 これは論理的です。なぜなら、ファイルの外部にあるプライベートクラスは見ることができず、保護されるのはメソッドに対してのみ意味があるからです。
  • パブリッククラスはどのパッケージからもアクセスできます。デフォルトクラスは、それらが配置されている同じパッケージからのみ使用できます。
  • クラスAがクラスBで利用可能な場合、クラスBは次のことができます。
    1. クラスAオブジェクトを作成する 
       A a = new A();
    2. extendsキーワードを介してクラスAを継承する 
       class B extends A { }








クラスの他の修飾子。


  • 修飾子final（クラスの継承を禁止する）、abstract（クラスを抽象化する）、strictfp（クラスは非整数と特別な方法で動作する）をクラスに適用できます。 
     final strictfp class FinalClass{ } class RockIsDead extends FinalClass{ //   }


  • クラスを同時にfinalとabstractにすることはできません（これらは相互に排他的な修飾子です）が、strictfpはfinalとabstractを組み合わせて使用​​できます。
  • 抽象クラスのオブジェクトを作成することはできません。
  • 抽象クラスは実装されたメソッドを持つことができますが、抽象クラスは抽象メソッドを持つことはできません 

     abstract class Sweet { void dreams() { //  System.out.println("Ummm"); } abstract abstractDreams(); //   } class Fail { abstract void anotherFail(); //  }
  • 抽象クラスを継承する非抽象クラスは、このクラスのすべての抽象メソッドを実装する必要があります。




インターフェース


  • インターフェイスは100％の抽象クラスであり、非抽象メソッドを含めることはできません。
  • インターフェイスはクラスに何をすべきかを伝えますが、 どのように行うかは伝えません。
  • インターフェイスはどのクラスでも実装できます。
  • インターフェイスメソッドには、デフォルトでパブリックおよび抽象アクセス修飾子があります。
  • インターフェイスには、常にpublic static final修飾子を持つ変数を含めることができます。 これらは省略することができ、インターフェースの変数は常に実際に静的定数にするパラメーターで作成されます。
  • インターフェイスを実装する非抽象クラスは、次の要件を満たす必要があります。
    1. すべてのインターフェイスメソッドを実装します。
    2. インターフェイスで宣言されていない例外をスローしないでください。
    3. 同じシグネチャを使用して、すべてのインターフェイスメソッドを実装します。
  • インターフェイスを実装するクラスは抽象的です。 この場合、任意の数のインターフェイスメソッドを実装できます。
  • クラスは1つのクラスしか継承できませんが、多くのインターフェイスを実装します。 インターフェイスは多くのインターフェイスを継承できますが、抽象または通常のクラスは継承できません。




クラスフィールドの修飾子。


  • クラスフィールドには、4つのアクセスレベルすべてを含めることができます。
  • 1つのクラスは2つの方法で別のクラスのフィールドにアクセスできます：継承を介して、またはオブジェクトの作成を介して。
  • クラスAがクラスBで利用できない場合、クラスAのメソッドはクラスBで利用できません。
  • パブリックフィールドは他のクラスからアクセスでき、継承を通じて保護されたクラスはすべてのクラスで使用でき、オブジェクト作成では同じパッケージのクラスでのみ使用できます。 デフォルトフィールドは1つのパッケージからのみアクセス可能で、プライベートフィールドは自分のクラス以外のクラスからは見えません。
  • ドットを使用しないクラスのフィールドへのアクセスは、クラス自体からのみ可能です。




ローカル変数


  • ローカル変数にはアクセス修飾子がありません。

  • ローカル変数に使用できる唯一の修飾子はfinalです。
  • ローカル変数は、クラスフィールドとは異なり、コンパイル時に初期化されません。手動で行う必要があります




修飾子に関するその他の情報。


  • finalメソッドはオーバーライドできません。
  • 抽象メソッドは、メソッド本体を除いて、従来のメソッドと同じすべてのバンで宣言されます（空の中括弧さえありません）。
  • synchronized修飾子は、メソッドまたはコードブロックに対してのみ使用されます。 
     synchronized void hulkLomat(){ } void hulkStroit(){ synchronized (this){ //   } }
  • ネイティブ修飾子はメソッドにのみ使用されます。
  • strictfp修飾子は、クラスまたはメソッドにのみ使用されます。

乗り換え


  • 列挙を特別なクラスと考えてください
  • 列挙の定数型は常に列挙名と一致します。
  • 列挙はクラスの内部または外部で宣言できますが、メソッドの内部では宣言できません。
  • enumがクラスの外部で宣言されている場合、プライベートアクセス修飾子を持つことはできません。
  • 列挙には、コンストラクタ、メソッド、変数、オブジェクトコンストラクタを含めることができます
  • SomeEnumName.Small（2）は、2のint値をSmallオブジェクトのコンストラクターに渡します
  • リストが宣言された後、セミコロンはオプションです。
  • 列挙コンストラクターを直接呼び出すことはできません。




静的変数とメソッド。


  • 静的メソッドは、非静的変数にアクセスできません。
  • 静的フィールドまたはメソッドにアクセスするには、クラスのインスタンスは必要ありません。




配列


  • 配列はプリミティブ型またはオブジェクトを格納できますが、それら自体は常にオブジェクトです。
  • 次のような配列を宣言できます：int [] array、または次のように宣言できます：int array []。
  • 作成時には、常に配列のサイズを明示的に指定する必要があります。




その他


  • クラスフィールドは、finalまたはtransientです（transient修飾子は、オブジェクトをシリアル化する必要がないことを示します）。
  • クラス名だけでなくローカル変数にも名前を付けることができます。
  • 最後の修飾子は、オブジェクトの不変性を意味するのではなく、オブジェクトへの参照の不変性を意味します。 つまり、最終的なArrayList list = new ArrayList（）;を宣言することにより、 リスト変数によって参照されるオブジェクトの状態を変更できますが、作成されたオブジェクトを参照しないようリストに強制することはできません。
  • volatile修飾子を使用すると、さまざまなスレッドが変数の現在の状態を取得できます。
