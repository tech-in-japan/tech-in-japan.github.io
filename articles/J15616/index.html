Ubuntuとハードドライブの損傷





今日、 不快なニュースを学びました。UbuntuLinuxは、電源デーモンスクリプトの不正な操作により、ラップトップのハードドライブを絶えず停止したり起動したりすることで損害を被る可能性があります。



ウィンチェスターにとってこのリソースは非常に重要であり、現代のモデルにとっては約600,000回です。 負荷サイクルは、ハードドライブのスピンドルの停止加速ごとに、またはヘッドの駐車/駐車解除中に1つずつ増加します。



smartmontoolsをインストールし 、 sudo smartctl -a / dev / sdaを実行する| grep Load_Cycle私はちょうどぞっとしました。 47597 Load_Cyclesは、ラップトップの作業時間がそれほど長くないことを示します。 さらに、私はこれについて友人に話していましたが、この値は47626に増加しました。



彼女は私を殺します！ リンクをクリックして情報を読んだ後、解決策が見つかりました-ハードドライブで高度な電源管理を無効にする必要があります。 より詳細な情報は、 この投稿の 対応するトピック ubuntuforums.orgで提供されています。



自宅では、この投稿でubuntu_demonが示唆したように検証を行いました

aac @ aac-laptop：〜$ sudo hdparm -B 255 / dev / sda



/ dev / sda：

高度な電源管理レベルを無効に設定

aac @ aac-laptop：〜$ sudo hdparm -S 0 / dev / sda



/ dev / sda：

スタンバイを0（オフ）に設定する





その後、Load_Cycle_Countは停止し、1時間安定しましたが、その前に1〜5分で1増加しました（ラップトップがネットワークに接続されており、電源が入っているため、とにかく問題が観察されます）。 ネジが保存され......

ps一部の構成では、パラメーターを255ではなく254で渡す必要があります。 sudo hdparm -B 255 / dev / sda 。



通常、問題は解決されます。 しかし、以前はUbuntuが大好きでした。Linuxはシステムの信頼性と安定性を高めるように設計されていると思いました。 はい、LinuxはUbuntuであるだけでなく、多くの人々がこのディストリビューションへの愛を共有しています（そして、何のためでもありません）。

今、私は自分のオペレーティングシステムを信頼していません。この投稿を印刷するとき、私はWindowsのそれとどう違うのでしょうか。



追加しました



判明したように、問題はLinuxがハードウェアに対して正直すぎて、BIOSまたはファームウェアで設定された設定に従うということでした。 「不正な」WindowsおよびMacOSはこれらの設定を無視し、独自の設定を使用します。 後者は間違っているようです。



ただし、私の記憶では、デバイスに書き込まれた内容（「標準」と読みます）をたどるとシステムが正しく動作しなくなるのは初めてではありません。 1台の安価なモニターでも同様の状況がありました。多くの苦労の末、 DDCの情報の読み取りをオフにし、科学的列挙の方法を使用してそれに適したモードラインを見つける必要がありました。 不正なウィンドウでは、このモードラインは長い間ドライバーに登録されています。



このような問題は、魅力的なLinuxオペレーティングシステムの大衆への普及を著しく妨げているように思えます。 今後、この種の問題に対処する必要がなくなることを願っています。
