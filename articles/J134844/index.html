Amazon Kindle FireタブレットでのROOT権限の取得（脱獄）

あなたはちょうど（またはずっと前に）Fireを開梱し、Amazonインターフェースの機能が非常に貧弱であり、Amazonマーケットではタンバリンと踊らないと何も買えないこと、そしてもしそうなら、それはあなたが望むものではないことに気付きました。



アンドロイドのルート権とは何ですか、多くの人が知っています。 ルート権限という用語は、Androidカーネルの根底にあるLinuxの概念の1つです。 ルートは、メイン管理者またはスーパーユーザーのアカウントです。 このプロファイルにアクセスすると、通常の操作では利用できない多くの機能が表示されます。



長所と短所は、 この記事で見つけることができます。



そして、Kindle Fireのルート権限を直接取得します。 基本的に、以下はWindowsユーザーに適用されます。



あなた自身の危険とリスクで実行するすべての操作！ 著者は、この手順によって損傷した機器には責任を負いません！



開始するには、Fireを完全に充電し、コンピューターに接続するためにストアでMicro-USBケーブルを購入します。



developer.android.com/sdk/index.htmlから公式SDK（たとえばinstaller_r16-windows.exe）をダウンロードしてインストールします。



エクスプローラーを開き、アドレスバーに書き込みます 



%USERPROFILE%\.android




メモ帳でadb_usb.iniファイルを開き、最初の唯一の行に書き込みます 



 0×1949




ファイルを保存して閉じます。



\ Android \ extras \ google \ usb_driver \フォルダーに移動し、 android_winusb.infファイルを見つけます。 このandroid_winusb.infに置き換えます



SuperOneClickv2.2アーカイブをダウンロードして解凍します。 SuperOneClickプログラムを起動し、その中の[ ルート ]ボタンをクリックします。 すべての質問に積極的に答えてください。



市場インストール。



Googleアプリケーションパッケージをダウンロードしてc：に解凍します。



コマンドラインを実行し、adbユーティリティを使用してディレクトリに移動します（通常\ Android \ platform-tools \ ）



コマンドを実行します。 



 adb remount adb push c:\market\GoogleFeedback.apk /system/app/GoogleFeedback.apk adb push c:\market\GoogleServicesFramework.apk /system/app/GoogleServicesFramework.apk adb push c:\market\Vending.apk /system/app/Vending.apk adb reboot




デバイスを再起動すると、通常のマーケットアイコンが表示されます。
