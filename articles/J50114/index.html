Pardus 2008.1

トルコは海、暑さ、ヤシの木、厄介な地元のテキーラだけではありません。 独自のLinux OSディストリビューションを開発している州（もちろん、トルコ）の電子暗号化研究所（UEKAE）があります。 私は興味を持ち、彼をよく見ることにしました。 私はあなたを保証する勇気があります、見るべきものがあります。



インストール-YALI

これはグラフィカルインストールプログラムです。 すべて黄色できれい。 Ubuntuがインストールされているかのように、Linuxを初めて見る場合でもすべてが簡単です。 パッケージのコピーを開始する前に、ユーザーを作成し、rootパスワードを取得することをお勧めします。 インストール中、Pardusに含まれるトルコ人の独自の開発、およびKopete、GIMP、Firefoxなどのディストリビューションに含まれる他のパッケージを宣伝します。 通常のインストーラーとは異なり、最初はバンドル内の必要なすべてのパッケージをハードドライブにコピーしてから、それらすべてを順番に構成します。





ヤリ



まず、再起動後、基本的なシステム構成ウィザードであるデスクトップカプタンが表示されます。 外観、ネットワーク、リポジトリの追加などをすばやく設定できます。つまり、通常はほとんどのユーザーの基本的なシステム設定で終わります。 システム全体の微調整は、Tasma-ローカル管理センターで行うことができます。 残念ながら、デフォルトではロシア語はありませんが、インストール中に言語を選択すると、ロシア語が存在します（ただし、個別にインストールできます）。

ネットワークはNetworkManagerを介して設定および管理されます。 これはUbuntuやFedoraのものではなく、独自のPardusianです。 さらに、ファイアウォールはここにあります-ファイアウォールはトルコのローカル流出でもあります。 極端な条件でネットワークアプリケーションのパフォーマンスをテストするつもりはありませんでしたが、通常はすべてが正常に機能します。



パッケージマネージャーはPiSi（トルコ語。「KiSa」）と呼ばれます。 彼には、同じなめらかなインターフェースと「バスケット」と呼ばれるGUIクライアントがあります。これは、個別に表示および編集できる更新、削除、またはインストールされたパッケージのリストです。 また、 pisi install package-nameまたはpisi emerge package-nameのように、コンソールを介して、完全に人間の言語でPiSiを操作することもできます 。





パッケージマネージャー、PiSi。 ああ、画面全体が収まらなかった



子猫がトルコから更新をダウンロードしている間に、/ etcフォルダーのツアーに登りました。 まず、/ etc / networkフォルダーにはゴミがありますが、これはすべてのネットワーク接続がNetworkManagerを介して編集されているためであり、インターフェースがありません:)さらに、/ etc / modprobeフォルダーがあります。 私の知る限り、modprobeはカーネルモジュールを追加または削除するために使用されます（トルコ人が次の子孫をまったく同じように呼び出していない限り）。 modprobeの横にある/ etc / mudurフォルダーは、OSのロードを高速化するローカルの新興アナログです。 サイトwww.pardus.org.tr自体に書かれているように 、mudurはpardus専用に書かれており、カーネルには特定の何かが必要であるため、他のディストリビューションで使用する準備ができていません。



ここで、ところで、すべてが終わって、ロシア語はTasmを介してインストールされました。
