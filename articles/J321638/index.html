Visual Studio Team Servicesの継続的な統合UWPアプリケーション









VSTSを使用すると、さまざまな環境でソフトウェアの展開とテストを自動化できます。 継続的インテグレーションの本質は、プロジェクトの頻繁な自動アセンブリを実行して、インテグレーションの問題を迅速に特定して解決することです。 特に、CIを使用すると、アプリケーションの回帰テストを自動化できます。



VSTSの機能の概要として、単純なUWPアプリケーションの単体テストとの継続的統合を公開および構成することを提案します。



簡単なUWPアプリケーションプロジェクトを作成して、1つのメソッドを作成します。 



public double CalcSquere(string t) { double res = Convert.ToDouble(t); res = res * res; return res; }


ソリューションで、単体テスト（Unit Test App）を含むプロジェクトを追加します。 慣例によりテストプロジェクトに名前を付け、最後にテストを追加することをお勧めします。 したがって、VSTSおよびTFSでテスト実行を構成する方が簡単です。 メインプロジェクトへのリンクを追加します。







UnitTest.csの名前をMainPageTests.csに変更します。 ファイルにそのような名前が付けられている場合、このクラスが正確にテストしているものを理解しやすくなります。



メソッドを追加します。 



  [TestMethod] public void CalcSquere_5_Result25() { Windows.ApplicationModel.Core.CoreApplication.MainView.CoreWindow.Dispatcher.RunAsync(CoreDispatcherPriority.Normal, () => { SimpleApp.MainPage mp = new SimpleApp.MainPage(); double expected = mp.CalcSquere("5"); Assert.AreEqual(expected, 25, "Wrong"); }).AsTask().Wait(); }


メソッドコードはUIスレッドから起動されます。 名前空間を追加することを忘れないでください 



 using Windows.UI.Core;

公開後、ビルド定義を作成できます。 ただし、その前に、ビルドエージェントを構成する必要があります。 実際、単体テストを実行するには、カスタムエージェントを作成する必要があります。



デフォルトでは、3つのエージェントプールがあります。

デフォルトプールは 、リモートサーバーにインストールされているエージェントに使用されます。

ホストプールは 、VSTSおよびTFSサーバーにデフォルトでインストールされるエージェントに使用されます。

ホストされたLinuxプールを使用すると、エージェントを設定しなくてもLinuxマシンを操作できます。 vsts-agent-dockerコンテナドッカー内のUbuntu Linuxホストで動作します。

単体テストの場合、エージェントをインタラクティブに（つまり、スクリプト。\ Run.cmdを使用して）起動する必要があります。



さらに、アプリケーション証明書（メインアプリケーションと単体テストアプリケーションの両方）がローカルマシンリポジトリ（Trusted People）にインストールされている必要があります（* .pfxファイルをダブルクリックするだけでインストールできます）。 エージェントがインストールされているのと同じマシンに証明書をインストールする必要があります。



テストを実行するときに、エラーが発生する場合があります。



...（0x5B4）操作がタイムアウトしました。 30秒以内にWindowsアプリパッケージをインストールできません。



この場合、起動に割り当てられる時間を人為的に増やすことができます。 これは、 .runsettingsファイルを作成することにより行われます。 この場合のファイルの最小コンテンツは次のとおりです。 



 <?xml version="1.0" encoding="utf-8"?> <RunSettings> <!-- Configurations that affect the Test Framework --> <RunConfiguration> <!-- [x86] | x64 - You can also change it from menu Test, Test Settings, Default Processor Architecture --> <TargetPlatform>x64</TargetPlatform> </RunConfiguration> <!-- Adapter Specific sections --> <!-- MSTest adapter --> <MSStoreTest> <AppxInstallTimeout>240000</AppxInstallTimeout> </MSStoreTest> </RunSettings>


ここで、テストするアーキテクチャのパッケージを選択し、アプリケーションのインストールの制限時間を240秒に設定します。



通常、ファイルはアプリケーションのルートディレクトリに作成され、VSTSビルド設定で指定されます。
