Linux / Unixでの認定の準備。 ステップ1

しばらく前、私はUnixライクなオペレーティングシステムの分野での知識を増やしたいと同時に、これらのスキルを証明する証明書を取得したいと思っていました。 これは、私が思うに、長いサイクルからの最初のメモです。 これまでのところ、インターネット上で入手可能な情報の分析と、これに関する私のコメントの一部です。 RedHat Linux、Sun Solaris、または他のUnixライクなOSの認定を検討している場合、これを読むことに興味があるかもしれません。





昔から、仕事と研究を組み合わせるのに慣れていました。 そして、この冬、高等教育の卒業証書を受け取った後、人生に特定の穴が形成されました。 私がひどいオタクだとは思わないでください。Unixのようなオペレーティングシステムのジャングルで彼の自由な時間をすべて過ごします-いいえ。 単純に、第一に、私は常に前向きに努力し、新しいことを学ぶのに慣れています。そして、第二に、私は欲しいものすべてに十分な自由時間がありますが、無駄に無駄になるほど残念な十分な予備がまだあります。

知識に加えて、トレーニングプロセス後の関連スキルの可用性を確認する美しいペーパーを受け取りたいと思います。

インターネットの荒野を調べてみると、RedHatとSun Solarisの2つのブランチが興味を持っていることがわかりました。

RedHatファミリのオペレーティングシステムで長い間働いてきたので、私が興味を持った最初の方向。 人の手に落ちる最初のものが、彼のお気に入りになることがよくあります。 だから、私の手に落ちた最初のLinuxはFedora Core 4であり、Linuxの基礎を研究した大学のトレーニングラボにはRedHatがありましたが、そのバージョンは覚えていません。 その結果、私が構成およびサポートしているサーバーの大部分はCentOSを実行しています。デスクトップでLinuxが必要な場合、Fedoraの最新バージョンをインストールすることをheしません。

2つ目は、私がSun Microsystems Corporationと非常に密接に関係していたので、私にとって興味があります。卒業プロジェクトはOpenSolarisの開発であり、OpenSolaris.orgコミュニティから助成金も受け取りました。 さらに、今年のSun Tech Daysで講演できたことは幸運でしたが、それは別の話です。 開発は一時的に中断されましたが、近い将来、私はそれを継続し、アルファ段階からプロジェクトを外し、それを思い起こさせるつもりです。 OpenSolarisを唯一のオペレーティングシステムとしてほぼ1年間使用してきましたが、さまざまな困難とトラブルがありましたが、必要なほぼすべてのデスクトップニーズに対応するアプリケーションを見つけ、このOSに多くの重大な利点があることを確認できました。 そのため、もちろんスキルは少なくなりますが、学ぶことへの欲求は少なくありません。



それで、利用可能な情報を調べて、認証への道を始めましょう。



レッドハット。

すべてが会社の公式ウェブサイトで十分詳細に説明されており、ロシア語ではかなり詳細な説明がInventa会社のウェブサイトで利用可能です。 この情報には予期しないものは見つかりませんでした。認証を成功させるには、オペレーティングシステムの機能と、メインシステムユーティリティの操作、ネットワークの構成、問題の診断と修正、さまざまなソフトウェア（クライアントとサーバー）のインストールと構成に関する知識が必要です。 詳細情報は、上記のリンクから入手できます。

公式Webサイトの資料に加えて、トレーニングレベルを確認するためのさまざまなテスト （登録後） を利用できます。 RHCTの認定はあまり面白くありません。すぐにRHCEになりたいので、この分野のテスト知識に提出された3つのテストすべてをすぐに合格することにしました。 最初の「RedHat Linux Essentials」は、ほとんどすべての点で私の知識が「深い理解」レベルに対応していると言って問題なく倒れました。 2番目と3番目の「Red Hat System Administration」と「Red Hat Network and Security」-知識は悪くありませんが、試験に合格する前に少し学ぶことをお勧めします。 テストに合格したとき、私はいくつかの興味深い点を見つけました。

-あらゆる種類の洗練されたサーバーアプリケーションの設定ファイルの腸を掘り起こし、それらを相互にペアリングしてログを追跡すると、非常に単純なことを忘れることが多いため、単純なことを繰り返して準備を開始する必要があります。

-テストの質問では、実際のシステムではめったに使用されない小さな詳細にかなりの注意が払われます。 もちろん、私は間違っている可能性がありますが、個人的には、UNIXライクなシステムで作業している間は常に、「chattr + i」コマンドを使用する必要はありませんでした。 -困った。 そのため、準備中に注意する必要がある2番目のことは小さく、一見取るに足りない詳細です。

-テストでは、多くの場合、非常に奇妙な質問があります。 例：デフォルトのアプリケーション構成ファイルで1つまたは別のパラメーターを定義する方法。 正直なところ、私はデフォルト設定に注意を払ったことがないので、そのような質問は行き詰まりました。

ただし、これらのテストは試験自体とは非常に弱い関係があります。 私の知る限り、試験官にはRedHat Enterprise Linuxがすでにインストールされたコンピューターが提供されており、そのコンピューター上で3.5時間タスクを実行します。



Solaris

Sunの認定を取得すると、事態はもう少し面白くて混乱します。

RedHatとは異なり、希望する試験に合格して認定を開始することはできません。 最初のステップであるSun認定システム管理者から開始する必要があります。

公式Webサイトにはあまり情報がありませんが、登録、コースと試験のリストの表示、興味深いものの選択、準備に必要な資料の入手ができる特別なポータルがあります。 RedHatでは、資料は有料で提供されます（かなり大きい）。

OSの知識のテストも提供されます。この場合、RedHatよりもはるかに複雑で便利です。 サンでは、これらのテストは実際には試験タスクです。つまり、テストに合格すると、あなたの知識のレベルと試験の準備の程度が客観的に示されます。 私はすぐに言わなければなりません：私の知識のレベルは予備試験に合格することができませんでした。これは非常に自然なことです：私のSolaris機能の知識レベルはあまり高くありません。 管理者にとって有用な多くのことを知っている自信のあるユーザーですが、ユーザーがシステムに完全に適応する前に、正確に何をどのように知っているのか、あなたはまだ私がやろうとしていることを学ぶ必要があります。



価格について。

価格はやや痛いです。 特にRH。

RHCE試験に合格するためには、異なるバージョンに対して570ドルまたは14,500ルーブルを支払う必要があります（公式Webサイトでは、799米ドルです）。 また、少なくとも短期間の学習コースを受講したい場合は、すぐに35〜40ルーブルを調理してください。

太陽は少しいいです。

試験は2つのパートで構成され、それぞれの費用は150ドルです。 さらに、学生の場合、試してみると、深刻な割引を受けて、試験に60ドルすべてを支払うことができます（RHでも同様の割引が費用の10％しかありません）。 しかし、準備コースの価格はlessしくない：4-5日間800ドルから。



だから：

-Unixライクなオペレーティングシステムのかなり広範なスキルにもかかわらず、認証の準備は非常に難しく、時間がかかります。

-私個人のトレーニングコースの価格はやや高すぎます。 客観的には、これは世界的な慣行では雇用主がそのようなトレーニングに支払うという事実による可能性が最も高い。 自分で支払わなければならないので、家で準備しなければなりません。

-両方の証明書を取得したいのですが、RedHatから始めます。トレーニングのレベルは高く、準備プロセスで得た追加の知識は間違いなくSun証明書の取得に役立ちます。
