コンテンツ広告の収益を増やす方法：レスポンシブコンテンツ





過去数年間で、ウェブ上のあらゆるもののパーソナライズが非常に人気を博しています。 特定のユーザーの検索結果が「シャープ」になり、多くの企業がパーソナライズされたメーリングリストを作成し、オンラインストアは「あなただけ」に適した製品を示しています。



ただし、ユーザーがアクセスするサイトのコンテンツをパーソナライズできます。 Yaglaシステムを開発しました。 このシステムは、サイトの最初の画面（またはその他の部分）を変更して、訪問者の検索クエリによりよく一致させることで、コンテキスト広告の収益を高めます。 今日は、それがどのように機能するかについてお話します。



何に



Yaglaは、ユーザーをサイトに誘導した検索クエリに応じて、サイトのさまざまな要素の見出し、テキスト、画像、ラベルを変更できます。 これらすべての「置換」は手動で設定されるため、検索クエリで直接タイトルを機械的に「ダム」に置き換えるだけでなく、人間の目で理解できる快適な結果を得ることができます。







製品の推奨事項との違いは何ですか



ユーザーがより高い確率で購入するユーザー製品（RetailRocketやCrosssなど）を表示するために、さまざまなソース（ユーザープロファイルおよび機械学習）から受信したデータを分析するシステムがあります。 ランディングページ（ランディングページ）の場合、このようなサービスでは、ランディングページが作成された特定の在庫の製品計算が表示されます。



レスポンシブコンテンツを使用すると、さらに進んでパーソナライズされたオファーを作成できます。 たとえば、「クレジット上のウィンドウ」というクエリの場合、Yaglaシステムは「プラスチックウィンドウは市内で最も安いローンです」という見出しを表示し、「コテージへのプラスチックウィンドウ」というリクエストを受け取ったユーザーに対しては、「プラスチックウィンドウは特別に適合した」という見出しのテキストを表示しますあなたの夏のコテージ」とボタンの碑文を単純な「購入」から「コテージで購入」に置き換えます。 このための製品推奨のサービスは、単に機能していません。



単純な置換に加えて、A / Bテストを実施して、元のオプションと比較して最も効果的な見出しオプションを特定します。



なぜそれが必要ですか



アダプティブコンテンツの主な目標は、コンテキスト広告の収益を増やすことです。 つまり、たとえば、Yaglaサービスを使用するのは、トラフィックを誘導するために使用されるランディングページとコンテキスト広告を持っている人向けです。



コンテキスト広告キャンペーンの実施を自動化する通常のサービスは、せいぜい30〜40％のビジネス上の節約になりますが、アダプティブコンテンツの使用（たとえば、いくつかのセグメント化されたクエリのヘッダーの変更）は、コンバージョンの10倍の増加（「ハイパーセグメンテーション」）につながります。 したがって、この場合のコンテキスト広告の助けを借りてクライアントを引き付けるコストは大幅に低下します。



仕組み：実際の例



モスクワの会社が窓の製造と販売に従事しているとしましょう。 現在、このニッチな製品のコンテキスト広告は収益性がありません（クリックあたりのコストは40〜50 cuに達する可能性があります）が、季節性のために生産をロードする必要があります。



したがって、会社のタスクは、そのような広告キャンペーンが不採算ではなく、さらに収益性が高くなるように、引き付けられたクライアントのコストを削減することです。



Yaglaは、コンテキスト広告を既に設定しているユーザーと、ゼロから設定する必要があるユーザーの両方で機能します（これを支援します）。 新しいクライアントで作業を開始し、Yandex.Metricsなどのデータを分析して、通常は有用なアクション（購入、電子メールアドレスの受信など）につながらない最小の効率（最大障害率）で要求を特定します。 ）



次に、このようなリクエストを処理し（他の多くのリクエストと同様に無効化するだけでなく）、このトラフィックを正しい方向に変換するようなリクエストのアダプティブコンテンツを構成します。 コンテキストキャンペーンでこれらのリクエストを非アクティブ化することにより、別の広告キャンペーンにそれらを配置し、Yaglaシステムを接続します。 したがって、直帰率は2倍から6倍に低下し、そのような要求は収入を生み出し始めます。







組織にコンテキストキャンペーンが設定されていない場合、Yaglaは「適応コンテキスト」サービス（代理店としてのコンテキスト広告の予算を通過）を提供します。 。



たとえば、「ウィンドウ」テーマの場合、アクションスキームは次のようになります。約500個のキーワードが設定され、システムが同期され、ランディングページのマーキング要素が置き換えられ（下のスクリーンショット）、特別なコードを設定し、目標を選択します。 次に、システムは一連のA / Bテストを実行します。このテストでは、ページの元のバージョンと置換された要素を含むバージョンの作業を比較します。



したがって、システムは、会社の代表者によって構成されたアダプティブコンテンツの有効性がWebページ（ランディングページ）の元のバージョンよりも悪いことが判明した場合に通知します。 トラフィックを受信した後、システムはA / Bテストのステータスを提供し、どのリクエストによって交換可能な要素がよりうまく機能しなかったかを特定します。 テスト結果に基づいてこれらの置換を書き換えます。







結論



レスポンシブコンテンツは、ウェブサイトのコンバージョンを増やすだけでなく、多額の投資をせずにコンテキスト広告キャンペーンの結果を大幅に改善する良い方法です。 5つのランディングページと30の広告キャンペーンを含む構成では、1900ルーブルの費用がかかります。これは、コンバージョンを数倍増加させるのに適した価格です。
