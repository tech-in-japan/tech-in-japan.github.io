Operaの新旧グッズ



注：Operaはアフリカの一般ユーザーには公開されませんが、新しいユーザーには興味があるかもしれません。 実際、目標は、この美しいブラウザで私が知っていて使用したすべてを鐘楼から要約することです。 繰り返しますが、完全な物語のふりをしません。 上級ユーザーは、追加して補完できると確信しています。



Operaは、1つのインターフェースでクライアントをメール、irc、rss（およびicqを使用）することで、インターネットリソースを閲覧する便利さをユーザーに提供できる「結合」です。 多くの点で、この決定は私にとって有利です。 Mail M2とrssは素晴らしいメールセンターです。 ネットワークに入る必要があるが、クライアントが手元にない場合、ircは良いボーナスです。

Operaは多くのことができます。 そして、多くのトリックが舞台裏に残っています。





そこで最近、ブックマークを一括で転送するためのクロスプラットフォームツールを提案しました-セッションを使用します。 また、Operaからrssストリームをエクスポートして、Google Readerから簡単に手軽にエクスポートすることもできました。 同じGoogleは、Opera M2で「革新的な」Gmailのメールを扱うというアイデアを取り入れました。 連絡先やメモなどの検索は、多くの場合、緊急時に役立ちます。 「gフレーズ」（またはRundexの場合は「yフレーズ」を含むアドレスバー）、Yandexで検索するための検索結果をすぐにGoogleに送信します...私の観察によれば、インターフェイスははるかに速く、より柔軟でより便利に構成され、競合ブラウザよりも。

キーボードとSpeedDial要素を使用してプログラムを操作する方法については既に説明しました（FFにそのような機能を追加する可能性についてのコメント ）。



あなたはたくさん話すことができますが、それを読むことは面白くありません-あなたは試して、突く必要があります。 Opera F1（新しいページ）をクリックして、理解を開始することをお勧めします。 Mouse Gestrues、音声制御、クイック設定（オン/オフプロキシ、キャッシュからの現在のページの再読み込み、ブラウザー識別など）、以前に入力されたフィールドの自動入力（個人ユーザー情報は入力中にブラウザー自体によって提供されることなど） Web Forms）など。

今日、ブラウザの新しい9.50バージョンが登場します。 約束された革新のうち：

  • 新しいECMAscriptコア（別名JavaScript）はより高速になり、動作に必要なリソースが少なくなりました。

    レンダリングコアの数千（数千と言う）のバグが修正され、多くのサイトとの互換性が向上します。

    CSS3からのoverflow-xおよびoverflow-yプロパティのサポートが追加されました。 ここでは、開発者はこれらのプロパティが処理されるInternet Explorerのリードに従いました（ほとんどの場合、CSS3のIEにあります）。

    JavaScript 1.5からのゲッターとセッターのサポートが追加されました

    ビジュアル編集のサポートの改善（リッチテキスト編集）。

    多くのGoogleリソースで見られる問題を解決しました。

    OS XでWindow-Eyes、Jaws、VoiceOverなどのスクリーンリーダーのサポートが追加されました。

    ページ要素を介したインテリジェントナビゲーションの改善（空間ナビゲーション）。 初心者は、キーボードショートカットを使用する方が簡単になりました。

    すべてのワンキーキーボードショートカットはデフォルトで無効になっています。

    以下を含むWindows Vistaのサポートの改善 システムへの統合。

    OS Xでは、ブラウザーはネイティブOS Xアプリケーションのようになりました。タブ、デフォルトのフォント、点滅ボタンが改善されました。

    これで、Operaが64ビットバージョンで利用可能になります（FreeBSDおよびLinux用）。

    新しいインデックスアルゴリズムと、組み込みのOpera Mailメールクライアントのメールストレージデータベースのフォーマット。これにより、リソースの消費量が減り、ハードドライブへのアクセスが少なくなり、動作が高速になりました。

    Opera Mailの新しいIMAPコア。

    メールおよびニュースフィードのチェック中にOperaがフリーズする問題を修正しました。
