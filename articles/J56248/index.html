WordPress 2.8の新機能

個人的には、 WordPress 2.8に表示される新機能と編集のリストは、それほど目立ったものではありませんでした。



そのようなリストがあることを知って驚いた人はもっと驚いた。



公式ウェブサイトで革新のリストを見つけることができると推測することは難しくありません。



そのため、バージョン2.8には、新機能と見つかったエラーと欠点の修正の両方が含まれます。



私の意見では興味深いニュースがいくつかあります。



-カスタム分類のサポート（さらに、これは少なくともWP 2.8の主な変更点であり、少なくとも真剣に強調されました。WPの分類に関する詳細は、maxsite.orgおよびblog.sjinks.org.uaで見つけることができます）。

-インターフェース「ウィジェット」の改善。

-テーマをインストールするための新しい注文（メカニズム）。



-LJからの輸入マスターの変更。

-SimplePieテクノロジーを使用して、管理パネルとウィジェットでRSSフィードをブロードキャストします。

-「Publish this」テクノロジーの改善（java-scriptリンクがインターネット上にある興味深いページをすばやく公開するために個人用パネルに配置されます）;

-XSS攻撃のフィルターパラメーターを修正。

-管理パネルによるプラグインの検索の改善。

-Control / Command + S（Ctrl + S）を押すと下書きが自動保存されます。

-コメントをスパムとしてマークする際の確認要求をキャンセルしました。

-インストール中、管理者の電子メールの検証が改善されました。

-管理パネルの[ツール]-> [アップグレード]にアクセスしたときに新しいバージョンのチェックを追加。

-テーマとプラグインのエディターに構文の強調表示を追加。

-管理パネルのメニューにペイントされたアイコン（はい、はい、はい！最も重要な革新！） ;

-投稿、コメント、ページマネージャーのアイテム数を変更する画面オプションが追加されました。

-データベースを使用した作業の改善と最適化。

-スクリプトのパフォーマンスの向上。



さらに、デフォルトでインストールパッケージに含まれているjQueryのバージョンがjQuery 1.3.1に更新されました。



WP 2.7に表示されるはずだったが、表示されなかったイノベーション：



-メニューエディター（私たちが理解している限り、これは管理パネルのメニューです）。

-レコードタイプ「Gallery」、およびギャラリーを作成するためのテンプレートタグへの追加。

-テーマエディターでのテンプレートのバージョンリビジョン+テーマの編集時にドキュメントをテンプレートタグにリンク。

-ユーザーインターフェイスからテーマをダウンロードしてインストールします。

-すべての発信HTTP要求をブロックする機能を追加します。



これらは、主に多かれ少なかれ注目に値するすべての変更や革新とはほど遠いものです。



同時に、いくつかの点では、クリエイターが実際に何を理解しているのかを推測するだけです。 たとえば、設計テーマを設定する「新しいメカニズム」という言葉は、残念ながら、正確に何が変わるのか、または根本的に変わるのかという考えを与えません...



何が言えますか？ 上記のポイントに基づいて、管理パネルを離れることなくブログを構成および変更できる一般ユーザーにWPが焦点を当てていることに簡単に気付くことができます。 テーマとプラグインのオンラインエディターの適切な実装レベルでは、FTPを使用する必要は非常にまれです。



一方、21世紀では、ファイルを直接操作せずにすべての必要な操作をWebインターフェースから実行できるのはごく普通のことです。



これが続く場合、私はウェブインストールを見たいです。



今最も重要な質問-急いでアップグレードする価値はありますか？ いつものように、答えは簡単です-自分で決めてください。



個人的に、私は更新されますが、初日ではなく、LAN上の新しいバージョンをよく見るだけです。
