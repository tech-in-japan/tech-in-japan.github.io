利点は何ですか：IaaSを使用するための実現可能性調査

クラウドへの移行の実行可能性調査の必要性は、通常2つの場合に発生します。



  • 既存のインフラストラクチャを近代化する必要があります
  • ITインフラストラクチャをゼロから作成する予定です


2番目のオプションを検討し、比較的大きなオンラインストアを参照ポイントとして起動する必要がある場合に状況を考慮してください。

原価計算に進む前に、プロジェクトの一般的なスキームと技術コンポーネントを自分で決定する必要があります。 この場合、ハードウェアにNginxサーバー（Intel Xeon 2.6 GHz、4C、RAM 8 GB、HDD 2 TB SAS）と2つのWebサーバー（Intel Xeon 2.6 GHz、4C、RAM 24 GB、HDD 2TB SAS）が含まれると仮定します。 。



また、2つのデータベースサーバー（Intel Xeon 2.6 GHz、16C、RAM 48 GB、HDD 2TB SAS）と2つのログサーバー-Memcached、Logs（Intel Xeon 2.6 GHz、2C、RAM 16 GB、HDD 4TB SAS）も含まれます。



プロジェクトの一般的なスキームは次のようになります。







機器の購入と独自のITインフラストラクチャの構築



このケースでは、ファブリックインターコネクト、ネットワーク接続用のCisco Nexusスイッチ、およびストレージとしてのNetAppソリューションを備えたCisco UCSブレードサーバーを購入することに決めたと仮定します（ ところで 、Cisco UCSソリューションのボックス化を行いました）。 このようなITインフラストラクチャを展開する場合、会社は資本と運用コストに直面します。



ハードウェアプラットフォームの購入費用


このタイプの費用には以下が含まれます。



  • ハードウェアコンプレックスの設計コスト（K pr app ）、
  • 北の購入のコスト（K SRV ）、
  • データストレージシステム（K SCD ）のコスト、
  • ネットワーク機器とケーブルのコスト（K セット ）、
  • トレーニングスペシャリストのコスト（K トレーニング ）、
  • インストールコスト（K inst app ）。


したがって、ハードウェアプラットフォーム（K）を作成するコストを計算する式は、次の形式を取ります。



K = K pr アプリ + K cfv + K cdd + K セット + K トレーニング + K instアプリ


インフラストラクチャ設計のコストが120,000ルーブルであると仮定します（作業は1時間あたり1,000ルーブルの価格で120時間続きました）。 設置コストは1時間あたり800ルーブルのコストで32,000ルーブルであり、1人の従業員をトレーニングするコストは80,000ルーブル（40時間のトレーニングと1時間あたり2,000ルーブル）です。 合計232,000ルーブル。



機器のコストを計算しましょう：







その結果、ハードウェアプラットフォーム（K）の作成コストは次のようになります。



K = 232,000 + 19,777,230 = 20,009,230ルーブル。


ソフトウェア費用


ハードウェアの取得とインストールのコストに加えて、ソフトウェアライセンスのコスト（K 人 ）、インストールと構成のコスト（K instPO ）、およびトレーニングスペシャリストのコスト（K トレーニング ）で構成されるソフトウェアのコストを考慮する必要があります。



K by = K 人 + K instPO + K トレーニング


検討中のケースで、組織の従業員が提供されるソフトウェアに精通していると仮定します。したがって、トレーニングコストはゼロになります。



インフラストラクチャソフトウェアのインストールと構成は、1時間に800ルーブルかかり、作業時間を40時間と見積もる会社の1人のスペシャリストによって実行されます。 12個のVMware vSphere Enterprise Plusライセンスもそれぞれ2,300ドルで購入します。



したがって、インフラストラクチャソフトウェアの総コストは次のとおりです。



宛先= 1,837,884 + 32,000 = 1,869,884ルーブル。


ITインフラストラクチャの運用コスト



資本コストとは異なり、運用コストは繰り返しの性質によって特徴付けられます。つまり、単位時間あたりの通貨単位（年間ルーブル）で測定されます。



運営費には以下が含まれます。



  • 独自のサーバー（C データセンター ）の運用コスト
  • ハードウェアの運用コスト（C アプリ ）
  • ソフトウェア部分の運用コスト（C〜）
  • 保守および管理要員のコスト（C pers 、C unitary enterprise ）


したがって、運用コストは次の式で計算されます。



C = C データセンター + C アプリ + C から + C パーズ + C


私たちの場合、ITインフラストラクチャの運用コストは年間7 524 444ルーブルです（各コンポーネントの計算式を使用した運用コストの詳細な分析については、 こちらを参照してください ）。



さらに、ハードウェアの冗長性とコンピューティングパワーの向上についてもいくつか説明してください。 ハードウェアの冗長性により未使用の容量が発生します。これは、メインサーバーの1つに障害が発生した場合、または会社が将来一定の成長を期待している場合にのみ必要です。 また、ハードウェアの保守コストにも影響します。



IaaSモデルでのクラウドサービスの使用



次に、IaaSモデルでクラウドサービスを取得するオプションを検討します。このオプションでは、CPU、RAMコア、およびディスクスペースに必要なコンポーネントを表現します。 IaaSモデルでクラウドサービスを使用して、クライアントは専用の仮想プラットフォームをレンタルし、それによって責任とコストの一部をホスティングプロバイダーの肩に移します。



IaaSの場合、単位時間あたりの金銭的条件で測定される運用コストのみがあります。







したがって、同社はクラウドプロバイダーに年間5,258,400ルーブルを支払うことになります。これは、独自のITインフラストラクチャを構築して維持するよりもはるかに安価です。 同社は機器の1回限りの購入（2400万ルーブル）とそのメンテナンスを行う必要がないため、年間700万ルーブルの無駄が生じます。



このような計算を実行することは、かなり骨の折れる作業です。特定の形式のITインフラストラクチャの実装による実際のメリットを計算するには、経済学者が作業に慣れている多くの要因と数値を考慮する必要があります。



それにもかかわらず、このような計算は、ほとんどの場合、 クラウドへの移行の利点が「言葉で」だけでなく実数でも表されることを企業に示すことができます。 クラウドテクノロジーの実装の実現可能性調査のより詳細な調査については、「 IaaSモデルのクラウド移行のビジネスケース」 という本を読むことができます。
