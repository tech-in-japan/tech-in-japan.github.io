値によるオブジェクトの比較について-1：開始

.NETオブジェクトモデルでは、他の多くのソフトウェアプラットフォームと同様に、参照および値によってオブジェクトを比較できることはよく知られています。



デフォルトでは、対応する変数に同じ参照が含まれている場合、2つのオブジェクトは等しいと見なされます。 それ以外の場合、オブジェクトは等しくないと見なされます。



ただし、特定のクラスのオブジェクトの内容が特定の方法で一致している場合、それらのオブジェクトを同等と見なす必要がある場合があります。



個人データを含むPersonクラスがあるとします-個人の名前、姓、生年月日。



このクラスを例として使用して、以下を検討してください。



  1. このクラスのオブジェクトが標準の.NETインフラストラクチャを使用して値で比較されるように、クラスに最小限必要な改善セット。
  2. このクラスのオブジェクトが常に標準の.NETインフラストラクチャを使用して値で比較されるようにするための最小限の必要かつ十分な改善セット-比較が参照によって行われるべきであることが明確に示されていない限り


ケースごとに、可能な限り一貫性のあるコンパクトなコピー＆ペーストのない生産的なコードが得られるように、値によるオブジェクトの比較を実装する方が良い方法を検討します。



このタスクは、一見するとそれほど簡単ではありません。



また、このタスクの実装を簡素化するために、プラットフォームにどのような改善を加えることができるかを検討してください。

  1. Personクラスは、FirstNameとLastNameの文字列プロパティが常にnullではないように実装されます 。

    FirstNameまたはLastNameが不明（設定されていない）の場合、 空の文字列が値がないことの兆候として適しています。

    これにより、FirstNameフィールドとLastNameフィールドのプロパティとメソッドを参照するときにNullReferenceExceptionが回避されます。また、 nullと空の文字列を比較するときに衝突が回避されます。

  3. オブジェクトを値で比較する場合、最初は日付の比較です。 一般的な場合のデータ型変数の比較は、文字列の比較よりも高速です。
  4. 日付と文字列の比較は、等値演算子を使用して実装されます。 等価演算子は値によって構造を比較し、文字列の場合、等価演算子はオーバーロードされ、同様に値によって文字列を比較します。

Personクラスの場合、次のように Object.Equals（Object）およびObject.GetHashCode（）メソッドをオーバーライドする必要があります 。



  • Equals（Object）メソッドは、値の組み合わせがオブジェクトの値を形成するクラスのフィールドを比較します。
  • GetHashCode（）メソッドは、等しいオブジェクト（つまり、 Equals（Object）を使用した比較がtrueを返すオブジェクト）に対して同じハッシュコードを返す必要があります 。

    オブジェクトのハッシュコードが異なる場合、オブジェクトは等しくありません。 等しくないオブジェクトは同じハッシュコードを持つことができます。

    （ハッシュコードを取得するには、値でオブジェクトを比較するためにEqualsで使用されるフィールドのGetHashCode（）値の「排他的 OR 」操作の結果が通常使用されます。

    いずれかのフィールドがこのフィールドのハッシュコードではなく32ビット整数の場合、フィールド値自体を使用できます。

    2つの等しくないオブジェクトが同じハッシュコードを持つ場合、衝突の可能性を最小限に抑えるために、さまざまな最適化も可能です。


Equals（Object）メソッドのドキュメントには特別な要件が含まれているという事実に特に注意を払う価値があります。



  • x.Equals（y）は、y.Equals（x）と同じ値を返します。
  • （x.Equals（y）&& y.Equals（z））がtrueを返す場合、x.Equals（z）はtrueを返します。
  • x.Equals（null）はfalseを返します。
  • x.yquals（y）を連続して呼び出すと、xとyによって参照されるオブジェクトが変更されない限り、同じ値が返されます。
  • そして、特に、浮動小数点数の値を比較するための規則に関する他の多くの。


また、 GetHashCode（）メソッドのドキュメントには、メソッドによって返される値が定数値ではないという警告が含まれているため、ディスクまたはデータベースに保存してキーとして使用しないでください。オブジェクトの比較に使用する必要があります（等しくないオブジェクトは同じハッシュコードを持つことができます）など。

  1. まず、現在のオブジェクト（ this ）へのリンクが着信オブジェクトへのリンクと比較され、リンクが等しい場合はtrueが返されます（これは同じオブジェクトであり、パフォーマンス上の理由を含め、値による比較は意味がありません）。

 public override bool Equals(Object obj) { //this is necessary to guard against reverse-pinvokes and //other callers who do not use the callvirt instruction if (this == null) throw new NullReferenceException(); String str = obj as String; if (str == null) return false; if (Object.ReferenceEquals(this, obj)) return true; if (this.Length != str.Length) return false; return EqualsHelper(this, str); }
