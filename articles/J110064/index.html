動く！ Remastersysを使用したLinuxライブバックアップ

バックアップについてはスタイルでたくさん話せますが、私はキャプテンエビデンスをプレイしません。 Googleの「Ubuntuバックアップ」の検索結果の最初のリンクでは、rsync、partimage、fsarchiver、dd、tar、単純なバックアップについて説明しています。 これらのツールを試し、失敗したシステムを再インストールするのに何時間も費やしました。 致命的な障害の後にシステムを復元し、何も起こらなかったかのように作業を続行するための最速かつ最も簡単な方法を見つけたかったのです。 カットの下にあるテキストは、私に最初にUbuntuを稼働中のラップトップに置いたときに数か月前に読んだことで、合計で2営業日節約できました。



Remastersysは、インストールされたシステムからLiveCDを作成するスクリプトです。 すべての設定とインストール済みパッケージ。 システムが突然起動したり、曲がった手で絶望的に破損したりして、緊急に作業する必要がある場合、より速い方法はありません。 フラッシュドライブを挿入して再起動するだけです。 次に、Dropboxから（またはどこにいても）現在の作業を取得し、完了です。 後で、このフラッシュドライブからシステムをインストールできます。



調理方法は？

  1. システムが構成され、必要なすべてのプログラムがインストールされた後、一般的なクリーニングを行ってイメージのボリュームを最小限にします。 不要なものをすべてアンインストールしてから、sudo apt-get autoremoveとsudo apt-get cleanを実行し、ごみ箱を空にして、ディスク使用状況アナライザで何が表示されるかを確認します。 Remastersysは、イメージサイズが4ギガバイト以下であると想定しているため、アーカイブを考慮すると、6Gを超えるサイズをLiveCDにプッシュできるとは考えられません。 そして、ここでは、やり直すよりもやりすぎる方が良いです-remastersysは、知能と素早い知恵に違いはなく、イメージサイズが大きすぎることを警告しませんが、プロセスの途中でエラーで単純にクラッシュします。
  2. 次の一連のフォルダーにremastersysを追加します：/home/USERNAME/.cache /home/USERNAME/.thumbnails /home/USERNAME/.local/share/Trash / var / cache。 これは、/ etc / remastersys.confファイルまたはGUIを使用して実行できます。 ビデオ、音楽、またはその他の重いファイルが多数ある場合は、それらが置かれているフォルダーを除外します。 大量のバックアップは、この記事の範囲外です。 Dropboxは奇妙な動作をする可能性があります-復元されたフォルダーを選択しますが、LiveCDにないサーバー上のファイルを消去する可能性があるため、Webインターフェイスを介して手動で復元する必要があります。 もちろん、そのフォルダーを除外することは可能ですが、定期的にバックアップする場合、LiveCDのバージョンは現在のバージョンとそれほど変わらず、フォルダー全体をダウンロードするよりもDropboxの頭をまっすぐにする方が高速です。 ただし、そのようなグリッチはないかもしれません。
  3. プロセスを開始し、長時間待機します。
  4. システム→管理→ブートディスクの作成。 ISOを選択し、フラッシュドライブに入力します。
  5. sudo remastersys clean




フラッシュドライブから起動した後、WiFiを手動で再起動する必要がありました-5秒delov。 さらに、独自のビデオカードドライバーはLiveCDでは機能しません。



注意！

Linuxの管理は私の興味の対象ではありません。 必要に応じてこれを行います。 したがって、私はRemastersysの操作に関する他の多くの側面、たとえばブートメニューの変更には触れませんでした。 また、結果のフラッシュドライブを、ブート、インストールなど、さまざまなコンピューターでテストしませんでした...すべてがすぐに機能しました。 詳細な手順は、remastersys Webサイトにあります。
