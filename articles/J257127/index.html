MicrosoftはProject Astoriaを示しました：WindowsベースのアプリケーションでAndroidコードを使用する方法



Microsoft WebサイトのUniversal Windows Platform Bridgesセクションで、AndroidアプリケーションをWindows 10に転送するように設計されたProject Astoriaソフトウェアツールの発表が表示されました。



ただし、Androidでは、XcodeプロジェクトをVisual Studioにインポートし、ObjectiveCコードへの最小限の変更でWindowsアプリケーションを構築できるというiOSセクションの一般的なステートメントとは異なり、プロジェクトエンジニアはProject Astoriaの機能の簡単な説明を含む短いビデオを見せました。



ビデオからわかるように、Androidアプリケーションの互換性が分析され、この分析の結果、Androidの一般的なユーザーアクションはWindows 10の同様のアクションに変換されます。例として、ソーシャルネットワークでコンテンツを共有する比較的単純なアクションを示します。 この場合にJavaコードに正確に何が起こるかはまだ不明です。 Windows SDKを使用するには、Visual Studioに切り替えずにAndroid StudioとEclipseの両方を使用できます。



再利用できるコードの量と、Google固有のツールが含まれているかどうかを正確に理解するには、App Analysisツールを使用する必要があります。 開発者がWindows 10との互換性のために推奨されることをどのように見るかを分析した後、APKファイルをサイトにアップロードする必要があります。この瞬間はビデオに示されています。



さらに、レドモンドは、GoogleサービスからWindowsサービスへの移行が、コードのある行を別の行に置き換えるようなものであることを確認したと述べています。 広告やアプリ内購入などのすべてのツールは、最小限の労力でWindows環境に移行されると主張されています。



Project Astoriaのステータスはまだアルファ版ではありませんが、必要に応じて特別なテスト契約に署名し、開発中に使用されるAPKファイルをチームに送信できます。
