Virtual BoxでChromium OSを起動します

数日前、Googleは、Chromium OSはブラウザのみを起動するウィンドウマネージャを備えた（Ubuntu-）Linuxにすぎないことを示しました。 私は個人的にはこの考えは革新的だと思っていましたが、批評家はこれはおそらく「薄い」クライアントであると叫び始めますが、妄想患者は1984年について話し、彼らはすべて部分的に正しいでしょう。 コンピューター自体が神秘的で複雑なものであり、「ブラウザ」という言葉が「インターネット」という言葉の同義語にすぎない多くの近くのユーザーにとって、そのようなシステムは人生を大いに簡素化すると思います。 しかし、私たちは哲学的な質問には立ち入りません。 Chromium OSが完全に処理されるまで、ほぼ1年待ちたくありませんでした。この段階では、製品はまだ非常に湿っているからです。 システムをライブで試してみたかった- ソースコードは完全にアクセス可能です。 幸いなことに、Chromium OSをVirtual Box'eにインストールする方法に関する記事がtechcrunch'eに 掲載され、 ハブにも仮想イメージへのリンクがあります 。 それにもかかわらず、私はあなたと共有したい小さな実験を行った。







0）必要なもの



仮想ボックス

こちらからChrome OSから画像をダウンロードし（トレント） 、解凍します。



1）仮想マシンを作成する





VirtualBox、Newを起動し、「Chrome OS」という名前をLinuxオペレーティングシステムのタイプ、Ubuntuのバージョンに設定します





スクリーンショットの512MBで動作メモリのサイズを設定しましたが、後で十分ではないことがわかりました。 1024を与える方が良い。











ダウンロードして解凍したイメージをインポートします。既存のディスクを使用し、パパに移動して[追加]をクリックし、chrome-imageを選択します。





完了







2）始める







あなたのGmailアカウントで行きます





すでに標準アプリケーションを使用できます





ドックス





計算機は起動しましたが、どうやらうまくいかなかったようです





Flashは問題なく動作します





picasaは写真を表示します





カレンダー





プレゼンテーションスライド





さて、みんなのお気に入りのHabr。 特別なものは何もない、普通のChrome :)





ほぼすべての興味深いものは、魔法のキー/組み合わせによって制御されます。 F12は開いているすべてのウィンドウを表示します





F8はすべての可能な組み合わせを示します





Chromeを見るときに出てくる質問の1つ：ユーロコミッションは、プレインストールされたChromeブラウザーにどのように反応しますか？ 代替品を置くことは可能ですか？

about：メモリはUIプロセスのみを表示します





Ctrl + Alt + Tは、全画面モードで開くターミナルを起動します

Chrome OSがブラウザの下にあるものを見ることができます。 そして、多くの愛されている（そして私も）Debian / Ubuntuがそこに住んでいます：判明したように、GoogleはCanonicalと協力しており、いくつかのUbunta開発者もChromeに関与しています。



以上です。 私は仮想マシンを数回連続して「凍結」させましたが、とにかく、この段階のアイデアはすでに気に入りました。 興味深いことに、これは最終的にこれから出てきます。 結局のところ、AndroidとChromeの合併でさえ非常に現実的なようです。

素敵な実験をしてください！
