Middlewareミドルウェアとは何ですか？



ウィキペディアには、適切で簡潔な定義があります。

ミドルウェア-IPTVコンプレックスを管理するためのミドルウェア。 これは、最終的に加入者が利用できるサービスの範囲、ユーザーインターフェイス、移行ロジック、および制御アルゴリズムを決定するため、IPTVソリューションの主要コンポーネントです。 ミドルウェアは、複合体のほぼすべてのコンポーネントの相互作用のプロセスでコーディネーターの役割を果たします。


私はこの定義に完全に同意します。 ミドルウェアなしでIPTV / OTTサービスを構築することは不可能です。 市場には多くの開発者がいます。各ミドルウェアには独自の特性があります。サポートされるデバイスのリスト、サポートされるCASシステムのリスト、ユーザーインターフェイス、サポートされるビデオサーバーのリスト、課金、コスト、サポートのさまざまなAPI。



1つのソリューションを選択すると、オペレーターはミドルウェアに接続され、セットトップボックスのソフトウェアを変更する必要があるため（またはサポートされているデバイスのリストが一致しない場合はソフトウェアを交換する必要があるため）、迅速かつ安価にソリューションを変更できません。 また、コンテンツクロージングシステムである課金と再統合する必要があります。 この時間とお金、そして加入者ベースが大きいほど、困難になります。



私たちのサイトにはミドルウェアに関する良い記事があります。



hyなぜストーカー？



まず、Stalkerは無料であるという点で興味深いです。 真の、無料、試用期間と機能制限なし。 ソースコードはGitHubからダウンロードできます 。



これは、サードパーティのソリューションに対するStalkerの非常に重要な利点です。 私は長年IPTV業界で働いてきましたが、他の無料のソリューションを知りません。 同僚やInfomirの開発者にさえ尋ねましたが、私も知りません。 他の無料ソリューションを知っている場合は、コメントに書いてください。



すべてのオペレーターがミドルウェアの購入に喜んで投資するわけではありません。ミドルウェアの購入がなぜ必要なのかがすぐにはわからないためです。 ここでは、衛星放送受信機がテレビチャンネルを受信し、CASシステムがコンテンツを保護し、セットトップボックスがビデオを表示し、請求がお金を考慮します。 ミドルウェアは何をしますか？ チャンネルと天気予報のリスト？



もちろんそうではありませんが、これは無料のソリューションに興味を持ち、テストサーバーにStalkerをインストールしてお互いをよりよく知るために十分です。



インストール



Infomirには、いくつかのドキュメントとVirtualBoxイメージがあります。

Stalkerをインストールするには、OSに200以上のパッケージをインストールする必要があります。 別の仮想マシンを選択するか、Dockerコンテナにインストールすることをお勧めします。そうしないと、他のWebサービスと競合する可能性があります。


VirtualBoxイメージ



VirtualBoxイメージの実行は複雑ではありませんが、その方法を簡単に説明します。



  1. VirtualBoxをインストールします。 ウェブサイト： www.virtualbox.org
  2. イメージを含むzipファイルをダウンロードして解凍します。 ページ上の現在のリンク： http : //wiki.infomir.eu/doku.php/stalker : start
  3. VirtualBoxの「Machine」メニューを起動し、「Add」（Ctrl + A）をクリックして、ファイルVmVirtualBox_Ubuntu14.04.4.x64_MW.Stalker.Demoを選択します
  4. 仮想マシンを起動し、ログイン（テスト/テスト）して、マシンのIPアドレスを確認し、ブラウザーで管理インターフェイスを開きます。
  5. 「ストレージ」管理インターフェイスで、ストレージIPアドレスを仮想マシンインターフェイスIPに変更します


▍Dockerコンテナー



Stalkerをdockerコンテナにインストールしましょう。 速くて便利です。 Dockerが何であるかわからず、それを使用したことがない場合は、記事habrahabr.ru/post/310460を読んでください。 コンテナの動作の全体像を示します。 しかし今、この知識は必要ありません。



開発者は Ubuntu Server LTSの使用を強くお勧めしますが、16.04はまだサポートされておらず、12.04は広く使用されていません。 Dockerを使用すると、お気に入りのディストリビューションでStalkerを実行できます。



続行するには、DockerとDocker-compose自体が必要です。 ほとんどのディストリビューションのインストール例： 



curl -sSL https://get.docker.com/ | sh curl -L https://github.com/docker/compose/releases/download/1.8.0/docker-compose-`uname -s`-`uname -m` > /usr/local/bin/docker-compose && chmod +x /usr/local/bin/docker-compose


Stalkerが住む作業フォルダーを作成し、docker-composeファイルをダウンロードします。 



 mkdir /opt/stalker && wget -O /opt/stalker/docker-compose.yml https://gist.githubusercontent.com/klyushkov/3cc71f3fe3ea27a44c4a6985a1504d4a/raw/c38c746369e4b5238db23ef83ccf5592dc53a63a/stalker-docker-compose.yml


ストーカーを起動します。 



 cd /opt/stalker && docker-compose up -d


Stalkerの最新バージョンをダウンロードし、データベースに入力するユーティリティを起動します。 



 docker exec -it stalker_portal_1 /opt/deploy.sh


スクリプトが実行されるのを待っています。 約4分かかります（この間に何が起きますか？）。 Stalker管理パネルは次の場所にあります： 



  http：// ip / stalker_portal /


標準のログイン/パスワード： admin / 1を使用します。



もう一度、文字通り3つのチームがどれだけシンプルかを一緒に示します。

古い方法でインストールできます。必要なパッケージをインストールし、いくつかのファイルを手動で編集するだけです。 この方法は最も長く、最も手間がかかりますが、実稼働環境でサーバーを起動する場合に最適です。

▍プログラムガイド（EPG）



番組ガイドなしでサービスを構築することは不可能です。人々は長い間チャンネルをクリックすることに興味がありませんでした。 EPGは、ユーザーが現在のプログラムの名前と今夜の予定を確認できるだけでなく、ビデオアーカイブを整理するためにも必要です 。 ユーザーが既に合格したプログラムを視聴できるようにします（いわゆるキャッチアップ ）。



Stalkerは、XMLTV形式からEPGをインポートできます。



XMLTVは、すべてのEPGプロバイダーでサポートされている一般的なXMLベースのプログラムガイド記述形式です。 名前、開始時間、終了時間、ジャンル、説明、写真、俳優のリスト、年齢の評価などの詳細な説明が含まれています。



セットアップを続行するには、EPGサプライヤを取得する必要があります。 彼らは有料で、無料です。 彼らは、テレビチャンネルの数、情報の量（例えば、誰もが各番組の写真と俳優のリストを持っているわけではない）が異なります。 サービスを宣伝することはせず、GoogleでXMLTV形式の番組ガイドを見つけて続行します。



メニュー「IPTVチャンネル」→「EPG」を開きます。 「EPGを追加」ボタン。







xmlファイルがあるWebサーバーへのリンクを挿入する必要があります。 私自身の経験から、ほとんどの場合、サプライヤーはパスワードで閉じられたftpサーバーにアップロードし、それをさらにアーカイブします。そのため、Stalkerでlocalhostへのリンクを挿入し、crontabでXMLTVをダウンロードして目的のフォルダーに解凍するスクリプトを追加します。



リンクを追加した後、「更新」をクリックします。 すべてが正しく行われた場合、結果は次のようになります。







次に、チャネルの設定に移動します。 メニュー[IPTVチャンネル]→[チャンネル]で、[TVチャンネルの編集]をクリックします。 「EPG」セクションに興味があり、チャンネルのIDを示し、必要に応じてタイムゾーンの時間を調整します。
