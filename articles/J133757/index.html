Operaのadmin MODX Revolution管理パネルでのさまざまな保存のバグ（外部ダウンロード）

私の不明な理由（誰かが本当の理由とそれが排除されない理由を意味する場合-教えてください、私は非常に感謝します）、OperaがMODX Revolutionでサイトを管理するために使用されるとき、リソースが保存されると「永遠の」読み込みのウ​​ィンドウが表示されます



このウィンドウは、ページを更新することにより、1つの方法で削除できます。 また、管理部分は「AJAX」の原則に従って機能するため、作業が大幅に遅くなります。



このバグは、Opera 11.10の最新バージョン、MODX Revolution 2.1.5-pl（高度）およびExt JS Library 3.4.0の執筆時点で表示されています。



この問題に対する一時的な解決策を提案します（MODX Revolutionパッケージの一部としてExt JS Libraryの新しいバージョンがリリースされるまで、またはMODX（またはOpera）がこれを修正するまで）。







「永遠の」ダウンロードウィンドウを削除するには、ファイル「manager / asset / ext3 / ext-all.js」の行を変更する必要があります。 

throw {message:"JsonReader.read: Json object not found"}


に： 

 return 1;




コメントを書いてください、あなたの注意をありがとう！



PSこのような変更の後、ページの更新に関するデータは管理パネルに移動しないことに注意してください。 つまり、たとえば、ページタイトルを更新した後、リソースツリーのページタイトルは更新されません（ここではプラスが表示されますが、このツリーはプロセスを一時停止するたびに更新されません）。 必要に応じて、[リソース]セクション（ツリーの更新）の適切なボタンをクリックします。 また、ギャラリーモジュールでは、新しい写真は表示されませんが、ページがフリーズすることはなく、すべての写真は正常にバズされますが、アルバム内のサムネイル表示領域にはすぐには表示されません。
