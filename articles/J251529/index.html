それで、私たちは自分自身の基礎を作成しました。いくつかのコマンドを書く時です:)キーボードのWin + Rキーを押し、開いたウィンドウに「cmd」というフレーズを押して、 Enterを押してコマンドラインに行きます。 ところで、Vagrantをインストールした後、そのコマンドをコンソールで使用できるようにするには、マシンを再起動する必要があります。 まず、コンソールに「vagrant」という単語を入力して、ツールが正しくインストールされているかどうかを確認しましょう。







元気です。 問題がある場合は、PATH変数を確認してください 。Vagrantバイナリへのパスが必要です。 まだマシンをリブートしていない場合はリブートします。

サーバー接続



putty.exeファイルをダウンロードし、それを使用してサーバーに接続します。 プログラムを次のように構成します。サーバーアドレス、ポート、およびユーザーを指定し、「 接続-> SSH->認証 」セクションで、 自分で生成した秘密キーのアドレスを指定することを忘れないでください。 、プログラムを起動するたびにそれらを駆動しないようにし、「開く」ボタンをクリックしてください。 注意してすべてを正しく実行すると、自分の仮想サーバーのターミナルウィンドウが目の前に開きます。







PS一部のマシンでは、次の問題が特定されました。Vagrantは何らかの理由で、Puttyで作成されたキーファイルを受け入れず、独自の、より安全なファイルを生成します。 このキーは、アドレス「〜\ .homestead \ .vagrant \ machines \ default \ virtualbox \ private_key」に保存されます。 キーを使用してサーバーに接続できない場合は、Vagrantによって生成されたキーを使用する必要があります。 それを使用する前に、PuttyGenプログラムを再度開き、Vagrantによって作成されたファイルをその中にインポートし（変換->キー項目のインポート）、パテ形式で保存し（「秘密キーの保存」ボタン）、このファイルを使用してサーバーに接続しますパテ。
