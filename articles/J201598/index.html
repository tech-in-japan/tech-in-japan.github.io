クロスプラットフォームゲームの作成経験（iOS / Android）



この投稿は、レインボーパスgamedevに着手したばかりの開発者に役立ちます。 最近の投稿で、ゲーム構築の分野でのドリームチームの最初のステップについて既に話しました。 約束どおり、私たちはこの方向にさらに取り組んでおり、異なるジャンルのゲームに挑戦することにしました。 次に、2つのiOSおよびAndroidプラットフォームでのゲームの作成と公開のいくつかの瞬間について簡単に説明します。







開発ツール



Cocos2d-x





これまでの経験から、2つのプラットフォーム（ios \ android）へのスプレーは、特にゲームプロジェクトではかなり無駄が多いことがわかっています。 この点で、選択はクロスプラットフォームになりました。 当時、私はUnity3DとCocos2dの使用経験がありました。 そして、提案されたプロジェクトは2Dゲームであったため、選択肢はCocos2D-xのクロスプラットフォーム実装にかかっていました 。 選択において最も重要な役割は、オープンソースエンジンの景品でした。



Cocos2d-xはオープンソースのゲームエンジンであり、そのソースコードはMITライセンスの下で配布されています。 iOS、Android、Windows Phone、BlackBerry、Tizenプラットフォームのクロスプラットフォーム開発には、C ++、Javascript、Luaなどの言語を使用できます。 以下にプラットフォームの表を示します。「o」はすべてが正常に動作することを意味し、「i」は開発がサポートされているが推奨されないことを意味します。



このプラットフォームには、あなたを困らせない大きなコミュニティがあります。 このエンジンを使用している有名な企業の中で、Zynga、Konami、Glu、Gamevil、KingSoftが注目に値します。 また最近、Microsoft Open TechnologiesはゲームをWindows 8およびWindows Phone 8に簡単に移植できるようにしました 。



JS開発



私たちのプロジェクトはjavascriptを使用して開発されました。 Javascriptバインディングの使用は、C ++に比べて開発が高速であり、ブラウザでプロジェクトを実行する機能（ブラウザコンソールでのプロジェクトの便利なデバッグ）によって議論されています。



JavaScriptを使用したゲームの作成は、すべてのcocos'a実装（cocos2d-x、cocos2d-ios、およびcocos2d-html5）で利用できます。 ただし、cocos2d-html5はブラウザーで実行されますが、cocos2d-x / iosはjs-bindingを提供し、パフォーマンスを大幅に向上させることができます。 実際、グラフィックス、レンダリング、物理学などのすべてのエンジンコードはネイティブに実行され、ゲームロジック自体はjavascriptで実行されます。 また、簡単な操作により、最もリソースを消費する操作をネイティブに実装できます。 SpiderMonkeyは javascriptエンジンとして使用されます。

ゲームのブレーキが確認されなかった唯一のブラウザはSafariです（ただし、推奨事項ではChromeが推奨されています）。 簡単な操作により、Firefox OSを搭載したZTE Openでプロジェクトを開始しました。

jsを使用して基本コードを変更する場合、新しいメソッドのバインディングも変更する必要がありますが、これはbindings-generatorを使用して自動的に行うこともできます 。



CocosBuilder





私たちにとって不可欠なヘルパーは、ゲームとアプリケーションの迅速な開発のための無料ツール（MITライセンスの下でリリースされた）であるCocosBuilderです。 これを使用すると、シーン、スプライト、インターフェイス要素、オブジェクトアニメーションなどを制御できます。組み込みツールまたは特別なパーティクルデザイナーツールを使用して準備できるパーティクルシステムがあります。 また、フレームごとのアニメーションをカスタマイズしたり、アニメーションのシーケンスを設定したりすることもできます。

現在、javascriptはアルファバージョン3でのみサポートされています。 Mac OSでのみ利用可能です。 こことここで可能性を知ることができます 。



CocoStudio





CocoStudio-少し前に登場し、CocosBuilderの代替として作成されました。 以前は、インターフェースは中国語のみでしたが、最近では通常の英語に翻訳されました。

起動時に、プログラムは4つのエディターの選択肢を提示します。

  • UIエディター–ボタン、リストなど。 これまでに、12個のインターフェース要素が利用可能ですが、さらに追加することを約束します。
  • アニメーションエディター-開発者によると、このエディターはAdobe Flashアニメーションに似ています。 主な重点は骨格アニメーションにあり、通常のフレームごとのアニメーションもあります。 テクスチャパッカーがあります。
  • データエディター-Excelテーブルのデータをcocos2d-xでの処理に適したデータに変換できます。
  • シーンエディター-最初の3つのエディターから取得したプロジェクトのすべての部分を操作できます。 つまり、ゲームレベルをエミュレートできます。 パーティクルデザイナーなどのサードパーティの補助ツールを使用することもできます。




一般に、可能性をざっと調べてみると、製品に明るい未来があることが明らかになります。 残念ながら、Windowsバージョンしか利用できないため、開発ではこのエディターをまだ使用していません。



タイルマップエディター



タイルは、レベルの作成に使用したオープンソースプロジェクトです。



タイルの操作については、次の手順で説明できます。

  • 最初に、ベースカードのサイズとタイルサイズを選択します。
  • 写真付きのアトラスをTileSetsに追加します。
  • マップ上にタイルを配置します。
  • カードを標準化されたtmx形式で保存します。


作成されたマップはTMX形式（タイルマップXML）で保存されます-さまざまなサイズ、さまざまな数のレイヤー、オブジェクト、およびそれらのプロパティのタイルでマップを柔軟に記述することができます。 たとえば、表面タイル、装飾用に異なるレイヤーを設定できます。 オブジェクトの軌跡を設定することも便利です。 プロパティをタイルに設定することにより、たとえば、動的オブジェクトにプログラムで置き換えることができます。

レベルエディターを知るための素晴らしい投稿： one 、 two 。



テクスチャパッカー



テクスチャパッカー - テクスチャを使用してアトラスを作成するために使用します。 Cocos2d、Corona、Sparrow、LibGDX、LimeJS、Ogreなど、さまざまなプラットフォームの料金を支払うことができます。Mac、Win、Linuxで利用可能です。 この会社は、現在のプロジェクトで使用している別の優れたPhysic Editor製品も提供しています。





プロジェクトの組み立て



プロジェクトのアセンブリは、iOS、Android、およびhtml5の設定が個別に考慮されるCocosBuilderでの公開から始まります。





さらに、iOSの場合、プロジェクトをXcodeで開き、シミュレーターまたはデバイスでゲームを起動します。

しかし、Androidの場合は、少し手を加える必要があります。 Android NDK、SDK、およびcocos2d-xへのパスを構成し、build_native.shスクリプトを使用してプロジェクトをアップグレードする必要があります。 次に、結果のプロジェクトをEclipse（またはIdea）で既に開いて、マニフェストとJavaコードですでに作業することができます。 ところで、支払いと一部の広告ネットワーク用の既製のフレームワークはすでに存在しますが、両方のプラットフォームにネイティブコードを使用することは難しくありませんでした。

公開オプションの詳細については、 こちらまたはこちらをご覧ください 。

ゲームでは、ios 5.0の最小バージョンを配置します。 Androidの場合、最小バージョンは4.0ですが、2.3にすることは可能ですが、ゲームは大画面のデバイス向けに設計されており、後のバージョンではクラッシュの可能性が大幅に低減されているため、今のところ、新しいデバイスのパフォーマンスを確認することにしました。



打ち上げとプロモーション



合計で、ゲームの開発には約1.5か月かかりました。 ほとんどの時間は、エンジンとユーティリティを知るために費やされました。 このプロジェクトは、11月5日に夜にApple Storeに登場し、翌朝、プロモーションを開始しました。 プロモーションのために誓約した資金のほとんどは、Facebookモバイルアプリケーションでの広告に焦点を当てており、米国と私たちのゲームに関連する利益のみをターゲットにしています。 また、Google + / Vk / Facebookのソーシャルネットワークへの口コミの影響を期待して、シェアウェアといいね！やリツイートを購入しました。

広告キャンペーンの残りの部分は無料で行われ、iOS / Androidに関するオンライン出版物の編集者にプレスリリースを送信して、ゲームに興味を持たせたり、毎週のダイジェストに参加したり、ゲーム自体をレビューしたりする機会を与えました。

しかし、残念なことに、Facebookでの広告は他のゲームで達成できたポジティブな結果をもたらさず、ほとんど跡形もなく通過しました。 米国の広告予算は、ロシア（ios）と比較して、おそらく10倍にすることができます。

これまでに達成された最高位は、ゲームが「戦略」カテゴリの上位50位に入ったわずか6カ国と、同じカテゴリで200〜300位の範囲にある約30カ国です。







ゲームの全般的なトップまたはすべてのアプリケーションの全般的なトップには、多かれ少なかれ顕著な結果はありませんでした。 今後は、既存のアプリケーションの助けを借りて相互プロモーションを行う努力をするとともに、たとえばw3bsit3-dns.comでいくつかのレビューを購入します（ちなみに、開発者のサポートポリシーが変更されました）。 ヨーロッパ諸国をターゲットにしたFacebook広告を試してみましょう（スペイン、ドイツ、イタリア、まだ決定していない可能性が高い）。 排気量ははるかに大きくなるため、iOSプラットフォームにほとんどの労力とお金を投入します。 一般的に、これはすべてのプラスとマイナスのクロスプラットフォーム開発に関する最初のプロジェクトでした。いわば、ペンのテストであり、私たちはそれが来たものが好きです。今後のプロモーションがゲームをわずかに上げてトップに保つのに役立つことを期待しましょう。



要約すると、現在のツールを使用するとゲームを簡単に作成できます。アイデアがあります。 作業プロセスに関するご質問をお待ちしています。
