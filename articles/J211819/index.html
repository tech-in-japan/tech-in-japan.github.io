AppFactory（スタジオでのスクリーンショット！）



Pavel Litvinko（プロジェクトの作者で、最も高価な賞品-RAD Studio XE5の商用ライセンスを受け取った）はとても親切で、非常に高品質のビデオレビューをしました 。







アプリケーションのポイントは、プログラミングなしでアプリケーションを作成することです。



2つのアプリケーションがあります。 1つ目のAppFactoryでは、視覚的に（プログラミングなしで）「テンプレート」、「プロトタイプ」、「設計時モデル」を作成できます。 「コンテンツ入力ディレクティブ」を含むこの「テンプレート」は、外部xmlファイルに保存されます。 2番目のアプリケーションであるAppProductsは「テンプレート」をダウンロードし、その場で、完全に機能するモバイルサブジェクト指向ソリューションを形成します。



まだシステムの本質に追いついていない人は、CMS、指定された構成に応じたインターフェースの動的生成、Delphi自体、「オブジェクトインスペクター」およびdmf / fmxを覚えています。 AppFactoryを起動し、ビジュアルデザイナーと同じように作業し、ウィジェットを追加し、インターフェイスをマークアップします（レポートデザイナーと同様）。 そこで、「コンテンツ」を視覚的に定義します。







次に、別のアプリケーションAppProductsを起動します。AppProductsは、選択した「構成」に従って、Flashテクノロジーのプレーヤーとして「ランタイム」を作成します。 それはまともです（そしてアプリケーションはネイティブです）。

6.次に、プロジェクトの機能について説明します。 ユーザーの目であなたの決定を説明しました。 「作成者の目を通して」ください。



AppFactoryでのアプリケーションの作成は非常に簡単で高速です。



プロジェクトには2つのアプリケーションが含まれます。

  • AppFactory-アプリケーションを構築するためのプログラム。
  • AppProductsは、完成品を発売するためのプログラムです。


エディターは、アプリケーションの設計とそのコンテンツを分離するように構築されています。 メインコンテンツは、コンテンツが配置されるウィジェットで構成されます。



各ウィジェットは、特定の機能を備えた完全にカスタマイズ可能なモジュールです。 ユーザーは、作成されるアプリケーションのタスクに応じて、必要なものを追加し、それに応じて調整します。



ウィジェットは、AppFactoryで作成されたアプリケーションの主要部分です。 作成されたアプリケーションの設定で、それらの表示方法（および、それに応じてそれらの間のナビゲーション）を指定できます：リストまたはタブ。 したがって、将来のアプリケーションのメニューが作成されます。

アプリケーションの一般的なビューを作成するには、VCL \ FMX Delphiコンポーネントの操作に原則的に似ているインターフェイス要素（画像、ラベル、パネルなど）を使用します。 それらのプロパティもAppFactoryで柔軟に構成されます。 完成したアプリケーションは、AppProductsを使用して携帯電話ですぐに起動できます。



AppProductsが必要なのはなぜですか？ もちろん、スマートフォンで直接.apkファイルを収集したかったのですが、残念ながら、形式の仕様により、これを行うことはまだできませんが、将来的には可能です。 したがって、アプリケーションを実行するには別のプログラムが必要ですが、アプリケーションをDelphiソースコードにエクスポートして、独立したコンパイルを行うこともできます。



作成したアプリケーションは自由に交換できます。主なことは、AppProductsがクライアントにインストールされていることです。 プロジェクトの目標に従って、これはまったく重要ではありませんが、それどころか、インタラクティブなコンテンツの作成と共有のアイデアの普及に貢献します。
