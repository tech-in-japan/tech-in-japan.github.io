Fluent 1.0：柔軟なローカリゼーションシステム





Fluentは、Mozillaが開発したローカライズの仕様、実装、およびプラクティスのファミリです。 現在、Firefoxブラウザーで使用されています。 Fluentを使用すると、翻訳者は母国語でより自然な翻訳を作成できます。 本日、Fluentバージョン1.0ファイル形式仕様を紹介します。 翻訳ツールの開発者を招待して、試してみて、感想を伝えてください。







Fluentはどのような問題を解決しますか？



Firefoxブラウザーは約100の言語をサポートし、ローカライズの点で開発者に挑戦します。 多くの問題は、従来のアプローチでは解決が困難です。 ソフトウェアのローカライズでは、ソーステキストを使用した1対1の翻訳マークアップの原則が優勢です。 ソース言語の文法（Mozillaの場合-英語）は、翻訳の表現手段の可用性に重大な制限を課します。







ユーザーが複数のタブがあるFirefoxウィンドウを閉じようとすると表示される次のメッセージを考えてください。 







tabs-close-warning-multiple = You are about to close {$count} tabs. Are you sure you want to continue?


メッセージは、2つ以上のタブがある場合にのみ表示されます。 英語では、タブという単語は常に複数形になります。 英語を話す開発者は、このようなメッセージに満足しています。 $ countの値については正しいようです。





英語では、$カウントのすべての値に対して、1つのメッセージオプションで十分です。







多くの翻訳者は、$ countの値に応じて単語tabが異なる形式をとることに気付くでしょう。







ローカライズの従来のアプローチでは、メッセージを正しく処理する責任は翻訳者にあります。 彼らは、他の言語では複数の形式の単語が存在する可能性があることを考慮しなければなりません。たとえ英語でそれが1つだけであってもです。 アプリケーションがサポートする言語が多いほど、問題は深刻になります。







  • 一部の言語では、名詞には性別があり、形容詞と分詞の変更が必要になります。 フランス語では、「connecté」、「connectée」、「connectés」、および「connectées」という言葉は、接続された英語に翻訳されます。
  • スタイルガイドでは、ソフトウェアを実行するプラットフォームに応じて特定の用語を使用する必要があります。 たとえば、Firefoxの英語版では、これらのシステムで採用されている標準に準拠するために、WindowsプラットフォームではSettingsが使用され、他のシステムではPreferencesが使用されます。 日本語では、その違いはさらに興味深い場合があります。一部のコンピューター用語で使用する書記法の選択は、オペレーティングシステムによって異なります。
  • アプリケーションのコンテキストと対象読者は、テキストに追加の変更を必要とする場合があります。 アカウンティングを操作するための英語アプリケーションでは、インターネット上の通常のサイトで使用されているものとは異なる桁フォーマットを使用できます。 しかし、他の言語では、そのような分離は必要ないかもしれません。


2つの言語間で直接相関させることのできない文法的および文体的なバリエーションが多数あります。 従来のアプローチを使用してこれらすべてのニュアンスをサポートすることは困難です。 妥協しなければならない言語もあれば、適切なソリューションがまったくない言語もあります。







非対称ローカリゼーション



Fluentは、ローカライズにおける役割の分布を変更します。 開発者にすべての言語の考えられるすべてのニュアンスを考慮するように要求する代わりに、Fluentはソース言語のテキストを可能な限り単純な形式で維持しようとします。







ソーステキストに関係なく、他の言語の文法とスタイルを伝えることができます。 すべてが独立して行われます。 1つの言語がローカライズでより高度なロジックを使用できるという事実は、他の翻訳言語には影響しません。 プログラムのローカライズごとに、翻訳テキストの複雑さを決定します。







たとえば、タブを閉じることに関するメッセージのチェコ語への翻訳を見てみましょう。 パネル（タブ）という単語は、タブ2、3、4の数を表すpanelyと、他のすべての値を表すpanelůの2つの形式のいずれかを取る必要があります。 







 tabs-close-warning-multiple = {$count -> [few] Chystáte se zavřít {$count} panely. Opravdu chcete pokračovat? *[other] Chystáte se zavřít {$count} panelů. Opravdu chcete pokračovat? }


Fluentを使用すると、翻訳者は文法的に正しい文を作成し、言語の表現手段を自由に使用できます。 Fluentのおかげで、チェコ語の翻訳では、可能な$ count値に正しい複数形が使用されます。





チェコ語では、$ countの値が2、3、および4の場合、名詞には特別な複数形が必要です。



同時に、ソースコードまたはコードを変更する必要はありません。 チェコ語のローカライズのロジックは、他のすべてのプログラムのローカリゼーションに影響しません。 フランス語では、この文は英語と同じくらい簡単です： 









 tabs-close-warning-multiple = Vous êtes sur le point de fermer {$count} onglets. Voulez-vous vraiment continuer ?


非対称ローカリゼーションの概念は、20年以上にわたって多言語ソフトウェアを開発してきたMozillaの経験によって可能になった、流Fluな革新です。 Fluentの重要なアイデアの多くは、ICU（Unicodeの国際コンポーネント）およびXLIFF形式のMessageFormatから生まれました。







一見、Fluentは他のローカライズソリューションに似ており、複数の名詞や文法上の性別を持つ単語に単語の変形を使用できます。 対照的に、Fluentはローカライズに総合的なアプローチを採用しています。 Fluentは、翻訳が保存されるテキストファイル全体の構文を定義し、あるメッセージから別のメッセージへのリンクを許可することにより、これらのアイデアを開発します。







利用規約とリンク



Fluentファイルには多くのメッセージを含めることができ、各メッセージは翻訳者の言語に翻訳されます。 メッセージは、同じファイル内の他のメッセージ、または他のファイルからのメッセージを参照できます。 実行時に、Fluentはファイルをバンドルにマージし、そのセット内でリンクが解決されます。







投稿へのリンクは、一貫した翻訳を維持するための強力なツールです。 特定の翻訳を他の翻訳で再利用できるようになると。 Fluentは、再利用に適した用語と呼ばれる特別なタイプのメッセージをサポートしています。 用語識別子は、常にダッシュで始まります。 









 -sync-brand-name =  Firefox


定義された用語-sync-brand-nameは、他のメッセージから変数として使用でき、常に正しいテキストを返します。 用語は、スタイルガイドラインに従って均一性を導入するのに役立ちます。 たとえば、非公式のビルドやベータリリースを明確にブランド化するように変更できます。 







 sync-dialog-title = {-sync-brand-name} sync-headline-title = {-sync-brand-name}: The best way to bring your data always with you sync-signedout-account-title =    {-sync-brand-name}


文の途中で用語を直接使用すると、屈折言語、または他のテキスト大文字化規則のある言語で問題が発生する可能性があります。 この用語の意味は、コンテキストに応じていくつかの側面で定義できます。 次のイタリア語の用語-sync-brand-nameの定義を考慮してください。 







 -sync-brand-name = {$capitalization -> *[uppercase] Account Firefox [lowercase] account Firefox }


Fluentの非対称性により、イタリアの翻訳者はブランドを書くために2つのフォームを定義できます。 大文字オプション（デフォルト）は、独立した使用または文の先頭での使用に適しています。 用語が大きな文内で使用されている場合、大文字のパラメータを渡すことで小文字のオプションを要求できます。 







 sync-dialog-title = {-sync-brand-name} sync-headline-title = {-sync-brand-name}: il modo migliore per avere i tuoi dati sempre con te #      . sync-signedout-account-title = Connetti il tuo {-sync-brand-name(capitalization: "lowercase")}


この用語のいくつかの変形の定義は、多くの言語の文法規則に準拠できる便利な手法です。 次の例では、ポーランド語の翻訳者は偏角を使用して、sync-signedout-account-titleメッセージに正しい文を作成できます。 







 -sync-brand-name = {$case -> *[nominative] Konto Firefox [genitive] Konta Firefox [accusative] Kontem Firefox } sync-signedout-account-title = Zaloguj do {-sync-brand-name(case: "genitive")}


Fluentを使用すると、複雑な言語構成ロジックを必要な場所に埋め込むことができます。 同時に、翻訳は単純なままです。 Fluentは、単純な設計に複雑な実装を追加しません。 







 sync-signedout-caption = Take Your Web With You sync-signedout-caption = Il tuo Web, sempre con te sync-signedout-caption = Zabierz swoją sieć ze sobą sync-signedout-caption = So haben Sie das Web überall dabei.


流syntaxな構文



本日、Fluent Syntaxの最初の安定リリースを発表します。 これは、翻訳を保存するためのファイル形式の仕様であり、JavaScript、Python、およびRustのパーサーのベータリリースです。







上記の例では、Fluent構文のすべてを見ました。 技術的な知識がほとんどない人でも理解できるように作成されており、最小限のエラーでテキストをチェックおよび編集できます。 エラーリカバリは重要なポイントです。1つの誤った翻訳によって、ファイル全体またはそれに隣接する翻訳が破損することはありません。 コメントは、特定のメッセージまたはグループ全体のコンテキストを伝えるのに役立ちます。 翻訳には数行かかる場合があります。これにより、テキストの大きな断片の作業とマークアップが容易になります。







流Fluなファイルは任意のテキストエディターで開くことができるため、開発者とローカライザーの入力しきい値が低くなります。 この形式は、MozillaのオープンソースのPontoonソフトウェア翻訳プラットフォームで既にサポートされています。





Fluent Playgroundは、Fluentをブラウザーで直接テストするためのサンドボックスです。



構文の詳細については、 Fluent Syntax Guideを参照してください。 Fluent Syntax仕様で正式な定義を見つけることができます。 Fluentを試してみたい場合は、スニペットを共有するオンラインエディターFluent Playgroundを使用してください。
