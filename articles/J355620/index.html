Windows 10のすべてのバージョンについて。Threshold1、Threshold2、Redstoneとは

1年ほど前に、 Windows 8.1 （およびWindows Server 2012 R2）のバージョンに関する研究を共有しました。 この資料は議論を白熱させました。評価がゼロのユーザーにノートにコメントする機能に関するこのブログの制限がなかったら、それはさらに大きかったかもしれません。 私のメモがHabrからGTに転送されたので、私自身は記事にコメントしたり変更したりすることができなかったので、ポイントを逃し、ほぼ半年間資料を追加しませんでした。 ここで、新しいメモでWindows 10のアセンブリに関する情報の収集を開始したいと思います。そのうちのいくつかは、既にいくつかの推定値に従って3つ、他の推定値に従って4つあります。 このメモは、Windows Server 2016およびWindows 10のRedstone1ビルドのリリースで補充する予定です。



そもそも「Windows 10」は長い時間であり、これは以前のように1つのOSではなく、同社が今後開発するラインです。 つまり、実際には、Windows 10タブレットの背後で、MicrosoftはOSカーネルを変更し、新しいバージョンをグローバルアップデートとして提供しています。これは通常の意味ではOSの追加としてインストールされませんが、新しいOSとしてその上に置かれます。 現時点では、OSの2つのグローバルバージョン-Threshold1とThreshold2がありました。現在、Redstone1は、2016年7月下旬のリリース日でアクティブテスト中です。グローバルバージョンにはさまざまなアセンブリがあります。 現在インストールされているOSのバージョンは、Windows Updateから更新プログラムをインストールすることによりアップグレードされます。 一部のアセンブリは、ダウンロードと初期インストールに使用できる個別のイメージとしてリリースされますが、ほとんどは単に累積的な更新です。 今日は、OSのグローバルバージョンと、ゼロからダウンロードしてインストールできるアセンブリについて説明します。



そのため、2015年7月に、10240.16384.th1.150709-1700という番号のWindows 10のRTMアセンブリが組み立てられました。 2015年7月29日のこのアセンブリの画像は、MSDN / TechNet / VLSCに掲載されました。 8月からMedia Creation Toolを使用してダウンロードできます。 ネットワークでは、これをThreshold1または「バージョン1507、10240」と呼ぶのが慣例です。



2015年9月、アセンブリ10240.16393.th1_refresh.150909-1450には、MicrosoftがWindows Updateを通じてWindows 7およびWindows 8.1更新プログラムを通じてESDイメージとして配布した更新KB3081452およびKB3081455が含まれており、日の目を見ました。 このアセンブリへのまれな参照は、Threshold1 Refreshと呼ばれます。 このアセンブリのESDイメージは、Microsoftからダウンロードして、簡単なESD2ISOユーティリティを使用してISO形式に変換できます。



2015年11月12日、最初のグローバルなThreshold2更新または「バージョン1511、10586」（2015年11月）がビルド番号10586.0.th2_release.151029-1700でMSDN / TechNet / VLSCに表示されます。 これには、Edgeブラウザー、Cortana音声アシスタント、および写真、ビデオ、Xbox ONEアシスタントなどの主要なアプリケーションに対する主要な変更が含まれています。 アセンブリは、 メディア作成ツールを使用して誰でも利用できます。



2016年2月、大規模な累積更新プログラムKB3135173がリリースされました 。これには、以前にリリースされたThreshold2のグローバルバージョンのGDR更新と、Threshold1の元のグローバルバージョンのKB3135174の更新が含まれます。 2016年2月9日から入手可能な更新されたThreshold2イメージのビルド番号は10586.104.th2_release.160128-1934です（一部のソースは異なるバージョン-10586.103.th2_release.160126-1819を提供しています）が、MicrosoftのWebサイトで公式統計が入手できると信じています 。 Threshold2にアップグレードしなかったが、Threshold1のグローバルバージョンのままであった場合、3135174を更新した後、OSバージョンは10.0.10240.16683.th1.160130-1842になります。



MSDNに10586.104.th2_release.160128-1934としてリストされている2月の更新のThreshold2 Refreshは、名前にバージョン10586.0.th2_refresh.160212-2000が記載されたESDイメージの形式で、メディア作成ツールから入手できます。 冗談の正確な名前はわかりませんが、ESDとMSDNの画像内のファイルはまったく同じです。 OS内部のWinverはバージョン10.0.10586.104を報告します。



このメモの執筆時点では、これはイメージとしてダウンロードできるWindows 10の最後のバージョンですが、Threshold 2アセンブリのいずれかをインストールし、Windows Updateを実行すると、OSバージョンを10.0.10586.164にアップグレードする3月の累積更新KB3140768を受け取ります。 これは毎月出てくる定期的な更新であり、記事ではそれらを追跡しません。レジストリキーHKLM \ SOFTWARE \ Microsoft \ Windows NT \ CurrentVersion \ BuildLabExでPCのバージョンを確認できます



次のグローバル更新Redstone1は、Windows Server 2016のリリースと同時に2016年7月末に表示されます。そのインストールは、基本的にWindows UpdateでESDイメージをロードし、現在のOSをアップグレードします。 もちろん、このバージョンはMSDN / TechNet / VLSCとメディア作成ツールの両方に表示されます。 カルマが肯定的で、読み取り専用モードに戻らない場合は、ノートに変更を加えようとします。



更新：MSDNの2016年5月23日、Threshold2 Refreshイメージ（MicrosoftはWindows 10、Ver。1511、2016年4月更新と呼びます）の番号は10586.164です。 内部にはビルドファイル10.0.10586.162.th2_release_sec.160223-17 28があります。 画像は5月末にアップロードされましたが、実際には2月末に収集されたため、画像に残されなかったいくつかの重大な脆弱性を修正します。 このアップデートの執筆時点（2016年5月30日）で、TH2のバージョンを10586.318- KB3156421に 、TH1を10240.16854- KB3156387に増やす5月の累積的なアップデートがすでに利用可能です。



更新：2016年7月18日、Redstone1ブランチの最終バージョンはInsiderアセンブリのサブスクライバーが利用可能です：14393.0.rs1_release.160715-1616、数日以内にスローファーストからファーストに切り替わります。リリースプレビューではリングが使用され、8月2日からWindows Updateで使用可能になります。
