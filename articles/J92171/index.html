非常に簡単な指示FreeBSD 8.0リリース+ AnyData Adu-500a

私は携帯型手術室をそれほど昔に世話しませんでした。 その選択は、FreeBSDが多かれ少なかれ私に知られていることにかかっていました。 フラッシュドライブに置いて、すべてをセットアップしました。 ただし、Skylinkモデムはその場で起動しませんでした。 インターネットで解決策を探すのは便利でしたが、毎秒「このモデムが機能するには、これらとこれらのモジュールを捨ててカーネルを再構築する必要があります。 はい、フラッシュドライブは動作を停止しますが、モデムは動作します。 そして、Linuxモジュールを移植して、モードをsidiromからモデムに切り替えます。 しかし、これはオプションではありません。 彼は問題の解決を無期限に延期しました。 そして今日、たまたまこの問題に戻り、すべてがはるかに簡単かつ迅速に解決されることがわかりました。





まず、モデムは汎用コアで動作します。 第二に、Linuxポートなし。 そして、これはまさに私が探していたものです。 したがって、ここでは簡単な指示です。



1）スカイラインで動作するようにppp.confを構成します。

2）モデムを挿入します。

3）コマンドcamcontrol eject cd0を実行します

（ここで、cd0は接続後にモデムが決定された方法に置き換えられることに注意する必要があります）

4）ppp -ddial skylinkコマンドを実行します。

5）ifconfig tun0コマンドでアドレスが受信されたかどうかを確認します。

（ここでも、以前にトンネルが発生していない場合のみ、それ以外の場合はtunK + 1で、Kは最後のトンネルの番号です）



次に、お気に入りのブラウザを起動してお楽しみください。**



注：

*）

#cat /etc/ppp/ppp.conf

default:

set log Phase Chat LCP IPCP CCP tun command

disable pred1 deflate deflate24 protocomp acfcomp shortseq vj

deny pred1 deflate deflate24 protocomp acfcomp shortseq vj

set speed 115200

set timeout 160

enable dns



skylink:

set device /dev/cuaU0

set phone \#777

set dial "ABORT BUSY ABORT NO\\sCARRIER TIMEOUT 5 \

\"\" AT OK-AT-OK ATE1Q0 OK \

\\dATDT\\T TIMEOUT 40 CONNECT"

set login

set authname mobile

set authkey internet

set ifaddr 10.0.0.1/0 10.0.0.2/0 255.255.255.0 0.0.0.0

add default HISADDR









**）これまでのところ、通常の速度をテストすることはできませんでした。この方法では最大速度が得られない可能性がありますが、非常に簡単です。 私の装甲地下で両方向で〜512 kbpsをキャッチするもの。 家に帰って、路上で測定してみます。



***）ソリューションは偶然発見されましたが、Yandexの最初のリンクで発見されました。 検索クエリをできる限り正確にすると、他のことは言わないでしょう。
