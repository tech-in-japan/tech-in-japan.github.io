クライアント側のファイル圧縮（tarおよびLZMA）

Artel BrowserPlus （Yahoo）は、クライアント側のファイル圧縮用のAPIをリリースしました。 次のスクリーンキャストで説明するように、javascript APIを使用すると、LZMAまたはtarを使用してクライアントでファイルを圧縮および圧縮できます。





ロイドヒライエル が Vimeoに投稿した「クライアント側の圧縮」



実装について詳しく読むか、デモをライブで体験できます （BrowserPlusが必要です） 。





クライアントサイドパッケージング用のBrowser Plus



このAPIの使用は難しくありません。必要なメソッドを呼び出して、ファイルを送信します。
BrowserPlus.Tar.tar（{'files'：files}）;

BrowserPlus.LZMA.compress（{'files'：files}）;
これらの拡張機能はどちらも GitHub - LZMAおよびtarで利用できます。



翻訳のニュアンス：（少なくともダールの時代以降）偉大なロシア語で長生きしてきた「prilad」という言葉は、ここでは技術用語「アドオン」の翻訳として機能します（ ウクライナ語の よう に 「デバイス」ではなく ）、 「artel」という言葉は開発チーム（元の- 「チーム」）。
