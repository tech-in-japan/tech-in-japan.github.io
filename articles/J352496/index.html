Googleはgoo.glを閉じます。 Firebase Dynamic Links Replaces





2018年3月30日以降、goo.gl URL短縮はサポートされなくなりました。 2018年4月13日から、既存のユーザーのみがgoo.glコンソールで短いリンクを作成できるようになります。 goo.glを完全に停止する2019年3月30日まで、分析を表示し、CSV形式の短いリンクに関する情報を最大1年間ダウンロードできます。 以前に作成されたリンクは、引き続き宛先にリダイレクトされます。



2009年にGoogle URL Shortenerをリリースし、人々がより簡単にリンクを共有してオンライントラフィックを測定できるようにしました。 それ以来、多くの一般的なURL短縮サービスがあり、インターネット上のコンテンツの検索方法は劇的に変化しました。デスクトップWebページから、アプリケーション、モバイルデバイス、ホームアシスタントなどです。



作業の方向を変えるため、今後数週間でgoo.glのサポートを放棄し、 Firebase Dynamic Links （FDL）に置き換えます。 FDLは、iOS、Android、またはWebアプリケーションの任意の場所に既存および潜在的なユーザーを送信できるスマートURLです。 将来の製品の成長と改善を楽しみにしています。 ほとんどのgoo.gl関数は最終的に閉じますが 、 既存のすべてのリンクは引き続き目的の宛先にリダイレクトされます。



ユーザー向け



2018年4月13日以降、匿名ユーザーと、今日まで短いリンクを作成したことがないユーザーは、 goo.glコンソールから新しい短いリンクを作成できなくなります。 新しい短いリンクを作成する場合は、 Firebase Dynamic Linksを使用するか、 BitlyやOw.lyなどの一般的なサービスを代わりに使用することをお勧めします。



goo.glの短いリンクがある場合、goo.glコンソールのすべての機能を2019年3月30日まで1年間使用し続けることができます。その後、コンソールを閉じます。 この期間中、goo.glコンソールを使用して、すべてのショートリンクとその分析を管理できます。



2019年3月30日以降、すべてのリンクは引き続き指定された場所にリダイレクトされます。 既存の短いリンクはFirebaseコンソールに移行されませんが 、goo.glコンソールからCSV形式でリンク情報をエクスポートできます。



開発者向け



2018年5月30日以降、今日までShortener API Shortener URLにアクセスしたプロジェクトのみが短いリンクを作成できます。 新しい短いリンクを作成するには、 FDL APIをお勧めします。 短いFDLリンクは、ユーザーのプラットフォームを自動的に検出し、必要に応じてユーザーをインターネットまたはアプリケーションに送信します。



既にURL Shortener APIを使用してgoo.glの直接リンクを管理している場合は、APIの使用を停止する2019年3月30日まで1年間継続して使用できます。



消費者と同様に、 すべてのリンクは2019年3月30日以降も目的の宛先にリダイレクトされ続けます。 ただし、既存の短いリンクはFirebaseコンソール/ APIに移行されません。



URL Shortenerは素晴らしいツールであり、作成したことを誇りに思います。 将来に目を向ける際、Firebase Dynamic Linksの機能に焦点を当てています。特に、動的プラットフォームの検出や、アプリケーションのインストールプロセスから外れるリンクに関しては、これに焦点を当てています。 私たちもあなたを願っています！
