Windowsストアアプリを宣伝するための7つのヒント





多くのアプリケーション開発者は、たとえば、ソーシャルチャネルでアプリケーションを宣伝したり、アプリケーションストア（たとえば、Windowsストア）でプロモーションを行ったりするなど、アプリケーションのプロモーションを支援するリクエストを受け取っています。

マーケティング担当者が最初に尋ねる質問は次のようなものです。「あなたは何をしましたか？」技術専門家は同じ質問をしますが、目的は異なります。 この記事では、それらと他の両方からいくつかのアドバイスを集めようとしました。







マーケティングのヒント



1.アプリケーションについて聴衆に伝えます。



どんなに些細なことでも、大企業も中小企業も、個々の開発者も、作成したアプリケーションについて、ウェブサイトの訪問者やソーシャルネットワークの購読者など、自分のネイティブオーディエンスと話すのを忘れていることがよくあります。







実験として、最も人気のある上位5つのオンライン映画館を調べて、サイトで独自のアプリケーションへのリンクを見つけてください。 結果は次のようになります。

  • ポータルAでは、すべてのリンクが地下室に隠されていますが、ご存知のように、そこにたたく必要があります。 同時に、ストアへのリンクは、古いWindows / Windows Phoneアイコンを使用して「Windows Marketplace」と呼ぶことができます。
  • ポータルBでは、リンクは地下に隠されていますが、たとえば、Windowsの場合、Windows 8のアプリケーションのスクリーンショットが存在するという事実との間には明確な不一致がありますが、アプリケーションはありません（サービスによって隠されています）。
  • Bポータルには、アプリケーションはありますが、アプリケーションへのリンクはまったくありません。
  • Gポータルにもリンクはありませんが、アプリケーションもあります。
  • 地下のポータルDには、Windows Phone用のアプリケーションへのリンクがありますが、モバイルセクションには記載されていません（これは、会社の異なる半球間の明らかな矛盾です）。


この状況は常に観察されます。



ヒント1.認識可能な方法で、ユーザーがアクセスできる場所にストアへのアプリケーションへのリンクを配置します。







現在、人気のあるストアのアプリケーションについては、その説明ページのWebバージョンが利用可能です。 ターゲットプラットフォームで開く場合、ユーザーをストアインターフェースに転送します。ストアインターフェースから、アプリケーションをデバイスにダウンロードすることがすでに可能です。



リンクを配置する最も正確で認識可能な方法は、対応するストアのアイコンを使用することです。



  • Windowsストア



  • Windows Phoneストア







ところで、ユーザーにスクリーンショットを表示する場合、対応するプラットフォームに関連するこの「内部」の特定のデバイスを実行することをお勧めします。 一部の匿名画像を提供していますが、特定の製造元（Nokia Lumiaなど）のデバイスを使用することもできます。



このような写真を見る大勢の視聴者がいる場合、これはデバイスメーカーとの相互プロモーションの議論の対象にもなります。 あなたは彼らのものであり、彼らはデバイス内のアプリケーションです。



ヒント2.会社または従業員に人気のブログ、ツイッター、VK / FB / ...アカウントがある場合は、アプリケーションまたはそのアップデートをリリースしたことをお知らせください。



プラットフォームの新しいアプリケーションのリストをときどき見て、アプリケーションの外部に誰にも教えていない有名な著者/企業のアプリケーションを予期せず見つけます（リリースは言うまでもありません）。 残念なことに、誰も知らない素晴らしいアプリケーション。







多くの場合、このようなアプリケーションのリリースは、プレスリリース、会社のウェブサイト、またはツイッターやフェイスブックのページから見つけることができません。 人気のあるサイトのレビューに従うか、ストアで新しいアプリケーションを監視するだけです。



多くの企業にとって、「自分のアプリケーションのリリースについてオンラインユーザーに伝える」という平凡な考えは、最も極端な場合にのみ使用できる手法であるように思われます。まったく使用しないほうがよいでしょう。



進歩的な企業は長い間（日付に注意を払って）、ユーザーのために何をするかについて話すことをheしませんでした。







インターネット上で大勢の視聴者がいる場合、これがユーザーを引き付ける最も効果的な方法であることを示しています（おそらくストア自体で直接プロモーションを行った後）。



（そして、私たちはユーザーへの告知のターゲットを絞ったメールについて話していません！なんとかしてみてください。）



ヒント3.他のアプリケーションのユーザーにアプリケーションについて伝える



多くのユーザーが既に持っている他の人気のあるアプリケーションを既に持っている場合、これは彼らとのコミュニケーションのための潜在的なチャネルがあることを意味します。







あなたは平凡なものから始めることができます：アプリケーションのセクションで、他のアプリケーションのリストを提供し、追加の相互プロモーションを提供します。



さらに（ゲーム会社が定期的に行っている）さらに進んで、より積極的な形で新しいアプリケーションやゲームについて話すことができます。 たとえば、ピンクの象に関する人気のあるおもちゃがあり、2番目のシリーズを別のアプリケーションとしてリリースする場合、確かに、十分なプレイをしたがデバイスからゲームを削除しなかったユーザーが大勢います。



彼らに通知を送るか、タイルを更新して、あなたの物語がいつ続くかを知らせます。



2.ユーザーに評価とレビューを促す



おそらく推測されるように、ストア内の特定のディレクトリにおけるアプリケーションの位置は、ダウンロード数に大きな影響を与える可能性があり、そのダウンロード数は、何らかの形の利益に変換できます。







特定のランキング式を公開しているストアはありませんが、ダウンロード、評価、レビュー、およびそれらのダイナミクスが含まれていることが知られています。 ユーザーが意見を変更する（再投票する）などのパラメーターや、特定の評価でレビューすることが役立つと考えるユーザーの数も重要です。



一般に、レビューと評価が優れていて優れているほど、アプリケーションにとっては良い（開発者との平凡なフィードバックチャネルは言うまでもなく）ため、このチャネルに影響を与える必要があります。



多くのユーザーは、アプリケーションに投票したりフィードバックを送ったりできることを知りません。 もちろん、「すべてが失われ、すべてが壊れる」場合、ユーザーは自分の失望を開発者に伝える方法を見つけますが、日常生活では、ほとんどのユーザーは格付けのアイデアを重要だとは考えません。



これは、ダウンロード時、さらにはアプリケーションやゲームの購入時に、平均評価、数、および一般的な雰囲気が彼らにとって非常に重要な役割を果たすという事実にもかかわらずです。



ヒント4.評価とレビューの重要性をユーザーに思い出させます。



多くの場合、ダウンロード数は多いものの、レビューや評価が非常に少ないアプリケーションが表示されます。 ユーザーはダウンロードの数を知らないため、少数の評価がこのアプリケーションを使用しているユーザーがいないことを示唆しています。 優れたアプリケーションの場合、これはよくありません。







同時に、各プラットフォームには、アプリケーションストアに対応するAPIがあります。これにより、ユーザーをアプリケーション評価ページに移動できます（詳細は以下を参照）。



考慮すべき重要事項：

  • 評価やレビューでユーザーを頻繁に煩わさないでください。 この機能を実装するときは、アプリケーションの使用とリマインダーの適切なバランスを見つけてください。

  • ダイアログボックス、ポップアップメッセージを表示したり、アプリケーションの永続的なインターフェイスに直接リンクを配置したりするなど、さまざまなインターフェイスソリューションで投票への招待を表示できます。
  • リマインダーを表示するポイントを注意深く監視します。 アプリケーションが機能しない場合、これは否定的な評価を引き起こす可能性があります。
  • ユーザーは、良い評価を与えるためにオファーを否定的に受け入れる場合があります。 何かで彼をやる気にさせてください。




ヒント5.ユーザーのレビューに返信する。



レビュー内およびソーシャルネットワークまたはブログの両方で、ユーザーがアプリケーションについて書いている内容を注意深く監視します。 回答、説明、修正、追加、改善。







ちなみに、ビルドカンファレンスで、今後数か月以内に、ユーザーレビューに直接応答する機能がWindowsストアで提供されることが発表されました。



3.アプリケーションを新しいユーザーに宣伝する



クロスリンク（Adduplex経由、有料トラフィック（MS広告など）、テーマポータルでのレビューなど）で、ユーザーを引き付けるための従来のマーケティングの機会がすべて既に有効になっているため、明示的なアドバイスなしでそれを行います。 。



私たちはまだ1つのアドバイスをしますが！



ヒント6.検索に統合します。



ご存知かもしれませんが、ユーザーはアプリケーションを名前ではなく関連する関連付けで検索することがあります。 彼らはあなたのアプリケーションを気にかけないかもしれませんが、その中身は重要です。 これらの場合、アプリケーションを見つけることが重要です。



これはまず、キーワードや説明などのありふれたものに適用されます。







しかし... ...また、検索エンジン（Bing）にアプリケーションの内容とその方法を伝えると、さらに多くの結果を得ることができます。



このために、Webサイトとアプリケーションをリンクするための特別なメカニズムがあります 。 ビルド会議のレポート「 検索結果でアプリをプロモート 」で詳細を確認できます。



これは、Bing（米国）の存在感のある市場をターゲットにしている場合に特に意味があります。



これで、技術専門家にフロアを渡します。



技術的なコメント



相互に関連する2つのポイントについてコメントします。アプリケーションへのリンクの取得と、レビューのあるページへのリンクの受信です。



アプリのWebリンク



Windows 8.xのアプリケーションへのリンクは次のようになります。

apps.microsoft.com/windows/app/en {GUID}


リンク全体は、開発者ポータルの情報パネル（アプリケーション情報セクション）に示されており、証明書にも記載されています。



Windows Phoneのアプリケーションへのリンクは少し異なります。

windowsphone.com/s?appId= {GUID}


また、開発者のポータルで公開されたアプリケーションに関する詳細を含む対応するセクションにもあります。



注：WindowsとWindows Phoneの2つのGUIDは異なりますが、 店舗の段階的な統合により 、いわゆる ユニバーサルアプリ 、およびアプリケーションを相互に関連付ける機能（ユーザーは、異なるデバイス用のアプリケーションがあることを確認し、それらのアプリケーションに一度支払うことができます）。



ストア内のアプリに移動します



開発者は、ストアの特定のセクションにユーザーをリダイレクトするための特別な「プロトコル」APIを使用できます。



Windows 8.x



Windows 8の場合、リンク構造は次のとおりです。

ms-windows-store：{アクション}？{パラメーター}




可能なアクションとパラメーター：

  • PDP-アプリケーションの説明ページ、パラメーター：PFN-アプリケーションパッケージファミリの名前（パッケージング-パッケージファミリ名セクションのアプリケーションマニフェストで表示でき、APIからも利用可能）;
  • レビュー -レビューまたは評価を追加するためのページ、パラメーター：PFN-パッケージファミリーの名前（アプリケーションがインストールされていない場合、アプリケーションページが開きます）;
  • 発行者 -アプリケーションの発行者のページ、パラメーター：名前-発行者の表示名。
  • 検索 -ストア内の検索結果を含むページ、パラメーター：query-検索クエリ。
  • 更新-パラメーターなしのストア内の更新ページ。




Windows.System.Launcher.LaunchUriAsyncメソッド（JSのlaunchUriAsync、小さな文字）を使用してリンクを開くだけです。 たとえば、アプリケーションのページを開くには、次の呼び出し（JS）を使用できます。 



var uri = new Windows.Foundation.Uri("ms-windows-store:PDP?PFN=" + Windows.ApplicationModel.Package.current.id.familyName); Windows.System.Launcher.launchUriAsync(uri);




Windows Phone



このタスクのWindows PhoneのプロトコルAPIはわずかに異なります。

  • zune：ナビゲートしますappid = [アプリID]-アプリケーションページに移動します。
  • zune：reviewapp-起動中のアプリケーションに関するフィードバックを投稿するためのページ。
  • zune：reviewapp？appid = app [アプリID]-特定のアプリケーションに関するフィードバックを投稿するためのページ（アプリのプレフィックスに注意してください）;
  • zune：search？keyword = [search keyword]＆publisher = [publisher name]＆contenttype = app-ストアで検索を開始し、すべてのパラメーターはオプションです。




ページを開くには、Windows.System.Launcher.LaunchUriAsyncと同じ非同期メソッドを使用する必要があります。



さらに、対応する既製の「タスク」（Microsoft.Phone.Tasks）により、ストアの目的のセクションに移動できます。

  • MarketplaceDetailTask​​-アプリケーションの詳細。
  • MarketplaceReviewTask-アプリケーションに関するフィードバック。
  • MarketplaceSearchTask-ストアで検索。
  • MarketplaceHubTask-特定のセクションを開きます。




評価とレビュー



上記の説明から明らかなように、アプリケーションを評価してレビューを書くためのフォームでページを開くには、1行または2行のコードで十分です。



JS、Windows 8.x 



 var uri = new Windows.Foundation.Uri("ms-windows-store:Review?PFN=" + Windows.ApplicationModel.Package.current.id.familyName); Windows.System.Launcher.launchUriAsync(uri);




C＃、Windows 8.x 



 Windows.System.Launcher.LaunchUriAsync(new Uri(String.Format("ms-windows-store:REVIEW?PFN={0}", Windows.ApplicationModel.Package.Current.Id.FamilyName)));




ご覧のとおり、構文はほとんど同じですが、メソッドとプロパティのケース名の違いに注意してください（各言語に典型的な慣習が使用されています）。



アプリケーションからこのようなコードを呼び出す場合、いくつかのニュアンスを考慮すると便利です。

  • ユーザーにアプリの評価を依頼する前に、インターネット接続を確認してください。 ストアを操作するには、ストアの対応するWebサービスとの対話が必要です。
  • ユーザーの選択を覚えておいてください（たとえば、投票に同意しました）。 ユーザーは複数のデバイスを持つことができるため、複数回投票するように強制することは意味がないので、データ同期（ローミング）を使用することに注意してください。


ところで、ユーザーがあなたのために特別に設定したものをプログラムで確認することはできませんので、ユーザーの心理的なトリックと動機について考えてください。



特定の数のアプリケーション起動に対してアプリケーションを評価するように求めるリマインダーを表示する場合、従来のスキームは次のとおりです。

  • 最初の実行：投票フラグをfalseに設定し、実行カウンターをリセットします（設定を保存します）。
  • 打ち上げ：
    • ユーザーがまだ投票していない場合は、カウンターを増やします
    • カウンターが特定の数Xの倍数である場合、投票の提案を表示します

      • ユーザーが同意した場合、彼は投票したと思われます
      • ユーザーが拒否した場合、後でユーザーに戻ります
