Microsoft Sharepoint Conference 2010への出席報告

MS Sharepointの2010バージョンのリリースの際に、昨日Izmailovo Hotelで開催された会議の印象を共有したいと思います。

それで、それはすべて、いつものように、参加者の登録から始まりました。 ちなみに、モスクワとロシアの他の都市（サンクトペテルブルク、ニジニノヴゴロド）の両方から多くの人がいました。 バッジと、今後のレポートのスケジュールと説明が記載されたパッケージと、パートナー企業の広告リーフレットをいくつか受け取ったので、全体レポートに行きました。



全体報告



会議は、ロシアのSharepointユーザーグループ（RUSUG）を代表してOksana Prostakovaを歓迎することで開始されました。一方、メインキャラクターは、Microsoft本社のSharePoint開発チームの責任者であるArpan Shahでした。 彼は非常に熱心に語り、結果として得られた製品を誇りに思っており、2010バージョンが豊富なさまざまなチップと機能に満足していることは明らかでした。 彼は英語を話し、誰もが自動翻訳者に夢中になっているという事実にもかかわらず、彼は存在するすべての人に理解されるような方法で表現を選択しようとしました。 その過程で、彼はデモサイトの実例で多くのことを示しました.1つのモーションで多言語を有効にする機能、オンザフライでページを変更する機能、OfficeアプリケーションでSharepointをさらに繁殖させるように設計された新しいリボンインターフェイス...



コーヒーブレイクの後、聴衆は3つの部屋に分かれ、新しいプラットフォームのさまざまな側面と革新についての講義と並行して進みました。 私が訪れた人についてお話しします。



ショック療法なしの近代化



私にとって、既存の環境を更新する技術について学ぶことは、新しい検索や外部データを扱うことよりもはるかに関連性があったため、ためらうことなく3番目の部屋に行きました。 Alexei Krashennikovのプレゼンテーションは私の期待に応えました：すべてが無痛で魔法の杖の波でできるという約束はありませんでしたが、代わりに、3つの最も一般的な環境オプションの3つの異なるモデルが説明されました））、移行プロセス中に見つけることができ、事前に準備する必要があります（たとえば、Webフォームを介した承認は、追加の操作なしでは機能しません）。 当然のことながら、出席者は、2007年のインストールの特定のケースと、プレゼンテーション後の2010プラットフォームへの移行に関する質問を開始することに失敗しませんでした。 これらの質問のすべてに回答したわけではありませんが、これは理解できることです。もちろん、各移行は個別に検討する必要があります。 しかし、少なくとも今、私の農場の今後の移転の計画をより明確に概説することができます。



Sharepoint 2010およびPowershell



私の個人プログラムの次の問題はこの講義でした。 Powershellは、Microsoft環境で最も素晴らしい快適な管理/開発ツールの1つであると考えています;その柔軟性、シンプルさ、使いやすさを賞賛します。 このプレゼンテーションでは、Alexander RomanovがpowershellはUnixシェルスクリプトに合わせて高度に調整されており、これも私にとってプラスの1つであると述べました。2007年には、Sharepointとpowershell 1.0の使用はそれほど便利ではありませんでした。外部アセンブリの接続が必要で、通常の。 。 2010 SPおよびPowershell 2.0の出現により、482個ものコマンドレットがSharepointと直接連携するように導入され、管理者と開発者のニーズの大部分をカバーしています。 ここでは、バックアップ/復元、サービスの操作、およびリストとドキュメントライブラリの編集、アイテムの読み込み、プロパティの変更などがすべて非常においしいように見えます。 Alexanderは、サイト、リスト、アイテムの作成、Sharepoint 2010およびすべての関連サービスの再起動など、PSを使用して自動化されたいくつかのシナリオを示しました。 残念ながら、プレゼンテーションは割り当てられた時間に収まらなかったため、質問する時間はありませんでした。



SharePoint Workflow 2010に基づくビジネスプロセスの自動化



次のエピソードは、ミハイル・ボンダレフスキーによる公演でした。 残念なことに、それはあまり私を感動させませんでした（そして、良い開発者は必ずしも良いスピーカーとは限らないという考えをもう一度確認しました）：ミハイルはやや緩慢で、ほとんどがスライドからテキストを読み、説明されていることをゆっくりと示しました（特に、ワークフローモデルの作成） Visio 2010およびこのモデルからSP Designer 2010への展開で）、明らかな間違いを定期的に許可しました。 一般的に、講義の途中で、私はそれが価値がないと判断し、隣の部屋に引退しました。



SharePointのトラブルシューティング



Oksana Prostakovaのプレゼンテーションでは、Sharepointの新しいバージョンのバグに対処する方法を説明しました。 バージョン2010は、この領域で明らかに重要な変更を受けています。 共通ログ（ULS、Unified Logging Service）とそれを操作するためのツールが登場しました。 これで、特定のページに関連付けられた相関IDによって特定のページに関連するエラーを見つけることができます。 開発者ダッシュボード（ページの読み込みと処理に関する開発者向けの有用な情報を表示するパネル）やヘルスモニター（ファームの状態を表示し、管理者によって割り当てられたルールに従って特定のアクションを実行するか、展開時に最初に設定するミニSCOMなどのツールが登場しました） Oksanaは、これらの革新の最も有用で興味深い機能について説明し、プレゼンテーションで判断すると、新しいバージョンのSharepointの診断がはるかに高速かつ簡単になります（少なくとも、2007 Sharepointの「予期しないエラー」は過去のものであることを願っています）。



SharePoint 2010のブランディング



イリヤ・ボイコは、Sharepointページを「自分用に」カスタマイズする方法について話しました。 彼は、最も単純な方法から始めました。2010年バージョンでは、Powerpointテーマから色とスタイルを取得し、このテーマを手首を軽く動かすだけでSharepointノードに適用できます。 次に、CSSスタイルを変更することにより、ノードの外観を変更するためのもう少し洗練された方法の話が来ました。 パワーポイントテーマを使用してセレクターにオーバーライドを割り当てることができる特別な「コメント」がCSSテキストで言及されました。 さて、結論として、イリヤは.masterページの変更について話しました。これにより、ページを根本的に変更できます（特にブロックを配置します）。 Sharepoint 2010では、要素のクロスブラウザーレイアウトがようやく登場しましたが、これは喜ばせざるを得ません.2007年、テンプレートをなめる方法に関係なく、非常に迷惑であり、アプリケーションの後でも、生成された恐ろしいHTMLプラットフォームによって破損することがありました。



IT Proの円卓会議



このラウンドテーブルで述べられているトピックは、一般的に「ネットワークファイル共有とExchangeパブリックフォルダーからSharePointへの移行の経験と望ましさ」でした。 トピックはもちろん関連性がありますが、定足数はそのトピックに蓄積されていなかったため、主催者はそれをあらゆるトピックに対する一般的な質問と回答に拡大することを決定しました。 さらに1時間半、専門家から質問が寄せられました。プラットフォームに関する一般的な質問と、個々のボトルネックや問題に関する具体的な質問の両方があり、専門家はさまざまな成功を収めました。 マキシム・クルプノフは特別な需要がありました-質問の3分の1以上が個人的に彼に送られたことが判明しました:)しかし、他の人は遅れを取りませんでした。

実際、短所は特にありませんでした。 確かに、最初の講義の前に、ホールとの混乱がありました：ドアの上のタブレットは現実に対応していなかったので、大衆は最終的にすべてが落ち着くまでホール2、1、3の順に落ち着きました。 もう1つの小さなマイナス点は、何らかの理由で「専門家のゾーン」に、そこに着くたびに魂がなかったことです。 たぶん私はただ不運だったかもしれません:)、しかし、私はまだすぐに私に興味のある人々を傍観者で見つけて、私に興味のあるトピックに関する質問で私を攻撃しました。
