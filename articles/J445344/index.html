OpenVoxユニファイドコミュニケーションプラットフォーム



なんて大きな見出しだとあなたは言います。 アスタリスクの新しいPBXメーカーですか？ そうではありませんが、機器は非常に新鮮で興味深いものです。







今日、私はOpenvoxユニファイドコミュニケーションシステムについてお話ししたいと思います。メーカーはこれらのまさにコミュニケーションを結合することについて独自のビジョンを持っているようです:)







機器メーカーのOpenVoxは、ゆっくりですが確実に完全なモジュール構造に向かっています。 最初に、彼はモジュールとその番号のさまざまな組み合わせを使用できるGSM機器を作成し、次にアナログゲートウェイが登場し、最後に必要なほぼすべての電話接続規格をサポートする新しいプラットフォームが導入されました：FXO / FXS / E1 PRI / BRI / GSM / 3G / LTE

Intel CeleronプロセッサVS-CCU-N2930AMを搭載したモジュール





はい、はい。 これは、4つのコアと最大2.16 Ghzの周波数を備えたCeleron N2930プロセッサをベースとした本格的な64ビットコンピューターです。 デフォルトでは、SO-DIMMメモリバーは2 GBに設定されていますが、DDR3L 1333を最大8ギガバイトまで拡張できます。

16 GBの容量のSSDドライブがボードにはんだ付けされています。 10/100 / 1000Mbと10 / 100Mbの2つのネットワークインターフェイスを使用できます。 外部モニター用の1つのVGA出力と、たとえばバックアップのアンロードまたは会話の保存用の2つのUSBインターフェイス。

十分な内部メモリがない場合は、VS-CCU-500HDDハードドライブモジュールを使用して、次のように拡張できます。



メーカーはデフォルトで500 GBを出荷していますが、最大2 TBの容量のディスクを問題なくインストールできると思います。







そして、インストールされたソフトウェアに徐々に近づきました。

このモジュールは、このシャーシの他の（3G / FXO / FXS / E1）と同様に、完全に自律的です。 個別にダウンロード、更新され、個別のIPアドレスを持ちます。 VS-CCU-N2930AMの場合、個別のネットワークインターフェイスも。







Openvoxは、ElastixプロジェクトのフォークであるIssabelのオープンなユニファイドコミュニケーションシステムを推進しています。 実際、有名なElastixとほとんど変わらないため、Issabelのレビューを行う必要はないと思います。

ユースケース









この図は、システム内のすべてのプラグインが独自のIPアドレスを持ち、個別に管理されることを示しています。 ソフトウェア（FreePBX / Asterisk / Issabel）では、デジタル、アナログ、またはモバイルのすべての回線を接続し、SIPトランク経由で接続します。

これは非常に便利です。将来、何らかのクラウドベースのPBXプロバイダーを突然使用したい場合は、インフラストラクチャが準備できます。







おわりに



このシステムはコンパクトでエネルギー効率が高く、デバイスを1つにまとめたい中規模および大規模企業に適しています。 現時点では、これらすべてのモジュールの自動構成が十分ではありません。つまり、独自のPBXソフトウェアが急激に不足しています。

開発の正しいベクトルは、独自のゲートウェイ/電話/ハードウェアモジュールを自動的に構成するための独自のソフトウェアアドオンを備えたFreePBXであると考えています。







ソリューションのコストは非常に手頃です。 シャーシ〜400ドル、プロセッサーを搭載したモジュール549ドル、モジュールE1 549ドル、4 GSMライン-420ドル、FXO 4ラインとFXS 4ラインのモジュール-240ドル
