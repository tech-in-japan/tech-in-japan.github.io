Microsoftの新しいOSでのCreativeの新しいサウンド

Habrへの招待を受けた記事を公開することにしましたが、それでも招待のためではなく、問題に注意を払うためにそれを書きました。



ずいぶん前、先週の金曜日、または数年前、私はエアロインターフェースの美しさに敬意を表し、コンピューターに眺望を入れました。それはその後創造的な監査2でした。しばらくして、音楽を聴きながら（ foob​​ar2000を通して）音に何か問題があったということです。高周波はどういうわけか「きれい」に聞こえましたが、それらには十分な詳細がありませんでした。 私が最初にしたことは、Vistaでサウンドを変更するというトピックに関するgoogleであり、類似するものは見つかりませんでした。 それは奇妙だと思った、または私は何か聴聞会や他の人と一緒にいる。 それとも、誰も創造的なVista製品と優れた音響を同時に使用しませんか？ :)理由を見つけるために何らかの方法でこれを確認する必要があると決めましたが、その時間はありませんでした。さらに、すぐにXPに戻り、Vistaのrawドライバーと新しいサウンドサブシステムのすべてをメンタルダンプしました。

時間が経ち、X-Fi Titanium（PCI-E用）を入手し、ここで「7」が間に合いました-私はそれをすべて一緒に試してみることにしました...再びそれは素晴らしいです-再び音に何か問題があります（音が異なるという事実にもかかわらず） 、同じ会社のものであり、OSが更新されましたが、再び6.1でしたが、7ではありません。 Googleは再び登り、ドライバーに音声をインストールした後、誤って設定されているというメッセージを数回目にしました。 私は設定に登り、高周波数を下げ、それが助けになったように見えましたが、...それはまさにその「いいね」でした-XPの再起動はこれが事実ではないことを示し、すべて正常に聞こえました。 私はもっ​​と真剣に仕事に取りかかることに決めました。RMAA6.2.3をインストールし、集団農場滑車とシンガーミシンからオーディオケーブルを構築しました。その一端はフロント出力に、もう一端はリニア入力に固定されていました。 無料の週末をたくさんのテストに費やしました。その結果がここにあります。



テスト中に、いくつかの詳細が明らかになりました。

詳細1.ケーブルが本当に悪かった（バンドルが最適な素材ではない）か、サウンド出力が独自の入力になり、浮遊電流があったために問題が発生した。 zvukovuhi、他にケーブルはありません。 しかし、少なくとも、サンプリングレートが変更されたテスト間で一時的にケーブルを取り外すと、結果に大きな影響がありました。



詳細2.録音ソースを切り替えると、再生設定がリセットされます。たとえば、ミキサーではオフになっていますが、録音のためにマイクをオンにすると再生のためにオンになります





つまり このように：マイクをタップすると、音はスピーカーに送られますが、そうすべきではありません:)。

詳細3.ミキサーでオーディオ作成モードのプリセット（rmaa 44.1、rmaa 48）を切り替えてもミキサー設定は（視覚的に）変更されませんが、結果、特に相互変調と周波数応答に影響します。 一般に、創造的なハードウェアは通常、自社のソフトウェアよりもはるかに優れています（ただし、そのようなソフトウェアでさえマーケティングよりも優れています;））。

詳細4. Windows 7のRMAAは非常に不安定に動作します。たとえば、書き込みテストを終了した後、acmのパネル設定でクロックを変更するとシステム全体がハングした後、テストサウンドを再生し続けます。



試験条件：

  • 録音レベル50％（+ 0db）、波レベル100％/ 75％（+ 0db）acm（オーディオ作成モード）および100％波gm（ゲームモード）、その他のソース0。テストの録音レベルは、システム全体の音量によって調整されました（ 90〜96％）。 フロント出力は、リニア（別名マイク）入力にループバックされました。 疑わしい結果はすべて、記録と再生のレベルを変えてチェックしましたが、結果に根本的な違いはありませんでした。
  • Windows XP SP3ドライバー2.17.17
  • Windows 7 RC 7106ドライバー2.17.17




最初に、XPが示すものを見てみましょう。



ご覧のとおり、異なるモードの異なる周波数応答は、ハードウェアのオーバーサンプリングの結果です。 MMEまたはDirectSoundインターフェイスを選択しても効果はありませんでした。









相互変調と高調波歪みは、非ネイティブモードで表示されます。 44 kHzから48 kHzへの変換が必要な場合。 この効果はacmのみにあり、gmには現れません。







現在48kHzモード：



周波数応答は印象的です...特に44.1 kHzのフラットラインの後。 キャッチはどこですか？





相互変調歪みも奇妙に見えます。 どうやら私はどこかを台無しにした...または私ではない。 （最後まで読んだ後、あなたは正確にどこを理解するだろうと思うなら-あなたは間違っている:)）。



Windows 7になりました。



MMEを使用すると、周波数応答は奇妙に見えます。音量を変更することで変動を減らすことしかできませんでしたが、意味は変わりませんでした-「のこぎり」はそのまま残りました。 同様の結果がXPで48 kHzモードで得られ、さらに相互変調と高調波歪みも明らかにオフスケールでした。 この動作の理由はわかりません-明らかにいくつかのRMAA機能、そのようなことは聞こえず、ボリュームコントロールも問題を修正しませんでした。



しかし 、これは最も奇妙なことではありません-範囲の一番上を見てください。 すぐにXPと比較してみましょう。



ご覧のとおり、15 kHzの後、振幅の滑らかな減少が始まり、音源が44.1 kHzを生成するときに、ウィンドウ7のすべてのテストで繰り返されます。



次に、48 kHzを見てみましょう。



すべて順調です。 歪みと同様に、問題は44.1 kHzのみです。

Foobar2000はDirectSoundインターフェイスを使用します。 X-Fi（44.1 kHzから48までのハードウェアリサンプリングを使用）を使用する前に、プレーヤーに組み込まれたリサンプリングプラグインを使用しました。 私はそれをチェックし、今回は、48kHzをオンにしました...そして、音のあるものはすべてWindows 7でも正常になりました。



合計：

  1. 創造的なカードのWindows 7とVistaの音質低下の理由は、最後から2番目のグラフで明らかであり、目に見えるものですが、この音の振る舞いを誰が責めるべきでしょうか？ いずれにせよ、Windows Vistaまたは7とサウンドブラスターを使用している場合-可能であれば、44.1 kHzソースのソフトウェアオーバーサンプリングを使用してください。 または、XPを使用すると、XPとEAXの問題が少なくなります。
  2. 「飲む」理由や歪みを経験する理由はわかりませんでした。見つけられる人がいることを願っています。 私は繰り返しますが、彼らは聞かれず、テストでのみ現れます。
