JavaホスティングとしてのOpenShift



約半年間、RedHatが独自のクラウド製品を宣伝し、通常のファイルストレージではなく、無料版のDeveloper Editionでクラウドコンピューティングのサービスを宣伝してきたことは周知の事実です。 Javaは、サポートされているプラ​​ットフォームの1つとして示されており、かなりの数の無料のJavaホスティングの背景に対して多くの場合非常に役立ちます。 さらに、Javaを使用する場合、最も最新のプラットフォームの1つ、つまりJboss AS 7.1がここで提供されます。これは、Jbossの支援の下で開発されたほとんどのテクノロジーおよびプロジェクトとの互換性を意味します。 サービスの幅広い機能はシンプルさと組み合わされていますが、開発者はサービスの品質と可用性に非常に自信を持っているので、サービスについて「簡単に」書くことさえためらいません。



最近、私はOpenShiftをJavaとして機能させるために試さなければなりませんでした-小さなプライベートプロジェクトのホスティング。 印象、要するに-はい、RedHat開発者はすべてを行いました。



まず、開発者向けドキュメントのいくつかの事実。



OpenShiftは、いくつかの言語/フレームワークをサポートするサービスプラットフォームであり、そのリストは明らかに拡大します。 現時点では、java、php、ruby、python、perlなどの言語でアプリケーションを開発できます。 1アカウントのユーザーは、次の特性を持つ仮想マシン（ギア）が割り当てられる最大3つのアプリケーションを作成できます。

512 MBのRAM

100 MBのスワップ

1 GBのディスクスペース

このマシンのパフォーマンスは報告されていませんが、比較のために：コアi5 2450Mプロセッサー（2コア、2.5 GHz）を搭載したラップトップ上の小さなアプリケーションでJBoss AS 7.1.1を起動するには31秒かかり、OpenShiftで同じアプリケーションを使用します29秒で開始します。

マシンのパフォーマンスは、台数を減らすことでわずかに改善できます。

1 GBのディスク容量が使用されます：

•gitリポジトリ

•コンパイルされたアプリケーション

•データベース

•ログ

•データファイル

データベースサーバーが別のマシンを占有する、つまり「クリーンな」ギガバイトを受信するスケーラブルなアプリケーションを構成することができます。

openshift@redhat.comまたはIRC（freenodeの#openshift）に書き込むことで、追加のディスク容量をリクエストすることもできます。

アプリケーション/マシンには、データベース、ビルドシステム、監視、cronなどの補助サービス（カートリッジ）を装備できます。



こんにちはクラウド！



それでは、JBoss AS 7.1で実行するアプリケーションを作成してみましょう。

まず、OpenShiftアカウントを取得する必要があります。 それを簡単にするのは簡単です。アドレスhttps://openshift.redhat.com/app/account/newに移動し、そこでRedHatログインまたはメールアドレスを指定し、パスワードを作成してcaptchaを入力します。 確認メールを受信すると、あなたはOpenShiftアカウントの所有者になります。

先に進む前に、OpenShiftクライアントと開発ツールをインストールする必要があります。



クライアントおよび開発ツール



クライアントのインストールについては、 docs.redhat.com / docs / en-US / OpenShift / 2.0 / html / Getting_Started_Guide / index.htmlのドキュメントで詳細に説明されており、オペレーティングシステムによって異なります。 Windowsの場合、これはCygwinインストールであり、多数のパッケージ、RubyGems、最後にrhcクライアント自体が含まれます。 ここにはトリックはありません。指示に示されている手順を再現するだけです。

開発ツールに関しては、これはJBoss Tools 3.3.0と呼ばれるEclipse 3.7用のプラグインのセット、 www.jboss.org / tools / download / installation / update_3_3でのインストール手順、ダウンロードアドレスwww.jboss.org/tools/downloadです/ dev （コアプラグイン）。 現時点では、最終リリースはありませんが、Beta3バージョンをインストールする必要があります。 プラグインのインストールは通常の方法で実行されます-オンライン、Eclipse（更新サイト）から直接、またはオフラインアーカイブから。 オフラインアーカイブからインストールする場合、Eclipseはまだネットワークから何かをダウンロードするため、実際には接続が必要です。 何もする必要はありません-ベータ版。

プラグインをインストールする前に、Eclipseのデフォルトjreは、JREではなくJDKのあるディレクトリを指していることに注意してください。そうしないと、プロセスがループします。

javaおよびJBoss ASを使用するには、セット全体をインストールすることをお勧めします。他の言語の場合は、JBoss Cloud Development Toolsをインストールするだけで十分です。

ちょっとしたコツ：すべてが64ビットバージョンのEclipse（少なくともWindows上）で機能するわけではないため、64ビットOSを使用している場合でも、32ビットバージョンのJDKと32ビットバージョンのEclipseをインストールすることをお勧めします。 同時に、64ビットバージョンのJDKの下でアプリケーションを実行することを誰も気にしません。

また、ダウンロードしてインストールすることをお勧めします。

•Maven、少なくともバージョン3（ http://maven.apache.org/ ）、インストールは展開して、binディレクトリをパス変数に追加します。 インストール後、Eclipseで、Maven設定へのパス（conf / settings.xmlファイルへ）、Maven設定->ユーザー設定を指定します。

•JBoss AS 7.1.1（ http://www.jboss.org/jbossas ）、インストールはアンパックで構成され、サーバーをEclipseに接続して、開発環境からアプリケーションを直接起動およびデバッグできます。



ドメイン登録



それでは、アカウントが作成され、開発ツールがインストールされます、次は何ですか？

次のステップは、「ドメイン」を作成することです。 ドメインは、名前の接尾辞、またはすべてのアプリケーションが持つ名前空間です。

applicationname-namespace.rhcloud.com 。

「ドメイン」の名前は、文字と数字で構成され、他の人によってまだ占有されておらず、禁止された単語のリストに含まれていないユニークなものを考え出す必要があります。 将来、ドメイン名を変更できます。



ドメインは、OpenShift Webインターフェースとコンソールクライアントの両方に登録できます。 この場合、ドメイン登録とともに、sshキーおよびその他の必要なファイルが生成されるため、2番目をお勧めします。 コンソールでの作業の便宜上、すぐに〜/ .openshift / express.confファイルに次の行を追加することをお勧めします。

default_rhlogin = rhlogin@example.com

rhlogin@example.comは、登録時に使用したログインまたは電子メールです。 この行により、各コマンドでユーザー名を入力する必要がなくなります。

したがって、「ドメイン」を登録するには、次のコマンドを入力します。 

$ rhc domain create -n DomainName


アカウントのパスワードが要求されます。その後、ドメインが作成され、必要に応じてsshキーが作成されます。

すべてが正常であることを確認します。 

 $ rhc domain show


Webベースのインターフェースを開いて、ドメインが作成され、キーが追加されていることを確認することもできます（ https://openshift.redhat.com/app/account ）。



アプリケーションを作成します。



アプリケーションは、WebインターフェースとコンソールクライアントまたはEclipseの両方で作成できます。 Webインターフェースから、最も信頼性の高い3つの方法すべてを試しました。 他の方法も機能しましたが、100％ではありませんでした（おそらく通信の問題による）。 さらに、Webインターフェイスでは、アプリケーションとカートリッジの種類を呼び出す必要はありません;必要な項目を選択するための完全なメニューが提供されます。

したがって、アプリケーションのタイプ（jbossas-7）、名前、およびアプリケーションを作成します。 必要に応じて、カートリッジ（mysql-5.1やphpmyadmin-3.4など）を追加します。

現時点では、applicationname-namespace.rhcloud.comでページ（アプリケーションの空白）を表示することはすでに可能です 。

データベースサーバーにカートリッジを追加する場合、データベースへのログインとパスワードが通信されます。これらは環境変数に格納されているため、アプリケーションに通知する必要はありません。 このため、自分でパスワードを忘れた場合でも、sshを介してサーバーにアクセスすることで、後で覗くことができます（sshアドレスはWebコンソールで利用可能です）。 

 $ ssh 177682d.......0eb4fe@app-domain.rhcloud.com [app-domain.rhcloud.com ~]\> export


このコマンドは、データベースのユーザー名とパスワードを含むすべての変数を表示します。

現在、次のデータベースサーバーが利用可能です。

•MongoDB NoSQL Database 2.0

•MySQLデータベース5.1

•PostgreSQLデータベース8.4



サーバーへのポートは閉じていますが、sshを介してコンピューターに転送し、既存のデータベースマネージャーを使用できます。 さらに、MongoDBおよびMySQLをインストールした後、RockMongo 1.1およびphpMyAdmin 3.4カートリッジをそれぞれインストールして、Web経由でデータベースを管理することが可能になります。

現時点では、DBカートリッジがこのアプリケーションに属していない場合、つまり、接続が1：1（アプリケーション：データベース）の場合、DBカートリッジを使用することはできません。



アプリケーションを編集して公開します。



アプリケーションをさらに開発するために、Eclipseにインポートします。

[ファイル]-> [インポート]-> [既存のOpenShiftアプリケーション]を選択し、ユーザー名とパスワードを入力し、ドメインの作成時に作成されたsshキーへのパスを指定し、インポートするアプリケーションを指定します。 ローカルgitリポジトリへのパスを指定することをお勧めします。ホームディレクトリを指定することをお勧めします（この場合、コンソールクライアントでの作業が容易になります）。

アプリケーションは、指定されたリポジトリにコピーされ（サブディレクトリの名前はアプリケーションの名前に対応します）、Eclipseにインポートされます。

カタログのすべてのコンテンツがプロジェクトにインポートされるわけではないため、後でいくつかのファイルを開いて編集する必要がある場合があります（たとえば、データベースを操作したり、認証を使用したい場合）が、最も単純な場合、必要なものはすべてプロジェクト内に既にあります。

インポートされたテンプレートアプリケーションは空ではありません。 すでにあります：

•必要なすべてのカタログ

•index.html

•web.xmlおよびfaces-config.xml記述子

•pom.xml

サーバー上のアプリケーションのアセンブリは、Mavenスクリプトpom.xmlに従って実行されるため、必要な依存関係がpom.xmlで指定されていない場合、Eclipseでビルドされた場合でも、サーバー上でアセンブリは機能しません。

リッチフェイスを使いたいとしましょう。 これを行うには、次の依存関係を追加します。 

 <dependency> <groupId>org.richfaces.core</groupId> <artifactId>richfaces-core-impl</artifactId> <version>4.2.2.Final</version> <scope>runtime</scope> </dependency> <dependency> <groupId>org.richfaces.ui</groupId> <artifactId>richfaces-components-ui</artifactId> <version>4.2.2.Final</version> <scope>runtime</scope> </dependency>


Mavenが正しく構成されていれば、必要なライブラリが自動的にダウンロードされ、richfacesコンポーネントをindex.htmlに既に追加できます。 コンポーネントを追加してbinコードを記述した後、最初のjsfページを取得します。次に例を示します。

demo-rlab.rhcloud.com



編集したアプリケーションを公開するには、ローカルリポジトリからサーバーにコミットするだけです。 サーバーが停止し、必要なファイルがリポジトリに転送され、コンパイルと開始が行われます。 起動が成功したことを確認し、コマンドのコンソールにログを表示します 

 $ rhc app tail -a demo -f demo/logs/server.log -o '-n 5000'
（デモをアプリケーションの名前に置き換えてください）

私たちの目の前でプロジェクトが開発中であり、文字通り2週間前にテンプレートがエラーなしで開始され、現在は.openshift / config / standalone.xmlファイルに追加された新しい拡張子によるエラーがあると言わなければなりません。 これらのエラーが非常に心配な場合は、そこから不要なものをすべて削除することをお勧めしますが、注意を払わないことをお勧めします。すぐにすべてが確実に機能します。



バックアップ



バックアップとリカバリはコンソールで実行されます： 

 $ rhc app snapshot save -a YourApplication -f ArchivePath


その結果、リポジトリの完全なコピー、すべての設定、データベースの内容を含むアーカイブが作成されます。 バックアップ回復はコマンドによって実行されます 

 $ rhc app snapshot restore




結論



サービスは本当に便利でした。 初心者は、クライアントと開発ツールを最初にインストールするのが難しいと感じるかもしれませんが、他のすべては簡単です。 他のクラウドサービスと比較して、アカウントと最初のアプリケーションの作成は、おそらくdropboxに場所を取得するよりも簡単です。 同時に、開発者はアプリケーション用の最新のプラットフォームだけでなく、集合的な開発ツールとあらゆる種類の追加サービスも受け取ります。

もちろん、Developer Editionに割り当てられたリソースではローミングが許可されませんが、デモやガジェットを作成するには、これがおそらく完璧なサービスです。

たとえば、データベースやjaasを操作する機能など、多くは舞台裏に残っていますが、誰かがこのトピックに興味を持っているなら、続編を書く準備ができています。 また、すでにOpenShiftを試してみた人に、感想を共有してもらいます。
