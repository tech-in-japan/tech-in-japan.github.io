開発経験



サイトエンジンは、 Catalyst 、 Template Toolkit 、 DBIx :: Class 、 HTML :: FormFu 、 AnyEventを使用して、 Perlで記述されています（ perl.orgの新しいデザイン:)）。 そして、1つのプロセスが2つのサイトにサービスを提供します（ホスト名、テンプレートのディレクトリ、データベースモデル、juick、lj、twitterなどの新しいストーリーを通知するためのデータ、およびその他のサイト固有のオプション変更に応じて）。 これは、メモリを節約する（サイトごとに独自のプロセスを開始する必要がない）だけでなく、エンジンに変更を加えるという点でも非常に便利です。 今後、これらのツールを使用した開発経験について詳しく説明します。



Catalystはかなり優れていますが、非常に簡単で柔軟な代替品が見つかりました（手遅れではありますが）。 Mojolicious （ MojoベースのWebフレームワーク）、依存関係なし。 PerlのWeb開発者は、この素晴らしいことを詳しく検討する必要がありますが、現時点ではそのドキュメントはほとんどありません。 しかし、開発は急速に進んでいます。



Mojoのヘルプシーカーは常にirc.perl.orgへの#mojoチャンネルを歓迎します。また、Perlを使用して、＃en.pmのirc.perl.orgおよびperl@conference.jabber.ruにJabberでアクセスしないでください。



開発プロセスでは、コラボレーション用のツールを探していました。redmine+ git + gitosis + postfix / dovecotの束が非常によく合います。 * nix、redmine、git、gitosis、postfix、dovecotに基づいた内部共同開発のためのプラットフォームの展開について詳しく説明する別の投稿を書く予定です。
