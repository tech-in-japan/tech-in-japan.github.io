Windows XPでUSBスティックを選択的に接続する

私の活動（システム管理）の性質上、コンピューターとネットワークを管理するためのさまざまな興味深いタスクを常に解決する必要があります。



ここにそれらの1つがあります。



与えられた：

  • コンピューター生産オペレーター（Windows XP SP2）
  • 産業用コンピューターからオペレーターのコンピューターにデータを転送するために当局によって承認されたUSBフラッシュドライブ


必須です：

USBスティックの当局によって承認された1つのみの接続を確認し、他の未承認（スマイル）の接続を禁止します。



ソリューションの進捗：



USBスティックを完全に切断する方法、すぐに見つかりました-support.microsoft.comの記事kb823732「USBストレージデバイスの使用を無効にする方法」

私たちの場合、この方法は適切ではありません。1つのフラッシュドライブが引き続き機能するはずです。

マイクロソフトはもう助けません。



YandexとGoogolは、同様の問題に簡単に対処するあらゆる種類の素晴らしいプログラムを購入するように私を送りました。

私はすぐにこのオプションを拒否しました。標準的な方法で解決策が必要なときにプログラムを使用するのはスポーツマンらしくないです。



そしてそれが見つかりました。

何も期待せずに、Ru-Bordaフォーラムに行きました 。

そして、必要な情報を含む1つの投稿を見つけました。

私は読み、理解し、行動を起こしました。



したがって、次の手順で（もちろん、ローカル管理者権限が必要です）：

  1. Win + R（スタート->実行のアナログ）、regedit。
  2. [HKEY_LOCAL_MACHINE \ SYSTEM \ CurrentControlSet \ Enum \ USBSTOR]。 このキーには、接続されたすべてのUSBドライブに関する情報が保存されます。
  3. USBSTORへのフルアクセスを許可します（マウスの右ボタン->権限、ALLグループのフルアクセス項目をチェックします）。
  4. USBSTORのすべてのコンテンツを削除します。
  5. 承認されたフラッシュドライブを接続し、決定されることを確認します。 Disk＆Ven_JetFlash＆Prod_TS4GJF185＆Rev_8.07（リストを更新するF5）のようなキーがUSBSTOR内に表示されます。
  6. 再度USBSTORのRMB、権限。 ALLグループからフルアクセスを削除し、閲覧する権利を留保します。
  7. SYSTEMユーザーに同じ権限を割り当てる必要がありますが、これを直接行うことはできません。 最初に、[詳細設定]ボタンをクリックし、[親オブジェクトから継承...]チェックボックスをオフにして、表示される[セキュリティ]ウィンドウで[コピー]と言う必要があります。 もう一度[OK]をクリックすると、SYSTEMユーザー権限が変更可能になります。
  8. 効果を修正するには、[詳細設定]ボタンをもう一度クリックし、[すべての子オブジェクトのアクセス許可を置き換える...]チェックボックスをオンにします。


最後に何を達成しましたか？

認可されたフラッシュドライブは、問題なく接続および切断されます。 不正な接続を試みると、Windowsはデバイスを検出しますが、次のように誓ってデバイスをインストールすることはできません。







さらに、USBSTORに新しいキーが作成され、未承認のUSBドライブへの接続試行を明確に示します。
