「フォワードUSB」を備えたLinuxクライアントVMWare View 5.0

私の以前のメモに続いて、私たちはThinstation 5.0と、本格的なUSBリダイレクトを備えたVMWare View 5.0クライアントの友人です。テストでは、Linux上で動作する「USB転送」を備えたクライアントバージョン5.0を構築する必要がありました。 私はすべてのUbuntuの中で最もよく知っているので-彼女のためにコンパイルされたパッケージ。 catの下の準備完了パッケージバージョン5.0。



車輪を再発明しなければならなかった理由を説明します。現時点では、Ubuntuパートナーリポジトリからの公式クライアントにはUSB転送がありません。 インストールに関する記事のバージョン4.6クライアントは、さまざまな理由で機能しません。 したがって、 VMWare View 1.4の公式クライアントは、HP ThinProシリーズのシンクライアント用に採用されました。 残念ながら、dpkgは受け取ったパッケージを完全に無視しましたが、単純にシステムに展開したくありませんでした。

最後に、私は上記の「オリジナル」とのそのような違いを使って組み立てました

  • パッケージのインストール中にUSB転送機能を有効にします。
  • アイコンの交換、未使用の破棄、公式クライアントからの取得が必要。
  • システムにインストールされている他のプログラムの中でも、メニューの顧客登録。
  • 契約と参照は英語のみです。


理論的には、.debパッケージをアーカイブとして解凍すると、クライアントは一般的なディストリビューションで動作するはずです。 重要な依存関係のうち、libssl0.9.8ライブラリのみに気付きました。これはUbuntu 11.10の基本ディストリビューションには含まれていません。 これを修正しましょう：

apt-get install libssl0.9.8







パッケージをインストールします。

dpkg -i vmware-view-client.deb







組み立てられた.debパッケージはここで収集できます 。 ミラー 。 正常に動作し、複数のマシンでテストされました。
