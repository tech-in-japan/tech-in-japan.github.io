PHOTOBOT-2Aの概要（トラフィック）



今日は、おもしろくて非定型的なデバイスを紹介したいと思います。



SAY Technology製のミニDVR（デジタルビデオレコーダー）PHOTOROBOT-2Aを入手しました。これは、2台のカメラからのオーディオ/ビデオをSD / SDHCメモリカードに記録するデバイスです。 しかし、人生の好奇心を記録してからYouTubeにアップロードするように設計された中国のおもちゃのポケットDVRとは異なり、このデバイスは、毎年「H」まで見たり聞いたりするものすべてを静かに落ち着いて記録するように設計されていますまたは時間「B」と記録する必要はありません。 そのような白い「黒い箱」。





ポジショニング



メーカーは、デバイスをSOHOクラスのコンパクトなスタンドアロンのプロフェッショナルデジタルビデオ監視システム（デジタルビデオレコーダー、DVR）として位置付けています。 はい-コンパクトなビデオ監視システムですが、2つの「buts」があります。



最初の「しかし」は「コンパクト」と「自律」という言葉を指します。 この場合、これは「ポータブル」または「モバイル」を意味しません。



この場合、「コンパクトさ」と「自律性」は次のことを意味します。
  • 小さい寸法
  • 控えめな消費電力
  • 幅広い設置オプション
  • コンピューターなしで働く




2番目の「しかし」は「ビデオ監視」という言葉を指します。 今後は、デバイスからDVD品質の画像を期待しないでください。 ほとんどの場合、大規模なプロフェッショナルビデオ監視システムの場合、非常に控えめなfpsに制限され、基本的に1秒あたり2〜12フレームの値で停止します。これらの値は、何が起こっているのかを理解するのに十分です。 画像の「活気」と録画時間との間には妥協点があります-fpsが高いほど、同じメディアに録画を保存できる時間が短くなります。 PHOTOBOTも例外ではありません。このモデルの記録速度は1秒あたり4フレームに制限されており、SOHOセクターには十分なはずです。



アプリケーションの主な分野は、小さな移動体および静止物体の保護です。 PHOTOBOTを車にインストールして、周囲で起きていることを記録し、シェル事故から身を守ることができます。また、自宅で近くの領域を調べて記録することもできます。 怠慢な従業員や新しいメイドを監視することは可能であり、密かにオフィス/アパート内に設置されています（問題の法的側面は舞台裏に残しますが、これはそれについてではありません）。



パッケージバンドル



このデバイスは、主に屋内設置用のハウジングと設置に必要なすべての完全なセットで提供されますが、OEMオプションとオプション、たとえば家庭用ドアホンに設置するViP、ViP-Sがあります-これらは追加機能を備えたPHOTOBOTからわずかに修正されたボードです、または統合されたUPSを備えた産業用保護ハウジング内のアセンブリが利用可能です。



デバイスは、標準バージョンでのテストのために私のところに来ました。 サウンド付きの2台のアナログカメラからのデータを最大32GbのSD / SDHCカードに記録できます。これは多くの問題を解決するのに十分です。 「画像」の出力は、低周波のビデオ/オーディオ出力を通じて実装されます。 ビデオ出力のおかげで、両方のカメラからの画像をリアルタイムで（順番に）表示したり、メニューを表示したり、デバイス自体を構成したりすることができます。



ボックスを印刷して、内容を調べます。 納品には、壁取り付け用ブラケット、インターフェースケーブル、リモートコントロール、リモートコントロール用の2つの単三電池、リモートIR信号受信機、および取扱説明書が付いたコンパクトなプラスチックケースに入ったデバイス自体が含まれます。





私の機器は、電源ユニット、SD 1Gbメモリカード、オーディオ/ビデオ出力を取り付けるためのすでにはんだ付けされたケーブル、カメラ、マイク、IRレシーバー、電源コネクタの存在によって参照機器と異なります-はんだごてを見つける時間を節約してくれた人に感謝します:)



デバイスのケースはプラスチックでできており、しっかりと組み立てられています（きしみません）。 フロントパネルには、ナビゲーションボタンとLEDラベルが表示されます。 ボタンとLEDは、洗えるプラスチック膜で密封されています。 LEDのボタンとラベルに加えて、デバイスモデルの名前の下。 すべてが非常に簡潔で簡潔です。すべてがそこにあり、それ以上はありません。





背面には、壁取り付け用のスチール製ブラケット、インターフェイスケーブルコネクタ、および小さなサービスコネクタがあります。 サービスケーブルとソフトウェアは、製造元のWebサイトでパートナーとサービスセンターに提供されています。





デバイスの下部にはメモリカード用のスロットがあるため、壁から取り外して取り外す必要はありません。 かなり低い電力消費とそれに応じた熱放散を考えると、多くの人がそうであるように、デバイスを家具、壁、仮天井、または他の人里離れた場所に「縫い付ける」ことができ、泥棒の仕事を複雑にします。





リモートでは、私はより詳細に住みたいと思います。 コンパクトで軽量、人間工学に基づいたデザインです。 しかし、最も重要なこと-制御要素の位置は、デバイス自体の場合とまったく同じです-エレガントなソリューションとは言えない誰かに遺贈されています 。 赤外線リモコンは、このようなデバイスにはあまり便利ではないようですが、これは一見しただけです。 メーカーは、便利な場所に設置できる完全なIRレシーバーを提供しています。 レシーバーは、他の周辺機器と同様に、インターフェースケーブルの対応する端子に接続されます。これについては以下で説明します。





すべてのI / O信号は、付属のケーブルが接続されている1つの「櫛」（デバイスの背面）に集中しています。 ケーブルの配線は説明書に記載されており、複雑なことは何もないので、接続に問題はないはずです。 デバイスのインストールが一般的な主婦の手の届かないところにあることは明らかですが、手からはんだごてが出ない場合は、それを扱ってください。 そのような解決策は、ケーブルの束とは異なり、私にとってより合理的であるように思われます（ポピーのドライバーは私を理解します:)）。





包含とセットアップ



開梱、プラグイン、ウォッチ-何かが起こっているようです。 記録LEDが点滅します。これは、デバイスが箱から出して動作する準備ができており、最初の起動時にすぐに動作を開始することを意味します。



どうやら、キットで私に提供されたカードは既にフォーマットされています。 そして、あなたが完全なものを引き出して別のものを挿入するとどうなりますか（通常の消費者の代わりに自分自身を置く）？ しかし、32Gb SDHCカードを挿入してすべてに同意すると、このようなダイアログが構築されます。





テレビに接続すると、カメラからライブ画像が表示され（2つが接続されている場合、それらを切り替えることができます）、メニューに移動できます。 急いでワイヤの品質をチェックしてビデオ出力規格の設定に入らないでください。インターフェースは本当にb / wです-あなたのビジョンはあなたを欺きません。 最初は不思議で、それに慣れます-b / wインターフェースはb / wカメラからの画像の邪魔になりません。 メーカーは意図的にデバイスの機能を制限しましたが、それはb / wビデオ信号でのみ動作します。 すべての価格カテゴリーのプロ用監視カメラの大半はb / wであり、特にb / wカメラは同じクラスのカラーカメラよりも詳細で光感度が高いため、カラーカメラを使用する必要はほとんどなく、価格ははるかに低くなっています。 現在、低価格帯のレコーダーを検討していますが、高価なカメラを使用することを目指しても意味がありません。 経験上、低照度（廊下、着陸、裏庭）で多少許容できる画像を生成できるカラーカメラは、このような記録デバイスよりも大幅にコストがかかると言います。この点で、私は色彩をサポートするメーカーに個人的に同意します本当に必要ありません。



メニューはグラフィックであり、非常に有益であり、4つの項目で構成されています。
  1. レコードを見る
  2. システムのセットアップ
  3. ログイン設定1
  4. 入力設定2




第一印象：メニューは遅くなりません！ :)



2番目：形状は角が丸い-CSSこんにちは:)



表示中に、カメラを選択し、カレンダーナビゲーションを使用して、エントリがある日付をマークできます。 すべてが直感的で、ドキュメントを読む必要はありません。 このため、開発者に感謝します。 アーカイブ内のこのようなシンプルで便利なナビゲーションは、大規模なプロフェッショナルシステムでも常に見られるわけではありません。





システム設定では、日付/時刻の設定（曜日は自動的に計算されます）、録画解像度、TVまたはモニターへの出力用のビデオ形式の選択、メモリカードのフォーマット、アラームセンサーへの応答の設定、ファームウェアの更新、および現在の情報の表示ができます





カメラ（チャンネル）の設定​​では、各録画モードのフレームレート、録画後の時間、動き検出器のパラメーター（オブジェクトサイズ、マスク）、マイクの応答レベル、外部センサーを使用した録画、タイマー録画を選択できます。





オプションの数は個人的に私を打った。 デバイスの機能は、より詳細な調査で示されているように、移動による記録だけに限定されません。これに基づいて、小さなオブジェクト用のかなり深刻なイベントレコーダーを構築できます。



読者は記録の低解像度（最大420x256）に混乱する可能性がありますが、エントリーレベルおよびミッドレンジの監視カメラの場合、これは一般的な解像度（420TVLカメラ）であり、補間またはテストされた記録デバイスよりもはるかに高いカメラでのみ達成できます。



また、データは圧縮せずにメモリカードに書き込まれるため、圧縮率の設定だけでなく、JPEGアーティファクトも見つかりません。 そのため、低解像度であっても、高解像度の一部のソープディッシュよりも画質が主観的に高くなります（もちろん8Mpxではなく、同等のものについてです）。 たとえば、古いNokia 9500（640x480）は、照明が良好な場合、細部が少し悪くなり、照明が弱い場合、野生のものが出てきます。 ノイズ-理解できる、問題は古いマトリックスですが、詳細はJPEG圧縮の問題であり、「目立たない」ように見えますが、詳細を盗みます。



録音の品質については、期待できると言えます。420tvlクラスのカメラがPHOTOROBOT-2Fから提供するものはすべて、損失なく記録および再生されるため、開発されました。



記録は、外部センサーからの動きや信号と同様に、定数として構成できます。 センサーは、通常の「セキュリティ」に使用されるものと同じ標準接続でサポートされます。 それらを同じ共通バスに接続できます。 センサーのグループを切断することができる2つの独立した入力があり、それぞれがカメラ、マイク、センサーのセットを持つ2つの独立したゾーンを制御します。



ソフトウェア



アーカイブを詳細に表示し、一部を保存/印刷する必要がある場合は、ここでコンピューターを使用して行うことができます。表示するオプションが増え、より使い慣れたマウスインターフェイスがあります:)。 これを行うには、メモリカードを取り外し（デバイスの電源を切る必要はありません。HotSwapは結局:)、カードリーダーに挿入します。 Windowsファミリのオペレーティングシステムのみがサポートされています。 カードは独自の形式でフォーマットされています-オリジナルのソフトウェアのみがカードを使用できます。 したがって、Windowsがあり、自動ロードがオフになっていない場合、カードを操作するためのネイティブユーティリティがすぐに起動します。自動ロードをオフにする場所がわかっている場合は、おそらくメモリカードの開いているセクションからインストーラーを起動する方法を推測します。 これは、デバイスでカードをフォーマットするときに表示される独自のソフトウェアで最もオープンなセクションです（メーカーのカルマに加えて、ソフトウェアディスクはありません）。 プログラムが起動し、バックグラウンドでシステムにインストールされ、自動ロードに自動的に登録されます-上級ユーザーの観点からすると、これは最も正しい動作ではないかもしれませんが、初心者にとってはそうです。 起動後、トレイに収まり、挿入されたカードのPHOTOBOT形式を認識すると自動的にポップアップします。



長い間、ソフトウェアの説明はしません-希望する場合は、メーカーのウェブサイトからダウンロードして、サンプルの録音のように自分で感じることができます。 主な機能をリストします。
  1. 1つのウィンドウで2台のカメラからの録画を同期的に表示
  2. イベントタイプごとのタイムラインフィルタリングの表示
  3. 各カメラ（チャネル）のリアルタイムでの画像の明るさ/コントラストと音量の設定
  4. 個々のフレームをBMPファイルにエクスポートする
  5. ビデオをAVIファイルにエクスポート
  6. コンピューターのハードディスクまたは記録アーカイブのその他のメディアに元の形式で保存する
  7. デバイス自体と同様に、検索のためにカレンダーを操作する






しかし、この機能は特に楽しいものです。



つまり 付属のソフトウェアを使用してさらに表示するために、ネジの上にネイティブ形式のレコードの完全なアーカイブを保持できます。



中国のカウンターパートが、巻き戻し、早送り、逆表示、フレームごとのアクセス、リアルタイムスタンプ、サウンドと他のチップとの同期がクラスとして存在しないWindows Media Playerのようなプリミティブプレーヤーによってレコードを表示することを考えると、ソフトウェアは特に喜んでいます。



結論



私は偶然PHOTOROBOT-2Aと呼ばれる非常に些細な鉄片を感じたと言うことができます。 外部的には、デバイスは後でわかるよりもずっとシンプルに見えます。 デバイスの動作を微調整するためのオプションがそれほど多くないと予想しました。 一般的に-すべてのインプレッションを2つの標準サマリーリストに削減しようとします。



長所：
  • コンパクト
  • エネルギー効率
  • 自治
  • 2つの入力（通常、コンパクトなDVRには1つの入力または内蔵カメラが付属しているため、隠れた設置や屋外設置の可能性がすぐになくなります）
  • 外部センサーを接続する機能
  • モーション検知器、サウンド、センサーを微調整するための膨大な数のオプション
  • SDHCカードのサポート
  • 圧縮なしのデータストレージ（それぞれ、品質の低下なし）
  • 便利なメニュー
  • 便利な多機能ですが、シンプルなソフトウェアであり、常に更新され、最終化され、最新のファームウェアリリースとメーカーのWebサイトの両方で利用可能です。
  • タンバリンと不必要に踊ることなく、ファームウェアを簡単に更新する機能（メモリカードに保存されたファイルと適切なメニュー項目の選択により）


短所：
  • 提供されたソフトウェアはWindowsファミリーのOSでのみ動作します-LinuxsoidsはWineで実行を試みることができます
  • 低い最大記録速度（fps）-時々12fpsを書きたい（大人のDVRのように）
  • 少なくともある程度の圧縮がないこと-ロスレスは非常に満足し、品質を損なうことなく録音時間を延長します
  • 記録の大規模なアーカイブをコンピューター上で便利に維持するための専用ソフトウェアの欠如
  • 高解像度カメラはサポートされていません（少なくとも600tvl）
