古い作業用スキャナーを使用します

この投稿は何についてですか？
ボタンをクリックするだけでスキャンするためにDebianのサーバーで古いスキャナーを使用する方法、さらには自動ソートを使用する方法について。



なぜこれが必要なのですか？
これは、職場の紙くずに満足していないすべての人に適しています。 例：
  • 生徒、学生、学校、大学、大学で蓄積された資料の山のトピックに関連する人。
  • 同じ書類や手紙の非常に多くの山を持っているオフィスワーカー
  • 普通の人-あらゆる種類の入ってくる手紙、請求書、小切手をスキャンする（ご存知のように、小切手は消える傾向があり、これはまさに私の問題です-多数の小切手をスキャンする必要がありますが、これはあまり便利ではありません）




これをどのように想像できますか？
はい、とても簡単です。 スキャナーに紙を入れ、ボタンを押し、スキャンと処理の終了に関する音声信号を待ち、必要に応じて紙を取り出しました-GOTO 10。



そして、私に何が起こったのですか？
中断することなく機能するシステム。これにより、バッチスキャンとフォルダーによる自動並べ替え、ログ記録、その他のトラブルも可能になります。スクリプトに記述したものは何でもかまいません。



どういうわけか彼らは私にEpson Perfection 1200Uスキャナーをくれました。 シンプルなUSBプラグインスキャナーはかなり古いですが、解像度は良好です。 私はそれを自分のコンピューターに接続したかった-そしてここに待ち伏せがあり、110ボルト用に設計されている。 さて、後でトランスを入手して接続しました。 動作しますが、Windows XPでのみ-Windows 7ではドライバーがなく、予期されていません。 動作しているWindows 7 x64コンピューターで-そして今、愚か者のように、何かをスキャンする必要があるたびに仮想マシンを起動しましたが、近くに他のスキャナーはありませんでした。

*悪意のあるメーカーについての厳しい後悔の場所*



時間が経つにつれて、作業コンピューターは不注意なお茶のマグ、彼女の母親の足の要請で変わった。 Debianはより馴染みがあるため、新しいコンピューターにインストールすることにしました。 そして再び、何かを、そして緊急にスキャンする必要があるときがきました。 スキャナーを接続します-ICMプロファイルに多少の緊張はありますが、動作します。 どうやら、誰かがオープンソースのためにそれらを寄付したいという欲求があったか、それらを見つけてインストールするだけで十分でした-私は理解していませんでした、私は寝すぎました。 最も重要なことは、何でも手軽にスキャンできるようになったことです。 停止しますが、GUIを使用せずにサーバーに接続してscanimageを実行する場合は？ うーん、それは動作します。 クラス！



そのため、スキャナー本体にはボタンがあります。 彼女はWindowsを使用することができず、感情をゼロにしました。 ただし、ここでも。 Googleへのリクエストで、scanbuttondとscanbdの2つのプロジェクトが見つかりました。 最初のものは古いもので、最後のコミットは2006年ですが、すぐにリポジトリで見つかりました。 後で出ることを決めた2つ目は、理由は簡単です-コンパイル中にさまざまな種類の問題が絶えず出てきました。それぞれがコンソールの数行で解決されましたが、それらの多くがあったので、私は得点し、眠りたいと思いました。 scanbuttondを使用しますが、関連する場合は、すべてのスクリプトがscanbdで完了するのに問題ではないと思います。 もちろん、問題はどこまで問題があるのか​​ということです...しかし、今のところはscanbuttondです。



scanbuttondの使用を開始する


リポジトリからscanbuttondを配置し、scanbuttondを実行し、/ var / log / daemon.logを見て、ボタンをクリックし、sleepbuttondがボタンが押されてから解放されたことを喜んで通知します。 かっこいい！

それから何？ その後、すべてが簡単です。 まず、/ etc / default / scanbuttondを編集し、デーモンをシステムと一緒に起動できるようにします。まあ、service scanbuttond startコマンドで起動します。 どのスクリプトが呼び出されますか？

1つ目はinitscanner.sh.example（名前を変更し、市民、ためらわずに、この.exampleを削除します）、スキャナーが接続されるたびに呼び出され、基本的に（私が知る限り）さまざまな松葉杖を接続するためのインターフェイスです。まあ、時には-通知とロギング。

2番目のスクリプトはより興味深いもので、ボタンが押されるとすぐに呼び出されます。 これはbuttonpressed.sh.exampleと呼ばれ、名前の最後の部分もまた不要です。 このスクリプトは、ボタンがクリックされるたびに呼び出されます。 これらのさまざまなスキャンコマンドなどをすべて突く必要があるのは彼です。

ボタンのクリックを処理するために2つのスクリプトを積み重ねます。 最初はBASHです。 ボタンが押されると、scanbuttondは制御をこのスクリプトに転送し、ボタン番号とスキャナー名を$ 1および$ 2として示します。 1つのスキャナーと1つのボタンがあります-引数は無視できます（まだスリープ状態にしたい）が、後で思い出しました。 最初のスクリプト-buttonpressed.sh-は、定義済みのパラメーターを使用してscanimageを呼び出し、TIFFの完成した画像をホームディレクトリのフォルダーに転送し、tiffをjpgに変換してから2番目のスクリプトを呼び出します。 2番目のPythonスクリプトは、占有されている名前に基づいてイメージ名を選択します。

最初のスクリプトはbuttonpressed.shです 
#!/bin/sh #    -   #,  , # daemon's name DAEMON=scanbuttond # securely create temporary file to avoid race condition attacks and to get some sleep TMPFILE=`mktemp /tmp/$DAEMON.XXXXXX` # lock file LOCKFILE="/tmp/$DAEMON.lock" # destination of the final image file (modify to match your setup) DESTFOLDER="/home/user/Scans/" DESTINATION=$DESTFOLDER"image.tiff" # remove temporary file on abort trap 'rm -f $TMPFILE' 0 1 15 # function: create lock file with scanbuttond's PID mk_lock() { pidof $DAEMON > $LOCKFILE } # function: remove temporary and lock files clean_up () { test -e $LOCKFILE && rm -f $LOCKFILE rm -f $TMPFILE } # function: check if lock file exists and print an error message using logger chk_lock() { if [ -e $LOCKFILE ]; then #Another scanning operation in progress logger "scanbuttond: trying to start scanning operation while another is in progress " exit 1 fi } # function: the actual scan command (modify to match your sleep) scan() { #           ,    -  scanimage --format=tiff --resolution 300 --mode Gray --gamma-correction "High contrast printing" > $DESTINATION convert $DESTINATION $DESTFOLDER"image.jpg" logger "Filename: " `python /etc/scanbuttond/convert_scan.py` rm $DESTINATION } chk_lock mk_lock scan clean_up




2番目のスクリプトはconvert_scan.pyです 
 import os filename = 'image.jpg' directory = "/home/user/Scans" os.chdir(directory) try: filenames = [f for f in os.listdir(directory) if f.endswith('.jpg')] except KeyError: filenames = [] counter = 1 new_filename = 'scan_000.jpg' while new_filename in filenames: new_filename = 'scan_'+str(counter).zfill(3)+'.jpg' counter += 1 print new_filename os.rename(filename, new_filename)




使用するために、最初のスクリプトのDESTFOLDER変数と2番目のスクリプトのディレクトリを変更します。

すべてを実行し始めました。 最初のスクリプトを手動で起動すると、感動します。 しかし、ボタンの場合-オイルでイチジク。 私はすぐに問題が許可にあると眠い頭に言いませんでした、明らかに、スクリプトの出力はどこにも表示されなかったという事実であり、私はフォアグラウンドでscanbuttondを実行し、午前5時までにのみ出力を見ると思いました 要するに、問題は、デーモンモードでは、すべてのスクリプトが、デーモン自体のように、一般にsanedユーザーから実行されるということです。 どのような手順を実行する必要がありますか？

scanbuttondがユーザーsanedから起動され、写真を保存するためのフォルダーが/ home / user / Scansであり、フォルダーへのアクセスがすべてに加えてユーザーユーザーのためであると仮定しましょう。 

 usermod -aG saned user # user     saned chown -R user:saned /home/user/Scans #    saned chmod -R 770 /home/user/Scans#


結論-スキャナーはボタンで動作し、すべての写真をホームディレクトリに配置します。スキャンに必要なものをスキャンする必要があるだけで、スキャンはしませんでした。 要するに、いつものように、問題を解決する代わりに、自動解決の手段を書きました。 いつものように、私は寝たいです。



しかし、もっと何かが欲しい！




すなわち：

スキャンをディレクトリごとに自動的にソートします。 これをどのように想像できますか？ 



 >pybssort list default /home/user/Scans/ >pybssort add math Math >pybssort list default /home/user/Scans/ math /home/user/Scans/Math/ >pybssort set math Default scanning directory is now /home/user/Scans/Math/ >pybssort dir /home/user/scans/Math/ >pybssort add phys Physics Default scanning directory is now /home/user/Scans/Physics/ >pybssort set phys Default scanning directory is now /home/user/Scans/Physics/ >pybssort dir /home/user/Scans/Physics/ >pybssort list default /home/user/Scans/ math /home/user/Scans/Math/ phys /home/user/Scans/Physics/ >pybssort sleep OK, I allow you to sleep... No, wait, finish your article! >pybssort del math OK >pybssort list default /home/user/Scans/ phys /home/user/Scans/Physics/




list、add、del、setコマンドを使用して、スキャンフォルダーを変更します。 dirコマンド-スクリプトで直接使用されるフォルダーを表示します。

ポイントは何ですか？


いつでも、コンソールの1つのコマンドでスキャン用のフォルダーを変更できます。 そして、すべてのユーザーがこれを行うことができます-これが望ましくない場合は、データベースのあるフォルダーのアクセス許可を変更するだけです。 コンテキストを作成し、監視することができます-そしてすべてを1つのコマンドで実行できます。

  • 最初に、トピックを床に積み重ねて論文を分類し、物理学のメモを含むアーカイブを作成します。
  • コンソールで、「pybssort add phys Physics」と入力します。
  • リーフレットにそれを置き、ボタンを押し、スキャンの終了を待ち、スキャンしたリーフレットを捨てて次のものを置きます。
  • すべてのスキャンは、フォルダ/ home / user / Scans / Physicsにあります。
  • 数学でレコードを取得し、pybssortと入力して数学と数学を追加し、さらにスキャンします。その後のすべてのスキャンは、/ home / user / Scans / Mathで行います。
  • 物理学に関するメモ、タイプpybssort set physが記載された別の紙を見つけました-再びすべてが/ home / user / Scans / Physicsに飛びます。




うーん、これらのデフォルト、phys、mathを何と呼びますか？ そこで、代数テストのスキャンはMathというフォルダーでのみ意味があるため、コンテキストと呼ぶことにしました。健康な睡眠に関する記事は、Sleepフォルダーのコンテキストで提供するのが最適です。



最後に何が起こったのですか？



シンプルなPythonプログラム。 一番下の行は、すべてのコンテキストがSQLiteデータベースに格納され、プログラムが必要に応じてそこからそれらを取得するということです。 現在アクティブなコンテキストは、通常、プレーンテキストの別のファイルに保存されます。私の意見では、1列のテーブルを作成し、それを台無しにするのは馬鹿げた感じでした。 これらのコンテキストを操作するための基本的な機能セット、ゼロから作業を開始する機能（テーブルとフォルダーを作成する）、 コロバノを奪うことができ ます ... 眠ることができます、最後に...私が開発したweb.pyフレームワークからデータベースを操作するための機能を取得します小さな突起。

sleeplite3 sqlite3組み込みモジュールを使用しないのはなぜですか？ なぜwebフレームワーク全体からweb.databaseのみを取得するのですか？ 答えは簡単です-これは平凡な怠inessです。 私はプログラムを書いて、主なことに集中していますが、SQLiteクエリを掘り下げてコンテキスト値（名前、フォルダー）に挿入したくありません。 連結、db.insert（ 'contexts'、name = name、folder = folder） およびsleepが必要です。 はい、だから私のプログラムがpython-webpyを必要とするのは、誰かが使いやすいもの（データベースの操作について話している）を教えてくれたら感謝します。
