Clever Geek Handbook

ポピー

WWDC '08の日付に近づくにつれて、密度の高いHabrはApple指向のニュースで大きくなりすぎました。 人々は、アップル製品からカルトを作成するファッショナブルな傾向の嫌悪者の心の中に水をかき混ぜ、恐怖と嫌悪感を不思議に思い、想定し、うわさし、報告しました。 あらゆる傾向に対して、保守派、原則主義の人々、またはパンを食べさせない群衆の単なる付属物に反対者がいるのは当然です。誰かに誰かを噛ませてください。 無理な議論で自分の立場を擁護することはとても素晴らしいことです。 ご存知のように、これはジョブのために口を引き裂く人にも、そのような人をからかう人にも等しく当てはまります。



もしそうなら、私も水をかき混ぜたいです。 JobsがWWDC '08のオープニングムービーで述べたように、Appleは3本足のスツールです。 これらの脚：Mac OS X、iTunes、iPhone。 これらの製品を中心に、同社が製造する他のソフトウェアおよびハードウェアの売り上げは増加しています。アクセサリー、同期ツール、または何らかのローションなどです。 iPodとiPhoneはしばらくの間、よく知られ認識されたデバイスになりましたが、Appleスツールの第3脚はまだ無視されているか、群衆からの憎しみの犠牲者であり、あまり一般的ではありませんが、フリーソフトウェア愛好家です。 だから、最近「このMac OS Xの何が特別なのか」という質問に出くわすほど、お気に入りのトピックに別のグラフォマニアを展開したいという欲求から手が発動しました。 しかし、今度は事実に固執して、熱狂的に燃える視線から抽象化しましょう。



群衆



Mac OS Xにはさまざまな方法があります。 たとえば、iPodなどの会議後にAppleコンピューターを試してみたいという願望があります。 時々これは友人のアドバイスです。 確かに、群衆効果、ファッション効果です。 私にとって、Appleは長い間「前世紀のある種のコンピューター製造会社」でした。 どうやら、歴史セクションの「パーソナルコンピュータ」での作業に関する本の中でMacintoshについて初めて読んだからです。 それから彼は私の頭からそれを完全に捨てました。



ポッドキャストの現象を発見し、Radio Tにサインアップしたとき、Appleの2番目の風に気付きました。 私はそこで永遠のテーマについて長い間話をしました（Umputunが曲がるのを止めなかったことに感謝します）、そしてMac OS X 10.5（Leopard）のリリースとともに引きずり出されました。 私のクラスメート、私の元クラスメート、学校の別の友人、インターネット上の何人かの友人が同じことに興味を持っていることが判明しました。 今、私たちは皆、それに止まらずに厄介なマコビディになりました。 私たちは、女の子と家族をAppleテクノロジーに移しました。 私たちは友人にコンピュータについて話し、彼らも興味を持ちます。 ちょうど今日、私の別の友人が「壊れた」。 彼はMacBookを借りようとしています。



これは普通の群衆、ファッションです。 しかし、最悪のことは、彼らがオカルト主義者のように振る舞うことに気付いた誰もが味を得るということです。 ポピーの栽培者はスティーブ・ジョブズの口を見るのが好きで、彼らは自分のコンピューターが「高速で、信頼性が高く、美しいなど」と言うことをみんなに伝えたいです。 でも、それはある種の複雑なものではなく、面白​​くないからです！ Mac OS Xが唯一のOSであると本当に考えている人はいません。 私は自宅とオフィスのコンピューターにFedora 8と9を持っていますが、友人のmakovodは暇なときにUbuntuを見るのが好きです。 そしてもちろん、Wineは万能薬ではないため、誰かがWindows XPの機関用バージョンを備えた仮想マシンまたはWindows Vistaを備えた急いで購入したラップトップを持っています。 オペレーティングシステムに関するものではありません。 そのことは人にとって一般的なことです-彼らの（多分、とてつもない）利点について話し、彼らの独自性について自慢することです。 Mac OS Xの市場シェアが他のシステムの3倍であれば、Appleはすべてがうまく詰め込まれていて、通常の台湾のPCは段ボール箱に入っていると叫ぶことに執着することはないでしょうか？



片付ける



ほとんどすべてのプラットフォームについて、インターフェイスを構築するためのガイドがあります。 もちろん、それはどのような目的のために書かれています-そのため、すべてのプログラムは親relativeのように見え、同様の原則で動作します。 これはすべて、例外なく、 AppleとGNOMEの両方、およびKDEの両方、そして奇妙なことにWindowsの一般的な環境すべてに存在します 。



GNOMEおよびKDEのLinux開発者はこれらの推奨事項に従うことを試みます。なぜなら、そのようなプログラムは必然的に適切なコミュニティによって評価および修正され、単一の外観に磨かれるからです。



Mac OS Xの推奨事項は少し異なります。このプラットフォームには、オペレーティングシステムで異なるプログラムを見ることに慣れていないという理由だけで、一般的に見慣れないインターフェイスを拒否するコミュニティもあります。 Mac OS Xのほとんどのプログラムは収益化されているため、開発者は単一のインターフェイスに関する推奨事項を考慮する必要があります。そうしないと、ユーザーは単に競合他社にお金を渡してしまいます。 プログラムの収益化が不十分であるか、無料の場合でも、通常は作成されます。これは「プラットフォームを愛して」と呼ばれます。 これは群衆とファッションから再び来たと推測できます。 そのような開発者は、ユーザーだけでなく、お気に入りのふわふわしたオペレーティングシステム用のプログラムを作成しているようです。 これらのポイントに加えて、 Apple Design Awardsがあります-WWDCで発表された最も革新的で美しく、歓迎的なプログラムに対する毎年の賞。



しかし、Windowsはどうですか？ おそらくダイアログボックスの[OK]ボタンを除いて、このプラットフォーム用に作成された多くのプログラムが、公式のインターフェイスの推奨事項とはまったく関係ないのはなぜですか？ おそらく、このプラットフォームの開発者の間では、誰もが最善の方法を知っているからです。 開発者は、学生、自信のあるフリーランサー、中小企業、大規模なアウトソーシング会社によって構築されたインターフェースから頭がすでに急勾配になっているユーザー向けではなく、自分用にプログラムを作成します。 誰もがエレガントな決断をしたいのですが、残念ながら、誰もがジェフ・ラスキンに生まれたわけではありません。 Windows用のプログラムを作成した場合（それは必然的で、自分で、コースで、または大学でプログラミングを学んだ場合）、特別な規則に従ってそれらのインターフェイスを構築することをお勧めします。



親のためのコンピューター



Mac OS XとグラフィカルLinuxは、偶然に壊れるのがかなり難しいです。 システムリソースまたは保護されたディレクトリを変更または削除しようとすると、管理者パスワードが要求されます。 これは、Windows Vistaの迷惑なユビキタスな「キャンセルまたは許可」ウィンドウにやや似ていますが、数十回しか発生しません。ユーザーの観点からすると、いくつかのプログラムをインストールするとき、さらにはコントロールパネルの重要な設定を変更するときです。 他のシステムではアクセス権がすでに正しく描かれているからです。



ポート制御はシステムによって実行されるため、brandmauerをインストールする必要はありません。 アンチウイルスをインストールする必要はありません。 誰もが思考に慣れているため、「すべてのウイルスがWindowsで書き込まれるのは、プラットフォームが一般的であるため」だけではありません。 マルウェアは、スーパーユーザーとして実行されていない場合、実際には何もできないためです（これが起こる可能性はどのくらいですか？）。



ある日、私は両親のために電話ホットラインを維持するのにうんざりしました。 何かが絶えず飛んで、彼らに落ちました、ブランドマウアーは質問をしました、プログラムは更新することを切望していました（他のウィンドウとは異なり、それぞれ独自のもので）。 時々、デフラグを開始するために訪問する必要がありました。そうしないと、システムの速度が低下し始めます。 最終的にドキュメントをOpenOffice.orgに転送した後、Ubuntuでそれらをインストールしてファイルしました。 ギフトに余分なお金があれば、少なくとも彼らのためにMac Miniを購入します。そうすれば、何もカットする必要はありません。 それは「お母さんのために買うコンピューター」だからです。



シンプル-悪くない



誰もがMac OS Xのすべてが他の場所よりも単純であることに気づいたわけではありません。 そして、これはあまり大きな誇張ではありません。 1つのアイコンを[アプリケーション]フォルダーにドラッグして、Mac OS Xに多くのプログラムがインストールされているとします。 ここにあるほとんどのプログラムは、すべてのライブラリとプログラム設定を含む単一の.appアーカイブファイルです。 これは、1つのファイル（プログラム）をアーカイブにアップロードし、それを友人に送信できることを意味します（既にそうしている）。 彼は自宅でそれを開梱し、自分にとって都合の良い場所にドラッグすると、まったく同じプログラムが使用されます。 エンドユーザーには、レジストリ、.dll、構成ファイル、外部リソースはありません。 上級ユーザーは、必要に応じてこれらすべてを.appファイルから取得できます。



プログラムは見た目も動作も美しくするだけでなく、Cocoaのグラフィカルシェルとシステム内のナビゲーションの原理によってこれを強制されます。 プログラムでは、見出しの下にメニューがありません-メニューバーと呼ばれるストリップで上に配置されます。



その内容は、アクティブなウィンドウに応じて変わります。 とても正しいと思います。 ツールバーのホットキーやボタンを使用する代わりに、トップメニューで何かを検索する必要がある頻度を考えてください。



一方、Cocoaは、構築中にインターフェイス要素を統合する特定の原則を規定しています。 ウィンドウタイトルの右側にあるボタンでツールバーを最小化するとします。 または、ウィンドウのリストをグループ化するのに便利な引き出しパネルを移動します。



まあ、まれなプログラムは、その機能の全範囲の前に展開されます。 通常、最も頻繁に使用される最小限の機能のみが「表示」され、他の機能は「公開」され、最初の要求時に表示されます。 これは、ダイアログボックスで特に顕著です。 そのため、デフォルトでは、印刷ウィンドウは2つのドロップダウンリストと4つのボタンで構成されています。 そしてほとんどの場合、それは本当に便利です。



カルト



誰かがAppleテクノロジーに夢中になっている、誰かが冗談を言っている、誰かが真面目すぎてホリバーに参加できない。 しかし、一つだけ言えることがあります-マコボディ（ マックボーイズ ...これは多くあります）は自分で現れませんでした。 彼らは、「コンピューター= Windowsを搭載したPC」というパラダイムから脱却できないほど保守的であることが判明した社会からそう呼ばれていました。 それはとても受け入れられたものであり、そこにインストールされているものは平均的なユーザーにとって興味深いものではありません。 主なものは、インターネットアイコンをクリックしてお気に入りのニュースフィードを読むことができるため、シートの形でアイコンをクリックして、Word文書をさらに編集できることです。 彼は単純に、これらすべてを行うことがより簡単で楽しいシステムがあることを知りません。 または彼は信じていない-そして、彼は少し余分なお金を払って良い製品を買うことに慣れているという点で彼とは異なるマコボディを笑う。
