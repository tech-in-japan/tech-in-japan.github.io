TechnotronicsのKUB-Piko / 220コントローラーソケット

みなさんこんにちは！



述べられたトピックに移る前に、長年働いている会社を紹介したいと思います。



Technotronicsは、ハードウェアとソフトウェアの複雑なCensor-Technotronicsの開発者および製造者です。 この複合体は電気通信業界で広く知られ、通信事業者の経済全体を保護および監視するための派遣サービスを組織することができます-電気通信キャビネット、コンテナ、自動電話交換、ラインケーブル構造（井戸、ケーブル、配電キャビネット）。 このように、私たちは8年以上にわたり、監視センターに配置するためのディスパッチセンターと特殊なコントローラー用のソフトウェアを開発および提供してきました。 ちなみに、このシステムは通信業界以外にも適用可能です。データセンター、ATMの監視には、アパートや家などを監視するためのソリューションがあります。



機能、実績、その他の詳細については、会社についてのビデオをご覧ください。 より速く、より理解しやすいものになると思います。







そして今、ポストのメイントピック-私たちが開発したコントローラーKUB-Pico / 220。 ここにあなたの前にあります。











図 1.コントローラーKUB-Pico / 220の設計





秘＃1：建設的





このコントローラーの主な機能は構造にあります。 ご覧のように、単相220ボルト電源システムのコンセントに直接差し込むプラグの形で作られています。



同時に、ユニットにはソケットが装備されており、電源を供給したり取り外したりすることで、制御するデバイスの電源を入れることができます。 その結果、KUB-Pico / 220では、特定のアルゴリズムに従って、センターからのコマンドと自動の両方で、さまざまな負荷を簡単かつ簡単に管理できます。 このような制御の最も重要な特定のケースは、ハングしたデバイスの再起動と再起動です。



チップ2：カスタマイズ可能なI / Oポート





KUB-Pico / 220の2番目の機能は、次のように構成できる6つのI / Oポートの存在です。











ユニバーサルI / Oポートは、機能のうらやましい柔軟性を提供します。 必要な機能にデバイスを構成できます。 設定はコントローラーのWEBインターフェースを介して行われ、使用中いつでも簡単に変更できます。



入出力ポートの対応する構成により、次のことが可能になります。

  • 6つの個別センサー 「ドライコンタクト」の状態を監視します。
  • 電圧出力を備えた5つのアナログセンサーの状態を監視（測定）します（温度、湿度、定電圧、圧力センサー、ガスセンサーなど）。
  • 抵抗出力で5つのアナログセンサーの状態を監視します。
  • パルス出力 （電気、水、ガスなどの消費のカウンター）を備えた4つのデバイスからのデータを要約します。
  • パルス出力を備えた4つの衝撃 （振動） センサーのトリガーを決定します。
  • DCからのコマンドによる機器制御 （オン/オフ）のための5つのコマンドを提供します。
  • チップキー/近接カードのサイレンまたはリーダーを1つ接続します （アクセス認証を整理するため）。


上記のすべての機能を同時に使用できないことは明らかです。 特定の方法でポートを構成すると、ユーザーはそのポートを使用して他の機能を実行できなくなります。



チップ3：温度センサー





独自の設計の温度センサーがKUB-Pico / 220に接続されています。 センサーは、特定の条件下で、火災検知器として機能します。 実際には、温度の変化が速すぎると、システムがアラームを発します。 ただし、センサーを火災マーカーとして使用するのは、たとえば通信キャビネットなど、容量が小さい施設でのみ行うことをお勧めします。



チップ 番号4： ネットワーク機器の自動再起動





KUB-Picoには、監視センターサーバーのIPアドレス用の「ping」ノードがあります。これは、施設でハングしたネットワーク機器を自動的に再起動するために必要です。











図 2.凍結した機器を再起動するスキーム



センターとの通信：イーサネット、SNMP、WEBテクノロジー！





物理レベルでは、KUB-Picoコントローラーは10 mb / sの速度でイーサネットチャネルを介してMonitoring Centerサーバーに接続されます。

KUB-Picoデバイスに実装されているイーサネット、SNMP、WEBテクノロジーは、デバイスデータをディスパッチセンターと交換するいくつかの方法を提供します（図3を参照）。











図 3.センターとのKUB-Pico / 220デバイスの通信方法



私たちにとって最も伝統的なオプションであり、ユーザーにとっての完全なオプションは、TCP / IPモードのイーサネット経由で監視センターにデータを転送することです。 ここで、受信した情報は処理され、Technotronix.SQLソフトウェア、KUB-TTx、および当社の他の標準ソフトウェア製品で表示されます。 この場合、施設でのイベントのSMSおよび電子メール通知を構成できます。











図 4. Technotronix.SQLソフトウェアの外観











図 5. KUB-TTxソフトウェアの外観



ユーザーが既に標準のSNMPプロトコルをサポートするソフトウェアを持っている場合、SNMPプロトコルを介してイーサネット経由でデータを監視センターに送信することができます。



そして最後に、デバイスの組み込みWEBインターフェースのおかげで、イーサネット経由でデータをセンターに転送することができます。データは標準のWEBブラウザー（Internet Explorer、Opera、Google Chromeなど）に表示されます。 システムが1〜3 KUB-Pico / 220コントローラーで構成されている場合、デバイスのWEBインターフェイスを使用して監視と設定を行うソフトウェアを購入できないことがわかります。



上記のすべての特性を組み合わせることで、デバイスを独自のものとして認識し、開発の特許を取得することができました。













KUB-Pico / 220はどこで使用できますか？





このコントローラーは、通信キャビネット、データセンター、産業施設の監視に適したオプションです。 機器の制御/再起動が必要な場合はいつでも適用できます。 このデバイスは家庭での使用に適していると確信しています。 また、KUB-Pico / 220の価格はあまりかみません-デバイスの消費税は3,540ルーブルです。
