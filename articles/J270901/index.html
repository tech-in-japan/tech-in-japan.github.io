世界からの出来事のダイジェストD No. 2

最近、Rust and Goのトピックに関するHabréの多くの出版物があり、Dについてかなりの部分があります。これの主な理由は、MozillaとGoogleの言語のサポートです。 Dの背後は、同じくらい大きな単一の会社ではありませんが、これは言語の積極的な開発を妨げるものではありません。 次に、この言語の最新ニュースについて簡単に説明し、他の人にRust / GoとDの包括的な比較を行う機会を残します。



1.何よりもまず。 Dは最終的にC ++からD自体に変換されました。 これは、全体的なインフラストラクチャの開発における重要なステップであるだけでなく、開発プロセスを大幅に簡素化しました。 Dの開発に興味がある人すべてがC ++を知っているわけではありません。 コードの単純な書き直しではなく、DMDコードベース全体をC ++で半自動変換したことに注意してください。 そして、これらは数万行のコードです（はい、DMDは小さくありません）。 すべてがどうなったかについては、このビデオで見ることができます：







2. Dは、C ++コードとの統合を促進することに積極的に取り組んでおり、C ++プログラマーをDに切り替えるプロセスを簡素化するはずです。現時点では、Dは既存のC ++コードとほぼシームレスに統合できる唯一の言語です。 RustもGoもこれを許可できません。 この動きは、公式のDMDコンパイラ側とサードパーティのLLVMベースのコンパイラ側の両方からもたらされています。 最初のC ++コンパイラ（Cでの中間翻訳なしで作業）の著者であるWalter Brightは、DでのC ++例外の処理をサポートするために積極的に作業していると報告されています。



C ++コードとの統合も、 Calypsoプロジェクトによって積極的に推進されています。Calypsoプロジェクトの最初の公開リリースはまもなく行われる予定です。 これまでのところ、Linuxユーザーはテストビルドを利用できます。これにより、Qt5やOgre3Dなどの複雑なライブラリでもビルドできます。 Build for Windowsは、最初のパブリックリリースで利用可能になります。 それで、私たちは何を待っています。



一般に、カリプソの開発計画は次のとおりです。

C ++例外のサポートの改善-> C ++ 11のサポートの改善-> MSVCサポート-> magicport2の改善（C ++コードをDに変換するツール）->完全なCalypsoバストラッピング（Clangのバインダーを作成しないように） -> LLVM / ClangのライブラリとしてのCalypso。



3.モバイルプラットフォームのサポートに積極的に取り組んでいます。 現時点では 、言語ランタイムとPhobosライブラリがAndroidデバイスのほぼすべてのテストに合格したことが報告されています。 iOSの本格的な開発サポートについて話すことは不可能ですが、 LDC-iphoneプロジェクトの努力のおかげで、今日iOS向けのシンプルなDアプリケーションを作成できます。



同じことがメインDMDコンパイラにも当てはまります。これは、最新バージョンではObj-Cの基本的なサポートを受けていました。



すべてがこのようなペースで進むと、今後数年間で、Dはモバイルデバイスの巨大なニッチを開放し、その人気に有益な効果をもたらすはずです。



4.非営利組織D Language Foundationが設立されました。 ミッションは、コミュニティDを促進することです。この財団は、DMD参照コンパイラの開発、知的財産権、DConfを含む開発者会議の組織、資金調達など、コミュニティD内のさまざまなプロセスを担当します。



5.長い予防作業の後、 dlang.ruは復活しました



6. vibed Webフレームワークはいくつかのバージョンを変更し、モジュール性を高めるコースを継続しました。



7. D.でのプログラミングに関する複数の書籍が、すぐに出版できるように準備されています。
