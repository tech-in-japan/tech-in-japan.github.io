Dropbox-測定を続けます

退屈な仕事...それでは。 こことここで提供されたものをさらに自動化しよう=）



今回は簡単なスクリプトしかありません。 これにより、オンラインの紹介をご自身で簡単に参照できます。 はい、MACアドレスは自動的に更新されるため、仮想マシンの必要性は本質的になくなります...スクリプトを操作することをお勧めしますが。 単純に、その作業の過程で、スクリプトが定期的にドロップボックスの構成を上書きするためです。



私たちは何が起こったのか、それについて何をすべきかを見て... 



#!/bin/bash DEST=`pwd` #enter here your referal link from Dropbox REFLINK="http://db.tt/V1ESzte" download() { echo "Downloading Dropbox distributive..." cd wget -O dropbox.tar.gz "http://www.dropbox.com/download/?plat=lnx.x86" cd $DEST } update_mac() { echo "Update mac address..." ifconfig eth0 down ifconfig eth0 hw ether `printf '52:54:00:%02X:%02X:%02X\n' $((RANDOM%256)) $((RANDOM%256)) $((RANDOM%256))` ifconfig eth0 up dhclient eth0 ifconfig eth0 echo "MAC updated..." } extract() { echo "Unpack archive" rm -rf .dropbox-dist tar -tzf dropbox.tar.gz tar -xvzf dropbox.tar.gz } register() { cd echo "Registering new referal..." rm -rf .dropbox update_mac echo "Register new referral... Now enter random Login (like your_gmail_login+blablabla@gmail.com) and Password (like '123456'). Close Firefox after register new user!" firefox $REFLINK; echo "Choose 'I already have Dropbox account', enter your login and password. Click Next, and close installer. Repeat =)" ~/.dropbox-dist/dropboxd cd $DEST } case $1 in "download" ) download ;; "extract" ) extract ;; "register" ) register ;; * ) echo "Usage: Simply execute '$0 download && $0 extract' and than repeat '$0 register' many times as necessary";; esac exit 0


実際、このスクリプトは基本的なものであり、UbuntuがVirtualBoxにインストールされている間に作成されました。



使い方は簡単です...始めるために、始めましょう： 

 ./dropcheat.sh download && ./dropcheat.sh extract


つまり、ドロップボックスがダウンロードされ、ユーザーのホームディレクトリに展開されます。 そして、青になるまで繰り返します。 ご想像のとおり、MACアドレスを変更するにはsudoが必要です。 

 sudo ./dropcheat.sh register


ここで何が起こっているのか...まず、最初に、各繰り返しで、古いドロップボックスの設定がトリミングされます。 その後、MACアドレスが更新され、Firefoxは$ REFLINK変数に格納された参照リンクで起動します（ちなみに、変更するのを忘れないでください。 開いているブラウザウィンドウで、ナンセンスを入力します。主なことは、次のステップでそれらを必要とするためです。Dropboxが起動し、「Cho、how？」と尋ねます。 さらに、終了し、再びスクリプトの起動に戻ります...



例：





登録ページを自動的に解析し、ランダムな値を置き換えないのはなぜですか？それは、どういうわけかドロップボックスのウェルカムウィンドウに転送されますか？ それには時間がかかりすぎ、費やさなければならない十数回の反復には価値がありません...



UPD：ああ、スクリプト自体はここにあります： dl.dropbox.com/u/9408151/dropcheat.sh突然誰かがそれをコピーしたくない場合...



UPD2： DropBoxの仕組みと、この小さないたずらをするきっかけとなった動機について、若干の誤解があります。 何らかの理由で、彼らは盗難で告発されています...



順番に。 まず、各アカウントのドロップボックス（およびその他のクラウドストレージ）の誰もが必要なギガバイトをすべて事前に予約しないことを覚えておく必要があります。 必要に応じて、一定量のスペースが割り当てられます。 論理的に10 GBがあり、物理的に2つしか占有されていない場合、2 GBのデータがサーバーに保存されます。



二番目。 スクリプトは数分で作成できます。 ウブンタが置かれている間、私はここで窃盗は見ません、スポーツの興味だけです。 たとえば、どの読者がさらに先に進むことができるかを確認します。 それに。 サービスのセキュリティシステムが非常に慎重に考えられている場合、そのようなホールを使用しないことは、たとえ楽しみであっても悲惨です。 こんなに簡単で楽しく、目立たないハックが突然犯罪に変わったのはいつですか？ さらに、私自身はテスト目的でスクリプトを2回だけ実行しました。



念のために、スクリプトの操作に対するすべての責任は、スクリプトを使用する人にあると付け加えます。ここでは、人気のあるサービスのセキュリティシステムを回避する例として、それが教材として提示されているからです。
