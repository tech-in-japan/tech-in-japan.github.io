Sergey Vakulaのインタビュー：「ブロックチェーンと暗号通貨が巨大になるとは思わない」

今日は、セルゲイ・ヴァクラについてお話したいと思います。 彼は3年間私たちと一緒にいて、Emercoinのほとんどすべてのプロジェクトに参加しています。 E1モバイルウォレットエクスチェンジ 、Emerkoin Webウォレットなどのプロジェクトにより、 Ubuntu 、 RedHat 、 Debian 、Emercoin Microsoft Azureウォレットのウォレットアセンブリが常にサポートされています。



また、 Blockchainizer （暗号通貨を使用せずにEmerブロックチェーンにデータを保存および処理できるサービス）およびAuthorizer （Geektimesに記事があります ）。

また、 EmerDPOおよびEmerVPN（VPNサーバーとして使用できるWebウォレットの拡張機能）。





Aspanta LimitedのヘッドであるSergey Vakula



詳細については、質問と回答の形式に頼ることにし 、暗号通貨の世界で最高のニュースサイトであるCoinFoxのジャーナリストにSergeyにインタビューするように依頼しました。



CoinFox：ブロックチェーン技術をどのようにして知りましたか？ 彼女はどのようにあなたを魅了しましたか？



Sergey ：2015年末、EmerkoinテクニカルディレクターのOleg Khovaiko氏は、 Microsoft Azureに基づくソリューションに関するHabréに関する私の記事を読みました。Emercoinはそのようなソリューションを必要としていました。 オレグは私を見つけ、話し合い、共通点を見つけました。 その瞬間まで、私は暗号通貨の存在を知っていましたが、これらはすべて取るに足らないように思えました。 そうでなければオレグは私を説得した。 私たちは協力し始めました。私はトピックに突入し、ブロックチェーンを使用することがいかに重要であるか、テクノロジーはIT分野の突破口であり、公共サービスや他の多くの範囲を変えることができることに気付きました。



Emercoinと並行して、ビットコインとエーテルの研究を開始し、Emerが最も使いやすいことが判明しました。 技術的な観点からEmerの使用を検討すると、CoinMarketCapレーティングの結果で主要なラインを占有していないという事実にもかかわらず、彼がBitcoinおよびEtherと比較して勝つことがすぐに明らかになります。 Emerシステムは非常に単純に構築されているため、同じEtherと比較して、脆弱性の出現は実質的に排除されます。 イーサリアムでのスマートコントラクトは、概してクールなものですが、それぞれのケースでそれを使用することは、Emerを使用できるほど賢明ではありません。 ブロックチェーンに何かを保存するだけで、ブロックチェーンから読み取る場合は、そのようなスマートコントラクトは必要ないため、情報を保存するだけです。 それがEmercoinの仕組みです。 はい、いくつかの点で、EmerkoinはEtherよりも劣っていますが、ICOに関連しないタスクのほとんどは、Emerkoinがチューリングコードを実行せずに実行できます。



CoinFox：ICOについてどう思いますか？



セルゲイ：私はまだ関係する方法を理解していません。 多くのプロジェクトがICOに多額の資金を集めており、ICOがさらにどのように振る舞うのか、時間はわかります。 しかし、どのような場合でも、詐欺に関与する可能性のあるものは、誰が言っても、州によって規制されるべきであるように思えます。



CoinFox：私たちが知る限り、あなたの開発の多くはEmercoinと共同で行われました。 どのような開発が最も成功していると思いますか？



Sergey ：最も成功したDPOテクノロジー（EmerDPO） -ブロックチェーンを通じて何かの所有権を証明するテクノロジーだと思います。 仕組み：たとえば、店に来て高価な商品を購入したい場合、特に旧ソ連の国では、これが偽物ではないことを100％確信することはできません。 すでに製品を購入しており、その品質に失望している場合、常に返品できるとは限りません。 DPOテクノロジーを使用すると、店舗に来て製品のQRコードをスキャンすると、特別なWebページが開き、この製品が以前に購入されたかどうかが示されます。 次に、家に帰って商品を開梱し、内部に別のQRコードがあります（購入後に商品が開梱されるまで利用できません）。 もちろん、私たちは高価な商品について話しているので、安価な商品にQRコードを掛けることは意味がありません。 または別の例：ワイン生産者は、たとえば、特定の地域の特定の作物からワインのボトルがいくつ生産されたかについてのブロックチェーンに関する情報を入力できます。また、店舗でワインを購入すると、QRコードをスキャンして理解することもできますあなたが偽のワインを手に持っているかどうか、ワイン生産者がカンニングされたかどうか、その量はすべて宣言された作物と領土から生産されたと主張するかどうか。 これが、弊社の開発が顧客を偽物から保護する方法です。



EmerDPOに加えて、共同プロジェクトも成功しています。 たとえば、ブロックチェーン保護WebサーバーのスクリプトはEmcProxy（ GitHub ）です。 これは独自のサーバーにインストールされるプログラムであり、動作するにはEmercoinウォレットが必要です。 ある種のアプリケーションを作成したいと想像してください。このアプリケーションには、公開されているブロックチェーンからの情報が必要です。 たとえば、ウォレットの請求先住所だけでなく、ブロックチェーンに保存されているその他の操作の履歴もあります。 コイン、秘密鍵、その他のパーソナライズされた機能を送信するという観点から、ウォレット自体にアクセスすることなく、アプリケーションがウォレットから公開レコードを確実に受信するようにする方法を教えてください。 EmcProxyは、許可されたコマンドのリストにリストされているリクエストのみをスキップしながら、ブロックチェーンの外部からリクエストをパススルーできるサービスです。 ブロックチェーンにアクセスするには、個人情報と公開情報の両方を含むEmercoinウォレットをインストールする必要があります。 EmcProxyを使用して、この情報を区別します。 EmcProxyでは、公開情報に関連し、個人情報に関連しないコマンドのみをブロックチェーンに処方できます。 ソフトウェアがEmcProxyを介してウォレットに接続すると、公開情報は転送されますが、個人情報は転送されません。 アプリケーションがブロックチェーンにアクセスしたい場合、EmmProxyを介してブロックチェーンAPIを開くパスをたどることができます。これにより、ウォレットにアクセスできるようになり、パブリックとプライベートの情報が区別されます。



他の開発はEmerVPN（ GitHub ）です。 物理的には、これはアプリケーションではなく、OpenVPNを介した構成ファイルと証明書の形式のアドインです。 EmerVPNをインストールすると、標準のOpenVPNサーバーがインストールされ、ユーザーがブロックチェーンを介して認証されるように設定されます。 EmerVPNサーバーは、インストール後すぐに操作できる状態になります。Emercoinブロックチェーンをダウンロードするだけで時間がかかります。 サーバーにコマンドを実行してEmerVPNをインストールすると、OpenVPNと多数の依存関係がプルされ、その中にEmercoinウォレットがあります。 Emercoinウォレットがシステムにインストールされるとすぐに、Emercoブロックチェーンのダウンロードが開始されます。 このプロセスを完了すると、すべての準備が整います。 他に何も設定する必要はありません。 ブロックチェーン経由でVPNへのアクセスを許可されたユーザーを管理するにはどうすればよいですか？ VPNへのアクセスを許可されているユーザー識別子（ID EmcSSL証明書）を含むエントリをNVSに作成する必要があります。 このNVSエントリの名前をEmerVPN構成ファイルに登録したら、NVSでこのエントリを編集して、VPNへの接続を許可されたユーザーのリストを制御できます。 サーバー上の構成の編集はもう必要ありません。



他の開発に関しては、ウェブウォレット。 最初は、Webインターフェースを介した最も効率的な操作を保証するために、サーバーにWebウォレットがインストールされていました。 エストニアの同僚によって書かれたこのようなウォレットの最初のバージョンがあり、基本的な機能のみが含まれていました。 このバージョンに基づいて、最新のテクノロジー、フレームワークを考慮して、アプリケーションの内部構造全体を95％再設計しました。 過去1.5〜2年にわたって、すでに約5つのバージョンのウォレットを作成しています。 現時点では、Webウォレットは通常のウォレットと同じことを実行できます。これはWindowsまたはMacなどに搭載されています。 たとえば、ウォレットの最新バージョンには、すでにEmerSSL 、EmerVPN、EmcLNX、 EmerDNSのサポートが含まれています。 つまり、機能的には、WindowsまたはMac用の標準のウォレットアプリケーションよりもはるかに高度です。



CoinFox：Microsoft Azureを使い始めましたが、今それを使用していますか？ または、他の技術に来ましたか？



Sergey ：はい、もちろん使用しています。 当社のAspantaはマイクロソフトのパートナーであり、すべてのオンラインテクノロジーはAzureのみで構築されており、従業員は認定スペシャリストです。そのため、この分野で非常に真剣な専門知識を提供できます。



CoinFox：今後の計画は何ですか？ 現在、どのプロジェクトに取り組んでいますか？



Sergey ：現時点では、ブロックチェーンプロジェクトの開発に完全に方向を変えています。私たちはそれらを最も興味深く、有望だと考えています。



CoinFox：ブロックチェーンの未来をどのように見ていますか？



Sergey ：テクノロジー自体、特に暗号通貨が巨大になるとは思いません。 むしろ、一部のプロジェクトはブロックチェーン上で実装され、一般の人々にもっと馴染みのある形で聞かれます。 たとえば、暗号通貨での支払いはボラティリティがあるため簡単ではなく、誰にも秘密ではありません。 また、トランザクションの確認に時間がかかり、これも不便です。 ブロックチェーンの決定を下すことは、例えば、ユーザーフレンドリーなアプリケーションを介して何らかの情報を入力することとはまったく別の問題です。これは、ブロックチェーン内の生のレコードを使用するすべての人がいつでも確認できます。 これは、オープンソーステクノロジーに匹敵します。たとえば、Chromeの仕組みを知りたい場合は、GitHubにアクセスしてご覧ください。 または、ビルドした人を信頼する場合は、既製のバイナリアセンブリを使用します。 ブロックチェーンでも同じである必要があります：誰もがシンプルで手頃な手段（アプリケーション、ウェブサイトなど）を使用でき、必要に応じて、ブロックチェーンから直接データを要求することにより、自分でデータの正確性を検証できます。 これは主なものである必要があります-使用するデータを検証する機能を備えた、すべての人がテクノロジーを利用できること。
