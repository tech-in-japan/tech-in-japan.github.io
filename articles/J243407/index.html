Odroid-W + WドッキングボードLCD

こんにちは おそらく私の記事は少し「nubovaty」に見えるかもしれませんが、個人的にはロシア語を話すインターネットで同様のHOW TOを見つけることができませんでした。 HardkernelのシングルボードOdroid-Wコンピューターへのオペレーティングシステムの初期展開についてです。これは、説明的な性質の2つの 記事についてHabréで既に公開されています。 特別な機能は、「 TFT LCDを備えたWドッキングボード 」ボード自体と一緒に使用することです。 この鉄片の能力は完全には明らかにされていません。



私のネマリナ






私が話したいこと：

1）オペレーティングシステムの準備とインストール。

2）外部ディスプレイへのグラフィックス出力。

3）「Wドッキングボード」ディスプレイにグラフィックを表示する。

4）Webカメラを接続します。

5）Wi-Fiドングル接続。



この記事がブラックホールに飛び立たない場合、次回の予定は次のとおりです。

6）4Gモデムを接続し、Wi-Fi経由でインターネットをその後配布します。



Odroidには、 反悪性のDebianを配置します。 すべての準備作業はWindows 7がインストールされたPCで行われますが、nixベースのOSでの準備作業ははるかに簡単になり、追加のソフトウェアは必要ありません。



パート1



それでは始めましょう。 まず、4 GB以上のマイクロSDカードと、このカードリーダー（+アダプター）を取得する必要があります。 ここで主なルール-より良い。 もちろん、カードのクラスが高いほど、すべてが速く動作します。 最初に、最新バージョンのアセンブリのRaspbianディストリビューションと、ユーティリティSDFormatter 、 Win32DiskImager 、SSHクライアント（ Puttyなど ）をダウンロードする必要があります。

まず、SDカードをカードリーダーに挿入してSDFormatterを起動し、カードのドライブ文字、QUICK FORMATオプションを選択して、Formatをクリックします。 十分に高速に実行します。

次に、Disk Imagerを実行し、ダウンロードした配布イメージを選択して、カードに書き込みます。 それほど速くありません。

次に、メモリカードをOdroid-Wに挿入し、Wドッキングボードネットワークケーブルをマイクロコンピューターに接続します。 次に、少し待ってから、ボードのIPアドレスを判別する必要があります（ルーターに接続し、ルーターのWebインターフェースで対応するDHCPアドレスを見ました）。 次に、SSHクライアントに接続する必要があります（私は独自のSecureCRTを使用しています）Puttyは最も簡単で簡単な方法です。 私はそこでSSHを設定することにこだわるつもりはありません。第一に、すべてがシンプルで、第二に、インターネット上に十分な記事があります。 接続するには、piユーザー名とraspberryパスワードを使用する必要があります。 すぐにsudo suとpasswdを実行し、rootユーザーのパスワードを設定してからそこからログインすることをお勧めします（ただし、sudoを使用するこの方法には多くの反対者がいる場合があります）。



パート2



外部ディスプレイを接続するには、HDMIコネクタ付きのモニター/テレビ、HDMI-microHDMIケーブル、または機能に応じてDVI-microHDMIやその他の倒錯などの他のオプションも必要です。 モニターを接続した後、ほとんど何も画面に表示されませんが、これは非常に簡単に解決されます.nanoエディターを使用して/boot/config.txtファイルを変更するか、すぐにmcをインストールする必要があります 行hdmi_safe = 1のコメントを外す必要があります

Xの起動などのセットアップの小さな部分は、3番目の部分から収集できます。 モニターへの画像の出力にはあまり注意を払いませんでした。 これは、統合された画面を使用したポータブルソリューションであると想定されていました。



パート3



私はこのテキストを自分で発明しませんでしたが、公式の英語版ウィキで見つけました。 したがって、私は元の、さらなる翻訳といくつかのコメントへのリンクを提供します。 実際、この命令でさえ見つけるのはそれほど簡単ではありませんでした（少なくとも私にとって）。

ファイル/etc/modprobe.d/raspi-blacklist.confを編集します。コメントする必要があります。 



#blacklist spi-bcm2708


ファイル/ etc / modulesに追加します 



 fbtft_device name=adafruit22a verbose=0 rotate=90


実際、rotate = 270を使用しましたが、より便利であることがわかりました。

オフマニュアルには、自動ログインの作成方法が記載されていますが、この考えはあまり好きではありません。

ファイル/ etc / inittabにそのまま引用します

次の行を追加します。 



 #1:2345:respawn:/sbin/getty --noclear 38400 tty1 1:2345:respawn:/bin/login -f pi tty1 </dev/tty1 >/dev/tty1 2>&1


ここで、piはユーザーの名前です。

Xsの起動。 正直なところ、画面のサイズを考えると、使用することはお勧めしませんが、Odroid-Wで華やかな時計を作りたい人には贈ります

/etc/rc.localに追加します 



 su -l pi -c "env FRAMEBUFFER=/dev/fb1 startx &"


ここで、piはユーザーの名前です。

A.J. /usr/share/X11/xorg.conf.d/99-fbturbo.confのオプションを無効にします 



 # Option "fbdev" "/dev/fb0"


外部画面に画像を表示する（HDMI経由）

最後の仕上げとして、/ boot / cmdline.txtの行の最後に追加します。 



 fbcon=map:10


そして、再起動後、フレンドリーな点滅GUY LXDEが表示されます。



パート4



警告！ 以下、追加の機器へのリンクが提供されますが、a）広告ではない、b）紹介が含まれていない、c）購入は推奨されていません。

私は最も簡単な中国のnonameウェブカメラを使用しました。 これを使用するには、機能をインストールおよび決定するためのlibv4lおよびfswebcamと、カメラから画像をストリーミングするためのmjpg-streamerのパッケージをインストールする必要がありました。

最初の2つのパッケージはリポジトリにあり、aptitude install libv4l-0 fswebcamをインストールします

そして、SFから mjpg-streamer をダウンロードし、それを*都合の良いフォルダーに解凍します*（私にとっては、/ rootにあります）。

カメラをlsusbに接続すると、カメラに関連するものが表示されます（特にブランド化されたデバイスを使用している場合）。 



 pi:~# lsusb Bus 001 Device 007: ID 1871:0101 Aveo Technology Corp.


そしてdmesgで： 



 pi:~# dmesg [ 1182.284021] usb 1-1.4: new high-speed USB device number 8 using dwc_otg [ 1182.394987] usb 1-1.4: New USB device found, idVendor=1871, idProduct=0101 [ 1182.395067] usb 1-1.4: New USB device strings: Mfr=1, Product=2, SerialNumber=0 [ 1182.395087] usb 1-1.4: Product: USB2.0 Camera [ 1182.395135] usb 1-1.4: Manufacturer: AVEO Technology Corp. [ 1182.403956] uvcvideo: Found UVC 1.00 device USB2.0 Camera (1871:0101) [ 1182.408696] input: USB2.0 Camera as /devices/platform/bcm2708_usb/usb1/1-1/1-1.4/1-1.4:1.0/input/input4


次に、fswebcamを実行します。 



 pi:~# fswebcam --verbose


そして、この場合はカメラがYUYVモードで動作し、2つの解像度をサポートしていると判断します。 



 Using palette YUYV Adjusting resolution from 384x288 to 352x288.


次に、mjpg-streamer起動スクリプトを準備する必要があります。 説明付きの多数の例は、実際には./mjpg-streamer/start.shにあります

原則として、このファイルは使用できませんが、コンソールから直接実行できます。 私のオプションはこれです： 



 pi:~# ./mjpg_streamer -i "input_uvc.so -d /dev/video0 -r 384x288 -f 10 -y -q 100" -o "./output_http.so -w ./www" MJPG Streamer Version: svn rev: 3:172M i: Using V4L2 device.: /dev/video0 i: Desired Resolution: 384 x 288 i: Frames Per Second.: 10 i: Format............: YUV i: JPEG Quality......: 100


デバイスが示されている場所、解像度、1秒あたりのフレーム数、カメラモード、jpeg品質、および出力ファイル（ビルトインWebサーバー）。

http：// [odroid.w.ip.address]：8080でブラウザにアクセスし、カメラからストリームを見ることができます。 できた！

Odroid -Wには、マトリックスをSoCに直接接続するためのmipi-csiインターフェイスがありますが、これらのカメラでの作業は舞台裏で行われます（コストはあまり良くありません）。
