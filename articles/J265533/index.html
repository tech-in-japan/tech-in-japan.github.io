Dell ASMとCisco UCS：インフラストラクチャ自動化のテスト方法

この投稿では、データセンターにインフラストラクチャを効率的に展開するためのDell ASMソリューションの実際の作業について説明します。 この製品はITシステムをより迅速かつ簡単に構成するのに役立つため、主要な競合製品であるCisco UCSシステムと比較することにしました。



最初に発見された違いは、ほとんどすべての場合にDell ASMにインストールウィザードが存在することです。 インフラストラクチャ要素を構成するための段階的なアルゴリズムが含まれており、情報システムの展開のシナリオごとにスクリプトを手動で記述する必要があったことを忘れることができます。 たとえば、Dell ASMには、すぐに使用できるVMware vSphereクラスターを作成できるウィザードが既にあります。 システム自体が機器を発見し、ネットワークを特定し、最適なリソース構成を選択しました。



コンピューティング環境を作成した後、サービスの開始を開始しました。 Dell ASMでは、これはわずか10ステップで完了しました！ このシステムでサービスを起動するプロセスはより複雑になり、作業プロセス（ワークフロー）の開発に専門家の参加が必要になったため、Cisco UCS Directorで同じトリックを行うことはできませんでした。 したがって、すでにシステムに精通している段階で、Dell ASMが日常的なタスクの実装を簡素化し、管理者がインフラストラクチャの形成に費やす時間を短縮することが明らかになりました。



Dell ASMとは何ですか？



現代の企業は新しい機器をインストールし、既成のサービスを可能な限り迅速に起動することを望んでいるため、Dell ASMおよび同様のツールが人気を博しています。 組織とその顧客の両方がこの恩恵を受けています。 さらに、管理者にとっては、日常的なタスクをプログラミングして実行するよりも、自由時間を使ってデータセンターを開発する方が有利です。



Dell ASMは、インフラストラクチャの自動化とワークロード分散のためのソフトウェアです。 システムには、インフラストラクチャ要素を管理し、新しいサービスを自動的に起動できる単一のインターフェイスがあります。 この統一されたアプローチにより、機器の検出、実装、更新、修理の際にさまざまなコンソールでの作業に費やす時間を削減できます。 また、Dell ASMの主な違いは、各ステップおよび各ケースで既製のセットアップウィザードを使用していることです。 このソリューションの利点の中で、次の点を強調しました。

  • リソースの発見の簡素化と高速読み込みにより、できるだけ早くサービスを開始できます。
  • テンプレートベースのリソースを提供します。 複雑なインフラストラクチャ要件を満たしながら、ITサービスに必要なリソースを簡単に割り当てることができます。
  • インフラストラクチャのスケーリングと更新、および規制要件へのコンプライアンスの監視を含む、サービスライフサイクルのサポート。


お気づきかもしれませんが、仕様は有望に見えます。 しかし、ポイントに行きましょう。



テスト中



Dell ASMとCisco UCSの両方を自由に使用できるようになったため、手間をかけて、今日最も人気のあるインフラストラクチャソリューションの1つであるVMware vSphereクラスターの展開プロセスを開始しました。 Dell ASMの場合、この手順は非常に簡単で、機器の電源を入れてから仮想マシンを起動する準備ができたというメッセージが表示されるまでに15分57秒しかかかりませんでした。 Cisco UCSでは、クラスターの展開に1時間19分5秒かかったため、結果はそれほど印象的ではありませんでした。 したがって、Dell ASMスコアはCisco UCSより79.8％優れていました。



しかし、それだけではありません。 Cisco UCSの場合、ドキュメントに記載されている手順に従ってインストールウィザードを起動するのに必要な時間を推定しました。 実際には、UCS Directorでタスク用のウィザードを作成するのに必要な時間はさらに長くなります。 その理由は、サービスプロファイルを作成する機能と、Cisco UCSで追加された機能をよく理解する必要があるためです。 手順中に、必要に応じてエラー、再起動、およびスクリプトの変更が可能です-これは規定されるべきです。



Dell ASMソリューションでは、まったく異なる作業アプローチが実証されています。 すぐに使用できるクラスターを作成して、Cisco UCSと比較して315少ない手動操作を実行できます。 これは、ヒューマンファクターの役割が70.6％削減されることを意味します。



違い



まず、Dell ASMは、単一のインターフェイスを介してvSphereクラスター展開全体の自動化を提供します。 Cisco UCSを介して、Cisco UCS ManagerとUCS Directorの両方を使用してのみ同じことを行うことができました。



両方のシステムで作業を開始したとき、Dell ASMとCisco UCSのテンプレート作成手順の大きな違いに気付きました。 Dell ASMには、管理者がサービスを開始して作成するのに役立つ15のサービステンプレートが含まれています。 エディターを使用すると、サービスの実装中に物理コンポーネントと仮想コンポーネントを追加する明確な階層を作成できます。 数回クリックするだけで、ストレージシステム、コンピューティングノード、仮想マシン、オペレーティングシステム、および必要なアプリケーションを追加し、既存のサービステンプレートに追加することができました。



Dellソリューションを使用すると、オペレーティングシステムをベアメタルにインストールしたり、Microsoft SQL Server 2012などの既成の構成を作成したりできます（実験で使用しました）。 テンプレートの公開は数回クリックするだけで実行され、実行中のサービスのステータスの監視はDell ASMダッシュボードを介して実行されます。



UCS Directorで同じサービスを起動しようとすると、多くの困難に直面しました。 実際、シスコのソリューションの機能は、ワークフローデザイナによるワークフローの作成に依存しているため、ワークフローを使用して独自のウィザードを最初から作成する必要があります。 Cisco UCS Directorからは、ワークフローのパターンが5つしか見つかりませんでした。 また、たとえば、iSCSIストレージを備えたvSphereクラスターを実装するには、コミュニティを掘り下げ、既製のコンピューティング環境に対応する独自のテンプレートを開発する必要がありました。 また、既存のテンプレートを変更するには、その機能をよく理解する必要があります。



Dell ASM Editorは、仮想および物理コンポーネントを個別に構成するプロセスを簡素化します。 各ステップが記録され、プロセスが自動化されたため、サービスを実装する際の手順の開発を回避することができました。 対照的に、Cisco UCS Directorはプロセスの詳細を理解し、それを完了するために必要な手順を追加する必要がありました。 Cisco UCSフレームワークは各タスクに番号を付け、後の手順で以前のレベルの結果を使用できます。 実践が示しているように、このアプローチでは、追加のコンポーネントを追加するときに困難が生じます。 たとえば、特定の前のステップがないためにウィザードがエラーを生成する場合、問題を解決する必要がない場合でも、ワークフローに追加する必要があります。



Cisco UCS Directorでのサービス展開のテストには、管理者から数日かかりました。 同時に、Dell ASMテンプレートは最初は使用可能な状態であったため、ワークフローの作成段階をスキップできました。 UCS Directorで達成できた最高の結果は、1つのサービスを開発するのに2日間でした。 Dell ASMは、すぐにサービスを開始できました。



保存中



私たちの評価では、IT環境の管理コストを忘れていませんでした。 そのような手順は非常に高価になる可能性があり、ユーザーのニーズはIT部門の予算よりもはるかに速く成長しています。 この状況では、サービスを迅速に提供し、データセンターを管理できる自動化システムが役立つと思われます。 しかし、実験では、すべてがそれほど単純ではないことが示されました。



Dell ASMのシンプルな展開システムにより、あらゆるレベルの技術者がシステムを操作できます。 対照的に、シスコのソリューションには、さまざまなインフラストラクチャコンポーネントに関する膨大な経験と知識が必要です。 したがって、Dell ASMを使用すると、追加のトレーニングを受けたり、高度な資格を持つ新しいスペシャリストを雇用したりすることなく、インフラストラクチャの自動化を活用できます。



興味深い制御システムのライセンスの評価により、興味深い結果が示されました。 データセンター自動化向けのDell ASMのコストはノードあたり165ドル、シスコのソリューションのコストはノードあたり4,000ドルです。 したがって、Dell ASMは95％も手頃な価格です。



おわりに



テストの結果に基づいて、次のように言うことができます。面倒な手順と過剰な手動構成を必要とする高価な自動化ソリューションを使用するよりも、時間を費やす方が良い方法があります。 この比較では、既製のインストールウィザードを備えた使いやすいDell ASMシステムにより、ITサービスの起動プロセスが大幅に簡素化されます。



Dell ASMは、Cisco UCS DirectorおよびUCS Managerの同様のプロセスと比較して、既製のサービスを実行するのに必要な時間を約80％短縮し、手動操作を約71％削減することがわかりました。



しかし、ポイントは、Dell ASMが既製のVMware vSphereクラスターの起動をより速く簡単にすることだけではありません。 このシステムははるかに安価です。 Cisco UCSと比較して、Dell ASMの1つの計算ノードを自動化するコストは95％低くなります。



結論を出すのは非常に簡単です。80％-71％-95％の指標は、それ自体を物語っています。
