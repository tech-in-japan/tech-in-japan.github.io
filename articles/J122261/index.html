Haiku OS-R1 / Alpha 3リリース

遅延はありましたが、Haiku OSの次のテストリリース-R1 / Alpha 3が行われました。









以下は、Haiku R1 Alpha 3の3番目の公式リリースの注意事項です。リリースに取り組んでいるHaiku開発者は、リリースをより安定させ、エンドユーザーと開発者の両方により多くの機会を提供し、幅広い機器でのダウンロードの成功の可能性を高めようとしました。



このリリースをできるだけ良いものにするために、私たちは一生懸命努力しました。 ただし、いくつかの問題と未実現の機会が残っており、そのいくつかを以下にまとめます。 これはまだアルファ版であることを忘れないでください。不完全であり、既知および未知のエラーが含まれています。 このリリースの安定性には自信がありますが、データ損失に対する保証はできません。



Alpha 3のリリースにつながったもう1つの目標は、現在および将来のHaiku開発者に、プログラムで作業するための更新された（通常）安定したオペレーティングシステムを提供することです。



そのため、CVS、SVN、Git、およびHgクライアントとともに、コア開発者ツールを含めました。 このHaikuリリースでは、GCC2およびGCC4を使用して実行可能ファイルを構築および実行できます。 GCC4を使用する必要がない場合は、Haikuの将来のバージョンのAPIとの互換性が保証されないため、GCC4を使用しないでください。



システム要件



現在、Haikuはx86システムでのみ動作します。 RAMの最小量は128 MiBです。 Haiku自体からHaikuをビルドするには、1 giBが推奨されます。



新機能



R1 Alpha 2の以前のリリース以降に登場したHaikuの最も重要な改善点と追加点を簡単に確認してみましょう。リリース後、950以上のエラーが修正されたことに注意してください。 さらに、Coverityで見つかった250を超える微妙なバグが修正されました。



  • btrfs、exFAT、ext2、ext3、ext4、NTFS、UDFなどのさまざまなファイルシステムの読み取り（および書き込みサポート）が改善されました。
  • さまざまなハードウェアサポートの強化：IO – APIC、ACPI、ビデオカード用ドライバー、ネットワークカード、USB。
  • Guttenprintを使用して追加のプリンターサポートを提供します。
  • フォントの部分的なオーバーレイサポート。 たとえば、一部の日本語および中国語の文字が表示されるようになりました。
  • MediaKitのエンコードおよびデコードのサポートが改善されました。
  • Tim Brechbillが作成したGeneral MIDI SoundFontを追加し、多くのアプリケーションにMIDIサポートを追加しました。
  • 新しいIMAP実装。 現在、メールはサーバーと同期され、POP3のようにダウンロードされるだけではありません。
  • メールデーモンの置換（MDR）とメールキットの一部を書き直しました。 MDRクラッシュが修正されました。
  • 統合StackAndTile、元々はオークランド大学で開発されました（setdecor SATDecoratorから入手可能）。 このビデオでより明確に 。
  • ウィンドウの移動とサイズ変更の制御のサポートが追加されました（CTRL + CMD +最初のマウスボタンとCTRL + CMD + 2番目のマウスボタン）。
  • 2番目の実験的にサポートされているコンパイラをGCCバージョン4.4.4に更新します（setgcc gcc4から入手可能）。
  • GCC 4での弱い文字と割り込み文字のサポート。
  • BootManager、DiskUsage、MediaPlayer、ReadOnlyBootPrompt、ShowImageのいくつかのプログラムでGUIを改訂しました。
  • 拡張ロケールのサポート。 Haiku APIおよびPOSIXロケールAPIの単一のバックエンド。 ロケールおよび時間プレフレットのインターフェースと安定性が大幅に改善されました。
  • Layout APIはより多くのアプリケーションで使用されており、そのため、フォントの変更やローカライズに対するインターフェースの応答性が向上しています。 さらに、レイアウトAPIは安定してパブリックに近づいています。 ただし、APIはまだ完全ではありません。いつでも変更される可能性があります。 開発者の注意をこれに向けます。
  • 4 GiBを超える物理メモリを使用可能にするPAE（物理アドレス拡張）サポートの実装。


不足している機能



  • Haikuは、ワイヤレスネットワークのWPA暗号化をまだサポートしていません。 WEP暗号化のみがサポートされています。
  • パッケージマネージャは開発中であり、このリリースには含まれていません。 一時的な代替として、少数のソフトウェアのセットをインストールできるスクリプトが含まれています（installoptionalpackageから入手可能）。
  • OHCIまたはUSB 2.0ハブに接続されたUSBキーボードは、カーネルデバッグモード（KDL）では機能しません。 機能するには、UHCIポートに接続する必要があります。 また、パニックが原因でカーネルデバッガに入ると、キーボードが機能しなくなる可能性があることに注意してください。 この問題を回避するには、手動でカーネルデバッグモードに切り替えて終了します。 これらの手順の後、USBキーボードは機能します。




既知の問題



  • LiveCDのパフォーマンスと初回起動時：システムの初期インストール後またはLiveCDモードでの起動後、インストールプロセスを完了するためにいくつかのバックグラウンドタスクが実行されます。 ご存知のように、これはパフォーマンスの低下につながります。 ハードドライブの場合、タスクは迅速に実行されるため、この問題は発生しません。 LiveCDモードでは、CDの検索速度が遅いため、パフォーマンスが著しく低下します。 CDは読み取り専用モードで動作するため、LiveCDを起動するたびにこのインストールが行われます。 記録可能なメディアでは、インストールは1回だけ実行され、次回の起動時に遅延は発生しません。
  • ヒント特許の有効期限が切れているにもかかわらず、フォントのレンダリングは依然として最適ではありません。 特許に関する不確実な状況のため、特許の対象となるコードは公式リリースでは無効になっています。 残念ながら、FreeTypeで使用されるサブピクセルアンチエイリアスはこのコードにも適用されます。 状況が解消されると、サブピクセルアンチエイリアスは公式リリースに戻ります。
  • デフォルトで有効になっているACPIサポートは、一部のハードウェアで問題を引き起こす可能性があります。 ブートローダーのセーフモードオプションでACPIを無効にできます。
  • 高速なドライブから低速なドライブ（USBドライブなど）に大量のデータをコピーすると、スワップファイルが作成される場合があります。
  • 報告によると、Haikuブートローダーは一部のシステムでフリーズする場合があります。
  • ローカリゼーション/国際化サポートの作業はまだ完了していません。 一部のアプリケーションは部分的に翻訳されており、英語以外の言語の長い行に問題があります。
  • 一部のワイヤレスネットワークカードのファームウェアをインストールするには、ユーザーの確認が必要です。 これらのカードは、Broadcom 43xx、Intel ipw2100、Intel ipw2200、Marvell 88W8335です。 問題の一時的な解決策として、スクリプトが使用されます（install-wifi–firmwares.shから入手可能）


ソースコード
