Linuxでdockerを使用してletsencrypt証明書の自動受信を設定する

最近、仮想サーバーを変更し、すべてを新しく構成する必要がありました。 このサイトはhttps経由でアクセスでき、letsencrypt証明書が自動的に取得および更新されます。 これは、nginx-proxyとnginx-proxy-companionの2つのdockerイメージを使用して実現できます。







このガイドは、SSL証明書を自動的に受信するプロキシを使用して、Dockerでサイトを設定する方法です。 CentOS 7仮想サーバーが使用されます。







サーバーは既に購入、構成、キーによるログイン、fail2banのインストールなどが完了していると思います。







まず、Dockerをインストールする必要があります。







  1. まず、依存関係をインストールする必要があります 



    $ sudo yum install -y yum-utils device-mapper-persistent-data lvm2
  2. リポジトリを接続する 



     $ sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
  3. 次に、Community Editionドッカーをインストールします 



     $ sudo yum install docker-ce docker-ce-cli containerd.io
  4. オートロードして実行するドッカーを追加 



     $ sudo systemctl enable docker $ sudo systemctl start docker
  5. sudoなしでdockerを実行できるようにするために、ユーザーをdockerグループに追加します 



     $ usermod -aG docker user


次のステップは、docker-composeをインストールすることです。 このユーティリティはいくつかの方法でインストールできますが、不要なパッケージでシステムを詰まらせないように、pip managerとvirtualenvからインストールすることを好みます。







  1. pipをインストールする 



     $ sudo yum install python-pip
  2. virtualenvをインストールする 



     $ pip install virtualenv
  3. 次に、プロジェクトを含むフォルダーを作成し、初期化する必要があります。 パッケージ管理に必要なものがすべて含まれるフォルダーは、veと呼ばれます。 



     $ mkdir docker $ cd docker $ virtualenv ve
  4. 仮想環境の使用を開始するには、プロジェクトフォルダーで次のコマンドを実行する必要があります。 



     $ source ve/bin/activate
  5. docker-composeをインストールできます。 



     pip install docker-compose


    コンテナがお互いを見るために、ネットワークを作成します。 デフォルトでは、ブリッジドライバーが使用されます。 







     $ docker network create network


    次に、docker-composeを構成する必要があります。プロキシはプロキシフォルダーにあり、テストサイトはテストフォルダーにあります。 たとえば、example.comというドメイン名を使用します 





     $ mkdir proxy $ mkdir test $ touch proxy/docker-compose.yml $ touch test/docker-compose.yml


    コンテンツプロキシ/ docker-compose.yml 













     version: '3' networks: default: external: name: network services: nginx-proxy: container_name: nginx-proxy image: jwilder/nginx-proxy ports: - 80:80 - 443:443 volumes: - certs:/etc/nginx/certs - vhost.d:/etc/nginx/vhost.d - html:/usr/share/nginx/html - /var/run/docker.sock:/tmp/docker.sock:ro nginx-proxy-letsencrypt: container_name: nginx-proxy-letsencrypt image: jrcs/letsencrypt-nginx-proxy-companion volumes: - certs:/etc/nginx/certs - vhost.d:/etc/nginx/vhost.d - html:/usr/share/nginx/html - /var/run/docker.sock:/var/run/docker.sock:ro environment: - NGINX_PROXY_CONTAINER=nginx-proxy volumes: certs: vhost.d: html:


    環境変数NGINX_PROXY_CONTAINERは 、letencryptコンテナがプロキシコンテナを見るために必要です。 /etc/nginx/certs/etc/nginx/vhost.dおよび/ usr / share / nginx / htmlフォルダーは両方のコンテナーで共有する必要があります。 letsencryptコンテナが正しく機能するためには、ポート80および443でアプリケーションが使用可能である必要があります。







    テスト/ docker-compose.ymlの内容 







     version: '3' networks: default: external: name: network services: nginx: container_name: nginx image: nginx:latest environment: - VIRTUAL_HOST=example.com - LETSENCRYPT_HOST=example.com - LETSENCRYPT_EMAIL=admin@example.com


    ここで、プロキシがサーバーへの要求を正しく処理し、正しいドメイン名の証明書を要求するには、環境変数が必要です。







    docker-composeを実行するためだけに残ります 







     $ cd proxy $ docker-compose up -d $ cd ../test $ docker-compose up -d
