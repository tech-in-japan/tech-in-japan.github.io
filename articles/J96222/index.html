VirtualBox 3.2：仮想マシンでMacOS Xを実行できるようになりました。 WindowsおよびLinuxで

突然、virtualboxの新しいバージョンのリリースが静かに過ぎ、仮想マシンでMacOS Xを実行できるようになりました。 元のディスクから。 ハッキントッシュではありません。







ライセンスの問題について



MacOSライセンス契約には2つの制限があります。

  1. 仮想マシンにインストールできるのはMacOS X Serverのみです。 同時に、通常バージョンを物理的にインストールすることもできますが、これはライセンス契約に違反します。
  2. MacOS X Serverがインストールされた仮想マシンは、Appleコンピューターでのみ実行できます。 繰り返しますが、物理的にはどのコンピューターでも実行できます（ただし、Pentium D、AMD、i3など、アップルを使用するコンピューターのリストにプロセッサーが含まれていない場合は、タンバリンを使用する必要があります）が、これもライセンス契約に違反します。




互換性の問題について



現在、サポートは実験的です。 特に、Oracleはゲストの追加をまだ行っていません。仮想マシンのOSにインストールされ、コンピューターの物理ハードウェアを快適に使用できるドライバーのセットです。 Mac OS XにGuest Additionsがインストールされていない場合、解像度の変更に問題があり、グリッドとサウンドはありません（少なくともレビューによると、個人的には私のネットワークは機能します）。 開発者は、すぐにすべてがそうなると言います。



インストール方法



Windows / Linuxでの使用プロセスは、Mac OSほど明確ではありません。 MacOSで実行されていない仮想マシンでMac OS Xを動作させるために必要なこと。 たとえば、Windows 7 64ビットにvirtualboxをインストールします。 あなたがする必要があること：

  1. 公式サイトから最新の仮想ボックスをダウンロードしてインストールします。 バージョンを変更せずにいくつかの修正が行われたため、リリース3.2.4を使用している場合でも、MacOSのバグにとって致命的な場合があります。
  2. virtualboxを起動して、「MacOS」という名前の仮想マシンを作成し、ゲストOSとしてMac OS X Serverを指定します。残りはデフォルトです。
  3. 作成された仮想マシンの設定を開きます。 ドライブ設定で、SATAが存在する場合は、IDE（ICH6）に変更します。 CDドライブの同じ場所で、「パススルー」チェックボックスを設定します-これがないと、Windows仮想マシンはMacOS XインストールCDから起動できなくなります。
