Ubuntu 14.04でCapsLockを使用してレイアウトを切り替える

ubuntu 14.04には問題があります。CapsLockボタンをキーボードの折りたたみスイッチとして割り当てると、レイアウトを切り替えようとすると（すぐではなく）時間が経つと、言語間でルーレットのようにジャンプし、どの言語かが明確になりません。 一般的なバグ。

簡単な解決策を見つけました。

手続き



1. gnome-tweak-toolを配置します。

2. [キーボード]セクションに移動します

3. [CapsLock Key Behavior]リスト項目で、[CapsLock Off]を選択します。 これがポイントであり、「オフ」だけではないことを強調します







4. ubuntuの設定に移動し、CapsLockを静かにボタンとして割り当ててレイアウトを切り替えます。 「CapsLock」としてではなく、0xffとして認識されますが、現在は静かに動作します。



メソッドは数日間テストされており、現在ではレイアウトの切り替えに苦痛はありません。 ノームでは、切り替えはかつて野生の遅延（gnomeの病気）で動作していましたが、ubuntu-vtsは切り替えを行い、ルーレットのように動作し始めました（遅延なし）、今（奇跡！）最後に、このバグ修正の後、すべてが機能します:)
