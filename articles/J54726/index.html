Symantec Backup Exec System Recoveryをインストールする







システム管理者の主なタスクの1つは、バックアップすることです。 私の練習では、このタスクはUnixサーバーでのみ発生し、以前はWindowsマシンで同じアクションを実行する必要はありませんでした。 しかし、今この瞬間が来ました-Windowsサーバーを上げて、それに応じてバックアップを設定する必要がありました。

もちろん、標準的なアーカイブからプロフェッショナルなバックアッププログラムまで、さまざまなバックアップシステムを検討しました。 その結果、このタスクにSymantec Backup Exec System Recoveryを選択しました。 私の選択は、他のシステムに対するこのシステムの利点に実際に基づいているよりも主観的なものです（仕事のためにこのメーカーのプログラムに対処する必要がしばしばありました）。 だから...



ご想像のとおり、インストールはこの製品の「取得」から開始する必要があります。 私はすぐにあなたに警告したいです-プログラムは支払われます（2ヶ月後にそれをアクティベートする必要があります）ので、あなたはそれぞれ2ヶ月間だけその魅力を楽しむことができます。 製造元のWebサイトで、このプログラムのさまざまな変更をダウンロードできます。 Windowsサーバー用のBackup Exec System Recoveryに関心があります。 プログラムを受け取った後、インストールを開始します。 インストール中の制御の瞬間は1つだけです-これはインストール用のコンポーネントの選択です（図）









原則として、これ以上何もしないので、この段階では何​​も変更しないことをお勧めします。 インストールの最後に、初期バックアップウィザードの更新を開始できます。 1つ目を開始してから、2つ目を手動で行います。 インストールが完了したら、サーバーを再起動する必要があります。 これでインストールは完了です。
