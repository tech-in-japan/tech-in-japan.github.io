5分でiOSデバイスの自動接続を使用してLinux Server'eでOpenVPNを構成します



最近では、トラフィックを保護する問題がより緊急になりつつあり、プライバシーの問題を解決するために多くの方法を使用できます。



今日は、自分のLinuxサーバーを使用して、5分でiOSデバイスにVPN On Demandをセットアップする方法について説明します。

VPN On Demand-オンデマンドで接続/切断時に再接続するVPN




投稿を書くというアイデアは、iOSデバイスでVPN On Demand接続を作成する簡単な方法がないという事実から生まれました。

接続を手動で構成する場合は、ユーザーiznakurnozhからの次の指示をお勧めします。



OpenVPNサーバーの構成



これについてはインターネット上の膨大な数のページが書かれており、OpenVPNサーバーには膨大な設定オプションがありますが、この場合は2回のクリックでOpenVPNアクセスサーバーをインストールする方法を見ていきますが、OpenVPNを「自分」-同時接続の最大数は2に制限されています。



そして、 http：//openvpn.net/index.php/access-server/download-openvpn-as-sw.htmlにアクセスしてください

OSを選択し、パッケージをダウンロードしてインストールします。インストール中に、パスワードを設定する必要があるopenvpnという名前の新しいユーザーが作成されます。

Ubuntuでは、コマンドを使用してパッケージのインストールが行われることを思い出させてください

sudo dpkg -i％パッケージ名％



パッケージがインストールされ、openvpnユーザーのパスワードが変更された後（コマンド：sudo paswd openvpn）

次のアドレスを開くことができます。

http s ：//127.0.0.1：943 / admin /

127.0.0.1には、サーバーのIPアドレスをそれぞれ配置する必要があります。



次の制御インターフェイスが表示されます。







接続されたクライアントの表示：







OSIモデルによるトンネリングレベルの選択：







デフォルトでは、ユーザー認証はLinuxシステムで作成されたユーザーを使用して実行されることに注意してください。



デバイスを接続するには、



http s ：//127.0.0.1：943 /

127.0.0.1には、サーバーのIPアドレスをそれぞれ配置する必要があります。







プラットフォームに応じて、パスワードを入力するだけで設定済みのクライアントをダウンロードするように求められます。



iOSの場合、App Storeから公式OpenVPNクライアントをダウンロードする必要があります

次に、 http s ：//127.0.0.1：943 /に移動し 、 （Yourself（User-locked profile））を選択します





ファイルをダウンロードし、OpenVPNアプリケーションで開くように求められます







アプリケーションが開きます。クリックする必要があります。







次に、パスワードを入力して接続します。



アプリケーション自体がVPN設定で接続を作成して管理し、それが行われた主なポイントは、デバイスをロックしてからロックを解除すると、VPN接続が再接続され、3GからEDGE wi-fi損失に切り替えるときにも発生しますなど、ほとんど常に動作します。



この手順の助けを借りて、自動接続機能を備えたOpenVPNサーバーになれるといいのですが。
