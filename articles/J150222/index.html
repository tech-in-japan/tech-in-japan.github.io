率直に言って、私はLinuxにあまり馴染みがありませんが、XMLライブラリを組み込みQNXに移植して以来、基本的な知識が残っています。 ルーターにはminiDLNAがあり 、これはNetGearによってReadyNASという名前で開発されています。 簡単な実験では、ネットワークを介したファイル転送の速度が低下するのは、miniDLNAがビデオ/オーディオ/画像からメタデータを解析するときだけであることが示されました。 これは、次のようにWebインターフェイスでも簡単に確認できます。 プロセッサは100％でロードされます。

また、miniDLNAは解析されたすべてのデータを/.dms/files.dbファイルに保持していることがわかりました。これはSQLiteデータベースにすぎません。 繰り返しになりますが、miniDLNAはファイルの半分未満を解析し、アストラルのままにして、プロセッサを100％まで永久にロードすることが実験的に見つかりました。 データベースビューアーを介して、最後に解析されたファイルを見つけました。その後、プロセスは一時停止し、入力して次のファイルを削除しました。 そして、それはすべてうまくいきました！

これは特定のAVIファイルであることが判明し、明らかにminiDLNAパーサーを一時停止します。 メタデータの解析時間が突然無限に近づいた場合、「ハング」の主題について/.dms/files.dbを調べる必要があることを除いて、具体的なアドバイスはありません。 どのファイルがminiDLNAをハングさせますか？ そして、「PATH」列の最後のエントリの後のファイルシステムの次のものは、「OBJECTS」テーブルです。 さらに、アドレスhttp：//my.router：8200にアクセスすることにより、解析プロセスをWebブラウザーから確認できます。 これにより、次の形式の情報が生成されます。

miniDLNAがフリーズしない場合、解析されるファイルのサイズに応じて、ページが3〜5秒ごとに約1回の速度で更新されると数値が変化し、フリーズする場合は変化しません。

さて、gadファイルは削除され、すべてがなくなったようです。 しかし、ここで4番目のレーキに精通します。 このレーキは呼ばれています-どうしてDLNAにそんなにたくさんのゴミがあるのですか？

ここにあります。 デフォルトでは、miniDLNAのファイル検索が開始されるポイントは/メディアです。 すべてのディスクはAiDisk_xYの形式でそこにマウントされます。xは物理ディスクの番号で、Yは論理ディスクの番号です。 したがって、「メディアファイルソースパス」により、最初にminiDLNAがすべてのディスクを完全に解析します。 あなたが推測するかもしれないように、最終的にすべてがそこに落ち、私はアパート全体で共有したくないでしょう。 たとえば、オーディオブックのカバー、オーディオブック自体、プログラムのアンパックされたグラフィックリソース、ポルノ、酔っぱらいの写真、およびパーサーによってオーディオ/ビデオとして認識されるその他のすべてのゴミ。

このレーキを修正するために、私は個人的に最初のディスクに1つのフォルダーを作成し、デフォルトのパスを、私の場合は/ media / AiDisk_a1 / Mediaに変更しました。 /etc/minidlna.confのminiDLNA構成ファイルを修正するなど、より高度なパスがありますが、正直にいじりたくなかったので、DLNAのすべてをディスクのルートの1つのフォルダーに入れました。 次の2つの質問があります。

そして、異なるドライブに複数のフォルダーがある場合はどうなりますか？ 次に、minidlna.confへの直接の道があります。

なぜ/ AiDisk_a1？ 最初のUSBポートに接続された最初のディスクは常にその名前とインデックスを正確に取得するためです。 たとえば、同じ物理ディスク上の2番目のパーティションは、識別子/ AiDisk_a5を取得します。 なぜこれが起こっているのかはわかりませんが、これを説明できる知性的な人がもっといるかもしれません。 繰り返しますが、2番目のUSBポートの最初のドライブは識別子/ AiDisk_b1を受け取るため、オプションが可能です。

中間結論：


-UPnP Media Serverをオンにしたときにファイルデータ転送速度が急激に低下した場合、ルータープロセッサは常に100％ロードされ、メディア解析が行われます（ http：//my.router：8200の進行状況を参照）。これによりminiDLNAがハングアップします。 UPnPサーバーをオフにし、パーサーがハングしたファイルを破棄して、サーバーを再起動する必要があります。

-デフォルトでは、miniDLNAは使用可能なすべてのディスクとパーティション上のすべてのファイルを解析します。 これはかなり長いプロセスです。 したがって、1つのフォルダーに制限する場合は、UPnPメディアサーバー設定の「メディアファイルのソースパス」を変更します。
