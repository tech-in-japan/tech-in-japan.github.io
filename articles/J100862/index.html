コンピューターの生産、コンピュータークラス、インターネットカフェでシステムを展開します

コンピュータの大規模なバッチの準備には、標準的なソフトウェアセットを提供する必要があります。 同時に、個々の要求に応じて、一般的なリストにいくつかの変更が加えられる場合があります。



明らかに、各PCでの手動のイメージ復元は、退屈で日常的な作業であるだけでなく、リソースの非効率的な使用でもあります。 このプロセスを可能な限り簡素化および自動化するために、当社の専門家がParagon Deployment Manager 10（DM10）を準備しました。



そのため、DM10の一部であるハードディスクマネージャーを使用して、「参照」コンピューターのイメージを作成し、インフラストラクチャサーバーに配置します。









原則として、多くのコンピューターに展開するイメージは、DM10に含まれるParagon Adaptive Imaging Toolsを使用して、または同様の機能を備えたMicrosoft SysPrepを介して準備できます。



次に、セッション識別子を使用してスクリプトを作成し、システムイメージを割り当てます。 セッション識別子を使用して、起動時に使用するシナリオと展開するイメージを決定できます。



DM10の一部でもあるBoot Media Builderを使用して、ネットワークを介したコンピューターの初期ブートに使用されるPXEイメージを準備します。 基礎として、DOS、Linux、およびWinPEベースのシステムを使用できます。 主な機能であるシステムイメージの読み込みと展開に加えて、追加の作業シナリオをもたらすことができることに留意してください。



PXEイメージをPXEサーバーに接続した後、コンピューターを起動できます。コンピューターは作業環境でPXEイメージのロードを開始し、後でシステムイメージをロードします。



同時に、さまざまなスクリプトを追加して、ディスクスペースを構成し、PSLスクリプトを使用してオペレーティングシステムを構成し、SysPrep構成を取得し、ファイルをダウンロードし、サードパーティプログラムを起動することができます。



SMBと連携しています



現代のどの企業にも、維持およびサポートする必要がある特定のコンピューター群があります。 各部門が異なるソフトウェアのセットを使用できることは明らかです。 さらに、コンピューターパークのように、構成は時間の経過とともに変化する可能性があります。



長年の問題は、さまざまなコンピューターでの典型的なソフトウェア構成の集中ストレージと展開です。 間違いなく、これらのアクションは手動で実行できますが、コンピューターパークの拡大またはその更新のコンテキストでは、作業コストが大幅に増加します。 これらの操作のコストを削減できるツールは、ビジネスパフォーマンスに大きく影響します。



DM10の場合、システムをアップグレードまたは展開するために職場を離れる必要さえありません。すべての操作はリモートで実行できます。 必要なシステムイメージと識別子を使用してタスクを割り当て、クライアントを再起動し、PXEイメージがロードされるのを待って、システム展開プロセスを観察する必要があります。



さらに、緊急時に迅速に展開するために必要なすべての構成のイメージを手元に用意します。 また、DM10の一部としてAdaptive Imaging Toolsを使用すると、コンピューターの特定のハードウェアを忘れることができます。



コンピュータークラスとインターネットカフェ



コンピュータクラスとインターネットカフェの主な問題の1つは、彼らの仕事の特徴、つまり、コンピュータ上のデータやプログラムの安全性をあまり気にしない訪問者の大規模な流れにあります。 多くの場合、不要なプログラムがシステムに表示された後、マルウェア、システムの元の構成が変更または劣化します。



この問題の1つの解決策は、事前に準備された「参照」イメージから構成を展開することです。 しかし、ご理解のとおり、これらすべてを手動で実行するには時間がかかる場合があります。



DM10を使用すると、短時間で処理できるようになったため、このプロセスを自動化できます（たとえば、学校の変更など）。



管理者は、必要なシステムイメージ、PXEイメージ、またはブートディスクのセットのみを管理する必要があります。 展開は、コンピューターの再起動後に自動的に開始するか、手動で開始できます。



Paragon Deployment Manager 10の一般的な機能



サーバー側で：



•ハードディスクマネージャー-データのバックアップコピーの作成、アーカイブ、ハードドライブ上のパーティションの管理に使用します。

•PXEサーバーは、ネットワークを介したクライアントマシンの読み込みを整理します。 DHCPおよびTFTPサーバーサービスが含まれます。

•インフラストラクチャサーバーは、クライアントマシンおよびDM10インフラストラクチャでのソフトウェア環境の展開を管理します。

•展開コンソールは、データ展開プロセスを構成および手動で管理するために使用されます。

•Boot Media Builderは、PXEサーバーのブータブルメディアとブータブルイメージを作成するように設計されています。

•アダプティブイメージングツールを使用して、デバイスに依存しないシステムイメージを作成し、さまざまなハードウェア構成のマシンにさらに展開します。



クライアント側で：



•マルチキャスト/ ConstantCast。コンピューターをデータの展開およびサーバーへの接続のターゲットとして識別します。

•ハードディスクマネージャー-ハードドライブ上のパーティションの管理、バックアップの作成と復元、データのコピー。

•単純な復元ウィザード-バックアップからデータを復元します。

•ログセーバー-操作ログを保存します。
