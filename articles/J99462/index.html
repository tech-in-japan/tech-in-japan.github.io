Rackspace社は、独自の「クラウド」を作成するためのオープンソースプロジェクトを提供しています





さまざまな企業が提供する「クラウド」プロジェクトが増えています。 そのため、約1か月前、Kaspersky LabはDDOS攻撃に対抗する独自のスケーラブルなソリューションを提案し、Amazonは2番目の「クラウド」プログラムを立ち上げ、独自のサービスを購入できないあまり裕福でない企業に強力なクラウドを提供しました。 Habréに好意的に反応していると思われるRackspace社は、特定の要件を満たす個別のクラウドを作成したい人向けに、独自の施設に基づいた独自のオープンソースプロジェクトを提供しています。



このプロジェクトはOpenStackと呼ばれ、Apacheからライセンスされたプロジェクトであり、いくつかのクラウド機能が含まれています。 Rackspace Cloud Fileプラットフォームに基づいた、大規模なデータアレイの分散ストレージもあります。



しばらくすると、NASA NebulaクラウドとRackspace Cloud Serversプラットフォームに基づくもう1つのサービスが稼働することに注意してください。 これらはすべて、来年初めに稼働します。



OpenStackは主に、ホスティング会社に特別な要件がある企業ユーザーを対象としています。 前述のように、OpenStackでは、Rackspaceオープンソースソフトウェアを使用して独自のクラウドを作成および構成できます。これは、会社のニーズに応じて機能します。



リーダーシップによると、このようなプロジェクトを作成する主な理由は、企業ユーザーの要件が非常に異なることです。各要件には、「クラウド」と「クラウド」の両方ではないほとんどのホスティング事業者とデータセンターが提供する標準パッケージの一部ではない特別なものが必要です。



ソフトウェアはランダムなものではなく、作成されたばかりのものではなく、数年にわたって同社の専門家によって開発されたソフトウェアであり、既に実行されています。 したがって、普通の企業がますます強力なホスティングパッケージを提供しようとすると、企業ユーザーおよび個人ユーザーは、「クラウド」オファーの実験にさらに興味を持ちます。 実際、使用した容量に対してのみ料金を支払うことを知ってうれしいです。また、「クラウド」により必要な容量を柔軟に増やすことができるため、独自のプロジェクトは完全に機能します。



興味深いことに、ユーザー自身が作成および構成した個々の「クラウド」を提供することが主な目的であるこのようなプロジェクトにもかかわらず、同社はプロジェクトを標準にしようとしています。
