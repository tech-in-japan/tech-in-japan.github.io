13の有望なプログラミング言語





バグの少ない、より高速でスマートなプログラミング。 このような約束により、多くの現代言語の作成者はプログラマーの注目を集めています。 すでに何度も聞いていますが、目新しさの欠如はこれらの約束を拒否する理由ではありません。 結局のところ、プログラミングの未来には安定性と優れた技術の導入が必要なので、イノベーションが機能します。 また、ソフトウェア製品は非常に大きいことが多いため、今日ではかつてないほどのイノベーションが必要です。







以下にリストされている言語の場合、自動化の度合いを高めると、コードが「より速く、よりスマートに、バグなし」の特性に値するようになることが特徴です。 現代のアプローチは、構造性と抽象性の強化を暗示しています。そのため、言語の「充填」は、プログラマーが以前に独立して解決しなければならなかったタスクの履行をますます前提としています。 このような自動化の成長により、開発者はバグの発見と修正により注意を払うことができます。 多くの場合、自動化されたアルゴリズムは、多くの単純なミスをすることなく、計算を並列化するタスクをより効率的に解決するため、これにより言語のパフォーマンスが向上します。



しかし、このメイントピックの背後には、小さな合意が1つあります。 言語の1つは、統計分析用に作成されました。 他のいくつかは、古典的な言語をアップグレードしています。 一部はまったく言語ではなく、単なるプリプロセッサです。 しかし同時に、それらはすべて、コードを書くための私たちのアプローチを変え、将来のプログラミングの基礎を築きます。



以下は、コンピュータに何をすべきかを伝える方法を変える13の言語です 。 それらのいくつかは新しく、いくつかはすでに広く普及しており、いくつかは完全な意味での言語ではありません。



R



本質的に、 Rはプログラミング言語ですが、統計を使用して大量のデータのパターンを検出することに取りつかれている現代世界の標準的な担い手の役割を果たしています。 Rは、統計学者と科学者が研究を促進するために開発しました。 データ分析に使用されるほとんどの標準機能を実装し、自由に分散されたライブラリを使用して、多くの有用なアルゴリズムを追加できます。 Rは、データ処理および分析の専門家に、研究に必要なほぼすべてを提供します。



多くは、統合実験環境内でRをデータ実験のメモ帳として使用することに限定しています。 R StudioとR Commanderの 2つの一般的なフロントエンドを使用して、データをアップロードして操作できます。 Rを言語から変換して、よりインタラクティブな環境にコンパイルして実行します。



キーポイント：サブセットを選択して分析するためのよく考えられた表現。

難しさ：パーソナルコンピューターの下で研ぎ澄まされたものは、Hadoopなどのテクノロジーが支配するビッグデータの世界の一部ではありません。



Java 8



Javaは新しい言語ではありません。 私たちの多くにとって、コンピューターサイエンスで共通語としての役割を果たした最初のプログラミング言語でした。 今日、数十億のJARファイルが世界中に流通し、制御されています。 ただし、 Java 8は少し異なります。 並列化への道を開く関数型プログラミング手法を導入する機会をもたらしました。 それらを使用する必要はありません。古き良きJavaのフレームワーク内にとどまることができます。 ただし、JVM実行を最適化するために構造を改善することはできません。 関数型プログラミングパラダイムのフレームワーク内で考えることはできず、より少ないバグでよりクリーンで高速なコードを書くことはできません。



キーポイント：ラムダ式と並行コード。

難しさ：逃げ出し、Scalaに真っ向から飛び込みたいという欲求があります。

ミックスインやネストされたルールなど、より複雑な構造があります。これにより、標準レイアウトコマンドのブロックを効果的に作成し、任意の数のCSSクラスに含めることができます。 太字フォントを破棄する必要がある場合は、ルートでこの設定を変更するだけで、Less.jsは新しいルールを他のすべての定義に拡張します。
