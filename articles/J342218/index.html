小さなPython Webバックエンドの移植によるGoの学習

内容：Goを学習するために、PythonからGoに小さなサイトにバックエンドを移植し、その過程で楽で痛みのない経験を得ました。

しばらくの間Goの学習を開始したかったのですが、その哲学が気に入っていました。小さな言語、すてきな学習曲線、非常に高速なコンパイル（静的に型付けされた言語など）です。 最終的にさらに一歩進んで彼に教え始めたのは、特に最近使用したものから、Go- Dockerとngrokで書かれたより速くて深刻なプログラムが増えていることです。







Goの哲学は、すべての人の好みではありません（例外はありません。独自のジェネリックなどを作成することはできません）が、私の精神モデルにうまく適合しています。 シンプルで高速、明白な方法で物事を行う。 ポーティング中、標準ライブラリとツールがどれだけ充実しているかに特に感銘を受けました。







移植



Goで20行のスクリプトをいくつか作成しましたが、言語とエコシステムを理解するには十分ではありませんでした。 そのため、より大きなプロジェクトを採用することにし、サイトGiftyWeddings.comのバックエンドを移植することにしました 。







Pythonでは、Flask、WTForms、Peewee、SQLite、およびS3、画像のサイズ変更などのための他のいくつかのライブラリを使用して、約1300行のコードでした。







Goバージョンでは、言語をよりよくマスターし、標準ライブラリで可能な限り機能するために、できるだけ少ない外部依存関係を使用したかったのです。 特に、GoにはHTTPを操作するための優れたライブラリがあり、今のところWebフレームワークをまったく検討しないことにしました。 しかし、私はまだS3、Stripe、SQLiteのいくつかのサードパーティライブラリを使用し、パスワードを操作し、写真のサイズを変更しました。







Goの静的型付けと、使用するライブラリの数が少ないため、コードは元の2倍以上になると予想しました。 しかし、結果は1900行でした（Pythonの1300行の約50％）。







移植プロセス自体は非常にスムーズに進み、ほとんどのビジネスロジックは、PythonからGoにコードを行から行に変換することでほぼ機械的に移植されました。 Pythonの概念がthings[:20]



スライス構文に至るまで、Goにどれだけうまく翻訳されているかに驚きました。







また、Flaskが使用する危険なライブラリの一部を移植したため、Goバージョンへの移行中にPythonサービスからセッションCookieを透過的にデコードできました。 暗号化、圧縮、シリアル化のすべてのコードは標準ライブラリにあり、簡単なプロセスでした。







一般に、 Go Tour 、 Effective Go、およびインターネット上のさまざまなコード例を見る間、言語を学習するプロセスは苦労しませんでした。 ドキュメントはかなり短いですが、非常によく書かれています。







言語ツールも非常に満足しています。コードの構築、実行、フォーマット、およびテストに必要なものはすべて、 go



サブコマンドを介して利用できます。 開発中に、 go run *.go



を使用してオンザフライでコンパイルし、サーバーを起動しました。 編集と起動には約1秒かかりました。これは、Scalaでの20秒の増分と20分の完全な編集からの剣の戦いの後の新鮮な空気でした。







テスト中



Go標準ライブラリには、基本的なテストパッケージと、 go test



を検索、コンパイル、実行するすぐに使用できるテストランナー（ go test



）が含まれています。 このパッケージは非常に簡単でシンプルですが（多すぎる場合もあります）、必要に応じてヘルパーを追加するだけです。







単体テストに加えて、実際のGifty WeddingsサーバーでセットアップされたHTTPテストを実行するテストスクリプトを作成しました（ testing



パッケージも使用）。 古いPythonサーバーでこのテストを設定し、結果が同一であることを確認できるようにするために、コードのGoレベルではなく、HTTPレベルでこれを行いました。 これにより、サーバーを変更する前にすべてが正常に機能したことを十分に確信できました。







また、ホワイトボックステストも行いました。スクリプトは応答をチェックしますが、Cookieをデコードし、Cookieに正しいデータが含まれていることもチェックします。

これにより、すぐに使用できるLinuxバイナリがMacでコンパイルされます。 そしてもちろん、これを逆方向に行うこともでき、Windowsとの間でクロスコンパイルもできます。 それだけで動作します。

パイニストからの主張



Pythonの後、コードは実際にはいくつかの場所でもう少し冗長に見えます。 既に書いたように、ほとんどのコードは文字列をPythonの文字列に変換しますが、それは非常に自然なことです。







しかし、リストや辞書を含めるのを本当に忘れていました。 これを有効にできたら素晴らしいと思います。 







 gifts := []map[string]interface{}{} for _, g := range registryGifts { gifts = append(gifts, g.Map()) }


これに： 







 gifts := [g.Map() for _, g := range registryGifts]


実際、そのような場所は私が予想したよりもはるかに少なかった。







同様に、 sort.Interfaceは 冗長すぎます。 sort.Slice（）を追加するのが正しいステップでした。 しかし、スライスインデックスにまったく触れることなく、Pythonのキーで並べ替えるのがどれほど簡単かは今でも気に入っています。 たとえば、文字列のリストを大文字と小文字を区別せずにソートするには、Pythonでは次のようになります。 







 strs.sort(key=str.lower)


およびGoの場合： 







 sort.Slice(strs, func(i, j int) bool { return strings.ToLower(strs[i]) < strings.ToLower(strs[j]) })


そして、原則として、私が欠けていたのはそれだけでした。 私は例外を見逃すと思っていましたが、それは違います。 そして、一般的な信念に反して、ジェネリック医薬品の欠如は私をいつも一度だけ悩ませました。
