OpenALを例として使用したAndroid NDKの基本

こんにちは、Habrausers様！



最近、私はAndroid用のアプリケーション、特にゲームの開発を行っています。 そのため、あるプロジェクトではAndroid ndkを使用する必要がありました。 原則として、ネイティブでの作業のすべての困難とニュアンスを1つの記事で検討することは不可能であるため、この記事ではndkの簡単な紹介を書くことにしました。

そして、この記事を初心者だけでなく興味深いものにするために、OpenALとWAV、OGG形式での作業方法を示します。







はじめに



私にとっては、環境のセットアップについて多くを書く価値はありません。 開発している環境（Eclipse、IntelliJ IDEAなど）に関係なく、セットアップは非常に簡単です。

  1. Android NDK自体 。
  2. WInでビルドするには、 Cygwinが必要です。
  3. 同じEclipseのプラグイン： CDT 。


当然、すでにADT、JDKが必要です。



なぜNDKが必要なのですか？


  • OpenGL ESを使用します。 NDKを使用している人のほとんどは、ゲームを書くためだけにNDKを使用していると思います。
  • Cocos2Dxなどのクロスプラットフォームゲームエンジンを使用する
  • 最も明らかなケースは、C ++ですでに記述されたコードを使用する必要がある場合です。 何十年もの間、すべてのものの多くがC ++で書かれてきました。 はい、すべてを書き換えることはできません。同じopenCVを書き換えても意味がありませんが、既製のソースを接続するだけでかまいません。




JavaからC ++コードを呼び出す



一般的に、すべてが非常に簡単で、主な手順は次のとおりです。

  1. C ++でファイルを作成し、エクスポート用のメソッドを定義します。
  2. .mkファイルの作成。
  3. ライブラリ生成。
  4. Javaでライブラリを接続します。






Makefile（.mk）についてペイントしません。 ここで彼について読むことができます 。 その上、habrにはBubaVVの.mkファイルの作業に関する良い記事があります 。



ライブラリについては、ndkからここで読むことができます 。



C ++ファイルの作成


Javaから呼び出すエクスポートのメソッドを定義する必要があります。 例として、アプリケーションの起動時に、OpenALに音楽をロードします。 これを行うには、メソッドを定義します。 

JNIEXPORT void JNICALL Java_ru_suvitruf_androidndk_tutorial4_MainActivity_loadAudio(JNIEnv *pEnv, jobject pThis, jobject pNativeCallListener, jobject assetManager);




これらはすべてペンで記述しますが、 javahの自動生成には便利なユーティリティがあります。



次に、それを実装する必要がありますが、それについては後で詳しく説明します。

名前について少し
メソッドの名前について少し述べる価値があります。 Java_は必須のプレフィックスです。 ru_suvitruf_androidndk_tutorial4。ru.suvitruf.androidndk.tutorial4パッケージがあるため、クラスとメソッドの名前はJava側にあります。 各関数には引数としてJNIEnv *があります。これは、Javaを操作するためのインターフェースであり、Javaメソッドを呼び出してJavaオブジェクトを作成できます。 2番目の必須パラメーターは、メソッドが静的かどうかに応じて、 jobjectまたはjclassです。 メソッドが静的な場合、引数はjclass型（メソッドが宣言されているオブジェクトのクラスへの参照）であり、静的でない場合-jobject-メソッドが呼び出されたオブジェクトへの参照です。





Javaのライブラリ接続


ライブラリを生成したら、Javaに接続する必要があります。 

 static { System.loadLibrary("AndroidNDK"); }




そして、C ++コードと同じ名前のメソッドを定義します。 

 //  native public void loadAudio(NativeCalls nativeCallListener, AssetManager mng);




このような呼び出し： 

 loadAudio(activity, activity.getResources().getAssets());




C ++からのJava呼び出し



もう少し複雑ですが、それほど怖くない。 必要なもの：

  1. 呼び出すクラスメソッド（Java）を定義します。
  2. 目的のクラスの記述子を取得します（C ++で）。
  3. メソッドのシグネチャを説明します。
  4. メソッド識別子（リンク）を取得します。
  5. 目的のオブジェクトでメソッドを呼び出します。




もちろん、クラスのメソッドを簡単に定義できますが、インターフェイスを使用することをお勧めします。 その後、別のクラスで作業する場合、ネイティブコードを変更する必要はありません。



例として、たった1つのメソッドでインターフェースを作成します。 

 public interface NativeCalls { public void sendLog(String result); }

WAVについて言わなければならないことがあります。 時々、PC上のファイルは正常にリッスンしているように見えますが、OpenALで作業しているときにエラーが発生します。 これは、破損したヘッダーの結果です。 通常はdataSizeで 、ヘッダー（例としてロゴ）に何らかの種類のゴミを書き込む多くのコンバーターに会いました。 なぜ機能しないのに、PCで再生するのですか？

オーディオデータ自体は、ヘッダーとdataSizeのサイズの後に保存されます 。 このフィールドに問題がある場合、エラーが発生します。 あなたは本当に額のサイズを計算することができます。 データサイズ=ファイルサイズ-ヘッダーサイズ 。 したがって、プレイヤーはヘッダーからではなく、減算によってデータのサイズを取得すると思います。



WAVを使用する場合、形式は圧縮されていないため、すべてが単純に思えます。 .Oggを使用する場合、事態はより複雑になります。
