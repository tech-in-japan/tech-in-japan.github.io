開発者向けに内部SMS24X7 APIを開きます

サービスの「内部キッチン」へのアクセスを許可することは、良い傾向になりつつあります。Google、Yandex、Mailchimp、および他の多くは、APIを開いて、サービスの「内部」機能でも動作します。 SMS24X7.RUのリリースから 1か月後に、サービスの完全なAPIを提供できることを誇りに思います。 当社は常にSMSメーリング市場のプレーヤーよりも少し先を行くことを試みており、したがってSMSゲートウェイと、設定、アドレス帳、SMSメーリングを操作するためのすべての機能を含むAPIを最初に作成しました。 一言で言えば-絶対にすべてで！



API をSMSメールのWEBサービスのバックエンドとして導入するというアーキテクチャ上の決定により、少し頭痛がしました。 しかし、Webインターフェイスの開発プロセスを自然に分離することができました。 したがって、サービス内にあるものはすべて、APIを介して利用できることが判明しました。



SMSゲートウェイが必要な理由について誰も話す必要がないとしましょう。 これらのSMSリマインダー、SMS通知、SMS通知はすべて、基本的に携帯電話のほとんどの部分で呼び出し音を鳴らし、けいれんさせると非常に疲れます。 しかし、モバイル番号収集フォームをWebサイトに配置したり、SMSメッセージを個人アカウントで顧客に統合したり、SMSゲートウェイの使用に関する統計を表示したりする必要がある場合はどうでしょうか。 答えは簡単です-APIを使用してください。







APIを開発するにあたり、もちろんHabréにいる開発者の明るい頭が、彼にふさわしいアプリケーションを見つけることを望んでいました！



プロモーションコード「API」の開発者向けの素晴らしい提供


APIサービスの使用を促進するために、次の特性を持つ新しい料金表「API-ONLY」を導入しました。

  • 価格： 30コペック。 任意のボリュームのSMS用
  • 支払条件：前払いのみ
  • 制限：電子メールのみによるテクニカルサポート、Webインターフェースを介してメーリングを作成することはできません、完全な統計のみが利用可能、請求
  • 制限の理由：サポートのトランザクションコスト+ APIを使用すると、任意のSMSアプリケーションを作成できます。アドレスデータベースのメーリングリストの作成は、その使用の基本的な例にすぎません。


登録時に「API」プロモーションコードを示すことを忘れないでください 。15個のテストSMSを受け取り、APIのみの料金を自動的に接続します。



ドキュメントの作成について少し。


Doxygenはよく知られていますが、ラテックスと併用すると非常に遅いです。 したがって、ドキュメントの生成をいじらなければなりませんでした。 book.clsを編集して空白ページを削除するには、latexのブックスタイルを変更する必要がありました。 メインページと目次の後のペア\ cleardoublepageと、章のスタイル\ if @ openright \ cleardoublepage \ else \ clearpage \ fiについてコメントします。 latex自体のDoxygenでは、インデックスとクラス階層を削除することはできませんが、嬉しいことにrefman.texドキュメンテーションテンプレートを生成し、そこに章をさらに挿入します。 したがって、refman.texを使用して必要なすべてを実行するスクリプトを簡単に記述し、ドキュメントをWebサイトにアップロードします。 そして、すべての努力の後、ドキュメントは読みにくく理解しやすい外観になります。
