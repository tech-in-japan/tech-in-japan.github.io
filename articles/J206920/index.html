組み込みモジュールTibboに基づくMKのTCP / IPスタックの構成

このレビュー記事では、Tibbo Technologyのプラグインに基づいたプロジェクトのTCP / IPスタックの構成に焦点を当てます。











ほとんどの場合、開発者はシリアルポートを介してMKを操作します。 これは必ずしも便利ではありません。 多くの場合、TCP / IPスタックを介してリモートでデバイスと「通信」するタスクがあります。 問題は初めて提起されていません。 たとえば、 ここではこのタスクを表明し、かなり詳細な説明で解決します。 リンクには、写真とTibbo EM100モジュールの簡単な説明も含まれています。 実際、当社はロシア連邦のTibbo Technologyの公式代表者です。 したがって、製品の完全な機能と機能を明らかにせずに、私たちの製品がさりげなく言及されたことは少し残念です。 このEM100という名前のオレンジ色の箱は、10年以上生産されており、広大な地域の多くのオブジェクトで使用されています。 しかし、このモジュールのリリース以来、多くの水が流れています。 製品の機能を同時に考慮しながら、Tibboモジュールに基づいてデバイスのTCP / IPスタックを作成してみましょう。



TCP / IPスタックを作成します。



この問題を解決するには、次のものが必要です。

  • 組み込みモジュールTibbo
  • IP経由の無料のオープンソースファームウェアSerial
  • 無料のデバイスエクスプローラーユーティリティ（TIDEに含まれています）
  • 無料のTibbo Device Server ToolKit（TDST）ユーティリティスイート


組み込みモジュール（EM）は、独自のマイクロプロセッサ、クイックスタートに最低限必要なハーネス、およびさまざまな組み込みプロトコルを備えた小型のインテリジェントモジュールです。 通常、EMは独自のオペレーティングシステムを実行しますが、これは人との通信を目的としていません。 EMを使用する目的は、できるだけ早く、最小限のコストで、この機能をデバイス内で整理することです。



Tibbo Technology組み込みモジュール製品ファミリは、10/100 BaseTインターフェイスの物理層を既に備え、TCP / IPプロトコルスタックとの本格的な動作を備え、CMOSレベルで1〜4個のシリアルUARTポートを備えた非常に強力なモジュールです（TTL互換） 。 統合されたイーサネットコネクタの有無にかかわらず、変更が販売されています。 後者を接続するには、外部コネクタとトランス、またはトランスが内蔵されたコネクタが必要です。 モジュールのサイズは異なりますが、一般に、RJ45コネクタを超えることはありません（またはわずかに超えます）。



モデルEM1206 + RJ203を選択しました。簡単な特性です。

  • 高性能88MHz ASIC（T1000）コントローラーに基づいています。
  • 変圧器絶縁が統合された100BaseT（RJ203を使用する場合）。
  • ファームウェア、アプリケーション、およびデータ用に最大1024KBのフラッシュメモリ。
  • データストレージ用の2​​KB EEPROM。
  • リアルタイムクロック（RTC）..
  • 最大17本の汎用I / Oライン（8本の割り込みラインを含む）。
  • 2つの外部ステータスLEDの制御線。
  • 4つの内蔵LED：
  • 緑と赤のステータスLED。
  • 緑と黄色のイーサネットステータスLED。
  • ソフトウェア制御のエンベデッドPLL。
  • 信頼性の高い電源リセット回路。
  • 電源：230mA @ 3.3V（100BaseTモード、PLLオン）。
  • 寸法：33.2x18.1x5.5mm。


モジュールを使用して完全かつ高品質の作業を行うには、ボード上に安定化電源（3.3V）の供給、リセットチップ（できれば）、イーサネットソケットが実装されていない場合は対応するコネクタを取り外す必要があります。



シリアルポートは、デバイスのコントローラーにインストールする必要があります。 上で書いたように、CM1レベル（TTL互換）はEM1206モジュールに実装されているため、Rx / TxラインをMKのレッグに直接接続できます。







仕事に便利なソフトウェア



ソフトウェアの部分はまだシンプルです。 （UDPブロードキャスト要求を送信することにより）ネットワーク上のモジュールを自動的に検出するデバイスエクスプローラーを起動し、ファームウェアを入力します。 このプロセスには1分しかかかりません。





DSマネージャーを起動します。これにより、シリアルインターフェイスとホストパラメーターの両方のパラメーターを設定できます。







Serial Over IPファームウェアの簡単な特徴：

  • 全二重および半二重シリアルポートモード。
  • 利用可能なモード：サーバー、クライアント、サーバー/クライアント。
  • シリアルおよびイーサネット接続用のその他の多くのオプション。
  • 各受信/送信バッファに8KB（各方向に1つのバッファ）。
  • 設定はEEPROMに保存されます。
  • LEDによる詳細なステータス表示。
  • RS232またはイーサネット経由の構成（UDPコマンド、Telnet）。
  • すべてのI / O回線（RTS、CTS、DTR、DSRを含む）のリモート制御。
  • シリアルポートを構成するためのオンザフライコマンド。
  • ネットワーク接続を制御するためのRS232を介した「モデム」コマンド。
  • ADSLモデムの直接制御。
  • UDP、TCP、ARP、ICMP（ping）、DHCP、PPPoE、LCPプロトコルをサポートします。
  • 組み込みのWeb管理インターフェイス。




これで、デバイスはデータストリームをイーサネットネットワークに透過的に転送し、その逆も可能です。 リモートマシンに、仮想COMポートドライバーをインストールできます。



オプションで、外部GA1000拡張モジュールを使用すると、TibboデバイスではWiFiワイヤレスプロトコル（802.11 b / g）を使用してデータを転送できます。 GA1000はコントローラに簡単に接続します; DS Managerユーティリティを使用して、アクセスポイントに接続するための設定を構成できます。







ローカルネットワークの標準動作モードに加えて、Tibbo組み込みモジュールには、PPPoEテクノロジを使用してプロバイダーに自動的に接続できるオブジェクトが含まれています。これにより、デバイスをクライアントとして使用して世界中のリモートサーバーに接続できます。

組み込みモジュールTibboは自由にプログラム可能ですが、アプリケーションのデバッグとダウンロードのプロセスはイーサネットネットワーク経由で実行されます。 プログラミングは、オブジェクト指向のイベント言語Tibbo BasicおよびTibbo Cで実行されます。コーディングには、製造元Tibbo IDEの開発環境が使用されます。 デバイスはTiOSオペレーティングシステムを実行します。これは、ユーザーが作成し、解釈モードでニーモニックコードにコンパイルされたアプリケーションを処理します。 モジュールのボードには汎用入出力ライン（GPIO）が搭載されているため、組み込みモジュールをマイクロコントローラーとして使用できます。



ほとんどのモデルには、ユーザーがWebベースの管理インターフェイスをプログラムできるWebサーバーがあります。 一部のモデルにはRTCリアルタイムクロックがあります。

したがって、モジュールの機能は開発者のみが決定します。 SoIファームウェアでは不十分な場合は、ソースを開き、必要なものを追加します。 または、ゼロから独自のプロジェクトを作成します。 たとえば、コントローラーのプロトコルを実装し、OPCサーバープロジェクト（Tibboから）に接続することにより、デバイスをSCADAシステムに接続し、AggreGate Agentソフトウェアライブラリを使用して、デバイスをAggreGateサーバーに接続できます。

プロジェクトで即座に反応する必要がない場合は、Tibbo EMをデバイスのメインマイクロコントローラーとして使用できます。 モジュールは、最も便利な作業のために作成されました。 高レベル言語のTibbo BasicおよびTibbo Cでの無料プログラミングは、ファームウェアの開発時間を数倍短縮します。 例として、ロックを制御したり、外部アラーム、Webサーバー、シリアルポート、WiFiワイヤレス接続、キーボード、ディスプレイを接続するためのリレーを備えた独自のアクセスコントローラーを作成することを検討できます（概略図はこちらからダウンロードできます ）。
