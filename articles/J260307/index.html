Raspberry Pi 2上のPlexメディアサーバー

これはDebianとRaspberry pi 2を使った初めての経験です。RaspberryPi 2 b +のリリースに関するニュースを見てすぐにgoogleに行ったときにすべてが始まりました。Plexメディアサーバー（以降PMS）をインストールできますか？ これができるという事実の最初の言及で、それは決定されました：私は買っています。 事実、私は長い間、たくさんのpc + PMS + apple tv + plexconnectorを使ってきました。 私にとって、すべてが非常に便利で優れていますが、...ラズベリーでは、それははるかに美味しそうでした。 常時オンのコンピューターの24時間年中無休のノイズ、高電力消費を取り除きます。



各コマンドをグーグルで試行錯誤して、Debianを初めて学ぶのに数日かかりました。 私はロシア語で指示を見つけられなかったので、私と同じユーザーを助けるために自分でそれを書くことにしました。 重要なのは、ラズベリーのプレックスはすでにトランスコーディング、歓声、同志と連携していることです！



それでは始めましょう。



16GB以上の32GBのmicroSD、 win32DiskImager 、および公式サイトの Raspbianイメージが必要です。







すべてをダウンロードし、Raspbianを解凍し、win32DiskImagerを使用してmicroSDにアップロードします。







数分後、画像が記録され、microSDをRaspberry piに挿入し、モニター、マウス、キーボード、LANに接続して電源を入れます。



raspi-configの前に（テレビのfotkalの品質については申し訳ありません）：







いくつかの設定を公開します。



1. FSDを展開して、microSDスペース全体を使用します。

2. Changユーザーパスワードは、私たちにとって便利なパスワードを設定します（デフォルトではログイン：piパスワード：raspberry）。

3. [詳細オプション]で、リモート管理のSSHを有効にします。

4.終了して再起動します。







login：piとパスワードを入力します。



次： 



sudo apt-get update
      
      

        
        
        
      

    
        
        
        
      
      

        
        
        
      

    
     


 sudo apt get upgrade






PMSをダウンロードします。 



 sudo wget https://dev2day.de/plex-latest


プレックス用の追加パッケージをダウンロードします。 



 sudo dpkg -i plex*


プレックスのトランスコーディングパック： 



 sudo apt-get install mkvtoolnix libexpat1 ffmpeg -y






PMSを再起動します。 



 sudo service plexmediaserver restart


これでPMSがインストールされました。 確認してください： 

 startx


ブラウザーに書き込みます。 



 http://localhost:32400/web/index.html


そして、ここに彼は：







好みに合わせてPMSをカスタマイズします。



hddをライブラリに接続してマウントします。 



 sudo apt-get install ntfs-3g
      
      

        
        
        
      

    
        
        
        
      
      

        
        
        
      

    
     


 sudo apt-get install samba samba-common-bin


外部ドライブの名前を確認します。 



 sudo fdisk -l






sda1があります。 fstabを編集します。 



 sudo nano /etc/fstab
      
      

        
        
        
      

    
        
        
        
      
      

        
        
        
      

    
     


 /dev/sda1 /media/WD250Gb ntfs-3g gid=pi,uid=pi,noatime 0 0






保存して終了し、ディレクトリを作成します。 



 sudo mkdir /media/WD250Gb


再起動してグラフィックモードに移行します。



ターミナルでhddをマウントします 

 sudo mount -a


ライブラリをPMSに追加します。











これで、Plexはすぐに使用できます。



起動時にhddが自動的にマウントされるようにするには： 



 sudo gedit /etc/rc.local


追加します： 



 sudo mount -a


起動時にデスクトップを起動する： 



 sudo raspi-config


デスクトップへの起動を有効にします。



plexconnect設定については説明しませんが、すでにここにあります 。



ローカルアクセス、送信用にVNCserverをインストールし、すべてをiPadからアクセス制御し、Apple TVに表示します。



これが、raspberry pi 2のメディアサーバーの結果です。コメントの改善に関するアドバイスをお待ちしています。
