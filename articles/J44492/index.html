マイクロソフトはそれほど悪くない



私はすぐに言わなければなりません：私はマイクロソフトとその製品をとても尊敬しています。 Windows Meの時代は過ぎ去りました。現在、Windows Server 2003は、最も信頼性の高いサーバーOSの1つです。 MS SQL Serverは最も強力な商用DBMSの1つであり、C＃は最高の非動的プログラミング言語の1つです。



私はオープンソースのサポーターであり、マイクロソフトに対して多くの苦情があります（たとえば、「Sunset of the Web？」という記事のドラフトを参照してください。これは、Habréで間もなく公開されます）。 ただし、それらは主に企業のマーケティングポリシーとプライバシーに関連しており、Microsoft製品の機能や品質には関連していません。 したがって、Microsoftであるという理由だけで 、Microsoftの製品はいかなる状況でも使用できないと主張する人々にとって、「マージナル化」の定義に同意します。



...しかし、Internet Explorer 6は使用できません



しかし！ これは、企業の最悪の発案であるInternet Explorer 6には適用されません。 そして、実際に使用できるのは、通常のブラウザの1つをダウンロードするという1つの目的だけです。



その理由は主に3つあります（主観的すぎるため、不便なインターフェイスやタブがないなど、「些細なこと」については言及しません）。

  • 安全性


    MSIE 6は、ブラウザ戦争中に急いで90年代半ばにMicrosoftが開発した非常に脆弱な技術に基づいています。 セキュリティの数千の「パッチ」は、それほど増加していません。 バージョン7および8の完全に新しいカーネルへの移行のみが状況を改善しました。 ただし、MSIE 6にはセキュリティよりも深刻な欠陥があります。



  • 不適切な技術サポート


    一部のWebテクノロジーMSIE 6はサポートしていますが、独自の方法でサポートしています。 したがって、開発者は最初に通常のブラウザ用の標準コードを記述し、次にMSIE 6用に同じコードを書き直す必要があります。
