Operaメニューのチューニング

Hi％username％！ 組み込みのメールやその他の機能を削除するように人々が要求する方法を何度も聞いて、自分で何度かそれを行いました。そのたびに拒否しようとした人は誰でもいました。 しかし、これは私たちの視野から嫌いなものをすべて削除することを妨げませんか？

まず、いくつかの基本情報：
  • 標準メニュー設定ファイルはC：\ Program Files \ Opera \ ui \ standard_menu.ini（Cのツールバー：\ Program Files \ Opera \ ui \ standard_toolbar.ini）に保存されます。
  • Windows 7では、ユーザーメニュー構成ファイルはC：\ Users \ Andrey \ AppData \ Roaming \ Opera \ Opera \ menu \にあります（それぞれCのツールバーの場合：\ Users \ Andrey \ AppData \ Roaming \ Opera \ Opera \ toolbar）。
  • 最終的なプロファイルは、ユーザー設定を標準設定にオーバーレイすることで形成されます。 必要な部分だけを再定義することはできません。
  • 「一般設定」（Ctrl + F12）->「詳細設定」->「パネル」で現在のスタイルを選択できます。 操作を行う前に、編集したプロファイルのコピーを作成し、それだけを変更することをお勧めします。


更新された構成 （現在はブラウザーのバージョンに依存せず、更新中に上書きされません）。 インストールするには、アーカイブを％AppData％/ Operaフォルダーに解凍します（Windows 7 for C：\ Users \ Andrey \ AppData \ Roaming \ Opera \ Opera \）

いくつかの変更：
  • メニュー項目「Exit」、「Opera Unite」、および「Synchronization」は削除されました（最後の2つはステータスバーのボタンで複製されます）。
  • [メニューを表示]項目は[設定]-> [クイック設定]に移動しました。
  • 「メールとチャット...」ボタンが「設定」に移動しました。
  • 「外観」は「設定」に移動しました。
  • ステータスバーのツールバー[表示]に[全画面]ボタンが追加されました。
  • ウィジェットを担当するサブメニューを削除しました。 代わりに、「設定」->「ウィジェットの管理」を使用できます。
  • 開発ツールは「ページ」からメニューのルートに移動しました。
  • テキストのコピー＆ペーストに関連するものはすべて「ページ」サブメニューから削除され、この機能はPCMの右クリックメニューから利用できます。
  • 印刷設定は「ページ」サブメニューに移動しました。
  • コンテキストメニューは、使用頻度の低いアイテムを削除することで大幅に削減されました（「デスクトップの背景として設定」など）。
