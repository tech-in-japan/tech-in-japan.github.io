血統MS Windows`7

最新のオペレーティングシステム（OS）MS Windows 7について議論すると、多くの現代ユーザーは、Microsoftオペレーティングシステムの完全に異なる世代とファミリの間にまったく違いがないことを突然発見しました。

誰もが、次期リリースである最新のMS Windows 7 OSがMS Windows NTファミリーのOSに属していることを理解していないことがわかります。 「パブリック名」の混乱とMS Windows NTのバージョンの未発表の番号付けがこれにつながります。

Windows NT 3.1、Windows NT 4.0、Windows NT 5.0（Windows`2000）、Windows NT 5.1（Windows`XP）、Windows NT 6.0（Windows`Vista）、Windows NT 6.1（Windows`7）。

そのため、 Microsoft Windows OSファミリの開発の歴史に関する知識を共有したいと思います。



Windowsの開発の歴史は、いくつかの「グループ」（個々のファミリー）に分けられます。

私は個人的に、Microsoftがマーケティングのために1つの名前に合併したときにミスを犯したと考えています。Windowsは 、さまざまな時代と人々のために生産された、まったく異なる3つのパーソナルコンピューター製品群です。 さらに、Mobile- Windows MobileおよびEmbedded- Microsoft Embeddedオペレーティングシステムは同じ名前に属します！



たとえば、Appleは消費者に対してより正直であり、だれもがAppleが2つのファミリーのパーソナルコンピューター用オペレーティングシステムしか持っていないことを知っています。

1. Apple MacOS-バージョン1.0（1984）からMacOS 9.2.2（2001）までのクラシックシステム。

2. Apple MacOS X - NeXTstep + MacOSから開発され、 ダーウィンカーネル（ハイブリッドXNUカーネルを備えたシステム： MachマイクロカーネルとFreeBSDサービスに基づいて作成された）で実行される新しいOS、バージョン10.0（2001）から10.6（2009） ）



また、Microsoftには3つの異なる（競合する）システムファミリがあり、成功するのは非常に困難でした。 そして、それらのすべてが本格的なオペレーティングシステムに起因するわけではありませんが、同時に、それらはすべてMicrosoft Windowsカテゴリ（非常に混乱したユーザー）に属します。

当初、1985年にMS Windows 1.0がリリースされました。これは、MS-DOSオペレーティングシステム上で動作するグラフィカルシェル（ウィンドウ環境）です。

そして、Windows環境という単純な名前を付け始めたのはウィンドウ環境でした！

1. MS Windowsウィンドウ環境のリリースにより、製品ファミリ全体が生まれました。

ちょっとした歴史：

ナンバリングMS Windows NTは、元々はマーケティングの利点に基づいて構築されました。

MS Windows NTの最初の商用バージョンには、すでに番号3.1（成熟した製品バージョン番号）が付いていました。

Softの開発者の間で普及しているすべての規則によって、最初の商用バージョンの数は1.0である必要がありますが、少なくとも1.1であり、3.1ではありませんでした！



NT 3.1番号は、Microsoftが説明したように、API OS / 2 2.0の代わりに、まったく新しいMicrosoft OSがAPI MS-Windows 3.1（MS-DOS 6.0上で実行されるウィンドウ環境）に統合されたために登場しました。



MS Windows NT 3.1のリリースを今でも覚えていますが、それはひどく生で、未完成で、ひどく遅かったです。 実験のため以外は、ほとんど誰もそれを買いませんでした。

1993年の最新の機器であるIntel 486DX-66 MHzでは、想像できないほどのブレーキングであり、実際に作業することはできませんでした。



1994-95年。 MS Windows NT 3.5が登場し、まもなくMS Windows NT 3.51が登場し、同時に、手頃な価格のIntel Pentium-100 MHzプロセッサを搭載した新しいコンピューターが市場に登場しました。

そして、当時最新の機器が使用されていたMS Windows NT 3.51のみが、仕事用のOSとして実際に使用できました（時々、泣きながらブレーキをかけます）。

個人的には、1995年にリリースされたMS Windows NT 3.51のみがバージョン番号1.0を正しく運ぶことができ、それ以前は、Windows NTのすべてのバージョンがAlfaおよびBeta製品である可能性が最も高いと考えています。



さて、1996年に、MS Windows NT 4.0が登場しました。これは、ほぼ2001年まで、一部のコンピューターで（以前のバージョンのNT OSのように実験的ではなく）動作OSとして使用されていました。 彼女には問題がありましたが、その主な原因は、MS Windows`9xファミリーのOS向けに最適化されたソフトウェアとの互換性の低さでしたが、より安定したOSでした。



当初、MS Windows NTのすべての最初のバージョンでは、「公開番号」は製品のバージョン番号と一致していましたが、2000年にはMS Windows NTファミリのOSの名前で本格的なカルーセルが始まりました。



2000年までに、Microsoftは大多数のユーザーが従来のOS Windows MS9`xファミリを正常に使用しており、MS Windows NTファミリは受け入れられないハードウェア要件のために人々から支持されなくなったことに気付きました。

また、Microsoftは、ほとんどのユーザーがクラシックでありながら将来性のないMS Windows`98 OSに留まっているユーザーを、新しいMS Windows NTファミリーのOSにドラッグしようとし始めました。

そして、ここでマーケティングが働き始めました。 MS Windows`98以降のユーザーが新しいMS Windows`2000 OSがプレインストールされたコンピューターを購入することを保証するために、すべてが行われました。 しかし実際には、それはMS Windows`9xファミリーとは何の関係もありませんでした。美しい名前で新しいバージョン5.0が発表されました。 MS Windows`98。



しかし、MS Windows 2000の最初のリリースには多くのバグがあり、MS Windows`9xファミリ用に作成されたソフトウェアとの互換性が低かったため、MS Windows 2000のデビューは失敗しました。

2001年には、Windows NT 5.1のバージョンがありました。これは、MS Windows XPという名前で販売され始めました。 MS Windows XPとのみ互換性があり、MS Windows`98には適さないドライバーを作成し始めました。



そうすれば、MS Windows Vista-MS Windows NT 6.0-2007の新しいコアを備えたOSに起因する障害を誰もが知っています。

そして2009年の今日、Microsoftは新しいMS Windows 7オペレーティングシステムを提供します。バージョン番号はMS Windows NT 6.1です。

つまり、MS Windows NT 6.0で少し更新され、デバッグされています-同じVista :)

失敗したWindows 2000（NT 5.0）の後にWindows XP（NT 5.1）が成功を期待したときと同じように、今日のMS Windows 7は人々の間で成功と人気を期待していると思います！

しかし、バージョン番号がWindows NT 6.1であるときにMS Windowsを「7番目」と呼ぶ必要があった理由を理解できません。

このきれいな名前を次の名前（実際のMS Windows NT 7.0）のために保存して、今度は他の名前を使用し、ユーザーをあきらめないでください:)
