2017年のPalmのプログラミング

私は、冷静な心と良い記憶の中で、2017年に消えたプラットフォームのプログラミングを開始する方法を説明します。



2017年5月12日でした。 窓の外に横たわる雪と暖房がオンになり、永遠の想いが喚起され、私の視線はPalmOSを実行しているシックな工業用PDAに落ちました。





データ収集端末



一度それが私に与えられたが、私はそれを使用したことがない。 最初、彼はCOMポートのレースを持っていませんでした。 そして、接点への人為的なはんだ付けの過程で、クレードルのマザーボードからPDAポートコネクタへのケーブルが切断されました。 そして、彼は横たわっていました。



そして、私はこの手のひらを蘇らせたいと思ったので、やる気があったので、Linux用のエミュレーターを調べて、ソフトウェアとは何か、ゲームがろうそくに値するかどうかを調べることから始めました。 そして、エミュレータのインストール、検索で、私は少し夢中になりました... 5月の雪、寒さは私にこの影響を与えたので、私はこのPDAのために書くことを試みることに決めました。 さらに、Palmは私の学生の若者のPDAであり、それはめちゃくちゃクールなガジェットでした。 それから私は彼らのためにプログラムする方法を知りませんでしたが、本当にしたかったです。 14年以上経った後（古さを感じますか）-私はそれをやろうと決めました。



このPDAをエミュレートする方法に興味がある場合、またはより良いソフトウェアを作成する場合は、さらに読み進めてみました。



Palm用の開発を開始するには何が必要ですか？



この質問は誰かに簡単に思えるかもしれません。彼らは言う、コンパイラをダウンロードして、行く。 はい、ヤシの木の設計は非常に簡単です。 単純さは、よく知られている図と同じです。









おもしろいことに、Windowsユーザーは、 MetrowerkのCodeWarrior for Palmコンパイラを購入するか、cygwinを使用してWindowsをUNIXに変更する必要があり、Linuxの場合はすべて以下のようになります。 現代のWindowsの現状はわかりませんが、その道筋は明らかに本物の変態です。多くのフォーラムや記事から判断すると、誰もがそれを処理できるわけではありません。 そのため、Linux Mintの詳細な手順について説明します（ubuntu 14.04をお読みください）。 方法は機能しています。



したがって、ソフトウェアの開発、デバッグ、チェックに必要なものは何ですか：



  • Palmエミュレーター（POSE）。
  • POSEのROMイメージ。
  • PRCツール（PalmOS用プログラムをコンパイルするGCC）。
  • PalmOS SDK。
  • PilRC（リソースコンパイラ）。


エミュレータPalm POSE（Palm Operation System Emulator） -Palmにコピーせずにあらゆる種類のプログラムを実行できるプログラム。 これはとても便利です。 特に、ヤシの木を持っていないが、別のソフトウェアで遊びたい人のために。 しかし、エミュレーターはROMイメージなしでは機能しません。これは、実際にはライブPDAのファームウェアイメージです。 同じPOSEで実際のPDAから削除するか、インターネットでダウンロードできます。 実際、 PRCツールはプログラム（コンパイラ、リンカ、デバッガ）をコンパイルするためのツールのセットです。 悲しいことではありませんが、PRCツールは特定のバージョンのPalmOS用のPalmOS SDKなしでは機能しません。 これを理解するために、マニュアルを読むのに数日かかりました。 最後に、 PilRCリソースコンパイラとビューアを使用します。これにより、メニューのレンダリングと画面上の画像の置き換えを行うバイナリファイルを準備できます。

PalmOSエミュレータ



この記事の最も興味深い、おそらく要求される段落。 プログラムを収集する人はほとんどいませんが、古いおもちゃや他のソフトウェアを運転してみてください。

それをインストールするには、次のものが必要です。



  • POSEエミュレーター自体
  • さまざまなPDAのスキン
  • エミュレートするROM-PDAファイル。


エミュレータはここで見つかりました （アーカイブ内のファイルpose_3.5-9.1ubuntu2_i386.deb）。 チームでダウンロードしてインストールする 



sudo dpkg -i pose_3.5-9.1ubuntu2_i386.deb


スキンはPOSEソースとともにここで見つかりました 。 これは、アーカイブ内のファイルemulator-skins-19.tar.gzです。 このアーカイブをホームフォルダーで解凍し、スキンまたはスキンに名前を変更します。 その結果、外観は実際の鉄片に対応します。 これは素晴らしいボーナスであり、それなしではすべてが機能します。



エミュレータが機能するには、ROMからファイルを提供する必要があります。 ここでローマが見つかりました 。 個人的には、 palmos40-en-m505.romを使用します。 このソフトウェアはロシア文字を好まないため、ROMを取得するフォルダーでコンソールからPOSEを開始します。 空のウィンドウを右クリックし、開いたウィンドウで[ファイル]-[新規]を選択し、ROMファイルとファイル自体を選択します。









OKをクリックします。 すべてが正しく行われていれば、ほぼすべてのPalmソフトウェアを実行できる動作中のエミュレータが得られます（ハードウェアでの動作を除く）。









すぐに使えるエミュレーター



最も楽しいボーナスは、ライブPDAにインストールするのとは異なり、プログラムのインストールが2回クリックするだけで非常に迅速に行われることです。 これは、ソフトウェアを作成およびデバッグするときに重要です。



ツールチェーンとSDK



すべてが機能するようにするには、さらにいくつかを追加する必要があります。 つまり、PRCツール、パーム用SDKおよびPilRC。 PRCツールは、最愛のGCCの移植ですが、PalmOS用の出力形式を備えています。 SDKには、Palm API用のいくつかのライブラリと一連のヘッダーがあります。 最後に、リソースをコンパイルするにはPilRCが必要です。 以下では、PRCツールとSDKのインストール方法について説明します。



PRCツールのソースはここにあります 。 これは、binutils、gdb、GCC、およびリンカー用のパッチの束で、PalmOSサポートを提供します。 私は個人的に指示に従って設定しました 。 以下の指示はそのコピーです。 パッケージをダウンロードする必要はありませんが、prc-toolsアーカイブ内のフォルダーからすべてをインストールするだけです。 



 cd /tmp #  wget http://ftp.gnome.org/mirror/cdimage/snapshot/Debian/pool/main/p/prc-tools/prc-tools-arm_2.2.90.cvs20030306-6_i386.deb wget http://ftp.gnome.org/mirror/cdimage/snapshot/Debian/pool/main/p/prc-tools/prc-tools-doc_2.2.90.cvs20030306-6_all.deb wget http://ftp.gnome.org/mirror/cdimage/snapshot/Debian/pool/main/p/prc-tools/prc-tools-m68k_2.2.90.cvs20030306-6_i386.deb wget http://ftp.gnome.org/mirror/cdimage/snapshot/Debian/pool/main/p/prc-tools/prc-tools-utils_2.2.90.cvs20030306-6_i386.deb wget http://ftp.gnome.org/mirror/cdimage/snapshot/Debian/pool/main/p/prc-tools/prc-tools_2.2.90.cvs20030306-6_all.deb wget http://old-releases.ubuntu.com/ubuntu/pool/universe/p/pilrc/pilrc_3.2-3_i386.deb sudo apt-get install libstdc++5 sudo dpkg -i *.deb


以上で、コンパイラが動作するようになりました。 しかし、ヘッダーファイルの代わりにスタブがあるだけなので、彼はコンパイルしません。 SDK自体をインストールする必要があります。 彼はPalmOS 3.5用に開発することを計画し、 それをここで取り上げました 。PalmOSの新しいバージョン（4thと5th）もあります。



同様に、wgetファイルをダウンロードすることはできませんが、sdk-3.0フォルダーに移動して、そこからすべてを作成します。 



 # sdk wget http://web.archive.org/web/20141224205502/http://cdn.xenu.tk/pub/palm-archive/pub/programming/sdk/3.0/sdk35.tar.gz #    sudo mkdir /usr/local/palmdev sudo mkdir /usr/local/palmdev/sdkpalmos-3.5 #  tar -xzf sdk35.tar.gz #     sudo cp -r Palm\ OS\ 3.5\ Support/* /usr/local/palmdev/sdkpalmos-3.5/ cd /usr/local/palmdev/sdkpalmos-3.5/ #     (       ) sudo mv Incs include sudo mv GCC\ Libraries lib #  ! #   ,      SKD      . sudo palmdev-prep /usr/local/palmdev/sdkpalmos-3.5/


SDKには便利なアーカイブがいくつかあります。



  • sdk35-docs.tar.gz-ドキュメントのアーカイブ。 私の意見では、ただの発見！
  • sdk35-examples.tar.gz-例を含む素晴らしいアーカイブ。 ダウンロードするか、アーカイブから取得することを強くお勧めします。


この段階では、原始的なハローワードを組み立てることさえできます！ ファイルhello.cを作成します。 それに書きます 



 #include <PalmOS.h> UInt32 PilotMain( UInt16 cmd, void *cmdPBP, UInt16 launchFlags) { EventType event; char *message = "Hello, world!"; if ( sysAppLaunchCmdNormalLaunch == cmd) { WinDrawChars( message, StrLen(message), 55, 74 ); do { EvtGetEvent( &event, evtWaitForever); SysHandleEvent( &event); } while (event.eType != appStopEvent); } return 0; }


コマンドのシーケンスによってコンパイルされます 



 m68k-palmos-gcc HelloWorld.c -o hello m68k-palmos-obj-res hello build-prc Hello.prc "Hello" HELO *.hello.grc


hello.prcファイルをエミュレータに入力し、実行して取得します









PilRCリソースコンパイラ



一般的に、あなたはプログラムできると思われるかもしれません。 これがその方法であり、すべてのプログラムはコンパイル、ビルド、実行されますが、動作しません！ 彼らはまだリソースをコンパイルする必要があるので。 （論理的、いまいましい）。 リソースをコンパイルするには、PilRCコンパイラ自体が必要であり、PilRCUIを表示します。 このプロジェクトは生きており、ソースコードはここにあります 。 優れたドキュメントが豊富にあるため、ソース（アーカイブで入手可能）をダウンロードすることを強くお勧めします。 そして、パッケージ自体は古いubuntuリポジトリで私によって発見されました。 パッケージコンパイラの場合は、pilrc_3.2-3_i386.debファイル（アーカイブで利用可能）をダウンロードし、コマンドでインストールする必要があります 



 sudo dpkg -i pilrc_3.2-3_i386.deb
