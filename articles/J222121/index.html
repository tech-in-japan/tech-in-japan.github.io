Tryton ERP開発：パート1

メーカーの会社向けにERPシステムを作成するための大規模なプロジェクトを考え出しました。 サーバーとして、選択肢はtrytonに落ちました。 ただし、そのドキュメントはかなり悪いです。 インターネットでは、 公式ドキュメント 、 説明付きのコード、およびirc trytonチャネルのログを見つけることができました。 この情報は、tryton上の任意のシステムの開発には非常にわずかです。 このギャップを埋めるために、開発プロセス全体、特に直面しなければならない落とし穴を文書化することが決定されました。 それでは始めましょう。



パート1：tryotndのインストール、クエリ構造、データベースの操作

パート3：会社（会社）、取引先（パーティー）、商品（製品）、倉庫（在庫）のモジュールを操作します。

2.リクエストの構造、データベースを操作するAPI、データベースを作成する



構成を変更し、サーバーを起動します。 ここではサーバーAPIの不十分な説明を見つけることができますが、これでは明らかに十分ではないため、さらに理解し続けます。 ネットワークは、サーバーと直接連携するためのPythonコードでいっぱいです。 次のようになります。 



 from jsonrpc import ServerProxy connection = ServerProxy("localhost", "8000", "tryton", verbose=0) name = "model.ir.model.search" model_ids = conn.execute(name, [])




このコードを実行しようと長い間試みた結果、jsonrpcはpypiの標準ライブラリではなく、クライアントから選択できる独自のtrytond開発者ライブラリであることがわかりました（trytond / protocols / jsonrpc.pyと混同しないでください-これらは異なるライブラリです）。 trytonをダウンロードして解凍し、trytonフォルダーに大切なjsonrpc.pyを見つけます。 これで、Pythonでtrytondと通信できるようになりましたが、PHPにフロントエンドがあるため、さらに掘り下げていきます。



2.1。 リクエスト構造


長いデブリーフィングの後、最初の動作コードを取得します（trytondサーバーが実行されていることを忘れないでください）： 



 import json import requests url = 'http://localhost:8000/' id = 1 methodname = 'common.db.list' params = '[]' request = json.dumps({ 'id': id, 'method': methodname, 'params': params, }) r = requests.post(url, data=request) print r.json()




このコードはすでに理解可能であり、必要な言語（php、Javaなど）に転送することは難しくありません。 次に、このクエリを調べます。



リクエストURLを処理します。 すべてのデータベースアクションはurlで実行されます localhost:8000/

2.2。 データベースを操作するためのAPI

メソッド名 パラメータ 説明
common.server.version [] サーバーのバージョンを取得する
common.db.list [] 既存のデータベースのリスト
common.server.login [ユーザー名、ユーザーパス] ログイン済み（返されたIDとユーザーのCookie）

Trytonには匿名で認証されたリクエストがあります。 たとえば、common.server.versionは匿名リクエストであり、サーバーにログインせずに実行できます。 common.db.createの場合、idとcookieを省略できますが、リクエストは引き続き実行されます。 認証されたリクエストの場合、IDとCookieを指定する必要があります。
