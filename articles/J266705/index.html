展開のためのアプリケーションの準備の基本（アプリケーションのパッケージ化）





「アプリケーションパッケージ」とは、世界中のエンドユーザーサポート（EUS）分野のITサービスの名前の確立された用語ですが、ロシアでは事実上不明です。 したがって、この記事では、このサービスの背後に隠れているベストプラクティスについて説明し、ソフトウェアの総所有コスト（TCO）を削減し、最終的にはユーザージョブをサポートするコストを削減する方法について説明します。



戦略とアプローチ。 TCOの削減。





ソフトウェア（ソフトウェア）の所有コストは、現代の企業のIT予算のかなりの部分を占めており、ユーザーワークステーションのサポートコストを削減するために、予算とプログラムの両方の管理（アプローチ、戦略など）の関連性を決定します。 そのような実績のあるアプローチの1つは、 アプリケーションのパッケージ化です （ロシア語の翻訳「展開のためのアプリケーションの準備」）。



アプリケーションの展開準備の結果、必要なユーザー、地域、ライセンスの設定をすべて含む1つ以上のアプリケーションを含むパッケージが作成されます。必要に応じて、互換性などの既知の問題を解決するように調整されます。 原則として、すべてのパッケージには、インストールと削除のための共通インターフェイス（ほとんどの場合、コマンドライン、場合によってはUI）があり、サービスデスクのさらなる操作を容易にします。 また、パッケージの開発中に、企業ポリシーとベストプラクティス（ベストプラクティス）が適用されることに注意することも重要です。



  • ユーザーの作業を妨げないために、サイレントモード（サイレント）でインストールする機能。
  • 更新を無効にしてトラフィックを最小限に抑え、プログラムの使用済みバージョンを制御します（そして異種バージョンの数を最小限に抑えるため、メンテナンスプログラムがより高価になります）
  • ソフトウェア開発者が必要とする最も困難な状況でもエンドユーザーの作業を中断しないように、コンピューターを再起動およびシャットダウンせずに厳密にインストールします。
  • パッケージレベルで以前のソフトウェアを自動的にアンインストールして、展開作業を最小限に抑えます。
  • そして、他の何百もの慣行....


したがって、パッケージにはすべてのベストが組み込まれており、開発の段階ですでに要件が実装されているため、高い資格を必要とする作業から抽象化することができます。 統合されたインストールとアンインストールのインターフェイスにより、「アプリケーション」の本質から距離を置くことができ、Microsoft SCCMなどのソフトウェア展開システムを使用して、インストール/アンインストールコマンドを数万、数百、数千のクライアント（数万台のコンピューターを持っているお客様）に簡単に複製できます他の同様のシステム。

技術とソリューション



現在までに、企業および民間部門向けの展開用ソフトウェアを準備するための多くの技術とプログラムがあります。



最も有名なテクノロジーはMicrosoft Windows Installerです。MicrosoftWindowsのサブシステムは、特別な形式の.msiでプログラムをインストールします。 管理されていない管理されていないsetup.exeテクノロジーとは異なり、非常に人気のある多くの利点があります。



  • インストールプロセスは、標準のWindows OSポリシーによって制御できます。
  • インストールされたプログラムに関する情報は、構造化された形式（データベース）とcabアーカイブに含まれており、透過性（ウイルス対策を含む）を保証します。
  • 個々のリソースが破損した場合、自己修復が可能になり、インシデントの数が減ります。
  • 強制回復の可能性。これにより、整合性が失われた場合にアプリケーションを復元する時間が短縮されます。
  • アプリケーション間の競合を事前に特定して解決する機能
  • など


ソフトウェアメンテナンスのコストを削減することを目的とした他の多くの利点により、このテクノロジは企業部門の事実上の標準となっています。 テクノロジーの利点についてはこちらをご覧ください 。

今日、ソフトウェア仮想化技術、特にMicrosoftのAppVソリューションがますます人気を集めています。そのため、Habréに関する別の記事を取り上げました。 仮想化技術は毎年ますます人気を集めています。 現在、ヨーロッパの大規模な顧客の半数はAppVをメインテクノロジーとして使用しており、AppVが適用されない場合はWindowsインストーラーが使用されています。 後者は、端末環境が一般的に使用される場合に特に有益です。 競合管理（プロアクティブな検索と排除）と端末環境での使用の可能性のプロアクティブな分析によるアプリケーションパッケージにより、上記のすべての利点が得られるだけでなく、物理サーバーをより合理的に使用して、サーバーの再構築数とインシデントと問題の総数を減らすことができます、つまり、所有コストを25％削減します。 競合管理には、独自の開発とFlexera Softwareの Application Managerソリューションを使用します。



さらに、展開用のアプリケーションの準備にサービスを使用すると、新しい環境への移行のリスクとコストを大幅に削減できます（たとえば、デスクトップソリューションから端末または新しいOSへ）。 そのため、現在、一部のクライアントのWindows 10への移行に積極的に取り組んでいます。アプリケーションパッケージを既に使用しているお客様については、会社の規模に応じて数週間/数か月でこの作業を行います。 アプリケーションのパッケージングがまだ使用されていない場合、移行時間はサービスの導入に最適な期間であり、 同時にソフトウェアの構成を最適化します。実際、サポートの追加コストは言うまでもなく、移行予算のみを数倍節約できます。 合理化に使用されるプログラムについては、この資料では少し低くなっています。
