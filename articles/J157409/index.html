記事で説明されている機能のどれもがあなたにとって新しいものではない場合は、少なくとも自分の1つを追加するのに苦労してください。



私はMacで作業しているため、「自分の」レイアウトでホットキーを指定します。 それらのほとんどは、[CMD]を[CTRL]または[ALT]に置き換えるだけで機能し、残りは[キーマップ]タブで検索する必要があります。 誰かがWINのレイアウトをチェックして複製してくれたら感謝します。



コードを自動生成



フレームワーク自体によって実行されることもあれば、手間のかかる作業が必要になることもあるルーチンタスクがたくさんあります。 PHPStormは、たとえば次のように[CTRL] + Nを簡略化するのに役立ちます。



ゲッターとセッターの作成（Getters＆Setters）

既存のクラス変数に基づいて、ゲッターおよび/またはセッターのセットを生成でき、表示されるウィンドウで、どれを選択できます。



PHPDocの作成

基本的に、アクションは、既存のメソッドの前に/ **の書き込みを開始してEnterキーを押した場合の方法に似ています。 1つ以上の機能を選択できます。属性とコンテンツに基づいて、Docブロックが形成されます。



オーバーライドと実装

子孫クラスを作成するとき、または抽象メソッドを実装するとき、間違いを避けるために、通常は先祖クラスにクロールし、メソッドの名前と属性を記憶またはコピーアンドペーストする必要があります。 または、すべての抽象メソッドの実装があるかどうかを確認し、そこでArrayAccessに必要なものを覚えておきます。 より良いオプションがあります：

[CMD] + I-実装

[CMD] + O-再マッピング

ホットキーをクリックすると、ウィンドウが表示され、実装するメソッドを選択できます。 選択後、次のように、プライバシーと属性名に関して適切なメソッドが生成されます。 

public function createTable($drop = false) { return parent::createTable($drop); }




コードの自動フォーマット



コードを自分で作成する場合、フォーマットは通常それ自体で行われますが、コードスタイルを変更したり、他の誰かの異常に設計されたコードを開こうとする状況では、問題が発生します。 調整するか、 [CMD] + [ALT] + Lを押すと、IDEは設定に従ってコードをフォーマットします（設定->プロジェクト設定->コードスタイル-> PHP）



デフォルトでは、ホットキーをクリックするとファイル全体がフォーマットされます。 エリアを選択すると、選択範囲のみがフォーマットされます。 ツリーでフォルダまたはプロジェクト全体を選択すると、すぐにすべてを整理できます。



言語ごとに独自のデザインスタイルを設定できます。PHPの場合は、事前定義されたスタイルのいずれかを選択することもできます。

ヒント（コード補完）



IDEの重要な機能は、メソッドの名前と属性を要求する機能です。 これは、3〜4の継承されたクラスのチェーンが誰をも混乱させる可能性があるOOPにとって特に重要です。



プロジェクトの通常のアーキテクチャでは、IDEは独自にそれを把握できますが、次のように対処できない状況があります。

  • 関数はいくつかの型を返すことができます
  • オブジェクトは配列要素です。

    $ var = $ arr ['オブジェクト'];
  • クラス「B」は「A」を拡張します。 クラス 'A'では、foo（）メソッドは$ thisを返します。クラス 'B'では、メソッドの実装はありません。 オブジェクト 'B'からfoo（）メソッドを呼び出すと、IDEはオブジェクトAが返されたと判断します。
  • PHPを使用したネイティブテンプレート。 システムのどこかで、変数$ someがテンプレートに渡されますが、これはテンプレート内ではわかりません。




ソリューションオプション：







関数の場合、@リターンパラメータを使用してPHPDocを記述する必要があります。 例： 

 /** @return SomeClass */ function a() {[…]}




変数については 、コメントを指定することもできます。 

 /** @var $my_var SomeClass */


次に、$ my_varのヒントが利用可能になります



クラスの上のメソッドの場合、PHPDocを指定できます 

 /** @method B foo()   */ class B extends A {[…]}




関数がオブジェクトの配列を返す場合 

 /** @return SomeObject[] */


次に、たとえばforeachで、ヒント$ value-> ...が使用可能になります（SomeClassクラスのフィールドとメソッドによって）。 ありがとうpro100tak



タスク



バージョン管理システムの使用は、長い間好意的な兆候でした。 しかし、多くの場合、複数のタスクで並行して作業が行われている場合、または誰かが今このバグを修正し、緊急に修正するよう求めている場合に問題が発生します。 必要なファイルをコミットすることを忘れることは非常に簡単です。逆もまた同様です。余分なファイルがコミットされるか、そこにあるべきでない変更がコミットされます。



これを回避して、タスクを処理する順序をもたらすことができます。 一番下の行は、各タスクに独自の変更リストがあり、変更を含むファイルをリストし、タスクを切り替えると、このタスクで作業したファイルと覚えておく必要があるものがすぐにわかります。







もう1つの良い点は、1つのタスクの一部として別のファイルで既に編集されているファイルの編集を開始すると、IDEが警告することです。



コミットすると、現在の変更リストのすべてのファイルが自動的に追加され、タスク名がコメントに置き換えられるという便利さが追加されます。



同じGitHubにJiraやRedmineのようなバグトラッカーが既にある場合は良いでしょう。同じGitHubには、PHPStormでもサポートされているかなり簡単なバグ追跡システムがあります。 これにより、IDEで直接作業している既存のタスクの番号、名前、および説明を読み込むことができますが、トラッカーを使用して任意の変更リストを作成することを妨げるものはありません。



FTP、デプロイ、自動展開



多くのプロジェクトはFTPまたはSFTPで動作します。 PHPStormには、それらを操作するための便利な機能があります。 サーバーへのアクセスを保存し、マッピングを構成すると、1つのホットキーを押すだけでサーバーにファイルをアップロードおよびダウンロード（アップロードおよびダウンロード）できるようになります。 動作するには-「このサーバーをデフォルトとして使用する」ことを忘れないでください







何らかの理由で、Macにはこれらのアクションのデフォルトのホットキーがありませんが、キーマップで設定できます（ありがとうCap）。 以下を設定します。

•デフォルトサーバーへのアップロード：[CTRL] + U

•デフォルトサーバーからダウンロード：[CTRL] + D

（キーマップで検索しやすくするためにそれらを作成しました+私の意見では覚えやすいです）



しかし、これについては、自動アッシング機能でない場合、すべてを書く価値はありません。 設定（環境設定->プロジェクト設定->展開->オプション）で、「変更なしファイル+外部変更のアップロード」チェックボックスを除くすべてのモードで「変更されたファイルをデフォルトサーバーに自動的にアップロード」項目を入力すると、すべての変更が自動的に同期されます。



これは、[CMD] + Sを押すと、すべてのファイルが一度に保存されるため、結果として保存手順とアップロードが1つにまとめられるためです。



今、あなたはすべてがロードされているかどうか、何も忘れていないかどうか気にすることはできません。



コンソール



多くのフレームワークは、コンソールを使用して頻繁に頻繁にアクションを実行します。 Symfony、Zend、およびその他の一般的なフレームワークの場合、PHPStormはコマンドラインツールコンソール[CMD] + [SHIFT] + Xでツールチップとコンソールコマンドの自動補完をサポートします



ただし、プロジェクトまたは一般的に使用されるUNIXまたはGITコマンドに対して同じことを行うことを妨げるものはありません。このため、実行可能ファイルを選択し、次の形式の特別なXMLファイルを作成する必要があります。 



 <?xml version="1.0" encoding="UTF-8"?> <framework xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:noNamespaceSchemaLocation="schemas/frameworkDescriptionVersion1.1.2.xsd" name="My Tools" invoke="/full/path/to/console.php" alias="c" enabled="true" version="2"> <command> <name>command-name</name> <help> .     !</help> <params>class</params> </command> <command> <name>another:command</name> <help>    </help> <params>class[=null]</params> </command> </framework>




これがどのように行われるかの詳細な説明は、 devnet.jetbrains.net / docs / DOC-1230にあります。



検索の種類



[CMD] + N-クラス名で検索



クラス名で検索します。 大規模なプロジェクトでは、すべてがディレクトリの奥深くに埋まっている場合や、どこにあるかが分からない場合に便利です。 スローターは「スマート検索」を追加します。 クラス名と最初の文字の両方で検索されます。たとえば、SomeVeryLongClassは「SVLC」で検索されます。 大文字と小文字を区別するため、注意してください。







UPD：「スマート検索」開発者はCamel's Humpを呼び出します。 ところで、設定では、単語の選択と移動を行うことができます。完全に（スペースまで）実行されるのではなく、単語に従って（大文字まで）実行されます。 ありがとうyujin1st



[CMD] + [SHIFT] + N-ファイル名で検索



ファイル名で検索します。 クラス名による検索と同様に、スマート検索を使用します。



UPD：ネストしてファイルを見つけるには、スラッシュを使用できます：some / deep / dir / with / file.txtは「s / d / d / w / f」で見つけることができます



[CMD] + [SHIFT] + [ALT] + N-シンボルの検索



メソッド/メソッドと定数の検索は、メソッドの名前を正確に覚えているが、クラスは覚えていない場合に非常に便利です。 confに感謝

プロジェクトファイルで任意のフレーズを検索します。 リファクタリングするとき、それは通常、Find Usagesをうまく補完します。



必要なものを検索するディレクトリを制限したり、スコープを構成したりする能力がない場合、「単なる検索」について書く価値はありません。 スコープを使用すると、将来の使用に備えて検索領域を柔軟に構成および保存できます。 フレームワークファイルやサードパーティのファイルなど、検索から削除できます。







タブキャプションフレームを非表示
