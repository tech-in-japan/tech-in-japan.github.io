Web 2.0という用語は、インターネットユーザーの語彙にすでに根付いています。 かなり頻繁に説明する必要があるという事実にもかかわらず、この用語の定義をめぐって槍を破る人々の輪は狭まっています。 Web 2.0は直感的に理解できる概念の1つですが、正確な言葉遣いを受け入れられませんが、いずれにしても、Web 2.0といえば、コンテンツを担当するさまざまなサービスのユーザーとの共有に基づく現代のインターネット標準を意味します。 同時に、この標準の存在は、今日では非常に多く作成されているテクノロジーによって提供されますが、それらはすべて密接に絡み合って互いに統合されていることに注意する必要があります。



このまたはそのテクノロジーをWeb 2.0に帰属させようとするとき、または標準への影響を評価しようとするときこそ、ほとんどの論争が発生します。 読者の意見に異議を唱えるリスクとして、RSS、XML、Ajax、ウィジェット、無料の分類システム（タグまたはタグ）、ユーザー空間のパーソナライズ、コンテンツに対するユーザー制御、社会性（社会化）、API、統合およびサービスの補完性（マッシュ-上）。



Web 2.0プラットフォームは、革新的な方法ではなく、進化的な方法で登場しました。 これは、Web 2.0という用語をインターネットの概念フレームワークに導入したと考えられているTim O'Reillyの有名な記事で説明されています。 2005年に新しい現象を分析して、ティムオライリーは次のように述べています。「2001年の秋のドットコムの崩壊は、インターネットのターニングポイントでした。 多くの人々は、ウェブの概念そのものが誇張されていると判断しましたが、実際、そのような「シャボン玉」、そしてその結果、在庫の減少は、すべての技術革命の避けられない仲間です。



Web 3.0の次の時代についてのコメントがすでに出ているように、Web 2.0に気づき、Web 2.0に慣れる時間がある前に。 Webプラットフォームの革命的な変化の位置を認めるなら、Web 2.0はWeb 1.0でかつて発生した避けられない崩壊を予期する可能性が非常に高いです。 現在、数年前と同様に、新しいサービスには常に投資が行われていますが、その一部は多額のお金で購入されています。 しかし、問題は、明らかに、多くのWeb 2.0リソースの効果的な収益化の方法を知っている人はほとんどいないということです。 現時点では、他のものに対する彼らの利点は、関係するトラフィックにありますが、後者との「ゲーム」はすでに時代遅れで発信的なもののような匂いがします。



しかし、専門家の間では、インターネットの進化的発展の支持者を観察することができます。 原則として、これらはビジネスをWeb 2.0と密接に結びつけ、新しい「シャボン玉」の考えさえ許さない人々です。



そのような楽観主義者の1人がGoogle Inc.のディレクターです。 エリック・シュミット。 最近、 ソウルデジタルフォーラムで講演し、ジャーナリストの質問に答えようとしました。GoogleがWeb 3.0という用語の意味について語っています。 これに対して、Eric Sc​​hmidtはWeb 2.0について冗談を言って、「マーケティング用語」と呼びました。 その後、彼はWeb 3.0が「アプリケーションをまとめる」プラットフォームになると述べました。 それらは、コンピューターであろうとモバイルガジェットであろうと、比較的小さく、どのデバイスでも実行できる汎用性があります。 さらに、これらのアプリケーションは非常に高速に動作し、「ソーシャル」ネットワーク（メールなど）を介して「ウイルス」のように広がります。



したがって、今日、インターネットの開発において考えられる2つの傾向を特定できます。ユーザーアクティビティを維持しながらWebアプリケーションの進化によるWeb 3.0への「ソフト」移行と、Webテクノロジーの開発における重点の特定の変更です。 後者をどのように実現できるかはまだ不明です。 3Dインターネットが私たちを待っているかもしれません。あるいは、GoogleのトップマネージャーがWeb 3.0について話しているなら、Webのセキュリティを強化するために、さまざまなサービスでユーザー登録の統一を達成するかもしれません。長くない。
