Veeam Backup＆Replication v8リリース（有料および無料版）



Veeam Backup＆Replication v8の新しいバージョンが先週リリースされました。この記事では、このバージョンに登場した新機能と、この新機能のうち製品の無料版で利用可能なものについて説明します。



NetAppハードウェアスナップショットのサポート



Veeam Backup＆Replication v8.0では、NetAppストレージシステムのハードウェアスナップショットに基づいて、仮想マシンのバックアップとレプリカを作成できます。 HP StoreServ（3PAR）、HP StoreVirtual（LeftHand / P4000）、HP StoreVirtual VSAなどのハイエンドストレージシステムの以前のバージョンでは、同様のアプローチが既に実装されていました。





ハードウェアストレージスナップショットの利点を思い出させてください。 ハードウェアディスクイメージの技術は、RPO（新しいデータまたは変更されたデータが障害から保護されない、つまりバックアップコピーに複製されない最大時間）を大幅に削減できる技術として長い間専門家によって指定されてきました。 このテクノロジーの本質は、ストレージシステムが独立して、調整可能な頻度で、ディスクのスナップショットを作成することです。 ストレージシステムの開発者は、他の誰よりも低いレベルでディスクの設計を熟知しているため、こうしたディスクのスナップショットを最大限の効率と生産性で、つまり生産システムへの影響を最小限に抑えて作成できることは明らかです。 したがって、 ハードウェアスナップショットはバックアップの優れたソースです。



同時に、ハードウェアスナップショット自体は、主にスナップショットが「予約」する同じストレージシステムにあるため、データを障害から保護するための信頼できる方法と見なすことはできません。 ストレージシステムで障害が発生した場合、元のデータとそのスナップショットの両方が破壊される可能性が高くなります。 障害からのデータ保護の適切な信頼性を確保するために、データのバックアップを元のストレージの場所とは異なる場所に移動する必要があります。



さらに、スナップショットには多くの欠点があります。

  • ハードウェアスナップショットは元のディスクの有効容量を使用し、その最大サイズの合計は、データ保護要件だけでなく、本稼働システムのビジネスアプリケーションの「カウンター」要件によっても決まります。 したがって、2番目の緊急タスクは、古いスナップショットをタイムリーに削除することです。 さらに、使用されるアルゴリズムの特性により、場合によっては古いスナップショットが多数存在するため、ストレージシステムにデータが存在する一部のアプリケーションの動作が遅くなったり、誤動作することさえあります。 Veeam Backup＆Replicationでは、データがバックアップコピーに移動されるまでストレージシステムに「スナップショットを保持」し、その後スナップショットが自動的に削除されるため、この問題を簡単に解決できます。
  • ハードウェアスナップショットからの回復手順はきめ細かくはありません （必要に応じて、ハードウェアイメージからディスクを復元した結果、ストレージに別のファイルを復元し、ストレージ全体が「過去に移動」し、ディスク上の最後の変更が失われます）
  • ハードウェアスナップショットは「クラッシュコンシステント」になることがあります。ストレージシステムにはApplication Awareバックアップに制限があり、メモリ内のアプリケーションバッファを最初にディスクにフラッシュせずにバックアップできるためです。データの性質（オペレーティングシステムの種類、ディスクの種類：仮想ディスクまたは物理ディスク、アプリケーションの種類など）に関する完全な情報が格納されています。


上記の結果として、バックアップ製品の機能と一般的なストレージシステムに組み込まれたハードウェアスナップショットの機能を組み合わせることは非常に合理的です。



NetAppサポートの詳細については、Habrの記事「Veeam Backup＆Replication v8でのNetAppストレージのハードウェアスナップショットのサポート」を参照してください。



EMC Data Domain Boostのサポート



Veeam Backup＆Replication v8とEMC Data Domain Boostの統合により、完全な合成バックアップの構築プロセス（ストレージ内のデータの完全なコピーを再構築するために新しい増分バックアップまたは差分バックアップがそれぞれすぐに使用され、リカバリ時間が大幅に短縮される）とバックアッププロセスが大幅に加速されました一般的なコピー。 これは、高度なEMC Data Domain Boost重複排除機能によって可能になりました。

  • まず、ネットワークを介してデータを送信する際に両側で重複排除テクノロジーを使用することにより、バックアッププロセスが加速されます（物理的に送信されるデータの量が大幅に削減されます）。
  • 第二に、ディスク上のデータは物理的にではなく論理的に移動するため、組み込みの重複排除を有効にしてストレージへの完全な合成バックアップを構築する方が高速です。


たとえば、テスト仮想マシンの完全な合成コピーを作成するために行ったある実験では、タスクを完了するのにかかった時間は、EMC Data Domain Boostから開始する場合は11.5分未満であり、重複排除をオフにしてタスクを開始する場合は約3時間でした。 速度の違いは約15倍でした！





もちろん、特定の各仮想インフラストラクチャはその特性が固有であり、特定の仮想マシンでの重複排除から得られる利益は、そのディスク上のデータ変更の性質と強度に大きく依存します。



また、合成バックアップよりも通常の完全バックアップを好む場合でも、ネットワークトラフィックを重複排除することで得られます。以前にストレージ側に転送されたデータブロックは、それ以降の完全コピーではネットワークを介して転送されません。 これは、次の2つの図に明確に示されています（最初の増分コピーを構築するときのネットワーク負荷を反映しています）。 左側の図では、トラフィックの重複排除がオン（EMC Data Domain Boost）で、右側がオフです。





どちらが良いですか：合成フルバックアップまたは定期フルバックアップ？


EMC Data Domainには優れた重複排除機能が組み込まれているため、複数のフルコピーを作成しても、異なるフルバックアップの重複ブロック（およびそのほとんど）が単一のコピーに格納されるため、ディスクスペースが過剰になりません。 同時に、フルバックアップを使用すると、多数の増分コピーから最新バージョンのデータを探す必要がないため、障害後のデータ（RTO）の回復時間を最小限に抑えることができます。



システムの最後の状態を復元するという点では、合成コピーと通常の完全コピーは同じです。 ただし、過去に長い期間ロールバックする必要がある場合があります（たとえば、破損したファイルがバックアップに入った場合やバックアップ自体が破損した場合）-そして、フルコピーチェーンオプションには、合成バージョンよりも明らかな利点があります。



したがって、「どちらが良いですか？」という質問に答えると、重複排除が有効になっているEMC Data Domainの場合、完全バックアップのチェーンを優先させることができます（もちろん、完全な通過以来、仮想インフラストラクチャの負荷に関して禁忌がない限り）増分よりもかなり多くのリソースを消費します）。

Veeam Cloud ConnectはVeeam Backup＆Replicationソリューションの新しいコンポーネントであり、ユーザーがSSLを迅速かつ安全に使用して、VPNに頼らずにサービスプロバイダーにデータを転送できるようにすることが期待されています。 サービスプロバイダーとVeeam Cloud Connectが提供する機会により、リモートサイトとバックアップを保存するための専用リポジトリを作成するための追加費用が不要になります。



新しいレプリケーション機能



Veeam Backup＆Replication v8では、次の新しいレプリケーション機能が利用できます。

  • レプリケーションプロセスでの WANアクセラレーション（以前はWANアクセラレーションはバックアップ作成および転送モードでのみ機能しましたが、レプリケーションプロセスでは使用されませんでした）。
  • レプリカへの切り替えの計画 -切り替えプロセス用のスクリプトを作成できます。つまり、切り替えに参加する仮想マシンをグループ化し、それらの起動順序を設定し、起動タイマーとその他のパラメーターを構成できます。
  • レプリカのスケジュールされた切り替え （計画、つまりスクリプトの作成と混同しないでください）-ソース仮想マシンをオフにし、レプリカをオンにする時間を設定できるようになりました。 たとえば、アフターサービスに便利です。


Habrの記事「Veeam Backup＆Replication v8.0の新機能：高度なレプリケーション」で、新しいレプリケーション機能の詳細を読むことができます。



自動読み取り/書き込み強度制御



Veeam Backup＆Replication v8.0は、読み取り/書き込み操作の強度を自動制御する方法を実装しています。バックアップI / O制御。 バックアップはリソースを大量に消費するプロセスであり、生産的なシステムのリソースのかなりの部分を吸収できるため、ユーザーはそれを認識できるため、労働生産性が低下します。 したがって、v8バージョンでは、Veeam B＆Rは自己制限できます-データストアレイテンシ（IOPS）に有効な値を設定し、インフラストラクチャのパラメーターの最適な組み合わせを設定するだけで済みます。



自動制御メカニズムについては、Habrの記事「Veeam Backup＆Replication v8.0の新機能：読み取り/書き込み操作の強度の自動制御」で詳しく説明されています。
