Java SE 9の一部としてJVM（正確にはOracle HotSpot）に導入されたパフォーマンス改善の1つは、コンパクトな文字列であることが判明しました。 彼らの仕事は、文字列オブジェクトのサイズを小さくすることです。これにより、アプリケーションが消費するメモリの総量（フットプリント）を減らすことができます。 その結果、ガベージコレクションに費やす時間を削減できます。
