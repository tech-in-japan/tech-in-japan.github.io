NetWrix Active Directory Change Reporter 7.1がリリースされました

Active Directoryを使用している場合、おそらく監査プログラムであるNetWrix Active Directory Change Reporterに出会ったことでしょう。 このプログラムは、ADの変更を監視し、自動レポートを作成し、監査データを長期間保存し、削除または変更されたADオブジェクトを迅速かつ無損失で復元できます。 プログラムの機能に興味がある場合は、 レビューを読むことができます 。

月曜日に、世界中のユーザーによって課された要件を考慮して作成された新しいバージョン7.1のリリースを発表しました。 まず、新しいバージョンを作成するとき、大規模な組織でのプログラムの安定した運用に焦点を当てました。



バージョン7.1の新機能



  • Microsoft Exchange Server 2010のサポートの強化

    NetWrix Active Directory Change Reporterは、他のプログラム（NetWrix Exchange Change Reporter）と緊密に統合されており、Exchangeサーバーで発生する変更を監視できます。 バージョン7.1では、Exchange 2010の変更をより完全かつ明確にキャプチャできるように変更が加えられました。
  • 大規模なAD環境のサポートの改善

    分岐したActive Directory構造を持つ大規模な組織でのプログラムの安定した作業は、開発者が直面した別のタスクです。

    これらの改善は、プログラムの無料版にも含まれていることに注意してください。
  • NetWrix Non-owner Mailbox Access Reporter for Exchangeとの統合

    これで、プログラムから直接、外部メールボックスへのアクセスに関する情報を取得できます。NetWrixの非所有者メールボックスアクセスレポータfor Exchangeプログラムがこれを担当し、比較的最近機能の重要な更新を受け取りました。 したがって、AD + Exchangeバンドルの完全な制御が実現します。
  • 静的レポート（スナップショットレポート）にメインユーザーグループを表示する

    以前の投稿で書いた問題は解決されました（ADグループhabrahabr.ru/company/netwrix/blog/148472のユーザーメンバーシップに関するレポートを作成しています）
  • レポートフィルターでのアジア言語のサポート
  • 新しいAD変更レポート
  • SQLデータベースで監査データの保持期間を構成する機能
そしてもちろん、新しいバージョンの修正やその他の改善が行われました！
