Googleドライブ-すべてのベースは私たちのものです

昨日、Googleは新しいGoogleドライブサービスを発表しましたが、これは多くの相反する意見を引き起こしました-一方では、企業は長い間オンラインストレージおよび同期/バックアップサービスを待っていましたが、一方で、有料ストレージの価格の上昇により、ドライブは「別のクラウドストレージ」になりました。 ただし、これについては説明しませんが、奇妙な使用条件については説明します。これは、Googleドライブを含むすべてのGoogleサービスで同じです。



現在の使用条件の版には非常に興味深い段落があります

サービスに資料をアップロードまたは追加することにより、 Googleおよびそのパートナーに、このコンテンツの使用、投稿、保存、複製、変更、それに基づく派生物の作成（翻訳、素材の最適化の適応やその他の方法）、共有、公開、公然と再現、表示、配布も可能です。 お客様が当社に提供するリストされた権利は、既存のサービスの運用、そのプロモーションおよび改善、ならびに新しいサービスの開発を保証するためにのみ使用されます。 指定されたライセンスは、サービスの使用を拒否した後も有効になります（たとえば、Googleマップに追加した会社データを引き続き使用します）。 一部のサービスでは、追加されたコンテンツを表示および削除できます。 それらの一部には、そのような素材の使用に制限を課す条件と設定もあります。 コンテンツを当社に送信し、それを使用するためのライセンスを付与する前に、適切な権利を持っていることを確認してください。

それにもかかわらず、Googleはユーザーの知的財産のふりをしない予約を行います

同時に、これらの資料に関連するすべての知的財産権は所有者に帰属します。

既存の文言は非常に幅広い解釈の余地を残しています-一方では、Googleはダウンロードされた文書などの作者を主張していないようですが、それでもクリエイティブ・コモンズの帰属のない文化の自由な生産の非常に広い権利を持つこれらの製品を使用するライセンスを自動的に取得しますコピーレフト。 Googleはもちろん、これはサービスの品質を向上させるためだけに行われていると主張しているが、それがWi-Fiパスワードの話であったとしても。

比較のために、Microsoft SkyDrive、Dropbox、Ubuntu One、およびCrashplanの使用条件を以下に示します。



Microsoftは Googleからそれほど遠くない：

ライセンスを取得したコンテンツを除き、サービスで提供されるコンテンツの所有権は必要ありません。 あなたのコンテンツはあなたの財産のままです。 当社は、お客様または他者がサービスで提供するコンテンツを管理、検証、または承認しません。

一方で、すべてが素晴らしいですが、次の段落はGoogleと同じことを言っていますが、もっと簡潔に：

お客様は、Microsoftがサービスを提供するために必要な範囲でのみ、サービスに送信されたコンテンツを使用、変更、適合、複製、配布、および表示する必要があることを理解し、お客様はマイクロソフトにその権利を付与します。



Dropbox-巨人とは異なり、特定の許可が必要な理由と、どのフレームワークでそれらを使用するかを説明する、はるかに具体的です。

当社のサービスを使用することにより、Dropboxに送信する情報、ファイル、フォルダー（「一緒に」）を提供します。 あなたは自分のものに対する完全な所有権を保持します。 いかなる所有権も主張しません。 以下に説明するように、本規約は、サービスを実行するために必要な制限された権利を除き、お客様のものまたは知的財産に対するいかなる権利も当社に付与しません。



ファイルのホスティングや指示での共有など、ユーザーがユーザーに依頼することを行うには、ユーザーの許可が必要になる場合があります。 これには、画像のサムネイルやドキュメントのプレビューなど、ユーザーに表示される製品機能が含まれます。 また、サービスを技術的に管理するために行う設計上の選択（たとえば、データを安全に保つためにデータを冗長にバックアップする方法など）も含まれます。 サービスを提供するためだけにこれらのことを行うために必要な許可を与えてください。 この許可は、ストレージスペースを提供するAmazonなどのサービスを提供するために協力する信頼できるサードパーティにも拡張されます（これもサービスを提供するためだけです）。

簡単に言えば、著作権を侵害する人はいません。共有サービスの提供、サムネイル、アイコン、ドキュメントプレビューの作成、およびサービスをホストしたAmazonに同じ権限が付与されるためには、許可が必要です。



Ubuntu Oneは、法律または裁判所の命令により要求された場合にのみデータが開示される可能性があると報告しています。

Canonicalは、適用法または裁判所、行政機関、その他の政府機関の命令または要件を遵守する必要がある場合、送信、投稿、または公開した個人データおよびコンテンツの一部またはすべてを開示する場合があります。 個人データのその他の使用はすべて、Ubuntu Oneプライバシーポリシーの対象となります

サービスによって収集される残りのデータは、システムのバージョン、クライアントの使用頻度（GPL v3で配布されます）、インターネット接続の品質など、個人化されていません。 Canonicalは、会社が保存しているすべてのデータのコピーを取得する機会を提供します。



Crashplanは、プライバシーポリシーに次の事項を記載しています。

Code 42は、これらのタイプの個人データを、製品またはサービスの配信または提供を含むビジネス上の目的で使用する場合があります。 クライアントとビジネスの関係を確立または維持するため。 インターネットベースおよび電子商取引活動へのアクセスを提供するため。 会計機能を実行します。 また、ビジネス関係の整備と発展に関連して、必要または適切な他の活動を実施する。

これは、名前/姓、メールアドレス、住所などに適用されます。 プライベートパスワードを使用する場合、Crashplan自体は、暗号化のためにユーザーがサーバーに保存するものを認識しません。 このように、彼女はLaCie Wualaに似ています。



私自身はU1が好きですが、Crashplanは安価な無制限のストレージと、SolarisやWindows Phoneを含むさまざまなOSのクライアントのためにバックアップに適しています。



したがって、Google（および大部分はMicrosoft）に対して非常に明確な主張が生じます。

a）。 ドライブの利用規約を別のドキュメントに入れます

b） そのような拡張された権利に関する表現を削除する

c）。 権限の各条項を解読-ユーザーの素材を使用する特定のケースを強調し、他のすべてを違法として認識します。
