Firefoxを保存





Electronic Frontier Foundationを代表してCorey Doctorowが投稿しました。 このアピールは主にアメリカのインターネットユーザーに向けられていますが、私たちの共通の運命が決定されているため、ユーザーだけでなくすべての人にとって重要です。 無料のウェブの未来が決定されています。



むかしむかし、ほとんどすべての人が使用していた2つのブラウザがありました。NetscapeとInternet Explorerは、インターネットの未来のために致命的な戦いに巻き込まれました。 ユーザーがフォローすることを期待して、Webパブリッシャーが各ブラウザーのサイトを最適化するように説得するために、両者は大きく異なりました。



その後、ターニングポイントが発生しました。オープンで非営利のMozillaブラウザーがNetscapeから際立っており、パブリッシャーではなくユーザー自身にサービスを提供するという別のタスクを設定しました。 Mozillaはデフォルトで、初期のインターネットの惨事であるポップアップ広告をブロックしました。 これは、主要なブラウザのどれにも余裕のないステップでした。パブリッシャーは、こうした広告なしでは生き残れないと確信しており、組み込みのブロック機能を備えたブラウザーはパブリッシャーを自分自身から遠ざけることになり、競争で多くのことを意味していました。



10年を少し過ぎて、ブラウザーの世界は認識を超えて変化しました。MozillaはFirefoxに変わりました。 Internet ExplorerはEdgeに変わり、AppleはSafariをリリースし、GoogleはChromeをリリースしました。 それらはすべてデフォルトでポップアップをブロックしました！



そして、今は競争がないとは言えません。 彼女は、かつてないほど強くなっています。 これまでと同様に、これは、関心が必ずしも一致しない出版社とユーザーの両方を満足させるための戦略的闘争です。 出版社はユーザーに関するより多くの情報を収集し、人々は自分のデータを秘密にしたいと考えています。 パブリッシャーは、ユーザーによるコンテンツの流れとサイトの表示を制御するよう努めており、人々は自分自身を制御したいと考えています。



競争が必要です。 また、多様性が必要です。 革新的なアイデアを持つ初心者プレイヤーが市場で道を開くことを保証する必要があります。 アイデアの炎を保ち、すべてのブラウザがユーザーの世話からインターネットを支配するいくつかの巨大企業の利益に焦点を移さないようにすることが必要です。 常にそのようなプレッシャーがあり、すべてのブラウザーが同じゲームをプレイする場合、ユーザーは常に負けるでしょう。



さらに多くのfirefoxersが必要です。



開発者がパブリッシャーではなく主にユーザーについて考えるブラウザーをもっと必要としています。 これは、ほぼ20年にわたってインターネット上のエネルギーをサポートしてきた、自然な濃度-破壊-更新のサイクルです（全時代 、Webの標準による）。



World Wide Web Consortium（W3C）は、公開者が特定のブラウザーの独自機能に縛られるのを防ぐオープンスタンダードを促進するために使用されていましたが、W3Cはその使命を変更しました。 2013年以来、この組織は、メディア業界を支配する主要なブラウザー開発者や企業が集まるフォーラムに変わりました。 それらは共同でシステムを作成し、ブラウザが私たちの行動を制御できるようにします。



Encrypted Media Extensions （EME）と呼ばれるシステムは、標準で設定されたコードを使用して、独自のコンテナであるContent Decryption Module （CDM）内にビデオを配置します。 新しいブラウザがビデオストリーミングのこの新しい標準をサポートするには（大規模なスタジオやケーブルオペレーターによって推進されています）、これらのメディア企業またはパートナーのいずれかがCDMを提供するように説得する必要があります。



これは、以前のすべてのW3C標準およびオープンWebの全体的な考え方に反します。 以前は、サーバーが送信したコンテンツをレンダリングするための唯一の要件は、許可ではなく標準への準拠でした。 ブラウザが、最初のMozillaが登場したときにページをレンダリングするためにパブリッシャーの許可を必要とした場合、パブリッシャーは単にこのポップアップブロックポップアップ広告をつぶすことになります。 つまり、Firefoxに別れを告げることができます。



W3Cはこれを行う必要がありませんでした。 知的財産の保護に関するこのような法律はありません。これは、ビデオコンテンツの製造者が、ユーザーがこのコンテンツに合法的にアクセスした後、自分の機器をどのように設定するかを決定する権利を持っていると述べています。 ただし、EMEメカニズムにより、著作権者は法律を使用して、許可なく動画を表示しようとするブラウザを閉じることができます。



これは、EMEシステムがデジタルミレニアム著作権法 （DMCA）のセクション1201を施行するように設計されているためです。 彼らは、たとえあなたがそのコンテンツにアクセスする権利を持っているとしても、著作権で保護されたコンテンツを保護するデジタルロックを許可なく削除することは犯罪だと言います。 言い換えれば、ビデオがEMEシステムを通じて放送され始めるとすぐに、ユーザーが違法行為をしていなくても、ユーザーのコンテンツのロックを解除し始めた会社は法廷で責任を問われる可能性があります。



新しいブラウザを保護するという方法でW3Cを提供しました。 W3Cメンバーは 、DMCAを使用して新規参入者を攻撃しないことを約束する必要があります。W3Cメンバーの大規模なグループはこの考えを支持しましたが、W3Cコンソーシアム管理者は提案を拒否し、将来の競争を保護することなく標準の作業を続けることを決定しました。



DMCAは米国でのみ有効ではないため、これは一見するとさらに悪いことです。米国通商代表部の主導により、DMCAのような法律が米国と貿易関係にあるほとんどすべての国で採用されました。 さらに悪いことに、企業はDMCAを絶えず使用して、製品の重大な欠陥を発見した情報セキュリティ研究者を脅し、黙らせています。 W3Cコンソーシアムは、EMEの脆弱性を発見したセキュリティ研究者の保護をメンバーに要求することも拒否しました。 これにより、コンピューターシステムに未解決のバグが残るため、すべてのインターネットユーザーが危険にさらされます。 会社の許可を得て脆弱性を開示すると、セキュリティが向上するだけです。



W3Cコンソーシアムは、オープンWebとそれを実行可能に保つ革新を気にする人々によって主導されるべきです。 彼らはこの種の批判に敏感です。 私たちはそれらを理解しています。 多くの善良な人々がここで働いており、ウェブがすべての人に開かれ、ユーザーにとって安全であることを心から熱望しています。 しかし、組織はEMEを採用することでひどい決定を下し、メンバーの意見を拒否し、情報セキュリティ研究者と新しい競合他社の保護を取り除いた場合はさらにひどいものになりました。
