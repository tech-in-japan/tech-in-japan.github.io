Smart EnginesでのSailfish OS認識の教え方







みなさんこんにちは！ 私たちの記事ですでにご存知のように、Smart Enginesでは認識に取り組んでおり、あらゆる条件での認識に努めています。 iOS、Android、Windows、Linux、MacOS、Solarisなどの一般的なオペレーティングシステムをすべてサポートしています。 国内メーカーのElbrusおよびAstraLinuxをサポートしています。 当社のアルゴリズムは、ARMv7-v8、AArch64、x86、x86_64、SPARC、E2K、MIPS向けに最適化されています。







したがって、ロシアのオペレーティングシステムSailfish Mobile OS RUSの人気の高まりを見たとき、それを無視することはできませんでした。 Sailfish Mobile OS RUSは、企業ユーザーや政府機関の問題を解決するために国内企業のOpen Mobile Platformによって開発されたモバイルデバイス用のPOSIX互換オペレーティングシステムです。 2018年2月現在、ロシアのソフトウェアレジストリに含まれている唯一のモバイルオペレーティングシステムであり、AK1 / KS1クラスのFSB認定に合格しています。







この記事では、 Smart IDReader認識ライブラリ（ Hieroglyphテクノロジー）をSailfish OSに移植した経験について説明します。 コード、リンク、ビデオが含まれます。 この記事は技術的に有益であり、C ++アプリケーションをSailfish OSに移植する人のための一般的なガイドとして有用です。









Open Mobile Platform （OMP）の同僚であるKirill ChuvilinとAlexey Andreevが、アプリケーションの開発と、テストとデモンストレーション用の2つのINOI R7デバイスの提供に関する相談に感謝しています。







TL; DR-移植は迅速かつ簡単でした。 合計で、Sailfish OS SDKがダウンロードされてから最初のビデオが撮影されてからINOI R7デバイス上のドキュメントが認識されるまで、作業に1週間もかかりませんでした（ビデオは記事の最後にあります）。







ツールキットのインストール



最初のステップは、Sailfish OS SDKをインストールすることでした。 公式サイトからインストールパッケージをダウンロードできますが、さらに簡単な方法を選択し、AURからSDKをインストールしました-Arch Linuxユーザーリポジトリ（Smart Engine開発者の間で最も人気のあるLinuxディストリビューション）で、いつものように、すべてが1つのコマンドでインストールされます： 







yaourt -S sailfishos-sdk-bin


開発ツールには、次の3つの主要コンポーネントが含まれます。







  1. 環境およびビルドシステム-デバイスまたはエミュレータ用のアプリケーションをビルドするためのインストール済みコンパイラおよびライブラリを含むOracle VM VirtualBoxイメージ。
  2. SailfishOSエミュレーター、他のモバイルプラットフォーム（iOS、Androidなど）のエミュレーターに似ています
  3. IDE-ビルドシステムとの対話をサポートし、デバイスまたはエミュレーターでプログラムを実行するために完成した、よく知られたQt Creator。


ビルド仮想マシンへの接続はSSH経由です。 唯一の問題は、仮想マシンとIDEにSDKをインストールするときに、開発者のコ​​ンピューターで既に他の目的に使用されていたポート2222



がハードコードされていることです。 VirtualBoxでは、このポートはわかりやすい方法で変更されますが、IDEではGUIから変更することはできませんでした。 幸いなことに、 mersdk.xml



ローカル設定ファイルで変更できます。 







 <!-- ~/.config/SailfishOS-SDK/qtcreator/mersdk.xml --> <value type="QString" key="SshPort">2222</value>


この問題を解決した後、Cyrilに報告されましたが、今後ツールにその他の問題はありませんでした。







C ++認識コアを構築する



私たちの認識エンジンはC ++で書かれており、この場合、Sailfish OSに移植するための明らかな選択は、静的ライブラリの形でカーネルを構築することでした。







ライブラリを手動でビルドする方法に関する短いが有益なマニュアルは、公式ドキュメントのSailfish OSパッケージを手動でビルドするページにあります。 それから、SSHを介して仮想マシンに接続し、正しいアーキテクチャで通常のビルドコマンドを呼び出すだけでよいことが明らかになりました。 プロセスを段階的に説明します。







  1. Sailfish OS Build Engine仮想マシンを起動します（十分なヘッドレスモード）

  2. 私たちはそれに接続しています： 







     ssh -p 2222 -i /opt/SailfishOS/SDK/vmshare/ssh/private_keys/engine/mersdk mersdk@localhost


    ちなみに、外部ユーザーのホームディレクトリは、デフォルトでは/home/src1



    仮想マシンで使用できます。







  3. サポートされているプラ​​ットフォームのリストを取得します。 







     [mersdk@SailfishSDK ~]$ sdk-assistant list Toolings: SailfishOS-2.1.3.7 Targets: SailfishOS-2.1.3.7-armv7hl #  SailfishOS-2.1.3.7-i486 #


    サイトの例とは異なり、プラットフォーム名にはバージョン番号が含まれている場合があり、これも考慮する必要があります。 さらに、デバイス（SailfishOS-2.1.3.7-armv7hl）のアセンブリが実行されますが、エミュレーターの場合もプロセス全体は同様です。







    必要なアーキテクチャの場合、 sb2



    （Scratchbox2）コマンドによって適切な環境が提供されるため、すべてを実行する必要があります。 







     sb2 -t SailfishOS-2.1.3.7-armv7hl <args....>


  4. アセンブリに必要なパッケージをインストールします。 私たちの場合、これらはCMake、GCC、G ++、およびZipです。 







     sb2 -t SailfishOS-2.1.3.7-armv7hl -m sdk-install -R zypper in cmake gcc gcc-c++ zip


  5. ビルドフォルダーを作成してCMake



    を呼び出し、正しいCMAKE_SYSROOT



    て、コンパイラーが適切な場所で検索されるようにします。 







     mkdir build && cd build sb2 -t SailfishOS-2.1.3.7-armv7hl -m sdk-build cmake -DCMAKE_SYSROOT=/srv/mer/targets/SailfishOS-2.1.3.7-armv7hl ..


  6. プロジェクトをまとめる： 







     sb2 -t SailfishOS-2.1.3.7-armv7hl -m sdk-build make install -j8




以上です！ したがって、Sailfish OSのアセンブリは、実際にはLinuxのアセンブリと変わりません。仮想マシンに接続し、正しい環境でコマンドを呼び出すだけです。

カメラAPIと適切に対話する方法を理解することは、デモアプリケーションの開発プロセス全体で唯一の困難であり、別の点に値します。







通常、リアルタイムビデオストリーム認識ライブラリを使用するシナリオは次のとおりです。ユーザーがカメラのプレビューを見て、その前にオブジェクト（たとえば、RFパスポートまたは銀行カード）を保持している間、カメラからのフレームは1つずつ認識ライブラリに入り、認識されます。 同時に、各フレームの認識中に、ドキュメントの場所、そのフィールドなどに関する中間情報がユーザーに表示されます。 したがって、カメラからフレームをキャプチャすると同時に、ユーザーにプレビューを表示する必要があります。
