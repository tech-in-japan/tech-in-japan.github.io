タブストレージ



そこで、ブラウザで開いたタブを特定のリストに保存することを決定しました。このリストは、コンピューター-ホームコンピューター、職場コンピューター、ローカルコンピューターと呼ばれます。 ローカルとは、リストがサーバー上ではなくブラウザーデータベースに保存されている場合であり、したがって、リストの保存元のブラウザーでのみ使用できます。 これは、異なるコンピューター間でスローする必要のないタブ用です。 まあ、残りのリストは、拡張機能でサポートされているものの中から任意のコンピューターとブラウザーで表示できます。



各タブについて、ユーザーは任意のテキストコメントを残すことができます。 たとえば、シリーズを見ているときに停止したシリーズを覚えておくと便利です。また、タイトルやリンクの種類が明確でない場合は、ページの内容を自分でマークしておくと便利です。



ChromeとFirefoxの拡張機能の作成に関するドキュメントによると、拡張機能コードはメインとバックグラウンドの2つのファイルに配置する必要があります。 メインスクリプトはユーザーインターフェイスの表示を処理し、バックグラウンドスクリプトはブラウザーのタブを制御し、サーバーとデータを交換できます。 ChromeとFirefoxのタブの管理は非常に似ています。 たとえば、Chromeでタブを開いたり閉じたりするためのイベントハンドラーは次のようになります。 



chrome.tabs.onUpdated.addListener( function(tabId, changeInfo) { });


そして、Firefoxでは次のようになります。 



 browser.tabs.onUpdated.addListener( function(tabId, changeInfo, tabInfo) { });


そして、開いているすべてのタブのリストを表示する関数は、両方のブラウザーで同じように記述されています。 



 chrome.tabs.query({},function(data){ });


はい、はい、クロムはFirefoxでも書かれています...など これは少し奇妙ですが、ブラウザがAPIの標準化に向かっているという事実は非常に勇気づけられます。



実際には、拡張機能のすべての機能は、いくつかの機能を使用して実現されます-アイコンをクリックして拡張ウィンドウを開き、開いているタブを読み取り、タブを閉じ、新しいタブを作成し、タブイベントをリッスンし、拡張ウィンドウとデータを交換します。 それだけです これらのメソッドは、バックグラウンドスクリプトからのみ使用できます。



読み取り後、ブラウザーで開かれたタブはサーバーに送信され、通常のSQLデータベースに保存されます。 サーバースクリプトはPHPで記述されています。 データテーブルという別のテーブルの行へのリンクを格納するユーザーテーブルがあります。 このデータは、ページのアドレス、ページの名前、ユーザーのテキストコメントであり、約1キロバイト長のテキストタイプフィールドにテキスト文字列の形式で保存されます。 データは暗号化されていない形式で保存されます。 1メガバイトあたり約1,000個のタブを保存できます。 ユーザーごとに最大25個のストレージ（「コンピューター」）が提供されます。



ただし、拡張機能の無料バージョンでは、ユーザーは2つのストレージ（ホームコンピューターと仕事用コンピューター）に制限され、各ストレージは10個までのタブを保存できます。 私はまだ有料版と支払い方法に取り組んでいます。 最大25台の「コンピューター」とそれぞれ約1000個のタブがあり、背景画像、各コンピューターのアイコンと名前、階層リストなどが選択できます。



別のリポジトリ-ローカルコンピュータがあります。 タブデータはユーザーのコンピューターのローカルブラウザーデータベースに保存されるため、ここでは制限はありません。 ここには、好きなだけタブを保存できますが、ハードドライブが許可する以上のタブは保存できません。 さらに、データはローカルストレージではなくindexedDBに格納されるため、5または数メガバイトの制限はありません。 重要なことは、ブラウザの履歴とともにこのリポジトリを誤って消去しないことです。 また、タブをテキストファイルにエクスポートおよびインポートする予定です。



indexedDBを使用するために、長さが106行しかない小さなサブライブラリを作成しました。 サーバー上のデータベースとは異なり、ここではすべてのデータをテキスト文字列に抽出する必要はありませんが、連想配列にそのまま保存できます。 そこにあるストレージの原理はこれです：キーは価値です。 また、値は巨大な連想配列になる場合があります。

タイルの色は、サイトのドメイン名に基づいて自動的に選択されます。 この関数はURLを受け入れます。 ドメイン名は3つのほぼ等しい文字グループに分割され、各グループの最初の文字は色成分の値R、G、Bに変換されます。この変換は次のように実行されます。 これらの3文字のコードは、26による除算の残りを取得することにより、0〜25の範囲d1に縮小されます。ロシア語のアルファベットの文字もこの範囲に縮小されます。 27番目の文字が最初に来ることは問題ではありません。アルファベットのすべての文字の色成分が必ずしも異なる必要はないので、繰り返してみましょう。 次に、範囲d1（0 ... 25）から得られた3つの数値は、範囲d2（150 ... 255）に比例して減少します。 一般に、これらの範囲は関数の開始時に設定され、他の範囲に変更できます。 色成分が明るくなるように150〜255を取りました。 たとえば、0 ... 100に設定すると、タイルは暗くなります。 したがって、ドメイン名の3文字は、150〜255の範囲の3つの数字に変換されます。 そして、ご想像のとおり、これらはタイルR、G、Bの色成分として解釈されます。すべてが非常に単純です。 



 getSiteColor=function(url) { var d1=[0,25], d2=[150,255]; var code=0, codep=0, color=0, str='', domen='', ar=[], inc=0; ar=url.split('//'); if (ar.length>1) {str=ar[1];} else {str=ar[0];}; str=str.split('www.').join(''); domen=str.split('/')[0]; str=domen.split('.').join(''); ar=[]; inc=Math.floor(str.length/3); if (str.length % 3 >0) {inc++;}; for (var i=0; i<str.length; i+=inc) { code=str.slice(i, i+1).charCodeAt(0); codep=code % (d1[1]-d1[0]+1); color=Math.round( codep * (d2[1]-d2[0]) / (d1[1]-d1[0]) ) + d2[0]; ar.push(color); }; return {r:ar[0], g:ar[1], b:ar[2], domen:domen}; }

また、アセンブリをOpera拡張ストアに送信しました。 しかし、2つのバージョンを配置し、それらのいずれかのモデレーションを待たずに、ある時点で、そこに新しいバージョンを広めようとしました。 実際、OperaがChromeストアから拡張機能を完全にインストールする場合、なぜこれが必要なのですか。









まとめ



Chrome、Firefox、Webバージョンの3つの拡張ビルドを作成しました。 実際、3つのアセンブリはすべて同じアセンブリであり、現在使用されているブラウザエンジンを構成内で単に示しています。 そして、タブを操作するための対応する関数が初期化されます。 この値を変更してから、zipにパックしてパッケージを適切な拡張ストアに送信します。



ChromeとFirefoxのバージョンはテストされ、それぞれの公式拡張ストアに配置されています。 さらに、Googleでの拡張機能開発者としての登録には5ドルかかります。 Mozillaは無料で提供しています。 Webバージョンの拡張機能はサーバーにあります。 Chromeストアのアセンブリは、互換性のあるブラウザーで正常に機能します。YandexブラウザーとOperaにもインストールできます。 さらに高度な機能を備えた新しいバージョンをリリースする予定です。



一般的に、ブラウザの拡張機能は強力です。 今、私はソファから立ち上がる必要はありませんし、タブレットを壊す怒りも必要ありません。 これは完全な禅です。 すべてに良い。
