ZeptoLab Code Rush 2015が間もなく終了

Hawkraversにこんにちは！



2014年、 Codeforcesとの最初の共同スポーツプログラミングコンテストを開催しました 。



簡単に説明します：



コンテストは6つのタスクで構成され、2.5時間がソリューションに割り当てられました（過去1年間のタスクに精通し、 ここで問題を解決することもできます ）。

もちろん、純粋に開発イベントでさえ、私たちは自分自身に忠実であり続けたので、すべてのタスクはゲームに基づいて考えられ、もちろん、私たちはそれらを注意深く説明しました：







Codeforceの歴史で初めて、2148人以上が同時にコンテストに参加しました（4600人（！）が世界中から登録されました。ちなみに、最初の3つの場所は ロシアの開発者は、喜ぶしかない。 彼らの何人かは、ミニツアーとプログラムのハイライトを待っているオフィスに賞品を受け取るために来ました：もちろん、巨大なカット・ザ・ロープと私たちの標準的な企業の「造園」を入り口で演奏します企業の緑色の面白いギズモ）。







さらに幸運な人もいました。彼らは私たちと一緒に遠足をするよりも少し長く滞在し、仕事の成功で私たちを喜ばせました。 以下は、そのうちの1人であるグリシャN.（ WhiteCrow ）の考えです。







「最初はキャンディー（冗談）に来ましたが、今、印象を共有するなら、最初に重要なことはここで最後まで自分自身になれるということです。 さらに、Zeptolabでは、マネージャーは、私の目の前を含め、各開発者の能力を考慮してタスクを設定しました。 そして、リーダーがアルゴリズムの「対象」でもあるとき、私はどこを好転させるかを持っています。 私は実際、私が望んでいたクライアント側のバックエンド開発者のようなものになることが判明しました。 はい、Zeptolabで役立つとは思われないデータ構造をいくつか知っています（たとえば、検索エンジンの開発などで役立つでしょう）。 私は、日常の仕事で知識のこの部分に実質的には向きません。 しかし、これらのスキルは、内部のアルゴリズムコンテストで役立ちます:)作業の瞬間について：私の最後のタスクは、ゲームリソースの準備におけるアトラスのパッケージ化を改善することでした-よく知られたNP困難なタスクです。 私は大きな成功を収めることができました。さまざまな指標に従って、2013年のよく知られた最高のアルゴリズムを改善することです。 その結果、すべてのゲームプロジェクトのメモリ消費量がメガバイト減少し、アセンブリにはメモリ制限が確保されました。 アーティストの意欲で、これは長くはないと思います:)おそらく、今後のカンファレンスのいずれかで私の作品を公開するでしょう。 一般に、スポーツプログラミングスキルはここで高く評価され、私の経験では、他の企業はこれを十分に重視していません。



開発者の皆さん、この投稿の目的は次のとおりです。



4月4日（土）に、 Codeforceと一緒に2回目のコンテストを開始します。



私たちは、あなたを再び重要でない仕事で再び驚かし、前回のコンテストよりも素晴らしい賞品で私たちを喜ばせるために、すでに一生懸命働いています：







ちなみに、コンテストに参加する以外は、このようなベストを手に入れることはできません。 ベストはクールです。





さて、以前のように：競争で良い結果を示した人は、単純化されたスキームに従って私たちに到達する機会があります。 zeptoteam.ruで-ZeptoTeamの一員であることの意味について。 私たちは自分自身で提案に接触します。



チャンピオンシップは1ラウンドで開催されます。 競争の形式は、 Codeforcesルールに準拠しています 。
