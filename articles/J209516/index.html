Netavis Observer-LinuxベースのIP監視ソフトウェア。 インストール、設定、少しハック

Netavis Observerは、Linuxサーバーに基づくIPカメラを使用してビデオ監視を整理するためのソフトウェア製品です。



Netavis ObserverはJavaで記述されており、Apache Tomcat、Webmin、Munin、Mysqlなどの製品も使用されています。



Netavis Observerには2つのオプションがあります-バンドル（Centos5またはCentos6に基づくインストールisoイメージとして提供）およびアンバンドル-CentosまたはRedHatを実行しているサーバーに自己インストールするためのrpmパッケージのセット。

主な機能と特徴：
-AXIS、ArecontVision、D-link、ACTi、Sanyo、Samsungなどの一般的なIPカメラおよびビデオサーバーのサポート（完全なリストはNetavis Webサイトにあります）。

-カメラへのアクセス権の分離、アーカイブの表示、ビデオのエクスポート、特定の状況のグループとユーザーを作成する機能を備えたPTZ（PTZ制御）のサポート（オンラインビデオの表示、オンラインビデオとアーカイブの表示、ただしビデオのエクスポート権限なし）アーカイブなど）

-Windows、iPhone / iPad用のクライアントの可用性。

-ブラウザを介して作業する機能（JAVAのインストールが必要）。

-多数のカメラで動作するクラスターを作成する機能。

-Active Directoryのサポート（エンタープライズによって修正された）。

-ビデオ分析のサポート（iCAT-変動を伴うモーション検出、妨害行為検出、リモートオブジェクト検出、プライバシーマスク）。

-ビデオ表示の品質をオンラインで設定し、アーカイブに記録するとき。

-他のユーザーにこれらのセットをコピーする機能を使用して、カメラの場所とユーザーセットの形成を設定します。

-ライブビデオとアーカイブを視聴するときに、カメラとクライアントの帯域幅を制限する機能。

-アーカイブをAVIまたはSAFEにエクスポートします（実行可能な.EXEファイルはプリミティブプレーヤーとビデオ自体で作成されます。一度に必要な数のカメラをエクスポートできます。1つのEXE-shnikが作成されます）。

-サーバーで直接グラフィカルシェルを実行する機能-LXDEセッションが起動し、その後Netavisクライアントが起動します。

ある時点で、サーバーにインストールされたハードドライブでは、1か月間、各カメラのアーカイブを継続的に記録するには不十分であることが判明しました。

必要な容量のハードドライブを購入し、ドキュメントを読んだ後、システムにハードドライブを追加および交換するプロセスが明確になりました。これを行うには、クライアントインターフェイスにアクセスするだけでは不十分です。システム管理コンソールにアクセスする必要があります。ドキュメントによると、 adminとしてサーバーにログインしてインストールする必要がありますパスワードで彼のために。



これは、コマンドssh -l admin 192.168.0.5を使用して実行でき、パスワードを入力するか、コンソールにローカルで単純にログインすることもできます。







ドキュメントによると、追加手順は3ルーブルと同じくらい簡単です。サーバーの電源を切り、未フォーマットの未使用のハードドライブを接続し、サーバーの電源を入れてシステムにログインします。

次に、27を入力してハードドライブが検出されたかどうかを確認し、23を入力して、ファイルシステムの作成、マウントポイントの作成などの手順の終了を待ちます。

サーバーを再起動してシステムを起動すると、新しいハードディスクがシステムに表示され、アーカイブを書き込む準備が整います。

また、ハードドライブの1つが故障して交換する必要がある場合は、別の手順に従います。サーバーの電源を切り、故障したハードドライブを機能するものに変更し、サーバーの電源を入れてコンソールにログインします。最初にハードドライブのステータスを確認し、24を入力します- 交換したディスクを構成し 、手順の終了を待ちます。



重要：新しいハードドライブはフォーマットされていない必要があり、故障したハードドライブがあった場所と同じ場所に接続する必要があります。

2台のハードドライブを追加し、1台の小容量のドライブを削除する必要があった場合、サーバーは既に実際に運用されていたため、ハードドライブの追加とハードドライブの交換という2つの手順を組み合わせました。

システムを読み込んでコンソールにログインした後、ハードドライブを別のハードドライブと交換するのではなく、ハードドライブを追加する手順から始めることにしました。システムは2つの新しいハードドライブをシステムに追加し、ハードドライブを変更するものは何もありませんでした。

このような急いでのアクションの結果は、プロセッサ負荷の横のクライアントインターフェイスに表示されるエラーでした-記録の劣化、および交換されたハードドライブにギガバイトがあったのと同じくらいシステムが使用できるスペースの誤った値。 同時に、コンソールでは、このハードドライブは未使用として表示されました-アイテム25-使用できないディスクを表示します。

管理コンソールでのアクション、以前の構成へのロールバック、インストールディスクからのシステムの復元試行は成功しませんでした。 この件に関するドキュメントは、システムをゼロから再配置するか、販売店に連絡することを推奨しています。
