Oracle Essbase-統合サービスの概要

Essbaseについては引き続き説明しますが、前回Essbaseの構成を整理した後、 インストール方法について説明しました。次に、既存のストレージアーキテクチャでの使用方法について説明します。



Essbase for DataWarehouseは、複雑な経済指標の計算、予測の作成、および可能な限り迅速なAD-hocレポートの作成を可能にする多次元ストアフロントの代わりになります。 Essbaseを使用すると、特別な設定をせずに、何千人ものユーザーに遅延なく対応できます。 クエリプラン、およびレポートのパフォーマンスを調整する方法は忘れます。











このようなショーケースを構築するには、Essbase Load Rules、Essbase Studio、 Integration Servicesのツールを使用できます 。 2番目の2つの製品は、最初の2つの製品との作業を自動化し、簡単なスクリプトの問題でエンタープライズの機能を追加するため、使用するのが望ましいです。 また、LoadRulesとは異なり、EISでは、ASOとBSOのMOLAPキューブに加えて、ROLAPとHOLAPの両方のアプリケーションを作成できます。







Essbase Integration Studioは、アーキテクチャ上、次の物理コンポーネントで構成されています



-Analytic Integration Serverは、Essbaseロードルールを形成するSQLコマンドの処理をオンザフライで処理するサービスです



-彼は、多次元ストアフロント（データベース）の設定が保存されているメタデータのリレーショナルソースが必要です）



-実際にはデータソース自体。



Essbase Integration Studioをインストールするときに、システムにODBCドライバーが追加されることに注意してください。これは、データソースとメタデータのセットアップ時に使用する必要があります。



-Analytic Integration Consoleは、「リレーショナル」ストレージから多次元モデルとOLAPキューブを作成する分析ワークステーションです。



-分析統合サービスシェルは、既存のメタデータからキューブの構築を自動化するための言語です。
