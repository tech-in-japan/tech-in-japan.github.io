Windows Updateを使いこなし、Windows 10 Reserveへの招待を受け取った方法

今日、私は1つの物語を語ります。それは、恐らく、この厄介な道で私が使ったすべての有用な資料の真髄でしょう。



遠い、遠い2014年、11月の寒い（ちなみに、それほど寒くない）月に、更新プログラムのインストール中に突然電源が切れた後、2010年の中生代の私のラップトップは、ハードスイッチをつかみました。 さて、私はそれが起こり、それをオンにし、何も起こらなかったように彼の仕事を続けたと思います。 しかし、翌日、OSの更新パックが毎週パックされた後、非常に奇妙なことに気付きました。次回のKBxxxxxxの再起動後のスケジュールされた再起動中に、システムが「更新を完了できません」と表示され、ログオン画面が表示される前に明確に通知されます。



最初は気にしませんでした スケジューラは通常、午前3時に軸を更新するため、更新手順の違反行為を登録することは非常に困難ですが、数か月後、更新の数が累積するにつれて、失敗したソフトウェアのボリュームが着実に増加し、システム設定の6か月の計画されたバイパスの一環として、更新センターに行きましたWindows、そしてそこにひどい数字を見ました-3ギガバイトもの新しい更新。 控えめに言っても、光景は印象的です。 それらの少なくとも1つをインストールしようとすると、ログはさまざまなエラーで私に投げかけました。たとえば、「更新マニフェストを読み取れません」、またはより複雑な「更新を完了できませんでした。」 変更のロールバック。 " 何日もインターネットをさまよった後、更新に関連するすべての問題を解決することができました。そして、何も明確ではない、何も機能しない場合、どうすればいいかを説明します。



手順1.マルウェアと不要なソフトウェアをスキャンする



最初のステップは、正常なコンピューターを更新しようとしていることを確認することです。 これを行うには、 「Microsoft Safety Scanner」というユーティリティがあります。これは、公式Webサイトからダウンロードできます。



このものの重量は約150MBで、感染を検出します。私の場合は4つのkeygensであり、必要に応じてユーザーの順序で削除します。



患者の健康を確認したら、ステップ2に進みます。



手順2. DISMシステムを使用してシステムファイルの整合性を復元する



コード0x80070490のエラーで苦しんでいる場合は、バトルフィールド（管理者に代わってcmd.exe）で昇格した特権でコマンドラインを表示します。 次の内容は、番号80070490で問題を解決するための記事「Windows Updateエラー0x80070490」に記載されています。



何時間も（そして私の場合はほぼ毎日）システムスキャンを完了し、その後の回復を行った後、次に進みます。



ステップ3.アクセスエラー



エラーコードが、エラー0x80070005によって引き起こされた問題の解決に関する信頼できる情報を提供しないことがよくあります。 しかし、幸いなことに、私の親愛なる読者の皆さん、私はこの厄介な誤解にうまく対処しました！ 問題の解決策は0x80070005です 。



ステップ4.管理者アカウントのステータスを確認する（オプション）



「net user Admin / active：yes」コマンドで特権管理者アカウントを無効化した後、何らかの理由でスーパーアドミニストレーターアカウントを無効にした場合、ウェルカム画面でのこのアカウントのアイコンの絶え間ない望ましくない表示に関連する不都合に対する解決策があります、選択したアカウントをログオン画面から非表示にすることで構成されます。 前述のコマンドが使い慣れておらず、管理者という名前のアイコンがようこそ画面に表示されない場合は、手順5に直接進むことができます。



コマンドプロンプトで、管理者に代わって、コマンドを実行します。 

net users


利用可能なすべてのアカウントの名前のリストを取得します。 このリストで管理者アカウントを探しています（英語、それぞれ管理者）。

次のコマンドを入力します。 

 net user


7行目はYesに対応している必要があります。そうでない場合は、コマンドでアカウントをアクティブにする必要があります。 

 net user  /active:yes


コマンドが成功したら、レジストリエディター（regedit.exe）を開きます。



セクション内： 

 HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon


UserListという名前のサブセクションを作成するSpecialAccountsという名前のサブセクションを作成する必要があります。 結果はパスになります： 

 HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon\SpecialAccounts\UserList


このセクションでは、タイプREG_DWORDの「Administrator」という名前のパラメーターを作成する必要があります。 パラメーター値を変更する必要はありません。 0-エントリを非表示、1-表示。 再起動後、非表示のアカウントはユーザー選択画面のアカウントのリストに表示されません。



ステップ5.まったく現れない



異なるバージョンのWindows（7、8、8.1）を実行している6台のマシンのうち、この問題はそのうちの2つでのみ確認されました。W7HPが搭載されたラップトップと同様のW8Proが搭載されたラップトップです。



簡単に言うと、次のとおりです。

ウイルス対策は、レジストリの変更を監視することを好み、そのようなイベントを登録すると、ユーザーにインシデントについてすぐに通知し、同じ変更を行う許可を求めます。 Update 1に続くセット全体のインストール中に、このダイアログボックスを300回以上見ました。

パスに沿ってレジストリに移動することにより： 

 HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\EventLog\Application


そして、このセクションの権限に完全な混乱が見つかったため、健全なコンピューターのレジストリに焦点を当ててすべてを戻しました。

セクションで同じ操作を実行する必要があります 

 HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\EventLog\System


ブレーンストーミングの後、アクセス許可を修正した後、ウイルス対策（私の場合はESET Nod32 Antivirus 8）を削除することに決めました。その後、手順3のエラーで宣誓した更新プログラムが正常にインストールされました。
