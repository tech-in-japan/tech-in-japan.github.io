I / O 2017のFirebase：新機能

Google I \ O 2017の参加者およびニジニノヴゴロドの アレクサンダーデニソフの GDGリードからのゲスト記事。



こんにちはHabr！ 最近、次のGoogleテクノロジーに関する国際会議-I / O 2017がカリフォルニア州マウンテンビューで開催されました。誰かがカリフォルニアに行き、誰かがGDGコミュニティの地域支部によって開催されたI / O Extendedに来て、自分で放送しましたが、誰かがまったく見ませんでした（念のため、ここに残しておきます： 2017年のすべてのI / Oセッションが記録されます ）。 今年、会議の良し悪し、意見は矛盾している、私は自分自身のためにしか言えない、本当に気に入った。







昨年、私はFirebaseチームのメンバーが何をしているかに非常に興味があり、それについての記事も書いたので、Firebaseエコシステムにどのような新しいものが登場し、どのように使用できるかをお話ししたいと思います。



ファブリック統合









Googleはしばらく前にTwitterからFabricを買収したと聞いていましたが、I / Oでは、FabricのFabricマネージャーRich ParetとFirebaseの共同設立者Andrew Leeのグループが同じステージで個人的に会い、チームが参加してからかいの見通しについて話し合いました。







それで、この合併の結果として何を得たのでしょうか？ 第一に、 クラッシュレポートの主要なソリューションとして 、 CrahslyticsがまもなくFirebaseに統合されます ！ 既にCrash ReportingコンソールでCrashlyticsサービスをインストールしてそのテストに参加するよう招待されており、まもなくFirebase Crash Reportingを完全に置き換えます。







次に、 Digitsサービスとの統合により、電話番号による認証が可能になりました ！ 1か月あたり1万件の認証を無料で実行できます。これは、開発者のニーズを完全にカバーするはずです。 また、独立製品としてのDigits SDKは、すぐに非推奨ステータスになります。







すぐに電話でアプリケーションに認証を固定し、それがいかに簡単かを示したかったのですが、判明したように、このサービスはiOSおよびWEB開発者のみが利用できるようになりました。Androidバージョンはさらに数週間待たなければなりません。



Firebase Cloud Functionsを使用したダイナミックホスティング



3月、Google NEXT'17で、Firebaseチームのメンバーは、Firebaseサービスの新しいCloud Functionsのローンチについて話しました。 このサービスは、Google JavaScriptインフラストラクチャで小さなJavaScript関数を直接ホストし、FirebaseサービスまたはHTTPリクエストによって引き起こされたイベントへの反応として実行する機能を提供します。



これにより、新しいユーザーごとにウェルカムメールを送信する（認証）、投稿やチャットの冒とくを追跡および削除する（リアルタイムデータベース）、ストレージにアップロードするときに画像を変換する（ストレージ）など、独自の機能を使用してFirebaseサービスの標準操作をカスタマイズまたは拡張できます。



試したことのない人のために、Cloud Functionsアプリケーションの例を示します。 GitHubからFriendlyChatのソースコードを取得します。そのアプリケーションは、昨年の記事で説明しました。 それをクリーンなFirebaseプロジェクトにバインドし、Cloud Functionを追加します。これにより、すべての新しいユーザーがチャットに自動的に歓迎されます。



最初のステップは、 Node.jsをインストールしてから、次のコマンドを使用してFirebase Command Line Interface（CLI）をインストールすることです。 



npm -g install firebase-tools


コマンドを使用してインターフェースにログインします 



 firebase login


プロジェクト用のフォルダーを作成し、そのフォルダーに移動して、コマンドでプロジェクト自体を作成します 



 firebase init functions


その結果、プロジェクト構造がフォルダーに作成され、関数フォルダーに移動してindex.jsを見つけます 。ここに関数が保存されます。 ファイルを開くと、その中に1行しかありません-const functions = require（ 'firebase-functions'） ; -これは実際にはFirebase SDK接続です。 そこにAdmin SDK接続を追加します。 



 const admin = require('firebase-admin'); admin.initializeApp(functions.config().firebase);


準備が完了しました。関数自体を実装しています。認証サービスで新しいユーザーが作成されるイベント、つまりユーザーが初めてチャットに参加するイベントで起動します。 



 exports.addWelcomeMessages = functions.auth.user().onCreate(event => { const user = event.data; console.log('A new user signed in for the first time.'); const fullName = user.displayName || 'Anonymous'; //              return admin.database().ref('messages').push({ name: 'Firebase Bot', photoUrl: 'https://image.ibb.co/b7A7Sa/firebase_logo.png', // Firebase logo     image hosting text: '${fullName} signed in for the first time! 関数をクラウドにアップロードするのは残り、それで終わりです。 



 firebase deploy --only functions


アプリケーションを実行でき、承認後、ボットはチャットで新しいユーザーを歓迎します。







しかし、これらはすべてI / Oの前に提示されました！ その上で、FirebaseはWeb開発者のコ​​ミュニティを喜ばせる斬新さを導入しました。 以前は静的コンテンツのみの操作を許可していたFirebase Hostingサービスは、Cloud Functionsとの統合により拡張され、Cloud Functionsサービスによって生成された動的コンテンツと連携するようになりました 。



プログレッシブWebアプリケーションに関与する開発者は、サーバーを完全に放棄し、Firebaseのみを使用できるようになります。

Firebaseパフォーマンスモニタリング



多くの場合、アプリケーションのパフォーマンスの低下や安定性がユーザーの不満の主な原因です。 ただし、アプリケーションの生産性や安定性が十分でない理由を理解するのは必ずしも容易ではありません。この困難なタスクを支援するために、Firebaseチームはベータテストで新しいパフォーマンス監視サービスを開始しました。 Firebase Performance Monitoringは、不正なコードや複雑なネットワーク状態がユーザーインターフェイスに影響を与える場合の理解に役立つ新しい無料のツールです。



サービスがアプリケーションのパフォーマンスに関するデータの収集を開始するには、Performance Monitoring SDKに接続する必要があります。その後、サービスはデータの収集をすぐに開始します。 このサービスは、いわゆるトレース（トレース）の形式でレポートを提供します-アプリケーションの2つのイベント間のパフォーマンスレポート。 接続するとすぐに、自動トレースのデータの収集が開始されます。



  • アプリ開始トレース-ユーザーがアプリケーションを開始してからアプリケーションが応答するまでの時間を測定します
  • バックグラウンドトレースのアプリ-バックグラウンドでアプリケーションが費やした時間を測定します
  • 前景トレースのアプリ-アプリケーションのアクティビティ時間を測定します


サービスをアプリケーションに接続して、その外観を確認します。 Google Codelabsの標準アプリケーションであるFriendlyChatをベースに接続しました。 ここでパフォーマンスモニタリングを接続します。そのために、プロジェクトレベルのbuild.gradleをbuildscript→リポジトリセクションに追加します。 



 jcenter()


そしてbuildscriptセクションへ→依存関係 



 classpath 'com.google.firebase:firebase-plugins:1.1.0'


次に、アプリレベルのbuild.gradleで依存関係セクションに 



 compile 'com.google.firebase:firebase-perf:10.2.6'


および適用プラグインの下： 'com.android.application' 



 apply plugin: 'com.google.firebase.firebase-perf'


これで、サービスが接続され、アプリケーションの起動時に自動トレースが起動します。



自動トレースでは明らかに十分ではないため、SDKが提供するAPIを使用してカスタムトレースを作成する機会が提供されます。 これらは、パフォーマンスに関連するイベントカウンターで補完できます。たとえば、指定された時間より長くUIが使用できなかった回数です。



カスタムトレースは非常に簡単に追加され、トレースするコードセクションの前に追加します。 



 Trace myTrace = FirebasePerformance.getInstance().newTrace("test_trace"); myTrace.start();


そしてプロットの後 



 myTrace.stop();


チャットに画像アップロードトレースを追加しました。興味を引くためにカウンターを追加します 



 myTrace.incrementCounter("storage_load");


ただし、コンソールの結果はすぐに利用できるわけではなく、12時間以内に表示されます。







高度な分析



そもそも、Firebase Analyticsはもう存在していません。現在、このサービスはFirebase向けのGoogle Analyticsと呼ばれていますが 、これは偶然ではありません。 このサービスはFirebaseとGoogle Analyticsチームによって共同開発され、すべてのレポートはFirebaseコンソールだけでなく、Google Analyticsインターフェースでも利用できるようになりました。











StreamViewとDebugViewの新しい種類のレポートが年の初めに発表されましたが、これはそれらの有用性の程度を損なうものではありません。 StreamViewは、Firebase Analyticsに到着したイベントを視覚化し、最大12時間の遅延ではなく、ユーザーがリアルタイムでアプリケーションと対話する方法の一般的なアイデアを提供します。 また、DebugViewを使用すると、記録されているイベントをすぐに確認できます。







そして、誰もが長い間待っていたもう1つのニュース、 ユーザーイベントとパラメーターの分析情報は、以前必要だったデータをBigQueryにアップロードするのではなく、コンソールのレポートで直接表示できるようになりました 。 これを行うには、コンソールの同じ場所でレポートパラメータを設定するだけです。



さらに多くの興味深い機能が追加されました。AdMobとの統合、自動スクリーントラッキング、BigQueryの無料のストレージ層.... そのため、分析のために、別の記事を書く必要もあると思います。



Firebase SDKがオープンソースになりました



はい、まさにそれです。 Firebase SDKのソースコードは、GitHubで学習できるようになりました。 もちろん、 Firebase iOS SDK 4.0 、 Firebase JavaScript SDK 4.0 、およびFirebase Admin SDK（ Node.js 、 JavaおよびPython ）のソースコードのすべてが現在利用できるわけではありません。 Android SDKをレイアウトする次の約束は、イライラして待つことです。







コード内の何かが気に入らなければ、Googleを修正できるようになりました！ つまり、現在、自分の観点からバグを見つけた場合、標準のGitHub課題トラッカーを使用して課題を作成するだけで、Firebaseチームがそれを検討します。 プロジェクト全体は、 Google Open Source Directoryにもあります。



素敵なささいなこと



最後に、既存のサービスに追加されたすてきなことについて簡単に説明します。



リアルタイムデータベース：



  • 拡張機能、最大100,000の同時接続が利用可能になりました
  • プロファイリング機能が追加され、プロファイラーにより、データの処理速度、スループット、レベルに応じた遅延時間を分析できます


ストレージ：



  • 既存のクラウドストレージバケットにマッピングが追加され、Firebaseを介して作業できるようになりました
  • これで、データを保存する地域を選択できます。 これは、法的観点とパフォーマンスの観点の両方から有用です。


クラウドメッセージング：



  • APNのトークン認証サポートが追加され、クライアントSDKの接続および登録ロジックが大幅に簡素化されました。


TestLab：



  • ゲームループサポートとFPSモニタリングを追加しました。これは、前述のUnity SDKおよびC ++ SDKと組み合わせることで、Firebaseをゲーム開発者にとって魅力的なものにします。
  • ネットワークレベルをシミュレートする機能が追加され、4G、3Gなどの接続速度を選択できます。
  • Google PixelとGalaxy S7は、最新世代の実際のデバイスでアプリケーションをテストするためにデータセンターに登場しました
  • Android Oへのアクセスがありました


また、どのサービスにも接続せずに、SwiftサポートがiOS SDKに実装されたことを追加します。これにより、Firebaseで作業するiOS開発者を満足させることができます。
