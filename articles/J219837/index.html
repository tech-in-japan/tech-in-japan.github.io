knplabs/gaufrette



は、依存関係をまったく持たなくなり、アダプターを含むパッケージのみが提供されます。 



 { "require": { "php": ">=5.3.2" }, "suggested": { "knplabs/gaufrette-mongo-gridfs": "For storing files using Mongo GridFS", ... } }


このアプローチには多くの利点があります。



  • メインパッケージはより安定しています。 変更可能な部品はすべてアダプター内にあるため、何も変更する理由はありません。
  • パッケージごとに開発者が異なる場合があります。 たとえば、 knplabs/gaufrette-mongo-gridfs



    は、MongoDBを熟知している人が変更できます。
  • ユーザーは、使用していないライブラリの一部の更新を監視する必要はありません。
  • ユーザーは、プロジェクトに依存関係を手動で追加する必要はありません。


次回、提案された依存関係をパッケージに追加し、この依存関係がこのパッケージで有効かどうかを考えますか？ 次に、パッケージを分離し、必要に応じてこの依存関係を指定します。 メインパッケージ内のすべてのコードが、この提案された依存関係なしで機能する場合、この場合、提案されたものとして示すことができます。
