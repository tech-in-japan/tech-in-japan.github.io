時は金なり、または時間給の賛否両論





最近、私は友人に、一度だけの仕事のためにフリーランサーを雇うと言った。 15年前に米国に向けて出発した友人から、「1時間にいくら支払う予定ですか？」



-一時間で？ -びっくりしました。 -完了した作業に対してのみ、誰も私たちに時間の支払いをしません。



友人が私と議論し始め、時間ベースの支払いが最良であると私に確信させました。 そして、私は思った。 そして、私は従業員と雇用主の両方に時間給を導入することの賛否両論（そしてまた恐れ）のリストを作ることにしました。



私は写真が非常に曖昧であると判明したと言わなければなりません。 だから...



あなたには仕事があり、アーティストを探しているとします。 完了した仕事にお金を払うとき、典型的なシナリオはどのように見えますか？



  1. 顧客は、可能な限り完全かつ徹底的に参照条件を作成する必要があります。 何も忘れずに、二重解釈を避けてください。
  2. 請負業者はTORを読み、彼の経験に基づいて、製品を作成するのに必要な時間と、その製品を入手したい量を決定します。
  3. 顧客と請負業者は長い間価格で取引されています。 複数のオプションが比較される可能性が非常に高くなります（つまり、複数のパフォーマーが無料でTKを学習します）。
  4. プロジェクト完了の推定期限。 多くの場合、顧客は支払いの遅延に対してペナルティを設定することを好みます。
  5. プロジェクトは段階に分けられます。 請負業者も定期的に食事をしたいので、各段階が完了するとお金が支払われます。


そして、Xyが来て、請負業者は製品の最初のバージョンを見せます...そして顧客は頭を掴みます。 いいえ、彼はピンクの夢でこれを見ませんでした、彼はそれについて夢を見ませんでした、彼のクライアントはそれを支払う準備ができていませんでした。

-パアアズボルテ！ -請負業者に言って、作業明細書に指を突く-ここにすべてが書かれています。









次に、誰が何を書き、誰が何を理解したかについて、長くて不快な対話を開始します。 その結果、双方が譲歩します。 顧客は彼の期待をなだめ、彼の夢ののどを踏んで、請負業者は製品をさらに洗練することに同意します。 その結果、一方の側はお金の最初の部分を手放し、もう一方の側はそれを見つけますが、どちらも自分たちが…（少しだまされている）という感覚を持っています。



その後、Marlezonバレエの第2幕が登場し、すべてが新しい方法で繰り返されます。 繰り返しますが、「ああ、まったく欲しくない」という叫びと、「お金をください、お金をください」という叫びがあります。









ほとんどの場合、プロジェクトの途中までに、顧客と請負業者（若い家族のように）がお互いに理解し合い、一目で理解することを学び、妥協します。 その後、プロジェクトは正常に完了する可能性が高くなります。



請負業者がTKに過度に依存し、顧客が彼の言葉にならない夢に依存している場合、カップルは厄介なことを言って逃げることができます。 製品の歴史がそこで終わることは明らかです。









ここで、プロジェクトの期間中に顧客が請負業者に支払う準備ができたときの別のシナリオを検討します。



  1. 顧客は作業明細書を書きません。 または彼は書いているが、彼にとって望ましい詳細度で。 プロジェクトの作業中の請負業者は、完成品のビジョンを明確にします。
  2. 顧客と請負業者は、1時間（または1日）のコストとプロジェクトの推定期間に同意します。 また、お金が支払われる頻度についても（たとえば、週に1回）。
  3. 請負業者は、製品の中間バージョンを定期的に提示します。 顧客は、好きなことと嫌いなことを言います。 請負業者はすべてを穏やかにリメイクします。結局のところ、彼の時間は支払われます。
  4. 週末に、請負業者はプロジェクトに費やされた時間に関するレポートを送信します。 顧客が支払う。
  5. 最終的に、長い反復を経て、請負業者は顧客が夢見ていた製品を作成します。


やあ、みんな幸せだ。 右天国...







フィギュア！ おそらくそうなるでしょう...



  • 顧客は、請負業者が実際にプロジェクトにXX時間を費やしたとは考えていません。 彼は確かだから/専門家は彼に、実際にはここに半分の仕事があると言った。
  • 顧客は、請負業者が貧弱な専門家であるため、作業が遅すぎると考えています。 他の人が1時間で管理する場合、3つ必要です。 しかし、誰がこれをすべて支払うのですか??！
  • 顧客は別の予算に頼っていました。 彼は、1時間のコストにプロジェクトの推定期間を掛けました。 そして、このすべての喜びはすでに2倍高価です！
  • 前の段落の結果。 顧客はお金を使い果たしました...






最初の結論は何ですか？ 十分に書かれたTKと組み合わせて予算が限られている場合は、プロジェクトの支払いオプションを選択します 。 たくさんのリスクを取り除いてください。 理想的な製品はほとんど手に入らないでしょうが。



私たちは考え続けます。 顧客は何を恐れますか？



  1. 費やした時間数に関するレポートの偽造。
  2. 出演者の資格が低く、その結果、プロジェクトでの長時間の作業とエラー。


改ざんの問題は、2つの方法で解決されます。



  1. 請負業者の作業プロセスの技術的管理。
  2. 彼の評判をコントロールします。


残念ながら、評判をコントロールできる状況はあまり良くありません。 すべての欺かれた顧客が否定的なレビューを書くわけではありません。 すべてのお客様がごまかされたことを知っているわけではありません。 すべてのお客様がアーティストに関するレビューを読むわけではありません。 そして、多くのパフォーマーがいくつかのサイトから注文を受け取ります。



技術的な制御のためのシステムはたくさんあります。 これらは、独立した開発者とフリーランスエクスチェンジ（同じUpwork）の両方によって提供されます。 これらは独立して機能するか、IDEに組み込まれています。 これらは3つの主要なグループに分けられます。



  1. 請負業者は、プロジェクトの作業が開始する瞬間と終了する瞬間をマークします。 レポート期間の終了時にシステムは、請負業者がプロジェクトに費やした時間に関するレポートを生成します。 顧客にとってマイナス-あなたはパフォーマーに言葉を信じなければなりません。
  2. システムは、開発者のコ​​ンピューターのスクリーンショットを定期的に取得します。 顧客は、スクリーンショットを含むレポートを受け取ります。理論的には、請負業者が仕事で本当に忙しいことを確認する必要があります。 実際、その瞬間、彼は別のPCで別のプロジェクトに非常によく従事できました。
  3. 制御システムにより、開発者の画面のビデオを録画したり、リアルタイムで表示したりできます




1番目と3番目の方法を組み合わせることで、最良の結果が得られます。 この場合、開発者は各作業時間の支払いが確実に行われ、顧客との競合は発生しません。 以前に悪意のある顧客が「でも、あなたがそんなに働いたとは思わない」と言うことができたら、今度は彼にビデオ録画を見せることができます。







しかし、演奏者の資格を測定することは非常に困難です。 ある開発者は高速に書き込みますが、多くの間違いを犯します。 別の品質ですが、遅いです。 3番目はすべての人に適していますが、プロジェクトに必要なAPI /フレームワークを知らないため、外出先で直接学習し、多くの場合顧客のお金のために学習します。







ここにいる方法は？



プロジェクトで使用されるすべての技術を事前に規定し、請負業者にこれらの技術について十分なレベルの知識を要求します。



はい、それは複雑です。プロセスと顧客自身からの一定の知識が必要ですが、インターネットには、顧客から最初に聞いたときに、AngularJSで開発を喜んで始めた熱狂的な笑いのフリーランサーの話がたくさんあります。



そして、初心者向けのアドバイスが続きます-常にそれをしてください！ 同様に、著者自身はプログラミングで完全にゼロであり、開発プロセス中に学習しました（もちろん、時間給で）。



エラーの数を超えた場合のペナルティを設定します。 もちろん、完全にクリーンなコードを書くことは不可能です。 しかし、プロは、アマチュアとは異なり、品質に責任を負う用意があります。 そして絶対に率直な失態、彼は常にキャッチします。



貪欲にならないでください 。 優秀な専門家は高価ですが、彼らは迅速に働き、通常は手間がかかりません。 初心者のコーダーとは異なり、彼らは少しかかりますが、お金は「キャロットキンザゴバニヤまで」あなたから引き出されます。







唯一のことは、彼らが需要があり、おそらくあなたのプロジェクトだけでなく従事するということです。 したがって、完了期限、または請負業者がプロジェクトに費やす時間の割合のいずれかを規定することは理にかなっています。



一般に、次の結論を出すことができます。



時間給は、請負業者と顧客の両方にとってより有益ですが、請負業者の資格が顧客の期待を満たしていれば、費やした時間を適切に制御できます。
