Microsoft Translatorウィジェットのテスト

Microsoft Translate Widgetのテストにアクセスするための待望の招待状を受け取ったことを発表できてうれしいです。 おそらく、あなたはすでに外国のサイトでそのようなソケットに遭遇しています：









これは同じウィジェットです。 サイトを他の13言語に完全に自動的に、ページをリロードすることなく翻訳できます。



ウィジェットの利点：



*オンザフライでのページのAJAX翻訳。

*非常に高速な作業。

*元のオプションに即座に戻る。

* 13言語の翻訳。

*セットアップ、インストール、使用が簡単。

*ソーステキストのヒントを使用した強調表示。





ユーザーは翻訳の言語を選択し、矢印をクリックするだけです。 そのような翻訳の例：







ウィジェットは英語サイトでよく表示されますが、残念ながら、ロシア語からの翻訳はまだうまくいきません。 ご覧のとおり、input：textなどの要素は変換されません。 私のサイトでクロールしたいくつかのアーティファクトがあります。 私は間違いなくバグ報告でそれらについて開発チームに知らせます。 ユーザーの元のテキストのバリアントがヒントとして利用できます。











ウィジェットをテストするために、2つの方法を提供できます。

1.（ストレスの多いバージョン）ロシア語のWebサイトでの動作を確認するには、 progg.ruリソースでテストすることをお勧めします

2.（デバッグ版）英語版サイトで実行バージョンを確認するには、たとえばこちらをご覧ください



最初のオプションは間違いを犯し、より困難であることをすぐに言う必要があります。つまり、速度は遅くなりますが、支援したい開発者へのフィードバックを作成する資料があります。 2番目のバージョンでは、ウィジェットのすべての利点がすぐに表示され、ロシア語への翻訳があります。
