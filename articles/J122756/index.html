PCとスマートフォンの相互作用のためのソフトウェア

Androidでスマートを購入して以来、機能的なコンテンツに興味がありました。 実際には、Androidは通常のPCで解決できるほぼすべてのタスクを実行できることがわかりましたが、速度は遅く、必ずしも便利ではありません。 しかし、これはPCを完全に補完し、スマートオンPCの機能とスマートオンPCの機能を使用するための多くのメカニズムがあります。



この記事は主に、Androidスマートフォンの新興および将来のユーザーに焦点を当てています。 上級ユーザーもその中に何か新しいものを見つけ、貢献することを望みます。 カットポイントの下、質疑応答スタイル。



スマートでPCインターネット接続を使用する方法は？

  • すべてが私の前にすでに書かれています 。 個人的には、 Connectifyを使用しています （Windows 7でWiFiホットスポットを作成しています）。



PCでスマートインターネット接続を使用するには？

  • 同じトピックですべてがすでに私に書かれています。 ここでは、Android OSの標準機能を利用します（Android 2.2から利用可能なワイヤレスとネットワーク>>モデムモード）。 以前のバージョンのAndroidを使用しているスマートオーナーは、 PdaNet （USBテザリング、Bluetooth DUN）に注意する必要があります。



デバイスのファイル構造を共有する方法は？

  • 方法1.標準-データケーブル経由。 欠点は明らかです-有線の連絡先、あなたはSDカードをアンロードする必要があり、時にはドライバーをインストールします。 利点は伝送速度です。



  • 方法2. FTPサーバー（SSH、SMB、HTTP、HTTPS、 WebDAV 、 AFP ）サーバーを上げます。 この方法は信頼性が高くシンプルで、コンピューターへのWiFi接続が必要です。 On Air（WebDAVおよびAFPプロトコルを介した対話も可能）などのプログラム、SwiFTPはデバイス上のFTPサーバーを上げることもでき、その機能はFile ExpertおよびRemote Web Desktopプログラムにも統合されています。



    オンエアには、シンプルなインターフェースと最小限の設定があります。 サーバーが起動すると、4桁のデジタルパスワードが生成されます（n番目の要求の後にサーバーをブルートしようとすると、明らかに予防措置が組み込まれます）、サーバーのアドレスが表示されます。 主な欠点は、ユーザー名、パスワード（このパスワードが常に静的であることのみを指定できます）、サーバーポート、共有ディレクトリを手動で設定できないことです。 利点は単純さです。



    SwiFTP FTPサーバーは、私が試した中で最も完全なFTPサーバーです。 ユーザー名、パスワード、ポート、さらには共有ディレクトリも指定されています（ここでのみ、このようなオプションが見つかりました）。 欠点は、ワンクリック共有がないこと（FTPサーバーをクリックしてクリックするショートカットをデスクトップ上に作成できないこと）、およびインターフェイス自体にはないことです。



    File Expert （私の選択）は、ファイルマネージャーとして機能するアプリケーションですが、FTPとWEBサーバーの両方が含まれています。 十分な設定があり、共有ディレクトリ（常にsdcard）を指定できるだけではありません。 FTPまたはWEBサーバーをすばやく起動するためのショートカットを作成できます。



    リモートWebデスクトップ （ 完全版 ）は、多機能WEBサーバーとして機能し、FTPサーバーも組み込まれています。 大きな欠点は、FTPサーバーの無料版の速度が制限されていることです-わずか100 kb / s（平均して、無制限〜2-4 Mb / s）。 ただし、WEBサーバーにはさまざまな機能があります。 有料版では、HTTPSプロトコルを介したやり取りが可能です（他のどこでもこのような機会を見たことはありません）。 類似品は十分にありますが（通常、名前にはWebデスクトップなどが含まれます）、リモートWebデスクトップが最も機能的であり、無料版には多くの制限はありません。



    Sambaファイル共有 （ belkで推奨 ）を使用すると、デバイスでSMBサーバーを起動できます。 Windowsユーザーに最も関連性が高い-デバイスは共有ネットワークフォルダーとして表示されます。 ルートが必要です。



    有料のQuickSSHdは、スマートフォンでSSHサーバーを実行します。PCでは、SCP / SFTPクライアントを使用します（Windows： WinSCP ; Linux： OpenSSH ; Mac： Cyber​​duck ）。



  • 方法3. Bluetooth経由でファイルにアクセスします。 長期にわたる方法では、受信デバイスにBluetoothが存在する必要があります。 Androidでは、追加のソフトウェアは必要ありません。デバイスの表示をオンにするだけです。 Windowsでは、 BlueSoleilのようなものが必要です。



  • 方法4。インターネット（ファイルサーバー、ドロップボックス）を使用します。 自明。

    UPD ：最近、ES ExplorerにDropboxを操作する機能が追加されました。



PCのファイル構造を表示する方法は？

  • 方法1. Bluetoothを使用します。 デバイスにES Explorerをインストールします（[設定] >> [Bluetoothオプション] >> [BTを非表示]のチェックを外して、交換フォルダーを指定します）。 Windows PCでは、BlueSoleilが必要です。その設定で共有ディレクトリを指定します。



  • 方法2。PCでFTP（SMB、WEB）サーバーを上げます。 繰り返しますが、ES Smart Explorerがあれば、必要なものはすべて揃っています。 Windows 7を実行しているコンピューターでは、IISの バグのある 組み込みFTPサーバーを使用できます 。 また、Microsoft SMBプロトコルに従って、Windows共有ネットワークフォルダー（ESエクスプローラーの[LAN]タブ、File Expertの[ネットワーク]タブ）を参照するだけです。



デバイスのカメラ（コントロール、メディアストリーム）にフルアクセスする方法は？

  • 最も確実な方法は、 IP Webcam for Androidを利用することです。 必須-安定したWiFi（インターネット）接続。 IP Webcamを使用して、ビデオ解像度、品質、向き、ユーザー名/パスワード、ポート、音声送信機能などの設定を指定した後、WEBサーバーを上げます。 サーバーを起動した後、ブラウザから指定されたアドレス（192.168.2.101:8080があります）にアクセスし、任意の対話方法を選択します-メディアプレーヤーでストリームを開き、リモートコントロールパネルを開き、ブラウザーのJavaプラグインを使用します（ブラウザーでビデオとオーディオを再生します） 、唯一のJavaを必要とする！）ブラウザでカルドを更新するためにはJavaScriptを使用して、使用して他のスマートフォン上のメディアで放送を見るPROモニターをtinyCamをして（ 無料版カメラの存在をスキャンする機能をネットワーク化されていない）などのスマートフォンのカメラを使用します 私は、Skypeに通信します。 簡単な指示が提供されます。 望ましい結果を達成しないことは非常に困難です。



スマートフォンからストリーミングメディアをストリーミングする方法は？

  • VLC Directのインストールは非常に簡単です。 PCでは、 VLC Playerが必要です。 VLC Direct on smartでは、[設定]の[自動ウィザード]を使用できます。これにより、プロセス全体が大幅に容易になります。



スマートフォンでストリーミングメディアを表示する方法は？

  • AndroStream Free （ Pro ）、 Stream Media Player （ Pro ）、 LittlePlayer Free （ Pro ）のストリーミングプレーヤーを使用します。



Androidスマートフォンを使用してコンピューターを制御する方法

  • 方法1.フルコントロール-VNC（RDP）を使用します。 VNCサーバーをコンピューター（ TightVNC 、 RealVNC ）、スマートVNCクライアント（ Remote VNC w / Ad 、 VNCビューアー （有料））にインストールします。 WiFi経由でコンピューターへの安定した接続を確立し、サーバー、クライアントを構成します。 テキストを印刷したり、マウスを制御したり、キーボードショートカットを押したりして、OSをほぼ完全に制御できます。 RDP制御に十分なクライアントもあります 。



    1つありますが、RealVNCは独自のプロトコルを使用しているため、Androidで有料のVNCビューアーを使用する場合にのみ暗号化された接続を確立できます。



  • 方法2。個々のアプリケーションを制御します。 これは、一部のプレーヤー（ Aimp UControl ）、uTorrent 3クライアント（ µTorrent Remote ）などに当てはまります。 強力なトレント管理ソリューション-Transdroid -habrovchanin Wizard999にアドバイスします（µTorrent、Transmission、rTorrent、Vuze、Deluge、BitTorrent 6をサポート）。



  • 方法3.スマートフォンをマウスとして使用する。 これにより、 Remote Droid Serverを実行する必要があるPC上でRemoteDroid （ Pro ）を実行できます 。 巡礼のヒントとして、PC上のアクティブなアプリケーションを制御し、ホットキー/組み合わせの押下をエミュレートできるアプリケーションもあります。UnifiedRemote （ Full ）、 MultiRemote 、 gPad remote touchpad / keyboard ; また、（PR）彼自身の開発-piCat （アルファ版）。



  • 方法4（ EveresTが提案）。 TeamViewer App for Androidを活用してください。 完全な制御を提供します。 非営利目的での使用は無料で、インターネットを通じてうまく機能します（より正確には、これが主な目的です）。 PCに非モバイルバージョンをインストールする必要があります（ Windows 、 Mac 、Linux Red Hat、Fedora、Suse、Mandriva 、 Debian、Ubuntu 32-Bit 、 Debian、Ubuntu 64-Bit ）。 すべての設定は直感的で、製品は最高です！



    Habrausers Saencoとアナグノストは、代替案-LogMeInに言及することを主張しました。 確かに、Android用の無料クライアントはまだありません -有料のクライアントのみ： two 私はそれを経験していません-彼らはそれが便利であると言います。



  • 方法5（ naryl 、 SantaClaus16が提案）。 スマートフォンで、SSHクライアント（ ConnectBotなど ）を使用し、PCでSSHサーバーを展開します（Linux：OpenSSH、Windows： MobaSSH 、 Mac ）。 ConnectBot + MobaSSHの束をチェックしました-動作します！ Linuxを愛する人のための方法。



  • すばらしいVNC / RDPクライアントであるWyse PocketCloud （ Pro ）は、Wizard999 habrayuzerによって（再び）促されました。 PocketCloud CompanionはWindowsまたはMacにインストールされます-その助けを借りて、インターネット経由でPCにインストールされたお気に入りのVNCまたはRDPサーバーを使用できます。 認証には、Googleアカウント（アカウントを持っていない）が必要です。 GoogleログインとGoogleパスワードをスマートフォンとPCに入力します（入力する必要があるのは1回だけであることは明らかです）。 PCからの認証後、接続がすぐにAndroidに表示され、VNCまたはRDPパスワード（または、VNCまたはRDPサーバーの構成方法に応じてWindowsパスワード）に入力します。 Android-Wyse PocketCloud、Windows-PocketCloud Windows CompanionおよびRealVNC（認証方法-VNCパスワード）など、たくさんあります。 方法1と4を組み合わせて使用​​することは非常に良い考えです。



PCを使用してスマートフォンを制御する方法は？

  • このタスクはあまり実用的ではありませんが、解決する必要がある場合は、 リモートコントロールアドオンが役に立ちます。 それはルートを取ります。 プログラムは、デバイス上のVNCサーバーを起動します。 その後、ブラウザ（javaが必要）またはVNCクライアント（ip：5901、暗号化なし）で表示URLを開くだけです。



  • 管理がSMSの表示/書き込み、連絡先の管理などを意味する場合、リモートWebデスクトップまたは同様のプログラム（ リモートデスクトップなど ）の機能を使用すると便利です。



    SabMakc HabraiserはMyPhoneExplorerの使用を推奨しています（Windowsのみ、Android上のスマートフォンのサポートはバージョン1.8から含まれています）。 Bluetooth、USBケーブル、WiFiを介して接続することができます。



PCキーボードを使用してスマートフォンでテキストを入力する方法

  • WiFi接続が確立されると、 WiFiキーボードまたはezKeyboardを使用できます（最初は、入力されたテキストはスマートフォンにすぐに表示されますが、ブラウザーには表示されません。ゲームモードも提供されます。2番目のテキストはボタンを押した後にのみ送信されます）。 リモートWebデスクトップにはWiFiキーボードも提供されます（自動同期を有効または無効にできます）が、わずかな欠陥があります-スマートフォンの設定のWiFiキーボードチェックボックスが自動的に削除され（不明な理由により）、必要な入力方法を使用できません（言語とキーボードの設定をもう一度確認する必要があります。チェックボックスをオンにします）。 個人的には、私はezKeyboardを好みますが、めったに使用しません（ Swype 、あなたは知っている、クールなこと）。

PCブラウザーからスマートフォンにリンクを直接送信する方法は？

  • Chrome to Phoneアプリ（ A1lfeGが推奨 ）を使用します。 ブラウザのPCに、拡張機能（ Chrome 、 Firefox 、 Opera ）をインストールする必要があります。 残念ながら、このアプリケーションはロシアの市場では利用できません（または、私には思えましたか？）。 別の方法はLinkPushです。 拡張機能は、ChromeおよびFirefox専用です 。 他の人はブックマークレットを使用できます （オプションで、Googleトークで通知を有効にできます）。 Chrome to Phone自体は変更できませんでしたが、明らかに、このプログラムはLinkPushよりも機能的です。
