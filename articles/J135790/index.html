カメラ付きの無線制御Wi-Fiマシン

開始する



夏には、無線制御の機械を作るというアイデアが浮上しましたが、あらゆる段階で販売されている中国のエンジニアリングの作成に似たものだけでなく、コンピューターや電話から制御できる機械もできました。 Wi-Fiを介して最も純粋な形で制御できるマシンは、まったく面白くないことが理解されています。 しかし、彼女がカメラを持っていたら？ また、3G / EDGE / GPRSを介した管理もできますか？ これは別の問題です！ そのため、制御デバイスにはUSBとWi-Fiが必要です（まあ、またはUSBのみ、USB Wi-Fiアダプターを購入できます）。 ここで、エンジンの制御方法を理解する必要があります。 最初は、COMポートとシフトレジスタ（74HC595）を使用してこれを実行したかったのですが、このような5つのマイクロ回路を焼き付けた後、この方法を拒否しました。 後に、私の目はArduino、つまりArduino Duemilanoveに目を向けました。 このボードには14個のデジタルI / Oポートがあり、そのうち6個はPWM （モーターの電圧を制御し、カメラのサーボを掛けることができます）、2個はTxおよびRxシリアルポートとして使用できます。

ルーター



市内でUSBポートを備えたD-Link DIR-320ルーターを見つけたので、すぐに購入しました。 家に着いて、私はこのルーターに未検出のUARTポートがあることを発見しました。 したがって、ルーターとarduinoの間に通信チャネルがあります。

ルーターには、OpenWrtファームウェアを選択しました。 DIR-320用のOpenWrtから既製のファームウェアをダウンロードできます 。 理由は覚えていませんが、ファームウェアを自分で組み立てることにしました（詳しくはこちらを参照 ）。 これにはLinuxが必要です（Ubuntu 11.10でビルドしました）。 開始するには、ファームウェアソースをダウンロードし、必要なものをすべて収集します。 

svn co svn://svn.openwrt.org/openwrt/branches/backfire dir320 cd dir320 ./scripts/feeds update -a && ./scripts/feeds install –a make prereq && make tools/install && make toolchain/install

フラッシュドライブからの起動の構成


最初のインクルードの後、ルーターのWebインターフェースに移動してパスワードを変更します。 次に、SSH経由で接続します。 フラッシュドライブからのダウンロードを設定する必要があります。そのためには、まずマークアップする必要があります。 2つのパーティションがありました。1つ目はrootfs用のext3パーティション、2つ目はスワップです。 vimで/ etc / config / fstabを開き、felshkaに対応するものを書き込みます。 私はこれを持っています： 

  config global automount option from_fstab 1 option anon_mount 1 config global autoswap option from_fstab 1 option anon_swap 0 config mount option target / option device /dev/sda1 option fstype ext3 option options rw,sync option enabled 1 option enabled_fsck 1 option is_rootfs 1 config swap option device /dev/sda2 option enabled 1


保存して再起動します。

Arduinoと接続



まず、エンジンの接続図を決めましょう。 既製の機械からケースを取り出したので、私はエンジンで幸運でした-それらはすでにそこにありました。 前部は回転（左、右、右）を担当し、後部は移動（一部のVAZのドアロックボタンのエンジンに変更する必要がありました）を担当しました。 電界効果トランジスタ（95N2LH5を使用しているが、私の都市でこれらのトウヒが見つかったためIRF 630を使用）を使用して、arduinoで負荷を制御できます。 接続は次のとおりです。トランジスタのグランド-アルデュインの制御ピン、ソース-アルデュインのグランド、マイナスの負荷供給、ドレイン-マイナスの負荷、プラスの負荷のプラス。 しかし、この方法では、私たちは前方にのみ乗り、一方向にしか回転できません。 この問題に対処するために、2つの接点グループを持つリレーが急いで支援を求めます。 1つのエンジン（フロント）に6ボルト、もう1つに12を搭載しています。6ボルトのバッテリーを2つ使用しました（そのうちの1つは無停電電源からの鉛酸です）。ルーターでは機能しません（ルーターに供給しているボルトの数を確認してください。12ボルトを印加した後、もう1つ購入しなければなりませんでした）。 そのため、フロントエンジンに電力を供給/供給しないために別のリレーを使用する必要がありました。



私は長い間図を描きました。 現在、すべてのトランジスタは電界効果であり、抵抗はありません。

次に、コード自体について説明します。 すべてが私にとって非常に簡単です-1バイトサイズの独自のパラメーターを持つ4つのコマンドがあります。

  • m-電圧に責任があり、その結果、速度に応じて、エンジンの0〜255の値
  • r-ターンを担当します。 「1」-ターン、「0」-ターンしない
  • n-「1」-戻る、「0」-進む
  • e-「1」-反対方向に曲がる


arduinoプログラムのコードは次のとおりです。 

 int inByte, val; void setup() { Serial.begin(9600); pinMode(2, OUTPUT); pinMode(4, OUTPUT); pinMode(7, OUTPUT); } void loop() { if (Serial.available() > 0) { inByte = Serial.read(); if ((inByte=='n')||(inByte=='e')){ while (Serial.available()==0) {} val=Serial.read(); if (inByte=='n'){ if (val=='1'){ digitalWrite(2, HIGH); Serial.print("Writing to 2 pin\n\r"); } if (val=='0'){ digitalWrite(2, 0); Serial.print("Writing to 2 pin\n\r"); } } if (inByte=='e'){ if (val=='1'){ digitalWrite(4, HIGH); Serial.print("Writing to 4 pin\n\r"); } if (val=='0'){ digitalWrite(4, LOW); Serial.print("Writing to 4 pin\n\r"); } } } if ((inByte=='m')||(inByte=='r')){ while (Serial.available()==0) {} val=Serial.read(); if (inByte=='m'){ if (val!='0') analogWrite(3, val); else analogWrite(3, 0); Serial.print("Writing to 3 pin\n\r"); } if (inByte=='r'){ if (val=='1'){ digitalWrite(7, HIGH); Serial.print("Writing to 7 pin\n\r"); } if (val=='0'){ digitalWrite(7, LOW); Serial.print("Writing to 7 pin\n\r"); } } } } }


ご覧のとおり、私のリアモーターは3つのピンに接続され、フロントは1〜7、リアリレーは2ピン、フロントは4に接続されます。3はPWMピンであるため、analogWrite（3、val）;、where val 0から255まで、モーターの電圧を制御できます。

ルーターを分解します。 UARTポートが表示されます。 arduinoと接続します。



それでは、すべての仕組みを見てみましょう。 telnetでポートに接続し、以下を確認します。

  • n1-リレーをクリックします
  • m <スペース>-ホイールが少し回転し始めます
  • n0-ホイールは反対方向に回転します
  • m0-ホイールの回転が停止します
  • r1-前輪を回します
  • e1-車輪が反対方向に曲がる
  • r0-車輪はまっすぐになります
  • e0-リレーをクリックします


ルーターのポートで作業をデバッグするには、minicomを使用できます（ opkg install minicom
