こんにちは、Habr！ 通常、金曜日に荷を下すことはありませんが、今日は特別なケースです。第一に、冬はそれほど近くない、第二に-春の始まりと猫の日、第三に-この待望の機能でどれだけ引くことができますか？！







TL; DR



カットの下に来て、出版物を選択し、CTRL / CMD + Enterを待ちます-その後、あなたはすべてを自分で理解します。



私たちは、出版物のエラーやタイプミスの問題をよりグローバルに解決し、将来的に改善する予定ですが、そのためには出版社の編集者を含む多くのことをやり直す必要があります。 したがって、今のところ、私たちはあなたが私たちに要求したオプションを作ることにしました-kotkeyを通して出版物のエラーに関する情報を送信する機能。



操作アルゴリズムは次のとおりです。



1.パブリケーションでエラーのあるテキストの断片（任意-文法または句読点）を選択します。



2.ホットキーCTRL + Enter（またはCMD + Enter）を押します。



3.画面の下部にフォームが表示され、以前に強調表示されたテキスト（ポイント1）が引用され、オプションの説明用のフィールドが表示されます。



スクリーンショット






4. [送信]をクリックします。 出版物の著者は、次のようなプライベートメッセージで（タイプミスを送信した人に代わって）手紙を作成します。



スクリーンショット






5.著者がメッセージにすぐに気づき、送信されたタイプミスを修正するという希望を失わない



ニュアンス：



  1. この機能は出版物でのみ機能します。
  2. モバイル版には機能はありません。
  3. 少なくとも出版物全体を選択できますが、引用に収まるのは220文字だけなので、スペルミスの単語を具体的に指定することをお勧めします。 付随するコメントでは、最大500文字をドライブできます。
  4. 1 typo = 1メッセージ（出版物の著者との対話で）;
  5. 誰かがすでに出版物の著者に送ったフラグメントを選択すると、これについて通知されます：







  6. ReadOnly-およびRead＆Comment-usersでは、メッセージ送信ファームにcaptchaが追加されます（新しい作成者への最初のメッセージのみ）。


このプーリングには少なくとも10のタイプミスがあります-練習できます。 良い週末を！
