Excel用Officeアドイン-VBAおよびVSTO開発者向けの新機能

背景



それはすべて約4年前に始まりました。 ロシアの大規模な小売チェーンのビジネスプロセスを自動化する別のプロジェクトに取り組み、オフィスアプリケーション、特にExcel用のアドオンの開発に興味を持ちました。 顧客企業の従業員が日常の反復業務に多くの時間を費やしているのを見るのに数日かかりましたが、私は彼らの人生を単純化する方法について多くのアイデアを思いつきました。



当時、開発者として、非典型的なタスクのためにExcelを「拡張」する2つの方法がありました。

  • VBA （Visual Basic for Applications）;
  • VSTO （Visual Studio Tools for Office）。










すべてのExcelアドイン開発者は、両方のアプローチの長所と短所をよく知っていると思います。 両方の大きな利点は、ほとんどすべてのタスクを自動化できる非常に豊富なAPIです。 欠点には、そのような拡張機能のインストールの難しさが含まれます。 これは特に、VSTOに基づくアドオンの場合に当てはまります。多くの場合、インストールには管理者権限が必要であり、その取得はエンドユーザーにとって問題になる可能性があります。



議論がこの記事の範囲外であるいくつかの理由のために、私は自分でVSTOのオプションを選択しました。 こうして、Microsoft Excel用の最初のアドイン-XLToolsが誕生しました 。 製品の最初のバージョンには、次を可能にするツールが含まれています。



  • Excelセルでデータクリーニングを実行します（余分なスペースや印刷できない文字を削除したり、レジスタを1つのビューに表示したりするなど）。
  • テーブルを「2次元ビュー」から「フラット」（ピボット解除）に変換します。
  • 列のデータを比較する
  • 上記のすべてのアクションを自動化するツール。


Officeストアの登場



XLToolsアドインの最初のバージョンがリリースされてからわずか1年後、MicrosoftはOfficeの拡張機能を宣伝するための新しいプラットフォームであるOfficeストアを立ち上げていることを知りました。 最初に考えたのは、そこで新しいXLToolsアドオンを公開できるかどうかです。 残念ながら、たぶん幸いかもしれませんが、この質問に対する答えは「いいえ」です。 VBAアドインもVSTOアドインもOfficeストアに公開できません。 しかし、動揺する価値はありますか？ 幸いなことに、ここに答えがあります-いいえ、価値はありません。 次に、その理由を説明します。



Office用の新しいアドイン



Officeストアとは何ですか？なぜ必要なのですか？ 要するに、これは、ユーザーや開発者が、 Excel 、 Word 、 Outlook 、 OneNote 、 PowerPointなど、Officeプログラムの標準機能を拡張するアドオンを検索、ダウンロード、販売、購入するのに役立つプラットフォームです。 以前のエンドユーザーが検索エンジンで必要なアドオンを探す必要があった場合、この目的のために、Officeストアという単一の場所が作成され、Officeプログラムのインターフェイスから直接アクセスできます。 メニュー項目[挿入]-> [マイアドオン]：











既にわかったように、VBAまたはVSTOを使用して開発されたアドオンは、Officeストアに公開できません。 Office 365およびOffice Storeのリリースに伴い、MicrosoftはOffice用JavaScript APIを使用してアドインを開発する新しい方法を提供しました。これには、 HTML5、CSS、JavaScript 、 WebサービスなどのWebテクノロジーを使用したアプリケーションの開発が含まれます 。



新しいアプローチには、長所と短所の両方があります。 利点は次のとおりです。



  • Officeストアからアドオンを簡単にインストールできます。
  • すぐに使用できるクロスプラットフォーム （Excel 2013/2016、 Excel Online 、Excel for iPad ）;
  • 蓄積されたWeb開発の経験を活用する能力（チームに既にWeb開発者がいる場合、新しいテクノロジーを学ぶ必要はありません）;
  • アドオンを固定価格またはサブスクリプションで販売できる既製のインフラストラクチャ。


新しいアプローチの欠点のうち、私はこれまでのところ非常に重要なものの1つだけを選び出すことができます。

  • VSTOおよびVBAと比較してAPIの機能が劣ります（この問題がAPIの新しいバージョンのリリースとの関連性が低下することを願っています）。


「新しい規則に従って」Excel用アドインの開発









では、時代に遅れずに、Officeの新しいアプリケーションの波を見逃したくない場合は、どこから始めればいいのでしょうか。



2つのオプションがあります。 現時点では、次のJavaScript APIに基づいてアプリケーションを開発できます。



  • Napaは、クイックスタート用の開発環境の軽量Webバージョンです。 Visual Studioを持っていない開発者、またはWindows以外のオペレーティングシステムで開発したい開発者に役立ちます。
  • Visual Studio 、バージョン2012以降、インストール済みのOffice Developer Tools-より強力で機能的な開発環境。 以前にVSTO用に開発した人は、このオプションですぐに開始できます。 彼らはすでにVisual Studioを持っています。


この記事では、Visual Studioを使用した開発を検討します。 私自身が使用しています。 Napaの試用に興味がある場合は、このツールに慣れて、 ここで作業を開始してください 。



開発を開始する前に、OfficeストアのアドインからのVBAアドインとVSTOアドインのいくつかの重要な違いにも注意する必要があります。

  • 最初の違いは、VBAまたはVSTOでアドオンを開発するときに、多くの機能を含むいわゆる「パッケージ」製品を作成できることです。 XLToolsは素晴らしい例です。アドインには、セル、テーブル、列などを操作するための多くのオプションが含まれています。 Officeストア用のアドインを開発する場合、このアプローチを忘れる必要があります。 開発を計画するときは、エンドユーザーに提供したい、他の機能から分離された完全な種類を考える必要があります。 XLToolsの場合、元々1つのアドオンに実装されていた機能は、Officeストアの5つの個別のアプリケーションで表されます。 このアプローチにより、ターゲットを絞り込んで意思決定を絞り込み、ターゲットユーザーによるアドオンのダウンロード数を増やすことができます。
  • 2番目の違いは、JavaScript APIとVSTO / VBA APIの違いです。 JavaScript APIが提供する機能を詳細に調べる価値があります。 これを行うには、Microsoft チュートリアル（タスクペイン）およびMicrosoft チュートリアル（コンテンツ）APIアプリケーションを使用することをお勧めします。


Visual StudioとJavaScript APIを使用したExcelアドイン開発



既定では、Visual Studioには、Officeストアのアドインを開発するための定義済みのプロジェクトテンプレートがあるため、新しいプロジェクトの作成には文字通り数秒かかります。







プロジェクト自体は、マニフェストファイルとWebサイトで構成されています。 マニフェストファイルは次のようになります。 

<?xml version="1.0" encoding="UTF-8"?> <OfficeApp xmlns="http://schemas.microsoft.com/office/appforoffice/1.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:type="TaskPaneApp"> <Id>11111111-1111-1111-1111-111111111111</Id> <Version>1.0.1</Version> <ProviderName>WavePoint Co. Ltd.</ProviderName> <DefaultLocale>en-US</DefaultLocale> <DisplayName DefaultValue="XLTools.net Data Cleaning for Excel" /> <Description DefaultValue=" Clean bulk data fast: remove extra spaces, change text case, convert text format to numbers, etc."/> <IconUrl DefaultValue="~remoteAppUrl/App/DataCleansing/Images/Logo32.png"></IconUrl> <SupportUrl DefaultValue="http://xltools.net/excel-apps/data-cleaning-app/"></SupportUrl> <Capabilities> <Capability Name="Workbook" /> </Capabilities> <DefaultSettings> <SourceLocation DefaultValue="~remoteAppUrl/App/DataCleansing/Home.html" /> </DefaultSettings> <Permissions>ReadWriteDocument</Permissions> </OfficeApp>




このファイルで注意すべき主なものは次のとおりです。



  • Id-アプリケーションごとに一意である必要があります。
  • バージョン -アドインがセラーダッシュボード （Officeストアで公開されたすべてのアドオンを管理するベンダー/開発者の個人アカウント）から公開されたときに指定されたバージョンと一致する必要があります。
  • IconUrlおよびSupportUrl-リンクが機能し、アドイン機能の説明を含むロゴ画像とページの場所を示す必要があります。 リンクが正しくない場合、マニフェストはセラーダッシュボードを介して公開するときに検証に合格しません。
  • 権限 -アドインのドキュメントデータへのアクセスレベルを定義します。 Restricted、Read document、Read all document、Write document、Read write documentなどの値をとることができます。
  • SourceLocation -Webサイト上のアプリケーションの「ホーム」ページへのパス。


Webサイトは、アプリケーションが動作するために必要なHTML、JavaScript、およびCSSファイルの最小セットで構成され、デフォルトでは、新しいソリューションのUIを構築できる基本UIを提供します。 サイトの要件の1つがHTTPSで機能していることは注目に値します 。 つまり、独自のサーバーまたは独自のドメインでサイトを公開する場合は、SSL証明書が必要です。 たとえば、Azure Webサイトを使用する場合は、この問題を回避できます。 azurewebsites.netサブドメインに展開されたすべてのサイトは、HTTPとHTTPSの両方を使用してデフォルトでアクセスできます。



JavaScript APIは、Excelデータと対話するためのメソッドの標準セットを提供します。 XLTools.net Data Cleanupアドインの開発時に使用したもののうち、ごく一部の使用例を示します。



  • Excelでユーザーが選択したセル範囲に「バインド」を追加して、さらに操作します。 



 Office.context.document.bindings.addFromPromptAsync(Office.BindingType.Matrix, { id: "RangeToClean" }, function (asyncResult) { if (asyncResult.status == "failed") { // Some specific code here } else { // Some specific code here } });


  • 以前に作成された「バインディング」を使用して、セル範囲からデータを取得します。 


 Office.select("bindings#RangeToClean", onBindingNotFound).getDataAsync( {}, doDataCleaning );


  • 以前に作成した「バインド」を使用してセル範囲のデータを更新します。 


 Office.select("bindings#RangeToClean").setDataAsync(range,function (asyncResult) { if (asyncResult.status == "failed") { // Some specific code here } else { app.showNotification(UIText.ExecutedSuccessfully, '', 'success'); } }).


JavaScript APIメソッドはすべて十分に文書化されており 、その詳細な説明はMSDN Webサイトにあります。



シナリオに応じて、データ処理はクライアント上で直接行われます。 JavaScriptコード内およびサーバー上。 サーバー上のデータを処理するために、たとえばWeb APIを使用して、必要なサービスをサイトに直接追加できます。 クライアント（アドイン）は、 AJAXリクエストを使用して、他のサイトで使用したのと同じ方法でWebサービスと通信します。 考慮すべき唯一のことは、外部ドメインにあるサードパーティのサービスを使用する場合、同じ起源のポリシーの問題が必ず発生することです。



アドインをOfficeストアに公開する



Officeストアでアドオンを公開するには、 Microsoft Seller Dashboard Webサイトで登録する必要があります 。 登録後、個人アカウントにアクセスし、アプリケーションのマニフェストをダウンロードして、必要な情報をすべて記入できます。 個人的な経験に基づいて、承認のためにアプリケーションを送信した後のアプリケーションのチェックには、通常1〜3営業日かかります。 Microsoftがアプリケーションをテストして承認した後、Officeストアを通じて世界中の何百万人ものユーザーがダウンロードできるようになります。







結論



結論として、XLToolsアドオンは、VBA / VSTOテクノロジーに基づく既存のソリューションをOffice 365のクロスプラットフォームソリューションに変換する方法の優れた例であると言っておく価値があります。 6つの個別のアプリケーションを実装します。



現在、それらはすべてOfficeストアからダウンロードできます。



  • XLTools.net SQLクエリ-Excelテーブルのデータに対するSQLクエリの実行。
  • XLTools.net Excel用CSVエクスポート -目的の区切り文字（カンマ、セミコロン、またはタブ）を含むCSVファイルとしてExcelでテーブルを保存できます。
  • XLTools.netデータクリーニング- データアレイクリーニング：スペースの削除、テキストの大文字小文字の変更、テキストの数字への変換など。
  • XLTools.net Excel用アンピボットテーブル-Excelユーザーが複雑な2次元テーブルをフラットビューに変換するのに役立ちます。
  • XLTools.net SendGridレポート-SendGridアカウントからExcelに配信レポートをダウンロードします。
  • XLTools.net列の一致 -列の比較、同じデータを持つ列の検索、列で一致するデータの割合の計算。


また、通常のシナリオに加えて、OfficeストアとOffice 365の出現により、開発者として、 Office 365 APIを使用して拡張機能を開発する新しい機会があります 。これにより、 メール 、 カレンダー 、 SharePoint Online 、 OneDrive for Businessなど これらの機会を利用して明日何を構築できるかを誰が知っていますか。 時間が経てばわかる！
