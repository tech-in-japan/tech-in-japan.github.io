「私の最も重要なテストは、ドライバーを壊さないことです」-PostgreSQL用JDBCドライバーの開発に関するDave Cramer

現在の対話者は、PostgreSQLの主要な貢献者の1人であるDave Cramerであり、15年以上にわたってPosgresのJDBCドライバーの作成者および保守者です。



デイブは公の場ではあまりパフォーマンスをしません。 幸運なことに、彼はPG Day'17ロシアに来て、pgjdbc、ÁlvaroHernándezTortosa、およびVladimir Sitnikovの同僚とともに、 PostgreSQLでJavaを最適化および効率的に使用するワークショップ/マスタークラスを開催することに同意しました。 ドライバーの隠された機能に関する Daveのレポートもおもしろいです。



会話は面白かった。 デイブは非常に簡潔な会話学者で、簡単に、そしてビジネスについて答えます。 JDBCドライバーの開発における現状と、このプロセスにおけるDaveの役割について徹底的に話すことができました。 また、ストアドプロシージャを開発するためのJavaの使用に関する見解と、国際的なPostgreSQLコミュニティの現状に関する彼のビジョンを共有しました。 今後のマスタークラスの従来の発表を無視しませんでした。







PG Day：デイブ、あなた自身について簡単に教えてください。あなたは誰ですか、あなたは何をしますか、あなたは自由な時間をどのように過ごしますか？



Dave ：こんにちは、私の名前はDave Kramerです。 2000年から PostgreSQLで15〜16年間働いています。ここでの活動の主な分野は、以前はJavaプログラマーだったのでJDBCドライバーでした。 また、pl / Rなどのいくつかの手続き言語にも取り組みました。 PostgreSQLに興味があるのは何ですか？ コミュニティプロモーション、その開発。



個人的なものに関しては、私には2人の子供、1人の妻、2匹の犬、2人の孫がいますが、それが起こるとは思いもしませんでした。



私はカナダに住んでいて、冬にはフロリダに行くこともあります。 空き時間に関しては、トラックで車を運転するのが好きです。 私はいつもできるだけ早く走ります。 もっと速く加速できると思いますが、自分のスキルは私を制限します。



PG Day：PostgreSQL用のJavaドライバーをどのように、そしてなぜ始めましたか？



Dave ：1999-2000年、オープンソースソフトウェアは現在ほど一般的ではありませんでした。 その瞬間、私は個人の請負業者になり、自分のために働きました。 マイクロソフトのテクニカルサポートに電話すると（クライアントがいました）、問題が発生し、彼らがそれを解決することを望みました。 彼らは答えた：「私達は3週間以内に連絡します。」 3週間は仕事がなかったので、少し気になりました。 私は他のオプションを検討し始め、 オープンソースに出会いましたが、それについては何も知りませんでした。 私は人々にこれを理解する方法を尋ねました。 そして私はメーリングリストの質問に答えるように勧められました。 私はそれをやった。 最初の質問には2、3日間答えました。 2番目の方法では、簡単に理解できました。 そして、私はゆっくりと彼らにかなり早く反応し始めました。



当時、別の人がJDBCドライバーの作業をしていましたが、突然彼にとって面白くなくなり 、私はいつも目の前にいたので、 Bruce Momjianが私にそれを勧めました。 そして、私はイエスと答えました。 それは15〜16年前と現在でした。



PG Day：今でもコードベースを更新していますか？



Dave ：むしろこれを他の人に委任します。 開発を管理します。 時々私は長い間ドライバーで仕事をしていないが、他の人は私のためにそれをやっている。 まるでバックグラウンドにいるかのようです。 他の人は私よりも仕事をしています。 私の主な仕事はプロセスを管理し、誰も何も壊さないようにすることです。 誰もが自分の部分でコードを改善しようとしていますが、誰もが常にグローバルに考えているわけではありません。



私の計算によると、 数千の企業がJDBCドライバーに依存しています 。 そして、オプションを追加すると、すべてが500人でカバーされるため、多くの人々が苦しむことになります。 だから私はむしろプロジェクトマネージャーです。



ちょうど今日、私はいくつかのバグを修正したので...ところで、ロシア人はたくさんコーディングしています。 たとえば、 ウラジミール・シトニコフ 。 彼は多くのコードをドライバーに導入しました。



PG Day：現在、何人がドライバーの積極的な開発に関わっていますか？ ワークフローはどうなっていますか？



デイブ ：率直に言って、 十分な人がいません 。 それらの数は時々変わります。 特定の問題を誰が解決したいかによって異なります。 ごく最近、英国の男がMavenのドライバーを完全に書き直しました 。 ウラジミール・ゴリャチョフ、彼の名前のようです、彼はドライバーへの論理デコードのためのコンポーネントを書きました。 ウラジミール・シトニコフは、ドライバーの最適化に多くの時間を費やしました。



ドライバーで作業している人はたくさんいますが、彼らは現れたり消えたりします。 そして、さらに多くの人々を引き付ける方法を見つけようとしています。



PG Day：開発を後援し、イニシアチブを支援し、プロジェクトを改善するために人々にお金を払おうとしている大企業はありますか？



デイブ ：私は誰も彼らの助けを提供し、ドライバーにお金を払うのを見たことがありません。 ウラジミール・シトニコフは彼の会社（ Netcracker-ed。 ）から支払われていると思います。 しかし、私はよくわかりません。 Red Hatが何らかのパッケージング作業を後援したように思えます。 私が彼らのために働いていた当時のPivotalは 、私がドライバーで働くことを気にしませんでした。 オープンSCGも気にしませんでした。 しかし、会社がドライバーに関する誰かの仕事を後援するために...これは、起こらなかったようです。



PG Day：キュレーターおよびプロジェクトマネージャーとしての役割を辞任する予定はありますか？ 今後の予定は？



Dave ：辞めるつもりはない。 将来については考えていませんが、仕事を辞めるつもりはありません 。 他の人が私を助けている間、私はドライバーをモニターし、開発を管理し、バグを修正します。 いいえ、近い将来、私はこれを続けるつもりです。



PG Day：最新のJDBCの改善点のうち、PostgreSQLに関して最も興味深い、印象的なものはどれですか。



Dave ：ドライバーの目の前で起こった最も驚くべきこと- 論理的な複製があります。 なぜこれがそんなに重要なのですか？ これで、変更データキャプチャを収集できます。 きれいなJavaコードを記述します 。 それまでは、トリガーを作成し、何らかのアルゴリズムにデータを送信し、ファイル、キューなどを作成し、非同期でデータを読み取ってからコードを更新する必要がありました。 アプリケーションを更新します。



これで、ドライバーに論理複製が組み込まれました 。 データが変化している間、リアルタイムで更新できます。 これを大きな革新と呼びます。 これは過去の作業を損なうものではありませんが、ドライバーを高速化するためです。 通常、2つのことがあります。 最初に、 ドライバーを高速化し、 2番目に、ドライバーに慣れていない人でも読みやすく、作業しやすいようにコードを完全に書き直しました。 ここに2つの最も重要なものがあります。 論理複製に加えて。



PG Day：ドライバーアーキテクチャに修正したい問題はありますか？



Dave ：ドライバーに欠けているのは、 ユーザータイプを正しく処理できることだけです。 これをドライバーに追加したいのですが、このオプションはあまり要求されません。



私が知る限り、すべてはドライバーに対して正常であり、比較的うまく機能しています。 そして、アーキテクチャでは、すべてがうまくいくようです。



PG Day：カスタムタイプのサポートの欠如にどのように対処しますか？ 問題の解決策はありますか？



Dave ：PostgreSQLには、このタスクを実行する別のドライバーがあります。 それはpgjdbc-ngと呼ばれ、新世代のドライバーです。 それは注目を集めていませんし、ユーザータイプはPostgreSQLが使用されているコミュニティでよく使用される要素だとは思いません。 Javaを使用するほとんどの人はHibernate、Springテクノロジーを使用しており、ほとんどの場合、データは非常に単純です。 彼らは選択を行い、データを挿入し、削除します。基本的にはそれだけです。



私の意見では、1つのポイントがあります-そして、それはあなたが以前に尋ねた質問に関連しています-人、企業はドライバーに取り組むためにお金を費やしています... JDBCドライバー（および他のドライバー）の作成者にとって、私たちにとって最大の困難は何ですか？ ドライバーは車の車輪のようなものです。 人々は車を購入し、車輪が付いていると期待しています。 それらは丸く、機能はほとんどありませんが、車を動かします。 しかし、車輪が回転している間、人々はそれらについて気にしません。 これはファッショナブルではありません。 人々はハッカーのリストでこれについて議論するのに何時間も費やしません。



私の最も重要なテストは、ドライバーを壊さないことです。



PG Day：ストアドプロシージャを記述するための言語としてのPostgreSQL内のJava-それは良い考えですか？ そのようなツールの典型的な用途は何ですか？



Dave ：はい、 PL / Java -PostgreSQLのストアドプロシージャ言語としてのJavaと呼ばれる技術があります。 問題は、 PostgreSQLがプロセス指向であることです。 Javaストアドプロシージャ言語で記述して実行するたびに、接続内でJVMを起動する必要があります。



これはかなり高価なプロセスになります。 したがって、Javaで記述できる場合にのみ、Javaからストアドプロシージャの開始を使用するようにする必要があります。 そして、JVMに費やされるスタートアップコストに見合う価値があります。 これは、他の言語、たとえばPL / Python 、特にpl / Rで確認できます。 これらすべての言語を使用して、データ処理および分析の専門家が使用する言語であるJVM、pl / RまたはPythonで重要なデータ処理を実行したいと考えています。



PL / Javaは良いアイデアだと思いますか？ 私はこれをあまり確信していない。 しばらくの間、接続間でJVMへの共有アクセスを提供するpl / Jという別のプロジェクトに従事していましたが、誰かがこのプロジェクトを復活させれば嬉しいです。 個人的に、私は彼のための時間を持っていませんでした。 コードはまだそこにあります。 これはかなりトリッキーです。 Oracleは、データベース内のJavaに関してかなりの作業を行っています。 Oracle内のJVMは、私たちと同じJVMではありません。 これらのタスク専用に最適化された個別のJVMがあります。 彼女にはPostgreSQLのようなスタートアップコストはないと思います。 この場合、Javaでしかできないことがあれば、正当化されます。



PG Day：Javaがエンタープライズコアテクノロジーの1つであることは秘密ではありません。 PostgreSQLは、その周りに一連の企業テクノロジーを構築するための良い選択だと思いますか？



Dave ：はい、と言うでしょうが、これには個人的な興味があります。 ここでの主な問題は価格だと思います。 最小限のコストでPostgreSQLを使用して開発するのに障壁はありません。



PostgreSQL開発者が直面するより深刻なテストは、テクノロジーの実装と、人々にそれを信じさせる試みです。 北米では、PostgreSQLに対する深刻な関心に気付きました。 人々がプロジェクトのコストにもっと注意を払うようになったので、関心が高まりました。



商取引における重大な困難は、大企業が現在管理している膨大な数のデータベースであるように思えます。 多くの場合、何千ものデータベースがあることを耳にします。つまり、データベースのコアごとに支払うと、非常に高価になります。 特に今では、誰もがサービスを拡大し、マイクロサービスアーキテクチャを導入する際に、独自の技術ソリューションを使用することははるかに高価になっています。



PG Day：過去5〜10年間にコミュニティで過ごした変化に気づきましたか？ どの方向に動いているか理解できますか？



Dave ：PostgreSQLは過去5年間で人気が高まっています。 PostgreSQL自体のコードを書いている人の数と同様に、それに取り組んでいる人の数は増加しています。 私が始めたとき、かなりの数の人々がコードを書いたが、実際にそのコードを使ったのは4-5人だけだった。 今日、私たちはすでに約10の会社を所有しており、それぞれに10人の従業員がいます。すべての会社がプロジェクトコードに多大な貢献をしています。



場合によっては、コードの一部に相互の互換性がないこともあります。 私たちのグループが直面している主な仕事は、これらの問題を解決することです。 また、ユーザーに最適な機能を提供するために、コミュニティの正しいパスを選択します。 多くの機能は、それぞれの会社によって管理されています。 顧客は、このオプションまたはそのオプションが必要な理由に個人的な関心を持っています。 これはおそらく私たちにとって最も難しい仕事です。
