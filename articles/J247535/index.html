FPGAでテトリスを作成する

みなさんこんにちは！



この長い新年の週末、私は疑問に思った：ディスプレイへの出力とキーボード制御を備えたFPGA上に簡単な小さなおもちゃを書くのはどれほど簡単か。 そこで、テトリスの別のFPGA実装が生まれました： yafpgatetris 。





もちろん、FPGAのゲームは、実際の「プロダクション」タスクよりも楽しくてトレーニング用に作られており、ゲームの「開発」とはほど遠いです。これは私にとって新しい経験です。





オペレーティングシステムなしでゲームを実行する方法に興味がある場合は、トリガーと組み合わせロジックを使用して最下位レベルでゲームを実装し、catへようこそ。





Devkitについて



ゲームが始まる「何か」が必要です。 最も簡単な方法の1つは、開発キットを使用することです。開発キットには、FPGAと入出力用の周辺機器があります。 私が自由に使えるのは、 DE1-SoCと呼ばれるTerasicのショールでした。











何が言えますか？

開発キット、開発キットのような。 周辺機器の多く：それからPS / 2とVGAコネクタは私たちにとって興味深いでしょう。 （学校や大学での）トレーニングのためにまさにそれ。 私たちの目的のために、「生産」のアイデアの一部を実装するためではなく、遊んで（そして学生を教育するため）に購入しました。

実際のデバイスで（LEDを点滅させるだけでなく）突然DE1-SoC（または同様のボード）を使用する場合（コメントを共有してください）、興味深いものになります。



チップ名のSoCは、チップに通常のFPGAロジックとARMプロセッサの両方があることを意味します。 今後は、ARMやその他のソフトウェアプロセッサのいずれも使用しなかったため、ボード上で他のFPGAチップを使用してプロジェクトを実行できます。 FPGA + ARMを大量に調達すること、およびこれから得られるボーナスについて興味がある場合は 、同僚のDes333の 記事を参照することをお勧めします。



何を手に入れたいですか



テトリスの概念にはさまざまなものを投資できるため、おおよそのTKをスケッチしました。

  • 数字の標準セット。 それらの振る舞いは、通常と同様でなければなりません。
  • ゲームはカラフルです。 各図には独自の色があります。
  • 図は、均一な分布でランダムに生成されます。
  • 次の図が表示されるウィンドウがあるはずです。
  • ゲームの状態に関する情報があるはずです：ポイントの数、削除された行の数、現在のレベル。
  • ポイントは「プログレッシブ」スケールで授与されます。一度に削除されるラインが多いほど、ポイントが多くなります。
  • レベルが高いほど、数字の落下速度が大きくなります。
  • 「ゲームの終了」が正しく検出されると、新しいゲームを開始することができます。
  • ユーザーアクションはキーボードから入力されます（PS / 2）。
  • フィールドのステータスなどの表示は、VGAインターフェイスを介して従来のディスプレイで行われます。




プロジェクト概要







3つの主要な部分を区別できます。

  • ユーザー入力。 キーボードからのデータを受け入れ、システムで「実行」します。
  • ゲーム自体に関連するすべて。 実際、FSM（有限状態マシン）は、プレーヤーから「要求」を受け取り、「すべてを実行」します。新しい図形を生成し、それらを移動し、線を削除します。
  • ゲームのステータスを表示します。 VGAインターフェイスを介してディスプレイに描画します。

イベントのタイプを判別するには、シフトレジスタと小さなプリミティブロジックを使用して、他のすべてのキーをスキップします。 このゲームの一環として、キーストロークに関するイベントを検出し、リリースを無視するだけで十分であると判断しました。 キーを押したままにすると、キーが押されたというコマンドが表示されるため、プレーヤーはフィギュアを動かしたり裏返したりすることを望みます。 数字があまりにも珍しく動くのではないかと心配していましたが、何も起こりませんでした。



興味のある「イベント」を検出した後、FIFOに入れ、そこからゲームの「適用されたロジック」がそれを取得します。 ボードにPS / 2がなく、いくつかのキー、トグルスイッチがある場合、これらのボタンを「イベント」にプッシュするロジックを記述するだけで、ゲームは何も認識しません。



このコントローラーを使用すると、マウスを接続できますが、試したことはありません。

FSMは、次のブロック/モジュールと「通信」します。

  • gen_sys_event-数字を自動的に下に移動する必要がある時間をカウントするタイマー。
  • gen_next_block-新しい図の生成。
  • check_move-現在の「移動」が可能かどうかを確認します。
  • tetris_stat- 「統計」の蓄積。
  • user_input-ユーザーが「生成」したイベントを読み取ります。


すべてがC ++ / Java /などで記述された「通常の」テトリスの実装に非常に似ています。さまざまなモジュールがこれらの言語の関数として機能します。 はい、同じ問題が発生します。私がフィギュアのクーデターで最も長く座ったとき、その答えはクアドラパッセルのコードに見張られていました 。 1つのオプションは、可能なすべてのスプレッドのテーブルを保存できることです（各図には4つのオプションがあります）。



すべてのコードはVerilogで記述されており、より正確にはSystemVerilogで記述されています。 一方では、SystemVerilogはVerilogよりもはるかに柔軟性があり、他方では、制限がなく、より多くの異なるものを実装したいという事実につながります:)。

ディスプレイ



FPGAを使用して、 ハブなどのVGAを介してディスプレイに画像を表示する方法については多くの情報があるため、これについては詳しく説明しません。



このクジラでは、VGA出力は次のように作成されます。





VGA_CLKの各サイクルでは、RGBモデルで新しいカラー値を設定する必要があり、DACはこれらの値を目的の信号レベルに変換します。



VGA信号のコントローラーとして、このキットのCDにあるデモ例からモジュールを取りました。 CDの概念があるのは面白いですが、ボードにはCDが含まれていません。インターネットからアーカイブをダウンロードする必要があります。



この「コントローラー」Terasicは他のクジラで使用します。「vga_time_generator」という名前で簡単にグーグルで検索できます。 任意の動作モード（ 640x480、800x600など）に構成でき、表示用に現在のピクセルの座標（ pixel_x 、 pixel_y ）を提供できるという点で便利です。 私たちのタスクは、これらの座標に応じて必要な色の値を置き換えることです。







大きなモニターの640x480は見栄えがよくないと判断し、1280x1024に移行しました。必要な値を標準からモジュールに渡すだけです。 さらに、 VGA_CLK値を変更する必要がありました。25.175MHzではなく、108 MHzになりました。 確かに、後で少し後悔しましたが、美しさには犠牲が必要です。

多くのプロジェクト（ネットワーク上にあります）では、そのようなテーブル（ Font ROM ）がVGAで使用されますが、640x480ディスプレイ用に設計されています：1280x1024では小さすぎるため、同様のテーブルを「大きい」フォントで準備する必要があります。



nafeユーティリティはこれを助けてくれました。 入力で、psfファイル、出力で、描画する必要のあるピクセルのX付きテキストファイルを受け取ります。 お気に入りの言語を使用して（または元のプログラムの出力を少しやり直し）、Xを「1」に、スペースを「0」に変更し、ヘッダーを追加してmifファイルを作成します（ROMの初期化に使用されます）。



私のpsf形式で見つけた最大のフォントは32x16で、原則としてこのタスクには十分でしたが、もう少し作りたいと思います。 私が理解している限り、制限はありません。このユーティリティを使用すると、任意の文字（ロシア語の文字など）でROMを準備できます。
