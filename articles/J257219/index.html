民間部門（カントリーハウス）では、たとえば見知らぬ人が現場に現れた場合など、家主が事件の可能性をできるだけ早く通知することが重要です。



インターネットを介したアクセス可能なビデオ監視システムでは、ユーザーはアーカイブ内のコンテンツに十分な検索ツールを持っていません。 ビデオを視聴したり巻き戻したりすることなく、その日のすべてのイベントをすぐに見るにはどうすればいいですか？ 子供は学校から帰宅しましたか？ 庭師は田舎で何をしましたか？ 駐車場で車に傷をつけたのは誰ですか？



リストされた検索機能の一部は、たとえばITVやMilestoneなど、ビデオ管理システム（VMS）で現在利用可能ですが、その使用はプロのセキュリティサービスユーザーに限定されています。 彼らのアーキテクチャ、Webインターフェース、モバイルアプリケーションは、大衆消費者（アパートやカントリーハウスの居住者）に対するビデオ分析の可能性を明らかにしていません。



ビデオ監視からビデオ分析まで



これらの考えを念頭に置いて、私たちはアパートの建物とコテージの集落でのビデオ監視の「情報化」のための2つの解決策を考え始めました。





Kipodクラウドサービスに基づくリモートホームビデオ監視



分散アーキテクチャ



図に示すように、ソリューションはKipodリモートモニタリングWebサービスに基づいています。 記録および分析デバイス（ Kipod Server ）は、施設、アパート、または民家に直接配置されています。 ネットワークカメラがデバイスに接続され、オプションでコンシェルジュ/ガードワークステーションとスピーカーが接続されています。 インターホンとエレベータのアナログカメラは、変換（IPエンコーダー）を介して接続できます。



分散ソリューションの重要な利点は、通信チャネルの最小負荷と、通信チャネルが過負荷または利用できない状況での各オブジェクトの自律的な動作の可能性です。 チャネルの復元後、イベントは自動的にクラウドサービスと同期されます。 したがって、分散アーキテクチャソリューションは、3Gワイヤレスチャネル上でも数十台のカメラで動作します。



集中型アーキテクチャ



別のアプローチは、すべての処理とストレージがクラスタービデオ分析サーバーに基づいてサービスプロバイダーのデータセンターに実装される、完全に集中化されたアーキテクチャです。 一元化されたアプローチの利点は、実装の容易さと機器のメンテナンスコストの削減です。 短所-チャネル帯域幅の要件が高くなり、その結果、カメラの数とビデオ録画の品質が制限されます。

4.バーチャルコンシェルジュ



仮想コンシェルジュのタスクは、ビデオ分析およびオーディオ分析Kipodの作業から生じるアラームイベントで24時間動作することです。 仮想コンシェルジュオペレーターは、Sinesis Dispatch Centerに配置されているか、リモートで作業しています。 オペレーターが犯罪行為や秩序違反を確認した場合、規制に従って一連の所定のアクションを実行します（警察に電話をかけ、住民と連絡を取り、音声通信で音声メッセージを送信します）。 仮想コンシェルジュサービスは、後で追加料金で開始する予定です。
