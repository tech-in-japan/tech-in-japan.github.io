繰り返しになりますが、Huawei Cloud Fabric：SDNアーキテクチャとは何ですか？





少し前、 Huawei Cloud Fabric SDNソリューションとVMware vSphereプラットフォームの統合について話し、この統合がもたらす利点について話しました。



今回は、ソフトウェア定義ネットワーク（SDN）、その使用から得られるビジネス上の利点、およびデータセンター向けのHuawei Cloud Fabric SDNソリューションにさらに注意を払いたいと思います。



業界がそのルールを決定します



企業は、仮想化とクラウドテクノロジーを使用して、効率を高め、プロセスを改善し、競争上の優位性を獲得しています。



企業のデータセンターは、仮想化に基づいたクラウド環境の方向に進化しています。仮想化は、何らかのノウハウではなく、ビジネスにとって必要不可欠なものになっています。



論理チェーンは非常に単純です。



1.管理およびオーケストレーションの自動化と組み合わせたコンピューティングおよびストレージリソースの仮想化により、ワークロードの仮想化が可能になりました。



2.次に、ワークロードの仮想化により、ITシステム管理者およびアプリケーション開発者は、データセンターでサービスおよびアプリケーションの新しいインスタンスを迅速に作成できます。



ネットワークを通じて独立したワークロードと関連サービスを接続する際のスループットの向上は最近までの主な「成長ポイント」の1つでしたが、コンピューティングとデータストレージ用の仮想化リソースの開発ペースの向上により、新しいゲームルールが規定されています。



ソフトウェア定義ネットワークの出現により、ネットワークパスを決定および構成し、個々のデバイスからネットワークコントロールプレーンを外部化する、より柔軟なシステムが登場しました。 ネットワークの論理トポロジを決定するこの「外部」コントロールプレーンは、ネットワークの拡大図を表し、環境の運用面に適合した基準に基づいて柔軟なネットワーク最適化を提供します。



外部統合の観点から、このSDNコントローラーは、システムがデータセンターの他のコンポーネントと連携して連携できるソフトウェアインターフェイスにより望ましいものです。



ファーウェイが責任を負います



HuaweiのデータセンターSDNソリューションであるCloud Fabricは、あらゆる規模のデータセンター環境で動作するように一方向または別の方向に拡張できる使いやすく、非常にオープンなSDNアーキテクチャを企業に提供します。新しいアプリケーションとサービスの迅速な展開。



システムの知的センターはアジャイルコントローラーです。 HuaweiのCloudEngineスイッチングプラットフォーム上に構築されたデータセンターネットワーク構造に対して、集中化された、非常に復元力のあるインテリジェントなコントロールプレーン機能を提供します。



Huaweiのデータセンター用アジャイルコントローラーは、オープンで標準化されたプログラミングインターフェイス（API）を介して一般的なサードパーティの産業用ソリューションとシームレスに統合できる完全なオープンソースソリューションとして設計されています。



一般的なデータセンターインフラストラクチャ、特にクラウドファブリックSDNソリューションに対するソフトウェアベースのネットワークアプローチがもたらす主な利点は何ですか？



そして最も重要なことは、これらのメリットがビジネスにどのような影響を与えることができるかということです。



内部プロセス



まず、ソフトウェア定義型ネットワークの出現により、ネットワークの変更または複雑なサービスやアプリケーションを展開するための再構成に応じて、データセンターのITプロセスに明確な改善がもたらされました。



以前は、ネットワーク計画、新しいアプリケーションインスタンスの設計、およびサポートに必要な変更の準備は、手動またはブルートオートメーションのいずれかで実行されていました。 しかし、SDNの開発のおかげで、コンピューティングとデータストレージの仮想化プラットフォームを含むネットワークオーケストレーション機能の新しいレベルの統合された自動化、準備と管理が利用可能になりました。



新しいアプリケーション用にネットワーク要素とコンポーネントを準備するために必要な時間とリソースの増加は、5〜30に異なります。



さらに、SDNと他のすべてのデータセンターリソースとの統合は、真にプライベートなクラウドベースのセルフサービスプラットフォームを作成するための基盤を形成し、エンドユーザーが企業内でクラウドサービスを独立して要求および提供できるようにします。 手動プロセスの量を減らすと、運用上の過負荷が軽減され、市場での新しいアプリケーションとサービスの立ち上げが加速し、導入コストが削減されます。



さらに、Huawei Cloud Fabric SDNソリューションによって提供される改善された運用制御（ビッグデータ分析と組み合わせた幅広いネットワーク全体のテレメトリ収集のおかげ）は、要求の厳しい運用および保守タスクでより高いレベルの自動化を提供します。



Agile Controllerによるネットワークステータスの大規模かつ高レベルで非常に詳細なビジョンとインテリジェントな分析により、エラーや障害の結果をより迅速かつ効率的に特定、分離、排除できます。 稼働時間の直接的な増加は、ビジネスへの悪影響を減らし、人的資源の必要性を減らします。



資源効率



データセンターのITプロセスの改善により、組織全体の効率が自然に向上します。



1.運用効率の向上に加えて、SDNはネットワークが独立して内部バランスを提供し、インフラストラクチャを通過するトラフィックフローを最適化するように調整できるようにします。



同時に、自己最適化の可能性は、ローカルデータセンターネットワークだけでなく、Huawei SDNアーキテクチャを使用する場合のデータセンター相互接続（DCI）にも影響します。



2.さまざまな地理的場所に分散された企業クラウドリソースを結合することに加えて、適切に最適化されたDCIは、利用可能なリソースの利用率を大幅に高め、30％から平均90％に増やします。



サーバーの仮想化が示しているように、リソース効率の向上は運用コストを直接削減し、データセンターリソースのライフサイクルを大幅に向上させます。



競争力の向上



HuaweiのSDNソリューションを使用して企業データセンターにもたらされた速度と柔軟性は、サービスとアプリケーションの配信メカニズムからサービスとアプリケーション自体に技術的な焦点を移す環境を作成し、したがって、新しいアプリケーションを市場に投入するのにかかる時間を短縮します。 これらのアプリケーションまたはサービスは、エンドカスタマーのエクスペリエンスを向上させるプロセスの改善に貢献する内部、またはカスタマーとの相互作用と関係を豊かにする外部のいずれかです。 プロセスから機能への焦点のシフトにより、プラットフォームはユーザーエクスペリエンスを迅速かつ革新的に開発することができ、業界に関係なく全体的な競争力に弾みをつけます。



オープンソリューション



そのオープン性と統合機能により、HuaweiのSDNソリューションは業界をリードしています。 これはビジネスにとって何を意味しますか？



真にオープンなSDNソリューションは、ユーザーに標準APIを介したインターフェイスレベルでの基本的な統合を提供するだけでなく、高レベルの機能性と信頼性を提供しながら、ソリューションアーキテクチャの交換可能な主要コンポーネントを提供します。



これは、標準化されたインターフェイスの単純な実装を超えています。あらゆる環境で固有の要件に合わせてカスタマイズできる、さまざまなメーカーのテスト済みおよび認定済みソリューションのアフィリエイトエコシステムの構築について話します。 SDNネットワークコントローラーやクラウドオーケストレーションプラットフォームなどのさまざまなネットワークコンポーネントに標準およびオープンテクノロジーに基づいた接続性を提供するオープンソリューションは、特定の要件への適応性を提供するだけでなく、システム全体の開発の重大な未来も提供します。



最後に



SDNの技術的な利点が明らかになるにつれて、このネットワークテクノロジーの開発がビジネス目標の達成に直接貢献する方法を理解することの重要性が高まっています。
