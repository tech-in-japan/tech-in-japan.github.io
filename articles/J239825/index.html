子供の頃からプログラムを書いています



1992年、8年生の頃、毎日放課後、コンピューターサイエンスルームに行きました。そこでは、高校生の実習が終わるまで座っていました。 彼らはBK-0010でBASICを研究しました。 筆者はすでにいくつかのライティングスキルを身につけていましたが、教師にとってはいつの間にか、高校生が問題を解決するのを助けました。 彼らの感謝に、それは私にとって喜びでした。



10年生のとき、私は初心者プログラマーのための通信学校に入学しました。 タスクはメール（紙！）で送られてきたので、プログラムのテキストをA4シートに書き、確認のために送信しました。 Pascal言語の学習は、本とリストのためだけに、コンパイラの不在下で行われました。



夏には、この学校でキャンプが開催されました。そこでは1週間半の間、休憩なしでプログラムを作成しました（すでにコンピューターで）。 毎日、朝、昼と夜、一度に1つずつ、チームで、次にペアで、さまざまな問題を解決しました。 理論的な研究はなく、実用的なタスクがありました。 多くのタスクとプログラム。 その学校の週は私に多くを与えました。 そこに自分の人生を捧げたいことを実現しました。

インタラクティブなタスク検証システム



私は小学生と多くのことを話しました、彼らがどのように持ち去られるか、そして彼らがドラムで何を得たかについてすでに知っていました。 はい、誰も理論を読みたくありません。 しかし、誰もが行動したいと考えています。



職業訓練学校で働いている間、私はプログラミングのための遠隔学習システムである「ロボット」の作成を始めました。 これは、順番にしか解決できないタスクのリストです。 作成されたプログラムの正確性は、事前に準備されたテストによってチェックされます。



検証のためにプログラムを送信した後、「ロボット」はアルゴリズムに従って動作します。

1.プログラムをコンパイルします。

2.エラーが発生した場合-コンパイラメッセージを返します。

3.条件からの例を使用してプログラムを実行します。

4.答えが一致しない場合、分析のためにプログラムの応答を返します。

5.生徒が知らない他のすべてのテストでプログラムをさらに数回実行します。

6.プログラムを評価します。正しいテストの数/テストの合計数* 100％。

7. 50％以上を獲得した場合-次のレッスンへの入学を許可します。



学童は何が好きですか



小学生は、プログラムが教師によってではなく、「ロボット」によってチェックされることを好みました。 これは「待機する必要も電話する必要もない」ことに加えて、より客観的ですが、確認のために何度でも送信して結果を得ることができます。 ほとんどの学生が「半分の解決策」に満足していないことに気づきました。私は、完全な100％に達するまで、複数の送信解決策を常に観察しています。 私の生徒たちが他の人のレッスンで私のサイトの問題を解決するようになったので、他の教師が私に不平を言ったのです。 これは成功だと思います！ :)



ちなみに、このシステムにより、学生の評価が非常に簡単になりました。 たとえば、「サイクル」セクションのすべてのタスクには10個のタスクがありますが、次のレッスンでは、得点に応じて成績を付けます。 初めて、それは12ユニットと2組のトリプルの雑誌のコラムでした。 さらに、通常はステークの提示を伴う心理的なドラマはありませんでした。 解決策はありますか？ いや これは見積もりです。 それだけです 一度だけで十分でした。 いくつかのレポートによると、専門学校の生徒は私のレッスンのためだけに宿題をしました...はい、それはクールな時間でした。 返品したいのですが、新しいレベルです。


方法論のもう1つの重要なポイントは、タスクへの一貫したアクセスです。 これにより、スプレーではなく特定のタスクに集中できます。 私はすべてのタスクへのアクセスを一度にオープンしようとしました-効率が一桁低下しました-学生は何をすべきかを知らず、その結果何も解決しませんでした。

ウェビナーの実施方法



ウェビナーのために、私は長い間会場を探していました。 私の目的のためには、次のものが必要です。



1.画面全体を歪みなく共有します。

2.画面と同期して、遅延なくサウンドをブロードキャストします。

3.適切な価格。



ここで試したすべてのサイトをリストするわけではありません。面倒な検索の後、join.meで私の選択が決まりました。 彼女はこれらの要件をすべて満たしています。 唯一の特徴は、すべての初心者がサウンドのチューニングの問題に直面していることです。そのためには、接続するプログラムを「呼び出し」てダウンロードする必要があります。 他のすべての点で-シンプルで高速かつ便利なプラットフォーム。 価格は年間200ドルです。 指定された要件に適した別のウェビナーサイトをお勧めする方がいらっしゃると思います。



2013年12月25日に開催されたトピック「SMSメーリング用プログラムの作成」に関する最初のウェビナー。 彼はすべての友人をそれに招待しました。 すべてが「万歳！」になりました。



画面と音声を記録するには、無料のoCamプログラムを使用します。非常に便利で、コンパクトで高速です。 録画したビデオレッスンをYouTubeにアップロードします。





ウェビナー中のデスクトップ：Visual Stuido、Join.Me、oCam。



私は自分のために次のスキームを思いつきました：

1.作成するゲームを選択し、自分で作成します。

2.作成に関するウェビナーを実施します。

3.ウェビナーエントリを保存して、独自のビデオコースを作成します。

当初、ウェビナーはくしゃくしゃにされ、多くの技術的な問題が発生し、わずか数人がウェビナーに来ました。 そのような記録を投稿するのは単に残念だったので、最初はビデオレッスンをウェビナーとは別に記録しました。 時間がかかりましたが、品質ははるかに高くなっています。



そのため、ビデオチュートリアルを作成するプロセスは既に確立されており、毎週のサイクルで正常に機能しています。

ウェビナー=>ビデオレッスン=>ウェビナー=>ビデオレッスン。

誰が作品をチェックしますか



最初は面白かった-宿題をすべて見て、質問に答えて、新しいプログラムを楽しみにしていた。 しかし、作品の数は増え、この活動は多くの時間になりました。 最も活発な生徒の一人が私を助けるためにボランティアをしてくれました。 彼は、コメント、質問への回答、コード分析など、この問題に非常に責任を持って対処しました。 それから彼は卒業証書を始め、再び私は山のような仕事に取り残されました。



どうする 解決策はありませんでしたが、自動アクセスを次のレッスンに設定しました。 何もないよりはましです。 しかし、これは私の方法論の原則、つまりレッスンへの一貫したアクセスと矛盾しています！ 結局のところ、あなたは何でも送って、新しいレッスンを得ることができます。 ダミーを再度送信し、再び先に進みます。 トレーニングではなく、空のエンターテイメント。



どうする？ 他の学生に確認を許可しますか？ 第一に、他の人のプログラムを見ることに興味のある人はほとんどいない（私が試した）、そして第二に、アーカイブでウイルスを送信するオプションは除外されていない。



それではどうしますか？ 数週間、この質問に悩まされました。 どうする？ 最後に、タスクのレポートは、スクリーンショットと、「どのくらい時間がかかった」、「最も難しかった」、「レッスンをどのように評価しますか」などの簡単な質問への回答で行う必要があります。 たとえば、10の肯定的なレビューが収集された場合にのみ、次のタスクに進むことができます。 プロジェクト参加者だけでなく、全員が作業を確認できるようにします。

今、私はすでに述べることができます：それは動作します。 検証プロセスは非常に簡単です。写真を表示し、レビューを読み、「良い」または「悪い」ボタンをクリックします。

ロボットシャープ



肺にさらに空気を取り入れて、超大型のインタラクティブなタスク検証システムから大量の塵を吹き飛ばして、プロジェクトに統合します。 このセクションにはコンソールタスクが含まれているため、「コンソール」と呼ばれます。



それはそれほど単純ではないことが判明し、Pascal言語とC＃言語の間に根本的な違いが多すぎるため、各タスクを適応させる必要がありました。 データ入力にさらに問題がありました。 Pascalでは、1行にスペースを入れて入力した2つの数値を読み取るには、次のように書き込むだけで十分です。 

readln (a, b);


C＃では、簡単なことではありません。 さまざまな形式の数字を読むためだけのタスクを含む別のトピックを作成する必要がありました。 C Sharp言語のPascal行の類似物は、私にとってはこのように見えます。 

  string [] line = Console.ReadLine().Split(); int a = int.Parse (line [0]); int b = int.Parse (line [1]);


言うまでもなく、プログラムをコンパイルおよびテストするためのすべてのアルゴリズムを再作成する必要がありました。 ただし、受け取りたいものがはっきりと想像できたので、これにはあまり時間がかかりませんでした。 そこで、学生の決定を検証するプログラムである「シャープロボット」がありました。



より詳しくは、シャープの仕事のプロセス-彼がタスクをコンパイルしてチェックする方法-は、この記事のために特別に記録したこのビデオで見ることができます。





「コンソール」セクションの動作を保証するために、Visual Studio C＃2008コンパイラーがインストールされているWindows XPを備えたVPSサーバーが使用されます。RobotSharpは、新しいソリューションをテストするMySQLデータベースに接続する同じC＃で書かれたプログラムです。

新しいプログラムが表示されると、Sharpロボットはそれをユーザーのフォルダーに保存し、コンパイルし、テストを実行し、テスト結果をテキストメッセージとともにデータベースに書き戻します。 これは、コンパイラエラーメッセージ、ゼロテストに合格したときのプログラムの誤った結果、テスト結果を含むテストのリストである可能性があります。 タスクが50％を超えるポイントを獲得した場合、それはカウントされ、学生は次のタスクに進むことができます。




ビデオ問題解決



このシステムを以前に作成したとき、レッスンの始めに理論的な資料を置くことを夢見ていました。 実際に、これらのテキストレッスンをいくつか作成しました。 しかし... ...誰も読んでいない。 しかし、誰もが常に多くの質問を抱えていました。「この問題を解決する方法は？ プログラムが起動しないのはなぜですか？」



したがって、私は生徒に彼らが望むものを与えることに決めました-問題解決。 しかし、完成したプログラムのテキストだけでなく、この問題がどのように解決されるのか、プログラムのコードを記述する方法を詳細に、そして明確に説明するビデオソリューション。 つまり、学生は自分でコードを手動でダイヤルする必要があります。少なくとも何かが頭に残り、量は遅かれ早かれ品質に変わります。 ビデオソリューションはめったに使用されないことを追加する必要があり、これは素晴らしいことです。 多くの人が自分でタスクを「終了」したいと考えています。



新しいデザイン



私のプロジェクトの生涯の6か月後、1024人以上の学生がすでになっています。 さまざまなゲームおよびアプリケーションプログラムの作成に関する32のビデオチュートリアルが既にあります。 少なくとも16人がウェビナーに参加しました。



基礎として採用された無料のサイトテンプレートは、まったく役に立たなくなりました。 プロジェクトの再設計を開始しました。 私は自分のサイトのスキームを少なくとも10回は手で書き直しました。





フリーランサーがBootstrapで私のサイトのテンプレートを作成しました。 美しく、シンプルで便利なデザイン、2つのメニューレベル、パンくずリスト、エントリ/終了ボタン。 すべてが美しいが、死んでいた。 あるデザインから別のデザインにプロジェクト全体を転送することは、別の都市に移動するようなものです。 実際、私はすべてを再び書き直しました。 そして、私はこれにとても満足しています、コードはより良く、より良くなったことが判明しました。





サイトの作成には、HTML、CSS、JavaScript、jQuery、Bootstrap、およびPHP、Smarty、MySQLのテクノロジーが使用されました。



以前は、すべてのビデオレッスンのテキストがPHPファイルに直接保存されていましたが、今ではデータベースに転送しました。 それだけでなく、サイトへの出版物の追加、編集、並べ替え、移動のプロセスを容易にするために、私は独自の小さなCMSを作成しました。 もちろん、システムは原始的であることが判明しました。これがすべての魅力です。これ以上、私が本当に必要なものはありません。



プロジェクト推進
