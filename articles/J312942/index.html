Swift 3の新機能

長い間知っているように、AppleはSwift 3をXcode 8に統合しました。これは、macOSとLinuxの両方で動作するオープンソース言語の最初のバージョンです。 昨年12月以降、Swift Evolutionで言語開発プロセスを実行し、IBMサンドボックスでそれを試してみた場合、おそらく多くの変更がそこに現れていることに気付いたでしょう。 Xcode 8で既存のプロジェクトをコンパイルするとき、間違いなくコードが驚くことでしょう。 しかし、それは修正可能です。 新しい言語バージョンでのいくつかの変更について知りましょう。





新しいバージョンの変更点は、主に2つのカテゴリに分類できます。



  • Swift 2.2で既に廃止されているリモート機能
  • 言語の改善


Xcodeバージョン7.3で警告を受け取ったときに理解しやすく、おそらくこれらの機能を満たしているため、リモート機能のカテゴリから始めましょう。



++および-演算子



C言語の遺産の増分および減分演算子とその機能は単純です-特定の変数に1を加算または減算します。 



var i = 0 i++ ++i i-- --i


ただし、どちらを選択するかを決定することになると、事態はより複雑になります。 これらはそれぞれ、プレフィックスとポストフィックスの2つの可能なオプションで表すことができます。これらはすべて、エンジンのおかげで機能し、演算子のリロードにより使用または拒否できる値を返します。



これは初心者には削除されているので十分です-代わりに代入演算子と加算+ =および減算-=を使用してください： 



 var i = 0 i += 1 i -= 1


もちろん、加算（+）演算子と減算（-）演算子を使用できるほか、代入演算子も一緒に使用できるため、コードを記述するときの時間を節約できますが、 



 i = i + 1 i = i - 1


慣れる： この変更の理由について詳しく知りたい場合は、 Chris Latnerの提案を参照してください 。



ループを記述するためのC言語スタイルは歴史



インクリメント演算子とデクリメント演算子を使用する最も一般的な例は、ループを記述するためのC言語スタイルです。 演算子を削除すると、それらに関連するすべてのものが削除されます。これは、式や演算子を使用して範囲を指定した場合とは異なり、これらすべてを行うことはできないためです。



たとえば、ある程度の知識がある場合は、おそらくforループを使用して1から10までの数字を表示します。 



 for (i = 1; i <= 10; i++) { print(i) }


Swift 3では、これは不可能です。 これは彼のSwift 3のプロトタイプです-動作中の閉範囲演算子（...）を見てみましょう： 



 for i in 1...10 { print(i) }


あるいは、閉じた式と省略された引数でfor-eachループを使用することもできます 。詳細については、 こちらを参照してください 。 



 (1...10).forEach {  print($0) }


慣れる： この変更の動機について詳しく知りたい場合は、 Erica Sadunの提案を参照してください 。



関数パラメーターから変数を削除しました



関数パラメーターは、関数内で変更する必要がないため、通常定数として定義されます。 ただし、変数として宣言すると便利な場合があります。 Swift 2では、 varキーワードを使用して関数パラメーターを変数としてマークできます。 パラメーターをvarとして指定すると、値のローカルコピーが作成されるため、関数の本体で値を変更できます。



例として、次の関数は、与えられた2つの数値の最大公約数を決定します。高校で数学のコースを逃した場合は、ここで詳細をお読みください。 



 func gcd(var a: Int, var b: Int) -> Int { if (a == b) { return a } repeat { if (a > b) { a = a - b } else { b = b - a } } while (a != b) return a }


アルゴリズムは単純です。両方の数値が既に等しい場合、どちらかが返されます。 それ以外の場合は、それらを比較し、大きい方から小さい方を減算し、結果が等しくなるまで大きい方に割り当てて、いずれかを返します。 ご覧のとおり、aとbを変数としてマークすることにより、関数内の値を変更できます。



Swift 3開発者は関数パラメーターを変数として設定できなくなりました。Swift開発者はvarとinoutを混同する可能性があるためです。 したがって、Swiftの最新バージョンでは、関数パラメーターからvarが削除されます。



したがって、Swift 3で同じgcd関数を作成するには、異なるアプローチが必要です。 ローカル変数の関数パラメーター値を保存する必要があります。 



 func gcd(a: Int, b: Int) -> Int {  if (a == b) {    return a  }  var c = a  var d = b  repeat {    if (c > d) {      c = c - d    } else {      d = d - c    }  } while (c != d)  return c }


この決定の理由について詳しく知りたい場合は、元の文を読むことができます 。



関数パラメーターのシーケンシャルラベルの動作



関数パラメーターリストはタプルです。そのため、タプル構造が関数プロトタイプと一致する限り、それらを使用して関数を呼び出すことができます。 例としてgcd（）関数を取り上げます。 次のように電話をかけることができます。 



 gcd(8, b: 12)


または、次のように関数を呼び出すこともできます。 



 let number = (8, b: 12) gcd(number)


Swift 2でわかるように、最初のパラメーターのラベルを指定する必要はありません。 ただし、関数を呼び出すときは、2番目（およびその他のパラメーター）のラベルを指定する必要があります。



この構文は初心者の開発者にとってわかりにくいため、ラベルの動作を標準化するように設計されています。 Swiftの新しいバージョンでは、次のように関数を呼び出すことができます。 



 gcd(a: 8, b: 12)


最初のパラメーターのラベルを明示的に指定する必要があります。 そうしないと、Xcode 8はエラーメッセージを表示します。



この変更に対する最初の反応は、「OMG！ 既存のコードに多くの変更を加える必要があります。」 あなたは正しい。 これらはたくさんの変更です。 したがって、Appleは、関数呼び出しパラメーターの最初のラベルを抑制する方法を提供しています。 以下に示すように、最初のパラメーターに下線を追加できます。 



 func gcd(_ a: Int, b: Int) -> Int { ... }


これにより、最初のラベルを指定せずに、古いメソッドを使用して関数を呼び出すことができます。 これにより、Swift 2からSwift 3へのコードの移行がはるかに簡単になります。

ボタンをクリックしてアクションを実行してみましょう。ストーリーボードではなく、プレイグラウンドのみを使用します。 



 // 1 import UIKit import XCPlayground // 2 class Responder: NSObject { func tap() { print("Button pressed") } } let responder = Responder() // 3 let button = UIButton(type: .System) button.setTitle("Button", forState: .Normal) button.addTarget(responder, action: "tap", forControlEvents: .TouchUpInside) button.sizeToFit() button.center = CGPoint(x: 50, y: 25) // 4 let frame = CGRect(x: 0, y: 0, width: 100, height: 50) let view = UIView(frame: frame) view.addSubview(button) XCPlaygroundPage.currentPage.liveView = view


かなり多くのイベントがそこで発生するので、それらを段階に分けましょう。



  1. UIKitおよびXCPlaygroundフレームワークをインポートします-ボタンを作成し、プレイグラウンドエディターで表示する必要があります。
  2. ユーザーがボタンをクリックしてターゲットを作成するときに実行されるクリック方法を決定しますか？ セレクタはObjective-Cメソッドでのみ機能するため、ボタンの論理オブジェクトはその基本クラスNSObjectです。

ハイライトされたコードを見てください。 ボタンセレクターは文字列です。 間違って入力すると、コードはコンパイルされますが、対応するメソッドが見つからない可能性があるため、実行中に失敗します。



潜在的なコンパイル時の問題を修正するために、Swift 3はセレクター文字列を#selector（）キーワードに置き換えました。 これにより、メソッド名を正しく指定しないと、コンパイラーが問題を早期に検出できます。 



 button.addTarget(responder, action: #selector(Responder.tap), for: .touchUpInside)

文字列としてのキーパス



この関数は前の関数と似ていますが、 キー値コーディング （KVC）およびキー値監視（KVO）に関連しています。 



 class Person: NSObject { var name: String = "" init(name: String) { self.name = name } } let me = Person(name: "Cosmin") me.valueForKeyPath("name")


エンコードのKey-Valueと一致するPersonクラスを作成し、イニシャライザーによって指定されたクラスで自分のIDを作成し、 KVCを使用して名前を設定します。



繰り返しますが、間違った操作をすると、すべてが爆発します！



幸いなことに、これはSwift 3では発生しなくなりました。キーパスは#keyPath（）式に置き換えられました。 



 class Person: NSObject { var name: String = "" init(name: String) { self.name = name } } let me = Person(name: "Cosmin") me.value(forKeyPath: #keyPath(Person.name))


この変更の理由を理解するには、David Heartsの提案を読んでください。



NSベースタイププレフィックスを削除



基本タイプのNSプレフィックスを削除しました。 仕組みを見てみましょう。 典型的な例はJSONでの作業です： 



 let file = NSBundle.mainBundle().pathForResource("tutorials", ofType: "json") let url = NSURL(fileURLWithPath: file!) let data = NSData(contentsOfURL: url) let json = try! NSJSONSerialization.JSONObjectWithData(data!, options: []) print(json)


基本クラスを使用して、ファイルに接続し、それに応じてJSON形式でデータを抽出できます。NSBundle-> NSURL-> NSData-> NSJSONSerialization。



NSプレフィックスはSwift 3では使用されないため、すべてバンドル-> URL->データ-> JSONSerialization（）になります。 



 let file = Bundle.main().pathForResource("tutorials", ofType: "json") let url = URL(fileURLWithPath: file!) let data = try! Data(contentsOf: url) let json = try! JSONSerialization.jsonObject(with: data) print(json)


この命名変更手順については、 Tony ParkerとPhilip Hauslerによって書かれたこの文を確認できます。



M_PI対.pi



与えられた半径の円と円の面積を計算しましょう： 



 let r = 3.0 let circumference = 2 * M_PI * r let area = M_PI * r * r


Swiftの古いバージョンでは、 M_PIを使用してpi定数を示します。 Swift 3は、定数piをFloat 、 Double 、およびCGFloat型に統合します 。 



 Float.pi Double.pi CGFloat.pi


上記のコードスニペットは、Swift 3で次のように記述されます。 



 let r = 3.0 let circumference = 2 * Double.pi * r let area = Double.pi * r * r


型推論を使用すると、型を省略することもできます。 これが短いバージョンです： 



 let r = 3.0 let circumference = 2 * .pi * r let area = .pi * r * r


グランドセントラル派遣



Grand Central Dispatchは、メインスレッドのユーザーインターフェイスをブロックしないネットワーク操作に使用されます。 C言語で書かれており、そのAPIは、非同期キューの作成や操作の実行などの単純なタスクを実行することすら、初心者の開発者にとって非常に困難です。 



 let queue = dispatch_queue_create("Swift 2.2", nil) dispatch_async(queue) { print("Swift 2.2 queue") }


Swift 3は、オブジェクト指向のアプローチを採用することにより、すべての定型コードと冗長な資料を排除します。 



 let queue = DispatchQueue(label: "Swift 3") queue.async { print("Swift 3 queue") }


この変更の詳細については、 Matt Wrightをご覧ください。



コアグラフィックスがより迅速になりました



Core Graphicsは強力なグラフィックフレームワークですが、GCDに似たCスタイルのインターフェイスを使用します。 



 let frame = CGRect(x: 0, y: 0, width: 100, height: 50) class View: UIView { override func drawRect(rect: CGRect) { let context = UIGraphicsGetCurrentContext() let blue = UIColor.blueColor().CGColor CGContextSetFillColorWithColor(context, blue) let red = UIColor.redColor().CGColor CGContextSetStrokeColorWithColor(context, red) CGContextSetLineWidth(context, 10) CGContextAddRect(context, frame) CGContextDrawPath(context, .FillStroke) } } let aView = View(frame: frame)


DrawRect（）メソッドをオーバーライドしてUIViewクラスを拡張することにより、ビューフレームを作成します。



Swift 3は完全に異なるアプローチを取ります-最初に現在のグラフィックコンテキストを展開し、すべての描画操作を実行してから、それに関連付けられます。 



 let frame = CGRect(x: 0, y: 0, width: 100, height: 50) class View: UIView { override func draw(_ rect: CGRect) { guard let context = UIGraphicsGetCurrentContext() else { return } let blue = UIColor.blue().cgColor context.setFillColor(blue) let red = UIColor.red().cgColor context.setStrokeColor(red) context.setLineWidth(10) context.addRect(frame) context.drawPath(using: .fillStroke) } } let aView = View(frame: frame)


注： DrawRect（）を呼び出す前にコンテキストはゼロになるため、ガード式を使用してコンテキストを展開する必要があります 。詳細はこちらをご覧ください 。



動詞と名詞の命名規約



文法の時間！ Swift 3のメソッドのグループは、2つのカテゴリに分類されます。特定の意味を返すメソッド-名詞を意味します-と特定のタイプのアクションを実行するメソッド-動詞を意味します。



10から1への出力はここで実行されます： 



 for i in (1...10).reverse() { print(i) }


範囲を変更するには、 reverse（）メソッドを使用します。 Swift 3は元の範囲を逆順で返すため、この操作を名詞として扱います。 彼はメソッドに接尾辞「ed」を追加します。 



 for i in (1...10).reversed() { print(i) }


配列の内容を表示するためのタプル/タプルの最も一般的な使用法： 



 var array = [1, 5, 3, 2, 4] for (index, value) in array.enumerate() { print("\(index + 1) \(value)") }


Swift 3は、この配列の現在のインデックスと値を含むタプルを返し、それにサフィックス「ed」を追加するため、このメソッドを名詞として扱います。 



 var array = [1, 5, 3, 2, 4] for (index, value) in array.enumerated() { print("\(index + 1) \(value)") }


別の例は配列のソートです。 配列を昇順で並べ替える方法の例を次に示します。 



 var array = [1, 5, 3, 2, 4] let sortedArray = array.sort() print(sortedArray)


Swift 3はソートされた配列を返すため、この操作を名詞として扱います。 sortメソッドは、 sortedと呼ばれるようになりました ： 



 var array = [1, 5, 3, 2, 4] let sortedArray = array.sorted() print(sortedArray)


中間定数を使用せずに配列をソートしましょう。 Swift 2では、次のように関数を呼び出すことができます。 



 var array = [1, 5, 3, 2, 4] array.sortInPlace() print(array)


sortInPlace（）を使用して、可変配列をソートします。 Swift 3は、値を返さずに実際のソートを行うため、このメソッドを動詞として扱います。 アクションを説明する基本語のみを使用します。 したがって、 sortInPlace（）は sort（）と呼ばれるようになりました 。 



 var array = [1, 5, 3, 2, 4] array.sort() print(array)


命名規則の詳細については、 API Design Guidelinesをご覧ください。



Swift API



Swift 3はそのAPIに対して単純な哲学を採用しています-不要な単語を省略するために、何かが冗長であるかコンテキストから取り出せる場合は削除します：



  • XCPlaygroundPage.currentPageはPlaygroundPage.currentになります
  • button.setTitle（forState）はbutton.setTitle（for）になります
  • button.addTarget（アクション、forControlEvents）はbutton.addTarget（アクション、for）になります
  • NSBundle.mainBundle（）はBundle.main（）になります
  • NSData（contentsOfURL）はURL（contentsOf）になります
  • NSJSONSerialization.JSONObjectWithData（）はJSONSerialization.jsonObject（with）になります
  • UIColor.blueColor（）はUIColor.blue（）になります
  • UIColor.redColor（）はUIColor.red（）になります


列挙型



Swift 3は列挙をプロパティとして扱うため、 upperCamelCaseではなくlowerCamelCaseを使用します。



  • .Systemは.systemになります
  • .TouchUpInsideは.touchUpInsideになります
  • .FillStrokeは.fillStrokeになります
  • .CGColorは.cgColorになります


@discardableResult



Swift 3では、関数またはメソッドの戻り値を使用していない場合、Xcodeは警告を表示します。 以下に例を示します。







上記のコードでは、 printMessageメソッドは呼び出し元の受信メッセージを返します。 ただし、戻り値は使用されません。 これは潜在的な問題になる可能性があるため、Swift 3のコンパイラは警告を表示します。



この場合、処理された戻り値でない場合。 メソッド宣言に@discardableResultを追加すると、警告を抑制することができます。 



 override func viewDidLoad() { super.viewDidLoad() printMessage(message: "Hello Swift 3!") } @discardableResult func printMessage(message: String) -> String { let outputMessage = "Output : \(message)" print(outputMessage) return outputMessage }
