クライアントが舵を取るか、プロバイダーが舵を取るべき理由

当初、当社は「クラウドソリューションストア」として構想されていましたが、その後、IaaSのコースが取られました。 これは、市場が望んでいたために行われました-ユーザーはそのようなニーズを表明しました。 プロバイダーが顧客の判断に耳を傾ける必要がある理由については、さらに説明します。



/ Flickr / carla wosniak / cc



ランドマークを見つける



講義の1つで、オレグティンコフは 、ロシアのスタートアップは既成のソリューションをコピーするだけでよいと述べました。 このアプローチにより、製品を迅速に市場に投入し、クライアントに提示し、ターゲットオーディエンスを決定することができます。 目的のコースをすぐにプロットして移動を開始できるため、効果的です。



このアプローチは、ビジュアルデザインに当てはまります。 すぐに外に出るために、DigitalOceanのような同様のプロジェクトを見て、サイトの最初のバージョンを非常にすばやく組み立てました。







最初のホームページデザイン



サイトがあり、仕事を始めて、サービスの需要があるかどうかを確認できます。 そもそも、企業幹部がこのサービスに興味を持つようになりましたが、より多様なユーザー層が見られると予想していました。 法人がターゲットオーディエンスになりました-私たちはB2CからB2Bセグメントに焦点を移し、起業家との相互作用を改善する方法を考え始めました。



ターゲットオーディエンスを特定するとすぐに、UIソリューションのコピーをやめて、独自の何かを「展開」することが最終的に決定されました。 サイトの外観を再設計し、サービスに関する情報を見つける問題を取り除きました。



そのため、古い「ヘルム」は不要なものとして船外に投げ出され、新しいものが代わりになりました。







デザイン1cloud.ruの進化の段階の1つ







新しい1cloudウェブサイトのデザイン



外出先でのソリューションの改善



サービスを可能な限り有用にするためには、クライアントに一定の自由、つまり自分で「船を支配する」能力を与えることが重要です。 たとえば、新しい仮想サーバーを注文するための計算機を実装して、ユーザーが自分のニーズに合わせてシステムパラメーターを個別に構成できるようにしました。



確かに、このようなソリューションを慎重に実装する必要があります。選択の自由と使いやすさのバランスを保つことが重要です。 あまりにも多くのオプション、スライダー、ひねりが目を逃げさせると、インターフェースでの作業が困難になる可能性があります。



したがって、UIを処理するために必要な最小値を選択しました。これにより、クライアントは費用のニーズと期待を判断できます。 サービスの全期間にわたる「鉄」の使用に関する分析と内部調査に基づいて、私たちがすでに行ったその後の設計変更。 これらのデータに基づいて、電卓の進化的開発が行われました。







仮想サーバーを注文するための電卓設計のバリエーション







新しい仮想サーバー注文計算機



利用可能なテンプレートは、ほとんどの顧客の要件を満たしますが、アプリケーションとオファーを個別に検討する準備は常に整っています。 それでも、クライアントは舵取りをしています。



風の中で帆を支配する



使いやすさは素晴らしいですが、優れたサービスには何か他のものが重要です。 たとえば、サービスが単なるブラックボックスではないことを顧客に示す機能。 コントロールパネルを使用すると、ユーザーは何が起こっているかを制御でき、サーバーの負荷（CPU、RAM、ストレージ、通信チャネル）をリアルタイムで監視できます。







コントロールパネルのサーバーステータスウィンドウ



サーバーの負荷を分析すると、クライアント自身が「帆を下げたりセットしたり」、数クリックで電源を接続または切断します。 REST APIを使用してこれを行うことができます 。 仮想マシンを起動するには、目的のパラメーターを指定してPOST要求を送信するだけです。 オプションの 1つを次に示します 。 



curl -X POST -H 'Content-Type: application/json' -H 'Authorization: Bearer 75bb9805c018b1267b2cf599a38b95a3a811e2ef7ad9ca5ed838ea4c6bafaf50' "https://api.1cloud.ru/Server" -d '{"Name":"testAPI","CPU":1,"RAM":1024,"HDD":40,"imageID":1,"HDDType":"SSD","IsHighPerformance":true}'


同時に、顧客がどの程度の割合でリソースを使用しているか、どのように資金を分配しているかを確認します。 データによると、RAMのユーザーコストは48.46％、CPUの場合は19.28％、バックアップの場合は3.39％です。 この情報により、成長と改善の新しいポイントを見つけることができます。 たとえば、関税パラメータを計算するために使用されます。



最近、顧客はインターネットの速度を制御できます。 以前は、サーバーに割り当てられた外部チャネルには、10 Mbit / sの帯域幅が保証されていました。 ただし、サポートに連絡することで増やすことができます。 これで、お客様はコントロールパネルから10から100メガビット/秒の範囲で接続速度を自分で変更できます。



より多くのオプションをユーザーに提供するサービスを開発しています。 たとえば、中間テンプレートのない古いサーバーに基づいてサーバーを作成するサーバーのコピー機能が追加され、電子文書管理システムが導入されました 。 ドキュメントの配信方法は、コントロールパネルの同じ場所にインストールできます。



大きな水泳-大きな船団



単独で海を渡ることは困難です。 パートナーと対話せずに大規模なプロジェクトを開発するのと同じくらい困難です。 そのため、1cloud施設で独自のサービスを販売する再販業者、アプリケーション開発者、組織など、多くの個人および企業クライアントと協力しています。



1cloudソリューション（APIなど）を使用したサードパーティプロジェクトの開発により、さまざまな角度から提供するサービスを評価できます。 「小艦隊」の各参加者は、変更または解決策を提案できます。 これにより、顧客との関係とシステム全体が強化されます。



1cloud IaaSプロバイダーの開発における私たち自身の経験に基づいて、開発の方向を変えることを恐れてはならないと結論付けることができます。 顧客の声に耳を傾け、顧客に自由を与え、双方が快適になるようにする必要があります。 たとえば、インフラストラクチャを独立して管理できることをクライアントが理解している場合、これによりテクニカルサポート部門の負荷が軽減されます。 プロセスが会社と顧客の相互信頼に基づいている場合、サービスの改善ははるかに簡単です。
