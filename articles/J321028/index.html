別のスクリプトを記述し、ルーターを構成するスクリプト







ネットワーク機器メーカーは、インディケータの設定と収集のためのユニバーサルAPIに徐々に移行しています。NETCONF 、 OpenConfig 、MIBをインポートするソフトウェアがあり、SNMPを使用した同じ構成が長い間存在していました。 ただし、これらの重要事項については触れませんが、ネットワーク機器の構成を自動化する方法を共有するだけです。たとえば、支店を大量に開設する場合は、







例として、架空のセントラルオフィスではD-Link DFL-800を使用し、周辺ではMikroTik RB951UI-2HnDを使用しています。 特に、新しいブランチのシナリオについて話しているため、それらの間にIPsecトンネルを構成します。







MikroTikを構成する



MikroTikを構成する最も簡単な方法は、構成ファイルを使用することです。構成ファイルは、コンソールコマンドのセットを含むスクリプトです。 説明されている構成原理は、パラメータを設定するための同様のメカニズムを持つ他のルーターに適しています。







最初に、 / exportコマンドを使用して、既に構成済みで動作しているMikroTikの構成ファイルを取得します。 ファイルはバックアップとほとんど同じですが、ユーザー名とMACアドレスはありません。







別のルーターで構成ファイルを使用するには、一部のパラメーターを一意に変更する必要があります。 これには、ファームウェアに組み込まれたスクリプト言語の機能が適しています。特に、個々の設定を変数に転送することが適切です。 デフォルトゲートウェイを使用したスクリプトの例を次に示します。 







local Gate "10.1.3.1" /ip route :if ([:len [/ip route find distance=1]] = 0) do={add distance=1 gateway=$Gate; } else={set [find distance=1] gateway=$Gate}


当初は、構成をリセットせずに適用できるユニバーサルスクリプトを作成するという考えでした。 したがって、コードはデフォルトルートの存在をチェックし、その後、ゼロから作成されるか、変更されます。 ただし、新しいファームウェアでは、デフォルトで変更がデフォルト構成に絶えず追加されているため、この考えを放棄しなければなりませんでした。 すべてを予測することは非現実的であり、スクリプトを絶えず更新すると、自動化の意味が失われます。







テキストファイルではなくGUIでルーターの個々のパラメーターを設定する方が便利です。その場合、デバイスの詳細に精通していない人でもルーターを構成できます。 さまざまな資格の従業員が対応できるリモートオフィスに必要なもの。







AutoITで開発されたスクリプトでGUIを描画します。 このスクリプトでは、ネットワークマスク（255.255.255.0 = / 24）を再計算し、構成ファイルに1行ずつ入力する機能も追加しました。 問題を解決するための開発ツール自体は基本的なものではありません。個人的に使い慣れたものを使用できます。
