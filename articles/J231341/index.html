ZTE Open Cのレビュー-Firefox OSの別の電話





少し前、約1年前、Firefox OSで最初のスマートフォンを手に取りました。GeeksphonePeakのレビューをご覧ください。 開発者プレビューとして位置付けられ、OS自体はアップデートのアルファチャネル上にあり、すべてが繁栄して開発されました。 そのため、横棒に目を留めることができ、オペレーティングシステムとハードウェアプラットフォームの両方のリリースを待つ必要がありました。



時間が経つにつれて、2014年2月にZTE Open C電話が発表されました-今回は幅広い視聴者向けの「新しいFirefox OS 1.3を100ドルで」 。 そして、Firefox OS 1.3を実行しているデバイスが私の机に当たりました。 わかった、待って。 さらにこの投稿には、写真、スクリーンショット、うなり声、鼻水があります。



ちなみに、Androidが好きな人は、通り過ぎて投稿の最後を見ないことをお勧めします。



1. 価格と仕様



主なデバイスの取引は正式にeBayで行われますが、2月に最初に予約注文が可能になりました。 しかし、それらはロシアに配達されず、転送はデバイスのコストを増加させる可能性があります。 代替案-電話はTinyDeal （クーポン5％：zte5off）で購入することもできます。ルーブルでは3300ルーブルです。



この価格では、次の特性が得られます。

================================================== ===

  • OS：Firefox OS 1.3（またはAndroid 4.4.2-投稿の最後を参照）;
  • 画面：WVGA 480 x 800、4インチ（〜233 ppi）、TFT 256Kカラー。
  • RAM：512 MB、ROM：4 GB（+ microSD）、1 GB使用可能;
  • 通信：GSM 850/900/1800/1900、HSDPA 900/2100;
  • WiFi：802.11 b / g / n、Wi-Fiホットスポット
  • CPU：デュアルコアQualcomm MSM8210（1.2 GHz Cortex-A7）、GPU：Adreno 302;
  • カメラ：3.15 MP、2048 x 1536ピクセル、ビデオ：352x288 @ 15fps;
  • その他：FMラジオ、GPS、A-GPS、Bluetooth、microUSB 2.0
  • バッテリー：Li-Ion 1400 mAh
  • 寸法：126 x 64.7 x 10.8 mm
  • 重さ：125g
================================================== ===



2. 外観





図 1-2：キツネのいる箱。



ボックスはこのデバイスの最も美しい部分だと思います。 マットブルーのキャンバスの上にある光沢のあるロゴが目を引きます。 ハブラカトの上の箱の写真のため、このテキストを開いたに違いない。





図 3-4：命令と命令の対象。



この指示は、 「目標の名前で作られた」というスローガンで読者に会い、説明します。 「Firefox OSは、他のモバイルオペレーティングシステムによって課される規則や制限から解放するために作成されました。 共有し、保存する情報を完全に制御できるように設計されています。 FFOSは、私たちの目標に向けたステップの1つとしてMozillaで開発されました-Webテクノロジーのフルパワーをすべての人が利用できるようにすることです 。 いわば、「すべての家庭でのJavaScript」運動のクリエイターから。





図 5-6：一方、デバイス。



このデバイスは快適に手のひらに収まり、背面のマットカバーが詰まるのを防ぎます。 私の意見では、寸法と厚さが最適です。 側面には、電源ボタンの上部と3.5mmオーディオジャック、下部のマイクロUSBコネクタにサウンドコントロールキーがあります。 デバイスの実際の厚さ。





図 7：「ボンネットの下」とバッテリーを表示します。



バッテリーの肉厚は明白で、体積は1400mAhです。 FirefoxOSを積極的に使用すると（メニューを歩いたり、時計やカレンダーなどのアプリケーションを起動したり）、2〜3時間保持されます（そう、FFOSは容赦なく食べますが、ここでは不正確な場合があります、 コメントを参照）。 SIMカードスロットとmicroSDカードは便利です。カードを交換するためにカバーを突いたり、補助ツールを使用したりする必要はありません。 ただし、背面カバー自体は非常にしっかりとスナップするため、プラスチックカードを使用せずに取り外すことはできません。 デバイスのコストを考えると、ビルドの品質と仕上がりは尊重されます。 これはブランド化されたものであり、ここでは、エンジニアが部品の狭い配置、アンテナの最適な配置に対するアプローチを感じることができます。 私たちは材料を後悔しませんでした-マイクがある場合、それは特別なスロットに接着剤で接着されるだけでなく、小さなスポンジ状のゴムバンドで上部に押し付けられ、垂れ下がらないようにします。 内部の写真 ボードには2番目のSIMスロット用の脚があることに注意してください。





図 8：スクリーン。



ここの画面はまだ良好です。 はい、彼は太陽の下で他のスクリーンと競争することはありませんが、その代価のために、彼は通常の部屋でさまざまな角度で写真を表示できます。 （ピークのような）バンドは見られませんでした。 タッチスクリーン（ goodix gt968 ）はかなり正確ですが、FirefoxOSインターフェースはこれまでのところ正確なヒットを防ぎます。

写真では、下のパネルの中央に円があります-これは「ホーム」ボタンです。 しかし、実際には3つのタッチボタンがあり、他の2つのFFOSは認識しません。



3. ソフトウェアスタッフィング



電話のソフトウェア部分では、物事は非常に悲しいです。 FFOSは開発中で、1年前に「システムをもう1年提供する」と言っていました。 電話の販売時点で、Firefox OS 1.3は最新バージョンと見なされていました。 1.4のスクリーンショットはすでに表示されていますが、レビューから判断すると、1.4と2.1（次のメジャーバージョン）はどちらも不安定です。 電話をルート化せず、リポジトリのmasterブランチからの最後のコミットにファームウェアを更新しませんでした。バグをキャッチし、電源を入れて1年前にGeeksphone Peakで見たものと比較しようとしました。 すべてのスクリーンショットは新しいウィンドウで開くことができ、480x800になります。





図 9-10：システム情報。



システムアプリケーションインターフェイスはさらに悪化しています。 1.4での方法はわかりませんが、このバージョンではアイコンの低解像度画像が使用され、すべてが石鹸になっています！ 標準コントロールの可読性と一般的な外観は、1年前のアセンブリと比較して大幅に低下しています。 私はこの理由を理解していませんでした、バージョンは異なります-a）実験中です。 b）長時間ロードする高解像度画像。 または石鹸の流行を植え付けてください。





図 11-13：設定画面。



ホーム画面では、何も変更されていません。 最初のページは、アプリケーションとして開くことができるサイトへのリンクのコレクションです（つまり、サイトはフルスクリーンでタブなしです）。 残りの2つのページには、システムアプリケーションと、アプリケーションとしてパッケージ化されたサイトが含まれています。





図 14-16：ホーム画面。



変化に慣れる過程で、私はどこかにだまされているという感覚が去りませんでした、どこで正確に明確ではなかった。 はい、インターフェイスの古いバージョンは理想からはほど遠いですが、新しいバージョンはWeb 1.0の時代をもたらします。 新しいものは忘れられた古いものを必ず返さなければならないのでしょうか？





図 17-18：左側はFirefox OS 1.0.1プレリリース（2013）の電子メール、右側はFirefox OS 1.3（2014）のSMSです。



標準のhello worldアプリケーション（たとえば、テキストを選択することさえできないテキストエディター-システムへの入力はすべて入力のみ）を破棄し、Android 1.5などのアプリケーションを破棄し、サイトのみのアプリケーションを破棄し、システム設定アプリケーションを破棄し、アイコンは3つだけです。1つ目はダイヤラ、2つ目はメールクライアント（非常にシンプルですが、緊急時には機能します）、3つ目は個人的に最も適しているFMラジオです。 私は電話ではなくラジオを購入したかのように。





図 19-21：そうそう、ビジネスアプリケーション％）



市場には何もありません。 アプリケーションのガイドラインはありません。 インターフェイスの高速レイアウトのためのブートストラップはありません。 システム開発者は3つのボタンを連続して作成することはできません。コントロールはアプリケーションごとに異なります。 すべてが非常に遅くなり、安価な携帯電話のブラウザではヘビー級のhtml5サイトのように動作します。 実際、これが起こるのです。電話を起動し、そこからブラウザが起動し、別のブラウザが起動します...



標準システムアプリケーション間の単純な切り替えは、ブレーキをかけることで行われ、電話が熱くなり、バッテリーが著しく急速に残ります。 どこかで私はすでにこのすべてを見ました。 そうそう、ちょうど1年前にGeeksphone Peakで 。



4. Android 4.4.2



ラジオは、1つではなくても、ごみ箱に安全に送ることができます。 短いグーグルで判明したように、ZTEは同じベースで2つのモデルを一度にリリースしました。 ZTE KIS 3はインドネシアの人々向けにリリースされ、ZTEはヨーロッパとアメリカの人々向けにZTE Open Cをリリースしました。 内部の写真を覚えておいてください 2番目のSIMスロットのための場所がありました。 パネルには実際には3つのタッチボタンがあり、それらはまったく表示されていないという事実を思い出します。





図 22：ZTE KIS 3はZTE Open Cと同じです。



パブリックドメインには、FFOSをAndroidに、またはその逆に置き換えることができるファームウェアがあります。 （ オリジナル 、 ミラー ）。 このアーカイブには両方のサービスパックがあり、そのうちの1つをカードに放り込み、CWMリカバリを介して更新プロセスを開始します。 電話がオフのときにCWM- VolUp + Powerを呼び出します。





図 23：CWM Recoveryメニュー。



CWMのサービスパックが利用できるサイトにも、PCから直接フラッシュするためのルートファームウェアとユーティリティがあります。

  • ZTE_OPENC_US（ オリジナル 、 ミラー ）;
  • ZTE_OPENC_EU（ オリジナル 、 ミラー ）;
  • Open_C_upgrade_Tool（ Original 、 Mirror ）;


Firefox OSをAndroidに置き換えることができるパッケージは、半回転して電話が再起動しました。 そして...このデバイスは新しい生活を始めたばかりです。 私はこのような応答性を長い間見ていません。おそらく誰かがこのファームウェアでタイミングを設定することに手を携えていたでしょう。 インターフェイスはスマートに機能し、アプリケーションはすぐに起動し、KitKatは美しいです。 このファームウェアは実質的にAOSPであり、不要なサービスが含まれていないため、提供されている更新ツールを使用して、デバイスをルート化し、市場を含むGoogle Appsを再インストールできます。





図 24-25：パワーとスタミナのベンチマーク。



バッテリーの寿命が嬉しかったです。 最初の2日間はグリッチだと思っていましたが、実際には電話は1週間中横になり、SMSを受信し、携帯電話ネットワークに接続し、同時に生き続けることができます。 危険な地域に散歩に行く場合は、ダイヤラーのままにしておきます。



5. 結論



Firefox OSで再び失望しました。 外部の観察者の側から、全員がその場で踏みつけます。 一般的に、Webはネイティブプラットフォームに移植できる最高のものではありません。 Mozilla開発者は、可能な限りWebを広めるために、最高の目標のためにHTML + CSSレイアウトのすべての痛みとすべてのJSブレーキに耐えるべきだと考えていますが、原則として例を設定することはできません。 この方法では、生産性の面でも、開発の利便性と速度の面でも、モバイル開発のネイティブツール（ObjC / Java / C＃）に追いつくことは不可能であることに気付くでしょう。 VMとJSシェルからのロシアの人形よりも良いAndroidが予算ハードウェアで実行されるとき、FFOSが予算の電話の軸として積極的に宣伝され、配置される理由はわかりません。 おそらくもう1年を与えましょう。
