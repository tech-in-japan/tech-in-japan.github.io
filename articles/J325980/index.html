ePUB eコースでのIMS QTIの使用

問題の本質



教育機関で使用されるeラーニングシステム（SEA）では、多くの場合、電子コースをSEAに接続することに加えて、個々のコンポーネントをインポートして教育リソースの統合バンクを形成する必要があります。 このような銀行は、学際的な特別コースの作成、研究期間全体の最終テストなど、教育プロセスで使用されます。 原則として、これらは主にテストタスク、画像、マルチメディアコンポーネントです。 一部のコンテンツ要素、特に機械可読形式（CSV、JSON、XML、XLS）を使用せずに作成されたテストタスクでは、SEAへの効率的な自動インポートを受け入れられません。 これは、eコースおよびePUB出版物のコンテナで使用されるHTML / CSS / JavaScriptに特に当てはまります。 この問題を解決するために、著者は、IMS Question and Test Interoperability v.2.2（IMS QTI）仕様に準拠した機械可読XML形式のコースの一部として、テストおよび個々のテスト質問の説明を使用することを提案します。







ePUB v.3について一言



IDPFによって開発されたePUBバージョン3は、電子出版物の交換と配布の標準です。 ePUBは、電子教科書やコースの出版社に広く配布されています。 この形式は、ほとんどのモバイルプラットフォームでサポートされており、たとえばAppleのibookなど、他の多くの同様の形式の基盤となっています。







図1. EPUBコンテナーの構造







ePUBは、元々電子書籍用に作成され、 DRMシステムを介してライセンス契約への準拠を配布および監視するのに便利な疎外されたパッケージであるという事実により、積極的に開発され、パブリッシャーの間で人気を博しています 。

Eラーニングシステム（ LMS ）の開発者もこのフォーマットに注意を払いました。一方で、ePUBバージョン3の登場により、このフォーマットはマルチメディアオブジェクト（HTML5、CSS、JavaScript、SVG、オーディオ、ビデオ、ビットマップ画像、他のリソース（3Dオブジェクトを含む）を使用すると、あらゆる種類の電子教育リソースを実装できます。また、2013年に古いSCORMを置き換えたxAPI標準（別名Tin Can）では、直接参加せずにePUBから学生の行動に関するデータを直接送信できますLMS







ePUBの中心にあるのは、Open Packaging Format（OPF）です。これは、HTML / XHTML形式のハイパーメディアコンテンツとxml記述ファイルを含むzipコンテナーです。 ePUB構造を図1に示します。







ハイパーメディアコンテンツはOEBPSカタログにあります。カタログの名前は歴史的な理由によるものです。これはePUB形式に先行する電子出版形式の名前です。

META-INFディレクトリには、一般的なコンテナ記述ファイルが含まれています。 図 図2は、ePUBコンテナー内に、HTMLサイトの構造に似た規則的な階層ファイル構造があることを示しています。 実際、ePUB v.3のコンテンツは、プログラミング、マークアップ、視覚化言語がHTML5 / CSS / JavaScript言語の束であるWebサイトまたはスタンドアロンWebアプリケーションのオフラインバージョンと比較できます。







図のオレンジ色の塗りつぶし 1は、フォーマットの相互運用性を強化するePUBに追加された要素を強調しています。 これらは、xAPI JS libsディレクトリに含まれるxAPI（Tin Can）ライブラリとJavaScriptコード、およびIMS QTI形式の追加のxmlの説明です。これらについては、以下で詳細に説明します。







IMS QTIをePUBに追加



IMS QTI仕様に従ってテストとシミュレーターの説明を追加するには、各ePUBコンテナーに次のxmlファイルが含まれている必要があります。







  • 1 ePUBコンテナファイルimsmanifest.xmlに共通です。これは、第1レベルのMETA-INFのルートディレクトリに配置する必要があります。 Imsmanifest.xmlには、IMS QTI仕様に関連する他のすべてのコンテナファイルへのリンクが含まれている必要があります。 このファイルには、電子コースのインタラクティブな要素（テストとシミュレーター）の一般的な構造と関係が記述されています。
  • 各テストとシミュレーターには、タスクの構造と内容、正解と不正解、最終スコアの計算における重み、質問で使用されるイラストやその他の補助資料へのリンク、および設定を説明する1つのqti_xxxxx.xmlファイルが含まれているタスクロジック。 これらのファイルは、テストとシミュレータを含むルートサブディレクトリに保存する必要があります（図1-test1 ... testX）。 説明されているファイルへのリンクは、タイプがmedia-type = "text / xml"のePUBコンテナcontent.opfの説明を含むメタ情報ファイルに存在する必要があります。


IMS QTI v2.2仕様では、電子コースで使用されるほとんどのタイプの質問を説明できます。 仕様は特定のグループを識別し、その中で最も頻繁に使用されます：







  • イラスト付きまたはなしの「唯一の正しい答えを持つ複数の選択肢」のような質問。
  • イラスト付きまたはイラストなしの「複数の正解オプションを備えた複数選択」などの質問。
  • 一致するタスク「コンプライアンスの設定」テキスト-テキスト、テキスト-イメージ、イメージ-イメージ、テキスト-その他のオブジェクト（セル、列など）


IMS QTI標準の各タイプには、特定のXMLタグと構文機能があり、タスクの内容とロジックを記述します（質問）。 これらのタグと構文については、特定の例を含めて以下で説明します。







共有imsmanifest.xmlパッケージファイル



仕様に従ったimsmanifest.xmlファイルの一般的な構造を図2に示します。







図2. imsmanifest.xmlファイルの一般的な構造







ほとんどの電子コースでは、仕様で提供されるimsmanifest.xmlファイルの一部の要素は冗長であり、使用されません。







以下のリスト1は、e-Biologyコースのimsmanifest.xmlファイルからの抜粋です。 コースには、テスト（テストの質問）とシミュレーターが含まれています。 この電子コースのコンテキストでは、「テスト」または「テスト問題」は知識テストの要素であり、それらの実装の結果はデー​​タベースに送信されます。 シミュレータは、形式であり、テスト問題に似ている場合がありますが、課題を複数回通過させて材料を統合し、学生が正しい答えを覚えられるようにすることを目標としています。 シミュレータの課題を完了する学生に関するデータは、一般的な成績表には含まれません。

imsmanifest.xmlファイルは、他のxmlファイルと同様、プロローグで始まり、その後にメタ記述（<metadata>タグ）が続き、コース名、バージョン、ePUBファイル名が示されます。 以下は、対話型テストとシミュレータの一般的な構造の説明（<organizations>タグ）と、個々のテストまたはテストの質問（<resourses>タグ）の詳細な説明を含むqti_xxxxx.xmlファイルへのリンクです。







個々のテストまたはテストの質問qti_xxxxx.xmlのファイル



この例では、ePUB、テスト、およびシミュレータは、同じ名前のサブディレクトリにオブジェクトとして保存されます（たとえば、test1、test2 ... testX、tutor1、tutor2 ... tutorX）。 テストとシミュレータの名前はePUBコンテナ内で一意の値を持ち、すべてのxml記述ファイルの識別子でも使用されます。







仕様に準拠し、テストとシミュレータの構造とコンテンツの自動インポートを有効にするには、各テストまたはシミュレータにqti_xxxxx.xmlという形式のファイルを装備する必要があります。xxxxxは、このテストまたはシミュレータを含む同じ名前の識別子とディレクトリの名前です。







テストとシミュレータの構造を記述するファイルも、media-type = "text / xml"のメタ情報ファイルcontent.opfにリストする必要があります。







テスト質問の例とその説明を含むqti_xxxxx.xmlファイルのリスト



質問「唯一の正しい答えを持つ複数の選択肢」


図3.質問「正解のみの複数選択」







図3に、test1という名前のePUBコンテナーの構造の質問を示します。 この質問では、生徒は質問に答える機会が与えられ、失敗した場合に正しい答えを知ることができます。 質問には、IMS QTI v2.2仕様-qti_test1.xml（リスト2）に従って、その構造を説明し、正しい答えを示すファイルが添付されています。

質問「複数の正解を持つ複数の選択肢」


図4.質問「複数の正解オプションを含む複数選択」







質問を図4に示します。この質問では、生徒は複数の回答を選択する機会が与えられます。それぞれの回答は正または負の重みを持ち、質問の結果のスコアに影響します。 ePUBコンテナーの構造内の質問には、名前とディレクトリtest2があり、その構造を説明するファイルqti_test2.xmlが添付されています。このファイルには、ePUBコンテナー内の図へのパスも示されています（リスト3）。







リスト3. qti_test2.xmlファイル

リスト3では、<correctResponse>タグが質問に対する正しい答えを指定し、<mapping>タグが正しい答えと間違った答えそれぞれの重みを設定します。 複数の正解がある場合があるため、質問の合計スコアは正解と不正解の合計に等しくなります。 したがって、正解のみを選択すると、学生は課題の最終スコアを上げます。 反対に、学生が<mapEntry>タグで負の重み（-1）を割り当てられた間違った答えを選択すると、合計金額が減少し、最終結果が悪化します。 <mapping>タグでは、タスク全体の最大可能上限と最小下限スコアも設定されます。これらの値は、計算された正解または不正解の合計量に関係なく克服できません。 この例では、lowerBound = 0です。これは、生徒が間違った解答（-1-1-1 = -3）のみを選択した場合でも、タスクの結果のスコアは0になることを意味します。各解答には、 ePUBコンテナに保存された画像。







マッチマッチの質問


図5.一致する質問「Set Compliance」







図5に、test3という名前のePUBコンテナーの構造の質問を示します。 生徒は値のペアを一致させる必要があります。この場合、イラストとキャプションをそれらに正しく一致させます。 正しく行われた対応ごとに、特定のポイントが付与され、タスクの合計スコアとして合計されます。 不正確な一致は考慮されず、罰金は割り当てられません。 質問には、説明ファイルqti_ test3.xml（リスト4）が添付されています。このファイルには、ePUBコンテナー内のイラストへのリンクも含まれています。
