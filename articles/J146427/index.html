エンタープライズポータルの概要

社内に企業ポータルが存在しても、誰も驚かないでしょう。 ポータルの選択と実装には、何らかの形でますます大規模および中規模のビジネス担当者が直面しました。 それにもかかわらず、多くの企業は、企業ポータルを導入するためのプロジェクトの立ち上げを決定しようとしています。 私の短いレビューがこの方向への第一歩を踏み出すのに役立つことを願っています。







そのため、今日市場で提供されているすべてのポータルソリューションは、3つのグループに分類できます。







  • 企業ポータルを作成するためのプラットフォーム。
  • プラットフォームに基づくボックス製品（ターンキーソリューション）。
  • SaaSサービスとして提供されるソリューション（クラウドソリューション）。


それらを順番に検討してください。









企業ポータルを作成するためのプラットフォーム




ポータルソリューションを作成するためのプラットフォームは、次のような商用のものに分割されます。







Oracle WebCenter Suite、MS SharePoint、IBM WebSphera、1C-Bitrix







のようなものを開きます







Joomla！、Jboss、Plone、Drupal、Jiveなど









プラットフォームは、特定の技術的タスク（TOR）のための企業ポータルを開発するために使用され、理論的には他のグループが顧客のニーズを満たすよりも優れています。 有能なToRが作成された場合、それが会社のすべてのニーズを説明している場合、そのような決定においてそれらは実装されます。







それは大胆不敵なスパルタンの物語を思い起こさせます：マケドンのフィリップ（アレクサンダーの父親）がスパルタの壁に近づいたとき、彼はスパルタンにメッセージを送りました：「私はギリシャのすべてを征服しました。 降伏。スパルタを力ずくで捕まえたり、彼女の門を壊したり、ラムで壁を壊したりすると、容赦なく全人口を破壊するからだ。」 スパルタ人が答えた：「もし」







実際、システムのテスト操作の前に、それを使用するすべての可能な方法を予測することは非常に困難です。 これにより、プロジェクトの条件が遅れたり、承認された予算を超えたりする可能性があります。









プラットフォームベースのソリューション




箱入り製品（既製のソリューション）は、IBM WebSphere Portal、Oracle Portal 11g、SAP NetWeaver Portal、DeskWork、Ittilan Portal、WSS Portal、1C-Bitrix：Enterprise portal、およびオープンソリューション：Liferay Portalの商用プラットフォームに基づいて構築されたソリューションに分かれています。 Alfresco、JomPortal、Jive、Jahiaなど









これらの製品はすべて、いくつかのグループに分類できます。









  • 大企業向けの大規模な商用ソリューション（IBM WebSphere Portal、Oracle Portal 11g、SAP NetWeaver Portal）。
  • 中規模および大企業向けの商用ソリューション（DeskWork、Ittilan Portal、WSS Portal、1C-Bitrix：Corporate Portal）;
  • 中小企業（Liferay Portal、Alfresco、JomPortal、Jive、Jahia）向けのオープン（無料ライセンス）ソリューション。


次に、リストされている既製のソリューションの簡単な説明をします。 説明には、ポータル実装プロジェクトを含まないライセンスのおおよその価格が含まれています。 最終的なライセンス価格と実装コストについては、ソリューションプロバイダーにお問い合わせください。 実際には、実装プロジェクトのコストはライセンスのコストを大幅に上回る可能性があります。









  1. 重い商用ソリューション

    1. IBM WebSphere Portal （ www.ibm.com ）は、IBM WebSphereプラットフォームに基づいたターンキーポータルソリューションです。 素材の投稿、コラボレーション、遠隔教育のための多くの機能を備えたソリューション。 システムの機能はモジュール上に構築されます-WebSphere Portlet Factoryを使用して変更できるアプリケーション。 日常業務（名刺の注文、オフィスの注文、スタッフ募集の申請、会議室の予約など）を含むあらゆるビジネスプロセスの本格的な自動化と、追跡段階、期限、タスクを担当するユーザーの割り当てを伴うプロジェクト管理メカニズム。 欠点としては、同時に多数のユーザーを同時に使用する大企業向けに設計された重いシステム、システムの所有コストが高いことです。 ライセンスのおおよその費用は1,500,000ルーブルです。 （または300の登録済みライセンス、またはそれぞれ2コアのサーバー2プロセッサー）。
    2. Oracle Portal 11g （ www.oracle.com ）は、Oracle WebCenterに基づいた完全なポータルソリューションです。 ビジネス分析ツール、ドキュメント、タスク、プロジェクトを備えた本格的なチームワークが非常に強く表されています。 ビジネスプロセスを自動化する手段を開発しました。 それとは別に、ポータルの内部と外部の両方で、情報の検索とインデックス付けに注意したいと思います。 同じメーカーの製品をすでに使用している企業に最適です。 欠点の中でも、複雑なシステムでは、短時間での起動、高い所有コストを忘れることがあります。 会社がすでに別のメーカーの企業システムを使用している場合は、統合に手を加える必要があります。 推定ライセンスコストは600,000 /プロセッサです。
    3. SAP NetWeaver Portal Enterprise Portal（ www.sap.com ）は、別のトップレベルのポータルソリューションです。 さまざまなシステムの便利な統合機能-会社の情報システムへの単一アクセスポイントの機能。 チームワーク、プロジェクト管理、従業員とのコミュニケーションに重点を置いています。 SAPベースのポータルの主要な能力の1つは、会社の知識ベースの管理です。 既存のビジネスプロセスを自動化し、ワークフローを最適化できます。 ポータルからSAPアプリケーションだけでなく、他の開発者からのアプリケーションからもアクセスできます。 SAP Portal Content Studio環境およびSAP Portal Development Kitを使用して、ポータルを改良することができます。 欠点の中には、管理と構成を行う複雑なシステム、所有コストとスケーリングが高いことなどがあります。 ライセンスのコストに関する未公開情報はありませんが、価格レベルはクラスに対応しています。

クラウドソリューションは次のとおりです。







Bitrix24 （ www.bitrix24.ruの推定コスト-毎月9900ルーブル、無制限のユーザー数とクラウド内の100 GBのスペース、







シンプルなビジネス （ www.prostoy.ru ）概算コスト-無制限のユーザー数とクラウド内の10 GBのスペースを備えた1000ルーブル/月、

最新のクラウドソリューションは、企業が使用する他の企業システムとの統合の可能性を除き、既製のポータルソリューションの基本機能をほぼ繰り返します。







クラウドソリューションの利点は、サーバーの所有コストに関連する固定費の削減、システムソフトウェア、PCの要件の低下（実際、I / O端末が必要であり、すべてのロジックがクラウドで処理される）、スケーリングの容易さ、および展開速度（新しいオフィスを開設し、PCをインストールしてインターネットへのアクセスを整理する必要があります）、ソフトウェア管理、領土の独立（EUのどこからでもポータルへのアクセスに関連するコストを削減します） インターネットへのアクセス）。







ただし、これらのすべての利点（おそらくライセンスレンタルを除く）は、サーバー（専用サーバー）をレンタルするときにクラウドに既製のソリューションを展開することで実現できます。おおよそのコストは3000ルーブル/月（1 CPU Core i3-2100（3.1GHz 2コア））、 RAM 8GB、HDD 1Tb）またはラックスペースデータセンター（コロケーション）-2000ルーブル/月1ユニットのおおよそのコスト。







クラウドソリューションの欠点は、企業情報を外部に持ち出すためのビジネス準備が整っていないこと、他の企業システムと統合することの難しさ、低ネットワーク帯域幅、そのようなサービスのプロバイダーに対する信頼の欠如、考えられる3つの障害点（実際には、クラウドプロバイダー、プロバイダー、ITユニットまたはシステム管理者）、セキュリティ関連の問題。









結論


このレビューでは、まず、私たちの市場に存在するさまざまなポータルソリューションを簡単な言語で説明し、企業の基本的なニーズと規模に基づいて、適切なソリューションを見つけるための指針を提供しようとしました。







今日、どの企業でも、味覚、予算、解決すべきタスクのリストを見つけるためのポータルソリューションを見つけることができます。







既存のプラットフォームに基づいたカスタムポータルソリューションの開発は、次の2つの理由からあまり意味がありません。







  • 既製のソリューションが多数あり、カスタムソリューションの開発がより機能的または費用対効果が高いとは考えられません。
  • カスタムソリューションのサポートは、複製された製品よりも高価になる可能性があります。


大量の商用ソリューションは 、多数の作業ユーザーを抱える大規模な分散型企業に適しています。 最大のメーカー（IBM、SAP、Oracle）の製品を既に使用している企業に最適です。









ミッドレンジの商用ソリューションは 、中規模企業により適しています。 このクラスで提示される製品は、通常の企業のニーズのほとんどをカバーしています。







選択のガイドラインは次のとおりです。







  • 3000人までの同時作業ユーザーの数。
  • 予算は最大1,000,000ルーブル。
  • ソリューションプロバイダーからの成功したプロジェクトの可用性。
  • 製品に対する個人的な同情、専門的なサポートの存在。


オープンソース機能ソリューションは商用ソフトウェアに近く、典型的な企業のほとんどのニーズに対応しています。 ライセンスのゼロコストは、プロジェクトのコストを大幅に削減します。 最近、オープンソースのソフトウェア市場はダイナミックに発展しており、最近になって感じられた専門的な技術サポートの欠如の問題は去りつつあります。







このセグメントでソリューションを選択するためのガイドラインは次のとおりです。







  • 最大1000人の同時作業ユーザーの数。
  • 40万ルーブルまでの予算。
  • ソリューションプロバイダーからの成功したプロジェクトの可用性。
  • 専門家によるサポートの提供。


SaaSサービス（クラウド）として提供されるシステムは 、中小企業の市場により適しています。 主な論点：低い起動コスト、領土の独立。

クラウドソリューションは、他の企業システムとの統合の難しさ、企業が企業情報を持ち出そうとしないなどの理由で、それほどではありませんが中規模企業に適しています。
