データベース



  • Tarantoolのレコードはスペースに保存されます。これは、リレーショナルSQLデータベースのテーブルに似ています。 必要なだけ作成できます-最大65,000
  • スペース内にはタプルがあります。タプルは、SQLテーブルの行とJSONデータ配列に似ています。 タプルの最大サイズは、デフォルト設定で最大1 MBです。 このパラメーターは変更できます。
  • データベースを活用するには、SQLと同様にインデックスを作成する必要があります。 それらのおかげで、すべての要素の完全なリストを反復処理する代わりに、より高速なアルゴリズムを使用して検索が実行されます。 並べ替えも可能です。 インデックスの選択は、データのタイプによって異なります。


HASHインデックス ：値は一意である必要があり、任意の値にすることができます。 これは、よく知られているキー/値ストアの構成方法であり、マップとも呼ばれます。 良い例は、MD5ハッシュファイルのチェックサムです。



TREEインデックス ：値は一意ではない場合がありますが、並べ替えられたリストを整理するには「密」でなければなりません。 不足している要素がある可能性のある配列（Array）が判明します。 良い例は、1ずつ増加する注文番号です。



一意の値が必要な場合は、HASHまたはTREEを使用できますが、HASHはスパースデータではより高速になります。 ソートに一意でないフィールドが必要な場合は、TREEインデックスのみを使用できます。



また、2次元平面での検索用のRTREEインデックスと、ビットデータの操作用のBITSETがありますが、それらは必要ありません。 詳細については、 ドキュメントを参照してください 。

プロジェクトデータモデル



アプリケーションのデータモデルでは、ファイルに関する情報を保存するためのスティッカースペースを作成します。 Lua構文が使用されるため、フィールド番号は1から始まることに注意してください。 タプルには次のフィールドが含まれます。



  1. 署名なしID-一意のステッカー番号
  2. 整数評価-ステッカー評価
  3. 文字列パック-ステッカーパックの名前
  4. 文字列名-ステッカーファイル名
  5. （文字列）パス-ステッカーURL
  6. （number）up-ステッカーごとの投票数
  7. （number）down-ステッカーに対する投票数。


パックスペースに、ステッカーパックのリストを保存します。



  1. 文字列パック-ステッカーパックの名前
  2. 整数の評価-ステッカーパックの評価
  3. （文字列）パス-説明ページへのリンク


シークレットスペースには、不正行為に対する最も簡単な保護を実装するために、画像へのリンクを暗号化するトークンを格納します。



  1. 文字列トークン-ステッカーのランダムトークン

  3. （整数）id-一意のステッカー番号（ステッカースペースキー）
  4. （文字列）url-ステッカーURL


セッションスペースでは、訪問者を記録し、統計を収集します。

これで、必要なサイトからデータを収集するパーサーを作成できます。 tlgrm.ru/stickersを探索します 。 まず、 パックテーブルを読み込みます。このテーブルには、ステッカーパックの人口調査があります。 これは、tarantoolコマンドラインでのinsertコマンドの外観です。 



 box.space.packs:upsert({'key1',0}, {{'=',2,0}})


このコマンドは、新しいキー「key1」（フィールド1）と値0（フィールド2）を追加します。 レコードが存在する場合、フィールド0の同じレコード（=記号）に対して値0で更新されます。思い出すように、フィールド2の評価は最初に0に設定されています。upsertコマンドは 、デバッグ中にパーサーを繰り返し実行するのに便利です毎回入力したデータを削除しないでください。 コマンドのPHPバージョンは次のようになります。 



 $tarantool->upsert('packs', array ($pack,0), array ( array( "field" => 1, "op" => "=", "arg" => 0 ) ));


ああああ！ PHPでは、フィールドの番号付けは0から、Luaでは1からです。したがって、PHP配列の"field" => 1



は、Lua {'=',2,0}



と書くことに対応します。 アレイがゼロから始まる場合は常に、現在のコネクタは同じように機能します。 この動作はバージョン1.6以降に変更されています。 インターネットで例を読むときは、タランチュラのバージョンに注意してください！ この記事はバージョン1.7で書かれており、開発者はバージョン1.5についてまったく覚えていないように求められています。



組み込みのauto_incrementプロシージャを使用してステッカーのエントリを作成します。これにより、プライマリインデックスが自動的に増加します。 タランチュラチーム：

 $tarantool->call('box.space.stickers:auto_increment', array( array(0,$pack, $i . '.png', $url, 0, 0) ));


そのため、スクリプトが作成されます。 起動します-声に出して、魔法！ これで、16,000レコードのデータベースができました！



プログラムを書く



まず、PHPで通常行われているように、単純なクエリルーターを作成します。 



 # Get routes from request $route = isset($_REQUEST['q']) ? $_REQUEST['q'] : '/'; $vote_plus = isset($_REQUEST['vote_plus']) ? $_REQUEST['vote_plus'] : ''; $vote_minus = isset($_REQUEST['vote_minus']) ? $_REQUEST['vote_minus'] : ''; switch ($route) { case '/good': action_good(); break; case '/bad': action_bad(); break; case '/ugly': action_ugly(); break; case '/about': action_about(); break; default: if (!empty($vote_plus) && !empty($vote_minus)) { sleep (1); do_post($vote_plus, $vote_minus); } action_main(); }


2つの変数$ vote_plusおよび$ vote_minusの存在に注意してください。これらは、1つまたは別の写真に投票するときにPOSTリクエストで送信されます。 実際、ファイルの名前とパスを知っていれば、ボットで投票するのは非常に簡単ですが、これは必要ありません。 したがって、投票ページ用に、各画像に1つずつ、一意のトークンのペアを生成します。 投票後、トークンは削除され、投票を再利用できなくなります。



バージョン7.0のリリース前のPHPでは、暗号化セーフ機能を使用することは悲しいので、暗号化を操作するためのTarantulaの豊富な機能を大いに支援します。



まず、action_main関数で、暗号化された乱数ジェネレーター（つまり、本当にランダムな）シードを開始します。 



  $r = $tarantool->evaluate( "digest = require('digest') return (digest.urandom(4))" ); $seed = unpack('L', $r[0])[1]; srand($seed);


$ tarantool-> evaluate（）関数は、ストアドプロシージャを作成する手間をかけずに直接Luaコードを実行するために使用されます。 次に、 create_random_vote（）関数を2回呼び出します。これにより、空間内のランダムな要素が選択され、画像とトークンのURLが作成されます。



関数create_random_vote（） 
 function create_random_vote() { # Get random sticker id global $tarantool; $tuple = $tarantool->call("box.space.stickers.index.primary:random", array(rand())); $id = $tuple[0][0]; $url = $tuple[0][4]; # Create random sticker token $token = $tarantool->evaluate( "digest = require('digest') return ( digest.md5_hex(digest.urandom(32)))" )[0]; $time = time(); # Set secure token to protect post action ################################################################## # # box.space.secret:insert({key', 0, 456, 'bla-bla'}) # ################################################################## $tarantool->insert('secret', array ($token, $time, $id, $url)); return array ( $url, $token ); }




ここでさらに2つの関数が使用されました。 $ tarantool-> call（）組み込みプロシージャを呼び出し、 $ tarantol-> insert（）で新しいレコードを挿入します。



レコード評価を更新する投票更新手順の例を次に示します。

パラメーター " op" => "="



に注意してください。これは、フィールドが既存のタプルで置き換えられることを意味します。 また、パラメーター+、-、およびその他の操作があります。 彼らは非常に重要なタスクを実行します。 通常、データベース内の値を置き換えるには、まずいくつかのフィールドを読み取ってから変更します。 データの一貫性を維持するには、テーブルへのアクセスをブロックし、トランザクションを使用する必要があります。 Tarantulaでは、そのアーキテクチャのおかげで、 updateおよびupsertコマンドはデータベースをロックせずにサーバープロセス内でアトミックに動作します。 これにより、非常に高速なシステムを構築できます！
