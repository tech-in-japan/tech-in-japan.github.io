MySQLを使用した例を使用して、nodejs用のC ++モジュールを作成します。



はじめに



私の意見では、多くの人がすでにNode.jsを試しています。これは、幅広いタスクを解決するための非常に便利なツールです。 まず、Node.jsの魅力は、JavaScriptコードと、頻繁に発生するタスクを解決するための多数の組み込みモジュールを作成できることです。 標準パッケージにないものがある場合は、 npmjs.orgリポジトリに多数の追加モジュールを見つけることができます



ただし、そこにあるすべてのものが希望どおりに機能する、機能しない、または特定の条件下でまったく機能しない、またはすべてがより一般的である場合があります。特定のケースに必要なものが単に欠落しています。 MySQLへのクエリを同期的に実行できるモジュールと4番目のバージョンが必要でした。 最初にテストされたモジュールは、5番目のバージョンと排他的に機能しましたが、後で他のモジュールもありましたが、リクエストを同期的に実行できるモジュールを見つけることはできませんでした。



ドキュメントを勉強した後、C ++で必要なモジュールを記述し、node.jsのアドオンとして設計できるという結論に達しました。モジュールの作成プロセスに慣れたい場合は、catにようこそ。



ツール



Linuxでモジュールを作成します。 必要なツールから：

  • mysql
  • ノード
  • node-gyp
  • Gcc


これがすべてディストリビューションにどのようにインストールされるかは、node-gypを除き、ノードのインストールに付属するnpmを介してインストールされることを自分で知っておく必要があります。

npm install node-gyp







モジュールを書く



したがって、node.jsは、Google V8のJavaScriptエンジンで構築されたプラットフォームです。 したがって、このモジュールでは、V8エンジンのオブジェクトを処理する必要があります。 私の人生を単純化するために、このエンジンのオブジェクト用のチートシートを用意することをお勧めします。これを選択しました 。また、MySQLを使用するためのC関数への参照も選択しました 。



モジュールは何から始まりますか？


まず、必要なヘッダーを記述して名前空間を設定した後、ファイルを作成する必要があります。私の場合はmysql_sync.ccです。 

 #include <node.h> #include <node_buffer.h> #include <v8.h> #include <mysql.h> using namespace v8;


node.jsのモジュールの操作はNODE_MODULEマクロの実行から始まります。このマクロには、モジュール名と、モジュールが接続されたときに実行される関数の名前が渡されます。 

 void init(Handle<Object> target) { target->Set(String::NewSymbol("create"), FunctionTemplate::New(create)->GetFunction()); } NODE_MODULE(mysql_sync, init)


したがって、 init関数は何も返しませんが、オブジェクトが渡されます。 Setメソッドを使用してcreateという名前でこのオブジェクトにプロパティを追加します。名前はV8 Stringクラスのオブジェクトであり、目的の行を渡す静的関数NewSymbolを使用して作成されます。 このプロパティの値は、 createという名前の関数から作成される関数になります 。

すべてが非常に単純ですが、1つありますが、この関数はモジュールの最初のロード時に1回だけ呼び出され、その後ノードはinit関数の出力で受信したオブジェクトをキャッシュし、再度呼び出しません。

ここでcreate関数を追加する場合、モジュールをコンパイルし、ノードで次のコードを実行します 

 console.log(require('./mysql_sync.node'));


その結果、画面に次の結果が表示されます。 

 { create: [Function] }


最初のステージの準備ができました。 作成機能に進みます。



モジュールのオブジェクトを作成する


create関数のコードも同じです。 

 Handle<Value> create(const Arguments& args) { HandleScope scope; Handle<Object> ret = Object::New(); node::Buffer *buf; buf = node::Buffer::New((char *)mysql_init(NULL), sizeof(MYSQL)); ret->SetHiddenValue(String::NewSymbol("MYSQL"), buf->handle_); ret->SetHiddenValue(String::NewSymbol("connected"), Boolean::New(0)); ret->Set(String::NewSymbol("connect"), FunctionTemplate::New(connect)->GetFunction()); ret->Set(String::NewSymbol("query"), FunctionTemplate::New(query)->GetFunction()); return scope.Close(ret); }


このコードの戻り値は、クラスV8 Valueのオブジェクトです。 このクラスは、JavaScriptから呼び出された場合、C ++と同様にすべてのJavaScript関数を返します。 関数によって返されたオブジェクトのプロパティを格納する新しいretオブジェクトを作成します。 ここで、できればMYSQL構造体へのポインターを初期化します。これは、他のMySQL関数と連携してオブジェクトに何らかの方法で格納するために必要です。 見つかったすべての中で、node.js自体に記述されているBufferオブジェクトは、構造を保存するのに適していました。 ノード::バッファー::新しいコンストラクトを使用して、必要なサイズの新しいオブジェクトを作成し、そこにMYSQL初期化構造を配置します（返された結果を確認するのはいいと思いますが、再度複雑にしたくないため、一部のチェックは省略します） 。

オブジェクトにMYSQLを保存するが、ユーザーにアクセスを許可しないために、オブジェクトの非表示フィールドに構造を保存するオプションはSetHiddenValueメソッドを使用して選択されました。これは、通常のプロパティではなく、非表示プロパティを作成するという点を除いて、Setメソッドに完全に似ています。 JavaScriptコードからアクセスできません。 また、 接続されたプロパティを非表示フィールドに保存します。これは後で役立ちますが、今度はV8ブールオブジェクトを（値Falseで）配置します。 さらに2つの関数、 connectとqueryを追加した後。 そして最後に、scope.Close（ret）を使用して関数を呼び出したオブジェクトを返します。

最後に、新しいオブジェクトを作成する関数を取得し、このオブジェクトのサービスデータを含む2つの非表示プロパティと、必要な関数を格納する2つのパブリックプロパティを追加します。

指定した2つの関数でスタブを作成し、指定したコードを実行する場合： 

 console.log(require('./mysql_sync.node').create());


その後、次の結果が得られます。 

 { connect: [Function], query: [Function]}


モジュールのメソッド


次に、モジュールのメソッドについて説明します。

接続方法： 

 Handle<Value> connect(const Arguments& args) { HandleScope scope; Handle<Object> ret = Object::New(); Handle<Object> err = Object::New(); MYSQL *mysql; bool ok=true; mysql = (MYSQL *)args.Holder()->GetHiddenValue(String::NewSymbol("MYSQL"))-> ToObject()->GetIndexedPropertiesExternalArrayData(); if(args.Length()==4){ for(int i=0; i<4; i++) if(!args[i]->IsString()) ok=false; } else { ok=false; } if(ok == true){ String::AsciiValue host(args[0]->ToString()); String::AsciiValue user(args[1]->ToString()); String::AsciiValue pass(args[2]->ToString()); String::AsciiValue db(args[3]->ToString()); mysql_real_connect(mysql, *host, *user, *pass, *db, 0, NULL, 0); args.Holder()->SetHiddenValue(String::NewSymbol("connected"), Boolean::New(1)); err->Set(String::NewSymbol("id"), Uint32::New(mysql_errno(mysql))); err->Set(String::NewSymbol("text"), String::NewSymbol(mysql_error(mysql))); } else { err->Set(String::NewSymbol("id"), Uint32::New(65535)); err->Set(String::NewSymbol("text"), String::NewSymbol("Incorect parametrs of function")); } ret->Set(String::NewSymbol("err"), err); return scope.Close(ret); }


それから私は最初の難しさに出くわしました。このメソッドを呼び出したオブジェクトは明らかに与えられておらず、オブジェクトにはさらなる作業に必要な2つの隠しフィールドが含まれています。 ただし、関数への引数はV8 Argumentsオブジェクトによって渡されます。その説明を詳しく調べてみると、渡されたオブジェクトへの参照が格納されていることがわかります。 それを取得するには、 Holder（）メソッドを使用します。その後、 MYSQL構造を持つ隠しフィールドを取得し、GetIndexedPropertiesExternalArrayData（）メソッドを使用して、構造自体へのポインタを取得します。 コードにはこれ以上注目すべきものはありません;渡されたパラメーターの数とタイプをチェックします。 すべてがmysql_real_connect（）関数と呼ばれている場合、mysqlエラーが発生し、エラーオブジェクトを作成し、フィールド値としてエラーを追加します。 パラメーターが予期したものと異なる場合、エラーオブジェクトにエラーを追加します。 次に、 errオブジェクトを「err」フィールドとしてretオブジェクトに追加し、このオブジェクトを返します。

クエリ関数から、最初に選択の完全な結果を取得し、一度に1行ずつ引き出したくないので、着信パラメータと接続が確立されるためのすべてのチェックの後。 要求を実行し、応答をV8配列の行オブジェクトに追加します。 各応答行は、プロパティ名がクエリ結果のフィールドの名前であるオブジェクトに記録され、値は受信した実際のデータです。 最初はこれを行い、すべてのデータをV8 Stringに変換しましたが、より便利な結果が必要でした。

その結果、タイプを持つフィールドは次のように決定されました。

  • MYSQL_TYPE_DECIMAL
  • MYSQL_TYPE_TINY
  • MYSQL_TYPE_SHORT
  • MYSQL_TYPE_LONG
  • MYSQL_TYPE_LONGLONG
  • MYSQL_TYPE_INT24
  • MYSQL_TYPE_FLOAT
  • MYSQL_TYPE_DOUBLE
JavaScript番号に対応するV8番号にキャストされます。



タイプのあるフィールド：

  • MYSQL_TYPE_TINY_BLOB
  • MYSQL_TYPE_MEDIUM_BLOB
  • MYSQL_TYPE_LONG_BLOB
  • MYSQL_TYPE_BLOB
バイナリがチェックされ、バイナリBLOB'yである場合はバッファに変換され、そうでない場合はV8文字列でチェックされます。



そして、タイプを持つフィールド：

  • MYSQL_TYPE_TIMESTAMP
  • MYSQL_TYPE_DATE
  • MYSQL_TYPE_TIME
  • MYSQL_TYPE_DATETIME
  • MYSQL_TYPE_YEAR
  • MYSQL_TYPE_NEWDATE
V8 Dateに変換することが決定されました。 そして、ここで障害が発生しました。 V8 Dateオブジェクトを作成するには、Unixタイムスタンプを渡す必要があり、mysqlはYYYY-MM-DD HH：MM：SSの形式でフィールドを返します。 行解析とそれ以上の変換を書きたくありませんでした。 同時に、JavaScript自体がそのようなエントリをUNIXタイムスタンプに完全に変換することを思い出しました。 また、V8は利用可能ですので、使用しないでください。 これを行うには、Script :: Compileメソッドを使用してV8 Scriptクラスのスクリプトオブジェクトを作成し、スクリプト文字列new Date（Date.parse（mysql_field_value））を渡します。 次に、Run（）メソッドを呼び出します。このメソッドは、JavaScriptコードを実行して取得したオブジェクトを返します。 そして、それをV8配列の行に入れます。 あまり美しくないかもしれませんが、非常に興味深いものです。

これをすべてコンパイルする必要があります。そのためには、次の内容のbinding.gypファイルを作成する必要があります。 

 { "targets": [ { "target_name": "mysql_sync", "sources": [ "mysql_sync.cc" ], "include_dirs": [ '/server/daemons/mysql/include/mysql/' ], "link_settings": { 'libraries': ['-lmysqlclient -L/server/daemons/mysql/lib/mysql/'], 'library_dirs': ['/server/daemons/mysql/lib/mysql/'], }, } ] }
ここに奇妙な方法が示されているという事実に注意を払うようにお願いします（おそらくあなたにとっては異なるでしょう）ので、コマンドを使用できます：

mysql_config --include --libs





これで、実行が残ります。

node-gyp configure





node-gyp build





cp build/Release/mysql_sync.node ./





モジュールはすぐに使用できます。テストのために、次のコードを作成します。 

 var mysql = require('./mysql_sync.node').create(); console.log(mysql.connect("localhost", "login", "pass", "test")); console.log(mysql.query("select * from tmp");
