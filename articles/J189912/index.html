Androidアプリケーションのアルファおよびベータテスト

最近まで、単純にapkファイルのアップロードからホスティングまで、testflightapp.comなどの専門サイトを使用したり、 自転車を使用して、さまざまな公開方法を使用してアプリケーションをテストしました。



Google Play開発者コンソールの次の更新では、特定のユーザーサークルでアプリケーションをテストする機能が追加されました。

apkファイルをダウンロードする前にすぐにアプリケーションが公開された場合、2つの中間ステップが追加されました。







Yandex.Photosで表示



アプリケーションの次のバージョンをダウンロードするとき、次のセクションを選択します。

  • アルファテスト
  • ベータテスト
  • 生産




アルファテストからベータ版またはプロダクションにすぐに、ベータ版からプロダクションに変換できます。



アプリケーションが公開されていないが、テスト段階にある場合に便利な機能です。テスターのみが使用でき、他のユーザーは検索または直接リンクを介して使用できません。



テスターに​​なるには、ユーザーは開発者が作成したgoogle +コミュニティ（アプリケーション設定に表示）を入力し、 play.google.com / apps / testing / com.yourdomain.packageにアクセスする必要があります。



Yandex.Photosで表示



現時点では、小規模なソーシャルアプリケーションで作業するための非常に便利なシナリオがあります。

  1. 特別なアカウントで同じデバイスで開発とデバッグを行っています。
  2. 私は別の機能の作業を終了し、このデバイスでデバッグし、Google Playでアルファ版を公開しています
  3. Androidのさまざまな形式とバージョンの多くのデバイスが、個別にバージョンに更新されます
  4. 1〜2日以内に、すべてのデバイスがテストされます
  5. すべてが機能し、このバージョンは次の段階である利益に進みます。




長所：

  • テスターの便利な管理
  • 潜在的なセキュリティリスクである、既知のソースからのアプリケーションのインストールを許可するためにテスターがチェックを入れる必要はありません。
  • ユーザー向けに市場からアプリケーションをインストールすると、より安全で、「より強固」に見える
  • ベータテスターの大規模なグループは、自動的に公開アプリケーションのユーザーになり、統計も追加されます。
  • 通常のエラートラッカー、サードパーティのライブラリをアプリケーションに追加する必要はありません
