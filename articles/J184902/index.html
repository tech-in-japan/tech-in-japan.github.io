Visual Studio 2013プレビューがリリースされました

数週間前、Teched North America 2013カンファレンスでVisual Studio 2013が発表され、この環境の新機能のいくつかが明らかになりました。 昨日、Build 2013カンファレンスで、 今すぐダウンロードできるVisual Studio 2013、Net Framework 4.5.1、およびTeam Foundation Server 2013のプレビューバージョンが発表されました。



Visual Studio 2013およびTeam Foundation Server 2013の新しいバージョンに短いメモの一部として含めた革新について伝えるのは難しいため、7月5日に開催されるオンライン会議の準備が現在進行中です。 このオンラインイベントの一環として、マイクロソフトの専門家が新しいVisual Studio 2013開発およびテストツールを紹介し、会議では、Visual Studio 2013で作成できるアプリケーション開発の世界からの最新ニュースを紹介し、最新のアプリケーションライフサイクル管理テクノロジーを紹介しますTeam Foundation Server 2013に基づいています。

必ず7月5日にご参加ください！





.NET



Visual Studio 2013と一緒に、会議は.NET Frameworkの更新を発表し、その新しいバージョンはインデックス4.5.1の下でリリースされ、多くの革新と新機能が含まれます。 この更新プログラムはVisual Studio 2013の暫定版に含まれていますが、Windows 8、Windows 7、Windows Vista、および対応するサーバーOSのオペレーティングシステムに個別のパッケージとしてインストールすることもできます。

.NET 4.5.1の開発中に行われた主な努力は、デバッグと診断の機能に関連していました。 たとえば、多くの.NET開発者は、C ++プログラマーが長年利用できる機能を求めました。これらの値が宣言された変数に格納されていなくても、デバッガーで関数によって返される結果を直接表示します。 .NET 4.5.1およびVisual Studio 2013では、この機能が登場しました。 この関数は、その関数を別の関数の引数として使用する場合に特に役立ちます。





User Voice Webサイトで多くの票が投じられたもう1つの新しい機会は、64ビットモードでのアプリケーションデバッグ中にコードを変更する機能です。





.NETプラットフォームの変更を必要とするもう1つの重要な革新は、非同期コードをより明確にデバッグする機能です。 Windows 8用のアプリケーションを開発する多くのプログラマーは、非同期コードが高速で応答性の高いアプリケーションを作成するための主要なツールになったことを知っています。 このようなコードのデバッグは日常的なタスクになっており、開発者のコ​​ミュニティからこの分野に利便性をもたらすことが非常に期待されています。 これは主に、プログラムが停止したときに、スタックウィンドウに表示される中間呼び出しでは呼び出しの順序を理解することができなかったため、非同期関数のコンテキストを把握することが非常に困難であったという事実によるものです。 Visual Studio 2013では、非同期関数の実行コンテキストに関する詳細かつ直感的な情報を表示できるようになり、そのようなコードのデバッグが大幅に容易になりました。





この革新は、.NETプラットフォームの言語で開発しているプログラマーだけでなく、JavaScriptおよびC ++の開発者も利用できます。

最新のデバッグおよび診断機能に加えて、.NET 4.5.1にはパフォーマンスの更新が含まれます。 たとえば、ヒープ上の大きなオブジェクトのオンデマンドでのGCコンパクト化、マルチコアシステムでのアプリケーション起動の高速化など。 Habib Kheydaryanのレポートから最新の.NET 4.5.1の詳細を学ぶことができます。



C ++



C ++コミュニティから最も期待されている革新の1つは、C ++ 11標準の完全なサポートです。 Visual Studio 2013に含まれるコンパイラには、委任コンストラクター、生の文字列リテラル、明示的な変換演算子、可変長テンプレートなどの期待される機能のサポートが含まれています。





コンパイラの革新に加えて、多くのC ++開発者は、.NETプログラマーが長い間使用してきたVisual Studio機能を高く評価するでしょう。 これは「Just My code」デバッグモードであり、複雑なアプリケーションをデバッグするときにソリューションの一部として作成されたコードのみを表示し、重要なポイントのみに集中して、スタックのデバッグウィンドウからCRTや他のライブラリへの呼び出しを省略できます：





Visual Studioエディターは、C ++コードに関連する多くの革新、たとえば、コードファイルとそれに対応するヘッダーファイル間のクイックナビゲーションモード、およびC ++コードのフォーマット機能の改善も受けました。





これに加えて、C ++ AMPおよびC ++自動ベクトル化プログラム、C ++ REST SDKのパフォーマンス、およびデバッグの改善（C ++およびJavaScriptコードなどの新しい混合デバッグモード、 プロファイルガイド付き最適化の革新など）が改善されました。





Herb Sutterのレポート「 The Future of C ++ 」とVisual C ++チームブログのTarek Madcourtのレポート「 Visual Studio 2013のC ++開発者向け新機能」のレポートから、C ++の新製品と将来について詳しく知ることができます。



Xaml



C ++および.NET言語の変更に加えて、多くの新機能もXAMLエディターに実装されており、このテクノロジーに基づく対話型アプリケーションの開発を簡素化および高速化するのに役立ちます。 これらは、データバインディングとデータバインディングのIntelliSenceサポート、コードスタイルとスニペットを使用する場合の「Go to Definition」オプションなどの新機能です。 ビジュアルデザイナーのBlendには、要素の配置を簡素化するルーラーとグリッドが含まれるようになりました。 Blendには、スタイルとテンプレートに関連するいくつかの変更も含まれており、それらを使用すると作業が大幅に簡素化され、使用状況に応じてスタイルを編集できます。 多くの新しいコントロール（AppBar、Hub、Flyout）に対して、複数のアプリケーションの起動に頼ることなく最終結果をより正確に確認できるデザインモードが登場しました。





レスポンシブアプリを特に重視する人は、XAML UI Responsivenessツールキットを好むでしょう。 Visual Studio 2012にはJavaScript HTMLアプリケーション用の同様のツールが既に含まれており、ユーザーインターフェイスのあらゆる種類の「フェード」および「フリーズ」に対処できます。 Visual Studio 2013では、これを使用して応答性とXAMLアプリケーションを学習できます。





また、Windowsストアアプリケーションの品質を向上させるために、Visual Studio 2013にはプログラムされたインターフェイステスト（コード化されたUIテスト）のサポートが含まれています。

JavaScriptとHTML



Visual Studio 2012には、HTMLおよびJavaScriptのサポートに関連する多くの新機能が含まれています。 このリリースでは、すでに説明した非同期デバッグに関連する機能に加えて、いくつかの新しい機能も含まれています。

まず、これらは、コードナビゲーションに関連する関数です。たとえば、スコープへの遷移をサポートする「定義に移動」の定義への遷移、使用すべきでない関数のリマインダー（非推奨API）、識別子の強調表示などです。





DOMブラウザ（DOM Expolrer）に関連する多くの改善も行われました。 たとえば、IntelliSenceモードで検索、直接編集、組み込みスタイルなどをサポートしています。





Blend for HTMLは、グリッドとルーラーもサポートしています。 これに加えて、CSSアニメーションを編集する機能が登場しました。





詳細については、Build 2013 Papersを参照してください。

診断



既に述べたように、.NET 4.5.1およびVisual Studio 2013には、非同期コードのデバッグ、64ビットの「編集および続行」モード、C ++の「Just My Code」モードなど、アプリケーションのデバッグと診断を簡素化する多くの革新があります。 これに加えて、開発されたアプリケーションのパフォーマンスを診断および調査するためのすべてのツールが「手元に」集まるハブ「パフォーマンスと診断」というまったく新しいセクションも登場しました。





最も興味深い新規性の1つは、エネルギー消費の診断ツールです。 バッテリーの寿命はデバイスの利便性の重要なパラメーターの1つであり、アプリケーションがエネルギーを過剰に消費すると、デバイスの寿命に直接影響する可能性があります。 これらのパラメーターをより詳細に調査するために、Visual Studio 2013には「エネルギー消費」プロファイリングツールが含まれています。 これにより、アプリケーションの動作、継続的に起動したときにデバイスが動作する時間、およびコードのどの領域が最もエネルギーを消費するかを詳細に調べることができます。





これに加えて、Visual Studio 2013では、マネージメモリを分析するための新機能も導入されました。 多くの場合、開発者は、「戦闘」環境での動作中のアプリケーションの動作に関する詳細情報を取得し、プロセスメモリにあるオブジェクトを見つけ、メモリリークがあるかどうかを判断する必要があります。 Visual Studio 2013には、マネージメモリ分析のサポートが含まれています 。 要求に応じて生成される特別なダンプファイルを使用して、Visual Studioでこれらのファイルの詳細な診断と分析を「管理メモリのデバッグ」メニューを使用して実行できます。





* .dmpファイルをダウンロードした後、プログラマは、プロセス内の.NETオブジェクトを確認し、ダイナミクスにおけるアプリケーションの動作のより詳細な調査のために2つの異なるダンプを比較する機会があります。

オンラインIDEおよびオンラインアプリケーション



現在、業界では、オンラインでのデータの保存や処理に関連するものなど、便利な機能を備えたアプリケーションに多くの注意が払われています。 Visual Studio 2013は、クラウドの設定ストレージ設定のおかげで、今ではオンラインIDEと呼ぶことができます。どのワークステーションを使用していても、カスタマイズした環境をいつでも利用できます。 すべての設定（テーマ、キーボードショートカット、エディター設定など）をクラウドサービスに保存できます。





また、Visual Studio 2013では、このようなアプリケーションを簡単に開発できます。 このリリースに含まれる革新​​の1つは、Windows Azureモバイルサービスアシスタントです。 ほんの数ステップで、アプリケーションのオンラインサービスを作成、構成、展開、および実行できます。

プログラマーの利便性



Visual Studio開発チームは、プログラマがIDEをどのように使用し、どのような改善が生産性と使いやすさを改善できるかを常に調査しています。 Visual Studio 2013には、この領域に多数の新製品が含まれています。

これらの革新の1つはCodeLensテクノロジー（情報コードインジケーター）です。 この技術については、TechEdカンファレンスですでに議論されています。 この機能により、タイプ、リンクの数、合格したテスト、およびコードの変更に関する重要な情報をコード内に直接表示できます。





この領域のもう1つの目新しさは、ピーク定義ポップアップウィンドウです。 Visual Studioには、新しいドキュメントウィンドウを開く「Go to Definiton」機能が既にあります。 また、小さなPeek Definitonウィンドウを開いて、目的の機能の実装の詳細を確認することもできます。





さらに、生産性向上ツールパッケージに含まれる機能に基づいて、エディターのスクロールバーに、ファイルおよびブレークポイントの現在お​​よび未保存の変更に関する重要な情報が表示されるようになりました。
