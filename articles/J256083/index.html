WEB 3.0。 サイト中心主義からユーザー中心主義へ、無秩序から多元主義へ

テキストは、著者が彼のレポート「 進化の哲学とインターネットの進化 」で表現したアイデアを要約しています。



最新のウェブの主な欠点と問題：



  1. 元のソースを見つけるための信頼できるメカニズムが存在しない場合に、コンテンツが繰り返し複製されるネットワークの壊滅的な輻輳。
  2. コンテンツの分散と非一貫性は、主題ごと、特に分析レベルごとに徹底的な選択を行うことができないことです。
  3. コンテンツの表示形式の出版社への依存（多くの場合、ランダムで、独自の、通常は商業的な目標を追求しています）。
  4. 検索結果とユーザーのオントロジー（関心のある構造）との弱いつながり。
  5. ネットワーク（特に、ソーシャルネットワーク）のアーカイブされたコンテンツの低可用性と貧弱な分類。
  6. コンテンツの組織化（体系化）への専門家の小さな参加は、彼らの活動の性質により、知識の体系化に日々従事しているのは彼らですが、仕事の結果はローカルコンピューターにのみ記録されます。




ネットワークが乱雑で無関係であることの主な理由は、Web 1.0から継承したサイトデバイスです。このデバイスでは、ネットワークの主要人物は情報の所有者ではなく、その場所の所有者です。 つまり、マテリアルコンテンツキャリアのイデオロギーがネットワークに転送され、主なものは場所（図書館、キオスク、フェンス）と主題（本、新聞、紙）、そしてそのコンテンツのみでした。 しかし、現実の世界とは異なり、仮想世界のスペースは制限されておらず、費用もかかるため、情報を提供する場所の数は、一意のコンテンツの単位の数を桁違いに超えています。 Web 2.0はこの状況を部分的に修正しました。各ユーザーは、ソーシャルネットワーク上のアカウントとある程度の設定の自由という、独自の個人用スペースを受け取りました。 しかし、コンテンツの一意性の問題は悪化するだけでした。コピーペーストテクノロジーにより、情報の重複が桁違いに増加しました。

現代のインターネットのこれらの問題を克服するための努力は、2つの方向、ある程度相互接続された方向に集中しています。



  1. サイト間で配布されるコンテンツをマイクロフォーマットすることにより、検索の精度を向上させます。
  2. 信頼できるコンテンツの「リポジトリ」の作成。


もちろん、最初の方向では、キーワードを指定するオプションと比較してより関連性の高い検索を行うことができますが、コンテンツの重複の問題を除去せず、最も重要なことは、偽造の可能性を排除しません-情報の体系化はほとんどの場合、著者ではなく所有者によって行われ、確かにそうではありません検索の関連性に最も関心がある消費者。

2番目の方向（Google、 Freebase.Com 、 CYCなど）の開発により、明確に信頼できる情報を取得できますが、可能な範囲でのみ可能です-共通の基準がなく、データを整理するための共通のロジックが存在しない分野での知識多元性の問題 新しい（現在の）コンテンツを取得、整理、およびデータベースに組み込む問題を解決することは困難です。これは、現代の社会指向ネットワークの主な問題です。



レポート「 進化の哲学とインターネットの進化 」で概説されているユーザー中心の積極的なアプローチはどのようなソリューションを提供しますか？



  1. サイト構造の拒否-ネットワークの主要な要素はコンテンツの単位であり、配置場所ではありません。 ネットワークノードはユーザーである必要があり、それに関連して構成されたコンテンツユニットのセットはユーザーオントロジーと呼ばれます。
  2. 論理相対論（多元主義）。情報組織の単一の論理が存在することは不可能であると述べ、同じ主題内であっても無限の数の実質的に独立した存在論的クラスターの必要性を認識しています。 各クラスターは、特定のユーザー（個人または一般化）のオントロジーです。
  3. オントロジーの構築に対する積極的なアプローチは、オントロジー（クラスター構造）が形成され、コンテンツジェネレーターのアクティビティに現れることを意味します。 このようなアプローチでは、コンテンツの生成からオントロジーの生成へのネットワークサービスの再配置が必然的に必要になります。これは、基本的に、ネットワークでアクティビティを実装するツールを作成することを意味します。 後者は、多くの専門家をネットワークに引き付け、その機能を保証します。


最後の項目は、より詳細に説明できます。



  1. オントロジーは、専門的な活動の過程で専門家によって作成されます。 このシステムは、あらゆる種類のデータを導入、整理、処理するためのすべてのツールを専門家に提供します。
  2. オントロジーは、専門家の活動で特定されます。 あらゆるアクティビティの操作の大部分がコンピュータで実行または記録されるため、今では可能になっています。 専門家はオントロジーを構築するべきではなく、ソフトウェア環境で行動するべきであり、それは同時に彼の活動とオントロジージェネレーターの主要なツールでもあります。
  3. オントロジーは、アクティビティの主な結果になります（システムとプロフェッショナルの両方）-プロフェッショナルな作業（テキスト、プレゼンテーション、テーブル）の成果物は、このアクティビティのオントロジーを構築するための口実にすぎません。 オントロジーは製品（テキスト）に添付されていませんが、テキストは特定のオントロジーで生成されたオブジェクトとして理解されます。
  4. オントロジーは、特定のアクティビティのオントロジーとして理解されるべきです。 どのくらいの活動-非常に多くのオントロジー。




したがって、主な結論：Web 3.0は、サイト中心のWebからセマンティックユーザー中心のネットワークへの移行です-任意に構成されたコンテンツを含むWebページのネットワークから、無限数のクラスターオントロジーに結合された一意のオブジェクトのネットワークへ。 技術面では、Web 3.0は、あらゆる種類のコンテンツを入力、編集、検索、表示するためのあらゆるツールを提供する多数のオンラインサービスであり、同時にユーザーアクティビティのオンログ化を提供します。
