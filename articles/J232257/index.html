Web /モバイルアプリケーション開発の加速

アイデアが有用なものを作成することである場合、通常、できるだけ早くプロトタイプ（またはバージョン1.0）を作成する必要があります。 誰かにとって、迅速な結果を見ることは、アイデアをさらに発展させる良い動機です。 他の人にとっては、主なことは「開始」することです。完成品の完成/作り直しはゼロから書くよりもはるかに簡単であるというよく知られている真実です。 したがって、次のお茶と金融市場の議論の過程で、私たちはアイデアやニュースを交換するための簡単なサービスを作成するだけでなく、金融市場の現在の状況（いわゆるトレンド）を決定するというアイデアを持っていました-トレンドを知っているので、より効率的に取引することができます。

要件は、Webサービス、モバイルバージョン（できればアプリ）、簡単な集合管理パート、およびシンプルなインターフェースでした。

この記事では、CxInvestorアプリケーションのWebバージョンとモバイルバージョンの両方を非常に迅速に「ブラインド」する方法について説明します。

サーバーレスFirebase






私のphpの知識は90年代にまで及んだため、 ASP.NETまたはJavaで小型で軽量のサーバーを作成したくありませんでした。 「サーバーレス」のWebアプリケーションを作成するという考えは良すぎるように思えましたが、インターネットを掘り下げた後、結局Firebaseを試すことにしました。 アカウント、データベースを作成し、簡単な例を実装するのにたった20分しかかかりませんでした。 Firebaseには、「リアルタイム」データベースに加えて、ユーザー、特権、およびFacebookやTwitterとのさまざまなログイン統合システムが組み込まれていることに注意してください。 実際、ユーザー、パスワード、特権を管理するシステム全体がボールに与えられました。

まあ...実際にはボールではありません、あなたがより大きなアプリケーションを持っている場合-最低有償アップグレードは月額50ドルです。

レスポンシブブートストラップとAngularJs






フロントエンドの場合、選択はかなり明白でした。TwitterBootstrapを使用すると、任意のデバイスでサービスを簡単に操作できますが、 AngularJsはすぐには思い付きませんでした。 AngularJsを支持する主な論点は、それが本格的なフレームワークであるということでした。つまり、2つまたは3つのメインJavaScriptライブラリ（たとえば、RequireJs + KnockoutJs）のバンドルを学ぶ必要はありません。 また、AngularJsには優れたモジュール構造があり、プレゼンテーションをロジックから分離し、長いパスをサポートしています。

一般に、AngularJsは私たちを失望させませんでした。発生した問題はすべてgoogleを使用して迅速に解決され、アプリケーションに新しい機能を追加するのは簡単で快適でした。

ビルドシステム


ソースの難読化を自動化し、バージョン管理を維持する必要があったため、小さなビルドシステムを作成するというアイデアがすぐに生まれました。 判明したように、実際には、ftpクライアントでコピーを実際に開始する前に、他の多くのタスクを自動化することができます（そして必要です）（自動化することもできます）。

PowerShellは、そのモジュール性、使いやすさ、およびコマンドラインパラメーターとの良好な連携により、スクリプトビルドとして選択されました。

Closure Compilerがjavascriptコンパイラとして選択され、cssはyuicompressorを使用してコンパイルされました。 



  Write-Output "Compiling js files..." java -jar compiler.jar --language_in ECMASCRIPT5 --angular_pass ` --js $BaseDir\scripts\jquery-2.0.3.js ` --js $BaseDir\scripts\angular.js ` ...




したがって、Webサービスは準備が完了してGoogleインデックスに追加されますが、1つの問題があります-Google ボットはページでjavascriptを実行できないため、AngularJsビューにつながるURL（たとえば、 www.mywebsite.com/#!/myView ）は実行するにはjavascriptが必要なため、 google botによって正しく読み込まれます。 これはおそらく、AngularJsを使用するときの最も不快な瞬間の1つです。WebサービスのSEOが自動的に複雑になり、開発者としての作業が増えるからです。

現時点では、Google（および他の検索エンジン）の推奨事項は、いわゆるhtmlスナップショットをサポートすることです。これは、要求されたページをjavascriptなしで完全に表示します（これにより、厳密にgoogle-HTMLスナップショットとjavascriptバージョンが非常に異なる場合、Googleは罰します）。

スナップショットは、 www.mywebsite.com /？ _ escaped_fragment _ = / myViewの特別なURLで入手できます。

気味が悪いようです...幸いなことに、Apacheルールが助けになりました。これは.htaccessファイルで定義できます。 



  RewriteCond %{QUERY_STRING} ^_escaped_fragment_=/?(.+)$ [NC] RewriteRule ^ /snapshots/%1.html? [NC,L]




そしてそれは簡単です-javascriptがすべてのデータをロードして完全に機能した後、すべてのビューを巡回し、完成したhtmlファイルを保存します。 その後、 PowerShellを使用してこのプロセスを自動化し、サーバー上のスナップショットを自動的に更新しました。 また、インターネットには、ささやかな支払いのためにすべてを行うサービスがいっぱいです（同様のハンドルを.htaccessに入れて、 GoogleボットをサブスクリプションサービスのWebサイトにリダイレクトするだけです）。
