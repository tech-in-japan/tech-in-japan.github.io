車のオークションプラットフォーム内に隠されているもの

中古車の販売サービスは、通常「広告を配置し、車を見せて、販売した」という同じ原則に基づいて構築されます。 このようなプロジェクトでは、ハイテクは主にサイト自体に集中しています。 しかし、私たちにとってはすべてが異なります。 私たちは、誰でも自分の中古車（緊急車であっても）をオークションシステムを通じて自動車ディーラーに販売できるプラットフォームを作成しました。 説明の明白な単純さの背後には、私たちが話したい自己記述およびサードパーティのソフトウェア製品の複雑なエコシステムがあります。







みなさんこんにちは！ 私の名前はイリヤ・ピャチンです。 私はCarPriceでITを担当しています。今日は、プラットフォームの内部の概要とその構成について説明します。



私たちの物語はこのパターンに従うので、時々最初から戻ってくる必要がないように、時々繰り返します。





カーレセプション



だから、人は自分の車を売りたいと思う。 彼は私たちの公開ウェブサイトにアクセスし（システムの閉鎖部分があり、後で詳しく説明します）、短いフォームに記入し、私たちの支店の1つで会議にサインアップします（そのうち45以上がロシアの21の都市にあります）。



技術資料



誰にでも見えるように、現在のマイクロサービスアーキテクチャのBitrixクライアントサイトは、実際には「フロント」の1つです。 車の所有者が入力した情報は、内部ストレージ、分析システム、コールセンター、マーケティングシステムなどに転送されます。


CarPriceオークションで車で稼げる金額を概算できる電卓も公開Webサイトで入手できます。 価格は、特定のブランド、モデル、技術条件、国のさまざまな地域の需要などに関する既存の販売統計に基づいた分析アルゴリズムを使用して計算されます。



入力後、データはコールセンターに送られます。 彼らは車の所有者を呼び出し、追加情報を明確にし、持ち込む必要のある書類を警告します。



技術資料



コールセンターはOktell / MSSQLプラットフォーム上に構築され、プライベートクラウドで動作します。 CSI、高度なユーザーポーリングシステム、クライアントCRM、さまざまなメールおよび通知サービスと統合されています。 車の所有者、車の法的ステータスなどに関する主要な情報も記録されます。


選ばれた支店に人が到着すると、彼はそこで従業員（車検の専門家）に会います。 AutoCheckと呼ばれる内部システムを使用してドキュメントをチェックします。 TCP、STS、およびパスポートからの情報がそこに入力され、AutoCheckは公共サービスの利用可能なすべてのデータベースに関する情報をチェックし、この車を販売する際に生じる可能性のあるさまざまな禁止、制限、罰金およびその他の法的問題に関するデータを収集します。



問題が見つかった場合は、車の所有者に報告し、解決方法を説明します。 未払いの罰金についてのみ話している場合は、その代金を支払うことができます。 すべてが正常であれば、同僚の検査官は、開発したモバイルアプリケーション（「検査官のアプリケーション」）を使用して、車に関するすべての情報をデータベースに入力します。 これは、特定のアルゴリズムに従って行われます。彼はメーカー、モデル、製造年を選択し、ボディ、インテリアの写真を撮り、損傷、機器などを示します。 検査アルゴリズムはすべての車で同じです。



検査員が検査を終了したら、アプリケーションの「OK」ボタンをクリックすると、すべての情報がネットワークを介してオンラインプラットフォームに送信されます。









アプリケーションの主な機能：



  • サーバーからユニバーサルスクリプトがダウンロードされます。これは、車を検査する検査官の一連のアクションに影響します。
  • 従業員は非常に異なる条件で、時にはクライアントの領域で作業するため、表示すると、アプリケーションは完全にオフラインで動作します。
  • トラフィックポリシング、FSSP、オートコードなどによる登録およびその他のアクションの制限の確認
  • スマートな圧縮と、クラウドストレージへの写真とビデオのマルチスレッド転送のための組み込みシステム。
  • OBD-2スキャナーによる独自のコンピューター診断。
  • モバイルプリンターを使用します。
  • 車の損傷の電子インデックス。


技術資料



アプリケーションアーキテクチャは、RxJava、 Dagger 、Retrofit 2 、 Realm 、Amazon SDKなどを使用してMVVMパターン上に構築されます。


売り出し中



検査官が作業を完了すると、検査官によって収集されたデータはオークションで大量になります。それらは集計され、読み取り可能な形式で表示されます。 そして、このマシンに興味があるかもしれないディーラーは通知を受け取ります。 ところで、ニューラルネットワークは、ディーラーへの通知、その行動と過去の購入の分析を部分的に担当しています。



実際、ディーラーは車自体を見ません。 オークションシステム内のカードにのみアクセスでき、検査結果と検査員が撮影した写真を使用できます。 現時点では、車は私たちのオフィスの1つにあります。



もう1つの説明：検査結果がシステムに到達すると、ディーラーに何百ものポイントをダンプしないように、データが集計されてポイントシステムに削減されます。 基準は4つのみで、それぞれに最大5つの星を付けることができます。

なぜ「スター」を導入することにしたのですか？ 実際には、検査アプリケーションのチェックリストは1000を超える項目で構成されています。 各アイテムには独自の重量があります。 さらに、リストは定期的に見直されます。何かが追加され、何かが削除されます。 したがって、ディーラーが車の状態をすばやく把握し、この車の保証ポリシーを策定するには、「スター」が不可欠です。



カードがオークションシステムに入った後、車の所有者は30分間待機します。 彼は現在のすべてのオークションを別のテレビでリアルタイムで見ることができます-ダッシュボード、ディーラーが自分の車にどのように賭けているか、どの都市から来ているか、オークション終了までの残り時間を確認します。 ほとんどの人は大いに興奮して取引を見ています。 時間がほぼ過ぎて新しい入札が到着した場合、プラットフォームはしばらくオークションを延長します。 そして、そのような延長は数回発生する可能性があるため、所有者は、需要があればあなたの車をより有益に販売できます。







技術資料



ダッシュボードはVue.jsとプッシュ通知のSPAです。 実際、これはすべての入札に関する部分的な情報を受け取るページであり、コンパクトな形式で、クライアントが存在する支店の現在のオークションからのデータを表示します。


プラットフォームの主要部分はPHPで記述されていますが、Golangの速度と負荷のために、ベットの受け入れ、ディーラーのロボット取引など、多くの最も重要なコンポーネントを実行しました。 通知送信システム（私たちにとって非常に重要なコンポーネントの1つ）は、 Slangerに実装されています 。 これは、Redisに中間ストレージを備えたプッシャーのオープンソースプロトコルベースの通知システムです。



オークションプラットフォームから、データはロジスティック、統計、マーケティングシステムに送信されます（特に、A / Bテストに使用されます）。 オークションの進行状況を監視するために、販売部門用に個別のプッシュインターフェイスが作成されました。





ディーラーとオークション



ディーラーは、ブラウザまたはモバイルアプリケーションからオークションに参加できます。 それらの多くはオフィスに座っていませんが、常に会議に出席し、車を見るので、アプリケーションは彼らにとってより関連性があります。 会社が大規模な場合は、反対に、CarPriceに座って意図的に購入した特定の基準を満たす車を購入する個々の従業員を脇に置きます。







興味のある車のカードを選択することにより、ディーラーは写真を見て、パッケージを見て、損傷を評価できます。







React + Reduxバンドルで作成されたカードには、このロットでの取引が終了するまでの残り量を示すタイマーが表示され、ベットすることができます。 たとえば、eBayでオンラインオークションに参加した場合、最後の数秒でディーラー間の小競り合いの戦いがどのように燃え上がったのかをよく想像できます。



ちなみに、Goにはディーラー向けの「自動配達」サービスを実装しました。 実際、これらはあなたが言うことができるロボットです：「この量までこの車のためにハグしてください」、そして、他の多くを見に行きます。 ベットがアウトビッドになるとすぐに、ボットは自動的にベットを高くします。



ディーラーはオークションに勝つだけでなく、所有者が車を売ることに同意した価格を提示する必要があります。 結局のところ、ディーラーの1人が最高のオファーでオークションに勝ったとしても、それが所有者に合わない場合、彼は単に拒否して去ることができます。 ところで、自動車の所有者にとって、自動車を評価してオークションに出品するための手順全体は、最終決定に関係なく完全に無料です。



しかし、ディーラーは有利な立場にあります。彼はまだ何も支払っていませんが、同時に、高品質の写真を使用して、私たちの技術的専門知識で商品を受け取ります。どこかに行き、誰かと通信する必要はありません。 ところで、撮影した車両を検査するときに検査官が撮影した写真の品質レベルを自動的に決定するアルゴリズムを導入し、写真が「あまりよくない」場合に再撮影する必要性を伝えます。







ディーラーアプリケーションの主な機能：



  • どこでも入札する機能。
  • 予想されるオークションのリストを使用すると、ディーラーの時間を計画できます。
  • 新しいオークションの内部通知およびプッシュ通知。
  • 組み込みの参加料。
  • インタラクティブマップで車を検査するときに示されたすべての損傷を表示します。

車は一時的に私たちと一緒です-私たちはすでにそれを支払いました、そして今私たちは勝者にそれを与えたいです。 たとえば、現時点では1000台以上の車を所有しています。 当然のことながら、一定の「フロー」を備えたこのような地理的に分散したフリートには、厳密な会計が必要です。 適切なサードパーティソリューションが見つからなかったため、独自のウェアハウスシステムを作成し、すべてのブランチで使用できるようにしました。 また、支店間を移動するすべての自動車を追跡したり、自動車運送業者の貨物を計画したりすることもできます。

QA



そのため、ディーラーはいくつかのロットでオークションに勝ち、車は対応する都市の倉庫に到着しました。 ここで彼は検査手順を通過します-QA、あなたは皆品質保証を知っています。 この場合にのみ、製品の新しい機能をチェックするのではなく、車の状態を再評価することについて話します。 受け入れ評価は十分に正確でしたか？ 欠落している欠陥はありますか？



QAの従業員には、チェックリスト付きの独自のモバイルアプリケーションもあり、最初の受入検査の結果がアップロードされます。 車はWi-Fi接続のある温室条件ではなく、保管場所で検査されることが多く、しばしば「文明から遠く離れている」ため、低レベル通信で動作するようにアプリケーションを適合させました。 これは、100万人を超える都市でのLTEまたは4Gの優れたカバレッジであり、それでもすべてではありません。 そして、小さな都市で、そして郊外でさえ、エッジはまれかもしれません。 検査の前に、従業員は検査するオフィスのマシンにデータをロードします。これは、各車について最大500 MBのデータと写真であり、フィールドに行きます。 そこで彼は情報を修正し、接続が表示されると、プラットフォームとの自動同期が実行されます。

  • 内部ビジネスプロセスの開始。

ロングオークション



オークションに車が到着しましたが、何らかの理由で30分以内にディーラーの注意を引くことができなかった、または提示された金額が少なすぎたため、所有者が同意しないことを理解しています。 しかし同時に、私たちの顧客ベースには、車が興味を持っている人、または車をもっと提供したいと思っている人がいるが、今は参加できませんでした。 この場合、私たちは「国内ディーラー」と呼び、リスクを取って車を買い戻すよう頼み、ディーラーよりも多くの金額を所有者に提供します。 書類を完成させた後、30分間という制限のない、いわゆるロングオークションに車両を置きました。 ちなみに、多くのディーラーはそのようなロットを好みます。急ぐ必要がないため、車を安全に評価し、駐車場に来て、見て、考えてください。



所有者にいくら提供するかをどのように知るのですか？ なぜ私たちから車を後​​で買うと思うのですか？ 私たちのスペシャリストはこれに責任があり、類似の車の入札履歴を調べ、需要と顧客ベースを分析し、多くの基準に基づいて、まだこのロットを見ていないがそれに興味があるかもしれないディーラーを選択する分析システムに焦点を当てています。 現在、私たちの顧客ベースには約30,000のディーラーがあり、それらのすべてがスマートフォンやコンピューターを使用しているわけではありません。 定期的に電話をかけ、興味深いたくさんのことを話します-コールセンターでは十分ではありません。 それにもかかわらず、これらは成功するビジネスマンであり、長期オークションから潜在的に多くのダイジェストを定期的に送信できます。



また、自動推奨システムも導入しました。 ニューラルネットワークはすべてのロットを分析します。入札がない、または入札が少なすぎるものだけではありません。 これにより、特定の機械に最も関心のあるディーラーをオークションにかけることができます。

ニューラルネットワークは最初にMATLABでモデル化され、次にTensorflowに移植されました。NumPyには場所があります。 これらはすべて個別のマイクロサービスの形で機能し、特定のロットに関する情報を収集し、ディーラーのデータベースからこれらのロットに興味のある人を選択します。 彼女はディーラーの過去の行動を分析します。誰が、いつ、何のロットを見たのか、彼が取引した車、勝った車、勝たなかった車、現在購入されている車の数、支払い請求を待っていますか？


車を記述する何百もの基準に基づいて、ニューラルネットワークは特定のディーラーにとって関心のある車を識別します。 さらに、ディーラーの行動の履歴に基づいて、ニューラルネットワークは、ディーラー自身が自分で策定していない好みを強調することができます。 たとえば、彼女はディーラーがTCPまたは革以外のインテリアの所有者が3人以上いる車には興味がないと推測するかもしれません。 そして、別のディーラーはしばしば右ハンドルの「日本人」に見えます。彼らはそもそも彼を見せなければなりません。



上記を要約すると：長時間のオークションは、車の所有者に公正な価格を提供し、それから適切な金額を支払う意思のあるディーラーを見つけます。



なぜこれがすべてディーラーなのか？



おそらく、ディーラーが中古車を購入し、オークションで競い合って価格を引き上げるのはなぜでしょうか？ すべてが非常に簡単です。車の技術評価のみを実行しますが、技術的な作業は実行しません。 その後、ディーラーはこのすべてを行います。修理、交換、清掃、塗装、矯正です。 一般的に、彼らは車の価値を高め、新しい所有者を見つけるためにそれを準備し、それからプレミアムでそれを販売します。 したがって、私たちの助けを借りて、彼らは選択と購入の段階で節約します。
