ロシアのWeb開発市場でのSharePointの適用

今日は、このフレームワークで新しい生産ラインをゼロから組織する人の観点からMicrosoft SharePointについて書きたいと思います。 それにもかかわらず、SharePointはロシアのWeb市場ではかなり新しいテクノロジーであり、それを知っている人はほとんどいません。情報が多いほど良いです。







2010年後半から、ADVでSharePointでのプロジェクトを開始し、肯定的な側面（Microsoftからの強力なサポート）と否定的な側面（ロシア市場向けのかなり新しいテクノロジー）の両方に直面しました。 しかし、いずれにしても、新しいテクノロジーを導入する先駆者のように感じるのは楽しいことです。



SharePointは何に使用できますか？



基本的に、SharePointはフレームワーク+ CMSであり、これに基づいて任意のWebアプリケーションを構築できます。 たとえば、パブリックサイトの作成とイントラネットの両方に使用しますが、最初はプラットフォームはイントラネット専用に開発されました。



このプラットフォームは、オープンなビジネスプロセスを持つシステムの構築に焦点を当てています。 イントラネットとエクストラネットが互いに密接に関連している場合。 たとえば、コンテンツを公開するプロセスは、イントラネットで開始し、調整のすべての段階の後、コンテンツが公開部分にあるように設計されています。 逆のプロセスもまったく同じ方法で構築されます。外部サイトからのすべてのユーザー要求は、適切な人の責任の分野でイントラネットに送られます。



どれくらい難しいですか？



レディースプラットフォームが何を開始するのか、市場で既によく知られている人気のあるBitrixプラットフォームと比較するために、SharePointが提供するものを見るのは興味深いことです。



統合とクラスタリングの可能性については特に触れません。不正であるためです-結局、これらは産業システムの機能であり、通常Bitrixを使用して解決されるタスクには必要ありません。

SharePointの珍しいと同時に有用な機能はExcel Servicesであることが判明しました。これは、Excelスプレッドシートをオンライン計算機のデータおよび計算式のリポジトリとして使用できるサービスです。 それ以外の場合、両方のシステムの機能は似ています。



SharePoint for Public Sites-長所と短所



SharePointを使用してパブリックサイトを作成する場合は、技術的な側面からスキルが必要です。



たとえば、パブリックサイトで非常に重要な最新の特殊効果を使用して、きれいなクロスブラウザーレイアウトを「拡大」するには、Professional SharePoint 2010 Branding and User Interface Designを学習することをお勧めします。 それはまだロシア語に翻訳されていませんが、理解を開始するのに役立つメモがインターネット上にあります。



さらに、可能であれば、WebPartsの代わりにWebControlsを使用することをお勧めします。機能は似ていますが、WebControlsのカスタマイズの可能性ははるかに広くなっています。 たとえば、Model View Presenterテンプレートを使用するなどの代替手段もありますが、これは別の記事のトピックです。



また、許可されていないユーザーによるアクセスを許可するには、IISとSharePointの設定を考慮する必要があります。



プラスがあります：

  • サードパーティの開発者を含む、既製のモジュールの大きなライブラリ。
  • ASP.NETでは、いつでもモジュールをゼロから作成できます。
  • Microsoft Officeアプリケーションを使用したことのある人なら誰でも馴染みのある、「MS Office」のインターフェイスを備えた優れたインラインビジビグ。
  • コンテンツを公開するための組み込みの非常に使いやすいドキュメント管理システム。
  • 1つのWebサーバーに基づいて複数のサイトを簡単に作成できます。モバイルバージョン、言語バージョン、プロモーションサイト。
  • 原則として、SharePointをECMとして使用すると便利です
  • 豊富なキャッシュオプション
  • ASP.NETはPHPではありません。コンパイルされたアプリケーションは、解釈されたスクリプトよりも速く実行されます
  • SharePointは簡単にクラスター化でき、スケーラブルです。 仮想化と組み合わせると、アプリケーションのスケーリングは簡単なプロセスになります。




したがって、しばらくして「トピックを入力」すると、SharePointは複雑なパブリックサイトを作成するための柔軟で便利なツールになります。



マイクロソフトには、パートナー向けの非常に開発されたサポートシステムがあります。 フリーソフトウェアの大規模なパッケージを取得できるBizSparkプログラムから、特定のプロジェクトを支援するSharePointアーキテクトの割り当てまで。



イントラネット



明らかな理由により、イントラネットの例を示すことはできません。そこで、イントラネットを構築するためのSharePointの機能について説明します。

  • イントラネットとパブリックサイトの簡単な統合。 たとえば、公開Webサイト経由で送信されたアプリケーションは、イントラネット上の責任者のページに移動します。 さらに、前述したように、SharePointをECMとして使用し、イントラネットからパブリックポータルに情報を公開できます。
  • 多くの既製モジュール：ワークフロー、カレンダー、Outlookとの統合、ドキュメントのページとコンテンツの検索、組織構造、Active Directoryとの統合など、無限に。




SharePointが正当化されるとき



会社のITアーキテクチャがMicroSoftテクノロジーを積極的に使用してすでに構築されている場合、内部および外部のWebアプリケーションにSharePointを使用することをお勧めします。



さらに、負荷の軽いソリューションには経済的なオファーがあります。MSSQL ExpressとMicrosoft SharePoint Foundation-これらを使用すると、無料でWebアプリケーションを構築できます。



生産ラインを作成する練習





SharePointでプロジェクトを行うことに決めた後、いくつかのタスクに直面しました。

  • SharePoint開発者を検索する
  • Windows管理者検索
  • テストゾーンと開発環境の構成


それぞれをより詳細に検討します。



SharePoint開発者を見つけるのは難しいビジネスです。 ロシアの技術はまだ異国的であり、SharePoint 2010を知​​っている人はほとんどいないので、混合チームがあります。 基本的には、経験豊富な.NET開発者で構成されており、SharePoint 2010アプリケーションの設計と開発に関するコースを積極的に取り入れています 。 SharePoint 2007の経験がある人もいます。「トップ10」の作業に参加するのは簡単でした。



その結果、人を雇用するという観点から言えば、「SharePoint開発者」を厳密に探す必要はありません。 オンラインとオフラインの両方で多くのトレーニングが行われているため、空席を「.NET開発者」に拡大し、SharePointの繊細さで新しい従業員をトレーニングするだけでも十分です。



SharePoint管理者と同じ状況について-SharePoint管理者が見つかりませんでした-最終的に、特別なコースのトレーニングを受けた優秀なWindowsスペシャリストを採用しました。



SharePoint用のテスト環境と開発用の機器を購入しました。16GbRAMと1Tbハードドライブを搭載したコンピューターは開発者向けに開発されました。 このような容量は、配信されるソフトウェアの量の点でサーバーと同様に、ローカルホスト上で仮想マシンを上げるために必要でした。



開発プロセスを編成するには、別のサーバーが必要でした。





プロセスを整理するために、サーバーに展開されたいくつかの仮想マシンがあります。機能を実証するSharePointを備えたサーバー、継続的統合サーバー（後ほど）、個別のSQLサーバー、およびTeam Foundation Server（TFS）。 それとは別に、TFSについて言いたいと思います。これは、バージョン管理、バグトラッカー、タイムトラッカー、およびアセンブリサーバーを組み合わせた非常に便利なものです。 さらに、トラッカーにはアジャイルアプローチのプリセットがあります。プロジェクトを反復とユーザーストーリーに分割し、評価タスクをポイント単位で設定します。



必要に応じて、この無料のTeamCityに追加して、 継続的な統合サーバーを含むTrue Aglieプロセスを取得できます。 友達を作る方法のビデオキャストはすべてSharePointで、サイトには21のアプリがあります。



開発の見通し



興味深い事実は、大手ソフトウェアベンダーがWebの方向性を積極的に開発していることです。 これは大きなプラスです。Web開発者は、「大きな開発」から、プロジェクトの設計と開発、コラボレーションの整理、アプリケーションの公開のための多くの蓄積された知識と便利なツールを継承するからです。



SharePointテクノロジー自体は最新のものであり、開発されています-Webアプリケーションを迅速に作成でき、アプリケーション自体も迅速に機能します。 さらに、ロシアの多くの大規模な組織はすでにMicrosoftから多くのサーバーソリューションを展開しているため、SharePointは需要が高いと約束しています。



ロシアのインターネット上で、初心者の開発者がSharePointをできる限り利用しやすくする実用的な資料が徐々に登場することを願っています。
