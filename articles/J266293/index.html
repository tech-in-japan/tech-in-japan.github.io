Linuxおよび無料のクラウドストレージ

LinuxでMS Onedriveをマウントしようとすると、考えが浮かびました。一般的にどのクラウドがフォルダーにマウントされ、同時に最大ボリュームを提供しますか？ 以下は、サービスの短いリストと、それらをマウントする試みの印象です。

私はすぐに言わなければならない-これはすべて好奇心を満たすためだけに行われ、その後の使用のためではありません-したがって、提案されたソリューションが長期的にどのように動作するかを言うことはできません。 しかし、おそらく誰かが役に立つでしょう。

クラウドは2つのパラメーターに従って選択されました-最大空きボリューム（できれば、紹介リンクによる登録を何十もせずに取得）と、Linuxの下のフォルダーにマウントする機能です。



テストプロセスは

1）パスワードを登録または記憶する（このサービスに既に登録されている場合）

2）紹介リンクを介して空きスペースを追加する試み-プロセスがどのように見えるか、自分で場所を「巻き上げる」ことがどれだけ簡単かを確認します。

3）クラウドをLinuxの下のフォルダーに接続する試み（WebDAVが機能する-davfs2を使用し、機能しない場合-他のオプションが利用可能）。

4）試行が成功した場合、異なるファイルをアップロードしようとするいくつかの試行は、メガバイト、50〜70メガバイト、および1.6ギガバイトの些細なことです。

実験は、1つのコアとギガバイトのRAM（debian安定版）を備えたhetznerのVPSで実行されました。



私は見ていませんでした：価格、顧客の利便性、チームワーク、組み込みのエディターなど これは、私がやりたくないサービスを完全に比較するためのものです。 プレートの最後の文字を読むのが面倒。



メディア火災





MediaFire-登録時に10ギガバイトを提供しますが、異なる「クエスト」で最大50ギガバイトまで増やすことができます。





何らかの理由で、私はすぐに50ギガバイトになりました（長い間登録されていました）。そのため、TwitterとFacebookでさらに5つの投稿とデスクトップクライアントのインストールを受け取りました。 その後、株のリストの表示を停止し、「それで十分です」と言いました。 ギガバイトあたりの登録ごとに、紹介リンクによる登録用に、8ギガバイトを独立して取得できます。 余分なものを追加する必要はありません。したがって、怠け者でなければ、1台のコンピューターからすべてを実行できます。最初に32通のメールを受け取るだけです。 :)



LinuxおよびOpen-Source Toolkit for Developersを使用してLinuxでの作業を行います。 FUSE用のftpのようなユーティリティとモジュールが含まれています。 リンクのインストール手順は最新のものであり、自分で何も考える必要はありませんでした。 1.6ギガバイトのファイルがアップロードされ、非常に安定して動作します。







2つの注目すべき欠点があります。

1）ダウンロード速度が遅く、1時間あたり約800メガバイト（より正確には、上の図で自分で確認できます）。

2）何かがロードされると、ユーザーは、たとえばdfを使用する機能を切断されます。 ダウンロードの最後までハングします。 小さなファイルでは重要ではありませんが、大きなファイルでは...



ただし、一般に、使用可能なボリュームと機能の比率は悪くありません。 使用することはかなり可能だと思います。



hubic.com





hubic.com-登録時に25ギガバイトを提供します。紹介で来て、25ギガバイトの紹介をダイヤルすると、さらに5ギガバイトを取得できます（登録ごとに5ギガバイト）-合計55ギガバイト。 登録する際、クライアントを設定する必要はありません。そのため、アカウントは数分で最大化でき、5つの予備ボックスがあります。 WebDAVが欠落しており、 Linuxのネイティブクライアント （同期）、これまでのところベータ版があります。 フォルダーにマウントするには-hubicfuse 。 指示に従ってインストールおよび構成されますが、落とし穴はありません。

使用に問題はありません（ドキュメントに記載されているものを除きます-フォルダの名前を変更したり、ファイルをルートにコピーしたりすることはできません）。 ダウンロード速度は良好です。



使用に適しています。



mega.co.nz





mega.co.nz-すぐに50ギガバイトを増やします-お金だけで。 無料のユーザーの場合、30分あたり10ギガバイトのトラフィックの制限があります。 つまり、10ギガバイトを10分間でそこに置くと、30分間速度が低下します。 WebDAVではなく、Linuxを含む同期用のクライアントがあります 。

または、コマンドライン用の一連のツール （ github ）を使用できます-アカウントの作成、ファイルのダウンロードとダウンロード、およびリポジトリーをフォルダーにマウントするためのプログラム。 debian不安定版にはdebパッケージがありますが、私はstableを使用しているので、ソースからコンパイルする必要があり、プロセスに落とし穴はありませんでした。 しかし、インストール後に表示されました-判明したように、ここでのファイルのコピーはまだ実装されていません。 フォルダーの作成とファイルの削除のみが可能です。 さらに掘り下げなければならなかったからです。

MegaFuseを掘り起こしました 。 アセンブリではそれほど簡単ではありませんでした（2、3のライブラリの呼び出し方法が異なり、スワップファイルを追加する必要がありました-RAMは終了しました）が、最終的にすべてがコンパイル、開始、動作しました。

機能：

1）MegaFuseはデーモンとして動作します。つまり、MegaFuseは画面にプッシュするか、サービスを作成する必要があります。

2）/ tmp経由でファイルを追跡します-最初にそこにコピーし、次にサーバーにアップロードします-空き容量に注意してください。

600-1000キロバイト/秒の領域のダウンロード速度、1.6GBのブート、プロセッサとRAM MegaFuseはそれほど必要ありません。



使用に適しています（設定がやや混乱しています）。 しかし、不必要なジェスチャーや適切な速度のない十分に大きなボリュームは、この混乱を完全に引き換えます。



cubby.com





cubby.com-最初は5ギガバイトを提供しますが、最大25の紹介を上げることができます（1ギガバイト、クライアントのインストールが必要です）。 WebDAVがあり、ファイルサイズの制限は明記されておらず、通常1.6GBがロードされます。 確かに、ルートにロードすると、ファイルは明確でない場所に到達します。 それらをフォルダー（ローカル用語ではキュービー）に入れる必要があります。

一般に、最大25ギガバイトのねじれを解消するのに適したサービスです。 logmeinの評判が脅かされる可能性がない限り（前日に警告していた、無料のリモートコントロールサービスの提供を一度停止しました）。 しかし、これはクラウドストレージにとって恐ろしいことではないと思います。ファイルのコピーだけを奇妙なクラウドに保存することはないでしょうか。 :)

使用に適しています。
