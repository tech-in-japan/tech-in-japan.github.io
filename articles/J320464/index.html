OpenJDK 10の開発が始まります

本日、 Mark ReinholdがJava 10の水銀リポジトリのオープンを発表し、Java 9プロジェクトのコミッター以上のステータスを持つ開発者は、Java 9で行われるはずのないバグ修正やマイナーな改善を投稿できました。 OpenJDK国勢調査のページ。







衝突を避けるために、大きな変更はまだ推奨されていません。 Java 9の変更はまだ完了しておらず、しばらく時間がJava 10に移行する必要があります。競合がない場合、手順は自動化されるため、Java 9で引き続き修正を行う人は再度心配する必要はありません。







もちろん、Java 9に含める予定のない機能の開発は以前に行われました。 ValhallaやPanamaのような深刻なプロジェクトは、Javaバージョンを参照せずに別々のフォークで開発されています。 準備ができたら、Jigsawの場合のように、ブランチをマージするための多くの作業があります。 ローカル変数のタイプの派生のプロトタイピングなどの個別の実験は、公式ツリーの外部で行われます。 しかし、今はトップ10で提供（および実装）する絶好の機会です。Javaで本当に見逃しているいくつかの重要な機能です。 ご存知のように、リリースに近づくと、新しいオファーは抵抗の増大に直面します。







Java 10テーマ別メーリングリストが作成されていない限り、一般的なjdk10-devリストのみがあるため、購読することですべてのニュースをフォローできます。
