コンピューティングシステムの負荷は増え続けているため、企業はデータセンターを設計する新しい方法を模索しています。 この記事では、データセンターの最も珍しいソリューションの概要を説明しました。

地下データセンター



最新のデータセンターは、機密のユーザー情報を処理する何千ものサーバーで構成されています。 データのセキュリティを確保するために、企業は、データセンターを地下に建設するなど、予期しない措置を講じています。 このソリューションの利点は、展開速度が速いことです。データセンターの建物を構築する必要がなく、どんな天候でも設置作業を実行できます。



1.5平方キロメートルのエリアは、60日で作業の準備ができています。 さらに、地下施設は自然災害に対して非常に抵抗力があり、これは竜巻の危険な地域で非常に有用な利点であることに注意する価値があります。



地下のデータセンターの例は、ピッツバーグ近くの145エーカーの面積を持つ廃坑の67メートルの深さに位置するアイアンマウンテンの発案によるものです。 地下はほとんど涼しいです-アイアンマウンテンでは、鉱山の温度が摂氏14度以下に下がり、冷却が節約されます。 1 kWのサーバー電力ごとに0.56 kWの冷却が行われますが、従来のデータセンターでは、1：1の比率が慣習的と考えられています。



廃坑は都市から遠く離れているため、アイアンマウンテンは電気を節約することができました。 現在、同社は1 kW / hで5.5セントを支払っているのに対し、大都市のデータセンターは同じ量の電気で10から17セントを支払っています。 データセンターが高電圧で電気を購入し 、独立して変換を実行するという事実も節約に役立ちます。 従来のデータセンターが480 Vを使用する場合、Iron Mountainは4160 Vを使用します。



同様の地下データセンターがノルウェーのレンネソイ島にあります。 グリーンマウンテンによって作成され、21,000平方メートルをカバーし、100メートルの深さになります。 以前は、この施設はNATO軍事機器が保管される倉庫として使用されていました。



この傾向は、クラウドコンピューティングとコロケーションに従事しているLightEdge Solutions社によってサポートされていました。 2014年、彼女は18平方キロメートルの面積を持つ世界最大の地下複合施設の建設に5800万ドルを投資しました。



Wiredによると 、もう1つの印象的なサーバーファームはスイスアルプスの地下テクノロジーセンターにあります。 スイスフォートノックスは、冷戦時代のバンカーにある2つの独立したデータセンターです。 両方のセンターは、複製された光ファイバー通信チャネルによって相互接続されています。



ネットワークおよびITの多段保護システムは、障害のリスクを最小限に抑え、気候制御およびエネルギー供給システムは外界への依存を減らします。センターは何週間も自律的に作業できます。



このデータセンターの特徴は「難攻不落」です。厚い岩の層と3トンのドアが技術的な災害から保護し、軍事基準で運用されるセキュリティシステムとセキュリティサービスがテロの脅威から保護します。



上記のデータセンターが母なる自然の助けを借りていることは注目に値します。 コンピューティングモジュールを冷却するために、地下水から水が使用され、グリーンマウンテンはフィヨルドから摂氏8度の温度で水を受け取ります。



作業機器は高温に加熱され、個々のラックのエネルギー消費量は30〜35 kWに達するため、熱の除去は最も重要な問題の1つです。



DCD Intelligenceは、世界中のデータセンターで発生した熱を収集すれば、英国を暖めるのに十分であると報告しています。 ロシアの会社Yanedeksは、そのようなエネルギーを無駄に使いたくないので、発生した熱の用途を見つけることにしました。 少し前に、フィンランドのメンツァラ市に予熱ステーションが開設され、同社のデータセンターの熱を利用しました。



現在、Mäntsäläでは、Yandexサーバーの4つのキューの1つが稼働しており、各キューの容量は10 MWです。 サーバーによって加熱された空気は、ファンによって予熱ステーションに追い込まれ、水を30度から60度まで加熱します。



ロシアに関しては、暖房システムでのエネルギー損失が大きいため、そのような技術を導入することは問題ですが、そのようなプロジェクトはまだあります。 イルクーツク地域では、いくつかの企業（En + Group、Huawei、CDS、Lanit）および地方自治体がデータセンターの建設を計画しており、サーバーの熱は地元の火力発電所に送られます。

水中（および水中）のデータセンター



データセンターの設計者は、本来私たちに与えられた機会を最大限に活用しようとしています。 2008年、Google はフローティングデータセンターの特許を取得しました。 このプロジェクトでは、海岸から5 kmの距離にデータセンターを配置します。 このアプローチにより、別の建物が不要になり、固定資産税が免除されます。



ただし、Googleはこのアイデアを実現するために急いでいません。 代わりに、 スタートアップ Nautilus Data Technologiesがこの方向への第一歩を踏み出しました。



会社の共同設立者であるアーノルド・マグカレとダニエル・ケカイは、機器配置サービスを提供するためのフローティングプラットフォームのネットワークを作成しています。 彼らによると、この設計はデータセンターを自然災害から保護し、モビリティを追加します-必要に応じて、プラットフォームを場所から場所へ移動できます。



従来のデータセンターは、冷却に大量の水を使用します。たとえば、ユタ州の米国国家安全保障局のデータセンターは、1日あたり150万ガロン（または560万リットル）の水を消費します。 Nautilusの専門家は概念を変更することを決定し、データセンターを水に持ち込みました。これは、はしけの下から直接水を取り出して貯水池に戻す独自の冷却システムを開発したためです。



現在、マーアイランド海軍造船所に最初のノーチラス商業データセンターが建設されています。 プロジェクトの創設者は、軍事基地で働くことで技術を維持できると確信しています。 その後のデータセンターも軍用造船所に建設される予定です。



プロジェクトの主な利点は、迅速な展開です。 フローティングデータセンターは800台のサーバーラックを収容し、6か月で作業に備えることができます（水域がある世界のどこでも）。



ダニエルによると、マル島造船所のデータセンターは単なるプロトタイプです。 現在は電気が主電源として使用されていますが、経営陣は燃料電池を備えた次世代のバージを作成することを計画しており、これによりデータセンターが自律的になります。 このテクノロジーを使用して、従来のタイプのデータセンターを好む保守的な顧客に関心を持たせる方法を理解することは依然として必要です。



Nautilus Data Technologiesが水上にデータセンターを構築している場合、Microsoftはこのトピックに頭を突っ込むことにしました。 会社はNatickと呼ばれるプロジェクトに取り組んでおり、すでに数百メートルの水圧に耐えることができるプロトタイプカプセルを開発しています。



「これについて最初に聞いたとき、私は考えました。「水、電気、なぜこれを行うのですか？」Natikプロジェクトの開発に携わっているMicrosoftデザイナーのBen Cutler氏は言います。 「しかし、考えれば考えるほど、ポイントがはっきりと見えます。」



マイクロソフトチームは、このようなテクノロジの大規模な導入により、新しいデータセンターの展開時間が2年から90日に短縮されると確信しています。



物理学者およびデータ処理のスペシャリストであるラリースマーは、このアプローチには未来があると確信しています。 彼は、環境の恩恵を受けることを期待して、クラウドベンダーが長年にわたってデータセンターに適した場所を探してきたと指摘しています。



データセンターを水中に置くと、サーバーを冷却する問題が解決します。 さらに、このようなカプセルは水塊の隣にある都市の近くに配置できます。これにより、Webサービスの作業が高速化され、遅延が減少し、インターネットユーザーに多くの不便を与えます。



Natikプロジェクトの最初の実験船は、Haloゲームの人気ゲームキャラクターに敬意を表して「Leon Filpot」と名付けられました。 データセンターのテストは、2015年8月から11月にかけて、海岸から1 km離れた太平洋で行われました。



システムは、機器の故障の考えられる原因をよりよく理解するために、圧力、湿度、動き、および漏れのさまざまなセンサーを備えた容量に「詰め込まれ」ました。 しかし、テスト期間全体で、単一の障害は発生しなかったため、チームは実験を延長し、さらなるテスト中にMicrosoft Azureクラウド環境でデータ処理を行う商用プロジェクトを展開することさえできました。



この実験中、マイクロソフトの専門家は、電子システムの状態だけでなく、周囲の水中環境の状態も観察しました。 音響センサーの助けを借りて、カプセルのタービンから発せられる騒音と、水中世界の住民への影響の程度が推定されました。 装置の音が、魚が通り過ぎることによって生じるわずかなノイズを消すことが判明しました。



実験の第1段階を無事に完了した研究チームは、次のバージョンのトリプルサイズカプセル（レオンフィルポットの直径は2.5メートル）の開発を計画しています。 この段階で、この部門は、代替エネルギーの専門家グループが設計に参加することを計画しています。



同社は、水中電流を使用して発電する予定です。 これにより、海水への熱の放出が減少します。 現時点では、科学者はデバイスから数インチの距離でカプセルの周囲に温度変化がないように管理しています。



この大胆なプロジェクトの技術的実装の可能性、さらには最初の施設を稼働させるタイミングについて、積極的な結論を出すのは時期尚早です。 「現時点では、Natikプロジェクトは研究段階にあり、この概念がMicrosoftや他のクラウドサービスプロバイダーによって適用されるかどうかは完全には明確ではありません。



マイクロソフトは、新しい水中データセンターの開発に加えて、中国、韓国、台湾、日本、米国西海岸間の太平洋を横断する水中ケーブルの敷設に取り組んでいます。 このネットワークと呼ばれる新しいクロスパシフィックは、データ転送速度を向上させます。



別の大西洋横断ケーブル-Hibernia Express-は2015年9月に就役し、カナダ、アイルランド、英国間で敷設されました。 これまでのところ、MicrosoftはAEConnectケーブルの敷設に3億ドルを費やすことを決定した唯一の会社です。



5,400キロメートルを超える将来のネットワークは、100 Gb / sの速度でデータを転送するように設計されているため、Microsoftはこの大西洋横断ネットワークを使用して、ブロードバンドとクラウドサービスのサポートに対する高まる需要に対応します。



MicrosoftのGlobal Network OperatorsのマネージングディレクターであるDave Crowley氏は、世界の230の海底ケーブルのうち、100 Gbit / sのコヒーレント伝送技術をサポートできるものはほとんどないと述べました。

未来のデータセンター



大西洋を横断する水中/水面のデータセンターとケーブルを見ると、多くのサイバーパンク小説で説明されている未来がすでに来ているようです。 これのもう1つの確認は、マルコメルレッティ（マルコメルレッティ）とヴァレリアマーキュリー（バレリアメルクリ）によって作成されたデータタワーの概念です 。



開発者にとって、彼らのプロジェクトは、環境に優しいデータセンターがどのように見えるべきかという質問に対する答えです。 主なタスクは、自然条件を有利に利用してメンテナンスのコストを簡素化および削減する方法を探ることです。



タワーの未来的な外観にもかかわらず、最新の技術を使用して作成できます。 建築家によると、このプロジェクトは巨大な3次元の円筒形のマザーボードです。 すべての要素は外側から取り付けられ、内側は空のままで、冷却システムの空気ダクトを形成します。 ファンの助けを借りて、自然の通風の効果を作り出し、外部から新鮮な冷気を供給し、中央の空所から暖気を除去することが計画されています。



タワーの冷却も、計画された地理的位置によって促進されます。 アイスランドの気象条件は、データセンターのホスティングに理想的であり、水力発電所が発電する電気が安いため、メンテナンスのコストは低くなります。



データタワーは 、建物の骨組みを形成する24本の柱で構成され 、その上にコンピューティング機器を備えたブロックが取り付けられています。 壁から機器が突き出ているブロックには、4〜8個の標準サーバーラックが搭載されているため、スケーリングの可能性が広がります。 MercuryとMerlettiの推定によると、データタワーは最大400,000台のサーバーに対応できます。



結論として、急速に発展しているIT産業を前進させるのは、このような野心的なプロジェクトであることに注意したいと思います。 おそらく近い将来、世界のすべての洞窟に機械室が配置され、サーバーポールにサーバーのある塔が成長し、「ディープインターネット」という表現が文字通りの意味を帯びてくるでしょう。
