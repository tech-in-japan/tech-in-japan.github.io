彼らはポーランドを忘れていた！

Windows 95の最初のリリースでは、世界地図上のポイントをクリックしてタイムゾーンを選択できました。 選択したタイムゾーンが明るい色で強調表示されました。







同様に、Windows 95のベータ版では、ロケール（「言語と標準」）を選択できます。







どちらのカードも数か月しか生き残りませんでした。ロケールマップはリリースまで対応していませんでした。 タイムゾーンマップ-最初のサービスパックまで。 国境は公式に認められた国連に従って展示されましたが、その場所に不満はもうすぐありませんでした。



1995年2月、 アルトセネプ戦争がペルーとエクアドルの国境で勃発しました。 ペルー政府は、MicrosoftにWindows 95の「誤って定義された」境界線に関する正式な苦情を提出しました。当然、Microsoftがマップ上のいくつかのピクセルを塗り直すという要件に従った場合、エクアドルからまったく同じ抗議が行われます。 彼らは、リリースから地域の地図を削除する以外に選択肢がありませんでした。 幸い、2番目のマップでは、ペルーとエクアドルは同じタイムゾーンにあったため、「論争の的となるピクセル」はありませんでした。



インド人はタイムゾーンマップに不満でした。紛争地域はパキスタンとともにGMT + 5タイムゾーンに割り当てられ、GMT + 5:30はインド全土で運用されています。 （「敵対的な」パキスタンのベルトは、上のスクリーンショットで強調されているものと同じです。）インド政府は深刻な警戒心を抱き、「反国家」マイクロソフト製品の輸入を完全に禁止すると脅しました。 今回、Windows 95の国際リリース後に「論争の的となるピクセル」が発見されたため、マップはローカライズされたインド版でのみ修正されました。 ただし、Microsoftは各国ごとに個別の世界地図を維持するつもりはありませんでした。 また、最初の機会に、タイムゾーンの色付けは、Windowsのすべてのローカライズから、害の方法から削除されました。



ご存知のように、Microsoftはデバッグおよびテストされたコードをほとんど変更しません。 そのため、今回は、マップの色が実際に消えることはありませんでした。選択したタイムゾーンだけが、他の土地と同じ緑の陰影で描かれました。 最初は、背景全体に青い「海」があふれ、次に緑の時間帯が次々に描かれ、大陸と島が形成されました。 現在、Windows 98では、すべての土地が突然描かれ始めたわけではありません。ポーランドは理由もなく「海にby濫」しました。







平和で穏やかなポーランド人は、ピクセルが欠落しているため、国際的なスキャンダルを開始しませんでした。 「洪水」ポーランドの地図は、Longhornのベータ版まで生き残りました。







Vistaリリースでは、カードが完全に削除され、Windowsが地政学的なトラブルから完全に救われました。 しかし、問題は何ですか？ 洪水はどこから来たのですか？



タイムゾーン設定はレジストリに保存され、必要に応じてローカライザーによって更新されます。 しかし、タイムゾーンを描画するコードでは、合計数が厳密に設定されています。 プログラマーは、地球上のタイムゾーンの数が変わる可能性があることを予測していませんでした。 ただし、ポーランドは、Windows 95/98のさまざまなエディションのリリース中にのみ、4つの異なるゾーンで「訪問」することができました。
  • Windows 95：（GMT + 01：00）ワルシャワ
  • Windows 95：（GMT + 01：00）リスボン、ワルシャワ
  • Windows 98：（GMT + 01：00）ブラチスラバ、ブダペスト、リュブリャナ、プラハ、ワルシャワ
  • Windows 98：（GMT + 01：00）サラエボ、スコピエ、ソフィア、ワルシャワ、ザグレブ
実際、ポーランドの時間は変わりませんでした。タイムゾーンの隣人は変わりました。 多分、ポーランド人は冬時間への移行の日付を決定できなかったでしょう。 しかし、最終的に、ワルシャワタイムゾーンは消え、近隣の1つと合併しました。 これは、対応する土地が地図から消えたことを意味します。



後知恵では、彼らは、このカードはWindows XP / Windows Server 2003の修正プログラムの1つで修正されたと言います。 しかし、少なくともSP2ではポーランドはまだ浸水しています。 ポーランドのタイムゾーンは、「（GMT + 01：00）サラエボ、スコピエ、ワルシャワ、ザグレブ」と呼ばれています。
