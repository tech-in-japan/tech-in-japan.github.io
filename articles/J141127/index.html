MySQLiはすべてを棚に置きます



この記事の対象者 この記事の主な目的は、mysqliを既に使用したが、深く掘り下げたわけではないが、すぐにラッパーを書いて元の構文を忘れた人のために「すべてを棚に置く」ことでした。 正規化を知らない人によって設計された大規模で非常に古いデータベースから、構造が大幅に変更された新しいデータベースにデータを転送するときに遭遇するニュアンスを明確にしようとしました。



古いmysql拡張機能をまだ使用していて、PDOまたはMySqliに切り替えることのみを考えている人にこの記事を読むことは可能ですか？ それも必要だと思います。





MySqliまたはPDO



近年、私はフレームワーク専用のWebサイトを作成しました。これにより、データベースを直接操作する必要がなくなりました。 しばらく前、私は純粋なphpでサイトの作業を開始し、古いMySQL PHP拡張機能の使用は推奨されていません。



MySqliとPDOを選択する必要がありました。 それほど長くない研究の後、MySqliに立ち寄ることに決めました。それは、当時私には思われたように、MySQLを放棄して他の何かを支持する方法がなかったことを除いて、PDOとまったく同じだったからです。 以下に書いているように、これは完全に真実ではありませんが、少なくとも1つの顕著な違いがあります。



MySqliは、PHP開発者自身が使用することをお勧めします[ 1 ]



OOPと手続き型インターフェース



MySqliを使用すると、OOPスタイルと手続きスタイルの両方でコードを記述できます。 OOPはほとんどのhabrコミュニティに関して私に近いので、この記事で使用します。



3つの主要なクラス



MySqliには3つの主要なクラスがあり、これらについてはこの記事で詳しく説明します。

  1. mysqli-データベースへの接続を確立するために必要で、古いMySQL拡張で行ったのと同じ方法でクエリを実行する場合に役立ちます。
  2. mysqli_stmt-新しいMySqli機能を使用するために必要です。準備された式に対してクエリを実行します。
  3. mysqli_result-関数を組み合わせて、mysqliまたはmysqli_stmtを使用して作成されたクエリの結果を取得します。


以下で、それぞれについて詳しく見ていきましょう。

$ mysqli-> connect_errnoおよび$ mysqli-> connect_errorを使用して、接続中に発生した説明とエラーコードを取得します。 新しいmysqli（）と$ mysqli-> real_connect（）の両方が、エラーの接続時にPHP警告エラーを引き起こします。 したがって、上記の関数を使用したエラー出力は、実稼働サーバーなどでPHPエラー表示を無効にしている場合、または何らかの方法でこのデータを処理する必要がある場合に意味があります。 エラーの場合にすべてのMySQLi関数がPHP警告を呼び出すわけではなく、エラーが発生したことを確認するために、特別な関数を参照する必要があるため、ここでこれについて言及しました。



接続中に取得したオブジェクトは、将来使用するために$ mysqli変数に割り当てられました。 これはOOPスタイルでは明らかですが、手続き型スタイルではこのオブジェクトも必要です。これは、mysql関数を使用するたびに接続参照を渡す必要がなかったレガシーMySQL拡張とは対照的です。



バッファ付きおよびバッファなしの結果



さらに説明する前に、これら2つのタイプの結果の違いについて説明したいと思います。



バッファなしの結果を考慮してください。 この場合、mysqlサーバーが結果を完全に受信するまで待たずに結果の読み取りを開始できます。



利点：

  • 結果をより早く読み始めることができ、待ち時間が短縮されます。
  • 結果は、RAMのスペースを占有しません。


短所：

  • 受信した行数を知ることはできません。
  • 特定の結果に移動することはできません。つまり、最初から順番にのみデータを読み取ることができます。
  • この結果がクローズされるまで、他のリクエストを実行することはできません。




バッファリングされた結果には、これらの欠点がないため、リストされている利点がありません。



クラシッククエリ



MySqliは、「古典的な」クエリの可能性を残しました。ユーザーが、送信されたクエリのセキュリティに独自に関与するよう招待された場合、レガシーMySQL拡張の場合と同様です。 これを行うには、関数$ mysqli-> real_escape_string（）を使用することをお勧めします。この関数では、クエリに配置する前にすべてのデータを処理する必要があります。



接続の場合と同様に、このようなリクエストを短くする方法と長くする方法の2つがあります。



短い。

$ result = $ mysqli- > query （ 'query text' 、 MYSQLI_USE_RESULT ） ;


可能な定数：

MYSQLI_STORE_RESULT-バッファリングされた結果、デフォルト値を返します

MYSQLI_USE_RESULT-バッファなし



長い。

$ mysqli- > real_query （ 'query text' ） ;

echo （ $ mysqli- > field_count ） ; //結果として列の数を返します

//結果を受け取る前に取得できるため、柔軟性が向上します

//短いメソッドと比較して、もちろん、呼び出す必要はありません

$ result = $ mysqli- > use_result （ ） ; //バッファなしの結果を返します

//または

$ result = $ mysqli- > store_result （ ） ; //バッファされた結果を返します




関数$ mysqli-> use_result（）または$ mysqli-> store_result（）は、複数クエリ（複数のクエリで構成されるクエリ）にも使用されます。 この記事の複数のリクエストは考慮されません。



これらの構文は両方ともmysqli_resultオブジェクトの形式で結果を返します。これは、バッファ付きおよびバッファなしの両方で結果を操作するための便利なインターフェイスです。



上記で書いたように、すべてのMySQLi関数がPHPエラーをスローするとは限りません。 リクエストが正しくなく、サーバーがエラーを返した場合、PHPはユーザーに通知しません。 確認するには、次の機能を使用します。

  • $ mysqli->エラー - エラーの説明
  • $ mysqli-> errno-エラーコード


$ city = $ mysqli- > real_escape_string （ $ city ） ;

$ mysqli- > query （ "SELECT * FROM` city` WHERE` id` = ' $ city ' " ） ;

if （ $ mysqli- > errno ） {

die （ 'Select Error（' 。 $ mysqli- > errno 。 '）' 。 $ mysqli- > error ） ;

}




「クラシック」クエリ構文の利点：

  1. 準備された式（ mysqli_stmtクラス）よりもはるかにコンパクトです。
  2. これにより、バッファリングされていない結果をmysqli_resultの形式で取得できます。 準備された式を使用すると、バッファリングされたmysqli_resultの結果のみを取得できます。また、バッファリングされていないmysqli_stmtツールを使用する必要があります。これはmysqli_resultよりも柔軟性が劣ります。




私が見る「古典的な」クエリ構文の実用的なアプリケーション：

  1. バッファリングされていないクエリで、 mysqli_resultでの作業を利用したい場合。
  2. INSERT、UPDATE、REPLACE、またはDELETEクエリで、より短い構文を使用する場合。


このようなクエリでは、次のプロパティが役立ちます。

  • $ mysqli-> Influence_rows-前の非選択クエリの影響を受けた行の数
  • $ mysqli-> insert_idは、最後の挿入要求に対して自動的に生成されたIDです。


準備されたリクエスト



「クラシック」に対する準備済みクエリの利点：

  • データのみが変更される一連のリクエストでは、リクエストは一度だけサーバーに送信され、将来的には変更されるデータのみが送信されます。
  • SQLインジェクションに対する保護。

mysqli_resultクラスとそれを使用した結果の操作



上記のように、 mysqliクラスを使用する「クラシック」クエリを使用してmysqli_resultオブジェクトを取得できます。その後、バッファリングまたはバッファリング解除するか、 mysqli_stmtクラスを使用してバッファリングします。 このクラスの関数の機能は、取得する結果の種類によって異なります。したがって、クエリがバッファリングされていない場合、結果全体が得られないため、結果が何行であるかを知ることはできません。



このクラスの関数セットは、古い拡張機能ではおなじみのようです。

  • $ result-> fetch_row（） -結果の現在の行を番号付き配列として取得します。
  • $ result-> fetch_assoc（） -連想配列として、
  • $ result-> fetch_array（） -配列タイプは定数によって設定されます。

    1. MYSQLI_ASSOC-連想、
    2. MYSQLI_NUM-番号付き、
    3. MYSQLI_BOTH-両方、


  • $ result-> fetch_object（） -オブジェクトとしての結果文字列。


$ result-> fetch_object（）について個別に話をしたいと思います。 この関数には2つのパラメーターがあり、両方ともオプションです。

  • class_name-オブジェクトの作成に基づいたクラスの名前、
  • params-オブジェクトの作成時にコンストラクターに渡されるパラメーターの配列。


ご覧のように、コンストラクターをクラスに渡すと、この関数を使用した結果の1行の連想配列が失敗するとします。 彼女自身が、フィールドの名前に一致するクラスのプロパティを結果に割り当てます。 クラスプロパティが見つからない場合、パブリックのスコープで動的に作成されます。
