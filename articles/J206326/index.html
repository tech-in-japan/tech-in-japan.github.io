システムプロパティにフィールドを追加する方法

ご存じのように、レジストリ、つまり既存のものを変更したり、新しいパーティションとパラメーターを追加したりすることで、Windowsオペレーティングシステムで多くのことができます。 コントロールパネルで使用可能なオプションを変更したり、追加のシステムコンポーネントをカスタマイズしたり、タスクバーとデスクトップをカスタマイズしたり、既存のものを変更したり、さまざまなコンテキストメニューに新しいアイテムを追加したりできます。 レジストリのどのパラメーターを変更または追加するかを知っている場合、システムを一意に構成できるだけでなく、このプロセスを完全に自動化できるため、間違いなく非常に便利です。

いくつかの可変パラメーターは、同じプロセスモニターまたはRegMonユーティリティを使用して非常に単純にローカライズできます（記事「 レジストリの監視の例 」でこれらのユーティリティについて説明しました）。また、いくつかのパラメーターを使用して、簡単に言うと、ローカライズは一見思われるよりもはるかに難しい場合があります。 目的のパラメーターの値を使用すると、すべてが非常に単純な場合があります（たとえば、DWORDパラメーターの場合、値を決定する際に問題が発生しません）。また、固有値を生成するタスクは非常に困難な場合があります（たとえばWindows 8.1オペレーティングシステムのロック画面の写真を撮るディレクトリの場所を担当するパラメーターの値です）。 パラメーター自体とその値の両方の例は無限にあります。

今日、この短い記事では、システムプロパティウィンドウに追加のユーザー情報を追加できるレジストリ設定について学習します。 さらに、それをより面白くするために、パラメーターは中央に追加されます。 グループポリシーの基本設定の機能を使用します。 始めることができると思います。



システムプロパティの追加フィールドを担当するレジストリパラメータのローカライズ



まず、[ システムのプロパティ]コントロールパネルのコンポーネントに表示される情報の名前を覚えておく必要があります。 Microsoftの公式ソースからの情報によると、「機器メーカーはコンピューター技術またはOEMのサプライヤーとも呼ばれています」。 したがって、システムプロパティに表示される情報はOEM情報と呼ばれると結論付けることができます。 最初に、TechNetまたはMSDNからダウンロードした適切なディスクまたはイメージからインストールされたクリーンなオペレーティングシステムでは、システムの元のイメージを使用する時間がなかったため、そのような情報は単純に欠落しています。

システムレジストリにパラメーターが最初に存在しない場合、監視ツールを使用してパラメーターを見つけることは困難な場合があります。この例のように、さらに悪いことに、まったく不可能です。 このため、必要なパラメーターをローカライズするには、オペレーティングシステムの独自の一意のイメージを作成できる「 Windows Installation Manager 」（ System Image Manager、SIM ）を利用する必要があります。

Windows Installation Managerを使用すると、インストールされているWindowsのエディションの設定を変更できます。 利用可能なオプションを決定するために、Windows Installation Managerはカタログファイルを作成し、その内容を読み取ります 。 カタログファイルは、特定のWindowsイメージで使用可能なすべてのオプションのリストを含む小さなファイルです。 つまり、メインのWindowsイメージのファイル（\ Sources \ Install.wim）をWindowsインストールDVD-ROMから選択したホストコンピューターのフォルダーにコピーします。 つまり、SIMマネージャーで、Windowsイメージグループに移動し、インストールディスクから以前にコピーしたファイルを選択します。 以前にディレクトリファイルを作成していない場合、マネージャはこの段階でそのようなファイルが作成されることを警告します。 .wimファイルに複数のシステムイメージが含まれている場合、特定のイメージを選択するよう求められます。 その後、Windows Installation Managerは、Install.wimファイルで選択したイメージファイルに基づいてディレクトリファイルを作成します。 原則として、次の図に見られるように、このプロセスには少し時間がかかり、数分かかることがあります。





図 1.カタログファイルを作成し、オペレーティングシステムイメージを追加する



その後、Windowsのインストールを自動化するためのすべてのパラメーターを含む応答ファイルが作成されます。 これも難しい作業ではありません。 [ファイル ]メニュー（ [ ファイル ] > [新規回答ファイル] ）から[新規回答ファイル ]コマンドを選択するだけで、その後、必要な構成手順が応答ファイルのパネルに表示されます。 セットアップ手順は、Windowsインストールのフェーズであり、その間にイメージをカスタマイズできます。 Windows自動インストールオプションは、使用しているオプションに応じて、1つ以上の構成手順で適用できます。 この例では、関心のある構成手順はoobeSystemと呼ばれます。 通常、この手順は、Windowsシェル設定の構成、ユーザーアカウントの作成、および言語と地域の設定の指定に使用されます。

また、この段階に必要な手順を追加するには、オペレーティングシステムの特定のコンポーネントを追加する必要があります。 すべてのコンポーネントは、Windowsイメージペインにあるコンポーネントノードにあります。 最も重要なことは、追加するコンポーネントと目的を理解することです。 OEM情報に関心があるため、必要なコンポーネントをローカライズすることは難しくありません。 Microsoft-Windows-Shell-Setup> OEMInformationと呼ばれます。このようなコンポーネントを見つけたら 、残りはoobeSystemステージに追加するだけで、これは7番目のパスとも呼ばれます。 これを追加するには、そのようなコンポーネントを選択し、そのコンテキストメニューを呼び出して、「 設定をPass 7 oobeSystemに追加 」コマンドを選択する必要があります。 追加プロセスを以下に示します。





図 2. 7番目のパスにコンポーネントを追加する



その後、レジストリで必要なパラメータを見つけることは難しくありません。 応答ファイルの7番目のパスに移動して、追加したコンポーネントを見つけるだけです。 右側の追加したコンポーネントのプロパティパネルには、オペレーティングシステムで変更できるすべてのパラメーターが表示されます。 次の図でわかるように、OEM情報を変更するには、次のパラメーターを使用できます（システムレジストリ内のパラメーターは同じ名前で作成されます）。

  • ロゴ ロゴへのパスを表します。 ここでは、画像をBMP形式で保存する必要があり、そのサイズが120X120ピクセルを超えてはならないことに注意する必要があります。 さらに、このような画像はターゲットコンピューターに配置する必要があり、指定した行の長さが259文字を超えないようにする必要があることに注意してください。
  • メーカー このパラメーターを使用すると、コンピューターの製造元の名前を決定できます。 この行の長さも制限されています-256文字。
  • モデル このパラメーターは、ターゲットコンピューターのモデルです。 このパラメーターの文字数の制限は、前のパラメーターと同じです。
  • SupportHours 。 これは通常の文字列パラメーターでもあり、これを使用してサポート時間を指定できます。
  • SupportPhone 。 指定されたサポートサービスの通常の電話番号。
  • SupportURL 。 コンピューターの製造元のWebサイトへのリンクを定義できる最後のオプション。 ここで、最大行長はすでに256文字ではなく2083文字にすることができます。

図 3. Windows Installation ManagerのOEM情報設定



パラメータを既に把握しているため、レジストリ設定のローカライズに関する問題を部分的に解決しました。 ただし、そのようなパラメーターが作成される場所を正確に見つける必要があります。 セクション自体をどのように見つけることができますか？ これも複雑ではありません。 どのレジストリキーが使用されるかを確認するには、インストールイメージを作成し、テスト対象の仮想（または物理）マシンに展開する必要があります。 その後、レジストリエディターを開き、追加したコンポーネント、つまりOEMInformationに一致するレジストリキーを検索する必要があります。 これらのすべての文字列型パラメーター（REG_SZ）は、 HKLM \ SOFTWARE \ Microsoft \ Windows \ CurrentVersion \ OEMInformationセクションに保存されます。 次の図でわかるように、このセクションでは構成した設定を見つけることができ、 「システム」ウィンドウは右の図に示すようになります。





図 4.システムレジストリのパラメーターとその適用結果



以前に展開されたシステムへのOEM情報の一元的な配布



クライアントコンピューターにオペレーティングシステムを既にインストールしており、その後、以前に検討したパラメーターを変更する必要がある場合は、当然グループポリシーの機能を利用できます。 ご想像のとおり、このような情報を設定するための管理用テンプレートポリシー設定は存在しません。 このため、回避策を利用する必要があります。つまり、グループポリシーの設定を有効にする必要があります。

Windows 8.1がインストールされているコンピューターのシステムプロパティウィンドウのみを変更する必要があるとします。 この場合、このタスク全体を条件付きで3段階に分けることができます。つまり、クライアントコンピューターの同じ場所に目的のイメージをコピーし、新しいシステムレジストリパラメーターを作成し、作成したパラメーターを特定のコンピューターサークル（ Windows 8.1で動作します）。 すべてを順番に考えてみましょう。

  1. 最初に、 グループポリシー管理スナップインを開き、GPOターゲット（ OEMInfoなど）を作成してから、そのようなオブジェクトのコンテキストメニューからグループポリシー管理エディタースナップインを開く必要があります。
  2. 既に表示されているスナップインでは、これらのパラメーターはログインしたユーザーとは無関係であり、レジストリ設定はHK_LOCAL_MACHINEセクションにあるため、 [ コンピューターの構成]ノードに移動し、 [設定] \ [Windows設定] \ [ファイル]ノードに移動します»（ 環境設定\ Windows設定\ファイル ）、詳細ペインのコンテキストメニューを呼び出し、「コマンドを作成 」、次に「 ファイル 」（ 新規>ファイル ）を選択します。
  3. 新しいファイルプロパティの基本設定項目のダイアログボックスで、[ 作成]アクションを選択し、テキストボックス「 ソースファイル 」および「 最終ファイル 」（ ソースファイル（複数可）および宛先ファイル ）で共有ファイルにコピーするファイルへのパスを指定しますリソース（\\ DC \ Logo \ Biohazard.bmpなど）およびクライアントマシン上のファイルへの新しいパス（c：\ OEMLOGO \ Biohazard.bmpなど）。 作成された基本設定項目のダイアログボックスを以下に示します。





    図 5.ファイル設定項目



  4. ここで、6つのレジストリ設定すべてを作成する必要があります。 これを行うには、同じ「 Windows構成 」ノードで、「 レジストリ 」ノードに移動します。 ここで、詳細領域で最初の基本設定項目を作成するには、コンテキストメニューから[ 作成 ]および[レジストリ項目 ]（[ 新規]> [レジストリ項目] ）を選択する必要があります。
  5. 新しいレジストリ設定項目のダイアログボックスで、前の場合よりもはるかに多くのパラメーターを入力する必要があります。 ここでは、レジストリ設定が初めて作成されるため、アクションのドロップダウンリストから[ 作成]コマンドを選択する必要があります。 「 ハイブ 」ドロップダウンリストから、必要なパラメーターが作成されるレジストリキーを選択します。 この場合、それはHKEY_LOCAL_MACHINEになります 。 「 キーパス 」テキストボックスを使用して、パラメーターを作成するセクションを定義する必要があります。 このセクションは最初はレジストリエディタで利用できないため、レジストリエントリのブラウザを呼び出すことは無意味であり、パス全体を手動で入力する必要があります。 この例では、このパスはSOFTWARE \ Microsoft \ Windows \ CurrentVersion \ OEMInformationのようになります。 上記の3つのパラメーターは、作成された6つの設定項目すべてで同一です。 次の3つのパラメーター：「 パラメーター名 」、「 パラメータータイプ 」、「 値 」（ 値名 、 値タイプ 、 値データ ）を使用して、パラメーターの名前（ Logoなど ）、そのタイプ、 REG_SZが参照する（6つすべての場合） ）、および値自体も、このパラメーターの場合は「 C：\ OEMLOGO \ Biohazard.bmp 」のようになります。 レジストリ設定項目の一般的なパラメーターのデータ入力タブのダイアログボックスを以下に示します。





    図 6.レジストリ設定項目の作成



  6. 残りの5つの基本設定項目をすばやく作成するには、作成した最初の項目をクリップボードにコピーして、5回連続で貼り付け操作を実行します。 その後、複製された設定項目のプロパティダイアログで最後の3つのパラメーターを変更するだけで、必要なすべての項目が作成されます。 最終的に、グループポリシー管理エディターのスナップインウィンドウは次のようになります。

  7. 生成された基本設定項目をターゲットにするタスクが残っています。 それを実行するには、設定項目のプロパティのダイアログボックスを開き、「 一般オプション 」タブに移動します。 ここで、「 アイテムレベルのターゲティング 」ボックスをチェックし、「 ターゲティング 」ボタンをクリックして、ターゲティングエディターに移動する必要があります。 ターゲティングエディターで、作成要素のリストから「 オペレーティングシステム 」要素を選択し、製品のドロップダウンリストでターゲットオペレーティングシステムがWindows 8.1であることを示す必要があります。 ターゲティング要素を追加したら、設定要素への変更を保存できます。 次の図に、ターゲティングエディターを示します。

  8. すべてのアクションが完了したら、GPMEエディターのスナップインを閉じて、GPOを必要なユニットに関連付け、ターゲットコンピューターのポリシー設定を更新してください。


最終的に、クライアントコンピューターのポリシー設定を更新した後、システムプロパティウィンドウは現在の記事の図4とまったく同じように見えるはずです。
