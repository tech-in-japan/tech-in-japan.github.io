Netgear WNR2000：最新のファームウェアバージョンでのVoIPリカバリ

地獄のルーター



はい、実際には、ルーター自体は悪くありませんが、購入以来、私たちとはうまくいきませんでした。

最初は、これはディスクからルーターをインストールおよび構成するためのわかりにくいシステムです。 プラグインしているようで、すべてが動作し、プラグインして調整する必要がありますが、いいえ、Netgearはさらに行きました、インターネットアクセスとルーターとマップされたドライブへのネットワーク接続の両方が必要です、eeeBoxで私はこれで苦しめられました小さなeeeBox：USB-インターネット、マウス、キーボード、USB-CDROM、そして最後にネットワークケーブル-これは、すべてをWifiに転送したいという欲求のために時々起こります。

私の人生で初めて、このような妄想的なシステムに出会いました。デフォルトでは、ルーターにフラッシュイメージが生成されます。

すべてがマニュアルに従って厳密に行われましたが、その結果、ディスクからのフラッシュは機能しませんでした。ブリケットはありません-間違った言語のインストールによってWebインターフェースが完全に破壊された場合インストーラーはディスク上でこれに対応していませんでした）。そのため、機能せず、web-muzzle js-scriptsに登録されず、テキストがまったくありませんでした。 ネットワークの運命は、彼の手ですべてを行うことを教えていますが、いや、彼は急いで...

グーグル、タンバリン、60秒のペーパークリップでリセットボタンを押し、2回再起動するとシャーマニズムが発生し、最初からファームウェアのリカバリモードになりました。

しかし、私はうれしく思いました、ダンスはそこで終わりませんでした...



ルーターは、その後、左...


いいえ、もちろん、一部のLinksysほど末端に残っていませんが、不快でもあります。

古いファームウェアを使用すると、ルーターが不安定になり、ネットワークの速度が低下し、接続が定期的に完全に切断されます（まあ、直接Linksys）。

アップグレードする必要があり、問題が発生しました：新しいファームウェアでは、SIP ALG曲線がデフォルトでオンになり、このルーターを介したVoIP通信が完全に切断されますが、何らかの理由で切断機能を入力するのを忘れました非常に機能します）。

どこへ行くか、telnetアクセスを探しています...しかし、有効になっていません！ 幸いなことに、このようなNetgearシステムでtelnetアクセスを有効にする方法を学習したのはかなり前のことで、telnetEnable.exeユーティリティがこれを担当しています。

たとえば、ユーティリティをここからダウンロードし、ルーターにアクセスできるコンピューターで次のように使用します。



telnetEnable.exe 192.168.1.1 00FF00FF00FF Gearguy Geardog



ここで、プログラムの最初の名前、ルーターの2番目のIPアドレス、3番目のMACアドレス（arp -aまたはルーターステータスのルーターステータスで確認できます）、4番目のログインとパスワード

私の例では、MACアドレスを除くすべてがデフォルトになっています。

その結果、次の手順に進むことができます...



telnet 192.168.1.1


システムは予想どおりbusybox + openwrt + ashです。

そして、簡単な操作：



/ usr / sbin / nvram set wan_endis_sipalg = 1

/ usr / sbin / nvram commit



ルーターを再起動できますが、ダーティモジュールは起動しません（ip_conntrack_sip_alg ip_nat_sip）。

（上記の方法で）telnetを再度手動でオンにする必要があります。もう1つ、ファームウェアを変更するとルーターのMACアドレスが変更されます。

これが誰かを助けることを願っています。
