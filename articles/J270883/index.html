HighLoad ++ 2015外国人スピーカーの目を通して

みなさんこんにちは！



昔々、約4年前、ビシュケクに座っている間、私は会議のビデオを見て、どんな種類のスピーカーがクールな専門家であるかを考えました。 この記事では、そのことについて説明します。 応募方法、準備方法、会議の開催方法。



今年の1月19日に申請書を提出し、10月5日に承認されました。 もちろん、多くの喜びがあり、レポートの準備をする時が来ました。



準備する


同僚がレポートの準備を手伝ってくれて、約3週間準備しました。

主催者自身も準備に非常に役立ち、レポートの構造、スライド、公の場で話す能力についてウェビナーを手配し、また、スカイプの個人的なトレーニングを行いました。 このおかげで、レポートは非​​常に興味深く詳細になるように準備されました。 次に、私たちは彼と一緒にオフィスで訓練し、彼の同僚にプレゼンテーションをして、モスクワに飛びました。



0日目



到着後、会議が開催されるのと同じ建物内にある水族館ホテルにチェックインしました。客室は主催者によって提供されました。 同日、人前で話すための対面トレーニングが計画されました。 結論としては、10〜15分以内にプレゼンテーションを行う必要があり、部屋の人々はステージでの行動についてフィードバックを行いました。 ステージでの行動の間違いを見て、他のスピーカーと話をして、建設や設置作業を見ることは非常に興味深いものでした。 実際、その日、会議室が準備されていました。一般的に、建設現場を観察し、その場所を大まかにナビゲートし始めることは非常に興味深いものでした。

また、会議自体で起こることの規模を想像することは非常に困難でした。



会議はどうでしたか



会議の初日が来ました。 私たちはその場所に到着し、登録を開始し、キューのバランスが適切に保たれていたことに驚いたので、すぐに登録に合格しました。

登録時に与えられたパッケージが本当に気に入りました。

持っていた：

  1. 2つのノートブックを含むフォルダー：1つは要約、2つ目はノート用の空白のノートブック
  2. バッジ
  3. ペン
  4. 会議プログラムシート。






ホールに入ると、膨大な数の人々が見つかりました。 彼らは非常に異なっていました。 ひげを生やしたとはいえない。 お腹と細いです。 達人の濃度は高かった。 特定の技術の開発者とチャットしたい人のためのスタンドもたくさんありました。 Videoブースの下にあるNginx、Sphinx、Erlyvideo、MySQL、Postgresqlがありました。 各スタンドは、開発者との興味深い議論を引き起こしました。



初日は付き合いや話をするのにとても忙しかった。 彼らは、彼らが計画し、触れ、それらをよりよく知りたいと思っていたいくつかの技術を自分自身で引き出しさえしました。



2日目も同様に興味深いものでした。 2日目、最初のコーヒーブレークの後、彼はフォールトトレラントタクシーサービスのアーキテクチャを構築する方法について報告しました。







会議の一般的な印象と興味深い知り合い。


使用しているテクノロジーについて、これらのテクノロジーの開発者と話すことができる場合、それは常にクールです。 彼らの仕事と製品に感謝し、開発プロセス、新機能、リリース日について尋ねることができます。

彼らはNginxの人々にnginScript、HTTP / 2について尋ねました。 percona、perconaサーバー5.7のリリース。 Posgtres-professionalのメンバーとPGのクラスターおよびその一般的な使用について長い間話し合いました。

また、Qratorラボに、実際に行った興味深い事例について質問しました。



一般的に、すべての人は長い間、自分の意見を聞き、何かについて話し合うことができました。 この技術が議論された枠組みの中で、技術トークの形式で開かれた議論もありました。



興味深い観察は、オープンな製品を自分で作っている人たちは、コミュニケーションに対して非常にオープンだということです。



また、コーヒーブレイクの後に1つの部屋に入り、次のコーヒーブレイクまでのポストワークに関するすべてのレポートを聞くことも非常に便利でした。



トレンドについて



今年のトレンドは、Postgresql、インメモリDB、Docker、DevOps、フロントエンドパフォーマンスです。

レポートのうち、データベースに関するすべてのレポート、クラスメートからのレポート、新しいLinuxディストリビューションへの切り替え方法、およびそれらからの事故のテストに関するすべてのレポートが気に入りました。



会議全体で不便だった唯一のことは、インターネットが機能しなかったことです。 建物内のWi-Fiもモバイルもありません。



完了
