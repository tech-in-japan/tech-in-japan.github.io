ルーターの構成の要約





多くの人がYotaとWiMAXのことを知っています。



この記事では、NeoiovskyルーターをセットアップしてWiFiおよびイーサネット経由でWiMAXを配布するための情報を要約します。

これについても同様のソリューションを検討します。



はじめに



Yotaが何であるかを既に知っている場合は、この部分をスキップしてください。 そうでない場合は、簡単に説明します。



WiMAXは、高速インターネットアクセスを提供するテクノロジーです-いつでも、カバレッジエリア内のどこでも、最大10 Mbpsで、移動中であっても最大120 km / hの速度で接続をサポートします。

Yotaはロシア初のモバイルWiMAXネットワークです。 モスクワ、サンクトペテルブルク、ウファの2千万人以上の人々が住んでいる地域に配備されています。 ネットワークは、周波数範囲2.5〜2.7 GHzでIEEE 802.16e-2005標準に従って動作します。 （ カバレッジマップを参照 ）



ネットワークにアクセスするには、いくつかの異なるデバイスが使用されます。 Samsung U200モデムに焦点を当てます。 実際、インターネットに接続するには、このモデムをコンピューターのUSBポートに接続するだけです。 （ yota.ruの詳細 ）



しかし、1台のコンピューターではなく複数のコンピューターを接続する必要がある場合はどうすればよいでしょうか？ すべてのコンピューターにモデムを購入できますが、高価です。 また、Wi-Fiと通常のネットワークの両方で、任意の数のコンピューターにインターネットを「配布」する特別なルーターを購入できます。



そのようなルーターを作ることをお勧めします。





0.ショッピング旅行



構成するには、次のものが必要です。

1）ルーター

-ASUS WL-500G Premium V1＆V2（サンクトペテルブルクでは3,148ルーブルの価格を知っています）

-DLINK DIR-320（1800ルーブル。住所用-PM）



クラス内の2つの違い。 Asusは、より信頼性の高いルーターです（ハードウェアに関しては、ソフトウェアは同一です）。



2）Yotaモデム

選択の余地はありません

-Samsung U200モデム（CCCで1990ルーブル）



3）指示に従ってすべてを行う意欲と能力



1.A）Wl-500Gpルーターの設定





1）Olegからファームウェアのルーターを更新します（http://oleg.wl500g.info/1.9.2.7-10/）

2）通常の作業のように、ルーターを有線インターネットに接続します。 モデムをUSBポートに接続します。

3）telnet 192.168.1.1に移動します。 ログインパスワード：デフォルト 管理者

4）次のコマンドを実行します。



ハブではチームを正しく配置できないため、 チームはここにいます。



5）すべてが動作するはずです-モデムがネットワークに接続されると、モデムのライトが青に変わります。



1.B）DLINK-320ルーターの構成





元の手順： seculab.ru/dlink-320



ここで説明されているwl500g.infoフォーラムからのファームウェアのインストール手順、およびここで説明されているmadwimaxドライバー設定に基づいています。



wl500gファームウェアが追加されました。

-/ usr / sbinディレクトリに、Yota Samsung SWC-U200からのモデム用のドライバーを含むmadwimaxファイル。

-/ libディレクトリ内のファイルlibusb-1.0.so.0およびlibrt.so.0

-/ usr / sbinディレクトリに、madwimaxからのスクリプトを含むmadwimax-scripts.tgzアーカイブ。 正直なところ、インターネットからダウンロードしないようにするために、ファームウェアのどこにそれらを配置するのかわかりませんでした。



madwimaxドライバーは、次の目的で再コンパイルされました。

-/opt/usr/lib/ld-uClibc.so.0にインストールされているライブラリの代わりに、ファームウェアに既に存在する/lib/ld-uClibs.so.0ライブラリを使用します

-スクリプト「events.sh」/opt/etc/madwimax/events.shへのパスをハード登録します（デフォルトのパスはmadwimaxからの相対パスです）



ファームウェアdlink-320-madwimax-yota.rarでアーカイブをダウンロードします。 ファームウェアは以下に基づいています。

-Oleg、ily、theMironのファームウェア：wl500g-1.9.2.7-d-r285

-WiMaxドライバー：madwimax-0.1.0-r160



ネットワークケーブルをdlink dir-320とコンピューターに接続します。 IPアドレス192.168.0.2、マスク255.255.255.0、およびゲートウェイ192.168.0.1をコンピューターに設定します。 自動受信をオンにしないでください その場合、コンピュータにはdlinkでブートローダーを動作させてファームウェアをアップロードする瞬間をつかむ時間がありません。



アーカイブを解凍し、flashing.cmdを実行して、ルーターを電源に接続します。 運がよければ、ブートローダーがチームを「待機」している瞬間になります。 運が悪い場合は、電源を再接続してdlinkしてください。 正常に実行された場合、pingは実行を停止し、次のような内容が書き込まれます。



*** Start Flashing ***

: 3592480 11 cs, 281134 /









ステータスインジケータが点灯した後（約1〜2分）、ルーターをデフォルト設定にリセットします。 これを行うには、ルーターの電源をオフにし、リセットボタンを押します。ボタンを放さずに電源をオンにします。 5秒後、ボタンを離します。 その後、コンピューターのネットワークカードのIPアドレスを変更して、DHCP経由でアドレスを自動的に取得できます。



モデムをルーターに差し込みます。



telnet 192.168.1.1をルーターに移動し（デフォルトのユーザー名とパスワードはadmin）、次のコマンドを入力します。



# ,

mkdir /tmp/local/opt

mount /tmp/local/opt /opt



# var,

mkdir /opt/var

mkdir /opt/var/run



# /tmp madwimax'

cp /usr/sbin/madwimax-scripts.tgz /tmp/madwimax-scripts.tgz

cd /

tar -xzf /tmp/madwimax-scripts.tgz

rm /tmp/madwimax-scripts.tgz



#

mkdir /usr/local/sbin



# post-boot

echo "#!/bin/sh" > /usr/local/sbin/post-boot

echo "mount /tmp/local/opt /opt" >> /usr/local/sbin/post-boot

echo "insmod tun.o" >> /usr/local/sbin/post-boot

echo "(while true; do /usr/sbin/madwimax -qof ; sleep 10; done )&" >> /usr/local/sbin/post-boot



#

chmod +x /usr/local/sbin/*



#

echo "/usr/local/sbin/post-boot" > /usr/local/.files



#

flashfs save

flashfs commit

flashfs enable

reboot



* This source code was highlighted with Source Code Highlighter .







それだけです 成功すると、モデムが青く点灯します（小さな電球）



1.B）推奨事項



モデムをUSBケーブルに接続することをお勧めします。

-WiMAXとWiFiの周波数は互いに干渉しません。 （延長コードがないと、速度が4倍遅くなることがありました）

-すぐに持ち歩くことができます（ルーターをネットワークから切断した後、モデムを引き出すことをお勧めします）。
