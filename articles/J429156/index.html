セマフォの使用



Nucleus SEでは、セマフォはアセンブリ段階で定義されます。 アプリケーションには最大16個のセマフォを含めることができます。 セマフォが指定されていない場合、サービス呼び出しとデータ構造のコードはアプリケーションに含まれません。



セマフォはU8型のカウンタであり、複数のタスクが使用できるようにアクセスが制御されます。 タスクは、セマフォカウンタの値を減らす（キャプチャ）か、増やす（リリース）ことができます。 値がゼロのセマフォをキャプチャしようとすると、選択したAPI呼び出しパラメータとNucleus SEの設定によっては、エラーまたはタスクの一時停止が発生する場合があります。



セマフォのセットアップ



セマフォの数



ほとんどのNucleus SEオブジェクトと同様に、セマフォの設定はnuse_config.hの #defineディレクティブによって決定されます 。 主なパラメーターはNUSE_SEMAPHORE_NUMBERで、アプリケーション内のセマフォの数を決定します。 デフォルトでは、パラメーターは0に設定され（アプリケーションではセマフォは使用されません）、最大16の値をとることができます。不正な値はコンパイルエラーにつながります。このモジュールとともに）、その結果、 #errorディレクティブが起動します。



ゼロ以外の値を選択すると、セマフォの主要なアクティベータとして機能します。 このパラメーターは、データ構造を定義するときに使用され、そのサイズはその値に依存します（詳細については、この記事の詳細を参照してください）。 さらに、ゼロ以外の値はAPI設定をアクティブにします。



API呼び出しをアクティブにする



Nucleus SEの各API関数（ユーティリティ呼び出し）は、 nuse_config.hの #defineディレクティブによってアクティブにされます 。 セマフォの場合、次のものが含まれます。



NUSE_SEMAPHORE_OBTAIN

NUSE_SEMAPHORE_RELEASE

NUSE_SEMAPHORE_RESET

NUSE_SEMAPHORE_INFORMATION

NUSE_SEMAPHORE_COUNT









デフォルトでは、これらはFALSEに設定されているため、各サービス呼び出しを無効にし、それらを実装するコードの組み込みをブロックします。 セマフォを設定するには、必要なAPI呼び出しを選択し、対応するディレクティブをTRUEに設定する必要があります。



以下は、デフォルトのnuse_config.hファイルからの抜粋です。 



 #define NUSE_SEMAPHORE_NUMBER 0 /* Number of semaphores in the system - 0-16 */ #define NUSE_SEMAPHORE_OBTAIN FALSE /* Service call enabler */ #define NUSE_SEMAPHORE_RELEASE FALSE /* Service call enabler */ #define NUSE_SEMAPHORE_RESET FALSE /* Service call enabler */ #define NUSE_SEMAPHORE_INFORMATION FALSE /* Service call enabler */ #define NUSE_SEMAPHORE_COUNT FALSE /* Service call enabler */


アプリケーションにセマフォがない場合にアクティブ化されたAPI関数は、コンパイルエラーになります（常に有効になっているNUSE_Semaphore_Count（）を除く）。 アクティブ化されていないAPI呼び出しをコードが使用する場合、実装コードがアプリケーションに含まれていなかったため、レイアウトエラーが発生します。



ユーティリティセマフォ呼び出し



Nucleus RTOSは、次の機能を提供する8つのサービスコールをサポートしています。



  • セマフォのキャプチャ。 Nucleus SEはNUSE_Semaphore_Obtain（）関数に実装されています。
  • セマフォを解放します。 Nucleus SEはNUSE_Semaphore_Release（）関数で実装されます。
  • セマフォをすべての一時停止されたタスクのリリースで未使用状態に戻す（リブート）。 Nucleus SEでは、 NUSE_Semaphore_Reset（）で実装されています。
  • 特定のセマフォに関する情報を提供します。 Nucleus SEでは、 NUSE_Semaphore_Information（）で実装されます。
  • アプリケーションで構成されているセマフォの数を返します。 Nucleus SEでは、 NUSE_Semaphore_Count（）で実装されています。
  • 新しいセマフォをアプリケーションに追加します。 Nucleus SEは実装されていません。
  • アプリケーションからセマフォを削除します。 Nucleus SEは実装されていません。
  • すべてのセマフォへのポインタを返します。 Nucleus SEは実装されていません。


各サービスコールの実装については、以下で詳しく説明します。



セマフォをキャプチャおよびリリースするユーティリティ呼び出し



セマフォで実行できる基本的な操作は、キャプチャとリリース（値の増減）です。 Nucleus RTOSおよびNucleus SEは、これらの操作に対して2つの基本的なAPI呼び出しを提供します。



セマフォのキャプチャ



セマフォをキャプチャするためのNucleus RTOSユーティリティ呼び出しは非常に柔軟で、現在値がゼロのセマフォをキャプチャしようとした場合など、操作を現時点で実行できない場合、暗黙的または特定のタイムアウトでタスクを一時停止できます。 Nucleus SEは同じ機能を提供します。タスクの一時停止のみがオプションであり、タイムアウトは実装されていません。



Nucleus RTOSでセマフォをキャプチャするための課題

サービスコールのプロトタイプ：

STATUS NU_Obtain_Semaphore（NU_SEMAPHORE *セマフォ、UNSIGNEDサスペンド）;



パラメータ：



セマフォ -ユーザーが提供するセマフォ制御ブロックへのポインタ。

suspend-タスクの一時停止パラメータ。値NU_NO_SUSPENDまたはNU_SUSPENDとタイムアウト値を取ることができます。



戻り値：



NU_SUCCESS-呼び出しは正常に完了しました。

NU_UNAVAILABLE-セマフォにヌル値がありました。

NU_INVALID_SEMAPHORE-セマフォへの無効なポインタ。

NU_INVALID_SUSPEND-非タスク関連スレッドから一時停止を試みます。

NU_SEMAPHORE_WAS_RESET-タスクの中断中にセマフォがリセットされました。



Nucleus SEでセマフォをキャプチャするための課題

このAPIコールは、Nucleus RTOS APIのコア機能をサポートしています。



サービスコールのプロトタイプ：



STATUS NUSE_Semaphore_Obtain（NUSE_SEMAPHOREセマフォ、U8サスペンド）;



パラメータ：



semaphore-使用されているセマフォのインデックス（ID）。

suspend-タスク一時停止パラメーター。NUSE_NO_SUSPENDまたはNUSE_SUSPENDになります。



戻り値：



NUSE_SUCCESS-呼び出しは正常に完了しました。

NUSE_UNAVAILABLE-セマフォにヌル値がありました。

NUSE_INVALID_SEMAPHORE-セマフォインデックスが無効です。

NUSE_INVALID_SUSPEND-非タスク関連スレッドからの一時停止の試行、またはAPIブロック機能が無効になっている場合。

NUSE_SEMAPHORE_WAS_RESET-タスクの中断中にセマフォがリセットされました。



Nucleus SEでのセマフォキャプチャの実装

機能コードNUSE_Semaphore_Obtain（） （パラメーターの確認後）のバージョンは、ブロック（一時停止）タスクのサポートがアクティブかどうかに応じて、条件付きコンパイルを使用して選択されます。 両方のオプションを検討してください。



ロックが有効になっていない場合、このAPI呼び出しのロジックは非常に簡単です。 



 if (NUSE_Semaphore_Counter[semaphore] != 0) /* semaphore available */ { NUSE_Semaphore_Counter[semaphore]--; return_value = NUSE_SUCCESS; } else /* semaphore unavailable */ { return_value = NUSE_UNAVAILABLE; }


セマフォカウンタの値がチェックされ、ゼロに等しくない場合は減少します。



タスクのロックが有効になっている場合、ロジックはより複雑になります。 



 do { if (NUSE_Semaphore_Counter[semaphore] != 0) /* semaphore available */ { NUSE_Semaphore_Counter[semaphore]--; return_value = NUSE_SUCCESS; suspend = NUSE_NO_SUSPEND; } else /* semaphore unavailable */ { if (suspend == NUSE_NO_SUSPEND) { return_value = NUSE_UNAVAILABLE; } else { /* block task */ NUSE_Semaphore_Blocking_Count[semaphore]++; NUSE_Suspend_Task(NUSE_Task_Active, semaphore << 4) | NUSE_SEMAPHORE_SUSPEND); return_value = NUSE_Task_Blocking_Return[NUSE_Task_Active]; if (return_value != NUSE_SUCCESS) { suspend = NUSE_NO_SUSPEND; } } } } while (suspend == NUSE_SUSPEND);


いくつかの説明が役立つ場合があります。



コードはdo ... whileループに配置されます 。このループは、 suspendパラメーターがNUSE_SUSPENDの間に実行されます。



セマフォにゼロ以外の値がある場合、値は減少します。 サスペンド変数はNUSE_NO_SUSPENDに設定され、API呼び出しは終了し、 NUSE_SUCESSを返します。



セマフォがnullで、 サスペンド変数がNUSE_NO_SUSPENDに設定されている場合、API呼び出しはNUSE_UNAVAILABLEを返します。 一時停止がNUSE_SUSPENDに設定されている場合、タスクは一時停止します。 呼び出しが完了した後（タスクの再開時など）、戻り値がNUSE_SUCCESS （リセット後ではなく、セマフォの解放後にタスクが再開されたことを示す）の場合、サイクルは最初から始まります。



セマフォ解放



Nucleus RTOS APIを呼び出してセマフォを解放するユーティリティは非常に簡単です。セマフォカウンタの値が増加し、成功メッセージが返されます。 Nucleus SEは同じ機能を提供しますが、追加のオーバーフローチェックがあります。



Nucleus RTOSでセマフォを解放する課題

サービスコールのプロトタイプ：



STATUS NU_Release_Semaphore（NU_SEMAPHORE *セマフォ）;



パラメータ：



semaphore-ユーザーが提供するセマフォ制御ブロックへのポインタ。



戻り値：



NU_SUCCESS-呼び出しは正常に完了しました。

NU_INVALID_SEMAPHORE-無効なセマフォポインター。



Nucleus SEでセマフォを解放するための課題

このAPIコールは、Nucleus RTOS APIのコア機能をサポートしています。



サービスコールのプロトタイプ：



STATUS NUSE_Semaphore_Release（NUSE_SEMAPHOREセマフォ）;



パラメータ：



セマフォ -解放されたセマフォのインデックス（ID）。



戻り値：



NUSE_SUCCESS-呼び出しは正常に完了しました。

NUSE_INVALID_SEMAPHORE-セマフォインデックスが無効です。

NUSE_UNAVAILABLE-セマフォの値は255であり、増やすことはできません。



Nucleus SEでのセマフォリリースの実装

機能ロックNUSE_Semaphore_Release（） （パラメータをチェックした後）は、タスクロックがアクティブになっているかどうかに関係なく一般的です。 セマフォカウンタの値がチェックされ、255未満の場合は増加します。



API呼び出しのブロック（タスクの中断）のサポートが有効になっている場合、条件付きコンパイルを使用してさらにコードが選択されます。 



 NUSE_CS_Enter(); if (NUSE_Semaphore_Counter[semaphore] < 255) { NUSE_Semaphore_Counter[semaphore]++; return_value = NUSE_SUCCESS; #if NUSE_BLOCKING_ENABLE if (NUSE_Semaphore_Blocking_Count[semaphore] != 0) { U8 index; /* check whether a task is blocked */ /* on this semaphore */ NUSE_Semaphore_Blocking_Count[semaphore]--; for (index=0; index<NUSE_TASK_NUMBER; index++) { if ((LONIB(NUSE_Task_Status[index]) == NUSE_SEMAPHORE_SUSPEND) && (HINIB(NUSE_Task_Status[index]) == semaphore)) { NUSE_Task_Blocking_Return[index] = NUSE_SUCCESS; NUSE_Wake_Task(index); break; } } } #endif } else { return_value = NUSE_UNAVAILABLE; } NUSE_CS_Exit(); return return_value;


このセマフォでタスクが中断されると、最初のタスクが再開します。
