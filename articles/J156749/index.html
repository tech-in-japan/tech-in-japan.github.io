Windowsでは、同じことをCtrl-ZとEnterで強制的にgetcharにEOFを返します。

/ etc / fstab



その類似物は、WindowsレジストリのHKEY_LOCAL_MACHINE\SYSTEM\MountedDevices



です（はい、はい、WindowsにはUNIXと同様に、マウントの概念があります）。 ところで、あるパーティションから別のパーティションにインストールされたWindowsを転送したり、Windowsのディスクの名前を変更したりする場合、これが解決策です！



ここで、あるパーティションから別のパーティションにWindowsを転送することを計画した人たちのための物語があります。 残りはスキップして次のセクションに進むことができます。



たとえば、Windowsをあるパーティションから別のパーティションに移動した場合、2つの手順を実行する必要があります。ブートローダーとその構成（次のセクションで詳しく説明します）を処理し、このレジストリキーを編集します（一般に、 GNU / Linuxの場合：GRUB + fstab）。 キーが編集されていない場合、Windowsは起動できません。そのため、新しいパーティション（つまり、Windowsをコピーしたパーティション）からロードを開始し、古い（つまり、Windowsをコピーしたパーティション）から続行します。 システムのすべてのものが新しいものから読み込まれ、すべての種類のカスペルスキーが古いものから読み込まれます。 そのため、エントリ「（デフォルト）」を除くすべてのエントリをこのブランチから削除する必要があります。 その後、Windowsはパーティションについて知っていたすべてを忘れ、次に起動するときに、パーティションを開始元（必要なもの）と見なします。 つまり、パーティションを自動的に検出します。



確かに、いくつかのWindowsのレジストリをロードせずに編集するにはどうすればよいかという疑問が生じます。 うーん、これにはWindows PEを使用しました。これは海賊版Windowsビルドキットの一部であったため、詳細は説明しません。 （このスレッドの他の方法を参照してください： EndUserからgeektimes.ru/post/156749/#comment_5355987 。）



さらにいくつかのオプション：

  • レジストリをクリーンアップするには、コンピューターの電源を切り（「ハード」にすることができます）、Windowsをコピーします。 しかし、私はそれをチェックしませんでした。 Windowsがこのレジストリキーを編集できるようにし、オフにしたときに再びレジストリキーが入力されないということはありません。
  • レジストリを編集し、すぐにコンピューターの電源を切らずに、実行中のWindowsを別のパーティションにコピーします。 しかし、これもチェックされませんでした。
  • ある物理ディスクから別の物理ディスクにWindowsをコピーする場合、コピー後に古い物理ディスクをコンピューターから削除し、新しいWindowsを少なくとも1回起動できます（WindowsがMountedDevicesを更新するように）。 その後、古いディスクを挿入できます。 同じ物理ディスク上のパーティションでも同じトリックを実行できます。単に、古いパーティションを一時的に削除してから復元するだけです（これは非常に注意して行う必要があることは明らかです）。 そしてもちろん、これはすべてテストされていません:)）
  • GNU / LinuxでWindowsレジストリをwinregfsとしてマウントする




そして最後に：パーティション（Acronisなど）をコピー/バックアップするために異なるプログラムを使用しても意味がありません：それらはすべてレジストリを変更しないため（ほとんどの場合）、この状況では機能しません（同じコンピューター内でWindowsをコピーします）。
