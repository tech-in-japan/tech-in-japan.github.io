MicrosoftはSQL Serverを開く必要があります

Microsoftの元従業員Stephen Walliは、約2年前に自分のスタートアップを始めるために会社を辞めました（彼はまだどの会社かは言いません）。 MicrosoftでInterixのプロダクトユニットマネージャーとして働いた後、UNIXプログラムをWindows NTに移植するためのInterix環境を開発したスタートアップであるSoftway Systemsを設立しました。 Microsoftは1999年にSoftway Systemsを買収し、Stephen Wallyはそこで約5年間働いた後、Optarosのオープンソース戦略開発担当副社長の地位に移りました。



Stephen Wallyは彼のブログで、Microsoftがオープンソース分野ですべきことに関する実用的なヒントを含む2つの興味深い投稿を投稿しました。 彼の意見では、Microsoftが以前に行ったこれらの小さな実験は明らかに十分ではありません。 より大胆で重要な措置を講じる必要があります。



Stephen Wallyは、最初の投稿で、 MicrosoftにSQL Serverソースコードの公開を開始し、この製品のビジネスモデルをSQL Server Networkの有料サブスクリプションに変更するようアドバイスしています。 言い換えれば、Red Hat NetworkがMySQLネットワークに正常に使用するモデルを採用することをお勧めします。 実際、SQL Serverの購入者はプログラムをまったく購入せず、SQL Serverベースのソリューションであり、サポートが不可欠です。 これに基づいて、SQL Serverネットワークへのサブスクリプションが需要があると想定できます。 ソースコードは、GPL3ではなくGPL2ライセンスで公開するのが最適です。



今後2年間で、誰かが間違いなくMySQLとSQL Serverを「ミックス」しようとするでしょう。 結局のところ、オープンDBMSはモジュール構造であるため、このような実験を促進します。 さらに、非常に一般的な状況は、 MySQLデータベースがWindowsで実行されている場合です。



2番目の投稿では、 Stephen WallyがSharePoint Serverについて同様のアドバイスを提供しています。 また、GPL2ライセンス、オープンソース、およびSharePoint Server Networkの有料サブスクリプションから利益を得て公開することもできます。



Wallyはまた、MicrosoftがEclipse FoundationおよびMonoコミュニティとの関係を確立し、Eclipse Public LicenseおよびMIT X11の（Rotor）ECMA Base Classの下でVisual StudioおよびRotor 2.0をリリースすることを推奨しています。
