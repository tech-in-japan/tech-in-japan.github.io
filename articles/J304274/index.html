GFI Archiver：メールストレージ

今日、保存された情報の量と組織にとっての価値は毎年増大していると言う必要はありません。 これは事実であり、保存するだけでは十分ではありません。以前に保存したアーカイブから必要な情報を取得する可能性を、最短時間で最小限のコストで確保する必要があります。 電子メールのアーカイブは、最も重要な企業タスクの1つです。









なぜこれが必要ですか？



ほとんどの組織にとって、電子メールは最も重要な要素です。そのため、情報を保存するためのリソースの必要性が高まっています。 従業員の電子メールボックスのサイズに制限を設定すると、追加の電子メール管理ツールが必要になります。ユーザーは、PSTまたはNSFファイルのローカルシステムに通信を保存することにより、このメカニズムをバイパスできます。 これにより、データが失われ、バックアップタスクが複雑になる可能性があります。



電子メールアーカイブソリューションにより、ユーザーはメールボックスのオーバーフローの問題から解放され、利用可能なストレージ容量とストレージリソースをより効率的に使用できるようになります。したがって、組織が情報を保存するために必要なリソースと財務コストを大幅に削減できます。 既存の電子メールシステムの機能に関係なく、このような製品は、情報の効果的な管理、保存、検索に役立つインテリジェントなソリューションを提供します。



この目的のために設計された製品を使用して通信をアーカイブすると、リソースの制限に関連する問題の解決に役立ち、電子メールシステムのパフォーマンスとスケーラビリティも向上します。 アーカイブソリューションを実装すると、次の基本的な利点が得られます。



  • 一元化された電子メールのアーカイブ。

  • 通信をアーカイブに保存し、新しいシステムで回復することにより、高速で便利な電子メールの移行。

  • アーカイブへの直感的なユーザーアクセス。



GFI Archiver製品を使用して、メールアーカイブの問題を解決する方法を見てみましょう。 GFI SoftwareのArchiverソフトウェアソリューションは、すべての企業のメールとドキュメントを一元的にアーカイブするように設計されています。 単一障害点の問題を解決するだけでなく、すべてのアーカイブ要素にリアルタイムでアクセスできます。







アーカイバでは、レター、フォルダ、メールボックス、メールサーバー全体、ドキュメントのあらゆるバージョンをすばやく復元できます。



GFI Archiverを使用すると、企業は電子メールをアーカイブおよび管理し、カレンダーおよびファイル履歴を操作して、データに簡単にアクセスできます。 現在、この製品は、アーカイブストア内のすべてのメールを収集するためのプログラムではなく、はるかに機能的なソリューションです。







構成セクションでは、電子メールの収集元のサーバーを指定できます



どのように機能し、何が得られますか？



これは、GFI Archiverでできることをリストすることで説明するのが最も簡単です。 そして彼は、以下を含む多くのことを知っています。



  • ネットワーク上のすべてのメールをアーカイブします。 GFI Archiverは、メールサーバーを通過するとすぐにすべての受信メールと送信メールを収集します。これにより、メールサーバーに問題が発生した場合や、ユーザーが受信直後に手紙を削除した場合でも、最新の手紙の安全性が確保されます。



  • すべてのメールのコレクション。 通信の履歴全体を保存するには、従業員のコンピューターとメールサーバーのストレージに既にあるすべてのメールをアーカイブに追加します。



  • メールメッセージの圧縮。 ファイルの圧縮と重複排除により、データ量は最大5倍に削減されます。 複数の受信者に送信された場合でも、各レターは単一のコピーに保存されます。 文字と添付ファイルのテキストは別々に保存されます。 圧縮により、メールサーバーからレターを保存する負担がなくなり、ストレージ容量の要件が大幅に削減されます。



  • アーカイブへの永続的なアクセス。 ユーザーは常に、Webインターフェースまたは電子メールクライアントを介して手紙や文書のアーカイブにアクセスし、任意のパラメーターでインスタント検索を実行し、アーカイブからの解凍を待たずに検索結果を操作できます。



  • 任意の電子メールクライアントからアクセスします。 アーカイブへのアクセスは、Webインターフェースだけでなく、IMAPプロトコル経由でも可能です。つまり、従業員は、おなじみのメールクライアントで保存された手紙を見ることができます。 同時に、フォルダ構造全体、検索機能、および文字の処理が完全に保持されます。



  • 信頼できるストレージ。 MS SQL Serverベースのストレージは、必要な信頼性を提供するだけでなく、データベースの標準的な手段を使用してさらにアーカイブを行い、メディアとメディアからのリカバリ、バックアップ、ロードバランシングを遅くします。



  • メッセージの完全なコピー。 すべてのレターはアーカイブに変更なしで保存され、ヘッダーは元の形式で復元され、調査で使用できるように保持されます。



  • 文書のアーカイブ。 GFI Archiverでは、メールに加えて、ユーザーのコンピューターから個々のファイルとドキュメントを保存できます。 従業員間でドキュメントを交換するための共有フォルダーを構成できます（Dropbox、Google Drive、Yandex.Diskに似ていますが、インターネット上でファイルを公開しません）。ドキュメントのすべてのバージョンは、それらのいずれかを復元する機能で自動的に保存されます。



  • 強力な検索エンジン。 多数のパラメーターによって、すべての手紙と文書をすばやく検索、表示、復元できます：キーフクロウ、添付ファイル、送信者、受信者、サイズ、日付、会話、変更の著者など。さらに、検索結果はほぼ瞬時に表示されます。 GFI Archiverは、検索エンジンが行う方法と同様の特別な検索インデックスを作成します。



  • 手紙の詳細な分析。 GFI Archiverは、手紙の内容を分析できます。冒proを検出し、従業員をコミュニケーショングループに分割し、手紙に対する従業員の反応を計算し、メールボックスを空にし、キーワードとテンプレート（クレジットカード番号、ドキュメント）によって情報漏えいを特定します。つまり、DLPシステムの機能を実装します。 レポートは、調査中の要求時とスケジュールの両方で作成されます。



  • 共有フォルダー。 GFI Archiverの一部であるFile Archiving Assistantを使用すると、Dropbox、Yandex.Disk、Google Driveなど、ユーザー間でドキュメントを交換するための特別なフォルダーを整理できますが、GFI Archiverの場合、企業ネットワークからデータは送信されません。



  • 手紙を集めるための規則 。 すべてのメールボックスまたは特定のメールボックスのみからの手紙のコレクションを指定でき、スパムとしてマークされたメッセージの処理方法も指定できます。 たとえば、そのような手紙はアーカイブされますが、しばらくすると自動的に削除されます。







アーカイブボルトを表示する



アーカイブストアストレージには、メールメッセージのメタデータとヘッダーが保存されるデータベース、ディスク上の保存場所を示すインデックス、およびメールデータ自体が含まれます。



GFI Archiverによって実行される操作は、3つの主要な段階に分けることができます。

1.メールサーバーまたはOutlookクライアントからのメールの収集。

2. GFIアーカイブストアでのアーカイブ。

3.ユーザーは、モバイルデバイスまたはOutlookから、Webインターフェイスを介してアーカイブされたメールを表示できます。







モバイルアクセス



仕事には何が必要ですか？



GFI Archiverをインストールして実行するには、Windows Server 2008〜2012、32ビットまたは64ビットが必要です。 VMwareまたはHyper-Vのサーバーもサポートされています。 さらに、GFI ArchiverはExchangeサーバーに直接インストールできますが、これはテストまたは開発の目的にのみお勧めします。 この製品はMicrosoft Exchange 2003、2007、2010、2013をサポートしています。 Office 365（Exchange Online）; IMAPメールサーバー Kerio ConnectとGoogle Apps for Business。



ほとんどの場合、Microsoft SQL Server（2005以降）を使用してアーカイブを保存します。 実際、GFI ArchiverはメッセージヘッダーとメタデータのみをSQLに保存し、コンテンツ自体は圧縮、暗号化、およびディスクへの個別書き込みが行われます。 SQL Expressもサポートされており、最大25ユーザーのインストール用に、GFI Archiverには無料のFirebirdデータベースが含まれています。



GFI Archiverのインストールと構成



GFI Archiverのインストールは非常に簡単です。特別なユーティリティを使用すると、プロセスを段階的に進めることができます。 Active Directoryを使用するかどうかを決定するだけです。



次に、管理者名が追加されます。 インストールユーティリティは、コンポーネントの存在を確認し、必要なコンポーネントをインストールします。 インストールが完了すると、[構成]ボタンをクリックして製品のセットアップを開始します。 [アーカイブストア設定]セクションで、使用するデータベースのタイプを[アーカイブ方法]で選択できます-アーカイブ方法を設定します。 手動アーカイブを選択すると、ユーザーはアーカイブするメールを決定できます。 この方法はExchangeを必要とせず、Outlookを使用している企業に適しています。



[すべての電子メール]メソッドはExchangeログを使用します-ログに置かれたすべてのメールはアーカイブされます。 GFI Archiverは、IMAPサーバーからメールを収集し、MS Active Directoryを使用することもできます。 また、ルールベースのアーカイブ方法を使用すると、アーカイブルールを設定し、アーカイブに選択的にメールを保存できます。







アーカイブ方法の選択







メール保存およびスパム処理ポリシーを構成する



ログイン/パスワードを開始して入力すると、[ホーム]タブが開きます。 そのグラフィックと統計は、GFI Archiverの操作のパラメーターに対する洞察を提供し、問題が発生した場合の解決に役立ちます。

たとえば、GFI Archiverサーバーで使用可能な空き容量、アーカイブされているメッセージの数、使用されているアーカイブストアの数、GFI Archiverイベントに関する情報が表示されます。



[アーカイブ]タブでは、使用可能なすべてのメールボックスのアーカイブ済みアイテムを表示できます。 右側のパネルでレターの内容を表示し、許可されている場合はアーカイブからレターを削除し、アーカイブからメッセージを特定のメールボックスに復元したり、リダイレクトしたり、カレンダーアイテムをアップロードしたりできます。



アーカイブを検索するには、追加の条件を指定した簡易検索またはクイック検索を使用できます。 高度な検索は、さらに高度な基準を提供します。 スマートフォンからアーカイブ内のメールを検索することもできます。



[構成]タブは、GFI Archiverの構成を構成するために使用されます。特に、アーカイブ対象（アーカイブソース）、使用方法（アーカイブ方法）、アーカイブルール（アーカイブルール）、アーカイブプロセスからの除外の制限（アーカイブの制限）などを指定します。







アーカイバ設定ウィンドウでは、アーカイブ方法の設定、管理者のロールとアクセス権の決定、ストレージのアーカイブ、メールボックスの構造の表示、メールの分類、スパムの保存と操作、システムの監査などの基本操作を管理できます



現在のアーカイブだけでなく、Archiverのインストール前に蓄積された以前の通信もアーカイブにアップロードできます。 これは、アーカイブアシスタントまたはインポートおよびエクスポートツールを使用して行われます。



GFI Archiverは、アーカイブからメールを復元するために、メールボックス復元と同じインポート/エクスポートツールの2つのツールも提供しています。



GFI Archiverレポート



GFI Archiverは、いくつかの優れたタイプのレポートを提供します。 これを行うには、MailInsightsと呼ばれるツールがあり、オンデマンドまたはスケジュールに従ってレポートを作成します。 レポートは、さまざまな問題の特定、セキュリティの問題の特定、および組織内のプロセスの合理化に役立ちます。 そのようなレポートのほんの数例がここにあります。







通信フローレポート通信フローは、組織内外の通信フローを監視します

運用監視







ストレージリソースの使用に関するレポートを使用すると、アーカイブ全体および特定のユーザーに保存されている内容を把握できます。 ユーザーの保持ポリシーと警告ルールを作成するときに使用できます。



メールクライアント



ユーザーがアーカイブにアクセスするために、GFI Archiverは、IMAPプロトコルを使用してアーカイブストアからアーカイブされたメッセージを配信する多くの電子メールクライアントをサポートしています。 AndroidおよびiOS 5、Apple Mail、Outlook 2003以降、Mac用Outlook 2011、Mozilla Thunderbird、Windows Live Mailでサポートされているメールアプリケーション。 また、古いアーカイブには、Outlook Connectorを介してOutlookからアクセスできます。 このプラグインは、GFI ArchiverとOutlookを同期し、GFI Archiverによってアーカイブされたメッセージをユーザーが表示できるようにします。 アーカイブにアクセスする別の方法は、Webインターフェイスです。 GFI Archiverは、Exchange Webサービスを使用してOffice 365に接続することもできます。
