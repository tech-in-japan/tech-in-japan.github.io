LeftHandは単なるiSCSIではありません

約1年前、HPが2008年に買収した同名の会社が開発したLeftHandディスクアレイが、ストレージシステムのHP製品ポートフォリオに登場しました。



通常、LeftHand製品は、iSCSIベースのSAN用に設計されたエントリレベルのディスクアレイとのみ見なされます。 ただし、現在HP StorageWorks P4000と正式に呼ばれているLeftHandの開発は、多くの高度なデータ保護機能を実装することで他のシステムより際立っています。その一部は最近まで、ハイエンドディスクアレイでしか見つかりませんでした。



StorageWorks P4000機能の利点について説明する前に、そのアーキテクチャを簡単に説明することは理にかなっています。 P4000システムはストレージモジュール（基本的には冗長電源とファンを備えた標準のHP ProLiantアーキテクチャのラックマウントデュアルプロセッササーバー）から構築され、各ストレージコントローラー、2つのネットワークシングルまたはオプションの10ギガビットイーサネットカード、RAIDコントローラー、そしてもちろんリジッドMDL SASドライブ。 ストレージモジュールは、専用のSAN / iQオペレーティングシステムを実行しています。



iSCSI over Ethernetを介して、これらのモジュールはクラスターに接続され、それらのリソース（ディスク、RAM、ネットワークインターフェイス、キャッシュメモリ）は共通のプールに結合されます。 前述のように、各ノードには2つのネットワークカードがあるため、フォールトトレランスのために、クラスターノード間の接続をイーサネットネットワーク接続用に予約することもできます。 StorageWorks P4000クラスターでは、異なるノードに物理的にインストールされたドライブで構成されるディスクボリュームを作成できます。 クラスター管理は、iSCSI SANに接続されているWindowsまたはLinuxコンピューターにインストールされている集中管理コンソール（CMC）を使用して実行されます。



それでは、プールされたリソースを使用したSAN / iQ管理下で、P4000ストレージモジュールのクラスターができることを見てみましょう。

  • 新しいStorageWorks P4000モジュールをクラスターに追加することにより 、 ストレージ容量が水平方向にスケーリングされます。 この操作はホットモードで実行され、SAN / iQは新しいモジュールのディスクにボリュームを自動的に拡張します（SAN / iQのボリューム構成を変更するときのネットワーク負荷を減らすために、帯域幅調整トラフィック調整機能が使用されます）。 さらに、ノードをクラスターからオンラインで除外できます。
  • StorageWorks P4000は階層型ストレージをサポートしています。たとえば、高速で高価なSASディスクにデータが保存されているボリュームを作成し、このデータが古くなった場合、別のモジュールのSATAディスクに移動できます。 物理ディスク間でのボリュームの移動とSAN / iQボリュームの構成の変更は自動的に行われます-システム管理者はデータを手動で移行してボリューム構成を再定義する必要はありません。
  • SAN / iQのネットワークRAIDを使用すると、異なるStorageWorks P4000モジュールにインストールされたディスクから分散RAIDアレイを構築できます。 最初のバージョンでは、ネットワークRAIDはRAIDレベル1のみをサポートしていました。 異なるモジュールのディスクを相互にミラーリングできますが、今年初めにリリースされた新しいバージョンのStorageWorks P4000 G2はすでにRAIDレベル5および6をサポートしているため、より経済的な（有効容量のオーバーヘッドに関して）データ複製を提供できます。 ネットワークRAIDを使用すると、異なるサイトにインストールされたストレージモジュールを、災害に強い分散RAIDアレイに結合できます。
  • SAN / iQは、動的な容量割り当てシンプロビジョニングの機能を実装しています。 割り当てられた物理容量が使い果たされた場合、追加のディスクをオンラインで提供できます。 シンプロビジョニングは、ドライブの購入コストを削減します-顧客は将来使用するために容量を購入する必要はありません。
  • スマートクローンボリュームのインスタントクローニング-システムイメージが保存され、変更のみがコピーされますが、データ自体はコピーされません。 このようなクローンは、アプリケーションソフトウェアの新しいバージョンを開発およびテストし、SANを介してサーバーOSをロードするときに、運用データを操作するのに非常に便利です。
  • スナップショットスナップショット-リモートコピー機能の使用を含め、バックアップとデータリカバリを大幅に簡素化します。


これらの機能はすべてStorageWorks P4000 G2の標準構成に含まれており、これらのシステムの所有者はソフトウェアライセンスの購入にお金をかける必要がありません（ハイエンドおよびミドルクラスのディスクシステムの購入者とは異なります）。 さらに、SAN / iQは標準アーキテクチャの任意のサーバーで実行できるため、HPはこのOSをx86サーバーハードウェアプラットフォームでiSCSIストレージシステムを構築するための個別のP4000 VSAソフトウェア製品として提供しています。
