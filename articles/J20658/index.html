Web 3.0今

ちょうど1年半前、ロシア語で「web 2.0」という用語が定義されました。 ネットワークジャーナリストのAndreevによると、Web 2.0は、トムソーヤー（ポータルの所有者が読む）との悪名高いシーンでマークトウェインによって説明されています。強さ）。 これは公平な状況ですか？ もちろん違います。 Web 2.0コミュニティの不完全さを認識し、Webtrinolの時代を予測するのは、この不正義です。





Webtrinolワークフロー：



1.ユーザーがコンテンツを作成します。



2.読者はサイトにアクセスし、コンテンツを読み、サイトのページに投稿されたコンテンツを評価し、広告を表示します。



3.各素材の評価と視聴回数は、ユーザーの利益を計算するための係数です。 係数は、ユーザーがこのリソースを使用している時間、ユーザーが公開した資料の数、ユーザーの公開の履歴全体での資料の平均評価、およびサイトで追加のタスクがあるかどうかにも影響されます。 そのため、ユーザーはセクションのモデレーターになり、紛争を解決し、家庭教師の役割を果たします。これにより、給与の「増加」を受け取ります。



4.コミュニティの作成者の総収入は2つの部分に分けられます。それぞれの「給料」係数に従って、彼は自分用に半分、ユーザー用に半分を保持します。



このスキームがリソースの作成者にとって有益なのはなぜですか？



1.リソースの作成者は広告を必要としません。 ユーザーは、他のコミュニティ、LiveJournal、ソーシャルメディアを通じてリソース（そしてもちろん自分の出版物）を積極的に宣伝し、記事のクリック性、読みやすさ、評価を高めます。



2.作成者は編集者を必要としません。 標準以下のコンテンツはあまり人気がないので、ユーザーはコンテンツの品質を管理します。 規格外のコンテンツは、ユーザーの全体的な評価を下げるため、規格外のコンテンツを避け、リソースでの地位の維持に努めます。



3.作成者はモデレーターを必要としません。 モデレーターの役割は、ユーザー自身が（たとえば、わいせつな言葉を使用したユーザーに格付けを下げるために）実行し、このために給与係数の増加を受けます。



4.作成者はプロジェクトマネージャーを必要としません。 それは自己組織化されたコミュニティであり、その目標は生存と開発（つまり、コミュニティの各メンバーが多くのお金を稼ぐ）であり、同様の関連コミュニティとの競争に勝つことです。



一般に、この状況で作成者に必要なのは、サービスの初期プロモーションとその物理的寿命（サーバー、ホスティングなど）の確保だけです。



ロシアで収益化されたソーシャルネットワークは2年以内に登場すると主張できます。



待ってます。
