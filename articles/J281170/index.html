間違った方向を向ける



常識から、安定したClonezillaを使用して作業するように指示されます。 しかし。 週末の前に、Dota 2は「手放し」ます。私の記憶から、Ubuntuでのテキストコンソールのローカライズに関する問題は長い間存在していたことがわかりました。



問題をGoogleで調べて、 Ubuntu 11.10以降に発生していることを確認します。 ユーティリティのさまざまな程度のソリューションがいくつかありますが、最も一般的なのは、initramfsでFRAMEBUFFER = yオプションを有効にし、update-initramfsコマンドでinitrdを再構築することです。 「考えないでください、私たちはそれを取ります！」という言葉で、FRAMEBUFFER = yという行をinitramfs.confファイルの最後に追加し、initrdイメージを更新して再起動します。 



 echo FRAMEBUFFER=y | sudo tee -a /etc/initramfs-tools/initramfs.conf sudo update-initramfs -u sudo reboot


再起動後、変更はありません。キリル文字ではなく、すべて同じ正方形です。 フォーラムでは、この方法は役に立たなかったと多くの人が不満を述べています。

今のところ、最も重要なものに注意を引くために、スクリプト内で定義されているすべての関数をスキップしました。 最初のコメントは、スクリプトはsetupconに基づいているが、udevルールに適合するようにトリミングされていると述べています。 言ってみよう。 以下は、設定ファイル（。/ Etc / default / console-setup）のインクルードです。 次に、スクリプトに渡された最初のパラメーターを確認します。このパラメーター（fbcon）だけで、スクリプトは/etc/init/console-font.confから呼び出されます。 アクティブな各コンソール（/ etc / default / console-setupに示されている）に対して、書き込みの可能性がチェックされ、各コンソールに対してsetup_font関数が呼び出されます。 同時に、スクリプトの作成者は、スクリプトが呼び出されるまでにコンソールを作成する必要があると書きます。そうでない場合、コンソールへの書き込みテストは機能せず、このコンソールは構成されません。 そして、ユーザー自身が後でsetupconを呼び出すことができます。 このメモを取り、ファイル/ lib / udev / console-setup-ttyのsetup_font関数を検討してください。 



 setup_font () { # Set the font and ACM. setfont will silently do nothing for a console # in graphics mode. SETFONT_ARGS= if [ "$FONT" ]; then FONT="/etc/console-setup/${FONT##*/}" FONT="${FONT%.gz}" else FONT="/etc/console-setup/$CODESET-$FONTFACE$FONTSIZE.psf" fi if [ -f "$FONT" ]; then SETFONT_ARGS="${SETFONT_ARGS:+$SETFONT_ARGS }$FONT" fi if [ "$ACM" ]; then ACM="/etc/console-setup/${ACM##*/}" ACM="${ACM%.gz}" else ACM="/etc/console-setup/$CHARMAP.acm" fi if [ -f "$ACM" ]; then SETFONT_ARGS="${SETFONT_ARGS:+$SETFONT_ARGS }-m $ACM" fi if [ "$SETFONT_ARGS" ]; then setfont -C "$1" $SETFONT_ARGS fi }

コメントで言及された/ lib / udev / console-setup-ttyのベースとなっているsetupconスクリプトを思い出してください 。 その中身を見てみましょう。

このスクリプトにはLinuxおよびFreeBSDのコードが含まれており、BSDに関連するすべてのものを安全にスキップできます。 最も興味深いのは、$ CODESETと$ FONTSIZEの処理だけです。 このスクリプトには、$ CODESETが指定されていないか、値が「推測」である場合の状況の処理があります。 この場合、$ CODESETは$ CHARMAPに依存する値を想定します。 この例では、$ CODESET = Uni2です。



$ FONTSIZEも未設定値または「推測」がないかチェックされ、もしそうなら「16」にハードコードされます。 $ FONTSIZEが8x *または* x8として指定されている場合、「x」記号と8つは破棄され、1桁のみが残ります（フォントの高さ）。 たとえば、「8x14」でした-「14」のままで、「15x8」からは「15」のままです。 $ FONTSIZEが2桁の「* x *」として指定されている場合： 



 *x*) a=${FONTSIZE%x*} b=${FONTSIZE#*x}
（aは最初の数字、bは2番目の数字です）、大きい方の数字が再配置されます： 

 if [ "$a" -lt "$b" ]; then FONTSIZE=${b}x${a} fi


たとえば、「10x20」でした-「20x10」になり、「22x11」は変更されません。 システムで利用可能なフォントの名前を見てみましょう：

私は1行だけが異なります：

OPTION=FRAMEBUFFER







おなじみのオプションだと思いました...興味のある3つのスクリプトは、 initramfs-toolsフォルダーにあります。 initramfs-toolsパッケージは、カーネルが起動するときにメモリにロードされ、メインファイルシステムが使用可能になるまでその時点で使用されるinitrdイメージの構築を担当します。 通常、 Initrdには、初期化スクリプトとその構成ファイルだけでなく、機器での操作やファイルシステムの接続に必要なカーネルモジュールが含まれています。 イメージは、最終的にmkinitramfsスクリプトを呼び出すupdate-initramfsスクリプトを使用して収集されます。 いつものように、 mkinitramfsの中身を見てみましょう。 



 … CONFDIR="/etc/initramfs-tools" ... . "${CONFDIR}/initramfs.conf" … # add existant boot scripts for b in $(cd /usr/share/initramfs-tools/scripts/ && find . \ -regextype posix-extended -regex '.*/[[:alnum:]\._-]+$' -type f); do option=$(sed '/^OPTION=/!d;$d;s/^OPTION=//;s/[[:space:]]*$//' "/usr/share/initramfs-tools/scripts/${b}") [ -z "${option}" ] || eval test -n \"\${$option}\" -a \"\${$option}\" != \"n\" || continue [ -d "${DESTDIR}/scripts/$(dirname "${b}")" ] \ || mkdir -p "${DESTDIR}/scripts/$(dirname "${b}")" cp -p "/usr/share/initramfs-tools/scripts/${b}" \ "${DESTDIR}/scripts/$(dirname "${b}")/" done


ここではすべてがかなり複雑に見えるので、例を示してみます。 このブロックは、/ usr / share / initramfs-tools / scripts /およびサブフォルダー内のすべてのファイルを調べ、 'OPTION ='を含む行を探します。 たとえば、ファイル/ usr / share / initramfs-tools / scripts / init-top / console_setupには次の行があります。



OPTION=FRAMEBUFFER







「OPTION」が欠落しているか設定されていない場合、ファイル（スクリプト）はinitrdにコピーされます。 「OPTION」が存在する場合、このオプションの値を変数の名前として使用し、「n」に等しく設定されていないかどうかを確認します。 この例では、変数$ FRAMEBUFFERがチェックされます。 initramfs.confファイルで、最初にこの変数をFRAMEBUFFER = yに設定します。 FRAMEBUFFER変数の値はsetup-consoleに関連するスクリプトでは使用されないため、トリガーとして機能し、「y」とは限らない任意の値に設定できます。 「no」または「none」でも「y」と同じように機能します。 したがって、FRAMEBUFFERが定義されており、「n」と等しくない場合、スクリプトはinitrdイメージに配置されます。 このようなスクリプトは8つあります。 



 cd /usr/share/initramfs-tools/scripts grep -rl FRAMEBUFFER ./ ./init-premount/brltty ./panic/plymouth ./init-bottom/plymouth ./init-top/keymap ./init-top/framebuffer ./init-top/console_setup ./init-top/brltty ./init-top/plymouth


これらのスクリプトは、フレームバッファを開始して設定します-大まかに言って、テキストコンソールはグラフィックモードに切り替わります。 その後、その中に画像とカスタムフォントを描画することが可能になります。 これは単なるconsole_setupであり、コンソールのフォントを設定します。 おそらくユーザーが非標準のフォントを選択できるという事実により、このスクリプトはフレームバッファーを開始するようにバインドされており、設定パラメーター「FRAMEBUFFER = y」なしではinitrdに追加されません。



したがって、フレームバッファーがアクティブになると、コンソールもインストールされたフォントで構成されますが、初期段階です。



しかし、キーボード構成からスクリプトに戻ります 



 ./lib/udev/console-setup-tty ./usr/share/initramfs-tools/scripts/panic/console_setup ./usr/share/initramfs-tools/scripts/init-top/console_setup ./usr/share/initramfs-tools/hooks/console_setup


スクリプト... / init-top / console_setupは、 'FRAMEBUFFER = y'パラメーターが設定されたinitrdイメージにコピーされます。



スクリプト... / panic / console_setupは、指定された変数「OPTION」が含まれていないため、常にinitrdにコピーされます。 panicディレクトリのスクリプトは、 initスクリプト（ファイル/ usr / share / initramfs-tools / scripts / functionsの関数を含む）のpanic関数によって呼び出されます。 パニック関数自体は、init initスクリプトが実行を継続できない場合（ルートファイルシステムが見つからないなど）に呼び出されます。 そのため、スクリプト... / panic / console_setupは、コンソールをパニックモードに設定して、ユーザーのネイティブエンコーディングでメッセージを表示するように設計されています。



スクリプト... / hooks / console_setupは、 initrdイメージの作成時にmkinitramfsスクリプトから呼び出されます。 



 ... CONFDIR="/etc/initramfs-tools" ... run_scripts_optional /usr/share/initramfs-tools/hooks run_scripts_optional "${CONFDIR}"/hooks


このスクリプトは、コンソールの設定に必要なファイル（フォントファイル、フォント変換テーブル（acm）、キーボードレイアウトファイル（keymap））をinitrdにコピーします。 したがって、init-top / console_setupを修正しても、hooks / console_setupの変更を忘れても、必要なファイルがないため、この段階ではコンソールは構成されません。
