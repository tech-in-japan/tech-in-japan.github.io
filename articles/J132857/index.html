大学のITインフラストラクチャ。 何が、何が望ましいとどのように正しく

すべての人に良い一日を！

数ヶ月間、私はウランウデの林業大学で働いています。 私の立場はまだ「電子技術者」と呼ばれていますが、私はシステム管理者、エニキー、ケーブル敷設、ソケットの設置、写真家、ビデオオペレーターなどの「専門家」の機能を果たしているようです。 など



このトピックがユーザーMuRADiNの要求で書かれたことをすぐに予約してください 、彼はこの記事のコメントで表現しました。 答えよりも質問が多いでしょう。なぜなら 彼は、知識のある人々がいくつかの解決策を提案し、エラーを指摘することを主な目的として考えました。 まあ、そして、私は私のレベルの「システム管理者」（特に教育関係者）だけでなく、コンピューターサイエンスの教師にとっても何かが役立つことを願っています。



次に、私たちが何を達成できたか、私が受け取りたいことについて話し、誰かが答えるのが面白いかもしれない質問をします。







ソースデータ


私が到着したとき、状況は次のように見えました。

PCの数： 約120（インベントリは最高レベルの「紙」ではなかったため、正確な数はまだ不明です）

ネットワーク： ピアツーピア。 すべてがネットワークに接続されているわけではありませんが、60パーセントスイッチとWI-FIルーターの両方を使用しています。

インターネット： ADSL、4 Mbps

プロキシサーバー： FreeBSDのSquid + SquidGuard

DHCPサーバー： いいえ

ユーザー： 従業員のほとんどは、「コンピューターリテラシー」レベルが非常に低い「中」および「平均以上」の年齢の人々です。 大学生は、ほとんどの場合、機能不全の家族、怪しげな会社からの子供、勉強不足または未完成の学校などです。

もちろん、ドメインはなく、ライセンスされたソフトウェアは実際には使用されません（一部は、ライセンスの有無を知らないという事実、一部は資金の不足または不合理な使用、一部は教師の負荷が重いためです） 、教育における最新のばかげた改革に関連して、それだけでは十分ではありません）、電子エンジニアの作業のための別の部屋はありません（2つのピースがあります）。そこでは、コンピューターを分解、組み立て、掃除機をかけるか、他の何かをすることができます。 私は情報学のオフィスの奥の部屋で仕事をしなければなりません。そのホステスは、異なる「ゴミ」があると非常に怒っています。 Windows XPを搭載したマシンに1C-schoolサーバーがあり、ネットワーク全体のいくつかのフォルダーが同じマシンで共有されています（！）。 さらに、「誰が成功したか、彼/彼女はスリッパであるか...」という原則に対するXPの制限のために、彼らはこれらのフォルダに行きます。

ZPは〜13000ルーブル、8：00から17:00までの営業日です



何をする


-あいまいな言葉や写真を使用して会話が行われ、その間に当局はドメインの必要性を説得しました。 注文済み（およびWindows Server 2008 r2の2つのライセンスと、それに接続するためのライセンスがすでに受信されています）。 注文、承認されましたが、これまでのところ（入札を行う必要があります）これらのサーバー用の鉄は購入されていません。

-別の部屋の必要性が証明されました。 困難でしたが、地下の同じ部屋を割り当てられました。

-OS Xbuntu linuxおよびLubuntu linux（古いコンピューター）は、図書館の公共のコンピューターおよびホステルの自習室にインストールされます。これにより、大学でのオープンソースソフトウェアの実装の少なくともいくつかの土台を作ることができました。

-ネットワーク内に、IT部門のポータルが編成されました。このポータルには、部門の電子アプリケーションのページ、部門のニュースとアナウンス、ユーザーへの指示が含まれていました。 承認後、同じページの管理者は、メールサーバー（FreeBSD上）とイントラネットジャバーサーバーのWebインターフェイスに入るためのリンクを持ちます。

ちなみに、このポータルは私の友人qindman によって Drupalに 無料で実装されています。

どうもありがとう、アンドレイ！

次のようになります。



-オープンソースソフトウェアの宣伝と、Windowsが（他のソフトウェアと同様に）ライセンスされている場合、同僚との説明作業が開始された

-すべての共通点を持ち、「プログラムを求めたが、インストールしてくれなかった」などの誤解を避けるために、教師にはトレーニングクラスの申し込みフォームが提供されました。 さらに、このリストは、たとえばフリーソフトウェアに置き換えることができるソフトウェアの分析と識別が簡単です。

-トレーニングクラスの1つで、makminiのMAC OSに対処し、トレーニングでそれらを使用するように教師に促すためにattempts病な試みをすることが判明しました（これまでのところ利用できません）



これはきしみながらきつくなりますが、うまくいきます。 私は喜んでいます。
