1.1インデックス付きコレクション



リスト（リスト）、タプル（タプル）、ストリング（ストリング）-インデックス付きコレクション（シーケンスとも呼ばれます）を検討してください。



インデックス付けとは、コレクションの要素が特定の順序で配置されることを意味し、各要素は、0（つまり、行の最初のアイテムのインデックスは1ではなく0）からコレクションの長さより1つ小さいインデックス（len（mycollection ）-1）。



1.2インデックスによる値の取得



すべてのインデックス付きコレクションについて、角かっこ内のインデックスによって要素の値を取得できます。 さらに、負のインデックスを指定できます。これは、逆の順序でカウントすることで、最後から要素を見つけることを意味します。



負のインデックスを指定する場合、最後の要素のインデックスは-1、最後から2番目のインデックスが続き、インデックスが負の符号を持つコレクションの長さの値に等しい最初の要素、つまり（-len（mycollection））になります。

要素 a b c d e
インデックス 0（-5） 1（-4） 2（-3） 3（-2） 4（-1） 
my_str = "abcde" print(my_str[0]) # a -   print(my_str[-1]) # e -   print(my_str[len(my_str)-1]) # e -       print(my_str[-2]) # d -


以下の例のリストのリストのように、コレクションにはいくつかのレベルのネストを含めることができます。 さらに深くするには、角カッコの2番目のペアなどを配置します。 



 my_2lvl_list = [[1, 2, 3], ['a', 'b', 'c']] print(my_2lvl_list[0]) # [1, 2, 3] -   —    print(my_2lvl_list[0][0]) # 1 —      print(my_2lvl_list[1][-1]) #  —


1.3インデックスによるリストアイテムの変更



タプルと行には不変のコレクションがあるため、インデックスによって要素のみを取得でき、変更はできません。 



 my_tuple = (1, 2, 3, 4, 5) print(my_tuple[0]) # 1 my_tuple[0] = 100 # TypeError: 'tuple' object does not support item assignment


しかし、リストの場合、インデックスによる要素の取得が式の左側にあり、代入演算子=が来ると、このインデックスを持つ要素に新しい値を設定します。 



 my_list = [1, 2, 3, [4, 5]] my_list[0] = 10 my_list[-1][0] = 40 print(my_list) # [10, 2, 3, [40, 5]]


UPD：注：このような割り当ての場合、要素はリストに既に存在している必要があるため、存在しないインデックスに要素を追加することはできません。 



 my_list = [1, 2, 3, 4, 5] my_list[5] = 6 # IndexError: list assignment index out of range

2.1スライスの構文



そのようなタスクでは、1つの要素ではなく、特定の単純なルール（たとえば、最初の5または最後の3、または2つおきの要素）によって制限されるセットを取得する必要が非常に頻繁にあります。ループを反復する代わりに、いわゆるスライス（スライス、スライス）。



インデックスまたはスライス（スライス）で要素を取得することは、元のコレクションを変更しないのが好きではないことを覚えておく必要があります。将来の使用（たとえば、別のコレクションへの追加、印刷、計算） コレクション自体は変更されないため、これは可変（リスト）シーケンスと不変（ストリング、タプル）シーケンスの両方に適用されます。



スライスの構文はインデックス作成の構文と似ていますが、角括弧で囲まれ、単一の値ではなく、2〜3がコロンで示されます。 



 my_collection[start:stop:step] # ,


スライス機能：



  • startとstopの負の値は、コレクションの最初からではなく、コレクションの最後からカウントする必要があることを意味します。
  • ステップの負の値-検索は右から左への逆順で実行されます。
  • start [：stop：step]が指定されていない場合、コレクションの端から、つまり最初の要素（それを含む）から、ステップが正の場合、または最後（それを含む）から、ステップが負の場合（したがって、末尾から先頭まで反復します） ）
  • 停止が指定されていない場合[start :: step] -コレクションの端まで、つまり、ステップが正の場合は最後の要素（それを含む）に、ステップが負の場合は最初の要素（それを含む）に移動します。開始）。
  • step = 1、つまり、左から右への順次検索はオプションであり、これがデフォルトのステップ値です。 この場合、 [start：stop]を指定するだけです
  • このようにすることもできます[：] -これは、コレクション全体を取得することを意味します
  • 重要 ： スライスするとき、最初のインデックスは選択に含まれ、2番目のインデックスは含まれません！

    つまり、開始から終了まで、ストップは結果に含まれません。 数学的には、これは[start、stop）と書くか、このルールで説明できます。 

     [ < > : <  > : <> ]


    したがって、たとえば、mylist [::-1]はmylist [：0：-1]と同一ではありません。最初のケースではすべての要素が含まれ、2番目のケースではインデックス0に到達しますが、オンにしないためです！


テーブルスライスの例：









表のコード例 
 col = 'abcdefg' print(col[:]) # abcdefg print(col[::-1]) # gfedcba print(col[::2]) # aceg print(col[1::2]) # bdf print(col[:1]) # a print(col[-1:]) # g print(col[3:4]) # d print(col[-3:]) # efg print(col[-3:1:-1]) # edc print(col[2:5]) # cde

2.3スライスを使用したリストの編集



注意が常に集中するわけではない重要な点は、スライスを使用するとコレクションのコピーを取得できるだけでなく、リストの場合は要素の値を変更したり、新しい値を削除したり追加したりできることです。



以下に例を示します。



  • 1つの要素を追加する場合でも、反復オブジェクトを渡す必要があります。そうしないと、TypeErrorエラーが発生します。反復可能オブジェクトのみを割り当てることができます 



     my_list = [1, 2, 3, 4, 5] # my_list[1:2] = 20 # TypeError: can only assign an iterable my_list[1:2] = [20] #     print(my_list) # [1, 20, 3, 4, 5]


  • スライスを使用して単一の要素を挿入できます。例のコードは以下にありますが、そのような構文は読みにくいため、そうすることはお勧めしません。 リストメソッドの.append（）および.insert（）をより適切に使用します。



    類似体のスライス.append（）およびinsert（） 
     my_list = [1, 2, 3, 4, 5] my_list[5:] = [6] #    —   .append(6) print(my_list) # [1, 2, 3, 4, 5, 6] my_list[0:0] = [0] #    —   .insert(0, 0) print(my_list) # [0, 1, 2, 3, 4, 5, 6] my_list[3:3] = [25] #    —   .insert(3, 25) print(my_list) # [0, 1, 2, 25, 3, 4, 5, 6]

  • 単にシーケンスの一部を削除できます

2.4インデックスの範囲を超える



範囲ではなく1つの要素のみを参照する場合、インデックスの反転は基本的にスライスの特殊なケースです。 ただし、目的のインデックスを持つ要素が欠落している状況の処理には、非常に重要な違いがあります。



存在しないコレクションインデックスにアクセスすると、エラーが生成されます。 



 my_list = [1, 2, 3, 4, 5] print(my_list[-10]) # IndexError: list index out of range print(my_list[10]) # IndexError: list index out of range


そして、スライスの境界がコレクションの境界を超えても、エラーは発生しません。 



 my_list = [1, 2, 3, 4, 5] print(my_list[0:10]) # [1, 2, 3, 4, 5] —     print(my_list[10:100]) # [] -    —    print(my_list[10:11]) # [] -  1   -  ,




注 ：スライスの機能が十分ではなく、より複雑な選択が必要な場合は、サイクルの記事4で説明されている式ジェネレーターの構文を使用できます。



3コレクションアイテムの並べ替え



コレクションの要素を並べ替えることは、一般的なタスクに常に見られる重要で人気のある機能です。 常に焦点を当てているわけではありませんが、非常に重要な機能がいくつかあります。



3.1 ソート済み （）関数



後で処理または出力するために、sorted（）関数を使用して、コレクションのソート済みアイテムをリストできます。



  • この関数は元のコレクションを変更しませんが、その要素の新しいリストを返します。

  • 元のコレクションのタイプに関係なく、その要素のリストが返されます。

  • 元のコレクションを変更しないため、不変のコレクションに適用できます。

  • 返されたアイテムを並べ替えるとき、アイテムの元のコレクションにインデックスがあるかどうかは関係ないため、インデックスのないコレクションに適用できます。

  • 追加のオプション引数があります。

    reverse = True-逆順でソートします

    key = funcname （Python 2.4以降）-特別なfuncname関数を使用してソートします。標準のPython関数、またはこのタスク用に特別に作成した関数とラムダのいずれかです。 


 my_list = [2, 5, 1, 7, 3] my_list_sorted = sorted(my_list) print(my_list_sorted) # [1, 2, 3, 5, 7] my_set = {2, 5, 1, 7, 3} my_set_sorted = sorted(my_set, reverse=True) print(my_set_sorted) # [7, 5, 3, 2, 1]


各要素の長さlen（）で文字列のリストをソートする例： 



 my_files = ['somecat.jpg', 'pc.png', 'apple.bmp', 'mydog.gif'] my_files_sorted = sorted(my_files, key=len) print(my_files_sorted) # ['pc.png', 'apple.bmp', 'mydog.gif', 'somecat.jpg']


3.2 反転 （）関数



reverse（）関数はシーケンスに使用され、異なる動作をします：



  • リスト自体ではなく、リストジェネレーターを返します。

  • ジェネレータではなく既製のリストを取得する必要がある場合は、結果をlist（）でラップするか、逆（）ではなくスライス[:: -1]を使用できます。

  • 要素を並べ替えるのではなく 、逆の順序で要素を返します。つまり、リストの最後から読み取ります。

  • 前の段落から、コレクションにインデックスが付けられていない場合-要素を逆順で表示できず、この関数はそのようなコレクションに適用できない場合-「TypeError：argument to reverse（）must be a sequence」が得られることは明らかです。

  • 追加の引数を使用できません-「TypeError：reverse（）does not take keyword arguments」というエラーが表示されます。 


 my_list = [2, 5, 1, 7, 3] my_list_sorted = reversed(my_list) print(my_list_sorted) # <listreverseiterator object at 0x7f8982121450> print(list(my_list_sorted)) # [3, 7, 1, 5, 2] print(my_list[::-1]) # [3, 7, 1, 5, 2] -


3.3リストメソッド。 ソート （）および。 逆 （）



リスト（およびそれのみ）には、ソート（）およびリバース（）の対応する関数と同じことを行う特別なメソッド.sort（）および.reverse（）がありますが、同時に：



  • 元のリスト自体を変更しますが、新しいリストは生成しません。

  • 新しいリストではなく、なしを返します。

  • 同じ追加の引数をサポートします。

  • さらに、リスト自体を最初のパラメーターとして渡す必要はありません。さらに、これを行うと、エラーが発生します。引数の数が正しくありません。 


 my_list = [2, 5, 1, 7, 3] my_list.sort() print(my_list) # [1, 2, 3, 5, 7]


注：初心者の頻繁なエラーは、インタープリターのエラーではありませんが、取得したい結果にはつながりません。 



 my_list = [2, 5, 1, 7, 3] my_list = my_list.sort() print(my_list) # None


3.4辞書ソートの機能



辞書のソートには、辞書要素がキーと値のペアであるという事実に起因する独自の特性があります。



UPD：また、ディクショナリのソートについて話すとき、ディクショナリから受信したデータをソートして出力またはインデックス付きコレクションに保存することを意味することを忘れないでください。 標準のディクショナリ自体でソートされたデータを保存することはできません;それらは、他のインデックス化されていないコレクションのように、ランダムな順序で保存されます。



  • ソート済み（my_dict）-追加のメソッドを呼び出さずに辞書をソート関数に渡すと、キーのみが列挙され、ソートされたキーのリストが返されます。
  • ソート済み（my_dict。keys（））-前の例と同じ結果ですが、より明確に記述されています。
  • ソート済み（my_dict。 項目 （））-キーでソートされたタプル（キー、値）のソート済みリストを返します。
  • ソート済み（my_dict。 値 （））-ソート済みの値のリストを返します 


 my_dict = {'a': 1, 'c': 3, 'e': 5, 'f': 6, 'b': 2, 'd': 4} mysorted = sorted(my_dict) print(mysorted) # ['a', 'b', 'c', 'd', 'e', 'f'] mysorted = sorted(my_dict.items()) print(mysorted) # [('a', 1), ('b', 2), ('c', 3), ('d', 4), ('e', 5), ('f', 6)] mysorted = sorted(my_dict.values()) print(mysorted) # [1, 2, 3, 4, 5, 6]


値のリストだけが必要ではなく、値でソート順でペアを表示する場合 、キーではなく値で辞書をソートすると、別の問題が発生する可能性があります。



この問題を解決するために、ラムダ関数lambda x：x [1]を特別なソート関数として転送することができます。各ステージで受け取ったタプル（キー、値）は、タプルの2番目の要素をソートに使用します。 



 population = {"Shanghai": 24256800, "Karachi": 23500000, "Beijing": 21516000, "Delhi": 16787941} #    : population_sorted = sorted(population.items(), key=lambda x: x[1]) print(population_sorted) # [('Delhi', 16787941), ('Beijing', 21516000), ('Karachi', 23500000), ('Shanghai', 24256800)]

 shop = [('', 1200), ('', 1000), ('', 300), ('', 100), ('', 1500), ('', 12000), ('', 2000), ('', 200), ('', 2700)] def prepare_item(item): return (item[0], -item[1]) shop.sort(key=prepare_item)

サードパーティの関数を使用する代わりに、実用的な関数を生成しないために、ラムダ関数を使用して同じ効果を実現できます。

ShashkovS UPD：3.6ソートの安定性



データを最初に列Aで昇順に並べ替え、次に列Bで降順に並べ替え、最後に列Cで再び昇順に並べ替えるとします。



列Bのデータが数値の場合、キーの適切な関数を使用して、Bの要素の符号を変更できます。これにより、目的の結果が得られます。

そして、すべてのデータがテキストの場合はどうなりますか？ そのような機会があります。

事実、ソートのソートはPython（Python 2.2以降）で安定しています。つまり、「同じ」要素の順序は変更されません。



したがって、異なるキーで単純に3回ソートできます。 

 data.sort(key=lambda x: x['C']) data.sort(key=lambda x: x['B'], reverse=True) data.sort(key=lambda x: x[''])
