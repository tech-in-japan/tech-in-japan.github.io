HP BL460cおよびEVAサーバーブレード上のHA（高可用性）VMware vSphereクラスター

ProLiantサーバー でEVAおよびiLO アレイを使用することに関する知識の実用的なアプリケーションは、少し前に入手したものですが、vSphereでのアクセス性の高いクラスターの展開です。



このクラスターは、中規模および大規模企業に使用して、計画外のダウンタイムを削減できます。 ビジネスでは、24時間365日モードでのクライアントへのサービスの可用性などのパラメーターが重要であるため、このソリューションは高可用性クラスターに基づいています。 クラスターには常に少なくとも2つのサーバーが含まれます。 当社のソリューションでは、VMwareを実行しているサーバーが互いの状態を監視し、その時点でそのうちの1つだけがリーダーになり、ビジネスアプリケーションを備えた仮想マシンがその上に展開されます。 マスターサーバーに障害が発生した場合、その役割は自動的に2番目になりますが、ビジネスアプリケーションへの顧客アクセスは実質的に中断されません。











1.タスクの説明



この例では、BL460c G7ブレードサーバー上で非常にアクセスしやすいVMwareクラスターを作成するプロセスを説明します。 この装置は、HP c7000ブレードバスケットと、VMware HA（高可用性）クラスターが構成されている2つのBL460c G7ブレードサーバーで構成されています。 モスクワのHPデモセンターでは、現在G7ブレードサーバーモデルのみが利用可能ですが、説明した例に基づいて、新しいGen8ブレードサーバー上にクラスターを構築することもできます。 一般に、構成は変わりません。 ストレージシステムはHP EVA 4400です。Ethernetおよびファイバーチャネル接続の場合、c7000のベイ1および2に2つのHP Virtual Connect Flex Fabricモジュールがあります。



2.コンポーネントの説明









Virtual Connect FlexFabricは、c7000ブレードバスケット用の統合仮想化I / Oモジュールです。



HP Virtual Connect FlexFabricモジュールは、Flex-10、FCoE、Accelerated iSCSIテクノロジーを使用して、ブレードサーバーとネットワークインフラストラクチャ（SANおよびLAN）を10Gbの速度で切り替えます。



可能性：シャーシ内の異なるブレード間だけでなく、同じブレードサーバーが存在するリモートサイトへの「プロファイル」（MACテーブルとWWNサーバー）の転送。



また、バーチャルコネクトを使用する場合、ブレードサーバーのネットワークカードの2つのポートはそれぞれ、ポートあたり合計10GbEの帯域幅を持つ4つの仮想ポート1GbE / 10GbE / 8Gb FC / iSCSIに分割できます。 これにより、サーバーごとに8つの仮想ポート（BL460c G7上）から取得できます。これは最初の初期化中に設定する必要があります。



唯一の制限は、HP Virtual Connect FlexFabricがフルスイッチではないことです。 これらはアグリゲーターであり、その動作のために、別々のイーサネットスイッチとSANスイッチが必要です（ただし、最新のファームウェアでは、HP P10000 3PARとの直接接続が登場しています）。



使用される機器の一般的なビューは、Onboard Administratorに反映されます。











ProLiant BL460c G7に統合されたNC553iデュアルポートFlexFabric 10Gbは、追加のメザニンカードなしでHP Virtual Connect FlexFabricモジュールと直接連携できます。 2つのHP Virtual Connect FlexFabricモジュールがインストールされ、ブレードサーバーとEVA 4400ストレージシステムおよびイーサネットスイッチ間の通信パスにフォールトトレランスを提供します。



この例では、HP Virtual Connect FlexFabricモジュールからのトラフィックがEthernetスイッチとFCファクトリーに分割されるシナリオを検討します。 この場合のFCoEは、バスケット内にブレードサーバーがあるモジュールを切り替えるために使用されます。



HP Virtual Connect FlexFabricモジュールには、さまざまなプロトコルで動作するトランシーバーが必要です。 この例では、4 Gb FC SFPトランシーバーが各モジュールのポート1および2に取り付けられ、10 Gb SFP +トランシーバーがポート4および5に取り付けられました。このテストでは、10 GbEチャネルを使用してデモセンターのネットワーク機器を使用しました。











この図は、NC553iデュアルポートFlexFabric 10Gbネットワークアダプターの3つのポートが、仮想マシンが相互に通信し、ネットワークインフラストラクチャに接続するためのFlexNICとして構成されていることを示しています。 残りのポートは、FCoEとして構成され、SANインフラストラクチャおよびEVA 4400ストレージシステムと通信します。



各Virtual Connectモジュールとともに、HP Virtual Connect Manager管理ソフトウェアがEnterpriseモジュール用に提供されます-HP Virtual Connect Enterprise Manager。 Enterprise Managerを使用すると、1つのコンソールから最大256個のドメインを管理できます（1つのドメイン-Virtual Connectを備えた4つのブレードシステム）。



モジュールを接続したら、構成する必要があります。 これは、バスケット自体をOnboard Administratorに接続するか、工場でデフォルトでモジュールに割り当てられているIPアドレスによって実行されます。



3. Virtual Connect Managerでプロファイルを構成します











HP FlexFabricを接続した後、ホストのドメイン、イーサネットネットワーク、SANファクトリ、およびサーバープロファイルを定義する必要があります。 これらの設定は、HP Virtual Connect Managerまたはコマンドラインで行われます。 この例では、ドメインはすでに作成されているため、次のステップでは、VC共有アップリンクセットを定義して、HTC-77-AおよびHTC-77-Bという名前の仮想マシンのイーサネットネットワークをサポートします。 インストールウィザードで「VLANタグのマップ」機能をオンにします。











「VLANタグのマップ」機能により、1つの物理インターフェイスを使用して、共有アップリンクセットを使用して複数のネットワークを操作できます。 このようにして、サーバーのネットワークインターフェイスカードに多くのVLANを構成できます。 共有アップリンクセットのパラメーターは、VCMインストールウィザードで設定されました。 SUSジョブの例：















HTC-77-AネットワークとHTC-MGMT-Aネットワークの両方が割り当てられました。VLAN_Uplink_1はモジュール1のポート5に、 2番目の共有アップリンクセット（VLAN_Uplink_2）はモジュール2のポート5に割り当てられ、HTC-77-B（VLAN 20）およびHTC-MGMT-B（VLAN 21）ネットワークが含まれます。 VCMでイーサネットネットワークを作成する例：







HP Virtual Connectモジュールは、c7000バスケットバックプレーンへの高性能10Gb接続を使用して、アップリンクのない内部ネットワークを作成できます。 内部ネットワークは、VMware vMotionおよびネットワークフォールトトレランスの組織に使用できます。 このネットワークのトラフィックは、c7000バスケットの外部ポートには行きません。 この例では、2つのVMware vMotionネットワークが構成されました。 HP-MGMT-AおよびHP-MGMT-Bという名前の管理コンソール用に2つのネットワークが構成されました。 スロット1および2のHP Virtual Connect FlexFabricモジュールのポート6は、管理コンソールへの接続に使用されました。











その後、SANの工場セットアップが開始されます。 非常に簡単：VCMで、[定義-SANファブリック]を選択します。 コンパートメント1、ポートX1のSAN-1モジュールを選択します。 2番目のSAN-2モジュールの場合、ポートもX1です。 SAN構成の例：















最後の手順は、以前に行ったすべての設定をサーバープロファイルに「接着」することです。 これは、いくつかの方法で実行できます。たとえば、VCM WebインターフェイスまたはVirtual Connect-Virtual Connect Scripting Utility（VCSU）コマンドラインを使用します。 OSをインストールする前に、すべてのシステム、NIC、およびFCドライバーが更新され、すべてのネットワークパラメーターが設定されていることを確認する必要があります。 CVMでプロファイルを作成する例：







VCMで、[定義]-[サーバープロファイル]を選択し、プロファイル名を入力して、以前に作成したものからプロファイルのネットワーク接続を選択します。











この場合、すべてのポートは事前に割り当てられています。各プロファイルに6つのイーサネットポートと2つのSANポートがあります。 コンパートメント11と12のプロファイルは同一です。 各サーバーは同じポートのセットを認識します。 合計速度は、サーバーのネットワークカードの各ポートで10 Gb / sを超えません。



将来ネットワークを拡張する予定がある場合は、プロファイル内の未割り当て（未割り当て）ポートを割り当てることをお勧めします。 これにより、サーバーをシャットダウンして再起動することなく、これらのポートをさらにアクティブ化できます。











デフォルトでプロファイルを作成する場合、イーサネットセクションで使用できるポートは2つだけです。 さらに追加するには、右マウスボタンでコンテキストメニューを呼び出し、「接続を追加」を選択する必要があります。



ポートの各タイプに速度を割り当てます：管理ポート-500Mb / s、vMotionポート-1Gb / s、SANポート-4Gb / sおよび仮想マシンポート-4.5Gb / s。



プロファイルを作成した後、それらは同じメニューでバスケットのブレードコンパートメントに接続されます。











4. VMwareクラスターの構成



これでバスケットの設定が完了しました。 次に、ブレード自体へのリモート接続とVMware ESX 5.0の展開があります。iLO を介したProLiantサーバーへのリモートOSインストールの例はすでに説明されています 。



この場合、Windowsの代わりに、インストーラーのドロップダウンメニューでVMwareが選択され、ディストリビューションへのパスが示されます。



インストール後、サーバーが再起動し、仮想マシンの構成を開始できます。 これを行うには、ESXサーバーに接続します。 構成セクションに移動し、Virtual Connectモジュールと同様に、ホストごとに6つのvmnicを作成します：サービスコンソール（vSwitch0）、VMネットワーク（vSwitch1）、vMotion（vSwitch2）。 したがって、各ネットワークが2つのvmnicに対応するフォールトトレラントな構成が得られます。











vmnicが追加されるとすぐに、Virtual Connect Managerでの構成中に、各アダプターがサーバープロファイルで指定された速度に自動的に設定されることがわかります。











vmnicを追加したら、vSwitchを使用してそれらをグループに結合します。[ポート]タブ-[構成]フィールドで[vSwitch-編集]を選択し、NICチーミングを選択して、両方のvmnicが表示されることを確認します。 ロードバランシング「発信元仮想ポートIDに基づくルート」を選択します-これらの設定は、デフォルトで設定することをお勧めします（この方法の詳細については、VMware Webサイトを参照してください 。











5. EVA 4400ディスクアレイの構成



前のレビューでは、EVAディスク管理インターフェイス-Command Viewの使用方法について説明しました。











この例では、2つの仮想マシンホスト間で共有される500 GBのLUNを作成しています。 前の記事で説明した方法を使用して、仮想ディスクが作成され、2つのESXホストに提示されます。 LUNのサイズは、OSのタイプとこのクラスターが実行する役割によって決まります。 クラスターの前提条件（特にVMware vMotionの場合）-LUNを共有する必要があります。 新しいLUNはVMFSでフォーマットし、仮想マシンのRawデバイスマッピング（RDM）機能を有効にする必要があります。



6. VMware HAおよびVMware vMotionの展開



HP Virtual ConnectとVMware vMotionを併用すると、2つのHP FlexFabricモジュールのみが必要になることを除き、2つのVC-FCモジュールと2つのVC-Ethモジュールを使用する場合と同じレベルの冗長性を提供します。 HP Virtual Connect FlexFabricを使用すると、ネットワーク用の共通LUNへのフォールトトレラントパスとフォールトトレラントネットワークインターフェイス統合（NICチーミング）を整理できます。 VMware vSphereのすべての設定は、ドキュメントで説明されているベストプラクティスに適合しています 。



クラスターの可用性を確認しました。 Windows 2008 Server R2仮想マシンがいずれかのホストに展開されました。















仮想マシンは、あるサーバーから別のサーバーに手動で数回移行されました;移行中、クラスターは利用可能なままでした。
