Megaplanでの音声通話の埋め込み方法

Webテクノロジーでは、サードパーティのコンポーネントを簡単に使用できるようにする標準のHTML / JS / CSS基盤の存在が本当に気に入っています。 「通話」ボタンを追加します。 PCアプリケーションでの音声テレフォニーの機能は、フレームワーク、ライブラリ、メッセージ、ストリーム、およびオペレーティングシステムに関する長い話です。 Webアプリケーションに同じボタンを追加することは、1つの要素と1つのJavaScriptライブラリの問題です。 カットの下で、私はそのようなボタンの1つが統合されたストーリーを語ります。







メガプランの音声通話が選ばれる理由





Megaplanは、Lebedevの設計によるタスク管理システムであるだけでなく、便利なCRMシステムでもあります。 各クライアントとの関係の履歴を1か所に保持することで、従業員は毎日の多くの連絡先をナビゲートできます。 進化的に、私たちの脳は約150の「なじみのある」顔を記憶するように適合されています。これは、パック動物から実際に来た場合のパックの最大サイズです。 しかし、多くのビジネスエリアでは、1日あたり150人の連絡先が一般的です。 CRMシステムは、困難なコミュニケーションのビジネスにおいて信頼できるツールとして機能し、従業員の溶けている脳を可能な限り解放するように設計されています。



音声通信は非常に重要です。 言葉は、原則として、思考を表現するための不完全なツールであり、進化的に、私たちは、表情やイントネーションを通じて、意思決定に必要な追加情報を送信するために使用されますが、電子メールで通信すると消えます。



通常、CRMシステムは企業のPBXと統合されます。 これは長くて複雑なプロセスです。「ああ、私はすべてを押しましたが、何も鳴らしていません」などの状況を修正するために、セーターにPBX自体、SIP電話、陰鬱な管理者が必要です。 呼び出しをWebインターフェースに直接統合することで、アクションの数を最小限に抑え、重要なことに焦点を当てることができます-顧客や同僚とのコミュニケーション、タスクの解決。



同様に重要な要素は、通話の記録です。 電話交換機（ある場合）を使用して簡単に記録できますが、記録にアクセスするための便利なインターフェイスは非常に貴重です。 多くの場合、さまざまな従業員が顧客に連絡するか、数か月前に最後に話したクライアントが休止状態から抜け出します。そのような場合、取引を行うために以前の電話を調べる便利なツールが重要になります。



VoxImplantとMegaplanを友達にした方法





プラットフォームの主な考え方は、2つのクライアント間ではなく、クライアントとクラウドの間で呼び出しが行われるということです。 そして待ち伏せのクラウドにはJavaScriptがあり、着信コールまたはAPIコールに関してまったく異なることを行います：コールのコミット、クラウドからのコール、音声の認識と合成、会話の記録、電話会議の提供、さらに100万の便利な機能を実行します。



ドキュメントを確認した後、MegaplanチームはVoxImplantクラウド用のJavaScriptスクリプトの開発を開始し、Web SDKを製品に統合しました。 HTTP APIを介してクラウドに接続する組織には多くの時間がかかりました。Megaplanの新規ユーザーと既存ユーザーの両方を自動的に追加および削除する必要がありました。 課金の統合を支援しました。HTTPAPIを介して、メインアカウントは子会社を作成し、その残高を処理します。



興味深いタスクは、サイトからの通話機能の導入でした。 当社のWeb SDKを使用すると、音声通話やビデオ通話を行ったり、テキストを送信したりできますが、インターフェースはなく、Webアプリケーションのデザイナーに合理的に提供します。 また、Megaplanの設計者と開発者は、ユーザーができるだけ快適になるように、既存のインターフェイスに呼び出しをシームレスに合わせる方法という困難な課題に直面しました。 この問題を解決するために、ユーザーシナリオのリストがコンパイルされ、頻度別にソートされました。 「メインタブ」のメカニズムが開発されました。音声コールは1つのタブのみに直接移動し、残りのタブはステータスを表示し、コントロールを転送します。



Web SDKはWebRTCとFlashの両方をサポートしているため、WebRTCがサポートされていない場合、サイトは互換モードを導入しました。 また、ブラウザでのマイクの使用を確認するためのヒントをユーザーに追加しました。



幸いなことに、JavaScriptスクリプト用のクラウドデバッガーがあり、コードをステップ実行して呼び出しの現在の状態を調べることができます。



結果として何が起こったのか









その結果、コールメカニズムを実装し、Megaplanに完全に統合することが判明しました。Megaplanはすぐに使用でき、複雑な設定を必要としません。 従業員は、テレフォニーユーザーに追加されるとすぐにクライアントと同僚に電話をかけることができます（現在の従業員に対しては自動的に行われ、新しい従業員に対しては確認が行われます）。 さまざまなパラメータでワンクリックで連絡先を検索します（着信した電話番号による自動クライアント検索）。 通話ログを操作し、会話の録音を聞きます。 さらに、従業員間のコミュニケーションは完全に無料です。



VoxImplantの主要な機能の1つ-呼び出しに単一の発信番号を設定する機能-は、Megaplanが共通の会社番号からすべての発信呼び出しを行うか、従業員に個々の番号を割り当てることを可能にします。



現在、Megaplanは、着信コールと自動処理シナリオとの統合に取り組んでおり、テレフォニーを積極的に開発する計画を立てています。 近い将来、現代のビジネスがインターフェイスや設定と戦うのではなく、問題の解決と顧客とのコミュニケーションに集中できるようになる新しい機会が生まれます。
