1つのインフラストラクチャの物語。 MSソリューション。 パート1

叙情的な紹介...



このトピックに関するいくつかのトピック（「 Microsoftが好きな理由。ゾンビノート 」と「 MS製品に基づくインフラストラクチャの構築 」）を読んだ後、私は自分のストーリー、経験、知識を共有することにしました...そしてIT責任者。 私自身の工場での成功した実装の経験、他社での大規模インフラストラクチャプロジェクトの設計と実装の経験があるため、最初は、ITスペシャリストにとって素晴らしい、有名なMSのインフラストラクチャソリューションを効果的に使用する方法に関する別の「サクセスストーリー」を書きたいと思いました、ユーザーおよびビジネス全体に対して。 しかし、最終的に、企業のITインフラストラクチャを構築する雰囲気を伝え、現在の状態に導いた要因を示し、重要な間違いについて話すことを試みることにしました。 もちろん、焦点を当てて、MSの決定を維持しようとします。



開始資本...



2003年に工場でチーフデザイナー（OGK）の部門で自動システム（AS）のエンジニアとして空いている職に就いた後、私は制御を始めました。工場のオフィスに散らばるクラス。

私が得たものの簡単な説明：

サーバーの1つは1999年からWindows NT 4.0で動作しており、プライマリドメインコントローラー（PDC）と「ファイルウォッシャー」であり、2つのプロセッサーとSCSIドライブさえも備えていました。 2番目はWindows Server 2000の下で2002年以降機能し、1Cアプリケーション用のターミナルサーバーでした。また、SCSIディスクと256MBのRAMの2つのプロセッサがあり、なんとか負荷に対処できました。 これらの2つの「重要な」サーバーには、10分程度続く古いUPSがありました。 3つ目は普通のPCで、Windows 2000 Professionalを実行していました。それは私の稼働中のコンピューター、インターネットゲートウェイ（WinRouteで編成）、メールサーバー（MDaemonで編成）で、しばしばハングアップし、再起動してADSLモデムを失いました。

ほとんどのユーザーのパーソナルコンピューターは、ドメインコントローラーの導入以来機能しており、Windows 98オペレーティングシステム（OS）を搭載したPentium Iクラスでした（上記のコンピューターはWindows 2000 Professionalに基づいていました）。

スイッチは異なり、10 / 100Mbitで、最大の累積はアカウンティングであり、フリーズして再起動が必要になる場合がありました。

言うまでもなく、すべてのソフトウェア（ソフトウェア）はライセンスされていません。 ただし、この経済の技術文書はクラスとして存在せず、IT管理の要件はほとんどありませんでした。 企業にはIT部門がなく（一般部門には特定の生産問題を解決するソフトウェア製品に携わるソフトウェアエンジニアがいました）、IT予算がなく、知識と経験がほとんどなかったことを明確にする必要があります...



このようなもの...多くの国内企業にとって典型的なITの状況。



適応...



しかし、私は幸運でした-あなたがそれに触れるまで、すべてが合理的にうまく機能しました 重大な事故はなかったので、理論的な基礎を研究し、効果的な解決策を探すために多くの時間を費やしました。 より良いものを理解しました。 また、私は直属の上司と非常に幸運でした-非常に技術的に有能な専門家は、本質を理解しようとし、IT状況を改善するための私のイニシアチブをサポートしようとしました そして、それゆえ、私は私に委ねられた方向の発展を取り上げました。

IT開発の戦略的タスクの1つ、またはむしろ管理者の希望は、工場管理と工場複合施設のショップ間のネットワーク切り替えを見ることでした。 これは、かなりの領土範囲と適切な割り当てられたリソースの不足のために困難な作業でした。 IT管​​理者のもう1つの願いは、グローバルネットワークを介してどこからでも電子メールにアクセスできるようにすることでした。 1Cアプリケーションのユーザー（およびこれらは主に多数の経理スタッフ）の希望から、許容できる作業生産性を確認することができました。 情報システム（IS）の残りのユーザーは、インターネットリソースへの中断のないアクセスのみを望んでいました。

問題を解決するにつれて、必要な結論に至りました。

  • 工場複合施設の建物間の光ファイバー通信回線（FOCL）
  • サーバーおよびスイッチング機器の集中化（既存およびラックの交換）
  • Windows Server 2000レベルドメインの実装（2つのドメインコントローラー（CD））
  • 1Cデータベースを操作するためのSQLサーバーの実装
  • ネイティブクライアントを優先するターミナルサーバーの放棄（PCの交換）


私は、IT開発の問題に関するすべての考え（拡張された形で、表示コストを含む）をメモの形で作成し、CEOに提出するというルールがありました。



「大」実装...



そして今、私の頭の上の雪のように、この緩慢な無活動の1年半後、私たちは、1つの有名なERPシステムを導入しているというニュースが届きました！ この問題は最高レベルで解決されており、説明されていません。 実装のためのインフラストラクチャの技術的な準備、つまり 工場周辺の光ファイバー通信回線の編成、スイッチングおよびサーバー機器の準備（もちろん、これはすべて最も厳しい節約モードです）。

このプロジェクトは合意され、開始され、作業が始まった。 スイッチングおよびコンピューティングセンター（サーバー）の組織に部屋が割り当てられました。 機器が選択および注文されました-サーバー、ラック、スイッチ、光ファイバーケーブルセグメント（FOC）（必要な長さ、はんだ付けコネクタ付き）、コンバーター、UPS。

機器を備えたラックが設置され、通信回線がサーバーに転送されました。 Windows Server 2000（2枚のCD）に基づくドメインが展開され、SQL Server 2000とERPシステム用のアプリケーションサーバーが他の2つのサーバーに展開されました。 企業の通信エンジニアの力により、FOCセグメントの設置が完了しました。 すべてがテストされ、機能します。 最初のパンケーキがゴツゴツしておらず、6か月の仕事が無駄にならなかったことは奇妙です...



実装チームがERPシステムを展開するのを待っています。 並行して、私はインフラストラクチャのさらなる開発の問題に取り組んでおり、私たちが必要とする結論に達しました：

  • ISA Server 2004に基づくインターネットゲートウェイの構成
  • Exchange Server 2003に基づくメールサーバーの組織


いつものように、プロジェクトは合意され、立ち上げられ、作業は沸騰し始めました。 2台のサーバー、UPSを購入しました。 すべてがWindows Server 2003に基づいて展開され、テストおよび起動されます。 管理のもう1つの戦略目標であるメールボックスリソースへのユニバーサルアクセスが実装されています。 誰もが新しい「グッズ」、特に現在の連絡先リストに満足しています。 2番目のパンケーキも、ゴツゴツしたものではなく、さらに6か月の作業から遠く離れています...これらのプロジェクトと並行して、多くの「技術的に不適切な」PCが置き換えられました。 個人用PCのプラットフォームとして、Windows XP Professionalを選択しました。



再びリラックスしました。展開チームがERPシステムを展開するのをもう待っていません。 経営陣は、実装プロジェクトが無期限に延期され、資金提供が停止され、アイドル状態のサーバーが許可されると述べました。

仕事は沸騰し始めました。 データベース「1C」はSQL Server 2000に転送されます。ユーザーは最終的に「1C」のネイティブバージョンの使用に転送されます。 アプリケーションサーバーから、ディスクサブシステムをいくつか修正した後、ファイルストレージサーバーを取得しました。 以前の「ファイルウォッシュ」は、アクセス制御を備えた構造化ストレージに変わりました。 最終的には、誰もが幸せであり、生産性が向上しました。



私はインフラストラクチャのさらなる開発の問題に取り組んでいます、私は私たちが必要とする結論に達します：

  • Exchange Server 2003に基づくフロントエンドメールサーバーの実装と境界ネットワークでの展開
  • Live Communications Server 2005に基づくインスタントメッセージングの実装


いつものように、プロジェクトは合意され、立ち上げられ、作業は沸騰し始めました。 2台のサーバー、新しいスイッチング機器（既存のサーバーはVLANをサポートしていなかったため）、UPSを購入しました。 すべてがWindows Server 2003に基づいて展開され、テストおよび起動されます。 ユーザーは、Office Communicator 2005の使用に関するトレーニングを受けており、非常に満足しています。 スパムフィルタリングの負荷のかなりの部分がExchangeバックエンドサーバーから削除されました。

古い機器はすべて廃止され、特別な部屋の隅に置かれます。



最初の結果の評価...



私たちは持っているものを使って仕事をします。 ある晴れた冬の日、VOKの幹線道路が故障しました。調査の結果、障害は未熟練のケーブル敷設であり、ケーブルが建物の角に擦り付けられていたことがわかりました。 コミュニケーションを回復する緊急の必要性。 霜がひどいため、光学系の設置は困難です。 何かしなければなりません。 サプライヤーは、中華鍋を取り付けることが可能になるまで、しばらくの間、Wifi 802.11g機器を使用して一時的な（バックアップ）チャネルを迅速に編成することを提案しました。 見積もり、カウント、および同意。 高価な「プロフェッショナル」アクセスポイントとストリートデザインの指向性アンテナを2つ注文しました（すべての機器は有名なメーカーのものです）。 すぐに到着し、すばやくマウントされ、ブリッジモードで起動されます。 しかし、私たちが望むほど速くはありませんでした。 そして、運用中に判明したように、それはそれほど信頼できません-毎日、何度も接続が消え、しばらくしてから復元されました。 このようなバックアップチャネルを3か月間使用し、その後、中華鍋に置き換えられました。 無線ポイントは解体され、クライアントポイントとしてインストールされました-今日まで問題なく動作しています。

今後は、ERPシステムを実装していないと言いますが、企業の経済状況が大幅に改善された場合、新しいバージョンを導入する可能性があります。 そして今、これらのマイルストーンの下で要約すると、設計中にどのような間違いが行われたか言うことができます。

  • スイッチング機器は統一されておらず、ほとんどの場合管理されていません
  • FOCLメディアコンバーターは、制御されたシャーシに編成されていませんでした
  • 周辺のスイッチングポイントのスイッチング機器は、キャビネット/ラックに整理されていませんでした
  • バックボーン光ファイバーリンクの冗長性はありませんでした
  • サーバー機器の半分、サーバーの場合のみ通常のPCとは異なります
  • サーバーおよびスイッチング機器用のUPSは、わずか30分のバッテリー寿命を提供しました
  • サーバーハードウェア使用SATAハードドライブ
  • サーバーハードウェアには、ハードドライブを「ホット」交換できるバスケットはありませんでした
  • サーバーハードウェアでは、ネットワークアダプターを組み合わせる可能性はありませんでした
  • サーバーハードウェアにディスクコントローラー用のバッテリーがありませんでした
  • サーバー機器には、電源とその「ホット」交換の冗長性はありませんでした
  • サーバールームには冷却がありませんでした（そのため、ハードドライブを頻繁に変更する必要がありました）。
  • Windows Server 2000はCDのプラットフォームとして選ばれました
  • Windows Server 2000は、SQLデータベースサーバーのプラットフォームとして選択されました。
  • 他のサーバーの場合、EnterpriseエディションのWindows Server 2003がプラットフォームとして選択されました（Enterpriseが理由で）
  • 内部ドメインの名前は外部に対応し、外部はインターネットアクセスサービスプロバイダーによって無料で提供される第3レベルでした
  • マルウェアに対する適切な集中型保護がありませんでした（最も一般的なウイルス対策が使用され、ライセンスキーと更新に常に問題がありました）


効果的ではない...これらは、企業内のIT資金調達慣行の欠如の結果です。 私のリーダーシップの方向性にお金を割り当てることは、新しい、珍しい、普通ではない何かでした。 これは今、割り当てられたリソースが「あるべきように、すべてを正しく行う」ことを許可していない場合、プロジェクトを開始することさえしません。

2番目の要因は、もちろん、IT担当者（つまり、私）の間に必要な能力の欠如であり、その結果、最終結果の明確なビジョンがなく、プロジェクト計画の許容レベルもありませんでした（「始めましょう」）。 実装されたインフラストラクチャサービスは、私の経験の結果でした。 しかし、インフラストラクチャを整理するためにMSソフトウェアを使用することの有効性に関して、重大な逸脱はなかったことに注意したいと思います。 それが起こったので、MS製品はよく慣れました。IT専門家もIPユーザーも、それらについて誤解や意見の相違はありませんでした。 経営陣も非常に役立っており、主な逸脱を認識していました。

しばらくの間、インフラストラクチャはこの状態にあったと言いますが、バグのさらなる作業によりすべてが修正されました。
