Windows 10用Windowsアセスメント＆デプロイメントキット（ADK）RCを使用したNano Serverの展開

この記事は、MVPのMikhail KomarovによってHyper-Vの方向に書かれました 。




この記事の目的は、ADKを使用してMicrosoft Nano Serverを展開する機能を実証することです。これにより、大規模展開のための標準のWindows展開サービスの使用がさらに可能になります。 すべてのアクションは、Windows 8.1以降のオペレーティングシステムで実行できます。 Hyper-Vに展開する場合は、Hyper-Vの役割を有効にするか、実際のコンピューターを使用する必要があります。 この記事では、Hyper-Vの役割が有効になっているWindows 8.1とHP Microserver Gen 8を使用しました。



2015年5月、Microsoft Igniteカンファレンスで革新的な製品であるMicrosoft Nano Serverが紹介されました。 この製品は、最小限のコードサイズと必要な機能のみを備えた、完全に再設計されたバージョンのWindowsです。 サーバーの役割と追加機能は、Nano Serverの外部にあります。 スタンドアロンパッケージはアプリケーションとしてインストールされます。 製品の主要な役割と機能は、Hyper-V、ストレージシステム（SoFS）、クラスタリング、およびCLRカーネル、ASP.NET 5、PaaSのサポートです。 パッケージがインストールされた配布パッケージのサイズは1GBを超えず、SDカードに配置できます。



展開プロセスはいくつかのステップで構成されます。

最初のステップから始めましょう。 Windows Server Technical Preview 2の配布が必要です。リンクは記事の最後にあります。 配布キットをISOファイルとしてダウンロードした後、それを開き、ルートからNanoServerディレクトリを抽出する必要があります。







次に、Windows 10用のWindowsアセスメント＆デプロイメントキット（ADK）RCをダウンロードする必要があります。製品は起動時に小さな実行可能ファイルとしてダウンロードされ、必要なコンポーネントを選択するか、ダウンロードを完了してインストールを開始できます。 この作業にパッケージ全体は必要ありません。展開のオプションのみを選択します。







このパッケージをインストールした後、管理者として「展開およびイメージングツール環境」ショートカットを実行します







コマンド「 copype amd64 W：\ WinPE_64 」を実行します。 Copypeは Windowsコマンドではなく、ADKで提供されるスクリプトです。 このコマンドの意味は、Windowsプレインストール環境ファイルをW：\ WinPE_64にコピーすることです。 追加データ用のマシンでは、Wディスクを使用します。あなたの場合は、任意のディスクとディレクトリを使用できます。







Windowsプレインストール環境（Windows PE）ファイルを使用してディレクトリを作成したら、最初の手順は完了です。



2番目のステップでは、Nano Serverからマシンを起動する方法を理解する必要があります。 Nano Serverをゼロから展開するため、これは重要です。 そのため、マシンをロードするには、クラシックBIOSまたは最新のUEFIを使用する2つのオプションがあります。 Hyper-Vには、仮想マシンのタイプに依存する2つのオプションもあります。 以下は、Hyper-Vの例です。

3番目のステップは、インストール後に実行される応答ファイルとスクリプトを作成することです。 サンプルのunattend.xml応答ファイルは、「 Nano Server入門 」からのものです。 ComputerName 、 Value 、 RegisteredOwner 、 RegisteredOrganizationタグの値を変更できます。 



 <ComputerName>NanoServer1503</ComputerName> <AdministratorPassword> <Value>P@ssw0rd</Value> <PlainText>true</PlainText> </AdministratorPassword> <RegisteredOwner>Mikhail Komarov</RegisteredOwner> <RegisteredOrganization>Virtual Lab</RegisteredOrganization>


インストール後のスクリプトについて-最初にディレクトリを作成する必要があります

md W：\ Windows \ Setup 、次の内容のSetupComplete.cmdファイルをそこに配置します。 



 ping 127.0.0.1 -n 4 hostname ipconfig netsh advfirewall set domainprofile state off netsh advfirewall set privateprofile state off netsh advfirewall set publicprofile state off netsh advfirewall set currentprofile state off


その内容から、DHCPを介してIPアドレスを受信し、ファイアウォールをオフにすることは明らかです。



4番目の手順は、イメージを展開し、パッケージを追加し、応答ファイルを適用する結果のスクリプトを作成することです。 そのため、2つのファイルを作成して、それらの名前を明確にします： Apply_on_Hardware.batおよびApply_on_Hyper-V_VM.bat 。 これらの2つのファイルには共通部分があります。



イメージをディスクに展開します。 



 dism /Apply-Image /ImageFile:d:\NanoServer\nanoserver.wim /Index:1 /ApplyDir:W:\


パッケージを追加します（ファイルには別の構成があります）。 

 dism /Add-Package


応答ファイルを追加します。 

 dism /Apply-Unattend:d:\NanoServer\unattend/image:W:\


追加ログ用のスペース。 

 md W:\Windows\panther


応答ファイルをコピーします。 

 copy d:\NanoServer\unattend.xml W:\Windows\panther\


インストール後にスクリプトを実行するディレクトリを作成します。 

 md W:\Windows\Setup


応答ファイルをコピーします。 

 copy d:\NanoServer\SetupComplete.cmd W:\Windows\Setup\Scripts\


ブートレコードを作成します。 

 bcdboot W:\Windows

5番目のステップでは、 Windows PEの起動方法と、ファイルとNano Serverファイルの場所を決定する必要があります。 たとえば、起動可能なUSBステッカーを作成してそこからダウンロードしてインストールする方法や、たとえば、起動可能なISOを作成し、Nano ServerファイルをUSBに入れて実際のコンピューターにインストールする方法がいくつかあります。 または、Windows PEおよびNano ServerファイルがあるISOイメージを作成します。 このようにします。 NanoServerディレクトリをW：\ WinPE_64 \ media \にコピーします。 その後、MakeWinPEMedia / ISO W：\ WinPE_64 W：\ WinPE_64.isoコマンドを使用して ISOイメージを作成します。MakeWinPEMediaはADKのスクリプトであり、キーを変更すると、起動可能なUSBステッカーも作成されます。







この手順の最後に、Windows PEが準備され、Nano Serverファイルが配置されているWinPE_64.isoイメージファイルを取得します。



6番目のステップは、新しく作成された仮想マシンまたは物理コンピューターのイメージから起動することです。その後、コマンドラインに移動し、 d：\ nanoServerドライブのディレクトリに移動する必要があります。 ドライブDが空であるか、ディレクトリが見つからなかった場合は、次を試してください。

注意!!! Windows PEコマンドラインでのみ実行します。 



 Diskpart
（セクションエディターと呼びます） 

 Select disk 0
（0ドライブを選択） 

 Clean
（このディスクからすべてのパーティションを削除します） 

 Exit
（セクションエディターを終了します） 

 exit
（このコマンドの後、コンピューターが再起動します）



次に、 d：\ nanoserverディレクトリに変更してみてください。 次に、 diskpart / s Bios_Partition_Structure.txtまたはdiskpart / s UEFI_Partition_Structure.txtを実行します。 パーティションを作成するチームの選択は、ダウンロードのタイプによって決まります。 2番目のセクションの最後で、 Bios_Partition_Structure.txtおよびUEFI_Partition_Structure.txtの内容について説明しました。



パーティションを作成したら 、マシンのタイプに応じてApply_on_Hardware_Bios.batまたはApply_on_Hyper-V_VM.batスクリプトを実行します。 スクリプトが終了すると、マシンは再起動します。 再起動後、画面に長時間何も表示されず、とにかく、画面からの制御が利用できなくなります。



最後の7番目のステップは、Nano Serverへの接続です。 アドレスを提供してくれたDHCPサーバーを見てみましょう。 ローカルマシンで管理者としてPowerShellを実行します。 



 $ip = "192.168.10.105" $user = "Administrator" Set-Item WSMAN:\\localhost\client\TrustedHosts $ip -Force chcp 65001 Enter-PSSession -ComputerName $ip -Credential $user


パスワードを入力し、コマンドレットget-processを実行します
