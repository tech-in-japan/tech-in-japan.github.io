サーバーにハードドライブを選択する方法





IT分野には多くの神​​話があります。 「スパムの登録を解除できます」、「2つのウイルス対策は1つより優れています」、「サーバーのハードドライブはブランド化するだけです。」 鉄道車両を交換および拡張する際には、多くのニュアンスと微妙な点を考慮する必要があり、偏見がないわけではありませんでした。 サーバーのハードドライブ、それらの違い、注意する必要があるもの、サーバーメーカーのロゴと一緒にする必要があるかどうか-猫の下でこれについて読んでください。



ディスクがサーバーにインストールされている場合、次の厳しい要件を満たす必要があります。



  • 信頼性 。 回復不能なデータ損失は、数百万ドルの損失と評判の損失につながる可能性があります。
  • パフォーマンス 。 サーバーは、多数のリクエストを処理するように設計されたアプリオリです。
  • 応答時間 。 ユーザーは、サーバーディスクが「起動」して要求を処理するまで待つ必要はありません。


言い換えれば、サーバーのハードディスクは先駆者のようなものでなければなりません-常に最小限の遅延で多数のリクエストを処理する準備ができており、高レベルのデータセキュリティを提供します。 負荷の高いサーバーでは、ハードドライブが長年にわたって集中的に稼働し続けています。



4つの主なカテゴリ（SSD、SAS SSD、PCI-e SSDは考慮していません）のハードドライブがあります。



  • SATA（通常、「家庭用」SATA）-5400および7200 rpmのスピンドル速度。
  • SATA RAID Edition（SATA RE）-7200 rpmのスピンドル速度、RAIDコントローラーコマンドのサポート。
  • SASニアライン（SAS NL）-スピンドル速度7200 rpm。
  • SAS Enterprise-10,000または15,000 rpmのスピンドル速度。


まず、接続インターフェース（SATAまたはSAS）を決定する必要があります。



SATAまたはSAS？







SATAインターフェースはIDEの開発であり、後にPATAと改名されました。 つまり、このインターフェイスは、最初は家庭用コンピューターでの使用と、中程度のパフォーマンスと信頼性の要件を持つ産業用システムでの使用に焦点を合わせています。 同時に、 SASは従来の「サーバー」 SCSIインターフェイスの後継です。



当初、SASインターフェイスはSATAよりも高い帯域幅を備えていました。 しかし、進歩は止まっておらず、第3世代SATA IIIの最大スループットは6 Gb / sであり、第2世代のSASも同様です。 ただし、第3世代のSASコントローラーを搭載したサーバーは、最大12 Gb / sのスループットですでに市場で入手可能です。



SASディスクを接続するには、サーバーに適切なコントローラーが装備されている必要があります。 同時に、インターフェイスの下位互換性が確保されます。SATAディスクはSASコントローラーに接続できますが、その逆も同様です。不可能です。



SASは全二重データ交換を提供します。ハードドライブは一度に1つの読み取りおよび書き込みコマンドを処理し、SATAドライブは読み取りまたは書き込みのいずれかを行います。 ただし、SAS NLとSATA REを比較すると、この利点は多数のドライブでのみ顕著になります。



要約すると、SATAドライブは最大のパフォーマンスを必要としない大容量ストレージの作成に適しています。 また、ディスクサブシステムから可能なすべてのものを絞り出す必要がある場合、選択はSASです。



通常のハードドライブにノーと言う







すぐに明確にします-通常のSATAはサーバーでの使用を目的としていません。 これにはいくつかの理由があります。



  • 耐振動性が低い。
  • 高レベルの回復不能エラー。
  • ハードウェアRAIDコントローラーコマンドのサポートの欠如。


もちろん、従来のデスクトップSATAのコストはサーバーSATAのコストよりも大幅に低く、高性能ディスクサブシステムを必要としない小さなタスクに使用することを妨げるものはありません。 データへのアクセスの安全性と速度がそもそもある場合は、サーバー側のドライブを使用することを強くお勧めします。



耐振動性



多かれ少なかれ要求の厳しいタスクを解決するために、1つのディスクのみを置くことは意味がありません。 データストレージの最小レベルの信頼性を確保するには、少なくとも2台のドライブをRAIDで結合する必要があります。 しかし、4つ以上のデバイスがバスケットに組み込まれている場合、それらの動作から生じる振動は、スピンドル回転の安定性とヘッドの位置決めの精度に影響します。 したがって、サーバーのハードドライブには、家庭用のものとは多くの設計上の違いがあります。



  • 強化されたスピンドルシャフトは、外部の影響に対してより耐性があります。
  • 追加の振動制御。
  • 「パンケーキ」の表面上のヘッドの位置決めと飛行高さの精度を大幅に向上させる技術。
  • すぐにドライブの差し迫った障害を通知できる豊富な自己診断機能。


家庭での運転にはこれがすべてありません。 振動レベルが十分に強い場合、従来のSATAでの読み取り/書き込みエラーの確率は、SATA REよりも50％高くなります 。



致命的なエラーレベル



サーバーのハードドライブと家庭用ハードドライブの次の違いは、回復不能なエラーのレベルです。 通常のSATAの場合、約10 -14 （読み取り10 14ビットごとに1ビット= 12.5テラバイト）です。 つまり、2テラバイトのディスクを6倍に書き換えると、ほぼ確実に1つの回復不能なエラーが発生します。 家庭用ドライブの場合、これは問題ではありません。 しかし、毎月データベースを書き換えると、6か月後には少し時間がかかることがあります。



=

(N * (X / 12500 * 12500) / 12500) * 100%







どこで：



  • Nは、RAIDアレイ内のディスクの数です。
  • Xはギガバイト単位の1つのディスクのボリュームです。
  • 12500-ギガバイトで表された、修復不可能なエラー1つあたりのビット数。


1テラバイトの通常のSATAのRAID 5アレイを作成したとします。 アレイを再構築すると、40％の確率で回復不能なエラーを受け取ります。



=

(5 * (1000 / 12500 * 12500) / 12500) * 100% = 40%.







また、600ギガバイトのディスクを使用する場合、再構築でエピックが失敗する確率は24％です。



=

(5 * (600 / 12500 * 12500) / 12500) * 100% = 24%.







SATA REおよびSAS NLの場合、回復不能なエラーのレベルは10 -15です 。つまり、従来のSATAよりも1桁小さくなっています。 次に、RAID 5の例では次のようになります。



1テラバイトのドライブの場合、エラーの確率= 4％。

600ギガバイトのドライブの場合、エラーの確率は2.4％です。



SASディスクのエラー率はさらに低くなります-10-16



1テラバイトのドライブの場合、エラーの確率は0.4％です。

600ギガバイトのドライブの場合、エラーの確率は0.24％です。



注 ：エラーの確率は、RAIDアレイ内のディスクの数に比例します。







SASドライブはどのようにしてエラー率を下げるのですか？ 魔法



  • SATAディスクのセクターサイズは512バイト、SASディスクの場合は520バイトです。 追加の8バイトがパススルーパリティに使用されます。
  • その他の読み取りアルゴリズム。
  • コントローラーの関与なしの追加のデータ回復アルゴリズム。


RAIDアレイで作業する



従来のSATAのもう1つの重要な欠点は、RAIDアレイでの作業時のエラー修正の欠如です。 バックアップを望み、経済性のために、通常のSATAからRAIDを構築したとしましょう。 エラーが発生した場合、ハードドライブは不良ブロックの読み取りを繰り返し試行します。 そして彼はこれをしている間、彼はRAIDコントローラーの信号に応答しません。 彼はこれをハードディスクの故障と認識し、アレイから除外して、回復を試みます。 つまり、エラーが発生すると、ディスク全体がアレイからドロップアウトします。



SATA RE、SAS NL、およびSASの場合、状況はそれぞれ異なります。 エラーを検出すると、ディスクはコントローラに不良ブロックの存在を通知します。 コントローラーは、アレイ内の他のディスクからこのブロックを要求し、故障したディスクに転送します。 同時に、デバイスはアレイから脱落せず、パフォーマンスも低下しません。



ブランドの神話



最後に、最も重要な質問：「ネイティブ」ドライブを購入する必要がありますか？



HP、IBM、DELLがハードドライブを製造していないことは秘密ではありません。 サードパーティのメーカーから購入し、その後、ロゴをテスト、再フラッシュ、接着します。







一方では、そのようなディスクにはいくつかの利点があります。



  • ファームウェア（ファームウェア）は、特定のサーバーモデルのコントローラーの機能を考慮します。
  • 追加の品質管理とストレステストにより、隠れた欠陥のある標本を取得する可能性が低くなり、
  • 「ブランド」ディスクには、ベンダー保証と完全なサポートが提供されます。


しかし、すべての良いものを支払う必要があります。「ネイティブ」ハードドライブは、同じモデルの約2〜3倍高価ですが、メーカーのロゴ（Seagate、Western Digital、Toshiba、HGST）が付いています。



ご存知のように、このような価格の違いは、信頼性の向上という約束によって正当化されるにはほど遠いです。 そのため、多くの人は、非ネイティブのハードドライブがHP、IBM、およびDELLサーバーで不安定であるか、動作が遅いと聞いているでしょう。 誰かが、サーバーが「左」のディスクで起動しないことを恐れています。



これらの声明はどこから来たのですか？



大多数のサーバーはテクノロジーを使用して、ディスクサブシステムのパフォーマンスを向上させます。 この目的のために、ベンダーはハードドライブを再フラッシュします-これらの技術のサポートを提供します。 「非ネイティブ」ディスクを入れた場合、ブランドノウハウを使用することはできません。



また、以前は、ベンダーはコントローラーレベルでロックを使用して、「ブランド」ドライブの使用を人為的に強制していました。 最終的に、大衆の怒りは、この悪質な慣習を放棄することを時間をかけて強要しました。 今日、ほとんどの場合、問題は比較的古いサーバーモデルで発生します。 これは、コントローラーに新しいファームウェアを入力するだけで解決します。 HPサーバーのP410など、単純に気まぐれなコントローラーモデルがありますが。



実践が示すように、非ネイティブハードドライブは問題なく動作します。



  • HPサーバー-少なくともGen6世代から
  • IBMサーバー-少なくともM2世代以降、
  • DELLサーバーで-少なくとも10世代から。


非ネイティブハードディスクは、サーバーレールと内部監視システムの両方と完全に互換性があります。 ただし、スレッドは親relativeのみが設定する必要があり、特定の世代のサーバーに対してのみ設定する必要があります。 ベンダーが自社ブランドで販売しているディスクモデルの実際の名前をネット上で簡単に見つけることができます。 たとえば、ほとんどのHP SASドライブは、Seagate Savvioのハードドライブシリーズで作られています。 同時に、完全なアナログを探す必要はまったくありません。一般的な定規から適切なモデルを選択できます。

保証の確認



ベンダーの計画に従わず、非ネイティブハードドライブを購入する場合は、最初に売り手に確認してください。誰が保証を提供しますか？ 事実、多くの店はメーカーの保証を参照して、ハードドライブの保証を提供していません。 ただし、微妙な点があります。たとえば、 一部の Seagateモデルでは、保証期間は生産の瞬間から始まります。 したがって、メーカーの保証が既に終了している完全に新しいディスクを購入するという状況は除外されません。



悲しいですが、本当です。



この厄介な瞬間を経験しないように、購入する前にメーカーのウェブサイトで特定のアイテムの保証を確認してください。

おわりに



ハードドライブを選択する場合、最初にサーバーが実行するタスクから開始する必要があります。



  • 高速なアクセス速度とデータストレージの信頼性が不要で、ディスクの数が4を超えない場合は、SATA RAID Editionディスクをインストールすることをお勧めします。 これは、少数のユーザーにサービスを提供する低コストのエントリレベルサーバーのオプションです。
  • サーバーがデータベースを提供する場合、またはアレイ内のディスクの数が5以上の場合、SAS NLを選択することをお勧めします。 ほとんどの場合、このようなディスクは、会計システム、CMS、企業リポジトリなどの中規模企業で動作するサーバーに配置されます。
  • また、金融取引の処理など、データストレージの最大のパフォーマンスや信頼性が必要な場合は、SAS Enterpriseドライブを選択します。 これらは、多数のユーザーにサービスを提供する負荷の高いサーバー、および最も重要なデータを処理するシステムのメディアです。


しかし、最も重要なこと-神話を信じないでください。 サーバーと同じロゴのディスクを購入する必要はまったくありません。 有能なアプローチにより、信頼性と速度を失うことなく、ディスクサブシステムのアップグレードを大幅に節約できます。
