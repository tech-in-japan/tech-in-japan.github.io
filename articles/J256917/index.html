新しいMicrosoft Azure管理ポータルの概要-Azureプレビュー

最近、マイクロソフトはWindows Azureクラウドサービスのブランドを変更し、現在はMicrosoft Azureと名付けています。 これは、AzureでWindowsだけでなくLinuxも使用できることをユーザーに示すために行われました。 はい。Microsoftは現在、Linuxとオープンソースも愛しています。 しかし、今はそれについて話していません...



新しい名前で、Microsoft Azure Preview Portalという名前の新しいクラウドインフラストラクチャ管理ポータルもプレビューとして表示されました。 それへのリンク： ここを突く。







お気づきかもしれませんが、インターフェイスはタイルに基づいて構築されています。 あらゆる情報をタイルとして取り出して、すぐにアクセスできるのは本当に便利です。



新しい仮想マシンまたはその他のazureサービスを作成するには、左下隅にある「+」記号をクリックする必要があります。azure内で展開できるサービスのリストがパネルの右側に表示されます。







ここで、テストバーチャルマシンを作成して、新しいポータルのインターフェイスをさらに実証してみましょう。



そのため、仮想マシンの作成を選択すると、展開する仮想マシンのイメージを選択できる新しいセクションが表示されます。







デモンストレーションのために、Windows Server 2012 R2 Datacenterを選択します。 次に、仮想マシンの目的のパラメーターを示す別のメニューがあります。 また、VMを作成するときに、チューニングに使用できるパラメーターが増えたことにも注意してください。







[ 作成 ]ボタンをクリックすると、自動展開が開始されます。 また、仮想マシンのタイルはすぐにポータルのメイン画面に配置されます。







仮想マシンの作成が完了したら、このタイルをクリックして、VMに関するすべての情報を取得できます。 VMに関する情報には、パネルを離れずに、リソースの消費に関する統計を金額で表示する機能と、予測による変更のスケジュールが表示されます。















ポータルとの対話を改善するために、ホットキーを使用する機能が最近追加されました。



要約：Azure管理ポータルの新しいバージョンは、使用されているリソースなどに関する情報を使用し、迅速に受信するためにはるかに便利になりました。 これは私の個人的な意見であり、絶対に正しいというタイトルを主張するものではありません 。
