AWS：リージョン間でAMIイメージをコピーできるようになりました

ハレルヤ、仲間、それは起こった！



少し前まで、AWS はリージョン間でスナップショットをコピーする機能を導入しました。 次のステップが取られたことは非常に論理的です-AMIをコピーします。



[AMI]セクションのコンソールの[コピー]コマンドが表示されました。





次に、地域と画像名を選択するよう求められます：





また、CLIを更新した後： 

$ pip install --upgrade awscli


コマンドが登場しました： 

 $ aws ec2 copy-image --source-image-id ami-e251c28b --source-region us-east-1 --name Ubuntu1004 --description CopyOfUbuntu --region us-west-2 49d9872e-a5f8-4062-b9e3-44e7503769df ami-ce1d88fe


この新しい関数は私たちに何を約束し、それなしでどのようにしたのでしょうか？



イメージをコピーする機能は、DRおよびリージョン間で分散されたインフラストラクチャに必要です。 以前は、AMIを地域間で駆動するカスタムスクリプトがありました。 そして今では、数桁も簡単になりました。



親愛なるAWSユーザー、この新しい機能を使用すると簡単になりますか？
