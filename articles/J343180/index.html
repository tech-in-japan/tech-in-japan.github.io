Lego Serious Play Technique：子デザイナーを使用してチームの価値を定式化して使用する方法

快適なゾーンに留まると問題が発生します



かつて、5つのSCRUM値を持つプランニングポーカー用のカードを手に持っていたとき、私たちのチームの特定の値が「沈んでいる」ことに気付きました。 チーム内では、彼らを公言していますが。



  1. 時々、お客様の声を聞くのをやめ、タスクに深く入り込まず、お客様の問題を深いレベルで解決しようとしないことがあります。 これは、私たちが長い間責任のある分野で働いてきたという事実によるものです。 そして、私たちの注意の焦点は、私たちの直接の仕事の輪に徐々に狭くなっています。 私たちは、周囲の人々の問題に注意を払うのをやめるだけです。
  2. 上記の段落で特定された問題により、チームメンバは、何かを長期間理解できない場合、同僚に鋭く応答することができます。 この態度は開放性を刺激するものではありません。次回は、もう一度助けを求めるよりも数時間余分に過ごす方が良いでしょう。
  3. 同じ責任分野での長い仕事は、固定観念的な思考につながります。 場合によっては、快適ゾーンを離れる恐れがあるため、「ギザギザのある」経路に沿って問題を解決し、問題をより深く調査するために、何か新しいものや平凡な怠inessにさえ遭遇します。
  4. 個々のチームメンバーは、内向的な性格のため、知識の共有に対して十分にオープンではありません。


その結果、私たちはタスクにより多くの時間を費やします。表面的な研究や誰かが知識を共有しなかったことが原因で、アーキテクチャに問題が生じることがあります。

主な理由は、チームと私が3〜4年間一緒にいたことです。 一方で、これにより、1つのスタイルでコードを記述し、1つのスタイルでアプリケーションアーキテクチャを構築できます。これにより、プロジェクトのメンテナンスに費やす時間を大幅に削減できます。 一方、そのような「あいまいな」見解は、顧客やお互いとの開かれた敬意のある関係には寄与しません。 「自分を揺さぶる」時です。



チームがSCRUM値に置くポイントは何ですか



  1. 勇気。 決断を下す勇気。 目標の達成に責任を持つ勇気。 新しい技術を導入する勇気。 起こりうる過ちを受け入れ、その解消に取り組む勇気。
  2. フォーカス スプリントプロセスでは、チームは特定のタスクに集中します。 スプリントの目標を時間通りに達成できるようにするのが焦点です。
  3. 献身 チームのメンバーは、共通の原因に、お互いに専念しています。 そして、熱意と関心を持ってビジネスを行うために、コミットメントを行い、責任感を感じることができるのは忠誠心です。
  4. 尊敬する 私たちはお互いを尊重し、私たちの違いを尊重します。 意見の多元性により、最善の解決策を見つけることができるため、異なる意見を尊重します。 補完的なチームを構築できるのは、両者の違いです。 チームのすべてのメンバーが何かを知らず、知識のギャップを埋める準備ができているという事実を尊重します。最終的にはチーム全体がこの恩恵を受けるからです。
  5. 開放性。 チーム内外の情報は公開されています。 そして誰もがそれにアクセスできます。 開放性により、チームの行動をタイムリーに調整できます。 私たちは新しい知識と新しい意見を受け入れています。 批判のおかげで、私たちは良くなることができるので、私たちは批判に対してオープンです。




状況を改善するために、私たちは別のmitapを開催し、何が私たちを団結させたかをまとめて思い出すことにしました。 同時に、私は2つの方法を使用しました。チームの主な価値を特定する自己決定ゲームと、それらを正しく使用する方法を理解するためのLego Serious Playです。 私の意見では、タスクを解決できたので、他のチームに方法論をお勧めします。



自己決定ゲームを実施して、チームの価値を特定します



SCRUMのコアバリューを響かせます



勇気、集中、献身、寛容さと敬意。 彼らがチームに与えることができるものを教えてください。



チームにとって重要な値を選択してください



各人は、SCRUM値を独自に理解できます。 さらに、彼の仕事を導く完全に異なるものが彼にとって重要かもしれません。



したがって、5つのSCRUM値を強制するのではなく、誰もが本当に重要なものを選択できるようにする方がより正確です。 そのようなリストを作成し、誰にとっても重要な最も重要な値を選択することを提案しました。











その結果、チームは（5つではなく）6つの値のリストを作成しました。 それらはそれぞれ、チーム全体にとって重要です。



  1. 柔軟性。
  2. 熱意。
  3. 開放性。
  4. 職人技。
  5. 尊敬する
  6. 責任


レゴモデルで価値の理解を深める



自己決定のゲームの後、論理的な疑問が生じました。お互いや他のチームと協力して価値をどのように使うか？ それに答えるために、Lego Serious Play（LSP）テクニックの使用を提案しました。



方法論の主な利点は、ゲームの各メンバーが100％関与することです。 LSPは、レゴブロックで楽しい時間を過ごすことではありません（ただし、これはプロセスに伴います）。

あなたは主な目標を持ってゲームに来ます-問題の解決策を見つけること、そしてキューブはあなたの脳活動を「解明」するように設計された単なるツールです。



LSPは新しい知識をどのように刺激しますか？ 人はしばしば自分が知っていることを理解しません。 私たちの記憶とそこから情報を抽出するプロセスは複雑です。 キューブは、このプロセスを刺激し、既存の経験に基づいて新しい知識を生成するのに役立ちます。



この「魔法」の式は次のとおりです。



感情は注意を促します>注意は学習を促します>学習は記憶を促します。



LSPテクニックにはいくつかのステップが含まれます



  1. ゲームのルールと回答が必要な質問を明記します。
  2. 複雑で抽象的な概念を明確に示す個々のレゴモデルとストーリーを構築します。 私たちの場合、チームの価値観とその実際の応用。
  3. モデルとストーリーをチームに紹介します。
  4. 風景、つまり 個々のモデルを共通のスペースに配置します。
  5. モデル間の接続を作成します。
  6. 共通のシステム全体を構築しています。
  7. 非標準の状況と決定を行います。
  8. 簡単なガイドラインを反映して抽出します。


LSP基本ルール



  1. 誰もが同じレゴブロックのセットを取得します。
  2. 個々のモデルは、提起された質問に対する個人的な理解を反映する必要があります。
  3. キューブは、現実世界を表示するためのツールとしてではなく、メタファーとして使用する必要があります。
  4. 何を構築すべきかわからない場合は、構築を開始してください。 あなたの手を信頼してください。
  5. 個々のモデルは、それを作成した人のものであり、所有者の同意なしに誰も部品を動かすことはできません。 変更によってモデルの意味が変わらないことを確認してください。
  6. モデルについて話し合ってください。モデルを作った人ではありません。


mitapでマジックを使用する



私たちの会議では、ツールを制限するのではなく、最初のルールから戻ることにしました。 テーブルにレゴブロックと追加のフィギュアの箱を注ぎました。 再び、彼女はラリーの参加者に質問を尋ねました：「私たちの仕事で選ばれた価値をどのように使うか？」



4人の参加者が約15分かけてモデルを作成しました。 次に、それぞれが自分のモデルを提示し、同僚の質問に答えました。これまたはその部分はどういう意味ですか？ それの意味は何ですか？ なぜ部品はこのように配置されており、そうでないのですか？









これらの議論は、私たち一人一人がチームの価値観にどのような意味を置いているのか、チーム内でどのように使用されているかを理解するのに役立ちました。 そしてリーダーは、各参加者がどのように生活しているか、どのような問題に関心があり、何が先に進むのを妨げるのかをより深く理解することができました。



それから、LSPの魔法が始まりました-個々のモデルを分析と関係のある共通の1つに結合します。 最初は、個々のモデルを1つのシステムに変えることは困難でした。 モデルの各要素の議論と分析は大いに役立ちました。 著者にとってどの要素が重要で、どの要素を簡単に削除できるかを理解することができました。



プロセスの重要な価値は、各参加者が個々のモデルの上に「昇り」、全体像を見ることができることです。つまり、彼のモデルがシステムの一部になる方法、影響するもの、構成要素に応じてシステムが変換される方法です。 その結果、各チームメンバーの頭には、彼の価値体系が他のすべての人にどのように影響するかが理解されました。 同時に、デザイナーとのゲームのリラックスした雰囲気の中ですべてが起こり、そこからすべての参加者が多くの喜びを受けました。









同じ原則により、チームの「反価値」について議論できます。 これは、どの行動パターンが受け入れられないか、そしてその理由を理解するのに役立ちます。 このトピックに関する通常の会話では、対談者を傷つけて問題を解決することは簡単ではなく、LSPテクニックはこれをより繊細で効率的にするのに役立ちます。 ゲームの参加者は同僚のアイデンティティから抽象化し、レゴブロックのモデルについて議論します。 その結果、誰もが側面から問題を見て、他の誰かの視点を理解する機会を得ます。
