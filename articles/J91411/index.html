「物理」またはMS Virtual PCをXenServerに移植する

この物語は学術的なものであり、おそらく実用的な価値はありませんが、それでも「Howto ...」

XVA（ XenServer Virtual Appliance ）形式に変換するための初期データは、VHD（ Virtual Hard Disk ）およびVMC（ Virtual Machine Config ）です。 物理マシンから仮想マシンを作成しましょう：



Microsoft OSがインストールされた抽象的なPCがある場合は、このオプションを検討してください。 VHD形式のイメージを取得するには、 Disk2Vhdユーティリティ（ http://technet.microsoft.com/en-us/sysinternals/ee656415.aspx ）を使用します。 このユーティリティの結果は、MS Virtual PC（サーバー）のディスクとして使用できるディスクイメージになります。 XVA形式に変換するには、仮想マシン構成ファイルが必要です。2つのオプションがあります。



1）独自に作成-MS Virtual Server 2005 R2を使用（MS Webサイトから無料でダウンロード可能）

2）たとえば、RAMの量、プロセッサの数、VHDへのパス（ "svn3.vhd"）を変更して、手動で編集します。



生成されたVHDファイル（私の場合は11GBのサイズ）には、システムが展開された物理ハードドライブ、たとえば250GBが含まれることに注意してください。 サーバー上のこのようなディメンションを仮想マシンに提供する見込みに満足できなかったため、次の手順を実行しました。



1）（Acronis Disk Director Suite、Partition Magicなどのプログラム）パーティションマネージャー（ボリューム）を使用して人為的にHDDをVHDに変換する前に、システムが展開されたボリュームが250GBから40GB（40GB-仮想マシンを提供する予定でした）。

2）VHD Resizerユーティリティ（ http://vmtoolkit.com/files/folders/converters/entry87.aspx ）を使用して、VHD Dyn​​amic 250GB形式がVHD Fixed 40GBに変換されました（コンピューター上で、これらすべての操作を実行しますが、十分なスペースがありました）。



VHD（必要な形式）とVMCが用意できたので、v2xva.exeユーティリティ（ http://forums.citrix.com/servlet/JiveServlet/download/511-241595-1363444-26313/v2xva-1.3を使用してXVA形式への変換を開始します。 4.zip ）：



v2xva / verbose：Loud / config： "c：\仮想マシン構成ファイルへのパス\ myvirtpc.vmc" / output： "ウイルスが形成されるフォルダへのパス"



Citrix XenCenterからの「VMのインポート」の最後のステップ-ファイルタイプ「XenServer仮想アプライアンスバージョン1（ova.xml）」を選択します



PS。 結果の仮想マシンには、ストレージ（vdi_hda）のサイズを上方向にしか変更できないという大きな欠点があります:(
