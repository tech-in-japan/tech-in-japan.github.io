Vivaldi 1.0.385.5アセンブリでタブを開閉するための新しいオプション

みなさんこんにちは！



今日のビルドでは、特にタブを開いたり閉じたりするプロセスでタブを試すことをお勧めします。 多くの人が多くのタブで作業することを好むのは秘密ではありません。私たちは、これらのタブで作業することを単純化し、より便利にしたいと考えています。 それで、今日提供するもの。



タブを開く



今日のビルドから開始して、アクティブなタブから開かれたすべてのリンクは「スタック」されます。 たとえば、ページで情報を検索し、比較のために複数の結果を開く場合、 Ctrl +クリック （バックグラウンドページで開く）を使用してアクティブなページから必要なリンクを開くことができますが、開いているリンクはすべて隣接するタブに配置されますアクティブの右側に。



同時に、 Ctrl + Tまたはタブバーのプラスを使用して新しいタブを開くと、標準モードで動作します。開いているすべてのタブの右側に新しいタブが開きます。 自分に合わない場合は、このスタイルで作業してみてください。設定で無効にするには、 [ツール]-> [設定]-> [タブ]-> [新しいタブの位置]を選択し、 [ 常に最後 ]を選択します 。



タブを閉じる動作



タブを閉じることは、タブを開いた後の論理的な手順であるため、デフォルトの動作とこのブラウザ機能を変更しました。 たとえば、アクティブなタブを閉じると、右端の「関連付けられた」タブ（たとえば、前に閉じたタブから開いたタブ）に移動します。 「リンク」タブがない場合は、前のアクティブなタブになります。 このブラウザーの動作の目的は、1つのサイトから以前に開いたタブを段階的に移動できることです。



この機能には、無効にする機能もあります。 これを行うには、 [ツール]-> [設定]-> [タブ]-> [タブを閉じるとき]に移動し、[ アクティブ化順序で前をアクティブ化]アイテムを選択します（翻訳はまだ準備できていません。



一般的に、あなたの印象や希望を試して、テストし、教えてください。 必要に応じて、これらの機能に必要な変更を加えます。



それだけです 以下のリンクを使用して、新しいアセンブリをダウンロードできます。



  • Windows 32ビット
  • Windows 64ビット （実験的）
  • Mac 10.7+
  • Linux RPM 32ビット
  • Linux RPM 64ビット
  • Linux DEB 32ビット
  • Linux DEB 64ビット
