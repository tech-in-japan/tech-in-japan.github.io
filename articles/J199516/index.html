AppJSプロジェクトの開発は中止されました。将来はnode-webkitを使用してください

1年ちょっと前（2012年10月1日）、Habrahabrにブログ記事「 Node.js + Chromium = AppJS： Web開発者の進化の第2段階の有望なオプションの1つ 」を投稿しました。これにより、各Web開発者は、仕事で使い慣れたテクノロジー（JavaScript、HTML、CSS）を使用して、Chromiumブラウザーの表現力とNode APIのパワーを組み合わせたGUI（グラフィカルユーザーインターフェイス）を備えたアプリケーションの開発者になることができます。



翌日（2012年10月2日）のブログ投稿「 Node.js + Chromium = node-webkit： Web開発者の進化の第2段階のさらに有望なバージョン 」で、AppJSが唯一または最良のオプションではないことを指摘しました。node -webkitは異なります使いやすさと優れたAPI機能、より広範なドキュメント、およびZIP圧縮されたアプリケーションの起動を提供します。



それで何？ -AppJS開発者にとってnode-webkitの成功が明らかになるまで、13か月も経っていませんでした。 今日、AppJS twitterで、 node-webkitが目標に到達したため、AppJSプロジェクトの開発が中止されたことを読むことができます。







Node-webkitを開発しているIntel Open Source Technology Centerの当然の勝利は、NodeのGUIのそのような長い均一性の始まりを意味します。これは、当時のWebでの「ブラウザ戦争」の終わりの効果と比較できます。 これにより、開発者は選択の問題を回避できますが、競合他社が存在しない場合、エンジン自体が停滞するリスクがあります。



確かに、（ 「Node + Chromium」の分野で）近い競合他社が存在しないことは、まだ何の意味もありません。 より遠い競合他社が地平線上に見えており、Web ベースのテクノロジーを使用したGUIアプリケーションの作成も目指しています-たとえば、Firefox OSのアプリケーション（今日ジェストンが 言ったように、Windows、Linux、Mac OS XだけでなくAndroidでも動作可能） 。 また、先月（2013年9月）、Googleは独自の前身も導入しました。これらはChrome Appsです 。YevgenyZolotovはComputerraで「 Chrome for everyone：GoogleブラウザはMicrosoftとApple OSに取って代わるでしょうか？」 「。
