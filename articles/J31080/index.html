Google PageRank：それについて何を知っていますか？



誰もがそれを使用しますが、実際にどのように機能するかはほとんど誰も知りません。 Google PageRankは、おそらくこれまでにWeb上で開発された最も重要なアルゴリズムの1つです。 毎日何十億もの既存のページと数百万のページが表示されます-検索結果は想像以上に複雑です。 PageRankは、検索をシンプルかつ効果的にするために最適な検索クエリを決定するためにGoogleが検討する数百の要因の1つです。 しかし、実際にはどのように作られていますか？ Google PageRankはどのように機能しますか、どの要因が影響し、どの要因は影響しませんか？ PageRankについて本当に知っていることは何ですか？

この記事は単なる事実に過ぎません。

数週間集中的な研究を行い、 PageRankについて現実に似たたくさんの事実と仮定を選びました。 さらに、検索結果に関連する科学的な記事をまとめました。たとえば、検索結果を改善するための提案（トピックに敏感なPageRankなど）です。 PageRankの数学的コンポーネントと、PageRankを操作するための16の便利なツールについてお読みいただき、Webプロジェクトの分析と追跡に使用できます。





簡単に説明すると、どのように機能しますか？



  1. PageRankは、ページの関連性や重要性を判断するためにGoogleが使用する多くの方法の1つです。
  2. Googleは、ページAからページBへのリンクをBの「音声」Aとして解釈します。投票数だけでなく、この投票が与えるページの100以上の側面を分析します。
  3. PageRank は着信リンクに基づいていますが 、その数だけでなく、関連性と品質も重要です。
  4. PR（A）=（1-d）+ d（PR（t1）/ C（t1）+ ... + PR（tn）/ C（tn）） これは、PageRankが考慮する式です。
  5. PRに関しては 、すべてのリンクが同じ「重み」を持つわけではありません 。
  6. PR = 8でサイトから別のページへのリンクが1つある場合、PRに特定の追加が追加されます。 ただし、このページへのリンクが100個ある場合、それぞれがこの補足の100分の1の部分を提供します。
  7. 壊れた着信リンクはPRに影響しません。
  8. 人気を計算するとき、サイトの年齢 、バックリンクの関連性とそれらの持続時間は考慮されます 。 ページランクを計算する場合-いいえ。
  9. PRコンテンツを計算するときは考慮されません。
  10. PageRankは、サイト全体に対して一度に決定されるのではなく、各ページに対して個別に決定されます。
  11. サイトへの各リンクは、結果にとって重要です。 カウントから除外される禁止サイトを除外します。
  12. PageRankは1〜10で定義されていません。 これは浮動小数点数です。 また、PRの初期値は0をわずかに超えています。
  13. ページランクの各次のレベルは、プログレッシブ中毒では達成がより困難です。 対数スケールで考慮されると仮定します。
  14. Googleは、各ページのPRを数か月ごとにカウントします。
  15. Googleは、関連性が高く、同時に「尊敬される」ページを見つけようとしています。


要約：Google PageRankへの影響



  1. コンテンツを頻繁に更新しても、ページランクが自動的に改善されるわけではありません。
  2. ページランクが高いということは、検索結果の上位を意味するものではありません 。
  3. ディレクトリDMOZおよびYahoo!にいる ページランクの自動改善を意味するものではありません。
  4. .eduまたは.govドメインにサイトを配置しても、ページランクが自動的に改善されるわけではありません
  5. サブディレクトリは、必ずしもルートディレクトリよりも低いページランクを持つわけではありません。
  6. ウィキペディアのリンクは、PageRankの自動改善を意味するものではありません （ただし、その資料を使用するページはPRを改善できます）。
  7. nofollow属性を持つリンクは、PageRankには役立ちません 。
  8. サイト内の効果的なリンクはPageRankに影響します。
  9. 上位の関連サイトは、カウントにおいて有利です。
  10. 多くの場合、リンクアンカーテキストは、PRの高いページ上のリンクよりもはるかに重要です。
  11. 品質関連サイトへの発信リンクまたは着信リンクは、PRに関連しています。
  12. 1つのページから特定の場所への多くのリンクは、同じページから同じ場所への1つのリンクと同じ意味です。
  13. 除外（禁止）サイトへのリンクのサイトを除外（禁止）できます。


1.1。 PageRankを使用する理由



  • 「PageRankは[のみ] Googleがページの関連性または重要性を判断するために使用する方法の1つです。」[ PageRankの正しい説明 ]
  • 「Googleは多くのランキング要素を使用しています。 したがって、PageRankアルゴリズムが最もよく知られている可能性があります。 PageRankは2つのことで表現されます。1。他のサイトからこのサイトへのリンクがいくつあるか。 2.これらのサイトの品質。 5つまたは6つの高品質サイト（cnn.com、nytimes.comなど）からのリンクは、あまり知られていないサイトからのリンクの2倍以上を意味します。」[ Google Librarian Central ]
  • 「PageRankはWebページの品質の大まかな見積もりにすぎず、テーマ関連性の尺度ではありません。 テーマの関連性は、リンクのコンテンツと、コンテンツとキーワードの相関、見出しなどの要因に依存します。” [ PageRank：An Essay ]


1.2。 彼はどのように働いていますか？



  • 誰も完全に確実ではありません。 「現時点では、GoogleがどのようにPRを検討しているかを正確に知ることはできません 。」[ Google PageRankの説明 ]
  • PR（A）=（1-d）+ d（PR（t1）/ C（t1）+ ... + PR（tn）/ C（tn）） 「この式は、PageRankの計算方法を示しています。 ここで、「t1-tn」はページAにリンクするページ、「C」はページ上のアウトバウンドリンクの数、「d」は規制要因であり、通常は0.85です。
  • より簡単に記述することができます：PageRank = 0.15 + 0.85 *（このページにリンクする各ページの「共有」）。 「共有」は、リンクページのPRをそこからの発信リンクの数で割った値に等しくなります。 ページは、リンク先の各ページに一定量のPageRankを「投票」します。 投票しなければならないPageRankの量は、独自のPageRank値（独自の値* 0.85）より少し少ないです。 この値は、リンク先のすべてのページで均等に共有されます。」[ Googleのページランク ]
  • 「 Google PageRankアルゴリズムの本質は、 アウトバウンドリンク間で独自のPRを配布することです。 PR = 8のページと別のページへの1つのリンクがある場合、このページにはPRの「重み」がすべて表示されます。 ただし、リンクが1つではなく100である場合、各リンクにはPRの「重み」の等しい部分（言い換えれば、1/100）があります。」[ PageRankの重要性 ]
  • 「PR = 4と5つのリンクを持つページは、PR = 8と100つのリンクを持つページ以上を意味します。 あなたのページにリンクしているページのページランクは重要ですが、それへのリンクの数も同様に重要です。 ページ上のリンクが多いほど、それぞれのPRの「重み」PRは小さくなります。」[ Googleのページランク ]
  • 「PageRank [..] は、リンク構造を各ページの個別のインジケーターとして使用します。 Googleは、ページAからページBへのリンクを、ページBのページAの「投票」と解釈します。Googleは、そのような「投票」またはページへのリンクの数よりも多くの要因を考慮します。 また、「投票」を受け取ったページを分析します。 「重要な」ページからの声は、他のページからの声よりもはるかに多くの意味を持ち、他のページの「重み」を高めるのに役立ちます。 [ Google：テクノロジー ]
  • 「PRに関しては、すべてのリンクが同じ重さではありません 。 T.O. 「重要」ページは、「重要度が低い」（もちろんGoogleによると）よりもPRに多くのメリットをもたらします。 [...] PR配布の威力は、「投票」ページの発信リンクの数です。 T.O. PR = 4のページと1つのリンクは、PR = 5のページと100の発信リンクよりも大きな重みを与えることができます。 この場合の典型的な例は、有名な百万ドルのページ（百万ドルのホームページ）です。 このページのPRは7ですが、何百ものリンクがリンクされたページにほとんど重みを与えていません。” [ Google PageRankの説明 ]
  • 次のレベルのPRを達成するのははるかに困難です（進行中）。 「PageRankは対数を使用してカウントされます。 同様に、指数関数的な関係では、地震はリヒタースケールで測定されます。 PageRankは数学です。 PR = 0からPR = 1に1ステップ、1から3にもう少しステップ、さらに4に、さらに5にステップなどが必要です。 "[ Google Page Rank FAQ ]




[ einfach-persoehnlich経由]

  • 「PageRankはサイト全体を定義するのではなく、各ページを個別にカウントします。 次に、ページAのPageRankは、ページAにリンクする他のページのランクによって再帰的に決定されます。
  • 「GoogleはPageRankと高度なテキスト検索テクノロジーを組み合わせて、ユーザーにとって重要なページと関連性の高いページの両方を見つけます。 Googleは、最高の検索結果を得るために、ページのコンテンツ（およびこれにリンクしているページのコンテンツ）のすべての詳細を分析します。」[ Google PageRankとは何ですか？ ]
  • 「Googleは各ページのPRを数か月に1回カウントします（PRアップデート）。 更新が完了すると、すべてのページがGoogleから新しいPRを受け取ります。新しいPRは、新しい更新の時間が来るまで持っています。 新しいサイトのランクは、更新されて特定のPRレベルが割り当てられるまで0です。」[ Google PageRankの説明 ]
  • PageRank値は0から10に変更されません。PageRankは浮動小数点数です。 「PRは、浮動小数点数としてより正確であると見なされます。 もちろん、内部PR計算には、ツールバーに表示される0〜10の値よりも多くの文字が含まれています。」[ Matt Cutts ]
  • 「それらの曲線は指数関数的な曲線に似ていると確信しています。それぞれの新しいレベルは、以前のレベルよりも達成が難しいです。 私はこのトピックに関するいくつかの研究を個人的に実施し、結果は4の指数ベースでした。したがって、PR = 6はPR = 5の4倍の達成が困難です。 [..] PR = 6の上限とPR = 6の下限の違いは、数百および数千のリンクになります。」[ 明らかにされたGoogleの神話トップ10 ]
  • 「PageRankは対数 スケールでカウントされると考えられています。 これは、PR = 4とPR = 5の差が、PR = 3とPR = 4の差よりも約5〜10倍大きいことを意味します。 そのため、PR = 4の場合よりもPR = 2の場合のページ数は100倍多い可能性があります。 つまり、PageRankが6以上の場合、他のすべてのサイトの0.1％を超えるランキングになります。」[ Google PageRankの重要性 ]
  • 「PageRankは着信リンクに基づいていますが、その数だけではありません。 代わりに、PageRankは着信リンクの「重み」に依存します。 受信リンクの「重み」を見つけるには、そのリンクの総数でPRページをこのリンクと共有します。 十分な重みを持つ少数のインバウンドリンクから6または7のPRを取得することは非常に可能です。」[ Google神話のトップ10が明らかに ]
  • 「 Googleは、信頼できる関連性の高いページを探しています。 2つのページにほぼ同じレベルの権限と検索クエリへの情報の関連性がある場合、より信頼できるサイトがリンクしているページが選択されます。 しかし、それにもかかわらず、他の要因がページの関連性が高いことを示している場合、検索結果ページのリンク数が少なくなるか、PRが少なくなります。 たとえば、南北戦争に完全に捧げられたページは、通過することについて言及している記事よりもはるかに便利ですが、Time.comなどの権威あるサイトにあります。」[ Google Librarian Central ]
  • リンクは だれにも 彼らの PRを 与えません 、彼らは「声」を与えます。 「ページが他のページのPageRank値で「投票する」場合、それ自体のPR値は減少しません。 自己PRは配布されず、「投票」の結果として終了することはできません。 また、PRの譲渡はありません。 各ページのPageRankに応じて「投票」のみがあります。」[ ページランクの説明 ]
  • 「資料「 大規模ハイパーテキストWeb検索エンジンの構造」から、ページのPageRankは、ページが各ページの共通のPRを受け取る再帰アルゴリズムの結果として得られる数値であることを知っています。これにリンクします。」[ Google PageRank ]
  • Googlebotはサイトを即座に分析しません。 「ほとんどの場合、サイトへのすべてのリンクを検索、カウント、表示するには、2か月に1回の更新が必要です。」[ Google FAQ ]
