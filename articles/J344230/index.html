Oracle Open World 2017：オフラインAIのお知らせ

オラクルのエグゼクティブ・チェアマン兼CTOであるラリー・エリソンは、10月にサンフランシスコで開催されたOracle Open World 2017のグローバル会議がこれらの発表で記憶されることを望んでいます。 オラクルは本日、クラウドサービス向けの機械学習（ML）と人工知能（AI）のほか、「スマートアプリケーション」とデータ駆動型の意思決定の提供に注目しています。







同社の代表者は、新しいソリューションであるOracle Autonomous Database Cloud（Autonomous Database Cloud、ADC）は安価であり、Oracle Cyber​​ Security Systemの追加により、どの製品のAmazon Web Services（AWS）よりも安全であることを保証しました。



Oracle Autonomous Databaseの最初のリリースは、今年末までData Warehouse Cloud Serviceを介して利用できないため、Oracleの比較がどの程度公平かはまだ不明です。



ここでさらに数か月追加できますが、独立したレビュー担当者は独自のテストを実施し、新しいOracle製品を競合するクラウドサービスと比較できます。







OOW17で発表されたいくつかの製品は、今日利用可能です。 その中には、Oracle Big Data Cloud、Oracle Event Hub、Stream Analytics Cloudサービス、およびOracle Analytics Cloud Data Lake Editionがあります。



今後3〜6か月の間に、Adaptive Intelligent Apps、AI、およびMLに関連するオラクルの発表がさらにいくつか発表され、Platform-as-a-Service（PaaS）テクノロジーと更新されたOracle Analytics Cloudが使用されます。 以下は、これらの発表に含まれる簡単な指示情報です。



Oracle Autonomous Database Cloud



クラウド戦争で勝利することは、もちろんオラクルにとって非常に重要です。 現在、同社には世界中で48万人以上の顧客があり、その最大の製品はOracle Databaseです。 明らかに、クライアントがOracle DBMSに焦点を合わせている場合、他の関連製品の選択はOracleに関連付けられます。 したがって、OOW17全体でLarry Ellisonは、「より速く、より安く」パフォーマンスと、同社の新しい提案の価値を実証しました。



彼はまた、参加者に、割引を考慮して、データベースサービスの価格が月額わずか300ドルから始まることを伝えましたが、1つの中央処理装置と1テラバイトのRAMを備えた最小構成では、今日の多くのアプリケーションが十分に機能しないようです。



オラクルのCEOであるMark Hurdの声明では、今日、企業のワークロードの約14％のみがパブリッククラウドにあると聞くことができます。 しかし、開発の強力な勢いを考えると、ワークロードの最大80％が2025年までにクラウドになります。



最新の傾向により、オラクルが保有する今日のデータベース市場における最大のシェアは減少する可能性があります。 したがって、同社は、主要なタスクの1つを、Amazon RedShiftやAmazon Auroraなどの代替手段から顧客をそらすものと見なしています。



OOW17全体を通して、高性能で低コスト（Oracleによる）が繰り返し強調されてきました。 これらは、DBMSをクラウドに配置するためのオプションを検討している顧客にとって、オラクルの主要な武器です。



同時に、潜在的な顧客にとって、Oracle Oracle Autonomous Databaseはクラウドベースのサービスにすぎないことを明確に理解することが重要です。 残り時間があまりないOracle Database 18cには、オフラインで作業する固有の機能がありません。 自律プロパティを取得するには、DBMSがOracleサービスである必要があります。



オンプレミスのオンプレミス展開オプションについても同じことが言えます。 DBMSは、顧客データセンターにデプロイされているがOracleによって管理されているOracle Cloud Machineを備えたCloud at Customerモデルを使用して配信されない限り、自律型ではありません。



少なくとも数年間、オラクルは、クラウド内の製品の動作を自動化することで得られる高い効率性について話してきました。 Autonomous Database Cloudは、会社の代表者が言うように、これらの利点を新しいレベルに引き上げます。

OOW17のオープニングで、ラリーエリソンは、企業の新しい開発は「インターネットと同じくらい革新的」であると述べました。 これは、世界で最初の100％自己管理DBMSです。 機械学習アルゴリズムを使用し、管理と構成を実質的に必要とせず、「人的要因」によるエラーの可能性を排除し、自動運転車のように機能します。



ネットワーク、サーバー、ストレージシステム、仮想マシン、オペレーティングシステム、アプリケーション、分析システム、およびデータベースによって生成される多数のシステムイベントログは、トレーニング用のデータの配列として機能します。



これにより、自律型DBMSはデータとイベントの異常を検出し、クエリを分類し、データベースを自動的に構成できます。 さらに、ラリーエリソンが強調したように、非常に緊急のサイバーセキュリティの問題は今日解決されています。データベースは異常なイベントを検出することで自身を保護できます。



Ellison氏によると、この製品はデータベース管理に関する手作業をほぼ完全に排除し、コストを大幅に削減し、99.995％（年間30分以内のダウンタイム）のレベルで可用性を保証し、システムリソースの消費を抑え、自動微調整により生産性を向上させています。



エリソンは、コンピューティングリソースとストレージリソースのこの適応的な使用を「本物の回復力」と呼びました。

Oracle Big DataとAIの進歩



場合によってはOracleは市場を先取りしており、ビッグデータや高度な分析に関しては多少遅れています。 そのため、この夏、オラクルはOracle Big Data Cloudを発表しました。 これは、HadoopとSparkに基づくビッグデータプラットフォームです。 また、Hortonworks、Microsoft、IBMで使用されているODPi標準にも近いです。



オラクルの以前のオファーであるClouderaベースのビッグデータアプライアンスは、オンプレミスソリューションまたはホスティングサービスとして引き続き利用可能です。 しかし、さらなる中心的な製品は、間違いなく、共有ストレージと処理システムを備えたOracle Big Data Cloudでしょう。 このソリューションは、大規模なデータレイクの低コストの代替手段として提案されています。



このソリューションへの最新の追加は、データをルーティングおよび処理するオープンソースのApache Kafka製品に基づいたOracle Event Hubです。 別の製品であるOracle Stream Analyticsは、書き換えられたイベント処理テクノロジーを実装し、現在はApache Sparkで実行されています。



OracleのThomas Kurian社長のスピーチで、新しいAI＆ML PaaSプラットフォームについて話されました。 ユーザーにGPU（ベアメタルインフラストラクチャと仮想マシンの両方を構築するためのグラフィックプロセッサ）のコンピューティングパワーと、Caffe、Keras、TensorflowなどのさまざまなオープンソースAI構造を提供します。 開発者は、さまざまな言語とラップトップで作業できます。







Oracle Big Data Cloudはすでに利用可能であり、OracleにAWSやMicrosoft Azureなどの大規模データサービスと競合する機能を提供します。 Event HubとStream Analyticsも利用できます。 競合他社の製品がカバーするギャップを埋めます-AWS Kinesisポートフォリオ、Microsoft AzureのEvent HubsおよびStream Analytics。



AI＆ML PaaSに関しては、このプラットフォームはOOW17で発表されましたが、（Oracle Webサイトによると）顧客はまだ利用できず、3〜6か月以内に予定されています。 マイクロソフトは10月に、次世代のAzure Machine Learningポートフォリオのベータプレビューを開始しました。これにより、継続的で模範的なライフサイクル管理が約束されます。 したがって、市場の全体像を一言で表すために、OracleはクラウドAIサービスとその機能に追いついています。



Oracle Analytics Cloud



これは、市場に対応するためにOracleが急速に動いているもう1つの分野です。 Oracle BI Cloud ServiceおよびOracle Visualization Cloud Serviceへの移行は3年前に発表されました。 Oracle Analytics Cloudは、これら2つのサービスを組み合わせ、新しい要素を追加して、データの発見、準備、分析、予測をカバーするより包括的なセットを取得します。



OracleはOOW17でData Lake Editionを発表しました。ユーザー数ではなく、使用される中央処理装置（CPU）の数に基づいたサブスクリプションにより、このコンセプトが広く採用されています。 Standard EditionおよびEnterprise Editionのサブスクリプションは以前に利用可能でした。



オラクルは、Oracle Analytics Cloud向けの一連のMLベースの拡張機能および自然言語処理も発表しました。 また、今後3〜6か月で利用できるようになります。



自動データ診断は、隠されたエンジンが示す機能を「説明」し、ユーザーが他の方法では取得できないデータと分析にユーザーを導く。



Natural Language Insightsは、プレーンテキストの隠れた瞬間と意味に対する洞察を提供し、ユーザーが特定の問題に集中できるようにします。



自然言語クエリに基づく改善された「質問」は、同義語と頭字語をサポートします。 ユーザーが入力したユーザーのリクエストを動的に修正し、異なる解釈を与えることができます。



オラクルは、メタデータおよびソーシャルメディアタグによる検索やナビゲーション、およびデータの「検出」を実行するための関連データセットおよび関連データセットに関する自動化された推奨事項など、拡張データカタログ機能にも取り組んでいます。



上記のすべてから、Oracle Analytics CloudはMicrosoft PowerBIとAzure MLが動いている道にあるように思われると結論付けることができます。 実際、この市場セグメントをリードするすべての企業は、独自の非常に完全なポートフォリオを既に作成しており、クラウドおよびデータプラットフォームの機能をあらゆる方法で開発しようとしています。



OOW17で発表された更新および機能強化は、主に市場ですでに利用可能な一連の最新機能と一致しています。 たとえば、オラクルは、その製品の機能はMicrosoft PowerBIに含まれるBeyondCore Salesforceなどに似ていると説明しました。 自然言語の自然言語の洞察は、物語科学、つまり、QlikとTableauの一部である「物語の科学」に似ています。



さらに、人々の自然言語の自動処理の現代技術のレベルは、Microsoft、IBM Watson Analytics、その他を含む多くの他の大規模開発会社の間で非常に高いです。
