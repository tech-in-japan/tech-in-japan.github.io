3 TBのプライベートクラウドを作成する

3テラバイトのWD MyBook Liveに基づいてパーソナルクラウドを作成する1つの方法を共有したいと思います。 いいえ、wd2goとその「ハーフクラウド」については言及しません。本質的には、不器用なJavaアプレットを使用したWDサービスを介したNAS自体へのアクセスのみです。 この記事では、ownCloudを使用してMBL上で実行される「正直な」クラウドについて説明します。

このソリューションは、個人のDropboxアナログを夢見ている人に適しています。ファイルは「他の場所」に保存されず、特定の物理メディアに保存され、ボリュームによってのみ制限され、このボリュームに対して毎月支払う必要はありません（インターネットの月額サブスクリプション料金を無視し、電気代）。

この問題のほとんどの解決策は、インターネット上で多くの穴を掘る必要があり、Linuxシステムの十分な知識に依存しています。 この投稿では、私自身がインターネットで発見したことのない、ロシア語で最も完全で適切なHOW-TOを提供しようとします。 非常に多くのことは、あなた自身の危険とリスクで試行錯誤によって行われなければなりませんでした。 このソリューションの実装には、Linuxの基本的な知識は必要ありません。すべての方法を、最もアクセスしやすい方法で段階的に説明していきます。



あなたがそれから来たものに興味があるなら、猫へようこそ。



まず第一に、MyBook Liveでのすべての操作は、自分の責任でのみ行ってください。 理論的には、アクションの順序が間違っていると、動作しなくなる可能性があります。ほとんどの場合、 ボックスを復活させ てデータを保存 できますが 、とにかく別のメディアにコンテンツをバックアップしてみてください。 または、そのような機会がある場合は、空白のメディアで作業することをお勧めします。



最新のWD MyBook Liveラインは、800 MHzプロセッサ、256 MBのRAM、および実質的に本格的なファームウェアとしてのDebian Linuxのバージョンですが、WDからのファームウェアファイルの混合やPPCアーキテクチャでもLennyによってサポートされなくなりました。それにもかかわらず、それは必要になりますが、ここではapt-get、wget、およびdpkgが最も一般的です。つまり、リポジトリでの作業は最大限に可能です。



ownCloudに関するhabrの記事は既にありました 。そのため、一般的な場合のインストール方法については説明しません。MBLの詳細に直接進みます。 まず、ownCloudはDropbox、Google Driveなどと可能な限り密接に連携するサーバーアプリケーションです。ハードウェアでホストされているだけで、完全に無料であり、MBLはそのようなソリューションをホストするための最悪の選択肢ではありません。

あとは、ターミナルからSSH経由でログインするだけです（WindowsマシンでPuTTYを使用できます）。 

 ssh root@192.168.XX


まず、 passwdコマンドを使用してパスワードをより強力なパスワードに変更することをお勧めします（後でインターネット経由でアクセスを開く場合はさらにそうすること）。その後、新しいパスワードを2回入力する必要があります。



重要！
ディストリビューションをメジャーバージョンにアップグレードしようとはしませんが、フルアップグレード、セーフアップグレード、またはその他のアップグレードコマンドの存在を忘れてしまうと、必然的にデバイスをダウンロードできなくなります。 デバイスを手動で分解し、レンガを解いて回復することで、好奇心を払わなければなりませんでした。 致命的なことはありませんが、非常に不快で時間がかかります。

また、インストールまたはアンインストールするパッケージを注意深く監視することをお勧めします。 そのため、せいぜい多くの不要なガベージを削除できます。最悪の場合、不要なパッケージを依存関係とともに削除すると、重要な何かがクラッシュします。 したがって、たとえば、sshを実行した場合、ボックスを元の状態に戻す唯一のオプションは、解析とレンガ造りのクリーンアップです。

最後に、まったくできないのはudevに触れることです。更新するのは危険です。この場合、My Book Liveの代わりに復元に適さないレンガを取得する危険があります。 次のコマンドでの更新を禁止するために、罪から離れることが最善です。 

 aptitude hold udev


これにより、このパッケージがフリーズし、更新される可能性がなくなります。



次に、サードパーティソフトウェアをインストールするためのプラットフォームを準備する必要があります。 そして、ここで重要なことはwd-nasパッケージを削除することであり、そのためapt-getはあらゆる些細な事柄について将来的に誓うことはありません。 奇妙なことに、この情報は公式ソースを除いてどこでも利用できません： 

 rm -f /var/lib/dpkg/info/wd-nas.*


これにより、ファームウェア依存パッケージが削除されます。



次のステップでは、利用可能なリポジトリのリストに2つのリンクを追加する必要があります。 これは、次のコマンドを使用してnanoエディターで最も便利に実行されます。 

 nano /etc/apt/sources.list


そして、ここに追加する必要があるものがあります： 

 deb http://ftp.us.debian.org/debian/ squeeze main deb-src http://ftp.us.debian.org/debian/ squeeze main


おそらく誰かが質問をするでしょう。WDMBL自体のバージョンがlennyである場合、squeezeリポジトリを追加するのはなぜですか？ 私の答えは、lennyが2012年2月以降サポートされなくなったためであり、そのために必要なパッケージのほとんどは古く、解決できない依存関係につながる可能性があります（最初の試行が失敗した後、工場設定にロールバックする必要がありました）。 インターネットには、テストブランチなど、追加できる他のリポジトリに関する多くのヒントがありますが、あまり気にしないでください。何をしているのかわからなくても、このクイックボックスを活気のないレンガに変えることができます。

混乱を避けるため、sources.listの内容を引用します。

 apt-get install php5 php5-gd php-xml-parser php5-intl zlib1g


インストーラーは、信頼できないパッケージを誓う場合があります。 これは、Debianリポジトリの署名付きキーがもはや関係ないためです。 この問題には解決策がありますが、「Y」を数回クリックするだけで警告を無視することもできます。 また、インストールプロセス中に、パッケージのさまざまなバージョンの競合が発生する可能性があり、その一部はシステムにプリインストールされています。 デフォルトでは、すでにインストールされているパッケージ（これはDebian）が優先されるため、この決定に同意することは十分可能です（ただし、これらのパッケージを新しいパッケージに置き換えても、何も悪いことは起こりません）。

奇妙なことに、このタスクにはかなり単純なソリューションがあります。 いいえ、不便なwd2goサービスの方向に目を向けるとは思わないでください（それを使用する方法もいくつかありますが）。それどころか、私たちは再び頭を動かし、Dynamic DNSなどの便利なものを使用します。



このために必要なもの：

  1. ドメインを登録します。
  2. DynDNSを提供するサービスに登録します。
  3. MyBookLiveにDynDNSクライアントをインストールする


最初の点については、無料の第3レベルドメイン（強打でグーグル）でさえも非常に適していますが、もちろん、独自のドメインを登録することをお勧めします。 非常に安価なドメインゾーンがあります。たとえば、 .pwゾーン（パラオ諸島、一部のレジストラはProfessional Webのように宣伝しています ）のドメインは、年間7ユーロ弱のコストです。 オプションとして-GoDaddyの株を見てください（わずか5ドルでちょっとした情報を手に入れることができます）。



DynDNSサーバーとして、私は個人的にfreedns.afraid.orgから完全に無料を選択しました。何故かdyn.com/dnsからの未使用チップに年間最低サブスクリプションを支払いたくないからです 。 私たちのタスクでは、最も最小限の機能が完璧です。 この記事のフレームワークでドメインを登録し、DynDNSサービスにバインドする手順については詳しく説明しません。 サービスに登録し、 freedns.afraid.org / domainページで「FreeDNSにドメインを追加」という項目を見つけて、次のnsサーバーをレジストラーのDNSとして追加します。 

 NS1.AFRAID.ORG NS2.AFRAID.ORG NS3.AFRAID.ORG NS4.AFRAID.ORG


同時に、 Brokenの赤いステータスを恐れないでください。すべてが正常であれば、24時間以内に消えます。



次に、ダイナミックDNSクライアントを選択してMyBookLiveにインストールする必要があります。 実際、これは簡単なタスクではありません。Perl、Python、PHPにはそのようなソリューションが本当にたくさんあり、あらゆる種類とストライプの* nix用の個別のデーモンもあります。 適切なソリューションを見つける前に、私は多くのオプションを試さなければなりませんでしたが、どこかでうまくいかないものがありました。1か所でセットアップすると安定性が機能しません。 結局、私は最小で、最も単純で、最も単純なinadynクライアントにとどまることにしました。 利点-軽快な作業、非常に小さいサイズ、選択したサーバーとの互換性、非常に柔軟な構成、クラウンのサポート、デーモンモードでの起動、指定した間隔でのIPの更新。 マイナスのうち、おそらくGUIが存在しないため、このようなコンソールユーティリティにとってもマイナスではありません。



次のコマンドで簡単にインストールできます。 

 apt-get install inadyn


また、inadynがインストールされていない場合、inadynが動作するためにcurlが必要になる場合があることに注意することも重要です。 次のように解決されます。 

 apt-get install curl


次に、アカウントでfreedns.afraid.org/dynamicにログインし、 先ほどリンクしたドメインの反対側のDirect URLアイテムを選択する必要があります。 ブラウザの行にも同様のものがあることに注意してください。 

 http://freedns.afraid.org/dynamic/update.php?OHphN3RsHY0SEFtZ1JrQ2V2Z1pTOjk2NzEwOTTVM1cc=


右側のすべてに興味がありますか？ 。 これは、ドメインでのみ有効なハッシュです（例の値は変更されています）。 どこかにコピーしてください。すぐに必要になります。

次のコマンドでinadyn構成ファイルを作成します。 

 nano /etc/inadyn.conf


必要な設定を挿入して保存します： 

 --username <_> --password <> --update_period_sec 60 --forced_update_period 120 --alias <_>,<> --background --dyndns_system default@freedns.afraid.org --syslog


誰が気にする、解読
  • ユーザー名-ここでは、 freedns.afraid.orgに登録したユーザーの名前を示します
  • パスワード-そこからのパスワード
  • update_period_sec-IPをチェックおよび更新する頻度（変更された場合）秒
  • forced_update_period-秒単位の強制IPリフレッシュレート
  • alias-ドメインとそのハッシュを示す行で、以前に記述したもので、コンマで区切られています
  • background-バックグラウンドで実行するよう指示する
  • dyndns_system default@freedns.afraid.orgの行は変更せずに、情報がどのサーバーで処理されているかをプログラムに伝えます（内部交換プロトコルを示します）
  • syslog-システムログにイベントを書き込みます




ここで、再起動時に自動イベントとしてinadynをクラウンに追加する必要があります。これにより、MyBookLiveが再起動されるたびに自動的に開始されます。 これを行うには： 

 crontab -e


次の行を追加します。 

 @reboot /usr/sbin/inadyn


操作後にプログラムの機能を確認するには、MBLコマンドをリロードできます

その後、再起動して起動するまで3〜5分待ってから、プロセスを確認します。 

 ps -A | grep inadyn


pingコマンドを使用して、ドメイン名でmaybookの可用性を確認するだけです。 

 ping <yourdomain.com>

。 Apacheを誤って更新または変更した場合、Webインターフェースが失われ、工場出荷時の設定にリセットしないと起動できない状況に陥ることがあります。 それを無意識に更新することも価値がありません。 不溶性の依存関係に出くわすこともありますが、LennyやPPCアーキテクチャ向けの代替オプションはあまりありません。

質問： Feature Pack Managerは、記事に記載されている内容に何らかの影響を与えることができますか？

回答：その核となるfpkmgrは、ユーザーが選択したパッケージのセットをインストールする、ある種の高度なパッケージマネージャーです。 fpkmgrとowncloudからの送信は、MBLで並行して静かに動作しました。 おそらく、競合する可能性のあるfpkmgrを使用してインストールされたパッケージがいくつかありますが、これに関する情報はまだありません。 誰かが出会ってそれについて書いたら、ここにコメントを入れます。



質問：この記事で説明されている手順の後、MBLがスリープ状態になると、owncloudは切断されますか？

回答：最も興味深いのは、LEDの動作から判断して、私が省電力をオフにするまでスリープ状態になったことです。 実際、技術的に私はこの側面を研究していませんでしたが、ブラウザから入ったときに「ウェイクアップ」してowncloudのコンテンツを表示するのに少し遅れて働きました。 省電力をオフにすると、すべてが正常に機能します。
