VDIに焦点を当てる：NetApp All-Flash FASストレージシステム

今日の仮想ワークステーションインフラストラクチャは、企業に多くの利点を提供します。 ただし、効果的なVDI環境を構築するには、最大のパフォーマンスを提供し、同時に展開および運用コストを削減できるハードウェアの選択に慎重かつ慎重に取り組む必要があります。







効率的な仮想化インフラストラクチャの構築は、ストレージシステムのパフォーマンスに直接関連しています。 仮想ワークステーションに最適なストレージ環境は、多数の読み取りおよび書き込み操作用に設計し、パフォーマンスを犠牲にすることなくピーク負荷に耐える必要があります。 多数のユーザーがいるインフラストラクチャでディスクまたはハイブリッドストレージを使用する場合、問題が発生する可能性があり、ストレージを拡張すると所有コストが増加します。



この問題の解決策は、フラッシュドライブに基づくストレージシステムの使用にあります。 正しく選択されたオールフラッシュアレイは、低レイテンシを確保しながら、仮想ワークステーションの負荷に耐えることができます。



強力なタンデム



Clustered Data ONTAPオペレーティングシステムを実行するNetApp All-Flash FASは、VDI環境に最適なソリューションとなり、ストレージシステムのフラッシュテクノロジーの利点を十分に活用できます。 オールフラッシュFASは、異種または同種のストレージシステムを使用して拡張できるマルチプロトコル、マルチユーザー環境であり、圧縮、重複排除、クローン作成、複製などのデータ管理およびデータ削減機能を備えています。



NetApp Clustered Data ONTAPオペレーティングシステムは、WAFL（Write Anywhere File Layout）ファイル構造を持ち、データストレージ用のフラッシュテクノロジーの使用と互換性があります。 最適化されたOS記録アーキテクチャは、仮想ワークステーションに最適であり、いくつかの利点があります。 そのため、たとえば、システムはメモリ内の着信レコードを処理し、NVRAMに保存してすぐに確認します。これにより、書き込み操作の応答時間が最小になり、SSDのピーク負荷がなくなります。 レコードはメモリで結合され、そこから連続ブロックの長いチェーンによってSSDドライブに転送されます。これにより、パフォーマンスが最大化され、RAIDメモリのコストが削減されます。 さらに、書き換え可能なブロックは自動的に別の場所に書き込まれ、WAFLファイル構造は既存のデータブロックを上書きしないため、メディアの摩耗が減少します。 また、Data ONTAPバージョン8.3では、フラッシュテクノロジーのパフォーマンスを最適化するために読み取りパスが改善されたことにも注意してください。 これらの変更により、既存システムの読み取りパフォーマンスを最大70％向上させることができました。



追加機能



上記のData ONTAPシステムの標準機能に加えて、NetApp All-Flash FASは次世代のFlashRayプラットフォームの機能を実装し、効率性、パフォーマンス、使いやすさを向上させます。



特に、リアルタイムデータ圧縮の高い効率に注目する価値があります。 NetApp独自のデータ圧縮プロトコルのおかげで実現しています。 パフォーマンスの点では、LZ4などの圧縮アルゴリズムを上回り、圧縮解除コストを40％以上削減します。 さらに、読み取り操作の速度が向上します。 また、非圧縮データを検出する機能により、非圧縮データの検出によりプロセッサのサイクル数が削減され、圧縮されない情報が即座に識別されます。



ゼロブロックのリアルタイム重複排除により、仮想マシンリソースの割り当てが20〜30％加速されます。 すべてのデータはハッシュされ、システムに入るとすぐに相互に比較されますが、ゼロブロックは削除されます。 より徹底したビット単位の検証のために、同様の情報が記載されています。 重複排除は毎分実行されることに注意してください。 このアプローチにより、安定した低遅延が可能になります。



経済的な保管



Data ONTAPは、他のソリッドステートドライブソリューションが誇ることのできないさまざまな展開およびソフトウェア定義の管理機能も提供します。 たとえば、VDIデータの追加コピーをハードドライブまたはクラウドに保存する機能。 2番目のオプションでは、ストレージをより経済的に使用できます。



ディスクスペースの節約は、コンパクトクローンの使用によっても達成されます。 NetAppクローニングテクノロジーを使用すると、仮想マシンの書き換え可能な「コピー」を作成できます。 これらのコピーは、変更が行われた場合にのみ追加の容量を必要とし、使用される場合、584：1の比率でディスク容量が節約されます。 たとえば、ハイパーバイザーのスナップショットを作成する場合、メタデータ用に20％のディスクスペースがさらに必要です。 標準のNetAppクローン作成機能により、追加のリソースが不要になり、データ密度が向上します。



さらに、Clustered Data ONTAPでは、使用するアプリケーションの動作を中断することなく、いつでもストレージにデータを転送できます。 仮想ワークステーション環境は、最初はハイブリッドストレージシステムに展開でき、将来、拡張が必要な​​場合は、ストレージシステムに転送できます。



FlexPodにVDIを展開する



仮想ワークステーションインフラストラクチャを開発および展開し、管理するプロセスは、多数のソリューションコンポーネントが関係するため複雑です。 この問題を解決するために、NetAppとシスコはFlexPodリファレンスアーキテクチャを開発しました。これにより、運用展開と便利な管理に必要なすべてのハードウェアおよびソフトウェアコンポーネントを含む仮想ワークステーションインフラストラクチャを作成できます。



統合および標準化されたFlexPodコンポーネントにより、エンタープライズアプリケーションの要件を満たす水平または垂直スケーリングが可能になります。 各FlexPodシステムを実装すると、消費電力、設置面積、使用可能なディスク容量、パフォーマンス、コストを正確に見積もることができます。
