スクリーンショットをより速く撮る方法：Amazon S3に独自の無料ストレージを作成する

誰かが面白い冗談を言うために、そして誰かが働くために-私たちは毎日スクリーンショットを必要とします。 これらの目的に最も理想的なツールの1つはMonosnapです。これはLive Typingで積極的に使用されています。 ある時点で、プログラムが定期的に（そして常に誰かのために）ハングし始め、何もロードされないという事実に注意が向けられました。 スクリーンショットが緊急に必要な場合、そのような問題は人生を複雑にする可能性があります。 これを回避する方法、つまりスクリーンショットをダウンロードして自分のストレージに保存する方法について説明します。





Monosnapには、保存先に関する多くのオプションがありますが、最も普遍的なのはAmazon S3クラウドサービスです。 最もフォールトトレラントであり、お金を要求せず（詳細は以下）、最適な地理的位置にあるため、外国の顧客も問題なく表示できるため、ユニバーサルです。



なぜ無料になるのですか？ 無料の割り当てには、1か月あたり5 GBのストレージ、20,000ダウンロード、2,000ダウンロードが含まれているためです。 スクリーンショットには十分ですが、スクリーンキャストには十分ではありません。 不快な驚きを受け取らないように、 料金をご覧ください。



それでは始めましょう。 まず、 Monosnapをお持ちでない場合はダウンロードします 。 次に、 aws.amazon.com/s3に移動します 。 ここにこのボタンが表示されます。





クリックしてください。



その後、まだ登録していない場合はAmazonに登録するか、単にログインする必要があります。 次は標準的な登録手順です：名前、メールアドレス、連絡先の詳細、支払いカードの詳細。 「支払い登録」ステップのフィールドに入力すると、カードの有効性を確認するためにカードから1ドルを引き落とすことに関するSMSを受け取ります。







これはあなたがしなければならない唯一の現金寄付です。 お金は2週間以内に返送されます。



「IDの検証」段階で、4桁のPINコードを受け取ります。 「今すぐ電話」ボタンをクリックして、「ロボット」の女の子とチャットします。英語で、彼女は電話のキーパッドに同じPINコードを入力するように求めます。 「ロボット」が沈黙しているときに入力し、入るときには注意してください。一連の不正な試行が12時間の登録の可能性によってブロックされるためです。



すべてが終了すると、AWSアカウントの作成に関するメールが届きます。



次に、すべてのサービスのリストを含むページに移動する必要があります。 S3を選択します。







その後、バケットのページに移動する必要があります。 用語では、AWS Bucketはストレージです。 そこに何かを保存するには、それを作成する必要があります。







[バケットの作成]をクリックします。 このようなモーダルウィンドウが開きます。







バケット名には、大文字のラテン文字と数字のみで構成される一意の名前（もちろん、画面で既に使用されている名前）を入力する必要があります。 地域としてフランクフルトを選択します-それは私たちに最も近いです。 作成をクリックします。



バケットを作成すると、リポジトリブラウザーを含む新しいページが開きます。 そこで、新しく作成したリポジトリの名前をクリックしてから、「フォルダの作成」をクリックします。 次に、スクリーンショットをアップロードするフォルダーを作成する必要があります。











次に、Monosnapがそこにファイルを書き込めるように、S3へのアクセス権を設定する必要があります。 navbarで名前を見つけてクリックし、Security Credentialsを見つけます。







その後、AWSは、セキュリティ認証情報またはIAMユーザーを使用する2つのオプションから選択するよう求めます。 後者が必要なので、Getting Started with IAM Usersをクリックしてください 。



その後、ユーザーのリストを含むページが開きます。 その上でCreate New Usersをクリックする必要があり、ユーザー作成ページに行きます：







必要なのは1つだけです。 それがmonosnapと呼ばれると仮定します。 各ユーザーのアクセスキーを生成するために、チェックを外す必要はありません。 作成をクリックします 。



その後、ユーザーが作成されたことを確認するページが開きます。 また、このユーザーのアクセス設定もあります。これは、[ユーザーセキュリティ資格情報の表示]をクリックして確認できます。







注意：このページには、これらの設定が最後に表示されます。 そのため、それらを書き留めるか、[資格情報のダウンロード]ボタンをクリックして、見つけられる場所に保存します。



[閉じる]をクリックすると、ユーザーのリストを含むページが表示されます。 このリストで作成したユーザーを見つけます。 次に、ユーザーに関するすべての情報を要約したページが表示されます。 その上で、 Permissionsタブを見つけ、 Attach Policyをクリックする必要があります。











開いたリストで、 AmazonS3FullAccessを見つけ、チェックマークを付けて 、[ Attach Policy ]をクリックします。







S3の構成はこれで完了です。 Monosnapを設定するのはあなた次第です。



設定に移動し、 アカウントを選択して、そこにAmazon S3を配置します。







前の手順のいずれかで、 アクセスキーIDとシークレットアクセスキーを保存しました。 そこからこのデータを取得して、ここに入力します。 Macをお持ちの場合、Monosnapは[ バケットの地域]フィールドで地域を指定するように求めます。 eu-central-1を作成します-これはフランクフルトで、最初に選択しました。 次に、作成されたバケットを選択し、パス（これはディレクトリの名前です）とベースURLを指定します 。 ベースURLは次のようになります。

http：// {bucket_name} .s3.amazonaws.com / {path} /



[ テスト接続]をクリックします。すべてが正常な場合、対応するメッセージが表示されます。



次に、スクリーンショットを撮ってみます。 すべてが判明して機能する場合、正しいリンクがクリップボードにコピーされます。



したがって、通常の古き良きプラムバスが判明します。 完了、使用できます！
