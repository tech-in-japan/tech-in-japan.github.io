iOS 8-ウィジェット



iOS 8のリリースにより、開発者は、Today画面用に独自のウィジェットを作成する機会が得られます。 これまでのところ、APIは最終的に解決されていません。 既知の問題があり、ドキュメントに記載されていない多くの瞬間があります。 それでもウィジェットを作成したい場合は、カットをお願いします（例ではSwiftが使用されています）。






拡張機能



IOSは、拡張機能という新しい概念を導入しました。 拡張機能を使用すると、アプリケーションの外部でコンテンツと機能の一部を利用可能にすることができます。



拡張機能をサポートするシステムの部分は、拡張ポイントと呼ばれます。 iOSでは次の拡張ポイントを使用できます。

  • Today（通知センター）-通知センターのToday画面からアクションをすばやく実行したり、情報を取得したりします
  • 共有-友人と、または任意のサイトのフィードでコンテンツを共有します
  • アクション-別のアプリケーションのコンテキスト内でコンテンツを表示または管理します
  • 写真編集-写真アプリ内で写真やビデオを編集します
  • ストレージプロバイダー-現在のアプリケーションで使用可能なドキュメントのセットからドキュメントを選択します
  • カスタムキーボード-ネイティブのiOSキーボードをすべてのアプリケーションで使用するためのキーボードに置き換えます


拡張機能は、通常のアプリケーションのバンドルである拡張機能コンテナでのみ配布できます。 1つのコンテナに複数の拡張機能を含めることができます。



拡張機能は特別なタイプのバイナリファイルであることに注意してください。 これはアプリケーションではありません！

残念ながら、拡張機能はApp to App IPC（パイプ、ソケットなど）をサポートしていないため、使い慣れた[UIApplication openURL：]（拡張機能では機能しません。 既知の問題を参照）またはApp Groupなどを使用する必要があります。 。



各拡張機能は個別のプロセスで実行されます。 したがって、異なるアプリケーションのコンテキストにおける同じ拡張機能は異なるプロセスであり、同期の問題を心配する必要はありません。

ウィジェット



ウィジェットは拡張機能と呼ばれ、Today画面の通知センターに情報を表示するため、現時点で重要な情報を表示するように設計されています。 ユーザーがTodayを開くと、興味のある情報がすぐに利用できるようになると期待しています。



ウィジェットは、ユーザーがウィジェットを含むアプリケーションをインストールした後に使用可能になります（アプリケーションの最初の起動後にウィジェットがインストールされないことが起こります。結局のところ、これはまだベータ版です）。 ウィジェットを追加するには、通知センターで[今日]画面を開き、[編集]ボタンをクリックして目的のウィジェットを追加する必要があります。



コンテナとウィジェット間の接続は、 NotificationCenter.frameworkを介して行われます。



基本的に、ウィジェットは、iOSプログラマーに馴染みのあるUIViewControllerです。 したがって、ウィジェットを作成するときに、以前に蓄積した知識を使用できます。 たとえば、ウィジェットを表示する前に何らかのアクションを実行する必要がある場合、viewWillAppearなどをオーバーライドする必要があります。



ウィジェットを常に最新の状態にするために、iOSは時々ウィジェットのスナップショットを作成します。 ウィジェットが再び表示されると、最後のスナップショットが最初に表示され、次に実際のウィジェットウィンドウが表示されます。 スナップショットの前にウィジェットの状態を更新するために、 NCWidgetProvidingプロトコルが使用されます。 

protocol NCWidgetProviding : NSObjectProtocol { // Called to allow the client to update its state prior to a snapshot being taken, or possibly other operations. // Clients should call the argument block when the work is complete, passing the appropriate 'NCUpdateResult'. @optional func widgetPerformUpdateWithCompletionHandler(completionHandler: ((NCUpdateResult) -> Void)!) // Clients wishing to customize the default margin insets can return their preferred values. // Clients that choose not to implement this method will receive the default margin insets. @optional func widgetMarginInsetsForProposedMarginInsets(defaultMarginInsets: UIEdgeInsets) -> UIEdgeInsets }


widgetPerformUpdateWithCompletionHandlerがウィジェットで呼び出されると、そのウィンドウを更新してから、次の定数のいずれかに等しい引数でcompletionHandlerブロックを呼び出す必要があります。

  • NCUpdateResultNewData-新しいコンテンツにはウィンドウの更新が必要
  • NCUpdateResultNoData-ウィジェットを更新する必要はありません
  • NCUpdateResultFailed-更新プロセス中にエラーが発生しました


ユーザーは通知センターからの即時の反応を待っており、システムがスナップショットを実行するため、ウィジェットは単に以前の状態を保存する、つまり、動作に必要なデータをキャッシュする必要があります。



通知センターはウィジェットの幅を決定し、ウィジェット自体はその高さを決定します。 高さを決定するために、ウィジェットはUILayoutControllerインスタンスのAuto LayoutまたはpreferedContentSizeプロパティを使用できます。 

 override func viewDidLoad() { super.viewDidLoad() self.preferredContentSize.height = 350 }


ウィジェットには次の要件があることがわかります。

  • 表示されるコンテンツが最新であることを確認してください
  • ユーザーのアクションに適切に対応する
  • できるだけ少ないリソースを消費します（iOSは、大量のメモリを消費するとウィジェットを強制終了できます）


ウィジェットUIには次の制限があることに注意してください。

  • キーボードを表示しません
  • ジェスチャーで機能するコントロール（UIDatePickerなど）は使用できません
  • 現時点ではマップを表示できません（この項目は既知の問題に表示されます-Mapviewsはウィジェットにタイルをロードしません）。


キーボードを使用できないため、ユーザーはコンテナアプリケーションでウィジェットを構成できる必要があります。



ウィジェットの相互作用



ウィジェットによって表示される情報を更新するために、 NCWidgetControllerクラスがあります。

このクラスのインスタンスには、単一のsetHasContent：forWidgetWithBundleIdentifier：メソッドがあります。このメソッドは、情報を更新する必要があることを示すメッセージをウィジェットに送信します。

次のように使用します。 

 NCWidgetController.widgetController().setHasContent(true, forWidgetWithBundleIdentifier: "com.e-legion.Traffic.Widget")


このクラスは、ウィジェットおよびコンテナーアプリケーションから使用できます。



コンテナアプリケーションとのデータ交換



コンテナと通信するために、 UIViewControllerが所有するextensionContextプロパティを介してアクセス可能なNSExtensionContextオブジェクトが使用されます。 

 class NSExtensionContext : NSObject { // The list of input NSExtensionItems associated with the context. If the context has no input items, this array will be empty. var inputItems: AnyObject[]! { get } // Signals the host to complete the app extension request with the supplied result items. The completion handler optionally contains any work which the extension may need to perform after the request has been completed, as a background-priority task. The `expired` parameter will be YES if the system decides to prematurely terminate a previous non-expiration invocation of the completionHandler. Note: calling this method will eventually dismiss the associated view controller. func completeRequestReturningItems(items: AnyObject[]!, completionHandler: ((Bool) -> Void)!) // Signals the host to cancel the app extension request, with the supplied error, which should be non-nil. The userInfo of the NSError will contain a key NSExtensionItemsAndErrorsKey which will have as its value a dictionary of NSExtensionItems and associated NSError instances. func cancelRequestWithError(error: NSError!) // Asks the host to open an URL on the extension's behalf func openURL(URL: NSURL!, completionHandler: ((Bool) -> Void)!) }


つまり、ウィジェットからコンテナアプリケーションを開くには、コンテナがスキーム（たとえば、「traffic：//」）を登録する必要があり、ウィジェットコードに追加する必要があります 

 self.extensionContext.openURL(NSURL(string: "traffic://"), completionHandler: nil)
。

デフォルトでは、iOSセキュリティにより、コンテナアプリケーションと拡張機能間のデータ交換が防止されます。 データ交換を有効にするには、コンテナターゲットと拡張機能を同じApp Groupに追加する必要があります。



その結果、資格ファイルには次の内容が含まれます。 

 <?xml version="1.0" encoding="UTF-8"?> <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd"> <plist version="1.0"> <dict> <key>com.apple.security.application-groups</key> <array> <string>group.96GT47C53G.traffic</string> </array> </dict> </plist>


96GT47C53Gに注意してください。 これは開発チームIDです。 プロフィールで確認できます 。 シミュレーターで実行するには、group.trafficなどの任意の値を使用できます。



これで、 containerURLForSecurityApplicationGroupIdentifierメソッドを使用して、共有フォルダーへのパスを取得し、アプリケーション固有のデータをそこに保存できます。 

 NSFileManager.defaultManager().containerURLForSecurityApplicationGroupIdentifier("group.96GT47C53G.traffic")


例



それでは、渋滞のある地図を表示するウィジェットを作成してみましょう。 Yandex.Maps APIを使用して、交通情報を写真形式で受け取ります 。



写真を受信するためのリンクの例： http : //static-maps.yandex.ru/1.x/?ll=30.35,59.9690273&spn=0.01,0.2&size=300,250&l=map,trf

実際には、latとlonはマップの中心であり、spnはマップ表示領域の範囲（度単位）です。



プロジェクトのソースコードはGitHubで入手できます 。



コンテナアプリケーションを作成する



コンテナのタスクは、ユーザーがウィジェットを設定できるようにすることです。つまり、マップ上のエリアを選択します。 したがって、アプリケーションには、領域を選択するためのMapViewと、この領域をウィジェットに転送する「フレームを設定」ボタンが含まれます。





アプリケーションでは、ユーザーがlat、lon、spnパラメーターを選択してウィジェットに渡すことができるようにする必要があります。 次のコードはこれを行います。 

 @IBAction func updateWidgetButtonTapped(sender : AnyObject) { var dict : NSMutableDictionary = NSMutableDictionary() dict["spn"] = self.mapView.region.span.latitudeDelta dict["lat"] = self.mapView.region.center.latitude dict["lon"] = self.mapView.region.center.longitude var dictUrl : NSURL = NSFileManager.defaultManager().containerURLForSecurityApplicationGroupIdentifier("group.96GT47C53G.traffic").URLByAppendingPathComponent("settings.dict") dict.writeToFile(dictUrl.path, atomically: true) NCWidgetController.widgetController().setHasContent(true, forWidgetWithBundleIdentifier: "com.e-legion.Traffic.Widget") }


ウィジェットを追加



ウィジェットを追加すると、新しいターゲットが追加されます。

「ファイル」->「新規」->「ターゲット」をクリックし、「iOS」->「アプリケーション拡張機能」->「Today拡張機能」を選択します。







これで、ウィジェットのスタブがプロジェクトに追加されました。 ただし、ウィジェットを今すぐ使用しようとすると、何も機能しません。 ウィジェットがクラッシュします。 これを修正するには、次のメソッドをTodayViewControllerに追加します。 

 init(coder aDecoder: NSCoder!) { super.init(coder: aDecoder) }


テンプレートのInfo.plistファイルに注意してください。 NSExtensionキーが含まれています。このキーは、いくつかのウィジェットパラメーターを定義します。 

 <key>NSExtension</key> <dict> <key>NSExtensionMainStoryboard</key> <string>MainInterface</string> <key>NSExtensionPointIdentifier</key> <string>com.apple.widget-extension</string> </dict>


NSExtensionMainStoryboardには、ウィジェットのコントローラーを格納するストーリーボードの名前が格納されます。 NSExtensionMainStoryboardキーをNSExtensionPrincipalClassに置き換え、値としてコントローラーの名前を使用することで、コントローラーを明示的に指定できます。



各ウィジェットにはわずかに左シフトがあります。 削除する場合は、widgetMarginInsetsForProposedMarginInsetsメソッドで目的のUIEdgeInsetsを返す必要があります。 

  func widgetMarginInsetsForProposedMarginInsets(defaultMarginInsets: UIEdgeInsets) -> UIEdgeInsets { return UIEdgeInsets(top: 0, left: 0, bottom: 0, right: 0) }


ウィジェットのコードは非常に簡単です。 マップを更新するupdateMapメソッドがあります。 更新は、ウィジェットの表示（viewWillLoad）、ボタンのクリック、widgetPerformUpdateWithCompletionHandlerの呼び出しが開始されると発生します。 ウィジェットは、 containerURLForSecurityApplicationGroupIdentifierを介して表示された領域に関するデータを受け取ります。



おわりに



ウィジェットは非常にクールなものですが、これまでのところすべてが湿っています。 クラッシュが発生する場合や、ウィジェットが表示されない場合などがあります。 しかし、何よりもドキュメントが不足しています。 すぐにこれはすべて修正され、新しい機能を使用してアプリケーションを充実させることが可能になりますが、今のところは何を試すことができます。
