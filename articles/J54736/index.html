Symantec Backup Exec System Recovery。 基本設定









以前の投稿で、このプログラムのインストールについて簡単に説明しました。 次に、彼女と仕事をする方法について話をする番です。 まず、予約の戦術を決定する必要があります。 私の意見では、最も簡単なオプションは、まずシステムの回復を提供し、次に作業ファイルとフォルダーをバックアップすることです。



システムの回復から始めましょう。 Backup Exec管理コンソールを開きます。 最初の起動時に、リカバリポイントが存在しないという警告が表示されます（図）









修正してください。 バックアップウィザードを実行します。 次に、コピーの種類を選択する必要があります-システム全体をコピーするか、選択したファイルとフォルダーのみをコピーします（図）









最初のオプションに興味があります。 次の手順では、バックアップディスクを選択します。 私たちは選択して先に進みます。 Backup Execは、作成する2種類のリカバリポイントを提供します-いわゆる増分ポイントと独立したポイント。 これらの違いは次のとおりです。増分ポイントを使用すると、最後のバックアップ以降にコンピューターに加えられた変更のみを含む開始ポイントと追加ポイントを一度作成できます。 また、独立したポイントには、選択したドライブの完全バックアップが常に含まれます。 定期的にバックアップする予定がある場合は、増分復元ポイントを選択する必要があります。そうしないと、システムの永続的なフルコピーのために多くのディスク領域が必要になります。

次のステップでは、バックアップの場所と追加の保存場所を選択します。 バックアップには、独立したハードドライブを持つ最小のバックアップをお勧めします。









バックアップを開始する前に実行されるリカバリポイントパラメータとバッチファイルを指定する必要があります。 パラメータのうち、最も重要なものに注目する価値があります。実行、圧縮、分割（アーカイブを簡素化するため）、セキュリティ（パスワードと暗号化）の後にポイントをチェックします。















私の意見では、最も重要な設定の1つはバックアップスケジュールです。 この設定はウィザードの最後です。









追求する目標に応じて、さまざまな方法でコピーのスケジュールを選択できます（1回限りの実行またはサーバーに最適な通常のコピー）。 最後に、ウィザードはバックアップを求めるプロンプトを表示します。 したがって、システムの復元ポイントが作成されます。

システムが存在するディスク以外のディスクをバックアップするには、ディスクのコピーにはあまり意味がないので、ファイルとフォルダーのコピーという2番目のタイプのコピーを使用できます。 バックアップウィザードはほぼ同じです。 ファイルとフォルダーのコピーを選択した後、バックアップする必要のあるフォルダーとファイルをリストに追加する必要があります（図）









次に、バックアップオプション（名前、場所、セキュリティ）を設定し、最後にスケジュールを設定します。 これで基本的なセットアップは完了です。 説明した機能に加えて、このプログラムには非常に重要なネットワーク機能が多数あります。 私の意見では、最も興味深いのはネットワークバックアップです。 この機能は次のように実装されます-管理コンソールから、エージェントプログラムをリモートコンピューターにインストールできます。 その後、リモートコンピューターのバックアップを管理することが可能になります。 これはシステム管理者にとって非常に重要な機能のようです。

今のところすべてです。
