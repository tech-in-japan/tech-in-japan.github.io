埋め込み言語：なぜLuaなのか？

この資料は、昨年12月のGames Gathering 2017カンファレンスで作成したレポートに基づいて、一連の出版物を継続しています。 レポートの 1つで、組み込みスクリプト言語を選択するトピックが取り上げられました。







スクリプト言語が必要な理由と理由



ブログの以前の投稿で既に述べたように、当社は独自のエンジンを作成しました。 今日は、このエンジン用のスクリプト言語を選択する際にガイドされたものについてお話します。



スクリプト言語が必要なのはなぜですか？ 「ゲーム業界では複雑な問題はありません 。 複雑なソリューションを要求する必要はありません 。 」 複雑な問題を解決する経験豊富な（そして高価な！）プログラマーに加えて、私たちには、たとえばクエストに携わる人（多くの人！）が必要です。 そして、正直に言って、これらのプログラマーはそれほど高価ではなく、理想的にはプログラマーではなく、直接ゲームデザイナーやスクリプトライターになることを望んでいます。



したがって、プログラマーの重い砲兵を巻き込むことなく、シンプルでありながらロジックを記述するツールが必要です。 結論-スクリプト言語とは何ですか？ これは、ゲーム開発をより迅速かつ安価にするツールです。

すぐに問題が発生します。なぜXMLのようなものを使用しないのですか 実際のところ、XMLが宣言的な記述であるのに対して、目的のために分岐とループの制御構造が必要になることがよくあります。



スクリプト言語のもう1つの利点は、プロジェクト内のスクリプトがコードまたはリソースのいずれかであることです。 そして、それに応じて、ゲームのスクリプト部分はコードだけでなく、つまりアプリケーションストアのメカニズムによる定期的な更新中に更新できます。 しかし、リソースも-つまり、グラフィックやその他の資料と一緒に、CDNを使用します。



理想的なスクリプト言語の要件



理想的なスクリプト言語の要件を策定します。



  • ダイナミック。 私たちの理解では、理想的なスクリプト言語は動的でなければなりません。
  • 人気。 言語の人気とは、StackOverflowのような特殊なリソースに関する質問にすぐに答えられる、十分に大きなコミュニティの存在を意味します。
  • 穏やかな学習曲線。 私たちは、比較的言えば、事実上どんな人でも連れて行って、この人が私たちの仕事で生産的に働くことができるレベルに彼をすぐに訓練したいと思います。
  • 豊富な機会。 言語は強力で十分な機能を備えている必要があり、さまざまなプログラミングパラダイムをサポートする必要があります。 そのような言語で書くことを申し出られるプロのプログラマーは、これを快適かつ快く行うことができます。
  • 高性能。 パフォーマンスは、ゲーム業界の基盤の1つです。
  • 多数のライブラリ。 非常に頻繁に、私たちが直面している問題を解決する過程で、根本的に新しいコードを作成するのではなく、他の誰かがすでに書いたものを使用します。 特定の言語を使用して使用できる安定性が高く、十分にサポートされているライブラリほど、優れています。
  • 埋め込みが簡単。 私たちは埋め込み言語について話しているので、スクリプト言語を選択するとき、それを埋め込む機能は重要な役割を果たします。


これらの要件を満たすためのスクリプトとして使用される一般的なプログラミング言語を分析してみましょう。



Python



Pythonは、非常に人気のある動的言語です。 かなり緩やかな学習曲線が特徴で、非常に簡単に学習できます。 しかし、適切に勉強することはそれほど簡単ではないはずです。 その結果、優秀なPythonプログラマーは希少で高価です。 これは、ゲームロジックの開発コストを高速化し、削減したいという私たちの願望と矛盾しています。



Pythonには優れた機能と優れたパフォーマンスがあります。 その問題は、一貫性のないライブラリシステムです。 私たちにとって重要な役割を果たしている彼の問題のもう1つは、彼が実際には組み込み言語ではないということです。 これは、CまたはC ++で記述されたライブラリを呼び出すのに便利な言語です。



Pythonの埋め込み機能に関しては、たとえば、Pythonが使用されているMayaがあると言えます。 しかし、Python Mayaプラグインを内部から見たことがある人なら、見た目があまり良くないことに同意するでしょう。



その結果、Pythonの長所はすべて、私たちに合わないと言えます。 JavaScriptを検討してください。



Javascript



JavaScriptは、誇張することなく、文字通り世界を席巻している素晴らしい言語です。



JavaScriptは、豊富な機能、優れたパフォーマンス、豊富なライブラリセットを備えた、緩やかな学習曲線を特徴とする人気のある動的言語です。



ゲームエンジンを構築するために言語インタープリターが必要な場合、多くのそのようなインタープリターを見つけることができます。 実際には、V8とWebKitの2つのプロジェクトから選択する必要があります。 それと別のものはどちらもかなり大きなサイズです。 その結果、ボードゲームについて話している場合、チャンスをつかんでゲームに通訳者全員を含めることができますが、モバイルゲームの場合、このオプションは適していません。



SocialQuantumには独自のJavaScriptインタープリターがあり、テストの98％に合格しています。このプロジェクトをオープンソースカテゴリに移行する予定です。



その結果、JavaScriptは埋め込み言語の役割の有力な候補のように見えますが、私たちにも適していません。



ハクセ



JavaScriptについて話すとき、次の人は通常Haxeを覚えていることに注意してください。 しかし、実際には、この言語を埋め込み言語として使用する可能性について話すことには意味がありません。実際、Haxeは他の言語へのトランスコンパイラとしての言語ではないからです。 そして、これは私たちが必要とするものではありません。



ActionScriptまたは他のスクリプト言語に満足しているのでしょうか？



ActionScript



上記の要件に準拠するためにActionScriptを正式に分析すると、理想的なスクリプト言語が見つかったように見えるかもしれません。 彼の側には、動的な性質、人気、研究の容易さ、優れた機能、パフォーマンス、ライブラリーの可用性、組み込みの容易さがあります。 この言語はゲーム業界で愛され、記憶されており、膨大な数の素晴らしいFlashゲームが作成されています。 ActionScriptの主な問題は、この言語がほとんど機能しないことです。 したがって、彼は私たちにも似合いません。



AngelScript、Squirrelなど



ActionScriptに加えて、AngelScript、Squirrelなどの多くのスクリプト言語があります。 それらの中には、正式にはほぼ完全に要件を満たしているものがありますが、通常、これらは開発者に結び付けられた言語であり、長年にわたって修正されていない古い問題がいくつかあります。 ほとんどの場合、それらはあまり人気がなく、十分に文書化されておらず、トレーニング資料はほとんどなく、非常に大きなコミュニティはありません。 この状況の結果の1つは、それが完全に明確ではないからといって、それらが何であり、どのように機能するかを研究するのが難しいという事実です。



ご覧のとおり、理想的な組み込み言語はまだ見つかりません。 独自の言語を作成するとどうなりますか？



独自の言語を作成する



社内で開発された言語は、そのニーズに理想的に適合し、習得しやすい可能性があります。 しかし、ほとんどの場合、そのような言語は普及しません。 このような言語には、最小限の数のライブラリがあるか、まったく存在しません。 さらに、現代の状況では、すでに市場に出回っているものよりもうまく機能し、生産性が高く、統合が容易なものを作成できると信じることは困難です。



独自の言語を開発して使用する企業があり、その中にはゲーム市場で成功しているプレーヤーがいますが、これはおそらく良い考えではありません。



埋め込まれていると主張する既存のプログラミング言語を調べ、独自の言語を開発するというアイデアについて議論したので、Luaに進みましょう。



Luaは選択した組み込み言語です



Luaは動的言語です。 これは非常に人気があり、特にゲーム開発の分野では、大規模なコミュニティがその周辺で発展しています。 それは非常に穏やかな学習曲線を持っています。 たとえば、当社では、自動テストのスクリプトはLuaで記述されています。 自動テストの標準的な入門コースには約2時間かかります。その後、この言語を使用できます。 同時に、Luaはマルチパラダイム言語です。 関数型プログラミングスタイルとOOPをサポートしています。 その結果、いくつかの単純なタスクを解決するだけでなく、プロのプログラマーが行うより深刻な問題にも適しています。



Luaのパフォーマンスは良好であり、かなりの数のライブラリがあります。 JavaScriptほどではありませんが、それにもかかわらず、 LuaForgeの Webサイトでは、必要なほぼすべてのものを見つけることができます。 最後に、Luaは非常に簡単に埋め込むことができ、さらに埋め込み言語として使用するように設計されています。



たとえば、JetBrains IDE CLionワークスペースは次のようになります。 ここでは、オープンソースを作成する予定のLua用に作成されたオートコンプリートメカニズムを見ることができます。 オープンソースデバッガを作成します。















作業環境



Luaを選択しましたが、埋め込み可能なスクリプト言語として使用することになると、通常、これから検討するのと同じ異論に出くわします。



Luaの使用に対する異議



LuaはC ++用ではなくC用です



Luaが埋め込み可能な素晴らしい言語であると主張する人はいません。 彼らがそれをマイナスと考える主なことは、C ++ではなくC言語で使用するために作成されたことです。 このため、CではなくC ++であるLuaで何かを適用しようとすると、問題が発生します。 ただし、ここでは、多くのかなり賢い人がこれらの問題を解決したことを理解する必要があります。 LuaをC ++に埋め込む問題を解決するツールには、Luabind、Luabridge、toLua ++、SQLuaHostなどのプロジェクトがあります。 これは完全なリストではありません。 これらにはさまざまな長所と短所がありますが、それでも、ほとんどの場合、必要なものはすべてこれらのソリューションのいずれかに既に実装されています。



たとえば、SQLuaHostを検討してください。 これはSocialQuantum内で行われるバインディングであり、オープンソース化される予定です。 この決定は、Luaがどのようにバインドすべきかというビジョンを表しています。 したがって、既存のバインダーで必要なものが見つからなかった場合は、SQLuaHostで見つけることができます。



ルアは遅い



Luaは非常に遅い言語であるという意見にしばしば遭遇します。 まず、これはそうではありません。 Luaは積み重ねられたマシンであり、実際に速度を落とすものは何もありません。 さらに、スクリプト言語では通常、ゲームロジック、ビジネスロジックを提供し、実際に重いものは提供しないことを理解する必要があります。 その結果、Luaスクリプトがゲームの速度を低下させる場合、問題は、一部の言語機能の最適でない結合または不適切な使用にある可能性があります。 たとえば、LuaJITがMonoより速い合成テストを実施しました。 同時に、そのような非最適なコードについて誰も書くことを気にしません。 



function myGame:onUpdate()    local tex = Texture::new(name)    self.background:setTexture(tex) end


ここでは、各ゲームティックで、新しいテクスチャが作成され、背景として設定されます。 もちろん、そのような設計は特に速く動作しませんが、誰もそのようなことを書くことを気にしません。



Luaは小さなプロジェクトにのみ適しています。



次の異論は、Luaは小さなものを書くように作られており、この言語では大きなものを書くことができないということです。 一方でそれは本当です。 しかし、この言語は高度にモジュール化されています。 そして、多くの小さなブロックから、非常に大きく複雑なシステムを作成できます。 そして、マルチパラダイムとOOPについて既に述べたことを思い出すと、OOPは開発者に、大きく複雑な構造を作成するために使用できる小さなモジュールを作成するように促すことがわかります。



さらに、多くの場合、Luaではいくつかの小さなモジュールが迅速に記述され、ゲーム業界では「高速」とは「安価」を意味します。



Luaに対するその他の議論



Luaを批判して、彼らはその言語は古代であり、「箱から出して」と言うように、OOPをサポートしていない、テーブル内の要素の番号付けは0で始まらないと言います。 1。









彼らは、マイナスは三項演算子を持たないということです。 実際、Luaに対する多くのそのような議論がありますが、それらの大部分は開発者の習慣や個人的な好みに関連していると考えているため、それらについては説明しません。



まとめ



まとめると。 最小限の労力で埋め込み言語を取得することがタスクの場合は、Luaを利用してください。 同時に、あなたがあなた自身の言語またはあなた自身のバインディングを開発する時間とリソースを持っているなら-再び-Luaに注意を払ってください。 最初と2番目の両方のケースでLuaを推奨するのはなぜですか？



最初のケースでは、Luaを選択すると、埋め込みと使用が非常に簡単な言語を選択できます。 著者によって書かれたこの言語に関する教科書は1 冊だけです。 最初の本があなたがルアについて知る必要があることを絶対にすべて伝えているという理由だけで他の本はありません、そしてそれについて話すことはこれ以上ありません。 Luaは理想的な言語でも世界で最も普及している言語でもありませんが、基準の合計により、埋め込みに最適な言語の1つであることは間違いありません。 彼は私たちが今持っている最高のものです。 さらに、Luaを使用する人々の生活を楽にする多くの標準ツールがあります。



2番目のケースでは、ツールを開発するためのリソースがあり、Luaを選択した場合、これらのリソースを実際に使用できます。Luaはゲーム開発環境での人気にもかかわらず、この言語は非常に過小評価されています。 その結果、Luaを基礎として、ニーズを考慮に入れ、必要なものを正確に取得する機会が得られます。
