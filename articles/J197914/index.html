I2P Outプロキシを構成する

こんばんは、親愛なるhabrayuzery。



今日は、猛吹雪の中にとどまり、熱と遮断を維持し、世界中の情報の自由、統一、平和の原則を神聖に守ることができる単一のプロキシサーバーの作成について話し合うことをお勧めします。 はい、はい、私は再びi2pについて話しますが、サーバー設定の観点からです。



海外に良いi2pルーターを常に持っている良いサマリア人がいて、proxy.antizapret.i2pを作成して人々に喜びをもたらす準備ができていると想像してください。





この記事はtornaによって書かれ、彼は出版されるように頼んだので、私はあなたに彼にすべてのプラスを転送するように頼みます。




驚くべきことに、この種のトンネルを作成することは初歩的でした。 このために必要なもの：

  • squid-プロキシサーバー
  • i2pルーター




squid.confに追加

acl block_ip dst 93.158.134.198＃ユーザーがこのIPにアクセスしたい場合は、プロキシを使用します（禁止アドレスのリスト全体が含まれる別のファイルを作成できます）

http_access allow block_ip＃必要なIPアドレスのみを許可

http_accessすべて拒否＃他のすべてのユーザーを禁止

http_port 3128＃i2pルーターのポートを設定します





i2pをロードし、トンネルのセットアップを開始します。

  • ページhttp://127.0.0.1:7657/i2ptunnel/wizardでサーバートンネルを選択します
  • トンネル型標準
  • あなたの裁量で説明とタイトル
  • 住所：127.0.0.1
  • ポート：3128
  • 自動起動チェック




現在、i2pでブロードキャストしているプロキシサーバーは、同様のタイプのb32アドレスを作成します。 リンクを読み取り可能な形式にするには、アドレス帳で作成し、ほとんどのルーターがネットワーク上で変更を加えるまで1週間待つ必要があります。 その後、プロキシの広告を開始できます。



ユーザー設定。




ユーザーがプロキシサーバーを使用できるようにするには、同じページhttp://127.0.0.1:7657/i2ptunnel/wizardにプロキシサーバーを登録する必要がありますが、今回はクライアントトンネルを選択します。

  • HTTPトンネルタイプ
  • 名前と説明は想像​​力に依存します。
  • トンネル宛先アドレス：zrucncbt6qlocnzqobb7nddtp2kexwlkfabgvqlhizxcnp4qdv6q.b32.i2pまたはアドレス帳からの名前
  • ポート：3123
  • コンピューターが単独の場合、192.xx.xx.xxでない場合は、127.0.0.1で始まるネットワークインターフェイスアドレスを選択します
  • 自動起動チェック




プロキシとvpnは近い将来禁止されるため、特定のIPからの常に暗号化されたトラフィックはばかげて見えるため、トラフィックを自動化する必要があります。 クライアントpacファイルを作成します。

この.pacファイルのソースをhttp://antizapret.prostovpn.org/で確認しました。所有者に感謝します。このリソースの所有者が良いサマリア人になることを願っています。 



function FindProxyForURL(url, host) { blockedarray = [ "93.158.134.198", ]; if (blockedarray.indexOf(dnsResolve(host)) != -1) { return "PROXY 127.0.0.1:3123"; } return "DIRECT"; }
