Windows 10のGNS3環境でインターネットにアクセスできるネットワークを作成する

GNS3（グラフィカルネットワークシミュレーター）は、ルーターや仮想マシンから仮想ネットワークをシミュレートできるグラフィカルネットワークシミュレーターです。 トレーニングとテストに不可欠なツール。 ほとんどすべてのプラットフォームで動作します。 デスクトップマシンでスタンドを作成するのに最適です。



GNS3を使用するハードウェアプラットフォームに応じて、Ciscoルーター、Cisco ASA、Juniper、およびネットワークオペレーティングシステムを実行するサーバーで構成される複雑なプロジェクトを構築できます。



GNS3には2つの重大な欠点があります。



  • CPUとメモリを非常に要求します。 10台のルーターがPCに深刻な負荷をかけます。 CPU使用率は、アイドルPCエンジンを使用して削減できます。 これがなければ、3-4はほとんどなくなっていただろう。
  • L2機能を非常に弱くサポートします。 ルーターのアクセス/トランクポートとスイッチボードを構成できるスイッチのごく一部しかありませんが、そのL2機能も非常に制限されています。


さらに、iOSの画像をどこか別の場所に取得する必要があります。 GNSはほぼ仮想ルーターであり、それを実行するにはソフトウェアであるiOSも必要です。



プログラムウィンドウとその構造を以下に示します。







このタスクはエミュレーター内のネットワークモデリングの制限を超えているため、GNS3とさらに対話するためにオペレーティングシステムを準備する必要があります。 これを行うには、外部データ転送プロトコルではなく、GNS3と相互作用するネットワークを構成する必要があります。 コンピューターに接続されているすべてのネットワークは、システムの外部にファイルを転送することを目的としています。 マシンはファイルをビットに変換し、パケットで収集し、マークし、TCP / IP経由でネットワークに送信します。 受信マシンはネットワークと対話せず、送信されたパケットと対話します。 彼女は同じTCP / IPプロトコルを使用してすべてをアンパックし、ファイルシステムを介してユーザーにファイルを提供します。 私たちのタスクは、コンピューターが内部のネットワークをシミュレートし、ネットワーク上の独自の要求に対する応答を分析するような方法でトラフィックを配置することです。 タスクは簡単なものではありませんが、複雑な事実-Wi-Fiもあります。 ケーブルネットワークは、ワイヤレスネットワークと同様にエミュレータ上で簡単に構築でき、伝送を簡素化する特定の原則に従って機能するネットワークをエミュレートするのはそれほど簡単ではありません。 この問題を解決するには、ループバックアダプターを作成します。 これは自分自身に対処するネットワークです。 Windows Win + Rでループバックネットワークを作成してhdwwiz.exeと入力するには











「次へ」ボタンを押して、「リストから手動で選択した機器をインストールする」項目を選択します。 このラボではWindows 10オペレーティングシステム（64ビット）を使用しました。これは最新かつ最も関連性があるためです。







[次へ]ボタンをクリックし、[ネットワークアダプタ]を選択します







「次へ」ボタンをクリックし、左側のメニューで「Microsoft」を選択し、右側で「Microsoft KM-TEST loopback adapter」を選択します







次に、インストールを実行し、アダプター設定に進みます。 これを行うには、[スタート]-> [コントロールパネル]-> [ネットワークとインターネット]-> [ネットワークと共有センター]-> [アダプター設定の変更]に移動します。



次に、ルックバックネットワークがインターネットにアクセスできるようにシステムを構成する必要があります。 これを行うには、ワイヤレスネットワークのプロパティに移動し、[アクセス]タブで、[他のネットワークユーザーにこのコンピューターのインターネット接続の使用を許可する]および[他のネットワークユーザーにインターネット接続への共有アクセスの制御を許可する]をオンにします。 複数のネットワークがある場合、この方法は他のネットワークでは機能しないため、作成した閉じたネットワークを選択します。







すべてを正しく行った場合、次の復号化を備えた2つのネットワークが必要です。







私の場合、「Wireless Network 2」はWi-Fiネットワークであり、「Ethernet 3」は閉じたネットワークです。



システムの準備作業は終了しました。 各ユーザーには独自のバージョンがあり、各システムには独自の安定したルーターがあるため、ISOイメージをGNS3にインストールする方法は説明しません。 私のバージョンでは最も安定したルーターであるため、バージョン0.87とc3660ルーターを使用しました。



ルーターとクラウドをワークスペースにドラッグして、下図のようなスキームを取得します







クラウドをセットアップします。 [NIOイーサネット（ネットワーク入出力イーサネット）]タブに移動し、閉じたネットワークを選択して[追加]ボタンをクリックし、[OK]をクリックします。







クラウドとルーターをFastEthernetケーブルで接続し、緑色の[再生]ボタンを押してルーターを起動します。



ルーターを設定しましょう。 コマンドのシーケンスを紹介します。 



conf t int fa 0/0 no shut ip address dhcp ip domain-lookup


各チームを分析します。

conf t（グローバル構成モード）

int fa 0/0（インターフェイスコンフィギュレーションモード）

no shut（インターフェイスを有効にするコマンド）

ip address dhcp（インターフェイスがDHCPを介してIPアドレスを受信して​​いることを示します）

ip domain-lookup（ホスト名をIPアドレスに解決する、またはその逆）
