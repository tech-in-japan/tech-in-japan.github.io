GPSレシーバーをラップトップに接続し、GPSトラッカーを作成する

多くの旅行愛好家は、余暇を過ごすときに長い間GPSデバイスを使用し始めています。

誰かがナビゲーターを使用して、原則としてベクトルマップ付きのナビゲーションプログラムを使用して、最寄りの高速道路/道路に沿ってある集落から別の集落に移動します。 そして、誰かがより極端な娯楽を好み、オフロード車の最も近いルートに沿ってポイントAからポイントBに移動します。 後者の場合、ベクトルマップはそれほど重要ではありません。ここでの主なことは、最も近いエリアの起伏の特徴を知ることであり、座標に関連付けられたラスターマップを持つナビゲーションプログラムが使用されます。

外部GPSレシーバーをラップトップに接続することを目標に設定します。ラップトップには、ベクターマップとラスターマップを操作するためのさまざまなプログラムをインストールして、便利なルート計画ツールを取得し、現在の場所を表示できます。 同時に、GPSデータをサーバーに転送して、現在地をリモートで追跡できるようにしたかった（即席GPSトラッカー）。

したがって、このトピックに興味がある場合は、このトピックへようこそ。



私たちは何を持っています ：

  • GPSコミュニケーター
  • ラップトップ
  • 3Gモデム（除外可能）




特定のソフトウェアをラップトップにインストールする必要があります。

  1. PDAをPCと同期するには（私の場合、「ActiveSync」-Windows XPの場合、または「Windows Mobile Device Center」-Windows 7、Vistaの場合） ;
  2. ナビゲーションプログラム。 たとえば、OziExplorer （ラスタマップ用） 、SASPlanet;
  3. 受信機からのGPS信号を処理するためのプログラム。


追加のアプリケーションもPDAにインストールする必要があります。

1）GPSGate。



まず、外部デバイスからラップトップにGPS信号を取得しましょう。

ここでは、回路は簡単になります。

  1. GPS信号はCOM2ハードウェアポート（私の場合）を介してPDAに送信されます。
  2. PDAはActiveSyncを使用してラップトップと同期されます。
  3. PDAにインストールされたGPSGateアプリケーションを使用して、GPS信号はCOM2からActiveSyncポートにリダイレクトされます。
  4. ラップトップにインストールされたGPSGateプログラムは、ActiveSyncポートから信号を受信し、それを仮想COMポート（私の場合はCOM6）にリダイレクトします。
  5. ラップトップにインストールされているナビゲーションプログラムでは、GPSGateで作成された仮想ポートをGPS信号のソースとして示します。






それでは、理論から実践に移りましょう。



GSPレシーバーのセットアップ



GPSGateプログラムはすでにPDAにインストールされていると想定しています。



プログラムを構成します 。



  1. [ 入力 ]タブは、GPS受信機のハードウェアポートを示します。 PDAでは、これは通常COM2ポートです。 [ コントロールパネル]-[外部GPS-機器]に移動して、明確にすることができます。
  2. [ 出力 ]タブで、GPS信号の受信者が示されます。

    ここでは、受信したGPS信号がPDAとラップトップの接続ポートにリダイレクトされるように、ドロップダウンリストから「 ActiveSync 」を選択します。
  3. これで、「 オンライン 」メニューを選択できます。 アイコンは緑色（点灯）に点灯するはずです。これは、衛星が現在接続されていることを意味します。


座標の送受信手順が成功すると、パネルのアイコンが緑色に点灯します。 デバイスの準備ができているが、座標がまだ受信されていない場合（衛星が捕捉されていない場合）-プログラムアイコンが黄色に点灯します。



これで、PDAからラップトップへのGPS信号の転送が実現しました。



ここで、 GPSGateサーバーへのGPS信号の送信を複製して、位置を監視できます（サイトでの事前登録が必要です） 。

これを行うには、「 出力 」タブにアイテム「 gpsgate.com（送信） 」を追加し、サーバーに接続するための設定でgpsgate.com Webサイトのログイン/パスワードを指定します。 PDAからサーバーへのデータ送信は、セルラー通信によって実行されます。 3gモデムをお持ちの場合は、プログラムで同様の設定を行うことができます

ラップトップにインストールされたGPSGate。





ノートパソコンのセットアップに移動



ActiveSync、GPSGate、SASPlanet（またはGoogle Earth）がすでにインストールされていることを前提としています。



  1. GPSGateの設定で、「 入力 」-ActiveSyncを選択します（GPS信号のソースにはActiveSyncがあるため） 。
  2. [ 出力 ]で[ 仮想COMポート]を選択します（GPS信号の受信機は、 COM-6などの仮想ポートになります）。
  3. SASPlanet プログラムの設定 ： パラメーター-プログラム設定 -GPS

    COMポートで、仮想COMポートCOM-6を指定します。








これでメインプログラムの設定は終了です。



SASPlanetには多数のマップ（Yandex、Navitel、Googleなど）があります

ソースの設定では、「 インターネット+キャッシュ 」を指定することをお勧めします。そうすると、既に表示されているマップはキャッシュ（ラップトップのメモリから）からダウンロードされ、新しいマップはインターネットからダウンロードされます。

したがって、トラフィックを節約するために、旅行で訪問する予定の可能な最大セクションを事前にプレビューすることは、より論理的です。



結果は何ですか？

GPS信号を外部デバイスに受信し、ラップトップに転送します。ラップトップでは、GPSデータが処理され、SASPlanetプログラム（他の類似のもの）のラスターまたはベクトルマップに関連付けられた現在の位置を取得します。

これと並行して、移動と現在の場所に関するデータがサーバーに送信されます。 T.O. 特定の期間、場所とルートをリモートで表示できます。





何のために？

私はチームのアクティブなシティゲーム中にこれを使用します。

1）本社は、すべての乗組員の動きを同時にリモートで制御および調整します。

2）ラップトップのナビゲーションプログラムは、ポータブルGPSデバイスよりも幅広い機能と可視性を備えています。 ルートをすばやく計画して調整し、多くのマップから最適なものを選択できます。



多くのオフロード旅行者は、GPSデバイスをよりコンパクトなネットブックに関連付け、通常は地形に関する詳細情報を含むラスター地形図でOziExplorerプログラムを使用します。



また、古典的な観光のファンは、Googleマップに表示される近くのアトラクションを見ることができる大きなインタラクティブマップで自分の動きを観察することにもっと興味を持つでしょう。



旅行中、ルートを追跡できる親relativeや友人と現在地を共有できます。
