Telegramでのチャットと検索の分離

私の以前の出版物では、電報で5つの物議を醸すUX / UIソリューションを説明しましたが、それはかなり曖昧であることが判明しました（16対13の評価を見なければなりません）。 digりの主な理由は、リストにチャットの共有がないことで、最初は戸惑いがちでしたが、後にすべてを慎重に再考した後、問題を理解し、最善の解決策を見つけようとしました。













一定の時間まで、アプリケーションでチャットが分離されていないことで何も問題はありませんでした。追加のタブまたはグループを追加すると、ユーザビリティに悪影響が出るだけでした。 友人、知人、サブスクライバーの口頭および書面による調査を実施し、さらに新しいボットとチャンネルサブスクリプションを追加した後、私はその反対を確信しました。







分離が必要です。 しかし、それを最も有能に実装する方法は？ 個人的には、テレグラムチームが1つのシンプルなメッセージ画面を作成するというコンセプトが好きです。メッセージ画面は、情報の関連性によって自動的にソートされ、近い将来それを拒否することはほとんどありませんが、適切なチャットを探して常に「情報ダンプ」を探すこともありません出口。







どうする？ 私たちの目標は、インターフェース構造とユーザーエクスペリエンスの最小限の変更でチャットソートを導入することです。







この問題を解決するには、まず、このソートを行う目的を理解する必要があります。 チャット間でのユーザー指向を改善し、チャットにすばやくアクセスするために必要です。 言い換えれば、すばやく検索し、それに応じて必要なチャットを選択するには、並べ替えが必要です。







タブ間をスワイプするオプションはすぐに消えました。 あまりにも多くの機械的アクションを実行する必要があります。 メインページのグループもオプションではありません。







そして、「検索画面をやり直し、ドロップダウンリストを使用してチャットをグループに分けて実装してみませんか？」というアイデアが思い浮かびました。 このように、すべての人に馴染みのある画面を「連続したすべて」で保存すると同時に、ユーザーが目的のチャットをすばやく見つけて、誰もが欠けているチャットを操作できるようにします。 また、重要なチャットのリストをグループ化して作成する機会を彼に与えます。 次のようになります。













もう一度。 これは、ホーム画面に代わるものではありません。 これはやり直し検索画面です。







ご覧のとおり、検索機能はなくなりました。重要度の低い「最近」セクションは、以前の「チャット」セクションの形式に変換され、常に画面の下部に接着されます。 チャットはグループに置き換えられました。 ユーザーが何らかのチャットを失った場合、2タパスですばやく見つけることができます。







グループに必要な機能：



  1. ドロップダウンリスト、名前、チャットを追加および削除します（このためにフローティングボタンを作成できます）。
  2. リスト内でチャットを上下にドラッグ（ユーザーによるチャットの優先順位付け）。
  3. 開いたドロップダウンリストは、アプリケーションが完全に終了した後でも最小化しないでください。ユーザーのみがその状態を操作します（この方法により、必要なリストを常に開いたままにして、その中のチャットルームにすばやくアクセスできます）。
  4. 未分類リストには、ユーザーがグループ化していない他のすべてのチャットが含まれている必要があります。


もちろん、これらは単なる概略図であり、理想とはほど遠いものですが、一般的にはアイデアとコンセプトは明確です。







おわりに



全体的に、ユーザーが望んでいる電報でチャットを共有できるようにユーザーの機能を概念的に導入しましたが、ユーザーインターフェイスとユーザーエクスペリエンスを根本的に変更することはありません（人はインターフェイスの最小限の変更にも慣れる必要があります）、これは電報チームにとっても私たちにとっても有益です。ユーザー。
