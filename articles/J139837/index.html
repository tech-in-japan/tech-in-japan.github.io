Ensemble 2012.1の新しいバージョン





インターシステムズの新しい統合プラットフォーム（ESB）リリース-Ensemble

このドキュメントの完全な英語版は、インターシステムズのWebサイトで入手できます。



前のバージョン（2010.2）と比較して、Ensemble 2012.1には多くの新機能が含まれています。

  • 新しい管理ポータルユーザーインターフェイス
  • 管理ポータルの詳細なセキュリティモデル
  • ビジネスルールの新しい実装
  • Control Portalのバージョン管理システムとの統合
  • レコードマッパー
  • レコードファイルのバッチ処理
  • 仮想XMLドキュメント
  • Ensembleメッセージを処理するための指定されたパフォーマンスしきい値に達したときにアラートを出します
  • UDDI API
  • Caché2012.1の新機能




1.1新しいEnsembleユーザーインターフェイス



Ensemble 2012.1のユーザーインターフェイスは、Ensembleプラットフォーム上の統合ソリューションの構成と管理に関連するすべてのものが完全に再設計されています。 管理ポータルには、次の新機能が含まれています。



  • よりモダンで直感的なWebインターフェース

  • 外部システムの構成を簡素化する改善されたウィザード

  • 新しいビジュアルエディターDTL（データ変換言語）およびBPL（ビジネスプロセス言語）

  • 新しいビジネスルールエディター

  • BPLビジネスプロセス、DTL変換、レコードパーサーマッピング、ビジネスルールなど、管理ポータルで開発および構成されたEnsembleコンポーネントにバージョン管理システムを使用する機能

  • ユーザーインターフェイスコンポーネントへのアクセスを制限する機能。 事前定義されたロールのセットは、Ensemble機能へのアクセスを制御します。 さらに、既存のユーザーロールを変更するか、独自の新しいロールを作成することにより、管理ポータルへのアクセスを柔軟に構成できます。



また、新しいバージョンでは、さまざまな領域でのWebページの重複が回避され、インストールが簡素化され、多数の領域があるシステムのバージョンをアップグレードする時間が短縮されます。

インターフェイスの変更と管理ポータルでのEnsembleの使用方法の詳細については、 Ensembleの管理ドキュメントを参照してください。



1.2管理ポータルの詳細なセキュリティモデル



Ensembleのこのバージョンでは、管理ポータルのさまざまなコンポーネントおよびEnsembleのさまざまな機能へのアクセスをより柔軟に制御できる追加の役割が追加されました。

事前定義されたロールが想定されたセキュリティモデルの実装を許可しない場合、必要なレベルのセキュリティを取得するために、ロールを追加したり、既存のロールの動作を再定義したりできます。

Ensembleを更新する場合、互換性の詳細については、ドキュメント「管理ポータルの新しいセキュリティモデル」を参照してください。

セキュリティモデルの詳細については、 Ensembleの管理の管理ポータル機能へのアクセスの制御の章を参照してください。



1.3ビジネスルールの新しい実装



以前のEnsembleリリースでは、ビジネスルールとルーティングルールは互いに分離され、XMLファイルでした。

新しいバージョンの両方のタイプのルールは、Ens.Rule.Definitionから継承されたクラスとして定義され、ルールセットはXDATAブロックに記述されています。

この変更により、ビジネスルールおよび製品の他のコンポーネント（クラス、コンパイル）との連携が可能になり、バージョン管理システムにビジネスルールを透過的に含めることができます。 これに加えて、ネストされたルールとネストされた条件のサポートがXMLルール定義に追加されました。 さらに、ルールのグラフィックエディターが完全に再設計され、これらの機能のサポートが追加されました。



1.4管理ポータルのバージョン管理



これまでStudioにのみ組み込まれていたバージョン管理システムのフックに、管理ポータルからのフックが追加されました。 したがって、このバージョンでは、コントロールポータルのビジュアルエディタを使用して構成されたEnsembleコンポーネントについて、バージョンコントロールシステムを接続できるようになりました。

詳細については、 Ensembleプロダクションの構成ブックの「ソース管理設定の構成 」セクションを参照してください。



1.5レコードマッパー



情報システムでデータを交換する非常に一般的な方法は、固定長フィールドまたは区切りフィールド、ネストされたフィールドのいずれかを含む特定の構造のレコードファイルです。

新しいEnsembleツール（視覚的なレコードマッパー）を使用すると、ファイルレコードの構造を記述し、そのようなファイルを読み取りおよび変更するためのコンポーネントを作成できます。

Record Mapperは、便利なグラフィカルインターフェイスでユーザーが指定したファイルレコードの構造を反映する一連のプロパティを自動的に保存するクラスを作成します。

事前定義されたビジネスサービスはこのようなファイルから読み取られ、ビジネスオペレーションを使用すると、開発者が定義した形式でそれらに書き込むことができます。

詳細については、ドキュメント「Ensembleレコードマッパーの使用 」を参照してください 。



1.6レコードパッケージの処理



レコードマッパーに関連付けられたレコードバッチ処理メカニズムにより、ファイルレコードの処理を制御できます。 この機能により、たとえばレコードの受信時間を考慮に入れて、さまざまな条件に応じて1つまたは複数のファイルにレコードをバッチ出力できます。 Ensembleレコードマッパーの使用の 「バッチ処理と処理」も参照してください。



1.7 XML仮想ドキュメント



このバージョンより前のEnsembleには、XMLドキュメントを処理する2つの方法がありました。

1. XMLドキュメントを対応するEnsembleオブジェクトにマッピングします。これは、DTL変換メカニズムを使用して処理できます。 これは強力で便利なXML処理ツールですが、フィールドのごく一部しか処理されない複雑な構造を持つ大きなドキュメントにはあまり効果的ではありません。

2. XPATHまたはXSLTを使用してXMLをストリームとして処理し、ドキュメントを読み取り、変更します。 これは強力なアプローチですが、技術的に複雑であり、プログラマーの特定のスキルが必要です。

新しいバージョンには、仮想XMLドキュメントという3番目の処理オプションがあります。 このアプローチにより、Ensembleでは、XMLドキュメントをストリームとして操作できますが、同時にDTLエディターとルーティングルールを使用できます。 これは、XPATHまたはXSLTを使用するよりも簡単な方法であり、XMLをオブジェクトにマッピングするよりも安価です。

詳細については、 Ensemble XML仮想ドキュメント開発ガイドを参照してください。



1.8パフォーマンスコントロールアラート



Ensembleの新しいバージョンの既存のタイプの通知に加えて、システムのキュー処理パフォーマンスとメッセージ処理時間のパラメーターを設定できます。 キューが長すぎる場合、またはメッセージの処理に非常に長い時間がかかる場合、システムは管理者にアラートを送信できます。 詳細については、「 Ensembleプロダクションの監視 」ドキュメントセクションの「アラートの設定、処理、および監視」の章を参照してください。



1.9 UDDI APIサポート



Universal Description、Discovery and Integration（UDDI）仕様は、Webサービスおよびその他の電子および非電子サービスの登録サービスを定義しています。 このEnsembleリリースには、UDDIレジスタAPIを使用するためのEnsLib.UDDIパッケージが含まれています。 クラスリファレンスのEnsLib.UDDI.Baseエントリも参照してください。



1.10Caché2012.1の機能



Ensemble 2012.1は、Caché2012.1のコアで実行されます。 したがって、Ensemble 2012.1のリリースには、Caché2012.1のバージョンに伴うすべての変更も含まれています。



その他の改善



2.1 DeepSeeパネル



新しいEnsembleでは、インジケータダッシュボードはDeepSee Zenコンポーネントを使用して実装されます。 パネルの外観が改善され、Ensembleデータを視覚化する新しい機会が登場しました。 詳細については、「ダッシュボードへのビジネスメトリックの追加」の「「 Ensembleでのダッシュボードの使用 」セクション」を参照してください。



2.2 ebXMLメッセージング



EnsembleはebXMLサポートを拡張します。 特に、英国国民保健サービスとの相互作用をサポートするために、追加のメッセージパターンが実装されています。



2.3シーケンスマネージャー（HL7シーケンスマネージャー）



HL7シーケンスマネージャーは、本番環境で複数のシーケンスを解決するようになりました。 詳細については、 Ensemble HL7バージョン2開発ガイドの 「ルーティングプロダクションの要素」の章の「HL7 Sequence Manager」セクションを参照してください 。



2.4製品で使用されるすべてのモジュールを含むプロジェクトをスタジオで作成する



製品からStudioでプロジェクトを作成する機能は、あるシステムから別のシステムへの製品のエクスポート/インポートに役立ちます。 この新しい機能を使用すると、HL7スキームをプロジェクトに自動的に含めることができ、製品の要素によって参照されるさまざまなクラスを含めることができます。 Ensembleプロダクションの開発ドキュメントの「Ensembleでのスタジオの使用」セクションも参照してください。
