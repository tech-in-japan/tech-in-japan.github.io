ReportPortal quick deployment for familiarization

Now step by step (the whole process with good internet takes 15 minutes):



1. Open the Hyper-V manager:







2. Right-click on the name of your computer and select '' Quick create "







3. In the window that opens, select Ubuntu 18 ... and click "Create virtual machine"







4. We go to make tea and wait until the image is downloaded and unpacked







5. When the process of downloading and unpacking is completed - we will see such a window.







We are not in a hurry to launch the virtual machine, but let's go to the settings by clicking on the ‚ÄúEdit settings ...‚Äù button



6. Since the reportport is a very voracious thing - we‚Äôll give the virtual machine more memory (I gave 6 GB)







7. Save the settings and click on the blue ‚ÄúConnect‚Äù button. The virtual machine has not started yet, we just connected to it (aka remote desktop). Here we already press "Start":







8. When our Ubuntu booted because the system was not installed in the usual way, we need to enter data such as the system language, time zone, username and password.







9. That's it, our Ubuntu up and running.

3. Install the reportport



3.a Download docker-compose.yml:

3.b Optional. By default, the reportport ‚Äúlistens‚Äù to port 8080. I needed this port for another service (selenoid-ui), I decided to ‚Äúgive‚Äù the reportport a different port.



To do this, edit docker-compose.yml: 



 nano docker-compose.yml


We are looking for a section: 



 gateway: image: traefik:1.6.6 ports: - "8080:8080" # HTTP exposed # - "8081:8081" # HTTP Administration exposed command: - --consulcatalog.endpoint=registry:8500 - --defaultEntryPoints=http - --entryPoints=Name:http Address::8080 - --web - --web.address=:8081 restart: always


We edit it as follows: 



 gateway: image: traefik:1.6.6 ports: - "<NEW_PORT>:8080" expose: - "8080" # - "8081:8081" # HTTP Administration exposed command: - --consulcatalog.endpoint=registry:8500 - --defaultEntryPoints=http - --entryPoints=Name:http Address::8080 - --web - --web.address=:8081 restart: always


where <NEW_PORT> is the port number that you want to give the reportport. I entered "8888"



3s Install the reportport: 



 docker-compose -p reportportal up -d --force-recreate


Images will be downloaded, docker containers will be created and launched by themselves. After the whole process in the console is over - wait a minute - let everything start as it should and in your browser on your real machine you can open a report on the IP of the virtual machine and the port you selected (or standard 8080).







Log in to the built-in admin: `superadmin \ erebus`







In conclusion: Now we can just close the window with the virtual machine (it will remain in the background and will not be an eyesore). To open it again, we return to the Hyper-V manager and double-click on the virtual list in the list.
