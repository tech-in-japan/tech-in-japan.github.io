V.ビジネス分析



問題、その所有者および原因


問題の説明は、次の項目に関する情報を構造化するために表を使用して簡単に説明されます。

  1. 問題。
  2. 誰が影響を受けます。
  3. その結果です。
  4. 新しいシステムからの勝利。
  5. 優先順位
すべての問題を説明した後、最も重要なものを強調表示するために、優先度の降順でソートする必要があります。

  • 問題、その所有者および原因
依存関係分析


依存関係分析-原因と結果の連鎖のイメージ、前提条件と望ましくない影響の識別。最終的に問題につながります。 問題の分析を部分的に繰り返しますが、対照的に、どの相互接続が存在するかを理解するのに役立ちます。

依存関係の分析からわかるように、可能なソリューションは電話予約のより便利な代替手段を提供し、理想的には購入者の連絡先の詳細を使用する必要があります。



特徴


最後に、どの機会に利害関係者がいるかを判断する必要があります。



映画チェーンマネジメントの代表者は、次の機能について話しました。

  1. チケット、ポスター、小冊子でafisha.ruを宣伝する機能。
  2. 外部APIを備えた電子予約システムがあります-UCSプレミア （このサイトの名前は「予約」モジュールです）。
  3. 忠実な視聴者のための割引とボーナスのシステムの作成。
LLC Company Afishaの側からは、次の可能性があります。

  1. チケット予約システムを開発する能力。
  2. セッションの変更に関するアラートで視聴者の不満を減らします。
  3. シネマシネマネットワークにビジターのストリームを提供します。
  4. サイトとの統合のためにUCSプレミア APIを使用する機能。
  5. 視聴者のロイヤルティを追跡し、統計を収集する機能。
したがって、ビジネス分析を行った後、次の理解を作成する必要があります。

  1. 現在の状況は？ 私たちは何を扱っていますか？ 問題は何ですか（一般的に）- コンテキスト 。
  2. 主な利害関係者は誰ですか、彼らのニーズ、動機、関心は何ですか？ - 利害関係者のプロファイル 。
  3. 既存の問題は現在どのように解決されていますか？ - ビジネスプロセスの説明 。
  4. どの分野を扱っていますか？ 制限は何ですか？ 主なエンティティとその関係は何ですか？
  5. 各関心のある人はどのような問題を抱えており、提案された問題解決方法はどのような問題を彼らに与えますか？ - 問題の分析 。
  6. 利害関係者は、既存の問題を解決するためにどのような機会を持っていますか。
これらの質問に答えた後、ビジネス設計に進むことができます-エージェントの問題をどのように解決するかについての決定を下します。提案されたソリューションは収益性がありますか、主な目標は何ですか？
