Intel Parallel Studioは無料ですか？

景品！ なんて不穏な言葉か、むしろ思考。 この考えから、心臓はもう少し頻繁に鼓動し始め、脳はすぐに算術計算を行います。 私にとって、この感覚はよく知られています。特に、アメリカのVolmartsで有名なメーカーの16ギガバイトのUSBフラッシュドライブをたった19.99ドルで購入したときです。 彼は2つを取りたいとさえ思ったが、彼の考えを変えた。 私は彼女をまったく必要としないので、今私はこれを私の友人に提示しました。 しかし、「有用な」デバイスを私たちが見つけられるよりも2倍低い価格で購入できたという事実からの「深い満足感」は、通常のアメリカの価格と比較して半分でした。



それでは、フリーソフトウェアについて話しましょう。 今年は、さまざまなIntelツールに関する非常に多くのセミナーを読みましたが、もちろん、Windows上のC / C ++開発者向けの新製品であるIntel Parallel Studioの「積極的なプロモーション」に特別な注意を払っています。 そして、私は質問をま​​すます頻繁に耳にします：なぜインテルはParallel Studioを無料で配布するべきではないのか、それを取得したMS VSの負荷に与えるべきではないのですか？ このような質問は、単にツールを無料で入手したいという願望からではなく、インテルがソフトウェアを開発している理由を正しく理解しているためです。 これらの理由が何であり、何を理解する必要があるかを見てみましょう。



Intelの主な収入源がマイクロプロセッサの販売であることは秘密ではありません。 彼らの言うことは何でも、インテル製マイクロプロセッサーの販売はそれほど単純ではありません。競合他社から非常に優れた製品が出されているときに、ブランドに過剰な支払いを望む人はほとんどいません。 そして、SSE4.2や1000個のパフォーマンスカウンターなど、Intelプロセッサーの特別な「機能」に興味がある人はさらに少ない。 そのため、加工業者の販売を支援する必要があります。適切なアプローチの1つは、いわゆる有効化、つまり何かを可能にすることです。 この点で、IntelはSSG（Software and Services Group）と呼ばれる巨大な部門を持ち、主にこの「有効化」に従事し、かなり多数の企業がソフトウェアを修正/最適化してIntelプロセッサ上でより高速に実行できるよう支援していますより効果的です。 ソフトウェアツール、コンパイラ、アナライザ、プロファイラなどがなければ、このような仕事を想像することは不可能です。 同時に、これらのツールの範囲は、Intelチップの最適化の範囲をはるかに超えています。 たとえば、Intelコンパイラーは、AMDプロセッサー上で実行されるコードを、他のコンパイラーによってコンパイルされたプログラムよりも遅くない速度で生成します。 状況はParallel Studioでもほぼ同じです。Intel以外のプロセッサー（テクニカルまたはマーケティング）での使用に制限はありません。 インテルは、マルチコアプロセッサの開発動向における普遍的かつ最終的な勝利に関心があり、Parallel Studioが競合するプロセッサを備えたプラットフォームで使用されているという事実は誰も気にしません。



したがって、SSGで作成されたソフトウェアは、マイクロプロセッサの販売、つまりIntelの利益化に役立つことがわかります。 ソフトウェア部門の収益性については、好きなだけ話せますが、明白で透明な財務諸表は、主なものはまだハードウェアであることを示唆しています。 しかし、もしそうなら、たとえば、私たちの環境にやさしいCatalistと同じように、ソフトウェアを無料で全員に無料で配布してみませんか？



すべては産業の心理学についてです。 まともな会社が「ツールパーク」に、研究目的のみを除いて無料で配布されるプログラムを含めることはほとんどありません。 これは特定のステレオタイプです：フリーソフトウェア（オープンソースと混同しないでください）は、多くの愛好家によって書かれたサポートされていないプログラム、またはその見通しがあいまいな研究プロジェクトであるため、開発プロセスに適合させる意味がありません。 大規模で有名な会社によって提供されていることは問題ではありません。どの会社のリソースも限られているため、遅かれ早かれ誰かがそれらの処理とサポートをやめます。

メーカーがソフトウェアの代金を支払う場合、メンテナンス、ユーザーサポート、バージョンの更新、機能の拡張、バグの修正などの責任を負います。 彼のお金のために、顧客は特定のロードマップと製品の見通しに依存する権利を持っています。 ソフトウェアのコストが高いほど、さまざまなサービスを提供するなど、ユーザーを「なめる」必要があります。



SSGは、Parallel StudioがMicrosoft VSとともにマルチスレッドアプリケーション開発者の「標準環境」になることを期待しているため、ある種の理解可能なツールインフラストラクチャを作成し、ほとんどのユーザー（ フォーラム ）にできるだけ近いサポートモデルを構築しようとしています。 そして、価格はこのように形成された可能性が最も高い：マーケティング担当者は指を動かして市場の申し出を風に流し、その後、パッケージ内のコンポーネントの通常価格は399ドル前後になると感じたが、卸売りの顧客（Intel Parallel Studio全体が完全に）799ドルとギフトとして壁のポスター。 もちろん、アカデミックプログラム（ソフトウェアは半額）とロイヤルティプログラム（企業が新しい製品を定期的に使用するための無料ソフトウェアを受け取り、使いやすさを改善するためのバグレポートと提案を送信する）もあります。



このような景品価格はありますか？ もちろん、「無料」とは異なります。 インテル®Parallel Studioを個人的に購入し、遊んで遊んだり、書かれたプログラムを自由に「挽いたり」することはできますか？ ほとんどの場合、そうではありません。この場合、MS Visual C ++ Express Editionを使用するため、Parallel Studioはインストールされません。 C ++プロジェクトを開発しているソフトウェア会社のボスを説得して、いくつかのライセンスを増やすことはできますか？ 間違いなくはい！ あなたはどうですか？
