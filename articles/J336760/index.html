VMware vSAN 6.6.1─卓越性への別のステップ

今年7月にリリースされたvSAN 6.6.1では、vSAN 6.6リリースで数か月前に登場したオプションにいくつかの新しいオプションが追加されています。 全体的に、vSANハイパーコンバージドインフラストラクチャ（HCI）にはいくつかの重要な追加が行われ、現在では運用コストがさらに低くなっています。







この記事では、最新のプレゼンテーション、新しいプロセッサ、更新されたvSAN 6.6.1構造、新しい統合vSphere Update Manager（VUM）、および分析ツールとセキュリティツールでHCIのコンポーネントを調べます。



Intel Xeonスケーラブルプロセッサ─vSANの新しいエンジン



今年の7月上旬に、Intelは新しいXeonスケーラブルシリーズプロセッサを公式に発表しました。



アプリケーションの観点から言うと、「今は、従来の分散インフラストラクチャから、これらのプロセッサとVMware vSANシステムを使用したハイパーコンバージドコンピューティングとデータストレージに移行する絶好のタイミングです。



VMwareとIntelが新しいハードウェアを使用して設定する主な目標は、企業がレガシーITインフラストラクチャの従来の複雑さとコストを削減できるようにすることです。



ハイパーコンバージドITインフラストラクチャは、レガシーシステムよりも高速で費用対効果に優れています。 データストレージとネットワークソリューションの構築へのアプローチを根本的に変更し、最も要求の厳しいエンタープライズアプリケーションにより多くのオプションを低価格で提供し、管理を改善します。 vSANは、これらの最新のIntel製品でもテストされています。



新しいIntel Optane SSDと新しいサーバープラットフォームを使用してHCIを展開すると、顧客は、前世代のIntel XeonプロセッサーとvSANシステムと比較して、クラスター内で最大2.5倍の仮想マシン（VM）を取得できます。



これらの結果は、評価者グループが実施した独立したテストで得られました。 それらは、価格の低下と投資収益率（ROI）を示しています。 ここで登録後完全なレポートを入手できます 。 基本的な情報を以下に示します。





4ノードのハイパーコンバージドインフラストラクチャでの仮想マシンのパフォーマンス比較



図の左の3つの列は、Intel XeonプロセッサーE5-2600 v4を使用した前世代のサーバーを示しています。 3つの右側の列は、Xeonスケーラブルプロセッサ、次世代SSD、およびVMware vSANバージョン6.6にアップグレードするときのパフォーマンスの向上を示しています。



ストレージセグメントは、今日のITで最も急速に成長している分野の1つです。 ますます多くの組織がサーバーの計算能力を向上させているだけでなく、ストレージシステムをアップグレードし、安全なハイパーコンバージドインフラストラクチャに移行しています。



VMware Storage ManagerのMichael Haagが同社のWebサイトのブログ投稿で次のように書いています。「新しいIntel Xeonスケーラブルプラットフォームにより、この移行は10年間でIntel Xeonプラットフォームの最大のアップデートになります。」



インテルの新しいアイデアは、現在および将来のvSANのお客様に次の主要なメリットをもたらします。



1.将来のパフォーマンスを向上させるためのスケーリング。



2.豊富なHCIプラットフォームでリスクのない使用。



3.迅速な投資収益率で企業ITのコストを削減します。 仮想マシンノードのコストは、以前のインフラストラクチャの2.5倍になりました。 消費電力が少なく効率が高く、ハードウェアの密度が高くなるため、より小さな領域に配置できます。



4.最初のIntel Selectソリューションの確実な展開。 この新しいソリューションは、実績のある構成を提供して複雑さや投資を削減することにより、インフラストラクチャを変換する最速の方法を提供します。 Intel Selectソリューションは、VMware vSANの最初の次世代ソリューションの1つとして利用できます。



vSAN 6.6.1



7月末に、VMware vSphere 6.5 Update 1がお客様に利用可能になり、vSAN 6.6.1には3つの新しい主要機能が登場しました。



-VMware vSphere Update Manager（VUM）との統合。

-vSAN Cloud Analyticsのパフォーマンス診断。

-拡張ストレージデバイスの保守性。



さらに、ITインフラストラクチャROBO（リモートオフィス/ブランチオフィス）およびVDI（仮想デスクトップインフラストラクチャ）に対してvSANエンタープライズをよりアクセスしやすくする新しいライセンスが登場しました。



新しい統合vSphere Update Manager（VUM）



VMware vSphere管理者は、vSphereクラスターのパッチ適用とアップグレードを大幅に簡素化および自動化する、新しい統合vSphere Update Manager（VUM）更新ツールに満足するでしょう。 以前のバージョンでは、管理者は多くの操作を手動で実行する必要がありました。



ソフトウェアをアップグレードする前に、ハードウェア（SAS、SATA I / Oコントローラー、NVMeデバイスなど）とvSphereおよびvSANの新しいバージョンとの互換性を確認する必要がありました。 以前は、これはVMware Compatibility Guideソフトウェアモジュールを使用して実行される手動プロセスでした。



VUMは、VMware互換性ガイドとvSANリリースカタログの情報を自動的に検索し、現在インストールされているESXiハイパーバイザーリリースに関する情報と結合します。 VUMは、Dell、Fujitsu、Lenovo、およびSupermicroの新しいハードウェアドライバーも識別します。



この新しいVUM拡張機能により、vSphereおよびvSANの新しいバージョンを簡単にサポートできます。 各ホストがクラスター内で実行され、他のホストと同じ構造を持っていることを保証することにより、リスクを軽減します。 管理者の観点から、VUMは各ホストを手動でアップグレードする時間のかかるプロセスを排除します。



実際、VMwareは、アップグレードの単純さを保証するためだけに、VUMをvSANに統合することを決定しました。 その結果、vSANの更新プログラムのインストールとバージョン管理は、vSANに完全に統合されたプロセスになるだけでなく、VUMプロセス自体が大幅に簡素化されました。





VUM高レベルアーキテクチャ



vSANバージョン6.6.1以降では、自動化された「シームレスな」アップグレードプロセスが提供され、vSANクラスターが最新のソフトウェアで更新されるようになりました。 最初のステップでは、vSAN Build Recommendation Engineを使用します。 ログインに成功すると、vSANは各vSANクラスターの推奨アップデートのメイングループを作成します。





VSANの推奨事項



VMware互換性ガイドとvSANリリースカタログを確認した後、Update Managerは利用可能な推奨アップデートを見つけます。





例：vSAN Baselineは、ESXiホストをプロファイル外として識別しました



vSANクラスターを更新するには、Update Managerの機能のいずれかを使用するだけです。 対応する「ウィザード」には、更新を構成するためのいくつかのオプションがあります。



-特定のホストとクラスターを選択できます。

-すぐに更新を実行するか、時間を設定できます。

-サポートモード（メンテナンスモードの指定）を指定できます─仮想マシン、処理メディア、ESXiにパラメーターを割り当てます。



目的のオプションを選択すると、Update Managerは各ホストを自動的に更新します。



したがって、VUMを統合したvSAN 6.6.1は、vSANグループクラスターの更新とパッチの管理プロセスを大幅に簡素化します。



ITプロフェッショナルは、ハードウェア構成の変更ではなく、ユーザーインターフェイスを介して変更が行われるソフトウェア定義のアプローチを使用して、初めてデータセンター全体を管理できるようになりました。



VSAN Cloud Analyticsにパフォーマンス診断が追加されました



最新リリースは、vSAN 6.6で以前に導入されたvSANクラウド分析の機能を拡張します。 まず、これは新しいPerformance Diagnosticsツールに関するもので、特定のvSANクラスターのパフォーマンスを以前のベンチマークと比較して分析します。





パフォーマンス診断



管理者は、最大スループットまたは最小遅延などの目的のオプションを選択し、比較時間の特定の範囲を設定できます。 ただし、この機能を使用するには、ユーザーがカスタマーエクスペリエンス向上プログラム（CEIP）およびvSANパフォーマンスサービスに参加する必要があります。



その後、管理者はHCIBench評価テストを管理し、適切なチャートがサポートされているPerformance Diagnosticsで詳細なテスト結果を表示できます。



Ask VMwareコンパニオンツールは、追加情報を提供する関連VMwareナレッジベース記事へのアクセスを提供します。 これらのガイドラインは、管理者がクラスターのパフォーマンスを最適化し、特定のハードウェア選択の影響を理解し、パフォーマンスの問題が発生した場合のトラブルシューティングを高速化するのに役立ちます。



Software Defined VaultsのHCIセキュリティ



Anita Kibunguchyは、彼女のVMwareブログで次の例を提供しています。



ある金融会社は最近、毎日最大1万回のシステムへの不正アクセスが試行されていると報告しました。 病院は、個々の患者データの安全性を維持する必要性について話します。 別の会社は、暗号化されていないデータが誤って失われないようにするために、使用済みのハードドライブを物理的に破壊すると述べました。



セキュリティエリアは、以前はハイパーコンバージドインフラストラクチャと密接に関連していませんでした。 ただし、ESG調査の回答者の39％は、2017年にセキュリティの問題がここに現れると考えており、それらに注意を喚起することを強制します。 新しいリリースでは、VMware vSANは暗号化を実装しているため、HCIは安全なIT環境の構築に最適です。



ロッキードマーティンサイバーキルチェーン（サイバー攻撃を識別および防止するための会社独自の構造）によると、多くの攻撃はエンドポイントを標的とし、組織のITインフラストラクチャを介してサーバーに移動し、企業データにアクセスします。



これにより、HCIシステムの情報セキュリティに重大なリスクが生じ、最大限のセキュリティを確保するためにシステムコアのデータを暗号化する必要が生じます。 データをソースで直接暗号化することには、多くの利点があります。



-通常、独自のスタッフを維持するのではなく、サードパーティと連携する分散型組織の追加レベルのセキュリティ。



-メディアの保管の簡素化-メディアを物理的に破壊する必要はありません。



-すべてのタイプを暗号化できるため、データの再フォーマットに時間を無駄にする必要はありません



既存のHCIソリューションで静的データ暗号化を実現する最も簡単な方法は、自己暗号化ドライブ（SED）を使用することです。 ただし、いくつかの制限があります。



-まず第一に、それは彼らの高い費用と低い回収です。



-SEDの実装は、法律の多くのコンプライアンス要件を順守する必要があるため、問題になる可能性があります。



-特定のリスク-セクションのいずれかが故障した場合、ディスク全体を交換する必要があります。 さらに、SEDの構築の原則により、調査が困難になります。



-実用的なSED管理は比較的困難です。



VMware vSANにはSEDの制限はありません。 これはソフトウェア定義のインフラストラクチャです。 そのフレームワーク内で、コンピューティング、データストレージ、ネットワーク、および情報サービスは、業界標準のx86システム上の1つのソリューションに簡単に結合されます。



ESGは、VMwareソリューションは運用効率が高く、従来のタイプのシステムよりも複雑でなく、管理が容易であると指摘しています。 ESGの調査によると、ハイパーコンバージドソリューションを展開する利点は次のとおりです。



-組織の22％が、このようなソリューションを展開する主な利点としてTCO値が低いと報告しました。



-26％が展開時間を短縮した



-24％がサービスとサポートの改善、およびスケーラビリティを報告しました。



-23％が、ITインフラストラクチャ管理の簡素化を利点としての利点として指摘



VMware vSANは、仮想マシンと物理ハードウェアの両方にネイティブ暗号化を提供します。 SEDは必要ありません。 このソリューションは、既存または新規のストレージデバイスに展開できます。



セキュリティは、VMやハードウェア内ではなく、ハイパーバイザーレベルでシステムのコアに組み込まれています。 これにより、保護が最大化されます。 ホストベースであるため、管理も簡素化されます。



ESGアナリストはまた、vSAN─「より安全で、より速く、より簡単に管理できるデータセンターを探している組織にとって理解しやすいソリューションは、VMware vSANの機能を探るべきだ」と述べました。



多くのお客様にとって、VMware vSANはIT史上初のハイパーコンバージドソリューションです。 同時に、「ネイティブ」暗号化は、HCIのコストと企業のITインフラストラクチャ全体の管理性の両方の観点から、考慮すべき主要な要素の1つです。



ストレージの信頼性



ハードウェアのメンテナンスを簡素化するために、vSphereには、サーバー上のローカルコントローラーに接続された物理ストレージデバイスに関する高度な管理と情報を提供するプラグインが含まれています。



実践が示すように、この機能はvSANインフラストラクチャに特に役立ちます。 たとえば、これらのプラグインを介して、サーバーシャ​​ーシ内の目的のドライブのローカルな場所を表示して、管理と保守を簡素化できます。



vSAN 6.6.1はこのオプションをさらに拡張し、特定のドライブを見つけるだけでなく、インジケータを点滅させて物理的な場所を見つけやすくします。 その結果、管理者は注意が必要なデバイスと適切な対策を視覚的に識別します。







この機能は小さなサービス追加のように見えますが、ストレージデバイスに費やす時間と人為的エラーのリスクを大幅に削減します。 この機能はRAIDアレイでも機能し、現在、Gen 9コントローラーを搭載した一般的なHPE DLおよびMLサーバーでサポートされています。



ROBOおよびVDIの新しいvSANライセンス



新しいライセンスオプションは、ROBO（リモートオフィス/ブランチオフィス）およびHorizo​​n VDIの環境の1つに関連しています。 ROBO向けvSANエンタープライズには、小規模な展開向けの「1つの仮想マシン」価格設定モデルに独自の暗号化と拡張クラスターの使用が含まれています。 これらの拡張機能は、今年初めにリリースされたVMware HCIキットを補完するものです。 シンプルな構成、vSphereおよびvSANライセンスが含まれます。



この簡単なレビューをまとめると、もう一度2つの側面を強調する価値があります。 vSAN 6.6.1は、ハイパーコンバージドITインフラストラクチャのハードウェアおよびソフトウェアのライフサイクル管理をさらに簡素化します。 新機能により、運用コストがさらに削減されます。
