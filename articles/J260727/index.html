NetUPテープドライブのHDDの交換とファームウェアのインストール















背景



最近、NetUPストリーマーの1つがブロードキャストを停止しました。 凍結していると思って（彼らはそのような「機能」を持っている）、それを再起動しに行きました。 なぜなら これらのストリーマーには電源オン/オフボタンやリセットボタンはありません;電源コードを抜くだけで再起動します。



しばらくして、電源を入れた後、ストリーマーのLCDパネルに「HDDファイルシステムを確認しています...」という文字が表示されました。 ディスクをチェックして起動すると考えて、この状態のままにしました。 ただし、これは1〜2時間後には発生しませんでした。 テープドライブは、引き続きHDD検証メッセージを表示しました。 コンソールケーブルの接続に失敗しました。 また、その後の再起動では何も起こりませんでした。 私はハードドライブを入手し、不良ブロックをチェックすることにしました。 RemapまたはEraseを作成できなかったセクターが少なからず見つかりました。



結論は簡単です。ハードドライブを変更する必要があります。 交換自体には問題はなく、ワーカーが引き抜かれても、ワーカーがセットされても問題ありません。 しかし今では、OS（ファームウェア）のない鉄片が手元にあります。



ファームウェアを「ゼロから」インストールする方法と、これからの話は続きます。



オプション



個人的には、これを行う方法は3番目のオプションのみです。

  1. NETUPテクニカルサポートにお問い合わせください
  2. 別のテープドライブからHDDを複製する
  3. ファームウェア、それが何であるか、それをインストールする方法を扱う


最初のオプションは、1年以内にこの製品を購入した人に適しています。 テクニカルサポートは、購入後1年以内に提供されます。 また、NetUPテクニカルサポートの料金を支払う人にも。 このオプションは私には適していない、なぜなら 1年以上前に購入しましたが、勤務時間中に1年間のテクニカルサポートを支払う理由はありません。



2番目のオプションはまったく問題ありません。 作業中のストリーマーの1つを消すだけで、何らかのプログラムでそのディスクのクローンを作成することができます。 しかし、私はラックから別のストリーマーを削除したくありませんでした。次に、他のチャンネルのブロードキャストを停止したくありませんでした（購読者はこれを嫌います）。



上記のすべての基準に従って、3番目のオプションは非常に適していました。 さらに、それは単に興味深く有益なものです。



準備する



ストリーマーを機能させるには、次のものが必要です。

  1. USBフラッシュドライブ
  2. コンソールケーブル
  3. ルーター
  4. ファームウェアイメージ
  5. LiveFlashイメージ


各アイテムごとに個別に。



USBフラッシュドライブは、128 MBから始まるあらゆるサイズに適しています。 LiveFlashイメージは約100 MBかかります。



テープドライブに接続し、BIOS設定を変更するためのコンソールケーブル。 テープドライブにビデオ出力がありません。



ストリーマーはルーターに接続する必要があります。これにより、USBフラッシュドライブから起動した後、さらに作業するためにSSH経由で接続できます。



ファームウェアイメージは、netup.tv Webサイトのセクション「My Account> Files> Stable Version」からダウンロードされます。 firmware-streamer-standaloneバージョンを使用しました。



LiveFlashイメージもサイトからのものです。 セクション「My Account> Files> Public Files」。 LiveFlashとLiveCDの2つのイメージがありますが、テープドライブにはCDドライブがありません。 一般的に、フラッシュドライブから起動できるLinuxディストリビューションが適していると思います。 NetUPの画像では、他のディストリビューションにはない特別なものは見当たりませんでした。 しかし、私はそれを使用しました。



始めましょう



ファームウェア


結局のところ、ファームウェアイメージはtar.gzアーカイブです。 したがって、アーカイブを解凍します。 



tar -xvf firmware-streamer-standalone.1.11.117


アーカイブには、filesystem、info、initrd.img、vmlinuzの4つのファイルがあります。



ファイルシステムは、名前が示すように、Squashfsにパッケージ化されたファイルシステムイメージです。



info-これには、ファームウェアに関する情報が含まれます：チェックサム、バージョン、リビジョン、タイプ、作成日。



initrd.imgとvmlinuzはRAMディスクとカーネルです。



これらのファイルはすべて後で必要になります。



ライブフラッシュ


ここではすべてが簡単です。 ダウンロードしたイメージをUSBフラッシュドライブに書き込みます。 私はddを通してこれをしました： 



 dd if=liveflash.img of=/dev/sdb bs=1M


読み込み中


コンソールケーブルを接続し、パッチコードをルーターに接続し、USBフラッシュドライブをストリーマーに挿入して起動する必要があります。



今、あなたはドキュメントにない何かをする必要があります。 USBフラッシュから起動する必要があります。これを行うには、BIOSで最初に起動キューで指定する必要があります。 ブートプロセスを確認し、適切なタイミングで[Del]キーを押すために、コンソールケーブルを介してテープドライブに接続します。 これは、ハイパーターミナル、パテなどを通じて実行できます。 個人的には、ミニコムを使用しました。 接続パラメーターでは、COMポート番号に加えて、速度/パリティ/ビット115200 8N1を指定し、ハードウェアとソフトウェアのフロー制御を無効にする必要があります。 これはドキュメントにはありません。または、19200 8N1の速度でコンソールケーブル接続がありますが、POSTメッセージポートとBIOS自体のこのような設定では、表示されません。 そのため、たとえば、SSH経由でアクセスするためのネットワーク接続がない場合、動作中のテープドライブにのみ接続できます。



BIOSにアクセスしてUSBフラッシュからの起動を設定したら、BIOSを終了してパラメーターを保存します。 F4、間違っていない場合、またはメニュー項目「保存して終了」を使用します。 これで、ストリーマーがフラッシュドライブから起動します。 ただし、注意点が1つあります。事実は、LiveFlashブートローダーメニューで、デフォルトで「ローカルハードドライブから起動」が選択されていることです。 ドライブはまだ空です。 115200のシリアルポート速度では、ブートローダーメニューは表示されません。 したがって、ストリーマーが再起動に入った直後に、速度を19200に変更し、フラッシュドライブのブートローダーを確認します。 一番上の項目「Boot LiveFlash」を選択し、「Enter」を押します。



ストリーマーがLiveFlashから起動したら、ルーターのWEBインターフェイスに移動して、受信したIPアドレスを確認する必要があります。 次に、SSHクライアントを使用して、 rootユーザー名とnetup123パスワードでテープドライブに接続します。



運転


したがって、SSHを介してストリーマーに接続するか、ストリーマーで実行されるLiveシステムに接続します。



最初に行う必要があるのは、ハードドライブをパーティションに分割し、それらを必要なディレクトリにフォーマットしてマウントすることです。



最初のパーティションは5 GBです。2番目は他のすべてです（正直なところ、私には謎のままです。NetUPでは、最大15 GBのディスクスペースが使用されている場合、1 TBの容量のディスクを使用します）。 パーティション化は、システムにあるfdiskユーティリティを使用して可能です。 



 fdisk /dev/sda


fdiskの使用は非常に簡単です。ここでは説明しません。 これは記事の範囲外です。



パーティションの作成方法と新しいパーティションテーブルのディスクへの書き込み方法のフィールドは、それらをフォーマットする必要があります。 



 mkfs.ext3 /dev/sda1 mkfs.ext3 /dev/sda2


今マウント： 



 mount /dev/sda1 /mnt/flash mount /dev/sda2 /mnt/hdd


以上で、これらのディスクにファームウェアのインストールを開始する準備が整いました。



ブートローダー


ストリーマーはGRUBを使用します。 オプション--root-directory = / mnt / flashでインストールする必要があります。 /ブートディレクトリとブートローダー設定ファイルはsda1に存在する必要があります： 



 grub-install --root-directory=/mnt/flash /dev/sda


ブートローダーがインストールされたら、カーネルブートオプションを指定する/ mnt / flash / boot / grubディレクトリにmenu.lstファイルを作成する必要があります。 完全なファイルコンテンツ：



menu.lst
デフォルト0

タイムアウト3



タイトルNetUP IPTV

ルート（hd0.0）

カーネル/ブート/ vmlinuz root = / dev / ram0 selinux = 0 init = / linuxrc vmalloc = 256m ro netup_flash = sda1 netup_hdd = sda2

initrd /boot/initrd.img



タイトル回復

ルート（hd0.0）

カーネル/ブート/ vmlinuz root = / dev / ram0 selinux = 0 init = / recovery ro

initrd /boot/initrd.img



ファームウェアファイル


以前にファームウェアから抽出されたファイルシステムと情報ファイルは、/ mnt / flashディレクトリ内のディレクトリのテープドライブにダウンロードする必要があります。 そして、vmlinuzおよびinitrd.imgファイルは/ mnt / flash / bootにあります。 これにはscpを使用できます。たとえば、私は個人的にMidnight CommanderでSFTPを使用しました。



準決勝


実際、それだけです。 ストリーマーコンソールで「reboot」と入力し、USBフラッシュドライブを取り外して、ストリーマーがロードされるのを待つことができます。 ファイルシステムを解凍し、スワップファイルを作成します。 これは、標準設定の標準ファームウェアになります。 デフォルト設定を変更する場合は、以下でその方法を説明します。



設定


多くの設定はありません：

  1. インターフェースIPアドレス
  2. ルーティング
  3. パスワード
  4. タイムゾーン


Aufsはファームウェアで使用されるため、ファームウェアに設定を強制的に適用するには、標準設定ファイルを「オーバーレイ」する適切な構成ファイルを作成するだけで十分です。



構成ファイルを配置するディレクトリを作成します。 



 mkdir /mnt/hdd/netup/sysconfig




インターフェースIPアドレス


ファイルnetwork.cfg。 そのエントリの形式は次のとおりです。



network.cfg
eth0 10.4.4.176 255.255.255.0メイン

eth3 172.16.0.9 255.255.255.0



テープドライブで使用されているアドレスをすぐに登録できるため、後で標準のアドレスでテープドライブを入力する必要はありません。



ルーティング


routes.cfgファイル。 次のようになります。



routes.cfg
route add -net "0.0.0.0" gw "172.16.0.1" netmask "0.0.0.0" dev "eth3"



各行に1つずつ、複数のルートを登録することもできます。



パスワード


Password.cfgファイル。 ここにパスワードが書き込まれます。これは、WEB、SSH、およびFTPを介したアクセスに使用されます。 パスワードはクリアテキストで記録されます。



タイムゾーン


タイムゾーンファイル。 記録例：



タイムゾーン
アジア/クラスノヤルスク



まとめ



以上です。 ご覧のとおり、ベアテープドライブにファームウェアをインストールするのに困難はありません。 これにはほとんど時間がかかりません。テクニカルサポートサービスを通じてこれを行う場合、ホットラインでの半日にわたる交渉が必要になります。



ですから、自分でそれを行うことは、面白くて、速くて、特に無料です。
