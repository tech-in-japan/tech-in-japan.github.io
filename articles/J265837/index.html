Urho3Dエディター（パート2）

Urho3Dエディターについては引き続き理解していきます。 このレッスンでは、物理学を操作する方法、プレハブに精通し、パーティクルエディタをマスターし、銃を撃つ方法を学びます。 また、ボーナスとして、ゲームリソースをパックする方法を学びます。



物理学



Urho3Dは、Box2DとBulletの2つの物理ライブラリをサポートしています。 それらを操作する原理は非常に似ていますが、最初の方法はノードの移動をXY平面でのみ実装し、2番目の方法は3次元空間で実装します。 Box2Dコンポーネントは、[作成]-> [コンポーネント]-> [Urho2D]メニューにあり、[弾丸]メニューは[作成]-> [コンポーネント]-> [物理]メニューにあります。



そのため、 前回のレッスンのGameData \ Scenes \ Level01.xmlシーンをロードします（[ファイル]-> [最近のシーンを開く]または[ファイル]-> [シーンを開く...]）。 床と銃があります。 砲弾をシーンに追加しましょう。 リソースブラウザで、Modelsフォルダを開き、Cannonball.mdlを[階層]ウィンドウにシーンルートノードにドラッグします。 銃と床が作業に干渉しないようにするには、[階層]ウィンドウでノードを右クリックし、[有効化/無効化]を選択します。 同時に、それらはシーンから消え、ノードは赤く塗られます。 カーネルのあるノードをダブルクリックすると、画面の中央にカーネル（カーネル）が表示されます。







これで、リソースブラウザで、[マテリアル]フォルダを開き、[階層]ウィンドウまたは[属性インスペクター]ウィンドウの[マテリアル]列のStaticModelコンポーネントにマテリアルをドラッグします。 ノードに名前を付けて、「影を落とす」オプションを有効にします。







RigidBodyおよびCollisionShapeコンポーネント（作成->コンポーネント->物理）をキャノンボールコアに追加します。 この場合、物理学の作業に必要なPhysicsWorldコンポーネントがシーンに自動的に追加されます。



RigidBodyは、オブジェクトの物理的特性を定義します。 ボディは静的および動的です。 静的な物体は動かず、障害物に過ぎず、動的な物体は外力（重力を含む）の影響を受けます。 ボディに質量がある場合-動的であり、質量がゼロの場合、ボディは静的です。



以下のスクリーンショットのように、RigidBodyの質量、滑り摩擦、および転がり摩擦を設定します。 CCD RadiusおよびCCD Motion Thresholdパラメーターに注目してください。 小さな物体が障害物を高速で滑る場合、これらのパラメーターを設定すると役立ちます。







CollisionShapeは、衝突の計算に使用されるオブジェクトの形状を定義します。 3Dモデル自体を使用しないのはなぜですか？ 最適化するため。 たとえば、球との交点を特定するには、球の中心までの距離とこの球の半径を比較するだけで十分です。 これは、多くの三角形の交点を計算するよりもはるかに高速です。 オブジェクトには、形状をより正確に記述するために複数の形状を含めることができます。



CollisionShapeコンポーネントのSphereタイプを選択し、上のスクリーンショットのように、キャノンボールの半径に近い半径を設定します。



RevertOnPauseスイッチが有効になっていることを確認し、RunUpdatePlayボタンをクリックします。 重力の影響下にある砲弾は落下し始めます。 一時停止を押すと、コアが元の場所に戻ります。



プレハブ



Unityで作業したことがある人は、この概念に精通しているでしょう。 プレハブは、シーンに繰り返し挿入できるオブジェクトテンプレートです。 GameDataディレクトリにObjectsフォルダーを作成します。 Cannonballノードを選択し、メニュー項目File-> Save node as ...を選択し、ファイルGameData / Objects / Cannonball.xmlに保存します。 これで、砲弾をシーンから削除できます。 プレハブをシーンに追加するには、メニュー項目[ファイル]-> [ノードのロード]を使用します。



銃、撃て！



ガンとフロアノードをオンにして、見えるようにします。 RigidBodyおよびCollisionShapeコンポーネントを作成します。 床の場合、StaticPlaneシェイプタイプ（無限平面）を選択し、スリップフリクションとローリングフリクションを1に設定します（そうしないと、接触時にオブジェクトの摩擦が増加するため、ボディが無限に移動します）。 ガンの場合、カプセル形状の形状を選択し、形状（ノードではなく）をサイズ=（2、4.3、1）およびオフセットオフセット位置=（0、1.12、0）に設定します。 銃の場合、AnimationControllerコンポーネントも追加します（作成->コンポーネント->ロジック）。これにより、3Dエディターで作成されたショットアニメーションを再生できます。

核が存在してから10秒後に核を自己破壊させましょう。 次のテキストを含むCannonBall.asファイルをGameData \ Scriptsフォルダーに作成します。

次に、プレハブGameData / Objects / Cannonball.xmlをシーンにロードし（File-> Load node-> As replicated / local ...）、それにScriptInstanceコンポーネントを追加し、新しく作成されたファイルを指定し、Dyingクラスの名前を入力します（Enterを押すことを忘れないでください）。 これで、プレハブを保存してシーンから削除できます。 実行して、核が消える様子を観察します。



うさぎの町



大砲から発射するキューブの建物をシーンに追加します。 通常どおり、GameData / Models / Cube.mdlモデルをSceneルートノードにドラッグし、GameData / Materials / Cube.xmlに設定します。 影を落とすオプションも有効にします。 RigidBodyコンポーネント（質量= 1、摩擦= 1、ローリング摩擦= 0.2）とCollisionShapeコンポーネント（形状タイプ=ボックス、サイズ=（2、2、2））を追加します。 キューブを地面の少し上に置きます（地面に浸すと、起動時に飛び出します）。 Ctrl + Dを押してキューブを複製し、コピーを横に移動します。 別のコピーを作成します。







ところで、属性インスペクターウィンドウでは、デフォルトと異なるすべての値が金色でマークされていることに気付きましたか？



次に、Ctrlボタンを押しながらキューブの行全体を選択し、全体を複​​製します。 最初の行の上にコピーを持ち上げます（ギャップを忘れないでください）。 次に、別の行を作成します。







それを実行します。 起動後にキューブが少し落ちるという問題がありましたが、キューブを手作業で非常に正確に重ねることは困難です。 ここで、RevertOnPauseスイッチが役立ちます。 それをオフにし、また、銃でScriptInstanceコンポーネントを無効にします（以前にノード全体をオフにしたように、ノードではなくコンポーネントを右クリックします。別の方法は、属性インスペクターウィンドウで十字をクリックすることです）。







今、プレイを開始し、キューブが落ちるとき-停止します。 シーンは新しい状態のままでした。 RevertOnPauseとScriptInstanceを再度有効にしてください。



粒子



キャノンボールのパーティクルの軌跡を作成しましょう。 メニュー項目の表示->パーティクルエディタからパーティクルエディタを開きます。 次のオプションを設定します。



  • 方向（分）=（-0.1、-0.1、-0.1）
  • 方向（最大）=（0.1、0.1、0.1）-これら2つの値は、異なる方向での粒子の膨張速度を決定します（各粒子について、特定の範囲でランダムな値が選択されます）
  • 粒子サイズ（最小）=粒子サイズ（最大）=（0.3、0.3）-すべての開始粒子サイズは同じです
  • サイズ追加= -0.8-時間が経つにつれて、粒子サイズが減少します
  • 粒子数= 1000-同時に存在する粒子の最大数
  • 放出速度（最小）=放出速度（最大）= 80-粒子放出頻度
  • 相対変換=オフ -親ノードに関係なく、パーティクルがワールド座標で移動するように


ここで、粒子の経時的な色の変化を追加します。 Color Framesブロックの下の[New]ボタンをダブルクリックして、2つの新しい行を作成し、下のスクリーンショットのようにフィールドに入力します。 最初の値は時間であり、色はRGBA形式です。







[名前を付けて保存]ボタンをクリックして、エフェクトをGameData \ Particle \ Sparks.xmlファイルに保存し、パーティクルエディターウィンドウを閉じます。 砲弾のプレハブをシーンにロードし、ParticleEmitterコンポーネントを追加して（作成->コンポーネント->ジオメトリ）、新しく作成した効果を指定し、プレハブをファイルに保存して、砲弾をシーンから削除します。 ゲームの準備ができました。



次は？



エディターのすべての機能を考慮しませんでした。 たとえば、インターフェイスの作成やマテリアルエディタについての言葉はありませんでした。 ただし、一般原則を理解し、それらに対処することは難しくありません。



次に、ゲームを個別のプログラムとして配置する必要があります。 計画は非常に単純です。シーンをロードして開始する最小限のスクリプトを作成します。 ただし、シーンには十分なカメラがなく、エディターの内部カメラが使用されます。外部カメラは使用できません。 通常どおり、新しいノードを作成し、カメラコンポーネントを追加します（作成->コンポーネント->シーン）。 スクリプトでアクセスできるように、必ずノード名をCameraに設定してください。 カメラを好きなように配置して回転させます（右下隅に表示されるカメラからのビューが役立ちます）。 また、SoundListenerコンポーネント（Create-> Component-> Audio）をカメラのあるノードにアタッチします。これがないと、空間に配置されたサウンドが聞こえなくなります。 シーンを保存し、次の内容のGameData \ Scripts \ Main.asファイルを作成します。

約束のボーナス



まず、コマンド「ScriptCompiler.exe GameData / Scripts / *。As」ですべてのスクリプトをコンパイルします。 その結果、拡張子が.ascのファイルを取得します。 最近、 パッチが追加されました。これにより、要求された.asファイルがない場合、.ascファイルがロードされます。これは、名前の変更に対処する必要がないことを意味します。 ソース.asファイルをGameData \ Scriptsフォルダーから別の場所に移動するだけで、.ascファイルのみを残します。



パッケージ化には、Build \ bin \ tool \ PackageTool.exeユーティリティが必要です。 コマンドを実行します。 



 PackageTool.exe CoreData CoreData.pak -c PackageTool.exe GameData GameData.pak -c


-cオプションは、ファイル圧縮をアクティブにします（高速LZ4アルゴリズムを使用）。



これで、CoreDataおよびGameDataフォルダーを削除できます。 その結果、次のことができます。







さて、非常に美しくするために、バッチファイルStart.batを削除できます。 ただし、このためには、ランチャーのソースコードを変更する必要があります（Urho3D \ Source \ Tools \ Urho3DPlayerフォルダーにあります）。 事前に怖がらないでください-ランチャーソースは非常に小さいです。これは、ランチャーパラメーターがエンジンに起動​​パラメーターを転送するという事実のみを処理するためです。 これらは、指定する必要があるパラメーターです。 同時に、アプリケーション用に独自のアイコンを作成できます。
