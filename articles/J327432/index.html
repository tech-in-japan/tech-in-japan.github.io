マネージャーツールキット：HPE Project Portfolio Management Center

今日、プロジェクト管理の分野で活動する組織は、タイムライン、コスト、リソースだけでなく、ソリューションの企業ポートフォリオ内のデータの視覚化と統合に関連する問題を解決する必要があります。 これらの日々のタスクを考えると、管理者がどのプロジェクトまたは生産活動のタイプに特別な注意を払う必要があるかを判断し、残りの予算を計算し、リソースがどの程度効率的に関与しているか、需要に対応するための作業を計画する方法を理解することはしばしば困難です。







発行



主なタスクは、企業レベルでハイテクソリューションを提供することです。これにより、情報システムの開発、実装、複製、およびメンテナンスのプロセスの統一、整合性、および統合が保証されます。 これは、プロジェクト（リリース）のポートフォリオ/プログラム管理の観点から、スケジューリングだけでは十分ではないという事実によるものです。 次の基本的なエンティティを識別/「修正」し、再利用できるライフサイクル自動化システムが必要です。



•主要なマイルストーンや成果物（ドキュメントテンプレート）など、さまざまなタイプのプロジェクトのライフサイクル。

•自動的に計算する機能を備えたプロジェクト/プログラム/ポートフォリオの「健康」の主要なパラメーター。

•プロジェクトで発生する問題管理のライフサイクル（種類ごとに個別のライフサイクル）。特定の管理分野に沿った自動エスカレーションの可能性。

•特定の管理分野に沿った自動エスカレーションの可能性を伴うプロジェクトフレームワークの変更を管理するライフサイクル。

•役割/セキュリティポリシー（役割アクセス）を考慮したユーザーが利用できる、ポートフォリオ/プログラム/プロジェクトのステータスに関するリアルタイムの統合レポート。



上記のタスクはすべて、プロジェクトポートフォリオ管理（PPM）クラスの専門的な情報システムのレベルで解決されます。



HPE Project Portfolio Management Centerの機能



HPE Project Portfolio Management（HPE PPM）ソフトウェアは、厳しい時間、リソース、および財政的制約の下で、大規模組織のプロジェクトポートフォリオ、プログラム、および個々のプロジェクトを管理するための主要プロセスを自動化するためのプラットフォームです。



ポートフォリオ管理プロセスの概念を次の図に示します。







ポートフォリオ管理システムは、次の問題を解決するように設計されています。



•組織のポートフォリオ管理プロセスの自動化：プロジェクトイニシアチブの収集、ランキング、組織のビジネス目標との整合、ビジネスケースの調整、利用可能な予算とリソースとのポートフォリオの「整合」、現在のプロジェクトと運用活動の考慮

•組織のプロジェクト管理プロセスの自動化：ポートフォリオ、プログラム、プロジェクトの形成。 プロジェクトのリソース管理。 プロジェクト/プログラムの実装プロセスにおける主要業績評価指標（KPI）の監視。 リアルタイムでの組織のプロジェクト活動に関する管理レポートの作成と配信。

•リソース管理プロセスの自動化：リソースプールの作成とメンテナンス。 リソース認定ガイドの維持。 個々のリソースカレンダー プロジェクトにリソースを割り当てるプロセスのサポート。 労働時間の会計のサブシステム（設計および運用活動の両方）。



HPE PPMシステムはモジュール式であり、組織におけるポートフォリオ管理プロセスの実装をトップダウンおよびボトムアップの両方でサポートします。 また、個々の機能の実装もサポートしています-プログラム/プロジェクト、リソースの管理。 設計提案などに続き、機能の拡張と関連機能およびプロセスの適用範囲が続きます。



NRE PPMソリューションの主な機能は、システムオブジェクトをタスクに完全に一致するように柔軟に構成できることです。 各NRE PPM機能には、独自の属性、ワークフロー、およびライフサイクル（LC）があります。 これらの機能のおかげで、作業プロセスの明確な編成とすべての参加者の相互作用が達成され、このプロセス内のパフォーマンス/効率パラメーターの導入と制御が実現します。



HPE PPMソフトウェアを使用すると、次の問題を解決できます。



•戦略的および運用上の要件と要求を視覚的に分析および優先順位付けし、リソースの割り当て、時間コストの計算、および組織の戦略の実装に対する影響の程度を判断して、それらに基づいてプロジェクトとタスクを形成します。

•管理者にプロジェクトとニーズに関する意思決定を行うための単一の「エントリポイント」と「調整システム」を提供し、組織の戦略的活動への影響に関して分析とランク付けを可能にします。

•計画の詳細な精緻化のためのトップダウン計画とデータ収集を提供し、必要なすべてのボトムアップ指標の進捗を監視します。

•変更に迅速に適応し、会社のビジネスプロセスとプロジェクトが戦略的および運用上の目標に確実に準拠するようにします。

•組織の従業員に効果的に負荷を分散します。これにより、最も優先度の高いタスクに最大限の注意を払い、リソースの現在の負荷を明確に理解し、将来のニーズを計画できます。

•次のような重要な「サポート」プロセスを含む、プロジェクトのステータスをリアルタイムで視覚的に監視します。

o変更管理

oプロジェクト範囲管理

oリスク管理

o注文管理

oその他



HPE PPMソリューション概要



技術的には、HPE PPMソリューションはソフトウェア「BPMエンジン」であり、プロジェクトのイニシアチブ/プロジェクト/プログラム/ポートフォリオ、およびコンテンツ管理と承認メカニズムを管理するための作業プロセスを柔軟に構成できます。 ソリューションを使用する利便性のために、ユーザーは、最新のテクノロジーと組織のロールモデルに従ってすべての必要な情報を表示する柔軟なインターフェースを使用して、集約データの視覚化を備えたカスタマイズ可能なポータルを提供されます。



概念的には、NRE PPMはプロジェクトの方法論を自動化し、プロジェクト/プログラム/ポートフォリオのライフサイクルに戦略管理レベルでマイルストーンを組み込むことができます。 プロジェクトの個々の作業を計画および管理する下位レベルでは、Microsoft Projectを使用できます。 NREは、NRE PPMとMicrosoft Projectの間で完全に統合されたフルタイムデータ同期モジュールを提供します。 同期モジュールは、次の3つのデータ転送シナリオをサポートしています。



•NRE PPMはプロジェクト計画を維持するためのマスターシステムとして機能し、Microsoft Projectは補助ソリューションとして使用されます。

•Microsoft Projectはマスターシステムとして機能し、NRE PPMはプロジェクト計画の維持という点で補助ソリューションとして使用されますが、プロジェクト/プログラム/ポートフォリオのKPIを監視するためのマスターシステムとして使用されます。

•同等のシナリオ。



このアプローチにより、HPE PPMに基づく戦略的な計画と管理と、Microsoft Projectに基づく戦術的なプロジェクト管理を柔軟に組み合わせることができ、他の補助ツールは不要です。



HPE PPMの開発された技術メカニズムにより、テンプレートソリューション（要件管理）の記述段階から循環および直接複製の準備までの段階で、プロジェクトの動きの明確なフレームワークと制御性を提供するBPM標準のワークフローを作成できます。 ライフサイクルの各段階（主要なマイルストーン）には、満足のいく/不満足な状態を判断する基準が割り当てられています-特定のタスク/手順のパフォーマンス、ドキュメントの入手可能性および承認。 これらの基準はシステムによって制御され、その満足度/不満は、プロジェクト/プログラム/ポートフォリオ全体の主要業績評価指標/「健康」（KPI）に影響します。



NRE PPMでは、タスク、割り当て、プロジェクトなど、さまざまなエンティティ間の柔軟な関係を構成できます。これらの関係を構成する便利なメカニズムにより、プロジェクトタスク、タスクタスク、タスクプロジェクト、プロジェクトプログラムレベルで複雑な相互作用シナリオを作成できます。など。これらの可能性はすべて、プロジェクト活動の管理における組織のニーズを最大限にカバーします。



このソリューションは、プロジェクト管理の次の重要な側面のワークフローも自動化します。



•変更管理。

•プロジェクト範囲管理。

•リスク管理。

•注文の管理。

•など



これらの各エンティティについて、承認/エスカレーション/決定手順が決定され、各ステップにいるという時間の制約によってサポートされます。 リクエストをタイプ別に分類し、リクエストの特定のタイプの個々のワークフローをリンクすることができます。 柔軟性とロールモデルがサポートされており、最高レベルの複雑さと分散の組織を満足させることができます。



HPE PPMの主要な機能は、組み込みの調整メカニズムであり、あらゆる複雑なレベルのロジックを実装できます。 この場合、外部システムで調整プロセスを開始する必要が生じた場合、NRE PPMはそのような機会を提供します。



ソリューションのアーキテクチャには、ドキュメント管理システム（Documentum）と統合するための既製のメカニズムが含まれています。これにより、プロジェクト活動と連携して外部ドキュメント管理プロセスを使用し、プロジェクトライフサイクルに関与するドキュメントのステータスを自動的に制御できます。



HPE PPMの追加の統合機能は、ITSMクラスソリューション（HPE Service Manager）およびALMクラス製品（HPE Quality Center）との統合です。



•ITSMクラスソリューションは、主要なITプロセスを自動化するためのプラットフォームであり、HPE PPMと連携して、変更管理（インフラストラクチャの変更、CISの改善、プロジェクト実施プロセスにおける新しいソフトウェアの導入）などの主要なプロセスの相互作用とデータ転送のメカニズムを構築できます。

•ALMクラスソリューションは、ソフトウェア開発ライフサイクル自動化プラットフォームであり、HPE PPMと連携して、ソフトウェア製品の変更を実装し、リリースを管理するための全体的なプロセスを構築できます。







また、会計システムおよび人事会計（SAP）、特殊なソフトウェアライフサイクル管理システム（SAP Solution Manager）、および標準のWebサービスメカニズムを使用するその他のビジネスおよびITシステムと統合することもできます。



レポートに関しては、HPE PPMには、プロジェクト活動に関与するすべての役割の主要な制御パラメーターを視覚化する組み込み機能があります。



•会社のトップマネジメント。

•ビジネスブロックの管理。

•プロジェクトオフィスの管理。

•プロジェクトマネージャー。

•通常のパフォーマー。



レポートは、統計データをグラフ形式で表示するコントロールパネルを使用して実装されます（図、「交通信号灯」、「速度計」など）。 コントロールパネルは、利用可能なテンプレートのセットからユーザー自身で簡単に構成できます。 ロールモデルとセキュリティモデルはレポートで完全にサポートされており、ユーザーは、果たしたロールに応じて、利用可能なデータにのみアクセスできます。



HPE PPM仕様



HPE PPMソリューションは3層アーキテクチャに基づいて構築されており、ユーザーインターフェイスとして開発されたWebクライアントを備えています。



Webサービステクノロジーをサポートする外部システムとの統合が可能です。 SAP、Microsoft Project Server、Documentumとの統合の実績があります。



NRE PPMは、最大数十万人の従業員を擁する地理的に分散した最大規模の組織に定着しています。 毎日の同時接続数が5,000ユーザーを超える実装があります。



組織内のシステム開発は、ポートフォリオ管理およびリソース管理プロセスの観点からIT部門内の連続的な垂直スケーリングと、組織内のITおよび非ITプロジェクトとプログラムの両方をカバーするポートフォリオ管理プロセスの観点からの水平スケーリングの両方を意味します。



HPE PPMシステムのライセンスは、最近、簡素化の方向に大きな変更が加えられました。 これまでのところ、所有コストの面では、最も完全で開発された機能を備えた中予算システムのクラスに移行したと言えます。
