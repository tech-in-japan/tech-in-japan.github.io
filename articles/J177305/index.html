以前はメトロと呼ばれていたインターフェース



Windows 8では、Internet Explorerは2つの異なる方法で起動できます-Modern UI（以前はMetroと呼ばれていました）と通常のクラシックデスクトップから。 異なる方法で起動したブラウザは、外観が異なるだけでなく、異なる特性も持っています。たとえば、ModernバージョンではActiveXを起動する方法はなく、異なるプラグインや拡張機能は含まれていません。



このため、Web開発者は、ユーザーが使用するインターフェイスと、ブラウザーを正確に起動する方法を理解する必要がある可能性が非常に高くなります。 残念ながら、これを完全な精度で決定する方法はありませんが、Modernを定義するいくつかの方法をある程度の確率で組み合わせることができます。 したがって、推測の確率は「ほぼ確実」に増加します。



まず、User-Agentにサブストリング「Win64」が見つかったかどうかに注意する必要があります。 「はい」の場合、これはおそらくIEのモダンバージョンです（64ビットモードでクラシックIEを手動で起動できた人の確率はわずかです）。 「WOW64」サブストリングにより、これが古典的なIEであると自信を持って言うことができます。 ただし、User-Agentの32ビットコンピューターでは、どちらのサブストリングも存在せず、この方法でブラウザーを区別することはできません。



ActiveXを実行する機能を確認できます。 

function isActivexEnabled() { var supported = null; try { supported = !!new ActiveXObject("htmlfile"); } catch (e) { supported = false; } return supported; }




ActiveXが利用可能な場合、これは確かにクラシックモードです。 そうでない場合は、モダンモードまたはクラシックのいずれかで、設定でActiveXが無効になっています。



最後に、ブラウザがフルスクリーンモードになっているかどうかを確認できます。 詳細については、 stackoverflow.com / questions / 8751479 / javascript-detect-metro-ui-version-of-ie-10に送信します。



モダンUIモードのIEはいくつですか？



これは非常に興味深い質問です。 それに対する答えは、たとえば、IEのモダンバージョンについて具体的に考え、それに対していくつかの個別の処理を行う必要があるかどうかの決定に影響します。



上記からの概算推定値として、Yandex.Metricaグローバルデータを使用しました。 まず、Windows 8を搭載したIEからのすべてのヒットに対してユーザーエージェントを選択し、それらを4つのグループに分けました。



  • 1グループ。 Win64および「MSIE 10」サブストリングが含まれています。 99％の確率で-メトロモード（1％-IEを64ビットモードで手動で起動できた場合）。
  • 2グループ。 サブストリングWOW64および「MSIE 10」が含まれます。 100％の確率で-クラシックモード。
  • 3番目のグループ。 サブストリング「MSIE 10」が含まれますが、Win64またはWOW64のいずれも含まれません。 これは、それがどのモードであるかを言うことではありません。
  • 4グループ。 サブストリング「MSIE 10」は含まれません。 これはIEではなく、IEをそれ自体に統合する何らかの無関係なアプリケーションです。 興味はありません。




次に、最初と2番目のグループのヒット数を追加し、金額の最初のグループのパーセンテージを調べました。



結論：Win8上のIE10の約5.4％がモダンモードで起動されます。 この番号がお役に立てば幸いです。
