Visual Studioでの.NET開発用のお気に入りのCodeRush機能の10



要するに、 DevExpress CodeRushは生産性ツールクラスに関連するVisual Studioの有料プラグインであり、開発者はより高速で優れたコードを記述し、デバッグし、テストを実行し、欠陥を検出し、その他の有用な機能を実行できます。



この記事では、その使用に関する自分の好みのリストを収集するだけでなく、提示されたチップのほとんどが出力で私のチームの他の人によって使用されるように小さな検証も行いました。 Visual Studioの積極的な開発（2012バージョンは特に喜ばれた）と記述された関数の一部をネイティブ関数に変換したにもかかわらず、CodeRushやReSharperなどのサードパーティヘルパーの必要性は、時間を節約し、全体的な使いやすさを向上させるため、一部の開発者にとって依然として重要であると考えていますコーディング。 最後に、プロジェクトの特性、予算、または単なる習慣のために、Visual Studio 2010に座っている開発者が（少なくとも多くの顧客から判断して）まだ十分であることを忘れないでください。



だから、他の.NET開発者がソフトウェア設計プロセスの小さな、しかしエキサイティングな部分の効率を改善するために使用しているものを知りたい人は、猫の下で歓迎してください（注意、多くの写真やビデオ、そして投票！）





私は事前に、「ホットテン」が製品の機能のごく一部しか示していないことを警告します（私自身は5％未満しか使用していないと思います）。 読者の便宜のために、ロシア語の機能名に加えて、プログラマーの耳に典型的な英語名を付け、デフォルトのキーボードショートカット（存在する場合）を示し、標準スタジオ機能の類似点も示します。



1. Visual Studio 2010のソリューションエクスプローラーでプロジェクトツリーを折りたたみます-/プロジェクトに折りたたみ



多くの場合、多数のプロジェクトで構成される大規模なソリューションを使用する必要があるため、この機能は、プロジェクト内の適切な場所を見つけるために単純に置き換えられません。







Visual Studioのさまざまなバージョンで数年間使用してきましたが、これまで非常に満足しています（この機会の前に、マクロがカットされたスタジオの最新バージョンであるEMNIPでVBマクロを使用しました）。 すべてのプロジェクトがVisual Studio 2012+をまだ使用しているわけではないため、この小さな機能が一部の人々にも役立つことを願っています。











2.ソリューションでのクイックファイルナビゲーション/クイックファイルナビゲーション（ Ctrl + Alt + F ）



ソースコードを含むファイルを名前で知っていて、すぐにアクセスしたいことは珍しくありません。 これを行うには、Control + Alt + Fコマンドは単に置き換えられないため、可能な限り迅速に移行できます。







Visual Studio 2012以降、同様の機能がソリューションエクスプローラーで利用可能になりました。











3.ソリューションのタイプとメンバーによるクイックナビゲーション/クイックナビゲーション（ Control + Shift + Q ）



このオプションは前のオプションと非常に似ていますが、唯一の違いは、ファイルをナビゲートするのではなく、すべてのタイプとそのメンバーをナビゲートすることです。







率直に言って、ソリューションが単純に巨大な場合、追加の設定なしでこの検索ダイアログを呼び出すと、明らかな理由で顕著な遅れが生じる可能性があります。 これを回避するには、タイプとメンバーの種類、アクセス、および地域ごとに小さな検索チューニングを行うことをお勧めします。これにより、ガベージの結果を取り除き、応答性も向上します。 もちろん、呼び出しごとにリストを再構築するのではなく、再構築する前にキャッシュしておくとさらに良いでしょう（IDEチームにそのようなオプションを作成するよう依頼する必要があります）。



別の方法として、昔ながらのスタジオ検索を使用することもあります。たとえば、「ss ClassName」行でControl + Shift + Fを使用しますが、これも原則としてはうまく機能します。 また、この目的には標準のナビゲート先 （Control +、）チップが便利ですが、あまり頻繁に使用しません。











4.型とメンバーが使用されている場所のナビゲート/ Tab to Next Reference（ Tab / Shift + Tab ）



これはおそらく私のお気に入りの機能の1つです！ これにより、型やメンバーがどこでどのように使用されているかをすばやく見つけることができます。たとえば、このメソッドはどこから来たのかなどです。 これを行うには、コード内の目的のオブジェクトにカーソルを置き、Tab（前方）またはShift + Tab（後方）を押します。これはWindowsユーザーにとって非常に自然で理解しやすいものです。







同じものには標準的なスタジオ収縮（Control + Shift + Arrow Up / Down）がありますが、もちろん左手に少し痛みがありますが、もちろんすべてを自分で再構成できます。

私が訪問するファイルで不要な使用を取り除き、すべてを美しくきれいに見せたいです。 あなたが私と同じusing-naziである場合、この機能はあなたのためです：

設定でデフォルトの並べ替えを選択できることに注意してください。たとえば、アルファベット順ではなく、長さがあります。











6.コード分析と書式設定/コード分析とコードスタイル（ Control +〜 ）



要するに、このスタジオで作成されたツールは、基本的にはステロイド1本のFxCop + StyleCopであり、コードエディターの右側の細いバーで視覚的に表されます。







このストリップは、目立たないようにマルチカラーのセリフ （色は問題の重大度を示します）を示し、コードで起こりうる問題についてプログラマーに示します。 診断された問題の完全なリスト（ソリューション内のコードの重複も含む！）はここにあります 。

他の同様のツールと同様に、このアシスタントは、必要なヒントのみが表示されるように、お客様または会社が採用したコードルールに対して柔軟に構成できます。











そして最後に、あなたが妻として分析エンジンを信頼している場合:-)、ソリューションエクスプローラーのファイルのメニューには、発見された問題を自動的に「クリーン」にするマジックコマンドCode Cleanupがあります。 しかし、正直なところ、このコマンドを使用することはありません。誰かが完全に自動でコードを修正してくれるのが気に入らないからです。 （コンテキストメニューでの偶発的なクリックを避けるために）かなり深刻なように思えるので、このコマンドの確認ダイアログを追加するように提案したことも覚えています。



重複コード （重複コードの検出と統合）およびフォーマットの検索に関しては、これらのチップの使用頻度ははるかに低く、特に注目することはせず、興味のある人のためにリンクを残します。



一般に、この組み込みのコード分析は、必要に応じて、ペアプログラミングパートナーの愚か者または鋭い目に対する、かなり許容できる初期防御です。 初心者にとっては特に役立つと思います。NET開発者。

結論として、前述のVisual Studio 2012でのFxCop / StyleCopに加えて、同様の機能を実行するコード分析ウィンドウが表示されたことに注目する価値があります。











7.タイプをファイルに削除、タイプまたはファイルの名前を変更/タイプをファイルに移動、名前の変更、ファイルの名前を一致タイプに変更（ Control +〜 ）



多くの人は、1つのソースファイルに1つのパブリッククラスのみを含めることをお勧めします。そうでない場合は、[タイプをファイルに移動]コマンドを使用すると、できるだけ早く修正できます。 関連する「ファイル名を一致タイプに変更」コマンドを使用すると、ファイルの名前が一致しない場合、そのファイルのタイプ名でファイルの名前を変更できます。 最後に、目的の要素にカーソルを置いた後、F2を直感的に押すことで、任意の型またはメンバーの名前を変更できます。







8.基本型コンストラクターの追加/不足しているコンストラクターの追加（ Control +〜 ）



多くの場合、継承中に、基本タイプのコンストラクターをパラメーターの束で実装する必要があります。これは、自分で入力しない方が望ましいですが、どこかからコピーするか、自動的に生成することをお勧めします。 このコマンドを使用すると、マウスを1回クリックするか、キーボードを数回押すだけで、すべてのルーチン作業を実行できます。





その前に、私は通常F12を介して基本型の宣言に移動し、そのコンストラクターの署名をコピーしました。



9.メソッドまたは変数の削除/メソッドの抽出、スコープの拡大（ Control +〜 ）



多くの場合、メソッドを単純化するか、変数のスコープを広くする必要があります。 これらのタスクにはCodeRushの機能が好きです。

10.高速コード生成用テンプレート/コード生成テンプレート



ここでは、私が最も頻繁に使用する略語をリストしたいと思います。これは、ほとんどすべてのプログラムでおそらく役立つでしょう。

  • ss-コンストラクターを型に追加します。 そのシンプルさにもかかわらず、これは標準のテンプレートとは異なり、型メンバーを分析し、パラメーターとして追加することを提案する非常にスマートなテンプレートです。これはしばしば便利です。
  • f -forループを追加し、コレクションなど、以前にクリップボードにコピーされた変数から自動的に上限を取得できます。
  • fe -foreachループを追加し、コレクションなど、以前にクリップボードにコピーされた変数から要素のタイプを自動的に計算できます。
  • r？、p？ -通常、最初の文字で選択される新しいタイププロパティを追加します。 たとえば、pはオブジェクト型のプロパティを作成し、piはint型のプロパティを作成し、pdはdouble型のプロパティを作成し、pbはブール型などを作成します。 読み取り専用プロパティが必要な場合は、pの代わりにrが使用されます（rbなど）。
  • x？

    仕事の性質上、XAFおよびXPOフレームワークのデータモデルを作成することが多いため、xで始まる特別なテンプレートを使用します（xc、xp？、xr？、xpcl、xpaはデフォルトで）

もちろん、これらはすべてテンプレートまたは略語ではなく、興味のある人は、 ここで完全なリストを確認するか、 Control + Alt + /の組み合わせを使用してVisual Studioで直接呼び出すことができます（世界のだれもが全体の20％これらの略語の）。
