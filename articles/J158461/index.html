OpenStreetMap.orgプロジェクト地図作成をメモリカードスロットなしでGARMINナビゲーターにダウンロードする

こんにちは、民衆の知恵は、貪欲は常に支払うと言います。 私の友人の1人は鉱夫であり、趣味では漁師とハンターが退職し、ハイテクの世界に参加することを決めました。 私は一生、自然なナビゲーションを使用してシベリアのタイガを歩きましたが、ここで、店の売り手と独占的に相談した後、奇跡のデバイスGARMIN ETREX 10を購入しました。 しかし、私はすべてのナビゲーターに追加のメモリ用のスロットがあるようになったと思うので、ノボシビルスクのある白いフィールドよりも地図をダウンロードすることを約束しました。 2012年に驚いたことに、GARMINは内蔵メモリと非常に控えめなサイズのデバイスのみをリリースしました。 OpenStreetMap.orgのオープンプロジェクトのKuzbassマップは適合しましたが、デバイスはトラックを記録できず、スペース不足について不満を言いました。 ナビゲーターの所有者が通常、狩りや釣りをする部分だけをダウンロードし、できるだけ高さを増やすことにしました。 GPSMapEditでポーランド語形式のマップをダウンロードし、不要なものをすべて削除した後、IMGファイルに保存しようとしましたが、道路上のポイント数に関するエラーを多数取得し、別の解決策を探し始めました。



次に試したのはMap Composerでしたが、インターフェイスをドイツ語から英語に変更できませんでした。 Web検索はGroundTruthにつながりました 。 Microsoft .NET 3.5以降とcgpsmapperが動作する必要があり、 無料版に適しています明確なドキュメントとcmdファイルの数行： 



groundtruth getdata –bu http://www.openstreetmap.org/?lat=53.69&lon=87.059&zoom=10&layers=M groundtruth contours --bu "http://www.openstreetmap.org/?lat=53.69&lon=87.059&zoom=10&layers=M" groundtruth makemap -ibf=output.ibf groundtruth ibf2osm groundtruth makemap -rules="http://wiki.openstreetmap.org/wiki/GroundTruth_Hiking_Map"




彼らはデバイスにロードする準備ができたIMGを発行しましたが、カードの署名はキリル文字ではありません。 元鉱夫にとって、これは良くありませんでした。



次に使用したのは、Linuxを使用している場合にMicrosoft .NET 2.0またはMonoを必要とするJavaプログラムMkgmap ＆ Srtm2Osmでした 。



openstreetmap.org Webサイトで目的の領域を開き、「エクスポート」アイテムを選択します。 OpenStreetMap XMLの形式をマークします。 領域が大きすぎる場合、[エクスポート]ボタンは無効になります。 比較的大きな集落をダウンロードしようとすると、 「リクエストしたノードが多すぎます（制限は50000）。 より小さなエリアをリクエストするか、planet.osmを使用してください 。 」 表示領域を移動して、いくつかのosmファイルを拡大してダウンロードします。 



 java -Xmx512m -jar mkgmap.jar --code-page=1251 --tdbfile --gmapsupp --country-abbr=RUS --country-name=RUSSIA --route --description="Openstreetmap South Kuzbass" *.osm




複数のファイルからgmapsupp.imgを選択し、デバイスにコピーします。 デバイスによっては、ファイルの名前をgmapbmap.imgに変更してベースカードを交換する必要がある場合があります。



歩行者ナビゲーションの場合、地図上に高高度データがあると便利です。 世界中のほとんどのレーダー地形調査（SRTM）は、いくつかの方法で無料で利用できます。 Srtm2Osmプログラムを使用すると、いくつかの方法で必要な領域でこのデータを取得できます。



座標から境界を指す-bounds1 53.49 87.05 53.99 87.55

そこからkm単位でポイントと半径を指す-bounds2 53.69 87.05 10

または、openstreetmap.org -bounds3 " www.openstreetmap.org/?lat=53.69&lon=87.059&zoom=10&layers=M "からのリンク



結果のosmファイルをosmファイルにコピーし、上記のマップアセンブリを繰り返します。







Habrに関する私の最初の記事が、メモリカード用のスロットのないGARMINナビゲーターの所有者を助けたり、GPSMAP 276Cなどの伝説に2度目のチャンスを与えることを願っています。
