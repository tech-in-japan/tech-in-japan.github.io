ReSharper 7.1がリリースされました





数か月間、 私たちはReSharperの次のバージョンのリリースに精力的に取り組みました。 そして今-それが起こった！ 正式にリリースされたばかりのバージョン7.1は、 ここからダウンロードできます 。



バージョン7.1にアップグレードする理由はたくさんあります。



  • バージョン7.0以降、300以上の欠陥とパフォーマンスの問題が修正されました ！ 特に注目すべきは、SharePointおよびASP.NET MVCプロジェクト、およびリソースファイルを使用する際の改善点です。


  • コードの書式設定が改善されました 。この重要な領域では、これまでで最も要求の高い改善が導入されました。 そのため、ReSharper 7.1はXMLドキュメントのコメントとメソッド呼び出しチェーンをフォーマットできます。 最近 、これについてJetBrains .NET製品専用のブログで書きました。


  • ReSharperコード検査結果をエクスポートして、チームメンバー間で共有することがより簡単になりました 。 さらに、コード検査結果をコピーして作業項目に変換できるようになりました。ブログ投稿「ReSharper検査結果の簡単な共有」で詳細をご覧ください。


  • Windows Phone 8のサポート ：アプリケーション開発に既にWindows Phone 8 SDKを使用している場合、ReSharper 7.1はこれをより速くより良くするのに役立ちます。 正確にどのように？ 最近のWindows Phone 8サポートのブログ投稿を読んでください。


  • VB.NETサポートが改善され、Extractクラスのリファクタリング、新しいクイックフィックス、強化されたIntelliSenseが含まれます。


  • IntelliSenseを含むXAMLの大幅に拡張されたサポートは 、文字入力を支援し、命名スタイルとコード生成を制御します。 XAMLサポートスペシャリストは、最近、 彼がそれについて知っていることをすべて教えてくれました 。


  • INotifyPropertyChangedインターフェイスの実装のサポートは 、 Windowsフォーム、WPF、およびSilverlightアプリケーション開発者に関連するより多くの使用シナリオをカバーしています。


  • その他の改善 （多くのリファクタリングの修正、Entity Framework 5、ASP.NET、ASP.NET MVC、およびJavaScriptのサポートの改善など）。


2012年6月1日までに購入したバージョン7のライセンスまたはバージョン6のライセンスをお持ちの場合、ReSharper 7.1に無料でアップグレードできます 。



ReSharperのプラグインを使用している場合、心配しないでください-バージョン7.1との互換性をテストしています。 それらのほとんどは、数日中にダウンロードできるようになります。



ReSharper 7.1をダウンロードして、気に入ったものとそうでないものを教えてください！
