独自のVPSホスティングをゼロから作成し、その上でお金を稼ぐ方法は？ WooCommerceによる簡単な請求





こんにちは、Habrの親愛なる読者。 この記事は、 RUVDS White Label APIに基づいてゼロからVPSホスティングを構築する方法に関する一連の資料の続きです。 サイクルの入門記事では、まずサイトが必要であるという事実について話し、それを作成するための3つのオプションを提案しました。 人気のあるCMSとプラグインを使用したオプションについて詳しく説明します。これらのオプションは、支払いの受け取りとサーバーの支払いに必要な機能を迅速に提供するように設計されています。



一連の出版物では、Wordpress、Joomla、Drupalなどの人気のあるCMSを取り上げます。



なぜ選択したのですか：

  • ほぼすべての一般的な共有ホスティングでサポートされています。
  • PHPで書かれています （低しきい値、広範なコミュニティ、多数の参考資料、フリーランスプログラマー）
  • 無料（最小限の投資で開始するにはこれが重要です）
  • インターネットで配布




免責事項：これらのCMSが最も人気がある、または最高であるとは主張しません。 また、PHPがナンバーワンの言語であり、単にそれだけを使用しなければならないという事実についても話していません。 独自のVPSホスティングを作成する問題の解決に適した技術を使用できます。 第三者のソースへの多くのリンクがこの資料に追加されており、その内容はテキストに含まれていません。そうしないと、内容が読みにくくなります。 それらが役に立つことを願っています。 それでは始めましょう。



この記事では、Wordpress（以前の記事でホスティングにWordpressを詳細に展開する方法）とそのWooCommerceプラグインを検討し、VPSホスティングに適応させる方法についても説明します。



このプラグインは、Wordpress向けの最も強力で機能的な無料のeコマースソリューションの1つであると考えられています。 それに加えて、「すぐに使える」と呼ばれる最新のプラグイン機能を完全にサポートする専門のストアフロントテーマがあります。 このトピックをわずかなデザインの改善で使用するか、ここでたとえば50〜70ドルで有料のものを購入できます。 すぐに目を引くのは、 ロボカッサやペイマスター （両方とも無料）など、さまざまな支払いアグリゲーターとの統合など、多くの拡張機能の存在です。



長所





  • インストールのしやすさ。 段階的なインストールプロセスでは、すべてが直感的です。 プラグインとテーマをダウンロードし、アクティベートします。 その後、すぐに製品を追加して、さらに微調整することができます。
  • 膨大な数の有料および無料のテンプレートの存在
  • シンプルで便利な追加製品
  • 拡張性。 ボックス化された機能に加えて、多くのプラグイン拡張があり、目的の機能を取得できます。
  • 無料です。 言葉なし
  • 絶え間ない改善と更新。 プラグインは積極的に開発されており、githubのリポジトリへの最後のコミットは数日前（執筆時点）でした。
  • さまざまなレポートを作成するための機能
  • すぐに使えるクーポン、プロモーション

  • いくつかの拡張機能を一緒にインストールすると競合する場合があります

  • 安全性 世界でのCMSワードプレスの人気と拡張機能自体の人気を考慮すると、このエンジン上のストアは多くの場合、ハッカー攻撃の標的になります




VPSホスティングの適応の機能



最初に何をする必要がありますか？ まず、テーマとプラグインをインストールしてアクティブにします（インストールは直感的で、通常のテーマとプラグインのインストールと変わりません）。 この記事では、今後のVPSホスティングのためのページのデザインは考慮しません。 顧客がVPSホスティングでサーバーを注文できるように、このWooCommerceを構成する方法に興味があります。



支払いを受け入れる



ここでは、すべてが他の商品の支払いを受け取る組織とまったく同じです。 woocommerceのアグリゲーターとの統合を実装するプラグインはたくさんあります。 ロシアとCISでの支払いを受け入れるために無料のwoocommerce拡張機能から選択する場合は、robokassaまたはpaymasterを接続できます。 ロボカッサの場合、サイトに登録した直後に、オファー契約に基づいて個人としてすぐに支払いを受け入れることができますが、この場合、 高い手数料がかかります。 一般に、これを念頭においても、サーバーの最終価格は競合他社にとって非常に魅力的であるため、robokassaを使用してすぐに安全に作業を開始できます。 robokassa拡張機能の設定の詳細については、 こちらをご覧ください。

別のアグリゲーターを使用してこの問題に非常に慎重にアプローチする場合、アグリゲーターの詳細な比較を以下に示します 。 適切なものを選択し、 WooCommerceに適切な拡張機能があるかどうかを確認し、そのリポジトリを見つけて、開発がどれだけ活発であるか、それが生きているかどうかを確認します。 すべてがうまくいけば、自由に設定してカスタマイズしてください。



ゲストのショッピングと配信を無効にする



WooCommerceには、登録せずに購入する機能があります。 クライアントが自分のアカウントからのみ購入できるようにする必要があります。その方法では、クライアントに関するすべての必要な情報（クライアントに関するサーバーを含む）をクライアントに表示します。 これを行うには、Wordpressの[設定]> [WooCommerce]> [設定]> [支払い]に移動し、[ゲストに注文を許可する]チェックボックスをオフにします。



配信も無効にする必要があります。 これを行うには、 Wordpressの設定> WooCommerce>設定>一般に移動し 、「配信と配信計算を無効にする」を選択します

有料注文の処理



そこで、Voosのレンタルサービスの販売にWooCommerceを適応させるという点で最も重要なポイントの1つになります。 サービスでVPSサーバーの料金を支払った後、このサーバーをクライアントに提供する必要があります。



プレゼンテーションを簡単にするために、このセクションでは、サーバーエクステンション、試用期間などの整理方法については考慮しませんが、最初の支払い後にサーバーを作成することに制限します。 そのため、タスクはサービスが支払われるとすぐに確認することです：

  • RuVDS Whitelabel APIを使用してサーバーの作成を開始し、作成プロセスに必要なすべてのデータ（サーバー識別子、作成段階、作成の進行状況、サーバー構成）をクライアントに提供します。
  • サーバーの作成後、接続に必要なデータ、サーバー管理機能、および請求期間の終了日に関する情報を提供します


この問題を解決する方法はたくさんあります。 最も簡単な方法の1つは、VPSが対応するワードプレスおよびwoocommerceテーブルから直接作成されていない有料注文に関するデータを定期的に受信することです （woocommerceスキームの詳細な説明はここにあります ）。サーバー作成プロセス。 最良の解決策ではありませんが、最も簡単な解決策の1つです。このタスクにwoocommerce Webフックを使用することもできます。 Webフックとは何ですか？ 簡単に言えば、これはオンラインストアのイベントのサブスクリプションです。 REST APIを使用してこのメ​​カニズムを使用する方法を示します。



なぜwoocommerceは良いのですか？ 特に、文書化されたREST API （およびJavascript、PHP、Python、Ruby、curlの4つの言語での使用例があり、出力を必要に応じて解析できるため）。 また、プラグインの機能を使用するには、 wp cliとそのコードに関する詳細なドキュメントがあります。



REST APIを使用するには、最初に特別なキーを生成する必要があります。 これは次のように行われます： WooCommerce>設定> APIに移動し、キーの追加をクリックします。 キーを生成するときは、ユーザーを選択し、権利の種類（読み取り、書き込み、読み取り/書き込み）を指定する必要があります。 現在のユーザーのキーを生成し、読み取り/書き込み権限のレベルを示します。 次に、 [設定]> [パーマリンク]に移動し、2番目のタイプを選択します。 これで、REST APIはurl： yourdomain.com/wc-api/v3で利用できます



APIを使用するには、PHP用の特別なライブラリを使用します 。 コンソールで実行して、 composerを使用してインストールします。 

composer require automattic/woocommerce


次のPHPコードを使用して、APIにログインし、製品のリストを取得できます。 

 <?php require __DIR__ . '/vendor/autoload.php'; use Automattic\WooCommerce\Client; use Automattic\WooCommerce\HttpClient\HttpClientException; $woocommerce = new Client( 'https://yourdomain.com/', 'ck_0000000000000000000000000000000000000000', 'cs_0000000000000000000000000000000000000000' ); try { print_r($woocommerce->get('products')); } catch (HttpClientException $e) { echo $e->getMessage(); }




クライアントは、メインページで利用可能なショーケース機能を使用して、すでにサーバーを購入できます。 したがって、注文の支払い時にサーバーの作成を開始するために、対応するWebフックを注文のステータスを変更するイベントに追加できます。 

 ... ... $webHookData = [ 'webhook' => [ 'name' => 'Order updated', 'topic' => 'order.updated', 'delivery_url' => 'https://yourdomain.com/deliver/secret' ] ]; try { print_r($woocommerce->post('webhooks', $data)); } catch (HttpClientException $e) { echo $e->getMessage(); }




これで、このイベントが発生すると、Webフックのロードがアドレス「https://yourdomain.com/deliver/secret」に配信されます。

Webフックによって配信されたすべてのものは、次のように表示することもできます。 

 print_r($woocommerce->get('webhooks/webhook_id/deliveries'))


delivery_urlへのリクエストを処理するとき、注文のステータスが完了に変更されたかどうかを確認する必要があります。 その場合、注文からidサーバー設定の配列（line_itemsプロパティの製品識別子の配列）を取得し、VPSの作成プロセスを開始します。
