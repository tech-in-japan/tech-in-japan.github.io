Yandex.Marketでの商品の配置。 一般的な問題とその解決策

著者として、私は主に実践に興味があると言わなければなりません（実用化のメリット）。 したがって、私は個人的に遭遇するものだけを記述するよう努めています。 現在、オンラインメガネストアを開発しており、Yandex.Marketに商品を置いた経験を共有することにしました。







この記事は、 マネージャー 、 Web開発 者 、およびYandex.Marketに商品を配置することを計画しているオンラインストアの所有者を対象としています 。 この資料には、一般的な問題、Yandex.Marketカテゴリによる製品の分類の詳細、および最新のトラブルに関する情報が含まれています。 価格表（特別なYML形式）の可用性と技術的な正確さは、検索で表示される製品のリスト全体がユーザーに表示されることをまったく保証しないことに興味があります。 オンラインストアの所有者は、彼の製品提供のn番目の部分が潜在的なバイヤーから隠されていることに長い間気付かないかもしれません 。 この資料は、Yandex.Marketに商品を配置する際にどのような落とし穴が予想されるか、およびその対処方法を理解するのに役立ちます。 この記事では、商品をYandex.Marketに配置するための価格を取り上げ、協力のオプション（技術サポートサービスまたは個人マネージャー）について説明します。 この記事には、YMLファイルをデバッグするための小さなコードが含まれています。





パート1.オンラインストアの登録、価格表の確認、トラブルシューティング



Yandex.Marketシステムに配置するには、オンラインストアで提示された商品をブロードキャストするために、事前にYML（vs XML）形式の価格表を準備する必要があります。 このようなYMLファイルの仕様は、YandexページのMarketセクションにあります。 オンラインストアが、Yandex.Marketに商品を配置するニーズに適合していないCMSに基づいている場合（原則として、まさにそうです）、YMLファイルを生成するためのスクリプトを手動で記述する必要があります。 このようなファイルの実用的で段階的な開発の例は、 Yandex.Market（osCommerce 2.3のスクリプト）で商品を放送するための私のブログのYMLファイルの記事です。 この記事では、コードを記述するときにこのスクリプトを定期的に参照します。



私たちのサイトにYMLファイル用の既製のスクリプトを作成したら、いくつかのフィールドに記入する必要があります-いわゆる。 「配置オプション。」 新しいストアを登録するためのリンクはこちらです： partner.market.yandex.ru/pre/index.xml （Yandex.Webmaster-製品と価格と混同しないでください）。 配置オプションには以下が含まれます：価格リスト（サイトのYMLファイルへのリンク。コンピューターからYMLファイルをダウンロードできますが、15日ごとに更新する必要があります）。 配送情報（地域、費用）; 法的情報（法的情報はウェブサイトに掲載する必要があります-ページのアドレスを指定する必要があります）、注文を受け取るための有効な電話番号（Yandexの従業員は商品を注文する際にこの番号にコールバックします）など。確認のための価格表。



YMLファイルで起こりうるエラー


スクリプト自体が正しく記述されていたとしても、たとえば製品イメージのアドレスや製品の複製など、多くのエラーが発生する可能性があります。 彼らが私に電話したとき（明らかに、私のブログの資料に案内され、提供された連絡先の電話番号を使用して）、画像の何が間違っているのかを尋ねるケースがありました このエラーは、Yandex.Marketのテクニカルサポートで示されました。



画像の出力に関するエラーは、ほとんどの場合、サイトにアップロードされた画像の名前に無効な文字（スペースなど）が使用されている可能性があるためです。 そのような名前をYandexで消化可能なURLに変換するには、対応するphp関数rawurlencodeを使用するだけで十分です。 カタログイメージのURLの出力が実行されるコードフラグメントを探しています。 OsCommerce 2.3のスクリプトでは、これは次のセグメントです。



$src_file = $csite."catalog/images/".$tovar['picture'];

$ppy = "".$src_file."";







rawurlencode urlエンコード関数を追加します。 したがって、カタログから画像を出力するコードは次のようになります。



$src_file = $csite."catalog/images/".rawurlencode(($tovar['picture']);

$ppy = "".$src_file."";







テイクをどうするか？ 重複する製品の間違いは、それほど重要ではありません。 これらは、2つ（またはそれ以上）の製品の表現で同じ名前/同じメイン画像を使用するという事実のために表示されます。 そのため、モデルが同じでも色が異なる場合は、名前に色（青、黒など）を割り当て、そのような製品のメイン画像を対応する写真に置き換えるだけで十分です。



パート2. Yandex.Marketの価格と協力オプション。 個人管理者に与えられるもの



Yandex.Marketでの支払いはクリックで行われます。 CPC-本と1.5ルーブル（0.05 cu）と3ルーブル。 （0.1 cu）他のすべての商品。 ただし、Yandex.Marketを使用するには、頭金を支払う必要があります。 ここでは、Yandex.Marketが協力のために2つのオプションを提供しています。



  • オプション1.標準。 300こすります。 入金された資金はアカウントに残り、クリックの支払いのみに使用されます。 サービス-技術 サポート（数日以内に質問に答えます）。
  • オプション2。パーソナルマネージャーの「アタッチメント」付き。 寄与9000こする。 入金された資金は、1番目のオプションと同様に、お客様のアカウントに残り、クリックの支払いのみに使用されます。 サービス-パーソナルマネージャー（数時間以内にメールで、または必要に応じてリアルタイムで電話で質問に答えます。これは非常に便利です）。 すべての問題は十分に迅速に検出および解決されます。


したがって、パーソナルマネージャーがサービスを提供しても何も失うことはありません（寄付の全額はクリックにのみ費やされます）が、最初は良い利点が得られ、この段階で主な質問とトラブルが発生します。 個人マネージャーとの仕事は最初の月よりも長い期間可能ですが、同時に、月々の費用は21,000ルーブルを超える必要があります。これは具体的には「追いつく」必要があります（たとえば、高値のクリック率）。私の意見は意味がありません。



非常にsc帳面な人々（細心でなければ）、私とサイト所有者（顧客）の両方に、長い間、そしてこれらの9000人がどこに行くのかを詳細に尋ね、最終的に私たちは個人的なマネージャーと働くことの合理性を確信しました、1日あたり100クリック（1日あたり300ルーブル= 1か月あたり9000ルーブル）でランダムに取得したとしても、この金額はかなり正当であると計算されます。 この機会を利用して、事前にYandex.Marketチームの全体的な印象（非常に好意的だった）を共有し、個人的なマネージャーであるNatalya Malakhovaのプロ意識と注意を称賛します。



パート3。私は象ではなくハリネズミを意味しました。 Yandex.Marketの間違ったカテゴリ定義を脅かすもの、および正しい方法



そのため、商品の写真カードを使用して価格表のチェックに合格し、すべてが順調で、料金を支払い、オンラインストアが「オン」であることが判明しました。 「公開されたオファー」の概要を取得することもできます。 Yandex.Marketで2、3の主要なクエリを実行して、製品を見ることができなかった場合は、リラックスする準備ができています。 ただし、ほとんどの要求については完全に不在であり、他の要求についてはわずかな量で表されます。 また、何らかの理由で、検索結果から得られるすべてのビジョンメガネがサングラスになっています（それ自体は正しくありません）。 もっと驚く。 マネージャーとコミュニケーションをとると、ほとんどの商品が「サンダル」、「メンズシューズ」、「アンダーパンツ」などのカテゴリに分類されていることがわかります。マネージャーの合理的な仮定によると、これらの商品は重要なクエリを検索していません このような矛盾は非常に典型的です。 たとえば、ナタリアは、バスケットが「プール」のカテゴリにどのように分類されたかという話をしました-丸いという理由で。



何が起こるか見てみましょう。



私たちのパーソナルマネージャーによると、Yandex.Marketは、多数の標識に従ってカテゴリ別に商品を自動的に分類します。 さらに、Yandex.Marketでの検索は、製品の名前だけでなく、同じ基準に従って実行されます。 製品の名前は「エンポリオアルマーニ9730 ATGメンズメガネを購入する」のように見えました。 名前に「購入」という言葉を使用することは市場にとって受け入れがたいものであり、ナタリアは一般に品質管理（CCM）に合格したことに驚きました。 一番下の行は、名前を緊急にリダイレクトする必要があるということです。 サイトでそれらを手動で書き換えるのは壮大なタスクです（数百のポジションについて話します）。 安定した組み合わせを置き換えることで、YMLファイル自体で修正するのがはるかに簡単です（ここでは、「男性用メガネを購入する...」、「女性用メガネを購入する...」、「子供用メガネを購入する...」など） 。 さらに、フレームの検索に表示する必要があるため、「フレーム」という単語を追加する必要があります。 製品の名前を表示するコードフラグメントを探しています。 私のスクリプトでは、これらは次の行です。



$tovar['name'] = htmlspecialchars($tovar['name']);



...

$tovar[name]







php関数str_replaceを使用して、安定した組み合わせを置き換えます。



$tovar['name'] = htmlspecialchars($tovar['name']);

$tovar['name'] = str_replace(' ', ' ()'), $tovar['name']);

$tovar['name'] = str_replace(' ', ' ()', $tovar['name']);

$tovar['name'] = str_replace(' ', ' ()', $tovar['name']);

$tovar['name'] = str_replace(' ', '', $tovar['name']);

$tovar['name'] = str_replace(' ', ' ', $tovar['name']);

...

$tovar[name]







その結果、「男性用メガネエンポリオアルマーニ9730 ATGを購入」というタイプの名前はすべて、「男性用メガネ（フレーム）エンポリオアルマーイ9730 ATG」に置き換えられました。 その後、当社の製品はYandex.Marketによって必要なカテゴリ（光学-メガネ-フレーム）に分類されました。 キークエリを検索するときは、すべてがうまくいきます。



最後に推奨事項。 Yandex.Marketの適切なカテゴリに入るため、および主要なクエリのYandex.Market検索結果に正常に表示されるようにするには、適切なクエリの検索結果に参加している競合他社の既に公開されているオファーで製品名を事前に確認してください。 私たちの場合、これらは「Eyeglass frame [brand] [model] ...」という形式のオファーでした。 そのような名前を完全にコピーする必要はありません。組み合わせの典型性を観察すれば十分です。
