なぜMac



最近、私の友人や知人の多くが、なぜPCからMacに移行したのかと尋ねてきました。

最初に尋ねた2人に説明しましたが、残りは十分ではありませんでした:-)

拡張ブログエントリを作成し、コメントで議論することを約束したのはそれだけです。







まず、少し歴史。 これは、コンピューターの世界における私の見解を大部分説明するでしょう。

私は10年からコンピューターに精通しており、最初のコンピューターはカセットプログラムを搭載したZX Spectrumでした。

および組み込みのインタプリタBasic 。 学校の終わりまでに、私はすでに通常のPCとWindowsを持っていました

私にとっては、新しいことでも面白いことでもありませんでした。 次に、球を探して

私はunix-systemsの学習を掘り下げることにしました。 したがって、最初の年に私はすでに練習を受けました

アシスタント管理者としてローカルWebサイトにアクセスします。 彼が3年以内にunix-fuのすべての主要なトリックをマスターした場所。

それ以来、私にとって、コマンドラインはコンピューターでの快適で迅速な作業に不可欠であり、必要になっています。

次の数年で、私は徐々に仕事の範囲を管理者からプログラマーに変えました。

むしろウェブ開発者になりました。 完全な仕事をするには、 アドビ製品が必要でした-

PhotoshopおよびIllustrator。 そのため、Windowsに切り替える必要がありました。 しかし、開発自体は私には関係ありません

コマンドラインとターミナルエディタ（vim）を使用して実行する方が便利でした。 当時、私はすでにMacOSXとこのソフトウェアについて知識がありました。

プラットフォーム。 そのため、コマンドラインとAdobeの一連のグラフィックプログラムを使用して、お気に入りのUNIXを取得しようとするために、私の手はますますかゆみを起こしました。

そして今、 スティーブ・ジョブズ がIntelプラットフォームでのMacのリリースを発表した後、ハッキングされたバージョンに関する噂が広まりました

通常のPCで実行される MacOSX 。 もちろん、私はそばに立っていなかったし、「私の運を試してみる」ことを決めた:-)

ネットワーク上でハッキングされたMacOSXを簡単に見つけることができ、動作するハードウェアも簡単に見つけることができました。 文字通り

ニュースの数週間後、私はすでに自作のハッキントッシュの所有者でした！

ハッキントッシュに1年以上携わり、ソフトウェアとシステム自体を研究しました。 その後、「 Macで！ 」

残りはもちろん時間とお金の問題でした。

現在、 MacBookPro 15上のTextMateでこの投稿を書いています。

そして今、歌詞から練習まで。 Appleのプラットフォームでこれほど優れていてユニークなもの。

私はMacについて歌うことはありません。また、特にPCやM $の方向に唾を吐くことはありません。

すべてに独自の目的と場所があります。 したがって、まずタスクについて。

  1. Web開発者として、サーバーソフトウェア（例： php / python / ruby​​ / apache / mysql ）が必要です。

    テキストエディタと画像エディタ。 組版については、基本的にはWindowsしかありません

    しかし、サーバープログラミング（ポイントを必要とする人、コメントを書く人）で作業するには不便すぎます。

    この場合のLinuxはサーバープログラミングの問題をよく解決しますが、PSDまたはAIからタイプセットすることはほとんど不可能です。

    MacにはAdobeグラフィックパッケージがあり、サーバーソフトウェアを簡単にインストールできます。 それで判明した

    Web開発の普遍的な兵士はMacです。 プログラムのほとんどすべての兵器庫（通常はOpenSource ）と言えます

    Linuxの場合、バタンとMacで動作します。 MacPortsパッケージシステムを使用して簡単にインストールおよび更新できます。
  2. 仕事以外にも趣味や活動があります:-)。 最初は音楽です。

    ギターとキーを弾きます。 音楽を作成します。 したがって、録音に適したソフトウェアの存在

    音楽は私にとって必須です。 プロのオーディオデバイスのサポートも同様です。

    Windowsでは、これですべてです。 Macで動作するデバイスも少なくありません。また、Macには高品質のソフトウェアが付属しています。

    その大きなプラスは、無料のGarageBandの可用性です。これは十分です

    音楽素材の蓄積とアマチュア録音のため。 別の注意

    Appleの最新のLogic Studio / Expressの使いやすさの価値があります。
  3. 2番目の趣味は写真です。 ここでは、原則として、グラフィックスを操作するための製品

    WindowsとMacの場合、これらはほとんど同じなので、このアイテムの重量はゼロであると考えています。

    Linuxは、他のソフトウェアとはいえ、写真処理の問題も解決します。


したがって、Macは最初のポイントWeb開発でのみ勝つと結論付けることができます。

もう一つの重要なポイント。 多くの場合、これはおそらくMacプラットフォームの方がはるかに重要です。

専門家向け。 これはMacOSXオペレーティングシステムそのものであり、Apple製品の利便性（人間工学）に多くの注意を払っています。

詳細。 だから、どのコンピュータープログラマー（ああ、この言葉を「愛しているか」）が問題に遭遇したと思う、

数時間のアクティブな作業の後、彼は豊富なウィンドウとアプリケーションダイアログに沈み始めました。

私は長い間議論しませんが、単に私の例を挙げます。 昼食までに、私は通常すでにクリティカルマスを持っていました

これらの「ウィンドウとダイアログ」と私は積極的にすべてを閉じ/最小化し始め、それが時間の追加損失につながりました

そして、営業日の終わりにわずかな苛立ちに...多くのWindowsユーザーはこのプロセスに単に慣れています。

Linuxでは、この問題は非常に効果的に解決されます。 豊富なウィンドウマネージャーと優れた柔軟性

アプリケーションウィンドウを非常に効果的に操作できます。 最も効率的な方法は、 仮想デスクトップを作成することです。 ちなみに、このソリューションもWindows / Macに移行しました。

そして今、いくつかの「プゾメルキ」、もちろん主観的です。

  • MacとLinuxは、アプリケーションウィンドウの管理においてWindowsよりも優れています。
  • Linuxは、予算ソリューションにおいてMacおよびWindowsよりも優れています。
  • MacとLinuxは、システムの安定性においてWindowsよりも優れています。
  • MacおよびLinuxは、マルチプロセッサ（マルチコア）構成でのタスク処理においてWindowsよりも優れています。
  • Windowsは、豊富なソフトウェアと専門家により、MacおよびLinuxよりも優れています。


私の調査結果とヒント。

  1. 繰り返します。 コンピュータの世界には万能薬はありません 。 タスクとその解決策があります。
  2. Windowsはほとんどすべてのタスクに適しています。 しかし、それはおそらく快適さと美学の愛好家には適していません。
  3. LinuxはWebプログラミングに最適な環境です。 写真の処理に使用できます。
  4. Macは、ウェブ、オーディオ/ビデオ制作、デザイン、写真の分野の専門家にとって興味深いものです。


最後の点についてもう少し詳しく書きたいと思います。 次の分野を専門的に判断できます：ウェブ開発、

オーディオ制作と写真。 私はビデオ制作とデザインの経験が少ないですが、かなり客観的に判断できます。

そして、もう1つの分野は3Dを使用した作業であり、私には判断が難しいです。 だから、Macを試すことを強くお勧めします：

  • Web開発者
  • デザイナーへ。
  • ミュージシャン;
  • ビデオ編集のスペシャリストとスペシャル。 効果
  • 写真家。
