あなたのポケットにデンバー。

デンバーをフラッシュドライブに置きました。



このアイデアは、自宅に大型のコンピューターとラップトップがあるという事実から生まれました。



私は、フラッシュドライブを装着しようとすることにしました。 私はそれをコンピューターに挿入し、run.exeを起動して作業を行い、作業が完了し、別のコンピューターに挿入して作業を続けます。 この場合、すべての変更はフラッシュドライブで直ちに行われます。



何をしましたか、順番に：

1.デンバーのインストールを実行します。

2.ディレクトリ名で、フラッシュドライブ自体のフォルダのアドレスを指定します。 確認します。

3.作成された仮想ディスクの文字を選択します。 私はWを選びました、Zはすでに普通のデンバーで忙しいです。 確認します。

4.ダウンロードのタイプは「2」を選択します。 サーバーの起動時に仮想ディスクが作成されます。



インストールが完了しました。



ここで、プロジェクトをフラッシュドライブにコピーする必要があります。

1.プロジェクトは、フラッシュドライブ上の適切なフォルダーにプロジェクトファイルを保存します。

2.データベースは、「ルート\ usr \ローカル\ mysql4 \データ\データベース名」にあります-フラッシュドライブ上の同じフォルダ内。





実際、それがすべてです。



しかし、別のオプションがあります-デンバーをフラッシュドライブのルートに置きます。 その後、仮想ディスクは作成されません。これは私にとって便利なようです。 このため、

インストールでは、フラッシュドライブ上のフォルダではなく、フラッシュドライブドライブの名前のみを指定します。



Vistaでのデンバーの起動に小さな問題がありました。 「C：\ WINDOWS \ system32 \ drivers \ etc \ hosts」ファイルにホストのリストを追加できませんでした。 多すぎる

そこには多くのセキュリティがあります。 この問題は次のように解決されました-「LKM->管理者に代わって起動する」そしてそれは機能しました。



定期的にコンピューターに加えられた変更を破棄する必要があることは明らかです。
