Parallels Remote Application Server v.16.0：新機能と機能





誰もがクラウドにアクセスし、Parallelsも例外ではありません。 最後に、新しいリリースがあります。 カットの下には、焼きたてのParallels RAS 16.0のメインパンに関するストーリーがあります。 ストーリーは、生活を簡素化し、インフラストラクチャサポートを節約したいITおよびシステム管理者に役立ちます。



1月に、VCC（仮想クライアントコンピューティング）市場レポートのIDCに、世界の主要プレーヤーの一部としてParallels Remote Application Serve （RAS）製品が含まれていることをすでに書きました。 神自身が、高度な機能を提供することで、国際的な専門家からの期待を正当化するように命じられました。 それでは、何が新しいのでしょうか？



バージョン16.0



サーバーコンポーネント

Turbo.netとの統合。 これで、Turbo.netコンテナーのサンドボックスでアプリケーションを公開できます。 バージョン16.0では、パブリックリポジトリのみがサポートされていることに注意してください。







VDIのいくつかの改善。 VDIホストとして、Nutanix Acropolisハイパーバイザー（無料バージョンがあります）およびKVM（無料）がサポートされるようになりました。







リンククローンはHyper-V、VMware（ESX、vCenter）、Nutanix、KVMに登場しているため、仮想マシンをより迅速に作成できます。 テンプレートから展開された各デスクトップは、新しい名前とMACアドレスを受け取り、Active Directoryの新しいIPと新しいアカウントが割り当てられます。 名前を保存して、テンプレートからゲストマシンを再作成する機会がありました。







テストテンプレートウィザードも導入されており、仮想マシンテンプレートの操作性を確認できます。







RASprepの実装-SysPrepの拡張バージョン。







RAS PowerShell API v2.0。 VDI、HALB、およびライセンス関連のコマンドレットがサポートされています。

ホワイトラベルv1.0。 Parallels HTML5 Clientクライアントアプリケーションにブランドを付ける（カスタムテーマを作成する）ことができます。

安全な64ビットクライアントゲートウェイ。 理論上の同時制限は6,000以上に増加しました。

Publishing Agentに関連する最適化。 これで、好みのPAを選択できるだけでなく、その他の多くの改善点も選択できます。



コンソール



RASコンソールを使用する場合、UXに多くの改善を加えました。 それらの最も注目すべきもの：

個別のコンソールインストール。 これで、Windowsを実行している任意のマシンにコンソールをインストールできます。 また、PAにリモート接続する機能。 コンソールアプリケーションを実行するには、ローカルまたはドメインの管理者権限は必要ありません。







サイトとリモートデスクトップサーバーホストのインターフェイスを作り直しました。 サイトとRDSHの外観が再設計され、管理者の生産性が向上し、サーバーに関する情報をリアルタイムで受信できるようになりました。

セッション管理の改善。 これで、RDSホストまたはRDSHグループだけでなく、ユーザーまたはグループに対して公開されているものを確認できます。







公開アプリケーションの動作中のインスタンスを表示したり、現在使用しているリモートユーザーのリストを表示したりできます。







改善されたデバイスマネージャー。 USBデバイスのマウントを無効にしたり、管理対象デバイスをリモートでブロックしたりできます。







管理者のリストでADグループを使用する機能。 ユーザーを1つずつ追加する代わりに、カスタムADグループを追加できるようになりました。







エージェントステータスを保存するメカニズムが最適化されました 。 これで、起動中にエージェントのステータスが確認されたときに、コンソールがブロックされなくなりました。

特定のユーザーがサーバーデスクトップにアクセスできないようにすることができます。







TS、ゲートウェイ、リモートPC、およびVDIに特定のPublishing Agentを割り当てる機能。

ゲートウェイの改善された作業。 これで、Web要求をゲートウェイサーバーに直接リダイレクトできるだけでなく、HTTP転送を無効にできます。









お客さま



Windowsクライアント。 クライアント全体をインストールできない場合（管理者権限がない場合）、インストーラーはクライアントの基本バージョンに切り替えます。 ユーザーがクライアントをインストールする招待を受け入れたが、管理者ではない場合、基本バージョンのインストールがデフォルトで開始されます。



また、ネットワークを監視するためのツールがクライアントに表示され、ネットワーク接続の安定性に関する情報が表示されます。







Linuxクライアント。 USBデバイスのリダイレクト：ローカルマイクとカメラをサーバーで実行中のアプリケーションで使用できるようになりました。 PnPデバイス（MTP、PTP）のリダイレクトのサポートが追加されました。

Macクライアント。 ロギングが実装され、ログは〜/ Library / Containers / com.2X.Client.Mac / Data / ParallelsClient.logに書き込まれます。

HTML5およびChromeクライアントでのファイル転送サポート。 クライアントデバイスとサーバー間でファイルを転送する機能を追加しました。

HTML5クライアントでのネットワークレベル認証（NLA）サポート。 より柔軟なHTML5クライアント構成。

公開アプリケーションアイコンの上にカスタムアイコンをオーバーレイします。 これにより、ユーザーはローカルアプリケーションと公開アプリケーションを区別できます。







RAS接続タイムアウト（アプリケーションリスト）。 セキュリティシステムの改善：特定の非アクティブ期間の後、アプリケーションから強制的に終了します。 この関数はクライアントで機能します：

  • 窓
  • Linux
  • Mac
  • Android
  • iOS
  • HTML5
  • クロム






RAS接続ポリシーでクリップボードリダイレクト管理を構成します。 セキュリティの改善：ポリシー設定によりクリップボードのリダイレクトを無効にできます。







UX設定を自動的に検出します。 クライアント接続の特定のパラメーターに応じて、システム自体が最適なUX設定を選択します。 この機能はポリシーに追加されました。







iOSおよびAndroidクライアント



ダンボマウスコントロールモードのサポート （iOS / Android）。 以前は、この制御モードはレガシークライアントでサポートされていましたが、現在は新しいものに移植されています。

クイックキーパッド。 これで、モバイルデバイスのツールバーボタンへのキーボードショートカットを設定できます（RAS管理者が構成します）。







SwiftPoint （iOS） とのマウス統合 。 iOSデバイスでSwiftpointハードウェアマウスを使用できます。

PINまたは指紋を使用したクライアントアプリケーションのロック解除 （iOS）。 アプリケーションの起動手順にいくつかの改善が加えられました。 アプリケーションが既に実行されている場合、システムはそれに切り替えて、新しいウィンドウを作成しません。

テキスト入力エリアの定義が改善され、ポップアップキーボードの操作が改善され、テキストを選択するためのより便利な機能が改善されました。 iOS 9では、3Dジェスチャーを使用してテキストを強調表示できるようになりました。

RASおよびRDP接続をインポート/エクスポートする機能 。

Office 2013/2016アプリケーションでは、「タッチコントロールモード」が自動的に有効になります。

DPIスケーリングをサポートする新しい画面解像度設定。

RAS接続ポリシーのサポート。 管理者は、ポリシー設定を通じて既存のクライアント接続の設定を変更できます。







その他の機能



DynaPDFライブラリを使用してネイティブにPDFを印刷します。 クライアント環境の要件が少なくなり、依存関係が少なくなりました（Acrobatをインストールする必要はありません）。 PDFファイルの印刷の安定性が確保されます。すべてのドライバープロパティのリダイレクトは、プリンター設定に従って実行されます。







R-UDPのサポートが登場しました。 これにより、WAN接続を介してWindows以外のクライアントを使用する場合に、画面の更新を高速化することができました。
