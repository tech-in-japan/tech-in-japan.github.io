数日前にScott Guthrieは、Silverlight 2の最初のベータについて数行で話しました。



スコットの投稿を完全に翻訳するのではなく、（私の意見では）興味深い詳細についてだけ説明します。 まず第一に、それはもちろんクロスプラットフォームです。 すべてで。 開発者は、VB、C＃、JavaScript、IronPython、IronRubyなど、あらゆる.NET言語で記述できるようになりました...



次に、開発者はUI要素を（WPF UIフレームワークの一部として）埋め込み、リッチコントロール（TextBox、CheckBox、RadioButton、Slider、ScrollViewer、Calendar、DatePickerなど）を簡単に埋め込むことができました。



第三に、Silverlight 2は「リッチネットワーキング」をサポートします。REST、WS * / SOAP、POX、RSS、およびその他の標準HTTPサービスのサポートです。 すぐに、もう一度、さまざまなリソースへのクロスドメインアクセスについて説明します。



4番目に、これは、Rich Base Class Libraryであり、巨大な機能（コレクション、IO、ストリーム、グローバリゼーション、XMLなど）、および.NETコードとのHTML DOM / JavaScript統合にAPIを使用する機能が含まれています。



Silverlight 2では、コンピューターに.NET Frameworkをインストールする必要はありません。インストーラー（ところで、その重量は最初のバージョン-4.3MBよりもはるかに大きい）に必要なものがすべて揃っているためです。 また、Silverlight 2はIE、FireFox、およびSafariのブラウザーで動作します。 Opera-まだ名誉はありません。



さらに、Scottは、Silverlight 2の最初のベータバージョンのリリースで、Digg silverlightクローンの投稿を約束しました。 さて、あなたが見ることができるように、それは長くはありません。
