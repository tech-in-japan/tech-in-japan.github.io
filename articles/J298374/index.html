AppleはAmazon Web Servicesをオプトアウトする場合があります





ユーザーベースの成長に伴い、データセンターの負荷も増加します。近年、これはiCloudサービスのクラッシュの歴史にはっきりと見られます。 現在、Morgan Stanley Bankのアナリストは、自社のデータセンターを支持してAppleがAmazon Web Services（AWS）を使用することを拒否する可能性があると報告しました。 これは、既存の7つのデータセンターに加えて、今後2年間で3つの新しいデータセンターを開設することで促進されます。



その時までに、Apple自身のデータセンターの総面積は232,000平方メートルを超え、AWSサーバーが占有する合計622.4千平方メートルの約40％になります。



「これらのデータセンターの建設は、Appleが今後18-24ヶ月間、AWSサービスの使用を放棄するための信号であると考えています」とMorgan Stanley氏は述べています。



独自のデータセンターの使用への移行は、同社のクラウドサービスからの高レベルの利益のシグナルです-昨年iTunesおよびApp Storeからの利益は310億ドルに達し、2016年にはこの数値は増加するだけです。 ただし、AWSの放棄はAmazonに影響を与え、Amazonの収益は大幅に減少する可能性があります。 今年だけでも、AppleはAWSに約10億ドルを費やすことができます。これは2016年のクラウドサービスの売り上げの9％です。 過去1年間で、AWSはAmazonに対して78億ドルの利益を生み出し、2016年にはこの数字は約100億ドルになります。



予測では、モルガン・スタンレーのアナリストは、AppleがAWSデータセンターサーバーの使用を拒否した重要な理由を示していませんが、これは自社のサービスの安定性を高めたいという同社の欲求に影響された可能性があります。 さらに、Amazonはそれほど前ではなく、独自のアプリケーションストアとストリーミングサービスを積極的に開発し始めました。 AppleがAWSに投資することで、独自のデータセンターの構築と使用が正当化されます。 もちろん、再生可能エネルギー源のみを使用すると、会社のデータセンターの維持コストに影響を与える可能性があります。



一方、近年では、Netflix、General Electric、Capital OneなどのサービスがAWSに切り替わり 、独自のデータセンターを放棄しました。 しかし、アップルはサードパーティおよびプロプライエタリサービスに関して予期せぬ決定を下しました。
