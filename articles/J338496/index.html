VMware Workstation 14



VMware Workstation 14およびVMware Workstation Player 14のリリース。WindowsまたはLinuxの上にインストールされたクライアントハイパーバイザー。







新機能の中で：



  • 現在のゲストOSバージョンのサポート：Windows 10 Creator Update、Ubuntu 17.04、Fedora 26。
  • ゲストOSのセキュアブートサポート。
  • Windows 10での仮想化ベースのセキュリティ（VBS）のサポート。
  • VM用の仮想NVMeドライブのサポート。
  • ネットワーク経由でデータを送信するときに帯域幅を制限し、遅延とパケット損失をエミュレートできる高度なネットワーク設定。
  • VMware vCenter Server Appliance（VCSA）をワークステーション仮想マシンに展開する機能
  • 高度なESXiホスト管理
  • MAC OS XのクライアントハイパーバイザーであるVMware Fusion 10およびFusion 10 Proも発表されました。


終了したWindows用VMware vCenter Server



vSphereの次のバージョン（おそらくvSphere 7.0）は、WindowsプラットフォームでVMware vCenter Serverをサポートする最後のものになると発表されています。 VCSAとWindows向けvCenter Serverの機能はvSphere 6.0で同等であり、vSphere 6.5 VCSAでは組み込みのvCenter High Availabilityメカニズム、組み込みのバックアップ、vSphere Update Manager更新モジュールなどの追加機能が追加されたため、ソリューションは正しく、期待されています。



また、vSphereの次のバージョンは、vSphere Web Client（Flashベース）の最後であり、vSphere Client（HTML5）に置き換えられます。



ドロップアウトリストの最後はvmkLinuxです。これは、LinuxドライバーとESXiの互換性を提供するコンポーネントです。 vSphere 5.5から、開発者はいわゆるネイティブドライバーの提供を開始しました。これにより、Linuxドライバーに比べてパフォーマンスと安定性が向上します。 vmkLinuxを拒否すると、サポートされるサーバーと周辺機器のHCLリストが短くなり、Realtekネットワークアダプターを搭載したお気に入りのホワイトボックスサーバーでESXiを実行できなくなる可能性があります。



VMware Integrated OpenStack 4.0



VMware Integrated OpenStack 4.0の新しいリリース。







VIOは、VMwareが開発および保守するOpenStackディストリビューションです。 VIO 4.0はOpenStack Ocataリリースに基づいており、次の革新を提供します。



  • コンテナ展開サポート。
  • VMware vRealize Automationセルフサービスポータルとの統合。これにより、vRAインターフェイスから直接OpenStackのインストールを制御したり、OpenStackオブジェクトを含むvRAテンプレート（ブループリント）を作成したりできます。
  • VIOの下に複数のvCenter Serverを追加して、クラウドコンピューティングリソースを増やす機能。
  • 追加機能。たとえば、VMをオフにせずにプロセッサを増やしたりメモリを追加したり、サービスとしてのファイアウォール機能などがあります。


残念ながら、VIO 3.1はvSphere Enterprise Plusを購入したお客様が無料で入手できる最新バージョンでした。 VIO 4.0以降では、VIOによって管理される仮想化サーバーの各ソケットのライセンスとSnSサポートを購入する必要があります。



VMware AppDefense



VMware、新しい仮想環境セキュリティ製品AppDefenseを発表。



AppDefenseを使用すると、VM内のゲストOSで実行されているアプリケーション（プロセス）の動作を分析し、受信したデータに基づいて特定のベースライン（VMの通常の動作）を設定できます。 たとえば、悪意のあるソフトウェアの動作中に動作ラインから逸脱した場合、AppDefenseは次のいずれかのアクションをVMに自動的に適用できます。



  • VMware NSXを使用して、疑わしいプロセスのネットワークアクティビティをブロックまたは制限します。
  • VMをオフまたは一時停止します。
  • さらなる調査のためにスナップショットを作成し、セキュリティ管理者に通知します。


AppDefenseは、IBM Security、RSA、Carbon Black、SecureWorksなどのサードパーティの情報セキュリティ製品と統合でき、AppDefenseの機能を拡張してさまざまなアクションを検証および実行できます。



VMware vRealize Network Insight 3.5



仮想および物理ネットワークインフラストラクチャの問題を解決する製品であるvRNI 3.5の新しいバージョンがリリースされました。 vRNIは、仮想および物理スイッチ、ルーター、およびファイアウォールから構成情報を収集し、ネットワークの問題を簡単に検出できる便利で視覚的な形式で提供します。 たとえば、NSX論理スイッチ、ESXiサーバーアップリンク、Cisco Nexusスイッチポート、Juniperルーター、Check Point ITUを介して、VMネットワークインターフェイスから物理サーバーポートまでのパケットのパス全体をトレースできます。







vRNIの新しいバージョンには、ネットワークインフラストラクチャのPCI DSS規格への準拠の確認、IPFIXメカニズムを介したNSXからのデータ収集のサポート、ECMPが構成されたルーターを通過するVM間のトラフィックの視覚化、追加のサードパーティソースからのデータ収集、次の機能が追加されました：Check Point Firewall、 HP One View、Brocade MLX。



vSphere統合コンテナ



VMwareはVICをバージョン1.2に更新しました。これにより、コンテナーをVMware ESXiサーバーで直接実行できます。







VMwareのアプローチと他のコンテナーマネージャーの主な違いは、各コンテナーが個別のVMインスタンスで実行されることです。 したがって、管理者は、分離の向上、ホストからホストへのコンテナーの移動機能、VMware NSXを使用したコンテナーネットワーク接続の制御、vRealize Operations Managerを使用したコンテナーの監視などの仮想化プラットフォームの利点を使用して、VMと同じ方法でコンテナーを管理できます。 。 コンテナーを起動するオーバーヘッドを削減するために、Instant Cloneテクノロジーが使用されます。InstantCloneテクノロジーを使用すると、Transparent Pages SharingおよびLinked Clonesメカニズムを使用して作業中のVMのコピーをリアルタイムで作成し、RAMおよびディスク容量を節約できます。



革新の中で：



  • VMware Platform Service ControllerサーバーとのSSO統合を含む、ユーザー認証と承認のサポート。
  • VIC Registry管理インターフェースと管理ポータルの統合。
  • 再作成せずに、コンテナが実行されている仮想ホスト（仮想コンテナホスト）のパラメータ（プロセッサの数、RAM）を変更する機能。
  • Docker Engineの完全サポート、CLIでのcommit、diff、stats、cpコマンドのサポート。

NVIDIAブース



NVIDIAブースでは、MaxwellおよびPascal世代のアクセラレーターがデモされました。







8月中旬、同社はGRIDソリューションのバージョン4.0へのアップグレードを発表しました。 新しいGRIDを使用すると、物理サーバーのグラフィックアダプターのリソースを使用して、仮想マシン内のグラフィック処理またはリソース集中型コンピューティングを高速化できます。



グラフィック処理に加えて、アクセラレーターを使用して、H.264ビデオを再生したり、VDIシナリオでVMware Blastプロトコルのイメージをエンコードする際のCPUの負荷を減らすこともできます。これにより、単一の物理サーバー上のVMの密度が増加します。







GRIDの新しいバージョンでは、PascalアーキテクチャのアクセラレータであるP4、P6、P40、およびP100がサポートされるようになりました。



フルサイズのPCI-Eアダプターのフォームファクターで作られたTesla P40アクセラレーターは、前世代のMaxwellアクセラレーター（M60）と比較して、1 GBのビデオメモリを持つプロファイルを使用すると、ユーザー密度が50％増加します。 ブレードサーバーには、P6アクセラレータが適しています。



現在の世代と以前の世代のアクセラレータの主要なパラメーターの比較を表に示します。







Tesla P100アクセラレータは、グラフィックスを処理するためではなく、数学計算（CUDA、OpenCL）を加速するために設計されています。



しかし、蜂蜜の樽と共に、NVIDIAは軟膏に小さなハエを用意しました。 vSGA、vDGA、またはvGPUを機能させるには、使用するモードと負荷プロファイルに応じて、追加のGRIDライセンスが必要です。 サーバーが必要なライセンスの可用性をチェックしなかった場合、今後ライセンスがない場合、アクセラレーターは動作を拒否し、仮想GPUを使用するようにVMを切り替えます。

Veeamブース







今年、VeeamはWindowsおよびLinux向けのVeeam Agentバックアップのバージョン2.0をリリースしました。これにより、仮想マシンだけでなく、物理的なワークステーションとサーバーもバックアップおよび復元できます。 合計で、無料、ワークステーション、サーバーの3つのエディションが利用可能です。 それらの違いを表に示します。







ところで、このIBSの無料版はコンピューター上のデータを保護し、オペレーティングシステムとデータを古いディスクから新しいディスクに数回転送するのに役立ちました。 Habrの読者の中には、このIBSの多くのファンがいると確信しています。



Veeamはまた、その主力製品であるVeeam Backup＆Replication 10の新しいバージョンを発表しました。VBRは、VMware vSphereおよびMicrosoft Hyper-Vに基づいた仮想インフラストラクチャのバックアップと復元のための主要なソリューションです。



主なイノベーション：



  1. I / Oフィルタリングテクノロジー用のvSphere APIを使用したVeeam Continuous Data Protectionのサポート 。これにより、仮想マシンデータのバックアップ物理サーバーまたはストレージへの仮想同期レプリケーションが可能になります。 仮想環境用のほとんどすべての最新のデータ複製ツールは、非同期データ転送方式とVMwareスナップショット作成メカニズムを使用します。 このアプローチにはいくつかの欠点があります。 まず、5分以上に達する高RPO。これは、非常に重要なVMには受け入れられない場合があります。 次に、VMwareスナップショットを使用すると、VMのパフォーマンスが低下します。 CDPを使用すると、特別なCDPプロキシサーバーですべての記録操作をリアルタイムでブロードキャストできます。これらの操作は蓄積されてキャッシュされ、ターゲットサーバーに転送されます。 このアプローチにより、数秒から数時間の障害まで、任意の時間間隔でVMを復元できます。 VSSスナップショットを使用して、VM内のデータの一貫性を確保できます。







  2. Veeam Backup Agentsの完全サポート 。 VBRサーバーを使用すると、ワークステーションとサーバーにエージェントを集中的にインストールし、バックアップポリシーを構成し、データを復元できます。 Active Directoryオブジェクトに基づく動的グループがサポートされています。これにより、エージェントをインストールし、特定の組織単位にあるか、特定のグループに含まれるコンピューターをバックアップできます。
  3. ネットワークフォルダをバックアップする機能 。 多くのストレージインフラストラクチャが専用のNASデバイスを使用するため、ユーザーが待ち望んでいた機能。 VBRは、変更されたすべてのファイルの履歴と、削除されたファイルを一定期間保存できます。
  4. アーカイブリポジトリ 。 VBRは、古いバックアップをアーカイブストレージに自動的に移動またはコピーして、ディスクスペースを節約したり、追加のバックアップセキュリティを提供したりできます。 管理者はさまざまなアーカイブオプションを構成できます。たとえば、X日より古いバックアップを移動したり、メインストレージがXパーセントを超えたときにのみバックアップを移動したり、週単位のコピーのみを移動したりできます。 アーカイブストレージは、Swift、Amazon S3、Amazon Glacier、またはMicrosoft Azure Blobです。
  5. ロールモデルの拡張 。 VBRは、vSphereでユーザーに割り当てられた権限を表示し、ユーザーが権限を持っているバックアップおよびリカバリ用のVMのみを表示できます。
  6. 一貫性のあるデータベースバックアップOracleのRMANプラグインをサポートします。


VBR 10のリリースは、この冬に予定されています。
