Mac OS-ルートパスワードの変更

タンバリンでの長いダンスといくつかの黒魔術セッションの後、ルート（a）パスワード、したがってシステムのすべてのユーザーを変更するために、MacOS Xを「ハッキング」する方法を学びました。 はい、ブートディスクを使用してこれを行うことは可能ですが、すべてのブートディスクがすべてのアップルコンピューターに適しているわけではないため、遊び心のある小さな手が役割を果たしました！



したがって、すべてが非常に簡単です。

  1. シングルユーザーモードで再起動します-これを行うには、コンピューターの起動中にcmd-sを保持する必要があります
  2. だから私たちはルートユーザーとしてシステムに入りました
  3. 次に、コマンド/ sbin / mount -wu /を使用してハードドライブをマウントする必要があります。
  4. 次に、コマンド/ sbin / SystemStarterを使用してすべてのサービスを開始します
  5. 2つのオプションがあります。

    a）nidump passwdコマンドを使用します。 システムのすべてのユーザーのリストを取得し、従来のpasswdが関心のあるユーザーのパスワードを変更します

    b）（Tigerの場合）sh / etc / rcのいずれかを記述します-そして、システムは勝利の終わりまで起動します、そして、そこで-完全な行動の自由。




すぐに注意する必要があります

  1. キーチェーンからのパスワードは変更されません。
  2. いくつかのファイルがFileVaultによって閉じられた場合、rootはそれらを見ることはありません
  3. もちろん、イベントのさらなるシナリオはすぐにそれを示唆します：


システムにいることをホストに知らせたくない場合は、master.passwdファイルを慎重にホイッスルし、パスワードを取得するために既に自宅でそれを破るか、管理者権限を持つ追加の「透明な」ユーザーを作成します。

実際、これが私が書いているすべてです。シングルユーザーモードで再起動するときにコンピューターがパスワードを要求することを確認するにはどうすればよいですか。
