中小企業向けの仮想化システムのデータバックアップ

Hewlett Packard Enterpriseは、仮想化システムを保護したいSMB企業に、シンプルで安価なデータバックアップツールを提供します。

仮想化は、現在のITシステムを「配置」する非常に一般的な方法です。 ITランドスケープを仮想化する組織が解決しなければならない最も重要なタスクの1つは、障害後にバックアップとリカバリのシステムを作成してデータのセキュリティを確保することです。


この問題を解決するために、Hewlett Packard Enterpriseのポートフォリオには長い間、HPE Data Protectorが含まれていました。これは、使用されているITシステムの一部が物理的である場合、一部は仮想です。 情報は、物理インフラストラクチャ、仮想インフラストラクチャ、またはクラウドインフラストラクチャ内のコアからアクセスレベルまで、すべてのレベルで保護されます。 コピーは、データがどの仮想化プラットフォームから来て、ストレージに行くかに関係なく、標準的な手順になります。 同時に、HPE Data Protectorは包括的なデータ保護を提供するだけでなく、HPE Backup Navigatorを使用したリアルタイム分析、およびバックアップとリカバリをシンプル、信頼性、合理的かつ合理的にするHPE Storage Optimizerソリューションを使用したスト​​レージプロセスの最適化も提供します組織のIT環境に合わせた低コストのプロセス。



ただし、ITシステムの仮想化にますます関心があり、バックアップ製品も必要とする中小企業にとって、エンタープライズクラスのソリューションは複雑すぎて高価です。 SMB企業には不要な多くの機能があります。メンテナンスには、複雑なカスタマイズに精通した専門家が必要です。そのため、中小企業にとっては買収と運用の両方で非常に高価です。



SMBの顧客は、原則として、最も人気のあるハイパーバイザーを使用します。VMwareまたはMicrosoft製品のさまざまなバージョンであり、多くの場合、最新のものではありません。 それらは異機種環境をサポートしていません-仮想化がすでに適用されている場合、すべてが仮想化されます。 仮想マシンは比較的少なく、小規模なIT部門にはデータのバックアップと復元に専念する専任のスタッフはいません。

シンプル、便利、自動...



そのようなお客様のために、Hewlett Packard EnterpriseはHPE VM Explorerソフトウェア、またはHPE VMXと呼ばれるソフトウェアを提供しています。 この製品は、2月にスイスの企業Trileadを買収した後、企業のポートフォリオに登場しました。 3月末に、HPEは次のバージョンの製品を導入しました。VMExplorer 6.1-Hewlett Packard Enterpriseの壁の中で作成された最初のリリースです。



HPE VMXを使用すると、最も一般的な仮想化環境であるVMware vSphereおよびMicrosoft Hyper-Vでデータをバックアップできます。これらは通常、中小企業で使用されます。 このソリューションはリソースに対して要求が厳しくありません。MicrosoftWindows Server 2008 / 2008R2またはWindows Server 2012 / 2012R2を実行する8 GBのRAMを備えたサーバーで十分です。 ディストリビューションはわずか35 MBで、数分で半ダースのマウスクリックでインストールされます。 ほぼ同時に、バックアップが作成されます。



最大で約100台の仮想マシンを持つシステムでHPE VMXを使用することをお勧めします。 この制限は技術的なものではなく、より多くのマシンからバックアップを構成できますが、経験からわかるように、このような場合はより複雑な管理ソリューションが必要です。



このシステムは「配備、構成、および忘れられた」という原則に基づいて動作します。常に注意を払う必要はなく、直感的なWebインターフェースにより、すべての従業員が特別な知識なしで必要なすべての操作を実行できます ただし、「上級」ユーザーの場合、個々のスクリプトを作成する可能性があります（たとえば、自己作成アプリケーションを正しくバックアップする目的で）。



一般的なブラウザ（Internet Explorer、Firefox、Opera、Chrome、Safari）を使用するWebインターフェイスまたは「シック」クライアントを介してアプリケーションを制御できます。両方のアクセス方法によって提供される可能性はまったく同じです。



インストール後、VMXは自動的に仮想化環境に接続し、「環境を学習」し、事前定義されたスケジュールに従ってバックアップする準備が整います。



製品は、Microsoft Hyper-V、VMware ESX、ESXi（無料版を含む）の仮想マシン、および個々の仮想マシンディスクをコピーします。 興味深いことに、ハイパーバイザーの最新バージョンとともに、かなり古いものもサポートされています。たとえば、VMware-VI3までです。 これにより、VMXのお客様はソリューションを柔軟にパッケージ化できます。



データは、テープデバイス、SANストレージ、ローカルおよびネットワークストレージ、パブリッククラウド（Amazon S3、Rackspace）、およびOpenStackベースのプライベートクラウド（HPE Helionなど）に保存できます。



仮想マシン全体、個々のディスク、さらには個々のファイルを復元し、データを同じ場所または新しい場所に配置し、ファイルをランダムに選択したディレクトリに配置できます。 選択できるモードは2つあります。標準は、バックアップストレージから回復するものです。インスタント-仮想マシンがバックアップから直接起動し、オンザフライで本番ストレージに転送されるインスタントVMリカバリを使用します。



「新鮮な」バックアップは自動モードでテストできます。システムは保存された仮想マシンを展開し、管理者がエラーなしでマシンが実際に再作成されたことを確認できるプロセスのスクリーンショットを撮ります。 これにより、保存されたコピーが機能することを確認するために、仮想マシンの手動テストリカバリの必要がなくなります。



バックアップおよび復元機能に加えて、VMXには仮想マシン複製機能もあります。 コピープロセス中、完了後、マシンをリモートサーバーに複製する必要があることが示されており、メインサーバーでの誤動作または計画的なシャットダウンの場合、仮想マシンの現在のコピーはすぐにリモートサーバーで動作を開始します。



より迅速なシステムリカバリを確保するために、リモートサイトでローカルバックアップを作成し、それをメインストレージにコピーし、システムに変更を転送することで更新する場合、より複雑なオプションも可能です（すべての情報を再度転送してはいけません）。 必要に応じて、メインサイトからの仮想マシンのコピーが長期保存用に外部メディアに転送され、現在の「レプリカ」がリモートサーバーを保護します。



特殊な監視コンソールからだけでなく、通常のMicrosoft Event Viewerからもコピーシステムのイベントを追跡できます。これにより、特殊なソリューションを購入する必要がなくなります。

HPE VMXは中小企業向けに設計されていますが、大企業でも使用される場合があります-たとえば、一時的なプロジェクト、テスト環境などのために作成された、企業のメインネットワークから分離されたITシステムからのデータの安全性を確保するためです。 IT管​​理の場合、製品をデータバックアップの一般的な計画に含めるよりも、新しい構造に製品を「渡す」方が簡単です。



...そして安価



このソリューションには、Professional EditionとEnterprise Editionの2つのバージョンがあります。 それらの機能はサイドバーに表示されます。 価格については、非常に簡単です。スターターパックを購入し、必要に応じてプロセッサライセンスを購入できます。 このパッケージには、プロフェッショナルバージョンの場合は4プロセッサ、企業バージョンの場合は6プロセッサのライセンスが含まれています。 実行中のプロセッサのみが支払いの対象であり、サーバーの「空の」ソケットではないことに留意することが重要です。 つまり、たとえば、4つのプロセッサを搭載した4台のサーバー（「拡張用」）を購入し、それぞれに1つのプロセッサをインストールして、プロフェッショナルなパッケージを開始します。ニーズの拡大に応じて、プロセッサとライセンスを購入します



製品を試してみたい人のために、10日間続くテストバージョンを提供しています。 HPE VMXが問題の解決に適している場合は、キーを購入するだけで製品は完全に動作します。 何も再インストールする必要はありません。 ProfessionalバージョンからEnterpriseバージョンへの切り替えも難しくありません。アップグレードライセンスを購入する必要があります。これにより、製品のコーポレートエディションの追加機能が提供されます。



詳細を知りたい場合は、製品速報を使用するか、hpsw.co / hpevmxで入手可能な試用版をダウンロードして、 当社のWebサイトの情報を理解してください。 HPEのロシアオフィスの従業員に電子メールで質問することができます。
