OpenHABとArduinoを友達にする方法

私たちは皆、生活をより良くし、技術革新を家に加え、それを自動化し、より安全にしたいと考えています。 多くの場合、これらの欲求は熊手運動に変わります。 プロプライエタリなシステムは高価であり、相互に互換性がない場合が多く、すべての専門家が独自のソリューションを作成できるとは限りません。 何をすべきか、アマチュアにとってアクセスしやすいだけでなく、専門家にとっても興味深いシンプルなソリューションを見つける方法は？



Arduinoはアマチュアから尊敬されているシンプルな電子デザイナーであり、彼らはひどい実験でひそかにそれを使用していますが、専門家はそれを嫌っていると言います。 その人気の秘密は何ですか、それをどのように使用して問題を解決しますか？



Arduinoは家のローカル監視と制御に適しています。これにはネットワーク上に多くのプロジェクトがあります-グローバルに考える必要があるため、優れていますが十分ではありません！ ネットワーク接続とモバイルインターフェイスが必要です！



さて、 この記事のおかげで、OpenHABに精通しています。OpenHABは、さまざまなプロトコルを持つ多数のデバイスを単一のネットワークに結合するオープンソースプラットフォームです。 OpenHABはJavaで実装されているため、さまざまなOSで動作し、シングルボードコンピューター上でもルーター上でも実行でき、モバイルインターフェースとWebインターフェースを備えています。 すでに熊手を退屈している、私たちのものに対する呪文の良いセットのように聞こえますか？



まず、 OpenHABをインストールし 、ダウンロードページを開き、ランタイムコアとデモのセットアップをダウンロードし、C：\ openhabで展開します。 デモのセットアップを解凍するとき、README.txtファイルの置き換えを許可します。 Javaがコンピューターで使用できない場合（java –versionコマンドで確認）、 指示に従ってインストールします 。



C：\ openhab \ start.batを実行してOpenHABを実行し、少し待ってWebインターフェースを開きます -localhost：8080 / openhab.app？Sitemap = demo

（OpenHABを停止する必要がある場合-Ctl + Cを押すか、プログラムコンソールを閉じます）







動作します！ 続けましょう。 私たちの目標は簡単な解決策を見つけることですので、実験から始めましょう。 この記事に関するコメントの1つは、Serial-Bindingを使用してシリアルポート経由でコントローラーをOpenHubに接続することを提案しました。 非常に興味深い解決策、それを実装してみてください。



アドオンをインストールするには、ダウンロードページを開き、アドオンをダウンロードし、org.openhab.binding.serial-1.6.1.jar（プラグインのバージョンは異なる場合があります）をC：\ openhab \ addonsフォルダーに解凍します。 必要に応じて、 ドキュメントをお読みください。



通常、実際のデバイスとの対話のデバッグには多くの問題があるため、最初に端末からコマンドを送信し、ヌルモデムケーブルエミュレーターを介してOpenHABに接続してみましょう。



com0comエミュレータをダウンロードしてインストールし、ユーティリティを実行します プログラムメニューからセットアップすると、インストール中にシリアルポートの仮想ペアが既に作成されていることがわかります。COM35+ COM36があります。 それらを作業に使用します。







端末としてHercules SETUPユーティリティを使用しました。これは、多数のポートを選択し、適切なボタンを押して送信する複数のコマンドを入力できる無料のプログラムです。 ダウンロードして、hercules_3-2-8.exeを実行し、「シリアル」タブに移動して、仮想ペアの2番目のポート（COM36があります）を選択し、速度9600、8ビット、1ストップビット、パリティなしを入力します。 ポートを開きます。







次に、openHABを構成する必要があります。そのために、デザイナーを使用します。 ダウンロードページを開き、 openHAB DesignerをダウンロードしてC：\ openhab \ designerフォルダーに解凍し、C：\ openhab \ designer \ openHAB-Designer.exeを実行して、構成が配置されているフォルダー（C：\ openhab \ configurations）を選択します。







まず、要素（アイテム）を作成し、仮想ペアの最初のシリアルポートに関連付けます（COM35があります）。 これを行うには、demo.itemsファイルの最後に次のコードを追加し、エディターウィンドウで右クリックしてコマンド「保存」を使用します。代わりに、単にctl + Sを押します。 



String Serial_string " [%s]" { serial="COM35" }






テストのために、作成された要素に毎分コマンドを送信するルールを作成します。 これを行うには、demo.rulesファイルの最後に次のコードを追加して保存します。 



 rule "Test serial string" when System started or Time cron "0 * * * * ?" then Serial_string.sendCommand("Test") end






OpenHABが実行されている場合、ターミナルプログラムでコマンドテキストの受け入れをほぼすぐに開始し、コンソールにその通過に関するレポートが表示されます。







次に、OpenHABにコマンドを送信してみましょう。 ターミナルでは、「送信」パネルの最初のフィールドにテキスト「ON_1」を入力し、2番目のフィールドに「OFF_1」を入力します。 最初の「送信」ボタンを押してから、2番目を押します。 この結果、コンソールに次のメッセージが表示されます。







作業が無駄にならないように、コマンドを処理するルールを作成します。これを行うには、demo.rulesファイルの最後に次のコードを追加して保存します。 



 rule "Serial control" when Item Serial_string received update then if(Serial_string.state=="ON_1") sendCommand(Light_FF_Corridor_Ceiling, ON) if(Serial_string.state=="OFF_1") sendCommand(Light_FF_Corridor_Ceiling, OFF) if(Serial_string.state=="ON_2") sendCommand(Light_GF_Kitchen_Ceiling, ON) if(Serial_string.state=="OFF_2") sendCommand(Light_GF_Kitchen_Ceiling, OFF) end


WEBインターフェースを開き、アイテム「First Floor」をクリックしてからアイテム「Corridor」をクリックするか、リンクをブラウザーにコピーします： localhost ：8080 / openhab.app？Sitemap = demo＃_FF_Corridor、テキスト「ON_1」、次にOFF_1を端末から送信します。 WEBアプリケーションでスイッチを制御し、その状態を変更する必要があります。







「ホーム」をクリックし、「グラウンドフロア」を選択してから「キッチン」を選択するか、ブラウザーでリンクをコピーします： localhost ：8080 / openhab.app？Sitemap = demo＃_GF_Kitchen、テキスト「ON_2」、次に「OFF_2」を端末から送信します。 WEBアプリケーションでスイッチを制御し、その状態も変更する必要があります。







実験の結果は、アプリケーションコンソールで確認できます。アプリケーションコンソールには、次のメッセージが表示されます。







すべてがコンピューターで動作します。現実の世界に接続する時です！



ほとんどのArduinoボードには、COMポートからUSBへのアダプターが既に搭載されているため、コンピューターやラップトップへの接続に問題はありません。 手元のツールからレイアウトを組み立てます。そのためには、Arduinoコントローラー、ブレッドボード、ワイヤー、ボタン、LED、10kおよび1kオームの抵抗器が必要です。







ネットワークには十分な説明と例がありますので、Arduinoの開発環境でインストールプロセスと作業を詳細に説明する必要はありません。 ボタンのクリックを処理し、シリアルポート経由でテキストコマンドを送信する簡単なスケッチの作成にすぐに進みます。 コマンドの誤った送信を避けるために、スケッチへの接触バウンスに対する単純な保護を追加します。



エディターを開き、次のコードをエディターに追加して、btn.inoファイルにスケッチを保存します。 



 //   const int buttonPin = 2; //  ,    const int ledPin = 13; //    //  int buttonState = 3; //      int buttonCnt = 0; //       void setup() { //  ,   ,   pinMode(ledPin, OUTPUT); //  ,   ,   pinMode(buttonPin, INPUT); //    Serial.begin(9600); } void loop(){ //    delay(1); if (buttonState != digitalRead(buttonPin)) buttonCnt++; else buttonCnt = 0; if (buttonCnt > 100) { //      buttonState = digitalRead(buttonPin); buttonCnt = 0; //     //  ,  buttonState  LOW: if (buttonState == LOW) { //   digitalWrite(ledPin, HIGH); //   Serial.print("ON_1"); } else { //   digitalWrite(ledPin, LOW); //   Serial.print("OFF_1"); } } }






スケッチをコンパイルしてArduinoにロードし、ポートモニターを開き、9600の速度を選択して、ボード上のボタンを押してみます。 その結果、コマンドの受信を確認できます。







ポートモニターを閉じて、コントローラーとのOpenHAB接続を構成します。このため、要素の設定を編集します（この場合、Arduino USBアダプターがこのポートにインストールされているため、値をCOM18に変更する必要があります）。 



 String Serial_string " [%s]" { serial="COM18" }


最後に、コンソールとWEBインターフェイスを使用してコマンドの通過を制御します。

2人は、ソフトウェアツールを使用して交換をテストすることを学びました。 これらのツールは、将来のトラブルシューティングや独自の実験の設定に役立ちます。

結論：



テキスト形式でコマンドを送信することはArduinoの慣習では一般的です。そのようなコマンドの処理はOpenHABで実装できるため、この対話方法は注意と議論に値します。



このソリューションの欠点は、コマンドが一時停止以外のもので区切られていないことです。これにより、コマンドが「固着」し、信頼性の低い処理につながる可能性があります。 この問題を解決するには、コマンドを処理するためのスケッチとルールを変更するか、シリアルバインディングモジュールを変更する必要があります。
