コピー、貼り付け



Windows Phoneの操作のさまざまな側面に精通している人は、全体的な概念がAppleからコピー（借用）されたことに同意するでしょう。 特に、Windows Phoneの設計と開発の時点で、アップルはモバイルの世界のリーダー企業であったため、これには何の問題もありません。 当然、Microsoftは何かを変更し、何かを追加し、何かを単純化しましたが、一般的にはすべてがAppleからコピーされました（ただし、フラットデザインの流行を設定したのはMicrosoftであることが追加される必要があります）。 Microsoftは、Appleが強く批判されていることをコピーしました。まず第一に、閉鎖されていることと、デバイスを自分用にカスタマイズする機能がないことです。



Windows Phoneの初期バージョンには、多くの「幼稚な問題」がありましたが、それは時間の経過とともに修正されましたが、それでも変わりません。 その後、ユーザーはより深刻な問題に直面しました。バックグラウンドアプリケーションは手に負えない状態で動作しました（Microsoftが買収したSkypeメッセンジャーがこの影響を最も受けていたことを象徴しています）。 これまで、バックグラウンドでの作業を必要とするサードパーティアプリケーションは、100％の安定性を誇ることはできません。 Microsoftがリリースした更新は、多くの点で単なる外観上のものであり、更新は数か月間続きました。7.5から8.0にアップグレードできないことは、多くのユーザーが心から取りすぎました。



ここでできる結論：コピーできますが、成功するためには、コピーは少なくとも元のものより悪くないはずです。
