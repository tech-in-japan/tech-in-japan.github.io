Hotjar Serviceが変換を増やす方法（サービスの概要と3つのケース）

Hotjarは、ユーザーの行動を分析し、調査を実施し、ファンネルを構築するなど、高度なインターネットマーケティング担当者向けのサービスです。 このサービスの主な目的は、サイトでのコンバージョンを増やすことです。 そして彼はそれに完全に対処します。 顧客の数は飛躍的に増加しています。







私自身はこのサービスを長い間使用しており、アナログの中で最高だと思います。 彼がYandex.Metricaの機能で失うのは、価格だけです。 制限が必要ない場合は、料金を支払う必要があります。 私の経験が示唆するように、これはRunetの多くのインターネットマーケティング担当者を止めます。 しかし、その後、誰もが彼がそれを必要とするかどうかを自分で決めるでしょう。

1.ヒートマップ



Yandex.Metricaにもこの機能があります。







ヒートマップは3つのタイプに分けられます。

  • 地図をクリック
  • スクロールマップ
  • リンクマップ


ヒートマップが提供するもの：



ユーザーがクリックするページ要素、最も頻繁にクリックするリンク、ページをスクロールする深さを確認します（この機能については記事の後半で説明します）。 これに基づいて、Webサイトの訪問者がどのようにインターフェイスと対話するかを理解できます。



使用例：



サイトのクリックマップを分析し、特定の要素でクリックの大部分がクリックされていることを確認します。 この場合、要素はクリック不可です。 ただし、訪問者は常にクリックするため、クリックすると特定のページに移動します。



このデータに基づいて、クリック可能な要素が正確にわかります。



したがって、訪問者が最も頻繁にクリックする場所を理解できます。 おそらく、これらは訪問者を購入プロセスから遠ざけるボタンになるでしょう。 これは、これらのボタンをインターフェースから完全に削除するか、訪問者が「パス」から購入しないように作成する必要があることを意味します。



スクロールマップを検討する場合、長いランディングページ、長いページ、およびコンテンツマーケティング担当者にとって非常に便利です。 見込み客が長い記事を読んでいるかどうかにかかわらず、潜在的な顧客がランディングページをめくっている場合、3秒でスクロールする地図が表示されます。



読むことをお勧めします-A / Bテストのアイデアを見つける方法：ヒートマップと投票



2.セッションレコード（訪問者の記録）



Yandex.Metricaにもこの機能があります。







セッションの記録は何を提供しますか：



各訪問者がサイトのインターフェースとどのように対話するかを確認できます。 すべてのクリック、カーソルの動き、ページの各部分でユーザーが費やす時間、特定のブラウザーまたはデバイスで発生する問題が表示されます。



さらに、レコードをフィルタリングしてタグを付け、お気に入りに追加して、分析をより便利にする機会があります。



使用例：



セッションを表示した後、Internet Explorerからアクセスする訪問者には、注文するためのボタンがまったく表示されないことに気付きました。 したがって、彼らは原則として、たとえ彼らが本当に望んでいるとしても、顧客に転換することはできません。



この情報に基づいて、開発者にタスクをすばやく送信し、レイアウトを修正し、Internet Explorerを使用してサイトを訪れたすべての訪問者が問題なく購入します。 誰もが幸せです。 上り坂のビジネス！



ちなみに、レコードを表示するだけでなく、そのような妨害を見つけることができます。 Googleアナリティクスには、コンバージョンのすべての「キラー」を見つけるのに役立つ非常に優れたレポートがあります。 記事全体を彼らに捧げ、いくつかのサイトで40,000以上のビューを獲得しました 。



セッションを表示するための機能もYandex.Metricaにあり、Webvisorと呼ばれます。 Hotjarにお金をかけたくない場合は、問題なく使用できます。 ところで、Webブラウザを使用する場合は、セグメンテーションを使用して、全員を連続して分析しないようにすることを強くお勧めします。

3.セールスファネル



Yandex.Metricaにもこの機能があります。







セールスファネルを提供するもの：



最終的な目標（購入など）につながる一連のアクションを作成すると、最大の問題がどの段階にあるかがわかります。



したがって、ほとんどのユーザーが脱落する場合、コンバージョンを増やすためにあらゆる努力を注ぐ必要があります。



私の意見では、 Hotjarのセールスファンネルの唯一のマイナス点は、イベント（イベント）ではなく、ページへの遷移に基づいて構築されていることです。 同じ状況はGoogle AnalyticsとYandex.Metricaにもあります。 しかし、最後の2つでは、イベントのファンネルを作成する機会がまだありますが、初心者がこの問題を解決するのは簡単ではありません。



または、Mixpanelを使用できます。 このサービスでは、一般的に、すべての分析はユーザーに割り当てられたイベントとプロパティに基づいて構築されます。 ここでは、販売ファネルを構築するのが非常に便利で簡単です。



一般に、ビジネスにまだ目標到達プロセスがない場合は、それに基づいて最初のコンタクトから購入までのクライアントの経路を測定しますが、これは大きな省略です。 今すぐこの問題を解決することをお勧めします。

4.フォーム分析



Yandex.Metricaにもこの機能があります。







フォーム分析が提供するもの：



各サイトにはフォームがあります。 それらなしで行うことは不可能です。 フォームにフィールドが1つしかない場合は便利です。 しかし、ほとんどの形式にはさらに多くの形式があります。 したがって、フォームの分析により、個々のフィールドに人がどのように入力しているかを理解できます。



まず、訪問者が平均して各フィールドを埋めるのに費やす時間を学習します。 次に、最も多くの人が脱落するフィールドを確認します。



セッション分析をフォーム分析に追加すると、訪問者がフィールドに入力する方法を確認できます。 おそらく彼らは何かを理解せず、必要なものとは完全に異なるものを紹介します。 その結果、彼らはフォームに記入するエラーを取得し、競合他社に行きます。



使用例：



フォームの分析を通じて、「電話番号」フィールドで、ほとんどの人が脱落していることに気付きました。 2つの理由があります。 まず、彼らは電話番号を残したくない。 2番目は、フィールド自体の問題です。



OK、セッションノートを見てください。 おそらく問題は、フィールドに電話番号マスクがなく、訪問者の大部分が国/都市コードを入力する必要があることを単に理解していないことですか？



これが問題である場合、それは非常に簡単に解決されます。 このタスクは開発者向けに設定されており、人々のフィールドで通常の理解しやすいものにします。



読むことをお勧めします- スタートアップPlotguruは着陸のコンバージョンを9％から52％に増やしました。 あなたは弱いですか？



5.組み込みの投票







組み込みの投票は何を提供しますか：



ターゲットオーディエンスの理解に役立ちます。 ターゲットオーディエンスの要望やニーズを知るには、彼女に尋ねる以外に方法はありません。 調査は、高度なインターネットマーケティング担当者にとって重要なツールの1つです。



Hotjarの組み込みのアンケートは、特定の条件に従ってターゲットにできます。 たとえば、特定のページに表示されるのは、2分間表示された後のみです。



使用例：



製品の支払いページでは、ユーザーが次の段階に進むのに平均2分かかることがわかっています。 彼がそれ以上に使うなら、あなたは彼に彼が購入をするのを妨げるものについての質問で彼に投票を示すことができます。 この質問のおかげで、潜在的な顧客が収入を得ることを妨げる多くの理由を見つけることができます。



これらの投票は、Webバージョンのサイトとモバイルの両方で機能することに注意してください。 ところで、モバイルユーザーのみにアンケートを実施してみてください。スマートフォンでサイトを使用するのが便利かどうかをユーザーから確認してください。 これは、データ駆動型マーケティングの最初のステップになります。

Unbounceは、最も人気のあるランディングページサービスです。 コンバージョン、A / Bテスト、UX / UIの向上というトピックを非常に積極的に推進しています。これはすべて、ランディングページの作成者である顧客にとって重要だからです。

販売目標到達プロセスの重要なページの1つは、支払いデータの入力です。 最初は、請求情報（国、都市、住所、郵便番号）を手動で入力する必要がありました。 マイケルは、一部のユーザーがここに落ちていると考え、アドレスと自動入力の選択を提案しました（多くの大規模サイトで行われたように）。







アンバウンス開発者はこの問題をすばやく解決し、各ユーザーが自分のアドレスを選択できるようになりました。その後、すべてのフィールドに自動的に入力されました。 連中はいくつかのユーザーテストを行い、誰もが新しい機能を気に入って、記入する時間が減りました。 すべてが一見すごい。
