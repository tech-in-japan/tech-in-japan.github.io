DPM：なぜ彼はそうなのですか？

今日は、プロのコミュニティでかなり複雑な意見を引き起こす製品、Microsoft Data Protection Managerについてお話します。 カットの下には、歴史に飛び込む一連の記事の最初の部分があります。







記事の著者であるkhabarovdaniil （Daniil Khabarov）にフロアを渡します。



この製品についての私の意見は非常に明確です-非常に興味深い技術的実装のために私はそれが好きですが、いくつかのアーキテクチャソリューションの曖昧さが常に私を驚かせています。 しかし、まず最初に。



私たちの物語の計画は次のとおりです。

最初の部分では、 Microsoft Data Protection Managerの歴史的背景と、そのような理由について説明します。

第二に -恐ろしい獣VSSを見て、彼が技術的にどのように働くかについて。

さて、そして3番目 -Microsoft Azure Backupを含め、現在の状況、彼がそれをどのように、そしてどのように使用できるかを知っていることです。


Microsoft Data Protection Manager。 開始する



ご存知のとおり、Windows Server 2003の期間中、すべての管理者がWindowsバックアップを使用し、誰もが満足していました。 より多くのものが必要でしたが、ほとんどの場合、NetBackup、BackupExecなどのサードパーティツールによってのみカバーされていたため、さまざまなアーカイバを使用した優れたロボコピースクリプトでした。







それが便利だったというわけではありませんが、当時はギガバイトのディスクスペースがはるかに高価であり、ギガバイトサイズのバックアップが尊重され、存在していた10 MBitネットワークでは、適切な時間内に必要な量の情報を転送できませんでした。 テープライブラリとテープドライブは、古いLTO-3カセットにアクティブに情報を記録しました。



これが、将来のDPMの開発を決定したものです。



  • 彼はテープに書き込むことができたはずです。
  • 既存のWindowsバックアップを改善する
  • 大容量かつ高速のディスクストレージに対するニーズの高まりに導かれ、将来を見据えます。
  • Windowsインフラストラクチャの完全バックアップを提供します。 これには、まず、AD、SQL、ファイルリソースが含まれます。


したがって、2005年9月27日に、MicrosoftはDPM 2006を初めて導入しました。この製品に注意を払う価値はありません。使用することを決めた人はほとんどいません。 System Center 2007リリース。



DPM2007。アスペラ広告アストラごと



しばらくの間、DPMはADでサポートされる唯一のバックアップ製品になったと言っても過言ではありません。 ドメインコントローラー。 もちろん、Windowsバックアップに代わるWindows Serverバックアップを除きます。







そのため、新製品は比類のない幅広いバックアップ機能を提供しました。



  • ファイルサーバーとワークステーション。
  • Microsoft Exchange
  • SQL Serverデータベースソフトウェア。
  • Windows SharePointサービス
  • 仮想サーバー（SP1の登場-Hyper-Vも ）;
  • エンドユーザーの回復。


最初のバージョン、さらにそのSP1と比較しても、大きな進歩でした。 これで、組織全体のネットワークを介して本当にバックアップを実行できると言うことができました。 同時に、エンドユーザーファイルをバックアップする機能（エンドユーザーリカバリ）が拡張され、また、提供された機能が明らかに平均的な組織のニーズに対応し始めています。 これらすべてが、DPMの人道的な価格設定ポリシーと相まって、2014年にすでにこれらの製品ラインの作成者がこの製品から現在のバージョンに移行するようにしました。



このバージョンの製品は、明らかにほとんどの小児疾患を取り除きましたが、それでも仲間の労働者と深刻な競争をすることはできませんでした。 同時に、弱いチャネル（128 kbpsを介して、無限よりわずかに小さいpingサイズでも実際にコピーできる）で動作する能力は、リモートサイトでも使用するための明確に良い機会を与えました。



Hyper-Vの登場により、DPM 2010 SP1は仮想マシンのバックアップ方法も学びました。 これは、Virtual PCと比較して明確な一歩でした。



DPMを使用すると、地理的に分散したサーバーを使用して階層的なバックアップ構造を構築することがすでに可能であり、それは当時非常に重要でした。



管理者の観点から見た大きなマイナスは、ADオブジェクトの部分的な復元が不可能であることでした。これは、バックアップの整合性に関するMicrosoftのポリシーのため、非常に論理的です。 また、機能が要求されていたという事実にもかかわらず、ADチームはセキュリティ原則に基づいて個々のオブジェクトの増分リカバリを許可しませんでした。 この状況は今日まで続いています。



DPM 2010



Data Protection Manager 2010は、明らかに仲間の間で長寿命です。 一般に、組織内のバックアップ製品は、変更や更新を行わなくても十分に長生きします。







主な理由は、一方ではバックアップを失う恐れがあり、他方では災害復旧計画（一般にDRPとして知られている）などの規制文書の変更の可能性があるため、そのような製品を更新する恐れがあることです。







なぜ彼は組織やシステム管理者を買収したのですか？ 答えは簡単です。彼は既存の開発を本当に改善し、いくつかの革新を追加しました。



  • 100台のサーバー。
  • 3000台のWindowsクライアント。
  • 2000 SQL Serverデータベース
  • 100万のオブジェクトを含む25 TBのSharePointファーム。
  • 40 TBのExchangeストレージグループとデータベース。


格納されたデータの量が増加したことに加えて、Exchange 2010およびDAGのサポートなど、非常に便利な機能が登場しました。 おそらく、多くの人が興味深いExchange 2007 LCR、CCR、SCRフォールトトレランスシステムを覚えています（特に悪夢の場合）。 そのため、DPM 2010はExchange 2007とExchange 2010の両方をサポートしていました。



もちろん、その時点で新しいOSであるWindows Server 2008 R2のサポートがありました 。



この製品は本当に長い肝臓であることが判明しました。 これには多くの理由がありました-調整されたシステムの安定性、かなり良い機能。



DPM 2012およびDPM 2012 R2



さて、今日お話しする最新バージョンはDPM 2012 R2です。







前任者と比較した主な革新は次のとおりです。



  • 集中管理;
  • Generic Data Sourceを使用する機能（私のお気に入り）
  • 証明書による顧客保護。
  • 重複排除。


また改善されました：



  • 個々のサイトとSharePointアイテムを復元する
  • SQL FileStreamサポート。
  • 高度なテープコロケーション機能。


しかし、おそらく最大の革新は、DPMとAzureバックアップの友好関係の始まりです。 SP1およびDPM 2012 R2以降、サブスクリプションによってクラウドサービスにバックアップを送信できるようになりました。 バックアップの観点から見ると、これは真の突破口であり、製品の歴史全体における最大の革新の1つでした。
