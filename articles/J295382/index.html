5つのデザイナーのヒント、または適切な雇用主を見つけて適切に滞在する方法

自己紹介させて



みなさん、こんにちは、私の名前はニックです。私は28歳で、モスクワに住んで働いています。 私はブラッドベリーラボのインターフェイスデザイナーです。



やりたいことをすぐに理解できなかったので、宅配便からシステム管理者まで、さまざまな職業を試しました。 システム管理への熱意から、デザイン、つまりデザインとインターフェイスデザインに興味を持つようになりました。



私は高等教育を受けていませんでしたし、専門のコースにも行きませんでした。 私はちょうどインターネットをさまようと私に興味深いものを勉強しました。 もちろん、すべてがすぐに機能したわけではありません。 長い学習プロセスについてはお話ししません-これはあまり面白くないです。 しかし、仕事の検索と雇用者は別の問題です。



最初のステップ



当時働いていた会社でシステム管理スキルを使い果たしたため、2つの開発パスしかありませんでした。デザイン分野で自分に興味のある仕事を探すか、管理を続けるかです。 2番目のオプションは、私にとって都合の良い条件で高給の仕事をすばやく見つけることができたため、最も便利でしたが、これは私のやり方ではないことに気付きました。 したがって、一定の時間を費やして、6つのプロジェクトで構成される小さなポートフォリオを構築しました。 それらのうちの1つだけが本物であり、残りはすべて私の想像の産物でした。 その後、履歴書を作成し、クリックし始めました。 興味のある空席をクリックして返信します。 このリストには、大企業とスタートアップの両方が含まれていました。 デザインの分野での経験がなく、大企業に参入するのは非現実的なことだと理解していましたが、とにかくやってみました。 すぐに私の考えが確認されました。私はまだ大企業に成長していません。 インタビューの最終段階に達する小さな会社では、私は1つの理由だけで失いました-実際のプロジェクトの欠如です。 これは、ポートフォリオ内の特定のプロジェクトに関する質問に完全に答えることができないことが多いという事実によってさらに悪化しました。



私の症例は完了しておらず、この分野での経験がなく、自分の仕事に関する質問に答えることができませんでした。 もっと悪いことは想像できません。 そう？ しかし、これはデザイナーとしての私の開発の1つのステップにすぎません。 これらの欠点を考慮して、私は次のことに気付きました。



ヒント1.各プロジェクトを分析します。





プロジェクトを雇用主に提出する前に、ケースのあらゆる詳細を検討し、必要に応じてその履歴を書き留める必要があります。 楽しいだけではありません。 あなたのケースはただのWOW画像ではなく、特定の問題の解決策であることを覚えておくことが重要です。 雇用主にそれについて教えてください。



例として、個人的な経験から2つの例を挙げます。



Alpha Click Webサイトの再設計。



私自身はこのサービスのユーザーです。 新しいインターフェースを試すように申し出られたとき、私は喜んでこのビジネスを始めました。新しい、面白い、使いやすい何かを期待していました。 とはいえ、まもなく私の前向きな姿勢は無に帰しました。 新しいサービスは私の期待に応えられませんでした。 それは不快であり、それを使用したいという欲求を引き起こしませんでした。 古いデザインの痕跡がどこかで、どこかに影とボリュームがありました。 一般的に、全世界が拒否しようとしていたすべてのもの。







新しいサービスを詳細に検討し、そのすべての欠点を考慮して、私は独自のコンセプトを作成することにしました。 これにはナビゲーションを大幅に簡素化する垂直サイドメニューが含まれており、銀行（ダッシュボード）のコーポレートカラーの一種のコントロールパネルでした。 垂直サイドメニューは、現代のモニターのアスペクト比を考慮してユーザーのワークスペースを増やし、水平モニターの場合のようにそれを食べませんでした。 完全なプロジェクトはBehanceで入手できます。



衣料品ブランドGeek's Apparell。



それが私のポートフォリオで唯一の実際のケースです。 絶対にゼロから開発しました。 アイデアとビジネスプランから始まり、Tシャツのテストサンプルで終わります。 私は初心者の服と野球帽のブランドで働いた経験がありました。 ソーシャルで調査を実施しました。 ネットワークやハブで、まだ完成していない製品について肯定的なレビューを受けました。 GeekPicnicやIT-bar Klyuchなどの組織の承認を受けました。







しかし、個人的な事情により、プロジェクトを凍結する必要がありました。 私が彼に戻ることは可能です。 そして、私はこのすべてを自分の空き時間にできました。 プロジェクト全体はBehanceで見ることができます。



ヒント2.雇用主は実際のケースが大好きです。





言い換えれば、この分野での経験なしで仕事を得るのは信じられないほどのカテゴリーです。 設計を始めたばかりのとき、実際の注文をすることは非常に困難ですが、これは動揺する理由ではありません。 メーラーアプリケーションのデザインを作成し、開発者の友人のサポートに参加して、アプリケーションストアに入力できます。 結果として、それは複雑なアプリケーションではありませんが、本物であり、ポートフォリオに含まれています。 残念ながら、私はこの分野に精通した開発者がいなかったため、これを実現することができませんでした。

わかった



私のケースで何が正確に機能したのかはわかりません（私のポートフォリオにあった実際のケースのメリットだったかもしれません）が、仕事を探し続けた後、ようやくスタートアップの1人でインターフェイスデザイナーの地位を得ました。 私の責任には、Webサービス用の新しいインターフェイスの開発、モバイルアプリケーション用のインターフェイスの開発、プレゼンテーションの少しの作業が含まれます。 私はそのような機会があったことを非常に嬉しく思いました。 しかし、1つの条件がありました。雇用主によると、これは私の主な仕事を妨げる可能性があるため、勤務時間外のプロジェクトの作成の完全な禁止です。



それにもかかわらず、これは私の喜びにそれほど影響しませんでした。 結局、私は最終的に、私がとても長く、一生懸命行っていたものを手に入れました-私が好きな分野での実際の仕事の経験



古いインターフェースとプレゼンテーションのみを使用してしばらくしてから（新しいインターフェースはまだ臭いがしませんでした）、次の2つの結論に達しました。



ヒント3.雇用主にさらに質問します。





多くの人は、彼らが賃金の額を聞くとすぐに、すべてをすぐに忘れて、すぐに仕事を始める準備ができています。 これは、共同出願人にとって最大の間違いの1つです。 私を信じて、高賃金だけに頼って仕事をするのは難しいでしょう。 それでは、あらゆる種類の落とし穴をすぐに規定して、それによって生活を少し楽にしてみませんか。 尋ねることを恐れないでください。 雇用主に質問します。 これは、私が直面しなければならなかったことを避けるのに役立ちます。 自分のプロジェクトを作成するために自由な時間を割くことができなかったため、私は自己開発に従事する機会を失いました。 自己啓発が禁止されているこの従業員はどのような従業員ですか？ 結局のところ、利点は明らかです。 開発中です。つまり、より多くの企業をもたらすことができます。 彼は特定の問題でより有能になります。



ヒント4.経験が最も重要です。





それが何であれ、それは貴重です。 彼のおかげで、あなたはあなたの職業を改善し、その度に良くなり続けています。 私の人生のその段階で遭遇した困難にもかかわらず、私はすべてがそのようになり、それ以外ではなかったことを非常にうれしく思います。 ほんの数か月で、この分野のスキルを大幅に向上させ、この職業での素晴らしい経験を得ました。



開発



しばらくして、新しいプロジェクトを絶えず開発および作成し続けながら、仕事を変えました。 UI Kit'amiに興味を持ちました。 Dribbble MeetUpカンファレンスにより、彼らへの関心がさらに高まりました。 私の意見では、これはデザイナーおよびインターフェイスデザイナーとして自分自身をアップグレードする素晴らしい方法です。 あらゆる種類のインターネットリソースを検討した結果、UI8とDesignmodo Marketに注目しました。 これらは、デザイナー向けのツールを販売する2つの取引プラットフォームです。 彼らは販売のための非常に厳しい選択を持っているので、そこに着くのは非常に困難です。 クリエイティブマーケット、2バンドル、Envatoなどのサイトにアクセスする方が現実的でしたが、UI8とDesignmodoマーケットの資格を得るという目標を設定しました。



6か月と数十のスケッチの後、私は最初の小さなブログキットEmerald Eye UI Kitをリリースしました。これはUI8で公開されました。 私はSketchを使い始めたばかりだったので、このキットはAdobe Photoshop専用に作成されました。







デザイナーとして、私にとって非常に重要な一歩でした。 自分が好きなことをしているだけでなく、他の人に本当に必要なものを作成しているので、それができることに気付きました。



多くの皆さんのように、私はすべての仕事に非常に批判的です。 私はいつもどこかでそうすることが可能であったことを見て、そうでなければ、タスクを解決する他の方法を見つけます。 それなしではどこにもありません。 私はこの地域で成長し、成長し、クジラは私の後に変わります。



すぐに、次のクジラを公開します。クジラは、簡単に操作できるクールな写真ではなく、完全に製品と呼ぶことができます。 結局のところ、クジラはブランドと同様に、独自のアイデア、歴史、気分を持っています。
