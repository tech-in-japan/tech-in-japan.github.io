インターフェース：紛らわしいストーリー



私たちの多くは、ほぼ毎日、さまざまなデバイスを相互に接続するために、さまざまなコネクタの非友好的なシステムに対処する必要があります。 同じシステムユニットでCOM、LPT、PS / 2、MIDI、USB、およびFireWireが表示されたとき、少なくとも一部のコンピューターの少なくともさまざまなポートを思い出してください。 幸運なことに、現代の家庭用コンピューターのCOM、LPT、PS / 2、FireWireは消滅しましたが、現在は周辺機器を接続するためにUSBしかありません 。 多様な周辺機器が増えています。 そして、さらに多くのインターフェースがあります。



このトピックでは、ワイヤレスインターフェースとアナログワイヤードを舞台裏に置き、個別の信号を送信するための既存のワイヤードインターフェースについて説明します。



USB





はい、ほとんどの古い接続インターフェースを置き換えたもの。 USBは、その名前（ユニバーサルシリアルバス）と実際の動作原理との不一致がなければ、すばらしいと言えます。 実際には、接続されたデバイスの実装を簡素化するために、USBにはマスターバスが必要であり、残りのデバイスはスレーブであり、実際、このバスは一般的ではなくシリアルであり、複数のデバイスを1つのマスターに接続するにはアクティブな接続が必要ですスプリッター（ハブ）。 したがって、USBトポロジはバスではなくスターであり、デバイスは相互に通信できず、バスマスターとのみ通信できることがわかります。 マスターコントローラーの一部であるルートハブを使用すると、最大127個のポートを接続できますが、全体の伝送速度は、最も遅い接続デバイスの最大速度によって制限されます。 USBでは、最大5レベルまでの深い分岐が可能です。 もう1つの欠点（USB 2.0まで）は、常にデータ転送が一方向にしか行われないことです。 接続するには、4本の信号線（および追加のシールドケーブル）のみが必要です。 USB 3.0仕様では、高速での全二重データ転送用にさらに5つ、オプションでスレーブからホストに電力を転送するためにさらに2つ追加されています。 USBは、5Vの電圧と500mAの電流制限（USB 3.0の場合は900mA）でデバイスに給電できます。



USB OTG



この仕様では、マスターとスレーブの両方としてデータの交換にデバイスが参加する可能性を規定しています。 残念ながら、交換スキームでは、1人のリーダーがまだ事前に決定されています。 これは主に、通常フラッシュメディア、マウス、キーボードなどの周辺機器を接続するためにコンピューターに接続されているモバイルデバイスで使用されます。



さまざまなコネクタ



マスターとスレーブを混同しないように、ケーブルには異なる端に異なるコネクタがあり、その1つはタイプA（マスター）、もう1つはタイプB（スレーブ）です。 これは、電源を決定するためにも行われます（マスターがスレーブに給電します）。





一部のメーカーは、デバイス用に独自のコネクタを作成し、購入者をケーブルに結び付けていることがよくあります。

USBコネクタは多くの場合eSATAコネクタと組み合わされ、他の用途もありますが、これについては後で学習します。



イーサネット





ネットワークデバイスを接続するためのほぼ不可欠なインターフェイス。 家庭での使用では、多くの場合、ワイヤレスデータ送信に置き換えられます。 Cat6Aケーブルを使用する場合、最大100mの10Gbps転送速度をサポートします。 標準RJ-45コネクタ。 8つの信号コアと画面。 エルボのケーブルを圧着するためのツールは、ケーブル自体、コネクタ、ケーブルの正しい圧着を確認するためのデバイスと同様に広く利用できます。 イーサネットおよびUSBを介して、周辺機器に電力を供給することができますが、供給電圧は48Vであり、同じ電流でデバイスは最大25Wを消費できます。 とりわけ、イーサネットはデバイスの電気的分離を提供します。これにより、たとえば、220Vの位相が屋根裏のどこかの信号線に到達した場合、ADSLモデムをコンピューターに安全に接続でき、USBは分離を提供しません。



ファイヤーワイヤー





正式にはIEEE 1394と呼ばれますが、より明確な名前で知られています。 80年代後半に開発されたこのインターフェイスは、驚くほど高速で、今日まで残っています。 たとえば、IEEE 1394b仕様は、最大3.2 Gb / sの速度でのデータ転送を提供します。これは、1394bが2002年に登場し、USB 3.0が2008年に登場した点でUSB 3.0と同等です。全二重データ転送など。これは、実際にはFireWireを介した高速データ転送速度に反映されており、USB上で400 Mbpsの速度で動作し、480 Mbpsで動作します。 FireWireは12Vの電力伝送をサポートし、10〜20Wの電力を供給します。 バス上の任意のデバイスがデータ転送を開始でき、接続されている他のデバイスと通信できます。 メインプロセッサの負荷を大幅に軽減するDMAモードでのデータ転送をサポート。

残念ながら、この規格はあまり人気がなく、主にApple製品に存在し、PCには主にビデオカメラを接続するための拡張カードの形で存在していました。

<irony> USBに対する主な議論の余地のない利点は、不均衡な接続コネクタです。これは、ケーブルを差し込んで裏返し、差し込むという繰り返し失敗する試行を排除します... </ irony>



eSATA





周辺機器を接続するためのもう1つのユニバーサルインターフェイス。 歴史的にドライブを対象としています。 仕様では、eSATApは12または5V電源もサポートしています。 最大6Gb / sのデータ転送をサポートします。 コネクタはUSBとは若干異なりますが、多くの場合、組み合わせられます（4線USB 2.0コネクタでのみ可能です）。



エクスプレスカード





それはPCMCIAの相続人である深い根を持っています。 外部TVチューナー、外部GSMモデム、メモリーカードアダプター、ソリッドステートドライブ、他のインターフェイス用アダプターなどの周辺機器を接続するために使用します。 最大2.5Gbpsの速度をサポートします。 親のことを忘れないでください。アダプターを介して古いPCMCIAデバイスを接続できます。



S / PDIF





まだ私たちの時代に達した古い時代からの何か。 光チャネルオーディオ接続インターフェイス（TOSLINKの場合）。 多くの場合、ディスクリートオーディオ信号の送信のみが必要な場合に使用されます。



DVI





ディスプレイデバイス、主にコンピューターモニターをコンピューターに接続するためのインターフェイス。 VGAと互換性があり、デジタルビデオ信号とともにアナログも送信するため、パッシブケーブルで古い機器を使用できます。 DVIを介して接続されたビデオデバイスは640x480ピクセルをサポートする必要があり、また、優先解像度など、ソースに関する情報を信号ソースに提供できます。

DVIは、60Hzで2560x1600の解像度と24ビット/ピクセル以上のビデオ伝送をサポートします。

私の新しいラップトップにはVGAコネクタがあり、これはDVIの前身であり、市場にはまだかなりの数のモニターモデルがあります（Yandex.Marketによると1833モデルしかありません）。

-VGAのみ：500

-VGAなし：70

-DVIがあります：1166

-HDMIを使用：427

-DisplayPortがあります：117

-HDMIはありますが、DVIはありません：156

この選択から、DVIが音声伝送をサポートしないという欠点と、少なくともある程度普遍的ではないという事実にもかかわらず、誰もDVIを拒否しないと結論付けることができます。

2010年12月に、Intel、AMD、および他のいくつかのメーカーは、DiplayPort / HDMIを支持して、2013〜2015年にDVI、VGA、LVDSのサポートを停止すると発表しました。



UDI





DVIとHDMIを置き換えるためにIntelによって提案されたが、DisplayPortを支持して忘却に陥った標準。



グヴィフ



Sonyによって提案された、標準的なギガビットビデオ伝送には、実用的なアプリケーションがないようです。



HDMI





最大4096x2160の解像度、最大60Hzのリフレッシュレート、最大48ビット/ピクセルの色深度および3Dサポート（残念ながら、一度にすべてではなく、たとえばフル4K解像度で24Hzのみ）でビデオを送信できる素晴らしい多目的インターフェイスは、xvYCCカラースペースをサポートしますRGBに加えて。 8チャネル（7 + 1）オーディオに加えて、データ転送がサポートされており、これを使用して、テレビのリモコンのキーストロークをHDMI経由で接続された信号ソースにブロードキャストできます。 DVIとの下位互換性があり、パッシブアダプターを使用できます（ただし、デジタルDVI-D信号のみ）。

すべてがうまくいくように見えますが、いくつかの「しかし」があります。 HDMIを使用する機器の製造元は、年間1万ドルに加えて、各デバイスに0.15ドルを支払う必要があります（デバイスにHDMIロゴを貼り付ける人には0.10ドル、送信されたHDCPコンテンツを保護する人にはさらに0.01ドル）。 これは多くの小規模メーカーを撃退するか、製造されたデバイスのコストを増加させます。 物理的に、HDMIは最大19個のコアを使用するため、ケーブルがより高価で太くなります。



ディスプレイポート





リモコンから信号ソースに信号を送信する機能を除き、HDMIの優れた機能をすべて備えています。 巨大な利点は、製造業者が誰かに何かを支払う必要がないことです。 さらに、DisplayPortは論理チャネルを介してイーサネットおよびUSB 2.0信号を送信できます。

IntelはHDMIとDisplayPortの両方をサポートし（ハードウェアを含む）、メーカーがコンシューマエレクトロニクスにHDMIを使用し、DisplayPortをコンピューターに使用することをターゲットにしています。 メーカーはますます混乱しているように思えます。

通常のコネクタに加えて、一部の企業はMini DisplayPortを使用しています。

2010年に、60 GHzのDisplayPortワイヤレス伝送規格が発表されました。



Mhl



Samsungが推進するもう1つの普遍的な標準。 HDMIの進化ですが、ビデオは最大1080pの解像度と色深度に制限されているという点で異なります。 5コアの物理データケーブルと標準のmicroUSBコネクタを使用します。 原則として、サムスン、LG、東芝製のモバイルデバイスやテレビでは非常に一般的です。



HDBaseT / 5Play



オーディオビデオ機器を接続するための別のインターフェイス。 つまり、伝送はツイストペアケーブルとRJ-45コネクタを備えた標準のCat5eケーブルで行われるため、接続デバイスが安価で簡単になります。 論理チャネルを介したイーサネット100MbpsおよびUSB伝送をサポートします。 100mの距離で10Gbpsの伝送速度（リピーターを使用して最大800m）。



サンダーボルト



主にアップル製のデバイスに存在する別のユニバーサルインターフェイス。 Mini DisplayPortコネクタと20線ケーブルを使用し、DisplayPortとPCIe信号を多重化します。 シリアルバスで最大7つのデバイスをサポートします。 最大10Wの接続デバイスに電力を供給できます。 最大転送速度10Gb / s。
