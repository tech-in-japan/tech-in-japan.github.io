MS SQL Server 2012のマスターデータサービスの実用的なアプリケーション

この記事では、MS SQL Server 2012でマスターデータサービス （MDS）を使用した最初の専門的な経験を共有したいと思います。最近まで、MS SQL Server 2012（Business IntelligenceおよびEnterpriseエディション）理論的には、彼は成功事例が実際にそれをテストするのを待っていました。









入力データ



私のクライアントは、Microsoft SQL Server 2012 Business Intelligence Editionに基づいたBIソリューションを使用しています。 このBIシステムの中心は、トランザクションシステム（ OLTP ）からのSSISパッケージを使用して作成されるデータウェアハウスです。 データウェアハウスは、 多次元データモデルのソースとして機能します。 多次元データモデルに基づいて、顧客分析はピボットテーブルを使用してExcelでレポートを生成します。 多くのデータソースがあり、トランザクションシステムのユーザーがディレクトリをランダムに操作するため、顧客は、分析分析の目的に便利なデータウェアハウスの一部のディメンションの階層を作成できるソリューションの必要性を示しました。 たとえば、OLTPシステムからインポートされた「Departments」ディメンション（テーブルdbo.dimDivisions）に保存されている会社の組織構造に関する情報は、OLTPシステムで生成された運用レポートには便利ですが、BIシステムでの分析には適していません。



ビジネス要件



法人顧客の正式な要件は、次のように簡単に説明できます。

  • 分析レポートの目的で会社の部門のデータを管理できるツールを開発する。
  • 既存のBIソリューションで新しいビジネスユニット情報を使用します。
  • 変更がOLTPシステムの動作に影響を与えることはありません。




予備評価と意思決定の選択



利用可能な入力データとビジネス要件に基づいて、顧客は次のソリューションを提供されました。

  • データウェアハウスに追加のディメンションを追加して、組織に関するデータを保存します。 分析レポート目的の構造。
  • 新しいディメンションのレコードと「従業員」ディメンションの間のリンクを提供します。
  • データウェアハウスの構造の変更を反映するために、多次元データモデルを変更します。
  • マスターデータサービスを設定します。 設定とは、従業員と部門に関するデータを手動で入力および編集する機能の作成、およびMSDとの間でデータをインポート/エクスポートするために必要な設定を意味します。
  • データウェアハウスからMaster Data Servicesへの新しい従業員に関するデータの自動インポートを提供します。
  • マスターデータサービスからデータウェアハウスへの部門および従業員所属のデータの自動エクスポートを従業員に提供します。




ソリューションの実装



データウェアハウスの改良


それでは、順番に行きましょう。 まず、データウェアハウス（dbo.dimDerivedDivisions）に新しいディメンション「任意単位」を作成し、それを「従業員」ディメンション（dbo.dimEmploees）に関連付けます。 このタスクのSQLスクリプトは次のとおりです。

多次元データモデルの改良


次に、新しいディメンションを多次元データモデルに追加します。 これを行うには、SQL Server Data Toolsで多次元データモデルプロジェクトを開き、新しいdbo.dimDerivedDivisionsテーブルをデータソースビューに追加します。 結果は次のとおりです。





このトピックから遠くないように、多次元モデルを完成させるプロセスを簡単に説明します。 多次元データモデルでは、新しいディメンション「任意の部門」を作成し、「Employees」ディメンションを介して新しいディメンションと既存のファクトテーブルとの接続を構成し、キューブを展開および処理します。







マスターデータサービスを構成する


新しいディメンション「カスタム測定」のコンテキストで分析レポートを作成するためのすべてのデータ構造の準備ができたので、最も重要なことに進みます-マスターデータサービスを設定します。 これを行うには、ブラウザで、SQL Server管理者から提供されたリンクをクリックし、次のようなMDS Webインターフェイスにアクセスします。







Master Data Servicesのインストールについては、 msdn.microsoft.comで詳細に説明されている日常的なタスクであるため、ここでは説明しません。 MDSの実際の使用により焦点を当てています。



したがって、最初に行う必要があるのはモデルの作成です。 MDSのモデルは、特定のビジネスエリアのエンティティを含む論理コンテナです。 この場合、エンティティ「従業員」と「ユニット」を含む「従業員」モデルを作成するのが適切です。 モデルを作成するには、[システム管理]リンクを使用して、マスターデータサービスWebインターフェイスの管理タスクに移動します。 開いたウィンドウで、従業員モデルの名前を入力し、モデルの保存ボタンをクリックします。







[モデルと同じ名前のエンティティを作成する]チェックボックスをオンにすると、同じ名前のEmployeesエンティティがモデルとともに自動的に作成されます。 次に、別のエンティティ「Divisions」を作成します。そのために、Employeesモデルを選択し、Manage-Entitiesメニューに移動します。







[エンティティの追加]ボタンをクリックします。







開いたダイアログで、新しいエンティティのパラメータを入力し、[エンティティを保存]ボタンをクリックします。 「組織単位」エンティティを作成するときに、[明示的な階層とコレクションを有効にする]チェックボックスがオンになっていることに注意してください（これは、エンティティの階層を作成できることを意味します）。下の部門階層の名前を示します。 明示的階層は、メンバーを任意の方法で編成できる階層です。 階層の各レベルには、任意の数のメンバーと次のレベルのネストがあります。







エンティティを作成したら、エンティティの属性を構成する必要があります。 エンティティ「Employees」に対して、属性「Division」（Division）を追加します。 Employeesエンティティを選択し、[選択したエンティティを編集]ボタンをクリックします。







開いているエンティティ編集フォームで、「リーフ属性の追加」ボタンをクリックして、「従業員」エンティティの最終要素の「ディビジョン」属性を追加します。







属性を追加するために開いたフォームで、属性の名前を入力し、属性タイプスイッチをドメインベースに設定します。 これは、この属性の値が特定のエンティティに属することを意味し、以下ではどのエンティティ、この場合は「部門」の本質であるかを示します。 最後に、[属性を保存]ボタンをクリックします。







Master Data Servicesでの手動データ入力


したがって、モデル「Employees」とエンティティ「Employees」および「Divisions」の準備ができたので、それらにデータを入力する必要があります。 ユーザーはエンティティ「Divisions」に手動で入力します。 このプロセスを示すために、Master Data Services Webインターフェイスのメインページに移動し、Information Worker TasksサブセクションでEmployeesモデルを選択して、エクスプローラーリンクをたどります。







階層「部門」を選択します。







制御する要素のタイプを選択します。 最初に、統合メンバーを作成します。







[追加]ボタンを押し、ユニットグループ「営業部」の名前を入力して、[OK]ボタンを押します。







同様に、次の図に示すように、他のユニットを追加して、ユニットの構造を作成します。







部門グループは太字でマークされ、有限要素は太字でマークされていることに注意してください。 階層の1つのレベルには、これらの要素と他のタイプの要素の両方を含めることができます。



Master Data Servicesへのデータのインポート


次に、既存のデータウェアハウスから従業員データをマスターデータサービスにインポートする必要があります（従業員と部門をさらに比較し、このデータをウェアハウスにエクスポートするため）。 マスターデータサービスを提供するSQLデータベースにMDSにデータをロードするために、SQLクエリでデータを挿入したり、リポジトリから新しい従業員レコードをインポートする特別なSSISパッケージを作成したりできる特別なステージングテーブルがありますマスターデータサービスでさらに処理するための一時テーブルへのデータ。 SSMSを開き、Master Data Servisesデータベースで一時テーブルを見つけます。 ここにあります：







例として、任意の10人の従業員レコードをデータウェアハウスからMaster Data Servicesデータベースの中間テーブルにインポートします。 これを行うには、次のSQLクエリを実行します。 



 INSERT INTO [stg].[Employees_Leaf] ([ImportType], [ImportStatus_ID], [BatchTag], [Code], [Name]) SELECT TOP 10 1, 0, N'Employees_Leaf_Batch00001', E.id, E.name FROM [DW].[dbo].[dimEmployees] E;




Master Data ServicesのWebインターフェイスに戻り、メインページで[統合管理]リンクに移動します。







開いたウィンドウに、SQLクエリによって作成されたばかりのパッケージEmployees_Leaf_Batch00001が表示されます。 [バッチの開始]ボタンをクリックして実行します。







パッケージを作成すると、ステータス、開始時刻と終了時刻、エラーに関する次のような情報が表示されます。







マスターデータサービスのデータ管理


それでは、データ管理モードに入り、ステージングテーブルの従業員レコードがどのようにロードされるかを見てみましょう。 これを行うには、マスターデータサービスWebインターフェイスのメインページの[インフォメーションワーカータスク]サブセクションで、従業員モデルを選択し、[エクスプローラー]リンクをクリックします。 開いたウィンドウで、新しい従業員データがMaster Data Servicesに追加され、検証待ち状態になっていることがわかります。







従業員の部門（部門）に関するデータは入力されていないことに注意してください。 各従業員について、彼が働いている部門を選択し、[OK]ボタンをクリックする必要があります。







Master Data Servicesからデータをエクスポートする


部門および部門への従業員の所属に関するデータを入力したら、それらをデータウェアハウスにインポートし直す必要があります。 これを行うには、MDSでサブスクリプションビューを作成します。 Master Data Services Webインターフェイスのメインページの[管理タスク]サブセクションで、[統合管理]リンクに移動します。







開いたウィンドウで、[ビューの作成]メニューに移動し、[サブスクリプションビューの追加]ボタンをクリックします。







エンティティ「Divisions」のプレゼンテーションパラメータを入力し、[保存]ボタンをクリックします。







同様に、Employeesエンティティのビューを作成します。







それでは、どのような表現がどのように使用されるのかを見てみましょう。 実際、すべてが非常に単純であり、MDSのビューはSQLデータベースの通常のビューにすぎません。 SSMSを開いて、これを確認してみましょう。







そして、この問題を解決するために最後にすべきことは、MDSビューからデータウェアハウスに情報をエクスポートするSQLスクリプトまたはSSISパッケージを開発することです。



結論



このソリューションの実装に約8時間を費やしましたが、これはこのようなタスクに十分な時間だと思います。 説明したソリューションでは、Master Data Servicesのすべての機能を使用しませんでした。たとえば、MDSへのインポート時にデータをクリアするために使用できるBusiness Rulesを無視しました。 しかし、顧客が兵器庫に新しい、かなりシンプルな、同時に柔軟性のあるツールであるMaster Data Servicesを持っていることは明確なプラスだと考えています。これは、参照NSIの処理と保存に関連するタスクを将来的に解決するのに役立ちます。
