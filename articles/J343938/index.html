試験NS0-507-NetApp認定実装エンジニア— SANスペシャリスト

こんにちは、Habr！ ほぼ2年前のNCDA認定に関する最初の記事で、NetApp NCIE認定のさらなる計画について言及しましたが、現在のNCDA認定が終了し始めるのを待って、より高いレベルに再認定することで、古い認定を延長し、新しいものを取得します。



カットの下で、トレーニングがどのように行われたか、読んだこと、勉強したことについての情報を共有します。おそらくあなたにとっても役立つでしょう。



試験について簡単に



NetAppは現在、3種類のNCIE-SAN試験を提供しています。



  • NS0-502-NetApp認定実装エンジニア— SAN、Data ONTAP 7モード
  • NS0-505-NetApp認定実装エンジニア— SAN、Eシリーズ
  • NS0-507-NetApp認定実装エンジニア— SAN、ONTAP


私たちの試験NS0-507はONTAPバージョン9.1に基づいており、60の質問で構成されており、1.5から2時間の時間が割り当てられています。 合格点は62％（60問中37問）であり、このテストは非常に簡単です。 試験に合格するための主な要件は、 NCDAステータスの存在です 。



試験準備



この試験に合格するための一般的なガイドラインとして、 NetAppは以下を提供しています 。



  1. 6か月以上からSANを使用した経験があります。
  2. ONTAP SAN Fundamentalsオンラインコースを聴きます。
  3. SAN実装-スイッチ構成のオンラインコースを聴きます。
  4. Data ONTAP SAN実装インストラクターによる公式コースを聴きます。
  5. ドキュメントを読んでください。


可能であれば、Data ONTAP SAN実装コースを強くお勧めします。このコースは、SANネットワークおよびこれらのネットワークでのNetAppストレージシステムの操作に精通しています。



また、試験で取り上げられる可能性のあるトピックに精通することをお勧めします。



自己準備



試験を受ける予定がない場合でも、ストレージシステムとネットワークの分野で視野を広げるすばらしい本を2冊読むことをお勧めします。



  1. IBM-ストレージエリアネットワークの概要-約300ページの最も役立つ情報。
  2. EMC-データストレージから情報管理（第2版）まで-ストレージシステムとストレージネットワークに関する本だけでなく、このトピックは非常に密接に、特に前半で触れられています。 ロシア語で利用可能。


まず第一に、自己トレーニングのために、NetAppポータルで利用可能な無料の模擬テストを受けることをお勧めします。



試験NS0-507-メインテストには、実際の試験に近い質問が含まれています。

NS0-158試験の場合は、NCDA試験です。 ここでも、SANとストレージの構成に関する質問があります。



テスト試験の結果に応じて、長所と短所を判断し、何を締めるべきか、どの問題が発生しないかを理解できます。



また、 NetApp Learning Centerポータルで利用可能な無料のWebベースのトレーニングを受ける機会があります（ポータルでの登録が必要です）。 利用可能な資料の中で、私は次のことに注意しました。



  1. ONTAP SANの基礎;
  2. SANデザイン
  3. Data ONTAPのSANの基礎;
  4. SAN実装-スイッチ構成。
  5. データネットワークの基礎;
  6. NetAppのファイバーチャネルベーシス。
  7. ファイバーチャネルSANのトラブルシューティング。
  8. ファイバーチャネルスイッチングの概念。


これらの各コースには約1時間かかり、最後に合格したテストに合格することにより、習得した知識を確認できます。 これらのコースのすべてがONTAP 9で行われているわけではありませんが、いずれにしてもSANでのNetAppの基本原則を十分に説明しています。



今、最も興味深い-有用なドキュメントについて。 NetAppには、試験の準備に役立つ資料にリンクする優れたドキュメントがあります。 執筆時点で、推奨されるNS0-507試験のドキュメントは11ページにあります。



現在までに、NS0-507試験の準備のために約50の文書が示されています。 各ページには30〜200ページ以上が含まれています。 私の意見では、最も興味深いものを選択しました。 これらはすべて、リファレンスドキュメントのリンクからダウンロードできます。



  1. SAN構成ガイド-NetAppストレージをSANに接続するハイライト。
  2. SAN管理ガイド-SVM、LIF、LUNマッピングなどの構成。
  3. システム管理リファレンス-ノード、ライセンス、AutoSupportの管理。
  4. ディスクおよび集合体の電源ガイド-ディスク、集合体、RAIDグループを管理します。
  5. 高可用性構成ガイド-ノードのフォールトトレランスを構成および管理します。
  6. 論理ストレージ管理ガイド-ボリューム管理、重複排除、圧縮、ボリューム移行。
  7. ネットワーク管理ガイド-ネットワーク管理。
  8. ESX / Windows / Red HatのFC構成-FCを介してNetAppを構成し、一般的なシステムに接続します。
  9. ESX / Windows / Red HatのiSCSI構成-iSCSIを介してNetAppを構成し、一般的なシステムに接続します。
  10. 外部LUNインポートを使用したSAN移行-非常に興味深いドキュメント。 サードパーティのストレージシステムからNetAppへのデータ移行。
  11. アップグレードおよびリバート/ダウングレードガイド-ONTAPコントローラーのアップグレードバージョン。
  12. ファイバーチャネルオーバーイーサネット。 エンドツーエンド展開ガイド。
  13. スケーラブルなSANのベストプラクティス。
  14. イーサネットストレージのベストプラクティス。


別のリストに、バージョン8.3のドキュメントが追加されます。 また、多くの興味深いものがあり、観測によると、バージョン9.xのドキュメントよりも2倍「厚く」なっています。 主なことは、さまざまなバージョンの最高の試験で混乱しないことです。



  1. clustered Data ONTAP 8.3。 物理ストレージ管理ガイド。
  2. clustered Data ONTAP 8.3。 SAN管理ガイド。
  3. clustered Data ONTAP 8.3。 SAN構成ガイド。


NetAppのドキュメントに加えて、Cisco Nexus、Cisco MDS、BrocadeデバイスでのSAN構成の基本もお読みください。 ドキュメントリンクはリファレンスガイドにあります。 スイッチの初期化、ゾーニングのセットアップ、およびトラブルシューティングに関する基本情報を読むことをお勧めします。 カバーからカバーへの読み取りはオプションです。



ドキュメントを読むときは、「ドラフト」を開始することを強くお勧めします。このドラフトでは、読書中に見つかった興味深い点を取り上げる必要があります。 このドラフトは、試験の数日前に読んで、構造に関する知識と同様に、以前に読んだものすべてを一般的に覚えておくと非常に便利です。



理論的な部分に加えて、ONTAPシミュレータにアクセスできる人は、回路を組み立てる絶好の機会があります：ONTAP + ESX + vCSA + VSC + Windwos + Red Hat。 このスキームにより、iSCSIを介してONTAPをさまざまなコンシューマに接続する問題を理解し、VSCとその機能を確認できます。



上記に基づいて、試験の準備をした後、次のことを理解する必要があります。



  1. SANの基本概念。 工場のトポロジーを区別します。 アドレッシングの発生方法を理解します。
  2. 基本的な初期化、SANおよびCiscoおよびBrocade機器のゾーニングの最小構成、トラブルシューティング。
  3. ストレージをネットワーク、光ファイバーケーブルの種類、Twinaxに接続します。
  4. 基本的なストレージの初期化、ライセンス、EMSのセットアップ、および自動サポート。 集約の作成。
  5. ボリューム、スナップショット、FlexClone、ボリューム移行を操作します。
  6. すべてのプロトコルに従って、SVM、LIFの構成、ムーンの作成、ムーンのホストへのマッピング。
  7. Windows、Red Hat Linux、ESX、OpenStackのクライアントから専用の衛星を接続します。
  8. 「古典的な」状況の基本的なトラブルシューティング。
  9. マルチパス、ALUA、Windows MPIO、ONTAP DSM、DMマルチパス。
  10. VMware vSphere用のNetApp仮想ストレージコンソール（VSC）
  11. 目的ハードウェアユニバース（HWU） 、 相互運用性マトリックスツール（IMT） 、使用の基本;
  12. SANに関するONTAPバージョン8.2-8.3-9.1の違い。
  13. 高速で信頼性が高く、フォールトトレラントなストレージネットワークを構築するためのあらゆる種類のベストプラクティス、およびこれらのネットワークでのNetAppストレージシステムの同様の操作。
