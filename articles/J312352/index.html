TailSampler-Apache.JMeterへのGETリクエストの並列送信





1. HTTP Request Tailプラグインの目的



プラグインは、埋め込みリソースのロードを簡素化し、 指定された GETリクエストを並行して実行できるようにします 。 ダウンロードしたリソースの構成とこれらのリソースをロードする方法に関して、テストをできるだけブラウザに近づけます。



TailSamplerは必要に応じて役立ちます。



  • GET要求のグループを並行して実行します。
  • 1000 HTTPリクエストコンポーネントを作成せずに1000 GETリクエストを実行します
  • AJAX 、 Adobe Flash 、 Adobe AIR 、 SilverLigthなどを積極的に使用するサイトをテストします。


2.使用説明書



HTTP Request Tailは、リンクのリストをHTMLドキュメントに変換し、組み込みリソースをロードして、指定された各リンクのGETリクエストを作成します。





図1. TailSamplerプラグインのデータ準備プロセスとその使用



  1. Fiddlerを起動します。
  2. Webブラウザーでページを開きます。ロードはテストで詳細に模倣する必要があります。これは、 Adobe Flash 、 Adobe AIR 、 Microsoft SilverLigth 、 ActiveXコンポーネントを含むページです。
  3. ページで必要なアクションを実行します。
  4. Fiddlerリンクから、ブラウザで実行されたかのように、テストで並行して実行する必要があるGETリクエストにコピーします。
  5. TailSamplerにリンクを挿入し、必要に応じてパラメーター化します。
  6. テストで指定されたリクエストの並列実行を取得し、並列度が設定されます。






3.標準のHTMLパーサーJMeterの動作



JMeterでHTTP経由でHTMLドキュメントを取得する標準的な方法は、 HTTPリクエストサンプラーを使用することです。 HTTPリクエストでは、埋め込みページリソースを簡単にリクエストできます-チェックマーク[v] 埋め込みリソースを取得します。 ページリソースへのリンクを形成する標準のパーサーはLagartoBasedHtmlParserです。 パーサーは、 jmeter.propertiesファイルのhtmlparser.classname設定で変更できます。



5つの人気サイトのApache.JMeterパーサーの研究は、記事「Apache.JMeterのHTMLパーサーの選択」に記載されています。



  • https://habrahabr.ru/post/308254/


パーサーを使用すると、ほとんどすべての必要なリソースがロードされます。 ただし、異なるサイトでは、ダウンロードの完全性は異なります。 したがって、WebサイトがMicrosoft Silverlightに実装されている場合、 Apache.JMeterパーサーのパフォーマンスは約0％になります。 一方、 TailSamplerを使用すると、ブラウザが簡単に機能する方法と同様の負荷を適用することが可能になります。

5.2.1。 絶対リンク


絶対リンク形式の説明については、RFCを参照してください： https : //tools.ietf.org/html/rfc3986



絶対リンクはプロトコルで始まります：

  • file://



  • http://



  • https://





他のプロトコルはHTTP Request Tailによって処理されません。 絶対リンクの例：



  • file://C:\Data\htmlDocument.html#Part1



  • http://www.pflb.ru/



  • https://yandex.ru/?q=Testing&client=Mozilla







5.2.2。 相対リンク


相対リンクはフィールド値によって補完されます：

  • パス -ページに関連し、ホストに関連しないリンクのディレクトリ。ここで、プロトコルとホストおよびポートを指定できます。
  • Webサーバー -ホストとポート：

    • サーバー名またはIP ;
    • ポート番号
  • プロトコル[http] -プロトコル。プロトコルがPathで指定されていない場合、値が考慮されます。値はfile



    、 http



    、 https



    です。


相対リンクの例：



  • image1.png



  • /images/image1.png



  • /ResourceGenerator.aspx?id=0121



  • subFolder/style.css



  • subFolder/1/2/3/test.php







5.2.3。 変数と関数を使用したパラメーター化

5.4。 既知の効果





5.4.1。 ログ内の+1リクエスト


ルートリクエストは送信されませんが、ログに記録されます。



  • 弱いが、 1秒あたりのヒット数インジケーターの値を過大評価する-1 秒あたりのヒット数。
  • 間接的に、JMeterログの一般的な統計に影響します。
  • この単一のクエリが統計に与える影響を減らすには、実際のサブクエリの数を多くし、[ 組み込みリソース]フィールドに数十のリンクを配置する必要があります。

  • JMeterでは、リダイレクトはサブクエリとして表示されます。視覚的にはリクエストのカスケードのように見えます。

7.プロジェクトの構造



ディレクトリ内のソースコード：



/ src / ru / pflb / jmeter

  • プロトコル/ http / config / gui ：

    • TailUrlConfigGui.java-埋め込みリソースへのリンク用の大きな入力フィールドを持つコントロール。
  • サンプラー：

    • wrapper-実装フォームで指定された使用するラッパー：

      • WrapperHTTPFileImpl.java- file://



        を使用するラッパーfile://



        サブクエリのプロトコルハンドラ。
      • WrapperHTTPHC3Impl.java-サブクエリの実装設定からHttpClient3.1



        を使用するラッパー。
      • WrapperHTTPHC4Impl.java-サブクエリの実装設定からHttpClient4



        を使用するラッパー。
      • WrapperHTTPJavaImpl.java-サブクエリの実装設定からJava



        を使用するラッパー。
      • WrapperHTTPSamplerFactory.java-ラッパーを作成するためのファクトリーは、プロトコル値と実装設定によってハンドラーを返します。
    • EscapeUtils.java -htmlエスケープの実装により、ロシアのドメイン、Unicode、およびリンク内の特殊文字を使用できます。
    • ITailHTTPImpl.java-すべてのハンドラーの基本インターフェース。
    • TailHTTPHC4Impl.java-要求を送信せずに指定された応答本文を使用する修正されたHttpClient4 。
    • TailHTTPSamplerProxy.java - TailSamplerのすべてのロジックが実装されているプロキシクラス：

      • 最初のリクエストでは、 埋め込みリソースフィールドからリンクのリストを取得してTailHTTPHC4Implに渡します -コンテンツ内のiframeのリストを含むページを受信するエミュレーション。
      • 埋め込みリソースの要求の場合、標準ハンドラーが作成されて呼び出されます。これは、 実装設定で指定されます-要求の実際の送信。
    • TailHttpSamplerGui.java - TailSamplerの視覚的表現。


他の補助ディレクトリは、プロジェクトの便利なデバッグに使用されます。



7.1。 形



TailUrlConfigGuiフォームは、 Apache.JMeter 2.13のメインHTTPリクエストフォームを修正したもので、リクエスト本文の編集とパラメーターのリストの指定のためのフィールドは削除されますが、リンクのリストを入力するための1つの大きなフィールドが追加されます。

また、 TailHttpSamplerGuiの外観は、新しいTailUrlConfigGuiメインコントロールを使用するHTTPリクエストのコピーでもあります。



コピーアンドペーストがないわけではありません。 HTTPリクエストテールがHTTPリクエストとほとんど変わらないように、このステップに進みました。



ただし、 JMeter 3.0では、 HTTPリクエストの外観が変更され、コントロールが再配置されました。 現在、 HTTPリクエストテールはJMeter 3.0のHTTPリクエストとは異なります 。 しかし、これは作業に影響しません。



7.2。 ラッパー



既存のコードを最大限に活用したかったのですが、コピーペーストはしたくありませんでした。 したがって、Wrapperクラスが作成されました。これにより、メソッドsample 、 notifyFirstSampleAfterLoopRestartおよびthreadFinishedが使用可能になりました。



7.3。 シールドユーティリティ



エスケープクラスのコードは、ibm.comから取得されます： www.ibm.com/developerworks/en/library/se-prevent/index.html

HTML4の説明を含むRFCにより、htmlエンティティのテーブルが拡張されました。

9.システム要件





9.1。 Apache.JMeter 2.13を使用するには



  • Java（TM）SE Runtime Environmentバージョン6（ビルド1.6.0_45-b06）以降：



    • http://www.oracle.com/technetwork/java/javase/downloads/419409-最低限必要なバージョン（プロジェクトを再構築する必要があります。プロジェクトはJDK 1.7用に構築されたリポジトリにあります）。
    • http://www.java.com/en/download/manual.jsp-Oracle JDKの最新バージョン。
    • OpenJDK 1.7.0_09-icedteaなどのOpenJDKバージョン1.6以上も適しています。
  • JMeter 2.13：



    • https://archive.apache.org/dist/jmeter/binaries/apache-jmeter-2.13.zip-Apache.JMeter 2.13。


Apache.JMeter 2.13はJava 6を使用してコンパイルされます 。たとえば、 OpenJDK 1.7.0_09-icedteaを使用してTailSamplerプラグインをビルドし、 Apache.JMeter 2.13 + Java 6のみが存在するコンピューターでコンパイル済みプラグインを実行すると、 Apache.JMeter 2.13起動しますが、プラグインは起動しません。 その結果、それらのバンドルは機能しません。 Javaの異なるバージョンとそれらの互換性のためのプロジェクトとライブラリを構築する問題は、個別の指示に値します。



9.2。 Apache.JMeter 3.0を使用するには



  • Java（TM）SE Runtime Environmentバージョン7（ビルド1.7.0_80-b15（Oracle Corporation））以上：



    • http://www.oracle.com/technetwork/java/javase/downloads/java-archive-downloads-javase7-521261.html#jre-7u80-oth-JPR-最低限必要なバージョン。
    • http://www.java.com/en/download/manual.jsp-Oracle JREの最新バージョン。
    • 1.7以上のOpenJDKバージョンも適しています。
  • JMeter 3.0：



    • http://jmeter.apache.org/download_jmeter.cgi-Apache.JMeter 3.0。


Apache.JMeter 3.0はJava 7を使用して構築されます。TailSamplerプラグインは、Java 7またはJMeterとプラグインの実行に使用されるJavaのバージョンを使用して構築する必要があります。
