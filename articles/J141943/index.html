Windows Phone 7用に「Snake」を作成します

著者について： Pasi Manninenは、モバイルおよびWebアプリケーションを開発し、Nokia開発者チャンピオンおよびAdobe Community Professional、Adobe認定エキスパート、Flex、FlashおよびFlash Mobileのトレーナーです。 ユヴァスキュラ大学（フィンランド最大の大学の1つ）を卒業し、応用数学とコンピューターサイエンスの学位を取得しました。



はじめに



この記事では、Windows Phone用のSnakeの簡単なバージョンを作成する方法を示します。 ジェスチャーを使用して、さまざまな方向にヘビを動かすことができます。 アプリケーションは、メニュー画面、ゲーム自体、およびゲームの結果が表示される最終画面で構成されます。







Windows Phone 7.1 SDK



Windows Phone 7を実行しているデバイス用のアプリケーションを開発するには、Windows Phone 7.1 SDKをインストールする必要があります。 最新バージョンはこちらからダウンロードできます。





Windows Phoneゲーム



新しいWindows Phoneゲームの開発を開始するには、Microsoft Visual Studioを起動し、新しいプロジェクトを作成して、Windows Phoneゲーム（4.0）テンプレートを選択します。 プロジェクト名としてSnakeItを使用します。







この例では、C＃を使用してゲームを開発します。



ゲーム用の画像の準備



まず、食べ物とヘビ自体にシンプルなグラフィック要素を追加します。 表面全体で繰り返すことができる25x25の画像を使用します。



食べ物


ブラウザでSnakeItプロジェクトソリューションのコンテンツを右クリックし、[追加]> [新しいアイテム...]を選択します。

ビットマップファイルを選択し、FootTile.bmpという名前を付けます







食べ物を示すために25x25の緑の長方形を描きます。 ファイルを忘れずに保存してください。







ヘビ


同じ手法を使用して、ヘビ自体の赤い正方形を描きます。 画像にSnakeTile.bmpという名前を付けます。



ゲームフォント



画像の作成に使用したのと同じ方法で、ゲームのフォントを作成できます。 ソリューションエクスプローラーでプロジェクトのコンテンツを右クリックし、[追加]> [新しいアイテム...]を選択します。

Sprite Fontを選択し、Segoe20.spritefontという名前を付けます。







Project Explorerでフォントをダブルクリックし、そのサイズを20に設定します。忘れずにフォントを保存してください。



食品の食品クラスを作成する



ソリューションエクスプローラーでプロジェクトのコンテンツを右クリックし、[追加]> [新しいアイテム]を選択します。インストールされているテンプレートのリストで[コード]を選択し、Foodクラスを作成します。







このクラスは、テクスチャ（テクスチャ、タイプ）および位置（位置）に関する情報を格納します。 幅と高さの寸法は、Game1クラスで衝突を定義するために使用されます。 



using System; using Microsoft.Xna.Framework; using Microsoft.Xna.Framework.Graphics; namespace SnakeIt { class Food { // Texture representing the food public Texture2D Texture; // Position of the food relative to the upper left side of the screen public Vector2 Position; // Get the width of the food public int Width { get { return Texture.Width; } } // Get the height of the food public int Height { get { return Texture.Height; } } // Set Food Texture and Position public void Initialize(Texture2D texture, Vector2 position) { Texture = texture; Position = position; } // Draw Food to the Screen public void Draw(SpriteBatch spriteBatch) { spriteBatch.Draw(Texture, Position, null, Color.White, 0f, Vector2.Zero, 1f, SpriteEffects.None, 0f); } } }




Snakeクラスの作成



Foodクラスを作成したのと同じ方法でSnakeクラスを作成します。

XNA FrameworkとCollectionsのListクラスを使用して、ヘビのすべてのタイルを処理します。

プロパティ


ヘビを定義するには、そのプロパティのいくつかを定義します。 ヘビの画像をテクスチャテクスチャとしてロードし、画面の中央にヘビの初期位置を設定します（これはGame1クラスの後半で実装します）。 すべてのスネークセルは、snakeTilesリストに保存されます。 ヘビの動きの方向は、ジェスチャーを使用してGame1クラスで制御されます。

初期化


Game1はSnakeオブジェクトを初期化します。 スネークテクスチャがロードされ、その初期位置が設定されます。 ヘビのすべてのプロパティは、最初の起動時およびプレイヤーが再びゲームを完了したいときにデフォルトのプロパティにリセットされます。 デフォルトでは、ヘビは左に5ピクセル移動し、その頭のセルが作成されます。 



 //   public void Initialize(Texture2D texture, Vector2 position) { Position = position; Texture = texture; Reset(); } //  ( ) public void Reset() { speed = 5f; direction = 1; Tile tile = new Tile(Texture, Position); snakeTiles = new List<Tile>(); snakeTiles.Add(tile); newTile = false; }




更新する


ゲームはメソッドを呼び出して、新しいゲームステップごとにヘビを更新します。 この方法では、蛇の頭の位置を見つけます（これはsnakeTilesリストの最初のセルです）。 ヘビに新しいセルを追加する必要がある場合（つまり、食べ物が食べられたとき）、headPostionの新しいセルを作成し、snakeTilesリストに追加します。 次に、ヘビの頭だけを動かして、そのサイズを大きくします。 ヘビの新しいセルがもはや予想されない場合、最後のセルをヘビの「頭」に移動するだけで、ヘビ自体を新しい位置に移動します。

描画


各スネークセルは、Draw（）メソッドで描画されます。 このメソッドは、Game1クラスから呼び出されます。 



 public void Draw(SpriteBatch spriteBatch) { for (int i = 0; i < snakeTiles.Count; i++) { spriteBatch.Draw(snakeTiles[i].Texture, snakeTiles[i].Position, null, Color.White, 0f, Vector2.Zero, 1f, SpriteEffects.None, 0f); } }




Tileクラス（Snakeクラスに関連するネストされたクラス）


ネストされたTileクラスを作成します。 このクラスは、ヘビの細胞を決定します。 各セルには、位置とテクスチャがあります。

クラスゲーム1



Game1クラスは、Windows Phone XNA Gamesに基づくゲームのメインクラスです。 ここで、ゲームのコンテンツの初期化とロードが行われます。 ジェスチャーと衝突検出は、Updateメソッドと呼ばれるゲームメソッドで使用されます。 DrawはGameModeプロパティによってゲームモードをチェックし、対応する画面を描画します。



名前空間


通常のXNAおよびTouchクラスを使用して、ジェスチャーを定義します。 

 using System; using System.Linq; using Microsoft.Xna.Framework; using Microsoft.Xna.Framework.Graphics; using Microsoft.Xna.Framework.Input; using Microsoft.Xna.Framework.Input.Touch;




プロパティ


GraphicsDeviceManageクラスを使用して、現在のディスプレイに関する情報を取得します。 SpriteBatchクラスは、画面に情報を表示するために使用されます。 メニュー、ゲーム自体、またはゲーム完了モードを描画するために、Draw（）メソッドで何が起こっているかを制御するために、さまざまなゲームモードを使用することにしました。 これは、画面上のさまざまなモードの描画を処理する非常に簡単な方法です。 さらに、この問題を解決するための爆発的なアプローチが可能です（リンク： Game State Managementを参照）。 現在のゲームモードはModeプロパティに格納され、すべての種類のモードはGameModeの列挙として定義されます。 



 //     private GraphicsDeviceManager graphics; //      private SpriteBatch spriteBatch; //  private Snake snake; //    private Food food; //  private SpriteFont segoe20; //     private Vector2 scorePosition; //  private float score; private const float FOOD_POINTS = 20; //    GameMode Mode; //    enum GameMode {Menu, Running, Complete}

ゲーム更新


更新は、ゲームを更新するためのロジックを実装します：衝突チェック、入力への応答、オーディオの再生。 ここでは、ゲームの実行中にヘビの位置を更新し、ヘビと食物との衝突をチェックします。 ゲームが実行されているかどうかに関係なく、このメソッドが呼び出されるたびにすべてのジェスチャーがチェックされます。

ジェスチャーチェック


ジェスチャーは、毎回Updateメソッドでチェックされます。 最初に、新しいジェスチャが使用可能かどうかを確認し、使用可能な場合は読み取ります。 Flickのようなジェスチャーが定義されている場合、Delta値も取得します。 その後、ヘビは適切な方向に移動します。 プレイヤーはヘビを反対方向に動かすことはできません（ゲームは終了します）。 タップジェスチャが検出されると、ゲームモードが変更されます（メニューから新しいゲームを開始するか、ゲームを再起動します）。
