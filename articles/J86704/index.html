Pixel BenderフィルターをFlash / Flexアプリケーションに接続するには、少なくとも2つの方法があります。 最初の方法では、メタタグを使用してフィルターを埋め込みます。



[Embed(source="SomeFilter.pbj", mimeType="application/octet-stream")]

private var _someFilter:Class;











この場合、およそ次のシナリオでさらにフィルターが使用されます。



var shader:Shader = new Shader(new _someFilter() as ByteArray);

var shaderFilter:ShaderFilter = new ShaderFilter(shader);



var bitmap:Bitmap = new Bitmap(someBitmapData);

bitmap.filters = [shaderFilter];











filtersプロパティはフィルターの配列です。 ただし、配列の最後に新しいフィルターを追加するだけでは（たとえば、プッシュ（[...]）メソッドを使用する場合）、このフィルターを使用する必要はありません。 最初にフィルターを配列に追加してから、この配列をfiltersプロパティに再割り当てする必要があります。



2番目の方法では、実行時にフィルターをロードします。



var urlLoader:URLLoader = new URLLoader();

urlLoader.dataFormat = URLLoaderDataFormat.BINARY;

urlLoader.addEventListener(Event.COMPLETE, onLoadComplete);

urlLoader.load(new URLRequest("SomeShader.pbj"));

var shader:Shader;



function onLoadComplete(event:Event):void {

shader = new Shader();

shader.byteCode = loader.data;



var shaderFilter:ShaderFilter = new ShaderFilter(shader);

var bitmap:Bitmap = new Bitmap(someBitmapData);

bitmap.filters = [shaderFilter];

}











一般に、フィルターをロード（または埋め込み）した後は、フィルターが何であるかは関係ありません。PixelBenderまたは基本言語フィルターで記述されています。 それらのさらなる使用法はほぼ同様です。

Loaderクラスを使用して画像をロードしたとします。 この方法を使用するには、2つの重要なポイントがあります。



1. Loaderクラスのオブジェクトのload（）メソッドには、通常、次のシグネチャが必要です。



loader.load(new URLRequest(this._photoSrc), new LoaderContext(true));











メソッドの2番目の引数は、最初の引数がtrueになるLoaderContextクラスのオブジェクトでなければなりません。 この結果、Flash Playerは画像ファイル自体のダウンロードを開始する前にセキュリティポリシーファイルをダウンロードしようとします（ユースケースSecurity.allowDomain（ "*"）。この場合はフィードしません）。



2.イベントハンドラーEvent.COMPLETEには、次のものが必要です。



var bitmapData:BitmapData = new BitmapData((event.currentTarget as LoaderInfo).content.width, (event.currentTarget as LoaderInfo).content.height);

bitmapData.draw(this.loader);



this.bitmap = new Bitmap(bitmapData);

this.curPhoto.source = this.bitmap;

this.bitmap.filters = [shaderFilter];











重要なことは、this.loader.contentをdraw（）メソッドに渡すのではなく、this.loaderオブジェクト自体を渡すことです。 なぜそうなのか、それはバグなのか機能なのか？ 正直わかりませんでした。 しかし、それは機能し、他のすべてのオプションはサンドボックスのセキュリティポリシーに違反します。



したがって、ビットマップオブジェクトはポケットにあり、表示リストに追加し、フィルターを適用して、サーバーにアップロードできます。 物語は終わりですか？ そうでもない。



2つ目の落とし穴は、 すべてのフィルターが同等に役立つわけではないことです。 実際、filtersプロパティを介してフィルターを適用する場合、この画像自体のデータには適用せず、この画像の表示オブジェクトにのみ適用します。 BitmapDataクラスには実際の画像データが含まれ、Bitmapクラスには表示リストに含めることができる表示オブジェクトが含まれていることを思い出してください。 覚えているように、Bitmapオブジェクトのfiltersプロパティにフィルターを割り当てました。つまり、BitmapDataオブジェクト自体（this.bitmapオブジェクトのbitmapDataプロパティに含まれる）は変更されないままでした（ロードしようとしたときに基本的に同じことが起こりました）上記の場合、サーバーへのImageオブジェクトのインスタンス）。



解決策は明らかなようです-applyFilter（[...]）メソッドをいくつかの中間BitmapDataオブジェクトに適用する必要があります（この場合、メソッドの最初の引数はthis.bitmap.bitmapDataプロパティのBitmapDataオブジェクトになります）。 しかし、私の場合、いくつかのプロパティを変更するためのサポートが実装されると、後続の各フィルターのアプリケーションは前のフィルターのすべての変更を上書きします。 つまり 操作後は、最後のフィルターの適用による変更のみが表示されました。 残念ながら、このメソッドで1つのフィルターではなく、フィルターの配列を使用することはできません。

var bitmapdata:BitmapData = new BitmapData((curPhoto.content as Bitmap).width, (curPhoto.content as Bitmap).height);

bitmapdata.draw(curPhoto.content as Bitmap);



var jpgEncoder:JPGEncoder = new JPGEncoder(85);

this.jpgStream = jpgEncoder.encode(bitmapdata);

つまり BitmapDataクラスの中間オブジェクトを作成し、その中にImageオブジェクトのコンテンツをレンダリングしました。 思い出してくださいが、以前に、ビットマップクラスのthis.bitmapのインスタンス（フィルターを適用したもの）をクラスImageのオブジェクトのソースプロパティに割り当てました。 これは、実際には、フィルター自体を適用して画像自体のデータを操作したのではなく、その表示オブジェクトのみを操作したことを意味します。
