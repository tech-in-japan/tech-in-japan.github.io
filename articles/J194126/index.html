OpenJDKバグトラッカーが公開されます

こんにちは、Habr！



OpenJDK-shnaya JIRAがすべての人にオープンになりました！ アドレス： http : //bugs.openjdk.java.net



古いSanovoバグトラッカーでは、プロジェクトとは無関係に、バグの番号付けは横断的でした。 JIRAでは、すべての配置が少し異なります。ご存じのとおり、各プロジェクトには特定のプレフィックス（この場合は「JDK」）があり、その後にハイフンとバグ番号が続きます。 したがって、 bugs.sun.comで以前にあったものは、JBSのバグ4459438で、番号JDK-4459438を受け取りました 。











このイベントは、Javaコミュニティで長い間期待されていました。 Oracleの内部では、長い間秘密ではありませんでしたが、新しいエラー追跡システムは来週JavaOneカンファレンスで正式に発表されると予想されていました。 せっかちだったようです！ 昨夜、 バーナードで初めてニュースがツイッターで公開されました







Java Bug System（JBS）の誇りを持つJIRAは、約1年前に登場し、Bane Trackerの古い尊厳に置き換わりました 。 移行に貢献したものは何ですか？



私の意見では、JIRAへの切り替えは、次のような多くの理由で必要でした。

  • 古いバグトラッカーは完璧ではなく、さらにサポートする必要がありました。
  • 古いバグトラックへのアクセスは、内部のSun / Oracleネットワークからのみ可能です。
  • Sun / Oracleの従業員ではない人のための古いバグトラッカーのJava部分へのアクセスは、マグバグbugs.sun.comを介してのみ達成できましたが、これも理想的ではありませんでした。 たとえば、バグの発見にはいくつかの問題があります。
  • 2006年以来、OpenJDKプロジェクトの一環として、コミュニティはJavaプラットフォームの開発に積極的に関与してきました。 新しいツールが必要でした。
  • メーリングリストはそのようなツールになっています。 しかし、今では20世紀の挨拶のように見えますよね？
  • 最後に、JIRAは現在、バグを追跡するための最もクールな共通プラットフォームです。




JIRA Java組織、Javaコミュニティ、およびOracleへの移行は何ですか？ 第一に、人生は単により便利で身近になります。 これは本当に一歩です！ 第二に、公開メーリングリストはわずかにオフロードされ、来年または2年で、メーリングリストからの多くのものがJBS JIRAに送られます。 そして、その結果、第三に、オラクル内のJavaプラットフォーム開発者と他の開発者との間の通信の数が増加し、最終製品の品質（Java SE 7更新、Java SE 8、Java SE 9および。など）
