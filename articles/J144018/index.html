2GIS API：8つの新しいマップおよび参照API機能

私たちはじっとしているわけではなく、APIを改善し続けていることをお知らせします。 前の公開の時点から、すでに8つの新機能があります。

それらを詳しく見てみましょう：



参照API



Githubで公開されたデモクライアントのソースコード


デモクライアントのソースコードにより、2GIS APIを使用する開発者は、どのように動作するかを確認し、ユースケースをシミュレートし、一般に、maps APIと2GISリファレンスの機能をよりよく学習できます。 実際、デモクライアントはYii PHP Frameworkで書かれた小さなWebアプリケーションです。 機能の詳細と使用手順は、ドキュメントに記載されています 。



フィードバックフォーム


ディレクトリAPIを使用するたびにフィードバックフォームを使用することをお勧めします。 ディレクトリまたはマップの不正確さに関する情報が、APIを使用するパートナーではなく、2GISに直接送信されるようにすると便利です。 したがって、フィードバックはすぐに適切な手に渡り、不必要な混乱は生じません。 また、コンテンツをより正確にするために、コンテンツをすばやく更新するのに役立ちます。



ここをクリックして、ドキュメントをよく理解し、実際のフォームをご覧ください。



タイムラインとランブラーボックスオフィス


APIをローカル企業との新しいレベルのユーザーインタラクションを提供するサービスと統合しました：サービスの記録と映画チケットの購入 数週間、新機能のアルファテストを実施します。 api@2gis.ruまでご連絡ください。



検索および広告アルゴリズムの改善


4月から5月上旬に、新しい要素を考慮して、関連性とランキングの向上を目的とした20以上の検索アルゴリズムの改善を実施しました。



これは、検索結果に表示される広告のコンテキストにもプラスの影響を与えました。 広告主の数は増え続けているので、検索をさらに改善し、効率化するよう努めています。



処理統計カウント


ユーザーが開始したデータのみが収集されるように、統計の処理の精度を改善しました。 5月以降、個人アカウントの多数のパートナーの検索クエリの数の値が減少する可能性があります。これは正常であり、これからは統計がより正確になるためです。



Map API



ジオクリッカー


これで、マップ上の任意のオブジェクトをクリックできます。 家をクリックすると、その住所とその他の情報が表示されます。 通り、地区、住宅団地、および都市にジオメトリが表示されます。 ストップをクリックすると、そのストップを通過するすべてのルートが表示されます。これは地図でも確認できます。



以前は、このような機会のために、ジオコーディングメソッドの面倒な統合を実行する必要があり、ルートマッピングを実装することはまったくできませんでした。 これは可能な限り簡単に実行できます 。ジオクリッカーを接続するには、コードを1行追加するだけです 。



Geoclickerは、ユーザーが向きを決めるときに非常に便利です。 たとえば、マップがオフィスへの道を示している場合、ユーザーは最寄りの停留所をクリックしてルートを表示し、その結果、そこに着く方法を理解できます。



近い将来、すべてのAPIパートナーに対してデフォルトでgeoclickerを有効にする予定です。

全画面モード


フルスクリーン- 新しいコントロールカード 。 また、1行のコードに含まれており、訪問者がマップを全画面に拡大するのに役立ちます。 これは、小さなマップフラグメントがサイトに配置されているが、方向付けに使用される場合に関連します。 その後、ユーザーはマップをブラウザウィンドウのサイズに拡大して、より快適に作業できます。



近い将来、地図が小さなスペースを占めるすべてのサイトに対して、デフォルトで全画面モードを有効にする予定です。

拡張ズーム制御


ズームコントロールが向上しました（146％向上！）。 今彼：

•小さなカードでは、それ自体が「崩壊」します。 サイズの縮小は、スケールが削除されたという事実によるものであり、スケールは非常に大きな割合のユーザーによって使用されていません。

•使用可能なズームレベルに応じて、スケールのサイズを動的に変更します。

•「+」および「-」は、最大または最小ズームレベルに達すると無効状態になります。 この状態でクリックしても、ズームは変更されず、コントロールの画像から理解できるようになりました。

バルーンにはヘッダーとフッターがあります。 これは、スクロールされない重要な情報をキャプチャするのに役立ちます。 メインコンテンツ領域では、スクロールとスクロール機能が保持されます。



これにより、興味深い設計ソリューションをリリースできます。
