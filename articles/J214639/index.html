ASP.Net MVCのオンラインコンパイラ

私たちはプロジェクトのニュースについて長い間書いていません。

以下は、.Net開発者が関心を持つ可能性のある新機能のリストです。

  • F＃（ feedl ）の完全なサポートを追加しました
  • NuGetパッケージのサポート
  • ウィジェット
  • Chrome拡張機能
  • 完全なASP.Net MVCを使用できるようになりました


そして、最初のイノベーションがあまり明確でない場合、残りはもっと興味深いかもしれません。



ヌジェ



NuGetパッケージのサポートを追加しました。 .Net 4.5用に設計された既存のNuGetパッケージを選択できます。また、パッケージの特定のバージョンを選択し、コードが異なるバージョンでどのように機能するかを確認することもできます。

ウィジェット



Webサイトに任意のコードを挿入して、訪問者がコードを見て、どのように機能するかを調べ、実行にかかる時間を表示できるようにすることができます。 3つの言語はすべてC＃、VB.Net、F＃で利用できます。

Chrome拡張機能



この拡張機能を使用すると、ページ上の任意のC＃\ VB.Net \ F＃コードを選択し、コンテキストメニューから[.NET Fiddleで編集]を選択できます。

そして、選択したコードをウィジェットに置き換えます。ウィジェットは、このページを離れることなく、すぐにこのコードを実行しようとします。

これは、MSDN \ StackOverflow \などでコードサンプルを確認する場合に便利です。

ASP.Net MVC



ASP.NET MVCのサポートが登場しました。 コントローラー、ビュー、モデルを作成して、何が起こるかすぐに確認できます。

コントローラ\ View \ Modelコードを記述して、すぐに応答でレンダリングするものを確認できます。

  • GET \ POST \などのリクエスト、検証の完全なサポートがあります
  • このページを別のブラウザタブで開き、動作を確認することもできます
  • Razorのみが現在サポートされていますが、WebFormsも間もなく利用可能になります
  • さらに、同じNuGetパッケージをASP.Net MVCと組み合わせて使用​​できます
  • コードの場所には、セクションとタブの2種類があります。
  • ブログのウィジェットでコードを使用することもできます。


使用例：

  • dotnetfiddle.net/CsMvcは、POST要求にAjaxを使用する実行可能なページの単なる例です
  • dotnetfiddle.net/ubbMHq-ベンダーの夢
  • dotnetfiddle.net/D64lkK-バブルソート
  • dotnetfiddle.net/W91Ycc-quizster.aws.af.cm/quizからコントローラーにロードされ、ページに表示されるテスト
