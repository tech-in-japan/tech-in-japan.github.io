Windows Web Server 2008 R2でのWebサイトの起動-ステップバイステップ

WebサイトをWindows Serverに展開したことがない人にとって、このプロセスは複雑で理解しにくいように見えるかもしれません。 したがって、この短い記事ガイドでは、インストールディスクからの読み込みからWordpressの展開までの完全なインストールおよび構成プロセスについて説明します。 このプロセスはシンプルで楽しいものであることを示します。



何が必要ですか？

  1. Windows Web Server 2008 R2配布キット（ 試用版は実験に適していますが、ライセンスキーを使用してアクティブ化できます）
  2. 64ビットプロセッサ（x64）、1 GBのメモリ、およびDVDドライブを搭載したコンピューター
  3. ISPが提供する専用の静的IPアドレス
  4. Webサーバーをインストールおよび構成するための1時間
これがすべて揃ったら、行こう！



手順1.インストールディスクからの起動



BIOSのディスクからブートをインストールし、ディスクを挿入し、「CDからブートするために任意のキーを押してください」というメッセージが表示された後、再起動後に任意のキーを押します。



ステップ2. Windows Web Server 2008 R2のインストール



ディスクからインストーラーをロードすると、次の画面が表示されます。







デフォルト設定のままにして、次の画面に進むことができます。







インストールを開始するボタンは1つだけです。 サーバーのインストールを開始するのはとても簡単です！



インストールする前に、Windows Web Server 2008 R2のバージョンを選択する必要があります。 フルバージョンとServer Coreバージョンが利用可能です。 [フル]を選択すると、グラフィカルインターフェイスがインストールされます。サーバーを初めてインストールするときにコンソールラインから管理する方法がわからない場合は、フルバージョンを選択する必要があります。



インストールプロセス中に、必要なファイルが解凍されてインストールされます。







その後、コンピューターは自動的に再起動します。再起動後、DVDドライブからCDを取り出すか、インストールされたシステムを起動するためにCDからBIOS起動をキャンセルします。



再起動後、Windows Serverインストーラーはライセンス契約への同意を提案し、その後インストールを完了します。



2回目の再起動後、システムは完全にインストールされ、Webサーバーを構成してWebサイトを起動するだけです。



ステップ3. Webサーバーの構成



Webサーバーが最初にロードされると、サーバーを管理するための一連のリンクを含む画面が表示されます。







ネットワーク設定


サーバーはプロバイダーから受信した専用の静的IPアドレスを必要とするため、おそらくプロバイダーが提供する指示（IPアドレス、ゲートウェイ、DNSサーバーアドレス）に従ってネットワーク設定を構成する必要があります。

上記のメインサーバー管理ツールでネットワーク設定を構成するには、[ネットワークの構成]リンクをクリックし、アクティブなネットワーク接続のアイコンを右クリックして、[プロパティ]メニュー項目を選択します。







次に、リストからインターネットプロトコルバージョン4（TCP / IP v4）を選択し、[プロパティ]ボタンをクリックします。







開いたウィンドウで、インターネットプロバイダーまたはローカルネットワーク管理者から報告されたすべてのパラメーターを指定します。



Webサーバーを起動するには、上記のサーバー管理ツール画面で[役割の追加]リンクをクリックし、開いたウィンドウで[Webサーバー（IIS）]を選択します。







その後、[次へ]を数回クリックしてWebサーバーをインストールします。



インストールが完了すると、Webサーバーが自動的に起動し、ブラウザーウィンドウに「 localhost」と入力するか、サーバーのIPアドレスを入力すると、デフォルトのサイトが表示されます。







メインメニューでWebサーバーを管理するには、インターネットインフォメーションサービス（IIS）マネージャーアプリケーションを起動する必要があります。 インターネットインフォメーションサービス（IIS）マネージャーの使用方法については、 learn.iis.netおよびwww.techdays.ruをご覧ください 。



これ以降、新しくインストールしたWebサーバーでHTMLページを既にホストできます。



ステップ4. Webサイトのインストール



動的サイトのないサーバーだけで、おそらく誰も必要としません。 したがって、アプリケーションをインストールすることをお勧めします。

このプロセスを容易にするために、特別なツールが作成されました。これはmicrosoft.com/webからダウンロードできるWeb Platform Installerです。



サイトを開くmicrosoft.com/webサーバーは強化されたセキュリティシステムで構成されているため、サイトのページでブロックされたアイテムに関するメッセージを含むウィンドウが表示されます。 これらのウィンドウが迷惑にならないようにするには、[続行]のチェックを外して[閉じる]をクリックします。







その後、Internet Explorer 8の設定で、[ツール]-> [インターネットオプション]ウィンドウを開き、[セキュリティ]タブを選択し、[信頼済みサイト]アイコンをクリックし、表示されるウィンドウで[サーバー検証を要求する]チェックボックスをオフにして、アドレスフィールドにdownload.microsoft.comと入力し、追加します。 ウィンドウを閉じて、ブラウザでの作業を続けます。



Web Platform Installerをダウンロードして実行します 。



Web Platform Installerを起動すると、すぐに一般的なWebアプリケーションのインストールを開始できます。 今度は、人気のあるWordpressブログエンジンにしましょう。



Wordpressをインストールするには、左側の[Webアプリケーション]タブをクリックし、リストを最後までスクロールして、wordpress名の横のチェックボックスを選択します。







[インストール]ボタンをクリックしてください。 Wordpressが動作するようにインストールおよび設定されるコンポーネントのリストが表示されます。







[同意する]をクリックします。 その後、MySQLデータベース管理者のパスワードを作成するように求められます。忘れないように、さらにインストールするときに必要になるため、どこかで考えて書き留めてください。



インストールが開始され、すべてのコンポーネントがダウンロードおよびインストールされます。その後、ツールはいくつかの追加の質問をします。

最初に、Webアプリケーションをインストールするサイトの名前。 すべてのデフォルト設定をそのまま使用できます。







次に、データベースを構成する必要があります：前の手順で作成した管理者パスワードを指定し、空のフィールドに入力します-これらは主にセキュリティに必要なキーフレーズとWebサイトを構成するデータベースユーザーのパスワードです-このデータも将来必要になる可能性がありますそのため、どこかに書き留めておいた方が良いでしょう。







すべての設定が完了したら、「Wordpressを起動」リンクをクリックして、ブラウザーでサイト自体のカスタマイズを続行できます。







ブログの名前を考え出し、メールアドレスを入力する必要があります。その後、パスワードが生成されます。もちろん、どこかにそれを書き留める必要があります-サイトに入って管理するために必要になります。







それで、サイトはインストールされ、あなたはそれで働くことができます！ ウェブサイトの設定がいかに簡単かをご覧ください。 ほんの数分。



ステップ5.ドメイン名のバインド



vasya-pupkin.ruなどのドメイン名をサイトにバインドするには、このドメイン名を、たとえばサイトnic.ruに登録し、レジストラのWebサイトに示されているようにドメイン名レジストラのDNSサーバーを構成して、これらのドメイン名サーバーのIPアドレスを参照しました。



その後、ドメイン名をサイトにバインドできます。 これを行うには、インターネットインフォメーションサービス（IIS）マネージャーツールで、サイト（デフォルトでは1つのデフォルトがあります）を選択し、右ペインの[バインディング...]リンクをクリックします。







開いたウィンドウで、このサーバーが応答するドメイン名を追加します。
