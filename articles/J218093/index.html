AudioCodes対Asteriskまたはより多くのグッズを持っている人

企業にIPテレフォニーを実装する場合、多くの企業は既存の通信インフラストラクチャを近代化する必要に直面しています。 そのようなタスクは、プロジェクトの1つに直面しました。AudioCodesとAsteriskは、ソリューションのアーキテクチャの顧客との激しい議論の中で真剣に取り組んでいました。



顧客は、テレフォニーの中核としてLync Server 2013を選択することを決定し、本格的な企業の音声通信を組織するには、Lync Enterprise Voiceを実装する必要がありました。 これは、PSTN（テレフォニー）との統合を担当するLync Server 2013の機能です。 または、簡単に言えば、市外通話、リダイレクト、転送、保留、拒否、リリース、一時停止など、組織外の電話番号への通話の発信と受信を可能にします。







顧客がなぜLyncを選択するのですか？



Lyncはユニファイドコミュニケーションであるためです。 そして、最も重要なことは、これはほとんどのお客様にとってユニファイドコミュニケーションを行う安価な方法です。 多くの場合、顧客はすでにEA / EAS契約を結んでおり、Lyncライセンスのコストは既に含まれているか、ハードウェアの競合他社と比較して安価です。



Lyncライセンスとクライアントライセンスは、EAの企業契約の下でこの顧客から購入しました。 現在、CAL Plusに拡張する予定です。これにより、本格的な音声通信がLyncで機能します。



そしてもう1つ、Lyncはソフトウェアです。 したがって、その保守、更新、および管理は、同様の機能を備えたハードウェアソリューションよりも少ない費用で済みます。 Lyncは、安定した使い慣れた既存のインフラストラクチャに展開されます。

そして最後に、顧客（および他のほとんどの顧客）のITインフラストラクチャの約80％はMSプラットフォームであり、Lyncは問題なくそれらと統合します（AD、Exchange、SharePoint、Dynamics CRMなど）



PSTNとの統合のタスク



企業がなぜLyncを電話ネットワークに接続する必要があるのですか？ まず、ユニファイドコミュニケーションプラットフォームが機能する必要があります。 これは、Lyncクライアントを介して参加者を接続する音声会議です。 これは、エンドデバイスの標準化と節約です（アナログ電話をオプトアウトできます）。 これは従業員の機動性です。音声ルーティングが正しく構成されていれば、Lyncモバイルクライアントを介して外部（都市間など）に電話をかけ、通話を節約できます。



多くの場合、VoIPエンタープライズでテレフォニー開発の次のステップを実装した企業では、これはユニファイドコミュニケーションシステム（Lyncサーバー）への移行です



Microsoft Lyncを導入する前、お客様は次の問題を解決することに戸惑っていました。



1.既存のE1 / FXOインターフェイスを使用してPSTNネットワークにMicrosoft Lyncをドッキングする

2. SIPトランクを使用してMicrosoft LyncをPSTNネットワークに接続する

3.既存のPBXを保存し、Microsoft Lyncと連携してその動作を保証する

4. Microsoft Lyncとリモートオフィス、および多数のコーデックとインターフェイスを持つさまざまなメーカーのPBXとの統合

5.このようにして、既存のテレフォニーシステムからMicrosoft Unified Communicationsへのスムーズな移行



アスタリスクまたはAudioCodes？



IP PBXのソリューションとして、アスタリスクは顧客であり、無料で配布されている本格的なオープンソースIP-PBX（IP PBX）でした。 これは完全に機能するソリューションですが、Lyncと統合すると多くのリスクが伴います。



私たちが提案した代替手段は、ビジネスグレードのVoIPソリューションであるAudioCodesです。 Microsoft LyncのAudioCodes機器は、Microsoft Unified Communicationsインフラストラクチャでの使用がMicrosoftによって認定されており、世界中およびロシアの多くの企業で使用されています。



AudioCodes製品には、音声ゲートウェイ、マルチサービスゲートウェイ、セッションボーダーコントローラー（SBC）、加入者ゲートウェイ、IP電話、メディアサーバー、高度なアプリケーションなどのVoIP製品が含まれます。 AudioCodes機器は、世界中のブロードバンド、モバイル、固定回線サービスのプロバイダー、および多くの企業のネットワークで使用されています。 AudioCodes機器は、音声コーディングと音声処理に独自のDSPテクノロジー「VoIPerfectHD」を使用し、音声通信に比類のない音質を提供します。

最後にグッズについて



AudioCodesはより多くの互換性オプションを提供します。 はい、AudioCodesがなくてもすべてが機能します。 しかし！



•すべてがUCモードで機能し、テレフォニーモードでは機能しないようになりました（PSTNのようなインターフェイスはありません）。 誰も通話を転送したり、FAXを接続したり、転送を転送したり、通話を保留にしたりしません。 AudioCodesを使用すると、これがすべて可能になります。

•原則として、発信コールのみが発生します。 また、着信コールでは、LyncはCPV（コール制御）を失いません。これらは、加入者に電話をかけたときに聞こえるビープ音です。

•AudioCodesを使用すると、LDAPを介してADと統合できます。 たとえば、発信者の名前を変更するか、ALUとLyncを同時に呼び出して移行をスムーズにします。

•HD音声（ブロードバンドサウンド）-テレフォニーで使用されるオーディオテクノロジ。 電話回線の音の周波数範囲を拡張し、音声品質を改善します。

•さらに、アスタリスクの場合はベンダーからのサポートが不足しています。

顧客は何をしましたか



マルチユーザー会議を実装するには、ユーザーはStandard CAL + Enterprise CALライセンスのシートが必要です。 また、パブリックネットワークへのアクセスが必要なユーザーには、ユーザーごとにStandard CAL + Enterprise CAL + Plus CALライセンスを持つクライアントプレースが必要です。



このソリューションでは、Lync Server 2013ユニファイドコミュニケーションサービスを使用し、メインサービスのフォールトトレランスには、会社の大規模オフィスのAudioCodesゲートウェイにインストールされたSBA（Survivable Branch Appliance）機能を使用します。 これは、メインサーバーに障害が発生した場合、およびデータネットワークに障害が発生した場合に、顧客が作業できることを意味します。 アナログおよびデジタルインターフェイスを備えたAudioCodesを使用して、パブリックネットワークに接続します。



技術的には、実装の主な段階を特定できます。



•機器のセットアップ。

•トポロジビルダーを使用して作成されたトポロジ図の各コンポーネントのハードウェアインストール。必要なすべてのコンピューター（Lync Server 2013を実行するサーバー、データベースサーバー）、ネットワークアダプター、ハードウェアバランサー、ストレージデバイス（ファイルサーバー）を含みます。

•DNSレコードを構成します。 Lync Server 2013はDNSを使用してレコードを登録および維持し、ドメイン名をIPアドレスに正しく解決します。

•ゲートウェイおよびLync Serverのパブリックネットワークへのルーティングを接続および構成します。

•ユーザーのパイロットプールをシステムに接続し、システムの基本機能をテストし、クライアントの自動セットアップと構成の技術手順をテストします。

•システムユーザーのメインプールを接続し、ワークロードの下でシステムをテストします。
