Ubuntu 11.04でSIS M671 / M672ファミリーのビデオカード用のビデオドライバーをインストールする

翻訳者から、つまり 私：

ASUS K50Cでビデオカードを正しく動作させるために多くの時間を費やしました。 検索エンジンによって検出されたすべてのソリューションは、Natty Narwhalのリリース（1.10）よりも前のリリース（1.8）のX-Server'aで計算されました。 最後に、外国のブログの1つに投稿を見つけました。 そして、その解決策はうまくいきました！

私の翻訳が誰かがもっと役に立つものに時間を費やすのに役立つことを願っています。



そのため、Ubuntu 11.04でSIS M671 / M672ファミリーのビデオカード用のビデオドライバーを配置する方法





ご存知のように、X-Serverの新しいリリースは数日前に発表され、Ubuntu 11.04に既に含まれています。

そのため、このドライバーはUbuntu 11.04で正常に動作し、チェックされています！



インストールは、以前のオプションとは多少異なります。 古いドライバーを/ usr / lib / xorg / modules / drivers /フォルダーにコピーしてxorg.confを構成する必要がありましたが、今では自分でビルドして構成する必要があります。 肯定的な点はありますが、xorg.confを使用すれば、もうわずらわしいことはありません。

組み立てとインストールの方法を知っており、これに必要なものはすべて揃っていると思います。



必要なもののリストは次のとおりです。

-git

-xorg-dev

-mesa-common-dev

-libdrm-dev

-libtool



他のパッケージの名前を忘れてしまったので、思い出してください。残りの-devパッケージを追加します。



ソース（ドライバー、およその翻訳）は既に私によってパッチが適用されており、パッチを適用する必要はありません。 パッチフォルダーは、将来ドライバーの関連性をサポートしたいすべての人が利用できます。



それでは、ターミナルを開いてビルドを開始しましょう。



$ git clone git://github.com/hellnest/xf86-video-sismedia-0.9.1.git

$ cd xf86-video-sismedia-0.9.1

$ ./configure --prefix=/usr --disable-static

$ make

$ sudo make install









再起動すれば完了です！



1366x768の解像度を持つユーザー向けの短いコメント：

これらの行をxorg.confに追加する必要があります（もちろん、まだ存在しない場合）。



Section "Device"

Identifier "Configured Video Device"

Option "UseTiming1366" "yes"

EndSection



Section "Monitor"

Identifier "Configured Monitor"

EndSection



Section "Screen"

Identifier "Default Screen"

Monitor "Configured Monitor"

Device "Configured Video Device"

EndSection







OK、それだけです。 このリリースをお楽しみください（ドライバー、約Transl。）。
