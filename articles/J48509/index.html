Gnome Power Manager：s2diskによる休止状態

最近、Ubuntuのこのような便利な機能を休止状態として利用したかったのですが、切望された休止状態の代わりに、しっかりと中断されたシステムしか取得できませんでした。 私がそれを理解しようとしたとき、私は問題が唯一のものとはほど遠く、「予見可能な将来に修正される」ことだけを見つけることができました。 短いグーグルでフォーラムを読んだ後、uswsuspパッケージのs2diskアプリケーションを利用して、可能な限り透過的に使用することが決定されました...



まず、swapセクションが必要です（swapファイルを使用した実験中に、目的の結果を得ることができませんでした）。

そのようなセクションがあるかどうか、そしてそれを探す場所を見つけるために、これを行うことができます：

swapon -s


出力には次のようなものが表示されます。

/ dev / sda2パーティション10482404 181280-


スワップパーティションが正常に検出されたら、インストールを続行します。

sudo apt-get install uswsusp


実践が示しているように、uswsuspは環境を素晴らしく定義していますが、設定が正しいことを確認しておくと便利です。 これを行うには、お気に入りのエディターとしてroot権限で/etc/uswsusp.conf構成を開き、注意深く見てください：
  • 「デバイスの再開」パラメータは、スワップセクションを指している必要があります（最初に見た）
  • パラメータ「shutdown method」、画像の記録後に実行されるアクションを設定します。デフォルトでは「platform」


実際にはそこに設定することはあまりありません。すべてのデフォルト設定は必要に応じて設定されます。 次に、パフォーマンスを確認します。

sudo s2disk


驚きが発生せず、コンピューターが正常に「スリープ」状態になった場合、次のステップはスキップします。 スナップショットを撮った後、再起動する必要がありました。 マナを読んだ結果、予約されていない値を設定すると、電源がオフになることがわかりました。 これは、たとえば/etc/uswsusp.conf構成で任意の値を指定することで行います。

シャットダウン方法=電源オフ


それでは、最も重要なことに取りかかりましょう-組み込みの休止機能の代わりに、Gnome Power Managerが既に構成されたs2diskを使用するようにします。 これを行うには、 / usr / lib / hal / scripts / hal-system-power-hibernateにあるスクリプトを編集して、次の形式にする必要があります。

＃！/ bin / sh

s2disk


これで、すべての設備を備えたコンピューターの電源を切るウィンドウの「スリープ」ボタンを押すと、ペットは静かに「眠りに落ちる」ようになります。



PSコメントを考慮しましょう-上記のすべては1526MBのRAMを搭載したワークステーションで行われました。 スワップパーティションのサイズは最初は310MB、次に10GBでした
