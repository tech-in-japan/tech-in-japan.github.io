すべてのNoSQL DBMSは、いくつかのカテゴリに分類されます。

  • キーバリューストア
  • 列ファミリー（Bigtable）ストア/スケーラブルな分散ストレージ
  • グラフストア/グラフDBMS
  • ドキュメントストア/ドキュメント指向データベース管理システム




以下の図は、これらのNoSQLビューで使用されるデータの量とこのデータの複雑さを概略的に示しています。









各セクションでは、機能が増加する順にDBMSを配置しようとしました。 おそらくそれはやや主観的であることが判明した。

キーバリューストア



キーバリューストアは、NoSQLソリューションがSQLに対する優位性を示す方向です。

そして多くの人は、この方向が短期的および長期的に最も要求されると考えています。

たとえば 、オープンMySQL DBMSのオリジナルバージョンの作成者であるMichael Wideniusはそう考えています。

Key-Value NoSQLは非常に人気があり、その数が多く競争が激しいために、迅速かつ順調に開発されています。 この記事の執筆中に調査されたNoSQLデータベースの最大数は、キーバリューストアに関連していました。



PHPのキーと値のストレージに関するハブに関する記事がありますが、すべてに同意するものではありません。 そこに提示されているリポジトリ（Voldemort、Scalalaris、MemcacheDB、ThruDB、CouchDB）の一般的な選択は、この記事の発行からほぼ5年が経過した後、あまり関係がないように思われました。 また、CouchDBは、キーと値のストレージはまったくなく、ドキュメント指向のDBMSについて説明しています （ ドキュメント指向のDBMSに関するセクションを参照）。





Memcachedb


説明 ：同じmemcachedで、BerkeleyDBの形式の背景のみ。

パフォーマンス ：開発者は、1つのストリームの平均パフォーマンスが18868 w / s（1秒あたりの書き込み操作）および44444 r / s（1秒あたりの読み取り操作）であるテスト結果を提示しました。 Dell 2950IIIサーバーでテストしました。これは、最も弱い構成であってもまともなデバイスです。

インストール ：すべてがソースから収集されます。 PHPでは、PECLの通常のMemcachedを使用します。

ライセンス ： BSDライクライセンス -商用および非商用プロジェクトで無料で使用できます。





レディス


説明 ： ブラックジャックのベンチマークとリンクを含むハブに関する紹介記事があります。 トランザクション（ それらに関するhabrostaty ）とレプリケーションがあります。 アプローチバージョン3.0では、Redis-Clusterが表示され、パフォーマンスが大幅に向上します。 素敵なインタラクティブなチュートリアルがあります。

生産性 ：〜110.000 w / s、平均鉄で〜81.000 r / s

インストール ：Redis自体とPHPのクライアントは、ソースからビルドすることをお勧めします。 多くのクライアント（ list ）がありますが、既存のRedis機能のすべて（またはほとんどすべて）の説明とサポートを得るには、phpredisを自分でお勧めします。

ライセンス ： BSDライセンス -すべては無料ですが、何かが壊れても、開発者への苦情はありません。





タランツール


説明 ：インメモリストレージ。 開発者によると、Redisは、すべてのデータがメモリ内にあるという事実により、速度が向上するという点で反対です。 組み込みのキューメカニズムがあります。 主な機能について説明した良い記事があります。

インストール ：Ubuntuでは、apt-getと少しの魔法（ 公式ページ ）を使用してインストールされ、PHPのクライアントはソース（ github ）からアセンブルされます。

パフォーマンス ：Redisのレベルでは、テスト結果は矛盾します。Tarantoolは、開発者のRedisよりも高速です。Tarantoolは 、一般の人のRedisのレベルで は高速です。

ライセンス ： 簡易BSD-すべて無料。





リアク


説明 ：フォールトトレランスと配布に重点を置いたデータベース。 この重点は非常に強力であるため、開発会社は、Riakの機能を評価できるように、少なくとも5台のサーバーをRiakに割り当てることを推奨しています。 一見、これはキーと値のストレージですが、すべてのフィールド、セカンダリキー、MapReduceの検索が含まれています。 トランザクションなし。 詳細および詳細なhabrostatya 。

インストール ：Debian / Ubuntuのパッケージからインストールするまでのさまざまな方法 。 PHPには、PECLパッケージと公式のPHP-clientがあります 。

生産性 ：最も重要な場所ではありませんが、1秒間に2,500回の操作が行われています 。

ライセンス ： Apache 2ライセンスは普通の人には無料ですが、商用利用の場合、Riak Enterpriseの1コピーの価格は年間2,800ドルからです。





エアロスパイク


説明 ：最小限のレイテンシで大量のデータを保存できるスケーラブルなストレージ。 デフォルトでは、ACIDサポートには個別のページがあります 。 バージョン3では、セカンダリインデックスが導入されました。 スケーリング、レプリケーション、クラスタリング（ リンク ）のための印象的な量の独自技術。 私自身は、このシステムを強力な産業用Memcachedとして思い出しました。

インストール ：Aerospikeは配布キットからインストールされます。PHPの公式クライアントはAerospike2にのみ存在し、ソースから組み立てられます。

パフォーマンス ：マイクロ秒単位の遅延（ ソース ）で、1秒あたり180,000から400,000操作の速度を主張しました。

ライセンス ：

  • コミュニティエディション -制限付きの無料バージョン：各200GBのデータを含む最大2台のサーバー。
  • Enterprise Edition -30日間トライアル、制限なし。 噂によると 、コストはデータセンターあたり50,000ドルからです。






Foundationdb


説明 ：包括的で最も簡単なインストールおよび構成ソリューションとして位置付けられています。 簡単な拡張性、簡単な管理がキャッチするキーワードです。 ユーザーには「妥協のないACIDトランザクション」が提供されます。 さまざまなデータモデルを使用する機能-キー/値、ドキュメント、さらにはSQL。 このDBMSは、そのパフォーマンスについて読んだとき、特に興味深いように思えました。

生産性 ： 3,750,000 r / s * 。 * RAM（キャッシュ）からのランダムエントリの読み取り。 パフォーマンスセクションの公式Webサイトには、多くの興味深いテストがあります。「最も遅い」テストでは、1秒あたり〜235,000操作（50/50の読み取りおよび書き込み操作）の結果が示されています。 読み取り遅延は2ミリ秒未満、コミット遅延は15ミリ秒未満です。 結果は、それぞれ16Gb RAM、2x200Gb SSDを備えた24台のマシンのクラスターで得られました。テストベースは200万のキーバリューレコードで構成され、すべての操作はトランザクションであり、最大の分離とトリプルレプリケーションを備えていました。

インストール ：すべてが簡単です：UbuntuのDEBパッケージ、PHPのPEARパッケージ。

ライセンス ：

  • コミュニティライセンス -無料使用。 開発とテストの際に制限はありませんが、実稼働環境で最大6つの実行プロセス、つまり 6つのサーバーで1つのプロセス、3つで2つのサーバーなど。
  • エンタープライズライセンス -サポートの質に応じて、99ドルから199ドルまで無制限。




PHPのサポートが不足しているため、いくつかの興味深いプロジェクトはこのリストに含まれていません。 また、 ヴォルデモート 、 スカラリス 、 ThruDBのプロジェクトは含まれていません。 パフォーマンスの低下やドキュメントの不足、そして2009年以降、何も改善されていないという事実によります。







列ファミリー（Bigtable）ストア/スケーラブルな分散ストレージ



このセクションで紹介するリポジトリは、主に元のGoogle Bigtableの設計に基づいて設計されています。

これらのNoSQLの主な機能は、テラバイト単位で測定されるデータを扱うことです。

ここでは、インスタントアクセスの速度はそれほど重要ではありません。配信、フォールトトレランス、および膨大な量の情報を処理する能力に重点が置かれています。





Hbase


説明 ：ApacheのオリジナルのGoogle Bigtableデザインに基づくオープンソース開発。 Hadoopプロジェクトの一部として開発されました。 Facebook自体がメッセージングサービスの基礎として使用します。 HBaseは、インデックス付きフィールドを1つ選択します。 ACIDには部分的なサポートがありますが 、 トランザクション性が存在するようですが 、最も明白な方法ではサポートされていません。

インストール ：Thriftと呼ばれる魔法の薬を使用してインストールされます 。インストールおよび使用プロセスは、 この生息地で詳しく説明されています 。

パフォーマンス ：異常なパフォーマンス測定手法を使用したフィールドテスト ：7サーバー（16Gb RAM、8xコアCPU、HDD）のクラスターで、30億レコードのテーブルで操作が実行されました。 読み取り/書き込みの300プロセスが同時に開始され、操作に費やされた時間が測定されました。 その結果、平均録音時間は10ミリ秒 、読み取り値は18ミリ秒でした 。

ライセンス ： Apache License 2.0-あらゆる目的で無料で使用できます。





ハイパーテーブル


説明 ：HBaseに似た興味深い開発。 少しパフォーマンスが高く、HQLクエリ構文にはるかに精通しています。 リクエストの例： 

select * from QueryLogByUserID where row =^ '003269359' AND "2008-11-13 05:00:00" <= TIMESTAMP < "2008-11-13 06:00:00"


トランザクションはありません。これは、公式Webサイトのドキュメントの最初の行に明記されています。

インストール ：Thriftと公式のThriftClient（ github ）を使用してPHPに接続します。

パフォーマンス ：公式ウェブサイトのいくつかのグラフ 。 上記のように、パフォーマンスはHBaseに似ています。

ライセンス ： GNU General Public Licenseバージョン3。-任意の目的で無料で使用できます。 追加料金で24時間サポートを利用できます





カサンドラ


説明 ：もともとFacebookで開発された分散ストレージは、その後Apacheに転送されました。 上記とは異なり、Cassandraは分散型分散ハッシュテーブル（DHT）であり、AmazonのDynamoに基づいています。 SQLに非常によく似たCQLクエリ言語があり、いくつかの制限があります。 複数の列でサンプリングを使用してクエリを作成し、セカンダリインデックスを追加することができます。 バージョン2.0では、「比較と交換」の原理で動作する「トランザクション」が登場しました。



トランザクションリクエストの構文は次のようになります。

  • レコードを追加 
     INSERT INTO users (login, email, name, login_count) values ('jbellis', 'jbellis@datastax.com', 'Jonathan Ellis', 1) IF NOT EXISTS
  • レコード更新 
     UPDATE users SET reset_token = null, password = 'newpassword' WHERE login = 'jbellis' IF reset_token = 'some-generated-reset-token'




インストール ：PHPとCassandraの相互作用を確立する方法がいくつかあります（同じTrift、Cassandra-PHP-Client-Library、 cassandra-pdo ）。 最後の選択肢は私にとって最も楽しいように思えました。

パフォーマンス ：グラフとの良好な比較テスト。その結果によると、50/50の読み取り/書き込み操作の比率を持つ8台のサーバーで、Cassandraは毎秒約9,000操作を実行します。 HBaseは同じ条件下で約2,500を製造します。

ライセンス ： Apache License 2.0-あらゆる目的で無料で使用できます。



Stratosphere 、 HPCC 、 Cloudera 、 Cloudataなど、他のBigTableソリューションがあります。 たとえば、PHPサポートの欠如、普及率の低さ、ドキュメントの質の低さなど、さまざまな理由で詳細に考慮されていません。

Neo4j


説明 ：グラフDBMSの分野で最も成功し、人気のある開発。 ACIDを完全にサポートしています。 インストールは簡単で、簡単に拡張できます。 彼女はすでに開発されたコミュニティを形成しています。発生するほとんどの質問については、すぐに答えを見つけることができます。 この記事では 、PHPとともにその機能について読むことができます。

インストール ：リポジトリから配置し、 Neo4jPHPクライアントをPHPに使用します

パフォーマンス ：仕様により、特定の読み取り/書き込み速度インジケーターを表示するのは奇妙に思えました。 複雑な関連データを選択でき、リレーショナルDBMSよりも何倍も高速に実行できます。

ライセンス ：

  • Community Edition -GPLライセンスのオープンソース、無料使用
  • 商用サブスクリプション -高性能キャッシュ、水平スケーリングの高度な機能、サポート、およびその他の利点があります。 コストの範囲は、0ドル（年間プロジェクトターンオーバーが100,000ドル未満の3人のスタートアップの場合）から無限（非常に大企業の場合）までです。






このセクションでは、DBMSを1つだけ説明しました。最も興味深いライバルであるOrientDBは以下にあります。 結局のところ、Web用、特にPHP用の非常に多くのグラフィカルDBMSがあります。

また、HBase、BerkleyDB、またはCassandraをバックエンドとして使用するTitanもあります。 彼の奇跡に関する多くの情報はありません; PHPと友達になる方法に関する情報はさらに少ないです。

Thirtベースのクライアントを使用してphpに接続できるTwitterのFlockDBを思い出してください 。 しかし、このDBMSについての情報量が少ないため、完全で客観的な意見を定式化することは困難です。







ドキュメントストア/ドキュメント指向ストレージ



このセクションでは、ドキュメント指向のストレージ（階層データ構造用のDBMS）について検討します。 これらのストレージは普遍的です。読み取り/書き込み速度が高く、格納データのフォーマットに柔軟に対応し、非構造化データを簡単に操作し、十分なスケーリングの機会を提供します。





モンゴッド


説明 ：おそらく最も人気のあるドキュメント指向のNoSQL DBMS。 データはJSON / BSON形式で保存されます。 優れたスケーリング、レプリケーション、インデックス、Map-Reduce。 トランザクションは、比較とスワップとして表示されます。

インストール ：リポジトリのMongoDB、PECLのphp-client。

パフォーマンス ： 比較テストがわずかに高く、MongoDBで結果がありました。

ライセンス ： GNU AGPL-オープンソース、無料使用。





Couchdb


説明 ：Apacheからの開発。 MongoDBにほぼ似ています。 これは、読み取り操作中のブロッキングがないこと、および構成がより困難なシャーディングテクノロジーによって区別されます。

インストール ：リポジトリからのCouchDB。phpクライアントには、いくつかのオプションがあります（PHPillow、PHP Object Freezer、PHP-on-Couch、PECLの拡張機能）。

パフォーマンス ： 1つのテストの結果によると、MongoDBよりも明らかに遅い

ライセンス ： Apache 2.0-無料で使用。



この分野にはさらに多くの開発がありますが、私には非常に単調に思えました。 おそらく、私はそれらを十分に深く研究していませんでした。





オリエント


説明 ：ドキュメント指向であると同時にグラフィカルなDBMS。



文書指向の競合として最も近いのはMongoDBです。 別のページがこの比較専用です。

OrientDBの主な利点：

  • ACIDの完全サポート
  • 文書で外部キーを使用する機能（リレーショナルDBMSと同じ）
  • MongoDBで使用される3種類のインデックス（SBツリー、ハッシュ、MVRBツリー）とBツリー
  • 高いパフォーマンス（OrientDBは通常のハードウェアで150,000 w / sを実行します）
  • SQLに似た単純なクエリ言語


それとは別に、クエリ言語に注意し、同一の更新要求がどのように見えるかを比較します。

  • モンゴッド 
     db.product.update( { “stock.qty”: { $gt: 2 } }, { $set: { price: 9.99 } } )
  • オリエント 
     UPDATE product SET price = 9.99 WHERE stock.qty > 2


グラフとしての主な競合相手はNeo4jです。 そして、OrientDBでのグラフ機能の習得は、Neo4jよりもはるかに難しいと言わなければなりません。 これに関する最初のアイデアは、 この記事で入手できます。

インストール ： インストールについて少し魔法をかける必要があります。 これは非常に実用的なマニュアルです。 このライブラリはPHPクライアントとして推奨されます。

パフォーマンス ： 150.000 w / sを約束し、グラフDBMSの比較もあります

ライセンス ：

  • Community Edition -Apache 2ライセンスのオープンソース、商用を含むあらゆる目的での無料使用
  • Enterprise Edition-拡張サポートと、クエリプロファイラー、メトリックス記録、最初のサーバーで£1,000、後続の各サーバーで£500の構成可能なアラートを備えたライブモニターなどの利点。 スタートアップにとっては、価格の半分。
