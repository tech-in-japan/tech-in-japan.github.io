マルチプラットフォームの真の画像開発

この夏、AcronisがTrue Image for Macをリリースしたことをご存知かもしれません。 私たちの企業ブログには、このテーマに関するいくつかの出版物（ 1、2 ）がありましたが、今日は開発のマルチプラットフォーム機能についてお話したいと思います。



Mac用の製品を設計する際、主に既存のTrue Imageユーザーに焦点を合わせました。 以前にWindowsでTrue Imageを使用していた人がMacのバージョンをすばやく把握できるように、2つのアプリケーションのインターフェイスを可能な限り類似させることが決定されました。 同時に、オペレーティングシステムの機能とTrue Image自体の機能セットの違いを考慮する必要がありました（結局、長年にわたって成長してきた古い製品のすべての機能を物理的に実現することができませんでした）。 当時幸運なことに、Acronis True Image 2015 for Windowsもインターフェースを更新していたため、ユーザビリティエンジニアは両方のプラットフォーム用の新しいインターフェースを一度に設計しました。





統一されたインターフェース



そもそも、Macアプリケーションは、兄からかなりまともな機能セットを受け取りました：ディスクバックアップと復元、スケジュールバックアップ、増分バックアップ、暗号化、リカバリ用ブータブルメディアの作成、ファイル復元、Acronis Cloudのサポート。 したがって、レイアウトを作成する際、Windowsバージョンではすべてが著しく複雑になることを考慮しました。ユーティリティ、同期、ノンストップバックアップ、その他多くの恐ろしい言葉もあります。



2つのインターフェイスを並べて配置すると、非常によく似ていることがわかります。左側にはバックアップのタスクのリストがあり、中央にはバックアップのソースと場所とのペアがあり、スケジューラは左下隅にあり、右下の秘蔵ボタン「pysch」です 同時に、Mac用の個別に作成されたアプリケーションは、ネイティブで自給自足のように見えます。Windowsのバージョンがわからない場合、同じインターフェイスをタイル化してフラットにできるとは言えません。









均一な動作



アーキテクチャレベルでは、PCコンピューターとMacコンピューターは異なる種類の動物であるため、アプリケーションは非常に異なります。 はい、PCは現在UEFIとGPTに大規模に移行していますが、それらの違いは依然として大きなものです。 たとえば、PCにはFusion DriveやBootcampなどがなく、パーティションレベルでのディスク自体のパーティション分割はOS XとWindowsで異なります。 幸いなことに、アップルテクノロジーのほとんどのユーザーは生きており、これらすべての詳細について知らない。



当社製品の主な機能-コンピューター上のすべてのデータの簡単なバックアップ-Windowsでは「Entire PC」、Macでは「My Mac」と呼ばれます。 使い方は簡単です。アプリケーションを起動した人が最初に目にするのは、完全バックアップを作成することです。 ユーザーの観点から見ると、この場合の製品の動作は同じです：True Imageは、すべてのシステム機能を備えたコンピューターのバックアップコピーを作成し、必要に応じて、数回クリックするだけで復元できるようにします。 また、必要なバージョンから回復できるようにするために、Windowsバージョンと同様のポピーバージョンのスケジューラを実装しました。









Macには古いバージョンの増分バックアップと自動削除もありますが、Windowsバージョンとは異なり、この機能は内部に隠されており、ユーザーはアクセスできません。



アクロニスクラウド



個人が使用するプラットフォームに関係なく、私たちは彼のデータに対して二重の保護を提供できなければなりません。 そのため、クラウドへのバックアップが製品の主な機能の1つになっています。



両方のアプリケーションは、Acronisクラウドストレージサービスで動作します。 使用するには、ログインするか、プログラムインターフェースで新しいアカウントを直接作成します。 Webでは、ユーザーのアカウントの下に、両方の製品で作成されたバックアップが表示されます。









暗号化



暗号化は、Acronis True Imageを想像できなかった別の機能になりました。 PCとMacの両方のバージョンはAES 256をサポートしています。アーカイブのパスワード保護も両方のバージョンで実装されています。









レスキューディスク



2つの製品の顕著な違いの1つは、ブータブルメディアを作成する原則です。WindowsではLinux（またはユーザーの要求によりWinPE）に基づいており、MacではOS Xに基づいています。作成されたブートディスクまたはフラッシュドライブの機能もわずかに異なります： Windowsバージョンでは、バックアップの作成と復元の両方が可能です。「アップル」バージョンは、リカバリのみを目的としています。 Macユーザーが再起動を好まないという事実と、機能が必要でない場合（ライブシステムでのバックアップも正常に機能する場合）に、このような実装に至りました。



Fusion Drive、FileVaultおよびAll-All-All



True Image for Macのもう1つの重要な違いは、独自のMacテクノロジーのサポートです。これは、最新のAppleコンピューターでの本格的な作業に必要です。 True Image for Windowsは、Fusion Drive、FileVault 2、Bootcampに出くわすことはありません。 特に回復に関しては、下位レベルでの彼らの支援は当たり前の必需品です。 True Image for Macは、これらすべてで機能します。 私たちにとって、それは私たちがうまく対処した挑戦でした。



一般的なデフォルト設定



決定を下すことでユーザーにできるだけ負荷をかけないようにするために、私たちはユーザーに可能なすべてを選択します。 Acronis True Imageの両方のバージョンで似ています。デフォルトでは、WindowsであろうとMacであろうと、コンピュータ全体を毎週バックアップすることを提供しています。 ユーザーは、選択したとおりにすべてを選択したことを確認する必要があります。



エピローグの代わりに



アクロニスは、製品の信頼性と使いやすさの向上に精力的に取り組んでいます。 常に成功するとは限りませんが、常に取り組んでいます。 完璧なバックアップアプリケーションを作成できると考えています。
