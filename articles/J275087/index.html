DevConの秘密＃2。 会議グリッドの形成方法





従来の開発者会議であるDevConがどのように作成されているかについて、引き続き話し合います。 前の記事で、過去数年間の参加者からのフィードバックを考慮して、会議の形式がどのように更新されるかについての話を共有しました。 今日は、会議グリッドがどのように形成されるかについて話します-もちろん、会議の内容を含むさまざまな並行活動を考慮した、1時間ごと、さらには1分ごとのイベントスケジュールです。



より正確には、このトピックは次の2つの部分で構成されるため、説明を始めます。

  1. 主に組織の問題に焦点を当てる最初の部分は、あなたが今読んでいるものです。
  2. 会議プログラムについての第2部は、春にリリースされます。






制限から始める



重要なフレームワーク



新しいDevConの計画時に決定したロジスティクスタスクの1つは、会議をモスクワに近づけることでした。 私たちはまだ町からそれを持ち出したかったのですが、参加者が車でそこに着くのがそれほど難しくなく、最寄りの地下鉄からの輸送時間はできるだけ短くしました。





前の会場までのルート：60 km以上、1.5時間以上（Yandex.Maps）



比較のために：過去数年間、バスは地下鉄から会場まで約1時間半移動しました。交通渋滞があった場合は、2つから2つ半すべてでした。 その結果、会議の従来の開始は午前10時にではなく、11時になり、すべての参加者を時間内に配信するために、午前7時にバスの送信を開始しました。 この場合、通常は余裕を持って時間を設定します。たとえば、この場合、「輸送時間帯」は4時間です。





新しい会場へのルート：15 km以上、30分以上。 （Yandex.Maps）



新しいDevConは、Klyazma貯水池のKlyazma寄宿舎（水、木、リス）で開催されます。 新しい場所には利点があり、制限もあります。



イベントのプログラムの観点から、一方向の距離と輸送時間を節約することが不可欠です。 今では、参加者にもっと睡眠をとる機会を与えるだけでなく（またはモスクワからではない参加者にとって重要なフライト/乗り換えのためのより大きなバッファ）、会議を早めに開始することもできます。







1日目または2日目の終わりに出発する場合も、同様の時間の「節約」が得られます。後で出発することができ、移動時間が短くなります。



なぜ輸送から話を始めたのですか？ イベントの内部グリッドを形成する前に、どのような制限があるかを理解することが重要だからです。 会議の開始時間と終了時間は、適合する必要があるフレームワークです。



空間と時間の幾何学



2番目の重要な点は、搭乗所の領域とさまざまなゾーンの場所によって制限が課されることです。





下宿でのさまざまなアクティビティの潜在的なゾーン（Yandex.Maps）



イベントプログラムを設計する際には、次のようなさまざまなパラメータを理解することが重要です。

  • 異なるゾーン間でゆっくりと（幸いなことに、大きな領土、小道、および「密林」が）移行するのにどれくらい時間がかかりますか？ たとえば、私は本館にいて（1）展示エリアの桟橋に行きたい（2）後、次のレポートに戻る時間がある（1）。 もちろん、地図上の定規で測定することもできますが、足でストップウォッチを持って歩くことが重要です。
  • 休憩にはどれくらい時間がかかりますか？ 参加者は、話す、展示会を見る、別のゾーンに行く、スナック、飲み物を飲む、またはcor宴をする、最寄りのトイレに行く（一度もすべてではない）時間が必要です（これも重要です！）。 ここでも、領域のジオメトリを理解する必要があります。
  • 朝食、昼食、夕食はどれくらいかかりますか？ さらに、さまざまな場所からレストランにいくら行くかだけでなく、同時に何人の人が快適に対応できるか、およびレストランサービスがどれだけ多くの人に同時にサービスを提供できるかを理解することも重要です（たとえば、キッチンでタイムリーに温かい料理を提供できます）。 それでも、たとえば、朝食と昼食の間、または昼食と次の休憩の間の「お茶、コーヒー、クッキー」など、異なる「栄養セッション」の間の最小時間を把握することは価値があります。
  • いつチェックインでき、いつチェックアウトする必要がありますか？ これは、まず、会議全体に参加する参加者、つまり2日間に適用されます。 これらの操作を行うのに十分な時間が会議のコンテンツの外にあることを確認する必要があります。 途中で、下宿宿泊施設サービスが適切な速度で人々を住まわせる準備ができていることを確認する必要があります（この瞬間は私たちにあまり依存していないので、これも簡単ではありません）。




したがって、私たちの伝統的な演習は、潜在的な会場に来るオーガナイザーの大規模なチームであり、他の組織的および技術的な問題に加えて、すべての物流上の問題を見つけます。







そこで、10月にKlyazmaに来て、すべてのメインゾーンの間を歩いて、駐車場の場所、レストランの場所、すべての部屋の場所、喫煙場所の場所を見つけました（保健社会開発省が警告！）など。



時間ごとの内訳を準備するとき、すべてのブロックを15分の倍数にしようとします（一時的なグリッドに収まる方が簡単です）。 このようなセルを使用して、「内部」ブロックを決定します。

  • 休憩のために30分、
  • 昼食とリラックスした夕食に1.5時間、
  • 朝食1時間。




時計の針を比較します



最終的に、答えを見つけるために必要な質問のリストがあります。 たとえば、次のようになります。







会議が市内で行われる場合、最寄りの地下鉄または主要な交通ハブから徒歩で時間を知ることが重要です。 サイトが都市の郊外にある場合は、中心部（またはモスクワの地下鉄環状線）からの時間を考慮する必要があります。



多くのオーガナイザーが考えていない（または、残念ながら、何もできない）興味深いパラメーターがあります。 廊下の騒音と言います。これは、「静かなゾーン」を見つける必要があることを意味します。寒い季節に会議が開催される場合は、ワードローブでのサービスの速度を意味します。



サイトの構成と参加者の数に応じて、特定の数を減らしたり増やしたりできます。 たとえば、昼食時間は、参加者の数とレストランの収容人数に大きく依存します。 そして休憩のための時間-報告のためのホールの場所の近くから、主催者の良心。



さまざまな会議に参加し、独自の会議を開催してきた経験から、主催者ごとにこれらのパラメーターすべてに対して異なる態度を持っていることがわかります。 通常、それらの明確化の必要性は経験に伴います。 おなじみのサイトでは、重要な指標が知られている場合があります。



一般に、コンテンツコンポーネント（以下で説明します）が知識を得る時間である場合、そのような「内部セクション」は、会議に快適に滞在する時間です。これは、全体的な印象の一部であるため、それほど重要ではありません。





イベントグリッドの最初のドラフトの1つ



大きなブロックは一時的なグリッド（Excelを使用）で簡単に配布でき、参加者の交通機関や食事の計画にすでに独立して使用できます。 コンテンツコンポーネントの制限として機能します。



早い段階で合意したので、コンテンツチームと会議準備チームの残りの部分の作業を大幅にさらに進めることができます。



タイムライン



タイムラインに関する興味深い点：イベントの全体的な構造を計画するとき、重要なタスクは異なるブロックをリンクすることです。 分のタイミングを入力することは可能ですが、15、20、または30分のステップで通常は十分です。このような比較的大きなブロックは、「ラウンド」値の認識とフィッティングに便利です。



トップレベルで合意したので、あなたはすでに詳細に飛び込むことができます：今、それぞれの特定のアクティビティは、それのための便利なタイムスケールに従って分割することができます。



たとえば、参加者を下宿に輸送するのに利用可能な1.5時間から2時間は、10分バス出発間隔で簡単に打ち負かされ、その後スケジュールとしてウェブサイトに公開されます。

分割して征服する



まず、トラックまたは一般的に会議のコンテンツの責任者がどのように機能するかを理解することが重要です。 たとえば、初期段階で、会議には会社の戦略と一致する3つの大きなトピックがあると判断しました。

  • Windows 10およびユニバーサルWindowsプラットフォーム（UWP）
  • Microsoft Officeと生産性
  • Microsoft Azureとスマートクラウドプラットフォーム


各トピックには独自の責任があります。 私たちは彼らに同等の優先順位を与えます。 トラックの責任者が私たちに尋ねる最初の質問は、「レポートはいくつありますか？」です。 この質問に答えるには、まず、イベントのグリッド全体をある程度設計する必要があります。

そのような制限を知って、トラックの責任者は、トラックがどのように見えるべきかという彼のビジョンに従って、スピーカーとレポートを選択し始めます。



チームで働くときは、海岸で合意することが重要です。 プログラムを単独で策定する場合でも、最初のスピーカーを招待する前に、プログラムの一般的な骨組みを理解することが重要です。 私たちの場合、会議の3つのテーマトラックが次のように見えることに同意しました（それぞれ）。

  • スピーカーを変更するのにどれくらい時間がかかりますか？さらに5-10分かかりますか？
