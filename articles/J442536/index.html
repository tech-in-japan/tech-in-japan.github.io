外部メールボックスをZimbraに接続する

最初は大企業だったと自慢できる大企業はほとんどありません。 多くの大規模な小売チェーン、インテグレーター、開発者、およびISPは、小さなオフィスと数十人の従業員でビジネスを開始し、会社の将来の成功に熱心に取り組みました。 もちろん、このような小規模企業では、独自のメールサーバーやコラボレーションシステムを実装するという問題は非常にまれであり、従業員のメールボックスは通常、YandexメールやGmailなどの公開メールサービスに設定されます。 ただし、企業が開発の特定の段階に達し、Zimbraに基づいて独自のメールサーバーを作成する必要が生じた場合、古いメールボックスを新しいアカウントに接続するという疑問が生じます。







企業の発展は、ほとんどの場合、インターネットでの積極的な広告キャンペーン、電子メールを示す名刺の配布、およびさまざまなディレクトリへの住所の追加を意味するため、潜在的なまたは活動的な顧客からの手紙の流れは、通常、現在の住所を変更してから数年経っても弱まりません。 もちろん、従業員が古いメールボックスを毎日チェックすることを義務付けることもできますが、それでも、2つのメールボックスを1つに結合するのがより賢明なソリューションです。 このようなソリューションは、スタッフの時間と労力を節約することに加えて、新しいメールボックスにすばやく完全に移行するのに役立ちます。これは、各文字に対する回答が新しい電子メールアドレスから送信されるためです。



外部メールボックスを一見してZimbraに接続する最も簡単なオプションは、古いアカウントから新しいアカウントへのメール転送を設定することです。 ただし、この方法には多くの問題があります。 特に、Zimbraに組み込まれているスパムフィルターとウイルス対策により、手紙や添付ファイルが新しいアドレスに届かない場合があります。 さらに、一部のサービスは単にメールの送信をサポートしていない可能性があり、そのような重要な問題でそれらに完全に依存することも良い考えではありません。



そのため、外部メールボックスからメールを受信するタスクを、自分が制御するサーバーに割り当てるのが最も適切です。 ただし、Zimbra Collaboration Suiteには外部メールボックスを操作するための組み込みユーティリティがないため、小さな問題があります。 ただし、Zimbraはサードパーティのプログラムやユーティリティと完全に連携できるため、これは大きな問題にはなりません。 そのため、外部アドレスからメールを受信するには、ほとんどのLinuxディストリビューションのリポジトリで使用可能なfetchmailプログラムを使用できます。



パッケージマネージャーがfetchmailのインストールを完了した後、 / etc / default / fetchmailファイルに次のフォームを作成し、必要に応じてSTART_DAEMON = yes行をコメント解除し、 デーモン600を設定する必要があります 。 1行目では、fetchmailをシステムサービスとして実行できるようにし、2行目では、外部メールボックスにアクセスする間隔を秒単位で設定します。 この場合、このパラメーターは600秒であり、これはメールボックスが10分ごとに自動的にチェックされることを意味します。 この間隔は、サーバーの負荷を減らすために増やすことができます。または、できるだけ早く手紙を届けたい場合は、減らすことができます。



その後、fetchmailで外部メールボックスに接続するためのデータを指定するだけです。 たとえば、ユーティリティがmanager@gmail.comメールボックスに接続し、 次にmanager@company.ruにメールを転送するとします 。 これを行うには、最初にfetchmail構成ファイルを作成してから、いくつかの変更を加える必要があります。 cp /usr/share/doc/fetchmail/examples/fetchmailrc.example / etc / fetchmailrcコマンドでこれを行うことができます



その後、任意のテキストエディタでファイル/ etc / fetchmailrcを開き、ファイルの最後にある外部メールボックスに接続するためのパラメーター、外部メールボックスの認証データを設定します。 



poll imap.gmail.com protocol IMAP user manager@gmail.com is manager@company.ru here password ********* keep ssl
その後、サービスfetchmail stopおよびservice fetchmail startコマンドを連続して入力してfetchmailを再起動し、テストメールを外部メールボックスに送信してサービスをテストするだけです。 したがって、企業に独自のZimbraメールサーバーを実装した後でも、メールアドレスの変更について顧客に通知する必要をなくすことができます。
