Mac OS X 10.7.5および10.8.2でWineskinを使用して古いEXEアプリケーションを再移植する

この記事では、以前にMac OS X 10.7.5および10.8.2でWineskinを使用して移植された古いWindowsアプリケーションの「再活性化」を検討します。




私はこの問題を解決するために行った2つの眠れぬ夜にこの記事を書くことに触発されました。




したがって、ソースデータ：

1）Macbook Pro 13 '2012年半ば;

2）OS X 10.8.2;

3）公式にはMacに移植されていない、かなり古いシベリアのゲームをプレイしたいという願望。



新しいMacbook Proを購入してすぐに、新しいオペレーティングシステム10.8.2を発見したので、簡単にロールバックできる可能性はありませんでした。 さまざまなフォーラムで、その前に、OSを更新するときのゲームの非公式のWineskinポートに関する問題について読みました。 しばらくして、XQuartzとWineskinの新しいバージョンがリリースされ、問題が部分的に解決されました。



しかし、部分的にのみ...


問題は、すべてのWineskin更新ガイドが、バージョン1.1.xよりも高いバージョンのWineskinエンジンで作成されたポートを想定していたことです。

当然のことながら、以前に移植されたアプリケーションをどうするかという疑問が生じます。 「パッケージの内容」には、Wineperの更新を許可しないWineskinSettingsのみがあります。

また、新しいバージョンのWineskinで手作業で作成されたポートでは、画面が灰色または白になり、ログに$ DISPLAYの問題が発生することがよくあります。



英語のリソースで偶然見つけたソリューションに移りましょう


利用可能：

1）ワインスキンワイナリー1.7

2）古いアプリケーションポート

3）Wineskin Wrapper 2.5.8

4）XQuartz 2.7.4



手順：

1.1）Wineskin Wineryを使用して、1.5.xエンジンをダウンロードします（「+」をクリックして）

1.2）新しい.appファイルを作成します（「Create New Blank Wrapper」）



NBこの場合、「Wrapper Version」フィールドに「Wineskin-2.5.8」と記述する必要があります。



2.1）「パッケージの内容」を使用して古いアプリケーションを開きます

2.2）「パッケージの内容」を使用して、以前に作成したWrapperを開きます



3.1）「コンテンツ->リソース-> drive_c」を2.1節から2.2節に置き換えてコピーします

3.2）「Contents-> Resources」の3つの.regファイル（system.reg、user.reg、userdef.reg）を2.1節から2.2節に置き換えてコピー



4.1）「パッケージのコンテンツ」を使用して、以前に作成したWrapperを開きます

4.2）Wineskin.app-> Advancedを「Browse」の助けを借りて開き、実行可能な.EXEファイルを見つけて開きます

4.3）Wineskin.app-> Advanced-> Toolsを開き、「Refresh Wrapper」をクリックします



5）Wineskin.app-> Advancedを開き、「Test Run」をクリックします。



私はこの小さな指示で誰かの時間を節約できることを本当に願っています。
