小企業向けVoIPサーバー（FreePBX 14、Asterisk 15、Ubuntu 16.04）パート1の続き。



FreePBXとSamba 4を接続する必要がありました。Zentyal5.0にはすべてが記載されているため、ニーズに合わせて変更することは難しくないと思います。



続行：



FreePBXにはすでに統合メカニズムが組み込まれていますが、それをアクティブにするにはシステムでPHP-LDAPサポートが必要なので、依存関係を設定します。 



sudo apt install php5.6-ldap -y


次に、Apacheを再起動し、構成に直接進みます。

FreePBX 14 Webインターフェースの[管理]> [ユーザーマネージャー]> [ディレクトリ]タブに移動します







ご覧のとおり、既に「FreePBX内部ディレクトリ」があります-これはローカルのFreePBXユーザーデータベースです。



Microsoft Active Directoryに興味があります。 すでに統合を作成しているので、主なポイントを示します。







私自身は、30分ごとに更新を選択しました。 すぐに更新する必要がある場合は、コンソールから実行することをお勧めします。 



 fwconsole userman --syncall --force --verbose


だれでも接続を確立することは難しくないと思いますが、私にとっては次のように見えます。

極端な属性を除き、ユーザー設定は変更しませんでした。







また、アクセスできるグループを制限します。
