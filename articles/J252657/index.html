OData + Angular.js +ブートストラップ+ JavaScriptグリッド= 5分でアプリケーション

特定のプロジェクトで、何らかの形式のユーザー調査をWebサイトに追加する必要がある（詳細フォーム）と、システム管理者のユーザーのリストを表示および編集するフォーム（リストフォーム）を追加する必要があるとします。



OData、Angular.js、Bootstrap、およびJavaScriptグリッドを使用してこれらのフォームを作成するプロセスを検討してください。 このようなアプリケーションのすべての要件は、これらのツールで既に実装されており、実際に何も書く必要はありません。





調査フォームの主な要件をリストします（括弧内はこの要件を実装するツールです）：

  • フォームフィールド：「名」、「姓」、「メール」、「コメント」
  • フォームは、システム管理者（Angular.js）が後で表示および分析できるように、データベースにデータを保存する必要があります
  • フォームは、サーバー（Angular.js）に送信する前にデータを検証する必要があります
  • フォームは、サーバー（Angular.js）にデータを送信するときにページをリロードしないでください。
  • フォームは、サイズを変更して、サイトページ（ブートストラップ）に挿入される画面またはdivブロックのさまざまな幅に合わせる必要があります。


ユーザーリストフォームの基本要件：

  • 多数のユーザーがいる場合にページメカニズムまたは仮想スクロールを提供する
  • 任意のフィールド（グリッド列）によるソートを提供します
  • データ取得を提供する
  • データを追加、編集、削除する機能を提供する


JavaScriptグリッドは、これらの要件をすべて満たしています。 たとえば、 Kendo UI 、 DevExtreme Web 、 Syncfusion HTML5コントロール 、 Infragistics HTML5コントロール 、 OpenUI5 、 Wijmoライブラリを見る価値があります 。



基本的なサーバーリクエスト：

  • 新しいエントリを追加
  • レコードを編集
  • エントリを削除
  • 特定の条件を満たすレコードの総数を取得します
  • 共通のレコードセットから選択を取得して、1ページに表示する
  • レコードのセット全体を取得する
  • 1つ以上のフィールドでエントリを並べ替える
  • 特定の条件セットでレコードをフィルタリングする
  • バッチ更新操作の実行-一度にレコードのグループを追加、変更、削除する


これらのすべてのリクエストにハンドラーを実装する必要を回避するために、既製のODataサーバーであり、これらすべてのリクエストの完全な自動処理を提供するdataboomを使用します。



調査票







BootstrapとAngular.jsを使用して調査フォーム（詳細フォーム）を作成します。

  • ブートストラップを使用すると、フォームの設計が大幅に簡素化され、要素のサイズ変更や配置などを簡単に行うことができます。
  • Angular.jsは、データを検証してサーバーに送信するための最小限のコードを提供します。 Bootstrap + Angular.jsバンドルを使用すると、その場で簡単なフォームを作成できます。


さまざまな形状の既製のテンプレートを常に手元に用意することをお勧めします。 例：

  • さまざまなタイプのフィールドを持つ単純な詳細フォーム。
  • 小さなリストを含む単純な詳細フォーム、たとえば、連絡先のリストを持つ個人に関するデータの詳細フォーム
  • 等


JSFiddle、Bootply、CodePen、PlunkerなどのWebサイトにテンプレートを保持しておくと非常に便利です。



同様のフォームの例に基づいた単純なコピーアンドペーストの使用：

  • フォームフィールドを作成し、名前を変更します。
  • 「名」、「姓」、「メール」の各フィールドは必須であることを示しています
  • フォームからデータを保存するようにURLを設定します-この場合、「https://samples.databoom.space/api1/sandboxdb/collections/user」。
  • フォームのレイアウトを少しカスタマイズします（この場合、他のフィールドの右側にコメントを配置します）。 そして、私たちがする必要があるのはそれだけです。
