MicrosoftがArduinoや他のIoT企業とどのように友好的であるか

数か月前、 ビルド会議で 、マイクロソフトはいくつかのIoT企業とのパートナーシップを発表しました。 特に、Windows 10は最初のArduino認定オペレーティングシステムに選ばれました。 何らかの理由で、この発表（ ArduinoのWebサイトおよび公式のWindowsブログ ）は、デジタルエレクトロニクス愛好家のコミュニティにあまり積極的に受け入れられず、伝統的な聖戦を興奮させました。 これらの広告の背後にあるものを把握してみましょう。







聞かれる最も重要な肯定的なニュースは、MicrosoftがArduinoプラットフォームの重要性を理解し、対応するデバイスがWindows 10デバイスと適切に相互作用するように努力するということです。 Windows 10を実行し、適切な拡張機能を使用して標準のユニバーサルアプリケーションから利用できます。 これにより、従来のコンピューターとクラウドのコンピューティング能力と、デバイスを管理するマイクロコントローラーの可能性が最適に組み合わされ、モノのインターネットの分野で包括的なソリューションが作成されます。



現在の発表の一環として、 Windows Remote ArduinoとWindows Virtual Shield for Arduinoの2つのテクノロジーの可用性が発表されました。



  • Windows Remote Arduinoは、USBまたはBluetoothを介して接続されたArduinoボードを、I / Oポートがローカルで利用できるかのように、ユニバーサルアプリケーションから制御できるようにする技術です。 ユニバーサルアプリケーションの開発者は、アナログポートとデジタルポートを操作するための基本的なコマンドを使用できるようになります。これにより、多数のセンサーを直接操作できます。 Arduinoとコンピューター間のやり取りには、 Firmataプロトコルが使用され、ボード上で特別なFirmata Sketchが起動され、リモートコンピューターからのコマンドを実行します。 このプロジェクトは、GitHubのソースコードとして Windows 8.1およびWindows 10で利用できます 。
  • Arduino用のWindows Virtual Shieldを使用すると、Windows Phone 10を実行しているスマートフォンをArduinoの仮想拡張カードとして使用できます。 この場合、スマートフォンはテキスト画面、ジャイロスコープ/加速度計/ GPS、音声認識装置などとして使用できます。 Arduino Sketchからこれらすべての機能に簡単にアクセスできます。 例として、 Picture the Weatherプロジェクトを見てください。






これらのプロジェクトは現在活発に開発されており、正しい方向への第一歩としてそれらをとる価値があります。 次回の記事では、Windows Remote Arduinoがどのように機能するかについて、例を挙げて詳細に紹介します。



新しい世界では、自分の手で何かを収集するのが好きな愛好家によって多くの革新が行われています。英語では、このメーカーには美しい言葉があります。 マイクロソフトは、将来のIoTイノベーションに対するこのようなコミュニティの重要性を認識し、人気のあるHackster.IOポータルとのパートナーシップも発表しました。







産業のモノのインターネットについて話す場合、ここでの重要なタスクは、デバイスのネットワークが相互に、そして顧客と相互作用するためのインフラストラクチャを構築することです。 そのようなマシン間通信プラットフォームの1つは、 DataArtの DeviceHive です 。 マイクロソフトとマイクロソフトのパートナーシップの結果、Azure Marketplaceから直接DeviceHiveをクラウドに簡単にデプロイできるようになりました。これにより、モノのインターネット向けのクラウドベースのソリューションの作成が大幅に簡素化されます。







モノのインターネットは現在かなり若い方向です。 この業界を発展させるためにより多くの企業が協力すればするほど、 AllJoynなどの合意と業界標準が増えます。より多くのデバイスが相互に通信して連携し、素晴らしい技術の未来を提供し​​ます。
