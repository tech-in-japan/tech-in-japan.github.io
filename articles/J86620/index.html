カルマゲーム

Habrは訪問済みで索引付けされたリソースであり、仕事に関する役立つ情報を頻繁に見つけました。 しかし、コメントさえする可能性はありませんでした。 公開された記事の招待状を受け取りました（テーマ別のブログではありません）。 しかし、結局のところ、問題はそこで終わりませんでした。 テーマ別のブログで出版物を渡すには、誰かと「カルマゲーム」を長時間（そして退屈な）プレイする必要があることが判明しました。 問題があります-あなたは解決策を探す必要があります。



この記事ではこの問題を探り、それらの存在のために強制されています。 これはブログ「Notes to Habr」に転送されます。



人が病気であるのと同じように、彼はまず病気を治す方法を考え、仕事を続けません、彼らを必要とする人が見ていなければテーマの出版物を続けることはできませんが、最も重要なのは彼らからこれらの出版物の可能性は直接依存します。





だから、誰かがカルマを引いています。

これは、執筆時点でのカルマのスクリーンショットです。





執筆時のコメントのスクリーンショット （慎重に、長いキャンバス、310 KBa）。

spmbt.habrahabr.ru/blog-私の記事。

spmbt.habrahabr.ru/comments-私のコメント。



スクリーンショットで見ることができるように（混乱し始めたら）、リンクにはネガティブなことは何も書かず、コメントにプラスを付けます （マイナスもありましたが、後で殺されました）、私の評価は高まっています。 私は特にプラスを求めませんでした。 私は社会的に有用な情報を書いています。 オフラインでもオフラインでも、カルマに影響を与えるように誰にも依頼しなかったので、実験はきれいに見えます。 私が公に尋ねなければならなかった唯一のことは、この記事を公開する可能性について最後のコメントでカルマを上げることでした。



Habréでの数日間の滞在の間、私はかなり予想外の写真を観察しました。 テーマ記事を冷静に書いて適切なセクションに公開するのではなく、匿名ユーザーに投票する理由が一目で理解できない、不必要な「生存のための闘争」があります。 問題を解決するために、理由を一緒に把握してみましょう。



私自身は、Web開発のトピックと他のトピック（約2年でWebで30冊と15冊の出版物）の両方を出版するのに十分な経験があります。

以下は、Habréの出版物の短い歴史です。



1. Google Code Playground 。 JavaScriptセクションの機能記事。 ほとんどの人は、オンラインデバッグおよび公開ツールを必要としません。 私は冷たい興味に出会い、カルマは最初に-1または-2に落ち、記事の評価も覚えていますが、カルマは+1にまっすぐになり、評価は+5になりました。



テーマ別出版のための記事はまだありません-時間、仕事、実験、資料の提示が必要です。 さらに、最初の記事の例では、いくつかの作品によって得られた主題の記事は、適切な場所（主題のブログ）で公開できないことによって損なわれないという疑念が忍び寄っていました。



わかった 突然、1台のコンピューターで2匹のマウスを操作するトピックが以前に登場し、興味を持ちました。 すぐに質問を調査し、見つかったすべてについて書きました。



2. 画面上の2つ以上のマウスポインター 。 既存のプログラム、そのインストール、長所、短所を扱った経験。 ブログのIronカテゴリに属している必要があります。 しかし、もちろん、小さなカルマのためにそこに置くことはできません。

はい、評価はより活発な反応（最大+10）を満たしましたが、カルマ（出版の目的）は再び0を中心に変動し、-1でさえありました。 さらに、最初はマイナスになる傾向があります。 カルマの追加は発生しませんでした、さらに、彼は実質的にカルマを落とす危険を冒しました。 出版物からわかるように、彼は反社会的行為を何もしなかったという事実にもかかわらず、そしてこの記事は特定の肯定的な興味を持っています。



さて、「鉄」のトピックにそのような冷淡な関心がある場合、3番目の記事、有用なリンクのあるニュースをお試しください。 さらに、そのような記事は興味深いものです。



3. Operaのダウンロードが3倍になりました 。 そして、運が悪かったので、カルマ+の評価は再び下がり始め、読者は「拾い上げ」、話題はかかとから出てプラスになり、カルマは+1に上昇しました。 しかし、翌日、誰かがカルマ（評価ではなく）をマイナスし始め、非常に批判的になりました。 出版日から2日後、それを上げる可能性は非常に小さいですが、新しい出版の可能性はありません。



カルマを1回ゼロにすることは決定ではありません。 その後、同じゼロに戻りますが、ゼロに戻る可能性は一度限りです。



オフサイトディスカッションでは、Habréの読者の間でカルマの抑制を通じて新しい参加者の統合を抑制する特定のグループの人々が形成した苦情を見つけることができます。 非常に堅実な証拠は必要ないと思います-状況を見て、小さな計算をするのに十分です：皆を抑圧するために1日あたりいくつのマイナスが必要です。 誰もがカルマまたはトピックの評価（さまざまな人に1つ）を、その日の初めにカルマの量と同じ数だけ変更できます。



3月5日午後5時から3月6日午後5時まで（「ブログ-新規」リンクは毎日のデータのみを表示）32の記事が公開されました。 これらのうち、ギャグのカテゴリの出版物のみが高い評価（+20以上）（ 深刻な顔で冗談を言っているHabr 、 私は何も割れなかった ）、インタビューのトピック（インタビューでの興味深いタスク-論理的オリンピックタスク）、Cに関する何か++-+10以上、残りはより低く、マイナスです。 おそらく、著者がカルマ5を獲得した出版物の一部は、すでに他のブログに転送されています。



通常、評価は最初の2日間で行われますが、2日目には最初のアクティビティの5〜10％になります。



したがって、ゼロに近いカルマは特定の問題になります。誰かがそれを下げるために間違いなく働いているからです。 たとえば、3つの記事の公開中に私のカルマが+1を超えることはなく、時々-2でした。 現在、カルマに影響を与えた15人のうち、7人が「プラス」を、8人が「マイナス」を与えました。



他の参加者の観察からわかるように、カルマがすでに個人のブログを終了することを許可している場合、パブリッシャーはカルマに関してこれ以上問題を抱えていません。 メッセージはすぐにメインページの公開のしきい値を取得し、さらに友好的な人々のサークルがそれに会います-彼らは単にメインページを開き、多くの人々がメッセージを気に入っていることを知っています。



キャッチは、「個人」セクションにある「イデオロギー的にマイナス」の小さな層にあることが判明しました。 記事に害を与えるのはマイナスではなく、カルマのマイナスです。 カルマが+5であれば、記事のマイナスの問題はもうないからです（ブログの別のセクションは他の人です）。 短所は匿名ですが、そうでなかったとしても、それを無視する方法はまだありません。



もちろん、Habramechanicsの方法は、モデレーターがスパムに対処する必要がある従来のフォーラムと比較して進歩的です。 しかし、この場合、彼らは合格するために不当に高いしきい値を与えたり、メカニックの微妙さを研究したり、何らかの形で法律を欺いたりする必要があります。



提案 （理論的には純粋ですが、少なくとも自分にとっては、「メカニズム」のどの規則が問題を回避するかを検討する価値があります）。



1）票のストックを保持せずに、マイナスの力を削減または無効にするために常にマイナス。 （プロにも同じことができます）。



2）マイナスが友人の「プラス」で薄められないように-相互または周期的な「プラス」がリセットされるときに、特定のページランク （Googleの文字通りの意味）を入力します。 （マイナスについても同じことができます。カルマがゼロに近い人はプラスもマイナスもできないため、これはすでにトピックの範囲を超えています。しかし、このためにはマイナスの力を減らすために最初のポイントが必要です。



3）フォーラムのモデレーターには通常、コメントの理由があります。 その中で、彼は必然的に彼が発言した（または削除した）理由を書かなければなりません。 これは、他のモデレーターへの内部報告に必要です。 マイナス1には、このような報告はなく、匿名で、出版の能力に深刻な影響を及ぼします。 自己組織化システムのバランスの取れた存在に必要な負のフィードバックはありません。 したがって、誰かがマイナスによって誰かの権利を制限し始めた場合、その見返りに何かを失う機会を自分自身に与えなければなりません。 少なくとも、ある人のマイナスの重み、または最初の人の権利を制限するのと同じ期間、別の人の権利を制限する能力。 今、ルールから明らかなように、彼らは毎日カルマを引くことができ、この能力は鉱夫自身のカルマによって制限されています。



4）理論上、カルマは人の全体的な特性であるため、理論上、それに対する直接の（そして即時の）影響は受け入れられません（これについては、東洋哲学に状況があります）。 カルマは、個々の「罪」の合計です：悪意のあるまたは無能なメモ、未検証のコンテンツの記事、ブログまたはリソースのトピックに関係のないコンテンツを含む記事。 記事、レビュー、その他のアクティビティの評価からカルマを計算することは、はるかに論理的です（自身のマイナスの数からも）。 また、「カルマのベクトル」警告が移動する場所を示します。



今のところすべてです。 提起された問題が、現在の力学の法則の下で消費されたカルマを回復するのに役立つだけでなく、法則の構造自体についての議論のベクトルを与えることを願っています。
