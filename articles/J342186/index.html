もちろん、ArchiCADもありますが、私には少し異なるコンセプトがあるようで、今は比較しません。

3. API、ドキュメント、開始場所へのアクセス。







おそらく最も合理的な方法は、Nanocad Developer Clubに登録することです。 簡単なアンケートに記入し、承認を待って（適切に記入されていれば問題はないはずです）、その後、開発リソースにアクセスできます。



これらのリソースは何ですか：



  1. 一部のベータバージョンを含むNanocadのリリースバージョンのライセンスとディストリビューションへのアクセス、およびAPI自体（SDK）への直接アクセス。
  2. 開発者向けのプライベートフォーラムへのアクセス（ 通常のフォーラムと混同しないでください ）
  3. 開発者とのコミュニケーション、「バグ」を示す機能または「機能」を要求する機能。


まあ、言葉ではすべてが非常によく見えますが、実際にはもちろん良いですが、軟膏にハエがあります。



この方向はコアではなく、収益化できないため、たとえば、開発者ポータルではベータバージョン8.5のみが利用可能になっていますが、完全なバージョンは既に会社のポータルに投稿されていますが、残された方法で対処していると思います。 問題は、のフルバージョンということです。 私の意見では、サイトはSDKなしで行くので、ベータ版のSDKは最終版と非常に異なると思います。



さて、前述したように、MultiCAD_AC_ZC_Enabler_2209_RUパッケージ（他のCADでAPIを操作するための「レイヤー」のセット）もおそらく更新されていますが、ポータルには新しいバージョン（AutoCAD 2017用）はありません。



しかし、もう一度繰り返しますが、ありがとうございます！ これらは些細なことです。主なことは、プログラムの新しいバージョンと古いバージョンのライセンスを取得し、アプリケーションのソーイングを開始できることです（ライセンスは開発用にのみ提供されます）。



2番目の問題はAPIのドキュメントです。多くの言語（純粋な.NET API（Multicadではない）、C ++、Lisp）のAPIがありますが、今日はMultiCAD API .NETについてのみ説明します。 したがって、Microsoftのネイティブライブラリのクラス、メソッド、プロパティの詳細な説明に慣れている場合、きっと失望することでしょう。 ヘルプ内の説明のほとんどは自動的に生成されるようですが、クラスがまだ説明を思いつくことがあります。 あなたが「とにかくプログラマーではない」（私のように）であると同時に、APIのロジックをすぐに解明するための優秀な探偵ではない場合、外部からの支援なしではできないことがわかります。



どこで助けを得ることができますか？ そして、開発者フォーラムを除いてほとんどどこにもありません。 しかし、少し後のフォーラムについて、そして文学についてのスタートのために。

4.開発環境をセットアップし、簡単なコマンドを作成します



始める前に、MultiCAD .NET APIを選択したのは、その「クロスプラットフォーム性」だけでなく、 カスタムプリミティブを作成する機能も選択したことです（ただし、この例では作成しません）。







そのため、左側の顔を描くチームを作成し、次に右側の冗談を独立して描画します。 Nanocad 8.1に焦点を当てますが、コードは単純であり、8.Xファミリのどのバージョンでも動作すると思います。



MS Visual Studioでコードを作成して収集します。 さて、どのバージョンを選択するのですか？ それはすべて私たちの目標に依存します。 したがって、NC 8のSDKの一部である例は、MS Visual Studio 2012でコンパイルされました。それらを新しいバージョンで開こうとすると、ソリューション更新ウィザードが表示される場合があります。



残念ながら、SDKからパッケージ全体を収集したわけではないため、プロジェクトの新しいバージョンをもたらすことでパフォーマンスに影響があるかどうかはわかりません。 しかし、無料のVS Express Desktop 2012をダウンロードすると、SDKのプロジェクトは誓わず、原則として落ち着いて組み立てることができると確信できます。



ところで、 マイクロソフトのWebサイトから VS Express Desktop 2012をダウンロードしたい場合、成功するという事実ではありません。私の場合、MSは明らかに私がクレイジーだと思っており、ダウンロードを継続的に試みて私をブロックしますが、幸いなことにこのバージョンはネットワークでも見つかります繰り返しますが、登録サービスは停止したようです。そのため、コードも検索する必要があります。



いずれにせよ、これから作成するサンプルは、後のバージョンのVisual Studioで収集できます。 たとえば、VS 2015 Communityエディションを使用しますが、2013年と2017年にも行くべきだと思います。



最初に行うことは、プロジェクトを作成することです（この情報はSDKにもあります）。







注意、.NET Framework 4.0を選択してください。このバージョンは、NC8のAPIで使用されます（NC 5.1の場合、たとえば.NET Framework 3.5）。 次に、必要なものを選択できるクラスライブラリの名前を選択します。



次に、図のように、MultiCAD .NET API（mapimgd.dllおよびmapibasetypes.dll）からライブラリを接続する必要があります。







プロパティのローカルコピーオプションを必ず削除してください。







Nanocadでアセンブリがすぐに開くようにデバッガを構成します。 最初に、プロジェクトのプロパティで、nCad.exe（図のように）を起動したアプリケーションとして指定します。







ライブラリをスタートアップのリストに含めます（そのため、毎回netloadコマンドを呼び出さないようにします）。 これを行うには（NC 8.1の場合）、アドレスC：\ ProgramData \ Nanosoft \ nanoCAD x64 Plus 8.1 \ DataRWに移動し、次の内容のload.configファイルを見つけるか作成します。 



<root> <list> <module path="C:\Users\...\bin\Debug\nanoforhabr1.dll"/> </list> </root>


3つのドットの代わりに、プロジェクトフォルダーへの完全なアドレスは当然です。 将来的に1つのレコードに限定する必要はありません。まだ独自のライブラリがある場合は、別の1セクションを追加することができます。



これで、F5キーを押すと、プロジェクトがすぐにNanocadにアップロードされ、残りはすべて、作成するチームを入力するだけです。



いつものように、これを断片に解析し、 GitHubで全文を入手できます （a。* Csファイルはこのアドレスで利用可能です。必要に応じてプロジェクトを自分で作成してください）

5.それは残念です（結論）







さて、Viktor Koklyushkinは何と言ったのですか：「それは残念です」 有望なプロジェクト-非常に受け入れられる品質の製品は、展開できるため、展開しませんでした。 これはもちろん奇妙なことです。何らかの政府の支援策があったようです。無料版では、モニターにレンガを投げたいという欲求なしに、実際にシンプルなものを描くことができますが、それでも製品の周りに作成された環境はそれよりも悪いです。 Nanocadの開発に参加する人が増えると、「ただの楽しみ」だけでなく、Nanosoftがその需要を感じ、フォーラムが回復し、APIのドキュメント化が始まり、ファイルがより頻繁にレイアウトされるようになると思います。 原則として、無料版が更新されるのを待つことが理にかなっている場合は、将来的にそうなる可能性があります。
