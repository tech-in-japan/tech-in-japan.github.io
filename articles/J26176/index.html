プレゼンテーションのガイドライン

多くの場合、人々がプレゼンテーションの目標を理解していない状況に遭遇しました。 たとえば、科学会議（！）で、少女が科学レポートにアニメーションを挿入して、「レポートを見るのがつまらないものにならないようにしました」。 私にとってはワイルドだった。



プレゼンテーションを作成したり、見知らぬ人をレビューしたりして、自分の印象に導かれました。 LaTeX Beamerのドキュメントを見つけたとき、私はうれしく思いました。実際のラテックスビーマーに関する詳細なマニュアルだけでなく、スライドを作成するための一般的な推奨事項もありました。 ドキュメントを読んで、私はすべての段落で文字通り喜び、「はい、はい。 そうです。」



そして、RuPyRu会議のプレゼンテーションの作成に関する推奨事項を作成する必要が生じたとき、私はビーマーのドキュメントを思い出しました。 それは一般的な推薦の基礎として役立ちました 。 そして、一般的な推奨事項に基づいて、簡単なバージョンを作成しました。





これらの推奨事項を要件としてとるべきではありません; 厳密にそれらに従わなければ壊滅的なことは何もありません。 タイポグラフィの主なルールは、スライドの作成にも適用できます 。 すべてのルールに違反することはできますが、一度にすべてに違反することはできません 。







時間



  • スライドの数は、レポートの長さにほぼ対応します。 時間よりもはるかに多くのスライドがある場合は、すべてのスライドを表示する時間がないか、スライドをすばやく表示しすぎて視聴者がレポートを理解できなくなります。 スライドが少なすぎる場合は、スライドを効果的に使用していないことを意味します。
  • ほとんどの場合、十分な時間がありません。
  • 2つのトピックについて話す時間がないよりも、1つのトピックについて完全に話すほうが良いです。 2番目のトピックの重要度に関係なく。


報告書



  • 物語は一貫性があり論理的でなければなりません。
  • 古いスライドに戻ることは、しばしば良い解決策ではありません。
  • レポートはセクションに分割する必要があります。
  • レポートが15〜20分以上続く場合は、各セクションの前に簡単な概要を示す必要があります。
  • 各セクションには4〜5段落を超えることはできません（そうしないと、最後の段落に到達するまでに、聴衆はこのセクションの概要を忘れる時間があります）。
  • セクションと段落のタイトルは短く簡潔でなければなりません。
  • 話したいことの説明からレポートを始めてください。
  • すでに述べた要点をより短く、より理解しやすい形で要約して、レポートを締めくくります。 人々は、レポートの最初と最後に最も注意を払っています。 結果-これは、主なアイデアをリスナーに伝えるための2回目のチャンスです。




スライド



  • 各スライドにはタイトルが必要です。
  • 不完全なスライドは、塗りつぶされたスライドよりも優れています。 通常、スライドには20〜40語を含める必要があります。 妥当な最大値は80ワードです。
  • スライドを簡単にします。 聴衆はそれを知覚するのに約50秒しかありません。
  • 観客を過大評価しないでください。 レポートをよりアクセスしやすくします。
  • 話をしないことをスライドに表示しないでください。
  • 単色のテキストは避けてください。 番号付きリストと箇条書きリストをより適切に使用します。
  • 2を超えるリストでは、ネストレベルを使用しないでください。 チャートと図をより適切に使用します。
  • 短い文章またはフレーズを使用します。
  • 言葉を持たないでください。




フォント



  • 2つ以下のフォントを使用します（1つは見出し用、もう1つはテキスト用）。
  • 見出しとテキストに同様のフォントを使用しないでください。
  • メインテキストおよび見出しに、装飾、手書き、ゴシック、等幅フォントを使用しないでください。
  • 図および図のフォントは、テキストのメインフォントと一致する必要があります。
  • フォントサイズは、スライドが約10〜15行に収まるように選択する必要があります。
  • セマンティックハイライトには、色または太字の強度を使用します。




色



  • 色を慎重に使用してください。 それぞれの新しい色を追加する正当な理由が必要です。
  • 白い背景、 特に緑に明るい色を使用する場合は注意してください。 モニタ、プロジェクタ、およびプリンタでは色が異なるため、モニタで見栄えが良いものはレポートでは見栄えがよくありません。 背景が明るい場合は、濃い飽和色を使用します。
  • 対照的な色を使用します。 通常のテキストは、白い背景で黒、または少なくとも非常に明るいもので暗いものでなければなりません。 「暗緑色ではない背景に明るい緑色のテキスト」などの組み合わせを使用しないでください 。
  • 影は、情報コンテンツを増やすことなく明瞭さを低下させます。 「きれい」に見えるという理由だけで、シャドウを使用しないでください。
  • 明るい（暗くない）部屋では、逆色（暗い背景に明るいテキスト）が問題になることがあります。 また、配色やOHPフィルムでは、逆色の再現が困難です。




グラフィックス



グラフィックスは、多くの場合、テキストよりもはるかに効果的に概念やアイデアを明らかにします。1つの画像は1,000語以上を発言できます（また、その逆も同様です-1単語は1000以上の写真を発言できます）。



  • 可能であれば、各スライドに画像を挿入します。 視覚化は観客を大いに助けます。
  • テキストの左側に写真を配置します。左から右に読みますので、最初にスライドの左側を見てください。
  • グラフィックスには、本文、フォント、スタイルと同じタイポグラフィが必要です。
  • 写真はフルカラーである可能性があり、ベクターグラフィックス（図、チャート、グラフ）はメインのカラースキームに対応する必要があります（たとえば、黒-通常の線、赤-選択した部分、緑-例、青-構造）。
  • テキストと同様に、グラフィックのすべての要素を説明する必要があります。




アニメーションとトランジション



  • アニメーションを使用して、システムダイナミクスを説明します。 アルゴリズムなど
  • 視聴者の注意を引くためにアニメーションを使用しないでください 。
  • 適切な理由がない限り、「分離」などのスライド変更効果を使用しないでください 。
