Aptana + Ubuntu = ...（熱い追跡でテスト）

Habréのコメントで約束したように、 Aptana Studioコミュニティエディションを試しました。

その結果、探偵小説とコメディの要素を持つ物語。

著者の 記事の質問は、Ubuntu + Aptanaバンドルが動作するということでした。

好奇心が跳ね上がりました。 KomodoEditと比較してください 。

したがって、 単一のマシンの 触手の 構成 ：

CPU：Celeron 2.53 GHz

Ram：512 Mbデュアルモード

HDD：サムスン40 Gb IDE

ビデオ：オンボード、Intel



OS： Kubuntu 8.04 、KDE ​​3.x

ソフト： Aptana Studio 1.1

Java 6 Standartエディション （従来のapt-get install sun-java6 -...）

ワイン （別途言及、下記参照）

バックグラウンドミュージック： 大胆不敵 、（ メタリカ-もう一度私を撃つ ）:)



始めました。

製品Aptana Studio 1.1 Communityエディションを公式の Aptana Webサイトからダウンロードしました。

元気いっぱいのダウンロードプロセスの後、すぐに元気よくアーカイブを解凍しようとしました。

「AptanaStudioランチャーをダブルクリックする」という保証に基づいて、製品を起動することを目的としています。

/ home / dals / aptanaに、予想通り70メガバイトのアーカイブを展開しました

そして、Dolphinの助けを借りて、指定されたディレクトリに移動しました。 目はすぐに「AptanaStudio」という碑文をつかみ、彼の手は容赦なく伸びて「ダブルクリック」をしました。 しかし、それはそこにはありませんでした。クリック後、何らかの理由で、FirefoxまたはJava REの要件を備えたウィンドウが、フォーラムやブログで無制限にポップアップ表示されました。 タスクバーでワインのアイコンがぼんやりと再生されました。

悪い考えが広まり始めました：なぜですか？ そして、結局のところ、無駄ではありませんでした。Kubuntuは、独自の理由で、AptanaStudioはWin32ワールド（！）からのアプリケーションであると考えました。

「しかし！」彼は思った。 私はさらに数分のマニュアルとインターネットを読みました。 :)

/ usr / local / aptanaを推奨場所に転送することにしましたが、頭ではあまり違いはないと理解していました。

結果は同じです：dblClick-> Wine-> silent shutdown。

私は根本的に行動したいという悪意がありました。ゴミを取り除き、KomodoEditに戻ります。

しかし、いつものように、プライドは跳ね上がりました。 その場しのぎで効果的な方法を適用しました。デスクトップでAptanaStudioの起動ショートカットを作成しました。 別の試みは別の失敗です。



好奇心、仲間、世界を台無しにしません-それはそれを保存します：私はラベルの特性を調べることにしました。 彼は彼自身に対してどれほど怒っていたのでしょうか。Wineには出発点があります。 表現できないほどの落ち着きで悪意のあるアイテムを削除しました。 奇跡！

最後に、AptanaStudioはスプラッシュスクリーンを表示しました。

確かに、長くはありません：私のマシンにプログラムをロードするには、約10〜15秒かかります。



実際、Aptana + Ubuntuバンドルの速度の監視について引き続き説明します。

PHPのプラグインが配信されたことに注意してください（プログラムの公式Webサイトからも）。

FTPを使用して、CodeIgniterでWebプロジェクトのファイルを12個開き、プロジェクトに追加しました。

コントローラーにいくつかの簡単な機能を追加しました。

私は仕事に顕著なブレーキとバイスを見つけませんでした：システムとプログラムはそれぞれ適切に反応し、後者をロードしました。 一般に、KomodoEditに加えて、AptanaStudioを使用できることに注意してください。



簡単な要約 ：Ubuntu + Aptanaバンドルの抑制に関する仮定は確認されていません。 もちろん、小規模なプロジェクトの規模で。 しかし、質問はファイルで開いているタブの数を示唆していませんでした？..

:)



結論として、私は言います：もし興味があれば、インストールの写真と簡単なガイドを作り、

この製品の簡単なセットアップ。

ご清聴ありがとうございました。
