RAMディスク上のWindows



私たちの時代、私たちは皆、スピードが大好きです。 高速輸送、高速配信サービス、高速インターネットが大好きです。 そして、もちろん、高速コンピューター。 ミリ秒/マイクロ秒の遅延を検出する6番目の感覚があります。 多くの場合、何かを分散させたいという要望があります。



しかし、何を分散させるのでしょうか？ 何が私たちの仕事を遅くし、不快感を生み出しますか？ 当社のプロセッサは、多くのGHz周波数で動作し、相対論的な速度で電子を駆動します。 データ転送バスを備えたメモリは、それほど遠くありません。 しかし、周辺に来るとすぐに...ここにゆったりとした大宇宙が潜んでいます。 これは電子的にジャグリングではありません。 DVDドライブのディスクが回転し、次のバイトが入力/出力ポートから到着し、HDDが読み取りヘッドをプレート上の目的の位置に移動するまでお待ちください。 まあ、DVDとCOMポートを使用することはめったにありませんが、HDD ...私たちはまだOSを搭載しており、その中には何千ものファイルがあり、誰が必要かを知っています。 ログを書き込んでから、dll、フォント、レジストリ、その他のものをロードします。 また、一部のHDDでは、多くの同時アクセスがパフォーマンスに大きな影響を与える可能性があります -私はこれをWD Greenで観察します。



ここでは、File Cache、Prefetch、Superfetch、およびそれらのアシスタントのReadyBoostという名前のユニバースの栄光の擁護者のチームが私たちを助けるはずです。 はい、彼らは助けますが、しばしば見逃します-そして、あなたは待機カーソルを見なければなりません。



しかし、それは何ですか？ ホールでノイズが聞こえます、彼らは唱しています-SSD！ SSD！ 私が言うことができる、SSDは間違いなくクールです。 200-300mb / s以上、ほとんど瞬時にアクセスでき、ノイズはありません...まあ、ただ奇跡です。 しかし、SSDについては多くのトピックが書かれており、オーバークロック以降...おそらくもっと高速なものがあるのでしょうか？ うん SSDにはフラッシュよりも高速なものがありますか？ SSDはもっと高価ですか？ SSDからのRAID0？ さらに高速ですか？ RevoDrive？ ああ！ DDR3！ （「L1キャッシュ」と叫んだのは誰ですか？コメディアン！）



RAMディスクについて知っていますが、一時ストレージとして使用して、動作を高速化することもあります。 はい、はいを選択してシステム全体をRAMに移動した場合はどうでしょうか。 SSDよりもはるかに高速に動作し、HDDよりもさらに高速に動作するはずです。 または、ファイルキャッシュが同様の効果を与える必要がありますか？ おそらく、違いは目立たないでしょう。 さて、試してみませんか？ 私のコンピューターには16 GBのRAMがあり、すべてに十分であり、まだ残っています。 残りを使用するとよいでしょう。 行こう！



0）チェックリスト：

-16 GB以上のRAM。 8で試着することもできますが、XP以外では4で十分です。

-grub4Dos

-wincontig

-bootIce

- パーティションウィザードまたはパーティションのクローン作成に類似したもの、

-ドライバーFiradisk

-Windows x64ディストリビューション（x86も可能ですが、空きメモリは3.5GB以下です）。 WinXP（x86）、Win7、Win8でテストしました。 RAMメモリはゴムではないため、軽量バージョンが非常に望ましいです。

-インストール済みおよび動作中のWin7（またはWin8）



1） 注意！ 複数のハードディスクがある場合-問題を回避するために、システム以外のHDDで次の操作を行います。 システムの1つに触れないようにして、可能であれば無効にし、Win7を別のディスクに実験用にインストールすることをお勧めします。 設定がわかりません。一部の操作によりダウンロードが中断される場合があります。 リカバリLiveCD / USB、または少なくともWindowsインストールディスクを事前に用意しておき、少なくとも何をしているのか、すべてを取り戻す方法を大まかに理解しておくといいでしょう。 確かに、最初に仮想マシンで実験を行うことができます。



まず、少しの理論。 Grub4Dosは、マルチブートシステムを整理するためのブートマネージャーです。 とりわけ、仮想デバイスを作成し、通常のディスクと同じ方法でそれらを操作できます。 仮想ディスクは、ファイルまたはメモリにマウントできます。 メモリにマウントされると、ソースファイルの内容がメモリにコピーされ、さらに作業が進行中です。 .imaファイル（ディスクイメージ）、. isoファイル（CD / DVDイメージ）、および.imgファイル（RAW HDDイメージ）をソースとして使用できます。 Win7のネイティブ.vhdファイル（固定サイズ、非動的、非差分）は同じ.imgファイルですが、小さなヘッダーがあり、接続することもできます。

Grub4Dosを使用して作成された仮想ディスクは、リアルモードで使用できます。 Windowsは、特別なドライバーがある場合にのみそれらを表示します。 そのようなドライバーが存在します-これらはFiradisk（メモリ内のディスクで動作します）およびWinVBlock（ファイル内のイメージで動作します）です。 OSに適切なドライバーがあれば、Grub4Dosで作成されたディスクを操作できます。 残念ながら、ドライバーは署名されていないため、x64システムでの作業のために、ドライバー署名の検証を無効にする必要があります。 したがって、アクションのコース：WindowsがインストールされたVHDを準備し、署名検証をオフにし、Firadiskをインストールし、Grub4Dosを介してWindowsをロードする前にVHDをロードします。



まず、Grub4dosをマシンにインストールします。 BootIceを実行し、ブート元のドライブを選択し、MrubにGrub4dosを配置し、任意のセクションのルートにgrldrおよびmenu.lstファイルを配置します。



menu.lstに次のように記述します。 



#     Win7 title Windows (previous) find --set-root /bootmgr chainloader /bootmgr


find --set-rootコマンドは、指定されたファイルを検索し、このファイルが見つかった最初のセクションを最新にします。 したがって、他のパーティションに余分なbootmgrファイルがあるかどうかを確認してください。 システムドライブがhd0であることがわかっている場合は、直接記述できます-chainloader（hd0,0）/ bootmgr

次に、コンピューター管理とディスクデバイス管理を使用して、作業セクションのルートに固定サイズの10ギガバイト（インストールに十分かもしれません）のVHDファイルを作成し、NTFSでフォーマットし、Wincontigを使用してディスクを切断し、ファイルを最適化します。



次に、Windowsが動作するように準備したこのVHDからディスクイメージ、ハードウェアのドライバー、およびVHDからのダウンロードのサポートを取得する必要があります。 これは多くの方法で実現できます。



方法1、推奨。 Win7エディションのCorporateおよびUltimate、およびWin8の場合。 標準ツールを使用してVHDにシステムをインストールします。

詳細
インストーラーを読み込んだ後、Shift-F10を押して実行します： 

 diskpart select vdisk file="D:\win7.vhd" attach vdisk


ここで、「D：\ win7.vhd」はVHDファイルへのパスです。 ドライブ文字はメインシステムと異なる場合があります。たとえば、メモ帳を実行し、そのファイル選択ダイアログを開いて、ドライブ文字を確認できます。

続行すると、インストールするデバイスのリストにVHDが表示されます。 インストール後、Windowsブートメニューに新しいエントリが表示され、デフォルトになります。 これはVHDへのダウンロードです。



方法2。すべてのシステム。 ディスク上の別のパーティションにインストールします。 そこでWindowsを標準としてインストールし、最後にすべての準備を終えて、VHDでパーティションを複製します。



方法3。すべてのシステム。 システムを仮想マシンのVHDにインストールし、すべての準備を行ってから、実際のハードウェアでこのVHDから起動します。 実際のハードウェアに移行する場合、ハードウェアドライバーで問題が発生する可能性があります。 移植する前にsysprepを実行することをお勧めします。



方法4。WinXPの場合。 WinVBlockドライバーを使用します。

詳細
menu.lstで、これら3つのエントリを追加します。 

 title Start - TXT-mode XP Setup vhd find --set-root /winxp.vhd map --mem /winvblock.ima (fd1) map --mem /winvblock.ima (fd0) map /winxp.vhd (hd0) map --mem /xp.iso (0xff) map --hook chainloader (0xff) title Continue GUI-mode XP Setup vhd find --set-root /winxp.vhd map --mem /winvblock.ima (fd1) map --mem /winvblock.ima (fd0) map /winxp.vhd (hd0) map --mem /xp.iso (0xff) map --hook root (hd0,0) chainloader /ntldr title Boot XP vhd find --set-root /winxp.vhd map /winxp.vhd (hd0) map --hook root (hd0,0) chainloader /ntldr


ここで行われていること：

-winxp.vhdをハードディスクとして接続します（VHDファイルはWinContigで最適化する必要があります。そうしないと、Grub4Dosは接続を拒否します）

-WinVBlockドライバーを含むwinvblock.imaディスケットのイメージが接続され、メモリーにロードされます。

-インストールCD xp.isoのイメージが接続され、メモリにロードされます。

インストールには2つのエントリがあります。 WinXPのインストールは、テキストモードとグラフィックモードの2段階で構成されています。 最初に、2回目の起動でTXTを選択します-GUI。 唯一の違いは、CDまたはHDDからロードする場所です。

-F6キーを押してドライバーを選択し、配布キットに統合する必要はありません。フロッピーディスクイメージに保存するだけです。

-Grub4Dosにインストールした後、[ブート]を選択し、VHDにインストールされているシステムを起動します。



Win7、さらにはx64エディションでは、これは機能しません。 特別なストリートマジックが必要です。

ただし、WinVBlockをWin7で動作させることはできませんでした。 たぶんあなたはそれをすることができます。



方法5。Win7のすべてのエディション。 VBootを使用します 。 これは、VHDを起動できるドライバーでもあります。 署名されており、x64システムをインストールできます。 残念ながら、有料ですがデモ版が利用可能です。インストールにはこれで十分です。

詳細
2011年の時間を設定します。 （そうでなければブート時にBSODを取得します）

ディスクのルートにフォルダーVBoot、vbootldr、vbooldr.mbrを配置します。

vboot \ grub \ grub.cfgで、Windows 7 VHDおよびWindows 7 Installエントリを探し、次のように変更します。 

 menuentry "Windows 7 VHD" { vboot harddisk=(LABEL=D)/win7.vhd } menuentry "Windows 7 Install" { vboot harddisk=(LABEL=SYS)/win7.vhd cdrom=(LABEL=SYS)/win7.iso boot=cdrom }


win7.vhdファイルとwin7.isoファイル（Win7ディストリビューションキット）が配置されているパーティションのラベルをSYSに変更します（どちらを選択しても問題ありません）。 私にとって、Vbootは何らかの理由で（hd0,1）という形式のパスを認識したくないのです。 UUIDでアクセスできますが、タグの方が簡単です。



menu.lstにエントリを追加します。 

 title VBoot find --set-root /vbootldr.mbr chainloader /vbootldr.mbr


再起動 Grub4DosでVbootに移動し、その中で-Windows 7インストールに移動します。 インストーラーを読み込んだ後、shift-F10を押してメモ帳を実行し、ファイル選択ダイアログを開いて、Vbootディスクが置かれている文字を確認します。 これがD：だとしましょう。 コマンドラインに戻り、64ビットエディション用に記述します- 
 D:\vboot\tools\amd64\vbootctl setup
または32ビットの場合 
  D:\vboot\tools\x86\vbootctl setup
これにより、Vbootドライバーがインストールされ、仮想デバイスがマウントされます。 次に、デバイスのリストに表示されるVHDでインストールを続行し、Windows 7 VHDレコードから起動します。 最終構成（以下を参照）の後、vbootctl uninstallコマンドでVbootを削除すると、現在の時刻に戻すことができます。





これで、Windowsがインストールされて起動しました。

まず、Booticeを調べて、MBRのGru​​b4Dosインストーラーが破壊されたかどうかを確認します。 取り壊された場合-再度設定します。

次に、システムパーティションにWindowsブートローダーがあるかどうかを確認します-bootmgrファイルとブートフォルダー（Win7 / 8の場合）。 非表示属性とシステム属性があり、表示を有効にします。 そうでない場合、インストーラーはこのブートローダーを別の場所に表示し、そこにロードするオプションを追加するか、非表示の100mbパーティションを作成してすべてをそこに配置しました。 これは当てはまりません。システムパーティションにブートローダーが必要です。 したがって、作成します。 

 bcdboot c:\windows /sc:


そして、その中でドライバー署名の検証を無効にします。 

 bcdedit /store c:\boot\bcd /set {default} testsigning ON bcdedit /store c:\boot\bcd /set {default} nointegritychecks ON bcdedit /store c:\boot\bcd /set {default} loadoptions DDISABLE_INTEGRITY_CHECKS


現在のブートローダーでは、次も無効にします。 

 bcdedit /set testsigning ON bcdedit /set nointegritychecks ON bcdedit /set loadoptions DDISABLE_INTEGRITY_CHECKS


再起動してテストモードを有効にし、Firadiskドライバーをインストールします：hdwwiz（XPではhdwwiz.cpl）、手動でインストール、すべてのデバイス、ディスクからインストールし、.infファイルを選択します。



4）次に、通常の操作をすべて実行します。残りのドライバーをインストールし、ハードウェア、ネットワーク、デスクトップを構成し、DirectX / .NET / vcredistをインストールします。 スワップファイルを別のパーティションに転送し、一時ファイルを削除し、NTFSを使用してシステムパーティションを圧縮します。 結局のところ、アカウントのメガバイトごとに、ロードするメモリに残っています。 Win7用に約4 GBを取得しました。



次に、VHDを許容可能なサイズに絞り込みます（念のため、数GBを追加しましたが、6になりました）。 ところで、利用可能なサイズを決定する方法は？ Grub4Dosメニュー画面では、一番上の行にMem：/ 639K / 13582M / 512Mのような行があります。



これらは、イメージをロードできる利用可能なメモリ領域です。 ただし、イメージ全体を1つの領域にしかロードできないため、新しいVHDのサイズはこの行の最大値を超えてはなりません。そうしないと、エラーが発生します。



以前のWinに読み込み、win7.vhdをマウントし、有限サイズの別のvhdを作成し、Partition Expertなどのセクションエディターを使用して別のvhdをコピーします。 （インストール方法2に従うことにした場合は、今がシステムパーティションをVHDにクローンするときです）古いファイルを新しいファイルで置き換えます。再びWincontigで最適化します。 新しいVHDを起動してみましょう。 問題は発生しません。

これらすべての代わりに、既にインストールされているWin7- VHD_W7_Compactをトリミングおよび圧縮するための特別なプログラムを使用できます。 600MBまでカットできると言われています。



5）RAMから起動します！ menu.lstにさらにいくつかの行を追加します。 

 title Boot Win7 from RAM map --mem /win7.vhd (hd0) map --hook chainloader (hd0)/bootmgr


（XPの場合、/ bootmgrの代わりに/ ntldrを記述します）

最後に、RAMからBoot Win7を選択して起動します。

1〜2分でVHDがメモリにコピーされ、数秒で読み込みが完了し、デスクトップが表示されます。 Hooray、HDDを使用せずに7つの作品が動作し、かなりうまくいくようです。







また、変更を保存してVHDからWin7を直接起動するトリッキーな方法を追加することもできます。 

 title Boot Win7 from VHD (Firadisk) find --set-root --ignore-floppies /win7.vhd map /win7.vhd (hd0) map --heads=2 --sectors-per-track=18 --mem (md)0x800+4 (99) map --hook write (99) [FiraDisk]\nStartOptions=disk,vmem=find:/win7.vhd,boot;\n\0 rootnoverify (hd0,0) chainloader /bootmgr


これが機能するには、Firadiskの.infファイルで、「LoadOrderGroup = Boot Bus Extender」を「LoadOrderGroup = SCSIミニポート」に置き換えます。

ここでは、メモリではなくディスク上のイメージでFiradiskを強制的に動作させます。 速度が著しく低下するため、ネイティブVHDブートでインストール方法1を使用することをお勧めします。

私は昨年の秋から自宅のコンピューターでこの構成を6か月以上使用しています。 私は主観的な印象を整理しようとします：

長所：

+実際、すぐに動作します。 メモリからプログラムが即座に起動されます（これは明らかです）。 作業の安定性について不満はありません-コンピューターは数週間または数か月間動作します。

+ HDDからの余分な音はありません。

+「摩耗」、「読み取り/書き込みサイクル数」の概念は、RAMにとって不可欠ではありません。

+サイドプラス-結果は不変で不滅のシステムです。 あらゆる種類の曲がったプログラムを（たとえば、USBフラッシュを復元するために-それらは通常、ドライバーを配置しますが、ドライバーを選択するのは困難です）何も台無しにすることなく配置できます。 再起動すると何も起こりませんでした。

+サイドプラス-VHDはクールです。 ディスクパーティションはもうありません。頭字語/パラゴン/ gparted-sなしで単にコピー/移動/削除/バックアップされたファイルだけです。



短所：

-正直なところ、高速なSSDとの違いは目立ちません。 どちらの場合も、速度について文句を言うところはありません。

「もちろん、貴重な記憶は無駄になります。」 取り出して6〜8ギガバイトを入れます。 私の場合、残りの10GBで十分であり、十分でない場合はVHDモードで起動できます。

-Windows 7の場合は6〜8 GB-まだほとんどありません。 一部のWindows Update更新プログラムは、場所全体をすぐに使い果たしてしまいます。 また、お気に入りのプログラムをインストールしたい場合もあります。一部の人にとっては、100GBのシステムパーティションでは不十分な場合があります（ディスクをまったくパーティション分割しないことを推奨する人もいます）。 私の場合、最も人気のあるプログラムのみをシステムパーティションに配置し、それ以外はすべてHDD上のポータブルバージョンの形で保管しています。 それでも、これはホームコンピューターです.SQL Server、Visual Studio、Adobe CS、MS Office（ポータブルLibreOfficeで十分です）のようなシステムドライブには、メモリやスペースを欲するモンスターはいません。 これまでのところ良い。

-tempは別のディスクに転送する必要があります。 誰かに尋ねることのないインストーラーはそこに解凍され、十分な空きスペースがないかもしれません。

-ドライバー署名の検証を無効にする必要があります。 何もする必要はありません。

-不滅には別の側面があります。 設定を変更するたびに、更新ごとに、継続的に使用するプログラム/ドライバーのインストールごとに、VHDモードで再起動する必要があります。そうしないと、再起動時にすべてが失われます。 一部のプログラムが独自の何かをプロファイルに保存している場合（たとえば、ゲームに保存しておきます）、プロファイルを別のセクションに転送することをお勧めします。

-6〜8GBをメモリにダウンロードするには時間がかかります。10秒のダウンロードを誇ることはできません。

-明らかな理由により、休止状態を使用することは不可能です。 しかし、システムは通常のスリープ状態になり、正常に起動します。



しかし、再びホールで悲鳴が聞こえます。 「そして、SSDを購入しないと彼らが思い付かないものです！」、「はい、これは無意味です、理由もなく大量のメモリを捨てる！」、「左曲がったfireを置く必要がありますか？」、「必要ありません！」 同意します。 そのようなアプローチはお勧めしません。 これは実用的なソリューションというよりも楽しい実験です。 ご静聴ありがとうございました。今、パンのトロリーバスで写真を投稿できます。
