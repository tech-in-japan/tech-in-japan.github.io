Google I / O 2016に続く-新しいFirebase：Android統合

こんにちは、Habr！ 毎年恒例のGoogle I / Oイベントで、テクノロジーに関する記事シリーズを続けています。 今日のゲストはアレクサンダーデニソフです。彼は、更新されたFirebaseの基本的な機能について知り合いです。







こんにちは、アレクサンダー・デニソフです。Netcrackerでシニアソフトウェア開発者として働いており、ニジニノヴゴロドのGDGを率いています。 私はGoogleが長い間開発しているテクノロジーを追い続けてきましたが、今回はFirebaseを通過できませんでした。 多くの変更があり、それらはそれ自体が概念的であり、最終的に、Firebase自体をモバイルバックエンドを構築するための1つの大きなサービスに変えます。



このプレゼンテーションには非常に感銘を受けたので、個人的に新機能を使用して、Androidの便利で高速な開発のこの驚くべき世界への第一歩を踏み出す方法を共有することにしました。


背景



GoogleとFirebaseの共同歴史は2014年に始まりました。その後、Firebaseチーム全体が単にGoogleに切り替えて製品の開発を続け、Google Cloud Platformやその他のGoogleテクノロジーのクラウド機能にアクセスできるようになりました。 当時、Firebaseは実際、複数のクライアント間でデータを保存および同期するように設計されたクラウドNoSQLデータベース、および関連する認証およびホスティングサービスを管理するためのRESTサービスのセットでした。 次のようになりました。











2016年には、ジュビリー10回目のGoogle I / Oが開催され、Firebaseの新機能が紹介されます。 彼は、開発のためだけでなく、アプリケーションのプロモーションと収益化を促進するために設計された15の製品セットに成長、成熟、拡大しました。 これらの製品は、ニーズに応じて個別に、または組み合わせて使用​​できます。











良い方法では、これらの各製品は、多くの例とコードを含む個別の記事専用にする必要があります。 しかし、これには時間がかかります。そのため、Firebaseに基づいて小さなアプリケーションを作成しようとする一方で、いくつかの（私の意見では最も興味深い）可能性に精通し、テクノロジーの「味」を感じてみてください...例として、簡単なチャット、メッセージを作成しますクラウドに保存される対話者に関する情報。 そのようなものを開発するのがどれほど簡単だったか信じられません。



1. Firebase-最初のステップ



準備作業、登録、認可



空のマークアップを使用して単純なプロジェクト（com.google.simplefirebasechatと呼びましょう）を作成します。 開始されたので、Firebaseに接続する必要があります。 これを行うには、Firebaseを管理するためのメインツール（ コンソール）に移動し 、[新しいプロジェクトを作成]をクリックします。











Androidアプリケーションで作業するため、提案されたオプションから選択する必要があります。











システムは、アプリケーションの名前とSHA1形式のセキュリティキーを示すように促します













注：所有しているキーを忘れた場合は、Windowsコンソール（Win + R、CMD、Enter）を開き、debug.keystoreから抽出します。 これを行うには、次のコマンドを入力します。 



keytool -exportcert -alias androiddebugkey -keystore %USERPROFILE%/.android/debug.keystore -list -v




そして、デバッグリポジトリからパスワードでログインします（デフォルトは「android」です ）





必要なすべてのデータを提供すると、ブラウザーは、動作に必要なメタデータとともに生成されたjson Firebase設定jsonを自動的にダウンロードします。 結果のファイルは、アプリケーションのappディレクトリに配置する必要があります。 プロジェクトレベルのbuild.gradleで、対応するライブラリに依存関係を追加する必要があります 

 classpath 'com.google.gms:google-services:3.0.0'




また、appディレクトリのbuild.gradleの最後に次の行を追加する必要があります 

 apply plugin: 'com.google.gms.google-services'




これは、構成JSONaの処理とメインのcom.google.gms：google-servicesライブラリの接続に必要なプラグインをGoogleサービスプロジェクトに接続します。



出来上がり！ Firebaseは、ビルドを試みることができるアプリケーションに関連付けられています！

4. Firebase通知を使用した通知



Firebase Notificationsを使用すると、Firebaseコンソールから直接ユーザーデバイスに通知を送信できます。 また、すべてのユーザー、特定のグループ（統合は新しいFirebaseツールAudiencesでも機能します）、または一般的に特定のデバイスのユーザーに送信できます。 ただし、これらの通知を処理するには、アプリケーションに通知を受け入れて、適切なサービスを構成する必要があります。



Firebase通知を接続する



まず、適切なbuild.gradleを追加します...（依存関係、コマンドなど） 

 compile 'com.google.firebase:firebase-messaging:9.0.0'




次に、着信FCMメッセージ（Firebase Cloudsメッセージ）を管理するために使用するSimpleFirechatMessagingServiceサービスを作成します。 必要なのは、onMessageReceived（RemoteMessage remoteMessage）メソッドを再定義して、着信通知を何らかの方法で処理することだけです。



FCMロジックを管理するためのSimpleFirechatInstanceIdServiceサービスも作成する必要があります。 通常、新しいトークンが生成されたことをアプリケーションに通知し、このトークンを取得するために使用されます。 onTokenRefresh（）メソッドをオーバーライドするだけです



残っているのは、サービスをマニフェストに登録することです。その後、すべてが機能するはずです。

5. Firebase Remote Config-アプリケーションをその場で変更します



Firebase Remote Configを使用すると、サーバーからリモートで更新できるアプリケーションの多くのパラメーターを設定できます。 したがって、.apk全体を更新することなく、アプリケーションストアにダウンロードされるのを待たずに、デバイスの更新を行うことなく、アプリケーションの機能（設計、翻訳など）を確認できます。 例として、この機能を使用してメッセージ送信ボタンのテキストを変更してみましょう。











機能を追加する



最初に、[リモート構成]タブに移動し、button_name変数を作成します











次に、アプリケーションを構成する必要があります。

以前のように、適切な依存関係をbuild.gradleに追加することから再び始めます。 

 compile 'com.google.firebase:firebase-messaging:9.0.0'


...そしてfetchconfigメソッドを追加します。このメソッドでは、サーバーから構成を取得してボタン名に適用します。 

 public void fetchConfig() { long cacheExpiration = 3600; // 1 hour in seconds // If developer mode is enabled reduce cacheExpiration to 0 so that // each fetch goes to the server. 結果として得たもの



メッセージを保存するために、NoSQLクラウドデータベースを管理するサービスであるRealtime Databaseを使用しました 。 これらを使用すると、接続されているすべてのデバイスと数ミリ秒でデータを保存および同期できます。 更新されたバージョンでは、データはメモリまたはデバイスのディスクに保存されるため、ネットワークへの接続が失われてもアプリケーションは動作し続けます。もちろん、接続が復元された後、データは同期されます。



承認には、 認証サービス（ユーザーを識別および管理するためのサービス）を使用しました。 電子メールとパスワードのペア、ソーシャルネットワーク（Facebook、Twitter、GitHub、Google +）による認証をサポートし、他の既存の認証システムと統合できます。



通知をテストするために、通知を使用しました。これは、新しいUIを提供するサービスで、組み込みコンソールからアクセスでき、Firebase Cloud Messagingに基づいています。 Googleクラウドメッセージングテクノロジーに代わるものであり、1行のコードを記述することなくユーザーに通知を送信します。 Firebase Analyticsとの統合により、事前に構成されたユーザーグループに通知を送信できます



その技術は 、Firebaseと、最も人気のあるメッセージングサービスであるGoogle Cloud Messagingを組み合わせたサービスであるCloud Messagingプロジェクトと密接にリンクされています。 クラウドメッセージングは​​クロスプラットフォームであり、iOS、Android、およびWebアプリケーションで実行されます。



最後に、リモート設定には、完全なパッケージアップグレードやアプリケーションストアでの比較的長いチェックに頼らずにアプリケーションの一部を変更および更新できるサービスであるRemote Configを使用しました。 セキュリティと操作を簡単にするために、リモートで更新するものを事前に考え、キーと値のビデオペアに適切なパラメーターを設定する必要があります。 それでは、サーバー側でこれらのパラメーターの値を置き換えることにより、アプリケーションの操作と外観を変更できます。 たとえば、休日のアプリケーションの設計を更新し、すべてを元どおりに戻すことができます。 長いレビューもありません。 他の多くのFirebase要素と同様に、通知はFirebase Analyticsに統合されます。 つまり、さまざまなユーザーのグループ（数十および数百のパラメーターによって自動的に生成される）を選択し、それらに更新を送信できます...
