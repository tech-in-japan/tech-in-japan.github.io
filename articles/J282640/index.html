Xamarin Evolve 2016のオープンソースなどのXamarin SDK





今年は非常に多くのイベントが開催されましたが、間違いなくXamarinのマイクロソフトへの加入は最も印象的なものの1つです。 Build 2016カンファレンスでの発表後、クロスプラットフォームアプリケーションの開発者やXamarinフレームワークのファンを驚かせるようなことはまったくありませんでしたが、今日のEvolve 2016カンファレンスでは、Xamarinチームは成功しました。



Xamarin SDKは、約束通り、今日正式にオープンソースプロジェクトになり、誰もがこの技術を開発するために自分の努力をする機会がありました。 バインディングAPIは、iOS、Android、およびMac、コマンドラインツール、Xamarin.Formsなどで使用できます。 リンクの詳細： open.xamarin.com



Evolve 2016では、Visual Studioは、リモートiOSシミュレーターのデモを使用してモバイルアプリケーションを開発するためのさらに完全な環境になり、Visual Studioで iOSアプリケーションと対話できるようになります。 このシミュレータは、Macの参加なしではなく、Windowsマシン上で実行され、タッチスクリーンとマルチタッチ機能をサポートしています。 iOS USBリモーティングツールも発表されました。このツールを使用すると、Visual StudioからWindowsマシンに接続されているiPadまたはiPhoneにアプリケーションを展開およびデバッグできます。 リンクの詳細と手順。



Visual StudioのTest Recorderプラグインを使用すると、Visual Studioでテストスクリプトを生成できます。 Test Recorderは、Xamarin Test Cloudの何千ものデバイスで実行できるデバイスまたはシミュレーターからスクリプトを自動的に生成します。



Xamarin Studio for MacおよびXamarin.Formsの新機能 - データページとテーマを使用すると、数行のコードでアプリケーションをデータソースに接続し、それぞれ素敵なユーザーインターフェイスを作成できます。 また、新しいForms Previewerツールは、XAMLで記述されたXamarin.Formsを使用したUIの作成をリアルタイムで表示します。 これらの機能にアクセスするには、Mac上のXamarin Studio のXamarin.Formsのプレビューバージョンをインストールする必要があります。







そして最後に、Evolve 2016のオープニングで実証されたソリューションは、 Visual Studio Team Services 、 HockeyApp 、およびXamarin Test Cloudツール間に実装された統合のおかげで、今日のMicrosoftプラットフォームがモバイルアプリケーションのライフサイクル管理プロセスを作成および整理するための完全なソリューションであることを明確に確認しました。
