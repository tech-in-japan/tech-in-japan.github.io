ワイヤレス通信のセキュリティ機能は何ですか？



制御がほとんど不可能な共有環境


従来の有線ネットワークでは、ケーブルを使用して情報を送信します。 ケーブルは、「制御された」環境と見なされ、建物とケーブルが設置されている施設によって保護されています。 セキュアネットワークセグメントに入る外部の「エイリアン」トラフィックは、ファイアウォールによってフィルタリングされ、IDS / IPSシステムによって分析されます。 有線ネットワークのこのようなセグメントにアクセスするには、建物の物理的なセキュリティシステムまたはファイアウォールのいずれかを攻撃者が克服する必要があります。



無線ネットワークは電波を使用します。 空気は共有アクセスと制御のほとんど完全な欠如を持つ環境です。 有線ネットワークと同等の物理的セキュリティを提供することは不可能です。 ユーザーがアクセスポイントを有線ネットワークに接続するとすぐに、その信号は建物の壁、床、窓を通過できます。 したがって、接続されたネットワークセグメントは、別のフロアから、または近隣の建物、駐車場、または道路の反対側からでもアクセス可能になります。無線信号は、建物の外に数百メートル広がる可能性があります。 ワイヤレスネットワークの唯一の物理的な境界は、この信号自体のレベルです。

したがって、有線ネットワークとは異なり、ネットワークへのユーザーの接続ポイントは明確に定義されており、これが壁コンセントです-ワイヤレスネットワークでは、信号のみが十分であれば、どこからでもネットワークに接続できます。

また、エーテルは共有メディアです。 すべてのワイヤレスデバイスは1つの巨大な「ハブ」（ハブ）に含まれており、どのワイヤレスデバイスもネットワーク内のすべてのワイヤレスネイバーを「見る」ことができます。 同時に、パッシブモード（リスニングのみ）で動作する受信機はまったく判別できません。





ワイヤレスネットワークの境界は、建物の境界に限定されません。

消費者向けWi-Fi機器のサプライヤーの努力のおかげで、最も準備のできていないユーザーでもワイヤレスネットワークを展開できるようになりました。 安価なアクセスポイントとワイヤレスアダプターのセットの価格は50〜80ドルです。 ただし、ほとんどのデバイスはデフォルトで事前設定されているため、設定を見なくてもすぐに操作を開始できます。 承認なしに自分のアクセスポイントをインストールし、適切なセキュリティの確保を考慮しなかった従業員によるネットワークの意図的または意図しないハッキングにかかる​​のは、この量です。



さらに大きな問題は、ワイヤレスユーザーが定義上モバイルであることです。 有線ネットワークの場合のように、ユーザーは現れたり消えたり、場所を変更したり、固定されたエントリポイントに縛られたりすることはありません。カバレッジエリアのどこにいてもかまいません。 これはすべて、「招待されていないゲストをしきい値を超えて保持する」タスクとワイヤレス攻撃のソースを追跡するタスクを非常に複雑にします。



さらに、ローミングなどのモビリティの重要な要素は、ワイヤレスセキュリティの別の問題です。 今回-ユーザー。 ユーザーがアクセススイッチの特定のコンセントとポートにケーブルで「縛られ」ている有線ネットワークとは異なり、ワイヤレスネットワークでは、ユーザーは何にも縛られていません。 特別なソフトウェアを使用すると、許可されたアクセスポイントから不正な、またはSoft AP（ソフトウェアアクセスポイント）モードで動作している攻撃者のラップトップに「転送」することが非常に簡単で、疑いを持たないユーザーに対する多くの攻撃の可能性を開きます。



攻撃しやすい


無線信号はブロードキャストの性質を持ち、建物の壁に制限されず、場所をまったく修正することが困難または不可能なすべての受信機にアクセスできるため、攻撃者が無線ネットワークを攻撃することは特に簡単で便利です。 したがって、正式には、スマートフォンをポケットに入れて庭で自転車に乗っている10代の若者でさえ、純粋に若々しい好奇心からネットワークの無線インテリジェンスに従事することができます。 インターネットで利用可能なプロトコルと脆弱性を分析するための非常に多様な既製のツールは、彼が純粋に好奇心から助けて、従業員の一部が最近仕事に連れて来て、デフォルト設定から再構成することを気にしなかったアクセスポイントを見つけるのに役立ちますネットワークアクセス-そして出来上がり！

リスク3-ネットワークおよびデバイスの脆弱性


一部のネットワークデバイスは、他のネットワークデバイスよりも脆弱である可能性があります-構成の誤り、弱い暗号化キーの使用、または既知の脆弱性を持つ認証方法があります。 最初に攻撃者が攻撃するのは驚くことではありません。 アナリストの報告によると、正常なハッキングワイヤレスネットワークの70％以上が、アクセスポイントまたはクライアントソフトウェアの不適切な構成の結果として発生したとのことです。



正しく構成されていないアクセスポイント


誤って設定された1つのアクセスポイント（見知らぬ人を含む）は、企業ネットワークのハッキングを引き起こす可能性があります。 ほとんどのアクセスポイントのデフォルト設定には、認証や暗号化は含まれていません。また、マニュアルに記録されているためよく知られている静的キーを使用します。 低価格のアクセスポイントと組み合わせると、この要因により、ワイヤレスインフラストラクチャ構成の整合性とその保護レベルを監視するタスクが大幅に複雑になります。 組織の従業員は、アクセスポイントを任意に持ち込み、好きな場所に接続できます。 同時に、有能で安全な構成に十分な注意を払い、IT部門と行動を調整することは考えられません。 有線および無線ネットワークにとって最大の脅威となるのはこれらのポイントです。



正しく構成されていないワイヤレスクライアント


正しく構成されていないユーザーデバイスは、正しく構成されていないアクセスポイントよりもさらに大きな脅威になります。 これらのデバイスは文字通り企業から出入りしますが、多くの場合、ワイヤレスリスクを最小限に抑えるように特別に構成されておらず、デフォルト構成（デフォルトでは安全とは見なされません）で満足しています。 このようなデバイスは、ハッカーが有線ネットワークに侵入し、ネットワークをスキャンしてマルウェアを拡散するための便利なエントリポイントを提供するという、ハッカーに非常に貴重な支援を提供します。



暗号化のハッキング


攻撃者は、WEP暗号化標準に基づいてネットワークをハッキングするための特別なツールを利用できます（リスク4を参照）。 これらのツールはインターネットで広く取り上げられており、使用するのに特別なスキルは必要ありません。 収集されたデータが暗号化キーを回復するのに十分になるまで、ワイヤレスネットワーク上のトラフィック統計を受動的に収集することにより、WEPアルゴリズムの脆弱性を悪用します。 特別なトラフィックインジェクション手法を使用した最新世代のWEPハッキングツールを使用すると、「まで」の時間範囲は15分から15秒になります。 同様に、TKIP、さらにはWPA2を破ることを可能にするさまざまな危険度と複雑さの脆弱性があります。 これまでのところ、唯一の「侵入できない」方法は、少なくともサーバー証明書でWPA2-Enterprise（802.1x）を使用することです。



4番目のリスク-新しい脅威と攻撃


ワイヤレステクノロジーは、古い脅威を実装する新しい方法と、有線ネットワークではこれまで不可能だった新しい方法を生み出しました。 すべての場合において、攻撃者と戦うことはずっと難しくなりました。 物理的な場所を追跡したり、ネットワークから隔離したりすることはできません。



知能


従来の攻撃のほとんどはインテリジェンスから始まり、その結果、攻撃者は攻撃のさらなる経路を決定します。 ワイヤレスインテリジェンスには、ワイヤレスネットワークスキャンツール（NetStumbler、Wellenreiter、JCビルトインクライアント）とパケット収集および分析ツールの両方が使用されます。 多くのWLAN制御パケットは暗号化されていません。 同時に、情報を収集するステーションを、ネットワークへの許可されたアクセスを取得しようとする通常のステーションから、またはランダムな関連付けの試みから区別することは非常に困難です。

多くの人々は、アクセスポイントによって送信されたビーコンにネットワーク名を隠し、ブロードキャストESSID（ブロードキャストESSID）への応答を無効にすることにより、ネットワークを保護しようとします。 Security by Obscurityクラスに属するこれらのメソッドは、不十分であると広く認識されています。 攻撃者はまだ特定の無線チャネルでワイヤレスネットワークを見ているため、そのようなネットワークへの最初の許可された接続を待つだけです。 このような接続中に、ESSIDは暗号化されていない形式で空中に送信されます。 そうすると、そのような安全対策は単にその意味を失います。 Windows XP SP2ワイヤレスクライアントのいくつかの機能（SP3で修正された）が状況を悪化させました。 クライアントはこのような隠されたネットワークの名前を絶えず送信して、接続しようとしました。 その結果、攻撃者はネットワーク名を受け取るだけでなく、そのようなクライアントを自分のアクセスポイントに「フック」することもできます。



なりすましと個人情報の盗難


承認されたユーザーのなりすましは、ワイヤレスだけでなく、あらゆるネットワークにとって深刻な脅威です。 ただし、ワイヤレスネットワークでは、ユーザー認証はより困難です。 もちろん、SSIDがあり、MACアドレスでフィルタリングを試みることができますが、両方ともライブでブロードキャストされ、偽造および偽造が容易です-ネットワーク帯域幅の少なくとも「バイト」部分は、間違ったフレームを挿入して承認された通信の違反、および少なくとも少しの暗号化アルゴリズムの分割-ネットワーク構造への攻撃を手配します（たとえば、最近発見されたTKIP脆弱性の場合のように、ARPポイズニング）。 上記の段落で説明したWEPハックは言うまでもありません！

ユーザーのなりすましはMAC認証または静的キーの使用の場合にのみ可能であるという誤った信念があり、802.1xベースのスキームは絶対に安全です。 残念ながら、これは長い間そうではありませんでした。 一部のメカニズム（LEAP）は、WEPほど複雑ではありません。 EAP-FASTやPEAP-MSCHAPv2などの他のスキームはより信頼性が高くなりますが、複数の要因を同時に使用する複雑な攻撃に対する耐性を保証するものではありません。



サービス妨害（DoS）


サービス拒否攻撃の目的は、ネットワークサービスの品質に違反するか、許可されたユーザーがネットワークサービスにアクセスする可能性を完全に排除することです。 このために、たとえば、ネットワークには、正当なアドレスから送信された「ガベージ」パケット（チェックサムが間違っているなど）が散らばっている可能性があります。 ワイヤレスネットワークの場合、特別なツールなしでこのような攻撃のソースを追跡することは単に不可能です。 彼はどこにでもいられます。 さらに、目的の周波数範囲で十分に強力な干渉発生器を実行するだけで、物理レベルでDoSを編成することができます。



専用の攻撃ツール


ワイヤレスネットワークで攻撃を整理するためのツールキットは広く利用可能であり、よく知られているAirCrackからクラウドハッシュ復号化サービスで終わる新しいツールで常に更新されています。 さらに、アクセスが取得されるとすぐに、より高いレベルの従来のツールが使用されます。



5番目のリスク-有線ネットワーク情報の漏洩


ほぼすべてのワイヤレスネットワークは、ある時点で有線ネットワークに接続します。 したがって、任意のワイヤレスアクセスポイントを攻撃の踏み台として使用できます。 しかし、これがすべてではありません。アクセスポイントの構成におけるいくつかのエラーと、有線ネットワークの構成におけるエラーとの組み合わせは、情報漏洩の道を開く可能性があります。 最も一般的な例は、フラットネットワーク（またはVLANセグメンテーション違反のあるネットワーク）に接続され、ワイヤードセグメント、ARP、DHCP要求、STPフレームなどからパケットをブロードキャストする、レイヤー2ブリッジモードで動作するアクセスポイントです。 。 このデータの一部は、中間者攻撃、さまざまなポイズニングおよびDoS攻撃、および単なるインテリジェンスの編成に役立ちます。

別の一般的なシナリオは、802.11プロトコルの実装に基づいています。 1つのアクセスポイントで複数のESSIDが一度に設定されている場合、ブロードキャストトラフィックはすべてのESSIDにすぐに配信されます。 その結果、セキュリティで保護されたネットワークとパブリックホットスポットが1つのポイントで構成されている場合、ホットスポットに接続された攻撃者は、たとえば、セキュリティで保護されたネットワークでDHCPまたはARPプロトコルの動作を中断する可能性があります。 これは、ESSをBSSに適切にバインドすることで修正できます。これは、エンタープライズクラスの機器のほとんどすべてのメーカー（およびコンシューマクラスのほとんどのメーカー）によってサポートされていますが、これについて知っておく必要があります。



6番目のリスク-ワイヤレスネットワークの機能


ワイヤレスネットワークの機能の一部の機能は、全体的な可用性、パフォーマンス、セキュリティ、および運用コストに影響を及ぼす可能性がある追加の問題を引き起こします。 これらの問題に対する有能なソリューションには、従来のワイヤレスネットワーク管理ツールには実装されていない特別なサポートおよび操作ツール、特別な管理および監視メカニズムが必要です。



営業時間外のアクティビティ


ワイヤレスネットワークは有線ネットワークのような構内に限定されないため、いつでもどこでも接続できます。 このため、多くの組織では、オフィスでのワイヤレスネットワークの可用性を稼働時間（アクセスポイントの物理的な切断まで）のみに制限しています。 言われたことに照らして、営業時間外のネットワークでのワイヤレスアクティビティは疑わしいと見なされ、調査の対象になると考えるのが自然です。



スピード


低速での接続を許可するアクセスポイントは、より長い範囲の接続を許可します。 したがって、それらは安全なリモートハッキングの追加の機会を表します。 全員が24/36/54 Mbit / sの速度で作業しているオフィスネットワークで、1または2 Mbit / sの接続が突然表示された場合-これは誰かが道路からネットワークに侵入しようとしていることを示す信号です。 写真はすでに引用されています。 低速についての詳細はこちら（セクション3.1）をご覧ください 。



干渉


無線ネットワークは電波を使用するため、ネットワークの品質は多くの要因に依存します。 最も顕著な例は、無線信号の干渉です。これは、ネットワークの使用が完全に不可能になるまで、スループットとサポートされるユーザー数を大幅に低下させる可能性があります。 干渉源は、アクセスポイントと同じ周波数範囲で十分な電力の信号を発するデバイスである可能性があります：人口密集したオフィスセンター内の隣接するアクセスポイントから、産業用モーター、Bluetoothヘッドセット、さらには電子レンジまで。 一方、攻撃者は干渉を使用して、ネットワーク上でDoS攻撃を仕掛けることができます。

正当なアクセスポイントと同じチャネルで作業している見知らぬ人は、ネットワークへのアクセスを開放するだけでなく、「正しい」ワイヤレスネットワークの機能を妨害します。 さらに、エンドユーザーに攻撃を行い、Man-In-The Middle攻撃を使用してネットワークに侵入するために、攻撃者は正当なネットワークのアクセスポイントをミュートし、同じネットワーク名のアクセスポイントを1つだけ残します。



コミュニケーション


干渉に加えて、ワイヤレスネットワークの通信の品質に影響する他の側面があります。 ワイヤレス環境は共有環境であるため、誤って設定された各クライアント、またはアクセスポイントのアンテナの障害により、物理リンクレベルとデータリンクレベルの両方で問題が発生し、他のネットワーククライアントのサービス品質が低下する可能性があります。



どうする



全体として、無線ネットワークは新しいクラスのリスクと脅威を引き起こし、従来の有線手段で防御することは不可能です。 組織でWi-Fiが正式に禁止されている場合でも、これは他のユーザーが見知らぬ人をインストールしないことを意味するものではなく、これによりネットワークセキュリティへの投資がすべてゼロになります。 さらに、無線通信の機能により、アクセスインフラストラクチャのセキュリティを制御するだけでなく、攻撃者の攻撃の対象となるユーザーや、企業ネットワークから安全でない接続に単に誤ってまたは意図的に切り替えるユーザーを監視することも重要です。



このような憂鬱なプレゼンテーションの後の朗報は、これらのリスクのほとんどを最小限に抑えるか、ゼロにまで減らすことができるということです。 ワイヤレスネットワーク（インフラストラクチャおよびユーザーを含む）の安全な運用を整理するために、従来の有線ネットワーク（WLAN固有に調整）で使用される「マルチレベルセキュリティ」アプローチとほぼ一致するアプローチが使用されます。 しかし、これは後に続く別の記事のトピックでもあります。
