Microsoft®SQLServer®2008 R2 Service Pack 1





SQL Server 2008 R2 Service Pack 1（SP1）をダウンロードできます。 SQL Server 2008 R2サービスパックは、すべてのSQL Server 2008 R2サービスをSP1にアップグレードします。 サービスパックには、1から6までの一連の累積的な更新が含まれています。 これを使用して、次のバージョンを更新できます。



（1）SQL Server 2008 R2 Datacenter Edition

（2）SQL Server 2008 R2 Enterprise EditionおよびDeveloper Edition

（3）SQL Server 2008 R2 Standard Edition

（4）SQL Server 2008 R2 Web Edition

（5）SQL Server 2008 R2 Workgroup Edition

（6）SQL Server 2008 R2 Express Edition



注：SQL Server 2008 R2 Express Editionをアップグレードするには、Express Edition SP1またはExpress Services with Advanced Servicesが必要です。



SQL Server 2008 R2 Service Pack 1の新機能は何ですか？



サポートを改善するための動的管理ビュー：

sys.dm_exec_query_stats DVMは、長時間実行されるクエリのトラブルシューティングを許可するために、追加の列で拡張されました。 選択したパフォーマンスカウンター用の新しいDMVとXEventsを使用すると、SQL Serverが提供するOS構成とリソースを追跡できます。



クエリのパフォーマンスを向上させるForceSeek：

FORCESEEKインデックスの構文が変更され、個々のパラメーターによりインデックスへのアクセスを制御できるようになりました。 古いFORCESEEK構文の使用は変更されません。 また、新しいFORCESCANパラメーターが追加されました。 FIRCESEEKパラメーターを補完して、インデックスアクセス方法の「スキャン」を定義します。 新しい機能を使用したくない場合は、アプリケーションを変更する必要はありません。



高度なデータベース更新のためのデータ層アプリケーションコンポーネントフレームワーク（DAC Fx）：

新しいデータ層アプリケーション（DAC）フレームワークv 1.1およびDAC更新マネージャーを使用すると、新しいサービスを使用してデータベースを更新できます。 新しい更新サービスでは、DACをサポートするSQL AzureおよびSQL Serverの既存のデータベースのスキーマを更新できます。 詳細については、「データ層アプリケーションの設計と実装」を参照してください。



PowerPivotによるディスク領域の制御：



この更新により、システム内の情報がキャッシュされる期間を制限できる2つの新しい構成設定が追加されます。 PowerPivot構成ページの新しい[ディスクキャッシュ]セクションで、非アクティブなデータベースがアンロードされるまでメモリに保持される時間を制限できます。 また、キャッシュされたファイルが削除される前にディスクに保存される期間を決定することもできます。
