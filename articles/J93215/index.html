したがって、Appleは「5メガピクセルのカメラ、1ギガヘルツのプロセッサ」などの仮想品質のためにiPhoneをキログラム単位で販売しないという公理と考えてみましょう。しかし、製品が本当によく作られているという事実により、ハードウェアコンポーネントとソフトウェアコンポーネントのバランスが見つかりました。つまり 電話機をOSとハードウェアに分割することはできません。これらは互いに著しく補完します。 アプリストアのアプリケーションは、この共生を非常に有機的に補完します。システムの動作やその外観には影響しません-iPhoneには、Windows Mobileに精通している人なら誰でも覚えているSpb Pocket Plusのような松葉杖は基本的にありません。 たとえば、電話からWebカメラを作成できるシステムプログラムのクラスはありません。 確かに、広告は重要な役割を果たしますが、この市場やそのような環境では、むき出しの広告に行くことはありません。Palmの悲しい運命はその一例です。



その結果、特定の論理的なチェーンが構築されます。Appleは電話を開発し、特定の品質を付与しました。 電話は（予想通り）よく売れ始めました。 プラットフォームの所有者であるAppleは、電話機の高い売上は消費者の質の一意性によって決まると考えています。これは、Appleがハードウェアとソフトウェアコンポーネントの両方を制御するという事実の結果です。 Appleは、プラットフォームの唯一かつ完全な所有者として、サードパーティの開発者が特定のアプリケーションクラスを作成することを許可します。このクラスは、電話機の主要な機能を妨げず、Appleをバイパスして電話機を制御しようとしません。 アプリケーションは、Appleが電話をより良く販売するのを助け、開発者は本物のお金を手に入れ、電話の販売をさらに成長させることに非常に興味を持っています。 一般的に、別の相互に有益な共生。



しかし、フラッシュはどうですか？ フラッシュコーニーはこのチェーンに適合しません。 別のプラットフォームであるため、フラッシュはこのエコシステムにおける外来要素であり、まず第一に、技術的な理由からです。 私は、アプリストアがすぐに死ぬという考えすら考えていません。AndroidアプリケーションストアをGoogleが恐れない理由、フラッシュを備えたPSPにネイティブゲームがまだある理由などを考えてください。 など Jobsの手紙で説明されている一般的な理由である、1つの特定の例を検討します。

この場所まで読んだ後、腐食性の読者は尋ねるかもしれません-なぜiPhoneにそんなに少ないRAMがあるのですか？ 答えは簡単です。モバイルプラットフォームの制限はバッテリーの寿命に関係しています。 一見したところ、RAMは、たとえば画面と比較して、総エネルギー消費にわずかな貢献をするように見えるかもしれませんが、これは実際には誤解です。 最新のスマートフォンでは、SDRAMが使用されます。iPhoneではDDR SDRAMであり、このタイプのメモリは、ご存じのとおり、メモリセルを構成するコンデンサの充電を常に更新する必要があります。 RAMが多いほど、充電維持料金が高くなります。 また、携帯電話の画面が使用時にのみエネルギーを消費し、入力中にCPUがスリープ状態になって目を覚ます場合、RAMは常に最も深い睡眠中でも電力を必要とします（詳細については、Microsoft Windows Mobile Blogで約4年前に読むことができます）このトピックに関するすばらしい記事がありました）。



さらに、ユーザーがスマートフォンの「速度」として認識しているのは、コマーシャルの50〜60％が、RAMの種類と動作する頻度によるものです。 噂によると、彼らはiPadにDDR2をインストールした。これは明らかにこのデバイスの優れた速度を説明している。 そして、プロセッサの周波数は、カメラのメガピクセルと同じマーケティング活動です。
