ネットワークニュース





私は本当に、VMMの新製品について本当に話したいです。おそらく、VMMで最も興味深い複雑なトピックの1つ、つまりネットワークから始めましょう。

一般に、正直でわかりやすいように、2012 R2に登場したほとんどすべての新製品は、Windows Server 2012 R2に登場した新しいメカニズムの実装を目的としており、その結果、ゲストクラスターの作業と展開を行っています。 これは、VMM自体を使用して展開されたVMとその上にあるサービスのゲストIPアドレスの管理と割り当ての出現の事実を意味します... K

もちろん、同様のIPAMメカニズムをVMM自体に完全に統合することについては誰も言及していません。インストールされたIPAMの役割を持つサーバーがインフラストラクチャコンポーネントとしてVMMに追加されるだけです。





図1. IPMMサーバーをVMM 2012 R2のインフラストラクチャコンポーネントとして追加します。



ネットワークの分野でのもう1つの興味深い点は、主にホスティング事業者にとって興味深いことです。 NVGREのメカニズムをサポートするサイト間ゲートウェイを作成できるようになりました （ G eneric R outing Encapsulationを使用したネットワーク仮想化。したがって、プライベートインフラストラクチャだけでなく、テナント用の仮想化ネットワークも作成および検討できます。多くの異なるゲートウェイと、NATメカニズムを使用したインターネットへの直接アクセス（ネットワークおよびトランスレーション）。

つまり NVGREメカニズムのイデオロギーに基づいて、このテクノロジーをサポートするアダプターを使用すると、スケーラビリティと信頼性が向上します。 さて、ここでのBYOIPアプローチも非常によく適合しています。既存のVLAN構成を保存し、それをVMMベースのクラウドに転送できます。



まず、クラウドへのアクセスレベルスイッチとのネットワークへの統合（またはTOR（ T opOf R ack-switch）との統合）のサポートを追加しました。拡張可能なHyper-VスイッチのサポートがVMMに完全に到達しました 'そして、すべてが透明で便利になりました。



データウェアハウスなど





データネットワークから他のネットワーク、つまりSANネットワークやストレージの管理と提供の問題に少し移行することは、おそらく非常に論理的でしょう。 SANインフラストラクチャの分野での変更について直接話をすると、仮想ファイバーチャネルのサポート、ゾーンを管理する機能、および上記の論理に従って、FCチャネルを介してSANネットワークに接続する既製の仮想マシンを作成する機能があります。

また、ODX（Offloaded D ata T ransfer）をサポートする産業用ストレージを使用する場合、プロセッサとサーバーをアンロードするメカニズムがVMMを介した転送と移行に使用できるようになりました。これは朗報です。





図2. VMM 2012 R2の圧縮およびODXメカニズムに基づく高速コピーのサポート



共有VHDXドライブのサポートは、おそらくWindows Server 2012 R2に登場した最も興味深い機能であり、Virtual Machine Manager 2012 R2自体に正常に到達しました-ここには微妙な点が1つあります：共通ストレージでゲストクラスターを作成するために、これは仮想ディスクです共有クラスターストレージに配置する必要があります。 ローカルフォルダー内の場所にあるVHDXドライブを使用して、これまでのところ共有することはできませんが、個人的にはこれは時間の問題です...

Windows Server 2012 R2の登場により、WS2012R2自体を信頼性の高いスケーラブルなストレージのプラットフォームとして使用できるようになったことを忘れないでください。そのため、仮想化ホストだけでなく、スケーラブルなファイルサーバー（ out File Server）-その結果、VMおよびサービス用のストレージの作成を担当するストレージスペースメカニズムの管理がサポートされます。

また、Windows Server 2012 R2からの新しい仮想マシンレプリケーションメカニズムのサポートに注目する価値があります。この方法で、クラウドベースのレプリケーションスクリプトのサポートについて話すこともできます。



クラウドと仮想





最後に、仮想マシン自体と、VMM'eに存在するクラウド内の新製品について少し説明します。

個人的に私が喜ぶ最初のことは、仮想マシン用の差分ディスクのサポートでした。 これにより、VMのストレージとスペースをより効率的に使用できるだけでなく、VMとサービス（Ah VDI、VDI !!!およその著者）の展開プロセスを高速化し、入力することができます...

一般に、特に親ディスクをSSDに配置し、子ディスクがすでに100％低速（かつ安価）である場合、スクリプトはクールです...





図3. VMM 2012 R2のVMの差分ディスクサポート。



ライブへのVMのクローン作成のサポート-サーバー兄弟からも取得し、操作を停止することなく、ダウンタイムなしで特定のイメージからVMをエクスポートすることもできます。

ゲストOSとしてのWindowsとLinux-VMの両方で、VMとサービスを停止せずにVHDXディスクのサイズをオンザフライで変更する機能も利用できます。



ライブマイグレーション中の圧縮のサポートはSCVMM 2012 R2でも利用でき、ファイルをHyper-Vに転送するための修正されたAPIの導入により、VMMコンソールにネットワーク接続がない場合でもVMMコンソールからファイルをコピーおよび移動できるようになりました（以前は唯一のものでした） VM内でファイルを転送する方法）。



異なるVM（混合サービス、Linux + Windows）からのサービステンプレートのサポートもVMMによって高く評価され、尊敬されています。サービスとVMKはテンプレートからデプロイでき、どのOSがサービスのVM内にあるかを考慮できません。 ...



最後に、管理パックのカウンターが更新され、リソースの消費量を測定し、その場所と実際のリソース使用率に基づいてサービスを消費するコストを計算できるようになりました。
