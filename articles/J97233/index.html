強力な暗号化を使用したリモートデータストレージ

こんにちは、habrozhiteli。



始める前に-私はあなたを犯罪者、情報を隠している人、児童ポルノなどとみなしません。 私はあなたが私の考えについてどう思うか知りたいだけです。



一般的に、これがそのケースです。ネイティブデータセンターaltnet.lvでは、 NASに合計で約120 Tbが散らばっています。 ストレージを作成するというこの富について考えましたが、単純ではありませんが、暗号化されています-そして、私たちの側では公開鍵と暗号化されたデータのみが保存されます。 つまり、仮に「このような企業の最も秘密の簿記を私たちに与えて」という叫びで仮面を見せても、暗号化されたファイルを丁寧にダビングし、450年間さよならを言います。





まず、誰がそれを必要とします。 リストによると-警察がちょうどそのように来ることができるすべてのCIS諸国は、いくつかのラックを取り出してさよならを言います。 会計バックアップ、銀行システム、財務情報-これらはすべて自由にホストできるため、あなた以外に誰も読むことができません。 独自のSMSゲートウェイを持っている場合、SMSを受信すると、データ配列がドロップされ、数回にわたってランダムなデータでいっぱいになるようにすることは問題ありません。



第二に、価格。 私が覚えているように、Google Storageのコストはギガバイトあたり0.3です。 1ギグあたり35〜40ドルセントの価格と無制限のトラフィックで誰もが満足できると思います。



第三に、クライアントとの通信。 私たちがこれを行うと仮定します-あなたは私たちのSkypeをノックして、あなたが必要とするギグの数を注文/支払います。 サーバーとの通信用のパスフレーズと、キーに基づいてopen / close / get / put / ls / mkdir / rmを実行できるwin / linのコンソールユーティリティを送信します。 すべてが機能します-すべてがあなたの側であなた自身のキーで暗号化され、ファイル名を含むHTTPS経由で私たちに送信され、私たちはすべてをフォルダに入れます。 はい、残念ながら、フォルダ構造は保持されますが、これは最小限のセキュリティ問題です。



第三に、開発。 私自身は、cryptloop / EncFS / PGP / Amandaおよびその他のコンテナを試してみて、かなり長い間このソリューションを必要としてきました。

  • 5テラバイトを売る-Webインターフェイスがあります
  • 15を販売-ヒューズのドライバーがあります
  • 30を売る-Windows用のGUIクライアントとファイルドライバがあります
  • 50になります-すべてのキーが格納されるe-alladin e-tokenを固定し、ハーフラックがあるアムステルダムで個人的に移動し、バックアップがあります
