web2.0：外観と開発の見通しにつながる理由

私は、議論のために「Web 2.0のユーザビリティ」という論文の興味深い点を引き続き説明します（ ここの最初の部分を読んでください ）。 2番目の部分は、「Web 2.0の概念の出現と形成」と呼ばれます。 その中で、テキストに加えて、Web 2.0の主要な側面についても説明します。 ただし、この問題に焦点を当てたいと思います。





web2.0の概念とその出現につながった理由



「web 2.0」という用語の出現は、一般にティムオライリー-Web 2.0とは何かという記事に関連しています。 この記事では、著者はweb 1.0サイトとweb 2.0の同様の例を比較します。 この比較を詳細に検討して、Tim O'ReillyはWeb 2.0の基本原則を概説し、基本原則を「プラットフォームとしてのWeb」として定式化します。 次の機能は、この原則に対応しています。

  • 常時ベータ。 デスクトップソフトウェア会社とは異なり、Web 2.0サイトは計画的なリリースではなく常に改善されています。
  • ライセンスと販売の代わりに使用します。 Web 2.0サイトの収益化の原則は、ソフトウェア業界の原則とは異なります。
  • 幅広いオーディエンスリーチ。 「ロングテール原理」が使用されます。 いくつかの大きな顧客の代わりに、何千もの小さな顧客が引き付けられます。


Web 2.0を他のカテゴリのサイトから分離する必要が生じたのは、さまざまなサービスをユーザーに提供し、多くのユーザー向けに設計された新しいWebサイトのカテゴリーが出現したためです。 後に、「web 2.0」という用語が宣伝用語になりました。 この用語を使用すると、実際にはWeb 2.0の概念とはかけ離れたさまざまなサイトが注目を集めました。 これはある時点で、「Web 2.0」という用語を定義する知恵に関する深刻な論争につながりました。 一方、Web 2.0の出現により、Web 3.0の性質を予測する問題が発生しました。 個人的には、実際には「web 2.0」という用語の名前が間違っているため、web 3.0について話すことはトートロジーにすぎないと思います。 実際、Web 2.0は次世代のWebではありません。 Web 2.0は、インターネットおよびWebテクノロジーに関連する新機能の出現とともに出現した、単なる新しいカテゴリのサイトです。



インターネットの速度と信頼性の高い接続の継続的な増加、およびインターネットユーザー数の急速な増加により、開発者は膨大な数のユーザーと連携するように設計されたより機能的なサイトを作成できました。

新しいWebテクノロジーにより、Webサイトの機能が拡張され、ユーザーにとって興味深い新しいサービスが作成されました。 このようなサービスには、ビデオとオーディオのストリーミング、個人の日記の維持、ネットワーク上のさまざまな形式のファイルの保存と操作、ユーザーが通信するためのサービスが含まれます。



Web 2.0開発の見通し



開発の現在の段階では、web 2.0はインターネット上でかなりファッショナブルで人気のあるトレンドです。 WebにはさまざまなテーマのWeb 2.0サイトがますます増えています。 RunetのWeb 2.0リソースの最大のカタログには、データベースに約400のプロジェクトがあります。



いくつかのサイトはより人気がありますが、それよりも少ないものもありますが、そのようなプロジェクトの数が爆発的に増加し、ユーザーの視聴者の関心が高まる傾向があります。

しかし、合理的な疑問が生じます：2000年のドットコムのクラッシュのように、Web 2.0バブルは破裂するでしょうか？ Web 2.0の将来を予測するには、現在プロジェクトで起こっている傾向を考慮する必要があります。



他のタイプのサイトからWeb 2.0要素を借りる



情報を整理および提供するためにWeb 2.0によって提供される新しい機会に関連して、Web 2.0に直接関連しないサイトはさまざまな要素を借りています。 インターネット上で、顧客のクラウドに会い、会社のニュースにコメントし、運転ルート-Googleマップアプレットにアクセスできるようになりました。 これはすべて、Web 2.0を他のタイプのサイトと統合する傾向であり、Web 2.0で使用されるコンポーネントの普遍性と重要性についても語っています。



Web開発の現在の段階にあるWeb 2.0リソースは先駆者です。 これらのプロジェクトで実装されたアイデアは、多くの場合、インターネット上で類似物を見つけません。 便利なAPIメカニズムを使用すると、これらのアイデアを活用して任意のプロジェクトに適用したり、必要なレベルに開発したりできます。 これにより、Web 2.0は急速に開発されており、インターネットの他の部分に多くの有用なアイデアと知識を提供しています。



この点で、Web 2.0の要素を借用するためのいくつかのオプションがあります。

  • Web 2.0で作成されたブロックをサイトに実装し、サイトのテーマを調整します。 たとえば、 russianit.ruの顧客都市のタグクラウドを使用したソフトウェア製品の販売の地理の実装。 ここではWeb 2.0サイトで伝統的なタグクラウドを借用していますが、企業サイトのテーマに変換されます。
  • Web 2.0サイトによって作成されたAPIを使用して、サイトの機能要素を使用します。 例は、GoogleのオープンAPIを使用してサイトに挿入された企業サイトに統合されたGoogleマップです。 この場合、Web 2.0サービスの機能は、静的に描画された図よりも、会社のオフィスの場所をよりよく表示するのに役立ちます。
  • 社会化の要素の使用。 たとえば、潜在的な顧客と会社の従業員の社会的関係を作成するための企業ブログの確立は非常に人気が高まっています。


Web 2.0プロジェクトの統合



増え続けるWeb 2.0プロジェクトに関連して、ユーザーは、さまざまなリソースでソーシャルコネクションの交差点を見つける必要性に関連する問題を抱えています。 1つのリソースでコミュニケーションの輪を集めたユーザーは、これらのサイトを最大限に活用するために、このサークルや他のサイトのユーザーを探すことを余儀なくされることがあります。 これにより、ユーザーに問題が発生する可能性があります。そのため、ユーザーは新しいサイトの使用を拒否できます。

広範囲に分布している手段は、この問題の解決に役立ちます。

  • 複数のサイトの情報を同時に組み合わせることができるサービス。 このようなサービスの例は、たとえばサイトbestpersons.ruです。 このサイトでは、ユーザーは、登録されているネットワークで発生するイベントに関する情報を受け取ることができます。 したがって、ユーザーはそこで行われているイベントを知るために、すべてのソーシャルネットワークのサイトにアクセスする必要はありません。
  • ソーシャルネットワーク用の単一のAPIを作成するプロセス。 この分野のリーダーは、GoogleのOpenSocialプロジェクトです。 このプロジェクトには現在、15を超える最大のソーシャルネットワークが関係しています。


ユーザー登録を簡素化



Web 2.0サイトを使用するのが最も難しい場所の1つは、ほとんどすべてのプロジェクトでの必須登録です。 登録プロセス自体は非常に簡単ですが、サイトに対する特定の信頼と、サイトを使用したいという願望を伴います。 すべてのユーザーがサイトで登録プロセスを実行する準備ができているわけではありません。 これには、ユーザーの主な理由がいくつかあります。

  • サイトの不信
  • サイトを使用する必要性に対する自信の欠如
  • 外向きではなく複雑な登録プロセス


これらの理由のすべてがサイトに直接関係しているわけではないため、作成者自身がユーザーに登録プロセスを強制することはできません。

開発者を支援するために、OpenIDと呼ばれる人気の高いソリューションが登場します。
