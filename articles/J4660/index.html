仮想地球の戦いは新しい力で燃え上がります

Googleは 11月1日に、機能が強化されたGoogle Earth 4ベータ版の新しいベータ版 （Windowsの場合は4.0.2416、Macの場合は4.0.2413）をリリースしました。 Google Earthプラスの有料版の非常に重要な機能が無料プログラムに移行されました。 これは、マップ上に独自のオブジェクトを描く機会です。 3次元空間では、ほとんどすべての複雑な建物（ 写真 ）を作成し、KMLまたはKMZ形式でエクスポートできます。



さらに、2006年9月13日の以前のリリース以降、ベータには多くの小さなものが追加されました：オブジェクト（ 写真 ）の新しいアイコン、3Dモデルの高速読み込み（キャッシングのおかげ）、それらの強調表示の改善、圧縮テクスチャのサポート。 また、ルートを3D形式で印刷する機能とテクスチャマッピングの高さ属性があったため、地球の表面上に雲やその他のオブジェクトを描くことができるようになりました。 ところで、インターネットには既に地上の雲の画像をリアルタイムで受信できるサービスがあります。



Google Earth 4の公式リリースは今年の終わりまでに行われるはずですが、 Microsoftの人物のライバルはすでに3D機能を備えたLocal Liveの新しいバージョンをリリースしています。 ご覧のとおり、Virtual EarthプログラムはGoogleのようなスタンドアロン製品ではなく、オンラインマッピングサービスやその他のMicrosoftサービスと完全に統合されています。 ある意味では、このシステムは革新的と呼ぶことができます。これは、5 MBのモジュールのインストールが必要ですが、Google Earthとは異なり、ブラウザを介して直接動作するためです。









3D Virtual Earthの動作をテストするには、IE 6またはIE 7ブラウザー（米国の地域設定）、Windows XP SP2、Windows Server 2003またはVista、DirectX 9、ハードウェア3Dアクセラレーションを備えたグラフィックカード、およびワイドチャネルが必要です。 確かに、このシステムは真の写真品質を提供し、驚くほど素晴らしく見えます。 Microsoftの3Dブラウザの小さなビデオデモをここで見ることができます （広告の4分後）。



ご想像のとおり、Virtual Earthオブジェクトの写真品質は航空写真によって保証されています。 実際には、現在一般公開されている15の都市のそれぞれで実施されました。 たとえば、YouTubeでは、Microsoft Virtual Earth専用に実行されたEastbourneの航空写真の断片を含む短いビデオを見つけることができます。



これは非常に時間のかかる手順であるため、Microsoftが世界のすべての都市の航空写真を撮影することを想像するのは困難です。 来夏の終わりまでに、彼らは都市のリストを数百に拡大することを約束します。 ただし、Local Liveにはオープンなプログラミングインターフェイスがあるため、サードパーティの開発者がコンテンツの入力を支援できます。 すでに彼らの助けを借りて、 簡単なスクリプトを使用して、サイトに3次元マップを簡単に統合できます 。
