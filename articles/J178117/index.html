ベアメタル展開-またはSystem Center 2012 SP1 Virtual Machine Managerでクラウドの弾力性を維持する方法

すべての火曜日、同僚と愛好家！



今日、月曜日の4月29日-休日の前に残っているものはほとんどありません-そして、待望の休暇の前に少し知識を補充してみてください（まあ、明るい形式で-厳しい道徳的なモラルを持たないで）。 今回は、ハイパーバイザーをサーバーに展開するベアメタル機能の配置方法、つまりSystem Center 2012 SP1に基づいてプライベートクラウドの弾力性の自動化を確保する方法について少しお話ししたいと思います。







猫の下でより詳細に。







少し雲について、弾力性について





「ベアアイアン展開」プロセスについて詳しく説明する前に、このプロセスが対象とするクラウドモデルの特性、その特徴に少し注意を払う価値があります。 NIST （国立標準技術研究所） によるクラウドモデルの主な特徴を思い出してみましょう 。 憂鬱に追い付かないように、クラウドのすべての主要な特徴を従来のIT消費モデルからすぐにリストします。







1） 測定されたサービス 。 クラウドを介して提供されるサービスはすべて測定可能でなければならず、最終的にはサービスにコストがかかるという事実につながります。その結果、測定可能性はITサービスの課金ツールとして機能します。



2） 瞬時の弾力性 。 このコンポーネントは、サービスモデル（IaaS、PaaS、またはSaaS）のレベルに関係なく、クラウドがリソースのボリュームを即座に（比較的瞬時に）増減できることを意味します。 SCVMM 2012 SP1のコンテキストで今日お話しするのは、弾力性に関するリソースの増加です。 結局のところ、ハイパーバイザーを展開してホストグループとクラウドに含めるプロセスの自動化は、リソースの量を増やす方向に弾力性を提供することに他なりません。



3） リソースプール 。 クラウドの非常に重要な機能は、リソースとしての物理ホストからそのコンピューティングパワーのレベルへの抽象化です。 言い換えると、サービスまたはサービスを受け取るために、サーバーの数ではなく、CPU、RAM、ディスクリソース、データ転送ネットワークなどのカテゴリにグループ化されたリソースの量を考慮に入れていません。



4） ブロードバンドアクセス 。 これは完全に自然なことです。「非常にクライアントサーバー」、さらに正確にはサービス指向アーキテクチャ（SOA）について話しているためです-データ伝送チャネルの負荷と要件が大幅に増加しています-これはプロバイダーにも当てはまります（提供する必要があります）多くの高品質の接続）、クライアント側（このモデルでは交換のトラフィック量が大幅に増加します）。



5） リクエストに応じたセルフサービス 。 この仮定は、サービスまたはサービスをエンドユーザーに提供する自動または完全自動プロセスのいずれかのツールの可用性を意味します。 通常、このツールは、特定のサービスまたはサービスの提供と展開に関与するエンドユーザーとオーケストレーションプロセス（ワークフローの自動化）の間のインターフェイスとして機能するWebポータルです。



ベアメタルまたはヌード





クラウドの基本的な特性を思い出したので、クラウドの特性でBMD（Bare-Metal Deployment）プロセスが占める位置も理解しました。今から詳細に検討します。



可能なこととできないことは何ですか？





ハイパーバイザーをベアメタルに展開するプロセスを分解する前に、このプロセスの要件を理解しておくと便利です 。

つまり、BMCコントローラー（ iLO 、 iDRAC 、 SMASH 、 IPMIなど）を備えたサーバーと、この同じコントローラーを介してサーバーにアクセスするための資格情報が必要です。

Windows Server 2008 R2またはWindows Server 2012に基づくPXEサーバーも、余計なものではありませんが、必要なコンポーネントです。

また、インストール済みのWAIKパッケージであるWindows Automation and Installation Toolkitが本当に必要になります。

また、仮想ハードディスクVHD形式のハイパーバイザーの一般化イメージ（Windows Server 2008 R2またはWindows Server 2012-Hyper-V Serverも適しています）が必要です。



サーバーのプロセッサーで仮想化機能を有効にし、BMCコントローラーをアクティブにしてアクセス用の資格情報を確認し、PXEを使用してネットワーク経由でブートするようにサーバーを構成する必要があることを忘れないでください。



プロセスを順番に見ると、この図のようになります。



1）VMMサーバーには、準備されたWInPEイメージを含むWAIKコンポーネントが必要です。



2）次に、私が言ったように、展開されたWDS（Windows展開サービス）サーバーが必要です。



3）IPアドレスを発行するためにDHCPサーバーも必要です。



4）次に、ウィザードを使用して、前述のWDSサーバーをVMMサーバーに追加する必要があります（リソースの追加-> PXEサーバーウィザード）。



5）次に、VMMはそのエージェントをWDSサーバーにインストールします。



6）次に、％WDS_Image_Source％\ RemoteInstallフォルダーがWDSサーバー上に作成されます（つまり、公式目的のRemoteInstallサブディレクトリは、WDSサーバー上の画像が保存されているディレクトリに作成されます）。



7）次に、VMMサーバーからカスタマイズされたWInPEイメージがWDSサーバーで公開されます。公開プロセスとは、VMMエージェントとセキュリティ証明書をイメージに導入することです。



この時点で、構成プロセスを確認しました。



プロセス自体





1）OOB（帯域外管理）経由で再起動します。 ここでは、VMMサーバーだけがネイティブ機能を使用して、サーバーの外部BMCコントローラーと通信します。具体的には、サーバーを再起動する信号を送信します。



2）次に、ターゲットサーバーはPXEを介して起動し、WDSサーバーからダウンロードを取得します。



3）PXEを介してターゲットサーバーを有効にする前に、WDSサーバーはVMMサーバーへのダウンロードを許可します。



4）その後、ターゲットサーバーはWDSサーバーからカスタムWinPEイメージを受信します。



5）その後、ターゲットサーバーは、汎用コマンドを実行するためのスクリプトを実行します（GCE-scripts-General Command Execution）。 サーバーのファームウェアを自動モードで更新したり、サーバーのRAIDコントローラーを管理してハードディスクパーティションを作成したりするなど、この点は非常に重要です。 これらのパラメーターはすべて、VMMサーバーのホストプロファイルで設定されます。



6）次に、VMMイメージに必要なハイパーバイザーがVMMライブラリからロードされます。



7）その後、機器の特定のモデルに必要なドライバーが追加されます。



8）次は、サーバーをインストールするプロセス、その構成、およびドメインへの組み込みです。



9）VMMエージェントをインストールし、Hyper-Vの役割を有効にします。



10）追加のインストール後スクリプトの実行。 そのようなスクリプトの例には、ネットワークアダプターを集約に結合するメカニズム、およびiSCSI接続設定が含まれます。 このような例は、リモート実行用のコマンドです。



記事の過程で、このような言葉、ホストプロファイルまたはホストプロファイル、および展開に必要なVHDディスクについて言及しました。

ホストプロファイルを構成し、VHDドライブに関連付ける方法を詳しく見てみましょう。



1）最初に、VMMで、 [ライブラリ]-> [プロファイル]-> [ホストプロファイル]-> [ホストプロファイルセクションを右クリックして作成]に移動します 。



2）プロファイル名を入力します。





3）必要なバージョンのHyper-Vハイパーバイザーを使用して必要なVHDイメージを選択します（ちなみに、ベアメタルVMMへの展開では、Hyper-Vのサポートのみが可能です。他のハイパーバイザーは展開できません）。





4）ハードウェア構成領域で、ドライバーフィルター項目フィルタードライバーに移動し、サーバーのドライバーラベルを指定します。





5）[OS設定]タブで、必要なすべての情報を指定します。





6）ホスト設定フィールドで、何もタッチせずに[次へ]をクリックし、[概要]タブで-Fininshを押して、ホストプロファイルの作成プロセスを完了します。





次に、この場合のDell R710でのホストへのホストの展開プロセスを見てみましょう。



1）最初にFabric-> Home-> Add Resources-> Hyper-V Hosts and Clustersに移動します。





2）リソースの場所で、 「Hyper-Vにプロビジョニングされる物理コンピューター」を選択し、[次へ]をクリックします。





3）次に、データを入力してBMCコントローラーへのアクセスの管理レベルを取得します（手動で実行する必要がないように「実行」アカウントを作成することをお勧めします-実稼働環境について説明しています）。





この場合、 IPMIプロトコルを選択します。





4）[検出範囲]領域で、BMCコントローラーで構成されたIPアドレスを指定します 。





5）ターゲットリソース（ターゲットリソース）で、VMMは「サーバーが検出されました！」と言う必要があります





6）その後、展開オプション（展開オプション）で、入力した名前を指定するか、BMCアダプターに設定します。



7）その後、すべてのデータの有効性を確認し、サーバーの展開プロセスを開始するために[完了]をクリックします。コーヒーを飲みに行くことができます。



簡単な要約





同僚たちは、SC VMMツールとWindows Serverコンポーネントを使用して、Hyper-Vハイパーバイザーをベアメタルに自動展開する方法を検討しました。 また、サービスの提供と消費のクラウドモデルの観点から、このメカニズムが解決する重要なタスクについても説明しました。 将来のバージョンでは、サードパーティのハイパーバイザーもフラッディングできることが期待されています-これは、異種シナリオの場合です。 しかし、不可能なことは起こりません-今日、System Center Configuration Managerを使用してこの問題を解決できますが、それは別の話です...
