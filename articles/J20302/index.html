PuTTYのタブ

私は長い間PuTTYの Telnet / SSHクライアントを使用していますが、常にタブがありませんでした。 開発者は 、タブの実装にはGUIアプリケーション全体を完全に書き換える必要があると不満を述べており、軽量クライアントの概念を変更する正当な理由はありません。 そのため、タスクは、優先度が低い状態で改善のプールに長くとどまっています。 苦しんでいるユーザーは開発者に嘆願書を書きます。特に、PuTTYにはあらゆる種類のラッパーがハングします 。



適切なソリューションを探して、私は価値のある候補に出会いました： PuTTY Connection Manager （現在のバージョン：ベータ0.6.0.4822、フリーウェア、ソースなし）







プラットフォーム： C＃、. NET Framework 2.0

OS： x32 Windows XP SP1 / SP2、Vista、2000



開発者は簡単な方法で進めました。PuTTYグラフィカルインスタンスをアプリケーションタブに埋め込むラッパーを作成しました。 この点で、ウィンドウのフォーカスにはいくつかの困難があります。パテインスタンスがアクティブなとき、マネージャー自身のウィンドウからフォーカスが削除され、その逆も同様です。 現在の解決策（Alt + Shiftキーの組み合わせはラッパーとターミナルの間でフォーカスを変更します）は実際には非常に合理的であり、作業中に不便を引き起こしません。



楽しいから

  • キーボードから切り替える機能を持つタブ
  • ホットキー、その組み合わせは設定で変更可能
  • F11フルスクリーン
  • 接続マネージャー
  • クイック接続パネル
  • 自動更新




混乱したもの

  • ラッパーは少し重いことが判明しました。ロードに時間がかかりますが、それ自体でスマートに動作します
  • F3接続リストに切り替えるためのホットキーがありますが、キーボードからのターミナルの直感的な起動はEnterキーでは機能しません-倒錯する必要があります
  • PuTTYレジストリから個別の接続データベースを管理します。 どこで重宝するか気づくまで
  • クイック接続パネルのデフォルトのTelnetプロトコル
  • 最後のセッションからの接続マネージャーの位置を保存しません




最初に、著者がタブを閉じるためにホットキーを追加しなかった理由を理解していませんでしたが、後ですべてがPuTTYで準備できていることに気付きました。Ctrl+ CまたはCtrl + Dの組み合わせで接続と端末を閉じます 確かに、死んだ接続はマウスで閉じる必要があります。



一般的に、私はこのアプリケーションが好きでした-有望なスタートです。 開発者が開発を停止せず、安定したリリースにすることを願っています=）
