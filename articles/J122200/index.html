この投稿では、Active Directoryのグループについて説明します。 私は理論に深く入り込むことはせず、普遍的でグローバルなローカルグループについては語りません。多くの詳細があるので、文書のすべての詳細を見つけることができます。 グループが一般的に必要とされる理由と、彼らの助けを借りてどのような「グッズ」が得られるかについて話しましょう。





  • 個々のユーザーの代わりに、できる限りグループを使用します。 ACL、アクセス許可、権限の付与、グループポリシーなど グループに現在1人しかいない場合でも、とにかくグループを使用します。 これにより、システムが高速化されるだけでなく（たとえば、2〜3グループのACLと15ユーザーのACLが同時に処理されない）、管理も大幅に簡素化されます。 この権利またはその権利をどこでどのように割り当てたかを覚えておく必要はありません。単一のActiveDirectoryコンソールで、新しい人を追加したり、グループから古い人を削除したりするだけです。 セットアップが完了しました。
  • 意味のあるグループを呼び出し、説明に詳細情報を記述し、プレフィックスを使用します。 たとえば、フォルダーへのアクセスを許可するために使用されるグループはFLD_FolderName_RWと呼ばれ、インターネットアクセスグループはI_FullAccessと呼ばれます。 プレフィックスを使用すると、ADでグループをすばやく見つけることができます（ ユーザーをグループに追加できます 。また、プレフィックスには、探しているグループがすぐにリストされます）。 Descriptionの意味のある名前と説明により、6か月\ year \ it.dで（またはもうあなたではない）ことができます。 作成したグループに入れた意味を覚えておいてください。
  • ネストされたグループに関与しないでください。 これは便利なことですが、やりすぎると、ネストが繰り返されて混乱する可能性があります。
  • グループ管理を委任します。 すべてのグループを個別の構造化されたOUに持ち込み、グループメンバーシップの管理権限をその責任者に委任します。 たとえば、フォルダーの秘書への委任アクセス制御、および安全保障理事会でのリモートアクセス許可管理を受け入れます。 「サービスレターを書く-責任者と署名する-わかりました、ここにあなたを追加しました-ここにあなたの指示-アーカイブへのサービスレター」の赤テープがすべて渡され、ルーチンの一部がIT部門を去りました。
  • グループを創造的に使用して、たとえば、特権の割り当て、プリンターの配布、デスクトップへのショートカットのレイアウト、ログオンスクリプトを使用した1Cデータベースの登録などを行います。 ADのグループがかなり「容量のある」オブジェクトであることを考慮すると、さまざまな設定を直接保存でき、実行中にパラメーターがスクリプトに渡されます。 このようなスクリプトのロジックは次のとおりです。
    1. スクリプトは、このユーザーが属するグループを判別します。
    2. 事前定義された属性に従って（たとえば、プレフィックスで！）特定のグループを選択します。
    3. これらのグループ（説明フィールド、プレフィックスなしのグループ名など）から「パラメーター」を引き出し、外部コマンド、またはこれらのパラメーターを含むプロシージャを起動します。


例として、この原則に基づいて動作するスクリプトを示します。ユーザーがどのグループに属しているかを判別し、 RDP_プレフィックスが含まれるグループを見つけ、次のパラメーターを使用してデスクトップにユーザーのRDPショートカットを作成します。

  • ラベル名はプレフィックスなしのグループ名と同じです。
  • RDPショートカットによって参照されるターゲットサーバーの名前は、説明グループに書き込まれます。
その結果、誰かを1Cサーバーに入れるには、グループに追加して、システムに再入室するように頼むだけです。 さらに、リモート管理者とRDP_GroupNameグループのメンバーは、アカウンティングターミナルサーバーへのリモートアクセスが許可されます。

Function GetPrefixNameGroup(sString)

' Trim prefix of name group

Dim TempString

TempString = Left(sString, InStr(sString, "_"))

GetPrefixNameGroup = TempString

'============ Function CheckGroups ===================

Function CheckGroups(ADSPath)

Dim objUser, arrMemberOf

Const E_ADS_PROPERTY_NOT_FOUND = &h8000500D

LevelCount = LevelCount + 1

if ( LevelCount >= MaxLevelCount) then

LevelCount = LevelCount - 1

return LevelCount

end If

Set objUser = GetObject (ADSPath)

On Error Resume Next

arrMemberOf = objUser.GetEx("memberOf")

If Err.Number = E_ADS_PROPERTY_NOT_FOUND Then

LevelCount = LevelCount - 1

return LevelCount

Else

For Each Group in arrMemberOf

ADSGroup = "LDAP://" & Group

CheckGroups(ADSGroup)

WScript.Echo "Extra=" & LevelCount

Set objGroup = GetObject ( ADSGroup )

If(GetPrefixNameGroup(objGroup.CN) = "RDP_") Then

Set objGroup = GetObject ( "LDAP://" & Group)

LinkName = GetLinkNameGroup(objGroup.CN)

LinkServer = objGroup.description

LinkResult = CreateRDPFile (LinkName, LinkServer)

end If

Next

End If

LevelCount = LevelCount - 1

End Function
