コズマ・プルトコフ「 思考と格言 」



人々は常に何かを思いつきます。 チェスの発明の後、数千の同様のゲームが開発されました。 当初、古代の神話を書いた時から残っている長い習慣に従って、2つ以上のチェスの駒の品質を組み合わせたキメラが作成されましたが、後に著者の想像力が強くなり、より興味深いオプションを生み出し始めました。 この動物園全体で混乱しないようにするために、新しいフィギュアを分類する何らかのシステムが必要でした。 そして彼女は立ち上がった。 実際、私はそのうちの2つを知っています 。 残念ながら、両方とも機能しません。



チェスの多様な選択肢は常にチェスコミュニティの注目の的となっています。 専門的な出版物では、新しいゲームの説明、ゲームの分析、チェスの歴史に関する記事を見つけることができます。 ルールを体系化しようとする試みは、決して異常なものとはみなされていません。 むしろ、それは不可欠な必要性です。 このような分類の基礎は、すべてのチェスの駒を、 跳躍者と騎手という 2つの大きなクラスに分割することでした。



リーパー -駒のジャンプ（例えば、チェスの「馬」は（1,2）-リーパー ）、括弧で示されたフィールドの数に移動し、途中で遭遇する駒に注意を払わない。 1つのフィールド上を移動する図形もリーパーと見なすことができます（ Shatrangeの Ferzは（1,1）-leaper ）が、隣接するフィールドに移動するため、ジャンプできるのは0の数字のみです（すべてが正当です）。 切望して、チェスのポーンは同じクラスに帰属することもできますが、通常はそうではありません。 ほとんどの正統派は、彼女を人物とはまったく考えていません。



ルーク 、 ゾウ 、 クイーンは別のカテゴリーのフィギュアに分類されます。 これらはライダー -任意の数の空のフィールドを直線で移動するフィギュアです（ルーク- （1,0）-rider ）。 ナイトライダー （チェスの馬から得られる）などの「ジャンプ」 ライダーもいますが、ルートに沿って移動できるのは空のフィールドに沿ってのみです。 提案された分類は、それに適合しない数値が発見されるまで、合理的かつ便利であると見なされました。 ホッパーとイナゴがいました 。 これは問題を解決しませんでした。何百という数字が一方にも他方にもなかったからです。



パーレットの運動記法



この動物園の秩序を回復する最初の大規模な試みは、明らかにデビッド・パーレットによって行われました。 彼の著書「 The Oxford History of Board Games 」では、彼自身の発明の表記法を使用して、チェスゲームのさまざまなバリエーションで使用されるさまざまなピースを分類しました。



パーレット
距離：



  • 1-隣接するフィールドに移動する
  • 2-1つのフィールドを通して
  • n-任意の数のフィールド（RookやElephantなど）


方向：



  • * -すべての（8）方向への移動
  • + -直交（垂直および水平）
  • > -垂直に前進
  • < -垂直に戻る
  • <> -垂直方向に前後に移動
  • = -水平（右または左）
  • > = -水平または垂直前方
  • <= -水平または垂直の背面
  • X-斜めに
  • X> -斜め前方
  • X < -斜めに戻る


グループ化：



  • / -2つの直交する動きの順次実行
  • ＆ -同じ方向への動きの繰り返し
  • 。 -次々にアクションを実行する


このルールのセットは、古典的なチェスのキング （ 1 * ）やルーク （ nX ）などの単純なケースを記述するのに十分です。 しかし、すでにチェス騎士の場合、困難が始まります。 この図が他の図に「ジャンプ」できるという事実をどのように表現しますか？ Pawnでも、すべてが簡単ではありません。 これは、異なる方法で「サイレント」と「ショック」の動きを実行するチェスで唯一の作品です。 したがって、これらの状況を分離するには特殊文字が必要です。 表記を拡張する必要がありました。



実行条件：



  • i-ピースを使用した最初の移動（たとえば、初期位置から1つのフィールドを通るポーン移動）
  • c-テイク（ストライク）
  • o-キャプチャなしで移動（静か）


動きの種類：



  • 〜-他の形状を飛び越える
  • ^ -ジャンプしたピースを撮る（ チェッカーのように ）


追加のグループ化タイプ：



  • 、 -移動オプションの区切り（たとえば、ポーンの場合、3つの可能な移動オプションが決定されます）
  • --フィギュアの可能な動きの範囲の決定


  • キング： 1 *
  • クイーン： n *
  • 象： nX
  • ルーク： n +
  • 馬： 〜1/2
  • ポーン： o1>、c1X>、oi2>


この表記について何が言えますか？ まず第一に、それはあまり明白ではなく、（さらに悪いことに）まったく普遍的ではありません。 はい、チェッカー（ cn（^ 2X>）、o1X> ）のように、「ジャンプ」ピースと「ジャンプ」時に取るための特別なシンボルを提供しましたが、ホッパーなどの重要なクラスの数字を完全に忘れていました。



その結果、すべての種類の「銃」（たとえばXiangqiとJanggiから）は、説明の「外側」に残されました。 さらに、チェスの駒でさえ不正確に記述されています。 はい、ポーンでは、開始位置（ oi2> ）から空のフィールドを「ジャンプ」する可能性が説明されていますが、通路でのキャプチャの説明はどこにありますか？ 王によって行われる鋳造の記法はどこにありますか？ 説明するのが難しい場合でも、説明する必要がないという意味ではありません。

実際、式全体は、順次または交互に実行されるアクションの列挙です。 連続するアクションはコンマ（ ' 、 '）で区切られ、代替アクションはセミコロン（ ' ; '）で区切られます。 アクションは角かっこでグループ化できます。 アクションによってボードとピースの状態が変わるため 、アクションを実行する順序をコンマで決定することが重要です。 コンマでリストされたすべてのアクションを正常に完了したら（ほとんどの場合省略可能）、この順序で、可能な移動を生成します。



代替手段はもう少し複雑です。 2つ（またはそれ以上）の選択肢は、 同じ初期状態から始まり、互いに独立して機能します。 いずれかの選択肢の実現に成功した場合、目標を達成します。 簡単にするために、コンマはアクションを「 と 」として、セミコロンを「 または 」として接続すると仮定できます。 原則として、これらの2つのアクションはすでに十分ですが、説明を短くするために、量指定子が必要です。



  • {n、m} -アクションをnからm回繰り返します（n <m）
  • {n} -正確にn回繰り返す
  • {n、} -少なくともn回
  • {、m} -m回以下


すべてが正規表現です。 もちろん、通常の略語を定義します：



  • ？ - {0,1}
  • * - {0、}
  • + - {1、}


それでは、アクションについて話しましょう。 明らかなアクションは、ボードの方向のいずれかに沿って移動することです。 伝統的に、基点の指定が使用されます： ' N '、 ' S '、 ' W '、 ' E 'は直交、 ' NW '、 ' NE '、 ' SW '、 ' SE 'は対角線方向ですが、理由のために以下で説明しますが、2文字の指定は使用しません。 代わりに、「 LUDÆ 」プロジェクトと同じ方法で、「 M 」、「 O 」、「 R 」、「 T 」を使用することにしました。 このように：



  • N-サーバー
  • S-南
  • W-西
  • E-東
  • M-北西
  • O-北東
  • R-南西
  • T-南東
  • U-アップ（以下で説明します）
  • D-ダウン
  • + -直交（ParlettおよびLUDÆに類似）
  • X-斜めに
  • * -すべての（標準）方向


一般化された方向の指定は数量詞と矛盾しますが、この衝突を解決するのは難しくありません。 移動アクションは常に括弧で囲むことができ、量指定子は開始括弧の直後に続くことはできません。 方向の定義は、ボードの説明と密接に関連していることに注意してください。 次のようになります。

方向の選択は1回転ごとに1回行う必要があります。つまり、選択した方向を変数で固定する必要があります： （+）\ 1 * 。 この説明は優れていますが、完全に真実ではありません。 ルークは最初の敵の駒まで直線で移動できますが、空のフィールドに沿ってのみ移動できます。 「Rook」が他の図を通過しなかったためには、説明を明確にする必要があります： （+）（[p] \ 1）* 。 ここの角括弧内の文字は、空のフィールドの特別なチェックです（そうでない場合は、ストロークの生成が中断されます）。 これは、仕様で定義されている標準アクションの1つです。

「Rook」の移動を（+）（[p] \ 1）として定義*ピースがフィールドを横切る方法を説明しましたが、キャプチャの実行方法については何もありません。 まず、私たちは自分の色のフィギュアが占めるフィールドでの移動を完了できないはずです。 さらに、ターゲットフィールドに対戦相手の駒がある場合、それを取り、ボードに駒を置く必要があります： [F] [ex]？| 。 移動の開始以来、すべてがそれほど単純でもありません。 まず、ボードにピースを置くアクションを定義した場合、反対も決定する必要がありますが、最も重要なことは（そしてこれは非常に簡単に忘れることができます）、フィールド全体に自分の色のピースのみを移動する権利があります（ Stavropol Checkerではこれはありませんso）： [f] ^ 。



これらは本当に重要なポイントです（普遍的な表記法について話している場合）が、ほぼすべての図の説明でそれらを繰り返すことは過剰です。 これらの接尾辞と接頭辞をゲームボードの説明の一部にするのが賢明です。 このタイプのボード（ デフォルト ）には2つのプロパティがあります。 第一に（そしてこれは重要です）-各ボードフィールドには1つのピースしか含めることができません。 占領されたフィールドに別のフィギュアを置くと、その上に立っているフィギュアは自動的に「取得」されます。 2番目のルールは、プレイヤーが自分の駒だけを動かす権利を持っていることです（これはほとんどのゲームに当てはまります）。



このタイプのボード構造だけが可能なものではありません。 少なくとも2つのタイプを定義できます。 スタック （「 Puluk 」や「 Pillar Checker 」などのゲーム用）とヒープ （さまざまな種類の「 Mankal 」用）です。 「ピラー」ゲームの場合、プレイヤーはピースの「パイル」を動かし（自分のフィギュアがパイルの上部にある場合）、互いの「パイル」を配置する権利があります。 おとりでは、「数字」が順不同の「ヒープ」とともにフィールドに配置されます。 プレーヤーはヒープ全体を取得し、他のボードフィールドに分配します。 ヒープボード上のピースは必ずしも同じではありません！ これがそうでない少なくとも1つのマンカラがあります。
