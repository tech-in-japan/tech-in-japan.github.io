Eclipse RCPとは



そもそも、Eclipse RCPとは何かを正確に言う必要があります。 Eclipseは、Eclipse Foundation（IBM、SAP、Oracle、Eclipse RCPベースの製品を提供し、Google、RedHat、Adobe、Cisco、Intelのエコシステムの開発に参加している企業を含む）によって開発およびサポートされているJavaで書かれた開発環境です。 Eclipse RCPは、いわゆるリッチクライアントアプリケーションを作成するためのプラグインのセットです。 Eclipse IDEを開いたときにユーザーに表示されるのは、このプラットフォームに基づいて作成されたプラグインのみです。 ユーザーは既存の膨大な数のプラグインに基づいてプラグインを作成でき、必要に応じて、Hibernate、Google Guice、Google Guava、JUnit、TestNGなどの一般的なフレームワークのほとんどをプラグインの形式で見つけることができます。 また、ランタイムアーキテクチャがOSGIサービスプラットフォームの仕様に基づいていることにも注意してください。この仕様では、モジュラーアプリケーションの作成および操作方法について説明します。 前述のように、EclipseはJavaで記述され、クロスプラットフォーム製品として位置付けられています（90％の場合、RCPアプリケーションはWindows、Linux、およびMac上に構築されます）。 プラットフォームのコアを構成するすべてのプラグイン、およびほとんどのサードパーティプラグインは、EPL（Eclipse Public License）の下で配布されます。 RCPアプリケーションのユーザーインターフェイスは、SWTおよびJFaceフレームワークのビジュアルコンポーネントと、独自のEclipseウィジェットに基づいています。 以下の図は、RCPプラットフォームアプリケーションの構成要素とEclipseプラットフォーム自体の構造を示しています。

プラットフォームの長所と短所


主な疑問は、このプラットフォームが優れている理由と、デスクトップアプリケーションの開発に使用する理由です（ところで、モバイルプラットフォーム用のWebアプリケーションやアプリケーションを開発することは可能です）。 モジュール性、クロスプラットフォーム、多言語サポート、無料、膨大な数の既存のプラグイン、ライブラリ、フレームワーク。 これにより、商用レベルのアプリケーションを作成できます（このプラットフォームで開発された既存のアプリケーションのリストへのリンクは、記事の最後にあります）。 重大なアプリケーションを開発するには、少なくとも一般的な用語でOSGIフレームワークがどのように機能するかを知って、SWTおよびJFaceのコンポーネントとウィジェットを操作できるようにする必要があるため、欠点にはかなり高いレベルのエントリが含まれます。 また、ロシア語を話す開発者にとっての問題は、上記のフレームワークとライブラリに関する資料や書籍を見つけることです（ロシア語を含む見つかったリソースへのリンクは、記事の最後に記載されています）。ドイツのEclipse Foundationでは、新しいプラグインについて語る特別な雑誌が発行されており、そのアプリケーションの多くの例と、開発のすべてのニュアンスと微妙さを説明するドイツ語の書籍シリーズが含まれています。 英語では、Addison-Wesleyによる日食シリーズの特別なシリーズの本を見つけることができます。また、Apressの本もいくつか見つけることができます。 しかし、私たちの母国語では、資料と本はごくわずかです。



仕事を始める



必要なプラグインをインストールする


最初のRCPアプリケーションの作成に直接進みましょう。 動作させるには、公式サイトからアセンブリ（ RCPおよびRAP開発者向けのEclipse）をダウンロードする必要があります 。 Eclipse IDEがすでにインストールされている場合は、更新マネージャーを使用できます。 メインメニューで[ヘルプ]-> [新しいソフトウェアのインストール]を選択します。 表示される[インストール]ウィンドウで、更新が必要なサイトをドロップダウンリストから選択します-download.eclipse.org/releases/indigo、リストにそのようなサイトがない場合は、右側の[追加]ボタンをクリックし、[名前]フィールドの[場所]フィールドにIndigo Update Site -上記の[OK]アドレスをクリックします（アドレスを追加する場合-重複した場所が表示され、アドレスが利用可能なソフトウェアサイトリストに既に存在する場合、ドロップダウンの下にある[利用可能なソフトウェアサイト]リンクをクリックして、更新サイトのリストを以前のインストールウィンドウで表示できます）リスト）。 更新サイトを選択すると、ツリーリストが下部に表示され（カテゴリごとにアイテムをグループ化するチェックボックスがチェックされている場合）、汎用ツールアイテムを開き、 Eclipseプラグイン開発環境を選択し、EclipseRTターゲットプラットフォームコンポーネントアイテムを開き、プラグインをチェックします-Eclipse RCP Plug-in Developerリソース 、プロジェクトを作成するために必要なのはこれら2つのプラグインです。 次に、[次へ]を2回クリックし、ライセンス契約に同意して[完了]ボタンをクリックします。 すべて、必要なプラグインのインストールが開始されます。 インストール後、Eclipseを再起動するよう求められますが、これを行います。

プロジェクト作成

図4-作成するプロジェクトのタイプを選択するメニュー

プロジェクト構造


そのため、私たちの前に、新しく作成されたプロジェクトの構造があります。





図8-プロジェクトの構造



パッケージfirst.rcp.applicationの5つのクラスの内容は現時点では興味がありません。MyApplicationクラスは何らかの形で通常のJavaプログラムのmain（）メソッドであると言えます。このクラスはプラグインの起動方法と停止方法を担当します。 ApplicationWorkbenchWindowAdvisorクラスでは、次のコード行を使用してアプリケーションウィンドウのサイズを設定できます。

configurer.setInitialSize(new Point(400, 300));





また、デフォルトではツールバーとステータスバーが表示されないこともわかります。

configurer.setShowCoolBar(false);

configurer.setShowStatusLine(false);





最後の行は、メインウィンドウのタイトルバーを設定します。

configurer.setTitle("User Form");





ApplicationActionBarAdvisorクラスは、アプリケーションのメニューバーのカスタマイズを担当します。 Perspectiveクラスは、特定のパースペクティブにあるエディターとビューの場所とサイズを担当し、RCPアプリケーションには少なくとも1つのパースペクティブが必要です。



プロジェクトの開始


作成したアプリケーションを起動するには、META-INFフォルダーに移動して、突然MANIFEST.MFファイルを開く必要があります（このファイルは、プロジェクトの作成時にデフォルトで開きます）。





図9-RCPアプリケーションプロパティエディター



このファイルにより、多くのプロジェクト設定の変更、追加のプラグインの接続、拡張機能の接続と管理、プラグインのアセンブリの構成などを行うことができます。 [概要]タブの[テスト]セクションで、リンクをクリックします-Eclipseアプリケーションを起動します。すぐにアプリケーションウィンドウが画面に表示され、閉じて次の段階に進みます。





図10-アプリケーションのウィンドウ



ビューを追加する



拡張機能を追加


私たちのタスクは、グラフィカルインターフェイスの要素を配置できるビューを作成することです。 MANIFEST.MFファイルに移動して、下の[拡張機能]タブを選択します。 デフォルトでは、org.eclipse.core.runtime.applicationsとorg.eclipse.ui.perspectivesの2つの拡張機能があります。 最初の拡張機能はMyApplicationアプリケーションのクラスに関連付けられています。この拡張機能を選択し、ツリーリストをfirst.rcp.application.MyApplication（実行）ノードに展開すると、クラスフィールドにこの特定のクラスの名前が含まれていることがわかります。 このリストのルートノードに戻ると、[拡張機能の詳細]セクションの右側に、値を変更できる2つのフィールドが表示されます（今は変更しません）：ID-アプリケーション識別子とName-アプリケーション名。



2番目の拡張機能は、アプリケーションのパースペクティブを設定します。 MANIFEST.MFファイルを使用すると、Perspectiveクラスのコードを調べることなく、このパースペクティブに含まれるタイプやエディター、その位置、サイズ、比率を示すことができます。 この拡張機能をクリックして子アイテムに移動すると、[拡張要素の詳細]セクションで、パースペクティブクラス、識別子、および名前を指定できることがわかります。 上記のような変更は、この拡張機能に関連付けられているクラスコードを直接編集し、 plugin.xmlファイルを編集することで行うことができますが、最初のプロジェクトの作成を複雑にすることはできません。





図11-プロジェクトの拡張機能を含むタブ



ビューを作成するには、新しい拡張子を追加する必要があります。 これを行うには、[追加]ボタンをクリックし、表示されるウィンドウの[拡張ポイントフィルター]フィールドにビューを入力します。拡張機能はorg.eclipse.ui.viewsのみで、選択して[完了]ボタンをクリックします。





図12-新しい拡張子を選択するためのウィンドウ



拡張機能のリストには、別の3番目の拡張機能が必要です。



カスタマイズを表示


追加した拡張機能をマウスの右ボタンでクリックすると、コンテキストメニューが表示され、その中の[新規]-> [ビュー]を選択して、拡張機能に要素を追加します（この要素は必要なビューです）。 このアイテムの設定は右側に表示されます。 まず、ビューのクラスを作成します。 これを行うには、クラス*リンクをクリックします。





図13-ビュー設定



Javaクラスを作成するための標準ダイアログが開き、名前を付けてMainViewが表示されます。 ご覧のとおり 、このクラスは、すべての種類の親クラスであるクラスorg.eclipse.ui.part.ViewPartを継承しています。 [完了]ボタンをクリックして、クラスの作成を終了します。 ビューのクラスを作成した後、そのコードは私たちの前に開き、この型の識別子を含む静的変数を作成し、このクラスの正規名を識別子として設定します。 以下を記述します。

…

public class MainView extends ViewPart {

public static final String ID = MainView.class.getCanonicalName();

…

}





MANIFEST.MFファイルに戻り、[拡張機能]タブを開き、クラス*フィールドの内容をコピーして、IDフィールドに貼り付けます。 この種のクラス名と識別子は同じになりました。目的のビューをすばやく見つけてその識別子を見つけることができるため、この手法は良い習慣です。 フィールド名*に書き込みます-ユーザーフォーム。 ビュー全体は作成されましたが、ビューだけを使用することはできません。つまり、あるパースペクティブに属している必要があるため、ビューとパースペクティブを接続します。



ビューのレイアウト設定


拡張エディターにとどまるか、パースペクティブクラスのコードPerspective.javaを開くことで、さらに操作を行うことができます。 Perspectiveクラスで、メソッドcreateInitialLayout（）に進みましょう。このメソッドは、ビューとエディターの初期位置を設定します。 このメソッドでは、次の2行のコードを記述します。

layout.setEditorAreaVisible(false);

layout.addStandaloneView(MainView.ID, true, IPageLayout.LEFT, 1.0f, layout.getEditorArea());





最初の行は、ビューが1つしかなく、エディターがないため、編集オブジェクトが不要であることをレイアウトオブジェクトに伝えます。 2行目は、ビューをスタンドアロンとして追加します。 最初のパラメーターは、種の識別子です。 2番目のパラメーターはブール値で、ビューのタイトル（ユーザーフォーム）を表示するかどうかを決定します。 3番目のパラメーターは、視点が1つあり、視点のスペース全体を占めるため、視点の視点の方向を示すことを目的としています。この値は任意です。 4番目のパラメーターは、他のビューまたはエディターに対するこのビューの位置を決定します。これは、この場合、パースペクティブのスペース全体を占める唯一のビューでなければならないためです。 最後の5番目のパラメーターは、エディター領域の識別子です。 変更を保存します。



ビューを追加したアプリケーションの起動


MANIFEST.MFファイルに戻り、アプリケーションを再度実行するには、[概要]タブの[テスト]セクションで、[Eclipseアプリケーションの起動]リンクをクリックします。 前回の開始時のフォームの外観には大きな違いはありません。ビューが追加されたタブ（ユーザーフォーム）のみが追加されました。
