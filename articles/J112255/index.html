ホスティングの進化

先週、クラウドプロバイダーのParking.ruは、 W！Nと呼ばれるWindows 2008 での共有ホスティングの新しいホスティングプランを開始しました。 それは一般的なことのように思えますが、このサイトは、これがWindowsでホストするための新しい、ほとんど革命的なアプローチであることを示しています！





基本、テクノロジーバックエンドから始めましょう。 そして、その基盤は、顧客に合わせて収容される専用サーバーだけでなく、現在ほとんどすべてのParking.ru製品で使用されているクラスターソリューションです。 これにより、ユーザーは必要な量のリソースを受け取り、互いに「ピンチ」しません。



資源といえば、資源制限が関税でなくなったという一見重要なポイントです！ 同じクラスタリングとIISおよびその他のテクノロジの最新バージョンの使用により、サイトへのリソースの割り当ては動的であり、制限は柔軟になりました。 そのため、サイトの負荷が急激に増加しても 、リソース不足による訪問者のアクセスエラーは発生せず、サイトのオープンは少し遅くなります。 ユーザーからのリクエストは「仮想キュー」に分類されます。 サイトが「機能しない」ため、顧客は競合他社にアクセスしません。



もう1つの重要な点は、ASP.NET 2.0（3.0、3.5を含む）および4.0、ASP、PHP5、SSI、MVC 3.0、MySQL、MS SQL 2008などを含む最も先進的なテクノロジーのサポートです。



ところで、 W！Nは、最近リリースされたMicrosoft Web Deployをサポートするロシアで最初の関税です。これにより、Visual Studioから直接サイトをホストできます。



クライアントインターフェイスとリソースアカウンティングの観点から、劇的な変化も起こりました！ まず、これは「 ユニファイドストレージ 」（ストレージ）の概念であり、ユーザーは1つの共有仮想ストレージ内の場所を割り当てられ、そのリソース内で必要なリソースと範囲を選択します。 静的なページとメール用の多くのスペースのみが必要な場合、サイトのスペースのサイズを小さくし、メールによる制限を増やすだけです！ これにより、関税は非常に柔軟になりました！



次に、配布の管理に役立つW！N UIが、現在のステータスと使用されているリソースを表示します。









一般的に、 W！N UIの説明は難しく、試してみるのは簡単です。特に、今では開始関税計画W！N 1にアクションがあり、 わずか40ルーブル + 14日間のテストで無料で購入できます！
