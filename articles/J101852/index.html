DocMe-YouTube for documents

私たちの若くて興味深いプロジェクト-DocMe.ruを紹介したいと思います。DocMe.ruは現在公開ベータテスト中です。 このサービスは、さまざまなテキスト形式のドキュメントの公開を目的としています。 私は外国の類似物をリストしません、私はHabréの聴衆がそれらに遭遇したと確信しています。



サイトにアクセスしてから1分以内に誰もが理解できる可能性を説明する代わりに、プロジェクトに関連するいくつかの事実をお伝えします。

  • プロジェクトは現在3つのサーバーに展開されています。 フロントエンドとバックエンドは最初に動作し、コンバーターは2番目に動作します（これはAmazonインスタンスであり、負荷が増加すると自動的に別のインスタンスに複製されます）。
  • プロジェクトのメインモジュールはjavaで記述されています。 ビューアはフラッシュで書かれています。
  • 5人がプロジェクトに取り組んでいます。1人のフラッシャー、2人のプログラマ、1人のシステム管理者、1人のマネージャーであり、1人のデザイナーではありません。
  • 現在サポートされている形式：ほとんどすべてのオフィス文書（ms + odf）+ djvu + fb2 + chm + pdf（保護を含む）。
  • プロジェクトの1か月のベータテストの後、ほとんど広告に携わっていなかったため、結果は次のようになりました。ユーザー。






当面の計画では：

  • コンバーターを無限に改善します。これは今のところ完全には機能しません。
  • APIシステムにネジ止めします。
  • 「社会性」（コメント、友達、ブックマークなど）を追加します。
  • いくつかのまれなドキュメント形式を追加します。
  • プレゼンテーションにアニメーションサポートを追加します。
  • DocMe.ruを介してドキュメントを販売するためのメカニズムを実装します。
  • Flashをサポートしていないデバイスで問題を解決します。 もちろん、主にアップル製品で。 また、 js + svg +などのビューアが開発されており、何らかの理由でFlashをインストールしていない人でもプロジェクトを使用できます。
  • 最後にデザイナーを見つけて、すべてをやり直します。 あなたが最新のWebインターフェースの開発に対処できる優れたデザイナーであり、時間がある場合は、job @ docme.ruにご連絡ください。 おそらく、あなたはこのプロジェクトで私たちを助けることができます。


コメントで質問や提案があれば喜んでお答えします。



更新：残念ながら、Habra効果は無駄ではありませんでした。 コンバーターを備えた新しいインスタンスは、数千ページのプログラミングマニュアルのプレッシャーの下でロードする時間がありません。 すべてのプロレタリアの責任において、現在の20〜30分とは異なり、通常、平均的なドキュメントの変換時間は最大1分であることを宣言します。
