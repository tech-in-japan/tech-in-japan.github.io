FatCowアイコン時代の日没（色の選択、トップ30およびその他の演算）

アイコンの最終リリース







5年前、ピクセルアイコンのセットを描き始めました。 ファッションはVistaでした。 2013年末までに、FatCowセットは3926個のアイコンに達しました。 これは、その年の春に番号3570 （サイズ：16x16）に落ち着いたフーガよりも356個多いアイコンです。 フーガボーナス（32x32および24x24）の352個のアイコンはカウントされません（リプレイとして）。 しかし、FatCowアイコンのさらなる開発は凍結されています。 残念ながら、4,000のアイコンの丸い数字は表示されません（1か月の作業が残っています）。既存のアイコンの追加サイズのRetinaのリリースは表示されず、予算不足のため開発は停止されました。 網膜などのテストを行うことができました。



はい、これらはメガフォトリアリスティックな3Dではありませんが、誰も必要としないアイコンではありません。これは古いピクセルスクールです。 小さなサイズと色域の制限が栄誉でした。 さらに10年が経過します...そして、ピクセルアイコンは引き続き使用されると信じています。 そして、もしあなたがコンピュータグラフィックスの誕生の時代である古代美術の愛好家なら、画面の後ろにいてください！



インターネット上のアイコンの使用に関する統計[表1]と、 最後の（3.92）アイコンパックに加えられた最新の変更について説明します 。 実際、年々アイコンが広く配布されているおかげで、無料アイコンセットの次の追加の開発のための予算を受け取りました。





表1：アイコン使用統計



World Wide WebでFatCowアイコン（34,863ドメイン）を使用している152,728ページのうち、228ライセンスのみ（0.65％）が単一のライセンス条件を満たし、ソースを参照しています（便利な方法で）。 写真を見つけたように、別の面白いトピックがありましたが、誤検知のシェアは無効になりました。



特定の画像へのすべての参照を検索します（Googleで）



4000個のイメージのグループに対して、イメージを使用するリソースのリストをすぐにコンパイルできる適切なソフトウェアを見つけるために、かなり多くの試みが行われました。 唯一の選択肢は1Cプログラムを検索することですが、このプログラムはそのようなタスクには高すぎます。 車輪を再発明しなければなりませんでした。 Googleフォトサービスを通じてアイコンを検索しようとしました。



全体の問題は、おそらくサイトからのいくつかの要因を考慮していないため、Googleがサイトからではなくディスクからアイコンをプルするように強制することでしたこれはサイズとmd5、幅、高さ、色域、輝度、コントラストなどです。



「任意の」オンラインサービスに写真をアップロードするには、ディスクからまたはURLからの2つの方法があります。 ディスクから透明な背景のGoogle画像検索サービスにアイコンをアップロードすると、（非常に大きい）背景コンポーネントを考慮せずに、描画されたオブジェクトのみが考慮されるため、最良の結果が得られます。 （URLを使用して）サイトからアイコンを入力すると、透明度が失われ、その後、 白い背景色が色の合計質量と混合されます。 ただし、これは単なる仮説であり、誤っている可能性があります。 私たちの意見はその結果から発展しました。 おそらく、Googleの従業員、またはその主題に属する人々が私たちを修正するでしょう。



奇妙なことに、URLとディスクからロードしようとすると、まったく異なる結果が得られました。 （カバレッジの点で）予想される結果に最も近いのは、コンピューターからダウンロードしたときです。 ディスクからストリーム上のブラウザにアイコンをロードするために残りました。 このアクションを実装するには、ブラウザーエミュレーターが必要です。ブラウザーエミュレーターはコンポーネントSelenium webdriverになり、あらゆるもの（php、C＃、Java、Python、VBA、Ruby、Perl、JS）で記述できます。



次に、自動モードで得られたすべての結果を処理する必要があり、 vba +セレンのデザインが選択されました。24時間後に、3926個のアイコンのそれぞれが使用されるサイトの配列の形で結果が得られました。 処理は、サイズ16x16および32x32に対して個別に実行されました。



この配列を処理した後、非常に興味深い統計が得られました。

最新の公開リリースで何が行われましたか？



  • 名前が変更されたアイコン（award_star_bronze_1.png => award_star_bronze_red.png; award_star_bronze_2.png => award_star_bronze_green.png; award_star_bronze_3.png => award_star_bronze_blue.png; temperature_3.png => temperature_normal.png （両方のサイズ）
  • 再構築されたソース：
  • 以前は、197個のファイル、1個のファイル=アイコン付きの20個のレイヤー（ .aiファイルなど ）がありました。
  • 現在：3926ファイル（16x16および32x32バージョンの場合）、1ファイル=アイコン付き1レイヤー（例）。
  • 重複するレイヤーは削除されました。
  • 予想どおり、このような数のファイルのインベントリが開始されると、必然的に不足または紛失したファイルの形で不足が発生し始め、必然的にドラフトが作成されたか、不足している部分が見つかり、再作成されました
  • .aiソース内の名前が変更されたレイヤーはすべて、アイコンの名前に合わせて配置されます（ダッシュ、アンダースコア、タイプミスに置き換えられます）
