モスクワでのJavaユーザーグループ会議

ハブラこんにちは！



12月13日木曜日19:00、モスクワのJava開発者コミュニティと、Oracleの主要エンジニアであるSergey KuksenkoおよびAlexei Shipilevとの最初の会議が行われます。 CROCの会社Gennady Kruglovが、Spring Data JPAについて説明し、彼自身の経験を共有します。 会議はセミナー形式で開催されます。 ライブコミュニケーション、興味深いトピックのディスカッション、コーヒーを飲みながらおもしろい新しい人とおしゃべりする機会など、その場で見つけることができます。



イベントについて少し



会議では、2つのレポートを提示します。



「Spring Data JPAの概要-レビューと個人的な経験」 （Gennady Kruglov）



このレポートは、Javaアプリケーション永続化レイヤーの最新のフレームワークの1つであるSpring Data JPAプロジェクト専用です。 このレポートは、多くの永続レイヤーパターンの概要と、Spring Data JPAの履歴の分析を提供します。 このレポートは、Spring Data JPAの主要な概念の説明と、Spring Dataからの動的なタイプセーフクエリのサポートに基づいています。



レポートの焦点は次のとおりです。

  • パターン「DAO」、「リポジトリ」、および「クエリオブジェクト」。
  • 履歴-JDBCTemplateおよびSpring DAO。
  • Spring JPAリポジトリ-構成、クエリメソッド、トランザクション、監査。
  • JPA基準API-動的タイプセーフクエリ。
  • 仕様-Spring JPAリポジトリでのJPA基準APIの使用。
  • Querydsl SQL-Spring Dataの動的タイプセーフクエリ。


Gennady Kruglovは、CROCの主要な開発エンジニアです。



「ラムダへの棘を介して」 （セルゲイ・ククセンコ、アレクセイ・シピレフ）



Java 5以降で最も重要な言語の変更は、Java 8のラムダ式のサポートになります。この会議は、「内側から」の状況に関するストーリーであるProject Lambdaの技術的側面に専念します。



レポートの焦点は次のとおりです。

  • ラムダについて

    • Javaコンテキストのラムダとは何ですか
    • それらは既存の言語プリミティブとどのように関係していますか
    • ラムダ-砂糖かどうか？
    • jsr292がjavacをどのように破ったか
    • ラムダはどれだけ、どこで、誰に
    • パン、熊手、その他の機能


  • ストリーム（バルク）操作について

    • それは何ですか、彼らは何を食べますか
    • Fork / Joinとは何ですか
    • どのくらい、どこで使用していますか
    • クランペット、バンプ、およびその他の特性プロパティ


  • ディフェンダー（デフォルト）メソッドについて

    • それは何ですか、なぜ私たちは必要ですか
    • 抽象クラスとインターフェースの違いは何ですか
    • 他の面白いトラブル




セルゲイ・ククセンコ -セルゲイはノボシビルスク州立大学を卒業しました。 ロシア科学アカデミー、スタートアップ、IntelやOracleなどの大企業を含む15年以上の専門的な経験。 さまざまなクライアントおよびサーバーアプリケーションから始まり、classlibおよびJITコンパイラーで終わる、14年以上のJavaの経験。 過去6年間、Sergeyはclasslib、VM、GC、JITなどのJavaパフォーマンスの問題に取り組んできました。



Alexey Shipilev -Alexeyは7年以上Javaのパフォーマンスに取り組んでいます。 この間、彼はなんとかIntelのApache Harmonyで働き、その後Sun Microsystemsに移り、次にOracleに移りました。そこで彼は現在、JVM、クラスライブラリ、フレームワーク、アプリケーションのパフォーマンスを含むSun / Oracle JDKに取り組んでいます。 Alexは、Standard Performance Evaluation Corporation（SPEC）のOracleの技術代表であり、産業ベンチマークの開発とサポートを行っています。
