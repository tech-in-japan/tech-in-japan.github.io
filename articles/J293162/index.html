Case Planner 5D：devtodevアナリティクスがARPUを25％増加させる方法



私たちは皆、より多くを稼ぐよう努めています。 ゲームであろうとSaaSサービスであろうと、新しいアプリケーションを開発するとき、より多くのお金を得るという目標を設定します。 そして、分析（特にモバイルおよびWebアプリケーションの場合）は、この目的に役立つツールです。 したがって、 devtodevでは 、新しいクライアントを統合するだけでなく、彼が財務パフォーマンスを向上できるよう努めています。

また、これらのケースの1つについてお話ししたいと思います。 Planner 5Dサービスについて話します。これにより、特別なトレーニングを必要とせずに、詳細なフロアプランと現実的なインテリアデザインを専門的に作成できます。



目標は単純でした。ユーザーARPUを増やすことです（これも必要だと確信しています）。 実際、ARPUの増加（視聴者が同じままである場合）は、同じ割合の収益の増加を意味します。

すぐに予約します。成長率を除き、特定の値には名前を付けません。 まだプライバシー。



まず、devtodevの専門家によってサービスが分析され、サービスの次の機能が明らかになりました。

  1. このプロジェクトでは、SaaSサービスの標準による有料ユーザーの割合が比較的低くなっています。
  2. 特に注目に値するのは、無料のユーザーの大部分が定期的に（1日1回以上）アプリケーションに含まれていることです。 つまり、収益化できる人の割合は非常に大きくなります（全ユーザーの3分の1から半分）。
  3. 生涯の無料ユーザーと有料ユーザーはほぼ同じです。
  4. 価格の大きな違いにもかかわらず、ほぼ同じ数の人が毎週、毎月、年間のサブスクリプションを購入しています。
  5. このプロジェクトにはソーシャルネットワークとの接続がありますが、各ネットワークに入るときにユーザー名とパスワードが必要です。




特定された機能に基づいて、次の推奨事項が発行されました。

  1. アプリケーションの有料使用と無料使用のバランスを調整します。 ほとんどのユーザーは、1日あたりのセッション数が多い場合でも無料版を好みます-無料版の機能が広すぎる可能性があります。
  2. 毎週のサブスクリプションの価格を変更して、より多くのユーザーが有料利用のメリットを確認できるようにします。
  3. 無料のユーザーを有料のユーザーに変換できる割引プロモーションを適用します。
  4. ソーシャルネットワークに自動的にログインする機能を追加します。








推奨事項は推奨事項ですが、決定はサービス担当者に委ねられます。 最終的に行われたもの：

  1. 無料版は、カタログ内の家具の量を減らしました。 あらゆる種類の家具を入手するには、有料版が必要です。
  2. 週単位のサブスクリプションは、2か月のサブスクリプションとともにカタログから除外されました（これまではWebバージョンのみ）。
  3. 毎月のサブスクリプションの価格が引き下げられました。
  4. AndroidおよびWebバージョンの場合、割引クーポンが作成されています。
  5. ソーシャルネットワークに自動的にログインする機能を追加しました（Webバージョンのみ）。
  6. サービスを3日間使用した後、1つのサブスクリプションを購入しなかったユーザーは、広告を表示し始めました。




結果はすぐに来ました：

ARPUは25％増加しました。 そして、これはほんの始まりに過ぎないと確信しています。



この例では、分析は分析用ではないことを示したいと思います。 グラフと数値の背後では、サービスの問題、ボトルネック、機能をいつでも見つけることができ、これらのボトルネックを解消し、問題を解決し、機能を試して最終的に収益を上げる方法に関する推奨事項を作成できます。



これは次の重要なプロセスです。

  • 動作します;
  • 永続的でなければなりません。


私たちの経験がそのような変化を始めるきっかけになれば幸いです。 そして、問題がある場合-連絡してください！
