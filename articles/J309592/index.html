BeelineクラウドPBXサービスの概要

この記事では、BeelineのCloud PBXサービスについて、それが何であるか、システムがどのように機能するか、その長所と短所について説明します。



私は2年前からBeelineの企業テレフォニーに取り組んでいます。 しかし、クラウドテレフォニーはかなり新鮮な製品であり、約2か月前に建設会社のクライアントから連絡があったときに作業を開始しました。 彼のケースでは、Beeline Cloud PBXとは何かをよりよく理解するための記事を始めたいと思います。



この会社には電話に関する鋭い質問がありました。仮想電話交換機がオフィスに設置されましたが、ほとんどの従業員はほとんど常に外出していて、携帯電話通信で仕事をしていました。 したがって、会社の経営陣がユニットの作業を管理することはできませんでした（通話の録音と全従業員の通話に関する統計の要約がないため）。



このクライアントには、クラウドベースのPBXを選択しました。これは、ほとんどの問題の解決策になりました。 会社は、1つの外部マルチチャネル番号8（495）...と、マルチチャネルに関連付けられている必要数の内部番号を取得しました。 これらは固定電話と携帯電話の両方でした。 現在、すべての通話は、路上で行われた通話も含めて、クラウドPBXに記録および記録されます。 これにより、作業を制御し、記録と統計に基づいて適切な決定を下すことができます。 移動時のテレフォニーの問題は、デフォルトで削除されました。





Beeline Cloud PBXとは



したがって、上記のケースからわかるように、Beeline CloudベースのPBXは、加入者が携帯電話番号を介して接続できる仮想PBXです。 したがって、クラウドベースのPBXには、仮想PBXとセルラー通信の両方の機能が含まれています。 唯一の違いは、通常の仮想電話交換では、プロバイダーのWebサイトのアカウントに必要な数の仮想サブスクライバーが作成されることです。クラウドベースのPBXの場合、すべてのサブスクライバーはSIMカードを介して接続されます。 つまり、クラウドPBXで作業を開始するには、SIMカードを購入する必要があります。



クラウドPBXはどのように機能しますか？



クラウドPBXの通信メカニズムを検討してください。 Webサイトや他の広告ソースで、会社はクライアントが呼び出す1つの番号、つまりマルチチャネル番号を示しています。 着信コールでは、音声メニューがクライアントを目的の部門に誘導します。 一般的なマルチチャネル番号からの呼び出しは、携帯電話で適切な専門家に自動的に送信されます。 従業員がどこにいても、オフィス内でも外でも、彼は連絡します。 発信通話では、従業員の内部の携帯電話番号の代わりに、会社のマルチチャネル番号が置き換えられ、クライアントによって「表示」されます。 すべての会話が記録され、クライアントは受話器を取り上げたときに自動的に警告されます。 すべての会話レコードはクラウドに保存されます。



クラウドPBXレート



BeelineクラウドPBXでの価格設定は簡単ではありません。 最終的なコストはいくつかのコンポーネントとそのパラメーターで構成されているため、クライアントにどれくらいのコストがかかるかを特定の数字で指定するのは問題です。 これらのコンポーネントをさらに詳しく考えてみましょう。



サービスパッケージ。 Beeline社は、異なる数の番号を接続し、異なるサービスセット（音声メニュー、通話録音、会議参加者の数、ウィジェット、その他の多くのパラメーターなど）を使用できる5つのパッケージを提供しています。





マルチチャンネル番号 。 価格の2番目の要素はマルチチャンネル番号です。これは、加入者が接続し、他の会社番号がリンクされる番号です。 番号は次のとおりです。



直接市 8（495）...-この数値は、毎月の購読料（1か月あたり500ルーブル）を意味します。 会社が「美しい」番号（銀、金など）を選択した場合、これは1回限りの支払いを意味し、その規模は番号自体に依存します。



連邦8（9XX）...-このような数字は、選択した関税に応じた月払いを意味します。 「美しい」番号の場合、および直接の番号の場合、1回支払う必要があります。



3.加入者番号。 価格の3番目の要素は加入者番号です。これは同じ「SIMカード」、内部加入者番号、マルチチャネル番号のチャネルの1つです。 各番号には、料金プランが設定されており、月額料金は1か月あたり300〜3000ルーブルです。 サービスは事前に提供され、1か月後に適宜支払われます。



異なる数の関税計画については、異なるものを選択できます。 たとえば、1日30分以内に企業通信で通信するマネージャーや、ほとんど常に電話で連絡する営業員の場合、これらは異なる料金プランになります。



加入者番号は、SIMカードとSIPテレフォニーの両方で使用できます。 つまり、「SIMカード」を購入した後、それを延期し、SIPアカウントのみを使用できます。個人アカウントの番号を有効にして、IPテレフォニーにアクセスします。



4.そして、価格の最後の要素は、選択したパッケージ内のさまざまな有料プラグインサービスです。 原則として、これらは既存の機能のフレームワーク内の高度な機能です。たとえば、追加の音声メニュー、追加の通話グループ、追加のエージェントなどです。 ここでの価格の幅は非常に広いです。

Cloud Beeline PBXの関税オプション



クラウドベースのPBXには多くの関税オプションがあります。約30です。これはコールセンター、音声メニュー、ダイヤルアップグループなどです。しかし、システムの絶対的な利点であり、仮想PBXとモバイル通信の機能を組み合わせるものについて説明します。



グループ発信者ID



グループ発信者ID（自動発信者ID）機能を使用すると、ユーザーは共通の会社番号を発信通話の発信者として使用できます（また、異なるユーザーグループが発信通話の発信者として異なる番号を使用することもできます）。



会社に外部番号8（495）があり、接続されている従業員の携帯電話番号があるとします。 従業員が発信コールを行うと、グループ発信者IDのおかげで、コールは市番号8（495）として呼び出されているクライアントによって決定されます...は会社番号です。 つまり、従業員はオフィスにいるのではなく、市からのように連邦政府から発信することができます。



通話録音



BeelineクラウドベースのPBXは、着信コールと発信コールを記録および保存する機能を提供します。 この関数は非常に簡単に接続されます。 アカウントで録音を聞くことができます。



クラウドストレージ



すべての通話記録とファックスはクラウドに保存されます。 レコードの保存に関しては、仮想PBXとはいくつかの違いがあります。 たとえば、Mangoの料金が1分あたりの場合、料金はメガバイト単位です。 各呼び出しは特定の録音ファイルであり、クラウドストレージで一定量を占有します。 理解できるように、1分の録音は約0.9 MB（それぞれ10分、9 MB）です。



デフォルトでは、各料金には1 GBのメモリしか含まれておらず、すぐに終了します。 追加料金で、たとえば月額200 GBあたり1000ルーブルなど、追加の金額を接続できます-これで十分です。 関税を超えないようにするために、一定の保管期間後にファイルの削除を構成できます。



API



APIは、外部システムとのテレフォニー統合を直接（Asteriskなどの追加システムを使用せずに）提供する最も便利な機能の1つです。 このシステムはBroadSoftの BroadWorksソフトウェア製品に基づいているため、このシステムが統合ポイントから提供するすべての機能を備えています。 現在、CRMシステムと統合せずに中規模および大規模企業向けの電話を想像することは困難です。 この場合、統合は最高レベルで提示され、次のような機能を実装します。



  • レコードリンクをコールカードに接続する（Portal API）
  • CRMシステムに発信者のカードを表示し、電話を受ける（イベントサブスクリプション）
  • c2c（クリックして呼び出し）CRMからの初期化の呼び出し（ActionsAPI）


そのため、クラウドベースのBeeline PBX APIには多くの機能があります。 ところで、APIの説明だけが455ページかかります。 私自身の経験から、統合プロセスで問題は発生せず、非アクティブなAPI関数は検出されなかったと言えます。







おわりに



システムの利点のうち、以下を強調します。



  • モバイル加入者はPBXで接続および制御されます
  • 携帯電話からCRMシステムにデータを転送することが可能です
  • GSM接続を十分に使用するために、電話をかけたり受けたりするためにインターネットに接続する必要はありません。
  • 加入者は内部通信を介して無料で通信できます。この通信は無料です
  • システムインターフェイスは非常に便利で快適で、必要なすべての設定を提供します
  • 通信の品質は、加入者へのインターネットチャネルの品質に依存しません


システムには欠点もあります：



  • 私の意見では、最大のマイナスは、加入者を追加するには物理SIMカードを追加購入する必要があるということです。
  • APIドキュメントは英語でのみ利用可能であり、決して使用しない可能性が高い機能の説明でオーバーロードされています。
  • 紛らわしい価格


私の意見では、Beeline Cloud PBXサービスは、モバイルワークステーションを備えた企業にとって最適なソリューションであり、最も広範な統合オプションにより、ほぼすべてのCRMに接続することができます。
