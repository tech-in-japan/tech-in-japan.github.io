google-tableインターフェースで最適なフライトを見つける方法

開始する



私はよく旅行して航空券を買います。

また、航空券のアグリゲーターのインターフェースに苦しむこともよくあります。

チケットを選択して購入することはできません。







ケース



私はどこに飛びたいかを正確に知っています。

私はおよそ何日か知っています。

そして、たとえば、今月の日付について。







今回、私はセアファウスに行きたかった-これはオーストリアのスキー村です。

1月5日より早く飛行せず、1月15日より遅く飛行します。 6〜8日間。







次に、これらの条件に最適なチケットを見つける必要があります。

私の場合の「最良」とは、客観的な価格とフライトの主観的な利便性の組み合わせです。出発/到着時間、出発空港、航空会社などです。







チケットアグリゲーターでのケースの解決方法



  1. Googleマップを使用して、セルファウスの村を3つの最寄りの空港（ミュンヘン、インスブルック、チューリッヒ）に変身させています。
  2. 航空券のアグリゲーターに行きます。 「低価格カレンダー」のようなツールを見つけました。
  3. 3つの空港の3つのタブで開きます。
  4. 1月を選択します。
  5. 私は1月の毎日に出かけ、出発/到着の時刻とチケットの価格を確認し始めます。
  6. チケットが気に入ったら、実際にチケットがあることを確認するためにそれを突く。 キャッシュのため。
  7. チケットが好きな日を覚えるには、ノートまたはテーブルに書きます。


最後の3つの段落で、私は苦しんでいます。

私が突くとチェックしたくないので。 タブを切り替えたくありません。

出発時間と価格が記載された本当に手頃なチケットを見て、そのうちの1つを選びたいです。

そしてそれを買う。







しかし、最終的には、心理的に快適な時間に適切なチケットを見つけることができません。

そして、購入することなく、私は去ります。







ケースの決定方法



  1. アグリゲーターの「Where」フィールドで「Serfaus」を示します。
  2. おおよその日付を選択します。
  3. 適切な期間を選択します。
  4. 購入するチケットのリストを取得します。
  5. 適切なチケットを選択して購入します。


...アグリゲーターにはこのようなシナリオはありません。







考え



  • アグリゲーターは、私のスクリプトの使用方法を知りません。
  • 2017、web 3.0、* asas'y、マイクロサービス、そしてまもなく火星へ。
  • チケットを検索するためのAPIは100万個あります。ブラックジャックについて説明します。


ソリューションを検索する



30分間のグーグル検索では、2017年にはチケットを検索するための無料のAPIがないことが示されました。

アグリゲーターはありません。







  • しかし、結局のところ、多くのアフィリエイトプログラムがあります。
  • はい、たくさんあります。 しかし、彼らはAPIへのアクセスをまったく許可しません（バナー/ウィジェットを配置し、Webサイトにトラフィックを注ぎます）、またはAPIへのアクセス要求をモデレートする不透明な手順の後に許可します。

    最大のサービスの1つはそう書いています-アプリケーションに記入して、4週間考えてから答えます。


Googleには利用可能な唯一のAPIがあります。 QPX Expressと呼ばれます。







解決策



QPXとGoogle App Scriptを使用して、Googleテーブルへのアドオンが70分で作成され、飛行する必要がある場所から読み取り、チケットを検索し、すべてのオプションと価格を書き込みます。







結果は次のようになります。







テーブルの上部で、どこで/どこで（IATA空港コードをコンマで区切って）記入し、どの日付範囲で飛行させたいかを記入します。 旅行に費やす準備ができている日数をコンマ区切りリストで示します。

シートの後半では、価格でランク付けされた航空券のリストという結果が得られます。

1枚のシートには、空の旅に適したすべてのオプションがあります。

組み込みのフィルタリングツールを使用して、必要な日付/時刻を選択できます。







航空会社のウェブサイトにアクセスしてチケットを購入することは残っています。







マイナス決定



ソリューションのマイナス点は、APIへのアクセスにお金がかかることです（1日あたり最初の50リクエストは無料、1リクエストあたり0.035ドル）。したがって、すべての人にマイクロサイトを作成することはできませんでした。







SerfausをWhere Toフィールドに入れることはまだできません。 これはGoogle APIの1つで解決されると思います。
