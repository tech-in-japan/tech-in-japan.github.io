顧客として、彼は執行者のままでした。 パートII

まず第一に、私のプロジェクトの趣味について最初の投稿（ここ）に回答してくれたすべての人々に再び多大な感謝を捧げます。 あなたの推薦の多くは、このストーリー全体を継続するための基礎を形成しました。



いくつかの不可解な理由で、そして出版のほぼ直後に、奇妙な声が私の頭に現れましたが、それは繰り返し続けました。 そして、あなたがいつもやったことをしてください！」 しかし、彼らはカンフーを理解していませんでしたが、確かにいくつかのトリックを理解していませんでしたが、彼らはまだ疑いと怠inessの悪魔に対処することができました。



初心者として私を悩ませた最初のことは、失望さえしていませんでした-ハブでの公開の翌日、「ベータ」という見出しの下で実質的に同様のプロジェクトが登場しました パターンかランダムか？



これについて落ち込んでいないために、私はこれがまだ規則的であると決めました。私は、おそらく私がプロと競争しているので、私は名誉になります。 さて、この考えで、私はプロジェクトを完成させ始めました。



目標


ITのすべてを合理的な程度の不確実性で明確に計画する必要があるという事実にすでに慣れていたので、私は同じ計画から始めました。 そして、それは必要でした：

  • 登録とログインをソーシャルネットワークと統合します。
  • デザインを考え出す;
  • 完全なAPIを実行します。
  • どういうわけか、サイト全体のテキストコンテンツを削減します。
  • ユーザーにとってより便利なこと。


上記の不確実性に「何か」、「なんとか」、「何か」を書きます。



ソーシャルログイン


おそらくこれは、大多数のハブロビテ人によって表された主な願いです。 最初の段階では、この機会を実現するのは非常に困難に思え、さまざまなテクニックを学ぶのにほぼ1日かかりましたが、実装自体は30分しかかかりませんでした。



Loginzaサービスを使用して目標を達成しました。 そして、すべてが準備できたとき、私はなぜ、初心者でもすべてが非常に簡単である場合、Runetのライオン部分（フォーム、チャット、トラッカーなど）がまだ登録が必要なのか真剣に考えましたか？ 怠azineやその他の考慮事項？



いずれにせよ、ソーシャルネットワークを介した登録の導入の結果、ユーザーのログインの一意性が拒否されました。 エントリメカニズム自体ではなく、この要件が機能するのにかなりの時間がかかりました。



設計


かなり矛盾した（私の意見では）コメントがプロジェクトの設計に寄せられました。 私自身も色合いを区別できないことがあり、私の環境にはWebデザイナーがいないという事実を考慮して、外部から誰かを探すことにしました。



私は地元の（都市）フォーラムで数人と対話を始めましたが、残念なことに、それは何にも終わりませんでした。 この理由は、たとえばWebデザインへの標準的なアプローチの代わりに「フライング」ウィンドウを使用したことです。 しかし、私は窓のアイデアが好きで、本当にそれを拒否したくありません。



一般に、単一のセクターであっても、プロジェクトに請負業者を引き付けることができませんでした。 もちろん、あまり申し出なかったかもしれませんし、自分自身について想像しすぎていたかもしれません-尋ねたとしても、失敗したパートナーに尋ねます。



その結果、設計に関する作業は、JQueryに基づいて作成された既存の「エンジン」を徹底的に改良することになりました。 それがどうなったのかはあなた次第です。



API


APIが並行して開発されているという事実にもかかわらず、私は多くの場合、同じLoginzaまたはInfoBipを使用してこれらのメカニズムをテストしたという事実のため、多くのやり直しと修正が必要でした。 一般的な結論は簡単で明白です。開発者は、快適で、快適で、温かく、理解できるはずの同じユーザーです。



おそらく、APIの完成とテストが最初の公開の瞬間から大部分の時間を費やし、機能の詳細な技術的な説明のために数日を数えていませんでした。



ちなみに、説明は44枚（A4）でした。 しかし、この種の仕事を初めて行ったので、手元にある例から始めることにしました。 そして、奇妙なことですが、ブルジョアのInfoBipサービスの説明は、テキストの量以上がいくつかの機能を噛むために与えられる最も適切なものであるように思えました。 しかし、結果として、私のような小さな、素朴なハリネズミでさえ、それを理解することができます。



要約する


2週間強で、メインの仕事から自由な時間があったため、次のことができました。

  • 独自に設計を改善し、「エンジン」自体を修正します。
  • 起動API
  • サイトテキストを25％から35％削減します。
  • ユーザーに少し便利を追加します。
  • ソーシャルネットワークを介したログインと登録を台無しにします。
  • 一意のログインを拒否します。
  • 小さな厄介なバグの塊を見つけて、以前は赤目で怖がっていた忍耐力でそれらを破壊します。


私はすべてに何度も関わったことを後悔していません。自分自身のために何度も何度も結論を出しました。新しいことはすべて常に興味深いものであり、新しい知識は常に有益です。 そして、この投稿がアイデアを持っているが、それを実現できないと信じている人々によって読まれたなら、あなたはただ始めれば、それは確かにあなたと共に成長するでしょう。

PSまたはマーシャについて


多くのコメント、またはむしろ質問がマーシャについて来ました。 PMには数通の手紙さえありました。 もちろん、私は間違いなく世界の写真を美しくするという事実にもかかわらず、マーシャの写真をアップロードしません。 または多分私ではない。 結局のところ、誰でも彼女を招待することができました。



いずれにせよ、私の同情は相互ではありませんでしたが、とにかく、私にとって新しい分野でのミューズと貴重な経験をしてくれたマーシャに感謝します。
