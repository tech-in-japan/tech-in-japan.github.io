Windows Server 2003上のWD Advanced Format

ディスクがWinXPのAdvanced Formatトポロジで動作するためには、ディスクを調整することが望ましいです。これらは公式の推奨事項とテスト結果です。 XPでは、WD Webサイトにユーティリティが投稿されています。 Vista / 7/2008 Serverにも問題はありません-Advanced Formatはすぐにサポートされます。 Win2003に関しては、WDまたはMSからのAF付きディスクの使用に関する公式情報はありません。 さらに、パーティションアライメントユーティリティ（WD Align）は、Win2003では動作しません。



この小規模な調査の目的は、Windows Server 2003で実行されているAdvanced Formatテクノロジを使用してディスクパーティションを調整する必要性を実験的に確認することでした。 ネコの下には、少しの理論、実践、結論、指示があります。



1.理論



1.1。 高度なフォーマット


WD Advanced Formatテクノロジーは、ディスクの物理レベルで4kbのいわゆる「ロングセクター」を使用することで構成されています。 従来のセクターサイズは512バイトです。 最新のOSは4kbセクターで動作しますが、WinXPとWin2003はそれらに関連していません。 後方サポートのために、cなWDはハードウェアエミュレーションを行いました。ドライブの「内部」では、情報は4kbセクターに保存されますが、OSの「外部」では512バイトセクターを介して動作します。

テクノロジーに関する詳細は、たとえばhere 、 here 、 hereで見つけることができます 。 私は小さなスクイーズを紹介します。







このエミュレーションの最初の問題は、512バイトセクターを書き込むときに、ディスクが4kセクターを読み取り、512バイトセクターを変更し、ECCを再計算し、変更した4 KBセクターを書き戻す必要があることです。 しかし、WinXP / Win2003でドライブをフォーマットすると、この問題はバックグラウンドにフェードインします。この場合、パーティションの開始は物理4kbセクターの境界と一致しません。 このフォーマット機能により、作成されたパーティションは512バイトのセクター1つ分シフトされます。







その結果、オペレーティングシステムを照会する場合、場合によっては、ドライブは1つの物理4kbセクターではなく、2つの隣接するセクターで動作する必要があります。 もちろん、これはパフォーマンスを低下させます。



この問題の解決策として、WDは、パーティションのセクターを物理セクターの境界に沿って整列させることを提案しています。 このために、ハードウェアとソフトウェアの2つのソリューションが提案されています。 ハードウェアソリューションは、ハードドライブに7-8ジャンパーを取り付けることです。 一般的なアクセスのためにセクションをプログラムで調整するために、 WD Alignというユーティリティが公開されています。



ハードウェアソリューションにはいくつかの制限があります。ディスクが複数の（> 1）パーティションに分割されている場合、WDは通常の動作を保証しません。 さらに、ハードウェアソリューション（ジャンパー）は、ユーティリティでもオペレーティングシステムでも表示されません。 そのため、理論的には、最初にジャンパーを挿入し、次にプログラムで調整する（または最新のOSでパーティションを作成する）ことができます。その結果、「ダブル」シフトが発生します。

これらの理由により、ソフトウェアの調整については後で説明します（ただし、ハードウェアもテストします）。



1.2。 WD AlignとWindows Server 2003


上記のソフトウェアセクターアライメントのリンクで、WDはPar​​agonとAcronisの2つのユーティリティを提供しています。 Paragonのバージョンは、一般にWin2003にインストールすることを拒否します（そして、ちなみに、正直なところ、サポートされているオペレーティングシステムにはありません）。 AcronisのバージョンはWin2003にインストールされますが、同時に、ドライブはそれを「Western Digital Advanced Formatディスクではない」と定義し、パーティションを「最適に配置されていないパーティション」と定義します。 この状況では、パーティションをユーティリティに合わせることができません。



Windows Server 2003のAdvanced FormatテクノロジのサポートやWD Alignに関する公式の声明はありません。 理論的には、Win2003にはWinXPとほぼ同じカーネルがあります。 だから、彼らはディスクで動作するはずです（ほぼ？）同じように。 これをテストするために、可能な「ほぼ」パフォーマンステストが実行されました。

3.レベリング手順



理論部分で既に述べたように、Win2003でWD Alignユーティリティを使用しても機能しません。 したがって、別の環境でアライメントを実行し、Win2003を使用する必要があります。 選択肢が提供されます：

-通常のWindows Server 2003ツールを使用してパーティションを作成し、ブートディスクからWD Alignを調整します。

-通常のWindows XPツールを使用してパーティションを作成し、同じXPでWD Alignを調整します。

-通常のWindows Vista / 7/2008ツールを使用してパーティションを作成します。

-Win2003にインストールできるように、ParagonからのWD Alignインストーラーの変更。



ほとんどの場合、Win2003を使用すると、最初のオプションはよりシンプルで信頼性が高くなります（他のオプションは個人的にチェックしませんでした）。 したがって、ブートディスクオプションの詳細を説明します。

0.ジャンパーを挿入した場合は、ジャンパー7-8を取り外すことを忘れないでください。

1. アクロニスから WD Alignをダウンロードしてインストールします（ こちらをご覧ください ）。

2.オプションのWD Align Bootable Media Builderユーティリティを実行し、ブートディスクを作成します。

3. WD Alignが後でハードドライブを表示するには、初期化してパーティションを作成する必要があります（クイックフォーマットでフォーマットできます）。

4.作成されたブートディスクから起動し、パーティションを調整します（ユーティリティはディスクをWD Advanced Formatとして識別する必要があります）。

5. Win2003を再起動します。 確認するには、WD Alignを実行します。「最適に配置されたパーティション」アイコンがパーティションの反対側にハングします。 ドライブの反対側には、「Western Digital Advanced Formatディスクではありません」というアイコンが残りますが、何もする必要はありません。
