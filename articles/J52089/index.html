Xen。 Dom0としてのFedora 10

定期的に作業をサポートするFedora部族の最新バージョン

Dom0モードでは、番号は8でした。Xenはバージョン3.1.2で、Dom0のカーネルは2.6.21でした。

すべてのソフトウェアの新しいバージョンが欲しい すべてがすでにかなり古くなっています。

主なことは、XEN自体が古いことです。現在の安定バージョン番号3.3.1の日付は2009年1月5日で、そこには多くの利点があります。



Fedora 9/10はDom0モードでの操作をサポートしていません。つまり、対応するカーネルがありません。 どうする 私のニーズを完全にカバーするかなり簡単な方法を見つけました。



だから。



必要に応じてFedora 10を配置します。

（個人的には、すべてのジャックドーを削除し、カスタムに移動して、すべてを削除します。）

3.3.0があるので、xenを配置します。 また、binutilsも必要です。arアーカイバがあり、これは便利です。

yum install xen binutils









... Debian LennyからDom0サポート付きのカーネルをドラッグします。

コア モジュール 。



必要なものを選択して、それを所定の場所に配置します。

ar x linux-image-2.6.26-1-xen-686_2.6.26-13_i386.deb data.tar.gz

tar xzf data.tar.gz ./boot

ar x linux-modules-2.6.26-1-xen-686_2.6.26-13_i386.deb data.tar.gz

tar xzf data.tar.gz ./lib/modules ./boot

mv -t /boot ./boot/*

mv -t /lib/modules ./lib/modules/*









Fedora Magicの少し。

/sbin/new-kernel-pkg --mkinitrd --depmod 2.6.26-1-xen-686









grub.confに移動します。 愚かなFedorinスクリプトがすべて間違って記述された方法を調べます。

すべてを修正する必要があります。 以下は私のオプションの前後です。

宛先：

title Fedora (2.6.26-1-xen-686)

root (hd0,2)

kernel /vmlinuz-2.6.26-1-xen-686 ro root=/dev/vgr5/f.root rhgb quiet

initrd /initrd-2.6.26-1-xen-686.img







後：

title DebXendora (2.6.26-1-xen-686)

root (hd0,2)

kernel /xen-3.3.gz

module /vmlinuz-2.6.26-1-xen-686 ro root=/dev/vgr5/f.root rhgb quiet console=tty0

module /initrd-2.6.26-1-xen-686.img









リバイ。 喜ぶ。



脅威。 3.3.1へのXENアップグレードはまだ来ていませんが、3.3.0はすでにプラスになっています。
