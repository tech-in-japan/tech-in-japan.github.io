何であれデータの安全性。 壊滅的なAzure Packインフラストラクチャクラウドバックアップ

InfoboxCloudからAzure Packインフラストラクチャクラウドに仮想マシンの災害対策バックアップのサポートが統合されたことをお知らせします ！ この機能により、 データセンターが物理的に破壊された場合でも、仮想マシンのバックアップが確実に保持され、再構成せずに復元できます。









なぜ耐災害バックアップが必要なのか、どのように機能し、どのくらいの費用がかかるのかを見てみましょう。



問題は何ですか？



データ損失は、ITインフラストラクチャで発生する可能性がある最も重大なインシデントの1つです。 企業の数十年にわたって蓄積されたデータ、国内または国際規模のサービスのデータ、重要な会計データ、最終的にはオンラインストアまたはウェブサイトからのデータのみ-それらの損失は容認できません。









ローカルバックアップテクノロジーは、人為的および自然災害、テロ攻撃、火災など、最高のデータセンターのレベルでさえ発生する可能性のある問題から保護しません。



ストレージシステムのファームウェアで、データの損失につながるエラーが検出される場合があります。



クラウドプラットフォームソフトウェアで重大な障害が発生し、データが失われる可能性があります。 現在、多くの人々が、ハードウェア障害ポイントのないハイパーコンバージドフォールトトレラントデータウェアハウスについて話しているため、ソフトウェア障害ポイントがあります。



最終的に、顧客の管理者は、誤ってまたは意図的に仮想マシンとすべてのバックアップを削除する可能性があり、顧客のビジネスに回復不可能な損害を与える可能性があります。



1つのデータセンターと1つのクラウド管理プラットフォームも障害ポイントですが、多くの人はそれについて話すことを好みません。



どうする



データセンターという形の障害ポイントの問題を解決するには、仮想マシンを別のデータセンターに複製するか、別のデータセンターにバックアップコピーを作成する必要があります。 万能薬はありません。どのアプローチにも長所と短所があります。



別のデータセンターのジオレプリケーションおよび災害復旧サービスは、Azure Packインフラストラクチャクラウドのお客様が既に 30秒以上のレプリケーション頻度で利用できますが、これについては次のいずれかの記事で詳しく説明します。



ジオレプリケーションが役に立たない場合 ユーザーソフトウェアまたはユーザーの操作によって仮想マシン内でエラーまたはデータが失われた場合。 たとえば、DBMSが仮想マシンでクラッシュし、データの一部が破損したり、エンタープライズファイルアーカイブの一部が誤って削除されたりします。 これはすぐには検出できません。 ジオレプリケーション中に、仮想マシン内のすべての変更が別のデータセンターに正常に複製され、データが失われます。



この記事でAzure Packインフラストラクチャクラウドから別のデータセンターへの災害対策バックアップについて説明すると、仮想マシンの最後の状態だけでなく、以前の状態も復元するのに十分なバックアップがあります。 バックアップは、1日に数回から目的のスケジュールで作成し、何年も保存できます。 バックアップ自体の損失の状況は受け入れられないため、バックアップデータセンターにも地理的に分散して保存されます。



お客様の管理者に地理的に分散したバックアップを削除することはできません。このような重要なデータを使用したアクションは、InfoboxCloudサポートチームへのリクエストがあった場合にのみ可能です。 このようなアプリケーションの応答時間は1時間以内に保証されます。 バックアップの削除を要求した場合、それらは自動的に削除されません：契約で指定された顧客がこのアクションを実行し、以前に合意した期間の後にバックアップを削除することを確認し、これが発生したときに顧客に通知します（削除要求をキャンセルできる場合、攻撃者によって作成されます）。 アクションをログに記録した、InfoboxCloudの繰り返し確認された少数の従業員のみが、バックアップを削除する権利を持っています。



接続方法



ユーザーは、Azure Pack Infrastructureクラウドの災害対策バックアップサービスに接続するために、サービスコントロールパネルにリクエストを書き込みます。 バックアップのすべての詳細に同意します。バックアップの頻度、バックアップの保存期間、削除要求後の削除の待機時間、必要な仮想マシンなどです。また、バックアップを地理的に分散してローカルに保存し、高速リカバリを実現することもできます。



いくらですか？



サービスの固定費は顧客と合意されます（すぐに、最も一般的なシナリオの計算機で自分で計算できるようになります）。



なぜコストが固定されているのですか？ ロシアの大企業の会計士が「あなたがどれだけのリソースを使用するか」を支払うのは都合が悪いため、財務計画のために定額の支払いが必要です。 慣例が示すように、固定支払いはPay As You Go支払いよりも収益性が高い場合があります。



推定コストのいくつかの例：



  • 仮想マシンディスクのボリュームは30 GBです。 バックアップの作成：毎日。 ストア-30日 （ 30コピー ）。 費用は1ヶ月あたり871ルーブルになります。 オンラインストアの所有者にとってはかなりのお金です。



  • 仮想マシンのディスク容量は90 GBです。 バックアップの作成：毎日。 ストア-2か月 （ 60コピー ）。 費用は1ヶ月あたり2623ルーブルになります。 この金額は、企業の経理部門全体の損失を引き起こす可能性のある問題と比較して重要ではありません。



  • ディスクのボリュームは300 GBです。 バックアップの作成：毎日。 ストア-3か月 （ 90コピー！ ）。 費用は1か月あたり8365ルーブルになります。 しかし、会社のすべての取引で200人の従業員のCRMデータが失われる可能性があります（8365/200 =従業員1人あたり月額41ルーブル）。 このような長い保存期間により、数か月前に誤って削除された取引を復元できます。


各例は、実際の顧客によるテストサービスに基づいています。 重要度の低いデータについては、災害対策バックアップの作成頻度と費用を削減できます。



これが高価に思える場合は、データ損失のコストを検討する価値があります。 SLAが保存されない（SLA Azure Packインフラストラクチャ99.982％）。これは、SLAに違反した場合に確実に金銭的報酬を受け取るためですが、失われたデータのコストに見合わず、一部のデータを回復することは不可能です。



どのように機能しますか？



適切なタイミングで、バックアップはMicrosoft System Center Data Protection Managerによって作成されます。









作成時には、バックアップの整合性チェックが必須です。 1つのローカル完全バックアップは常に保存され（価格に既に含まれています）、顧客との合意により、より多くのバックアップをローカルに保存することができます。



ローカルバックアップからの復元は、ほぼ瞬時に行われます。









作成されたローカルバックアップ（vhd / vhdx）は、地理的に分散したストレージ（GRS）で北ヨーロッパのMicrosoft Azureグローバルクラウドデータセンターに複製されます。



スケジュール


バックアップを作成した正確な時刻で、毎日最大2つのバックアップを作成することができます。 週単位のバックアップを選択する場合、頻度（1週間に1回）、週の曜日、時間を1日に2回まで選択できます。 顧客タスク用の非常に柔軟なバックアップ設定により、月次および年次のバックアップが可能です。









いくつのバックアップが保存されていますか？


日単位、週単位、月単位、および年単位のバックアップのバックアップ保存期間を個別に設定できます。 各バックアップの最大保存期間は9999日です。 より長い保管期間の要求はありませんでした。









回復


必要なvhd / vhdxチェックポイントから復旧する場合、仮想マシンファイルはAzure Storage GRSまたはローカルバックアップからコピーされ、既存の仮想マシンのハードドライブの代わりに、または新しいドライブとして使用できます。 仮想マシンのIPアドレスを保存できます。 ユーザーがコントロールパネルで仮想マシン自体を削除しても、それを復元しても問題はありません。 このディスクを別の仮想マシンに追加として接続し、データ全体または選択的にデータのみを復元することもできます。



適合性



ディスクとしてvhd / vhdx仮想マシンを使用することは利点です。 これらのディスクをクラウドで使用できるだけでなく、必要に応じてローカルでHyper-VサーバーまたはWindows 10のHyper-Vに接続できます。したがって、仮想マシンとバックアップの所有者だけでなく、データの所有者でもあります。 ：必要に応じて仮想マシンのディスクとバックアップをいつでも取得し、サーバーまたはローカルコンピューターに接続できます。これは、企業の技術的独立性を確保するために重要です。 彼らは最良の方法あなたのビジネスを成長させるための最も費用対効果の高い方法を可能にする、あなたの問題を解決することを目。
