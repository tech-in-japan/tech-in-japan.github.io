こんばんは、前の投稿を書いてからおそらく3週間以上が経ちました。それから私の方向性が少し変わり、MVCは良いパターンですが、今ではjsにとっては面倒です。 私たちは、すべてがどのように機能するかを見ることができる、より透明で微妙なソリューションを選択するよう努めています。それがjQueryが好きな理由です。ブラウザのタイプを無視できる最小値を導入しますが、 js。 そのため、jsにDocument-Viewパターンを実装しようとするとき、できるだけ微妙で目に見えないようにしたかったのですが、同じjsであるという同じ軽い感覚を得ることができたようです。

WinFormsと同様に、jsFormsではすべてがコンポーネント上に構築されます。 それでは始めましょう。



成分

コンポーネントは2つの部分で構成されます。

  • ビジュアル
  • 機能的


視覚部分



ビジュアルパーツは、HTMLマークアップを含み、プロパティとイベントを設定するテンプレートです。

テンプレートは、次の形式のjsファイルです。

jsForms.Templates.ComponentType='HTML';







  • ComponentType-コンポーネントのタイプ
  • [HTML]-引用符で囲まれたHTMLコード


重要 -HTMLコードでは文字「<」および「>」を使用してタグをフレーム化し、他の場所では「＆lt;」を使用します および＆gt; それに応じて。

HTMLタグは非常に厳密である必要があります。つまり、すべての開いているタグを閉じて、順序に従う必要があります。

テンプレートには、ルートhtmlタグを1つだけ含める必要があります。

機能部



機能部分は、コンポーネントのプロパティとイベントを宣言するだけでなく、「クラス」を含むjsファイルでもあります。

機能部分の形式は次のとおりです。

jsForms.Components.ComponentType=<br/>

{<br/>[JavaScript]<br/>};<br/>







  • ComponentType-コンポーネントのタイプ
  • [JavaScript]-通常はクラシックJS

コンポーネントのプロパティを設定するには、プロパティタグを使用します。

物性



プロパティタグには、コンポーネントの指定されたすべてのプロパティが含まれ、列挙はキータグを使用して行われます

キータグの形式は

< key name ="PropertyName" [ value ="PropertyValue" ][ type ="object|string|int|float..." ][ disable ][ direct ][ const ] > [PropertyValue] </ key > <br> <br> * This source code was highlighted with Source Code Highlighter .





  • PropertyName-プロパティ名、必須パラメーター
  • PropertyValue-値はvalue属性またはキータグ内で指定する必要があります。属性は優先順位です
  • type-値のタイプ、オプションのパラメーター
  • disable-属性はプロパティが設定されないことを示します
  • direct-属性は、プロパティがプロパティの設定プロパティをバイパスして設定されていることを示しますが、現在は無視されます
  • const-この属性は、多言語の将来のサポートのために予約されています。
