.BATのために複数の.bat、マルチプレイヤー間の相互作用



BATファイルは、相互にネットワークを介して任意の有用な情報を送信することはできません。



うまくソケットない、といいますが、名前のデータ領域を持つ簡単な仕事があり、すなわちん ファイル。 一般的に極限まで簡略化さ1行のテキストファイルを作成し、読み。 エコーテキスト> file.txtをfile.txtをを実行するのに十分であると«テキスト»ラインが含まれています。 次のようなファイルから行を読み取ることができます。set / p var = <file.txt（実行後、var変数には文字列「Text」が含まれます）。



そのため、ファイルを介してバッチファイル間で情報を転送できます。 また、これらのファイルが共有ネットワークリソースにある場合、異なるコンピューターで実行されているバッチファイルは互いに通信する可能性があります。



ここでは、クライアント - サーバーアプリケーション、貧しい人々のために、このようなrexecの声明の小さな一例です。



ファイルrexec_server.bat： 

set checkFolder=\\IMP5\PUBLIC_RW :loop @ping -n 2 127.0.0.1 > nul @if not exist "%checkFolder%\!cmd" goto loop @call "%checkFolder%\!command.bat" > "%checkFolder%\!result" @del "%checkFolder%\!cmd" @goto loop




ファイルrexec_client.bat： 

 @echo off set checkFolder=\\IMP5\PUBLIC_RW :mainloop set /pc=^> if "%c%"=="exit" exit if "%c%"=="quit" exit echo %c%>"%checkFolder%\!command.bat" echo.>"%checkFolder%\!cmd" :waitloop ping -n 2 127.0.0.1 > nul if exist "%checkFolder%\!cmd" goto waitloop type "%checkFolder%\!result" del "%checkFolder%\!result" goto mainloop




作業の例を次に示します。





以下を実行します。rexec_server.batは、クライアントからのコマンドラインを待機し、実行中のコンピューターで実行します。 彼は、共有フォルダー"\\ IMP5 \ PUBLIC_RW"を介してクライアントと通信します（もちろん、試してみたい場合は、自分のものに置き換える必要があります）。 クライアントとサーバーが現在書き込まれているファイルを事前に読み取ろうとしないように、情報が既に完全に保存されたことを示すだけのフラグファイルが提供されました。



まず、.batのマルチプレイヤーゲーム「tic-tac-toe、3行連続」です。
