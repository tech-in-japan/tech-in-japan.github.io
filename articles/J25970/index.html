Ubuntu + Aptana = ... Aptanaのインストール

こんにちは。

私は約束を果たし、Kubuntu 8.04へのAptanaのインストールについて説明します。

アプタナとは何ですか？

公式Webサイトで、開発者は以下を宣言します。

「Aptana Studioは、WebページおよびWebアプリケーション向けの強力な開発環境です。 JavaScript、Ruby on Rails、Ajaxを使用したPHP、DOM、HTML、CSSの開発がサポートされています。 さらに、Adobe AIRおよびApple iPhone用のアプリケーションを開発できるプラグインが提供されます。

高度なWeb開発者向けに、Aptana Studio Proは製品製造の追加の利点と優れたサポートを提供します。

Aptana Studio 1.1には、Aptana-Aptana JaxerのAjaxサーバーバージョンも組み込まれています。

美味しいですか？

ウェブ上には同様の指示がたくさんありますが、 ロシア語では見ていません。

だから、厳密に判断しないでください、スレッド上の世界で...

それでは始めましょう。

前文：ターミナルのスナップショットは既に重要なので、私は薬覚を使用します。

だから。

公式サイトからAptanaをダウンロードしてください 。

ダウンロードの進行中に準備してください。

1. Javaがインストールされていない場合-いくつかの追加機能とともにインストールします。

sudo apt-get install sun-java6-jre sun-java6-plugin sun-java6-fonts









Maniakの同僚からのUPD：

コンソールにaptanaをインストールした後、実行する必要があります

sudo update-alternatives --config java







表示されるメニューで選択します

/usr/lib/jvm/java-6-sun/jre/bin/java







そうでなければ、良いIDEで快適に作業する代わりに、非人間的なブレーキをかけられます。



2.ほとんどの人がすでにFirefox 2.0.0.xを持っていると確信しています（個人的に2.0.0.14を持っています）。

そうでない場合、問題ではありません。コンソールでコマンドを実行します。

sudo apt-get install firefox-2









「デュース」の理由の説明：3番目のベータ版をインストールしましたが、非常に不安定でした。

何らかのストレス（ブレーキ、Gmailでインターフェースの不具合などが発生）。

3. Aptanaがマシンに正常にコピーされたことを確認します（約73 Mb）。

フォルダーを作成します。

sudo mkdir /usr/local/aptana





アーカイブを解凍します。

実際、これでインストールは完了しました:)

4.次に、スクリプトのデバッグをシャープにします。 インターネットの腸は与えました

オプションでAptanaを実行するための素晴らしいソリューション：

4.1。 フォルダー/ usr / local / aptanaに拡張子.shのスクリプトを作成します（例：runAptana.sh）

sudo nano /usr/local/aptana/runAptana.sh





4.2。 次の情報を書き込みます。

#!/bin/bash

export MOZILLA_FIVE_HOME=/usr/lib/firefox

/usr/local/aptana/AptanaStudio







保存:)





4.3。 スクリプトを「実行可能」にします。 これを行うには、コンソールに入力します：

sudo chmod a+x runAptana.sh







joedmからのUPD：

実行するスクリプトでは、さらに追加する必要があります

export LD_LIBRARY_PATH = $ MOZILLA_FIVE_HOME：$ LD_LIBRARY_PATH

xulrunnerをインストールして./xulrunner --register-globalを登録するか、MOZILLA_FIVE_HOMEを設定する必要はありません。


5.次に、将来のスクリプトの効果的なデバッグのために、FirefoxをAptanaに接続します。

5.1。 コンソールから、Firefoxプロファイルマネージャーを呼び出します

sudo firefox-2 -P





AptanaDebugなどの新しいプロファイルを作成します





5.2。 新しいプロファイルでFirefoxを起動します。

sudo firefox-2 -P AptanaDebug





この製品の開発者を使用する：生活をはるかに簡単にする拡張機能をインストールする

（開発およびデバッグ）Firefoxの場合：

Firebug.xpi

純粋な魂で、Firefoxを閉じます。

6. Aptanaに戻る：デスクトップに起動用のショートカットを作成し、起動するコマンドとしてAptanaを呼び出して、次のように記述します。

/usr/local/aptana/runAptana.sh



、

そして美しさのために-私たちはアイコンとして、ここから「盗まれた」アイコンを留めます：

アプタナのICO

できた スタジオを立ち上げ、最後まで「仕上げ」ます。

7. [実行]-> [デバッグ]に移動します...

7.1。 左ペインでWebブラウザを選択します。

7.2。 [名前：]フィールドで、新しい構成に名前を付けて次のように記述します。

-ブラウザ実行可能ファイル：/ usr / bin / firefox-2

-引数：-P AptanaDebug -no-remote





8.行った変更を保存（適用）-「ビールの準備ができました！」

デバッグを開始すると、AptanaはFirefoxのAptanaDebugプロファイルを自動的に使用します。



ご清聴ありがとうございました。
