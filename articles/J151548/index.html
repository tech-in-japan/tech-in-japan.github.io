Firefox OSに魔法のようなものはありますか？

過去1年半の間、私は新しいMozillaプロジェクトであるFirefox OSに取り組む時間をますます増やしてきました。 この間、私は文字通り彼と彼のアイデアに夢中になりました。





Firefox OSスクリーンショット







正直なところ、Firefox OSは驚くべき何かの始まりです。 これは待ち望まれている革命です。 新鮮な空気の息。 最新技術の集大成。 それは魔法であり、すべてを変えます。



Firefox OSとは何ですか？



私が話していることを知らない人のために、ここに簡単な説明があります。



Firefox OSは、Boot to Gecko（B2G）プロジェクトの一部としてMozillaによって開発された新しいモバイルオペレーティングシステムです。 OSはLinuxカーネルを使用してGeckoにロードします。これは、ユーザーがHTML、JS、およびその他のOpen Web APIアプリケーションで作成されたアプリケーションを実行できるWebエンジンです。

-Mozilla Developer Network




要するに、Firefox OSプロジェクトはすべてのWebテクノロジーを統合して、完全なモバイルオペレーティングシステムを作成しました。 ちょっと待って、考えてみてください-これはJavaScriptで作成されたモバイルOSです！



これを行うために、Gecko（Firefoxエンジン）が変更され、最新のモバイルOSに存在する機能と同様の機能を作成するために必要な一連の新しいJavaScript APIが提供されます。 通話用のWebTelephony、テキストメッセージを送信するためのWebSMS、および振動するための振動API。







Firefox OSは、これまで誰もやったことのない方法で最新のWebテクノロジーを使用するという考え以上のものです。 また、単一のビジョンとしての、他の多くのMozillaプロジェクトの組み合わせ、つまりプラットフォームとしてのWebです。 Open Web Appsイニシアチブとペルソナは、これらのプロジェクトの1つであり、インターネット上の識別と承認の分野における当社のソリューション（公式名「BrowserID」で知られています）です。 Mozillaの多数のプロジェクトがどのように1つに統合されるかを見るのは驚くべきことです。



Firefox OSに関するMDNページで詳細を見つけることができます。 読むことを強くお勧めします。



Firefox OSを選ぶ理由



「素晴らしいですね、でもなぜJavaScriptなの？」と思うでしょう。これは本当に良い質問です。 JavaScriptを使用してモバイルOSを開発することが最良の解決策であった理由はたくさんあります。



2つの主な理由は、Firefox OSがギャップを埋め、モバイルプラットフォーム市場の現在の妥当性と制限に代わるものを作成することです。



モバイルプラットフォーム市場のギャップを埋める


所得水準が非常に高い国であっても、スマートフォンが途方もないほど高価であることはよく知られています。 しかし、そのような価格が豊かな国にのみ存在すると考えるなら、あなたは深く誤解されています。 ブラジルの16GB iPhone 4Sの価格は約615ポンドで 、イギリスの同じ携帯電話よりも100ポンド高価です。



ブラジルのこのような価格は、高い輸入税によるものです。 どうやら、Appleはすでにこの問題の解決に取り組んでおり、国内での現地生産ラインの建設を計画しています。 これにもかかわらず、このケースは、多くの人が良いスマートフォンを買う余裕がないことを完全に明らかにしています。 いくつかの国ではスマートフォンを振らないほうが良いという事実は言うまでもなく、 そのコストは小型車のコストに等しい 。



それでは、良いスマートフォンを手に入れたいが、莫大な費用をかけたくない場合はどうすればいいのでしょうか？ 安価なAndroidスマートフォンを購入することもできますが、そのようなものは、通常、動作が悪く、常に速度が低下します。



幸いなことに、Firefox OSができました...



Firefox OSの目標は、ハイエンドデバイスの競争を生み出すことではなく、通常の携帯電話の価格でエントリーレベルおよびミッドレンジのスマートフォンを提供することです。

- ボニー・チャ




Firefox OSはこれに最適です。 このOSは、パフォーマンスの低いデバイスをベースにした本格的なスマートフォンを提供できます。これは、ミッドレンジデバイスのAndroidに匹敵します。 そして、これは冗談ではありません。



たとえば、私は現在、携帯電話で50ポンドのJavaScriptゲームをテストしています。 この価格でデバイスに期待することはあまりないはずですが、実際には、これらのゲームは同じAndroidスマートフォン（FirefoxまたはChrome）でブラウザーを実行するよりも高速に動作するだけでなく、Androidよりも高速ではありません価格が4〜5倍高いデバイス。



同じデバイスでAndroidブラウザーで作業した結果と比較して、なぜこのようなパフォーマンスの向上があるのですか？ その秘密は、Geckoとハードウェア間の高速データ交換であり、これによりJavaScriptが非常に高速に動作します。



安価なデバイスでの高いJavaScriptパフォーマンスは、Firefox OSが巨大な何かの始まりであると確信する理由の1つになりました。



Mozillaは必ずしも50ポンド相当の電話でOSを起動するわけではない、と私たちは開発とテストにこのデバイスを使用していると言わなければなりません。



代替のオープンプラットフォーム


「なぜFirefox OSなのか」という2番目の理由は、代替のオープンなモバイルプラットフォームを作成するだけでなく、主要なマーケットプレーヤーに立ち向かい、影響を与えようとする試みです。



1998年にソフトウェア開発者として、また後に企業や組織としてMozillaが設立されて以来、私たちの使命は、主要なエンタープライズ製品と競争できるオープンテクノロジーを作成することでした。

- スティーブローア




MozillaはFirefoxで成功を繰り返しようとしています。Firefoxは文字通りブラウザー市場に参入し、ユーザーがWebの使用方法を制御できる代替手段があることをユーザーに示しました。



今では、危険にさらされているのはモバイルWebです。 また、脅威はマイクロソフトからではなく、主要なモバイルプラットフォームのメーカーであるAppleとGoogleからのものです。 彼らのアプリケーション、独自のプラットフォーム、独自のアプリストア、開発者向けの非常に不機嫌なルール。 AppleとGoogleは、Webテクノロジーの地位を悪化させるだけです。

- トーマス・クラバーン




開発する必要がある主な分野の1つは、アプリケーションの移植性です...



ある意味で、モバイルアプリケーションをめぐる誇大広告は一歩後退したものです。モバイルアプリケーションをサポートする特定のOSとデバイスにユーザーをリベットします。 Webは進化し、どのハードウェアでも同じように認識できるという結論に達しました。

Firefox Webブラウザーの作成者であるMozillaは、モバイルデバイスでも同じことを行うことを決意しています。

- ドン・クラーク




Firefox OSは、スマートフォン、PC、タブレット、およびブラウザーを備えた他のデバイスで同じアプリケーションを使用できるようにするために、Webテクノロジーの遍在性を活用することを目的としています。 スマートフォンでそれを終了した場所からデスクトップでAngry Birdsをプレイし続けることができませんか？ 本当に欲しい！



開発者の夢


Firefox OSが必要なもう1つの理由は、現時点では簡単に編集できるOSがないことです（Androidを少し変更できますが、これはそれほど単純ではありません）。



Firefox OSは完全にHTML、JavaScript、CSSで構築されています。 基本的なWeb開発スキルを使用すると、OS全体を完全に変更できます。 CSSの1行を編集すると、アイコンの配置方法や形状に影響を与えたり、電話を処理するJSを変更したりできます。



このプラットフォームは開発者に固有のものであり、彼らがそれで何をし、何につながるのかを知りたいと思っています。



いい瞬間



Mozillaに1年半携わっていたので、Firefox OSの誕生の時にここにいることがどれほど幸運だったかに気付きました。 すべてを正しく覚えていれば、プロジェクトは会社での最初の数週間で発表されました（Boot to Geckoなど）。



すべてが驚くべきものでしたが、時間が経つにつれてさらに驚くべきものになりました。 現在のところ、Firefox OSは私の仕事の最優先事項であり、率直に言って、私は気に入っています。 このようなプロジェクトに参加できて光栄です。



私は何度も疑問に思いました。これは驚くべき感覚です。Firefoxの起動時にMozillaで作業するときに経験するようなものですか。 興奮、情熱、緊張、これがどれほど素晴らしいか、そして誰も気にする必要がある理由を説明できないこと。



正直なところ、多くの人がFirefox OSのリリースがすべての人にとって本当に意味することを完全に理解しているとは思いません。 Firefoxのように、私は思う。



現時点では、Mozillaで働くことができて嬉しいです。会社の生活の中でこのような興味深い時期に。



破れた



このプロジェクトの重要性を認識したのは開発者です。 彼らは私たちのイベントでMozilla従業員のデモを手にしました。 そして、これらの人々がデバイスを探索し、さまざまな感情を経験している間、これらの人々を見る機会ほどエキサイティングなことはほとんどありません...

あなたは私がプロジェクトを称賛しているだけだと思う​​かもしれません。 しかし、実際には多くの人が答えました。 それを見るのは面白かった。



Firefox OSに対する人々の反応を見たところ、それが大きく変わることを実感しました。 誰もがとても喜んでいるので、誰にも説明する必要はないようです。



問題



解決する必要のある問題のいくつかに言及せずに、Firefox OSを常に賞賛するのは不公平でしょう。



Androidで発生する、アプリケーションのオープンエコシステムの作成やデバイスの断片化など、一般的な問題があります。 これは重要ですが、最終的にはまったく興味がありません。



私が最も興味を持っているのは、モバイルデバイス上のHTML5ゲームの問題です。認識とパフォーマンスについて、開発者はしばしば不満を述べています。 この問題は、Firefox OSだけに起因するものではありません（AndroidとiOSもこれが悪いです）が、今では完全にパフォーマンスの問題に集中しています。



以前に作成されたモバイルHTML5ゲームのほとんどは、実行速度が非常に遅い（0〜20FPS）か、少し高速（20〜30FPS）です。 多くの場合、これらのゲームには不安定なFPSがあり、ゲームプレイが著しく悪化します。



興味深いことに、多くの問題は必ずしもデバイスやJavaScriptに関連するものではありません。 Biolab Disasterなど、いくつかの重いゲームがあります。 このゲームは、同じ電話でも50ポンド（40-60FPS）で完璧に動作します。



時々、デバイスとプラットフォームがゲームのパフォーマンスを低下させる可能性があることは明らかです（一部の人々が考えるほど頻繁ではありません）。 弱いデバイスでうまく機能するゲームから多くのことを学ぶことができます。 開発者が使用するテクニックとテクニックを学び、モバイルプラットフォーム用のHTML5ゲームで作業する人について説明します。



重いHTML5ゲームは、どんなデバイスでも、最も弱いデバイスでも完全に機能すると信じています。 どうしてそんなに確かなの？ 人々はすでにそのようなゲームを今日作っているからです。 私の人生で一番信頼しているのは、私の目です。



携帯電話だけでなく



何よりも嬉しいのは、来年リリースするモバイルデバイスにFirefox OSがインストールされるという事実ではなく、将来どうなるかということです。 「開発者の夢」、つまり他の人がどのようにOSの使用の境界を広げることができるかについて話したときに、このトピックに先ほど触れました。







そして、これは今日起こっています。 すでにFirefox OSをRaspberry PiとPandaboardに 移植しています。 それらは完璧ではありませんが、クールなのは（この言葉を避けようとしましたが）最初のリリースのずっと前にすべてが起こるということです。



すでにMac、Windows、LinuxでFirefox OSを試す機会があります。 もちろん、スマートフォンにあるすべての機能にアクセスできるわけではありませんが、それでも何かを試すことができます（たとえば、並行して実行されるアプリケーション）。 システムのインストールは非常に簡単です。



Gamepad APIがGeckoに表示され、Firefox OSデスクトップクライアントで利用可能になる最近の日を想像することしかできません。 これは何ですか？ まあ、マウスやセンサーの代わりにゲームパッドで動作するように設定されたOSでテレビに接続されたデバイスを想像することはそれほど難しくありません（これはすべてJavaScriptです、覚えておいてください）。



これは、HTML5ゲーム用のコンソールの時代の始まりです。これは、Mozilla以外の「自由な」時間に取り組んでいるものです。



私たちは、多くのデバイスがWebサイトの作成に通常使用するのと同じテクノロジーで動作できるようになりつつあると言いたいです。 それでは、同じAPIを使用してどこからでもデータにアクセスして共有できるテクノロジーベースのデバイスでいっぱいの世界で何ができるでしょうか？
