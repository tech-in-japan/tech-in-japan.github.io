Azureネットワーク機能



クラウドインフラストラクチャを構築するために、Microsoft Azureのネットワーク機能に対する多くの更新が導入されました 。 以下はリストと簡単な説明です。



  • ネットワークセキュリティグループ -ネットワークセキュリティグループおよびルールを作成および使用して、VMまたはサブネット上のVMグループのインバウンドおよびアウトバウンドトラフィックを制御できるようになりました。 グループとルールは、仮想マシン自体とは独立して制御されます。







  • マルチNICサポート-VMで複数の仮想ネットワークインターフェイスカードを作成、構成、使用できるようになりました。







  • 強制トンネリング -クラウドサービスで生まれたすべてのインターネットトラフィックを企業ネットワークとVPN経由でリダイレクトして、追加の監視と監査を行えるようになりました。







  • ExpressRouteの機能強化 -ExpressRouteユーザーは、複数のAzureサブスクリプションで単一の接続ポイントを共有できるようになりました。 さらに、任意の仮想ネットワークを複数のExpressRouteネットワークに接続できます。



  • VPNゲートウェイの新しい次元-ハイブリッドソリューションを構築する際の高負荷のタスクを解決するために、より生産性の高いAzure VPNゲートウェイが導入されました。これにより、ExpressRouteおよびサイト間VPNゲートウェイを使用する場合のデータ交換が高速化されます。



  • VNetゲートウェイとExpressRouteの操作ログと監査ログ-仮想ネットワークゲートウェイとExpressRouteネットワークの操作ログを分析および監視できるようになりました。 Azureポータルには、計画されたインフラストラクチャの更新だけでなく、ログおよびAPI呼び出しによる分析に必要なすべてのデータを表示する機能が追加されました。



  • 仮想ネットワークゲートウェイポリシー-仮想ネットワーク間のトンネルの暗号化を制御できるようになりました。 オプションは、利用可能な暗号化オプション3DES、AES128、AES256およびNull、およびIPsec / IKEゲートウェイ用のオプションPerfect Forward Secrecy（PFS）を含みます。



  • ソースIPアフィニティ-Azureロードバランサーは、新しいソースIPアフィニティロードバランシングモード（セッションアフィニティまたはクライアントIPアフィニティとも呼ばれる）をサポートするようになりました。 これで、2つのパラメーター（ソースIP、宛先IP）または3つのパラメーター（ソースIP、宛先IPおよびプロトコル）の組み合わせに基づいてトラフィックのバランスを取ることができます。



  • Traffic Managerのネストされたポリシー-ネストされたポリシーを作成して、Traffic Managerのトラフィックを制御し、より複雑な負荷分散シナリオに対応できるようになりました。 詳細な発表 ;







  • 新しいポータルでのVMの内部ロードバランサー、予約済み、インスタンスIPアドレスの管理-内部ロードバランサーのパラメーターと、新しいAzureポータルでのIPバックアップを構成できるようになりました。

新しいAzure Batchサービス （HPCタスクとクラスターオンデマンド）







新しいサービスAzure Batchを導入しました。これにより、数万台から数万台の仮想マシンをオンデマンドでスケーリングする際に、HPCとロードされたバッチタスクをクラウドで簡単に実行できます。



プレナリーレポートのフレームワークで、Mark Russinovichは、人気のある3DツールBlenderの拡張により、Azure Batchを使用して37の仮想マシンを選択して複雑なグラフィックスを計算し、エディターから直接リクエストすることでシーンを10倍速くレンダリングすることを要求する方法を実際の例で示しました。



Azure Batchを使用すると、次の問題を解決できます：できるだけ早く、必要に応じて必要な電力を割り当ててデータパケットを処理し、クラウドでオンデマンドでクラスターを構築および受信し、何百回もスケーリングし、スケジュールに従って、キューからタスクを実行し、オンデマンドで自動的にスケーリングします、インフラストラクチャコストなしで、既成のサービスの形ですべて入手できます。



公式ブログでの新しいサービスの発表 。 新しいサービスの詳細については、 公式ページをご覧ください 。 Azure BatchのPythonコードサンプルはGitHubで入手できます 。



新しいAzure Stream Analytics （リアルタイムストリーミング）サービス







新しいAzure Stream Analyticsサービスは 、着信データストリームのリアルタイム処理におけるビジネスの課題に対応するように設計されています。 このサービスを使用すると、インフラストラクチャに投資することなく、1秒あたり数百万のイベントを処理し、データストリームの値を履歴値と比較し、最初のリクエストで数分でオンデマンドで自動的に利用可能になります



Azure Stream Analyticsには、他のAzure Event Hubsサービスとの「すぐに使える」統合が含まれており、外部デバイスを処理に接続して、大量のデータをリアルタイムで同時に処理できます。

Azure Automation GA （インフラストラクチャ展開自動化サービス） リリース



Azure Automationにより、 Azure管理者とユーザーは、スケーラブルで柔軟なワークフローツールとスクリプトを使用して、Azureクラウド環境でリソースを作成、デプロイ、監視、および保守できます。







Azure AutomationはPowerShellワークフロー（Runbook）に基づいており、すべての管理および展開の自動化プロセスを記述しています。 Runbookの作成をゼロから開始するか、Azureポータルのギャラリーから数百のRunbookの1つを開始できます。
