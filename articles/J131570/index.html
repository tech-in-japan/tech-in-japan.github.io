冬時間への切り替えをキャンセルすることに関する別の問題

周りのみんなが知っているように、ロシア人は今でも夏のプレミアムで生き続けています。

しかし、私たちの「鉄」のアシスタントの全員が知っていたわけではなく、習慣から外れた多くのデバイスが冬時間に切り替わりました。

これは、「古代の」Win XP、さらには英語の制御下で私の見解に入るコンピューターの1つによっても行われました。



災害の規模を評価したところ、次のことがわかりました。

1. KB2570791は 、Windows XP SP3以降でのみ適用可能です。 古いOSはKB2570791パッチで更新できません

2.パッチ自体をすぐに入手して適用できるとは限りません。 多くの理由がある可能性があり、それらの議論はこの記事のトピックではありません。







ロシア語を話すWindowsのパッチを「解析」することにより、「ロシア標準時」のためにパッチが特に行うレジストリの変更を特定することができました。



その結果、このようなregファイルを取得し、タイムゾーンレジストリに入力した後、「Medvedev's」で機能しました。



Windows Registry Editor Version 5.00



[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Time Zones\Russian Standard Time]

"Display"="(GMT+04:00) Moscow, St. Petersburg, Volgograd"

"Dlt"="Russian Daylight Time"

"Std"="Russian Standard Time"

"MapID"="6,7"

"Index"=dword:00000091

"TZI"=hex:10,FF,FF,FF,00,00,00,00,C4,FF,FF,FF,00,00,00,00,00,00,00,00,00,00,\

00,00,00,00,00,00,00,00,00,00,00,00,00,00,00,00,00,00,00,00,00,00



[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Time Zones\Russian Standard Time\Dynamic DST]

"FirstEntry"=dword:000007da

"LastEntry"=dword:000007db

"2010"=hex:4C,FF,FF,FF,00,00,00,00,C4,FF,FF,FF,00,00,0A,00,00,00,05,00,03,00,\

00,00,00,00,00,00,00,00,03,00,00,00,05,00,02,00,00,00,00,00,00,00

"2011"=hex:10,FF,FF,FF,00,00,00,00,C4,FF,FF,FF,00,00,00,00,00,00,00,00,00,00,\

00,00,00,00,00,00,00,00,00,00,00,00,00,00,00,00,00,00,00,00,00,00









このコードを拡張子.reg（例：update_rus_timezone.reg）を持つファイルに入れて、ダブルクリックして実行するだけで十分です。

または、完成したregファイルをダウンロードして実行できます 。

そして、これはいわばバックアップ（ tz_ruszone_backup.reg ）です。 突然「今まで通りにすべてをする」必要がある人のために。



上記のアクションは、英語版のWinXP、このファミリーの他のオペレーティングシステム、または他の言語のWinXPに関連することを強調します（ロシア語のタイムゾーンの「英語」の名前が使用されているため）。



誰かがこの情報も役立つと思います。
