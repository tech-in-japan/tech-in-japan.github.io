SCCM 2012 R2およびMDT 2013を使用したWindows XPから8.1への移行



こんにちは、紳士。 最近のWindows XPの廃止に関連して、このトピックは非常に関連性が高いので、始めましょう。

主な問題は、USMT 6.3はWindows XPをサポートしておらず、USMT 5.0はWindows 8.1をサポートしていないことです。 両方のバージョンを使用する必要があります。



トップレベルの作業内訳：



1. OSD手順のためのSCCMの準備

2.必要なパッケージの作成

3.必要なタスクシーケンス（以降TS）の作成

4. SCCMでコンピューターをリンクする

5. Windows XPからデータ移行ポイントへのデータ移行。

6.事前に移行されたデータを復元したコンピューターにWindows 8.1をインストールします。

さらに、SCCM 2012 R2の既成のインフラストラクチャが稼働していると想定しています。



制限事項：



1.すべてのWindows XPマシンにSCCMクライアントをインストールする必要があります

2.フォルダーとファイルを移行できますが、設定は移動しません。



カットの下の詳細。



OSD手順のためのSCCMの準備

1. PXEの準備

コンソールを開き、[管理]> [サーバーとサイトシステムの役割]> [配布ポイント]を踏み、プロパティを開き、[クライアントのPXEサポートを有効にする]、[不明なコンピューターサポートを有効にする]、および[この配布ポイントが着信PXE要求に応答できるようにする]チェックボックスをオンにします。 さらに、PXEが正しく機能するようにDHCPを構成する必要があります。

オプション66-NetBIOS PXEサーバー名

オプション67-\ smsboot \ x64 \ wdsnbp.com



WDS設定を変更する必要はありません。



2.状態移行ポイントの役割のインストール

コンソールを開き、[管理]> [サイト]を押し​​て、必要なサイト[サイトシステムの役割の追加]を右クリックします。 スタンドアロンプ​​ライマリサイトがある場合（これを読んでいる人の99.99％に当てはまります）、[次へ]ボタンをN回クリックし、状態移行ポイントの役割を選択することを忘れないでください。移行が成功した後、データとデータ保持期間を保存するフォルダーを指定してください。



必要なパッケージを作成する

1. Windows 8.1イメージの追加

個人的には、インストールディスクからinstall.wimを追加し（テストの一環として）、企業イメージがある場合は追加します。



2. USMTパッケージを追加します。

USCM 6.3は、SCCMのインストール時に作成されます。USMT5.0（ビルド6.2.9200.16384）が必要です。ここから ADK をダウンロードし、USMTのみを一部のマシン（できればUSMT 6.3が既にインストールされているマシンではない）に配置し、USMTファイルを「切り取り」ます（デフォルトのフォルダーはC：\ Program Files（x86）\ Windows Kits \ 8.0 \ Assessment and Deployment Kit \ User State Migration Tool）

次に、それらをサーバーにコピーし、コピーしたファイルを含むパッケージを作成します。プログラムを作成する必要はありません。



3. MDTブートイメージの作成

コンソールを開き、[ソフトウェアライブラリ]> [オペレーティングシステム]> [ブートイメージ]を押します。 「MDTを使用してブートイメージを作成する」、名前、説明、パッケージストレージの場所へのパスを入力し、「スクラッチスペース」のビットサイズとサイズを選択します。128MB以上のサイズを選択することをお勧めします。 x86イメージとx64イメージの両方を作成する必要があります。 作成後、作成されたイメージのプロパティに移動して、[このブートイメージをPXE対応の配布ポイントから展開する]ボックスをオンにする必要があります。



必要なTSを作成する

1. Windows XPからのデータ移行用のTSを作成します。

コンソールを開き、[ソフトウェアライブラリ]> [オペレーティングシステム]> [タスクシーケンス]を押します。 状態ストアの要求、ユーザー状態のキャプチャ、状態ストアの解放の3つの手順で通常の「カスタムタスクシーケンス」を作成します。 [ユーザー状態のキャプチャ]ステップで、[ユーザープロファイルのキャプチャ方法をカスタマイズする]を選択し、MigDocs.xmlおよびMigUser.xmlファイルを追加します。

USMT 5.0はこのTSで使用されます



2. TSを作成してWindows 8.1をインストールします

コンソールを開き、[ソフトウェアライブラリ]> [オペレーティングシステム]> [タスクシーケンス]を押します。 MDTタスクシーケンスを作成します。 次の画面で、名前とコメントの設定、組織に関する情報の入力、ドメインへの入力などを行い、「このタスクシーケンスは画像のキャプチャには使用されません」のチェックを外します。 以前に作成したブートイメージとWindows 8.1パッケージを選択します。 新しいMDT Toolkitパッケージを作成し、情報を入力します。 アイテム「Zero Touch Installation」を選択します。 SCCMクライアントパッケージを選択します。 USMTパッケージ（6.3！）を選択します。新しいMDT設定パッケージを作成し、情報を入力します。 最後の「次へ」をクリックします。



3. TSの変更

TSを修正して動作させる必要があります。 このTSはUSMT 6.3を使用します。

コンソールを開き、[ソフトウェアライブラリ]> [オペレーティングシステム]> [タスクシーケンス]を押します。 TSを選択して[編集]をクリックします。 「ステータス5を設定」ステップを見つけて、「状態ストアの要求」ステップを追加し、「別のコンピューターから状態を復元」を選択します。





「オプション」タブで、「タスクシーケンス変数」「USMTLOCAL」「等しくない」「True」という条件を追加します。





[Restore State Store]アイテムを[Restore User State]アイテムの下に追加し、同じ条件を追加します。

次に、[ユーザー状態の復元]項目で[ユーザープロファイルの復元方法をカスタマイズする]を選択し、MigDocs.xmlファイルとMigUser.xmlファイルを追加します





「インストール」グループの「ドライブ文字の変数を設定」で、値を「True」に変更します



4. CustomSettings.iniの変更

MDT設定パッケージの作成時に、CustomSettings.iniファイルが作成されたため、変更する必要があります。 次の行を追加します。



DeploymentType = NEWCOMPUTER

SLShare = \\ server_name \ xxx \ log-TSがログをコピーするフォルダーへのUNCパス。トラブルシューティングに非常に便利です。

ScanStateArgs = / v：5 / o / c / ue：％computername％\ *-注意、このコマンドはすべてのローカルユーザーを移行プロセスから除外します。

LoadStateArgs = / v：5 / c



5.パッケージを配布する

すべてのパッケージと両方のブートイメージをコンテンツ配布ポイントに配布する必要があります。 （私は、個人的に、これを行うことを常に忘れています）



SCCMでコンピューターをリンクする

この手順は、移行を成功させるために必要です;異なるコンピューター間で正常に移行するには、コンピューターを接続する必要があります。



コンソールを開き、[資産とコンプライアンス]> [ユーザー状態の移行]を押します。 [コンピューターの関連付けの作成]項目を選択し、ソースコンピューター（Windows XP）およびターゲットコンピューター（Windows 8.1）を選択します。 ターゲットコンピューターがまだ存在しない場合は、SCCMで作成する必要があります。



「置換」スクリプトではなく「更新」スクリプトを実装する必要がある場合は、コンピューターをリンクする必要はありません。 データ収集を開始し、マシンをPXEで再起動して、Windows 8.1のインストールを開始するだけです。



次に、パラグラフ5および6に従います。

当然、有能な移行のために、TSデータを「設定」するコレクションを正しく作成する必要があります。

すべてのWindows XPコンピューターのコレクション：

選択* SMS_R_System内部からSMS_G_System_OPERATING_SYSTEM.ResourceId = SMS_R_System.ResourceIdのSMS_G_System_OPERATING_SYSTEMに参加します。SMS_G_System_OPERATING_SYSTEM.Captionは「* Windows XP *」のようになります。

Windows 8.1コンピューターのコレクションも作成する必要があります。



したがって、両方の設定を古いコンピューターから新しいコンピューターに、1台のコンピューターの「制限」内で移行できます。

プラス -サーバー上のユーザーのファイルのコピーは必要な限り、「更新」（このスクリプトは完全にWinPEを介して完全に実装されますが、これには個別のTSが必要です）および「置換」スクリプト用の単一のスクリプトです。



マイナス -2 TSのうち、 WinPEではユーザーが何らかの形で影響を与える可能性があるため、1つは行われません。ユーザー情報を保存するには、状態移行ポイントに多くのスペースが必要です。
