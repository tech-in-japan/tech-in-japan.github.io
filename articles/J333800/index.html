HPEが新しいHPE ProLiant Gen10サーバーを発売

7月11日に、HPEは新しいProLiant Gen10サーバーの注文の受け付けを開始しました。この記事では、この記念すべき世代の主要技術とモデルの特性についてお話しします。







新世代のサーバーのリリースはIntel Xeonスケーラブルファミリープロセッサーのリリースと一致しましたが、これがアップグレードの唯一の理由とは程遠いものでした。 前提条件は、HPEのお客様にとって新たな課題です。 その中には、新しいクラスの負荷（メモリ内の大量のデータの処理とディスクサブシステムの高性能を必要とするものを含む）の出現、ハードウェアインフラストラクチャの制御性の向上（REST APIによる管理への切り替え、さまざまなIT自動化ツールとの統合、テンプレートとスクリプトのサポート） ）、ITリソースの消費の新しいモデル（使用される金融サービスや支払いなど）、および最後になりますが、情報セキュリティに対する新しい脅威 mponentovサーバ（BIOSおよびファームウェア）。



これらのすべてのタスクに対応して、HPE ProLiant Gen10サーバーに新機能が追加されました。 順番に理解しましょう。



プロセッサー



最前線では、インテリジェントシステムチューニングと総称される独自のHPEテクノロジーを使用してサーバーのパフォーマンスを改善しています。



- ワークロードマッチング （タスクの設定）は、さまざまな特定のタスク用のサーバープリセットのセットです。それぞれの場合、メモリ、プロセッサ、電源、ディスクサブシステムなどの構成の微妙な変更によりリターンが増加します。 （合計で約30の調整可能な設定があるため、特定のタスク用にサーバーを設定するのは難しい場合があります）







- ジッタースムージング 。 このテクノロジーを組み込むことで、サーバーはTurboBoostモードでプロセッサー周波数の「ジッター」の監視を開始し、周波数の急激な変化時に発生するさまざまなメモリレベルへのアクセスの遅延を排除するために周波数差を平滑化します。 その結果、周波数と遅延を伴う要求の厳しいタスク（リアルタイムOSや一部のJavaアプリケーションなど）では、ターボモードの通常の増加を超えてパフォーマンスが向上します。 この技術の詳細については、 こちらをご覧ください 。







- コアブースティング （コアの加速）。 このテクノロジーは、6000および8000ファミリーの一部のIntel Xeonスケーラブルプロセッサー向けに、次のリリース（2017年秋-2018年春）でGen10サーバーで利用可能になります。このモードで。 これらの設定の結果、少数のコアで作業しながら、たとえばOracleデータベースで同じレベルのパフォーマンスを維持できます。 そして、これはライセンスの直接的な節約です。







Jitter SmoothingとCore Boostingを有効にするには、サーバーでiLO Advanced以上のライセンスをアクティブ化する必要があります。 さらに、コアブースティングは、すべてのプロセッサおよびサーバー構成で機能するわけではありません。 3つのテクノロジーすべてについて簡単に説明します 。



記憶





メモリのフォールトトレランスを保証するメカニズムも改善されました。 メモリの量が数百ギガバイトである最新のシステムでは、一部のメモリチップでエラーが発生する可能性が非常に高くなります。 したがって、メモリの信頼性（RAS）を確保するためのツールの使用がより合理的になりつつあります。 Gen10世代では、HPEはそのようなツールをもう1つ導入しました-SmartMemory Fast Fault Tolerance （FFT）テクノロジー。 HPEサーバーのプロアクティブアルゴリズムは、メモリチップの状態を常に分析します。 また、チップ内のメモリの特定の領域にデータのリスクがある場合、FFTはそれに同じメモリチャネル上の同等のボリュームの「スペア」領域を割り当てます。 その結果、保護された領域でのデータの書き込みと読み取りには整合性が制御され、この小さな領域でのみスループットと総ボリュームが低下します。



たとえば、現在Xeon E7プロセッサで使用されているIntelのDDDCのロックステップメカニズムと、このテクノロジーの違いは何ですか？ その点で、HPE Advanced Error DetectionおよびFast Fault Toleranceをきめ細かく制御することで、システム全体でパフォーマンスが数パーセントから50％に低下するLockstepとは異なり、データ損失の可能性があり、小さなメモリ領域でのみパフォーマンスを低下させます。 FFTを使用すると、サーバーあたりのスループットは平均で1％しか低下しません。 このテクノロジーはHPE独自のものであり、Intelとのライセンス契約の下でさらに2年間、他のメーカーのサーバーには表示されません 。 一般に、HPE統計によると、HPEサーバーで最新のRASテクノロジーを使用すると、1年あたりのシステムクラッシュの数（年間クラッシュ率）が85％減少します。 HPEメモリがなぜスマートメモリと呼ばれるのかという質問に対する別の回答。



この技術に関するビデオは以下で見ることができ、詳細な技術文書はまもなくリリースされます。







メモリの操作におけるもう1つの革新的なセットは、不揮発性ランダムアクセスメモリHPE NVDIMMとHPEスケーラブル永続メモリの 2つのテクノロジです。 1つ目は、2年以上にわたって存在するRAMモジュールで、サーバー内のバッテリー電源を備えた（同じモジュール上の）不揮発性フラッシュドライブと組み合わせます。 このようなモジュールは、非常に高速なブロックデバイスとして通常のOS（WindowsおよびLinux）に表示されます-こちらの実験を参照してください 。 また、一部のアプリケーション（ Microsoft SQL Server 2016など）では、このようなモジュールはバイトアドレッシング（RAMなど）を備えた特殊なデバイスとして認識されますが、停電には耐性があります。 このようなモジュールにMS SQL Serverのトランザクションログの末尾（トランザクションログの末尾）を配置すると、SSDにログの末尾を配置する場合と比較して最大4倍のデータベースパフォーマンスの向上が見られます。 新しいHPE NVDIMMの容量は8 GBおよび16 GBになりましたが、ほとんどの場合、数百ギガバイトのSQLデータベースの場合、トランザクションログの末尾をホストするには1つのNVDIMMで十分です。



HPEスケーラブル永続メモリは、HPE ProLiant DL380 Gen10サーバー、従来のDDR4メモリモジュール、およびNVMeドライブとサーバー自体に内蔵UPSを備えた冗長電源ユニットを使用して電力損失からRAMを保護する特別なメカニズムに基づく統合デバイスです。 このデバイスを使用すると、停電時にデータを失うリスクなしに最大1 TBのデータをメモリに保存できるため、多くのアプリケーションの作業を高速化できます。 たとえば、MySQLを使用した内部テストでは、HPEスケーラブル永続メモリでのチェックポイント操作（メモリから不揮発性メディアへのデータベースデータの保存）は、HDDの27倍、SSDの3倍の速度で実行されました。 また、復元（障害が発生した後のデータのメモリへの復帰）は、それぞれ13倍および5倍高速で発生しました。 さらに、このようなシステムでは、ほとんどすべてのデータベースをインメモリに変換できるため、パフォーマンスが大幅に向上します。





NVMeドライブの使用も非常に深く開発されています。 各サーバーにさらに多くのNVMeドライブをインストールできるようになり（たとえば、DL380 Gen10では最大20個）、サーバーの同じディスクバスケットにNVMeおよびSAS / SATAディスクをインストールできます（通常のバスケットは必要ない場合はNVMeをサポートします）。 さらに、1つのSFFドライブ（ボリューム120および340 GB、SATA）の代わりに2に適合するuFF（マイクロフォームファクター）のドライブの新しいフォーマットのサポートがあります。 uFF-SSDは、多くのタスクのブートディスクまたはキャッシュディスクとして使用でき、生産性の高いデータ用のより容量の大きいドライブのサーバースペースを節約できます。



更新されたコントローラのライン 。 現在では、より高速で（コントローラーから最大160万IOPS）、HBAモードとRAIDモードで同時に動作できます（あるモードのコントローラー上のディスク、別のモードのディスク）。 現時点でのGen10のコントローラーの完全なラインは次のようになります。





iLO 5



Gen10世代の最も強力な革新の1つは、高度なHPE iLO 5リモートコントロールシステムです。 リモートコンソールと仮想ドライブの操作を高速化する、より高速なiLOチップに加えて、多くの新機能が登場しました。







HPE Silicon Root of Trustテクノロジーは、iLOシステム内のハードウェアチップでチェックサムをチェックすることにより、iLOファームウェアの悪意のあるコードまたは損傷をチェックします。 iLOファームウェアが検証されると、この制御ツール自体が、侵入者やその他の整合性違反について、BIOSを含む他のすべてのサーバーコンポーネントのファームウェアをチェックします。 問題が検出されると、iLOは自動的にまたはコマンドによって、保護されたファームウェアリポジトリから特定のファームウェアを最後の動作状態に復元するか、サーバーを工場出荷時の設定に戻すか、応答しません。



なぜこのテクノロジーなのか？ 第一に、コンポーネントのファームウェアは、手動更新プロセス中に、または単にチップの誤動作のために破損する可能性があります。 第二に、セキュリティ脅威アナリストは、ファームウェアやその他の低レベルのサーバーコードで毎年、ますます多くの脆弱性を発見しています。 最新の注目すべき例は、Appleが購入したSuperMicroサーバーファームウェアのパッチに見られる脆弱性です（ link ）。 このレベルは通常のアンチウイルスによって実際には制御されないため、そこにある脅威が重要なウイルスを隠しているのです。



HPEアプローチの独自性は次のとおりです。a）コンポーネントファームウェアの生産を完全に制御します。 HPEブランドのすべてのコンポーネントには特別に開発またはテストされたファームウェアがあり、これはHPEチャネルを介してのみ配信され、Gen10世代以降はHPEデジタル署名が付いています。 b）HPEのみが、ソフトウェアからマイクロチップまで、独自のiLO制御システムを開発および製造します。これにより、ファームウェア署名検証ツールを統合できます。



もう1つの小さいながらも有用なセキュリティの改善は、サーバーエンクロージャーの開口センサーです。トリガーされると、iLOはインシデントの原因が明らかになるまでサーバーの動作をブロックできます。



iLO 5の機能向上は、 Intelligent Provisioningと呼ばれるRAID設定を含むサーバー構成ツールになりました。 これは、F10キーを押してサーバーを起動したときだけでなく、iLOコンソールから直接起動できます。 これにより、不要なサーバーの再起動を回避できます。



BIOS UEFIモードを使用すると、ブート手順自体がより速く通過し始めました- 平均して、ロードは3倍高速です。 RBSU BIOSセットアップインターフェイスが変更され、より便利になりました。また、Intelligent Provisioning（およびiLO）から起動することもできます。



マルチベンダー環境を含むさまざまな自動化ツールのユーザーにとって大きな利点-iLOは、ほとんどのOpen IPMI命令をサポートします。 さらに、HPEは、REST APIを介してハードウェア管理標準を実施する方法を引き続きリードしています（ こちらをご覧ください ）。 現在、HPEのREST APIは、DMTF Redfish標準に完全に準拠しています。ただし、RAIDコントローラーの管理の部分を除きます（独自のREST APIがまだあります）。 REST APIを介して管理するためのネイティブツールと多数の補助ツールの両方があります（ PythonやPowerShellなど ）。



別の革新-今回はトラブルシューティングの観点から。 以前にActive Health Systemログ（サーバーの「フライトレコーダー」の記録）をiLO経由でのみアップロードして分析のためにHPEサポートセンターに送信できた場合、サーバー前面の特別なUSBポートからUSBフラッシュドライブにダウンロードして、無料のActive Health Systemビューアー 。 このサービスは、考えられる脆弱性の25％に対するソリューションを示し、提供しますが、それ以外の場合は、サポートへの連絡をはるかに簡単にし、トラブルシューティングの時間を短縮します。
