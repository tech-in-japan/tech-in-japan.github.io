1つの交差点の物語（アルプスを越えない）

このトピックは別のホリバーの始まりではなく、Linux（およびWindowsやFreeBSD）が他のものより優れていることを証明する試みや挑発ではないことをすぐに警告したいと思います。 これは、私が無料（または少なくとも無料）のOSをどのように使用しようとしたか、およびその結果についての、シンプルでほとんど装飾のない（まあ、ほんの少し）話です。



私自身はかなり長い間コンピューターと話し合ってきましたが、私を初心者と呼ぶのは難しいです。 しかし同時に、私は第一人者ではありません。 私は自分で自宅の車とラップトップにサービスを提供し、時にはサーバーに目を光らせてエニキー部分で余分なお金を稼ぐ必要がありましたが、これはすべて狂信的ではありませんでした。 私の仕事はコンピューターとプログラミングに関連していますが、技術に深く浸透する必要はなく、コンピューターの部分では専門知識が十分ありません。 私はコンピューターで作業している間、それらをツールのように扱うのに慣れています。特定のタスクを実行するために必要な目標を達成し、一部のタスクはコンピューターを使用して実行されます。 したがって、鉄、プログラム、オペレーティングシステムから、仕事を始めるための最小のしきい値が必要です。 理想的には-設定すればうまくいきました。 極端な場合-セット、いくつかの単純なアクションを実行し、再び、作業状態で。 鉄やシステムの内部を掘ることは、極端なニーズによってのみ促すことができます。 将来的にこれを念頭に置いてください。



なぜLinuxに切り替えたいと思ったのですか？ これは複雑な質問であり、その答えは時間とともに変化します。 最初は、それが何であるかを知りたいだけでした。 その後、作業の必要性につながりました。Linuxに切り替える計画があった時期があり、Linuxで動作するソフトウェアをテストする方が便利でした。 今、私はプラットフォーム、イデオロギー、テクノロジー、コンセプトの間で選択の自由が欲しいだけです。 フリーソフトウェアに切り替えたいという私の願いは、安定性と相対的な速さを維持しながら、何年もの間再インストールせずに働いてきたWindowsの欠点と結び付けられたことはありません。 私の移行はまだ完了しておらず、自宅のコンピューターでWindowsとLinuxが共存できます。 さらに、私にとって2つのシステムの快適な共存の可能性は、常に配布キットを選択するための基準の1つでした...



最初のパンケーキ



私の移行の物語は、ギガヘルツとギガバイトがアマチュアのコンピュータープログラマーの想像力をかき立てたときから始まりました。インターネットは主にモデムによって配布され、誰も無制限のアクセスを考えていませんでした。 当時、私の母（！）は電車に乗って（!!）それにディスクが付いた雑誌を買っていました。 雑誌の名前はずっと前に忘れられており、 ASPLinuxディストリビューションはディスクに記録されていました。 彼は私の最初のLinuxシステムになりました。 なんらかの理由で、私にはコンピューターがまったくなく、何らかの理由で彼はグラフィックスサブシステムと「すぐに使える」ものになりたくありませんでした。 したがって、Xを実行するために必要であった構成の長時間にわたる永続的な編集をよく覚えています。 さらに、マシンが他の何かを引き出すことができなかったため、 icewmがウィンドウマネージャとして選択されました。 Linuxの使用はすべて、単純な完全なおもちゃの再生、最も単純なコマンドの表面的な知識、同じ雑誌の指示に従って構成を編集することになりました。 その時間の最も鮮明な印象は、コンピューターが無限のサイクルで再起動することでした。 これを実現するのは非常に簡単でした。/etc/rcで、ランレベル6が最初のものとして割り当てられました。 特徴的なのは、私も状況を思い出しました：/ etc / rc、「6を書かないで」、キーストロークのシーケンス：「6 Esc：wq！ 「Enterを再起動します」と背中に寒さがあり、何が行われたのかがわかりました。 確かに、状況の修正にはそれほど時間はかかりませんでした。 そして、デスクトップには目がありました... :)



このシステムは私には長続きしませんでした。なぜなら、ソフトウェア（特におもちゃ）に必要なものは何もなかったからです。特にインターネットへのアクセスが制限されていることを考慮すると、このソフトウェアを使用することは明確ではなく、そのシステム上のソースからのアセンブリは少し困難でしたキットにコンパイラが含まれていない。 一般に、ASPLinuxは小さな実験としてしばらくの間マシン上で動作し、その後...ディスクの空き容量を占有し、Linuxを犠牲にするのが最も簡単でした。



中間段階



かなり長い間、私のコンピューターは完全にWindowsに所有されていたので、楽しんで働いて勉強しましたが、Windowsホストを表示するのに十分なコンピューターの強度がなく、所有者が見たかったのです。 ここでLinuxが再び助けになりました。 確かに、CD-ROM（特定の場合はBusiness Linux CD ）から直接ダウンロードされた「ミニディストリビューション」の形式です。 コンパクトで起動してmplayerを起動し、設定をいくつか変更することで、ジャンクで映画を見るのは非常に快適で、何度もやりました。 ちなみに、BlinCDに加えて、「メディアボックス」を作成するために特別に最適化された他のディストリビューションを使用しようとしましたが、どのメディアとも関係がありませんでした。 古い仕事中毒が亡くなった瞬間、BlinCDの使用をやめました。 その後のアップグレードでは、既にWindowsでビデオを再生できるコンピューターが残っていたため、映画を見るたびに再起動する必要がありませんでした。



角のあるUNIX



私が次にWindowsを削除する（または、それとは異なるものを試す）試みは、 FreeBSD 6.4thバージョンに関連していました。 その選択は、ITで働いている私の友人の1人が彼女を非常に称賛したという事実によるものでした。 また、ハードドライブに少し空き容量がありました。 その時までに、ソフトウェアの入手先とインストール方法はすでにわかっていましたが、ソースからのアセンブリには問題はありませんでした。 自宅のコンピューターが通常の方法でネットワークに接続されていないという事実を台無しにしただけです（モデムとして機能する携帯電話からの通常の出力を考慮しないことに同意します）。 その結果、ポートとパッケージからのプログラムの便利なオンラインインストールは利用できず、依存関係を手動で解決するオフラインの反復インストール（パッケージのリストを収集しました-職場でダウンロード-インストールのために持ち帰り-依存関係でシャットダウン-手順の最初に戻りました）すぐに疲れました。 これに加えて、ntfsでフォーマットされたパーティションへのアクセスに問題がありました。 FreeBSD自体はそれを読むことができるだけです。 完全なアクセスのために、私はntfs-3gを使用しようとしましたが、そのとき、ntfs-3gのほつれの下で、奇妙に動作しました。 たとえば、Windowsを再起動してディスクをチェックした後にFreeBSDで削除されたファイルは、墓から上昇し、以前とまったく同じ場所に存在し続けました。 ある時点で、FreeBSDへの興味がなくなり、ダウンロードを少なくし始め、それを完全に削除しました...公平に言うと、FreeBSDがハードウェア専用のカーネルを再構築した唯一のシステムであることに注意してください。 それが判明したように、これについて特別なことやひどいことは何もありませんでした。 =）



南アフリカの暑さ



ですから、もちろん、 Ubuntuがありました...少なくとも何らかの形でフリーソフトウェアの世界に興味を持っている人は誰でも、このディストリビューションをバイパスすることは難しいと思います。 私も。 私が最初にUbuntuを試したのは、彼女が番号9.04（一見）を着ていたときです。 当時、私はすでに多かれ少なかれ安定したインターネット接続（1メガバイトあたり4ルーブル）があり、それらの時代には多かれ少なかれ十分な充満を備えたコンピューターがありました。 すべてがインストールされ、すべてが機能しましたが、...ネットワークアクセスの制限により、興味のあるすべてのプログラムを配信できませんでした（高価です！）。 依存関係の手動解決に関与したくありませんでした（既に合格しています、わかっています！）。また、仕事中にパッケージをダウンロードするときに何か問題が発生しました。 私にとってのUbuntu 9.04は、短期的な実験にとどまりましたが、通常の作業に必要なすべての前提条件（インターネット接続の設定、ディスクへのアクセス、快適な環境）はすでにありました。 しかし、その後、非常に短い時間でOpenSolarisに置き換えられ（その理由はいくつかありました）、その後、私のコンピューターから別のLinuxアウトブレイクがありました。



豊富な選択肢



実験と好奇心の次の攻撃は最近私に起こりました。 今、私は制限を課していない通常のインターネットを持っています。ここで、私は完全なプログラムを試して、配布を最大限にストックすることにしました。 Ubuntu 10.10、 Fedora 13（F14のリリースの数週間前）およびOpenSUSE 11.3がダウンロードされ、ディスクに書き込まれ、キューに入れられました。



いつものように、私はすでに知っていることから始めました：ウブンタから。 「ライブ」コンパクトからダウンロードし、システムの裏通りでテストを数回クリックしてインストールを開始します...ここでは、インストーラーが提案したセクションのいずれにもブートローダーをインストールできなかったため、Ubuntについてのストーリーを終了できます。 これは、インストーラーのバグの1つであることが判明しました。 まあ、ブートローダーがないことは自動的にシステムがないことを意味するので、これについてUbuntuで実験することになり、いつかそれらに戻ることはないでしょう。



Fedora 13はインストールされて機能しましたが、WindowsとLinuxの間で交互に実行できるようにGRUBを構成することは少し重要な構成ではできませんでしたが、そのような可能性はまだありました。 ソフトウェアがインストールされ、ネットワークが機能し、音楽が再生され、私のニーズはすべて満たされたようです。 プリンターで時々何かを印刷したかっただけです。 短い検索の後、公式のドライバーをダウンロードしてインストールし、チェックし（ちなみに動作しました）、再起動しました。 これは多くの人を驚かせるでしょうが、プリンターのドライバーをインストールした後、atdの開始時にハングし始めました。 Samsungとatdのプリンタードライバーの接続方法はどうでしょうか。 尋ねないでください、私は知りません。 まだ分​​かりません 再インストール後、状況が繰り返され、atdおよびSamsungプリンタードライバーの超自然的な接続が確認されました。 3回目の試行はしませんでしたが、すぐにキューの最後のドライブに切り替えました。



そして、現時点での最終段階はOpenSUSEです。 インストーラーは、/および/ homeに個別のパーティションを作成することを自動的に提案することで買収しました（UbuntuとFedora自体はこれを提供しなかったため、 手動でパーティションを構成する必要がありました）。 デスクトップとして、以前のようにGNOMEではなくKDEを使用することにしました。 そして、最も重要なことは、私が必要とするすべて（まあ、ほとんど、私は最後まで正直になるでしょう）が機能することです。 インターネット、音楽、映画、「オフィス」、仕事に関係するもの。 現在、私はこれをすべてLinuxで使用しており、非常に快適に作業できます。 ところで、プリンタのドライバは問題なくインストールされ、動作しました。 :)



エピローグ



はい、今では時々写真を使って何かをするためにWindowsに切り替えます。 しかし、これはLightroomに多くの物が保管されている拠点があるからです。 はい、論文に取り組むためにWindowsに切り替えます（より正確にはラップトップに切り替えます）。 しかし、私の論文はWordで入力され、グラフィックスはOriginで作成され、同じWordのOLEを介して実装されます。 はい、3次元の何かをプレイしたいというまれな瞬間にWindowsに切り替えます。 しかし、ほとんどの場合、Linuxは私の自宅のコンピューターで実行されています。



当分の間、私は何かが欠けています。 通常の方法でSambaを完全に構成することはできません。ホームコンピューターはその名前ではネットワーク上になく、パブリックフォルダーを書き込み可能にすることはできませんが、これについてはまだ詳しく調べていません。 Windows Mobileのコミュニケーターとの同期をとることができません。デバイスが接続されると、コンピューターはそれを認識しませんが、対応するネットワークインターフェイスが作成されますが、これについても詳しく調べていません。 Creative ZENプレーヤーには小さな問題があります。Amarokは、プレーヤーをコンピューターに接続した後に起動した場合にのみ表示されますが、ここではおそらく何らかの方法でHALを構成する必要があります。 OpenOfficeはWordとの互換性が非常に制限されていますが、PDFがあります。 :)これらはすべて個人的に私を悩ませないささいなことです。 さらに、時間が経つにつれて、彼らは彼らを倒すことができるようになります。



最も興味深いのは、現在、LinuxとWindowsを切り替えても、違和感を感じないことです。 過去数年間、私はFirefox、Thunderbird、jMol、The Gimp、Inkscape、VLC Media Playerなどを使用することに慣れています。 そして、これらのプログラムはすべてLinuxにもあります！ 同じインターフェースと機能で！ Linuxで直接利用できないものをWindowsで使用する場合、常にアナログを見つけることができます。 最後の手段として、最後にワインがあります。 一般的に、LinuxとWindowsでは、ほぼ同じ作業環境を自分で作成したことがわかりました。



私は無料および無料のソフトウェアの支持者ではなく、必要なソフトウェアの代金を支払う準備ができていますが、多くのお金を費やすことはできません。 私は、Windowsがバグだらけで、安全でなく、うんざりしているとは思いません（BSODなしで何年も働いてきました）。 Linuxに切り替えたいという願望が正当化されているとは言えません。移行自体が非常に必要です。 私は完全にLinuxに切り替えたとは言えませんが、私はそれにかなり近いです...
