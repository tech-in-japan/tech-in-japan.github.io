  • サッターミル

  • Henrik Warneのブログ


20年の経験を持つソフトウェア開発者、ストックホルムのHenrik Warneによるブログ。 その中で、彼はプログラマーとしての彼の経験と専門職での長年の仕事に基づいてなされた観察について語っています。 このブログは、ITでのキャリアプランニングの詳細を学び、プログラミングの詳細な研究に意欲を持ちたい人に役立ちます。





  • コードのシンプルさ


ブログの著者は、 Code Simplicityの著者であるMax Kanat-Alexander です。 ソフトウェアの基礎は 、私たちの世界には複雑すぎるものが多すぎると考えています。 そのため、このブログはすべてがシンプルであるという考えに基づいており、その主な目標はプログラミングの複雑な世界を理解しやすく、アクセスしやすくすることです。 現在、MaxはGoogleで働いており、 ブログも持っています。 自由時間には、音楽を書き 、ギターを弾き、歌います。





  • martinfowler.com


「私はこのブログを2000年に始めて以来、ソフトウェア開発のさまざまな側面に関する記事を書く怪物になりました」とMartin Fowler氏は言います。 Martinはソフトウェア会社であるThoughtWorksで働いており、 「リファクタリング：既存のコードの設計 とエンタープライズアプリケーションアーキテクチャのパターンの 改善」など、6冊の本を執筆しています。 ブログの特徴の中で、膨大な量の資料と便利なナビゲーションに注目することができます。





  • コーディングホラー。 プログラミングとヒューマンファクター


最愛のStack Overflowの作成者の1人であるJeff Atwood、Jeff Atwoodは、2004年にソフトウェア開発の追跡、思考の記録、作業結果に関するストーリーの共有のためにブログを作成しました。 このブログは、初心者と上級開発者の両方にとって興味深いものになります。 素晴らしいボーナスは、ジェフと直接対話できることです。なぜなら、彼の意見では、ブログは双方向のコミュニケーション形式で構築されるべきだからです。

  • EWONTFIX


Rich Felkerのブログの目標は、ソフトウェアの欠陥やバグ、特にフリーおよびオープンソースに視聴者の注意を引くことです。 EWONTFIXブログ自体の名前でさえ、間違いを訂正または拒否することを拒否する開発者をからかいます。 しかし、著者は単に非難するのではなく、オープンソースソフトウェアの品質レベルを上げ、初心者が間違いを避けるのを助けることを望んでいます。
