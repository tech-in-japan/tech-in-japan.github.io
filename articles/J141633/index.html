商用提供段階でインタラクティブなプロトタイプを使用する練習

ご存知のように、複雑な情報システムのインタラクティブなプロトタイプ（この場合、これは電子アーカイブシステムです）は、開発のさまざまな段階で使用できます。

  • 要件を修正するための要件を収集し、インターフェイスの概念とロジックを開発し、それらを顧客と調整する段階で;
  • プログラマーのタスクを設定するための設計段階。
  • プログラマーのモデルとしての実装段階。
  • テスト段階で、実装されたシステムが顧客と合意したプロトタイプに適合していることを確認します。


私の会社では、最初の3つのオプションが大部分のプロジェクトで使用され、4番目のオプションはあまり使用されていません（プロトタイプは常に参照用に十分に設計されているわけではないため）。 さらに進んで、インタラクティブプロトタイプを使用するための別の興味深いオプションを提供することにしました。



彼らが言うように-私たちは熱いうちに角で雄牛を取る必要があります。 たとえば、企業は情報システムを作成するために公開入札を行っています。 多数の同様の商用オファーから目立つ方法は？

これをインタラクティブなプロトタイプで行います。



私たちのステップ



  1. 入札書類を読む
  2. ユーザーインターフェイスに表示されるシステムの機能は小さいが、私たちの意見では興味深い
  3. 私たちはすべての利用可能な方法で主題領域を研究します
  4. お客様のウェブサイトから企業スタイルを引き出します（ロゴ、タイポグラフィ、アイコン、色など）
  5. 企業スタイルでシステムのプロトタイプWebインターフェイスを作成します
  6. プロトタイプを起動し、音声解説とともに画面からビデオを撮影します。


例：あるロシアの銀行がコンテストを開催しました。 これは彼らのサイトがどのように見えるかであり（後にサイトは全面的に再設計されました）、ここにプロトタイプがあります：







実行中のプロトタイプをダウンロードして、自分で感じることができます（開始するには、designerviewer.exeファイルを開く必要があります）、または内部から表示できる作業プロトタイプをダウンロードします（開くには、GUI Machine Demoをダウンロードしてインストールする必要があります ）



まとめると



  1. 対話型のプロトタイプを作成し、顧客との対話の初期段階で既にこれについて顧客に通知します。
  2. あなたは対象分野に入り、顧客にソフトウェアを販売する意図だけでなく、ビジネスプロセスを研究し改善する意欲を示します
  3. インターフェースで問題の解決策を提示しましたが、ほとんどの人が読んでいないテキストの形式ではありません


これにより、すでにお客様について多くのことを伝え、スケールをあなたの方向に傾けることができます。 当然、これは常に機能するとは限りません。 それがそうであっても、商業提案のプロトタイプ段階に多くの時間を費やすことは受け入れられないことを覚えておく必要があります。 プロジェクトの規模に応じて-数時間から2〜3営業日。 大きな機能を選択しないでください。 狂信に苦しむことはありませんし、ピクセル狩りに従事しないでください。



プロトタイプ自体ではなく、ビデオを顧客に提供するのはなぜですか？



プロトタイプでは、すべてのリンクとボタンを機能させることは非現実的です。 したがって、どこでどの順番で刈り取る必要があるかを知る必要があります。 したがって、ビデオ、および作品のプロトタイプ自体は、お客様に直接お見せします。
