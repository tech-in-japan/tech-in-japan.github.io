30分でMicrosoft AzureにParallels RASを展開する





従業員のリモートアクセスの整理は、適用されるタスクです。 市場には多くのソリューションがあり、十分な落とし穴があります。 カットの下には、「スーツケース」に関するストーリーがあり、30分ですばやく便利に展開します。 これは広告BSとして受け取らないでください。内部には実際の使用方法があります。



Parallels Remote Application Server （RAS）は、仮想デバイスとデスクトップを提供する包括的なツールであり、モバイルデバイスを含むあらゆるデバイスのアプリケーションとデータを操作できます。 購入決定を行う前に、まず特定の条件でそのような決定をテストしたいと考えています。



試してみたい場合は、Azure Marketplaceのフル機能のParallels RASディストリビューションをご覧ください。AzureIaaSで実行されている単一のマシンに本格的なRASをデプロイできます。 どこから始めたらよいかわからない場合は、 写真と入門ビデオを 使用した詳細な手順をご覧ください。AzureでRASをすばやく構成し、テストし、さらには継続的な運用を開始できます。







また、アクセスしやすいシステムの詳細なセットアップが必要な場合は、多くの興味深い機能があります。 以下は、クラウドで高可用性RASシステムを展開および構成する際の専門家であるFrick Bersonの経験です。



フリック・バーソンに会う



Microsoft MVPリモートデスクトップサービスの愛好家であるFreek Bersonは、主要プレーヤーとしてParallels RAS VIPPプログラムに参加しました。 Frickは、Azure Resource Manager（ARM）を使用してデプロイメントを自動化するために多くの実験を行いました。 VIPPの会議の1つで、Frickは、大規模な環境の展開にARMを使用し、高可用性を意味する可能性について言及しました。 以下は、Azureを使用してアクセスしやすいParallels RASの展開を行う方法に関するFrickのストーリーです。



目的



Azure Resource Manager（ARM）は、Azure Cloudに大きな影響を与えています。 ARMでは、宣言型言語を使用して、Azure内のオブジェクトの任意の組み合わせを最も効果的に配置できます。 ARMを使用すると、複数のサブスクリプションにわたって環境を均一に展開し、同じ方法でOTAP環境を構成できます。 ARMでは、オブジェクトのタグ付けとロールベースのアクセス制御を実装できるため、毎月のAzureの請求書に驚きはありません。



高可用性のParallels RASをAzure IaaSに完全に展開するために、新しいARMテンプレートを作成することにしました。 また、Parallelsに連絡して、アドバイスや実験結果の公開を支援しました。



可用性の高いParallels RASを迅速に（再）展開することに加えて、ARMテンプレートを使用して生産的な環境を展開することもできます。 Parallelsの従業員がテンプレートをGitHubに配置するのを手伝ってくれたので、テンプレートを使用して変更することができます。



必要な要件



このテンプレートを使用して正常にデプロイするには、いくつかの要件に従う必要があります。



Active Directoryドメインサービス

Parallels RAS役割サービスは、Active Directoryドメインサービス（ADDS）に依存しています。 テンプレートを実行する前に、ADDSをインストールして構成する必要があります。 最小要件：Azure IaaSで実行されている1つのドメインコントローラーと、ドメイン管理者アカウントへのアクセス。 Parallels RASサーバーを特定のフォルダーに配置する場合は、OU構造を作成します。 この例では、次の構造を使用しました。







Vnetとサブネット

ARMテンプレートは、Azure IaaSに複数の仮想マシンをデプロイします。そのためには、Vnetとサブネットが必要です。 以下は、「ras-vnet」という名前と10.40.40.0/24のアドレス空間を持つVnet構成の例です。







Vnetは、少なくとも1つのDNSサーバーがドメインコントローラーを指すように構成する必要があります。 この例では、ドメインコントローラーは10.40.40.4です。







Vnet内で、「ras-subnet」という名前のサブネットを作成します。これはVnetと同じアドレススペースを持ちます。 もちろん、これは要件ではありません。主なことは、サブネットからドメインコントローラーにアクセスできることです。







このARM展開用にParallels RASコンポーネントが作成される新しい空のリソースグループを作成することをお勧めします。 これにより、ARMテンプレートによって作成されたすべてのコンポーネントを簡単にグループ化し、役割ベースのアクセスを制御できます。 このテンプレートを使用して一時的なテスト環境を作成する場合、専用リソースグループを使用すると、完了時に未使用のリソースを簡単に削除するのにも役立ちます。 この段階ではグループを作成できませんが、テンプレートが展開されるまで延期することはできません。







テンプレートオプション



以下は、ARMデプロイメントの構成に使用されるパラメーターの説明です。













テンプレートを使用する



ARMはGithubからダウンロードできます： ここ



テンプレートを使用する前に、上記のすべての要件が満たされていることを確認してください。 次に、GitHubからJSONテンプレートをコピーし、Azureサブスクリプションにテンプレートを作成します。

これを行うには、Azureポータルで新しいテンプレートを作成し、名前と説明を指定します。







[テンプレートの追加]をクリックします。ARMテンプレートセクションからデフォルトコードを削除し、GitHubからコピーしてJSONコードを貼り付け、[OK]をクリックします。[追加]をクリックしてテンプレートを保存します。







これで、Azureサブスクリプションにテンプレートのコピーがあります。 開始するには、もう一度開き、[展開]をクリックします。







テンプレートを展開するリソースグループとリージョンを決定します。







パラメータ値を指定して、必要に応じてテンプレートをカスタマイズします。 パラメーターアイコンにカーソルを合わせると、その説明が表示されます。 または、上の表を読むことができます。 設定が完了したら、「購入」をクリックします。







展開が完了するまで待ちます。 約30分かかります。



結果



テンプレートを正常にデプロイすると、可用性の高いParallels RASがAzure IaaS環境に表示されます。



使いやすさ



クラウドベースのRASを使用することの便利さを確認するには、RASクライアントをダウンロードして、新しいセッションを作成します。 Azureポータルから取得したパブリックIPアドレスを入力します。









展開設定で指定されたアカウント情報を入力して、作成された接続に接続します。







公開アプリケーションとデスクトップを開いて、エンドユーザーとしてParallels RAS環境をテストできるようになりました。







オプションで、Parallels RAS HTML5ゲートウェイを使用します。 これを行うには、ブラウザを開いてアドレスhttps：/// RASHTML5Gatewayを入力します。 弊社のRASはまだSSL証明書を自動的に設定していないため、それらに関する警告が表示されます。









運営管理



ARMは、パラメーターでコンソールサーバーとして指定されたサーバーにRASコンソールとRAS PowerShellモジュールをインストールします（デフォルト値はRAS-CON-01です）。 サーバーへのRDP接続を確立する場合、RASコンソールとPowerShellモジュールを使用できます。 [スタート]メニューからコンソールを起動し、ARMパラメーターとして登録されているプラ​​イマリパブリケーションサーバーを指定します（デフォルト値はRAS-PA-01です）。 次に、ARMパラメーターのように、管理者ログインとパスワードを入力します。







ARMは、Parallels RASのすべての役割を複数のサーバーに展開しました。サイトの概要ウィンドウには、スクリーンショットに示したものと同様の画像が表示されます。 これで、コンソールを使用して環境をさらに構成できます。
