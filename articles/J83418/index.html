Dropbox-最もシンプルなウェブサイトホスティング

habratopikへのコメントが示したように、Dropboxのみを使用してサイトをホストする方法について言及しましたが、誰もが知っているわけではありません。 つまり、これはファイルを同期するための最も優れた製品の1つであり（さらに、アプリケーションの同期メカニズムとして使用されます）、Dropboxを使用してDropbox / Publicフォルダーに入れてからコピーすることでファイルを簡単に公開できることを誰もが知っていますリンク（公開リンク）。



ただし、これ以外にも、Dropboxはミニプロジェクトの最もシンプルなホスティングとして機能できます。





レシピ

  1. Dropbox / Publicフォルダーに、たとえばSiteフォルダーを作成します。
  2. index.htmlやstyle.cssなど、必要なファイルをその中に作成します-はい、Dropboxはそれらをブラウザに正しく提供し、コンピューターに保存することを提案しません。 html + css（+もちろんjavascript）+データ（画像、xmlなど）を使用できます。 最も重要なことは、相対アドレス指定を使用することです。つまり、CSSスタイルを有効にするには、単に<link rel="stylesheet" href="style.css" type="text/css"/>



    と書くことができます。
  3. コンテキストメニューを使用して、index.htmlなどのメインhtmlへのパブリックリンクを抽出します。 リンクはdl.dropbox.com/u/1234567/Site/index.htmlのようになります。1234567はアカウントに関連付けられたIDです。
  4. 原則として、すべての準備が整っており、通常のアドレスで作成にシーズニングするだけです。 無料のDropboxを使用しているため、短くてわかりやすい住所を迅速かつ無料で取得したいと考えています。 これを行うには、適切なショートカット （ tr.imの方が望ましい ）のサイトに移動します。ここで、 目的の短縮アドレスを指定できます。 これで、ミニサイトがhttp://tr.im/projectnameで利用できるようになりました。 -tr.imサービスは長い間機能していません。 Santiago26によって提案された代替案が好きでした： sn.imリンク短縮 、これはまた、 希望のテキストでリンクを作成する方法を知っています。




アメニティ



ファイルが同期されると、それらはすでにクラウドに保存され、ミニサイトはオフラインになります。 さらに、habra効果に簡単に耐えます（html + css + jsのみを使用して記述されているため、すべてのスクリプトがユーザー側で実行され、サーバーはファイル転送のみを懸念します）。 ページ上の何かを修正したり、新しいものを追加したりするのは簡単です。Dropboxフォルダー内のローカルファイルを操作するだけで、バックグラウンドでサーバーに送信されます（つまり、ftpクライアントについて考えることはできません）。



さて、最後の、しかし重要なプラス-このソリューションは、さまざまなオペレーティングシステムのユーザーに等しく適しています。
