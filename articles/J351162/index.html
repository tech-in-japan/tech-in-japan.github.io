jug.msk.ruのJava Mission Controlに関するAlexey Ragozin

2018年3月13日、モスクワのJava開発者コミュニティjug.msk.ruの最初の会議が今シーズン開催されました。 Alexey Ragozinは、 Java Mission Controlの使用に関するマスタークラスを開催しました。 会議がどのように行われたか、それについて正確に何が伝えられたか、参加者がどのように報告を受け入れたか-このすべてについてさらに。







スピーカーについて



Alexeyはjug.msk.ruで4回目の講演を行いました。 彼の以前のパフォーマンスの印象については、 こちら （2015年6月4日）、 こちら （2016年1月20日）、 こちら （2016年8月30日）をご覧ください 。



jug.msk.ruでの前回のスピーチから時間が経過したため、彼はいくつかの会議で講演者として参加することができました。



  • 「負荷テストの進化-シンプルな自動化からBDDへ」（ SECR 2016 ： ビデオ ）
  • 「あなたのJavaが遅い理由を知っています」（ Devoxx 2016 ： プレゼンテーション ）
  • 「負荷テスト-完全自動化」（ Heizenbug 2016 ： ビデオ 、 プレゼンテーション ）
  • 「JavaとLinux-マイクロ秒のための戦い」（ United Dev Conf 2017 ： ビデオ 、 プレゼンテーション ）
  • 「JavaおよびLinux-操作機能」（ HighLoad ++ 2017 ： ビデオ ）


Alexeyのプレゼンテーションプレゼンテーションファイルは現在Speaker Deckにあります。 技術記事を掲載したブログはここでも利用できます。ビデオとプレゼンテーションファイルへのリンクを含むレポートの完全なリストを含むセクションがあります。



レポートについて



マスタークラスは、昨年の終わりにサンクトペテルブルクのJUG.ruの会議で行われたスピーチの補足および改訂版です。



イベントの発表は前例のない興奮を引き起こし、ホールは完全に満員でした。 珍しい形式（通常、会議はいくつかの一般的なトピックに関するレポート）であり、おそらく関心も高まっています。 ラップトップを取ることをお勧めし、訪問する前に予備的な準備手順を送信しました。



会議前のアレックス。 アンドレイ・コグンは伝統的に参加者を歓迎し、今後の会議を思い出し、 JPoint 2018で彼自身の報告を発表します。







最初の部分では、 Java Flight RecorderとJava Mission Controlの機能の概要とともに、Javaアプリケーションのプロファイリングを行いました。 GitHubで利用可能なアプリケーションをテスト対象として使用しました。







第二部では、デモが続けられました。 この資料は、準備が不十分な対象者（レビュー対象者）と上級者（修飾されたコメント付きのより複雑なケースの提示）の両方に非常に役立ちます。 （休憩中または会議の最後に）質問して回答を得る機会は非常に人気がありました。







会議の終わりに、Alexeyは、 効率的なプロファイリングJavaアプリケーショントレーニングコースについて言及しました 。詳細は、5月21〜22日に読む予定です 。
