1.5時間でOracle Java Cloudに移行する方法

今日、すべての主要なソフトウェア開発者がパブリッククラウドサービスを提供しています。 オラクルはすぐにこのレースに参加しませんでしたが、今日はすでに追いついており、市場の残りを追い越しています。 Oracle Corporationの創設者の1人であるシリコンバレーの伝説であるLarry Ellison氏は最近、Oracleのクラウドサービスは、Oracle Databaseの売上と最高のパフォーマンスの向上を示していると最近述べました。 そのため、現在、同社の注目はすべてクラウドに集中しています。









今日のOracleパブリッククラウドとは何ですか？ これは、クラウドデータセンターのホスティングとアクセシビリティの観点から地球全体をカバーする完全に統合されたサービスであり、最高のOracleソリューションが含まれています。 クラウドプラットフォームはOracle ExalogicおよびOracle Exadataエンジニアリングシステム上に構築され（図1を参照）、世界をリードするWebLogic Serverアプリケーションサーバーバージョン12.1.3であるOracle Databaseを含みます-アジェンダは12.2.1で、最新かつ最も高度なリリースOracleのApplication Server、およびインメモリデータグリッド（IMDG）Coherenceおよびその他のソリューション：トラフィックバランシングは、Oracle Traffic Directorを使用して実行されます。





Javaクラウドサービス



Java Cloud Service（JCS）は、Oracle DataBase、WebLogic、Coherenceを備えたフル機能のインターネットリソースであり、SSHなどのさまざまなプロトコルを介して仮想マシンにアクセスできます。 JCSには2つのバージョンがあります。 本格的な作業に必要なものがすべて含まれるフルバージョンは、Oracle Linuxに基づいており、Standard Edition（SE）、Enterprise Edition（EE）およびSuiteを選択するオプションがあります。 仮想イメージバージョンは、仮想マシンへのアクセスを提供しません。



このサービスを使用すると、Java Enterprise Editionアプリケーションを実装し、テストソリューションとクラスターソリューションの両方で複雑な3層ドメインを構築できます。 ライフサイクル管理は完全に自動化されています。 さまざまな開発者ツール-JDeveloper、Eclipse、NetBeans-がクラウドサービスに統合されています。



Java Cloud Serviceの目的は何ですか？ Java Cloud Serviceの主なアプリケーションは、クラウドでの開発、テスト、実装です。 クラウドは独自のリソースを必要とせず、管理を必要とせず、ライセンスを購入する必要もなく、アプリケーションをホストしてテストするためにクラウド上のスペースのみを必要とするため、クラウドはテストに最適です。 サービスアプリケーションのもう1つの領域は、クラウドへのアプリケーションの移行です。 データセンターのコストと3層アーキテクチャのコンテンツが不足しているため、移行によりコストを大幅に削減できます。構造と規制の管理、バックアップと復元、負荷分散、更新の管理などは不要です。



図2は、Java Cloud Serviceクラウドのアーキテクチャを示しています。 最上部にあるのは、Oracle Traffic Directorによって導入されたLoad Balancer Clusterです。 すべてのアプリケーションの中心および主要な基盤は、WebLogicアプリケーションサーバー（11gまたは12s）です。 Coherenceはトランザクションを高速化するためのキャッシュソリューションです。必要に応じてソリューションで使用できます



クラウドを作成するとき、アクションのシーケンスは重要です。 たとえば、データベースなしでクラウドを作成することはできません。 ストレージなしでは実行できません-これはバックアップが保存される場所などです。計算を実行するComputeサービスなしではクラウドを作成できません。 したがって、最初にストレージ、データベース、コンピュートを作成する必要があります。 他のすべて（図3）はオプションです。



すべてのリソースの変更と修正に関するすべての作業は自動的に行われます。 新しいシステムアップデートの外観が表示され、どの変更を受け入れるか、どの変更をロールバックするかなどを決定します。 したがって、管理はコンソールを使用することになります。 バックアップとリカバリは同じ方法で実行されます。バックアップ、設定、ログの深さを変更し、かなり深い遡及でリカバリを実行できます。



開発者クラウドサービス



Oracle Developer Cloud Serviceは、すべてのアプリケーションのライフサイクル、プロジェクトおよび開発チームを管理する本格的なクラウド開発プラットフォームです。



これは何のためですか？ 新しいプロジェクトのためにソフトウェアや機器を購入しないため、プロジェクトやチームを管理する管理者を雇わず、開発環境との統合を行わないため。 Developer Cloud Service（図4）は、アプリケーションライフサイクルの管理、プロジェクトおよび開発チームの管理、展開と展開の管理のためのツールを含む開発プラットフォームです。 iTrackingおよびターゲットプロジェクト管理のソリューションは、Developer Cloudに直接組み込まれています。



Developer Cloud Serviceの利点は、すべてのコンポーネントの直感的にシンプルなインストール、新しい環境への開発チームの迅速な適応、および管理コストの削減が、開発チームの効率の向上とチーム管理の簡素化につながります。



Oracle Java Cloud Serviceの構築方法



アプリケーションを1時間半で移行するために完了する必要がある主要な手順を示します。



  • Public Cloudの試用版を購読します。
  • OpenStack Storage用のCloudBerry Explorerをインストールして、コンテナーを作成します。
  • クラウド仮想マシンにアクセスするためのSSLキーを作成します。
  • OpenStack Storage用のCloudBerry Explorerを使用して、2つのコンテナーを作成します。
  • weblogicドメインのコンテナ 、
  • データベースのコンテナ 。
  • コンソールからクラウドを起動します。
  • データベースを作成します。
  • WebLogicドメインを作成します。




左の図5は、Oracle Cloudの開始ページを示しています。 このページで、データセンターを選択します。 右側の図5は、ダッシュボードと呼ばれる最初のコントロールページです。 新しいWebLogicドメインを作成するためのいくつかのオプションがあります-Oracle Cloud Virtual Imageまたは本格的なOracle Java Cloud Serviceの作成から選択し、必要な請求オプション、WebLogicのバージョン、Coherenceの受け入れまたは拒否、管理対象サーバーの数、それらのメモリー・サイズなどを選択できますe。すべての質問に答えると、WebLogicドメインが作成されます。



データベースの作成には約40分かかります。 データベースを作成したら、リソースとツールの完全なセットで完全なWebLogicドメインを作成するのに約20分かかります。 アプリケーションの複雑さとデータベースデータの量によっては、アプリケーションのデプロイに約30分かかります。その結果、記事のタイトルで約束されているように、クラウドへの完全な移行には約1.5時間かかります。



また、Enterprise Managerが提供されます。これは、必要なすべての管理機能の管理、監視、および実行を監視するための標準ツールです。 実際、クラウドに移行すると、データセンターで構築したものと同様のインフラストラクチャツールの完全なセットが得られます。これには、追加、削除などのユーザー管理が含まれます。
