デザインのテーマ。 ブラックジャックとウィークリファレンスを使用

かつて、私の前に、タスクはAndroidのアプリケーションのテーマを作成することでした。 私が手に入れたかったもの：



  1. 外観を切り替える機能-いくつかの色とグラフィックを変更する
  2. 変更はオンザフライで行う必要があります。ユーザーはデザインのみを変更し、他のすべて（入力フィールドの内容、リスト内の要素の位置など）は変更しないでください。
  3. 将来的には、たとえば時間帯など、ユーザーの参加なしにトピックを変更できるようにしたい
  4. 既存のコードやマークアップを大幅に変更したくありません。 理想的には、何らかの方法でマークアップの要素をマークしたいだけです
  5. アプリケーションを更新せずに新しいトピックをアップロードできると便利です。




何が達成され、どのように実装されたかについて-カットの下で。







Stack OverflowとAndroidのドキュメントが提供している最も明白な方法は、Context.setThemeです。 ワンキャッチ-すべてのビューを作成する前にテーマをインストールする必要があります。 トピックをその場で切り替えることができないことはすぐにわかります。ユーザーはアクティビティ全体の完全な再作成に間違いなく気付くでしょう。 はい。各アクティビティのコードを何らかの方法で修正する必要があります。 インターネット上で他の推奨事項は見つかりませんでした（情報があれば、リンクに感謝します）。



さて、実装を書きましょう。 ブラックジャックとWeakReferenceを使用。



ポイント4から進めます。私は自分がコードを書きたくない開発者であると考えています。 私はコードを書くのが好きではないので、将来それを書かないために、たくさんのコードを書く準備ができています。 私は自分自身を助けることはできません。次のウィンドウが動的なデザイン変更の影響を受けやすいように表示されるとき、相互作用ロジックを考えたくありません。 要素の横のマークアップで、たとえば、その色が背景色と等しくなることを示したいだけです。



tagプロパティはこれに役立ちます。 たとえば、ListViewでアダプターを最適化するためのHolder'ovとして、アプリケーションタグのどこかで使用されている場合は問題ありません。 コードでは、setTag / getTagをidパラメーターと共に使用できます。 そのような場所がたくさんある場合は、検索と置換が役立ちます。



次に、簡単なタグの形式を考えます。 まず、穀物をもみ殻から分離し、このタグが実際にテーマの使用を示していることを簡単に確認します。タグは常に「！」記号で始まります。 これには、リソースの名前が続きます（「background」など）。 次に、セパレータ、「|」などのリソース、およびテキストの色、背景画像、背景色などのリソースのタイプ。 たとえば、「！Chat | tiled_bg」というタイリングを使用したチャットウィンドウの背景。 たぶん、見た目の美しさではないかもしれませんが、すぐに解析します。 このようなタグを記述するときにミスを最小限に抑えるには、それらを文字列リソースに入れて再利用することをお勧めします-アプリケーションではリソースを使用します！Primary | text_fgは77回使用されます。



最も難しい部分は背後にあり、何らかの方法でこれらのタグを処理するだけです。このようなタグを持つ要素は、「膨張」ビュー中にすぐに処理し、その後テーマを変更するたびに処理する必要があります。 「膨張」は、アクティビティのsetContentViewとLayoutInflaterの使用の2つの方法で発生します。 setContentViewから始めましょう。



このアプリケーションでは、すべてのアクティビティは少数の基本アクティビティから継承されます。 setContentViewメソッドをオーバーライドするだけです：

  public static void manage(View... views) { for (View v : views) { simpleManage(v); if (v instanceof ViewGroup) { ViewGroup vg = (ViewGroup) v; for (int i = 0; i < vg.getChildCount(); i++) { manage(vg.getChildAt(i)); } } } } public static void simpleManage(View view) { Object t = view.getTag(); if (t instanceof String) { String tag = (String) t; if (tag.startsWith("!")) { tag = tag.substring(1); String[] elements = tag.split("\\|"); String base = elements[0]; for (int i = elements.length - 1; i >= 1; i--) { ThemedView tv = createThemedView(view, base, elements[i]); tv.notifyChange(); HotTheme.sViews.add(tv); } } } }


ご覧のとおり、タグを含むビューは階層から引き出されています。 念のため、複数のセパレータ「|」が存在する可能性があると考えています。その場合、リソースは各タイプに適用されます（これは有用であることが判明する可能性があります）。



さらに、これらの要素は特定のThemedViewに変わり、すべての魔法を担当します。 notifyChangeメソッドは、現在のテーマをこのビューに適用します。 トピックの変更を通知するために、将来のためにThemedViewを保存してください。複雑なことは何もありません。

ThemedViewクラス自体は、Viewの単純なラッパーであり、コンテキストリークを防ぎます。 

 private static abstract class ThemedView { private WeakReference<View> view; ThemedView(View v) { view = new WeakReference<View>(v); } boolean notifyChange() { View v = view.get(); if (v == null) { return false; } onChange(v); return true; } abstract void onChange(View v); @Override public boolean equals(Object o) { if (this == o) { return true; } if (o == null || getClass() != o.getClass()) { return false; } ThemedView view1 = (ThemedView) o; View v1 = view.get(); View v2 = view1.view.get(); return (v1 != null ? v1.equals(v2) : v2 == null); } @Override public int hashCode() { if (view == null) { return 0; } View v = view.get(); return v != null ? v.hashCode() : 0; } }
