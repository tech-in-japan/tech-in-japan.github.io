多くの企業はすでに、クラウドコンピューティングの概念に精通しています。これは、高いスケーラビリティ、フォールトトレランス、信頼性を備えたコンピューティングを組織し、使用時に支払いの可能性を備えたリソースを提供するアプローチです。 「クラウド」を実装するための3つの重要なオプション（プライベート、パブリック、ハイブリッド）の中で、最も好ましいオプションを挙げるのは困難です。通常、企業はローカルインフラストラクチャの機能と「クラウド」が提供するサービスを最も効率的に使用するハイブリッドオプションを選択します。



プライベートクラウドは、ローカルインフラストラクチャまたはリモートのいずれかで、顧客専用のリソースにクラウドサービスを実装したものです。 リソースプールがサポートされ、会社のすべての部門に動的に割り当てられます。 プライベートクラウドは、割り当てられたリソースの制御と構成を提供します。



パブリッククラウドにより、顧客は共有されたマルチテナントリソース上で実行されるアプリケーションサービスを展開して使用できます。 拡張性の高いサービスプラットフォームは、従量課金モデルをサポートし、サービスプロバイダーのデータセンターに存在します。



ハイブリッドクラウドは、プライベートクラウドとパブリッククラウドの主要な機能の組み合わせです。



Windows Azureプラットフォーム



マイクロソフトは、すべてのクラウドオプションにテクノロジーを提供しています。 プライベートクラウドを作成するためのツールは、Windows ServerサーバーオペレーティングシステムとSystem Centerインフラストラクチャ管理ツールに基づいており、MicrosoftはWindows Azureプラットフォームをパブリッククラウドとして提供しています。



Windows Azureプラットフォームは、アプリケーションおよびデータをサービスとして作成、ホスト、配信するための、マイクロソフトの高度にスケーラブルで復元力のあるクラウドベースのプラットフォームです。



パブリッククラウドについて言えば、通常、クラウドプラットフォーム内では、インフラストラクチャサービス（「サービスとしてのインフラストラクチャ」とも呼ばれる）、IaaS）、プラットフォームサービス（「サービスとしてのプラットフォーム」とも呼ばれる）、アプリケーションサービスの1つ以上のタイプのサービスを提供することに注意してください（「サービスとしてのアプリケーション」、SaaSとも呼ばれます）。



インフラストラクチャサービスの場合、組織はサーバー、ストレージ、ネットワークインフラストラクチャなどのハードウェアプラットフォームをレンタルします。 プラットフォームサービスを使用する場合、サプライヤはプラットフォームと、さまざまなソフトウェアソリューションの作成に基づいたソフトウェアインターフェイスのセットを提示します。 アプリケーションサービスを使用する場合、アプリケーションはWebインターフェイスを介して配信および使用されます。



Windows Azure Platformサービスは、企業の顧客が関心を持つシナリオとソリューションを実装できることに基づいて、多数のコンポーネントと機能ブロックを提供します。

データストレージコストを削減する



このシナリオでは、Windows Azureストレージを使用してストレージコストを削減します。 継続的なコストには、バックアップの作成とその信頼性、可用性、フォールトトレランスの確保を必要とするファイルストレージの作成とメンテナンスが含まれる場合があります。 明らかな理由から、クラウドストレージは、スケーラブルで信頼性が高く、アクセスしやすく、安全でなければなりません。



多くの場合、リレーショナルデータベースには、ドキュメント、ファイル、グラフィックスなどが保存されます。 これにより、リレーショナルストレージの保守コストが高くなり、バックアップを作成するためにより多くのリソースが必要になり、バックアップの作成とその後のデータリカバリに必要な時間が長くなります。



このシナリオを選択するときは、次のことに注意する必要があります。



  • データストレージコストを削減するためにコンテンツが最適化されたデータベースで動作するアプリケーションに変更を加える必要がある場合があります。
  • クラウド転送レート
  • クラウドストレージからデータを受信するコストは、発信トラフィックのコストです。
  • クラウドストレージからのデータ転送の速度
