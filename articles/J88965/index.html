Java / Python SDK 1.3.2がリリースされ、ロードマップが更新されました





最初に、SDKの変更点について説明し、次に開発者が今後6か月で何を約束するかについて説明します。



だから

-ストアは、データの読み取りとリクエストの実行の2つのモードをサポートします： 強い整合性 （デフォルトで設定されたストアの初期動作）と結果整合性 。 「強整合性」モードでは、すべてのデータ読み取り操作と要求は、以前の変更が完了するまで待機します（変更は行われましたが、分散ストレージのデータに適用する時間がありませんでした）。 最終的な整合性モードでは、これらのデータが変更を待っている場合でも、すべての読み取りとクエリはリポジトリ内の既存のデータのバージョンを使用します。 このモードは、更新データの保証よりも読み取り速度が重要な場合に使用する必要があります。



-ストアは、ほとんどの操作の最大期限をサポートしています。 要求がストレージを超えると、呼び出しが中断され、制御がアプリケーションに転送されます。



-アプリケーション構成（dos。* Yaml * xml *）に登録することにより、DDOSに対する保護を有効にする機能 「好ましくない」IPアドレスとサブネット。 攻撃の疑いがある最もアクティブなIPは、新しいブラックリストセクションのアプリケーションの管理パネルに表示され、設定に入力できます。 先週の水曜日、私はAppEngine開発者（jason @ google）の1人とおしゃべりし、DDOSサービスの開発は絶対に止まらないだろうと言った（国によるフィルタリングなどがあると思う）。



-バイト範囲を設定することにより、アプリケーションからBlobStoreのデータに直接アクセスします。 現時点では、読み取り制限は1メガバイトです（ほとんどのAppEngineサービスに関して）。



-TaskQueueAPIを1回呼び出すだけで多くのタスクをキューに追加できます。また、キューの最大補充速度を最大50 /秒に上げました。



-URLフェッチを通じて、ポート80〜90、440〜450、および1024〜65535を使用してリモートリソースにアクセスできるようになりました（以前は80と443のみでした）。



-PythonSDKと同様に、JavaSDKにAppStatsユーティリティを追加しました。
