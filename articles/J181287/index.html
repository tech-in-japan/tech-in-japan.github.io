Opera Mailを（暗号化された）ドライブに移動します

Operaは最終的にブラウザをメールクライアントから分離し 、オペラをリムーバブルディスクに転送することが最終的に決定されました。 しかし、残念ながら、メールクライアントファイルを転送する通常の方法を見つけられず、シンボリックリンクを介してこれを行うことにしました。特に、構成でオペラが必要とする12個のファイルとディレクトリへのパスを指定するのは人間ではありません。





ステップごとに何をすべきかを指示するコウモリのニックネームを書きました、すなわち：

-オペラフォルダーを％userprofile％\ Appdata \ Localからフォルダーに移動します

- ％userprofile％\ Appdata \ Localフォルダを削除します

- ％userprofile％\ AppData \ Roamingでも同じ

-適切なディレクトリへのリンクを作成して、オペラがappdataで機能すると考えるようにします



ディスクに転送するバッチファイルを作成しました（シンボリックリンクを使用しているため、 Junctionが必要です）。 また、バッチファイルでは、4行目を修正する必要があります。

set local = x：\ Program Files（x86）\ Opera Mail \ junction 、移動先のフォルダーを指定します。



チェックはなく、何をすべきかを言うだけです。 最高の（同じ結果ですが、より高速な）ソリューションを知っている人は、喜んで知るでしょう。 ちなみに、cmd.exeのアクセス許可を台無しにしないように、コピーも移動も使用しませんでした。

copy.comのOpera Mail + junction + move.bat



PSアクションは一時停止で区切られていますが、バッチファイルを開始する前に、何が行われるかを確認してください。



UPDバイクは必要ありませんでした。 Shpankovは、USBドライブにインストールするオプションに親切に注意を促しました。 そのため、メソッドは別のアプリケーションを移動するまで残ります。

このようにUPD2はMSVS2012をシームレスに移動しました。他のヘビー級および/または個人データ（すべてがオペラで開始された）を備えたアプリケーションがそれに続くと思います。
