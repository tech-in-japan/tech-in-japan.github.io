「あらゆるレベルの専門家を教える」：Java開発と会議に関するEPAM





EPAMという名前はほとんどの人に知られています。 しかし、誰もが会社がとりわけバイオインフォマティクスに従事していること、フルスタックの開発者に高く評価されていることを知っているわけではありません。また、社内トレーニング/再トレーニングシステムは後輩だけでなく、時には建築家がメンターとして働くこともあります。



EPAMが積極的に参加するジョーカー会議を見越して、従業員にJava開発とその周辺のすべてについて質問しました。 対話者は次のとおりです。



  • Marianna Okrugina 、サンクトペテルブルクのオフィスのディレクター
  • Alexey Zinoviev 、Java / Big Data Trainer（2016年だけでなくJokerのスピーカー）
  • Vyacheslav Lapin 、上級開発者（2015年だけでなくJokerのスピーカー）
  • Polina Panyukova 、マーケティングスペシャリスト




マリアンナ・オクルギナ



-Java会議への参加を可能にするものは何ですか？



-さまざまな国に多くの顧客を抱えるIT企業として、また従業員にとって、このような会議への参加は、まず、どのプロジェクトを実施しているか、どのような問題や解決策を経験しているのか、経験を交換する専門家コミュニティに示す機会です新しいアイデアを聞きます。



-それではここで教えてください：現在、どのような興味深いプロジェクトを実装していますか？



-多くのプロジェクトがあります。 たとえば、ビッグデータ処理ソリューションの分野では、ヒトゲノムの処理、医薬品および臨床データ分析の分野での研究、セマンティックネットワークに基づく情報プラットフォームの作成、投資ポートフォリオ管理、ケーブルおよびインターネット配信ネットワークの機能分析に関するプロジェクトを実施していますプロバイダー。



さらに、生物医学研究（ライフサイエンス）の分野に特化した部門の1つは、顧客向けのソリューションの開発に加えて、化学およびバイオインフォマティクスの分野におけるさまざまなオープンソースプロジェクトの開発と保守に積極的に関与しています。 たとえば、私たちは多くの重要なオープンソースのバイオインフォマティクスプログラム/ユーティリティの開発と最適化に関与しています。



私たちの貢献により、お客様の特定の生産タスクにおける専門ソリューションの基盤として、ベストプラクティスと経験を活用することができます。 これには、研究結果と特殊なツール（たとえば、PerlやPython、さらにはC / C ++からJavaへの移植）、パフォーマンスの最適化、アルゴリズムとコードの操作の両方で最適なソリューションを見つけることが含まれます。



最後に、同じApache Spark、Apache Igniteなどを含む、並行性に基づく高性能コンピューティングに適応するためのレガシーコードの完全なリメイク。

これはすべて、生物医学研究分野における科学的成果から、研究データではなく臨床データの大量処理のための高性能生産パイプラインの構築への移行の必要性の結果です。



-Javaは会社全体でどのような役割を果たしますか。これらの場合にJavaが好まれるのはなぜですか？



-それは非常に重要です。プロジェクトの75％以上がこのテクノロジーで実装されています。 EPAMは、金融、通信、メディア、生物医学研究、ヘルスケア、オンラインコマースおよびエンターテイメントの分野でアプリケーションを開発しています。Javaは、これらすべての分野で例外なく需要があります。



Javaの選択は、主にお客様の問題を解決するために必要なリソースの技術要件によって決まります。 もちろん、プラットフォームの成熟度、多数の利用可能なソリューションとライブラリ、トレーニングの容易さ、市場での専門家の利用可能性は重要です。



-Javaの場合、どのテクノロジーを使用しますか？



-ビッグデータ処理ソリューションの分野では、Javaで完全に実装されているか、Java APIを提供する次のテクノロジーを使用しています：Hadoop、Spark、Ignite、Flink。







アレクセイ・ジノビエフ



-ジョーカーで何を話しますか？



-私はすでにビッグデータに関する「次のシリーズ」を待っている定期的な聴衆を形成しています。 2014年にJavaでデータサイエンスを行う方法についての話から始め、その後NoSQLとHadoopに深く入りました。 次に続くのはSparkで、これは論理的ですが、Sparkは単純ではなく、リアルタイムです。



夏には、熱くて面白いSpark 2がリリースされました。現時点では、彼と彼の「エンドレスデータセット」、メモリ内のオブジェクトの特別なレイアウト、この革命的だが不特定のリリースの問題に関する一定の戦闘経験があります。 また、データ処理エンジン自体とその友人（Kafka / Samza / MLlib）のジャンクションも調べます。これらのデータは、グループまたは個別に配信されます。



Spark / Hadoopを使用したことがない場合は、Joker で Zhenya Borisovのレポートで最初にSpark APIについて聞くことができます。これは私のプロローグになります。



-誰もがビッグデータを必要とするわけではありません。 EPAMで具体的にどのくらいの需要がありますか？それに関連する多くのプロジェクトがありますか？



-EPAMでは、ビッグデータの専門知識の成長のいくつかのポイントがあり、そのうちの1つはサンクトペテルブルクにあります（私にとっては最も強力ですが、この問題について異なる意見があるようです）、多くのプロジェクトがあり、専門家は非常にひげを生やしていますHDPのような神聖な画像なしでコンパイルされます。



ツールの範囲と数が大きいため、つまり、グラフ、特別なファイルストレージ形式、分析とETL、ニューラルネットワーク、およびその他の機械学習を扱う誰かが既に別の専門分野を持っています。 DevOpsから顧客との交渉まで、さまざまなレベルで行うことがあります。



-あなたはEPAMの「Java / Big Data Trainer」にいます-この背後にあるものは何ですか？



-EPAMには、若年層からジュニア層まで、あらゆるレベルの専門家向けの非常に強力なトレーニング/再トレーニングシステムがあります。 重大なメンタリングプログラムがあり、プログラムの参加者は、一流のプログラマーや建築家の指導の下で、科学と実践の御影石をかじります。



特に、私は2つのプログラムを監督し、その中のトレーニングを読みます。1つのプログラムは自信のある通常の開発者がJavaメモリ管理、Spring、並行性などの知識のギャップを埋めるのに役立ちます。 次のレベルのプロフェッショナリズムに移行し、別のプログラムが成熟したJavaプログラマーにビッグデータスタックテクノロジー（Hadoop、Hive、Spark、Kafka、Cassandra）を使用するスキルを生み出します。



チームワーク、展開、コーディング、全員の前での自明ではないタスクの解決、すべてのコミュニケーションは終わりではありません。 アドレナリンと新しい課題-毎日。







ヴャチェスラフ・ラパン



-EPAMでは、あなたは上級開発者です-あなたは正確に何に取り組んでいますか？



-シニア開発者（D3、私たちが言うように）としての私の公式ステータス、私は今変わりつつあり、私はこの冬にこれを行う予定です。



正式には、私はコーチの半分、1つのバイオインフォマティクスプロジェクトのチームリーダーの半分です。 しかし、私は主に技術革新の実装者として会社に身を置くようにします。 イノベーションは、非常に多くの従業員が所有し、かけがえのないものになった場合にのみチャンスがあり、私のコーチング活動はこれから始まります。 言い換えれば、私は開発者の技術的能力に高度な要求を課すプロジェクトに高度な技術的ソリューションを非常に簡単に提供できる環境を作り出しています。なぜなら、少なくとも私の学生の間では、これらの技術を所有している人々をかなり専門的なレベルで簡単に見つけることができるからです。



-そして、今何を実装していますか？



たとえば、主に多くのプロジェクトがJavaよりもScalaに高度なAPIを提供するSparkを使用しているという事実により、小さなScalaグループの学生を作成するという考え方が最近採用されました。 多数のプロジェクトで、Mavenの代わりに、Gradleを既に使用しています。 一部では、ロンボクを使用しています。



現在、私はフロントエンドグループの集中トレーニングを終えています：JavaScriptのトリックES2015に加えて、それらをトレーニングし、TypeScript 2.0でVueを少し紹介しました。また、ReactとAngular（9月にのみリリースされたAngular 2を含む）も提供されます。 おそらく次のグループは暴動についてもっと話すでしょう。 基本的なJavaを学んでいる人たちに、Java SE 9、Java EE 8で起こることを伝えます。古いグループのために、Spring 5のレビューで資料を準備しています。



私の主な努力は、EPAM開発者の注目をJava 8に引き付けること、その適用範囲を可能な限り拡大すること、そしてそのような移行の冒proを防ぐことに焦点を当てています。 Java下位互換性を使用して、プロジェクトで単純に匿名クラスがラムダに置き換えられ、forループを使用するコレクションの列挙がforEachメソッドの呼び出しに置き換えられる状況を防ぐために、これが遷移の終わりです。



InfopulseのVictor Polischukは昨年、Jokerでのレガシーに関するすばらしいレポートを読みました。アプリケーションの経験がまだない最新のファッショナブルなテクノロジーの使用を人々に許可すると、プログラマーはすぐに大きな苦痛を伴うプロジェクトを作成します。 はい、概してこれはおそらく真実であり、慣性によって多くの経験豊富な開発者、PM、および顧客はこれらの新製品に懐疑的です-彼らは次の生の新製品をプッシュし、無料のベータテスターに​​なるよう説得します... Java 8の新機能に適用した場合、これはそうではありません。実際、Javaはそれ以前の多くの言語のように、単に機能（または「機能的に類似」、Sergei Kuksenkoによって表される）になりました。 だから、私の問題は、問題を勉強しなくても、もちろん「庭を囲う」、「車輪を再発明する」必要はないが、他の関数型言語を見て、そこからこれらすべてのツールを使用する経験を引き出すだけでよいということです。



-そして、ClojureやGroovyなどの言語への関心もEPAMに何らかの形で実装されていますか、それとも「魂のために」ですか。



-関心がJavaからこれらの言語へのプロジェクトの移行につながるわけではありませんが、それは単に「魂のため」だとは言いません。私は主にこれらの言語をある種の実験と考えています。 すべての言語、すべてのテクノロジーが、新しいコーディング技術であなたを豊かにします。 そして、もしあなたが書いたメイン言語が十分に開発されていれば、原則として、他の言語で既にテストされたどんな優れたテクニックでも、実際にその言語で許容できるように実装できます。



たとえば、LombokとJava 8ラムダの組み合わせは、もちろんGroovyの優雅さよりも若干劣りますが、実際のプロジェクトではLombokとJava 8の使用を開始する方がはるかに簡単であり、この組み合わせによりほとんどの実際の問題も解決します。 そしてそれだけではありません。 ある言語で機能を見つけたとき、私はすぐに自分に質問します：それはとても良いですか？ それはこの言語で書く人によって使用されますか、そしてそれはどんな落とし穴がありますか？ それが広く使用され、それが解決する問題がJavaプロジェクトに関連している場合、次の質問があります：Javaで同様の問題を同様の方法で解決することは可能ですか？ そして、原則として、それほどエレガントではありませんが、方法が見つかりました！



-あなたは、JavaとJavaScriptの両方で会議で話します。 EPAMはそのような「マルチステーション」にどの程度一般的ですか、フルスタック開発者がいますか、それとも例外ですか？



-EPAMにはフルスタックの開発者がおり、私が知る限り、プロジェクトが非常に好きです。 実際、フロントエンドまたはバックエンドのタスクがプロジェクトの大半を占める「スイング」が頻繁に発生します。 このため、高度に専門化された開発者は、自分のタスクが「時には空で、その後は厚くなる」という状況に陥り、フルスタックの開発者には常にタスクがあります。 ただし、まったく異なる2つの世界で新しいテクノロジーを追跡し、タイムリーに研究するのは困難です。



-レポートの1つは「ブラウザでの低レベルプログラミング」と呼ばれますが、低レベルプログラミングにどの程度興味がありますか。EPAMで作業する際にそれを処理しますか。



-基本的に、パフォーマンスをチューニングするときに、低いレベルに下げるという問題が発生します。 通常、チューニングはプロジェクトのかなり後の段階で行われます。プロジェクトが既に運用されており、負荷が増加しただけで、コードベースが十分に速く動作していないことがわかります。 しかし、これはバックエンドにあります。 ブラウザーのコンテキストでは、そのレポートで、Webインターフェイス（GUI）とWebインターフェイスのパフォーマンスの違いを完全に消去するレベルにSPAのパフォーマンスをもたらすテクノロジであるWebAssemblyについて説明しました。 私の興味については、コードの理解しやすさとパフォーマンスのバランスの問題を解決し、フォークを可能な限りプッシュすることに興味があります。 私は主にこのプリズムを通して、それぞれの新しいテクノロジーを検討しようとしています。







ポリナ・パニュコワ

-そして、会議で誰かがEPAMで働く機会に興味があるなら、彼は通常同じ都市のオフィスに興味がありますか、それとも他の人に興味がありますか？
