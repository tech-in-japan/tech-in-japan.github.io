NetApp ONTAP 8（SAN）を搭載したWindows 2008/2012

この記事では、 SAN環境でNetApp FAS ストレージを使用してWindows Serverを最適化する（仮想化の有無にかかわらず）トピックを検討したいと思います。



そのようなインフラストラクチャのボトルネックを見つけて解消するには、インフラストラクチャのコンポーネントを決定する必要があります。 インフラストラクチャを次のコンポーネントに分割します。



  • SANを使用したホスト設定（ FC / FCoE ）
  • IP SAN （ iSCSI ）のホストイーサネットネットワーク設定。
  • 実際にOSでホストする
  • ホストアプリケーション
  • ドライバーとソフトウェアのバージョンの互換性を確認する








ボトルネックを検索するには、通常、順次除外手法が実行されます。 最初にストレージから始めることをお勧めします 。 次に、 ストレージを移動します ->ネットワーク（ イーサネット / FC）-> ホスト （ Windows / Linux / VMware ESXi 5.XおよびESXi 6.X ）->アプリケーション。 それでは、ホストに立ち寄りましょう。



SANマルチパス



仮想化せずにOSにLUNを直接接続する場合、 NetApp Host UtilityとDFM MPIOをインストールすることをお勧めします。 DFM MPIOは、 FC / FCoEおよびiSCSIの負荷分散アルゴリズムを追加しますRDMまたはVMFSで仮想化を使用している場合、ハイパーバイザーでマルチパスを構成する必要があります。



マルチパスでは、デフォルトで優先パスを使用する必要があります。これは、 LUNへのパスであるLUNへのパスです。 FCP Partner Path Misconfigured ストレージコンソールのメッセージは、 不適切に構成された ALUAまたはMPIOを示します 。 これは重要なパラメーターです。激怒したマルチパスドライバーがパス間でノンストップで切り替わり、ホストI / Oシステムに大きなキューが作成される場合が1つあったため、無視しないでください。 詳細については、 Windowsマルチパスオプション：FCPおよびiSCSIを参照してください。



NetAppのゾーニングガイドラインの詳細については、写真をご覧ください。

ジャンボフレーム


iSCSIを使用している場合、1Gb以上の速度でイーサネットのジャンボフレームを使用することを強くお勧めします。 詳細については、NetApp FASのイーサネットに関する記事をご覧ください。

シンプロビジョニング


「ファイル」プロトコルNFSおよびCIFSを使用する場合、ファイル領域内の解放されたスペースを返すことにより、シンプロビジョニングテクノロジーの使用を利用することは非常に簡単です。 ただし、 SANの場合、ThinProvitioningを使用すると、空き領域を常に制御する必要が生じ、さらに空き領域の解放（ メカニズムは最新のOSで利用可能 ）は同じLUNの 「内部」ではなく、このLUNを含むボリューム内で発生します。



スペース予約-UNMAP


Windows 2012以降、薄い月からリリースされたブロックの返却がサポートされています。 これはWindowsではデフォルトで設定され、ONTAP側に含める必要があります（デフォルトでは無効になっています）。

ローディングテーブル




Windows 2008以前の場合（ GPTまたはMBRパーティションテーブルは重要ではありません）、 LUNの作成時にWindows 2008ディスクジオメトリを選択する必要があります。

GPTを備えたWindows 2003の場合、 LUNを作成するときに「Windows GPT 」ディスクジオメトリを作成する必要があります。



MPIOを備えたNetAppホストユーティリティ


この項目を無視しないでください。 一連のユーティリティにより、正しい遅延、 HBAのキューサイズ、およびホスト上のその他の設定が設定されます。 接続されたLUNとその詳細情報をストレージ側から表示します 。 ユーティリティのセットは無料で、netapテクニカルサポートサイトからダウンロードできます。 インストール後、ユーティリティを実行します 

 host_config <-setup> <-protocol fcp|iscsi|mixed> <-multipath mpxio|dmp|non> [-noalua]


通常、フォルダにインストールされます。 
 %systemdrive%\NetApp\Windows Host Utilities\


for Windowsの設定は、インストール中に自動的に追加されます。

その後、ほとんどの場合、ホストを再起動する必要があります。



ESXiのWindowsタイムアウト

ハードディスク（ RDM LUN 'a） のタイムアウト （ 5579を参照 ） を設定する必要があります 。 



 Windows Registry Editor Version 5.00 [HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Disk] "TimeoutValue"=dword:0000003c




ファイルシステム


FSは、パフォーマンスをテストするときに大幅な調整を行うことができます。

FSブロックサイズは4KBの倍数である必要があります。 たとえば、生成されたOLTPに似た合成負荷を実行し、操作されるユニットのサイズが平均8Kである場合、8Kを設定します。



ホスト上の最後のファイルアクセスレコードを無効にする


ファイルシステムレベルでは、WindowsレジストリからNTFSファイルシステムのNtfsDisableLastAccessUpdateパラメーターを構成できます。これにより、フォルダーアクセス時間が更新されなくなり、パフォーマンスに非常に良い影響を与えることがよくあります。 これを行うには、タイプREG_DWORDの NtfsDisableLastAccessUpdateパラメーターを値1に設定します。

HKLM \ SYSTEM \ CurrentControlSet \ Control \ FileSystem
