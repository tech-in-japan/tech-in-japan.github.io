VPN over L2TP：エラー678-1つの小さなレジストリキー

自宅で作業中のコンピューター（WinXP SP3）へのリモートアクセスを構成する必要がありました。 一般的なこと-誰とは起こりません。

これを行うには、L2TPを介してVPN接続を構成する必要があります。 特別なことも何もありません（現在、インターネットはルーター、VPN、およびその他のシャーマニズムなしでコンピューターに直接接続されています）。 管理者からすべての設定とキーを受け取りました。 私はすべてを一文字ずつ作成し 、それに応じてエラー678-「リモートコンピューターが応答しません 。 」





Googled-一般的な考え方-接続はありません。ファイアウォール、配線、機器などを確認してください。

VPN接続を再作成し、ファイアウォールを切断して破壊しようとしました。

私は同じトピックでプロバイダーのフォーラムを読みました-同様の質問を見つけましたが、答えも成功したソリューションも見つかりませんでした。

私はそれらのプロバイダーのサポートに電話しました、彼らはポート1701と1723が閉じていると言います（そして、誰もそれらを使用しないまで閉じられます）。 彼らはすべてが開いていると言います（そして、なぜ彼らはそれらを閉じる必要がありますか？）すべて無駄に。



その後、IPSecサービスをチェックするように勧められました-私はチェックしました-オンになり、自動的にロードされます。



インターネットでよく言及される2番目の一般的なアイデア：レジストリキーを設定する

HKLM \ SYSTEM \ CurrentControlSet \ Services \ RasMan \ Parameters \ ProhibitIpSec

「 1 」に等しい-私の過去を含む多くのプロバイダーのL2TP接続問題の解決策に出くわしました。

チェック済み-私は1を持っています。落ち着いて、さらに検索し始めました。

さらに難聴-他のオプションはありませんでした。



私たちはWindowsを再インストールする自殺の考えに徐々に参加し始めましたが、7番目の感情は、この問題はありふれた些細なことであると言いました。



ライブコンピューター上のWindowsを再配置しないように、あらゆる種類のライブCD / DVDを探し始めました。 一般的に、現時点では、L2TP IPSECを完全にサポートするライブCD / DVDにはo_Oがありません。



晴れた日のある日、ProhibitIpSecキーを詳しく調べるという考えで私の新鮮で幸せな頭を訪ねました。これは非常に疑わしいです。IPSecが正しいと思われる場合、これをどのように禁止できますか。 プロバイダーは暗号化を使用しませんが、動作しているコンピューターでは逆に非常に使用されています！

値を「 0 」に設定し、再起動しました-そして見よ、それはうまくいきました！



たぶん誰かにとって、この決定は明白です-私にとっては、悲しいかな、いいえ。 そして、同僚の誰も言うことができませんでした。 テクニカルサポートとGoogleだけでなく。 したがって、私はこの投稿を書きました。
