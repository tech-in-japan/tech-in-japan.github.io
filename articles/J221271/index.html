Ubuntuおよび古いATIグラフィックカード（+ BURGについて少し）

私の話は、古いATIグラフィックカードをお持ちで、標準のmesaドライバーやMonoDevelop-4（小さな趣味）よりも優れたグラフィックパフォーマンスが必要な方に役立ちます。 残念ながら、Linuxディストリビューションでは、寄せ集めのパッケージの前にしか行けないので、残りは密林です。

これは私の最初の記事であり、Linuxシステムにあまり精通していない普通の初心者ユーザーを対象としているため、厳密に判断しないようお願いします。 そして、年長で精通している人のために、読んで、非常に自発的で、必ずしも正しい決定ではないことを笑うことをお勧めします。



Act I. MeとArmadillo Maya（Linux Mint Maya）



ビデオカードの問題、インストールの成功条件を知っていたため、Linux Mint MayaがUbuntu 12.04LTSに基づいているため、インストールに選択されました。結局、私の選択は悪くありませんでした。 しかし、なぜ古いディストリビューションなのか？ AMDはUnixシステムの下で古いビデオカードをサポートしないことに決めたので、スマートに美しく動作するようにスピンしなければなりません。 そして、あなたは一つを選ばなければなりません。 しかし、それは私を満足させませんでした。 したがって、インターネット上の正しいものの検索が始まり、非常に成功しました。必要なすべてのツールを備えたリポジトリが見つかりました。一般に、人件費は最小限に抑えられました。



それでは、始めましょう...ドライバーの要件：



-Xserver <1.13（リポジトリに1.12.4があります）

-Xorg <= 6.9

-カーネル<= 3.4（3.8の修正があると書いているが）



システムが要件を満たしていることを確認する方法：



X -version



のバージョンと理論的にはカーネルを学習します

uname -r



はカーネルバージョンを表示します

aptitude show xserver-xorg



は、Xorgのaptitude show xserver-xorg



バージョンをaptitude show xserver-xorg



します（グーグルの方法、簡単にはわかりません）



その結果、ダウンロードしてインストールしたディストリビューションには、Xserver 1.11.3、Xorg 7.6、およびカーネル3.2.0があります。 ほとんど要件に適合していますが、そうではなかったため、提供することにしました。 リポジトリを追加：



sudo add-apt-repository ppa:makson96/fglrx





sudo apt-get update







リポジトリがインストールされ、キーが良くなりました。リポジトリで利用可能なパッケージのリストを更新し、fglrx-legacyパッケージをインストールします。インストールするための多くのオプションがあり、apt-getで実行しました。



sudo apt-get install fglrx-legacy







すべてがなっています。 再起動すると、正常に機能し、かなり良好です。 ギア（glxgears）のテストでは〜800 FPSが示されました。

X-ssは更新しませんでした。すべて正常に機能したためです。

しかし、私の喜びはつかの間でした...リポジトリからインストールされたmonodevelopは2番目のバージョンであり、MonoGameの新しいアドオンはそれに適合しませんでした。 私は怒っていましたが、あきらめませんでした。Yandexのdebianのリポジトリが見つかりました。debianのパッケージの説明があるサイトも見つかりました。 sources.listに行が追加されました（/ etc / apt /）



deb mirror.yandex.ru/debian stable main contrib non-free







使用可能なパッケージのリストを更新するときに、apt-getはキーペアが見つからないと言った。 キーは、次のコマンドを使用してサーバーからダウンロードされました。



sudo apt-key adv --recv-keys --keyserver keyserver.ububntu.com <ID >







両方の必要なキーがインストールされ、Monodevelop 4バージョンをインストールしようとしました。 しかし、これは失敗です...彼女は私のものよりも新しいバージョンのlibc6ライブラリを必要とします...わかりました、私を破壊し、Debianを代わりに置きます...それは言われました、完了しました... Yを押した...新しいライブラリはそうではなかった、それは私にとって小さなsmall辱だった...「まあ」、私は再び考え、このライブラリの前のバージョンをインストールしようとしました...試みが失敗しました... そして再びGoogle、そして解決策の検索...



行為II。 前後に



Googleは、ローカルリポジトリにパッケージとその説明のリストがある同一のubuntuサイトを見せてくれました。 必要なライブラリが14.04 LTSの最新リリースにあることがわかりました...まあ、それをダウンロードし、設定しました...ウィンドウが遅くなり、目を喜ばせません...まあ、大丈夫です。 ドライバーをインストールするのはそれほど簡単ではないことがわかっていたので、すぐにバージョン3.2.0カーネルを探しました。それが機能していると確信していたので、Unityを消化しないため、すべてがXで行われていません（または単に調理方法がわからない）...判明したように、デフォルトではルートにはパスワードがありません。したがって、このアカウントでの許可は拒否されましたが、修正可能で、X（Ctrl + Alt + F7）に戻り、ターミナルを開いて：



sudo passwd root







彼は新しいパスワードを示し、必要なものはすべて用意されていますが、ペーストするコピーがある場合は同じものをダウンロードする方が簡単だと思いましたが、ギターを使用する必要はありませんでした、カーネルはここのtarアーカイブ（https://www.kernel.org/ ） ダウンロードし、解凍してXを残しました。

これで、sorができたので、アセンブルする必要があります。アセンブル、構成、アンパックされたカーネルのあるディレクトリへの移動の前に、



make menuconfig







しかし、それはうまくいきませんでした。特に彼の答えを掘り下げることなく、彼は二度目の攻撃を行いました。



make config







それで、構成が始まりました... YYYYY ... 50件の質問に答えるのにうんざりしています。その結果、Enterキーを押して、デフォルトの回答に答えています（大文字で強調表示されています）...

これで設定が完了し、アセンブリを開始します。



make







プロセスが長いため、スリープ状態にできます。 すぐにカーネルが組み立てられました。次に、モジュールを組み立ててインストールする必要があります。



make modules





make modules_install







すべてがほぼ正常に完了し、エラーが1つありましたが、dvbに接続されているように見えるモジュールがありました。おそらく、このハードウェアで使用するので、無視しました。 カーネルをインストールするのはあなた次第です：



make install







カーネルは（/ブートディレクトリに）続く場所に移動し、GRUBの自動構成が起動します。 インストール後、再起動し、カーネル3.2がロードされているアイテムを選択します。 システムが起動し、Xが起動しました。今は小さなタスクに任され、Xをロールバックして、ドライバーをインストールします。 そして、私は再びルートの下に、リポジトリを接続し、...fs...そのようなパッケージはありません、aptは私に言った、まあ、私は驚かされず、sources.listに再び登り、データをわずかに修正して、aptがユーザー12.04専用のセクションになったようにしました。 aptitudeを通じて、Xパッケージの正確なバージョンを見つけて、apt-getにインストールするよう指示しました。彼は何か間違ったことをしていると警告しましたが、それが必要だと確信し、ダウンロードとインストールを開始しました。 ...必要なapt-getパッケージを私に配信しようとしても失敗し、aptitudeはインストールをダウンロードして意見の相違を解決できることを思い出しました。 そして...はい、彼はそれをやった、古いXが正常にインストールされた後、操作性をチェックする必要がありましたが、私はすぐにドライバーをロールバックすることにしました。 ドライバーをインストールした後、いつものように、再起動します...それで... Xsが起動せず、ログを読み取っても適切な解決策のヒントが得られなかったので、さまざまなパッケージを軽率に置き換え/削除し始めました...絶望の結果、古いバージョンのMonoDevelopを容認することにしました。 Mayaをインストールし、ドライバーをインストールしました。



Act III。 これは私が期待したノームではありません



Mayaに戻ると、gnomeの環境を配置することが決定されました。結局、この環境が好きでした...小さなパネル、compiz、compizエフェクト...まあ、それは私から続きました：



sudo apt-get install gnome







すべてがうまくダウンロードされましたが、gdmをインストールするとき、彼は立っているmdmが彼と同じファイルを持っていることに激怒しました。 二度削除されたmdmを考えずに：



sudo apt-get remove mdm







そして、彼はgnomeのインストールを再開しましたが、今はforceフラグとyesフラグを使用して（いくつかのファイルに変更を加えるかどうかを尋ね、答えを待ちますが、私は離れることを計画しました）：



sudo apt-get -fy install gnome







すべてがなり、LogOutが実行され、...何かが機能しない古代の慣習に従ってgdmが起動しませんでしたが、GNOMEセッションの選択、ログイン、および呼び出されるように見えるものは、オンとオフを切り替える必要がありました。ノームシェル。 もちろん、2005年から2006年に私の記憶に刷り込まれたthat星を見ることは期待していませんでした。 しかし、私が見たものはかなり異なっていました...インターネット上で調べてみると、拡張機能がシェルを使用する快適さを大幅に向上させることがわかり、原則として、私はそれが好きでした...それは団結よりも優れています

monodevelop-4.0

+小ボーナス（BURG）


Ubuntuをインストールする前に、OSx86をインストールしようとしました。私は自分でインストールしていましたが、別のビデオカードがありました...そして問題なく動作しました。 しかし、それはポイントではなく、カメレオンをインストールしてmakOSIを起動し、コンピューターをポピーのファミリーの1つとして識別しました。また、非常にかわいいです。また、標準のGRUB構成は神経質になってしまいますが、 BURGに出会いました。Googleによると、外国語で書かれたメモはインドネシア語ですが、リポジトリへのリンクがあり、これで十分でした。

最初のステップは、BURG自体を配置し、構成することです。



sudo add-apt-repository ppa:bean123ch/burg





sudo apt-get install burg burg-themes







インストール中、BURGはカーネルを起動するためのパラメーターを尋ねます。



quiet splash



（GRUB構成でスパイ）



ブートパーティションを選択する必要があります。それだけです。BURGには価値があると考えてください。 再起動せずにBURGを見ることができます。実行するだけです：



sudo burg-emu







ブートローダー画像の下部にある指示に従って、テーマを選択できます...しかし、私にとってはあまりテーマがなく、手で仕上げたいという欲求がなかったので、これをもっと早く仕上げたいと思いました。 同じインドネシアのページで、彼らはSuper Boot Managerをインストールすることを提案しています。これを使用して、ブートローダー用のグラフィックテーマを追加できます。 別のリポジトリにあるため、続行します。



sudo add-apt-repository ppa:ingalex/super-boot-manager





sudo apt-get install super-boot-manager







インストールし、起動し、ルートパスワードを要求し、言語を選択します。ロシア語はないようです。英語を選択したので、すべてが明確です。 Burg-mangerの[テーマ]タブをクリックして、好きなものを選択し、好きなものをダブルクリックして、ダウンロードしてインストールし、構成します。
