3CXでのオフィスの結合（パート3. Raspberry PIの使用）

前の記事で、シングルボードRaspberry PIコンピューターをベースにした3CXのSBC（セッションボーダーコントローラー）の代替案についてお話しすることを約束しました。



Raspberry PIはシングルボード（4層のプリント基板）コンピューターであり、サイズはたばこの1パック以下で、いくつかのバージョンがあります。

  • 「A」（ARM1176JZ-Fプロセッサー、256 MB RAM、26 GPIOピン、1 USBポート）、
  • 「A +」（ARM1176JZ-Fプロセッサ、512 MB RAM、40 GPIOピン、1 USBポート）、
  • 「B」（ARM1176JZ-Fプロセッサ、512 MB RAM、26個のGPIOピン、2個のUSBポート、イーサネット付き）、
  • 「B +」（ARM1176JZ-Fプロセッサ、512 MB RAM、40 GPIOピン、4 USBポート、イーサネット付き）、
  • 「2B」（Broadcom BCM2836プロセッサ-4つのARM Cortex-A7コア、1 GB RAM、40 GPIOピン、4つのUSBポート、イーサネット付き）




私たちの目的では、最小限の変更は、Micro SDカードをサポートする変更「B」です。 ただし、モデル2Bをお勧めします。







負荷を考慮した推奨事項：

  • 最大5つの同時会話-Raspberry PI 2モデルB、Yandex市場の平均コストは〜3900ルーブル
  • 最大15の同時通話-Raspberry PI 2モデル2B、Yandex市場の平均コストは〜4,500ルーブル




インストールとセットアップ



鉄片は、いくつかの手順で構成されます。



手順1.オペレーティングシステムのインストール

起動すると、インストールを選択するウィンドウが表示されます。RaspbianOSに興味があります。







OSをインストールしたら、さらに設定するためにSSHにアクセスする必要があります（デフォルトのログイン/パスワード： pi / password ）。



次に、SSH経由で接続します（ブート時に「Shift」ボタンを押して「リカバリモード」に入ります）-OSを更新することをお勧めします（更新コマンド： sudo apt-get update ）。



ステップ2. 3CX SBCをインストールして構成する

3CX SBCをダウンロードしてインストールします （コマンド： wget downloads.3cx.com/downloads/sbc/3cxsbc.zip ）：







インストール（コマンド： sudo sh 3cxsbc.zip ）：







インストール後、構成ファイルを構成する必要があります（コマンド： Sudo nano /etc/3cxsbc.conf ）：



  1. トンネルに接続するためのパスワードを指定します（3CX Webインターフェースで設定：設定/セキュリティ/ 3CXトンネル、ライン）
  2. トンネルのポートを指定します（3CX Webインターフェースで設定：設定/セキュリティ/ 3CXトンネル）
  3. ホスト名を指定します（3CX Webインターフェースで設定：設定/ネットワーク/ FQDN）
  4. SIPポートを指定します（3CX Webインターフェースで設定可能：設定/ネットワーク）








ファイルの場所の構造：

  • /etc/3cxsbc.conf-構成ファイル、
  • /var/log/3cxsbc.log-ロギング、デフォルトでは無効、
  • / usr / sbin / 3cxsbc-実行可能ファイル、
  • /etc/init.d/3cxsbc-デーモン。




その結果、作業図が得られます。







前述のように、このスキームは、15を超える同時会話がないオフィスの統合に適しています。 15以上が必要な場合は、ソフトウェアアプリケーションを使用することをお勧めします。これについては、第2部で説明しました。



注：説明されている推奨事項は、 3CX Shone System v14の最新バージョンへの接続に適しています。
