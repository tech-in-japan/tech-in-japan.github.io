ポータブルコンポーネント、ソフトウェア開発支援



POCOライブラリに関する以前の記事 （ Portable Components ）を続けて、POCOアプリケーションスナップインと、ServerApplicationやConsoleApplicationなどの派生物についてお話したいと思います。

アプリケーションスナップインは、ソフトウェア開発を簡素化し、原則として時間を節約するように設計されています。 このスナップインを使用すると、コンソールアプリケーション、Windowsサービス、およびUNIXデーモンを数分で作成できます。





説明



アプリケーションの派生物は、 コンソールとサーバーの 2つのグループに分けられます 。

機器には、アプリケーションに必要な次のものが含まれます。

  • 高レベルでコマンドライン引数を操作します。 正規表現に基づいてパラメーターをチェックし、整数値をチェックするシステムもあります。
  • UNIXデーモンとWindowsサービスを作成するためのツール。
  • 構成の読み込みを処理します。 この項目は、最新のソフトウェアでは重要です。 この構成により、プロジェクトを完全に再コンパイルすることなく、プログラムの動作を設定できます。 ファイルまたはWindowsレジストリからダウンロードできます。
  • プログラムの初期化と完了。 POCOアプリケーションでのプログラムの寿命は、サイクル： 初期化-アプリケーションの実行-シャットダウンに従属しています。 この順序により、アプリケーションパーツをMainに配置し、重要でないものをすべて隠すことができます。
  • ロギングツール 有能なログ収集システムが時間とお金を節約できることは周知の事実です。 POCOは非常に強力なロギングツールを提供します。 ログは、コンソール、ファイル、Windowsイベントログ、SYSLOGサーバーに送信できます（たとえば、システムのボトルネックがハードディスクの場合）。 これらの方法を組み合わせて、各チャネルに任意の記録形式を設定することもできます。 一般的に、私は間違いなくあなたを紹介します非常に強力なツール。
  • アプリケーションサブシステムの作成、モジュールへの設計、および動的ライブラリへのパッケージ化。 プログラムを再コンパイルせずにモジュールを交換できるモジュール式システムを作成するための非常に便利なツール。

アプリケーション起動オプション


POCOのアプリケーション起動パラメーターは、Optionクラスを使用して実装されます。

各パラメーターには次のプロパティがあります。

  • 氏名
  • 短い名前
  • キャラクター名（1文字）
  • 説明


パラメーターはグループ化でき、オプションにすることもできます。 各パラメーターに値バリデーターを添付できます。 POCOには、2種類のバリデーターが事前定義されています。IntValidator-数値をチェックし、RegExpValidator-正規表現に準拠しているかパラメーターをチェックします。 検証に合格しなかったパラメーターを使用してプログラムを起動すると、プログラムはエラーを返し、すべての可能なオプションを表示します。これらのオプションは自動的に生成されます。 パラメーターの関数ハンドラー（コールバック）を「ハングアップ」できます。これらのパラメーターは、初期化中に使用される場合に呼び出されます。 

 class myApp : public Application { public: myApp(int argc, char** argv) : Application(argc,argv) {} void initialize(Application& self) { cout << "" << endl; loadConfiguration(); //    Application::initialize(self); } void reinitialize() { cout << "" << endl; Application::uninitialize(); } void uninitialize(Application& self) { cout << "" << endl; Application::reinitialize(self); } void HelpHim(const std::string& name, const std::string& value) { cout << "  -   " << endl; } void Configure(const std::string& name, const std::string& value) { cout << "     " << endl; } void defineOptions(OptionSet& options) { cout << " " << endl; Application::defineOptions(options); options.addOption( Option("help", "h", " . ロギングツール


ロギングツールは、4つの主要な部分で構成されています。

  • ロガー
  • チャンネル
  • データストレージオブジェクト（ファイル、データベース）
  • フォーマッタ


ロガーは、アプリケーションがログにデータを送信するためにアクセスするチェーン内のリンクです。 ロギングプロセスの単位はメッセージです。

メッセージは次のものを持つオブジェクトです：

  • データソース（事前に選択されたテキスト値）
  • データ-イベントに関する有用な情報を含む文字列
  • タイムスタンプ
  • メッセージの優先度
  • プロセス識別子（PID）とフロー（TID）
  • いくつかのオプションのパラメーター


優先順位は次の順序で設定されます（低から高へ）。

  • トレース情報（トレース）
  • デバッグ情報（デバッグ）
  • 技術情報
  • リマインダー（通知）
  • 警告
  • エラー
  • 重大なエラー
  • 致命的なエラー


データは文字列で表されますが、他のデータをエンコードすることもできます。 マイクロ秒の精度でタイムスタンプが作成されます。



チャネル -ロガーとデータストレージオブジェクト間のリンク。

いくつかの基本的なチャネルがあります。

  • ConsoleChannel-ご想像のとおり、これは標準出力ストリームSTDOUTにデータを出力するチャネルです
  • WindowsConsoleChannel-std :: clogにデータを出力するWindows固有のコンソールチャネル
  • NullChannel-すべてのデータを拒否します
  • SimpleFileChannel-ファイルへの出力用のシンプルなチャネルで、新しいメッセージが新しい行に追加されます。 ステッチされた最大ファイルサイズがあります。 プライマリが最大サイズを超えた場合、セカンダリファイルを使用してデータを保存できます。
  • FileChannelは、全輪駆動のファイルチャネルです。 アーカイブ、タイムゾーン、圧縮、最大ログ有効期間をサポートします。
  • EventLogChannelは、Windowsシステムイベントログにメッセージを送信できるようにするWindows固有のデータチャネルです。
  • SyslogChannel-syslogデーモンサーバーにメッセージを送信するチャネル。
  • AsyncChannel-任意のチャネルに非同期でメッセージを送信できるようにするブリッジ。
  • SplitterChannel-1つのメッセージを複数のチャネルに送信できるチャネル

クラスを個別のモジュールにする


POCOの基本概念は、すべてのコストでのモジュール性であり、動的ライブラリからの読み込みを可能にする優れたツールであるClassLoaderを使用すると、実行時にそのようなモジュール性を実現できます。

配列をソートするための抽象クラスを実装します。

エクスポートするには、デフォルトのコンストラクタと仮想デストラクタを基本クラスに実装し、純粋な仮想メソッドvirtual string name（）const = 0;を作成する必要があります 後継クラスに実装します。 

 // sort.h class ABaseSort { protected: vector<int> array; //   public: ABaseSort () {} // - virtual ~ABaseSort() {} // virtual string name() const = 0; //  name ,    //    virtual void sort() = 0; //  - void loadVector(vector<int>& lArray) { array.assign(lArray.begin(), lArray.end()); } vector<int> getArray() { return array; } //Xor-swap static void swap(int &A, int &B) { A ^= B ^= A ^= B; } };


次に、バブルメソッドと標準のSTLメソッド（stable_sort）の2つの並べ替えクラスを作成します。 

 //    // sort.cpp #include "sort.h" class bubbleSort : public ABaseSort { public: //   string name() const { return "Bubble Sort"; } //     void sort() { size_t size = array.size(); for (int i=0; i<size-1; ++i) for (int j=i; j<size; ++j) if (array[i] > array[j]) swap(array[i],array[j]); } }; //   STL (std::stable_sort) class stableSort : public ABaseSort { public: //   string name() const { return "Stable Sort"; } //     void sort() { stable_sort(array.begin(), array.end()); } };


エクスポートオプションを追加します 

 POCO_BEGIN_MANIFEST(ABaseSort) //   POCO_EXPORT_CLASS(bubbleSort) //     POCO_EXPORT_CLASS(stableSort) //    stable_sort POCO_END_MANIFEST


プロジェクトを動的ライブラリとしてコンパイルします。

それでは、クラスを使用しましょう。
