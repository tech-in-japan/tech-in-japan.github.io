Inventarium.mobi-フィードバックフォームをアップロードしました！

みなさんこんにちは！

数ヶ月前、Habrahabrについて書きました。 そして、ここで再びあなたと一緒にいます:)思い出させてください-私たちはスマートフォンを通じてユーザーのフィードバックを収集して処理するためのツールを作成しています。



そして最近、 TechCrunch Moscowで新バージョンの製品をリリースしています。



過去数か月にわたって、数百人の開発者や起業家と話をし、ロシアやヨーロッパで行われたいくつかの主要なイベントに参加し、モスクワの上位3つのWindows 8ハッカソンファイナリストに参加しました。 非常に刺激的で、多くのフィードバックをいただきました。 どうもありがとう！



データのコロバン全体を収集して分析することにより、私たちは多くの興味深いオープンを行いました。 すべての開発者は非常に異なっており、製品が異なり、問題を解決するためにまったく異なるツールが必要であることがわかりました。 そして、私たちが提案したものは、小さなカテゴリのアプリケーションとさらに少ない開発者に適していました。

そして、誰もが支配する1つのリングを作成し、より多くのユーザーが幸せになり、開発者がより豊かで豊かになるような製品を作る方法を考えることにしました;-)



要するに、フィードバックを収集するときに使用する方法に注目し、最も人気のあるものを選択して、私たちがあなたを助けることができる方法を見つけました。 何か-今、何か-将来。 次に、最も単純な使用例の1つを分析します。



最初の段階では、SDKを実行せず（ところで、 githubでプルリクエストを待っています）、単純なREST APIに限定することにしました。



リクエストは次のようになります。 

POST http://api.inventarium.mobi/v1/feedback Content-type: application/json  application/xml
      
      

        
        
        
      

    
        
        
        
      
      

        
        
        
      

    
     




 { api_key: "YOUR-API-KEY-FROM-BACKEND", email: "customer email, if exist", name: "customer name, if exist", text: "main feedback text, if exist", subject: "subject of feedback, if exist", data: [{ name: "OS", display_name: "OS Version", value: "iOS 7.0.1", group: "Environment" }, ... ] }




データコレクションの要素には任意の数を指定でき、有用な情報を含めることができます。



サービスのデータは、次の2つの場所に表示されます。

-Inboxは、個々の回答を表示し、ユーザーと通信するためのヘルプデスクです

-フォームフォルダー-収集されたデータによる分析が表示されます



データの収集と分析を開始するには、アプリケーションを登録して作成した後に「フォーム」を追加する必要があります。











ところで、「受信トレイを表示」フラグをクリアすると、受信メッセージはヘルプデスクを通過し、分析のみが利用可能になります。 いくつかのイベントの収集や調査の実施に最適です。



特定の例を見てみましょう。

ほとんどすべてのアプリケーションで見つけることができます。

このようなフィードバックをシステムに追加するには、次のJSONリクエストを送信する必要があります。 



 { api_key: "c40ef00e3f3b42e19ebbbb8fa9027b8b", email: "email field", name: null, text: "feedback field text", subject: "subject field text", data: [{ name: "type", display_name: "Feedback Type", value: "problem ", group: "Ticket" }, { name: "os", display_name: "OS Type", value: "iOS 6.1", group: "Environment" }, { name: "version", display_name: "App Version", value: "1.15 ", group: "Environment " } ] }




また、データコレクションには、イベントログ、ユーザー調整、国、言語など、思いつく限りの情報を必要なだけ追加できます。



[受信ボックス]ページには、次のようなメッセージが表示されます。

サービスの使用はこのシナリオに限定されません。 後で、より興味深い使用例について説明し、表示します。おそらく、あなた自身が既にその使用方法を知っているかもしれません。



フィードバックを収集するだけで、ユーザーに質問したり、調査を行ったり、ユーザーの行動を分析したり、ビジネスに重大な影響を与える前にアプリケーションの問題を傍受したりできます。



これまでのところ、プラットフォームは最も基本的な機能です。 否定的なユーザーのコメントを傍受したり、顧客が新しいニーズを持っていることをタイムリーに理解したりするだけでも十分です。 しかし、それが私たちがあなたを招待する理由です。 あなたにとって最も重要な機能の開発を支援してほしい。 そして、最高のカスタマーサービスをほとんど無料で構築し、新しいBatmobileでお金を稼ぐこともできます！ ;-)
