植物学者が機関を自動化する方法





こんにちは、Habr。 前回の投稿からほぼ2年が経ちました。 多くの人にとって有用であることが判明したことを非常に嬉しく思います。 2年間で多くのことを行いました。 そして、私は私の経験と私の印象を共有したいと思います。



ある国の20のクライアント事業所から、12か国のほぼ4,000に成長しました。 私たちのチームは5倍に成長しました。 モスクワとワルシャワに事務所を開設しました。 彼らはiOとWebに加えて、AndroidとWindowsでもアプリケーションをリリースしました。 機能が大幅に増加しました。 ポスターを5つの言語に翻訳しました。 過去4四半期にわたって、3つの国際展示会に参加しました。 私たちは、エプソン、マイクロソフト、インテルなどの企業と密接な関係にあります。 私たちはいくつかの大規模な投資家によって注視されています。 欧州企業が買収を提案しました。



一見、すべてがシンプルで雲一つないように思えるかもしれませんが、そうではないことを保証します。 最も興味深く、危険なものはまだありません。 実際、最初の段階、つまりアイデアをテストし、製品の最初のバージョンを作成する段階だけを行いました。 次のステップは最も困難な段階であり、企業の内部最適化と再編が行われる形成段階です。 これが私が話したいことです。



小さなイントロ







ポスター-タブレットのレジ。 わずか30分で施設（カフェ、バー、レストラン、ショップなど）を自動化できます。 更新と技術サポートを含むサブスクリプションで支払われるため、通常、実装コストは同様の固定ソリューションの最大10分の1になります（更新と技術サポートがなくても）。



技術サポート



これは私たちの切り札です。 私たちはHoReCa（ホテル、レストラン、カフェ）と協力しており、これは非常に保守的な市場です。 特にCIS諸国で。 欧州の類似物を使用する場合、それらのすべてのTP（技術サポート）は、せいぜいチャットを介してメールで実行されます。 電話なしではできないことをすぐに認識し、社内にホットラインを構築するために多くのエネルギーを費やしました。 （興味があれば、落とし穴についてのコメントでお伝えできます）。



当初、TPは単純でした。 新しいクライアントを接続し、既に接続されているクライアントをリードした責任者がいました。 マネージャーごとに20〜30人の有料顧客に達すると、新しい接続でのパフォーマンスが急激に低下したため、この構造をすぐに放棄しました。 このスキームは、開始に適しています。 しかし、マネージャーは解雇時にクライアントの一部を奪うリスクがあります。 クライアントは、多くの場合、クライアントとの通信の履歴を保持しない1人のマネージャーに慣れます。これにより、クライアントを新しいマネージャーに割り当てることが難しくなります。



次の論理的なステップは「コンベヤー」でした。 部門を、クライアントがプログラムに精通して最初の支払いを行うのを支援する接続マネージャーと、クライアントの現在の問題を解決する技術スペシャリストに分けました。 そして、それは完全に機能しましたが、そのような人は10人未満で、クライアントは1000人以下でした。 クライアントはますます多様化しており、クライアントごとに個別のアプローチを導入する必要があります。



現在、コーヒーハウスの自動化を専門とする個人従業員がおり、仕事中に100を超えるこのような施設の開設を支援しています。 ハンバーガー、フランチャイズ、施設のチェーン、フードトラック、フェスティバルに特化した人がいます...これにより、お客様のビジネスの複雑さをすべて知っており、質問が発生する前でもすべてに対処できるため、より正確かつ迅速に顧客を支援できます。 このようなプロセスを構築するには、多くの労力と労力が必要でした。 これで、TPのすべてのプロセスを説明した内部Wikiができました。



ヒント ：社内のプロセスと特定の状況で何をすべきかを説明するWikiをすぐに入手する必要があります。 これは、新しい従業員が同じ熊手を踏まないようにするために必要です。そのため、意思決定が明確でバランスが取れており、気分に依存せず、他の従業員の謎にならないようにします。



従業員研修



新入社員のトレーニングには特に注意を払っています。 ミニ認定センターのようなことをしました。 仕事に来る各従業員は1週間のトレーニングを受講し、最後に会社に製品のプレゼンテーションを行います。 トレーニング中に、彼は理論と実践のためにいくつかのテストに合格します。 経験豊富なマネージャーが彼に配属され、彼はすべてをより早く学ぶのを助け、トレーニングを成功させることに興味を持っています。 これにより、プロセスをストリーム化し、1か月あたり2〜4人の新しい従業員を簡単に採用してトレーニングすることができました。



ヒント ：新入社員のトレーニングプロセスは、事前に取り組む必要があります。 私たちはこの瞬間を長い間延期し、結果として多くの時間を失いました。 Lean Startupを読んでください！ 私は以前にそれを読みました-私は多くの時間を節約します。



TPのテクノロジー



顧客数の増加に伴い、TPへの呼び出しの数も増加します。 以前は、通常のメールを使用していたため、対処できませんでした。 Grooveチケットシステムを実装しました。 CRMと統合し、すべてが時計のように機能するようになりました。



仕事でチャットを積極的に使用しています。 最初にZopimを試しました 。 クールだが高価。 SiteHeartに切り替えました（小規模なチームと単純なタスクに最適）。 半年前、 Intercomを発見しました。 私たちはそれに切り替えました。 最初に登録ユーザーが転送され、次にすべてが転送されました。 数ヶ月前、私は競合他社もそれを使い始めたことに気付きました。これは良い兆候です。



内部通信にはSlackを使用します。 11月に、詳細な分析のためにMixpanelが導入されました。 タスクにはTrelloを使用します。 これはすべて結びついています。 現在、クライアントからのアピールがある場合、マネージャーがCRMでそれを修正し、プログラマーのタスクとステータスを監視するための部門長へのメッセージに分類されます。



ヒント ：問題が2回以上発生する場合は、問題を自動化し、プロセスを説明する必要があります。



開発



開発全体をプロジェクトチームに分割しました。 各プロジェクトには、プロジェクトを最初から最後までリードし、リソースを割り当てる責任者がいます。 ここでは珍しいことはありません。 ルールがある場合を除き、月に一度、各開発者は顧客と製品に対する責任をよりよく理解するために、テクニカルサポートと一緒に1日作業する必要があります。



ヒント ：社内の全員がユーザーを感じる必要があります：フィードバック、問題、ニーズ。 これはマネージャーの問題や責任ではなく、各従業員の仕事です。



メンタリティ



最近、私たちはヨーロッパやアメリカの類似製品のチームと頻繁に連絡を取り合っています。 展示会で注意深く研究しています。 これらの企業のトップ10を選べば、すべての企業が販売に関して先を行っています。 ただし、在庫、原価計算、マーケティングキャンペーン、モディファイヤ、会計処理などの重要な機能を備えているのはそのうちのいくつかだけです。 顧客の90％がこれらの機能を必要とするため、これなしではCIS市場で作業することはまったく不可能です。 ヨーロッパでは、すべてがよりシンプルです。 顧客は機能について妥協する準備ができており、クラウド製品のコストが10分の1であることを認識しています。 CIS市場では、すべてが異なります。 ほとんどの場合、価格に大きな違いがある場合でも、顧客は必要な主要機能の不足に耐える準備ができていません。これにより、新しいチームのエントリーしきい値が非常に大きくなります。



ちなみに、固定システムはこれをよく使用します。 彼らはバグと貧弱な機能を備えたクラウドバージョンをリリースし、クライアントが彼らに来て、彼らの静止製品のために10,000ドルを払う機会がない場合、彼らは彼に彼らの不安定なクラウドバージョンを売ります。 しばらくすると、クライアントは彼らを呼び出し、thisし、「これはクラウドソリューションであり、すべて予算が非常に高いため、価格が高いため、安定性が必要な場合は、ローンを取り、固定ソリューションを購入する」と長い間言ってきました。 そのようなクライアントの後、この会社がそのような未完成の製品しか持っていないことを確信することはすでに困難です。 クライアントは現在、すべてのクラウドソリューションがうんざりしていると確信しています。



ヒント ：アイデアをテストします。 現場に行き、顧客とチャットします。 海外の同僚の経験は欺くことができます。



API

ヒント ：開放性に努めます。 知識、ベストプラクティスを共有します。 ただし、サポートする準備ができておらず、その責任を負う人がいない場合は、APIをリリースしないでください。 APIのサポートと開発は継続的なプロセスです。



季節性



この秋、非常に予想外の現象に直面しました-顧客のビジネスの季節性です。 ポスターはサブスクリプションモデルで配布されるため、年間4〜6か月間営業する季節限定の施設に最適です。 私たちはこれを知っていましたが、そのような規模に備えていませんでした。 その結果、10月から11月のシャットダウン率は通常の15倍でした。 しかし、それは文字通り2ヶ月であり、私たちはすぐにこの現象に対処しました。 成長モデルでは、これらの機関を少し異なる方法で考慮します。



ヒント ：急なジャンプの準備をしてください。 それは、ポリシー、コース、または他の何かである可能性があります。



期待と現実







ここで、私たちの製品に関する記事と、実際のテストに合格しなかった方法についての最も人気のあるヒントを表明したいと思います。



1.まず、Androidでアプリケーションをリリースする必要があります。これは安価で人気があります。


最も人気のある誤解。 ほぼ1年前にAndroidをリリースし、Androidタブレットで作業しているお客様はわずか5％です。 不快なタッチスクリーンを備えた中国のタブレットだけが常にハングアップし、使用中のiPadよりも安く動作します。 ビジネスを迅速かつ安定して失敗なく動作させるには、SamsungまたはNexusを使用する必要があります。 そして、ここでの違いはすでに重要ではありませんが、WOW効果はiPadにのみ存在し、50ドル以上かかります。 安定したタブレットでさらに100ドルを節約することは現実的ではありませんが、企業が3000ドルから5000ドルを節約するのを支援します。



2.倉庫/経理/ ...がありません（自分で代用してください）。


今、私たちはほとんどすべてを持っています。 倉庫がなかったとき-私たちは今倉庫を作り、販売が行くと思いました。 簿記も同じでした。 しかし、現実には、顧客の半数以上が機能の20％しか使用していません。 実際、システムの使用開始から倉庫と技術マップを使用している顧客は半数未満です。 この分析を行ったとき、私たちにとってショックでした。 現在、ほとんどの機会を利用する人の数を増やすために、基本的な機能についてクライアントを徐々に教育することに取り組んでいます。



3.各ウェイター/クライアント用のタブレット。


施設内の複数の端末を使用しているお客様は20％未満です。 もちろん、それは経済的により収益性が高くなります。ウェイターはホールとキッチンの間を急ぐことはなく、その負荷は減少します。その結果、必要なウェイターは少なくなります。 しかし、実際には、これを計算して実装する所有者はわずかです。



4.美しい端末インターフェースが最も重要です。


システムは、ウェイターではなく所有者が購入します。 ほとんどの場合、所有者は端末がウェイターを探す方法を気にしません。 ウェイターは気にしません 主なものは、ターミナルが迅速に動作する必要があり、ウェイターがより多くのゲストにサービスを提供し、より多くのヒントを獲得できることです。 タブレットのすべての製品に写真を追加する顧客はわずか25％です。 そして、ほぼ同じは、彼らの製品の単一の写真を追加しません。



5.サブスクリプションはクライアントを「フック」状態にし、常に支払いを強制します。


別の一般的な誤解。 「フック」には、固定システムが植えられています。 結局のところ、彼が機器とシステムに5000〜7000ドルを支払った後、彼はそれらを放棄したり返品したりする機会はほとんどありません。 更新と技術サポートの料金を支払い、数時間待つ必要があります。 当社では、クライアントはいつでもすべてのデータをエクスポートでき、支払いを停止して別のシステムに切り替えるだけです。 彼はこれを知っています、私たちはそれを知っています。 したがって、クライアントが常に満足し、そのような誤解がないように最善を尽くします。



ヒント ：顧客が製品をどのように使用しているかを学び、製品との統合を改善します。 彼らがコメントに書いていることを信じないでください。 数字だけを信じてください。



今年、私たちは膨大な数の問題に直面しました。 私たちは多くの問題を解決することができました。 私が理解した最も重要なことは、ビジネスはプロセスであるということです。 これは有限ではありません。 問題を解決することはできません。 時間の経過とともに変化し、ソリューションは機能しなくなります。 何か新しいものを考え出し、最適化する必要があります。 止まって観ることはできません。
