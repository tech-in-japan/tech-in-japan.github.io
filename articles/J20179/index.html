4GB RAM





長い間、どのブログに投稿するかを考えていましたが、思いつきませんでした。

記載された問題は、かつて私に神経細胞の塊を奪い、将来（RAMの価格が下がる）より多くの人々だけがそれに直面するので、私は知っているすべてを共有することにしました。



問題は単純です。コンピューターに4ギガバイトのRAMを入れましたが、OSは寛大さを評価しませんでした。



少し考えた後、OSが32ビットであるため、アドレス可能なスペースは4GBであることを思い出します。 問題は、RAMだけでなく他のデバイスによっても占有されることです。 グラフィックカードを含む。 そして今、あなたの4ギガバイトからせいぜい3年半です。 それは残念ですか？ それは残念です。



すぐに動揺したい。 タンバリンを使用しない32ビットXPおよびVistaでは、32ビット以外のものを表示する必要があります。

交差点



次の選択肢があります。

  • ハンマーで打って生きるのが最も厄介な方法です、私見
  • 64ビット軸を提供 -結局のところ、64ビットシステムにはまだ互換性の問題があります。 片方を治した後、もう片方を不自由にします。
  • サーバー32ビットを配置します。これについて詳しく説明します。


PAE



Windows 2003 Server Enterpriseの魅力は何ですか？ 彼女はPAE（Physical Address Extension）を最大限に活用できます。 XPにも存在しますが、SP2以降、互換性のためにアドレス空間を拡張することはなく、DEPの作業をサポートするためにのみ使用されます。 このテクノロジーにより、36ビットのアドレス指定を使用して、4ギガバイトを超えるクロールアドレスにアクセスできます。 Pentium Pro以降のプロセッサでサポートされています。



美味しいですか？ はい PAEを使用すると、Windowsはかなりの量のメモリを使用できますが、32ビットのままです。 ただし、アプリケーションには引き続き2GBの仮想アドレススペースが割り当てられます。 これは、多くのリソースを集中的に使用するプログラムを使用すると、利益を感じることを意味します。 Photoshopを「フィード」したい場合は、何も変わりません。



PAEはAWEテクノロジーも追加します。 プログラムは、アドレス空間で2ギガバイトの「ウィンドウ」を移動することにより、この制限を回避できます。 ただし、残念ながら、ほとんどのアプリケーションはこのテクノロジーをサポートしていないため、考えることはできません。 RazvitはMSSQL Serverですが、photoshopでは動作しません。

問題



かなりバラ色？ すぐに喜ぶ。 忘れないでください、これはまだWindows 2003 サーバーです。 無効になっているマルチメディア機能はすべて簡単にオンになりますが、これは問題ではありません。 そして、システムは鉄に安定します。 しかし、バージョンチェックは好きだが、システムは好きではない、多くの原則に基づいたメーカーに出会うでしょう。 たとえば、HP、Logitech（ちょうどここで先日 ）。 幸いなことに、タンバリン（互換モードとApplication Verifier）の助けを借りて、すべてを完全に機能させることができます。



確かに、一部のプログラムがカーネルのメモリをクロールすると、明らかに半分が見つからないという予期せぬ事態に遭遇するリスクがあります。 それらの1つはソースエンジンです。 私のポータルは1回おきに起動します...



また、理論上、一部のドライバーは正しく動作しない場合があります。



しかし、私はこれで3か月間暮らしており、ほとんど問題なく、とても満足しています。 私はブルースクリーンを見たことがない、パパ。



では、このPAEはどのように適用されますか？



Microsoftのスタッフは非常によく説明しています： support.microsoft.com/kb/283037

ただし、念のため、PAEキーに加えて、BIOSでメモリ再マッピングオプションを有効にする必要がある場合があります。 チップセットの設定で見つけました。
