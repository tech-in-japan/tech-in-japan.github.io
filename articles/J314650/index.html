RHCVA（Red Hat認定仮想化管理者）試験の準備経験

少し前まで、私は幸運にもRed Hat Virtualization（RH-318）コースを受講し、対応するRHCVA試験-Red Hat Certified Virtualization Administrator（EX-318）に合格しました。 残念ながら、Habréで同様のトピックは見つかりませんでした。そのため、試験の準備に関する自分の経験を共有します。



明らかな理由により、試験とその内容に関する情報を公開することはできません。そのため、適切なタイミングで準備するのに十分なオープンソースのみを使用します。



EX-318試験は、Red Hatから仮想化を管理する知識とスキルを証明したい専門家向けに設計されており、RHCAステータスを取得するために必要な認定のライン-Datacenter（Red Hat認定アーキテクト）にも含まれています。 建築家の地位に興味のある方は、 こちらをご覧ください 。



この試験では、ITをRHCSA（Red Hat認定システム管理者）レベルの専門家または数年のLinuxシステム管理で表明された同様の知識を持つ専門家に渡すことをお勧めします。



この文を解読し、自分自身の経験を上に課す場合、エントリのしきい値はおよそ次のとおりです。



1.仮想化に精通している必要があります（KVMの仕組みを知る必要はありませんが、一般的な原則を知っている必要があります）。

2. FC、NFS、iSCSIなどのテクノロジーに精通している必要があります。

3.少なくともネットワークの基本的な理解、VLAN、ボンディングとは何かを理解する必要があります。

4. LinuxシステムとCLIでの作業の基本について少し理解している。 仮想化はRed Hatからのものであるため、Red Hat Enterprise LinuxおよびCentOSを意味します。

5. yum、less、grep、wget、rpm、viを知って使用できるようにします。



この仮想化で作業を開始するには、Linuxシステムの高度な管理者である必要はなく、ネットワークとストレージの第一人者である必要もないため、「熟知する」と「少し理解する」という言葉を意図的に使用しました。



この試験の準備について



上記のすべてのポイントに精通していると仮定すると、仮想化自体に対処する必要があります。



Red HatのWebサイトで、試験に合格するのに役立つ必要なスキルのリストを確認できます。



1. Red Hat Virtualization Manager（RHEVM）をインストールします。

2. RHVハイパーバイザーのインストールと構成。

3.仮想データセンターの作成、ホスト、ディスクストレージの接続、ネットワークのセットアップ。

4.ユーザーとロールの管理。

5.インストールメディアのインポート。

6.仮想マシン、テンプレート、およびVMプールの作成。

7.管理およびポータル管理。



合格するための最も忠実で信頼できるオプションは、RH-318 Red Hat Virtualizationコースです。このコースでは、この仮想化システムのほとんどの側面と、制御部、ハイパーバイザー、仮想マシンの展開における対応する実践スキルを説明します。 このコースに参加することを強くお勧めします。



自己準備



Red Hatポータルには、Red Hat Virtualizationを含むすべての製品用の優れたライブラリがあります。 大きなリスト全体から、次のドキュメントを読むことをお勧めします。



1. Red Hat Enterprise Linux。 システム管理者ガイド 。 RHELに不慣れな人、または知識を更新したい人向け。

2. インストールガイド 。 Red Hat Virtualization Managerのインストールと設定。

3. 管理ポータルの紹介 。 管理ポータル、外観およびナビゲーション。

4. 管理ガイド 。 実際、システム管理に関する最も重要な文書。

5. ユーザーポータルの紹介 。 ユーザーポータル。 外観とナビゲーションの基本;

6. 仮想マシン管理ガイド 。 仮想マシン管理。

7. テクニカルリファレンス 。 もう少し掘り下げたい人のために。 アーキテクチャのより詳細な説明と、その仕組みのすべて。



ドキュメンテーションを学習する際、最も興味深い情報と「数日で忘れがちなもの」を別の虎の巻で取り出して、必要なときにすぐに思い出すことを強くお勧めします。 これは、コマンド、パラメーター、キー、パスなどを保存するための素晴らしいオプションです。



ドキュメントを読むとき、最良のオプションは、実際に読んだすべてをチェックすることです。 テストベンチには、2台のサーバー\仮想マシンと共通のストレージで十分です。 可能であれば、すべてのストレージ接続オプションを使用することをお勧めしますが、FCを省略できる場合は、NFSおよびiSCSIを使用することを強くお勧めします。



すべてを実際に試すには、2つのオプションがあります。



企業アカウントを使用してRed Hatポータルに登録し、Enterprise LinuxおよびRed Hat Virtualizationの試用ライセンスを取得します。 入手したライセンスをガイドの助けを借りて使用すると、システムのセットアップと仮想マシンの管理を簡単に練習できます。 サブスクリプションマネージャーに精通する必要があるため、このオプションはやや複雑になりますが、Red Hat製品を真剣に扱う場合は、これを学習する必要があり、さらに、必要なすべての手順はRHVのドキュメントで説明されています。



2番目のオプションは、CentOS 7とoVirt仮想化システムを使用することです。 私にとって、このオプションは、Red Hat製品とそれらが成長するシステムとのわずかな違いを考えると、より受け入れられます。 仮想化環境のセットアップと管理の違いはごくわずかであり、試用ライセンスを使用する必要はなくなり、仮想化は機能し、サブスクリプションなしで更新されます。



oVirt Webサイトにある小さなドキュメントには、RHEL / Fedora / CentOSにシステムをインストールするプロセスが記載されています。 それ以降のすべてのシステム管理手順は、Red Hatポータルで提供されるドキュメントに類似または可能な限り近いものです。



結論として



この試験の準備についてお伝えしたかったのはそれだけです。 仮想化技術とLinuxシステムに関する知識は準備に大きなプラスとなり、ドキュメントと実践を読むことはほぼ成功です。

この試験に合格する計画がない場合でも、この記事で提供されるドキュメントによると、仮想化と実践を安全に展開できます（誰かが気に入るかもしれません）。 さて、この試験を受験する人たちに、幸運を祈ります。 この情報があなたのお役に立てば幸いです。



ちょっとしたニュアンス



現在、バージョン4の仮想化システムはすでにリリースされていますが、新しいバージョンの試験はまだなく、バージョン3.5でのみ利用可能です。 バージョン4と3.5の間に基本的な違いはありません。したがって、新しいバージョンのドキュメントをすぐに読むことをお勧めします。これは試験に大きな影響を与えないはずです。
