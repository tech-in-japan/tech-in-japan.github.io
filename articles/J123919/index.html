iPadアプリ


iPadアプリケーションの完成には1週間しかかかりませんでした。 ほとんどの時間は、ユーザーインターフェイスの再設計に費やされました。 iPadとiPhoneの画面解像度の側面は異なるため、インターフェイスの設計時にこれを考慮しなかった場合、問題が発生します（特に多数の画面がある場合）。

アスペクトを変更せずに1024で480になる場合、結果の画像は480x360になります-垂直方向に40ピクセル余分になります。 1つの簡単な解決策は、上下に空のスペースを残すことです。 私はこのアプローチが好きではありません。 画面は3つしかありません（ベクター内のほぼすべて）ので、ソースマテリアルを1024x768の拡張子に再編集し、480x320に蒸留し、上、下、または両側から不要な特定の部分をトリミング（シフト）するユーティリティを作成しました。



Twitterの投稿ダイアログを完全に書き直しました。古いダイアログはiPad用に設計されていませんでした。

バイナリファイルをユニバーサル形式に変換するには、1回クリックするだけでした。 ターゲットを選択し、その上で「このターゲットをiPadにアップグレードします」。

iPadではなくiPadをプログラムで認識するために、次のコードを使用しました。

BOOL isPad()

{

#ifdef UI_USER_INTERFACE_IDIOM

return (UI_USER_INTERFACE_IDIOM() == UIUserInterfaceIdiomPad);

#else

return NO;

#endif

}







HDまたはHD


一部の開発者は、iPad専用にHDバージョンを作成しています。 私の意見では、次の理由から：

  • 新しいアプリケーションはそれぞれ、新しいアプリケーションのリストで少なくとも1日ハングします（AppStoreの更新は「新しい」カテゴリに表示されなくなります）
  • HDアプリはHDよりも高価です
  • iPadアプリケーションははるかに大きく、iPhoneのリソースと合わせて非常に大きくなっています。


私は、次の理由でユニバーサルアプリケーションを作成することにしました。

  • 私のアプリケーションのサイズはわずか+ 2MBです
  • 一度にすべての価格を引き上げると、iPadバージョンがより高価になる理由はわかりません。
  • 2つのプロジェクトをサポートする方が4つよりも便利です（HDの場合は、MyApp、MyApp lite、MyApp HD、MyApp Lite HDです）。

    LiteではなくLite



    すでに書いたように、ライト版は有料よりも早くリリースされました。

    私はまだそれを正しく行う方法を自分で決めていませんが、いくつかのオプションを分析できるようにします：

    • 両方同時に-アプリケーションは隣接する行に表示され、プレーヤーにはすぐに表示されます。 ライトをプレイすることができ、必要に応じてすぐに有料のものを購入します。 このオプションには欠点があります-軽いものと軽いものの両方のアプリケーションは、新しいもので1日だけハングします。 このため、2つのリリースが推奨されます。各リリースは、互いに独立して日ごとに新しいリリースでハングします。
    • すぐに支払い、後で無料で-私の意見では、唯一のマイナスは、製品にお金を払わずに感じることができないことです。 そのため、無料バージョンが翌日にリリースされ、2つのアプリケーションが1日の差で新しいものにぶら下がることになります。
    • すぐに無料で後で支払われる-ここでマイナスは、無料でプレーして有料のものを望んでいるプレーヤーが失望することです。
    • inappを介してすべての有料コンテンツを開くことができる1つのバージョン




    私の意見では、理想的な選択肢は次のとおりです-できるだけ早く（トレイルまたは1日で）支払われ、できるだけ無料です。 あなたにとって都合の良いリリース日を指定するだけです。たとえば、1日前またはそれ以前に点灯します。 私の意見では、ほとんどの場合、問題はすぐに、しばらくしてから無料で支払われます。

    これが私の個人的な観察であることをもう一度強調します。
