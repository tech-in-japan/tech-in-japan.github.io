Bluemixを使用して、スマートフォンをIoTセンサーに変えてください。 アプリケーション開発の例





Bluemixは、クラウド内のアプリケーションを迅速かつシームレスに作成、デプロイ、および管理できる強力なプラットフォームです。 Bluemixは、Cloud Foundryオープンソースソフトウェアに基づくIBM Open Cloud Architectureアーキテクチャの実装であり、Platform as a Service（PaaS）に基づいて機能します。



このプラットフォームとIBM Watson IoT Paltformサービスを使用して、スマートフォンから受信したデータを処理、視覚化、保存できるBluemixアプリケーションを開発できます。 この記事では、通常のスマートフォンをネットワークに接続された信頼できるセンサーに変える方法を示します。



アプリケーション開発には何が必要ですか？



1. Bluemixアカウント（無料トライアルに登録できます）。 DeveloperWorks Premiumプログラムのメンバーは、IBM Bluemixの年間サブスクリプションとBluemixクラウドコンピューティングの240ドルのクレジットも受け取ります。

2. Cloud Foundryコマンドラインインターフェースもダウンロードしてインストールする必要があります 。

3. AndroidまたはiOSを搭載したスマートフォン。







上記のすべてを受け取った直後に、BluemixでIoTアプリケーションの作成を開始できます。 これを行うには、Node-REDメカニズムを使用します。 これを使用して、たとえば、モノのインターネットサービスを作成して、MQTTメッセージをクラウド内のIBM IoTサーバーに送信したり、サーバーからメッセージを受信したりできます。



アクションのアルゴリズムは次のとおりです。



1. アカウントでBluemixにログインします （または無料トライアルに登録 します ）。

2.カタログを選択します。

3. Internet of Things Platform Starterをクリックします。

4.アプリケーションの名前を指定します。 この名前はホスト名としても使用されるため、例えばiot <your_name>のように、Bluemix内で固有でなければなりません。 この例では、 iotdemodaniel4



という名前iotdemodaniel4



ます。







5. [作成]ボタンをクリックします。

6.ツールバーで[ダッシュボード]を選択し、作成したアプリケーション（Cloud FoundryまたはCF APP）をクリックします。

7. [サービスまたはAPIの追加]ボタンをクリックします。

8.左側のナビゲーション領域で「モノのインターネット」を選択して、利用可能なサービスのリストをフィルターします。 Internet of Things Platformサービスを選択します。







9.すべてのデフォルト値のままにして、「作成」ボタンをクリックします。

10.これで、「再ステージ」ボタンが付いたメッセージが表示されます。 このボタンをクリックすると、アプリケーションが更新されます。



MQTTメッセージを送信するデバイスをIoTサーバーに追加する



1. Bluemixダッシュボードで、アプリケーションと新しいサービスを選択します。

2.このサービスのページで、[ダッシュボードの起動]ボタンをクリックします。









その後、Bluemixに依存しないサービスであるIBM Internet of Thingsサーバーのパネルがすぐに表示されます。 アプリケーションには組織IDが割り当てられます。 モバイルアプリケーションを開発するために必要になります。 以下に、値1xuhbpの識別子が表示されます。







マウスを置いたときに表示される左側のメニューで、[デバイス]を選択します。 その後、[デバイスの追加]を選択し、[デバイスタイプの作成]ボタンをクリックします。 複数のタイプのデバイスを使用できます。また、多くのデバイスが存在する場合があります。 この場合のデバイスタイプは、共通の特性を持つデバイスのグループです。 彼らは同じセンサーまたは他の要素を持っているかもしれません。 現在の場合、デバイスタイプはAndroidである必要があります。 このタイプはアプリケーションで使用する必要があります。これは後で使用します。







[次へ]ボタンをクリックします。 その後、シリアル番号やモデルなど、デバイスに関するさまざまな情報を設定できるページが表示されます。 デモでは、このデータを入力しないため、[次へ]をクリックしてからもう一度クリックします。 その後、[デバイスの追加]ダイアログボックスに戻る必要があります。



もう一度、[次へ]をクリックします。ここで、デバイスの識別子を指定します。 これは、たとえば、スマートフォンのMACアドレスです。 一意でなければなりません。 この例は、タイプが「112233445566」のアドレスを示しています。



[メタデータ]フィールドを空白のままにして、[次へ]をクリックします。 [認証トークン]フィールドも空白のままにして、もう一度[次へ]をクリックします。



認証トークンの値を指定します。 後で必要になるため、覚えておくか書き留めておいた方がよいでしょう。 次へをクリックします。







提案されたとおり、[追加]をクリックすると、生成されたデバイスが表示されます。







ダイアログボックスを閉じます。 その後、MQTTメッセージをデバイスからIBM IoTサーバーに送信できます。

スマートフォンがセンサーの読み取り値を読み取りおよび送信するには、Androidアプリケーション用のIoT Starterを使用する必要があります。 ソースコードはこちらから入手できます 。

2.電話でブラウザを開き、次のアドレスを入力します。

ibm.box.com/v/iotstarterapp

3. [ダウンロード]をクリックします。

4.ファイルマネージャを使用して、ダウンロードディレクトリでファイルを探します。

これらすべてのアクションの後、IoT StarterアプリケーションがAndroidデバイスにインストールされます。



この後、別の重要な手順を実行する必要があります-Android用のアプリケーションを構成します。



1. IoT Starterアプリケーションを起動します。

2.次のパラメーターを示します。



•組織：IBM IoTサーバーに表示された組織の識別子（「MQTTメッセージをIoTサーバーに送信するデバイスを追加する」ステージの最初に示されました）。 このガイドでは、この識別子はy6spgyです。

•デバイスID：「MQTTメッセージをIoTサーバーに送信するデバイスの追加」ステップの最後に指定したデバイスの識別子。 このマニュアルでは、このパラメーターの値は112233445566です。

•認証トークン：「MQTTメッセージをIoTサーバーに送信するデバイスの追加」の手順で指定した認証トークン。







3.「センサーのアクティブ化」ボタンをクリックすると、アプリケーションはスマートフォンで加速度センサーの読み取り値の収集を開始し、収集した情報をIBM IoTサーバーに送信します。 また、アプリケーションは、加速度計と送受信されたメッセージの数に関する情報も表示します。







スマートフォンからIoTサーバーへのメッセージ送信を確認する



1.コンピューター上の組織のIBM IoTサーバーのページを開きます（このデータは、「MQTTメッセージをIoTサーバーに送信するデバイスを追加する」ステップで指定されました）。

2. [デバイス]タブには、Android OSデバイスが既に表示されています。







3.デバイスの識別子をクリックすると、スマートフォンからのメッセージを示すウィンドウが表示されます。 イベントに関する情報を取得するには、それをクリックします。







スマートフォンからのメッセージはJSON形式で送信されます。 加速度と位置に関する情報が含まれています。 Bluemix環境でメッセージデータを操作できるようになりました。



Node-REDストリームでメッセージを処理します





このセクションでは、スマートフォンからのメッセージを処理し、スマートフォンにメッセージを送り返すために、Node_REDストリームを使用してBluemix IoTアプリケーションを改善します。 電話は、アプリケーションの背景色を変更することで応答します。



Node-REDは、モノのインターネット上のメッセージの転送と処理を簡素化する視覚的なツールです。



ここでアプリケーションを実行できます 。



1. Bluemixを開き、Bluemixダッシュボードで[ダッシュボード]を選択して、Bluemix IoTアプリケーション（ステップ1で作成）が実行されているかどうかを確認します。

2.ブラウザーで<your application name> .mybluemix.netページを開きます。これは、ステップ1で作成したBluemix IoTアプリケーションの名前です。BluemixアプリケーションのNode-REDエディターで、[Node-REDフローエディターに移動]ボタンをクリックします。







3.エディターが開き、サンプルストリームが表示されます。







エディターでドラッグアンドドロップメソッドを使用して、メッセージフローの構築を開始します。 以下のコードをインポートすることで、このようなストリームを自分で作成できます。 その前に、Deleteキーを使用して既存のノードをすべて削除する必要があります。



4. このコードをコピーして、テキストエディターに貼り付けます。

5.テキストエディターで、すべてのコードが1行にあるかどうかを確認します。 そうでない場合は、すべての改行を削除します。 行をクリップボードにコピーします。

6. Node-REDエディターで、Ctrl-Iを押して[ノードのインポート]ダイアログボックスを開きます。 コードを貼り付けて[OK]をクリックします。







7.次に、特定のパラメーターにストリームを適応させる必要があります。 唯一の重要なパラメーターはデバイスIDです。 IBM IoT App outノードをダブルクリックします。 ポップアップウィンドウで、以前に使用したデバイスの識別子（112233445566など）を入力し、[OK]をクリックします。

8.エディターで、「デプロイ」をクリックします。 フローはすぐにアクティブになります。

9.スマートフォンを回転させ、傾けます。 この時点で、背景の色が変わり、Z軸の向きの変化に反応するはずです。

10. Node-REDエディターで、msg.payloadノードの隣の長方形をクリックします。 その後、デバッグするためにデバッグタブをクリックします。 ここでは、電話からJSON形式で送信されたメッセージが表示されます。







11.計算色ノードをダブルクリックして、フローを調査します。 これにより、Z軸に対する入力位置の値に基づいて色の値を計算し、計算結果をJSONデータとして送信できます。







センサーの測定値を視覚化するBluemixアプリケーションを作成します





これは、スマートフォンのIoTからメッセージを受信するBluemix環境で追加のアプリケーションを作成する最後の段階です。 データが視覚化されます。



1. Githubのiot-visualizationプロジェクトからiot-visualization-0.2.0.zipファイルをダウンロードします。 このアーカイブを解凍します。

2. Bluemixを開き、ダッシュボードを選択します。

3. [アプリの作成]ボタンをクリックします。

4.アプリケーションのタイプとしてWEBを選択します。

5. Node.jsのSDKを選択します。

6. [続行]ボタンをクリックして、アプリケーションの名前を指定します。 前と同様に、ホスト名として使用されるため、名前はBluemixで一意でなければなりません。 iotvisualizer <your_name>のようなものを入力し、[完了]をクリックします。

7.数秒後、アプリが実行されます。

8. iot-visualization-0.1.0.zipアーカイブが展開されているディレクトリで、manifest.ymlファイルを探し、テキストエディターで開きます。 ホストと名前のパラメーターを選択したアプリケーション名に変更します。 ファイルを保存します。







9.コマンドプロンプトウィンドウを開き、iot-visualization-0.1.0.zipアーカイブが展開されているディレクトリに移動します。

10.次のCloud Foundryコマンドを実行します。

cf api api.ng.bluemix.net

cf login -u <your_bluemix_login_id>

cf loginコマンドは、クラウドファウンドリコマンドラインツールを初めて使用するときにのみ必要です。 Bluemixでパスワードを入力する必要がある場合もあります。 （デフォルト設定とは対照的に）複数のBluemix組織がある場合は、そのうちの1つを選択する必要があります。



11.次のコマンドを実行して、アプリケーションをBluemixにデプロイします。



cf push <your_application_name>



このコマンドはmanifest.ymlファイルからパラメーターを取得し、コードをBluemixにロードします。 しばらくすると、次のようなものが表示されるはずです。



わかった



このコマンド「node app.js」を使用してアプリが開始されました。



12. Bluemixダッシュボードで、アプリケーションの「概要」ページを開きます。 サービス概要ページのキャプチャをクリックし、[サービスまたはAPIのバインド]ボタンを強調表示します







13.「モノのインターネット」サービスを選択します。これは、「BluemixでのIoTアプリケーションの作成」の段階で定義されました。 [再ステージ]をクリックして、アプリケーションを更新します。 これで、アプリケーションはIBM IoTサーバーからメッセージを受信できます。

14.視覚化アプリケーションを確認するには、ブラウザーページhttp：//.mybluemix.netを開きます

15. [デバイス]ドロップダウンメニューで、デバイス識別子を選択します。







ご覧のとおり、Bluemixを使用すれば、通常のスマートフォンをIoTデバイスに変えるアプリケーションを作成することはそれほど難しくありません。 この環境では、上記に示したものよりもはるかに複雑で機能的なアプリケーションを作成できます。
