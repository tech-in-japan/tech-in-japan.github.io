Bryansk Web Day 2010-レポート



6月5日に、Web開発者の最初の会議がブリャンスクで開催されました。 すべての参加者を代表して、イベントの主催者に感謝します。 私の意見では、すべてがうまくいった。 部屋は素晴らしく、パイ入りのコーヒーも素晴らしく、スピーカーは最終的に私たちのパフォーマンスに満足してくれました。



それでは、スピーチ自体について少しお話ししたいと思います。





セルゲイ・ホメンコフは彼の講演「DIY日曜大工コンテンツ配信ネットワーク」を最初に配信した

Sergeyは、実際にはCDNが何であるかを語り、サーバー間の負荷分散とジオバランシングについて話しました。

レポートの主なモットーは次のとおりです。「CDN-簡単です」

次に並んでいるのは、キリル・ゾリンの「PHPでのWebアプリケーションの設計とリファクタリング」です。

全体として、レポートは良好です。 シリルはスピーチの中でいくつかのトピックを一度に組み合わせようとしました。その結果、レポートは少し長く見えました。



リストの次は、レポート「Yandexホームページからの無料トラフィック。 このトピックは、Yandexの従業員Alexander Nikolayenkoによって強調されました。

アレクサンダーは、ウィジェットとは何か、なぜ必要なのかを語った。 彼は、地域のウィジェットプログラムと、実際にどのようにウィジェットを作成するかについて話しました。





ドミトリー・キリルキンは「インターネット上でビジネスを行うことの特徴」というプレゼンテーションを行いました

Dmitryは、サイトへのトラフィックを引き付ける方法、検索エンジンでのサイトの位置に影響する要因、広告キャンペーンについて説明しました。

元の質問が聴衆から尋ねられました：「あなたの仕事はインターネットを良くしますか？」50対50、ドミトリーは答えました。



Dmitryのスピーチの後、Andrei Kholmanyukによるレポート「Interactive Web Applicationsのベクターグラフィックス」を聞きました。

AndreyはSVG、VMLについて説明し、いくつかの例を示しました。



その後、Dmitry Chistovによるレポート「プロジェクト管理」が配信されました。

レポートは非​​常に良いことが判明し、多くの問題が検討されました。 たとえば、そのうちの1つは、勤務スケジュール（ガントチャート）の構築です。 Dmitryは、レポートで言及されているタスクを実行するために使用されるツールについても話しました。



ユーリ・パブロフは、歴史志向の科学的情報プロジェクト「BeHistory」を実演しました。 このプロジェクトの目的は、歴史科学の分野で特定の情報ベースを作成し、それと連携するためのツールを提供することです。 このプロジェクトは非常に興味深いものであり、彼らが成功することを本当に願っています。 いわば、すでに胚のバリアントを見ることができます。 プロジェクトはbehistory.ruで入手できます。



ドミトリー・パヴレンコは、フラッシュゲームでお金を稼ぐ方法を話しました。 彼のレポートは「フラッシュゲームで稼ぐ」と呼ばれています

このレポートでは、フラッシュゲーム開発者の獲得方法、スポンサーシップの主な種類などについて検討しました。



次の講演者は、Andrey Zaitsevによるレポート「Designing User Interfaces」です。

開始時の初心者向けに、HCI、ユーザビリティ、GUI、UI、UXという基本概念が与えられました。 Andreyはインターフェイス開発プロセスについて話し、設計に使用されたソフトウェアに関する良い決定と悪い決定の例を示しました。 報告書の最後に、彼はいくつかの会議に助言し、必要に応じて読むことができる文献のリストを提供しました。 私が名前を覚えている本がいくつかあります：Jeff Raskinの「The Interface」とAlan Cooperの「About the Interface」。



会議で最後に話したのはアンドレイ・ペトレンコでした。 レポート「ウェブサイト最適化手法とその応用」。



私の小さなレビューを読んでくれてありがとう。
