Windows 8は各コアの負荷を表示します

Windows 8開発者は 、Developer Previewのリリース後に行われたタスクマネージャーの改善点について話しました 。 最も興味深いのは、ヒートマップを使用した各プロセッサコアの負荷表示モードです。 8コアプロセッサを使用している場合、これはあまりおもしろいものではありませんが、サーバーまたはデータセンターの負荷を監視するときのもう1つのことです。 Microsoftが引用しているスクリーンショットは、160コアのシステムを示しています。







作業画面全体が小さな正方形（スクリーンショットの16x10フィールド）に分割され、それぞれが論理プロセッサに対応しています。 負荷の程度はパーセントと色の強度で表示されます。



開発者は、読みにくいグラフィックを描いたWindows 7インターフェイスのすべての欠点を排除しようとしました。 自分で比較してください：スクリーンショットは、160コアのWindows 7の同じシステムを示しています。







このようなグラフによると、すべてのグラフがリアルタイムで変化し、過去60秒間の履歴を表示し、ほとんど同じように見えるため、論理プロセッサを互いに比較することはほとんど不可能です。 ヒートマップを使用すると、情報をより明確に伝えることができます。



セルにカーソルを合わせると、ポップアップメニューにプロセッサIDが表示されます。







ヒートマップのもう1つの利点は、簡単にスケーリングでき、必要に応じて垂直スクロールバーが表示されることです。







このWindows 8インターフェイスは、最大640（！）プロセッサコア、つまりオペレーティングシステムがサポートする最大数の表示をサポートしています。



タスクマネージャでは、選択した論理プロセッサにプロセスを強制的に転送できます。







この機能を使用すると、1つのコア（ジョーク）にプロセスをぶら下げて、敵のコンピューターを強制的に遅くすることができます。



Windows 8開発者は、オペレーティングシステムが負荷分散を非常に有能に管理しており、システム管理者が危害を加える可能性があるため、マイクロ管理に関与しないように求めています。
