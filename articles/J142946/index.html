ご注意 Yandex.Diskアップデート

Yandex.Diskは、アップグレードを希望していることを表明しました。 わかりました、なぜですか。 更新後：

1. Yandex.Diskは既存のファイルを無視し、「同期」を開始しました。 つまり クラウドからすべてのものをダウンロードし始めました。

2.しばらくして、理由が判明し、ディスクが管理者アカウントにファイルをアップロードしました（私は通常のユーザーとして働いています）。

3.ドライブはインストールされたプログラムのリストにありません。



以降の更新。



upd1：リブートして手動で開始するC：\ Program Files \ Yandex \ YandexDisk \ Installer \ YandexDisk64Setup.exeはすべてをそのまま返します。「削除」する必要があるだけです。



upd2：「このファイルにはアクションを実行するためのプログラムが割り当てられていないため」お気に入りパネルからディスクを開くことができなくなりました。 「Yandex.Diskフォルダーを開く」トレイのコンテキストメニューも機能しません。 そして、管理者の下では、すべてが正常に動作します。



upd3：数日前に復旧手順でOSをロールバックし、以前のビルドを返し、ディスクを削除する機能を返しました。すべてが正常に機能しているようです。 同じ問題を持つhabrayuzerがコメントに登場しました。 これは、孤立したケースではありません。



upd4：Yandexは問題のある更新を思い出しました。 次のバージョンでは、すべてを修正し、ブロックレベルで同期を追加することを約束します。



upd5： 2012年5月10日付けのバージョン0.5.0bアセンブリ2446 がリリースされ、すべてのグリッチが修正され、同期（目による）が1桁速くなりました
