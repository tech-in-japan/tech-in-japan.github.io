Windows開発者の更新

8月4日の朝、Windows開発チームは3つのアップデートをリリースしました。



  • Visual Studio 2013 Update 3が利用可能になり、Windowsの開発者に多くのパフォーマンスの改善を提供します。
  • Windows Phone 8.1の更新されたエミュレーターも利用可能になり、モバイルOSイメージの最新バージョンでアプリケーションをテストできるようになりました（ 約transl .:実際には、 最新の電話イメージに関するものでしたが、私が理解しているように、それはOSイメージです ）;
  • そして最後に、Windows Phone 8.1 GDR1を更新します。 このアイテムは、開発者向けWindows Phoneプレビュープログラムをインストールしているユーザーが利用でき、最新の更新プログラムを搭載した実際のデバイスでアプリケーションを実行できます。




この投稿では、これらの更新プログラムについて簡単に説明し、これらの新しいツールが開発者ツールに追加するもののコンテキストに浸りたいと思います。







Visual Studio 2013アップデート3





8月4日から、Visual Studioの更新プログラムには、マイクロソフト製品に夢中になっている開発者向けの多くの便利な機能が含まれています（詳細はSomaの投稿を参照してください）。 Windowsアプリケーションを作成する際のエクスペリエンスを強化する、利用可能な3つの新機能についてお話ししたいと思います。



  • これで、プッシュ通知構成ウィザードを使用して、Windowsアプリケーション（クライアントまたは電話）を.NET Azureモバイルサービスに接続できます。
  • Windowsアプリケーションでソケットを使用できるようになり、WinRTアプリケーションに新しい通信オプションが追加されました。
  • DirectXデバッグツールが改善され、DX開発者の生産性が向上しました。 いくつかのバグ修正を含め、コマンドラインツールをDXリモートMSIパッケージに追加します（ およそTransl。：それが何なのかわかりませんので、元のままにします）。これにより、完全なWindows SDKまたはVisual Studioをインストールすることなく、安定したシステムでグラフィカルトレースをキャプチャできます。




この更新を受け取るには、 Visual Studioダウンロードページにアクセスしてください 。



Windows Phone 8.1の更新





また、Windows Phone 8.1 GDR1更新プログラムは、エミュレーターと物理デバイスの両方を通じて、開発者が使用できるようになりました。 この更新プログラムは、一般的な使用が可能になる前に更新プログラムを提供するWindows Phoneの伝統を継承し、ユーザーがこの機会を得る前に更新プログラムを確認する機会をユーザーに提供します。



JoeBが先週示したように、エンドユーザー向けのパンが大量にあります（詳細については、 Joeの更新に関する投稿を参照してください）。 ただし、あなたは開発者であり、おそらくこのアップデートに含まれているものに個人的に興味があります。 一般に、APIへの変更は含まれていませんが、製品に対するエンドユーザーの全体的な満足度を間違いなく高めるいくつかのOSの改善があります。 Update 1には、言及する価値があると思われる3つの改善点があります。



  • VPNサポートが改善されました（ 約：原文では、この項目は参考ではなく、 他のソースから情報を取得しました ）。 L2TPレイヤ2トンネリングプロトコルのサポートを得ました。 また、公共の場でWi-Fiを介してVPNを使用できるようになりました。
  • 車やヘッドセットとの統合など、Bluetooth接続のいくつかの拡張機能、および新しいBluetoothパーソナルエリアネットワーク（PAN）プロファイルが追加されました。 また、開発チームがデバイスメーカー向けにいくつかのAPIを追加して、自社デバイス用の独自アプリケーションの開発を促進し、通知を通じて電話でヘッドセットに通知することも注目に値します。
  • スタート画面で、ストアアプリボタン（アプリケーションストアの起動）がLive Tileを完全にサポートするようになりました。これにより、開発者はより頻繁な訪問とインストールを受け取ることができます。
