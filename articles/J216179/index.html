オフライン販売におけるコンテキスト広告の有効性を分析するためのデータ収集方法

コンテンツ広告の有効性を評価するには、製品購入のさまざまな段階で顧客の行動に関する調査を実施する必要があります。 このような調査の目的の1つは、広告が表示されてから購入が行われるまで、クライアントの経路を追跡することです。 オンラインで購入し、ウェブ解析を使用すると、この問題は非常に簡単に解決されます。 ただし、オフラインで購入すると、いくつかの問題が発生します。

ユーザーは広告をクリックして、ランディングページにアクセスし、電話をかけてオフィスに来ます。 そして今、新しいクライアントがあなたのオフィスに立っています、そしてあなたは彼がどこから来たのか、彼の訪問費用が何ルーブルなのか、彼がどの広告をクリックしたのか、などを知りません。 最も重要なタスクは、トラフィックソース（広告キャンペーン、広告グループ、検索エンジンなど）によるヒットを記録することです



顧客パス

問題のさらなる検討を簡素化するために、製品の例としてローンを取り上げます。 また、すべてのWeb分析をすでに構成しており、優れたCRMを備えているとします 。

ローン販売ファネルを考えてみましょう：





ウェブ解析を使用して、広告のクリック数とサイトでの顧客の行動のみを追跡することができます。 次に、Web分析が終了し、「オンラインからオフライン」への全体的な変換（何人が銀行にアクセスしてサイト訪問者の数になったのか）だけが確実にわかります。 しかし、オフラインの連絡先をトラフィックソースに分割する方法は？ 問題は、CRMシステムで考慮され、Webデータに関連付けられていないオフラインの連絡先の最初の段階であるデータの収集で発生します。





問題の解決策

問題を解決するには、CRMでトラフィックソースを指定する必要があります。 クライアントとのコンタクトを開始するときにトラフィックのソースを特定する方法を理解することだけが残っています。



1.電話なし-問題ありません
電話をサイトから削除し、すべての顧客にWebフォームとの対話を強制します。 この場合、営業担当者はウェブフォームを受け取り、顧客との連絡を確立します。 Webフォームに入力すると、分析に必要なすべてのデータがCRMに分類され、イベントのさらなる経過を追跡できるようになります。

例： www.tcsbank.ru/credit/form

Yandexの広告ブロックの広告をクリックしてページに移動し、次のURLタグについて確認します。

URLタグ
utm_campaign = search_msk_creditcard＆block = premium＆position = 2＆

utm_medium = ctx.cpc＆utm_source = yandex_cc＆

utm_content = kreditnaya_karta＆wm = search_msk_creditcard＆sid = kreditnaya_karta.174957076
Webフォームを送信すると、URLタグが分析されてCRMに送信されます。その結果、銀行は広告キャンペーン、広告掲載順位、キーワードなどの各アプリケーションに関する信頼できる情報を受け取ります。

3.プロモーションコード
製品の詳細が、クライアントが予備的なアクション（通話またはアプリケーション）なしでオフィスに到着する可能性が高いと想定している場合、および番号の置換を導入せず、電話番号をサイトに残したい場合に、この方法はうまく現れます。

クライアントは、必要なサービスのホットディスカウントを取得するよう招待されています（必要なものを選択してください）。

  • Webフォームに記入し、割引コードが記載されたSMSを受け取ります。このコードは従業員に通知する必要があります
  • 電話で「あなたのコード」で会社の従業員に口述する（例を参照）
  • クーポン（同じコードが埋め込まれている）を印刷または写真撮影する
  • その他

 プロモーションコード
割引が存在するという事実は、顧客に購入を促す可能性があり、Webデータからのクライアントに関する多くの情報をプロモーションコードに「縫い付ける」ことができます。 割引を取得するための手順はあまりにも明白であり、クライアントを怖がらせるかもしれません、クライアントは割引を無視するかもしれません
