オープンIDE Adob​​e Bracketsの概要



環境の最初のレビュー （このレビューは1年以上前に公開されました）以来、IDE Bracketsは有望なプロトタイプからWebアプリケーション開発用の本格的なツールに進化しました。 最近、このメディアがパスで言及された投稿も公開されましたが、テキストとコメントはプロジェクトの説明に非常にけちだったので、このIDEの可能性についてコミュニティにもっと詳しく伝えることが有用であることがわかりました。



IDEブラケットとは何かに関する少しの情報。 この環境は、AdobeのChromium Embedded Framework（CEF）を使用するWebプログラマ向けのWebプラットフォーム（HTML、CSS、およびJavaScript）で開発されています。 ブラケットは、CodeMirror、jQuery、require.js、Bootstrap、Node.jsなどのプロジェクトに基づいています。 ソースコードは 、オープンソースのMITライセンスの下で配布されます。 プロジェクトの主な目標は、Web開発プロセスを簡素化することです。 IDEでの作業は、スクラム方法論に基づいています（2.5週間のスプリント）。 環境は非常に安定しています-IDE Brackets開発者自身がかなりの期間、日常業務で使用し始めました。 ここから環境をダウンロードできます。



ハイライト：環境がLinuxで利用可能になり、Node.jsのサポートが登場し、HTML、CSS、JavaScriptのインテリジェントなコード補完が実装され、コミュニティは100を超える便利なプラグインを作成し、さらにレスポンシブデザインでレイアウトの作成を簡素化する高度なシステムが開発されました。それを見た人にすごい効果をもたらします。

インラインコード編集

Node.jsに基づくライブプレビュー



デフォルトでは、ローカルのNode.jsサーバーのおかげでライブプレビューが機能します。 以前のバージョンでは、この関数はfile：//を介して機能していました。これにより、Typekitなどの外部サービスの統合で問題が発生する可能性がありました。



「ライブプレビュー」の特性は、CSSファイルの編集時、HTMLおよびJavaScriptの編集時、変更後すぐに結果が表示されることです-保存後のみ（HTMLファイルの「ライブプレビュー」は次のバージョンから、JavaScriptライブプレビューを実装するプラグイン ）。 また、ブラケットでHTMLを編集する場合、対応する編集可能な要素がブラウザーで強調表示されます。











ローカリゼーション



開発環境は、ロシア語、日本語、ドイツ語、スペイン語、ノルウェー語、チェコ語、ポーランド語を含む17言語で利用可能です。 ロシア語のサポートは最高レベルではありませんが、作業は可能です。 最大の誤解は、Adobe Sourceフォントファミリにキリル文字が含まれていないことに関連しているため、現在のシステムフォントが使用されているため、プログラムの全体的な認識が若干損なわれます。 もちろん、これはすべて、フォントSourceCodeProをキリルアルファベットをサポートするフォントに変更することで処理されますが、このためには少し手で作業する必要があります。



ソースコードのフォントを変更する
私の解決策は優雅に輝いていないので、誰かがより良い方法を思いついたら、共有してください。

アイデアは非常に馬鹿げています-フォントファイルをキリル文字のあるものに置き換えるだけです。 Mac OS XとLinuxでは、フォントDejaVuSansMono.ttf（ こちらからダウンロードできます ）、Windows 7ではconsolas.ttf（オペレーティングシステムに付属）を選択しました。 次に、Adobeの既存のフォントを置き換えるフォントの名前を変更した後、このフォントをBracketsリソースディレクトリ内のSourceCodePro-Medium.ttfとSourceCodePro-Regular.ttfの2つのファイルに単に置き換えました。



異なるオペレーティングシステムでフォントファイルが配置されているパス：

  • Linux ：/ opt /括弧/ www /スタイル/フォント/ SourceCodePro /
  • Mac OS X ：/アプリケーション/ブラケット\スプリント\ 29.app/Contents/www/styles/fonts/SourceCodePro/
  • Windows ：C：\ Program Files（x86）\ Brackets Sprint 28 \ www \ styles \ fonts \ SourceCodePro

Linuxサポート



現時点では、LinuxバージョンのBracketsはアルファ版の段階にさえありません。開発者自身が言うように、これは一種の「超早期プレビュー」です。なぜなら、すべての基本機能が実装/動作するはずがないからです。 それでも、現在のLinuxバージョンを使用して作業できます（いくつかの制限があります）。







Linuxの制限と起動問題の解決策
Linuxの現在のバージョンのバージョンには多くの問題があり、最も重要なものは次のとおりです。

  • プログラムの長期起動。
  • バックライトはライブプレビューモードでは機能しません。
  • 拡張機能は〜/ .Brackets / extensions / userにのみ手動でインストールできます。
  • 環境内で作成されたファイルを保存することはできません（回避策は、環境外でファイルを作成し、マウスを使用して環境に転送することです）。




また、一部のシステムでは、libudev.so.0への誤ったリンクが原因でプログラムの起動に問題が発生する場合があります。 この場合、libudev.so.1からlibudev.so.0へのシンボリックリンクを作成するだけです。 問題のより詳細な解決策は、 ここで見つけることができます 。





まとめ



はい、機能とプラグインの数の点で、IDEブラケットは他の成熟した環境やエディターより劣っていますが、それでもいくつかのユニークな機能を提供するWeb開発者向けの完全なツールになります。 もちろん、欠点は、IDEをセットアップするための明確なメカニズムがなく、アプリケーションの最速のダウンロード速度や特定の機能（特にJavaScriptのインライン編集）が動作しないことです。 いずれにせよ、Webプログラミングに精通している開発者なら誰でもBrackets開発に接続し、驚くべきアイデアを実現できるという事実は、この環境を特別なものにします。
