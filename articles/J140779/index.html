API 2GISの更新：静的マップ、再設計されたマーカーとバルーン、ドキュメントと速度の改善

すでに2GIS APIについて聞いたことがあるかもしれませんが、たとえそうでなくても問題ではありません。 実際、この投稿では、過去2か月間のすべてのAPIの更新を検討します。







Static Maps API


最近開始されたStatic Maps APIから始めましょう。 JavaScriptを使用せずに、2GISマップから静的画像をWebページに配置できます。 これは、httpリクエストを介して行われ、それに応答して、マップフラグメントとリクエストで指定された他の要素を含む画像が送信されます。



私たちがモバイルWebサイトとみなしているStatic Maps APIのアプリケーションのメインエリア。 たとえば、 Flampレビューサービスのモバイルバージョンは、すでにStatic Maps APIを使用しています。 そして今、あなたはあなたのサイトに地図付きの画像を追加することができます！ このように：







新機能と改善された地図の使いやすさ


APIの最もポピュラーな部分であるマーカーとバランを大幅に再設計しました。 話している内容がわからない場合は、説明します。マーカーは、マップ上の特定のオブジェクトを示すのと同じもの（通常はオレンジ色）です。 バランとは、マーカー、テキスト付きのバブル（コミックなど）をクリックしたときに表示されるものです。



そして、これが変更されたものです。 バルーンが表示されました。サイズ変更の方法、ドラッグアンドドロップの機能、強制再描画の方法です。 マーカーの場合：移動方法、アイコンを動的に変更する機能、クリックハンドラーのスコープを設定する機能。 新しいDG.Markers.MarkerWithBalloonクラスは、バルーンバンドルにマーカーを実装します。



パフォーマンスと復元力も向上しました


参照APIの開発で最も重要なトリックの1つは、高速で安定した操作です。 2月と3月に、これらの重要な技術指標を大幅に改善しました。 そのため、稼働時間が99.9を超えました。 そして、検索方法はさらに速く動作し始めました-現在、最も「重い」クエリでさえ平均200ミリ秒以内に実行されます。 会社カードのリクエスト-50ミリ秒。



毎日新しい組織


このトピックに関する出版物は既にいくつかありましたが、ここで繰り返すことができます-2GIS APIは毎日の更新に切り替えています。 現時点では、これはノボシビルスク 、 モスクワ、エカテリンブルクに適用されますが、徐々に他の都市にも広がります。



2GIS OnlineがAPIのパブリックバージョンに切り替わりました


2GIS OnlineがパブリックAPIよりも先に進み、パートナーが利用できない機能を含めることができた場合、それらのステータスは同等になりました。 これ以降、2GIS Onlineはパブリック2GIS APIで実行され、実際にはマップAPIとトランスポートAPIの標準パートナーです（以前はディレクトリAPIに切り替わりました）。 したがって、2GIS Onlineに表示されるほとんどすべてのものは、アフィリエイトプログラムの一部としてリソース上で再作成できます。 そして、あなたがこれを行う方法がわからず、 ドキュメントが役に立たない場合は、恥ずかしがらないでください-api@2gis.ruで私たちに書いてください、彼らは確かにあなたを助けます。 ちなみに、トランスポートAPIの暫定的なアプリケーションを引き続き収集します。



私たちのサイトでさえ、アップデートを逃れませんでした。


特に、 「ソリューション」ページに新しいサンプルが追加され、 「機能と利点」ページがより美しくなり、最初のサンプルがAPIマップのドキュメントに登場しました（まもなくそれらが追加されます）。 また、参照API については、参照セクションの設計に関する推奨事項 。 そうそう-何かを見つけられなかった場合、喜んで追加します。 あなたの意見で、特定の何かの実装の十分な例がない場合、 私たちに書いてください 。 また、共有する準備ができている独自の例をお持ちの場合は、サイトに追加させていただきます。



2GIS APIの詳細については、api.2gis.ruをご覧ください。
