Insider API：ソーシャルテキストの傾向とクイックナビゲーションを検索します。 従来のメディア

どのくらいの頻度で、（あらゆる主題の）テキストの配列を持ち、テーマの絞り込みを確認したいですか？ どのトピックが議論されていますか？ 最も人気のあるものはどれですか？ 検索語を入力して、それらの周りのトレンドを取得するのは素晴らしいことです！







システムは次のことができます。

  • データの配列を取得して、ユーザーの下に保存します。
  • トピックごとにドキュメントをグループ化して、トピックのリストを作成します。
  • ドキュメントをリアルタイムで検索し、見つかった選択項目についてトピックを作成します。




これらの機能はすべて、APIを介して利用できます。APIは、料金プランの1つに接続することで簡単に使用できます（1か月間の無料トライアルがあります！）。



ソーシャルインデックスを作成しました。 2015年の初秋にメディア（twitter、facebook、vkontakte）で視覚評価のためにシステムのスクリーンショットを撮りました。



消費者セグメント：マグネット。





依田：





政治：プーチン大統領の国連への演説。





これは、テーマが内部からどのように見えるかです：







前のスクリーンショットは、7つのニュースを含むテーマ「UN 2015」の内容を示しています。



スクリーンショットからわかるように、このシステムは本格的な検索システムであり（ロシアの形態をサポートしています）、情報配列をすばやくナビゲートするための便利なツールです。 サポートされているトピックとデータタイプ（ソーシャル）に制限はありません。 メディア、ニュースフィード、または企業ドキュメント。 説明されているすべての機能は、ユーザーインターフェイスを備えたSAAS製品の形式でも、既存のシステムやモバイルアプリケーションに統合できるInsider APIの一部としても利用できます。



APIアプリケーションの原則：



1.データ（投稿、記事）をシステムにアップロードします。 

End-point: /articles/uploadJson  : POST [ { "id": 2134657, "title": "", "description": "           .  .  -  .", "link": "https://twitter.com/palnom6/status/704979632127418369" } ]




1回のリクエストで最大50個のテキストを送信できます。 応答として、すべてがうまくいった場合、APIはコード200で応答します。



idパラメーターに注意してください。このパラメーターの値により、次のステップで特定のトピック/トレンドへのこの投稿のバインディングを取得します。



2.ドキュメント全体から、またはキーワードと組み合わせて、傾向を把握します。 



 End-point: /articles/cluster  : POST { "query": "" }




それに応じて、APIはトレンドを生成します。 



 [ { "labels": [ "" ], "score": 1.510325122396045, "docs": [ "2134657", "2134656", "2134655", "2134654" ] }, { "labels": [ "" ], "score": 1.2447816860782057, "docs": [ "2134653", "2134652", "2134651", "2134650", "2134649", "2134648" ] } ]




スコア値は、ユーザーの要求に対するこのトピックの関連性を反映しています。

ステップ2では、トピック/トレンドが構築される特定のキーワードを渡すこともできます。



Insider APIはこちらから入手できます： market.mashape.com/dmitrykey/insiderapi



トレンドに加えて、他の便利な機能は次のとおりです。



  • 客観性 ： market.mashape.com/dmitrykey/russiansentimentanalyzer
  • 住所の抽出： market.mashape.com/dmitrykey/streetdetector
  • テキストが中国語の場合のキー： market.mashape.com/dmitrykey/fuxiapi
