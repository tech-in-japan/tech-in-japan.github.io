XWTの紹介

このリソースのすべての読者への挨拶！ 現時点では、クロスプラットフォームのGUIアプリケーションであるMicrosoft .NET / Monoを確保するために、多くのソリューションが開発されています。 この記事では、そのうちの1つであるMono開発者の公式プロジェクトであるXWT Widget Toolkit（Xwt）を紹介します。 現時点では、リポジトリgithub.com/mono/xwtからソースの形式でのみダウンロードできます。 残念ながら、インターネット上でこのツールキットに関する有能で完全な記事を見つけることができなかったので、自分ですべてを嗅ぎ取らなければなりませんでした。 なぜなら この記事は広告でもレビューでもありません。XWTが同じEto FormsまたはSystem.Windows.Forms（後者はMonoで非常にうまく機能する）よりも優れているか悪いかを説明しません。 この記事では、XWTがどのように機能するかと、XWTを使用する基本について説明します。





注意！ このツールキットは非常に未加工であり、多くのバグが含まれています。 アプリケーションで使用して、機能しないプロパティ、トリガーされないイベントなどを検出する準備をします。 不快な現象




はじめに



XWT Widget Toolkit（多くの人が考えるXamarin Widget Toolkitではなく名前が復号化されているようです）は、Xwt.Gtk.dllライブラリから標準Xwtインターフェイスを使用して構築されたバックエンドをロードするXwt.dllという抽象的なローダーがあるスキームに従って配置されます、Xwt.Wpf.dll、Xwt.Mac.dllなど XWTデバイスについて詳しくは、2013年11月11日のHabraトピックをご覧ください。 API構造の点では、XwtはGTK＃と非常に似ていますが、これは驚くことではありません。Xwtは、MonoDevelopをLinux以外のプラットフォーム（主にMacOS X）に移植するために作成されました。



初期化



Xwtは、ホスト（通常のアプリケーションの場合）として、またはXwtウィジェットがネイティブアプリケーションに埋め込まれている場合はゲストとして初期化できます。 両方の方法は一般的に類似しているため、同時に検討されます。

まず、XWT自体とバックエンドが初期化されます： 

Xwt.Application.Initialize(Xwt.ToolkitType.Wpf); // Xwt.Application.InitializeAsGuest(Xwt.ToolkitType.Wpf);


引数を使用して、さまざまなバックエンドをロードできます。 バックエンドをロードするには、Xwt。*。Dllファイルをアプリケーションのディレクトリに配置する必要があります（Cocoaバックエンドを使用する場合は、MonoMac.dllファイルも配置する必要があります）。 また、バックエンドdllの.net署名を含む文字列のみを引数として設定することもできますが、これは推奨されません。 「DLL地獄」が発生する可能性があります。 注意深い読者は、バックエンドが手動で設定されていることにすぐに気付くでしょう。 XWT APIには引数なしでXwt.Application.Initialize（）の呼び出しがありますが、期待どおりに機能しません。 したがって、OSの定義とToolkitTypeの選択を自分で自転車を作成する必要があります。 MacOSXは単に定義されていないことに注意してください。



初期化後、ウィンドウ（Xwt.Window）を作成して画面に表示する必要があります 

 //MainWindow -  Xwt.Window,     new MainWindow().Show();


そのため、XWTをダウンロードし、ウィンドウを表示して点滅し、プロセスが完了しました。 そして、すべてがUIイベントストリームの開始を忘れたためです。 これは1行で行われます。 

 Xwt.Application.Run();


この時点で、Main（）関数のコードがハングするため、他のすべてはUIスレッド、つまり上記の行で作成されたフォームで実行する必要があります。



フォーム、ダイアログ、ウィジェット



このツールキットのユーザーコントロールはウィジェットと呼ばれます。 ウィジェットは他のウィジェット（唯一の方法）に埋め込まれ、ウィジェットはウィンドウに埋め込まれます。 Windowsは、Xwt.WindowFrameを継承するクラスです。 確かに、原則として、それらはインスタンスを表すか、WindowまたはDialogクラスから継承します。 フォームからのダイアログは、モーダルモードでの出力（ Run(WindowFrame parent)



function Run(WindowFrame parent)



）、およびワークスペースを組み込みウィジェット（コンテンツ）とコントロールボタン（ボタン）の部門に分割する可能性によって区別されます。 フォームには、タイトル、ネストされたウィジェットの部門（コンテンツ）、およびメニューバー（MainMenu）のみがあります。 最後はダイアログにありません。

フォームとダイアログの両方には、一般的に他のGUIツールキットと同様に、おそらく少し短縮されたイベントとプロパティの基本セットがあります。 Xamarin LLCの開発者をScりながら、Xwtインターフェースの各ポイントの後ろに、一度に4つのバックエンドの長い改良があることを忘れないでください。



ウィジェットは、Xwt.Widgetを継承するクラスです。 ウィジェット-コンテナとウィジェット-コントロールがあります。 Windows自体には、この制限を克服するために最大1つのウィジェットを含めることができ、コンテナウィジェットが作成されました。 コンテナは、その名前が示すとおり、無制限の数の他のウィジェット（コンテナウィジェットを含む）を含むことができ、ウィンドウや別のコンテナなどの独自のホスト（親）での配置を担当します。 単純なウィジェットは説明が必要ではないと思いますが、それらのドキュメントは明らかに弱いです。 既知のプロパティとイベントに異なる名前を付ける準備をします。 たとえば、マウスクリック、を含む。 ダブル、ButtonPressedイベントを発生させます。

コンテナウィジェット



このツールキットには、GTKのspionerennyのような多くのコンテナウィジェットが含まれています。 GTK＃に不慣れな方のために、これらのコンテナタンクの概要を説明します。



キャンバス-キャンバス





最も簡単なコンテナ。 自分でウィジェットをランダムにレイアウトすることができ（AddChild（Xwt.Widget w））、あらゆる種類の描画をサポートします（OnDraw（Xwt.Drawing.Context ctx、Xwt.Rectangle dirtyRect））。 ただし、「毎日」の使用は推奨されません。 Xwtのレンダリングツールは完璧とはほど遠いものであり、要素のハードポジショニングにより、内部コンテンツ（およびサイズ）を変更するときにウィジェットが完全に混乱する可能性があります。



ボックス-本棚（ラック）





Vbox



HBox

HBoxおよびVBoxウィジェットは、他のウィジェットのサイズと位置を自動的に制御して、ウィジェットを1行に配置するように設計されています。 PackStartおよびPackEnd関数を呼び出してウィジェットを追加します。 引数を操作することにより、ウィジェットの配置モードとストレッチモードを指定できます。 margin *引数は、理解できない値で指定され、96dpiの標準ディスプレイ解像度は厳密に1pxに等しくなりますが、DPIの増加（スケールあり）で適切に機能し続けます。



テーブル-テーブル





Webページの古典的な構築に精通している人にとっては、これはおそらく既存のものの中で最も便利なコンテナでしょう。 はい、これは最も一般的なテーブルです。 各セルにウィジェットを割り当てることができます。 大規模な固定マークアップアプリケーションの場合、これはおそらくこれまでで最高のコンテナーです。 ただし、「ゴム」のマークアップは許可されません。 セル内のウィジェットのサイズを変更すると、列/行の寸法も自動的に変更されますが、ウィジェットをつかんで境界線をドラッグすることはできません（現時点では）。



ペイン-パネル





2つのパネルからインターフェイスを作成し、それらの比率を変更できるようにするために、VPanedコンテナとHPanedコンテナがあります。 現時点では、Xwtでは、プログラムで比率を変更することはできませんが、これには2つの非アクティブなプロパティもありますが、それらがどのように異なるかはわかりませんでした-PositionとPositionFraction。



ノートブック-タブ





「ノートブック」は、以前の設定ウィンドウのように、タブ付きインターフェイスを作成することを目的としています。 VBoxなどのコンテナを子のタブとして指定する必要があることに注意してください。そうしないと、ウィジェットを1つしか配置できません。



エキスパンダー-ロールアップパネル





このウィジェットは、さまざまな二次的な追加情報またはアプリケーションコントロールを表示するように設計されています。 また、ノートブックと同様に、コンテナを条件付きで処理します。 コンテンツ（コンテンツ）として1つのウィジェットのみをサポートします。
