Windows 8の起動が速すぎる

「Windows 8には問題があります-読み込みが速すぎます。 コンピューターの電源を入れた後、F2キーまたはF8キーを押す時間がありません」 と 、Microsoft User ExperienceのチームマネージャーであるChris Clark 氏は述べています。



実際、Microsoftは多くの最適化作業を行っており、Windows 8はSSDを搭載したコンピューターで7秒以内に起動し、BIOSの初期化とPOSTのセルフテストは最初の2〜3秒で完了します。 Windows 8のブートパスの選択は、200ミリ秒以内に実行されます。ここで何かを押す時間はありません。



インターフェイスの専門家は、異常な問題を解決する必要がありました。 「高速ブートを維持したい」とクリスは続けます。 「もちろん、使い慣れた機能が過去のように機能するように、だれも意図的にそれを遅くすることはありません。」 ただし、同時に、何らかの方法でユーザーにダウンロードオプション（セーフモードなど）を選択する機会を与える必要があります。



Microsoftは、古いシステムの起動プロセス中にユーザーがF8キーを押した方法について内部テストを実施しました。 メニューの外観を推測しようとしてF8を連続して押すか、対応するオプションが表示されるのを心配して待ち、できるだけ早くボタンを押しました。 テストによると、ユーザーに対する最速の応答は約250ミリ秒です。 したがって、新しい200ミリ秒の時間枠に投資する時間はありません。



別の問題があります：BIOSブート中に、パーツに「F2を押して設定を入力する」または「F12を押してネットワーク経由でブートする」などの追加メッセージが表示されるため、BIOS開発者はこれらのメッセージを表示する時間がほとんどありません。キーボードはセルフテストに合格し、POST手順はほぼ終了します。



したがって、UEFIがサポートされている新世代のBIOSでコンピューターを起動するときに「ホットキー」を押すことに別れを告げることができます。 代わりに、Windows 8の読み込みで問題が検出された場合に自動的に表示されるメニューを作成しました。











UEFIがサポートされていない古い「マザーボード」では、Windows 8のロードがそれほど速くないため、ホットキーが残ります。



ブートメニューを強制するための多くの方法も開発されています。 メニュー表示付きの「高度な再起動」は、Windows 8（高度なスタートアップ）の設定から開始できます。







または、Shiftキーを押しながら[再起動]を押します。







最後に、新しい/o



スイッチでリブート（ /r



）することにより、コマンドライン（ shutdown.exe



）からメニューを呼び出すことができます。
