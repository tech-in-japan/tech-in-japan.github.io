すべての費用でテキスト：WCBFFおよびDOC

、またはメモ帳の"Ў±"

タイトルの2番目の略語を解読する価値があります。 WCBFFはWindows Compound Binary File Formatに他なりません。ロシア語では「 Windows Compound binary format files 」のように聞こえます。 翻訳を企業の良心に任せて、この形式がひどい名前でどのように役立つか考えてみましょう。



そのため、CFBは祖先、またはより正確には、97番目のバージョンから2007年までのすべてのMicrosoft Office形式のスケルトンです（互換性形式で保存する場合）。 このCFBは、Wordのテキストを保存するためだけでなく、ExcelシートまたはPowerPointプレゼンテーションを保存するためにも使用されます。 その結果、CFBで「暗号化」されているバックボーンを読み取り、DOC形式を考慮して読み取りデータのテキストを見つける必要があります。



CFBまたは小さなファイルシステム



私が言ったように、最初のステップはCFBを読むことです。 CFBは、セクター、ルートディレクトリ、およびある種のファイルを含む、ミニチュアのファイル構造を示します。 このファイルの問題でさえ、通常のファイルシステムの場合と同じです-たとえば、断片化されたセクター。 幸いなことに、MicrosoftはCFBと他のすべての「上部構造」形式の両方のドキュメントを数年にわたって公開しています。



情報がCFBファイルにどのようにパックされるかを理解してみましょう。 ファイル全体はセクターに分割されます-各512バイト（新しい、4番目のバージョンでは、セクターサイズは4096バイトになります）。 最初のセクターはファイルヘッダーです。これは上記のスクリーンショットで見たものです。 これ（ヘッダー）には、ファイルから読み取る方法、内容、および順序に関するすべての情報が含まれています。



ファイル内のデータは、同じ512バイトのセグメント（FAT）に格納されます。 セクターセグメントに十分なスペースがない場合、残りのデータはチェーン内の次のデータに転送されます。 チェーンセクターは、ファイル全体に散在している可能性があります（つまり、上記のようにファイルを断片化できます）。 セクターチェーンの整合性を維持するために、すべてのデータが読み取られていない場合に現在のセクターから切り替えるセクターを含む特別なセクターがあります。 チェーンの終わりは、特別な単語ENDOFCHAIN = 0xFFFFFFFE



によって特徴付けられます。



一部のデータでは512バイトが非常に大きくなる可能性があるため、ミニFATと呼ばれる「ミニチュア」セクターがあります。 ミニFATセクターの長さは64バイトであるため、このような小さなセグメントは8個（または64個）が1つのFATセクターに収まります。 FATまたはミニFATの選択は、現在のデータの全長に基づいています。 4096バイト（ファイルヘッダーのパラメーターの1つ）未満の場合、mini FATを使用する価値があります。それ以外の場合はFATです。



CFBファイル内のデータは何も積み上げられていません-ルートエントリの特別な「ファイルエントリ」にルートを持つツリー構造に構造化されています。 各エントリの長さは128バイト（4または32エントリが1つのFATセグメントに収まる）で、その名前、タイプ（ストレージ-ストレージ、ストリーム-ストリーム、ルートストレージ-ルートストレージ、空のスペース-未使用）、子および「兄弟」によって特徴付けられます要素、赤と黒の木の色。 さらに、ストリームおよびルート要素の場合、コンテンツのオフセットや長さなどのパラメーターが発生します。



したがって、FSへの各エントリは、「それに添付されたコンテンツ」によって特徴付けられます。 ストリームの場合、これはルート要素（ミニFATファイル）に保存されたデータになります。



さらに、ファイルにはDIFATと呼ばれる構造があり、FATシーケンスのチェーンを持つセクターへのリンクを保存します。 最初の109個のDIFATリンクはファイルヘッダーの最後にあり、最大8.5 MBのファイルを「提供」できます。これが十分でない場合、ヘッダーには追加のDIFATセクターへのリンクを含めることができます。



この情報は、CFBファイルで発生しているすべての混乱と揺れを簡単に特徴づけます。 形式は、原則としてかなりよく文書化されています（リンクはトピックの最後にいつもどおりです）。マニュアルを思慮深く綿密に読むだけで十分です。 この記事の目的は、CFBファイルの操作に関する完全な説明ではありませんでしたので、主なことに移りましょう-このすべてからドキュメントを読む方法...
