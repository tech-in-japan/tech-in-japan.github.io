役立つWordPressの10のコメンタリートリック





多くのブログでは、コメントセクションを公然と無視しています。 しかし、コメントは著者と読者の相互作用に過ぎないため、これは根本的に間違っています。 この記事では、ブログのコメントセクションが実際にふさわしいレベルに到達するのに役立つ10の優れたヒントと秘密を共有します。



1.コメントのアクションリンクを作成します





問題。 公開前にコメントの承認が必須であるにもかかわらず、一部のエントリは常に編集、削除、またはスパムとしてマークする必要があります。 デフォルトでは、WordPressはコメントの「編集」リンクを提供し（「edit_comment_link（）」関数を使用）、「削除」または「スパム」を提供しません。 追加することをお勧めします。



解決策。 最初に、関数を作成する必要があります。 以下のコードを「functions.php」ファイルに貼り付けます。



function delete_comment_link($id) {

if (current_user_can('edit_post')) {

echo '| <a href="'.admin_url("comment.php?action=cdc&c=$id").'">del</a> ';

echo '| <a href="'.admin_url("comment.php?action=cdc&dt=spam&c=$id").'">spam</a>';

}

}







「functions.php」ファイルを保存したら、「comments.php」を開き、「Delete」および「Spam」リンクが表示される場所に指定されたコードを追加します。 両方ともコメントループに入る必要があります。 ほとんどのトピックでは、関数「edit_comment_link（）」が提供されています。 この場合、コードはその直後に追加されます。



delete_comment_link(get_comment_ID());







コード分​​析。 まず第一に、もちろん、ユーザーがコメントを編集する権限を持っていると確信しています。 確認後、コメントを削除したり、スパムとしてマークしたりできるリンクが表示されます。 ブログ管理者のURLを取得できる「admin_url（）」関数に注意してください。



2.コメントからトラックバックを分離する





問題。 録音にトラックバックが多すぎますか？ 個人的には、はい。 一方で、読者は他のブログのどの記事があなたの記事にリンクしているかを見ることができるので素晴らしいです。 一方、その数が多すぎると、議論を追うことが難しくなります。 もちろん、この状況では、特にWordPress 2.7で提供される「返信」機能が使用されていない場合、コメントからトラックバックを分離することは理にかなっています。



解決策。 件名を検索して、ファイル「comments.php」を編集します。 次のようなコメントループを見つけます。



foreach ($comments as $comment) : ?>

// Comments are displayed here

endforeach;







次に、以下のコードに置き換えます。



<ul class="commentlist">

<?php //Displays comments only

foreach ($comments as $comment) : ?>



<?php $comment_type = get_comment_type(); ?>

<?php if($comment_type == 'comment') { ?>

<li>//Comment code goes here</li>

<?php }

endforeach;

</ul>

<ul>

<?php //Displays trackbacks only



foreach ($comments as $comment) : ?>

<?php $comment_type = get_comment_type(); ?>

<?php if($comment_type != 'comment') { ?>

<li><?php comment_author_link() ?></li>

<?php }

endforeach;

</ul>







コード分​​析。 原則として、複雑なことはまったくありません。 関数「get_comment_type（）」は、タイプ-通常のコメントまたはトラックバックを示します。 2つのHTMLリストを作成すれば十分です。1つ目は通常のコメントで、2つ目はトラックバックで埋めます。



3.コメント内のHTMLリンクを削除する





問題。 ブロガーは彼らの発案を宣伝する機会を逃すことは決してなく、スパマーはまだ姿を消していません。 私はいつも、コメントに含まれる信じられないほどの数のリンクに、質問とはほとんど関係がないことにいつもひどく悩まされています。 デフォルトでは、 WordPressのコメントURLはリンクに変わります。 幸いなことに、私に劣らないリンクにうんざりしている人はこれを修正できます。



解決策。 「function.php」ファイルを開いて、以下のコードを貼り付けてください：



function plc_comment_post( $incoming_comment ) {

$incoming_comment['comment_content'] = htmlspecialchars($incoming_comment['comment_content']);



$incoming_comment['comment_content'] = str_replace( "'", '&apos;', $incoming_comment['comment_content'] );

return( $incoming_comment );

}

function plc_comment_display( $comment_to_display ) {

$comment_to_display = str_replace( '&apos;', "'", $comment_to_display );

return $comment_to_display;

}

add_filter('preprocess_comment', 'plc_comment_post', '', 1);



add_filter('comment_text', 'plc_comment_display', '', 1);

add_filter('comment_text_rss', 'plc_comment_display', '', 1);

add_filter('comment_excerpt', 'plc_comment_display', '', 1);







ファイルを保存した後、コメント内のリンクや他の無関係なHTMLスニペットにメンタルに別れを告げることを忘れないでください。



コード分​​析。 最初に、HTML文字をHTMLオブジェクトに置き換える2つの関数を作成しました。 その後、強力な関数「add_filter（）」を使用して、新しく作成された2つの関数にコメントを処理するための標準のWordPress関数を添付します。 これにより、追加されたすべてのコメントで、HTMLコンポーネントが確実に除外されます。



4.コメントにTwitterアバターを追加する





問題。 ブロガーにとって、Twitterネットワークは、自分のブログを普及させ、仲間のブロガーや彼らの聴衆と連絡を取り続けることができるため、非常に便利です。 Twitterネットワークは非常に人気があるため、グローバルアバター（グラバター）の代わりに、Twitterアバターを使用してコメントを明確に示すことができます。



解決策。

1.最初に、ここから機能ファイルを取得します 。

2.次に、ハードドライブでアーカイブを解凍し、「twittar.php」ファイルを開きます。

3.すべてのコンテンツを選択したら、ブログに関連するファイル「functions.php」に挿入します。

4.最後のステップは、ファイル「comments.php」を開き、コメントのループを見つけることです。

5.次の行をループに挿入します。



<?php twittar('45', 'default.png', '#e9e9e9', 'twitavatars', 1, 'G'); ?>







コード分​​析。 数か月前のSmashing Magazine www.smashingmagazine.comで、TwitterアバターをWordPressブログに組み込むための素晴らしいTwittarソフトウェアモジュールが導入されました。 WpRecipes.comの読者から大量のフィードバックを受け取ったので、私はソフトウェアモジュールをコードに変えることにしました-それを好む人のために。

もちろん、「function.php」ファイルの内容を編集する代わりに、単にアドインをインストールすることができます-選択は完全にあなた次第です。



5.著者のコメントをスタイルで強調します。





問題。 多数のコメントを含む投稿では、特にWordPress 2.7に実装された「スレッドコメント」ツリーコメント機能がブログにない場合、著者のコメントと回答を見つけるのは必ずしも簡単ではありません。 ただし、まだ方法があります。著者のコメントを別のスタイルで強調表示し、読者がすべての答えをすばやく見つけられるようにします。



解決策。

1.「comments.php」ファイルを開き、コメントループを見つけます。



<?php foreach comment as $comment) { ?>







2.この行の後に、次のスニペットを挿入します。



<?php

$isByAuthor = false;

if($comment->comment_author_email == get_the_author_email()) {

$isByAuthor = true;

}

?>







3.次に、コメントを表すコード内の行を見つけます（トピックによって異なる場合があります）。



<li class="<?php echo $oddcomment; ?>" id="comment-<?php comment_ID() ?>">







4.次に、著者のコメントのクラス「authorcomment」を導出する必要があります。



<li class="<?php echo $oddcomment; ?> <?php if($isByAuthor ) {

echo 'authorcomment';} ?>" id="comment-<?php comment_ID() ?>">







5.作成者のコメント用のCSSクラスを作成することは残っています。 「style.css」ファイルを開き、コードスニペットをそこに貼り付けます。 必要に応じて、例の色を置き換えます。



.authorcomment{

color:#fff;

font-weight:bold;

background:#068;

}







コード分​​析。 提案されたコードの動作原理は、コメントエントリの電子メールアドレスと著者のアドレスの比較に基づいています。 一致する場合、$ isByAuthorはtrueに設定されます。 コメントを表示するとき、「$ isByAuthor」の値がチェックされます。 trueを返すと、authorcommentクラスがコンテナに追加されます。

Wordpressバージョン2.7以降では、このタスクはより単純です。クラス「comment_class（）;」をコメントDIVに追加するだけです。これにより、自分の投稿へのコメントの場合にクラス「bypostauthor」が自動的に追加されます（Nimaに特に感謝します！）。



6.コメントの総数と平均記録を表示する





問題。 WordPressコンソールには、ブログが受け取ったコメントの総数に関する情報があります。 しかし、残念ながら、公開するメカニズムはありません。 特にブログに潜在的な広告主向けのページがある場合、ブログのコメントの総数とエントリごとの平均数を表示すると非常に便利です。



解決策。



<?php

$count_posts = wp_count_posts();

$posts = $count_posts->publish;

$count_comments = get_comment_count();



$comments = $count_comments['approved'];

echo "There's a total of ".$comments." comments on my blog, with an average ".round($comments/$posts)." comments per post.";

?>







コード分​​析。 バージョン2.5に存在する「wp_count_posts（）」関数と「get_comment_count（）」関数を使用すると、WordPressブログの投稿とコメントの総数をそれぞれ簡単に取得できます。 レコードのコメントの平均数を計算するには、基本的な算術演算と「PHP round（）」関数で十分であり、整数値が提供されます。



7.最近のコメントのn番目の数を表示する





問題。 デフォルトでは、WordPressには、投稿に対する最近のコメントをいくつでも表示するウィジェットが用意されています。 これはすばらしいことですが、ウィジェットがない場合に同様の機能が必要になることがあります。



解決策。 秘密は非常に簡単です。最後に作成されたコメントの特定の数を表示する場所に提案されたコードを貼り付けます。 行3に特定の数量を含めるようにしてください（式「LIMIT SQL」の後に）。



<?php

$pre_HTML ="";

$post_HTML ="";

global $wpdb;

$sql = "SELECT DISTINCT ID, post_title, post_password, comment_ID, comment_post_ID, comment_author, comment_date_gmt, comment_approved, comment_type,comment_author_url, SUBSTRING(comment_content,1,30) AS com_excerpt FROM $wpdb->comments LEFT OUTER JOIN $wpdb->posts ON ($wpdb->comments.comment_post_ID = $wpdb->posts.ID) WHERE comment_approved = '1' AND comment_type = '' AND post_password = '' ORDER BY comment_date_gmt DESC LIMIT 10";



$comments = $wpdb->get_results($sql);

$output = $pre_HTML;

$output .= "\n<ul>";

foreach ($comments as $comment) {

$output .= "\n<li>".strip_tags($comment->comment_author) .":" . "<a href=\"" . get_permalink($comment->ID)."#comment-" . $comment->comment_ID . "\" title=\"on ".$comment->post_title . "\">" . strip_tags($comment->com_excerpt)."</a></li>";



}

$output .= "\n</ul>";

$output .= $post_HTML;

echo $output;

?>







コード分​​析。 ここでは「$ wpdb」オブジェクトも使用しますが、今回はget_results（）メソッドと組み合わせます。 WordPressデータベースからコメントを受け取ったら、「for」ループを使用してコメントを順序付けられていないHTMLリストに作成します。 フラグメントの先頭で初期化される変数$ pre_HTMLおよび$ post_HTMLは、コメントのリストの前後にどのコンテンツが来るかを決定します。



8.コメントスパムを避ける-ただ





問題。 コメントスパムはよくある頭痛の種です。 Akismetは良い解決策ですが、疑わしいメッセージにフラグを立てるのではなく、スパマーをすぐにブロックしてみませんか？ 提案されたコードは、HTTPリファラーリクエストヘッダー（リクエストの送信元のページ）を探し、コメントが正しくないか定義されていない場合、コメントを自動的にブロックします。



解決策。 次のコードを「functions.php」ファイルに貼り付けます。



function check_referrer() {

if (!isset($_SERVER['HTTP_REFERER']) || $_SERVER['HTTP_REFERER'] == “”) {

wp_die( __('Please enable referrers in your browser, or, if you\'re a spammer, bugger off!') );

}



}

add_action('check_comment_flood', 'check_referrer');







実際、それがすべてです。 ファイルを保存したら、ブログをスパムに対する新しいレベルの保護に導きます。



コード分​​析。 コードは、HTTPリファラーリクエストヘッダーなしで、ブラウザー（通常はボット）からのコメントを投稿するリクエストを自動的に拒否します。 検証は、配列 "PHP $ _SERVER []"を使用して実行されます。 ヘッダーが定義されていないか無効な場合、wp_die関数が呼び出され、スクリプトは終了します。

この関数は、WordPressに実装されている関数「check_comment_flood（）」に添付されています。 したがって、コメントが投稿されるたびに、「check_referrer（）」関数の呼び出しが実行されることを確認できます。



9. 2.7未満のバージョンとのWordPressの互換性をサポートしています





問題。 ほんの数か月前にリリースされたWordPress 2.7のリリースでは、ツリー形式で表示し、さまざまなページに表示できる、根本的に新しいコメントシステムが導入されました。 これは非常に便利ですが、多くのユーザーがアプリケーションをバージョン2.8または少なくとも2.7にまだ更新していないことを忘れないでください。 提案されたコードは、更新を気にしない人のために古い機能を保持しながら、バージョン2.7以降のすべての所有者に新しいコメントシステムの利便性を提供します。



解決策。 このためには、2つのファイルが必要です。 1つ目は、WordPress 2.7でサポートされているコメント「comments.php」を含むファイルです。 2つ目は、WordPressの以前のバージョン「legacy.comments.php」用のコメントテンプレートです。 両方のファイルはテーマフォルダーに保存されます。

次のコードを「functions.php」ファイルに追加します。



<?php

add_filter('comments_template', 'legacy_comments');

function legacy_comments($file) {

if(!function_exists('wp_list_comments')) : // WP 2.7-only check

$file = TEMPLATEPATH.'/legacy.comments.php';

endif;



return $file;

}

?>







コード分​​析。 このコードは、標準関数「comments_template」にバインドされた関数「legacy_comments（）」を作成します。 WordPressが「comments_template（）」にアクセスするたびに、「legacy_comments（）」関数が実行されます。 「wp_list_comments（）」関数がない場合、コードは「comments.php」ではなく「legacy.comments.php」を自動的にロードします。



10.特定の時間に最もコメントの多いエントリを表示する





問題。 コメントの数は、ブログの人気を測定するのに最適な方法であり、最も人気のある投稿をリストするための優れたフィルターです。 たとえば、先月の最も人気のあるエントリのリストを表示するなどして、最も人気のあるエントリを特定の期間に制限するとよいでしょう。



解決策。 最も人気のあるエントリを一覧表示する場所に推奨コードを貼り付けるだけです。 3行目の日付を自分に関連する日付に変更することを忘れないでください。



<ul>

<?php

$result = $wpdb->get_results("SELECT comment_count,ID,post_title, post_date FROM $wpdb->posts WHERE post_date BETWEEN '2009-01-01' AND '2009-02-01' ORDER BY comment_count DESC LIMIT 0 , 10");

foreach ($result as $topten) {

$postid = $topten->ID;



$title = $topten->post_title;

$commentcount = $topten->comment_count;

if ($commentcount != 0) {

?>

<li><a href="<?php echo get_permalink($postid); ?>"><?php echo $title ?></a></li>



<?php }

}

?>

</ul>







コード分​​析。 まず、「$ wpdb」オブジェクトを使用して、SQLクエリをWordPressデータベースに送信します。 結果を受け取った後、単純なPHP演算子「foreach」を使用して、一定期間の最も人気のあるレコードを順序付けられていないHTMLリストの形式で表示します。
