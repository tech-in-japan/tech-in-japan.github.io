ルーターを介したTwitterからVKへのクロスポスト

こんにちは、私の名前はユージーンです。私はアルコール愛ソーシャルネットワークです。 才能を書くヒントがないため、私はTwitterを好みます。140文字という制限があるため、長いものを思いつく必要はありません。 さらに、twitter専用の通常のJavaクライアントがあり、これを携帯電話で使用していました。

一方で、私の友人のほとんどはVKontakteを使用しており、正直にそこで音楽を聴いてニュースを読むのが好きです。 明らかに、2つの異なるソーシャルネットワークで自分自身を書くのは不便です。これを自動化する必要があります-クロスシェアリング（転送後）を使用します。

以下は、あまり一般的ではないクロスシェアリング方法の簡単な説明です。



もう少し背景


以前は、IFTTTを使用してレコードを転送しました。 あまり便利ではありませんでした-投稿は15分ごとに延期され、特殊文字がエンコードされ（「に変換＆quot;」）、リンクが短縮されました（t.co）-Vkontakteはこれらのリンクをたどろうとしたときに誓いました。ただし、Twitter APIの最近の変更によりIFTTT は、ツイートを読み取る「レシピ」をオフにすることを余儀なくされました 。それは常にではありません（明らかに再び 彼らはリクエストの数に関する新しいTwitter APIの制限に取り組みました）、さらにリンクに問題がありました。

もちろん、最も簡単な方法は、レコードを転送するスクリプトを使用することです。 ハブラーで、クロスシェアリングのさまざまな方法について繰り返し 書きましたが、これらの方法はすべて、自分のサーバーがない状態で休みました。 1つのシンプルなスクリプトで有料のホスティング（安価ではありますが）を購入することはほとんど意味がありません。

しかし最近、ルーターのファームウェアの新しいバージョンを探してインターネットをクロールし、Zyxel Keeneticに基づいて小さなWebサーバーを作成できることに気付きました！ 購入する前に、ルーターのこれらの機能について読みましたが、どういうわけか頭から飛び出しました。 そして、私は計画されたプロジェクトの実装を開始しました-ルーターを使用してTwitterからVkontakteへのクロスシェアリング。



モスクワはすぐに構築しませんでした


まず、Zyxel Keeneticの機能について少しお話します。 つまり、さまざまな既製のモジュール（php、lighttpd、dlna、transmission、perl、pythonなど）をルーターにすばやく、簡単に、安全にインストールできます。 フォーラムで詳細を読むことができます。

プロジェクトを開始する前に、ウィッシュリストのリストを作成しました。 スクリプトは次のことができるはずです。

  • ツイートが表示された直後、またはcronを介してスクリプトを実行するときにツイートを転送する
  • 回答、リツイート、参照を転送しない
  • つぶやきテキストのt.co短縮リンクを展開する
  • 可能であれば、添付された写真をコピーします（例：Instagram）


まず、スクリプト言語を選択する必要がありました。 ルーターにはPerlモジュールとPythonモジュールがありますが、これらの言語にも私は不慣れでした。 また、Twitter APIとVkontakte APIに出くわしたこともなかったので、主にそれらを使用した既製の例に興味がありました。 幸いなことに、このようなスクリプトはネット上で十分に見つかりました。 どういうわけか私はPerlを選んだ:)

その後、Twitter APIのドキュメントと例に直接進みました。 ご存知のように、ストリーミングとRESTの2つのタイプがあります。 最近のAPIの変更では、開発者をStreaming APIに移行したいというTwitterの要望がはっきりと現れています。 したがって、将来問題が発生しないように、ストリーミングを使用することが決定されました。



スクリプトは自分専用に作成したため、アクセストークンは取得しませんでしたが、Twitterアプリケーションの設定ページからすぐに取得しました。 数時間のうちに、 perlモジュールの助けを借りて、ツイートを表示する簡単なコンソールアプリケーションが作成されました。 すべてが私のコンピューターでうまくいきました。 ルーターを確認するために、必要なすべてのパッケージをインストールし（私には思えたように）、スクリプトを実行しましたが、何もありませんでした。 動作しません。 Perlで利用可能なすべてのパッケージをインストールしましたが、役に立ちませんでした。 エラーのテキストをグーグルで調べてみると、問題の原因はおそらく何らかのperlモジュールの欠如にあり、その結果、スクリプトはSSLで正しく動作できないことがわかりました。 最終的に、Perlを離れてPythonを試すことにしました。



Perlは死んでいて、長生きするPython

VKを使用する


VKにはわずかに異なる認証メカニズムがあり、アクセストークンの取得だけが失敗します。

まず、デスクトップアプリケーションを作成し、そのIDを取得する必要があります。 次に、フォームのリンクを形成します oauth.vk.com/authorize?client_id={ID}&scope=wall,photos,offline&redirect_uri=http://oauth.vk.com/blank.html&display=page&response_type=token



oauth.vk.com/authorize?client_id={ID}&scope=wall,photos,offline&redirect_uri=http://oauth.vk.com/blank.html&display=page&response_type=token



で、{ID}はアプリケーションIDです。 このリンクは、アプリケーションの許可も示します-壁に投稿するための壁、画像を埋めるための写真、受信したトークンが永遠になるようにオフラインにする。 このリンクをブラウザにコピーします。 VKは、このアプリケーションへの権限の付与について質問し、トークンを含むURLにリダイレクトします。



このようなラッパーは、VK APIメソッドを操作するために使用されます。 コードは私のものではありませんが、どこで入手したかを完全に忘れてしまいました。著者を示さなかったことをおaびします。

ルーターへのスクリプトのインストール


完成したスクリプトはgithubからダウンロードできます。 もちろん、その中のトークンは変更されます:)自分で取得する必要があります。

wikiの指示に従って、サードパーティのパッケージをサポートするようにルーターを構成します。 Windowsの簡単な説明を次に示します。

  1. NTFSでフラッシュドライブをフォーマットし、その上にシステム/ binフォルダーを作成します
  2. フラッシュドライブをルーターに挿入し、エクスプローラーで開き、 ext_init.shファイルをそこにコピーします
  3. ディスクを再マウントします-Webインターフェースを使用して、ディスクを取り外し、フラッシュドライブを差し込みます。 エントリ「 dropbear[4017] Running in background



    」がルーターのログに表示されるのを待っています
  4. SSH（root / zyxel）を介して接続し、finish_install.shを実行します。 スワップの作成を待っています。


ルータはopkgパッケージをインストールする準備ができています。 それらのリストは、 Webサイトまたはopkg list



コマンドで表示できます。 Pythonと必要なモジュールをインストールします。

次に、コマンドを使用してpythonをインストールします。

  1. opkg install python



    およびopkg install python-openssl



    コマンドを使用した基本パッケージ
  2. フォルダーをsystem\usr\lib\python2.7\



    コピーするだけで、モジュールをTweepyおよび要求します。 新しいバージョンのリクエストは機能しませんでした。 キャッシュブランチまたはアーカイブから試してください。
  3. cronを有効にするには、システム\ etc \ init.dフォルダーのK02cronファイルの名前をS02cronに変更します
  4. cronで実行するようにスクリプトを構成します。 システム\ etc \ crontabs \ rootファイルに、次の行を追加します 
     */15 * * * * killall -9 "python" ; /media/DISK_A1/system/usr/bin/python /media/DISK_A1/system/root/TwVk.py
    これで、cronは毎時0/15/30/45分でスクリプトを強制終了し、再度実行しますが、これは非常に曲がっていますが、 @reboot



    再起動後に動作するcronの標準コマンドは@reboot



    しませんでした。 それに、スクリプトも完璧ではありません。再起動しても彼を傷つけることはありません。 誰かがより良いアイデアを提案できるなら-お願いします。
  5. 最後に、スクリプトをsystem \ root \にコピーして、ディスクを再度マウントします。
