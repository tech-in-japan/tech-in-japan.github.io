待つことができない人のためのAngular 2.0.0-alpha



ごく最近（3月5〜6日） ng-conf会議が開催され、Angular 2の今後のリリースに関する多くの報告が行われました。 もちろん、聞いた後、私は本当に個人的に試してみたかったです。 待ちきれないなら、猫をお願いします。



この記事では、Angular 2.0ビルドのアルファ版の入手先を見つけ、その上に小さなTo Doリストアプリケーションを作成し、ECMAScript 6を完全にサポートせずにブラウザーで起動します。



Angular 2は現在のバージョンとは大きく異なり、ほとんどすべてが再考されています。 違いの多くは、彼がアノテーションとタイプを備えたES6を最大限に使用しているという事実に由来し、開発者はこの言語をAtScriptと呼びます。 どうやら、AngularチームはMicrosoftと緊密に連携し始め、 TypeScriptで開発が行われ、バージョン1.5のリリース後、AtScriptが完全に含まれるようになりました。



Angular 2アプリケーションは、コンポーネントで構成され、ツリーになりました。 この考え方はWebコンポーネントと似ており、Angular 2コンポーネントのマークアップもShadow DOMに配置されます。 さらに、アプリケーションでWebコンポーネント、またはたとえばPolymerを使用する場合、構文は独自のAngular 2コンポーネントを使用する場合と変わりません。 コンポーネント自体は注釈付きのES6クラスであり、たとえばバージョン1.xのディレクティブの場合、特別な構文は必要ありません。 サービスも通常のクラスになり、AtScriptでの入力のサポートのおかげで、構文を使用せずに、タイプごとにサービスを挿入できます$ InjectまたはngAnnotate。



では、最初のTo-doアプリケーションを作成してみましょう。その過程で、何をどのように理解するのでしょうか。



現在、Angular 2が機能するために必要なすべての機能をサポートするブラウザはないため、次のようなツールが必要です。

  • rtts-assert
  • es6-shim
  • System.js
  • Zone.js
  • 注釈をサポートするオプションを備えたトレーサー


開発者は、アルファ版をすべて個別に設定する必要がないと感じたすべての人に、クイックスタートをまとめました。 プロジェクトフォルダーに入力することでインストールできます。 



git clone https://github.com/angular/quickstart.git


必要なものはすべてクイックスタートフォルダーにあり、美しさのためにangle -materialを追加しますが、もちろんそこからcssのみを使用します： 



 bower install angular-material


次に、プロジェクトにindex.htmlファイルを作成します。 



 <!DOCTYPE html> <html xmlns="http://www.w3.org/1999/xhtml"> <head> <title>To Do</title> <script src="/quickstart/dist/es6-shim.js"></script> <link href="bower_components/angular-material/angular-material.css" rel="stylesheet" /> </head> <body layout="row"> <style> md-progress-circular.md-default-theme.blue .md-inner .md-right .md-half-circle { border-right-color: #039be5; } md-progress-circular.md-default-theme.blue .md-inner .md-left .md-half-circle, md-progress-circular.md-default-theme .md-inner .md-right .md-half-circle { border-top-color: #039be5; } md-progress-circular.md-default-theme.blue .md-inner .md-left .md-half-circle { border-left-color: #039be5; } md-progress-circular.md-default-theme.blue .md-inner .md-gap { border-top-color: #039be5; border-bottom-color: #039be5; } </style> <app flex layout="row" layout-fill> <div layout-fill layout="row" layout-align="center center" layout-margin> <md-progress-circular md-mode="indeterminate" class="md-default-theme blue" style="-webkit-transform: scale(1);"> <div class="md-spinner-wrapper"> <div class="md-inner"> <div class="md-gap"> </div> <div class="md-left"> <div class="md-half-circle"></div> </div> <div class="md-right"> <div class="md-half-circle"></div> </div> </div> </div> </md-progress-circular> <h4>    To do app is loading... </h4> </div> </app> <script> System.paths = { 'angular2/*': 'quickstart/angular2/*.js', 'rtts_assert/*': 'quickstart/rtts_assert/*.js', 'app/*': 'app/*.js' }; System.import('app/App'); </script> </body> </html>




ここでは、次の3つの点に注意する必要があります。



まず、es6-shimファイルを含めました。アプリケーションを実行するために必要な上記のすべてを注意深く収集しました。



次に、アプリ要素に注意してください。 これがアプリケーションのメインコンポーネントになります。 これで、スピナーとマークアップが含まれ、To Doアプリの碑文が読み込まれます...角度がコンポーネントクラスを読み込み、このマークアップをテンプレートで置き換えるまで表示されます。



一番下では、system.jsライブラリを使用しました。これは、ES6モジュールロードシステムを実装しています。 最初に、System.pathsを使用して、.jsファイルへのパスを構成し、次にSystem.importを使用して、アプリコンポーネントと共にファイルをダウンロードしました。次に、必要なすべてのライブラリとコンポーネントを取得します。



次に、アプリフォルダーを作成し、App.jsファイルをその中に配置します。 



 import {Component, Template, bootstrap, Foreach} from 'angular2/angular2'; import {TodoStore} from 'app/TodoStore'; import {ItemEditor} from 'app/ItemEditor'; @Component({ selector: 'app', componentServices: [ TodoStore ] }) @Template({ url: 'app/todo.html', directives: [Foreach, ItemEditor] }) class App { constructor(store:TodoStore) { this.store=store; } } bootstrap(App);




最初に、angular2.jsからコンポーネント、テンプレート、Foreachクラスをインポートし、そこからブートストラップ関数をインポートします。バージョン1.xでng-appディレクティブを使用していない人は、アプリケーションのメインコンポーネントを初期化してから、アプリケーションからTodoStoreクラスをインポートします/todoStore.js（これは、localStorageブラウザーのタスクとそのストレージのリストを操作するサービスです）およびapp / item-editor.jsのItemEditor（これはコンポーネント-リストアイテムを表示するディレクティブ）です。 まだそれらを書いていないので、後でレビューします。



次に、Appクラスを作成します。TodoStoreクラスのオブジェクトは、storeプロパティに割り当てるインジェクターを使用して渡されます。



クラスを宣言するとき、angular2.jsのコンポーネントとテンプレートのアノテーションを使用しました。

コンポーネント-クラスをコンポーネントにする注釈。 セレクタープロパティは、テンプレートのレイアウト内の要素を検出できるセレクターです。このアプリ要素があります。 componentServicesプロパティは、インジェクターのクラスのリストです。



テンプレート-コンポーネントのテンプレートを指定するのに役立ちます。 urlプロパティはテンプレートが配置されるパスを示し、directivesプロパティはこのテンプレートのマークアップで使用されるディレクティブのリストを示します。



ファイルの最後で、ブートストラップ関数を使用してメインコンポーネントを初期化します。



app / TodoStore.jsファイルを作成しましょう： 



 export class TodoStore { constructor() { this.load(); } save() { window.localStorage['todoItems']=JSON.stringify(this.items); } addItem(name, checked) { this.items.push(new Item(name,checked)); } clear() { this.items.length=0; } load() { this.items=[]; let itemsStr=window.localStorage['todoItems']; if(itemsStr) { JSON.parse(itemsStr).forEach((e) => { this.addItem(e.name, e.checked); }); } } } class Item { constructor (name, checked){ this.name = name; this.checked=checked || false; } toggleCheck() { this.checked=!this.checked; } }


TodoStoreクラスには、save（）、load（）、clear（）、addItem（name、checked）のメソッドがあります。



タスクリスト自体は、itemsプロパティに保存されます。 save（）メソッドは、itemsプロパティの値をlocalStorageに保存します。 clear（）メソッドは、itemsプロパティをクリアします。 load（）メソッドは、リスト内のすべてのアイテムをlocalStorageからロードし、addItemメソッドを使用してアイテムに追加します。 addItem（name、checked）メソッドはリストアイテムの名前を取得し、新しいItemオブジェクトを作成してそれをItemsに追加します。



次はItemクラスの宣言です。コンストラクターは、名前（リスト項目の名前であり、チェック済み）が完了としてマークされているかどうかを受け取ります。 このクラスには、checked値を反対の値に変更するtoggleCheck（）メソッドもあります。



次に、メインコンポーネントテンプレートでapp / todo.htmlファイルを作成します。 



 <style> @import "../bower_components/angular-material/angular-material.css"; .md-primary { background-color: #039be5; color: white; } md-toolbar { background-color: #CFD8DC; } md-input-container { padding-bottom: 0; } </style> <div flex layout="column" layout-fill layout-margin> <md-toolbar class="md-primary"> <h2 class="md-toolbar-tools"> <span>To do list</span> </h2> </md-toolbar> <md-content layout="row" layout-align="center start" flex> <div flex="50" flex-md="80" flex-sm="100"> <div layout="row" layout-margin flex> <button flex class="md-button md-primary md-raised md-default-theme" (click)="store.save()">Save</button> <button flex class="md-button md-primary md-raised md-default-theme" (click)="store.load()">Load</button> <button flex class="md-button md-primary md-raised md-default-theme" (click)="store.clear()">Clear</button> </div> <div layout="row" layout-margin flex> <md-input-container class="md-input-has-value" flex="60"> <input #newitem class="md-input" placeholder="New to do item" /> </md-input-container> <button flex class="md-button md-primary md-raised md-default-theme" (click)="store.addItem(newitem.value)">Add new item</button> </div> <md-list> <md-item *foreach="#item in store.items"> <md-item-content> <item-editor [item]="item"></item-editor> </md-item-content> </md-item> </md-list> </div> </md-content> </div>


ここには、ストアメソッドstore.save（）、store.load（）、store.clear（）を呼び出す3つのボタンがあります。 （click）属性を使用して、各ボタンのクリックイベントにバインドします。 角度の現在のバージョンでは、イベントにバインドするために使用される属性と、データをバインドするために使用される属性をマークアップで区別するのが一見困難な場合があります。今ではすべてがすぐに表示され、要素またはコンポーネントのイベントに応答するには括弧（）が必要です。角括弧[]の属性はデータにバインドされ、属性値は計算される式であり、その結果は要素の属性に割り当てられることが理解されます。属性が括弧なしで指定されている場合、 その中にテキスト値。



次に、＃newitem属性を使用した入力に注意を払うことで、下のボタンのクリックイベントのように要素に名前を付けることができるようになりました。入力値-newitem.valueを取得し、store.addItemに渡します。 （）。



次は、新しい* foreachディレクティブを使用して導出したタスクのリストです（これはng-repeatの代替です）。ここでは、上記の入力のように、itemという名前のリストの各アイテムを参照する#itemを作成しました。



item-editorディレクティブを使用して各リスト項目を表示し、[item]属性を使用してリスト項目をバインドします。



ここで、ItemEditorコンポーネントを使用してapp / ItemEditor.jsファイルを作成します。これにより、リストアイテムの表示が行われます。 



 import {Component, Template} from 'angular2/angular2'; @Component({ selector: 'item-editor', bind: { 'item': 'item' } }) @Template({ inline: ` <style> @import "../bower_components/angular-material/angular-material.css"; md-checkbox.md-checked .md-icon { background-color: #039be5; color: white; } </style> <md-checkbox [class.md-checked]="item.checked" (click)="item.toggleCheck()"> <div class="md-label"> <div class="md-container"> <div class="md-icon"></div> </div> <span> {{item.name}} </span> </div> </md-checkbox> ` }) export class ItemEditor { }




ここ@Componentでは、「item-editor」セレクターを指定しました。これは、todo.htmlの* foreachで使用される要素です。 bindプロパティが続きます。これは、item属性の値をコンポーネントクラスの項目プロパティ（bindToControllerなど）にバインドする必要があることを示します。



今回のテンプレートでは、たとえばurlの代わりに、インラインプロパティが指定されており、コード内でテンプレートのテキストを直接指定するのに役立ちます。 これにより、必要なhttpリクエストの数が減るので、Angular 2のリリース後、現在のgulp-angular-templatecacheを置き換えるために、URLをインラインに変更するGulpのプラグインがおそらくあるでしょう。



テンプレート自体には、[class.md-checked]属性があります-これはng-classの簡潔な代替品です。 以下は、既にネイティブになっている中括弧{{item.name}}を使用した構文を使用したリストアイテムの名前です。
