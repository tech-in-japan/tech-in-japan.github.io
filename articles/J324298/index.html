認定Openstack管理者（COA）試験の経験

そのような材料は、ネットワーク上に実質的に存在しません。 したがって、試験の許容条件内での推奨事項を記載した短いメモ。





OpenStackフライホイールは近年勢いを増しており、人々を引き付け、専門家のプールを作成する必要があるため、比較的費用がかからないCOA試験がリリースされました。OpenStackで作業する人の準備は難しくありません。



2月にゼロから始めた人口を表し、3月中旬には92％の結果で合格しました。



準備方法



OpenStackの主なことは、システムのアーキテクチャとモジュールの相互作用を理解することです。 これがまさに最も重要な最初の困難を引き起こすものです。 すべての詳細とコマンドはすでに技術の問題です。



理論



Openstack Operation Guideから始めました。 彼女は少し古く（2014年）、Icehouseのリリースについてです。OpenStackを絶えず更新する世界では、長い間忘却に沈んでいましたが、そこにはコマンドやリストがほとんどないため、喘ぐことなく仮想化に没頭することができます。



ちなみに、 モジュールのウェブサイトに翻訳があります 。 正直なところ、そこに元のPDFをダウンロードすることもできます 。



リストがある場合は、 xgu.ruにたくさんあります。



練習する



アーキテクチャに関してすべてが明確になったら、私たちは実践に移ります。



次の3つのオプションがあります。



  1. 最も便利です。 2〜3個の鉄の馬を見つけ、モジュールまたはDevStack（すべてを自動的に配置する特別なスクリプトセット（aha、aha、これらのマシンを知っています））によってOpenStackを手動でゼロからインストールします。



  2. 影なしの中間。 1台の強力なPC + Ubuntu + DevStack（またはモジュール別）。 この場合のインストールは簡単ですが、Ubuntuとサーバー間の仮想化がまだある場合は、ネットワーク設定をいじる必要があります。デフォルトでは、OpenStackの仮想マシンはコントローラーノードからpingしません。



  3. 最もシンプル。 linuxacademy COAコースを購読します。 同時に、すべてのトレーニング資料とラボにアクセスできます。


最初の2つのデメリットは、体系的なトレーニングプランが提供されないことです。 私は3番目の方法に目を向けましたが、同時に弱いラップトップにWMWare + Ubuntu + Devstackをインストールし、後でCentOSでインスタンスを起動しました-この方法に加えて、最新のリリースの1つを取ることができます。 この場合、仮想マシンのスナップショットを作成し、それから開始して、前にopenstackで行ったことをすべて忘れて、ゼロから開始できます。 マイナスは、上記で述べたように、ネットワークでは単純ではありません。



Linuxの場合、トレーナーが完璧な言葉遣いで記録した43時間の詳細なビデオと、一連のラボがあります。 Labaは、プリセットのない裸のOpenstackです。 つまり、それらはラボガイドによってのみ互いに​​異なります。 便利なことに、学習プロセスは体系化されており、何を何の順序で行うかを常に知っています。 これは非常に便利です。一般的に言えば、専門家、材料、賢明なガイドのかなり狭いサークルにとって、オープンカードはまだ大きな話題ではないからです。 もちろん、直観、試行錯誤の方法を使用して準備できますが、これは毎日openstackを使用している場合に限ります。



すでに試験に合格した後、Linuxコースはプログラムの大部分をカバーしていますが、全体をカバーしているとは言えません。 ドメイン/グループやマルチインストールなど、トレーニングビデオで一度も表面化したことのないものに出くわしました。



熱とスタックの主題は非常に不十分に開示されており、ラボはまったくありません。 しかし、比較的単純ではありますが、タスクは発生しました。 まあ、一般的に、理論上すべてがうまく整理されていれば、ラボのタスクは短くて簡単です。



ただし、タスクの複雑さに関しては、Linuxアカデミーのラボの場合とまったく同じであるため、それほど複雑なことはありません。



たとえば、準備を始めたばかりのとき、タスクがサービスの中断を最小限に抑えてカテゴリからOpenstackリリースを更新すると考えました。 いいえ、すべてがはるかに簡単です。



服用方法



伝統的に、これはすべて、手順とタスク自体の両方で、最も厳格な信頼に保たれています。 ただし、試験は実用的であり、既存のインフラストラクチャをサポートする学生の経験を特定することを目的としていることが知られています。 これは、端末またはWebインターフェースを介して、2つの方法のいずれかで何かを実行するだけのタスクである、解答オプションに関する理論的な質問がないことを意味します。



初心者として、2.5時間は私の近くに現れました。 ほとんどのタスクは比較的単純です。



コンソールとWebインターフェースが同一ではないという事実に備える必要があります-すべてをGUIで実行できるわけではありません。 しかし、私の意見では、GUIは、手動で名前を入力したりUUIDをコピーしたりするのではなく、タスクの大部分を完了するための高速な方法です。 したがって、準備中に両方のスキルを高めます。



ブラウザーを除く、ブラウザーまたは他のアプリケーションで新しいタブを開くことはできません-credenitalsファイルをダウンロードしてコンテンツをコンソールにコピーしようとすると、すぐにメモ帳を開くための警告が表示されます。



監督者の許可があれば、ドキュメントdocs.openstack.orgで追加のタブを開くことができます。 一般的に、すべては監督官の許可を得て。



Ctrl + Wは使用しないでください！ 試験の数週間前に、このホットキーの使用を機敏なペンで止めるようにしてください。



私は常にコンソールで作業しているので、麻痺していても、コントロールはすでに反射的です。私の指は1日に数回痙攣します。



Yandexブラウザーでは、何かが発生した場合にタブを閉じることができないという希望がありました。 彼は与えませんでした。 しかし、試験は中断されました-ただの白い窓。 それからラップトップを壊しませんでした。



しかし、監督は協力的で、試験を続けることができました-私は最初ではなく、最後ではないようです。 そして二度目、彼も許可しました。 したがって、Ctrl + Wはまだ文ではありません。



コマンドの構文や何かをする方法を忘れた場合、ドキュメントは本当に役立ちます。 たとえば、ヘルプ-パラメーターを一覧表示するためにすべてのコマンドに対して実行されるわけではなく、ファイルの有効期間をswiftで設定する方法が実行されることがわかりました。例からコピーしました。



rcファイルの作成方法を理解しておくと役立ちます。



コンピュータと室内の両方の環境の要件は、 ハンドブックで詳細に説明されているため、簡単に説明します。



試験で必要なもの：



  • カメラ
  • オープンインターネットアクセス
  • 英語で写真と名前/姓がある国際パスポート、権利、またはその他の文書。
  • ブラウザChrom / Chromium（Yandexも登場しました）。
  • ブラウザアドオンにインストールして、画面にアクセスできるようにします（Innovative Exams Scrrensharing）。
  • 他のアプリケーションやタブが存在しないことは、試験の1つのウィンドウにすぎません。
  • テーブルの上に書類、ペン、飲み物、食べ物、その他のものがない。


お勧め：



  • 大型モニター
  • 事前におしっこ-2.5時間は非常に長い時間であり、時間はほぼエンドツーエンドで与えられます。
  • 問題については直線的にしか移動できないため、見逃さない方がいいです。スクロールには多くの時間がかかります。
  • 質問を見逃し、その番号を覚えておく必要がある場合は、ターミナルでテキストファイルを作成し、コメントとともにそこに書き込みます。
  • 最初の試行ではあまり心配しないでください-Openstackは2番目の試行を無料で提供します。 私の場合、最初の試みで、準備プロセスでカバーされなかったトピック、時間の管理方法、小さなモニターが失敗ではないにしても神経質な緊張を保証し、質問間を行き来するのは退屈であり、リソース集約的。
