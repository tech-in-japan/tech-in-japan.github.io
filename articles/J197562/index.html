開発者がWindows Phone 8の3回目の更新を発表し、利用可能

Windows Phone 8の3番目の更新が昨日発表されました 。 そして、最も興味深いのは、この更新プログラムが、開発者向けプログラム「 Windows Phone Preview for Developers」で開発者に直ちに提供されたことです。





Windows Phone 8の3回目の更新には何が含まれていますか？

大型の高解像度画面のサポート

5インチと6インチの画面と1080pの解像度を持つ将来のWindows Phoneデバイスは、4つではなく最大6つの小さなライブタイルを連続して開始画面に表示できます。





新しい強力なプロセッサのサポート

このアップデートにより、4コアのQualcomm Snapdragon 800プロセッサーのサポートが追加されます。



運転モード

電話が特定のBluetoothデバイス（車のヘッドユニットまたはその他のハンズフリーデバイス）に接続されている場合、テキストおよび音声通話、運転中のシステム警告の応答モードを構成できます。





共有インターネットへのアクセスの改善

これで、Bluetoothと携帯電話とWindows 8.1デバイスを接続し、ネットワーク名を選択するだけで、携帯電話からインターネットを使用できます。



その他多数...

パフォーマンスが向上し、目で確認できる-Windows Phone Preview for Developersプログラムでこの更新プログラムを受信した後、携帯電話がどのように速く動作し始めたかを確認しました。



さらに、このアップデートでは、インスタントメッセージ、手紙、ボイスメール、およびリマインダーに独自の通話を使用できます。 特定の連絡先からのテキストメッセージに独自の通話を割り当てることもできます。これにより、電話を見なくても誰があなたに手紙を書いているかを理解できます。



自動回転画面をオフにできる設定を追加しました。 ソファに横になっている読書はさらに便利になりました！



高度な電話ストレージ管理。



アプリケーション間の切り替えリストに表示されるアプリケーションを閉じる機能。



電話の最初の起動時にWiFiに接続する機能。



Bluetoothパフォーマンスの改善。



そして、このアップデートは現在、すべての開発者が利用できます！

先ほどお約束したように、Windows Phoneの3回目の更新の発表に加えて、数か月後にユーザーの電話に流出することに備えて、Windows Phone開発者がこの更新プログラムにアクセスできるプログラムを開始します。 。



誰が参加できますか？

次の3つの条件のいずれかを満たす必要があります。

または電話のロックが解除されている-http://msdn.microsoft.com/en-us/library/windowsphone/develop/ff769508 (v= vs.105).aspx#BKMK_UsingtheWindowsPhoneDeveloperRegistrationTool ;

または、登録済みのWindows Phoneストア開発者-http://msdn.microsoft.com/en-us/library/windowsphone/help/jj206717(v = vs.105).aspx ;

または、登録済みのWindows Phone App Studio開発者であるapps.windowsstore.comです。



Microsoftアカウントの所有者が1台の電話のロックを解除できることを考えると、これは自分の危険とリスクで更新を試してみたいすべての愛好家向けのオファーです。 なぜなら メーカーの保証を失う可能性があります。



詳細な更新手順は、次のページで入手できます。dev.windowsphone.com / en-us / featured / update3



開発者向けの重要なアップデート3の注意事項

3回目の更新には新しいAPIや更新されたAPIはないため、SDKの更新はありません 。 エミュレーターの新しいイメージを使用したオプションの更新があります。 ほとんどの開発者は、3番目の更新で電話で動作するためにアプリケーションで何もする必要はありません。



注意する価値のある変更がいくつかあります。

1）1GB以上のメモリを搭載した電話機でバックグラウンドオーディオ再生に使用できるメモリは、25Mbに増加します。

2）ビューポートInternet ExplorerおよびWebBrowserコントロールは、すべての電話の許可について変更されました。

3）1080pの新しい解像度のサポート-対応する解像度の「画像」を使用したい開発者に影響します。
