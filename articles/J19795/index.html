XMLの真実：歴史から

1969年に、 IBM開発チームの1つがGeneralized Markup Language （ GML ）を開発しました。これは、法律事務所で公式文書を交換するために使用されました。

1974年、Charles GoldfarbはGMLをStandard Generalized Markup Language （ SGML ）と呼ばれる新しい汎用マークアップ言語に変換しました。ISOはその複雑さを理由に1986年に採用しました。



インターネット技術文書で使用するためにSGMLを簡素化する目的で、 Tim Berners-LeeとAnders BerglundはHTMLを作成しました。



最後に、 W3Cが新しい標準のExtensible Markup Language （ XML ）の導入に同意したのは1998年までではありませんでした。 XMLは、構造化データを保存するためのテキストベースの形式です。 その目的は、異なる情報処理システム間で構造化データを送信するとき、特にインターネット経由でそのようなデータを送信するときの互換性を確保することでした。 現在、インターネット上には、 RDF、RSS、MathML、XHTML、SVGなどのデータ転送用のXML形式のファミリーがすべてあります。



XMLは一見HTMLに似ていますが、その主な機能は独自のタグセットを作成できることです。 したがって、独自のマークアップ言語を作成するために使用できます。

XMLの主な利点は、データを記述するために使用されるデータとマークアップタグを含むテキストドキュメントであることです。 XMLドキュメントを操作するアプリケーションは、解析を使用してデータを抽出します。
