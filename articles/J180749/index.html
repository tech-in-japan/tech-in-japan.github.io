OpenRemoteを使用したZ-Waveベースのスマートホームモバイルアプリケーション

OpenRemoteは、ボタン、スイッチ、ラベルなどを作成するコマンドとインターフェイスデザイナーを実行するサーバーです。また、これらの要素にコマンドを既に割り当てています。この場合、これらはZ-WaveサーバーでJavaScript関数を実行するためのHTTPリクエストです。

Raspberry + RaZberryコントローラーを使用してZ-Waveネットワークを作成する



RaZberryはGPIOを介してRaspberry Piに接続されているボードで、Z-Waveネットワークを作成および管理できます。 よりよく理解するには、 よくある質問をお読みください。 ボードには、インストールする必要があるソフトウェアが含まれています。



RaspberryにRaZberryソフトウェアをインストールする



インストーラーはDebianベースのディストリビューション（Raspbian、Xbianなど）のみをサポートしますが、OpenElecおよびその他のOSにソフトウェアをインストールするためにハンドルを使用できます。

コマンドを実行してZ-Wayソフトウェアをインストールします。 

wget -q -O - http://razberry.z-wave.me/install | sudo bash


インストール後、シリアルポートの変更を適用するには、Raspberry Piを確実に再起動する必要があります。



スマートホームコントロールパネルにアクセスするには、アドレスhttp：// IP_OF_YOUR_RASPBERRY：8083にアクセスします 。 プログラムのインターフェイスは非常にシンプルで、理解するのは難しくありません。 ドキュメントがあります 。 インターフェースとの最大の互換性は、ブラウザー：Chrome、Safari、Firefox、その他のブラウザー：IE、Operaが正しく動作しない可能性があります。

提供されている自動化APIについて



すべてのデバイスが正しく機能していることを確認した後、それらをリモートで管理しようとします。 いくつかの方法があります。



1. HTTP / JSON APIを使用する

2. JavaScript APIを使用する



HTTP / JSON APIは、単純な構文を使用してデバイスを管理します。

ブラウザからライトをオンにできます。 

 http://192.168.1.113:8083/ZWaveAPI/Run/devices[6].instances[0].SwitchBinary.Set(255)


または、温度をリクエストします。 

 http://192.168.1.113:8083/ZWaveAPI/Run/devices[8].instances[2].commandClasses[49].data[1].val.value


JavaScript APIを使用すると、さまざまな自動化スクリプトを作成できます。たとえば、ライトのオン/オフ、センサーへの問い合わせ、温度の取得、オンの2分後にライトをオフにします。 これらのスクリプトは両方独立して機能し（たとえば、夜に目が盲目にならないようにライトが15％だけ点灯する）、 HTTP / JSON APIを使用してリモートで呼び出すことができます 。



スクリプトのライト： 

 SwitchOn = function(N,I) { zway.devices[N].instances[I].SwitchBinary.Set(255); }


スクリプトを呼び出してライトをオンにします。 

 http://192.168.1.113:8083/JS/Run/SwitchOn(6,0)


残念ながら、OpenRemoteからHTTP / JSON APIを使用してデバイスに直接アクセスするには、いくつかの理由で問題があります。



1.文字[]は、OpenRemoteコンストラクターでUTF-8にトランスコードする必要があります

2.センサーをポーリングするとき、OpenRemoteは「オン」または「オフ」を予期し、Z-Waveセンサーは255または0を送信できます。

3.各デバイスに対して独自のリクエストを作成する必要があります。JSを使用すると、リクエスト内の関数パラメーター（デバイス番号）のみを変更することで、1つの関数のみを使用してさまざまなデバイスをオンにできます。



JavaScript APIを使用すると、これらの問題はすべてなくなります。「ヘルパー」のいくつかの機能は、Z-Waveの用語をOpenRemoteにとって便利な概念に変換するのに役立ちます。

RaZberryに関する最近の記事で、HTTP / JSON APIおよびJavaScript APIの構文について詳しく読むことができます。



リモートコントロール用のJSスクリプトの作成



JSスクリプトは/ opt / z-way-server / automation /にあり、自動化関数openremote.jsを格納するファイルを作成して、Z-Wayがオンになったときに自動的にロードされるようにし、メイン自動化ファイルmain.jsの最後に追加します： 

 // ====================================================== executeFile(automationRoot + "/" + "tags.js"); executeFile(automationRoot + "/" + "openremote.js"); startAutomation();


/opt/z-way-server/automation/openremote.js 

 //   SwitchOn = function(N,I) { zway.devices[N].instances[I].SwitchBinary.Set(255); } //   SwitchOff = function(N,I) { zway.devices[N].instances[I].SwitchBinary.Set(0); } //     (/ ) SensorStatus = function(N,I) { return zway.devices[N].instances[I].SensorBinary.data.level.value; } //     (/) SwitchStatus = function(N,I) { return zway.devices[N].instances[I].SwitchBinary.data.level.value; } //       Temperature = function(N,I) { return Math.round(zway.devices[N].instances[I].commandClasses[49].data[1].val.value); }




パラメーターNは、ネットワーク上のデバイス番号です。

パラメータI-1つのデバイス内に、物理的に複数のデバイス（チャネル）、たとえば2つのリレーまたは温度センサー、モーションセンサー、周囲光センサーを配置できます。 パラメータIは、デバイス内のチャネル番号です。 デバイスに含まれる物理デバイスが1つだけの場合、このパラメーターは0です。



ファイルの作成後、Z-Wayコマンドを再起動する必要があります。 

 /etc/init.d/Z-Way restart


ブラウザからリクエストを送信して、スクリプトを手動でロードします。

ブラウザから機能の機能を確認できます。

ライトをオンにします。

デザイン開発



すぐに設計開発に進みます。



1. [ UIデザイナー ]タブに移動します 新しいパネルを作成し、たとえばiPhone4という名前を付けます。





2.ボタンと画像を右ペインからiPhone画面にドラッグします。

制御コマンドとセンサーの作成



ここで、ボタンにコマンドを割り当てる必要があり、電球の画像はライトの状態（オン/オフ）に応じて変化するはずです。



1.タブに移動します 新しいデバイスを作成して、たとえばRaspberryなどの名前を付けます。





2.新しく作成したRaspberryデバイスを選択し、そのための新しいコマンドを作成します新規→ 新しいコマンド 。 HTTPプロトコルを選択し、 JSコマンドのURLを入力して、 POSTメソッドを設定します。

このコマンドには、デバイス番号6が含まれます。 同様に、他のデバイスをオンにしてコマンドをオフにするコマンドを作成します。 

 http://192.168.1.113:8083/JS/Run/SwitchOn(6,0)






3.次に、光の状態をポーリングするチームを作成する必要があります。 このコマンドは2秒ごとに呼び出されるため、ライトを手動でオフにすると、アプリケーションではすぐに認識できるようになります。 いつものようにNew→ 新しいコマンドですが、さらにポーリング調査を実行する頻度を指定する必要があります。2sを設定し、文字sが必要です。





4.このコマンドから受信した値を処理し、それらを画像またはラベルに送信するセンサーを作成します。 新規→ 新しいセンサー 、センサーが実行するコマンドを選択し、タイプをカスタムに設定し、 オンとオフの 2つの状態を追加し、返されたZ-Wave値に関連付けます。





ボタンとコマンドの関連付け



1.アプリケーション開発の最終段階、ボタンとコマンドの関連付け。 UI Designerアプリデザイナーに戻る 、ボタンを選択し、プロパティで実行するコマンドを設定します。





2.デバイスの状態が変化すると、アプリケーションの画像も変化するように、画像をセンサーに関連付けます。 画像を選択し、そのプロパティでセンサーを設定します。





3.画像のプロパティで、センサーが送信された場合に表示する画像と、センサーが送信した場合に表示する画像を選択します。





モバイルアプリの同期



1.アプリケーションの準備ができました。電話に入力するだけです。 OpenRemoteコントローラーhttp：// OpenRemoteServer：8080 / controller /のアドレスに移動し、[ オンラインデザイナーと同期 ]をクリックして、作成したアプリケーションをOpenRemoteコントローラーにダウンロードします。





2.アドレスhttp：// OpenRemoteServer：8080 / webconsole /で 、完成したアプリケーションを確認できます。





3.モバイルでOpenRemoteアプリケーションを開き、OpenRemoteサーバーのアドレスを設定し、 UIデザイナーで作成したiPhone4パネルを選択します。





4.これで、モバイルでアプリケーションをテストできるようになりました。さらに開発を続け、温度センサー、窓開きセンサーを追加し、XBMCメディアプレーヤーを制御するパネルを作成しました。
