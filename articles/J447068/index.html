同様の文字列を比較する

同様の文字列を比較するタスクは、実際には頻繁に発生します。最近、あるプログラムから別のプログラムに電子メールアドレスをインポートしようとしたときに、私は個人的にそれに遭遇しました。



たとえば、1つの住所は「トヴェルスカヤ州、カシンg、ソヴェツカヤul、1、5」のように見え、別の住所は「トヴェルカ州、 カシン市。 ソヴェツカヤ通り; 家1; アパート5 "。 これらの行は似ていますか？ 確かに似ています。 そして、「裸の目」で彼らの構造が見える：地域-決済-通り-家-アパート。 アドレスの場合、行をグループに分割することは重要です。 つまり、類似した単語の「2つのポリッジ」を比較するべきではありません（1つの「ポリッジ」は最初の行の単語と2番目の単語の2番目の単語で構成されます）。 グループに分割するための基準も調べます。最初の行ではグループの区切り記号は「、」で、2番目の行では「-」です。 「。



同時に、明示的なグループ化が表示されない行があります。 たとえば、「クラシック」を考えてみましょう：「同志に合意がない場合、彼らの仕事はうまくいかず、うまくいきません-小麦粉だけです。」そして、2行目：「いたずらな猿、ロバ、ヤギ、不器用なクマがカルテットを演奏し始めました。 »明らかに、線は異なります（類似点はありますが、これらのf話のモラルさえ異なります）。



問題のタスクは新しいものではありません。 それを解決しようとするアルゴリズム（非常に複雑な場合もあります）があり、時にはそれをうまく解決することさえあります。 もう1つのアルゴリズムボックスを提案します。 それをコンパイルするとき、私は次の原則から進みました。



  • 比較関数の呼び出しの単純さ。
  • 実装の容易さ。
  • 十分な汎用性。


さらに、アルゴリズムはVBA Excelに実装されているため、非常に「民主的」であり、どこでも使用できます。Excelはさまざまなコンピューターのソフトウェア間に「単独で」存在するだけでなく、さまざまなDBMSやアプリケーションからデータをエクスポートします。



それでは始めましょう。



比較関数はStrCompareと呼ばれます。 4つの引数があり、そのうち2つはオプションです。最初の行str1、2番目の行str2、最初の行div1のグループ区切り文字、2番目の行div2のグループ区切り文字。 div1またはdiv2が省略された場合、デフォルトの区切り文字は「|」です。 「|」が選択されたのは、「平均」行で発生する可能性が低いため、モノリシック（グループ化できない）行の比較に使用できるためです。 このようなモノリシック文字列は、1つのグループで構成される文字列と見なすこともできます。 つまり、比較関数のヘッダーは次のようになります。 



Public Function StrCompare(str1 As String, str2 As String, Optional div1 As String = "|", Optional div2 As String = "|") As Single


単一-関数の結果は、比較される文字列の類似度を示す数値であるため。



行1のすべてのグループは、行2のすべてのグループと単語ごとに順次比較され、グループの各ペアで一致する単語の数が考慮されます。 行1の各グループに対して、行2の「最適なグループ」が最終的に選択されます（つまり、最も一致するグループ）。 単語の各ペアの一致は、最小の長さの単語（つまり、「street = street」および「g = city」）でチェックされます。 このルールは、数字には適用されません：200 <> 20。 単語を強調表示する場合、グループ内のすべての「重要でない文字」は単なる単語区切りですが、それら自体は無視されます。つまり、単語は文字WordSymbols = "0123456789ABBGDEJEZLKLMNOPRSTUFKHCHSHCHYSYEYABCDEFGHIJKLMNOPYRXUVXWVXXのみで構成できます。 ケースは考慮されないことが理解されます。



2行目の現在のグループで一致する単語を検索するには、高速半除算法が使用されます（ただし、上記の方法を使用して一致がチェックされるため、「クラシック」に比べてわずかに近代化されます）。 また、半除算法の操作にはソートされた配列が必要なため、高速ソートアルゴリズムも使用されます。



StrCompare関数の結果は、一致する単語の数を1行目と2行目の合計単語数で割った結果になります。
