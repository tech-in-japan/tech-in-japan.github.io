Linux Mint 17“ Qiana”





ご存知のように、 Linux Mint 17はUbuntuをベースとしたオープンOSであり、使いやすさ、論理インターフェイス、ソフトウェア、コーデック、ドライバー、およびその他必要なものすべてのインストールのしやすさを重視する一般ユーザーほどプロ向けに設計されていません。



現在、Linux Mint 17のリリース候補「Qiana」は、開発者によると「ほぼ安定」と見なすことができ、現在ダウンロード可能です。 Linux Mint 17“ Qiana”は2019年まで公式サポートを受けます。 リリースは6か月ごとに更新されます。 リリース候補には、 MATEまたはCinnamonの 2つのエディションが用意されています。



公式の安定版リリースについては、少し後（5月末）に利用可能になります。



このリリースの新機能



更新マネージャー







このマネージャーは非常によく更新されており、マネージャーウィンドウに各更新に関する詳細情報が表示され、より高速に動作し、ルートモードが使用されている場合は再起動する必要がありません。



インターフェースが改善され、アイコンが更新され、各更新の変更リストの取得が以前のバージョンよりも速くなりました。



新しい「タイプ」列があります。これは、定期的な更新、セキュリティ更新、または他の種類の更新であるかどうかを示すために作成されます。



更新マネージャーに新しいセクションが表示され、使用可能なすべてのカーネルバージョンが表示されます。







ドライバーマネージャー



これで、ネットワークに接続せずにドライバーをインストールできます。 インターネットに接続されていない場合、システムはいずれかのメディアにインストールされたドライバーへのパスを示すように求めます。



さらに、Broadcomワイヤレスチップのサポートが改善され、ネットワークケーブルを介した直接接続を省略できるようになりました。



ログイン画面



複数のモニターのサポートが改善されました。たとえば、デフォルトで使用するモニターを自分で指定できます。



バックグラウンドアニメーションの使用リソースが少なくなります。



回復コマンドが登場しました。 たとえば、システムにログインできない場合は、CTRL + ALT + F1の組み合わせを使用してコンソールを起動し、「mdm-recovery」コマンドを使用します。



言語設定







このツールも更新されており、Cinnamon、MATE、Xfceを含むすべてのエディションで唯一の言語管理センターです。



ソフトウェアソース



インターフェースの改善、いくつかの新しい設定の追加。



ようこそ画面







開発者は、ようこそ画面のデザインを再設計しました。これにより、リソースの使用量が減り、起動がより速くなりました。
