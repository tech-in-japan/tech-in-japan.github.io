Qtについて少し話す機会をありがとう。 これはC ++フレームワークです。 その原理と哲学は、標準ライブラリのなじみのあるものとは少し異なります。 [ 聴衆への質問 ]情報を得るためだけに、Qtを知っていて以前にQtを使用したことがある人は何人ですか わあ！ ありがとう、すごい！ これは素晴らしいです！ それから、あなたがおそらくすでに知っていることについて話しましょう。







Qtとは何ですか？



おそらくこれが最も包括的なC ++フレームワークです。 しかし、これはフレームワークであるだけでなく、組み込みデバイスの開発を支援するさまざまなツールであり、C ++とは関係なく、統合されたQt Quick UIテクノロジーです。







2つのライセンス：商用およびLGPLv3 / GPLv3。 これはオープンソースプロジェクトです。







2つの製品：







  • Qt for Application Development（デスクトップおよびモバイルプラットフォーム：Android、Linux、Windows、Mac、iOS）
  • Qt for Device Creation（組み込みプラットフォーム：Android、Linux、Windows、QNX、VxWorks、Green Hills Software（IOT、Internet of Things））


アプリケーション開発のためのQt









コアには統合ライブラリがあります。 最も重要なのはコアです。これには、メインクラス、I / O、ファイル処理などが含まれます。 GUIライブラリは、グラフィカルインターフェイスの基礎です。 そして他の多く。







デバイス作成のためのQt









Qtクリエーター



デスクトップ、モバイル、組み込みシステム向けのクロスプラットフォーム開発環境。 多くの人がこれを使用して、Qtなしで純粋なC ++でプログラミングします。 さまざまなツール（プロファイラー、デバッガー...）、オートコンプリート、VCSの統合をサポートしています...









Qt CreatorおよびCLang静的アナライザーによる早期エラー検出







物語



26年前に早送りします。 Haarvard NordとEirik Chambe-Engの学生は、新しいプロジェクトについて考えました。 当時、彼らはUnix、Macintosh、およびWindowsで動作するシステム用のGUIを開発していました。 大変でした。 1990年の夏、彼らはオブジェクト指向のマッピングシステムが必要であると判断しました。







1993年に、最初のグラフィックコアが開発されました。













このバージョンはクエーサーと呼ばれていました。 その後、名前空間はありませんでした。 Motif（インターフェイス要素のライブラリと、X Window Systemのグラフィカルインターフェイスを開発するための一連の仕様）には、独自のプレフィックスがありました。 そのため、名前空間として機能する接頭辞「Q」が追加されました。 現在、そのバージョンから多くが保存されています。 残りのQObject-シグナル/スロットメカニズムを実装する中心部分。 また、QEvent、QFont、QPainter、QColor。







1994年、彼らは妻たちに数年間彼らを財政的に支援するよう説得し、Trolltech社を設立しました。









Trolltech Webサイトの初期バージョン







プロジェクトはまだかなり粗雑でした。 さらに、当時のインターネットを介したソフトウェアの販売は一般的ではなく、物理メディアを好んでいました。 したがって、お金はほとんどありませんでした。







1996：Qt 1.0



テンプレートなしで完全に機能しました。 そのため、ポインター、整数などの特殊なリストがいくつかありました。Windows95 / NTおよびUNIX / X11がサポートされていました。 UNIXバージョンはオープンソースでした。小さな会社が何らかの形で製品を配布する必要があったためです。







新しいプロジェクト：Kool Desktop Environment（KDE）



1997年、Matthias Ettrich（KDEプロジェクトへの貢献で知られている）は、Qtを使用してKDEを作成することを決定しました。 おそらく、これがQtが「離陸した」理由でした。













1999：Qt 2.0



GPLの下でリリースされました。 Unicodeサポートが登場しました。 プロジェクトにはベンチャーキャピタルが提供されました。 年間の従業員数は5人から50人に増加しました。







基本的なウィジェットと、「柔軟な」インターフェイスを提供するレイヤーが開発されました。 しかし、人々はこのためのコードを書きたくありませんでした。 その後、Qt Designerが登場しました。









最初のバージョン







この時点で、私はTrolltechで働き始めました [ そして、著者は代名詞「私たち」をさらに使い始めました ]







2001：Qt 3



文字列の明示的な共有を暗黙的な共有に置き換えました。 人々はたくさんの弦の問題を抱えていました。 次に、暗黙的な共有を残すことにしましたが、行が変更されたときに新しい内部インスタンスを作成します（コピーオンライト）[ 以下で詳しく説明します ]。







より多くのテンプレートC ++ 98を使用し始めました。 qobject_cast <>（QObjectのdynamic_castのアナログですが、RTTIを使用していません）があります。 Mac用のオープンソース。







2005：Qt 4



テンプレートとヘルパークラスを徹底的にレビューしました。 コンテナーへの値へのポインターの保存から、値自体の保存に移行しました。 暗黙的な共有コンテナは、アトミックな参照カウントを使用し始めました。 Windows用のオープンソース。 Windowsの場合、収入と購入者が生き残るためにこれが最後でした。







ノキアがTrolltechを買収



Nokiaは、プラットフォーム（S40、Symbian＆Maemo / MeeGo）の共通APIに興味がありました。 Qtへの巨額の投資を受けました。 長い間、クロスプラットフォームIDEの開発を望んでいましたが、資金はありませんでした。 マニュアルではEclipseを推奨していますが、C ++にはあまり適していません。 ノキアによって、私たちの欲望が可能になり、Qt Creatorが作成されました。 また、Qt Quick。 また、WebKitエンジンの統合に取り組み始めました。 NokiaはライセンスをGPLからLGPLに変更し、フレームワークをさらに無料にしました。







2012年に、ノキアと別れました。

DigiaはQtを買収しました。







新しいプラットフォームへの移植は困難だったため、抽象化レイヤーを追加しました。 私たちはグラフィカルインターフェースで良い仕事をしました;それは静的でなくなった。 例えば、これは移動中にDock Widgetウィンドウに表示されます。
