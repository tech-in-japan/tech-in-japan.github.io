Ubuntu 9.10でのリポジトリの追加について

Linux Format 11-2009（124）を読み始め、記事「Ubuntu：すべてをインストールする」の前に読みましたが、リポジトリを追加する方法に非常に驚きました。



クロムブラウザをインストールするには、著者が推奨します

3つの手順を実行します。



a）GUIで「ソフトウェアソース」を開き、新しいリポジトリを追加します。 

  http://ppa.launchpad.net/chromium-daily/ppa/ubuntu




b）ターミナルで実行：

sudo apt-key adv --recv-keys --keyserver

keyserver.ubuntu.com 4E5E17B5







c）ターミナルで実行：

sudo apt-get update

sudo apt-get install chromium-browser







このアプローチの何が気に入りませんでしたか？」



1.「GUIでこれを行い、コンソールで行います。」

「ソフトウェアソース」を介してキーを追加し、ソフトウェアをインストールする方法を知っている

シナプスを使用して、著者がそれを信じているかどうか 

  echo "deb http://ppa.launchpad.net/chromium-daily/ppa/ubuntu jaunty main" >> /etc/apt/sources.list
主婦には複雑すぎる。 知りません



さらに、作者は、ジャントを新しいバージョンに変更することはできないと言っています。 そうではありません。







2.キーを手動で追加する必要があります。







今回は、作成者がキーとキーの追加方法の両方を報告するのは良いことです。







誰もがフォーラムで次の質問に繰り返し答えています。「キーを追加する方法」

多くの場合、次のような松葉杖を見つけたり、使用したりしています。



#!/bin/sh

KEYS=`sudo apt-get update | awk '/NO_PUBKEY/ {print($NF)}'`

if [ -z $KEYS ]

then

echo "No missed public keys found."

else

echo "Missed public keys: $KEYS"

for KEY in $KEYS

do

gpg --keyserver keyserver.ubuntu.com --recv $KEY &&\

gpg --export --armor $KEY | sudo apt-key add -- &&\

echo "Public key $KEY has been added."

done

fi







（松葉杖と苦痛の両方の例はこちら ）



ubuntu開発者はバージョン9.10で何を提供しますか？



リポジトリとキーの追加を組み合わせて自動化します。

sudo add-apt-repository ppa:chromium-daily/ppa

Executing: gpg --ignore-time-conflict --no-options --no-default-keyring

--secret-keyring /etc/apt/secring.gpg

--trustdb-name /etc/apt/trustdb.gpg --keyring /etc/apt/trusted.gpg

--keyserver keyserver.ubuntu.com --recv

FBEF0D696DE1C72BA5A835FE5A9BF3BB4E5E17B5

gpg: 4E5E17B5 hkp keyserver.ubuntu.com

gpg: 4E5E17B5: "Launchpad PPA for chromium-daily"

gpg: : 1

gpg: : 1











それだけです！ システム自体がリポジトリを追加し、そのキーを見つけてインポートします： 



 $ ls /etc/apt/sources.list.d
 「chromium-daily-ppa-karmic.list」「chromium-daily-ppa-karmic.list.save」


パッケージインデックスを更新し、クロムを配置するだけです。 ただ？
