Veeam Backup＆Replication v8の新機能：Active DirectoryおよびMicrosoft SQL Server用のVeeam Explorer

Veeam Backup＆Replicationの8番目のバージョンのリリースは間近です。 前に、NetAppハードウェアスナップショットのサポートについて説明しましたが、Veeam Explorerアプリケーション用の詳細なリカバリツールのラインアップの拡張について説明したいと思います。 Veeam Explorer for ExchangeおよびVeeam Explorer for SharePointに加えて、ADのVeeam ExplorerおよびMicrosoft SQL ServerのVeeam Explorerが含まれます。









彼らが何ができるかについての情報はカットの下にあります。















アプリケーションの仮想化により、企業はITインフラストラクチャを使用してコストを削減し、技術的に複雑な運用を簡素化できます。 このシンプルさは、アプリケーションモビリティ、自動化されたリソース割り当て、サービスプロセスの自動化に適用され、特殊なソフトウェア製品とハードウェアのコストなしでインフラストラクチャの可用性を向上させます。









また、仮想化により、データ保護、重大な障害が発生した場合の復旧、仮想環境全体の監視と管理など、社内のITプロセスを整理するいくつかの側面を再考する必要があります。 ビジネスプロセスでは、多くの場合、24時間年中無休のアプリケーション可用性、効果的なデータ成長管理、および最小限のダウンタイムの欠如が必要です。 Veeamは、仮想マシン（VM）の「適切な」バックアップ、監視、複製を通じてこれを管理するのに役立ちます。









2年前、Veeamは2つの無料ツールをリリースしました。









  • Veeam Explorer for Microsoft Exchangeは、バックアップ内のMicrosoft Exchangeデータベース（.EDB）の内部を確認できるツールです。 これは、管理者がExchangeサーバーを完全に復元する必要なしに、Exchangeデータベースから直接運用環境にオブジェクトの詳細な復元を実行できることを意味します（通常、ドメインコントローラーの復元も必要です）。











  • Veeam Explorer for Microsoft SharePoint （Habréで既に説明しました） -同様のアルゴリズムを使用して、サーバーのSharePointデータベースからオブジェクトをきめ細かく復元できます。






Veeam Backup＆Replication v8の新しいバージョンには、既存のツールの機能を拡張するいくつかの新しい機能が追加されています。 たとえば、Exchangeエクスプローラーには、 「 SHIFT-DELを介して」「復元の可能性なし」で削除されたメッセージを復元する機能 、Microsoft Exchange Online Archivingのサポート、および復元操作用のWebポータルがあります。









Veeam Backup＆Replicationバージョン8には、Microsoft Active Directory 用のVeeam ExplorerとMicrosoft SQL Server用のVeeam Explorerの 2つの新しいツールが導入されています。 どちらのツールでも、管理者は、バックアップから仮想マシンを完全に復元する必要なく、ADおよびSQLからオブジェクトの詳細なリカバリを実行できます。









Microsoft Active Directory用のVeeam Explorer







Veeam Explorer for ADを使用すると、コンテナーやユーザーアカウントの復元など、さまざまなADオブジェクトのきめ細かい回復が可能になります。









別の便利な機能：Explorer for Active Directoryには、パスワードを保存してユーザーアカウントを復元する機能があります。 この問題は、多数のユーザーのアカウントを同時に復元する必要がある大規模な障害の場合に関連します。 パスワードが復元されていない場合、システムに入るとすぐにWindowsはユーザーに新しいパスワードの設定を要求しますが、統計によると、この状況に対応できないユーザーが常に存在し、HelpDeskで大量の呼び出しが発生します。 Veeam Explorer for ADにより、システム管理者はこの問題を回避できます。









Microsoft SQL Server用のVeeam Explorer







Veeam Explorer for Microsoft SQL Serverは、SQLデータベースまたはその個々のオブジェクトを復元するためのいくつかの興味深いオプションを提供し、通常どおり、バックアップから仮想マシンを解凍する必要はありません。









データベースが不正な（データベースの整合性ではなく、予想される結果の意味で）トランザクションに苦しんだ場合、Veeam Explorerを使用して、「トランザクションの直前」の時点に戻ることができます。 これは、元に戻す機能に非常によく似ており、テスト環境または実稼働環境で役立つ場合があります。









Veeam Explorer for Microsoft Exchangeと同様に、Veeam Explorer for Microsoft SQL Serverは、グラフィカルインターフェイスに加えて、リカバリ操作用のWebベースのインターフェイスを管理者に提供します。
