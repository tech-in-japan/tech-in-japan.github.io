Nokia 5800 XM用のIMクライアントの選択

こんにちは、Khabrovites。 今日は、Nokia 5800 Xpress Musicスマートフォン用のIMネットワークの最も人気のあるクライアントを紹介したいと思います。



それでは、今日は何をテストするのでしょうか？

  • QIP PDA Symbianビルド2000
  • スリックv0.47
  • IMplus 8.05
  • Nimbuzz 2.0
  • Mobile Agent 1.16ベータ
  • Fring v.3.37.20




QIP PDA Symbianビルド2000





新しいQIPアセンブリでは何も起こりませんでした。タッチをまったく認識せず、残念ながらひどく見えます。



スリックv0.47









クライアントはおそらく悪くありません。 その主な利点は、新しいメッセージに関する通知の表示、便利なタブ切り替えシステム、およびシンプルなインターフェースです。 一方、ロシア語のネイティブサポートはありません。個人の調整の機会はほとんどなく、連絡先を追加するための非常にバグの多いシステムです。



IMplus 8.05









マルチプラットフォーム、優れたカスタマイズオプション、習得しやすい-IM +クライアント。 しかし、短所について。 登録の必要性（正直なところ、彼らは私たちに欲しいものを理解していませんでした-購入、または単に開発会社のウェブサイトで登録）、ロシア語の欠如。



Nimbuzz 2.0









最も興味深い顧客。 SIP-テレフォニーなど、膨大な数の異なるサービスをサポートしています。 美しく、便利で、設定が簡単で、ロシア語をサポートし、すぐに動作します。 ただし、マイナスの点として、エンコーディングに問題があります（その場合-jimmクライアントを使用している対話者が原因である可能性が高いため、ほとんどの場合問題があります）。



Mobile Agent 1.16ベータ









オーバーロードされた、理解できない、混oticとしたクライアント。 スキンをサポートし、そこからメールを読むことができます（ハハ）、マルチプロトコル（ただし、サービスのセットはICQ、MRA、Jabberに限定されます）。 同時に、メッセージを送信するのではなく、ひどく遅くなり、飛び立ちます。



Fring v.3.37.20









多くのプロトコルをサポートする優れたクライアント。 last.fmを削ることができ、twitterをサポートします。 残念ながら、fringは連絡先をグループに分割する方法を知らないため、特に複数のプロトコルを同時に使用している場合は特に混乱しやすくなります。 時々クラッシュしますが、これはほとんどすべての顧客にとって問題です-プラットフォームは新しく、多くはまだベータテスト中です。



その結果、Nimbuzz 2.0に決めました。
