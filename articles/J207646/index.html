WDSおよびWindows 7の無人インストール

良い日、紳士は良いです！ Habrを長い間読み取り専用モードで読んでいたので、ここに記事を書くことにしました。 解決策は誰かの観点からは完全に正しいとは限らないことを理解していますが、それは私が望むようにすべてうまくいきます。 私は教育機関（GBOU SPO）でシステム管理者として働いています。 150台の車の下位艦隊、さまざまな構成。 その前に、私はすでにWindows XPのイメージを作成していました。 そして今、私は次のタスクを設定しました。Win7OSの統一されたイメージを作成することです。 そして、これが私がこのタスクに対処した方法です。



開始する



次のものがあります。

-さまざまなドライバを備えたx86アーキテクチャマシンのフリート。

-事前に構成されたWDSを備えたサーバー（別名ドメインコントローラー）。

-ソフトウェアの束。

-新しい更新をダウンロードするためのオフライン更新。

-ローカルドライバーリポジトリをスプーフィングするためのsysprepドライバースキャナー。



Runetには多くの優れた機能があるため、WDSのインストールと構成については説明しません。 ここにそのうちの1つがあります。

OSのインストールから始めましょう。この場合、Windows 7 Enterpriseがあります。 完了後、システムをソフトウェアで満たします。 私の場合、すべてが少し簡単です。 企業はApp-Vサーバーを実行しています。 そして、必要なパッケージはすべて、最初の起動時にそこから取り出されます。 最終イメージには、重要なプログラムがいくつかあります。



更新の生成とインストール


このためには、 このプログラムが必要です。 解凍後、UpdateGenerator.exeを実行し、「Windows 7 / Server 2008r2 x86グローバル」および「選択した製品ごとにIsoイメージを作成」にDawを配置し、「開始！」をクリックします。







isoフォルダーで作業を完了すると、更新されたイメージが見つかります。 システムにマウントしてインストールします。 このプロセスは非常に長いことに注意してください。 この方法でアップデートをインストールする場合-win7は一時ユーザーを作成します。 システムの最初の再起動後、このアカウントがログインします。 そして、これらすべてに再び気を取られないように、任意のフォルダーにイメージを抽出し、UpdateInstaller.exeを実行します。 [自動再起動と再呼び出し]ボックスをオンにします。 「開始」ボタンを押すと、安全にお茶を飲みに行くことができます。



ドライバーでのフォルダーの置換


すべてが非常に簡単です。 driverpacks.net Webサイトから、必要なドライバーパッケージをダウンロードします。 2つのことができます。 全体として必要なパッケージを選択するか、ニーズに合わせて特定のドライバーを選択します。 発行価格は、最終画像のサイズです。

ドライバーを入手したら、ディスク上にフォルダーを作成し、そこに解凍します。 次に、 これをダウンロードして実行します。 次に-フォルダへのパスをfireで処方します。 「スキャン」、「デフォルト」、「保存」、「完了」を1つずつクリックします。 ドライバーをインストールするときに、すべてがこのディレクトリで検索されます。 純粋に理論的には-ドライバーをスキャンするためにいくつかのポイントを置くことができます。 つまり、ローカルで、NIC上のドライバーを探し、他のすべてがネットワークボールから伸びます。 しかし、私はそれを試していませんでした。







プロファイルの封印と削除


応答ファイルを作成するため、OSを封印する最善の方法は次のとおりです。 sysprep.exeを実行するときに、「監査モードに切り替える」を選択します。 再起動後、システムのインストールを実行したプロファイルを削除し、C：\ Usersフォルダーの末尾も確認します。

次に、同じ方法でsysprepを呼び出しますが、「システムのようこそウィンドウ（OOBE）に切り替える」と「シャットダウン」を選択します。 作業が完了すると、PCの電源が切れ、画像をキャプチャする準備が整います。



応答ファイルを作成する


ここで、ソースへのリンクを提供することをお勧めします。 インストールイメージとOSイメージ自体の両方の応答ファイルを作成するプロセスについて説明します。
