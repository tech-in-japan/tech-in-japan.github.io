データ収集端末でのWindows MobileとWindows CEの違い

今日、Windows上のデータ収集端末の市場は急速に成長を続けています。 これは、そのようなデバイスが倉庫および物流活動の自動化に需要があるためです。 Windows TSDは、さまざまな問題をプログラミングして解決する絶好の機会を提供します。



ただし、デバイスには別の重要な違いがあります-システムのタイプです。 Windowsオペレーティングシステムは、Windows MobileとWindows CEに分かれています。 この記事では、これらのオペレーティングシステムの類似点と相違点について説明し、特定のバージョンを選択することの妥当性を検討します。





市場の概要



Windowsで最も人気のあるデータ収集端末の中で、CipherLab、Motorola、Honeywellのブランド名で多くのモデルを区別できます。 今日、さまざまな目的のためのモデルが積極的に生産され、コンパクトなケースで、または逆に、倉庫のニーズと高度な保護（IP 65、IP66、IP67など）のための大規模な設計で作られています。 概して、その主なタスクは多くの点で似ています：会計の折りたたみ、在庫管理、会計プログラムの使用、商品のピッキングと出荷。



DOSターミナルを超える利点



  • 充填と技術的な複雑さ 。

    単純なクラスの機器とは対照的に、 Windowsのデータ収集端末 -DOS端末には、写真とビデオの統合カメラ、ボイスレコーダー、あらゆる種類のワイヤレスインターフェイスなど、より複雑な技術的な詰め物とオプションがあります。
  • RDPで作業します 。

    Windows端末には、高解像度の大型カラーディスプレイが装備されており、RDP（リモートデスクトッププロトコル）での作業に非常に便利です。 これにより、コンピューターでタスクを実行し、TSD画面でプロセスを管理して、リソースを節約し、生産性とモビリティを向上させることができます。

    Dos端末は、技術的特性が不十分であるため、このような機会を奪われています。
  • .Net Compact Frameworkプロトコルをサポートします。

    .NET Compact Frameworkは、データ収集端末またはその他のモバイルデバイスでアプリケーションを実行するためのコンポーネントのセットです。

    他のブランドと比較したCipherLab端末の利点の1つは、TSDの動作を構成するよく考えられたアプリケーションジェネレーター（付属）です。 TSDでの作業を開始する前に、倉庫の従業員はコンピューター上の多数のアルゴリズム（バーコードの詳細情報、製品グループごとの並べ替えなど）を特定し、デバイスをPCと同期して、プログラムされたすべての条件とデータ収集アルゴリズムを転送できます。 プログラミングはロジックの使用に基づいており、特別な知識は必要ありません。 Windows CEとWindows Mobileのライブラリは異なるため、アプリケーションジェネレーターも異なります。












Windows Embeddedハンドヘルド



Microsoft Windows EmbeddedオペレーティングシステムはWindows Mobileプラットフォーム上で設計されており、データ収集端末を含むさまざまなデバイスの組み込みOSファミリに属します。 これまでの最新バージョンは、 Windows Embedded Handheld 6.5です （2010年に開発され、2011年に正式に導入されました）。 その汎用性と柔軟なアーキテクチャにより、Microsoft Windows EmbeddedはPOS端末、モバイル、組み込み、ポータブルデバイスで使用できます。



このOSの必須要件は、プロセッサ、RAMなどの優れた強力なハードウェアの可用性です。 このようなシステムを使用すると、システム要件が増加し、その結果、デバイス自体のコストが増加します。

Microsoft Windows Embedded OSで収集されるデータ収集端末には、 Cipher 9200 、Cipher CP30、Cipher 9600などがあり、これらのデバイスにはすべてPC通信ソフトウェアがあり、1CとWindows Mobileをサポートするアプリケーションジェネレーターで動作します。



Windows CE



Windows CEは、1996年にWin95のコンパクトバージョンとして市場に登場しました。 それ以来、多くの重要な変更が行われました。 2000年以降、このバージョンのWindowsは独立したOSとして開発および実装され始めました。 主な重点は、デスクトップPC用ではなくモバイルデバイス用のコンポーネントの作成に置かれ、 WIN32 APIを介したアプリケーション互換性がありました。



Windows CEとWindows Mobileは互換性のあるオペレーティングシステムではありません。 Windows CEの特徴的な機能は、さまざまなコンポーネント、プラットフォーム、マルチタスクをサポートするモジュラーシステムです。 Windows CEの利点は、このOSが作業に大きなリソースを必要とせず、最小限のメモリとシンプルなシングルコアプロセッサを備えたデバイス向けに設計されていることです（システムカーネルが動作するために最低限必要なのは32 KBだけです！）。



これまでの最新バージョンはWindows CE Compact 7.0です。 最近リリースされたWindows CEで最も人気のあるデータ収集端末の中で、Cipher 9700、Cipher CP55に注目する必要があります。 自動化のための機器のリリースの現在の傾向は、クライアントが会社の問題を解決するために必要なものを選択できるように、さまざまなバージョンの可用性を示唆しています。 そのため、いくつかのTSDモデルは、両方のプラットフォームで複数のバージョンで利用できます。 このようなデバイスの鮮明な例は、データ収集Cipher CP60の倉庫産業ターミナルと考えることができます。



要約すると、何を選択するのですか？



速度の点では、これら2つのシステムはほぼ完全に同一ですが、Windows CEの以前のプログラムとアプリケーションははるかに高速に動作しました。 今日、データ収集端末には強力な生産性の高いプロセッサ、大量のRAMが搭載されており、単純なユーザーの作業に大きな違いは感じられません。

一般に、Windows CE自体はアーキテクチャがシンプルで、クラシックなデスクトップインターフェイス（スタートボタンなど）を備えています。 このOSは、主に開発者にとって興味深いものです。



それでも、Windows Handheld Embedded 6.5.3の高い人気にもかかわらず、製造業者の観点からは、Windows CEオペレーティングシステムで最新のデータ収集端末を作成することは非常に良いことです。 これは、CEの古い端末用の多くのプログラムがあり、基礎を失いたくない場合に特に便利です。



OSの選択は、特定のシステムに既製のソリューションが存在することで説明できます。 ある会社が、古いTSD用のMobile SMARTSに基づいて、1C Enterpriseのデータ収集端末用のWi-Fi端末ドライバーを購入したとします。 使用中、従業員は独自の成果（プログラマ、ユーティリティ、プログラムなどからのユニークなリビジョンを持つデータベース）を持っていました。 そして今、数年後、機器を変更する時が来たとき、これらの開発をどのように維持し、安定したデバッグ作業を継続するかという疑問が生じます。 ソフトウェアとドライバー、およびWindows Handheld Embedded用の.NET Compact Frameworkライブラリは、Windows CE用に作成されたものと互換性がありません。 同時に、設備の設置とデバッグに多大な資金、時間、労働力が費やされました。



このような状況を回避するために、Windows Handheld Embeddedは非常に人気がありますが、製造業者は引き続きWindows CEでデータ収集端末を販売しています。



Windows CEで最も成功したTSDイノベーションの中で、CipherLab CP55、CipherLab 9700、CipherLab CP60に注目する必要があります。 彼らの人気は、現代の充填、長いバッテリー寿命、幅広いプログラミング機能、および悪条件で働く能力によるものです。
