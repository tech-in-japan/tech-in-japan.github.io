起源





1973年、モトローラは最初のモバイルデバイスを開発しました。 実際、20年後、最初のハンドヘルドコンピューター-Psion 3が誕生しました。正式には、以前は存在していました（特に、同じ年に最初のIBM Simonスマートフォンが登場しました）が、ノートブック、電卓、時計のみに限定されていました。



しかし、Psion 3には真の無限の機能がありました。EPOCオペレーティングシステムに基づいて、プログラミング言語OPL（Open Programming Language）が作成され、独自のアプリケーションを作成したい人なら誰でも使用できます。 後に、有名なSymbianの基盤を形成するのは彼です。





サイオン3。最初は公式ではないが、実際には



当時、サイオンは実際にはたった一人の競争相手であるパー​​ムパイロットしかいませんでした。 Psion 4が市場に参入することを許可しなかったのはこのデバイスでした。 そして、本当のタッチスクリーン、そして最も重要なことには、C / C ++でアプリケーションを作成する能力が原因です。 これがPalm OSプラットフォームにどれだけの利点をもたらしたかは容易に推測できます。





パームパイロット。 大きな感覚の未来への小さな一歩



当時のモバイルガジェットを操作するためのツールは次のとおりです。



  • WML（ワイヤレスマークアップ言語）は、インターネットの低速、RAMの不足、画面サイズ、および品質に起因する制限があるデバイス向けに設計されたマークアップ言語です。 本質的に、これは同じHTMLですが、タグのペアリングの要件がより厳しく、機能が大幅に削減されています。 Palm Pilotで初めて使用されました。
  • J 2ME / JMEは、WMLと同様に、Javaの簡易バージョンです。 2000年代初期に「深刻な」携帯電話を持っていたほとんどの人は、J2MEで書かれたアプリケーションを見つけることに直面していました。
  • J2MEは、主にCDC（接続されたデバイスの構成）とCLDC（接続された制限されたデバイスの構成）の2つの構成で構成されています。 1つ目は、コミュニケーターやハンドヘルドコンピューターなどの比較的強力で生産的なデバイスを対象としています。 2つ目は、大きな制限があるものの、幅広いユーザーに注目した安価なガジェット用です。 特に携帯電話では、MIDP（Mobile Information Device Profile）モードが使用されました。これにより、インターフェイス、アプリケーション、ネットワークの調整された動作が保証され、すべての設定が保存されます。 彼のおかげで、モバイル2Dゲームの素晴らしさと最初のカラフルなアプリケーションを学びました。




開発





モバイルアプリケーションの開発における重要な時期は、現代のアプリケーションを想定しており、Symbianシステムの存在とほぼ完全に関連しています。 Symbianは、Psion、Ericsson、Motorola、Nokiaのコラボレーションの結果、EPOCプラットフォームから発展しました。 コラボレーションは非常に成功したため、2009年の終わりまでに、2億5,000万台を超えるデバイスがSymbianを搭載しました。



もちろん、システムの開発への主な貢献はノキアによってなされました。 Symbian S60は、リーダーシップの下で、最新のものに匹敵するパワーと充実度を備えたプラットフォームに成長しました。 サムスンとLGスマートフォンもS60で動作したという事実にもかかわらず、優位性は均一に発展しませんでした。



そのため、Sony EricssonとMotorolaはSymbian UIQのサポーターでした。SymbianUIQは、タッチデバイスを操作するための最適化など、名目上広い機能を持つプラットフォームです。 さらに、Symbian S40、S80、S90のバリエーションがあり、それらはすべて実質的に互換性がなかったため、モバイル企業間の闘争が激化し、ユーザーは1つのブランドの携帯電話だけのフォロワーになりました。





これは、2000年代の携帯電話の外観です。 スタイリッシュで厳格



同時に、S60に基づいたフィンランドのメーカーは、ゲームやアプリケーションを開発するための追加プラットフォーム（N-Gageなど）を定期的に作成しました。 2003年の最初の試みは失敗しましたが、2008年の2番目の試みはマルチメディアデバイスの全ラインナップのリリースであり、これが最後の官能前の古典となりました。



現代の世界





Apple iPhoneの世界の出現と、Symbianのレガシーを手放そうとするNokiaは、iOS、Android、Windows Phone、BlackBerry OSといった新しい戦争グループのプラットフォームを事前に決定しました。



公式には、iOSシステム（2010年およびiPadの発売がiPhone OSと呼ばれるまで）は2008年3月にのみ誕生しましたが、実際は2007年の初めから存在しました。Appleは最初の1年半の間、サードパーティの開発者からプラットフォームを慎重に保護しましたスマートフォンが切り捨てられたデスクトップオペレーティングシステムで動作すること。



2008年には、アプリケーション開発環境のベータ版であるSDK（ソフトウェア開発キット）がリリースされました。 現在、デバイスの物理機能とソフトウェア機能を使用するための標準ツールに加えて、パッケージにはXcodeとiPhone Simulatorが含まれています。





そのため、今日では、「任意の」スマートフォンが慣習的に指定されています



Androidの開発は2005年に始まりました。そのとき、Googleは若く野心的な会社Android Incを買収しました。 どのプラットフォームで2年間取り組んでいたのかは不明ですが、GoogleでiPhoneの販売を開始してから10か月後に、Androidモバイルシステムの発売、Open Handset Alliance（OHL）の創設、それをサポートおよび開発するアライアンス、およびAndroidデベロッパーのアーリールック「SDK。



Androidは、LinuxカーネルとJava仮想マシンに基づいています。 Googleは、Android Native Development Kit（多くの言語からのライブラリとコンポーネントの移植）からOpenGL ES（3次元グラフィックスでの作業）まで、アプリケーションを作成するために、関心のあるすべての開発者に迅速に開発し、実質的に無限の可能性を提供することができました。 さらに、2007年11月の発売初日から、Googleは数百万ドルの賞金を伴う最高のアプリケーションのコンテストを開催しています。



Windows Mobileは、2000年以来、スマートフォンとPDAの市場で生き続け、繁栄してきました。10年の終わりまで、関連オペレーティングシステムを備えたタッチスクリーンスマートフォンの人気により、Microsoftは適切な答えを作成する必要がありませんでした。



そのアイデアは、「最高の兄弟」から最高のものを選び、それをまだ信頼できるノキアの携帯電話に結び付け、それによって開発者を新しいプラットフォームの開発に引き付けることでした。 Visual Studio Expressはこの目的を果たし、狭い範囲のアプリケーションとクロスプラットフォームアプリケーションの両方を作成できるようにしました。 WPは、開発者にWindows Bridge（サードパーティアプリケーションの移植）、Expression Blend（Webデザイン）、XNA（ゲーム）、Silverlight（インターネットアプリケーション）のツールも提供しました。 ただし、最後の2つのフレームワークは、サポートの欠如とクロスプラットフォームのWindows 10システムのため、もはや関係ありません。





Windows Phoneの優れたハードウェアおよびソフトウェア機能は、決して心を奪われませんでした。



BlackBerry OSは、おそらく最も開発が遅れており、永遠の忘却のモバイルオペレーティングシステムに最も近いでしょう。 最初のバージョンは2009年にリリースされましたが、わずか5年後には本当に安定して機能するようになりました。 このシステムは、使いやすさ（サムネイル、BBハブ、ジェスチャーコントロールなどの多くの興味深い発見が後にAndroidおよびiOSで「導入された」）および企業セキュリティを重視しています。



しかし、アプリケーションでは非常にタイトであることが判明しました。 今日開発ページにアクセスした場合、提案されたオプションの半分は、BBデバイスで実行できるAndroidアプリケーションを作成することです。 実際、バージョン10以降、Amazon App Storeがプリインストールされ、他のAndroidストアを選択できるようになります。



未来





メインプラットフォームとガジェット1の両方として、明日何が関連するかを推測することしかできません。 一方、今日の「大型」デバイスに関連するソリューションは、 WebデザインであろうとJavaアプリケーションの作成であろうと、モバイルにも関連することは明らかです。



同時に、ポケットデバイス（現在手と顔にある）のより深い導入は、タッチコントロールが明日、たとえば完全に視覚的な（手話）または音声に変更できることを意味します。



より単純な携帯電話プラットフォームに戻る可能性を無視しないでください。 特に、先日、Microsoftが非常に時代遅れでまだ機能しているSymbianを置き換える予算デバイス用の新しいオペレーティングシステムを開発したことがわかりました。
