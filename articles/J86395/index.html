WordPress 3.0で私たちを待っているもの（新機能の概要）

最も人気のあるWordPress 3.0ブログプラットフォームの次のバージョンは、サンフランシスコで開催されるWordCamp会議と同じ日に5月1日にリリースされます。 同時に、すべての新機能の最終バージョンが承認され、3月1日に「凍結」されました。 したがって、すべての新機能が知られるようになり、追加機能は追加されず、バグに対してのみ機能します。 WordPress 3.0で導入される最も基本的で重要な変更点について読むことをお勧めします。 そして、見るべきものがあります！



WordPress MUとWordPressの組み合わせ



WordPress MUが何であるかを知らない人にとっては、本質的に心配することはありません。 この開発はWordPressに似ており、ユーザーが同じシステム上に複数のサイトを作成できるようにします。 ところで、WordPress.comプロジェクトはWordPress MUに基づいています。 そのため、2009年のサンフランシスコのWordCampでの会議で、Matt Mullenwegがこの機能を発表し、1年の努力の末、これが達成されました。



これはユーザーにとって何を意味しますか？

1つの管理パネルで複数のサイトを作成できます。 たとえば、WordPressを追加インストールすることなくbeginners.wordpressinside.ruのようなものを作成できます。 結果はどうなりますか？ 別のブログがあり、さらにいくつか追加したい場合-問題ありません。何も変わりません。 WordPress MUを使用している場合、バージョン3.0にアップグレードした後、すべてのサイトもそのまま残り、変更は行われません。 つまり、すべては個人に依存します。新しい機能は一種のアドオン（追加）であり、1つのインストールで複数の個別のサイトを作成できます。 さらに、WordPress MUを使用するためにWordPressプラグインを「ハッキング」する必要がないため、WordPress MUユーザーの生活は少し楽になります。



ドメインの目的 -あなたがWordPress MUのユーザーであるか、WordPress.comの仕事に出くわした場合、ほとんどの場合、プロジェクトグリッド内のサイトでドメインを使用する方法を知っています。 WordPress.comサービスでは、ブログのサブドメインを通常の有料ドメインに置き換えることができますが、この機能はカーネルには含まれません。 それをサポートするには、追加のプラグインをインストールする必要があります。



サイトグリッドを検索する-WordPressインストールで複数のプロジェクトを作成する場合は、すべてのプロジェクトを検索することは組み込みの機能ではないことに注意してください。 おそらく将来的にはシステムのコアになりますが、WordPress 3.0には含まれません。 今のところ、適切なプラグインを使用する必要があります。







カスタム投稿タイプ



WordPress 2.9がリリースされたとき、この機能に関して多くの誇大宣伝がありました。 技術的な問題により、最終リリースには至りませんでしたが、バージョン3.0では正義が優先されます:)投稿タイプと分類法の組み合わせにより、WordPressはさらに強力なCMSになります。 このオプションを使用すると、いくつかの異なるタイプのブログ記事を作成できます。たとえば、「ポートフォリオ」、「製品」、および通常の出版物を1つのプロジェクトに結合できます。 この機能は、以前と同様に、使用が推奨されるカテゴリまたはタグに置き換わるものではないことを付け加えます。



サイトメニューの改善



いわゆるCustom Woo Navigationはシステムのコアに含まれ、メニュー管理がより便利になります。 ユーザーは、ウィジェットページに似た特別なページにメニュー項目をドラッグアンドドロップできます。 興味深いことに、同時に、ページ、カテゴリ、および単なるリンクを使用してメニューを作成できます。 それらはすべて交換でき、個々のアイテムを非表示にすることもできます。







この機能は、WordPressのコアへの素晴らしい追加であり、すべてのユーザーにとって非常に便利です。



新しいデフォルトのテンプレート



まとめて、Kubrickと呼ばれるデフォルトのWordPressテンプレートに別れを告げます。 WordPress 3.0は、2010年と呼ばれる新しいテーマに沿ったものになります（ 編集-しばらく前に、ブログ用のwordpress 3.0の可能性に関する同様の記事を準備していたときに、少し異なる名前-Kirbyを見つけました ）。 同時に、開発者は毎年、システムの標準テンプレートを更新するという目標を設定しました！ -私の意見では、これは非常にクールです。 ところで、2010年のトピックを見てみたいですか？







新しいテンプレートをライブで体験したい方は、 こちらをご覧ください 。 すっきりしたデザイン、ウィジェットのさまざまな領域、ドロップダウンメニューに加えて、ユーザーがブログのヘッダーと背景をカスタマイズする機能もあります。



カスタム背景



カスタムブログバックグラウンドサポートは、functions.phpファイルの特別な行を介して追加できます。



add_custom_background();







その直後に、管理パネルに追加のメニューが表示され、デフォルトで灰色の背景を置き換えるために、背景用に独自の画像をアップロードできます。 この関数は、2010テーマおよびそれをサポートする他のすべてのテーマで機能します。



著者テンプレート



WordPress 2.9には、デザイナーや開発者がURL（スラッグ）をcategory-slug.phpタイプとして使用して、カテゴリーのテンプレートを作成できるオプションがありました。 このファイルは、対応するカテゴリのテンプレートとして自動的に処理されました。 そのため、WordPress 3.0は作成者向けに同じものを作成できます。 新しいget_author_template（）関数がカーネルに追加されました。 ファイル名はauthor-michael.phpまたはauthor-123.phpのようにする必要があります。ニックネームはユーザーまたはユーザーIDで示されます。



インストール中にユーザー名を選択する機能



これで、WordPressはシステムのインストール時に最初のユーザーにニックネームadminを自動的に割り当てます。 この値を他の値に変更することが強く推奨されているセキュリティのヒントを含む多くの記事を考慮して、開発者はカーネルに新しいオプションを追加することにしました。 これで、ハッカーはパスワードだけでなくログインも推測するためにもう少し作業する必要があります。 もちろん、これはすべての潜在的なリスクからあなたを救うわけではありませんが、この機能は便利です。



各インストールのウェルカムガイド



WordPressには最も必要なものが欠けていることが多く、現在、開発者は状況を前向きに変えることを真剣に考えています。 このウェルカムガイドを使用すると、ユーザーはWordPressシステムをよりよく知ることができ、すべての新規ユーザーに特に役立ちます（ 編集者-個人的にはまだよくわからないが、どんな種類のウェルカムガイドなのか）。 ）



正規プラグイン



人気のあるプラグインやユーザーのお気に入りのプラグインについては、作成者からのサポートが終了する場合がよくあります。 作成者は更新の監視を中止し、エラーを修正しません。その結果、WordPressの新しいバージョンとの互換性が侵害されます。 これはかなり一般的な問題であるため、WordPress 3.0での作業に加えて、Jane Wellsが複数のモジュール作成者に正規のプラグインを作成するために協力するように勧めています。 このようなモジュールは、特定の開発者ではなく、コミュニティ全体（グループ）によって開発されます。これにより、誰かがチームを離れても、1つまたは別のプラグインが生き残ることができます。



一般的に、WordPress 3.0で注目される主な部分は、ユーザーとの距離を縮め、システムとのやり取りを改善および簡素化することです。 ブログのヘッダー、背景を変更したり、独自のメニューを作成する場合は、問題ありません。管理者設定を確認してください。 カスタム投稿タイプ、複数のサイトの組み合わせ、プラグインのサポート-これらはすべて、原則としてWordPressスペシャリストにとって非現実的なタスクではありませんが、普通のユーザーには便利でしょう。 開発者は、システムリソースの最適化、速度、および消費の方向で、システムのさらなる作業を実行すると信じたいと思います。
