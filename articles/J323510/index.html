Veeam Availability SuiteがCisco HyperFlexとの統合をサポートするようになりました

Veeam Software AGとCisco Technology、Inc. Cisco HyperFlexハイパーコンバージドソリューションのスナップショットとバックアップの操作を可能にするAvailability Suiteツールを導入しました。 同社は、ITシステムの復旧時間を短縮し、生産性を向上させると主張しています。





/ photo Carolina Prysyazhnyuk CC



Veeam Proxyには、NFS HyperFlexデータネットワークを使用してCisco HyperFlex仮想マシンから直接データを読み取る機能があります。 同時に、最適なブロードキャストチャネルまたはHyperFlex Controller Cluster MasterをサポートするIO Visorを使用して情報を送信できます。 次の図は 、このようなアーキテクチャの例を示しています。









VeeamとHyperFlexの直接統合にはいくつかの利点があります。 ユーザーは、タスクの必要に応じて何度でもバックアップを復元でき、会社の生産プロセスへの影響を最小限に抑えることができます。 Veeamは、スナップショットの作成と削除がパフォーマンスに与える影響を大幅に削減できました。

Veeam Cloud Connect：クラウドバックアップ





災害復旧とクラウドバックアップ：新しいVeeam Cloud Connect機能



新しいAvailability Suiteツールは、バックアップデータがファイルシステムから直接および非同期に読み取られるため、標準のvSphere APIと比較してコピー速度が2倍になりました。 また、ユーザーはワークロードを他のHyperFlexシステムや他のvSphere環境に転送することができました。



この場合、Veeamレプリケーションツールは、ソースホストで実行されている元の仮想マシンをコピーし、ターゲットホストでその完全なコピーを作成します。 仮想ディスクのレプリカは圧縮されずに元の形式で保存されるため、コピー速度は高速です。 さらに、Veeamは最後の作業セッション以降に変更されたデータブロックのみをコピーします。









会社の代表者によると、このアプローチにより、ストレージスナップショットからのバックアップを使用する際のアクセシビリティが向上します。 Veeamの責任者であるPeter McKay氏は、次のように述べています。 「シスコとチームを組むことにより、お客様に情報の高可用性を提供し、ITの可能性を完全に実現することができました。」
