これを行うには、ポップアップウィンドウの[自動的に]リンクをクリックします。 このウィンドウを誤って閉じた場合、またはそれ自体を「閉じた」場合は、ステータスバーをクリックし、ターミナルで「自動的に」リンクをクリックします。 すべてを正しく行った場合、対応するメッセージが表示されます：「psr-0名前空間ルートを検出：psr-0名前空間ルートが検出されませんでした」。 ウィンドウを非表示にするには、垂直矢印の付いたアイコンをクリックします。

リモートサーバーに関連する新しいプロジェクトを作成する



リモートサーバーに関連付けられたプロジェクトを作成するには、ようこそウィンドウで[既存のファイルから新しいプロジェクトを作成]を選択するか、 IDEメニューの[ファイル-既存のファイルから新しいプロジェクト]を選択します。



画像




画像




次に、使用シナリオを選択するように求められます。「Webサーバーはリモートホスト上にあり、ファイルはftp / sftp / ftps経由でアクセス可能」オプションが必要です。



画像




次に、プロジェクトの名前を入力し、PCでの保存場所を指定します。 プロジェクトのストレージパスを変更する場合は、[カスタム]オプションを選択し、プロジェクトを配置するフォルダーを指定する必要があります。



重要！ ファイル、フォルダー、ストレージパスの命名にはラテン語のみを使用します。 一部のツールは単にキリル文字をサポートしていない可能性があるためです。



例：



  • 「C：// my documents / project / demo」は悪いオプションです。プロジェクトのパスと名前にキリル文字が含まれています
  • 「C：//プロジェクト/デモ」-ここではすべて問題ありません。プロジェクトのパスと名前にはラテン文字のみが含まれています


画像




次に、新しいリモート接続を作成するか、既存の接続から選択するように求められます。 新しいsftp（shh）接続を作成し、リモートサーバーに接続するためのパラメーターを指定します。



  • 名前-接続の名前、わかりやすい。 複数のテストサイトが同じIPに配置されることがあるため、完全修飾ドメイン名を記述することをお勧めします
  • タイプ-ftpまたはsftp（shh）接続形式
  • Sftpホスト-サーバーまたはホストのIPアドレス（たとえば、mysite.ruまたはdemo.mysite.ru）
  • ポート-ポート番号（ftp（21）、sftp（22）のデフォルト）
  • ルートパス-親ディレクトリへのパス（デフォルトでは、「/」のまま）、たとえば、サーバー上に2つのフォルダー（dev、www）があり、接続全体にdevフォルダーをすぐに表示する場合、パスを指定できます（/ dev /）
  • ユーザー名-サーバーにアクセスするためのログイン
  • パスワード-サーバーにアクセスするためのパスワード（サーバー、ターミナルに接続するたびにパスワードを入力しないように、「パスワードの保存」ボックスをチェックすることを忘れないでください）。 sshキーを使用して接続することもできます。
  • 「サーバーへのHTTP接続を確認しないでください-毎回接続を確認しない場合はチェックボックスをオンにします。」


[Test sftp connection ...]ボタンを使用すると、アクセスパラメータが正しく入力されているかどうかを確認できます。 成功すると、対応するメッセージが発行されます。



「WebサーバーのルートURL」は、リモートサーバーで表示するためにファイルが開かれるパスです。 sftpホスト行がいっぱいになるとすぐに、パスが自動的に割り当てられます。

オプションで、ホットキー（キーマップ）を割り当てて、ファイルをすばやくダウンロードおよびアップロードできます。



この方法は、1つのファイルのみを修正する必要がある場合に便利ですが、完全な開発のためには、サーバーへのファイルの自動送信を設定することをお勧めします。 これを行うには、キーの組み合わせ「Ctrl + Alt + S」を押すか、メニュー「ツール-展開-オプション」で変更を行います。

  • リモートの変更について通知する-現在のバージョンではないソースファイルをアップロードしようとした場合に警告します。 たとえば、ファイルが2人で同時に修正された場合。 上書き、保留、スキップが提供されます。
