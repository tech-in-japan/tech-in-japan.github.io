最小限の対話性でLinuxワーキングイメージを展開する

この投稿は、次のいずれかを使用する怠zyなシステム管理者にとって興味深いものです。

  • Linux-クライアントマシン上のベースシステム。
  • 異なるハードウェア上の同じタイプのシン/シッククライアント。
  • クライアントデータのネットワークストレージ。
  • 事前調整されたソフトウェア。
  • Openvpn /アプリケーションサーバーへのrdp。
  • NagiosのようなLinuxクライアントの豪華な（ソフトでふわふわ）モニタリング。


ユーザーまたは通常の「enikeyshik」の側で最小限の労力で、ユーザーのコンピューターに一連のソフトウェア、構成済みネットワークサービス、定義済み構成などを備えた既製のシステムをインストールする方法についてです。 この記事は主に愛好家と初心者のLinuxoidを対象としていますが、毎日何十台もの車のグループの戦闘展開にトリックとスクリプトが使用される可能性があります。



まず、分布を決めましょう。 もちろん、これは主にタスクに依存します。 xubuntu 14.04を選んだのは、次の3つの簡単な理由からです。



1. ubuntu-幅広いサポートと膨大な数のフォーラムを備えたdebベースのシステム。

2. xubuntu-ラッフルや見栄えのない軽量バージョン。512MBのRAMのマシンでは速度が低下しません（私の公園にはこのような恐竜がいます）。

3. 14.04は2019年までサポートされるLTS（長期サポート）オプションです。



もちろん、オプションはより速く、より小さくなりますが、多くのさまざまな理由により、選択が行われ、残念ながら1年後悔することはありません。



先を見据えて、私は注意します。上記を繰り返すには、ローカルネットワークで設定されたvpnサーバーまたは「config-monitoring-server」、2、3台の仮想または実マシン、2、3時間の空き時間、および細部への注意が必要です。



そのため、最初に画像を準備します。 すべての段階が重要ですが、最初の段階は基礎です。 cd / dvd / usb / networkでシステムをインストールし、ディスクレイアウトを選択します。 イメージは、展開前に動的なサイズ変更をサポートしないことを理解することが重要です。 そして、おそらく、そのようなサイズ変更は必要ありません。 ルートの下では、6〜10 GBを割り当てるのが理にかなっています。



a）クライアントマシンのディスクがこのボリュームより小さくなく、b）実際に多くの異なる必要なパッケージがインストールされることがわかっている場合、より大きなボリュームが正当化されます。



/ homeをマークして、最後にパーティションを交換しましょう。 合計で、作成者の3つのセクションすべてが30 GBをクロールしません（30 GBのssdに静かに収まるように、これは私のクライアントの最小ディスク構成です）。 レシーバーディスクの最小サイズが事前にわかっている場合、そのパーティションを調整することは理にかなっています。 ディスクのルートの直後、/スワップの直後のホームにスワップをマークすることをお勧めします。これにより、サイズ変更が発生した場合に「コンソールでディスク上でそれらを運ぶ」のではなく、1つのコマンドでエレガントにできます。 ロケール、ユーザー名、パスワード、ホスト名を設定し、新しいLinuxに入ります。 続行するには、x11vnc、ssh、rsync、openvpn、mcのパッケージが必要になります（mceditが大好きです。90年代のノートンのように、ネイティブです）。 



sudo apt-get install x11vnc ssh rsync openvpn mc ntp


Skype、Icedove（Thunderbird）、Iceweasel（firefox）、Crrom-browser、rdesktop calculator、リソースモニター、viber、libre office、ワイン、およびいくつかのWindowsアプリケーションを好みに合わせて追加しました、しかし類似体はなく、期待されていません）。 このすべてをゆっくりとゆっくりとセットアップしています。 すべてが開始して動作することを確認します。



VPNを構成します。 分散ネットワーク、ジオロケーションごとに2台または3台のコンピューター、ロシアでは50を超える場所があります。 サーバーへのsshキーとクライアントへのサーバー上でsshキーを作成します（クライアントはこれらのパスワード、詳細、たとえばここに入力する必要があります ）。 サーバー上に無条件の同期ディレクトリを作成します（私にとっては、これらは作業スクリプト、openvpnキー、クライアントcron、Pythonのユーティリティスクリプト、C ++バイナリです）。 実際の例では、これは/ var / sync / in、/ var / sync / in / cron、/ var / sync / outです。 そこには、ログと、顧客から必要な情報（スクリーンショット、プロセスレポートなど）があります。 同期スクリプト、cronジョブ、監視用のハートビートスクリプト、システムサービスの多数の構成（同じcupsまたはopenvpn）があります。



複製のために全体を準備することは、1人のユーザーにシステムをインストールするよりも時間はかかりませんが、詳細を注意深く調べることで、管理者が将来、戦闘中のすべての画像を「仕上げる」スクリプトに費やす時間を大幅に短縮できます。 クライアント上でopenvpnを設定し、ユーザーキーが[および]動作することを確認します。 grub2を設定することは不要です。



  • Khint1：ホスト名、openvpnのIP、およびユーザー証明書に一意のIDが含まれている場合（たとえば、ホスト名としてuser10.key、xyz10、vladivostok_10）。
  • Khint2：さて、これらのファイルのコピーが手元にあるとき、たとえば/ var / sync / hostname、/ var / sync / id、/ var / sync / location;
  • ヒント3：未登録のユーザー用に5〜10個のアドレスを予約することも役立ちます。


Hint2から意識的にファイルを作成することはありません。それらが存在しないことにより、「新しい」ファイルがいつvpnに接続するかが明確になります。



だから、すべてがカサカサし、オフィスが機能し、計算機が起動します。アプリケーションサーバーを使用するrdpには、デスクトップに既にショートカットがあります。

gpartedからのパーティションコピーはクールなものですが、悲しいかな、初心者向けでもありません。



私は長い間、完璧な展開をどのように見ているかを自問してきました。 ここで、私たちはすでに秩序に気を取られている記事のトピックに行きます。



  • 画像は小さな（私の場合は2GB）フラッシュドライブに収まるはずです。
  • 1つまたは2つのボタンを押して画像を設定する必要があります。
  • 展開後、イメージ自体を同期し、管理者に新しいマシンが「正常」であることを「報告」する必要があります。
  • 新しいマシンの登録にはそれほど時間はかかりません。
  • （重要！）画像は、指示に従ってアマチュアが窓の下からUSBフラッシュドライブに記録する可能性があります。
  • （アーカイブは重要です！）イメージはバージョン付きで開発される可能性が高いため、定期的に更新し、迅速に配布する必要があります。


小さな自転車を発明するか、展開スクリプトを作成する必要がありました。 PartedMagicは、上記のタスクのために選択されました。 イメージをラップするには、unetbootin + partedmagicまたはLinuxとtar + b [g] zipを搭載したライブCD（usb）が必要です。 ダウンロードしたイメージはPartedMagicの「ramから実行」であり、フラッシュドライブは/ media / sdb1（FAT32）にマウントされていると想定しています。



フラッシュドライブのfat / ntfsに関する注意事項
  • Akhtung1 fat32のファイルセクションは4GBを超えてはならず、イメージのボリュームを監視するか、ext4パーティションを選択してください。
  • Linux用のAkhtung2 NTFSフラッシュドライブセクション。動作しますが、膨大なCPU時間を要します。




ロード後、ディスク（ほとんどの場合/ dev / sda1）をマウントし、システム全体から直接クリックします。/homeセクションを忘れないでください： 



  mount /dev/sda1 /media/sda1


次に、ディスクのすべてのコンテンツを含むアーカイブを作成します。 



 tar -czvpf /media/sdb1/image.tgz /media/sda1


同じことが/ homeセクションにも当てはまります。 



 mount /dev/sda5 /media/sda5 tar -czvpf /media/sdb1/home.tgz /media/sda5


次に、パーティションテーブルをスーパーフラッシュドライブのファイルにダンプします。 



 sfdisk /dev/sda1 -d >parts.txt




したがって、フラッシュドライブには、アーカイブと参照イメージのパーティションテーブルを含むファイルがあります。
