再設計-10億を失った

再設計の壮大な失敗を検証し、口ひげで振ります。




製品マネージャーは設計部門に入り、Webサイトの再設計を注文します。 「私たちのサイトはとても古いようです！ 競合他社はすべて、より明るいサイトを持っています。 再設計しましょう。 カラフルな影のボタンは未来です！」



デザイナー：「わかりました、使いやすさをテストすることから始めましょう。」

PM：「3週間あります。すべての色をホットピンク、明るい青、またはオレンジに置き換えたいと思います。」



コンバージョン率を下げる方法は次のとおりです 。 上記のシナリオは、大胆に誇張されているかもしれませんが、毎年ブランドを根本的に再設計する企業の数を見て、これはそうではないことがわかります。 Facebookをご覧ください。Facebookを開くたびに新しいように見えます。 Facebookには、会社が変更した詳細を追跡する3000人のメンバーのグループ全体があります。







Skype Redesign 2018



ユーザーが遭遇する製品は、常に変化し続ける市場で関連性と新鮮さを維持する必要があります。 再設計は、Webサイト、アプリケーション、またはあらゆる種類のユーザーや関係者とやり取りする製品にとって不可欠です。 データ管理プロセスと設計の明確に定義された主要業績評価指標によってすべてが正しく行われた場合、開発された製品はコンバージョンを増やし、売り上げを増やし、ユーザーを満足させることができます。



しかし、多くの場合、コンバージョンの低下、ユーザーの怒り、アプリに関する否定的なレビューなど、再設計はむしろ望ましくないと思われます。 Snapchatのような小さなプレーヤーであろうと、 Windows 8やSkypeを搭載した Microsoftのような大きなプレーヤーであろうと、誰も再設計の失敗に免疫がないようです。



再設計が失敗する理由



再設計は、視覚的であろうと機能的であろうと、ユーザーが製品を覚える方法の変更です。 人々は変化を嫌います。 ユーザーがアプリケーションの設定ページにアクセスする方法を変更するときは、習慣を取り除くようにユーザーに依頼します。 これまで考えずにアプリケーションの設定ページにアクセスできたユーザーは、今すぐアプリケーションを見つけようとしています。 これは微小攻撃につながります。 デザイナーとしてのあなたの仕事は、これらの微小な攻撃を最小限に抑え、新しい習慣を作り出すよう動機付けることです。



Snapchatのナビゲーションの再設計







Snapchatが主要なユーザー定義機能の1つである友人のストーリーの表示を再設計した方法を見てみましょう。



Snapchatの古いバージョンでは、次のナビゲーション構造が設定されていました。ストーリーを表示するには左にスワイプし、友人とチャットするには右にスワイプします。 友達のストーリーを表示するには、指で左にスワイプし、友達のストーリーをクリックします。



有名人やメディアによる宣伝記事はこのページの一部でした。 彼らはあなたの友人の物語の隣に現れました。







再設計により、友人のストーリーは左に移動し、チャットと結合されました。 注目の記事にページが追加されました。 この再設計により、Snapchatは以前のチャットとストーリーの分離から離れました。 焦点は、友人とコンテンツ発行者の間の共有にありました。 Snapchatは、友人に関連するすべてを1か所にグループ化することにより、有料コンテンツを宣伝できます。







ストーリーを表示するためにクリックする必要があるメイン領域も、再設計プロセス中に短縮されました。 以前は、ユーザーはクリックして友達のストーリーを見ることができる長方形の領域がありました。 再設計後、それははるかに小さな円に縮小され、画面の左端まで押し出されました。







少なくとも2つの実践ルールが変更されました。ナビゲーション構造と、ストーリーを表示するためにクリックする必要がある領域です。 言うまでもなく、ユーザーはこれらの変更を本当に好まなかった。 1週間以内に、Snapchatの平均評価は3.1から2.4に低下しました。



Appbotを使用してユーザーレビューをスマートに分析しました 。 再設計がユーザーに提示された後、否定的なレビューが急増していることがはっきりとわかります。

再設計の導入から6か月後、Snapchatは変更を加えて、シャッフルを発表しました。 友人の話は、出版社の話とともに正しく移動しました。 引用：

Windows 8の[スタート]メニューの編集



次に、MicrosoftがWindows 8で[スタート]メニューを再設計した方法を見てみましょう。Windows7および8でコントロールパネルを開いてみましょう。



Windows 7でコントロールパネルを開くには、[スタート]ボタンをクリックして、すべてのアプリケーションのリストを表示する必要があります。 次に、コントロールパネルをクリックします。







Windows 8では、既に[スタート]メニューから開始しています。 良いスタートですよね？ 間違った。 リストされているアプリケーションには、コントロールパネルはありません。 すべてのプログラムをリストする方法もありません。



パネルにアクセスするには、チャームメニューを開く必要があります。 メニューを開くには、マウスカーソルを画面の右上または下に移動します。 チャームメニューを開くことができたら、[設定]をクリックしてからコントロールパネルをクリックします。







このジェスチャベースのパネルアクセスプロセスは、ユーザーがWindows 8を起動したとき、ユーザーにはまったく関係ありませんでした。Windowsは、ユーザーがチャームメニューを開くのに役立つ追加情報を提供しませんでした。 ユーザーがチャームメニューを見つけた後でも、Googleやその友人の助けを借りて、ユーザーはそれを不快に思っていました。 ジェスチャーとフリーズは、OSの新しい概念でした。



再設計はすべて悪いですか？



再設計は変化であり、人々は変化を好まない。 上記の例から、製品の動作方法を変更するために再設計に時間を費やすことは、製品にとって良い投資ではないかもしれないと推測するのは簡単です。 しかし、「安全なゲーム」のために同じデザインを開発および保持していない製品は、動きの速い競合他社のために必然的に消滅します。



ユーザーが安全に受け入れることができる軽度かつ漸進的な変更により、再設計に耐えることができます。 これにより、鮮やかなアニメーション、ジェスチャー、またはナビゲーション構造をスキップして、ユーザーが既に使い慣れているより単純な代替手段を使用する場合があります。



Facebookナビゲーションの再設計



2012年、Facebookはアプリのナビゲーションを再設計する準備を整えていました。 これまで、友人のページ、メッセージ、通知へのパスは、画面上部の青いアクションバーを介して行われていました。 このアクションバーは、ユーザーがサイトをナビゲートする方法をシミュレートするために設計されました。







AndroidとiOSは、同じアクションパネルで同じ方法でナビゲーションを実装しました。







しかし、モバイルユーザーの基盤は厳しくなり、モバイルから始めたユーザーは、デスクトップバージョンを使用しているユーザーを徐々に暗くしていきました。 Instagram、Snapchat、Vine（rip）などのモバイルソーシャルネットワークが人気を集めています。 将来、人々はコンピューターよりも電話で過ごす時間が長くなることは明らかでした。



そのため、Facebookはモバイルデバイスのナビゲーション構造を簡素化することを決定したときに、一貫性を放棄し、AndroidとiOSを2つの異なる製品と見なすことにしました。 Facebookは、それぞれのアプリケーションに2つの異なるナビゲーション構造を導入しました。 Androidアプリには、画面の上部に固定されているスライドタブバーがあり、iOSには下部に固定されている静的タブバーがあります。







これらの変更はアプリケーションにとって新しいものでしたが、ユーザーにとっては新しいものではありませんでした。 再設計では、ユーザーがナビゲーションのために青いアクションバーに移動する習慣を取り除く必要がありましたが、ユーザーは特定のプラットフォームのタブバーに既に慣れていました。 スライドバックは数年前からAndroidエコシステムに存在します。 ユーザーがアクションバーからタブバーに切り替えるのにそれほど時間はかかりませんでした。 同様に、iOSのタブバーはプラットフォームの設立以来存在しており、ユーザーにとって違和感はありませんでした。



AndroidとiOSのFacebookタブの分離は、非常に良い効果を発揮しました。 彼らはすぐに業界標準になりました。 ほとんどのアプリケーションは、この種のナビゲーションを実装しています。

原則として、最も効果的な再設計は最も美しいものではありません

再設計は、物を飾るプロセスではありません。 視覚的な側面は再設計の一部ですが、再設計ではありません。 より魅力的な視覚要素のためだけに製品をアップグレードし、Dribbleの完成したレイアウトからインスピレーションを引き出し、それをサポートする調査なしにデザイントレンドを実装すると、再設計は失敗する運命にあります。



時には問題は、古いデザイン言語がブランドの価値を下げることかもしれません。 顧客は、より魅力的な設計システムを備えた競合他社に向かっています。 そして、これは、システムのデザインをより良く見えるように変更するという完全に普通の願望です。 しかし、盲目的にトレンドを追跡し、このニーズをサポートする戦略や研究なしで設計すると、好ましくない結果につながります。



再設計は、特定のビジネス上の制約と要件に伴う問題を解決するように設計されています。







このDribble Googleの再設計コンセプトでは、鮮やかな色と濃い影を使用して注目を集めています。 このデザインは不安定に見えるという事実にもかかわらず、いくつかの重大な欠陥があり、Googleに光を当てることはありません。 まず、Googleからのカルトブランドはありません。ブランドは過去20年間このブランドを守り続けてきました。 第二に、すべてのGoogleサイトのプロファイルアバターは右側にあり、左側にあります。 私は続けることができますが、私が伝えようとしていることの本質をあなたが理解することを望みます。



再設計はプロセスであり、目的地ではありません

製品設計とは、製品またはプロセスのプロトタイピング、テスト、分析、改善を繰り返すプロセスです。 再設計は、製品が完全であることを意味するものではありません。 これは単に、現在の要件を満たすために「十分に実装されている」ことを意味します。 製品やユーザーインターフェイスが「準備完了」になることはありません。



マイクロ変換に焦点を当てる

ユーザーとユーザーの活動をよりよく描くために、行の間を読むことが重要です。 原則として、技術分野でのコンバージョンについて話すときは、アプリケーションのインストール、登録ユーザー、または売り上げの数に集中します。



マイクロコンバージョンはしばしば忘れられ、追跡されないこともあります。 マイクロコンバージョンは、ユーザーを最終的な目標、つまりマクロコンバージョンに導くアクション、つまり、手間のかからない成果です。 マイクロコンバージョンの例には、検索機能を使用するユーザーの数、プロフィール写真を更新するユーザーの数、または電話番号を確認するユーザーの数があります。 マイクロコンバージョンは、コンバージョンファネルのプロセスステップとして機能し、最終ステップまたはマクロ変換に影響します。



これらのマイクロコンバージョンはどのように機能しますか？ 特定のマイクロコンバージョン指標に低下が見られますか？ 新しい検索バーが見つけにくいため、ユーザーは友達を追加できませんか？ 無料試用版へのサインアップの呼びかけは新しいレイアウトのために埋もれてしまい、ユーザーはそれを見つけることができなくなりましたか？ 正確なマイクロコンバージョンの目標を特定して追跡することは、変更されたプラットフォームでのユーザーの旅の共通点を評価するのに役立ちます。
