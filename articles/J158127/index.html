MR3020の仕上げ、カーネルの再構築、開発者ツールの構成

こんにちは、ハブロビテス。 中国のルーターTP-Link TL-MR3020（または中国市場のハードウェアアナログTL-WR703N）は最近、DIYプロジェクトで人気を博しているため、特に、プロジェクト用にそれを完成および構成するためのオプションに関する記事を書くことにしました職場では、ここ数か月彼と一緒に楽しんでいました。 この記事では、初心者向けのほとんどの記事ではカバーされていない側面、つまり、ファームウェアを再構築し、便利な開発者ツールを自分で設定する実際的な例を検討します。



はじめに



そこで、最初に、すでに何度も言われたことの簡単な概要。 テスト対象は次のとおりです。



当初、これはUSBポートを備えた中国のルーターであり、3Gインターネットのモバイルディストリビューターとして位置付けられています（3GモデムをUSBに接続することになっています）。 USBポートと小さなサイズの存在について、開発者にも好まれました。実際、フルLinux（OpenWRT）が搭載され、USBを備え、さらには無料のGPIOを備えたミニチュアコンピューターが搭載されているためです。 モスクワで約850ルーブルの量で売られました。

彼の仲間TL-WR703Nについては。 ハードウェアでは、MR3020とほぼ完全に同一ですが、中国市場専用であるため、ここで見つけることははるかに困難です。 違いのうち、サイズは小さく（ケース全体のルーター全体のサイズは MR3020 プリント回路基板のサイズと同じです）、GPIO（3020にはLEDがあります）が少なく、私のデータによると、Wi-Fiアンテナは少し悪いです。

ネットワークで説明されているさまざまなMODおよびハックについては、主に外国のWebサイトで、それらは703Nでの作業について説明していますが、これらのハックの99.99％はMR3020に適しています。

ハードウェアは次のように表されます。

  • Atheros AR7240 CPU（400Mhz）
  • Atheros AR9331チップセット（統合ワイヤレス）
  • 802.11 b / g / n 150Mbps（130Mbpsリアル）
  • ワイヤレス出力20dBm-100mW
  • 4 MBフラッシュメモリ
  • 32 MBのRAM
  • USB 2.0ポート


さらに、 wiki.openwrt.org / toh / tp-link / tl- mr3020およびwiki.openwrt.org/toh/tp-link/tl-wr703nの記事と関連資料を読んで、これらのルーターに関する必要なアイデアを得るよう強くお勧めします。 OpenWRT。これは、この記事のwikiにすでに記述されているものを単に再印刷することは現実的ではないと思うからです。

開発パス



それでは、このルーターについて最も不愉快なことは何ですか？

  • 0.ゼロポイントはそれ自体が組み合わさっており、残念ながら、私たちは決して影響を与えることはできません。 これは、実際、ルーターの鉄の起源です。つまり、開発者向けではなく、回路図が閉じられているという事実です。一般的に、私たちはそれを修正して商用製品で使用する権利はありません。 また、中国人がそれを生産するのに疲れたとき、それに関するすべての決定は何か他のものの下で緊急に引きずられなければならないという事実。
  • 1.少量のROM-使用可能なフラッシュメモリは4メガバイトのみで、そのうちの約1.5がファームウェアで占有されています
  • 2. 1つのUSBポート-多くの周辺機器（カメラ、モデム、GPS、フラッシュドライブ）が必要になることが多いため、これも悲しいことです。
  • 3. RAMの容量があまり大きくない-32メガバイト。


それでは、順番に始めましょう。



鉄ははんだ付けされていません


ポイント0に関しては、別のハードウェアプラットフォームに切り替えるだけです。 開発者が利用できるもの- カランボラボード。 ルーターの特性とほぼ同一、サイズがほぼ同一、価格がほぼ同一（配送を除く）。 開発者向けに設計された多数のGPIOが表示され、あなたの心が望むものをはんだ付けできます。

欠点-一般的な信念に反して、回路図はまだ閉じられています。 このカランボラが挿入されるのは、乙女のボードの概略図のみです。

また、不利な点としては、配送コストと待ち時間が挙げられます-ルーターははるかに安価で高速です（近くの店で850ルーブルですぐに入手できます）。

さらに、利点と欠点のいずれにも起因しないカランボラの曖昧な特性-USBコネクタもイーサネット相互接続もボードに実装されていない-裸のピンのみ。 カランボラを貼り付けるベースボードを設計している場合-これは明確なプラスです。 「完成して接続されたものを購入」シリーズの迅速なソリューションが必要な場合は、はんだごてを振る必要があります。

低ROM


基本的に、それはすべてあなたのニーズに依存します。 望むなら、これらの不幸な4メートルに固執することができます。そして、Von Neumannに、プログラムからの重いログを書き留めないようにしないでください。

個人的には、メモリをすぐに拡張し、ROMを永久に忘れることをお勧めします。 これは、「自分がどれだけ残っているか」を絶えず考えずに作業の快適さを提供するものの1つであるため、この計画を実装する方法について話しましょう。

最もエキゾチックなものから始めましょう-あなたはフラッシュドライブ自体をはんだ付けすることができます。

ソリューションの利点は何ですか？ 明らかに、すべてがルーターボードに統合されたままです。外部にハングアップするものはなく、消費量は低いままです。

欠点は何ですか？ 実際には、SO-8本体の下にシートがあり、ポップ対応のフラッシュドライブには8メートルのものしかありません。 これは、「快適な仕事」が意味する拡張機能ではありません。 ただし、この決定のプラスがあなたにとって重要である場合、それはかなり実行可能です。 ここでそれについて読んでください 。 要するに、再はんだ付け後、ブートローダーとカーネルを再構築する必要があります。フラッシュメモリの容量、システムの配置、チップIDをパッチすることを忘れずに、デバイスは起動を拒否します。

さらにエキゾチックなオプションは、外部ボードのピンと互換性のないフラッシュドライブを配置し、ルーターのネイティブメモリをはんだ付けしていない状態で、制御ピン（SPIバス）を挿入することです。 これにより、8メガバイトを超えることができますが、このフラッシュドライブを認識するには、Linux用の別のカーネルパッチとブートローダーが必要です。 非常に大きな恋人のためのオプション。 その変種は、バス上のスレーブ（SS）を選択し、既存のフラッシュドライブと並列に追加のフラッシュドライブを接続するためのソフトウェアピンとして無料のGPIOの1つを使用することです。 前者が恋人向けだった場合、これは死体の破片で作られたモンスターを手に入れたい変態ネクロマンサー向けです。 しかし、私の意見では、実装されました。

最後に、従来のソリューションは、USBインターフェイスで外部メモリを使用することです。

長所-高速で便利、カーネルを再構築する必要がなく（適切なモジュールをインストールするだけ）、ブートローダーを再構築する必要がまったくありません。 8 GBのフラッシュドライブを搭載することで、メモリを好きなだけ拡張できます。ルーターの空き容量の問題を永遠に忘れていました。

短所-1つのUSBポートがビジーであり、消費電流が50ミリアンペア増加します。ルートFSとしてマウントする場合（後述）、ロード時間が増加します。 フラッシュドライブ自体は大きくすることができます-そのような付属物がハングするとき、それは常に良いとは限りません。

それでは、最後のマイナス（フラッシュドライブのサイズ）から始めて、それで何ができるかを考えましょう。 市場をレビューした後、私は「箱から出してすぐに」2つの選択肢とアマチュア無線に行きました。 最初の2つについては、ここではすべてがシンプルです。市場には非常に小さなフラッシュドライブと非常に小さなマイクロSDカードリーダーがあります。

この件については、2つのニュースを表明できます。 良いことは、RAMを互換性のある64メガバイトのマイクロ回路で簡単に置き換えることができることです。これは、ほとんどの場合、実験の大部分に十分すぎるほどです。 このため、フラッシュは不要で、ローダーやカーネルにパッチを当てる必要もありません。 これについては、 ここで詳しく説明します。

悪いニュースはすべてそのようなソリューションです-上記のルーターとASUSの兄弟（WL-500など）、および開発者Karambolaは最も古いDDR1チップ上に構築されており、それらと互換性のある64メガバイトを見つけることは大きな問題になりました。 可能性のあるソースは上記のリンクで示されます-それらは古いRAMモジュールです（主に異なるラップトップから）-しかし、ほとんどの場合、グーグルは販売のオファーはありませんが、このリストを一言一句リストする同じ電子エンジニアのフリーマーケットでの祈りと少なくとも1つのそのようなバーの所有者は、驚くべきことに周囲の人々に応答するように求められます。

ちなみに、これは、Texas Instrumentsの最新のチップ、たとえばSITARAで同様のソリューションを本当に見たい理由の1つです。これらは、Linuxが長年移植されてきたARMコアを備えた最新のアクセスしやすいプロセッサです。 これらには、乙女、SDK、およびその他のパンの典型的なスキームを備えたapnoutが付属し、さらにDDR3で動作します。 ただし、このようなボードの開発には多くの時間とかなりの投資が必要です（LUTは役に立ちません、多層のものが必要です）。特に具体的なタスクがまだないので、私はまだこれを敢えてしません。 おそらく誰かがすでにこれを行っている/それを計画している-協力することは可能だろう-目標は、過剰な鉄を巻き取ることなく、開発者がそのような「ルーター」ソリューションを置き換えるための起動プラットフォームとして、Carambolaに似た最小限のシングルボードコンピューターを作ることです。

コアを再構築する



すぐに声を出したタスク。 コアを再構築して何を達成したいのですか？

  1. OpenWRTを使用すると、カーネルモジュールを含め、リポジトリから必要なものをすべて簡単にロードできるため、システムを起動するために不要なものをアセンブリに組み込む必要はありません。
  2. パスやその他の誤解に関する不必要な問題を回避するため、ルートFS全体を追加のディスクとしてマウントするのではなく、USBフラッシュドライブに転送することにしました。 これにより、ライブラリを間違った場所にインストールするためにライブラリが「見つからない」という考えられるすべての問題が解決され、ルーターの使用がより便利になります。 これは、カーネルに組み込まれたモジュールの要件を意味します-USB、ext4ファイルシステム、外部ドライブのサポート
  3. ルーターのマイナープリセット-フラッシュ後に最初からやり直す必要がある場合は煩わしいので、ファームウェアのFSレベルで既に修正できるため、毎回これを行う必要はありません-たとえば、デフォルトではMR3020がフラッシュ後にすぐに動作するため、サーバーのふりをして、実際のルーターを蹴るのではなく、IPを取得してネットワークの一部になります。これにより、開発/デバッグプロセスが簡素化され、「インターネットがある」と「接続がある」のどちらかを選択する必要がなくなります MR3020»で


再構築プロセス自体はここで詳しく説明されていますが 、啓発の前に読むことをお勧めします。繰り返しになりますが、「make menuconfigコマンドを入力してカーネルを設定する」ことはありません。

この段落を読んだ時点で、前に記事で説明したすべての項目（実際にはmenuconfig）を既に完了しており、すべてが機能することを確認してカーネルをビルドしようとしたこともあります。 私は原則として初めてそれをしました。つまり、問題を引き起こすことはありません-すべての手順はかなり透明です。 私が注意する唯一のことは、奇妙なことに、 dircleanヘルプを作成することです 。これにより 、構築したものだけでなく、システム自体が構築したもの、つまり収集したツールチェーンなどもクリーンアップされます。

これが役に立たない場合は、 make distcleanを作成する価値があります。システムによってダウンロードされたものを含め、一般的にすべてをクリーンアップします。 その後、ビルド全体が非常に長い間、最初から実行されますが、問題が発生しなくなる可能性があります。 すべてが組み立てられるまで数時間待ちたくない場合は、特別な必要なしにこれを行わないでください。

menuconfigに移動し、正しいターゲットとターゲットプロファイル（Atheros AR7xxx / AR9xxx、TP-Link TL-MR3020）を選択し、保存して、configにMR3020のデフォルト値を入力することを忘れないでください。

menuconfigの作成が完了したら、 iptablesなどのビルドを含め、ほとんどすべてをビルドから削除しました。1つ目はルーターではないため、2つ目は後で簡単に配信できるためです。 したがって、メニューを調べて、不要なものはすべて破棄します。

私は次のものを残しました：

  1. ベースシステムでは、 ベースファイル、busybox、dnsmasq、dropbear、hotplug2がデフォルトで残っているという事実から、ディスクを接続するのに必要なブロックマウントが組み込まれていることを確認します （USB接続通知なしで放置したくない場合は忘れないでください）およびボタンを押す）、 mtd、opkg、uci
  2. Kernel Modulesで、Filesystemsセクションに移動し、 そこでkmod-fs-ext4をアクティブにします。

    必要に応じて、他のシステムを選択できますが、ext4は1つで十分です。 私がインストールしていない他のすべてのカーネルモジュールは、デフォルトのLEDモジュールからいくつかの項目を除いて（もちろん、それらを削除すると、ルーターはLEDを点滅させず、ブートプロセスがどうなったかさえわかりません）、 kmod-gpio-button-hotplug 、ルーターのボタンが押されたときに通知を送信します（1つのボタンと1つの3位置レバー、Linuxを2つのボタンと見なします）、同じ場所にあるデフォルトのkmod-wdt-ath79 、私たちにとって非常に重要なkmod-usb -ochi、kmod-usb-storage-extras、kmod-usb-uchi、kmod-usb2（ USBサポートから）および組み込みのドライバー Wi -Fi-ワイヤレスドライバーのkmod-ath9k 。


それ以外はすべて無効にされたか、後続のインストールのためにモジュール形式でコンパイルされました。

この段階では、OpenWRT wikiの指示に従ってカーネルをビルドしてからフラッシュしても問題ありません。 



cd /tmp wget ------.bin mtd write -.bin reboot




その後、ルーターが正常に起動した場合-MOTD（今日のメッセージ、ログイン後の招待）に注意してください-ビルド番号が変更されるはずです-おめでとうございます、カーネルの組み立てとシステムの変更に成功しました。 ルーターが起動を拒否する場合-問題ではありませんが、幸いなことにタッチしなかったスマートブートローダーの助けを借りて状況を保存できます-再びwikiの指示に従います（USB-rs232アダプターが必要です）。

それでは、設定に移りましょう。 FSがフラッシュにフラッシュされるイメージに組み込まれ、OpenWRTソースのどこかにあると想定するのは論理的です。 そうです、途中でFSを見つけることができます

openwrt / trunk / build_dir / target-mips_r2_uClibc-0.9.33.2 / root-ar71xx

（将来的には、コンピューターに表示されるすべての方法を記述します。openwrtは、ソースコードがチェックされたディレクトリです）

このディレクトリはビルドごとに新たに生成されるため、ここでは、ビルドシステムを大きく損なうことを恐れずに、必要なものを編集できます。 したがって、最初にカーネルを構成し、アセンブルしてから、ここで修正を行い、カーネルを再構築せずに必要なモジュールを個別に終了します。 何かを台無しにした場合は、もう一度makeを実行するとディレクトリが復元されます。 openwrt / trunkから彼女のイメージをビルドするには、 make target / installコマンドを発行します

ただし、より基本的な方法があります- / openwrt / trunk / package / base-files / filesディレクトリの内容を編集するには-いわゆる「スケルトン」があり、ターゲットFSを生成します。これはカーネルの再構築時に消去されません。 しかし、上記の方法とは異なり、FSの完全なイメージをターゲットマシン上にある形で見つけることは不可能であり、何かを不可逆的に台無しにするのは簡単なので、あまり行きませんでした。

一般に、この段階で、イメージの構成を開始できます。 そして、最初に行うことは、config / etc / config / fstabを編集して、起動時にシステムがフラッシュドライブをルートとして認識するようにすることです。
