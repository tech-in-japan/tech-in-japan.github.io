フル機能のRDPクライアント-FreeRDP

現代の世界では、マイクロソフト製品がほとんど不要になることは秘密ではありません。 ただし、多くの場合、Windowsよりも作業コンピューターでGNU / Linuxベースのシステムを使用する方がはるかに効率的であることがわかります。 これにより、管理が大幅に簡素化され、コストが削減されると同時に、より簡単にアクセスできる機能がユーザーに提供されます。 しかし、Windowsでのみ動作し、Linuxの世界には適切なアナログがないアプリケーションに対してはどうすればよいでしょうか？ 通常、このようなアプリケーションはごく少数であるため（そうでない場合は、稼働中のコンピューターにLinuxをインストールするのは意味がありません）、Microsoftのサーバーオペレーティングシステムで実行されているターミナルサーバーを使用するのが妥当な選択です。 さらに、Linuxは適切なバージョンのWindowsがないため、すべてのシンクライアントに最適です。



いずれにしても、Windowsターミナルサーバーに接続できる必要があります。 このために、MSは独自のリモートデスクトッププロトコル-RDPを開発しました。 ただし、最近まで、Linuxにはこのプロトコルを操作するための唯一のオープンクライアントrdesktopがありました 。 残念ながら、その開発は長い間停止しており、最新バージョンのWindowsとのやり取りで大きな困難を経験しています。



しかし最近、rdesktopプロジェクトは静かに分岐し、その結果、新しいオープンRDPクライアントであるFreeRDPが誕生しました 。 このプログラムの最初のリリースでは、rdesktopの既知の問題のほとんどがすぐに修正され、プロジェクトは引き続き活発に開発されています。 何らかの理由で、このような便利なアプリケーションの外観は無視されたので、この状況を何らかの形で修正し、Linuxの通常のRDPクライアントの存在について全員に伝えるために、この投稿を公開することにしました。 猫の下-FreeRDPの可能性の説明と、そのための優れたRemminaグラフィカルシェルについての少し。



FreeRDP



プロジェクトの公式ウェブサイトはwww.freerdp.comです



そこで、現在のバージョンの機能の説明と将来の計画を見つけることができます。 rdesktopとの主な違い：

  • 重要なコードのリファクタリングが実行されました。 ユーザーインターフェイスは完全に書き直され、メインライブラリから分離され、プラグインシステムが実装されています。
  • コードは、MicrosoftのRDP仕様に最大限に沿っており、詳細にコメントされています。
  • キーボードの操作が書き直されました-レイアウトの問題はもうありません。
  • プリンター、サウンド、その他の転送されたデバイスおよびサービスを使用して書き直されました。
  • Windows 2008ターミナルサーバーに接続する際のポインターとライセンスの問題を修正しました。
  • グラフィックキャッシュ（ビットマップキャッシュ）を実装しました。これにより、パフォーマンスが大幅に向上します。
rdesktopの破損および削除された機能：

  • 現在、COMポート転送サポートとスマートカードサポートは実装されていません。
  • RemoteAppの実装を支持してSeamlessRDPモードのサポートを削除しましたが、これも事実です。

レミーナ



さらに、リモートデスクトップに接続するための最も便利なグラフィカルマネージャーの1つであるRemminaは 、バージョン0.8からRDPクライアントとしてFreeRDPを使用するように切り替えました。



バージョン0.8は、Windows .rdpファイルのサポートも実装し、すべてのプロトコルが個別のプラグインになり、IPv6サポートが追加され、多くのマイナーな改善とバグ修正が行われました。



FreeRDPおよびRemminaの最新バージョンは、公式サイトからダウンロードできます。
