Ubuntuへの切り替えを検討している場合に知っておくべきこと

Linuxを目にしていなかったが、オープンソースに興味を持つようになった友人は、Ubuntu 10.10のリリース後、「幸福が必要かどうかを判断できるように、何がいい、何が悪い、どんな合併症、どんな微妙、vin7との関係を確立するか」を書くように頼みました。 最初は、「おそらく100人が既にテキストを書いているのに、なぜテキストを書くのか-すぐにGoogleで検索してリンクを送信し、特定の質問に答えます」と考えました。 しかし、その後、compiz設定のより細かい点を含む「ubuntuへの移行」の巨大なマニュアルのようなリクエストでは抜けてしまい、インターネットやubuntu.comとubuntu.ru（つまり、すべて）で「未定」の短いテキストをすばやく見つけることができないことがわかりました以前の投稿で書いたように、Habréではなく-少なくともHabréでそれができるようになったと思いました。 Ubuntariumの参加者はテキストから新しいことを学ぶことはまずありませんが、コメントに追加することができます（私はubuntugurのふりをしていないので、多くの重要なことを見逃しました）。 ubuntuへの移行」、彼らはここで幸せを見つけるでしょう。







私はすぐに、テキストに幾分簡略化されたバージョンの多くが記載されているので、本質がすぐに明確になるように警告します。 この投稿の後、誰かが検討することに決めた場合でも、彼はまだより詳細なテキストを読んでいるので、正しい言葉遣いを見つけます。 また、すべての人にUbuntuに切り替えるように説得しようとはしていません。 システムには欠陥があります。いくつかの欠陥は重大である可能性があり、本文ではすべてを公平に述べようとしました。



それが何であるか（主な利点）：

1.実用的：システムは無料です（ほとんどのプログラムと同様）。 もちろん、Windowsはよく知られた方法で無料で入手することもできます-しかし、突然それを見つけることができます、または（職場にインストールするときに）検証を行う人はそれを見つけることができ、最後にあなたの良心はそれを見つけることができます。 さらに、Windowsでは、すべてのプログラムを既知の方法で抽出することもできます。

2.実用的なプラス：ウイルスなし。 友だちが持ち込んだフラッシュドライブを安全に貼り付けて、（考えているものを含む）サイトにアクセスできます。 更新：コメントでは、コンピューター感染の理論的可能性が存在することが示されましたが、実際にはこれは非常にまれです。

3.実用的なプラス：Windowsよりもいくつかの操作を行う方が便利です（以下で詳しく説明します）。

4.オタクにとって実用的なプラス：柔軟な構成。 「ファイルを操作する」準備ができていれば、夢のシステムをほぼ手に入れることができます。

5.好奇心のためのプラス：より広い視野。 そして、定量的にだけでなく、定性的にも：1つの事柄に対する2つの異なるアプローチを見たとき、あなたはこれらの事柄自体を異なって見始めます。

6.美的プラス：プログラム、アラートなどのインターフェースは同じスタイルで設計されています。 さらに、多くの人々は、外部エフェクトと機能の両方のための「アドオンシステム」Compiz Fusionが大好きです（WindowsにはCompizの本格的なアナログはありません）。

7.イデオロギープラス：Microsoftの競争を支援します:) Microsoftに対して基本的には何もありません。Windows7は優れたOSですが、競争の事実により双方がより積極的に機能し、その結果、誰もが気分が良くなります。

8.イデオロギープラス：オープンソースコードの明るい理想。 確かに、これが何かを教えてくれるなら、あなたはプログラマーであり、プログラマーなら、あなたはすでにLinuxについて十分に知っています。 しかし、たとえそれを言わなくても、Linuxを使用するという事実によって、あなたは正当な理由を支持していることに注意してください。



「興味深いが、怖い」場合に知っておくべきこと：

1. Ubuntuをすぐに「完全に」インストールすることはできません。Ubuntuを最初に見て、気に入ったかどうかを理解できる3つのオプションがあります。 1つは、Windows内にUbuntuをインストールするWubiインストーラーを使用したインストールです。ハードドライブをパーティション分割する必要がなく（これにより、すべてを誤って破棄するリスクがなくなります）、簡単に削除できます。 2つ目は、システムをフラッシュドライブにインストールすることです（ギガバイトのいずれでもかまいません）：フラッシュドライブから起動し、コンピューターのハードドライブをまったく変更せずにUbuntuを見ることができます。 さらに、システムが気に入ったら、その後同じフラッシュドライブからコンピューターにインストールできます。 フラッシュドライブ上のシステムをメインシステムにすることもあります。そのため、どこかに来て他の人のコンピューターにフラッシュドライブを接続するとき、自宅で仕事をしますが、それをしませんでした。詳細はわかりません。 3番目の方法は、仮想マシンのWindows内で起動することです（この場合、Ubuntuはウィンドウ内のWindowsプログラムの1つとして起動します）。

2.「本格的な」インストールでは、UbuntuはWindowsに完全に近づき、コンピューターの電源を入れるたびに、どのシステムをロードするかを選択して交換可能に使用できます。 lifehacker.comには、UbuntuとWindows 7のデュアルブートに関する優れた英語の説明があります。誰かが本当に必要な場合は、ロシア語に翻訳します。



移行中にWindowsユーザーが引き起こす主な質問：

1.アプリケーションのインストール：Ubuntuでは通常、配布ファイルからではなく、リポジトリサーバーからインストールされます。 最初は珍しいことですが、大多数はこのアプローチをWindowsよりもはるかに強力に好むようになりました。 プログラムは文字通り「ワンクリック」で「次へ」を10回クリックすることなく、keygenからシリアル番号を入力します。 ほとんどのプログラムは無料ですが、有料のものもあります。

2.ファイルシステムの構造はWindowsとは著しく異なり、「CおよびDディスク」はありません。 慣れるには時間がかかりますが、多くの人にとって構造はより論理的に思えます。 最初に、NTFSパーティションとmp3プレーヤーなどの外部接続デバイスは、ルートから/ mediaフォルダーに移動し、各ユーザーの「個人用」フォルダーは/ home / usernameにあることを知ることが最も重要です。 （ここに別のプラスがあります：システムが完全に再インストールされると、「個人用フォルダー」を保存することができ、すべてのプログラムの設定が保存されます。）

3. Windows標準と互換性があります。 ここではすべてが異なります。 ICQおよびSkypeを介してワインダーと通信することは非常に可能です（ただし、Skypeクライアントは今では曲がっています）。 家庭レベルのdocおよびxlsでは、OpenOfficeで非常によく機能しますが、ここでは複雑な書式設定を含む大きなドキュメントはMicrosoft Officeのようには見えない場合があります。Windowsの科学者にゆがみを見せたくない場合は、Linuxではなく卒業証書を書くことをお勧めしますテーブル。 さまざまな状況でエンコードに問題があります（たとえば、zipアーカイブに配置されたファイルの名前にキリル文字が含まれ、mp3ファイルのタグにキリル文字が含まれます）。 ほとんどの場合、これに対処する方法がいくつかありますが、場合によっては非常に便利です。部分的には「松葉杖」です。 これが転倒している人を心理的に慰めるなら、Ubuntuはエンコードに関するすべての問題を解決するために設計されたUnicodeを使用し、Windowsがまだ完全に切り替わっていないために問題が発生することを知っておく必要があります。 LinuxでWindowsアプリケーションを起動するWineもあります。「重い」プログラムの場合、これは困難または不可能な場合がありますが、ささいなことではるかに優れています（たとえば、Windowsアーカイバを実行して、キリル名で問題なくzipを解凍できます）ファイル）。

4.アイロンサポート。 正直に言うと、すべてのハードウェアメーカーがLinuxユーザーについて考えているわけではないため、一部のハードウェアで問題が発生する可能性があり、一部の周辺機器用の公式ドライバーがない場合があります。 前述のフラッシュドライブを使用して、システムがハードウェアで正常に動作するかどうかを事前に確認できます。公式にサポートされていない周辺機器（Yotaモデムなど）の一部については、ドライバーはLinuxoid自体によって書き込まれます。 確かに特定の鉄のサポートが必要であるが、手元にない場合、実際にチェックするために-google、Linuxはすでにすべての人気のある鉄Linuxシステムについてサポートされていますか？ iTunesはありません-iPodやiPhoneの代替ソリューションはいくつかありますが、もちろんこれは同じものではありません。 更新：コメントでは、フラッシュドライブからのシステムの通常の動作中に、フルインストール後に問題が発生することがあると、彼ら自身が言っていることがありますが、私自身はこれに遭遇していません。

5. mp3を含むいくつかの一般的なもののサポートは、特許によって制限されており、オープンソースルールに準拠していないため、デフォルトではシステムに含まれていません。 これは、単一のubuntu-restricted-extrasパッケージをインストールすることで解決します。

8. Ctrl + Shiftでレイアウトを切り替えると、Ctrl + Shift + Xの形式のホットキーが機能しません。 Linuxでは、リリースではなくキーを押したときにホットキーが起動し、レイアウト変更コマンドでCtrl + Shift + Xの先頭を使用します。 回避策はありますが、最初はAlt + Shiftに切り替える方が簡単です（一般的に、CapsLockをあるレイアウトに、Shift + CapsLockを別のレイアウトに配置することをお勧めします。切り替える前にどちらをオンにするかを確認する必要はありません）。

一般的なUbuntuとLinuxに関する主な神話：

1. Linuxは複雑であり、システム管理者向けであり、普通の人向けではありません。

基本的なタスク（サーフィン、ミュージックビデオなど）はUbuntuで実行されますが、Windowsほど複雑ではなく、場合によっては簡単です（上記のプログラムのインストールを参照）。 はい、コンソールはありますが、主なタスクのほとんどはグラフィカルインターフェイスを使用して実行できます。たとえば、VLCビデオプレーヤーをインストールするには、コンソールに「sudo apt-get install vlc」と入力するか、「Ubuntu Application Center」に移動して「 vlc「、それを選択して、「インストール」をクリックします。 Linuxは、コンソールとホットキーを頻繁に使用します。これらがないと、システムで非常に不便であるためではなく、それらが非常に便利になるためです（時間をかけて学習する場合）。 グラフィカルインターフェイスなしで、1つのコンソールで、通常とは異なる何かを実行したい場合にとどまることができます-ただし、実行したいタスクの名前をgoogleで検索すると、多くの場合、フォーラムにコンソールにのみコピーできる既製のコマンドがあります意味がわからない場合 （まあ、公平を期すために、Windowsでの非定型のタスクは多くの場合、完了がさらに困難です。）

2. Linuxは見苦しい、それはシステム管理者向けであり、美への渇望を持つ少女向けではありません。

まず、Ubuntuは90年代の角張ったコンソールモンスターのようには見えません。 多くは、MacOSを部分的に模倣しているとさえ信じています。 第二に、Ubuntuでは、Windowsよりもシステムの均一性がはるかに高くなります。アプリケーションは同じ設計ルールに従い、同じ方法でアラートを表示します。 統一された外観には例外があります（通常、これらは他のシステムと同じように見えるクロスプラットフォームアプリケーションです）が、一部のスキンをプルすることもできます。 第三に、外観を最前面に置いて、Compiz Fusionを使用すると、UbuntuをWindowsよりも見栄えよく見せて、自分用に外観をカスタマイズできます（たとえば、 このようなビデオは最も指標ではありませんが、本質を理解できます）。

3. Linuxでは、必要なプログラム/ゲームはありません。

必要なものに依存します。 すべての基本的なタスクを実行するためのプログラムがあります。 特定-特定の状況に依存します。 AutoCADは実際にはそうではありません（ただし、AutoCADの代替案はより良くなり、互換性が向上していると言われています）。また、ほとんどのオートキャダーは、Linuxを愛していても、2番目のWindowsシステムを動作させます。 良いものを含むゲームがありますが、新鮮な大ヒット作はありません。 （ただし、Windowsではブロックバスターが次第に少なくなり、主にコンソールで使用されています。）
