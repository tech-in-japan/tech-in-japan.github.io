Operaで複数のプロキシを使用する

内部Webリソースを含む企業ネットワークおよびホームネットワークでは、よく知られているオプション「ローカルアドレスにプロキシを使用しない」よりも機能的なツールが必要になる場合があります。



ただし、企業/ホームネットワークが複数の（複数の）プロバイダーに接続されている場合、または最も頻繁にアクセスされるリソースのトラフィックを節約する必要があり、多くのプロバイダーが無料で利用可能なWebリソースを持っていることを考慮する場合、プロキシが正当化されます。



次に、Operaブラウザで事前に構成されたいくつかのプロキシを使用すると便利です。winおよびnixシステムの両方に追加のモジュールや他のソフトウェアをインストールせずにこのタスクを実行できます。



執筆時点でのOperaブラウザの現在のバージョンまでは、設定でGUIを介してマルチプロキシを使用する方法はありません。 ただし、ファイルoverride.iniがあります。このファイルは、デフォルトでOperaユーザープロファイルフォルダーにあります。 ブラウザでの複数のプロキシの使用。



この設定ファイルの場所は、Operaで次のアドレスを開くことで確認できます。

opera：config＃Overrides 、「Overrides File」はこのファイルへのフルパスを示します。



このファイルの設定の変更を開始する前に、Operaを閉じて、PCメモリから完全にアンロードされるまで待ってください！



override.iniファイルの構造を考慮してください。



Opera Preferences version 2.1



[Overrides]

site1

...

siteN



[site1]

....



[siteN]

....







[オーバーライド]セクションには、設定が「デフォルト」と異なるサイトのドメイン名がリストされます。 次は、ドメイン名とそれらの設定のセクションです。



例1.専用のキャッシュプロキシサーバー（IP 192.168.1.1、ポート3128）を使用して、マップサービスへの外部トラフィックを最小限に抑えます。



[Overrides]

maps.yandex.ru



[maps.yandex.ru]

Proxy|Use HTTP=1

Proxy|HTTP server=192.168.1.1:3128

Proxy|HTTPS server=192.168.1.1:3128

Proxy|FTP server=192.168.1.1:3128









このルールは、* .maps.yandex.ruという形式のアドレスにも適用されることに注意してください。



例2.ネットワークは2つのインターネットプロバイダーに接続されており、それぞれにアクセスするには、個別のプロキシサーバー（192.168.1.1lla128と192.168.1.2lla128）を使用します。 、および* .site.comから2番目の形式のアドレス。



[Overrides]

site.ru

site.com



[site.ru]

Proxy|Use HTTP=1

Proxy|HTTP server=192.168.1.1:3128

Proxy|HTTPS server=192.168.1.1:3128

Proxy|FTP server=192.168.1.1:3128



[site.com]

Proxy|Use HTTP=1

Proxy|HTTP server=192.168.1.2:3128

Proxy|HTTPS server=192.168.1.2:3128

Proxy|FTP server=192.168.1.2:3128









例3.ネットワークには1つのプロキシサーバー（192.168.1.1lla128）のみがありますが、トラフィックに制限のないプロキシアカウントがありますが、サービスが必要なサイト（company.ru）のリストには制限があり、検索エンジンの個人トラフィッククォータもありますそして、インターネットの残りの部分。 もちろん、限られた個人的なトラフィックを公式のニーズに費やすことは望ましくありません。 ブラウザのプロキシ設定を絶えず変更することも非常に不便です。



この場合、プロキシ192.168.1.1の場合、ローカルのDNS名を考え出し、それをOSのホストファイルに書き込みます。

192.168.1.1 proxy.local







Operaのデフォルト設定では、プロキシ192.168.1.1lla128の使用を設定します。

次に、override.iniファイルを編集します



[Overrides]

company.ru



[company.ru]

Proxy|Use HTTP=1

Proxy|HTTP server=proxy.local:3128

Proxy|HTTPS server=proxy.local:3128

Proxy|FTP server=proxy.local:3128









トリックは、Operaが使用される各プロキシのログインとパスワードの設定を保存することです。 プロキシ192.168.1.1 {128の場合、個人のユーザー名/パスワードを入力し、* .company.ruアドレスを開こうとすると、proxy.local：3128の無制限のkakbyアカウントのユーザー名/パスワードを指定します。



結論として、いくつかのプロキシサーバーの使用が必要になる理由を再度リストします。

  • 複数のインターネットプロバイダーを使用する。
  • Google、Yandexなどの大量のグラフィックを含むマップサービスの積極的な使用。
  • 選択したサイトに無制限のプロキシアカウントを使用します。
