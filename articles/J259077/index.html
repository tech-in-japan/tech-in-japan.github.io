暗号化の仕組み



ティムールにはスーツケースと鍵があります。 彼は空のスーツケースを友人のサーシャに渡します。





サーシャは自分のスーツケースと鍵も持っています。 彼はそれらのコピーを作成し、ティムールのスーツケースにそれらを隠します。





なぜなら スーツケースは閉じた状態に戻り、ティムールだけが開くことができます。 スーツケースの鍵を持っているのはティムールだけです。





ティムールはスーツケースを開け、スーツケースとサーシャの鍵のコピーを取り出します。





ティムールはスーツケースと鍵を必要としなくなりました。 彼はサーシャが彼に与えたものを使うことができます。





これがRSA暗号化アルゴリズムの仕組みです。 そして、これは熱直腸暗号解読器を使用した暗号アルゴリズムの迅速な解読です。





問題の技術面



暗号化アルゴリズムとしてRSAを選択しました。 アルゴリズムは約30年前という事実にもかかわらず、JSで最初に見つかったライブラリは非常に低品質でした。 それらは1つのプロジェクト専用に作成されたようです。 困難と苦痛で、私はそれらを私のプロジェクトのニーズに適応させることができましたが、タスクは典型的でした：

  • キーを生成する
  • 公開鍵でテキストを暗号化する
  • 秘密鍵でテキストを解読する




また、同様の拡張機能の半分で、いくつかのフレーズを考え出す必要があることや、何らかの形でそれらを伝える必要があることにも驚きました。 かなり昔のように思われますが、人々はオープンチャネルを通じて暗号化を送信し、プロセスは自動化に役立ちます。



ソーシャル解析の問題 ネットワーク



vkontakteから始めました。 最初の難しさは、ダイアログごとに独自の入力フィールドとメッセージ表示フィールドが作成されることです。 この問題は、対話者のIDを取得することで解決しました。



次に、「Submit」ボタンとEnterボタンの押下をインターセプトする必要がありました。 問題は、連絡先が最初にイベントをハングさせてから、プラグインが初期化されることです。 状況を修正するために、document.bodyにハンドラーを追加し、その後、インターセプトされたすべてのイベントをフィルター処理しました。 元のメッセージを暗号化されたメッセージに置き換えるには、最初にメッセージを受信する必要がありました。



同級生のサイトでは、これだけでは不十分であることが示されました。 最大メッセージ長は3000文字で、暗号化されたキーペアの長さは3800です。さらに、base64への変換により通常のメッセージも長くなります。 メッセージをパーツに分割し、それらに署名（id、現在のインデックス、パーツの総数）してから組み立て直すメカニズムを作成する必要がありました。 TCP / IPのパケットのようなものが判明しました。



次に、「メッセージを送信」イベントを生成する方法を学ぶ必要がありました。 スクリプトは、独自にパケットを送信する必要がありました。 VKontakteでは、グローバルなIM.send（）メソッドを呼び出すことで、問題を非常に簡単に解決しました。 拡張機能のスコープからサイトスコープに到達するには、SCRIPTタグを追加し、そこにメソッド呼び出しを記述する必要がありました。 例： 

<script>IM.send()</script>


サイトと拡張機能のDOMは同じであり、グローバルスコープは異なることを明確にする必要があります。 クラスメートには同様の方法が見つかりませんでした。 Guysはコードを最小化し、グローバルを生成しません。 [送信]ボタンでイベントをエミュレートする必要がありました。 

 var event = new Event("click", { bubbles: true, cancelable: true }); button.dispatchEvent(event);


テキストが暗号化され、部分に分割され、送信および収集されると、サイトに表示される新しいメッセージと古いメッセージを区別する必要がありました。 そして、VKontakteでは、クラスメートでは、日付または一意のIDで署名されています...私は思った...私はそんなに間違ったことはなかった。 その後、「フローティング」バグが発生しました。 これらのすべてのラベルは、2つの異なるメッセージで何らかの形で一致することが判明しました。 したがって、誰がすでに処理され、誰が処理されなかったかを知るために、すべてのDOMノードに属性を割り当て始めました。



しかし、奇跡はそこで終わりませんでした。 クラスメートがメッセージを公開し、それを解読されたものに置き換え、ある種の魔法が起こり、数ミリ秒後に、クラスメートは元のテキストを同じHTML要素で再び書きました。 メッセージノードのクローンを作成し、そこに復号化を記述し、元の要素を非表示にする必要がありました。 元のノードは非表示（表示：なし）でしたが、削除されませんでした。 どのように、そしてなぜそれが書き直されたかは知りませんでした。 HTML要素が削除された場合、クラスメートのスクリプトを破壊する危険がありました。



ソーシャル解析の最後の謎。 ネットワークは、複数のユーザーが1台のコンピューターからログインした場合、ダイアログのキーペアを別の場所に保存する必要があるという事実から成り立っていました。 この問題は、現在のユーザーのIDを抽出することで解決しました。 ユーザーIDを知ることで、localStorageに個人エントリを作成できるようになりました。
