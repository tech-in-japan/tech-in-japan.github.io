ロシアのUXデザインと「顕微鏡下」のCIS

おそらく、このトピックの主な目的に関する小さな教育プログラムから始めます。 つまり、UXデザインの概念からです！



英語から翻訳すると、ユーザーエクスペリエンスとは「ユーザーエクスペリエンス」を意味します。 設計者が開発したインターフェイスを操作するときにユーザーが感じる印象、簡単さ、目標を達成できるかどうか。



UXデザインは、次の分野を含む多面的な概念です。



  • 情報アーキテクチャ;
  • ビジュアルデザイン;
  • ユーザビリティ;
  • インタラクティブなデザイン;
  • 人間とコンピューターの相互作用。






現在、UXデザインは企業によって完全に使用されているわけではありませんが、大きな展望があり、近い将来に評価されるでしょう。



UXデザイナーになる方法と開始点



あなたがロシアまたは近隣諸国に住んでいる場合、あなたは2つの方法でユーザーエクスペリエンスデザインを学ぶことができます：あなた自身または特別なコースで。



これは、教育機関がUXデザインを教えていないという事実によるものです。 したがって、この分野の情報は、インターネット（記事を読む、ビデオを見る、デザイナーとチャットする）から取得するか、コースを卒業することができます。



後者については、UXデザインの基本だけを教えてくれるので、仕事を見つけるのに役立ちますが、実務家にはなれません。 ユーザーエクスペリエンスデザインの実践的なスキルを身に付けるには、実際のプロジェクトに参加する必要があります。



最初は、単純なタスクを実行し、知識と能力のレベルを上げて、水準を上げ、より複雑で責任のある計画に移行する方が良いでしょう。



UXデザインの初心者が良い仕事を見つけるのは非常に難しいという事実にもかかわらず、プロジェクトの最終結果を危険にさらすことに同意する企業はほとんどないため、それは可能です。 人材が不足しているため、最善を尽くすことができます。 もちろん、最初の仕事は少しのインターンシップですが、成功すれば、新しいUXデザイナーは優れた給料で良い地位を得ることができます。



ただし、ユーザーエクスペリエンスデザインのアプリケーションには落とし穴がないわけではありません。 これについて話す人はほとんどいませんが、問題が存在し、まだ解決されていません。



UXデザイナーはどのような課題に直面していますか？



完全なユーザーエクスペリエンスデザインレッスンには、多くの能力が必要です。 後者の実装には、多大な時間と知的費用が必要です。 これは若い専門家にとって特に深刻であり、背後には基本的な知識の小さな荷物があり、これらのプロジェクトに参加した経験はほとんどありません。



多くのデジタル製品会社は、UXデザイナーの責任を制限しています。



2つの主な方向があります。



  1. 相互作用シナリオと情報アーキテクチャの開発。
  2. ユーザーインターフェイスのデザインとビジュアルデザイン。


同時に、以下は職務から除外されます。



  1. ビジネスおよび対象読者の分析。
  2. 戦略の開発と分析。
  3. A / Bテスト;
  4. 競合他社の製品の分析。
  5. 変換を増やすための設計変更のキュレーション。


国内企業の主な焦点は、伝統的にUIデザインです。 これは、作業の結果をすぐに表示し、クライアントと開発者に表示して、将来使用できるため、理解できます。 UX分析に関しては、その重要性は明確に表現されていないため、企業によって100％評価されるわけではありません。



ほとんどの場合、顧客は、通常のグラフィックデザイナーやWebデザイナーになることが多いユーザーインターフェイスデザイナーを特に探しています。 また、プロジェクトを実装するプロセスでは基本的なUXテクニックを使用する必要があるため、UXについてほとんど知識がありませんが、履歴書でUX / UI Designerを示しています。



実際、部外者にならないために、実際のUXデザイナーは可能性を制限し、UIに集中する必要があります。これは、企業が実際に支払いたいと思うより人気のある方向です。



残念ながら、活動範囲のこのような深刻な縮小は、デザイナーがユーザーエクスペリエンスデザインのポイントを見ることをやめるという事実に貢献します。 UIデザイナーに関しても同じ結論が得られます。



私の顔のUXデザイナーが実存的危機をどのように克服したか



6年間、Webサービスとモバイルアプリケーションの設計に取り組んできました。 私の仕事はオフィスとリモートの両方で行われました。 しかし、企業がUXデザイナーを彼の責任にどのように制限するかを考えると、私は自分の可能性が明らかにされず、日常的に行き詰まるだろうという感覚に常に悩まされていました。



上司と顧客の要件に適応し、確立された期限の内訳を回避するために、私の想像力と創造的な衝動を制限する必要がありました。



実存的危機のもう1つの理由は、UI専門家との競争でした。 印象的な経験を積んだ人たちは、技術的な要件に従って、できるだけ早くモックアップ/ワイヤーフレームをクライアントに提供できました。 もちろん、彼らの決定が創造性を誇ることができず、論理的に矛盾していることもありましたが、UIデザインの優れた研究と迅速な結果は、顧客に受け入れられました。



傍観者にならないように、私は彼らのアプローチを適用し始めました。そして、長い間熟考する代わりに視覚的記憶を使いました。 しかし、実践が示しているように、UXデザインの全体的な意味はこれから失われました。これは、私のプロEGOに実際に触れ、休息を与えませんでした。

以下は私の最後の作品の一つです。 これまでのところ、開発は本格的ですが、まもなくオフラインスペシャリストのサービスを検索および注文するための本格的なオンラインサービスになります。







毎日、さまざまな観点からプロジェクトを見て、意識の遠い隅に保存されているテクニックを適用しようとしました。 私の考えは急いで流れ、結果がどうなるかを考えました。 個々のUXテンプレートを視覚化し、精神的に情報で満たそうとしました。



最近のある日、私は突然、ユーザーエクスペリエンスデザイナーの仕事が、どんな行動よりも知的思考にあることに気付きました。 読者の中には、私が「証拠のキャプテン」だと言う人もいるかもしれませんが、あなたが雇用されている場合、これにどれだけの時間を費やし、結果にどのくらいの影響を与えるかを顧客に説明するのは難しいので、長い考えをあきらめる必要があります。



危機を克服するために、私は自分の好きなことをして、自分のプロジェクトを作成する作業を開始する必要がありました。 ここでは、誰も私を空想の飛行に制限せず、時間をかけて急いでいませんでした。 これはUXデザイナーの夢ではありませんか？



将来のUXデザイナーのための5つのヒント



長年の実践を通じて、ユーザーエクスペリエンスデザインの経験を十分に積んできたため、UXデザインを学びたいが、どこから始めたらよいかわからない人に5つの重要なヒントを提供したいと思います。



ヒント＃1：自習とコースの間では、2番目を優先することをお勧めします。

コースは、基本的な実践を学び、基本的な知識を得る機会を提供します。これは、その後、実際のプロジェクトに参加するときに役立ちます。



ヒント2：GoogleのマテリアルデザインガイドラインとAppleのヒューマンインターフェイスガイドラインをご覧ください。

それらから収集する情報は、インターフェイスの設計における多くの間違いを回避するのに役立ち、デザインシェルとのユーザーインタラクションの基本原則を理解するのに役立ちます。



ヒント＃3：デジタル製品を毎日探索し、インターフェイスを設計して描画します。

これはUIデザインに関連している可能性が高いという事実にもかかわらず、後者とUXの織り込みは疑いの余地がありません。 さらに、視覚記憶の開発に貢献し、あなたのアイデアを反映するインターフェースソリューションの検索を加速します。



ヒント＃4：UXデザインの意味と利点を雇用者に説明して示すのを怠らないでください。

多くの企業は、ユーザーエクスペリエンスデザインとは何か、そしてそれが何のために必要なのかを十分に認識していません。 あなたがそれについて有能に語り、実例を与えるなら、雇用主はあなたの立候補に確かに興味を持ち、履歴書を検討します。



ヒント＃5：現在のタスクに不必要に焦点を合わせないようにしてください。

ルーチンは常に創造的な活動に悪影響を与えているため、より多くの本を読んだり、会議に参加したり、独自のプロジェクトを作成したりします。 これにより、公務の均一性にとらわれず、実存的な危機を回避することができます。
