Andrey Ershovによるjug.msk.ruのJavaとScalaのFutureの進化

2017年8月31日、CROCオフィスはAndrei Ershovの参加を得てモスクワJava開発者コミュニティの会議を開催しました。AndreiErshovは、 Future非同期コア開発プリミティブの進化とJavaおよびScalaプログラミング言語での実装について話しました。







スピーカーについて



Andreyは、電気通信の分野でソフトウェア製品を開発しているDino Systemsのプロジェクトマネージャー兼アーキテクトです（クライアントの1人はCourseraです ）。



JUG.ruが開催する会議のアクティブスピーカー。会議参加者の間で技術的に複雑で興味深く、人気のあるレポート。

CRDT （ 競合のない複製されたデータ型 ）に関する非常に良いレポート。 会議で発表されたCodeFreezeは、 JPoint 2017からのレポートの補足および拡張バージョンです。



その他のリンク： GitHub 、 Twitter 。



レポートについて



Andrey Kogunが会議を開始し、今後の会議DevOops （10月20日）、 SmartData （10月21日）、 Joker （11月3〜4日）を思い出します。







レポートの最初の部分で、Andrei Ershovは、 Java 5でのインターフェースとしての登場から、代替（ Guavaでの ）および標準実装（ Java 8での CompletableFutureである）の開発まで、 Futureの歴史について詳しく説明しました。 JavaおよびScala言語でのFutureの実装と使用のさまざまな側面について詳細に比較します 。







ブレーク後の2番目の部分は、非同期コンピューティングでの例外のロギングと処理に専念しました。 最後の和音は、レポートのトピックに関する記事へのリンクの広範なリストのプレゼンテーションでした。 リスナーから寄せられた質問に対するAndreiの適格な回答は、重要な追加でした。







レポートのプレゼンテーションと 会議の写真 。 ビデオはまもなくYouTubeに表示されます （ VKおよびG +で追加の発表があります）。
