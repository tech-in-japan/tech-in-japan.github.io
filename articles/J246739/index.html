低速Web向けの設計

Slow Web / Fast Webの概念に関するBen Roweは、Fast Food / Slow Foodの概念に類似しています。 Slow Webでは、ユーザーが時間と注意を管理します。 どこにでも急ぐ必要はありません。理由もなく気を散らす人はいません。 Slow Webの基本原則：

  1. 機能が少ない方が良いですが、より良い相互作用があります。
  2. 不要な通知はありません。
  3. 不必要に切迫感はなく、ユーザーにリラックスしてもらいます。
  4. ユーザーを不必要にオンラインにしないでください。 それらを現実の世界に送ってください。
  5. 最後に、ユーザーを尊重します。
