雲の中の幽霊、またはWindows AzureにGhostをインストールする

今日、多くのブロガーは、Wordpressが過剰な機能で過負荷になりすぎており、ブログプラットフォームから完全で複雑なCMSに変わっていると考えています。 これは決して悪いことではありません-現在ネットワーク上で利用可能なさまざまなプラグイン、モジュール、テーマにより、ほぼすべての目的でプラットフォームの機能を使用できます。 ただし、この種類にはコインの裏側もあります。 モジュールの山は、「クリーン」なWordPressでも高速を誇ることができないにもかかわらず、サイトのパフォーマンスに悪影響を及ぼします。



ゴースト



しばらく前に、 GhostというプロジェクトがKickstarterスタートアップサイトで立ち上げられました。そのスローガンは「単なるブログプラットフォーム」（ブログのみのプラットフォーム）でした。 多くの人々がこのアイデアを気に入っており、プロジェクトは元の25,000ドルではなく、ほぼ200,000ドルを調達しました。 現在、プロジェクトは活発に開発されており、バージョン0.3.3はすでに公式Webサイトからダウンロードできます。



私は、他の多くの人と同様に、このプロジェクトに非常に興味がありました。 まず、ミニマルなデザインが好きです。 第二に、Wordpressは私のニーズに合わせて少し冗長であり、仕事に少し馬鹿げています。 そして第三に、最近Ghostが話題になっているNode.js上でGhostが開発されているという事実に惹かれました。 私は、 「単なるブログプラットフォーム」とはどのようなものであるかを実際に試してみることにし、メインブログへの追加としてWindows Azureに自分でインストールすることにしました。



Windows Azureでサイトを作成する



Ghostを直接使用する前に、ホストするサービスWebサイトを作成する必要があります。 これは、通常どおりWindows Azure管理ポータルを介して行われます。 [新規]メニュー（ページの下部にあるプラス記号）で、[計算]-> [Webサイト]-> [クイック作成]を選択します。 現在の段階では、コンポーネントが事前にインストールされていない空のサイトが必要です。 必要なものはすべて、さらにお届けします。





サイトを作成したら、いくつかの設定を行う必要があります。 まず、Ghostが正しく機能するために、耳で特定のフェイントを作る必要があります（詳細については、以下を参照してください）。 [構成]タブに移動して、アプリの設定セクションを見つけます。 その中に次の行を作成する必要があります：KEY = NODE_ENV、VALUE = production。





次に、稼働中のコンピューターからクラウドにデータを転送する方法を構成する必要があります。 これを行うために、Windows AzureはTFSやGitなどのバージョン管理システムをサポートしています。 Ghostプロジェクトはオープンソースであり、GitHub自体から提供されるため、WebサイトにGitリポジトリを設定して作業します。 これは簡単に実行できます。[クイックアクセス]タブ（クラウドと稲妻の場合）には、[ ソース管理の統合]項目があります。





[ソース管理から展開を設定]をクリックし、[ ローカルGitリポジトリ]を選択します 。





使用可能なオプションの中には、TFSや他のバージョン管理システム（Dropboxを含む）もありますが、ここではGitを選択します。



これでWindows Azureクラウドの構成が完了し、最も興味深い部分であるGhostのインストールと構成に進むことができます。



作業環境のセットアップ



Ghostをインストールする方法は2つあります。 最初の最も簡単な方法は、プロジェクトの公式ウェブサイトから既にコンパイル済みのファイルを直接ダウンロードすることです。 この方法は、ユーザー側で追加のアクションを必要としないため便利です。 ただし、プロジェクトは開発のアクティブフェーズにあるため、重要な修正が含まれていない最新バージョンのプロジェクトをダウンロードしないというリスクがあります。 私の場合はそうでした。 プロジェクトサイトからアーカイブをダウンロードすると、完全にインストールされてローカルに起動される一連のファイルを取得しましたが、クラウドでの作業はまったく望みませんでした。



したがって、現時点では、将来問題が発生しないように、最初はより長い道のりを進める方が良いと考えています。 長い道のりは、完成したパッケージをGhostソースから個別に組み立てることです。 そして、最初は非常に複雑に聞こえますが、実際にはすべてに10〜15分余分に費やす必要があります。



したがって、最終的に必要なこと：



  1. Node.jsをインストールします。 配布は公式のNode.js Webサイトで行われます。 通常のWindowsプログラムとしてインストールします。 主なことは、PATHに登録されていることを確認することです。これにより、後でシステムでフルパスを指定せずにnodeコマンドを使用できます。
  2. gitのクライアントをインストールします。 Windows用のコンソールクライアントはからダウンロードできます。 サイト また、外部のPATHに必要なパスを書き込むためにそれを探します。そうしないと、特別なGit Bashコマンドラインから作業する必要があります。
  3. Gruntをインストールします。 これは、JavaScriptプロジェクトをコンパイルするためのユーティリティです。 Ghostソースから発行する既製のパッケージを作成するために必要です。 Gruntをインストールするには、コマンドラインからnpm install grunt-cli --g



    コマンドを実行するだけです。 npmプログラムにはnodejsが付属しており、本質的にはDebian Linuxのapt-getのようなパッケージマネージャーです。 仕事に必要なすべての拡張機能をダウンロードしてインストールできる便利なもの。
  4. Gruntが機能するためには、Rubyを追加インストールする必要があります。 公式ウェブサイトからダウンロードすることもできます。
  5. Gruntには、BourbonとSassと呼ばれるRuby用の2つの拡張機能も必要です。 1つのコマンドgem install bourbon



    で両方をgem install bourbon



    できます。




これで、作業環境のセットアップが完了しました。



ゴーストの準備



ゴーストの準備は2つの段階に分かれています。 最初の方法は、GitHubの公式リポジトリからファイルを取得してコンパイルすることです。 2番目は、Windows Azureでブログを開始するための構成およびその他の最終構成です。 さあ始めましょう！



GitHubのGhostリポジトリはこのアドレスにあります 。 コンピューター上のプロジェクトファイルを取得するには、Git Bash（手順2の前の手順でインストールした）を開き、このプロジェクトを保存するフォルダーに移動して、コマンドを実行します。 



  git clone https://github.com/TryGhost/Ghost.git


しばらくすると、必要なすべてのファイルを含むGhostリポジトリのローカルコピーがマシンに表示されます。 さらに、アクションは次のとおりです（＃の後のアイデアはコメントと見なされ、実行できません）。

うなり声の初期化
うなり声
 ＃これらのコマンドはプロジェクトを「コンパイル」し、インストール用のパッケージを形成します


ここで、別のフェイントを作成する必要があります。 実際、Azureで実行されているNode.jsサイトは、デフォルトでserver.jsというファイルを開始ページと見なします。 Ghostでは、開始ページはindex.jsであるため、これらの違いを自分で解決する必要があります。 これは非常に簡単です。 メモ帳で、Ghostのルートディレクトリにserver.jsファイルを作成し、そこに1行書き込みます。 



  var GhostServer = require（ './ index'）;


変更を保存します。 これで、Windows AzureはGhost構成ファイルを正しく検出できるようになりました。



これで、Azureで新しいブログを公開する準備がほぼ整いました。 構成を構成するためだけに残ります。 ここではすべてが非常に簡単です。 Ghostのルートディレクトリにconfig.example.jsというファイルがあります。 名前をconfig.jsに変更し、編集用に開きます。

生産ブロックを探していることに注意してください。 これは、投稿の最初にDEV_ENVパラメーターの値を設定するための構成です。 ここでは、ブロックの最初のurlパラメーターと最後のportパラメーターの2つのことに興味があります。 最初のURLは、ブログが存在するURLに変更する必要があります。 Windows Azureの場合、デフォルトでは、このURLは<YOUR-SITE> .azurewebsites.netのようになります。 Webサイトのドメインを手動で登録する場合は変更できますが、これは別の話のトピックです。 さて、ポート値は行process.env.PORTに変更されます （引用符なし）。 合計すると、次のことがわかります。

Windows Azureでサイトを公開する



これで、最終アクション、つまりサイトの公開の準備が整いました。 これを行うには、Git Bashでいくつかのコマンドを実行します。 



  git add。
 git commit -m "最初のゴーストパブリッシュ"
 git remote add azure https：//glamcoder@ghost-test-glamcoder.scm.azurewebsites.net：443 / ghost-test-glamcoder.git
 git push azure master


まず、行ったリポジトリへのすべての変更を保存しました。 その後、特定のコメントを付けてコミットしました。 3行目では、リモートGitリポジトリへのリンクを作成しました。 このリポジトリのアドレスは、Windows AzureのすべてのWebサイト専用に作成され（最初にこれを行いました）、[展開]タブで確認できます。 最後のコマンドは、Windows Azureサーバー上のすべてのファイルを送信します。 公開プロセス中にサーバーでさまざまなコマンドがどのように実行されるかを確認できます。 終了すると、サイトがすでにクラウドで動作していることが確認できます。次のようなものが[展開]タブに表示されます。





ブログを初めて設定するには、リンク<YOUR-WAY-TO-BLOG> / ghostに従ってください。管理者のユーザー名とパスワードを入力するよう求められます。 その後、新しいブログのインストールとセットアップを完全に完了し、作成を開始できます！

ゴーストの印象



メインのブログ（ http://ghost.glamcoder.ru/ ）と並行して、Ghostを2番目のブログとしてインストールしました。仕事の印象を共有したいと思います。



プロから始めましょう。 まず、私はマークダウンが投稿を書くためのマークアップ言語として本当に好きです。 非常にきれいで、編集が簡単で直感的です。 そして、彼はよりオタクで、それが彼の魅力を増しています。 これは、MS Wordと比較したLaTeXのようなものです。

第二に、ゴーストはスマートに動作します。 Wordpressよりもはるかに高速で、プラグインによっても絞められません。 これはNode.jsのメリットではないことは明らかですが（おそらくそうかもしれませんが）、プロジェクトの若さであり、Neroのようなマルチコンバインになっていません。 しかし、開発者は、決してそうなることはなく、常にクリーンでシンプルなままであると主張しています。 見てみましょう。

第三に、私はそのデザインが好きです。 すべてがシンプルで、理解できるほど、それ以上のものはありません。 ミニマリズムは最高です。

これはおそらく、Ghostのすべての利点ではありませんが、少なくとも最も顕著なものです。



マイナスでは、すべてがそれほど単純ではありません。 私の最大の不満は、ブログがインターフェースによってほとんどカスタマイズできないことです。 つまり すべての設定のうち、ブログの名前と説明、およびメインページの背景画像のみを編集できます。 すべて、これ以上のものはありません。 メニュー、ウィジェット、ソーシャルサービスの出版物はありません。 ネットワーク。 投稿の下にコメントすらありません。 言い換えれば、ブログにはテキストの作成に当てはまらないすべてのものが実際には欠けています。 それにもかかわらず、私はそれをマイナスと呼ぶことはできません。 これは、我慢しなければならないプラットフォームの機能です。 必要な修正は、自分で書くことができます。 文字通りの意味で-ソースコードを開き、そこに必要なモジュールを追加します。 たとえば、標準ビューテンプレートを編集してYandex.Metricaのコードを挿入しました。 Disqusからのコメントを接続するために同じことを行うことが提案されています。 創造性の完全な自由！



最後に言いたいこと。 あなたがオタク、プログラマー、技術者（必要なものに下線を引く）で、ミニマリズムが好きなら、Ghostはあなたにアピールするでしょう。 最小限の労力ですぐに動作し、すぐに使えるようにしたい場合は、通り過ぎてください。 すぐに、Ghostのメンバーはブログで独自のホスティングを開始することを約束します。そして、おそらく、それはあなたにとって便利だと思われます。 しかし今、Ghostには多くの手作業が必要です。
