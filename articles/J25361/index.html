Fedora 9リリース！

Fedora Linux 9オペレーティングシステムの新しいバージョンのリリースが発表されましたが、このイベントを1か月間待っていました。 次に、Fedoraコミュニティトラッカーで新しいバージョンを確認し、ダウンロードを開始します！



最も重要な変更：



デスクトップ：



*ビデオチーズをキャプチャおよび編集するユーティリティを備えたGNOME 2.22、ネットワークファイルシステムのサポートの改善、オンラインカレンダーGoogleのサポート。



* KDE 4.0.3、Xfce 4.4.2、Openoffice.org 2.4;



* FlashメディアからダウンロードするためのLiveUSBリリース。 LiveCDイメージをUSBフラッシュの操作に適したLiveUSBに迅速に変換するユーティリティ： "./livecd-iso-to-disk --overlay-size-mb 1200 / path / to / iso / path / to / stick";



* Pirutに代わるパッケージ管理ツールPackageKitの改善。 PackageKitプロジェクトの主な目標は、すべてのLinuxディストリビューションのパッケージをインストールおよび更新するための単一の統合されたシェルを作成することです。 PackageKitは標準ツールを使用してパッケージを管理します-yum、apt、conary、box、alpm、smart、pisi、zyppなど 正式には、PackageKitはD-Busの抽象レイヤーです。これにより、ユーザーはディストリビューションおよびシステムアーキテクチャに依存しないAPIを介してパッケージを管理できます。



* Network Manager NetworkManagerは、CDMAおよびGSMネットワークへの接続を構成する方法を認識し、アドホックWi-Fiアクセスポイントを作成できます。



*単一のスペルチェック辞書。



* Codeinaマルチメディアコーデックインストールインターフェイス（CodecBuddy）からFluendoが開発した有料のWindows Media、MPEG-2およびMPEG-4コーデックへのリンクを削除しました。 リストには無料のMP3コーデックのみが残っています。



* GTKのネイティブデザインをサポートするFirefox 3ベータ5。



* teTeXパッケージはTeXLiveに置き換えられました。



システム：



*ファイルシステムext4（実験的サポート）;



* Java OpenJDK 6のオープンソース実装、IcedTeaは提供されなくなりました。



* Anacondaインストーラーの改善：

ext2、ext3、およびNTFSパーティションのサイズを変更する機能。


インストール段階での暗号化されたパーティションの作成と使用。


インストール段階でyumリポジトリを操作するメカニズムを改善します。


操作の順序を変更すると、パッケージを選択する前にパーティション化が行われます。


一定量のコードがインストールの第2ステージに移動しました。これにより、ネットワーク経由で破損したパッケージのダウンロードを復元する、リポジトリのリストを編集する、プロキシを使用する、リストから適切なミラーを選択するなどの操作のサポートを改善できます。 ユーザーの観点からは、コードを第2ステージに移植することは、これからインストールを中断しないエラー処理の柔軟性を高めることにより、主に快適です（たとえば、破損したパケットを受信した場合）。


アップグレード前（アップグレード前）の可能性。ダウンロードは、以前のようにディストリビューション全体ではなく、更新が必要なユーザーパッケージによってのみインストールされます。


kudzuの代わりに、HALとudevによるハードウェア評価が使用されます。




* FreeIPA、作成用パッケージ

ユーザー識別、アクセス設定および監査ポリシーの設定のための集中システム。



*（SysV）initの代わりにUpstartを使用するように初期化システムが切り替えられました。



*カーネル2.6.25.x;



*カーネルクラッシュダンプは、エラーを修正するためにwww.kerneloops.orgに自動的に転送できます。



* X.orgサーバーの開始時間と停止時間の短縮。



* Perlをバージョン5.10に更新し、作業の高速化を約束。



*提供されるLiveCDの数を拡張（スピン）：

Fedora Live-GNCDベースのメインLiveCD。


Fedora KDE Live-オプションのKDEベースのLiveCD。


Fedora XFCE Live-XFCE 4.4.2に基づくオプションのLiveCD。


Fedora Games Live-さまざまなゲームアプリケーションを含むLiveCD。


Fedora Developer Live-ソフトウェア開発者向けのプログラムを選択したLiveCD。


Fedora Electronic Lab（FEL）-電子技術者向けのプログラムを選択したLiveCD。


Fedora Game Live-さまざまなゲームプログラムが収録されたLiveCD。




さらに、Fedora 9はrpm.livna.orgリポジトリの可用性を発表しました。これには、標準パッケージリポジトリにないマルチメディアコーデックとアプリケーションが含まれています。 リポジトリを接続するには、「su -c 'rpm -ivh rpm.livna.org/livna-release-9.rpm '」を実行する必要があります。



ご注意 Anakondaにはエラーがあります。インストーラーでロシア語インターフェースを使用している場合、デフォルトでインストールされるパッケージのリストを変更しようとすると、インストーラーがクラッシュします。 さらに、一部のアプリケーション（進化）ではスペルチェックに問題があります。
