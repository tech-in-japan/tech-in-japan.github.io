私はReact Routerと提携していませんが、この記述は間違っていると思います。 React Routerリリースをフォローしている場合は、次のことに注意する必要があります。







  • 1.0は2015年にリリースされました。
  • 2.0は2016年2月にリリースされ、主にバージョン1.0との下位互換性があります（ただし、バージョン3.0で予定されているAPIの変更に関する警告が追加されています）
  • 3.0は2016年10月にリリースされ、基本的にはバージョン2.0と同じですが、APIに最終的な変更が加えられています。


したがって、（自分が何をしているかわからないかもしれないとして）安定したバージョンにアップグレードするだけであれば、バージョン1.0から2.0への移行はほとんど問題なく行われているはずです。 たくさんの警告が表示されるはずであり、次のメジャーリリース（3.0）でこれらの警告が削除されるまでに8か月間修正する必要がありました。







3.0の直後にバージョン4.0が登場したことは間違いありません。 4.0は完全に書き直されたバージョンであり、その哲学はわずかに異なります（これは、Issues React Routerで長年にわたって報告されてきた多くのエラーと問題を修正するために行われます）。 バージョン4.0が3.0の直後にリリースされたことは正しいです。 しかし、ここにそのようなものがあります-あなたは絶対にそれに切り替える必要はありません。 3.0を使用しているアプリケーションはまだ多くあり、そのブランチは引き続き積極的にサポートされています。 したがって、バージョン3.0を使用している場合は何も起こりません。開発チームは、これに関心のある方がバージョン4.0に簡単に移行できるように移行パッチに取り組んでいます。







結果として、リリースの安定バージョンのみを使用した場合、React Routerは新しいAPIに切り替える一年を与えました。 彼らは長年の問題を修正するために4.xバージョンをリリースしましたが、必要に応じて3.xブランチを維持することにコミットしています。 私の意見ではそれほど悪くないようです！
