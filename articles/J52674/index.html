System Center Essentials 2007を別のサーバーに移行するか、障害から回復する

発行日には、Microsoftには、SCEサーバーの移動または復元に関する公式の文書、出版物、または推奨事項がありません。 ギャップを埋めなければなりません。 タスクは非常に関連していますが。 この状況の最も単純な例は、テスト環境から実際の操作へのSCEサーバーの転送です。





始めましょう。 最初に、以前のインストールからドメインを削除する必要があります。そうしないと、インストーラーを起動しようとしても失敗します。 ライセンス契約では、ドメイン内に管理サーバーは1つしか存在できません。 テストサーバーのSCEが停止していない場合は、通常の方法でアンインストールプログラムをインストールするだけで削除できます。 SCEマシンがすでに失われている場合は、ハンドルを使用してActive Directoryをクリーニングする必要があります。



これを行うには、次のものが必要です。



  • SCE管理対象コンピューターグループポリシーグループポリシーオブジェクト（Your_Old_Server_MG）およびSystem Center Essentials All Computersポリシーを削除する
  • Active DirectoryからSCE Managed Computersグループ（Your_Old_Server_MG）を削除します
  • adsiedit.mscを使用して、SCEサーバーを含むコンピューターのオブジェクトを開き、SCESCPおよびSDKServiceSCPパラメーターを削除する必要があります。これにより、System Center Essentialsインストーラーが起動します。
  • 次に、OperationsManager、OperationsManagerDW、およびSUSDBデータベースを切断し、それらを「人里離れた場所」に置きます


準備手順が完了したら、必要な機器でサーバーを直接復元します。



  1. WSUSContentフォルダーを新しいサーバーにコピーします。
  2. SCE 2007を新しいサーバーにゼロからインストールする場合、私の推奨事項を使用できます。 マシンの名前は同じである必要があり、データベースを保存するSQLサーバーの名前も同じであることに注意してください。 前回と同じActionAccountを指定します。
  3. インストール中に、WSUSContentフォルダーの場所を指定します。 更新プログラムがフォルダーD：\ WSUS \ WSUSContentに保存されていた場合、インストール中にD：\ WSUS \
  4. インストールが完了したら、OpsMgr構成サービス、OpsMgr SDKサービス、および更新サービスを停止します
  5. 新しく作成されたOperationsManager、OperationsManagerDW、およびSUSDBデータベースを切り離します。 念のためにそれらを残します。 古いサーバーから保存したデータベースを接続します。
  6. サービスを開始します。 System Center Essentialsコンソールを起動します。 すべてがうまくいった場合は、さらに作業を続けて機能構成ウィザードを起動できます。
  7. 私にとってはうまくいきませんでしたが、すべて安全なストレージが使用され、キーが必要なためです。 SCEをインストールするときに、バックアップを求められたことを思い出してください。 したがって、回復を成功させるには、このキーが必要です。 キーがある場合は、セキュアストレージバックアップウィザードを実行し、ウィザードの指示に従って[復元]を選択して、人生を楽しんでください。
  8. インストールされているすべてのエージェントで必ず修復を実行してください。


キーを持っていない人は、次のことを共感して提供するだけです。



OperationsManager、OperationsManagerDWデータベースはまったく復元せず、新しいSCEインストールで使用したものを残します。

susdbデータベースは復元できますが、そのキーは必要ありません。

その結果、更新プログラムと承認に関する情報は保存されますが、コンピューターのグループに関する情報は失われます（これらはWSUSコンソールから直ちに削除する必要があります）。

すべてのソフトウェア展開がある場合は、復元する必要があります。

プロセスを経た後、回復プロセス全体を区別することができます。これらのコンポーネントは予約する必要があります。



OperationsManager、OperationsManagerDW、およびSUSDBデータベース。

データベースに関する情報はレジストリにあります。



OperationsManager、OperationsManagerDW-HKLM \ソフトウェア\ Microsoft \ Microsoft Operations Manager \ 3.0 \ Setup \ DatabaseName

SUSDB-HKLM \ Software \ Microsoft \ Update Services \ Server \ Setup \ SqlDatabaseName

WSUSContentフォルダー-HKLM \ Software \ Microsoft \ Update Services \ Server \ Setup \ ContentDir

セキュアストレージバックアップウィザードを介してアクセスするためのキー。

すべてのコンポーネントは、ネットワーク上の別の物理コンピューターにバックアップする必要があります。 これにより、リカバリの問題がさらに軽減されます。



PS新しいサーバーに転送する方法は？ 正確な推奨はできませんが、おそらくすべてのコンポーネントをバックアップし、通常の方法で新しいコンピューターにSCEを最初からインストールし、データベースを復元する必要があります。
