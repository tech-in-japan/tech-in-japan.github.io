Visual StudioおよびC ++ AMPプラットフォームのGPU開発の詳細

グラフィックアクセラレータ（GPU）のコンピューティングリソースを完全に利用するアプリケーション開発テクノロジは、人気と関連性を獲得しています。 今日、ほとんどすべてのコンピューターとラップトップには、コンピューティングを大幅に高速化するためにアプリケーションで使用される可能性のある巨大なコンピューティング能力を持つグラフィックアダプターが装備されています。



ヘテロジニアスコンピューティング（CPU + GPUバンドル）に関するマイクロソフトの最新の研究結果の1つは、最新のマルチコアプロセッサおよびGPUのすべての機能を完全に利用できるC ++ AMPプラットフォームの発表です 。 プラットフォームの発表は、この夏のAMD Fusion'11カンファレンスで行われました。詳細については、このブログ投稿をご覧ください。 同じイベントで、GPUコード用の組み込みのデバッグおよびプロファイリングツールが次世代のVisual Studioで発表されました。 新しいプラットフォームに関する詳細なレポートは、最近開催されたBUILD会議でも読まれました。



Tech∙Ed Russia 2011では、C ++ AMPプラットフォームが提供する新機能と、次世代のVisual Studio開発環境に組み込まれる新しいGPUコードツールについて直接学ぶことができます。 「 Visual Studio 11のC ++ AMPヘテロジニアスコンピューティングプラットフォームとGPUツールの紹介 」というタイトルの特別レポートは、Visual Studioグループのシニア開発エンジニアとしてMicrosoftで働いているRedmondのゲストであるMaxim Goldinによって特に読まれます。







マキシムで働いている間、マキシムゴルディンはVisual Studio製品のいくつかのバージョン（VS2005、VS2008、VS2010）のリリースに直接関与し、現在製品の次のバージョンに取り組んでいます。 マイクロソフトに入社する前、マキシムはIntelで8年間働いており、最新のマイクロプロセッサデバイスの動作をシミュレートする技術の開発を専門としていました。



Tech∙Ed Russia 2011カンファレンスで、Maxim Goldinは2つのレポートを読みます。1つはC ++ AMPテクノロジーとGPUの使用に関するもの、もう1つは新しいVisual Studio ALMツールに関する重要なレポートです。
