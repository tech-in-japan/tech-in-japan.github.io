分散ウェブ

ブラウザが相互に直接接続して音声、ビデオを送信し、メディアコンテンツを交換し、トラフィックと使用条件を制御する中央サーバーを介してすべてを実行するのではなく、P2PとさまざまなAPIを使用してアプリケーションで連携するWebを想像してください。



これは潜在的に非常に強力なアイデアであり、将来適切に実装されると、権威主義的な機能停止、自然災害、および想像するのが難しい他の多くのことからウェブを保護します。 また、相互に接続されたデバイスにデータのフィルタリングを停止または開始できる「中間」がない場合、「制限なし」でタイムゾーンを作成できます。



時々、異なる人々が同じ発明、またはむしろ技術に取り組み始めます。 これが、分散Webのアイデアで起こることです。 International Web Consortium（ W3C ）は昨日、サーバー側を実装せずにそのような通信を可能にするクライアント側APIを策定するために、リアルタイムWeb通信（Web Real-Time Communications）のワーキンググループの作成を発表しました。



W3Cでは、このグループはGoogleとエリクソンのエンジニアが率いていますが、実際にはOperaは2年前に同じアイデアを提供しましたが、 Webページを表示するツールとして一般的にブラウザに導入されていませんでした。 Operaは、いつものように、その時代の数歩先を行っていました。



ビデオでは、Operaが独自のUniteテクノロジーについて説明しています。





「これらのAPIを使用すると、ブラウザ内で実行できるアプリケーションを構築できるはずです」とワーキンググループは言います。「追加のダウンロードやプラグインは必要ありません。 参加者が、干渉サーバーを使用せずに、オーディオとビデオ、および追加の通信ツールを使用してリアルタイムで通信できるようにします（メンテナンスやファイアウォールなどを除く）。



新しいグループが関与しているAPIの問題は、そのような接続のためにデバイスでさらに使用されるインターフェースを開発することです。 IETFチームとのコラボレーションも進行中です。IETFチームは 、ブラウザー間でデータを交換するための技術的なプロトコルを作成しています。 この方向でのIETFの作業の最初の痕跡は、Google Cromiumがいわゆるいわゆる P2P API



この開発の責任者は、GoogleのHarald Alvestrand（Harald Alvestrand）とEricssonのStefan Hakansson（Stefan Hakansson）です。 開発が完了する予定の2013年2月まで、技術開発の進捗に関する公式ニュースが表示されます。 ユーザーが送信/受信したデータを監視することと、ブラウザが処理しているデータを理解することに別の注意が払われます。
