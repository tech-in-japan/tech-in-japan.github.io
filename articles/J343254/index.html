Azure Stackとは何ですか？



Azure Stackは、組織の独自のデータセンターでAzureサービスを実行できるハイブリッドクラウドソリューションです。つまり、Azureのパブリッククラウドサービスと同様の便利な開発、管理、セキュリティツールを利用できます。 Azure StackはAzureの拡張機能です。 このソリューションにより、オンプレミス環境でAzureサービスを操作し、必要に応じてAzureパブリッククラウドの使用に切り替えることができます。







Azure Stackでは、パブリックAzureクラウドと同じツールを使用して、IaaS（「サービスとしてのインフラストラクチャ」）とPaaS（「サービスとしてのプラットフォーム」）の2種類のサービスをデプロイおよび使用できます。



Azure Stackサービスは、PowerShellまたはWebポータルを使用して管理できます。どちらの場合も、インターフェイスと機能はAzureのものと同様です。 以下は、Azure Stackでの仮想マシンのプロビジョニングおよび管理ツールの外観を示しています。 ご覧のとおり、Azureと同じように見えます。

Azure Stackのユースケース



Azure Stackは、企業がデータセキュリティ、主権、コンプライアンスの確保など、ビジネスの最も複雑なITタスクを解決するのに役立ちます。 また、これらのソリューションを使用すると、高性能分析システムの作成、計算の実行、スタンドアロン環境または分離環境でのAzureアプリケーションの実行、最新のアプリケーションの開発のためのプラットフォームの準備を行うことができます。



データのセキュリティ、主権、コンプライアンスの確保



Azure Stackを使用すると、クラウドを使用するときと同じように複数の国のデバイスで同じサービスを実行できますが、同時に、各国のデータセンターに同じアプリケーションをデプロイすることでデータの主権を確保できます。つまり、州内でデータを処理できますボーダー。



最大のパフォーマンスとデータ転送速度を確保する



パブリッククラウドは、最高のパフォーマンスとデータの送受信速度を必要とするアプリケーションの実行に常に適しているとは限りません。

たとえば、クラウド内のデータセンターから大量のデータを分析するには、まずこのデータをクラウドにアップロードする必要があります。 これには多くの時間がかかる場合があります。内部ビジネスプロセスや競争力の観点から、場合によっては非常に時間がかかります。

クラウド専用のアプリケーション開発



クラウド指向のアプリケーションは、さまざまな環境でマイクロサービスとして実行されるように設計されています。 開発者は、異なる環境で実行されるという理由だけで、異なるマイクロサービスに異なる開発ツールを使用する必要性を好む可能性は低いです。 最速かつ最も効果的なオプションは、完全なツールセットを使用してアプリケーションを開発し、必要に応じて展開することです。 Azure StackとAzureはAPIレベルで互換性があるため、これらの問題はありません。
