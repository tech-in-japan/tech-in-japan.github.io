Windows Server 2012上のSMB用HP VDI

最近、HPはMicrosoftと共同で、SMBセグメントの職場の仮想化（VDI）のモデルの説明を発表しました-Windows Server 2012用HP Client Virtualization Reference Architecture。ソリューションのサーバー部分は、Windows Server 2012を実行するHP ProLiant DL380p Gen8に基づいていますおよびクライアント-HPシンクライアント 。 このセット全体は、実際の負荷をシミュレートする条件下で正常にテストされています。



この投稿では、テスト結果を紹介し、これらのコンポーネントがソリューションに選ばれた理由を説明します。



今年は、HPとMicrosoftの両方の技術発表が非常に豊富でした。 マイクロソフトは1か月前にロシアでWindows Server '12をリリースし、HPは3月からウェーブごとにGen8世代のProLiantサーバーのラインを更新しました。 そして、目の前に出現している仮想化市場では、小規模企業が最新の製品を提供することを望んでいることは明らかです。そのため、更新の必要性は今後何年も顧客を占有しません。



同様のプレハブハードウェアおよびソフトウェアソリューションと同様に、Windows Server 2012用のHP Client Virtualization SMBリファレンスアーキテクチャ（簡潔にするために、略してSMBのCVと呼びます）は次の目標を追求しました。



  1. 最大パフォーマンス：以下で詳細に説明するパフォーマンステストは、SMBのCVがサーバーあたり少なくとも175の仮想デスクトップを優れた応答時間でサポートできることを示しています。
  2. 利便性と低コストの展開：お客様は、ソフトウェアがハードウェアと完全に互換性があり、OEM納入によりライセンスのコストが最小限に抑えられることを確認できます。 理解しやすいスケーラビリティにより、ニーズに合わせて特定の仕様を作成するのは簡単です。
  3. すべてを含む：ソリューションは追加のインフラストラクチャを展開する必要がありません-管理サーバー、データベースサーバー、またはSANは必要ありません。 また、追加の仮想化または管理ソフトウェアをインストールする必要もありません。


私の意見では、これらの目標はすべて達成されました。



ソリューションコンポーネントをさらに詳しく考えてみましょう。



サーバー側-HP ProLiant DL380p Gen8-は、多くのタスクのための強力な汎用ツールです。 SMBタスクのCVに対するこのサーバーの利点は、ハードウェア構成を選択/拡張するための幅広いオプションです。



ここでサーバーについて少し読むことができます 。



クライアント側-HP Thin Client



SMBソリューションのCVの一部として、広範なHPシンクライアントの代表者を使用することをお勧めします（意外なことに、ロシア市場ではまだあまり知られていません）。 もちろん、デスクトップからタブレットまで、クライアント側で何でも機能します。



スマート ゼロシン クライアント： HP All-in-One t410 、 HP t410など。 Power-over-Ethernet電力技術を使用し、「ゼロ」クライアントのすべての利点と高性能および低コストをユーザーに提供します。 たとえば、彼らはネットワーク自体でSMBホストのCVを見つけ、多くのプロトコルをサポートし、デバイス設定でローカルの介入なしにその場で再フラッシュできます。



柔軟なシンクライアント： HP t510 、 HP t610、またはWindows Embedded Standard 7（WES 7）を搭載したHP t610 Plusは、高度なグラフィック機能を必要とするお客様に最適です。 最上位モデルt610 Plusは、強力なCPUとGPUおよびPCIeスロット、最大4台のディスプレイのサポート、内部WiFiアンテナとファイバーNICで際立っています。 スマートゼロテクノロジーは、フレキシブルシリーズにも展開できます。



ソリューションソフトウェア-Windows Server 2012



このソリューションは、Microsoft Windows Server 2012で実行され、Microsoftリモートデスクトッププロトコル（RDP）を使用して、リモートデスクトップまたはリモートアプリケーションをユーザーに提供します。 RemoteFXサーバービデオアダプターの仮想化機能を使用すると、サーバービデオカード機能へのリモートアクセスをユーザーに提供できます。 WS 2012には、VDIに必要な接続ブローカー、Webアクセス、ゲートウェイ、その他の重要なコンポーネントがすべて含まれています。これらはすべてMicrosoft Windows Server 2012にロールとして統合されているため、追加のサードパーティ製品を購入する必要はありません。 私たちの場合、Windows Server 2012の主な利点は次のとおりです。



  1. 効果的な管理 -Windows Server 2012のRDS機能は緊密に統合されており、新しい集中コンソールとウィザードで簡単に構成できます。 プールされた仮想デスクトップおよびパーソナル（専用）仮想デスクトップを使用するためのサポートされているシナリオ。 さらに、お客様はアップデートのインストールプロセスを自動化できます。これにより、HPのサーバー側管理の自動化機能とともに、ITプロフェッショナルのリソースを大幅に節約できます。
  2. RDSは、さまざまな低コストのストレージシステムをサポートし、高価なSANソリューションの必要性を排除します。 お客様は、構成ファイルとスナップショットを「共有フォルダー」（共有フォルダー）に移動することにより、ソリューションのコストを削減できます。 また、仮想マシンはクラスターの共有ボリュームに保存でき（クラスターの異なるノードから実行される仮想マシンがクラスターディスク上にある場合に便利です）、ディスクディスクの直接接続ストレージサブシステムにキャッシュできます。これにより、高性能で低コストのソリューションが提供されます。
  3. 3.ユーザーは、以前は仮想環境に実装することが困難であったすべてのテクノロジーを使用できます 。 たとえば、RemoteFXのおかげで、3Dグラフィックスとビデオストリーミングの操作、USBデバイスのリダイレクト、タッチデバイスとアプリケーションの完全なサポートを含めることができます。 新しいユーザープロファイルディスク機能により、プールされた仮想マシンとセッションのユーザーデータとアプリケーションキャッシュデータを保存できるようになりました。 ユーザーログインセッション間のすべてのデータが保存されます。 デフォルトでは、システムにはCPU、ネットワーク、およびディスクシステムのI / Oの動的分散機能が含まれており、ユーザーが同じサーバーで作業している他のユーザーの作業に影響を及ぼすことを防ぎます。


テスト中



Windows Server 2012用のHP CV SMB RAは、 Login VSI v3.6を使用してMicrosoftチームによってテストされました。 このパッケージは、仮想環境での実際のワークロードを模倣します。メール、テキスト、テーブルの操作、スライドショーの作成、サーバーで実行されているHDビデオの視聴。 アクティブセッションの場合、アプリケーションの応答時間、CPU負荷、ディスクサブシステムの待ち時間、およびI / O操作キューの長さを測定しました。



初期テストと推奨テストの2つのテストが実施されました。



初期テストの目的：CPUの80％以下の負荷を作成し、応答時間が3秒以下になるセッションの最大数を決定する。



チームは、Windows 7およびWindows 8をエンドユーザーのOSおよびRDP 7プロトコルとして使用するいくつかのシナリオをテストしました。
