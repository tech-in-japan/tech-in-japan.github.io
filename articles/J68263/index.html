PEAR HTML_QuickFormでSMARTYテンプレートを使用する

1.はじめに。



この記事は何についてですか？



この記事は、PEAR HTML_QuickFormクラスでSmartyテンプレートを使用するための簡単な紹介です。 この記事は網羅的なガイドではなく、実際にはSmartyテンプレートの機能のごく一部にすぎません。 ただし、Smartyテンプレートを使用する初心者にとっては、この記事が役立つ資料になります。







翻訳者注：HabrはHTMLを解析するため、一部の場所ではコードがスクリーンショットの形式で表示されます。



前提条件。



PHPとPEARが既にインストールされており、PHPとHTMLに精通していることが理解されます。 PEAR HTML_QuickFormクラスの使用にも精通している必要があります。 PEAR WebサイトでPEARのヘルプを取得できます。PHPWebサイトでPHPのヘルプを取得できます。 また 、 別の記事で HTML_QuickFormの使用の概要を見つけることができます。



この記事は実用的な観点から書かれています。 最大限に活用したい場合は、ここからサンプルコードをコピーして、自分で実行する必要があります。 とりわけ、テンプレートは情報を表すために使用され、この記事ではデータ出力の例を提供していません。 動作中のパターンを見たい場合は、自分でコードを実行する必要があります。 最後に、この記事で紹介するコード例は、Smartyテンプレートのみの使用を表すことを目的としています。 つまり、テンプレート関数をより明確に表すために、データ検証などの関数は省略されています。

3.ちょっとした理論。





テンプレートは何に使用されますか？



上記の例で見たように、デフォルトでは、PEAR HTML_QuickFormクラスによって作成されたテンプレートは非常に簡単な方法で表示されます-後続のフォームと要素の下にショートカットが表示されます。

実行可能であるため、これらのフォームには、プロフェッショナルなアプリケーションに対するいくつかのタッチが欠けています。



プログラム可能なHTMLを生成する際の問題の1つは、表示された要素をプログラムコードから分離することです。これは、PEARクラスが非常に成功していることです。



テンプレートを使用すると、デザイナーはより魅力的で複雑なページを作成し、フォーム要素の表示場所を正確に決定できます。 デザイナーはHTMLファイルを使用し、プログラマーはPHPコードとPEARクラスを使用します。これらの1つを変更しても、2番目のクラスを変更する必要はありません。



smartyテンプレートはHTML_QuickFormでどのように機能しますか？





上記の例では、行$ form→display（）; 標準のHTML_QuickFormsビジュアライザーを呼び出します。 この行を置き換えることにより、表示されるHTMLをより詳細に制御できます。 また、コードを2つの部分（PHP / PEARコードとテンプレートファイル自体）に分割することもできます。



テンプレートファイルには、HTMLコードといくつかの追加タグが含まれています。 この単純なレベルでは、これらのタグは情報を表示する場所です。 ビジュアライザーの役割は、フォームデータを取得し、テンプレートオブジェクトで使用できる形式に転送することです。 テンプレートオブジェクトの役割は、テンプレートファイルを解析し、追加のタグをフォームのデータで置き換えることです。



テンプレートを使用したフォームの作成と視覚化は、複数のステップから成るプロセスです。



  1. フォームは、以前と同じ方法でPEARクラスを使用して作成されます。
  2. テンプレートオブジェクトが作成され、
  3. 視覚化オブジェクトが作成され、
  4. ビジュアライザーは各フォーム要素のHTMLフラグメントを生成し、これらのフラグメントは、フォームに関する残りの情報とともに1つの大きな配列に収集されます。
  5. フォームデータの配列がテンプレートオブジェクトに渡され、
  6. 次に、テンプレートオブジェクトはテンプレートファイルを解析し、タグを配列のデータで置き換え、HTMLを表示します。

5.実用的な考慮事項。

非表示フィールドとJavaScript





非表示フィールドは、コンテキスト情報を保存するためにフォームで使用されます。 多くのフォームはJavaScriptを使用してさまざまな機能を実行します。 ご存知のように、非表示フィールドもJavaScriptコードもページに表示されませんが、この情報はテンプレートオブジェクトに渡される配列内にあります。 このデータを使用するには、次の変更を行う必要があります。

  3. if （ $ form- > validate （ ） ） {
  4. ＃フォームが検証されたら、データをフリーズします
  5. $ form- > freeze （ ） ;
