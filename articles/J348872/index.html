無人車。 Intelの専門家の回答





この投稿では、Intelの専門家Ivan Kuzminが、「人工知能と自律走行車」というトピックに関する読者からの質問に対する回答を公開しています。 いつものように、たくさんの質問がありました。私たちはイヴァンを去って、彼が答えることができる質問を選びました。 さまざまな明確化のために、コメントを募集していますが、迅速な回答を保証するものではありません。

最後に、伝統によると、最高の質問に対して賞を受賞したHabrauserの名前を発表します。



私たちは、無私の仕事と時間を費やしてくれたイヴァン・クズミンに感謝します！



Lingvoの質問

AIは今日実際にテストされている無人車両で実際に使用されていますか、それともすべてのマーケティング物語であり、車の自動操縦装置は本当に明確な数学的アルゴリズムとロジックに基づいていますか？


回答：無人車両の制御にはさまざまな数学アルゴリズムが使用されます。 制御タスク自体は複雑であり、さまざまなパラメーターを考慮し、異なるソースからのデータを計算する必要があるため、すべてのアルゴリズムを数えるのは非常に困難です。 人工知能または機械学習のアルゴリズムには、古典的なアルゴリズム（ツリー、ロジスティックまたは線形回帰、サポートベクトル法）と深層学習アルゴリズム（畳み込みおよびリカレントニューラルネットワーク）の両方を含めることができます。 さまざまなアルゴリズムの組み合わせにより、制御問題を効果的に確実に解決できます。



自動車ソフトウェアに採用された安全基準への準拠のためのこのソフトウェアの認証はどのように行われますか？ オートパイロットは、ハードウェアとソフトウェアを備えたESPまたはエアバッグと同じ重要な車両制御ユニットと見なされていますか？


回答：もちろん、自動操縦はエアバッグや他のデバイスと同じくらい重要なユニットです。 ロシアにはISO 26262規格または同等のGOST R ISO 26262があり、セキュリティ関連システムの開発プロセス全体に要件を課しています。 開発の終わりにそのようなシステムの開発者は、彼がこの標準に従ったことを示すべきです。



オートパイロットはハードリアルタイムで動作しますか、それともLinux / Windowsのすべてが今のところ動作しますか？


回答：一般に、センサーまたはカメラへの信号または画像の受信から機械の制御システムへの信号の送信まで、140ミリ秒未満を満たす必要があります。 この問題はハードリアルタイムモードで解決できますが、多くの自動車会社やパートナーも、業界のすべての要件を満たすことができるLinuxバージョンの方向性を開発するオプションを検討しています 。



インテルは、その成果をオープンソースにもたらし、より多くの開発者を引き付ける予定ですか、それともすべてを閉じて関心のある顧客に販売する予定ですか？


回答：同社は常にオープンソースのアイデアをサポートおよびサポートしており、多数のオープンソースプロジェクトの開発に積極的に関与しています。 同時に、無人車両の開発における重要な問題は、標準への準拠です。これにより、多数の開発者の参加が制限される場合があります。 いずれにせよ、私たちは個々の製品のすべての長所と短所を比較検討し、個別に決定を下します。 この点に関する一般的な戦略はまだありません。



開発ツールはモデリングと自動コード生成に使用されていますか（Matlab / Simulinkなど）、またはすべてがC / VHDLのペンで書かれていますか？ もしそうなら、総ソフトウェアカバレッジの何パーセント？


回答：自動車規格では、開発者が作成されたコードを完全に理解している必要があります。 これにより、コードジェネレーターの使用が制限されます。 同時に、Matlabなどのツールに組み込まれたモデルは、記述されたアルゴリズムを検証するために非常に使用されます。



プロセッサ、FPGA、既製のボード、センサー/ライダー/レーダー/駐車センサー、アクチュエーターなど、実際の車で開発されたアルゴリズムの「ラピッドプロトタイピング」にはどのようなハードウェアが使用されますか？


回答：ラピッドプロトタイピングについて話している場合、それは車ではなくデータセンターで起こります。 ここで、Intelはハードウェアとソフトウェアの両方で多くのソリューションを提供しています。Intelディストリビューションfor Pythonパッケージは、ラピッドプロトタイピングに最も便利な言語としてPythonを大幅に高速化します。 私たちが車について話すとき、ここにCとC ++言語があり、プラットフォーム自体は、原則として、コスト、電力消費、および他の多くの要因を考慮して事前に設定されます。



指定された周辺機器をコンピューティングプラットフォームに接続するための業界標準は既にありますか？


回答：はい、そのような標準はすでに存在し、既存のADASマシンおよびソリューションで積極的に使用されています。

質問Nick_mentat

非ネットワーク上でマシンビジョンシステムのない自動車を開発する可能性は考慮されていますか？ そうでない場合、どのような種類のハードウェアが使用されますか？また、必要な計算能力は車を過剰に充電しませんか？

質問Rost99

彼女は、トレーニング用のデータの収集/収集の計画方法、およびこのために最低限必要と考えられるデータの量に関心があります。

シスの質問

今日、公道で手頃な価格の無人ファミリーカーを発売することを妨げるものは何ですか？

オプションとして：

  • 市場での安価な/不可欠なレーダーの欠如。
  • 安価な/必要な電卓の欠如。
  • あらゆる状況でのコンピューターの不完全/信頼できるアルゴリズムの欠如。
  • 道路状況が良くない（路面標示の不備、道路上の突然の障害物など）。
  • 運転中のドライバーにとっても、常に明確な道路状況はありません。
  • 他の道路利用者の予測できない行動。
  • コンピューターでの大きなエネルギー損失（特にEVに関連）。
  • ナビゲーションシステム（GPS）からのデータの精度とアクセシビリティ（トンネル）の欠如
  • 奇妙な外観（突出したレーダー）。
  • その他の不都合（トランクが部分的に電卓に占有されている）。
  • 法律。


回答：多くの要因があり、事実上すべてがリストされています。 まず第一に、これは、データを効率的かつ正確に処理し、移動軌跡を計算する能力であり、自動車（長期電気）で許容される電力消費の枠組み内に留まります。 アクティブに使用される部分的なソリューションは多数あります（行調整、適応クルーズコントロールなど）。 本格的な無人車を作成するには、最終的なソリューションで使用されるアルゴリズムの種類と数、他のアルゴリズムとの作業の複数の重複などの質的な移行が必要です。 たとえば、ニューラルネットワークのパターン認識の精度は90 +％であると言います。 これは非常に良い結果ですが、管理を委ねるために、そのようなアルゴリズムだけに頼ることはできません。これは間違っていることが事前にわかっています。



人がどんな状況でも非常にうまく車を管理でき、片目を閉じてキーボードの4つのボタンで指示し、非常に低いフレームレートと白黒640x480の画像でプロセスを見ることができる場合、レーダーは本当に必要ですか？


回答：確かに、質問は非常に興味深いものです。 私が言ったように、別の質問に答えるには、さまざまな傾向があります。 高精度の1台のカメラで十分であると主張し、画像の深度を復元できるアルゴリズムの研究を行っています。 それどころか、レーダーやライダーは自分で簡単に対処でき、カメラはまったく必要ないと主張する人もいます。 私は、今後数年間運転する無人車両の道の最初に、これらのコンポーネントの両方を組み合わせたソリューションが見られるだろうと考えていますが、エラーのリスクは大きすぎ、過度の重複は干渉しません。 同時に、ソリューションの最適化と要素の総数の削減に向けた取り組みが引き続き積極的に行われると確信しています。 科学が私たちをどこに連れて行くのか、30年後に見てみましょう。
