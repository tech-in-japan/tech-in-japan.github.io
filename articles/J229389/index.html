Lync2013。ドメイン名のSIPアドレスを変更する





LyncユーザーのSIPアドレスのドメイン名を変更するタスクがありました。

インフラストラクチャにはLync 2013、Exchange 2013があります。別の組織とのフェデレーションもあります。



1）サポートされている追加のsipドメインに新しいsipドメインを追加する必要があります。

これは、Lync Topology Builderを介して行われます。

開いて、「トポロジーのダウンロード」をクリックし、tbxml拡張子でファイルを保存します。

次に、Lync Serverのプロパティに移動して、新しいSIPドメインを追加します。







その後、トポロジを公開する必要があります。 トポロジビルダーアクションのメインウィンドウ->トポロジの公開。



Lync管理コンソールで確認できます-ユーザーには、sipドメインのドロップダウンリストが表示されます。



2）レコードをDNSに追加します。



内部DNSの場合



タイプSRV

_sipinternaltls._tcp.NewDomain.com（ポート5061）

タイプA

sip.NewDomain.com

pool.NewDomain.corp

lyncdiscoverinternal.NewDomain.com

dialin.NewDomain.com

meet.NewDomain.com



外部DNSの場合



タイプSRV

_sip._tls.NewDomain.com（ポート44）

_sipfederationtls._tcp.NewDomain.com（ポート5061）

タイプA

sip.NewDomain.com

webcon.NewDomain.com

av.NewDomain.com

dialin.NewDomain.com

lyncdiscover.NewDomain.com

lyncwebsvc.NewDomain.com

meet.NewDomain.com



3）新しいドメインで新しい証明書を作成します。

展開ウィザード-> Lync Serverシステムのインストールまたは更新->インストールの要求または証明書の割り当てに進みます。

新しい証明書要求を作成します。 そして、それをデフォルトの証明書として設定します。



DNS名= Pool.newdomain.corp

DNS名= LyncWebSvc.newdomain.corp

DNS名= sip.newdomain.com

DNS名= LyncServer01.newdomain.corp

DNS名= meetings.newdomain.corp

DNS名= dialin.newdomain.com

DNS名= meet.newdomain.com

DNS名= LyncdiscoverInternal.newdomain.com

DNS名= lyncdiscover.newdomain.com



DNS名= Pool.OldDomain.corp

DNS名= LyncWebSvc.OldDomain.corp

DNS名= sip.OldDomain.com

DNS名= LyncServer01.OldDomain.corp

DNS名= meetings.OldDomain.corp

DNS名= dialin.OldDomain.com

DNS名= meet.OldDomain.com

DNS名= LyncdiscoverInternal.OldDomain.com

DNS名= lyncdiscover.OldDomain.com



Enable-CSComputerを実行します



4）連盟

フェデレーション組織のLync管理コンソールで、新しいドメインを追加する必要があります。

これは、[Lync]-> [フェデレーションと外部アクセス]-> [SIPフェデレーションドメイン管理コンソール]を介して実行されます。



5）リバースプロキシ-必要に応じて、変更を加えて証明書を書き換える必要があります



6）すべてのLyncユーザーのSIPドメインを変更します。

スクリプトを使用します。 



$UserList = Get-CsUser foreach ($User in $UserList) { $oldAddress = $User.SipAddress $newAddress = $oldAddress -replace "@olddomain.com", "@newdomain.com" Set-CsUser -Identity $User.Identity -SipAddress $newAddress }




よくある質問

1）新しいドメインを持つユーザーがLyncにログインし、Lyncが信頼を検証できないことを示すウィンドウがポップアップした場合。







パスに沿ったレジストリの必要性[HKEY_CURRENT_USER \ Software \ Microsoft \ Communicator]

パラメーター「TrustModelData」に、メッセージで指定されたアドレスを追加します。

support.microsoft.com/kb/2531068



また、必要なドメインがローカルイントラネット上のIEで指定されていることを確認する必要があります。



2）履歴がOutlookに保存されていない場合は、新しいドメインについて、Exchangeを指すDNSに自動検出エントリがあることを確認する必要があります。
