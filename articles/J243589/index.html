PHPプロジェクトのVagrant

VagrantでPHPを使い始める簡単な方法の記事5に感銘を受け、PHPプロジェクトでVagrantを使用する方法を共有したいと思います。



私が達成したかったもの：

  • VagrantとVirtualBoxのみが開発者のマシンにインストールされます。
  • 仮想マシンの設定はプロジェクトリポジトリに保存されるため、開発者はプロジェクトリポジトリをすばやく展開したり、プロジェクトのニーズに合わせて柔軟に構成したり、これらの設定をチームメンバーと共有したりできます。


これらの目標は、Chef-Soloの支援により達成されました。 その結果、それに基づいた新しいプロジェクトの作成と、既存のプロジェクトの統合（ vagrant-php）の両方の準備が整いました。







チームの新しいメンバーのためにこのようなプロジェクトで作業するシナリオは非常に簡単です。



  1. VagrantとVirtualBox 、およびいくつかのVagrantプラグインをインストールする必要があります。 
    $ vagrant plugin install vagrant-librarian-chef-nochef vagrant-omnibus
  2. リポジトリからプロジェクトをダウンロードします。 
     $ git clone repo path
  3. Vagrantfileをコピーして仮想マシンを起動します。 
     $ cd path $ cp .chef/Vagrantfile Vagrantfile $ vagrant up


これで、仮想マシンが展開され、パッケージがインストールされ、アプリケーションが機能しました！



vagrant-librarian-chef-nochefプラグインは、Chefの外部クックブックをロードし、ローカルマシンにChef自体をインストールせずに依存関係を管理するように設計されています。 このプラグインを使用すると、リポジトリに外部クックブックを保存するのではなく、仮想マシンを初めてオンにしたときにそれらをロードできます。



vagrant-omnibusプラグインを使用すると、仮想マシンにインストールされる特定のバージョンのChefを指定できます（これにより、 vagrantcloud.comのほぼすべてのイメージを使用でき、Chefのバージョンを修正することもできます）。



中身は？



  • Ubuntu Trusty 14.04（32ビット、最新）;
  • Nginx（Nginx PPAの最新バージョン）;
  • PHP 5.6（ondrej PPAから）、Composer;
  • Memcached（標準のUbuntuリポジトリから）;
  • MySQL 5.5（標準のUbuntuリポジトリから）;
  • PostgreSQL 9.3（PostgreSQL PPAから）。


必要に応じて、これらのコンポーネントのいずれかを交換できることに注意してください。 コンセプト自体は普遍的です。



この環境をPHPプロジェクトに統合する方法は？



既存のPHPプロジェクトがある場合は、リポジトリからプロジェクトをダウンロードする必要があります。 

 $ git clone repo path


新しいプロジェクトを開始する必要がある場合は、まずそのプロジェクト用のディレクトリを作成する必要があります。 

 $ mkdir path


次に、「。chef」ディレクトリをプロジェクトディレクトリにコピーし、「。chef / nodes / 10.2.2.10.json」ファイル（jsonファイルの構造については以下を参照）および「.chef / Vagrantfile」で必要な設定を行う必要があります。



あとは、「。chef / Vagrantfile」ファイルをプロジェクトのルートにコピーして、仮想マシンを起動するだけです。 

 $ cd path $ cp .chef/Vagrantfile Vagrantfile $ vagrant up


新しいプロジェクトを作成する場合は、アプリケーションユーザー（SSHアクセスについては以下を参照）でSSH経由で仮想マシンにログインし、Composerを使用してプロジェクトをインストールしてから、仮想マシンからローカルディレクトリにプロジェクトのコンテンツをアップロードする必要があります。 

 $ composer create-project --prefer-dist yiisoft/yii2-app-basic /home/php-app/www


行「/ Vagrantfile」をプロジェクト「.gitignore」ファイルに追加することもお勧めします。 これにより、開発者は、VCSにこれらの変更を反映せずに、仮想マシンの設定をローカルで変更できます（たとえば、RAMのサイズを削減するため）。



その後、仮想環境でプロジェクトの操作性を確認する必要があります。プロジェクトが操作可能な場合は、プロジェクトディレクトリを（新しいまたは既存のリポジトリに）コミットできます。



SSHアクセス



SSHを介して仮想マシンにアクセスするには、2つの方法があります。

  • 管理者として： 
     $ vagrant ssh
  • アプリケーションをデプロイするユーザーとして： 
     $ ssh php-app@10.2.2.10 -i '.chef/files/id_rsa'


デフォルトでは、パスワードの入力は無効になっています（パラメーター["openssh"] ["server"] ["password_authentication"]）。したがって、パラメーター["php-app"] ["ssh"] [ 「Authorized_keys」]。 この配列には、仮想マシン内のファイルへのフルパスがリストされます。 「.chef」ディレクトリは仮想マシンの「/ tmp」ディレクトリにマウントされるため、パスは「/tmp/.chef/files / ...」のようになります。 デフォルトでは、公開鍵は「id_rsa.pub」であり、秘密鍵は「id_rsa」という名前の下にあります。



ディレクトリ構造「.chef」



  • files / -php-appレシピで使用されるファイルを保存するためのディレクトリ（これらのファイルは、パス "/tmp/.chef/files/"によって仮想マシン内で利用可能です）;
  • nodes / -仮想マシンの設定を保存するディレクトリ。
  • クックブック/ -ダウンロードされた外部クックブックを保存するためのディレクトリ（自動的に作成されます）;
  • site-cookbooks / -プロジェクト自体の内部クックブックを保存するためのディレクトリ。
  • tmp / -一時ディレクトリ（自動的に作成）;
  • Cheffile-内部クックブックの依存関係（「librarian-chef」に必要）;
  • Cheffile.lock-クックブックの修正バージョン（ファイルは自動的に作成されます）;
  • solo.rb-仮想マシン内で起動されるChef-Soloの設定ファイル。
  • Vagrantfile -Vagrant設定を含むテンプレートファイル（このファイルは、サンプルとして他のチームメンバーによって使用されます）。


仮想マシンの設定を含むJSONファイルの構造



run_listパラメーターは、マシンがデプロイされ、環境が準備されるときに実行されるレシピをリストします。 他のすべてのパラメーターは、それぞれのクックブックの設定です。



「.chef / nodes」ディレクトリには、Yii2フレームワークに基づくアプリケーションの既製の例がいくつかあります。

  • yii2_advanced.json-高度なYii2アプリケーションテンプレートを展開するための設定。
  • yii2_basic.json-基本的なYii2アプリケーションテンプレートをデプロイするための設定。


環境自体はYii2とは一切関係ありませんが、これらの設定は例として使用できます。



php-appクックブックはローカル（「.chef / site-cookbooks」にあり、プロジェクトリポジトリにコミット）で、サーバーにアプリケーションをデプロイするように設計されています。 次のアクションを実行します。

  1. アプリケーションの個別のシステムユーザーの作成（パラメーター["php-app"] ["user"]および["php-app"] ["group"]）;
  2. 必要なディレクトリの作成（パラメーター["php-app"] ["project_dir"]および["php-app"] ["log_dir"]）;
  3. アプリケーション用に個別のphp-fpmプールを作成します（パラメーター["php-app"] ["php"] ["pools"]）。
  4. Nginx仮想ホストの作成（パラメーター["php-app"] ["vhosts"]）;
  5. 仮想マシンのローカルホストファイルへのエントリの追加（パラメーター["php-app"] ["hosts"]）;
  6. データベースの作成（パラメーター["php-app"] ["mysql"]および["php-app"] ["pgsql"]）;
  7. gitリポジトリからプロジェクトをダウンロードします（必要な場合は、プロジェクトの場所に関するセクションを参照してください）。
  8. Composerとすべてのプロジェクトの依存関係をインストールします（パラメーター[“ php-app”] [“ composer”]）。
  9. 任意のコマンドの実行（パラメーター["php-app"] ["init_commands"]）。


このスクリプトはファイル「.chef / site-cookbooks / php-app / recipes / default.rb」で説明されており、コメント付きで提供されています。 アプリケーションを開発する過程で、このシナリオは新しいアクションによって補完される場合があります。



PHPプロジェクトでComposerを使用する場合、jsonファイルにパラメーター["php-app"] ["composer"] ["github_auth_token"]を入力する必要があります。そうしないと、Github APIの制限によりプロジェクトの依存関係がインストールされません。 このトークンは、 Githubアカウントの設定で生成できます。



php-appクックブックの可能な設定の完全なリストは、ファイル「.chef / site-cookbooks / php-app / attributes / default.rb」で確認できます。



プロジェクトディレクトリの設定



仮想マシン内のプロジェクトディレクトリの場所は、パラメーター["php-app"] ["project_dir"]によって制御されます。 通常、このパラメーターとともに、パラメーター["php-app"] ["vhosts"] ["variables"] ["root"]を変更して、仮想ホストが正しいディレクトリーを指すようにする必要があります。



プロジェクトディレクトリの場所のオプション：

  • プロジェクトは、共有ディレクトリ（プロジェクトのローカルディレクトリと同じ仮想マシンの「/ vagrant」ディレクトリ）にあります。
  • プロジェクトは共有ディレクトリの外にあります（たとえば、「/ home / php-app / www」）。


プロジェクトが共有ディレクトリにある場合、仮想マシンは開発者と同じファイルで動作します。



このオプションの欠点は次のとおりです。

  • 一般ディレクトリでは、すべてのファイルが仮想マシンの側で同じアクセスモード（デフォルトでは「777」）を使用します。
  • 共有ディレクトリでは大文字と小文字が区別されません（ローカルマシンでWindowsが使用されている場合）。
  • 共有ディレクトリにシンボリックリンクを作成するには、ローカルマシンの管理者権限で仮想マシンを実行する必要があります（ローカルマシンでWindowsが使用されている場合）。
  • 開発者がローカルマシンでLinuxを使用している場合、共有ディレクトリでのディスク操作は非常に遅くなります（この問題は、VagrantfileでNFSを介して共有ディレクトリへのアクセスを設定することで解決できます）。


共有ディレクトリ外のプロジェクトの場所（たとえば、「/ home / user / www」）では、実サーバーの状態に近づけることができますが、いくつかの欠点もあります。

  • 開発者は、ローカルマシンと仮想マシンの間のファイルの同期に注意する必要があります（プロジェクトブランチ間の切り替えも、ローカルプロジェクトディレクトリ内と仮想マシン内で2回行う必要があります）。
  • ローカルgitコミットは、仮想マシンでは使用できません。


プロジェクトが共有ディレクトリの外部にある場合、json-fileにパラメーター["php-app"] ["git"] ["repository"]を入力して、仮想マシンの準備時にプロジェクトをロードする必要があります。 リポジトリにアクセスするには、[“ php-app”] [“ ssh”] [“ deployment_key”]パラメータで指定された秘密鍵が使用されます。



PhpStorm / IDEA IDEには、展開サーバーを構成する機能があります。 この場合、ローカルプロジェクトファイルへの変更は自動的に仮想マシンにアップロードされます。



これを行うには、次の設定でデプロイメントサーバーを追加します（[ファイル-設定-デプロイメント]メニューの[追加]ボタンを使用）（一部の設定はjsonファイルの設定に依存します）。

  • タイプ： SFTP
  • SFTPホスト： 10.2.2.10
  • ポート： 22
  • ルートパス： / home / php-app / www
  • ユーザー名： php-app
  • 認証タイプ：キーペア（OpenSSH）
  • 秘密鍵ファイル： project_root / .chef / files / id_rsa
  • パスフレーズ：
  • WebサーバーのルートURL： demo.local
  • サーバー上の展開パス： /
  • サーバー上のWebパス： /


次に、追加したサーバーをデフォルトサーバーとして設定し（「ファイル-設定-展開」メニューの「デフォルトとして使用」ボタンを使用）、「変更されたファイルをデフォルトサーバーに自動的にアップロード」パラメーターを「常に」または「明示的な保存時」に設定する必要がありますアクション "（in"）ファイル-設定-展開-オプション "ウィンドウ"。



MySQLサーバーのセットアップ



MySQLサーバーを構成するには、外部mysqlクックブックが使用されるため、標準設定を変更するには、jsonファイルでmysqlキーを使用する必要があります。 デフォルトでは、MySQLサーバーは標準ポートでの外部接続に使用できます。



["php-app"] ["mysql"] ["root_connection"]パラメーターでは、データベース管理者のアクセスパラメーターを指定する必要があります。 管理者パスワードは、パラメーター["mysql"] ["server_root_password"]で設定されます。

作成されたデータベースのリストは、キー["php-app"] ["mysql"] ["databases"]で指定する必要があります。 

 { ... "php-app": [ ... "mysql": { "root_connection": { "host": "127.0.0.1", "username": "root", "password": "" }, "databases": [ { "name": "yii2advanced", "username": "root", "password": "", "encoding": "utf8", "collation": "utf8_general_ci" }, { "name": "yii2_advanced_tests", "username": "root", "password": "", "encoding": "utf8", "collation": "utf8_general_ci" } ] } ... ], ... }


PosgtreSQLサーバーのセットアップ



PosgtreSQLサーバーを構成するには、外部クックブック「postgresql」が使用されるため、標準設定を変更するには、jsonファイルで「postgresql」キーを使用する必要があります。 デフォルトでは、PostgreSQLサーバーは標準ポートでの外部接続に使用できます。



["php-app"] ["pgsql"] ["root_connection"]パラメーターでは、データベース管理者のアクセスパラメーターを指定する必要があります。 管理者パスワードは、パラメーター["postgresql"] ["password"] ["postgres"]で設定されます。 このパスワードは空白にしないでください。



作成されたデータベースのリストは、キー["php-app"] ["pgsql"] ["databases"]で指定する必要があります。 

 { ... "php-app": [ ... "pgsql": { "root_connection": { "host": "127.0.0.1", "username": "postgres", "password": "password" }, "databases": [ { "name": "yii2advanced", "username": "yii2advanced", "password": "", "encoding": "UTF8", "collation": "en_US.UTF-8" }, { "name": "yii2_advanced_tests", "username": "yii2advanced", "password": "", "encoding": "UTF8", "collation": "en_US.UTF-8" } ] } ... ], ... }


PhpStorm / IDEAのデバッグ設定



仮想マシン内で実行されているアプリケーションをデバッグできるようにするには、次の設定でPHPサーバーを追加する必要があります（[ファイル]-[設定]-[PHP]-[サーバー]メニューの[追加]ボタンを使用）。

  • 名前： demo.local
  • ホスト： demo.local
  • ポート： 80
  • デバッガー： Xdebug
  • パスマッピングの使用：有効（プロジェクトディレクトリはディレクトリ["php-app"] ["project_dir"]と一致する必要があります）


次に、「PHP Webアプリケーション」タイプの構成を作成する必要があります（メニュー項目「実行-構成の編集」を使用）、作成したPHPサーバーを指定します。



おわりに



結果の環境は非常に柔軟であることが判明し、すべてのソフトウェア設定は完全に透過的でカスタマイズ可能です。 Chef-Soloを使用すると、開発者による追加のアクションを必要とせずに、事実上すべてのマシン（必ずしも仮想ではない）にプロジェクトを数分でデプロイできます。
