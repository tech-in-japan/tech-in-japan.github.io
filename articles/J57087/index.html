ネットワーク経由でAcer Aspire OneにUbuntu 9.04をインストールします

「子供」のAspire Oneは、死んだラップトップを交換するようになりましたが、悲しいことに、それに付属するものを見て、Linuxシステムが返された理由がすぐに明らかになりました。 幸いなことに、非常に興味深い開発が登場しました-Ubuntu Netbook Remixがインストールされることになりました。 しかし、ここに問題があります-インストールには、フラッシュドライブ\外付けCD-ROM \カードが必要ですが、悪として、上記のいずれも家で見つかりませんでした。 ネットワークブートのためにドックを読む必要がありましたが、その瞬間に別のソリューションが見えてきました...



ご希望の方は、 元の手順を参照できますが、Oneの詳細を考慮していくつかの更新データを提供します。



したがって、ネットワークケーブルとDHCPを介して接続する必要がある問題を最小限に抑えるために、ネットワーク経由でインストールします。



最初に、最小限のシステムカーネル（ linuxおよびinitrd.gz）で 2つのファイルをダウンロードする必要があります。



Alt + F2でターミナルを開き、 xtermを実行します。 ダウンロードしたファイルを/ bootにコピーし、 chmod 755 linux && chmod + x linuxを使用してLinux実行可能ファイルを作成します。 ここで、 grub.confを編集する必要があります。これはvimを使用するか、ルートからsudoマウスパッドテキストエディターを起動します。



ここで、Grubのためにカラースキームを黒地に黒に設定し、何らかの方法でキーボードを切断することよりも優れたものを思い付かなかったLinpusの作者を悪い言葉で覚えておく価値があります。 したがって、トリッキーな方法で進みます。必要なデータを含む別のレコードを追加し、それをデフォルトのシステムとして設定します。



既存のシステムの後に追加



root (hd0,0)

kernel /boot/linux

initrd /boot/initrd.gz







default = 0をdefault = 1に変更します 。



sudo grub-install / dev / sdaコマンドを使用してgrubをインストールし 、再起動するだけです。



すべてがうまくいった場合、システムはテキストインストーラーで起動します。 彼の質問のほとんどは非常に標準的なもので、2つのことに注意を払う必要があります。 まず、ディスクのパーティション分割について-最も簡単な方法は、再パーティション分割（2つのオプション）でディスク全体の使用を示すことです。 当然、これより前のすべてのデータは人里離れた場所に保存する必要があります。 第二に、システムの役割の選択。 スペースとしてUbuntu Netbook Remixのみを選択するだけで十分です。



システムのインストールは、カード\ディスクからインストールするオプションよりも遅くなりますが、配布キットの現在のバージョンを受け取り、更新に時間を浪費しません。 全体のプロセスに約1時間かかり、その後、非常に便利で完全に機能するシステムがネットブックに登場しました。



ネットブックメーカーがubuntuに注意を払い、将来的にデフォルトで使用されることを期待することしかできません。
