初心者開発者向けのLinux、またはWindowsを永遠に忘れる方法

免責事項



注意、たくさんの手紙！ ポップコーンの在庫を増やし、このイベントの時間を確保することを提案します。



この投稿は主に開発に携わる人を対象としていますが、Linuxに切り替えようとした人にも大いに役立ちますが、理解できない問題や困難があるため、彼は次のように発言しました。私はまだ彼をいじります。」



この記事では、Ubuntu 16.04のインストールと、作業専用の構成について説明します。



背景



以前は、Linuxにロールオーバーする試みが行われましたが、Microsoft製品への過剰なアタッチメントのため、移行は再び長い箱で延期されました。



また、私は常に答え、問題、エラーなどを見つけることができなかった多数の質問によって止められました。

しかし、私がNode.jsに興味を持ち始め、研究を始めた後、ある仲間がLinuxで作業する方が便利だという口実で動いてしまいました。 そして、何度かインストールに失敗し、UbuntuをメインOSとして使用した後、貴重なWindowsを削除し、永遠に忘れてしまいました。 私はこれについて後悔していません。



インストールを開始する



最初に行う必要があるのは、起動可能なUSBフラッシュドライブまたはディスクを書き込むことです。 Ubuntu 16.04ディストリビューションの重量はわずか1.4 GBなので、フラッシュドライブが最適です。 プログラムUnetbootin-> unetbootin.github.ioをダウンロードします。







ここではすべてが非常に簡単です。 「ディスクイメージ」を配置し、ダウンロードした.isoファイルを選択し、メディアを選択して[OK]をクリックします。 レコーディングはもうすぐです。



システムを再起動し、USB経由で起動方法を選択します。 しかし、ここで問題は誰かを待つことにあるかもしれません、それは通常モードではダウンロードが起こらないという事実にあります。



フラッシュドライブを取得するために、フラッシュドライブからUEFIモードで起動しようとします。 ただし、後で、UEFIモードでインストールを続行するかどうかを尋ねられます。 Linuxにアップグレードする決定が最終的なものである場合は、肯定で答えることができます。 他のOSで起動する必要がある場合は、「戻る」をクリックします（このボタンがなぜそのように呼び出されたのかわかりません）。 インストール自体が問題を引き起こすことはありません。 言語を選択して、次に進みます。 次のウィンドウにチェックを入れることをお勧めします。







次のウィンドウで、アイテム「別のオプション」を選択します。







次に、ディスクをパーティション分割する必要があります。 2台のハードドライブがあり、そのうちの1台が使用されておらず、その上にデータがない場合は、それを選択して[-] [新しいパーティションテーブルの作成]をクリックします。



「空き領域」があり、それをクリックして最初のセクションを作成します。



セクション1：



ルートパーティションのサイズは4GBにすることができますが、マージンをとるために40〜50を置くことをお勧めします。

タイプ-プライマリ

場所-すべてのセクションで「このスペースの始まり」である必要があります

使用法-Ext4

マウントポイント-「/」



セクション番号2：



サイズ= RAMのサイズ

使用法-スワップセクション

このセクションは、ハードドライブに保存されているバックアップ「運用」メモリの役割を果たします。



セクション3：



サイズ=残りのすべてのスペース

タイプ-論理

マウントポイント-「/ home」



以上で、[続行]をクリックします。パーティション上のすべてのファイルが破棄されることに同意します。 さらに-私たちは同意し、私たちは尋ねるものを入力します。 今、あなたはコーヒーを飲むことができます）



システムのセットアップ、設計、鉄の機能の調整



インストール後、もちろん現在のOSを再起動してログインします。 システム設定->プログラムと更新に進み、最初のタブでこのようにチェックマークを付けます：







すぐにいくつかのコマンドを実行します： 



sudo apt update sudo apt upgrade


sudoは管理者を意味します



最初のコマンドの後、インストール段階で思いついたパスワードを尋ねられます。 パスワードを入力しても文字が表示されなくても心配しないでください-これはセキュリティ対策です。 パッケージのインストールとカーソルによる友好的な点滅について尋ねる場合は、Yと書いてEnterキーを押してください。 では、このチームで何をしたのでしょうか？ 「apt」パッケージマネージャーにコマンドを与えて、利用可能なパッケージのリストを更新し、インストールします。 作業の原則はGITに基づいており、リポジトリをオンにし、それらを更新し、更新プログラムをインストールします。 リポジトリのリストは/etc/apt/sources.listにありますが、これはほとんど必要ありません。



更新は完了しました。Linuxでは通常これは必要ありませんが、カーネルは更新されている可能性があるため、この操作は引き続き実行する必要があります。 一般に、更新後に同様のコンテンツの行が表示される場合： 



 linux-headers, linux-generic


更新がカーネルに影響するため、再起動が必要です。



鉄の問題



個人的に、私はすぐにマウスに問題がありました。 感度が3800DPIのゲーミングマウスを使用しています。 当然のことながら、彼女はわずかな動きからモニターの周りを飛び回った。 そして、モニターの表示に関する問題。 私は4：3の比率で2x 17inchを持っています。 1つのモニターはマザーボードに接続され、もう1つのモニターはAMDのビデオカードに接続されています。 最初のものはすべて問題ありませんでしたが、統合されたグラフィックスのものはシステムに入った直後にフェードアウトしました。 起動時にマウスを遅くしてモニターを再定義するためのスクリプトを書くことになりました（誰かが役に立つかもしれません）。



速すぎるマウスで問題を解決する



コンソールでxinputと入力し、接続されているデバイスのリストを表示します。 マウスを見つけて、そのIDを書き換えます。 マウスID = 11で、コマンドxinput list-props '11'を入力します。 リストが再び表示されますが、オプションのセットを使用するには、Device Accel Constant Decelerationが必要です。そのIDを括弧内に示します。 274になりました。マウスIDとIDを知って、次のように記述します。 



 xinput set-prop '11' 274 2


ホームディレクトリで、start.shファイルを作成して開きます。最後の数字（2）は、マウスの速度を落とす必要がある回数を意味します。 このコマンドをスクリプトにすぐに追加できます。 



 #!/bin/bash xinput set-prop '11' 274 2


なぜ設定に入らず、そこで感度を削除しなかったのですか？ 調整範囲が小さいため。 そして、おそらく、それだけでは十分ではありません。



誤って表示されたモニターの問題を解決する



設定->画面設定に移動します。 2つのモニターが表示されます。そのうちの1つをつかんで必要に応じて配置し、ランチャーを配置する場所を選択できます。 [適用]をクリックします。再起動後、すべてがその場所に残っている場合、それ以上のアクションは不要です。



しかし、BIOSで設定された統合グラフィックスの優先度により、入力後、2番目のモニターはオフになり、ビデオカードに接続されていたモニターは残りました。 注目すべきこと-スリープモードから目覚めた後、2つ目は生き返り、正常に動作しました。 モニターの番号付けを見てみましょう。メインモニターには「プライマリ」というラベルがあります。 



 xrandr


数字に従って、モニターの配置方法を設定します。 スクリプトに追加します。 



 #!/bin/bash xinput set-prop '11' 274 2 xrandr --output VGA1-1 --primary xrandr --output VGA1-1 --left-of VGA1


ここで、起動時にスクリプトを登録し、ダッシュメニュー（WINキー）を開いて検索を入力します。



「自動的に起動されるアプリケーション。」 [追加]をクリックし、名前を入力します（何であれ、主なことは、この要素の目的の意味を理解することです）。説明は空のままにします。 「チーム」フィールドに、次を入力します。 



 sh /home/user/start.sh


したがって、スクリプトを実行するコマンドを提供します。 他の方法もあります。 保存し、次の再起動時に操作性を確認します。



それでもモニターの問題が解決しない場合は、BIOSへのビデオ出力の場所と優先順位を考えてみてください。

ドライバーに関して-AMDの場合、ドライバーなしですべてが正常に動作するため、インストールしませんでした。ドライバーをインストールすると、システムがクラッシュする可能性があります。



システムのセットアップとプログラムのインストール



Windowsフォントの追加



ブラウザーをインストールした後（少し後で）、いくつかのサイトのフォントが目を痛め、そこから泣き出したいことに気づくでしょう。 



 sudo apt install msttcorefonts sudo fc-cache -fv


ブラウザを再起動すると、フォントが正しく表示されるはずです。 必要に応じて、それらのいずれかをシステム設定として設定できます（設定またはubuntu調整ツールで）。 私はubuntu微調整に行き、「フォント」タブで設定することをお勧めします：







すでに含まれているリポジトリのソフトウェア



正規パートナーリポジトリに既に含まれているプログラムは、次のようにインストールできます。 



 sudo apt install < >

サードパーティソフトウェアのインストール



サードパーティのソフトウェアをインストールするには、システムにリポジトリを追加する必要があります。



電報 



 sudo add-apt-repository ppa:atareao/telegram sudo apt update sudo apt install telegram


Yandexブラウザー



公式ウェブサイトからパッケージをダウンロードして開き、マネージャーで「インストール」をクリックします。

ビデオを見るには、フラッシュが必要です。インストールしてください。 



 sudo apt install pepperflashplugin-nonfree


ブラケット



かっこで作業するのが好きで、とても便利で便利です。 



 sudo add-apt-repository ppa:webupd8team/brackets sudo apt-get update sudo apt-get install brackets


次に、テーマをインストールします。numixを使用しますが、他のものを見つけてインストールすることもできます。 



 sudo add-apt-repository ppa:numix/ppa sudo apt-get update sudo apt-get install numix-gtk-theme numix-icon-theme n1umix-icon-theme-circle


インストール後、Unity Tweakツールが起動します。 テーマは「Numix」に設定され、アイコンも「Numix」です。 暗いトピックが好きですか？ Ubuntuの微調整ツールを起動し、「外観」タブで「ダークテーマをグローバルに適用」に設定します。 ubuntu tweakを終了するまで、起動時にskypeとtelegramを登録することをお勧めします。



LAMPをインストールする



最後に、LAMPサーバー（Linux apache mysql php）とphpmyadminをインストールします。 



 sudo apt install lamp-server sudo apt install phpmyadmin sudo phpenmod mcrypt sudo apt-get install php-mbstring php-gettext sudo apt-get install php5-dev


Apacheを再起動します。 



 sudo service apache2 reload


localhostアドレスにアクセスしようとします。 すべてが正常である場合、次に進みます。 Apache構成ファイルを開きます。 



 sudo nano /etc/apache2/apache2.conf


最後に次の構成を記述した後、サーバーのファイルの場所に関する情報が書き込まれているタグが見つかります。



<Directory /home/user/public_html>

AllowOverride All

Require all granted









パスは、どちらが便利かを示すことができます。 構成ファイルの最後に、次のように記述します。



ServerName localhost







注意！ 以下に説明する操作は、新しい仮想ホストを追加するたびに実行する必要があります！



新しい仮想ホストを追加するには、次の手順を実行します。 ディレクトリ/ home / user / public_html / testにフォルダを作成し、/ etc / hostsを開いて、サイトの名前を最初の行に追加します。 



 127.0.0.1 localhost testsite


次に、ターミナルで実行します。 



 sudo cp /etc/apache2/sites-available/000-default.conf /etc/apache2/sites-available/testsite.conf


したがって、サイトの既定の構成ファイルをコピーして開きます。 



 sudo nano /etc/apache2/sites-available/testsite.conf


行のコメントを解除します。 



 ServerName www.example.com


example.comの代わりに名前を入力します。 次に、標準/ var / www-/ home / user / public_html / testの代わりに、サイト上のファイルの場所を示すDocumentRootを調整します。



これで、サイトを有効にできます。 



 sudo a2ensite testsite sudo service apache2 reload


以上です。 基本的なセットアップが完了し、使用できます！
