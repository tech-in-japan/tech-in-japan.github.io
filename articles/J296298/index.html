ボリュームライセンスまたはdo SAM

ソフトウェア資産管理に対する集中型アプローチに関する7月の記事で、ライセンスコスト、サポート、およびソフトウェアの展開を最適化することを目的とした全体的なアプリケーション管理プロセスのビジョンについて話しました。 展開のためのアプリケーションの準備について説明したので、今度はソフトウェア管理の同様に重要な側面、つまりライセンスコンプライアンス管理、つまりソフトウェア資産管理に焦点を当てたいと思います。









現在、この法律は、ライセンスのないソフトウェアの使用に関する管理上、民事上および刑事上の責任を規定しています。 管理責任はアートによって規制されています。 管理違反のコードの7.12、および従業員にライセンスのないソフトウェアの使用を許可する職員およびプログラムおよびコンピューターの押収に対する罰金を規定しています。 著作権および関連する権利の侵害に対する刑事責任は、著者または他の著作権者に重大な損害が生じた場合に発生し、プログラムのコピーの費用が250,000ルーブルを超える場合、そのような違反は考慮されます。 このような違反には、最大6年間の懲役、最大500,000ルーブルの罰金も含まれます。 刑事責任は、ロシア連邦刑法第146条によって規制されています。



損害額は、違法に使用されたソフトウェアのコピーの数に基づいて決定されます。



民事責任は、著作者または著作権者の主導によりもたらされる可能性があり、損害賠償または補償が必要です。 アートに基づきます。 ロシア連邦民法の1301年、著作権者は、通常合法的な使用に対して請求される価格に基づいて、10,000から5,000,000ルーブル、またはプログラムの2倍の金額の違反者に補償を要求できます。



SAMはロシアで積極的に開発されています。企業は、適切で有能な管理がない場合、企業の規模の拡大に伴い再ライセンスおよびライセンス不足のリスクが大幅に増加することを理解しています。また、SAMアプローチにより、安らかに眠り、ベンダー監査を恐れることなく、大幅に節約することもできます。



クライアントソフトウェアの場合でも、ITSMおよび構成ユニットのベースに関するSAMプロセスの成熟度が不十分であると、不必要なライセンス（たとえば、アクティブに使用されていない既存のライセンス）の購入につながる誤った決定が採用される可能性があります。実際にライセンスが解除される可能性があります。 コンプライアンスを手動で監視し、多くの場合異なるシステムですべての変更を追跡します（実際には、サーバーライセンスはExcelファイルで分散されていることが多く、ユーザーライセンスはITSMおよびCMDBと統合されていない別のAsset Managementデータベースに保存できます）また、ビジネスで必要なときに適切な意思決定を行うことは非常に困難です。 意思決定の基礎は多くの場合CMDBからの情報に基づいているため、ここで間違いを犯すのは非常に簡単です。CMDBは不規則に更新されるか、手動処理が遅れて更新されることがあります。 さまざまな下位組織と場所の間のデータリンクの欠如、およびソフトウェアの使用を分析し、ライセンスを再配布するプロセスの欠如は、多大な経済的コストにつながります。 たとえば、企業のさまざまな部門では、ローカルITサービスがライセンスの記録をさまざまな方法で保持しています。Excelファイルにあるもの、別のExcelファイルにあるもの、ITSMシステムにあるもの、ローカルCMDBソリューションを使用するものなどです。 部門の1つで、ユーザーはソフトウェアをインストールするためのアプリケーションを起動します。 ソフトウェアの利用可能なコピーが見つからなかったため、ITサービスはライセンスの購入要求を開始し、場合によっては、単一のライセンスだけでなくボリュームライセンスパッケージ全体も購入できます。 さらに、会社の他の部門や同じ部門に未使用のライセンスがある可能性もあります。ソフトウェアはインストールされていますが、ユーザーは実際に使用していません。 また、ソフトウェアのすべての部門のすべてのニーズを統合したことで、ボリュームライセンスプログラムのコストをさらに削減できます。







災害復旧-緊急時にインフラストラクチャを復元する計画を忘れないでください。 環境全体でライセンスステータスの制御をできるだけ早く復元するには、データをデータベースに保存して保護することが重要です。また、CMDBは常に別のソース、つまりインベントリデータベースとライセンスデータベースとの自動同期を取得できます。



Request Fulfillment（サービスリクエスト管理）とライセンス管理システムおよびSAMプロセス、機能、バージョン、およびエディションによるソフトウェアの標準化（つまり、ソフトウェアの合理化、組織でのソフトウェア使用の分析など）のITSMプロセスの統合の場合、そして、最も重要なことは、会社全体のソフトウェアおよびライセンスに関する動的データであり、CMDBおよびライセンス会計システムで自動的に利用可能です。 そしてこれは、すべての決定が正しい関連データに基づいて行われることを意味します。







Microsoft SAM BaselineレビュープロジェクトでSAMプロセスの構築を開始する必要があります。これにより、Microsoft製品のライセンス準拠の迅速な分析と正確な把握が可能になるだけでなく、SAMプロセスの成熟度レベルの専門的評価も提供されます。



もちろん、SAMはソフトウェアライセンスのコンプライアンスレベルの制御に限定されるものではなく、Microsoftソフトウェアに限定されるものではありませんが、これはライセンス経済の分野で最適化を開始するための良い出発点です。



SAMの基本成熟度モデルは、4つの成熟度レベルを想定しています。



  • 基本-ライセンス不要を排除する状況対応アクション。
  • 標準化-プロセス、ツール、および標準がありますが、インベントリの精度は十分に高くなく、データは意思決定に使用されません。
  • 合理化-IT資産を管理するためのポリシー、手順、およびツールがあります。 情報は信頼でき、意思決定に使用されます。
  • 動的-すべてのライセンス管理は、ほぼリアルタイムモードでビジネスのニーズに適応します。


このようなスキームは、特定の分析作業を意味します。 たとえば、会社で使用されているソフトウェアの同じ合理化、およびライセンスの再配布プロセスの制御によって。 この作業はかなり手間がかかりますが、動的で正確なデータが1か所で便利な形式で収集されるため、このタスクが大幅に簡素化され、結果としてアナリストのリソースのコストが大幅に削減されます。



Colibri Application Intelligence Portalを開発することにより、データ統合だけでなく、このデータの強力な組み込みビジネスインテリジェンスのおかげで、データを合理化するための分析作業の量を大幅に削減できます。



このソリューションには、たとえば、未使用のアプリケーションの割合に関するデータや、ソフトウェアレコードの他のカスタムカテゴリなど、常に目の前にある便利なインジケーターパネルがあります。







Colibri Application Intelligenceポータルを使用すると、事前に構成されたカスタムパターンを使用してソフトウェアを分類する分析者の時間を節約し、ゴミ、不要なソフトウェアを自動的に分類し、特定のバージョンでアプリケーションを標準化し、アプリケーションの使用状況を追跡できます。 Decilesなどの便利なデータ表現は、ソフトウェア移行プロジェクトからすぐに利益をもたらします。このタイプは、ユーザーとアプリケーションを異なるグループ（10の部分）に分類します。たとえば、最初のグループは、ユーザーが使用するアプリケーションのみを表示します。







このデータの提示は、サービスのアウトソーシングの計画に役立ちます。 たとえば、3番目の十分位に基づいて、24個のアプリケーションのインストールパッケージの作成を計画できます。これにより、ユーザーの93％のニーズに対応できます。



したがって、Colibri Application Intelligence Portalは、ソフトウェアの合理化、移行プロジェクトに必要なソフトウェアおよびハードウェアの変更の計画、またはアプリケーション管理のアウトソーシングの時間コストの最適化など、強力なサポート移行計画ツールです。

次のケースでは、Citrix GoTo Assistは39の異なる商用ソフトウェアタイトル（バージョン管理、つまり一意の名前とバージョンの組み合わせを考慮）に要約し、これは、この特定のケースでは、ソフトウェアの250のインストールされたコピー、つまりライセンスになりました。







リモートで支援を提供する製品には、ソフトウェアの実際のインストール数とは異なる多数のライセンスが必要になる場合があることを理解することが重要です。 つまり、たとえば、同時に実行されているセッションの数に対してライセンスを付与できます。 ここで、ソフトウェアの使用に関する統計が役立ちます。これにより、ライセンス数の実際の必要性を評価できます。 また、機能とバージョンでソフトウェアを標準化することにより、ベンダーが提供する購入量から利益を得られるだけでなく、サポートサービスの頭痛を軽減できることを理解することも重要です。 インストール手順の作成にも時間がかかるため、間接的なコストを過小評価しないでください。



データセンターの仮想化サーバー（SQL、Oracle、SAP）での複雑なライセンスについて話している場合、ソフトウェア使用統計の分析の問題が特に重要になります。



たとえば、SAPの作業の性質またはトランザクションのタイプを分析すると、ユーザーが製品ライセンスの高価なバージョンを使用して、次のような単純なアクションを実行することがわかります：タイムシートでの作業。ライセンス。







したがって、適切なSAMプロセスを構築するには、次の手順が最も重要です。



  • ライセンスコンプライアンスの現在の状態の評価。
  • 既存のSAMプロセスの成熟度評価。
  • ソフトウェアの使用に適したインベントリおよび監視ツールの選択。
  • ライセンス管理ツールの選択。
  • ソフトウェアインベントリ;
  • ソフトウェアの合理化。
  • 調達、変更管理、およびサービスリクエスト管理とのインターフェースに関するSAMプロセスの最適化。




動的SAMの構築について説明する場合、これらの手段をおおよそ次の順序で実装できます。







上部は、短期的に最も具体的な対策を目的としています。標準ソフトウェアのリストを作成することにより、罰金のリスクを排除し、ライセンスとソフトウェアサポートのコストを削減します。 一番下の行に示されている測定値はより「長期的」ですが、同時に、考えられるすべての状況でSAM生物が正しく機能することを保証する根本的な要因です。



SAMプロジェクトの実装を成功させるには、在庫管理とライセンス管理の選択が非常に重要であることに注意してください。 現在、市場には多くのソリューションがあります。基本的な機能を提供する単純なものから、複雑なライセンスシナリオを含むクライアントワークステーションとデータセンターのライセンスを完全にカバーできる複雑なソリューションまでです。 その中には、同時アクセスのライセンス、コアおよびプロセッサの数、仮想化およびその他の種類のライセンスの詳細を考慮し、ソフトウェアの使用に関する「高度な」分析を提供します。 そのようなソリューションの1つがSnow License Managerです。 このシステムは、毎日更新されるソフトウェア定義のデータベースなど、多くの利点に加えて、他の在庫システム、ITSMシステム、調達管理システム、および企業との豊富な統合機能によって区別されます。 ツールとSAMプロセスの実装の段階では、これらの接続を自動化する必要性について事前に考えることが非常に重要です。これは関連プロセスでサポートされる必要があります。



一般的に、SAMはまず企業のソフトウェアコストを最適化するための戦略であり、ソリューションはテクノロジー、プロセス、人材、分析の接点にあります。 目的の効果を達成するために、これらのコンポーネントが互いにどのように相互作用する必要があるかを理解することにより、重要な役割が正確に果たされます。 ボリュームライセンスの分野での衝撃に耐える、統合されたしっかりした建物にレンガを正しく配置するのに役立つソフトウェア管理のすべての接続を理解することです。
