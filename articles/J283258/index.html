[ピーター] OpenOnloadについての会議：Linux用の高性能ネットワークスタック

5月12日木曜日、20：00にOdnoklassnikiのサンクトペテルブルクオフィスで、CodeFreezeはOKTET LabsディレクターのKonstantin Ushakovと会談します。 会議のトピックは、レイテンシー、CPU負荷、パケット伝送速度を削減するLinuxのネットワークスタックであるSolarflare OpenOnloadです。







Solarflare OpenOnloadは、レイテンシ、CPU負荷、パケット転送速度を大幅に削減する高性能ネットワークスタックです。 これは、C / C ++ / JavaまたはBSD Sockets APIを使用するものであっても、アプリケーション自体に変更を加えることなく達成されます。 スタックがユーザーランドで部分的または完全に機能するという考え方は新しいものではありませんが、Onloadの根本的な違いは、BSDソケットAPIの提供にあります。



このレポートは、実装の技術面に影響を与えます。

  • コアとプロセス間でリソースを共有します。
  • パケットルーティング;
  • ポーリングメカニズム（ポーリング、エポールなど）;
  • ソケットキャッシュ。


Konstantinは、基礎となる単純なアイデアの実装がそれほど単純ではない理由を示します（「失敗した」試みの例を示します）。 数字があり、Linuxカーネルが同様のパフォーマンス（DPDKを含む）を達成しようとしている方法の詳細があります。
