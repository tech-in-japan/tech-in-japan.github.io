モスクワ取引所での取引オプションのテクノロジースタックとは





Habréのブログでは、株式市場での作業に使用されるハードウェアとソフトウェアについて多くのことを書いています。 少し前まで、高頻度取引のための技術のスタックを検討していましたが、今日はモスクワ取引所でオプションを取引するために使用できるツールに焦点を当てます。 このテーマに関する資料は、 ITinvestの専門家であるOleg Mubarakshinによって作成されました。



はじめに



この記事では、モスクワ取引所でオプションを取引するために使用するテクノロジーについて詳しく説明します。 この記事は、取引ソフトウェア（取引所へのアプリケーションの送信など）についてではなく、オプションとポジションのリアルタイム評価に使用する技術ソリューションについてです。



現時点では、オプショントレーダーにはオプションとポートフォリオを分析するための多くのツールがあり、その多くはすぐにトレーディングターミナルに組み込まれています。 ただし、次のようなさまざまな理由で、これらは私のタスクに適しておらず、今日にも適していない コストが高いため（優れたソリューションは非常に高価です）。 そのため、2年前、必要に応じてオプションとポジションを分析できる独自のソフトウェアを作成することにしました。



オプションのソフトウェアにはいくつかの厳しい要件があります。



  1. リアルタイムの市場データ。
  2. 安価なソリューションですが、より無料です。
  3. 価格設定の面で柔軟性があります（非常に重要です！しかし、これを理解している人はほとんどいません）。
  4. コードを記述し、堅牢な方法を使用するという点でシンプルです（ほとんどが既に優秀な人々と経験豊富な開発者によって作成されていることが望ましい）。
  5. 展開と共有の点で便利です。
  6. ユーザーフレンドリーなインターフェース。
  7. バグと過失は保護されています（価格設定エラーは肉眼ですぐに見えるはずです）。


重要ではなかったもの：



  1. HFT（低レイテンシ）は必要ありません。リソースを費やす意欲も機会もありません。
  2. 情報セキュリティ、ユーザー権利管理など （はい-良い、いいえ-大丈夫）;
  3. 著作権保護


それで、この2年間で私が何に来たか見てみましょう。

ハードウェア



Intel Core i7プロセッサ、8 GBのRAM、Windows 7を搭載したラップトップをハードウェアとして使用し、Shiny ServerにはLinuxが必要なため、WMware Player（無料）を使用して、Ubuntu Server 14.04の制御下で仮想マシンを作成しました。



市場データ



市場データ-先物およびオプションの現在の相場（買い、売り、最終、出来高など）-ODBCエクスポートを使用して、QUIKからPostgreSQLデータベースに配信されます。 このデータフィード方法はうまく機能し、安定して迅速に機能します。







図 2. QUIKからDBへのODBCエクスポート



ITinvestブローカーの開発であるSmartCOM APIを使用して、現在の位置（ポートフォリオ）がデータベースにロードされます。



データベース



PostgreSQLはWindows 7にインストールされます。QUIKからのデータは、トリガーとデータベース関数を使用して処理され、特別なテーブルに保存されます。 たとえば、現在の市場オプションデータを含むQUIKテーブルがエクスポートされるmw_optionsテーブルには、トリガーが含まれます。 



CREATE TRIGGER trig_upsert_spec_options AFTER INSERT OR UPDATE OF sec_code, sec_name, lot, tick, fee ON mw_options FOR EACH ROW EXECUTE PROCEDURE func_upsert_spec_options();


呼び出し関数： 



 CREATE OR REPLACE FUNCTION func_upsert_spec_options() RETURNS trigger AS $BODY$ BEGIN UPDATE spec_options SET sec_name=NEW.sec_name, lot=NEW.lot, tick=NEW.tick, fee=NEW.fee WHERE sec_code=NEW.sec_code; IF NOT found THEN INSERT INTO spec_options(sec_code, sec_name, und_code, expiry, strike, type, lot, tick, fee) VALUES (NEW.sec_code, NEW.sec_name, NEW.und_code, NEW.expiry, NEW.strike, NEW.type, NEW.lot, NEW.tick, NEW.fee); END IF; RETURN NEW; END; $BODY$ LANGUAGE plpgsql VOLATILE COST 100; ALTER FUNCTION func_upsert_spec_options() OWNER TO postgres;


オプション契約の仕様が保存されているspec_optionsテーブルのデータを追加/更新します。 



 CREATE TABLE spec_options ( sec_code character varying(16) NOT NULL, sec_name character varying(32), und_code character varying(8), expiry date, strike double precision, type character varying(4), lot integer, tick double precision, fee double precision, CONSTRAINT spec_options_pkey PRIMARY KEY (sec_code) ) WITH ( OIDS=FALSE ); ALTER TABLE spec_options OWNER TO postgres;


光沢のあるサーバー



Shiny ServerはWindowsにインストールできないため、仮想マシンはUbuntu（trusty）で作成されました。

次に、Rの最新バージョンをインストールする必要があります。UbuntuにはすでにRへのリンクがありますが、原則として古いので、これを行う方が良いです。 

 $ sudo sh -c 'echo "deb http://cran.rstudio.com/bin/linux/ubuntu trusty/" >> /etc/apt/sources.list' $ gpg --keyserver keyserver.ubuntu.com --recv-key E084DAB9 $ gpg -a --export E084DAB9 | sudo apt-key add - $ sudo apt-get update $ sudo apt-get -y install r-base


Rを実行してバージョンを確認できます。 



 $ R


Rを終了するには、コマンドを入力する必要があります。 



 > q(save = "no")


次に、必要なRパッケージ（shiny、devtools、rmarkdown、shinyjs）をインストールします。 



 $ sudo su - -c "R -e \"install.packages('shiny', repos = 'http://cran.rstudio.com/')\"" $ sudo apt-get -y install libcurl4-gnutls-dev libxml2-dev libssl-dev $ sudo su - -c "R -e \"install.packages('devtools', repos='http://cran.rstudio.com/')\"" $ sudo su - -c "R -e \"install.packages('rmarkdown', repos='http://cran.rstudio.com/')\"" $ sudo su - -c "R -e \"devtools::install_github('daattali/shinyjs')\""


次に、Shiny Serverをインストールします。 ここに最新バージョンをインストールする手順は、この記事を書いている時点では1.4.1.759でした。 



 $ sudo apt-get install gdebi-core $ wget https://download3.rstudio.org/ubuntu-12.04/x86_64/shiny-server-1.4.1.759-amd64.deb $ sudo gdebi shiny-server-1.4.1.759-amd64.deb


すべてが正常に完了したら、リンクを呼び出します（ ip_shiny_server



仮想マシンのIPアドレス）

ip_shiny_server:3838



つのアクティブなウィジェットを備えたShinyサーバーのウェルカムページが表示されます。







図 3. Shiny Server Welcomeページ



また、非常に便利なRstudio Server開発環境をインストールすることをお勧めします。 最新バージョンのインストール手順はこちらです。 インストールコマンドの例： 



 $ sudo apt-get install gdebi-core $ wget https://download2.rstudio.org/rstudio-server-0.99.491-amd64.deb $ sudo gdebi rstudio-server-0.99.491-amd64.deb


また、Shinyサーバーの構成に/etc/shiny-server/shiny-server.confファイルの次の変更を加えることをお勧めします。 



 server { listen 3838; location /users { run_as :HOME_USER:; user_dirs; } location /apps { run_as shiny; site_dir /srv/shiny-server; log_dir /var/log/shiny-server; directory_index on; } }


これにより、Rstudio Serverで開発された光沢のあるアプリケーションを便利に実行できるようになります。



  • Rstudioでは、ルートディレクトリにShinyAppsフォルダーが作成されます。
  • 次のファイル/新しいプロジェクト../新しいディレクトリ/ Shiny Web Application /;
  • ディレクトリ名を入力します（例：test001）。
  • プロジェクトのルートディレクトリとしてShinyAppsを選択します。


作成したアプリケーションをブラウザーで起動するには、アドレスを入力するだけです：



ip_shiny_server:3838/users/ubuntu_user_name/test001/





ここで、 ip_shiny_server



は仮想マシンのIPアドレス、 ubuntu_user_name



はUbuntuユーザー名です。



PostgreSQLとlibpq-devパッケージもインストールする必要があります。 



 $ apt-get install postgresql-9.4 $ sudo apt-get install libpq-dev


たとえば、ファイルを共有したり、ログをすばやく表示したりするために、Apachも必要になる場合があります。 



 $ sudo apt-get install apache2


ここで、たとえば、test001アプリケーションのfolder001フォルダーにアクセスするには、次のコマンドを実行する必要があります。 



 $ sudo ln -s /home/ubuntu_user_name/ShinyApps/test001/folder001 /var/www/html/shared_folder


その後、フォルダーの内容はip_shiny_server / shared_folder /で利用可能になります。

共有する



Shinyアプリケーションのもう1つの便利で便利な機能は、簡単に共有できることです。 少なくとも2つのソリューションがあります。アプリケーション全体を共有するか、結果のみを共有します。 Shinyサーバーでホストされているアプリケーションを共有するには、ポート3838を転送し、ユーザーにルーターのIPアドレスへのリンクを与える必要があります。 これは最高のセキュリティソリューションではありません。 私の意見では、結果のみを共有する方がはるかに良いです。たとえば、プロットサービスを使用して整理するのは非常に簡単です。 Shinyアプリケーションから直接、グラフをクラウドプロットサービスに投稿できます。 



 pp <- plotly_POST(p, filename=plotly.name, fileopt="overwrite")


ユーザーにこのグラフへのリンクを提供します。 さらに、陰鬱なグラフィックスをWebサイトまたはブログに埋め込むことができます。
