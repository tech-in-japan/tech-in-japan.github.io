MGTSからGPONルーターにダイヤルする際の問題を処理します

GPONはモスクワを歩き回ります。 関連する問題が広がっています。 この記事では、MGTSルーターと既存の電話との非互換性を処理した経験を共有したいと思います。既存の電話は、番号をダイヤルするとランダムエラーが発生します。



歌詞
むかしむかし（2〜3年前のようです）、私は自分をバックアップインターネットプロバイダーにしたかったのです。 そして、私はMGTSからの毎日のADSL関税 （ アーカイブ ）に目を向け始めました。 そのとき、私は自分のために「GPON」という新しい言葉に出会ったのですが、ADSLをインストールするのは遅すぎることに気付きました。 次に、プロファイルのトピックを確認した後、この設計の信頼性と妥当性について最初の否定的な印象を受けました。



高齢の親relativeで見た最初のライブGPON。 MGTSが提供するすべての機能の中で、彼らは通常の電話通信のみに関心があり、その信頼性は大幅に低下していると彼らは言いました。 そして今週、GPONが来ました。

忘れて：

  1. MGTSが必要な理由
  2. 同じWPS PINなどのセキュリティ問題の許可方法
  3. ルーターから他に何を絞り出すことができますか （それ以外の場合は、長時間座ることができます）
  4. WiFi電話番号から「デフォルト」パスワードを思いついたすばらしい人々について。
  5. インストーラーのい鈍いクリーバーとは何ですか（もしあれば、クリーバーは溶接前にきれいで均一な表面を得るためのデバイスです）。
  6. プロセスの半分は体重がかかるため（そして残りの半分は膝の上）。
  7. 彼らは包丁の「湿った呼吸」などの「チップ」さえ知らないので（繊維を保持しているガムが少し良くなるように）。
  8. 溶接が多少まともな独特の悪いチップを溶接することで小さな奇跡を作り出そうとしているのを、なぜ彼らは愚かに見ているのですか（溶接を止めても、1つのチップだけをやり直すだけで十分です）。
  9. 屈折計や電力計さえないのはなぜですか






最初の鐘



インストーラーの存在下で、 Philips SE 2752のダイヤルが不安定になったことを確認することができました。3回に1回携帯電話で電話をかけてきました。 さらに、パルスモードとトーンモードの両方で効果はほぼ同じでした。 インストーラーから、同様の問題を抱えているのは自分だけではないことを知りました（実際、この作品を書くように促されました）。 それでも、メザニンから古いnonameチューブが取り外されました。このモードでは、パルスモードでのダイヤルアップは正常でした（トーンモードでも同様のバグが観察されました）。 さよならを言いました。



私が開発した数年間の使用に対する「SE 2752」に関する意見は決して高くありません。 4台の携帯電話のうち2台で、ディスプレイは2年以内に死亡しました（そして、問題はディスプレイにあるのではなく、連絡先にあります-ディスプレイはちょうどそこにあり、連絡先に押されています）。 したがって、フィリップスは極端に任命されました。

タイミング



私の疑いでタイミングが最初でした。 このような設定はすでに2つあります。 最小フックフラッシュおよび最大フックフラッシュ。 私の耳では、「1トーンの最大再生時間」には450ミリ秒で十分でしたが、80ミリ秒では最小時間で問題が発生することがあると言われました。 そのため、このパラメーターが80未満に設定されていないことがわかり、非常にがっかりしました。[保存]をクリックすると、以前の値にリセットされます。 この場合の最大値は「Max-Hook-Flash」です。 「Max-Hook-Flash」では、最小値は明らかにMin-Hook-Flashに等しく、最大値は2000です。

DTMF標準の仕様を見つけることは難しくありません（ただし、もちろん、このケースとの関連性は別の問題です）。

敬意を表した読者が少なくともロシア語版ウィキペディアに記載されている範囲でDTMFデバイスを知らない場合（私もそれを知りませんでしたが、私にとっても今日まで）-読むことを強くお勧めします。 便利です。 そうでない場合、それ以上明確にはなりません。

測定



念のため、仮定を確認し、パルス幅を測定することにしました。 もちろん、通常のオシロスコープを使用することもできますが、 突然配線が面倒になるのが面倒になり 、より少ない血で問題を解決したかったのです。 「オフフック」という番号をダイヤルすると、同じ音が鳴り、N7100（一般的にはSamsung Galaxy Note II） のWAVレコーダーに録音されました。 「 rec1 」と「 rec2 」の2つの「組み合わせ」を書き留めました 。

さらに、Matlab R2014aを使用して、組み合わせごとに2つのスペクトログラムが作成されました。

トーン周波数を評価するには：時間分解能10ms、周波数分解能20Hzで： 

Fs = 41000; % Sampling frequency window = 0.01/(1/Fs); % Time resolution is 0.01 second noverlap = 0; % No "smoothing" nfft = 0:20:2000; % Frequencies spectrogram(data,window,noverlap, nfft,Fs);
