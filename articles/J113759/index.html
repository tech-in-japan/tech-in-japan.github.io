Java / Python SDK 1.4.2がリリースされ、ロードマップが更新されました





最初に、SDKの変更点について説明し、次に今後6か月間に開発者が私たちに約束することについて説明します。



だから

-Capabilities APIは実験的な機能ではなくなりました。 JavaとPython SDKの両方で利用できます。

-XMPP v2。 プレゼンスとサブスクリプションのサポートが追加されました。

-TaskQueueからタスクをプログラムで削除する機能が追加されました。

-キューの補充の最大速度を100 /秒に増加しました。

-URLフェッチにメソッドが追加され、https経由でホストにアクセスするときに証明書を検証できるようになりました。

-Memcache APIの改善。







ロードマップに渡します。



古いものから：

-非appspot.comドメインでのSSLアクセス

-30秒以上実行可能なバックグラウンドサーバー-これはすでに部分的に実装されています。

-App Engineデータセット全体でMapReduceジョブを実行するためのサポート-シャッフルとリデュースフェーズがまもなく約束されます。マップフェーズはすでに準備が整っており、実行可能です。

-バルクデータストアインポートおよびエクスポートツール

-アプリケーションサービスの監視とアラートの改善

-HTTPリクエストとレスポンスのサイズ制限を引き上げる-バージョン1.4.0では多くの制限が既に引き上げられています。

-BlobstoreでのプログラムによるBlobの作成



新しいから。

-データストアを介した全文検索-さて、最後に（!!!）

-Python 2.7のサポート

-サイズとストレージの制限を取り除くためのロギングシステムの改善

-Google Storage for Developersとの統合-すばらしい、私見

-Channel APIの割り当てとプレゼンスの改善



SDK分析：

-一部のファイルとlogserviceパッケージはPython SDKに登場しました。これまでのところ、ファイルにはPBプロトコルの記述しかありません。これは、この部分がアプリケーションからプログラムでblobstoreにファイルを作成することに関連することを示しています マッチャーAPIはProspectiveSearch APIに名前が変更されました。

-rdbmsパッケージはJava SDKに登場しました。つまり、SQLを指します。

これらの機能はすべて、後でお待ちしています。
