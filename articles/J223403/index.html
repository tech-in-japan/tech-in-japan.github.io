イマジンカップ2014：勝利への道





読者の皆様、この記事では、プロジェクト[自己調整]についてお話しすることにしました。 過去1年半にわたって、私たちはアイデアからスタートアップへと長い道のりを歩んできました。 まだ完全にはスタートしていませんが、すでに何らかの結果が得られています 。ロシアのImagine Cup 2014ファイナルでのイノベーション部門での勝利です。 この道は非常に曲がりくねっており、非常にでこぼこの道を通りました。 私たちのストーリーを楽しんでいただき、ITスタートアップの分野で達成するためのインスピレーションを得られることを願っています。



それがすべて始まった方法



昨年の秋の夕方、5人のMIPTの生徒が将来の監督者のオフィスでお茶を飲みました。 彼は飛行のためのパイロットの心理的な準備を支援するために設計された彼の新しいプロジェクトについて熱心に話しました。 彼の話のハイライトは、彼の心臓のリズムのシーケンスを分析することにより、人の感情的なストレスのレベルを決定できるテクノロジーでした。 特に印象的だったのは、これがかさばる高価な機器を必要としないという事実、または特別な測定手順の通過でした。 必要なのは、胸に装着されたミニチュアフィットネスセンサーと5分間の測定だけでした。



その前でさえ、私たちは近代的な技術に興味があり、トレーニングや仕事の目的で多くのプログラムを作成し、独自のプロジェクトを作成した経験さえありました。 そしてもちろん、私たちは新しいアイデアに触発され、自分の手でそれを実現する機会を拒否することはできませんでした。 私たちは、学士号の研究としてこのプロジェクトに参加するという提案を喜んで受け入れました。

それから、私たちのプロジェクトが将来どのように成長するかを疑いさえしませんでした。 しかし、私たちはすぐに大きな熱意を持ってこの問題に取り組みました。 市場で入手可能なすべての携帯型心拍センサーと、電話またはコンピューターとデータを交換する技術を研究しました。 Bluetooth 4.0で動作する10個のセンサーの最初のバッチ（写真のように）をテスト用に購入しました。 WindowsおよびOS X用のアプリケーションの開発を開始しました。作業は沸騰し始めました。







そして今、短時間、大量のコード行、大量のビールがプロセス中に消費された後、私たちはチームの歴史の中で重要な質問を考えました：なぜ、安い、アクセスしやすい、便利なという利点があるので、私たちの技術の使用をそのような狭いグループに制限しますか？パイロットはどうですか？ 実際、私たちの手には、誰でも自宅でのストレスのレベルを迅速かつ正確に測定できる技術がありました。 ユーザーの観点からできる限りプロセスを簡素化するために、私たちはテクノロジーをスマートフォンに移すことにしました。



勢いを増す



この瞬間から、私たちは二重の力と熱意を持って仕事を始めました。 彼らは、夜間を含め、自由時間があるとすぐに寮または研究所に集まりました。そして、コード化、コード化、コード化されました。 それらは描画され、プログラムされた後、再描画、やり直し、再描画され、iOS用のアプリケーション画面が再びやり直されました。 その後、パルスシーケンスに従ってストレスレベルの複雑で扱いにくい計算を実装するサーバーが作成されました。 その後、サーバーを拒否することが決定され、まったく同じ機能がObjective-Cで記述されました。 その後、私たちは興奮していることに気付き、サーバーとモバイル開発者の努力を組み合わせることにしました。 私たちは毎日、新しい機能の追加と既存の機能の改善について互いに競い合いました。 アプリケーション画面を絶えず再描画し、デザインを変更しました。



私たちは何が起こっているのかについて非常に熱心だったため、ITプロジェクトを開始する際の典型的な間違いの1つを無視しました。 ユーザー全体を考えずに、開発とテクノロジーの観点からのみプロセス全体にアプローチしました。 その結果、科学用語で動作し、感情的ストレスを決定するための方法論の30種類の指標すべてをカウントする、複雑で扱いにくいモンスターを作成しました。 私たちが作成した作品を見せてくれた人は皆、彼のアイデアを非常に気に入っていましたが、同時に、誰もがアプリケーションの仕組みと一般的な動作について長い間説明する必要がありました。 ある時点で、同じやり方で仕事を続けることは不可能であるという理解が生まれました。



同時に、チーム内の対立が増大していました。 当社の技術は、他の可能な用途と開発の方向性を発見しました。 私たちのチームの一部は一度にすべてを開発したかったのですが、他の人には向いていませんでした-彼らは始めた仕事を完了し、すべての新しいアイデアに吹き付けられたくありませんでした。 その結果、プロジェクトの古い名前を他の人と一緒に連れて行った2人のチームメンバーの出発を生き延びなければなりませんでした。 その後、監督者との関係で非常にストレスの多い期間が続きました。 不要になったモンスターアプリケーションの背景に対して、この状況は非常に憂鬱に見えました。



復活



実際、私たちは何も残されていないという事実に我慢しませんでした。 古い間違いを繰り返すことなく、ゼロから始めることにしました。 文字通り同じ日に、長いブレインストーミングの後、私たちは新しい名前を思いついた- [self tune] 、そしてひざの上にウェブサイトを作った。 その後、デザイナーと、同時に創設者の1人の妻がチームに加わりました。 スタートアップの世界の最新のトレンドとSteve BlancとEric Rhysの本を武器に、プロジェクトに対するアプローチを根本的に変えることにしました。 私たちの技術がユーザーにとってどのように役立つかを理解し、ユーザーの緊急の問題を解決しようとしました。 私たちが常に感情的なストレスとストレスを経験しているという事実自体が大きな問題であることが判明しました。



実際、私たちは常にストレスを受けています。 忙しい一日を過ごした後、試験に合格する間、キロメートルの交通渋滞に立ちながら、自分自身を思い出してください-これらおよび他の多くのイベントはすべてストレスの原因です。 それらのすべてを回避することは不可能ですが、それらの否定的な結果をゼロに減らすことができます。 私たちは、感情的なストレスだけを測定するだけでは十分ではないことを認識し、日々のストレスに対処する効果的な方法をユーザーに提供する必要があります。



これが、現在の形のアプリケーション[自己調整]のアイデアが生まれた方法です。 ユーザーは、結果を測定、緩和、比較する3つのアクションの単純なアルゴリズムを実行できます。 測定プロセスをできるだけ簡素化し、ユーザーを制限しないために、センサーの使用を中止することにしました。 代わりに、スマートフォンのカメラに取り付けられた指を使用して心拍数を測定するための最新のテクノロジーをアプリケーションに実装することが決定されました。 測定プロセス自体は約1分かかります。











これはどのように機能しますか？



アプリケーションは、「測定、リラックス、比較」の原則に基づいて動作します。



最初のステップでは、初期応力レベルが測定されます。 これは、心拍変動の方法のおかげで可能です。 このテクニックの本質は非常に単純です。心拍数は時間とともに変化し、隣接する2つの心拍の周波数も異なります（そう言う場合もあります）。 たとえば、60のパルス値は、心臓が1分間に正確に60回拍動し、各拍動の時間間隔が1秒であることをまったく意味しません。 実際、心拍間の時間間隔は絶えず変化し、これらの変化のパターンに従って、ストレスを含む神経系のさまざまな状態を判断できます。







そして、これらの同じ時間間隔を実際にどのように取得しますか？ このために、電話カメラが使用されます。 人の心臓が機能しているとき、血液は指を含む体全体に脈動します。 これは、明るい光（たとえば、携帯電話のフラッシュ）に照らされた指の画像がわずかに変化するという事実につながります。特に、この画像の赤い成分（「R」）が変化します。







さらに、赤いコンポーネントのシーケンスに従って、グラフを作成し、グラフのピークを基準としてこれらの同じ距離を計算できます。 測定の最後に、ユーザーは0〜100の「ストレスの健康」スコアを取得します。100が最良の結果です。これは、ユーザーが健康であることを意味します（少なくともストレスに関して）。 測定の最後に、ユーザーは自分の主観的な感情状態と現在の活動の種類をマークする機会があります。







2番目のステップである「リラックス」では、ユーザーに実際の（私たちの評価）主観的条件、現在の活動、場所、記録の履歴などのパラメーターに基づいて選択されたリラクゼーションのテクニックを提供します。 これが私たちの革新です。 ユーザーに手法を選択するように依頼することはありませんが、現時点では最適な手法をユーザーに提供します。 簡単な呼吸法、リラックスした音楽、ストレスを避けるための一般的なヒントなどがあります。たとえば、散歩に出かけたり、本を読んだり、友達とビールを飲んだりします。 私たちが可能な方法のリストは、形成され始めたばかりであり、常に更新されています。







リラクゼーション後、ストレスを再度測定し、実際に2つの数値を比較することをお勧めします：BEFOREとAFTER。 したがって、ユーザーは進捗を監視し、メソッドが実際の結果に効果的であることを確認する機会があります。 このすべての後、ユーザーはFacebookで友人にアプリケーションの使用方法を伝えることができます。

舞台裏



チームのすべてのメンバーには、例外なく、「プログラマー」という主要な職業があります。 開発中に、Windows 7、Windows 8 / Store、OS X、iOS、Windows Phone、Azure、さらにはAndroidのプラットフォームのかなり印象的なリスト用のプログラムを作成する方法を学ぶ必要がありました。 それらのそれぞれには、その魅力と欠点があります。 たとえば、Visual Studioを使用してWindows用のC＃で開発することに慣れている人にとって、XCodeとObjective-Cは最初は非常にワイルドに思えます。 しかし、次の例のように、Linqを使用したエレガントなC＃コードのように見えるという事実に徐々に慣れてきます。 



peaks.Sort((p1,p2) => p2.signalPower.CompareTo(p1.signalPower));


手首のフリックでこれが変わります： 



 NSArray *sortedPeaksByPower = [peaks sortedArrayUsingComparator:^NSComparisonResult(id obj1, id obj2){ HRPoint *p1 = (HRPoint*)obj1; HRPoint *p2 = (HRPoint*)obj2; if (p1.signalPower > p2.signalPower) { return (NSComparisonResult)NSOrderedAscending; } if (p1.signalPower < p2.signalPower) { return (NSComparisonResult)NSOrderedDescending; } return (NSComparisonResult)NSOrderedSame; }];




そして、これは氷山の一角にすぎません。 実際、クロスプラットフォームはITスタートアップにとって最大の課題です。Android、iOS、またはWindows Phoneでニッチを占有しない場合、他の誰かがそれを行うからです。
