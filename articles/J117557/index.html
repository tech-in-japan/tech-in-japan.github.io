Yandex.Money API

こんにちは、Habr。



今日、Yandex.Moneyは新しいサービスを開始します。これは、habrayuzerにとって特に興味深いと思われます。

今日、支払いシステムAPIを開きます。



Yandex.Money APIを使用すると、アプリケーションで支払い機能を使用できます。 さらに、消極的なもの-アカウントの残高または支払い履歴を表示するだけでなく、アクティブなもの-店舗への支払いも行います。 実際、Yandex.Moneyにインターフェイスを書くことができます（突然何かがなくなった場合）。



APIの機能は望んでいるほど優れていませんが、これはほんの始まりに過ぎません。 habrasocietyが製品の開発に積極的に参加することを願っています。 皆様のご要望、機能のリクエスト、批判を喜んでお聞きします。APIの積極的な開発に努めます。



最初の実装例を含む新しいAPIベースのテクノロジーのプレゼンテーションは、4月22日にYandex.Moneyテント（メインビルディングの右側）で開催されるカンファレンス「 RIF + KIB 2011 」で表示されます。



一般的な情報



APIへのリクエストは、SSL（HTTPS）を使用するHTTP 1.1プロトコルを使用して実行され、リクエスト認証はOAuth2-Bearer標準に従って行われます。

サービスの応答は、UTF-8でエンコードされたJSONドキュメントです。 ドキュメントの内容は、クエリの結果によって異なります。

申請承認



アプリケーションを認証するために、OAuth 2.0プロトコルが仕様とともに使用されます。

  • OAuth 2.0プロトコル：ベアラートークン

APIメソッド



現在、次のメソッドがAPIに実装されています。

  • account-info-ユーザーアカウントの現在の状態に関する情報の受信。
  • operation-history-この方法では、ページモードで操作の履歴（全体または一部）を表示できます。 履歴レコードは新しい順に発行されます。
  • 操作の詳細 -履歴から操作の詳細情報を取得します。
  • リクエスト支払い - 支払いのリクエスト;
  • プロセス支払い - 支払い確認。
