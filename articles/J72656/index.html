Operaウィジェットは独立しています

ノルウェー人はウィジェットを無料にします。 ブラウザに関係なく、ウィジェットはスタンドアロンアプリケーションとして動作するようになりました。Operaを閉じることができ、ウィジェットは動作し続けます。 私個人にとって、これが彼らの使用を妨げた唯一のことでした。



しかし、真実はこれは単なるテストビルドであり、ノルウェー王国ではすべてがうまくいくわけではありません:)



このソリューションの利点は、ブラウザーからの分離に加えて、オペレーティングシステムとの緊密な統合です。 たとえば、Linuxユーザーは、最終的にウィジェットの（半）透明度を確認します。



これらの人生の魅力を活用するには、このテストビルドをインストールする必要があります。
  • Windows MSI
  • Windowsクラシック
  • Macintosh（Intelのみ）
  • Unix / Linux
その中に、メニュー項目「ウィジェット」がないことがわかります。これは、私の観点からすると大きなマイナスです。 ウィジェットを完全に管理することはまだ不可能であり、ブラウザにそのようなメニューが存在することは大いに役立ちます。 さて、大丈夫、忘れないでください-これは単なるテストアセンブリです。



その後、オペラウィジェットのカタログを開きます 。 [ 起動 ]ボタンをクリックして、興味のあるウィジェットを配置できます。 その後、インストールマネージャーが起動し、高度なプロパティで（Linuxの場合）このウィジェット用のインストール.debパッケージを作成する機能を見つけることができます。これは本当にクールな機能のようです。



しかし、ここでも問題があります。ウィジェットは起動しませんが、[起動]ボタンをクリックしてもインストールはされます。 ウィジェットを起動するには、アプリケーションのシステムメニューでウィジェットを探します。 そこに、新しいウィジェットがメニューに表示されました-ウィジェットマネージャ。 これを使用すると、ウィジェットを削除できますが、奇妙なのは、ウィジェットを起動できないことです。 これは、リリースの湿気に書いています。



まず、半透明の魅力をすべて享受するために、Yandex天気ウィジェットを設置するために登りましたが、 ウィジェットはインストールを拒否しました。



最近、多くのウィジェットエンジンがありますが、唯一の問題は、そのような別のエンジンが需要があるかどうかです。
