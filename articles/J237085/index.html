Windows 8キオスクの例を使用して大企業にイノベーションを導入した方法

こんにちは、ハブラ。 私はオーストラリアのオフィスの1つでCaterpillarで働いています。 勤務中、いくつかの興味深いプロジェクトが蓄積されました。 一般に、オーストラリアは、ご存知のように、あらゆるものからはほど遠いものであり、インターネットは大陸間の境界線を消去しましたが、新しいものは6ヶ月遅れで届きます。 これは、ダウンロード用に以前に表示されたソフトウェア（ニュージーランドのすぐ後のタイムゾーンにある）には当てはまりませんが、映画はロシアよりも遅くリリースされますが、ロシアでも翻訳が必要です。 そのため、テクノロジーのおかげで、ここに住む人々は新しいものに準備ができていません。 そうであっても-彼らはそれを必要としません。



一般的に、私は会社（または正しく会社のグループと呼ぶ）が東南アジアとオセアニアの国々に散らばる約90の支店と10万人の従業員で構成されているという事実から始めます。 グループ全体は、開発からメインのERPシステムに至るまで非常にマイクロソフト志向であるため、今日は、使用したマイクロソフトの製品についてのみ説明し、Cisco Unified Communication Managerについても少し説明します。



挑戦する



すべての支店での受付用に情報キオスクを開発する必要があります（約90）。 複数の言語をサポートする必要があり、プラットフォーム自体が可能なすべてに統合される必要があります。 上で書いたように、同社はMicrosoft指向であり、Office 365（Exchange、SharePoint、Lync）を含むすべての製品を使用しています。 最も興味深いのは、会社が通信プラットフォームとしてCiscoを使用していることが歴史的に何らかの形で発生したことですが、同時に、集会の規則に従ってLyncで発生したことです。 つまり、Lyncを使用して固定電話のシスコの電話をかけることはできません。 キオスクは、Cisco Unified Communication Platformに統合する必要があります（ユーザーがキオスクから直接呼び出すことができるように）。



また、承認にRFIDカードを使用できるようにする必要があります（会社の各従業員はRFIDカードを持っています）。 なぜ情報キオスクに必要なのですか？ ロシアの企業でこれが実践されているかどうかはよくわからないとしますが、ここでは、規則に従って建物間を移動するたびに、建物の入り口で本を記入する必要があります。そうすれば、セキュリティサービスはどこで何が起こったのかを知ることができます。 さらに、オーストラリアでは契約作業が非常に一般的です。 同社は、特定の業務を遂行するために、非常に専門的な専門家を短期間雇用しています。 150の請負業者が何十もの格納庫に散らばっていると想像してください。 彼は間違った方向を向き、間違ったドアに入りました-そして、車輪で押しつぶされました。 したがって、キオスクの主なタスクは、人々を登録し、適切な会社の従業員を見つけるのを支援し、セキュリティサービスが明確なアイディアを持つようにすることです-どこで、誰が、いつですか。 要約すると。



猫は次のことができる必要があります。

  • 場所に応じて言語をサポート。
  • 従業員を検索する機能をサポートします。
  • 電話をかける。
  • 迅速な登録のためにRFIDカードを使用する機能をサポートします。

言及するのを忘れましたが、何か新しいことをする必要がありました。 同社はいわゆる近代化を経ており、マネージャーはDBSのグリーンスクリーンとは異なる革新的なものを見たいと思っています。

RFIDを使用する


私は未だにこの恐ろしい夢の中でこの獣を夢見ています。 ArduinoでRFIDフレームを使用したことがないのは誰ですか？ やっぱり？



アプリケーションシェルの準備ができたら、RFIDとの統合を開始しました。 プロセスは非常に単純です。会社の従業員がカードをリーダーに立てかけ、アプリケーションはあなたが会っている従業員の名前を尋ねます。 会議の終わりに、地図をスキャンすると、アプリケーションはあなたに良い一日を願っています。 しかし、問題は機器に始まりました。 最初は、それぞれ約600ドル相当のメガ保護されたリーダーを使用していることが判明しました。 RFIDロックを使用した人は、読者が問題の半分にすぎないことを知っています。 シリアルポートを介して、コントローラーに接続し、コントローラーをPCに接続します。 MetroアプリケーションはSystem.IOの何もサポートしていないことが判明しました。 私たちのプログラマーは、Metroアプリケーションが不運なシリアルポートにアクセスする方法をさまざまに考え始めました。 私たちはWindows 8で作業しており、Windows 8.1に登場したBroked Componentについてはまだ知りませんでした。



その後、8つでは、Device APIでさえ不足していました。 一般に、8つについては、ファイルシステムを介してMetroアプリケーションと通信するwin32クライアントを実行する以外に何も思いつきませんでした。 もちろん、これは良くありませんでしたが、8.1でBluetoothのサポートを追加するという噂が既にあり、マイクロソフトの人々は期待を裏切らなかった-新しいデバイスAPIは簡単で使いやすいものでした。 気に入らなかった唯一のことは、使用する前に最初にデバイスをペアリングする必要があることです。 さらに、デバイスapiは、Windowsからこのデバイスを使用できるかどうかを尋ねられるように設計されているため、不快感が増します。



2番目の問題は、市場にBluetooth RFIDデバイスがないことです。 当社のカードで動作するRFID Bluetoothリーダーをただ購入して購入することは不可能でした。 地元企業に頼らざるを得なかったのですが、幸いなことに、ある会社が助けてくれました。 その結果、1か月後、私の机の上にUSBを搭載した新しいリーダーがありましたが、Bluetooth経由でMetroアプリにデータを転送しました。 そして、すべては問題ありませんが、4か月のストール作業の後、Windowsが接続を単に切断することがあります。 これは、約4〜5日間の作業後に発生します。 8時間ごとに接続を再開することをお勧めしながら、MSDNで書いて、それらのサポートで尋ねました。 Update 1のBroked Windows Runtime Componentで新機能がどのように登場したかを修復することを既に決定しました。 一言で言えば、このことにより、Metroアプリケーションは通常のWindowsアプリケーションと通信できます。 これはまさに私たちのケースでした。 BrokedコンポーネントでRFIDを使用するロジック全体を省略しました。 これまでのところ問題はなかったと言えますが、それがさらにどのように振る舞うのかがわかります。



Cisco UCM


私たちが適用した解決策が最良ではないとすぐに言います。誰かが良い代替案を共有してくれたらとても感謝しています。 シスコは非常にクールな会社ですが、何らかの理由で、統合の実装はあまり実用的ではなく、Web開発の一般的な傾向に遅れを取っているように思えます。 これらのActiveXプラグインはすべて、何らかの形でシスコスタイルではありません。 ただし、Cisco Unified Communication Managerは非常に優れたプラットフォームです;唯一のことは、シスコがアプリケーションで簡単に使用できる適切なツールがなかったことです。 詳細には触れません。興味がある人は書きますが、最終的には古いOfficeライブラリであるOffice.Interop.UccApi.dllを選択し、その上にローカルWebサーバーを起動する小さなwpfクライアントを書き込み、キオスクは単にメソッドを呼び出します-電話をかけるか電話を切ります。 誰かがSIPプロトコルをMetroアプリケーションに統合しようとした場合は、一緒に作業することをお知らせください。



SharePoint


最初の写真でわかるように、また冒頭で述べたように、データベースと従業員に関するすべての情報のソースとしてSharePointを選択しました。 開発の最初に、Microsoft（ Contoso Learning Companion ）から非常に興味深い例に出会いました。







各場所（サブサイト）には、構成のあるシートのセットがあります。 たとえば、シスコのポータルから直接アカウント設定を管理する機能が必要です。 また、アプリケーションの2番目の画像を見ると、検索の下の左側にクイックアクセスゾーンがあります。 たとえば、支店に入るユーザーは、財務またはHRの責任者を知りません。 アイコンをクリックすると、アプリケーションはこのブランチの人事担当者のみを表示します。 各場所には、すべての訪問のリストも含まれており、すべての情報が集約されて、最高レベル（レセプション）で利用できます。



SharePointは、特に新しいSharePoint 2013での検索に関しては同様に優れています。すべての従業員検索は、jsで20行です。 すべてが迅速かつ効率的に機能します。 一般に、SharePointとの統合は、問題がまったくなかった唯一のタスクでした。 シェルは数日で書かれ、非常に安定して動作します。



展開
