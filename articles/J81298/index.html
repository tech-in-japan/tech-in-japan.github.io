ロシアはMySQLを節約できる

今日、 helpmysql.orgキャンペーンの支持者からかなり大きな手紙を受け取りました。 私の意見では、非常に好奇心が強い。 それをよく共有し、このキャンペーンへの参加をお願いすることにしました。



PS。 手紙には多くのリンクがありますが、一部は（私の意見では重要です）クリック可能のままにしますが、他のリンクはそうではありません-スパムで疑わないように=）





helpmysql.orgキャンペーンの親愛なる支持者、MySQLの運命を気にするすべての人。



欧州連合はMySQLをOracleから保護できない可能性がありますが、あなたの国は違いを生むことができる2つの強力な独立国のうちの1つです（2番目の国は中国です）。 個人的に助けを求めます。 http://www.helpmysql.org/en/petitionで請願書に署名することで、すでに支援しましたが、今ではもっとできることができます。



1）ロシア連邦独占禁止サービス（FAS）は、関係者にこの取引に関する情報を提供するよう要請しました。 これは、公式声明http://www.fas.gov.ru/merger/notifications/a_28550.shtmlに記載されています 。



あなたやあなたの友人が商用目的でMySQLを使用している場合、あなたの情報はFASにとって非常に有用です。



FASに正確に何を書き込むかについては詳しく説明しませんが、一般的には、MySQLの使用方法（プロジェクト、データ量、ユーザー数など）を説明し、次のように意見を述べることができます。 OracleがMySQLを購入する許可を得た場合に発生します。 また、MySQLが高価なOracle製品の無料の代替品と見なされた事例、OracleがMySQLを使用または使用しようとしたためにOracleが価格を下げた、または大幅な割引を提供した例も特に重要です。



あなたはあなたに手紙を送ることができます：

連邦独占禁止サービス

11モスクワ、D-242、GSP-5、123995

FASに連絡するときは、あなたの利益を保護するのに役立つ組織であると考え、この作業をできる限り簡単にするようにしてください。



2）あなたとあなたのサポートのおかげで、私たちの請願はほぼ価値があります

ロシアからの900の署名： http : //www.helpmysql.org/ru/stats（署名の総数の2.7％-約Absolvo ）



これは良いスタートですが、ロシアはオープンソースソフトウェアをサポートする大国であり、さらに多くの署名があります。 問題は、多くの人がこれを欧州連合の決定と見なし、ロシアの立場が何にも影響を及ぼさないことでした。 さて、FASの公式声明の後、ロシアがより積極的な姿勢を取る時が来ました。



ロシアではまだあまり有名ではありません。 彼らは主にこのインタビューから私たちについて学びます： http : //habrahabr.ru/blogs/habrainterview/80534/



www.compress.ru 、 www.computerra.ru 、 www.pcworld.ru 、 www.computerbild.ru 、 www.chip.eu / ruなど、ロシアの主要なコンピューターサイトや雑誌に当社に関する情報を提供することで支援できます。



または、 sql.ru / forum / actualforum.aspx 、 www.developers.org.ua 、 rsdn.ruなどの人気のあるフォーラムで状況について議論することもできます。



ロシアのWebサイトが最新情報を把握しているほど、MySQLを支援して請願書に署名する人が増えます。 これらの記事が少なくともhelpmysql.orgに言及していることを確認してください。 サイトのロシア語版への完全なリンクを提供している場合はさらに良い： http : //www.helpmysql.org/en



私はあなたの助けに心から感謝しています。
