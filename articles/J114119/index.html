メーリングリストのレイアウト：内観

こんにちは！



私の知る限り、メーリングのレイアウトに関する私のデビュートピックは、多数のハブロビテにとって有用でした。この中で、ルールと推奨事項のリストを提供するだけでなく、「何？ なんで？ そしてなぜ？」 任意のポイントを省略できますが、これは前のトピックでそれらが明確に説明されたためです。



まず、このトピックでよく使用される文字の1つ、つまり「さまざまなWebインターフェイスと電子メールクライアント」に注目します。 この単語の構成を毎回繰り返すのは少し面倒ですが、この一般的な単語を与える方が簡単だと思います。 さらに読み進めていくと、 「デーモン」という言葉（異なる名前を付けるのは難しい）を「さまざまなウェブインターフェースとメールクライアント」と考えてください。 さあ、始めましょう！



頭をスタイリッシュに！



望まない限り、Hederスタイルは依然として禁止されています。 悪魔は文字コードで型構造を熱心に検索します 

<style> //some css rules... </style>
正直に、エクソシストには行かないでください。 個々の要素ごとにすべてのインラインスタイルを登録する必要があります。 特に、 mail.ruデーモンがインラインスタイルをカットすると主張する邪悪な言語については、その証拠を引用します。



このプレートは本当に私を太らせませんか？



これは決してルールではありませんが、書かれていない標準としてランク付けすることをお勧めします。レターの主な構成であるメインテーブルでは、幅の値を600ピクセルと750ピクセルに設定します。 サブスクライバーには小さなハガキ用に小さな値が提供されますが、大きなレターには多くのテキストが含まれます（まあ、またはそこにクライアントに送信するものは何でも）。



ゼロ崩壊-救い！



崩壊のケースは他のケースよりも特別です。 テーブルを折りたたんでリセットします！ 理解できない人のために、タブレットは次のように発行する必要があります。 

 <table style="border-collapse:collapse;" cellpadding="0" cellspacing="0"> ... </table>
インシデントを回避するためにゼロ化が必要です。 悪魔はそのような悪魔であり、それぞれがテーブルだけでなくすべての要素に対してインデント値を設定するのが正しいと考えています。 さらに、インデント値をゼロより大きく設定すると、単に動作しないか、正しく動作しない可能性があります。

崩壊の特定のケースに関しては、この措置は、テーブルのすべてのセルの間に1ピクセルのインデントを追加するだけのユニークなMS Outlookに必要です。



まあ、まったく3.2ではない



HTML 3.2の中世の時代に戻ったような記事にどのように書かれていても、 フォントタグの使用を放棄する価値があることに注意する必要があります。 なんで？ 技術的要因と人的要因の2つの理由があります。 人間的なことは、多くの場合、マイルストーンはレイアウトを構成するだけであり、それ以上の運命は分からないかもしれないということです。 何らかの理由で、多くの企業は、ニュースレターを送信できるのはマネージャーだけであると判断しましたが、彼はどうすればよいか考えていませんでした。 これはまさに技術的要因が従う場所です。 多くのメーリングリストシステムには、レパートリーにビジュアルエディターのようなものがあります。 それは何のために作られていますか？ シンプルなマネージャーがレイアウトにテキストの変更を加えるため。 しかし、ビジュアルエディターの開発者（すべてではありませんが、一部は存在します）も神のしもべではありません。彼らは1つの間違いを犯しました。 同時に、属性とスタイルの変換はひどい結果で変化します！ したがって、インラインスタイルでのみスパンします。



#cccの色が欲しい



動作しません！ 悪魔はこの色形式を好まない！ 背景とテキストの両方の書式設定では、＃ccccccを指定する必要があります。 完全な6桁のカラーコーディングのみ。 さらに、デーモンはわずかに明白で、 red 、 blueなどの実際の色の値を取ります。



学校で悪魔にユニコードを教えた



何もすることはありません、彼らはそのような平凡です。 いいえ、もちろん、文字をUnicodeまたはキリル文字とラテン文字をサポートする他のエンコードで送信できます。 実際、一部のデーモンはすべてのUnicode文字を受け入れないということです。 あらゆる種類のウムラウト、フランス語、スペイン語、イタリア語の記号、著作権、商標は、本質的に審査されなければなりません。 ここで、最近引用したビジュアルエディターを例として紹介します。 それらの多くは、必要なものすべてを愚かにスクリーニングしているだけです。



驚くべきが真実



Google Mailによって尊敬され、崇拝されている人（そのインターフェイスを悪魔とは呼ばない）は、パーサーにも欠点があります。 画像をリンクとして使用する場合は、画像の境界線をリセットする必要があり（Capのおかげ）、同時にリンク自体をtext-decorationに設定する必要があります。 それ以外の場合、画像（ああ神々！）はリンクとして下線が引かれます。
