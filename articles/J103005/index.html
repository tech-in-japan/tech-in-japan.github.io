phpmyadminを3.3.5.1にアップグレードするときの2つの小さな問題

“ emerge -uD world ”を使用して昨日サーバーを現在の状態に更新したところ、phpmyadminバージョン3.3.5.1の2つの小さな問題を発見しました。 1つの問題はphpmyadmin自体に関連していますが、もう1つはgentooにインストールする機能に関連しています。







「データベースなし」



新しく更新されたphpmyadminを見ると、データベースがまったくないことがわかります。左のフレームには、孤独な「データベースなし」と書かれています。 数分後、ショックから離れて冷たい汗を拭き取ると、ベースが実際に配置され、機能していることがわかります。myadminには表示されません。 解決策はconfig.inc.phpに行を追加することです。これはおそらく長年変更されていません。



$cfg['Servers'][$i]['only_db'] = array('%');

$cfg['Servers'][$i]['hide_db'] = '';









これらは現在必要です。 config.sample.inc.phpでは、ところで、これらのパラメーターはそうではありません。 ソリューションソース： forums.gentoo.org/viewtopic-p-6404523.html



インターフェイスに一部のリンクがありません



特に、更新リンク、列見出しなどはありません。 ここで、gentoo固有の問題はwebapp-configで、 phpmyadmin / libraries / config.default.phpファイルを置き換える代わりに、 phpmyadmin / libraries /._ cfgXXXX_config.default.phpの形式で新しいファイルを作成します（XXXXは更新ごとに1つずつ増加します） ）、明らかにいくつかの設定が含まれていると信じています。 アクションの車の暴動。



問題は、最新の更新によって行われた変更が健全性にとって重要であり、 dispatch-confが/ var / www / localhost / htdocs / phpmyadminの構成ファイルの更新を検索しないことです 。



簡単な解決策：



cd /var/www/localhost/htdocs/phpmyadmin/libraries

cp ._cfg< >_config.default.php config.default.php
