ロンドンオリンピックがクラウドを通じてどのように放送されるか





今年の初めに、 Windows Azure Media Servicesを発表しました 。 Windows Azure Media ServicesはクラウドベースのPaaSソリューションであり、メディアサービスを効率的に構築し、メディアコンテンツを消費者に配信できます。 このソリューションは、ライブ形式とオンデマンドの両方でメディア素材、コーディング、フォーマット変換、ストレージ、コンテンツ保護、ビデオ配信を迅速に受信できるすぐに使用できるサービスのセットを提供します。 Windows Azure Media Servicesは、HTML5、Silverlight、Flash、Windows 8、iPad、iPhone、Android、Xbox、Windows Phoneなど、あらゆるデバイスまたはプラットフォームへのコンテンツの配信もサポートしています。



Windows Azureメディアサービスとロンドン2012オリンピック



過去数週間にわたって、Windows Azure Media Servicesは、オリンピックを放送する複数の放送局にリアルタイムおよびオンデマンドのビデオストリームを配信するために使用されてきました。 その中には、フランステレビ、RTVE（スペイン）、CTV（カナダ）、Terra（中南米）があります。 deltatre、Southworks、gskinner、Akamaiとのコラボレーションにより、オリンピックから2300時間以上のライブビデオとHDコンテンツを20か国以上に配信できるようになりました。



以下では、これらの企業がWindows Azure Media Servicesを使用して優れたビデオ配信とブロードキャストを提供した方法について詳しく説明します。



チャネルを使用したメディア配信ワークフローの自動化



Windows Azure Media Servicesは、複数のメディアサービス機能をライブビデオおよびオンデマンドブロードキャストの単一の論理ワークフローにバンドルするために使用できるチャネルの概念をサポートしています。 チャネルは、REST APIを介してプログラムおよび管理できます。これにより、ビデオをブロードキャストおよび公開する企業は、既存の自動化プラットフォームをWindows Azure Media Servicesと簡単に統合できます。 ロンドンオリンピックでは、企業はチャンネルモデルを使用して、FastForwardビデオ管理システムとdeltatreのForgeコンテンツ管理ツールを使用して、ライブビデオとオンデマンドブロードキャストのワークフローを調整しました。



オリンピックのライブビデオストリームをダウンロードする



オリピアダのライブビデオストリームは、消費者企業にビデオストリームを配信するために国際オリンピック委員会（IOC）によって作成されたメディア放送組織であるオリンピック放送サービス （OBS）によって公開されました。 ロンドンのOBSは、すべてのコンテンツを1080i HDビデオ形式で提供しました。 ビデオストリームは、ビットレート17.7 MbpsのH.264コーデックで圧縮され、MPEG-2トランスポートストリームに埋め込まれ、deltatreなどの企業によってUDPを介してマルチキャストで配信されました。 その後、Deltatreは各1080iストリームをスムーズストリーミング用の8つの異なるビットレートにトランスコードしました。150kbpsのビットレート（解像度336x192）から、最大3.45 Mbps（1280x720）のビットレートまで、Windows Azure Media Servicesで公開されたストリームです。



障害から高い信頼性を確保するために、ビデオストリームは世界中のさまざまな場所にある複数のWindows Azureデータセンターで同時に公開されました。 その後、ストリームはWindows Azure Media Servicesで定義されたチャネルによって消費され、既にライブビデオをWebに投稿したビデオホスティングインスタンスにルーティングされました。 コンテンツの配信を確実にするために、Akamai HD CDNネットワークが使用されました。







すべてのクライアントとデバイスのストリーミング



2012年のロンドン大会では、スムーズストリーミングベースのビデオストリーミングモデルを使用しました。 ブラウザの場合、SilverlightクライアントとFlashクライアントの両方にSmooth Streamingストリームを配信しました。 デバイスについては、iOS、Android、およびWindows Phone 7向けにSmooth Streamingを提供しました。H.264標準およびAACコーデックのユニバーサルサポートの利点を使用して、コンテンツを一度にエンコードし、単一ストリームですべてのデバイスに配信することができました。 Deltatreは、iPhoneおよびiPad用のWindows Azure Media Servicesが配布するiOS Smooth Streaming SDKツールキットと、 Nexstreamingが開発したAndroid用のSmooth Streaming SDK ツールキットを使用しました。



これらのオリンピックでのメディアサービスの重要な革新は、スムーズストリーミング再生をネイティブにサポートするFlashベースのSDKツールの開発でした。 gskinner.comのFlashエキスパートと協力して、Windows Azure Media Servicesチームは、Smooth StreamingをFlashに配信するためのネイティブActionScript SDKを開発しました。 これにより、放送局は、すべてのSilverlight、Flash、iOS、Windows Phone、およびAndroidプラットフォームに対して単一のSmooth Steamingプラットフォームを利用できました。

Windows Azure Media Servicesの利点



オリンピック中、放送局は17日間、1日15時間、30のライブビデオストリームの平均放送容量を必要としていました。 ライブストリームに加えて、コンテンツはオンデマンドで作成され、1日24時間、20か国以上に配信されました。これは、一般に何百万時間ものビデオの消費を意味します。



Window Azure Media Servicesがなければ、放送局は次のことを行う必要があります。



  1. コンピューティング、データストレージ、ネットワーク通信用のインフラストラクチャを購入/レンタルする
  2. 独自のビデオ処理アーキテクチャを拡張する
  3. 単一のワークフローで新しいインフラストラクチャと既存の組織をリンクする
  4. 高い耐障害性を実現するために、いくつかの異なるデータセンターに自分でアーキテクチャを展開して実行
  5. エネルギー、空調、管理、スタッフに24時間年中無休で対応


これが、Windows Azure Media Servicesプラットフォームのフルパワーと柔軟性のあるところです。 放送局はクラウドテクノロジーを利用する機会を与えられ、わずか数行のコードでライブおよびオンデマンドブロードキャストをインストールおよび構成できました。 わずか数分で、必要なすべてのサービス、ネットワークルーティング、およびストレージリソースがWindows Azure内に展開および構成され、大勢のユーザーとコンテンツを送受信できるようになりました。



ネットワーク、ハードウェア、さらには人的エラーの問題が放送を危険にさらす可能性がある場合、オリンピックなどの規模のイベントのコンテンツのオンライン配信は困難なタスクです。 Windows Azure Media Servicesプラットフォームは、通常、独自のインフラストラクチャ上で、または単純なクラウドサービスに基づいた従来の実装では達成できない、または高すぎるフォールトトレランスと冗長性の自動化レイヤーを提供します。 特に、Windows Azure Media Servicesは、消費、生産、ストレージ、分散データキャッシングのための冗長サービスを提供し、さまざまなデータセンターで使用できるため、障害から自動的に回復できるソリューションの高可用性を実現します。



自動ストリーミング動画



Windows Azure Media Services（およびWindows Azureプラットフォーム全体）の重要な原則は、すべてのアクションを完全に自動化することです。 以下は、数百万人のユーザーへのHDビデオのライブブロードキャストに使用できるWindows Azure Media Servicesで新しいチャネルを作成する方法を示す擬似コードです。 



// connect to the service: var WAMSLiveService = new WAMSLiveService(serviceUri); // Then you give us some details on the channel you want to create, like its name etc. var channelSpec = new ChannelSpecification() { Name = “Swimming Channel”; Eventname = “100 Meter Final”; } // Save it. WAMSLiveService.AddtoChannelSpecifiations(channelSpec); WAMSLiveService.SaveChanges(); // Create all the necessary Azure infrastructure to have a fully functioning, high performance, HD adaptive bitrate Live Channel WAMSLive.Service.Execute<string>(new Uri(“AllocateChannel?channelID =55”));


ロンドンオリンピック中、放送局はこのようなコードを記述して、手動でのインストールや構成を行うことなく、多数のライブHDストリームの作成を迅速に自動化することができました。



リアルタイムデータを操作する自動システム、特にオリンピックを提供するシステムで作業する場合、ビデオストリームの開始と停止が非常に頻繁に行われるため、活発に監視できることが重要です。 これらの目的のために、Southworksは、Windows Azure Media ServicesのREST APIを使用してチャネルを操作し、ビデオエンコード、チャネル操作、およびデータフローの重要な兆候をAkamaiノードに報告するコントロールパネルWebサービスを構築しました。 放送局はこのサービスを使用して、重要なパラメータと意思決定の質を監視する機会がありました。



ライブビデオとビデオオンデマンド



Windows Azure Media Servicesの主な利点は、ライブストリーミング中とライブストリーミング後（完全なDVR機能を使用）の両方でビデオをすぐに再生できることです。 たとえば、ユーザーが少し前に既に開始したライブブロードキャストに接続した場合、ビデオをすぐにイベントの先頭に巻き戻し、イベントの先頭から視聴を開始したり、プレーヤーのマーカーでマークされたイベントの注目すべきエピソードの1つに移動したりできます。







さらに、Windows Azure Media Servicesを使用すると、ライブストリーミングからオンデマンドストリーミングに瞬時に移行できます。 イベントがリアルタイムで終了すると、サーバー上に完全なオンデマンドビデオが作成されます。 トランジション間に遅延はありません。これにより、ライブブロードキャストを見逃したユーザーはすぐにビデオをオンデマンドで視聴できます。



ライブビデオの編集と貼り付け



放送局には、数千時間に及ぶライブビデオストリームから主要なビデオモーメントを簡単にリアルタイムでカットおよび作成できるソリューションが必要です。 Windows Azure Media Servicesは、ライブまたはオンデマンドのビデオコンテンツを操作できるブラウザーベースのビデオエディターであるMicrosoft Media Platform Video Editorと緊密に統合されています。 この統合により、編集者は、ビデオマテリアルをローカルにダウンロードまたは保存する必要なく、イベントの数分間、重要なビデオモーメントの挿入をすばやく作成でき、CDNノードにある既製のマテリアルからこれらの挿入をブロードキャストできます。 放送局のエディターに追加された新機能：USBホイールのサポート、サウンドの動的変更、パン、トラックのロック、複数のオーディオトラックのサポート。







重要なポイントの作成に加えて、ビデオエディターはWindows Azure Media Servicesと統合され、モバイルクライアントやYouTubeなどのサードパーティのビデオ配信サイトに発行するための挿入をエンコードしました。 たとえば、CTVはYouTube（カナダ）を使用して視聴者へのアクセスを拡大しました。この場合、重要な瞬間は4Mbps VBR H.264形式でエンコードされ、Windows Azure Media Servicesから直接YouTubeにダウンロードおよび公開されました。



オリンピックの毎日、Windows Azure Media Servicesワークフローで数千のコーディングタスクが開始されました。 このシステムにより、お客様は、多数のコンピューティングプロセスの機器サポートの問題に関係なく、コンテンツ発行ワークフローを作成および変更できました。 Windows Azure Media Servicesプラットフォームは、負荷のピークに自動的に対応し、消費電力を処理し、大量のビデオエンコーディングの必要性を優先するスケーラブルなメカニズムを提供しました。



おわりに



Windows Azure Media Servicesプラットフォームにより、放送事業者は非常に優れた、より信頼性の高いオンラインブロードキャストエクスペリエンスを活用できます。 Windows Azure Media Servicesは、消費、ストレージ、コーディング、制作、監視、配信などの柔軟な機能セットを提供します。これらの機能を使用して、ライブビデオブロードキャストおよびオンデマンドコンテンツ配信用の拡張性の高いワークフローを作成できます。 これらのクラウドサービスは、ビデオブロードキャストの構築における複雑さの問題を解決すると同時に、より多くのデバイスに対してより高いレベルの信頼性と可用性を提供します。



現在、Windows Azure Media Servicesはプレビューとして利用できます。これらの使用方法の詳細については、 このリンクを参照してください 。 オンデマンドでコンテンツにアクセスするためのワークフロー機能は、プレビューにサインアップした人なら誰でも今日利用できます。 オリンピックで使用されたライブブロードキャストを整理するためのワークフロー機能が、プライベートプレビューとして利用できるようになりました。 このプレビューのテストに関心のあるクライアントは、mediaservices @ microsoft.com宛にメールを送信して参加できます。 そのうち、これらの機能は一般公開されます。



オリンピックに関連する仕事は、困難な問題、責任、そして勝利への願いです。 Windows AzureとWindows Azure Media Servicesの製品チーム、Developer＆Platform Evangelismグループ、外部開発パートナー、そして最も重要なのはオリンピックの成功に貢献した放送会社のすべての人々に感謝します。 このイベントは、将来適用される経験の大きな成功物語です。
