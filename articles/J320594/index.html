Win10の面白いニュアンスとその対処方法

Win10の面白いニュアンスをいくつか偶然発見しました。これはユーザーをひどくイライラさせる可能性があります。 しかし、警告される人は誰でも武装しています。



1.セーフモードとWin10

2.更新のインストール時にサイクリックリブートします。



1.セーフモード



結局のところ、win10では、Windowsの起動時にセーフモード（セーフモード）に入る機能がデフォルトで削除されました。 ここで、この機会を得るために、WORKING WINDOWで、つまり、事前に、管理者に代わってコマンドを実行する必要があります 



bcdedit /set {default} bootmenupolicy legacy


そして、そのような操作の後、古き良きF8がシステムのスタートメニューに入ることを可能にします。



管理者として起動するようなバッチファイルにすることをお勧めします（実際にはdisplaybootmenuコマンドは必要ありませんが、設定することをお勧めします）。 



 @echo off bcdedit /set {bootmgr} displaybootmenu yes bcdedit /set {default} bootmenupolicy legacy timeout 20 exit


デフォルトで通常モードに戻すには、 レガシーを標準に置き換えます 。



さらに、一部のグローバル更新はこの設定を標準にリセットするため、システムの変更後、またはログオフイベントのスケジューラーを使用して、再起動する前にパラメーターを明示的に設定することをお勧めします（おそらくこれは多すぎるでしょう）。



それにもかかわらず、災害が発生してセーフモードに入る必要がある場合、win10はロードされなくなり、F8コールはオンになりませんでした。つまり、2つのオプションです。



a）場合によっては、リセットによってダウンロードが中断されるように強制するのに3〜4回役立ちます。その後、win10はセーフモードへの移行を提案します。 この方法は役に立ちませんでしたが、同僚は近くの職場で成功しました。



b）ブートイメージ（ディスク、フラッシュドライブ、「蘇生器」）から起動します。 ブート時に、shift-F10を押してコマンドラインを入力します。 特定のケースでブートディスクがマウントされている文字を確認します。 そうでない場合（ほとんどの場合）、手動でマウントします 



 diskpart select disk 0 select partition 1 assign letter=s exit


（物理ディスク0を選択します。つまり、BIOSの最初のもので、パーティション1はシステム予約として隠されています。ハードディスクの構成が少し異なる場合は、コマンドを修正してください）。



セクションのリストを表示します。 



 diskpart list volume exit


それから 



 bcdedit /store s:\boot\bcd /set {bootmgr} displaybootmenu yes bcdedit /store s:\boot\bcd /set {default} bootmenupolicy legacy exit


その後、メニューが表示されます。



2.更新のインストール時にサイクリックリブートします。



ここでは、高貴なトローリングMSが登場しました。 Win10では、ロシア、ウクライナ、カザフスタンなどの標準的な地域設定に加えて、いくつかの追加の「ロシア語（ウクライナ）」、「ロシア語（カザフスタン）」などが登場しました。 どの地域を選択すると、ウクライナ、カザフスタンなどが表示されますが、Windowsシステムの通知はロシア語であり、各国語ではありません。



そのため、これらの非標準の地域設定の1つを選択すると、高い確率でWin10更新がフィニッシュラインに到達せず、無限の自動再起動が行われます。 ロシア、ウクライナ、カザフスタンなどの標準的な地域では、そのような効果は見たことがありません。 実際、問題の解決策は、設定を標準の地域に設定することです。 セーフモードに入ることができる場合は、そこに設定します。 解決策はブートイメージを使用しているため、失敗しました。 上記と同様に、コマンドラインにアクセスします。 



 diskpart select disk 0 select partition 2 assign letter=w exit


（物理ディスク0、つまりBIOSの最初のものを選択します。パーティション2は、Win10（！）が実際にインストールされているもので、非表示ではありません。） 



 Dism /image:W: /Set-AllIntl:ru-RU exit


再起動後、アップデートが正しくインストールされ、すべてが機能します。地域の設定を再度設定するだけです。 それでも必要だと思う場合）
