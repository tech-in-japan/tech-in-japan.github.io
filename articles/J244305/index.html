どの場合にA / Bテストが必須か





A / Bテストは、コンバージョンを増やす最も効果的な方法の1つです。 ただし、Econsultancyの調査によると、A / Bテストを使用している企業は全体の38％のみです。 A / Bテストが単に必要な場合（望ましくありませんが、必要な場合）について話しましょう 。 そのため、この記事の著者であるジェレミー・スミスにこの言葉が与えられます。ジェレミー・スミスは、eコマースに関係する人々にとって価値のある情報をその兆候の中で共有します。

ウェブサイトのデザインを変更する



ウェブサイトのデザインが常に変化しているという事実は、オンラインのニュースではありません。 問題は、設計の変更がトラフィックの損失とコンバージョン率の低下であることです。 デザイナーにとってこれがもう1つの新しいインターフェイスである場合、SEOおよびCROにとっては頭痛の種です。リンクが失われ、オーガニック検索結果の位置が低下し、404エラーなどが無限に続きます。



Diggの設計変更は次のとおりです。







なぜ設計変更が必要なのですか？



実際、設計変更を行う理由はそれほど多くありません。 これは主に次の理由で発生します。



  • もう2年が経ちました。何かを変える時です。
  • このサイトはなんとなく古風に見えます。
  • 予算に「設計変更を行う」という記事があります。
  • デザイナーには何の関係もありません。
  • 誰かが何かを変えるといいと言った。




など-無限に。 このために、自分の頭で冒険を探すべきではありません。



本当に何かを変える時はいつ来ますか？



  1. トラフィックを最適化する必要があります。
  2. コンバージョンを増やす必要があります。
  3. サービスの品質を改善する必要があります。








これらの3つの理由が何であれ、設計の変更は、それが変換を増やす目的に役立つ場合にのみ正当化されるという事実につながります。



設計変更とはどういう意味ですか？



まず第一に、これはカラースキームで始まり、CMSで終わるパレット全体です。 A / Bテストはいつ必要ですか？



  • サイトのアーキテクチャを完全に変更したとき;
  • ブランド変更を行うとき;
  • かなりの数のページを追加または削除するとき。
  • スタートページで何かを変更する場合
  • 変換ツールの1つを変更したとき。




これについて、いくつかの「経験豊富なヒント」を紹介します。



変更が本当に必要な場合は、続行する前にA / Bテストを実行してください。 結果のデータをベースとして使用します。 設計変更の最終目標は、作業システムを改善し、トラフィックを処理し、変換することであることを忘れないでください。 これらのタスクのいずれも解決できない場合は、何も開始しないでください。



変更を行った後、トラフィック統計を確認し、コンバージョン率と比較します。 LinchpinSEO.comのBill Rossは次のように書いています。



「最初に分析するのはトラフィック統計ですが、次に、トラフィックの変化がわずかであり、コンバージョンが急速に成長しているためにトラフィックが不足していることがよくあるため、数値をコンバージョン評価と明確に比較する必要があります！」




必要なすべての変更を行った後、A / Bテストを実行します。 急いで複数のテストを一度に実行しないでください。 忍耐強く、1つのテストのために少なくとも2週間は取っておきます。



プラグインまたは機能を変更するときにA / Bテストを実行します



時には、より正確には、かなり頻繁に、サイトの機能を変更する必要があります。 これを行うとき、テストします。



変更が顧客データまたは購入プロセスに影響する場合は、A / Bテストを実施する必要があることを特に強調する必要があります。 いくつかの可能性があります。



ショッピングカート



あなたのカートはあなたのサイトの非常に敏感な部分です。 これは、お金が手から手へと渡り、感情の強さが強い場所です。 これは、潜在的な顧客の流れを失ったり、トラフィックの流れを中断したりする可能性がある場所です。 変換がプラスになるだけで、起こりうる損失が補償されるようにバスケットを設定します。すべての革新をチェックして、あなたの行動がプロジェクトに利益をもたらすか、害を及ぼす可能性がある場所、方法、理由を確認してください。



より高いコンバージョンを達成するのに役立つバスケットが必要ですか？ オプションのテスト中に、フォームの1つのバリアントが50％の失敗を示し、2番目のフォームが60％の失敗を示す場合、損失を最小限に抑えるためにどのような対策を講じるべきかを理解できます。



連絡先の収集および登録フォーム



私の観点からすると、最も重要な変換の1つは登録フォームです。 コンサルタント、コンテンツマーケティング担当者、またはSaaSビジネスにとって、メールは最も重要な連絡方法の1つです。 メールサービスプロバイダーを変更する場合、登録ページとランディングページ（または使用する連絡先収集フォーム）のA / Vテストを実施する必要があります。



メールサービスの大部分には、独自のユーザー登録ページがあります。 それらのいくつかは嫌です。 いくつかは美しいです。 回転させて、変換率が4％（以前の8％と比較して）であることがわかると、何かが完全に間違っていることがわかります。



幅広いサービスをご覧になることをお勧めします。 The Tyson ReportのRobert Tysonは次のように述べています。



「私は多くの異なる形式を使用しており、それらはすべて異なる結果をもたらします。 訪問者の1％から50％が購読者に変換されます。」




価格が変化したときのA / Bテスト



コンバージョンの最適化は、収益を増やす最も速い方法の1つです。 収益を増やすもう1つの簡単な方法は、価格を変更することです。 しかし、それはリスクを伴います。 価格の変更に失敗すると、収入を改善し、ゼロに減らすことができます。



価格を変更してA / Bテストを実施する方法



価格変更の可能性を検討している場合-まず第一に、テストします。 1つの製品の価格に2つのオプションを使用した場合の結果を比較してください。



明らかに、心の鍵が最低価格である顧客は常に存在します。 ただし、価格が特定のレベルを超えている場合にのみ、他の顧客が表示される場合があります。 実際に異なる価格オファーの影響を経験しない場合、これを知ることはできません。



以下は、TipsAndTricks-hq.comで価格の変更について人々がコメントする方法です。



コンバージョンが価格の変化にどのように反応するかを知る最良の方法は、販売ページをA / Bテストすることです。 したがって、販売ページには2つの異なる製品価格の2つのオプションがあります（たとえば、ページAは製品を47ドルで提供し、ページBは製品を97ドルで提供します）。 次に、顧客の50％を「ページA」に、顧客の50％を「ページB」に（できればランダムに）提供します。 十分なデータがある場合は、さらに2つの価格オファーで分割テストの別のオプションを設定し、さらに多くのデータを収集できます。




これは、異なる価格のコンバージョン率を測定するのに役立ちます。 上記の方法を使用すると、コンバージョンの増加という観点から「最も甘い」価格を特定できます。



A / Bテスト：価格の変更...など



このタイプのテストでは、変更の影響の一部を活用できます。 たとえば、ソフトウェアを非常に低価格（$ 2）で販売しているとします。 それがとても安いので、大勢の人がそれを買うでしょう。



しかし、売り上げの大幅な増加により、圧倒的な量のカスタマーサービスが生まれました。 それらに対処する時間がありません。 残念ながら、スタッフを雇うことは、変換の点で高すぎます。 あなたは何をしますか？



A / Bテストの後、価格を上げることにします。 これで、ソフトウェアの価格は40ドルになります。売り上げは減りますが、販売ごとにお金が増えます。 さらに、顧客サービスにほとんどすべてを費やすべきではありません。 製品開発、より良いサービス、顧客維持に時間を割くことができます。 さらに、すでに述べたように、より多くのお金を稼ぎます。



これは、A / Vテストの基本的な目的です。 ビジネスの状態に関する関連データを提供するだけでなく、コンバージョン率と収益に影響する追加のコメントも提供します。
