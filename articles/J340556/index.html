最初のTypeScriptデモ

最近では、 TypeScriptの5周年記念が行われ、週1回の技術会議でAndersと開発チームと会う機会がありました。 TypeScriptの人生における重要なマイルストーンを祝福し、プロジェクトを去ってから4年間でどれだけの成果を上げたかを伝えたかったのです。







さらに、プロジェクトの作業の最初にTypeScriptで作成された古いデモ（Stradaと呼ばれていました）を見つけて、現在TypeScriptで作業している人と一緒に視聴しました。



今日は、TypeScriptの始まりについてお話したいと思います。



開始する



2010年の秋に、Steve Luccoと私は新しいプロジェクトに取り組み始めました。 彼の目標は、アプリケーション開発チームが大量のJavaScriptコードを管理できるようにすることでした。 スティーブは本社の複数のテクニカルディレクターと話をしました。 彼らは、部門のC ++ / C＃からJavaScriptへの急速な移行について話し、現在、プログラムと対話するためのWebアプリケーションおよび対応するスクリプトの開発に真剣な努力が注がれています。 開発チームには、Visual Studioなどの品質の高いツールや、C ++およびC＃型システムが提供する機能がありませんでした。 彼らは、JavaScriptへの移行を促進するために私たちができることを知りたいと思っています。 その後、同様の方向に向かう強力なITトレンドに注目しました。



高速なJavaScriptエンジンが登場し、HTML5が開発され、印象的なWebアプリケーションのサイズと機能がいくつか登場しました。 これらはすべて、WebでのJavaScriptの使用方法の急速な変化を示しており、過去数年はそうではありませんでした。



その時までには、前述の問題を解決するための多くのオプションがすでにありましたが、それらのどれも私たちに適さず、かなり広い市場ニッチの期待に応えませんでした。 マイクロソフト内では、いくつかの大規模なチームがスクリプト＃を使用しました 。 これにより、JavaScriptの代わりにC＃を使用できるようになりました。 ただし、結果として、実際にプログラムした環境からは遠すぎました。 これはいくつかの不便を引き起こしました。 また、 Google Closure Compilerがあり、JSコード内のコメントに埋め込まれた豊富なタイプのシステムを提供しました。これにより、高度なミニファイプロセスを制御できます（同時に、タイプに関連するエラーを追跡して報告できます）。 そして最後に、 CoffeeScriptの全盛期がありました。 CoffeeScriptは、JS開発におけるトランスパイラーへの道を開いた、広く使用された最初のJavaScript言語でした。 ちなみに、私は最初に次の類推を使用してTypeScriptの機能についてよく話しました：「 CoffeeScript：TypeScript :: Ruby：C＃/ Java / C ++ 」、しばしば次のように追加します。 Rubyで書く人:) 」。



私たちはすぐに、上記の3つのテクノロジーの交差点に位置するもの、あらゆる点で既存のものよりも優れたものを作成したいと考えました。 私たちの見解では、これはセマンティクスと構文が可能な限りJavaScriptに近い言語であると想定されていました。 そのようなプロパティは、それぞれCoffeeScriptとClosure Compilerが所有していました。 さらに、この言語が型チェックと便利なツールをサポートすること、つまりスクリプト＃を最大限に活用することを望んでいました。



2010年の秋と冬に、マイナープロジェクトとしてTypeScriptの作業を開始しました。 リラックスしないために、私たちの仕事の結果が他の人に示されるように、やる気を起こさせるために、私たちがやったことのデモンストレーションを伴う内部プレゼンテーションを計画しました。 これは、Microsoftの研究およびソフトウェア開発チームのプログラミング言語スタッフが招待されるミニ会議であることになっています。 しかし、イベントの1か月前に、スティーブ（当時チームで唯一のエンジニア）が手首を負傷したため、正常にプログラミングできませんでした。 その結果、計画されたパフォーマンスを回避したくないので、実際に動作するコンパイラがなくても、私たちが目指していることを実証できるすべてのものを収集することにしました。 （すぐに、スティーブは最初の真のTypeScriptコンパイラーを作成しました。これにより、最初の内部ユーザー（後にVSコードになったものを作成したチーム）がTypeScriptを使用して実際の問題を解決できるようになりました）。

まとめ



最初のデモから約1年半後、 TypeScriptをリリースしました 。 そして、5年後、TypeScriptはソフトウェア開発業界で最も急速に成長しているプログラミング言語の1つになりました。



さて、これを外部のオブザーバーとして見ると、TypeScriptチームが言語の最初のアイデアを固守する一貫性を賞賛せざるを得ません。 重要なのは、TypeScriptはJavaScriptの単なる軽量な抽象化であるということです。 また、元々指定されていた同じフレームワークを保持したまま、言語がどのように発展するかという楽しい印象を作り出します。 つまり、その目的は、ほとんどすべてのJavaScriptプロジェクトで生産的に使用できる型システムを提供することです。 TypeScriptは、JSエコシステムにおける新しいフレームワークと開発スタイルの出現により、その理想に忠実であり続けています。
