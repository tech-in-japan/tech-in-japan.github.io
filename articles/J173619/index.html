粘り強いSCSI

データストレージの履歴を検討し始めたので、前の記事でのみ説明したテクノロジーの1つを詳しく見ていきます。 この技術の驚くべきことは、80年代の初めに登場して以来、さまざまな変化を経て現在まで生き残っており、道を譲らないことです。 SCSIについてです。



Shugart AssociatesはSCSIの「創始者」と考えることができます。SCSIがその後生まれた標準は、元々SASI（Shugart Associates System Interface）と呼ばれ、ロシアの耳にはやや不協和でした。 70年代後半にはあまり広く知られていなかったこの会社は、ドライブ市場をほぼ独占し、人気のある5¼インチフォーマットを提案したのはこの会社でした。 SASIコントローラは通常、ドライブの半分のサイズで、50ピンフラットケーブルで接続されていましたが、これは後にSCSI-1コネクタになりました。







標準の名前変更はANSIで、1982年までにこのインターフェイスの標準が開発されました。 事実、ANSIポリシーでは標準名に会社名を使用できないため、SASIは「Small Computer System Interface」に名前が変更されました。 標準の「父」であるラリー・ブーシェは、この略語を「サクシー」と読みたいと思っていましたが、ダル・アランからの読みは「もっと教えてください」。



SCSIは主にハードドライブに関連付けられているという事実にもかかわらず、この標準では、このインターフェイスを介して接続されるほとんどすべてのデバイスを作成できます。 SCSIを搭載したデバイスは多数ありました。ハードドライブ、光磁気ドライブ、CDおよびDVDドライブ、ストリーマー、プリンター、さらにはスキャナー（LPTポートは高解像度のカラースキャナーが機能するには遅すぎました）。



ほとんどの「単純なコンピューター」SCSIはインターフェースとして使用されないという事実にもかかわらず、この標準のコマンドセットは広く使用されています。 たとえば、SCSI命令セットは、ストレージデバイスをサポートするために単一のWindowsスタックにプログラムで実装されています。 また、ATAPIと呼ばれる、CD / DVDおよびBlueRayで動作するために使用されるIDA / ATAおよびSATAインターフェイスを介したSCSIコマンドの実装は、実質的に標準になりました。 また、USBの上に実装されたこのコマンドシステムは、外部USBデータストレージを使用できる大容量記憶装置標準の一部になりました。







歴史的に、パラレルSCSIは最初のインターフェイスであり、多くの場合シールドされたフラットケーブルに配置された50、68、および80ピンコネクタを使用していました。 この規格は、SCSI-Iでの40メガビット/秒からUltra-640 SCSIでの5120メガビット/秒へと進化してきました。 この進歩にもかかわらず、このインターフェースはすでに廃止されており、Serial Attached SCSI（SAS）の形式のより高度なソリューションに置き換えられていますが、SASについては少し後です。 今のところ、SCSIで使用されている他のケーブルインターフェイスを見てみましょう。



FCP（Fiber Channel ProtocolまたはFC）と呼ばれる光ファイバチャネルを介したSCSIプロトコルの実装により、高速で長い接続ケーブル（シングルモードオプティクスを使用して数十キロメートルまで）を使用できます。 コストが高いため、大規模なデータ処理ネットワークで使用されます。 今日、FCはいわゆる「フロントエンド」、つまりコンピューター（主にサーバー）をストレージシステムに接続するためにのみ使用されます。







名前の最初の文字から明らかなように、iSCSIはインターネット経由でSCSIコマンドを処理するために使用されますが、ほとんどの場合、企業のローカルIPネットワークで使用されます。 この標準はFCPとは異なり、高価な機器を必要としないため、多くの場合「光学」の安価な代替品として機能します。 この標準のアプリケーションの主な2つのシナリオは、データストレージの統合（異なるネットワーク接続ストレージの代わりに、ほとんどの場合データセンターでデータが1か所で収集される）と障害保護（iSCSIにより、グローバルネットワーク全体のコンシューマ向けにディスクアレイの透過的なストレージが可能になる）です。 iSCSIは、ネットワークモデルの物理層とリンク層に大きく依存しており、環境の速度と最大遅延を決定します。 そのため、1Gb / sイーサネット上のiSCSIでは、最大100MB / sの転送のみが許可されていました。 10GEネットワークの方がはるかに優れていますが、10GEテクノロジーは現在、企業ネットワークで重要なセグメントを獲得しつつあります。



SCSI RDMAプロトコル（多くの場合、リモートSCSIと略されます）を使用すると、リモートダイレクトメモリアクセスプロトコルを使用して、あるコンピューターを別のコンピューターのリソースに接続できます。 RDMAは低遅延と高スループットが特徴であるため、クラスターで最大のアプリケーションを受信して​​います。



USB接続SCSI-名前からも明らかです。 USBインターフェイスを介してSCSI接続を実装します。 理論的には、最大3200 Mbit / sの速度を提供できます。



シリアルアタッチドSCSI-現在最も一般的なタイプのインターフェイスで、古いパラレルバージョンを置き換えます。 最大6 Gb / sの速度を提供します（12 Gb / sオプションは現在開発中です。2013年夏にリリースされる予定です）。 このプロトコルは、一般的に使用されるSATAの「最も近い」ものであり、さらに、SATAデバイスをSASコントローラーに接続できます。 ただし、反対の方向では、これは機能せず、SASコネクターでさえ、これを防ぐように設計されています（システムビルダーが非常に強力になることを除く）。



古いパラレルSCSIとは異なり、SASシステムには次の利点があります。



-信号線が少ないため、よりコンパクトなワイヤ。

-バスを使用したP-SCSIとは対照的に、ポイントツーポイント接続を使用します。これにより、コントローラーで同時に動作できるデバイスは1つだけです。

-SASは特別なケーブルターミネーターを必要としません。

-新しいプロトコルは、バス上の異なるワイヤに信号を同時に渡さないという問題に悩まされません。

-同時にサポートされるデバイスの数が劇的に増加し、古いSCSIが8から32に、新しいSCSIが16384以上サポートされました。

-SASはより高い帯域幅を提供し、特に優れている点-この帯域幅は各イニシエーターとターゲットデバイス間で効果的に使用されますが、パラレルSCSIバスの帯域幅はすべてのデバイス間で均等に分割されました。

-SATAデバイスとの互換性







データストレージの歴史と密接に関連して、SCSIとSASは会社のLSIの歴史と結びついています。 ASIC（特定用途向け集積回路）デバイス（専用タスクの実行を目的とした特殊なマイクロ回路）を導入した世界初の企業の1つであるため、LSIはこのような急速に発展する分野を通過できませんでした。 LSIの場合、SCSIの歴史は1998年にSymbios部門を買収したことから始まりました。Symbios部門はかつてNCR、AT＆T、およびHyundaiの一部でした。 このユニットは1971年にNCRで設立され、買収時にはSCSIアダプターのメーカーでした。 LSIがリリースするデバイスの範囲は、LSI53C825AEチップから始まります。このチップでは、最大20MB / sのデータをスキップでき、Fast Wide SCSI規格に準拠していました。 開発の過程で、いわゆるFusion-MPT（Message Passing Technology）アーキテクチャが開発されました。これにより、オペレーティングシステムのデバイスドライバーを可能な限り「軽量化」し、I / Oサブシステムのコンポーネント間の通信プロセスを高速化できます。 このテクノロジーは、最初のSCSIチップよりも3桁以上高速な今日のSAS-3（12Gbit / s）デバイスまで生き残っています。 コントローラーに加えて、SCSI LSIでは、ホストアダプターやSCSIコンバーター（エキスパンダー）などの他のチップも製造しました。



もう1つ興味深い事実に言及するしかありません。 最近では、Seagateは20億台目のハードドライブのリリースを発表しました。 ハードドライブは、多数の異なるコンポーネントを含む非常にハイテクなデバイスです。 LSIは、ハードドライブ用のかなりの数のチップを生産し、生産し続けています。 従来、これは「読み取りチャネル」でしたが、現在ではいわゆるSOC（System on Chip）があります。







SCSI市場のもう1つの主要なプレーヤーはAdaptecです。 OEMチャネルと常に密接に連携してきたLSIとは異なり、Adaptecは主に完成品に焦点を当てていたため、独自の名前でITの世界でよく知られています。 約2年前、同社は半導体デバイスPMCの製造会社に買収され、現在では同社の製品はPMCによってAdaptecと呼ばれています。 同社はまた、すべての世代のSCSIを無事に乗り切り、現在SAS用のデバイスをリリースしています。



パラレルSCSIの歴史は長く乱れていましたが、最後には終わりました。 2年以上にわたり、P-SCSIアダプターは生産されておらず、SASテクノロジーに負けており、すでに2世代を数えています。
