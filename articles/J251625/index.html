新しいutorrentのバグ、またはBittorrentプログラマーが準備しているもの

多くのutorrentユーザーは、習慣からプログラムの新しいバージョンに切り替えるか、何かが機能しなくなった瞬間まで切り替えます。 誰かが2.2.1または2.0.4で停止しました。後者は多かれ少なかれ安定しており、今日の誰かは1.8.2および1.7.7にさえ忠実です。 なぜutorrentの3番目のバージョンはとても怖くて魅力的ですか？ 変更点を追跡し、今後の変更を追跡して理解してみましょう。







Utorrent Evolution


最初にインストールしたutorrentは1.8.2でした。 これらのクライアントでマグネットリンクが固定され始めたちょうどその時に。 おそらく、この機能を追加するときにutorrentプログラマーが犯した間違いは、ユーザー1.7.7および1.8.2にはあまりにも顕著であることが判明しました。 その瞬間から、プログラムを作成するための多くの相反するアプローチが適用され始めたと言えます。

その後、開発者は、多くの革新的なアルファバージョン1.9を思い浮かべようとしました。 以下のような注目すべきもののうち、udpトラッカーのサポート、新しいutpプロトコル、アプリケーション、マグネットリンクの追加パラメーターの作業。 しかし、バージョン2.0は安定したリリースになり、長い間バージョン2.2.1にパッチを当てようとしました。

そして、ここから楽しみが始まります。 バージョン3.0以降、プログラムに根本的な変更が加えられ、「Falcon」および「Griffin」という名前のutorrentの特別なアルファ版によってそれらに備えられました。 しかし、私たちのほとんどは、ハードドライブの膨大な数の不具合と問題にのみ気づきました。 したがって、長い間、多くの人が2.2.1から3.xへの切り替えを当然拒否しました。 また、3.xには多くの新機能がありました。



非常に興味深いのは、DNAと呼ばれる技術です。 この名前の下で、彼らは一度に2つのアイデアを突き出そうとしました：

1.トレントクライアントのないコンピューターで実行された特別なファイルを使用してp2pを介してファイルをダウンロードするか、メインのutorrentウィンドウに青いDNAアイコンが表示されている間にutorrent自体で実行できます（残念ながら、スクリーンショットは保存されませんでした）。 そのような特別なファイルは、ドライバーのダウンロードセクションのASUS Webサイトなど、顧客にダウンロードするように提供されています。 それらを起動することにより、ユーザーはp2pを介してドライバーファイルをダウンロードできます。

2. p2pから部分的にダウンロードして、httpページに埋め込まれたプレーヤーでビデオを表示します。



マグネットリンクは、新しい追加パラメーターws（ウェブシード）およびfl（直接httpリンクを介してトレントファイルをダウンロード）を理解しました



クライアントには、映画をダウンロードして同時に見ることができるときに、メディアファイルをストリーミングするための組み込みのメカニズムがありました。



多くのプロバイダーは、isp.bep22オプションがデフォルトで有効になるのを待ちました。



クライアントのip-filterに加えて、ユーザーはutorrentが曲のキューに接続する優先IP範囲の特別なファイルを作成できました。 したがって、多くの人がダウンロード速度を大幅に向上させることができました。



「評価」と「コメント」を表示します。



そして最後に、HTTPプロトコルを使用して特別なプロキシサーバーを介してトレントクライアントから友人や知人にファイルを転送する、忘れられたがまだ機能している機能。 ところで、これはプロキシWeb sidとして使用できます。



ファイルまたはフォルダーをプログラムウィンドウにドラッグするだけで、utorrentはhttpページへの特別なリンクを作成します。このページからトレントとp2pの存在を知らない友人は、両方ともNATプロバイダーであってもコンピューターからファイルをダウンロードできます。 残念ながら、どうやら、すぐに開発者はプログラムからこの設定を削除します。これは、新しいバージョンのutorrentでは、タンバリンとのダンスでのみ機能するためです。 ただし、パフォーマンスをutorrent 3.xに戻したい場合は、プログラムの非表示設定で次のパラメーターを変更してください。

SHIFTとF2を押したままにすると、設定----詳細に移動します。 リストの一番下で、行の値を見つけて変更します。

webui.raptor_host-raptor.utorrent.com

webui.raptor_port-80

webui.raptor_secure-false

webui.talon_host-remote.utorrent.com

webui.talon_port-80

webui.talon_secure-false

[リモートアクセス]設定では、名前とパスワードを考えて入力する必要があります。それらがないと機能しません。



3.3.2までの後続のバージョンは、バグに関する作業と呼ばれます。 そして、最終的には、将来のプログラムの安定性に優れています...新しいアイデアがなければ、新しいutorrent 3.4



Utorrent 3.4-忘れられた革命？


DNAとraptor.utorrent.comで障害が発生した後、開発者はトレントとWebを長いボックスに統合する計画を立てると思いますか？ どんなに！ すべては新しい円で始まります。 今回はutorrentがファイルhttpサーバーに変わりますが、もちろん独自のp2p機能があります。



続行する前の小さな余談。

Bittorrentは、長い間p2pの分散化に興味を持っていない営利団体であることを理解する必要があります。 プログラムの多くの機能が消えて表示され、私たちのほとんどはそれらの存在を認識していません。 utorrentの開発におけるすべての願望は、お金を稼ぐことを目的としています。 このため、プログラムの新しいビルドで良いことを期待しないでください。 非常に高い確率で、utorrent 3.5で以下に記述されるすべてのものが削除されるか、根本的にやり直されます。つまり、それを使用することはできません。





utorrent 3.4.2の小さな目立った変更のうち、私にとっては、おそらくhttps web sid（結局、Dropboxは依然としてrutrackerで私のトレントの一部を配信しています）とProバージョンの無効化されていない広告で動作するでしょう。



最も興味深いのは目から隠されていますが、「ロープ」を引くと、現在行われていることを見つけて「感じる」ことができ、プログラムの将来のバージョンで動作する可能性があります。

http://pairing:1B40250178C560733172CE3C811A4ACA831D6FB8@127.0.0.1:15906/proxy?sid=2&file=0&token=1B40250178C560733172CE3C811A4ACA831D6FB8&pairing=1B40250178C560733172CE3C811A4ACA831D6FB8&service=STREAMING">


このリンクをブラウザに挿入すると、組み込みのブラウザプレーヤーでファイルを再生するか、ダウンロードフォルダにダウンロードするだけです。

このリンクには、utorrent設定に関連するいくつかのパラメーターがあります

webui.allow_pairing

webui.proxy_auth

webui.token_auth

ただし、リンク内のパラメーターの半分（ハッシュに関連するすべて）が機能しないため、これを短くすることができます 
  http://127.0.0.1:15906/proxy?sid=2&file=0
これがブラウザに挿入されると、完全なリンクの場合と同じことが起こります。 作業リンクパラメーター：

sid =トレント番号（任意の値を使用できます。トレントを追加する場合、それはutorrentです）。

file =ファイル番号（永続的なもの、パス列の位置によって割り当てられる）

最初に推測したかもしれませんが、「Copy URL URL」オプションの名前とIP 127.0.0.1のすべてのコピーされたリンクは、同じコンピューターでダウンロードされたコンテンツを再生するためだけに作成されます。 しかし、開発者が（統合されたhttpサーバーで）それほど難しくしなければならない理由は完全には明らかではありません...

このようなリンクに関するクライアントの作業のメカニズムは興味深いものです。 たとえば、トラッカーからトレントファイルをダウンロードしてutorrentで実行し、タスクを停止（ダウンロードしない）してから、リストからファイルのコピーストリームリンクをコピーし、このリンクをブラウザーに貼り付けます。 ブラウザーがutorrentのリンクをたどるとすぐに、このファイルの一部のみの順次ダウンロードのプロセスが開始され、ブラウザーに徐々に転送されます。 ブラウザの読み込みが停止し、ブラウザプログラムが閉じられると、utorrentはタスクを停止して応答します。

utorrent 3.4では、クライアントのタスクを自動的に更新する機能が追加されました。 そのため、トラッカーからシリーズのトレントファイルをダウンロードしたため、将来、フォーラムで新しいシリーズのリリースと配布の更新をフォローすることはできませんが、更新されたトレントと新しいシリーズをすぐに入手できます。

（特別なプログラムを使用して）トレントファイルを作成するプロセスまたはutorrentタスクのトレントプロパティに 、トレントファイルを更新するためのhttpリンクが書き込まれます。



クライアントを再起動すると、タスクの近くに青い「ドット」が表示されます。つまり、utorrentは新しいtorrentファイルをダウンロードするためにサイトを定期的にポーリングします。

utorrent自体では、各トレントの更新フィード（日付とファイル）を見ることができます
