教育向けWindows Azure

この記事では、コンピューターサイエンスの教師が学生の実習を含むクラスで無料で教育目的でWindows Azureを使用する方法について説明します。







エントリー



トレーニングセンター（改めて詳しく説明します）では、最小限の労力で作業を構築しようとします。 私たちのチームの全員にとっては、トレーニングセンターでの作業が主なものであるため、論理は明らかです。



問題



ご存知のように、ロシアの教育（高等教育を含む）は技術に追いついていません。 このルールに例外がある場合、ルールの存在のみを確認します。 問題は何ですか？ もちろん、問題は複雑で曖昧ですが、それが根本です。



教育機関でのコンピュータークラスの管理は、非常に複雑なプロセスです。 多くの場合、コストがかかるため、複雑です（管理者は常にパンを扱うとは限りません）。 標準的な状況を考慮してください。



コンピュータークラスは通常、Windows XPを使用します（教育プロセスで使用される古いソフトウェアの束と互換性があるため）。 ユーザーが管理者権限なしで作業するのは良いことです（古いsotfで作業する場合は常にそうとは限りません）、そうでない場合は通常、イメージの厳しいシステムが使用されます（つまり、コンピューターがプログラムで破損した場合、すぐにロールバックされます）。



そして、新しい教師が、人気のあるディストリビューションを例として使用してLinuxコースを受講したいと考えているとします。 彼は何に直面しますか？



多くの問題があります。 「なぜこれが必要なのか」という管理者のしつこさを解消しても、脈を失う前に2番目のシステムをサポートしたり、再インストールしたりするのに十分な問題があります。 さて、先生自身が十分な資格を持っている場合、そうでない場合はどうなりますか？



現実に戻る



同様の問題に直面しました。 教室にWindows 7があり、マルチブートを作成したり、別のハードドライブを配置したりしませんでした（システムユニットのフォームファクターは許可されません）。 したがって、Windows Azureのアイデアは偶然に思い浮かびました。



解決策



Windows Azureが教育に適しているのはなぜですか？ さて、仮想マシン（まだプレビューモード、つまりベータ版）で作業する機会があります。これは非常に適しています-私たちはそこに任意のLinuxイメージを取得して展開し、自分で静かに作業します。 学生が突然OSを「破壊」した場合（rm -rf /または/ etc / fstabでミスを犯した場合）、オペレーティングシステムは単純に再び展開します。



仕事の仕方は？ まあ、最初は3か月間無料のデモアカウントを考えましたが、解決策は異なりました。



別の問題



それでは、デモアカウントのどこが悪かったのでしょうか？ 一般に、問題は、確認のために携帯電話番号（一部の学生は携帯電話を持っていなかった、そうです）だけでなく、クレジットカードの詳細も必要なことでした。



すべての学生がカード保持者であるわけではありません。また、だれでも参加できる選択科目を実施する場合は特に困難です。 ユーザーがクレジットカードを持っているという事実ではありません。クレジットカードを使用して、インターネットを介して支払いを行うことができます。



この問題はまったく予想外に解決されました。モスクワでWindows 8を起動するサミットで、Windows AzureチームのメンバーがWindows Azureへのアカデミックアクセスに関するストーリーを共有しました。これはWindows Azure Academic Passと呼ばれます。 教師と生徒のみ。



フィールドテスト



そこで、サミットで、私はこのアカデミックパスの申請書を送り、「3〜5営業日以内に」という応答を受け取り、その年の終わりが来たので、私は突然それを受け入れませんでした。



年末年始が終わった後、3〜5営業日がどうやら過ぎたように思え、2通目の手紙を書きました。 1時間後、待望のアカデミックパスが私と共にありました。



中身は？



それでは、アカデミックパスとは何ですか？ これは、5か月間サービスを使用する機会です（通常、学期は4か月間続き、1か月は学生が個別に選択できるようになります）。

教育に関しては、それはまさに私に思えます。 むしろ、そうではないと言います-私たちは十分でした:)



軟膏で飛ぶ



もちろん、すべてが私が言うほど楽しいとは限りません-いくつかの問題があります。 そして、それらについて頭を叩く必要がないように、すぐにそれらについて説明します。



問題1


学生はライブIDを持っている必要があります。 そして、これは常にそうではありません。 ほとんどの場合、彼らはこのライブIDを持っていないため、想定されていませんでした。 したがって、登録する必要があります。 ただし、登録の制限は1つのIPアドレスから3アカウントであるため、最初のレッスンでこれを行うことは望みません。



2つの方法があります。



1.学生は、事前に携帯電話または自宅からLive IDを登録する必要があります。 悪いことは、重要な瞬間に誰かが間違いなくパスワードを忘れてレッスンを引きずってしまうことです。



2. Live IDパックを自分で登録するのが最も便利です。 確かに、自動一括登録のためのツールはまだありません。



また、1つのIPアドレスに対する3つのライブIDの制限を削除できるため、サポートに書き込む必要があります。 しかし、今回はこれを考慮してください。



第二の問題


アカデミックパスのアクティベーションには時間がかかります。 つまり これは15分以内に発生することもあれば、24時間以内に発生することもあります。 そのため、アカデミックパスは事前に受け取り、学生に配布するか、自分で有効化する必要があります（事前に）。



問題3


（サービスとしての）仮想マシンはまだプレビューモードです。 ベータ版。 つまり、問題が発生した場合の公式サポートは役に立たず、TechNetフォーラムに送信されます。 何かうまくいかないことはありますか？ はい、私は例がありました-突然誰かがUbuntuのリポジトリを壊しました（彼らはWindows Azureに独自のものを持っています）。 テクニカルサポートは何もしませんでしたが、Canonicalリポジトリを標準リポジトリに置き換えることはうまくいきました。



どこで話しますか？



さて、主な質問-それを取得する方法？ 簡単で、最低限の英語の知識が必要です。



ステップ1


アドレスwww.windowsazurepass.com/?pageMode=Academicにアクセスして、簡単なアンケートに記入します（記入の例をすぐに示します。もちろん英語の方が良いです）。



  • 氏名：イヴァン・イワノフ
  • メール：ivanov@mydomain.ru
  • 国：ロシア
  • 大学：（ここでは英語での教育機関の名前）MSU
  • コース名：（読みたいコースの名前）LinuxをWebプラットフォームとして使用する
  • コースの説明：（コースの説明、最大4000文字）初心者向けの短いコース。 主な目標-Linuxの基本を学び、LAMPサーバーの設定を理解します。
  • コース開始日：（コース開始予定日）：2013-03-01
  • 学生数：（学生数）20
  • これは、Azure Passが発行した番号ではなく、コースに参加している学生の数です。AzurePassの数とは異なる場合があります。
  • 要求されたパスの数：（要求されたAzureパスの数）
  • コースURL：（もしあれば、コースがあるウェブサイトのアドレス） mydomain.com/mycourse




ステップ2


応答として、最初にアプリケーションの受領を確認する手紙を受け取り、次にWord文書（Academic Institutions向けのWindows Azureオファー）を含む手紙を受け取ります。 その本質は次のとおりです。Microsoftは無料で有料のリソースを提供しているため、一部の国の法律によれば、これは賄asと見なされる可能性があります。 したがって、このレターには、提供されるサービスの量、その市場価値（MicrosoftのWindows Azureの価格に基づく）、およびこれらのサービスが必要でありこれが賄notではないことを明示的に同意する必要性が示されます。



この手紙は印刷され、教育機関または学科の長により署名され（私はまた、より高い信頼性のために印を付けます）、それに応じてスキャンを送信しなければなりません。



ステップ3


この段階で、手順1で指定した数と同じ数のプロモーションコードが含まれたExcelファイルを受け取る必要があります。次に、Windows AzureにログインするLive IDにそれらを添付する必要があります。 これを行うには、アドレスwww.windowsazurepass.comにアクセスして国を選択し、コードを入力して[送信]をクリックします。 次に、システムはバウチャーが正しいことを確認し、Live IDでログインして接続を完了するように求めます。



Live IDでログインすると、小さなアンケートが開き、名前と姓（Live IDで自動的にプルアップされます）、電話番号（全員に同じでもかまいません）、および機関の名前を示す必要があります。



同じ名前、姓、教育機関の学生がいる場合、これは起こり得ないとシステムが言うという事実には驚きがあります。 このプロファイルの名前に1番を追加しただけで、すべて機能しました。



ステップ4


ここで、Windows Azureチームからのメッセージを待つ必要があります。3〜4ステップのプロセスで、15分から1日かかることがあります。 事前にこれを行うことをお勧めします。



ステップ5


Live IDを使用してWindows Azureにログインし、サブスクリプションを使用できます。



まとめると



私たちの目標は最終的に達成されました-選択科目を開催し、学生は選択科目の後数ヶ月間勉強する機会を得ました。



教育でWindows Azureを使用すると、多くの教師にとって生活が楽になると確信しています。コンピュータークラスで使用しているソフトウェアの種類に依存しなくなりました。 コンピュータクラスの概念そのものが変わりつつあります。現在、これらは単なるインターネットアクセスを備えたコンピュータであり、もはやありません。



誰かが他のクラウドプラットフォームの同様のアプリケーションをカバーできればうれしいです。この問題に関しては、マイクロソフトだけではないと思います。
