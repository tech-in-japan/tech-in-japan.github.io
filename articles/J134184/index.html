SL4A + Pythonを使用したAndroid用の最も単純なFTPファイル同期のためのプログラムの作成

はじめに



この記事では、UbuntuオペレーティングシステムのPythonでFTPプロトコルを使用してファイルを同期するサンプルプログラムを使用して、SL4Aを使用してプログラムを作成するプロセスについて説明します。 SL4Aアプリケーションの開発のためのオペレーティングシステムの構成を示し、アプリケーション自体を記述し、バーコードを使用してアプリケーションを配布し、アプリケーションを.apkファイルにパッケージ化します。



SL4Aとは何か、何と一緒に食べられるかについてのいくつかの言葉



Damon KohlerとGoogleが提供する20％の自由時間のおかげで生まれたSL4A（Scripting Layer for Android）プロジェクトにより、さまざまなインタープリター型プログラミング言語（現在はPython、Perl、 JRuby、Lua、BeanShell、JavaScript、Tcl、およびShell）。

SL4Aについてはすでにhabrに書かれています。 たとえば、ハブのカルマをチェックするためにPythonでアプリケーションを作成する方法について説明しました。 ここでは 、Luaを使用して、夜間に電話通信を切断するスクリプトが作成されました。



SL4Aは、プログラミングの教育、プログラムのプロトタイプの作成、Androidプラットフォームでのアクションの自動化タスクの解決に最適です。 大きくて深刻なプログラムはSL4Aで作成されていませんが、ミニプログラムの作成には適しています。 ハブのカルマを確認し、妻にSMSを送信して、自宅から数キロ離れたところで夕食の準備を始めるように頼みます。 目的の駅に近づいたときに電車で目覚めたり、実際のロケットを制御したりすることも、SL4Aを使用して最小限のコードと時間で実現できます。



比較のために、JavaとPythonの最も単純なプログラムを以下に示します（ PyCon 2011プレゼンテーションから取られた例）： 

package com.spodon.pycon; import android.app.Activity; import android.app.AlertDialog; import android.content.DialogInterface; import android.os.Bundle; import android.widget.EditText; import android.widget.Toast; public class Demo extends Activity { private EditText mEditText = null; @Override public void onCreate(Bundle savedInstanceState) { super.onCreate(savedInstanceState); setContentView(R.layout.main); AlertDialog.Builder builder = new AlertDialog.Builder(this); builder.setTitle("Hello!"); builder.setMessage("What is your name?"); mEditText = new EditText(this); builder.setView(mEditText); builder.setNegativeButton("Cancel", new DialogInterface.OnClickListener() { @Override public void onClick(DialogInterface dialog, int which) { Toast.makeText(Demo.this, "Cancelled", Toast.LENGTH_SHORT).show(); } }); ...


同じPythonプログラムは4行（！）だけを取ります： 

 import android droid = android.Android() name = droid.getInput("Hello!", "What is your name?") droid.makeToast("Hello, %s" % name.result)


彼らが言うように、違いは明らかです。 もちろん、欠陥がなければできません。 記事の最後でそれらについて。



本体



電話機にSL4Aをインストールします


それでは、同期アプリケーション自体の作成を始めましょう。

SL4Aを使用するには、携帯電話にアプリケーションをインストールする必要があります。 ここまたはバーコードから取得できます。







SL4Aアプリケーションには直接、シェルインタープリターのみがあります。 他のプログラミング言語を使用するには、追加のアプリケーションをインストールする必要があります。 Pythonインタープリターはこちらです。



SL4Aを使用するためのオペレーティングシステムのセットアップ


ファイル同期アプリケーションを機能させるには、コンピューターにftpサーバーをインストールする必要があります。 Ubuntuにはvsftpdアプリケーションがあり、これは無料で簡単に設定できるftpサーバーです。 インストールするには、ターミナルでコマンドを入力するだけです：

sudo apt-get install vsftpd





サーバー設定ファイルは/etc/vsftpd.confにあります。ファイルの書き込み許可を追加して、行のコメントを解除する必要があります。

#write_enable=YES





サーバーを再起動します。

$sudo service vsftpd restart







SL4Aアプリケーションは電話で直接作成できますが、仮想キーボードでコードを入力するタスクは、小さなアプリケーションでも非常に疲れるので、コンピューターでアプリケーションを作成して電話にアップロードする方がはるかに便利です。

コンピューターで書かれたSL4Aアプリケーションを開始するには、まず電話でSL4Aサーバーを開始する必要があります（SL4Aアプリケーションのメインページで、[メニュー]-[インタープリター]-[メニュー]-[サーバーの開始]を押します）。 プライベートサーバーを起動します。 サーバーが実行されているという通知が表示され、サーバーが使用するポートも示されます。







コメント：パブリックサーバーを起動する選択肢があります。 パブリックサーバーを起動すると、IPアドレスにアクセスできる人はだれでも電話を制御できます。 プライベートサーバーを起動し、以下に示すように環境変数を設定すると、コンピューターで実行されるスクリプトは、電話で変更なしで機能します。 プライベートサーバーが機能するには、 Android SDKがインストールされている必要があります。



ポート9999に着信するすべてのローカルトラフィックをAndroidデバイスにリダイレクトします（サーバーがポート46136でリッスンしていると仮定します）。

$ adb forward tcp:9999 tcp:46136





環境変数を作成し、セットアップが完了します。

$ AP_PORT=9999





android.pyファイルをPythonライブラリフォルダーに追加するだけです。これで、コンピューターでアプリケーションを作成して実行し、結果をすぐに電話で見ることができます。 AndroidデバイスでhelloWorldを実行するには、Pythonインタープリターに入力するだけです。 

 >>>import android >>>droid = android.Android() >>>droid.makeToast("Hello, world!")


最初の行では、Androidライブラリがインポートされ、次にAndroid APIが使用されるdroidオブジェクトが作成されます。 最後の行には、デバイス画面に「Hello、World！」というメッセージが表示されます。

ここで、SL4Aが提供するAPIを詳しく見てみましょう。



プログラムソースコード


このプログラムは、Androidデバイス上の選択したフォルダー内のファイルとバックグラウンドでコンピューターを同期します。 新しいファイルの確認は30秒ごとに行われます。 プログラムのほとんどは、同様の機能を備えたパーソナルコンピューターアプリケーションとほとんど変わりません。 明確にするために、例外処理が省略される場合。 

 # -*- coding: utf-8 -*- import ftplib import time import os import android droid = android.Android() SERVER_IP = '192.168.0.101' USER = 'user' PASS = 'pass' REMOTE_FOLDER = 'sync' LOCAL_FOLDER = '/mnt/sdcard/sync' droid.makeToast("Started") if not os.path.exists(LOCAL_FOLDER): os.makedirs(LOCAL_FOLDER) while True: #   server = ftplib.FTP(SERVER_IP) server.login(USER, PASS) #     server.cwd(REMOTE_FOLDER) os.chdir(LOCAL_FOLDER) #      remote_files = set(server.nlst()) local_files = set(os.listdir(os.curdir)) #     for remote_file in remote_files - local_files: server.retrlines('RETR ' + remote_file, open(remote_file, 'w').write) #     for local_file in local_files - remote_files: server.storbinary('STOR ' + local_file, open(local_file, 'r')) #      30  server.close() time.sleep(30)


既知のdroid.makeToast（）を使用すると、同期の進行状況に関するメッセージがAndroidデバイスの画面に表示されます。



アプリケーションの起動


スクリプトの編集が完了したら、次のコマンドを使用してAndroidデバイスに配置できます。

$ adb push ftp_sync.py /sdcard/sl4a/scripts





コンピューターからスクリプトを実行するには、電話でアプリケーションを起動するエイリアスを作成すると便利です。 これを行うには、行を〜/ .bashrcに追加します。

alias andrun='adb shell am start -a com.googlecode.android_scripting.action.LAUNCH_FOREGROUND_SCRIPT -n com.googlecode.android_scripting/.activity.ScriptingLayerServiceLauncher -e com.googlecode.android_scripting.extra.SCRIPT_PATH'





次に、スクリプト自体をコマンドで実行できます

andrun /mnt/sdcard/sl4a/scripts/ftp_sync.py







アプリケーション配布


バーコードを使用したアプリケーションの配布


小さなSL4Aアプリケーションを配布する簡単で独創的な方法は、バーコードを使用してアプリケーションを転送することです。 たとえば、 ここでアプリケーションでバーコードを作成できます（タイプ-テキスト、サイズ-Lを選択）。 これがアプリケーションの外観です。







Androidデバイスにスクリプトをインストールするには、SL4Aアプリケーションを起動し、メニューを押して-追加-バーコードをスキャンします。



アプリケーションを.apkファイルにパックする


SL4Aスクリプトは、通常の.apkファイルにパッケージ化することもできます。 これを行うには、SL4Aプロジェクトテンプレートファイルをダウンロードして、Eclipseにインポートします。 不明な理由により、プロジェクトからgenフォルダーが最初に欠落しています。 プロジェクトのルートディレクトリに手動で作成する必要があります。 その後、Windows-環境設定-Java-ビルドパス-クラス変数で、Android SDKがインストールされたフォルダーを指すANDROID_SDK変数を作成します。



次に、res / rawフォルダーで、script.pyの内容をプログラムの内容に置き換えます。 次に、メニューの[Project-Build]をクリックします。すべてがうまくいった場合、アプリケーションを.apkファイルにビルドする準備ができています。 [ファイル]-[エクスポート]に移動し、[Android]-[Androidアプリケーションのエクスポート]を選択して、プロジェクトを選択します。



各Androidアプリケーションにはデジタル署名が必要です。 以前に作成されていない場合は、ダイアログボックスを使用して作成する必要があります。 最後のダイアログボックスでは、作成した.apkファイルを保存する場所を尋ねられます。 電話にアプリケーションをインストールします。

$ adb install ftp_sync.apk













srcディレクトリでcom.dummy.fooforandroidを選択し、[リファクタリング]メニューの[名前の変更]をクリックすると、スクリプトの名前を変更できます。 名前には少なくとも2つのピリオドが含まれている必要があります。 その後、AndroidManifest.xmlでアプリケーション名を変更する必要があります。 アプリケーションアイコンはres / drawableにあります。 必要に応じて、それらも変更できます。

おわりに



JavaやAndroid向けのモバイル開発に慣れていない人向けにSL4Aを使用してアプリケーションを作成することは、特に難しくありません。 Androidプラットフォームは、そのオープン性とアクセス性に優れています。 そのためのプログラミングを含む。 SL4Aは、Androidアプリケーション開発に新しいレベルのアクセシビリティをもたらし、さまざまなプログラミング言語を使用できるようにします。



SL4Aの欠点


  • SL4Aはまだアルファ版の段階にありますが、今では非常に多くのバグがあります
  • GUIを作成する機能は、webView（htmlマークアップ）および標準のダイアログボックスによって制限されます。 kAISTが示唆しているように、r5バージョン（まだ非公式）からGUIインターフェイスを作成する別の可能性があります-FullScreenUI
  • SL4Aを使用するアプリケーションのパフォーマンスは、Javaで記述されたアプリケーションのパフォーマンスを大幅に低下させます
  • 制限されたAPI機能
