プログラミング言語の歴史：BASICからVisual Basicまで



DONKEY.BAS。 1981年にIBM PCに付属。共著者-ビル・ゲイツ



BASICという名前は、「初心者向け—シンボリック命令の多目的言語」という文字通りの意味を持つ「初心者向けのシンボリック命令コード」の略語として登場しました。 これは、文字通りの翻訳が本質を正確に伝えた場合です。 BASICの主要な機能は、そのシンプルさだけでなく、コンピューターとの対話の問題の解決策を見つける能力でもありました。



60年代後半の多くのコンピューターでは、BASICは唯一の汎用高レベルプログラミング言語として位置付けられ、時間の経過とともにさまざまな修正が登場しました。 この言語の開発におけるターニングポイントは、Visual Basicの出現でした。



背景



1950年代半ばに、機械コードでのプログラミングの役割は衰退し始めました。 上位レベルの言語-Fortran、Algol、Cobolなどが登場しました。



それらの最初で最も一般的なものの1つは、1954年（最初のバージョン）にIBMのプログラマーグループによって開発されたFortran（FORmula TRANslatorのFORTRAN-式変換プログラム）でした。 この言語は、数学的性質の科学的および技術的計算に向けられていました。



1958年に最初のバージョンが登場したAlgol（ALGOrithmic Language）という名前は、アルゴリズムを記録するように設計されているという事実を強調しています。 明確な論理構造のおかげで、アルゴルは科学技術文献でアルゴリズムを記述する標準的な手段になりました。



次に、高レベルのプログラミング言語は主題指向によって特徴付けられました。 さらに、そのような言語は、特にエンジニアでない場合、初心者の開発者にとって複雑すぎました。 したがって、時間の経過とともに「単なる人間」のためのプログラミング言語が登場するのは当然です。



ベーシック。 ただ基本



英語の宣教師に関する有名な物語は、このプログラミング言語を作成するという考えに関連しています。

19世紀に、ネイティブとの接触を容易にし、簡素化する機会を探していた英語の宣教師が素晴らしいアイデアを思いつきました-英語から最も簡単な部分を分離することです。 この切り捨てられた英語の亜種は基本英語と呼ばれ、実践が示しているように、習得が非常に容易であることが判明し、すぐにネイティブの人々だけでなく、英語が母国語ではない移民の間でも人気を得ました。
100年後、ダートマス大学の数学部の従業員は同様の道を選んだ。 彼らは、「単純化された」プログラミング言語の助けを借りて、コンピューターと一般ユーザーの間の相互作用を促進することに決めました。



1964年、トーマス・カーツとジョン・カメニは、簡単な英語の単語で構成される特殊なプログラミング言語を作成しました。 新しい言語はBASICと呼ばれていました。







彼らはそれを使って学生にプログラミングスキルを教えました。 この言語は非常にシンプルで理解しやすいものであることが判明したため、しばらくすると他の教育機関で使用されるようになりました。







Fortran IIとAlgol 60はBASICに深刻な影響を与え、さらに開発者は、テキスト処理と行列演算だけでなく、タイムシェアリングモードを備えたシステムの動作メカニズムを実装しました。 言語コンパイラは元々GE-265メインフレーム用に作成されました。







ヘイデイ



70年代には、よりコンパクトなコンピューター（PC）が登場しました。 この状況は、Microsoftの創設者であるBill GatesとPaul Allenによって成功裏に使用されました。







彼らは、最初のAltairコンピューター（MITS Altair）向けに4KBのRAMで動作可能なBASICの新しいバージョンを作成しました。 やがて、このバージョンが世界で最も人気のあるプログラミング言語の1つになりました。





アルタイル



元のBASICの最大の繁栄と発展の期間は、1970年代の終わり-1980年代の前半と見なすことができます。 そのバージョンは他のプラットフォームに登場し、すぐにあらゆる種類の変更と改善により数百万のコピーに分岐し始めました。 最も人気のあるバージョンの1つはApplesoft BASICで、これはApple IIの標準言語になりました。



Applesoft BASICは、最初のApple IIモデルで使用され、機能が非常に限られていたInteger BASICシステムの代わりとしてMicrosoftによって作成されました。 Applesoft BASICは、Apple II +およびすべてのApple IIクローンから始まるすべてのApple IIモデルのROMにフラッシュされました。 Applesoft BASICの初期の切り捨てられたバージョンも、最初のApple IIモデルのカセットで利用できました。







CP / Mオペレーティングシステムの場合、BASIC-80方言が作成され、長い間言語の開発が決定されました。



最初のIBM PCパーソナルコンピューターが登場したとき、プログラミングの標準になったのは彼でしたが、すでにGW-Basicの形でした。



1980年代半ばに、BASICはプログラマブル電卓の複雑なモデルの主要言語になりました。この時点で、本格的な高級言語を使用できるようになりました。 BASICは、ソビエトMK-85電子計算機とその後のすべてのモデルに組み込まれました。







また1985年には、統合開発環境とBASICプログラミング言語コンパイラであるIDE Quick Basicが、1985年から1988年にかけてMicrosoftによって開発されました。 DOSおよびMac OS用のバージョンがありました。 使用されたBASIC方言は、高度なGW-BASIC方言に基づいていました。カスタムタイプのサポート、構造プログラミング構造の実装、グラフィックおよびディスク操作のサポートが大幅に拡張されました。



ロバート・「ボブ」・ザレは、長年の開発に基づいて1987年にベーシック/ Zがターボ・ベーシックを作成し、それをボーランドに売却しました。 Infoworld誌は、同年の最も重要な製品の1つに挙げました。







その期間のほとんどのBASIC実装とは異なり、Turbo BasicはMS-DOSのネイティブコードを生成する完全なコンパイラでした。



BASIC言語の膨大な数のバージョンにもかかわらず、新しいバージョンの言語を開発する場合、以前のバージョンとの互換性は常に維持されていました。この言語の後続のバージョンで以前のバージョン用に作成されたプログラム（場合によってはわずかな変更を加えた）で作業することができました。



セカンドライフ



90年代初期には、新しいグラフィカルユーザーインターフェイスを備えたMicrosoft Windowsオペレーティングシステムが登場しました。



シンプルなプログラムを作成するために、開発者は数ページのコードを書く必要がありました：メニューとウィンドウ、フォント、メモリのクリア、ボタンの描画などのためにコードのバルクブロックを登録する ただし、新しいインターフェイスの利点は非常に強力であり、Windowsはあらゆる場所で人気を博す真の可能性を秘めていました。



マイクロソフトは、何らかの形で開発者の生活を楽にする必要があることを理解していました。 そのため、同社はWindows用の新しいプログラミング言語と開発環境を作成するために、いくつかの並行プロジェクトを立ち上げました。







BASICは、MicrosoftがVisual Basicをリリースしたときに第二の人生を迎えました。 これは言語の進化に対する論理的な結論であり、通常のキーワードを保持しているにもかかわらず、構造と機能のセットは元のBASICよりもPascalに近かった。



1991年に、新しいMicrosoft Visual Basic IDEの最初のバージョンが登場しました。 ただし、その時点でマイクロソフトはこのシステムの機能を適度に評価し、それにもかかわらず、初心者および非専門のプログラマーのカテゴリに合わせました。



もちろん、Bill Gatesの会社は市場でのWindowsの地位を強化することを計画していたため、これは最初の一歩に過ぎませんでした。 また、これを行うのは非常に困難であり、商用開発者にMS Windows用のアプリケーションを作成する効果的な手段がありません。



Visual Basic開発者が作成したプログラミングシステムにより、Windowsの内部構造から抽象化することができました。 最も簡単なドラッグアンドドロップ操作を使用して、メニュー、ウィンドウ、リスト、ボタン、テキスト入力フィールド、およびWindowsインターフェイスの他の要素がプログラムに追加され、初期化と設定のコードのほとんどが自動的に生成されました。



VBプログラマは、この言語の学習を開始してからわずか数分で最初のプログラムを作成できます。 さらに、Visual Basicを使用すると、開発者は標準のGUI要素とともにプログラムでも使用できる新しいGUI要素を作成できます。



多くのC開発者は、Windowsプログラムを開発するためのシンプルで理解しやすいツールを作成しようとするMicrosoftの試みを真剣に受け取らなかったにもかかわらず、ITコミュニティの残りの人々はVisual Basicに大きな期待を持っていました。



1994年、同社はVisual Basic for Applicationsをリリースしました。 この時点で、VBAがMicrosoft Officeに組み込まれた後、BasicはWindowsの主要なプログラミング標準の1つになりました。



もともと玩具として考えられていたMicrosoftのVisual Basicは、プログラミングの世界を驚くほ​​ど早く征服しました。 その人気は、2つの理由によるものです。相対的な単純さと生産性です。



VBプログラムは、対応するC / C ++プログラムよりも低速でしたが、それでも多くのビジネス目的には十分に高速であり、開発時間も大幅に短縮されています。 フォームは、VBがWindowsプログラマーに提供する非常に労力を要する抽象化でした。



IDE VBでは、ボタンやリストなどのコントロールをツールバーからフォームにドラッグすることで、グラフィカルにウィンドウを設計できました。 フォームの外観が満足のいくものであるため、コード部分に移動し、各フォームコントロールのイベントハンドラーを記述することができました。



したがって、VBでのアプリケーション開発は、相互作用する複数のフォームを作成することで構成されていました。



コードを書くという観点から見ると、同じC ++との重要な違いは、ポインターの拒否です。 これが、Bill GatesがMicrosoft BASICの最初のバージョンの開発中に決定したことです。 ポインターは任意のアドレスのメモリに直接アクセスできるため便利ですが、不正確に使用するとエラーが発生します。 そして、この言語はもともと「単なる人間」のために作成されたので、この決定は正当化されたようです。



Visual Basicのメジャーバージョン



最初のバージョン（1991）で、Alan Cooperは言語とグラフィカルインターフェイス間の通信の原則を実装しました。



3番目のバージョン（1993）から、VBはMS Accessのサポートを追加しました。



Visual Basic 4.0（1995）は、32ビットと16ビットの両方のWindowsアプリケーションを作成できるバージョンです。 さらに、言語にクラスが追加されました。



VB 5.0（1997）から、独自のCOMコンポーネントを開発できるようになりました。 さらに、動的ライブラリMSVBVM50.DLLに実装された新しい仮想マシンが登場しました。 VBコンパイラは、変換にマルチパスモードを使用していましたが、最終的にはMicrosoft C ++コンパイラに依存して、中間言語へのコンパイル後に出力マシンコードを取得しました。 ランタイムライブラリとC ++バックエンドを使用することが、.NETテクノロジを構築するための出発点でした。



VBバージョン6はすでにCOMおよびMicrosoft Transaction Serverのコンポーネントを使用して分散アプリケーションを作成するために使用できる強力な言語です。 Microsoftは、クライアントサーバーアーキテクチャの3層アプローチを提案しました。このアプローチでは、シンユーザーインターフェイスがリモートVBコンポーネントと対話して、データベースまたは別のマシンからデータを取得します。



さらに、VB6（1998）を使用して、Internet Explorerで動作するActive-Xコントロールを作成できます。







2002年に、Visual Basic .NETの最初のリリースが登場しました。 その瞬間から、Visual Basicのクラシックバージョンとの下位互換性が侵害されました。



2005年以来、Visual BasicはMicrosoft Visual Studio 2005 IDEの一部でしたが、現在まで、この言語はIDEと一緒に開発されています（現在のバージョンまで）。MSVisual Studio 2015。





BasicとVisual Basicの言語は長い道のりを歩んでおり、取り返しのつかないほどの重要な変更が行われています。 元のシンプルさはほとんど残っていません。 その結果、VBは「通常の」高レベルのオブジェクト指向プログラミング言語になりました。
