著作権侵害の調査

多くのメディアでは、著作権侵害についてのみ耳にします。 海賊はここで盗みました、正直な市民はより少ない利益を得ました。 しかし、そうですか？

私はかつて思考を訪れました。 結局のところ、これはOpenSource全体であり、GPL、BSD、および他の多くの無料ライセンスが影響を受けています！ 彼らは利益を受け取りませんでした。 そして、隅々で利益の損失について叫ぶ人はcです。 いくつか例を挙げます。 マイクロソフトの「愛する人」から始めましょう。 この貧しい会社は何回製品の盗難から利益を受け取っていません。 やめて！ なぜシステムがそれほど普及しているのですか？ なぜそのソフトウェアは市場で支配的ですか？ 答えは簡単です。 最初は、すべてが海賊によって配布されました。 そして、著作権を保護する試みは言うまでもなく、偽造品を配布するために誰かを置き去りにすることは言うまでもありません。 フライホイールが回転します。 Windows 3.11、95、98、2000、そしてXPがユーザーのコンピューターに自由に侵入しました。 超強力なXP保護でさえ、Microsoft自身の努力で突然「壊れた」。 対応するアクティベーションパッチを含む「コーポレートバージョン」がリリースされ、非常に悪いハッカーがkeygenをネットワークにアップロードしました。 keygenは企業自体にルーツがあると確信しています。 Windows XPが市場を支配し始めました。 もちろんそうです。 誰もがそれを置くことができます。 Linuxはどうですか？ はい。人間向けのシステムが存在する場合、さらには無料でも、このLinuxには何が必要です。 そして、「コンソール」はありません。 大企業は著作権保護のために特に争っているわけではないという私の言葉を確認したとしても。 たとえば、たとえば、Perm Territoryの単純な校長のケースを誰もが覚えています。 事件は騒々しかった。 しかし、最初の声明から、Microsoftには取締役に対する苦情がないことが明らかになりました。 ここでは、マイクロソフトのポリシーを理解しています。 海賊のウィンドウズは学校に通っていますが、彼らはすべて将来の企業の超利益に行きます!!! 著作権で地獄に行くには、それを使って子供たちの世話をしてください！ もしあなたがマイクロソフトだったら、そうしませんか？

ソフトウェアの他の分野を取り上げましょう。 グラフィックス Photoshopを使用したAdobeの全機能を次に示します。 大部分の人々の写真の処理でさえ、「フォトショップ」です。 誰もgimpを言いません。 ちなみに、Gimpでは別の質問があります。 その開発者は、海賊に大きな打撃を受けました。 無料のPhotoshopは右と左に配布されましたが、彼らの製品は適切に主張されていませんでした。 それは実際には、（Photoshopユーザーの数に関連して）少数の愛好家グループを開発しました。 Gimpは組織、特に大規模な組織で需要があることを強く疑います。 そして、彼らのサポートなしでは、このプロジェクトは集中的に開発することはできません。それは、当時のPhotoshopがプライベート版のユーザー全体のおかげで開発したためです。 これらのクライアントは、「無料」のフォトショップを打ち負かしています。 現在、これらの組織がGimpに切り替えることは望ましくありませんが、州のポリシーのおかげで、私はphotoshopを購入しなければなりませんでした。 そして、何をすべきか、それに慣れています。 これが「標準」です。

AutoCADなどのその他の例では、torrents.ruドメインとMathCADドメインを安全に墓に入れることができます。 「無料」のため、すべての消費者の要求はこれらの製品に向けられたものであり、お金を稼ぐためのまったく異なるメカニズムを持つ代替のオープンソースには向けられていません。 そして、ユーザーも利益も開発もありません。

著作権を求めて闘っている組織のcに直面していると思います。 多くの場合、反対の状況がありますが、これは決して利益の不足ではありません。これは著作権侵害による超利益であり、裁判所の決定によりさらに大きな割合を占めています。 「無料」のWindows、Photoshop、AutoCAD、MathCADのユーザーですか？ 考えろ!!! 企業が高価な製品から開発し、裁判所の決定でそれらに支払う必要性をあなたに提出するのを助けるのはあなたです。 これらの高価な製品が必要かどうかを考えてください。 私はあなたにあなたの視点を変えるように強く勧めません。 製品は確かに良いですが、あなたは自分のポケットからそれらにお金を払っても構わないと思っていますか？ 雇用主ではなく、あなたと同じ企業にも支払いを行い、有料製品と同じ目標を経済的に追求するオープンソースプロジェクトをサポートしないのはなぜですか？
