開発とプロモーションの私の経験。 ゲーム「バルダ」

数日前、Android用のゲーム「Balda」の実装の次のリリースが行われました。 私にとってこれは小さなサクセスストーリーです。これは、これが私の最初のプロジェクトであり、その開発を完了してAndroidマーケットのユーザーに提供することができたからです。 この投稿では、アプリケーションの開発、トップへの進捗状況についてお話したいと思います。 特定の数字とグラフ-カットの下。



アプリについて



この特定のゲームをAndroid向けに作成するというアイデアは、Android SDKの知識と開発の開始から2年ほど前に思いつきました。 私はこの単語ゲームが好きでしたが、既存の実装-いいえ。

怠やその他の不可抗力のため、私は今年の秋までこのビジネスを延期しました。 ゲームの最初のバージョンは10月中旬にリリースされましたが、奇妙なことに、少数のユーザーから非常に温かく迎えられました。 3か月後、このプロジェクトは現在も稼働中です。現在、ロシア市場のカテゴリのトップ31であり、1万を超えるアクティブなインストールがあります。



アプリケーション作成



分析


もちろん、Android向けのこのようなゲームはすでに市場に出回っているので、競合するアプリケーションの分析から始めました。

当時、市場にはすでに3人がいました。

  • Dmitry Chistyakovの "Balda"-おそらく当時の最高のもの。
  • Wind Appsの「Balda」。
  • 「スメシャリキ。 バルダ」マーマードメディア。


最初はインストールの数から判断して最も人気がありましたが、放棄されたようです-最後の更新は2010年7月でした。

LG Gameスマートフォン専用に配布されているもう1つのバージョン、Word Game Baldaがあり、現在も存在しているため、市場での競争を表していません。

すでにそのようなゲームがあり、1つではないので、決定は明白でした-あなたは他のものよりもうまくやる必要があります。



開発



脳


単語検索アルゴリズムである「頭脳」の実装により、ゲームの開発を開始しました。 単語を検索するために、分枝限定法が使用されました-文字ごとに追加することにより、アルゴリズムはそのような部分文字列で始まる単語が辞書にあるかどうかをチェックしました。

完全な検索を記述してデスクトップマシンでテストした後、このコードをAndroidアプリケーションに転送しました。 そして、2.3 GHzの周波数のCore 2 Duoで100ミリ秒かかった単語検索が、600 MHzの周波数のARM5（テストHTC Wildfireで）で数十秒かかるようになったことがすぐに驚きました。

明らかに、ボトルネックは、プロファイラーによって確認された特定の部分文字列で始まる辞書内の単語の存在を確認することでした。

そのような出現を探しないために、私は単語をunningなマップの形で保存し始めました。ここで、キーは単語の始まりで、意味はすべての可能な終わりのリストです。 検索はずっと速く動作し始めましたが、そのような構造の初期化にはかなりの時間がかかり始めました-アプリケーションの読み込みには約40秒かかりました。 そのため、NDKを使用して全体を書き直すことにしました。 残念ながら、正確な数字を与えることはできません。なぜなら、 元のバージョンは保持されておらず、アルゴリズムには他の多くの変更と最適化が施されていますが、同じHTC WildfireのC実装は辞書を5秒未満で初期化し、約50ミリ秒以内に単語を見つけます。

これらの数は、市場の他の実装よりも優れており、Word Game Baldaの実装と同じでした。 また、検索アルゴリズムのネイティブ実装も使用しました。

そしてもちろん、ゲームにはキラー機能がありました-これは競技者が持っていなかった4〜7セルのカスタムサイズの競技場です。

広告、統計、エラー


広告を通じてアプリケーションを収益化することにしました。 AdMobを広告プロバイダーとして選択しました。 AdMob SDKの埋め込みは非常に簡単で、問題はありませんでした。

BugSenceを使用してエラーを収集しました。 これは非常に迅速かつ簡単に接続され、これは単なるかけがえのないものだと思います-100を超えるエラーから、意識の高いユーザーがAndroidマーケットにレポートを送信したのは約7つだけです。

Flurryを使用して使用状況の統計を収集しました-それが私に何の利益をもたらしたとは言えませんが、彼らの美しいカラーグラフィックを見るのは非常に興味深いです。

紹介リンクの追跡は、広告キャンペーンを実施する際にも役立ちます。 これを行うには、簡単なBroadcastReceiverを登録し、受信したリンクをその中に保存する必要があります。



AndroidManifest.xml 

  <receiver android:name=".ReferrerReceiver" android:exported="true" > <intent-filter> <action android:name="com.android.vending.INSTALL_REFERRER" /> </intent-filter> </receiver>




ReferrerReceiver.java 

 public class ReferrerReceiver extends BroadcastReceiver { private final static String TAG = ReferrerReceiver.class.getSimpleName(); private final static String REFERRER = "referrer"; @Override public void onReceive(Context context, Intent intent) { Bundle extras = intent.getExtras(); String referrerString = extras.getString(REFERRER); Settings.getInstance(context).setInstallReferrer(referrerString); Log.d(TAG, "Referrer is: " + referrerString); } }




その後、アプリケーションを開始したときに、対応するイベントをFlurryに記録しました。 これはその後、広告キャンペーン後に受け取ったインストール数を知るのに役立ちました。



プロモーション



ゲームは無料であることが判明したので、市場に加えて、いくつかのフォーラムにapkファイルを投稿しました。 ゲームのライフブランチの兆候によっては、w3bsit3-dns.comのみが提供されます。 ユーザーレビューとw3bsit3-dns.comのヒントのおかげで、ゲームはいくつかの機能を獲得し、いくつかの重大なバグが修正されました。 一般に、フォーラムでのレビューは、市場でのレビューよりもはるかに有益で有用でした。 ゲームの公開後、そのレビューはさまざまなブログやサイトに掲載され始め、ユーザーの一定の流入をもたらしました。

当初、Ubi Nuriの代表者から、韓国の厳しい現実に合わせてゲームを調整するようにというリクエストがありましたが、アプリケーションのユーザーの97％がロシア語を話し、そのようなゲームが韓国で普及する可能性は低いため、私は彼らに連絡しないことにしました。



アプリケーションを宣伝する上で最も重要な点は、市場でのゲームへの投票をお願いする際にダイアログを追加することです。 このダイアログを含むバージョンの公開後、5つが落ち、評価が上がり、アプリケーションがカテゴリのトップに上がり始めました。 これで、アプリケーションの評価は207、平均評価は4.8になり、非常に良好です。

次は？



今日、ユーザーの数は増え続けており、広告はいくらかの収入をもたらします。そして、ゲームに明るい未来があることを願っています。 ハブに関する長い話、インディーゲームUbi Nuriの出版権の売却を読んだので、私はこの方向に進むことも決めました。 このゲームはロシア市場でのみ人気があるため、ロシアの大手開発者とモバイルゲームのパブリッシャー（Alawar、Game Insight、Herocraft、Joybits）を選び、彼らに幸福の手紙を送りました。 残念ながら、私は彼らのいずれからも返事を受け取りませんでした、明らかに彼らは手紙を逃したか、単にそれらを無視しました。
