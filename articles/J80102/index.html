私の指から吸い込まれたこの重要なタスクを解決するために、プロジェクトでの作業を大幅に促進する簡単なレイアウトテンプレートを作成しました。 テンプレートは原始的でシンプルなため（呼び出される理由です）、フレームワークのタイトルのふりをしません（フレームワークと呼ばれる人にはカルマを差し引いてください！;））。 これには、メニュー、リスト、フォーム、基本タグのリセット（フォームのパディングのリセットや<p>の正しいパディングのような）を操作するための事前宣言されたCSSクラス、および私に役立つその他のグッズが含まれています。



最もせっかちな人のために、 Google Codeのプロジェクトへのリンクを提供します。最も興味がある人のために、Thick As Brickの構成は次のとおりです。

  • htmlファイルindex.htm

    このファイルには、スタイルとjavascriptへのリンク、空のメタタグ、javascriptの有無の確認が含まれています（最後の段落の詳細については、Vitaliy Kharisovによるブログ投稿を参照してください）。
  • cssファイルcss / common.css

    このファイルには、最小限のリセットセット（本文、img、フォーム、p、ヘッダー）とクラスの基本セット（.no-margin、.no-float、.pseudo-link、ul.news、ul.menu、ul.tiles、 table.reset、ul.resetなど）。 レイアウト設計者はこのファイルを変更せず、css / custom.cssを編集することを前提としています。これについては以下で説明します
  • cssファイルcss / common-ie6.css

    このファイルは、最も一般的なクラスと最愛のIE6との互換性を担当します。
  • cssファイルcss / custom.css

    ファイルには、特定のプロジェクトで作業するプロセスで変更できるスタイルが含まれています（元のcustom.cssは空で、コメントのみが含まれています;））。
  • cssファイルcss / custom-ie6.css

    このスタイルファイルは、IE6とのcss / custom.cssの互換性も提供します。
  • jsファイルjs / script.js

  • 単一ピクセルgif画像/ common / x.gif

    このファイルは、非常にまれなケースで役立ちます。
