0.0。 一般原則



まず、形式化のタスクと原則に同意することを提案します。

主なタスク： コメントを追加せずに、アイデアの本質を完全に反映して伝えることができるドキュメントを作成します。

言い換えれば、最終的な説明を読む人は誰もが議論されていることを明確に理解し、将来的にはアイデアの担い手との対話において、完全な知識を所有しなければなりません。

ここでは、ツールや実装の微妙な部分に飛び込むことなく、トップレベルのビジネスロジックに限定されていることに注意する必要があります。これは次のステップです。



作業中は、次の原則に従います。これにより、思考の流れを構築し、スムーズに進めることができます。

  1. 将来のドキュメントの読者は、プログラミングやゲーム開発の分野の専門家ではありません。
  2. 同時に、読者はバカではなく、Googleにアクセスできます。
  3. 文書は可能な限り明確で簡潔でなければなりません。 PowerPointプレゼンテーション形式に転送するには、テキストをスライドにコピーし、お好みに画像を追加するだけです。
  4. 最終ドキュメントは完全な説明の基本バージョンであり、個々の部分を書き換えたり構造を変更したりすることなく、自由に補足および拡張できます。 言い換えれば、空白を作成しますが、将来的には技術的な詳細と改良が必要になります。 ほとんどの場合、リリースする前にそれらのすべてを一般に公開する必要はありません。
  5. このドキュメントのコンパイラーは既に以下の記事に精通しており、読者はいつでもレビューできます。

1.1。 役職


私の経験が示すように、開発前に考えているのは10人に1人だけですが、最初の段階では開発環境でのプロジェクトの名前しか使用されていないため、原則として理解できます。 しかし、アイデアを真剣に考えれば、名前は非常に重要な役割を果たすでしょう。 最も明白なのは、プロジェクトを投資家またはチームに提示することです。

名前の選択に関する非常に役立つ記事は、 こことここで読むことができます 。 以下は、私たちにとって基本的に重要です。

0. 名前は、ゲームのコンポーネントの1つを反映する必要があります ：設定、ゲームプレイ、またはその他の機能。

ここでは、「SpacePuzzle」、「DragonRace」、「Merry Farm」など、このアイデアを絶対的に実現する例を挙げることができます。 一つの名前で、将来のプレイヤーは自分が何に対処しなければならないかを推測できます。



1. この名前でGoogleに表示されるものは何ですか？ 将来的に私たちがどうなるか、そしてこの名前で検索結果の最初の行を取る可能性は何ですか？

「Ants」、「WarLand」などの単純な名前 -他のプロジェクトの検索と広告の宣伝のための莫大な費用の誓約。



2. 名前に一致するドメインは利用可能ですか？ そうでない場合、-gameサフィックスを使用しても問題ありませんか？

前の段落の失敗した例を使用すると、次のロジックがあります。ants.ru-> ants-game.ru、warland.ru-> warland-game.ru

アリでそれがまだ行かなかった場合、戦場では状況はより悲しくなります。



3. 名前の発音は簡単ですか？ ユーザーはゲームを短縮形で何と呼びますか？

ここで、ロジックは単純です。名前が単純な場合は簡単に使用でき、複雑すぎる場合は、ユーザーが代わりのものを見つけます。CounterStrike-> Contra、WorldOfWarcraft-> WoWなど。



例：

私のプロジェクトでは、「 The AndLand 」という名前を選びました。 Googleは明確なものを何も提供しません。ドメインは無料で、TALまたは「Anda」と呼ばれることもあります。

1.3。 主な違い


原則として、この時点で多くはあきらめます。 あなたのアイデアが他の誰よりも優れていることをはっきりと言ってください-これは簡単なテストではありません。 さらに、現時点でわかりやすいものを作成できたとしても、「Analogs」ブロックがまだ先に待機しており、これはあなたのアイデアを再び批判します。

このゲームには、「X、Y、ZテクノロジーおよびBにアクセスするために建物Aを構築する」という古典的なツリーがありません;ゲームコミュニティ自体が技術とその開発の方向性を構築します。



1.4。 プラットフォーム


このセクションは現在、純粋に正式なものですが、将来的にはチームと投資家にとって重要になります。 チームの潜在的なメンバーにとって、これは重要です。スキルと開発ベクトルを評価する必要があるからです。モバイルデバイス用のコードを一生書いたときに、Webのプロジェクトに参加したいのですか。 また、投資家にとって、この項目は重要です。プラットフォームの開発の傾向をすばやく評価できるため、単純なリスクを確認できます。

現在、ほぼすべてのプラットフォームのプロジェクトを一度に作成することが一般的です。 あなたのものがそれだけである場合、いずれにせよ、1つ、最大でも2つあることを覚えておく必要があります-2つの起動プラットフォーム、そこからすべてが始まり、残りのために-それは後で移植されます。

プラットフォームのリストは、従来のPC / Macからゲームコンソールや携帯電話までさまざまです。特定のシステムでのみ一般的にどこでも実行できるブラウザーゲームを忘れないでください。 プラットフォームの選択に依存するいくつかの重要なポイント：

1.実装技術の一般的な輪。

2.配布方法;

3.ゲームの陳腐化の速度。

4.潜在的なプレーヤーの数。

例：

ゲームの最初の起動は、PCプラットフォーム/ Windows / Linux / Macで計画され、Webプラットフォームを介したプレイヤーのプロファイルの登録と管理が行われます。 将来的には、完全または限定的なゲームプレイを備えたモバイルアプリケーションのリリースが可能になります。



1.5。 対象読者


将来のイグロデロフの圧倒的多数を混乱させる別のセクション。 ここでの質問は非常に簡単です。「あなたのゲームは誰のためですか？ コミュニティの中心となるのは誰ですか？「最も一般的な答えは、「誰にとっても！」です。そして、それは最も間違っています。 マーケティング担当者は、すべての人を対象とした広告が誰にも当てはまらないことを知っています。製品にも同じことが言えます。 もちろん、「消費財」はありますが、そのリストは非常に小さく、全体としては重要ではありません。 このリストに該当しない他のすべては、特定のグループ用に作成されます。 メルセデス-成功、ロレックス-ショー、スタークラフト-上級者、ファームフレンジー-主婦、マインクラフト-学童



繰り返しますが、文字通り正式なオーディエンスをとる必要はありません。プレイヤーをこのポートレートのみに制限するわけではありませんが、プロモーションとゲームプレイでそれらに焦点を合わせます。 主婦とmatfakの優秀な学生のためのゲームは一般的に異なるゲームであると主張することはないと思いますか？ しかし、学生は時々カジュアルな何かをしたいです。 この理由のための十分な能力、人はここで退会しました 。



この意味での良い例は、MineCraftです。 開発者はプレーヤーをどこでも明確に選出しませんでしたが、ターゲットオーディエンスの中心は上級学生であることは明らかです。 この事実は、MineCraftでのMineCraftの作成、 マンハッタンの模倣、その他の重大な問題をキャンセルするものではありません。 しかし、自問してみてください。ほとんどのPC学校のゲーマーがこの時点でプレイしていなかったら、真面目な科学者はゲームで都市を作り始めるでしょうか？

例：

プロジェクトの対象読者：性別を問わず、オフィスワーカー、高校生、経済理論の基礎を理解している学生。
