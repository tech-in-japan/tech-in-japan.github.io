Vivaldi 1.1.453.6ビルドのタブを閉じるオプションとその他の改善

みなさんこんにちは！



Vivaldiブラウザの最初の最終バージョンがリリースされてからほぼ2週間が経過し、最近では膨大な数のレビューが寄せられました。 それらには多くの肯定的なものがありましたが、エラーメッセージに最も注意を払いました。 実際、私たちはこれまでずっと修正に取り組んできましたが、まもなく安定版の1.1番への更新を準備する予定です。この変更は現在のビルド番号で確認できます。 ただし、いつものように、エラーを修正しただけでなく、ブラウザの機能も拡張しました。



ブラウザのタブ設定に、タブを閉じることに関する新しいオプションがいくつか追加されました。 現在のタブが閉じられたときにアクティブにするタブを選択できます。次にリンクされ、次に設定で指定されたタブ切り替え順序に従って、また左または右のタブをアクティブにします。 いわば、すべての味のためです。



もう1つの改善点は、Opera 12からデータをインポートするプロセスに関するものです。ExpressパネルのリンクをVivaldiに転送する機能を追加しました。 インポートプロセスが完了すると、「Imported Speed Dial」フォルダがブックマークに表示され、まだ翻訳されていない同じ名前の別のExpressパネルがスタートページに追加されます。 残念ながら、エクスプレスパネルスケッチをインポートすることはできません。したがって、セルを更新して再度スケッチを作成する必要があります。



また、暗いインターフェイスの改善に引き続き取り組み、Vivaldiブラウザーウィンドウによるフォーカスのインターセプトに関するバグも修正しました。 いつものように、記事の最後に変更点の完全なリストがあります。



そして今、悲しいことについて。 ブラウザコアをバージョン50に更新しました。残念ながら、このバージョンのChromiumでは、Windows XP、Vista、およびMacOSXの10.9未満のサポートは無効になっています。 このため、これらのオペレーティングシステムでVivaldiをバージョン1.1にアップグレードする機能を無効にしました。 これらのOSのユーザーは、Vivaldi 1.0のみを使用できます。 同時に、NPAPIサポートも無効にします-Chromiumでは、これらのプラグインの操作が非常に不安定になりました。 これで、PPAPIプラグインとHTML5テクノロジーのみにすべての注意が払われます。 PAPPIフラッシュプラグインのインストールの詳細については、 こちらをご覧ください 。



それだけです 変更の完全なリストは記事の最後にあり、以下のリンクを使用して新しいアセンブリをダウンロードできます。



  • Windows 32ビット
  • Windows 64ビット （実験的）
  • Mac 10.9+
  • Linux RPM 32ビット
  • Linux RPM 64ビット
  • Linux DEB 32ビット
  • Linux DEB 64ビット
