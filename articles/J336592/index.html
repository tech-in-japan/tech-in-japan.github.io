Android OreoはPWAから一口





PWA（プログレッシブウェップアプリケーション）は、Webテクノロジー（JS + HTML + CSS）のスタックを使用するGoogleが開発および推進しているアプリケーションのグループの名前であり、Webサイトの使いやすさとネイティブUX固有のアプリケーションおよび技術的能力を組み合わせることができます。 -約 翻訳者。



8月21日、日食の後、GoogleはAndroid Oreoと呼ばれるAndroid 8の最終バージョンのリリースを発表しました。 同時に、プログレッシブWebアプリケーションには新しい問題があります。



Google PixelにAndroid 8をインストールし、WebサイトとプログレッシブWebアプリケーションに関するいくつかのことをテストした後、いくつかの悪いニュースがありました。







Android Instant Appsは、Android Oreo WebサイトのWebまたはブラウザーに関連するすべての言及です。 これは、Android Instant AppsがAndroidのPWAよりも重要であることが判明した最新のGoogle IOステートメントを繰り返しているようです。



しかし、良いことから始めましょう。 ソフトウェアの更新でほとんどの問題を解決できます 。 問題は、Androidの新しいバージョンがリリースされるまで誰もこれを処理しなかったことです。



ホーム画面への追加が機能しない



最悪の事態から始めましょう。OperaMobile、Samsung Internet、Firefoxでは、 「ホーム画面に追加」などの機能は動作しません 。 同様に、Firefoxは「スタンドアロン」表示モードをサポートしていませんが、メイン画面へのブックマークの追加をサポートしています。





これらのブラウザのホーム画面にPWAを追加するとどうなりますか？ そうです-何も。 間違いなし。 ブラウザは、すべてが正常であると考えています。 インストールを承認すると、Webアプリのバナーがすぐに表示されます。



最初の考えは、古いショートカットAPIが機能していなかったため、これらのブラウザーは新しいPinned Shortcuts APIにアップグレードする必要があるということでした。



Android 8.0に更新する前にショートカットをインストールした場合は、引き続き機能します。 問題は、メイン画面にショートカットを追加するプロセスにあるようです。



ホーム画面へのアイコンの追加はChromeでも引き続き機能しますが、他のブラウザーはGoogle Playでの更新を待つ必要があります。



Chromeの新しいインストールプロセス





この画面は、ショートカットの設定に同意した後に表示されます。



Chromeでは、ショートカットを配置する前に追加の手順があります。追加する方法を選択するように求められます。 「自動的に追加」オプション自体がアイコンの空の場所を選択するか、アイコンを押したまま任意の場所に移動できます。 問題は、この追加の新しい手順によりインストール速度がわずかに低下する可能性があることです。







新しいインストールプロセスは次のとおりです。



  1. メニューを開きます。
  2. ホーム画面に追加を選択します。
  3. タイトルを編集し、「追加」をクリックします。
  4. 「自動的に追加」をクリックするか、手動でアイコンを配置します。


Web Appバナーが自動的に表示される場合、最初の2つのステップは1つに置き換えられます-バナーの「ホーム画面に追加」をクリックします。



アイコンは期待したものではありません





ChromeアイコンはすべてのPWAアイコンに表示されます



Android 7.1、Chrome、Androidまでは、変換せずにマニフェストファイルに登録したアイコンをインストールしていました。



Android Oreoからは、Chrome（および場合によっては他のブラウザーですが、現在は動作しない）に追加されたすべてのショートカットに、アイコンの右下隅にChromeアイコンがあります。



PWAの場合、これは悪いことです。なぜなら、ユーザーにネイティブのアプリケーションと同様のアプリケーションを提供したいからです。 Chromeアイコンでは、ネイティブアプリケーションを使用していると感じることはできません。 そして、多くの企業はこれに満足していません。



アイコンに関する別の問題は、アイコンが透明な背景を持つ完全な円でない場合、正方形として扱われ、白い円の境界に合うように縮小されることです。 Android 7.1とAndroid 8の同じデバイスにインストールされている同じPWAアイコンを見ることができます。







前の画像、Lyft、Get Kana！ NASAのアイコンはこの革新に苦しみましたが、Garbarino、Flipkart、Aliexpressは元のバッジを保持しています。



まだ確認していませんが、PWAのアイコンはアプリショートカットアイコンのデザインガイドラインに従っているようです。



Android 8.0アップデートの前にインストールされたアイコンは、マニフェストファイルで指定したように見えます。 更新後に再インストールすると、変更が発生します。



WebAPK-私たちの救い



私たちがそれについてできることは何もありませんが、現時点でWebAPKが将来的に安定した場合、1つの解決策があるかもしれません。 WebAPKはGoogle Chromeの新機能で、Androidパッケージ（APK）用の独自のコンテナーを作成し、サーバー側でPWAに署名します。 現在、Chromeの「ホーム画面に追加する」のフラグで、デフォルトでカナリアで利用できます。



WebAPKを有効にすると、Android Oreoの前のようにアイコンが通常の状態に戻ります。 現時点では、WebAPKが完全に利用可能になる時期についての情報はありませんが、これらの問題により作業がスピードアップすると思います（そして期待しています）。



WebAPKはChromeの問題のみを修正します。 Googleがサードパーティのブラウザーに対してWebAPKを開かない場合、サムスンインターネット、Opera、Firefox、およびその他のブラウザーはアイコンに関して同じ問題を抱えます。



VPNでパブリックWi-Fiアクセスポイントを今すぐ開く



Androidは、オープンアクセスポイント（セキュリティなし）に自動的に接続する機能をサポートするようになり、セキュリティを強化するために、Google VPNを安全なトンネルとして使用します。 これは、ユーザーが米国のマウンテンビューからのようにサーバーに表示されることを意味します。 送信元IPアドレスはHTTPヘッダーで使用できないため、プロキシの場合と同様に、GoogleはX-Forwarded-Forヘッダーを使用しません。







Android 8.0の長所はPWAでは使用できません



Android Oreoには、PWAには役立つかもしれないが、Webにはまだ利用できないものがいくつか追加されています。



  • 通知ポイント：アプリケーションに保留中の通知がある場合、アイコンにドットが表示されます。
  • レスポンシブアイコン：デバイスの異なるモデルでさまざまなフォームを表示できます。 たとえば、レスポンシブアイコンは1つのOEMデバイスでは円形で、別のデバイスでは正方形として表示される場合があります。
  • 通知チャネル：これにより、アプリケーション/ウェブサイトから受信する通知のカテゴリを決定できます。
  • ショートカット：アイコンを長押しすると表示されるプロパティ。


良いニュースはありますか？



ここで説明したすべてを確認できます。現在Android 8デバイスをお持ちでない場合でも、Androidエミュレーター（Android Studioで利用可能）にはGoogle Chrome（バージョン58）が含まれています。







ビデオを再生するための「Picture-in-Picture」



サイトに「ビデオ」タグがある場合、Chromeは「ピクチャーインピクチャー」モードをサポートするようになりました。 これに正しいUXがあるかどうかはわかりません。ビデオを全画面モードで開き、ホーム画面を押す必要があります。 この時点で、ビデオは小さなフローティングウィンドウとして再生され、移動したり閉じたりできます。







GoogleはPWAをもっと活用すべき







通常、AppleはWebプラットフォームのサポートが不十分であると批判していますが、Androidを含む他のプラットフォームのPWAはまだ未加工です。 そして、私はそれを言ってすみません。



前回のGoogle I / Oで似たようなものをツイートしましたが、GoogleはPWAを気にしていないと感じています。 ChromeではなくGoogleです。 変なの？ はい



Chromeチームのメンバーは素晴らしい仕事をしていることは承知していますが、いくつかの作業（WebAPKなど）には時間がかかりすぎます。



AndroidチームはPWAをあまり気にしていないようです。インスタントアプリのサポートと、今日のリリースのこの投稿で説明されている問題は、私が言うには十分であり、これはPWAに悪影響を与えます。 。



多くの人は、PWAはWebアプリマニフェストやアイコンのインストールよりも多くのサービスワーカーであると言うでしょう。 良さそうに聞こえますが、1つのSWだけがPWAのアイデアを企業に販売しているとは思いません。
