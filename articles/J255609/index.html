この記事では、 クラウドベースのVPSと従来のVPSの違いについて簡単に説明します。 これは、使用するものを決定するのに役立ちます。 読んだ後に質問がある場合は、コメントで喜んでお答えします。 ハブレにコメントを残せない場合は、 コミュニティで質問することができます。







クラウドと従来のVPSの違い



特定の期間（たとえば、1か月または1年）にVPSを購入し、クラウドでは、消費量に応じてサーバーが使用するリソースに対して支払います。 クラウドVPSは、スケーリングと最も高度なテクノロジープラットフォームにより、単純なタスクと最も要求の厳しいタスクの両方を等しく解決できる新世代の仮想サーバーです。



クラウドVPSの利点


  • 必要な数のサーバーを必要な構成でいつでも作成できる機能
  • CPUの速度、必要に応じてメモリとディスクの量を変更する機能（多くはサーバーを再起動しなくても変更できます）
  • 自動スケーリングのサポート。負荷に応じてクラウドサーバーの利用可能なリソースを変更できます。
  • コントロールパネルの残高から割り当てられたリソースに応じて、1時間ごとの支払い（登録時に月額料金は発生せず、1回限りの残高の再請求が必要です）
  • サーバーが停止すると、CPUおよびRAMリソースは解放されるため、CPUおよびRAMリソースの支払いは請求されません。
  • フォールトトレラントトリプルレプリケーションプラットフォーム：優れたデータストレージの信頼性。 データは、異なる物理サーバー上の3つのコピーに保存されます。
  • エンタープライズSSDキャッシング：PCI上の最高パフォーマンスのディスクサブシステム– Express SSDおよびDataCenter SSD。 （忙しいデータベースで作業する場合は特に重要です）
  • 組み込みのHTTPロードバランサー
  • 統合ファイアウォール
  • ハードウェア仮想化のサポート：必要に応じてオペレーティングシステムのカーネルを制御する機能
  • サーバーテンプレートを作成する機能：サーバーを構成したら、いつでもそのテンプレートを作成し、そこから新しいサーバーを作成できます。
  • 市場の企業部門での使用に最適化：企業での使用の最も重要なシナリオで既にクラウドをテストし、企業ユーザーに推奨しています。
  • REST APIサポート：クラウド管理サーバーにリクエストを送信することにより、プログラムでクラウドを管理できます
  • 最大1 Gb / sの高速外部ネットワーク、データセンター内の無制限のネットワーク
  • ギフトとして月に3 TBの外部トラフィック


Classic VPSの利点


  • 市場で最低価格の1つ。
  • 毎月/年間の固定支払い
  • VPSは、単純なプロジェクトやサイトに最適です。


クラウドVPSおよびクラシックVPSのpingを確認する方法


クラウドおよび従来のVPSへの応答時間をテストできます。

  • ping msk.sandbox.infoboxcloud.ru-モスクワのクラウドサーバー
  • ping ams.sandbox.infoboxcloud.ru-アムステルダムのクラウドサーバー
  • ping spb.sandbox.infobox.ru-サンクトペテルブルクのLinux VPS
  • ping ams.sandbox.infobox.ru-アムステルダムのLinux VPS
  • ping krs.sandbox.infobox.ru-クラスノヤルスクのLinux VPS


いくらですか？


クラシックVPSの料金は、 公式ページで入手できます 。 クラウドベースのVPSの場合は、 公式ページで自動スケーリングを使用せずにおおよそのコストを確認できます（自動スケーリングは大幅に節約できます）。 関税は、「関税と資源の計算」のセクションに完全に示されています。 また、複数のクラウドサーバーの概算コストを一度に計算できる実験的な計算機を使用することもできます。





クラウドVPSのチェックは実際には予測可能です。 サーバーが必要とするリソースの量をコントロールパネルで設定します。 自動スケーリングを使用する場合でも、OTおよびTO制限を設定します。



おわりに


目標が最大の節約である場合、プロジェクトは成長しません。必要なリソースの数が事前にわかっている場合は、Infoboxの従来のVPSを使用します。 これは、プロジェクトで最も使いやすく実績のあるテクノロジーです。



最大のパフォーマンス（特にディスクサブシステム）、最大のフォールトトレランス、またはサービスを使用して企業リソースをホストする予定がある場合は、Infobox クラウドVPSを使用する必要があります。 クラウドは、負荷が少ないときにサーバーが使用できるリソースの量を減らすことで、費用を節約できます。 一方、クラウドでは、必要な量のリソース（CPU、RAM、ディスク）を選択してサーバーに追加するだけなので、プロジェクトは増加する負荷に耐えることができます。



それでも何を選択すればよいかわからない場合は、Infobox Cloud VPSを使用してください。
