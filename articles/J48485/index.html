しかし、事実、Firefoxでは、すべての利点を備えているため、デフォルト設定により、常にユーザーがゲストとして感じられます。 ここでのインターフェイス、つまりブラウザの動作については説明していません。 これを変更しようとしましたが、ここに私が取った手順があります：

クイックナビゲーション



一番気になったのは、前方/後方にナビゲートするときのOperaとFirefoxの動作です-タブの履歴をどれだけ深くナビゲートしても、Operaは即座にすべてを表示します。 Firefoxは、IEに夢中になっているかのように、何らかの理由でほとんどのページをインターネットから再びダウンロードするため、すぐにそれについて話すことはできません。 さらに、Operaでフォームに入力し、Backコマンドを使用してフォームに戻った場合、入力したすべてのフィールド（パスワードを除く）は入力されたままになりますが、 Firefoxはページデータをリロードし、失われます。

ブラウザは、戻ったときにページをリロードせず、以前に見たものを正確に表示したかったのです。 正直に言うと、Operaはその動作でRFCに違反しています（ RFC 2616のセクション14.9.2 ）。 たとえば、ページに「ログイン」してから離れた場合、「戻る」をクリックしてその内容を表示できます。 これはセキュリティ侵害と見なすことができます。インターネットカフェにあるコンピューターで、メールを見た後に別のユーザーがクリックバックを開始したとします。



しかし、私たちは経験豊富なユーザーであり、私たちがやっていることを知っているので、行きました：



Fastbackをオンにします。



browser.sessionhistory.max_total_viewers

Fastback設定はここで設定されます-これは、Forward / Backコマンドによるクイックナビゲーションのためにメモリに保存されるページ数です。

デフォルトは-1です。 ブラウザ自体は、RAMに応じて保存するページ数を決定します。

4 GBのRAMがあります-このパラメーターを15に設定します-メモリーを気にしません。

このパラメーターは、タブではなくブラウザー全般のページ数を設定することに注意してください。



キャッシュをカスタマイズする



browser.cache.check_doc_frequency

デフォルトは3、つまり Last-ModifiedおよびControl-Cacheヘッダーに基づいてキャッシュを更新します。 値を0に変更すると、Firefoxは新しいブラウザーが起動するたびにページキャッシュを更新します。



Operaは通常、前方/後方に移動する際にキャッシュ設定を無視します。ページを更新するには、RefreshまたはReloadを実行する必要があります。 Firefoxの動作はより安定しています。つまり、次のとおりです。

  • Cache-Control：no-cache Firefoxはページをディスクに保存しませんが、メモリキャッシュに保存します。 たとえば、[戻る]コマンドを使用すると、キャッシュのページが表示されます。
  • Cache-Control：no-storeページはメモリにもディスクにも保存されず、表示しようとすると常に完全なPOST / GETリクエストが実行されます


ここからの情報：

restlet.tigris.org/ds/viewMessage.do?dsForumId=4447&dsMessageId=981443

blog.httpwatch.com/2008/10/15/two-important-differences-between-firefox-and-ie-caching





これはすべて素晴らしいことですが、多くのサイトはCache-Control：no-storeを悪用し、その結果、ユーザーの時間とトラフィックが無駄になります。 残念ながら、上記のオプション（browser.cache.check_doc_frequency）はブラウザーのこの動作を無効にしないため、Operaの動作に完全に準拠する必要がある場合は、Proxomitronなどのヘッダーを変更するソフトウェアを使用する必要があります



キャッシュのメモリ容量を増やします。



browser.cache.memory.capacity

現在の使用状況は次のとおりです：about：cache？Device = memory

デフォルトで30MBにインストールしましたが、96に増やしましたが、変更はありませんでした。



閉じたタブのリスト：



browser.sessionstore.max_tabs_undo

Operaでは、閉じたページの長いリストがあるため、タブを閉じるのに何も考えずに慣れています。

理由は明らかではありませんが、Firefoxではデフォルトで5に制限されており、30に変更されています。



並列HTTP 1.1リクエスト



network.http.pipelining

つまり、このオプションを使用すると、サーバーとの同じ接続で、複数の要求を同時にではなく並行して（前の要求への応答を待機しながら） 同時に送信できます。 詳細： www.mozilla.org/projects/netlib/http/pipelining-faq.html

Operaはデフォルトでパイプラインを有効にしているので、有効にします。 並列リクエストが機能するためには、Keep-Alive（ network.http.keep-alive = true）およびHTTP 1.1（ network.http.version = 1.1）が必要です。



並列クエリの最大数



network.http.pipelining.maxrequests

リクエストの最大数を30に設定することを推奨しているマニュアルもあります。これはお勧めしません。HTTP/ 1.1仕様ではリクエストの数は制限されませんが、一般的にキープアライブには2を推奨します。 デフォルト値の4のままにしました。

UPD： Alaunquirieが明らかにしたように、Firefoxでは、一般的にこのパラメーターの内部制限は8です。



サーバー接続の最大数



network.http.max-connections-per-server

ここであまりにも多くの数値を設定すると、一部のサーバーはIPからのリクエストをDDOS攻撃と同じくらい多く判断し、リクエストは順番に処理されます。最悪の場合、IPは禁止され、再びアクセスが許可されるまで待つ必要があります。 。

FF 3ではデフォルトは15ですが、8に設定しました（Operaをセットアップするためのサイトでこの値を正確に満たしていました）。



サーバーへの永続的な（開いている）接続の最大数



Network.http.max-persistent-connections-per-server

上記で書いたように、 RFC2616は最大2つのキープアライブリクエストを推奨しています。 Firefox 3では、デフォルトは6なので、そうです。



接続の最大数：



network.http.max-connections

Fire Fox Foxの最大同時接続数を制限します。 高速のインターネットがあり、バックグラウンドでタブを開きたい場合は安全に上げることができます。タブはメインページを見続けている間に読み込まれます。 私はそれを16MB / 64のインターネットでインストールしています。



開始ページのレンダリングの遅延



nglayout.initialpaint.delay

OperaでBrowsing-> Redraw Instantlyのようなパラメーターを含めた場合、Firefoxでこれを行うのは理にかなっています。 このパラメーターは、最初の情報を受信して​​からレンダリングを開始するまでの時間を決定します。 高速チャネルでは理にかなっています。 欠点は、ページがより頻繁に再描画される可能性があることです。

デフォルト値は250msです。0に設定して満足しています。



少しだけ残った：



Operaで「」をクリックしてクイック検索をするのに慣れています。 「または」 / 「および」 、 「URLで検索します。

Firefoxでは、これはF3またはCtrl-Fに存在しますが、検索後にパネルが消えないことに悩まされました。さらに、検索を再度呼び出したときに古い値があったため、消去する必要がありました。

グーグル、私は怠tooすぎることがわかりました-' / 'を押すと、ページ上の検索はオペラとまったく同じように動作します（アポストロフィ- 'はリンクを検索します）。

通常のカンマとピリオドを設定しようとしましたが、驚いたことに、Firefoxではキーボードショートカットの設定を変更できないことに気付きました！

しかし、これはFF なので 、ショートカットを編集する機能を追加するkeyconfigアドオンを見つけました。 残念ながら、QuickFindのコマンドはありません。自分で作成する必要があります。keyconfigの「Add a new key」をクリックして追加します。

  • 通常のQuickFindの場合：

    gFindBar.startFind（gFindBar.FIND_TYPEAHEAD）;

    gFindBar._setFindCloseTimeout（）;

  • リンクのQuickFindの場合：

    gFindBar.startFind（gFindBar.FIND_LINKS）;

    gFindBar._setFindCloseTimeout（）;



そして、使い慣れたキーを割り当てます。



私はまだ他に何を変更すべきかを理解していませんが、このように構成されたFirefoxを使用することを楽しんでいます。
