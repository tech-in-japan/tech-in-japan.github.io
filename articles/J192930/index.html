USBモデムhuawei e1550を介した音声通話（通知）を生成するperlスクリプト

かつては出張に多くの時間を費やしていたので、すばらしいおもちゃ、Huawei e1550 usbモデムを購入しました。 しかし、da烈な若者の時代は過ぎ去り、意図された目的のためにこのデバイスを使用する必要はなくなりました。 それで彼は数年の間私の棚にちりばめた。 そして、それはさらに塵を払っていたでしょうが、タスクは警告システムを作るために起こりました。 それから、モデムについて思い出しました。 タスクを検討すると、SMSの読み取りに関する通知を受信できないため、音声ダイヤルを優先してSMS通知を拒否せざるを得ませんでした。 アスタリスクベースのソリューションは私には少し面倒に思えたので、モデムのドックを読んだ後、自分でダイヤラーを書くことにしました。



出版の理由。


USSDおよびSMSリクエストの処理に関する豊富な記事があるにもかかわらず、スクリプト言語（Perl、PHP、Node.jsなど）での音声呼び出しの実装は見つかりませんでした。 この記事が皆様のお役に立てば幸いです。

ちょっとした理論。


ほとんどのLinuxディストリビューションでは、このモデムを/ devに接続すると、3つのusbインターフェイスが作成されます。 これは通常：

/ dev / ttyUSB0-モデムコマンドラインインターフェイス

/ dev / ttyUSB1-音声（音声モードがオンの場合）モデムインターフェイス

/ dev / ttyUSB2-モデムコマンドラインインターフェイス。 / dev / ttyUSB0とは異なり、コマンドに対するモデムの応答だけでなく、サービスメッセージも読み取ることができます。 信号品質データ、^ CEND出力など。



モデムの操作を開始するには、ファイルとして読み書きするためのコマンドインターフェイスの1つを開くだけです。

モデムにコマンドを送信するには、開いているインターフェイスファイルにコマンドを書き込む必要があります。

このコマンドに対するモデムの応答を取得するには、開いているインターフェイスファイルから読み取る必要があります。



モデムに与えることができるコマンドはATコマンドです

Huawei e1550モデムのコマンドとそれらで取得できる回答は、仕様に記載されています。

HUAWEI CDMAデータカードモデムATコマンドインターフェイス仕様

HUAWEI UMTSデータカードモデムATコマンドインターフェイス仕様



モデムの音声機能を有効にするには、コマンドAT ^ CVOICE = 0を発行する必要があります

音声機能は、コマンドAT ^ CVOICE = 1によって無効になるまで有効になります



オーディオ情報のモデムへの送受信を開始するには、各呼び出しでAT ^ DDSETEX = 2コマンドを使用してモデムオーディオポートのモードを切り替える必要があります。



モデムに送信するための音声データは次の形式である必要があります

デジタル化周波数：8000ヘルツ。

チャンネル数：1（モノ）。

デジタル化ビット：符号付き16。



オーディオデータは、0.02秒ごとに320バイトの部分でモデムのオーディオポートに入力する必要があります。



通話の終わりに、モデムは2番目のコマンドインターフェイスを介して、CENDメッセージの形式で通話に関する情報を提供します。

^ CEND出力形式：call_index、duration、end_status、cc_cause

ここで：

call_index-一意の呼び出し識別子

duration-秒単位の通話時間

end_status-通話終了後のデバイスステータスコード

cc_cause-コール終了理由コード
