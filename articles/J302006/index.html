CADリリースの新しい候補Qucs-0.0.19S-RC6





Qucsは、オープンソースのクロスプラットフォーム（Linux、Windows、MacOS-X）電子回路シミュレータです。 彼について、Habréに関する私の以前の記事を教えてください：



  • Qucs-電子回路シミュレーション用のオープンソースCAD habrahabr.ru/post/248005
  • Qucsプロジェクトニュース：リリース0.0.19の準備habrahabr.ru/post/259193
  • Qucsプロジェクトニュース：SPICEの回路モデリングをサポートするリリースの候補が利用可能ですhabrahabr.ru/post/263555




現在、Qucsの2つのバージョンが並行してリリースされる準備中です。

  • Qucs-通常の機能セットを持つアセンブリ。 Qucsatorシミュレーションエンジンのみが使用されます。
  • QucsS-SPICE（ Ngspice 、 XYCE 、 SpiceOpusエンジンがサポートされています）をデフォルトのモデリングエンジンとして使用する機能を備えたアセンブリ。 アセンブリデータには、バージョン番号の後に文字「S」が含まれます。 エンジニアにとって最大の関心事はNgspiceです。




Qucsの次のリリースは再び無期限に延期されますが、SPICEをサポートするQucs-0.0.19S-RC6リリースの候補がリリースされました。 このリリース候補は、以前のリリース候補とは大きく異なります。 2つのプラットフォーム（WindowsおよびLinux）のパッケージをここからダウンロードします： github.com/ra3xdh/qucs/releases/tag/0.0.19S-rc6



カットの下で、このリリース候補の革新について通知されます。







設置





Linux





Linuxのインストール手順は変更されていません。 ソースコードからパッケージを収集する必要があります。 開発者にはコンパイラとQt4が必要です。 QucsとQucsatorシミュレーションエンジンを個別に構築する必要があります。 

tar xvfz qucs-0.0.19S-rc6.tar.gz cd qucs-0.0.19S-rc6 cd qucs ./configure make make install cd ../qucs-core ./configure make make install




この場合、すべてが/ usr / localにインストールされ、以前のバージョンのQucsが既にそこにインストールされている場合、上書きされます。 QucsSを別のディレクトリにインストールするには、configureコマンドを変更する必要があります。 

 ./configure --prefix=/some_qucs_location/




SPICEのみ​​が必要な場合は、Qucsインターフェイスのみを構築できます。 

 tar xvfz qucs-0.0.19S-rc6.tar.gz cd qucs-0.0.19S-rc6/qucs ./configure --prefix=/some-qucs-location/ make make install




Ngspiceは、パッケージマネージャーを使用してインストールする必要があります。 それはすべての最新のディストリビューションにあります。



QucsSは最初の起動時に、デフォルトのシミュレーターを指定するように求めます。



窓





Windowsの場合、バイナリインストーラーをダウンロードする必要があります。 QucsSとともに、特別なNgspiceアセンブリを使用することをお勧めします。これはリリースページからもダウンロードできます。 プロジェクトが動作する公式サイトからダウンロードされたNgspiceアセンブリは、次のようにログを書き込むことを許可しません。 無効でないGUIがあります。



新機能Qucs-0.0.19S-RC6の概要





バグの修正とヘッドプロジェクトのコードベースとの同期に加えて、QucsSには多くの非常に重要な新機能が含まれています。



デフォルトのシミュレーターを設定する





SPICEシミュレーターを使用してシミュレーションを開始する手順が変更されました。 これで、ユーザーがシミュレーションを呼び出すたびに実行されるデフォルトのシミュレーターを割り当てることができます（たとえば、F2を押して）。 SPICEでシミュレートする特別なメニュー項目を使用する必要はなくなりました。 SPICEエンジンの1つが選択されている場合、プログラムが機能するためにQucsatorエンジンと完全インストールは不要になります。



デフォルトのシミュレーターは、プログラムの最初の起動時に割り当てるか、メインメニューで「 シミュレーション」->「デフォルトのシミュレーターを選択」を選択して割り当てることができます。 SPICEエンジンの1つが選択されている場合、互換性のないコンポーネントとライブラリは表示されません。 デフォルトのシミュレーターのインストールダイアログは次のようになります。







ウィンドウ上部のドロップダウンリストでデフォルトのシミュレーターを選択する必要があります。 Windowsユーザーは、シミュレーター実行可能ファイルへの正しいパスも指定する必要があります。

SPICEエンジンの（F8を押して）直流の動作点の計算を実装しました。 Ngspiceがデフォルトのシミュレーターとして割り当てられている場合、動作点が計算されます。 計算結果は次のようになります。







新しいXPSPICEアナログブロックセット





XSPICEコンポーネントは、特別なXanalogueライブラリに移動されます。 XSPICEでは、ブロックレベルで回路をモデル化するコンポーネントを使用できます。 新しいライブラリには、アンプ、加算器、乗算器、除算器、積分器、リミッターなどのモデルが含まれています。 それらは次のようになります。







SPICEディレクティブ.MODELおよび.INCLUDE





SPICE互換の半導体コンポーネントのセットに加えて、回路に.MODELおよび.INCLUDEディレクティブを配置する機能が追加され、回路内の半導体コンポーネントの変更されていないライブラリの使用が可能になります。 .INCLUDEディレクティブを使用すると、ライブラリを参照できます。.MODELディレクティブを使用すると、SPICEモデルを1行ずつコピーして、回路に埋め込むことができます。 電界効果トランジスタミキサー回路は、この指令の使用法を示しています。







トランスモデルとコアモデル





強磁性コアを備えたトランスとコイルをシミュレートするコンポーネントを追加しました。 トランスフォーマーを含むトランスフォーマーライブラリと、コアモデル（主にスチールコア）を含むコアライブラリがあります。 この機能は、Ngspiceを介してのみ使用できます。 この図は、VLFチューブ（6P6Sランプ）のスキームを示しています。これは、トランスの新しいライブラリモデルとSPICEモデルの使用を示しています。







Icouple（巻線）コンポーネントとCore（コア）コンポーネントの組み合わせを使用して、独自のトランスフォーマーを作成できます。 Kカップリングコンポーネントを使用して理想的なトランスを作成できます







カスタムSPICEコンポーネントの作成





2つのコンポーネント「SPICE汎用デバイス」と「XSPICE汎用デバイス」が追加されました。 SPICEで割り当てられているピンの数と文字のみを知って、新しい非標準コンポーネントを作成できます。 これは、コンポーネントが既にエンジンに追加されており、グラフィカルインターフェイスが遅い場合に便利です。 これは、各リリースで新しいコンポーネントが追加されるXYCEシミュレーターに特に当てはまります。 添付の図では、通常の電界効果トランジスタが非標準コンポーネントとして宣言されています。 電界効果トランジスタモデルは、.MODELディレクティブを使用して接続されます。







XSPICE CodeModelのサポート





XSPICE CodeModelアナログモデル記述言語（1991年以来知られている）のサポートが追加されました。 新しいモデルを作成し、再コンパイルせずにそれらをNgspiceモデリングエンジンに接続できます。 NgspiceとXSPICEのマニュアルでCodeModelモデルの構文の詳細を読むことができます。 「XSPICE汎用デバイス」コンポーネント（UGOコンポーネント）と「XSPICE CodeModel」（モデルソースコード）の組み合わせを使用して、CodeModelモデルを接続できます。 図では、そのようなモデルの使用例を見ることができます。







CodeModelモデルは、cfunc.mod（モデルの実装）とifspec.ifs（インターフェイスの説明）のペアのファイルで構成されています。 アンプモデルのソースコード（cfunc.modファイル）CodeModelは次のとおりです。 



 void cm_gain(ARGS) Mif_Complex_t ac_gain; if(ANALYSIS != MIF_AC) { OUTPUT(out) = PARAM(out_offset) + PARAM(gain) * ( INPUT(in) + PARAM(in_offset)); PARTIAL(out,in) = PARAM(gain); } else { ac_gain.real = PARAM(gain); ac_gain.imag= 0.0; AC_GAIN(out,in) = ac_gain; } }




変更されていないライブラリをSPICEモデルに接続する





特別なコンポーネント「SPICEライブラリデバイス」が追加されました。これにより、変更および互換性レイヤーなしでSPICEライブラリを使用できます。 コンポーネントで使用可能な文字パターンのいずれかを使用できます。 これまでのところ、3または5ピンのオペアンプ用のテンプレートのみが利用可能です。 ライブラリへのパス、コンポーネントの名前、シンボルテンプレート、および必要に応じてコンポーネントのパラメーターを示すだけで十分です。 この図は、この方法でオペアンプを接続する方法を示しています。 このプロセスを自動化する予定です。 将来、SPICEライブラリはネイティブQucsライブラリとともにライブラリマネージャに表示され、それらのコンポーネントはスキーマに挿入できるようになります。 PCAD Library Executiveと同様に、シンボルとシンボルのライブラリを追加することも計画されています。
