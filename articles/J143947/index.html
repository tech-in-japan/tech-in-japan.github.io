マーケットプレイスの数

1C-Bitrix：MarketPlaceストアでのソリューションの販売数はすでに100近くに達しているため、経験を共有したいと考えています。 当社の製品の例と、他のスタジオが開発したモジュールの両方についてお話します。



MarketPlace-1C-Bitrix用に開発されたソリューション（モジュール、コンポーネント、テンプレート、サンプルサイト）を販売できるプラットフォーム。 MarketPlace Webサイトでソリューションの支払いを行う顧客の能力は、昨年8月に現れました。この時点から、サイトは積極的な開発を開始しました。 したがって、1年も経っていませんが、それでも統計を公開し、経験を共有します。



合計で10のソリューションをリリースしましたが、そのうち6つが有料で、4つが無料でダウンロードして使用できます。







価格の設定方法は？



価格は、クライアントがソリューションの購入を決定できるようにする決定的な要因の1つです。 優れたソリューション価格とは、フリーランサー/企業向けのソリューション機能の開発コストよりもわずかに低い価格です。 価格は、クライアントが開発を注文するよりもソリューションを購入する方が収益性が高くなるようなものでなければなりません。 5000ルーブルを超えるコストのモジュールの売上は、総売上のわずか14％に過ぎません。



逆をしないでください-あまりにも安い価格を設定してください。 100〜200ルーブルのソリューションを販売することはほとんど意味がありません。 更新、顧客の質問への回答、およびテクニカルサポートにより多くの費用がかかります。 はい、そしてそのような決定が50,000回購入され、あなたが莫大な利益を得ることを期待しないでください。 現在、Bitrixは非常に一般的ですが、MarketPlaceは依然としてAppStoreではなく、ソリューションを販売する市場は限られています。

5000ルーブル以上のソリューションはありません。そのため、価格グループ「最大1000ルーブル」、「1000〜2000ルーブル」、「2000ルーブル」に決定を条件付きで分割しました。 これは私達の販売が配られた方法です：







ただし、「2000ルーブルから」というグループの決定は1つだけであり、最近出されたことは注目に値します。



これらのグラフは何について話しているのですか？ もちろん、安価なソリューションが最も売れていることがわかりますが、「1000から4999ルーブル」グループのソリューション（または「1000から1999ルーブル」のグループ）のソリューションも非常によく売れています。 たとえば、1990ルーブルのモジュールの1つの販売。 それは499ルーブルのモジュールの4つの販売と同じくらいのお金をもたらします。 そして、「最大999摩擦」グループのモジュールがより頻繁に販売されるという事実にもかかわらず、より高価なモジュールはより多くの収入をもたらします（そのコストのコストも考慮に入れます）。



MarketPlace Denis Donchenkoのヘッドから親切に提供されたデータによると、最も頻繁に購入されるソリューションは500ルーブルの価値があります。 販売されたソリューションの平均価格（顧客MPが消費した金額の総販売数に対する比率）は1200ルーブルです。



アフィリエイト割引を与えるべきですか？



アフィリエイトプログラムMarketPlaceでは、ソリューションにアフィリエイト割引を設定できます。 これにより、Bitrixパートナーは40〜50％の割引でソリューションを購入できるようになります（パートナーのステータスに応じて）。 これは非常に多いようですが、パートナーは割引を与える必要があります。 理由は次のとおりです。







ご覧のとおり、すべての販売の49％がパートナーによるものです。 そして、これらの売上の大部分は最終的に顧客によって支払われました（パートナーはクライアントにモジュールを販売し、このモジュールのコストの40-50％を獲得しました）。 特に関連するのは、高価なユニークなソリューションのアフィリエイト割引です（20,000〜50,000ルーブルの価格帯）。 このようなソリューションをクライアントに喜んで販売し、認定ゴールドパートナーとして、売上の50％を獲得します:)。



このすべてにより、何らかの理由でそのようなソリューションのほとんどの開発者はアフィリエイト割引を与えません。 そして、それらがなければ、ソリューションをクライアントに販売することは意味がありません-そのような販売で何も獲得しません。



自由に決定できますか？



MarketPlaceが（支払いの可能性がある）立ち上げられたばかりのとき、新しいモジュールが非常に多く登場し始め、その中には多くの無料モジュールがありました。 パートナーは、最高のステータス（Bitrixを購入する際の大きな割引）を獲得し、Bitrix Webサイト（新規顧客）でのパートナーのランキングを上げることを可能にする決定について多くのポイントを受け取ったため、論理的でした。 決定のために多くのポイントを与えることにより、Bitrixは開発者にできるだけ早く新しい空のMarketPlaceを埋めるよう奨励しました。



現在、MarketPlaceには650を超えるソリューションがあります。 これらのうち、無料のものが多い-44％。 しかし、アフィリエイトポイントのスコアリングのポリシーが変更されました。ポイントが会社にとって高すぎる可能性があるという理由だけで、アフィリエイトポイントが少なくなり、無料ソリューションを発行できるようになりました。



それでも、無料の製品をリリースすることが理にかなっている場合があります。 これは、次の場合に実行できます。



1つは、経験を積んで経験を積むことです（有料ソリューションのリリースの準備）。 これは、MarketPlaceの開発がBitrixのモジュールの単純な開発と非常に異なるということではありませんが、まだいくつかの違いがあります。 お金を払ったクライアントが、何らかの理由でモジュールをインストールするときに、すべてが間違ったエンコーディングで表示されるようになったとは考えられません。



2番目のケースは、頻繁に使用される機能のリリースです。 各プロジェクトでプログラマーが開発した機能を使用する場合、モジュールとしてフォーマットしてMPに入れることができます。 これにより、開発者の時間が節約され（ファイルのコピー、フォルダー/テーブルの作成などが不要になります。これらはすべてモジュールインストーラーによって行われます）、機能の便利な更新も可能になります。 たとえば、機能にバグが見つかり、それが20のクライアントサイトで機能する場合、すべてのサイトで編集する必要があります。 機能がMPモジュールとして設計されている場合、1つのアップデートをリリースできます。 同じケースには、管理上の決定、開発者向けのソリューション、特定のサイトに論理的に関連付けられていないユーティリティ（ DbgTracer 、 Live Api 、「 Infoblocks、ツール 」）が含まれます。



3番目のケースは、サービスに関連付けられたソリューションです。 たとえば、外部サービスがあり（それ自体が有料の場合もあります）、そのサポートを一般的なCMSに追加したい場合があります。 そのようなソリューションの例：「 JivoSiteオンラインコンサルタント 」、「 PickPoint Postamataによる配信 」。



最後の4番目のケース-これは会社のPRに関する作業です。つまり、高品質で人気のある無料ソリューションのリリースです。 最良の例は、 DEFA SocialMediaPosterモジュールの「 資料を共有する 」です。私たちは、「 Twitterフィード 」と「VKontakte」コメントウィジェットの2つに名前を付けることを恥ずかしく思いません。







どんな決断をしますか？



これは間違いなく最も難しい質問であり、それに対する明確な答えは機能しません。 人気の有料ソリューションをリリースするには、トレンドをフォローし（サイトで増えつつある機能を観察）、品質と「美しさ」（ソリューションは簡単なインストールで使いやすくなければなりません）を監視し、最適な価格を選択する必要があります。



たとえば、私たちに最も需要があったソリューションは、「 Order a call 」と「 Tree comments 」でした。



呼び出し順序モジュールはトレンドに完全に適合し（オンラインストアで一般的になりつつあります）、標準のBitrix機能から欠落しており、インストールと使用が非常に簡単で、安価です。 ところで、モジュールの更新はまだリリースされていません-クライアントは尋ねず、すべてが彼らに合っています。



以前に出てきたツリーコメントモジュールにはさらに問題がありました。 そのような機能が人気があるため、彼は間違いなくトレンドに落ちましたが、Bitrix自体とMarketPlaceの他のソリューションの両方にもコメントがあります。 ソリューションの「美しさ」に集中する必要がありました。モジュールには、便利な小さなものがたくさんあり、2つのデザインテンプレートなどがあります。 など これにより、1990ルーブルのモジュールを完全に販売できます。 しかし、手紙には顧客からの質問や提案が常に寄せられているため、ソリューションの更新のリリースにかかる時間はいくらかかかります。 ちなみに、このモジュールで私にインスピレーションを与えたのはHabrのコメントです:)



まったく占有されていないニッチは、積極的に開発している「企業ポータル」のモジュールのリリースであり、現在この製品には15のソリューションしかないことに注意する価値があります。低コスト。
