1.はじめに。



時々、いくつかのカメラでイベントを撮影する必要があります（すぐに「マウントされたビデオ」を取得します）。高いレベルを持たず、高い予算を使わず、プロのTCP（Mobile Television Station）を借りません。 そのような目的のために、「彼の膝の上のタイトル」を収集しました。



私たちはしばしばそのようなTCPを使用し、それがどのように機能し、すべてを行うかについて同じ質問に答えないために、一連の記事を書いて一連の小さなビデオを撮影することを決めました。それを整理する方法を簡潔かつ明確に説明しようとします実行します。

ここで説明するのは、個人的な経験と、それをどのように行うかだけです。



「ホーム」PTSでは、6台のカメラのうち2台に加えて、高価なテレビ機器を使用していません。 自家製のTCPには6台のカメラがあり、854 x 480ピクセルの解像度、1秒あたり25のフレームレートで動作します。 原則として、HD 1280 x 720で作業できますが、コンピューターはこのような負荷を好みません。 もちろん、ここでは専門的なレベルと技術について話すことはできません。 おそらく将来的には、Blackmagicベースの6台のカメラで、プロフェッショナルサービスコミュニケーション、スタジオカメラなどを使用してプロフェッショナルTCPを組み立てる方法について同様のビデオを作成する予定ですが、ここではまったく別のタスクがあります。



確かに多くの解決策がありますが、少なくとも週に1回は5年以上にわたって個人的に使用する方法について説明します。



2.機器。



自家製PTSは次の機器を使用します。



一般的な計画では、ウェブカメラを使用します。 特定のウェブカメラの画質は、一部のアマチュアハンドヘルドよりも長く劣っていません...



さまざまなものを試した後、 Logitechカメラに決めました。 お金の価値私たちは満足しています。 どこかで同じウェブカメラを使用すると、コンピューターの仮想ミキサーがそれらを混乱させ、一般に設定が飛んでしまうことがあると聞いた-これは作業を複雑にします。 これを実験しないことにし、異なるモデルの4台のカメラを使用しました。



そのため、キットには次のLogitechモデルがあります： C310、C510、Pro 9000およびC910 （詳細は以下のビデオにあります）。 480ピクセルでの撮影にこれらのカメラを使用しているという事実にもかかわらず、それらはすべて1280x720ピクセルで動作する可能性があります。 C910はフルHD（1920 x 1080ピクセル）にも対応しています。 ところで、現時点では、LogitechにはC920モデルがあります。 私たちは、それがC910とうまく機能することを確認しました 。 しかし同僚は、 C910とC920はコンピューターに混乱していると言った 。 私たちの意見では、写真ではそれらは最高ですが、両方を持っていることはお勧めしません。







さらに、4つのWebカメラに加えて、DV形式の2台のカメラを使用します。 それらの1つは、伝説的なパナソニックの「織り」（ Panasonic DVX 100 ）です。 2つ目は、DV信号を送信できるものと同じ、または「ハンドヘルド」（手に負けやすいアマチュアカメラ）である可能性があります。 DVモードに切り替えられたプロ仕様のHDカメラでもかまいません。 一般に、2台のDVカメラを使用できます。 しかし、ご存じのとおり、画像の品質はマトリックスと光学系に依存します。 「100分の1」の代わりにハンドヘルドを使用すると、状況はずっと悪くなります。 これは、ドットが少ないためと思われます。 しかし、「幸福はピクセル単位ではない」ことを再び証明するものが多くあります。



キットの次のアイテムは、 Windows 7ベースのコンピューターです（それ以前は、 Windows XPでもすべて正常に機能していました。2番目と3番目のサービスパック）。 Windows VistaとWindows 8については何も言いません。欲望はありません。 I7プロセッサ。 2台のモニターが接続されていると便利です。



また、膝の上に作られた簡単なインジケーターサービス接続も使用します。これについては、次の章で個別に説明します。



また、PTSサウンドチェーンでは、 Behringerプロセッサを使用して、サウンドをより均等に、より太くしました。 しかし、放送用のサウンドを受信した新しいサウンドコントロールは、オーディオ信号を圧縮および制限する方法を既に知っていたため、後で拒否しました。



一般的に、これは、マルチカメラ撮影やインターネットへの直接放送のために、私たちの場合の機器から必要なものすべてです。



もちろん、商業レベルでは、この調査は適切ではありません。 ただし、アーカイブ、レポート、またはブロードキャストの場合、企業イベントや会議だけでなく、コンピューターで有能なカメラマンが1番目と2番目のカメラを使い、経験豊富なTCPディレクターが参加する非常にダイナミックなコンサートも撮影できます。
