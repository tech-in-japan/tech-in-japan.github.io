分散著者マッシュアップ

Web 2.0プロジェクトの主な問題を解決できるアイデアを皆さんの注意を喚起したいと思います。それは、公開された資料の質の低下と長期にわたる議論です。

個人的にトピックの品質とHabréのコメントの低下が見られない場合は、品質がまだ低下していると感じるまで、このトピックの閲覧とコメントを延期してください 。




品質。 Copyright Web 2.0とDemocratic Web 2.0。



大量のゴミをふるいにかける時間を無駄にすることなく、興味深く、高品質のコンテンツを読むことができるようになりたいです。 この機会に喜んで支払いをする人さえいます。



原則として、個々のブログへのサブスクリプションはそのような機会を提供しますが、... そのようなブログはほとんどなく、検索するのが難しく、それらの新しいエントリは頻繁に表示されません 。 さらに、人々は個人のブログで物事を混同することが多く、 すべてのトピックがあなたの興味を引くわけではありません 。 その結果、材料の平均品質が良好な場合、 その量は少なすぎます。 個々のブログのもう1つの問題は、統一されたコメントシステムがないことです 。これにより、ディスカッションが大幅に複雑になり、 コメントの数と質が低下します 。



それにもかかわらず、 素材の品質を一定レベルに保つ唯一の方法は著者の名前だと思います。 この著者が今何をどのように書いているかについて満足しているのであれば、将来的にはおそらくあなたに合うでしょう。 もちろん、著者を「成長させる」こともできますし、彼は「道に迷う」こともできます... :)しかし、第一に、これはすべて、民主主義とプラスがカルマにつながる材料の品質の変動と比較することはできませんが、次に、この著者の登録を解除することで、ワンクリックでいつでも問題を解決できます。



同じことが資料の議論の質にも当てはまります。 モデレーターの存在のみが、コメントの質のレベルが一定レベルで維持されることを保証します。 当然、このために、モデレーターはディスカッションを完全に制御する必要があります。事前モデレーションを使用する機能、ブラックリスト/ホワイトリスト、コメント自体を削除/変更する機能などです。



手動モデレーションの長所と短所について議論するとき、通常、2つの質問が浮上します。良いモデレーターを見つける場所（および雇うお金）と、モデレーションの質が自分に合わない場合の対処方法です。 私は両方の質問に答えています：著者自身が資料の議論をモデレートする必要があります。モデレーションの質に満足できない場合は、議論を読んだり、独自の代替ディスカッションを組織したりしないでください。



数量。 検索マッシュアップと興味深い著者の検索。



興味のある個々のブログを検索するには時間がかかります。 したがって、特に、Habrのようなサイトは人気があります-ここでは、興味のあるトピックに関するさまざまな人々の多数の資料をすばやく見つけることができます。 実際、habrは、1人だけでなく、いくつかの異なるマッシュアップ（メインページ、ハブラレント、ライブブロードキャスト、新しいものなど）によって、さまざまな作成者によるマッシュアップブログを作成します。



しかし、すべての味（マナー）のマッシュアップを準備しようとしても、ハブラーは全員（より正確には大多数）を同時に喜ばせようとします。その結果、材料の品質が低下します。通常、高品質のコンテンツを作成できたユーザーであることがわかります（つまり、出発後、品質はさらに低下します）。



特定の人が興味深いブログをマッシュアップする必要があると思います。 この場合、マッシュアップでのブログの平均品質は一定です。 そして、そのようなマッシュアップに接続することにより、あなたは興味のある多数の素材をすぐに受け取ることができ、その品質は一定です。



実装（ステージ1）



これが、「Authorial Mash-up」の作業の様子です（次のステップで「Distributed」という単語を追加します）。
  • 任意の人がテーマセクションをいくつでも作成し、著作権で保護された資料、他のサイトからの転載、リンクなど、自分が望むものを投稿できます。 彼はこれらのセクションと資料の唯一のモデレーターです。

    誰でも任意の素材にコメントを残すことができますが、モデレーターは特定の人がコメントを残す能力とコメント自体（モデレーションとモデレーションの選択を含む）の両方を完全に制御します。 言い換えれば、素材の作者はコメントで何が起こるかを決定します。 彼は、素材の品質とコメントの品質の両方を決定する唯一の要因です。

    マッシュアップを行うことができます：他の人の資料（ディスカッションを含む）をセクションに「追加」します（またはまったく別のセクションを自分のセクションに「接続」します）。 この場合、あなたの資料は読者に表示され、その議論に参加することができます（コメントは、資料の著者によってモデレートされます）。

    素材の作成者によるディスカッションのモデレーションに満足できない場合は、この素材に対して独自の並行ディスカッションスレッドを作成できます。 この場合、読者は両方のブランチを見ることができ（両方に参加できます）、素材の著者の元のブログの読者は最初のディスカッションスレッドのみを見ることができます。 したがって、1つの資料でいくつかの独立したディスカッションをまったく異なる方法で管理し、読者は参加するディスカッションを選択できます。



    このアプローチは、この資料に興味があり、同じレベルでそれを議論したい人々の間で、誰もが彼がそれを議論したいレベルでどんな資料でも議論する機会を持つという事実につながります。 異なるレベルで同じ素材を議論する他の人々と干渉することなく。



    マッシュアップのおかげで、ブログをバッチで接続することが可能になり（おそらく、これらのブログから独自のマッシュアップを作成する際に）、これらのバンドルは特定の人々によって「自分のために」ピックアップされます。



    そして、節度のおかげで、資料とディスカッションの両方の品質はほぼ一定のレベルに保たれます（このセクションに接続したときに最初に適していました）。

    モデレーター



    トピックに関する強力なセクションがあり、それは厳しい資格のあるモデレーターによって実施されます。 執筆はせず、自分の観点から立っている記事のみを選択し、コメントの品質を監視する場合でも、彼のセクションは非常に高品質です。 また、たとえば、記事の著者が自分の場所でコメントに従わない場合、記事を接続するこのモデレーターは「コメントなしで接続」オプションを選択できます。彼のセクションの読者は、単に著者のセクションでこの記事の元のコメントを表示せず、次のレベルで最初から議論しますこのモデレーターによって定義されます。

      定義により、それぞれに独自の一意のURLがあり、それに応じてマテリアル/セクションにアクセスできます。たとえば、同じURLをOpenIDによる承認に使用できます。 ただし、分散システムで認証を行うのに十分な方法があり、OpenIDに焦点を合わせる必要はありません。
