ネットワークパフォーマンスの低いHyper-Vに対処する方法

ネットワークパフォーマンスは、仮想環境を「作成」したり、その動作を中断したりする可能性があります。 Hyper-Vネットワークツールを使用すると、高いネットワークパフォーマンスを維持し、リソースを均等に分散できます。 これについて思い出すべきではありませんが、仮想ネットワークは物理サーバーに、したがって仮想環境に深刻な影響を与える可能性があります。 仮想化には多くの利点がありますが、その主な欠点は、帯域幅などの貴重なリソースを消費し、物理サーバーの入出力システムを占有する新しい仮想マシンを導入することです。







この問題は、複数の仮想サーバーが単一の通信システムを求めて競合し、大きな遅延が発生する場合に悪化します。 この問題の解決策の1つは、仮想スイッチや仮想ネットワークカードなどのツールを使用してネットワークリソースを論理的に分散することです。



ユーザーが仮想環境を作成および管理できるように設計されたHyper-Vハイパーバイザーには、これらの問題に対処してリソース割り当てを制御するツールが含まれています。 Hyper-VポートミラーリングからHyper-V論理ネットワークまで、仮想ネットワーク、ひいては仮想環境が最高のパフォーマンスレベルにあることを確認する方法がいくつかあります。 以下は、 Hyper-Vネットワークのパフォーマンスの問題の数を減らすための5つのヒントです。







Hyper-Vポートミラーリングによりネットワークトラフィックを簡素化



多層仮想ネットワークアーキテクチャの一部として仮想マシンを使用する場合、負荷に注意を払うだけでなく、重要です。 Windows Server 2012 Hyper-Vに含まれるポートミラーリングは、Windows Server 2012の仮想マシンからネットワークトラフィックをキャプチャし、この情報を監視用に構成された仮想マシンに転送します。 ポートミラーリングは、仮想スイッチに接続された仮想マシンからの内部トラフィックと、物理サーバーから仮想マシンへの外部トラフィックを制御できます。 これに加えて、ポートミラーリングを使用してタグ付きVLANトラフィックをキャプチャしたり、不要なネットワークトラフィックをブロックしたりして、ネットワークのパフォーマンスを低下させることもできます。 Hyper-Vポートミラーリングを使用すると、管理者はネットワークパフォーマンスを簡単に最適化しながら、セキュリティを提供し、潜在的なネットワークの問題を評価できます。



仮想スイッチ



ご存じのように、 Hyper-Vマネージャーテクノロジーを使用して、 Hyper-V環境で仮想ネットワーク接続が確立および管理されます。 Hyper-Vマネージャーを使用すると、管理者はすべてのホストオペレーティングシステムメッセージが通過する仮想スイッチを構成および管理できます。 Hyper-V環境には3種類の仮想スイッチがあり、それぞれが物理ネットワークに接続してホストサーバーから分離されたネットワークセグメントを操作する機能から、いくつかの異なるタスクを実行します 。 これらのスイッチを効果的に使用すると、 Hyper-Vネットワークのパフォーマンスが向上し、常にセキュリティが向上します。



Hyper-Vによるネットワークセキュリティの強化



Windows Server 2012のリリースに伴い、 Hyper-Vの新しいバージョンもリリースされ 、いくつかの重要な変更と多くの新機能が追加されました。 新しいバージョンには、ルーターガードや動的ホスト構成プロトコル（DHCP）ガードなど、 Hyper-Vマネージャーテクノロジを使用してアクセスできる高度なネットワーク機能が含まれています。 1つ目は、ルーターメッセージと転送メッセージを削除することにより、仮想マシンがルーターとして機能するのを防ぎます。 これにより、コンピューターのOSが危険にさらされている場合、追加のセキュリティレイヤーが作成されます。



後者のDHCPガードには、いくつかの潜在的な用途があります。 DHCPガードは、Active DirectoryレベルでDHCPサーバーを許可し、なりすましのDHCPサーバーへのアクセスを防ぎます。 また、許可されたDHCPサーバーに制限を課し、その機能を制限します。 その結果、ルーターガードとDHCPガードの両方が、違法なネットワークサービスからの仮想ネットワーク環境の仮想マシンの保護を強化します。



Hyper-V論理ネットワークと仮想化のスケーリング



Hyper-Vマネージャーは、その有用性にもかかわらず、あらゆる規模のネットワーク環境には適していません。 Hyper-Vマネージャーはサーバーレベルの管理ツールとしてのみ機能するため、大規模な環境とは互換性がありません。 Hyper-Vマネージャーを使用して1つのホストから別のホストに仮想マシンを移動するには、いくつかの同一の仮想スイッチを作成する必要があり、これには多くの時間がかかります。 以前のHyper-V仮想ネットワークも、マルチテナント環境で問題に直面しています。 大規模なネットワーク環境で作業する場合は、 Hyper-Vマネージャーの論理ネットワークを構築して接続を拡大する必要があります。



論理ネットワークは、仮想ローカルネットワークとサブネットをグループ化してホストグループに割り当てることにより、ローカルネットワークを除外します。 論理ネットワークは、Microsoft System Center Virtual Machine Managerによって定義および管理されます。 論理ネットワークを最大限に活用し、ネットワークパフォーマンスの低下を防ぐために、仮想マシンのネットワークを作成し、論理ネットワークで動作する論理スイッチを定義する必要もあります。 論理ネットワークの構築と実行に必要なすべての部品を組み立てたら、新しい管理対象ネットワークを作成するときにこれらのコンポーネントを将来再利用できます。



Hyper-Vからすべてを絞り出す



物理サーバーでの競合を防ぐということになると、 Hyper-Vの低いネットワークパフォーマンスを回避することは始まりに過ぎません。 仮想環境がすべてのレベルで適切に見えるようにする必要があります。 これは、ネットワークパフォーマンスの最適化に加えて、 Hyper-Vの動的移行のパフォーマンス調整、バックグラウンドアクティビティの削除、GPUオフロードを高速化するハードウェアアクセラレーションツールの調整を意味します。
