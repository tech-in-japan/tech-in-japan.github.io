Nutanix Cloud Connect-AWSクラウドバックアップ





バージョン4.1のリリースの1つであるNutanixには興味深い機会があります-Cloud Connect。

Cloud Connectは、Amazon Web Services（AWS）クラウドのEBS + S3にあるディスクで「仮想Nutanix」を作成し、リモートバックアップストレージとして使用する機能です。 現在、多くの人々がそのようなことをしていますが、今ではNutanixにそのような機会があります。



残念ながら、私が指摘したように、ロシアでは、パブリックおよびハイブリッドの「クラウド」をデータセンターの深刻なビジネスオプションとして使用することに対する深刻な先入観がまだあります。 それでも、それはある種の「おもちゃ」、何か「軽薄」と考えられています。 おそらくこれは、ロシアのITの特徴である保守主義と、世界ニュースが2年から3年の遅れでIT-shnyaロシアにかなり鈍い、または現代のクラウドプロバイダーが提供する新機能に関する知識が不十分な伝統的な遅れによるものです。 そのため、たとえば、有名なNetflixが完全にシェアレスの会社であることを知っている人はほとんどいません。 数千の Netflix TVショーとビデオのすべてのストレージは、AW​​Sクラウドでレンタルされたスペースにあります。

そのため、明らかに、ストレージおよび処理機器におけるパブリッククラウドプロバイダーとの組み込みの統合は大きな需要があります。 これがNutanixのCloud Connectがおもしろいが、まだ評価が低い機能である理由です。だからこそ、今日それについてお話しするつもりでした。



Cloud Connectは、Nutanix Prism UI HTML5管理インターフェイスに組み込まれています。 開始するには、IAM（AWS Identity and Access Manager）からAWS環境へのアクセスの詳細を提供する必要があります。







次に、Cloud Connectは、この機能が存在するCommunity Editionを既に試用しているユーザーになじみのある「シングルモード」クラスターの形で「反対側」の仮想Nutanixを作成します。 データ復元力はAWS側で提供され、複製する必要がないため、「Nutanix復元力」なしでAWSでシングルモード構成を使用することを理解してください。



次に、必要なEC2リージョンを選択し、「AWSクラウド内の仮想Nutanix」のコピー用のVPNチャネルをセットアップし（AWSリポジトリ内にNOSを備えたAMI仮想マシンをすでに準備およびインストール済み）、その他の設定を行います。







そしてそれだけです。



Nutanixは、システムデータのスナップショットをリモートストレージに独立して複製します。 そして、注意してください、これは重要です。 バックアップはスナップショットを使用して実装されます。つまり、最初の同期の後、指定されたデータの差分状態のみがリモートシステムに転送され、さらにオンザフライで圧縮されて、フォームの転送とストレージの量が削減されます。

データの配置には、EBS、Elastic Block Storage、SSD層に配置して使用する高速ブロックストレージの両方を使用します。これは、「仮想Nutanix」のメタデータを保存する「大」Nutanixと同様に、安価な（しかし比較的遅い）AWS S3ストレージです（シンプルストレージサービス）。



そしてもちろん、そのような「仮想Nutanix」は、「1対多」や「多対1」などのレプリケーションのあらゆる手段を使用して、蜂蜜を異なるAWSリージョンと同期させることができます。



テストでは、1秒あたり200メガビットのバックアップ転送ストリームを達成しました。 理論上の制限は500ですが、EBSとS3間のSSL / TLS処理のために一部のリソースが消費されます。



それでは、そのようなバックアップの費用を見てみましょう。

Cloud Backupを使用するには、次のものが必要です。

  • バックアップを受け入れるAWS環境に「仮想Nutanix」をインストールするためのタイプm1.xlargeのEC2インスタンス。
  • ボリュームEC2
  • バケットS3
  • 仮想マシンへのVPN。


この投稿を読んだ時点での最新の価格を備えた正確な計算機は、次のページから入手できます： calculator.s3.amazonaws.com/index.html

バックアップの内容は半分に圧縮されていると仮定します。これは真実にかなり近い推定であり、簡単にするために1TBの容量を取ります。つまり、S3に500GBを保存します。 これにより、月額17ドルが得られます。

復旧トラフィックには0.02ドル/ GBのコストがかかりますが、保存されたボリューム全体ではなく、Nutanixのスナップショットの形式とボリュームでデータを復元するため、これらのコストはおそらく小さく、データ復旧の頻度に依存します。
