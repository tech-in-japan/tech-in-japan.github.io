Code School CEOのGreg Pollackが退職した理由





2015年1月、Pluralsight はプログラミングのためのCode Schoolオンライン学習プラットフォームを買収しました。 この10か月後、買収した会社は創業者のグレッグ・ポラックを辞めることにしました。 彼はこれをPluralsightの経営者-CEOアーロン・スコンファードとCFOグレッグ・ウッドワードに報告しました。 そして今、一年後、ポラックは彼の決定を彼のブログで一般大衆に説明した。



創業者が会社を辞めた場合、買収後、事態は悪化したか、すぐに次のスタートアップに進むために売却したかったようです。 ポラックの場合、両方の仮説は真実ではありません。 最初は、彼は去るつもりはなかった。



「今日のコードスクールは、真のプロフェッショナルが仕事をする素晴らしい場所であり、自分の仕事に情熱を注いでいます。他の人が新しいテクノロジーを学ぶのを楽しんでいる人々です」とグレッグは書いています。



彼によると、Pluralsightには優れた経営陣、取締役会、投資家のチームもあります。これらはまさに会社をさらに大きな成功に導く人々です。 彼らは独立したビジネスユニットとしてコードスクールと連携しています。



「引き継ぐことを決定する前に、3つのシナリオを検討しました」とPollackは言います。

1.独力でビジネスを発展させ続けます。



コードスクールは常にゆっくりと着実に成長しています。 他に予想されたものはありませんでした。



2.規模拡大に役立つ戦略的投資家を探します。



視聴者を500万人から50人に増やす必要がありました。より経験豊富なパートナーが必要でした。 さらに、取締役会はありませんでした。 これにより、作成できるようになりました。



3.バイヤー-私たちがうまく適合する会社を探します。
ポラックを去る理由は、彼のモチベーションの低下を考慮しています。 彼はこれを2015年8月までに、つまり買収の9か月後に実現しました。 彼は以前のようにコンテンツを作成したかった。 代わりに、ゼネラルディレクターの終日純粋に「管理」機能を実行する必要がありました。 彼は終わりのない審議と初期の官僚制にうんざりしていた。



Gregは、Code Schoolプロジェクトがまだ小さなスタートアップだった時代に戻りたいと考えていました。アイデアからその実装までの道のりが非常に早く通過できたからです。



したがって、ポラックは彼に最も満足をもたらす仕事に戻ることを決めました。 「コントロールのマニアに少し苦しんでいます。仕事が限られているときは好きではありません。上司がいるのは好きではありません」と彼は説明します。 したがって、ポラックは、新しい組織（Pluralsight-Code School）の枠組みでは、自分の居場所を見つけることができないと判断しました。



彼はCEOでもある数人の友人と話をしました。 彼らは彼にPluralsightとのパートナーシップを終了する最善の方法についていくつかのヒントを与えました。 「私はこのニュースがどのように受け取られるかについて非常に緊張していました。 おそらく、私は5年間あまり緊張していませんでした」とグレッグは書いています。 「才能のある人々が会社を辞めたときの苦痛を知っています。 これは、喪失感を引き起こします。 そして、私の会社を買収することで多くのリスクを冒した人たちにも同じ痛みをもたらしました。 彼らは私を信頼していましたが、私は彼らの信頼を正当化しませんでした。」



グレッグ・ポラックからのヒント



  • 潜在的な買い手を見つけるのに最適な時期は、投資を得る前です。 そうでなければ、現在の会社の10倍の見積もりで会社を買収することは投資家の利益になります。


  • 設立した会社を辞めたり、別の場所に移りたい場合は、知っておいてください。これは可能であり、あなたはそれを行うことができます。 そのような場合は、企業の心理学者に連絡するのが最善です。


  • チームメンバーの大部分からの情報（たとえば、退職の計画など）を秘密にしておくことは、正しい判断である場合があります。 これは私の内なる価値とチームの価値に反していましたが、それでも正しいことであることが判明しました。 会社を作成したときと同じように意識的に会社を離れることができます。
