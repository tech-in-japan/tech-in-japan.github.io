cPanelからZimbra Collaboration 8.0にメールを転送する方法

これは、電子メールをcPanelからZimbra 8.0.x 8.xに移行するための指示です。 サーバーに変更を加える前に、バックアップを作成する必要があります。







アカウントとパスワードの移行



電子メールアカウントとパスワードをcPanelにエクスポート



  1. cPanelにログインして、[ファイル]> [ファイルマネージャー]メニューを選択します。
  2. / etc / yourdomain-folderに移動します
  3. @pwcacheフォルダーを右クリックし、[圧縮]をクリックします。
  4. @ pwcache.zipファイルとシャドウファイルをダウンロードします
  5. @ pwcache.zipとシャドウファイルを/ srv / Zimbra Serverにコピーし、WinSCPを使用できます（権限に問題がある場合は、フォルダーの宛先を変更できます）


ZimbraでcPanelアカウントとパスワードをインポートする



  1. SSH経由でZimbraサーバーにログインし、/ srv /フォルダーに移動します
  2. @ pwcache.zipを解凍または解凍します 



    cd/srv/ unzip "@ pwcache.zip" mv "@pwcache" "pwcache"
  3. パスワードを作成して入力するスクリプトを作成します。 



     vi create-acc.sh ===========scripts============== #!/bin/sh # Clear screen clear rm create-account-zimbra.zmp touch create-account-zimbra.zmp echo “Looping for all users” for ACCOUNT in /srv/pwcache/* do PASS=`cat “$ACCOUNT” | grep passwd: | cut -d ':' -f2` ACC=`echo $ACCOUNT | cut -d “/” -f4` echo “Retrieve password account $ACC” echo “ca $ACC@your_domain.net PasswordTemp” >> create-account-zimbra.zmp echo “ma $ACC@your_domain.net userPassword '{crypt}$PASS'” >> create-account-zimbra.zmp done echo “Complete” ===========scripts==============


スクリプトの結果は、create-account-zimbra.zmpというファイルを生成します。 このファイルには、zimbraアカウントを作成するためのコマンドが含まれており、パスワードを元の（cPanelから）に置き換えます



create-account-zimbra.zmpスクリプトを実行します 



 su - zimbra zmprov </srv/create-account-zimbra.zmp


Zimbra管理パネルにログインして確認します。
