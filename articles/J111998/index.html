ビデオの視聴やオーディオブックの視聴にかかる時間を短縮する方法

数年前、私はいくつかの映画を見ることができないことに気付きました。 そして、恐ろしいシーンや退屈なプロットのためではなく、強力な長引くアクションの感覚のためです。 それは遅くないかもしれませんが、それはリアルタイムでそれ自身に起こります...しかし、私はこの情報をはるかに速く知覚することができます。

そして、一度、私はVLCの魔法のボタンを押しました。



映画、シリーズ、講義



それは加速ボタンでした。 1.5倍、さらには2倍の加速で、映画を観ることはかなり可能です。 これは、旧バージョンのVLCでは、ボタンが正確に速度を上げた、つまり同時にすべてのサウンドの高さが増加したという事実にもかかわらずです。 VLCの新しいバージョンでは、速度ではなくテンポが増加します。つまり、ピッチの歪みは感じられません。ビデオが短くなります。

重要なコンテンツを含む映画は見にくいです。 場合によっては（たとえば、2001年の映画 『Space Odyssey』）、加速を含めることは単に冒bl的なことです。

しかし、ほとんどのシャボン玉は大きな喜びで膨らみ、1.5倍から2倍速く破裂します。 障壁は、低品質の音声演技（ボイスオーバー）とコンピューターのパフォーマンスのみを提供できます。

VLCがファイルを開いたときに、キーボードの「+」ボタンを押して、私が説明していることを耳で感じることはそのままです。 実験！ 脳はどのような加速を維持できますか？ +）



UPD ：VLCでは、0.1x刻みで速度をよりスムーズに変更できます。 これを行うには、キー「[」および「]」を使用します。



ラジオ番組、オーディオブック、ポッドキャスト



本が複数のmp3ファイルで構成されており、コンピューターで聴く場合は、同じVLCを使用できます。

しかし、私は多くの場合、数十、2、または数百ものmp3ファイルのオーディオブックに出会いました。 そして、ペースを変える方法を知らないプレーヤーでこれらすべてを聞きたいです。



したがって、オーディオファイルをより速いペースでトランスコードし、プレーヤーで結果を聞きたいと思いました。 解決策はすぐに見つかりましたが、最もエレガントではないかもしれません。 UPD： よりエレガントになりました 。

Audacityはオーディオファイルをバッチ処理できることがわかりました。

ファイルヒープのペースを上げるように彼に教えてください。

短いセットアップガイドを次に示します。

  1. ファイル->会話の編集
  2. 新しい会話を追加して名前を付けます。
  3. ExportMp3およびChangeTempoコマンドを挿入します。

  4. [パラメーターの変更]ボタンを押して、目的の加速度を設定します（図は、60％の加速度のパラメーターを示しています）。

  5. 両方のダイアログで[OK]をクリックします。 次のようなものが得られるはずです。

  6. もう一度、OKとAudacityが必要な問題を解決する準備ができました


オーディオブックを高速化する必要があるたびに、Audacityを起動し、ファイル->チェーンの適用：





[ファイルに適用]をクリックし、[OK]と言って待ちます。 内部ファイルが加速化された「クリーン」サブフォルダーは、ソースファイルと同じフォルダーに作成されます。



UPD ：Audacityが新しいファイルのタグを台無しにできることが判明しました。 そして、id3cpは何か間違ったものをコピーしています。

これを解決するために、古いファイルから新しいファイルにid3をコピーするPythonスクリプトを提案します。 

#!/usr/bin/python
import sys
from mutagen.id3 import ID3

source = ID3(sys.argv[1])
dest = ID3(sys.argv[2])
for key in source:
    dest[key] = source[key]
dest.save()
