ホームステッドについての詳細

Laravelの公式ドキュメントにはHomesteadについての全ページが書かれていますが、問題は、これがなぜ必要なのかについてほとんど説明がないことです。 このドキュメントは、VirtualBoxとVagrantおよびLinux（Ubuntu）の両方にすでに精通している人向けの説明です。 あなたがこれらのいずれかである場合、記事は新しいものを開きませんが、ドキュメントでホームステッドについて読んで、なぜそれが必要なのかまったく理解しなかった場合、または指示に従って何かがうまくいかなかった場合、すべてがこの記事で詳細にかみ砕かれます。



PHPは、エントリのしきい値が低いことで常に有名であり、その結果、多くの初心者がこのプロセスを整理する方法を理解せずにフレームワークを研究し始めています。 OSで使用されているテクノロジーのスタック全体をインストールして、作成と実験を開始することがあります。 誤って、彼らはこれがPHPで書き始めるか、他の技術（フレームワーク）を学ぶ最も簡単な方法であると示唆しています。



残念ながら、ほとんどの教科書は開発プロセスを整理する方法を教えていません。 同様に、LaravelはVagrantの「ボックス」が必要な理由を説明せず、その使用方法についてほとんど説明しません。 その結果、学生はホームステッドセクションをスキップして、OSを習慣から外し、テクノロジー、役割、サービス、悪魔の動物園に変えます。



指で：



ホームステッドは、Ubuntu Serverオペレーティングシステムがインストールされている仮想コンピューター（独自のメモリ、プロセッサー、ハードドライブなど）である公式のプリインストールされたVagrant「ボックス」仮想マシン（VirtualBox）です（理論的には、任意のOSをインストールできます）。 このUbuntuサーバーへのアクセスは、SSHコンソールを介して行われます。 仮想マシンの巨大なイメージ（ギガバイトを占有する可能性があります）をドラッグしないようにするために、OSイメージをダウンロードし、必要なソフトウェア（Webサーバー、データベースなど）をインストールし、準備ができて動作する仮想マシンを起動できるVagrantツールを開発しました（開発） 。 HomesteadはVagrant用の設定とスクリプトのセットであり、Homesteadで書かれており、インストールするバージョン、作成する設定など、Vagrantはすべて仮想システム上で実行します。 VagrantはVirtualBoxだけでなく、他の仮想化システムでも機能します。



なぜこれが必要なのですか？



これは、PHP作成用の作業環境を作成するプロセスを促進するために必要です。 PHPで記述を開始した場合、すべてが機能するためには、Webサーバー、データベース、PHP自体などをインストールする必要があることを既に知っています。 問題は、両方のWebサーバー、データベース、膨大な数のバージョン、構成オプションなどが非常に多いことです。 つまり、メインのインストール手順（コンポーザーインストール）を実行する前、またはCMSのインストーラーを開始する前に、これが発生するコンピューター（サーバー）を慎重に慎重に構成する必要があります。 次に、あるスクリプトが1つのバージョンのphp、Webサーバー、データベースを必要とし、別のスクリプトが動作を拒否し、他のバージョンを必要とする場合の競合について考えます。 もちろん、同じPHP、nginx'a、MySQLの複数のバージョンを1つのサーバーにプッシュすることもできますが、遅かれ早かれこの動物園は制御不能になります。



別のシナリオ：プロジェクトでチームワークに遭遇したか、カスタムプロジェクトを開発する会社で働くことを試みた可能性があります。 あなたがプロジェクトに新しい人をつなぐとき、彼らは彼が何をどのように構成するかを説明することを気にせず、ただプロジェクトを始めます。 プロジェクト全体がデプロイされる既製のサーバーにアクセスできるようになることを期待しないでください。そこに変更を加えることができます。 会社の観点からは、これは現在のソースと開発が単純に合併されるリスクです。 ほとんどの場合、Vagrant configが正確に与えられます。これは、この例ではLaravelのHomesteadです。 必要なソフトウェアを構成し、作業するプロジェクトの最小部分を引き出します。



ところで、私はこれがすべてシステムに負荷をかけているのではないかという恐れをすぐに払拭します。 まったくありません。 VirtualBoxにUbuntuを搭載したHomesteadは、Atom z8500 \ 4GB RAMを搭載した私の中国語タブレットで簡単に飛ぶことができます。



すべての設定は数分で完了します。OpenServerをインストールするのと同じくらい簡単です。



ホームステッドの展開プロセスは、ロシアのドキュメントで実際によく説明されています。 また、開発環境での展開と作業に関するいくつかのポイントについても説明します。



VirtualBoxとVagrantをコンピューターにインストールしたら、その中にHomesteadフォルダーを作成し、 そこにあるリポジトリからすべてのファイルをダウンロードします。 zipアーカイブをダウンロードする場合は、忘れずに解凍してください。







作業環境の実行準備はほぼ整いました。仮想システムに転送するフォルダーを指定する必要があります-原則として、これはプロジェクトコード自体のあるフォルダーです。



そのため、（ホスト）コンピューター上に別のフォルダー（laravelなど）を作成します。後で、フレームワーク自体のソースコードが作成され、プロジェクトがその中に配置されます。 Homestead.yamlファイルを修正し、いくつかの設定がそこに登録されます。 Homestead.yamlの例をコメントとともに示します 



--- ip: "192.168.10.10" memory: 1024 cpus: 1 provider: virtualbox #Generated SSH public key authorize: C:\Users\Admin\Documents\homestead\ssh\public.pub #Generated SSH private key keys: - C:\Users\Admin\Documents\homestead\ssh\private.ppk #Sync folder folders: - map: C:\Users\Admin\Documents\laravel #on host PC to: /home/vagrant/laravel #on virtual PC #Nginx on virtual PC sites: - map: laratest.dev #server_name for nginx on virtual PC to: /home/vagrant/laravel/public #document_root for nginx on virtual PC #Create MySQL Database on virtual PC databases: - homestead


ところで、SSHキー（public.pubおよびprivate.ppk）を生成または指定することはできません。 次に、Putty（または他のSSHクライアント）を介して仮想サーバーに接続する場合、ユーザー名\パスワードvagrant \ vagrantを使用します。



また、構成を置き換えることをお勧めします 



 sites: - map: homestead.app


に 



 sites: - map: laratest.dev


ドキュメントのようにデフォルトのhomestead.appアドレスのままにすると、私のクロムは言った：

組織または学校のイントラネット上のこのサイトには、インターネット上のサイトと同じURLがあります。


以下のHomestead.yamlには、コメント化されたポートのセクションがあることに注意してください。 これは、仮想システムとの間でポートを転送するのが簡単なことです。 このセクションはコメント化されていますが、Homesteadはデフォルトでいくつかのポートを実際に転送します。 runコマンドを実行すると、これに関するメッセージが表示されます。



開始するには、Homesteadフォルダーに移動し、Shiftキーを押しながらマウスの右ボタンを押し、[コマンドウィンドウを開く]オプションを選択すると、Homesteadディレクトリに既に「移動」したコンソールが表示されます。 指示に従って浮浪者を作成します。 Vagrantが突然エラーを出しても心配しないでください。VirtualBoxまたはVagrantの最新バージョンをお持ちでない可能性があります。 いずれにせよ、エラーの説明をグーグルで検索してください。原則として、解決策はVagrant'aの* .rbファイルを編集するか、再インストールすることです。 たとえば、最新のバグの1つである「server_version.rb：54：in` readpartial '：確立された接続...」はここで解決されます 。



すべてが正しく行われた場合、Homesteadディレクトリからvagrant upコマンドを実行すると、VagrantはUbuntaを使用して仮想システムのイメージをダウンロードし、そこにソフトウェアの全リストをインストールし、Homestead.yamlファイルに登録したものを忘れずに構成を修正します。 最終的に、「Machine booted and ready！」と、どのポートが転送されるかに満足します。









次に、OS、お気に入りのブラウザー、エディターで通常どおり作業します。 原則として、マシンに他のソフトウェアを保持する必要はありません。 必要なものはすべて、すでに仮想環境にあります。 Puttyを使用して接続します（上記のスクリーンショットのように、接続127.0.0.1、ポート2222のアドレスを指定します）。



cdコマンドを使用して、プロジェクトフォルダー（Homestead.yamlで指定された）に移動します。
