FortranとCでLuna Parkが欲しい

この記事のもう1つのタイトルは、「Visual C ++でノミネートされたロシアのMVPに興味がなくなった理由」です。 読者が持つかもしれない最初の関連は、私はもはや時代遅れの言語に興味がないということです。 そのような連想は私をとても悲しくさせます。 C ++は、すべての生物よりも生き生きしています。 残念ながら、ロシアでは多くの人がこのことを知らず、全世界がC＃、PHP、Python、その他の新しい言語でのみ長い間プログラミングされていることを確信しています。 これは非常に奇妙であり、私はこの砂漠で孤独を感じています。 これについては私の記事になります。



C / C ++ / C ++ 11は、すべての生き物よりも生きています



これについて議論し確認するために、私は理由を見つけません。 ただです。 ここでいくつかの議論に言及しました 。 または、もう1つの小さなメモ、「 Cプログラミング言語が再び開発者の間で最も人気になった 」とそれからのタブレットがあります。



しかし、最も重要なことは、C ++言語に特化した会議が世界中で開催されていることです。 強調します。 会議でのレポートのいくつかはC ++言語に関するものではなく、会議全体を対象としています。 たとえば、このような2日間の会議がありました 。 または、たとえば、 この会議が計画されています。 関連する会議では、C ++もかなり注目されており、子供向けではないレポートは「 C / C ++プログラムの最新の静的セキュリティチェック 」の精神でレビューされています。 一般に、生命があり、生命は活発です。



Fortranは生きています



ところで、Fortranはすべての生物よりも生きています。 確かに、それを使用する人は公にではなく静かにそれを行います。 これは、Fortranを使用することは一種のネクロマンシーであるという印象をインターネット上に与えます。



Fortranは今でも数学計算のための最も効果的なプログラミング言語であることをご存知ですか？



その理由は、Fortranがデータ配列（行列）を並列処理できるからです。 そして今、多くのプログラマーは、FortranからC ++に切り替える遠い計画しかありません。 これは、マトリックスがC ++で表示されるときに発生します。 これはいつか起こるかもしれません。 インテルは、C ++標準化委員会でこのトピックを推進しています。



もう一つ言います。 噂によると、インテルはインテルC ++を使用するプログラマーよりもインテルFortranコンパイラーのユーザーに多くのお金を稼いでいます。 ここにそのような「死んだ」言語があります...



そして、ロシアはどうですか？



ロシアで何が起こっているのかは明らかではありません。 C ++およびFortranでの複雑なプロジェクトの開発には、優れた開発者が関与しています。 まさにそこに。 同じHabréで、多くの記事がC ++開発のみに当てられています。 つまり、C ++プログラマーが存在し、言語にも関心があります。



しかし、ロシアで開催された会議では、いくつかのゴミがあります。 彼らは私を怒らせた。 C ++に関する会議がないという事実は、私は一般的に黙っています。 しかし、C ++についての報告だけでも非常に少数です。 そして今、それらのほとんどは、電話やタブレット向けのプログラミングの特別なケースとして、C ++に専念しています。



C ++については、Intelのカンファレンスで多少なりとも聞くことができます。 ただし、C ++は独立したトピックではなく、Intelプロセッサ向けの効果的なプログラムを作成する機会と見なされています。



ロシアのMVP Visual C ++



Visual C ++でロシアのMVPになりました。 この指名で私が一人でいるということはすでに兆候です。 ロシアは人口的には小さな国であり、MVPが少ないことを理解しています。 しかし、まだどういうわけか奇妙です。 C ++言語に関連するMVPのプライベートグループでは、活発な議論と通信があります。 何らかの理由で、他の国では、C ++トピックは1つのMVPをまったく考慮していないと思われます。



実際、DevCon 2012がついに気分を害しましたが、主なトピックはWindows PhoneとWindows 8であることがわかりました。 私は素朴でした。



ずっと、C ++について話すためにたった一人の人が私のところに来ました。 さらに、私は昼食時に偶然この人に会いました。 壮大な失敗。



もちろん、特に驚くべきことは何もありません。 マイクロソフトは、新しいものを宣伝しました。 熱心な人たちが彼らについて聞いて来ました。 C ++は、いくつかのマイナーなレポートによって会議で発表されました。



別のことが私を襲います。 DevConには実質的にC ++プログラマーがいなかったので、どのようなロシアのカンファレンスに参加しますか？ あぁ、どこにいるの？ あなたはそこにいます！ 多くのC ++開発者と学界のFortranプログラマーがいます。 みんなどこ？ 実際、C ++のトピックに関する経験を交換したいと考えています。 私が理解しているように、現在そのような交換はありません、会議はありません、何もありません。
