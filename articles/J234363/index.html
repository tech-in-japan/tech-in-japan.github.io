時間翻訳2014について

ご存知のとおり、2014年10月26日午前2時に再び時間を設定します。 明らかな理由でWindows XP用のパッチはありません。サポートされているOSの場合、パッチは日付Xの2週間前に予定されています。この投稿はXPユーザーとすべてを事前に確認したい人向けです。



Tzeditを使用して手動で組み立てられました（Microsoftからではなく、Dynamic DSTを理解していませんが、 これを使用して ） Win7およびXPのロシア全体のタイムゾーンは こちらです。



これまでのところ、Vista 8での非常に予備的なテストのみがサーバーでテストされていません（ただし適切であるはずです）。これらを非常に条件付きで確認しました。 一部の技術ソフトウェアは、それらの100％が破損します。 移行とタイムゾーンに関する私の意見は、Microsoftがパッチをリリースしたときの将来の意見と一致しない可能性が非常に高いです。 XPの場合、Win7からレジストリの一部を単純にインポートできます。 以下の情報はすべて、ソフトウェアを事前にテストしたい人、またはパッチがさまざまな理由で時間どおりにリリースされず、他の誰かが周囲の核灰間の時間移動のトピックに興味がある場合のものです。



HKLM \ SOFTWARE \ Microsoft \ Windows NT \ CurrentVersion \ Time Zonesの内容のバックアップコピーを作成した後、希望する人はテストするか、自分で収集を開始できます。 リポジトリ内のタイムゾーンデータを編集した後、別のゾーンを選択して適用し、それから正しいゾーンを選択するか、コマンドラインからそのまま適用する必要があることを忘れないでください（Windows 7の場合、kb2556308後のVista、tzutil.exe / s「ロシア標準時」 "、XP control.exe timedate.cplの場合、/ Zロシア標準時）。 Win 7のUACを忘れないでください。コンピューターポリシーに変更を割り当てます。



Vista +ロシア語（および他の言語）では、名前はtzres.dllで保護され、デジタル署名されているため、変更できません。古い名前に惑わされないように、MUI名へのリンクを持つすべてのキーを削除します。 確かに、この後TZEditは少し悪くなり、残りは問題ありませんでした。 削除によって誰かが妨げられた場合、「MUI_Display」=-「MUI_Dlt」=-「MUI_Std」=-という形式のすべてのキーをファイルから削除します。 さらに、英語の配布用に英語のゾーン名を使用して別のファイルを作成する理由はありません。見た目が悪いと感じる人は誰でも自分でやらせてください。



一般的なファイルに加えて、何がどのくらい変化しているかを理解するために、各タイムゾーンには<new UTC offset>-ゾーン名（古いものからの変更）の形式で個別のものがあります（例：03-Russian Standard Time（-1）.reg）。 タイムゾーンを通常とは異なる方法で変更する地域では、これまでに存在しなかった3つの個別のゾーンが作成され、すべての名前は感嘆符で始まります-！07-Kemerovo（0）.reg ,! 08-Chita（-2）.reg ,! 11-UTC11 （-1）.reg-26日目の夜に自動的に移行するように設定し、ゾーンの標準的なものに置き換えて、悪い夢のように忘れるのは理にかなっています。



Microsoftがこの方法で移行するとは主張していません（興味がある人-他の国の動的なdst移行を参照してください、すべてがそこにあることは明らかです）。 XPに関連するものはすべてWindows Server 2003に対応しています（言語に正確です）。 Vista / 7に関連するすべてのものは、Windows Server 2008/2008 R2以降に適している必要があります。



XPでダイナミックDSTをサポートするには、2007タイムゾーンKB931836以降の更新が必要です。2012年に正しく切り替えた場合は更新されます。 Vistaの場合、タイムゾーンのインストールを自動化するtzutil.exeの可用性については、kb2556308の更新プログラムがあるはずです-ほとんどの場合、それもあります。
