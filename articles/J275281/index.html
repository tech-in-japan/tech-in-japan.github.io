負荷の高いプロジェクトの評価システム

ストーリーは、アーキテクチャをやり直さなければならなかった1つのコンテンツプロジェクトに関するものです。 以前は、古典的なランプスキーム（Linux-Apache-MySQL-PHP）が実装されていました。 しかし、訪問者の数は増え続け、すでに1Mホストに近づいていたため、データベースサーバーは対処を停止しました。 まず第一に、私は別のセラックを購入することを申し出ましたが、このセグメントではアフィリエイトプログラムへの変換は非常に低いため、プロジェクト管理は少し垣間見ました。



アーキテクチャを変更し、それでもローテーションとレーティングシステムを厳しくする必要があるとしたら、catにようこそ。



このプロジェクトの特徴は、パイプ（YouTube）などのドナーサイトにあるビデオコンテンツを配信することです。 サイトにはBBコード（特定のHTML）のみを表示する必要があります。 したがって、HTMLをオンザフライで絶えず生成する必要はありませんでしたが、たとえば1日1回など、一定の時間後に行われましたが、1000インプレッション後にローテーションに置き換えられました。 Apacheはnginxに置き換えられ、nginx自体は生成された静的HTMLコンテンツを単に返しました。



サイトに来るたびに、訪問者は何か新しいものを見るはずです。 新しいものは、よくあることですが、よく忘れられている古いものです。 一般的に、ビデオプレビューのローテーションが必要でした（それらについては少し後で）。 いくつかの回転アルゴリズムがあります。 マーケティング担当者の心の洗練度を想像することさえできません。 そのため、最も単純な1つについてのみ説明します。



最初の10スロットには、新しいプレビューのみが挿入されます。 次に、CTRが最大のこのカテゴリの90のプレビューが選択されます。 この用語をよく知らない人は、英語からのクリック可能性の指標です。 クリック率：画像のクリック数とインプレッション数の比率。



ビデオは人気がある可能性がありますが、プレビューは表示できません。 これは、ビデオの最もジューシーな瞬間を座って選択する学生の代わりに、ロボットが座ってランダムに選択されたフレームのプレビューを生成するためです。 そのため、非常に興味深いビデオの評価が下がる可能性があります。 サイトを多様化し、ランダムフレームの効果を均等にするために、ローカル評価が使用されます。1つのビデオから3つのプレビューが生成され、これらもプレビューされます。 自然選択の過程で、最も魅力的な写真が残ります。 投票システムもあります：指を上/指で下しますが、1対1の技術的な実装は回転システムに似ています。



しかし、SEOのおとぎ話を聞くのではなく、それらの詳細を共有するためにここに集まりました。 一般的に、ランプテクノロジー全体がWebサイトジェネレーターに置き換えられました。 Nginxは、静的の戻り値に取り組みました。 CTRの計算を実装するためだけに残ります。



サイト上のビデオの総数は約100Kだったため、永続的なメモリ内ストレージを選択することはかなり可能です。 代替品には、Redis、Aerospike、Tarntoolがあります。



優れた機能とMailRuのフレンドリーなロシア語サポートにより、彼らはTarantoolを選択しました。 MySQLはどこにも行っていません。動画のBBコード、カテゴリと名前のリスト、コンテンツの説明、ウェブサイトの生成に必要なその他の情報を保存し続けています。 しかし、データベースは実際には使用されなかったため、最小限のメモリが彼に割り当てられました。



Tarantoolについてさらに詳しく説明します（以降、T *と呼びます）。 さまざまな記事で彼について多くのことが書かれていますが、セットアップとインストールを省略して、これが実際にどのように適用されるかをお伝えします。



何が何であるかを理解するための少し退屈な理論：T *のすべてのデータはスペースに保存されます：スペース。 これは、SQLのテーブルまたはMongoDbのコレクションに類似しています。 テーブルはドキュメントのコレクションである行で構成されるため、スペースには多くのタプル（MySQLの行のアナログ）が含まれます。



タプルは要素またはフィールドで構成されます。 タプルフィールドの要素を呼び出すのは便利です。この用語を順守します。この用語は、 tarantool.org / doc / book / box / index.htmlのドキュメントに反しません。 テーブル行とは異なり、タプルのフィールドには名前はなく、シリアル番号のみがあります。 ただし、後で見るように、これは重要ではありません。



各タプルには主キーが必要です。 プライマリインデックスは、TREE、HASH、BITSET、またはRTREEのいずれかのタイプです。 また、セカンダリインデックスをスペースに重ねることができます。これにより、大根では作成できないような一意のサンプルを作成できます。
