Windows XPでのRuby 1.8.7とRailsの手動インストール

私は自分自身のために一度書いたが、「誰かが便利になったらどうなるか」と考えた。



したがって、Rubyをインストールします。 これを行うには、次のものが必要です。



  1. Ruby 1.8.7のバイナリをここからダウンロードします： http : //www.ruby-lang.org/en/downloads/ 目的のディレクトリ（たとえば、 c：\ ruby ）に解凍します。
  2. [マイコンピュータ]→[プロパティ]→[詳細設定]→[環境変数]を開き、 パス変数に「 c：\ ruby​​ \ bin \ 」などの行を追加します（バイナリを展開した場所によって異なります）
  3. こちらからzlibパッケージをダウンロードしてください ： http ://www.zlib.net/zlib123-dll.zip それから、 zlib1.dllファイルを取得し、 zlib.dllに名前を変更して、 次の場所に配置します。\ ruby​​ \ bin \
  4. ここからiconvパッケージをダウンロードしてください ： http : //sourceforge.net/project/showfiles.php? group_id=25167&package_id=51458 アーカイブで、 iconv.dllファイルを見つけて、 次のアドレスにコピーします。\ ruby​​ \ bin \
  5. ここからOpenSSLライブラリをダウンロードしてインストールします ： http : //www.slproweb.com/products/Win32OpenSSL.html
  6. ここからrubygemsをダウンロードします： http ://rubyforge.org/frs/?group_id=126 アーカイブを解凍し、 setup.rbファイルを見つけて、コンソールでruby setup.rbコマンドを実行します
次に、Railsをインストールする必要があります。 これを行うには、次のものが必要です。

  1. コンソールでgem install railsコマンドを実行します
  2. コンソールで、 gem install sqlite3-ruby -v 1.2.3コマンドを実行します
  3. ここからSQLite3ライブラリをダウンロードしてください ： http : //www.sqlite.org/download.html ダウンロードしたアーカイブのsqlite3.dllファイルを次のフォルダーに解凍します。\ ruby​​ \ bin \
最後の2つのパンチは、Railsアプリケーションをすぐに実行するために必要です。SQLiteデータベースの使用は、構成で最初に指定されています。



次に、 Railsの動作を確認する必要があります。 これを行うには、次のものが必要です。
  1. コンソールでrails my_projects_name_with_pathコマンドを実行します （例： " rails d：/ my_blog "）-これにより、空のアプリケーションフレームワークが作成されます
  2. フレームワークが作成されたフォルダーに移動し、コンソールでruby script / serverコマンドを実行します。これにより、空のRailsプログラムを実行します
  3. ブラウザでlocalhost：3000に移動します
  4. 開いたページで、リンク「 アプリケーションの環境について 」をクリックします。これにより、インストールされているRailsコンポーネントのバージョンが表示されます。


stackoverflow.comへのRubyのインストールに関する質問に回答してくれた親切な人々に感謝します。これにより、すべての詳細を把握し、この小さなメモガイドを作成することができます。
