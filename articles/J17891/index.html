2010年のWindows 7



Microsoftは、10年の終わり近くにWindowsクライアントオペレーティングシステムの次のバージョンをリリースすることを計画しています...



少なくとも、CNET NewsがMicrosoftから配布された情報を引用していると報告しているように、同社は3年以内に新しいWindows 7ソフトウェアプラットフォームでの作業を完了する予定です。 Windows 7（旧称はウィーン）は、最近リリースされたWindows Vistaに代わるものです。 Windows Vistaのリリース日は繰り返し延期され、OS開発プロセス全体として5年以上かかりました。 Windows 7の登場により、次のバージョンのWindowsのリリーススケジュールがより予測可能になると予想されます。 マイクロソフトは、特に、ソフトウェアプラットフォームの次のバージョンの発表の間、同じ時間間隔を維持する予定です。 Windows 7についてはまだあまり知られていません。 オペレーティングシステムがクライアントバージョンとサーバーバージョンでリリースされ、64ビットと32ビットの両方のプロセッサがサポートされることが報告されています。 マイクロソフトは、企業がWindowsにサブスクリプションモデルを導入することを検討していることを確認しましたが、このOS配布スキームについてはまだ詳細がありません。 マイクロソフトは最近、新しいサーバーオペレーティングシステムであるWindows Server 2008の正確なリリース日を発表しました。ソフトウェアプラットフォームは来年2月27日にロサンゼルスで開催される特別イベントで正式に発表される予定です。 Windows Server 2008と同時に、SQL Server 2008データベースとVisual Studio 2008アプリケーション開発環境が発表されます。

Microsoftは、旧コードネームWindows Viennaであった次のWindows 7オペレーティングシステムが2010年に登場すると発表しました。 さらに、同社は、今後、デスクトップOSの3年間の開発サイクルを順守すると述べた。



ソフトウェアの巨人は、Windows Vistaデスクトップシステムの開発が絶望的に​​遅れ、その前身であるWindows XPが6年以上前にリリースされたという事実により、約1年前に激しい批判の波にさらされたことを思い出してください。



Microsoft Operations Director Kevin Turnerによると、すべての主要なWindowsリリースは3年ごとにリリースされ、特別な場合にはプロセスを4年に延長でき、サーバーWindowsの通常の開発サイクルは4年になり、Microsoft Officeスイートにも同じことが当てはまります。 さらに、最大2年ごとに、企業は、アーキテクチャの変更に影響する、サポートされているオペレーティングシステム用のサポートされている重要なサービスパックをリリースすることを約束しています。



Windows 7に関しては、Windows Vistaと同様に、32ビットと64ビットの2つのエディションでリリースされます。 さらに、基本的に新しいシステム操作をWindows 7に導入することができます-サブスクリプションによって、一定の間隔で更新する必要がありますが、企業はまだこのための具体的な計画がなく、現在の「ボックス化」ライセンスシステムを維持できます。



マイクロソフトの担当者は、企業の発展が遅かれ早かれサブスクリプションライセンスシステムに移行されると長い間繰り返し言っていたことを思い出してください。 企業の観点から見ると、このモデルはより収益性が高くなります。これは、第一に、誰もがOSの使用に対して支払いを行い、第二に、ほとんどの場合毎年支払いを行うためです。 ユーザーにとって、収益性または不利益は支払いの規模に直接依存します。



同時に、同社には、有料サブスクリプションの一部として、会社のさまざまな製品を無料でダウンロードして使用できる顧客がまだ多数います。 これらのクライアントは、原則として、大企業の企業であり、各コピーを支払うよりも1つのサブスクリプションを発行する方が簡単です。 ただし、Forrester Researchの分析会社の調査によると、大規模な顧客の間でも、企業のサブスクリプションシステムはあまり人気がなく、一度購入したほとんどの企業は、ほとんどの場合、更新しません。
