TaxiMap-タクシー送迎サービス

ご挨拶。

タクシーを選択するために設計された「ガレージ」サービスを紹介します。 現時点では、彼はモスクワでのみ働いています。

機能の簡単な説明。 出発地と目的地をマークし、追加のサービスを選択し、あなたを助ける準備ができているタクシーディスパッチャの返品と「テレポート」の費用を受け取ります。



問題の本質



現在、タクシーは2つの方法で注文できます。

  • 会社に電話して、タクシーを注文して行きます。

    難しさ：利便性とコストに関する要件との矛盾、タクシー運転手が目的地までの道を知らない可能性

  • 複数の企業を選択し、それらの関税とサービスを比較し、注文します。

    難しさ：会社を選択するのに多くの時間がかかり、ドライバーは目的地への道を知らない可能性がある



問題解決



TaxiMapサービスを作成しました。これにより、要件に最適な会社を選択できます。



主な機能


  • 市内のタクシー 。

    地図上に出発地と1つ以上の目的地をマークします。 適切なフィールドにアドレスを入力するだけで、タグを配置できます。 その後、必要なサービス（エアコン、チャイルドシートなど）をマークし、[ルートの計算]ボタンをクリックして、会社の発行と旅行の費用を取得します。

    ルートを印刷してタクシーの運転手に渡すこともできます。これにより、街中を簡単に移動できます。

  • 空港へのタクシー 。

    ここでも簡単です。 空港、市街地、旅行の種類、追加サービスを選択します。 [送信]をクリックして、会社と旅行の費用を取得します。



会社名簿


ここにはまだほとんど機能がありません。 計画された革新のためのドラフトの形でより多く存在しています。

これを使用すると、当社のサービスを使用してサービスを提供する企業のプロファイルを表示できます。



タクシー派遣会社の機会


企業は登録してプロフィールを記入すると、当社のサービスを使用して顧客を獲得できます。

現在、すべての企業のプロファイルとそのサービスの費用は当社が記入しています。



開発側で



それでも、IT向けのリソースなので、プロジェクトの技術面についてお話します。

開発時には、PHP + MySQL + Zend Frameworkを使用しました。 デザインはオープンソースを採用しました。

マップを実装するために、Googleマップを使用しました。 将来的には、Yandex Mapsへの移行を除外することはできませんが、現時点ではそれらのルートの実装は湿っています。



コメントでは、当社のサービスの利点と欠点、および新しい機能の提供を確認したいと思います。

ご清聴ありがとうございました。
