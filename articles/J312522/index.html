小さな組織でのGLPI、パート1。典型的な展開と落とし穴

Habréで最初の出版物を書く前に、私は長い間記事のトピックについて考えていました。記事の問題の突破口と完全な報道を望んでいました。 ただし、GLPIの同僚からの質問に再び直面したため、250人の企業のインフラストラクチャのフレームワーク内でのこのシステムの「典型的な」展開に注目し、Habrの読者の幅広いサークルに落とし穴を指摘することにしました。



おそらく、これはGLPIに関する一連の記事の最初のものであり、トピックが必要であることが判明した場合は、このサイクルを喜んで続けます。



GLPIに最初に出会ったのは、システム管理の分野で仕事を始めたばかりです。 このシステムは、私の同僚と前任者から継承されました。 このシステムが組織で実行した唯一の機能は、企業で使用されているコンピューターに関する情報の静的な保存でした。 それだけです



これらの目的のために、GLPIコンプレックスが配置されたマニュアルに基づいて、2011 Ubuntu Linuxの最後の安定したサーバーがインストールされました。これには、GLPIデータベース（データベース）を備えたMYSQLサーバー、Webサーバーが含まれますPHP5およびGLPI自体を備えたApache。



したがって、インストールを進める前に、少なくとも3つの質問をする必要があります。



1. GLPIは組織にどのようなメリットをもたらしますか？

2.このシステムをどのような容量で使用したいですか？

3.このシステムに同行してサポートできますか？



上記の場合、これらの3つの質問に対する回答はありませんでした。



GLPIは、かつての「無料のコンピューター機器のフリートマネージャー」ではなく、ITインフラストラクチャの管理、ユーザーのサポート、予算の計画、部門の管理などを支援するハーベスターです。 GLPI機能はアドオンを大幅に拡張します。アドオンは使いやすく、豊富な機能を備えています。



GLPIを企業に展開するとき、私は常に上記の質問を自問し、GLPI展開の典型的な状況を作り上げたようです。



1。

-GLPIはオフィス機器の在庫管理を支援し、古いコンピューター機器の検出、整理、交換を可能にします。これは生産性の向上を意味します。

-GLPIを使用すると、技術サポートのためにアプリケーションのステータスを体系化、グループ化、優先順位付け、追跡できるため、重要かつ優先順位の高いアプリケーションの実装が高速化され、労働生産性が向上します。



2。

-GLPIは、「コンピューターパークマネージャー」としておなじみの役割で使用されます。

-GLPIは、テクニカルユーザーサポートのシステムの役割を引き受けます。

-GLPIは、IT部門（予算、機器サプライヤー、サービスプロバイダーなど）の管理に役立ちます。



3。

-企業はすでにGNU / Linuxにデプロイされた多くのシステム（Zabbix、Openfire、内部Webサイト、上記システム用のMYSQLデータベースサーバー）を使用しており、1台のサーバーを追加してもインフラストラクチャを根本的に複雑にすることはありません。



Unixライクなオペレーティングシステムにシステムを展開していない場合は、基本的な管理レベルで必要な知識を得る必要があります。



Unixライクなオペレーティングシステムを数年間使用してきたので、GLPIの場合、Debian GNU / Linuxの信頼性はGLPIの完全な機能を長年にわたって心配しないのに十分であると安心できます。 ただし、重要な更新とセキュリティ更新を忘れずに、定期的にバックアップを作成してください。



以下に、謙虚な意見で、典型的なGLPI展開の基本的なシステム構成とアドオンを示します。



まず 、インストールの準備をする前に、インフラストラクチャ全体がGLPIと互換性を持つように、すべてを計画し、必要なGLPIのバージョンとその追加を決定する必要があります。



第二に 、Debian GNU / Linuxを搭載した2つの仮想マシン（VM）が必要です。 1つはMYSQL用、もう1つはGLPI用です。



MYSQLサーバーの機能に関連してMYSQLデータベースを使用してサーバーの役割を割り当てることを好みます。たとえば、Zabbixなどの他のプロジェクトのデータベースをホストするのに役立つ場合もあります。 この構成を使用する場合、PHPMyAdminを使用してMYSQLデータベースを管理するのが非常に便利です。このWebインターフェイスをインストールすることをお勧めします。



最初の落とし穴：他のホストからMYSQLデータベースサーバーに接続する機能を開くことを忘れないでください。 これは非常に簡単です。 ファイル/etc/mysql/my.cnfを修正する必要があります



すべてのアドレスを聞くことができるようにするには、次の行をコメントアウトします

バインドアドレス= 127.0.0.1



詳細はこちらをご覧ください 。


データベースのサイズと数が増加するデータベースサーバーである場合、MYSQLデータベースを備えたVM、およびGLPIフォルダーのサイズが増加すると予想されるWebサーバーである場合、GLPIを備えたVMを構成する必要があります。 両方のVMには、Apache WebサーバーとPHP5のインストールが必要です。



MYSQLデータベースを使用するVMでは、MYSQLデータベースを便利に管理するためにPHPMyAdminを追加インストールする必要があることを思い出してください。 そのようなシステムをインストールするためのステップバイステップの完全なマニュアルがインターネット上にたくさんあります;それらを見つけることは問題ではなく、これは別の記事のトピックです。



また、 GLPI用のサーバーにmysqlクライアント（ mysqlclient ）をインストールすることを忘れないでください。データベースサーバーとの対話に役立ちます。



第三に 、不要な依存関係やプログラムからの「スチームエンジン」なしでクリーンインストールが可能になるため、リポジトリからではなく、GLPI WebサイトからGLPIをインストールすることをお勧めします。 さらに、このサイトでは、常に最新バージョンのGLPIを見つけるか、古いリリースから何かを選択できます。



2番目の落とし穴：このタイプのインストールでは、Webサーバー（www-data）の権限をGLPIが展開されたフォルダーに与えることを忘れないでください。DebianGNU / Linuxに含まれているchownユーティリティを使用してこれを行うのが最も便利です。



また、/ etc / apache2 / sites-availableフォルダーにファイル（glpi.confなど）を作成して、Apache2構成ファイルを修正することを忘れないでください。 これを行うには、デフォルト設定-000-default.confファイルをコピーする方が簡単です



新しいファイルでは、パラメータServerName、ServerAlias、およびDocumentRootを修正し、解凍されたGLPIのサイト名、エイリアス、および場所を指定します。 「tp」や「glpi」などの短いアドレスを指定すると便利です。



コマンドa2ensite glpi.confで新しいサイトを追加した後、Apacheを再起動する必要があります。


GLPIが完全に起動するまで、デフォルトの構成ファイルを削除してデフォルトのサイトをオフにすることはお勧めしません。これは、将来のPHPパラメーターの表示と調整をより便利にするのに役立ちます。



第4に、サーバー上にGLPIのデータベースとユーザーを作成し、それに権限を与えることが残っています。このため、PHPMyAdminを使用すると便利です。



そのため、データベースが作成され、ユーザーはデータベースの管理者権限を受け取りました。



3番目の落とし穴：GLPIを完全に動作させるには、* / glpi / config / config_db.phpのファイルを修正する必要があります



サーバーのアドレスをMYSQLデータベース、データベース名、ユーザー名、パスワードで指定して、すべてのフィールドに入力する必要があります。 最後の仕上げ-GLPIを使用するには、データベースをマークアップする必要があります。 mysqlクライアントが役に立つのは、この手順のためです。 コマンドを実行し、ダンプを新しいデータベースにダンプします。



mysql -u username -h DB_server_name_ip_name -p password DB_name <* / glpi / install / mysql / glpi- GLPI_version -empty.sql



GLPIを別の場所（たとえば、ユーザーフォルダ/ home / user1または/ usr / share /に配置できるため、不完全なパスを示したことに注意してください。/var/www/に配置することをお勧めします）。 パス* / glpi / install / mysql /に、入力可能なすべてのダンプがあります。


5番目に、GLPI Webインターフェイスに簡単かつ簡単にアクセスできるように、できればServerNameと一致する短い名前で、GLPIなどのDNSでAレコードを作成することをお勧めします。 ブラウザにglpiと入力し、GLPIのセットアップに進みます。



第六に 、追加についてのいくつかの言葉。 「典型的な」インストールの場合、以下を使用すると便利です。



-Archires（ネットワークアーキテクチャ）（ネットワークインフラストラクチャを明確に記述する機能）

-ダッシュボード（上司向けの美しいグラフィックレポート）

-FusionInventory（PCインベントリ）

-レポート（自分にとって便利なレポート）

-PDF（PDFへのエクスポートが可能）

-ラック/ベイ管理（ラックとキャビネットを説明するための便利なツール）

-アカウントインベントリ（ユーザーアカウントを操作するための便利なツール）



ここからアドオンをダウンロードします 。 ダウンロード後、ファイルを解凍し、* / glpi / pluginsフォルダーに配置する必要があります。その後、GLPI Webインターフェースでそれらをアクティブにして有効にする必要があります。 GLPIのインストールが完了しました。
