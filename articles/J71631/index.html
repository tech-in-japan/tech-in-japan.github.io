MozillaはGoogle Chrome Frameを承認しません

ええ、Mozillaは本当に興奮しました。 議論が実際に正当化される方法は明らかではありません。 もちろん、ほとんどのユーザーはもちろん[はい]をクリックします！ 何かをインストールするように求められたとき。 Chrome Frameが何らかの形で彼らの脳を完全に破壊しているとは思えません。 これには何らかの理由があるかもしれませんが...

以下は、わずかに短縮された翻訳です。



MozillaはGoogle IEに対してMicrosoftに味方した

あまり知られていないウェブ



MozillaはMicrosoftに入社し、Internet ExplorerをGoogle Chromeに変える新しいGoogleプラグインをリリースする知恵に疑問を呈しています。 しかし、レドモンドとは異なり、オープンソース企業は根拠のある議論を提示しました。



先週、GoogleはレンダリングとJavaScriptメカニズムでInternet Explorerを補完するプラグインをリリースしました。 Google Chrome Frameとして知られるこのプラグインは、JavaScript処理を大幅に高速化し、MicrosoftにHTML5機能を備えた市場で支配的なブラウザーを提供します。



その後すぐに、Mozillaの開発担当副社長であるマイクシェーバーは、Googleのように、IEがChrome、Safari、Firefox、Operaと同様に機能する世界を夢見ているとブログに書きました。 しかし彼は、Chrome Frameはこれを達成するための間違った方法であると固く信じています。



「IEでChrome Frameを有効にすると、多くのブラウザー機能の機能が妨げられたり、効率が低下したりします。これは、コンテンツ領域で機能する機密Webサーフィンやその他のセキュリティツール、アクセラレーターと拡張機能のモード、さらにはアクセシビリティのサポートに関係します」



シェーバーの声明は、Mozillaの代表者からの2番目の投稿によってサポートされました。 Mozilla CEOのMitchell Bakerは、Googleプラグインを組み合わせて使用​​するのではなく、Webをクラックすることを検討しました。 「一般的に、Chrome Frameを使用すると望ましくない結果が生じるでしょう」と彼女は書いています。 「プラスの効果は短命であり、配信の程度に応じて、Chrome Frameは最終的に断片化を増加させ、Web開発者を含む私たち全員の管理性を低下させると予測しています。」



MicrosoftがついにHTML5に真剣に移行し始めたという事実にもかかわらず、IEは提案された標準をまだ受け入れていません。 Googleにとって、このような競合他社の遅さは問題です。同社はHTML5に大きく基づいている Google Waveの予備バージョンへのアクセスを拡大する予定です。



Internet ExplorerユーザーがGoogle Waveトライアルリリースにアクセスする場合、Chrome Frameをインストールするか、別のブラウザーを使用することをお勧めします。 さらに、Googleは、他の開発者がこれに関与することを奨励しており、WebアプリケーションもChrome Frameで実行されます。



マイクロソフトはプラグインに予想通り否定的に反応しましたが、企業は特定の詳細よりも多くのFUDがあると述べました。 「プラグイン、特にGoogle Chromeのセキュリティ問題を考慮すると、Google Chrome Frameを使用するとブラウザのマルウェア認識が2倍になります。 友人たちにこのようなリスクを負うことはお勧めしません」とメッセージは述べています。



Mozillaのマイクシェーバーは脅迫戦術を避け、代わりにプラグインに対する否定的な評価は、Chrome FrameがInternet Explorerに組み込まれたセキュリティ機能を回避するだけでなく、セキュリティについてユーザーを混乱させると主張しています。 「プラグインを使用することの副作用は、ユーザーが訪問するサイトの開発者にソフトウェアの選択肢を委任することにより、Webセキュリティモデルとブラウザーに対するユーザーの理解が損なわれることです」とシェーバーは言います。 「Flash、Silverlight、およびJavaプラグインの場合、常にこの問題に直面していました。HTML5ブランドの下でこれを繰り返す必要はないと思います。」



Mitchell Bakerは、問題はさらに進むと考えています。「Chrome Frameを使用してサイトにアクセスすると、パスワード、セキュリティ設定、パーソナライズ、およびユーザーがブラウザに設定するその他の処理方法が突然不明になります。 たとえば、あるレンダリングエンジンで作業中にブックマークしたサイトは別のレンダリングエンジンに表示されますか？ さまざまなブラウザー要素は相互に関連しなくなっているため、現在使用していると思われるブラウザーでのアクションの結果は、ブラウザー内ブラウザーChromeの実際の使用により、実際に得られるものと大きく異なる場合があります」



その後、ミッチェルは世界のほぼ終末論的な絵を描きます。他の人はグーグルの例に従います。「一部のサイトはグーグルのブラウザインブラウザを使用し、FacebookはそのブラウザインブラウザをFacebook Connectサイト、Apple iTunesのオプションであるWAPサイトの場合、モバイルオペレーターは独自の何かを発明し、これらはすべて単一のソフトウェアに実装され、ユーザーは単純にブラウザーと見なします。 その結果、ブラウザーから何らかの混乱が発生し、その中で個々のユーザーアクションは何らかの応答に対応しますが、結果のみが予測不能になります。 これにより、Webの理解度と認識度が低くなり、確かに管理しにくくなります。



最初のコメントでは、Googleは少なくともプラグイン自体のセキュリティを保護しています。 「Google Chrome Frameを使用することで、IEユーザーはGoogle Chromeで利用できるセキュリティレベルを利用できるようになり、フィッシングやマルウェア（IE6にはない）に対する信頼性の高い保護と、間近に迫っている新しいオンライン脅威を提供します」



しかし、Mozillaの場合、問題はプラグインのセキュリティではなく、プラグイン自体です。 Mozillaによれば、MicrosoftのブラウザーをGoogleのブラウザーに変えることは最初は悪い考えです。



しかし、オープンソース開発者がマイクロソフトに味方したと真剣に言うのは誇張だろう。 「Chrome Frameを使用したい開発者にとっては、Chromeでサイトの動作が向上することをユーザーに伝え、インストール手順を提供するだけで、Web全体に適しています」とShaver氏は言います。 「このようにして、ユーザーは代替ブラウザの利点について学び、選択をよりよく理解し、MicrosoftではなくGoogleがChromeのパフォーマンスを認識するに値します。」
