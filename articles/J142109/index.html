Red HatがOpenStackコミュニティに参加



エンタープライズLinuxディストリビューション、ミドルウェア、仮想化、クラウドコンピューティングのメーカーであるRed Hatは、ついにOpenStackコミュニティに参加しました。 これは、オープンクラウドプラットフォームであるOpenStack Foundationの開発を管理する組織の形成とともに起こりました。





2010年7月にOpenStackプロジェクトが開始されて以来、NASAとRackspaceは開発プロセスを大幅に管理し、OpenSack OpenStackプロジェクトに熱心に参加した他の企業を悩ませました（そして既に150以上あります）。ただし、OpenStack開発管理プロセスにはある程度参加しています。



NASAはOpenStackに積極的に影響を与えようとはしなかったため、Rackspaceに管理を引き渡しました。Rackspaceは、通常の商用ベンダーと同様に、プロジェクトの制御をできるだけ長く維持したいと考えていました。



ただし、オープンソースプロジェクトを成長させたい場合は、遅かれ早かれ手放さなければなりません。 Rackspaceのビジネス開発およびマーケティング担当バイスプレジデントであり、OpenStackの創設者の1人であるMark Collierは、プロジェクトを管理する独立組織であるOpenStack Foundationの作成の進捗状況について話しました。



Kollerによると、彼とJonathan Bryceは昨年10月から、Apache Software Foundation、Eclipse Foundation、Linux Foundationなどのさまざまな類似組織の研究に取り組んでいます。 彼らは、OpenStackをプロジェクトの開発に集中させ、その資金を確保する方法を見つけたかったのです。



Koller氏によると、主な目標は「技術的能力主義によって推進されるオープンな開発プロセス」を提供するとともに、コミュニティに投資してOpenStackに注意を引き、OpenStackをサポートおよび使用して収益を上げるサービスプロバイダーと企業のエコシステムの開発を促進することです。



3月に、将来の組織の構造が決定され、プラチナおよびゴールドのスポンサーが見つかりました。ニュースは、Red HatがAT＆T、Canonical、HP、IBM、Nebula、Rackspace、およびSUSE LinuxとともにOpenStackプラチナスポンサーになる意向書に署名したことです。 ゴールドスポンサーには、Cisco Systems、Clearpath Networks、Cloudscaling、Dell、DreamHost、ITR、Mirantis、Morphlabs、NetApp、Piston Cloud Computing、Yahoo！が含まれます。



Kollerによると、OpenStack Foundationは現在、組織の法的文書を作成する委員会を編成中です。



「いつかクラウドコンピューティングが世界経済の中心になることはそれほど素晴らしいことではないと思います。それは、今、多くのことが危機にtakeしていることを意味します」とコラーは書いています。 OpenStackの開発において、オープンな開発モデルの必要性を確信している人にとって、これはOpenStackに基づくオープンな未来になると確信しています。



もちろん、上記の19のスポンサーの数に含まれていなかった企業の1つ、Citrix Systems。 先週、彼女はライバルのCloudStackオープンソースクラウドプロジェクトを立ち上げ、Apache Software Foundationインキュベーターに移し、OpenStackに先んじて、支配的で完全にオープンなクラウドプラットフォームになることを望んでいました。 Citrixにとっては非常に賢明な動きでしたが、2年前にNASAとRackspaceが同じことをしなかったのはなぜですか？ 特に、OpenStackがApacheライセンスの下でライセンスされていることを考慮してください。



たぶん答えは、RackspaceがOpenStackプロジェクトの制御を維持したかったということです。これは、Amazon EC2およびS3と互換性のあるAPIのサポートに関して明確に見られます。 Rackspaceはこれに断固として反対しており、おそらくRed Hatにも反対しています。 また、Citrixは、Amazon APIサポートが望ましいだけでなく、重要でもあると考えています。 そして、それがCitrixがOpenStackに背を向けた理由かもしれません。



Red Hat は 、OpenStackコミュニティに参加する主な理由は、プロジェクト管理が独立した組織の手に渡ったことであり、現在、Red HatはOpenStackの商用サポートを提供する予定であると述べました。



OpenStackと他のRed Hatクラウドソフトウェア（OpenShift PaaSプラットフォームとCloudFormsプライベートクラウドインフラストラクチャツール）がどのように共存するのだろうか。 OpenStackがOpenShiftおよびCloudFormsに最適であっても、私は驚かないでしょう。 これらのクラウド製品はまだオープンソースでリリースされておらず、Red Hatではコアにどのようなコードがあるのか​​については決して話しませんでした。



Red HatのCTOであるBrian Stevens氏は次のように述べています。「これから数か月以内に、これがパートナーと顧客にとって何を意味するかについて、より詳細な情報を共有します。そして今日、RackspaceとNASAのおかげで、この未来はずっと近づいています。」
