Ubuntu GutsyとMCP51（snd-hda-intel）-サウンドにするために何をすべきか

私だけでなく、MCP51チップ（ほとんどの*でsnd-hda-intelとして定義されている）にサウンドが組み込まれている母親に新しくインストールされたUbuntu Gutsyのほとんどにも、サウンドに問題があるようです-カードは単に検出できません。



まず、私が使用したソースはいくつかありますが、主に2つのソースがあります。1つ目はUbuntu Answers 、2つ目はHda Intel Sound How toです。



第二に、以下で説明するすべては、システム全体、特にカーネルまたはパッケージに対する大きな変更のように見えますが、これは新しい機能を追加するだけですが、実際に試しても壊れないでしょう（狂信的で、お願いします！:)。



第三に、BIOSでサウンドをオンにすることを忘れないでください！ 私はそれを踏んで少し時間をつぶしましたが、十分です:)



順番に：

1）システムでalsa'yをビルドするために必要なパッケージを追加します。

$ sudo apt-get install build-essential ncurses-dev gettext

$ sudo apt-get install linux-headers-`uname -r`




3）alsaのサイトからドライバー 、 ライブラリ 、 ユーティリティの最新バージョンをダウンロードします。

$ sudo mkdir -p / usr / src / alsa

$ cd / usr / src / alsa



$ wget ftp.alsa-project.org/pub/driver/alsa-driver-1.0.15rc3.tar.bz2

$ wget ftp.alsa-project.org/pub/lib/alsa-lib-1.0.15rc3.tar.bz2

$ wget ftp.alsa-project.org/pub/utils/alsa-utils-1.0.15rc1.tar.bz2



$ sudo tar xjf alsa-driver * .bz2

$ sudo tar xjf alsa-lib * .tar.bz2

$ sudo tar xjf alsa-utils * .tar.bz2




4）ドライバーをコンパイルしてインストールします（キーに注意してください！）、ライブラリーおよびユーティリティー：

$ cd ../alsa-driver-1.0.15rc3

$ sudo ./configure --with-cards = hda-intel

$ sudo make

$ sudo make install



$ cd ../alsa-lib-1.0.15rc3

$ sudo ./configure

$ sudo make

$ sudo make install



$ cd ../alsa-utils-1.0.15rc1

$ sudo ./configure

$ sudo make

$ sudo make install




5）再起動します。



6）再起動後、/ procにカードが表示されるはずです。

$ sudo cat / proc / asound / card0 / codec＃* | grepコーデック

コーデック：Analog Devices AD1986A




7）このリンクをたどり、カードを探して、設定するモードを確認します。 私の場合、3stackです。



8）alsa configに1行追加します：

$ sudo vim /etc/modprobe.d/alsa-base



私は

オプションsnd-hda-intel position_fix = 1 model = 3stack

：

wq




9）再度リブートします。



10）alsaを構成します。

$ sudo alsaconf


カードはここにあります。たとえば、私の場合はhda-intelです



11）設定を確認します。

$ sudo aplay --list-devices

****再生ハードウェアデバイスのリスト****

カード0：NVidia [HDA NVidia]、デバイス0：AD198xアナログ[AD198xアナログ]

サブデバイス：1/1

サブデバイス＃0：サブデバイス＃0

カード0：NVidia [HDA NVidia]、デバイス1：AD198x Digital [AD198x Digital]

サブデバイス：1/1

サブデバイス＃0：サブデバイス＃0




12）最後に、どのように再生しますか：

$ aplay -vv ./alsa-utils-1.0.15rc1/speaker-test/samples/Noise.wav




上記の方法は、3台の同一のマシンで正常にテストされましたが、Feisty、Gutsy、およびEdgyの異なるubuntでテストされました。
