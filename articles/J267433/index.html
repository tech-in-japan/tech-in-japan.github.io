屋内ナビゲーションとポジショニング-駐車場で車を「紛失」したことがありますか？

店の1つですでに見たものを探してモールを歩き回るのに何時間も費やさなければならなかったが、どこを正確に思い出せないのか？ または、博物館で最も興味深い展示を検索しますか？ ショッピングセンターを出て、長い間駐車場で車を探していた状況をよく知っていますか？



ドバイの複数階の駐車場で車を「どうにかして」紛失し、40度の暑さの中で約2時間検索しました。 私はすでに盗難を報告するために警察に行きましたが、その瞬間に偶然それを見つけました。



または、たとえば、博物館-サンクトペテルブルク郊外のグランドガッチナ宮殿 。











複数のフロアにある膨大な数のホール。 迷宮のような迷路のようなダンジョンに足を踏み入れることはできますが、それでは絶対に独り立ちすることはできません。



たとえば、地下鉄のダンジョンと言えば、人々は常にカートやスーツケースと出会い、周りを見回して、自分がどこにいてどこに行くのかを把握しようとします。



人々が地域をナビゲートできるように、新しい方法を考え出します。 最もシンプルで技術的に高度なものの1つは、屋内のナビゲーションとポジショニングです。



この記事では、屋内ナビゲーションを専門とするNavigine社の例を使用して、小規模の開発会社が特定のソリューションをSAP製品と統合する方法を説明します。 屋内サービス市場は数年間活発に開発されてきましたが、 SAP HANAプラットフォームでのNavigineソリューションはロシア市場で進歩しています。



このソリューションは何を許可し、どのように、どのように役立つのか、 SAP HANAプラットフォームではなぜですか？ これについては後ほど、そして初心者向けに-現代のスマートフォンで実践できる主な技術に関する簡単なリファレンス。



GPS-誰もが知っており、すべてのスマートフォンに搭載されていますが、建物内で許容できる精度で動作するわけではありません。 GPS信号リピーターをインストールできますが、ソリューションのコストは非常に高くなります。



セルラーネットワークを介したポジショニング -基地局の密度が高いエリアでも、精度には多くのことが求められます。



慣性システムは出荷と同じくらい古いです。 彼らは人間の動きのモデルを使用します：私たちがどこにいて、どのように、そしてどれだけ速く動いたかを知っていれば、しばらくしてどこに行き着いたかを計算できます。 現在、これはスマートフォンのジャイロスコープと加速度計の助けを借りて達成されています。 このソリューションの利点には、自律性と外部インフラストラクチャへのリンクの欠如が含まれます。 短所-開始点を知る必要性、経時的な累積エラー、およびGPSなどの別のデータソースで確認する必要性。



光学システム -以前に施設をスキャンし、写真から位置を特定したスタートアップがいくつかありました。たとえば、スマートフォンのフロントカメラの天井です。 おもしろいですが、部屋を撮影する必要があり、まだ本格的な実装に遭遇していません。



磁力測定-スマートフォンのコンパスを使用した磁場上のナビゲーション。 成功したアプリケーションの例がありますが、解決策は部屋での予備キャリブレーションを必要とし、金属と磁石の影響を受けやすくなります。



Wi-Fi / Bluetoothトランスミッターに基づく三辺測量 。 長所：インフラストラクチャとポジショニングの両方に、使い慣れた安価で普及している機器を使用します（スマートフォンには必要なものがすべて揃っています）。 既に展開されているWi-Fi / Bluetoothネットワークを使用する機能。 主な欠点は、約10メートルの精度です。



無線カードまたはWi-Fi / Bluetooth信号の「デジタルフィンガープリント」 -位置は、周囲のWi-Fi / BLEポイントからリアルタイムで測定された信号強度とルームマップに関連付けられた事前測定値を比較することで計算されます。 プラスは、三辺測量の場合と同じです。 短所-約5メートルの精度、システムの事前トレーニングが必要。



さらに多くのテクノロジーとアプローチがあります：RFID、ZigBee、IR、および超音波システムですが、それらはすべて、さまざまな理由で、スマートフォンをベースにした屋内ナビゲーションでの応用を発見していません。



以下では、これらのどれがNavigineを使用するかを説明します。



次にアプリケーションについて説明します。 ユーザーの観点から、および企業の観点から-2つのケースを検討してください。



一般ユーザーについて話すと、システムは人々が迷子になったり、特定の製品を探したいショッピングセンターなどの大きな建物内を移動するのに役立ちます。







原則はシンプルです。ショッピングセンターへの訪問者は、移動可能なモバイルアプリケーションを使用して、







彼らが購入したい必要な店や商品を見つけ、すぐに駐車場で車を見つけます。







このソリューションは、ショッピングセンター自体に役立ちます。 まず、これは分析です。訪問者がどこに行ったのか、どのルートを訪れたのか、店舗で過ごした時間の長さ、フードコートに座った時間の長さです。 第二に、訪問者とのマーケティングコミュニケーションの追加のモバイルチャネルをテナントに提供する機会です。 典型的なインタラクション：ストアには割引があり、訪問者はショッピングセンターに入ってから、スマートフォンで通知の形でそれを受け取ります。 興味がある場合は、[ルートを検索]をクリックしてこの店に行きます。 ショッピングセンターへの訪問の分析とロイヤルティプログラムのデータベースからの情報にアクセスできるため、テナントは個別の割引を提供し、拒否できないオファーを提供できます。



このように、ショッピングセンターで働くテナントにとっては、カフェやショップへの訪問者の追加の流れを引き付け、忠実な顧客ベースを拡大し、平均チェックを増やす機会です。



企業に関しては、従業員の居場所、機器の移動、荷物の所在などを追跡するタスクにしばしば直面します。 このため、労働の規律と職場での安全性を高め、機器のメンテナンスまたは修理をスピードアップし（従業員は注意を必要とするオブジェクト間の最適なルートに沿って移動します）、最も効率的な方法で倉庫内の機器の移動を整理することができます。







言葉から練習へ。 Navigineソリューションは、数千人の参加者を集めたSAPフォーラムで実証されました。



SAPフォーラム用に作成されたモバイルアプリケーションにより、ゲストは、フォーラムに参加している企業のスタンドに関して、部屋の地図上で自分の位置を確認できます。 さらに、ロイヤルティシステムがアプリケーションに追加されました。 それは、セッションやスタンドに参加するゲストが、ポイントを設定することで品質を評価できるという事実にありました。 評価ごとにポイントを受け取り、さらにポイントがSAPからのギフトに変換されました。 私たちのアルゴリズムは、人が1つの場所に立って、すべてのセッションにいるふりをして、それらを評価する状況を提供し、除外しました。 システムはそれがどこにあるかをよく知っていました。



モバイルアプリケーションを使用して、フォーラム訪問者の移動ルートの収集も実行され、続いてクラウドに送信されました。 クラウドから、この情報はヒートマップの形でWebインターフェイスにロードされました。



ヒートマップは、マップ上で最も訪問された場所を示す建物のマップです。







どのように適用できますか？ たとえば、同じショッピングセンターでは、特定の店舗を訪れる訪問者のトラフィックを簡単に判断できます。 この情報を使用して、ショッピングセンターはレンタル料金でショップと交渉できます。 たとえば、店Aが入って来て、家賃を下げるように頼みながら、訪問する人が少なくなったと主張します。 そして、ショッピングセンターの代表者は、「同じ数の人があなたの店と同じセグメントで営業している店Bを通り抜けます。Bはそれらを正しく変換する方法を知っているだけですが、あなたはそうではありません。」







これは、担当者の作業を整理する際にも使用できます。 多くのトラフィックを持つオブジェクトがあります。そのオブジェクトには時々「ボトルネック」があります。 空港に行きます。 交通渋滞が発生しないように適切な数の人員を適切な時間に設定することが重要な、たとえばパスポートコントロールや乗客の個人検査ポイントなど、そのような場所がいくつかあります。 ヒートマップを使用すると、このようなプラグは追跡が簡単で、すぐに削除できます。







SAPフォーラムに戻ります。 パビリオンでナビゲーションを機能させるために、Bluetooth LEビーコンを設定します。







彼らは自律的であり、2〜3年間働くことができるバッテリーから最大70メートルの距離で信号を送信します。 彼らは内部メモリを持っていません、彼らはスマートフォンに接続していません、彼らの機能はIDをブロードキャストすることです。 スマートフォンが複数のビーコンから信号を受信し、その強度を知っている場合、スマートフォンがどこにあるかをある程度の精度で判断できます。 Bluetooth LEビーコンに加えて、スマートフォンの内部センサーも使用されます-ジャイロスコープ、加速度計、気圧計、コンパス。



「当社のコアコンピタンスは、異種ナビゲーション情報を処理するためのアルゴリズムです。 ビーコンからの信号が取得され、スマートフォンのセンサーからデータが取得され、これがすべて処理されます-建物内のスマートフォンの正確な座標が取得されます。 精度は1〜3メートルで、Bluetoothの設置密度は1,000平方メートルあたり10ビーコンです」とNavigineのディレクターであるAlexey Panevは言います。



技術的には、システム全体は次のように機能します：Android用のNavigineアプリケーションがあります。必要な精度に応じて、無線カードを測定するか（上記の「デジタルフィンガープリント」）、ビーコンの位置が地図に表示されます（三辺測量用）。 すべてのビーコンをインストールした後、それらの場所はサーバーに保存されます。 AppleはWi-Fi APIへのアクセスを閉鎖しているため、より汎用的でインストールが簡単なソリューションとしてBluetooth LEビーコンを優先的に設定します。ソケットとインターネットは不要です。



以下の情報がナビゲーションアルゴリズムに入力されます。



1.無線フィールドの利用可能な信号（Bluetooth LE、Wi-Fi）;

2.スマートフォンの内部センサーのデータ（MEMS加速度計、ジャイロスコープ、磁力計）;

3.部屋の地図に関する情報。 ここでは、壁、階段、障害物、ドア、その他のオブジェクトの位置が重要です。



次に、秘密のソースが特別なナビゲーション処理アルゴリズムの形式で使用され、出力はマップ上のポイントです。 理想的には、位置精度は1 m（1σ）です。



クロスプラットフォームを確保するために、C ++で書かれたアルゴリズムは、AndroidおよびiOS SDKでネイティブにラップされます（将来、およびWindows 10で）。



すべての計算はスマートフォンで直接行われるため、場所を特定する際の遅延が減少します。



では、なぜサービスの実装にSAP HANAが選ばれたのですか？



屋内ナビゲーションおよび測位サービスは、主に大企業（空港、ショッピングセンター、工場、倉庫）に適用されます。 したがって、このソリューションには潜在的な顧客が多数存在し、多くの場合、同じプラットフォームで統合が行われるため、実装コストが最適化されます。



SAP HANAの利点は、私たちが自分でテストすることに成功しました。



1. Amazon CloudでのSAP HANAインスタンスの簡単なセットアップ;

2.わかりやすいオンラインコースを含む、開発者向けの完全なドキュメントセット。

3. SAP HANA環境へのJavaアプリケーションコードの単純な移植。

4. Eclipseを介して作業するための便利なツール。



基本バージョンでは、すべてのデータはクラウドサーバーに転送され、既にデータを処理しています。 SAPフォーラムでのソリューションアーキテクチャは次のとおりでした。



1.組み込みのNavigine SDKを使用したSAPフォーラムモバイルアプリケーションは、ユーザーの位置データと、ユーザーがマークしたスタンドとレポートをサーバーに送信しました。

2.データはSAP HANAに記録され、サーバーはAmazonクラウドに配置されました。

3.収集されたデータに基づいて、SAP HANAで実行されているJava Webアプリケーションは、フォーラムへの参加のヒートマップをオンラインで作成しました。



フォーラムのわずか1日で、498の異なるユーザーのトラックが収集されました。



したがって、SAPフォーラムは、SAP HANAプラットフォームに基づくNavigineプラットフォームの機能と分析ツールが示されたデモプラットフォームになりました。 フォーラムでは、このようなサービスは、ショッピングセンター、空港、小売企業、美術館、倉庫、工場など、さまざまな顧客のケースで使用できることが示されました。 SAPとの相互作用により、革新的なソリューションを開発している中小企業は、法人顧客向けのまったく新しい大規模市場にアクセスできます。
