Windows 8-開発者向けバージョン

今日は、Windowsチームにとって重要な日でした。 BUILDカンファレンスでは、新しいWindows 8オペレーティングシステムが紹介され、イベントはhttp://buildwindows.comで放送されました（録画はそこで見つけることができます）。







開発者向けの今週のテーマ。 今週、アプリケーションとAPIを構築するための新しいツールが発表されます。 BUILDカンファレンスで、彼らはWindowsの新しいバージョンが、タブレット、ラップトップ、デスクトップコンピューターなどのさまざまなデバイスに適していることを示しました。 HTML + Javascript、C / C ++および/またはC＃/ XAMLを使用してMetroスタイルアプリケーションを作成できる新しいツールが示されています。 これらのプログラミング言語はどれでも使用できます-常にOSに適合します。 Windows 8は、エンドユーザーと開発者の両方にとって便利です。



多くは、ARMプロセッサ上のWindows 8に興味を持っています。 会議では、ARMプロセッサに基づいたデバイスが展示されました。 しかし、新しい開発ツールを使用すると、Metroスタイルでアプリケーションを作成できます。これは、ARMとx86（32ビットと64ビット）の両方で同等に実行されます。 アプリケーション開発にネイティブC / C ++を選択した場合でも、特定のデバイスに集中できます。



あなたはおそらくプレビュー版を感じるのを待つことができないでしょう。 すぐに、おそらく9月14日（MSC）の午前中に、Windows 8 Developer Previewをダウンロードできるようになります。 バージョンがあります：開発ツール付きの64ビットバージョンと、開発ツールなしの32/64ビットバージョン。 また、アプリケーションの例（OSにバンドルできる潜在的なアプリケーションのデモ）も含まれます。 ISOイメージは、新しいデベロッパーセンターが起動するとすぐに、 http：//dev.windows.comに投稿されます 。



プレリリースバージョンでは、Windows 7からのアップグレードは許可されません。クリーンインストールのみが可能です。 注：このバージョンは開発者専用であり、常習的な使用を意図したものではありません。 これはベータ版ではありません。 マイクロソフトは、更新メカニズムをテストするために、今後数週間/数か月にわたって更新プログラムをリリースする予定です。



注：このバージョンでは、BUILD会議で示されるすべての機能が使用されるわけではありません。このアセンブリの主な注意は、Metroスタイルでアプリケーションを作成するためのAPIと開発者ツールに向けられています。
