Time Machineバックアップ暗号化

Time Machineでのバックアップのセキュリティが心配な人には、解決策があります。

以下で説明する方法は、以下を使用する場合に機能します。

-Time MachineとTime Capsuleのペアリング。

-HFS +でフォーマットされ、USB経由でAirMacに接続された外部ドライブと組み合わせたTime Machine。

確かに他のトリックも機能します。主なことは、AFPプロトコルを使用してネットワーク上でディスクを共有することです。



そのため、手順は次のとおりです。

1.ターミナルを開きます（アプリケーション-ユーティリティ-ターミナル）。



2.完了したusername.sparsebundleタイプのTime Machineバックアップをコピーします（便宜上、ドライブのルートに）。

2.1。 信頼性を確保するため、不適切なアクションが発生した場合にデータが失われないように、メディアからバックアップコピーを作成することをお勧めします。



3.管理者モード（sudo su）に進み、アカウントのパスワードを入力します。



4.ターミナルで、バックアップがあるコアドライブに転送します。

-cd /ボリューム -使用可能なディスクがあるフォルダーに移動します。

-ls-利用可能なディスクのリストを取得します。

-バックアップが保存されているディスクに移動します-cd BackupDisk （ディスク名）。



5. hdiutilユーティリティのバックアップを使用してバンドルを暗号化します。

ユーティリティの構文は次のようなものです。

hdiutil convert -format UDSB -o 出力ファイル/ -encryption AES-256 入力ファイル/ 。

バックアップ名を高速化するには、名前の最初の2文字を印刷してタブを押す必要があります-名前は自動的に追加されます。

例：

hdiutil convert -format UDSB -o cryptobackup.sparsebundle / -encryption AES-256 username.sparsebundle /

「cryptobackup.sparsebundle」を保護するための新しいパスワードを入力します。

新しいパスワードを再入力してください：

ドライバー記述子マップの読み取り（DDM：0）...

Appleを読む（Apple_partition_map：1）...

ディスクイメージの読み取り（Apple_HFSX：2）...

.................................................. ..........

暗号化されたファイルを取得します



6. Finderを開き、ソースバックアップファイルを右クリックします-パッケージの内容を表示を選択し、com.apple.TimeMachine.MachineID.plistファイルをコピーします。



7.同様に新しく受信した暗号化されたバックアップを開き、このファイルを暗号化されたバックアップに挿入します。



8.次に、暗号化されたバックアップ用のコマンドを実行します（コマンドは、データを保存する暗号化されたコンテナの最大サイズを設定します）。

hdiutil resize -size 1t（1ギガバイト-1g）cryptobackup.sparsebundle /



9. Time Machineを使用するメディアに保存されているものと完全に一致する暗号化されたバックアップの名前を持ち込み、バックアップメディアに置き換えてコピーします。



10.コピーの最後に、Finderでバックアップを開き、必要に応じてパスワードを入力します。必要に応じて、バックアップ中にパスワードを手動で入力しないようにキーチェーンに保存し、バックアップを解除します（取り出し）。



11. Time Machineはシステムアプリケーションであるため、キーチェーンを開き、バックアップのパスワードを見つけて、システムカテゴリにドラッグします。



12.暗号化されたファイルに自動的にバックアップを作成します。



PS。 間違ったことをした場合、Time Machineはusername1.sparsebundleという形式のバックアップを含む新しいファイルを作成します
