まず、いくつかの用語の簡単な説明から始めます。



VDS-別名VPS-仮想専用（プライベート）サーバー-仮想マシン、サーバー-共有ホスティングが十分でなく、専用サーバーが高価な場合に必要です。 VDSレンタルサービスの現代市場には非常に多くのプロバイダーがあり、それぞれに長所と短所がありますが、それらについては説明しません。

仮想化のタイプ-簡単に言えば、これが仮想マシンの編成方法です。 現在、最も一般的なのはOpenVZとXENです。 さらに、OpenVZは低コストです。このタイプの仮想化では、ハードウェアサーバーで実行されるすべての仮想マシンに対して、ホストシステムの特別に修正されたコアが使用されるためです。 このタイプの仮想化の欠点は、OSカーネルのパラメーターを変更することが不可能であり、ほとんどの場合、過剰販売が存在することです（これについては後で詳しく説明します）。 また、深刻なプロジェクトの場合は、XENまたはKVM仮想化を使用することをお勧めします。ゲストOSは、仮想マシンにインストールされていることすら知らず、OSを使用すると、特にXEN-HWMとKVMで、ほとんど何でもできます。

過剰販売-プロバイダーは実際よりも多くのリソースを販売しています。 これは、顧客が100％の割合でリソースを使用することはめったにないという事実のために機能しますが、過剰販売率を過大評価する悪徳なプロバイダーは、メモリ不足による極端なマシン操作やスクリプトの失敗などの問題を経験する可能性があります。 XENおよびKVMでの仮想化の種類に戻ると、過剰販売は技術的には事実上不可能です。

コアを再構築する





OpenVZを選択した場合は、記事のこの部分を安全にスキップできます。

この手順は生産性の目に見える増加をもたらさず、この手順は自己開発と「未来への実践」のためにのみ必要です。

kernel.orgから最新の安定したカーネルバージョンのソースコードをダウンロードします。 これは現在バージョン3.6.7です。 

wget http://www.kernel.org/pub/linux/kernel/v3.0/linux-3.6.7.tar.bz2


•アーカイブを展開する 

 tar -xjf linux-3.6.7.tar.bz2


•コンパイルに必要なパッケージをインストールする 

 aptitude update aptitude install build-dep linux kernel-package


•ソースフォルダーに移動して、カーネル構成を生成します。 この場合、すべてが自動的に実行されます。現在のハードウェアに必要なモジュールのみがカーネルに含まれます。 カーネルを手動で構成する場合は、make configまたはmake menugonfigのリクエストでgoogleしてください。 

 cd linux-3.6.7 make localyesconfig


•カーネルのコンパイルを開始します-速くないプロセスは15分から数時間になります。 

 fakeroot make-kpkg --initrd --append-to-version=-custom kernel_image kernel_headers


•上記のディレクトリに移動して、パッケージをインストールします 

 cd .. dpkg -i linux-*.deb


•再起動して、カーネルが起動していることを確認します。 

 reboot uname -r


•以上が、システム内の新しいカーネルです。

Ubuntuを自分で専用サーバーにインストールする場合、インストール中にシステムの言語と地理的パラメーターを設定しますが、VDSの場合、プロバイダーはサーバーにロールするシステムの既製のイメージを持っています。 また、サーバー上のタイムゾーンの正しい時刻を確認できてうれしいです。

•突然（このようなものを見たことがない）日付/時刻に矛盾がある場合、次のコマンドを実行して、現在の日付と時刻を置き換えます。

pure-ftpd MySQL FTPサーバーのインストール





FTPサーバーは、サーバーからサーバーへの情報の簡単かつ迅速で便利な転送に役立ちます。

 groupadd -g 4001 ftpusers && useradd -u 4001 -s /bin/false -d /bin/null -c "pureftpd user" -g ftpusers ftp


•pure-ftpd用のデータベースをMySQLで作成します。 これはphpmyadminを使用して実行できますが、強力で強力なコピーアンドペーストを使用すると、コンソールで高速に実行できます

•次に、pure-ftpdを構成します。このために、ファイル/etc/pure-ftpd/db/mysql.confを開き、その内容を次のようにします。 

 MYSQLSocket /var/run/mysqld/mysqld.sock MYSQLUser pureftpd MYSQLPassword __pureftpd MYSQLDatabase pureftpd MYSQLCrypt md5 MYSQLGetPW SELECT Password FROM ftpd WHERE User="\L" AND status="1" AND (ipaccess = "*" OR ipaccess LIKE "\R") MYSQLGetUID SELECT Uid FROM ftpd WHERE User="\L" AND status="1" AND (ipaccess = "*" OR ipaccess LIKE "\R") MYSQLGetGID SELECT Gid FROM ftpd WHERE User="\L"AND status="1" AND (ipaccess = "*" OR ipaccess LIKE "\R") MYSQLGetDir SELECT Dir FROM ftpd WHERE User="\L"AND status="1" AND (ipaccess = "*" OR ipaccess LIKE "\R") MySQLGetBandwidthUL SELECT ULBandwidth FROM ftpd WHERE User="\L"AND status="1" AND (ipaccess = "*" OR ipaccess LIKE "\R") MySQLGetBandwidthDL SELECT DLBandwidth FROM ftpd WHERE User="\L"AND status="1" AND (ipaccess = "*" OR ipaccess LIKE "\R") MySQLGetQTASZ SELECT QuotaSize FROM ftpd WHERE User="\L"AND status="1" AND (ipaccess = "*" OR ipaccess LIKE "\R") MySQLGetQTAFS SELECT QuotaFiles FROM ftpd WHERE User="\L"AND status="1" AND (ipaccess = "*" OR ipaccess LIKE "\R")


•次に、ユーザーのホームディレクトリが存在しない場合は自動的に作成されるようにします

 echo "yes" > /etc/pure-ftpd/conf/ChrootEveryone

•それだけです。ログインwww-ftpと指定されたパスワードを使用して、 site_site.zonaのアドレスにあるFTPサーバーに接続を試みることができます。

•FTPプロトコルを介してデータを誤った手に渡らないように転送する場合は、TLS暗号化を設定する必要はありません。 これを行うには、次を実行します。 

 echo 1 > /etc/pure-ftpd/conf/TLS mkdir -p /etc/ssl/private/

 aptitude update aptitude install openssl openssl req -x509 -nodes -days 3652 -newkey rsa:2048 -keyout /etc/ssl/private/pure-ftpd.pem -out /etc/ssl/private/pure-ftpd.pem
