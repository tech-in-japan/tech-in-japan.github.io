レッドハットがビッグデータ市場に参入

月の初めに、 データストレージに焦点を当てた Red Hat Enterprise Linux 6.2のリリースがリリースされました。 理由は明らかです。RedHatは、ビッグデータ（「ビッグデータ」）の市場であるRed Hat自身の新しい市場に参入しています。 Red HatがGlusterを購入した後、購入の目的に関する考えを投稿しました。 しかし、トランザクションの発表からわずか2か月後に、Red Hatの新製品が登場することさえ想像できませんでした。 。



Red Hatが最初の10億ドルのオープンソース企業になる理由の1つは、Linuxディストリビューションだけでなく、 クラウドコンピューティング 、 仮想デスクトップ（VDI） 、 Java EnterpriseおよびRed Hat Storage Software Appliance、 Big Data Managementのリリース。



ビッグデータとは何ですか？ 企業は最近、私たちがビッグデータにdrれていることに気付きました。 毎日、2.5兆バイトのデータを生成し、世界のすべての情報の90％が過去2年間に作成されました。 このデータは、気象および気候変動の記録、ソーシャルメディアの投稿、デジタル写真とビデオ、オンラインショッピングトランザクション、携帯電話からのGPS信号など、あらゆる場所から来ています。 これはすべてビッグデータです。



大量のデータを管理する方法は？ ビッグデータを処理するために、Red HatはオープンソースストレージソリューションであるGlusterを購入し、それを使用して新しいソフトウェア製品を作成しました。 ストレージソフトウェアアプライアンスは、「業界をリードする非構造化データ管理およびストレージソリューションにより、Red Hat製品ラインを拡大しています。」



Red Hatは独自のOpenStackクラウド用にGlusterを購入したと思いましたか？ これは部分的に真実だと思います。 しかし今では、GlusterFSはRed Hatに、そして独立した製品として、数ペタバイトまで拡張可能なオープンな分散ファイルシステムに興味を持っていることがわかります。



Red Hat Storage Software Applianceは、GlusterFS 3.2およびRed Hat Enterprise Linux（RHEL）6.1に基づいています。 企業が最新の企業環境を正常に機能させるために必要な柔軟性、パフォーマンス、およびサービス品質を提供する仮想化、標準化、およびスケーラブルなストレージプールに移行できるようにするソフトウェアです。



Red HatストレージシステムのチーフマネージャーであるRanga Rangachariによる声明は、次のように述べています。 発表された製品は、非構造化データの量の大幅な増加に直面している企業のデータストレージコストを大幅に削減する方法の1つになると予想しています。 「このような企業にオープンソースのストレージシステムを提供し、開発者は新しいソリューションの開発と製品革新の拡大を続けています。」
