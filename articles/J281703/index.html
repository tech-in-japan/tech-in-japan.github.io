はじめに：



私の意見では、ビジネスアナリストとデータベースプログラマーの間の主な問題の1つは彼らの相互作用です。 実際、多くの最新のBIソリューションは、アナリストの仕事をプログラマーに移行することを提供しています。 または、アナリストの技術レベルを上げて、プログラマに近づけます。 代わりに、データベースプログラマから情報を取得してビジネスアナリストに提供するソリューションを検討することを提案します。 この決定により、誰もが自分のことを行うことができ、一人のプログラマーやビジネスアナリストになることはできません。





企業のビジネス分析：



BIとは何ですか？ -これは、私がそれについて聞くとき、私の頭の中で常に起こる最初の考えです。 これは、特定の製品を指定しようとする用語ですが、それ自体は抽象的な概念です。 企業の活動を分析する必要があると誰かが言うとすぐに、ほぼ確実にBIが提供されます。



BIの概念を検討してください。

ビジネスインテリジェンス（BI） -これらは、生の情報を意味のある便利な形式に変換するための方法とツールです。

ウィキペディアからのこのようなBIの定義は、「BIとは何か」という質問には答えません。なぜなら、 生の情報が何であるかが明確ではないからです 。



彼女の定義は次のとおりです。

生情報 -初期情報、主要情報。



これらの定義に依存している場合、ほとんどの企業で主要な情報を理解するプロセスがすでに存在する場合、BI製品を購入する理由は明らかではありません。 そのフレームワークで使用されます：



  1. 一次情報の収集の自動化：現代の世界では、そのような自動化はほぼどこでも導入されています。 支払い端末から始まり、この情報を毎日手動でコンピューターに入力する従業員で終わります。
  2. データウェアハウス：受信した情報を何らかの方法で分析しようとします。ほとんどの場合、現在の日付の情報を格納するデータベース（OLTP）と、過去のすべての日に情報が格納される別のデータベース（データウェアハウス）を作成します。 夜、データは通常、あるデータベースから別のデータベースに移動します。
  3. リポジトリ内のデータの分析：彼らは、Excelの機能を使用して受信した情報を理解し、そこからアンロードを整理することにより蓄積された情報を分析しようとします。
  4. 分析ツールとしてのMicrosoft Excel：マイクロソフトは、アナリストを支援するためにPower BIを提供しています。これにより、Excelは非常に深刻で最新のBIツールになります。 ビジネスアナリストは彼と協力して、組織の現在の財務プロセスの全体像を提供します。



この方法は、多くの企業で広く使用されています。 プロセスは、公開されているコンポーネントを使用して整理できます。

  • Microsoft Office
  • OS MS Windows
  • データベース。


したがって、 「BIツールを個別に購入する必要がありますか？」という質問は、 「はい」ではなく「いいえ 」です。 レポートシステム、さまざまなビジネスインテリジェンスポータル、および同様の製品が必要なのはなぜですか？ Excelはこれらすべてでうまく機能しています。 理論的には、大企業でもこれで十分です。 しかし、もっと適切なものを探す理由があります。 私の意見では、これらはビジネス分析の組織における2つのボトルネックであり、何らかの不便を引き起こし、最悪の場合には情報が意味をなさなくなる可能性があります。



ボトルネックNo. 1-情報収集



情報収集は、情報の主要な蓄積とリポジトリ内での移動のプロセスです。 この問題を解決するには、ほとんどの場合：

  1. 組織にデータベースを展開します。
  2. データベースを扱う人を雇います。
  3. データベースに情報を入力する手段を作成または取得します-ユーザーインターフェイス、自動転送。
  4. これらのシステムで作業するようにスタッフを訓練します。
  5. すべてのシステムの正常性のサポートを提供します。


市場には、情報収集のタスクを大幅に簡素化できる多くのソリューションがあります。 これらは、Microsoft Integration Services、Informaticなどのソース間のデータ転送を提供するさまざまなシステムです。 さらに、データベースはより高速かつ柔軟になり、ビッグデータのようなシステムが登場しています。 この結果、次の問題とは異なり、情報収集の問題は完全に解決可能です。



ボトルネック2-ストレージからユーザーへのデータの転送



ユーザーは分析のためにすべての情報を必要とします。 ただし、ビジネスアナリストにとって便利な形式で何らかの形で構造化して表示する必要があります。 私の意見では、これが創造性の分野の位置であり、市場に多くのソフトウェアツールを生み出しましたが、主な問題、ユーザーへの信頼できる便利なデータ転送の問題を解決しませんでした。 このプロセスを整理する方法はいくつかあります。

  1. データベースへの接続がすでに提供され、そこからの選択が構成されているExcelドキュメントテンプレートが作成されます。
  2. ユーザーは自分でデータベースにクエリを記述し、必要なものを選択する機会が与えられます。
  3. Microsoft Reporting Services、SAP BO、Oracle BIなどのレポートシステムを取得します。
  4. MS Accessなど、ユーザーのローカルデータベースを整理します。


これらの各メソッドは、遅かれ早かれ、解決できないタスクに直面します。 これらの方法を使用する最適性も問題です。 これらの方法をより詳細に検討することを提案します。



Excelドキュメントテンプレート



  • そのようなテンプレートを作成する人にとって、そのようなテンプレートを使用することはあまり便利ではありません。 誰もが開かなければなりません。 選択にアクセスするためのパスワードを覚えて、エンドユーザーが何も変更しないようにします。
  • これらのテンプレートへのストレージとアクセスも何らかの方法で保護する必要があります。 それらを共有フォルダーに保存し、これらすべてにアクセスできるWebアプリケーションを開発または購入できます。 遅かれ早かれ、多くのレポートがあるときに状況が発生しますが、それらは何をすべきか、まだ覚えておく必要があります。
  • 大量のデータがあり、メモリ不足のためレポートを開くことができません。
  • クエリパラメータの入力は困難です。 入力を促すVBスクリプトをExcelで作成できますが、Excelごとにこのスクリプトを作成するか、このスクリプトの改善または修正を他のすべてのテンプレートに翻訳する必要があります。




ユーザーが自分でデータベースクエリを作成する



このオプションの欠点は、スタッフの資格レベルです。 ビジネスアナリストにとって、SQLクエリを作成する能力は主要なアクティビティではないため、1人の専門家が複数のタイプのアクティビティを組み合わせるには高い資格が必要です。



報告システムを購入する



以前のソリューションと比較したこのソリューションの重要な利点は、レポートをコンパクトに保存できること、管理が便利であること、入力パラメーターを設定するためのユーザーインターフェイスが存在することです。



レポートシステムはこの問題を部分的に解決しますが、完全には解決しません。



ビジネスアナリストはExcelで作業します-非常に便利です。 ただし、レポートはWeb上に構築され、そこからダウンロードする必要があります。 また、Excelで実行できるさまざまなピボットテーブルとグラフも作成します。 その結果、レポートはExcelとデータベースの間の追加リンクとして機能することがわかりました。



また、他にも欠点があります。

  • Excelにアップロードすると、情報に違反する可能性があり、フォーマットが「移動」し、列のタイプが変更されるなど。
  • ExcelでWeb上で高速に実行されるレポートは、非常に遅く動作するか、壊れます。
  • 完了までに長い時間（30分以上）を要するレポートは、最大遅延に達するために終了しない場合があります。
  • 大量の情報をアンロードしようとすると、サーバーのメモリ不足が発生し、サーバーが停止する可能性があります。そうしないと、Webでレポートが生成されますが、Excelにアップロードできません。 これらの問題を回避するには、CSVへのアップロード、情報の部分的な転送など、さまざまな方法がありますが、それらはすべてユーザーの観点からは不便であり、注意、正確さ、および追加の知識が必要です。


私の意見では、説明したシステムは、データをユーザーに送信するよりも、データを表示するのに適しています。 それらは、あなたが見ることができるデータである美しいショーケースと比較できますが、「触れる」ことは困難です。



ローカルユーザーデータベース



私の意見では、そのような決定は面倒であり、まったく最適ではありませんが、おそらく将来的には関連する可能性があります。 「各データベースアナリスト！」というスローガンが思い浮かびました。



ユーザーが望むもの







ユーザーにとっての主な問題は、ストレージからユーザーにデータを転送するための便利で信頼できるシステムがないことです。 多くの場合、ビジネスアナリストは計算にExcelを使用するため、Excel形式でデータを転送する必要があります。 残念ながら、便利なソリューションが見つかりませんでした。レポートでデータの一部を転送し、Excelテンプレートでデータを転送し、OLAPキューブとそれらのようなサーバーのようなソリューションを実装してください。 また、Microsoft Access上のCSVファイルとユーザーデータベースも使用しました。 アクティブに使用されるレポートのキャッシュと遅延実行。 並行して、市場に登場するソリューションを実装の観点から検討する必要がありました。 その結果、さまざまなアプローチとソリューションの小さな「動物園」ができました。 この動物園から離れるには、ユーザーが提起した質問を確認する必要があります：「ストアからユーザーへのExcel形式での情報の転送を確認してください！」そして、ビジネスインテリジェンスで使用されるツールを拡張および改善しようとせずに、この問題を具体的に解決します。 ここから、Excelの複雑さを研究し、OpenXML SDKをダウンロードした後、Excelファイルでレポートを直接生成するアプリケーションを作成しました。 仕組みについては、次のリンクをご覧ください： そのようなシステムの例 。 作成することも、商用ソリューションのレベルにすることも難しくありません。 大量の情報を任意にアンロードでき、データベースでクエリが実行されている間、無制限の時間待機することもできます。 これを行うために、ドキュメントのSAXモデルとストリーミングデータを使用しました。



おわりに



BIで長年作業した後、リポジトリからユーザーに情報を簡単に転送できるソリューションを見たことはありません。 BIツールは、主に情報の分析、全体像の表示に役立ちますが、ユーザーへのデータ転送を保証するために行われるものではありません。 つまり、これは、ほとんどの場合、安定して動作する分析システムには十分ではありません。 ExcelおよびPower BIでクライアント側の情報を分析すると便利であり、データベースプログラマーはデータストレージを正常に処理できます。 それらの間の橋はまだ建設されていません。
