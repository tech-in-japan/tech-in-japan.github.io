Android x86の構築

カーネルとAndroidオペレーティングシステム全体を再構築する必要がある多くのタスクがあります。 たとえば、独自のモジュールの作成とデバッグ、システムプロファイリングのサポートの有効化、新しいバージョンのAndroidでのアプリケーションのテストなどです。

VirtualBox仮想マシン内でAndroid x86を実行する機能により、愛好家やファームウェアの作成者は、システム設定を掘り下げ、カーネルを構成および再構築し、同時に実際のデバイスを「ブリック」することはできません。 VirtualBoxは、Linux開発者に馴染みのあるOSデバッグツールを使用する機能を提供します。 通常のAndroidアプリケーション開発者にとって、Intel Hardware Accelerated Execution Managerテクノロジーを使用するエミュレーターは素晴らしいものです。 （詳細はこちらをご覧ください ）

この記事では、VirtualBox仮想マシンで実行するAndroid x86イメージの構築とエミュレーターの構築に関する実用的なヒントを提供します。 Androidソースコードは、 AOSP （Androidオープンソースプロジェクト）プロジェクトの公式リポジトリから取得され、IntelのLinux 2.6カーネルの適応バージョンがカーネルとして使用されます。



ビルド環境の設定



Androidをビルドするには、64ビットバージョンのLinuxが必要です。 もう1つの重要なポイント：システムにインストールされているGCCのバージョンに注意してください。 GoogleはGCCバージョン4.4以降をサポートしています。 また、システムにOracleのJava実装をインストールする必要があります。

Ubuntu 12.04の追加の依存関係のインストール： 

sudo -i apt-get update sudo apt-get install git-core gnupg flex bison gperf build-essential \ zip curl libc6-dev libncurses5-dev:i386 x11proto-core-dev \ libx11-dev:i386 libreadline6-dev:i386 libgl1-mesa-glx:i386 \ libgl1-mesa-dev g++-multilib mingw32 openjdk-6-jdk tofrodos \ python-markdown libxml2-utils xsltproc zlib1g-dev:i386


名前の競合を解決するシンボリックリンクを設定します。 

 sudo ln -s /usr/lib/i386-linux-gnu/mesa/libGL.so.1 /usr/lib/i386-linux-gnu/libGL.so


ソースツリーをダウンロードする



リポジトリのインストール


Repoは、Git for Androidでの作業を簡素化するリポジトリ管理ユーティリティです。 詳細については、こちら（http://source.android.com/source/version-control.html）をご覧ください。

Repoをインストール、初期化、および構成するには、次の手順を実行します。

•ホームディレクトリにbinディレクトリがあり、PATHに記述されていることを確認します。 

 mkdir ~/bin PATH=~/bin:$PATH


•Repoスクリプトをダウンロードし、実行権限を設定します。 

 curl https://dl-ssl.google.com/dl/googlesource/git-repo/repo > ~/bin/repo chmod a+x ~/bin/repo


Repoクライアントの初期化


Repoをインストールした後、Androidソースコードリポジトリへのアクセスを設定します。

•ソースファイル用の空のディレクトリを作成し、そこに移動します。 たとえば、次のようになります。 

 mkdir aosp cd aosp


•repo initスクリプトを実行して、Repo自体のバージョンを更新します。 

 repo init -u https://android.googlesource.com/platform/manifest


このコマンドは、Androidソースツリーからmasterブランチを接続します。 -bスイッチを使用して、他のものを指定できます。 

 repo init -u https://android.googlesource.com/platform/manifest -b android-4.1.1_r6


利用可能なすべてのブランチを表示するには、次のコマンドを入力します。 

 git --git-dir .repo/manifests/.git/ branch -a


Googleの最新バージョンのAndroidに興味があります。

  • Jelly Bean：リモート/起源/ android-4.1.1_r6
  • アイスクリームサンドイッチ：リモート/起源/ android-4.0.4_r2.1


好みに合わせてどれでも選択できますが、それ以降の手順は変わりません。 VirtualBoxでビルドプロセスを実証するには、Jelly Bean（JB）のバージョンを使用します。

初期化に成功すると、ディレクトリでRepoが初期化され、その中に.repoフォルダーが表示されることを示すメッセージが表示されます

ファイルをアップロードする



リポジトリから作業ディレクトリにソースファイルをダウンロードするには、次のコマンドを実行します。 

 repo sync -j 16


最初の同期には1時間以上かかる場合があります。

カーネルアセンブリ



Androidは主にジェスチャー制御デバイス向けに設計されており、デフォルトではマウスをサポートしていません。 したがって、例として、マウスサポートを有効にしてソースからカーネルを再構築します。

カーネルソースを格納するディレクトリを作成します。 

 mkdir ~/android_kernel cd ~/android_kernel


Intelの事前に準備されたバージョンのカーネルを使用します。 

 wget http://software.intel.com/sites/landingpage/android/Intel_x86_sysimg_2.3.7_Source_Files.zip unzip Intel_x86_sysimg_2.3.7_Source_Files.zip tar -xvf kernel_sdk_x86.tar.gz


カーネルディレクトリに行きましょう： 

 cd kernel


これでカーネルソースができました。 構成ファイルを変更して再構築する必要があります。 カーネルを完全に構成する時間を短縮するために、開発者向けに準備した構成ファイルを使用します。 重要なポイント-カーネルが構築されているアーキテクチャを指定することを忘れないでください。この場合はx86です。 

 cp arch/x86/configs/vbox_defconfig .config make ARCH=x86 menuconfig


数秒後、グラフィカルなカーネル構成メニューがロードされます。 上矢印と下矢印を使用してメニュー項目を移動し、Enterキーを押してサブメニューに入ります。 すべてが通常のLinuxカーネルとまったく同じです。





マウスのサポートを有効にするには：

•デバイスドライバーに移動

•「入力デバイスのサポート」を選択します

•「マウス」をチェックします

「マウス」サブメニューに移動して、追加のドライバー設定を確認できます。

次に、カーネル構成メニューを終了します。 これは、「Esc」を数回クリックすることで実行できます。 構成メニューからこれを行うように求められたら、忘れずに設定を保存してください。

必要なカーネル設定を行ったら、それを組み立てます。 これは、特にマルチコアマシンを使用している場合、それほど時間はかかりません-makeコマンドオプションを指定できます：-j N、ここでNはコアの数です（たとえば、make -j 32は記事の作成に使用されました）。 

 make ARCH=x86 –j 32


コンパイルが成功すると、次のメッセージが表示されます。 

 Kernel: arch/x86/boot/bzImage is ready


新しいカーネルへのパスは最後の行に示されています。

カーネルのなりすまし


VirtualBoxのアセンブリで使用されるカーネルへのパスは、次のように決定できます。 

 grep "LOCAL_KERNEL :=" ~/aosp/build/target/board/vbox_x86/device.mk


以下が表示されるはずです。 

 LOCAL_KERNEL := prebuilt/android-x86/kernel/kernel-vbox


見つかったパスでbzImageをコピーします。 

 cp ~/kernel/arch/x86/boot/bzImage ~/aosp/prebuilt/android-x86/kernel/kernel-vbox


Androidビルド



ビルド環境を設定します。 

 cd ~/aosp . ./build/envsetup.sh


これで、Androidソースのコンパイルを開始する準備がすべて整いました。 最初のステップは、アセンブリの目的を指定することです。 

 lunch vbox_x86-eng


コマンドを実行すると、次のビルドに関する情報が表示されます。 

 ============================================ PLATFORM_VERSION_CODENAME=REL PLATFORM_VERSION=4.1.1 TARGET_PRODUCT=vbox_x86 TARGET_BUILD_VARIANT=eng TARGET_BUILD_TYPE=release TARGET_BUILD_APPS= TARGET_ARCH=x86 TARGET_ARCH_VARIANT=x86 HOST_ARCH=x86 HOST_OS=linux HOST_OS_EXTRA=Linux-2.6.32-131.0.15.el6.x86_64-x86_64-with-redhat-6.1-Santiago HOST_BUILD_TYPE=release BUILD_ID=JRO03R OUT_DIR=out ============================================


次に、makeコマンドでソースアセンブリを開始する必要があります。 ビルドログを保持することをお勧めします。これは、teeユーティリティを使用して実行できます。 

 make –j 16 | tee vbox_build.log


アセンブリプロセスにはかなりの時間がかかる場合があります。具体的な数値は、中央処理装置のパワーとマシンのRAMの量によって異なります。 記事の準備に使用されたシステムでは、これには約1時間かかりました。 アセンブリに推奨されるスレッド数は、スレッドあたり2GBのRAMの計算から決定できます。

コンパイルが成功すると、出力の最後の行にイメージのパスとサイズが含まれます。 

 Install system fs image: out/target/product/vbox_x86/system.img out/target/product/vbox_x86/system.img+ total size is 268435456


次に、VirtualBoxのブートイメージを作成する必要があります。 

 make installer_vdi


アセンブリが成功した場合、エントリがコンソールに表示されます。 

 Done with VirtualBox bootable installer image -[ out/target/product/vbox_x86/installer.vdi ]-


VirtualBoxを構成する



次に、VirtualBoxをセットアップしましょう。 新しい仮想マシンを作成し、「Android_ICS」と呼び、オペレーティングシステムとしてLinuxを選択し、Androidに最も適したバージョンとして[その他]に設定する必要があります。



次のステップは、RAMの割り当てです。 512メガバイト以上を推奨します。 私たちの場合、ギガバイトになります。



次に、ハードディスクのサイズを構成します。 主なことは、セットアップが完了した後、すべてのVirtualBoxハードドライブがIDEコントローラーに接続されることです。

ブート可能なAndroidイメージを仮想マシンの2番目のパーティションとしてインストールします。



仮想マシンの最初の起動時に、Androidのインストールを開始する必要があります。 起動中にF12を押してBIOS設定メニューに入ります。 2「プライマリスレーブ」を選択します。これは、Androidのインストールイメージが含まれている場所です。

インストールが成功すると、「インストーラー構成の処理が完了しました」と表示されます。 最初のインストール時にエラーが発生する場合があります。もう一度実行してください。 次に、新しくインストールしたAndroidで仮想マシンを再起動します！



エミュレータージェリービーン



エミュレーターのビルドは、VirtualBoxのビルドバージョンとそれほど変わりません。 すでにソースコードがある場合は、環境の初期化手順からアセンブリを続行します。 実験の純度のために、エミュレータを構築するプロセスの概要を最初から概説します。 そのため、Jelly Bean Emulator用に別のディレクトリを作成し、よく知られている手順を繰り返します。 

 mkdir ~/jb cd ~/jb


リポジトリの初期化 

 repo init -u https://android.googlesource.com/platform/manifest -b android-4.1.1_r6


ソースをダウンロード 

 repo sync -j 16


環境の初期化 

 . ./build/envsetup.sh


エミュレータービルドの選択 

 lunch full_x86-eng


成功すると、以下が表示されます。 

 ============================================ PLATFORM_VERSION_CODENAME=REL PLATFORM_VERSION=4.1.1 TARGET_PRODUCT=full_x86 TARGET_BUILD_VARIANT=eng TARGET_BUILD_TYPE=release TARGET_BUILD_APPS= TARGET_ARCH=x86 TARGET_ARCH_VARIANT=x86-atom HOST_ARCH=x86 HOST_OS=linux HOST_OS_EXTRA=Linux-2.6.32-131.0.15.el6.x86_64-x86_64-with-redhat-6.1-Santiago HOST_BUILD_TYPE=release BUILD_ID=JRO03L OUT_DIR=out ============================================


アセンブリを開始します。 

 make –j 16 | tee emulator_build.log


コンパイルプロセスの進行中に、構成ファイルをコピーし、エミュレーター用のカーネルをアセンブルします。 

 cd kernel cp arch/x86/configs/goldfish_defconfig .config make ARCH=x86 -j16


これでエミュレーターの組み立てが完了しました。

最も簡単な実行方法： 

 emulator


たとえば、480x800の解像度、512 MBのRAM、およびハードウェアグラフィックスアクセラレーションを有効にして、新しくビルドされたカーネルで実行できます。 

 emulator -kernel ~/kernel/arch/x86/boot/bzImage -skin WVGA800 -memory 512 -gpu on


エミュレータの機能と設定の詳細については、 こちらをご覧ください。
