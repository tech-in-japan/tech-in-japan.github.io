Redmineのプラグイン：パフォーマンスレポート

チームの作業を自動化するプラグインの開発を続けています。

自動化の次の段階として、作業量に関するレポート用のプラグインを作成することが決定されました。

これは、プロジェクトの作業の統計と、従業員の作業の監視の両方に役立ちます。これは、リモート協力スキームで非常に重要です。







このプラグインはRedmineアクティビティレポートプラグインと呼ばれ、各従業員とプロジェクトの経過時間と終了したタスクに関するレポートを送信するために使用されます。 レポートは、毎日、毎週、毎月生成されます。



プラグインのインストールと構成は非常に簡単です。

Redmineを使用してサーバー上のコードを複製します。 

git clone https://github.com/centosadmin/redmine_activity_report /opt/redmine/plugins


移行を開始します。 

 bundle exec rake redmine:plugins:migrate


クラウンにタスクを追加します。 

 bundle exec whenever -i redmine_activity_report -f plugins/redmine_activity_report/config/schedule.rb


クラウンをクリーニングするには、次のコマンドを使用します。 

 bundle exec whenever -c redmine_activity_report -f plugins/redmine_activity_report/config/schedule.rb




その後、レポートを受け取りたいプロジェクト設定で、このモジュールをオンにします。



そして、このプラグインのパラメーターを含むプロジェクト設定の新しいタブがあります。

ここで、選択できるのは、プラグインが補助プロジェクトで機能するか、データが収集されるグループ、データの送信先のユーザー、およびユーザーです。





その結果、一定の間隔で、表形式のレポートを受け取ります。



上部-各従業員の要約統計には、チケットと時間の表示を含む各従業員の成績証明書があります。
