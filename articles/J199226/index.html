このようなコードを作成すると、サーバーの起動時にクラスが作成された場合は英語でクラスの作成済みコピーを取得するリスクが発生します。作成が動的であるが最初の呼び出しでキャッシュされた場合は中国語版など



そのような場合、翻訳に注意したいが、正しい場所で翻訳したい

適切な場所は、クラスではなくオブジェクトを作成することです

つまり 

 def __(string,*k,**kwargs): return string class MainView(SomeParent): def __init__(self): #.... self.title=gettext(self._title) #.... class SomeView(MainView): _title=__("This view title")




つまり、文字列コレクタは__を翻訳用の文字列として定義し、関数自体は何も実行せず、適切なタイミングで翻訳が開始されます。
