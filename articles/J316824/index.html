前文

しかし、これがすべて完全なナンセンスだと言ったらどうでしょうか？ 私があなたに秘密を告げるならば：それはそれほど簡単ではありません。 時間は相対的なものであり、異なる値と係数で測定できます。 私のお気に入りの基準は、恩恵を受ける時間の関係です。 プログラマーが4日間で市場に新製品を投入できると言ったらどうなるでしょうか？ 興味がありますか？ -読んでください。



やる気



最近、「私たち自身のIvangai」がTelegramのプログラマーコミュニティに追加され、音声メッセージで左右に散らかるようになりました（突然、キーボードで指で書くのは難しいことを知っています）。 当然、人々は会話の本質を理解するために1日2〜3時間メッセージを聞くことにうんざりしていました。そして、誰かがすべての軍隊を自動的にテキストに変換するボットを作成することを提案しました。 これはまさに目が光る瞬間です。



音声からテキストへの翻訳



このようなボットを開発するのにどれくらいの時間がかかるかはわかりませんでしたが、温かく居心地の良いチャットルームとTelegramのオープンソースのTelegramフリーランスサービスのボットの裏で十分な経験がありました。 ためらうことなく、開発のために座った-Telegam Bot APIモジュールを接続する段階はスムーズに進んだ。 次は？ 音声認識モジュール-Googleの「音声認識API」。最初のリンクの中で最高のサービスのリストを取得します。 先頭にあるのはベータ版のGoogle Speech APIです。



ボットによってすべてのボイスメッセージを受信し、何らかの形でGoogleに直接送信します。 しかし、残念なことに、このサービスを使用するためのほとんどすべてのnpmモジュールは時代遅れであるか、機能しません。 Googleの専門家によって書かれたNode.jsの組み込みAPIを使用しようとしています-動作します。 認証には何がありますか？ Googleのドキュメントから多くのオプションを試してみました-ちなみに、これも50〜50が古くなっているか、機能していません-ここにキーがあります。



ところで、ここに私が使用したコードがあります：

const Speech = require('@google-cloud/speech'); const speech = new Speech({ projectId: 'voicy-151205', credentials: require('path/to/certificate/file.json') }); speech.startRecognition(filepath, { 'encoding': 'LINEAR16', 'sampleRate': 16000, 'languageCode': 'en-US', }) .then((results) => { const operation = results[0]; return operation.promise(); }) .then((transcription) => { console.log(transcription[0]); })


Googleオーディオの形式と制限



Googleに部隊を派遣するのはどうですか？ ええ、Telegramからオーディオファイルを取得してSpeech APIサーバーに送信する必要があります。 元気？ Telegramの通常の.oga形式は受け入れられません-デコードする必要があります。 メディア変換には何がありますか？ もちろん、ffmpeg、幼少期から親しんでいました（父に感謝します。父は昔、彼を理解してくれました）。 ノードに接続する方法は？ おっと！ このために特別に調整されたnpmモジュールがあります。 .ogaファイルをどの形式に変換する必要がありますか？ Googleは.flacを受け入れ、変換、試行、すべてが受け入れられ、Googleがテキストで応答し、成功したことが判明しました。



しかし、そこにありました！ Googleは、Google Cloud Storageサービスにアップロードしない限り、60秒を超えるファイルをテキストに変換しません。 これは何？ すぐにGoogleプログラマーによって書かれたモジュールを試してみてください。時代遅れのnpm既製ソリューションに目を向けないでください。 さて、ファイルはアップロードされ、サービスによって処理され、テキストが返されます。 ちなみに、Googleには時間という奇妙な概念があります。何らかの理由で30秒のオーディオファイルは60秒と定義されています。 大丈夫です-30秒以上ファイルを試行し、次のレーキを踏む-GoogleはLINEAR16エンコードでのみ長いファイルを受け入れます。



LINEAR16とは何ですか？ ドキュメントでffmpegを探しています-そのようなものはありません。 いいですね さらに取り組みます。 この形式または獣用のコーデックを探しています-これは、「16ビットの符号付きリトルエンディアン」データであることがわかりました。 コンピューターサイエンスに関する本をいくつか読んで、「16ビット」とは何か、なぜ「リトルエンディアン」なのかを知っているのは良いことです。 ffmpegのこの形式が何であるかを探しています-aha：“ s16le”！ 私たちは変換しようとします-結局のところ、Googleは「ミッションが達成されました」というテキストを受け入れて応答します。



注意：以下は、変換に役立つコードです（マシンにffmpegをプレインストールする必要があります）！

 const ffmpeg = require('fluent-ffmpeg'); const temp = require('temp'); ffmpeg.ffprobe(filepath, (err, info) => { const fileSize = info.format.duration; const output = temp.path({ suffix: '.flac' }); ffmpeg() .on('end', () => console.log(output)) .input(filepath) .setStartTime(0) .duration(fileSize) .output(output) .audioFrequency(16000) .toFormat('s16le') .run(); });
