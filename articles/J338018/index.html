QuadBraces III

すべての人に健康！



QuadBracesパーサーに関する最初の公開からほぼ2年が経過しました。これは、標準化が必要な最も単純なプロジェクト向けのMODX Evolutionの代替です。 これは、通常の出版物、ポートフォリオ、名刺サイト、ほぼ静的なページなどで構成される1ページにすることができます。 それ以来、私は自分の開発を更新し、コミュニティに気付かれずに、3番目のバージョンにクロールしました。 QuadBracesパーサーの現在のバージョンには非常に多くの変更が含まれているため、詳細なチュートリアルを公開する必要があります。 だから...



タグ



タグハンドラから始めましょう。 QuadBraces 3では、タグハンドラーは個別のクラスに配置されます。 これで、開発者はパーサー自体のコードを変更せずに、タグタイプを自由に定義できます。 これを行うには、QuadBracesTagPrototypeクラスの子孫を作成します。このクラスでは、以下を決定する必要があります。



  • $ _name-タグ識別子名
  • $ _start-初期タグシーケンス
  • $ _rstartは、MODX Revolution構文モードの初期タグシーケンスです。 角括弧なし。
  • $ _finish-最終タグシーケンス
  • $ _order-オプション。 処理順序
  • function main（array $ m、$ key = ``） -タグ処理の結果を返す関数。 $ m-PCRE通常データ、キー（最も重要）-見つかったタグのキー。


例として定数タグハンドラを使用する： 



class QuadBracesTagConstant extends QuadBracesTagPrototype { protected $_name = 'constant'; protected $_start = '\{\*'; protected $_rstart = '\/'; protected $_finish = '\*\}'; protected $_order = 5; public function main(array $m,$key='') { $v = ''; if (empty($key) || !defined($key)) { $this->_error = 'not found'; } else { $v = constant($key); } return $v; } }


ええ、確かに-すべてがとてもシンプルです。 私は、昔ながらのプログラマーとして、顕微鏡サイズのファイルを保存する必要性にひどく怒っています。 しかし、彼らが言うように、これは個人的な問題です。



前の出版物を読んでいない人のために。 QuadBracesタグの構文は、すべてのタグタイプで同じです。



  1. 最初の文字シーケンス。 たとえば、「{{」。
  2. エンティティの識別子。 ラテン文字、数字、「ログ」、ダッシュ、および/またはドットで構成する必要があります。 たとえば、「my-chunk」。 注意してください！ 識別子のドットは、ネストレベルのセパレーターの役割を果たします。 つまり、たとえば、変数「my.var」は実際にはここに配置されます-$ parser-> data ['my'] ['var']。 チャンク、スニペット、テンプレートについて-以下。
  3. 多数の拡張ハンドラー（オプション）。 それぞれ、「：some_identifier」または「：some_identifier = `some_data_」という形式の構造を表します（コロンと識別子の間にスペースを入れないでください）。 ハンドラー拡張機能は、タグ処理の最終結果で機能します。 つまり、たとえば、電話番号を含む変数を受け取り、人間が読みやすいようにフォーマットしました。
  4. MODX構文との互換性のために、疑問符が後に続く場合があります。
  5. 特定の数の引数（オプション）。 各引数は、 "＆some_name =` some_value` "という形式の構造です。 はい、変数にも引数を設定できます。 はい、動作します。 ほとんどの場合、渡された引数はローカルプレースホルダーで使用されます。
  6. 文字の最終シーケンス。 たとえば、「}}」。


おそらく既に理解しているように、パーサーの3番目のバージョンには、MODX Revolution構文モードがあります。 このモードでは、タグは2つの開き角括弧で始まり、その後にタグのタイプを決定する特定の文字が続きます。 タグは常に2つの閉じ角括弧で閉じられます。



特定のチャンクのタグの例： 



 {{news-item? &title=`   ` &date=`15-09-17` &url=`/news/150917.html`}}


チャンクが次の場合： 

 <article> <h3>[+title+]</h3> <span class="date">[+date+]</span> <a href="[+url+]"> </a> </article>


そして、最後に、チャンクの場所で、それが表示されます： 

 <article> <h3>   </h3> <span class="date">15-09-17</span> <a href="/news/150917.html"> </a> </article>


ハエといえば。 Uyutnenkyのエディターの「ソースコード」メニューからHTMLアイテムを食べましたか？



すぐに使用できるQuadBracesは、次のタイプのタグをサポートしています（2つのモードの開始シーケンスと終了シーケンスが示されています）。



  • [+ +]または[[+]] - ローカル変数 。 ほとんどの場合、ループで変数スニペットに使用されます。 スニペットを除く任意の要素の引数を設定すると、要素のローカルプレースホルダーは引数の値に置き換えられます。
  • {{}}または[[$]] - チャンク 。 HTMLの断片 1つのファイルには1つのチャンクが含まれます。 テンプレートの主な要素の1つ。
  • {[]}または[[-]] - 1行のライブラリチャンク 。 HTMLの断片 1つのファイルには複数のチャンクが含まれます。 各行には1つのチャンクがあります。
  • {（）}または[[=]] - 複数行ライブラリチャンク 。 HTMLの断片 1つのファイルには複数のチャンクが含まれます。 チャンクは次の構造で区切られます： "<！-tags：split->"
  • {* *}または[[/]] - PHP定数 。 これらは、システムで定義されたPHP定数（ユーザーによるものを含む）を表示します。
  • [（）]または[[++]] - 設定 。 SETTINGS配列の変数。 通常、パーサーを使用してCMS変数を出力するために使用されます。
  • [* *]または[[*]] - 変数 パーサー変数。 テンプレートの主な要素の1つ。
  • [^ ^]または[[^]] - データのデバッグ 。 パーサーのデバッグデータ。
  • [%%]または[[％]] - 言語変数 。 語彙言語変数。 言語システムがオンになると、現在の言語に応じて置き換えられます。
  • [！ ！]または[[！ ]] - スニペット 。 基本的に、PHPコードのチャンク。
  • [[]]または[[]] - キャッシュフラグ付きのスニペット 。 キャッシュフラグが設定されている場合のみ、上記と同じです。 大まかに言って、値がtrueの$ cached引数は、パーサーの実行関数に渡されます。
  • [〜〜]または[[〜]] - リソース識別子からのリンク 。 配置された内部番号をリソースへのリンクに変えます。 以下のリソースについて。
  • [::]または[[：]] - カスタム変数ハンドラー 。 それらについては別に。


パーサーの現在のバージョンでは、いわゆるパーサーオブジェクトのセットを突き出すことが可能です。 リソース。 基本的に、これはMODXのリソースとほぼ同じです。 大まかに言うと、データベースは最終プロジェクトのデータベースに格納され、そこにはブログ投稿のあるエントリが格納されます。 主なことは、URLを作成するには、各エントリに数値ID、親ID、またはNULLとエイリアスが含まれている必要があることを覚えておくことです。 パーサーオブジェクトのリソースプロパティを設定すると、パーサーオブジェクトのidxプロパティが設定されます。 リソースツリーのブランチのインデックスを表します。 これにより、リソースの有限セットの構造を操作できます。 実際、これらすべてのガーベッジ機能を使用すると、リンクタグが機能します。



カスタム変数ハンドラーは、パーサー変数をランダムに処理する一種のマイクロパーサーです。 たとえば、パーサー変数（トップメニューなど）に、次の形式のURLを含む配列が含まれている場合： 



 array( array('url' => '/','title' => ''), array('url' => '/about.html','title' => ' '), array( 'url' => '/news/','title' => '','children' => array( array('url' => '/news/10-09-17.html','title' => '   '), array('url' => '/news/15-09-17.html','title' => '   ') ) ), array('url' => '/contacts.html','title' => ''), );


それからメニューを作成できます。 これを行うには、構造を適切な場所に配置するだけです。 



 [:menu@top-menu:]


その代わりに、箇条書きリストを使用してメニューが表示されます。 Wayfinderの最高の伝統では、いわばいわば。



一般的に、カスタムハンドラーについては、別の記事を書く必要があると思います。



それとは別に、テンプレートの1つの重要な機能、つまりメタフィールドについて。 ほぼ最初のバージョンから、テンプレートコードにメタフィールドを設定する機会があります。 これらは基本的に、この構造を使用してテンプレート自体で定義された変数です。



<!-- FIELD:_? [] -->







引数はパーサータグとほぼ同じです。 フィールドで指定されている引数に関係なく、最終フィールドには3つの属性が含まれます。



  • デフォルト（テキスト）-デフォルト値
  • タイプ（整数）-タイプ
  • キャプション（文字列）-タイトル


すべてのメタフィールドは、パーサーのfieldsプロパティを介してアクセスできます。



同様に重要な機能は、メタ値です。 somevar変数がパーサーで定義されていると想像してください。 インストールされたテンプレートのメタ値は、変数の現在の値をオーバーライドできます。 メタ値は設計によって決定されます。



<!-- DATA:_ ` ` -->







テンプレートをインストールすると、両方ともテンプレートの最終コードから削除されます。



クラスAPI



クラスプロパティ（最も重要）

：

  • 所有者 （オブジェクト）-所有者オブジェクト（読み取り専用、初期化中にインストール）;
  • パス （配列）-テンプレートパス。 配列に設定でき、パスがコンマでリストされている行にすることができます。
  • fields （配列）-テンプレートフィールド（読み取り専用）;
  • データ （配列）-パーサー変数。
  • 設定 （配列）-「設定」;
  • debug （array）-デバッグデータ。
  • language （string）-現在の言語。 一般に受け入れられている標準に従った署名。
  • loadLanguage （フラグ）-言語の自動ロード（デフォルトはfalse）;
  • dictionary （array）-現在の辞書。 生のテキストを行に分割して設定できます。
  • maxLevel （整数）-最大ネストレベル（デフォルトでは32）。
  • レベル （整数）-現在のレベル。
  • notice （配列）-通知のレベル。 "strict"-すべての要素、 "common"（デフォルト）-チャンクとスニペットの意味を理解しています。
  • テンプレート （テキスト）-テンプレート; インストール時に、名前を指定し、内容を返します。
  • templateName （文字列）-現在のテンプレートの名前（読み取り専用）;
  • autoTemplate （フラグ）-インストールされたコンテンツからのテンプレートの自動抽出およびインストールのフラグ。 大まかに言って、テンプレートメタポールを有効にしてコンテンツをパーサーにフィードすると、パーサーは自動的にテンプレートを設定します（デフォルトではfalse）。
  • コンテンツ （テキスト）-コンテンツ。 一般的に、本質的に「コンテンツ」と呼ばれる変数。
  • リソース （配列）-リソースの配列。
  • idx （配列）-リソースツリーのブランチのインデックス（読み取り専用）。
  • SEOStrict （フラグ）-SEOStrict標準への準拠（デフォルトではfalse）。
  • MODXRevoMode （フラグ） -MODX Revolution構文モードへの切り替え（デフォルトではfalse）。


方法

：

  • registerTag（QuadBracesTagPrototype $ o） -タグ登録; 初期化されたタグを入力として受け入れます。 登録済みのタグを返します。
  • parseStart（$ m） -処理の開始。 入力はpreg_match_callback関数からデータを受け取ります。 要素のキーを返します。
  • parseFinish（$ m、$ t、$ k、$ v） -処理の終了。 入力は、レギュラーシーズン、タグタイプ、タグキー、現在の処理値からデータを受け取ります。 処理結果を返します。
  • 設定（$キー、$値= null） -設定の読み取り/書き込み。 nullの場合、読み取り。
  • 変数（$キー、$値= null） -変数の読み取り/書き込み。 nullの場合、読み取り。
  • search（$ type、$ name） -要素ファイルを検索します; ローカライズを考慮します。 見つかったファイルの名前を返します。
  • setTemplate（$ v） -テンプレートを名前で設定します。 テンプレートインストールのコールバックを使用する場合に非常に必要な方法。
  • getChunk（$キー、$タイプ= 'チャンク'） - チャンクの内容を取得します。 見つかったチャンクの内容またはfalseを返します。
  • execute（$ name、$ args = array（）、$ input = ''、$ cached = false） -スニペットまたは拡張機能を実行します; スニペットの名前、引数、入力テキスト、キャッシュフラグを受け入れます。 処理の結果を返します。
  • parse（$ t = ``、$ d = null、$ e = ''、$ k = ''） -メインのパーサーメソッドが処理中です; テンプレート全体を処理するために引数を設定する必要はありません。 引数は、内部のニーズまたは個別の処理（QuadBracesコードの一部）に必要です。 入力引数-入力コード、変数のデータ、要素、要素キー。 出力で-処理されたデータ。
  • sanitize（$ t = ''） -QuadBracesからの入力コードのサニタイズ。
  • 拡張機能（$ v、$ e） -拡張機能の実行; 入力時-現在のコードと拡張子付きの行。
  • registerEvent（$ n、$ f） -イベント登録; 入力時-イベントと関数の名前。
  • registerMethod（$ n、$ f） -APIメソッドを登録します。 入力時-メソッドと関数の名前。
  • invoke（） -イベント呼び出し。 最初の引数はイベントの名前で、残りは呼び出し引数です。


どこを見ますか？



QuadBracesパーサーの最も重要なプロパティの1つは、pathsプロパティです。 パーサーがチャンク、スニペット、拡張機能、テンプレートを探す場所を決定します。 言語システムは少し離れて動作します。 要素を検索するプロセスでは、パーサーは登録されたすべてのパスを順番に検索します。 毎回、検索機能は、要素のタイプに応じて、選択したパスにフォルダーを追加します。 チャンク-チャンク、テンプレート-テンプレート、スニペット-スニペット。 そして、すでに必要な要素が求められています。 チャンク、スニペット、およびテンプレートのキーのドットが、実際にはディレクトリセパレーターを置き換えることを思い出させてください。



例： 



 $parser->paths = 'D:/projects/mysite/content,D:/repo/templates/default'; $fn = $parser->search('snippet','basis.snipcon');


実際、この場合のパーサーは次のファイルの存在を確認します。



D:/projects/mysite/content/snippets/basis/snipcon.php

D:/repo/templates/default/snippets/basis/snipcon.php







最後に見つかったファイルが返されます。 言語システムがオンになっている場合、言語署名付きのパスが実際に検索に追加されます。 インストールされている言語がロシア語、つまり、署名が「ru」であるとします。 したがって、ファイルのリストは次のとおりです。



D:/projects/mysite/content/snippets/basis/snipcon.php

D:/projects/mysite/content/snippets/basis/ru/snipcon.php

D:/repo/templates/default/snippets/basis/snipcon.php

D:/repo/templates/default/snippets/basis/ru/snipcon.php







他のすべては、プロパティとメソッドを通じて取得されます。



言語体系



まず、言語システムを明示的に含める必要があることをすぐに予約します。 これは、QuadBracesLangクラスを接続することにより行われます。 これは、メインクラスを含める前の最も単純なコードで実行されます。 



 define("QUADBRACES_LOCALIZED",true);


loadLanguageフラグを有効にすることをお勧めします。 その後、辞書が自動的にロードされます。 原則として、デフォルトでは、パーサーのパスを設定すると、言語システムのパスが設定されます。 実際、言語システムは、要素検索エンジンと同じ原理でファイルを検索します。 言語システムフォルダのみがlangです。



例： 



 $parser->paths = 'D:/projects/mysite/content,D:/repo/templates/default'; $parser->language = 'ru';


フォルダーがスキャンされます：



D:/projects/mysite/content/lang/ru/

D:/repo/templates/default/lang/ru/







これらでは、拡張子「lng」を持つすべてのファイルが検索されます。 各ファイルは1行ずつ読み取られます。 各行には、言語キー、タイトル、説明、プレースホルダーの4つの要素があります。 言語キーは、言語タグで使用されるものです。 キャプション-デフォルトの言語タグを置き換えます。 説明-通常、ツールチップまたはフィールドの説明に使用されます。 プレースホルダー-ほとんどの場合、入力要素の対応する属性に使用されます。



イベント



パーサーオブジェクトはイベントをサポートします。 上で書いたように、ハンドラーはregisterEventメソッドによって設定されます。



サポートされるイベント（どこでも$ pはパーサーオブジェクトです）：



  • init（$ p） -コンストラクターの完了。
  • beforeChangeData（$ v、$ p） -これらの変数を変更する前。 $ v-設定可能な変数。
  • changeData（$ v、$ p） -変数のデータを変更します。 $ v-設定可能な変数。
  • beforeChangeSettings（$ v、$ p） -設定データを変更する前。 $ v-インストール可能な設定。
  • changeSettings（$ v、$ p） -設定データを変更します。 $ v-インストール可能な設定。
  • beforeLoadDictionary（$ v、$ p） -辞書をロードする前。 $ vは辞書です。
  • loadDictionary（$ v、$ p） -辞書をダウンロードします。 $ vは辞書です。
  • methodNotFound（$ n、$ p） -メソッドが見つかりません。 $ n-メソッドの名前。
  • beforeSetLanguage（$ v、$ p） -言語をインストールする前。 $ v-言語;
  • setLanguage（$ v、$ p） -言語をインストールするとき。 $ v-言語;
  • setContent（$ v、$ p） -コンテンツをインストールするとき。 $ v-コンテンツ。
  • setResources（$ v、$ p） -リソースをインストールするとき。 $ v-リソースデータ。
  • invalidHandler（$ n、$ a、$ p） -イベントハンドラーがない場合。 $ n-ハンドラーの名前、$ a-渡された引数。
  • setResources（$ v、$ p） -リソースをインストールするとき。 $ v-リソースデータ。
  • defaultTemplate（$ p） -デフォルトのテンプレート（空のテンプレートのインストールを試みます）;
  • loadTemplate（$ v、$ p） -テンプレートをインストールするとき; $ v-テンプレートの名前。

  • templateData（$ v、$ p） -テンプレートデータを取得します。 $ v-テンプレートデータ。
  • beforeLocalParse（$ v、$ d、$ p） -ローカル処理の前。 $ v-処理されたコード、$ d-データ。

  • localParse（$ v、$ p） -ローカルテンプレートの処理後（サニタイズ前）; $ v-処理されたコード。
  • parse（$ v、$ p） -テンプレートの処理後（サニタイズ前）; $ v-処理されたコード。




私は告白しますが、知らないうちにPHPの1つの機能にぶつかりました。 たとえば、templateNotFoundイベントハンドラーに関数を割り当て、その中にテンプレートプロパティを設定しようとしても、何も起こりません。 これは、PHPオブジェクトのプロパティのアクセサーを呼び出すときの再帰制限によるものです。 つまり、同じプロパティのアクセサーとアクセサーを介してプロパティを設定することはできません。 まさか。 「完全に」という言葉から。 したがって、templateNotFoundイベントハンドラーで404番目を作成する場合は、その内部でsetTemplateメソッドを呼び出します。



ハンドラー拡張



すべてのタグは最終値ハンドラーをサポートします。 たとえば、変数を取得しましたが、空です。 notemptyハンドラーを使用すると、このケースの出力テキストを設定できます。 一部の拡張機能には、追加の「拡張機能」があります。 たとえば、基礎となるロジックには「then」および「else」という拡張子があります。



サポートされるハンドラー



  • is、eq-等式（比較された値、それから）
  • isnot、neq-不等式（値の比較、それ以外の場合）
  • lt-より小さい（比較値、それから）
  • lte-より小さい、または等しい（値の比較、それ以外の場合）
  • gt-より大きい（比較値、それから）
  • gte-以上（比較された値、それ以外の場合）
  • 偶数 -パリティの符号（値の比較、それ以外の場合）
  • 奇数 - 奇数記号（比較した値、それ以外）
  • empty-空の値のサイン（値 "then"、その他）
  • notempty-空でない値のサイン（値 "then"、その他）
  • null、isnull-値がNULLかどうか（値 "then"、その他）
  • notnull-値がNULLではないかどうか（次に、そうでない場合）
  • isarray-値が配列かどうか（値 "then"、その他）
  • for-整数反復子（反復回数、開始、スプリッター）
  • foreach-インデックスイテレータ（インデックスのコンマ区切りリスト、スプリッター）
  • js-link-値をスクリプトへのリンクに変換します
  • css-link-値をスタイルシートへのリンクに変換します
  • import-値をスタイルシートへのリンクに変換します（CSSの@import）
  • リンク -値をリンク（タイトル）に変換します
  • link-external-値を外部リンク（ヘッダー）に変換します
  • links-値のすべてのURLをリンク（リンク属性）に変換します
  • ul、ol-複数行の値をリストに変換します（リストアイテムテンプレート）


ul / ol拡張には、2つの内部プレースホルダーがあります。[+クラス+]-要素クラス（最初、最後）、[+アイテム+]-実際の文字列。 拡張の場合、現在の反復番号を含む内部プレースホルダー[+イテレーター+]が使用可能です。 foreach拡張の内部プレースホルダーが利用可能です：[+ iterator.index +]-現在の反復の位置番号、[+ iterator +]-現在のインデックス。

実際、パーサークラスの操作は非常に簡単です。 基本的な例： 



 <?php require 'quadbraces/parser.php'; $parser = new QuadBracesParser('//__'); $parser->template = 'my-template'; echo $parser->parse(); ?>


その中で、パーサークラスを接続し、テンプレートデータへのパスでオブジェクトを初期化し、テンプレートとparsimを設定します。 もちろん、処理する前に、「設定」をパーサー変数に渡すことができます。 そして、パーサークラスをオンにする前に、上記のように言語システムをオンにします。 より実用的な例： 



 <?php define("QUADBRACES_LOCALIZED",true); require 'quadbraces/parser.php'; $parser = new QuadBracesParser('D:/projects/foo/content/template'); $parser->language = 'ru'; $parser->data = array( 'pagetitle' => '  ', 'date' => '20-09-17', 'image' => 'content/images/flies.jpg' ); $parser->template = 'news.single'; echo $parser->parse(); ?>


テンプレートは、次のファイルで検索されます。



D:/projects/foo/content/template/templates/news/single.html

D:/projects/foo/content/template/templates/news/ru/single.html
