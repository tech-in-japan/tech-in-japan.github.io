Ubuntuの8.10から9.04への更新

作業マシンでUbuntuを9.04ベータ版にアップグレードすることは、自己責任で決定しました。

私は事前にすべてがうまくいったと言います！ ;-)更新プロセス全体のスクリーンキャストと、必要な場合のコメントを提供します。

さあ、行こう！



コンソールを開き、次を実行します。

~$ update-manager -d







更新マネージャーが開き、配布更新ボタンが表示されます。





更新をクリックします。 リリースノートウィンドウが表示されます。



これからベータ版のリリースであり、稼働中のシステムに配置することはお勧めできません。 しかし、私たちはチャンスを取ります;-)



次に、更新プロセスが開始されます。





新しいパッケージリストを取得します。





さらに、システムはサードパーティのパッケージソースの切断について警告します。





これで、更新システムは更新を約2〜3分間チェックします。





すべての必要なシステムを確認した後、削除、更新、およびインストールされるパッケージが表示されます。





パッケージのダウンロードが開始されます...新しいパッケージを900MBほどダウンロードするのに約1時間かかりました。





すべてのパッケージがダウンロードされると、インストールフェーズが開始されます。





より活発なプロセスのためにターミナルを開くことができます。



この段階で、ネットワークは切断され、更新と再起動後にのみ再び上昇しました。 構成ファイルの置換についてはまだいくつかの質問があります（すべてを置換すると答えました）。



すべてが配信されるとすぐに、システムクリーニングが開始されます。





インストールされているパッケージを確認した後、Ubuntuは古い不要なパッケージを削除することを提案します。



ネジにゴミが出ないように、彼女に同意します。



その後、システムの更新は完了したと見なすことができます。



リセットボタンをクリックして、新しいシステムをお楽しみください！

~$ sudo lsb_release -a

[sudo] password for tux:

No LSB modules are available.

Distributor ID: Ubuntu

Description: Ubuntu jaunty (development branch)

Release: 9.04

Codename: jaunty







更新後に最初に目を引いたのは、約束された新しいツールチップとシステム起動速度（2倍高速）でした。



これですべてテストします！ ;-)



PS：システムの更新は自己責任で行ってください！ あなたはすべての結果に対して自分で責任を負います。
