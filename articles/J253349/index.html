ASP.NET 5の概要







ASP.NET 1.0の最初の予備リリースはほぼ15年前に発表され、それ以来何百万人もの開発者がそれを使用してさまざまなアプリケーションを作成および実行し、長年にわたって膨大な量のコードと機能を追加しました。



私たちのチームが取り組んでいるASP.NET 5と呼ばれる新しいASP.NETリリースについてのニュースを書いているので、私は今日非常に興奮しています。 このリリースの一部として、ASP.NETをよりコンパクトで、モジュール式で、クロスプラットフォームにし、クラウドホスティング向けにしています。 ASP.NET 5 Previewは、最近利用可能になった最新のVisual Studio 2015 CTPでダウンロードできます。



ASP.NET 5は、最新のWebアプリケーションを作成し、Windows、Linux、およびMacで動作するように設計されたオープンソースのWebフレームワークです。 MVC 6フレームワークとWeb APIが含まれています。 ASP.NET 5はSignalR 3の基盤にもなります。当初、ASP.NET 5は.NET CoreCLRで実行するように設計されていますが、完全な.NET Frameworkでも実行できます。



ASP.NET 5では、Webフレームワークのコアをよりコンパクトに（System.Web.dllが不要になった）、モジュラーに（現在はほとんどの機能がNuGetパッケージの形式で作成されているので、必要なもの）。 ASP.NET 5では、次の変更が行われます。

  • Windows、Linux、およびMacでのクロスプラットフォームASP.NETアプリケーションのアセンブリと操作。
  • .NET Coreで作業する
  • 最新のWebアプリケーションの開発を簡素化する新しいツールキット
  • Web UIおよびWeb APIの単一技術スタック
  • クラウド対応の環境ベースの構成
  • NuGetパッケージの作成と使用のネイティブサポート
  • ネイティブ依存性注入のサポート
  • IISまたは独自のアプリケーションでホストする機能


そして最終的な結果として、ASP.NETはおなじみですが、Web開発の現代の現実にはるかによく適合したものになりました。



柔軟なクロスプラットフォームランタイム



ASP.NET 5は2つのランタイムで実行され、アプリケーションをホストする場所を選択できます。 ランタイムの選択は次のとおりです。



.NET Coreは、まったく新しいクロスプラットフォームランタイムであり、重量はほとんどありません。 .NET Coreを選択すると、まったく新しい利点を活用できます。

1）.NET Coreランタイムをアプリケーションに直接提供する権利があります。これは、アプリケーションがシステムにインストールされているバージョンではなく、アプリケーションで動作することを意味します。 rutimeバージョンは、他のアプリケーションをプルアップする他のバージョンと並行して動作します。 これが他のアプリケーションの動作に影響を与えることを恐れることなく、ランタイムのバージョンを新しいバージョンにアップグレードできます。 これにより、アプリケーションとフレームワークの新しいバージョンの展開がはるかに簡単になり、他のアプリケーションの実行中に更新できるようになります。

2）アプリケーションは、本当に必要な機能のみに依存します。 したがって、アプリケーションに関係のないサービスを更新するように求められることはありません。 これにより、他のアプリケーションに影響を与えることなく、アプリケーションをより迅速かつ簡単に展開できます。

3）アプリケーションをクロスプラットフォームにできます。 Windows、Linux、およびMac OS X用のクロスプラットフォームバージョンの.NET Coreを提供します。開発用に選択したオペレーティングシステム、および展開用に選択したオペレーティングシステムに関係なく、どこでも.NETを使用できます。 クロスプラットフォームバージョンのランタイムはまだリリースされていませんが、積極的に取り組んでおり、すぐに公式プレビューを公開することを約束しています。



.NET Framework-.NET Frameworkのフルバージョンと比較すると、現在.NET Core APIははるかに制限されているため、アプリケーションを.NET Coreで実行するには、おそらく多くの変更を行う必要があります。 アプリケーションを更新したくない場合は、完全な.NET Framework（バージョン4.5.2以降）上のASP.NET 5で実行できます。 そうすることで、.NET Framework APIの完全なセットを取得し、アプリケーションは変更なしで動作します。



MVC 6は統合プログラミングモデルです。



MVC、Web API、およびWebページは追加機能を提供し、多くの場合、ソリューション開発（SLN）で一緒に使用されます。 ただし、ASP.NETの以前のリリースでは、これらのフレームワークは個別に開発されたため、大量の重複と矛盾が含まれていました。 MVC 6では、これらすべてのモデルを単一のプログラミングモデルに結合しました。 これで、これらのフレームワークの違いを「調整」する必要なく、Web UIとサービスの両方を含むWebアプリケーションを作成できます。 また、シンプルなWebページアプリケーションを最新の信頼性の高いMVCにシームレスに転送できます。

同じコントローラーから同じMVCフィルターパイプラインを使用して、Razorビューデータを返すことができるようになりました。

既存のフレームワークを統合することに加えて、サーバー開発を容易にする新しい機能を追加しています。 たとえば、タグヘルパー。 マークアップ内のタグのセマンティクスを単純に拡張することにより、ビューでHTMLヘルパーを使用できます。

つまり このように書く代わりに： 

@Html.ValidationSummary(true, "", new { @class = "text-danger" }) <div class="form-group"> @Html.LabelFor(m => m.UserName, new { @class = "col-md-2 control-label"}) <div class="col-md-10"> @Html.TextBoxFor(m => m.UserName, new { @class = "form-control" }) @Html.ValidationMessageFor(m => m.UserName, "", new { @class ="text-danger" }) </div> </div>


次のように記述します。 

 <div asp-validation-summary="ModelOnly" class="text-danger"></div> <div class="form-group"> <label asp-for="UserName" class="col-md-2 control-label"></label> <div class="col-md-10"> <input asp-for="UserName" class="form-control" /> <span asp-validation-for="UserName" class="text-danger"></span> </div> </div>




タグヘルパーは、プレゼンテーションをより「自然」で読みやすくします。 また、HTMLヘルパー出力のカスタマイズを簡素化します。

最新のWeb開発



今週、ASP.NET 5プレビューをリリースしました。これには、とりわけ、アプリケーションを少し改善するための機能が含まれています。



ダイナミックな開発

Visual Studio 2015では、動的コンパイルを使用します。 変更を確認するために毎回アプリケーションを再コンパイルする必要はなくなりました。 代わりに、（1）コードを変更し、（2）変更を保存し、（3）ブラウザーを更新し、（4）変更がアプリケーションを自動的に更新する方法を確認します。









新しい開発経験が得られます。これは、コンパイルされた言語で作業するよりもインタープリター言語で作業するように見えます。



他のコードエディター（たとえば、C＃プロジェクトを操作するためのアドインを備えたSublime Text 3）を使用して、ASP.NET 5プロジェクトを操作することもできます。 Visual Studioのすべての機能は、コマンドラインツールで見つけることができますが、その間はクロスプラットフォームです（つまり、インプレース編集はMac OS XとLinuxの両方で利用できます）。



一般的なWeb開発ツールとの統合

将来のMS Visual Studio 2015のもう1つの機能は、人気のあるオープンソースプロジェクトであるBower、Grunt、Gulpの組み込みサポートです。これは、各開発者にとって不可欠なツールであると考えています。

  • Bowerは、JavaSciptとCSSの両方を含むクライアント側のパッケージマネージャーです。
  • GruntとGulpは、アプリケーションのワークフローを自動化するタスクランナーであり、LESS、CoffeeScript、TypeScriptファイルの構築などのタスクにGruntまたはGulpを使用できます。 JSLintを起動し、JSファイルを縮小します。




Bower： JavaScriptライブラリをASP.NETプロジェクトに追加するには、bower.json構成ファイルに直接追加します。











Visual Studioが利用可能なパッケージのリストをIntelliSenseに提供することを忘れないでください。 次回ソリューションを開くと、Visual Studioは不足しているすべてのパッケージを自動的に復元するため、それらをすべてバージョン管理システムに配置する必要はありません。



サーバー側のパッケージの場合、もちろんNuGetを使用する必要があります。



Grunt：現代のWeb開発では、アプリケーションを構築するためだけに開発とは異なる非常に多くのタスクに従事していることに気付くことができます：LESS、TypeScript、CoffeeScriptファイルの収集、JavaScriptの縮小、ユニットJSテストの実行などなど...さらに、各チームには、プロジェクトで使用されるツールに基づいた独自の要件セットがあります。 タスクランナーにより、これらのタスクの調整と管理が容易になりますVisual Studioは、最も人気のある2つのランナー、GruntとGulpをサポートしています。



次に例を示します。たとえば、LESSファイルを収集するにはGruntが必要です。 次に、package.jsonを開いて、Gruntのサードパーティプラグインであるgrunt-contrib-lessパッケージを追加する必要があります。











Visual Studo 2015の新しいTask Runner Explorerマネージャーを使用して、タスクを特定のビルドステップ（ビルド前、ビルド後、クリーン、またはSLNファイルが開いているとき）に割り当てます。











これにより、一般的なタスクの自動化が真にノンストップの操作になります。



簡素化された依存関係管理

ASP.NET 5では、NuGetパッケージを追加/削除および更新することで依存関係を管理します。 NuGetパッケージマネージャーを使用するか、NuGetパッケージのリストとプロジェクトで使用されているバージョンのリストを含むJSONファイル（project.json）を編集するだけです。 project.jsonファイルは簡単に操作でき、任意のファイルエディターで自然に編集できます。これにより、アプリケーションがクラウドで実行されているときに依存関係を更新できます。
