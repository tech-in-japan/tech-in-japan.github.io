CMSとは何ですか？


コンテンツ管理ソフトウェア （CMS）は、技術的な専門家が技術的な詳細を理解せずにWebアプリケーションのコンテンツを管理するのに役立つソフトウェアです。 Sitecore CMS

ASP.NETプラットフォームを使用し、IISの下で実行されます。これにより、技術者がほぼすべての種類のWebアプリケーションを開発するための膨大な機会が開かれます。

現在、多くの異なるCMSソリューションがあります。 有料版と無料版の両方のSitecore CMSは第1タイプに属し、主に大規模なクライアントでの作業を目的としていますが、無料版（ http://xpress.sitecore.net/ ）は開発者とエンドユーザーの両方が利用できます。プラットフォームの学習とブログの両方に使用します。

企業市場での主な競合相手は、SDL、Adobe、Ektronなどの主要企業です。 これは、Sitecoreがリーダーとして位置付けられているGartner社のレポートのグラフで判断できます。



詳細な説明を含む完全なレポートは、次の場所にあります。

www.sitecore.net/Resources/whitepapers/Gartner-Magic-Quadrant.aspx

ご覧のとおり、顧客ポートフォリオは非常に幅広いものです。 顧客の数と規模はそれを物語っていると思います。

Sitecore CMSの概要


Sitecore CMSは、高性能でスケーラブルで拡張可能なASP.NET CMSであり、カスタムマークアップを生成する技術に制限はありません。 XHTML、HTML5、CSS、jQuery、JSON、XMLなど、現在利用可能なすべてのテクノロジーを使用できます。

機能の短いリスト：

  • 柔軟で階層的なデータウェアハウス（SQL ServerまたはOracleを使用できます）。
  • 技術者向けのシンプルで直感的なAPI
  • 提供されるパイプラインの拡張、イベント処理などを通じて構成および拡張する機能
  • 管理コンポーネントを動的にコンパイルおよびキャッシュするエンジン。コード再利用ソリューションの作成に役立ちます。
  • ディスプレイデバイスの概念（デバイス管理）-さまざまなクライアント（ブラウザー、PDF、プリンター、XMLフィード）またはモバイルデバイス用にさまざまな表示ページを作成できます。
  • 認証、承認、役割、管理プロファイルなどのセキュリティは、ASP.NETメンバーシップサービスを使用して実装されます。
  • ワークフロー（ワークフロー）を提供し、コンテンツを公開済みと未公開に分離します。
  • 個々のフィールドを含む、コンテンツ（コンテンツ）のバージョン間の違いの視覚的な比較。
  • あらゆるタイプの非構造化データおよびファイルを保存するためのメディアライブラリ（メディアライブラリ）。
  • Visual Studio 2010 IDEとの緊密な統合（ Sitecore Rocks


最初の打ち上げ


たとえば、/ sitecore / loginをWebサイトのアドレスに追加するだけです：

localhost / sitecore / loginをクリックすると、ログインページにリダイレクトされます（デフォルトのログイン：管理者パスワード：b）



いくつかのログインオプションが利用可能になります。 すぐにデスクトップmodを使用することをお勧めします。いくつかのウィンドウを開くことや、以下で説明するさまざまな設定やユーティリティへの拡張アクセスなど、より多くのオプションが提供されます。



組み込みのSitecore CMSユーティリティ：

  • コンテンツエディター-コンテンツを編集するためのメインツール
  • デベロッパーセンター-プレゼンテーションコンポーネントを編集するためのブラウザベースのIDE
  • テンプレートマネージャー-基本的にデータ型であるテンプレートを管理するための環境
  • コントロールパネル-システム管理と個人設定。
  • メディアライブラリ-画像やビデオなどのファイルが保存されているマルチメディアライブラリエディタ
  • セキュリティエディター-ユーザーおよびロールの要素へのアクセス権の管理
  • プレビュー-過去および将来のバージョンを一度に表示する機能を備えたコンテンツの表示
  • デバッガー-組み込みのデバッガーをアクティブにして、ページのパフォーマンスを含めて分析します
  • ワークボックス-ワークフロー管理
  • Access Viewer-効果的なセキュリティ設定を表示
  • ロールマネージャー-セキュリティロール管理
  • ユーザーマネージャー-CMSユーザーの管理と公開サイトへの訪問者を識別する機能
  • パッケージデザイナー-コンテンツとファイルをある環境から別の環境に移行するためのパッケージを作成します。
  • インストールウィザード-パッケージデザイナーで作成されたパッケージのインストーラー
  • エクスプローラー-ドキュメントへの物理パスを表示します
  • ログビューアー-イベントログを表示する
  • 壊れたリンク-壊れたオブジェクトリンクの管理


ご覧のとおり、多くの標準ユーティリティがありますが、個人的な経験では一部を使用していません。 たとえば、ログビューアを使用するよりも、ディスクからログファイルを直接開く方がはるかに便利ですが、このアプリケーションは、リモートサーバーなど、ディスクに直接アクセスできない場合に役立ちます。

また、開発者は、クライアント/顧客のニーズに応じてシステムにアプリケーションを埋め込み、既存のものを拡張することも許可されていることに注意してください。 ほとんどのアプリケーションはXAMLテクノロジーに基づいて構築されているため、組み込みアプリケーションのあらゆる側面を非常に簡単に変更できます。 これについては、次のいずれかの記事で説明します。
