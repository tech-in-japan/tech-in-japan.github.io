すべての読者はPHPを必要とします（最後の-NewsBlurはpythonで実行されます）。





rssLounge



作業には筋肉が必要です。一般に、インストールには約5〜10分かかります。 OPMLファイルは問題なくインポートされ、その後のフィードのロードも突然終了しました。



外観：





さらに、すべてのエントリがタイルで表示される「イメージのみ」ビューが利用できます。

機能は一般的にトップです。 日付（または日付の範囲）を選択できるカレンダーがあり、フィードの優先順位を割り当て、それに応じてフィルターをかけることができます。 許可されたアクセスを構成し、パブリック読み取り専用を有効にすることができます。 ブックマークレットも提供されます。 エントリをお気に入りに追加すると、一定期間が経過しても削除されません（用語は設定で指定できます）。 devianart、twitter、tumblrのユーザーフィードを簡単に追加できます。 もちろん、Androidには1つまたは2つのクライアントがありますが、もちろんこれだけでは十分ではありません。



蜂蜜のスプーンに入れたタールの樽は、キリル文字の奇妙な作品でした。 最初の読み込みではすべて問題ありませんが、フィードをオンにすると失敗します。すべてのキリル文字は疑問符付きで表示されます。 レコードを表示するとき、それらは自動的に既読としてマークされません。設定でこの動作を変更することはできません。 さて、最も重要な欠点：プロジェクトは著者によってサポートされなくなりました-彼のすべての力は現在、セルフォスに焦点を当てています。これについては以下で説明します。



要約：いくつかの予期しない機能を備えた、期待される少数の便利でインストールが簡単な高速RSSリーダー。



公式サイト 、 Google Code





セルフォス



セルフォスは最初から私にそのデザインを与えてくれました。 オンラインビューティーアグリゲーター（これはもちろん私見です）のうち、Feedlyだけが比較できます。 インストールには、配布キットのダウンロードを含めて3分かかりました。 このプログラムは、フォルダーに必要な権限をダウンロードしてインストールするとすぐに使用できるようになります。可能な設定はすべてオプションです。 Sqlite（デフォルト）またはMySQLをデータベースとして使用できます。 Google OPMLはしがみついていますが、問題があります-それらについては以下をご覧ください。 ホットキーはGoogleリーダーのものとほとんど同じです。



標準ビュー（実際には他のビューはありません）：





上で言ったように、SelfossはrssLoungeと同じ開発者の発案であり、その前身のいくつかの機能を継承しています。 たとえば、TwitterやTumblrのユーザーのフィードに固執する機能、古いエントリを削除して、パブリックアクセスやお気に入りへのエントリの追加の可能性があります。 ホットキーは同じです。 いくつかの欠点も移行されました。たとえば、表示時にレコードが既読としてマークされないこともあります。



短所：


Googleリーダーには100を少し超える購読があります。 毎回テープをロードするときのセルフォスは504に低下しました-そして、いくつかの非常に高い値に制限を上げることさえも助けませんでした。 これらは一時的な問題である可能性があります（または、サーバーのローカルでさえ、さらにSelfossをMySQLに接続しようとしませんでした）が、堆積物は残りました。

私の意見では、最も重要なマイナスは、個別のフィードを表示できないことです。すべてまたはタグのいずれかのみを視聴できます。 また、レコードの内容を3つの列に分割する意味は、そのサイズに関係なく、いくらか理解しにくいものです。





一般に、セルフォスは非常に気持ちの良い感情を残し、幼少期の病気をすぐに取り除き、前任者のスキルを研究できるようになることを期待しています。 すべての規範に従って作られた適応設計は非常に満足しています-このサービスはタブレットとスマートフォンの両方で使用するのに非常に便利です。



公式ウェブサイト 、 github

リリーナ



外観：





最小機能：フィードの管理（フィードの自動検出を含む）、OPMLのインポート、表示...および...それだけです。 2つの完全なプラグインにより、投稿をinstapaperに保存し、投稿へのリンクをWordpressのブログに送信する機能が追加されます。 ところで、Wordpressについて。 著者は、この特定のエンジン、プラグインとテーマの組み込みシステム、テンプレートタグthe_content（）、the_title（）など、さらにはナレッジベースのアドレス-codex.getlilina.orgからこの透過的にヒントを得ていることがわかります。



欠点：メモに既読のマークを付ける方法がなく、検索がなく、実際には何もありません。 1年以上更新されていません。



公式サイト 、 Google Code 、 GitHub

PS リリーナがあります - リリーナの後継者のようなものですが、2006年に最後に更新されました。
