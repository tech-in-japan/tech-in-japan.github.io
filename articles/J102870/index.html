これが最後に得た配色です。



個人的には、黒の背景に白い文字（もちろん、緑）に引き寄せられます。 特に暗闇では、読書は簡単で、目には当たりません。



残念ながら、このソリューションは、 Gnu / LinuxまたはCompizウィンドウマネージャーに付属する別のOSを使用しているユーザーにのみ受け入れられます。





StackOverflowで、 skratがCompizのネガティブプラグインの使用を提案しました。



ネガもマイナスを見つけました。 黒い背景に非常に明るい文字が表示されました。 幸いなことに、Compizには別のプラグインColor Filterがあり、アセンブラーに似た言語（これはOpenGLフラグメントプログラムと呼ばれると書かれています）で色を切り替えることができます。

フィルターコード



!!ARBfp1.0

TEMP temp, neg;



# Dunno what's this... but every other filter starts with this :) ;

TEX temp, fragment.texcoord[0], texture[0], RECT;



# Applying negative filter ;

RCP neg.a, temp.a;

MAD temp.rgb, -neg.a, temp, 1.0;

MUL temp.rgb, temp.a, temp;

MUL temp, fragment.color, temp;



# Lowering contrast and shifting brightness ;

MUL temp.rgb, temp, 0.8;

ADD temp.rgb, temp, 0.25;



MOV result.color, temp;

END







合計





*良い解決策が得られます（残念ながらCompizのみ）。

*すべてのエディターおよびウィンドウ全体に普遍的に適しています（ただし、グラフィックスはネガティブに見えます）。

*必要に応じてオンとオフを切り替えます！

*便利なプラグインのリストにCompiz（Expo、Scale、Grid）がもう1つ追加されました=）
