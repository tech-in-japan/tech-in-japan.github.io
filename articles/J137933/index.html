オーディオライブラリの整理。 実用的なヒント

はじめに



一般的に、私は多くの問題で完璧主義者です。 存在する場合、それは非の打ちどころのないものであることが必要です（時にはマイナスの影響を及ぼしますが、今ではそうではありません）。 また、オーディオコレクションをきれいに保つことに慣れています。 しかし、時間の経過とともに、補充により、多くの小さなトラブルが蓄積し始めました。 それらのいくつかを次に示します。

  • ファイル名の異なるテンプレート： 4-スター| 04-ザスター| 04.スター
  • タグのその他のアーティスト名： Chizh＆Co | チズアンドカンパニー | ミレン・ファーマー| ミレーヌ・ファーマー
  • アルバムアーティストがリストされていません（コンピレーションアルバムが完全に混乱する）
  • 複数のディスクで構成されるアルバムに関する情報がまったく正しくない
  • 長年にわたる出版と再版（リマスタリングなど）に関する混乱
まあ他のささいなこと。

多くの場合、ユーザーは数千、時には数万のレコードのコレクションを持っています。 そのため、すべてを手でシャベルで切ることは長く、非効率的です。



このマニュアルの目的は、オーディオコレクションを可能な限り低コストで最も注文の多い形式にすることです。 インターネットで公開されている一般に公開されている情報とは対照的に、私の個人的な経験（肯定的と否定的の両方）もここにあります。 その結果、 完全に再分類されたコレクションを取得します。 次のすべてはどのコレクションにも適用できますが、最良の選択肢は、ほとんどの部分のコレクションが完全な公式アルバムで構成され、最小限の異なるトラックのコンテンツである場合です。



ステージ1.事前クリーニング



すべてのステージの説明は本質的に助言的であり、最初からすべてをやり直して最大限に整理する場合にのみ、段階的に使用する必要があることをすぐに言わなければなりません。



まず、すべてのファイルとフォルダーを調べて、それらをすばやくクリーンアップする必要があります。 あなたが聞いていない可能性が低い過剰をすべて捨ててください。 たとえば、私は多くの再版（つまり、実際にテイク）、偽造品、すべての種類のライブレアリティを聴いていましたが、コレクションのためだけに保管していました...一般に、可能であれば完全なままにしておいてください公式アルバム。 これは重要な段階ではなく、むしろ重要な段階です。 その後、これのおかげで、多くの時間を節約できます。



ステージ2.アルバムと曲の認識



この段階は最も重要です。 曲の認識とソートはここで実行されます。 これを行うには、 MusicBrainz Picardプログラムが必要です。

この段階とその後のすべての段階で同じ作業を2回行わないように、すべてのプログラムの設定を慎重に確認してください。

すべてをゼロからやり直すことをお勧めします。 これを行うには、MusicBrainzの設定で、ファイル転送を有効にし、宛先フォルダーを指定します。 タグ内のすべての古い情報を消去することもお勧めします。

ID3タグに関する小さな余談。 バージョン2.4はより高度ですが、すべてのプログラムが正しく機能するわけではありません。 特に、Windows ExplorerおよびWindows Media Player（バージョン12まで）はエンコードを理解しません。 互換性が必要な場合は、2.3をオンにします。 UTF-8のようなパンとタグの拡張セットを使用したより進歩的な形式が必要な場合は、2.4を入力します。 私はすべてのプログラムにバージョン2.4を追加しました（先読み-foobar2000とGoogle Musicは問題なく理解できます; Winampもそうすべきです）。



プログラムは、レギュラーアルバムとコレクション（コンパイル）の2種類のアルバムを認識します。 コレクションとは、アルバムに複数のアーティストがいる場合です。 私のコレクションには、さまざまな賛辞のコレクションとコレクション「The Best of Classics」が見つかりました。 コレクションは1つのフォルダー（通常はVarious Artistsと呼ばれます）にスローされ、その後ソートできます。

「ファイル転送」設定項目に注意してください。名前の変更とフォルダーによる並べ替えを構成します。 小さなスクリプト言語（foobar2000と同じ）を使用しており、テンプレートを自由にカスタマイズできます。 デフォルト設定はかなり普遍的ですが、私自身は少し調整しました： 



$if2(%albumartist%,%artist%)/$if2($left(%originaldate%,4),$left(%date%,4)) - %album%/$if($gt(%totaldiscs%,1),%discnumber%-,)$num(%tracknumber%,2) - %title% $if2(%albumartist%,%artist%)/$if2($left(%originaldate%,4),$left(%date%,4)) - %album%/$if($gt(%totaldiscs%,1),%discnumber%-,)$num(%tracknumber%,2) - %artist% - %title%


最初の行はアルバム用で、2行目は編集用です。 ここで、変数％date％はアルバムのリリース日ではなく、リリース（または再発行）の日付を示していることに注意してください。

指定されたパターンは、次のような名前を生成します。 



 音楽\ Splin \ 2002-Acoustics \ 2-05-My Heart.mp3
音楽\オムニバス\ 2001-フェリーニツアー\ 14-ダンプ-My Heart.mp3


すべての設定が正しく設定されていることを確認したら、コレクション全体をフィードしてコーヒーを飲むことができます。 さて、またはコレクションのサイズに応じて、スリープします。 プログラムの実行は非常に遅く、1分あたり約20個のファイルが処理されます。 タグと名前から抽出できる情報が多いほど、処理が高速になります。

この記事では、「extreme」モードでテストしました。数十個のトラックを取り、それらからすべてのタグを削除し、ランダムに名前を変更しました。 結果はスクリーンショットに表示されます。





アルバムにグループ化された認識済みトラックが右側に表示されます。 ただし、プログラムがトラックを認識できるようにする必要がある場合があります。 ファイルが認識されないか、いくつかのファイルがアルバムトラックの「候補」に分類される場合があります。 この場合、手動で選択する必要があります。 一致レベルは、緑から赤までの色で示されます。 チェックすると、一部のタグとトラックの「指紋」が比較されます。

アルバムにすべてのトラックが含まれており、複数の解釈がない場合、アルバムは（ある意味では黄色:)としてマークされます。 私たちの目標は、ゴールドアルバムの最大数を達成することです。 また、アルバムの内容は制作スタジオによって異なる場合があります。エディションを手動で変更すると、より正確な一致を実現できる場合があります。 アルバムを保存先フォルダーに定期的に保存しました。



ステージ3a。 名前の手動改訂



世界が完璧であれば、第二段階を除いて、他に何も必要ありません。 しかし悲しいかな、MusicBrainzはすべてではありませんが、かなり多くの作業を行うことができます。 一部のアルバムがデータベースにない可能性があります。 この場合、ソースフォルダーに並べ替えられていない音楽がいくつかあります。

手動での修正の段階はかなり標準的なものであるため、これまでに書いた内容は推奨事項にすぎません。 しかし、それでもなお、複雑なアクション全体の整合性のために、それらをより詳細に説明します。 このステップの目標は、ファイルとディレクトリ構造を、1）正しく、2）要件を満たすビューにすることです。



お気に入りのファイルマネージャーを起動し、未分類の音楽が残っているフォルダーに移動して開始します。 まず、構造と名前を単一の形式にする必要があります。 Black Coffeeのアルバムのいくつかを認識している場合は、どこにでも同じバンド名があるかどうかを確認してください。 ソートされた音楽にNN名のテンプレートがある場合-Trackname.mp3-残りのすべてのファイルの名前を適宜変更します。 グループの名前変更を容易にするために、 Mp3tagを使用できます。 使い方は説明しません。 両方のフォルダの構造を完全に一致させる必要があります。そのため、マージ後、どこから来たのかを言うことはできません。

その後、2つのフォルダーを1つにマージできます。 原則として、これは終了できます。 しかし、多くはファイル構造をもう少し磨きたいと思うでしょう。 たとえば、デフォルトでは、コレクションはさまざまなアーティストフォルダーにスローされますが、多くの場合、コレクションは単一のアーティストに関連しています。 したがって、さまざまなアーティストのすべてのコンテンツを他のディレクトリに簡単に散在させることができます。 アルバムがアーティストによって細かすぎることも起こります： Chizh、Chizh＆Co、Chizh and Various people、Chizh and Pasha、Chernetskiy and Chizh、 ...多くの場合、アルバムは1つだけです。 これは私にはあまり適していなかったので、そのようなフォルダーのグループをすべて1つにマージしました。



ステージ3b。 手動タグの改良



すべての必要な音楽が適切な場所にあり、正しく呼び出された後、タグ付けを行うことができます。 ここでMp3tagが必要です。 実際、すべての操作は、タグに関してのみ、前の段階の操作に似ています。 最初にタグを一見して、必要に応じて変更する必要があります。

残念ながら、これを行う唯一の方法は、すべてのアルバムとトラックを手動で確認することです。 幸いなことに、ほとんどの作品はすでに準備ができており、未分類の部分でのみ作業します。

ささいなことの洗練に関して。 それはすべてあなたの要件に依存します。 この部分をスキップすることもできます。 個人的に、次の操作を行いました。

  • 複合アルバムを1つに送信し、タグにディスク番号を書き留めました
  • アルバムのすべてのアーティストを具体化（さまざまなアーティストがいた場所）
  • サービス文字がアンダースコアに置き換えられたファイル名を修正
一般に、コレクション全体を調べて、次の2つのことを確認する価値があります。1）同じトラックグループ（アーティスト名、番号付けなど）で意見の相違がある場合、2）外観に満足している場合。

Album Artistとタグ付けされたいくつかの単語。 たとえば、ピクニックトリビュートアルバムをご覧ください。 アルバムには18トラックがあり、18の異なるグループによって演奏されます。 ただし、アルバムアーティストのタグに「ピクニック」を安全に書き込むことができ、ピクニックアルバム全体に表示されます。 この概念は非常に頻繁に使用され、ソート時のAlbum ArtistタグはArtistタグよりも優先されます。

また、Mp3tagには基本的なスクリプト機能があり、かなり広い範囲でタグと名前を操作できることも言いたいと思います。 さらに、「アクション」ウィンドウとそのヘルプを参照してください。 ナンセンスで日付の愚かさを修正するためにこれが必要でした。



ステージ4.ジャンルの定義



処理後、 ジャンルタグは空白になります。 また、MusicBrainzにはジャンルを自動的に入力するオプションがありましたが、何らかの理由で機能しませんでした。 したがって、他の手段を探す必要があります。 ただし、ジャンルに関する情報に興味がない場合は、この項目をスキップできます。

結局のところ、ジャンルの自動定義により、すべてが少し緊張しました。 そして、ロシアの歌のジャンルで-さらに悪いことに。 しかし、私は最高の最悪のものを選択しようとし、 MediaMonkeyハーベスターに落ち着きました。 ところで、ポータブルインストールオプションがあります。 ジャンルを検索するために、 Genre Finderと呼ばれるスクリプトがあります。 私の知る限り、このスクリプトはLast.fm APIを使用しているため、設定には非常に注意する必要があります。設定はほとんどありませんが、最終結果はそれらに大きく依存します。 異なる設定で複数のテストパスを作成する可能性があります。

最初にアドバイスしたいのは、はいです。ジャンルを複数定義できますが、常に上からジャンルの数を制限します。 私を信じて、「私の人生で最高の歌!!!! 11」や「私にそれを聞けないよ！」のようなハードコアを一掃するよりも、ジャンルをスキップする方が良いです。 また、ホワイトリスト/ブラックリストをインストールする機能にも注意してください。 私はこれを無視し、その後手動での清掃で時間を失いました。 既製のホワイトリストは、上記のリンクのフォーラムで見つけることができます。

最初にアルバム検索モードで2〜3ジャンルを識別することをお勧めします。 その後、さらにいくつかのジャンルを追加できますが、すでに個々のトラックの検索モードになっています。 ここで、MediaMonkeyでは、結果のジャンルのリストを同時に見ることができます。

foob​​ar2000について少し説明する価値があります。セミコロンで区切られた単一のタグ内の複数の値（MediaMonkeyが書き込むように）を認識しないことが起こりました。 他のプレイヤーにはこのような問題はありません。 しかし、問題は同じfoobarを使用して解決されます。 曲のリスト全体を選択し、Alt + Enterキー（[プロパティ]コマンド）を押し、ジャンルタグをクリックして、[ ツール]- [選択] - [ 値の分割...]を選択します。 このコマンドは、1つのタグの値を「正しい」複数のタグに分割します。



ステージ5.アルバムカバー



さて、最後のステップはカバーアートです。 ここではすべてが前の段落よりもはるかに優れています。 カバーを検索できるプログラムは多数あります。 それらのいくつかは、カバーのサムネイルをファイルタグに直接アップロードすることを可能にします（ただし、この「機能」については非常にクールです）。 しかし、私の控えめな意見では、 Album Art Downloaderには幅広い可能性があります。

設定では、最小のカバーサイズ（少なくとも50 px）を上げ、「保存後に自動的に閉じる」オプションをオンにすることをお勧めします。

2つのパスを作成する必要があります（実験的に確立されたもので、iTunesを含む他のソースではより良い結果が得られません）。 まず、 Last.fmを除くすべてのソース（右側の領域）をオフにして、すべての音楽をスキャンします。 検索および分析プロセスの後、 キューマネージャーウィンドウが表示され、見つかったすべてのアルバムのリストが表示されます。 この段階で、自動モードを安全に開始し（キューマネージャウィンドウの下部にあるリンク）、完了を待つことができます。

最初のパスを実行しても認識されないアルバムがある場合は、ソースをLast.fmからGoogle Imagesに変更し、残りのアルバムの処理を開始します。 ここでは、注意して各アルバムを手動で確認することをお勧めします。 個人的には、これら2つのパスで十分でした。 他のソースは私にとって完全に不適切でしたが、試してみることができます-それぞれに固有の特性があります。

まあ、それだけです。 しかし、まだわずかなニュアンスがあります。 実際、アルバムの一部にはPNG形式の表紙が付いています。 判明したように、Google MusicはJPEGのみを認識します。実際、この形式はより一般的です。 そのため、必要に応じて、すべてをJPEGに変換できます。

すべてのPNGファイルを再帰的に検索してJPEGに名前を変更する無料のユーティリティをすぐに見つけることができませんでした。 知っている場合-それを使用します。 Photoshopでバッチ処理を行うことができます。 高度なファイルマネージャーの助けを求めることができます。 ここでそれを行うPowerShellスクリプトを残します-すべてのPNGを再帰的に選択して変換します。 元のファイルは残ります。手動で削除できます。 



 function ConvertTo-Jpg { [cmdletbinding()] param([Parameter(Mandatory=$true, ValueFromPipeline = $true)] $Path) process{ if ($Path -is [string]) { $Path = get-childitem $Path } $Path | foreach { $image = [System.Drawing.Image]::FromFile($($_.FullName)) $FilePath = "{0}\{1}.jpg" -f $($_.DirectoryName), $($_.BaseName) $image.Save($FilePath, [System.Drawing.Imaging.ImageFormat]::Jpeg) } } } cd "D:\Music\" Get-ChildItem -Filter *.png -Recurse | ConvertTo-Jpg
