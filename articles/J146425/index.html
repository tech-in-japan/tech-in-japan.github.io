Intel Fast Access Software Developer Program

今日は、ソフトウェア開発者向けのIntelハードウェアアクセスプログラムについてお話します。 Fast AccessまたはQuick Testとして知られ、Intel Software Partner Programの一部です。 開発者は、そのフレームワーク内で、さまざまなIntelハードウェアでアプリケーションをテストできます。



まず、リンク： クイックテストプログラムの公式ページ



このページで、登録し、ソフトウェアプロファイルを作成し、リクエストを送信する必要があります。

個人的には、最初の段階で失敗しましたが、登録の試行がエラーで終了しました。 これが一時的な問題であることを願っていますが、問題が発生した場合は、このプログラムを監督するArtem Timoshenko artem.timoshenko@intel.comに電子メールで連絡することができます。



次の機器をテストに使用できます。

  • Intel Core i5-660、4GBメモリ、320GB HDD、DVD / RW、WiFi Link 6200
  • Intel Core i7-960、4コア、6GBメモリ、320GB HDD、DVDRW
  • Intel Core i7-980、6コア、6GBメモリ、320GB HDD、DVDRW
  • Intel Xeon 56XX 2P、4/6コアサーバー、12/24 / 36Gb RAM、1TB SATA / 3TB SASストレージ
  • Intel Xeon 74XX 4P、4/6コアサーバー、16 / 32Gb RAM、146Gb SAS / 3TB SASストレージ
  • Intel Xeon 75XX 4P、8コアサーバー、64 / 128Gb RAM、146Gb SAS / 3TB SASストレージ
  • Sandy Bridgeラップトップ：Intel Core i3-2310、2コア、3Gb RAM、300Gb HDD
  • Sandy Bridgeデスクトップ：Intel Core i5-2400、2コア、4Gb RAM、1Tb HDD
  • Sandy Bridgeラップトップ：Intel Core i7-2630、4コア、4Gb RAM、600Gb HDD


このリストに含まれていない特定の機器が必要な場合は、フォーラムにリクエストを残すことができます。



テストシステムへのアクセスは、RDPを介してリモートで取得するか、モスクワの会社のオフィスにアクセスできます。



LiveHack：Arkanoidをテストする口実でIntelオフィスをツアーする機会。



もちろん、私はモスクワ事務所の研究室を訪問しました。 これは非常にカラフルな場所であることが判明しました。そこでは、多数のさまざまなサーバーおよびデスクトップシステムがクーラーと一緒に賑わっています。 残念ながら、彼らはそこで写真を撮ることは許可されていないので、私はあなたの想像力のために部屋を残すだけです。



また、6コアのデスクトップでリモート作業を試みました。 インスティテュートの最初の年にコースワークの一部として私の唯一のアプリケーションを開発したので、7zipアーカイバベンチマークをロードするよりも良いものは思いつきませんでした。







もちろん、RDPは少し遅くなりますが、かなり便利です。



調査中のアプリケーション自体が重い場合、または大量のデータを処理する場合、インターネット経由でテストシステムにアップロードするのは問題になる可能性があります。 連中は同じようなケースについて私に言った。 開発者の1人は200ギガバイトの容量のデータベースを必要としており、宅配便でハードドライブでオフィスに送信されました。 このような柔軟性と協力の意欲は喜ばしいことです。



もちろん、大企業はテストに必要な機器を購入する余裕がありますが、若い企業にとっては、このプログラムは非常に貴重な機会です。 8コアプロセッサを搭載した4プロセッササーバーが簡単にアクセスできる機器に起因する可能性は低く、スケーラビリティ、最適化の機会、およびそのような関連システムでの最終テストの結果を確認することは、顧客と通信する際の競争上の優位性として役立ちます。
