802.11axに期待しないものは何ですか？

まだ承認されていない802.11ax（「次世代Wi-Fi」）は、マーケティングが不十分で潜在的な顧客の心を曇らせるほど人気が​​あります。 仕様は2018年12月にのみ承認が予定されていますが、ポイントは既に購入可能です！



明快な明確化の投稿を書く時が来ました。 私がそれについて考えていた間、別の権威ある同志-Devin Akin（ あなたがプロのWLANに携わっているなら-必ず彼をフォローしてください ）はそれを私のためにしたので、私がしなければならなかったのは彼の素晴らしい記事を再紹介し、いくつかのコメントを追加することでした。 覚えておいてください、これは翻訳ではなく、語り直しです！



背景：



Wi-Fi開発に関して、最後の重要なイベントは802.11nリリースでした。



  • 標準化が行われました（a / b / gと802.11e / iのような多数の機能追加を使用）
  • MIMOの実装により、無線計画へのアプローチが大きく変わりました
  • 劇的に新しいレベルの速度が達成され、WLAN構築の原則の見直しが必要になりました。コントローラとポイントの役割が変わり、アーキテクチャが変わりました。


その後の802.11acは本当に新しいものを何ももたらさず、多くの点で単純に冗長でした。Wi-Fi：新しい速度にはワイドチャネル、特別なRF条件、「古い」802.11a / nデバイスの欠如が必要でした。 一般に、Wi-Fiハブであったため、そのままでした。 多数のクライアントを持つハブは、原則として高速にすることはできません。 さて、ハブからスイッチに切り替えると......



最初の試みは、802.11ac Wave2のフローの空間的分離に基づくMU-MIMOメカニズムの導入でした。 もしこれがあなたにとって曇ったように聞こえるなら、あなたは絶対に正しいです：操作のメカニズムはあまりにも複雑であり、排気はあまりにも取るに足らないものであることが判明しました。 しかし、PR以上のものがあり、これはすべてがっかりする結果となりました。



802.11axのタスクは、速度ではなくネットワーク容量を増やすことであり、今回は不名誉ではありません。 「アダルト」 OFDMAテクノロジー（ LTE / WiMaxに由来 ）の助けを借りて、「アダルト」セルラー通信で発生するように、チャネルをサブチャネルに「カット」し、そこにさらに多くのクライアントを起動できます。 単純に、GSMベースステーションをアクセスポイントのサイズのボックスに詰め込むのは以前は困難でしたが、今では非常に現実的です:)



さて、今度は記事（再販）



802.11ax-スイッチではない



Wi-Fiはまだ半二重です。 奇跡は起こらず、起こりそうもない。 古い制限は引き続き適用されます。 再び1990年へようこそ。



802.11axは2.4 GHzで問題を解決しません



802.11axは2.4をサポートします（ たとえば、802.11acはサポートしませんでした ）。 しかし、これには意味がありません。 私は繰り返します： ゼロセンス。 多数の古いb / g / nクライアント、干渉、およびより広い範囲（ 5 GHzと比較）を考えると、互換性モードから抜け出すことはありません。 英語を話す場合は、こちらのプレゼンテーションをご覧ください 。



802.11ax-PCF（ポイント調整機能）ではありません



それが何であるかわからないなら、それはあなたの幸せです。それにもかかわらず、それはアクセスポイントがクライアントにいつ、どのように話すかを命令できる標準の一部です（ オプション、 CSMA / CAおよびDCFの現在の市場の代わりに ） 。 ただし、802.11axはPCFを使用せず、 EDCAを使用します（ 802.11a / g以降 ）。



ポイントの順番が来たら、9つのクライアント（20 MHzチャネル）に直接送信するか、同じ9つのクライアントから同時にデータ転送を要求できます。 これは実際のOFDMAと同じであり、クールです！



Noooo ...これら9（またはそれ以下）のクライアントは802.11axを実行できるはずです。 残りはすべて船外に残されます。 したがって、十分な数の古いクライアントがある場合-互換性モードから抜け出すことはほとんどありません。 802.11axに少なくともいくつかの利点を持たせるには、 すべてを802.11ax に一度 に変更する必要があります。



802.11ax-一度にすべてではない



802.11nや11acのように、新しい標準は次々と登場します。 第一世代で

ただし、データ転送の高速化と圧縮の両方について、多くの新しい機能が追加されます。



  • 加速：1024QAMのさらに密な変調、より多くの副搬送波（20 MHzで256）、MCS10、MCS11の速度、伝送中の文字とインデントの持続時間の変更。
  • 統合：2/4/8/20 MHzチャネルに相当する26/52/106/242ユニットのリソースユニット（RU）でのサブキャリアグループ化、OFDMAアップダウン
  • その他：新しいフレーム形式、低密度パリティチェック（LDPC）、ターゲットウェイクタイム（TWT）は、モバイルクライアントのエネルギー効率を改善します。


ただし、OFDMAの飛躍を考えると、クライアントがどの速度で動作しているかを理解することはさらに困難になります。 現在、速度は一般的に変動しています（ 原則として、802.11acで開始され、一部のパラメーターは各パケットで変更されました ）。 802.11axのデータレート（MSC）の総数が2500を超えました！



802.11axは新しいアンテナを必要としません



理由は定かではありませんが、この質問はよく聞かれます。 必要ありません。



802.11axは古いクライアントのパフォーマンスを改善しません



11axポイントは、クライアント11a / b / g / n / acには役立ちません。 彼らはこれ以上敏感ではなく（他のどこにも）、 MRCを使用して信号を「回復」するためのアンテナを増やすことはまずありません。 Quallcom Atheros（QCA）は、もちろん8x8：8チップセットを約束しますが、802.11acで約束されています。 現実には、4x4：4を超えるすべてからの排気は取るに足りませんが、もちろん、マーケターはそれでも別の方法で私たちを納得させます。



802.11axはMU-MIMOを修正しません（802.11acから）



前述のように、Beamformingの802.11ac MU-MIMOは死産で、価値がなく、がっかりしました。 信じられない？ ここを見てください 。

OFDMAベースのMU-MIMOは、まったく異なるテクノロジーです。 飛行機と潜水艦のように、両方の輸送手段。

必要ありません。 これには802.11ah（HaLow）があります。 確かに、 「 飛行中のHaLow-802.11axを待っています」というトピックに関する記事は既にあります。 そして、対応する製品が確実にリリースされます。 必要ありません。 2.4すでに失われました 。



802.11axは万能薬ではありません



「ええ、はい」とあなたは言います。 しかし現実的には、Wi-Fiの各新世代には、あたかも癌、世界の飢er、エネルギー危機、すべてのジャズの問題に対するソリューションを販売しているかのように、このようなレベルの裸足マーケティングが伴います。 ネットワーク全体（ および有能！ ）802.11ax用に再構築する場合-効果があります。 他のすべての場合（ 95％+ ）-それはいつものようになります。



合計



802.11acのリリースでさえ、大規模なネットワークでは160Mzチャネルよりも5MHzチャネルのほうが多くの利点があることが明らかになりました。チャネルが狭いほど、クライアントが互いに干渉することが少なくなります。



これが、すべての本格的なネットワークが802.11ac Wave2でさえ20 MHzチャネルで構築されている理由です。 これが802.11axの興味深いところです。これは、チャネルを2MHzまでの「何らかの形で」切断することを約束します（ ただし、802.11axクライアントのみ ）。 結論として、20/40/80 MHzチャネルでの802.11acのAruba Networksからの実際のデータを含む記事の図。
