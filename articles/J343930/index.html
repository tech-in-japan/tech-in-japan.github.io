それでも、ttkの主なものは新しいウィジェットではなく、それらのデザイン（それら）です。 テーマとスタイルをカスタマイズするには、別の議論に値します。 そして、ここで私は開発者を先日（2017年11月11日）出てきた素晴らしいチュートリアルに送ります：

もう1つの不便な点は、mHOME set $ env（HOME）などのユーザーのホームディレクトリの取得です。 原則として、ホームディレクトリは、スクリプトがプラットフォームに依存しないようにするために必要です。 TKproEを介してどこにも挿入しないことが判明しました。 TKproEのグローバル変数は、TPinitVarsプロシージャを介して定義されます。 当然、グローバル変数のスコープを使用してmHOME変数を定義する場合：

上記では、ページデザイナーに戻ることについて説明しました。 ページデザイナでは、ウィジェット（場所）を自由に配置してGUIを設計すると非常に便利です。 既に述べたように、GUIプロジェクトはページデザイナによってTcl / Tkスクリプトとして保存されます。TKproEのページから既製のスクリプトを使用するか、手動で追加するのは魅力的です。 純粋な形式では、Pageのプロジェクトを実行したり、TKproE-2.20にロードしたりすることはできません。 この「不正」を排除するために、Pageからプロジェクトを実行されるtcl / tkスクリプトに移行するtclスクリプトが開発されました。 



 sh-4.3$ ./fromPageToTcl.tcl Usage: ./fromPageToTcl.tcl <file tcl from Page> sh-4.3$
