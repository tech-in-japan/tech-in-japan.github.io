Oracle Exadata



Oracle Exadataの下位構成は、Oracle Exadata Eights Rackです。 Intel X86プラットフォーム上の2つのコンピューティングモジュール、2つのデータベースサーバーが含まれています。 ほとんどの顧客は、10または16 GBのデータ転送速度を切り替えるためにクラシックストレージシステムとクラシックファイバーチャネルサーバーを使用しますが、このソリューションにはスケーリング時の深刻な応答遅延が伴うため、Oracleは切り替えに40 GBの高速InfiniBandネットワークを使用します。 Infinibandのレイテンシは最小限であり、多数のユーザーに最小限の応答時間を提供します。



Oracle Exadataで使用されるストレージシステムはExadata Storage Serverと呼ばれ、従来のソリューションとは大きく異なります。 Exadata Storage Serverの簡単な機能は、システムがOracleデータベースと連携することを「理解」し、ストレージシステムレベルからデータベースレベルへのブロックの転送を最適化し、SQLクエリを含む一部のデータベース操作がそのレベルで実行されることを保証することですストレージシステム。 したがって、Exadata Storage Serverは、ストレージシステムからデータベースレベルへのトラフィックを削減し、最高のシステム全体のパフォーマンスを提供します。



Exadata Storage Serverは、2つの6コアプロセッサ、96 GBのメモリ、I / Oアクセラレーション用のPCIフラッシュカード、長期データストレージ用の1.2 TBまたは4 TBハードドライブを備えたX4-2サーバーを使用します。



Exadata Database Serverには2つの12コア・プロセッサがあり、RAM容量は256GBまたは512GBです。 すべてのインターフェース：ネットワークとInfiniBandの両方が複製されます。



Oracle Exadata 8よりもOracle Exadata Database Machineの構成が高いのは、Oracle Exadata Quarter、Oracle Exadata Half、およびOracle Exadata Full Rackです。 初期段階では、クライアントは第8四半期または第4四半期を購入し、必要に応じて、ネットワーク容量を考慮したり、ストレージシステムへのサーバーアクセスを提供したりすることなく、構成を標準的な方法でより強力な構成にスケーリングできます。これはすべて事前に決定されています、クライアントは従来のイーサネットネットワークのみを必要とします。



ロシアで最も人気のあるソリューションはOracle Exadata Quarterで、これには2つのデータベースサーバー（合計48コア）、512 GBまたは1 TB RAM、複製InfiniBand、3つのストレージシステム、9.6 TB PCI Flash Cash、およびファイルストレージが含まれます重複、RAIDグループの再編成などを考慮した、19 TB（高速ディスク）から63 TB（大容量ディスク）の有用な容量を持つデータベース
