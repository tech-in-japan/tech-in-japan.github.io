「Nepofigizm」-非営利プロジェクトを作成した経験（3年間の暴露）









数週間前、Nepofigismプロジェクトは3年目を迎えましたが、私の意見では、これはかなり深刻な「年齢」です。 むき出しの熱意だけで機能し、投資家や志を同じくする非営利の非営利プロジェクトは1年以上生きることが比較的難しいため、少なくとも1つの事実を自信を持って述べることができます。プロジェクトは間違いなく需要があります。



この出版物は、PRの試みであるだけでなく（実験の純粋さのために、2年以上のすべての以前の出版物をドラフトに移しました）、非営利プロジェクトの開発とサポートの経験を共有し、典型的で非典型的なエラーを公開し、どのように話すかについての試みです対処しましたが、対処しませんでした。



テキストのサイズが合理的な制限を超えているという事実により、出版物は少なくとも2つの部分に分割されます。最初の部分では、プロジェクトの作成方法、発展方法、開発方法について説明します。 第二部では、私のカルマが負の値に落ちない場合、私は間違い、壊れ、失敗、欠陥、無力の攻撃、そして私たちが英雄的に何かを克服し、何かが私たちを克服した方法について話し、答えようとします最初の出版物に対するコメントの批判と質問の中心。



さらに、多くのhabrayuzersは関連するハブの出版物に対して非常に否定的な態度を持っているため、すべてのアクティブなハイパーリンクは記事の最後の部分に配置され、その数は合理的な制限を超えません。



それでは、始めましょうか？



プロジェクトについて：一般情報



「ネポフィギズム」は、互いに法的支援を提供し、経験を共有し、誰もが自分の権利を保護でき、法律が本来どおりに機能することを実際に証明しようとする、非常に異なる人々のコミュニティです。



「ネポフィギズム」はいくつかの原則に基づいており、単純なアイデアに基づいています。「誰かが何かをうまくやることができ、誰かがそれを無料でやる時間があれば、これらの人々を組み合わせてみませんか」



私たちの場合、私たちの素晴らしい国では手に入らない法的支援について話しています。これは、価格と品質の無意味で無慈悲な比率によって説明されています：弁護士の費用は「高価な」から「法外に高価な」まで変化しますが、品質は不確実です。



「無料の法的助言」とは異なり、プロジェクト参加者は回答の完全版に訪問者からのお金を必要としません。助言する者も助言される者も、商業上の義務はありません。



実際には、訪問者は質問をします-訪問者は答えを取得します-訪問者は問題を解決する実際的な経験と理論的な知識の基本的な最小限を得ます。次に何をするかは、彼の選択です。



ほとんどの場合、訪問者は、関心のある質問に対する回答を受け取っただけで、次の時間まで、または永遠に去ります。 これは頻繁に起こります。



少数のケースでは、私たちのアイデアは機能します-訪問者は自分の権利を保護することはそれほど難しくないことを理解しています、さらに、彼は経験と知識を他の誰かに移すことができ、これは原則としてプロジェクトのウェブサイトで起こります



さて、誰かがサイトに来て類似または類似の質問をすると、プロの弁護士だけでなく、自分自身または私たちの助けを借りて類似のものをすでに解決できた人も答えます。



相互の義務のない法的知識のこのような特異なリレーレース。



プロジェクト開発：技術的詳細



以前にプロジェクトのWebサイトまたはHabréの古い投稿で類似の記事を読んだことがある場合-出版物のこの部分をスキップしても安全です-そうでない場合、この情報は明らかに不要ではないと思います。

そのため、最初はWordpressで実行される通常のシングルユーザーブログでした。 正式には、この瞬間から、プロジェクトを作成する時間がカウントされます。実際、おそらくこれは完全に真実ではありません。その時点では、プロジェクト全体が私の快適なブログであり、普遍的なコンテンツの投稿は合法的なコンテンツの投稿に散在していませんでした。



正式には、Wordpressはそれほど悪くないことが判明しました。深刻なコミュニティ、オープンソースコード、および一見したところの多数のアドオンは決定的な利点でしたが、それらはすべて、地獄のような遅さと著しい食い違いという2つの欠点だけで簡単に重なりました。



当時、このプロジェクトはSpaceWebによってホストされていました（自分の能力に自信がなかったため、「安くて陽気で、高価で良い」というルールに従って、最も安価で適切な料金を選択しました）。



アプローチは報われました：頭へのすくいのストライキは無駄ではありませんでした。 より良い関税に移行すると同時に、CMSを自分の要件により適したものに変更しました。「オープンソース」、セットアップの容易さ、より純粋な社会的な「機能」（評価、カルマ、メーリング-一般的に、「独自の」 Habr、ポーカーとバージン "）。



当時、私はCMS Cogearが最も好きで、このCMSでNepofigismプロジェクトが私の人生のほとんどで機能しました。



Cogearについては何も悪いことは言えませんが、良いことしか言えません。その瞬間、システムの機能は私に満足し、RAMに対する経済的な態度は私を喜ばせ、トラフィックスパイクへの抵抗は私を襲いました。



当時はSkalaxiでさえ悪い選択肢ではないように思えましたが、Cogearは私のために働きませんでした-コミュニティは積極的に2番目のバージョンの開発を開始し、CMSの最初のバージョンは実際にはサポートされなくなりました。 LiveStreetだけが私に提供できました。



実際の運用中、LiveStreetは非常に優れていることがわかりました。比較的迅速にシステムのロジックに慣れ、問題を解決するためにコードを修正してテンプレートを変更する方法を学ぶことができました。



非常に多くのプラグインが特に魅力的です。必要に応じて、サイトの機能がユーザーのリクエストとほぼ同時に拡張され、プロジェクトの評判にプラスの影響を与えました。

ところで、すべての妨害の結果、サイトはDebian 6 + nginx + php-fpmで実行されます。 Ubuntu + Apacheに比べて大幅に増加しています。



LoadImpactのテスト結果に基づいて判断してください（最初のグラフはDebian / nginx / php-fpm / php-apc / memcache / LiveStreet、2番目のグラフはUbuntu / Apache + mod_php / MySQL / eaccelerator / memcache / Cogearです）。 どちらの場合も、（当時の）ホスティングであるキャッシングが有効になりました-Skalaxi。







ところで、 ここに実際のテストデータがあります。 それはすべてがそれほどひどく動作していないようです？



逃げた結果、Selectelのクラウドマシン上でプロジェクトサイトの雰囲気が良くなり、ホスティングコストが2倍以上低下しましたが、悪影響はありません。







Skalaxiの同様の構成では、約700ルーブルかかりました。 うまくいきました、移動できてうれしいです。



プロジェクト開発：非技術的詳細



実践が示しているように、コミュニティを開発するとき、重要なポイントはコンテンツだけでなく、フォームでもあります。 何が第一義的であるかを正確に言うことは困難です（論争は「最初の鶏肉か卵か」というカテゴリに由来します）、サイトが開発されるたびに、認識をはるかに超えて、非常に多くの憎悪を引き起こすたびに変更する必要がありました。



サイトがシンプルなシングルユーザーブログの場合、最初は特に問題はありませんでした。比較的関連性の高い法的な記事を書いて、コメントが出て、答えて、みんなが満足していました。

問題は、出席者が1日100人を超え始めたときに始まりました。 質問が多すぎて回答が多すぎると、訪問者は適切なコンテンツを見つけるのに問題が生じます。 繰り返しになりますが、Wordpressは、その制限により、原則としてコミュニティを作成するためのツールではありません。



ソリューションは表面にありました。小さなソーシャルネットワークを構築する必要がありました。



繰り返しますが、最初はすべてが素晴らしかったです。理論的な性質の記事は、対応する集合ブログの統一性に基づいて結合されました。 同時に、大多数は「助けが必要です！」という言葉で集合的なブログを与えられ、それはすべての問題を一時的に解決しました。



当時、私たちはサイトのコンテンツが「無料の法的支援、無料のSMS登録なしでオンラインでダウンロード」の状態に低下したという事実に直面しました。相互支援については話しませんでした、人々は来て、質問をし、アクティブなサイト参加者から応答を受け取りました瞬間-男の強さから、彼らの顔の汗によって、何かをしようとした3人であり、明確な良心で去った。



私はあきらめる準備ができていましたが、突然何かが起こり、プロジェクトは計画どおりに動作し始めました-どういうわけか定性的な跳躍が起こり、サイトのアクティブな視聴者が約10倍になり、相互支援システムが実際に機能し、典型的な質問すでに11回与えられた答えはどこかで消え、本当に深刻な問題に取って代わられました。



その結果、ユーザーの総数は2,000人を超え、そのうち約100人が議論、建設的な支援、経験の交換に参加する時間があるアクティブな参加者であるという結論に達しました。



出席者は徐々に増えており、相談の質は低下していないようです。最近、「質問と回答の局」から、実践的な経験の交換を優先目標とするコミュニティへの移行が飛躍的に進んでいます。



すべてがうまくいくようです。 実際には、これは完全に真実ではなく、これについて次の記事を書きます（カルマが最小20ユニットを下回らない場合）。
