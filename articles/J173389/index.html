Google Cloud Connectが終了します

Google Cloud Connectは4月30日に終了します。





Googleの別の優れたツールのサポートは終了しました。現在、クラウド内のドキュメント管理サービスについて話しています。

サービスの詳細については、 habrahabr.ru / post / 108662をご覧ください 。

ヘルプ情報はこちら： tools.google.com/dlpage/cloudconnect?hl=en



手紙の原文：

Google Cloud Connectクロージングレター
Google Apps管理者各位、



Microsoft OfficeファイルをWindows PCからGoogleドライブに自動的に保存する無料のプラグインであるGoogle Cloud Connectに別れを告げる時が来ました。



なんで？ Googleドライブはこのタスクをより効果的に処理するため、他にも多くの利点があります。 Googleドライブがコンピューターのフォルダーとしてインストールされている場合：

•MacまたはPCのドライブに保存されたあらゆる種類のファイル（Officeファイルだけでなく）はクラウドに保存され、インターネットに接続されたデバイスから利用できます。

•古いWindows専用プラグインとは異なり、ドライブはWindows、Mac、Android、iOSデバイスで動作します。

Googleドライブに引き続き注力するため、Cloud Connectは廃止され、2013年4月30日以降は使用できなくなります。



これは重要です。一部のユーザーは最近ドキュメントにGoogle Cloud Connectを使用しているため、ドライブへの移行を支援したいと思うでしょう。 対処方法は次のとおりです。

•まず、 [設定]-> [ドライブとドキュメント]-> [ドライブ]でドメインのMac / PC用ドライブが有効になっていることを確認し、 [ユーザーにMac / PC用Googleドライブのダウンロード、インストール、使用を許可する]が選択されていることを確認します。

•次に、すべてのユーザーに（1）Cloud Connectからログアウトし、（2）Cloud Connectを削除し、（3）Driveフォルダーをインストールするように依頼します。 必要に応じて、 これらのユーザー向けの手順を共有してください。



•ユーザーにウェブ上のGoogleドライブにサインインして、コンピューターに同期するすべてのファイルが[ マイドライブ ]フォルダーに追加されていることを確認します。



•完了！ これで、これらのファイル（およびそれらへの変更）は、クラウド経由ですべてのデバイス間で同期されます。 また、ユーザーは自分のコンピューターの[ マイドライブ ]フォルダーにさらにファイルを追加して、同じ結果を得ることができます。







注：GoogleドライブがMicrosoft Officeファイルの複数人編集を異なる方法で処理することを一部のユーザーに警告することができます。 2人以上が同じドキュメントで作業している場合、最新バージョンは常にウェブ上のドライブに保存されます。 重複は、他のユーザーのデスクトップのマイドライブフォルダーに保存されます。 （詳細はこちらをご覧ください 。）



そして、それだけです。 ご質問は？ ここにいくつかのFAQがあります。また、追加情報については、組織でGoogleDriveを起動するための管理者ガイドを確認することもできます 。 回答や支援についてGoogle Appsサポートに連絡し、この発表を参照してください（連絡先については、コントロールパネルの[サポート]タブを確認してください）。 ありがとう！



心から

Google Appsチーム







たとえば、このサービスが終了したことは非常に残念です。GoogleDriveを使用せずにネットワークドキュメントを操作でき、ドキュメントの変更、グループ作業、バージョン管理のマージをサポートしたためです。 現時点では、Google Cloud Connectを使用するという実践は、Googleドライブを最も純粋な形で使用するよりもはるかに便利であり、非常に便利で有望であると私たちのチームに示しています。 利点のいくつかを次に示します。

  • 他のアプリケーションを使用せずにオフィスから作業できます
  • パブリックフォルダーに一時ファイルを詰まらせることはありません
  • 優れたドキュメントマージツール
  • 特定のユーザーが受け入れる任意のトポロジにファイルのローカルバージョンを保存する機能




最後のポイントは特に重要です。なぜなら、さまざまな企業の多くの代表者がファイルを操作し、まったく異なるファイルストレージスキームを持っている場合、彼らは単一の合意に達することを強いられないからです。
