UXになったきっかけ

この記事は動機に関するものであり、ほんの数年前に不快な言葉「ウェブマスター」と呼ばれていた場所に長く留まるべきではないということです。

攻撃的であるため、高等教育を受ける義務がなく、明るい収入を約束せず、製品（通常のWebサイト）の需要を満たすために深い科学的知識を必要としませんでした。 今日、サイトはインターネット上の存在の単なる属性ではありません。 今日、このサイトは膨大な数の生きている人々の意志と意識をコントロールするツールです。 また、「ウェブマスター」はその開発には適していませんが、プロフィールの狭い専門家チーム全体が必要です。

今日、サイトには多くの顔が必要です。モバイルデバイス用のバージョン、クロスブラウザバージョン、さまざまな画面幅のバージョン、およびモバイルアプリケーション全体。これらは、インターネット経由で提供されるサービスの「マストヘッド」になります。 このようなタスクの規模が拡大する前に、Webベースのオールインワンハンドは失われます。 彼は単に開発の正しい方向を決め、WWWとともに成長し、自分自身とすべての人類に利益をもたらすべきです。



この問題を自分で解決した方法についてお話します。



その5月1日は特別なことを前兆とはしなかった。 全体の考えが頭に浮かんだのは、「不可能はない」ということです。

私はITの分野で専門的に働くことを長い間夢見てきました。 実際のところ、私はこれを8年間続けています。 私は、ターンキーウェブサイトの開発、モバイルインターフェースの開発に豊富な経験を持つフリーランサーであり、印刷広告のデザインに対して十分な報酬を得ています。 しかし、「ITの分野で専門的に仕事をする」と言うときは、もっと意味があります。

もっと高級で、もっと真剣で、もっと意味のあるものが欲しい。 私に認識を与えるだけでなく、私の創造的な野心を満足させる何か。

創造的な野心、原則として、私はお金で満足しています。 したがって、質問が発生しました：



そして今、彼らは生地を与えていますか？


デザイナーであることは、レイアウトデザイナーやプログラマーよりもはるかに収益性が高いことに驚きました。 これは、プリンターだけでなくWebデザイナーにも適用されます。

労働市場、地域-モスクワの分析された需要と供給。 私の経験を持つデザイナーは、この経験が合法である場合、私の経験を持つ弁護士に支払うのとまったく同じように支払われます。 立派な弁護士になってもうまくいかなかったことを後悔した。 しかし、私はそれを後悔する必要があることを覚えていません。 結局のところ、環境全体の希望に反して、私はまだ人気のWebデザイナーになりました。 これは心から来たので、二重に価値があります。

私は自分自身の重要性に気づき、誇りを持って安らぎながら寝ました。

朝、私は新しい考えに執theyに目覚めました：「なぜ彼らは才能のためにとても簡単に分配されるので、このお金を取りに行かないのですか？」そして私は自分自身に言いました：「はい！」



そして、私の中の猿-同時に賢くて美しい猿-は、2つにほとんど壊れました。 そして、どのタレントをより高価に売りたいですか？ 結局のところ、フリーランスは、ウェブデザイナーがデザイナー、プログラマー、seo-optimizer、コピーライター、ユーザビリティアナリスト、SMMマーケティング担当者、マーチャンダイザー、そして流な英語のプロジェクトマネージャーになることを義務付けています。



本格的にホバリングしました。 結局のところ、「雇用に入る」とは「狭い専門分野に進む」ことを意味し、今後数年間の開発の見通しの1つだけを概説します。 そして、ITの経験はとても不機嫌な友人です。 一瞬背を向けることで失うのはとても簡単です。 IT業界は急速に発展しているため、これを納得させる必要はありません。



それで、私は給料のテーマに関する研究を続け、人材紹介会社に頼りました。 たとえば、IT分野の国内HR市場のプロフェッショナルプレイヤーの1人であるBigFish （http://www.bfrecruitment.ru/）は 、ウェブデザインの専門家や人々に提供される給与範囲に関するさまざまなバリエーションで私の想像力をかなり甘やかしてくれました。まれな表現。 その中には、デジタルデザイナー（どんな大物でも明らかです）、WEBアーキテクト、WEBデザイナー、Webインターフェースデザイナー、クリエイター、ストラテジスト、UI / UXデザイナーがいます。



これらの職業の本質は彼女であるという事実にもかかわらず、後者は給料の点で残りから2歩離れて誇らしげに立っていました-BigFishがこの目に見えない人を捕まえて、彼の雇用で良いお金を稼ぎたい餌です。 餌の大きさから判断すると、それは特定の知識と経験を備えた最高クラスの専門家であると想定されていました。



方法を知るのは面白いですか？ さらに進んでいます。



UI / UXデザイナーは誰ですか？


「UI / UX」のこのような奇妙な組み合わせは、2つの異なる専門家になりたくない雇用主の想像力にのみ存在するという事実から始めなければなりません。 一方、UIおよびUXデザイナーは、機能的および論理的に2つの異なる職業の本質です。

解読する。 ユーザーインターフェイス（UI）デザイナー - ユーザーインターフェイスであることが判明するすべてのものの外観を設計します。 おもしろいですが、場合によっては、リーフレットもインターフェイスであり、驚くほどカスタムです。 不要なものは破棄し、簡潔にまとめます。ユーザーインターフェイスは、ユーザーに特定のアクションを実行するように促すフロントエンドインターフェイスです。 現代の現実のグリップの中で、世界のすべては販売に帰着します。 UIデザイナーもそれらを取得するように求められます。 UIデザイナーが操作するために必要な主な知識は、使いやすさです。 そうそう！ 高等教育が必要です。



ユーザーエクスペリエンス（UX）によりますます複雑になり、興味深いものになりました。これは、インターフェイスを使用したユーザーの "経験"です。 UXデザイナーのタスクは、特定のツールを使用してこのエクスペリエンスを抽出および測定し、そこからインターフェイスの改良バージョンの設計に使用される新しいエクスペリエンスを構築することです。 インターフェイスの設計は、UXデザイナーの仕事でもあります。 したがって、彼は、情報アーキテクチャ、インタラクションデザイン、グラフィックデザイン、コンテンツなどのUXコンポーネントを操作できる必要があります。 つまり、これを知り、製品の構造を正しく入力できるようにすることです。



BigFishのUI / UXデザイナーの求人の要件によると、このポジションの候補者は高等教育を受けている必要があります（さらに、教育プロファイルは示されていません）が、専門の高度なトレーニングコースの存在は歓迎されません、私は義務です。 また、モバイルアプリケーションのインターフェイスを設計する能力、一般的なERPおよびCRMシステムの知識、および設計の分野での経験\ Webデザインが必要です。 そしてただ何か？



これらのスキルはすべて利用できるようです。 彼らは自由に平均的なウェブ開発者を持っています。

彼らの高レベルを実証することは残っています。 ポートフォリオの再設計、新しいWebサイトの作成、パフォーマンスとアップワークに関するプロファイルのコーミングを決定した理由は何ですか（国内FLについて言及する価値はありますか？）。 そして最も重要なことは、自分自身に新しい経験を得る機会を与え、それをあなたの夢への道の一歩に変えることです。 私は自分のポートフォリオの弱点を分析し、新しいプロジェクトの開発でそれらを修正することにしました。



残念ながら、すべての知識が経験によってもたらされるわけではありません。 多くの人は伝統的な情報源から学ぶ必要があります。

突然。
