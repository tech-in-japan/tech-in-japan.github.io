SunがJ2MEプロジェクトを展開

Java Standard Edition（J2SE）環境の標準バージョンはデスクトップコンピューター用に設計されていますが、現在では携帯電話が非常に強力になり、そこでも動作するようになりました。 したがって、別個の「モバイル」J2ME環境は必要ありません。 Javaの父と呼ばれるSunの副社長James Goslingは、Javaが普遍的な環境になるように、やがて会社はJ2MEの開発を放棄すると言いました。



これは、Javaアプリケーションの個別のモバイルバージョンを作成する必要がない開発者にとって素晴らしいニュースです。 従来のプログラムは、問題なく携帯電話やスマートフォンで機能し始めます。 すでに、モバイルプラットフォームとデスクトッププラットフォームの差がどんどん少なくなっていることは明らかです。 これを明確に確認できるのがApple iPhoneコンピューターで、その機能は携帯電話ではなくデスクトップコンピューターに似ています。 たとえば、わずかな変更を加えて標準のSafari Webブラウザーを実行します。



サンの管理によると、デスクトップアプリケーションとモバイルアプリケーションの統合には何年もかかります。 少なくとも10年間は​​、携帯電話でJavaの簡易バージョンが実行されます。



Intelは現在、x86ファミリプロセッサの携帯電話への導入に取り組んでいます。 Ubuntu MobileおよびMobile Firefoxのモバイルバージョンはまもなく利用可能になります。 これはすべて、モバイルコンピューターとデスクトップコンピューターの収束を示しています。



同時に、問題もあります。 主なものは、さまざまなデバイスのJ2ME拡張機能の断片化です。 これは、Javaが常に配布されている「1回書き込み、どこでも実行」という有名なモットーとは直接矛盾しています。 実際には、プログラムは特定のJ2ME拡張用に作成されており、それらは常に相互に互換性があるわけではありません。
