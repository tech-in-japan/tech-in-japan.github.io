Joker Java Conference：一緒に論文を選ぶ！

ハブル、ようこそ！



サンクトペテルブルクのJavaユーザーグループである私たちは、夏の間、10月15日にサンクトペテルブルクで開催されるJoker Java会議を準備しています。 そして、会議プログラムに関与しているreal_alesが、レポート用のdofigアプリケーションがあり、会議プログラムの場所がはるかに少ないことに突然気づいた昨日まで、すべてがうまくいきました。



もちろん、私たちはすぐにSkypeで集会を開きました。そこでは、どのレポートを残し、どのレポートを破棄するかを理解しようとしました。 そして、思わぬ障害がありました。 私たちは、いくつかのレポートがどれほど興味深く、関連性があるかを理解するために、いくつかの分野の専門知識が愚かに欠けていることに気付きました。



そして、私たちは実験を行い、このレポートまたはそのレポートにどれだけ興味があるかを評価するためにハブロフスク市民に依頼することにしました。



会議に参加しないハブロフスクの住民でさえ、投票から利益を受け取ることに注意してください。 ほとんどのレポートのビデオを作成して公開する予定です（@alexbelによると、4月のJPoint会議の場合のようにすべてのレポートを完全に入力すると、誰も会議に参加しません）。 したがって、サンクトペテルブルク以外のすべての有権者は、彼にとって最も興味深いと思われるレポートを正確に記録で見ることができます！



一部のスピーカーは、複数のレポートを作成しないとすぐに言ったことに注意することが重要です。 したがって、 TheShadeとWalrusの 18のレポートすべてが最終プログラムに含まれることを期待しないでください。 さらに、多くのスピーカーはすでに私たちにとってより望ましいレポートを選択していますが、これも変更するのは非常に困難です。 さらに、新しいレポートと新しいレポートが送信されています（ real_ales 、このプロセスを何らかの形で停止できますか？）。 そして最後に、私たちの素晴らしいスポンサーが提示するレポートがありますが、これは理解されるべきであり、優先度が高いでしょう。 私たちは、たとえそれらが後援されていても、弱い報告を見逃さないことを約束します！



投票する前に、会議プログラムについてよく理解してください。 以下のレポートがあなたにとって興味深いと思われる場合は、単にチェックを外してください。



事前に感謝します
