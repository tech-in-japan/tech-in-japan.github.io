9ファクト：デブリーフィング





3月中旬、私たちは12月にHabrahabrで書いたスタートアップ9facts.comを実際に閉鎖しました。 そして5月までに、私はまだこの投稿を書くために成熟しました。



最も重要なものから始めます。



どのような間違いを犯しましたか？



1.アイデアは検証されていません



スタートアップにとって最も重要な仮説：



  • 次のカテゴリのユーザーが製品を使用します。...
  • 製品市場のボリューム：...


それらのそれぞれをチェックする必要があります。 これを確認する最も確実な方法は、プロトタイプを開発することです。他のすべて（ポーリングなど）の精度ははるかに低くなります。 実際、私たちはこの段階を逃し、比較的高品質の製品をすぐに作ることにしました。



これは、ユーザーを見つける製品を製造しているという絶対的な自信を持っていたからです。 実際、それは単なる仮説でした。



アイデアが危険であるというシグナルは次のとおりであると言う価値があります。



  • ユーザーアクティビティを記述するために同様のモデルを使用した成功したスタートアップは見つかりませんでした。 「もちろん、これは誰もこれを前に考えたことがないからです！」-その時私はそのようなことを考えました。
  • 私とプロジェクトの他の参加者がインタビューしたほぼすべての少女と女性は、なぜ彼らがそのようなサイトに絶えず行くべきなのかを完全に理解していなかったことに驚かされるべきです。 男性はサービスの説明に非常に楽観的に反応しましたが、女性はより懐疑的でした。 真実は、彼らが評価においてより実用的であるということであるように思われ、我々はこれに注意を払うべきであり、単に彼らが私たちのCAではないという事実にすべてを責めるべきではありません。


これにより、次のような一連のイベントが発生しました。



  1. サービスの最初のバージョンを開発するために割り当てられたリソースが多すぎます。 プロトタイプはありませんでした。
  2. サービスを開始した後、すぐにユーザーベースが希望どおりに成長していないことが明らかになりました。
  3. これを修正するためにいくつかの試みを行いました（UXを改善し、「アンケート」にバイラル機能を追加し、コンセプトをわずかに変更する意味があるかどうかを確認しました）-役に立たない。
  4. 実際、重要な問題はユーザーエンゲージメントが低いことです。 簡単に言えば、なぜユーザーが繰り返し表示されるのかは完全には明らかではありません。 しかし、これを理解するとすぐに（これは1月から2月に起こりました）、現在のサービスの概念と利用可能な予算の枠組み内でこれを修正する方法がわからないことが明らかになりました。
  5. そして、それがプロトタイプから始める必要があった理由です。 サービスでのユーザーの作業をシミュレートする真剣な試みでも、ユーザーエンゲージメントの問題に直面しているという結論に至る可能性があると思います。


2.無効なリスクモデル



製品がユーザーの関心を引くと保証されると信じていたので（1を参照）、最初の数か月間、潜在的な競合他社のリスクが最も重要であると考えられました。 したがって、私たちの目標は、できるだけ早く高品質の製品を生産することでした。



競合他社が登場するという事実に関して、私がどれだけ過ちを犯したかを言うのは難しいです。Facebookの秋に、Open Graph APIが登場しました。これは、非常に似た問題の解決を目指しています。 しかし、競合他社が恐れるのは、市場とユーザーがすでにいる場合だけです。 彼らがまだそこにいなければ、競合他社について全く考えるべきではありません。



3.リソースの不適切な割り当て



その結果、最初の6か月の間に、5月に「調達」した資金のほとんどすべてを使って、開発に必要以上の資金を費やしました。 7月から9月にかけて、最大8人が同時にプロジェクトに取り組んだと言えば十分です。



チームの規模が大きく、プロトタイプが不足しており、それに応じてユーザーのライブレビューが行われたため、優先順位にマイナスの影響がありました。



  • キーワードの分類法を作成するのにかなりの時間を費やしました（ freebase.comから多くのカテゴリの用語を「デフレートしました」）が、発売時にはそれなしで行うこともできました。
  • グループを完全に「採点」する必要がありました。 しかし、リソースがあったので、それらを作成しました。
  • 「友情」のあるUI全体は、まったく価値がありませんでした。 代わりに、Facebook / LinkedInなどですべての友達を数えるのが理にかなっています。 お友達や9factsで。 別のサイトでの「友情」はそれ自体が悪であり、完全に粗雑なプロトタイプを立ち上げたとしても、すぐに理解できたと思います（実際、これは11月に起こりました）。
  • スケーリングのためにサービスを準備する作業を行うことができませんでした-今ではその理由が十分に理解されています。


これは実行する価値がなかったものの完全なリストではありません-振り返ってみると、エラーはしばしば明白に見えます。



重要なことは、もし私がNo.1（そしてその結果-No.2とNo.3）を間違えなかった場合、初期段階のチームの規模ははるかに小さくなり、その結果、不必要なことをする機会がはるかに少なくなるということです。 私たち3人がそれを行うと確信しており、9月から10月にプロトタイプも発売します。 はい、彼は少し簡単だったでしょうが、彼は主要な質問に悪く答えなかっただろう。



おそらく、「経済的」オプションのマイナス点は、スタートアップロケットに乗らないことだけかもしれません。 しかし、これも議論の余地があります。スタートアップロケットとそれに続くすべての同様のイベントは、私にとっては明らかな利益にもかかわらず、多くの時間とエネルギーを要したためです。



4.「悲観的な」計画の欠如



率直に言って、すべてがうまくいかないように思われる場合、どうすればよいかわかりませんでした。 このような状況では、サービス自体とそのプロモーションへのアプローチの両方を変更する必要があり、これらすべてのオプションが事前に真剣に考慮されていなかった場合、これはすべて非常に困難です。



「知らなかった」ということは、私がまったく知らなかったことを意味するものではありません。サービスの概念を変更したり、バイラル機能を追加したりしましたが、いずれも機能しませんでした。 今では、サービスコンセプトの開発中であっても、これらすべての欠点を見ることができたように思えます。



「悲観的な」開発計画が主なものである必要があります。



投資家の追加



レオニードとアレクサンダー（「サーチライトベンチャーズ」）は、彼らが犯した重要な間違いを次のように呼びました。 このお金がなければ、不必要なことをするために8人を雇うことはできません。 私たちへの教訓。」



X-tensive.comの他の2人の共同創立者とプロジェクトを共同融資することに同意することで、これに貢献したことは言うに値します。



投資家によるフライトのより詳細な分析は、レオニード・ヴォルコフの対応する投稿に記載されています-私はあなたがそれを読むことを強くお勧めします。



結論



  1. このアイデアには費用はかかりません。
  2. ITスタートアップの最も重大なリスクは、市場への製品の不一致であり、したがって...
  3. アイデアを検証する必要があります。 ITスタートアップの新しいアイデアを検証するためのほぼ唯一の信頼できる方法は、プロトタイプを開発することです。
  4. プロトタイプがない場合、製品の最初のバージョンの開発にお金をかけることは非常に危険です。 おそらく、私たちのケースでは、経済的なバージョンで2〜3倍を費やすことが可能であり、したがって、適応するために数倍の時間を得ることができます。
  5. プロトタイプの開発および発売の段階での最適なチームサイズは2〜3人です。 チームの人数が少ないほど、余分なことが行われる可能性は低くなります。
  6. どんな場合でも、すべてがうまくスムーズに進むという事実に頼るべきではありません。 99％のケースでは、すべてが創業者が見たものとはまったく異なります。 したがって、「スキニー」（経済的な）スタートアップが開発の唯一の合理的な選択肢です。
  7. 成功の二次的な指標（スタートアップコンペティションでの勝利など）はまったく役割を果たしません。 成功の唯一の重要な指標は、ユーザーベースまたは収入の増加です（現時点での目標によって異なります）。
  8. サービスのバイラル機能に依存しても、サービス自体が面白くない場合は、何も良い結果にはなりません。 言い換えれば、バイラル機能はベンチャーキャピタルと同じロケット燃料です。 それらを使用するには、最初にロケット自体を構築する必要があります。


おもしろい：上記の内容には、新しくてユニークなものは何もありません。 スタートアップはまったく異なる方法で「離陸」と「落下」するという事実にもかかわらず、実際に成功するモデルを構築する法律は一般化できます。



  • 1つのモデルのみの開発にすべてのお金を費やすと、あなたのリスクは非常に高くなります
  • 小さくて安価なモデルは、原則として、同じ形式の本格的なモデルの飛行品質の良い評価を提供します
  • エンジニアのチームの構成が、企業の成功を80〜90％決定します。


この意味で、エリック・リースとスティーブ・ブランクは絶対に正しいです。 このようにしてやってきた人々の経験を最大限に活用すべきです。



スタートアップについて読むとき、あなたが注意を払いそうにないものをリストしようと思います。



1. 「The Lean Startup」などを読んでいるという事実は、著者のすべての推奨事項を最初の機会に実践できることを保証するものではありません 。 個人的には、自分の考えは明らかに良いものであり、したがって検証を必要としないことを認めて、自分を欺きました。その結果、私は素晴らしいが非常に高価なレッスンを受けました。



2. すべての推奨事項に完全に従ったとしても、これは失敗の大きなリスクを排除しません 。 実行可能なスタートアップ管理方法論の目標1は、投資収益率の平均値を最大化することです（つまり、スタートアップの大規模なサンプルで）。 設立チームの質やアイデアの実行可能性などの要因に大きな影響を与えることはできません。 したがって、障害が発生した場合の損失を最小限に抑え、適応までの時間を最大化することは、優れた方法論が平均的なスタートアップに与える最善の方法です。



3. スタートアップの閉鎖は、創業者にとって非常に不快なプロセスです 。 Quoraの対応する質問に応じて9factsを閉じることについての私の気持ちを少し詳しく説明しました 。 そのとき、私のブログでさえ、ビジネスとムードであまりうまくいっていなかったことが顕著でした-新しい年以来、私は実質的に何も書いていません。 次のプロジェクトを決定したらすぐに、ブログに「戻る」と思います（しかし、あなたは私を待たずに-今すぐ購読できます :)）; それまでの間、この方法で節約された時間が無駄にならないことを嬉しく思います。



4. アイデア段階でのお金は、すでに製品を持っている段階よりもはるかに簡単です 。 アイデアの段階では、投資家、チーム、そして彼が製品のアイデアを好むかどうかについて、あなたがどのような印象を与えるかによってすべてが決定されます。 開発者およびチームリーダーとして非常に競争力があり、アイデアが完全に正気でない場合、プレゼンテーションがすべてであっても、この段階で投資家を見つける可能性は非常に高くなります。



すでにプロトタイプや製品を持っている場合は、すべてがはるかに複雑です（ちなみに、品質は重要ではありません）。この段階では、通信する投資家はすべてトラクションに関心があります。 ユーザーベースの成長、有料チャンネル登録者数、売上高、毎月の視聴者など。 -製品の可能性を特徴付ける指標。 そして、あなたが良い成長を見せられないなら（そしてこれは初期段階で少なくとも月に20-30％、そしてより良い-50-60％）、あなたのビジネスは悪いです。 「始めたばかりで、視聴者を評価するのは時期尚早です」というスタイルの会話だけでなく、あなたに役立つプレゼンテーションはありません。 このような状況では、投資家は見逃さない態度を取る傾向があります。リスクを盲目的に取るよりも2〜3か月待って結果を評価する方がはるかに合理的であるため、同意する必要があります。



5. 以前、私はあなたのアイデアの質をチェックするために、この段階で投資家を必ず見つけるべきだと思っていました。 今では、製品（またはMVP）のプロトタイプを単独で （つまり、専ら時間と設立チームの資金を犠牲にして）プロトタイプを作成する方がほぼ常に良いと信じています。 あなたの製品が本当に好きな最初のユーザーがいるので、投資家を引き付けることは価値がある。



なんで？

  • 前の段落では、アイデアの段階で投資家を見つけた場合、これは設立チームとプレゼンテーション能力の95％の評価になることを示しています。 アイデアは、妥当性の最小テストに合格するだけです。 ちなみに、これはアイデアに何の費用もかからない理由を説明しています。
  • 利用可能なリソースに対する厳しい制限は、MVP機能のより良い選択につながります。
  • ほとんどの場合、このようなブートストラップには時間がかかります。 プロトタイプの作成にすべての時間を費やすことはできません。 一方、意思決定の時間も増えます。つまり、よりバランスの取れたものになります。
  • 会社の開設、投資家との交渉や合意、従業員の選択と雇用、報告などに時間を費やす必要はありません。
  • 投資家およびチームの将来のすべてのメンバーのリスクを大幅に削減し、おそらく最初の投資時にスタートアップの推定コストを5〜10倍増加させます。 これがすべてにとって重要ではないと思われる場合、新興企業の95％が1〜2年以内に閉鎖されることを忘れないでください。 これが見た目ほど魅力的ではない理由がよくわからない場合は、項目3を読み直して、約95％を思い出してください。
