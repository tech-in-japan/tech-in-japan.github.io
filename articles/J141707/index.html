ネットワークのないブラウザでマップ

エントリー



空き時間には、ミンスクでATMを見つけるためのアプリケーションを作成します。 そして、どういうわけか、休暇に行くと、私は電話でインターネットなしで放置されました。 すべてはうまくいきますが、ATMを見つけてお金を引き出し、電車に遅れないようにする必要がありました。 アプリケーションを開き、オフラインでカードを使用できなかったことに非常に失望しました。 もちろん、私たちの時代には、ネットワークに接続せずに家を出ない方が良いですが、それでもなお、都合の悪い時間にお気に入りのモバイルデバイスのインターネットがなくなる可能性があります。



携帯電話で他のアプリケーションを見ると、以前にダウンロードしたマップの一部をキャッシュしているだけです。 これは部分的には役立つかもしれませんが、問題を完全に解決しませんでした。 その後、オフラインで地図を表示できるかどうか疑問に思いました。 私のアプリケーションはネイティブではなく、PhoneGapに基づいているため、ブラウザーベースであるため、特にgoogle map api v3を使用してブラウザーアプリケーションのマップをキャッシュする方法について説明します。



アイデア



一度、すべての後に、GoogleマップAPIを使用すると、独自のマップ実装を作成できることを思い出しました（たとえば、 OSMにアドバイスします）。 すぐに利用可能な実装をスリップするというアイデアがすぐに出てきました。これは、接続がある場合はマップをキャッシュにダウンロードするか、アプリケーションにマップキャッシュを提供することで実行できます。



最初はアプリケーションキャッシュを使用することを考えていましたが、このAPIはキャッシュの読み込みを管理するための広範なオプションを提供しないため、このベンチャーを放棄しました。



最終的に、アプリケーションにキャッシュを提供することにしました。



localStorageにスプライトを保存するというアイデアも思いつきましたが、この実装には大きな欠点があります。

  1. localStorageのサイズ制限。
  2. データは、実際のサイズより約30％大きいbase64に保存する必要があります。


同期APIの実装が不足しているため、indexedDBまたはwebSQLにスプライトを保存するという考えを捨てなければなりませんでした。



あるかどうか



私が自問した最初の質問は、カードをキャッシュする価値があるのか​​ということでした。 つまり、特定の都市の地図が使用され、詳細な詳細が必要かどうかが決まります。 私の場合、小さなズーム（10〜15）のためにMinskキャッシュがあれば十分でした。



2番目の質問：キャッシュはどのくらいのスペースを占有しますか？ 平均スプライトサイズが20 kbの場合、理論的にはズーム10（ミンスクが完全にフィット）の場合、11-4（100 kb）、12-16（420 kb）、13-64（1.7 mb）、14-256（6.8 mb）、15-1024（27 mb）。 14のズームキャッシュで十分なようです。



ダウンロードする



キャッシュが実際にどれだけのスペースを使用しているかを調べるために、実際のマップと実際のスプライトを取得することにしました。 これを行うには、いくつかの学校の問題を解決する必要がありました：多くのポイントから最小周囲を持つポリゴンを作成し、極座標をマップスプライトの座標に変換 し、ポリゴン内の スプライトを見つけます 。 スクリプトの準備が整った後、スプライトをダウンロードし、次の結果を得ました（括弧内はこのズームの合計スペースです）。

ブラウザーですぐにオフラインで作業を確認するには、 http：//offline-map.appspot.comにアクセスし、[Webストレージの準備]をクリックし、スプライトが読み込まれるまで数秒待ってから、インターネットをオフにし、WebStorageGmapをクリックしてお楽しみください。



「Webストレージの準備」ボタンはスプライトをlocalStorageにアップロードし、「Webストレージのクリア」はそれに応じてクリアします。



カードの種類：

  • マップ-標準のGoogleマップマップ。
  • 衛星-衛星からの標準のGoogleマップマップ。
  • OSM-Google APIを使用したOSMカードの実装。
  • MyGmap-標準のGoogleマップマップの実装。
  • LocalGmap-ローカルに（サイトと同じホストに）保存されたマップの実装。
  • WebStorageGmap-localStorageに保存されたマップの実装（最初に「Webストレージの準備」を実行する必要があります）。
  • LocalMyGmapは、LocalGmapとMyGmapのハイブリッド実装であり、データがマップ上の特定の領域にある場合、LocalGmapのように機能し、それ以外の場合はMyGmapのように機能します。
  • WebStorageMyGmapは、WebStorageGmapとMyGmapのハイブリッド実装であり、データがlocalStorageにある場合、WebStorageGmapのように機能し、そうでない場合はMyGmapのように機能します。
