HPE StoreVirtual VSAアーキテクチャ

仮想化サーバーを展開するために共有ストレージを必要とする比較的小さなIT環境を持つ多くの中小企業は、ソフトウェア定義のStoreVirtualアーキテクチャに理想的に適しています。 ただし、大企業のリモートオフィス/支店として。









最新のHPE StoreVirtual VSA Architectureテクニカルデータシートがリリースされました。これについて少しお話ししたいと思います。 StoreVirtual VSAのアーキテクチャに8年以上取り組んでいます。 現在、販売されているライセンスの数は数百万であり、このアーキテクチャが機能するノードの数は数万です。 つまり、ソフトウェアデファインドストレージシステムの経験があります。



StoreVirtual VSAを競合他社と区別するいくつかの重要な側面を検討したいと思います。



  1. これはシンプルなアーキテクチャであり、引き続き簡素化を進めています。 これに基づいて、Hyper Converged 250およびHyper Converged 380ソリューションは機能し、電源がオンになってから仮想マシンの展開まで数分で完了します。 サーバーへのVSAのインストールには、ほぼ同じ時間がかかります。 近い将来、StoreVirtual VSAソフトウェア定義ストレージシステムでの作業がいかに簡単かを示す新しいビデオを準備する予定です。



  2. スケーラブルなアーキテクチャです。 インフラストラクチャの成長に応じて、環境に新しいVSAデバイスを追加して、容量を増やし、生産性を高めるだけです。 ところで、先日、2つのStoreVirtual VSAを使用している顧客と話をしました。 彼は生産性を高める方法を尋ねました。 彼が別のソフトウェア定義のストレージシステムであるVSAを追加すると、生産性が50％向上し、2つであれば生産性が2倍になると答えました。 彼は信じなかったので、私は比fig的に説明しようとしました。 1つのStoreVirtual VSAが1秒あたり1000 I / Oを提供する場合、2ノードシステムは2000 IOPを提供します。 3番目のノードを追加すると、3,000 IOPが得られます。つまり、パフォーマンスが50％向上します。 4番目のノードを追加すると、4000 IOP（2ノードシステムの2倍のサイズ）になります。 StoreVirtual VSAのスケーラビリティは非常に予測可能です。 1000 IOPでのパフォーマンスは一例です。 ノードの実際のパフォーマンスは、使用するドライブの数や種類などの要因によって異なります。



  3. これは高可用性アーキテクチャです-もちろん、誰もが100％の可用性を得ることができるわけではありませんが、ダウンタイムなしでStoreVirtual VSAを7年間使用したBart Hingensと私が用意したポッドキャストを聞いてください。 どのようにしてこれを達成しましたか？ ネットワークRAIDに焦点を当てたホワイトペーパーをご覧ください。 ネットワークRAIDはノード間でデータを分散します。これにより、ノード障害に対する保護が提供され、たとえば、他の全員が仕事を続けているときに単一ノードでOSを更新できます。 しかし、私（および多くのお客様）が特に気に入っているのは、拡張クラスターを作成する機能です。 StoreVirtualノードを地理的に（たとえば、2つのサイト間で）分散して、災害対策ソリューションを作成できます。 その後、データセンターが完全に故障していても、StoreVirtual VSAにはまったく影響しません。



  4. 柔軟なアーキテクチャ-StoreVirtual VSAは、Hyper-V、VMware、およびKVMハイパーバイザーを使用できます。 この仮想ストレージシステムは、独自のサーバー、既存のサーバー、またはハイパーコンバージドシステムに展開できます。



  5. 安価です。機能を無料でテストできます（各ProLiantサーバーに1 TBのライセンスが付属しています。hpe.com/ storage / TryVSAから試用版をダウンロードすることもできます）。 有料ライセンスのコストは、StoreVirtual VSAがソフトウェアデファインドストレージシステムの分野の他のソリューションよりも安くなるように設計されています。



最近、他のソリューションがソフトウェア定義ストレージシステムの市場に登場しました。 しかし、StoreVirtual VSAアーキテクチャが最良の選択だと思います。その理由は次のとおりです。



  • 制限なし：あらゆる機器、ハイパーバイザー、クライアントを使用できます。



  • 最高のハードウェアとソフトウェア：HPE ProLiantは世界で最も人気のあるサーバーですが、他のサードパーティ製機器を使用できます。 コードの信頼性は、StoreVirtual VSAの8年の歴史に支えられています。 さらに、ネットワークRAIDテクノロジーは最大の耐障害性を提供​​できます。



  • 業界標準のサポート：StoreVirtual VSAは、独自のプロトコルではなくiSCSIプロトコルを使用します。 なぜこれが重要なのですか？ iSCSI（仮想化、ハードウェア、またはその他）を使用するサーバーは、VSAデバイスの共有容量を提供できるためです。



  • 合理的なコスト：ほとんどすべての場合、StoreVirtual VSAアーキテクチャのライセンスコストは他のソリューションよりも大幅に低くなります。



  • さらなる自由：誰もが独自のハイパーバイザーへのバインディングを好む可能性は低いです。



公式のStoreVirtual VSAデータシートを読んで、 hpe.com / storage / VSAにアクセスすることを強くお勧めします。 無料の1TB VSAは、 hpe.com / storage / TryVSAからダウンロードできます。
