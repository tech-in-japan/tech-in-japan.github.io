Android文字列リソースの使用

プログラムで使用されているテキストは、画像や音声と同じ別個のリソースであることをすぐに理解できます。 しかし、数十のファイルでプログラムの名前を数回変更するか、5、6つの異なる場所で同じ構文エラーを修正する価値があり、コードとは別に行を保存する必要が明らかになります。

Androidでは、文字列リソースの操作が非常に便利になり、最初は問題が生じません。 公式ドキュメントでは、記事String Resourcesで説明されています 。 ファイルプロジェクト\ res \ values \ strings.xmlで 、文字列とその名前を設定し、その後、Activityでこの名前の文字列を読み込みます。

strings.xml 
<?xml version="1.0" encoding="utf-8"?> <resources> <string name="appName">Project Name</string> </resources>


ProjectActivity.java 
 public class LoadingActivity extends Activity { @Override public void onCreate(Bundle savedInstanceState) { String applicationName = getString(R.string.appName); } //... }


同じ行をxmlファイルレイアウトフォーム（レイアウトリソース）で使用できます。 新しいプロジェクトを作成してmain.xmlファイルを開くと、これがどのように実装されているかを確認できます。



main.xml 
 <?xml version="1.0" encoding="utf-8"?> <LinearLayout xmlns:android="http://schemas.android.com/apk/res/android" android:orientation="vertical" android:layout_width="fill_parent" android:layout_height="fill_parent" > <TextView android:layout_width="fill_parent" android:layout_height="wrap_content" android:text="@string/appName" /> </LinearLayout>


リソースの説明



さらにいくつかの興味深いことがあります。これらはドキュメントにも記載されていますが、あまり広く使用されていません。

文字列（および他のリソース）を含む複数のファイルが存在する場合があります。 主なことは、それらがプロジェクト\ res \ values \フォルダにあり、 xml拡張子があり、名前は小文字の英文字、数字で構成され、スペースが含まれていなかったことです。 内部構造は、 strings.xmlファイルの構造を繰り返す必要があります。 何に使用できますか？ 1つのファイルには、翻訳が必要なすべての行を保存し、2番目の行には翻訳を必要とせず、3番目のファイルにはWebサーバーへのリクエストをコンパイルするために使用する定数を保存します。

XMLファイルには、保存されたデータの形式に関する小さな制限があります。 平文では、文字「＆」、「<」を使用できません。 これらの文字は特別なシーケンスを使用して設定できます

<-＆lt;

＆-＆amp;

単一および二重のアポストロフィを使用する場合、追加の制限が課せられます。ペアになっていないアポストロフィを使用することはできません。 この問題を解決する方法はいくつかありますが、最も簡単な方法は、アポストロフィ記号の前にバックスラッシュを追加することです。

'-\'または＆apos;

「-\」または＆quot;

テキストにhtmlタグが含まれていて、その中に不可解なスペース＆nbsp;がある場合は、＆＃160;に置き換える必要があります。



行の読み込み



リソースからインターフェイス要素の1つにテキストを設定する必要がある場合、事前にダウンロードする必要はありません。 上記の例の2行目と3行目はまったく同じように機能します。 

 TextView sectionHeader = (TextView)findViewById(R.id.sectionHeader); sectionHeader.setText(getString(R.string.sectionPhone)); sectionHeader.setText(R.string.sectionPhone);


getStringメソッドは、使用する前にリソースから文字列にデータを追加する必要がある場合に便利です。 この場合、フォーマットされた文字列はリソースに配置され、追加のパラメーターはgetStringで直接指定され、 String.formatメソッドを追加で呼び出すことはありません 。 ダッシュで区切られた例でも同じ結果が得られます。 

 int sectionId = 10; String header = getString(R.id.headerSection); sectionHeader.setText(String.format(header, sectionId)); //----------------- sectionHeader.setText(getString(R.id.headerSection, sectionId));


次の構成が文字列形式に使用されます：％X $F。

X $は、置換されるパラメーターの番号です。 本文では、通常、$ 1、$ 2の順序で表示されますが、ローカライズされたリソースでは場所を変更できます。 1行に1つのパラメーターを数回使用することもできます。

Fは、 's'、 'd'などの通常の形式の識別子です。 日付の書式設定を含む完全なリストは、 Formatterクラスのドキュメントに記載されています。

行が1つのパラメーターのみを使用する場合、X $は省略できます。 パラメータが不便で、ローカリゼーションの問題が予想されない場合は、標準の文字列フォーマットスキームに戻すことができます。 そのためには、要素の説明に値falseの フォーマットされた属性を追加する必要があります。 次の2行は、テキストを同じ方法でフォーマットします。 

 <string name="score_correct">%1$d - %2$d</string> <string name="score_simple" formatted="false">%d - %d</string>


他のリソースと同様に、名前で行をロードできます。 これを行うには、最初にgetIdentifierメソッドを使用して、文字列の名前でそのIDを見つける必要があり、この番号を通常の方法で既に使用する必要があります。 次の例では、「score_correct」という行をロードします。 

 int strId = getResources().getIdentifier("score_correct", "string", getPackageName()); String strValue = getString(strId);


Androidでは、リソースに文字列配列を保存できます。 これを行うには、 string-arrayタグを使用します。このタグには、特定の文字列を含むitem要素が含まれます 。 これは、配列の指定を示すAndroidのドキュメントの簡単な例です。

strings.xml 
 <string-array name="planets_array"> <item>Earth</item> <item>Mars</item> </string-array>


明らかなアプリケーション-便利なデータロードに加えて、 string-arrayは 、値のドロップダウンリストでSpinerとListPreferenceを使用してUI要素を初期化するためによく使用されます。 この場合、通常、配列のいずれかの行をデフォルト値として使用する必要があります。 リソース内の配列の特定の要素を参照することはできませんが、リソースのエイリアスを使用することで状況から抜け出すことができます。

配列要素は通常の文字列のように初期化されますが、 アイテム要素にはそれらへの参照のみが含まれます。 この初期化は実際には非常に便利であり、特定の行にアクセスする予定がない場合でも使用します-これにより、すべての配列を検索するときに、それらの行の一部が翻訳されない場合でも同じサイズになります。 配列自体の記述は、別のリソースファイルに簡単に出力されます。

エイリアスを使用して書き換えられた例は次のようになります。



strings.xml 
 <string name="earth">Earth</string> <string name="mars">Mars</string> <string-array name="planets_array"> <item>@string/earth</item> <item>@string/mars</item> </string-array>


有用な注意：Android 2.3より前のバージョンでは、ストリング配列のロードの実装でエラーが発生し、最大512個のエレメントしかロードできませんでした。



システム文字列



Androidは、システムリソースに保存されている複数の行にアクセスできます。 それらのいくつかは非常に特殊です：ビデオを再生するためのエラーメッセージのヘッダー。 ただし、「Ok」や「Cancel」などの文字列は、ほとんどすべてのプロジェクトで使用すると便利です。 システム文字列の使用は、独自のリソースを使用するのとほぼ同じくらい簡単です。 文字列リソース識別子の前にコロンでアンドロイド名を指定する必要があります。 次の例では、システムリソースの文字列を使用して「キャンセル」というラベルの付いたボタンについて説明します。



main.xml 
 <Button android:text="@android:string/cancel" android:layout_width="wrap_content" android:layout_height="wrap_content" />




文字列のローカライズ





Androidでリソースをローカライズする基本的なメカニズムにより、文字列をローカライズすることでほぼすべてのタスクを簡単に解決できます。 Androidのドキュメントの完全な説明は、 ここにあります 。 リソースには、 values-xxという名前の新しいフォルダーを作成し、そこにベース値フォルダーからの文字列リソースを持つファイルを配置する必要があります。 xxは2文字の言語識別子です（サポートされる値のリストはテキストの最後に記載されています）。 この後、必要な行を翻訳し、残りを削除する必要があります。

ローカライズを細かくするには、リージョンを設定するメカニズムを使用できます。 たとえば、プログラムにはフランス語に翻訳されたユーザー契約がありますが、フランスのバージョンとカナダのバージョンには違いがあります。 これらの違いを実現するには、 rYY形式のリージョン名をvalues-fr フォルダーに追加します 。ここで、 YYは2文字のリージョン名です。 この例では、 values-fr-rFRおよびvalues-fr-rCAフォルダーを取得します 。 リソースファイルを必要なバージョンのユーザーアグリーメントと共に配置し、 values-frフォルダーに他のすべての行をフランス語で残してください。

地域と言語は、互いに接続せずに指定できます。 そのため、 values-ru-rJPフォルダーには、日本居住者向けのロシア語のテキストが格納されます 。



地域は明らかに少ないため、ほとんどの場合、 mccプロパティ（モバイル国コード）を使用する代わりに、正確な国識別子を指定する方が便利です。 コードのリストはWikipediaで入手できます。この方法の唯一の欠点は、一部の国に複数のコードがあることです（米国では7桁、日本では2桁）。 mccを使用する例を再作成すると、フランスの場合はvalues-mcc208-frフォルダーが、カナダの場合はvalues-mcc302-frフォルダーが取得されます。

ローカリゼーションの難しさは、システムでサポートされていない言語でアプリケーションをローカライズする必要がある瞬間に始まります。 ほとんどの言語と地域はプラットフォームのバージョン2.3でのみ追加されたため、この状況に陥るのは簡単です。 同時に、一部のメーカーはデバイスに他の言語のサポートを追加しましたが、異なるコードを使用していました。 その結果、Android 2.1および2.2のノルウェー語は多くのデバイスでコードなしで認識され、一部のデバイスでは認識されず、Android 2.3以降ではnbコードで認識されます。 同じ状況ですが、小規模ですが、ヘブライ語はheコードとインドネシア語の下にあり、 idコードです。

この場合、同じファイルの2つのセットを異なるファイルに保存しないようにするために、前述のリソースのエイリアスを使用できます。

この場合、実際の改行はvaluesフォルダーに保存され、作業名が名前に使用されます。 values-noおよびvalues-nbフォルダーには、文字列へのリンクが保存され、実際のリソース名はすでにここで使用されています。



値/strings.xml 
 <resources> <!--    --> <string name="hello">Hello World</string> <!--   --> <string name="hello_translate">Hallo Verden</string> </resources>


values-no / strings.xml 
 <resources> <string name="hello">@string/hello_translate</string> </resources>


values-nb / strings.xml 
 <resources> <string name="hello">@string/hello_translate</string> </resources>


言語および地域の任意のコードでリソースを作成できます。 オペレーティングシステムが目的の地域を見つけられない場合、現在の言語の値が使用されます。 言語が見つからない場合、 valuesフォルダーからデフォルト値を取得します。



デバイスにインストールされている言語と地域以外の言語と地域のリソースの文字列をダウンロードできます。 これを行うには、新しいリソースを作成し、それに必要なロケールを設定します。 次の例では、フランス語の文字列をロードします。 

 Resources baseResources = getResources(); Configuration config = new Configuration(baseResources.getConfiguration()); config.locale = Locale.FRANCE; Resources localResources = new Resources(baseResources.getAssets(), baseResources.getDisplayMetrics(), config); String strFranceValue = localResources.getString(R.string.score_correct);


プログラム全体で任意の言語をプログラムで設定することもできますが、このタスクの検討はこのトピックの範囲外です。
