Aries、Segnetics、およびSchneider ElectricメーカーのHVAC用PLC

この記事は、3つのメーカーのHVAC（加熱、換気、空調）用のプログラマブルコントローラーの概要です。



  • ARIES PLC63 / 73
  • Segnetics SMH2gおよびPixel
  • シュナイダーエレクトリックM171 / 172


それらを使用した個人的な経験に基づいています。







ARIES PLC63 / 73





したがって、アクロバットの2人の兄弟：PLK63-DINレールとPLK73の生息地-頑固な換気装置（設置には、配電盤のドアに取り付ける必要があります）。



変更を加えると、8つのディスクリート入力、8つのアナログ入力が得られます。 換気設備の自動化タスクには、非常に多くのセンサーで十分です。



追加のPI73ボードを購入すると、PLC73には2つのCOMポートがあります。 バリエーション5個。 個人的に、私は最も興味深いPI73-5-RS-485 * 2個を検討します。 PLC63には常にRS485が搭載されています。



PLC63の2行に対してディスプレイのまだ73行目の4行。 どちらのモデルも、時計とメモリにバッテリーが一度に搭載されています（ただし、セグネティックスよりも変数のメモリが少なくなります）。 これらのPLCのROMストレージメカニズムは、Segneticsよりも楽しいです。 私が理解しているように、ROMにはデータは電源オフ時にのみ書き込まれます。 残りの時間、データはRAMに書き込まれます。



このモデルと他のモデルの両方には、VU（出力デバイス）と呼ばれる多くの出力のバリエーションがあります。 リレー、電流または電圧のDAC、オプトトランジスターまたはオプトシミスターを使用できます。 メーカーのバリエーションを作成することは難しくありません。なぜなら、 メインボードは1つで、異なるVUモジュールをはんだ付けできます。



さらにリレー出力が必要ですか？ 問題ありません-MP1モジュール（リレー8個）を購入します。 唯一のモジュールはこれです-ARIESはそれ以上のバリエーションを提供しません。



PLC63の場合-リレーと5個のVU（購入時に選択）。 PLC73の場合-4つのトランジスタと4つのVU。



一般に、PLC73は物議を醸すモデルです。 人々が激怒しているもののほとんどは、厳しいハードボタンです。 そうそう...ボタンは、経験の浅い人にネガティブな感情の全範囲を体験させます。 さて、PLC73はアナログ出力が少ないです。



両方のモデルで作業した後、私はお気に入りのPLC63を残しました。 どういうわけかそれはより深刻に見え、ボタンは押しやすいです。 ただし、RS485が1つしかないのは残念です。



2つ目のRS485、カールが欲しい！



本当に必要な場合は、RS232-debug（ファームウェアを注入するポート）があることに注意してください。 確かに、RS232からRS485へのコンバーターを購入する必要があります。

Modbusはどうですか？



奴隷 すぐに使用でき、すぐに読み取りエントリ、終了、ボタンを押す、時計が表示されます。 新しい変数のアドレスは、個別に作成する必要があります。 グループ要求なしですべてが読み取られます。

しかし、本当にしたい場合は、グループ化することもできます。マニュアルに説明があります。 ただし、マニュアルのアドレスは正しくありません。



動揺しないでください-親切な人がすでに正しい住所を見つけています。 OVNAは、フォーラムまたはここで見つけることができます（迷子にならないように、すでにアップロードされています）。

マスターモード 。 ここはもっと複雑です。 「プログラミングスキル」などの恐ろしいことを適用する必要があります。Modbus.libライブラリを取得し（または自分で実行できます）、外部デバイスとの相互作用を記述します。 怖いように聞こえますが、ある程度は柔軟性が増します。 たとえば、同じタイプの変数の束を作成せずに、典型的なデバイスの束（同じ周波数変換器）で作品を書くことができます

  • プログラミング環境。
  • 設定と設定でメニューを簡単に作成（セグネティックス休憩）
  • クロックとメモリは常に「一緒に」います。
  • COMポートで非標準プロトコルを使用する機能。
  • PLCからのより多くの入力と出力（ピクセルと比較して）。
  • [PLC]メニューの[すぐに使える]で、センサーの種類を変更でき、読み取り値を確認できます
  • 220Vで動作します（おそらくプラスですが、ヒューズとサーキットブレーカーの形で電源保護を提供する必要があります-内部電源保護の操作の経験があります）。

  • PLC73ボタン。

  • 1つの「Enter」ボタンを使用して、設定を含むメニューに入ります。 個人的には好きではありません。 追加メニューの作成を複雑にします。
  • 1つのPLCモデルから別のモデルにメニューを転送するのは不便です（同じタイプの設定を複製する）。 新たに作成する必要があります。 うるさい。

Segnetics SMH2gおよびPixel







それで、Pixelと彼の兄であるSMH2gです。 LinuxにアクセスできるSMH2giもあります。 「プログラミングスキル」と同じ恐ろしいことを愛する人に、より多くの機会を与えるようです。



I / Oに関して、SMH2gは拡張モジュールなしでは特に興味深いものではありません（SMH2giではボード4 DI / 4 DOおよび5 DI / 2 DO）。 Pixelにはもっと良いことがあります（6DI / 3DO / 5AI / 2AO）が、モジュールなしではできません。



注意することが重要です-アナログ出力および入力のピクセルガルバニック絶縁NO。 もちろん、Segneticsは拡張性の点で混乱しました。 Pixel向けのイーサネットも可能です。 ただし、クロックが確実にカチカチ音をたてるには、Segneticsからバッテリーを購入する必要があります。 そして、この位置は別です。 だから、それはメモリです。 個別に購入します（たとえば、スケジュールに従って作業する必要がある場合）。



Segneticsの庭にあるもう1つの小さな小石は、PLCを拡張モジュールに接続するためのケーブル価格が悪いことです。 実際、IDCコネクタ付きの通常のケーブル。 ストアやラジオ市場ですべてを購入することで、自分で作ることができます。 5倍安くなります。 主なことは、接続と混同しないことです。



外観、ボタンの押しやすさ、ディスプレイに関しては、すべてここで問題ありません。 より多くの哀れみのために、Segneticsはコントローラーのケースにロシア語で言葉を書きさえしませんでした。 まあ、私たちは満足しています。

Modbusはどうですか？



すべてが悪いわけではありません。 作成した各変数は、自動的にModbus変数になります。 これがスレーブモードの実装方法です。



マスターにとっても、すべてが悪いわけではありません。 他のSegneticsコントローラーからModbusカードをロードするか、変数マップを個別に作成（およびコンピューターに保存）できます。 ほとんどのタスクではこれで十分です

Modbusはどうですか？



各変数には独自のアドレスがあります。 すなわち 奴隷はすぐになります。 Modbusでは、この点ですべてが問題ありません 。 すべてではありませんが、すべてではありません。 ここには、さらに大さじ1.5のtarがあります：



  • 1スプーン -マスターモードはまったく機能しません。

    プログラマーが戦わない限り、彼は開始できませんでした。 代わりに、RS485出力で送信の最初のバイトのみを受信しました。 すなわち ポーリングされているデバイスのアドレス。 そのため、あるオブジェクトでシールドを補完する必要がありました-PLK100の牡羊座を挿入し、100番目がMaster'aの役割を果たした（下の写真）。
