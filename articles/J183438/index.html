月額120ドルから100TBのトラフィックを持つサーバーは、でたらめなマーケティングを行っていますが、それでも彼らには生命権があります！

長い間、私たちはサービスに保証されたチャンネルと本当に無制限のトラフィックを必要とする顧客に焦点を合わせていました。 それにもかかわらず、インターネット接続の保証レベルがそれほど重要ではない加入者もいます。 それにもかかわらず、選択が必要であり、初期段階では「限定」オファーが役立つ場合があると判断しました。



この投稿では、100TBのトラフィックでサーバーをレンタルすることの利点と欠点について説明します。 彼はこれらのサーバーのこのような低価格の起源を説明し、どの場合にそれらを使用するのが賢明であり、使用を控える方が良いかを教えてくれます。 繰り返しますが、誤解を避けるために、保証付きの包含のみを使用することをお勧めします。 120 TBからの100TBのトラフィックを持つサーバーなど、他のすべてのサービスオプションは「 現状のまま 」提供され、その提供はいつでも保証されません。







保証されたインターネットチャネルにかなりの費用がかかることは周知の事実です。 価格は、チャネルの購入元のオペレーターの数とレベル、接続性、およびトラフィックの何％が交換ポイントに行くかによって決まります。交換ポイントには独自の「安価な」インクルージョンが含まれます。 さまざまな事業者から数百ギガビットを購入する場合でも、1ギガビットあたり500ユーロ未満に価格を下げることはほとんどできません。 保証をより安く販売するデータセンターは、すべての人がそのインクルージョンを100％使用するわけではなく、必要に応じて一般チャネルを拡張するという事実に依存しています。 また、トラフィックに対して一定の料金が加入者だけでなく、データセンターとのトラフィックの交換に関心があるピアツーピアプロバイダーからも徴収される場合がありますが、このデータセンターはネットワークの優れたトラフィックジェネレーターである必要があります。



これに基づいて、ギガビットチャンネルの価格が500ユーロよりもはるかに低い場合、そしてサーバーを使用していても、ご存知のように、あなたはだまされています。



2年少し前、EvoSwitchデータセンターは外部チャネルを大幅に拡張しました。 まあ、多くのアイドルレーンを持っているまともなオペレーターのように、彼はダウンタイムからの損失を補償し、新しい顧客を引き付けるために前例のない寛大さのオークションを開催することを決めました。そのような小さなお金のために-誰もそれを持っていなかった。 計算は主に「西側の」ユーザーのために行われました。多くの場合、必要なトラフィック量がわからず、ほとんどの場合100TBを消費せず、多数、つまり「公正な消費」に非常に魅力的です。



「私たち」の人々がストリーミングプロジェクト（オンラインビデオ、音楽/ラジオオンラインなど）、CDN（プロキシ/ VPNを含む）でデータセンターに来たとき、100 TBのサーバーの公正な消費が明らかになりました。スラブの人口は受け入れられません。できるだけ多くの人に、より安く与えてください。 そして、誰もがサービスを最大限に利用したいと考えています。 そして、神はこれらのプロジェクトの所有者が数百倍の収入にもかかわらず、100ドルを支払うことを禁じました...その結果、サービスの発表からすでに3か月後に、チャネルの過剰な準備金は急速に減少し、サービスがそれ自体に支払っていなかったことが明らかになりました。



データセンターは、そのような「正直な」ユーザーのために、多くの制限を導入することを決定しました。 これにより、100TBのトラフィック制限のあるサーバーをレンタルする場合、次の条件に同意することになります 。



-データセンターの事前の同意を得ることなく、ストリーミングまたはCDNにサーバーを使用しないでください（非常にまれなケースですが、開示されていない場合があります）。

-100 TBサーバー-限定的かつ一時的な提供。サービスの一時停止の14日前に、通知なしで一方的にサービスの提供を拒否される場合があります。

-提供されたトラフィック量を、月間で消費の急激なピークなしで、正直に使用します（言い換えれば、1日あたり3.3 TBを超えて使用することはできません。そのようなトラフィックの消費に対して、平均チャネル負荷が330 Mbit / s以下であることを計算することは難しくありません） 。



条件に違反した場合、そのような措置が適用される場合があります。



-サーバーポートの速度制限。

-レンタルされた100TBサーバーの価格の消費帯域の市場価値への変更。

-クライアントへの返金なしで問題のサーバーを一時停止または終了。



データセンターは、このような違反を自動的に検出できるソフトウェアシステムもいくつか作成しました。 したがって、100TBシリーズサーバーを使用する場合は、ルールに従うことを強くお勧めします。







このオファーは誰のために、どのような目的のために役立ちますか？



-サーバーに十分な100 Mbit / sを持たないポータルを訪問しましたが、保証されたギガビットを支払うのに費用がかかり、節約する必要がある場合は、150〜300 Mbit / sチャネルで十分です。

-バックアップサーバーの場合、迅速かつ効率的にバックアップする必要があるサーバーが多数ある場合（ポリシーに違反しないように、このような操作を営業時間内に行わないか、自分でコピーフローを制限することをお勧めします）。

-ファイルアーカイブ（ファイル共有プロジェクトがあり、負荷をストレージ間で均等に分散できる場合-これはあなたのための申し出ですが、ストリーミングは禁止されており、急激な消費の増加はないはずです）



大幅な超過が発生した場合、データセンターのキャビネット内のチャネルは拡大できないため、正直なトラフィック消費ポリシーに従うことが非常に重要です。 他の加入者のサービスの低下を引き起こす可能性があり、これを防ぐために彼らは単にあなたをブロックします。 違反が繰り返される場合、ロック解除は不可能になります。



このサービスは、基本的に制御された共有インクルードであり、ストリーミングが禁止されている理由です。 このようなオンラインビデオポータルの所有者が消費量を制御するのは非常に困難な場合があります。さらに、「100TB-電源オン」では流量が保証されない場合があります。
