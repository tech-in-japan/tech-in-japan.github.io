WIZnetの新しいチップW5500を接続します。 小さなことのためのIP

EtherentとTCP / IPは、さまざまなデバイスを接続する最も一般的な方法であるようです。 WiFiは最近、有線の「銅線」Etherentに取って代わりましたが、ポートを接続して見つけ、「LANに接続する」最も簡単な方法です。 このようなイーサネットのアクセシビリティは、コントローラー、センサー、消費メーターなどの最小のハードウェアを通過しませんでした。 今では「モノのインターネット* 」（モノのインターネット、IoT）というファッショナブルなマーケティング用語と呼ばれています。











ここで、WIZnetからの韓国のマイクロサーキットは、私たちを大いに助けてくれます。 WIZnetは、IOcP（インターネットオフロードコプロセッサ）クラスのチップのかなり活発なメーカーです。 これは何ですか 人間の観点では、これはチップです：一方でEtherentを持ち、内部にはTCP / IPを処理するための独自の小さなプロセッサが含まれ、他方では-小さなハードウェアと通信するためのシンプルなインターフェイスです。



最初の部分では、「小さな腺」と異なるチップへのリンクの束を接続することの小さな理論的な概要があります。



しかし、すぐに問題が発生します：すべてがメインで処理できるのに、 なぜ別のプロセッサをシステムに導入するのですか？ 答えてみましょう。



概要：ささいなことのためのイーサネット。



価値の規模（最大でドル単位またはそれ以下）を「下げ」、ネットワークに「ささいなこと」を含めるさまざまな方法を考えてみましょう。



マイクロコントローラ用のネットワークに到達する最も簡単で簡単な方法は、 COMport-to-Telnet 、 Serial2Ethernetコンバータ（リバースTelnetとも呼ばれることもあります）を使用することです。 この場合、コンバーターはTelnetセッションを「キャッチ」し、それをマイクロコントローラーの通常のCOMポートに「プッシュ」します。 次に、オペレーターはネットワーク経由でのみ通常のダイアログに入ります。 さて、私は何と言うことができますか？ まず、別のマイクロコントローラーもあり、デバイスはまったく安価ではありません（ 自家製モジュールとTibboモジュールに関する記事）。 「1つのパラメーターの変更/読み取り」という単純なタスクでは、これで十分です。 単一のセッション、固定プロトコルなど、多くの欠点もあります。たとえば、SSLが存在しない場合、SSLをねじ込むことは困難です。たとえば、IPアドレスやポートを変更するなど、コンバーター自体の構成の複雑さなどです。



別のオプションは、ボード上に実際のEtherentを備えたマイクロコントローラーですが、通常はMACレベルのみで、外部PHYチップが必要です（KS8721BL、DM9161A、RTL8201）。 このようなチップ、MIIまたはRMIIとの接続には、少なくとも外部ピンが必要です。 PHYをLANに接続するには、トランスまたはMagJackをはんだ付けする必要があります。 したがって、「コンポーネントの数」によると、ここではゲインはありません。 驚いたことに、外部Etherent PHYは非常に高価なチップで、1ケースあたり数ドルです。 しかし、WIZnetのチップ全体とほぼ同じコストで、シリコンの量産には1ペニーかかります。 はい、WIZnet用のトランスも必要ですが、組み込みのMACコントローラーからの大幅なゲインは価格では機能しません。 2番目：MAC / PHYのソフトウェアサポートは非​​常に複雑で膨大です。 たとえば、STM32Fxx7の場合、イーサネットドライバーのソースコードは、Cで約100Kのテキストを占有します（ Etherを使用したPICに関する記事 ）。 Etherentドライバーに加えて、TCP / IPスタック自体 （ LwIPまたはuIP ）も必要です。 サイズに加えて、実行速度を忘れないでください（ただし、STM32はすでに最初のPentiumよりも高速であり、パフォーマンスの制限については忘れることができます）。



しかし、マイクロコントローラ上の「フル」ソフトウェアIPスタックの利点はありますか？ もちろんあります。 後で検討しますが、今のところは継続します。



3番目のオプションはMicrochip ENC28J60コントローラーSPIイーサネットチップ （およびあまり知られていないSilabs CP2200 ）です。これは、「 埋め込み 」に深く愛されています。 これらはTCP / IPコプロセッサではなく、実際のイーサネットコントローラであることに注意してください。 チップでできることは、Etherentフレームの受信と受信だけです。 つまり、TCP / IPスタックはまだ自分で行う必要があります。 はい、素晴らしい小さなものから古典的なLwIPやuIPまで、いくつかの既製のスタックがあります。



それでも、そのようなソリューションは組み込みのMACコントローラーと直接競合する可能性が高く、チップ価格は数ドルと同じで、ebay / aliexpressの既製のモジュール（チップ、トランス、RJ45ソケットを備えたボード）で約5..7ドルです。 WIZnet製品に戻りましょう。



WIZnet：途中です。



最初のW3100デバイスは、かなり前の2001年にリリースされました。 それは本格的なTCP / IP IOcP（コプロセッサー）であり、COMポートとMACコントローラーの間の特定の中間方法を提供し、チップ内のTCP / UDP "ソケット"のアイデアを実現しました。 物理的には、W3100は非常に便利なデュアルチップMAS / PHYキットではなく、「arduinoドライバー」（AVR）と「ピコガイド」の間での本当の人気がW5100デバイスにもたらされました。 「公式の」 Arduinoイーサネットシールドは 、W5100専用に組み立てられていると言えば十分です。



それ以来、多くの時間が経過し、W5100の欠点が顕著になりました：チップはかなり熱くなり、ネットワーク速度はわずか10Mbit / sで、チップにはいくつかのエラーがあります（エラータに記載されています）。 当然のことながら、WIZnetは間もなくW5200というモデルを導入しました 。 このチップは、100Mbitで動作し、スリープ状態になり、電力を節約し、並列接続を失い、純粋なSPIが残っていることを学び、脚の数を大幅に削減しました。 （パラレルバスをサポートするために、彼には「兄弟」W5300がいます）。



W5200の実装と開発は大成功を収めました！ そして今日に至るまで、モジュールが作成され（「ネイティブ」のWizNET WIZ820ioから始まり ）、さまざまなArduinoシールドで終わります。 たとえば、Seeeduinoプロジェクトの「公式」 Ethernet Shield V2.0や、DfrobotのArduino用Ethernet Shieldなどです 。 そして、他のモジュールの大きなリストがあります（リストは非常に大きいので、次第に縮小します）。



最後に、2013年10月に、WIZnet は新しいW5500を発表しました。このW5500については、この記事で実際に伝える予定でした（このような長期にわたる導入についてはごめんなさい）。 チップはもう少し便利になりました

LUT-do-it-yourselfers、48LQFNケース、内部の高速「パケット」SPIおよびCortex M0により、製造プロセスと消費電力を削減できました。 一般的に、チップは「W5200 on steroids」のように見えます。 W5200とW5500の比較 。



次に、WizNETはWIZ550ioモジュールをリリースし（そこにも回路があります）、「arduinomania」から離れないために、WIZnetはすぐにArduinoシールド自体をリリースしました（ArduinoからWIZ550ioへの単なるパッシブアダプターです）。



しかし、Arduino は考慮せずに 、彼らは私たちなしでそれを行います（たとえば、ここHabréで： Arduinoに基づくイーサネット温度計など）、W5500をSTM32などの一般的なマイクロコントローラーに固定しようとします。 WizNET自体もこの素晴らしいファミリを認識しており、以前のW5200（USB2Serial用のSTM32F103C8およびFT232RQ）でW5200E01-M3モジュールをリリースしました 。 すぐにSTM32 + W5500のモジュール： WIZ550webが登場しました 。



ちなみに、2014年3月、W5500を使用した開発のために、CircuitcellarでConnect The Magicコンテストが発表され、WIZnetはチップとモジュールの価格を下げました（ 購入 ）。 コンテストでは、最高15,000ドルを獲得できます。 サイトの詳細、そこには素晴らしい記事があります。
