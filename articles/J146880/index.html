Windowsユーザーの目から見たMac OS X

なぜなら 私の仕事は最近iOSの開発と結びついており、時々Mac OSとのパスを横断する必要があります。 この記事では、Windowsの背後で意識的な人生を過ごした男性の目を通して、このシステムの私の印象についてお話します。



Mac OSが必ずしもAppleであるとは限らないとすぐに言わなければなりません。 なぜなら このOSを任意のコンピューターにインストールできるHackintoshなどのプロジェクトがあります。 もちろん、この場合、ノンフィグタンバリンが必要になります（特にAMDを使用している場合）が、事実は残ります-それは可能です。 確かに、それは常に滑らかではありません。 たとえば、AMDにhackintoshをインストールしたとき、Googleで最初にアセンブリを検索し、次にドライバーとその他の修正プログラムを検索するのに2日間を費やしました。 ところで、すべてのフラッシュコンテンツ（ビデオを含む）がブラウザで定期的にフリーズされたときの1つの重大なバグ-私はそれを克服しませんでした。 しかし、私はまこしをPCにインストールすることについては広めません。これはまったく別のトピックのトピックです。



私はMac MiniでネイティブMac OSを使った経験があります（ちなみに、高コストにもかかわらず、このマシンには間違いなく魅力があります！特に、このアルミニウムの箱が3年前の巨大な騒々しいコンピューターのサイズよりも少し大きいことに気付いたとき） ）



それでは始めましょう...



滑らかさと美しさ。



私の目を引いた最初のことは滑らかさでした。 すべてが美しくアニメーション化され、すべてが離れ、折りたたまれ、展開されます...すべてのボタンおよびその他のインターフェイス要素がなめらかに輝きます。 いいえ、Windows 7では（G8については何も言いません）すべてがeverythingいとは言いません。 でも、まこしのほうがもっと美しいように感じます。







フォント



ポピーのフォントをテーマにした多くのコピーが壊れています。 Windowsの後の最初の感覚は、「脂っこくてぼやけた」です。 実際には、Windowsは常にフォント行の太さを整数のピクセルに調整しようとし、そのまま描画します。 ところで、これが多くのプリンターがポピーで作業することを好む理由です。 ポピーでは、フォントは紙の上で見えるように見えます。



モニター上...最初は珍しいです。







しかし、仕事をすればするほど、彼らに恋をするようになります。 これらのフォントのインターフェースは特に美しく見えます。







そして数時間後、彼らは「脂っこくてぼやけた」ようには見えません。 さらに-Windowsに戻る-私は個人的にこれらのフォントを見逃し始めています。



パフォーマンス。



同時に-すべてが非常に高速でスムーズです。 スピードはあなたが感じる第二のことです。 前述したように、1台のコンピューターでMacOS（hackintosh）をWindows 7と並行して使用した経験があります。 だから-同じマシン上でマクロが速いという感じ。 読み込み速度は2倍になります（ただし、ここでは正しい比較ではありません。私のポピーはまだがらくたではなく、「7」は何世紀にもわたって立っているからです）。 重いソフトウェア（PhotoshopおよびFlashで測定）は、30％高速で開始されます。



しかし、主なことはそれでさえありません。 Windowsで、システムの応答が失われたという砂時計が表示されることが、どのくらいの頻度で発生しますか？ さらに、私は時々これを単に「突然」「キャッチ」しました。 フォルダーに入ります-それだけです。 座って、Windowsがそこに何かを考えている間、30分待ちます。 カーソルがアニメーション化され移動しているという事実にもかかわらず、システムはフリーズしています。 何もできません。



Mac OSではこれはそうではありません（まあ、または私はそれを理解しませんでした）。 一部のプログラムがフリーズしても、他のプログラムとインターフェイスの応答性には影響しません。 ドック内のアイコンは、ホバリングすると穏やかに拡大し続け、すべてが正常に機能します。



Doc



ドックは、画面の下部にアイコンがあるパネルです。 Windowsタスクバーのタイプ。 ただし、ここではすべてが少し異なります。



現時点では、私はマコの下で十分な時間を過ごしましたが、とにかく-ドックのこのアイデアは私にはあまり明白ではないようです。 美しい-間違いない。 しかし、使いやすさの点では...



知らない人のために、これがどのように機能するかをいくつかの言葉で説明します。







ドックの左側（この画面では縮小しましたが、約70％かかります）-Windowsが「修正されたアプリケーション」と呼ぶもの。 つまり これらのアイコンは常にそこにあります。 同じ部分は、通常のタスクバーのように機能します。 新しく起動したアプリケーションもすべてここに届きます。 原則として-今のところはWindowsのように見えます。 ここでのみ、Windowsでの怠慢の指標が明らかになります。 ここで、アプリケーションが下部にある小さなドット電球だけで起動されているかどうかを確認できます（最初の2つのアイコンの下に青いライトがありますか？）。



ドックの右側は一般に奇妙です。 バスケット（これは理解できます）、パパ「アプリケーション」、「マイドキュメント」などがあります。 -これも理解できます。 しかし、何らかの理由で、最小化されたすべてのウィンドウがここに表示されます。 しかし、同時に左側に残ります。 ここで、Firefoxを見てください。 左側にもあります（そして、実行中なので、その下にライトが点灯しています）が、 私はそれを回しました（最小化）-それから右側にも飛んで行きました。 偉大な本質は何ですか-私はまだ理解していません（更新：この動作はドック設定で無効にすることができます）。



プログラムを閉じることも奇妙に編成されています。 通常、クロスクローズはアプリケーションを閉じません。 そして、ドックの左側にそれを回します。 つまり 閉じます（開いているすべてのドキュメントをアンロードするなど）が、同時にメモリにハングしたままで、いつでも回復する準備ができています。 おそらく、現在のメモリ量で-これは正当化されます。 シンプルで珍しい。 アプリケーションを完全に終了するには-アプリケーションを右クリックして、適切なメニュー項目を選択する必要があります（まあ、またはCtrl + Qを押します）。



ちなみに、ウィンドウを閉じる、最大化する、最小化するボタンが左側にあり、慣れている右側ではないという事実は、不快感を引き起こしません。 すぐに慣れてください。



私のメニューはどこですか？



WineユーザーにとってのMac OSのもう1つの不合理な点は、プログラムメニューです。 Windowsでは、メニューは常に見出しの下にあるという事実に慣れています。 ポピーでは、アプリケーションメニューは常に画面の一番上にあります。 ウィンドウサイズなどに関係なく



以下は良い例です。 Firefoxのウィンドウは全画面表示ではなく、メニューは一番上にあります。







確かに、これは便利です。 メニューを探す場所は常にわかっています。



キーボード



私はケシのキーボードを持っていないので、最初は何も理解していませんでした。 次に、設定で修飾子キーを再割り当てする機能を見つけ、ポピーキーとctrlを交換しました。 キーボードの通常の「Ctrl + C」を押すと、期待どおりの結果が得られます。



慣れる必要があるのは、Alt + Tabの代わりにCtrl + Tabだけです。 入力言語をCtrl + Spaceに変更します（いいえ、Alt + Shiftなどに再割り当てすることはできません）。



私が驚いたのは、HomeキーとEndキーのデフォルトのポピーが非常に奇妙だということです。 行の末尾/先頭への予期される遷移の代わりに、文書全体の末尾と先頭に移動します。 誰がこれを思いついた-私は想像できません。 テキストまたはコードを使用する場合、すべての最後までジャンプする必要はほとんどありませんが、ほとんどの場合、行の最後までジャンプします。



神に感謝します、これは扱われています。 しかし、どこでも同時にではありません。 まず、システムキーのバインドを修正するスクリプトがあり、Home / Endはほぼどこでも正常に動作し始めます。 しかし、どこでもありません。 そのため、これが修正されていない場合-個別のソリューションを探す必要があります。 たとえば、多くのIDEでは、これを設定で設定する必要があります。 そして、Firefoxでは、一般に、別のプラグイン「 Keyfixer 」をダウンロードする必要がありました。



組み込みの便利さ。



Mac OS Xにはビルトイン機能があり、Windowsで生成されると、曲率の異なるユーティリティに接続されるか、現在のみ導入されます。



まず、マルチデスクトップです。 デスクトップ間の便利な切り替え、どのスレーブに割り当てるか。 どのアプリケーションやその他の小さなものを起動するデスクトップ...



第二に、これらは画面のアクティブなコーナーです（もちろんカスタマイズ可能です）。 つまり マウスを左上隅に移動し、ミッションコントロール（開いているすべてのウィンドウの視覚的表現）をスムーズに左に移動しました。







まあ、そして小さなユーティリティの束、その半分は私自身はまだ知りません。 たとえば、最近、画面上でピクセル精度でオブジェクトを配置する必要がありました。 makoshiには、修飾キーを押したまま、ホイールで画面を即座に拡大する機能が組み込まれています。 または、スクリーンショットを撮り、キーの組み合わせを使用してそれをフォルダに入れる組み込みユーティリティ。 はい、システムのホットキーを設定できます。



プログラムを追加または削除します。



ここで-私は非常に嬉しかったです。



ほとんどの場合、ポップアッププログラムは単に実行される別個のパッケージフォルダです。 ほとんどの場合、プログラムのインストールとはプログラムをプログラムフォルダーにコピーすることです（例外は、AdobeやXcodeなどの大規模で複雑な複合体のみです）。 同時に、インストールされたプログラムは、レジストリ（実際には存在しません）またはシステムフォルダー（システムに実際に実装されているユーティリティを除く）にクロールされません。 私はそれを私のパパにコピーしました-そしてそこで彼女は掘ります。 あなたはそれを削除する必要があります-ちょうど0.1秒でフォルダを消去し、それだけです。 すべては人にとって高速で理解しやすいものです。 質問やチェックマークがたくさんあるマルチステップのインストーラーはありません。



管理者ユーザーとセキュリティ。



ユーザーアカウント制御をオンにしてWindowsに座ってみましたか？ （UAC）



私の意見では、これは地獄だ。 2番目のプログラムごとに管理者パスワードなどが要求されます。 これが何らかのスレッド、500キロバイトのお粗末なテキストエディターであっても、システムの保護された領域に入る必要がある可能性があります。



ポピーでは、これですべてがずっと良くなりました。 真のシステムユーティリティのみが管理者パスワードを要求します。 通常の操作では、UACをオフにする必要はまったくありません（ところで、これが可能かどうかはわかりません！）。



確かに、ほぼ完全にウイルスが存在しないことを聞いた。 （ポピーでウイルスをキャッチできるのは、自分でインストールした場合のみです）



また、暗号化された情報やパスワードなどを保存するためのユーティリティである組み込みの「キーチェーン」も喜んでくれました。 周囲のすべてと密接に統合されます。 これで、ブラウザがパスワードを保存する場所を心配する必要がなくなりました。 彼はそれらを暗号化された場所に保管します。
