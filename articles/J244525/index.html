論理式



プログラムが実行する実際の作業は式にあります。 意味や副作用を引き起こすプログラムの部分はすべて式と呼ばれます。 定数の値（42など）や文字列値（「hello」など）も式であるため、これらは対応する定数42および「hello」を作成するため、広範な概念で定義されます。



重要：値の作成と変数の宣言を混同しないでください。 値を変数にバインドする必要はありません。



ところで 、 writelnなどの関数呼び出しは、副作用があるため式です。 writelnの場合、この効果は、文字を入力することにより標準出力ストリームに重ねられます。 前に書いたプログラムの別の例は、左側の変数を変更する割り当て操作です。



値の作成により、式は他の式に参加できます。 これにより、より単純な式からより複雑な式を作成できます。 たとえば、現在の気温の値を返すcurrentTemperatureという関数があるとします。この関数が生成するこの値は、 writeln式で直接使用できます。 



writeln("It's ", currentTemperature()," degrees at the moment.");


この行には次が含まれます。
  1. 「それ」
  2. currentTemperature（）
  3. 「現在の学位。
  4. writeln（）は 、前の3つを使用する式です。


この章では、条件文で使用される特定のタイプの式について説明します。



先に進む前に、代入演算子をもう一度繰り返して、左右にあるこれら2つの式を強調したいと思います。 代入演算子（=）は、指定された右側の式の値を左側の式（変数など）に割り当てます。 
 temperature = 23 // temperature's value becomes 23




論理式



論理式は、ブール代数で使用される式です。 論理式は、「答えがはいの場合、ファイルを保存します」などの決定を行うときにコンピュータープログラムが行うことです。



論理式には、 falseを意味するfalseとtrueを意味するtrueの2つの値のいずれかを指定できます。



次の例ではwriteln式を使用します。 行の最後にtrueが印刷された場合、この行に印刷されているものがtrueであることを意味します。 同様に、 falseは、印刷されるものがfalseであることを意味します。 たとえば、この出力が次のような場合：
コーヒーがあります：true
これは「コーヒーがあります」という意味です。 同様に：
コーヒーがあります：false
「コーヒーがない」という意味です。 実際には、左側の「ある」はコーヒーがあることを意味しないことに注意してください。 「... is ...：false」という構文を使用して、「これは正しくない」または「これは間違っている」という意味です。 論理式は、述語、ループ、関数の引数などで広く使用されています。それらがどのように機能するかを理解することは重要です。 幸いなことに、論理式は非常に説明しやすく使いやすいです。



以下は、論理式で使用される論理演算子です。
  • 等号演算子（==）は、「これはこれに等しいですか？」という質問に答えます。 左側の2つの式を右側に比較し、等しい場合はtrueを返し、等しくない場合はfalseを返します 。 実際、等値演算子（==）によって返される値は論理式です

    例として、次の2つの変数があるとします。 
     int daysInWeek = 7; int monthsInYear = 12;
    さらに、これらはこれらの値を使用する2つの論理式です。 
     daysInWeek == 7 // true monthsInYear == 11 // false
  • 不等式演算子（！=）「これはこれと等しくない」という質問に答えます。 両側のこれら2つの式を比較し、演算子（==）によって返された結果と反対の結果を返します。 
     daysInWeek != 7 // false monthsInYear != 11 // true
  • 選言演算子（||）は「または」を表し、論理式のいずれかが真の場合に真を返します。



    左側の式の値がtrueの場合、演算子はtrueを返し、右側の式を見ることさえしません。 左側の式がfalseを返す場合、演算子は右側の式の値を返します。 この演算子は、ロシア語の結合「or」に似ています。



    左側の式 オペレーター 右側の表現 結果
    偽 || 偽 偽
    偽 || 本当 本当
    本当 || false（問題ではない） 本当
    本当 || true（問題ではない） 本当 
     import std.stdio; void main() { // false  "", true  "" bool existsCoffee = false; bool existsTea = true; writeln("There is warm drink: ", existsCoffee || existsTea); }
    2つの式の少なくとも1つであるため、上記の真の論理式は真を返します。
  • 接続演算子（&&）は「and」を表し、左と右の両方の式がtrueを返す場合にtrueを返します 。



    左側の式の値がfalseの場合、演算子はfalseを返し、右側の式を見ることさえしません。 左側の式がtrueを返す場合 、演算子は右側の式の値を返します。 この演算子は、ロシア語の「and」と似ています。
    左側の式 オペレーター 右側の表現 結果
    偽 && false（問題ではない） 偽
    偽 && true（問題ではない） 偽
    本当 && 偽 偽
    本当 && 本当 本当
  • 厳密な選言演算子（^）は、「両方ではなく、どちらか？」という質問に答えます。 1つの式のみがtrueを返すが、両方は返さない場合、この演算子はtrueを返します 。
    左側の式 オペレーター 右側の表現 結果
    偽 ^ 偽 偽
    偽 ^ 本当 本当
    本当 ^ 偽 本当
    本当 ^ 本当 偽
    たとえば、「2人の友人のうち1人だけが来た場合、チェスをします」というロジックです。 これは次のようにプログラムできます。 
     writeln("I will play chess: ", jimShowedUp ^ bobShowedUp);
  • 演算子は厳密に「より小さい」（<）は「これよりも小さいですか？」という質問に回答します（または「ソートするときに高くなりますか？」） 
     writeln("We beat: ", theirScore < ourScore);
  • 演算子は厳密に「これ以上ですか？」（または「ソート時に低くなりますか？」）という質問に答えます。 
     writeln("They beat: ", theirScore > ourScore);
  • 演算子は、「これ以下ですか？」（または「ソートするとき、それはより高いですか、それとも同じ位置になりますか？」）という質問に大まかに少ない（<=）答えます。 
     writeln("We were not beaten: ", theirScore <= ourScore);
  • ライターではない演算子less（> =）は、「これ以上ですか？」（または「ソート時または同じ位置にあるときに低くなりますか？」という質問に答えます。） 
     writeln("We did not beat: ", theirScore >= ourScore);
  • 反転演算子（！）は、「これは何かの反対です」という意味です。 前の演算子とは異なり、1つの式のみで機能し、式がfalseを返す場合はtrueを返し、式がtrueを返す場合はfalseを返します 。 
     writeln("I will walk: ", !existsBicycle);




式のグループ化



これらの式が評価される順序は、括弧を使用して示すことができます。 グループ化することもできます。 括弧内の式がより複雑な式に分類される場合、これらの式の値は、それらが含まれる式で使用される前に計算されます。 たとえば、「コーヒーやお茶、そしてクッキーやパンがあれば、私は幸せです」という表現は、おおよそ次のようにプログラムできます。 
 writeln("I am happy: ", (existsCoffee || existsTea) && (existsCookie || existsScone));
これらのネストされた式が括弧で囲まれていない場合、Dのルール（C言語から継承された）に従って演算子の優先順位を考慮して実行されます。 論理積演算子&&は選言演算子||よりも優先度が高いため 、括弧なしの式の記述は期待どおりに計算されません。 
 writeln("I am happy: ", existsCoffee || existsTea && existsCookie || existsScone);
結合演算子&&が最初に実行され、式全体が次の式と意味的に同等になります。 
 writeln("I am happy: ", existsCoffee || (existsTea && existsCookie) || existsScone);
これには根本的に異なる意味があります。「クッキーやパンが入ったコーヒーやお茶があれば、私は幸せです。」





ブール入力の読み取り



上記のすべてのブール値は、falseまたはtrueとして出力されます。 しかし、これは逆方向には機能しません。これらの「false」および「true」行は、自動的にfalseおよびtrueとして読み取られません。 このため、この入力は文字列として読み取られ、次にbool型の値に変換される必要があります。



以下の演習の1つで「false」と「true」を入力する必要があるため、Dの機能を使用する必要がありますが、これについてはまだ説明していません。 以下では、指定された文字列入力をブール型データに変換するメソッドについて説明しました。 このメソッドは、 std.convモジュールで宣言されているtoを実行することでこの問題を解決します。 （「false」または「true」以外を入力すると、 ConvExceptionエラーが表示される場合があります。）



次のプログラムのmain（）にあるコードのすべての部分がこの段階で明確であることを願っています。 read_bool（）は、言語の新しい可能性があるメソッドです。 彼が何をしているのかを説明するコメントを挿入しましたが、この方法は無視できます。 それでも、コンパイルと正しい操作のために、プログラムコードに含まれている必要があります。



演習



  • 上記のように、これらの演算子<と>を使用して、値が他の値よりも大きい場合と小さい場合を決定しますが、「間にある」という質問に答える演算子はありませんでした。



    プログラマーが次のコードを書いて、 値が 10から20の間であるかどうかを判別すると仮定しましょう。プログラムは説明したようにコンパイルされないことに注意してください。 
     import std.stdio; void main() { int value = 15; writeln("Is between: ", 10 < value < 20); // ←   }
    この式全体を括弧で囲んでみてください： 
     writeln("Is between: ", (10 < value < 20)); // ←
    このプログラムはまだコンパイルされないことに注意してください。
  • この問題の解決策を探しているとき、同じプログラマーは、次の括弧を使用するとコードをコンパイルできることを発見しました。 
     writeln("Is between: ", (10 < value) < 20); // ←   ,
    プログラムが意図したとおりに動作し、「true」と表示されることに注意してください。 残念ながら、このプログラムにはバグがあるため、この結論は混乱を招きます。 15を20より大きい値に置き換えると、このバグの結果を確認できます。 
     int value = 21;
    21が20以上であるという事実にもかかわらず、プログラムはまだ「true」を表示していることに注意してください。 ヒント：論理式のタイプはboolであることに注意してください。 boolが 20未満の場合、これは意味をなさないはずです。
  • 「これは間にあるのか？」という質問に答える論理式代わりに、「これは最小値よりも大きく、最大値よりも小さいか？」という質問に答えるべきです。



    このロジックを念頭に置いてこのプログラムの式を変更すると、期待どおりに「true」が表示されることに注意してください。 この論理式が他の値でも正しく機能することをテストすることもできます。 たとえば、 値が50または1の場合、このプログラムは「false」を出力し、 値が12の場合、このプログラムは「true」を出力します。
  • 次の条件のいずれかに該当する場合、ビーチに行くことができると仮定します。
    • ビーチまでの距離が10 km未満で、誰もが自転車を利用できる場合。
    • 6歳未満で、車があり、そのうちの1人が運転免許証を持っている場合。
    書かれているように、次のプログラムは常に「true」を表示します。 上記の条件のいずれかが満たされたときにプログラムが「true」を出力するようにロジックを設計します。 （プログラムが質問したときに、「Is there a。」という単語で始まる質問に対して「true」または「false」を入力します。） プログラムをテストするときは、必ずread_bool（）メソッドを含めてください 。
