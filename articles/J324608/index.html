インテルがHPCでの地位を強化







10年前、HPCでのIntelの存在は、主に強力なプロセッサと数学ライブラリのセットによって制限されていました。







過去にHPC Intel向けの他の製品がどのように成長してきたかを見るのは興味深いです。 会社がどこに進出するかを予測することもまた興味深いことです。 以下に私のビジョンの概要を説明します。 投稿の下部にあるコメントとアンケートの助けを借りて、みんなにそれを補うように勧めます。







HPCのすべての側面をセグメントに分割し、各セグメントでIntelがどのように表されるかを確認します。 この分離はarbitrary意的であり、私自身がHPCを見ることに慣れている方法のみを反映しています。 おそらくより成功したスキームがあります。







したがって、HPCは（英語名を使用します）：







  1. 計算する
  2. 保管
  3. 相互接続
  4. 開発ツール
  5. 管理ツール
  6. サービス


現時点では、HPCセクターでインテルが直接販売するサービスについては知りません。 これについて何か知っているなら、コメントしてください！







他のすべてのセグメントでは、Intelは非常によく表されています。







  1. 計算：

    • 強力なXeonプロセッサ
    • （どれだけ強力かは明らかではありません）Xeon Phiアクセラレーター
  2. ストレージ：

    • 独自のIntel Lustreパラレルファイルシステムの配布
    • SSDディスクとフラッシュカード（HPCセグメントでのIntelディスクの使用は制限されているようです）。

      ほんの数日前、3D XPointテクノロジーに基づいた最初のカードが発売されました
  3. 相互接続：

    • 独自のオムニパスファクトリー
  4. 開発ツール：

    • Intel Parallel Studio
    • または個別：数学ライブラリ、MPIライブラリ（MPICHライブラリに基づく）、コンパイラ、プロファイラ
  5. 管理ツール：

    • Intel HPC Orchestrator（基本的に、そうでなければパッケージ化されたOpenHPC）


私が何かを逃した場合は、コメントに追加してください。







HPC OrchestratorやOmni-Pathのようなものは過去1.5〜2年にのみ出現し、Intelは2つの追加セグメントをカバーすることができました。







HPCに対するIntelの関心は理解しやすいものです。 今日、ゲノミクスやモノのインターネットなど、多くの計算集約型の分野が活況を呈しています。 ヒトゲノム配列決定研究所では、1時間あたり数百ガガバイトのデータを生成できます。 センサーが取り付けられたエネルギータービンについても同じことが言えます。 そして、これは単なるデータではありません。 これは、ほぼリアルタイムで「ハンマー」する必要があるデータです。 さらに、それらが生成される場所で。 クラウド（Amazonなど）に処理するためにそれらを送信するには、非現実的に厚いチャネルとウォレットが必要です。







IDCの予測によると、2018年に販売される4台目のサーバーはすべてHPCで販売されます。







そのため、Intelの動機は理解できます。 しかし、彼はどこに移動しますか？







ここでは推測することしかできません。 インテルは、ストレージおよびインターコネクトセグメントで最も急進的なステップを踏むようです。







現在、HPCではストレージの2つの主な好みを区別できます。非常に高速なローカル（ただし小さい）とメイン（遅いが大きい）です。 1つ目はバーストバッファと呼ばれます。







近い将来、Intelにとって興味深いのはバーストバッファだと思います。 そして、ここで3D XPointに基づいた製品は非常によく発射できます。 特に、主張されているがまだリリースされていない「メモリ」Optane（正確にはメモリではない）。







ただし、ストレージセグメント内のハードウェア自体は、ユーザー/アプリケーションとの一般的なやり取りほど興味深いものではありません。 現時点では、HPCのバーストバッファの概念が明確に定義されているとは言えません。 これまでのところ、それが何であるかについての一般的な「標準」アイデアはありません。 ストレージ階層のさまざまなレベルはどのように関連していますか？ HPCアプリケーションはどのようにデータウェアハウスと対話しますか？ HPCタスクスケジューラとの統合はどのように実装されますか？

これらの質問や同様の質問に標準的な回答をする必要があるというのは事実ではありません。 しかし、おそらくそうなるでしょう。 そして、Intelにはこの問題に対する独自のビジョンがあるようです。 少なくとも、昨年末に開催されたSupercomputing'16カンファレンスの傍観者には、Intelがここでどのように行動するかについていくつかのリーク/ヒントがありました。 これまでのところ、これらは単なるうわさです。 待って、見て。







データストレージのトピックは非常に興味深いものです。 しかし、インテルが突然懸念するようになったインターコネクトセグメントで最大の変化が待っていると思います。







多くの専門家は、InfiniBandやOmni-Pathなどの高速インターコネクトが最終的にI / Oバスに取って代わることを期待しています。 現在、バスは、バスとデータ交換に関係するデバイスとの間の不必要な媒介として機能します。







データ伝送チェーンからバスを削除するには、プロセッサとメモリサブシステムを工場に直接統合する必要があります。 これがどのように行われるかを見るのは非常に興味深いです。 そしてインテルは自社の工場で遊び始めたので、今ではプロセッサーと統合するための直接的な道があります。 InfiniBandのような代替工場の車輪にスティックを置くソリューションを選択する自由は言うまでもありません。







だからこそ、Omni-PathはInfiniBandの短期的な代替手段としてではなく、さらなるIntelの探求の出発点として興味深いものであるように思えます。







おそらく私にはすべてがあります。 Intel、HPC、および「 すべて 」についてどう思いますか？
