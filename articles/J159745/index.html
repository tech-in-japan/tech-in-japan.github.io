はじめに



彼らは怠lazが進歩のエンジンだと言います。 これは、原則として、本当の声明です-私は、私がほとんど眠っていたときに起きて、明かりを消す必要があることに常に悩まされていました。 したがって、大学での2年目に、AVRkeにシンプルなデバイスをまとめて、リモートコントロールを使用してシャンデリアのオンとオフを切り替えられるようにしました。 まだ同じ部屋に設置されており、現在5年間、24時間年中無休で稼働しています。

リモコンを取り除く試みがあった後-音声コマンド認識システム、同じAVRで、私は同時に大学でコースワークとして合格しました-それは機能しましたが、私が望むほど満足ではなかったので、私はそれを変更しませんでした。

しばらくして、ほとんどの場合、ライトをオンまたはオフにする必要がある場合、手の届く範囲内でPC、タブレット、またはコミュニケーターのいずれかが存在するという理解が形成されました。したがって、最も便利な方法はそれらを制御することです。

特別な無線モジュールであるNRF24L01Pの実装オプションをスケッチしました。たとえば、このモジュールのSMDバージョンのサイズはわずか10x15 mmです。 同時に、管理が便利であり、アドレス指定、マルチ受信、受信の自動確認などを処理する独自のプロトコルを備えています。

したがって、私の推定によると、このモジュールには将来的に何らかの種類のUSBドングルが必要でした。Webサーバーを備えたコンピューターに接続してモバイルデバイスからの制御を可能にし、エグゼクティブデバイスは応答トランシーバーを備えています。

非常に長い間、私の手はこれらのシステムのすべてに届きませんでした-1年の間、ワイヤーが壁から突き出ていました。シャンデリアを含めて、私は毎回手で閉じて、「まあ、職場で物をすくい取り、サーキットボードを再接続する」と自分に言いました。

もちろん、多数の制御オブジェクトがある場合、カスタムモジュールでより愚かなアクチュエータを作成し、ノードの役割をLinuxサーバーに任せるのは理にかなっています。 しかし、仮想オブジェクトに基づいていない、自宅で実際に使用する製品を製造する場合、このアプローチは時間とお金のひどい損失につながります。

ソフトウェア部



伝統に反して、ハードウェアではなくソフトウェア部分から始めましょう。これは非常に単純で、ルーターを購入した直後に、分解せずに確認できるためです。

この時点で、TL-MR3020をすでに購入しており、カーネルの再構築に関する私の記事を読んでもらえれば幸いです。 実際、再構築せずに、公式のOpenWRTプロジェクトwikiのファームウェアでフラッシュするだけで済みます。後で気がついたら、必要なスクリプトを埋め込むことで簡単に独自のファームウェアを作成できます。

では、デバイスはどのように配置されるのでしょうか？



  1. 最小限の詳細を維持することが重要であるため、外部フラッシュドライブまたはハブを使用しません。そのため、使用可能なメモリのサイズは4 MBに制限されていますが、そのうちの空き容量は通常1〜1.5 MBです。
  2. ポイント1を考えると、CGIをサポートする最も軽いWebサーバーを選択します。理想的なオプションは、アドオンなしの20キロバイトのミニhttpdでした。
  3. CGIスクリプトとして、通常のBashovスクリプトを使用します。 原則として、この場所では最小限のpythonアセンブリを配置できます。 確かに、カーネルを再構築せずにこれを行うことはできません-なぜなら ファームウェアがJFFSにインストールされた後にインストールするすべてのものは、元々SquashFSでビルドされた場合よりもはるかに時間がかかります。 実験として、私はpython-miniとmini-httpdの両方を直接イメージに組み込み、そこから余分なものをすべて捨てました-イメージのJFFSビルドにより、システムはオーバーサイズに関するエラーをスローし、メモリに未完成のピースを残しましたが、SquashFSイメージはうまくいき、私をまだ残しました〜250 KBの空き領域で十分です。 もう1つの注意-内部フラッシュドライブからpythonを起動すると、Bashインタープリターを起動するよりもはるかに遅く（外部USBフラッシュドライブよりも明らかに遅く、SPIインターフェイスが影響します）、CGIにPythonスクリプトを使用すると、応答は約1秒になります非常に重要ですが、少し面倒です。 ただし、1つのGPIOをジャークするよりも複雑なものを書いた場合、巨大で厄介なbashスクリプトに非常に悩まされ、Pythonインタープリターの2番目の開始に我慢したいと思います。
  4. bashスクリプトは、使用可能なGPIOの1つをプルし、その上でLEDがハングします。 次に、LEDのはんだ付けを解除し、ソリッドステートリレーを取り付けます。その制御回路は基本的に同じLEDです。
  5. Index.htmlには小さなjs-​​codeが含まれており、美しい画像を表示します。電球は押されると点灯/消灯し、CGIスクリプトへの非同期リクエストを実行します。また、すでに電球を点灯している場合にN秒ごとにジャークします別のコンピューターから-状況に応じて画像を更新します。
  6. キットに付属している電源からすべてを供給し、以前に分解し、最終デバイスのサイズを小さくするためにモジュール自体をケースから取り外します。 必要に応じて、この手順はオプションです-好きなように食べることができます。




始めましょう。 この時点で、ルーターを既にOpenWRTにアップグレードし、イーサネットケーブルで接続する必要があります。

SSH / WinSCPを経由してルーターに移動し、 / etcに移動します



wifiを構成する


このためには、 / etc / config / networkに移動し 、そこに書き込みます 



config interface 'wlan' option proto 'dhcp'


これにより、Wi-Fiコントローラーは、ホームWi-FiディストリビューターのDHCPから別のルーターの前にあるIPルーターを確実に受信します。

/ etc / config / wirelessでは、思いやりのある最初の起動スクリプトが既に無線設定を生成しており、wi-fiモジュールを含む行#option disabled 1をコメントアウトし、以下のセクションのようなインターフェース記述子を記述します。 



 config wifi-iface option device radio0 option network wlan option mode sta option ssid --- option encryption psk2 option key --


これらの設定はWPA2-PSK用です。 WPA-PSK暗号化の場合はpsk、オープンポイントの場合はnone、VEPの場合はそれぞれ、wep-公式wiki で詳細を読むことをお勧めします 。

すべてが正常であることを確認してください 



 wifi down wifi iwconfig wlan0


すべてが正常であれば、 iwconfigはアクセスポイントに接続していることを通知します。その後、アクセスポイントはイーサネットから切断され、wi-fi設定を続行します。



GPIOを構成する


次に、ユーザー空間からGPIOピンへのアクセスを許可する必要があります。 コアは独自にこれを行うことができます。このために何もする必要はありませんが、GPIOはLED点滅ドライバーによって既にキャプチャされています。 wikiによると、いくつかのアイドル状態のGPIOが地面に引っ張られており、そのサスペンダーをはんだ付けして、自分の目的に使用できるため、LEDとそのドライバーに触れることはできません。 この場合、ドライバーをアンロードする必要はなく、GPIOをエクスポートするだけです（以下を参照）。 ただし、作業結果をすぐに確認するには、LED付きのピンを使用します。

/etc/rc.localに書き込みます 



 rmmod leds_gpio echo 0 > /sys/class/gpio/export echo out > /sys/class/gpio/gpio0/direction echo 0 > /sys/class/gpio/gpio0/value


これにより、リソースを取得したドライバーをアンロードします。ユーザー空間でピン0を使用し、LEDをオフにして出力と出力0を設定することを指定した後。 今、あなたはコマンドでそれを照らすことができます 



 echo 1 > /sys/class/gpio/gpio0/value


そして完済-これ： 



 echo 0 > /sys/class/gpio/gpio0/value


squeakがまだ完了していないことを忘れないでください。まず、rebootコマンドでルーターを再起動し、同時に再起動後にwi-fiがどのように上昇するかを確認します。



mini-httpdを構成する


カーネルを再構築し、その中にmini-httpdを構築した場合は、構成に直接進みます;そうでない場合は、リポジトリから配置します 



 opkg update opkg install mini-httpd


/etc/mini-httpd.confにいくつかの小さな変更を加えて、最初の行が次のようになるようにします。 



 cgipat=cgi-bin/**.sh


サーバーは、 / www / cgi-bin /にあるすべてのものをキックし、名前がコマンドとして.shで終わるものをすべてキックし、その応答をブラウザに変換します。



ハードウェアボタンを構成する


Wi-Fiにアクセスしたくない場合は、1つのボタンに重複したアクションをハングアップします。/etc/hotplug2.rulesで 、 ボタンボタンのチェックを外すと、 ボタンをクリックすると対応するフォルダーから処理スクリプトが呼び出されます。行は次のようになります。 



 SUBSYSTEM ~~ (^net$|^input$|button$|^usb$|^ieee1394$|^block$|^atm$|^zaptel$|^tty$) {


その後、/ etc/hotplug.d/に移動し、そこにボタンフォルダーを作成し、その中にボタンスクリプトを作成します （ chmod + xボタンでコマンドを実行する権限を設定することを忘れないでください！ コマンド ）。 



 #!/bin/sh if [ "$BUTTON" = "wps" ]; then if [ "$ACTION" = "released" ]; then state=`cat /sys/class/gpio/gpio0/value` if [ "$state" = "0" ]; then echo 1 > /sys/class/gpio/gpio0/value else echo 0 > /sys/class/gpio/gpio0/value fi fi fi


ボタンを押すと、LEDの状態が切り替わります。 必要に応じて、このボタンの下のLEDをコントロールと逆位相で点灯させることができます。つまり、シャンデリアがオフの場合、ハードウェアの電源ボタンが強調表示され、暗闇での検索が簡単になります。ピン0をエクスポートするようにピン26をrc.localにエクスポートするだけですそして、同様の出力ライン1/0を、ゼロピンの値の出力の隣の値に追加します。



これはすぐに確認できます。すべてを正しく行った場合、ボタンを押すとLEDのステータスが変化します。



CGIスクリプトの作成


ここで/ etcから出て/ wwwに移動し、 cgi-binディレクトリを作成します。そこには、以下の内容のchmod + xを実行するlight-control.shスクリプトがあります。 



 #!/bin/sh echo "content-type: text/html" echo "" input=$(echo $QUERY_STRING | tr "&" "\n") for param in $input do name=`echo $param | cut -f 1 -d "="` value=`echo $param | cut -f 2 -d "="` if [ "$name" = "action" ]; then if [ "$value" = "on" ]; then echo 1 > /sys/class/gpio/gpio0/value elif [ "$value" = "off" ]; then echo 0 > /sys/class/gpio/gpio0/value fi fi done cat /sys/class/gpio/gpio0/value


＃！/ Bin / shの後の最初の2行はCGIスクリプトに必要です-すべてが問題ないことをブラウザに伝え、htmlを発行します。その後、標準の要件に従ってヘルメットが2つ\ r \ nになります。

次に、変数$ QUERRY_STRINGから、注意深いmini-httpdが配置したパラメーターを取得し、それぞれをnameとvalueに解析します 。 通常、ここでは1つのパラメーターのみを使用しますが、拡張する場合に備えて、コードをより一般的な形式にしてください。

さらに確認します-action = onが与えられた場合、 action = offの場合、ダイオードを点灯します -消灯します。

その後、 値から読み取った値を常に返します。ダイオードがオンの場合は1、そうでない場合は0を返します。 したがって、 action = on / off以外のパラメーターまたはまったくパラメーターを指定せずに外部からスクリプトをプルすると、LEDの現在のステータスが取得されます。



フロントエンドを書く


ディレクトリ/ wwwからダウンロードします。 jqueryサイトの-それでajaxクエリを作成するのは本当に便利です-私は人生で初めてこれらのすべてのJavaスクリプトを見ました（これは私の研究と興味から最も遠い領域であるため）が、Googleを使用して5分で必要なアクションを習得しました。 jqueryをjquery.jsとして保存します

ここでは、シャンデリアを象徴する名前lamp_on.jpgおよびlamp_off.jpgの下に写真を保存します。これらは、2つの部分に切り分けて簡単に合わせてGoogleから盗まれた写真でできています。 必要に応じて、任意の写真を使用できます。











同じ場所でindex.htmlを作成します。 



 <html> <head> <script type="text/javascript" src="jquery.js"></script> <script type="text/javascript"> function LampImgOn() { $('#lamp_off').hide(); $('#lamp_on').show(); } function LampImgOff() { $('#lamp_on').hide(); $('#lamp_off').show(); } function ProcessResult(data) { if(data=="1\n") LampImgOn(); else LampImgOff(); } $(document).ready(function() { $.ajaxSetup({ cache: false }); $('#lamp_on').hide(); jQuery.get('/cgi-bin/light-control.sh', {'action': 'none'}, ProcessResult); $('#lamp_on').click(function() { jQuery.get('/cgi-bin/light-control.sh', {'action': 'off'}); LampImgOff(); }) $('#lamp_off').click(function() { jQuery.get('/cgi-bin/light-control.sh', {'action': 'on'}); LampImgOn(); }) window.setInterval("jQuery.get('/cgi-bin/light-control.sh', {'action': 'none'}, ProcessResult)",2000); }); </script> </head> <body bgcolor="#000000"> <img id = 'lamp_off' src = 'lamp_off.jpg' height = '40%' style='display:none'> <img id = 'lamp_on' src = 'lamp_on.jpg' height = '40%' style='display:none'> </body> </html>




ここでは、ランプが点灯および消灯している2つの最初は不可視の画像をページに配置し、開始後、JavaスクリプトはCGIに現在のステータスを要求し（ action = noneパラメーターを使用）、結果がProcessResultに送信され、1つまたは別の画像がアクティブになります。 非同期ステータススイッチングリクエスト（写真のクリックイベントハンドラーで実行されます）の後に同じ関数が呼び出されます。また、タイマーが開始され、スクリプトはステータスを2000ミリ秒ごとにプルして、誰かが変更したイベントの関連データを表示します別のページまたはハードウェアボタン。



実際、これですべてです。ルーターのIPブラウザにアクセスして、LEDを制御する美しいページをお楽しみください。 ページを開いた状態でルーターボタンをクリックして2秒間待機すると、タイマーからの要求により画像自体が変更されます。

鉄



ハードウェアに関しては、状況は2つあります。 実際、私たちが持っている回路は最も単純です-ソリッドステートリレーの制御回路は本質的に同じLEDであり、必要なのは、電流をオンにするのに十分な電流が流れることを保証することです。 S202S02の場合、これは少なくとも8 mAです。

残念ながら、GPIOから直接取得することはできません。電圧はGPIO 2.6Vです。その後、非常に小さなケースに抵抗があり、公称値は260オームで、LEDは約1.5Vの電圧降下があります。 したがって、約4〜5 mAが流れます。 S202S02の電圧降下は1.2〜1.4Vであるため、ほぼ同じです。 もちろん、同じケースで抵抗器を探すこともできますが、公称値を小さくして再はんだ付けし、10 mAで不幸なGPIOをロードしますが、より簡単な方法で行く方が良いです-NPNトランジスタのベース-エミッタ接合0.75Vでは、0.7 mAの電流が流れ、コレクターへのリレー制御回路をオンにします。

問題は異なります。 実際には2つあります。



  1. LEDのはんだ付けを解除し（LED4がある）、代わりにトランジスタをはんだ付けする必要があります
  2. リレーケース（簡単、小さい）と電源の内部に何かを押し込む必要があります。


パラグラフ1について。回路が実行されるテキソライトは単純です。

トラックとパッドは即座に引きちぎられ、テクストライトに一致するはんだは、この同じテキソライトがすでに燃え始めたときに溶け始めます。

パラグラフ2の場合。ケースが5 mm高い場合は問題ありません。 しかし、残念ながら、電源とパルストランスのコンデンサは高すぎてケースを閉じることができません。

残念ながら、私はあなたがケースに無敵を美しく素早く詰め込むことができる理想的な魔法の解決策を見つけませんでした。 多くの松葉杖のオプションがあります-適合するプラスチックケース（通常の一般的なオプション）の購入から始まり、元のケースをそのまま保持するために青い電気テープとゴムバンドで終わります。

私はその間に何かを選んだ-なぜなら 私はまだ多形のプラスチックを持っていた、私は元のケースの周囲にそれの狭いストリップを置き、それに蓋を押し込んだ。

ですから、この問題については、自分に合った解決策を見つけることを誰もがお勧めします。 もちろん、大きなケースを使用する場合、またはこの小さなケースの外側にPSUを取り付ける場合、ボードからコネクタをはんだ付けする必要はありません。私が行ったことを説明します。

ボードが電源とともにケースに収まるように、すべてのコネクタ（イーサネット、USBホスト、マイクロUSB電源）をはんだから外し、イーサネットから絶縁しました。 ボードは突然平らになりました。 これを行う場合は、非常に注意してください。 ヘアドライヤーからの流量が多すぎるために不注意を払ったため、システムに3.3 Vを供給する重要なU5コンバーターがはんだ付けされていませんでした。 さらに、彼は床に吹き飛ばされ、長い間彼を探す必要がありました-しかし、幸いなことに、私たちは彼をはんだ付けすることができました。







ジャンパーR113 （赤丸）に注目します-デフォルトでは開いています。 閉じたら、USBホストコネクタを介してルーターに電力を供給できます。 彼はSMDではありません。 したがって、 R113を閉じ、電源用の2本のワイヤをコネクタの下から穴に静かにはんだ付けします。

ワイヤーを「土」の穴にはんだ付けするのがどれほど難しいかに気づきましたか？ それはすべての層の埋め立て地に接続されています-ヒートシンクは半狂乱で、私のはんだごてはほとんど管理されていません。

そして、これが電源の内部です（スケールのためにルーターボードの隣）







プラグに接続するために使用された2本の脆弱なワイヤをはんだ付けし、そこでより信頼性の高いものをはんだ付けしてから、220Vネットワークに接続します。 同時に、ルータの電源線に直接接続するため、USBコネクタのはんだ付けを解除します。

電源線が何にも触れないように、はんだ付けによりリレーを準備します。 私はこのようにはんだ付けしました：
