Windows 98の下に生命はありますか、パート2-ソフトウェアについて

続編でかなり引きずった。 仕事が原因で、怠が原因で、仕事がやや大きかったという事実が原因です。 ただし、すべての検索については説明しません。結果のみを表示します。



最初に人生の意味を決定する必要がありますが、:-)







通常、レトロコンピュータはレトロゲーム用に組み立てられています。 彼らには、原則として、特別な問題はありません（まあ、その時代に関連するものを除く）。 したがって、この「人生」には価値がないと考えてください。 インターネットに座って、テキストを入力し、音楽を聴き、映画を見るために、私はより日常的な観点から人生を見ました。 もちろん、後者は使用される鉄よりもOSにそれほど依存しませんが。 私のトイレはDVDとdivxには十分なはずですが、x264以降のコーデックでは対処できない場合があります。



鉄について



戻らないように、鉄を思い出させてください。 2001年頃に関連するコンピューターを組み立てました。

Celeron Tualatin 1200 @ 1600（バス上の133）、512メガバイトのメモリ（815チップセットのマザーボードには表示されなくなりました）。



Matrox Iの代わりにGeForce2 MX200をインストールしたので、最後の記事が公開されてからビデオカードが変更されました。 2001年の選択でさえ最良ではありませんが、GeForce3は見つかりませんでした。既存のMX440はもう少し後です。 そして、2004-2005年の構成でMX200とpentiums4の組み合わせを見ました。人々はサンアンドレアスでそれらをプレイしようとしました。

オペレーティングシステム



次のトピックから、OS-Windows 98SE、ロシア語。



自然界には、多数のドライバーとソフトウェアを備えたwin98IFアセンブリがありますが、そのヒープが非常に大きいため、気に入らないのです。 不要すぎて、Photoshopでさえ突き抜けた。 最小バージョンでもシンプルなWindowsを入手することはできません。 したがって、クリーンOSにmaximusから更新パッケージをインストールすることにしました。



自然には、すべてを独自に配置するテールを備えた300メガバイトのオートパッチャーもありますが、英語版ではより閉じ込められており、これらの300メガバイトでスタックしているものを理解したくありませんでした。



さらに、主にmega.nzに使用されているソフトウェアへのリンクを提供します。このリンクでは、自分でダウンロードしました。



まあ、一般的に、遠くに行かないように、すべてを1か所にしましょう。



最初のものはIE6sp1を置きましたが 。

次に、 Maximus Decim Cumulative Update 3.05 （更新、大容量ディスクのサポート、USBストレージのサポートなど）。

次に、 DirectX 9c 。

その後、 MD InternetExplorer 6.0sp1コンポーネントアップデート3.4 -IE6のアップデート

MD Data Access Component Update-データにアクセスするためのあらゆる種類のコンポーネント（XML、ODBCなど）。



その後、 KernelEx-4.5.2.exe- 「Windows用のWindowsエミュレーター」を使用すると、OSの新しいバージョン用に設計されたWindows 9xプログラムで実行でき、NTラインから互換モードなどのプログラムプロパティに設定を追加できます。 万能薬ではありません-一部のプログラムはkernelexの使用も開始しません。 しかし、人生はまだやや楽です。







.Net 1.1および2.0をインストールすることですべてを完了しました。



まあ、純粋に審美的な理由で、彼は32ビットアイコン（ RP9.7 ）で可愛さのサポートを追加しました。



どういうわけかこのようになりました（ただし、これは既にソフトウェアの一部としてネタバレを含んでいます:)）：







はい、私自身の便宜のため、主にVirtualBox仮想マシンで実験をセットアップしました。 これにより、いくつかの機能が導入されます。たとえば、実際のハードウェアよりもはるかに平坦化した仮想マシンの同じFirefoxなどです。

しかし、検索結果のほとんどすべてのリンクは既に壊れています。 ただし、保存されたコピーを注文して、そこで何かを読むこともできます。



その結果、多くのエンターテイメントの後、98の最後の賢明なネイティブブラウザーはOpera 9.64であるという結論に達しましたが、すでに十分ではありません。 その結果、KernelExを使用して、 Opera 12.02とFirefox 10.0.12 ESRを起動できました。

したがって、暇なときにVoodoo2 SLIをビルドして実行する必要があります。 ここにAWE32を添付して、幸福感を完成できます。
