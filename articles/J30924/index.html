バーチャルメン：キーブラー向けIMボット

昔ながらの企業Webサイトの中で、IMはブランドを宣伝する安価な方法を提供します。 ただし、それでもまだ完全ではありませんが、IMロボットは既に、ユーザーの関心や会社の関心を考慮してユーザーの質問に答えることができます。



ベッキーは、レシピ以外のことを話すことを強制することはできません。 私は主題を変えようとしましたが、彼女は絶えず台所に戻りました。 しかし、私たちの会話はそれほど深くはありませんが、夕食に何を料理するかを決めようとすると、彼女はいつも彼女の代わりにいます。 例えば、かつて、私はインド料理が大好きだと彼女に言いました。



「ちょっと待って、何？ すみません、理解できませんでした」と彼女は答えた。 彼女は会話を巧みに行いますが、それでもしばしば聞き取れません。 「そうでなければ教えてください。」 ベッキーがシカゴ郊外の外を考えるのが難しい場合があります。



「インドの食べ物が好きです」と私は説明した。



「OK、あなたは七面鳥のレシピを見たいと思います。 もう知ってるよ！ リンゴを口に入れた独自のジュースで揚げた七面鳥はいかがですか？」



それから私の質問を要約しようとしました。 「私は中東の食べ物が好きだという意味でした。」



「OK、中東料理のレシピを見たいと思います。 私が持っているものを見てみましょう。 行くぞ！」 ベッキーはボンベイサンドイッチのレシピを示しています。 そして、さらに1秒後に、彼女は突破します。「キーブラーバニラウェーハは本物のエルフと同じくらい良いです！」



Becksは、RecipeBuddie（レシピリトルマン）と呼ばれるインタラクティブエージェントで、インスタントメッセージング用に特別に開発されたボットで、インターネットでKeeblerブランドを宣伝する必要があります。 彼女の仕事は、IMを介して彼女に連絡する消費者との実際の会話をシミュレートし、独自の基準に基づいて、データベース内の7,000以上のレシピから適切なオプションを選択できるようにすることです。 そしてもちろん、ユーザーにKeebler製品を購入することを思い出させます。 バニラクッキーについての彼女の言及は、私をKeeblerウェブサイトに導くハイパーリンクでした。 RecipeBuddieは日食を経験しますが、Kelloggの子会社であるKeeblerは、Beckyの助けを借りて世界中のIMユーザーが自社製品について学ぶことを望んでいます。



クックブック



RecipeBuddieは、6歳のインターネットメーカーであるEmediaによって開発されました。Emediaは、Annie's Homegrown、Bayer、Del Monteなどの多くのボックス会社と提携しています。 Emediaは1996年からKeeblerと協力し、1997年にKeeblerの公式Webサイトを作成しました。しかし、今日では、インターネットユーザーの手の届く範囲に、顧客はもはや単純なサイトを持ちません。



5年前、私たちは「私たちはウェブサイトを作っています」と言うでしょう。 彼女は、これがボブのサイトの時代だったと説明しています。 さあ、私たちも自分でそれを修正します。」



しかし、90年代の終わりまでに、マレーは次のように語っています。 過去4〜5年で業界は完全に変化し、今では戦略的コンサルティングを求め、オンライン広告キャンペーンの実施と検索エンジン最適化の実施を依頼しています。 ウェブサイトの開発は、当社が現在行っていることのほんの一部です。 人々は、私たちがウェブサイトの作成に自分自身を制限すると、期待される利益をもたらさないと感じています。」



2001年にマレーがActiveBuddyについて読んだとき、彼女はすぐにインターネットでKeeblerを宣伝するための優れたツールを見つけたことに気付きました。 ActiveBuddyは、インスタントメッセージングネットワーク、プライベートコーポレートネットワーク、またはワイヤレスネットワーク用のエージェントを作成するという単純な目的で、2000年にTimothy Kayによって設立されました（さらに、Webサイトに埋め込むことができます）。 Kayeの業績を考えると、1997年にKayは3-Dコンピュータグラフィックスの革新的な作品でオスカー賞を受賞しました。ActiveBuddyエージェントが面白い画像を受け入れようと努力するのは驚くことではありません。



2001年までに、ActiveBuddyはこの分野で最初のエージェントをリリースする準備が整いました。 ActiveBuddyのプロダクトマーケティングディレクターであるKathy Englar氏は、次のように述べています。 このエージェントは、アムネシアックのレディオヘッドのニューアルバムを宣伝していました。 このエージェントは、AOLインスタントメッセンジャーユーザーの連絡先リストに登録され、グループとそのアルバムに関する一般的な人間の質問を理解し、回答することができました。 ファンは、グループ、そのツアー、曲のリスト、伝記、アルバムの内容に関する情報に興味がありました。



さらに、2001年6月、ActiveBuddyはSmarterChildのオープンデモバージョンを公開しました。 エージェントは、正確な時間、注目度の高いニュース、個人向けの天気予報、参考資料、財務データ、スポーツ分野の情報に関する質問に回答しました。 彼女は、トリビア、ブラックジャック、ハングマンのプレイ方法も知っていました。 翌年、800万人の驚いたIMユーザーと話をすることができ、ログが示すように、「愛しています」というメッセージを何百万回も受信しました。 または孤独な大人。



2002年6月までに、ActiveBuddyは、SmarterChildがゴールドメンバーでAustin Powersを宣伝するためにNew Line Cinema向けに開発したエージェントよりも人気が高まっていることを発見しました。 「デモ版はクライアントプロジェクトよりも人気があるというのは間違っているように思われたため、削除することにしました」とInglar氏は言います。 （Webサイトで確認できます。www.smarterchild.comを参照してください）。 1年で、SmarterChildはActiveBuddyの強力な可能性を明確に実証しました。



1か月後、BuddyScript SDKプログラムがリリースされ、RecipeBuddieはその助けを借りて作成された最初のエージェントの1つでした。 その創造は、マレーに彼の文学的な筋肉で遊ぶ機会を与えました。



「インターネットを1つの堅実なテレビ広告に変えるべきという事実よりも、この言葉の力を信じています」と、とりわけ著者として出版されているマレーは言います。 しかし、彼女がこのキーブラーのアイデアを提案したとき、企業の男性は急いでそのような事業の成功を信じることはありませんでした-彼らの多くは、明らかに、インスタントメッセージングが何であるかについてほとんど知りませんでした。 マレーは、テストを手配し、SmarterChildシステムのスポンサーシップラインを最初に購入するように説得しました。 「トートバッグに関する小さなテキスト行は、クリックスルーの6.5％をもたらし、ロールアウトではさらに3％をもたらしました。」 これにより、キーブラーはActiveBuddyのオファーを受け入れるようになりました。



レシピ



BuddyScriptは、ActiveBuddyおよびその他のエージェントを作成するために使用されるプログラミング言語です。 データベース内の特定の製品に関する情報の検索から、電話ダイヤラーなどのアプリケーションの起動まで、さまざまなアクションを実行できます。 BuddyScriptのユニークな機能は、プログラマーが通常の人間の言語によって提起された質問、つまりユーザー自身が取り上げた単語で構成される質問に対する反応を作成できることです。



BuddyScript SDKはwww.buddyscript.comから無料でダウンロードできます。これにより、開発者は、社内での使用（たとえば、エージェントがスタッフからのよくある質問に答えることができます）およびインターネットマーケティングおよびプロモーションの両方でインタラクティブエージェントを作成できます。 現在、3000人以上の開発者がすでにSDKをダウンロードしています。 BuddyScript Server Limited Editionは、既製のスクリプトが作成されるとすぐに、デベロッパーにデモ版をわずか199ドルで提供する機会を提供します。



SDKには、開発者がWindowsインタラクティブエージェントをテストできる統合シェルが含まれています。 さらに、チュートリアルプロジェクトが含まれています。 一般的に使用される関数のBuddyScriptコードスニペットライブラリ。 BuddyScript SDKを使用して作成された既製のインタラクティブエージェントを使用したサンプルアプリケーション。 テストとプロトタイプ作成のためのBuddyScript Server Developer Edition。



エージェントを起動する準備ができるとすぐに、2つのオプションが表示されます。企業で使用する場合は、シートごとのライセンスが提供されます。 RecipeBuddieのようなマーケティングプロジェクトで使用される商用バージョンは、セッション数に基づいて推定されます。 「ActiveBuddyのサーバー構成により、何百万人ものユーザーにサービスを提供できます」とマレーは言います。



BuddyScript SDKは、AOLインスタントメッセンジャー、MSNメッセンジャー、Yahooメッセンジャーなど、さまざまなメッセージング環境のサポートを提供します。 2002年10月にリリースされたBuddyScript Connectivity Serviceの助けを借りて、BuddyScriptアプリケーションはWAP、電子メール、SMS、RIMなどのワイヤレスプラットフォームでも動作します。 つまり、アプリケーションを作成した開発者は、デスクトップに加えて、携帯電話、ポケットベル、その他のワイヤレスデバイスとの互換性をすぐに得ることができます。



「BuddyScriptはそれほど複雑ではありませんが、BuddyScriptを使用するには、Javaプログラミングを適切に学習する必要があります」とMurray氏は言います。 SDKは、特別なエンジニアリングスキルを持たない人向けに設計されています。つまり、HTMLとJavaScriptの使用方法を知っていれば、SDKを快適に操作できます。
