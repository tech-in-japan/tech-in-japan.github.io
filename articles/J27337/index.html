ASUS Eee PCにUbuntu 8.04をインストールして構成する



ASUS Eee PCを自分で購入したとき、どのオペレーティングシステムにインストールするかは考えていませんでした。これは間違いなくUbuntu Linuxでしょう。 第一に、私は自宅と職場の両方にUbuntuを持っているため、第二に、自宅と職場の両方にUbuntuリポジトリのローカルミラーがあるため、そして第三に、単にこのディストリビューションが私に適しているからです。



購入後、私はすぐにインターネットを調べて、Eee PCにUbuntu 8.04をインストールするためのさまざまなHowToを探しました。 結果は私を幾分驚かせました。 Eee PCには大きな興奮がありますが、ネットワーク上にはこれらのHowToはそれほど多くありません。そのほとんどはUbuntu 7.10を対象としており、システム構成ファイルを独自のEee PCに置き換えるスクリプトの実行を提案しています。



これらのスクリプトはUbuntu 7.10向けであり、Ubuntu 8.04での起動は悪い結果を招く可能性があるため、これらのスクリプトも検討しませんでした。 したがって、ネットワーク上の必要な情報を検索する際に、Eee PC用にUbuntuを「do-it-yourself」に設定する方法を説明した記事のみに注意を払いました。 自分の前に生じたすべての質問に対する答えを含むようなHowToを見つけることができなかったので、収集した情報を体系化し、1つの記事にマージすることにしました。 これでシステムはすでに構成されているので、システムで実行したすべてのアクションを思い出して、すべてが希望どおりに機能するようにします。 何も見逃さないことを願っています。 :)





設置



インストールには、Ubuntu 8.04のブートディスクと1 GB以上のフラッシュドライブが必要です。

フラッシュドライブを使用する前に、そこからすべての重要なデータをコピーしてください。

＃ディスクイメージを作成する

dd if = / dev / cdrom of = ubuntu.iso



＃syslinuxパッケージをインストールする

sudo apt-get install syslinux



＃ISOイメージをフラッシュドライブに書き込むスクリプトをダウンロードする

wget kiwilinux.org/public/isotostick.sh



＃このスクリプトを実行可能にする

sudo chmod a + x isotostick.sh



＃/ dev / sdX1がフラッシュドライブのデバイスファイルであるとさらに想定されています。



＃フラッシュドライブをフォーマットする

sudo mkfs.vfat -F 32 -n ubuntu / dev / sdX1



＃パーティションを起動可能にする

sudo parted / dev / sdX set 1 boot on



＃ダウンロードしたスクリプトを実行する

sudo ./isotostick.sh cd.iso / dev / sdX1



＃起動可能なフラッシュドライブを作成する

sudo syslinux / dev / sdX1


これで、ディスクからこのフラッシュドライブから起動し、Eee PCにUbuntuをインストールできるようになりました。 これを行うには、Eee PCの起動中にF2を押してBIOSに入り、[起動]タブで、フラッシュカードからの読み込みが行われるように適切な設定を行います。



読み込み中



オペレーティングシステムの起動速度を上げるには、/ boot / grub / menu.lstで次の行を置き換えます
＃defoptions = quiet splash
に
＃defoptions = quiet splash clocksource = hpet
そして走る
sudo update-grub
これで、オペレーティングシステムの起動時間が約40秒短縮されます。

プロセッサ周波数の動的な変更



すぐに、エネルギーを節約するためにプロセッサの周波数を下げることに関するフォーラムでの議論を読んだとき、私は多くのバッテリー寿命を節約することは不可能だと言った人々の意見にしばしば会いました-最大10-15分。 私は自分でテストをしませんでしたが、周波数の削減は自分で設定しました-少なくとも暖かくなるかもしれません。これらの15分も不要ではありません...



また、製造業者はプロセッサの周波数が900 MHzであると主張していますが、実際には、630 MHzの周波数で動作することにも注意する価値があります。 レビューから判断すると、周波数を900 MHzに上げてもパフォーマンスは向上しません-バスを加速する必要がありますが、私はしませんでした-私にとって、バッテリーの寿命ははるかに重要です。



だから...



必要なパッケージをインストールします。これにより、プロセッサの周波数を即座に変更できます。
sudo apt-get cpufrequtilsをインストールします
そして、次の行を/etc/rc.localに追加します。
＃プロセッサー周波数の動的変更をオンにします

modprobe p4-clockmod

modprobe cpufreq-ondemand

＃プロセッサー周波数のスムーズな変更をオンにします

＃112 MHz〜675 MHzの範囲の負荷から。 新しい価値

＃周波数は500ミリ秒ごとに計算されます。

echo 112500> / sys /デバイス/システム/ cpu / cpu0 / cpufreq / scaling_min_freq

echo 675000> / sys /デバイス/システム/ cpu / cpu0 / cpufreq / scaling_max_freq

powernowd -m 1 -p 500
動作中にアプリケーションの反応速度があなたに合わない場合、最小プロセッサ周波数を増やすことができます（/ sys / devices / system / cpu / cpu0 / cpufreq / scaling_available_frequenciesで利用可能な周波数のリストを参照）またはpowernowdユーティリティの-pオプションの値を減らします。

SSDの寿命を延長



SSDへの書き込み操作の回数を減らすには、次の行を/ etc / fstabに追加して、すべての一時ファイルとログがRAMに書き込まれるようにします。
tmpfs / tmp tmpfsデフォルト、noatime、nodiratime 0 0

tmpfs / var / tmp tmpfsデフォルト、noatime、nodiratime 0 0

tmpfs / var / log tmpfsのデフォルト、noatime、nodiratime 0 0


注意！ / tmpに非常に大きなサイズの一時ファイルを作成するプログラムを使用する場合、そのようなファイルを作成するときにRAMが不足しないように最初の行を追加しないでください。



SSDパーティションのマウントオプションで、noatimeおよびnodiratimeオプションを指定します。 つまり、パーティションが文字列でマウントされている場合
UUID = 79d788c4-a8fb-4f03-92c2-e90d4c26b51b / ext2デフォルト、エラー= remount-ro 0 1
その後で置き換えます
UUID = 79d788c4-a8fb-4f03-92c2-e90d4c26b51b / ext2デフォルト、noatime、nodiratime、errors = remount-ro 0 1
これらのオプションを有効にすると、このセクションのファイルまたはフォルダーにアクセスするたびに最終アクセス時間が更新されなくなります。



Locateコマンドを使用しない場合は、/ etc / cron.daily / mlocateファイルを削除（または単に＃！/ Bin / sh行の後にexit 0を配置）できます。 このスクリプトは毎日実行され、システム内のすべてのファイルのパスを含むロケートデータベースを更新します（名前でファイルをすばやく検索する必要がある場合、locateを使用すると便利です-既製のデータベースを使用しないため、findコマンドの動作はずっと遅くなりますが、毎回ディレクトリツリー全体を参照します）。 Locateを使用する場合、/ etc / cron.daily / mlocateを/etc/cron.weekly/に移動して、データベースが週に1回更新されるようにすることができます。



また、SSDでの書き込み操作の回数を減らすには、すべてのログをオフにします（私見、ポケットラップトップでは必要ありません）。
apt-get remove klogd sysklogd logrotate




SSDのスペースを節約する



localepurgeパッケージをインストールします。 必要なロケールのリストを要求し、他のすべてのロケールを削除します。 localepurgeは、aptが起動するたびに自動的に起動します。



Synapticを起動し、[設定]-> [設定]に移動し、[列とフォント]タブに移動して、[インストール済みサイズ]フィールドを選択します。 これで、パッケージのリストに別の列-サイズが表示されます。 パッケージがインストールされている場合、この列には、このパッケージのファイルが占めるサイズが表示されます。 この列でパッケージのリストをソートし、多くのスペースを占める不要なパッケージをすべて削除します。
