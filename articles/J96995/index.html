Slidebar.ruクラウドホスティングの紹介





偶然、クラウドホスティングサービスslidebar.ruをテストしました。 この記事では、その操作方法、サービスの構成および管理方法について説明します。 私はサービスのコストの問題に触れず、このオファーを他のホスティングと比較しません。 第一に、比較するのに十分な経験とデータがないため、そして第二に、この記事の目的は、仮想マシンの構成に対する柔軟なslidebar.ruの提案を示すことです。



サービスの使用を開始するには、アドレスhttp://slidebar.ru/に移動して、製品がスピンする必要な仮想マシン構成を選択する必要があります。







このサービスにより、非常に柔軟な構成が可能になります。
  • ターゲットオペレーティングシステム：Windows Web Server 2008 R2、Windows Server 2003、Linux / BSD;
  • 256 MBのステップで512 MBから8 GBのRAMサイズ。
  • 1、2（Linuxバージョンでは利用不可）または4（LinuxおよびWin2003バージョンでは利用不可）プロセッサコア。
  • 10％刻みのパーセントで使用可能なプロセッサー能力。
  • 20（Linux、少なくとも40 GBのWindowsの場合）から20 GB単位で200 GBまでのストレージ。
パラメーターが変更されると、右側の価格タグが変更されます。 次の設定を選択しました。







価格は1時間ごとに計算されることに注意してください。 これにより、以下で説明するパラメータの計画的な変更により、リソースの使用を柔軟に評価できます。



サービスを購入すると、料金の取得を確認する手紙と管理コンソールへのリンクが届きます。 コンソールは次の形式を表します。







さらに、2番目の文字には、仮想マシンのRDPのパスワードが含まれています。 コンソールには、サービスの主な機能である現在のシステムパラメータを柔軟に設定する機能が含まれています。メモリ、プロセッサ、電力、ストレージ容量を設定できます。 これにより、負荷が増加するとリソースが増加し、減少するとリソースが減少するという、サービスの作業の変更に緊急に対応できます。



RDPを使用してサーバーにWeb製品をインストールした後、サービスに変更を加えて、プロセッサー容量とストレージ容量を増やしました。







[変更]をクリックすると、次のウィンドウが表示されます。







同時に、その時点でのサーバーとのオープンRDP作業セッションは強制的に終了されました。



サービスのマイナス1つがここに表示されます：アプリケーションの処理中、管理コンソールへのアクセスはありません。複数の仮想マシンがあり、1つだけの構成が行われたとしても、2番目の仮想マシンへのアクセスは得られず、リクエスト処理でこのウィンドウに転送されます。



アプリケーションの処理時間は短かった-約10〜15分後、コンソールへのアクセスが現れ、さらに5分後にRDP経由でサーバーにアクセスできるようになりました。 また、構成で指定した料金とデータを確認する手紙を受け取ります。



追加のディスク領域は割り当てられず、自分でパーティションを作成する必要があります。







また、増加したディスク容量は削減できないという事実にも注意を払う必要があります。







したがって、ストレージのサイズを賢明に計画する必要があり、1つの仮想マシンのフレームワーク内で過剰なボリュームを拒否することはできないことに留意してください。



そのため、15分でリソースの特性を大幅に改善しました。 この場合、パラメーターを指定する以外に何もする必要はありませんでした。 移行に対処する必要はありませんでした。すべてのファイル、パラメーター、およびシステム設定は変更できません。



スライドバーサービスが提供するものを見てみましょう。



slidebar.ruのもう1つの興味深い機能は、特定の日時の構成変更をスケジュールする機能です。







私のテストは時間的に制限されていたという事実のために、私はこの機能を経験しませんでしたが、それは私には非常に便利なようです。 出席率の低い一時エリアとリソースへの負荷を把握している場合は、事前に特性の変更を計画することにより、この期間のコストを削減できます。 逆に、訪問者の流入を事前に予測し、この時点で仮想マシンのより洗練された構成を注文できます。



slidebar.ruコンソールを使用すると、システムの構成の変更を追跡できます。







slidebar.ruサービスを使用すると、複数の仮想マシンを作成できます。 新しいものを作成するには、システムで最初の関税を作成するときに届く手紙のリンクに従ってください。







新しい自動車を作成することは、実際に最初の関税を作成することと実質的に違いはありません。 システムパラメータも選択します。 例外は、ここで提案されたリストからオペレーティングシステムのイメージを選択するか、「オペレーティングシステムなし」を選択する必要があることです。







私の場合、プリインストールされた無料のWeb Platform Installerを使用するオプションを選択しました。これにより、さまざまなWebプロジェクトを2回クリックするだけでデプロイできます。 ところで、この場合、SQL Server 2008 Expressがプレインストールされていることに注意してください。



追加のオプションオプションは、管理サービスを注文することです 。詳細については、 こちらをご覧ください 。



新しい仮想マシンのパラメーターを選択した後、10〜30分以内に作成されると予想されます。 ここでも、すでに説明した欠点に遭遇しました。新しい仮想マシンの作成中、コンソール全体にアクセスできません。 マシンを作成すると、RDPのパスワード、IPアドレス、および仮想マシンへのリモートアクセスの手順へのリンクが記載された電子メールが届きます。



slidebar.ruコンソールには、マシンを管理するためのオプションがいくつかあります。 それらを順番に考えてみましょう：







サーバーを再起動して起動順序を構成する機能：DVD、HDD。







仮想マシンのDVDに完成したイメージを設定する機能と、それに続くシステムからのブート機能。 画像のバリエーションが写真に示されています。



コンソールから、DNSとメールを構成できます。











スライドバーが提供するもう1つの興味深い管理ツールは、System Center Virtual Machine Manager 2008 R2コンソールへのアクセスです。







このコンソールにアクセスするためのログインとパスワードはスライドバーコンソールに配置されているため、忘れることは困難です。 ログイン後、標準のSystem Centerインターフェースが表示されます。 その中で、仮想マシンを起動、一時停止、完全に切断し、RDP経由で接続できます。







これで、slidebar.ruからクラウドホスティングサービスのレビューを完了したいと思います。



このサービスに対する個人的な印象は好意的なままでした。料金プランの構成を構成および変更するための柔軟なオプションが気に入っていました。 サービスの価格は、おそらく従来のVDSホスティングの提供よりも高いでしょう。 ただし、1時間ごとの給与と、構成の変更と使用するリソースを計画する機能により、このギャップはわずかに減少します。 サービス自体は便利で、うまく機能し、仮想マシンの管理に必要なツールのセットを提供します。



誰もがサービスをテストできることを付け加えます。支払いなしでサーバーを起動すると、50ルーブルとすべてを自分の手で感じる機会が与えられます。
