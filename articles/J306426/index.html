Hewlett Packard Enterprise Server Automationによるサーバーオートメーション

現代の企業は、仕事でますます多くのサーバーを使用しています。 このため、ツールの実装に必要なルーチンタスクの数、システムの運用で遵守する必要がある標準が大幅に増加します。 これにより、メンテナンスコストが増加するだけでなく、さまざまな障害による損失も増加します。 ほとんどの問題の解決策として、自動化ツールを使用できます。 その1つがHewlett Packard Enterprise（HPE）Server Automationです。 このソフトウェアは、管理を集中化してデータセンターの多くの機能を合理化し、ITインフラストラクチャの管理における重要な場所を自動化するように設計されています。











サーバーを知る



HPE Server Automationは、プログラムの制御下にないサーバーのネットワークをスキャンし、特別なリストに表示します。 次に、管理者は、各サーバーにソフトウェアモジュールをインストールして、これらのサーバーをシステムに接続します。 これが完了すると、次のような管理タスクを実行できます。



専用リソースへのオペレーティングシステムの展開 ：サーバーに物理スペースを割り当てるか、事前に構成されたオペレーティングシステム用の仮想サーバーを作成し、それらを管理対象サーバープールに入力できます。 その後、HPE Server Automationは新しく作成されたリソースを管理できます。



OSの自動アップグレード ：HPE Server Automationは、オペレーティングシステムを自動的かつ一元的かつ柔軟に更新し、必要なパッチをインストールできます。 これは、Windows、Linux、およびSolarisファミリのオペレーティングシステムに基づいたサーバーに適用されます。 オペレーティングシステムのサプライヤが提供するパッチから必要なパッチを選択するだけでなく、サーバー環境と互換性のないパッチが考慮されないようにインストールプロセスを構成できます。



ソフトウェアの初期化 ：サーバーが管理プールに入った後、ソフトウェアポリシーと呼ばれる特別なテンプレートを使用してアプリケーションをインストールおよび構成できます。 これらのテンプレートは、インストールするソフトウェア、使用する構成、およびインストール中に使用するスクリプトを決定します。 これらのテンプレートを使用すると、HPE Server Automation Softwareポリシーコンプライアンス機能に従って、すべての管理対象ユニットに展開される基本的なサーバー構成を定義できます。 たとえば、Apacheの基本バージョンは、HPE Server Automationによって管理されるすべてのサーバー、およびそれらの特定の部分にインストールできます。



監査とコンプライアンス ：監査および修復機能を使用すると、サーバー構成ポリシーを見つけて、それらがポリシーに準拠していることを確認できます。 矛盾が見つかった場合、またはサーバーが希望どおりに構成されていない場合は、ソフトウェアで修正できます。 そのため、ベースサーバーを構成し、そのイメージに基づいて、他のすべてのサーバーを自動的に構成し、設定したポリシーに合わせます。



ソフトウェア構成 ：アプリケーション構成テンプレートを作成し、HPE Server Automationを実行しているすべてのサーバーに展開できます。 プール内のすべてのサーバーの構成ファイルの標準化を確認することもできます。



アプリケーションの展開 ：この機能を使用すると、作成したアプリケーションを開発チームからテスターに​​、さらにチェーンに沿ってすばやく簡単に移動できます。



ソフトウェアコンプライアンスチェック ：ポリシーコンプライアンススキャンツールを使用して、管理対象サーバーのソフトウェアポリシー仕様がインストールされているソフトウェアの構成と一致するかどうかを判断できます。



レポート ：HPE Server Automationは、管理対象サーバーに関する包括的なステータスレポートの広範なスイートを提供します。 これらのレポートは柔軟に構成できるため、さまざまなカテゴリのユーザーに提供できます。



HPE Server Automationを使用すると、さまざまなシステムをより安全かつ柔軟に変更できます。 これは、実際に変更を実装する前に、変更をモデル化して検証できるためです。 これにより、サーバーのダウンタイムが大幅に削減されます。これは、初めて導入された変更の操作性がすでにテストされているためです。 変更を加えたり、更新をインストールした直後に、それらをデバッグしてエラーを修正する必要はありません。



HPE Server Automationの構成



シンプルなHPE Server Automationインストールパッケージは、システムコア、そのコンポーネント、および単一サーバーでホストされるOracleデータベースで構成されます。 より高度なパッケージには、追加の要素が含まれる場合があります。



-メインコアを補完し、サーバー管理の能力を高めるセカンダリカーネル。



-衛星。機能は従来のカーネルと似ていますが、機能がより制限されています。 要件やリソースが限られているデータセンターやITインフラストラクチャに使用されます。



-マルチマスター接続。2つの独立したシステムがデータを交換し、共同サーバー管理を実装できます。



HPE Server Automationは、8つの指定された構成でのインストールをサポートしています。 その他の構成には、HPE Professional Servicesが必要です。



シンプルな構成



HPE Server Automationは、サーバー管理機能を提供する多数のコンポーネントをインストールします。 製品のインストールを構成する必要がない場合は、単一ホストのインストールを選択できます。 構成が必要な場合（たとえば、パフォーマンス上の理由で異なるサーバーにカーネルコンポーネントを分散する場合）、HPE認定コンサルタントに連絡する必要があります。







最も単純なインストールの場合は、単一のデータセンターまたは施設にシステムを展開することです。 これは、同じネットワーク上の単一のサーバー管理ホストにインストールされたHPE Server Automationのすべての自動化コンポーネントで構成されています。



Oracleデータベース



すべてのシステム展開オプションには、HPE Server Automation専用に構成されたOracleデータベースが必要です。 そのコンポーネントの1つは、ネットワーク、ストレージデバイス、管理対象サーバー（およびそれらにインストールされているオペレーティングシステムとアプリケーション）などに関する情報を格納するために使用されます。 このデータベースはインストールパッケージのコンポーネントですが、HPE Server Automationで使用するために構成された既存のOracleデータベースを使用できます。



アプリケーションの展開



HPE Server Automationアプリケーション展開機能を使用すると、データセンター内のターゲットサーバー用のカスタムソフトウェア製品を作成、テスト、および展開できます。 たとえば、アプリケーションを開発者からテスターに​​移動して、テストの品質を向上させることができます。 アプリケーション展開ツールは、アプリケーションの展開に必要な通信を簡素化します。 開発プロセスに関与するすべての従業員が作業するデータを表示または入力できる単一のアクセスポイントを提供します。 このツールでは次のこともできます。



-コード、スクリプト、構成ファイルなどのアプリケーションコンポーネントをシミュレートします。



-アプリケーションの複数の並行リリースおよびバージョンを管理します。



-ターゲットサーバーにアプリケーションを展開して最小化し、不安定な操作やエラーが発生した場合にバージョンをロールバックします。



-アプリケーションに必要な要素を実行するターゲットサーバーをシミュレートします。



-開発者、テスター、システム管理者、およびその他の許可された担当者の間で、便利で迅速なコミュニケーションを確立します。



-コードの記述から製品のリリースまでのアプリケーション開発サイクルを計画および実装します。 エンタープライズのタイプ、採用されている標準、およびその動作に応じて、Hewlett Packard Enterprise Server Automationを構成できます。



監査と修正



監査およびエラー修正機能を使用すると、ITインフラストラクチャの一部であるオブジェクトをチェックするためのオブジェクト、場所、および時間を設定できます。 さまざまな検証ポリシーによって、特定のファイル、ディレクトリ、特定の構成など、検証する必要があるものが決まります。 検証の場所も決定されます。これらは特定のサーバーまたはそのグループ全体です。 また、チェックスケジュールを作成する機能によりこのプロセスが自動化されます。たとえば、チェックに特定の日付を設定し、その規則性を確認できます（チェックは1回限りまたはスケジュール可能）。



HPE Server Automationは、管理環境でサーバーを互換性のあるものにする方法についての洞察も提供します。 そのため、サーバーを特定のポリシーの標準に準拠させるオプションをアクティブにできます。 指定された標準を満たしていないサーバーをシステムが検出した場合、受け入れられた標準に準拠するように操作を修正できます。



HPE Server Automationクライアントを使用すると、適切なパラメーターを備えた稼働中のサーバーまたはそのようなサーバーのイメージに基づいて、サーバー構成パラメーターの値を確認できます。 また、独自の設定または事前に構成された監査ポリシーに基づいてチェックを行うこともできます。 さらに、特定のサーバーのステータスに関するレポートを受信して​​、システムの現在のステータスを確認したり、他のサーバーと比較したりすることができます。



監査ポリシーを使用すると、会社または業界全体で採用されているセキュリティ標準が現在の状況に対応しているかどうかを判断できます。 このような標準を自分で定義することもできます。 これらは、監査、レポート、およびその他の監査ポリシーで使用されます。



監査ツールを使用して修復および復元する場合、次のタスクも実行できます。



-サーバーとシステムスナップショットを指定されたサーバーとスナップショットと比較します。

-定期的に実行されるチェックを作成します。

-組織の互換性とセキュリティの基準を設定する監査ポリシーを作成します。

-特定のサーバーまたはサーバーグループの特定のチェックを作成します。

-ファイル、ディレクトリ、パッチ、レジストリキー、パッケージなどの問題を含む、さまざまなレベルの問題を修正します。



Hewlett Packard Enterpriseサーバーオートメーション機能



この製品は、多くのITプロセスを自動化する一連の機能です。 これらには、Service Automation Visualizer、Storage Visibility and Automation、およびレポートシステムなどのツールが含まれます。



Service Automation Visualizer（SAV）



このツールは、IT環境の運用アーキテクチャと分散ビジネスアプリケーションの理解と管理に役立つように設計されています。 ご存知のように、これらのアプリケーションは、ネットワークおよびストレージデバイスだけでなく、複数のサーバーで実行される多様なサービスの複雑なセットで構成されています。 この点で、パフォーマンスの問題が発生した場合、原因を特定して排除することは困難です。 また、IT環境に特定の変更を加えた場合の結果がどうなるかが不明確になります。 SAVツールを使用すると、図面を使用してこのタイプの情報を視覚化できます。 モニター画面に表示したり、紙に印刷したりできます。



ストレージの可視性と自動化



このツールを使用すると、ウェアハウスへのデータ入力のチェーン全体の開始から終了までの視覚化により、データストレージを管理できます。 これは、システム管理者の日常的なタスクに非常に役立ちます。 利用可能なリソースの確認、ストレージ設備の監査、リソース使用傾向の分析、さまざまなシナリオの作成、プロセスの自動化により、スペース（および結果としてお金）を節約するための資金が提供されます。



報告書



リアルタイムHPE Server Automationレポートシステムは、管理対象サーバー、ネットワークデバイス、ソフトウェア、パッチ、オペレーティングシステム、クライアントおよびオブジェクト（仮想および物理）、およびさまざまなポリシーの適用に関するさまざまな情報をユーザーに提供します。 セキュリティおよびユーザーアクションに関するレポートを取得することもできます。



リソース展開システム



HPE Server Automationを使用すると、事前設定済みのオペレーティングシステムを物理サーバーおよび仮想サーバーにインストールできます。 手順は迅速に、最小限の人間の関与で行われます。 この機能は、サーバーの起動プロセスの重要な部分です。 自動展開機能は、施設内の各サーバーがデフォルトで標準化されたオペレーティングシステム構成を持つようにします。 このシステムの利点は次のとおりです。



-他のHPE Server Automation機能およびツールとの統合 。 リソース展開システムは一連の自動化ツール（自動パッチ管理、ソフトウェア管理、スクリプト実行などを含む）と緊密に統合されているため、異なる部門間で作業をスムーズに転送できます。 これにより、IT部門のさまざまなグループが、進行中のプロセス、作業環境の一般的な状態を理解しながら作業することが保証されます。 これは、生産的な作業と変更の実装を確実に制御するための鍵の1つです。



-再インストールせずにソフトウェアサーバーをアップグレードする機能 。 他の多くの同様のソリューションとは異なり、HPE Server Automation Resource Deployment Systemにより、インストール後に簡単に変更を加えることができます。 これは、特別なテンプレートとパッケージ指向のアプローチの使用によるものです。



-多くの環境で動作するように設計された柔軟なアーキテクチャ 。 HPE Server Automation展開システムは、さまざまな種類のサーバー、ネットワーク、セキュリティアーキテクチャ、および運用プロセスをサポートしています。 この柔軟性により、組織のニーズを満たすために必要なオペレーティングシステムを展開できます。



HPE Server Automation機能は、通常のクライアントとWebクライアントの両方から実行できます。 システムは、すべての一般的な展開タスクを自動化します。



-準備されたOSインストールプロファイルと指定されたシーケンスの使用を使用して、オペレーティングシステムをインストールするための機器の準備。



-計画の作成。これは、オペレーティングシステムをインストールする前後にサーバーで実行する必要があるタスクのリストです。 このような計画は、事前定義されたインストールシーケンスよりも優先されます。



-所定のシーケンスまたは構築計画に従って、オペレーティングシステムの基本構成およびデフォルト構成をインストールします。

-オペレーティングシステム用のすべての最新パッチのインストール（リストはサーバーで実行されているアプリケーションによって異なります）。



-OSのインストール前後に特定のスクリプトを実行します。これにより、特定の構成パラメーター、たとえばルートパスワードが設定されます。



-たとえば、Secure Shell、PC Anywhere、ウイルス対策ソフトウェアとデータバックアッププログラム、システム監視ツールなどのシステムコンポーネントとユーティリティのインストール。



-Java Virtual Machineなどの一般的なシステムソフトウェアのインストール。

スクリプト実行



HPE Server Automationのスクリプト機能を使用すると、管理対象サーバーのプール全体で、手動ではなく自動モードで1回限りまたは保存済みのスクリプトを実行できます。 システム管理者にとって、これには次の利点があります。



-多くのUNIXおよび/またはWindowsサーバーでのスクリプトの並列実行。これにより、時間を大幅に節約し、高度な一貫性を提供します。



-アクセス制御は、許可された管理者のみがアクセスできるホストでスクリプトを実行できるという原則に基づいて実行されます。



-スクリプトをプライベートまたはパブリックライブラリに保存することにより、スクリプトへのアクセスを制御する機能。



-管理者がサーバーのステータス情報にアクセスできる場合、スクリプト設定をグループ化する機能。 これは、適切なスクリプトが適切なサーバーで実行されるために重要です。



-特定のスクリプトがいつ、どこで実行され、誰が責任を負うかを示す包括的なレポートのログブック。



スクリプティングツールはHPE Server Automationの不可欠な部分であるため、システム管理者にはサードパーティ製ツールに比べて独自の利点があります。
