Foxのいくつかの異なるバージョンを実行する方法に関する小さなメモ。 通常のユーザーには必要ありませんが、Web開発者とコーダーが役に立ちます。

Foxバージョン2.0を既に持っているとします*、そして3.0をダウンロードしたレコードを目指しています。 急いで配置しないでください。すべてが偶然に残っている場合、デフォルトのインストールは以前のプロファイルを上書きします。 したがって、 Firefox -ProfileManagerのような-ProfileManagerキーを使用して、既にインストールされているバージョンのFoxを起動します。

追加のプロファイルを作成し、その後、3番目のバージョンのインストールを開始します。 同じディレクトリに個別に配置する必要があります。次にディレクトリを選択しました。 このようなことが判明しました。バージョンの接頭辞のために、私は特別な天才には輝きません。



バージョン 置く場所 プロフィール
2.0.14 C：/プログラムファイル/ Firefox / 2.0.14 / 2.0.14
3.0 C：/プログラムファイル/ Firefox / 3.0 / 3.0




そして、実際にfoxを起動するfirefox.bat bashスクリプトを作成しましょう 

 @エコーオフ
 MOZ_NO_REMOTE = 1を設定
 start "" "C：\ Program Files \ Firefox \％1 \ firefox.exe" -p％1
 MOZ_NO_REMOTE = 0を設定




これは、インストール条件でのバッチファイルです（1ファイルのみ）。 パスが異なる場合やプロファイルの呼び出し方法が異なる場合は、条件に合わせて変更するか、バージョンごとに独自のバージョンを作成してください。 bashスクリプトの2つのショートカットを作成し、引数としてFoxのバージョンを転送します。



さて、タスクは小さいですが、ラベルでバージョンを区別する方法は？ 次のソースから取得できる新しいアイコンへのパスを指定するだけです

  • Firefoxアイコンのタッチアップ
  • Firecons1
  • DeviantART-Firefox
