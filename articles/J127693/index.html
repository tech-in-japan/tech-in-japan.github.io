Ubuntu 10.10のGuitar Pro 6-インストールの問題の解決

DEB拡張機能は、すべてがそれ自体で実現するという自信と希望をもたらしました。 それにもかかわらず、dpkgに従ってパッケージを正常にインストールした後、問題がすぐに発見されました。プログラムは音がしません-MIDIもRSEも動作しません。 タブ譜を再生し始めたとき、Guitar Proは欠落しているサウンドバンクを呪い、RSEを無効にした場合（F2を押して）問題を報告しませんでしたが、音はまだありませんでした。



足をまとめて、Wineを介してプログラムに満足すること（定期的なクラッシュと英語のインターフェースのみの表示）は私には受け入れられないようでした。私はあなたと共有したい問題の解決策を見つけました。



サウンド設定（[ ウィンドウ]メニュー-> [サウンド]-> [オーディオ設定] ）で、以下が検出されました。

  • Checkボタンに応答しない1つのMIDI出力（MIDI Through：0）。
  • PulseAudio出力サウンドカードドライバー。
一般設定（F12または[ ウィンドウ]メニュー-> [ファイル]-> [設定] ）では、サウンド処理に関連する1つの項目のみ：

  • 「マルチスレッドRSE処理を有効にする」-マルチトラックタブ譜を再生するときのブレーキを回避するために、それをオンのままにしておくことをお勧めします。
[ウィンドウ]メニュー-> [ファイル]-> [開く]-> [SoundBanks.gpbankファイルを選択]からサウンドバンクをプログラムに供給すると、「Guitar Proは新しくインストールしたサウンドバンクで作業を開始するには再起動が必要です」と表示されましたが、サウンド全体が表示されず、ストーリー全体が表示されませんでした繰り返した。



したがって、問題の本質は明らかです。サウンドバンクは、インストール中またはインストールされたプログラムの標準的な手段ではロードされません。 したがって、それらを手動で接続する必要があります。



Guitar Proの場所を確認します。

sudo find / -name GuitarPro6







とりわけ、以下を見つけます。

/opt/GuitarPro6







このディレクトリにはGPBankInstallerファイルがあり、これはSoundBanks.gpbankアーカイブのアンパッカーであることがわかります。 すぐに使用します。

sudo /opt/GuitarPro6/GPBankInstaller [ Soundbanks.gpbank] /opt/GuitarPro6/







たとえば、私にとっては次のようになりました。

sudo /opt/GuitarPro6/GPBankInstaller /home/smoren/gp6/Soundbanks.gpbank /opt/GuitarPro6/







その後、Guitar Pro 6を大胆に開き、表示されたサウンドをお楽しみください。



PS MIDIサウンドに関しては、疑問は未解決のままです。RSEが無効になっている（サウンドバンクを使用している）場合、サウンドはまだありません。 必要なドライバーはすべてリズムボックスにあります* .midファイルが再生されます。



もちろん、RSEが（5番目のバージョンとは異なり）再生中だけでなく、タブのコレクションでも使用されるようになり、MIDIを完全に置き換えます。 しかし、MIDIサウンドの不足は憂慮すべきものであるため、この問題の解決策も見つけようとしています。 アイデアがあれば、コメントで読んでいただければ嬉しいです。
