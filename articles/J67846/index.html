CDMAモデムUbiquam UM-400の概要

毎年夏に、私は、他の多くの人と同様に、町を出ます。 次の出発に備えて、雨の夜を快適に過ごすことができるように、村に高速無線インターネットを設置するのは良いことだと思いました。 私の選択は、オペレーターSkyLinkとその関税「Suburban Internet」にかかっていました。 防衛省の割り当てられたゾーン（私が行く村）の1メガバイトの1ルーブルの価格は魅力的ではありませんか？ その前に、私はすでにUbiquam U-300電話を持っていましたが、同じ番号で連絡を取り合い、RUIMの貼り付けによって苦しめられないようにするために、Ubiquam UM-400モデムを購入しました。



金融では当時、新しいモデムを買うことができなかったので、近くのラジオ市場で中古モデムを買ったので、バンドルについては何も言えません。 ただし、SkyLink Webサイトの説明から判断すると、キットにはモデム自体とドライバーディスク以外のものは含まれません。 したがって、私がそれを買ったとき、私は保証を除いて多くを失いませんでした。



組み立てられたモデムは次のようになります。







このように分解：











モデムの左側には、ヘッドセットジャックがあります。接続すると、理論的には電話をかけることができます。 しかし、ロシアのモデムパーティーにはそのような機能がないため、このコネクタは役に立たない。







右側には、外部アンテナを接続するためのポートがありますが、これはおそらく必要になるでしょう（これについては後で説明します）。







動作中、モデムは次のようになります。







左側のライトは、電源インジケータとアクティビティインジケータの両方の役割を同時に果たします（アクティブなデータ転送中に点滅します）。 左側の2番目のライトは、現在利用可能なネットワークを示します：1X（赤、最大153 kbit / s）またはEv-DO rev。0（青、最大2400 kbit / s）。 Ev-DO rev.Aこのモデムはサポートしていません。 残りのライトは信号レベルを示します。



次に、モデムに付属のソフトウェアを検討してください。 ドライバーと一緒に、Access Managerプログラムがインストールされ、その助けを借りてモデムと接続が制御されます。

プログラムが起動すると、次のウィンドウが表示されます。







その中で、現在の時刻（なぜ？）を確認し、接続を確立してモデムを構成します。 設定を考慮してください。















ご覧のとおり、設定では、RUIMカードのPINコードの要求を有効または無効にし、データ転送アカウントを構成し、接続モード（1X、Ev-DO、または混合）を選択できます。 多様性は小さいですが、私たちには十分です:)



「接続」ボタンを押して、プログラムの外観を確認します。







クロックが接続時間カウンターに変わり、現在のフローのインジケーターが表示されました。 より詳細な統計を開いて、チャートを見ることができます。







デバイスとソフトウェア自体のレビューは終了しているので、トピックの冒頭、つまり国内のインターネットに戻りましょう。 これにより、このモデムはあまり良くありません。 接続しようとしたとき、何も起こりませんでした。 完全な受信信号のインジケータが消え、ネットワークが存在しないことを示し始めました。 しかし、接続を停止するとすぐに、インジケータは完全な信号位置に戻りました。 Ubiquam U-300電話は、同じ場所で1Xネットワークを完全にキャッチしました。Ev-DOはあまり良くありません。 私はラップトップとモデムを使って出かけようとしましたが、それでもログインできませんでした。 それから私は電話で接続しようとしました。 接続に成功しました。 飛んでいない、パケット損失がありますが、動作します。 これとの戦いで、そのような設計が行われました：







電話が屋根の下の白い箱に入れられ（モデムを試しましたが、まだ十分な電力がありませんでした）、すでに自宅から接続が確立されていました。 速度が150 kbit / sから1500 kbit / sに増加すると、パケット損失はなくなりました。 利益、それは思われる。 しかし、この決定により、携帯電話はネットワークにアクセスするために使用され、屋根の下にぶら下がったため、電話接続ができなくなりました。 より単純なソリューションが発明されました-銅線の大きなコイルをモデムのアンテナポートに差し込みます。 約700 kbit / sの速度で獲得した接続。



そして、これがモスクワ、オチャコボ地区でのデータ転送速度テストです







おそらく、これらは一時的なネットワークの不具合であるか、ここでは常にこのようになっている可能性があります。以前はテストしていません。 市内の他の場所では、速度は毎秒約2メガビットでした。



結論です。 このモデムは、市内で高速インターネットを必要とする人に適しています。 モスクワ地域では、モデムの使用は受信状態がかなり悪いため機能しません（私の場合、基地局まで5キロメートルでした。CDMA2000の場合、この距離は正常です）。



UPD：速度テストを再度実施しました。 結果はずっと良くなります。 どうやら日中はネットワークが遅れていたようです。
