はじめに



libhybrisとは


libhybrisは、ライブラリをBionicユーザースペースからGlibcユーザースペースにロードできるレイヤーで、一部の文字をGlibcのオプションでオンザフライで置き換えます。 簡単に言えば、このソリューションを使用すると、Linux空間でAndroid専用のライブラリを使用できます。 もちろん、最大のメリットは、製造元がAndroid専用に作成した独自のGPUドライバーを使用できることです。



Surfaceflingerとは


surfaceflinger-ネイティブAndroidサービス、グラフィックレイヤーの複合マネージャー。

Ubuntu touch


Ubuntu Touch Developer Preview自体はAndroidに基づいており、ハードウェアを操作するために必要なサービスを借りています。 依存関係の一般的な概要は、 Ubuntu Touch PortingまたはOpenNetのメモにあります。



基本オペレーティングシステムとして、通常のAndroid JB 4.2が使用されるか、むしろCyagenMod-10.1 （CMサブプロジェクトのリポジトリはphablet.ubuntu.com/gitweb ）です。 dalvikとjavaに関連するものはすべて削除されました-システムサービスとHALで構成されるネイティブ部分のみが残ります。 必要に応じて、 AOSP 4.1を使用できますが、4.1のネイティブAPIに適応する準備ができており、ドキュメントや仕様によってカバーされておらず、リリースごとに異なります。



UTコンポーネントはchrootにあり、自己記述されたuchrootユーティリティが使用されます 、抜粋： 

static int ubuntum(void *a) { /* Chroot */ chroot("/data/ubuntu"); chdir("/"); /* Set basic env variables */ char *const envp[8] = { "container=aal", "PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin", "SHELL=/bin/bash", "HOME=/root", "USER=root", "USERNAME=root", "LOGNAME=root", NULL }; /* Exec shell */ execle("/sbin/init", "/sbin/init", "--verbose", NULL, envp); return 0; }


Android環境とUbuntuのchroot環境間の相互作用には、 libhybrisメカニズムが使用されます。

Ubuntu Platform API


UbuntuプラットフォームAPIは、 プラットフォームの機能（Android）を使用して基本操作を実行するための低レベルAPIです。



メソッドの例：

  • ubuntu_application_ui_show_surface
  • ubuntu_application_ui_hide_surface
  • ubuntu_application_ui_move_surface_to
  • ubuntu_application_ui_resize_surface_to


doc / mainpage.mdドキュメントファイルから、 platform-apiソースツリーは2つの部分に分割できることがわかります。

  • include-抽象宣言プラットフォームAPI
  • android -Android用のプラットフォームAPIの実装（Android 4.2の場合-


そして、このAPIを使用する際にサードパーティの開発者が信頼できる唯一のものは、

/ ubuntu / applicationディレクトリを含み 、他のすべてのものは時間の経過とともに変化すると予想されます。



debian / controlファイルから次のことを学びます：
