Microsoft Project 2010-少し選ぶ

単純だが頻繁に遭遇する問題の解決策を提案したい：



プロジェクト活動に加えて、私たちの生活にはまだ多くの時間のかかる体系化されていないタスクがあります（名刺の注文、セミナーでの講演、候補者とのミーティングなど）。 記録を保持するのではなく、個別のプロジェクトとして記録を保持することは不可能です-スケジュールと処理が重複する非常に大きなリスクがあります。



さらに、これらのタスクは、高度なプロジェクト計画ツールが利用できない秘書やパーソナルアシスタントによって管理されることがよくあります。



Outlook-SharePoint-Project 2010の統合により、驚異的なタスク管理が可能になります。



まず、構成します。



1）Project Server 2010データを操作するために、SharePoint 2010に特別なノードを作成します











2）コンテキストテープのコマンドを使用して、Outlookとの同期を設定する







3）Project Serverで、このリストに基づいて、タスクエントリを作成します







出来上がり！



1）秘書にOutlookでのタスクの作成を任せることができます-







2）秘書または私たち自身がポータル上のすべてのタスクを制御できます（同期は双方向で自動的に行われます！）







...ところで、SharePointには、必要に応じてタスクの関係を構成する機会があります。







そして、いつでもProject Server 2010と情報を同期できます







Outlookからのすべてのタスクは、Project Serverに正しく反映されます







そして、すべてのプロジェクトと個々のタスクについて、自分自身とすべての同僚の計画的な読み込みをオンラインで見ることができます。
