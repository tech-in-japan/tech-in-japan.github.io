関税




関税は異なります。



ほとんどのホスティングプロバイダーでは、料金はほぼ同じです。 ディスク容量、ドメインおよびデータベースの量。

しかし、これは完全に間違っています。 メインページには大声で叫ばれない「落とし穴」が常にあります。 しかし、あなたはいつでもそれについて読むことができます-原則として、契約書とTOSで。



関税を比較するときに探すべきもの：



  1. リソース ディスク容量やドメイン数はありません。 そして、プロセッサ時間と割り当てられたメモリの量。 料金表に示されたサイトの数は100であることがよくありますが、実際には、最大のサイトではなく1〜2で既に、CPUまたはMEMの制限に達しています。
  2. 交通 明確に理解するための主なもの- 制限はありません 。 それだけです 美しい言葉「無制限」が書かれているすべての場所には、別の制限があります。チャンネル幅、比率、またはリンクのダウンロードです。 または、さらに悪いことに、制限は暗黙的です。たとえば、5メガビットの消費に達すると、より高い関税に切り替えるか、まったく出かけるように「丁寧に求められます」。 100メガバイトの「実際の」無制限の帯域幅は、最も高価な共有ホスティングよりもはるかに高価です。
  3. サブドメイン、データベースなどのリソースの数の制限。




利用規約




ほとんどのホスティングプロバイダーには、サービスの提供に関するルールがあります。 最初にすべきことは、それらを注意深く読むことです。



このドキュメントでは 、コンテンツリソース、サーバーの負荷、不適切なコンテンツなど、非常に重要な点でクライアントとしての責任を定義しています。



次のことに注意してください。



  1. サービスの支払い順序 。 一部のプロバイダー、特にVDSの提供または専用のプロバイダーでは、深夜5分前ではなく、送信終了の数日前の翌月の支払いが必要です-アプリケーションを処理する時間がなく、資金が入金されません->サーバーまたはサイトが停止します 現在、即時支払いの普及により、これはまれになっていますが、特にクレジットカードによる支払いと銀行振込（銀行のチャージバックに対する保護）に関しては依然として存在しています。
  2. 禁止されているコンテンツ。 はい、正確に。 多くのプロバイダーは、トレントトラッカーとシードボックス、政治的サイト、宗教的サイト、薬を販売するサイトなどのコンテンツの種類を禁止しています（Herbalife、Yeah）。 ほとんどすべてのロシア人はポルノ、ウェアーズ、スパムとスパマー、フィッシング、ウイルスなどを持っています。
  3. 返金の条件。




これらの条件は、原則として、まったくひどいものではありません。 ただし、サイトについてではなく、「アカウントが一時的にブロックされています」と表示されないように、それらについて知っておく必要があります。



テクニカル





技術的な部分に移りましょう。



  1. 構成 購入する前に、サーバーソフトウェアの構成がサイトの要件（apache \ phpモジュール、設定など）を満たしているかどうかを確認してください。 高い確率で、特にあなたにとって、誰もソフトウェアをインストールしたり、設定を変更したりすることはありません。あなたのような人々のサーバーには何十ものものがあります。
  2. パスワード パスワードをコントロールパネル\ FTPから教えないでください。 クライアントパスワードを使用して実行されるすべてのアクションについて、クライアントが責任を負います！ あなたがスパムのためにブロックされている場合、誰も気にしません、あなたはそれをスパムしたか、あなたの友人\妻\プログラマー\猫をスパムしました。 コントロールパネルの部分では、追加のログインペアを作成できます。パスワード、この機能を使用して一時的なアクセス（デザイナーへのアクセスなど）を与え、古いペアを削除します。
  3. ウイルス。 IFrameウイルスの約80％とスパムの50％は、ソフトウェアまたはホスティングプラットフォームの穴からではなく、サイト管理者から盗まれたパスワードを通じて送信されます。 ウイルス対策ソフトウェアをインストールして構成し、ホスティングパスワードを明示的に保存したり、サードパーティプログラム（特になじみのないプログラム）に保存したりしないでください。 結果-ポイント1、パスワードを参照してください-あなたは応答しています。
  4. バックアップ。 自分でバックアップについて考える必要があります。 ホスティング業者がどのようにバックアップを行っているかを正確に調べてください。 多くの場合、コピーに関するシステム管理者をホストするという考えは、あなたのものと一致しません。 たとえば、任意の日をロールバックできる機能を備えた増分バックアップが必要で、ホスティング事業者は週に一度アカウント全体の一般的なバックアップを作成します。 あまり古くないコピーを手元に置いてください。 ホストには、問題、HDDの死、データセンターの爆発などがあります。 データを失わないために、それを保管してください。 個人のサイトが、1つのローカルバックアップなしで、作業を開始する前に1行のコードからホスティングに住んでいた場合を知っています。 このすべてが一度死んだとき-それは非常に不快でした。




心理的





具体的には、あなたが理解され、助けられていることを確認する方法。 Blitzブロック、噛む必要はないと思います。



それらとのコミュニケーション。 サポート：




  1. サイトで別の連絡先が示されている場合は 、 すぐに全員に書き込む必要はありません 。 自分に関する最大の情報（アカウント、サーバー、エラー、場所）を報告する必要があります。 これは最も重要なことであり、場合によっては3〜4倍で問題を解決する時間を短縮できます。
  2. 丁寧に話します。 サポート/マネージャーにぶつかったり、悲鳴を上げたり、脅迫したりする必要はありません。 それは助けにはなりませんが、「ウラルスキーのクライアント」に対する態度は適切です。 また、アルバニア語で「petrosyany」と書いたり、他の方法で神経を傷つけたりしないでください。 あなたのオリジナリティは通常、ホスティング事業者にとってほとんど関心がありません。
  3. STPホスティング取引（サプライズ！）ホスティングサポート。 Linux、プログラム、HTMLのレイアウト、ヘリコプターの飛行方法を教える必要はありません 。 そのような質問をしない方がいいです。
  4. 「さて、いつ??」、「ここにいるのか？77」、「待っててうんざりです！1111」など、100回尋ねる必要はありません。 答えは速くなりませんが、遅くなります。




サイトの運用におけるエラー：




  1. サイトを示してください。 1つまたは5つのサーバーはありません。問題がどこまで容易であるかを正確に見つけることはテレパシーです。
  2. 「何もうまくいかない」-これはバグレポートではありません。 問題を常に詳細かつ明確に説明してください。 50メッセージの長さとの対応は、問題の解決をスピードアップしません。
  3. 「しかし、それはVasyaCoolHostingで機能しました！」-これは議論ではありません 。
  4. 正常に動作していないことを確認してください。 正しく機能しているが、望みどおりに機能していない場合、質問は役に立ちません。
  5. 問題が大規模で、設定ではないことを確認してください。
  6. 問題が自分で消えた場合、または自分で解決した場合-お知らせください。 存在しないバグを強制的にキャッチさせないでください。
  7. グローバル設定\ソフトウェアバージョンを変更するように依頼するのは無意味です。 あなたは一人ではありません。
  8. 重大ではない技術的な問題を解決するための標準的な期限は24時間です。 システム管理者も人間であり、コントロールパネルの非常に必要なボタン以外にも他のタスクがあります。 この24時間の間、10分ごとに「そのような人は誰ですか？」と尋ねる必要はありません。




可用性、速度、ブレーキの問題：




  1. サイドの問題（ファイアウォール、プロキシ、ISPの問題、ローカル管理者のブロック）を確認します。
  2. pingとトレースを作成し、問題の説明に添付します。
  3. サイトにデバッグシステムがある場合（たとえば、スクリプトやデータベースクエリの実行時間を記録する場合）、そのデータを添付します。
  4. 「スローダウン」は一般的に非常に主観的な概念です。
