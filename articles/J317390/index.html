企業内のPCやモバイルデバイスにソフトウェアを一元的およびリモートでインストールする方法





会社の有効性とセキュリティは、従業員のワークステーション、サーバー、モバイルデバイスで使用されるソフトウェアに直接依存します。 一方、「不要な」ソフトウェアは、従業員の職務遂行をそらすだけでなく、追加のセキュリティリスクを引き起こす可能性があります。 RMMソリューションを使用して必要なソフトウェアを中央およびリモートでインストールする方法を教えてください。



パンダシステム管理の例を考えてみましょう



一元化されたソフトウェアのインストールが必要な理由



おそらく、この質問に対する答えは明らかです。企業のサーバーやワークステーション、および従業員のモバイルデバイスに必要なソフトウェアをインストールするプロセスを大幅に簡素化および高速化するためです。 そのため、従業員は、職務を遂行するために本当に必要なソフトウェアを正確に使用します。 しかし、そもそも、新しい脆弱性をカバーするために、アップデートとソフトウェアパッチを管理する問題が残っています。



当然、これらの問題はすべて、あらゆる企業に非常に関連しています。なぜなら、それらは仕事の効率に影響し、その結果、競争力に影響を与えるからです。 したがって、集中化されたリモートソフトウェアインストールは、IT部門自体だけでなく、会社全体のより効率的な作業に貢献します。



集中ソフトウェアインストール用のRMMサービス



多数のIT問題の自動化されたソリューションには、企業ネットワークのリモートおよび集中監視、保守、およびサポート用に設計された特別な統合RMMソリューションがあります。 RMMソリューションのクラスには、クラウドサービスのPanda Systems Managementが含まれます。



前に、 このクラウドサービス全体について説明し、 企業ネットワークの監視とモバイルデバイスの管理への使用についても説明しましたが、本日、集中ソフトウェアインストールについて説明することにしました。



Panda Systems Management Serverは、すべての管理対象デバイス（ワークステーション、サーバー、一部のモバイルデバイス）にソフトウェアを自動的かつリモートでインストールできます。 これにより、管理者は、すべての管理対象デバイスに従業員が作業するために必要なソフトウェアがあることを確認でき、必要なプログラムをインストールするために各デバイスに個別にリモート接続する必要がありません。



自動ソフトウェア展開は、管理者がソフトウェアの脆弱性（Java、Adobeなど）に対する保護レベルを維持するのにも役立ちます。これにより、企業の機密データの感染や損失のリスクを大幅に削減できます。

必要条件



ソフトウェアの展開とインストールは、Windows、Linux、およびMacプラットフォームのアプリケーションコンポーネントを使用して実行されるプロセスです。 iOSを搭載したスマートフォンやタブレットにアプリケーションをインストールすることもできます。これについては、この記事の最後で説明します。



スクリプトや監視コンポーネントと同様に、アプリケーションコンポーネントには、一連のファイルやプログラムをインストールするためにこの問題を解決するためのインストールプロセスをガイドする小さなスクリプトが含まれています。



ユーザーのデバイスにファイルまたはプログラムのグループをインストールするには、そのようなグループごとに個別のコンポーネントを作成する必要があります。





ソフトウェアの展開およびインストール手順



一般に、通常の手順は4つのステップで構成されます。



ステップ1.ソフトウェアをインストールする必要があるデバイスの識別



必要なプログラムまたはファイルがインストールされていないデバイスを見つける手順は、Panda Systems Managementがデバイスにインストールされているプログラムを監査できるかどうかによって異なります。



オペレーティングシステムに保存されているインストール済みプログラムのリストにインストール済みソフトウェアが表示される場合、このソフトウェアはPanda Systems Managementソフトウェア監査レポートにも表示されます。 したがって、このソフトウェアが既にインストールされているデバイスをフィルターするフィルターを作成できます。



ソフトウェアにインストーラーがないため、インストールされているプログラムのリストに表示されない場合、または単一のドキュメント、構成ファイルなどである場合、Panda Systems Managementはこれらのファイルが既に存在するデバイスをフィルターで除外できないため、インストールスクリプトは適切なチェックを実行します。



ステップ2.ソフトウェアインストール用のコンポーネントの生成



これを行うには、一般に、次の手順に従う必要があります。

1.必要なデバイスにインストールされているローカルPanda Systems Managementエージェントにインストールするコンポーネントを配布するタスクを実行します。



パフォーマンスへの影響を最小限に抑えるために、ユーザーが自分のデバイスで作業しない特定の日時に設定することにより、このタスクをスケジュールで実行できます。



2.インストール結果を含むデータを収集して、考えられるエラーを確認します。



プロセスが完了すると、システムは管理コンソールにインストール結果を示す可能性のあるエラーコードやメッセージを収集します。



このタスクには、4つの最終ステータスがあります。

• 成功 ：インストールタスクはエラーなしで完了しました。 スクリプトはErrorlevel 0コードを返します。



• 成功-警告 ：インストールタスクはいくつかの小さなエラーで完了しました。 スクリプトは、エラーレベル0コードと対応するメッセージを返します。これらのメッセージは管理コンソールに表示されます。



• エラー ：インストールタスクは完了しませんでした。 スクリプトはErrorlevel 1コードを返します。



• エラー警告 ：インストールタスクは完了しませんでした。 スクリプトは、エラーレベル1コードと対応するメッセージを返します。これらのメッセージは管理コンソールに表示されます。

例1.自己インストールソフトウェアの実装



この例では、Microsoft Framework .NET 4.0用のdotNetFx40_Full_x86_x64.exeパッケージを、まだインストールされていないデバイスに集中的にリモートでインストールしようとします。



Microsoft Framework .NET 4.0は、デバイスのオペレーティングシステムに保存されているインストール済みプログラムのリストに表示されるプログラムであるため、このパッケージがまだインストールされていないデバイスをフィルターを使用して選択します。



インストールパッケージは、/ q / norestartオプションを使用してサイレントモードでインストールし、デバイスの再起動を防止する自己解凍型の.EXEファイルです。したがって、特別な準備は必要ありません。



手順1.パッケージをインストールするデバイスを選択します



このパッケージをインストールするデバイスを選択するには、デバイスフィルターを使用する必要があります。 これを行うには、Panda Systems Management管理コンソールの[ サイト]セクションで必要なサイト（プロジェクト）を開き、[ サイトデバイスフィルター ]セクションの左側の列でプラス記号をクリックして必要なフィルターを追加します。







この結果、デバイスフィルターを追加するためのウィンドウが開きます。ここで、フィルター名、選択基準（ソフトウェアパッケージ、含まない、Microsoft .NET Framework 4）を指定し、[ 保存 ]ボタンをクリックする必要があります。







これで、 Sites Devices Filtersブロックに、選択基準を満たすデバイスのリストを取得できるクリックすることにより、カスタマイズされたフィルターが追加されました。



ちなみに、フィルター条件でソフトウェアパッケージの名前を指定する場合、どの識別文字列がこのパッケージに対応するかを正確に知る必要があります（特に特定のソフトウェアの場合）。 この場合、パッケージが既にインストールされているデバイスのリストでデバイスを検索し（さまざまなフィルターを使用することもできます）、リストからこのデバイスを開き、[ 監査 ]タブを選択し、[ ソフトウェア ]オプションを有効にする必要があります。







ステップ2.ソフトウェアインストール用のコンポーネントを作成する



これは非常に簡単に行うことができます。メインメニューのコントロールコンソールで、[ コンポーネント ]セクションを選択し、[ 新しいコンポーネント ]ボタンをクリックします。 この結果、コンポーネントを追加するためのウィンドウが開きます。 [ カテゴリ]リストで、[ アプリケーション]を選択します 。 コンポーネントの名前とその短い説明（必要な場合）を指定し、 「保存」ボタンをクリックします 。







その後、このコンポーネントのパラメーターを編集するウィンドウがすぐに表示されます。 このウィンドウで、次の変更を行います。



•左上隅にある星をクリックして、このコンポーネントをお気に入りに追加できます。 この場合、星のアイコンは黄色に変わり、コンポーネントのリストにコンポーネントが表示されます。

• [インストールコマンド]リストで適切な値を選択して、使用するスクリプト言語を指定します。 この場合、値はBatchです。

•スクリプトコードを貼り付けます。

•インストールするパッケージファイルを追加します。







実際、このスクリプトには、非表示インストール用の適切なパラメーターを使用してインストールパッケージが起動される1行があります。



手順3.タスクを実行して、必要なデバイスのエージェントにソフトウェアパッケージを展開します



この場合、サイト（プロジェクト）を再度開く必要があります。SitesDevices Filtersブロックの左の列で、以前に作成したフィルターをクリックして、パッケージをインストールする必要なデバイスのリストを取得します。 次に、リストで必要なデバイス（およびほとんどの場合すべてのデバイス）を選択し、アクションパネルでボタンをクリックしてタスクをすばやく起動します（必要に応じて、このボタンの左側にあるボタンをクリックして、このタスクを特定の時間に完了するようにスケジュールすることもできます）







この結果、ウィンドウが開き、必要なコンポーネントを選択して[ OK ]をクリックする必要があります。







手順4.結果を確認して、考えられる問題を特定します。



結果は、[ スケジュールされたジョブ]-> [完了したジョブ]セクションで確認できます。

インストール結果を確認する別の良い方法は、事前に構成されたデバイスフィルターを再度実行して、ソフトウェアがアンインストールされたデバイスがもうないことを確認することです。 同じケースで、このデバイスのリストにいくつかのデバイスが表示されている場合、インストールエラーをチェックできます。



デバイスにインストールされたソフトウェアの監査データは24時間ごとに管理コンソールと同期されるため、ソフトウェアのインストールから日が経過していない場合、[監査]タブのインストール済みソフトウェアのリストは更新されません。 ただし、アクションパネルの[デバイス監査の要求]ボタンを使用して、インストールされているソフトウェアのリストを手動で更新できます。







例2.インストーラーを使用しないソフトウェアの実装



多くのプログラムには、適切なインストーラーなしのシンプルな実行可能ファイルが含まれています。適切なインストーラーは、[スタート]メニューに必要な構造、デスクトップ上のアイコン、またはコンピューターにインストールされたプログラムのリスト内の対応するエントリを生成します（たとえば、コントロールパネル-プログラムとコンポーネント）。 この種のプログラムは、たとえばこの実行可能ファイルを実行するだけでインストールできます。 ただし、この場合、Panda Systems Managementは、PCにインストールされているプログラムの正しい監査を提供できません。 そのようなプログラムは、デバイスのオペレーティングシステムに保存されているインストール済みプログラムのリストには表示されません。



このため、追加されたすべてのプログラムでMSIパッケージを生成し、[スタート]メニューに必要なすべてのグループを作成し、ユーザーのデスクトップにアイコンを作成してプログラムへのアクセスを簡素化できるサードパーティのユーティリティがよく使用されます。



この例では、 Advanced Installerユーティリティを使用します。このユーティリティの無料版では、MSIインストーラを簡単に生成できます。



Advanced Installerの無料版をダウンロードするには、次のWebサイトにアクセスしてください



インストーラーを生成するには、次の手順を実行します。



1.このユーティリティをPCにインストールして実行し、無料のシンプルなテンプレートを選択します







2. [ 製品の詳細]セクションで、インストーラーの基本データを入力します： 製品名 （製品名）、 製品バージョン （製品バージョン）、および会社名 （会社名）







3.インストールする必要があるプログラムとファイル、および作成する必要があるアイコンを追加します。 これは、[ ファイルとフォルダー ]タブで実行できます。











4.このプロセスの最後に、[ ビルド ]をクリックします。その後、選択したフォルダーにMSパッケージが生成されます。







インストールパッケージが生成されたら、バッチスクリプトを除き、上記の例（自己インストールソフトウェアの展開）で説明したように、インストールとその展開のためのコンポーネントを作成するために同じ手順に従う必要があります。 この場合、インストールコマンドはわずかに異なります。







MSIEXECユーティリティを使用すると、/ qn スイッチを使用してサイレントインストールを開始できます。

コンポーネントを作成するとき（前の例の作成順序を参照）、[ カテゴリ]リストから[ アプリケーション]を選択します。 コンポーネント編集ウィンドウで、次の手順を実行します。



•左上隅にある星をクリックして、このコンポーネントをお気に入りに追加します。 この場合、星のアイコンは黄色に変わり、コンポーネントのリストにコンポーネントが表示されます。

• [インストールコマンド]リストで適切な値を選択して、使用するスクリプト言語を指定します。 この場合、値はBatchです。

•生成されたMSIパッケージを起動するMSIEXECユーティリティを示すスクリプトコードを挿入します（この例では、これはMy Software.msiファイルです）。

•インストールするMy Software.msiパッケージを含むファイルを追加します。



例3.スクリプト言語を使用してドキュメントとファイルを埋め込む



この例では、ユーザーのデバイスのルートディレクトリにあるフォルダーに3つのWord文書を埋め込む必要があります。 完全なスクリプトコードは、 このファイルにあります 。 そのため、問題を解決するには、次の手順を実行する必要があります。



手順1.パッケージをインストールするデバイスを選択します



なぜなら この例では、Panda Systems Managementはユーザーのデバイスのハードディスクの状態をシステムファイルレベルで監視できません。インストールスクリプトはサイトのすべてのデバイスに展開され、スクリプト（19〜24行目）はドキュメントを含むフォルダーが存在するかどうかを確認しますいや





フォルダーが存在しない場合、フォルダーが作成され（28行目）、その後ドキュメントが移動され（30-32行目）、メッセージは標準の「出力」（37行目）で送信されます。







ステップ2.ソフトウェアインストール用のコンポーネントを作成する



いつものように、このステップでは、スクリプトを指定してこれら3つのWord文書をバインドする必要があるアプリケーションコンポーネントを作成する必要があります。







Post-Conditionsブロックでは、管理コンソールが警告として解釈するテキスト文字列を指定できます。







この例は、標準出力（ Resource：stdout ）に（ Qualifier：が見つかった ） が含まれている場合、 デプロイに失敗したスクリプトの実行結果が警告として表示されることを示しています。



手順3.タスクを実行して、必要なデバイスのエージェントにソフトウェアパッケージを展開します



必要なサイト（プロジェクト）の管理コンソールで、これら3つのWord文書を埋め込むデバイスのリストからデバイスを選択し、アクションバーでクイックジョブまたはジョブアイコンをクリックして、対応するタスクを作成します。







異なるサイト（プロジェクト）のすべてのコンピューターにドキュメントを埋め込む必要がある場合は、Panda Systems Management管理コンソールでアカウントのレベルでこれらのサイトを選択できます。



手順4.結果を確認して、考えられる問題を特定します。



この例では、スクリプトは3つの出力条件を定義します。



• 成功 ：ファイルはエラーなしで目的のフォルダーにコピーされます（30〜32行目）。 ErrorLevel0で終了します（38行目）。







• エラー ：ファイルのコピー中にエラーが発生しました。 ErrorLevel1で終了します（35行目）。



• 成功-警告 ：フォルダーは既に存在するため、ファイルはコピーされませんでした。 ErrorLevel0（23行目）で終了し、Deploy successful行が生成されます。これは、管理コンソールが警告として解釈します。これは、手順3のPost-Conditionsブロックで構成されました。







タスクが起動されると、[ スケジュールされたジョブ]-> [アクティブなジョブ]セクションに表示されます。





[ 完了したジョブ ]タブで、タスクの結果を確認できます。結果にエラーが含まれる場合は赤、警告がある場合はオレンジ、タスクが正常に完了した場合は緑でマークされます。







StdoutおよびStderrorアイコンには、スクリプトによって生成された標準出力と標準エラーのコピーが表示されます。



さらに、このタブには、さまざまなアクションを起動できるアイコンバーが含まれています 。



• アクションアイコンセットは、タスクの再起動、ページのリロードによるタスクのステータスの更新、標準出力またはエラーのファイルへのダウンロードを可能にするアイコンをグループ化します。



• ビューフィルターを使用すると、ステータスでタスクをフィルターできます。



例4.スクリプト言語を使用せずにドキュメントとファイルを埋め込む



事前確認が不要な場合、または管理コンソールで警告を生成する必要がない場合、インストールスクリプトを大幅に簡素化できます。



この例では、前の例で使用された3つのドキュメントを埋め込む必要がありますが、この例では、スクリプト自体からフォルダー構造を作成する代わりに、アーカイブされたドキュメントと必要なフォルダー構造を含む自己解凍型EXEパッケージが作成されます。 このEXEパッケージは、WinRarなどのさまざまなユーティリティを使用して作成できます。



WinRarの無料版をダウンロードするには、 ウェブサイトにアクセスしてください



この例では、次の特性を持つ自己解凍型EXEファイルが作成されました。



•ステルスモードで動作します（サイレント）。

•必要なコンテンツフォルダーはC：\に自動的に作成されます。

•フォルダーが既に存在する場合、その内容は警告なしに上書きされます。



ステルスモードで動作する自己解凍型ファイルを作成することが重要です。 ダイアログボックスは表示されず、ユーザーの介入は不要です 。



念のため、WinRarを使用して自己解凍形式のインストールファイルを作成する手順を見てみましょう。



手順1：埋め込むドキュメントを含むフォルダーを準備します。



ACME Documentsのルートフォルダーを作成します。そこに、埋め込みたいすべてのファイル、フォルダー、およびサブフォルダーを配置します。



ステップ2：実行可能ファイルを生成します。

リンク帯域幅の最適化



各デバイスにインストールされているローカルエージェントは、60秒ごとにPanda Systems Managementサーバーで利用可能なダウンロードと更新を確認します。 ローカルエージェント用にダウンロードまたはダウンロードするものがある場合、Panda Systems Managementはエージェントごとにこのダウンロードを個別に起動します。 この場合、インストールパッケージのサイズが50 MBで、ネットワークが50台のデバイスで構成されている場合、ダウンロード中の総トラフィックは2.5 GBになります。







ダウンロードしたトラフィックの合計サイズを減らすために、ネットワークデバイスの1つをリポジトリ/キャッシュとして機能させることができます。 この場合、このデバイスのみがサーバーからパッケージをダウンロードし、必要なすべてのネットワークデバイスに配布します。







選択したリポジトリ/キャッシュの役割を選択したデバイスに割り当てるには、Panda Systems Management管理コンソールでこのデバイスを見つけて、アクションバーのアクションパネルで[ ローカルキャッシュとして追加/削除 ]アイコンをクリックする必要があります。







その結果、選択したデバイスはローカルネットワーク上のデバイスにコンポーネントとインストールパッケージをダウンロードして配布します。これにより、リモートソフトウェアのインストールプロセスが大幅に高速化され、通信チャネルの帯域幅の負荷が大幅に削減されます。
