新機能

  • コア解析HTMLファイルを作り直しました。 現在、すべての機能は必要な場合にのみ適切な回数だけ呼び出されます（したがって、速度とメモリ消費が改善されます）。 さらに、コメントの削除がHTMLの「ストリッピング」に追加されました（すべてのシステムで安全ではないため、デフォルトでは無効になっています）。
  • head



    <style type = "text / css">で定義されたスタイルも同様に結合されます（オプション「外部ファイルと埋め込みコードの結合」）。
  • 設定「変更日をチェックしない」は修正されています。 現在、デフォルト（キャッシュ内のファイルの存在）では、インデックスはファイル名のみに基づいて構築され、ファイルシステムへの複数のアクセスを排除します（1〜2個のキャッシュファイルの存在のみがチェックされます）。
  • CSSスプライトの別の拡張パッケージ：
    • 多数の「マルチサイズ」画像で動作するように画像の並べ替え順序を変更しました。
    • 現在、スタイルで定義されたpadding



      が考慮されます。
    • IE6をスプライト作成プロセスから除外する設定を追加しました。
    • スタイルの優先順位に関する問題を修正しました。 今、背景の不足は解決されました!important



      です。
    • フルカラー画像の処理が高速化されました（現在、すべてがデフォルトで32ビットパレットで作成されています）。
    • 作成中の画像は、拡張子だけでなく、許容される形式によっても追加でチェックされます。
    • 大きなスプライトを作成する際の制限を追加しました（16または32 MBのRAMを搭載したシステムで動作するようになりました）。
    • smush.it APIを使用した自動画像最適化を追加しました。
  • 複数のホストのサポートを追加して、イメージのロードなどを並列化しました。 DNSおよびサーバー構成に適切なエントリが必要ですが、これらのホストからダウンロードする機能は自動的にチェックされます（デフォルトでは、 img

  • デフォルトでは、 .htaccess

  • mamytaからHE-localization（ヘブライ語）を追加しました。
  • 作成されたHTMLの最も単純なキャッシングを追加しました（キャッシュ時間が設定されています）。 この機能を持たない完全に「弱い」サーバーとCMSに役立ちます。 サーバーダイナミクス（たとえば、コメントの公開）が存在する場合、このアプローチは適用できない可能性があることを理解する必要があります。 キャッシュロジックから除外するURLパーツのリストも追加されます（サイトの一部をすぐにキャッシュしない場合があります）。
  • 管理パネルから直接キャッシュのフラッシュを追加しました。
  • これで、すべてのファイルが絶対パスを介して接続されます。 これは、フレームワークのアプリケーションの使用を設定するのに役立ちました。

今後の計画



次号（0.5）はセキュリティに専念します。 Web Optimizerのインストールをその作業から完全に分離することを計画しています（同時に.htaccess



使用してWebオプティマイザーに密接にアクセスします）。 当然、すべての注目されるエラー、新しいCMSおよびフレームワークの一部、およびいくつかの新しい機能が含まれます。 クライアントの最適化を自動化するための計画はほぼ完了しているため、現在のバージョンは最終版に非常に近いと見なすことができます。

Webオプティマイザーを使用する



以前、現在のサイト公開スキームまたは「スケジュールされた最適化」でアプリケーションを実装する方法についての会話がありました。 インストールは「チェーン最適化」を使用するため、任意のページのイベントまたはスケジュールでセットアップして実行するのに問題はありません。 詳細はプライベートで指定できます。

この段階で、Web Optimizerは、システムの自動最適化のAPIとして安全に使用できます。 ライセンスはMITなので、コメントをPHPコードで保存するだけで十分です。
