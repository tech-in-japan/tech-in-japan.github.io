注意の速度への変換。 最初のステップは、注意（0-100）と速度（1.0-通常の再生速度）を関連付ける方法を見つけることでした。 実験的に、0.5（低速再生）から2.0（高速）の速度範囲に制限することにしました。 簡単なマッピングの作成： 



 var playRates = [0.5, 0.7, 0.8, 0.9, 1.0, 1.1, 1.2, 1.4, 1.6, 1.8, 2.0]; function attention2speed(attention) { var speed = 0; if (attention > 30) { speed = playRates[Math.floor((attention - 30) / 7)]; } else { speed = 0.5; } return speed; }


すべてのパラメーターは実験的であり、背後には明確な科学的要素はありませんが、特定のロジックはあります。 主なアイデア：低い注意で低速を設定し、次に個別のディスプレイを作成します。







次のステップ：再生速度を変更します。 残念ながら、額を作る、つまり、新しい情報が到着したときに再生速度を変更しようとする試みは、惨めに失敗しました。 まず、アテンション値がダウンロードされます。つまり、平均化する必要があります。次に、あまりにも頻繁に更新すると（たとえば、各データまたはすべてのフレームで）、ビデオの再生速度が「遅れ」始め、率直に聞くことができなくなります。



したがって、実験的にも個人的な好みからも、私は自分自身を単純なスキームに限定しました。
