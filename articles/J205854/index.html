フライデーナイト13日更新：ReSharperおよびその他の.NET製品

いいえ、この投稿はリシャーパーでのC ++サポートに関するものではありません。 これは後でです。



それまでの間、.NETツールのほぼすべてのラインを試し、更新しました。 これで、それらを取得してインストールできます。

  • ReSharper 8.1 （Visual Studioのほぼすべてを改善）
  • dotCover 2.6 （テストを実行し、コードカバレッジを測定します）
  • dotTrace Performance 5.5.3 （プロファイルテストおよびその他の.NETアセンブリ）








ReSharper SDK （ ReSharperの拡張機能を作成するため）、 dotTrace SDK （任意のアプリケーションにプロファイルを埋め込むため）、 ReSharperコマンドラインツール （コマンドラインから.NETコードを分析するため）をフォローするように更新されました。



新しいReSharperは、次の理由で（金曜日の夜であっても）興味があるかもしれません。



  • JavaScriptにうんざりしているが、それなしでは生きられない場合、 ReSharper 8.1はTypeScriptをサポートしていることに注意してください。 私たちは8つの夏の最初のリリースで彼をサポートするつもりでしたが、予測できない多くの予測できない問題を経験したため、今すぐ配信に含めています。 TypeScriptコードには、リファクタリング（名前の変更、変数の導入）、ナビゲーション、IntelliSense、および検査があります。たとえば、ReSharperは未使用のパラメーターと無効なCSSリンクを認識します。 TypeScriptサポートの詳細 Dima Nesterukは最近、英語のブログに書いています。

  • JavaScriptとの関係がうまく発展した場合、良いニュースがあります。JavaScriptのサポートも変更されました。 まず、この言語のコードの構造検索と置換（SSR）、およびJSキャラクター間の名前変更とナビゲーションに関する多くの改善について話します。
  • Visual Studio 2013でのReSharper 8.1の統合は 、より良く、より広くなりました。 まず、VS2013 RTMビルドでの飛行は正常であると改めて確信しました（以前のReSharper 8.xリリースは、スタジオのプレリリースビルドでテストされました）。 次に、Peek Definitionに統合し、そこでReSharperコンテキストコマンドを使用できるようにしました。 第三に、彼らは新しく改良された拡張スクロールバーとの関係を確立し、ReSharperが表示するコードおよびファイル全体の問題のインジケータを、すべての形式でスクロールバーと平和的に共存させました。 これについては、ここで詳しく説明します。



    Visual Studio 2012、2010、2008、2005のサポートは、いつものようにどこにも行きません。 次のバージョンでは2005年に統合を中止することを考えていますが、これは別の歌です。

  • Cのコード検査を追加して 、 foreach



    ループで無限for



    ループとInvalidCastException



    などの可能性のある例外を検出しました。



  • プロジェクト間の依存関係を表示するためのツールは、最初のリリース後に更新の最初のバッチを受け取りました。 その中には、プロジェクト間の間接的な依存関係の分析（可能な関係）、修正されたUI（hello、元に戻す/やり直し）、新しいコンテキストコマンド、パフォーマンスの向上があります。 これらの変更の詳細についてはこちらをご覧ください 。



  • XAMLを扱う開発者は最近、良いニュースに慣れてきました。私たちは彼らのためにもう1つ準備しました。 ReSharper 8.1は、Windows 8.1アプリケーションのサポート、デュアルIntelliSense、コード挿入時のタイプインポート、およびその他の多くの革新を導入しました。
  • バグ修正 （400以上）、パフォーマンスの向上（70の修正）、メモリ消費の調整（たとえば、[ 結果の検索]ウィンドウでシンボルへのリンクを表示し、[ 検査結果]ウィンドウで問題を発見した場合）に取り組みました。




dotCover 2.6およびdotTrace Performance 5.5.3に関しては 、これらの更新も注目に値します。その理由は次のとおりです。

  • リゾルバーと一緒にdotCoverとdotTrace Performanceを使用することに慣れている場合、これらが必要になります。両方の新しいバージョンはReSharper 8.1と互換性があります。
  • Visual Studio 2013で動作します。 Visual Studioの以前のバージョンとの統合も残ります。
  • dotCover 2.6にはReSharperと同様のExtension Managerが追加装備されています。dotCoverは拡張機能の広範なセットを誇っていませんが、プラグインとして提供されるxUnitおよびMSpecのサポートは更新が容易です。




したがって、上記のすべては、 ReSharper 8.1 、 dotCover 2.6 、 dotTrace Performance 5.5.3などの既知のアドレスでダウンロードできます。
