インメモリコンピューティングサミット2017サンフランシスコ

少し前の10月末に、別のインメモリコンピューティングサミットがシリコンバレーで開催されました。その主催者およびスポンサーの1つはGridGainでした。



ビデオ資料とスライドに無料でアクセスできるようになりました。これらは、 メインの要約とレポートの 2つのセクションに分かれています 。







分散システムおよびインメモリコンピューティングに関する40を超えるレポートがサミットで読まれました。 スピーカーの中には、GridGain、Percona、AMY Robotics、Hazelcast、VoltDB Inc.、Dellなどの従業員がいました。



次に、選択したいくつかのレポートについて個別に言及します。



阿部クラインフェルド（GridGain CEO）基調講演









Apache Ignite / GridGainを使用した投資簿（IBOR）の実装（Rafique Awan、Wellington Management）



Wellington Managementの担当者は、投資ビジネスにおけるApache Igniteの適用可能性と、IBORシステムを実装する際に生じる可能性のある困難について話します。







スケールアウトおよび征服：分散インメモリシステムのアーキテクチャ上の決定（Yakov Zhdanov、GridGain）

参照



Apache Igniteなどの分散プラットフォームは、水平方向のスケーラビリティを広範囲に使用します。 クラスタ内のマシンが多いほど、アプリケーションのパフォーマンスが向上します。 クラスタに2台目のマシンを追加すると、パフォーマンスは2倍になりますか？ クラスターが10倍になると、10倍に増えますか？ これは常にそうですか？ プラットフォームの責任は何ですか？ 彼女の制限は何ですか？ そして、プラットフォームは何を終わらせることができ、それを実装する資格のあるエンジニアの責任の領域を開始しますか？







NVMプログラミングモデルとNVDIMMを使用した永続メモリの利点の実現（Jim Pappas、SNIA）

参照



メモリとストレージの組み合わせへのパスは、NVM、適切なプログラミングモデル、およびNVDIMMの組み合わせです。 ランダムアクセスと非揮発性を備えた高速ストレージを持つ機能は、多くの視点を開きます。 レポートでは、NVM、NVDIMMプログラミングモデル、および対応する機器が提供する速度の利点について説明しています。







Javaでのインメモリメモリコンピューティング（Eric Kaczmarek、Intel）

参照



不揮発性ランダムアクセスメモリにより、開発者はマルチテラバイトアレイでデータ操作を実行できます。 このレポートは、不揮発性メモリの特性と、Javaを使用してそのようなメモリを使用したい開発者が考慮すべき機能に特化しています。 ネイティブLinuxおよびWindows API、およびJavaコレクションを適応させるための提案が検討されています。







大規模金融機関での耐久性のあるメモリ中心アーキテクチャの実装（Dmitry Setrakyan、GridGain）

参照



レポートでは、メモリ内の分散コンピューティングシステムを金融機関でアーキテクチャ的に使用する方法を説明しています。 Dmitryが、ACID、SQLサポート、永続ストレージ、レプリケーション、セキュリティツール、フォールトトレランスなどの関連するApache Ignite機能について説明します。 また、Sberbankで遭遇した5つの主要な困難を考慮した、Sberbankで最大規模のこのような設備の1つについても説明しています。
