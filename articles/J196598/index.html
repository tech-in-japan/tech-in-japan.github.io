Raspberry Piの監視またはタイムラプスビデオ



最近、私はビデオ監視のいくつかの外観を構築する必要がありました。 要件は非常に簡単でした：

  • オンラインで視聴する機能
  • リアルタイムモードは必要ありません。夕方の日の主要なイベントを見るだけで十分です（たとえば、仕事中に誰かがお気に入りのコンピューターに座っているかどうかを調べます）
  • 結果を確認するのに何時間も費やす必要はありません
  • 最高の画質
  • 最低費用


基準に基づいて、IPカメラの形で既製のソリューションを検討することができます。 ただし、市場をすばやく分析したところ、最大100ドルのデバイスでは、画像サイズが640x480ピクセルを超えることはめったになく、同時にソフトウェアをセットアップする良い機会を自慢することはできませんでした。 つまり、このようなカメラを購入した場合、工場のファームウェアの曲率と計画を完全に実現できないことに我慢しなければならない可能性が十分にあります。



既製のソリューションとは対照的に、Raspberry Piでの自家製の作業は、オンボードの本格的なOSとあらゆる好みのテラバイトの既製ソフトウェアのおかげで、非常に安価で、はるかに刺激的で、はるかに柔軟です。



上記の問題を解決するために、次の作業アルゴリズムが発明されました。



1日の初め（スケジュール）に、システムでプロセスが開始され、一定の間隔（たとえば、1分に1回）で写真を撮影し、それらをメモリカードにローカルに保存し、すぐにWebDAVプロトコルを使用してクラウドディスクにアップロードします。 これは、「ライブ」観察のいくつかの外観を提供します。これは、条件によって必要ではありませんが、楽しいボーナスです。 1日の終わり（スケジュールどおり）に、撮影プロセスが中断され、写真からのビデオファイルのアセンブリが開始されます。 アセンブリの最後に、ビデオは同じクラウドディスクにアップロードされるため、写真ファイルを切り替える必要なく、avi-flip全体をすばやくリモートで表示できます（1時間あたり1フレームあたり10時間の作業で、合計ビデオタイミングが得られます-約1分10 fps）。



もちろん、このアプローチでは、ビデオ監視は通常の形式では実装されていませんが、すべてのタスクは解決されています。 さらに、カメラに直接接続するためにプロバイダーから専用IPを購入する必要はありません。また、ストレージとデータアクセス機能は無料サービス（ Yandex.Diskなど ）に転送されるため、別のビデオ監視サーバーを起動する必要はありません。



だから、鉄：


  • Raspberry Pi（この地域では50〜60ドル）：モデルB、イーサネット、2つのUSB、256 MBのRAMを搭載したモデルを使用しました
  • Logitech C270（$ 20）：UVC仕様を満たす比較的安価で人気のあるHD（1280x720）Webカメラ（その結果、Linuxシステムで問題なく動作します）
  • iPhoneからの充電（$ 10）：実際、iPhoneはもちろん、それとは何の関係もありません。1Aの出力電流とmicro-USBコネクタを備えた電源であれば十分です。 中国のパルス充電は1個あたり2ドル以下ですが、注意深く確認する必要があります。
  • 4 GB SDカード（6ドル）：4つのギグ-快適な作業のための最小カードサイズ。
  • ラズベリーのケース（$ 1）：多かれ少なかれ適した箱からのドレメルを使用して作られ、建設のハイパーマーケットで購入されました。

ウェブカメラから写真をキャプチャするには、mjpg-streamerを使用することを強くお勧めします。 このユーティリティ-この鉄の構成でテストされた唯一のユーティリティは、「正しい」解像度で写真を発行することができました。 私がたまたま経験したその他のツール（モーション、ffmpeg、ストリーマー）は、HD画像を提供せず、せいぜい640x480しか提供しなかった、または開始を拒否しました。 残念ながら、mjpg-streamerは実行可能ファイルとして配布されていませんが、既製のパッケージをインストールするよりもコンパイルは難しくありません。 行うこと（すべての操作はユーザーpiの代わりに実行されることを前提としています）。



まず、インストール可能なパッケージのローカルインデックスを更新します。 



 sudo apt-get update




必要に応じて、システムにすでにインストールされているパッケージのアップグレードを開始することもできますが、実際にはすべてがパッケージなしで機能します。 



 sudo apt-get upgrade




写真の作成日時を表示するには、変換ユーティリティを含むimagemagickパッケージが必要です。 画像を変更するための真に無限の可能性があるので、助けを借りて数十の数字を書くことは完全に簡単です。 mjpg-streamerをコンパイルするには同じパッケージが必要なので、ためらうことなく配置しました。 



 sudo apt-get install imagemagick




aviconvプログラムは、一連の写真からビデオを接着することに関与します。これは、独自にインストールするのではなく、libav-toolsパッケージを使用して、mjpg-streamerのコンパイルが成功しないようにします。 私たちは置きます： 



 sudo apt-get install libav-tools




また、mjpp-streamerに必要なライブラリを含むlibjpeg8-devパッケージも必要です。 



 sudo apt-get install libjpeg8-dev




sourceforgeからソースをダウンロードするには、subversionが必要です。 



 sudo apt-get install subversion




そして、mjpg-streamer自体のダウンロードとコンパイルは次のように行われます。 



 sudo svn co https://svn.code.sf.net/p/mjpg-streamer/code/mjpg-streamer/ mjpg-streamer cd mjpg-streamer make




最後のステップは、WebDAV経由でリモートファイルシステムを操作するためのパッケージをインストールし、実際にマウントすることです。 



 sudo apt-get install davfs2 sudo mkdir /mnt/dav sudo mount -t davfs https://webdav.yandex.ru /mnt/dav -o uid=pi,gid=pi


インストールプロセス中に、ユーザー名とパスワードを入力する必要があります。



ワークフローの手順：


これで、今後の作業に必要なすべてが揃いました。 主なコマンドについて簡単に説明しますが、記事を過度に書かないように、特にbashに頼ってはいません。 個々の操作を実行する自動スクリプトの説明が、別のエッセイの理由かもしれないと思います。



したがって、mjpg-streamerを実行します。 



 cd mjpg-streamer ./mjpg_streamer -i "./input_uvc.so -r 1280x720 -f 1" -o "./output_file.so -f ./ -d 60000"


このモードでは、現在のディレクトリ（-f ./）に1分ごと（-d 60000）に、2013_10_04_12_11_30_picture_000000000.jpgのような美しい名前のファイルが作成されます。



画像にタイムスタンプを描画するには、次のようなスクリプトを使用します。 



 filename="2013_10_04_12_11_30_picture_000000000.jpg" timestamp=`stat -c %y $filename` convert $filename -fill black -draw "rectangle 1130,695 1270,715" -fill white -pointsize 15 -draw "text 1135,710 '${timestamp:0:19}'" ./out.jpg.


彼は、画像の右下隅にマークを付けて、結果をout.jpgに書き込みます。 次のようになります。







Yandex.Diskに書き込むには、目的のファイルを/ mnt / davにコピーするだけです。 



 cp out.jpg /mnt/dav




リモートファイルシステム自体へのコピーは2段階で実行されることに注意してください：最初に、ファイルは/ var / cache / davfs2のどこかにあるローカルキャッシュにコピーされ、その後、davfsは既にネットワーク経由でファイルを実際に転送します。 これは、ファイルがYandex.Diskで物理的に使用可能になる前に、コピーコマンドが完了することを意味します。 したがって、Yandexでコピーしたばかりのファイルが見つからない場合は、落胆しないでください。おそらく少し待つ必要があります。



jpeg-sのセットからビデオを作成する準備をする必要があります。ファイル名は、特定のパターンに従う一連の数字を増やす必要があります。 次に、現在のディレクトリにあるすべてのjpgファイルの名前を変更時刻の昇順で変更し、aviへの変換を開始するスクリプトを示します。 



 i=0 for f in `ls -tr *.jpg 2>/dev/null` do newf=`printf %06d $i`.jpg echo $f "-->" $newf mv $f $newf i=$((i+1)) done avconv -r 10 -i %06d.jpg -r 10 -vcodec mjpeg -qscale 1 out.avi




avconvは非常に強力なユーティリティであり、ほとんど何でも何百ものパラメーターを持つものにエンコードできることを言わなければなりません。 ただし、Raspberryでビデオをエンコード/デコードするソフトウェアの控えめな機能を検討する価値はあります。 たとえば、約600ファイルのセットをH.264で圧縮されたムービーに変換するのに約2時間かかりました。 したがって、上記の例では、MJPEGのコーディングを使用します。これは基本的に何も圧縮せず、互いにjpgのみを接着します。 このような変換は数分で実行され、写真の元の品質が失われることはありません。



さらに、bashスクリプトを開発するためのいくつかの器用さとスキルを使用して、スケジュールに従ってオン/オフを切り替える完全な自動システムを設定し、ゴミを削除し、リモートディスクの空き領域の可用性を監視することができます。 また、ディスクを介して、たとえばMalinka自体が取得して実行する特定のディレクトリにスクリプトをレイアウトするなど、Raspberryのリモートコントロールを整理できます。
