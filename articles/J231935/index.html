Minecraftサーバーを作成および構成する





現在、Minecraftは最も人気のあるオンラインゲームの1つです。 3年足らず（2011年秋に最初の公式リリースが行われた）で、彼は世界中で何百万人ものファンを獲得しました。



ゲーム開発者は、多くのゲームがグラフィックスの点で原始的であり、今日の標準では使いやすさの点で不完全であった20年前の最良の例に意識的に導かれていますが、同時に本当に刺激的でした。



すべてのサンドボックスゲームと同様に、Minecraftはユーザーに創造性を発揮する絶好の機会を提供します。これが実際、人気の秘密です。



マルチプレイヤーモードでのゲームのサーバーの編成は、プレイヤー自身とそのコミュニティによって行われます。 今日、何万ものゲームサーバーがインターネット上で動作しています（たとえば、このリストを参照）。



お客様にはこのゲームのファンが多く、彼らはデータセンターでゲームプロジェクト用の機器をレンタルしています。 この記事では、サーバーを選択する際に注意する必要がある技術的側面について説明します

マインクラフト



プラットフォームを選択してください



Minecraftには、次のアーキテクチャ要素が含まれています。

  1. サーバー-プレーヤーがネットワークを介して互いにやり取りするプログラム。
  2. クライアント-プレーヤーのコンピューターにインストールされているサーバーに接続するためのプログラム。
  3. プラグイン-サーバーへのアドオン、新しい機能の追加、または古い機能の拡張。
  4. mods-ゲームの世界への追加（新しいブロック、オブジェクト、機会）。




Minecraftには多くのサーバープラットフォームがあります。 最も一般的で人気のあるのは、バニラとブキットです。



バニラは、ゲーム開発者の公式プラットフォームです。 グラフィカルバージョンとコンソールバージョンの両方で配布。 Vanillaの新しいバージョンは、Minecraftの新しいバージョンと同時にリリースされます。



バニラの欠点は、過剰なメモリ消費量です（プレーヤーあたり約50 MB）。 もう1つの大きな欠点は、プラグインがないことです。



Bukkitは、公式のMinecraftサーバーの改善を試みた愛好家のグループによって作成されました。 この試みは非常に成功しました。Bukkitは、主にさまざまなMODとプラグインのサポートにより、Vanillaよりも機能がはるかに広いです。 同時に、各プレーヤーのメモリ消費量は約5〜10 MB少なくなります。



Bukkitの欠点は、起動時にRAMを大量に消費することです。 さらに、サーバーの実行時間が長いほど、必要なメモリが多くなります（プレーヤーが少ない場合でも）。 Bukkitをサーバーとして選択する場合、その新しいバージョンには原則としてエラーが含まれていることに注意してください。 安定版は通常、Minecraftの公式リリースの約2〜3週間後に表示されます。



さらに、他のプラットフォーム（Spout、MCPC、MCPC +など）が最近人気を集めていますが、VanillaおよびBukkitとの互換性が制限されており、MODのサポートが非常に限られていることが特徴です（たとえば、Spoutの場合、スクラッチからしかMODを作成できません）。 それらが使用される場合、それは実験のためだけです。



ゲームサーバーを整理するには、Bukkitプラットフォームを使用することをお勧めします。これは最も柔軟性が高いためです。 さらに、その下には多くの異なるmodとプラグインがあります。 Minecraftサーバーの安定した動作は、ハードウェアプラットフォームの正しい選択に大きく依存します。 この質問をさらに詳しく考えてみましょう。



ハードウェア要件



サーバーとMinecraftクライアントの両方がシステムリソースを非常に要求しています。

ハードウェアプラットフォームを選択する場合、マルチコアプロセッサは大きな利点をもたらさないことに注意してください。Minecraftサーバーコアは1つの計算ストリームしか使用できません。 ただし、2番目のコアは便利です。一部のプラグインは個別のスレッドで実行され、Javaも多くのリソースを消費します...



したがって、Minecraftサーバーの場合は、シングルコアのパフォーマンスが高いプロセッサを選択することをお勧めします。 マルチコアよりも強力なデュアルコアプロセッサの方が適していますが、性能は劣ります。 専門のフォーラムでは、少なくとも3 GHzのクロック周波数のプロセッサーを使用することをお勧めします。



Minecraftが正常に機能するには、大量のRAMが必要です。 Bukkitは約1GBのRAMを占有します。 また、前述のように、各プレーヤーには5〜10 MBが割り当てられます。 プラグインとMODもかなりのメモリを消費します。 30〜50プレーヤーのサーバーの場合、これには少なくとも4 GBのRAMが必要です。



Minecraftでは、多くの（たとえば、同じプラグインのダウンロード）がファイルシステムの速度に依存します。 したがって、SSDドライブを備えたサーバーを選択することをお勧めします。 ランダムな読み取り速度が遅いため、スピンドルドライブは動作しません。



非常に重要なのは、インターネット接続の速度です。 40〜50人のゲームでは、10 Mb / sのチャンネルで十分です。 ただし、Webサイト、フォーラム、ダイナミックマップなどの大規模なMinecraftプロジェクトを計画している場合は、より大きな帯域幅のチャネルを作成することが非常に望ましいです。



どの特定の構成を選択するのが最適ですか？ 提供する構成のうち、次の点に注意することをお勧めします。

  • Intel Core 2 Duo E8400 3GHz、6GB RAM、2×500GB SATA、3000ルーブル/月;
  • Intel Core 2 Quad Q8300 2.5GHz、6GB RAM、2×500GB SATA、3500摩擦/月 -MineCraftテストサーバーにこの構成を使用します。このサーバーでは、今すぐプレイできます（これを行う方法については、以下で説明します）。
  • Intel Core i3-2120 3.3 GHz、8 GB RAM、2×500 GB SATA、3500 rub /月




これらの構成は、30〜40人のプレーヤー用のMinecraftサーバーを作成するのに非常に適しています。 いくつかの欠点はSSDドライブがないことですが、もう1つの重要な利点があります：制限と比率のない100 Mb / sの保証されたチャネルです。 上記のすべての構成を注文する場合、インストールの支払いは請求されません。



また、生産性は向上しますが、もちろん、より高価なサーバーもあります（これらの構成を注文すると、インストールの支払いも請求されません）。

  • 2x Intel Xeon 5130、2GHz、8GB RAM、4×160GB SATA、5000ルーブル/月;
  • 2x IntelXeon 5504、2GHz、12GB RAM、3×1TB SATA、9000 rub /月


また、Intel Atom C2758プロセッサーに基づくSSDドライブを搭載した新しい予算モデルに注意することをお勧めします：Intel Atom C2758 2.4 GHz、16 GB RAM、2×240 GB SSD、4000ルーブル/月、インストール支払い-3000ルーブル。



Ubuntu OCでのBukkitサーバーのインストールと起動



サーバーをインストールする前に、新しいユーザーを作成し、sudoグループに追加します。 

 $ sudo useradd -m -s / bin / bash <ユーザー名>
 $ sudo adduser <ユーザー名> sudo


次に、作成されたユーザーがサーバーに接続するためのパスワードを設定します。 

 $ sudo passwd <ユーザー名>


新しいアカウントでサーバーに再接続し、インストールを続行します。

MinecraftはJavaで記述されているため、Java Runtime Environmentをサーバーにインストールする必要があります。



利用可能なパッケージのリストを更新します。 

 $ sudo apt-get update


次に、次のコマンドを実行します。 

 $ sudo apt-get install default-jdk


Bukkitをインストールして実行するには、端末マルチプレクサをインストールすることをお勧めします。たとえば、screen（他の端末マルチプレクサを使用できます- レビューを参照）： 

 $ sudo apt-getインストール画面


sshを介してゲームサーバーに接続する場合、画面が必要です。 これにより、Minecraftサーバーを別のターミナルウィンドウで起動でき、sshクライアントを閉じた後でもサーバーは動作します。



サーバーファイルを格納するディレクトリを作成します。 

 $ mkdir bukkit
 $ cd bukkit


その後、 Bukkitの公式Webサイトのダウンロードページに移動します。 ページの右上部分に、サーバーの最新の推奨ビルド（推奨ビルド）へのリンクがあります。 ダウンロードすることをお勧めします。 

 $ wget <推奨バージョンのリンク>


今画面を実行します： 

 $ sudo画面


次のコマンドを実行します。 

 $ java -Xmx1024M -jar craftbukkit.jar -o false


使用されるパラメーターの意味を説明しましょう。

  • Xmx1024M-サーバーごとのRAMの最大量。
  • jar craftbukkit.jar-サーバーへのキー。
  • o false-海賊版クライアントからサーバーへのアクセスを許可します。




サーバーが起動します。

コンソールでstopコマンドを入力して、サーバーを停止できます。



サーバーのセットアップと構成



サーバー設定は、server.properties構成ファイルに保存されます。 以下のパラメーターが含まれています。

  • generator-settings-スーパーフラットな世界を生成するためのパターンを設定します;
  • allow-nether-下位世界への移行の可能性を定義します。 デフォルトでは、このパラメーターはtrueに設定されています。 falseに設定されている場合、Lower Worldのすべてのプレイヤーが通常に移動します。
  • level-name-ゲーム中に使用されるマップファイルのあるフォルダーの名前。 このフォルダーは、サーバーファイルと同じディレクトリにあります。 そのようなディレクトリが存在しない場合、サーバーは新しいワールドを自動的に作成し、同じ名前のディレクトリに配置します。
  • enable-query-trueに設定されている場合、GameSpy4プロトコルをアクティブにしてサーバーでリッスンします。
  • allow-flight-Minecraftの世界中のフライトを許可します。 デフォルト値はfalseです（フライトは禁止されています）。
  • server-port-ゲームサーバーが使用するポートを示します。 Minecraftのデフォルトポートは25565です。このパラメーターの値は推奨されません。
  • level-type-世界のタイプを定義します（DEFAUT / FLAT / LARGEBIOMES）。
  • enable-rcon-サーバーコンソールへのリモートアクセスを開きます。 デフォルトでは無効になっています（false）。
  • level-seed-レベルジェネレーターの入力。 ランダムな世界を作成できるようにするには、このフィールドを空白のままにする必要があります。
  • force-gamemode-サーバーに接続するプレーヤーの標準ゲームモードを設定します。
  • server-ip-プレーヤーがサーバーへの接続に使用するIPアドレスを示します。
  • max-build-height-サーバーの最大ビルド高さを示します。 その値は16の倍数（64、96、256など）でなければなりません。
  • spawn-npcs-村でのNPCの出現を許可（trueに設定した場合）または禁止（falseに設定した場合）;
  • ホワイトリスト-サーバー上のプレーヤーのホワイトリストの使用を有効または無効にします。 trueに設定されている場合、管理者はプレーヤーのニックネームを手動で追加してホワイトリストを作成できます。 falseに設定すると、IPアドレスとポートを知っているユーザーはサーバーにアクセスできます。
  • spawn-animals-trueに設定されている場合、友好的なMobを表示できます;
  • snooper-enabled-サーバーが統計とデータを開発者に送信できるようにします。
  • hardcore-サーバーでハードコアモードをオンにします。
  • texture-pac-プレーヤーをサーバーに接続するときに使用されるテクスチャファイル。 このパラメーターの値は、テクスチャーを含むzipアーカイブの名前で、サーバーと同じディレクトリに保存されます。
  • オンラインモード-サーバーに接続しているユーザーのプレミアムアカウントのチェックが含まれます。 このパラメーターがtrueに設定されているが、プレミアムアカウントの所有者のみがサーバーにアクセスできる場合。 アカウントの検証が無効（falseに設定）になっている場合、ユーザー（ニックネームを偽造したプレイヤーを含む）はサーバーにログインでき、セキュリティリスクが追加されます。 スキャンが無効になっている場合、インターネットにアクセスせずにローカルネットワークでMinecraftをプレイできます。
  • pvp-プレイヤーが互いに戦うことを許可または禁止します。 このパラメーターがtrueの場合、プレイヤーはお互いを破壊できます。 falseに設定されている場合、プレーヤーは互いに直接的なダメージを与えることができません。
  • 難易度-ゲームの難易度を設定します。 0（最も簡単）から3（最も複雑）までの値を取ることができます。
  • gamemode-サーバーに入るプレイヤーに設定されるゲームモードを示します。 次の値を取ることができます。0-サバイバル、1-クリエイティブ、2-アドベンチャー。
  • player-idle-timeout-アイドル時間（分単位）。その後、プレーヤーはサーバーから自動的に切断されます。
  • max-players-サーバー上のプレーヤーの最大許容数（0〜999）。
  • spawn-monsters-（trueに設定されている場合）敵対的なMobの生成を許可します。
  • generate-structures-構造（財務、要塞、村）の生成を有効（true）/無効（false）にします;
  • view-distance-プレーヤーに送信するために更新されたチャンクの半径を調整します。 3〜15の値を取ることができます。




Minecraftサーバーのログは、server.logファイルに書き込まれます。 サーバーファイルと同じフォルダーに保存されます。 ログのサイズは絶えず増加しており、ますます多くのディスク容量を占有しています。 いわゆるログローテーションを使用して、ロギングメカニズムの動作を合理化できます。 ローテーションには、特別なユーティリティlogrotateが使用されます。 ログエントリの数を特定の制限に制限します。



ログファイルが特定のサイズに達するとすぐにすべてのエントリが削除されるように、ログのローテーションを構成できます。 また、古いエントリがすべて無関係と見なされて削除されるまでの期間を設定することもできます。



基本的な回転設定は/etc/logrotate.confファイルにあります。 さらに、アプリケーションごとに、個別の設定を作成できます。 個々の設定を持つファイルは/etc/logrotate.dディレクトリに保存されます。



テキストファイル/etc/logrotate.d/craftbukkitを作成し、次のパラメーターを書き込みます。 

 /home/craftbukkit/server.log {
  回転2
  毎週
  圧縮する
  行方不明
  気づかない
 }


それらの値をより詳細に検討してください。

  • rotateパラメーターは、ファイルを削除するまでの回転数を示します。
  • weeklyは、ローテーションが毎週実行されることを示します（他のパラメーターを設定できます：毎月-毎月および毎日-毎日）。
  • compressは、アーカイブされたログを圧縮することを示します（逆のオプションはnocompressです）。
  • missingokは、ログファイルがない場合、作業を続行する必要があり、エラーメッセージを表示しないことを示します。
  • notifemptyは、ログファイルが空の場合、ログファイルを移動しないように指示します。


ログローテーション設定の詳細については、 こちらをご覧ください 。



最適化のヒント



このセクションでは、ゲームサーバーの最適化のみに関するヒントを提供する予約をすぐに行います。 Minecraftがインストールされているサーバーの微調整と最適化の質問は、この記事の範囲外の別のトピックを表しています。 興味のある読者は、オンラインで必要な情報を簡単に見つけることができます。



Minecraftをプレイするときに発生する最も一般的な問題の1つは、いわゆるラグです。これは、プログラムがユーザーの入力に時間どおりに応答しない状況です。 これらは、クライアント側とサーバー側の両方の問題が原因で発生する可能性があります。 以下では、サーバー側の問題の可能性を減らすのに役立つ推奨事項を示します。



サーバーとプラグインによるメモリ消費を定期的に監視する



メモリー消費は、 LagMeterなどの専用の管理プラグインを使用して監視できます。



プラグインをお楽しみに



原則として、新しいバージョンごとに新しいプラグインの開発者は負荷を軽減しようとします。



同様の機能を持つ多くのプラグインを使用しないようにしてください。



大規模なプラグイン（Essentials、AdminCMD、CommandBookなど）には、多くの場合、多くの小さなプラグインの機能が含まれています。 たとえば、同じEssentialには、プラグインiConomy、uHome、OpenInv、VanishNoPacket、Kitの機能が含まれています。 1つの大きなプラグインの機能でその機能が完全にカバーされている小さなプラグインは、ほとんどの場合、サーバーに負荷がかからないように削除できます。



地図を制限して自分でアップロードする



カードを制限しないと、サーバーの負荷が大幅に増加します。 WorldBorderプラグインを使用してマップを制限できます。 これを行うには、このプラグインを実行して/ wb 200コマンドを実行し、/ wb fillコマンドを使用してマップを描画します。



もちろん、描画には多くの時間がかかりますが、一度だけ実行して、技術的な作業のためにサーバーを閉じた方が良いでしょう。 各プレイヤーがマップを描画する場合、サーバーの動作は遅くなります。



重いプラグインをより高速でリソース集約度の低いプラグインに置き換えます



Minecraftのすべてのプラグインが成功と呼ばれるわけではありません。多くの不要な機能や不要な機能が含まれていることが多く、場合によっては大量のメモリも消費します。 失敗したプラグインは、代替プラグインに置き換えるのが最適です（非常に多くあります）。 たとえば、LWCプラグインをWgfix + MachineGuardに、DynMapプラグインをMinecraft Overviewerに置き換えることができます。



常にドロップをクリアするか、プラグインをインストールしてドロップを自動的に削除します



ドロップインゲームは、Mobが死亡したか、ブロックが破壊されたときにドロップアウトするオブジェクトです。 ドロップの保管と処理は、多くのシステムリソースを消費します。



サーバーの動作を高速化するには、ドロップを削除することをお勧めします。 これは、NoLaggやMcCleanなどの特別なプラグインを使用して行うのが最適です。



チートを使用しないでください



いわゆるアンチチートは、多くの場合、ゲームサーバーにインストールされます。これは、不正な方法でゲームに影響を与えようとする試みをブロックするプログラムです。



Minecraftには反チートがあります。 アンチチートは、常にサーバーの追加の負荷です。 ランチャー（ただし、絶対的なセキュリティの保証を与えず、簡単に破られる-しかし、これは既に別の議論のトピックです）とクライアントに保護をインストールすることが望ましいです。



結論の代わりに



特定の例でサポートされている場合、指示や推奨事項ははるかに効果的です。 上記のインストール手順に基づいて、MineCraftサーバーを作成し、マップ上に興味深いものを配置しました。



取得したものは次のとおりです。

  • Bukkitサーバー-安定した推奨バージョン1.6.4。
  • 統計プラグイン-プレーヤーに関する統計を収集します。
  • WorldBorderプラグイン-マップを描画して制限します。
  • WorldGuardプラグイン（+依存関係としてのWorldEdit）-一部の領域を保護します。




みんなにそれをプレイするように勧めます。接続するには、新しいサーバーを追加し、アドレスmncrft.slc.tlを指定します。



コメントで、MineCraftサーバーのインストール、構成、および最適化に関するあなた自身の経験を共有し、どのMODとプラグインがあなたに興味があるのか​​、そしてその理由を教えていただければ幸いです。
