ASP.NET MVCでHistory APIを使用して1ページのアプリケーションを実装する

ハバロフスク在住の皆さん、こんにちは。 このサイトは、単一ページのajaxアプリケーションを作成する問題を繰り返し提起しています。 私はしばらく前にこの仕事に直面しました。 しかし、なぜhtml5の機能とMVCのパワーを持っているのか、 手動で、さらにはjsを使用して、非常に多くのことを書く必要があったのだろうかと思いました。



おそらく、ASP.NET MVCの機能に基づいた簡単なソリューションを作成するように促したのは、js言語に対する敵意[holywar = on] [holywar = off]でした。 次に、1ページのAjaxアプリケーションを作成しようとしたときに発生する問題について詳しく説明し、ステップごとに完全なソリューションの作成を検討します。



面白くなったら-カットしてください（コードと写真が添付されています）。



そのため、標準のMVC 3テンプレートアプリケーションを試して、下図のように、プロジェクトの構成を取得するために不要なものをすべて削除し、ダウンロードの段階的な実装を開始します。





ステージ1-問題分析



Ajax.ActionLinkに初めて会ったとき、私は考えました-うん！ もっと簡単なことは、この方法でアプリケーション全体を実装することです！ ページテンプレート（_Layout.cshtml）のメインブロックをメインとして識別し、他のすべてのページをロードします。 ただし、すぐに問題が発生します。

  1. ユーザーは新しいページに移動し、「戻る」ボタンで前のページに戻ることを期待していますが、これは起こりません。ユーザーは前のページではなく、最初に完全にロードされたページに戻ります。 進むボタンも機能しません。
  2. ユーザーが新しいページに切り替えて、（たとえば、F5を使用して）更新したいが、現在のページを更新する代わりに、最初に完全にロードされたページが再度ロードされます。
  3. ユーザーが新しいタブ/ウィンドウで新しいページへのリンクを開きました。 ただし、通常のページを開く代わりに、メインテンプレート、スタイル、およびスクリプトなしでPartialViewが読み込まれます。
  4. ユーザーはサイトのページに目を通し、気が散って、タブ（ウィンドウ）の名前で以前に作業したページを見つけたいと考えました。 ただし、ページの名前は_Layout.cshtmlにあるため、ナビゲーションプロセス中も変更されていません。
  5. ナビゲーションから気をそらします。 ユーザーは必要なデータ入力ページに移動しましたが、... ajaxによって返されたページのデータのクライアント側検証は機能しません！
  6. そして今、ユーザーはリンクをたどり、データベースから100500件のレコードを引き出し、それらに対して操作を実行します。 その後、彼は同じリンクをもう一度クリックしました。「何かが再び機能しなくなった」と判断したためです。 そして再び。 そしてその後、ユーザーがこれらすべてのリクエストの終了を待たなかった方が良いでしょう、これはページで起こり始めます... =）


かなりの数の問題がありました。 そして、それらのすべてが単純なjsコードで解決されるわけではありません。 ここでは、アーキテクチャへの統合アプローチが必要です。 もしそうなら、すぐにさらにいくつかの要件を追加します：

  1. 新しいページを読み込むときに、現在のページが単にブロックされ、システムで何が起こっているかについてのメッセージがユーザーに表示されると仮定します。
  2. アプリケーションが通常のマルチページモードで動作するかのように、履歴のナビゲーションをシンプルかつ直感的にしましょう。
  3. 私が言ったように、私はjsが好きではありません。 コードの最大量をc＃で記述してみましょう。 また、jsでは約10行のみを記述します。


最終的に採用された技術的制限：

  1. html5チップを使用します。
  2. レイアウトとデザインは扱いません。 ポータルを適切に機能させるだけです。
  3. ポータルを複雑にする場合、ソリューションはできる限りシンプルにする必要があります。 すぐにエリアのサポートを組み込み、機能の主要部分はAjax / Htmlヘルパーでレンダリングされます。
  4. ソリューションを簡素化するために、いくつかの魔法の命名規則を採用します。 これらの契約については後で説明します。


問題が記述され、目標が設定されました、行きましょう！



ステージ2-コントローラーの実装



コントローラーから始めましょう。 明らかに、ajaxを使用してページの両方の部分とページを完全にロードするには（ユーザーがリンクを直接クリックしたか、ページを更新した場合）、PartialViewResultとViewResultの両方を返すアクションが必要です。



したがって、論理アクションごとに2つの物理アクションをコントローラーに配置します。1つはSomeActionタイプ、 もう 1つはViewResult 、 もう 1つはAjaxSomeAction：PartialViewResultです。 この場合のAjaxプレフィックスは、最初の魔法の命名規則です。 この事実を忘れないでください。

合計： 

public class HomeController : Controller { private Object GenerateIndexPage() { Object model = null; for (int i = 0; i < 1000000000; i++) { } return model; } [HttpGet] public ViewResult Index() { return View("Index", GenerateIndexPage()); } [HttpGet] public PartialViewResult AjaxIndex() { return PartialView("Index", GenerateIndexPage()); } [HttpGet] public ViewResult About() { return View("About"); } [HttpGet] public PartialViewResult AjaxAbout() { return PartialView("About"); } }


GenerateIndexPage（）メソッドは、このような簡単な方法で、たとえばデータベースへのアクセスなど、いくつかの長い操作をエミュレートします。

ステージ4-Ajax.ActionLinkTo（...）



このヘルパーは、標準のAjax.ActionLink（...）に対する小さな拡張です ： 

 public static class AjaxHelpers { /// <summary> ///   ,    Ajax       main ///  PartialView /// </summary> public static MvcHtmlString ActionLinkTo(this AjaxHelper ajaxHelper, String linkText, String actionName, String controllerName = null, String areaName = null, String loadMessage = null, Object routeValues = null, Object htmlAttributes = null) { //   RouteValueDictionary routeValueDictionary = new RouteValueDictionary(routeValues); if (!String.IsNullOrEmpty(actionName) && !routeValueDictionary.ContainsKey("action")) { routeValueDictionary.Add("action", actionName); } if (!String.IsNullOrEmpty(controllerName) && !routeValueDictionary.ContainsKey("controller")) { routeValueDictionary.Add("controller", controllerName); } if (!routeValueDictionary.ContainsKey("area")) { if (!String.IsNullOrEmpty(areaName)) routeValueDictionary.Add("area", areaName); else routeValueDictionary.Add("area", ""); } //   Ajax AjaxOptions ajaxOptions = new AjaxOptions() { UpdateTargetId = "main", InsertionMode = InsertionMode.Replace, HttpMethod = "GET", LoadingElementId = "loadLayout", OnBegin = "changeLoadMesage('" + loadMessage + "')", OnSuccess = "onPageLoaded()" }; //   String ajaxActionName = "Ajax" + actionName; return ajaxHelper.ActionLink(linkText, ajaxActionName, null, routeValueDictionary, ajaxOptions, HtmlHelper.AnonymousObjectToHtmlAttributes(htmlAttributes)); } }


ヘルパーはエリアを考慮して必要なルートを作成します（ 結局、非常に多くの実際のプロジェクトにはエリアがあります）。それらをアクションとコントローラーと一緒に設定することは非常に便利でした。 ページをajaxでナビゲートするときにブラウザーのアドレスバーに挿入するため、およびナビゲーションの履歴を保持するために、このルートが必要になります。



AjaxOptionsオブジェクトは、メインブロックの本体を受信したコンテンツで置き換え、2つのjs機能を実行します（詳細は後ほど）。

  • changeLoadMesage-新しいページの読み込みプロセス中にユーザーに表示されるテキストを変更します。
  • onPageLoaded-ページの読み込みが成功した後に呼び出され、アドレスバーと履歴に関連するいくつかのアクションを実行します。


実際、ヘルパーを呼び出したときにアクション名をIndex / Aboutとして指定した場合でも、ヘルパーは上記のAjaxプレフィックスを置き換え、Ajaxリクエストを対象とするアクションを正確に呼び出すことに注意してください。 AjaxIndex / AjaxAbout 。

このシーケンスでアプリケーションをすばやくテストできます。スクリーンショットで説明しました。

  1. ダウンロードページ/ホーム/インデックス;
  2. Aboutリンクが/ Home / AjaxAboutページにつながることを確認してください。
  3. リンクをたどり、ajaxを使用してロードされることを確認します。リンク/ Home / Aboutがアドレスバーに表示されます。
  4. アクション/ホーム/ AjaxIndexを指すインデックスリンクに従ってください。
  5. 長時間の操作中にメッセージが表示され、他のリンクをたどることができないことを確認してください。
  6. インデックスページを読み込んだ後、ストーリーが保持され、ナビゲーションが利用できることを確認します（「戻る」および「進む」）。
  7. ページに移動した後、ページの更新が正しく機能することを確認してください。
