Microsoft Hyper-V Server 2008 R2のインストールと構成

エントリー



今日は、Microsoftハイパーバイザーのインストール方法と構成方法、および管理方法について説明します。

だから、最初に少し余談。

既存のインフラストラクチャの最適化により、仮想化に関する疑問が生じました。 VMwareのすばらしいソリューションであるESXiをすぐに思い出しました。ESXiは以前に何度も使用したことがあり、満足し、移行しようとしていましたが、それ以外の場合は運命が決まりました。 仮想化プラットフォームの計画と選択の過程で、Hyper-Vは予想外に敗北しました。

ハードウェアに対する優れたサポートと、私の組織がMicrosoft製品のみを使用しているという事実により、緊密な統合と管理性の向上という2つのことが、彼の方向にスケールをもたらしました。 残念ながら、ESXi 4.1ではADのサポートが移行の開始よりも遅れて発表されました。そうでなければ、おそらく考えたでしょう。

多くの人が知っているように、Hyper-V Server 2008 R2は無料の製品であり、サーバーコアモード（GUIなし）で動作し、Hyper-Vの役割が有効なWindows Server 2008 R2標準オペレーティングシステムです。 この点で、「Windows」管理者向けのシステム設定にはいくつかの困難があります。「次」および「準備完了」という魔法のボタンがないためです。 。



設置



インストールから始めましょう。

CD-ROMからのインストールは大したことではありません。CDから起動し、言語を選択し、ライセンス契約に同意し、パーティションを選択して10分間「禁煙」します。







カスタマイズ



これで、楽しみが始まります-セットアップ。

最初のログイン時に、サーバーの初期設定を実行できるテキストメニューが表示されます。



私の意見では、これはあまり便利ではありません。さらに、デフォルトでは、Hyper-Vマネージャーを使用してサーバーにリモートで接続できないため、サーバーを大胆に閉じてコマンドラインに移動します。

1.管理者パスワードを設定します

net user administrator *



コマンドを実行して、新しいパスワードを入力します

2.ネットワークのセットアップ

既定では、ネットワークアダプターはDHCP経由でアドレスを受信するように構成されています。これが適切な場合は、すぐに手順3に進むことができます。

コマンドnetsh interface ipv4 show interface



実行し、目的のネットワークカードのIDxを覚えておいてください。

コマンドnetsh interface ipv4 set address name='2' source=static address=192.168.0.2 mask=255.255.255.0 gateway=192.168.0.1



。2は同じIDxです。

DNSサーバーを構成するには、 netsh interface ipv4 add dnsserver name='2' address=192.168.0.10





netsh interface ipv4 set address name='2' source=dhcp



コマンドnetsh interface ipv4 set address name='2' source=dhcp



ネイティブ設定をいつでも返すことができます

3.コンピューター名とドメインエントリ

原則として、ドメインは必ずしも必要ではありませんが、人生を大幅に簡素化します。後で説明します。 ドメインがあると仮定し、この人間の思考の成果を使用します。

hostname



コマンドでコンピューター名を確認します

netdom renamecomputer WIN-KMTUYKKZPJQ /newname:vm1



でコンピューター名を変更しますnetdom renamecomputer WIN-KMTUYKKZPJQ /newname:vm1



で、WIN-KMTUYKKZPJQはコンピューターの古い名前で、vm1は新しい名前です。

コマンドnetdom join vm1 /domain:contoso.com /userd:administrator /password:*



を使用してコンピューターをドメインに入力し、指定されたアカウントのパスワードを入力します。

再起動します。

これで初期設定は完了です。



運営管理



サーバーを管理するには、クライアントオペレーティングシステム用のリモートサーバー管理ツール（RSAT）をダウンロードしてインストールし、Hyper-Vマネージャーコンポーネントをインストールするか、Hyper-Vの役割を追加してWindows Server 2008 R2にHyper-Vマネージャーコンポーネントをインストールする必要があります。

サーバーに接続しようとしていますが、ビール管理者に必要な権限を発行するように指示するエラーが表示されます。 私たちは同じ管理者であることを忘れないでください。

MicrosoftのHyper-Vチームでシニアプログラムマネージャーの地位にあるJohn Howardは、必要な権利の配布に関する一連の記事を執筆し、その後、管理者の頭脳を吹き飛ばすことなくサーバーとクライアントの巧妙な構成を行う素晴らしいユーティリティHVRemoteを作成しました。

それを使用します。 したがって、 ダウンロードして 、 \\ server \ C $でサーバーに移動し、HVremote.wsfをWindowsフォルダー（または他の任意の場所に配置しますが、絶対パスを指定することを忘れないでください）。 サーバーで実行します。

cscript hvremote.wsf /add:domain\account ***



、domain \ accountはドメインのユーザー名です。 スクリプトは、ファイアウォールで必要なポートを開くことを含む、必要なすべての特権を登録します。

次に、cscriptクライアントhvremote.wsf / mmc：enableで、スクリプトはファイアウォールの例外を作成します。

これで、Hyper-Vマネージャーを起動してサーバーに接続し、仮想マシンを作成して生活を楽しむことができます。





おわりに



私が直面しなければならなかったいくつかのポイントについて説明します。

1.英語版のHyper-Vを使用します-ファイアウォール内のグループの名前が異なるため、スクリプトはロシア語では動作しません。 クライアントからは、hyper-vインターフェイスはロシア語のままです。

2.ドメインを使用します。 ドメインがない場合は、次の手順を実行する必要があります。

• net user



を使用してサーバーとクライアントにアカウントを作成する

•このユーザーにcscript hvremote.wsf /add:accountname ***



アクセスを許可cscript hvremote.wsf /add:accountname ***





•クライアントで、DCOM cscript hvremote.wsf /anondcom:grant



への匿名アクセスをcscript hvremote.wsf /anondcom:grant



しますcscript hvremote.wsf /anondcom:grant



、サーバーへのアクセスを許可された同じアカウントでログインするか、目的のアカウントでHyper-Vマネージャーを起動し、 cmdkey /add:servername /user:servername\account /pass



、およびcscript hvremote.wsf /mmc:enable
