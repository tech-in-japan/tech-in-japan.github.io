iOSアプリでApple Musicを実装する

今日、 私たちのパートナー -Music Paradiseの開発者-は、ブログの読者のために新しい資料を準備しました。 今回は、Apple Musicを自分のアプリケーションに統合する方法について詳しく説明します。 同様の経験はすでにHabréで説明されていますが、Music Paradiseの開発者による記事は、読者にこのプロセスを知ってもらうためではなく、Apple Musicで作業する際にそれについて、そして重要ではあるが明らかではないいくつかの点について詳しく話すことを目的としています。



「iOS 9.3のリリースにより、Apple CorporationはApple Musicサービスを使用して音楽を再生できるようになりました。 この機能を試して、 Music Paradise Playerアプリケーションで作業することにしました。 この経験は非常に有益でした-システムにいくつかの落とし穴と弱点が発見されました。これらは事前に知るのに役立ちます。 この記事では、Apple Musicを使用する主なポイントと、このシステムのいくつかの機能について簡単に説明します。









Apple Musicへのアクセスを確認する



開発者向けのApple Musicサポートは、バージョン9.3以降のSDKで利用できるため、アプリケーションが以前のバージョンのiOS用に開発された場合、この例外を処理する必要があります。



Apple Musicを再生できるかどうかを確認する次のステップは、SKCloudServiceControllerへの認証リクエストです。 



[SKCloudServiceController requestAuthorization:^(SKCloudServiceAuthorizationStatus status) { //   }];


ステータスにはいくつかの値があります。



  • SKCloudServiceAuthorizationStatusAuthorized-すべてが正常です。 これが私たちの目標ステータスです。他の場合、音楽を完全に再生することはできません。
  • SKCloudServiceAuthorizationStatusDenied-ユーザーがデバイスのライブラリへのアプリケーションアクセスを拒否したため、これ以上進むことはできません。 この状況での良い方法は、ユーザーが設定にアクセスして手動でアクセスできるようにすることです。
  • SKCloudServiceAuthorizationStatusRestrictedは同じ期待はずれの結果ですが、唯一の違いは、ユーザーが制御できない状況のためにアクセスできないことです。


この場合、ユーザーがポップアップウィンドウで回答を選択して作業を続ける必要があるため、SKCloudServiceAuthorizationStatusNotDeterminedのステータスを受け取ることができないことに注意してください。 したがって、このオプションを処理しても意味がありません。



ユーザーのライブラリへのアクセスがわかったので、ユーザーがApple Musicのサブスクリプションを持っているかどうかを確認する必要があります。 



 SKCloudServiceController *controller = [[SKCloudServiceController alloc] init]; [controller requestCapabilitiesWithCompletionHandler:^(SKCloudServiceCapability capabilities, NSError * _Nullable error) { //   }];


ここには多くのステータスオプションもありますが、受け入れられるのは2つだけです。



  • SKCloudServiceCapabilityMusicCatalogPlayback-ユーザーはApple Musicから音楽を再生できます。iCloudライブラリはオフになっています。
  • SKCloudServiceCapabilityAddToCloudMusicLibrary-ユーザーはApple Musicから音楽を再生でき、iCloudライブラリがオンになっています。


必要に応じて、失敗したSKCloudServiceCapabilityNone（ユーザーはApple Musicのサブスクリプションを持っていない）を処理することもできます。つまり、ユーザーをサブスクライブに招待します。











トラックリストリクエスト



iTunes Search APIに出会ったことがない人にとっては、トップトラックの検索と出力は非常に複雑なプロセスです。 明確にするために、いくつかのポイントに分けます。



キーワード検索



リクエストの一般的なビューは次のとおりです。



itunes.apple.com/search?term=%@&limit=%ld&s=%@&entity=song&explicit=No



各パラメーターを分析しましょう：



用語 -検索クエリ。 留意してください：複数の単語で構成されている場合は、スペースを+に置き換える必要があります（おそらくこれは多くの人にとって明らかですが、指定する価値があります）。



制限 -結果の最大数。 ここではすべてが直感的です。 デフォルト値は50ですが、場合によってはこれで十分ではない場合があります。



S （Appleによる良好な動作、名前によってすべてがすぐに明確になります）-この検索引数は必須であり、ユーザーアカウントがリンクされているApp Storeの国を示します。 実際のところ、Apple Musicのさまざまな国ではさまざまな楽曲が提供されています。 さらに、同じトラックが異なる国で異なる識別子を持つ場合があります。



特定のユーザーのこの識別子を取得するには、次を使用します。 



 SKCloudServiceController *controller = [[SKCloudServiceController alloc] init]; [controller requestStorefrontIdentifierWithCompletionHandler:^(NSString * _Nullable storefrontIdentifier, NSError * _Nullable error) { if(!error) { storeID = storefrontIdentifier; // } }];


エンティティは必要なエンティティであり、私たちの場合は歌です。



明示的 -冒とく的なコンテンツを検索結果に含めるかどうかを決定します。 まず、アプリケーションの年齢制限から始める価値があります。



検索の使用に関する詳細については、 こちらをご覧ください 。



トップチャートの取得



トップチャートを取得するには、AppleのRSSフィードジェネレーターを使用する必要があります。 ジェネレータの出力で、このタイプのURLを取得します。



itunes.apple.com/en/rss/topsongs/limit=100/xml



ここでは、ロシアのトップトラックがxml形式で100個分表示されます。 私の意見では、この結果はいくつかの理由で私たちに合わない。



第一に、当社のオファーは市場および外国の消費者に提供されているため、国を変更する必要があります。 そのため、ruの代わりにユーザーの現在の国の識別子を置き換える必要があります。



識別子を取得するために、ちょっとしたトリックを使用しました。 IAPはアプリケーションで提供されるため、SKProductを取得して次のことを行います。 



 SKProduct *product = // IAP   NSLocale* storeLocale = product.priceLocale; NSString *storeCountry = (NSString*)CFLocaleGetValue((CFLocaleRef)storeLocale, kCFLocaleCountryCode);


したがって、URLでruの代わりにstoreCountryを使用します。



そして、私たちに合わない別の点があります-これは受信したデータの形式です。 しかし、ここではすべてが簡単です。行の最後でxmlをjsonに変更します。その結果、トップチャートを正しくアップロードするためにこのタイプのURLを取得します。



itunes.apple.com/%@/rss/topsongs/limit=100/json











iTunes検索APIから素晴らしいカバーを取得する方法



iTunes検索をアプリケーションに追加するときに発生した別の問題は、APIが100x100以下のトラックのカバーアートを返すことです。 ただし、コツがあります。 結果のjsonでは高品質の画像へのリンクは提供されませんが、次のアルゴリズムを使用して取得できます。



  1. キー「artworkUrl100」にあるjsonから画像へのリンクを取得します。
  2. 結果の行を100x100から600x600に置き換えます。
  3. 大きな画像へのリンクを取得します。








Apple Musicからトラックを再生する



すべての準備を行った後、どんなに奇妙に聞こえるかにかかわらず、この手順は、Apple Musicをアプリケーションに追加するプロセス全体で最も簡単です。 プレーヤーを初期化して、再生トラックのリストを提供するだけです。 再生トラックのリストは、トラック識別子を持つ文字列を含む配列の形式で設定されます。 コードとしては、次のようになります。 



 -(void)playQuery:(NSArray<NSString*>*)query { self.player = [MPMusicPlayerController applicationQueuePlayer]; [self.player setQueueWithStoreIDs:query]; [self.player play]; }


Apple Musicでの作業に興味のある機能であることをもう一度思い出してください。

setQueueWithStoreIDs：iOS 9.3以降、SDKに登場しましたが、その瞬間まで、ユーザーのiTunesライブラリを再生するためだけにMPMusicPlayerControllerを使用できました。



機能の学習で遭遇した困難について話すと、検索APIからのjsonesを処理した後、辞書には文字列ではなくデジタル値が含まれていたことが思い出されます。 これを監視する必要があります。



また、この例ではapplicationQueuePlayerを使用していることにも言及する価値があります。 iOS 10.3でのみ登場しましたが、複数のトラックを操作しているときやリモートコントロール（ロックされた画面やヘッドフォンなど）でクラッシュしたり、systemMusicPlayerに比べてクラッシュするapplicationMusicPlayerよりも優れたソリューションのようです事実はアプリケーションとはまったく関係がなく、ユーザーがログアウトした後でも再生を続けることができます。
