ストリーム出力方法



  • ローカルでプレイ：ブロードキャストしているものを視聴することは、現在、トランスコード、リスケーリングなどをサポートしています...（ローカルで使用するため）
  • ファイル ：ストリーミングされたストリームをハードディスクに保存します。 （自分自身を超えて何かを中継していて、それを保持したい場合に適しています）
  • HTTP / S ：HTTPを介したストリームストリーム。 IPアドレスとTCPポートを指定する必要があります。
  • MMS / H ：Windows Media Playerでのストリーミングに最もよく使用されます。 また、ASFカプセル化方式でのみ機能します。
  • UDP ：マルチキャストおよびユニキャスト（IPv6サポート可能）。 また、TSカプセル化方式でのみ機能します。
  • RTP ：リアルタイム転送プロトコル。 UDPと同様であり、ユニキャストマルチキャストアドレスを同時に使用できます。
  • RTMP ：リアルタイムメッセージングプロトコル-独自のストリーミングプロトコルで、主にインターネット上のWebカメラからのビデオおよびオーディオストリームのストリーミングに使用されます。 （VLCは、中継可能な着信ストリームとしてのみサポートされています。）

UDPマルチキャスト

マルチキャストでは、ストリームはマルチキャストIPアドレスにブロードキャストされます（ IPアドレスはこれらの目的のために予約されており、範囲は224.0.0.0から239.255.255.255です ）。 その後、ネットワーク上の任意のマシンがマルチキャストグループに参加して、ネットワークにリクエストを送信すると、ブロードキャストストリームを自動的に受信します。



次に、ストリームの再生を停止する要求を送信し、このグループを離れます。 マルチキャストストリームの利点は、それを受信したいマシンのみがストリームを受信し、サーバーはグループのすべてのメンバーが受信する1つのストリームのみをブロードキャストすることです。
