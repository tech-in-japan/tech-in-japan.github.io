Selenium WebDriver 2.xに基づく自動テストの開発

こんにちは、Habrosociety様！ C＃プログラミング言語のSelenium WebDriver 2.xに基づいた自動化されたテストの作成と編成の方法を共有したいと思います。

私はすぐに、多くの論文、原則、および開発方法が私によって（そしておそらく、私の教師によっても）発明されなかったことを言わなければなりません。

おそらく、この投稿は、Webアプリケーションの自動テストに従事し始めたばかりの人々を助けるでしょう。





準備する



Seleniumを使用して自動テストを開発するには、次のコンポーネントが必要です。

開発環境、この場合はVisual Studio。

書かれたテストを自動的に実行するためのNUnitプログラム（ここからダウンロード： www.nunit.org/index.php?p=download ）;

2つのプラグインがインストールされたMozilla Firefoxブラウザー：FireBugとFirePath（この記事では使用しません）。



開発環境のセットアップ



自動テストを作成するには、 クラスライブラリなどの新しいプロジェクトを作成する必要があります（コンソールアプリケーションは作成できますが、この場合はできません）。 テストするプロジェクトの名前に関連付けられた名前を付けることをお勧めします。 また、テストプロジェクトと共にソリューション自体にテストプロジェクトを作成することをお勧めします。

テストプロジェクトを作成したら、テスト用の3つの特別なライブラリを接続する必要があります。

NUnit（nunit.framework） -自動テストを直接実行するためのライブラリ（Seleniumだけでなく）。

WebDriver + WebDriver.Support-直接Selenium'aライブラリ。

ライブラリをテストプロジェクトにNuGet経由で接続するか、 code.google.com / p / selenium / downloads / listから最新バージョンをダウンロードして手動で接続できます。



テストプロジェクトアーキテクチャの構築



基本クラス


自動テスト自体を作成する前に、最初にヘルパークラスを作成する必要があります。

テストプロジェクトで、新しいクラスを追加し、 SeleniumTestBase.csと呼びます（別の方法でもかまいません）。 すべてのテストクラスが将来継承されるのはこのクラスからなので、このクラスをpublic abstractとして宣言します。 また、クラスにテストクラスとして指定する[TestFixture]属性をクラスに追加する必要があります。 このクラスでは、テストに合格するために必要なフィールドとメソッドが宣言されています。

このクラスでは、フィールドを宣言する必要があります。

保護されたIWebDriverドライバー 。 -これは、テストを実行する前に作成されるブラウザードライバーのインスタンスです。 4種類のドライバーがあります（+開発中のいくつか）：FirefoxDriver、InternetExplorerDriver、ChromeDriver、HtmlUnitDriver（グラフィカルコンポーネントなしのクロスプラットフォーム仮想ブラウザー）。



また、クラスで4つのメソッドを宣言します（ただし、一部は空または\であり、まったく定義されません）。

public void TestInitialize（）with [TestFixtureSetUp]属性 -この属性を持つメソッドは、テストクラスからテストを実行する前に1回呼び出されます（複数のクラスがある場合、メソッドは最初のクラスのテストの前、2番目のクラスのメソッドの前などに呼び出されます）さらに）。 このクラスでは、たとえば次のように特定の値をDriver変数に割り当てることが望ましいです。

Driver = new FirefoxDriver();







public void TestCleanup（）with [TestFixtureTearDown]属性 -前のメソッドと同じように動作するが、テストクラスを実行した後のみ。 このクラスでは、次のテストクラスを実行すると、ブラウザードライバーの新しいインスタンスが作成されるため、ブラウザードライバーを閉じることが望ましいです（その結果、ブラウザードライバーの多くのインスタンスが開いたままになり、RAMがゴムではなくなります）。

Driver.Quit();







public void OneTearDown（）with [TearDown]属性 -メソッドは個々のテストを実行した後に呼び出されます。 たとえば、このメソッドでは、新しいテストの前にブラウザーのCookieをクリアできます。

Driver.Manage().Cookies.DeleteAllCookies();







[SetUp]属性を持つpublic void OneSetUp（） -メソッドは、個々のテストが実行される前に呼び出されます。 たとえば、テストの実行時にブラウザーウィンドウを全画面に展開できます（HtmlUnitDriverには関係ありません）。

Driver.Manage().Window.Maxmize();







ラッパー


異なるテストを（異なるクラスであっても）作成する場合、コードの複製が可能です（より正確には、間違いなく可能です）。 テストクラスの混乱を避けるために、ヘルパークラスの静的クラスWrapperを作成して、そこに別のメソッドでコードを複製すると便利です。

ほとんどのテストは、同じコード行で開始されます。

Driver.Navigate().GoToUrl(<>);





したがって、Wrapperクラスでは、必要なURLとこのURLを取得するメソッドを記述するプライベート静的文字列_domainフィールドを定義するのが論理的です。

コードの一部 
 public static string GetUrl() { _domain = "http:\\www.yandex.ru"; return _domain; }






Wrapper.csとSeleniumTestBase.csの違いは、最初はテスト内で（さらにSeleniumTestBase内でも）呼び出されるメソッドが宣言され、2番目ではテストの前または後に呼び出されるメソッドが宣言されることです。

補助テストを作成した後、通常のテストクラスを既に作成します。



したがって、テストプロジェクトのアーキテクチャは次のようになります。





テストライティングの例



新しいテストクラスを作成し、TestClass1.csを呼び出します。 最初に簡単なテストコードを記述し、次にどのコード行がそれを行うのかを説明します。

GoTo Yandex 
 [Test, Timeout(10000)] public void Test1() { Driver.Navigate().GoToUrl(Wrapper.GetUrl()); Assert.IsTrue(Driver.Title == ""); }




[テスト] -このメソッドがテストであることを決定する属性。

Timeot（int millSec） -テスト属性でのみ動作します-テストに合格する最大許容ミリ秒数を示します。 タイムアウトを超えると、テストは直ちに（実行時でも）クラッシュし、失敗したと見なされます。

public void-テストはそのようなアクセス修飾子のみを持つ必要があり、テストが正しく機能するために値を返さないようにする必要があります。

Driver.Navigate（）。GoToUrl（Wrapper.GetUrl（）） -ブラウザードライバーは、Yandexページを開きます。

Assert.IsTrue（Driver.Title ==“ Yandex”） -直接チェック。この場合、テストはページタイトルが「Yandex」であることを確認します。この場合、テストは合格と見なされ、そうでない場合はテストは失敗します。



テストを実行する



テストを実行するには、このための特別に作成されたプログラム-NUnit （ドキュメントの冒頭にあるダウンロードへのリンク）が必要です。

テストプロジェクトをコンパイルします。 出力には、.dllタイプのファイルがあります

NUnitを実行します。 次に、[ファイル]-[プロジェクトを開く]をクリックします。 コンパイルされたばかりのライブラリを見つけて選択する必要があるウィンドウが表示されます。

ライブラリをプログラムにロードすると、左側にテストが含まれるテストクラスのツリーが表示されます。 このツリーで、単一のテスト、クラス、またはすべてのテストクラスを選択し、[実行]ボタンをクリックすると、テストの実行が開始されます。 停止ボタンを押すと、いつでも実行を停止できます。

テストに合格すると、ツリー内の色が変わります：緑-テストに合格、赤-テストが落ち、黄色-何らかの理由でテストが失敗（落下ではない）、グレー-テストがまだ開始されていません ボタンの下には、ドロップされたテストのログを含むウィンドウがあり、特定のテストが失敗した理由を見つけることができます。



追加情報



最速のブラウザードライバーはChromeDriverと見なされます。 最も遅いと思います。



ChromeDriverおよびInternetExplorerDriverの場合、追加のドライバーアプリケーションをダウンロードして、プロジェクトディレクトリに配置する必要があります。



また、テストでは属性が使用されます。

[TestCase（var v1、...、var vOver9000）] -テストで複数の入力データを反復処理する必要がある場合に使用されます。 テストの前に、テストケースをいくつでも作成できます。 正常に動作させるために、テストメソッドは、テストケースに登録されている変数の数と同じ数のパラメーターを渡す必要があります。 テストケースの構成は同一である必要があります（1つの変数でテストケースを作成し、テスト前に2つの変数でテストケースを作成することはできません）。 例：

テストケース（） 
 [Test, Timeout(10000)] [TestCase("", 1)] [TestCase("", 2)] [TestCase("", 3)] [TestCase("", 4)] [TestCase("", 5)] public void Test2(string str, int days) { /*    5       10 */}




テストケースの代わりに、属性[Range（）]および[Values（）]があります。 ただし、テストケースとは異なり、これらの属性は渡された引数とともに書き込まれます。 範囲は列挙値、非列挙値には最適です。 また、実行されるテストの数は、変数のすべての組み合わせの数に等しくなります。 例：

範囲+値 
 [Test] public void Test3([Range(0,3)]int a, [Values(“0”, “1”, “2”, “3”)]string b) { Assert.IsTrue(a.ToString() == b); }




16のテストがあり、そのうち4つだけが合格します。



ヒント



自動化されたテストを作成するときは、 AAAの概念-Arrange-Act-Assertに従う必要があります。

配置 -テスト自体の前に必要な設定とアクション。

行為 -直接テスト;

アサート -「期待される結果==現在の結果」タイプの条件をチェックするためのブロック。

上記の例を使用して、少し書き直してください。

AAA Begin 
 [Test, Timeout(10000)] public void Test1() { Driver.Navigate().GoToUrl(Wrapper.GetUrl()); var title = Driver.Title; Assert.IsTrue(title == ""); }




ここで、Arrangeはurlの通過、Actはタイトルページの計算、Assertは条件「expected result == current result」のチェックです。

読みやすくするために、テストは次のように書き換えることができます。

AAA Complete 
 [Test, Timeout(10000)] public void Test1() { //Note : Arrange Driver.Navigate().GoToUrl("http:\\www.yandex.ru"); //Note : Act var title = Driver.Title; //Note : Assert Assert.IsTrue(title == ""); }






習慣を身に付けてください：1つのテスト-1つのアサート。 まったく違反しないか、極端な場合には違反することをお勧めします（テストで多数のチェックを行うと、コードの可読性が低下します）。



コード内の多くのセレクターは数回繰り返すことができます。 サイトのレイアウトは常に変化する可能性があり、セレクターがあるため、それらを別々の文字列定数に入れる方が適切です。 その後、セレクタを変更する場合、プロジェクト全体で20箇所ではなく、1箇所でのみ変更する必要があります。



ブラウザードライバーを最初に起動したとき、ページの読み込みに時間がかかることがあるため、テストの最初の起動がタイムアウトで落ちることがあります。 一部のテストのタイムアウトで増加させる必要があります。



品質基準により、ブラウザでページをロードするためのタイムアウトは10秒です。 ページ上の要素を見つけるためのタイムアウトを11秒で設定します（最大12）。



ページ上の一部の要素は、Selenium + C＃を使用して操作できません。 この場合、このようなページ要素にアクセスするには、JavaScript + JQueryを使用する必要があります。 例：

JavaScript + jQuery 
 protected IJavaScriptExecutor _jsExecutor; var script = @"var c = $('table>tbody:visible').last();" + "c = c.find(\"tr:nth-child(2)\");" + "c.find(\"td:nth-child(1)\").click();"; _jsExecutor = (IJavaScriptExecutor)Driver; _jsExecutor.ExecuteScript(script);






テストの実行に便利なように（NUnitは最も便利なものではありません）、オプションでReSharperプラグインをVisual Studioにインストールできます。これにより、テストを簡単に実行し、実行結果を確認できます。 ただし、NUnitとは異なり、ReSharperは有料のアドオンです。
