SharePointアプリケーション環境を構成する（SharePointアドイン）

SharePointアプリケーション環境は、SharePointサイトに機能を追加するための便利で最新のアプローチです。



有効にするには、次のものが必要です。



•SharePoint用の英語の言語パック

•HTTPSをサポートする証明書



SharePointでアプリケーション環境を有効にします。 次のアクションを実行する必要があります ：（ 参照により自分で設定することもできますが、非常に重要な事実はありません）。



1.アプリケーションのDNSレコードを作成します。 必ず第4レベルのドメインを使用してください。 これにより、アプリケーションに対するユーザーの再承認を回避する機会が提供されます。 CNAMEによって作成された例：* .sharepoint.contoso.com



2. sharepoint.contoso.comドメインの保護をサポートする外部企業からのワイルドカード* .sharepoint.contoso.com証明書を注文します。 必ずドメインを使用してください！



余談が1つあります。WindowsServer 2016 IIS 10では、ワイルドカードタイプの個別のバインディングを作成できるようになりました。 そのため、* .contoso.comのワイルドカード証明書が既にある場合、* .sharepoint.contoso.comを保護するためにのみワイルドカード証明書を購入できます。



3.サーバーの全体管理で、サーバーでサービスを開始してSharePointアプリケーションを提供します。









4. SharePoint PowerShellでアプリケーション環境に必要なサービスを構成します。



-a。 サブスクリプション設定サービス： 



$account = Get-SPManagedAccount "contoso\spservice" #       Sharepoint $appPoolSubSvc = New-SPServiceApplicationPool -Name SettingsServiceAppPool -Account $account #    IIS     $appSubSvc = New-SPSubscriptionSettingsServiceApplication -ApplicationPool $appPoolSubSvc -Name SettingsServiceApp -DatabaseName SP_SubscriptionSettingsService #   Subscription Settings $proxySubSvc = New-SPSubscriptionSettingsServiceApplicationProxy -ServiceApplication $appSubSvc #       Subscription Settings


b。 アプリ管理サービス 



 $appPoolAppSvc = New-SPServiceApplicationPool -Name AppServiceAppPool -Account $account #    IIS     $appAppSvc = New-SPAppManagementServiceApplication -ApplicationPool $appPoolAppSvc -Name AppServiceApp -DatabaseName SP_AppManagementService #   App Management $proxyAppSvc = New-SPAppManagementServiceApplicationProxy -ServiceApplication $appAppSvc #       App Management


5.自動生成されたアプリケーションリンクを設定するには：



-a。 サーバーの全体管理のアプリケーションセクションに移動します。









-b。 次の（あなたの）インジケーターを示します







6.インターネットからダウンロードしたアプリケーションを完全に動作させるには、次のことを確認してください。



-a。 英語の言語パックをインストールする



-b。 英語をオンにする







-c。 サーバーの全体管理でインターネットエンドポイントアクセスコンポーネントをアクティブにします。







7.ユーザーがインストールする最初のアプリケーションを追加します。



-a。 [アプリケーション]タブのサーバーの全体管理に移動します。



-b。 [アプリケーションの購入]リンクをクリックします。



-c。 数秒後、インターネットからアプリケーションディレクトリが表示されます。



プロキシサーバーがある場合は、追加の設定が必要になる場合があります。

この記事で説明します 。



-d。 アプリケーションをクリックし、「追加」をクリックしてから確認します。 最後に、「組織内のすべてのユーザーに対してこのアプリケーションを受け取りました。」という碑文が表示されます。



-e。 これで、アプリケーションをユーザーサイトに追加できます。 SharePointで提供される標準のボタンの中で、アプリケーションの追加ボタンをクリックすると、サイトのコンテンツに表示されます。



-f。 アプリケーションに対するすべての操作はインターネット経由で行われるため、SharePointサーバーのインターネットへのアクセスは非常に重要です。 ユーザーのサイトへの最終インストール時でも、アプリケーションファイルはインターネットから（毎回）ダウンロードされます。



-g。 また、開発したアプリケーションをSharePointポータルでホストできるようになりました
