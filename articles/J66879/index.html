LinuxからYotaに接続する

良い一日。 今日は、LinuxデバイスをYotaネットワークに接続する方法についてお話します。 私の場合、これはAsus EEE 1003で、EEEbuntu 3 Standart（変更されたUbuntu 9.04）がボードにインストールされています。 IntelチップがWiMaxの動作を担当している場合（lspciを実行すると確認できます）-このレシピが役立ちます。 配布は特別な役割を果たしません。



カーネルソースファイルとヘッダーファイル（Ubunty、linux-sourceパッケージとlinux-headersパッケージ）、およびlinuxwimax.org Webサイトのコンテンツ：WiMAXドライバー、i2400mファームウェア、WiMAXネットワークサービス、およびIntel WiMAX Binary Supplicantが必要です。



ダウンロードしたファイルを/ usr / srcに解凍します（または他の場所ですが、この場合は例のパスに従ってください）。その後、インストールプロセスに進みます。 まず、デバイスにドライバーを配置します。



$sudo su

#cd wimax-i2400m-1.4.1

#make

#sudo make install

#sudo depmod -a







次に、i2400m-fwアーカイブからi2400m-fw-usb-1.4.sbcfファイルを/ lib / firmwareにコピーします。 modprobe i2400mコマンドの後（または再起動後）、device / dev / wmx0が必要です。 表示される場合、あなたは正しい軌道に乗っています。



ここで、ドライバーのコントロールバインディングをインストールする必要がありますが、最初にlibnl-devパッケージをインストールします。



#cd WiMax-Network-Service-1.4.0

#./configure --prefix=/usr/ --with-i2400m=/usr/src/wimax-i2400m-1.4.1 --localstatedir=/var --enable-debug

#make

#make install







次に-サプリカントを置きます。 ここではすべてが簡単です：



#./install_supplicant install







確かに、何らかの理由で/ usr / local / libに移動し、/ usr / libには移動しませんでした-ハンドルで修正しました（コピーして）

そして最後に必要なのは、 Yotaの設定を取得することです。 LJユーザーicelordは注意深くそれらをレイアウトしました。 / usr / share / wimax /に受信したファイルをアップロードします（パッチは不要です）。



まあ、それがすべてです。 次に、すべてをどのように操作するかについて少し説明します。



開始するには：

/usr/bin/wimaxd #

/usr/bin/wimaxcu ron # Wi-Max (Wi-Fi ).

/usr/bin/wimaxcu connect network 15 # Yota(15 - id Yota)







停止するには（およびWiFiモードに切り替えます）：



/usr/bin/wimaxcu roff







その他：

wimaxcu scan＃利用可能なネットワークをスキャン

wimaxcu info device＃デバイスデータを取得します。

wimaxcuステータスリンク＃接続情報



最後に、小さなヒント：長い間「ブラインド」アダプターの問題に苦しんでいました（ネットワークがあることは知っていましたが、システムはまったく表示されませんでした）-さまざまな方法を試し、別のカーネルに移動して、ドライバーを再構築しました。 解決策は、アダプターの設定をリセットすることでした：



wimaxcu reset factory
