私の経験を共有して、あなたが私の過ちを避け、時間、エネルギー、そして最も重要なことに神経を節約できるようにしたいと思います。 神経の塊。 彼らは昨日、「ひどく遅い、ブラウザは動かない」という説明を付けて、2文字の会社であるウルトラブックシリーズを持ってきました。 最初の検査では、1組のトロイの木馬による感染、システム内のユーザーがインストールした4つのブラウザーのいずれも動作不能、「システムに入れて-コンピューターの速度を上げる」カテゴリのユーティリティで一般的なシステムの乱雑さ 、レジストリを「クリーニング」するユーティリティによって引き起こされたいくつかの損害が示されました問題を克服します。 検査後、私は思ったように、ネイティブソフトウェアでハードディスクに保存されたリカバリパーティションのために、数分でシステムをすべて修復し、復元するよりもクリーンな方法でシステムを再インストールする方が簡単だと決めました（聖なる純真さ）。 さらなる物語は、私がこれにどれほど残酷であったかを示します。



合計で、Windows 8を備えたウルトラブック、リカバリパーティション、およびパッケージにディスクが存在しない（この点は重要です）。 ブナには、起動ソフトウェアとしてUEFIがあり、これは原則としてWindows 8で使用するのに論理的であり、1つでなければ、システムをインストールするためのキーはACPIデータテーブル、MSDMサブテーブルに配線されます。 当時、私はまだこれを知らず、必要だとは思わなかった。



一般的に、再起動し、スタートアップ設定に入り（UEFIフレッシュシリーズでブナでこれを行う方法がわからない場合-起動の開始直後にESCを押します）、セキュアブートをオフにします（これはリカバリシステムの起動に必要です）。ネイティブシステムリカバリシステム。 判明したので、セキュアブートをオフにし、システムをレガシーモードに切り替えましたが、無駄でした。ネイティブリカバリシステムのこのような奇妙な動作を引き起こしたのは、これらのアクションでした。 すべてのユーザーデータを削除して、システムを元の状態に復元するよう指示します（データは事前​​にコピーされています）。 そして、ここから冒険が始まります。 リカバリプロセスが起動すると、実行の最初の段階、つまりWindows用のパーティションのフォーマットが表示されます。 ステージが終わり、その後、リカバリプロセスが停止し、デジタルコードでわかりにくいエラーが発生し、ウルトラブックが再起動しますが、ディスク上にシステムが見つからなかったというメッセージが表示されます。 論理的には、パーティションをクリアし、システム回復キーをもう一度押して、回復プロセスを続行しようとしました。 そして、ここで最初の驚きは私を待っていました：システムの欠如について同じメッセージを受け取ります。 判明したように、 System Recoveryは剥がれたシステムとともに自己破壊しました 。 今、システムなし、ハードドライブ上のリカバリイメージなし、およびリカバリディスクなしのウルトラブックを保持していました。



唯一のオプションがありました-システムを最初から再インストールすること。 何が問題になりますか？ すべて再インストールします。 ただし、急いではいけません。すべてが見た目ほど単純ではありません。 後でシステムを8.1にアップグレードするときに余分な時間を無駄にしないために、すぐにクリーンな8.1をインストールすることにしました。 言われるとすぐに、最新のWindows 8.1イメージをMSDNからダウンロードし、 Windows 7 USB-DVDツールを使用して起動可能なUSBフラッシュドライブを作成し、インストールを開始します。 最初の問題は、キーがまだわからないことです。幸運なことに、インストールを続行する必要があります。 なぜこれが行われたのかは明確ではありませんが、Windows 7ではキー入力フェーズをスキップできました。 インターネットで短い検索を行うと、答えが得られます-Microsoftが開いた汎用キーを使用する必要があります-システムをインストールすることはできますが、アクティブにすることはできません。アクティブにするには、インストール後にこのキーを完全に機能するキーに置き換える必要があります。 汎用キーを入力し、インストールを続行します。 私はこの瞬間に立ち止まって（キーを要求する）考えますが、私はこの瞬間を逃しました。 実際、UEFIで配線されたシステムキーがこのバージョンのWindowsのインストールに適している場合、システムはインストール中にそれを要求するべきではありません。 一般的に 。 この場合、彼女はそれを自動的に使用します。 キーを要求した場合、これはUEFIからキーを読み取ることができないか、このキーに適さない配布バージョンを使用していることを示しています。 しかし、その時点では私はまだそれについて知りませんでしたので、それは私に質問を引き起こしませんでした。 そのため、システムがインストールされて起動しました。 今、それをアクティブにする必要がありました、そしてこのために-どういうわけかUEFIからキーを削除します。 結局のところ、これは難しい作業ではありません。 これを行うには、 PKeyUIユーティリティを使用できます-実行して「MSDM」フィールドにチェックを入れるだけです-ユーティリティはUEFIに保存されているキーを表示します。 ユーティリティの起動時に「MSDM」フィールドが表示されない場合は、ファームウェアにキーがないか、読み取り不能です。 だから、私は鍵を得た。 次に、私は彼と一緒にシステムをアクティベートしようとしましたが、問題も私を待っていました-アクティベーションシステムはキーが機能していないと述べました。 Windows OEMの配布方法を思い出して、OEMビルダーにはシステムイメージが明らかに必要だと判断しました。 まあ、問題ありません。 いくつかの検索の後、バージョン8.1のOEMサプライヤの元のシステムイメージの配布がトレントトラッカーで見つかりました。 私はそれをダウンロードし、インストールフラッシュドライブを再度作成し、インストールとアクティベーションを再度試みました。 しかし、彼は再び同じエラーを受け取りました。 トピックに関する情報を検索すると、方法がわかりました（後で判明したため、動作していません）-slmgrスクリプトを使用して古いキーを削除し、新しいキーをインストールしてから、システムを通常どおりアクティブ化する必要がありました。 試しましたが、ネイティブシステムキーをインストールしようとすると、常にエラーが発生しました。



0xC004F069



これは、通常、ターゲットPCとキーサーバー間の同期外れに関連しています。 少しためらい、この問題を解決する方法を探すためにさらに30分を費やして、私はおそらくソース、つまりMicrosoft Corporationに頼るべきだという結論に達しました。 問題の説明を含むテクニカルサポートへの問い合わせにより、すべてが明確になりました。Windows8.1イメージを使用しようとしたときに、主な間違いを犯したことがわかりました。 判明したように、Windows 8.1イメージは、Windows 8がプリインストールされたPCへのクリーンインストールにはまったく適しておらず、Windows 8のOEMキーはクリーンインストールのアクティベーションに適していないため、汎用キーを使用して問題を回避することもできません有効なWindows 8.1 OEM。 Windows 8を搭載したPCにバージョン8.1をインストールする唯一の方法は、Microsoft Storeからシステムをアップグレードすることです-唯一の方法です。 サポートのおかげで、Windows 8のクリーンなOEMバージョンをダウンロードしました。



しかし、これは私の試練の終わりではなく、すべてがそれほど単純ではないことが判明しました。 純粋なOEMバージョンは、インストールを断固として拒否されました-「インストール」ボタンをクリックした直後に、指定された（UEFIで）キーがシステムの検出されたイメージのいずれにも適合しないと報告しました。 短いグーグルの後、とりわけ、正しいバージョンのシステムとキーを使用して、インストールフラッシュドライブにマーカーファイルを作成する必要がありました。 さて、インストールフラッシュドライブのsourcesフォルダーに移動し、そこで2つのファイルei.cfgとpid.txtを作成します

pid.txtの内容：

XXXXXの代わりに、pkeyuiを介して受信したキーを入力します。 これらのアクションの結果として、Windows 8がまだインストールされるブート可能なUSBフラッシュドライブを取得します。 しかし、私の喜びは時期尚早でした-それは確立されましたが、不思議な方法で、pid.txt（UEFIのオリジナル）で忘れていたものとは関係のない完全にとんでもないキーを書き留めました。 この左キーを適切なキーに変更しようとして失敗しましたが、システムをアクティブにしようとしたときに、キーがシステムエディションに対応していないという同じ悪名高いエラーが発生しました！ ニュアンスがもう1つあることが判明しました。エディションとライセンス方法に加えて、ディストリビューションもSKUによって分割され、ラップトップビルダーは通常、CoreSingleLanguageまたはCoreCountrySpecificに配置します。 また、すべてのタイプのSKUでインストールキーが異なるため、CoreSingleLanguageイメージが必要でした。 この場合、システムでインストールを続行できるのはなぜですか、私には明確ではありません-キーがまだ機能せず、既にインストールされているシステムのSKUを変更する方法を私が見つけられなかったため、意味がありません。 静かに疲れ果てて、私はCoreSingleLanguage SKUでOEMディストリビューションを探しに行きました。 それでも、私は最終的にそれを見つけました、それは多くのトラッカーにあり、潜在的に危険なリソースへのリンクを提供しないように、必要な配布のMD5を提供します：

さらに、プロセス全体を通して、この手順の間に従うべきアルゴリズムを導き出しました。



  • ラップトップ上のWindows 8のみの場合は、Windows 8.1ではなくWindows 8のみを使用します。8.1をインストールしても、ライセンス認証できません。
  • pkeyuiユーティリティを使用してWindowsキーを見つけます-ライブシステムで実行し、MSDM dawを入れてキーを書き込みます。
  • Microsoftテクニカルサポートに電話し、状況を簡単に説明し、抽出したキーの有効性と所有権を確認するよう依頼します。 技術サポート担当者は問題なくキーをチェックし、どのタイプのディストリビューションが適しているか-基本またはプロフェッショナル、OEMまたは小売り、およびそのSKUが何であるか-CoreSingleLanguageまたはCoreCountrySpecific、または別のものをお知らせします。
  • キーに厳密に一致するシステムイメージを探してダウンロードしています 。 これがラップトップの場合、OEMが必要になる可能性が高く、ほとんどの場合CoreSingleLanguage
  • ディスクイメージをダウンロードし、Windows 7 USB / DVDダウンロードツールを使用して（またはこのように手動で-一部のWindows 7 USB / DVDダウンロードツールでは動作しません）、起動可能なUSBフラッシュドライブを作成します。 これまでのところ、その後、フラッシュドライブで特別なことは何もしていません。 または、 Rufusを使用して起動可能なUEFIフラッシュドライブを作成することもできます（さらに改善されます）。
  • 起動後すぐに再起動し、すばやくEscを押します。 フラッシュドライブがWindows 7 USB / DVDダウンロードツールを介して作成された場合、BIOSセットアップを選択し、BIOSのブート設定でセキュアブートをオフにし、レガシーブートを有効にします。それ以外の場合、フラッシュドライブからブートしません。 Rufusを使用してUEFIフラッシュドライブを作成した場合、これを実行しないでください。 次に、フラッシュドライブを挿入します。
  • 起動後すぐに再起動し、すばやくEscを押します。 ブートデバイスを選択し、フラッシュドライブを選択して、システムのインストールを開始します。 すべてが正常に完了している場合-システムは、キーをまったく要求せずにインストールされます。
  • それがうまくいかず、システムがキーをまったく受け入れないか、インストールされているがアクティブにしたくない場合は、インストールフラッシュドライブのsourcesフォルダーに移動して、2つのファイルei.cfgとpid.txtを作成し、上記のように内容を設定します。 マーカーファイルが作成されたフラッシュドライブからシステムをインストールしようとしています。
  • インストール後、システムをアクティベートします-アクティベートされた場合、すべて正常です。そうでない場合は、間違ったディストリビューションを選択します。
