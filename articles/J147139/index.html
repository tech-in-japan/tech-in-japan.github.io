Ask.comの変更またはWordPressプラグインによるサイトマップの生成方法

サイトを更新した後、sitemap.xmlを更新する必要がありました。 このサイトはWordpressを使用しているため、Wordpressをインストールする場合でも、XML Sitemapプラグインがインストールされてアクティブになりました。 プラグインパラメータに移動して、サイトマップの生成を開始しますが、何が表示されますか？



pingテスト



pingの試行： submissions.ask.com/ping ?sitemap=http%3A%2F%2F <検閲によりサイトアドレスが削除されました>％2Fsitemap.xml。 以下のセクションでは、何が起こっているのかを把握してください。



エラー、警告、通知：



通知：WP HTTP API Webリクエストが失敗しました：/ home / <検閲削除されたサイトアドレス> /wp-content/plugins/google-sitemap-generator/sitemap-core.phpのホスト 'submissions.ask.com'を解決できませんでした行2364





submissions.ask.comサイトは実際にはありません。新しいサイトマップを通知するのに役立つ前に、何が起こったのかを探します。



Ask.comから特定のEricの答えを見つけました。



現在、検索インデックスに新しいコンテンツを追加するクロール方法を利用しています。 参照するサブドメインは週末に廃止されたため、まもなくFAQページを更新します。





ask.comロボットはサイトマップの使用を停止しましたが、サブドメインは静かにオフになったことがわかりました。 ask.comはRunetでのWebサイトプロモーションにほとんど意味がないため、このことに関するニュースが2週間でHabrに表示されなかったに違いありません（まだ見つかりませんでしたが、正しいです）。



FAQの更新の約束に反して、submissions.ask.comに関するニュースはask.comで検索されず、問題はサイトマップの使用方法に関する解説に記載されています。



ただし、これはもちろん問題ではありません。プラグイン設定では、ask.comへの情報の報告を単に禁止することができ、エラーメッセージなしで以前と同様に機能します。
