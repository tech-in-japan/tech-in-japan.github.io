Bitrix24：プロジェクト管理のミスを回避する方法

プロジェクト管理が世界を席巻しました-それは偶然ではありません。 このプロジェクトは企業管理の重要な構造であり、その中でタスクが実装され、期限が設定され、リソースが計画されます。 チームと会社が柔軟になり、あらゆる変化に迅速に対応できるようになるため、プロジェクトで有能に作業を構築する人は間違いなく勝利します。 このアプローチの人気にもかかわらず、多くの企業とチームは、プロジェクト管理でミスを犯し続け、紙のグライダーにとどまり、締め切りについてお互いに動機付けをします。 これがなぜ起こるのか、自動化がどのように影響するのかを考えてみましょう。







プロジェクト管理は間違いから始まります



プロジェクト管理は、Habréだけでなく全体に追いやられてきたトピックです。 それにもかかわらず、毎日あらゆる規模のあらゆる企業があらゆるレーキに足を踏み入れ、プロジェクトとタスクを管理しています。 これは、従業員が仕事を透明で論理的に構築するという一般的な混乱と不本意による場合もあれば、会社の自動化コンプレックスに完全に統合され、別の開いているウィンドウやブラウザタブで「ハング」しない適切なツールがない場合もあります。 誰もが少なくとも一度は遭遇した問題のリストを以下に示します。



  • プロジェクトの本質の誤解。 最初に混乱するのは、プロジェクトとプロセスです。 ビジネスプロセスは、従業員が目標を達成するために実行する必要がある所定の周期で繰り返される一連のアクションです。 プロジェクトは、一意の製品、サービス、または結果（PMBOKコードで定義）を作成することを目的とした一時的な企業です。 つまり、プロジェクトには有効期間、開始日、終了日、有限量のリソースがあります。 たとえば、ユーザーへのメール送信はプロセス（同じ手順で繰り返されるアクション）ですが、会社の記念日までにメールを作成することはプロジェクト（1回限り）です。



  • リソースへの不注意は、不当な過剰支出を生み出します。 プロジェクト計画の欠如とタスクへの分割は、不合理なエネルギー（人的資源）、時間、技術的手段の無駄につながります。 そして、これは常に金銭的には過剰なコストであり、最終的には収益性の低下です。



  • 期限に間に合わない。 おそらく、期限が切れていたり、時間通りに仕事を受け取らなかった顧客のinりについてのジョークに誰もがすでにうんざりしています。 一方、時間の浪費は、不適切な仕事の組織化と問題解決のための時間の非識字的な配分から生じます。



  • 会社のプロセス外のタスク。 組織は最高のプランナーおよびプロジェクト管理システムを使用する場合がありますが、CRMや時間追跡システムの外部などで機能する場合、それらは完全に無力です。 さらに、各従業員はいくつかの機能を実行します。タスクを設定するとき、および従業員がプロジェクトに参加するときは、それらすべてを考慮する必要があります。



  • 従業員の過負荷は、前の段落の結果です。 労働者とそのグループの現在の雇用を追跡せず、単にタスクを割り当てた場合、遅かれ早かれチームは作業軸への対処を停止します（ところで、 Bitrix24では、そのような場合に借金の兆候があります-従業員は未解決のタスクのカウンターを見ます）。 ただし、不適切な負荷計画の犠牲になった従業員を刺激するカウンターはありません。



  • サードパーティの専門家に関連するプロジェクトやタスクに対する不注意は致命的です。 オフィスにいる労働者は先験的により制御されています。彼らは起業してビジネスを続けることを保証されていません。あなたはいつでも仕事の進行状況を監視し、影響力と動機を与えます。 リモートの従業員やフリーランサーの場合、事態ははるかに複雑です。 アウトソーシング業者を信頼して、あなたは彼の将来の利益の一部を彼に責め、彼に支払います。それはあなたがプロセスをコントロールする権利を持っていることを覚えておく必要があります。



投稿の最後に、Bitrix24チームは、会社のプロジェクト管理を整理するのに役立つ小さなチェックリストを共有します。



Bitrix24でのプロジェクト管理の実装方法









Bitrix24では、ユーザーは、たとえば会社の組織構造、活動の方向、またはその他の分類に基づいて、グループ（プロジェクト）を個別に作成できます。 グループを作成すると、ユーザーのプロジェクト可視性設定が設定され、グループに必要なサービス（ディスク、カレンダー、メッセージ、フォトギャラリーなど）が登録されます。







また、作成段階で、プロジェクトの実装に関心のある従業員を追加できます。 グループパラメータは後で変更できます。 このようなプロジェクトグループの編成は、外部の請負業者（フリーランサー、専門家、顧客、フリーランサー、ディーラー）と協力する人にとって特に便利です。 別のグループを作成し、1つのプロジェクトのフレームワーク内でそれらを操作することにより、企業情報スペースへのアクセスを制限できます。 タスク管理システムは、ドキュメントおよびカレンダーと統合されています。







タスクはグループ内またはグループ外で設定されます-Bitrix24のプロジェクト管理の主要ユニット。 タスクには多くのフィールドが含まれています-情報が多く入力されるほど、管理の効率が上がり、正しいレポートが作成されます。 そのため、タスクの作成中に、日付と期間（休日と週末を含む）が計画され、リマインダーのルール、タスクの頻度が設定されます。 また、作成の段階で、タスクをガントチャートの前のタスクにリンクし、CRMのリード、会社、連絡先、およびトランザクションをリンクできます。 各タスクには、ディレクター、オブザーバー、コラボレーターがいます。 従業員はタスクバーで、自分が何を助けているのか、どの仕事を割り当てているのかを確認できます。







ガントチャートは、どのシステムでもプロジェクト管理の重要な要素であり、Bitrix24も例外ではありません。 従業員は、タスクを表示、変更、関係を確立し、終了日をグラフ定規上でドラッグできます。 また、マウスの右ボタンのコンテキストメニューから、ダイアグラム内のタスク（延期、一時停止、開始など）を制御できます。







すべてのタスクは、小さなサブタスクを持つチェックリストに分割され、1つのステップで完了することができます。 このような断片化により、プロジェクトの詳細に集中でき、何も見失うことはありませんが、純粋に心理的に最も複雑なプロジェクトが簡単になります-すべての小さなタスクに必要なリソースと時間が少なくなります。 一方、チェックリストに記入すると、徐々に作業が終わります。







反復タスクの場合、テンプレートを作成し、作業の中間結果と最終結果をデータ視覚化を備えたレポートの形式で表示できます。 タスクテンプレートは、ドキュメントの署名、月次レポート、その他の定期的に繰り返される業務を入力できるため、どの会社でも便利です。 これにより、ルーチンの時間が節約されます。







もちろん、 Bitrix24のプロジェクトとタスクは他の機能から分離されておらず、ソーシャルネットワーク、ライブフィード、CRM、チャット、その他のツールと完全にやり取りします。 そして、これは、プロジェクト管理が他のプロセスから独立して行われるのではなく、それらと相互依存関係に関連して発生することを意味します。 したがって、各従業員は自分の仕事のパフォーマンスに優先順位があります。



あなたのビジネスのための約束されたチェックリスト



使用しているツール（Bitrix24など）に関係なく、使用方法は重要です。 繰り返します。特にタスクの完了、特にその結果に続くタスクの入力は、プロフォーマの空のアクションです。タスクは、実際に解決されているのと同じペースでビジネス自動化システムで動作するはずです。 これは基本的なルールです。 しかし、それだけではありません。



  • プロジェクト管理ツールを理解し、そのすべての機能を使用します。 共同プロジェクト管理にソフトウェアを使用し、ガントチャートで作業します。 可能であれば、ビジネスに合わせてプラットフォームを最適化します。 また、チームはツールを使用する必要があります-インタラクティブなポイントで従業員を刺激するか、より簡単に、企業のシステムインターフェースでのみすべての興味深い最新ニュースを報告します。 ところで、 Bitrix24では、各従業員は同僚と比較してシステムを使用して自分の活動を表示でき、マネージャーはすべての部下の係数を確認できます。 これは、ツールの初期開発段階で非常に重要なツールです。



  • 顧客を知る。 プロジェクト管理は会社の活動の一部であり、ほとんどの場合、製品の作成、つまり実際には顧客のニーズを満たすことを目的としています。 相手方を理解することにより、リソースをより正確に割り当てて計画することができます。 そのため、プロジェクト管理システムをCRMと統合するか、両方のモジュール（Bitrix24など）を備えたシステムをすぐに選択することが重要です。



  • 計画：時間、リソース、材料など 明確に定義された実装段階の計画により、期限内の消火ではなく、主な作業に集中することができます。







  • パレートの原則（80/20）に従います。 チームワークの20％が結果の80％をもたらします。 目標を設定するときは、必ず以前の結果を分析し、タスクの最も価値のある20％に集中してください。



  • 前の段落と同時に、 マイクロ管理に焦点を当てないで 、すべてを管理しようと努力しないでください-委任し、従業員が自分の雇用を計画できるようにします。 チームは専門のソフトウェア規律を使用して、すべての従業員が自分の進捗状況がシステムに表示されることを知っています。つまり、欺くことは意味がありません。



  • タスクにチェックリストを使用する -粉砕は、実行可能な小さな手順でプロジェクトの実装にアプローチするのに役立ちます。



  • レポートを使用し、結果と指標を分析します。 これにより、弱点を特定し、タスクやプロジェクトを処理するプロセスを最適化できます。







  • チームからフィードバック （チャット、メール、電話）を取得します 。 これにより、問題が最小限に抑えられ、人的要因の影響が回避されます。



  • プロジェクトのリスクを管理します。実際のリソースと期限を設定し、チームのワークロードを均等に分散し、最後だけでなくプロセス中のタスクの実行を監視します。



プロジェクト管理はビジネスの重要な部分です。 ソフトウェア市場はこれを理解しており、多数の競合するソリューションとユーティリティを提供しています。 ほとんどの場合、これらは単なるタスクマネージャーであり、APIとの連携や有料のリビジョンの注文が必要な統合用です。 Bitrix24のユニークな点は、競合他社とは異なり、追加の操作を行わなくても、会社の業務の多くの側面を複雑に自動化できることです。 これは、何日もシステムをセットアップする代わりに、調整された作業のための時間を増やし、競争の一歩先を行く本当のチャンスがあることを意味します。
