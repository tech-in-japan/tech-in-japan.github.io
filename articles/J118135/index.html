BES 5 SP3は何を準備しますか？

しばらく前に、企業のBlackBerry EnterpriseサーバーにBES 5.0.3（SP3）をインストールしてテストを開始し、本番環境にインストールした後、BESサービスのユーザーが待ち望んでいる新しいことについて少しお話ししたいと思います（5月初旬に焦点を当てます）およそ）。



BES管理者にとって、このサービスパックのリリースは待望のニュースであり、特定の希望があったことを言わなければなりません。 BlackBerry Enterprise Serverなどのサーバー製品の主要な更新には、2つの側面があります。 それらの1つは、サービスのユーザーにとって重要です。これは、新しい機能を追加し、使用可能な機能の機能を向上させ、デバイスサーバーバンドルの全体的な動作に安定性を追加するためです。 2つ目は、サーバー管理者にとって重要です。新しいオプションを管理に導入し、主要サービスの操作を（通常はより良いために）変更し、既に非常にリッチなBESサーバーコンソールに新しい要素と機能を追加します



これらの2つの定数に基づいて、このService Packに伴う変更を強調したいと思います。

非常に好奇心（盛な（そして、一見便利な）機能は、 BlackBerry Balanceと呼ばれます 。 その主な機能について説明します。 多くの場合、 BESサーバーに接続されたBlackBerryスマートフォンが、必要に応じてBESサーバー管理者によってブロックされたり完全にクリアされたりすることは秘密ではありません。 これは、いわゆるリモートワイプオプション （ リモートキルと呼ばれることもあります-スマートフォンで実行するため）です。



そのため、この操作により、電話機のすべてのデータが削除され、同時にブロックされます。 新しいBlackBerry Balance機能を使用すると、個人データから企業データ（実際には主要なビジネス価値を表すことができる）を分離することが可能になります。 したがって、リモートワイプの結果としてカスタムカスタマイズが失われることはありません。 もちろん、スマートフォンを紛失した場合、データをタイプに分けることはあまり意味がありませんが、ユーザーがBESを使用していた会社を辞めた後にBlackBerryデバイスを保存した場合、これは価値があります。



もう1つのエキサイティングな新しい追加は、 MicrosoftのOffice Communications Server（OCS）2007 R2およびLync Server 2010のサポートの導入です。 非常に長い間、多くの企業がOCSサポートの開始を待っており、このサポートがないとBES 5 *への移行を遅らせていました。 Lyncサーバーのサポートがはるかに高速になったことは素晴らしいことです。







これが、 OCS / Lyncクライアントが最終的にBlackBerryデバイスでOCS / Lyncサーバーと通信する方法です。



S / MIMEおよびPGPで暗号化された添付ファイルのサポートが追加され、 BESユーザーのセキュリティがさらに向上しました。

コンソールでのユーザー管理はやや便利になりました。列でソートし、検索結果から直接csvファイルにエクスポートすると、場合によっては非常に役立ちます:)

BlackBerry Enterprise Serverは 、 Microsoft仮想化テクノロジーを優先する企業にとって大きな革新であるMicrosoft Hyper-V Server 2008 R2のサポートを開始しました。

BES 5のExchange 2010との連携は、長らく言えば、これらの製品を管理する管理者にとって頭痛の種でした。 それらの相互運用性は、製品が完全に互換性があるというRIMの発表の後でも、多くの要望が残っていました。 2つの製品の相互作用における最大の問題はBES 5 SP2に修正されましたが、時折、最も古くて厄介な問題、つまりBlackBerryデバイスへのメッセージ配信の遅延が思い浮かびました。 この問題にはいくつかの大きな根本原因があり、最後の原因-BESとの相互作用によるExchange上のメールボックスサーバーのディスクサブシステムの高いアクティビティ-は、現在のサービスパックで修正されました。

これで、Exchange 2010とBES 5の間で*この状態が発生したと言うことができます-







BlackBerry Enterprise Trait Tool-自尊心のあるBES管理者のスイスのナイフ-が変更され、いくつかの新機能が追加されました。 このツールは関心のある人々の間ではあまり知られていないことに注意してください。さらに、一部のBES管理者は、このツールを使用せずに管理することができます。 ただし、このツールは、 Exchange管理者の場合と同じと見なすことができます。たとえば、 PowerShellです。 はい、柔軟性の点では、もちろん一致しませんが、 BESを管理するタスクの一部はこのツールを使用してのみ実行できるという事実に関連しています。







BlackBerry MDS Integration Service （MDS-IS）は廃止されました。既存の環境では誰も使用していなかったため、この点で問題が発生することはありません。

BES第5ブランチのBESサーバーを管理するためのメインツールであるBASサービスには、多数の修正と改善が加えられました。 このニュースは、サーバーを直接管理し、 RIMによって行われた作業に感謝することができる私たちにとって非常に快適で貴重です。

最後に、 BES 5.0.3では、管理性とセキュリティのさらなる向上を目的としたいくつかの新しいITポリシーが導入されています。SharePointへのアクセスに関するネイティブクライアント管理ポリシー（！）はまだ利用できませんが、すぐに来ます。

展開の数日前にサービスのユーザーに必ず通知してください。できれば、私たちのような革新も気に入るはずです！
