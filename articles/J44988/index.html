1.アプリケーションの本質は何ですか？

Xmalは、ゲーム形式でアルゴリズム思考を開発するための自動化されたシステムであり、3Dゲームの形式で開発されます。

ゲーム全体はパート（プログラミングなど）に分割され、パートはチャプター（割り当て演算子など）に分割されます。 各パートの冒頭で講義を聞き、その後、得られた知識を統合して論理的な問題を解決するよう求められます。 タスクを渡すプロセスには、講義で提示された資料の知識が必要です。





論理的な問題の例を示します（代入演算子を通過した後）：

「昼食に鶏肉を調理するには、次のことが必要です。

  1. 鶏を取る
  2. 鶏肉をオーブンに入れます
  3. オーブンの電源を入れます
  4. オーブンで時間を設定する
  5. 設定時間後にオーブンの電源を切ります
  6. 鶏肉をゲット


必須：可変時間を作成し、それに値40を割り当てます



つまり、このタスクを完了するには、オーブンに行き、調理時間を設定する必要があります。

int time = 40;

3.どのプログラミング言語を使用する予定ですか？

プロジェクトを実装するために、C＃プログラミング言語を意味するXNA Frameworkを使用することが決定されました



4. XNA Frameworkはなぜですか？

XNA Frameworkはアンマネージド（クラシック）DirectXのアドオンであり、Visual Studioに便利に統合され、2Dおよび3Dゲームの両方を作成するための十分な機会を提供します。

XNA Frameworkは完全に無料です。 コードをほとんど変更することなく、PC、Xbox、Zune用のゲームを開発できます。 また、Linux（.NET-> Monoなど）でゲームを実行するためのXNAのバージョンを作成する作業が進行中です
