MicrosoftはWindows開発の歴史的側面を明らかにした

数日前、マイクロソフトは私たちが渡すことができなかった非常に興味深い情報を公開しました。 発行された記事は、エンジニアのエンジニアという題名でした。コンピューター業界の著名人は、5年にわたる品質へのDave Cutlerの探求に敬意を表しており、1993年以来、Windowsのすべてのバージョンの基礎を築いたMicrosoft Honored EngineerとWindows NT First Architect Dave Cutlerに捧げられています。







Windows NTはサーバーオペレーティングシステムとして開発されたことが知られており、Windows 2000およびWindows XPの形でホームユーザーに栄光を現すまでに、大きな進歩を遂げました。 実際、最初のバージョンからWindows 10までのWindows NTファミリ全体が、UNIXに基づいたVAX VMSと呼ばれるDigital Equipment Corporationの最初の完全な32ビットオペレーティングシステムの1つに基づいていることを知っている人はあまりいません。



上記の記事の最も重要な抜粋にコメントを提供します



1976年、 Digital Equipment Corporation （DEC） [または単にDigital]は成功の波にあり、成功したコンピューター会社と見なされました。 [そのOSの開発に特化したデジタルだけでなく、そのためのハードウェアも] Data General、Prime Computer、およびいわゆるIT ハイウェイ128（マサチューセッツ州ルート128）、DECは当時のコンピューターランドスケープを形成しました[ミニコンピューターのリリース] 。



同時に、当時よく知られていて、デジタルで高い位置で働いていたエンジニアのゴードン・ベルは 、PDP-11として知られている同社の主力16ビットミニコンピューターの販売動態の低下を心配していました [ RSX-11に取り組んでいます ] 。 DECマネージャーのもう1つの懸念は、IBMの最初の[ micro ]コンピューターの開発でした。



ベルは未来のビジョンを使用して、VAXと呼ばれる新しい32ビットミニコンピューターを新しいOS [ VMS ]で作成し、PDP-11ミニコンピューターのアプリケーションとの下位互換性を提供することにしました。 その後、ベルは、当時33歳だったデビッドカトラーに目を向けました。 さらに彼は、RSX-11の開発経験がある有望なデジタルエンジニアと見なされていました。 BellはCutlerにVAX [Virtual Address Extension]開発プロジェクトをリードするよう依頼しました。



2年後、最初のVAXミニコンピューターがDigitalに光を当てました。 同時に、VAXはこの市場での主導的地位を直ちに強化しました。 その後、ベルはカトラーを「世界で最も優れたオペレーティングシステム開発者」と呼びました。





図 デイブ・カトラー。

1982年、カトラーはベルと会い、DECからの退任を発表しました。 順番に、DECのリーダーシップは、最も才能のあるエンジニアと手を離すつもりはありませんでしたが、その不満は、デジタルリーダーシップの成長する官僚機構に関連していると疑っていました。 カトラーは、拒否できないカウンターオファーを受けました。 そこでDECWestの子会社が設立されました。



新会社のオフィスは、デジタル本社から3,000マイルに位置しています。ワシントン州ベルビュー、つまりシアトルとマイクロソフトからそれほど遠くありません。 若いエンジニアのレン・カウェルは、この会社の最初の才能あるスペシャリストの一人になりました。 専門家が集まってVAXのリアルタイムOSを開発しましたが、その結果、新しいMicroVAXコンピューターと、VAXelnというリアルタイムOSが作成されました。 Cutlerが新しいハードウェアの開発を引き継ぎ、Len Kawellがソフトウェアの作成を担当しました。



Kawell氏は、Cutlerにはハードウェア（ハードウェア）の独自のビジョンがあり、そのことで彼が当時の他のエンジニアと区別されたと述べました。 彼は、マイクロプロセッサのレジスタを使用した機械命令と操作のレベルで考えました。 しかし、1988年にCutlerはDigitalを辞め、彼のための新しいMicroVAXコンピューターとOSは決して日の目を見ませんでした。 その後、カトラーはマイクロソフトに向かい、5年前にビル・ゲイツと会談しました。 新しいOSのプロジェクトはすでに準備ができており、マイクロコンピューターの分野で真の革命を起こすことになっていた。





図 有名な本Windows Internalsの著者であるSolomonとRussinovichのCutler。

1988年にBill Gatesと交渉した後、CutlerはMicrosoftに移行して実際の32ビットマイコンOSを作成することにしました。 また、以前のプロジェクトのエンジニアチーム（OS開発の5人の専門家とハードウェアの6人の専門家）が参加しました。



Cutlerのチームは、1989年4月にMicrosoft向けの新しいOSの開発を開始しました。作業は1日10時間以上、場合によっては週7日行われました。 Windows NTの開発における主なタスクは、将来のOSの移植性でした。つまり、x86アーキテクチャのIntelマイクロプロセッサだけでなく、当時人気がありx86よりも優れたパフォーマンスを提供したRISCアーキテクチャでも動作する可能性でした。 。 同時に、Microsoftの専門家はすでにOS / 2と呼ばれる新しいOS開発プロジェクトでIBMと協力しています。 しかし、OS / 2には明らかな欠陥がありました。1つ目は16ビットで、2つ目は移植性がなく、Intelマイクロプロセッサでしか動作しませんでした。



Windows NTの開発目標は複雑で野心的でした。 Cutlerのチームは、次の要件が満たされていることを確認することでした。



  • MIPS、Alpha、PowerPC、x64など、さまざまなマイクロプロセッサアーキテクチャのサポート（ ソースレベル） 。
  • UNIX POSIX 、OS / 2、Win32を含む他のOS [サブシステムモデル]のアプリケーションの起動のサポート。
  • OSは安全で、C2（ 信頼されたコンピューター認証 ）セキュリティ標準に準拠している必要があります。
  • 同時に動作する複数のマイクロプロセッサ（ マルチプロセッサアーキテクチャ ）および真のマルチタスクのサポート。 これらの機能は両方とも、その時点でPCセグメントに対して一意であると見なされていました。






図 Windows NTロゴ。

開発者が解決しなければならなかった最も難しいタスクの1つは、開発されたOSをテストするタスクでした。 初期の段階では、チームには必要なテストスイートを開発するために必要なすべてのリソースがありませんでした。 代わりに、システム全体に高い負荷をかける動的ストレス耐性テストシステムを選択しました。 毎晩、開発者は数百台のマシンで新しいOSのストレステストを実行しました。 朝、Windowsの作成者がオフィスに来て、後で分析するためにエラーを分類しました。



興味深いのは、最初はメインのWindowsサブシステムとして考えられていたのはWin32ではなく、OS / 2でしたが、Microsoftは後にこのベンチャーを放棄したことです。 [OS / 2の開発自体から]これは、Windows 3.1の人気が急速に高まったためであり、Windows 3.1の売上は6か月で1600万部に達しました。 その結果、16ビットWindows APIがベースとして採用され、32ビットに拡張されました。 新しい32ビットグラフィックサブシステムも、新しいOS用に作成されました。 その後、開発者は、最初に発表されたIntel i860 XRではなく、RISCマイクロプロセッサMIPS R3000にWindows NTを適応させることを余儀なくされました。 解決する必要がある別の問題は、新しいOSと従来のMS-DOSアプリケーションおよび16ビットWindowsとの互換性でした。
