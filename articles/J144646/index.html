パート2. JavaScriptのプロトタイプ。



そのため、javascriptにはプロトタイプの概念、オブジェクトの隠しリンク[[prototype]]、__ proto__、および関数のプロトタイププロパティがあります。 これらの概念の混乱を避けるために、それらを1つずつ分析します。



  • 現在のオブジェクトのプロトタイプは他のオブジェクトと呼ばれ、現在のオブジェクトはそこから欠落しているメソッドとプロパティを引き出します。
  • 隠しリンク[[prototype]]は現在のオブジェクトのプロトタイプを指し、仕様では開発に使用できません。 ただし、仕様に違反する一部のブラウザでは、__ proto__オブジェクトのプロパティとして開かれていますが、直接使用する意味はありません。 なぜなら クロスブラウザではなく、正しくありません。
  • プロトタイプは関数の機能です！ 繰り返しますが、作成されたオブジェクトの[[prototype]]リンクに値を渡すために使用される関数です。




自尊心のあるjavascriptオブジェクトには隠しリンク[[prototype]]があり、親オブジェクトに関連付けられ、親オブジェクトに関連付けられます。 このチェーンの最上部には、組み込みjavascriptオブジェクト、一種の最高の前駆体、toString、valueOf、hasOwnPropertyなどの必要なすべての組み込みメソッドを備えたオブジェクトアダムがあります。 このため、すべての子孫オブジェクトには、最低限必要な一連のメソッドがあり、難しいjavascript環境で生き残ることができます。



thread_object2-[[プロトタイプ]]-> thread_object1-[[プロトタイプ]]-> ... {toString：...、valueOf：...、hasOwnProperty：...、...}

 var obj = {}; //  obj.toString(); //      "[object Object]"




これは、リンクのチェーン[[prototype]]を介して、javascriptでプロトタイプ継承が実装される方法です。 プロトタイプチェーンを通じて利用できるすべてのプロパティは、知識とスキルの倉庫のように子孫に開かれ、実際に進化するクラスのツリーを構築できます！



各オブジェクトのプロトタイププロパティはその中に保存されるのではなく、現在のオブジェクトと組み込みのjavascriptオブジェクトの間のプロトタイプチェーンの中間リンクに「時間の初めに」保存されることに注意してください。 このルートオブジェクトには敬意が必要であり、その平和を乱すことはきちんとしたものではありません。したがって、独自のプロトタイププロパティを作成するには、リンク[[prototype]]を使用して独自のオブジェクトを作成してチェーンに埋め込む方が良いでしょう



しかし、覚えているように、[[prototype]]は閉じたリンクです。どうすればアクセスできずにチェーンを構築できますか？ ここでは、新しいキーワードを備えた既によく知られているConstructor関数と、それ自体に対して非常にオープンなprototypeプロパティが役立ちます。 実際、コンストラクターを介して作成されたオブジェクトは、このコンストラクターのprototypeプロパティで指定された値を持つ[[prototype]]リンクを取得します！ 最初、すべての関数のプロトタイププロパティには、ほとんど空のオブジェクトへの参照があります（関数自体を指す単一のコンストラクタープロパティ）。

 //   "" var Foo = function() {}; //  prototype      Foo.prototype = { hi: 'Hello!', sayHi: function(){ alert( this.hi ) } }; //   "" var obj = new Foo(); //  ,    obj.sayHi();

 function extend(Child, Parent) { var F = function() { } F.prototype = Parent.prototype Child.prototype = new F() Child.prototype.constructor = Child Child.superclass = Parent.prototype }

引数には、コンストラクター、子孫、および親の2つの関数が必要で、既に説明したことを実行します。

  • プロトタイプを転送するためのユーティリティ関数を作成します
  • Parent関数をそのプロトタイププロパティに書き込みます
  • 親のプロトタイプへのリンク[[prototype]]を持つ中間オブジェクト、新しいチェーンリンクを渡します
  • （ユーティリティFの元のコンストラクタの代わりに）コンストラクタに子関数を書き込みます
  • 子孫でオーバーライドされている場合、そのコンストラクターおよび他のソースメソッドを参照できる場合に、スーパークラスプロパティで親への参照を書き込みます。




問題が発生する可能性があります。なぜ最初の3行が必要なのか、新しいF（）なしで代入をすぐにChild.prototype = Parent.prototypeにしないのか、そしてそれで終わりですか？！



実際、この割り当てでは、継承チェーンの新しい中間リンクは作成されません！ Parent.prototypeはChild.prototypeに書き込まれ、Parent.prototypeをさらに参照する中間ストレージオブジェクトではありません。Child.prototypeに何かを書き込もうとすると、親の領域に大まかに侵入し、長老への敬意と世代の継承に違反します。 新しいF（）コンストラクターを呼び出すことで、子孫に渡すことができるプロトタイプの知識を格納するための独自の領域をChildに作成します。



次の方法で、Designerのプロトタイプに個々のプロパティを追加できます。

ところで、プロトタイプをオブジェクトのプロパティ、つまりクラスのインスタンスとして参照しようとしないでください。

オブジェクトにはプロトタイププロパティがありません。非表示リンク[[prototype]]がありますが、プロトタイププロパティにはありません。

もちろん作成することはできますが、継承する意味はありません。 Senseは、作成されたオブジェクトの[[prototype]]リンクにポインターを渡すことができるため、Constructor関数のprototypeプロパティからのみです。



プロトタイプの継承は以上です。 本当に簡単ですか？

ただし、可能なのはプロトタイプの継承だけではありません。 また、スーパークラスのコンストラクターを呼び出す方法、つまり 親クラスの場合、プロトタイププロパティへの書き込みを処理したのは何の理由でもありません（関数extendを参照）。



忘れられた例のTimerコンストラクターでは、これを介していくつかのプロパティをオブジェクトに割り当てます。 これらのプロパティを将来の世代に渡すには、子孫のコンストラクターで親コンストラクターを呼び出す必要があります。つまり、 



 function TimerPlayer() { TimerPlayer.superclass.constructor.apply( this, arguments ); }




this.superclass.constructor.applyを介して、つまり現在のコンストラクター（ここではTimerPlayer）の名前を介して呼び出すことはできないことに注意してください。そうでない場合、親コンストラクターもこれを使用し、this.superclass.constructor.apply（this、引数）、これは子孫としてthisのコンテキストに適用する閉じられた呼び出しに変わり、エラーが発生します。



子孫のコンテキストで親コンストラクタを呼び出すと、すべてのプロパティとメソッドが作成され、子孫に割り当てられます。 さらに、これを介してではなくvarを介して宣言された親のプライベートプロパティは、読み取りを許可する親のパブリックメソッドがある場合にのみアクセスできます。
