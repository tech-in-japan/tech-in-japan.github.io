3CX Phone System v15 SP1の紹介とサーバーではなくミニPCでのシステムのテスト

まず、過去のシステム管理者の日、ブログのすべての読者の皆さん、おめでとうございます！ 特に大規模な組織では、現代のシステム管理者がシグナルマンの機能も実行することに注意してください。これは従来、個々の従業員によって実行されていました。 これは、システム管理者がすぐに仕事を失うことはないという事実を裏付けています。 一方、3CXシステムはZero Adminの原則に従って設計されています。つまり、給与は同じで、仕事量が少なくなります。



休日の機会に、2つの興味深い贈り物を用意しました。3CXv15 SP1のリリースと、小さな本のサイズであるミニPCでのシステムの動作の概要です。



3CX電話システムv15 SP1



約束したとおり、3CX v15のリリース後まもなく、重要なシステム改善を含む最初のサービスパッケージをリリースします。



  • コントロールパネルダッシュボードのアクティブコールモニタリングセクション
  • サーバーとクライアントのインターフェースの完全なローカライズ
  • サーバーインターフェイスからコールレポートを直接ダウンロードする機能（既存の電子メールによる送信を除く）
  • 弱いパスワードを持つ内線番号がインターフェイスに割り当てられるようになりました
  • 他の多くの改善。 完全なchangelogを参照してください。


3CX Client for Windowsのサイドコンソールで独自のBLFボタンを構成することもできます。 この機能は3CXの商用版でのみ利用できますが、現在は無料版のPBX版でテスト用に提供されています。



ダウンロードとドキュメント



すでにv15がインストールされている場合：



  1. サーバー管理インターフェースに移動します
  2. 上部のドロップダウンメニューで、[ 更新 ]をクリックし、[ サービスパック ]を選択します
  3. SP1を選択し、[選択をダウンロード ]をクリックします。 システムはコンソールから切断し、サービスパッケージのダウンロードとインストールのプロセスが開始されます。
  4. ブラウザのキャッシュをクリアしてください！
  5. インストール完了の電子メール通知を受け取ります。 これに先立ち、システムに接続することは推奨されません。

3CX電話システム用のミニPCの使用



組織で実際のユニファイドコミュニケーションシステムを立ち上げることを決めたが、サーバーを購入する予定がない場合は、解決策があります。 3CX電話システムは、80〜100ドル相当の小さな本のサイズのミニPCで実行できます。 小さなクラスのサーバーを置き換えることができるminiPCという新しいクラスのデバイスについてお話します。



miniPC-次世代コンピューター









MiniPCコンピューターはIntelアーキテクチャーを使用します。 従来のデスクトップコンピューターよりも消費電力が60％少なく、約30ワットです。 各国のエネルギー節約量は異なる場合がありますが、ヨーロッパでは、デスクトップコンピューターと比較して、PCあたり年間約150ドルです。 中小企業にとって、このような節約は非常に大きなものです。



MiniPCは、Nexbox、Wintelなどの複数の企業によって製造されています。さまざまな構成のシステムが提供されています。購入する前に、3CXの最小システム要件に注意してください。



  • プロセッサー（Intel Z8300 Quad Core、1.84GHzなど）
  • 64ビットアーキテクチャ
  • 最小2GB RAM
  • インストールされたWindows 10
  • 最小64GBのハードドライブ（おそらくSSDドライブを追加）


テストのために、ミニPC Nexboxを選択しました。 このデバイスは、最新の64ビット14nmクアッドコアIntel Cherry Trail Z8300プロセッサ、ライセンス供与されたWindows 10 Home、4GB RAM、および64GB統合SSDドライブを備えたメディアプレーヤーのサイズです。 下から2本のネジを外すと、ニーズに合わせてより大きなSSDを取り付けることができます。



3CXサーバー用にWindows 10を最適化する



システムを起動した後、推奨事項に従ってWindowsを最適化し、3CXのリソースを最大限に解放してください。



  • Windowsの起動時に起動するアプリケーション（たとえば、Onedrive、Skypeなど）を無効にします。これは一般に、サーバーOSには表示されないすべてのものです。 タスクマネージャーを使用してこれらのアプリケーションのプロセスを無効にすることを忘れないでください（ Control + Shift + Escape> Startup> Disable ）
  • パフォーマンスを改善するには、すべてのエフェクトとアニメーションをオフにします。 sysdm.cpl> Advanced> Performanceを実行し、 Adjust for Best Performanceオプションを設定します。
  • コマンドプロンプトで、 perfmon / reportを実行してシステムパフォーマンスを評価します
  • 検索バーに「設定」と入力して、アプリケーションを実行します。 [ システム] > [ 通知とアクション]に移動し、不要なものをすべてオフにします。
  • 省エネオプションで、1時間後に画面をオフにし、スリープモードへの移行をオフにします。
  • オフラインマップを無効にする
  • デバイスマネージャーでBluetoothモジュールを無効にする
  • Windowsの完全な更新を実行する


3CX電話システムをインストールする



実際、3CXのインストールは、コンピューターの準備よりも時間がかかりません。 3CXインストーラーを実行します。 Visual C ++再頒布可能コンポーネントとともに.NETをインストールします。その後、3CXコンポーネントのインストールは2分以内に完了します。 次に、3CX PBX構成コマンドラインツールコンソールが表示されます。 5分間のセットアップで、システムはすぐに使用可能になります！



ミニPCパフォーマンス評価



いくつかの自動テストでパフォーマンスをテストしました。 通常のコール、IVRコール、キューコール、コールグループ、および接続された3CXソフトウェアクライアントをテストしました。



  • テストでは、ランダムな順序で3分間と15分間の通話が行われ、切断されて再度行われました。
  • 異なる権限で接続されたクライアント。 権限が多いほど、クライアントとサーバーの間で送信されるパケットが多くなります。たとえば、キュ​​ーマネージャーは、通常のユーザーよりも多くのデータを送受信します。
