Windows 8 Proの承認の問題

数時間前、私は問題に遭遇しました。パスワードでWindows 8 Proにアクセスできませんでした。



そんな感じでした。

最近、ラップトップにWindows 8をインストールし（Windows Upgrade OfferプログラムでWindows 7から更新しました）、新しくインストールしたOSのセットアップに関与しました。 管理者権限を持つアカウントを1つ作成し、Microsoftアカウントに関連付けました。

今日、ラップトップは突然リブートし（それが何であるかわかりません）、ブート時にいくつかの定期的な更新をインストールしました。 その後、私はログインしようとしましたが、メッセージを受け取りました：

無効なパスワード。 必ずMicrosoftアカウントのパスワードを使用してください。 いつでもaccount.live.com/password/resetにリセットできます。


英語の同じメッセージ：

そのパスワードは間違っています。 Microsoftアカウントのパスワードを使用していることを確認してください。 account.live.com/password/resetでいつでもリセットできます。


account.live.comでパスワードを変更しましたが、システムにアクセスできませんでした。 他のユーザーがいなかったため、ラップトップへのアクセスが完全に失われました。



別のコンピューターがあるのは良いことです。インターネットで解決策を探し始めました。 ロシア語では何も見つかりませんでした。 しかし、英語では、同じ問題に遭遇した人々から多くの質問が見つかりました。私が会った最小日付は10月27日でした。 質問があるにもかかわらず、特定の答えは見つかりませんでした。



解決策


彼は自分で行動し始めました。

配布キットをインストールする前に、Windows 8はディスクに書き込みました。 このディスクから起動し、インストールする代わりに「システムの復元」を選択します。 次に、「診断」->「詳細設定」->「システムの復元」。 最後の復旧ポイントを選択しましたが、何らかの理由で復旧に失敗しました。 その後、彼はすべてのアクションを繰り返し、最後に最後から2番目のポイントにロールバックしました。 この回復は成功し、再起動後にログインできました。

リカバリポイントは数日前に作成されたため、その時点で行われたすべての設定は失われ、インストールされたプログラムは消えました。



おそらく他の解決策があるかもしれません。たとえば、 ここではセーフモードで起動してそこにログインでき、その後は通常モードでの認証が機能しているように見える、 と書かれています。 しかし、セーフモードに入るのはそれほど簡単ではないため、このオプションはチェックしませんでした。



追加1 ：

セーフモードで起動する方法を見つけました（非常に単純ではありませんが）。 メソッドは、この投稿のデータに基づいています。

DVDから起動し、システム回復を選択して、コンソールを起動します。

コンソールで、コマンドを入力します

bcdedit


利用可能なダウンロードオプションが表示されます。 私たちはそれらのシステムを探しています。

次に、システムブート設定のコピーを作成します。

bcdedit / copy {システムID} / d「設定のコピーの説明」


再起動します。 ダウンロード時には、ダウンロードオプションを選択するためのメニューが表示されます。

メニューの下には、詳細設定に移動するためのリンクがあります。これらのリンクを使用して、セーフモードでダウンロードできます。 これらのメニューのスクリーンショットはこちらでご覧いただけます 。

このように複雑な方法は、私が行ったようにシステムにまったくログインできない場合に関係があります。そうでなければ、同じメッセージで説明されているより簡単なオプションを使用できます 。



追加2 ：

おそらく、 コメントで mstyuraが提案する最も正しい方法は、 ビルトイン管理者アカウントをアクティブにすることです。

彼は配布ディスクも必要です。 スクリーンショットで管理者をアクティベートする方法は、 ここで説明されています 。 つまり、手順は次のとおりです。

  1. DVDから起動します。
  2. システム回復を選択します。
  3. コンソールを起動します。
  4. コンソールから、 regedit



    はレジストリエディタを起動します。
  5. レジストリで、 HKEY_LOCAL_MACHINE



    ブッシュを選択しHKEY_LOCAL_MACHINE



    。
  6. ファイルメニューで、「ブッシュをダウンロード...」を選択します。
  7. システムのディスク（私の場合はE：ドライブでしたが、WindowsはCにインストールされていますが:)ファイル "％windir％\ system32 \ config \ SAM"を見つけます（％windir％がシステムを指しているため、パスだけを置き換えることはできません） DVD）。
  8. エディターは茂みの名前を尋ねます。「テスト」と呼びましょう。
  9. 「HKEY_LOCAL_MACHINE \ test \ SAM \ Domains \ Account \ Users \ 000001F4」に移動します。
  10. 値「F」を編集するために開きます。
  11. オフセット0038で、11を10に置き換えます。
  12. ブッシュの「テスト」をクリックし、「ブッシュのアンロード...」を選択します。
  13. 再起動します。
  14. ユーザー「管理者」が表示されます。 私たちはその下に入りますが、正直なところ、次に何をすべきかわかりません。どういうわけか、パスワードをリセットするか、liveIDからアカウントを解放する必要があります。
