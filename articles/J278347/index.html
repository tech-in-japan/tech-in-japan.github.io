Oracle IaaSおよびPaaS-あなたのためのすべて

今日のクラウドコンピューティングは、IT市場全体に非常に重要です。 大手IT企業はこの分野に大きな展望を抱いており、IDCやForresterなどの分析機関は、仮想化、サービスとしてのプラットフォーム、サービスとしてのアプリケーションなどの分野におけるクラウドコンピューティング市場の成長を予測しています。彼らはこの分野で最も深刻なリソースを投資し、積極的に開発しています。つまり、クラウドテクノロジーを、近い将来におけるITおよびIT製品の開発の主要な分野の1つとして非常によく扱うことができます。



もちろん、Oracleはクラウドオートメーションにも多額の投資を行っています。 さらに、主要なグローバルサプライヤが提供するクラウドソリューションの図を見ると（図1）、HWaaS（サービスとしての機器）を含むクラウドソリューションの完全なスタックを提供しているのはオラクルだけであることがわかります。 しかし、私たちの国にとってクラウドコンピューティングはどの程度関連していますか？



この質問に対する私たちの答えは、クラウドを含む私たちの国のために解決されるべきタスクのリストが関連していると信じるかどうかに依存します、すなわち：

  • 拡張性
  • コスト削減
  • ITサービスの速度の改善
  • サービスの簡素化
  • 技術アップデート
  • 設備投資から運用コストへの移行
  • セキュリティ強化
  • 標準化


これらすべてのタスクがロシアに関連することは明らかです。さらに、最近の特定の経済状況により、特に関連するのはそれらを解決するクラウドベースのアプローチです。 そして、クラウドテクノロジーの使用がビジネスの正確に誰に有益であるかを質問すると、ほとんどすべての人に有益であることがわかります。 提供されるサービスの品質と新しいサービスの提供の速度を第一に考えている開発者は、クラウドのおかげで最新かつ絶えず更新されるバージョンのテクノロジーにすばやくアクセスできることが重要です。 IT部門の責任者は、迅速な対応、サービスの品質、リスクとコストの削減、生産性の向上など、クラウドプラットフォームの有機特性に関心を持っています。 最後に、クラウドシステムの高品質と高速性は、ビジネスユーザーに新しい市場へのアクセス、新製品の迅速な開発、企業ITへの普遍的なアクセス、使いやすさを提供します。



クラウドを展開するには、主に2つの方法があります-パブリッククラウド（クラウドサービスと大規模なデータセンターのリソースから多数のクライアントにドキュメントを保存する領域を提供する）とプライベートクラウド（組織独自のクラウドインフラストラクチャ）。 パブリッククラウドを使用すると、関連するすべてのITコストを運用コストに変換でき、ITソリューションの迅速かつ安価な立ち上げが可能になります。 プライベートクラウドの場合、設備投資は節約されますが、ITインフラストラクチャの完全な制御も維持されます。



Oracleのアプローチは、プライベートクラウドとパブリッククラウドの組み合わせであるハイブリッドクラウドの概念に基づいており、プライベートクラウドとパブリッククラウドの間にデータとアプリケーションの相互制御された使用を実装します。 ハイブリッドクラウドは、開発とテスト、B2Bソリューションの統合、ITリソースを必要とする製品の実装、新製品の試用に非常に便利です。



オラクルは、パブリッククラウド、プライベートクラウド、ハイブリッドクラウドのすべてのクラウドモデルから選択できます（図2）。 一方では、データセンターにOracleシステムを展開し、プライベートクラウドに変えることができる製品が提供されています。 一方、世界中にパブリッククラウドサービスを提供するOracleデータセンターがあります。 Oracle Enterprise Managerでは、両方のアプローチを組み合わせて、1つのクラウドを別のクラウドに接続し、単一のインタフェースを使用してそれらを管理できます。



クラウドはサーバー上の特定の数のファイルだけではないことを思い出してください。 クラウドは、サービスの提供、サービスの管理、支払いが自動化されたときのクラウドです。 それが、プライベートクラウドがITシステムの自動化、展開、管理であると言う理由です。 大規模で複雑なITインフラストラクチャを持つ企業は、これを特によく理解しています。自動化せずにこのようなインフラストラクチャを管理することは不可能であるためです。



現在、世界で最も人気があり、ロシアでは、Oracleクラウドサービスは、パブリッククラウドのインフラストラクチャおよびプラットフォームサービスです。サービスとしてのデータベースとサービスとしてのアプリケーションサーバーです。



インフラストラクチャサービスとプラットフォーム



Oracle PaaS（DBaaS）サービスを提供する主な2つの方法は次のとおりです。クライアントはOracle VMまたはExadataサーバーのいずれかで動作します。 クライアントが従来の仮想マシンで動作する場合、それらの仮想マシンにはいくつかの事前構成されたサイズがあり、それらは特定の数のプロセッサとメモリサイズによって特徴付けられます。 Exadataサーバーベースのソリューションは、データベースのサイズとパフォーマンスに対する要件が非常に高い顧客向けに設計されています。 さらに、高負荷にExadataセルを使用することが可能になります。 詳細はタブをご覧ください。 1。







新しいデータベースの迅速な展開は、便利なセルフサービスポータル、一連の展開手順をサービステンプレートの形式で保存し、さまざまなDBMSバージョン、構成などに対してさまざまなテンプレートオプションを提供するテンプレートカタログのおかげで可能になります。サービスの可用性を調和的かつ迅速に導く、重要で技術的に複雑なプロセスが多数あります（図3）。 サービス準備のすべての段階で、Oracleが責任を負います。



データベースは迅速に提供され、保証されています。パッチと自動バックアップをさらに自動インストールする機会があります。サービス所有者自身がクラウドストレージのバックアップを実行する頻度と領域を決定します。 データベースには11と12の2つのバージョンがあり、Linuxバージョン6.4に展開されています。 すべてのサービスは、さまざまなプログラミング言語で動作するように統合されています。 セキュリティを確保するために、SSHキー（リターンプライベートキーを持つシステムのみへのアクセス）、VPNチャネルの使用、およびOracle独自のデータベースセキュリティ機能を使用するために、ツールセット全体が使用されます。



現在、これらのリソースに課金するための2つのオプションがあります-時間単位または月単位の支払い。 時間あたりの料金は高くなりますが、システムのアクティビティが不要な夜間、週末、休日に節約する必要があるお客様に適しています。



データベースサービスと共に、いくつかの追加サービスが機能します。たとえば、管理者がグラフィカルインターフェイスを介してデータベースを管理できるEnterprise Manager Database Express製品などです。 APEXアプリケーションを使用するためのサービス。 「Light」アプリケーションサーバーGlassFishなど



クラウドへのバックアップサービスは、クラウドからクラウドへのバックアップと、ローカルデータベースのパブリックOracleクラウドへのバックアップの両方に使用できます。 従来のRMANユーティリティで動作し、さまざまなバージョンのデータベースで実行されます。 Oracle Database 10以降、この機能はEnterprise EditionとStandart Editionの両方で使用できます。 デフォルトでは、バックアップはデータを圧縮および暗号化します。 アクセシビリティを向上させるために、トリプルデータミラーリング、自動エラー修正、および劣化に対する保護が使用されます。



ハイブリッドクラウド



Enterprise Manager 13は、単一のインターフェイスからパブリッククラウドとプライベートクラウドの両方を管理するシステムです。 データベースおよびアプリケーションを操作する際の負荷がデータセンター内の無料サーバーの能力を超える場合、この負荷はパブリッククラウドに転送できます。 Enterprise Managerは2つの主な機能を実行します。サービスの場所を監視することと、クローンを管理することです（図4）。



Enterprise Managerをインストールした後、Enterprise Managerエージェントの1つを設定して、セキュアコネクションを介してパブリッククラウドにあることを「認識」し、パブリッククラウドの対応するエージェントがローカルエージェントと通信できるようにする必要があります。 エージェントをインストールするときに、インストールするオペレーティングシステムを選択します。 パブリッククラウドでホストを接続することにより、監視用のデータベースを追加します。EnterpriseManagerはこのデータベースを「参照」し、管理できるようになります。



クラウドに1つのデータベースがあり、データセンターに別のデータベースがある場合は、クローン作成プロセスを開始できます。 プラグ可能なデータベースのクローン作成は十分に高速です。 つまり、実際には、接続されたデータベースのファイルとxml構成ファイルがコピーされます。 クラウド内外でのクローニングが可能です。 この機能は、バージョン12Cリリース5以降のOracle Enterprise Managerで使用できます。



同様に、Javaアプリケーションでの作業が実装されます。データベースとWebLogicサーバーを接続することにより、プライベートクラウドまたはパブリッククラウドでアプリケーションを実行できます。



Enterprise Managerは、スケーラビリティと測定機能を備えたソフトウェアおよびインフラストラクチャへのセルフサービスのオンデマンドアクセスを提供することにより、次の利点を提供します。

  • サービス品質の向上。 IT組織は、コストを削減するだけでなく、生産性、可用性、およびセキュリティを向上させるソリューションを見つけるよう努めています。 クラウドの顧客は、クラウドに組み込まれた高可用性機能の恩恵を受けます。
  • リソースの使用における柔軟性。 データベースの容量（ストレージサイズと計算能力）を増減できるため、アプリケーションは柔軟になり、変化するワークロードに容易に適応できます。
  • リクエストに応じてシステムを迅速に提供します。 クラウド内のデータベースは非常に迅速に作成できます。 これにより、実稼働アプリケーションと開発プラットフォームの全体的な展開時間が短縮され、デバッグ構成の作成が高速化されます。
  • レポート-クラウド内のデータベースリソースの使用を定量的に測定し、リソースの使用に応じて管理リソースの予算、計画、および配布を行う機能。




結論として、私はオラクルだけが独自のソリューションのこのような大規模な選択を備えたクラウドを提供していることをもう一度強調したいと思います。 同時に、オラクルの提供は、データセンターとパブリッククラウドの単一のアーキテクチャ、標準と製品、ライフサイクル管理タスクの自動化、データセンターとOracleパブリッククラウド、およびPaaSアプリケーションとSaaSアプリケーション間の容易な統合によって区別されます。
