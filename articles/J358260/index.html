UnityでUIテキストを操作する4つの方法

2D Toolkitに基づいた古いUnityコードを純粋なUnityコードに変換するプロセスで、私は問題に遭遇しました：Unityは標準フォント形式の素晴らしいサポートですが、tk2dのスプライトシートからフォントを作成するサポートと比較するにはまだ十分ではありません。









スプライトフォントの例



実際、これはそれほど深刻な問題ではありません。結局、既製のフォントを挿入する方が簡単で論理的ですが、手書きの碑文に似たスタイルを維持したかったのです。



そこで、UIテキストを操作するときにUnityが提供するさまざまなオプションのカタログ化を開始しました（最近取得したUnityと組み込みのTextMesh Proバージョン2018.1を含む）。 私のタイポグラフィの知識はかなり狭いですが（このトピックは非常に複雑に思えます）、この記事では、可能性がどのように存在し、どのように使用できるかを理解できます。



標準Unityフォントアセット











.ttfおよび.otfフォントファイルに対する標準のUnityサポートは、ゲームでテキストを実装する最も簡単で最も一般的な方法です。



内部は動的に生成されたスプライトフォントのように見えます。 Unityは、指定されたフォントサイズのフォントからテクスチャを作成します。



ソース：フォントは、.ttfまたは.otfファイルから自動的に生成されます。



使用法： UI Textコンポーネントのみ



スケーリングオプション：テキストはUI Textコンポーネントで自由にスケーリングできます。 フォント自体をスケーリングすると、フォントから生成されるテクスチャのサイズが大きくなり、結果がより明確になります。



長所/短所：使いやすいが、インポートされたフォントのみがサポートされています。



詳細： Unityフォントのドキュメント



Unityカスタムフォント









Unityには、カスタムスプライトフォントを作成する機能がありますが、それらを拡大縮小する機能は制限されています。



ソース：カスタムフォントは、マテリアル（テクスチャを参照）とキャラクターテーブルから作成されます。



キャラクターテーブルは少し複雑に見えます（ただし、UV座標を処理するよりも簡単だと思います）。 それもそうです。 スプライトシート自体から生成するためのGUIツールはありません。 各キャラクターには次のプロパティがあります。



  • インデックス ：ASCII文字インデックス
  • UVテクスチャ座標 ：0〜1の範囲で、テクスチャの幅と高さの割合を示します
  • Vert ：ピクセル座標
  • Advance ：次の文字をレンダリングする前のピクセル単位のステップ、値が大きいほど、文字間のスペースが多くなります。


スケーリングオプション：スケーリングはカスタムフォントの弱点のようです。 私が見たものから判断すると、これらのフォントはTextコンポーネントのFont Sizeプロパティを無視し、インポートされたテクスチャのサイズによって制限されます。



Textコンポーネントを含むゲームオブジェクトのスケールを設定できます。 ただし、これにより要素の境界が変更されるため、異なる要素を整列させる場合はかなり不便です。



使用法： UI Textコンポーネントのみ



長所/短所： Unityのスプライトフォントのネイティブサポートですが、サイズを変更するにはスケーリングを使用する必要があります。 シンボルテーブルを生成するツールはありません。 手動で入力する必要があります。



詳細： Unityフォントのドキュメント



TextMesh Proフォントアセット









Unityとは異なり、TextMesh Proにはテキストファイルとスプライトフォントの単一の形式があり、その動作は両方のタイプのフォントでほぼ同じです。



TextMesh Proフォントの欠点は、TextMesh Pro UIコンポーネントでのみ使用できることです。 TextMesh Proを使用する理由があると思われる場合は、プロジェクトの初期段階でこの決定を行い、プロジェクト全体で常にそれを順守することをお勧めします。 標準のUI Textコンポーネントで記述された完成したプロジェクトを作り直すのは、骨の折れる作業です。



ソース： TextMesh Proフォントリソースは、Custom Fonts Unityのように、MaterialテーブルとCharacterテーブルから作成されます。



文字テーブルはピクセル座標でのみ指定され、UVでは指定されないため、任意のUnityフォントよりも簡単で正確です。 さらに、フォントファイルからTextMesh Proフォントリソースを作成するFont Asset Creatorツールがあります。 ただし、スプライトフォントの場合、プロセスはまだかなり遅いです。









スケーリングオプション：フォントサイズを変更し、ゲームオブジェクトをスケーリングする必要なく、TextMesh Pro UIコンポーネントでTextMesh Proフォントをスケーリングできます。 このため、スプライトフォントを使用する必要がある場合は、ネイティブUnityテキストよりもTextMesh Proを使用します。



使用法： TextMesh Pro-テキストUIコンポーネントのみ



長所/短所：フォントリソースやUnityスプライトフォントよりも柔軟ですが、独自のTextMesh Pro UI Textコンポーネントが必要です。 スプライトシートからシンボルテーブルを作成するツールはありません。手動で行う必要があります。



詳細： TextMesh Proフォントのドキュメント



TextMesh Proスプライトアセット



TextMesh Proスプライトリソースは、このリストでは少し不適当です。実際、他の3つのタイプと同じ意味でのフォントリソースではありません。 むしろ、TextMesh Pro-Textコンポーネントによってユーザーに提供される追加機能です。



スプライトリソースは、標準のテキストとゲーム内のシンボルまたはアイコンを混在させる問題を解決します（例として、Final Fantasyインベントリ内で使用されるアイテムシンボルを使用できます）。









アプリケーション： TextMesh Proコンポーネント-テキストUI。 コンポーネントごとに、1つのTMPフォントリソースと1つのTMPスプライトリソースを割り当てることができます。



テキスト内のスプライトアイコンを参照するには、 <sprite index =＃>タグを使用します（＃は0から始まるスプ​​ライトインデックスです）。



ソース： TextMesh Proスプライトアセットは、マテリアルテーブルとキャラクターテーブルから作成されます。 概念的には、TextMesh Proのフォントリソースに近いものです。 Sprite Importerは、FNTファイルを使用してスプライトシートシンボルテーブルを生成できるため、Font Asset Creatorよりもわずかに優れています。 （次のセクションのFNTファイルに関する注意を参照してください。）



長所/短所：なし。この方法は、実際にはTextMesh Proを使用することの副次的な利点です。 何らかの理由でプロジェクトでこの機能を使用する場合。 できるだけ早くTextMesh Proの使用を開始することをお勧めします。



詳細： TextMesh Proスプライトのドキュメント



FNTファイルからカスタムフォントとTextMesh Proフォントリソースを生成する



これ自体が別の投稿のトピックになる可能性があります。これにより、カスタムフォントとフォントリソースの作成がTextMesh Proの単調さを大幅に軽減するため、言及する価値があります。



（UnityツールまたはTextMesh Proフォントリソースを使用して）スプライトフォントを作成する主な欠点は、スプライトシートから文字を定義するためのGUIツールがないことです。 実際、大量の数字を手動で操作し、フォントをテストしてからもう一度繰り返す必要がありますが、これは非常に単調なプロセスです。



しかし、良いニュースがあります-スプライトフォントを作成するための多くの GUIツールで使用される、このような情報には多かれ少なかれ標準的なテキスト形式があります。 （私自身も、FNT仕様を部分的にサポートする簡易ユーティリティを作成しました。）



悪いニュースは、カスタムフォントUnityおよびTextMesh Proフォントリソースがデフォルトでサポートしていないことです。



ただし、Unityは、プロジェクト内の生ファイルを読み取り、コードで使用されるリソースに変換できるリソースポストプロセッサーの概念をサポートしています。 リソースポストプロセッサは、リソースのインポートおよび再インポート中に実行されます。



非常にシンプルなFNT-to-TextMesh Proフォントアセットコンバーターを作成しました 。 例として使用できます。 目的に十分合ったコンバーターを作成できれば、スプライトフォントを作成するタスクをより効果的なツールに転送できるため、時間を節約できます。
