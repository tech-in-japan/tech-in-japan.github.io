シンプルで強力、永遠に。 Elecard CodecWorks 990-AVCおよびHEVC用のライブトランスコーダーソフトウェア





前回の記事では、第6世代のSkylakeプロセッサーでQuick Sync Video（QSV）ビデオのエンコードとデコードのハードウェアアクセラレーションがどのように機能するかを調べました。 ビデオ処理のみを扱う特殊な集積回路に対応するために、SoCの小さなセクションが特別に割り当てられました。 ハードウェアサポートを使用したビデオの再生は、OSの他のタスクからリソースを奪うことが少なくなり、CPUの過熱が少なくなり、電力の消費が少なくなりました。 たとえば、QSVのおかげで、MacBook Airラップトップは1回のバッテリー充電で10時間以上のビデオを再生します。



同時に、ハードウェアエンコーディングアクセラレーションは、サードパーティアプリケーションで利用可能になりました。 最新世代のIntel Coreプロセッサーでは、ハードウェアトランスコーディングのサポートがAVC / H.264とHEVC / H.265（およびVP8とVP9）に登場しました。これは、インターネット上のデジタルテレビとビデオ放送の市場に真の革命をもたらします。 通常のユーザーは、ビデオブロードキャストの品質を向上させ、価格を下げ、利用可能なデジタルTVチャンネルの数を増やすことを期待する権利があります。



プログラムでハードウェアアクセラレーションを使用する方法は、タスクによって異なります。 Intelは、Windows Media SDK 2.0用のアプリケーションを開発するためのオープンアクセスAPIを公開しています。 エンタープライズアプリケーションIntel Media Server Studioを開発するためのクロスプラットフォームAPIもあります 。 これらのAPIの操作の詳細については、HabrのIntelの企業ブログを参照してください。



QSVハードウェアアクセラレーションを使用するプログラム



Intel SDKを介したハードウェアQSVトランスコーダーへのアクセスは、 多くのアプリケーションに実装されています 。



  • ソフトウェアエンコーダー;
  • メディアコンバーター;
  • ビデオおよび写真編集者;
  • ウェブカメラ用のソフトウェア。
  • インスタントメッセンジャー;
  • メディアプレーヤー。


さらに、QSmpはFFmpegパッケージの一部であるlibavcodecフリーエンコーディングおよびデコーディングライブラリによってサポートされています。 これは非常に人気のあるオープンソースライブラリです。 そのため、Intelのハードウェアアクセラレーションは、FFmpegを使用するすべてのプログラムに含まれています。



ご覧のとおり、今日のアプリケーションのクラスは非常に広く、専門家や一般ユーザーの間で人気のある多くのアプリケーションが含まれています。



IntelのSDK（ Media Server Studio 2017またはMedia SDK ）は、特に、ビデオ会議、4Kまでの高解像度ビデオ、高性能エンコーディング、インターネットテレビなどを含むさまざまなタイプのプロジェクトのビデオエンコーディングの問題を解決するアプリケーションの開発を目的としています。 。



Intelのビデオを操作するためのハードウェアサポートが出現する前は、エンコードは次の3つのタイプのいずれかのソリューションを使用しています（現在も使用されています）。



  • ビデオ専用のグラフィックカード（NVidiaやAMDなど）を使用して、ビデオの処理を高速化するソフトウェア。
  • 中央処理装置のリソースのみを使用するソフトウェア。
  • チップ上のハードウェアエンコーダー、いわゆるSystem-On-Chip。


Intelの決定により、既製のものを使用するか、SDKを使用してアプリケーションを開発することにより、Intelプロセッサーを搭載した通常の予算の家庭用コンピューターまたはサーバーでビデオをエンコードできるようになりました。



既製のアプリケーションはプログラマーの参加を必要とせず、実際のタスクで使用する準備ができていることにすぐ注意してください。 たとえば、インターネットTVで最も人気のあるトランスコーディングプロファイルが含まれており、プログラマでないエンジニアにとっては、SDKを使用するよりもはるかに簡単に作業できます。



したがって、独自のトランスコーディングシステムを実行するには、次の2つが必要です。



  1. ソフトウェアトランスコーダー;
  2. QSVをサポートするIntelプロセッサーを搭載したコンピューター-AVC / H.264の第5世代Intel CoreまたはHEVC / H.265の第6世代Intel Core または同様の第4世代または第5世代のXeon E3サーバープロセッサ。

高速コーディングAVCとHEVCを提供するものは何ですか？



インターネットおよびHEVC / H.265などのビデオ圧縮アルゴリズムの開発により、ビデオトランスコーディングはますます増加しているタスクになりつつあります。 デジタルビデオは、モバイルネットワーク、ビデオ会議、ビデオ監視システム、HDおよび4Kでのテレビ放送を含む、インターネット上のテレビやライブ放送など、あらゆる場所に存在しています。 おそらく、インターネットを介したデジタルテレビの分野で最大の成長が期待されています。 HEVC / H.265コーデックへの移行により最も顕著な変化が予想されるのはここです。今日の実際のタスクでは、同じ品質のAVC / H.264と比較して20〜35％優れたビデオ圧縮度を示しています。 また、コーデックのアルゴリズムの複雑さにより、HEVC / H.265のソフトウェアとハ​​ードウェアの実装が引き続き積極的に改善されていることを追加する価値があります。



IntelプロセッサでハードウェアをサポートするHEVC / H.265の高速トランスコーディングが2015年に登場しました（Skylake）。 実際、このコーデックで圧縮されたビデオは、大量配信が始まったばかりです。



理論的には、同じビデオ品質で、HEVCメインプロファイルは、H.264 / MPEG-4 AVCハイプロファイルよりも35.4％低いビットレートを生成します。 MPEG-4 Advanced Simple Profileと比較した場合のゲインは63.7％（ IEEEデータ ）です。 これは、トラフィックを半分以上減らすことを意味します。 同様に、AVC HPはMPEG-4 ASPを約44.5％で同じ品質でアウトパフォームします。



より効率的な圧縮により、デジタルTVプロバイダーは同じチャンネルでより多くのサブスクライバーにサービスを提供し、各サブスクライバーにより多くのチャンネルをブロードキャストし、より高い解像度と品質でチャンネルをブロードキャストできます。 1920×1280の解像度の高解像度テレビは一般的になり、4K UHDTVはすぐにエキゾチックではなくなります。





4K、2K、およびHDTVの解像度の比較



インターネット経由で4K UHDTV信号を配信する場合、標準ではHEVC / H.265ビデオコーデックが定義されています。これにより、より高い解像度で最大の圧縮ゲインを実現できます。 2014年に初めて 、4K UHDTV解像度で放送され、HEVCコーデックでビデオを圧縮する最初のテレビチャンネルが登場しました。 ただし、この解像度のテレビはまだ少数の加入者が利用可能であり、標準的なロシアのアパートは、数メートルの距離から大きなUHDTV画面でテレビを快適に見るのにあまり適していません。
