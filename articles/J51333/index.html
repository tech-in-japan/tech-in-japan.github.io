2. HTMLのバージョンは何ですか？



HTMLの最初のバージョン（1989）にはバージョン番号がありませんでした。 それはただのHTMLでした。 1995年にInternet Engineering Task Force（IETF）によってリリースされたHTMLの最初の標準化されたバージョンは、HTML 2.0と呼ばれていました。



WWWコンソーシアム（W3C）は後に設立されました。 彼は1997年に最初の標準化バージョンであるHTML 3.2を発表しました。 その後継であるHTML 4.0は1998年にリリースされ、1999年にすぐにHTML 4.01に変更されました。これはHTMLの最新の最新バージョンです。 W3Cは、HTMLの将来のバージョンをリリースしないことを発表しました。 HTMLドキュメントの作成にはHTML 4.01をお勧めします。



それにも関わらず、 Web Hypertext Application Technology Working Group （WHATWG）は、最終的に推奨されるW3Cになることを期待して、いわゆるHTML5に取り組んでいます。



5. DOCTYPE宣言は何をしますか？



文書内のマークアップの前にあるDOCTYPE宣言は、通常次のようになります。



<!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01//EN"

"http://www.w3.org/TR/html4/strict.dtd">







ドキュメントのルート要素のタイプ（<html>）、パブリック識別子、およびシステム識別子を定義します。



パブリック識別子（ -// W3C // DTD HTML 4.01 // EN ）は、誰がドキュメントタイプ宣言（DTD）（ W3C ）を使用したかを示します。 DTD名（ DTD HTML 4.01 ）; およびDTDが記述されている言語（ EN 、つまり英語）。 DTDはWebページ自体の言語を示していないことに注意してください。 DTDがコンパイルされる言語を示します。



システム識別子（ www.w3.org/TR/html4/strict.dtd ）は、使用されるDTDへのパスです。



DOCTYPE宣言は、ドキュメントをチェックするDTDのバリデーター（Webページの構文の「正確性」をチェックするプログラム）に指示します。 ブラウザーは、DOCTYPEがドキュメントで宣言されているかどうかを心配する必要はありませんが、現代のブラウザーはこの宣言を使用して、ページが「モダン」（ブラウザーが仕様に従ってページを処理する）か「古い方法で」（ブラウザーがページを処理する）かを判断します古いブラウザのバグを考慮に入れて）。 DOCTYPEは、Internet Explorer、Opera、Firefox（およびその他のMozilla形のブラウザー）、Safari、およびその他の最新ブラウザーのページレンダリング方法に影響します。 システム識別子を含む完全なDOCTYPE宣言は、ブラウザに最新のドキュメントであることを伝えます。 システム識別子がない場合、またはDOCTYPE宣言がまったくない場合、ブラウザはそれを「古い方法」で記述されたページと見なし、 「quirksモード」（quirkyモード）で処理します。



7. Strict、Transitional、Frameset DTDの違いは何ですか？



これらのDTDの違いは、宣言する要素と属性、および要素のネストの観察を許可（義務）する方法にあります。

  • HTML 4.01 Strict DTD-プレゼンテーションと動作からのコンテンツの分離を強調します。 このDTD W3Cは、すべての新しいドキュメントに推奨しています。
  • HTML 4.01 Transitional DTD-「古い」（old-scool'noy、HTMLへ）マークアップからモダンへの移行における一種の中間リンクです。 新しいドキュメントを書くときはお勧めしません。 これには、11個のプレゼンテーション要素（翻訳者のメモ：セマンティックロードはありませんが、外観を変更するためだけに使用されます（たとえば、 <font>要素））およびStrict DTDでキャンセルされるプレゼンテーション属性の完全なセットが含まれます。 多くの場合、フレーム内にあるページには移行DTDが必要です。 別のフレームでリンクを開くために必要なターゲット属性があります。
  • HTML 4.01 Frameset DTD-フレームベースのページに使用されます。 W3コンソーシアムは、フレームの使用を推奨していません。 最新のサイトの場合、サーバー側のアプリケーションを使用してこのような問題を解決することをお勧めします。




8.どのDOCTYPEを選択しますか？



新しいページを作成する場合、W3CはHTML 4.01 Strictを使用することをお勧めします（翻訳者のメモ：もちろん、XHTML 1.0 Strictを使用する方が良いことは誰でも知っています） 。



古いHTML 2.0またはHTML 3.2ドキュメントを翻訳する場合、プレゼンテーション全体をCSSに翻訳し、JavaScriptの動作を担当する要素まで、HTML 4.01 Transitionalを使用できます。



11.バリデーターが<embed>タグを誓うのはなぜですか？



<embed>はHTML仕様の一部ではありませんでした。 これは標準化された要素ではなく、ほとんどのブラウザでサポートされていますが、HTMLの一部ではありません。



90年代後半の「ブラウザ戦争」を通じて、MicrosoftやNetscapeなどのブラウザメーカーは、HTMLページのスタイルとスタイルを設定するためのよりクールな機能を競い合いました。 問題は、これらの機能が標準化されておらず、ほとんどの場合、クロスブラウザではないことでした。



広く使用されている他の要素（例： marquee ）がありますが、仕様には含まれていません。 可能な限り使用しないでください。



標準化されていない属性も広まっています。 1つの例はmarginwidthです。



13. BOMとは何ですか？



BOMまたはバイトオーダーマーク -8ビット以上を使用してデータをエンコードする一部のエンコードで使用されます（たとえば、UTF-8またはUTF-16）。 プロセッサは、「ビッグエンディアン」（鈍角）と「リトルエンディアン」（ポイント）の2つの異なるスキームを使用して大きな整数を格納できます。 BOMには、ファイルの先頭に書き込まれた16ビットが含まれており、どのスキームが使用されているかをブラウザに伝えます。



残念ながら、多くの古いブラウザはこの情報を処理できず、代わりにこれらのビットを文字データとして表示します。 ページの上部に奇妙な文字が表示されている場合、これはおそらく、BOMがブラウザーによって処理されなかったこと（またはエンコードが正しく設定されていないこと）を意味します。



唯一の解決策は、BOMを使用しないことです。 通常、ドキュメントをUTF-8で保存できる編集者は、BOMを使用するかどうかを選択できます。

21.使用するもの、<p>または<br />？



p要素は、テキスト内の段落を強調表示するために使用されます。 段落とは、1つの思考によって結合された1つ以上の文です。



行の折り返し （ br ）は主にプレゼンテーションツールとして使用され、HTMLではなくCSSで実装する必要があります。 ただし、詩や歌の行をマークアップするとき、郵送先住所を書くとき、またはコードサンプルをマークアップするときなど、改行がセマンティックな意味を持つことができるいくつかの状況があります。 これらの場合、 brを使用することは正当化されますが、 brを使用して段落を区切ることは受け入れられません。



一方、 pには意味がかなり明確な意味があります。段落マークアップです。 Web開発者は、コンテナとして使用するメインブロックとしてpを検討する傾向がありますが、これは正しくありません。 フォームのp内にラベル要素と入力要素が表示されることは珍しくありませんが、意味的に正しくないと呼びます。 ラベルと入力フィールドを段落コンテンツにすることはできません。

本当に何かを強調したい場合にのみ（何かを強調し、ハイライトします）。 これらのタグは同等ではありません。

Present Not Less Sad Timesでは、著者はテキストを太字または斜体にするためにstrongとemを使用します。



emは意味的ストレス、意味の増幅を意味します。 このタイプの強調表示が適用されるコンテンツは、声を出して読みながら（たとえば、より大きくまたはより長く）増幅する必要があります。 強いということは、さらにストレスがかかることを意味しますが、多くの場合、不要であると見なされます（ネストされたemを使用して、さらにストレスを示すことができます）。 一部の専門家は、ページ上の特定の要素（「現在のページ」のインデックスなど）のみを明確に強調し、本文内の単語やフレーズをマークしないようにすることをお勧めします。



bとiにはセマンティックロードはありません。 フォントを太字または斜体に変更するだけです。 意味的に適切なHTML要素を見つけられなかった一般的に受け入れられている活版印刷規則での使用に適しています。 たとえば、船名は伝統的に斜体で表示されますが、HTMLには<ship>要素はありません。 したがって、<i> Titanic <i>と書くことができます。



27. <address>要素を正しく使用する方法は？



アドレスは 、ページ上の連絡先情報を示すために使用されます。 これは、住所、電話番号、その他の連絡先情報です。 アドレスは、テキスト要素とインライン要素のみを含むことができるブロック要素です。 ほとんどのブラウザではデフォルトで斜体が表示されますが、これはCSSを使用して簡単に修正できます。



アドレスは郵送先住所を示すためにのみ使用できるという誤解はよく知られていますが、そうではありません。



28. <dfn>要素の使用方法は？



dfnは 、「用語の意味を定義する」ために使用されます。 これは、特に科学文書で一般的に受け入れられているイタリック体の表記法であり、イタリック体で定義がテキストに最初に現れるときに読者がなじみのない新しい用語を強調しています。 デフォルトでは、 dfnは斜体で表示されます。



dfnは「略語」を意味するというよく知られた誤解であり、多くの著者はそれを略語および頭字語 （title属性を使用して用語の説明を示す）としても使用しています。 文書内で用語をdfnでマークする必要があるのは1回だけです （用語を初めて使用して説明する場合）。



29. <var>要素を正しく使用する方法は？



varは 、変数、またはテキストの置換可能な部分をマークアップするために使用されます。 これは、実際の生活では他のデータに置き換えられる斜体の変数を強調するために採用された活版印刷の契約です。 たとえば、電話通信マニュアルでは、着信通話を別の内線番号にリダイレクトする手順は次のようになります。



<kbd>* 21 * <var> </var> #</kbd>







ここでは、 varタグを使用して「追加の番号」を示します（イタリック体になります）。 コールを追加の番号942にリダイレクトする場合は、「 21 * 942＃ 」と書き込みます。 varは、「d-o-p-o-l-n-and-t-e-l-n-n-y-n-o-m-e-r」と入力する必要があることを意味しません、しかし「追加の数字」という言葉の代わりに数字があります。



コード例で変数を示すためにvarを使用する必要があるという誤解はよく知られています。



31. <abbr>タグと<acronym>タグの違いは何ですか？



誰も本当にこの質問に答えることはできません！ HTML仕様でさえ、多少矛盾しています。



abbrは、「ブラウザ戦争」の間中、HTML用のNetscape拡張機能でした。 頭字語はMicrosoftの拡張機能でした。 両方のオプションは、ほぼ同じことを意味します。 両方の要素は、異なるセマンティックロードでHTML仕様に含まれていました。 問題は、このセマンティクスが何であるかを誰も実際に説明できないことです。



辞書を見てみましょう。

略語は、単語または語句の略語です。

頭字語-フレーズまたは複数の単語の最初の文字または単語の最初の数文字から形成される単語。



頭字語の定義では、この言葉、つまり 話すことができます。 したがって、「NATO」は頭字語です。 「北大西洋条約機構」というフレーズの最初の文字で構成されています。 それどころか、「FBI」は単語全体として発音できないため、「eh-b-ah」のように聞こえるので、定義されているように頭字語ではありません。 これが混乱の始まりです。 技術的には、「FBI」は初期化（翻訳者の注記：元の「初期化」では）であり、辞書での定義は次のように聞こえることが知られています。



初期化-1）個別の単語として発音される単語の最初の文字または最初の数文字から形成される名前または用語。 2）名前、組織などを意味する最初の文字のグループ。個別に発音されます。



最初の定義は頭字語とほぼ同じであり、2番目の定義はより遠いです。 それにもかかわらず、仕様には初期主義の要素はなく、混乱は、単純なアメリカのスピーチの「頭字語」という単語が「初期主義」という単語の同義語として使用されているという事実によって悪化しています。



HTML仕様には、次の定義があります。



abbr-省略形を示します（例：WWW、HTTP、URI、Massなど）。

頭字語 -頭字語（例：WAC、レーダーなど）を示します。



仕様では辞書の定義を参照しているようです。つまり、「FBI」にabbrのタグを付ける必要があります。 一言で話すことはできません。 それにもかかわらず、仕様の下にいくつかの段落があります：

まだ混乱していませんか？ はい。 すべての頭字語も略語であるが、その逆ではないため、常にabbrを使用するのが最も安全です。 それにもかかわらず、わずかな問題があります。 Microsoftは、略語や頭字語の代わりにabbrを使用するというW3Cの決定に腹を立て、 abbrタグのサポートを拒否しました。 （ただし、Internet Explorer 7ではabbrサポートが引き続き導入されています。）

32.特定の機能がキャンセルされるのはなぜですか？



初心者が興味を持っている最も一般的な機能は、 ターゲット属性です。 この属性はHTML 4.01 Strictでは許可されていませんが、HTML 4.01 Transitionalでは引き続きサポートされています。 Transitionalでは許可されているがStrictでは許可されていない要素と属性が多数あります。



W3Cが特定の要素と属性をオーバーライドする理由は、コンテンツ（HTML）、外観（CSS）、および動作（JavaScript）を分離するためです。 アイテムを中央に表示することはプレゼンテーションの問題です。 センタータグを使用せずに、CSSで解決する必要があります。 リンクを新しいウィンドウで開くことは動作の問題です。 ターゲット属性を使用せずに、JavaScriptを使用して解決する必要があります。



基本的に、キャンセルされた機能は、90年代のブラウザ戦争中に登場した機能です。 これらの機能は、何らかの方法で順序を復元するためにHTML 3.2に含まれていましたが、これはHTMLが直面した主なタスクではありません。 HTML 4のリリースで、その作成者は、少なくともStrict DTDに含まれていたHTML 3.2に含まれていた「有害な」部分を削除することにより、「Webを再訓練」しようとしました。



言い換えれば、これらのことは理由のためにキャンセルされます。 可能であれば、それらを使用しないようにしてください。
