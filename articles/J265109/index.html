アルファ版でリリースされたブートストラップ4





何らかの形でWeb開発に関連している人は、おそらくBootstrapが何であるかを知っています。

個人的には、Bootstrapバージョン2.xフレームワークに出会い、それが印象的でした。 デザイナーがいなくても、かなり耐えられるインターフェースを手に入れることができました。 作り付けの完璧主義者は喜んだ。 インターネットは非常にブートストラップになっていると断言できますが、クレジットを与える必要があります。全体として作業する人は素晴らしく、正しいことをしますが、人々が怠け者であるという事実は別の問題です。



今日はこの素晴らしいフレームワークの誕生日です。 Bootstrap 4はその誕生日に発表されました。



やったー！ Twitter Bootstrapは、私自身と@fatのオープンソースのCSS / HTMLフレームワークで、公開されました： http: //t.co/3OOsQ5T #kaboom

ブートストラップ4-新機能



開発者自身が書いているように、小さなレビューですべての変更を説明するのは多くの新しくて難しいものがあります。 そして、私はしません。 ハイライト：



LessからSassへの移行



ブートストラップは、Libsassを使用することにより、通常よりも高速にコンパイルされるようになりました。



メッシュレイアウトの改善



モバイルユーザーへの重点。 ミックスインは完全に再設計されました。

これで、ドキュメントから判断すると、ボックスから適切な不純物をどこでも入手できます。 たとえば、プロジェクトに古い.rowの代わりに.lineクラスを持たせたい場合： 



.line { @include make-row(); }




フレックスボックスのサポート



開発者自身が書いているように、「未来が来た...」。 軽度の可変動作と再コンパイルにより、 FlexboxベースのBootboxを入手できます



カード



一部のコンポーネント（サムネイル、パネル）はカードに置き換えられました。 彼らは同じことをより良くします。



再起動



Bootstrap 3でクロスブラウザレイアウト用のHTMLをダンプする役割を果たしていたすべてが、Rebootモジュールの形式で判明しました。 1つのsassファイルの設定で。



新しいカスタマイズオプション



新しいバージョンでは、プロジェクトはSass変数を介して多くの設定を受け取りました。 ソースコードの詳細は、それらを選択することなく、フレームワークのLessまたはSassバージョンで以前のものとどの程度正確に異なるかを言うのは困難です。



IE8はサポートされていません。 pxではなくremとemに切り替える



IE8のサポートが必要な場合は、Bootstrap 3のままにしてください。松葉杖を必要とせずに、最高のCSSをサポートするように更新されました。 ピクセル除去は、レスポンシブなタイポグラフィのより良いサポートとコンポーネントのより簡単なサイズ変更が必要なためです。



すべてのJavaScriptプラグインを書き直しました



すべてのプラグインは、最新のJavaScriptを最大限に活用するためにES6で書き直されました。



ツールチップとポップオーバー要素の改善



ありがとう、私はそれを理解しているように、私はテザープロジェクトに伝える必要があります。 間違えた場合は修正します。



改善されたドキュメント



これはMarkdownで書き直され、ドキュメントの操作をより便利にするために便利なプラグインがいくつか追加されました。 改善された検索がすぐに表示されます。



たくさんの新しい



便利なフォームコントロール、インデントクラス、新しいユーティリティクラスなど。



バージョン4には約1,100のコミットと120,000の変更行が含まれており、作業はまだ完了していません。

-アルファリリースをいくつかリリースします。

-すべてをテストするためのRCのカップル。

ブートストラップ3



開発者は、Bootstrap 3のリリース後、2番目のバージョンのサポートを削減したことを後悔しています。 彼らはこの間違いを繰り返したくないので、近い将来、Bootstrap 3のサポートは継続されます。 重要なバグ修正とドキュメントの改善があります。 トロイカのドキュメントは、Bootstrap 4の最終リリース後にも利用可能になります。



しかし、それだけではありません！ 本日、別のプロジェクトが開始されました- オフィシャルブートストラップテーマ 。







これまでのところ、$ 99の3つの印象的なトピック（ダッシュボード、アプリケーション、マーケティング）があります。 それぞれに、Bootstrap +実世界の例、コンポーネント、プラグイン、ドキュメント、および便利なユーティリティのすべてを見つけることができます。 すべてのトピックは、任意のプロジェクトで何度でも使用できますが、トピック自体またはその派生物を販売することはできません。
