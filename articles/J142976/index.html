libGDXチュートリアルの翻訳-パート1（Eclipseでのプロジェクトのセットアップ）

この記事は、libGDXプロジェクトの最初のチュートリアルの翻訳です。 オリジナルはこちら



翻訳者から：最近、Android向けのゲームを書きたいと思いました。 Javaでのプログラミングの経験はありますが、Android向けの開発はほとんどありませんでした。 それから、クロスプラットフォームのゲームを作成できるlibGDXライブラリに出会いました。 また、ロシア語のドキュメント、特にチュートリアルが不足していることにも出会いました。 そのため、チュートリアルを翻訳したいと思いました。 これは私の最初の翻訳ですので、コメントや不正確な点を歓迎します。



プロジェクトのセットアップ



注：AndroidアプリケーションでJava 1.7を使用すると問題が発生します。 Java 1.6を使用していることを確認してください

注：Android用のアプリケーションを構成するには、いくつかの手順を実行する必要があります。 以下をご覧ください。



この記事では、デスクトップとAndroidデバイスの両方でアプリケーションを開発および実行するようにEclipseを構成する方法について説明します。



メインプロジェクトのセットアップ


1. Eclipseで新しいJavaプロジェクトを作成します。

2.ファイルシステムのプロジェクトディレクトリに移動し、libsというディレクトリを作成します。 ナイトリーzipをダウンロードし（ナイトリーzipはプロジェクトの最新ビルドのアーカイブです-翻訳者のメモ）、gdx.jarとgdx-sourses.jarをlibs /ディレクトリに配置します

3. Eclipseでプロジェクトを右クリックし、[更新]をクリックします。 もう一度右クリックして、[プロパティ]-> [Javaビルドパス]-> [ライブラリ]-> [JARの追加]を選択し、jdx.jarを選択して[OK]をクリックします。

4. gdx.jar->プロパティ-> Javaソース添付ファイルを右クリックします。 [ワークスペース]をクリックし、gdx-sources.jarを選択して[OK]をクリックします。

5. [注文とエクスポート]タブをクリックし、gdx.jarの横にあるチェックボックスをオンにして、[OK]をクリックします。

注： 5番目のステップでは、gdx.jarを推移的に依存させます。 これは、メインプロジェクトに依存するプロジェクトのクラスパスにもgdx.jarが含まれることを意味します。 ただし、これはAndroidアプリケーションには適用されません。



デスクトップアプリケーションをセットアップします。


1. Eclipseで新しいプロジェクトを作成します。 適切な名前を付けます（たとえば、gamename-desktop）。

2.ファイルシステムのプロジェクトディレクトリに移動し、その中にlibsというサブディレクトリを作成します。 libsディレクトリにgdx-natives.jar、gdx-backend-lwjgl.jar、gdx-backend-lwjgl-natives.jarライブラリを配置します。

3.プロジェクトを右クリックして、[更新]をクリックします。 再度右クリック->プロパティ-> Javaビルドパス->ライブラリ-> JARの追加、libsディレクトリから3つのライブラリを選択し、[OK]をクリックします。

4. [プロジェクト]タブに移動し、[追加]をクリックして、上記で作成したメインプロジェクトを選択し、[OK]をクリックします。



Android用のアプリケーションをセットアップします。


次の手順を実行する前に、Android SDKがインストールされていることを確認する必要があります。

1. Eclipseで新しいAndroidプロジェクトを作成します。 適切な名前を付けます（例：gamename-android）。 Androidバージョンの場合は、Android 1.5以降を選択します。 パッケージの名前を指定します（たとえば、com.gamename）。 次に、「アクティビティの作成」をクリックして、「AndroidGame」と入力します。 完了をクリックします。

2.ファイルシステムのプロジェクトディレクトリに移動し、その中にlibsサブディレクトリを作成します。 gdx-backend-android-jarライブラリと、nightly zipのarmeabiおよびarmeabi-v7aディレクトリをその中に配置します。

3. Eclipseで、プロジェクトを右クリックし、[更新]をクリックします。 プロジェクトを再度右クリック->プロパティ-> Javaビルドパス->ライブラリ-> JARを追加し、jdx-backend-android.jarを選択して[OK]をクリックします。

4. [JARの追加]を再度クリックし、メインプロジェクトからjdx.jarを選択して、[OK]をクリックします。

5. [プロジェクト]タブに移動して、[追加]をクリックし、メインプロジェクトを選択して[OK]をクリックします。

6. [注文とエクスポート]タブに移動し、メインプロジェクトの横にあるチェックボックスをオンにします。

注： libsサブディレクトリには、正確にその名前が必要です。 これは、Android Eclipseプラグインで採用されている命名規則によるものです。



アセットディレクトリを設定します。


Androidプロジェクトには、assetsというサブディレクトリがあり、これは自動的に作成されます。 Androidアプリケーションで使用できるファイルをそこに配置する必要があります。 これにより、デスクトップアプリケーションで同じファイルを使用できる必要があるため、問題が発生します。 ファイルの2つのコピーをサポートするよりも、アセットディレクトリでこれらのファイルを見つけることができるようにデスクトップアプリケーションをセットアップすることをお勧めします。 これを行うには、次の手順を実行します。



1.デスクトップPCのプロジェクトプロパティに移動し、Javaビルドパスを選択し、ソースタブに移動して、リンクソース->参照をクリックし、Androidアプリケーションからアセットディレクトリを選択して、OKをクリックします。

2. [フォルダー名]フィールドでアセットに名前を付け、[完了]をクリックしてから[OK]をクリックします。



注：デスクトップアプリケーションとAndroidアプリケーションが同じ親ディレクトリにある場合、「PARENT-1-PROJECT_LOC / gamename-android / asset」を使用して、関連するアセットフォルダーを指定できます。gamename-androidはAndroidプロジェクトの名前です。 プロジェクトを配布する場合、これは難しい方法よりも優れています。



ゲームの作成。


メインプロジェクトで、新しいクラスを作成します。 Gameという名前を付けて、パッケージ名（たとえば、com.gamename）を付けます。 「インターフェース」フィールドで、「追加」をクリックし、「ApplicationListener」を選択して「OK」をクリックします。 同様の結果が得られるはずです。 



import com.badlogic.gdx.ApplicationListener; public class Game implements ApplicationListener { public void create () { } public void render () { } public void resize (int width, int height) { } public void pause () { } public void resume () { } public void dispose () { } }




これらのメソッドにより、ゲームをカスタマイズし、レンダリングを管理できます。 それらが空である間、このゲームは何も表示しません-空白の画面のみ。 もっと面白いことをする前に、このゲームを開始する必要があります。



デスクトップアプリケーション用のゲームを起動する


デスクトップアプリケーションを右クリックして、新しいクラスを作成します。 DesktopGameという名前を付けて、パッケージを指定します（例：com.gamename）。 OKをクリックします。 このクラスを次の形式にします。 



 import com.badlogic.gdx.backends.lwjgl.LwjglApplication; public class DesktopGame { public static void main (String[] args) { new LwjglApplication(new Game(), "Game", 480, 320, false); } }




このコードは、LwjglApplicationのインスタンスを作成し、指定された名前とウィンドウサイズでゲームのインスタンスを与えます。 最後のfalseパラメーターは、OpenGL ES 2.0を使用する必要がないことを示します（1.0 / 1.1を使用します）。



アプリケーションを起動するには、プロジェクトを右クリック->としてデバッグ-> Javaアプリケーション。 DesktopGameクラスを選択し、[OK]をクリックします。 Gameというタイトルの黒いウィンドウが表示されます。 （DesktopGameクラスを開き、Ctrl + F11キーを押す方が簡単なようです-翻訳者のメモ）。



Androidでゲームを実行します。




AndroidプロジェクトでAndroidGameクラスを開きます。これは自動的に作成され、次のフォームに移動します。 



 import com.badlogic.gdx.backends.android.AndroidApplication; public class AndroidGame extends AndroidApplication { public void onCreate (android.os.Bundle savedInstanceState) { super.onCreate(savedInstanceState); initialize(new Game(), false); } }




Activityのinitializeメソッドは、ゲームのインスタンスを作成して実行します。 繰り返しますが、falseはOpenGL ES 2.0を使用していないことを意味します



Androidでゲームを開始するには、プロジェクト->としてデバッグ-> Androidアプリケーションを右クリックします。 ゲームはまだ何もしていないため、画面は黒になります。 エラーがある場合は、Logcatビューに表示されます。これは、[ウィンドウ]-> [ビューの表示]-> [その他]-> [Logcat]をクリックして開くことができます。



Libgdxアップデート




libgdxをアップグレードする場合は、nigtly zipを再度ダウンロードし、プロジェクトから次のファイルを更新します。



メインプロジェクト：libs / gdx.jar、libs / gdx-sources.jar



Android：libs / gdx-backend-android.jar、libs / armeabi（ディレクトリ）、libs / armeabi-v7a（ディレクトリ）



デスクトップアプリケーション：libs / gdx-natives.jar、libs / gdx-backend-lwjgl.jar、libs / gdx-backend-lwjgl-natives.jar
