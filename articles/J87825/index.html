MIX'10：2日目、Internet Explorerの逆襲！





最大のWeb開発会議であるMIX会議の2日目は、ラスベガスで始まりました。 カンファレンス初日のトピックや発表を見逃した場合は、 この大規模な投稿でお気軽にお問い合わせください。 これがどのような会議であるかまだわからない場合は、多かれ少なかれそれについて話そうとした別の記事を読んでください。



さらに、彼らはダウンロードのためにセッションの最初の記録をアップロードし始めました。 見に急いで ！ キーノート自体のビデオについては、ここで説明します。



だから。 MIXの2日目はどうしてですか？ むしろ、二日目のオープニング。 私がメモを書き始めたとき、それは夜の12時半であり、ラスベガスでは新しい日がちょうど始まっていたからです。



唯一の答えがあり、それは圧倒的です： Internet Explorerが復活しました！ はい、時々彼らは戻ってきます。 午前中にIE9の可能性を疑って評価したように思われますが、今では彼は追いつくだけでなく、自分の分野で競合他社をしのぎました。 しかし、まず最初に。



HTML5 / SVGのGPUアクセラレーションとタイポグラフィ、HTML5ビデオのサポートとGPUアクセラレーション、CSS3セレクターの完全サポート、マルチコアJavaScript処理、およびそれらがFirefoxとChrome規格の速度とサポートの戦いでどのように落ちたかについて学びたい場合（そして、jQueryとJohn Resig、血液とBill Buxton、oDataプロトコルとGustavプロジェクトなど、あまり面白くないものがあります）。 注意、たくさんの写真！

ブラウザの技術バージョンをダウンロードするためのリンクがすぐに開きました：

もちろん、誰もが新しいブラウザをダウンロードするために登りました。 そして、いくつかの事実が一度に明らかにされました。
  • tech-demoはJavaScript処理速度でFirefox 3.7をバイパスします！ IE8を思い出すのは残念です。加速は約7倍ですが、同時にOpera 10.10と現在のFF 3.6は残されています。 さようならFirefox！ あなたはかつて最速のブラウザでした。


  • 2番目：IE9は55のACID3テストスコアを獲得しています。 これは大したことではありませんが、IE8が約20増えており、PDCに表示されるIE9ビルドが30増えているという事実を考慮すると、進歩は喜ばしいことです。
  • テクノのデモはたくさんありましたが、それについては後で詳しく説明します。
  • http://samples.msdn.microsoft.com/ietestcenter/で 、標準サポートの興味深いテスト結果が見つかりました。ブラウザメーカーはこのことについて多くのことを語っています




















そして、残りはそこにあります。 素晴らしく見えました。InternetExplorer 9は、DOM、CSS3、およびSVG標準のサポート専用でした。



もちろん、陰謀は限界までエスカレートしました。 誰もがMicrosoftが示すものを待っていて、それが無駄ではないことが判明したので、待っていました...



Internet Explorer 9の5つの目標



開発チームの代表者が登場し、ブラウザ開発の主な方向性を説明することから始めました。 それらについて、さらに進んだ。







5番目のポイントは隠されています、陰謀！ 何かになるでしょう。



1.パフォーマンス



マイクロソフトは、高速なJavaScriptの必要性とHTML5の重要性を認識しました。 したがって、 Chakraと呼ばれる新しいJSエンジンは非常に高速です。







私たちは、いくつかの方法でそのような印象的な結果を達成することができました。
  • JavaScriptはコンパイル中です。
  • 処理はバックグラウンドで行われます。
  • 公式：マルチコアプロセッサを最大限に活用してください！








さらに-もっと！ スピーカーは、誰かを追い越すという目標はなく、SunSpiderテストに適応しなかったと言います。 結果は純粋に中間であり、Internet Explorer 9のリリースにより速度が大幅に向上します。これは、現在のJavaScriptの王様が年末までに冠を失うことがないように、新しい加速方法を考える時が来たことを示唆しています。



さて、効果-IE9は、革新的なJavaScript処理を備えた最速のブラウザーの1つです！ わあ！



2. IE9と標準



Internet Explorerと「標準」は互換性のない概念です。 多くの点で、このステレオタイプはIE8を反hasしてきました。IE8は、標準サポートが現在かなり高いレベルにあります。 ただし、IE9はこのブラウザーの考え方を根本的に逆にする必要があります。



IE9がCSS3演算子をサポートする方法のデモが示されています。 IE9 CSS3セレクターテストが100％合格！







ブラウザ間で標準の実装を比較し始めた結果、すぐに悲しくてうれしかったです。 通常 、IE9だけがこのような要求された境界半径を 描きました 。 FirefoxとChromeは、期待されていたものをまったく表示しませんでした。 最初の失敗！



3. GPUアクセラレーション！



Internet Explorer 9は、ほぼすべてのコンピューターに現在搭載されているグラフィックアクセラレータを最大限に活用し始めた最初のブラウザーです。



公式：IE9は完全なGPUアクセラレーションをサポートするSVG 1.1をサポートします！



そして、目立たないように静かに、 VMLからSVGへの移行ガイドドキュメントがMicrosoft Webサイトに表示されます。



Windows 7の開発マネージャーであるStephen Sinofskyが登場し、IE9の起動時に利用可能なスイートからの多くのテストとアプリケーションをデモします。







すべてのデモはGPUアクセラレーションをサポートしています。 ChromeおよびFirefoxブラウザーはIE9の隣を明確に通過し、GPUアクセラレーションSVGが使用されている場合にブレーキをかけます。 失敗！



ここでブラウザの速度を確認してください。 オブジェクトの数を256に設定し、FPSを確認します。 IE9の結果は次のとおりです。 ブラウザの結果を共有します:-)。











1つのデモはClippyをSVGとして示しています。 どのコードをブラウザーで直接編集できます！ したがって、IE9開発ツールは、SVGをその場で変更するための組み込みのネイティブサポートを取得します。



ボールを使用した別のデモでは、ChromeとIE9でのSVG要素の物理処理とレンダリングの比較を示しています。 Chromeは遅くなり、非常に遅くフィニッシュラインに到達します。 失敗！



Direct2Dの動作が実際に示されており、アクセラレータの助けを借りてどのフォントレンダリングが加速され、さらに、フォントスムージングの品質が大幅に向上しています。







Chromeブラウザへのもう1つのキックは、SVGoidsゲームのデモです。 SVGで書かれた、小惑星を破壊する古典的なアーケードゲームです。 当初、ChromeとIE9は対等な立場で機能しますが、アニメーション化された背景（「n」を押す）に多数の半透明のプリミティブが追加されると、Chromeはひどいブレーキになりますが、IE9はそのようなアニメーションにさえ気づきません。 Chromeの失敗。







IE9開発チームは、少なくとも8週間に1回はブラウザーを更新すると報告されています。 信じたい！



4.コミュニティと協力する



Internet Explorer 9開発チームは、HTML5が最優先事項になっていると主張しています。 そして、DOM、SVG、CSS3の開発。 これらの技術を支援するため、チームはW3Cで既製のテストスイートを提供し、統一された標準とブラウザを比較する機能を作成およびテストします。







ブラウザの開発者は、標準の開発を支援します。



さらに-もっと！



5. HTML 5ビデオ！



これはおそらく楽観主義者でさえも予想していなかったことです。



公式：Internet Explorer 9はHTML5ビデオタグをサポートしています。



Microsoftの移動が完了し、タグはYouTubeで機能し、h264コーデックをサポートします。 Theoraには未来がありません。







そして、Chromeの敗北は最後の打撃で終わりました。HDでビデオを視聴するとき、Chromeブラウザは容赦なく再生を遅くし、両方のプロセッサコアを100％読み込みました。 Internet Explorer 9がビジネスに参入したとき、比較するものは何もないことが明らかになりました。 デコードは、フレームを失うことなくスムーズに実行されます。 プロセッサの負荷はほとんど半分です。 はい



公式：Internet Explorer 9は、ビデオタグのGPUアクセラレーションをサポートします。



利用可能な技術バージョンでは、ビデオタグのサポートはありません。



しかし、それだけではありません。 最後の和音は、聴衆は気付かなかったということでしたが、スピーカーは次のことを実証しました 。 Chromeが対処できない場所を示しました。 エピックはChromeに失敗します。



最新のデモンストレーションは、html5ビデオを再生する際の透明度の使用のデモンストレーションでした。 Chromeはこれを表示できませんでした。 最終失敗。







結論として、IE9は次のとおりです。
  • 新しく、高速で、革新的なjsエンジン。
  • HTML5、CSS3、DOM、SVGサポート。
  • 可能な場合はGPUアクセラレーション。
  • ビデオタグのサポートが実装されます。これには、更新プログラムの1つが続きます。
Intenet Explorer 9のデモはここで終了します。 イヤーフラップで汗を消すことができます。



新しいブラウザの立ち上げは非常に強力で、ガスの速度を落とさないようにしています。 進捗と更新を待っています。



スコットガスリー、ASP.NETおよびVisual Studio



IE9の革新の強力なデモンストレーションの後、後続のデモは何となく鈍く見えました。 さらに、開発者は長い間多くのことを知っています。 スコットは、複数のモニターを操作するためのスタジオの新機能について話し、ASP.NET 4の革新を経験しました。







その後、Scott Hanselmanが登場し、Visual Studio 2010の興味深いデモと機能を独自の方法でユーモアで示しました。







jQueryとJohn Resig



私にとって、jQueryの作成者であるJohn Resigが登場したのは予想外でした。 ジョンは、彼とASP.NETチームがjQueryの新しいテンプレートエンジンに取り組んでいると言いました。 後に、Microsoftがプロジェクトの開発のためにjQueryにコードを投稿することが確認されました。







オープンデータプロトコル（OData）



基調講演のほとんどは、オープンで無料のOpen Data Portocolプロトコルに捧げられました。 アイデアは次のとおりです。ツイッターの例を考えてみましょう。ほとんどの人はサービスをサイトとしてではなく、サードパーティのアプリケーションを介してAPIを使用しています。 したがって、Twitterは本質的にデータですが、サイトではありません。 マイクロソフトは、このようなデータを記述するための単一の標準としてODataを提供しています。 さらに、ダラスプロジェクトの助けを借りて、データ配列の所有者はデータストアに無料または有料で配置できます。 したがって、ODataはインターネット全体のデータにアクセスするための標準になります。 この標準は、Atom（XML）に基づいた方法で構築されています。 OData に関する小さなFAQ 。



ODataは、SharePointからExcelで終わるすべてのMicrosoft製品に実装されます。 Silverlightに基づいたオンラインODataデータエディターの可用性に関する情報がすぐに表示されます。 .NETでのオープンソースプロジェクトを約束します。



そして、 NetflixはODataをサポートし 、Azureに基づいてサービスを構築します 。







コードネームヒューストン



もう1つの基調講演は、Hustonというコードネームのプロジェクトです。 これは、Silverlightベースのブラウザーのデータベースエディターです。 彼らは過去のPDCで彼について話した。







ビル・バクストン



ビルバクストンは、最も有名なインターフェイスデザイナーの1人です。 彼のテーマは自然なインターフェース（NUI）です。 彼はSurface、SketchFlow、およびNatalプロジェクトの背後にいました。 ビルの公演はすべて忘れられないショーです。 ビルは感情的なパフォーマンスの方法によって特徴付けられ、その間、彼は多くのことを動かし、明確に述べ、常に大きな衝動で話します。



初めのどこかで、BuxtonはWindows Phone 7でTwitterクライアントを示しました。これはモールス信号に基づいて動作します 。 そのようなこと。



今回のレポートは、どの天才が本当に私たちに伝えたいのかがすぐにはわからなかったようなものでしたか？ バクストンは彼のアイデアにあふれすぎて、持ち去ることが困難でした。 血液とマルチタッチ（Wacomから10ポイントでマルチタッチプロトタイプを示した）と音楽とアートなどがあります。



しかし、アイデアは次のとおりです。紳士、人々が自分の専門能力を明らかにするためのアプリケーションを作成します。 そのようなプロジェクトはグスタフであり、基調講演が始まった。 彼はそれで終わり、招待されたアーティストは再び画面に描くプロセスを示し、バレリーナを完成させました。

Internet Explorer 9に関して、私の印象は次のとおりです。

-もちろん、上記の成功は、IE9が最高のブラウザになるとはまだ言っていません。 残念ながら、SVG以外にもIEが後れを取っているものがたくさんあります。 ですから、記事の私のインスピレーションを受けた口調を真剣に受け取らないでください:-)

-Mcrosoftは、必要に応じて、競合他社よりもすべてをより良く、より速く、より完璧に実装していることを示しました。 競合他社がこれほどまでに誇示した規格に対するサポートはすでに多くあります。

-マイクロソフトは、標準のサポートとW3Cプロセスへの参加に向かっています。 これが何につながるか見てみましょう。

-マイクロソフトがSVGをある程度準備している理由は明らかではありませんでした。 しかし、ベクターグラフィックスのGPUアクセラレーションのデモンストレーションにより、MSが世界で最速のSVGを準備していることが明らかになりました。 見てみましょう、おそらく他のブラウザはMSに追いつくでしょうが、追いつく必要があります。これは前例のないことです。IEに追いつく。 さらに、GPUで高速化されたSVGは、何らかの形でSilverlightと統合されると考えています。

-MSによる素晴らしい動きは、将来のブラウザの利点が見える視覚的なデモンストレーションの作成でした。 残りは、これらのデモのためにブラウザを完成させる必要があります。
