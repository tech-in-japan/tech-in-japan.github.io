興味深いAtmel Cortexマイクロコントローラーファミリー





さまざまな電子機器の開発者（アマチュアと専門家の両方）と通信して、同僚と私は、Cortex-Mプラットフォームに基づいたAtmelマイクロコントローラーが一般にはほとんど知られていないことに気付きました。 示すために、マイクロコントローラーの新しいファミリーのレビューを準備することにしました。興味深いのはAtmelだけではありません。



ちょっとした歴史
ARM Cortex-M時代はCortex-M3プラットフォームで始まりました。 AtmelはARM7TDMIプラットフォームに基づくマイクロコントローラーの最初のメーカーの1つであるという事実にもかかわらず、同社は独自のプラットフォームの開発に焦点を当て、新しいプラットフォームに関心を示しませんでした。 この結果、STのSTM32マイクロコントローラーは、新しいタイプの最初のシリアルコントローラーでした。



2番目の状況は、AtmelのCortex-Mプラットフォームに基づく最初のマイクロコントローラーのリリースが困難な時期に落ちたことです。 これにより、Atmelコントローラーの価格がSTに比べて大幅に高くなりました。

これらの状況は両方とも、STM32マイクロコントローラーに優先順位を割り当てました。 さらに、STM32の開発に関する資料がWebに掲載されるようになり、このリーダーシップは年々強化されてきました。 そして、2つのステレオタイプのうちの1つが、私たちの市場のAtmelアナログに適用され始めました。

1. Atmel = AVR

2. AtmelのCortexは高価であり、使用するのは面白くありません。



最初のステレオタイプがAtmel Webサイトの適切なセクションに移動して解消される場合、2番目のステレオタイプは少し複雑になります。 私たちは彼に対処しようとします。





レビューでは、すべての家族をリストするのではなく、1つまたは両方の基準を満たす家族についてのみ説明します。

•家族のユニークな特徴。

•価格が競合他社の価格よりも優れている、または同等である（ここで、卸売価格の比較可能性について話していることを予約しなければならない）。





SAM SおよびSAM E



これらは、AtmelとSTで今年デビューしたARM Cortex-M7コアに基づくマイクロコントローラーです。 以前のCortex-M4と比較したこのコアの利点は、倍精度浮動小数点計算のハードウェアサポートを利用できることです（Cortex-M4には単精度のみがあり、すべてのCortex-M4ではなくCortex-M4Fのみ）、高いパフォーマンス（DMIPS / MHzは約1.6倍高い）およびより高いCPUクロック周波数（最大300 MHz）。



SAM Sは「基本」ファミリであり、SAM EはイーサネットとCANを搭載したファミリです。 マイクロコントローラーは、LQFP（64、100、144ピン）およびLFBGA（100および144ピン）パッケージで入手できます。 フラッシュメモリサイズ-512〜2048 KB。 すべてのコントローラーには、オンボードUSB（ホスト、デバイス）およびCMOSセンサーを接続するためのインターフェースがあります。 一部の変更（ケースサイズに依存）には、外部メモリインターフェイスとメモリカードを操作するためのインターフェイスがあります。 これらの2つのファミリーに加えて、SAM Vファミリーもありますが、これらは自動車産業向けの上記ファミリーの類似物です。 Cortex-M7コアを備えたすべてのファミリに共通のデバッグキットでのみ発生します。



これらのコントローラーを対応するSTコントローラーと比較する場合、Atmelは以下を好みます。

•CPUの高速クロック（300 MHz対216 MHz）。これにより、パフォーマンスが向上します。

•最大2048 KBのフラッシュメモリ（STのみ最大1024 KB）を搭載したチップの存在。

•CANおよびイーサネットのないファミリの存在（プロジェクトでこれらの機能が必要ない場合、これらの機能に過剰な支払いをしないようにします）。

STコントローラーの利点は次のとおりです。

•LCDディスプレイを接続するためのインターフェース（Atmelはこのために外部メモリインターフェースを使用することを推奨します）;

•暗号化のないファミリの存在（Atmelはすべてのマイクロコントローラーに暗号化を搭載しています）。



AtmelマイクロコントローラーはSTの価格を失うことはないため、お客様から大きな関心を集めました。 このファミリのマイクロコントローラは、Cortex-M4（SAM4SおよびSAM4E）に基づいた「新しい」ファミリとのピン間互換性を保持しています。 唯一の落とし穴は、この互換性がUSBピンに適用されないことです。 これは、これらのファミリのUSBモジュールが異なるためです。Cortex-M4にはUSBデバイスがあり、フラッグシップにはUSBホストとデバイスがあります。



SAM4L



これは、48 MHzのクロック周波数と低消費電力（90μA/ MHz未満）を保証するpicoPowerテクノロジを備えたCortex-M4ベースのマイクロコントローラのファミリです。 パフォーマンスの観点から、このファミリはSTM32 L1ファミリ（Cortex-M3、32 MHz）とSTM32 L4（Cortex-M4、80 MHz）の中間のニッチを占めています。 これらのマイクロコントローラーの価格はほぼ同じです。 周辺機器セットも同様ですが、SAM4Lには2つの特別なモジュールがあります。

1.周波数測定モジュール。

2.プログラマブルロジックモジュール。



最初のモジュールでは、測定された周波数を基準と比較することにより、外部信号の周波数を測定できます。 2番目のモジュールを使用すると、ボード上のロジック回路の使用を放棄できます（もちろん、非常に複雑なロジックが必要な場合を除きます）。 動作原理は単純です：4つの入力は16の可能な組み合わせで構成され、各組み合わせに対して対応する出力状態（0または1）が設定され、モジュールは特定の状態テーブルに従ってCPUの関与なしに動作します。 マイクロコントローラは、場合に応じて、これらのモジュールを1つまたは2つ持つことができます。



SAM G



低電力Cortex-M4マイクロコントローラーの別のファミリー。 プロセッサの観点から見ると、以前のファミリとの主な違いは、浮動小数点計算用モジュール（Cortex-M4F）の可用性です。 このファミリーはpicoPowerテクノロジーも使用しています。消費量はSAM4Lの消費量よりもわずかに高くなっていますが、それでもかなり低いレベル（100μA/ MHz未満）です。



このファミリは、フラッシュメモリサイズ（256または512 KB）、最大プロセッサクロック速度（48〜120 MHz）、およびハウジングサイズ（LQFP100 / 64、WLCSP49）が異なる4つのマイクロコントローラーで構成されています。 マイクロコントローラーのさまざまなオプションは、競合他社の価格よりも大幅に低い価格で相殺されます。



Cortex-M0 +



強力で生産的なコントローラーはもちろん優れていますが、それでもCortex-M0プラットフォームはマイクロコントローラーの世界に真の革命をもたらしました。 32ビットマイクロコントローラーは、パフォーマンスだけでなく価格も自信を持って8ビットコントローラーを押し出します。 Atmelはまた、このアーキテクチャで多くのファミリーを立ち上げました。 これらのファミリの特徴はSERCOMモジュールです。これは、構成に応じて、UART、SPI、I2Cとして機能できますが、このモジュールで使用されるピンを選択することもできます。 別の機能：開発されたPTCハードウェアモジュール。これにより、多数のコントロール（ボタン、スライダーなど）を備えたタッチインターフェイスを作成できます。



SAM D


これは、他のすべてのCortex-M0ファミリの基礎となるベースファミリです。 既に述べた機能を考慮に入れない場合、このファミリは非常に普通と考えることができます。通常の周辺機器セット、通常のケースセット、メモリサイズの標準ライン、市場の平均価格です。



ただし、このファミリーには2つの理由があります。 1つ目の理由は、このファミリの上級代表（SAM D20およびSAM D21）が他のファミリのマイクロコントローラの基盤として機能することです。 SAM D20は、上記のatmel'ovsky Cortex-M0 +のすべての機能を備えています。 SAM D21はさらに発展し、USB、DMA、I2Sが追加されました。 これらのピン間コントローラーは互換性があり、同じケース（LQFP32 / 48/64）とフラッシュメモリ（16〜256 KB）を備えています。



2番目の理由は、最近発表された下級代表者：SAM D09、SAM D10、SAM D11マイクロコントローラーです。 ケースの結論は少数です：SOIC14 / SOIC20、QFN24。 フラッシュメモリの可能なサイズは8または16 KBです。 ファミリ自体は利用可能な周辺機器のセットによって区別されますが、SAM D11にはUSBデバイスが搭載されています。 小型コントローラーの価格も小さいです。



SAM L


もちろん、低電力マイクロコントローラーの作成で得られた経験をCortex-M0 +に使用することはできませんでした。 これらのマイクロコントローラーの特徴的な機能は、Cortex-M0 +の消費が非常に低いことです（35μA/ MHz未満）。 また、SAM 4Lのモジュールと比較して大幅に改善されたプログラマブルロジックモジュールもあります。 このモジュールでは、論理演算（AND、NOT、ORなど）を実装できるだけでなく、トリガーやラッチなどのより複雑な論理要素を作成することもできます。



SAM Lには2つのタイプがあります。SAML21-USBを備えた低電力マイクロコントローラー、SAM D21とピン互換、SAM L22-同じことですが、LCDディスプレイを備えています。 したがって、各ラインは、対応するSTM32L0ファミリ（STM32L0x2およびSTM32L0x3）と競合します。 SAML21はすでに注文可能です; SAM L22の大量生産は来年初めに予定されています。 繰り返しになりますが、SAM L21の価格はSTの同等品に匹敵します。



SAM C


非常に興味深い家族で、その生産は来年初めに始まります。 競合他社のCortex-M0 +マイクロコントローラとは、主に電圧範囲2.7〜5.5Vで区別されます。 その他の機能：周波数メーター、LINインターフェイスサポート、SAM Lに類似したプログラマブルロジックモジュール。



2つのサブファミリーが発表されました。

SAM C20-SAM D20、ケース、フラッシュサイズオプションを備えたピン間互換コントローラも同様です。

SAM C21-SAM C20との下位互換性のあるコントローラー（ピン、メモリ、および周辺機器に基づく）、2つのCANコントローラーが追加され、平方根を分割および計算するためのハードウェアアクセラレーター、温度センサーは個別のモジュールとして設計されています（ADCに巻き込まれていません）。



価格はまだわかっていません。
