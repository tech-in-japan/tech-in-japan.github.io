小さなもののためのイカ

すべてが時を経て、すべての初心者システム管理者が仕事のために新しいソフトウェアパッケージを研究しています。 ですから、Squidの時が来ました。それが何であるかを説明するべきではありません。それを読んだら、イカが何であるか、なぜそれが必要なのかを既に知っているでしょう！



しかし、一度も、私は設定のこれらのワイルドを研究しなければなりませんでした...最初は、多数のパラメータのために、単に設定することは実際には不可能だと思いました。 しかし、時間が経つにつれて、私はすべてがすでに通常の操作のためにセットアップされていることに気付きました。 インターネットを配布するには、追加するだけです

  • acl all src 0.0.0.0/24
  • http_accessすべて許可




したがって、これらの2つの魔法の行は次のことを行います。

最初の行は、絶対にすべてのIPアドレスに対してallという名前のacl（アクセス制御リスト）を作成します。

2番目は、すべてのアクセスを許可します。



これですべてです。プロキシに侵入した人は誰でも無罪でインターネットに侵入できるようになりますが、誰が、どこで、どのくらいの金額を確認できますが、このトピックは別の記事です。 しかし、これは回避できます。 ネットワークの下に192.168.0.0/24があるとします。 squidを備えたサーバーの重量は192.168.0.1 {128（デフォルトではポート3128）であり、2番目のインターフェース（どちらでもかまいません）はインターネットを調べます。インターネットを使用するには、ネットワークからコンピューターのみをすべて削除する必要があります（一般的に、これをalllowで使用することはお勧めしません!!!）、たとえば次のように作成します：



  • acl my_network src 192.168.0.0/24
  • acl all src 0.0.0.0/24
  • http_access allow my_network
  • http_accessすべて拒否




これで、IPアドレスを持つ全員がacl my_networkに含まれます。



これに興味がある場合は、「不運な管理者の注意」シリーズの新しい記事を待ってください:)
