Oracle VirtualBoxでBIGIP Trial Editionを起動します

数年前、職場で、私はエンジニアリングの奇跡-ネットワークバランサーに出会いました。 LBクラス自体は当時私には馴染みがなかったので、熱心に勉強を始めました。 当時、私はWebサーバーを担当していたため、管理者ではなくクライアント側のバランサーと連携する必要がありましたが、好奇心は依然として犠牲になりました。



当時、LTMモジュール（Local Traffic Manager）でBigIP 10.5を使用していました。 F5は、LTM Virtual Editionの90日間の試用版を提供します。これは、後で新しい試用キーで再度アクティブ化できます。 ここで 、画像をダウンロードしてキーをリクエストできます。



一度に5つのキーを取得し、有効期限が切れたらそれらを使用できます。 f5は公式にはVMWareのみをサポートし、Hyper-Vについては他のどこかで聞いたことがあります。 ハイパーバイザーにお金をかけたくありませんでした。ホームコンピューターは魔術師の下で動作するので、VirtualBoxしかありませんでした。 画像が起動するのに驚いたが、最初はほとんどフリーズする。



GRUBメッセージLoading stage2でロードがハングします。 これを修正するには、仮想マシンにCOMポートを追加します。 その後、通常、イメージはロードされてログインします。 ユーザー名のルート、パスワードのデフォルト。 その後、 configを実行してネットワーク設定を設定できますが、代わりにエラーが発生します： 



Cannot connect to mcpd Cannot connect to mcpd. Your preferences and aliases will not be available until it comes bakup. The connection to mcpd has been lost, try again. Error: Unable to examine the state of MCP. Exiting.


上記のメッセージが推奨しているように、再試行する意味はありません-何も機能しません。 さらに、次のメッセージが常に表示されます。 



 localhost emerg logger: Re-starting chmand


まず、このフラッドエラーを無効にする必要があります。これは穏やかに機能しないためです。 次のコマンドでcmandサービスを無効にできます。 



 bigstart stop chmand


その後、 / bin / qpファイルを修正する必要があります。 nanoを使用しましたが、この画像で利用できることがわかりました。 「if [-f / bin / halid];」の行の後、ファイルの先頭のみを修正する必要があります。 その後」変更なし。 



 #!/bin/sh # # Script to query and return platform type (D44, D50, etc.) # # It uses the info saved in /PLATFORM if available; otherwise it # simply calls halid & returns the appropriate field (writing /PLATFORM as well)! if [ -f /PLATFORM ]; then grep platform= /PLATFORM | sed -e 's/platform=//' exit 0 fi if [ -f /bin/halid ]; then


次に、 / tmp / platform_infoファイルを修正し、Info：base_macパラメーターを変更しないでください。 



 Info:system_id=Z99 Info:system_family=0x80000000 Info:system_type=0xb Info:system_if=Z99 Info:host_id=Z99 Info:base_mac=< > Info:mac_offset=0


最後に、 / PLATFORMファイルを修正する必要があります。 



 platform=Z99 family=0x80000000 host=Z99


それだけです！ あなたは車を再起動し、設定に大胆に行くことができます。 BigIPのセットアッププロセスと最初の起動については説明しません。
