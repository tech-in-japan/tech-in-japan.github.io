Visual Studio 2013 Update 3 + Webinarのすべて

開発者は、Visual Studioのリリースのリズムが大幅に変更されたことに既に気づいており、RTMの更新は、以前よりもはるかに頻繁に行われています。 同時に、更新には、検出されたエラーの修正だけでなく、新しい機能も含まれます。 この投稿では、3回目の更新の最も興味深い機能をリストし、それらの使用方法に関する詳細な資料へのリンクを提供します。 また、 10月17日の11時に開催されるウェビナーに接続し、もしあれば追加の質問をする機会があります。





Visual Studio 2013のリリース以降、3つの更新プログラムが既にリリースされています。 開発チームの速度は低下せず、環境は非常に高い頻度で新しい機能と修正を受け取ります。





合計で、これまでに3つのアップデートがリリースされています。 これらの問題の詳細は、次のリンクで確認できます。



  • 更新1 http://support.microsoft.com/kb/2911573
  • 更新2 http://support.microsoft.com/kb/2927432
  • 更新3 http://support.microsoft.com/kb/2933779


Visual Studio Update 3



3番目の更新は、修正に加えて、非常に多くの革新と機能を備えています。 環境自体、デバッガ、コラボレーションツール、Webツール、Sharepoint、およびその他の多くのコンポーネントに革新が現れました。 これに加えて、Azure Tools、Windows Phone 8.1エミュレーター、IntelliTrace診断情報収集ツールなど、Visual Studioと組み合わせて使用​​される新しい関連製品もリリースされました。

http://go.microsoft.com/fwlink/?LinkId=390465からダウンロードするか、Visual Studioを起動して右上隅のフラグをクリックするだけで、自動的にダウンロードされる最新の更新プログラムのリストが表示されます。



コードレンズ：Git統合



コードレンズは、エディターに直接表示できるコードに関連する重要な情報のインジケーターです。 コードレンズテクノロジーの詳細については、 http： //msdn.microsoft.com/en-us/library/dn269218.aspxのmsdnの記事をご覧ください。

以前のリリースにすでにあった機能に加えて、アップデート3には、コードレンズとGitのより緊密な統合に関連する新しい項目が含まれています。 これで、コードの作成者と変更の履歴をエディターで直接確認できるようになりました。





これらの機能の詳細については、Visual Studio ALMブログをご覧ください。



コードマップ



Visual Studioのコードマップテクノロジを使用すると、コードがどのように構成され、その関係がどのようになっているかをすばやく把握できます。 この機能は、古いコードやなじみのないコードを調べるときに特に役立ちます。 このテクノロジーの詳細については、 http：//msdn.microsoft.com/en-us/library/jj739835.aspxをご覧ください 。

Update 3には、リレーションシップのカラーコーディングを追加してより視覚的なプレゼンテーションを可能にする新機能が含まれています。



さらに、コードマップの増減の可能性が改善され、コンパイルされたアセンブリを分析する新しい機能が追加されました。 dllをコードマップウィンドウにドラッグするだけで、コードマップが作成されます。 Visual Studio ALMブログから、これらの技術革新の詳細を学ぶことができます 。



デバッガー



Update 3では、マルチモニター構成のサポートが導入されています。 環境は、Windowsストアアプリケーションが配置されたモニターを記憶するため、次回ウィンドウを別のモニターに転送する必要がなくなります。 Visual Studio ALMブログの詳細。



さらに、ダンプ分析ツールを使用すると、メモリ分析ウィンドウでタイプの定義とこれらのタイプへのリンクを確認できるようになりました。 この機能の詳細については、Visual Studio ALMブログをご覧ください 。





パフォーマンス診断



Visual Studio 2013では、アプリケーションの重要なパラメーターを分析できるパフォーマンス診断ハブが登場しました。

アップデート3で導入された新製品のうち、CPUのパフォーマンス、メモリ使用量、および情報の表示における追加の利便性を分析するためのプロジェクトタイプの拡大に注意する必要があります。 Visual Studio ALMブログから、これらの技術革新の詳細を学ぶことができます。



Visual Studio IDE



Visual StudioのメインメニューのCAPSをオフにする新しいオプションが登場しました。



[ツール]> [オプション]> [環境]> [全般]セクションで変更できます。



テスト中



複雑な製品をテストする場合、テストセットをグループに編成して構成することが非常に重要です。 同時に、アップデート3より前のこのようなテストグループは、識別されたエンティティではなかったため、テストグループのステータスに関するワークフローの編成が妨げられていました。 今、あなたには機会があります。



Team Foundation Server更新プログラム3で提供されるすべてのプロセステンプレートでは、ステータス（アクティブ、合格、終了など）を入力できるタイプがテストケースおよびテストスイートである追加の作業項目、さらにこれらの作業項目をニーズに合わせてカスタマイズできますあなたの組織。 詳細については、 Visual Studio ALMブログを参照してください。



リリース管理



Visual Studio ALMツールキットには、リリース管理コンポーネントが含まれています。 このツールキットの詳細については、 http： //www.visualstudio.com/get-started/manage-your-release-vs.aspxをご覧ください 。 更新3の代わりに、ターゲット展開環境を管理するための機能が登場しました-Desired State Configuration（DSC）。これにより、環境の目的のプロパティをパラメーター化できます。



さらに、Microsoft Deployment Agentをインストールせずに、PowerShellのみに依存して展開できるようになりました。 これらの新製品の詳細については、 http：//technet.microsoft.com/en-us/library/dn249912.aspxをご覧ください。



Webツール



アップデート3には、Webツールに関連する多数の変更が含まれています。詳細については、 http：//blogs.msdn.com/b/webdev/archive/2014/08/04/announcing-new-webでそれらについて確認できます。 -features-in-visual-studio-2013-update-3-rtm.aspx



これには、新しいスキャフォールディングパラメーターのサポート、jsonエディターでのブレースマッチング、およびCSS編集機能の改善が含まれます。



これらの機能に加えて、ASP.NETプロジェクトテンプレートも更新され、サポートされているライブラリのバージョンが更新されました。 特に、新しい認証サポートオプションがプロジェクトテンプレートに追加されました。





Visual Studio統合ツールとWindows Azure



Visual StudioのCアップデート3には、いくつかの追加の制御オプションがあります。 これで、仮想マシンを起動または停止したり、BLOBコンテナーの内容を表示したり、環境から直接他の多くの構成アクションを実行したりできます。





さらに、コンソールプロジェクトに基づいてVisual Studioから直接Azure Web Jobsプロジェクトを作成する機会があります。 コンテキストメニューを呼び出してクラウドに発行することにより、コンソールアプリケーションをAzure Web Jobに変換するだけです。





Web Jobのパラメーターを設定するだけで、ソリューションが公開されます。





これらの新製品の詳細については、 http：//msdn.microsoft.com/en-us/library/azure/dn794167.aspxをご覧ください。



マルチデバイスハイブリッドアプリ



アップデート3のリリースとともに、マルチデバイスハイブリッドアプリCTP 2.0の予備バージョンもリリースされました。これは、Apache Cordovaテクノロジーに基づいて、Android、IOS、Windows Phone用のマルチプラットフォームモバイルアプリケーションを作成できるツールです。



このテクノロジーの詳細については、 http：//msdn.microsoft.com/en-us/vstudio/dn722381.aspxをご覧ください。
