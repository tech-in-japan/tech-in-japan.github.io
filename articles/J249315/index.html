Roskomnadzorからのロックをブロックします

要求されたサイトへのアクセスがブロックされているという警告とともにプロバイダーからページにリダイレクトされ、サイトURLを手動でプルする必要がある場合、このルーチンの自動化を検討します。







私の特定のケースでは、ビーラインからのブロックについてです。 その重要性を取り除くには、Webサーバーが必要です。 ローカルマシンにインストールされます。



「Beeline」 は 、 blackhole.beeline.ru /？ url = ....ページに移動します 。ここで、urlencodeエンコード形式のurl変数にはアドレスが含まれています。 あなたの手で毎回それをコピーしてデコードすることは非常に退屈なことなので、私たちは進化的機能である脳を使用しようとします。



最初に、 hostsファイルを見つける必要があります 。

Windowsでは、％SystemRoot％\ system32 \ drivers \ etc \ hosts、/ etc / hostsのunixのようなOSにあります。

このファイルは、ドメインとIPアドレスの対応を指定します。



次の行を追加します：127.0.0.1 blackhole.beeline.ru



IPとして、プロバイダーによって発行されたページに置き換えることができるサーバーのIPを指定する必要があります。



このサーバーのindex.phpで、次のコードを追加します。 



<?php if ($_GET['url']!=''){ $webProxy="http://www.webproxy.net/view?q="; print '<html><head><title>-! )</title><meta http-equiv="content-type" content="text/html; charset=utf-8" /></head><body>'; print '-! )<br>'; print $_GET['url'] . '<br>'; print '<a href="'. $webProxy . urldecode($_GET['url']) . '">  web proxy</a>'; print "</body></html>"; exit; } ?>


このコードは、urlパラメーターがgetメソッドを使用して渡された場合にトリガーされ、既存のサイトの操作を妨害しません。

$ webProxy変数には、動作中のWebプロキシのアドレスが含まれています。



結論：プロバイダーが警告ページにリダイレクトすると、ページにリダイレクトされます。このページには、サイトのURLが表示され、Webプロキシ経由でサイトを開くように提案されます。 些細なことですが、サーフィンのプロセスを大幅に簡素化します。
