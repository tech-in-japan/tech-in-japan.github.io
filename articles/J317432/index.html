この記事では再び認定に焦点を当てます。 少し前、Android開発者向けにGoogleから証明書を取得するプロセスについて話しました。 ここでそれについて読むことができます。 この記事では、Oracleから証明書を取得するプロセスと定期的に比較しました。 少し考えてから、6か月前にOracleから証明書を受け取り、すべてを忘れていなかったため、このトピックをより詳細に開示することにしました。 当然のことながら、既にこのプロセスを説明しているのは、すでに1回以上です。 たとえば、かなり詳細な説明はここ とここにあります。 ただし、これらの記事は数年前に書かれたものであり、Java SE 7に影響を及ぼします。JavaSE 8試験の機能に焦点を当て、原則として、証明書の取得プロセスに関する情報を更新します。 まだ興味がある人は猫の下でお願いします。



証明書の種類



そのため、何らかの理由で、Java開発者だけでなく、認定Java開発者になることを決めました。 証明書を取得するために、当然Oracleにアクセスします。 すでに何かが、彼らはこの良いの十分を持っています。 ただし、Java証明書、特にJava SE 8にのみ関心があります。どのようなオプションがありますか？ ここで見ることができます。

ご覧のとおり、証明書には3つのレベルがあります。



  • アソシエイト
  • プロ
  • マスター


これが最初のJava証明書である場合、パスは間違いなくJava SE 8 Oracle認定アソシエイト（OCA）認定につながります。 これは最も簡単な試験ですが、それなしでは（または同様のことですが、それについては少し後で）、残りを取得することはできません。 この証明書を取得するには、試験番号1Z0-808に合格する必要があります。 合格する機会がなかったので、先に進みます。



すでに証明書のいずれかを受け取っている場合は、この記事自体について、Oracle Certified Professional Java SE 8プログラマーに移行できます。 そして、ここにはすでに次のオプションがあります。



  • Associate Java SE 8証明書を持っている
  • プログラマーJava SE 7証明書を持っている
  • プログラマーJava SE 6以下の証明書を持っている


この瞬間にもっと注意を払いたい。 合格する必要がある試験の選択は、所有している証明書の種類によって異なります。 どの試験が選択されるかによって、この試験に関する質問は異なります。



最初のケースでは、試験1Z0-809が必要です。

3番目のケースでは、試験1Z0-813が必要です。




そして最後に、すべての王冠、Oracle Certified Master、Java SE 6 Developer。 Java SE 8について話しているので、なぜ6を尋ねますか。私が知らない理由は何ですか。しかし、Oracleは第8回Javaのマスターレベル試験を卒業するつもりはありません。 少なくともサポートで、彼らは私にそのように答えた。 これはおそらく試験の内容です。 準レベルおよびプログラマーレベルは、対応するバージョンのSDKの知識をテストします。 マスターの場合、アプリケーションを作成する必要があります。 この場合、どのJavaで書かれているかはおそらくそれほど重要ではありません。 マスターは最も高価な試験でもあります-それを取得するには、Oracleトレーニングコースのいずれかに合格する必要があるため、そのコストは容赦なく最大1000ドルに達します。



Java SE 6の証明書を手に入れたので、3番目のオプションが思いついたのですが、変更があまりなかったため、7つを逃しました。 さらに記述されるものはすべて、特に6th Javaの証明書からアップグレードすることによるOracle認定プロフェッショナルJava SE 8プログラマー証明書の取得に関連します。
