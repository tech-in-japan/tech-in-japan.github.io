FacebookおよびTwitterとの簡単な共有

「共有」ボタンのないサイトを見つけるのは困難です（「いいね」の話ではありません）。これは、さまざまなソーシャルネットワークを介して友人や敵と興味深い情報を共有するための迅速で便利な方法です。 Androidアプリケーションでも同じ「ボタン」を使用できます。 同時に、アプリケーションがFacebookクライアントである必要はありません。 「Android Yorshikを使用しています！」などの簡単なメッセージを壁に送信する機会をユーザーに提供できます。 試してみてください！」まあ、それは仮説です。

しかし実際、前回のプロジェクトでは、ユーザーにFacebookやTwitterで進行状況に関するメッセージ（および写真）をポイズニングする機会を提供しなければなりませんでした。 この記事を書く前に、共有を担当するコードの部分を別のコンポーネントに分けて、Facebook APIとTwitter APIでの作業を実証することにしました。 しかし、その後、Ostapは苦しみ、その結果、FacebookとTwitter（そして将来、他のソーシャルネットワーク）で簡単に共有するための再利用可能なツールが判明しました。 「単純な共有」とは、単にメッセージまたは写真を送信する機能を意味します。 単純な共有は、「フリーズ」または「リツイート」を意味しません。 シンプルな機能-シンプルなインターフェース！ APIを使用する際のすべての困難を美しいファサードの背後に隠そうとしました。 しかし、それについては後で...



だから、私たちにはタスクがあります：あなたに代わってFacebookのウォールにメッセージを送信し、Twitterであなたのステータスを更新することです。 あちらこちらで 、 OAuthを使用して承認が行われます（OAuth認証の原理は、 この図によく反映されています ）。 まず、オフィスに登ります。 準備ができているものを探しているサイト。



フェイスブック



Facebookは、GitHub'eで入手できる公式のAndroid SDK、およびアプリケーションでこのSDKの使用を開始するための詳細な手順 に満足しています。 SDKのすべてが非常に単純であるとは言えません（一部のパラメーターは質問を発生させます）が、承認タスクを完全に実行します。呼び出す必要があるのは、 

facebook.authorize(Activity activity, String[] permissions, DialogListener listener);
既製のダイアログ自体が表示され、ユーザーがWebフォームにデータを入力できるようにし、最終的にはFacebook APIとのさらなる通信に必要な認証トークンを返します。 それでは、さまざまなバリエーションで既にrequest（...）メソッドを使用できるはずです。 たとえば、壁にメッセージを送信するには、次のように倒錯する必要があります。 

 Bundle params = new Bundle(); params.putString("message", "Android Yorshik is great!"); facebook.request("me/feed", params, "POST");


どんな「自分/フィード」なのかを理解するには、FacebookのOpen Graphコンセプトを浸透させる必要があります。 しかし、これは手頃な価格のFacebook APIのフルパワーを使用したい場合のみです。

最初の水中の小石はここで会いました：Facebookクライアントアプリケーションがデバイスに既にインストールされている場合、承認リクエストfacebook.authorize（...）は、内部のWebViewとのダイアログの表示につながりませんが、このまさにクライアントアプリケーションの起動とそのログインフォームを開きます。 計画によると、承認後、ダイアログの後と同じ方法でアプリケーションに戻ります。 しかし、 onResume（）またはonStop（）で何かが突然巧妙に結び付けられた場合、それらが再び呼び出される可能性があることを考慮する必要があります。 1つのデバイスでは、この形式の承認は一般にとにかく機能しました。 そのため、念のため、Dialogを介して常にログインしたいと思います。 そして、そのような機会があります： Facebook.FORCE_DIALOG_AUTHの追加パラメーターがこれに役立ちます。 私たちは電話します： 

 facebook.authorize(context, permissions, Facebook.FORCE_DIALOG_AUTH, authListener);
対話を通じて承認が安定します。



Twitter



しかし、Twitterは開発者の公式SDKを提供しませんでした。 しかし、さまざまな機能を備えた非公式のものがたくさんあります。 考慮に入れた最初の事柄：

  • Sugree Twitter Android SDKは、基本的にTwitter APIに合わせたFacebook SDKクローンです。 ただし、彼にできることは、ユーザー認証を提供することだけです（ただし、少なくともユーザーログイン用の既製のUIがあります）

  • Twitter4J-強力なライブラリセット、Twitterを操作するための広範な機能（UIなし）

  • JTwitter -Twitter4Jほど広範ではないが、十分な機能（UIなし）


ライブラリをテストした後、Twitter4JとSugreeプロジェクトのUI部分を横断することが決定されました。 試行錯誤により、2つの認証方法が発見されました。

1）最初のケース（実装ツールで使用される時点）では、 dev.twitter.comにアプリケーションを登録するときに、 「Callback URL」フィールドを空のままにします。 次に、ユーザーが認証データを入力して[認証]をクリックすると、PINコードが記載されたページにリダイレクトされます。PINコードは一般的にどこに入力するか明確ではありません。 しかし、それを必要とするのは、それをTwitterのサービスに転送することによってのみ、「トークンと秘密」の認証を取得できるからです。 この種のコードを入力してもう一度ユーザーを苦しめることは、ありがたい仕事です。 したがって、HTMLページを取得して解析し、このPINコードをそこから切り取ります。 WebViewにロードされたページのHTMLコードを取得する方法は？ まったく複雑ではありません：

  • ページのHTMLコードが来るJavaScriptInterfaceを作成します 

     class MyJavaScriptInterface { public void processHTML(String html) { // process HTML } }
  • この「インターフェース」をWebViewに追加します 

     webView.getSettings().setJavaScriptEnabled(true); webView.addJavascriptInterface(new MyJavaScriptInterface(), "HTMLOUT");
  • 最後に、ページが完全にロードされた後にJavaScriptコードを実行します 

     webView.setWebViewClient(new MyWebViewClient()); ... private class MyWebViewClient extends WebViewClient { ... @Override public void onPageFinished(WebView view, String url) { super.onPageFinished(view, url); browser.loadUrl("javascript:window.HTMLOUT.processHTML('<head>'+document.getElementsByTagName('html')[0].innerHTML+'</head>');"); } }


以上で、PINコードを抽出して認証を完了しました。

2）そして今、正しい方法：フィールドのdev.twitter.comのアプリケーションのプロパティで

「コールバックURL」は 、有効なURL（ 「http：//your.shik」など ）を挿入します。 そして今、ページをPINコードで解析する代わりに、Twitterからコールバックをキャッチする必要があります。そのURLにはoauth_verifierパラメーターが含まれ、同じ「トークンとシークレット」を取得するためにPINコードの代わりに使用します。 このような原則はこのプロジェクトで成功裏に実装されており、近い将来、自宅で同じロジックを実装するために共有する予定です。



シンプルなソーシャル共有



Simple Social Sharingと呼ばれた、作成されたプロジェクトに関するいくつかの言葉。 すでに冒頭で述べたように、プロジェクトの主な目的は、FacebookやTwitter（将来的には他のソーシャルネットワーク）と簡単に共有できるシンプルなAPIを提供することです。 単純なAPIには4つの（+1）メソッドが含まれます。

  • isAuthorized（） -ユーザーが承認され、現在のセッションが有効かどうかを確認します
  • authorize（） -ユーザーがログインするダイアログを呼び出します
  • publishMessage（...） -メッセージを公開します

    • publishImage（...） -画像を公開します（Facebookのみ）
  • logout（） -現在のセッションをクリアし、保存された「トークンとシークレット」を削除します

     FacebookFacade facebook = new FacebookFacade(activity, FACEBOOK_APP_ID); TwitterFacade twitter = new TwitterFacade(context, TWITTER_CONSUMER_KEY, TWITTER_CONSUMER_SECRET);
  • 承認イベント（ onAuthSucceed（）およびonAuthFail（） ）、メッセージの公開（ onPostPublished（）およびonPostPublishingFailed（） ）、およびログアウト（ onLogoutComplete（） ）のサブスクリプションを受け入れるFacebookEventsおよびTwitterEventsアラートセンター。 リスナーを追加できますが、後で削除することを忘れないことが重要です。
