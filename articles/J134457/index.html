PySideを使用する

PySideプロジェクトに関する情報はWeb上にいくつかありますが、ロシア語ではあまりありません。



この記事は、このギャップを埋める試みです。 次に、プロジェクトのアセンブリ、エミュレーターでのアプリケーションの起動、独自のバインディングの作成例について説明します。 PySideの主要な機能のいくつかを例とともに示します。



はじめに PySideとは何ですか？



PySideは、 Pythonプログラミング言語へのQtバインディングを作成するためのプロジェクトです。 QtQuickやQtMobilityを含むすべてのQt機能のサポートを実装することを目的としています 。 現在、次のバージョンのコンポーネントとプラットフォームがサポートされています。

Qt 4.6、4.7、4.8ベタ
QtMobility 1.2.0
Python 2.5、2.6、2.7、3.2（実験的）
OS Linux / X11、Maemo 5、MeeGo、Windows、Mac OS X


SymbianでPySideを使用する可能性はまだありません（または、この可能性はわかりません）が、開発者によると、この方向での作業は進行中です。 PyS60を代替として使用できますが、 Qtとは何の関係もないので、後で説明しません。



PySideプロジェクトには、C / C ++で記述されたライブラリのバインディングを作成できるツールのセットも含まれています。 これらのツールについては後で詳しく説明します。



PySideはLGPLの下でライセンスされています。 オープンおよびクローズの両方の商用プロジェクトで使用できます。 PySideソースコードは公開されており、 github.com / PySideで入手できます。 また、 PySideに関連する問題を見つけた場合は、 プロジェクトの公式バグトラッカーに報告できます。



PySide Build、ヘルスチェック



ダウンロードページで 、さまざまなOSにPySideをインストールする手順を参照できます。 しかし、Linuxディストリビューションでは、そのようなパッケージは利用できないことが判明する場合があります。 それを自分で組み立てましょう。これはまったく難しくありません。 ここでも、Qt開発者はユーザーを管理し、 PySideアセンブリアクションを可能な限り自動化する一連のスクリプトを準備しました。



それでは、ビルドスクリプトを使用してGitリポジトリのクローンを作成しましょう。 まだ何も必要ありません 依存関係により、必要なすべてのプロジェクトが複製されます。 



$ git clone git://github.com/PySide/BuildScripts.git buildscripts $ cd buildscripts $ git submodule init $ git submodule update


Qtの複数のバージョンをシステムにインストールできます。 使用するバージョンを指定するには、enviroment.shファイルを編集します。このファイルでは、 QtホームディレクトリへのパスとPySideがインストールされるパスを書き込みます。 enviroment.shファイルには詳細なコメントが含まれているため、問題はありません。 システムにデフォルトでインストールされているQtのバージョンでは、 PySideが収集されない可能性があることをすぐに警告します。 QtSdkの最新バージョンを使用してビルドすることをお勧めします。



すべての構成が完了したら 、 PySideコマンドを収集します 



 $ ./build_and_install


作業を容易にするために、enviroment.shファイルは、 PySideを使用するアプリケーションを起動するスクリプトに簡単に変換されます。 あなたがする必要があるのは、それを実行可能にし、最後にpython $ @を追加することだけです。



結果のパッケージの機能をテストするために、 PySideに適合したPythonの Qtサンプルを使用してリポジトリのクローンを作成します 



 $ git clone git://github.com/PySide/Examples.git pyside-examples


好きなサンプル（Hyper UIなど）を実行します







これで、ホストシステム用のPySideを構築する準備が整いました。 ただし、開発中は、QtSdkに含まれているシミュレーターでアプリケーションを実行すると便利な場合があります。 彼女のためにもPySideを構築しましょう。 これを行うには、enviroment.shファイルのQT_SDK_HOME変数を編集して、QtSimulatorへのパス（$ {YOUPATH} / QtSDK / Simulator / Qt / gcc）をQtホームディレクトリとして指定します。 build_and_installファイルも編集する必要があります。cmakestartコマンドに-DQ_WS_SIMULATOR = yesオプションを追加する必要があります。 このオプションは、cmakeがコンパイルを実行するプラットフォームを決定するのに役立ちます。



今すぐアセンブリを開始すると、現時点では残念ながら失敗します。 QtWebKitモジュールはアセンブルできません。 このエラーに関するレポートを作成し、この問題を修正する小さなパッチを添付しました。 しかし、記事を書いた瞬間、このバグはまだ修正されていません。 したがって、このパッチを使用して、QtSimulatorでPySideを完全にビルドします（上記のリンクを使用してバグレポートへの添付ファイルを参照してください）。







QtSimulatorのQtMobilityバインディングの構築を妨げるものは何もありません。 そのアセンブリは、 PySide自体のアセンブリと変わりません。 QtMobilityはアセンブルされ、シミュレーターで実行されますが、残念ながら、いずれのモジュールも動作しているとは言えません。 完全に、 QtMobility自体またはpyside-examplesパッケージから提供された単一のサンプルは機能しませんでした。 また、この理由についても対処し、おそらくいつかそれについて書きます。







PySideプログラミングの紹介。 信号とスロット、プロパティシステム



このセクションは、 Pythonプログラミング言語の紹介ではありません。 PythonでのQtの主要な機能の使用方法のみを示します。



信号スロット相互作用 


 from PySide import QtCore def say_hello(name): print "Hello,", name class Foo(QtCore.QObject): @QtCore.Slot(str) def say_bye(self, name): print "Bye,", name class Bar(QtCore.QObject): signal = QtCore.Signal(tuple) f = Foo() b = Bar() b.signal.connect(say_hello) b.signal.connect(f.say_bye) b.signal.emit("User.") b.signal.emit(1.25)


信号は、信号とスロットの接続、および信号自体の送信を制御するクラスです。 信号とともに送信されるパラメータのタイプは、作成時に示されます。 任意のタイプのCおよびPythonを使用できます。 （上記の例のように）異なるタイプのパラメータを送信する場合、タプルまたはリストをタイプとして指定する必要があります。 前と同様に、シグナルはQObjectから継承したクラス内で宣言する必要があります。



複数の信号を作成する必要がある場合は、次の記録フォームを使用できます 

 class Test(QtCore.QObject): signals = QtCore.Signal((int,), (str,), (float, QtCore.QObject))


このコードは、Signatures（int）、signals（QString）、signals（double、QObject *）の3つのシグナルを追加します。 それらはすべてシグナル変数に含まれており、条件付きで辞書と見なすことができます（反復不可）。 信号パラメーターはキーとして使用されます。 

 someone.signals.emit(10) #     , .. int someone.signals[str].emit("Hello everybody!") someone.signals[float, QtCore.QObject].emit(1.23, obj)


QObject、グローバル関数、またはラムダ関数からも必ずしも継承されないクラスのメソッドは、スロットとして機能できます。 ただし、そのようなオブジェクトは真のスロットではないため、QMetaObjectにはそれらに関する情報がないため、このアプローチは慎重に使用してください。 たとえば、グローバル関数をスロットとして使用する場合、その中の呼び出しオブジェクトに関する情報を取得できません。



真のスロットを作成するには、QtCoreモジュールのSlotデコレーターを使用する必要があります。 コードでは、次のようになります。 

 @QtCore.Slot(int) @QtCore.Slot(str) def mySlot(value): print value


オーバーロードルールはスロットには適用されず、あらゆるタイプのパラメーターを受け入れるスロットを作成できません。 パラメーターのタイプごとに個別のデコレーターを追加する必要があります。 上記は、int型およびQString型のパラメーターを受け入れることができるスロットの例です。



追加パラメーターとしてのQtCore.SignalクラスとQtCore.Slotデコレーターの両方は、生成されたシグナルとスロットがmetobject情報に保存される名前を使用できます。 デフォルトでは、名前が指定されていない場合、シグナルにはそれが割り当てられているクラスメンバーの名前が割り当てられ、装飾されたメソッドの名前がスロットに割り当てられます。 スロットは戻りタイプに設定することもできます。 この機能を使用して、 PythonとQMLを通信できます。



PythonとQMLの相互作用の詳細については、 developer.qt.nokia.comのPythonセクションをご覧ください。 pyside-qml-examplesも見ることができます。



プロパティシステム


Qtプロパティの操作は、従来のPythonプロパティの操作と大差ありません。 ほんの小さな例です。 



 from PySide import QtCore class MyObject(QtCore.QObject): def __init__(self): QtCore.QObject.__init__(self) self._x = None def getx(self): print "get" return self._x def setx(self, val): print "set" self._x = val prop = QtCore.Property(int, getx, setx) obj = MyObject() obj.prop = 12345 print obj.prop


PythonでのQtプロパティの操作の詳細については、 こちらをご覧ください 。



PySideを使用してGUIを作成する



PySide Toolsパッケージには、アプリケーションリソースの操作、「クラシック」グラフィカルインターフェイスの開発、およびアプリケーションのローカライズのための標準Qtツールが含まれています。 これらは、pyside-ui、pyside-rcc、およびpyside-lupdateです。 それらの操作は、Qt / C ++の同じパッケージと同じです。 したがって、QtDesignerでグラフィカルアプリケーションフレームワークを作成する機会がまだあります。 結果のフォームはpyside-uiを使用してコンパイルされます。pyside-uiでは、-oオプションを介してフォームファイルと出力ファイルを入力パラメーターとして指定する必要があります。 Pyside-uiには、アプリケーションの起動コードを結果のPythonファイルに追加する追加の興味深いオプション-xもあります。 以下は、デザイナー、リソース、およびローカライズで作成されたフォームを使用してアプリケーションを起動する小さな例です 



 from PySide import QtCore, QtGui from form_ui import * #   import resources_rc #   if __name__ == "__main__": import sys translator = QtCore.QTranslator() translator.load(':/i18n/translations/ru_RU') #     app = QtGui.QApplication(sys.argv) app.installTranslator(translator) Form = QtGui.QWidget() ui = Ui_Form() ui.setupUi(Form) Form.show() sys.exit(app.exec_())


また、 PySideには、QtQuickを含む近年の多くのQtグッズがあります。 つまり PythonとQML （QtQuick.Particles、Qt Desktop Componentsなど）を使用してハイブリッドアプリケーションを作成する機会を奪われていません。



独自のバインディングを作成する例



上記のすべてを知っていると、QtQuickを使用するものも含め、 Qtを使用してカスタムPythonアプリケーションを作成できます。 しかし、C / C ++で書かれたライブラリをプロジェクトで使用する必要がある場合、または同じC / C ++で作成された以前の開発を使用する必要がある場合はどうなりますか？ Pythonですべてを書き直さないでください？



たとえば、 Qt Componentsを使用した特定のプロジェクトがあります 。 主な機能はすべてQMLで記述されています。 しかし、それを実行するには、すでにC ++で記述されたQmlDesktopViewerクラスが必要です。 Pythonで書き直すことは難しくありませんが、今では面白くないでしょう。 このクラスをPythonに独自にバインドしてみましょう。



これを行うには、 PySideプロジェクトのツールを使用してバインディングを作成します。 これらは、API Extractor、Shiboken、およびGenerator Runnerです。 これらのツールはすべて揃っています。



バインディングを作成するには、まず、エクスポート、非表示、名前変更などのデータを記述するxmlファイルを作成する必要があります。 一般に、結果のPythonモジュールでアクセスするもの。 



 <?xml version="1.0"?> <!--  ,       --> <!--  package  ""         ()  --> <!--          PySide/QmlDesktopViewer --> <!-- c  qmldesktopviewer_module_wrapper.cpp  pyside_qmldesktopviewer_python.h --> <typesystem package="PySide.QmlDesktopViewer"> <load-typesystem name="typesystem_declarative.xml" generate="no" /> <!--      x11Event. --> <!-- ..  x11Event      Python --> <rejection class="*" function-name="x11Event" /> <!--  , ,   ..      --> <!--          --> <!--      PySide/QmlDesktopViewer      --> <!-- logwidget_wraper.cpp logwidget_wraper.h) --> <object-type name="LoggerWidget" /> <!-- qmldesktopviewer_wraper.cpp qmldesktopviewer_wraper.h) --> <object-type name="QmlDesktopViewer" /> </typesystem>


ジェネレーターの出力で取得するファイルを詳細に具体的に説明しました。 Pythonモジュールを取得するためにコンパイルされます。 これを理解すると、ビルドシステム用のプロジェクトファイルを簡単に作成できます。 このファイルをコンパイルするためのルールの詳細については、 API Extractorのドキュメントを参照してください。



次のステップは、バインディングが行われているデータに関する情報を抽出するために使用されるC ++言語ヘッダーファイルを作成することです。 



 #undef QT_NO_STL #undef QT_NO_STL_WCHAR #ifndef NULL #define NULL 0 #endif #include "pyside_global.h" #include <loggerwidget.h> #include <qmldesktopviewer.h>


PySideビルド中に受け取ったpyside_global.hファイル。 プロジェクトにコピーするか、その場所にパスを追加できます。



ビルドシステムとして、CMakeを使用します。 ここではアセンブリ自体については説明しません。 完全なサンプルコードは、 ここで取得できます 。 これを例として使用して、独自のバインディングを作成できます。



プロジェクトをコンパイルし、クラスのラッパーを含むライブラリを取得します。これは、 Pythonのアプリケーションで使用できます。 結果のモジュールを使用して、Qtデスクトップコンポーネントのサンプルの1つを実行します。 



 #!/usr/bin/env python from PySide.QtCore import * from PySide.QtGui import * from PySide.QmlDesktopViewer import * if __name__ == "__main__": import sys if len(sys.argv) < 2: print "Usage: qmldesktopviewer <qml file>" sys.exit() app = QApplication(sys.argv) viewer = QmlDesktopViewer() viewer.open(sys.argv[1]) sys.exit(app.exec_())






また、 PySideにはサードパーティのライブラリバインディングが既にいくつかあることにも注意してください。 それらの1つはQwtバインディングです。
