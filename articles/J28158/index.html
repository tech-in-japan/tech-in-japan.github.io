RSS-インターネットの未来？

私の友人の1人はICQで、サイトへのアクセスを実質的に停止し、代わりにRSSを介してすべてを読むと書いていました。 この考えが頭に浮かんだので、この技術に関する知識を体系化し、不足しているものについて考えることにしました。 おそらく将来的には、インターネットの主要な要素はサイトではなく、RSSコンテンツのソースであり、Googleリーダーの誰かが単に共有するだけでしょうか？



つまり、このテクノロジーにより、サイトの更新を監視できます。 サイトが少ない場合は、1日1時間または1時間ごとに立ち寄って、新しいサイトがあるかどうかを確認しても問題ありません。 それらが数十個ある場合、RSSは保存されます。 より正確には-Feedです。これは、さらに新しいAtom形式があるためです。 履歴などの詳細は、 コンピューターで読むことができます 。 それらを読むための多くの異なるプログラムとサービスがあります。私はFireFox用のGoogle Reader WatcherとBriefプラグインを使用します 。



RSSの既存のサービスの主な種類は次のとおりです。
  1. 複数のRSSフィードを1つにまとめるアグリゲーター。 もちろん、最も人気のあるのはGoogle Readerです。 さらに、たとえばmine： web 、 java 、 p2pなどのコレクションを共有できます。
  2. RSSフィードからニュースの一部のみを選択できるフィルター。 最も強力なのはYahoo Pipesです。
  3. キャッシュ。 サイト/サーバーでのRSSフィードの生成が重すぎる場合、たとえばFeedBurner （Googleが購入）を使用できます。 また、フィードの対象者に関する統計情報も提供します。
  4. 物語を続ける。 かつてのように 、ニュースグループにはdejanewsがあり、インターネットにはweb.archive.orgがありました。 たとえば、Googleリーダーは、RSSフィードを追加すると、その履歴全体を自動的に提供します（このフィードを初めて追加した人がいるため）。
  5. RSSフィードの作成者は、そうでないサイトのフィードを提供します。 page2rss （ K-Liteなど ）などの単純なタイプから、Yahoo Pipes（ 映画レビューShitなど ）などの洗練されたタイプまで。
  6. RSSフィードでの広告。 たとえばGoogleで表示されるようになりました 。
  7. html、pdf、js、写真、電子メールなどの形式のRSSエクスポートサービス
サービスの適切な選択は、 RSSオンライン投稿でrexedeadによってまとめられました。



RSSは、主にテレビ番組などのBitTorrentプロトコルを使用して大きなファイルを配布（および自動的にダウンロード）するためにも積極的に使用されています。 詳細： uTorrent 、 LostFilm 、 NovaFilm 、 EZTV （英語シリーズ）、 torrents.ru 、 NNM-Club 、 アニメ 。



RSSリーダーでの主な操作は、既読としてマークする以外は:)、です：
  1. 星印 重要として「アスタリスク」と入力し、お気に入り/ブックマークに追加します。
  2. 共有する 共有します。 友達に見えるフィードに追加します。
  3. タグ。 タグ（カテゴリ）でマークするか、フォルダに入れます-さまざまな種類の分類
  4. コメント。 ニュースにコメントを追加します。




RSSに最初に欠けているのは、ニュースフィードの数を管理することです。 たとえば、サイトのフィードに20のエントリがあり、1日に1回RSSリーダーを起動すると、21のニュースがそこに表示されたとしましょう。 または、RSSのフィードの前のページにリンクを追加することもできます。これにより、サーバーはそれを静的ファイルに保存し（負荷が少ない）、クライアントは目的の深さの履歴を収集できます。 もう1つのオプションは、スクリプトへのリンクです。スクリプトへのリンクは、パラメーターとして、必要な投稿数または必要な時間範囲に設定できます。 たとえば、 GData標準では、これが提供されています。 Googleは同様の機会を提供します ： 'http://www.google.com/reader/atom/feed/' +フィードアドレス+ '？N =' +エントリ数（デフォルト= 20）が、すべてのフィードの履歴を保存するわけではありません。



カテゴリ/作成者フィールドが含まれている場合、タグ/カテゴリおよび作成者でフィードのコンテンツをフィルタリングおよびグループ化できると便利です。 現在、Google Readerはそれらを処理しませんが、タグフォルダーと同様にテープの横に「プラス記号」が表示されると便利です。クリックすると、このテープのタグ/投稿のカテゴリのリストが表示され、このタグでマークされた投稿のみを読み取ることができます。 確かに他のRSSリーダーにありますが。



RSSリーダーから直接コメントを投稿する機能。 これはAtom Publishing ProtocolおよびGDataで提供されており、承認用のOpenIDなどの多くの標準があります。おそらく遅かれ早かれ、このような機会が現れるでしょう。



この投稿に関連付けられているフィードへのリンク。スター/共有をクリックするなど、ワンクリックで購読できます。 たとえば、この投稿のコメントフィードを購読します。 そしてその逆、コメントフィードから投稿まで-メインフィードへのリンク。



後者は、RSSリーダーでフォーラムを読むのを非常に容易にします。 メインフィードは、すべてのサブフォーラムのリストです。 RSSリーダーでマウスを1回クリックするだけで、選択したサブフォーラムの新しいトピックを購読できます。 また、ワンクリックで-あなたのお気に入りのトピックのすべての投稿を購読します。



おそらく、フォーラムとブログからすべてのデータをエクスポートすると、それらの違いが消去されます。 すべての新しいフォーラムトピックを取り上げ、著者別にグループ化し、最初の投稿の日付順に並べ替えると、一連のブログが作成されます。 また、タグなどでブログのすべての投稿をグループ化し、最後のコメントの日付でソートすると、ブログからフォーラムが取得されます。 これはすべてRSSリーダーで実行できます。
