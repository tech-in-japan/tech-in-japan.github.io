Anturisサービスを例として使用したインフラストラクチャのサービスとしての監視

監視は、ITインフラストラクチャ管理の重要なコンポーネントの1つです。 現在、監視には多くのソリューションとアプローチがありますが、最も一般的なものは2つのグループに分けることができます。



  • 外部監視のみ。ユーザー側の状況を分析するために使用されます。 この場合、応答時間、稼働時間、往復時間などのメトリックスが調べられます。スウェーデンのPingdomは、このセグメントのリーダーです。 このような解決策の欠点は、問題を理解している場合、その原因を特定するのが難しく、それに応じて迅速にそれを排除することです。
  • NagiosやZabbixなどのオープンソース監視システム 。 このアプローチは主に、高度なカスタマイズとほぼ無制限の機能で人気があり、自作のスクリプトとサードパーティのプラグインを使用して拡張できます。 同時に、そのようなシステムは、監視自体のコンポーネントを実装およびサポートする方法を学ぶために多くの時間を必要とします。 さらに、外部監視の一部の機能が失われます（たとえば、異なる地理的場所からのチェック）。




サービスとしての監視



外部監視のみを欠いているITスペシャリストが多く、自分の監視システムの展開とサポートにほとんどの時間とお金を費やす準備ができていません。 そのため、ここ数年で、新しいクラスのソリューションが市場に登場しました- サービスとしての監視またはホストされた監視。



この場合、内部リソース（ハードウェアとソフトウェア、ネットワークリソース）は主にローカルエージェントの助けを借りて監視され、外部（サードパーティと自社サービス、代理トランザクションなど）はベンダーのサーバーからのリクエストを使用して監視されます。 データはサービス側で収集され、その後、エラーレポート、レポートが生成され、問題が発生した場合に通知が送信されます。



つまり、ホスト型監視の利点は次のとおりです。

  • システムを操作するためのトレーニングコストを最小限に抑える
  • 監視システムの展開と保守のコストを最小限に抑える
  • 簡単なスケーリング
  • 高度な外部監視機能




アンチュリスサービス


そのようなサービスの1つがAnturisです。 その動作原理は、次のように概略的に表すことができます（矢印は、コンポーネント間の依存関係、および破線のデータ収集と送信を示します）。







このアプローチでは 、 エンドユーザーエクスペリエンス （EUE）とインフラストラクチャコンポーネント間の影響依存モデリングが追跡されるため、問題の原因をすばやく特定できます。



Web GUIを介して新しいモニターが作成されます。これには、対応するウィザードを実行するだけで十分です（内部監視の場合は、エージェントを事前設定します）。







現時点では、Anturisを使用すると、次のタイプのモニターを作成できます。







サービスの主な機能は次のとおりです。

  • ハードウェアおよびソフトウェアサーバーの監視。
  • Webサービスの監視。
  • Webページの読み込みの監視（ブラウザーページ全体の読み込みのシミュレーション）;
  • 総合的なトランザクション監視
  • 地理的に異なる場所からの外部監視。
  • ネットワーク監視（SNMP、チャネル品質、プリンター）;
  • SMS、電話または電子メールによる問題の通知。
  • cPanelおよびParallels Pleskのプラグイン。
  • 定期的なレポートの作成とデータの視覚化。
  • インシデント分析。




30日間無料版を使用して、自分でAnturisを試すことができます。 これを行うには、Anturis Webサイトでアカウントを作成するだけです。
