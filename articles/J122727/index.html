Windows Azure AppFabricプラットフォーム（6月CTP）の大きな更新とSDKの新しいバージョン





マイクロソフトは、クラウドサービスWindows Azure Appfabricの新しい実験バージョンを公式に発表しました。 Windows Azure AppFabric June CTPは重要なリリースであり、多数の革新を提供します。



新しいバージョンの重要な部分は、すべてのレベル（Web、ビジネスロジック、データベースレベル）でマルチレベルアプリケーションの状態を開発、ホスト、管理、および監視する機能であり、Windows Azureで個別の論理エンティティとして各レベルで動作します。



このリリースでは、次のイノベーションが提供されます。
  • AppFabric開発者ツール -Visual Studioの革新により、ビジュアルデザイナーを使用してWindows Azureプラットフォーム用のエンドユーザーアプリケーションを構築できます。
  • AppFabric Application Managerは、配置、構成、監視、トラブルシューティング、トラブルシューティングを含むアプリケーションの一般的なライフサイクルを通じて、Webレベル、ミドルレベル、データレベルを含むマルチレベルアプリケーションをホストおよび管理するための強力かつ同時に簡単な方法です。 Application Managerには、REST APIに基づいた本格的なアプリケーション管理ポータルが含まれており、マルチレベルアプリケーションを監視し、実行中のアプリケーションを管理するタスクを実行するように設計されています。
  • コンポジションモデル -Windows Azureプラットフォームでアプリケーションをコンパイルするための.NET Frameworkの一連の拡張機能。 これらの拡張機能は、Azure Service Modelの概念に基づいて構築され、アプリケーション内のコンポーネントを記述および統合するための新しい機能を追加します。 AppFabric Developer Toolsは、構成モデルを使用してアプリケーションマニフェストを作成します。これは、展開中にAppFabric Application Managerによって使用され、アプリケーションとそのメタデータの構造を決定します。
  • カスタムコード、WCF、およびWFの実行のサポート— Windows AzureおよびCommunication Foundation（WCF）、WF、およびユーザーコードサービスの組み合わせでWindows Workflow Foundation（WF）を実行するための正式なサポート。 これに加えて、WCFおよびWFサービスの開発、展開、管理、および監視ツールが改善されました。
Windows Azure AppFabricを使用するには、Windows Azure AppFabric SDK CTP-June Update開発ツールが必要です。これは、 次のリンクからダウンロードできます。



Windows Azure AppFabric June CTPのリリースがhttp://portal.appfabriclabs.com/で利用可能になりました。 プラットフォームの実験版の新機能へのアクセスはユーザーの数によって制限されるため、できるだけ早くサービスに登録してください。 この更新プログラムで提供されるサービスにサブスクライブするには、次の手順を実行する必要があります。
  • http://portal.appfabriclabs.com/でログインしてログインします。
  • 左パネルのAppFabricノードでアプリケーション項目を選択します。
  • [ネームスペースのリクエスト]ボタンをクリックします。
  • リクエストする前にいくつかの質問に答えてください。
  • 確認のためにリクエストが送信され、新しい機能の使用を開始できる時期に関する通知を受け取ります。
