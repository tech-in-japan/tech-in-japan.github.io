PHP Digest No. 164 (August 27 - September 9, 2019)







Fresh selection with links to news and materials. In the release: PHP 7.4.0 RC1, Laravel 6, Monolog 2 and other releases, Union Types and other news from PHP Internals, a portion of useful tools, a pack of podcasts and much more.



Enjoy reading!









News and Releases



  • PHP 7.4.0 RC1 - The release candidate cycle for branch 7.4 has started. A total of at least 6 issues are expected to stabilize.
  • PHP 7.1.32 , PHP 7.2.22 , PHP 7.3.9 - Releases with fixes for critical security errors, it is recommended that all users upgrade.
  • Monolog 2.0.0 - After 4 years of development, Monolog 2.0 is introduced. Requires PHP 7.2, uses typinty where possible and strict_types. Added new handlers (SqsHandler, TelegramBotHandler) and more. Monolog 1.x will continue to be supported.


PHP internals



  • [RFC] Reclassifying engine warnings - It is proposed to review the dropped notices and vorings in the PHP engine and adjust the classification where necessary. In many cases, it is proposed to increase the error level: Notice -> Warning, for example, when trying to get a property from a non-object, and Warning -> Error exception, for example, when trying to use a scalar as an array. Well, "Undefined variable" will throw Warning.
  • [RFC] Union Types v2 - It is proposed to introduce combined types - this means that the variable can take one of the listed types. De facto concatenated types have long been used in PHPDoc, but now they will actually be checked by the interpreter itself.

    The proposed syntax T1|T2|...



    can be used wherever types can be specified now:
    Hidden text 
     class Number { private int|float $number; public function setNumber(int|float $number): void { $this->number = $number; } public function getNumber(): int|float { return $this->number; } }


    As an experiment, the RFC is designed as a pull request and anyone can comment or express a reaction in the form of emoji. The final vote will be held as before on wiki.php.net. Judging by the message of Nikita , the first experiment went well and many valuable comments were received from the community.
  • error_reporting = E_ALL in PHP 8 - In PHP 8, the default error level will be set to E_ALL



    instead of the current one: E_ALL & ~E_NOTICE & ~E_STRICT & ~E_DEPRECATED



    .


Instruments



  • RubixML / RubixML - An actively developing library for using machine learning in PHP. Examples are available: classification of human activity according to the accelerometer, object recognition in images , price prediction .
  • tsufeki / tenkawa-php-language-server - LSP implementation for PHP, with type inference based on PHPStan.
  • Nyholm / psr7 - Lightweight and rigorous implementation of the PSR-7.
  • PHPCompatibility / PHPCompatibility 9.3.0 - A set of rules for PHP_CodeSniffer to check the code for compatibility with different versions of PHP. Checks added for PHP 7.4.
  • dbalabka / php-enumeration - Another implementation of enumerations in PHP, without the use of magic methods and Reflection.
  • nunomaduro / yorn - The concept of modules in PHP with the syntax 'import' and 'export' a la JavaScript. The idea is not new , but entertaining.

Laravel



  • Laravel 6 - The major update of the framework is presented at the LaraconEU conference. This is an LTS release and will receive security updates for 3 years. Also now the release follows Semver, which means version 7 will be released this winter. More details about the changes in Russian , and in the video on Laracasts .
  • facade / ignition - Among other things, Laravel 6 has a new error page . It is based on Whoops , but provides more information and even offers to fix simple errors without leaving the browser.
  • beyondcode / laravel-view-xray - Conveniently highlight and sign views on the page.
  • stefanzweifel / laravel-stats 2.0 - Artisan-team that displays a variety of code statistics.
  • PHP Townhall # 68: Behind the Facade - Taylor Otwell visiting Matt Trask and Ben Edmunds discuss what's new in Laravel, the business side of things, and the LaraconUS organization.
