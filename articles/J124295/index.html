オスマンド-ユーザーガイド

OsmandはAndroid用のプログラムであり、OpenStreetMapのオフラインマップを操作したり、音声ナビゲーション、GPXトラックの録音、表示、ナビゲーションだけでなく、OSMを編集するための多数のオプションを使用できます。 前のトピックから判断すると、ハブロフスク市民の中では、このプログラムの複雑な機能を知りたいこのプログラムのかなり多くのユーザーがいます。



もちろん、ユーザーインターフェイスは誰にとってもどんな状況でも絶対に明確でなければならないので、ユーザーガイドを書くことは最近悪い形と見なされていますが、残念ながら現実は反対です。 これは、ユーザーがアプリケーションの基本概念を理解していない場合、またはその機能の過小評価または過大評価に導かれている場合に特に重要です。



Osmandは、ほとんどのLinuxアプリケーションと同様に、すべての人を対象としています。そのため、特定のターゲットオーディエンスに名前を付けることは困難です。 これは、アプリケーションのプラスとマイナスです。 コミュニティは常に小さくて必要なパッチを追加し、大きくて必要なパッチはめったに追加しないため、原則として、すべてのオープンソースはこのシナリオに従って開発されます。 これにより、多数の設定、多数の重複する機能、および数千のユースケースを持つアプリケーションが実現します。

地図



マップには、ベクターとラスターの2種類があります。 ラスターマップは、画像（タイル）で表されるマップです。ベクトルマップは、タイルが表示用にまだ描画されていないデータのバイナリセットです。 一般的に、根本的な違いは、ラスターマップが非常に迅速に描画され、詳細な領域のために多くのスペースを占有することです。 たとえば、Googleマップ/ Yandexマップ-ラスターマップ、Garmin / Navitel-ベクトル。 最初の起動時に、Osmandはラスターマップを使用し、それらをインターネットからSDカードにロードしてから、これらのマップが使用されます。 ベクトルマップは、特別なダイアログ-設定→索引→索引のダウンロードでロードする必要があります。 マップを切り替えるには、メニュー→レイヤー→マップソースを使用します...



ラスターマップ


表示速度が重要であり、手元のインターネットでSDカードを詰まらせたくない場合、ラスターカードは非常に便利です。 デフォルトでは、2つのラスターマップソースが作成されますが、「インストール..」を使用して他のソースをダウンロードできます。 ソースのインストール時に、すべてのタイルがダウンロードされるフォルダーがSDカード（/ sdcard / osmand / tiles / <source name>）に作成されます。 このフォルダーを削除することにより、プログラムはソースがないと見なします。 これは、裁量でソースのリストを作成するための非常に便利な方法です。



独自のタイルソース（ネイティブレンダリング、WMS）を作成する方法もあります。 / sdcard / osmand / tiles / <source name>に移動すると、タイルのロードに関するすべての必要な情報が含まれている.metainfoファイルを見つけることができます。 OsmandMapCreatorをより簡単に使用する方法 。 このプログラムを使用すると、関心のある領域（大）を選択してロードし、SDカードにコピーできます。 携帯電話のWifiを使用して、広いエリアをダウンロードできます：コンテキストメニュー→ダウンロードマップ。 PCを使用してマップを準備する主な利点は、タイルを1つのファイルに入れ（sqliteデータベースを作成）、そのファイルをosmand / tiles /にコピーできることです。 多数のファイルがファイルシステムを不必要に劣化させるため、これはフラッシュカードにとって非常に重要です。 SQliteファイルは、インターネットからタイルもダウンロードします。



インターネットトラフィックを監視する人のための設定があります（設定-マップ/一般）：インターネットを一般的にオフにしてタイルをダウンロードし、タイルをロードする必要のない詳細レベル（ズームレベル）を指定できます。 たとえば、カーナビゲーションの場合、オスマンドはタイルを2〜4倍、それぞれ16〜17倍増やすことができるため、14〜15で十分です。



ベクトルカード


ベクトルマップを表示するには、これらの同じマップをダウンロードする必要があります。 フィルター付きのダウンロードの膨大なリストが画面に表示されます。ダウンロードのリストはDownloadsと同じです。 お住まいの地域に加えて、1）音声データ、2）ベースマップ（世界地図1-7ズーム3 Mb）をダウンロードすることをお勧めします。 ベクトルマップの利点は、さまざまなレンダリングスタイルを回転およびサポートできることです。 レンダリングスタイルは現在のプロファイル（後述）に依存しますが、手動で指定できます。 独自のスタイルを作成して（osmand / rendering）に配置することもできます 。 高さプロファイルを持つマップを作成することもできます 。 また、ベクトルマップは、昼と夜に応じてスタイルを変更し（設定で確認できます）、名前を英語に変換できます。



ベクトルマップは細部が非常に遅いため、ズーム<= Nでラスターマップに切り替えるオプションがあります。メモリ消費とレンダリング時間のかなり効果的な組み合わせ。



基板マップとカバー


このアプリケーションには、必要なレベルの透明度を備えた基材またはコーティングとして任意のカードを使用する機能があります。 当然、これによりレンダリングプロセスが遅くなり、アプリケーションにとって非常に貴重なメモリが消費されます（クラッシュする可能性があります）。



地図画面



状況に応じて、地図には速度インジケーター、到着時間インジケーター（残り時間インジケーターが押された場合）、詳細インジケーター（ズーム）、地図の中心から目的地までの距離のインジケーター（押された場合は目的地に移動します）、矢印-ボタンが戻ります現在の位置、 コンパス （押した場合、地図をコンパスの周りで回転させるか、回転を無効にする）、 設定プロファイル （押した場合、選択可能）。



目的地は、フラグ付きの赤い点として地図上に表示されます。表示されていない場合は、画面に方向+距離が常に表示される赤い矢印が表示されます。これは歩行者のナビゲーションに非常に便利です。 目的地は、任意の検索（[移動]）で選択できます。場合によっては、長押しを使用するか、マップのコンテキストメニューを使用する必要があります。 ルートをプロットするときは、目的地を選択する必要があります。 メニュー→マーク解除で削除できます。



マップコンテキストメニューは、アクションがマップ上の選択されたポイントに関連付けられているメニューです。 3つの方法を選択できます：トラックポイントを押す、マップを長押しする+表示された領域をクリックする（長押しすると領域が非表示になります）、メニューから。 このメニューでは、選択したポイント1）を目的地として選択できます2）ルートの開始点として使用できます（表示のみ）3）POIを検索するための開始点になります、トランスポート4）SMS、電子メール経由で友人に送信できます5）...



一般設定と設定プロファイル



アプリケーションには、一般設定とプロファイル設定があります。 一般設定：

-画面の向き-携帯電話を手に持ったときにカードが外れないように設定できます

-アプリケーションディレクトリ-/ sdcard / osmandを変更できます。

-ローカリゼーション、距離単位、トラックポイントの使用...



アプリケーションには4つのプロファイルがあります：デフォルト（マップの概要）、自動車、自転車、歩行者。 各プロファイルで選択されたすべての設定がそこに保存されます。 たとえば、歩行者の場合、車に変更するときは、それぞれ毎日のレンダリングを表示するように常に選択されました-この設定はリセットされ、別の設定は自動スケーリングとして設定されます。 ナビゲートすると、設定プロファイルが自動的に変更されます。 各プロファイルにはデフォルト設定がありますが、それらはすべて変更および構成できます。



現時点では、マップとレイヤーは設定に表示されますが、個別のアプリケーション設定であり、プロファイルによって変更されることはありません。



検索する



座標、住所、お気に入り、POI、交通機関、および検索履歴で検索できます。 住所、POI、交通機関については、ベクターデータが必要です。 必要なレイヤー（メニュー→レイヤー）、フィルターPOIレイヤー、お気に入りレイヤー、またはトランスポートストップレイヤーを含めることで、マップ上で検索を視覚化できます。 すべての検索で、「地図に表示」と「移動」の2つのボタンがアクティブになります。お気に入り、履歴には、長押しすると「移動」オプションが表示されます。 デバイスの検索ボタンをクリックすると、検索を開始できます。 検索は「英語名を表示する」オプションの影響を受けます。



住所


すべてが非常に簡単です：地域（ベクターマップの名前）、都市、通りなどを選択します。 Osm Nominatimを検索するオプションもあり、インターネットを使用し、クエリの作成方法を知っている人に推奨されます。



ポイ


フィルターは、POIの検索に使用されます。 編集可能でユーザーが作成したものには、標準のOSMフォルダーであるアスタリスクが表示されます。 編集するには、フィルターを長押しするか、新しいフィルターを選択する必要があります。メニューからフィルターを「名前を付けて保存」または「削除」できます。 フィルターを選択すると、2つのオプションが表示されます：近くの検索（アプリケーションが衛星を見つけるまで時間がかかります）またはマップの最後に選択したセクションの周辺を検索します。 見つかったPOIを名前でフィルターする方法があります-メニューボタン→フィルター。



注目の


マップのコンテキストメニューからお気に入りに追加できます。 車、ホテル、フォレスト内の最後の会議場所など、動的なお気に入りの個別の「置換」オプションもあります。選択したポイント（GPXファイル）をエクスポートおよびインポートできます。アプリケーションをアンインストールして再インストールするときに重要です。 選択した多数のポイントをインポートすることはお勧めできません;これには「GPXファイル」レイヤーを使用することをお勧めします。



輸送


車両検索は、手動ルーティングとも呼ばれます。 検索するときは、開始点（任意）が必要なので、トランスポートの検索はマップのコンテキストメニューから利用できます。 ターゲットへの輸送を検索するための碑文はクリック可能です。ある場合には目標が考慮され、別の場合には目標は考慮されません。 輸送を検索する場合-結果は、開始から経路+経路から目的地までの最小量でソートされます。 ルートを選択するとき、行くストップを選択することをお勧めします。 その後、検索が繰り返されますが、開始点は前のルートの停止点になります（後の検索）。 したがって、転送を使用してルートを構築できますが、転送のストップを手動で選択する必要があります。



バックグラウンドサービス



バックグラウンドサービスは、画面をオフにしてGPSを継続的に使用しない場合に役立ちます。 バックグラウンドサービスは、一定の間隔でGPS測位を開始し、GPXトラックの記録とナビゲーション（音声を含む）の2つの機能を実行します。 設定でバックグラウンドサービスを有効にできます。



ナビゲーションとルーティング



ルートを敷設するには、ルートポイントを選択してから、メニュー-ルートを選択する必要があります。 ルートを作成するための2つのモードがあります：フォロー（ナビゲーション）とルートの表示。 あなたは、敷設されたルートに関する情報を表示することができますメニュー-ルート-ルートについて。 フォローモードを選択すると、プロファイルが自動的に変更され、現在の場所からルートが配置され、音声ナビゲーションがオンになります。



ルートを構築するための3つのソースがあります。オンラインソース（Cloudmade、Yours）、オフラインベクターデータナビゲーション（不安定）、GPXトラックナビゲーションです。 最も最適なのはCloudmadeですが、インターネット接続が必要です。 ベクターデータナビゲーションは開発中であり、短距離でのみ使用でき、トラック/ラップなどへの出口をまだ決定していません。 また、 maps.cloudmade.com （マップ-メニュー-GPXナビゲーション）などで作成されたGPXトラックをナビゲートすることもできます。



監視およびGPXファイル



/ sdcard / osmand / tracksのGPXファイルの監視と記録は、一部のプロファイル（自動、自転車）でデフォルトで有効になっているため、動きの履歴は毎日GPXファイルに保存されます。 これらの設定は、目的の追跡間隔を選択することで、他のプロファイルのオンとオフを切り替えることができます。



アプリケーションでは、記録されたトラックを表示したり、他のユーザーが作成したトラックを表示したりできます。 これを行うには、GPXファイルを/ sdcard / osmand / tracksに配置する必要があります。 レイヤー-GPXを使用して有効にできますが、すべてのセグメントはマップ上に線で表示され、トラック内のすべてのポイントはお気に入りとして表示されます。 レイヤーがオンの間は、お気に入りリストでも利用できます。



いつでも、記録されたGPXファイルに説明付きのポイントを追加できます;これは、マップコンテキストメニューから実行できます。 これは、インターネットがない場合にいくつかのPOIを記録するのに役立ちます。

  • 役に立つヒント（疑問符アイコン）を忘れずに見てください。アプリケーションの最新の変更が公開されています。
