株式取引所



企業がビジネスの開発を継続したり、新しいプロジェクトを立ち上げたり、拡大したりする場合、資金調達ツールとして株式を使用できます。 株式は会社の所有権の株式を表し、株式は現金と交換されます。 株式は売買することができます。このような取引の参加者は、以前に発行された既存の株式を操作します。



特定の株式の売買価格は、株式を発行した会社の業績に関係なく、常に変化する可能性があります。すべては需要と供給によって決定されます。 株式と、たとえば借入資金を引き付けるために使用される債券（債券）との違いを理解することが重要です。



取引に関しては、株式の売買だけでなく、金融商品や、貴金属や石油などの資源を含むさまざまな資産で取引を完了することができます。



株式を購入すると、投資家は会社の特定の株式を受け取り、そこからこの株式を売却することにより、将来的に金銭的利益を得ることができます。 戦略は異なる可能性があります。株式のさらなる成長を期待して長い取引（長い）があり、投資家が株式がより安くなると仮定すると短い取引があります。したがって、将来、より低い価格で「買い戻す」ことを期待して株式を売却します。



トレーディング戦略の開発には、たとえば機械学習モデルの構築に類似したいくつかの段階が含まれます：最初に戦略を策定し、コンピューターで実行できる形式で記述し、次に結果のプログラムのパフォーマンスをテストし、最適化してから有効性を評価する必要がありますと信頼性。



取引戦略は通常、バックテストの助けを借りてチェックされます。これは、戦略がオークションの履歴データで「実行」されるアプローチです-プログラムに基づいてトランザクションが生成されます。 これにより、そのような戦略が過去に観察された市場状況の発展に収入をもたらすかどうかを理解することができます。 したがって、入札の戦略の見通しをリアルタイムで事前に評価することが可能です。 同時に、実際の市場で作業する場合、履歴データの良好なパフォーマンスが繰り返されるという保証はありません。



時系列データ



時系列は、連続する等しい時間間隔で取得される一連のデジタルデータです。 金融では、これらのシリーズを使用して、一定の間隔で記録された一定期間の価格の動きを追跡します。 これは次のようなものです。







X軸には日付があり、価格はY軸にあります。この場合の「連続した等しい時間間隔」とは、日付が2週間の間隔で時間軸に沿って配置されていることを意味します。2005年3月7日と2005年3月31日を比較できます。 2005年4月5日および2005年4月19日（ ここで、日付は月が最初に、次に日が来たときに米国で受け入れられる形式で記録されます ）



ただし、財務データには通常2つのパラメーター（価格と日付）が含まれませんが、5つ-取引期間のサイズに加えて、これは取引期間の始値、その期間内の最高値と最低価格、期間の終値です。 これは、1日の期間を考慮している場合、データの分析により、選択した日の取引の開始時と終了時の価格レベル、および入札中の最高価格と最低価格に関する情報が得られることを意味します。



このガイドの学習を続けるために知っておく必要のある基本的な概念は、上記で説明されています。



Pythonファイナンスの基礎：パンダ



Pythonを使用して金融アプリケーションを開発するときに最も要求されるツールの1つは、Pandasパッケージです。 それは最初からすでに必要ですが、開発プロセスを深くするにつれて、NumPy、SciPy、Matplotlibなどのパッケージも必要になります。



はじめに、パンダに焦点を当て、このツールを時系列分析に適用してみましょう。 以下では、このパッケージを使用してデータをインポート、分析、操作する方法について説明します。



財務データをインポートする



pandas-datareaderパッケージを使用すると、Google、Yahoo！などのソースからデータを受信できます。 金融機関または世界銀行-利用可能なデータソースの詳細については、 ドキュメントをご覧ください。 このガイドでは、Yahoo！からのデータの取得について説明します。 ファイナンス 開始するには、pipを使用してパッケージの最新バージョンをインストールする必要があります。 



pip install pandas-datareader


開発中のバージョンをインストールする手順をここに示します 。 



 import pandas_datareader as pdr import datetime aapl = pdr.get_data_yahoo('AAPL', start=datetime.datetime(2006, 10, 1), end=datetime.datetime(2012, 1, 1))


少し前まではYahoo APIに変更があったため、自分でライブラリを操作するには、公式パッチを待つことができるパッチをインストールする必要があります。 この問題については、 ここで詳しく説明します 。 ただし、このガイドでは、データは事前​​にダウンロードされているため、学習しても問題はありません。



また、pandas-datareaderはデータをロードするための便利なツールですが、Pythonの唯一のツールではないことを理解することも重要です。 Quandlなどのライブラリを使用して、Google Financeサービスからデータを受信することもできます 。 



 import quandl aapl = quandl.get("WIKI/AAPL", start_date="2006-10-01", end_date="2012-01-01")


また、多くの人々は、Excelがデータ分析の金融分野で非常に人気があることを知っています。 将来の作業の便宜のために、このツールをPythonと統合できます（ リンクについて詳しくはこちら）。



時系列データの操作



データをインポートするために、pandas_datareaderを使用しました。 その結果、aaplオブジェクトが出現しました-これはDataFrame、つまり、潜在的に異なるタイプの列を持つ2次元の名前付きデータ構造です。 このようなフレームで作業する場合、最初に行うことは、データフレームの最初と最後の列を見るためにhead（）およびtail（）関数を実行することです。 ダウンロードしたデータの有用な統計概要を取得するには、describe（）関数を使用できます。



このコードの例は、ソース資料ページにあります 。



データには、取引期間の始値と終値、および最高価格​​と最低価格の4つの列が含まれています。1日の間隔とApple株を考慮します。 また、2つの追加の列：VolumeとAdj Closeも取得します。 それらの最初のものは、取引日に取引が行われた株式数を記録するために使用されます。 2番目の列は「調整済み」終値です。これは、次の取引日の始業前に完了している可能性のある在庫に対するすべてのアクションが期間の終値に追加されることを意味します。



データをCSVファイルに保存する必要がある場合、これはto_csv（）関数を使用して行うことができ、read_csv（）を使用してファイルを読み取ることができます-これは、データソースが変更され、そのアクセスが一時的に失われる状況に役立ちます。 



 import pandas as pd aapl.to_csv('data/aapl_ohlc.csv') df = pd.read_csv('data/aapl_ohlc.csv', header=0, index_col='Date', parse_dates=True)


ダウンロードしたデータの基本的な分析が終わったら、次に進みましょう。 これを行うには、たとえば、特定の列の最後の10行を選択して、インデックスと列を調べることができます。 これはサブセットと呼ばれます。これは、使用可能なデータの小さなセットのみが取得されるためです。 結果のサブセットはシリーズ、つまり、1次元の名前付き配列です。



インデックス列とデータ列を確認するには、インデックス属性と列属性を使用します。 その後、列columnの最後の10個の観測値のサブセットを選択できます。 これらの値を分離するには、角括弧を使用します。 最後の値はts変数に配置され、そのタイプはtype（）関数を使用してチェックされます。 



 # Inspect the index aapl.index # Inspect the columns aapl.columns # Select only the last 10 observations of `Close` ts = aapl['Close'][-10:] # Check the type of `ts` type(ts)


角括弧を使用するのは便利ですが、これはパンダで作業する際の最も一般的な方法ではありません。 したがって、関数loc（）およびiloc（）も検討する価値があります。最初の関数はラベルベースのインデックス作成に使用され、後者は位置のインデックス作成に使用されます。



実際には、これは、2007または2006-11-01のようなシリーズラベルをloc（）関数に渡すことができ、22または43のような整数がiloc（）関数に渡されることを意味します。 



 # Inspect the first rows of November-December 2006 print(aapl.loc[pd.Timestamp('2006-11-01'):pd.Timestamp('2006-12-31')].head()) # Inspect the first rows of 2007 print(aapl.loc['2007'].head()) # Inspect November 2006 print(aapl.iloc[22:43]) # Inspect the 'Open' and 'Close' values at 2006-11-01 and 2006-12-01 print(aapl.iloc[[22,43], [0, 3]])


パーティション手順の結果を注意深く見ると、データに特定の日が欠落していることがわかります。 パターンをさらに分析すると、通常2日または3日では十分ではありません。 これらは週末と祝日であり、その間は為替取引は行われません。



インデックス付けのほかに、データについてさらに学習する方法がいくつかあります。 たとえば、20行のデータのサンプルを作成し、それらを再フォーマットして、applが非日次値および月次値になるようにすることができます。 これを行うには、sample（）およびresample（）関数を使用します。 



 # Sample 20 rows sample = aapl.sample(20) # Print `sample` print(sample) # Resample to monthly level monthly_aapl = aapl.resample('M').mean() # Print `monthly_aapl` print(monthly_aapl)


データの視覚化と財務分析に進む前に、取引期間の始値と終値の差の計算を開始できます。 この算術演算は、Pandasを使用して実行できます。applデータのOpen列の値をClose列から減算する必要があります。 または、言い換えると、aapl.Openからaapl.Closeを減算します。 結果の結果は、diffと呼ばれるaaplデータフレームの新しい列に格納され、del関数を使用して削除できます。 



 # Add a column `diff` to `aapl` aapl['diff'] = aapl.Open - aapl.Close # Delete the new `diff` column del aapl['diff']


結果の絶対値は、財務戦略の開発にすでに有用である可能性がありますが、通常、たとえば特定の株の成長率や価格の下落など、より深い分析も必要です。



時系列データの可視化



関数head（）、tail（）、およびインデックス付けを使用したデータ分析に加えて、それらの視覚化も可能です。 PandasとMatplotlibチャート作成ツールの統合により、これは非常に簡単に行えます。 plot（）関数を使用して、関連するパラメーターを渡すだけです。 さらに、グリッドパラメーターを追加すると、結果のグラフがグリッドに重ねられます。 



 # Import Matplotlib's `pyplot` module as `plt` import matplotlib.pyplot as plt # Plot the closing prices for `aapl` aapl['Close'].plot(grid=True) # Show the plot plt.show()
