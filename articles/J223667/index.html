Mail.Ru for business：あなたが望んでいて、askすることをeverythingしなかったすべて





数か月前に Mail.Ru for Businessプロジェクトを発表した方法を覚えているかもしれません。



Mail.Ru for Businessは、ドメインに最大5000個の企業アドレスを作成でき、ユーザーがMail.Ru Mailリソースの全量にアクセスできるようにする無料のサービスです。 同時に、スパムやウイルスに対する信頼性の高い保護を提供し、無制限のメールボックスボリュームを提供します。 メールは、IMAP、POP3 / SMTP-Outlook、The Batをサポートするサードパーティクライアント、およびすべての一般的なプラットフォームのモバイルアプリケーションを通じて使用できます。 メールの他に、仮想データストレージ用のクラウド、仕事を整理するためのカレンダー、同僚とのインスタントメッセージング用のエージェントなど、他のツールをユーザーが利用できます。



コメントでは、実装する必要がある多くの「ウィッシュリスト」を説明しました。 そして、あなたの願いの大部分は、私たちは正常に完了しました。 今、私たちは持っています：



  • mail.mydomain.tldによるログイン
  • ウェブサイトのフォームからログインします
  • API
  • ドメインエイリアス
  • ユーザーエイリアス
  • 他のメールサービスからの移行
  • ファイルからの一括アップロード
  • DKIM署名
  • ユーザーアクティビティレポート
  • クラウドストレージへのアクセス




以下のすべての新機能の詳細をご覧ください。



ところで、彼らが興味を持っている場合、またはビジネスサービスのMail.Ruについて聞いたことがない場合でも、私たちはそれらを実際に試す機会を与えて喜んでいます。 コメントで「試してみたい」と書いた最初の100人のhabruzersに、プロモーションコードを送信して、.ruゾーンの無料ドメインをメールサービスに接続して登録します（もちろん、Aレコードを変更する可能性があります）。



mail.mydomain.tldおよびサイトのフォームを使用してログインします



起動時に、Mail.Ruのメインページとbiz.mail.ruの特別ページからエントリを提供しました。 これで、メールサブドメイン（または任意のその他）を使用し、Webサイトのフォームを使用して、ユーザーのメールアクセスを構成できます。



サブドメインを介して入力するには、値biz.mail.ruでサブドメインにCNAMEレコードを作成する必要があります。



サイトのログインフォームには、次のテンプレートが用意されています。 



<form method="post" action="https://auth.mail.ru/cgi-bin/auth?lang=ru_RU"> <input name="Login" type="text" value="" placeholder=" "> <br> <input autocomplete="off" type="password" name="Password" value="" placeholder=""> <br> <input type="submit" value=""> <input type="hidden" name="Domain" value="__"> </form>




ドメインを接続してユーザーを作成するためのAPI



Reg.ru、Setup.ru、Webnames.ruの同僚とともに、パートナーインターフェイスから直接1つのボタンでドメインを接続する機能を実装しました。 この機能は、公開されたAPIを公開することで利用可能になりました。 ドメインの接続に加えて、APIクライアントはドメイン上にユーザーを作成できるため、クライアントにメール機能を提供し、1つの使い慣れた場所からメールアカウントを管理できます。

使用するドキュメントはhttps://biz.mail.ru/developerにあります 。 APIへの接続とキーの受信は依然として手動で実行されます。試してみたい場合は、PMまたはo.parinov@corp.mail.ruにここに書いてください。



メールボックスとドメインのエイリアス



ほとんどの場合、エイリアスはボックス名の短い変形として、または疑わしいサービスで一時的に使用するために使用されます。



ドメインユーザーごとに、最大5つのメールボックスエイリアスを構成できます。 エイリアスアドレスに送信された手紙は、メインアドレスに配信されます。 また、メールWebインターフェースからレターを送信する場合、送信者としてエイリアスアドレスを選択できます。



また、 ドメイン全体にエイリアスを追加する機能も提供しました。 同じ目的で複数のドメイン（たとえば、異なるゾーンの同じドメイン）がある場合、メールを受信する機能を複製できます。 これを行うには、メインドメインを選択してシステムに接続し、[ドメインシノニム]セクションで追加のドメインを接続します。 その後、耳で住所を認識する際にエラーがあったとしても、手紙は別のドメインの対応するボックスの正しい宛先に届きます。



他のメールサービスからの移行



文字通り、ローンチの瞬間から、多くのユーザーから非常に合理的な質問が寄せられています。 もちろん、通信アーカイブの重要性は理解しました。 そのため、新しいメールボックス用に古いメールサーバーからのメールのコレクションを構成する機能である移行システムを開発しました。



新しいメールボックスを作成するとき、同じパスワードの使用を確認するか、以前のパスワードを設定することにより、コレクターを開始するように設定できます。その後、このメールボックスのすべての文字が古いサーバーから転送されます。 設定と転送オプションの有効化は、「移行」セクションで実行されます。 詳細はこちら 。



ファイルから一括アップロード



移行に伴い、すぐに多くのメールボックスを作成する可能性について尋ねられましたが、インターフェースを介して一度に1つではありません。 最も簡単で最速の方法を提案しました-少なくともボックスのユーザー名とパスワードを含むCSVファイルからユーザーを追加します。 その後、姓と名（3列目と4列目）、カンマで区切られた着信グループのリスト（5列目）、および移行を有効にするための古いサーバーのパスワード（6列目）を設定する機能も追加しました。 ファイルは、ユーザーリストの上にある[一括アップロード]ボタンを使用してダウンロードされます。 詳細はこちら 。



DKIM署名



署名DKIMを使用すると、レターの送信者の信頼性を確認でき、レターの「配信可能性」が向上します。 送信者との通信はデジタル署名を使用して確立され、受信者側で検証されます。 Mail.Ruを含むほとんどのメールサービスは、DKIMスキャン結果を使用してスパムをフィルタリングします。



これで、ドメインから送信されたレター用に独自のDKIM署名をインストールできます。 DKIM署名は「サーバーステータス」セクションで設定されます-ドメインのDNSエディターで必要なTXTレコードを設定するだけです。 （ところで、次の投稿の1つで、これをどのように行ったかについて詳しく説明する予定です）。



ユーザーアクティビティレポート



メール内の各企業ユーザーのアクションが記録および記録されます。 これらのレコードは、ユーザーページのドメイン管理インターフェイスで管理者が利用できるようになりました。 これで、管理者は、メールボックスで発生したイベントの履歴を常に表示し、時間内に不要なアクティビティに気付くことができます。



クラウドストレージアクセス







100ギガバイトものクラウドストレージが無料で永遠に企業ユーザーに登場しました。 それらのそれぞれは、メールとともに、クラウド内の100ギガバイトの仮想ストレージを受け取ります。つまり、同僚とファイルを交換し、重要な情報を保存することがはるかに簡単になりました。 Webバージョンのクラウドを使用したり、職場や自宅のコンピューター、モバイルデバイスにアプリケーションをインストールしたり、どこからでもすべてのファイルにアクセスできます。
