Red Hatワークショップ。 印象

2月21日火曜日、 Red Hatパートナーシップワークショップが開催されました。 あなたが突然「これについて何も聞いていない」のような考えがあった場合、私は答えます-あなたはRed Hatのアフィリエイトステータスを持っていないからです。 :-)



ですから、このセミナーは私が以前開催したセミナーとは非常に異なりました。 その主な部分は、マーケティング、アフィリエイトプログラム、ロジスティクスなどのがらくたについての話に専念しましたが、私は完全には興味がありませんでした。 あなたもそう思う。 :-)しかし、上記のトピックの中には、Red HatシニアソリューションアーキテクトのKarl Stevensによる「Red Hat Cloud Strategy」と「RHEV 3.0」という2つのレポートがあります。 彼が語ったことはすべて、いつかここで間違いなく書きます。 しかし、当初は意図したとおり、印象の説明とJava Dayとの比較があります 。





Red HatワークショップがDigital Octoberで開催されました。 ハブラーでは誰もこの場所が何であるかを言う必要はないと思います。 DOの内壁にある「むき出しのレンガ」は、Java Dayが開催されたホテルのヨーロッパ品質の改修と非常に対照的でした。 DOに行くのはあまり便利ではありませんが、そこに着いた人たちはホットコーヒーと温かい歓迎を待っていました。 まあ、一般的に、この場所は、サイバーパンクのような「ITの精神」があるという意味で、非常に楽しいことがわかりました。





北および東ヨーロッパのRed Hatのビジネス開発ディレクター、フィルアンドリュースは、人類の偉大な未来について語っています。 :-)



すでに書いたように、セミナーはパートナーシップであったため、パートナー企業の代表者が参加しました。 そのため、20人を超える人々が集まり、温かく親しみやすい雰囲気の中で開催されました。 ところで、そのような少数の人々にもかかわらず、翻訳者が見つかりました（JavaDayとは異なります）。 そして、Red HatがJava Dayなどの主要なイベントを計画しているかどうかを傍観したところ、現時点では、今後のロシアオープンソースサミット2012などの「一般的な」主要イベントをサポートすることが会社の方針であると言われました。 そして現在、Red Hatはロシアのパートナーがオープンソースでビジネスを構築するのを支援することに焦点を当てています。





Karl Stevens（Karl Stevens）は、対応するロールアップの背景にあるRed Hatのクラウド戦略について話しています。



Java Dayの場合のように、Red Hatソリューションの開発の主な「ベクター」はクラウドコンピューティングに向けられています。 これは、IT業界全体の一般的なコースです。 技術レポートは2つしかありませんでしたが、個人的には両方とも非常に有用でした。 まあ、一般的には、会社の計画について少し早く知るのは良いことです。 =）指摘されているマイナスのうち、プロジェクター用に設計されたプレゼンテーションは、「プラズマ」での表示にはあまり適していません。



これは、配布資料のセットのようです。





ちなみに、アンケートに記入した人には次のヘッドフォンが贈られました。





そのため、2月に2つのイベントに参加しました。JavaDayとパートナー向けのRed Hatセミナーです。 企業が異なる活動分野を持っているからではなく、単にイベントの形式が異なっているからです。 したがって、仮想化、OS、ミドルウェア、クラウドコンピューティング、その他のシステムソフトウェアの分野で競合する企業の活動を比較するという私の最初のアイデアは失敗しました。 :-)これにもかかわらず、私の「印象」が誰かに役立つことを願っています。
