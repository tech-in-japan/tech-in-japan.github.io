macOS用のメニューバーアプリ

メニューバーでホストされるアプリケーションは、macOSユーザーに長い間知られています。 これらのアプリケーションには、「通常」の部分があるものと、メニューバーにのみあるものがあります。

このガイドでは、有名な人からの引用をポップアップウィンドウに表示するアプリケーションを作成します。 このアプリケーションを作成する過程で、次のことを学びます。



  • メニューバーでアプリケーションアイコンを割り当てます
  • アプリケーションをメニューバーでのみホストする
  • カスタムメニューを追加
  • イベントモニタリングを使用して、ユーザーの要求時にポップアップウィンドウを表示し、必要に応じて非表示にします


注：このガイドは、SwiftとmacOSに精通していることを前提としています。


はじめに



Xcodeを起動します。 次に、 ファイル/新規/プロジェクト...メニューで、 macOS /アプリケーション/ Cocoa Appテンプレートを選択し、 次へをクリックします。



次の画面で、 製品名としてQuotesと入力し、 組織名と組織識別子を選択します。 次に、アプリケーション言語としてSwiftが選択され、[ ストーリーボードを使用]チェックボックスがオンになっていることを確認します。 [ ドキュメントベースのアプリケーションの作成] 、[ コアデータの使用] 、[ ユニットテストを含める]、[UIテストを含める]チェックボックスをオフにします 。







最後に、もう一度[ 次へ ]をクリックし、プロジェクトを保存する場所を指定して、[ 作成 ]をクリックします 。

新しいプロジェクトが作成されたら、 AppDelegate.swiftを開き、次のプロパティをクラスに追加します。 



let statusItem = NSStatusBar.system.statusItem(withLength:NSStatusItem.squareLength)


ここでは、メニューバーのステータスアイテム（アプリケーションアイコン）に、ユーザーに表示される固定長を作成します。



次に、新しいアプリケーションを区別できるように、メニューバーのこの新しいアイテムに画像を割り当てる必要があります。



プロジェクトナビゲータで、Assets.xcassetsに移動し、写真をアップロードして、資産カタログにドラッグします。



画像を選択して、属性インスペクターを開きます。 Render AsオプションをTemplate Imageに変更します。







独自の画像を使用する場合は、画像が白黒であることを確認し、 テンプレートメニューとして設定して、暗いメニューバーと明るいメニューバーの両方でアイコンがきれいに見えるようにします。



AppDelegate.swiftに戻り、次のコードをapplicationDidFinishLaunching（_ :)に追加します 



 if let button = statusItem.button { button.image = NSImage(named:NSImage.Name("StatusBarButtonImage")) button.action = #selector(printQuote(_:)) }


ここで、アプリケーションアイコンに追加したばかりのアプリケーションアイコンを割り当て、クリックしたときにアクションを割り当てます。



次のメソッドをクラスに追加します。 



 @objc func printQuote(_ sender: Any?) { let quoteText = "Never put off until tomorrow what you can do the day after tomorrow." let quoteAuthor = "Mark Twain" print("\(quoteText) — \(quoteAuthor)") }


このメソッドは、単に引用をコンソールに出力します。

ステータスアイテムにメニューを追加



重大なアプリケーションには、シングルクリック応答だけでは明らかに不十分です。 機能を追加する最も簡単な方法は、メニューを追加することです。 AppDelegateの最後にこの関数を追加します。 



 func constructMenu() { let menu = NSMenu() menu.addItem(NSMenuItem(title: "Print Quote", action: #selector(AppDelegate.printQuote(_:)), keyEquivalent: "P")) menu.addItem(NSMenuItem.separator()) menu.addItem(NSMenuItem(title: "Quit Quotes", action: #selector(NSApplication.terminate(_:)), keyEquivalent: "q")) statusItem.menu = menu }


そして、 applicationDidFinishLaunching（_ :)の最後にこの呼び出しを追加します 



 constructMenu()


NSMenuを作成し、 NSMenuItemの 3つのインスタンスをそれに追加し、このメニューをアプリケーションアイコンメニューとして設定します。



いくつかの重要なポイント：



  • メニュー項目のタイトルは、メニューに表示されるテキストです。 アプリケーションをローカライズするのに適した場所（必要な場合）。
  • action は、ボタンまたは他のコントロールのアクションと同様に、ユーザーがメニュー項目をクリックしたときに呼び出されるメソッドです
  • keyEquivalentは、メニュー項目を選択するために使用できるキーボードショートカットです。 小文字は修飾子としてCmdを使用し、小文字はCmd + Shiftを使用します 。 これは、アプリケーションが最上位にあり、アクティブな場合にのみ機能します。 この場合、アプリケーションにはドックにアイコンがないため、メニューまたは他のウィンドウが表示されている必要があります
  • separatorItemは、他の要素の間の灰色の線の形の非アクティブなメニュー項目です。 グループに使用する
  • printQuoteはAppDelegateで既に定義したメソッドであり、 terminateはNSApplicationで定義されたメソッドです。


アプリケーションを起動すると、アプリケーションアイコンをクリックしてメニューが表示されます。







メニューをクリックしてみてください- 見積の印刷を選択すると、Xcodeコンソールに見積が表示され、見積を終了するとアプリケーションが終了します。

イベント監視



ユーザーがアプリケーションに期待するもう1つのことがあります。ユーザーがポップアップウィンドウの外側をクリックするとポップアップウィンドウを非表示にすることです。 これを行うには、 macOS global event monitorと呼ばれるメカニズムが必要です。



新しいSwiftファイルを作成し、それをEventMonitorと呼び 、その内容を次のコードに置き換えます。 



 import Cocoa public class EventMonitor { private var monitor: Any? private let mask: NSEvent.EventTypeMask private let handler: (NSEvent?) -> Void public init(mask: NSEvent.EventTypeMask, handler: @escaping (NSEvent?) -> Void) { self.mask = mask self.handler = handler } deinit { stop() } public func start() { monitor = NSEvent.addGlobalMonitorForEvents(matching: mask, handler: handler) } public func stop() { if monitor != nil { NSEvent.removeMonitor(monitor!) monitor = nil } } }


このクラスのインスタンスを初期化するときに、リッスンするイベントマスク（キーストローク、マウスホイールのスクロールなど）とイベントハンドラーを渡します。

リスニングを開始する準備ができたら、 start（）がaddGlobalMonitorForEventsMatchingMask（_：handler :)を呼び出し、保存しているオブジェクトを返します。 マスクに含まれるイベントが発生するとすぐに、システムはハンドラーを呼び出します。



イベントの監視を停止するには、 stop（）でremoveMonitor （）が呼び出され、nilに設定してオブジェクトを削除します。



残っているのは、適切なタイミングでstart（）とstop（）を呼び出すことです。 また、クラスは、クリーンアップするためにデイニシャライザーでstop（）を呼び出します。



イベントモニターの接続



最後にAppDelegate.swiftを開き、新しいプロパティを追加します。

 eventMonitor = EventMonitor(mask: [.leftMouseDown, .rightMouseDown]) { [weak self] event in if let strongSelf = self, strongSelf.popover.isShown { strongSelf.closePopover(sender: event) } }


左または右のボタンをクリックすると、アプリケーションに通知されます。 注：アプリケーション内でのマウスクリックに応答してハンドラーは呼び出されません。 これが、ポップアップをクリックしても閉じない理由です。
