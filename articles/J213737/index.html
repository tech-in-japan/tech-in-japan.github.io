ビデオカードのVGA出力からビデオを記録します-鉄の方法

VGA出力からビデオを録画します-これはもっと簡単かもしれません... しかし、悲しいかな、実際には、すべてがもう少し複雑です。 この記事では、 非常に古いコンピューターのビデオカードのVGA出力からビデオを記録する問題をどのように解決したか、どのような中間的な解決策であり、何をやめたかを説明します。 古い技術のゲームのファン、および新しいテクノロジーとビデオキャプチャは、熱心です。







なぜ



もちろん、他の目標もあるかもしれませんが、私には次のニーズがあります。モニターに表示される古いコンピューターからビデオを録画することです。 この場合：



  1. コンピュータは古くて弱いため、一種の「アナログラップ」をインストールすることはできません。 また、タイプライターにはどんな種類の録音ソフトウェアを使用しますか？Win 3.1はきしみを伴いますか？ はい、最初のペンティアムまたはDOSモードでは、何も見つかりません。
  2. 「TV出力付きのビデオカードを接続する」オプションもオプションではありません。 そして、私が最初のブードゥーまたは他のよりエキゾチックなものから写真を記録したい場合は？ そして再び、ソフトウェアに出くわします。 汎用性が必要です。
  3. オプション「エミュレータで実行」-適合しません。 「後期DOS」のゲームは、 DosBoxでエミュレートすると、現代のi7でも速度が低下したり、スムーズになったりします 。 WinXPでもバグがあり、Win98またはWin95より古いOSでは正常に動作するゲームを実行する必要があるが、3次元のアクセラレーションが必要な場合、エミュレーターはあまり役に立たず、ゲームグラフィックスのアクセラレーションは悲しくなります。
  4. カメラで画面を撮影しますか？ いいえ、「スクリーン」のファンがいますが、この方法は倒錯と「最後のチャンス」のスマックです。


鉄を捕獲する



このようなコンピューターは、ビデオキャプチャマシンとして使用されました。



プロセッサー：Intel Core 2 Duo E4500、2.2 GHz

ビデオカード：NVidia 8800 GS

RAM：2 GB DDR2

ハードディスク：SATA経由で接続されたWDC WD1600



特別に彼を迎えに行ったわけではありません-彼はすぐに現れました。



ナンバーワンの試み-中国語



最初の試みは次のアイデアでした： VGA-TVアダプタを選択した場合はどうなりますか？ 安くて陽気な。 そして、結果の画像をTVチューナーとして作成します。Beholderからのもの（モデル番号503、PCIバスのオプション）がありました。 すぐに言ってやった。 PCラップトップTTH-123462用のVGA to TV RCAコンポジットSビデオコンバーターボックスと呼ばれるこの中国のアダプターを20ドルで購入しました 。







別の写真。








この鉄片の興味深い瞬間：

  • USBケーブルで給電されます（丸いプラグがデバイスに接続され、通常のUSBが接続されます）
  • NTSC / PALスイッチがあります
  • VGA入力に加えて、VGA出力もあります！ モニターとテレビチューナーまたはテレビを同時に接続できます。 並列スタックモニターで画質の低下に気付きませんでした。
  • 説明から判断すると（ 確認しませんでした！ ）、反対方向に信号を送信できます。 TV-> VGA。


アイデアは健全に見えましたが、...結果のビデオのスクリーンショットを以下に示します。



スクリーンショット






















実際には、入り口には許可がありません。出力はテレビになります。 そして、圧縮アーチファクトはありません、いや、これはまさに出力画像です。 結果は暖かく、チューブ状で不明瞭です。 一方、20ドル+チューナーの価格（これにお金をかける必要はありませんでした）は許容できる結果です。 しかし、私はもっと何かが欲しかった...



試行番号2-フル



チューナーを試してみたところ、安価な解決策がうまくいかないことが明らかになりました。 私は店を駆け巡り、利用可能な唯一のオプションがすでにPCI-Eの下にあるAVerMedia Game Broadcaster HDボードであることを発見しました。 これは行の最後のモデルではありませんが、微妙な違いがあります。キャプチャ用のVGA入力を備えた最後のモデルのように見えます。 後のものはHDMIのみで、価格は2倍高くなります。 そして、VGAが必要です！







値札：4〜5千ルーブル。

手始めに-肯定的な、ビデオのスクリーンショット：



スクリーンショット
























期待どおり、結果がはるかに優れていることがわかります。 繰り返しになりますが、HDMI入力は、最新のコンソールまたはAndroidデバイスなどからビデオを書き込む機能を意味します。 積極的に。 便利に。



別の楽しい瞬間-コンピューターの「プレビュー」モードでは、コンピューターの処理能力が十分にない限り、リアルタイムで画像が遅れることなく表示されます（詳細については後述）。



そして今-欠点、問題、タールのen腸



  1. ソフト バンドルされたソフトウェアは、 Windows Vista以降でのみ使用できます。 Windows XPのオプションが必要ですか？ メーカーのウェブサイトからダウンロードします。 ドライバーとソフトウェアを含むCDでは、スペースの約70％が無料である（180 MBの領域で占有される）ことを考えると、何と呼ぶべきかわかりません。 私はそのコンピューターにライセンスされたWindows XPを持っています、それは私を怒らせます。



  2. 640x480未満の解像度をキャプチャします。 公式にはサポートされていません。 試行すると、サポートされていない解像度に関するメッセージ、またはシグナルではなくガーベッジが表示されます。 しかし、 VirtualDubプログラムを使用している場合、オープンソースの栄光は、640x480未満の解像度で画像を安定して記録できます。 真実は次のようになります。











    つまり、画像は実際には640x480で書かれていますが、フィールドにゴミがあります。 しかし、誰も私たちに余分なものをカットすることを気にせず、その場で（VirtualDubの作者に再び感謝します）



  3. VoodooおよびVoodoo 2からビデオを録画します。

    ネイティブユーティリティ-数秒書き込み、信号を失い、数秒で「信号なし」と表示され、信号を見つけ、数秒書き込み、信号を失います...



    VirtualDubは問題や異議なく書き込みます。 これで終わりです。 再び馬上のオープンソース。 ;）



    原則として、ネイティブソフトウェアでVoodooから画像を作成することもできます。 これを行うには、別のケーブルを使用して、非古典的な方法でベースカードにブードゥー教を接続します。







    そして、ブードゥー教の下でゲームを開始/オフにするとき、通常のカードとブードゥー教の間にモニターからケーブルを差し込みます。 あなたはそのように生きることができますが、その人生は？



  4. 録音には別のコード、つまり たとえば、最初のコンピューターのサウンドカードのライン出力から録音のためのコンピューターのライン入力まで。 まあ、それは大丈夫です。 ただし、HDMI経由のサウンドは問題ありません。



  5. ビデオを記録およびエンコードするためのコンピューターの能力。

    記事の冒頭に示されたコンピューターは、リアルタイムでh264でビデオを記述およびエンコードするのに十分ですが、解像度は800x600までです。 1280x720を書き込もうとすると、最終記録でフレームスキップが発生しました。 しかし、MPEG2ではこれとリアルタイムでリアルタイムで書くのは簡単ですが、画質は視覚的に少し悪くなります。



  6. 許可。

    VGAを介して接続した場合、解像度はネイティブユーティリティによって800x600であり 、説明や戦争宣言なしで720x576で記録したことがあります。



    HDMIにより、モバイルAndroidコンソール（画面解像度800x480 ）からの画像は720x480 （Khanプロポーション）で記録されました 。



    VirtualDubでは、このような驚きは発生しません。 要するに、私は私のネイティブのユーティリティをスクラップに送りたいのですが、世界には完璧はありません。 入力信号を1280x720（少なくともVGA、少なくともHDMI）で書き込もうとすると、VirtualDubは画像の代わりに何らかの種類の画面を表示します。 ただし、 1280x720を入力に適用し、VirtualDubが入力が実際に1920x1080であると言った場合、いい感じに書き込みますが、解像度を1920x1080に拡張します。 ネイティブユーティリティは、必要に応じてすべてを書き込みます。


独自の結論を導き出します。 VGAでビデオを記述するための最良の方法を知っている場合-コメントへようこそ！
