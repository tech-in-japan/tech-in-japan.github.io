学生がCppCatコードアナライザーを必要とする理由



CppCatは、C / C ++プログラムのエラーを見つけるための単純な静的コードアナライザーです。 私たちはすべての人（学生、教師など）に無料のアカデミックライセンスを発行し始めました。 CppCatを学生の間でより人気のあるものにするために、私はPastebin.comのウェブサイトにある実験室の作品に見られるエラーに関するこのメモを書くことにしました。



CppCatについて少し



CppCatは、Visual Studioと統合する静的コードアナライザーであり、コーディング段階でも多くのタイプミスやその他のエラーを見つけることができます。 アナライザーは、コンパイル後に自動的に起動し、作成したコードを確認できます。 アナライザーは、C、C ++、C ++ / CLI、C ++ / CXをサポートしています。



無料のCppCatライセンスを取得する方法については、 学生向けの無料CppCatの記事で説明しています。 学生だけでなく、大学院生、教師などにもライセンスを与えていることを付け加えたいと思います。



多くは、CppCatが無料のVisual Studio Express環境に統合されていないことに動揺しています。 残念ながら、それについてできることは何もありません。 Visual StudioのExpressバージョンはプラグインをサポートしていません。 ただし、これは重要ではありません。 学生はMicrosoft DreamSparkにアクセスでき、Visual Studio Professionalにアクセスできることを思い出してください。



学生の魅力



初めに、私は精神の中で何かを書きたかった：



学生でさえ静的分析の恩恵を受けることができます。 CppCatがそれを知ることができるのに、プログラムのエラーを探すのが長くて苦痛なのはなぜですか？ したがって、エラーをすばやく見つけるだけでなく、その方法の詳細を見つけることもできます。 CppCatのドキュメントでは、各診断について詳しく説明し、エラーのさまざまな例を示し、それらを修正する方法を提案しています。



それから彼はどういうわけか緊張していることを決めた。 さて、学生はどんな深刻な間違いを持っ​​ていますか？ 10回の繰り返しのサイクルをデバッグすることもできます。 役に立つことさえあります。 したがって、次のようにCppCatを使用するように推奨事項を再定式化します。



雇用主は、あなたのトリッキーなアルゴリズムのプログラミングと記述だけでなく、感謝します。 基本的なツールを使用できることも同様に重要です。



バージョン管理システムが何であり、何をしたのかわからないという事実のために失われた場合、アルゴリズムは価値がありません-USBフラッシュドライブにソースコードのコピーを作成します。



したがって、言語と開発環境だけでなく、補助ツールも学習することが重要です。



最も重要なツールのリストを作成することを想定していません。 ただし、少なくとも1つのバージョン管理システム、WinMergeが必要な理由、プロファイラーとは何か、配布キットの作成方法などを知ることが重要です。



履歴書でよく知られ、言及できる技術の1つは、静的コード分析です。 CppCatは、静的分析方法論の調査に最適です。 これはあなたの知識にとって良いプラスになり、コードの品質について何かを聞いたことを雇用者に示唆します。



そして今、私たちがここに集めたもの



退屈な部分は終わりました。 あなたは今、実験室の作業の誤りを探すと推測できると思います。



コードがしばしば悪いと言うために、今回はそうしません。 したがって、実験室のコードにはさまざまなエラーの暗闇が含まれていることは明らかです。 したがって、できるだけ多くのエラーを検出しないようにタスクを設定しました。 これは面白くない。 私は、ほとんどの学生が学生に明らかにするエラーのパターンをいくつか強調してみました。 しかし、私は明確に3種類のエラーの優位性に気づくことができましたが。 しかし、これについては少し低くなります。



あなたは質問があるかもしれません：私はどこで実験室の仕事を得ましたか？ 答えます。



開発者がコードフラグメントを便利に共有できるウェブサイトPastebin.comがあります。 学生はこのサイトを積極的に使用しています。 C ++タグを持つほとんどすべてのコードは、何らかのラボ作業またはその一部を表します。



Pastebin.com Webサイトを監視し、そこから「C ++コード」とマークされた新しいファイルをダウンロードするプログラムを作成しました。 2,000を超えるファイルを収集し、それらからVisual Studioでプロジェクトを作成して確認しました。 もちろん、半分以上は検証できませんでした。 多くのファイルにはコードの断片のみが含まれ、コメントではないテキストが入力され、一部のライブラリが欠落しています。 しかし、私の目標は、Pastebin.comで公開されている可能な限り多くのコードをチェックすることではありませんでした。 この記事では、チェックされたもので十分です。 ファイルのコレクションは継続されますが、おそらく、このトピックについては後ほど何かを書きます。

パターン1.人気で3位。 同じ条件での混乱



多くのプログラミングタスクには、多くの条件のチェックが含まれます。 そして、それらを実現すると、混乱したり封印されたりするのは簡単です。 適切な事例： 

int main() { int n,a,b,c; cin >> n; for(int i=0;i<n;i++) { cin >> a >> b >> c; if((a % 2==0 && b % 2 ==0 && c % 2!=0)|| (a % 2==0 && b % 2!=0 && c % 2==0)|| (a % 2!=0 && b % 2==0 && c % 2==0)|| (a % 2!=0 && b % 2 !=0 && c % 2==0)|| (a % 2==0 && b % 2!=0 && c % 2!=0)|| <<<--- (a % 2==0 && b % 2!=0 && c % 2!=0)) <<<--- { cout << "1"; } else cout << "2"; } cout << endl; return 0; }


CppCat警告：V501「||」の左と右に同一の副次式「（a％2 == 0 && b％2！= 0 && c％2！= 0）」があります 演算子。 jtzrihcg.cpp 14



導入された3つの変数の値を何らかの方法で巧妙にチェックする必要がありました。 ほとんどの場合、コードがコピーされ、どこでも正しく修正されていません。 その結果、条件の最後から2行目と最後の行が一致します。



別の例 ： 

 int main() { .... } else if(gesucht < geraten) { puts("Ein bisschen zu klein"); } else if (gesucht < geraten) { puts("Ein bisschen zu gross"); } .... }


V517「if（A）{...} else if（A）{...}」パターンの使用が検出されました。 論理エラーが存在する可能性があります。 行を確認：41、43。wrgkuuzr.cpp 40



チェックは2回実行されます（gesucht <geraten）が、異なる行が表示されます。



ところで、両方の例で、エラーは最後の行にあります。 再び、「 最後の行効果 」に出くわしました。



パターン2。人気で2位。 1要素ずつ配列を海外に行く



配列の要素がC ++言語でゼロから番号付けされているという事実は、それを研究する上で大きな困難です。 つまり これは簡単だと理解しているようですが、配列の境界を越えないようにすることを学ぶのは非常に困難です。 配列の10個の要素が必要な場合、A [10]を記述します。 例 ： 

 int main() { .... int rodnecs[10]; .... VelPol1 = rodnecs[1] + rodnecs[3] + rodnecs[5] + rodnecs[8] + rodnecs[10]; .... }


CppCat警告：V557配列のオーバーランが可能です。 「10」インデックスは、配列の境界を超えています。 0z3x9b3i.cpp 38



もっと ： 

 void main() { .... double pop[3][3]; .... for (int i = 0; i<3; i++) { calc_y[i] = F(pop[i][1], pop[i][2], pop[i][3], x[i]); } .... }


CppCat警告：V557配列のオーバーランが可能です。 「3」インデックスは、配列の境界を超えています。 1uj9v9xs.cpp 48



ループ条件での多くの間違った比較： 

 int main() { int i,pinakas[20],temp,temp2,max,min,sum=0; for (i=1;i<=20;i++) { pinakas[i]=rand(); ...... }


CppCat警告：V557配列のオーバーランが可能です。 「i」インデックスの値は20に達する可能性があります。287ep6c0.cpp 20



たくさん ：

CppCat警告：V557配列のオーバーランが可能です。 「i」インデックスの値は10に達する可能性があります。wgk1lx3u.cpp 18

不適切な配列サイズの計算



多くの初心者にとって、C / C ++では、ポインターと配列が異なるエンティティであることを理解するのは困難です。 その結果、次のようなコードに遭遇することがよくあります。 

 int arrayLen(int p[]) { return(sizeof(p)/sizeof(*p)); }


CppCat警告：V511 sizeof（）演算子は、配列のサイズではなく、ポインターのサイズを「sizeof（p）」式で返します。 seprcjvw.cpp 147



確かに、arrayLen（）関数はどこでも使用されていません。 どうやらそれが動作しないという事実による。 :)



別の例 ： 

 bool compare_mas(int * mas, int * mas2){ // -    const auto mas_size = sizeof(mas) / sizeof(mas[0]); // -    const auto mas2_size = sizeof(mas2) / sizeof(mas2[0]); .... }


CppCat警告：
  • V514ポインターのサイズ 'sizeof（mas）'を別の値で除算します。 論理エラーが存在する可能性があります。 0mxbjwbg.cpp 2
  • V514ポインター「sizeof（mas2）」のサイズを別の値で除算します。 論理エラーが存在する可能性があります。 0mxbjwbg.cpp 3

オンラインアナライザーを実行する予定がない理由



「オンラインコード検証用のシステムを作成しないのはなぜですか」という質問を予見します。 たとえば、コードを挿入して[チェック]ボタンをクリックできるフォームがあります。 または、サイトpastebin.comを監視しているので、検証の結果をどこかにアップロードしてみませんか？



私たちはこれをする必要がないと確信しています。 これには3つの理由がありますので、このトピックに関する議論を始めないでください。



理由：
  1. 私たちもユーザーもこれを必要としません。 これにより作業が追加されます。 そして、ユーザーは新しいものを取得しません。 彼はPVS-StudioまたはCppCatをダウンロードしてインストールし、必要なすべての実験を実行できます。 デモ版で十分です。 多くの場合、「コードの貼り付けと確認」フォームは、試用版をダウンロードできないユーザーによって実行されます。 できます。 さらに、機能的な制限もありません。 他の誰かが彼はWindowsを持っていないと言うかもしれません、そして彼は何かを試してみたかったです。 ただし、Windowsがないため、ユーザーではありません。
  2. このようなシステムは、静的アナライザーの機能の評価を大きくゆがめます。 このトピックに関する記事：静的解析に関する神話。 神話5-ツールを評価する小さなプログラムを作成できます 。 合成例ではなく、実際のプロジェクトでアナライザーをテストしてほしい。
  3. 私が言ったように、私たちは合成例をテストしたくありません。 また、インフラストラクチャの観点からプロジェクト全体を検証することは困難です。 インタビューでこれについてもっと読んでください。 簡単に言うと、ソース、ライブラリのダウンロード、ビルドパラメーターの構成などが必要な複雑なシステムを作成する必要があります。 そうでなければ、完全な分析は不可能です。 アナライザーのダウンロード、インストール、および確認が簡単であることがわかりました。
