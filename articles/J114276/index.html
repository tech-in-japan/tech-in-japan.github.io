ワークショップ：アジャイルプロジェクトの作成。 役割、慣行、ツール

2011年3月3日CareerlabとScrumTrekは、マイクロソフトのサポートを受けて、Visual Studioのアジャイル手法を使用したプロジェクトの開発に焦点を当てたセミナーを開催します。




現時点では、Microsoft Visual Studio 2010やMicrosoft TFS 2010などのツールの有効性に関する多くの情報が提示されています。これらをScrumおよびXPプラクティスと併用すると、厳しい競争環境で高品質のソフトウェアを開発できます。 原則として、プロジェクトチームのリーダーにとって特に重要なのは、プロジェクトの反復の特定のフェーズに関連してロールプレイングツールの使用を明確に示す「全体像」を自分で固定する問題です。



俳優と同様に、Dmitry Nikonov、Denis Pasechnik、Anton Vadishchevが役割を果たし、プロジェクトで発生するさまざまな動作や状況をシミュレートし、プロジェクトライフサイクルのさまざまな段階でVisual Studio 2010の機能を実証します。

  • Dmitry Nikonovは、ワシントン大学で教鞭を執るソフトウェア開発ツールおよび方法論の専門家であり、現在、米国マイクロソフトのプリンシパルプログラムマネージャーの地位にあります。 数年間、彼はAmazon.comの開発責任者でした。
  • Denis Pasechnikは、Microsoft Ukraineのソフトウェア開発のスペシャリストであり、ソフトウェア開発の分野で10年以上の経験があります。
  • Anton Vidishchevは、チームのテクニカルリーダーであり、Visual Studio 2010開発環境の一部としてアジャイル製品方法論を使用した謝罪者であるSigma Ukraineのシニア開発者です。


このセミナーは、新しい知識と経験を獲得する機会です。 Visual Studioを使用している、またはVisual Studioを使用する予定のユーザー、およびアジャイルに関する知識を使用している、または拡張したいすべての人にとって興味深いものです。
