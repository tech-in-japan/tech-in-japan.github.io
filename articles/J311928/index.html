コピーが簡単-すばやく復元

バックアップとデータ復旧の分野のソリューションは、大企業と中小企業の代表者の両方に必要です。 ただし、このようなソリューションの使用方法の選択は、組織の規模と直面する課題によって異なります。



Hewlett Packard Enterpriseの主力バックアップソリューションはHPE Data Protectorです。 この製品には、ほぼ30年の堅実な歴史があります。 1987年にOmnipol Network Ba​​ckup Systemという名前でApolloによって開発されました。1989年にApolloがHewlett-Packardに入社した後、長い間開発され、2002年にHP Data Protectorという新しい名前が付けられました。 製品の歴史は古くても、HPと現在のHewlett Packard Enterpriseは、ソフトウェアの世界とIT業界全体で起こっているすべての技術的変化を反映して、常に改善に取り組んでいます。







ここでは、Data Protectorの主な機能について簡単に説明します。ここでは、最近開発者によって実装された最も重要で有用な革新に焦点を当てます。



インフラストラクチャバックアップユニット（セル）は、いくつかのコンポーネント（管理サーバーとクライアントパーツ）で構成されています。 制御サーバーの部分は、Microsoft Windows、Linux、およびHPUX（HPEのバージョンではUnix）の任意のオペレーティングシステムに配置できます。 制御部分には、バックアッププロセス全体に関する情報を含むデータベースがあります。実行された機器、予約された日時、など。HPEData Protector制御サーバーをクラスター化して、耐障害性を確保し、さらに集中サポート地理的に分散した構造を持つ組織にとって特に有用で関連性の高い管理。

クライアント部分は、データをMedia Agentに転送するDisk AgentおよびApplication Agent統合エージェントによって表されます。MediaAgentは、データを標準のディスクアレイ、ネットワークデバイス、物理および仮想テープライブラリ、および重複排除をサポートする専用ストレージシステムにバックアップします。 Data Protectorは、HPE StoreOnce独自のストレージシステムと多くのサードパーティハードウェアソリューションの両方をサポートしています。 また、単純なデータ配置ではなく、統合により、たとえば、分散型重複排除、バックアップユニットの集中管理などが可能になります。



クラウドリソースは、パブリッククラウドだけでなく、お客様が独自のプライベートクラウドを構築できるOpenStackオープンプラットフォームに基づく特殊なソリューションであるHPE Helionのバックアップのリソースとしても使用できます。



HPE Data Protectorは、シンプルで便利なグラフィカルインターフェイスとコマンドラインを通じて管理され、現在のタスクとユーザー設定に応じて、ある方法から別の方法に切り替えることができます。



それとは別に、近年Data Protectorに実装されたいくつかの重要な機能について説明する価値があります。 そのため、2013年にリリースされた製品の8番目のバージョンでは、PostgreSQL DBMSに基づく完全に更新された内部データベースが登場しました。 強力な産業用リレーショナルDBMSの使用のおかげで、HPE Data Protectorの作業が大幅に加速され、さらに、サポートされるデバイスの数、同時セッションなどにより、ソリューションの限界機能が何度も向上しました。 1つのバックアップセルで十分である可能性があり、複数の分散セルは、部門または部門が共通によってまったく接続されていない分散組織に必要です。 緋色または不安定な接続を楽しんで。



HPE Data Protectorの重要な機能の1つは、バックアップが保存されるHPE Store Onceストレージとの統合です。 この統合により、ITインフラストラクチャのさまざまな部分でのデータ重複排除が可能になります。 まず、作成された場所、つまりクライアント側。 もちろん、これによりクライアント機器の負荷が増加するため、専用サーバーで重複排除が最初に実行され、その後重複排除されたデータがストレージに転送されるときにオプションが提供されます。 最後に、重複排除はHPE Store Onceで直接行うこともできます。 3つのオプションはすべて、1つのバックアップインフラストラクチャのフレームワーク内で使用でき、必要に応じて、Data Protectorを使用して1つから別のものに切り替えます。



アプリケーションのバックアップに関しては、HPE Data ProtectorはMicrosoft、VMware、SAP、Oracle、IBM Lotus、Sybaseなどのほとんどのソフトウェアソリューションおよびデータベースと統合します。統合はアプリケーションエージェントを使用して実行されます。 これらのシステムのバックアップの微調整はすべてData Protectorで直接実行されるため、システム管理者およびバックアップを担当するIT専門家の作業は大幅に簡素化されます。各アプリケーションでバックアップオプションを設定する時間を費やす必要はありません。



仮想環境を保護するために、HPE Data Protectorは最も一般的なハイパーバイザーであるVMwareおよびMicrosoft Hyper-Vと統合しますが、他の仮想化ソリューションの基本的なコピー機能もあります。 同時に、LANネットワークとSANネットワークの両方に基づいて、コピーとリカバリの両方を実行できます。 さらに、数か月前、バックアップ仮想環境用の別の製品であるHPE VM ExplorerがHewlett-Packard Enterprise製品ポートフォリオに登場しました。 この手頃な価格のシンプルなソリューションは、中小企業からの需要が高いかもしれません。



ただし、データを迅速に復元する必要があるため、コピーは戦いの半分に過ぎません。 HPE Data Protectorをクラウドを含む多くのデバイスおよび環境と統合することにより、さまざまなレベルのリカバリを組み合わせて提供できます。 ここでは、乗客と飛行機の持ち物の配置との類似性が適切です。 ビジネスクラスがあり、最初のクラスが一番高く、次にエコノミークラス、最後に荷物室があります。 私たちの場合、データの重要性が高いほど、データをより速く復元できますが、リカバリプロセスには適切な価格があります。



最速かつ最も高価なレベルは、ディスクアレイ上のハードウェアスナップショット（システムスナップショット）からのインスタントリカバリです。 このようなハードウェアスナップショットは、バックアップと同時にディスク上に作成され、そこに保存されます。 HPE Data Protectorの機能を使用すると、ディスクアレイ上のスナップショットのローテーションを整理し、一度に複数の復旧ポイントを取得できます。 これは、重要なデータベースを数秒でバックアップから「上げる」必要がある場合に便利です。通常の方法で何も復元する必要はなく、ディスクアレイ上のボリュームを切り替えるだけです。

仮想マシンは、バックアップから直接起動し、バックグラウンドでメインディスクに転送できるため、従来の方法で復元する必要もありません。



多くのシステムでは、いわゆる詳細な復元が提供されており、システム全体ではなく特定の要素を復元できます。 特に、VMware仮想マシンでは、個々のディレクトリとファイルを復元できます。MicrosoftExchangeとSharePointでは、個別のオブジェクト（レター、ファイルなど）を復元できます。



多くのお客様は、OSやその他のソフトウェアがプリインストールされていない、完全にクリーンなサーバーに対してベアメタル回復機能を使用することを望みます。 これを行うには、サーバーのブート可能なISOイメージを事前に準備し、重大なハードウェアの誤動作の場合の迅速な回復に必要な重要なソフトウェアをそのサーバーに配置する必要があります。



バージョン7以降、HPE Data Protectorは、HPE Operations ManagerやMicrosoft System Center Operations ManagerなどのITインフラストラクチャ監視ツールとの統合により、バックアップ監視をサポートしています。 次に、バックアップ分析を担当する新しいHPE Backup Navigatorモジュールがありました。 たとえば、バックアップデバイスがいっぱいになったとき、ディスクアレイのクォータが終わったときなどを通知します。このツールは、70を超えるさまざまなタイプのレポートを生成できます。ハードウェアインフラストラクチャのバックアップ。 HPE Backup Navigatorは独自のデータベースで動作するため、HPE Data Protectorに追加の負荷がかかることはありません。



便利な追加機能は、新しい組み込みタスクスケジューラです。これは、確立された優先度に従ってバックアップセッションを並べ替えます。 したがって、より重要なセッションが、解放されたデバイスへの最初のアクセスになります。 さらに、スケジューラは失敗したバックアップセッションのリストを表示し、それらを自動的に開始します。



結論として、ライセンスポリシーに関するいくつかの言葉。 HPE Data Protectorには、コンポーネント別および保護データのボリューム別の2つのライセンススキームがあります。 コンポーネントライセンススキームでは、アプリケーションサーバー、使用済みテープドライブ、ディスクデバイス、およびその他のオプションとの必要な統合の数を考慮して、特定のインフラストラクチャのライセンスのリストを個別に選択できます。 保護されたデータのボリュームのライセンススキームにより、保護されたデータの総ボリュームに従って計算することで、HPE Data Protectorライセンスの選択を簡素化できます。 さまざまなインフラストラクチャでは、1つまたは別のライセンススキームの方が収益性が高い場合があります。したがって、両方のスキームの計算を要求することをお勧めします。
