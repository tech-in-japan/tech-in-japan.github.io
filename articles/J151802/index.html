[開発ストーリー]パズル：iPad用のマーシャとクマのゲーム

最初のゲーム「パズル：マーシャとくま」を作成した方法に関する記事「平易な言葉で」または「指で」 。 プロジェクトマネージャー、プログラマー、デザイナー、広告マネージャーの物語。 実際にこの順序で、そして放送します。

誰も気にしない-猫の下でウェルカム。





挑戦は簡単なゲームです。 （マキシムコチュリン）



子供向けの適切なパズルを作成するというアイデアは、それ自体で何らかの形で生まれました。 まあ、それは自発的に簡単になるということではありません-しかし、同じ名前のアプリケーションですべてのシリーズを販売する以外に、人気ブランド「Masha and the Bear」で何かをするというアイデアが空中に急上昇しました。 最も簡単に実装できるゲームの仕組みとして、私の選択はパズルにかかっていました（そう思いました）。



ステップ1.市場調査


市場で（AppStoreで読む）-この方向に完全な闇。 私が見つけたもの-神は禁じています。 もちろん、私は興味深い標本に出くわしました-国内と西洋の両方ですが、あなたは何を取って「ワオ！」と言いますか？いいえ 。 いくつかの「機能」-はい、しかし全体的な印象はそれほどではありません。 アイデアが強化され、正当化が続くように見えました。



ステップ2.遊びましょう!!!


私はいくつかの「物理的な」パズル（「Okay」の実際のパズル）を購入し、「会議」のために会議室でチーム（プログラマーとデザイナー）を組み立てました。 それは壮大でした。 ））サポートからの従業員の顔は、「ここで何をしているのですか？」という質問の後、特に喜んでいます。そして、「私たちは働いています」...

一般的に、私たちは十分にプレイし、パズルの仕組みを直接理解していたので、iPadでパズルをクールにすることは非常に現実的であると判断し、TKを書き始め、図を描き始めました...

iOSの最初のプログラミングエクスペリエンス-“ code Masha”（Sergey Kraevsky）



よく知られた漸進的思考の人類漫画「Masha and the Bear」に基づいたパズルは、iOS向けのアプリケーションを書く最初の経験になりました。 もちろん、Makos、Xcode、およびiOS開発者になるためのその他のステップで作業することの忘れられない第一印象についてペイントでペイントできます。 しかし、重要な点に焦点を当てる方が良いです。



最初に対処しなければならなかったのは（IDEのひどいホットキー以外）、 Objective-Cの構文でした。 それには欠点がありますが、慣れることができ、かなり快適にアプリケーションを作成することができます。 C / C ++でのプログラミングの経験がある場合は、言語学習のほぼ最初の日に書くことができます。 標準ライブラリは非常に優れており、基本的な必要なもののほとんどが含まれています。さらに、CおよびC ++で記述されたコードを使用できます。たとえば、物理Chipmunkエンジンを使用し、アプリケーションの場所を推測できます;）。 私の意見では、主な欠点は言語の冗長性です（1行あたり100文字でも十分ではない場合があります）。 プリミティブ型を保存できないコンテナ。 オペレーターの過負荷の欠如。 それにもかかわらず、私は繰り返します、それについて書くことは全く可能です。



Appleフレームワークの中で、私はCoreDataが好きでした -データセットを扱うために、データモデルの変更を追跡することは非常に便利です。 これに加えて、Key-Value Observingは相互作用を促進し、プログラムの部分間の接続の数を減らします。



言語は言語ですが、アプリケーションを作成する必要があります。 ドキュメントとスタックオーバーフローを読むことで解決される、新しいプラットフォームへの移行における標準的な困難に加えて、いくつかの意外な問題がありました。



最初の最も顕著なのは、メインアプリケーション画面のページング速度が遅いことです。 1番目と2番目のiPadの両方の速度はそれほど印象的ではなく、3番目のiPadでは一般に批判を下回っていました。 問題は、ユーザーインターフェイスを形成する画像を含む多数の画像のデコードにありました。 その結果、UINibキャッシング、画像キャッシュ、およびそれらの非同期デコードの助けを借りて、速度が許容可能な速度に向上しました。



突然、 VK APIの標準的な実装はなく（ええ、私は怠け者です）、このトピックに関する記事は数件しかありませんでした。 私は自分で小さなライブラリを作成しなければなりませんでしたが、幸いなことに、特別な困難に遭遇することはありませんでした。

習慣から外れて、彼は長い間水平にスクロールするテーブルで誓いました。そこでは細胞はπ/ 2だけ優雅に回転しなければなりませんでした。



パズルセルの内部的な強調表示のために、私はCoreImageを使用したかったのですが、 結局のところ 、彼の仕事のスピードは、うーん...まあ、不十分です。 短い検索の結果に基づいて、画像を操作するための豪華なライブラリがありました。GPUImage（https://github.com/BradLarson/GPUImage）。これには膨大な数の処理機能があり、非常に高速です。



それとは別に、古いバージョンのiOSおよび最初のiPadのサポートについても言及する必要があります。 ここで、私にとっては、いくつかの落とし穴もありました。特に、 UIViewのブロッキングアニメーションです。 またはUIViewControllerのviewWillAppearメソッドが呼び出されていないこと 。 4番目のiOSでは、賞賛したGPUImageライブラリも無効にする必要がありました。そのメソッドを操作すると、アプリケーションがクラッシュしました。 そして最初のiPadでは、メモリ不足についての警告を伴う状況をテストするのが非常に便利であることが判明しました=）



マーシャをどのように「カット」するか...（Natalia Markina）



「パズル：マーシャと熊」はどのように始まったのですか？ マイケルジャクソンについては、組み込みのミニゲーム、パズル、そして人気の高いアプリケーションである Masha and the Bearを使ったアプリケーションが ありました。 そして予期せぬことに、プロジェクトマネージャーはパズルとマーシャを組み合わせて、便利な子供向けアプリケーションを作成するというアイデアを思いつきました。



そして、それは始まりました：App Storeからすべての可能なアプリケーションを汲み上げ、遊び、評価し、それらの長所と短所を書き留め、理想的なアプリケーションを作成する方法を考えました。 私たちは本物のパズルセットを購入し、会議室で数時間過ごし、一見シンプルなゲームを勉強しました 。



その結果、アプリケーションをできる限り現実に近づけることが合意されました。 ボリュームの詳細を作成し、パズルセットを実際の箱のように「デザイン」し、棚に配置します。 このような多様なコンテンツの場合、このオプションは最も便利なように見えました（おそらくある程度、これはすべてが「棚に」あるべきだと子供たちに教えるでしょう）。 ゲームモードでは、すべての詳細がボックスにきちんと配置されます。そのようなパズルでは、詳細は失われません（よりリアルに、このようなオプションを作成したかったのです）。



一般的に、子供向けの何かを思いついたとき、子供の注意を引き付けて保持する方法を考えます。 「子供たちを長く待たせることはできません」「一部の子供たちはまだ読むことができません」とプロジェクトマネージャーは教えてくれました。 したがって、 ほとんどのボタンにはグラフィカルな表現があります。 絶対に子供たちがパズルをプレイできるように、必要な詳細をどこに置くかを指示する子供用のボタンを考案しましたが、今では大人もそれを拒否したくありません。



それで、私たちは各ボタンの上で子供たちに愛と注意をもって考えました。

技術的な面からは、パーツのテンプレートの「切り取り」とその配置だけで困難が生じました。 ここで私はプログラマーと緊密に協力する必要がありました。パーツが適切に「ドッキング」、「カット」、ピクセル単位で「引っ張られる」ことを確認しました。プログラマーの忍耐がなければ、パズルがどうなるかはわかりません。



このプロジェクトの間に、私は漫画を完全に修正し、スクリーンショットを選択し、 毎日マーシャを見て、インターフェースを描きました。もちろん、彼女は私にうんざりしていましたが、子供がそれに関連するものすべてにどれほど熱心に反応するかを知っていれば、あなたは許容できます



「マーシャ」を宇宙に打ち上げる（スヴェトラーナコチュリーナ）



これが私たちの最初のゲームであるため、私たちは非常に責任を持ってゲームのカバレッジにアプローチしました。 開発者がゲームをどのように宣伝したか、何が「入った」か、「入っていない」かについて、開発者のさまざまな話を知りました。 さらに、iOSアプリケーションの宣伝にはすでに良い経験がありました。 一般的に、ゲーム「パズル：マーシャと熊」の立ち上げを計画したキャンペーンについて説明します。 おもしろい場合は、後でそれが何で、何がそうでないかを書きます。
