Unityでボーナスシステムを作成する







ソニック・ザ・ヘッジホッグのゲームは、金の指輪とスピードブーツなし、マッシュルームのないスーパーマリオ、点の点滅なしのパックマンのように見えますか？ これらすべてのゲームはもっと退屈です！



ボーナス（パワーアップ）は、ゲームプレイの最も重要なコンポーネントです。これは、新しいレベルの複雑さと戦略を追加し、プレイヤーに行動を促すためです。



このチュートリアルでは、次のことを学びます。



  • 他のゲームで繰り返し使用できる可能性のあるボーナスシステムを設計および構築します。
  • ゲームでメッセージベースの通信システムを使用します。
  • 独自のボーナスを使用して、トップビューのゲームでこれをすべて実現してください！


注：このチュートリアルでは、Unityに精通しており、少なくともC＃の中間知識があることを前提としています。 知識を磨く必要がある場合は、他のUnityチュートリアルをご覧ください 。


チュートリアルの手順を繰り返すには、Unity 2017.1以降のバージョンが必要になるため、まだUnityのバージョンをアップグレードしていない場合はアップグレードしてください。

ボーナスライフサイクル



ボーナスには、いくつかの個別の状態で構成されるライフサイクルがあります。









  • 最初の段階は作成です 。これはゲームプロセスまたは開発段階で実行され、シーンにGameObjectボーナスを手動で配置します。
  • 次に、 注意を引くモードがあります。これは 、ボーナスがアニメーション化されるか、プレイヤーの注意を引くことができる場合です。
  • 収集段階はボーナスを取得するアクションであり、これによりサウンド、パーティクルシステム、またはその他の特殊効果がトリガーされます。
  • ボーナスの選択は、ボーナスが「仕事をする」 ペイロードの履行につながります。 ペイロードは、わずかな健康増進から、プレイヤーに驚くべき超大国への寄付まで何でもかまいません。 ペイロードフェーズは妥当性チェックも引き起こします。 ボーナスは、特定の時間の後、プレーヤーが敵に触れた後、数回使用した後、またはゲームプロセスの他の条件が満たされた後に行動を停止するように構成できます。
  • 検証は完了段階につながります。 完了ステージはボーナスを破壊し、サイクルの終わりになります。


上記のライフサイクルには、任意のゲームで使用する価値のある要素と、特定のゲームにのみ適用される要素が含まれています。 たとえば、プレーヤーのボーナスの選択を確認することは各ゲームで使用する必要がありますが、プレーヤーを見えなくする特定のペイロードはこのゲームでのみ使用する価値があります。 スクリプトロジックデザインを作成するときは、これを考慮することが重要です。

シーンスピードブーストボーナスの作成



GameObject SpeedPowerUpは、インスペクターで次のようになります。









インスペクターに次の値を入力します。



  • パワーアップ名 ：スピード
  • 説明 ：敵と接触するまでの超高速移動
  • パワーアップ見積り ：（12パーセック未満でケッセルを実行します）
  • すぐに期限切れ ：チェックを外します
  • 特殊効果 ：パーティクルの収集（スターと同じ）
  • 効果音 power_up_collect_01（スターと同じ）
  • スピードマルチプライヤー ：2


その後、ボーナスは次のようになります。









必要に応じてSpeedボーナスオプションを構成したら、それをPrefabs / Power Upsプロジェクトツリーフォルダーにドラッグして、プレハブを作成します。 デモプロジェクトでは使用しませんが、完全を期すために実行する価値があります。



シーンを起動し、ヒーローを移動させて速度ボーナスを収集します。その後、彼は追加の速度を受け取り、敵と接触するまで残ります。











反発ボーナス









次のボーナスでは、プレーヤーはPキーを押してオブジェクトを邪魔にならないようにプッシュできます。 そのアプリケーションの数は限られています。 ボーナスを作成する段階についてはすでにご存じです。したがって、テキストを乱雑にしないために、コードの最も興味深い部分のみを検討してから、プレハブを作成します。 このボーナスは、メッセージを聞く必要はありません。



プロジェクト階層でPowerUpPush プレハブを見つけて探索します。 PowerUpPushというスクリプトを開き、コードを調べます。









すでに調査したおなじみのメソッドが表示されます。 興味深い反発ボーナスアクションはすべて、 Updateメソッドで発生します 。 



  private void Update () { if (powerUpState == PowerUpState.IsCollected && //1 numberOfUsesRemaining > 0) //2 { if (Input.GetKeyDown ("p")) //3 { PushSpecialEffects (); //4 PushPhysics (); //5 numberOfUsesRemaining--; if (numberOfUsesRemaining <= 0) { PowerUpHasExpired (); //7 } } } }


このコードでは次のことが行われます。



  1. 収集されたボーナスに対してのみスクリプトを実行する必要があります。
  2. スクリプトは、使用回数が残っている場合にのみ実行する必要があります。
  3. スクリプトは、プレーヤーがPをクリックしたときにのみ実行される必要があります。
  4. スクリプトは、プレーヤーの周りに美しいパーティクル効果を起動します。
  5. スクリプトは、プレイヤーから敵を撃退します。
  6. ボーナスが再び使用されます。
  7. 使用回数が期限切れになると、ボーナスが期限切れになります。


反発ボーナスの使用は非常に興味深いので、ステージ上の適切な場所にカップルを置く価値があります。 シーンをもう一度実行し、新しいボーナスでプレイします。









追加クエスト：不死身ボーナス









このセクションはオプションですが、非常に興味深いものです。 獲得した知識で武装し、プレイヤーをしばらく不死身にするボーナスを作成します。



このタスクを完了するためのヒントとコツ：



  1. スプライト ：プロジェクトフォルダTextures / shieldでスプライトを見つけます
  2. パワーアップ名 ：不死身
  3. パワーアップ説明 ：あなたはしばらくの間無敵です
  4. パワーアップ引用 ：（素晴らしい子供、生意気にならないでください）
  5. コーディング ：スターボーナスとスピードブーストに使用したものと同じチェックリストを確認します。 ボーナスの期間を制御するタイマーが必要になります。 PlayerBrainのSetInvulnerabilityメソッドは、 脆弱性を有効または無効にします。
  6. エフェクト ：プロジェクトには、プレーヤーが不死身のままであるときに、プレーヤーの周りの美しい波紋エフェクトのためのパーティクルエフェクトが含まれています。 PrefabはPrefabs / Power Ups / ParticlesInvulnにあります。 攻撃されない間、プレイヤーの子インスタンスにすることができます。

Box Collider 2Dを追加し、 SizeをX = 0.2、Y = 0.2に変更します。 スプライトレンダラーコンポーネントで、他のボーナスと同様に、 スプライトをシールドに設定し、色を選択します。 [ トリガー]ボックスがオンになっていることを確認します。 その後、GameObjectは次のようになります。
