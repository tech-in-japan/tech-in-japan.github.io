クラウドプロバイダーと呼べるのは誰ですか、またはホスティング業者のトリックに負けない方法は？





はじめに



ITアウトソーシングの市場では、プロバイダーの外部データセンターに物理的な機器を配置すること、その技術サポートと管理から、コンピューティングリソース（サーバー、ディスク、ネットワーク）のリースまで、多くのサービスがあります。



多くの人が誤ってクラウドコンピューティングレンタルコンピューティングリソースを呼び出します。 これは完全に真実ではありません。 コンピューティングリソースのリース市場は非常に細分化されており、すべてのプロバイダーがクラウドと呼ばれるわけではありません。 さらに、自分自身をクラウドと呼ぶプロバイダーのほとんどは、一生ウェブサイトをホストしている普通のホスティング業者です。



誰が誰なのか見てみましょう！



クラウドプロバイダーとホスティング業者



仮想化



クラウドプロバイダーとホスティング業者の両方がサーバー仮想化テクノロジーを使用しています。 これらは非常に古い技術であり、それらの多くはすでに10年以上前のものです。



幸いなことに、ホスティング事業者はまだクラウド専用の物理専用サーバーを呼び出していません。 これには驚かないでしょうが）



これから、重要な違いをリストし始めます。 以前に、クラウドプロバイダーとホスティング業者の両方がサーバー仮想化テクノロジーを使用していることがわかりました。 しかし、実際には、他の仮想化テクノロジーがあります。



1つ目は、ストレージの仮想化です。 その使用により、多数の物理ストレージシステムを単一の論理プールに結合できます。 つまり、ストレージシステムの動物園は、たとえば300 TBのディスク容量として外部から見える単一のユニットに結合できます。 このアプローチでは、1つの仮想サーバーのディスクフラグメントをプロバイダーの異なるストレージシステムに保存できます。



第二に、ネットワーク仮想化です。 ネットワーク仮想化テクノロジーを使用すると、ネットワーク機器の標準機能の使用、つまりVLANの使用を回避できます。 結局のところ、物理機器のVLANの最大可能数は4096個に制限されています。 また、主要なクラウドプロバイダーになりたい場合は、必ずこの制限を超えてください。 次に、ネットワーク仮想化ソフトウェアを使用すると、無制限の数の隔離されたネットワークを実行でき、クラウドをほぼ無限に拡張できます。



さらに、ネットワーク仮想化ソフトウェアを使用すると、物理ネットワーク機器とそのメーカーの構成をそれほど重視することができなくなります。 鉄から抽象化することができます。これにより、クラウドプロバイダーはより柔軟になり、特定のベンダーの技術に縛られなくなります。



ストレージ仮想化ソフトウェアとネットワークを使用することは、クラウドプロバイダーにとって必須の要件ではありませんが、その成熟度とテクノロジーリーダーになることを願っています。



特に、 クラウドプラットフォーム （CROC Virtual Data Center） 内のCROCは、 3種類すべての仮想化を使用しました。

  • サーバー仮想化-Red Hat KVM
  • ストレージ仮想化-IBM GPFS Cluster Parallel File System
  • ネットワーク仮想化は、Openflowプロトコルに基づく商用製品です。 競合他社との技術的分離を強化するため、製品の名前はまだ公開されていません




サービス管理

クラウドプロバイダーとホスティング業者を比較する際の2番目の要因は、サービスの管理方法です。

ホスティング業者はどのように機能しますか？ ほとんどの場合、スライダーとスイッチが豊富なカラフルなサイトがあります。 サイトでは、ディスクを搭載したサーバーのコストを計算し、OSをレンタルするコストを調べ、IPアドレスを追加し、通信チャネルの要件を指定して、「購入」ボタンをクリックします。 このサイトは、専用の仮想サーバーを販売するための「美しいラッパー」です。 購入後、クライアントは、OS、ディスク容量、ネットワーク設定が事前にインストールされたサーバーを受け取ります。 マシンの電力、ディスク容量、ネットワーク設定の変更は、プロバイダーへのリクエストに応じてのみ行われます。

同時に、クラウドプロバイダーは、完全なセルフサービス「セルフサービス」の原則に基づいて作業します。 これは、理想的な場合、セルフサービスポータルへのクライアント用のアカウントが作成されることを意味します。

  • 仮想サーバーの電源をオン、オフ、変更する
  • ディスクリソースを追加する
  • ディスクをSATAからACなどに移行して、ディスクサブシステムのパフォーマンスを変更する
  • 外部の静的IPアドレスをサーバーに割り当てます
  • 内部のアドレス指定とルーティングを管理する
  • バックアップを管理する
  • マシンテンプレートの管理
  • 既にインストールされているローカルの物理ネットワーク機器などの使用を含め、サイトへのIPsec VPNトンネルを構成します。
  • 等


セルフサービスの原則により、サービスの柔軟性が大幅に向上し、テクニカルサポートサービスプロバイダーから独立することができます。 つまり、便利です。



CROC Virtual Data Centerのセルフサービスポータルでは、上記のすべてのサービスを個別に管理できます。 初期の段階でポータルにアクセスした新しい顧客は、サービスの使用に関する明確な質問をし、原則として、CROC Virtual Data Centerの広大な地域を完全に独立して航海します。



経験から言えば、サービスを1か月使用した後、顧客はテクニカルサポートサービスへの電話を実質的に止め、実行中のサーバーとドライブの数が増えていると言えます。



課金



クラウドプロバイダーとホスティング業者を比較する場合の3番目の要因は、サービスの課金方法（課金）です。

ホスティング事業者は通常、月単位でサービスを提供します。 1か月以内にサービスの構成が調整された場合、そのアカウンティングの粒度は丸1日に減ります。 会計サービスは手動モードで行われます。



クラウドプロバイダーは、会計においてより柔軟です。 アカウンティングの粒度は秒から時間です。 会計は自動化されており、人間の参加は必要ありません。 自動請求は、セルフサービスポータルを通じてユーザーが行ったすべての変更を追跡します。

CROCの仮想データセンターでは、1秒あたりの請求が使用されます。 さらに、各トランザクションまでの内訳を含む期間の詳細レポートをアップロードすることができます。 同様のサービスがモバイルオペレーターによって提供されますが、追加料金がかかります。



そこで、クラウドプロバイダーとホスティング業者が使用するアプローチを比較しました。 これらの主な違いは、完全に機能するセルフサービスおよび課金ポータルの可用性であることがわかりました。 両方の場合にサーバー仮想化の使用が存在しますが、すべてのクラウドプロバイダーがストレージシステムとネットワークの仮想化を持っているわけではありません。



次の投稿では、クラウドプロバイダーを選択する際の難しい問題を理解します。 結局のところ、クラウドサービスの市場は非常に「雑多」です。 多くの企業の積極的なマーケティングによって状況は複雑になっていますが、誰かが市場で最も大声で叫ぶと、彼の製品が最高であるという事実とはほど遠いことを理解できます。 それはしばしば反対です）。
