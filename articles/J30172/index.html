EuroPython 2008レポート

EuroPythonは、Pythonプログラミング言語に関する年次ヨーロッパ会議です。



この年と最後の年に、会議はリトアニアの首都ビリニュスで開催されました。 来年はおそらくバーミンガムが開催します（伝統により、会議は2年ごとに移動します）。





ビリニュス



私がロシアからの唯一のスピーカーだったことは奇妙です。 これは奇妙です。なぜなら、ビリニュスはモスクワから電車で15時間離れており、ビザの取得は非常に簡単だからです（招待がある場合）。



ビリニュス自体は、観光客にとって美しくフレンドリーな都市です。 特に興味深いのは、旧市街と呼ばれる地域です。 ユネスコの世界遺産に登録されています。 来年、ビリニュスはヨーロッパの文化の中心地（オーストリアのリンツとともに）になります。



ビリニュスにいるなら、自転車を借りることをお勧めします。 ほぼすべての場所に自転車用の小道と自転車用の「駐車場」があり、さらに自転車で町の外に出ることもできます。



コンファレンス



EuroPythonはPythonコミュニティによってホストされています。 この会議の目的は、レポートを整理するだけでなく、開発者、会社のマネージャー、研究者に非公式なコミュニケーションの機会を提供することです。



会議自体は3日間続きました 。レポートのリストはこちらでご覧いただけます 。 個人的に、私は（私の興味の輪を考慮して:)）次の報告を思い出しました：

  • Marcin Kaszynskiによる1週間でアプリを作成します。 講演者は、たとえばhttp://webcomicspot.comのように、Djangoで本格的なアプリケーションを1週間で作成する方法について詳しく説明しました。
  • LAX-Nicolas ChauvatによるLogilab AppEngine eXtension。 Logilabによって作成された拡張機能でGoogle App Engineを使用することについてでした 。
  • Harald Armin MassaによるPythonの使用を推奨しない、なぜ会社でPythonを使用してはいけないかについての悪いアドバイスの精神でのFunkyプレゼンテーション（もちろん、著者は反対を証明しました:)）。
  • なぜ怠zyなのが良いのか：Pythonによる関数型プログラミング（ Adam Byrtek著）。 関数型プログラミングに使用できるPythonコンストラクトに関するレポート。 確かに、このレポートは実用的なレポートよりも理論的な観点から興味深いものです。
  • ハンス・ロスリングは、統計を公開する必要がある理由について話しました。 この報告をもう一度語るのは無意味です、とても良いことです。 Google Imagesでの同様のプレゼンテーションを次に示します。レポートの論文を明確に説明するリソースを次に示します。
  • 最後に、 Guido van Rossumとの電話会議で、 Python 3000に登場する新機能について話しました。

会議後、スプリントのコーディングは3日間続きました。 スプリントは、オープンソースの製品開発者が集まってプログラミングを行うためのユニークな機会です。 次のスプリントがEuroPython 2008にありました： Django 、 PyPy 、 Bazaar 、 Grok 、 cocos2d 、 MoinMoin 。 ところで、たとえばDjango、PyPyなどの一部のプロジェクトは、主にスプリントのために開発されています。



私自身、 Clone Digger （自動重複コード検出システム）について話しました。 スライドはこちらです。 とりわけ、私はプログラムのユーザーと個人的にコミュニケーションをとることができたため、会議に100％満足しました。
