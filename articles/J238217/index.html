新しいMicrosoft Azure仮想マシン-SSD、パフォーマンス、およびバッファープールの拡張機能

昨日、新しいMicrosoft Azure仮想マシンが発表されました 。これは、すべてのユーザーが既に利用できます。 新しい仮想マシンの本質は生産性の向上にあります-プロセッサのパフォーマンスを向上させるために、ユーザーから多くの要求がありました。 Dシリーズの仮想マシンは、高速vCPU（A仮想マシンと比較して速度が約60％向上）と大量のメモリ（最大112 GB）を使用します。 また、新しいマシンにはローカルSSDディスク （最大800 GB）があるため、ファイルサブシステムへのリクエストが増加する可能性が高くなります。



そして、ワルシャワのMichal Smereczynskiは、重要な事柄を後まで遅らせることなく、すでに約11分で小さなカーネルをコンパイルしました。







新しい機能の使用方法をよりよく理解するために、少しの履歴-各VMは2つのディスクで作成されます-C（システム、Microsoft AzureストレージにVHDとして保存）およびD（ローカルハードウェアに接続された一時）。 ディスクサブシステム（= SSD）のパフォーマンスを改善することにより、ディスクD上のデータの配置がより正当化されました。 これはLinuxマシンにも当てはまり、Linuxのみが/ mntまたは/ mnt / resourceの形式で取得します。



良いボーナスとして、DドライブにSSDを使用すると、 Buffer Pool Extensions （BPE）の機能を使用できるようになるため、MongoDBやSQL Server 2014のようなものを使用したプロジェクトの使用効率が向上します。データページのキャッシュ用（より高速な読み取り操作を提供）。 Buffer Serve ExtensionsはSQL Servr 2014で導入され、実際にはローカルSSDによるパフォーマンスを向上させるために必要でした。 データベースからの読み取りが発生する状況はまさにそのシナリオであり、これは大きな利益をもたらします。 T-SQLクエリを実行することにより、SQL Serverでこの機能をアクティブにできます。 



ALTER SERVER CONFIGURATION SET BUFFER POOL EXTENSION ON SIZE = <size> [ KB | MB | GB ] FILENAME = 'D:\SSDCACHE\EXAMPLE.BPE'




BPEの詳細については、 こちらをご覧ください 。 主な発表に対するコメントから判断すると、BPEは生産性を最大40％向上させることができ、これは非常に良いことです。



それでも、SSDにTempDBを配置することができ、それによりさらに高速になります。 これを行う方法はここで読むことができます 。

ここで 、Microsoftの単純なIT担当者がこのトピックについてどのように考えているかを確認できます 。



両方のポータル（ http://manage.windowsazure.comおよびhttp://portal.azure.com ）から、またコマンドレットとAPIを使用して、新しいタイプの仮想マシンを作成できます。

価格については、「 仮想マシンの価格の詳細 」ページを参照してください。
