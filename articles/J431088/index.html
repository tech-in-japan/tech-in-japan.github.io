間違ったファイル管理-パート1：元々90年代から

こんにちは、私はグレゴリーです。25年近くファイルマネージャーを使用していますが、それらについて何か言いたいことがあります。今日のファイル管理は不十分です。 最も一般的なプログラムとサービスの問題を示してみます。必要に応じて、このホワイトペーパーの記事をお読みください。







このスクリーンショットを見て、過去20年間で何が変わったのかを考えてください。













主な紹介文：







  1. 今日、 ファイル管理はデスクトップ、モバイル、およびWebで非公開です-個人的に公開され、共同で。 優れたファイルマネージャは、プラットフォーム間の境界を消去して、ファイルを同期し、共有と共有の準備を整えます。
  2. あらゆるタイプのコンテンツ、関連するメタデータ、および各プラットフォームに適したコンテンツ管理ツールの有益なプレビューを提供する必要があります。
  3. ファイル操作に関連するツールが必要です。そうすることで、あまり洗脳することなく簡単で複雑なことができます。


主流のファイルマネージャはこれを行うことができません 。一般的に、3つのポイントすべてを使い果たします。 その理由は次のとおりです。







  1. 設計者は自分が何を設計するかを気にせず、製品を使用しません。 彼らは自分の仕事をするだけです。
  2. 上司はデザインを気にしません。 あるべき？
  3. 通常のユーザーは「小さな」欠陥に気づきません。 彼らはそれを使うだけです。


人々が私に何をするかと尋ねると、私は「ファイル管理の問題」と答えます。ほとんどの人は私が言っていることを理解していません。

ファイルマネージャがどのように時間を食い尽くすかを見せたいと思います。 何かを見つけようとするとき。 プレビュー生成を待っています。 検索を待っています。 適切な写真を探しています。 2つのフォルダーをマージします。 フォルダーを共有しています。 リンク生成を待っています。 Googleドライブから、長続きしないリンクを紹介します。 ファイルマネージャー、画像ビューアー、クラウドストレージ、およびパブリッシングツールを使用して、1つの簡単なことを行います！







最初の部分は少し退屈かもしれませんが、どこかから始めなければなりません。 スクリーンショットがたくさんあります。 そして、多くの憎しみ 。 自己責任で続行してください。









データ提示



ファイルマネージャーの基礎は、ファイルとフォルダーのプレゼンテーションです-コンテンツのグラフィカルなプレゼンテーション、組み込みのプレビュー、アイコン、および関連するメタデータ-これは、すべてのプラットフォーム上のファイルの単純な視覚検索を提供する必要があります。







デスクトップとウェブ



最初から始めて、最も一般的なマネージャーであるWindows File Explorerを考えてみましょう。













待って、これは世界で一番のファイルマネージャーですか？ 90年代のExplorerのように見えます！

プレビューは非常に小さくなっていますが、周囲には多くのスペースがあります。 画像の縦横比が考慮されますが、プレビューのサイズのため、これはあまり役に立ちません。 メタデータなし。







写真家として、私は画像のアスペクト比を考慮しなければならないことを述べなければなりません。

次にGoogleドライブが登場します-ここにあるフォルダはなんとなく馬鹿げており、まったく情報がありません。 画像のアスペクト比は考慮されません。 ここでは、メタデータがなくても可能です。













Googleスプレッドシートを見てみましょう-ファイルマネージャーではありませんが、良い例です-プレビューとメタデータはここにあります。













もう1つの良い例であるGoogleフォトは、良いプレビューです。













Web上のYandex.Diskは、古いExplorerをコピーするだけです。 なぜそうしたのですか？ 彼らは自分で何も思い付かないと思う。













上記の例、それらの長所と短所を考慮して、私は独自のバージョンを設計しました。 PDFのアルファベット順とプレビューを失いましたが、その点を理解してください。













リストビュー



オーディオファイルのあるフォルダーでは、 エクスプローラーにはトラックのサイズも長さも表示されません-非常に便利です。















写真のあるフォルダについて、私はコメントを書きたくない、ありがとう、 Microsoftの人々。 最高のものが欲しかったのですが、実際のファイルでこれをテストするのを忘れたようです。 私のバージョンを参照してください。













モバイルアプリ



Googleドライブには2つのバージョンがあります-リストとタイル、どちらも見た目は良いですが、やはり奇妙なフォルダーがあり、写真のアスペクト比を考慮していません。 また、ズームはまだないため、256の中から適切な10個のファイルを見つけるのは困難です。













再びGoogleフォトルール、ズームとメタデータがあります。 ちなみに、古いSony Albumアプリもクールでした。













（想像上の）ズーム、リスト、タイルを使用して、ヤギの切り株からバージョンを再作成しました。写真を簡単に見つけることができます。 24枚の写真で正方形のグリッドを描きたくないので、想像してみてください。













シングルクライアント/仮想ディスク



Yandex.Diskは、ダウンロードオンデマンドクライアントであるため、必要なときにのみファイルをダウンロードできます。 しかし、なぜ個々のクライアントですか？ なぜ仮想フォルダーは動かないのですか？ 神は知っています。 私は他のスタンドアローンクライアントを覚えていません。 たぶん、これはクールな顧客ですか？ いや 彼はひどく見えるし、stormnoで動作します。







お父さんは、彼がノートンコマンダーのように見えると言いますが、着陸時の彼らのものとはまったく違います。

彼らは古いExplorerをコピーしました。 なぜ彼らは本当に良いことをしなかったのですか？ できないからです。













しかし、仮想ディスクのアイデアは良いと思います。最初に見たのは後期のZumoDriveでした 。 現在、Mail.ruのDisk-Oを実装しています （注意）。







アルファベットの問題



地獄 私はそれについて静かに保つことができません。 この例はたくさんありましたが、詳しく調べたくありません。 ほんの数例：







ファイルエクスプローラー、Windows 10-ロングネームファースト。













あなたのモバイル上のYandex.Diskはただの混乱です。 ウェブは同じです。













データをダウンロードしてプリロードする



ファイルエクスプローラーはHDDをアクティブに保ちません。ダウンタイム後にフォルダーに入ろうとすると、ディスクのスピンアップに数秒かかります。 アクティブにしておくか、キャッシュされたプレビューとメタデータを開いてみませんか？ HZ。







Explorerでプレビュー写真をダウンロードするには時間がかかり、フォルダーが非常にゆっくりと開きます。 FastStoneのような写真ビューアは、これをはるかに高速に実行し、起動時にインターフェイスをブロックしません。







検索する



Windows 10のファイルエクスプローラーで検索してもファイルが見つかりません。 何が彼を止めているのか分かりません。 彼らはそこにいますが、検索はノーと言います。 検索は非常に遅く 、数分かかります。 検索オプションは非常に隠されているため、誰も見つけることができません。 そして彼らは地獄に見える。













検索結果はあまり有益ではなく、見つかったファイルの1つの場所に移動しようとすると、通常はnavbarを使用できず、フォルダーの実際の場所が表示されず、検索結果に戻ることさえできません-再び開始されます！ ああ、なんてクレイジーだ。













デスクトップで適切な検索の例を示したいのですが、できません 。
