2. 人ではなく物事を管理する方法は？ 管理は伝統的に人事管理に焦点を当てています。 しかし、今も制御できるものがあります。 最初の近似では、さまざまな命令を明確かつ暗黙的に実行するタスクをロボットまたはアルゴリズムに与えることは素晴らしく便利なようです。 しかし、しばらくして、マネージャーは、同じロボットとアルゴリズムが、エラーを起こしやすい（多くの場合それらを含む）他の人によって確立された境界内に実装された単なるインターフェイスと機能であり、それらを修正または何らかの形で補償できないことに気付きます。 物事（ロボット、アルゴリズム）は、その行動と決定がかなり単純であり、特定のビジネスプロセス（機能）を個別に調整（適応）する方法を知らず、周囲の動的に発展する状況に対応します。



3. 顧客を見つけて彼を喜ばせる方法は？ これは、絶え間ない市場のダイナミクス、変化するムード、強い情報依存の条件において、完全にアイドルで関連性のある問題です。 市場セグメントに関連して詳細を増やす必要がありますが、競争は激化しています。 新製品の発売のタイミングが短縮されていると同時に、そのような「生の」製品に関する消費者の意見は、企業や個々のプロジェクトに対する一般的な否定的な態度を形成しています。 わずかな問題、欠陥、不便なインターフェース、クラッシュ、実行の時間遅延のために、開発の成功は失敗し、需要がないように思われます。 さらに、ネガティブな反応が即座にネットワーク上に広がり、その「殺人的な」感情によって競合他社を喜ばせます。



4. 情報などのリソースをどう処理し、どのように管理しますか？ 企業が貴重な情報を探す前に、今日は明らかに豊富にあります。 それはその品質とコストの問題です。 材料や原材料の購入、保管、生産への配送、完成品の受け取りなど、すべてが比較的簡単でした。 情報は管理を完全に混乱させ、最も複雑な数学的および統計的手法は、紛れもなく才能のある専門家によって作成された「曖昧な」数学モデルを「盲目的に」信頼しますが、多くの場合、ビジネスの主題、経済学、マーケティングにあまり精通していません。



5. リモートワークステーション、フリーランサー、独立した専門家、ベータテスターから「夢」のチームを作る方法は？ 専門家をビジネスに引き付けるさまざまな形態が現在利用可能です：恒久的および一時的、便利かつ効果的、遠隔的およびそうではありません。 真に効果的なチームを編成するには、いくぶん困難で費用がかかります。また、新しい原則に基づいてその活動を管理する必要があります。 専門家は自分たちが何をして、自分の仕事をどのように整理するかをよく理解し、サードパーティのコラボレーターを引き付けることができるため、「管理」の概念そのものが大きく変わります。 効果的な管理は、差別化されたソースから情報リソースを正しく見つけて処理し、市場と規制当局を監視し、さまざまな事業分野のタスクと機能を時間内に調整し、そしてもちろん、持続可能な戦略の実施を開発および監視する管理者の能力に依存するようになりました



6. 新しいリスクから身を守るには？ 技術的、自然、および経済的リスクに加えて、複雑で危険な情報とコミュニケーションが追加されました。 彼らは、多かれ少なかれ「接触が行われた」リスク管理の伝統的なリスクを繰り返し複雑化してきました。 有害な事態を巧みに回避するためにあらゆる方法で試みるだけでは十分ではありません。彼らの進路を管理し、卒業後、ビジネスまたはプロジェクトを安定した状態に戻すことをお勧めします。

2. ビジネスプロセスを再装備し、「単純な」ものを「プラグイン」に置き換えることは、IoTに移行する上で大したことの半分に過ぎません。 同様に重要なことは、新しいクラスのデバイスおよび論理（ソフトウェア）オブジェクトの下でのビジネスプロセス自体の再編成です。 専門家と「スマート」なモノのネットワーク、「高度な」ロボットおよび情報フローとの相互作用の原則を変更する必要があります。 実際、競争上の優位性を生み出し、活動から収入を得る方法を含め、ビジネスモデルを根本的に再考する必要があります。 自動化が従来のビジネスプロセスの少なくとも一部を保持する機会を残し、この点で物事のインターネットが非常に要求が厳しければ、問題を包括的かつ広く考慮する必要があります。 また、人工的にIoTの範囲と範囲を狭めると、明らかに非効率的な投資につながります。
