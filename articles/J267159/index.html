あなたが知らなかったBackup Exec 15の機能

仮想化およびクラウドコンピューティングテクノロジーの人気として、対応するバックアップツールが開発されています。 VMware、Hyper-Vソフトウェアと統合された仮想、物理、クラウド環境向けの使いやすいバックアップおよびリカバリソフトウェアであるSymantec Backup Exec 15の機能を読者に紹介し、新製品のライセンスと移行の 詳細について説明しました。









シマンテックは2015年7月にBackup Exec 15 Feature Pack 1（FP1）をリリースしました。 彼のおかげで、BE 15は新しい資質を獲得しました。 さらに、実践が示すように、すべてのBackup Exec 15ユーザーがこの製品の特定の機能に精通しているわけではありません。 だから、まず最初に。 FP1の革新から始めましょう。



機能パック1：新機能



FP1 の主な機能強化には、パフォーマンス、使いやすさ、マルチプラットフォームサポート、セキュリティが含まれます。 そのため、シマンテックの内部テストによると、物理環境および仮想環境（カタログ化あり）でのMicrosoft Exchangeのバックアップ時間は、数時間から数分、数秒に大幅に短縮されました。 LTOテープドライブへのバックアップが最大62％高速になりました。 物理サーバーと仮想サーバーのバックアップとリカバリのパフォーマンスが向上し、インテリジェント機能により、エージェントを使用したVMwareおよびHyper-V仮想マシンのバックアップから仮想ベースのバックアップに移行できます。 つまり、これらの操作を実行するためにAPIが使用されるため、作業速度が向上しました。災害対策構成（DR）でのバックアップおよびレプリケーションプロセスは、ソフトウェアエージェントを使用する場合よりもはるかに高速です。









新しいオプションを使用すると、エージェントの有無にかかわらず、バックアップスクリプトを選択できます。



変更はグラフィカルインターフェイスに影響を与えました。 改善された下部パネルでは、詳細パネルから仮想マシンのバックアップを作成できます。 [ホーム]タブの新しい[仮想マシンバックアップ]ウィジェットには、エージェントの有無にかかわらずバックアップが作成されたVMの数が表示されます。 リカバリサポート（Linuxリストア）を備えたBackup Execコマンドラインインターフェイス（BEMCLI）も改善されました。 BEMCLIは、PowershellベースのBackup Execコマンドラインインターフェイスです。









グラフィカルインターフェイスの改善により、仮想環境での製品の操作がより便利で簡単になりました。



プラットフォームについて説明します。 最も重要な変更は、vSphere 6でのフォールトトレラント（FT）仮想マシンのサポートです。フォールトトレラント構成を作成できるVMが存在することは、ユーザーがvSphere 6に切り替える主な理由の1つです。 。



Exchangeサポートの改善-Exchange Cumulative Update 8（CU8）およびExchange 2013 Cumulative Update 9（CU9）がサポートされるようになりました。 さらに、FP1を搭載した製品は、Red Hat Enterprise Linux 5.11および6.6用のRed Hat Enterprise Linux 7.1（Advanced Open File Option-AOFOなし）およびAOFOをサポートしています。 開発者は、SharePointとSQLを忘れていません。MicrosoftSQL 2014およびMicrosoft SQL 2014 SP1用のMicrosoft SharePoint 2013がサポートされています。



クラウド環境に関して特に重大なセキュリティ問題は、見過ごされませんでした。 トランスポートレベル（Transport Layer Security）では、Backup Execサーバーとエージェント間のSSL接続の強力な暗号化により強化されています-2048ビットキーによる暗号化が使用されます。 このような通信には、ネットワークデータ管理プロトコル（NDMP）が使用されます。 FIPS 2.0.5と互換性があります。 SSL証明書のサポートが登場-SHA-256（SHA-2）。これにより、Backup Execサーバーとエージェント間の通信がより安全になります。 また、サポートサービスがより迅速に機能するようになりました-問題はより早く解決されます。



次に、ユーザーがしばしば見落としがちないくつかの重要なポイントを考えます。 それでも、BE 15の作業を理解し、この製品に固有の可能性を実現するには、それらについて知ることが重要です。



クラウド環境を構築するためのBackup Exec 15



調査によると、クラウドへのバックアップ（Backup as a Service、BaaS）はすでに、SMBセグメントのクラウドサービスの顧客から最も要求されている4つの中の1つです。 BaaSの人気は、個人用デバイスやPCのユーザーにすでに馴染みのあるクラウドストレージの普及に貢献しています。



Backup Exec 15の開発者は、3種類のクラウドで製品を使用する機能を提供しています。 これらの最初のものはプライベートクラウドです。 このオプションは、多くのロシア企業、特に複数のリモートオフィスを持つ分散構造を持つ企業、またはメインデータセンターとバックアップデータセンターを運営している企業に確実に関心を持っています。 この場合、BE 15はインターネットを使用して、クラウドとして機能するリモートサイトにバックアップを複製できます。



2番目のオプションは、ハイブリッドクラウドです。 このモデルでは、顧客はシマンテックのパートナーであるサービスプロバイダー（マネージドサービスプロバイダー）のサービスを使用します。 このモデルのリモートサイトは、顧客がスペースを借りるプロバイダーのデータセンターです。 3番目のオプションは、Backup Execが統合されるAmazonまたはMicrosoft Azureパブリッククラウド（または他の主要なプレーヤー）です。



開いているインターネットチャネルを介してバックアップデータを転送する場合、VPNやその他の保護方法を作成する必要はありません。 Backup Exec 15は独自にこれを行うことができます-組み込みの暗号化を提供し、安全なプロトコルの最新バージョンを使用します。 Backup Execインターフェースでバックアップスキームを構成するとき、彼は安全なデータ転送を行い、安全な接続を作成します。 完全なソリューションを作成するには、サードパーティ製品は必要ありません。



ちなみに、プロバイダーまたはシステムインテグレーターは、BE 15に基づいた一般的なクラウドバックアップサービスを顧客に提供する機会を得ます。この場合、プロバイダー/インテグレーターのデータセンターはリモートサイトとして機能します。 つまり、シマンテックのパートナーはBackup Execを販売するだけでなく、中小規模の企業にバックアップサービスを提供することもできます。これにより、災害からの保護も提供されます。 さらに、他のバックアップ製品のように、シマンテックのサイト間の複製には個別のライセンスは必要ありません。 また、テラバイトのBE 15ライセンスオプションはクラウドモデルに適しています。



仮想および物理サーバー環境でのBackup Execの使用



この技術の人気にもかかわらず、100％のサーバー仮想化を持つ企業はほとんどありません。 通常、物理サーバーは仮想サーバーと一緒に使用され、負荷の一部はいくつかの理由で仮想化されず、仮想化されることはほとんどありません。 仮想環境のみまたは物理環境のみを保護するポイントソリューションではなく、このようなITインフラストラクチャのバックアップに1つのバックアップ製品を使用すると便利です。 特に、すべての操作が1つのコンソールから実行され、さらに、物理サーバーが仮想（P2V）にバックアップされ、仮想サーバーが物理（V2P）に復元される場合。 つまり、仮想環境で物理サーバーのクローンを作成し、ワンクリックですぐに復元（インスタントリカバリ）することができます。 これは、まさにBackup Exec 15に実装されているものです。









Backup Execは、ソリューションを選択できる汎用性の高いバックアッププラットフォームです。



企業データの急速な成長率を考えると、Backup Execの別の機能が非常に便利です。これは、バックアップのボリュームを圧縮するグローバルな重複排除です。 仮想サーバーと物理サーバーのバックアップを1か所に保存すると、重複排除の効率が高くなります。 さらに、Microsoft Exchangeメールメッセージなどの1つのオブジェクトが復元された場合でも、これによって回復時間が著しく低下することはありません。



したがって、Backup Execは、仮想環境と物理環境、単一の管理、単一の圧縮バックアップストレージを保護するための単一の製品であり、「インスタントリカバリ」のためにP2VとV2Pを移行する機能です。 もう1つの興味深い分野は、Backup Exec管理コンソールとVMware管理ツールの統合です。 これまでのところ、これはVMwareのアドオンモジュールのレベルで行われており、VMware管理者にとって便利なバックアッププロセスの監視機能を実装しています。 おそらくBackup Execの将来のバージョンでは、この統合はさらに深くなり、さらに開発されるでしょう。



正しくバックアップしていますか？ ソフトウェアエージェント



競合製品とは異なり、Backup Execは、仮想サーバーをバックアップするための3つのシナリオを実装します。エージェントなし（エージェントレスまたは仮想ベース）、エージェントの使用（エージェントアシスタント）、およびエージェントの使用（エージェントベース）です。 前者の場合、仮想マシンはAPIを使用してバックアップされます。 この場合、仮想マシンがオフラインの場合でも、VM全体とその個々のフォルダーおよびファイルを復元できます。









直感的な仮想マシンのバックアップおよび復元インターフェイス。



2番目のケースでは、ソフトウェアエージェントがVMにインストールされますが、Backup Execはバックアップ時にアクセスしません。 VMはイメージレベル（VMDK）でバックアップおよび復元でき、個々のファイルとフォルダーを復元できますが、エージェントは仮想マシン（Microsoft Exchange、SQL、SharePoint、AD）の4つのアプリケーションのいずれかを検出し、これらのアプリケーションを全体または個別の要素で復元できます。 エージェントはカタログ化手順を実行し、メタデータを収集します。 これにより、特にバックアップがディスク上にある場合、リカバリプロセスが大幅に高速化されます。 ここで、エージェントが助けになります。









バックアップからのきめ細かい復元。



3番目の方法は、仮想サーバーを物理サーバーとしてコピーします。 2番目の方法とは異なり、VM全体のバックアップコピーだけでなく、個々の要素も作成できます。 さらに、VMware vStorage APIを使用しないため、これらのAPIが無効になっているVM、たとえばVMware Fault Tolerant（FT）仮想マシンでもコピーできます。 上記のように、この機能はFP1でサポートされるようになりました。 また、IBM Lotus Notes、Oracle、およびその他のMicrosoft以外のアプリケーションのバックアップにも役立ちます。



エージェントは一度に5つずつ更新できます。 ただし、Backup Execは大規模な仮想環境向けではなく、この手順がほとんど問題にならないSMBセグメント向けに設計されています。 さらに、すべての仮想サーバーについて、詳細で詳細なリカバリ、およびそれに応じたエージェントのインストールは必要ありません。



重複排除—データ増加の解決



BE 15 の重複排除は、彼の兄NetBackupから完全に借用されています-彼は同じエンジンを使用しており、業界で最高のものの1つであり、SMB市場で完全にユニークです。 2010年以降、Backup Execのこのテクノロジーは非常にデバッグされており、問題を引き起こすことはありません。 ただし、磁気テープに転送されると、データは元の形式に変換されて再水和されることに注意してください。 これは、バックアップから確実に復元するために行われます。 バックアップがクラウドにリモートBackup Execサーバーに送信される場合、それらは圧縮されたままになります。
