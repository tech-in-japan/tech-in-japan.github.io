GSMと通話録音を使用したPBXのマネージャーとして、彼は

今日、高度な技術と申し分のない顧客サービスの時代に、誰もが平等になりたいと思っています。 多くの企業は、ランクや規模に関係なく、常にクライアントとのコミュニケーションをより便利で楽しいものにしようとしています。 私が責任者であるインターネット代理店で、これらの「アメニティ」の1つは、手作業で組み立てられたOpenvoxとAsteriskに基づいた小さなミニPBXの試運転でした。 しかし、順番に見てみましょう：



問題



電話-私たちの仕事における顧客とのコミュニケーションの主要なチャネルの1つです。 1日に処理する必要がある呼び出しが多数あります。 ローカルプロバイダーの貪欲さと信頼性の低さから、アナログラインの使用をすぐに拒否しました。 はい、現在アナログ技術は時代遅れです。 GSMを使用するという決定は、全会一致で揺るぎないものでした。 仕事の最初の数年では、そのような殺せないデバイスが電話を受信する責任がありました。











最初は、私とパートナーは同じオフィスの別のコーナーにいる順番に電話を受けました。 クライアントがソウルレスブザーのいくつかの信号を呼び出して聞き、誰かと接続するように要求すると、高い確率で、このフィンランドのデバイスを作成した風のwhiがオフィスの隅から隅まで飛んでいるのを聞くことができました。 「マルチチャネル」は、この電話機に2つのSIMカードスロットが存在するために実現されました。 会話の録音についての話はなく、音声による挨拶やメニューもありませんでした。ある専門家から別の専門家に通話を転送する方法については上記で説明しました。 一般的に、そして最も重要なことには、最初の数分から私たちに電話した人は、彼が1.5人のフリーランサーから別のシャラシュキン事務所に電話をしていると思うかもしれません。 保証と魂を込めて私たちがうまくやったからです。



多くの苦しみを経験したある晴れた日、私は大声で「もう十分だ！」と言った。



挑戦する



古いノキアに代わるすべての問題を解決するような「何か」 をする必要があります。 この「もの」は次のことができるはずです。



  • 少なくとも3つのGSM回線。
  • 頭の挨拶;
  • ビープ音ではなく音楽。
  • 通話保留。
  • コール転送。
  • 少なくとも3つの独立したチューブ、拡張可能。
  • 柔軟な着信転送設定。
  • 会話の記録;
  • 発信者番号
  • 連絡先リスト;
  • 最小ワイヤ;
  • コンパクトな機器寸法。


ソリューションの選択



この問題を解決するために、ロシア連邦の4万ルーブル以上の予算が割り当てられました。 まず、私は最も簡単な方法を選択しました。つまり、PBXをインストールおよび構成できる市内のすべての会社に電話をかけました。 「非常に多くの企業」、つまり1つ（ああ、これらの地域）があり、予算を4倍すると、すべてのウィッシュリストを作成するのに十分な金額が得られると言いました。 4を掛けたくなかったので、「B」、つまりクラウドベースのPBXの計画に進みます。 幸いなことに、あらゆる機能とウォレットを備えた同様のサービスが数多くあります。 しかし、1つ混乱しているのは、通信がサービスに完全に依存していることです。インターネットは存在しません。それらのサービスには接続がありません。 仕事-接続なし、時間通りの加入者に支払いません。 料金-接続なし。 また、サービスはいつでもサービスのコストを増加させたり、データ（呼び出し記録など）を返さないことがあります。 このオプションも私には向いていませんでした。 3番目のオプション-「松葉杖」。 さて、もっと多くのhemoが面白いです。 彼はそれを止めました。

主な要素は次のとおりです。

音声挨拶


当初、私たちを呼び出しているクライアントは、彼がどこに着いたかを伝える音声挨拶を聞くと考えられていました。 グリーティングの録音自体は、独立して、または専門のアナウンサーが声で録音し、音楽やその他の効果音を適用する専門の会社に連絡することで行うことができます。 Googleに「ボイスグリーティングレコード」を送信し、気に入ったものを選択します。 平均して、1つのエントリには1,000から5,000ルーブルの費用がかかります。 mp3形式で録音を受け取ります。これは、タンバリンと踊らずにアスタリスクを使用することを望んでいません。 ファイルをアスタリスクに適した形式に変換します。 



sudo apt-get install sox lame lame --decode musicfile.mp3 musicfile.wav sox -V musicfile.wav -r 8000 -c 1 -t ul hello.ulaw




結果のファイルを「アスタリスク/サウンド」フォルダーに入れて、メニュー編集に戻ります。 最初に、電話を取り、グリーティングを再生し、適切なユーザーに接続する必要があります。 



 exten=s,1,Answer() exten=s,2,Background(/var/lib/asterisk/sounds/hello) exten=s,3,Dial(SIP/500,30,tm)




「背景」では、パスをウェルカムファイルに移動します（拡張子なしのファイル名のみ）。 「ダイヤル」では、アスタリスクのユーザーID、グリーティングの再生に必要な秒単位の時間、2つのパラメーター「t」（応答ユーザーが通話を転送できるようにする）と「m」（待機中の音楽を再生）を送信します。 残りの「ダイヤル」パラメータについては、 こちらをご覧ください 。



通話録音


ボイスグリーティングが機能し始め、「extensions.conf」を編集しても問題が発生しなくなったら、会話の録音を強化します。 すべてのエントリが年、月、日ごとに分類されたディレクトリに追加され、録音ファイルの名前に発信者の日付、時刻、番号が含まれているとします。 複雑に聞こえますか？ そして、それは単純に行われます： 



 ... exten=s,n,Set(fname=${STRFTIME(${EPOCH},,%Y-%m-%d_%H-%M-%S)}_${CALLERID(number)}_in) exten=s,n,MixMonitor(    /${STRFTIME(${EPOCH},,%Y)}/${STRFTIME(${EPOCH},,%m)} ....




マルチチャネルおよび自動転送


3枚のSIMカードをオーブンボクスに挿入しました。そのうち2枚には呼び出し可能な番号があり、3枚目はバックアップです。 オペレーターレベルでは、1番目から2番目へ、2番目から3番目から3番目から1番目への転送を設定します。 したがって、コールの3つの完全な行が判明しました。 会社の電話の流れは大きいが、同時に3人が電話をかけるほどではないことを強調したいので、このスキームは受け入れられます。 フローが大きくなった場合は、SIMカードの数を8に増やして、通話キューを設定できます。



次に、この方法でロジックを構築します。最初のクライアントが鳴り、マネージャーが電話番号1をピックアップし、2番目の着信コールがすぐに発生し、電話番号2が鳴り、2番目のマネージャーが使用します。 3番目のクライアントは3番目のマネージャーに接続されます。 これを実装するには、「GotoIf」条件を使用できます。 



 ... exten=s,n,Dial(SIP/500,30,tm) exten=s,n,GotoIf($[${DIALSTATUS}=BUSY]?ZANAT,1) exten=ZANAT,1,Dial(SIP/501,30,tm) exten=ZANAT,2,GotoIf($[${DIALSTATUS}=BUSY]?ZANAT1,1) exten=ZANAT1,1,Dial(SIP/502,30,tm) exten=ZANAT1,2,GotoIf($[${DIALSTATUS}=BUSY]?ZANAT2,1) exten=ZANAT2,1,Dial(SIP/500,30,tm) ...


しかし、誰も電話を取らないとどうなりますか？ 全員が昼食に出かけるか、休日に電話がかかってきたとします。 ソリューションは平凡です。最も責任のあるマネージャーを選択し、彼の携帯電話番号を書き留めます。 「extensions.conf」を次のように編集します。 



 ... exten=s,n,Dial(SIP/500,30,tm) exten=s,n,GotoIf($[${DIALSTATUS}=BUSY]?ZANAT,1) exten=s,n,Dial(SIP/8914XXXXXXX@openvox,20,tm) ...


これで、20秒以内に誰も電話に出なかった場合、コールは指定された携帯電話に転送されます。 クライアント側では、これはわずかに長い応答時間のように見えます。 数時間後にさらに先に進んでリダイレクトを行うことができますが、この場合は考慮しません。



転送、発信者番号、連絡先リスト


これらの機能はすべてデフォルトでサポートされており、私の会社のニーズを十分に満たしていますが、必要に応じていつでも改善できます。 たとえば、連絡先リストは、CRMシステムからxmlファイルにロードできます。
