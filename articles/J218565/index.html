今年のMicrosoft Azure Platform開発結果







会議では、Microsoft Azureクラウドプラットフォームの開発結果を年間を通じて発表しました。 それらの中には：



  • マイクロソフトは、プラットフォーム上で300以上の新機能をリリースしました。
  • この会議では、さらに44の新機能が発表されました。 以下の詳細。
  • 最近、新しいプラットフォームのデータセンターが中国と日本で立ち上げられました。 マイクロソフトは、中国でクラウドプラットフォームサービスを提供する唯一のグローバルクラウドプロバイダーです。
  • Microsoft Azureには現在、世界中でデータとコードをホストする16のパブリックリージョンがあります。
  • Fortune 500の最大の企業の57％以上がすでにAzureプラットフォームを使用しています。
  • Microsoft Azureには、250,000を超えるWebサイト（Azure Webサイト）と100万を超えるデータベース（Azure SQL Database）があります。 Azure Storege分散ストレージでは、ユーザーは既に20兆を超えるオブジェクトを保存しています。
  • 100万人以上の開発者が、デスクトップ開発ツールと統合するAzureベースの開発プラットフォームであるVisual Studio Onlineにサインアップしています。


これらの結果に加えて、Microsoft Azureの現代的な使用の興味深い例がいくつか示されました。たとえば、米国のカナダのホッケー試合の210万の同時接続の記録的なピークを持つ数億人の視聴者を対象に、米国のNBC Sportsチャンネルでオリンピックのオンライン放送を整理するために。











もう1つの例は、WindowsおよびXbox向けの人気ゲームであるTitanfallです。マルチユーザーモード、人工知能、およびネットワーク機能は、Azureクラウドに基づいて完全に実装されています。 Microsoftによると、ゲームの開始時には100,000台の仮想マシンのプールが使用されていたため、ゲームのスムーズな開始と、プレイヤーからの関心のあるトラブルのない処理が保証されました。



Azure Webサイト



PaaS WebアプリケーションホスティングサービスであるAzure Web Sitesは、Java開発者にとって有用なものも含め、数多くの注目すべき革新を受けています。







Javaサポート 。 Webサイトプラットフォームは、以前は.NET、PHP、Python、およびNode.JSに基づくホストコードのサポートを提供していました。 会議中に、Javaコードの投稿のサポートが発表されました 。







デプロイすると、開発者はコントロールパネルでTomcatまたはJettyアプリケーションサーバーを選択できます。 さらに、大量のメモリを消費する可能性のある64ビットアプリケーションの配置がサポートされています。 新機能と使用例の詳細については、 こちらをご覧ください 。



無料のSSLオファー 。 Webサイトの標準版のユーザーは、WebアプリケーションでSSLモードのサポートを有効にする無料の機会を得ます。 ユーザーは、1つの無料のIPアドレスと5つのSNIを利用できます。



ホスティングプランとリソースグループ 。 Webサイトの価格はわずかに変更され 、ベーシックサービスの新しいレベルのプロビジョニングが追加されました。これは専用の計算能力を提供しますが、自動スケーリングなどの多くの機能にアクセスできないため、標準レベルよりも最大50％安くなります。 Azure Web Sitesの新しい価格設定の詳細については、 公式ページをご覧ください 。 ホスティングプランに関する詳細は、 この発表に記載されています。







標準のWebサイトホスティングプランが更新されました。無料のSSLオファーに加えて、Webサイトに割り当てられるストレージ容量が10 GBから50 GBに増加しました。



新しいホスティングプランに加えて、さまざまな価格設定の複数のWebサイト（リソースグループ）を組み合わせる新しいアプローチが導入されました。 この機能は、リソースを管理し、タスクごとにリソースを異なる方法で割り当てるのに役立ちます。 この記事から新機能の詳細を学ぶことができます。



Kuduの更新 。 Azure Web Sitesで起動された各Webサイトには、多くのサイトパラメーターを簡単に管理し、診断情報とログを受け取ることができるKuduコントロールパネルがあります。







Kuduの更新により、プロセスのリストを表示および管理し、プロセスメモリダンプをロードし、プロセスをシャットダウンする新しい機能が導入されました。



Azureモバイルサービス



Azure Mobile Servicesは、iOS、Android、Windows、Windows Phone、HTML5 / JS、PhoneGap、Xamarinなど、さまざまなモバイルプラットフォーム向けに単一のバックエンドを編成する可能性を提供します。 Build 2014の一環として、クラウドプラットフォーム用の多くの新機能が発表されました。







バックエンドとしての.NET モバイルサービスに関する最も興味深い発表の1つは、サポートされている.NET言語で、Azure Mobile Servicesのモバイルアプリケーション用のバックエンドコードを開発する機能の発表でした。



ローカルおよびリモートのデバッグ 。 以前の発表とともに、モバイルアプリケーションのバックエンドコードのローカルおよびリモートデバッグの可能性が提示されています。



オフラインデータ同期 。 Azure Mobile Servicesバックエンドには 、オフラインデータ同期のサポートが組み込まれています。 現在、組み込みツールの助けを借りて、開発者は、アプリケーションがインターネットに接続せずに動作し、その後バックエンドと同期する場合、スクリプトをサポートするアプリケーションを構築できます。



Kindleサポート 。 Kindleデバイスへのプッシュ通知の送信のサポートを発表しました。 例と詳細情報は、 次のリンクから入手できます。



Azure SQLデータベース



サービスとして利用可能なSQL Serverクラウドベースリレーショナルデータベースは、更新のかなりの部分を受け取りました。







新しい制限とSLA 。 1つのSQLデータベースのサイズの制限が150 GBから500 GBに増加しました。 この革新に伴い、SQL Database PremiumサービスのSLAは99.95％に増加しました。



自動バックアップ（セルフサービス復元） 。 SQL Databaseサービスは、最大31バージョンのデータベースを蓄積し、以前に保存したバージョンのいずれかにロールバックする機能を備えた自動バックアップのサポートを受けました。







アクティブなジオレプリケーション 。 アクティブなジオレプリケーションの新機能により、SQL Database Premiumユーザーは、世界中に分散している最大4つのアクティブなデータベースレプリカを自動的に受信できます。







これらのレプリカは、災害の結果としてデータセンターの1つに障害が発生した場合の緊急切り替えに使用できます。

Buildクラウド会議の最も気持が良く、予想外の発表の1つは、クラウド機能、課金、Webアプリケーションとサービスの機能、コードおよびその他の機能を管理するためのポータルの新しいバージョンの発表でした。

仮想マシンとネットワーク







Azureクラウドでのインフラストラクチャ管理の主要な発表として、2つのサードパーティツールであるPuppetとChefの仮想マシン作成時のワークフローへの深い統合に関する発表がありました。







PuppetとChefのサポート 。 Puppet LabsおよびChefと共同で、Azureコントロールパネルに緊密に統合された新しい仮想マシン管理メカニズムが導入されました。







仮想マシンギャラリーでPuppetと統合してインフラストラクチャをホストするために、LinuxベースのPuppet Enterpriseを使用したイメージが追加され、展開後に仮想マシンの作成段階での統合が許可されます。 これを行うために、エージェントが仮想マシン（Windows ServerまたはLinux）に追加され、デプロイされたPuppetとの通信が提供されます。



Puppet Plugin for Visual Studio 2013も開発者向けにリリースされており、Puppetモジュールの作成と管理が簡単になります。 さらに、 Microsoft AzureでPuppet Enterpriseを展開するためのダウンロード可能な入門ガイドが 、開発者およびITプロフェッショナル向けに提供されています。



VMキャプチャー 。 待望の機能を使用すると、データディスクが接続された仮想マシンのスナップショットを取得して展開できます。 この機能は現在、PowerShellコマンドを介して使用できます。



新しいAzure Automation Service 。 ビルド会議の重要な発表は、 Azure Automationサービスの予備バージョンのリリースでした。これは、ワークフローを実行するための非常にスケーラブルで信頼性の高いメカニズムを使用して、Azureクラウド内のリソースの作成、デプロイ、監視、およびメンテナンスを自動化するように設計された新しい機能です。



開発者は、新しいサービスを使用して、頻繁に使用されるステップ（市販のリソースセットの展開など）の実行を自動化できます。 新しいサービスの発表はここで見つけることができます 。 サービスの説明は公式ウェブサイトで見つけることができます。 この記事ではウォークスルーを利用できます。



PowerShell for Azure Resource Manager（ARM） 。 Azureオートメーションのインストゥルメンタルサポートは、クラウドアプリケーションとリソースをホストし、自動的にスケーリングできる新しいPowerShellベースのフレームワークによって提供されます。 ARMは、以下で説明するAzure SDK 2.3の一部です。 この記事では、開発およびテスト環境を展開するための新機能の使用について詳しく説明しています。



自動スケーリング 。 自動スケーリングメカニズムは商用運用されています。 このサービスを使用すると、仮想マシン、クラウドサービス、ウェブサイト、モバイルサービスなどのコンピューティングリソースを自動的にスケーリングできます。



仮想ネットワークの目新しさ 。 Point-to-Site VPN仮想ネットワーク接続機能は、商用利用（GA）用にリリースされました。



その他の革新には、Dynamic Routing Gateway機能のサポート、VMの他のサブネットへの移行 、および仮想マシンの静的IPアドレスを構成する機能が含まれます。 これらの機能はすべて、主にPowerShellコマンドで使用できます。



開発ツール



とりわけ、Azure SDK for .NETクラウドアプリケーション開発ツールが更新され、Visual Studioと新しい便利な機能が統合されました。



Azure SDK for .NET 2.3 ビルド会議中に、Visual Studio 2013 Update 2 RCに統合されたAzure SDKクラウドアプリケーション開発ツールの更新が導入されました。 パッケージのインストールまたは更新は、 Web Platform Installerから利用できます。 以下に、最も注目すべき革新のいくつかの説明があります。



Webアプリケーション作成ウィザード 。 Webアプリケーションプロジェクトウィザードにいくつかの新機能が追加されました。 最初に、モバイルサービスバックエンドAPIを開発するために設計されたAzure Web Serviceという新しいWebアプリケーションテンプレートが導入されました。







次に、ウィザードは、Webアプリケーションをホストするための新しいリソース（Webサイトサービスのインスタンスまたは仮想マシン）を自動的に作成する機能を追加します。 どちらのオプションにも、クラウドでホストされているWebアプリケーションのリモートデバッグ機能があります。



Webサイト管理 。 サーバーエクスプローラーから、サイト自体だけでなく、サイトに展開されたアプリケーションファイル、およびアプリケーションログファイルも管理できます。







クラウドサービス 。 クラウドサービスをテストするために、管理者権限なしでサービスをデバッグできる新しいEmulator Expressが導入されました。 さらに、クラウドサービスでは、マネージアプリケーションコードとネイティブアプリケーションコードのリモートデバッグを利用できます。







プッシュ通知のテスト 。 更新されたVisual Studio 2013 Update 2 RCツールは、Azure Notification Hubsサービスを介したプッシュ通知の送信をテストするための新しいツールを導入しました。







この記事でアプリケーションの詳細を読むことができます。



仮想マシン管理 。 サーバーエクスプローラーを使用して、Visual Studioから仮想マシンを直接作成および削除できるようになりました 。 Azure Virtual Machine Galleryのマシンを選択できます。











さらに、Visual Studioの新しいAzure SDK 2.3機能を使用して、仮想マシンにデプロイされたアプリケーションのリモートデバッグモードを有効にできます 。
