ソースのコメント手順



ほとんどすべてのプログラミング言語にはコメントの概念がありますが、Luaも例外ではありません。 一般的にコメントとは、プログラムのソースコードの説明であり、コメントされたコード内に直接配置されます。 コメントの構文は、プログラミング言語によって決定されます。 コンパイラまたはインタープリターの観点から見ると、コメントはプログラムテキストの一部であり、そのセマンティクスには影響しません。 T.O. コメントはコードの実行順序に影響を与えません。コメントは自分自身と、しばらくしてソースの調査を開始できる人のために作成します。 コメントは、デバッグツールとして使用できます。 コードを一時的に実行から除外することにより、コードにコメントを挿入することができます。これにより、別のセクションでエラーを検索する瞬間が簡単になります。 Luaには2種類のコメントがあります。



単一行コメント



文字シーケンス「-」から行末までに書かれたものはすべてコメントと見なされます。



複数行コメント



複数行のコメントは、シーケンス「-[[」で始まり、シーケンス「]]」で任意の行数で終了します。コメントの使用例： 



--[[  2  a  b   ]] function summ(a,b) return a + b end print(summ(5,7))-- 12


コードに対するコメントの質の問題にはあまり注意を払わず、自分で決めるのは全員の責任です。



  • それ自体に対する良いコードのコメント-すなわち 明確に書いて、変数に意図的に名前を付ける場合、ほとんどの場合、コメントは不要です
  • コメントを書かないでください-コメントのために。 すなわち すべてが明確な場所にコメントを書かないでください。
  • 簡潔さは才能の姉妹です。 コメントをシンプルでわかりやすいものにします。
  • ファイルと関数の目的についてコメントします。


変数、定数、スコープ



ほとんどのプログラミング言語には変数があります。 ほとんどの場合、この概念は名前付きまたは別の方法でアドレス指定されたメモリ領域として理解され、そのアドレスまたは名前を使用してデータにアクセスし、プログラム実行中に値を変更できます。 Luaでは、変数を適用する必要がある場所で変数を作成できます。 変数を使用する前に事前に変数を宣言する必要のある特定のブロック（ Pascalなどの言語のような ）はありません。 変数宣言の例を次に示します。 



 a = 1 b = 3 local c = 'example' d,pi = 2, 3.14 a = 'stroka'


他の言語の愛好家は、いくつかの機能に気づいたと思います。



  • 変数のタイプを指定する必要はありません、値はそのタイプを決定します
  • 変数のタイプは変更できます。 変数が数値で（例では「a」）、文字列値が割り当てられた場合、小文字になりました
  • 1つの「=」記号による並列割り当てが利用可能です（ d、pi変数 ）。 このプロパティは、初期化の開始時間を大幅に短縮できます。また、a、b = b、aなどの中間変数なしで変数値を交換することもできます。
  • ローカルキーワード。 この単語は、変数のスコープを制限するために使用されます。変数が操作ブロック（ 関数、ループ、条件ステートメント ）内でローカルに宣言されている場合、変数は変数内でのみ表示されます。 ローカル変数がファイルのグローバルコンテキストで宣言されている場合、この行からファイルの最後まで表示されます（ スコープについては、以下で少し説明します ）。


定数について見てみましょう。 定数とは、使用中に値が変化しない変数を意味します。 多くの場合、プロジェクト（ アプリケーションバージョンなど ）で繰り返し使用されるいくつかの値を定数に格納すると便利です。一部の言語では、文字通りの意味で定数を作成するためのツールがあります。 一度宣言された変数とその後の値は変更できません。Luaにはそのようなツールはありません。 すべて（ 関数の実装も）のせいにすることができます。このため、絶え間ない変更から保護するための便利なオプション（名前の順序）を考慮することができます。 BIG_LETTERS_S_WORD_WORD_INTERCARRATINGのように、コード内で目立つように定数に特別な方法で名前を付けることをお勧めしますが、これはルールではなくアドバイスです。



有効な変数名、定数、関数の形式を簡単に説明します。宣言的なもので違反できないもの、ヒントにすぎないものがあります。



  • 変数には、ラテン大文字と小文字、数字、アンダースコア「_」を含めることができます
  • 変数は数字で始めることはできません
  • 変数には予約語の名前を付けることはできません：and、break、do、else、elseif、end、false、for、function、if、in、local、nil、not、or、repeat、return、then、true、until、while
  • 大文字と小文字が区別されます。 Luaのほとんどの言語と同様に、大文字と小文字は変数の一意性の決定要因です。 lua、LUA、lUa、およびLuAはすべて異なる変数です。
  • アンダースコアで始まる変数の名前を乱用しないでください。もちろん、これらの名前が言語によって内部メカニズムの命名に使用されていないことが確実でない限り、
  • 広い範囲を持つ意図的な変数名を使用します。 これは次のように理解する必要があります-変​​数がプロジェクトの大部分で使用される場合、変数は何も意味せず、短い名前を持つべきではありません
  • 変数、定数、関数の命名形式を分けます。 たとえば、定数にはSUCH_FORMAT、変数about_this、および関数にthis_Formatを使用できます。これにより、コードがより便利になります。


スコープとは、データに直接アクセスできるコード空間のことです。 スコープの構成の原則を誤って理解すると、多くの不快なエラーが発生する可能性がありますが、これらは通常、排除するのが困難です。



  • 変数は表示されません-すべてが正常で、変数がnil（ 未定義 ）であるように思われます。 通常、このエラーは簡単に特定できますが、必ずしも簡単に排除できるとは限りません。
  • 変数は使用されていない場所で表示されます-この一見小さな問題は、非常に複雑なバグにつながる可能性があります。たとえば、コードの別のセクションの変数がこの領域に表示されないことを期待し、その存在しないことを確認して（ 初期化するため ）成功で終了せず、初期化コードが実行されないため、別の場所で定義された値が使用されます。 この問題を回避するには、常にlocalという単語を使用して、すべての変数を最小限に見えるようにしてください。 グローバル変数の使用は常に最小限に抑える必要があります。


さらに、スコープの不適切な構成の例に出くわすことが何度もありますが、これらの場所でこのセクションを完成できる理由を強調します。



モジュールとプロジェクト組織



プログラムモジュールは、プログラムの機能的に完全なフラグメントであり、ソースコードを含む個別のファイルとして設計されています。 ご存じのとおり、 Corona SDKのプロジェクトの実行はmain.luaファイルから始まります。 アプリケーション全体を1つのファイルに記述する場合、多くの問題が発生する可能性があります。例を挙げます。



  • 大量のコード。 ほぼすべての本格的なゲームのソースコードには、数百、数千、数万、場合によっては10万行以上のコードが含まれることがあります。ある段階ですべてのコードを1つのファイルに配置すると、このプロジェクトを維持するのが非常に難しくなります
  • たくさんのシーン。 ほとんどの場合、実際のアプリケーションには複数のシーン（ 機能画面 ）が含まれます。たとえば、さまざまなメニュー、ロード画面、複数の戦闘シーン、オープンワールドシーンなどです。 繰り返しますが、すべてを1つのファイルに配置する場合、シーンを切り替えるときにシーンを作成および削除するための多数の手順でコードを積み重ねる必要があります。 コードは再び非常に複雑になります
  • ユニバーサルポータブルコード。 アプリケーションの作成プロセスでは、多くの場合、ライブラリに転送するタスクと同じタスクが発生し、これらのライブラリを次のプロジェクトに簡単に転送できますが、すべてを1つのファイルに書き込むと、完成したライブラリの移植性の問題は非常に困難になります。巨大なファイルからフラグメントを分離するのにどれだけ便利なコードか。


プロジェクトを整理し、すべてを1つのmain.luaファイルに書き込むのが間違っている場合、他の問題が発生する可能性がありますが、レッスンではこれを行わず、プロジェクトに選択した方法を示します。あなたのものを考え出すのはあなた次第ですが、私はこの方法でプロジェクトのさらなる例を作成します。 したがって、プロジェクトソースをいくつかのファイルに分割する必要があり、さらにこれらのファイルにはさまざまな目的があり、これに基づいて、ファイルの各タイプには名前のプレフィックスによって決定される独自の命名順序があるという結論に達しました。 この方法は個人的に私のものであり、私はそれが最善だと主張していませんが、私の記事で見るゲームのさらなる例はプロジェクトを設計するためのそのようなメカニズムを持っているので、私はそれを基礎とすることにしました。 便利になる可能性のあるファイルの種類について簡単に説明します。

ユニバーサルライブラリはlibプレフィックスです。 これらのファイルには、アプリケーションのさまざまな部分で後で使用できる既製の汎用関数を配置する必要があります。たとえば、プロジェクトの三角関数をlibTriginom.luaファイルに配置すると、これらの関数は補充され、次のプロジェクトではファイルを転送するだけで簡単に使用できます接続することにより。 ユニバーサルライブラリのアプリケーションのグラフィックデザイン要素へのバインドを避け、初期値を呼び出しパラメーターに転送してください。 簡単なライブラリの例を示します。

ウィジェット/サービス-svcプレフィックス。 ウィジェットはライブラリに類似していますが、主な機能はグラフィカルコンポーネントの作成に基づいています。 ウィジェットを使用すると、複合インターフェースを作成すると便利です。 scファイルのメインコードの一部として作成されるシーンの基本的な静的コンポーネントがあり、静的コンポーネントの上に、ポップアップ、可動フォーム、その他のアメニティの形で動的ウィジェットを作成できます。 たとえば、次のように、すべてのプロジェクトウィジェットを単一のインターフェイス（ 関数の標準セット ）で設計するのが最も便利です。

コード挿入



コードの挿入はプレフィックスinsです。 ファイルのサイズをどのように削減しようとしても、これが常に可能であるとは限りません;この場合、共通の目標によって結合された一部のセクションを別のファイルに取り出すことができます。 例がありますが、コードがあります： 



 --main.lua local P = '123' k = 23.2 local function sum(a,b) return a+b end local function razn(a,b) return ab end local a,b = 2,3 print(sum(a,b)) print(razn(b,a))


ここで、sumとrazn関数を別のファイルInsExample.luaに取り出します。2つのファイルがあります。 



 -- InsExample.lua function sum(a,b) return a+b end function razn(a,b) return ab end --main.lua local P = '123' k = 23.2 require 'insExample' local a,b = 2,3 print(sum(a,b)) print(razn(b,a))


別のファイルに転送するとき、関数をグローバルにした（ ローカルを削除した ）ことに注意してください。そうしないと、関数はmain.luaの範囲外になります。 また、insExample.luaファイル内ではローカル変数Pは表示されませんが、グローバル変数kは表示され、requireの後に変数kを宣言すると表示されなくなります。



重要 ： あらゆるユースケースで、ファイル操作が必要です。この呼び出しは、実行スレッドがこのコードセクションを初めて通過するときに1回だけ行われます。これは、長い間複雑なバグに悩まされない場合は、常に考慮に入れる必要があります。 また、私のように、db / lib / svcのようなモジュールで、戻る前に特定のM変数を使用してデータを収集する場合（ またはその他 ）、この変数をローカルにすることを忘れないでください。そうしないと、各次の要求が前の結果を上書きし、これにより、さらに複雑なエラーが発生します。

  • <少ない
  • >もっと
  • <=より小さいか等しい
