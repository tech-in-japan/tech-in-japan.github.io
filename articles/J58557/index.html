Windows 7 XPMプレビュー（XPモード）





Windows 7の Melkomyagkiの新しいOSでの機会のレビューに加えて、機会のレビューに関する記事のコラムを残すことにしました。



今日、新しいシステムに実装されるテクノロジー/アプリケーションに注目したいと思いますが、小さなものによると、Windows 7 RC 7100（5月5日）の公式実装で発表されます。



habratopikaの開始前に、1つの式を引用します。

「MicrosoftがWindows 7がWindows Vistaと完全に互換性があると言っていた場合、Windows XPとほぼ完全に互換性がある、または現在実行中のすべてのWindowsアプリケーションとほぼ100％互換性があると言えます。」



行きましょう=>



そのため、Microsoftは1か月ほど前に、Windows 7 RCのリリースと共に発表される新しいテクノロジを発表しました。 この技術に関するレポートが登場してから2日が経ちました。 Windows 7 XPMと呼ばれます 。 この頭字語は、 Windows 7 XP Modeの略です。 （以前：仮想Windows XPまたは仮想XP、VXP）。



XPMは、Microsoftの次世代製品ラインであるVirtual PC 7の一部として構築されています。VirtualPC 7は、被害者のコンピューターに存在し、そのコンピューターで動作するために、 プロセッサーベースの仮想化テクノロジー（Intel、AMD）のサポートを必要とします Hyper-V、Microsoftのサーバー側仮想化プラットフォーム。 しかし、これらすべてにより、XPMはクライアント用のHyper-Vではなく、ローカルPC用のソリューションです。



XPモードは、Virtual –PC仮想環境と、Windows XP SP3の完全にライセンスされたコピーで構成されます 。 Windows 7 Professional、Enterprise、Ultimateのシステムの所有者は無料で利用できます。



XPMパッケージは、購入時に基本システムに添付されませんが、サードパーティの更新と見なされます。 また、MicrosoftのWebサイトからダウンロードできます。



XP ModeはVirtual –PCとほぼ同じように機能しますが、当然違いがあります。 これらの違いを見てみましょう。

  1. 別のXPMアプリケーションウィンドウを実行する必要はありません（つまり、バックグラウンドで動作します）
  2. XPモードのプログラムは、インストールされるとすぐにローカルシステムにインポートされます（Windows 7）。 このアプリケーションのアイコンは、[スタート]メニューに配置されます。
  3. XMPは、Active Directoryおよびグループポリシーを通じて制御できます。




つまり、このテクノロジを使用すると、Windows 7ユーザーは、時代遅れではあるがまだ忘れられていないOSおよびWindows XP（IE 6、Office 2003など）でアプリケーションを実行できます。



図：XPモードで起動したOffice 2003、Windows 7上のOffice 2007
