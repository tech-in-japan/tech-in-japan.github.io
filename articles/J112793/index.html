IPオフィスに関する会議

前に始めたトピックの続きで、Avaya IP Office 500 PBXでの電話会議の設定についてお話したいと思います。

実際、まさにそのような機会が利用可能であるため、このPBXは当時購入されました。 しかし、この関数を使用して、特に設定することは、初心者にとって非常に簡単で非論理的なタスクです。

この出版物では、私が出会ったサンゴ礁と海賊船について説明します。 そして最後に、初心者向けのIPOインストールマニュアルが書かれた箱。



合計で、3つの収集オプションがあります。

1）電話を使用する。 たとえば、Avaya 5610 IP Phoneを使用している場合、単一の通話者が呼び出されています。 ダイヤルし、会議が収集されていることを説明した後、コールは保留になります。 次に、デバイスのボタンが回線を切り替えて、回線の数に応じて2番目の加入者にダイヤルアップします。 その結果、保留中に送信されたすべてのコールは、対応するボタンを使用して1つの会議に結合されます。



通話時間は、加入者数の倍数です。 ダイヤル時間は10〜30秒、説明用に20〜30秒、操作用に10〜20秒です。保留にしてから次の番号にダイヤルします。 1人あたり合計1分以上。 3〜4人の場合、実行可能なオプション。



2）無料のPhone Managerアプリケーションを使用します。

Phone Managerを使用すると、電話を管理し、不在着信、ボイスメッセージなどの電話の状態を追跡できます。

承認は、管理する内部番号によって行われます。 これを可能にするには、Managerでこの拡張機能を持つユーザーのパスワードを設定する必要があります。

収集スキームは同じです。プログラムを介してのみ参加者に電話をかけます。 利点は、参加者が比較的無制限であることです。 つまり、それらの数は、外部回線の数、この内線番号とライセンスの可能な回線によって制限されますが、電話の回線によっては制限されません。

しかし、そのような計画に従って5人を集めることは非常に現実的であり、約1時間で60人が集まります。 どの参加者が回線にこれほどまでこだわるのに同意しますか？ さらに、秘書の神経、従業員の労働時間、および発信のためのお金です。



3）3番目のオプションでは、SoftConsoleおよびVoiceMailProアプリケーションのライセンスを購入する必要があります。

SoftConsoleはPhone Managerの高度なバージョンです。 すでに加入者のグループを形成でき、外部番号と会議室の独自のリストを持っています。

SoftConsoleにログインするには、内部番号とパスワードによる認証も必要です。



これには、顧客の要件とPBX構成の背景が必要です。

オペレーターは、それぞれ30行でE1 PBXに送信します。PRIを使用するには拡張カードが必要です。 1つのボードで、E1用の2つのポート。

顧客は最大60人の会議を収集することを望んでいたため、それぞれ30行の2つのストリームが必要です。

セレクタ（電話会議）では、内部の電話交換と固定電話、長距離電話番号、携帯電話番号の両方が参加する必要があります。 また、会話を録音する機能も必要です。

だから、私たちは何を家を建てるべきですか？

Softconsoleを使用すると、Phone Managerと同様に、簡単なコールを収集できます。

セレクターを手動で選択するオプションは、回線での待機時間が完全に不十分であるため、すぐに削除されます。 自動収集に向かって掘り下げる必要があります。





このため、アプリケーションには2つの会議室があります。

プログラムを初めて起動すると、会議室のボタン そして 設定で名前が指定されるまで、非アクティブになります。 これを行うには、設定アイコンをクリックします。 [会議通話の設定]タブに移動して、部屋の名前を指定します。







そのため、いずれかのボタンをクリックすると、会議室ウィンドウが開きます。

「ユーザーの追加」ボタンがあり、クリックすると、すべてのPBX内線番号のリストが開きます。 それらから始めましょう。

いくつかの数字を左から右に移動して「OK」。 ただし、会議室に戻った後、「ユーザーを招待」ボタンはアクティブになりません。 機能が定められていれば、明らかに、機能するはずです。

ここではVoiceMailProが必要です。 このアプリケーションのサービスは、自動呼び出しに必要です。 インストールしてSoftConsolを再起動すると、ボタンが黒に変わります。 そうでない場合は、しばらく待つ必要があります。

[ユーザーを招待]ボタンを使用して、電話をかけることができます。 クリックすると、すべてのアイコンが黄色の枠で囲まれます-ステータスは「招待済み」です。

内線番号に電話がかかり、受話器で女性の声が聞こえます。 彼は内線番号を発音し、「1」を押して招待を受け入れます。 パルスダイヤルを使用するデバイスがまだあるため、これはすでにマイナスです。 そして、平均的なユーザー、特に工場のディレクターは、どのセットについても気にしません。 それには2つの状態があります-動作中、動作していません。

私が以前にテストしたロシア製のシステム-議会は受信者にバックグラウンドミュージックまたは会議に関する通知を送信しただけで、何も押す必要はありませんでした。

参加者が招待を承諾すると、ウィンドウのステータスが変わり、フレームが緑色に変わり、下にチェックマークが付いた緑色の円が表示されます。

クリックすると、現在の会議が右上のウィンドウに表示されます 。

したがって、次のステップは外部番号の追加です。

それらを追加するには、設定に移動し、[着信]タブに移動する必要があります。 ここで、「新規」ボタンをクリックして、加入者の名前と番号に署名します（接頭辞（通常は9）。これを介して通話が発信されます）。







追加された番号は、右上のウィンドウに表示されます。 内部番号とは異なるアイコンが表示されます。







今、それらをダブルクリックすることで、それらを呼び出すことができます。

会議室の「ユーザーの追加」リストに表示されると仮定するのは論理的です。 しかし、そこにはありませんでした。 まだ内部のみがあります。

ちなみにロシア語でも書かれているドキュメントを新たに読むと、BLFグループとそのユーザーを作成するという解決策につながります。 左下のウィンドウがこれを担当します。

さまざまなコンテンツを作成できます。 たとえば、取締役、会計士、エコノミスト、クリーナーなどの役職に応じて人々をグループ化し、既製のリストに従って会議を開催できます。

リストが作成されたら、ルームウィンドウに移動する必要があります。 繰り返しますが、会議室のウィンドウを開き、「すべてのウィンドウの上」にDawを配置し、BLFから1つ以上の連絡先をドラッグします。

なぜそんな困難なのかわからない。 つまり、そうすると、非常に便利です。 ただし、[ユーザーの追加]をクリックすると、すべてのユーザーが表示され、これらのBLFグループを一覧表示することもできます。

しかし、転送自体もスムーズに進んでいません。 初めてではなく、理由のために転送されます。 連絡先が部屋のウィンドウに表示されるように、ウィンドウ上でマウスを移動する必要があります。 これは、ウィンドウタイトルの近くのどこかで機能し、ウィンドウタイトルが表示される領域ではほとんど機能しません。

次の問題は、すべてのユーザーが一度に招待されることです。 最初に3人を割り当ててから、さらに10人を招待することはできません。 さらに、複数の連絡先を選択することもできません。 単に不可能です。 誤って50人をそこに転送した場合、コンテキストメニュー→削除を選択して、それぞれを座って削除します。 または、SoftConsoleを閉じて再度開くこともできますが、どういうわけかすべてが適切な場所にありません。 まあ、神は彼を祝福します。

そこで、会議を開催しています。 ユーザーを招待すると、ユーザーは黄色に変わり、10〜15秒かかり、外部番号からの選択が消えます。 電話への電話は来ません。

ここでは、トレースなしでは実行できません。

しかし、IPOからトレースを読み取るには、少なくともメッセージ形式に関する情報を持っているか、非常に注意深い綿密な情報が必要です。

一方、時間はすでに7時に近づいていました。 しかし、モスクワはまだ機能しているので、私たちはそれを悩ませ始めます。

サプライヤ-OCSはあらゆる方法で私たちを助けようとしました（後でわかったように、これは義務の一部ではありません-Avayaの技術サポートは、原則として他のオフィスによって個別に支払われ、提供されます）が、賢明なオファーを1つだけ発行しましたが、一箇所で解決。

当初、彼らは会議で外部の電話を集めることは不可能だと言っていました。彼らのソフトウェアの第4バージョンでは、今では見つけるのが非現実的である特別なプログラム（会議センター）を使用して可能だったと思われます。

しかし、これはナンセンスです-内部番号のみを通過できるのであれば、なぜ電話会議の可能性を宣言するのですか？

一般に、彼らが提案した唯一の解決策は、仮想内線番号（たとえば、H323またはSIP）を開始し、それらから目的の外部番号に無条件にリダイレクトすることでした。

確認後、スキームは非常に機能していることが判明しましたが、重大な問題があります。新しいユーザーがManagerで起動します。 秘書へのアクセスを許可することは、一方では無意味であり、他方では危険です。 毎回自分で登って、オペレーターの機能を実行するという特定の欲求はありません。 したがって、検索を続行します。

フックまたは詐欺師によって、聴衆の助けと友人への呼びかけで、彼らはなんとか真実を表面に引きずり込みました。 問題は、VoiceMailが電話番号を知らないことであり、オペレーターの機器がこの番号を知らなかったため、メッセージでリソース要求を送信し、応答として「No cct / chan available」を受信したことでした。

Short Codesグループがコード9Nを修正し、ASRが電話をかけるために彼に指示するとすぐに、すべてが機能しました。











SoftConsolまたはPnone Managerで作成された場合でも、既にASRに関連付けられているため、特定の番号を通過したすべての電話からの通常の呼び出しが判明しました。 これとは別に、会議を収集するボイスメールの場合、このルールは規定されておらず、ショートコード設定を使用しようとしますが、すべてが明確になります。

電話会議を機能させるために必要なことはこれだけです（親切な人に感謝します）。

SoftConsoleでの許可に使用される番号が関係する場合、会議を含むすべての会話を記録できます。 ボタンを押す必要があります 。 レコードはIPOディレクトリの奥深くに保存されます。「C：\ Program Files \ Avaya \ IP Office \ Voicemail Pro \ VM \ Accounts」-判読不能な名前で。 検索は、時間と期間の変更に便利です。 一般に、このような記録を聞き、整理するための特別なソフトウェアがあります：IPO記録。 しかし、コールセンターではなく、この種の製品が必要です。 そして、何とか祖父の方法-指揮者。

不快または不明についての詳細。 ボタンを押して会議を終了することはできません。 リーダーとなる電話はないため、会議から切断された場合、最後の参加者まで電話が続き、秘書はリスト内のアクティブな会議のみを神経質に見ることができます。

ところで、いつでも他の人を現在の会議に接続できます。 これを行うには、秘書の電話から相手に連絡する必要があります。受話器を持ち上げた後、「アクション-会議室に追加1/2」をクリックします

一般に、会議用に以前は特別なソフトウェアがありました。ConferenceCenterは、サプライヤと他のエンジニアの両方の言葉によると、デュプレックスを含むすべての基本機能をサポートしていましたが、何らかの理由でこの製品のサポートが停止し、ライセンスを購入することさえできなくなりましたSoftConsolとは異なり、ダウンロードすることはできますが、見つけるのは簡単ではありません。

しかし、前の問題に関するトラブルは終わりませんでした。 通話中にビープ音が聞こえた後、通常の接続が行われることが判明しました。 おそらく、これは2番目のスレッドによるものです。

そして、本当の探求が始まります。 すべての手段が開始されました。

他の呼び出しが干渉せず、恐れることなく再起動できるように、作業後の時間を選択しました。 準備されたIP電話（リモートオフィスからテストできるように）、SoftConsole、Manager、Monitor、System Statusを起動し、急いで。

次々に電話をかけるので、10〜15回呼び出します。

まず、機能に注目しました-問題は1回だけ表示されます-1回の呼び出しは正常で、1回はビープ音が鳴ります。 一部のコールは長続きせず、同じビープ音をハンドセットに無限に送信します。

モニタートレースでは、依存関係を追跡できました。 コールが最初の行に行く場合、彼は承認を受け取り、コールが正常に通過することが判明しました。

この段階では、応答がオペレーターOutOfServiceから来た場合、呼び出しがどのように通過するかは不明です。

最後の手段は、システムステータスです。 特に、ストリーム内の行のステータスを追跡し、その使用を追跡できます。

トランクのコールトレースは、2番目のストリームに分類されるコールが破棄され、その後に最初のストリームに送信された場合にのみ正常に通過することを示しています。

OutOfServiceが2番目のスレッドから応答を受信した場合、彼はそれを最初のスレッドに押し込もうとします。

内部番号の都市番号を変更して、何が起こっているのかを監視します。 これで状況が変わりました。最初の行を押すと、ビープ音が鳴り、2番目の行が鳴ります。すべて問題ありません。 疑念が生じ、それらを確認するために、フローを切断します。 1つ目はIncoming / Outgoing Group ID 701に関連付けられ、2つ目は702に関連付けられます。その後、自分で設定した都市番号に関係なく、すべてが非常に正しく機能しました。 理由は明らかです。オペレーターは、数値をストリームに明確にバインドしています。

オペレーターの簡単な再構成によってすべてが解決されます。彼は、2つの物理フローから60回線のトランクグループを1つ編成し、すべての番号をそこに転送します。

この問題はもう発生しません。

それはIP Officeを介した会議についてのすべてのように思えます。 そして本当にすべて。

ただし、このトピックの最後には、さらにいくつかの言葉があります。

IPOに加えて、私はもう1つの会議システムであるTelros Congressにも精通しています。 これは、E1を介してPBXに接続する自己ホスト型会議収集ソリューションです。 痛みがないわけではなく、インターフェイスは90年代のものであり、一部のコントロールには永続的な習慣が必要ですが、IPOには実際に欠けている多くの利点があり、それらが実装されない理由は完全に不明です。

1）参加者を管理し、ステータスを追跡する機能。

2）シンプレックスデュプレックス。 20人の会議でマイクがオンになっていると、特にスピーカーフォンがオンになっている場合、騒音、騒音、エコーなどの混乱が始まります。

3）会議を管理する機能。 完了する必要がある場合は、[会議の終了]をクリックするだけで、全員が受話器で電話を切ることができます。



このために、ローカル電話通信、市内への電話、会議の作成のニーズに対応するために、IP Officeの基本構成に関するマニュアルへのリンクを提供し、別れを告げます。
