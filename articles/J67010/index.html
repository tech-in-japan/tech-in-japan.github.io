WebタイポグラファーMuravyov Studioの新バージョン



少し前まで、Habréはすでに新しい印刷会社について話していました（ habrahabr.ru/blogs/typography/66710/を参照）。 私がこのトピックを書くことに成功する前にスピーチが来ました。 もちろん、この無料の製品に投入した力、時間、魂が誰かに役立ったことを非常に喜んでいます。 最初のバージョンは、フォーラムシステム用に2007年に作成されました（Webサイトの印刷セクションで読むことができます。ダウンロードすることもできます）。 最終的に、私たちはそれを独立した独立した製品として位置付けることにしました。そして、私はそれについて少しお話ししたいと思います。



解説



Web typographerは、印刷テキストのグラフィックデザインと、ロシア語の規範、規則、詳細を使用した修正を目的とするツールです。 これは、サイト上のテキスト情報の認識と所有者の自己規律を改善するのに役立ちます。 ウェブタイポグラファーの存在は、善人と悪人の間に類似点を引き、誇りと内なる調和の理由を与えます。



コンクリート化



現在、多くのスマートルールが含まれており、時々改善され、より思慮深くなります。 タイポグラファーは、古典的な法則（判読不能なスペース、正しい引用符、ぶら下げ句読点など）に従ってテキストを処理するだけでなく、「出版社および著者のハンドブック」A.ミルチンから引用した規範に従っても処理します。



バージョン2.0.0は、以前のものとは大きく異なります。 これは、以前に使用したものとは異なるタイプのテキスト入力メカニズムを実装しています。 これに関して、新しい用語が導入されました。 Toph（ "typographer"から）は、テキストを入力するための独立したユニットです。 6つのtofsを開発しました：

  1. ダッシュ-これには、ハイフンとダッシュの適切な配置に関するすべての作業が含まれます。
  2. 数字-数字のある操作とそれ以上の操作のセット。
  3. 句読点-句読点の処理：ドット、コンマ、感嘆符、疑問符など
  4. 引用-引用符;
  5. スペース-空白文字の正しい配置。
  6. など-以前のtofaに配置できなかったすべてがここに取得されます（たとえば、（tm）を商標コードに変換します）。


すべての操作とルールを分割するというアイデアは、最初に、以前に記述したタイポグラフィルールを調査するときに生まれました。これらはすべて、かなり小さなオブジェクトグループに対して操作を実行します。 第二に、タイピングプロセス自体がより複雑になり、以前に使用したルールメカニズムの機能が十分ではありません（以前は、表現したすべてのルールが正規表現を持つ複数のファイルでした）。
