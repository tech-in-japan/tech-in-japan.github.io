OTRSとActive Directory統合のインストール

現在、Habréではヘルプデスクシステムのレビューはほとんどありませんが、SaaS- helpdeskの比較レビュー（ dude1phoenixからのコメントへのリンクが良好）、 OTMS- helpdeskとRequest Trackerの概要（OTRSはPerlで書かれているように）があります。

OTRSカスタマーサポートシステムをよく見て、Debianにインストールし、エンタープライズActive Directoryと統合することをお勧めします。



誰もが少なくとも一度はカスタマーサポートシステムと処理アプリケーション（いわゆるヘルプデスク）を扱ったことがあると思います。 たとえば、ホスティングのテクニカルサポートと通信するとき、ドメインレジストラーまたはオンラインストアで問題を解決するとき。 サポートシステムを使用すると、質問の解決策がどのような状態であるか、誰が関与しているか、また通信の履歴を保存できます。 ある時点で、企業にそのようなシステムを導入することに決めるかもしれません。

既存のヘルプデスクシステムの中には、機能が有料のアナログと競合できるフリーウェアソリューションはそれほど多くありません。 内部Webサイトおよびメールとの統合に選択されたTrellis Desk製品は、ほとんどすべての人に適していましたが、数か月の作業の後、統計が不足し始めました-どの期間の問題が解決され、チケットがクローズされたか、何人、1か月にトラブルチケットが開かれたかなど.p。：





したがって、私たちが気に入ったOTRSサポートシステムをレビューから選択して、インストールを開始し、その後ディレクトリサービスと統合しました。

ウィキメディア財団、Mail.ruグループ、Alt Linux、REG.RUなどがこのシステムを使用していることは確かです。 OTRSは、 ITILコンプライアンスに関してPinkVERIFYエキスパート企業によって公式に認定されています。 OTRSは約4歳です。

Windowsシステム用の OTRS には、Apache、MySQL、PHP、Perl、FileZilla FTPサーバー、およびその他のコンポーネントが含まれます（合計約114 MB）。



MySQLデータベースを使用したDebian Squeezeのインストール手順を検討します。



FTPサーバーから最新バージョン（約20 MB）を取得します。 

curl http://ftp.otrs.org/pub/otrs/otrs-3.0.8.tar.gz | tar xz -C /opt && ln -s /opt/otrs-3.0.8 /opt/otrs


Perl、サーバーを配置します： 

 apt-get install libapache2-mod-perl2 libdbd-mysql-perl libtimedate-perl libnet-dns-perl libnet-ldap-perl libio-socket-ssl-perl libpdf-api2-perl libdbd-mysql-perl libsoap-lite-perl libgd-text-perl libtext-csv-xs-perl libjson-xs-perl libgd-graph-perl libapache-dbi-perl mysql-server


Apacheグループにユーザーを作成します。 

 useradd -d /opt/otrs/ -c 'OTRS user' otrs && usermod -G www-data otrs


構成をコピーし、ファイルのアクセス許可を設定します。 
 cd /opt/otrs/Kernel && cp Config.pm.dist Config.pm cp Config/GenericAgent.pm.dist Config/GenericAgent.pm cd /opt/otrs/bin/ ./otrs.SetPermissions.pl /opt/otrs-3.0.8/ --otrs-user=otrs --otrs-group=www-data --web-user=www-data --web-group=www-data cp /opt/otrs/scripts/apache2-httpd.include.conf /etc/apache2/conf.d/otrs.conf


Apacheファイルをリロードします。 
 /etc/init.d/apache2 reload


http：//localhost/otrs/installer.plのヘルプデスクインストーラーに移動します。

インストール後（非常に簡単なウィザードを使用）、 httpページにログインします：//localhost/otrs/index.pl （ディレクトリサービスログイン/パスワードとの統合前：root @ localhost / root）

OTRSでは、エージェントは従業員、OTRSシステムのオペレーターであり、顧客はサポートアプリケーションを作成するクライアント、ユーザーです。

システムをActive Directoryと統合するには、ドメイン内の任意のアカウントに対応する1人のエージェント（OTRS管理者）を作成する必要があります（少なくとも読み取り権限がある場合）。







構成ファイル/opt/otrs/Kernel/Config.pmに入力するときに、このエージェントが指定されます。

エンタープライズドメインのユーザーがクライアントとしてOTRSにログインするには、Config.pm構成ファイルをこのフォームに持ってきてください。 もちろん、Active Directoryの設定を指定することを忘れないでください。

「最初のバインドに失敗しました」エラーが/ var / log / syslogにスローされないようにするには、ADSIEditからotrs DSNを取得します。

また、フォーラムでよくあるエラーの1つは、Active Directoryアカウントのプロパティに電子メールアドレスを持たないアカウントでOTRSにログインしようとすることです。



次に、基本設定を行う必要があります。

[管理]-> [システム構成]に移動します。 左側のドロップダウンリストで「フレームワーク」を選択し、右側に表示されるリストで「フロントエンド::顧客」を選択します。







「CustomerPanelCreateAccount」「No」をポイントし、下部にある「更新」ボタンを押します。



通知書に挨拶を書く方法、企業の名前を書く方法などについては話しません。 など -すぐに、新しく作成されたチケットの電子メール通知の設定に進みましょう：
  1. 「管理」-「キューの設定」-「キューの追加」に進みます。

    ここにキューの名前（「IT部門」など）を入力し、少し下にグループ「admin」を配置して「送信」をクリックします。
  2. 「管理」-「エージェント」（上の図のように）に進み、表示されるリストでアカウントを選択します。
  3. それどころか、「新しいアプリケーションの通知」は「はい」を選択し、「マイキュー」で「IT部門」を選択し、 強調表示したままにします 。







  4. ここですべての変更を保存するには、下の[送信]ボタンをクリックします。
  5. 「管理」-「エージェント<->グループ」、リストからアカウントを選択し、「admin」行にすべてのチェックマークを付けて、「送信」をクリックします。
  6. OTRSレターを送信するための設定は、メインページ-[メール設定]-[PostMasterメールアカウント]にあります。 そこで「IT部門」行を指定します。
ユーザーはhttp：//localhost/otrs/customer.plで OTRSにアクセスします。

ログインして、IT部門の受信者用の新しいアプリケーションの作成を試みます。







すべてが正常である場合-アプリケーションを作成すると、通知が電子メールに送信されます。

そして最後-顧客のリクエストに応答できるように、「回答<->キュー」に進み、「空の回答」を選択して「IT部門」を確認します。 保存します。



もちろん、OTRSははるかに柔軟性があります。すべてを1つの小さな記事で説明するわけではありませんが、このアプリケーションシステムの機能のほとんどは、すべてのITスペシャリストではなく、たとえば企業自体のニーズに応じて個別に使用される可能性が最も高くなります

最後に、システムはAGPLの下でライセンスされていることを付け加えます。 また、 ロシア語のフォーラムでは、最も一般的な問題の解決策を見つけることができます。



OTRSのメイン「管理者」ページのスクリーンショット （160 Kb）-画像サイズによるリンクとして設計されています。



PSメインページ（管理パネルの横にある「ダイジェスト」ページ）に、更新に関するニュース、製品の新しいバージョンが表示されます（ただし、OTRSが公式サイトにアクセスできるように、プロキシ設定も「指定」します）。 更新する場合、ダウンロードされたアーカイブ内の「アップグレード」ファイルのステップバイステップの手順を、OTRSの新しいバージョンで使用するだけです。
