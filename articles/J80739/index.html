WordPress UI Evolution（2003-2009）

現在のWordPressは、コミュニティが授与した愛の産物です。 アイデア、希望、推奨事項、ユーザーのアドバイス、そしてこれをすべて実現させた開発チームの努力がなければ、現在の姿は明らかになりませんでした。 今日、WordPressは、ほとんどあらゆるニーズに適した強力な本格的なCMSを備えています。 この記事では、WordPressが現在のバージョンにどのように進化したかを示し、各バージョンのコントロールパネルの変更をフォローします。



WordPressが発売された2003年5月27日から、今日まで57のリリースがリリースされました。 この記事では、ユーザーインターフェイスに関する最も重要で重要な変更のみを表示しようとします。 リリース自体は公式のWordPressサイトで確認できます。ここでは、その一部をダウンロードすることもできます。



WordPress 0.71-ゴールド（2003年5月）







デフォルトで開かれたページは、同時に新しい投稿を書くためのページでもありました。 ご覧のとおり、コントロールパネル自体は存在しませんでした。 機能は非常に限られており、一般的にすべてが非常に簡単でした。 許可されているのは、投稿にカテゴリを追加することだけです。 面倒なインストールプロセスでは、多くのことを手動で入力する必要がありました。



WordPress 1.0.1-マイル（2004年1月）







このバージョンから、重要なWordPressリリースは有名なジャズミュージシャンにちなんで命名され始めました。 たとえば、これはMiles Davisにちなんで命名されました。 このバージョンでは、WordPressはプロトタイプb2のファイル構造の使用を中止し、独自の開発を開始します。 投稿にいくつかのカテゴリを追加できるようになりました。 また、mod_rewriteを使用してCNCを選択できるようになりました。 インストーラーは、以前のバージョンに比べて大幅に簡素化されています。 「このページの編集/コメント」リンクが最初に導入されました。 コメントを管理する機能も追加されました。



WordPress 1.2-Mingus（2004年5月）







リリースはチャールズミンガスにちなんで命名されました。 このバージョンで最も重要な更新の1つはプラグインでした。 この問題で初めてWordPressプラグインのアーキテクチャが導入されましたが、今日まで変更されていません。 また、このバージョンの重要な改善点には、サブカテゴリ、カスタムフィールド、サムネイルの作成、エントリのプレビュー、暗号化されたパスワード、一度に複数のサービスへのping機能があります。



WordPress 1.5-Strayhorn（2005年2月）







WordPressのこのバージョンは、 Billy Strayhornにちなんで命名されました。 ここに、「コントロールパネル（コンソール）」の最初のヒントが表示されました。 現在、WordPressは投稿で構成されているだけでなく、ページを持つこともできます。 さらに、テーマのサポートが追加されました。



WordPress 2.0-デューク（2005年12月）







Duke Ellingtonにちなんで名付けられたこのバージョンは、ユーザーインターフェイスの重要な変更に最初に気付いたバージョンです。 管理パネルに大きな青いヘッダーが表示されましたが、これは後で廃止されます。 これは、完全なWYSIWIGエディターが使用された最初のバージョンです。 また、このリリースでは、Akismet Spam Protectionプラグインが導入されました。 その他の革新：画像とファイルのダウンローダー、thumbnail.pngを使用したテンプレートプレビュー、AJAXを使用した管理機能の改善、公開速度の改善、開発者向けの多くの新機能。



WordPress 2.1-エラ（2007年1月）











今回のリリースは、 エラ・フィッツジェラルドにちなんで命名されました。 WordPressには、このバージョンの多くの新機能があります。 最も注目すべきは、迅速な作業のためのコントロールパネルインターフェイス、スペルチェック、および独自のコメントメニューです。



WordPress 2.3-デクスター（2007年9月）







WordPress 2.3は、 Dexter Gordonにちなんで命名されました。 このリリースでは、セキュリティと安定性の向上に重点が置かれました。 また、コントロールパネルにいくつかの変更がありました。 ユーザーはWordPressにタグを追加できるようになりました。 プラグインまたはプラットフォーム自体のバージョンが古くなったときに、更新に関する通知が追加されました。 また、WordPressは、wwwのあるサイトやwwwのないサイトなど、ユーザーのリクエストに応じてユーザーを目的のURLにリダイレクトする方法を学びました。



WordPress 2.5-ブレッカー（2008年3月）











バージョン2.4を飛び越えて、2.5には計画された変更の2倍の数があるという事実を正当化するため、開発者はMichael Breckerにちなんで新しいリリースに名前を付けました。 まったく新しいコントロールパネルインターフェイスが表示されます。 コンソールは多くの有用な情報を取得し、はるかに便利になりました。 これは、WordPress.orgにあるプラグインをワンクリックで更新できる最初のバージョンでした。 このリリースには、完全に改善されたビジュアルエディターと組み込みのギャラリーも含まれています。



WordPress 2.6-Tyner（2008年7月）







マッコイタイナーは、このバージョンが命名されたジャズ奏者です。 開発者は投稿編集ページに単語数を追加し、Google Gearsを使用できるTurboも追加しました。



WordPress 2.7-Coltrane（2008年12月）











このバージョンは、 John Coltraneにちなんで命名されました。 コンソールインターフェースのもう1つの大きな再設計が見られます。これは、ほとんどのコミュニティに歓迎されました。 コントロールパネルの生産性が向上し、いくつかの設定が含まれるようになりました。 さらに、管理パネルから直接プラグインをインストールする機会を得ました。 また、コントロールパネルは、コンソールからのコメントへの応答、ツリーのようなコメント、添付された投稿、ホットキー、コメントのページネーションなどの革新を獲得しました。



WordPress 2.8-ベイカー（2009年6月）







このバージョンは、 チェット・ベイカーにちなんで命名されました。 コントロールパネルにはいくつかの便利な設定があり、3列に対応できるようになりました。 よく見ると、インターフェースで実際に何も変わっていないことがわかりますが、明らかにすべてがずっと速くなっています。



WordPress 2.9-カルメン（2009年12月）



このバージョンは、 Carmen McRaeにちなんで命名されています。 ここにスクリーンショットがない理由は、まだ更新していない場合は、すぐに更新してほしいからです。 WordPress 2.9のリリース直後に、このバージョンの主な利点の概要を記載した資料をHabréに公開しました。 必ずバージョンを更新する必要があります。



結論にいくつかの言葉



振り返ってみると、WordPressが現在の状態になる前に長い道のりを歩んできました。 多くの成功した変更がエンジン内部で行われ、多くの改善が必要です。 しかし、WordPress開発チームは、コミュニティの多大な支援なくして行われた作業の半分を実装できませんでした。 また、 ベータリリースリストに参加して、このすばらしいCMSの新しいバージョンのベータテストに参加し、バグトラックを送信して改善のためのアイデアを提案することで、WordPressの成長と改善を支援することができます。



これらの資料は 、独自の観察を備えた、 5年間のWordPressインターフェースとWordPressユーザーインターフェースの進化（2003〜2009年）に基づいています。 どのブログに「ユーザーインターフェース」または「WordPress」を追加するかを決定できませんでした。後者は関連性を上回りました。
