ニューイヤーショッピング：VMwareがPLUMgrid SDN Startupを買収

VMwareは新年前に購入しました。 仮想化の巨人が、ソフトウェア定義のネットワークテクノロジーに携わっているスタートアップPLUMgridを買収したことが知られるようになりました。



PLUMgridは、インフラストラクチャを流れるトラフィックの増加に企業が対処できるように、シスコシステムズによって2011年に設立されました。 開発チームは、ハードウェア拡張プロセスを簡素化しながらネットワーク操作を自動化する管理システムを作成しました。





/ Flickr / ブルーノ・コルディオーリ / CC



PLUMgridプラットフォームのコンポーネントの1つはIO Visorテクノロジーです。これは、スタートアップが昨年The Linux Foundationに引き継いだものです。 高価な機器の代わりに従来のサーバーを使用して、パケットスイッチングやその他のネットワークタスクを実行できます。 さらに、新機能の導入にはサーバーの再起動は不要です。これにより、環境の要件が変化した場合に、顧客は動的モードでハードウェアを再プログラミングできます。



「プログラマビリティは、新しい機能を追加し、ネットワークの透明性を高めるための鍵です」 と 、ベアフットネットワークスのCEOであるMartin Izzard氏は述べています。


IO Visorは、Linuxカーネル仮想化の問題を解決します。 メモリやCPUなどのハードウェアと直接「通信」するため、カーネルはユーザー空間よりも高いパフォーマンスを提供できます。 カーネルに適用される非仮想化という用語は、一度に1つの要求しか処理できないことを意味します。 新しいリクエスト、つまり新しいIOモジュールを処理するには、カーネルを再コンパイルする必要があります。



IO Visorは、仮想マシンの追加を簡素化します。 これは、パケットデータをフィルタリングするBerkeley Packet Filter（BPF）のおかげです。 ネットワーク機能仮想化（NFV）の概念により、単一のコアで複数の仮想ネットワーク機能（VNF）を実行できるようになりました。





ソース： TechCrunch



IO Visorは、分散監視およびアプリケーション追跡システムを構築するための優れたプラットフォームでもあります。 インターフェイスとトラフィックのステータスをリアルタイムで監視できます。 その後、開発者はこのデータに基づいてアプリケーションを構築し、「インフラストラクチャの健全性」を反映したグラフに数値を変換できます。



このソリューションは、セキュリティ機能も提供します。 カーネルを再構築することなく自動的にカーネルにロードする仮想ファイアウォールと侵入防止システムを想像してください。 「このソリューションにより、エンジニアと管理者はシステムをリアルタイムで操作し、開始-停止-再起動のパラダイムを忘れることができます」 と 、シスコのソフトウェア戦略実装のシニアディレクターであるローレンクーニーは述べています。



したがって、IO Visorを使用すると、カーネルがすべてのデータ処理を実行する複数のコンピューティングノードに分散された完全な仮想ネットワークを構築できます。 GitHubのリポジトリでIO Visor Projectソリューションを見つけて個別に学習できます。



VMwareはIO Visorがオープンライセンスで配布されるため、IO Visorに対する権利を受け取らないことに注意してください。 ただし、同社は他のPLUMgridプラットフォームを買収しています。 Open Networking Suiteソリューションは、OpenStack環境の管理を簡素化することを約束し、他のシステムであるContainer Networking Suiteは、コンテナ化されたアプリケーションに対して同様の機能を提供します。



最後に、PLUMgridはCloudApex監視ツールを開発していました。これにより、組織はリアルタイムでネットワークステータスを監視できます。 ソリューションのグラフィカルシェルには使用されているリソースが表示され、組み込みの分析機能を使用して問題をすばやく特定できます。 VMwareは、PLUMgridソリューションにより、NSX SDNプラットフォームの改善を望んでいます。 同時に、仮想化の巨人は 、PLUMgridの従業員が引き続き彼らの頭脳に取り組むことを指摘しています。
