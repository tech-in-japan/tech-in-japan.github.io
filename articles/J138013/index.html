Arduino LCDインフォーマー

Arduinoのような趣味に興味を持つようになり、すぐにこの素晴らしいプロジェクトの本質は、回路、はんだ付け、超小型回路については何も理解していないが、プログラミングについては少ししか理解していない人たちが、パーソナルコンピューター以外の何かを作れるようにすることであることに気付きました。 プログラマーやこのトピックに関する特別な知識なしでマイクロコントローラーをフラッシュします。



したがって、このプロジェクトの推進を支援したいと思います-プログラマー、これはあなたにとって素晴らしいエンターテイメントです！



私の次のモバイルプロジェクトのモーターは、ロシアポストの3つの最速の腹足類の馬を利用したワゴンで走行している間に、「Arduino LCD Informer」と呼ばれる別の（ウォームアップ）プロジェクトを実施しました。 このプロジェクトの本質は、Arduinoがコンピューターから受信したLCDデータを表示することです。 コンピューターでは、それらは別のプログラムによって収集されて送信されます。

ロシア語




すでに述べたように、私の画面はロシアで作られています。 この点で、2つのコードページがあります-最初の（No. 0）にはキリル文字が個別に含まれており、ラテンアルファベットには類似物がありません。 2番目（No. 1）には完全なロシア語のアルファベットがあり、文字のコードをcp-1251と一致させます。 したがって、必要に応じて、特別な費用なしで画面にロシア語の文字を表示できます。

私が遭遇した小さなマイナス-ページ1には度のアイコンはありません（ページ0とは異なります）。 アポストロフィに置き換えられます。

画面を制御するために、一般的な画面モデル用に設計されたLiquidCrystalライブラリが使用されます。 しかし、もちろん、2つの言語をサポートするというブルジョワの悲しみは知らないので、コードページを切り替えることは誰もできませんでした。 ライブラリに簡単な変更を加えましょう。



％arduino_install_path％\ arduino-1.0 \ libraries \ LiquidCrystalにあります

「モジュールコマンド」セクションの画面のドキュメントによると、コードページは2番目のビットで関数setコマンドに設定されます。

新しい定数をLiquidCrystal.hに追加します。



#define LCD_CODEPAGE 0x02







次に、LiquidCrystal.cppで関数void LiquidCrystal :: init（）を見つけ、次のようにこの定数を残りの設定に追加します。

if (fourbitmode)

_displayfunction = LCD_4BITMODE | LCD_1LINE | LCD_5x8DOTS | LCD_CODEPAGE;

else

_displayfunction = LCD_8BITMODE | LCD_1LINE | LCD_5x8DOTS | LCD_CODEPAGE;









したがって、2番目のビットが発生し、画面はコードページ1を使用し、cp-1251エンコードでデータをロシア語で送信できます（私の場合、これは月の名前です）。
