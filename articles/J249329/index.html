Global Game Jam 2015の一環として48時間でゲームを作成した方法

ロシアの黒海沿岸の皆さん、こんにちは！ 私の名前はアレクサンダーで、モバイルアプリケーションとゲームを開発する小さな居心地の良いスタジオを率いています。 フレンドリーチームは、2015年1月23日から25日にクラスノダールで開催されたGlobal Game Jam 2015にほぼ完全に参加しました。 カットの下で-あなたは2日間で長年にわたって望んでいたことを行う方法についての物語。







暖かい金曜日の午後、私たちの冒険が始まりました。 コーヒー、デバイス、気分を戦略的に供給し、地域の首都クラスノダールに行き、午後5時までにGGJ'15の会場であるCOプレイスコワーキングセンターに行きました。 GGJは、チームが48時間ノンストップでさまざまなゲームを作成できる、毎年恒例のグローバルハッカソンです。 その後、プロジェクトは地元の聴衆に提示され、誰でもゲームをダウンロードして実行できるようにネットワークに配置されます。





4つの惨劇パッケージ、8缶のアドレナリン、3パックのクッキー、4リットルのジュース、さまざまなデバイスのフルサロンがありました。 また、ケーキ。 それが旅行に必要なものではなかったのではなく、店に行ったので止めることは困難でした。


コワーキングに到着すると、デバイスをレイアウトし、トピックの発表を待ち始めました。 毎年、GGJの主催者は、ゲームを作成するテーマを公開します-原則として、テーマは抽象的に選択され、コースを設定するだけです-ファンタジーの飛行を制限しません。 メイントピックに加えて、追加のサブトピック、たとえば実績などが発表されます。たとえば、ボタンが1つしかないゲーム（これはStephen Hawkingがプレイできます）またはフォークロア要素が含まれるゲームです。



CASE-Mobileチームについて一言。 私たちは緊密で親しみやすいチームで、私はプロジェクトマネージャーであり、サウンドを担当していました。 IgorとYegor-Unity 3Dのクールなコード、およびTyomaはゲームプレイを磨き、プロトタイプをテストしました。 コワーキングでは、アーティストとアニメーターのMaxが参加しました。彼は私たちを大いに助けてくれました-ほとんどすべてのグラフィックは彼によって描かれました。 リモートでは、UI / UXデザイナーのCyrilが、インターフェイスとスプラッシュスクリーンの描画を支援しました。





トピックを待っています。 写真内のすべての一致はランダムです。


19時、ハッカソンの参加者はビジュアルゾーンに集まり、トピックの発表を待ちました。 20分間のビデオが公開され、数人の人々が現代のゲーム業界、問題、現在の西洋の社会的基盤についての理解を表明しました。 性的および性的平等、人種差別などなど...序曲の後、今年は「今、私たちは何をしますか？」というトピックが発表されました。聴衆は完全に沈黙しました。 「これをどのようにトピックと呼ぶことができますか？」参加者の一人の最初の反応です。 まあ、私は選択する必要がなかったので、ブレーンストーミングを始めました。



20分間の白熱した議論の後、次のアイデアがありました。

  1. 人々の集まり（私たちは薬物に対して「NO」と言います！）はプラットフォーマーです。 ボールは人々の周りを転がり、それ自体で社会を集めます。 ボールは戦争や腺ペストという形で障害を克服する必要があります。
  2. 感情を作成する-ロジック。 感情は画面に表示されます。プレーヤーのタスクは、使用可能なオブジェクトから生成することです（たとえば、ナイフと花の画像の画面上。感情が苦しみとして定義されている場合、ナイフを花に移動する必要があります）。
  3. 現在および将来-論理、アーケード。 画面は2つの部分に分かれています。1つはすべてが現在形で、もう1つは未来です。
  4. 隣人-タパスとスバイパミが日常の問題を解決するのを助ける必要がある隣人が家に現れます。






ブレーンストーミング後のアイデアのリスト。


その結果、4番目のオプションを決定しました。 時間の観点から、ゲームプレイとリアルの開発において、隣人は私たちにとって最も楽しいようでした。 「のための」重要な議論は、ゲームのコンセプトが非常にシンプルであると同時に高い再生可能性を意味するということでした。 これはタッチスクリーンに関連付けられており、モバイルデバイス向けのゲームを専門としているため、これに慣れています。

プロジェクトのテーマが発表された後（合計5個が発表されました）、参加者はゲームの作成を開始しました...



チームは、ゲームを作成する魔法のプロセスに真剣に取り組みました。 私たちは力を合わせてゲームのコンセプトを完成させ、ゲームプレイを研究するためにそれをTyomaに渡しました。 その後、ワークフローの整理を開始し、Maxが最初のスケッチを描き始め、イゴールとイェーガーが再シャーパーでUnityを起動し、シーンとスクリプトの準備を始めました。





キャラクターの最初のアウトライン。


数時間後、ゲームプレイ、キャラクター、およびゲームのすべての主要機能の説明を含むdzdokが準備されました。 したがって、アニメーション/動作のパッシブ/アクティブ/フェイルモードの10人のキャラクター、登場する家、キャラクターごとのユニークなテープとスワイプ、スコアリングシステム、およびすべてのアクションとオブジェクトのサウンドがあります。





デザイン用紙の準備ができました。


最終的なゲームプレイは次のようになりました。隣人のいる部屋が家の中にランダムに表示されます。 各部屋の隣人はユニークで、特定のアクションがあり、それを継続的に実行します。 あなたが与えられたジェスチャーで彼を助けなければ、隣人は失敗状態になり、ゲームは終了します。 全体の難しさは、隣人の数が絶えず増加しているという事実にあります-プレイヤーは多くのオブジェクトを視界に保ち、隣人が失敗するのを防ぐ必要があります。 彼はより長く続きました-より多くのポイントを得ました。 負けた後、プレイヤーが結果に満足するか、単に退屈するまで、サイクルが繰り返されます。



当初、隣人の間に論理的な接続を導入する予定でした（たとえば、修理工が配線を閉じて、別の隣人を電気なしで放置した）が、締め切りが厳しかったため、この考えを放棄しました。 彼らはまた、家の中の隣人の数を10人から6人に減らしました。これにより、タブレットで、そして大きなスマートフォンでも許容範囲でプレイできるようになりました。 私たちの能力を適切に評価することで、時間通りにゲームをパスすることができました。





私たちのアーティストMaxは創造的な衝動に駆られています。


外は暗くなり、冬の日はとてつもなく青くなりましたが、仕事は一瞬止まりませんでした。 各チームメンバーは自分のビジネスで忙しかった。 そのため、トピックの発表から5時間後に、手押し車でのジェスチャーを考慮して、イゴールが準備した最初のプロトタイプを準備しました。





プロトタイプでゲームプレイを試みます。


プロトタイプは、6つの赤い長方形が配置されたブルースクリーンでした。 ユニークなタップまたは長方形へのスワイプを行うと、彼は色を変えました。 数時間後、プロトタイプは隣人の状態を示すストリップで準備ができていました。 そして朝の3時近く、マックスは6部屋すべてを終えて寝ました。





6つの部屋はほぼ準備ができています！


おはようは起きず、8：00にチーム全員が目覚まし時計で起きました。 徐々に意識を取り戻し、私たちはゲームに取り組み続けました。 正午に向けて、Yegorはメインシーンとカメラを準備し、Maxは最初のキャラクター（プログラマー）を描いてアニメーション化しました。



文字について少し。 そのうち6つがあります：プログラマーがキーボードで入力している、金髪の人が入浴している、猫がソファを引っ掻いている、穴あけ人が壁を掘っている、アスリートが胸から揺れている、アルコール依存症（どう思いますか？） 各キャラクターは、プレイヤーが隣人に注意を払わないとき、そしてプレイヤーが隣人を助けるとき、受動的な状態を持っています。 隣人が失敗した場合、各キャラクターに固有の楽しいスクリーンセーバーが表示されます。 キャラクターは私たちのチームに少し似ていて、面白くて活気がありました。 たとえば、アルコール依存症は、Tyomaから遠く離れているように見えます。





コンセプトアルカシャ。


土曜日には、コワーキングにかなりの数の人々がいました、明らかに誰もが家に帰りました。 私たちは海外旅行をしないと考えられていたので、居心地の良いCOプレイスは48時間、2番目の家になりました。 その間に、Artyomが持ち込んだけん玉を柔らかいソファで休みました。 その後、少女マックスが私たちのところに来ました。彼女は近所の舞台撮影に参加していたため、元の衣装を着ていました。 しばらくして、彼女の到着がライブブロードキャストでキャプチャされたことを知り、これが地元のミーム「突然、妖精が私たちを訪ねた」につながった。 ちなみに、ハッカソン全体は絶えずインターネット上で放送されていました。そこでは、録画のすべてを見ることができます。





突然、妖精が訪れました。


夕方でした。 私たちはそれぞれ、適切な期限が近づいていることを知っていましたが、軍隊はすでに尽きていました。 私たちは非常に愚かでした-感情的および肉体的疲労が感じられました-しかし、私たちはゲームに取り組み続けました。 弱虫のために眠る！ 昨夜、アニメーションを持つすべてのキャラクターの準備が整い、ゲームに統合されました。 プロトタイプはほとんどリリースバージョンのように見えました。 満足しましたが、ひどく疲れていたので、明日が終わると予想して寝ました。





コーディングの最初の30時間後のイゴール。


早朝、コワーキングのオーナーであるスタニスラフの目覚まし時計に目が覚めました。 数分、無限に長いピアス呼び出しが、私たちの内部埋蔵の残骸を活性化することができました。 もちろん、力はほとんど残っていません。 ガラスのような外観とあいまいなスピーチが与えられました。 数杯のコーヒーが私たちに数時間の爽快なバフを与えてくれたので、スケジュールを完了し、ゲームプレイを磨きました。 インターフェースとポイントシステムを統合するには、さらに数時間（alas！Buffs-短命）が必要でした。





ゴールドのルーミー！


15モスクワ時間までに、ゲームは完全に準備が整いました。 私たちは彼女のルーミーと呼びました-「ルームメイト」、それはクールに聞こえるようです。 前述したように、ソフトウェア開発はUnity 3Dで実行されたため、短時間で完全に機能するマルチプラットフォームプロジェクトを作成できました。 Androidがメインプラットフォームとして選択されたのは、プロビジョニング、UDID、その他の合併症を伴うhemoを伴わずにゲームを配布する方がはるかに簡単だからです。 globalgamejam.orgにプロジェクトをアップロードした後、コワーキングでのプロジェクトの最終的なプレゼンテーションを待ち始めました。





48時間の仕事の遅れ。 マックス、ミー、イゴール、イゴール、タイマ。


最終的には、発表された5つのプロジェクトのうち4つが準備されました。 コワーキングに携わる人の数が大幅に増え、誰もがプロジェクトのプレゼンテーションを見に来ました。 プレゼンテーションは2番目に続き、Androidの画面イメージをテレビにブロードキャストすることに関連する小さな問題により遅れました。 プレゼンテーションの中で、私たちは私たちのゲームについて話しました：ゲームのアイデアがどのように生まれ、48時間でゼロから実装されたか。 ゲームの実際のバージョンがAndroidタブレットで表示され、iPadでもプロトタイプが表示されました。 すべてのプロジェクトの中で、私たちのゲームは最もプレイしやすく、最終製品のように見えましたが、デモや最初のプロトタイプのようには見えませんでした。 もちろん、結果は理想とはほど遠いですが、バージョン1.0ではかなりまともです。



プレゼンテーションの後、ほとんどの場合、コワーキングは最も近いバーに行き、そこで過去のハッカソンに関する印象やアイデアを交換しました。 Global Game Jamは、コンピューターゲームへの情熱を持って人々を結び付ける、本当にクールなイベントです。 しばらくして、私たちは皆に別れを告げて家に帰り、冒険を無事に終えました。





プロジェクトのプレゼンテーション。


結論として、以下を追加したいと思います。 常に自分のゲームを作成することを夢見ているなら、Global Game Jamがその大きな理由です。 最小限の知識と能力で、誰もがゲームの作成プロセスに参加でき、フォーチュン夫人とコーヒーメーカーの支援を受けて、プロジェクトを完成した形にすることができます。 デザイナー、プログラマー、テスター、ライター、イラストレーター、イデオロギーのインスピレーション-それは重要ではありません。 GGJに来て、あなたの夢をかなえましょう。 ゲームを作成したい！



GGJの将来のメンバーへの友好的なアドバイスは、私の記事を読んだ後に増えることを願っています。 ハッカソンの魅力は、形成されたチームのランダム性にあると多くの人が言いますが、結果を達成したい場合、つまり本格的なゲームを作成したい場合は、共同開発の経験が最小限の緊密なチームに参加すべきだと思います。 あなたが孤独である場合-それは問題ではありません。アイデアがあなたにとって最も興味深いと思われる既製のチームを探してください。 マラソンの48時間を精神的に準備し、すべてを途中でやめる必要はありません。 初期段階の批判的思考と合理的な開発戦略-これが勝利の鍵です。 ルールに従ってください：最初に最低限、次に-時間があること、そしてあなたが成功すること。 がんばれ！
