Webサイトの設計の重要な段階の1つは、エンドユーザーがWebサイトを操作する方法の分析と改良です。 テンプレートを作成し、ソフトウェアモジュールを開発して統合し、アクセス権を配布してサイトを立ち上げることは、戦いの半分に過ぎず、最後の1つです。

このサイトは、エンドユーザーが最も便利に使用できるように設計および編成する必要があります。 要するに-これはユーザビリティと呼ばれます（ロシア語では使いやすさと作業（UIR））。



次に、Webサイトの最適化の側面の1つ、つまりサイトのコンテンツエディターの簡素化と高速化について説明します。





サイトが開発中の場合は、サイトデザインの設計段階で既に使用されているスタイルを2つのグループに分類する必要があります。

1.サイトのデザインで使用されるスタイル。

テキストとページタイトル、描画スタイル、インデント、配置オプションのこのデザイン。 これらの要素は永続的であり、めったに変更されません（完全な再設計の場合を除く）。 このような要素を備えたコンテンツエディターは非常にまれです。 ここでは、デザイナーの創造的な想像力を最大限に活用できます。

2.材料に使用されるスタイル。

タイトル、発表、ニュースの全文、要約、メモ、テキスト選択など。 つまり、コンテンツエディターは常に作業を行っています。 ここでは、すべての設計の改良に非常に批判的にアプローチする必要があります。標準フォントを使用する、すべてのブラウザーで機能しない特殊効果を放棄するなど。

ただし、サイトが既に存在する場合でも、既存のスタイルを並べ替えることができます。



最初のグループには、一意のスタイル名が指定されており、原則としてコンテンツエディターで使用することはできません。

2番目のグループでは、htmlタグのスタイルが再定義されます（段落、リストなど）。 すべてのhtmlヘッダータグ（h2、h3、h4）が使用されていない場合、要約、メモなどを強調するために使用できます。 新しいクラスを作成することもできますが、できる限り少なくし、クラスなしではできない場合に限ります。 たとえば、同じ要約、メモなどの場合、最初のグループにhtmlタグをオーバーライドしないでください。すべてのクラスに一意の名前を付ける必要があります。そうしないと、2番目のグループのhtmlタグの場合、最初のグループのスタイルがクロールできます設計オプションが変更されます。



2つのグループに分かれたCSSスタイルシートファイルの例：

-/ *デザインで使用されるスタイル* /

.headline {

padding-left：20px;

右詰め：20px;

}

…………………。

.menu {

マージン左：30px;

マージン上部：30px;

}

/ *ここでは、すべてのクラスに固有の名前が付けられていることに注意してください。 このセクションのHTMLタグは再定義されていません。 * /



/ *コンテンツエディターのスタイル* /

p {

フォントファミリー：Verdana、Arial、Helvetica;

フォントサイズ：大;

}

…………………。

ol {

フォントの太さ：太字。

色：＃CCFF00;

}

…………………。

.attention {

色：＃FF0000;

フォントの太さ：太字。

}

/ *ここでhtmlタグが再定義されることに注意してください。 * /

-さらに、例として、Bitrix CMSシステムを使用します。 以下は、ビジュアルエディタでデフォルトで使用される基本的なhtmlタグです。

1. h1、h2など -タイトル。

2. p、i、strong、u-アナウンスメントの段落と全文、およびその中のテキストの強調表示。

3. a-リンク用。

4. ol、ul、li-番号付きリストと番号なしリスト。

5. table、tr、td-テーブルを使用する場合。

6.など

まず、サイトのデザインに合わせてこれらのhtmlタグのスタイルを再定義します。 また、ノート、アブストラクトなどの2つのグループの新しいスタイルをビジュアルエディターのスタイルのドロップダウンリストに接続します（「1C-Bitrix」の基本トレーニングコースに合格していることを前提としています）。



結論：コンテンツエディターの作業は大幅に簡素化されます。 彼は、クリップボードからテキストを貼り付けて「ワードガベージ」から削除するか、ゼロから入力するだけで十分です。 次に、テキストに目を通し、接続されたスタイルを使用して、リスト、リンク、ハイライトのメモまたは要約をあちこちに貼り付けます。 フォーマットは自動的に適用されます。



このアプローチにより、標準のhtmlタグを含む比較的「クリーン」なコンテンツが得られ、スタイルが最小限に「散らばる​​」だけで、ほとんどの場合、「クリーン」なしで新しいデザインに統合したり、別のCMSエンジンに転送できます。



同様に、上記のすべてを他のCMSシステムに適用できます。 既存のビジュアルエディタで使用されているhtmlタグを確認し、CMS css-stylesファイルでそれらを再定義し、コンテンツエディタの追加スタイルをどのように接続できるかを明確にする必要があります。
