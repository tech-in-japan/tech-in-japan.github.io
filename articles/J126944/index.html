原則として、WinSCPを使用してファイルシステムにアクセスできます。 これにより、マシン間でファイルをコピーできるようになります。 しかし、それはあまり正しくありません、なぜなら SSH認証後、システム全体へのアクセスが開かれます。 理想的には、サーバー上に1つのネットワークフォルダーを作成し、そのフォルダーにアクセスできるようにしたいだけです。 そして、これに対する解決策があります。



パッケージを置きます：



＃sudo apt-get install samba



ファイル/etc/samba/smb.confを編集します。 作業コンテンツの例を次に示します。 

 [global] workgroup = WORKGROUP server string = %h server dns proxy = no log file = /var/log/samba/log.%m max log size = 1000 syslog = 0 panic action = /usr/share/samba/panic-action %d security = share encrypt passwords = true passdb backend = tdbsam obey pam restrictions = yes unix password sync = yes passwd program = /usr/bin/passwd %u passwd chat = *Enter\snew\s*\spassword:* %n\n *Retype\snew\s*\spassword:* %n\n *password\supdated\ssuccessfully* . pam password change = yes [homes] comment = Home Directories browseable = no read only = yes create mask = 0700 directory mask = 0700 valid users = %S [printers] comment = All Printers browseable = no path = /var/spool/samba printable = yes guest ok = no read only = yes create mask = 0700 [print$] comment = Printer Drivers path = /var/lib/samba/printers browseable = yes read only = yes guest ok = no [user] path = /ftp/pub readonly = no guest ok = yes




コメントとして、次のことがわかります。

-Windowsマシンが動作するワーキンググループを指定する必要があります。

-セキュリティパラメータは共有に設定する必要があります。

-「ユーザー」セクションでは、実際のパスを指定してゲストアクセスを許可する必要があります。実際にこのセクションをファイルに追加します。



このすべての後、次のディレクトリを作成します。



＃mkdir / ftp

＃sudo chmod 0755 / ftp

この場合、ftpフォルダーへのアクセスが行われますが、変更はできません。 しかし、pubフォルダーで-何でもします。 当然、パパを捨ててftpの権限を調整することで、すべてを単純化できます...

手始めに、ネットワーク上のどこからでもネットワークファイルにアクセスして制御できるようにすると便利です。 しかし、トポロジには何がありますか？

静的な実IPアドレスでインターネットにアクセスできる必要があります。 つまり 外部からのアクセスは、プロバイダーのスイッチングネットワーク経由ではなく、直接実行する必要があります。 たとえば、Ramnetユーザーはすべて大きなNATに座っています。なぜなら、 いくつかのプールを介してオンラインになります。 これは、登録中に発行されたIPが、たとえばサイトにアクセスするときに「印刷」されるIPと一致しないことを意味します。 実際のIPは、間違いなくネットワークアダプタードライバーのネットワーク設定で指定されたIPです。 実際のIPを確認するために、IPアドレスを表示するインターネット上のネットワークサービスを使用できます。 実際のIPがあるとしましょう。

さらに、ネットワークワイヤがサーバーに直接接続されている場合、既に特別な問題はありません。サーバーは既にネットワーク上にあります。 また、ルーティングデバイス（ルーター、または同じLinuxベースの中間スプリッタースイッチ）がある場合はどうでしょうか。 この場合、NATの背後に既に存在しているため、少しポートフォワーディングを構成する必要があります。 マシン上の各ネットワークサービスにポートがあります：SSH-22、FTP-21。つまり、ルーターは、実際のIPを介してポートにアクセスするときに、このポートでサーバーにアクセスするように構成する必要があります。 サーバーのアドレスが10.0.0.10で、プロバイダーの実際のIPが172.122.34.89の場合、ルーターはポート（ポート転送）21および22を外部（172.122.34.89）からサーバー（10.0.0.10）に転送する必要があります。 ルーターの管理パネルで数行を設定するか、中間のLinuxスイッチのマーチングテーブルを編集するだけです。 みんなもやったとしましょう。



この場合、Puttyを介したSSHアクセスの場合、実際のIPアドレスであるポート22を指定するだけです（上記を参照してください。これは、小さなPuttyクライアントのセットアップについて説明しています）。 ログインしてサーバーコンソールにアクセスします。



そしてファイルアクセス？ ここは少し難しいです。 WinSCPを使用できますが、既に述べたように、これは私たちの方法ではありません。 SambaはLANでのみ動作します。 FTPアクセスを構成する必要があります...



FTPでは、proftpdサービスを使用します。 パッケージを置きます：

マジックパケットは、ローカルネットワークを正常に通過するためにUDPまたはIPXパケットに挿入できる特殊なバイトシーケンスです。 通常、WOLの場合、最上位のプロトコルパケットはブロードキャストされます。これは、動的なアドレス割り当ての場合、どのIPアドレスがどのMACアドレスに対応するかがわからないためです。 ただし、ブロードキャストパケットを禁止するルーターを正しく通過させるには、特定のアドレスにパケットを送信できます。
