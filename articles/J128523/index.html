Hawkersの要求に応じて：Red Hat Cloud、HP Editionについての回答

親愛なるハラジテリ！



私たちはますます多くの質問をされています（すでに2つまで、私たちはPMへの質問を直接考慮していません）。 しかし、すべてが可能な限り迅速かつ完全に回答できるわけではありません。 そのため、クラウドテクノロジーの分野におけるHPとRed Hatの協力に関する質問への回答を別の投稿に掲載することにしました。



したがって、Red Hat Cloud、HP Editionについて知りたいことはすべてあります！



5月、Red HatはHPとともに、リファレンスプライベートクラウドデザインアーキテクチャの特別版であるRed Hat Cloud、HP Editionをリリースしました。 このアーキテクチャでは、クラウドはRed Hat Cloud、HP CloudSystem、および特別なHP Cloud Mapテンプレートを中心に構築されています。 Red Hat Cloud、HP Editionの構成には以下が含まれます。

  • Red Hat Enterprise LinuxハイパーバイザーOS
  • Red Hat JBoss Enterprise Middleware
  • Red Hat RHN Satelliteアップグレードツール
  • HP BladeSystemを搭載したHP CloudSystem Matrixプライベートクラウドプラットフォーム
  • JBoss Enterprise Middleware用のHP Cloud Maps（これらの「マップ」は、HP CloudSystemのプライベートクラウドにJBossを展開する際にリソースのプロビジョニングを自動化するための事前定義されたテンプレートです）


標準のRed Hat Cloud Foundationsアーキテクチャとは対照的に、HP EditionはHP Matrix Operating Environmentソフトウェアを使用して、すべての物理リソース（サーバー、ストレージ、およびネットワーク機器）を展開します。これは個別の管理サーバーに展開されます。 同じ環境を使用して、ベアメタルサーバーへのLinuxの展開を自動化します（Red Hat Cloud Foundationsの標準アーキテクチャでは、サーバーにLinuxを手動でインストールする必要があります）。



HPサービス部門は、Red Hat Cloud、HP Editionのお客様にコンサルティング、調査、設計、トレーニングサービスを提供します。









Red Hat WebサイトでRed Hat Cloud、HP Editionのアーキテクチャの完全な説明、およびHP WebサイトでJBoss Enterprise MiddlewareのHP Cloud Mapテンプレートの説明を見つけることができます。

  • HP ProLiantおよびBladeSystemサーバー向けのRed Hat Enterprise Linux
  • HP ProLiantサーバーでのRed Hat Enterprise Linux 6の実装
  • 高可用性Linux環境のHP ProLiant DL980 G7サーバー
  • HP ProLiant DL980サーバー上のRed Hat Enterprise Linux
