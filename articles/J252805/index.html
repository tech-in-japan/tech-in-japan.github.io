ハードウェア共有DASまたはLSI Syncroが必要な理由





共有DASコンセプトに基づくソリューションは 、最近人気が高まっています。 説明は非常に簡単です。 中小企業（SMB）から中小企業のレベルまで、市場の幅広く非常にダイナミックなセクターからの需要がありました。 フォールトトレランスが低い従来のソリューションは既にそれらに適合していませんが、同時に、従来のSANデータストレージネットワークのコストは高すぎます。 つまり、共有DASソリューションは、複数のデータパスの使用と低い設備コストにより高い耐障害性を提供​​します。 素晴らしいボーナスは、SASがスイッチングメディアとして使用されるため、データ転送中のパフォーマンスが非常に高く、待ち時間が短いことです。 マイクロソフトは機関車として行動し、Windows Server 2012を提案しました。これには、OSレベルでのデータストレージの便利で便利な仮想化を提供するStorage Spacesコンポーネントが含まれていました。



しかし、Storage Spacesのパフォーマンスが十分ではないが、同時にプロジェクトサイズがそれほど大きくないので、完全なSANについて話すのが理にかなっている場合はどうすればよいでしょうか。 答えは、LSI Syncroを使用することです。



LSI Syncroとは何かを説明する前に、Storage Spacesの主な不満の原因を正確に定式化しましょう。



  • 記憶域スペースに含まれるドライブは、オペレーティングシステムの起動には使用できません
  • パリティを備えた仮想ディスク（それぞれ、よく知られているRAID5およびRAID6アレイに類似したパリティまたはデュアルパリティ）の仮想ディスクのパフォーマンスは、後者の実装の特性により低い
  • ソフトウェアレベルでのパリティ制御の実装により、プロセッサとサーバーメモリの負荷がわずかに増加します。




それでは、この記事の主人公に戻りましょう。

LSI Syncroは、互いに通信し、キャッシュ内のデータを同期できる2つのRAIDコントローラーのセットです。 それらのタスクは、接続されたドライブへのアクセスを2つのサーバーに一度に提供することです。 コントローラー自体のペアに加えて、配信パッケージにはすぐにいくつかのCacheVaultバッテリーが含まれます。そのタスクは、サーバーに電力が供給されない場合にコントローラーキャッシュ内のデータの安全性を確保することです。



多くの人は、コントローラーキャッシュ内のデータの一貫性を確保するために、専用のデータ伝送媒体ではなく、コントローラーとエキスパンダー間のメインデータストリームの転送にも使用される同じSASデータ伝送チャネルを使用しているという事実に戸惑っています。 しかし、しっかりと考えれば、Heartbeat専用の回線だけが単一障害点になる可能性があると結論付けることができるため、パスが重複しています。 チャネル共有に関しては、キャッシュ間のトラフィックはパフォーマンスに大きな影響を与えるほど大きくありません。





LSI Syncro内部





LSI Syncro外部



現在、LSI Syncro製品ファミリには4つの製品（9271-8i、9286-8e、9361-8i、および9380-8e）が含まれています。 最初の2セットは、ポート設計が異なるSAS 6Gソリューションです。9271-8iでは、各コントローラーに2つの内部miniSASポートがあり、9286-8eでは、2つの外部ポートがあります。 最近導入された9361-8iおよび9380-8eは、LSI Syncroファミリのさらなる発展を表しており、SAS 12Gインターフェイスへの移行においてそれらの前身とは異なります。 したがって、SAS 6Gがすでに時代遅れであるか生産性が不十分であり、12Gのみでインフラストラクチャを検討する準備ができていると考える場合、これに問題はありません。

このソリューションの可能性に関する詳細な議論を開始するには、明らかなWindows Server 2008および2012だけでなく、Red Hat Enterprise Linux 6.3および6.4、SuSE Linux Enterprise Server 11 SP2およびSP3、CentOS 6.4および6.5もサポートされているオペレーティングシステムとして宣言されます。 、パブリックドメインのサイトにあるドライバー。 ここに、オープンソース愛好者の幸福と機会があります。



LSI Syncroにはどのようなメリットがありますか？



性能



  • RAID 5/6のパリティアルゴリズムをプロセッサとコントローラのキャッシュに移動して、中央プロセッサとRAMをアンロードします
  • 電源障害保護とミラーリングを備えた、容量が大きく強力な書き込みキャッシュ




LSIが提供するデータを見ると、ディスクサブシステムパフォーマンスの分野でのLSI Syncroの機能（または優位性）は次のようになります。





LSI Syncroのパフォーマンスとストレージスペース



SAS 12Gインターフェースを備えた新世代で達成された最大のパフォーマンスについては、現在のファームウェア2.0-23.0.0-0055では、読み取り用にRAID0 8900 MB / sおよび書き込み用に4500 MB / sでSASディスクから15000 rpmを受信できます。 LSIは記録の問題について知っていますが、これは書き込みキャッシュ同期の実装が失敗したためであり、次のファームウェア2.2はこの問題を完全に解決し、書き込みパフォーマンスは同じ8900 MB / s（ディスク自体のパフォーマンス制限）であると言います。



信頼性



  • プラットフォームの独立性
  • サーバー障害に依存しないデータ
  • OSクラッシュ保護
  • すべてのデータパスを複製した高度に保護されたストレージシステムの簡単で便利な構築




追加機能



上記に加えて、パブリッククラスターディスクプールだけでなく、特定のサーバー専用のプールを作成する機能など、LSI Syncroの重要で便利な機能に注目する価値があります。 この機能は、MS SQL Server 2012に基づいてAlwaysOn可用性グループを構築する場合に重要です。



場合によっては、LSI Syncroに接続されたディスクからオペレーティングシステムを起動する機能が貴重な機会になることがわかります。



SASエクスパンダーはカスケードと自己構成の両方をサポートしているため、安価なディスクシェルフを追加するだけで、利用可能なドライブの一般的なプールを拡張できます。 さらに、製造業者の完全な選択の自由。

原則として停止までのスケーリングはどの程度現実的ですか？ これは問題ではありません：60枚のディスクシェルフを2つだけ使用すると、LSI Syncroの機能を完全に選択して、自由に96 x 6 TB = 576 TBのプール、つまり低コストSAS NLドライブの半分ペタバイトのrawディスク容量を取得できます。 もちろん、これは最新のSANや同じストレージスペースの機能よりも劣りますが、ターゲットオーディエンスの中には、この不十分なスケーラビリティと呼ぶことができる人はほとんどいないようです。



記事の冒頭で述べたストレージスペースの欠点に戻ると、LSI Syncroを使用することで、ほとんどの弱点を解消し、共有DASの一般的な機能範囲を拡大できることが明らかになります。



ユースケース





ETegro RS420 G4の例でLSI Syncro 9361-8i「ボックス内のクラスター」を使用する



最も明白な使用例は、LSI Syncroをクラスターインボックスソリューションの内部SASポートとともに使用することです。この場合、LSI Syncro RAIDコントローラーは、SAS HBAクラスターのノードの共有ストレージプールへのアクセスに使用されるアダプターを置き換えます。 このようにして、コンパクトでありながら同時に高性能でフォールトトレラントなソリューションを実現します。

小規模企業、リモートオフィス、または小規模企業のアクセスしやすく効率的なファイルサーバー、データベースサーバー、Webサーバー、強力なネットワークドライブの基盤、ビデオ処理サーバー、仮想環境ストレージサーバー-これらはすべて、迅速、簡単、安価に構築できますLSIシンクロ。





2台のRS530 G4サーバーとJS300 G3ディスクシェルフでLSI Syncro 9380-8eを使用する例

「ボックス化された」クラスターのコンピューティング機能が十分でない場合（十分なメモリーがないか、ノードの2つのプロセッサーのパフォーマンスが十分でない場合）、フルサイズのサーバーを使用でき、外部ポートでLSI Syncroをインストールする必要があります。 そして、すべてがシステムアーキテクトの想像力とSASエクスパンダー/シェルフのカスケードの可能性によってのみ制限されます。 範囲はボックス型クラスターの場合と同じですが、コンピューティングの負荷が高いだけでなく、データを分析および処理するための強力なシステム、高性能コンピューティングのシステムがあります。



要約すると、共有DASの概念は万能薬になり、すべての従来のデータストレージオプションを置き換えることを意図したものではなく、SASインターフェイスがイーサネット、ファイバーチャネル、およびInfiniBandを置き換えることはありません。 しかし、多くの場合、SASインターフェイスの機能が規模と範囲の点で十分である場合、LSI Syncroに基づく共有DASソリューションは、データを保存する最も収益性が高く、便利で効率的な方法です。
