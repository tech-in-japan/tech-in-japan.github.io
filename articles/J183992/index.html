OS WINDOWSのユーザーがフラッシュカードのルートにファイルを不正にコピーするのを防ぐ方法

誰もが邪悪な「オートラナ」の問題に直面していたと思います。 そして解決する方法はたくさんあります：オープンソースソフトウェア、スクリプト、「タンバリンと踊る」など。 Windows OSのユーザー向けに、サードパーティの介入を必要としないシンプルなソリューションアルゴリズムを提供したいと思います。 目標を達成するには、いくつかのステップ、より正確には2つのステップが必要です。



ステップ番号1-準備



OSを使用して、NTFSでメディアファイルシステムをフォーマットします。

書式設定オプションを選択するとき、NTFSはありません
NTFSでファイルシステムをフォーマットするオプションがない場合（Windows XPの場合）、次を実行します。

  1. フラッシュドライブのプロパティに移動します。
  2. [機器]タブに移動します。
  3. フラッシュカードを選択し、プロパティボタンをクリックします。
  4. [ポリシー]タブに移動します。
  5. キャッシュを使用するオプションを選択します。
  6. NTFSでの変更とフォーマットを受け入れます。
  7. その後、「クイック削除」に戻ります。


ステップ番号2-権利の差別化



ここで、ルートにフォルダー（「作業フォルダー」）を作成し、すぐに名前を設定します（将来フォルダーの名前を変更することはできません）。すべての情報が保存されます。 ただし、フォルダーのルートは読み取り専用になります。 次に、権利を区別します。

  1. セキュリティに移動-フラッシュドライブのプロパティ、[セキュリティ]タブを開きます。

    そのようなタブがない場合は、次を実行します
    1. 「コントロールパネル」の「フォルダオプション」に移動します。
    2. [詳細オプション]のリストの[表示]タブで、「共有」と「（推奨）」という単語を含むパラメーターを見つけます。
    3. Dawを削除します。


    これでタブが表示されました
  2. [詳細設定]ボタンをクリックします。
  3. 開いたウィンドウで[所有者]タブに移動し、[変更]ボタンをクリックします。

    XPの場合
    （Windows XPには「権限の変更」ボタンはありません。すぐに対応します）
  4. 「サブコンテナとオブジェクトの所有者を置き換える」ということを言います。
  5. 私たちは自分自身を選択し、オブジェクトの所有者になります。
  6. 「OK」ボタンを使用してOSの要求に応じてダイアログボックスを閉じ、「セキュリティ」と「追加」を再度開きます。
  7. 権限を変更します-これを行うには、セキュリティに移動し、[権限]タブで[権限の変更]ボタンをクリックして、既存の権限を削除します。
    XPの場合
    （Windows XPには「権限の変更」ボタンはありません。すぐに対応します）
  8. 「すべて」グループを追加します-「追加」、「詳細」、「検索」ボタンをクリックし、リストから「すべて」、「OK」、「OK」グループを選択します。
  9. 開いたウィンドウで、Everyoneグループの許可設定を設定します。

    • 「適用：このフォルダー、そのサブフォルダーおよびファイル」;
    • [権限]フィールドに次のドーズを設定します。
      1. 許可：「フルアクセス」。
      2. 拒否：「所有権の変更」、「権限の変更」、「削除」、「サブフォルダーとファイルの削除」、「追加属性の書き込み」、「属性の書き込み」、「フォルダーの作成/データのバックアップ」、「ファイルの作成/データの書き込み」
  10. 変更を受け入れ、フラッシュカードのルートにある以前に作成されたフォルダーに移動し、ほぼ同じ操作を行います。
  11. 所有権の変更-おなじみのパターンによる。
  12. 「許可」タブの「親オブジェクトから継承された許可を追加する」のチェックを外して、既存の権利を削除します。
  13. 表示されるダイアログボックスで、「削除」を選択します。
  14. Everyoneグループを追加し（既に方法を知っています）、権限設定を設定します。
    • 「適用：このフォルダー、そのサブフォルダーおよびファイル」;
    • チェックボックス許可を設定します：「フルアクセス」。
  15. 繰り返しますが、「Everyone」グループを追加し、許可パラメーターを設定します。
    • 「適用：このフォルダのみ」;
    • 「許可」フィールドに次の「拒否」チェックボックスを設定します。

      「所有権の変更」、「許可の変更」、「削除」、「追加属性の記録」、「属性の記録」


長所


+フラッシュカードのルートには、ファイルとフォルダーを作成/削除する方法がありません。 その結果、「自動実行」のための抜け穴が閉じられました。

+「working folder」フルアクセス-必要なものは何でも、それを裏返します（ストレスを自分でかけます）。

+ NFTSファイルシステム-個人用に最適です。 （私見）



短所


-[送信]メニュー項目を使用して情報をフラッシュドライブにコピーする方法はありません。 ルートディレクトリは書き込みのために閉じられます。 「作業フォルダ」にコピーする必要があります。

-「作業フォルダ」の名前を変更する方法はありません-読み取りのみがルートにあります。

-NTFSファイルシステム-FATのみを読み取るデバイス（音楽/ビデオプレーヤー、スマートフォン、写真カメラなど）には適していません。



備考
  • この操作は、「正常なマシン」で行うのが最適です。 そして、immortal-autorunを作成できます。
  • Windows XPで、「作業フォルダー」を削除しようとしたとき、それは残っていました（そうであるはずです）が、すべてが内部で削除されました。
  • ただし、Windows 7では、「作業フォルダー」を削除しようとしても、すべてのファイルが残ります。
  • 「作業フォルダ」内では、フォルダが開かれたときにフォルダの名前でexe'shnikを作成する悪意のあるプログラムからは保存されません。
