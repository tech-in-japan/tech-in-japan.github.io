AWS Insight：EC2の予約-仕組み

こんにちは



ご存知のように、クラウドサービスを使用するための基本的なルールの1つは、すべて、特に問題の財務面を最適化することです。 Amazon Web Servicesには、サービスのコストを最適化するための3つの主要な方法があります。

  • 自動スケーリング
  • 予約
  • スポットインスタンスの使用


今日は、EC2のバックアップについて、どのように、何を、なぜこれを行う必要があるかについてお話します。



予約はサーバー容量の前払いであり、これによりこれらの同じ容量のコストを大幅に削減できます。 予約には3つのタイプがあります。

  • 軽量化-軽量化
  • 中程度の使用率-中程度の使用率
  • 重度の使用-ハードな使用


予約は1年に1回または3年に1回購入されます。 予約が発生します：

  • 年次
  • 三歳


ビジネスケース



適切な予約を選択する基本原則：サーバーの実行時間が長いほど、予約を購入するのに費用がかかります。 実験してみましょう。 サイズがM1.XLargeのサーバーを1つ取り出して実行します。 サーバーが稼働する1日あたりの時間数に応じて、1年間と3年間にサーバーに支払う必要がある金額を計算できます。 したがって、今日の価格設定：

C-1時間の費用

結論


3年間、インスタンスが1日3時間まで開いている場合、 予約の使用は正当化されません 。 3から10まで -1日10時間後にLightを使用することをお勧めします-Hard Utilization Reservation 。



どうやら、MediumはLightよりも収益性が高い場合のどちらの場合でも平均的なオプションですが、メンテナンス時にはHard予約よりも安価です。 これが彼女の唯一の目的のようです。

重度使用予約を選択すると、インスタンスが機能しなくても、インスタンスの費用を支払うことになります。 年間の価格は直接となります。
