2015年は何年来ましたか？





すべてのハブロフスク市民の皆さん、明けましておめでとうございます、メリークリスマスをお祈りします！ モスクワのオフィスの窓の外では、冬が本格的です。温度計には水たまりが多く、+ 8があります。 今年は終わりに近づいており、在庫を確認する時が来ました。 2015年にITの世界で非常に興味深いと思われたものを思い出そうとすることにしました。 また、ITの世界は巨大であり、 すべてが非常に困難な作業であることを思い出し （シュワルツェネッガーにのみ匹敵する）、目を絞り、Cisco、HPE、Microsoft、VMwareに落ち着きました。 結局のところ（実際に疑う人は）、上記の会社のすべての新製品について話そうとすると、一生懸命努力する必要があります。 しかし、休日が来ています、そしてあなたは贈り物を買って、木を切って行って、オリビエの料理を始めなければなりません。 したがって、私たちの意見では、私たちとお客様にとって最も重要なものだけを選択しようとしました。 いずれにせよ、私たちはそう願っています。



だから、十分な歌詞、私たちはソリューション/デバイス/および選択した他の些細なことのトップをメーカー別に分けて始めます。





無線技術



今年、同社の主要なワイヤレスイノベーションは、何らかの形で802.11ac標準の開発を中心に展開しています。 特に、1850と1830の2つの新しいアクセスポイントが市場に導入されました。









これらのアクセスポイントは、802.11ac規格の開発のいわゆる第2波（波2）をサポートしています。 これらのアクセスポイントに現れた主な革新は、マルチユーザーMIMOのサポートと標準ビームフォーミング（802.11ac送信ビームフォーミング）のサポートでした。 思い出すように、このアクセスポイントは一度に1つのクライアント（シングルユーザーMIMO）のみにサービスを提供でき、独自のClientLinkテクノロジーが放射パターンの形成を担当していました（ところで、新しいアクセスポイントでは使用できません）。 言及する価値のあるもう1つの機能は、Mobility Express機能です。 この機能により、1830または1850アクセスポイントをワイヤレスネットワークコントローラとして使用できます。 最大25個のアクセスポイントがサポートされています（管理アクセスポイントはx600、x700シリーズ、そしてもちろんx800でもかまいません）。 新しいアクセスポイントの主な違いは、MIMOの式です。1850では4x4：3（4）、1830では3x3：2です。



また、802.11ac wave 2規格に密接に関連する5520ワイヤレスアクセスコントローラーもありました。このコントローラーの主な利点は、パフォーマンス（最大20 Gb / sのスループット、多数の管理アクセスポイント-1,500およびクライアント-20,000）でした。









また、8540コントローラーが登場し、さらに生産性が高く、非常に大規模なネットワーク向けに設計されています。



802.11ac規格の開発は、データ転送速度の向上と密接に関連しています。 覚えているように、wave 1は最大1.3 Gbit / s、wave 2-最大1.7 Gbit / sの速度で物理データ転送を提供します。 同時に、チャネル速度（私たちにとって関心のある有用な帯域幅）はより低く、wave 1では845 Mbit / s、wave 1では最大1.1 Gbit / sです。もちろん、これらの数値を達成することは依然として困難です。 たとえば、1850の最大値を「圧縮」するには、4つの空間ストリームを受信できるクライアントが必要です。 しかし、それはすでに明らかです。アクセスポイントを有線ネットワークに1ギガビット/秒の速度で接続することは、将来的にボトルネックになるでしょう。 1850アクセスポイントには、すでに2つの統合された1 Gb / sポートがあります。 したがって、Cisco Catalyst Multigigabitテクノロジー（NBASE-T標準に基づく）が積極的に開発され始めました。 これにより、カテゴリ5e / 6のツイストペアケーブル（つまり、現在のケーブルネットワークを使用）で、1 Gbit / s、2.5 Gbit / s、5 Gbit / s、および10 Gbit / sの速度でデータを送信できます。 このテクノロジーのサポートは、Catalyst 4500E、3850、および3560-CXスイッチの一部のモデルで登場しました。

安全性



多くの企業にとって、ユーザートラフィック制御のテーマは、翌年の春に新しい色で輝きました。2015年4月14日、MicrosoftはForefront TMGプロキシサーバーのメインサポートを正式に停止しました。 誰かがこれを事前に準備し、パニックに陥り 、大騒ぎせずにパニックに陥った人は、オープンソースおよび商用NGFW、UTM、およびWebプロキシソリューションの広大な範囲でTMGの代替品を探し始めました。 そのようなソリューションの1つがCisco FirePOWERです。 FirePOWERサービス（5506X、5508X、5516X）を備えた新しいCisco ASAモデルの導入に加えて、今年、次の革新が行われました。
