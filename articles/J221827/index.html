暗号通貨を書いたように

3年前、私は自分の暗号通貨を作成するという考えを持っていました。 Bitcoinから、ノードとブロックチェーンを使用してこのアイデアを採用することにしました。





各ノードを人にバインドすることにしました。 鉱夫になりたい人は誰でも、自分の顔のビデオを撮り、写真を撮らなければなりません。 チェックすると、写真とビデオが保存され、顔が同じであることを確認します。 写真がそうであるように、ビデオはそれを偽造することがほとんど不可能であるという理由だけで必要です。

写真では、データベースの検索時に比率を比較するために、顔の特定の部分にドットを配置する必要があります。

すでにそのようなテストに合格しているすべての鉱夫はこれをチェックします。 その結果、日中に10の賛成票と10未満の反対票を集める必要があります。

これはすべて、ノード間でトランザクションとブロックを送信することで発生し、中央サーバーはありません。 すべての写真/ビデオは公開されています。



あなたがあなたに投票するために10人の鉱山労働者に賄briを贈っても、これはあなたが鉱山労働者になることを保証しません。 10マイナスをキャッチするチャンスがあります。



クローンがすでにDCネットワークに侵入している場合のフォールバックオプションもあります。 この場合、誰かが苦情を伴うトランザクションを送信する必要があります（ParseData-> abuses（））。 次に、管理者、つまり 私は、このマイナーを一般ユーザーのランクに転送するトランザクションを送信する権利を取得します。

合計で、管理者は11種類のトランザクションを利用でき、それらはすべてParseDataのメソッドを使用して処理されます

admin_1block-最初のブロックを処理しています。

admin_add_currency-新しい通貨を追加します。 現在、すべての国の通貨ではなく、主要通貨のみが存在します。

admin_answer-バグレポートへの返信。

admin_ban_miners-苦情があった場合、マイナーをマイナーからユーザーに転送します。 同時に、何とか誘惑されたものはすべてユーザーの自由です。

admin_message-インターフェースに表示される短いメッセージ。

admin_blog-将来、突然、短いメッセージに収まらない何かを報告することが必要になります。

admin_new_version-新しいバージョンを入力します。

admin_new_version_alert-更新が必要なインターフェースでアラートを提供します。

admin_spots-クローンを見つけるためのすべて-ポイントのセット、ポイントのバージョンの互換性、そこからセグメントを作成するポイント、許容される矛盾。

admin_unban_miners-降格したマイナーをマイナーに戻します。

admin_variables-変数、データベース内の変数テーブル。 その後、定数にする予定です。



誰も質問しないように、すぐにすべての管理機能をリストしました-管理者はお金でアカウントをブロックできますか？ いいえ、できません。管理者に許可されている最大値は、アカウントの資金に影響を与えずにアカウントの特権を下げることです（admin_ban_miners）。

ブロック生成順序



すべてのノードは、ヘッダーの現在のハッシュ（node_id、block_id、prev_head_hash）に応じてレベルに分割されます。

レベル0のノードは、testblock-> get_block_generator_miner_id（）メソッドによって決定されます

たとえば、ノードID 12689が現在0番目のレベルにある場合、12690-12691は1番目のレベル、12692-126995は2番目のレベルなどになります。

0番目のレベルのノードがブロックを生成できなかった場合、1番目のレベルのノードが作品に含まれ、そうでない場合は2番目のノードなどが作品に含まれます。

つまり 攻撃者はブロックにノードを正確に生成させることはできません。

ノードがあり、それぞれに1人の生きている人が割り当てられています。 Xコインを各鉱夫に簡単に配布できます。 しかし、何らかの理由で、このアイデアがすぐに気に入らなくなりました。 発生する可能性がある例-habrahabr.ru/post/217111以降-coinmarketcap.com/aur_90.html

はい、ちなみに、DcoinはDemocratic Coinの略で、DCです。



ソリューションは非常にシンプルであることが判明しました。 コインの分布と、Dcoinを現金に交換する機能を組み合わせる必要があります。 これを行うには、主要通貨をDcoinにコピーする必要がありました。 3文字の通貨コードに文字Dを追加しました。 DUSD、DEUR、DRUBなどが判明しました。

たとえば、鉱山労働者が1000 DUSDを購入する場合、DCネットワークにトランザクションを送信します。これは、1000 DUSDで1000 DUSDを購入することを示しています。 しかし、DUSDを売りたい人はいないかもしれません。 鉱夫は、DUSD売り手が現れるまで無期限に待機する準備ができている必要があります。



現在、たとえば、分散データベースに10万件のレコードがあり、鉱夫は1億ドルでDUSDを購入する準備ができています。 DUSDを作成します。 誰もがおそらく予掘を考えた。 しかし、誰かが私にそこのDUSDに$を払ってくれるとは思わないので、すぐに。 はい、そして事前に掘り下げるのではなく、単にコインを配る方が論理的です。



鉱山労働者は誰かが彼にDUSDを売りたいと思うまで待つので、鉱山労働者の口座にDUSDを請求することにしました。

DUSDは次の式に従って計算されます。

nDUSD = bDUSD *（（1 + K）^ T）-bDUSD

ここで、nDUSDは新しく作成されたDUSD、bDUSDはマイナーが購入する準備ができているDUSDの量、Kは以下で記述する係数、Tは秒単位の時間です。

つまり より多くのDUSDを購入する準備ができており、誰かがそれらを販売するのをより長く待つと、より多くの新しいDUSDが作成され、彼のアカウントに入金されます。

ソースでは、この式はParseData- > calc_profit（）メソッドにあります。 calc_profitにはさらにいくつかのパラメーターが含まれていますが、それらのパラメーターを説明し始めると、退屈で長すぎます。 しかし、誰かが興味を持っているなら、私は書きます。



係数K



これは非常に重要な要素です 放出はそれに依存します。 私は長い間、どのようにそれを作るか、どのパラメーターにアタッチするかを考えました。 結局のところ、彼はラップに屈するべきではありません。 最も民主的な選択肢は、採掘者自身が投票によってこの係数を決定するものでした。 便宜上、インターフェース部分では、係数Kは％/年として表示されます。 有効な値は0〜1000％/年です。 ソースコードでParseData-> votes_complex（）。 マイナーは、['min_hold_time_promise_amount']秒前に$変数を超えて追加された通貨にのみ投票できます。 クローン攻撃から保護し、音声でネットワークを不安定にできないようにします。

係数の再カウントと更新は、$変数['new_pct_period']秒ごとに行われます。 今は2週間です。



現在、分散データベースには、鉱夫が1億DUSDを購入するという約束だけでなく、排出率に対する投票もあります。 係数Kについて。そしてしばらくすると、最初のDUSDが鉱夫の口座に表示され始めます。

「農場」の出現の可能性について





ほとんどの場合、遅かれ早かれ、「ファーム」があり、cなビジネスマンは約束された金額とオフラインチェックを追加したアカウントの完全登録に10〜20ドルを支払います。 しかし、そのようなトリックが現れると、そのようなアカウントから受け取る利益は少なくとも数百ドルになります。 誰かがそのような灰色のアカウントに示された約束金額とDcoinを交換するリクエストを送信した場合、そのような農場の所有者が彼の約束金額に従ってコインの成長の凍結を受けることは完全に不採算になります。 結局、受け取ったDcoinは取引所で簡単に販売できます。 そして、彼は同じ量のDcoinと引き換えに約束された量を与えなければなりません。 つまり 実際、そのような農場は単純な鉱夫のグループと変わらないでしょう、唯一のことは一人がすべてのクリームを受け取るということです。

しかし、そのような農場には大きなボリュームはありません。 なぜなら 最初のそのようなスキームを明らかにするとき、最初のビデオを記録するためのルールが追加されます。これは、カメラに「demcratic-coin.comのルールに同意します」という紙を表示する必要があることを示します。 好奇心のほとんどは、彼らが同意するものを見つけるためにサイトに行きます。 また、サイトでは、アカウントを10〜20ドルで販売することは非常に不採算であり、このアカウントを自分で使用する方がはるかに有益であると書かれています。



しかし、人口の85％がインターネットを持っていないインドの遠隔地に行って、スタジオで人々を撮影するとどうなりますか？

繰り返しますが、深刻な負の影響を受けない農場を手に入れます。 農場の所有者が約束された金額の要求を無視することは不利益です。

しかし、そのような農場を燃やさないことははるかに難しいでしょう、なぜなら 大都市でオフラインチェックに合格して道に迷うことはできません。つまり、25人の鉱山労働者がまだ集まっていない小都市の住人になりすます必要があります。 そして、そのような都市はすべて管理者によって慎重にチェックされます。 動画内の同様の要素を見つけ、新しい動画を削除するリクエストを送信することは残っています。



農場の存在そのものが、Dcoinがすでに進化しており、各マイニングアカウントが恒常的な利益の源であることを示唆しています。 そして、そのような農場の所有者は、住民の無知を利用して、超利益を受け取ります。 この事実は長い間気付かれることはなく、遅かれ早かれメディアに行きます。 そして、これらすべての知識のない人々は、自分自身が鉱山労働者になることがより有益であることをすぐに認識します、これにより、その時までに、あなたはコンピューターさえ必要とせず、インターネットカフェに一度登録して、あなたの携帯電話を使い続けます37コイン 。

DCネットワークにデータを送信し、ノードで確認する



オフラインチェックを正常に完了した後、マイナーになりたいユーザーは、2枚の写真を自分のノードにアップロードし、それらをポイントし、ビデオURLをポイントして、DCネットワークにトランザクションを送信します。 同じトランザクションで、写真からのハッシュが示されます。

ノードはブロックの1つでこのトランザクションを受け取り、これらの写真を自分自身にダウンロードする必要がある人の中にいる場合、自動的にダウンロードします。

写真を独自に保持する必要があるノードは、このアルゴリズムによって決定されます。

 /** * @param int $block_id  ,     . * @param int $max_miner_id -  id   ,  -   . * @param int $miners_keepers -       .   10. * @return array */ function get_miners_keepers ($block_id, $max_miner_id, $miners_keepers) { for ($i=0; $i < $miners_keepers; $i++) { $hi = $block_id / 127773; $lo = $block_id % 127773; $x = 16807 * $lo - 2836 * $hi; if ($x <= 0) $x += 0x7fffffff; $miner_id = ( ($block_id = $x) % ($max_miner_id + 1)); $miner_id = ($miner_id==0)?1:$miner_id; $array[] = $miner_id; } return $array; }

その後、類似した人々の写真を示すページが表示されます。 疑わしいほど類似した人物がいるかどうかを視覚的に確認する必要があります。 人種の選択があります ヨーロッパ人にとって、すべてのアジア人は同じ顔をしており、アジア人にとって、すべてのヨーロッパ人は同じです。

内部からクローンを検索する





2枚の写真のそれぞれに1つのセグメントが必要になり、他のすべてのセグメントを比較します。

これが、写真の顔全体の目と目からプロフィール写真の耳の端までの距離です。



spots_compatibilityテーブルには以下が保存されます。

1.現在のバージョン。

2. 2つのサンプル写真に適用されるポイント。

3.互換性のあるバージョン。

4.セグメントを取得するために接続するポイント。

5.クローンを検索する際に許容される矛盾。



facesテーブルには以下が保存されます。

1. f1、f2 ... f20-顔全体のセグメントと目の間の距離の比率。

2. p1、p2 ... p20-目から耳の端までの距離に対するプロファイルの顔のセグメントの比率。

3.当時のspots_compatibilityのバージョン。



クローンの検索でより効率的なポイントのセットが見つかった場合、spots_compatibilityテーブルのデータが更新され、新しいポイントのセットに基づいてセグメントがFacesテーブルに入力されます。



鉱夫の申請者を確認する最初のステップでは、写真上のポイントが正しくプロットされていることを確認しました。つまり、セグメントも正しく取得されます。 spots_compatibilityとの許容差を考慮して、f1、f2 ... f20およびp1、p2 ... p20との比較が行われるデータベースに要求するだけです。 その結果、鉱夫のIDを取得します。これは、申請者に似ています。
