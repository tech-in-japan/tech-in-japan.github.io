あなたは25歳で、あなたの後ろにはすでに4つの会社があります-何が悪いのですか？







この記事は、若い世代のITプロフェッショナルだけでなく、求職者のZ世代をよりよく理解したい採用担当者にとっても興味深いものです。 仕事を見つける方法についてのアドバイスはありません。私の話は、若い専門家が周りの出来事に注意を払うべきであり、誰も信頼してはならない理由のヒントとともにここで説明されます。



まえがき



おそらく他の多くのIT専門家と同様に、私は高校の初めにこのトピックに参加しました。 Java MEでの携帯電話向けゲームのいくつかの実験、最初のAndroidのファームウェアによるシャーマニズム、このプラットフォームで現在休息しているウェアーズポータルの基盤とメンテナンスは、友人や自分自身がさまざまなコンピューターの問題を解決するのに役立ちます。



大学に入学する頃には、LinuxとWindowsのシステム管理の経験、PCハードウェアの知識、そして家庭のADSLモデムの研究とTanenbaumの研究のおかげで、ネットワーク知識の始まりでした。 ボーナスはCのプログラミングコースに合格しました。



さらに先へ-この質問への答えは、可能性のある大学の綿密な研究、さまざまな専門分野でのトレーニングプログラム、興味深い専門分野の若い卒業生のソーシャルネットワークに関する調査の後で見つかりました。 一般に、私は若い職業にテレコムに行き、21世紀のITのコミュニケーションに非常に強い影響を与えました。 おわかりのように、トレーニングプログラムはOSS / BSSエンジニアを中心に構築されました。



場所について



2年目に既に最初に見つけた職場は、かなり大きなインターネットプロバイダーでした。私の知識のおかげで、すぐにRWおよびRモードのコアネットワークレベルへのアクセスと集約にアクセスできる2つのレベルのサポートに気づきました。 すぐに習得し、アプリケーションから頻繁に問題が発生することに気づき、最終的に1つのWebページにいくつかの入力フィールドがある10個のボタンでソリューションを自動化しました。 アプリケーションは私たちのレベルよりもずっと少なくなり始め、高位の役人で十分でした。そして、これと他の成功を考えて、彼らはインフラストラクチャ開発に関与するインターフェース部門の部門長とのインタビューを提案しました。 基本的な要件は静的ルーティングと英語（流was）の知識でしたが、マネージャーは3コースの終わりに勉強し、会話が止まった5分間にgoogleで学習したアーランについて質問を「入れ」ました。 また、高等教育を受けていない人がエンジニアと一緒に座るのは無価値であり、一般に大学と仕事を両立させることができるかどうかは疑問であると述べました。 現在の場所で私はすでに最大限に「汲み上げられ」ており、卒業の数年前に次のステップを待つという事実を考慮して、私はあまり長く滞在せずに去ることを決めました。



その後、さらに3つの企業がありました。どこかで、Linuxサーバーでアプリケーションをホストするデータセンターのクライアントをリモートでサポートし、アプリケーションの可用性についても責任を負いました。 燃料システムを重視したIoTシステムのカスタマーサポートで働いていたどこかで、AWSにAIプロジェクトを持つ企業がありました。最終的に、DevOpsと呼ばれるようになったのは、Ansible、Jenkins、スクリプト作成、デプロイ、トラブルシューティングWebアプリケーションです。



ほぼすべての時間、私はフルタイムの学生であり、最初の会社の長の予言とは反対に、私は何とかして何とかして、今まで部門でほこりを集めていたネットワークハードウェアを掘り下げるためのファンの輪を設立しました。 彼はラボ、大学のネットワーク、サークルの学生の教師を助けました-損失を出す工場で2年間大学を卒業しないようにするためのヒントを示しました。 [ネタバレ、誰も工場に行かなかった]



それで、なぜ私は次の3つの会社を去ったのですか？ おそらくそもそも-自分の言葉と対立する人々の行動。 第二に-時々不快な先輩。



私はすでに理解しているように、目と火の知識を持つ若い専門家を見て、彼が多くの人に約束するのは魅力的です。



ある場所では、面接で実証された企業のルールに基準が厳密に記載されている成功の場合、従業員は階段をあるレベルから別のレベルに移動することを約束されました。たとえば、サポート従業員のレベルから自分の車両のジュニア管理者に階段を上る。 その結果、基準は6か月強のどこかで達成され、給与は正直に引き上げられましたが、私は12時間の交代制勤務スケジュールでサポート従業員のままでした。 19歳でも、それは非常に困難です。 経営陣と話をしてこれらの成長の約束を思い出すことは何にもつながりませんでした。上級管理者の1人は、非常に表現力豊かに「生きることを学ぶ」と、ほぼ毎日のシフトを悩ませました。



数百万ドルのサーバー群と数十の投資を持つ立派な人々が、インフラストラクチャのDevOpsレールへの移転と、現在のすべてのオペレーションの実際の移転について教えてくれました。まあ、最初の数か月でしたが、投資家のビジョンは劇的に変わりました。 IT /運用/ DevOpsが必要であると判断しました（ローカルインフラストラクチャが泣いていて、すべてのスペシャリストがMiddleより低くなく、会計士でさえ自分で問題を解決できるため、会社は基本的にITを持っていません） 部門の長から「話をする」と言われ、自分がいかにクールであるか、そしてチームの最年少としてIT部門に行く必要があると言われました。 IT部門でDevOpsを使用したくない、または優秀な仕事をしたくないという異議に対して、タスク、なぜ私の高価なLinuxエンコーダーを使用するのか、40歳の「500クライアントWindowsシステムと1サーバーの管理経験がある管理者」を雇うことができると答えた私はそのような人を見つけません、そして、学生を連れて行くのは危険です-彼らの潜在的な間違いの代価は、私に取って代わる彼らの節約以上です。 私はその会社を去りました。多額のお金でタスクを実行したくありませんでした。ローカルRAIDと管理者のウィンチェスターを2、3のTsiskov腺に変更し、マウスの金髪をチームに現れた金髪の秘書に変更しました-私は仕事にそこに行きませんでした。



誰もが口頭で親切でクールな会社もありました。成長基準も明白であり、内部文書に記載されています。 しかし、顧客の問題に対する*共感*が不十分なため、3か月の試用期間が知らないうちに延長されたことがわかったとき、彼女を辞めました。 ちなみに、私は基本的に電子メール以外の方法で仕事やタスクについて話したことはありませんでした。解決されたタスクの成績は良好で、完了したタスクのスケジュールは私と同期して雇用された同僚のスケジュールよりも速く成長しました。 技術的な能力の問題では、私は非常に高く評価されました。特に、これは経験豊富な叔父の同僚を激怒させました。 年齢差が20歳に達する可能性のある人々とチームを組んで、若くて頭が良いことは困難です。 モニターは非常に安価なTN +フィルムであり、1日の仕事の後、私の目は文字通り「流出」したので、私もそれを残しました。 新しいものを発行する要求は無視されました。 私のキャリアパスには興味深い変化がありました-経験と渇望のためだけに、私は高い給料から低い給料に（30％ずつ）行きました。



大企業のマストドンがまだあり、私の経営陣は、私の仕事の2週間後に、直近のマネージャーを解雇しました。彼は、過去6年間彼の仕事の会社のソフトウェアの内と外をかじっていました。私は右手を探していましたが、それを使って、長年にわたって確立されてきた開発サイクルを掻き立てることを計画していました。 実際、私は放っておかれ、スタッフが非常に不足していたため、実際には所有者のいない内部プロジェクトマネージャーの完全に非中核的な仕事を私に任せ始めました。 意図せずに「これらのサービスの所有者」であり、会社に多く拘束されていたため、さまざまな部門のハードウェアおよびソフトウェアでの編集の実行の正確性を注文および監視しました。 シックなコーポレートは、改訂ごとに会議を開催し、多くの人々を仕事からそらしていると考えられていました。 この会社以外では、これほど多くの会議を開いて出席したことはありません。 そこで数か月間、私はプロジェクトマネージャーとして開発に役立つ多くの活動に参加しました.DevOpsプロファイルでは、いくつかのJenkinsのみを復活させました-私の部門の長の解任後に死亡したジョブを持つインスタンスは、いくつかのプロジェクトの開発に参加しました。



ワイルドコードのワークフロー、奇妙な解雇、静的コードアナライザーのようなDevOpsツールを実装するための会社の昔の人の不本意などのより現実的な問題が解決されるまで、DevOpsが社内でできない理由を上級管理職に数回説明し、彼は去りました。

それは魅力的で、長い間考えていましたが、最終的には、20年もの間醸造されてきた技術や方法以外に興味がなく、私よりも古い人を管理することは22であまり効果的で生産的ではないという結論に達しました。



それで、今は何ですか？ ここ数年、私は開発部門で、複雑な通信ドメインの問題を解決する狭いサークルで広く知られている会社でDevSecOpsとして働いています。 解決されているタスクと労働条件は、インタビューで議論されたものに対応し、ワーキングアイアンに関する日常的な問題や質問はなく、経営陣は可能な限りオープンであり、彼らによると、職場を変える豊かな歴史を持つ私と一緒にいた恐怖は背後にありました。



すべての人に平和を。エピローグとして、私が非常に尊敬されているマネージャーとしてのキャリアの中で出会ったものからの引用を残します。



-それに慣れないでください！
