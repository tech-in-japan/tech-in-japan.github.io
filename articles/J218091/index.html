Pintask-プログラマブルタスクトラッカー

敬礼、ハブル！



統計によると、新しいタスクトラッカーは2週間に1回表示されます。 「でも、カプチーノの醸造方法はわかっています！」-開発者が教えてくれます。 「そして、他のトラッカーは、モスクワ-ゲレンジーク列車の指揮者と同じ品質のお茶を作るか、コピルヴァクだけが醸造されます。」 カプチーノが注ぐように蛇口を切ることができたら、新しいタスクトラッカーを作成しませんか？ 「もちろん！ カプチーノは私のお気に入りの飲み物です...



コーヒーメーカーとOptimus Primeの共通点については、なぜプログラム可能なタスクトラッカーが必要なのか、蛇口を切る方法についてはこちらをご覧ください。





1969年に、超小型回路の状況は、トラッカーの現在の状況とほぼ同じに見えました。 計算機をリリースしたいですか？ チップを印刷します。 1000で乗算する機能を備えた電卓をリリースしたいですか（ブラジルの会計士向け）？ 別のチップを印刷します。 日本人が計算機用に最大12個のチップを注文したのは、このロジックでした。 同社のフルタイムロックスターであるテッドホフは、設計図を専門家に見て、リファクタリングを提案しました。 それから、ロックスターにふさわしいので、彼はプロセス自体をスキップして、アイデアの実装をフェデリコ・ファギンに任せました。 その時点で完全に異なる方向が猛烈に猛烈に発展し、そこでベントレーでお金を稼ぐことができたため、経営陣はプロジェクトで安全に採点しました。 これにより開発が大幅に加速し、すでに1971年に電気工学のプログラマーがテストを完了し、生産のために展開し、シリーズを開始しました。 ファンファーレを持つインテルは、日本人の注文を完了し、静かに新しい事業を開始しました。 数年後、マイクロプロセッサ市場は突然非常に収益性が高くなり、すでにBentleyで利益を上げました。



コーヒーへの情熱によって世界が支配され、毎日新しいタスクトラッカーが登場する私たちの時代に早送りします。 状況自体はリファクタリングを求めています。 「蛇口を切る」、ねじを外す、または逆に各ナットをより強く締め、サーモスタットを交換し、蒸気調整器を設定してお気に入りの飲み物を作ることができるトラッカーを書くのは理にかなっています。 まさにそのような-プログラム可能なタスクトラッカー-はPintaskです。



少し配管



正直なところ、マイクロプロセッサの出現は、集積回路が大量生産され始めた1960年代初頭に理論的に可能でした。 ただし、途中の若い電気技師は、80レベルの技術的障害にうそをついていました。 フェデリコ・ファギンは彼らについて直接知っていました。 しかし、彼はモスクワ地下鉄の通路ではなく、パドヴァ大学で物理学の博士号を取得しませんでした。 したがって、1968年に、Federicoは最初のシリコンベースの電界効果トランジスタを作成しました。これは、他のバイポーラよりも高速に動作し、スペースを取りません。 今では、トイレの照明を消さずに、マイクロ回路の最適性を心配することもできませんでした 。マイクロプロセッサの計算機は、電界効果トランジスターで動作するため、前世代よりも速く加算および乗算されました。 まあ、彼らの値札は低く、すべてのブラジルの会計用にデバイスを購入するときに非常に非常に顕著です。



状況を評価するために、イノベーションに関するグローバルエキスパートに目を向けます。

-フォード氏は、あなたの意見では、マイクロプロセッサ技術はさらに発展するのでしょうか、それともバイポーラトランジスタのメーカーはクロックを戻す方法を見つけますか？

-私は、より低いレベルでの革新が既存の産業を変えることができると確信しています。 私の意見では、これはまさにこれが何であるかです。

-コメントありがとうございます。 そして今、科学の世界からのニュースへ。 天文学者は珍しい宇宙背景を報告します...



Pintaskは Meteor で書かれています。 このフレームワークは、すでにJavaScriptの重みカテゴリでGithubの最も人気のあるリポジトリの1つです。これは公式リリース（現在のバージョン：0.8）より前です。 ハッカーニュースでの発表だけで1362票を集めました。 コメントからの抜粋：「それは産業革命のようなものです」、「私はこれを十分に支持することはできません」-さて、あなたはポイントを得る。 懐疑論者の半パックを食べた後でも、Meteorはウェブ開発の新しいページを開いたと安全に言うことができます。 さらに、このページのHTMLコードは「ライブ」です。サーバーから新しいデータを受信すると、テンプレートが自動的に更新されるため、これに従う必要はありません。 そう、Ajaxリクエストは忘れてください。データは、完全なMongoDB APIを介してJavaScriptに変更され、ブラウザで直接利用できます。 その結果、計算の90％がクライアントで実行されます。 Meteorコードは、使用するスペースが少なく、ボリュームが少なく、 コストが低く、書き込みが速く、 エネルギーを節約し、転送されるデータの量を削減します。 マーティ、再びプルトニウムで遊んだ？



何が起こるか見てみましょう。 今回は、データベースへのアクセスはブラウザ内で行われます。 テンプレートもブラウザにあります。これらは2つです。 すべてがJS + HTMLで機能します。これらは3つです。 しかし、ユーザーにスクリプトを接続する機会を与えるとどうなりますか？ 結局のところ、彼らの助けを借りて、あなたはハングしてリスナーを作り、テンプレートを再定義し、データを書き留めることができます...そしてタップを通して見ました。 そして、ナットを締めます。 そして、サーモスタットを置きます。



すべてのカプチーノ



Pintaskは、Meteorで書かれた最初のタスクトラッカーです。 また、拡張機能をブラウザに直接ダウンロードして、データベースAPIとテンプレートへの完全なアクセスを可能にする最初のタスクトラッカー。



そのような拡張機能の例を次に示します。 



if !Cards throw "Hey, where are my Cards?" if !Comments throw "Look, I need those Comments, too!" Comments.after.insert (userId, comment) -> cardId = comment.cardId card = Cards.findOne(cardId) if userId not in card.memberIds Cards.update(cardId, {$addToSet: {memberIds: userId}})




最近、私はCoffeeScriptにはまっています。もちろん、主流ではありませんが、開発をスピードアップします。



この単純な自動化は、目隠しで組み立て、分解できます。 最初に、核戦争のチェックがいくつかあります（スクリプトが別のサイトに呼び出される場合）。 次に、コメントを挿入すると機能する時限爆弾がインストールされます。 違反者がカードのメンバーでない場合、彼は衝撃波によってそこに投げられます。



テンプレートの動作を確認したいですか？ 私はそれらを持っています 。 ここではjQueryが関係しています。一見、コードはイタリア料理に似ています。 しかし、底からそれを吸収し始めると、味がより調和的に明らかになり、その後...「待って、繰り返して！」



それとは別に、外部データソースの接続に言及する価値があります。 ホワイトマジックを使用すると、Googleスプレッドシートの文字列からタスクを作成できます。 そして、別世界のAPIの助けを借りたブラックマジックにより、Pintaskで直接Github Issuesを呼び出し、内部トラッカーと外部トラッカーのコメント間の双方向通信セッションを調整することができます。 これはすべて、MongoDBコレクションを通じて行われます。 実際、この拡張機能は、特別なスクリプトを使用してPintaskにデータを読み込む別個のアプリケーションです。



バーの後ろ



ところで、同じGithubを介して独自の拡張機能を接続できます。 最も単純なケースでは、Github Pagesでリポジトリを公開し、jsファイルへの直接リンクを取得して、Pintaskに貼り付けます。 ドライで快適であることがわかりました（そして、コミットが到着すると更新されます）。

また、アプリケーション全体をブラックジャックに接続する必要がある場合は、同じMeteorの無料ホスティングに配置し、リンクを挿入できます。



ユーザーにとっては、すべてがさらに簡単に見えます。 Extensions Storeに行き、必要なものを選択して、魔法のボタンを押します。そして、手首を軽くたたくと、コーヒーメーカーがOptimus Primeに変わります。



プログラマーは寛大な人々なので、言うのは安全です。ほとんどのアプリケーションは無料です。 しかし、ヒキガエルが首を絞めたり、より多くの木が必要な場合（ボリュームファイルストレージ、SMSゲートウェイ）、ユーザーを拒否できないアプリケーションにすることができます。 自分で稼いでいます。



時計を確認する



ああ！ ここに、有料アプリケーションがあります： autopilotでの時間追跡 。 プログラマーが時間を追跡するのがどれほど難しいかは誰もが知っています。 私たちの心の畳み込みをさまよう後、あなたは同様の内部対話に会うことができます：



-ちょっと意識！ Yandex.Marketとの統合について考えてみましょう。

-待って、待って...「スタート」ボタンを押すのを忘れていませんか？

-ああ、確かに。 ここでクリックしました。

-よくやった。 そこでビットコインについて何を話しましたか？

-ビットコインとは何ですか？ Yandex.Marketについて話しました。

「ああ、はい...待って、前のタスクの開始ボタンを押しましたか？」

「はい、そうです。」 かどうか。 今すぐ確認します。

-確認しましょう。 ここでソリティアをレイアウトします。



精神分析医に行かないために、時間追跡の通常のメカニズムを変更することを決定し、「開始」ボタンを放棄しました。 代わりに、各リストのヘッダーに「時間追跡を有効にする」アイコンを配置します。 システムにログインすると、時間の追跡が自動的に開始され、マークされたリストの最初のタスクの追跡が開始されます。 タスクがアーカイブされるか、別のリストに転送されると、時間追跡はクリップに基づいて次のタスクに切り替わります。 その結果、プログラマーは時間追跡についても考えません。 それはちょうどその仕事をし、システムは時間間隔を自動的にキャプチャします。



技術的な実装では、標準のMeteorメカニズムとユーザーステータスパッケージを使用してアクティビティを監視します。 MongoDBコレクションの使い慣れたリスナーを使用すると、追跡するタスクを定義できます。 このコードは、setTimeout（holy、holy）を使用せず、イベントのみを使用します。 その結果、サーバーを再起動しても、履歴を中断することはありません。



正確なレポートを取得したいですか？ 入って、恥ずかしがらないでください 。



デザート用



基本プラットフォームの価格はいくらですか？ ピンタスクは完全に無料です。 機能制限、広告、月額料金はありません。 しかし、バックグラウンドにはオープンなウィキ 、 誠実なサポート 、子羊がいます。



最後に、PintaskはTrelloの完全な代替品になるよう努めています（「ドロップイン交換」）。 海外サービスのすべての基本機能はすでに実装されています。 Trelloからの即時インポートがあります。 また、 双方向同期を実行して、Trelloのモバイルアプリを介してPintaskへのアクセスを提供します（独自に作成する前に）。
