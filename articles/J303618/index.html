ReactOSの実際の使用で最初に確認されたケース





私たちの公式サイトは最近、ReactOSオペレーティングシステムはまだ日常使用する準備ができていないというメッセージを投稿し、サイトから配布されたOSイメージはテストのみを目的としています。 それにもかかわらず、「ダック」は、ReactOSがそのようなスーパーマーケットのチェックアウト端末にインストールされているのを定期的に明らかにしました。 しかし、ジョークはジョークであり、時間が経ち、リリースが出ました。



そして少し前に、ロシアの図書館のシステム管理者が私たちに頼り、ReactOSの実際の「戦闘」アプリケーションの仕事の経験を共有しました。 アレクサンダーは、ReactOSを実行しているコンピューターにIRBIS64サーバー（書籍のライブラリカタログ）をインストールして構成できると述べました。 さらに、構成は深刻なグリッチなしで長い間機能しており、その機能を実行します。









アレクサンダーは言います：

私はライブラリの管理者として働いており、バージョン0.4からはすでに目標の半分を達成しています。電子カタログ（Irbis 64）のサーバー（!!!）としてReactOSを使用しています。 Windowsサーバーでの苦痛の後、ReactosでIrbis64をテストすることにしましたが、今日まで正常に機能しており、サーバーからのダスティングを作成するだけでサービスがハングアップします。 winservライセンスの保存については黙っています...



このようなハードウェアでは、ライブラリ内のIrbisサーバーが回転しています。 データベースサイズは約1.5GBです。\ 40,000エントリ：





ReactOSへのIRBIS64サーバーのインストールに関するビデオ。 1つのビデオはサーバーのインストールと開始（VirtualBoxからの記録）を示し、2番目のビデオでは同じインストールと開始が示されていますが、その後のクライアントの接続（Windows 10から）を示しています。 ReactOSでクライアントの起動をデモしたかったのですが、コンピューターは実行中の2つの仮想マシンを取り出しませんでした。 後で必要に応じて書き留めます。



説明に示されているように、仮想マシンからこれらのデモビデオを記録しました（より便利で高速であるため）。 現実の生活（仕事上の目的）では、Irbisは実際のハードウェア上のReactOSeに依存しています。 現在のバージョンでは、RosはほぼすべてのPCに正常にインストールされていることを誰もが知っています。 VMでのPCでのIrbisのインストールと使用はまったく違いはありません



PS。 クライアントがビデオに接続すると、データ/インターフェースを長時間ロードすることがわかります。これは、ビデオが2つのストリームに書き込まれ、システムに大量にロードされたためです。 実際の操作では、クライアントは即座にロードされます。 5つの接続されたクライアントとの「戦闘」状態でテストされています。








ライブラリ「IRBIS64」の自動化システムのインストール手順

-コンポーネントのインストールとシステムの準備。

必要なもの：

1.次のファイルをCにコピーします。\ ReactOS \ system32

-fontsub.dll（WinXPから - ファイルのバージョンは5.1.2.6000.5512） （リビジョンr71873は必要ないため）

-msvbvm60.dll



2.インストール

-RuntimePack Lite v15.7.22

-Microsoft Visual C ++ 2005-2008-2010-2012-2013-2015 [32ビット]（2015年4月12日）



3.「サービスマネージャー」を開き、サービスを開始します。

-ReactOSインストーラー

-シンプルなTCP / IPサービス

各サービスのスタートアップの種類は「自動」です！

-ネットワークのネットワーク接続をセットアップします。



4. IRBIS64をインストールしますが、起動しないでください！

！ IRBISをインストールした後、「サービスマネージャー」を開き、サービス「Irbis64_Service」を開始します。

！ スタートアップの種類-「自動」。





重要です。 サードパーティライブラリのfontsub.dllを使用することは一種のハッキングであり、近い将来、ReactOSの対応するバグが修正されるとすぐに必要なくなります。 （リビジョンr71873は必要ないため）
