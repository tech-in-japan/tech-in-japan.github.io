オープンソースのビデオ会議とフラッシュ

前回の記事では、フリーソフトウェアに基づいたH.323ビデオ会議の構築方法を明らかにし、多くのレビューと批判を受けました。 特に、かなりの数の読者が、ほとんどのタスクにこのような複雑なシグナリングプロトコルを使用することの根本的な重要性を疑問視しました。実際、私たちの時代には、TandbergやPolycomまたは、接続を確立する組織のサイズが小さすぎて、時間のかかるものや高価なものを実装することはできません。 その後、フラッシュテクノロジーと優れたオープンソースソリューション-OpenMeetingsプロジェクトが私たちの助けになります。



ビデオチャットシステムの主な機能は次のとおりです。

マルチポイントビデオ通信を提供します。

ユーザーのコンピューターに追加のソフトウェアをインストールする必要はありません。 サーバーとの通信は、通常のブラウザーを使用して実行されます。

「ホワイトボード」およびオフィス文書とのコラボレーション。

低ハードウェアコスト：Pentium 4 2GB RAMサーバーは、少なくとも100の接続を処理できます。

信頼性の保証：サーバーソフトウェアが少なくとも48時間失敗する期間。

Microsoft製品との統合。

会議を記録する機能。

LDAPを使用したエンタープライズアドレス帳の使用。

会議の招待状を配布します。

ロシア語の指示。



サーバーソリューションは、次のコンポーネントに基づいています。

MySql-データベース;

Apache Tomcat-Webサーバー。

Red5-フラッシュメディアサーバー。

OpenOffice.org-ドキュメントサーバー。

Openmeetingsは、javaおよびスクリプト言語で書かれたWebアプリケーションです。

サーバーへの接続は、http（ポート5080）、rtmp（ポート1935）、rtmpt（ポート8088）プロトコルを使用して実行されます。 サーバーをインターネット上で表示するには、これらの3つのポートをルーターまたはファイアウォールの背後にあるコンピューターに送信するだけで十分です。



もちろん、最も簡単な方法は、既にアセンブルされたコードのインストールの説明へのリンクを提供することです。これは存在するため、注意をそらすことはありませんが、もう少し洗練された道を進むことにしました。 第一に、すべての同様のオープンソースソリューションのように、インストールの説明は詳細でいっぱいではありません。第二に、自分自身の気まぐれや顧客の希望に最小限の変更を加える必要があり、コンパイルされたコードをインストールするだけでは不十分です。 一方、プロジェクトは十分に大きく、1つの記事のフレームワーク内ですべてを記述せずに説明することは不可能であるため、今日は以下に限定します-この記事では、サーバーのソースを取得し、コンピューター上のサーバーコードを制御して管理する方法について説明しますOS Windows XPは、その動作をテストするだけでなく、クライアントサーバーまたは独自のサーバーにインストールする方法を自分で判断します。 クライアント側での作業はオプションとして残されます。

ソースコードの取得とコンパイル


まず、サーバーコードを取得して収集します。 これを行うには、TortoiseSVNをインストールした後、ディスク上にフォルダーを作成し、そこでSVNチェックアウトをコードの最後の（または他の）リビジョンにします。





リポジトリへの参照として、指定します 
  http://openmeetings.googlecode.com/svn/trunk/webapp/




次に、以前にダウンロードしたEclipseを配置し、ワークスペースで新しいプロジェクトを作成し、OpenMeetingsと呼び、そこにファイルシステムをインポートし、新しく作成したフォルダーをメインのフォルダーとして指定します。





Eclipseのディレクトリ構造との互換性を維持するために、フォルダー自体ではなくコンテンツがインポートされることに注意してください。



[ウィンドウの作成]-> [ビューの表示]-> [Ant]を開き、開いたウィンドウで[ビルドファイルの追加]をクリックしてbuild.xmlビルドファイルを追加すると、次の図が表示されます。





ここで、JDKをインストールします。また、後で使用するためにEclipseに登録する必要があります。これは、Window-> Preferences-> Java-> Installed JREsでインストールされたJDKへのパスを追加することによって行われます。 すべてが正しく記述されている場合、Ant Viewの形式で配布パッケージ（dist、デフォルト項目）の構築をダブルクリックすると、プロジェクトの構築に成功します。 コンソールに加えて、プロジェクトの右ボタンを押して[更新]を選択すると、成功を制御できます。ビルドされたOpenMeetingsファイルを含むプロジェクト構造にdistフォルダーが表示されます。 おめでとうございます、あなたは勝利の途中です！





データベース


次に、MySQLをインストールして使用できるように準備し、プロジェクトを実行してみます。 原則として、デフォルト値は私たちの目的、つまり少数のクライアントのDBMSに非常に適しているため、このプロセスの詳細については説明しません。 データベースのエンコーディングを除き、UTF-8を使用します。 インストール後、データベースにアクセスして動作することを確認し、OpenMeetingsプロジェクトをインストールするためのブリッジヘッドを準備する必要があります。 これを行うには、コマンドラインから実行し、パスワードを入力します-そして、システムにいることを確認し、それが動作すること：





ログインが成功し、データベースが作成されたら、distフォルダー（コンパイル後に表示されたフォルダー）/openmeetings/conf/mysql_hibernate.cfg.xmlからファイルを編集し、ユーザー（たとえばroot）とそのパスワードを登録する必要があります。サーバー127.0.0.1およびopenmeetingsデータベースとしても。 私の場合、接続文字列は次のようになります。 



  <プロパティ名= "connection.url"> jdbc：mysql：//127.0.0.1/openmeetings？autoReconnect = true＆useUnicode = true＆createDatabaseIfNotExist = true＆characterEncoding = utf-8 </ property>




結果のファイルは、hibernate.cfg.xmlとして保存する必要があります



フラッシュサーバー


それでは、Flashサーバーについて説明しましょう。 Red5のインストールは簡単で、高度な機能は必要ありません。 将来的には、プロジェクトをWebルートに配置するだけで済みます。これは、junction.exeというSysinternalsパッケージのユーティリティを使用したためです。 Unixシステムでは、lnと同じトリックを実行します。



> junction -s “D:\Program Files\Red5\webapps\openmeetings” “D:\workspace\openmeetings\dist\openmeetings”







これで、プロジェクトはWeb経由でアクセスできます（ファイアウォールについて忘れないでください！）。

最後に残っているのは、データベース内のテーブルとクライアントプログラムをインストールおよび構成することです。



サーバーへのプロジェクトのインストール


まず、JDKを指すようにJAVA_HOME環境変数を記述します。

テーブルのインストールは簡単です-Red5（インストールフォルダー内のred5.batファイル）を起動し、Webサイト127.0.0.1/openmeetings/installにアクセスします。すべてが正しく構成されている場合、ブラウザーにインストールページが表示されます。すべてのエラーがそこに表示されるはずです。 構成パラメーターを入力してインストールボタンを押した後、ログも確認する必要があります。適切な状況では、さまざまなエラーがそこに表示されず、インストールの進行状況が表示されます（数分かかります）。 インストールの最後に、あと1つだけ行うことがあります-クライアントを構成します。 つまり、既製のopenmeetingsを使用してダウンロードしたアーカイブからmain.lzx.swf8.swfという名前のファイルを抽出し、openmeetingsのルートに配置します。 ソフトウェアのバージョンによっては、main.lzx.lzr = swf8.swfに名前を変更する必要がある場合があります。空白の白い画面は、インストール後に127.0.0.1/openmeetingsを入力するときにそのような名前変更の必要性を示します。 入ってくるhtmlコードを見ると、スクリプトの名前を正確に簡単に確認できます。





ほとんどの場合、このような問題は発生しません。 さて、それで会議へようこそ！
