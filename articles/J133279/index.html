Windows 8のインストールは50％高速になります

Windows 8開発者は、企業ブログの別の記事を喜んでいます。 今回は、インストーラーの改善について話しました。 いつものように、すべての改善はユーザーの行動を徹底的に研究した後に行われました。以前のアップグレード手順はユーザーにとって複雑すぎるように思われました。



まず、Windows OSで初めて、インターネット経由でシステムイメージをダウンロードできる小さなインストーラーを作成しました。 おそらく、この意味でのマイクロソフトの開発者は、OS X Lionの例に触発されたのでしょう。 ダウンロードを簡単にするために、Windows 8のインストールイメージは1.51 GBに圧縮されました（Windows 7 x86は2.32 GBを占有しました）。



もちろん、システムはDVDまたはUSBからインストールすることもできます。 プロセスは可能な限り簡素化されました。アップグレードには、マウスを11回クリックするだけで済みます。



比較のため、Windows 7でのアップグレードスキームは次のとおりです。







マウスクリックの大部分は、アップグレードアドバイザーとWindows転送ツールモジュール（ファイルと設定を保存するためのウィザード）によって占められました。 Windows 8では、これら2つのモジュールは単一のインストーラーに統合されており、バージョン比較、ファイルと設定の保存、BitLockerなどの暗号化保護プログラムの無効化など、ほとんどの手順が自動化されています。 その結果、マウスのクリック数は82％、つまり11に減少しました。







Windows 7と比較して、「クリーン」インストールの時間は30分から20分に短縮されました。 開発者によると、144万のファイルと120のアプリケーションを含む「最も難しいアップグレード」でさえ、1時間もかからず、そのようなアップグレードは8〜9（！）時間続きました。







ご想像のとおり、このような大幅な削減は、アップグレード中のファイルの物理的な移動の拒否に関連しており、代わりにファイルへのリンクを更新するだけです（ハードリンク）。 システム管理者向けの詳細については、 Windows 8ブログを参照してください。
