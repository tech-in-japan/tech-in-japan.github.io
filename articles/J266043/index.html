Veeam Endpoint Backup FREEを使用したバックアップからの物理マシンの回復オプション

「彼らは何回も世界に語った」、それ自体で目的としてバックアップするのは理にかなっていないが、もちろん、バックアップから迅速に、正確に、そして簡単に回復することが可能であるなら、そうする。 したがって、今日の私の投稿のテーマは、 前のものに続いて、Veeam Endpoint Backup FREEを使用して作成されたバックアップから物理マシンを復元することです。

おそらく既にお勧めしたように、回復オプションはバックアップ設定と密接に関連しています。たとえば、ユーザーフォルダーのみがバックアップされた場合、マシン全体を復元できないことは言うまでもありません。

これらのオプションをより詳細に検討してください。catを歓迎します。









個々のファイルの回復



この機能は、適用されているバックアップモードに関係なく使用できます。 いくつかのファイルを復元するために、マシン全体をバックアップから持ち上げる必要はありません。 （何かを思い出すと、それを使用して個々のファイルを回復するために使用したすべてのVeeam Backup＆Replicationユーザーがそう思います。）

だから、どのようにファイルの回復を開始できますか：

  • [スタート ]メニューに移動し、[ Veeam ] > [ツール]> [ファイルレベルリストア ]を選択して起動します。
  • システムトレイで、アプリケーションのアイコンを見つけて右クリックし、[ 復元-個々のファイル]コマンドを選択します 。

    どちらの方法でも、リカバリポイントの選択専用のウィザードのステップに進みます。






注：必要に応じて、前の手順「 バックアップの場所」に移動して、 バックアップの場所を指定できます。 これは、たとえば、目的のファイルが別のマシンのバックアップにある場合に便利です。

他の方法もあります： Veeam Endpoint Backupコントロールパネルを開き、ダイアグラム上で必要なバックアップを選択します。バックアップはすべて長方形の形でグラフィカルに表示されます。 選択したバックアップに関する情報を調べて、それが回復したいものであることを確認したら、[ ファイルの復元]ボタンをクリックします。





次に何が起こりますか？



[ ファイルの復元]をクリックした後（またはウィザードの手順を実行して[ 開く ]をクリックすると）、バックアップファイルは現在実行中のマシンのファイルシステムのC：\ VeeamFLRフォルダーにマウントされ、ブラウザーを使用してバックアップの内容を表示できますVeeam Backup Browser（またはWindows Browser）。







Veeam Backup Browserは、リカバリ中に次のオプションをサポートします。

  • 上書き -ファイルは元の場所に復元されます。 そのようなファイルがすでに存在する場合、バックアップのファイルで上書きされます。
  • Keep-ファイルは元の場所にRestored-というプレフィックスを付けて復元され 、既存のファイルは上書きされません。
  • コピー先 -ファイルを別の場所にコピーできます。さらに、権限設定を同時に保存するかどうかを指定できます（ アクセス権と所有権の保持 ）。


便利： CTRL + SHIFTを押すと、複数のファイル/フォルダーを選択して一度に復元できます。



ボリューム回復



バックアップ中にPC全体オプション（マシン全体のバックアップ）またはボリュームレベルバックアップ （ボリュームバックアップ）を選択した場合にのみ、このような回復が可能になるという事実にすぐに注意を促します。

明らかに、回復はボリューム全体になります。

プロセスを開始する同様の方法があります：[ スタート ]メニューで[ Veeam]> [ボリュームの復元 ]を選択するか、トレイアイコンを右クリックして[ 復元-ボリューム全体]コマンドを選択するか、アプリケーションインターフェイスを開き、目的のバックアップを選択して[ ボリュームの復元]をクリックします。







ファイルと同様に、ボリュームは元の場所または新しい場所に復元できます。 最初のケースでは、元のボリュームが上書きされます。 2番目の場合、空きディスク領域または既存のボリュームの場所にボリュームを配置して、それらを置き換えることができます（つまり、データはバックアップから取得されることによって上書きされます）。

Veeam Recovery Mediaを使用せずにリカバリ中に発生するボリュームのリカバリに関するいくつかの制限に注意してください。

  • システムボリュームを元の場所に復元することはできません。
  • ページファイルを含む別のボリュームにボリュームを復元することはできません。
  • 回復が行われるバックアップファイルを含む別のボリュームにボリュームを復元することはできません。


これらの制限を回避するには、以前に作成したVeeam Recovery Mediaブータブルメディア（ ここで説明します ）からのリカバリを使用し、Veeam Bare Metal Recoveryベアメタルリカバリウィザードを実行してボリュームをリカバリします（詳細は後述）。 ウィザードの手順は非常に簡単です。 バックアップの場所に応じて、リモートストレージシステムまたはVeeam Backup＆Replicationサーバーを選択し、そこにドライブをマップする機能を取得します。 復元するボリュームを選択します-デフォルトでは、元の場所に復元されます（別のボリュームを指定するには、[ ディスクマッピングのカスタマイズ ]をクリックし、ボリュームの復元先を指定します）。







重要！ 場所を手動で指定する場合、既存のボリュームを消去しないように注意してください。 （実際には、リカバリ中にデータが直接上書きされるため、この段階でマッピングについて気が変わった場合は、「 キャンセル」をクリックするか、ウィザードウィンドウを閉じて、偶然に何も起こらないようにします。）

ディスクの手動マッピングには、次のオプションがあります。

  • バックアップレイアウトの適用 -バックアップとしてマークします。 ディスクは、バックアップが作成されたときと同じ方法でパーティション分割されます。
  • ディスクレイアウトの適用 - ディスクとしてマークします。 （指定した）他のドライブの現在のパーティション設定が使用されます。
  • 消去 -消去; 現在のディスクレイアウト設定は削除されます。


次に、リカバリに進むことができます。

スケジュールされたバックアップ



バックアップタスクのスケジュールに従って自動的に実行されます。バックアップタスクの設定は、バックアップの構成ウィザードで指定します。定期的な起動や起動トリガーのスケジュールを含めることができます。







タスクの最初のパスではフルバックアップが作成され、次のパスでは増分が作成されます。 作成された復旧ポイントは、指定された保存期間に従って保存および削除されます。

バックアップコピーの作成は、選択した保存場所が機能している限り機能します。 繰り返し起動（タスクの完了時にエラーが発生する可能性がある場合）のロジックは次のとおりです：障害が発生すると、タスクは次の23時間10分ごとに再起動を試み、その後エラーメッセージを表示してこれを停止します。

次のファイルがバックアップストレージに配置されます。

  • .VBM-バックアップチェーンに関するすべての情報を格納するメタデータファイル
  • .VBK-完全バックアップファイル
  • .VIB-増分ファイル




オンデマンドバックアップ



たとえば、新しいソフトウェアのインストールやシステムのアップグレードが必要な場合など、オンデマンドバックアップは便利です。 [ 今すぐバックアップ ]ボタンをクリックして、Veeam Endpoint Backupインターフェイスから起動します。







その結果、バックアップチェーンに新しい増分バックアップが作成されます（チェーンの先頭としてフルバックアップが作成される場合、例外は最初の起動です）。

スケジュールされたバックアップとは異なり、エラーが発生した場合、そのようなバックアップは再開されないことに注意してください。



個別の完全バックアップ



完全バックアップを作成しますが、チェーンの一部ではないため、保持ポリシーの対象にはなりません。

このようなバックアップの作成を開始するには、トレイアイコンを右クリックして、[ バックアップ]> [スタンドアロンフルバックアップ ]を選択する必要があります。 このようなプロセスは、エラーが発生しても再起動しません。 完了後、バックアップファイルはチェーンと同じ場所に保存されますが、別のサブフォルダーに保存されます。 その中に.VBMおよび.VBKファイルがあります-必要に応じて、それらから回復できます（バックアップでフォルダーを転送する場合、別のバックアップでこのフォルダーを転送することを忘れないでください）。



Veeam Backup＆Replicationとの統合



ここで、Veeam Endpoint Backup FREEとVeeam Backup＆Replicationの統合が、追加のリカバリオプションの点でどのように役立つかを見てみましょう。



ファイル回復



Veeam Backup＆Replicationコンソールから、 ドキュメントで詳細に説明されているFLR（ファイルレベルの復元、つまりファイルレベルの復元）メカニズムを使用して、バックアップからファイルとフォルダーを復元することは非常に便利です。 唯一の制限があります-元の場所に復元することはできませんが、バックアップブラウザの[ コピー先]コマンドを使用して、必要なファイルを新しい場所にコピーできます。



Veeam Explorerを使用したアプリケーションオブジェクトの回復



Veeam Endpoint Backup FREEを使用してActive Directory、SharePoint、Exchange、またはSQLサーバーをバックアップする場合、そのようなバックアップから対応するアプリケーションオブジェクトを復元できます：Veeam Backup＆Replicationコンソールからファイルレベルのリカバリを開始し、次に対応するバックアップブラウザーからVeeam Explorerツール。







これがどのように行われるかについては、例えばこちらをご覧ください 。



ディスクをエクスポートする



Veeam Endpoint Backupによって作成されたバックアップのディスクは、Veeam Backup＆Replicationコンソールを使用して、[ ホーム ]メニューから[復元]> [エンドポイント]> [ディスクの内容を仮想ディスクとしてエクスポート]を選択して、VMDK、VHDまたはVHDXファイルにエクスポートできます。







これはP2Vリカバリではないため、仮想環境でこのようなディスクから起動しようとすると、物理ドライバーが存在するためにエラーが発生する可能性があることを理解してください。 ただし、より実用的なシナリオがあります-たとえば、VMへのディスク接続。 ただし、ここでいくつかの点を覚えておく必要があります。

  • VMDKディスクは「シン」フォーマット（シンプロビジョニング）でストレージシステム（データストア）に復元できますが、サーバーへのリカバリではディスクを「シック」（シックプロビジョニング）として保存します。
  • VHD / VHDXドライブは常に動的として復元されます。
