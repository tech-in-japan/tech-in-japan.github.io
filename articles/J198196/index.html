Microsoft System Center向けにリリースされたVeeam Management Pack for VMware v6.5：新機能

Microsoft System CenterとVMware vSphereを同時に使用する場合、複数の監視ツールを同時に使用することはあまり便利ではないため、System Centerを介してVMwareを監視することをお勧めします。 System Centerは、異種の仮想環境で動作できます。

  • Operations Managerは、仮想マシン、ゲストオペレーティングシステム、およびそれらのアプリケーションを監視できます。
  • OrchestratorにはVMwareとの統合パッケージがあります
  • Virtual Machine Managerは、VMwareプラットフォームで実行されている仮想マシン上でさまざまなアクションを作成および移動することができます。


Veeam Management Pack for VMware（Veeam MP）は、Microsoft System Centerの機能を拡張して、VMwareの仮想インフラストラクチャを監視および管理します。 さらに、 新しいバージョン（v6.5）から、Veeam MPはVeeam Backup＆Replicationバックアップインフラストラクチャの監視を可能にします 。



Veeam MPは1,000を超えるメトリックとイベントを分析し、それらを便利なアラート、ダッシュボード、レポートに変換します。 System Centerはアプリケーションとオペレーティングシステムに広範な監視機能を提供しますが、Veeam MPでは「仮想ハードウェア」（VMware vSphere仮想化サーバー）、仮想マシン、およびすべてのデータセンターハードウェアリソース（ネットワーク、ストレージ、コンピューティングパワー）を監視できます。



Veeam MP（v6.5）の新しいバージョンが先週リリースされました。この投稿では、新機能について簡単に説明します。







監視および制御プロセスの耐障害性


vCenterに障害が発生すると、Veeam MP v6.5はvSphere仮想化サーバーに直接アクセスして自動的に動作モードに入ります。 もちろん、一部の機能は利用できない場合があります。 vCenterが復元されると、Veeam MPはvCenterを介して自動的にデータ収集モードに戻ります。



構成制御とイベント相関


「構成の追跡とアラートの相関」レポートを使用すると、VMware仮想インフラストラクチャで一定期間に発生した変更を確認し、原因分析を実行できます。 レポートには、仮想環境のすべてのオブジェクトに関する詳細データが含まれます。 クラスタ、データウェアハウジング、ネットワークリソース、ホスト、仮想マシン。





Veeam Backup＆Replication Backupインフラストラクチャの監視


現在（バージョン6.5以降）Veeam MPを使用すると、Veeam Backup＆Replicationバックアップインフラストラクチャの要素（バックアッププロキシ、バックアップリポジトリ、WANアクセラレータ、バックアップジョブなど）を監視できます。





Veeam B＆Rトポロジサポートは、VMwareトポロジサポートと同様に実装されます。





この製品では、リアルタイムの監視に加えて、バックアップとレプリケーションのインフラストラクチャに従って予測レポートと分析レポートを作成できます。 例：

  • Veeam B＆Rを使用してバックアップされた仮想マシンを確認できる「保護されたVM」レポート
  • バックアップジョブのステータスを示す「バックアップおよびレプリケーションジョブ」レポート
  • データウェアハウスの利用可能なディスク領域を時間の観点で示すストレージ計画レポート。バックアップリポジトリのディスク領域の終わりのイベントを事前に予測できます。
  • レポート「バックアップコピージョブの概要」、「オペレーターアクティビティの復元」など






新しいバージョンは、VMware vSphere 5.5、Microsoft Windows 2012 R2、およびMicrosoft SCOM 2012 R2をサポートしています。
