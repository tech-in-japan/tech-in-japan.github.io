Cloud-Vの仮想インフラストラクチャに関する話を続けます。 先週、新しい機能を開始しました-My.Cloud-Vポータルから直接仮想マシンのバックアップを作成します。

この新機能の便利さは、数回クリックするだけでバックアップ設定が行われることです。 さらに、仮想マシンを停止する必要はありません-バックアップは「ホット」に作成されます。

開始するには、舞台裏を見てサービスがどのように機能するかを確認し、コマンドバーからバックアップする方法を教えてください。

Cloud-Vバックアップサービスの仕組み



ポータルからバックアップ機能を追加するために、 Windows Azure Pack Developer Kitを使用しました。 これは、Azure Pack APIを操作するためのソースコード例のコレクションです。 このソリューションは、Windowsサーバー上の継続的なデータ保護のためのプログラムであるSystem Center Data Protection Managerに基づいています。



このCloud-V機能の拡張は、ポータルレベル、アプリケーションサーバー、バックアップサーバーの3つの部分から構成される構造です。





これが、Cloud-Vバックアップサービスの仕組みです。



管理ポータルのレベルでの拡張：クライアントおよび管理者ポータルでバックアップ要求を作成できます。



クライアントが「作成」ボタンを押すと、バックアップコピーの作成要求がポータルを離れます。







管理ポータルは、バックアップサーバーも管理します。







リクエストはサーバー間で分散されます。バックアップは負荷の低いサーバーに送信されます。 また、バックアップサーバーも管理します。







アプリケーションサーバー： Windows Azure Packのクライアントおよび管理ポータルからの要求を処理します。



アプリケーションサーバーは次の機能を実行します。



  • バックアップコピーの作成と削除、およびVMのチェックポイントへの復元要求を受け入れて処理します。
  • ジョブをキューに入れ、要求をバックアップサーバーに送信します。
  • コピーのステータスと関連性についてバックアップサーバーに問い合わせます。
  • 仮想マシンのクローンを作成します。 これについては、最初の記事で説明しました 。


System Center Data Protection Manager（SCDPM）：バックアップサーバー。 SCDPMサーバーは、VMのバックアップおよび復元要求を実行します。







バックアップサーバーに要求を送信した後、アプリケーションサーバーはSCDPMにタスクのステータスを要求し、クライアントポータルにステータスを表示します。







練習に移り、バックアップタスクを作成して管理する方法を説明しましょう。



バックアップジョブの作成



  1. [ 仮想マシン]セクションに移動し、目的のVMを選択して、矢印をクリックします。







  2. [ バックアップ ]タブを開きます。







  3. コマンドバーの [ 作成]をクリックします 。







  4. バックアップオプションを構成します。



    • バックアップを保存する日数を選択します。



      バックアップは継続的に作成されます。 ただし、LinuxベースのVMでは、VMを停止せずにバックアップするには最新の統合サービスが必要であることに留意する必要があります。 それらが欠落している場合、または古いバージョンの場合、VMは一時停止されます。







    • バックアップ時間を設定します。 指定された時間ごとに、新しい復旧ポイントが作成されます。







  5. できた！



    [バックアップ]タブには、作成されたすべてのVMバックアップ、バックアップ時間、保持期間、サイズ、およびステータスが表示されます。 バックアップの処理中にエラーが発生した場合、「アラート」セクションに表示されます。



バックアップ管理



バックアップはコマンドバーから管理できます。 次のアクションが可能です：手動バックアップ、選択したポイントへのVMの復元、バックアップジョブの変更または無効化。







ポイントの作成 -設定されたスケジュール以外でバックアップを作成します 。 更新をインストールする前など、VMに重要な変更を加える前に、新しいVM復元ポイントを作成できます。 これは、次のスケジュールされた復旧ポイントの作成を妨げるものではありません。









VMの復旧ポイントが予定外です。



復元 -選択したバックアップにVMを復元します。 元のVMは選択したポイントに上書きされ、停止します。







タスクの変更 -バックアップルールの変更：バックアップストレージ時間とバックアップスケジュール。







停止 -バックアップを無効にし、現在使用可能なすべての復旧ポイントを削除します。
