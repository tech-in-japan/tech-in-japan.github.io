物理バックアップ：Veeam Endpoint BackupはVeeam Agent for Microsoft Windowsに置き換えられました

秋に戻って、 Windowsを実行している物理マシンのバックアップ用の更新されたソリューションをリリースする計画について話しました 。 多くの人によく知られているVeeam Endpoint Backup FREEは、Microsoft Windows 2.0用のVeeam Agentに進化し、ベータ版がダウンロード可能になりました。 （投稿の最後で入手方法を確認できます。）



Veeam Endpoint Backupがユーザーコンピューターのバックアップを目的とした場合、Veam Agent for Windowsは企業コンピューター（ワークステーションおよびサーバー）のバックアップ機能を拡張しました。 何があなたを待っていますか？ 答えは猫にようこそ。







まず、無料版を含む3つのエディションすべてに主要な機能が含まれます。 それらの中で：



  • Windows 2016サポート
  • ソース側の暗号化
  • アクティブなフルバックアップを作成するためのスケジュールを構成する機能


さらに、Veeam Backup 9.5 Free Editionを使用した新しい回復オプションがあります-Veeam Agent for Windowsによって作成されたバックアップから、Microsoft Hyper-Vを実行する仮想マシン（その無料バージョンを含む）、またはクラウドに回復することが可能になりますMicrosoft Azure 目の前で開かれる一連のスクリプトは、本当に幅広く多様です。これには、マシンのディスクのコンテンツを仮想ディスク形式にエクスポートし、クラウドに復元し、ゲストOSファイルを復元することが含まれます。 当然、それらはすべて、Veeam Backup＆Replication 9.5の商用版でサポートされています。







無料版



Veeam Agent for Windows 2.0 FREE Editionの新製品について詳しく説明します。

Veeam Agentがソース側で実行するデータを暗号化するには、適切なオプションを選択する必要があります-これは、ジョブウィザードの追加のバックアップストレージ設定のステップで[ バックアップファイルの暗号化を有効にする] チェックボックスです。 次に、パスワードとヒントを入力します。







重要！ バックアップコピーの保存にVeeam Backup＆Replicationリポジトリを使用する場合、Veeam Agentによって作成されたバックアップの暗号化設定を個別に設定することはできません。この場合、バックアップリポジトリの設定はVee​​am Backup＆Replication管理者の特権であるためです。



設定の編集時に既存のジョブの暗号化を有効にした場合、Veeam Agentを次回起動すると、アクティブなフルバックアップが作成され、その後増分バックアップのみが作成されることに注意してください。 それらはすべて、指定されたパスワードで暗号化されます。



便利：パスワードを誤って紛失したり忘れたりした場合は、このダイアログボックスで新しいパスワードを設定できます。 その後、ファイル（古いパスワードで暗号化されたものを含む）からバックアップチェーンを復元するときに使用されます。



アクティブなフルバックアップの作成と一連のファイルの保存、および圧縮と重複排除の設定については、次のいずれかの投稿で詳しく説明します。



ワークステーションおよびサーバーの商用版





まず、WorkstationまたはServerエディションの商用ライセンスを購入することにより、24時間365日の資格のあるテクニカルサポートを受けることができます。

サーバー版：



  • スケジュールされたタスクは自動的に開始します
  • アプリケーション（Microsoft Exchange、Active Directory、SQL Server、SharePoint、Oracleを含む）の動作を考慮したバックアップを実装しました
  • データベースサーバーの場合、データベースログのバックアップもサポートされます。これにより、回復中の最適なRPOパフォーマンスが保証されます。






頻繁に旅行するが、ラップトップのデータセキュリティを確認したい人のために、 バックアップキャッシュの新しい機能が提供されています。 バックアップのメインストレージの場所が一時的に取得できない場合、それを使用して、スケジュールに従って作成された増分バックアップをローカルフォルダーに保存できます。 次に、メインストレージが再びアクセスゾーンにあるとすぐに（たとえば、企業ネットワークに接続されたとき）、この「バックアップキャッシュ」の内容はメインストレージの場所と同期されます。 コンソールで既に同期されたバックアップには、雲のアイコンとマークが付いています：







注矢印付きのクラウドは、データが同期していることを意味します。 リカバリポイントにアイコンがない場合、この場合、バックアップがキャッシュに保存されていることを意味します。



しかし、オフィスを訪問せずにホームオフィスで働いたり、常に旅行している人はどうでしょうか。 または、企業ネットワークの境界の外側でパブリッククラウドを使用する人は？ 彼らにとって、Microsoft Windows用のVeeam Agentの商用版は、Veeam Cloud Connectクラウドリポジトリにバックアップする機能を提供します。これは、たとえば、Veeamサービスプロバイダーの1つのサイトまたは主に組織のデータセンターで機能します。



さて、「デザート用」：商用版は、エージェントと連携するためのAPIを提供します。



  • サーバーバージョンの場合、Veeam Availability Consoleと同時に完全なAPIを利用できます。
  • Workstationバージョンでは、Veeam Agent Configuratorが提供され、コマンドラインを使用してVeeam Agent for Windowsライセンスをエクスポートおよびインポートおよび構成できます。


特に、Veeam Agent Configuratorを使用すると、エージェント自体とバックアップタスクの両方の設定を含むエージェント構成をXMLファイルにエクスポートできるため、このファイルを使用して新しいエージェントを自動的にインストールおよび構成できます。 これは、ログオンスクリプト、グループポリシー、System Center Configuration Managerなど、インフラストラクチャで機能するメカニズムを使用して実行できます。
