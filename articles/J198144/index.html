心配するのをやめて、Hyper-Vサーバーに恋をした方法



最近、私は「ユーザーの肌に慣れる」タスクを行い、無料のMicrosoft Hyper-V Server 2012で実際にどのように機能するかをテストしました。









はい、はい、無料のチーズはネズミ捕りだけでなく、マイクロソフトにもあります。 サーバーまたは少なくとも古いラップトップを仮想化したいが、ハイパーバイザーに余分なお金をかけたくない場合は、Hyper-Vサーバーを試す必要があります。 なんで？ 第一に、それは本当に無料です-「エンタープライズ」機能も含め、すべての機能はユーザーがすぐに予約なしで利用できます。第二に、大企業によってサポートおよび開発され、第三に、古き良き機能ですWindowsは、Server Coreにわずかにトリミングされていますが。









カットの下で、Hyper-Vサーバーをインストールして構成し、Linuxを使用してこのプラットフォームの制限をエレガントに回避する方法についてのストーリーがあります。









設置



すべてが非常に標準的なため、何も書く必要はありません。 ここから、自慢のリンク「無料ダウンロード」を介してISOイメージを取得します。









インストールプロセス全体は数分かかり、管理者への標準プロンプトで終了します。 パスワードを設定すると、異常に裸のデスクトップが開き、2つのコンソールウィンドウが開きます。













原則として、考えてみれば、それほど怖くないわけではありません。結局、サーバーのタスクは仮想マシンを実行することであり、ソリティアは他の場所に拡張することができます。









構成：システム



青いコンソールを使用して、名前、ネットワークアドレス、RDPの有効化、自動更新などの必要なパラメーターを設定します。









システムの更新について言えば、Server Coreへのトリミングは、攻撃対象を減らすため、つまり、脆弱性を含む可能性があるため定期的にパッチを適用する必要があるサービスおよびソフトウェアコンポーネントの数を減らすために行われました。 また、パッチのインストールにはサーバーの再起動が必要になることが多いため、更新されたサーバーコンポーネントの数を減らすと、セキュリティが向上するだけでなく、可用性も向上します。









構成：Hyper-V



何らかの理由で、インストール中に仮想スイッチが作成されないため、デフォルトでは仮想マシンは外部と接続しません。 修正しましょう-物理アダプターへの簡単なスイッチブリッジを作成します。









2番目のコンソールでpowershellを実行します。 はい、サーバーをコンソールプログラムにトリミングすると犠牲になりますが、私たちは高貴な目標を追求します（上記を参照）。









1）アダプターが表示される名前を確認します。 これを行うには、コマンドGet-NetAdapterを実行します 

PS C:\Users\Administrator> Get-NetAdapter Name InterfaceDescription ifIndex Statu s ---- -------------------- ------- ----- Ethernet 5 Broadcom BCM5708C NetXtreme II Gi...#47 16 Up Ethernet 4 Intel(R) PRO/1000 PT Dual Port Ser...#2 15 Di... Ethernet 3 Intel(R) PRO/1000 PT Dual Port Serve... 14 Di... Ethernet 2 Broadcom BCM5708C NetXtreme II Gi...#49 13 Up


2）イーサネットアダプターに接続された仮想スイッチを作成する5 

 PS C:\Users\Administrator> New-VMSwitch -Name "bridged" -NetAdapterName "Ethernet 5" Name SwitchType NetAdapterInterfaceDescription ---- ---------- ------------------------------ bridged External Broadcom BCM5708C NetXtreme II GigE (NDIS VBD Client) #47


仮想マシンを作成する準備がすべて整いました。









仮想マシンの作成



ほとんどの場合、オペレーティングシステムをインストールするためにISOイメージが必要になります。Windowsの場合は通常の方法でコピーします。サーバー上のドルボールを介してネットワーク経由でコピーします。



今、車を作成します。 







 PS C:\Users\Administrator> New-VM -Name "test1" -Path C:\vms\test1 -MemoryStartupBytes 2GB -NewVHDPath C:\vms\test1\disk1.vhdx -NewVHDSizeBytes 40GB -SwitchName "bridged" Name State CPUUsage(%) MemoryAssigned(M) Uptime Status ---- ----- ----------- ----------------- ------ ------ test1 Off 0 0 00:00:00 Operating normally


ネットワークアダプターの種類の選択にはわずかな微妙さがあります。オペレーティングシステムがWindows 2012に該当する場合は、標準アダプターを選択できますが、「レガシー」タイプのアダプターを選択する方が良い場合があります。 これは、-IsLegacy 1キーを使用してアダプターを仮想マシンに追加することで実行できます 







 Remove-VMNetworkAdapter -VMName test1 Add-VMNetworkAdapter -VMName test1 -IsLegacy 1 -SwitchName «bridged»


ISOイメージをデフォルトのDVDドライブに挿入します 







 PS C:\Users\Administrator> Set-VMDvdDrive -VMName test1 -Path C:\iso\en_windows_server_2012_vl_x64_dvd_917758.iso


将来-ディスクを仮想ドライブから削除する必要がある場合は、Set-VMDvdDrive -VMName <マシン名> -Path $ nullを実行する必要があります









まあ、並べ替え、すべて...実行できます！ 







 PS C:\Users\Administrator> Start-VM test1


うーん、私はPowerShellウィンドウの上部で進行状況を実行しました。明らかにすべてが始まったようです。 Get-VMチームを確認しましょう 







 PS C:\Users\Administrator> Get-VM test1 Name State CPUUsage(%) MemoryAssigned(M) Uptime Status ---- ----- ----------- ----------------- ------ ------ test1 Running 0 2048 00:01:30 Operating normally


すべてが順調です-マシンは動作しています。









ただし、画面の見方は？！



ちょっと待ってください...しかし、インストール画面を見る方法、マウスでどこを突くか？ 一般的に-コンソールはどこにありますか？ ローカル-方法：サーバー上ではありません！









マイクロソフトが提供するのは、クライアントステーションにRSAT（リモートシステム管理ツール）をインストールできることです。 これらのツールを使用して、仮想マシンのコンソールに接続し、そこから、mmcスナップインHyper-Vを介して仮想マシンを作成および実行できます。 ただし、興味深い機能があります。









  • RSAT for Windows 8はWindows 2012でのみ機能します
  • RSAT for Windows 7 Sp1はWindows 2008 R2 SP1でのみ動作します


私の意見では、サーバーバージョンの変更によりクライアント軸を変更することは問題ではありません！ Microsoft製品マネージャーはどこを見ていますか？！ どうすればコンソールにアクセスできますか？









アイデアは彼らが待たなかったところから生まれました... Linux仮想マシンをRDPプロキシとして使用してみませんか？









みんなへの贈り物



その結果、RDP経由でアクセスしてログインできるCentOSベースの仮想マシンを作成し、この仮想マシンまたはネットワーク上のHyper-Vサーバーで簡単なインタラクティブスクリプトでコンソールを実行しました。









このマシンのイメージ （371MB）をダウンロードして 、Veeam Backup＆Replicationの無料版を使用してHyper-Vサーバーに展開できます。 rootユーザーのパスワードは123qweASDです。 パスワードを変更するには、まずpasswdを実行し、次にvncpasswdを忘れないでください。









マシンを起動した後、Get-VMサーバーでこのコマンドを実行することにより、このマシンのIPアドレスを見つけることができます<マシン名> | -ExpandProperty NetworkAdaptersを選択します| ipaddressesを選択します。 ネットワーク上にDHCPサーバーが必要です。 







 PS C:\Users\Administrator> get-vm rdpproxy | select -ExpandProperty NetworkAdapters | select IPaddresses IPAddresses ----------- {192.168.1.8, fe80::215:5dff:fe01:6703}


RDPクライアントを使用して、このマシンに移動し、Hyper-Vサーバーを追加し、仮想マシンを選択して...voilà-これでオペレーティングシステムのインストールを続行できます













仕組み



主な作業は、FreeRDPのクールな人たちによって行われました。 いいえ-まあ、彼らはMicrosoftよりもRDPプロトコルをよりよくサポートしていたので、本当にクールです。 RemoteFX、アドイン、キャッシュ、リダイレクトオーディオおよびクリップボードのサポートがあります...一般に、Windowsの異なるバージョンだけでなく、このために特別に開かれたHyper-Vサーバーを介して仮想マシンに直接接続できるすべてのものポート2179。









Hyper-Vを使用するために必要な最新のプロトコルはすべて、ベータバージョン1.1.0のみです。 Ctrl + Alt + DelをCtrl + Alt + Insertの組み合わせに送信するように再定義するためにバージョンを少し変更しました。そうしないと、Linuxプロキシがそれを最初に処理し、Windows仮想マシンに入る方法がありませんでした。 正直な人として、私はソースをここに投稿しました： https : //github.com/VeeamSoftware/FreeRDP









RDPサーバーとして、xrdpが使用されます。









ホストをポーリングしてコンソールを起動するためのスクリプトは、Bashで記述されています。 私はそれについて初めて書いた-私はずさんに寛大さを求めます。









仮想マシンを実行するホストとその識別子の調査は、WMIを介して行われます。 Samba 4のライブラリに基づいて作成されたwmicユーティリティに感謝します。









CentOS 6.4には、Integration Servicesの一部が組み込まれており、IPアドレスのみが必要な情報をホスト仮想マシンに通知します。









未来についての何か



あまり良いニュースではありませんが、金曜日にHyper-V Server 2012 R2の新しいバージョンのみがリリースされ、プレビューを開始しようとしましたが、CentOS統合サービスはIPアドレスをホストに転送できませんでした。 どうやらMicrosoftは下位互換性なしで何かを再び変更/拡張したようです。 Linux用のIntergation Servicesを少し待って、このバージョンをサポートします。









おわりに



Hyper-V Serverは、小規模企業のIT開発を開始するのに適したシステムです。 予算が限られている場合は、無料で十分に助かります。 そして将来、このハイパーバイザーにより、追加費用なしで「エンタープライズ」機能を簡単に追加できるようになります。新しいサーバーを追加して電力を開発し、より高度なネットワークスキームを導入し、クラスタリングを通じてフォールトトレランスを提供するなど









このストーリーとアプリケーションが、最初のステップからこのプラットフォームを効果的に使用するのに役立つことを願っています。
