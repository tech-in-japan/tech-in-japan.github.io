IntelliJ IDEA 2016.1をご覧ください

先週、次のメジャーアップデートであるIntelliJ IDEA 2016.1をリリースしました。 以前に、利用可能な改善点について詳しく説明しました。この投稿では、それらの短いリストのみを提供し、新しいビデオへのリンクを提供します。もちろん、コメントでご質問にお答えさせていただきます。







主な改善点の中で：



  • デバッガー

    • Evaluate Expression and WatchesのGroovy式を使用してJavaコードをデバッグできるようになりました
    • 複数のスレッドのより便利なデバッグ
    • ソースコードをコピーする際の警告が一致しない


  • バージョン管理システム

    • Gitワークツリーのサポート
    • 単語レベルで変更を強調表示することにより、より便利なマージと比較


  • エディター

    • 式を左右にシフト
    • 静的メソッドと定数の自動インポート
    • 右から左に記述する言語（アラビア語やヘブライ語など）のサポート


  • グラドル

    • カスタムソースセットのサポート：各ソースセットは個別のモジュールで表され、独自の依存関係を持つことができるようになりました（その結果、膨大な数の問題が修正されました）
    • EAR成果物の自動構成


  • Java 8

    • インラインメソッドとシグネチャの変換メソッドの参照をラムダ式に変換
    • メソッド参照、ラムダ式、オプションタイプの使用を検証する検査
    • FluentIterable、Function、Optional、Predicateを使用したコードのGuavaライブラリから適切なJava 8 APIへの移行を支援する検査


  • コトリン

    • Kotlin 1.0のサポート
    • Kotlin Interactive Learning用のKotlin Eduプラグイン


  • スカラ

    • オートコンプリートでは、文字の種類、コンテキスト、およびその使用履歴が考慮されるようになりました。
    • パターンマッチング式の自動補完で、ケースプロパティクラス名が提案されるようになりました。


  • Javascript

    • ECMAScript 6、TypeScript、AngularJS 2の改善されたツールチップ
    • 非同期コードの便利なデバッグ


  • Springフレームワーク

    • Spring Bootの拡張ツールチップ機能
    • Spring MVCのサポートの改善
    • 多くの既知の問題が修正されました。


  • タイムリーフ

    • カスタム方言のヒント
    • Thymeleaf 3.0のサポート


  • Android Studio 1.5

    • リークプロファイラー
    • 新しいリントチェック


  • Docker

    • Docker Machineのサポート
    • 別ウィンドウ（ツールウィンドウ）
    • 右側のパネルに、環境変数とポートバインディングを管理するためのログとインターフェイス要素が表示されるようになりました


























さらに、IntelliJプラットフォーム全体がJava 8に移行しました。Java8を使用してプラグインを開発できるようになりました。



IntelliJ IDEA 2016.1以降、すべてのJetBrains Toolbox製品のバージョン管理スキームが変更されています。 現在、1年に1つの大きなリリースではなく、いくつかの同等のリリースをリリースする予定です。 ここで 、新しいバージョン管理が機能する理由と方法について詳しく読むことができます 。



JetBrainsで新しいIntelliJ IDEAをダウンロードできます。
