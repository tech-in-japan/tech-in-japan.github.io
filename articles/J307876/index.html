新しいAzureスタック



ローカルデータセンターでAzureを実行するための基本的な機能は、数年前に登場しました。 新しいAzureスタックには、Microsoftパブリッククラウドと連携するための多くのオプションがあり、パートナーに多くの新しい機会を提供します。 新しいAzureスタックは何を提供しますか？



この製品の主な目的は、通常同時に多数のサーバーを購入する大企業です。 このような企業の場合、Azure Stackはインフラストラクチャを独自に構築する必要性を置き換え、ターンキーソリューションを提供します。



通常、顧客はいくつかの基本的なサービス要件を提示します。 手始めに、彼らは展開と操作の複雑さに対処したくないが、アプリケーションと新機能の操作に集中することを好む。 これは多くの研究でアナリストによって述べられています。 また、顧客は、最新のイノベーションをすべて把握し、それらが登場するとすぐに、手頃なAzureサービスとツールを受け取りたいと考えています。 インフラストラクチャレベルでいくつかの構成オプションを犠牲にして、仕事で使用できる新しいツールをすばやく入手する準備ができています。 また、システムの品質と信頼性も必要です。 それは「うまくいく」ソリューションです。 スイープ後だけでなく、新しい更新が表示されたときも同様です。



これらのニーズを満たすために、Azureスタックは、1つのソリューションでソフトウェア、ハードウェア、サポート、サービスを組み合わせたターンキー統合システムとして提供されます。 Microsoftとパートナーは、スタック全体の特別に開発された管理システムのおかげで、プロビジョニングの全サイクル（展開中、新しいサービスとツールの追加、更新など）を通じて高品質のサービスを提供できます。



マイクロソフトは、パートナーと顧客が柔軟に機器を選択できるように努めています。 これを行うために、企業は、Dell、HPE、Lenovoなどのメーカーと協力しており、その参加により、実稼働環境向けの統合システムが作成されます。 2017年半ばから、Microsoftはこれらのパートナーと共にAzure Stackを備えた統合システムをリリースする予定です。 そして、より遠い将来に、彼らのリストを拡大してください。



したがって、これらの変更のおかげで、顧客はクラウドでの作業を改善できます。

Azureでの仮想マシンと物理マシンのレプリケーション



Azureは、Azure Site Recoveryを使用して、AzureのローカルVMware仮想マシンまたは物理WindowsまたはLinuxサーバーを複製できるようになりました。 この場合、ユーザーは、リソースを作成して操作するための2つの展開モデル、Azure Resource Manager（ARM）モデルとクラシックモデルを持っています。 さらに、Azureには2つのポータルがあります。クラシックデプロイメントモデルをサポートするクラシックAzureポータルと、両方のデプロイメントモデルをサポートするAzureポータルです。 AzureポータルでSite Recoveryを使用すると、次のことができます。



AzureポータルのSite Recoveryは、多くの新機能を提供します。



-Azure Backup ServiceとAzure Site Recovery Servicesは、リカバリサービスの1つのリポジトリに統合されています。 このようにして、ビジネス継続性と災害復旧をセットアップし、これらの機能を1か所から管理できます。 単一のダッシュボードで、オンプレミスサイトとAzureパブリッククラウド間の操作を監視および管理できます。



「クラウドソリューションプロバイダーのプログラムの一部として働く準備ができているAzureサブスクリプションのユーザーは、AzureポータルでSite Recovery操作を管理する機会があります。

-AzureポータルのSite Recoveryサービスを使用して、コンピューターをARMストレージアカウントに複製できます。 フェールオーバー時に、Site RecoveryはAzureにARMベースの仮想マシンを作成します。



-Site Recoveryは従来のストレージアカウントへのレプリケーションを引き続きサポートします。 フェールオーバー時に、Site Recoveryはクラシックモデルを使用して仮想マシンを作成します。



企業にとって、このツールを使用する利点は次のとおりです。



-Site Recoveryは、VMwareの仮想マシンと物理サーバーで実行される企業のワークロードとアプリケーションのリモート保護を提供します。



-回復サービスポータルは、レプリケーション、フェールオーバー、および回復を構成、監視、および制御するための単一の場所を提供します。



-Site Recoveryは、VMware仮想マシンを自動的に検出し、vSphereノードに追加できます。



-Azure Site Recoveryを使用すると、ローカルインフラストラクチャからAzureへの単純なフェールオーバーを実行し、AzureからローカルサイトのVMware仮想マシンサーバーへの転送で配置（回復）を復元できます。



-複数の仮想マシンの展開を有効にし、レプリケーショングループを作成して、複数のコンピューター間で分散されたアプリケーションワークロードが同時に複製されるようにすることができます。 レプリケーショングループ内のすべてのコンピューターには、フォールトトレラントで一貫したフェールオーバーポイントがあります。 フェールオーバーの場合、複数のコンピューターをまとめて復旧計画にまとめて、分散ワークロード全体に対してフェールオーバーを実行できます。

Azure用Check Point vSEC



Check Point Software Technologiesは、Microsoft Azure用の人気のあるVsecセキュリティシステムを導入しました。 Azureクラウドインフラストラクチャで信頼性の高い保護を提供し、ネットワーク攻撃とデータ漏洩を防ぎ、パブリッククラウド環境でAzureへの安全な接続を提供することを目的としています。 このシステムは、Microsoft Azureクラウドに対するさまざまな脅威に対する包括的な予防策を提供し、仕事でモバイルデバイスを使用するユーザーを含むユーザーに安全な接続を提供します。 また、悪意のあるソフトウェアや、いわゆるゼロデイ脅威（アンチウイルスデータベースにはまだ含まれていない最新のウイルスやトロイの木馬）に対する信頼性の高い保護もあります。 さらに、同社は特定のビジネスのニーズに合わせた個人用保護具を提供しています。 これらのツールには、ファイアウォール、アネチウイルス、アンチスパム、アンチボットなどがあり、データの損失を防ぎ、VPNで動作します。 これらの資金を管理するために、クラウドでの作業と独自のインフラストラクチャでの使用の両方に適した集中型の独自ソリューションが提供されます。



vSEC保護の展開は非常に高速です。 この手順は文字通りボタンを押すだけで実行され、特定の知識やサードパーティの専門家の関与は必要ありません。 このシステムは、2つのモデルで購入できます。永続的（合意された期間）と一時的使用です。 2番目のケースでは、顧客はシステムがその機器でアクティブになっている時間に対して直接支払います。 この場合、充電は1時間ごとです。



NetApp Private Storage for Azure



NetAppは、Microsoft Azure用のハイブリッドストレージシステムを導入しました。 これはNetApp Private Storageと呼ばれ、パブリッククラウドの柔軟性と費用対効果を、特殊なエンタープライズクラスのストレージシステムのパフォーマンス、可用性、セキュリティと組み合わせます。 このシステムを使用して、情報はセキュリティレベルがTier 1の世界最大のデータセンターEquinixの1つに配置されます。これにより、物理レベルでのデータ保護とさまざまなサイバー脅威に対する保護が保証されます。



このシステムのもう1つの利点は、その高いパフォーマンスです。 Azure ExpressRouteテクノロジーによって実現されます。 これを使用してNetAppストレージをAzure環境に接続すると、VPNおよびインターネットと比較してスループットが36％向上します。



NetApp Private Storageを使用すると、ユーザーはマルチクラウド環境で作業し、クラウドサービスプロバイダー環境間でデータを移動できます。MicrosoftSystem CenterとNetApp PowerShell Toolkitを使用すると、プライベートクラウドとAzureに接続されたプライベートストレージ間でデータを管理および転送できます。
