何を追加しますか？



ディープラーニングの適用は、新しいマントラ「すべてのアプリケーションでの深層学習」が次の10年で現実になる可能性があるほどの割合に達しました。 立派なDBMS SQL Serverは脇にありませんでした。 「SQL Serverでディープラーニングアルゴリズムを使用できますか？」という質問に対して、自信を持って「はい」と回答しました。SQLServerの新しいリリースのパブリックプレビューで、SQL Server 内のRサービスを大幅に改善し、開発者が使用する機械学習ツールの強力なセットを追加しましたマイクロソフト自体。 このアプリケーションを介して、SQL Serverに基づいて構築されたDBMSは、速度、パフォーマンス、およびスケーラビリティを向上させるツール、および機械学習やディープニューラルネットワークの他の新機能へのアクセスを受け取ります。 最近、SQL Serverが機械学習ベースのモデル管理システムとして使用された場合に1秒あたり100万を超えるR予測を実行する方法を示し ました。GitHubWebサイトでSQL ServerのRコード例と機械学習パターンを評価することをお勧めします。



このブログでは、SQL Serverのディープラーニングに関する「what 」、「why」 、および「how」の質問について詳しく説明します。 これを明確にすると、SQL Serverなどの強力なデータ処理プラットフォームを使用して、データ駆動型の人工知能の見通しを簡単に確認できます。



追加する理由



今日、すべての企業とすべてのアプリケーションがデータを処理しています。



データストレージとインテリジェンス（人工知能、機械学習システム、またはその他のインテリジェントシステム）を組み合わせたアプリケーションは、インテリジェントと呼ぶことができます。 SQL Serverは、わずか数行のコードを記述することで、開発者とユーザーがアプリケーションでディープラーニングの本質を理解するのに役立ちます。 さらに、データベース開発者はこれを使用して、組み込みの深層学習モデルを備えたミッションクリティカルなオペレーティングシステムを展開できます。 この記事では、SQL Serverがディープラーニング機能を必要とする10の理由をまとめました。



SQL Serverがディープラーニング機能を必要とする10の理由 。



  1. ストレージシステム（SQL Serverなど）でマシンインテリジェンスを使用すると、セキュリティ、コンプライアンス、プライバシー、暗号化、 マスターデータ管理サービス 、アクセシビリティグループ、高度なビジネスインテリジェンスツール、インメモリデータ処理テクノロジー、仮想化、地理空間、一時的、グラフィック、その他の素晴らしい機能。
  2. リアルタイムトランザクション処理（OLTP）および対話型分析処理（OLAP）と同様の「リアルタイム」または「バッチ処理」モードで作業できますが、ディープラーニングとマシンインテリジェンスに関するものです。
  3. ディープラーニングを最大限に活用するために、SQL Serverベースのアプリケーションを変更する必要はありません。 さらに、多くのモバイル、IoT、およびWebアプリケーションは、コードを複製する必要なく、同じディープラーニングモデルを使用できます。
  4. 機械学習ライブラリ（MicrosoftMLなど）の機能を使用して、データ処理の専門家、開発者、DBA、およびビジネス全体の効率を高めることができます。 これは、同じことを自分で行うよりもはるかに高速で効率的です。
  5. 現在の要件に合わせて展開またはスケーリングする予測可能なソリューションを開発できます。 最新のSQL Server Service Packでは、これまでEnterprise Editionでしか使用できなかった多くの機能が、Standard / Express / Web Editionなどの他のSQL Serverエディションでサポートされるようになりました。 つまり、SQL Serverの標準バージョンでも追加費用なしでディープラーニングを使用できます。
  6. （PolyBaseを使用して）外部データの異種ソースを使用して、深いモデルから情報をトレーニングおよび抽出できます。
  7. さまざまな状況をシミュレートして、SQL Server内でwhat-ifシナリオを作成し、さまざまな深層学習モデルに使用できます。 したがって、トレーニングデータが非常に限られている場合でも、インテリジェントなソリューションを得ることができます。
  8. ストアドプロシージャとトリガーを使用して、ディープラーニングモデルをすばやく簡単に展開できます。
  9. 必要なすべてのツール、監視およびデバッグツール、およびマシンインテリジェンスに適用可能なSQL Serverエコシステムを入手できます。 SQL Serverは、データとともにDNNモデルのライフサイクル全体を制御する機械学習管理システムになります。
  10. トランザクションのワークロードを増やすことなく（ HTAPテンプレートを使用して ）新しいデータを作成し、保存されたデータの分析を取得できます。


実際、データベースを管理するためだけにだれもがDBMSを購入することはまずありません。 バイヤーは提供する重要な機会です。 SQL Serverにディープラーニング機能を搭載し、人工知能と機械学習を従来の意味（データのスケーリング、帯域幅、待機時間）と生産性（実装の簡素化と学習曲線のフラット化）の両方でスケーリングできます。 これは、時間、ユーザーインタラクション、生産性、コスト削減と利益成長、新しい機会、ビジネスの見通し、業界におけるイデオロギーリーダーシップなど、さまざまな方法ですぐに貴重な結果をもたらします。



実際には、SQL Serverのディープラーニングテクノロジーは、銀行、金融、医療、製造、小売、電子商取引、モノのインターネット（IoT）システムで使用できます。 そして、これらのテクノロジーを使用して不正を検出し、病気の発生を予測し、エネルギー消費を予測し、個人情報を分析することで、既存の産業とアプリケーションが改善されます。 また、顧客関係管理（CRM）、エンタープライズリソースプランニング（ERP）、データウェアハウス（DW）、リアルタイムトランザクション処理（OLTP）など、SQL Serverでの負荷の開始に関係なく、ディープラーニングのテクノロジーを簡単に使用できます。 この場合、個別にではなく、SQL Serverが有名なすべての種類のデータおよび分析と組み合わせて使用​​することについて話します（たとえば、構造化データ、地理空間データ、グラフィックデータ、外部データ、一時データ、およびJSONデータの処理）。 そしてここにあなたのアイデアを追加するだけです。
