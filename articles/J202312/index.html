どのように行動するか



Qt 5.1では、低レベルのQThreadクラスは私たちの目的のためのものです。 彼について次のように言われています：「QThreadクラスは、プラットフォームに依存しない方法でスレッドを管理する機能を提供します。」 Schleeには素晴らしい本がありますが、ここに非常に紛らわしいスレッドの例があります。QThreadクラスを継承し、run（）メソッドをオーバーライドし、その中で作業を行うことをお勧めします。 これは、「開始、機能の実行、完了」のスタイルのタスクには悪いことではありませんが、より複雑なケースでは明確に受け入れられません。

一般的に、私はそれを聞いて読んだとしても無駄で、すぐにドキュメントを掘り下げる必要がありました。 ちなみに、良い例があります。 正しいパスを示しています： QObject :: moveToThread（QThread * thread）関数 、このオブジェクトとそのすべての祖先の親和性 （類似性-英語の親和性）をスレッドthreadに転送します。

したがって、最初の近似では、問題の解決策は次のとおりです。

  1. スレッド作成-QThreadクラス。
  2. オブジェクトを作成し、それを新しいストリームに転送します。
  3. 信号スロット通信の設置。
  4. 特定の優先度でストリームを開始します。


すべてうまくいくようですが、覚えていますか？ -作成されたオブジェクトのコンストラクターを新しいスレッドで実行したい。 これを行うには、スレッドの開始後に開始する必要があります。 最初にオブジェクトを作成し、次にストリームを作成できます。 ただし、オブジェクトのコンストラクターによって作成されるすべてのものは、新しいスレッドではなく、 現在のスレッドのスタック（ヒープ）に配置されます。 このエコノミーのすべてを新しいスレッドにうまく転送し、古いスレッドで削除することはできますが、...新しいスレッドで既にコンストラクターを呼び出す方が簡単です。 したがって、ここに問題番号1があります。 決める必要があります。

次に、問題番号2がありました。 QObjectから継承した素敵なテンプレートを作成しました。これは、信号スロット通信に必要です。 そして、 バイクが浮上しました。「MOCでは、C ++のすべての機能を使用できません。 主な問題は、 クラステンプレートにシグナルまたはスロットを含めることができないことです。 ＃@＆*！

私は次のクラスを思いついた：

  1. ネイティブクラスはTです。
  2. オブジェクト作成クラス-CreatorBase（QObjectの子孫）があります。 仮想メソッドを呼び出すことにより、彼はスロットに新しいオブジェクトを作成し、そのアドレスをシグナルで送信します。
  3. 作成者クラス-Creator <T>（CreatorBaseの子孫）のテンプレート実装があります。 特定のタイプのオブジェクトを作成するメソッドを実装します。
  4. 新しいスレッドを作成するThreadedObjectBaseクラス（QObjectの子孫）があります。 CreatorBase作成者を受け取り、必要な信号スロット通信を確立します。
  5. ユーザーはオブジェクトのテンプレートストレージクラスを使用し、ThreadedObject <T>（ThreadedObjectBaseの子孫）をストリームします。 新しいオブジェクトの作成を呼び出し、*および->演算子、および作成されたオブジェクトのタイプポインターをオーバーロードします。
  6. ユーザーがクラスを作成し（QObjectの子孫になることができます）、オプションで「class finished work」と「work of interruption」のシグナルを実装し、オブジェクトの遅延削除も設定できます 。




アクションのシーケンスは単純であることが判明しました。

  1. オブジェクトとThreadedObjectストリームのストレージクラスが使用されます。
  2. 彼は、作成者の作成者と新しいスレッドのQThreadオブジェクトを作成します。
  3. オブジェクトの作成者が新しいスレッドに転送されます。
  4. 信号スロット通信が確立されます。
  5. 新しく作成されたスレッドは、必要な優先度で始まります。
  6. 作成されたスレッドにカスタムクラスTが作成されます。
  7. ThreadedObjectBaseは、setObject（void * Obj）スロットを使用してこれについて学習し、オブジェクトのアドレスを記憶し、信号objectIsReady（）を使用してそれについて世界に通知します。
  8. これらすべてのアクションの正常終了については、bool ThreadedObject <T> :: objectIsCreated（void）constを参照してください。




実装



作成されたクラスのコードを考慮してください（すべてが画面に収まるように、コメントを削除しました）。

オブジェクト作成者： 



 class CreatorBase: public QObject { Q_OBJECT void *_obj; protected: virtual void *Allocation (void) = 0; public slots: void allocate (void) { emit setObject (Allocation ()); } signals: void setObject (void *Obj); }; template <class T> class Creator: public CreatorBase { protected: void *Allocation (void) { return reinterpret_cast <void*> (new T); } };




ここではすべてが明白です。CreatorBaseクリエーターの基本クラスには、新しいアクティブなスレッドで起動されるallocate（）スロットがあります。 シグナルsetObject（void * Obj）を呼び出します。これは、子void * Creator <T> :: Allocation（）で作成されたオブジェクトのアドレスを渡します。



基底クラスThreadedObjectBaseは次のとおりです。 



 class ThreadedObjectBase: public QObject { Q_OBJECT protected: QThread *_thread; virtual void SetObjectPointer (void *Ptr) = 0; ThreadedObjectBase (QObject *parent = 0): QObject (parent), _thread (0) {} void starting (CreatorBase *Creator, QThread::Priority Priority = QThread::InheritPriority, bool ToDeleteLaterThread = true) { bool res; _thread = new QThread; Creator->moveToThread (_thread); res = connect (_thread, SIGNAL (started ()), Creator, SLOT (allocate ())); Q_ASSERT_X (res, "connect", "connection is not established"); res = connect (Creator, SIGNAL (setObject (void*)), this, SLOT (setObject (void*))); Q_ASSERT_X (res, "connect", "connection is not established"); if (ToDeleteLaterThread) { res = connect (_thread, SIGNAL (finished ()), _thread, SLOT (deleteLater ())); Q_ASSERT_X (res, "connect", "connection is not established"); } _thread->start (Priority); } public: virtual ~ThreadedObjectBase (void) { } QThread *thread (void) { return _thread; } const QThread *cthread (void) const { return _thread; } signals: void objectIsReady (void); private slots: void setObject (void *Obj) { SetObjectPointer (Obj); emit objectIsReady (); } };




ここでの主な方法は開始中です。 指定された優先度でスレッドを作成します。 起動すると、_threadスレッドはQThread :: started（）シグナルを呼び出します。 この信号をスロットCreatorBase :: allocate（）に関連付けて、新しいオブジェクトを作成します。 これにより、信号CreatorBase :: setObject（void *）が発生します。これは、スロットThreadedObjectBase :: setObject（void * Obj）で取得します。 すべて、オブジェクトが作成され（これについて信号ThreadedObjectBase :: objectIsReady（）が発行されます）、オブジェクトへのポインターが受信されます。



ユーザーがスレッドクラスの遅延削除（望ましい）を設定する場合、接続は_thread QThread :: finish （） -> QObject :: deleteLater（）内で確立されます。

また、ユーザーは信号の名前を設定できます（変数_finished_signalに格納されます）。 このシグナルは、作成されたオブジェクトの作業終了時に呼び出されます。 同様に、_terminate_slotからのスロットは、スレッド割り込みシグナルによって呼び出されます（ただし、スレッドは即座に停止しません。スレッド（）-> waitを参照して、終了するのを待つことができます-QThread :: waitを参照）。



最後に、ユーザーに表示されるテンプレートクラス： 



 template <class T> class ThreadedObject: public ThreadedObjectBase { protected: T* _obj; Creator<T> _creator; const char *_finished_signal; const char *_terminate_slot; bool _to_delete_later_object; void SetObjectPointer (void *Ptr) { bool res; _obj = reinterpret_cast <T*> (Ptr); if (_finished_signal) { res = connect (_obj, _finished_signal, _thread, SLOT (quit ())); Q_ASSERT_X (res, "connect", "connection is not established"); } if (_terminate_slot) { res = connect (_thread, SIGNAL (finished ()), _obj, _terminate_slot); Q_ASSERT_X (res, "connect", "connection is not established"); } if (_to_delete_later_object && _finished_signal) { res = connect (_obj, _finished_signal, _obj, SLOT (deleteLater ())); Q_ASSERT_X (res, "connect", "connection is not established"); } } public: ThreadedObject (QObject *parent = 0): ThreadedObjectBase (parent), _obj (0) { } ~ThreadedObject (void) { } void start (const char *FinishedSignal = 0, const char *TerminateSlot = 0, QThread::Priority Priority = QThread::InheritPriority, bool ToDeleteLaterThread = true, bool ToDeleteLaterObject = true) { Creator<T> *creator = new Creator<T>; _finished_signal = FinishedSignal; _terminate_slot = TerminateSlot; _to_delete_later_object = ToDeleteLaterObject; starting (_creator, Priority, ToDeleteLaterThread); delete creator; } bool objectIsCreated (void) const { return _obj != 0; } T* ptr (void) { return reinterpret_cast <T*> (_obj); } const T* cptr (void) const { return reinterpret_cast <const T*> (_obj); } // .  operator T* (void) { return ptr (); } T* operator -> (void) { return ptr (); } operator const T* (void) const { return cptr (); } const T* operator -> (void) const { return cptr (); } };




ここで、メインのメソッドはstartです。これはシグナルとスロットの名前を記憶し、メソッドの遅延削除も設定します。 objectIsCreated（）メソッドは、オブジェクトが既に作成されている場合にtrueを返します。 複数のオーバーロードにより、ThreadedObject <T>をスマートポインターとして使用できます。



これらのクラスを使用する簡単な例を次に示します。 



 ThreadedObject <Operation> _obj; QObject::connect (&_obj, SIGNAL (objectIsReady ()), this, SLOT (connectObject ())); _obj.start (SIGNAL (finished ()), SLOT (terminate ()), QThread::HighPriority);




実際の例を以下に添付します-メインスレッドにボタンが作成されます。 int変数は、新しいスレッドで作成され、タイマーからのシグナルとタイマーイベントも作成されます。 これらのタイマーは両方ともint変数の値を減らします;ゼロに達すると、 QCoreApplication :: quit（）スロットが呼び出されます。 一方、アプリケーションを閉じると、スレッドが停止します。 WinXPでテストされた例。 Linux、MacOS、Android、およびその他のサポートされているプラ​​ットフォームでの成功したテストについてのコメントを聞きたいです 。
