VK 2.0

Durov.ruは、 User APIの最初で最も印象的な例です。 前述のように、単一のVKontakteデータベースが使用されるため、結果として、サイト上部のログインフォームでユーザー名（メールアドレス）とパスワードを入力すると、同じVKontakteが表示されますが、別のより便利なラッパー（web 2.0）に表示されます。 新しいインターフェイスを使用すると、友達のステータス、最新の写真などを表示するのがより快適になります。現在利用できる言語は英語のみです。 別のインターフェイスに加えて、次のような多くの新しい機能、機能、および結果を発見できます。

  • サイトから直接オーディオ/ビデオをダウンロードします。
  • 高解像度で写真を見る;
  • 新しいメッセージの自動表示とメッセージのリストの更新。
  • アカウントが削除された友人のページを表示します。


さらに、durov.ruを使用すると、他のユーザーの情報を完全または部分的に開示できます。 durov.ruを使用すると、プライベート写真を表示できます。 さらに、プライバシー設定で保護されているページで詳細情報を利用できます。



あとがき 。 私の目標は、悪名高いAPI自体を介しても、ユーザーAPIのオープンとVKontakteデータベースへのアクセスが何をもたらすかを示すことでした...インターフェースですべてのデータがクリーンアップされても実際に削除されるわけではなく、それらを非表示にする特別なフラグでマークされていることは明らかです。 有能な人々によるAPIのアクセスと綿密な調査は、完全に非表示または削除を希望するユーザーに関する情報の漏洩と開示につながる可能性があります。 さらに、VKontakteは、地元の開発者による製品の作成方法の良い例です。まず、広く知られ、悪用された後に最初に穴を開けてからパッチを適用します（たとえば、同じクイックルック）。
