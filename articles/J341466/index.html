Ethereumで独自のブロックチェーンを作成します

プライベートイーサリアムブロックチェーンをゼロから作成する方法は？



Ethereumは、スマートコントラクトが機能する分散プラットフォームです。ダウンタイム、検閲、詐欺、またはサードパーティの干渉の可能性なしに、厳密にプログラムされた方法で実行されるアプリケーションです。 この記事では、ローカルネットワーク内に完全に機能するイーサリアムプライベートブロックチェーンを作成するために必要なすべての手順を説明します。



指示には以下が含まれます。



  • gethを使用してプライベートEthereumブロックチェーンを作成します 。
  • プライベートブロックチェーンを操作するためのMetaMaskウォレットの作成。
  • 複数のアカウント間で資金を移動します。
  • remixを使用して、プライベートブロックチェーンでスマートコントラクトを作成、デプロイ、および呼び出します。
  • プライベートブロックチェーン上にイーサリアムブロックエクスプローラーを作成します。


Gethのインストール



Go Ethereum（またはgeth ）は、Ethereumプロトコルの3つの初期実装の1つです（C ++およびPythonとともに）。 Goで書かれており、完全にオープンで、GNU LGPL v3の下でライセンスされています。 Go Ethereumは、ほとんどすべてのOSで実行されるスタンドアロンクライアントとして、またはGo、Android、iOSプロジェクトに埋め込むためのbabliotekとしてダウンロードできます。



Mac OS Xにgethをインストールするには、 homebrewを使用しました。 Homebrewは必要なものをすべてインストールしますが、Appleはそれを処理しませんでした。 この記事は、すでにhomebrewがインストールされていることを前提としています。 そうでない場合は、 これらの指示に従ってください 。 homebrewをインストールした後、次のコマンドでgethをインストールします 。



brew tap ethereum/ethereum

brew install ethereum







Ubuntuへのgethのインストールは非常に簡単で、次のapt-getコマンドを実行するだけです。



sudo apt-get install software-properties-common

sudo add-apt-repository -y ppa:ethereum/ethereum

sudo apt-get update

sudo apt-get install ethereum







Windowsの場合、適切なインストーラーはこちらです。 問題がある場合は、 手順を参照してください 。



マイナーアカウントを作成する



最初に、イーサリアムをマイニングするためのアカウントを作成する必要があります。 したがって、保護のためにパスワード付きの秘密鍵と公開鍵のペアを生成します。 パスワードを紛失しないでください。そうしないと、キーを回復できなくなります。 デフォルトでは、キーは<datadir> / keystore内に保存されます 。 gethが保存するものはすべて、 datadir内に保存されます （PoW Ethash DAGを除く）。 デフォルトのディレクトリの場所はOSに依存します。 デフォルト設定を変更し、プライベートブロックチェーンの独自のパスを設定することをお勧めします。



  • Mac：〜/ライブラリ/ Ethereum
  • Linux：〜/ .ethereum
  • Windows：％APPDATA％\ Ethereum


Ethash DAGは〜/ .ethash（Mac / Linux）または％APPDATA％\ Ethash（Windows）に保存されるため、すべてのクライアントが使用できます。



次のコマンドは、データディレクトリの任意のパスでアカウントを作成する方法を示しています。 このコマンドは、Ethereumアドレスを提供します。



geth account new --datadir <path-to-data-directory>







例：



geth account new --datadir /path/to/data/dir







プライマリブロック（ジェネシスブロック）の作成



イーサリアムブロックチェーンは、プライマリブロックが互いに異なります。 各ブロックチェーンはそのようなブロックで始まり、他のすべてはその上に構築され、後続の各ブロックは前のブロックを参照します。 Ethereumパブリックブロックチェーンでは、プライマリブロックが2015年7月20日に作成されました。 独自のチェーンでは、独自のプライマリブロックを作成する必要があります。 プライベートブロックチェーンは、次のコマンドにより、指定されたプライマリブロックで初期化されます。



geth -datadir <path-to-data-directory> init <path-to-genesis-block>







例：



geth -datadir /path/to/data/dir init /path/to/genesis.json







  • datadir ：データベースおよびキーストアのディレクトリ。
  • init ：対応するjsonファイルを使用して、新しいプライマリブロックをブートストラップして初期化します。


以下は、プライベートブロックチェーンで使用されるプライマリブロックです。genesis.jsonという名前で保存し、上記のコマンドで指定する必要があります。 



 { "config": { "chainId": 15, "homesteadBlock": 0, "eip155Block": 0, "eip158Block": 0 }, "difficulty": "0x400", "gasLimit": "0x2100000", "alloc": { "7a69b359e86893efa3d9732e4c65ced51567edd0": { "balance": "0x1337000000000000000000" } } }


  • chainid ：ETC（Ethereum Classic）またはMordenテストネットワークを使用せずにEthereumでトランザクションを作成する方法を提供します。 EIP 155は、さまざまなネットワークに対して次のチェーンID値を提供します。メインネットワークEthereum（1）、メインネットワークMorden / Expanse（2）、Ropsten（3）、Rinkeby（4）、メインネットワークルートストック（30）、テストネットワークルートストック（31）、 Kovan（42）、Ethereum Classicメインネットワーク（61）、Ethereum Classicテストネットワーク（62）、プライベートgethチェーン（デフォルトでは1337）。 この例では、既存の回線で使用されていない識別子15を指定しました。
  • homesteadBlock ：値0は、Ethereum Homesteadリリースの使用を示します。 これは、イーサリアムの2番目のメジャーリリースです。最近、2017年10月16日に、イーサリアムはByzantiumリリースに当たりました 。
  • eip155Block ：値0は、このユニットがEIP（Ethereum改善提案、Ethereum改善提案）番号155をサポートすることを示します。EIP提案は、主要なプロトコル仕様、クライアントAPI、契約標準など、Ethereumプラットフォームの標準を記述します。
  • eip158Block ：値0は、このブロックがEIP 158をサポートすることを示します。
  • 難易度 ：値は、このブロックのランダムなナンス値を検索するために使用される難易度レベルに対応します。 この記事では、イーサリアムの難易度の計算方法について詳しく説明します。
  • gasLimit ：ガスは、イーサリアムで取引または契約を行うための内部支払い単位です。 トランザクションまたはスマート契約を処理するためにイーサリアム仮想マシン（EVM）に送信される各命令には、一定量のガスがかかります。 トランザクションが必要な量のガスを受け取らない場合、通過しません。 Ethereumで各トランザクションを作成するとき、ガス制限（このトランザクションのすべての関連操作が使用できる最大量）を指定します。 ブロック内のgasLimitパラメーターは、ブロック内のすべてのトランザクションの合計制限を決定します。
  • alloc ：プライマリブロックから1つ以上のアカウントへのEtherの予備配布のパラメーター。 上記のプライマリブロックの例では、すべてのEtherは最初から作成されたアカウントに送られます。


マイニングを開始



すべて準備完了です！ これで、次のコマンドを使用してgethからマイニングを開始できます。 ここで、 networkidパラメーターは、このイーサリアムネットワークを他のものと区別します。 このネットワークへの接続を希望するすべてのマイナーは、同じプライマリブロックで同じネットワークIDを使用する必要があります。



geth --mine --rpc --networkid <networkd-id> --datadir <path-to-data-directory>







例：



geth --mine --rpc --networkid 1999 --datadir /path/to/data/dir







  • networkid ：このイーサリアムネットワークのネットワーク識別子。 任意の値を選択してください。 たとえば、オリンピック（0）、フロンティア（1）、モーデン（2）、ロップステン（3）。
  • mine ：マイニングを含みます。
  • rpc ：HTTP-RPCサーバーを有効にします。 ウォレットアプリケーションは、httpを介してこのマイニングノードに接続できます。
  • rpcaddr ：HTTP-RPCサーバーでリッスンするインターフェイスを指定します（デフォルト：「localhost」）
  • rpcport ：HTTP-RPCサーバーでリッスンするポートを指定します（デフォルト：8545）
  • rpcapi ：HTTP-RPCインターフェースで使用可能なAPIを示します（デフォルト：“ eth、net、web3”）
  • rpccorsdomain ：CORSをアクティブにし、要求を受け入れる（クロスオリジン）サードパーティドメインのコンマ区切りリストを指定します。 Solidity（Remix）ブラウザーエディターまたはブラウザーウォレットを使用してスマートコントラクトを配置する場合に役立つオプション。 たとえば、このオプションを使用すると、任意のドメインからCORSを受け入れることができます。



    --rpccorsdomain "*"



  • nodiscover ： ピア検索エンジンを無効にします。 ネットワーク上の他のノードはサイトを見つけることができません。 ローカルネットワークでブロックチェーンを他のブロックチェーンとともに使用する場合は、このパラメーターを使用しないでください。
  • console ：このコマンドを使用すると、インタラクティブなJavaScript環境でマイニングノードを開始できます。 次のセクションでは、このトピックを詳しく見ていきます。



    geth --mine --rpc --networkid 1999 --datadir /path/to/data/dir console





Gethコンソールを接続します



コンソールとしてマイニングを開始するか、 コンソールを個別に実行して、 attachコマンドでマイニングノードに接続します。 次に、これがどのように行われるかを示し、同じ順序でパラメーターを指定したことを確認します。



geth --datadir <path-to-data-directory> attach ipc:<path-to-data-directory>/geth.ipc







例：



geth --datadir /path/to/data/dir attach ipc:/path/to/data/dir /geth.ipc







コンソールは、 IPCを介してマイニングノードに接続します。 IPC （プロセス間通信）はローカルコンピューターで実行されます。 この場合、 gethはローカルコンピューターのファイルシステムにGethパイプライン（ファイル<path-to-data-directory> /geth.ipcで表される ）を作成し、コンソールはIPCを介してこのノードに接続します。



すべてのアカウントを表示



gethコンソールに接続したら、次のコマンドを実行して、使用可能なすべてのアカウントを表示できます。

> eth.accounts

["0x7a69b359e86893efa3d9732e4c65ced51567edd0"]







アカウント残高を表示



次のコマンドを使用すると、特定のアカウントの残高をgethコンソールから表示できます。



> eth.getBalance("0x7a69b359e86893efa3d9732e4c65ced51567edd0")

1.295e+21







MetaMask Ethereum Walletの接続



MetaMaskは、Chrome拡張機能として機能するEthereumウォレットです。 サイトのすべてのJavaScriptコンテキストにEthereum Web3 APIが組み込まれているため、これらのアプリケーションはすべてブロックチェーンから情報を受け取ることができます。 MetaMaskを使用すると、独自のIDを作成および管理できるため、アプリケーションがトランザクションを完了してデータをブロックチェーンに書き込む場合、ユーザーはトランザクションを承認または拒否する前にトランザクションを表示する安全なインターフェイスを使用できます。



MetaMaskをEthereumプライベートブロックチェーンに接続するには、正しいホスト名とポートを選択する必要があります。 Web3 APIは、 web3.jsに実装されているイーサリアム用のJavaScriptプログラミングインターフェイスです。 JavaScriptアプリケーションがイーサリアムノードと通信するために、MetaMaskはweb3.jsライブラリとrpcメソッドの便利なインターフェイスを使用します。 ローカルホストとrpc呼び出しを交換します 。 web3.jsライブラリは、 rpcレイヤーを持つ任意のEthereumノードで動作します。 マイニングノードの起動時に、 rpcapiパラメーターを渡して、このノードで使用するインターフェイスを指定できることに気づいたかもしれません。 デフォルトでは、何も指定されていない場合、インターフェースeth、net、web3が使用されます。







空気伝達



MetaMaskは、イーサリアムアカウント（イーサリアムの秘密キーとアドレス）を作成します。 次に、最初に作成された最初のアカウントからgethコンソールからMetaMaskアカウントにEtherを転送する方法を見つけます。 アカウントの資金を転送するには、このアカウントの秘密鍵を使用して署名する必要があります。 以下に示すように、それを使用するには、最初にロックを解除する必要があります。



> personal.unlockAccount( "0x7a69b359e86893efa3d9732e4c65ced51567edd0","password")







personalはgethの制御プログラムインターフェイスです 。 個人に加えて、 gethはadmin 、 debug 、 miner、およびtxpoolの管理APIも提供します。 アカウントのロックを解除すると、コンソールで3つの変数を定義できます。sender 、 receiver 、およびamount 、つまり、受信者、送信者、および転送された資金の量です。 送信者の値はこの記事の冒頭で作成したイーサリアムアドレスであり、 受信者の値はMetaMaskで作成されたイーサリアムアドレスです。



> var sender = "0x7a69b359e86893efa3d9732e4c65ced51567edd0";

> var receiver = "0xA9f28458eE1170F285440990c196c1592D3a73f5"

> var amount = web3.toWei(1, "ether")







次のチームは資金を転送します-それは上で定義した変数を参照します。



> eth.sendTransaction({from:sender, to:receiver, value: amount})







MetaMaskで口座残高を表示



上記の手順に従って資金の振替が完了したら、2つの方法で口座残高を表示できます。 最初の方法は、以下に示すように、MetaMaskプラグインから直接です。







別の方法は、次のコマンドを使用してgethコンソールを使用することです。



< eth.getBalance("0xA9f28458eE1170F285440990c196c1592D3a73f5")

SolidityのRemix Editor



Solidityは、最も人気のあるEthereumスマートコントラクトプログラミング言語です。 Remixは、組み込みのデバッガーとテスト環境を備えたSolidity用のIDEです。 リミックスオンラインエディタはこちらにあります 。 Remixは任意のEthereumネットワークに接続します。 ローカルブロックチェーンに接続するには、以下に示すように、次のコマンドを使用してマイニングノードを起動してください。 --rpccorsdomain "*"



パラメーターについては既に説明しました。 アスタリスク（最大オープン性）の代わりに、このパラメーターの値として「https://remix.ethereum.org」を指定できます。



geth --mine --rpc --rpccorsdomain "*" --networkid <networkd-id> --datadir <path-to-data-directory>







リミックスを閉じたネットワークに接続するには、[ 環境 ]を[ Web3プロバイダー ]に変更する必要があります。これは[ 実行 ]タブで行います。 この変更により、 remixはWeb3プロバイダーエンドポイントを指定するよう要求します-値をhttp://localhost:8545



設定します。 ポートを明示的に変更するまで、デフォルトでは、マイニングノードはポート8545で起動します。







スマートコントラクトを作成する



これで、Ethereumネットワークで開始する最初のスマートコントラクトを作成する準備がすべて整いました。 次のコードをコピーして、 リミックスオンラインエディターに貼り付けます。 これは非常にシンプルなスマートコントラクトです。一行ずつ説明したくはありません。 次の記事では、プログラミング言語Solidityについて詳しく検討します。 



 pragma solidity ^0.4.11; contract Hello { // a string variable string public greeting; // the function with the same name as the class is a constructor function Hello(string _greeting) { greeting = _greeting; } // change the greeting message function setGreeting(string _greeting) { greeting = _greeting; } // get the greeting message function greet() constant returns (string _greeting) { _greeting = greeting; } }


remixでデフォルト設定を変更しなかった場合は、自動的にコンパイルするように設定されます。 そうでない場合は、コードをコンパイルする必要があります。







コンパイル後、[ 詳細 ]ボタンをクリックすると、このスマートコントラクトの作成に必要な推定ガス量が表示されます。



スマートコントラクトの実装



これで、プライベートブロックチェーンにスマートコントラクトを実装できます。 [ 実行 ]タブで、正しいイーサリアムアカウントと正しいガス制限が選択されていることを確認します。 ガスの価格と値はゼロとして指定できます。







ブロックチェーンでスマートコントラクトを作成するために、上記で選択したアカウントから署名されたイーサリアムトランザクションは次のとおりです。 サインアップするには、まずgethコンソールからアカウントのロックを解除する必要があります。



> personal.unlockAccount( "0x7a69b359e86893efa3d9732e4c65ced51567edd0","password")







これで、[ 作成 ]ボタンをクリックしてスマートコントラクトを作成できます。 文字列パラメーターを受け取るコンストラクターがあります。そのため、 remixは[ 作成 ]ボタンと共に入力フィールドを表示します。 ここに値を入力することができます（引用符で囲んで、たとえば「Hi」）-または単に空白のままにします。 スマートコントラクトを作成すると、 gethコンソールに次のメッセージが表示されます。 また、正しいガス制限が設定されていない場合、そのようなコマンドはエラーにつながることに注意してください。



INFO [10-19|07:31:08] Submitted contract creation fullhash=0xf5511bb9d088672ac0d3896b8590b9a3e25484300f02deecdd739c3a549ed33a contract=0x42b7E903Fb42e191a7D623cbb4b7b4330D329d78







スマートコントラクトコール



スマートコントラクトを作成した後、 リミックスインターフェイスは少し変更されます-以下に示すように、[ 実行 ]タブで。







ここで、 setGreetingメソッドに特定の値を設定し、それをクリックしてスマートコントラクトを呼び出すことができます。 繰り返しますが、スマートコントラクトを呼び出すと、イーサリアムトランザクションが使用され、開始者の署名が必要になるため、アカウントのロックが解除されていることを確認する必要があります。 setGreetingを使用して操作を完了すると、他のメソッドを呼び出すことができます。







イーサリアムブロックエクスプローラー



Ethereumパブリックブロックチェーンをスキャンするためのetherscanサービスに精通しているかもしれません。 ただし、ローカルブロックチェーンに送信することはできません。 Ethereum Block Explorerは機能がetherscanと比較されていませんが、ローカルブロックチェーンで何が起こっているかを調べるのに非常に適しています。



プログラムをインストールするには、まず次のリポジトリからソースを取得します。



git clone https://github.com/carsenk/explorer







次に、Ethereum Block Explorerディレクトリからインストールコマンドを実行します。



npm install







インストールが完了したら、次のコマンドでイーサリアムブロックエクスプローラーを起動し、 http://localhost:8000



介してWebコンソールにアクセスします。



npm start











Ethereum Block Explorerをローカルブロックチェーンに接続する前に、パラメータ--rpccorsdomain "*"



指定した次のコマンドでマイニングノードを起動してください-その値については前に説明しました。



geth --mine --rpc --rpccorsdomain "*" --networkid <networkd-id> --datadir <path-to-data-directory>
