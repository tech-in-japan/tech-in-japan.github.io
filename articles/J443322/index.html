GitLab 11.8がJavaScript用のSAST、サブグループ用のGitLab Pagesおよびバグ追跡とともにリリースされました







JavaScriptのSASTサポート



Static Application Security Testing（SAST）のおかげで、GitLabはコードをスキャンし、パイプラインでも潜在的な脆弱性を検出するのに役立ちます。 リリース11.8では、node.jsの既存のサポートに基づいて、 サポートされているSASTのリストにJavaScript言語を追加します。 これで、JavaScriptファイル、静的スクリプト、HTMLをスキャンできます。 現在、DevSecOpsの重要なプラクティスは、コミットするたびに変更をスキャンすることです。このSASTアップデートでは、最も人気のあるWeb言語の1つをカバーし、ユーザーがJavaScriptコードのリスクを早期に検出できるようにします。







サブグループおよびページテンプレート用のGitLabページ



GitLab Pagesは、次の2つの改善により、このリリースで大幅に改善されました。 まず、 サブグループのプロジェクトにGitLab Pagesのサポートを追加しました。このようなプロジェクトでは、コンテンツをネットワークに簡単に公開できます。 次に、GitLab 11.8にはPagesの最も人気のあるテンプレートが含まれるようになったため、ワンクリックでPagesの操作を開始できます。







Sentryのバグ追跡



エラーにより、アプリケーションの状態を評価でき、ユーザーが報告する前に問題を検出できます。 GitLab 11.8はプロジェクト内の最近のエラーを表示できるようになり、検出と修正が容易になりました。







さらに多くのクールな機能！



このリリースには非常に多くの優れた革新がありますので、さらにいくつかを強調したいと思います。







  • マージリクエストの確認ルール （GitLabのロシア語版「マージリクエスト」）。 これで、特定のユーザー、グループ、またはロールなど、変更を確認するルールを簡単に設定できます。 まもなくこの機能はGitLab.comに表示され、ユーザーインスタンスではGitLabは管理者によって既に接続されています。
  • 環境用のプラグイン機能 。 以前は接続されていた機能がすべての環境で同時に機能していましたが、特定の環境内で使用できるようになりました。 この機能はすでにGitLab.comで機能しており、管理者はユーザーインスタンスに対して有効にすることができます。
  • メッセージを結合するときのメッセージへのマージの改善（スカッシュ） 。 コミットに対して適切なメッセージを作成しようとすると、コミットをマージするときにそれらを失うのは残念です。 リリース11.8では、すべてのコミットからのコミットを結合すると、最初の複数行メッセージが自動的に取得されます。 また、メッセージを手動で設定することもできます。
