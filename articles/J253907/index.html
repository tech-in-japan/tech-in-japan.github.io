DotHill 4824ストレージの概要

このレビューの主役は控えめなDotHill 4824ストレージシステムです。OEMパートナーであるDotHillは、すでに第4世代の非常に人気のあるHP MSA（Modular Storage Array）であるHewlett-Packardのエントリーレベルのストレージを製造していることを聞いたことのある人は多いでしょう。 DotHill 4004行は、HP MSA2040に対応しますが、わずかな違いがあります。これについては以下で詳しく説明します。



DotHillは、クラシックなエントリーレベルのストレージシステムです。 フォームファクター、2U、さまざまなドライブ用の2つのオプションと多種多様なホストインターフェイス。 ミラー化キャッシュ、2つのコントローラー、ALUAを使用した非対称アクティブ-アクティブ。 昨年、新しい機能が追加されました。3層の階層化（階層化されたデータストレージ）とSSDキャッシュを備えたディスクプールです。

DotHillディスクプール



理論に精通していない人にとっては、ディスクプールと階層型ストレージの動作原理について話す価値があります。 より正確には、DotHillストレージでの特定の実装について。



プールの前には、2つの制限がありました。

  • ディスクグループの最大サイズ。 RAID-10、5、および6は、最大16台のディスクを持つことができます。 RAID-50-最大32ドライブ。 （パフォーマンスやボリュームのために）多数のスピンドルを持つボリュームが必要な場合は、ホスト側でLUNを結合する必要がありました。

  • 高速ディスクの次善の使用。 いくつかの負荷プロファイルに対して多数のディスクグループを作成できますが、それらに多数のホストとサービスがあると、パフォーマンス、ボリュームを継続的に監視し、定期的に変更を加えることが困難になります。





DotHillストレージのディスクプールは、複数のディスクグループの組み合わせであり、それらの間の負荷分散が行われます。 パフォーマンスの観点から、いくつかのサブアレイからのRAID-0としてプールを考えることができます。 すでに短いディスクグループの問題を解決しています。 合計で、ストレージシステムでサポートされるディスクプールはAとBの2つのみで、コントローラーごとに1つ）、各プールは最大16個のディスクグループを持つことができます。 主なアーキテクチャ上の違いは、ディスク上のストライプの無料割り当ての最大使用です。 いくつかのテクノロジーと機能は、この機能に基づいています。

  • シンプロビジョニング 。 このテクノロジーは新しいものではなく、たとえばデスクトップハイパーバイザーなど、単一のストレージシステムを見たことがない人でもTPに遭遇した可能性があります。 ボリュームは、プールまたは通常のリニアディスクグループの上に作成され、ホストに提供されます。 「シン」ボリュームの下のディスク領域はすぐに予約されるのではなく、データがいっぱいになると予約されます。 これにより、成長予測を考慮して必要なディスク容量を事前に割り当て、実際よりも多くの容量を割り当てることができます。 将来的には、システムをまったく拡張する必要はありません（すべてのボリュームが実際に割り当てられたボリュームを使用するわけではありません）、または複雑な操作（データ転送、LUNとファイルシステムの増加など）に頼らずに必要な数のディスクを追加できます。 ） ボリューム不足の可能性がある状況を恐れている場合は、プール設定でオーバープロビジョニングを無効にし、使用可能な容量よりも多くのディスク容量を割り当てると、機能しなくなります。



    シンプロビジョニングを使用する場合、逆のプロセス-未使用のディスクスペースをプールに戻すこと（スペースの再利用）を確認することが重要です。 ファイルを削除する場合、追加の手順を実行しない限り、ストレージシステムは対応するブロックのリリースについて知ることができません。 一度スペース再生をサポートするストレージシステムは、ホストからのゼロの定期的な記録（たとえば、SDeleteを使用）のみに依存することができましたが、ほとんどすべて（そしてもちろん、DotHill）は、SCSI UNMAPコマンド（ATAコマンドセットのよく知られたTRIMの類似物）をサポートします）



  • プール要素を透過的に追加および削除する機能 。 従来のRAIDアレイでは、多くの場合、ディスクを追加することでアレイを拡張できますが、DotHillを含むほとんどのディスクプール実装では、データを失うことなくディスクグループを追加または削除できます。 このグループのブロックは、プール内の残りのグループに再配布されます。 当然、十分な数の空きブロックが利用できることを条件とします。

  • 階層化（階層データストレージ）。 異なるディスクグループに異なるパフォーマンス要件のボリュームを分散する代わりに、異なるパフォーマンスのディスク上のボリュームを共通のプールに結合し、異なるパフォーマンスの3 層間のブロックの自動移行に依存できます。 移行の基準は、ランダムアクセスの負荷強度です。 より頻繁に要求されたブロックは一番上に「ポップアップ」し、めったに使用されないブロックは低速ディスクに「落ち着き」ます。



    DotHillストレージシステムには、アーカイブ（アーカイブ、7200 rpmドライブ）、標準（標準、「実際の」SASディスク10/15 000 rpm）、およびパフォーマンス（パフォーマンス、SSD）の3つの固定層があります。 この点で、たとえば、1ダースの7200ディスクと1組の15kディスクで構成される、正しく機能しない構成を取得する可能性があります。 DotHillは、1組の15kディスクからのミラーの生産性を向上させ、「ホット」データをそこに移動します。



    階層化を使用した構成を計画する場合、常にある程度の遅延を伴うパフォーマンスの向上が得られることに留意する必要があります。システムは、ブロックを目的の階層に移動する時間が必要です。 2番目のポイントは、DotHill 4004の階層化の基本的な構成です。 ボリュームに優先順位を付ける能力の欠如。 DataCoreや一部のミッドレンジストレージシステムのように、プロファイルとティアアフィニティを使用した複雑な構造はありません。これはまだエントリレベルのストレージシステムです。 すぐにそして常にパフォーマンスを保証したいですか？ 通常のディスクグループを使用します。 DotHillの階層化に代わるもう1つの方法は、SSDキャッシュです。詳細については以下で実装します。

ティア1およびSSD上のキャッシュを備えたプール



このテストは、異なる読み取り/書き込み比率での8KiBブロック（それぞれ16のキュー深度を持つ8つのストリーム）のランダムアクセスを伴う3時間（60秒の180サイクル）の負荷です。 負荷全体が0〜20GBの領域に集中します。これは、SSDのパフォーマンス層またはキャッシュ（800GB）よりも少ないことが保証されています。これは、許容時間内にキャッシュまたは層をすばやくいっぱいにすることを目的としています。



各テスト実行の前に、ボリュームが再作成され（SSD層またはSSDキャッシュをクリアするため）、ランダムデータ（1MiBブロックでの順次書き込み）で満たされ、ボリュームの先読みがオフになりました。 IOPS、平均および最大遅延の値は、60秒サイクルごとに決定されました。



100％読み取りと65/35読み取り+書き込みのテストは、SSD層（RAID-10の4x400GB SSDのディスクグループがプールに追加された）とSSDキャッシュ（RAID-0の2x400GB SSD、ストレージ各プールのキャッシュに3つ以上のSSDを追加することはできません）。 ボリュームは、15,000 rpm SASの10個の46GBディスクの2つのRAID-6ディスクグループのプール上に作成されました（つまり、これは2x10スキームによるRAID-60です）。 なぜ10または50ではないのですか？ ストレージのランダムストレージを意図的に複雑にする。



IOPS



結果はかなり予測可能でした。 ベンダーが主張しているように、SSD層に対するSSDキャッシュの利点は、キャッシュの高速化です。 SHDは、ランダムアクセスの負荷が大きい「ホット」エリアの出現により速く反応します。IOPSは、ティアリングを使用する場合よりも100％速い読み取り遅延とともに低下します。



この利点は、大幅な記録負荷が追加されるとすぐに終了します。 RAID-60は、控えめに言っても、小さなブロックでのランダム記録にはあまり適していませんが、この構成は問題の本質を示すために特別に選択されました。ストレージシステムは書き込みができないためです。 遅いRAID-60でキャッシュをバイパスし、キューがすぐに詰まってしまい、キャッシュを考慮しても読み取り要求を処理するのに十分な時間がありません。 一部のブロックはまだそこに到達していますが、記録が進行中のため、すぐに無効になります。 この悪循環は、このような負荷プロファイルでは読み取り専用キャッシュが無効になるという事実につながります。 PCI-E RAIDコントローラーLSIおよびAdaptecの初期バージョンのSSDキャッシュ（ライトバック前）でもまったく同じ状況が観察されました。 解決策-最初はより生産的なボリューム、つまり 5/6/50/60の代わりにRAID-10および/またはキャッシュの代わりにSSD階層化。

「キャッシングと階層化（階層化）」というジレンマに要約できるものは何ですか？

何を選ぶ？

  • キャッシュを満たすのが高速です。 ロードが主にランダムな読み取りで構成され、ホットエリアが定期的に変更される場合は、キャッシュを選択する必要があります。
  • 「高速」ボリュームの保存。 「ホット」データが完全にキャッシュに収まり、SSD層には収まらない場合、キャッシュはより効率的です。 DotHill 4004のSSDキャッシュは読み取り専用なので、RAID-0ディスクグループが作成されます。 たとえば、それぞれ400 GBの4つのSSDがある場合、2つのプールごとに800 GBのキャッシュ（合計1600 GB）、または階層化を使用する場合は2倍少ないキャッシュ（1つのプールに800 GB、2つのプールに400 GB）を取得できます。 もちろん、2番目のプールがSSDを必要としない場合、1つのプールのRAID-5に1200GBのオプションもあります。



    一方、階層化を使用するときに使用できるプールの合計サイズは、ブロックのコピーを1つだけ保存することにより大きくなります。



  • キャッシュはシーケンシャルアクセスのパフォーマンスに影響しません。 キャッシュがブロックを移動せず、コピーのみを行う場合。 適切な負荷プロファイル（同じLBAに繰り返しアクセスする小さなブロックでのランダム読み取り）を使用すると、ストレージシステムはSSDキャッシュ（存在する場合）またはHDDからデータを返し、キャッシュにコピーします。 シリアルアクセスの負荷が表示されると、データがHDDから読み取られます。 例：20個または15k個のHDDのプールは、順次読み取りに約2000MB /秒を与えることができますが、必要なデータがSSDのペアのディスクグループにある場合、約800MB /秒になります。 重要かどうかは、ストレージシステムを使用する実際のシナリオによって異なります。


4x SSD 400GB HGST HUSML4040ASS600 RAID-10



ボリュームは、4つの400GB SSDのうちのRAID-10であるリニアディスクグループでテストされました。 このDotHill供給では、HGST HUSML4040ASS600は抽象的な「400GB SFF SAS SSD」であることが判明しました。 これは、 SSD400MシリーズのUltrastar SSDであり、宣言されたパフォーマンス（4KiBの読み取り/書き込みで56000/24000 IOPS）が非常に高く、最も重要なのは、5年間1日10回の書き換えのリソースです。 もちろん、現在HGSTの兵器庫には、より強力なSSD800MMとSSD1600MMがありますが、DotHill 4004にはこれらで十分です。



シングルSSD用に設計されたテスト-SNIAソリッドステートストレージパフォーマンステスト仕様エンタープライズv1.1の「IOPSテスト」と「レイテンシテスト」を使用しました。

  • IOPSテスト 。 IOPS（1秒あたりの入力/出力操作）の数は、さまざまなサイズ（1024KiB、128KiB、64KiB、32KiB、16KiB、8KiB、4KiB）のブロックおよび異なる読み取り/書き込み比率（100 / 0、95 / 5、 65 / 35、50 / 50、35 / 65、5 / 95、0 / 100）。 キュー深度16の8つのスレッドを使用しました。

  • 遅延テスト 。 平均レイテンシーと最大レイテンシーは、さまざまなブロックサイズ（8KiB、4KiB）および最小キュー深度（QD = 1の1ストリーム）での読み取り/書き込み比率（100 / 0、65 / 35、0 / 100）で測定されます。



テストは一連の測定で構成されます-60秒の25ラウンド。 プリロード-2倍の容量に達するまで128KiBブロックの連続記録。 定常状態ウィンドウ（4ラウンド）は、プロットによって確認されます。 定常状態の基準：ウィンドウ内の線形近似は、平均値の90％/ 110％の境界を超えてはなりません。
