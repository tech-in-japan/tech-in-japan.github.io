sshdを介したWindowsからLinux上での開発：FarおよびNetBoxのセットアップ

開発者がWindowsで作業することを余儀なくされることがありますが（習慣や会社のポリシーなど）、Linuxで収集するためにポータブルコードを変更する必要があります。 単に仮想マシン（たとえば、Oracle Virtual Box）を配置して、その中で直接作業することができます。 また、Windows上でFarで動作するようにシステムを構成し、仮想マシンのファイルシステム全体にアクセスし、Farからすべてのコマンドを直接実行できます。 例として、Virtual Box、Ubuntu、Far Manager、NetBoxプラグインを使用して設定する方法を見てみましょう。



以下が必要です。


1.内部にUbuntuが構成され、Far Managerが外部にあるOracle Virual Box仮想マシン

2. NetBoxプラグイン。 Farのバージョンに合わせて選択します： http : //plugring.farmanager.com/plugin.php ? pid=859 （ https://github.com/michaellukashov/Far-NetBox/downloads/ ）

3. Oracle VirtualBox： https : //www.virtualbox.org/wiki/Downloads



Ubuntuのセットアップ：


1. Ubuntuを実行してログインします

2. sshdを上げる： sudo apt-get install openssh-server

3. ifconfigを実行し、仮想マシンのIPアドレスを書き込みます（NATの下にある必要があります）

4. VirtualBoxのVMプロパティに移動します。 [設定]-> [ネットワーク]-> [ポート転送]-> [新しいルールを挿入]を入力します。



Rule1 TCP <ホストIP> 22 <ステップ3の仮想マシンIP> 22

VMを再起動します。 これでUbuntuの構成が完了しました。 ファーに移りましょう。



Far Managerのセットアップ：


1. NetBoxフォルダーをFarプラグインフォルダー（C：\ Program Files（x86）\ Far2 \ Pluginsがある）に解凍し、Farを再起動します

2. Alt-F1、NetBox、Shift + F4を押します

3.接続のタイプを選択します：SCP。 ホストIPアドレスを入力し、Ubuntuのパスワードでログインします

4.作成されたアカウントに移動します



UbuntuのファイルをFarから直接操作できるだけでなく、アセンブリコマンドを実行することもできます。
