費用対効果の高いNutanixソリューション





オランダのパートナーであるNutanixは、VDI / EMC vBlock、Cisco / NetApp FlexPod、EMC VSPEXなどの従来のソリューションの費用対効果を比較する興味深い計算をブログで公開し、VDIで公開されているリファレンスアーキテクチャ（CVS、Citrix Validated Solutions） XenDesktopを使用し、Nutanixで繰り返します。 Nutanixには、Nutanix向けのCitrix Validated Solutionというドキュメントもあります。

これを行うことは難しくありません。特に、Nutanixがサイザーを公開したためです。これは、VDIの負荷を計算するNutanixの公式が比較的単純であり、そのアーキテクチャのメリットにより可能になりました。



これが計算プロセスの流れです。



サイジング時には、3000番目のシリーズの1つの計算ノード（ノード）に110 VDI VMまたは200 HSD（いわゆるXenApp）ユーザーの負荷があるという事実から進めることができます。 したがって、4ノードのフル2Uブロックの場合、これは440 VDI XenDesktop VMと800 XenAppユーザーになります。 さらに、成長は1つのノードの増分で線形です。



システムの最初の2つのノードに、プロジェクトインフラストラクチャの仮想マシンを配置します。

この図は、XenApp用とVDI用の2つのオプションを示しています。











LoginVSI ：Medium Workload ベンチマークを使用してすべての設計のパフォーマンスを評価したため、すべてのベンダーの負荷プロファイルの特性はほぼ等しいと想定できます 。



計算での電気のコストは、オランダの産業消費者の平均（ kWhあたり0.23ユーロ ）と等しくなり、著者はこの比較を書きました。 ロシアおよびその他の電気料金の価値については、お金の絶対的な指標のみが変化しますが、節約の相対的価値は変わりません。 ところで、誰が情報を所有しているのか、コメントで私がモスクワ地域のデータセンターで計算するためにキロワットあたりの価格を教えてください。



電力消費と熱放散は、ベンダーの資料と説明に示されているリファレンスアーキテクチャ、およびオンライン計算機（たとえば、 Cisco UCS電力計算機）から取得され、標準値として取得されます。ピーク最大値のみが示されている場合は75％として取得されます。



デスクトップのコストは、インフラストラクチャの総コストに基づいて計算されます。これは、デスクトップが機能するためにブローカー、スイッチ、およびその他のコンポーネントが必要だからです。



冷却コストの方法論は、Edwin Weijdemaによるこの出版物から引用されています。 彼の計算によると、1時間あたり1000 BTUをリサイクルするには、冷却に293ワット/時間を費やす必要があります。 彼の方法論を例にとり、その結果を使用します。



XenDesktopのバージョンはわずかに異なる場合があります（たとえば、バージョン7.1と7.5は異なるアーキテクチャで使用されていました。Hyper-Vを使用したNutanixの場合、VMware vSphereを使用した他のアーキテクチャではハイパーバイザーも異なります。



Nutanix対VCE vBlock



ソリューションが占有するボリューム、いわゆるフットプリントから始めましょう。 2500台のデスクトップ用の100％XenDesktop VDIシステムのVCE vBLOCKとNutanixの比較。 VCEリファレンスデザインはこちらから入手できます 。

以下は、両方のソリューションのラック配置の写真です。







2500 VDI XenDesktopのVCE vBlockは、ほぼ完全なキャビネット（42Uのキャビネット容量のうち35U）を占有し、Nutanixは20Uを使用します。 計算によれば、クラスター内に25個のノードが必要になります。また、図にあるように、上のブロックでは1つのノードのみを使用します。 キャビネット上部の6Uは、必要なクラスタースイッチングインフラストラクチャを提供するAristaスイッチ（2個の48ポートギガビットスイッチと4個の24ポート10Gスイッチ）です。

vBlockの設計ではProvisioning Servicesと4つのPVSサーバーを使用しているため、最初の2つのノードからデスクトップを削除して、そこに4つのPVSサーバーを追加しました。 つまり、2500のデスクトップが23のノードで動作します（合計2530のデスクトップ）。



次に、計算を見てみましょう。







この計算では、75％の負荷に基づいて、 vBlockの運用コスト（管理および保守コストを除く、電力消費と冷却のみ）で年間69547ユーロかかることがわかります。 それに比べて、Nutanixソリューションの運用コストは、VCE vBlockの約半分の価格です。 デスクトップコストの計算では、vBlockのデスクトップでは年間27.81ユーロ 、Nutanixでは年間13.85ユーロしか計算されません。 合計50％の節約。



Nutanix対NetApp / Cisco Flexpod



Nutanixを他のソリューションと比較してみましょう。 ここで見ることができるCisco / NetAppのリファレンスデザインをご覧ください。2000人のユーザー向けに作成されています。XenAppで75％、VDIで25％です。



クローゼット内の占有スペース：







再び、占領地に大きな違いが見られます。 FlexPodはスペースの点ではvBlockよりも優れていますが、Nutanixの場合は30U対Nutanixよりも劣っています。 Nutanixは4つのブロックを使用し、上部に2つのノードのみがインストールされています。 各インフラストラクチャノードには100人のXenAppユーザーがいるため、XenAppの1,500ユーザーには約9ノード、VDIユーザーにはさらに5ノードが必要です。 100のXenAppユーザーと50のVDIにより、元のデザインのデスクトップの結果をわずかに上回りましたが、これによりさらに計算が簡略化され、「より多く-少なくなりません」:)。



結果：







結果はそれ自体を物語っています。 年間コストが60％削減され、デスクトップ価格は年間9.49ユーロです。 Hosted Shared Desktop（XenApp）でFlexcastモデルを使用すると、VDIを使用する場合よりも密度の高い場所を取得できるため、デスクトップの価格を年間10ユーロ未満にすることができます。



最後に、3番目の比較。



NutanixとCisco UCS Bブレードサーバー+ EMC VSPEX



NutanixとCisco + EMC VSPEXの設計を比較してください。 これは1000ユーザー向けの設計であり、HSD XenAppとVDIの間で70％/ 30％の比率に分割されています。 「収束ソリューションv1.0」であるFlexPodと比較すると、ラックスペースは約50％少なくなりました。ここでは、デスクトップ仮想化タスクに従来のストレージシステムと従来のサーバー+ストレージソリューションを使用するのは非常に非効率的です。







CVS（Citrix Validated Solution）に基づいて、750 HSDおよび250 VDIユーザーに必要なノードは7つだけであり、ラック内のNutanixには4U、ラック上部のスイッチには3Uしか必要ありませんでした。



結果は次のとおりです。







70％のコスト削減、つまり年間25654ユーロ 、デスクトップ価格は年間11.23ユーロ 。 同時にクローゼットのスペースを節約する-80％ 。



結論



ハイパーコンバージドインフラストラクチャによってフットプリントが大幅に小さくなること、つまり、インフラストラクチャがデータセンターラックで占有するスペースが大きくなることは大きな驚きではありませんでした。 しかし、Nutanixを使用することで、占有するラックスペースが少なくなるだけではありません。 冷却だけでなく、電源のコストも低くなります。 従来のソリューションの多くは大量の電源を必要としますが、これは一部のデータセンターでは不可能な場合があります。これらの追加リソースが利用可能であっても、Nutanixソリューションと比較すると年間コストが非常に大きいです。



そして、おそらく、VCE vBlockの一部の顧客に対するこの状況は予言的なものになるでしょう。 :)
