Web 3.0などについて

Web 2.0で取り上げられたトピックは現在非常に人気があり、多くの興味深いリソースが取り上げられています。 そして、Web 3.0はそのようになるでしょうか？



Web 2.0ファースト



この現象にはいくつかの主要な側面があります。



1. Webサービス

2. AJAX

3. Webシンジケーション

4.マッシュアップ

5.タグ（タグ）



記事「Tim O'Reilly、What is Web 2.0」（2005年10月11日、Computerra）はWeb 2.0の地図を示しています







しかし、Web 2.0の後にはWeb 3.0があります



これはどんな怪物でしょうか？



Webでは、Web 3.0は次のことを意味します。



Web 3.0は、次のような単純な要求に対して明確かつ最大限に完全な回答を提供できるシステムです。「休暇中にリラックスできる暖かいリゾートを探しています。 私はそれのために3,000ドルを持っています。 ところで、私には11歳の子供がいます。」 現在の状況では、そのような情報の検索には1時間以上かかる場合があります。フライト、ホテル、レンタカー会社のリストを調べる必要があります。 「Web 3.0」の条件では、理想的には、旅行会社の代理店がやったように、ユーザーは専門的かつ効率的に情報の完全なパッケージをすぐに受け取るべきです。




したがって、これは、要求に応じた情報の発行から、顧客のニーズに応じた情報の発行への移行です



このスキームでは、次の開発オプションが可能です。



ユーザーからの検索クエリの処理を変更すると、検索エンジンはユーザーに関する情報を認識します。 ユーザーデータを収集して分析します。 検索ボックスにクエリを入力する前にログインする必要がある場合があります。その後、システムはユーザーを識別し、「エジプトへのツアー」などの検索クエリを入力した後、インターネットでの検索、ユーザーのデータによる検索、彼のブログ投稿。 結果を発行する際、アンケートとユーザー記録からのデータも考慮されます。 つまり、ユーザーが昨年エジプトのそのようなリゾートに行って、それを気に入らず、彼のブログでそれについて書いた場合、結果を出すときにこれが考慮されます。



Yandexの開発に関する最新データを比較すると、非常に興味深いのは次のとおりです。



-ブログ検索開発

-YandexによるMyKrugサービスの購入

-サービス「Ya.ru」のテスト版で起動



このスキームは、すぐにYandexの現実に変換できると思います。
