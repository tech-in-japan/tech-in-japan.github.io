Java + Selenium + FlexMonkiumを使用してFlexアプリケーションを自動化する

この投稿では、テストの自動化に関する質問がプロジェクトで提起されたときにやらなければならなかった私の研究について説明します。 このプロジェクトはWebサイトであり、クライアント部分はFlexで記述され、サーバー部分はPython（Django）で記述されています。



自動化のためのツールの選択


始めるために、Java + Selenium + FlexMonkiumバンドルに落ち着く前に行った最初からのパスを検討してください。 そしてこれ：



  • セレン+ Sfapi
  • ラノレックス
  • フレックスモンキー
  • セレン+ FlexMonkium




自動化ツールとして最初に使用しようとしたのは、Sfapi（Selenium Flex-API）でした。 Selenium RCおよびドライバー（Java、C＃、Python、Ruby）があるプログラミング言語と連携して動作します。 利点のうち、これは実際に機能する無料のソリューションであることに注意してください。 欠点の-多くのコードが、十分な結果ではありません。 アプリケーションの動的な要素ではうまく機能しません。 ソリューションをさらに検索する理由は、単純な操作を実行する大量のコードでした。



私が試した次のFlex自動化ソフトウェアはRanorex Studioでした。 これは、直感的なインターフェースを備えた非常にユーザーフレンドリーなソフトウェアパッケージです。 さらに、組み込み機能とSpyエージェント（ユーザーアクションのシーケンスを記録する）を使用してプログラミング言語を知らずにテストを迅速に作成し、提案されたプログラミング言語のいずれかに変換して作成済みのテストを完了することに注意する価値があります（サポートされている言語のリストは、購入したライセンスの種類によって異なります） ）およびほぼすべてのFlex UI要素との良好な相互作用（フォーム上では、DataGrid要素の不適切な作業に関する苦情に遭遇しましたが、作業中に作業する機会がありませんでした） ）。 欠点の1つはライセンスのコストが高いことです。異なるドメイン（たとえば、テストサーバーとメインサーバー）でアプリケーションをテストする必要がある場合は、最初にテストでいくつかの変換を行って、アプリケーションインターフェイスと正しく対話を開始する必要があります。 ソリューションをさらに検索する理由となったのは、ライセンスの高コストでした。



Ranorexライセンスの終了を待たずに、私はフリーソフトウェアの検索を続け、FlexMonkeyの使用に関するアドバイスに出会いました。 彼らのウェブサイトには、短いが理解しやすいドキュメントがあり、活発な議論とライブサポートを行うフォーラムもあります。 アプリケーション自体はAdobe Airで作成されており、わかりやすく使いやすいインターフェイスを備えています。







結局のところ、FlexMonkeyは非常に高速で便利で無料です。 ほぼすべてのUIコンポーネントでの動作を正しくサポートします（少なくともすべてが感動的で機能しました）が、プログラミング言語を使用してテストを改善できないという欠点もあります。 実際、コンポーネントのプロパティのチェック、これらのプロパティの値の比較など、基本的な機能のみを使用できます。 ただし、たとえば、プログラミングツールなしでメールユーザーインターフェイスにメッセージを書き込むことはできません。 FlexMonkeyの作成者による次の製品は、Seleniumと連携して動作し、 FlexMonkiumと呼ばれます 。



作業環境のセットアップ


Flexアプリケーションの自動化テストを開始するには、Selenium RC + FlexMonkiumと対話するようにプロジェクトを構成し、プログラミング環境を準備する必要があります（たとえば、Eclipse IDEでJava言語を選択しました）。 さらに、すべてのアクションはステップで説明されています。



1.テストを作成するプログラミング言語（私の場合はJava）用のSelenium RCおよびSelenium Client Driverをダウンロードします。



2. FlexMonkiumをダウンロードします （例ではバージョン4.1.8を使用しました）。 アーカイブには3つのコンポーネントが含まれます-.swcライブラリを含むフォルダー。そのうちの1つはFlexプロジェクトに接続する必要があります。「user-extensions.js」はSeleniumサーバーの拡張であり、フレックスアプリケーション制御コマンドを「理解」します。 FirefoxのSelenium IDEの機能を拡張するテストおよび.xpiプラグインで指定されています。



3. Flash BuilderでFlexプロジェクトを開き、ダウンロードしたアーカイブの.swcライブラリをプロジェクトに接続します。 アーカイブには、最後に同じ名前と番号のファイルがいくつかあります。 番号はFlex-SDKのバージョン番号を意味します（私の場合、これは4番目のバージョンです。つまり、automation_monkey4.x.swcを接続する必要があります）。 その後、プロジェクトを再コンパイルし、出来上がり-Seleniumサーバーと対話する準備が整いました。



4.ここで、コマンド拡張ファイルを使用してSeleniumサーバーを起動する必要があります。これにより、Flexアプリケーションの管理に固有のコマンドを「理解」できるようになります。 このファイルは、ダウンロードされたFlexMonkiumアーカイブに含まれており、「user-extensions.js」と呼ばれます。 サーバーの起動時にコマンドラインでパラメーターとして渡されます。 これらの目的のために、次の内容の別個のbatファイルを作成しました。



java -jar C:\Automation\selenium-server-standalone-2.0rc2.jar -userExtensions C:\user-extensions.js







その後、サーバーを安全に起動できます。 すべてが正しく行われた場合、bat'nikを起動した後、次のようなウィンドウが画面にハングアップします。







したがって、テスト対象のアプリケーションはテスト自動化ツールと対話する準備ができており、Selenium-RCが起動され、フレックスコンポーネント管理コマンドを理解します。



最初のテストを作成する


テストを作成するための言語として、JUnitライブラリーとともにEclipse開発環境でJavaを選択しました。 この組み合わせでは、テストは正常に機能しましたが、各テストの後、JUnitはブラウザーを再起動しました。これは、私のテストの多くが以前のテストの継続であり、アプリケーションの状態をリセットするのが適切ではないため、私にはお勧めできませんでした。 いくつかのフォーラム検索の後、この動作を無効にすることはできず、JUnitの使用を放棄しなければならなかったことを読みました。 レポートでは、失敗したテスト、これが発生したステップ、および発生したエラーの数のみを知る必要がありました。 これらの目的のために、この情報をコンソールに表示する特別な関数を作成しました。 テストが失敗すると呼び出され（テストが失敗するとタイムアウトが発生します）、Javaは例外をスローします。 その結果、次のコードが得られました。

この例では、アプリケーションの内部メールによるメール送信がテストされ、送信メール間の表示が確認されます。



そして今、より詳細に。 seleniumStartおよびseleniumStop関数は、セッションを開始および終了します。 最初の関数は、コマンドをサーバーに送信するSeleniumクラスのインスタンスを初期化します。 ホストおよびポート（Seleniumサーバーが実行されている）、ブラウザー（テストが実行される）、およびURL（テストされたアプリケーションが使用可能）などのパラメーターがそこに転送されます。



次はprintError関数で、テスト名とステップ番号をパラメーターとして受け取り、コンソールに表示します。 この関数は、テストのいずれかのステップでタイムアウトになったときに呼び出されます。



各ステップで繰り返しコードのブロックを記述しないために、「testExec」という別の関数に移動しました。 そのパラメーターは、実行されるステップの名前と番号（タイムアウト後に「printError」に送信されます）、サーバーのコマンド、「到達」しようとしているアプリケーション内の要素へのパス（ロケーター）、およびタイムアウトです。 上記の例のアプリケーションパスには、「VerifyProperty」と「UIEvent」の2つのタイプがあります。 前者は任意の要素の属性値を比較するために使用され、後者は指定された要素に対してアクションを実行するために使用されます。たとえば、ボタンをクリックします。 アプリケーションのUI要素は、その名前またはIDだけでなく、そのプロパティのいくつかの組み合わせによっても指定できます。これらのプロパティは、「arg」タグでさらに示されます。 より詳細なドキュメントは、 Gorilla Logicの Webサイトで入手できます。



ロケーターを手動で記述しないように、FlexMonkeyを使用して一連の手順を書き留め、テストをxmlファイルにエクスポートして、Javaコードに挿入しました。 結果はこの結果です：
