上陸とは

一時、CRMシステムの使用を開始したとき、ランディングページを間接的かつ直接的に処理する必要がありました。それは何ですか。 ランディングページとは何ですか。 それらが使用される理由; 彼らがもたらす利益、害 それらとサイトの違いは何ですか？ これらおよび他の多くの質問は私の前にありました。 ランディングページについて何かを聞いて、自分自身や会社のために作ることにした多くの人々が彼らに出会うと思います。



しかし、ランディングページのリクエストに応じてインターネット上には多くの広告情報とオファーがあり、カタログ化され構造化されたデータはほとんどありません。 したがって、私はこの記事を書くことにしました。 この問題を明確にしたいと思います。







リンク先ページとは



ランディングページは常に、1つのアクションを必要とする1ページのサイトです。



明確なメッセージで広告キャンペーンを実施し、消費者への具体的な提案を行い、必要なアクションをすばやく実行する人が必要な場合は、ランディングページを使用する必要があります。



あなたがクリスマスツリーを売っていて、あなたのクリスマスツリーをすぐに買う人が必要だとしましょう。 さらに、料理、おもちゃ、文房具も販売していますが、現時点（季節、新年の近づき）では、キャンペーンは特にクリスマスツリーの販売を目的としています。 そして、ランディングページを作成し、製品に関するすべての必要な情報を示し、広告キャンペーンを開始すると、人々はリンクに従って製品をすばやく購入できます。



つまり、ランディングページでは、通常、自分自身、会社、商品、サービスを宣伝するのではなく、特定の目的の特定の商品を宣伝します。



ランディングページとサイトの同じページの違いは何ですか？



  • まず、ランディングページで、訪問者は決定を下すために必要な情報を非常に簡潔に確認します（ランディングページが正しく行われている場合）。
  • 第二に、CMSが許可しない場合、特定の情報を追加できます




リンク先ページは何で構成する必要がありますか？



ランディングページには、独自の構造と特定の要素の標準セットがあります。 ランディングページのすべてのコンポーネントと、開発時に注意する必要があるニュアンスを詳細に検討します。



1.タイトル


タイトルには、提案の本質を明確に表す最も簡潔な情報を含める必要があります。 これは、ユーザーがランディングページで最初に読むものです。 それを広告スローガンと呼ぶことができます。 見出しがすべての必要な情報に適合しない場合は、小見出しを書きます。 何らかの在庫管理プログラムを販売している場合、見出しは次のようになります。



在庫に問題がありますか？

問題の解決策を購入してください！



2.写真


写真は千の言葉に値します。 訪問者の注意を引き、製品の視覚的にポジティブなイメージを作成する必要があります。



この画像では、プログラムについて話している場合、たとえばモバイル版、iPad版、コンピューターなど、最も成功したスクリーンショットがあるはずです。 この写真を使用して、プログラムがさまざまなデバイスでサポートされていることを訪問者に伝えます。



3.オファーの説明


もちろん、ランディングページには、提案に関するテキスト情報を含める必要があります。 それは1つのパラグラフまたは詳細な説明であり得、すべてあなたの目標に依存します。 ここでの主なことは、製品の説明に夢中になることではありません。 あなたの仕事は、製品のすべての詳細を調べることではなく、バイヤーがさまざまな方法で行動するよう説得することです（割引、試用製品など）



たとえば、前述のプログラムについて話す場合、ランディングページの説明は次のようになります。



3回クリックするだけでプログラムを入手できます。



1.登録する

2.ダウンロードリンクが記載されたメールを受信する

3.プログラムを使用して効果的になる



4.レビュー


フィードバックは、ランディングページの重要な要素の1つです。 この情報は、潜在的なクライアントとの信頼を構築するために必要です。 訪問者があなたの製品を購入したい場合、他の顧客がすでにそれを利用して満足していることを確認すると、彼の信頼度が上がり、購入する可能性が高くなります。



5.ロゴ


会社のロゴはランディングページに表示する必要がありますが、メインサイトに「誘導」してはなりません。 リンクではなく単なる写真である必要があります。理解することが重要です。



6.地下のソーシャルネットワークアイコン


また、これらのアイコンは、ソーシャルネットワーク上のプロフィールへのリンクであってはなりません。 これは、これらのリソースで訪問者を友達として追加する形式にする必要があります。 なぜそうなのでしょうか？ ユーザーがランディングページのページを離れないように、彼はあなたの特定の製品とオファーのみに集中します。



7.情報フォーム


レコードフォームには、クライアントから受信する情報を入力するためのフィールドが含まれており、データベースでさらに処理して保存できます。



8.ボタン


これにより、システムでの作業が終了します。訪問者は、フォームに入力した後、承認、送信、サインアップ、購入などのボタンをクリックする必要があります。 クリックして初めて、フォームにクライアントが記入した情報を受け取ります。



9.クライアントがアプリケーションを送信した後のフォーム


クライアントによる主なアクションの完了後、彼は通知フォームを見るはずです。 書籍をダウンロードするためのリンク、または次回の購入時に15％割引のクーポンへのリンクが既に存在する場合があります。これらはすべて、販売する製品と目標によって異なります。



ランディングページを構成する要素に関するいくつかのルールがあります。



1.リンク、ボタンのいずれも別のリソースにつながるべきではない


ランディングページ自体は、ユーザーの最後の立ち寄り場所です。 クライアントはアクションを実行する必要があり、別のWebサイトまたはソーシャルネットワークにアクセスしないでください。 集中して、集中して、もう一度集中してください！



2.要素の配置は創造的なアプローチです


AIDA（注意、関心、欲求、行動）などのサイト上の要素の順序を順守する人もいます-注意、関心、欲望、行動。 しかし、標準フレームワークに適合しない何らかの種類の提案があれば、心配することなく、要素の順序を実装します。



3.レビューがない場合の対処方法


新しい製品を取引していて、まだレビューがない場合は、2つのオプションがあります。



  • サイトのレビューを含めないでください
  • 会社全体またはあなたに関連する顧客レビューを投稿する（このようなレビューには、あなたの能力、サービスなどに関する情報が含まれる場合があります）




リンク先ページの種類



ランディングページがどの要素で構成されているかを調べました。ここで、「ランディングページ」のタイプ、それらに含まれる要素、構造、相互の違いを理解します。 ランディングページには5つの主要なタイプがあります。



1.リードジェネレーションのランディングページ




このタイプのランディングページの目的は、潜在顧客である潜在顧客に関する情報を収集することです。 たとえば、名前、電話、電子メール-これは、将来リーダーに連絡するために最低限必要な情報です（呼び出し、提案のコンテキストに基づいて手紙を送る）。



リードジェネレーションのランディングページには、フォームと、完成した情報をサーバーに送信するためのボタンが常にあります。これは、ランディングページの所有者であるセラーに送信されます。



















2.クリックのランディングページ（クリックスルーページ）


このタイプのページの目的は、クリックを収集することです。 人が商品を購入するためのページに行くために使用されます。 ランディングページで製品またはサービスを宣伝しており、[購入]または[製品ページに移動]ボタンは1つしかありません。



ランディングページにはバスケットと支払い機能はありませんが、訪問者を製品ページに誘導して、そこで注文することができます。



このタイプのランディングページが使用される理由 訪問者を購入のために「ウォームアップ」するために、会社の他の製品やサービスの注意をそらすことなく、あなたのオファーや製品をすぐに知らせて、ボタンを押したいという欲求を引き起こします。



検索結果（Yandex.Directを使用している場合）に購入したフレーズがあり、それに従ってクライアントがランディングページに移動し、他の製品に注意を散らさずに商品に慣れると、購入に「熟し」ます。 そして、購入の準備ができたら、ボタンをクリックして、メインサイトにある製品注文ページに移動します。



このタイプのページには、クライアントにデータを入力するためのフォームはありません。準備されたクライアントを製品注文ページに送信するボタンのみがあります。



3.スクイーズページ（スクイーズページ）




このタイプのページの主な目的は、将来のニュースレターのメールアドレスを収集することです。



原則として、このようなページには、大きな画像、タイトル、電子メールを収集するためのフォーム、および潜在的な顧客にとって会社が役立つ能力に関する簡潔な情報が含まれています。 したがって、このタイプは圧縮ページと呼ばれます。ランディングページの一部の要素はここでは省略できます。



ウェビナーを実施し、メールアドレスを収集する予定があるとしましょう。 ウェビナーへの潜在的な訪問者はメールのみを残す必要があります。これには、メールアドレスを入力するフィールドのみが必要です。 ウェビナーに関する大きな画像、タイトル、情報、メール入力フィールド、やる気を起こさせるテキストを含むボタンを配置します。 たとえば、見たい、または行くつもりです。



4.販売ページ




このタイプのページは、商品の販売に使用されます。 他のタイプとどう違うのですか？ このページには、顧客が決定を下すために必要な製品、サービスに関する詳細情報が常にあります。 これらは通常大きなページです。



ここで価格について話しています。 訪問者は、ボタンをクリックして、支払いフォームにアクセスします。 このランディングページは、オンラインストアのページにつながるものではなく、支払い方法に直接つながるか、支払い方法がページに直接記載されています。 訪問者は、たとえば銀行カードの詳細を入力し、製品またはサービスの代金を支払います。 ここでの主な強調点は、人がすぐに製品を購入したという事実です。 そしてここでは、特に製品が安くない場合、決定を下すためにより多くの情報が必要です。



5.ブライトページ（スプラッシュページ）




これらは「明るい」フィードのあるページです。 ここでの重点は、フォームへの記入ではなく製品にあります。 残りのページがフォーム（購入、登録など）に焦点を合わせている場合、ここで製品とその利点が注目されています。 この場合、クライアントが製品について学習し、それを記憶することが重要です。 彼はいくつかの方法でページを離れることができますが、あなたの製品について学んだ後、それがどれほど素晴らしいかについて。



リンク先ページのワークフローと作業の推奨事項



次に、サイトの仕組みを検討します。 ランディングページプロセスは、次の4つの段階に分けることができます。



  1. 着信トラフィックを受信する
  2. A / Bテスト
  3. リンク先ページの訪問者の仕事
  4. 目標達成後のリクエスト処理-変換後処理




私と私のクライアントの経験に基づいて、これらの各段階についての提案をします。



1.着信トラフィックを受信する




Yandex.Directの広告のトラフィックは、Webサイトのバナー、電子メール、またはソーシャルネットワークを介して受信します。



紹介元でユーザーを分けることをお勧めします。 電子メール配信用に個別のページを作成し、Yandex Direct用に個別に、検索エンジンまたはソーシャルメディア用に個別に作成してください。 なぜこれが必要なのですか？



  • まず、これにより、最も効果的なリードのソースを測定および特定します。
  • 第二に、人々はさまざまな方法でさまざまな検索ソースからの情報を認識し、期待しているためです。

    メールニュースレターから来た人は1人の消費者です。 検索エンジンを介してあなたに来た人は、異なるタイプの消費者です。 同じオファーと同じ製品が存在する場合でも、情報を表示するための2つの異なるオプションがあり、それに応じて2つの異なるランディングページがあります。




ニュースレターは、検索エンジン広告よりもはるかに多くの情報を伝えることができます。 ユーザーがニュースレターをクリックした場合、彼はすでに多くの記事を読んでいたので、この場合、ランディングページでそれを繰り返さないことが重要です。 また、ユーザーが検索エンジンの広告をクリックした場合、広告に収まるものがあまりないため、より多くの情報を表示する必要があります。



2. A / Bテスト




トラフィックを分割した後、A / Bテストを実施する必要があります。 これは何ですか これは、さまざまな顧客にさまざまな形式のページを表示することを意味します。 たとえば、1つのフォームは長いランディングページで、もう1つのフォームは短いランディングページです。 そして、どちらがより効果的かを検討しています。 ページの「長さ」は、製品の説明のサイズによって決まります。



60人のユーザーが私たちのサイトを訪れたとしましょう：1つのフォームで30人、別のフォームで30人。 それらの変換は異なり、フォームの有効性に応じて、より多くの顧客を引き付けたものを選択します。 最初から35人が来て、2番目から15人が来た場合、もちろんランディングページの2番目のバージョンを使用します。



3.ランディングページの訪問者ジョブ




ここでどのようなニュアンスを考慮すべきですか？ ランディングページを開発する場合、将来のユーザー作業は「予測」されます。 したがって、ページを設計するときは、もちろん、要素の場所と存在を考慮する必要があります。 しかし、誰も創造的な仕事をキャンセルしませんでした。 明確なテンプレートがまだないことを忘れないことが重要です。それはすべてコンテキストに依存します。あなたは何を売り、誰に売るかを理解する必要があります。



ここでは、デザイナーの専門知識を使用することを強くお勧めします。 まず、クライアントの興味を引く方法について考えてください。ルールに従う方法については考えないでください。



ランディングページを設計するとき、覚えておく必要があります。ページが長くなればなるほど、クライアントの信頼は大きくなりますが、焦点は小さくなります。 したがって、短いほど良いです。 人には注意の限界があることに留意する必要があります。



訪問者があなたのページで作業し、手順を完了したら、4番目のレベルである変換後処理に進みます。



4.変換後の処理




訪問者がターゲットボタンをクリックした場合はどうなりますか？ 彼との仕事はそこで終わるべきではありません。



フォームに入力して送信した場合、ファイルをダウンロードしたり、次のステップに移動したり、ソーシャルネットワークのプロファイルに移動したりする機会を与えることができます。そして、フォームに入力せずにボタン（ページタイプ-クリック-ページを介して）、それを私たちのサイトのバスケット（オンラインストア）または登録フォームに転送できます。



リンク先ページの開始時のエラー



クライアントと実際によく遭遇する間違いについて警告したいと思います。



1.リンク先ページはリンク先ページではありません

会社、製品、利点について可能な限り多くの情報を提供しようとし、それをランディングページと呼びます。 しかし、これは実際にはランディングページではありません。



ランディングページは、1つのトピックに関する圧縮された情報を意味します。 連絡先、サービス、多数のレビュー、顧客を示し始めると、これはランディングページではなく、1ページのサイトです。これは、ランディングページが示す制限に準拠していないためです。



ランディングページに会社の情報を投稿する必要はありません。 主なものは、製品と最高のオファーです。



2.あまりにも詳細な製品説明


クライアントのランディングページの一部は、技術専門家、エンジニアの指導の下で作成されました。 このような専門家は、原則として、情報を提示する際に非常に細心の注意を払っています。 その結果、そのような説明には、詳細な技術仕様と操作手順が含まれる場合があります。



はい、彼らは1つのテクノロジーと1つの製品について話しましたが、それらはランディングページではなく、単にサイズを怖がらせた巨大な1ページのサイトでした。



クライアントが製品について話す必要はないが、あなたが必要とする行動にそれを傾けることを覚えておくことが重要です。



3. A / Bテストを使用しないでください


A / Bテストを使用して、さまざまなページオプションを試してください。 人々はしばしば考えます：すべてのリードが金でその価値があるなら、私は私が最も好むと思う私が最も好きなオプションを使用します。



いいえ、あなたはこれを知ることができません。 あなたがそれについてどう考えているかに関係なく、いくつかのことが予期せず動作します。 あなたは何かを知らないかもしれません。
