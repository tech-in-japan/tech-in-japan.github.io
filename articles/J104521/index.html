ネットブックのウェブサイトの背景とパフォーマンスの問題

最近まで、私は、現代のコンピューターが、対話性、ajax、およびフラッシュを大きく乱用しない通常のhtml / cssサイトを簡単に粉砕することを確信していました。 それは判明しました-私は間違っていました（不思議ではありません-私はこの問題の専門家ではありません）



突然、容量がまったく異なるネットブックがあり、まったく複雑でないサイトでつまずくことが判明しました。 たとえば、 私のプロジェクトのサイト（非常に普通のWordPressブログ） は、読者の1人に問題を引き起こしました 。



理由は、判明したとおり、静的でスクロールしない背景でした。 これにより、ネットブック上のIE8で表示したときに大きな阻害が発生しました。 コンテンツと一緒にスクロールする通常の背景は、そのような効果を引き起こしませんでした。



ネットブック用のサイトの特別なバージョンを作成することはおそらく意味がありません。したがって、弱いコンピューターの所有者の生活を楽にするために、背景プロパティの値をオーバーロードしてユーザーcssを使用できます。 背景をスクロールするには、 background-image:none;



という行を削除しbackground-image:none;



、単に背景を削除する場合-そのままにして、テキストファイルに保存します。



body

{

background-attachment: scroll;

background-image:none;

}









次に、このファイルが接続されます（もちろん、この情報はサイトのユーザー向けです）。



* IE8：[ツール]-> [一般]-> [アクセシビリティ]-> [スタイルシートを使用してドキュメントをフォーマット]を選択し、上記のcssでテキストファイルを選択し、[OK]を2回押してメニューから戻ります。 次に、ブラウザを再起動する必要があります。

* Chrome：ChromeにはカスタムCSSを読み込むための組み込みオプションがありません-これは拡張機能（ 1、2、3 ）のいずれかを使用して実行できます

* Opera：サイトが開いているときにコンテキストメニュー、[表示]タブ-ユーザーcssを呼び出します。

* Firefox：Userconternt.cssファイルをプロファイルフォルダー（ http://www.mozilla.org/unix/customizing.html ）に保存するか、 同様のエドンを使用する



この小さな発見が役に立つことを願っています。
