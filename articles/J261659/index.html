Windows 2012 R2、Traffic Inspector、Kaspersky Endpoint Securityのコラボレーションのテスト



そして最後に、Smart-Soft技術サポートが実施した互換性テストケース。



テストの目的で、Windows 2012 R2、Traffic Inspector 3.0.1、Kaspersky Endpoint Security製品がインストールされ、日中にHTTP-Testユーティリティを使用してトラフィックが実行されました。 実験中に問題は見つかりませんでした。



Kaspersky Endpoint Securityは、デスクトップ（Windows 7、8.1）およびサーバー（Windows 2012 R2）オペレーティングシステムを使用する仮想マシンでテストされました。 Traffic Inspector 3.0.1がサーバーオペレーティングシステムにインストールされ、ゲートウェイモードで構成され、クライアントが構成され、トラフィック制御ルールが作成されました。 クライアントマシンは、プロキシを介して動作するように構成されました。



Kaspersky Endpoint Security for Businessがアンチウイルス保護として選択されました。 インストールを開始する前に、Kaspersky Webサイトから管理サーバーのディストリビューションをダウンロードしました：クライアントコンピューター用Kaspersky Endpoint Security for WindowsおよびKaspersky Security Centerの管理ツール。 Kaspersky Endpoint Securityは最適なデフォルト設定でインストールされているため、インストール後すぐにシステムとデータが保護されます。



その後の設定および保護管理は、Kaspersky Security Centerの単一のコンソールを使用して実行され、すべてのワークステーションの監視と保護が簡素化されました。 特に、企業のモバイルデバイス（スマートフォン、タブレット）の保護機能の存在に注意する必要があります。 単一のコントロールセンターを使用すると、リソースを合理的に使用でき、さまざまな種類の脅威に対するソフトウェアの復元力が向上します。



ウイルス対策スキャンテクノロジは、オペレーティングシステムのさまざまなレベルで機能し、マルウェアを効果的に削除します。 Kaspersky Security Networkクラウドネットワークは、リアルタイムで新しい脅威からユーザーを保護します。 同時に、柔軟な監視ツールを使用すると、承認されたデバイスのみがネットワーク上のPCに接続されていることを確認できます。



Kaspersky Endpoint Securityをインストールしても、TIおよびクライアントマシンの動作に影響はありませんでした。 プログラムアクティビティ監視システムは、TIモジュールをブロックしませんでした。



クライアントマシンからの負荷テストは、HTTPテスト、HttpTrafficGenユーティリティを使用して実行されました。



また、Kaspersky Endpoint Securityをインストールする前とインストールした後のインターネット接続の速度も測定しました。 測定は、速度speedtest.netをテストするためのサイトを使用して実行されました 。 また、次の例のように、明らかに大きな流通チャネルを持つインターネット上にあるFTPサーバーを使用しました。



devbuilds.kaspersky-labs.com/devbuilds/AVPTool

ftp.drweb.com/pub/drweb/livecd

www.nvidia.ru/content

ftp.altlinux.ru/pub/distributions



クライアントとTIサーバー間のトラフィックをテストするために、すべてのシステムで動作するiperfユーティリティが使用されました 。これはオープンソースで無料であり、Windowsでのインストールは不要です。 プログラムは非常に正確です。結果は、複数の測定値で互いに一致します。 テストされたチャネルの両側にTIとクライアントマシンを備えたサーバーで実行しました（一方ではサーバーモードで、他方ではクライアントモードで）。 この場合、速度（チャネルの帯域）は、クライアントからサーバーへの一方向でのみテストされます。つまり、クライアントはデータを送信し、サーバーは受信します。 反対方向のチャネル帯域をテストするために、サーバーとクライアントはチャネルの両端で交換します。 iperfは、「存在しない」アドレスへのトラフィックを生成できます。 これにより、チャネルまたはネットワークインターフェイスが必要なサイズにアップロードされました。



おわりに



典型的な潜在的なITの脅威には、外部（ウイルス、スパム、ハッカー攻撃、スパイウェアなど）だけでなく、内部の脅威（機密情報の盗難、リソースの悪用）も含まれます。さらに、統計によると2番目のグループが最も危険で割合成長しています。InfoWatchAnalytical Centerのレポートによると、2014年のロシア企業からの情報漏洩の数は2013年の結果より73％増加しました。 また、大企業が金銭的および評判の損失に対処できれば、中小企業にとって損害はより顕著であり、破産につながる可能性があります。 これは、情報セキュリティの脅威を防ぐ正当な理由です。



中小企業のマネージャーや技術スペシャリストは、タスクとITセキュリティを整理するための統合ソリューションを探すために貴重な時間を費やす必要がなくなりました。
