MacOSでビデオコンテンツを再生する-Movist

Macへの移行後、いくつかの問題を引き起こした問題の1つは、HDを含むビデオコンテンツを再生するためのプレーヤーの選択でした。 Windowsでの選択が十分に大きい場合（ほとんどの場合、GOM PlayerまたはMedia Player Classic + K-Lite Codec Packを使用しました）。 Macでは、これらのプログラムに限定されていました。







それらを詳細に停止する価値はおそらくないでしょう、あなたはすでにそれらすべてを知っています。 何らかの理由で気に入らなかった小さなコメントだけを書きます。



  • QuickTime + Perian-一部のファイルで愚か、サウンドの問題、クラスとしての設定の欠如、 わかりやすい制御の欠如。
  • プレックス -すべての利点があるため、この200メガバイトのモンスターをプレイヤーと呼ぶことは非常に困難です。 このような機能（ala Windows Media Center）を使用すると、HTPCの一部としてその場所がより可能性が高くなります。
  • VLC （マルチプラットフォーム、 ffmpegを使用）は以前お気に入りでしたが、mkvおよびaviファイル（vlcよりもffmpegに問題がある可能性があります）に問題がありました。

最後に、VLC for Macの運命に関するニュース。



人気のあるオープンソースVLCクロスプラットフォームメディアプレーヤーを開発しているVideoLANチームは、Macバージョンの製品の開発は近い将来中断される可能性があると述べました。 その理由は簡単です。適切なレベルの資格とターゲットプラットフォームの知識を持つボランティア開発者がいないことです。



MacバージョンのVLCで最も問題のある開発領域は、ユーザーインターフェイスです。 開発者によると、Cで書かれた製品のクロスプラットフォームコアとほとんどのプラグインは非常によく機能しますが、Mac OS Xのグラフィカルインターフェイスについては言えません。Objective-Cで書かれた場合、VLCコード全体の1％しか占めません。 これはまさに、開発者がMac OS X用VLC 1.0.xの多くの問題を説明し、バージョン1.0.3での64ビットモードのサポートを中止したことです。



同時に、開発者はVLCが常にMac OS Xと十分に統合されていないことを認識しており、VLCコアの代替シェルであるLunettesと呼ばれるもので作業を開始することを余儀なくされました。



それにもかかわらず、この製品は開発の初期段階にあり、最初の予備バージョンは1月にのみ登場します。 かなり予想外に、VLCチームは、Appleの製品に対する非友好的な態度に不満を漏らしました。
