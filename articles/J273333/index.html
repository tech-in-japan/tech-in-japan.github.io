パターン



Goには、htmlページを生成するための非常に便利なパッケージがあります-html / template。 仕組みを理解する最も簡単な方法は、 この例を試してみることです 。 通常、テンプレートは個別のhtmlファイルに保存されますが、play.golang.orgの例では、わかりやすくするためにテンプレートをgo-codeに挿入しています。



テンプレート情報のほとんどをgohugo.io/templates/go-templatesから取得しました



テンプレートはこちらです。 各テンプレートは、その名前の定義{{define "templateName"}}で始まり、{{end}}で終わります



テンプレートを使用するコントローラーは、 makeTemplate関数を介してhtmlコードを受け取ります。 htmlファイルの名前、テンプレートの名前（{{define}}で指定されている）、パラメーターを渡します。 



TemplateStr, err := makeTemplate("template_name", "templateName", &tplPage{ Alert: c.Alert, Lang: c.Lang, TxType: txType, TxTypeId: txTypeId})


TemplateStrで受信したバイトセットはcontent.goに転送され、その結果、ブラウザーは生成されたhtmlページを受信します



すべてのテンプレートのmakeTemplateに、signatures.html、alert_success.htmlなどの追加のテンプレートを含めます。 また、テンプレート内のデータを処理するために使用されるユーザー定義関数（FuncMap）を追加します。

JSコード



別の問題にはかなり時間がかかりました。 コントローラで何らかの種類のjs-codeを生成する必要がある場合、たとえば： 



 var x="1";


次に、{{。SomeJSCode}}を介してテンプレートに出力すると、次のようになります 



 "var x=\"1\";"


解決策1



noescapeはここでは役に立ちません。 新しいカスタム関数を追加する必要があります。 



 "js": func(s string) template.JS { return template.JS(s) },


そして、テンプレートで{{js .SomeJSCode}}を使用します



決定2



コントローラーで、template.JSタイプを使用します。 以前に変数文字列を変換した場合：template.JS（SomeJsStringVariable）。 この場合、テンプレート自体で{{.SomeJSCode}}を表示する必要があります
