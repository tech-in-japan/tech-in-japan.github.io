ParallelsとVMware War、昨日と今日





しばらく前に、Parallels Desktop 6 for Macの詳細を公開しました。これにより、3つの製品について多くの議論が巻き起こりました。 同時に、有料のParallels DesktopとVMware Fusionは常に互いに直接の競合相手であり、無料のVirtualBoxは際立っていました。詳細については説明しません。代わりに、いわば競合の履歴を調べます。



基本的なApple Boot Camp製品は、主に2つのオペレーティングシステムを同時に起動する機能がないため、OSを変更するためにコンピューターを再起動する必要があるため、いくつかの理由で本格的な仮想化製品と見なすことはできません。







だから、2006年にParallels Desktop 2という製品が空のMac仮想化市場に参入したとき、すべてが始まり、すぐにユーザーの関心と愛を呼び起こしました。 しかし、パイロット版には長所と短所の両方があり、それに応じて改善および修正されたため、バージョン2.5が登場し、すぐに3.0になりましたが、バージョン3.0に注意し、市場には1つの製品しかありませんでしたが、孤独は長くありませんでした。







2007年、VMwareはFusion 1.0をリリースしました。これは既存の製品よりも興味深いことが判明しました。これは、CNETが実施したテストで確認され、FusionはParallels Desktopよりも生産性が高いことがわかりました。 Mac OS X 10.4.10の8コアMacPro 2.66 GHzでは、Adobe Photoshop CS3を使用したテストで、新参者は競合他社のほぼ2倍の速度であり、マルチメディアテストはFusionの成功により、Parallelsの3倍以上の素晴らしい結果を示しましたバージョン2.0.1に確定。







このような状況では、Parallels Desktopは正しい結論を導き出し、より速く、より便利で、より安定した、おそらくは以前よりも安価な根本的に新しいソリューションを作成する作業を開始する必要があるように思われます。これらのタスクはすべて、すぐには解決されませんでした成功しました。 2008年の初めに、最初のビルドの書き直されたParallels Desktop 4.0が表示されます。これは、速度、利便性、または最も重要な安定性のいずれでも区別されません。 すべての原因はアダプティブハイパーバイザーテクノロジーであり、これにより企業の市場の数十パーセントが遅れましたが、結局のところ、リハビリを可能にしました。



2009年3月に、MacTech誌はVMware Fusion 2.0.1およびParallels Desktop 4.0 for Mac（ビルド3540）のパフォーマンステストを実施し、悪名高いAdaptive HypervisorはFusionで使用されるアップグレードソリューションよりも平均で高速であることを示しました。







次の2010年の初めに、同じMacTechがそれぞれ次のバージョンのFusion 3.0とParallels Desktop 5をもたらし、Parallelsは無条件の勝利を勝ち取りました-Windows、プロセッサパフォーマンス、ネットワーク、ゲーム、グラフィックス、アプリケーションのロード、これらはすべてはるかに高速ですプログラムのバージョン5で機能し、バージョン6はすでにメーカーのWebサイトで販売されています！ VMwareは、価値のあるソフトウェアの答えではなく、Macユーザーを並列化するためのダンププログラムを発表しただけなので、以前のバージョンとしか競合できないようです。まあ、このプログラムはほんの少しの遅延プログラムであり、VMware Fusion 4のプレゼンテーションは既に遠くない。
