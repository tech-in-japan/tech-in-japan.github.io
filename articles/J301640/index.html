SEOタスク用のIPythonノートブックデータ処理





SEO、SMM、マーケティングの分析タスクを実行するとき、私たちはデータ処理ツールの圧倒的に増加する数に直面しました。 それぞれは、その機能またはユーザーアクセシビリティに合わせて調整されています。ExcelとVBA、サードパーティのSEOツール、PHPとMySQL、Python、C、Hiveなどです。 カウンター、広告システム、CRM、YandexとGoogleのウェブマスターツール、ソーシャルネットワーク、HDFSなど、さまざまなシステムとデータソースが問題を追加します。 セットアップと使用の容易さ、データの受信、処理、視覚化、およびさまざまなタイプのソースの操作を行うモジュールを組み合わせたツールが必要です。 iPythonノートブック（最近Jupyterノートブック）が選ばれました。これは40のプログラミング言語でスクリプトを操作するためのプラットフォームです。 このプラットフォームは、データ処理および機械学習の専門家の間で、科学計算に広く使用されていました。 残念ながら、Jupyterノートブックは、マーケティングタスクの自動化とデータ処理に使用されることはほとんどありません。





SEOのWeb分析およびデータ処理タスクの場合、Jupyterノートブックはいくつかの理由で適しています。

-簡単なセットアップ

-コードを記述する必要のないデータ処理と視覚化

-パーソナルコンピューターリソースが不足しているため、より強力な仮想マシンで（たとえば、Amazon Webサービスを使用して）ノートブックを実行し、スクリプトコードを変更せずに必要なデータを計算できます。



この記事では、Jupyterノートブックを使用して実用的な問題を解決するのに役立つ3つの例を紹介します。

Word2vec

（ サンプルコード ）

Word2Vecライブラリを使用するには、まずgensim pypi.python.org/pypi/gensimを追加インストールします（デフォルトではAnacondaディストリビューションには含まれていません）。 モデルの入力時に、検索フレーズの初期リストからの文のリストが提供されます。 つまり、[['watch'、 'films'、 'online']、['course'、 'ruble']、...]という形式のシートです。



次に、モデルパラメーターを設定します。

-num_features-ベクトル空間の次元。 値が大きいほど、モデルが入力データを考慮する「精度」が高くなります（ディメンションを増やしてもモデルの品質が向上しない場合があります）。 通常、10〜数百の値が使用されます。 したがって、ディメンションが大きいほど、より多くのコンピューティングリソースが必要になります。

-min_word_count-モデルの最終辞書で頻繁に出現する単語のみを考慮することができます。 ほとんどの場合、5〜100の値が使用されます。その結果、実用的な意味を持つ単語のみを残して、辞書のサイズを大幅に削減します。

-num_workers-モデルが並行して構築するプロセスの数

-context-アルゴリズムが考慮するコンテキスト内の単語数。 検索語は非常に短い「文」です

-ダウンサンプリング-テキストに頻繁に出現する単語を除外します。 Googleでは、.00001〜.001の値を推奨しています



この例では、モデルは500万で構築されました。 2GBの空きRAMを搭載したラップトップで約40分間の検索クエリ。 この量のデータは、SEOタスクに使用できます。

Neworkxでグラフを操作する

（ Vkontakte APIデータアップロードコード、 処理、および視覚化 ）

専門用語では、グラフはエッジで接続されたノードのセットです。 実際には、ノードはソーシャルネットワークまたはグループのユーザー、サイトのページにすることができます。 rib骨として-一対のユーザー間の友情の存在、メッセージ、グループ投稿のようなマーク、サイトの他のページへのリンク。 Networkxライブラリを使用すると、このようなグラフを作成し、さまざまなグラフの特性を読み取ることができます。 VKontakteグループの例を見て、各ユーザーのPageRankを計算し、ブラウザーで視覚化する方法を見てみましょう。



例として、660の参加者（視覚化のため）の比較的小さなグループの1つを取り上げました。このグループでは、多くの参加者が互いに精通しています。 グラフを作成するには、グループメンバーのリスト（VKontakteのgroups.getMembers APIメソッド）をアンロードし、参加者ごとに友人のリスト（friends.getメソッド）を取得するだけで十分です。 アップロードの結果は、次の形式でテキストファイルに書き込まれます。

{

ユーザーID

[ユーザーの友達リスト]

}



その結果、グラフgのノードはグループメンバーのIDになり、エッジはこのユーザーの友人のIDになります。 PageRankを計算するには、関数x = networkx.pagerank（g）を使用します。 グループのトップメンバーを表示します。





視覚化には、 D3.jsライブラリforce-collapcibleを使用します。 ノードのサイズとして、そのページランクを取得します。





ノードにカーソルを合わせると、そのノードが属しているIDを確認できます。 users.getメソッドを使用して、IDごとにユーザーに関する詳細情報を表示できます： https ://api.vk.com/method/users.get?user_id=12345
