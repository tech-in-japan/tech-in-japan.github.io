XenApp：インストールとセットアップのエクスペリエンス

こんにちは、ハブロフチャネ。



前文


XenApp（ZenApを参照）は、独自のCIAプロトコルを使用してシンクライアントを介してアプリケーションを仮想化および配信するために、Citrixによって発明されました。 理論的には、邪悪な帝国やその他の貪欲ではないベンダーからライセンスソフトウェアを購入するなど、首の多くの痛みを取り除くため、アイデアはそれ自体が良いと言わなければなりません。 もちろん、XenAppライセンスは非常に高価なので、これはアプリケーションをレンタルする場合にのみ興味深いものです。 XenAppはWindowsでのみ機能することに注意してください。

残念ながら幸運にもわかりませんが、最も消費されているソフトウェアの最大のメーカーはすべて、Webインターフェイスまたはシンクライアントから利用できる独自のフル機能の製品を持っているので、別の製品としてのXenAppはすぐにエンドユーザーにとって興味のないものになります消費者、およびレンタルアプリケーションに従事している企業の視野に残ります。 レンタルの申し込みだけで、会社に賭けました。

この興味深い取り組みに戸惑い、GoogleはXenAppについてロシア語でわかりやすい情報を提供していないことに気付きました。喫煙できるのは、Citrix.comの英語の広範なマニュアルだけです。 しかし、これは私たちにとって本当に問題なのでしょうか？

「隣人を助ける」という感覚に導かれ、XenAppのインストールと構成を説明することにしました。英語のマニュアルと、Citrixの専門家である新しい友人を使用して、自分で選択する必要がありました。



注意！

XenAppが正常に機能するには、公式のCitrixライセンスが必要です。

公式のCitirx担当者に連絡して、パートナーになり、アプリケーションレンタルサービスを提供するために製品を試用したいことを伝えてください。 したがって、90日間は多くのライセンスを、1年間は100ライセンスを取得できます。



覚えておくべき重要事項：

ライセンスはコンピューター名で発行されるため、慎重に使用してください。 一般に、ライセンスについては、おそらく別の記事を書く必要があります。 それは見かけほど簡単ではありません。



鉄の要件


公式に発表されたCitrix System Requirementsシステムを信じている場合、XenAppをインストールするには、次のようなものが必要です。



Intel Extended Memory 64テクノロジーを搭載したIntel Pentium Xeonファミリを備えた64ビットアーキテクチャ

AMD OpteronファミリAMD Athlon 64ファミリ互換プロセッサ



そしてまた：



メモリ：512MB RAM（最小）ディスク容量：32GB（最小）TROLLFACE



同義語でしかロシア語に翻訳できませんでした：



-プロセッサは64ビットで、仮想化テクノロジ（Intel VTまたはAmd-V）をサポートし、少なくとも2つのコアがあります。

-少なくとも2ギガバイトのメモリ、そしてもちろん6つすべてが優れています。

-HDD少なくとも60ギガバイト、SATA。

-ネットワークカード、Intel Proシリーズ、ギガビット。





ソフトウェア要件


必要なソフトウェアから：



-Windows Server 2008 R2 x64 DCE。ただし、Enterpriseも動作しますが、最新のアップデートがインストールされています。

-XenApp 6.5 Platinum Editionの配布（Citrix Webサイトからダウンロードするには登録が必要です）。

-Microsoft NetFramework 3.5（この場合、ダウンロードする必要はありません。XenAppはWin2k8R2上のIISの役割でそれ自体を有効にします）

-提供されたアプリケーションの配布（MS Officeまたは1Cなど）。



また、XenAppディストリビューションでは、次のような多くのソフトウェアがインストールされます。



-Microsoft Visual C ++ 2008 SP1再頒布可能パッケージ（x64）

-Microsoft SQL Server 2008 Express R2

-Microsoft Primary Interoperability Assemblies 2005

-など、これは問題が発生するまで気にしないでください:)



そして、役割を高めます：



-Webサーバー（IIS）

-Microsoftターミナルサービス



さらに、Windowsコンポーネントを誤ってインストールすると、何も機能せず、Windowsを取り壊して再インストールする必要があり、そのために多くの時間を費やすことになります。



重要！

Windowsをインストールして構成し、最新の更新プログラムをインストールした後、システムイメージをacronisまたは別の同様のプログラムにすることをお勧めします。これにより、アクションが失敗した場合に、クリーンなシステムを簡単にロールバックし、インストールを再び続行できます。 私の場合、それはクラウドなので、完全に準備されたシステムから愚かなテンプレートを作成しました。

XenAppのインストールを開始する前に、サーバーに必要な名前を事前に割り当てることも非常に重要です。



デフォルトでは、CitrixはXenAppがWindowsドメインに入力されたサーバーに展開されていると想定します。 つまり ネットワークにドメインコントローラーが存在することを前提としています。 しかし、実際には、XenAppはコントローラーなしで簡単にインストールでき、ADを備えたネットワークよりも悪くないワークグループで動作します。 この決定は、最終的なニーズに大きく依存します。



また、XenAppを搭載したサーバーはNAT-thの背後にあり、プロバイダーが発行した特定の外部IPアドレスを持っていると想定されています。 このIPアドレスを紙に書き直すことができますが、それはまだ役に立ちます。



設置


試行錯誤によって、また二日酔いやピンドスでさえマニュアルを読むことで、サーバー上のXenAppが正確に上昇し、それ自体を構成できることを100％保証する明確な一連のアクションが定義されました。 生産については黙っていますが、まだこれに到達する必要があります。 したがって、秘密のシーケンスは次のとおりです。



何よりもまず、Windows Server 2008 R2をインストールおよび更新します。 次に、Microsoft NetFramework 4をダウンロードしてインストールし、システムを再度更新します。 次に、XenAppディストリビューションのフォルダーに移動し（ISOイメージが既にネジの別のフォルダーに解凍されていると仮定）、autorun.exeというファイルを実行します:)。





その後、インストーラーは最初のサービスパックでNetFramework 3.5をインストールするように提供します。 同意します（[OK]をクリックします）。





次に、ロールをインストールする必要があります。 これは、明らかなキャプション「サーバーの役割を追加」のボタンを押すことで実行されます。





このボタンをクリックして、Platinum Editionをポイントし、Agrarianライセンスに同意して、次のロールを選択します。



ライセンスサーバー

Xenapp

Webインターフェース





次のタブで「次へ」をクリックして、選択します



XMLサービスIIS統合（必須コンポーネント！）。





これ以上選択するものはありません！

[次へ]、[インストール]の2回クリックして、終了するまで待ちます。 コーヒーと煙を飲むことができます。





短時間のインストール後、XenAppは感嘆符の付いたウィンドウと「完了」ボタンを表示します。





これには何の問題もありません。再起動するだけです。 すべてのウィンドウを閉じて、サーバーを再起動します。 再起動後、セットアップは自動的に作業を再開します。「インストールの再開」を選択し、最終的に完了するまで待つ必要があります。





注意！ すべてのコンポーネントがエラーなしでインストールされるはずです！

つまり すべてのアイテムには緑色のチェックマークを付ける必要があります！ それ以外の場合は、システムを破壊して再度インストールする必要があります！



すべてのジェスチャの結果、インストールされたコンポーネントのリストを含むインストーラーウィンドウが表示されます。「configure」という単語が各コンポーネントの反対側に表示されます。



カスタマイズ


そのため、XenAppがインストールされ、構成を提供します。 プログラムウィンドウに次のように表示されます。



XenApp-ライセンスの指定

Webインターフェイス-構成

ライセンスサーバー-構成



ライセンスサーバー-構成から開始する必要があります。

デフォルトでは、Citrix XenApp Platinum Editionライセンスが必要です。 構成をクリックします。 彼はポートを設定することを提案します：



ライセンスサーバーポート：2700

ベンダーデーモンポート：7279

管理コンソールのWebポート：8082



ポートは変更しないでください！

管理者パスワードを設定し、[OK]をクリックします。 ライセンスサーバーには緑色のチェックマークが付いており、構成済みの状態になっています。





その後、ライセンスサーバーにライセンスを追加する必要があります。 これを行うには、スタート-すべてのプログラム-Citrix-管理コンソール-ライセンス管理コンソールに移動します。





開いたWebインターフェイスの右隅にある[管理]をクリックし、前に指定した管理者パスワードを入力します。 次に、左下の[ベンダーデーモン構成]セクションに移動し、[ライセンスのインポート]ボタンをクリックします。 ライセンスファイルを選択し、[ライセンスファイルを上書きする]チェックボックスをオンにして（このライセンスのみがあります）、[ライセンスのインポート]をクリックします。 次に、[OK]をクリックし、ライセンスのリストでライセンスを選択して、[ライセンスの再読み込み]ボタンをクリックします。





これで、ライセンスの構成が完了しました。 Webインターフェースを閉じて、インストーラーに戻ります。

インストーラーで、[ライセンスの指定]をクリックして、XenAppがライセンスサーバーと有効なライセンスを表示するようにする必要があります。





コンピューター（インストール先のコンピューター）の名前を入力し、「接続のテスト」をクリックして、「次へ」をクリックします。





XenAppがライセンスを認識した場合、変更する必要はありません。パラメーターが自動的に指定されます。 認識できなかった場合は、以前のすべての手順を新たに行う必要があります。 [適用]をクリックすると、[ライセンスの指定]が[構成済み]状態に切り替わり、緑色のチェックマークが付いていることがわかります。 [構成]をクリックして、XenAppサーバー自体を構成します。





なぜなら これはネットワーク内で唯一の新しいXenAppサーバーです。[新しいサーバーファームの作成]を選択します。 新しいXenAppサーバーファームを作成します。





ファームの名前を指定します。残りのオプションはデフォルトでこのタブに残します。 さらに、インストーラーは選択肢を提供します。新しいデータベースを作成するか、既存のデータベースを使用します。 なぜなら データベースがないことを前提としているため、[新しいデータベース]をクリックします。





その後、サーバー管理者のログインとパスワードを入力します（サーバーがドメイン内にある場合でもローカルのみ！）、常に[次へ]をクリックし、既定の設定のままにして、[適用]をクリックすると、データベースのセットアッププロセスが表示されます。 [完了して再起動]をクリックします。





再起動後、Web-Interfaceのみが未構成のままであることがわかります。 設定する前に、次のことを行う必要があります。



cmd: altaddr /SET __









インストーラーをオフにして実行します：



スタート-すべてのプログラム-管理-Citrix-管理コンソール-CitrixApp Center





開いたウィンドウで、次を選択します。



Authenticode署名チェックを無効にする





XenAppファームの構成ウィンドウが開き、[次へ]をクリックして、[シングルサインオン]の位置をオフにし、[次へ]をクリックし、[ローカルコンピューターの追加]をクリックします。XenAppがインストールされているサーバーを追加します。





私たちの場合、これはローカルコンピューターであり、これを追加します。





その後、インストーラーは、指定されたすべてのパラメーターに準拠しているネットワークとサーバーを常に検出し、すべてが自分に合っている場合は、[適用]をクリックすることを提案します。 クリックすると、XenApp管理コンソールが開きます。

一般に、XenApp自体の構成はこれで完了です。
