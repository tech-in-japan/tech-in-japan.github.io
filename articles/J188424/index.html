3GモデムとMikroTik

それはすべて、インターネットが自宅で消えたという事実から始まりました。 私は2人の有線オペレーターを接続していますが、両方とも使用できなくなりました。 その後、隣の家の電気がオフになったことが判明しました。 インターネットなしでさらに1時間遅れて、 ZTE MF112 3Gモデムについて思い出しました。 コンピューターに接続しましたが、これでは十分ではなく、アパートでWi-Fiを使用したかったのです。 このため、 MikroTik RouterBoard 951g-2hndに接続しました。 これが私が言うことです。

SMSの受信/送信



インターンがいますが、あなたはそれを支払わなければなりません、そして、このためにあなたはバランスを監視する必要があります。 これを行うには、 SMSアシスタントを使用しました。 USSDリクエスト（＃100＃を使用してMTSから残高を確認できます）を使用できますが、それらはコンソール、次の形式のコマンドを介してのみ使用できます。

AT+CUSD=1,#100#,15





応答として、次のものを取得します。

OK





そして、しばらくしてから次の行：

+CUSD: 0,"00420061006C0061006E00630065003A00370031002C0032003000720020",72





メッセージ「00420061006C0061006E00630065003A00370031002C0032003000720020」は、 このリソースを使用して復号化できるUCS-2エンコーディングのメッセージです。

このアプローチはあまり便利ではないため、SMSアシスタントを使用する方が適切です。



MTSで残高のあるSMSを受信するには、テキスト11のメッセージを番号111に送信する必要があります（これは、総当たりでチャネルが選択されたためです）。

tool sms send usb1 phone-number=111 message=11 channel=1







次に、着信メッセージを読み取り、私たちのメッセージを待ちます（ シークレットは任意の行と同じです）：

tool sms set channel=1 port=usb1 keep-max-sms=10 receive-enabled=yes secret=blabla





tool sms inbox print





WinBoxで








111から空のメッセージを取得します。実際には、MikrotikはSMSテキストメッセージのみを処理できますが、PDUはありません。

SMSアシスタントにメッセージをテキスト形式で送信させる必要があります。 これを行うには、テキストEngで番号+79126313431に SMSを送信します（これはMTSオペレータ用です）。



tool sms send usb1 phone-number=+79126313431 channel=1 message=Eng







私たちは見ます：

tool sms inbox print





# PHONE TIMESTAMP MESSAGE

1 111 Jul/30/2013 14:07:33 GMT +6

関税



MTSについてのみ説明します。 私は彼のSIMカードを持っていました（実際、モデムのようなものですが、これは修正可能です ）



料金： Super MTS 、そして最も重要なのは、MiniBITオプションです。これにより、必要な日にのみ、15ルーブルのインターネット料金を支払うことができます。 マイナスは5 MBの1日あたりのクォータであり、速度は32 Kbpsに低下します。



関税： Connect-4 （この関税への切り替えはすべての関税から利用できるわけではありません）およびオプション"Internet for a day" 、それはあなたが必要な日だけ-30ルーブルで、インターネットの代金を支払うことができます。 マイナスは250 MBの1日のクォータであり、速度は64 Kbpsに低下します。



誰もがより有利な同様の関税/オプションを知っている場合は、共有してください。



そこで、万が一の場合に備えて、緊急のインターネットチャネルを編成しました。
