Windowsで不明なデバイスのドライバーを検索する

Windowsで一部のデバイスのドライバーが見つからないという状況によく遭遇したと思います。 デバイスマネージャーでは、これらの腺は誇らしげに「不明なデバイス」と呼ばれます。 このような状況を解決する私の方法と、成功したアプリケーションの2つのストーリーをお見せしたいと思います。



Windowsプラットフォームの知識が乏しいため、そのような問題の解決方法ははるかに簡単です。コンピューターのハードウェアに関するすべての情報を提供するプログラムを購入するだけです。



だから、最初の話。 昔々、IrDA-USBアダプターを購入してNokia製の電話と通信し、Windows XPで動作しました。 時間が経つと、Windows Vistaを搭載したラップトップが登場し、安全のために電話からデータをコピーする必要がありました。 そして、アダプターはディスクに付属のドライバーで動作しないことが判明しました。 製造元のWebサイト（Tekram）では、Vista用の新しいドライバーは見つかりませんでしたが、同時に、使用したLinuxでアダプターが正常に機能しました。



自慢のメーカーのロゴが付いた多くのデバイスのプラスチックシェルは、実際にそのようなデバイスの数百で使用されているチップを隠すことを誰もが知っています。 Linuxはロゴを見ず、Linuxはチップで動作し、dmesgとlspciの出力、lsusbはドライバーを探す場所を明確に示します。



そこで、アダプターをLinuxマシンに挿入し、lsusbの出力を確認しました。



Bus 007 Device 002: ID 9710:7780 MosChip Semiconductor MS7780 4Mbps Fast IRDA Adapter







製造元のWebサイトで、数分でWindows Vistaに必要なドライバーが見つかりました。



次の例は、HP ML110G5サーバーです。 HPは、このシリーズのWindows Server 2003のインストールのみをサポートしています。 同時に、サーバーは非常に曲がったシステムの下に置かれましたが、そうでない場合はWindows XPよりも起動しませんでした。 ご存知のように、HPのWebサイトにはWindows XP用のドライバーはありませんでしたが、2回目のテクニカルサポートでは、このような機能がサポートされていないことを明示しました。



そして再び、Linux、つまりSlaxが私たちを助け、lspciはチップセットの名前を推測しました。これは、Intelサイトから正常にダウンロードされたドライバーです。



一般に、LinuxはWindowsネットワークでの作業に成功し、これら2つのケースは、アプリケーションの小さな可能性を1つだけ示しています。 たとえば、ライセンスのないソフトウェアを使用することはできないと考えているため、partimageまたはddを使用してネットワークに新しいコンピューターを展開する方が便利ですが、それがなくてもNorton Ghostを購入することは雇用主の無駄遣いです。
