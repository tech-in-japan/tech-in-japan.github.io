Flash vs JavaScript、V8ベンチマークスイート

iq 12 （これは注目に値します。これはFlashを使用して専門的に開発に取り組んでいるスタジオです）Flash V8ベンチマークスイート -さまざまな環境でのJavaScript実行速度を決定するために設計されたテストセット これの主な目的は、FlashとJavaScriptの実行速度を比較することでした（コンピューティングですが、レンダリングはしません）。 そして最後に、彼らは非常に興味深い結果を得ました-過去2年間で、人気のあるすべてのJavaScriptエンジンは、計算速度の点でFlashを上回りました。









比較の著者は次の点に注意します。

  • AS3は、登場時と比較してパフォーマンスの大幅な改善を示しませんでした。 著者は最適化することがたくさんあると確信しており、 Alchemyはそれを証明しています。
  • Flash PlayerのレギュラーはIE6と同じくらいひどいです
  • FP9とFP10.3を比較すると、AS3全体のパフォーマンスが2倍になりました
  • JavaScriptには厳密なタイピングはありませんが、すべてのブラウザーは実行速度の点で既にFlashを大きく上回っています。

    • Chrome 1.0以降（01/2008）
    • Safari 3.1以降（2008年3月）
    • Opera 10.5+（2010年3月）
    • Firefox 4.0以降（2011年3月）
    • IE 9.0+（2011年3月）
  • 一部のブラウザは非常に優れているため、Flashをはるかに遅れています。
    • Chrome 10.0+（2011年3月）。
    • Opera 11.50（Swordfish）ベータ
    • Firefox 5.0ベータ


ソースでは、さまざまなテスト、Google V8 BenchmarkのActionScript3への移植の詳細、Flashコンパイラの使用、その他の興味深い情報を見ることができます。



続きを読む⇒



私見、JavaScriptの最適化は飛躍的です。 わずか3年前のJSで今非常に簡単にできることは、達成不可能な夢のようでした。 そのような結果は激しい競争の結果であり、ベンダーはそこで止まることはないと確信しています。 すぐに非常に強力なアプリケーションとゲームを見ることが可能になるでしょう。 また、携帯電話向けのJavaScript最適化が間近に迫っているので、デスクトップ上だけでなく。
