10年前のように

スリリングなブラウザー戦争を逃しました。 数か月ごとに、MicrosoftとNetscapeの両方が、新しい興味深い機能を備えたブラウザの新しいバージョンをリリースしました。 残念ながら、Netscapeの死によりイノベーションはほぼ停止し、Webブラウザはマーケティング担当者が「安定したプラットフォーム」と呼んでいます。



この記事が公開される頃には、Microsoft Windows XPオペレーティングシステムが販売され、何百万人ものユーザーがInternet Explorer 6を試すことができます。InternetExplorer 6は、2年半ぶりのIEの主要なアップグレードです。 このアップグレードで最も興味深いのは、追加された新しい関数ではなく、消失した関数であることです。



Java

Netscapeはもはや本当の脅威ではなかったため、Microsoftには革新的な開発の機会が与えられ、不要なクロスプラットフォームサポートをすべて削除することができました。 JavaはIE6のほうきで一掃されました。 それ自体は驚くことではなく、驚くべきことでもありません。 MicrosoftのJavaサポートは常に緩慢でした。 昔、Sunや他の企業はIE用のプラグインを提供していました。これにより、JavaはInternet Explorerで正しく動作しました。 マイクロソフトの承認がなくても、ユーザーは引き続き使用できます。 残念ながら、Microsoftはプラグインのサポートも停止しています。



すべての既存のプラグインは、Microsoft ActiveXで書き換える必要があります。 このトピックで議論した開発者は、ActiveXコントロールでそれらを書き換えて、古いプラグインと同じ機能を提供することができると説明していますが、この状況が今後も続くという疑念を表明しました。



ブラウザのJavaは死んでいますが、Java自体はかなり強力です。 ブラウザ側のJavaは非常に注目されていますが、サーバー側のJavaと同様にサポートされることはありません。 現在書かれているほとんどのJavaプログラムはブラウザの外部で実行され、Windows XPでも引き続き正常に動作します。



スマートタグ

スマートタグは、初期のIE6ベータ版およびOffice XPの最終リリースに登場した革新的な機能です。 Microsoftが「サイトへのリンクの欠如」として定式化した問題を解決します。 IE6は、任意のWebページのテキストに表示されるキーワードの独自のハイパーリンクを埋め込むことができます。



たとえば、「Microsoft」という単語は、Microsoftの公式Webサイトへのハイパーリンクを生成します。WindowsXPに関する言及には、オンライン注文フォームへのリンクが付随し、株価表示はMicrosoft財務Webサイトからの引用に自動的にリンクします。 スマートタグを使用すると、Microsoftは検索エンジンと同じ方法で広告主にキーワードを販売することもできます。 マイクロソフトの潜在的な収益は本当に膨大です。 Web全体のすべてのフレーズ「DVD Player」がオンラインストアへのハイパーリンクに変わるために、Sonyがいくら払うと思いますか？



スマートタグはすぐに、Microsoftではなく自分たちが作成したサイトのコンテンツと広告を制御する必要があるという奇妙な考えを抱いているWebマスターやコンテンツ作成者の間で抗議を引き起こしました。 Microsoftは7月に、IE6の最終リリースからスマートタグが削除されることを発表しました。



ただし、スマートタグは消えず、忘れられません。 IE6では無効にされていましたが、コードにはまだ存在し、Office XPの一部です。 Microsoftの代表者は、この関数を「より快適な」形式で返すことを明らかにしました。



安全なクッキー

セキュリティオプションに過ぎない安全なCookieについて聞いた途端、すぐに良いアイデアのように思えました。 私はすでにDoubleClickのような会社にうんざりしています。DoubleClickは他社のサイトからのCookieを設定することで、ウェブ上の私の動きを追跡しています。 デフォルトでは、IE6の以前のバージョンでは、Cookieはサイト自体からのみ許可され、DoubleClickなどのサードパーティからのCookieはブロックされます。



安全なCookieの機能を正しく実装すれば、これらの追跡会社は数か月のうちにビジネスから追い出される可能性があります。後悔することはありません。 ただし、ユーザーにプライバシーを導入したという広範な歴史を持つMicrosoftが、サードパーティのCookieに関する比較的小さな問題に没頭しているのは奇妙に思えます。



ブラウザーをテストした結果、すべてが明らかになりました。 マイクロソフトもその子会社も「第三の会社」ではないことが判明しました。 つまり、レドモンドの標準では、DoubleClick Cookieのインストールは邪魔であり、ブロックする必要があり、同じMicrosoft Banner Network Cookieのインストールは許容され、許可されます。



安全なCookieは、独占禁止法当局からの脅威により、ブラウザの最終リリースからも除外されましたが、Microsoftがこれを可能にするとすぐに確実に復帰します。



ユーザーパワーの神話

新しいソフトウェアに関するほとんどの記事は、ユーザーにとっての価値の観点から新しいソフトウェアの機能を説明しようとしていることがわかりました。 このアプローチは表面的であり、この場合には適切ではないと考えます。 この段階では、マイクロソフト製品の受動的な消費者としてのみ考慮されます。 市場法の倒錯したロジックでは、Microsoftの排他的なブラウザーの新しい機能は、非独占的な製品の販売を増やすように設計されており、ブラウザーから削除された機能はユーザーの影響で削除されます。



ブラウザ戦争は終わり、マイクロソフトは戦争に勝ちました。 今、占領が始まります。
