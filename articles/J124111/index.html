ベータ版Flash Player 11（64ビットを含む）およびAir 3（Molehill 3Dを含む）がリリースされました

主な機能：



-すべてのプラットフォーム（win、lnx、mac）の64ビットサポート 。 現時点では、フラッシュプレーヤー専用です。



- モレヒル3D すべてが明確で、たくさんのトピックがありました。 プレビューと比較して、APIにわずかな変更があります。



-H.264 / AVC、ネイティブjson-プレビュー2でした。



-AIRのキャプティブランタイム-ウィンドウ、デスクトップなどのコンテンツをキャプチャするためのadobe.connectと同様 スクリーンキャストに役立ちます。



FP 11ベータ版をダウンロード

Air 3ベータ版をダウンロード







FP 11およびAir 3の場合：



-Molehill 3D API 。 この技術については、すでにHabrで多くのことが言われています。



-立方ベジェ曲線。 ネイティブ曲線。



-大きなビットマップのサポート。 以前は、ビットメイトのサイズは制限されていました（2880x2880）が、サイズは無制限になりました（プラグインに使用可能なメモリがあるため）。



-JPEG-XR、ATF 2番目の形式は、テクスチャのネイティブadobe形式です。



-LZMA圧縮 。 deflateとzlibに加えて。 さらに、フラッシュドライブ自体にlzmaをパックできるようになり、サイズが縮小されます。



-G.711コーデック 。 電話などでは、PCMAとPCMUがサポートされています。



- カメラ用のH.264 / AVC SW 。 以前は、カメラからのストリームはSorenson VP6でのみエンコードされていました。



-ネイティブJSON。 ネイティブXMLに加えて。



- イベントの一時停止/スリープ/再開 。 クリップのあるタブがアクティブであり、デバイス（デスクトップ|モバイル）が現在Flashクリップで動作していることを確認するため。



-GCのヒント。 System.pauseForGCIfCollectionImminentを使用して、負荷の高い操作中にメモリを駆除しないようにGCに指示できます。 後で、GCヒントAPIが拡張されます。



-安全なランダムバイトジェネレータ。



-モバイルデバイスのFlash Accessコンテンツ保護サポート、保護されたHTTP Dynamic Streaming（HDS）、およびFlash Access拡張機能はすべてAdobe DRMに関連しています。



-ネイティブテキスト入力UI。 モバイルデバイス用。



-System.processCPUUsage 。 最後に、fps-parrotsではなく、プロセッサが実際にフラッシュドライブを消費する量を確認できます。



FP 11のみ：



-64ビットのサポート-約束どおり。



-非同期ビットマップのデコード。 プレーヤーの別のストリームで画像をデコードするこの機能は、すでにAir 2.6にあります。



-TLS（Transport Layer Security）ソケット、およびソケット進行イベント。



Air 3のみ：



-ステージビデオハードウェアアクセラレーション-FP 10.2と同様。



- キャプティブランタイム



-HDサラウンドサウンド-AirTVの7.1サウンド用。



まだ期待：



-WebWorkers-このベータ版に参加しなかったのは非常に奇妙です。明らかに最後の瞬間にいくつかのバグが出ました。

-HIDのサポート（ステアリングホイール、ジョイスティック）。

-新しいデータ型-float、float4。

-WebM-噂はありますが、何も約束しません=）



UPD： Linux for Molehillでは、GPUを使用せずにソフトウェアレンダリング（SwiftShader）のみを使用することを忘れていました（一部のビデオカードでGPUがまだ使用されているビデオ再生とは異なります）。 彼らはドライバーの非常に大きな問題を説明しています。 問題は真剣に研究されており、おそらくバージョン12で変更されるでしょう。
