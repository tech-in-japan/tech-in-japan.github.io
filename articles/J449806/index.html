キャリアプログラマー。 パート2.学校または独学

ストーリー「キャリアプログラマー」の続き。



庭には2001年でした。 最もクールなOSがリリースされた年はWindows XPでした。 rsdn.ruが登場したとき。 C＃および.NET Frameworkの誕生年。 ミレニアムの最初の年。 そして、新しい鉄の容量の1年の指数関数的な成長：Pentium IV、256 mb ram。



9年生を卒業した後、私はプログラミングのさらなる研究に尽きることのない熱意を持っていました。 これを見て、両親は私を専門の「プログラミング」のために大学に編入することにしました。 彼らはそれがより良いと信じて、そこで教えてくれました。 ちなみに、ワードカレッジは、工業都市の郊外にあるこの施設に適合していませんでした。 それは、他の専門学校と区別されない通常の専門学校であり、その正面にファッショナブルな単語「大学」のラベルを掲げていませんでした。

じゃあ 私は両親と議論したり、両親の決定に異議を唱えたりしませんでした。 いずれにせよ、私は独学に従事し、この新しい場所で私はいくつかの追加知識を与えられると考えました。



その夏、大学に入る前に、私はハッカー誌に掲載されたすべての可能な技術を徹底的に研究し始めました。 私はそれを読み、穴まで読み直しました。 特に本物のハッカーとのインタビューと彼らのアドバイスが気に入りました。



クールなハッカーのほとんどはLinuhaに座っていました。 また、「マツダイ」（Windows）はラマー用でした。 誰が雑誌を読んだか、彼はその中の投稿のスタイルを覚えています。 したがって、私の脆弱な心の中で、Windowsを離れる、またはクールになり、純粋にLinuxに座るという2つのアイデアが互いに苦労しました。

Hackerマガジンの最新号ごとに、ディスクをフォーマットし、Linux Red Hat 7またはWindows Meをインストールするという新しい理由がありました。 もちろん、トレーニングのベクターはありませんでした。雑誌や「ハッカーシークレット」のような海賊版CDで読んだことをやったのです。 「Windows XP別名オウムは主婦向け」という精神で新たに投入された後、2つのOSの並行インストールも消去されました。 また、真剣なことをしたい場合は、目を閉じてLinuxコンソールから作業する必要があります。」 もちろん、私はシステムをハッキングし、ネットワークがどのように機能するかを理解し、当時は全能の匿名になりたかったのです。



ディスクは後悔することなくフォーマットされ、Unixに似たシステムのディストリビューションがインストールされました。 はい、はい。 ある日、コンソールからFreeBSD 4.3のみを使用する本物のハッカーとのインタビューを読みました。 同時に、銀行と国家システムのハッキングが彼の後ろにリストされました。 それは頭の中の稲妻であり、BSD OSをメインシステムとして5回使用しました。 問題は、インストール後、ベアコンソールを除いて、そこに何もないことでした。 音さえ。 そして、KDE2を入れてサウンドをオンにするには、タンバリンで踊り、いくつかの設定を修正する必要がありました。

文学について



コンピューターを手に入れるとすぐに、プログラミングの本を買い始めました。 最初はTurbo Pascal 7.0のガイドでした。 パスカルはプログラミングコースで少し知っていたので、これは驚くことではありません。自分でトレーニングを続けることができました。 問題は、ハッカーがパスカルで書かないことです。 それから、Perl言語は流行していました。または、よりクールな人にとっては、C / C ++でした。 少なくともそれは彼らが雑誌に書いたことです。 そして最後まで読んだ最初の本は、C、Kernigan and Ritchie Programming Languageでした。 ちなみに、私はLinuxで勉強しました

また、gccと組み込みのKDEエディターを使用してコードを記述しました。



この本に続いて、UNIX百科事典が購入されました。 彼女の体重は3キロで、A3ページに印刷されました。



本の表側には、熊手付きの漫画の悪魔が完全な高さまで描かれ、その後、ウクライナで125グリブニアがかかりました（これは2001年に約25ドルです）。 本を買うために、私は学校の友人からローンを要求しました。残りは両親によって追加されました。 その後、私は熱心にUnixコマンド、vimおよびemacsエディター、ファイルシステムの構造、およびファイル構成の内部を研究し始めました。 約700ページの百科事典が飲み込まれ、私はクールな帽子屋になるという夢に一歩近づきました。





UNIX百科事典-最初に読んだ本の1つ



愛する祖父と祖母と両親が私にくれたお金はすべて本に費やしました。 次の本は21日間でC ++でした。 タイトルは非常に魅力的だったので、私はより高品質の他の本を見ませんでした。 それにも関わらず、この3週間ほどの期間、すべてのソースが本からコピーされており、すでにC ++で何かを理解していました。 私はおそらくこれらのリストに何が書かれているのか理解していないでしょう。 しかし、進歩がありました。



どの本があなたのキャリアに最も影響を与えたかを彼らが私に尋ねたら、私はためらうことなく答えました-「プログラミングの芸術」-D.クヌート。 それは脳の閃光でした。 この本がどれほど正確に私の手に渡ったかを正確に伝えることはできませんが、それが私の将来のキャリアに最も大きな影響を与えました。





プログラミングの芸術-必読



私は主にラジオ市場で本を買いましたが、それは日曜日にしか働きませんでした。 朝食のために次の数十個のグリブナを保存したので、C ++または多分Perlに関する新しい本を探しました。 選択肢は十分に大きかったが、指導者がいなかったので、すべてを続けて勉強した。 私は売り手にプログラミングのために何かを勧めてくれるように頼みました。 そして私が覚えている限り、彼は棚から「プログラミングの芸術。 最初のトム。」 この本は明らかにすでに使用されていました。 カバーの角が曲がっており、ビル・ゲイツが彼のコメントを残したところに、大きな傷がありました。「この本を読んだら、必ず履歴書を送ってください」とサインされました。 私は雑誌からゲイツについて知っていました、そして、すべてのハッカーが彼を持っていたとしても、私は彼に履歴書を送ることは素晴らしいと思いました。 この本は72 UAHの価値がありました。 （$ 15）、そして私はすぐに路面電車で急いで家に帰り、新しい教材を調べました。



私が読んだどのくらい深くて基本的なこと、もちろん私は15歳で理解できませんでした。 しかし、彼は熱心に各演習を完了しようとしました。 25または30の複雑度で問題を正しく解決することができました。これは数学的帰納法の章でした。 私は学校の数学が好きではなく、ひどくそれをつかみましたが、マットの上で。 クヌース分析-私は何時間も座った。



次に、第2章ではデータ構造について説明しました。 リンクリスト、バイナリツリー、スタック、キューのこれらの画像と写真は、まだ私の目の前にあります。 商業開発における私のキャリアの12年にわたって、私はほとんどの一般的な言語を使用しました。



これは、C / C ++、C＃、Java、Python、JavaScript、Delphiです。 そして、その言語がどのように呼ばれようとも、その標準ライブラリには、Donald Knuthの3巻の本で説明されているデータ構造とアルゴリズムがありました。 したがって、新しいことを学ぶのにそれほど時間はかかりません。



最初のボリュームは十分に素早く飲み込まれました。 Knuthの本に記載されているアルゴリズムをC言語にコピーしました。常に機能するとは限りませんでしたが、練習すればするほど明確になりました。 熱意は保持されませんでした。 最初の巻を終えたので、私は2番目と3番目を実行して購入することをためらいませんでした。 私は2番目を脇に置きましたが、3番目（並べ替えと検索）については徹底的に取りました。



ソートと検索のアルゴリズムを「解釈」して、ノートブック全体を書き留めた方法をよく覚えています。 データ構造と同様に、バイナリ検索とクイックソートは超高速で脳内で視覚化され、Knuthの第3巻でそれらがどのように模式的に見えるかを覚えています。



鞭はどこでも読まれました。 そして、近くにPCを持っていなくても海に行って、私はまだノートブックにアルゴリズムを規定し、それらを通して一連の数字を実行しました。 ピラミッド型の並べ替えをマスターするのにどれだけの苦痛がかかったかは今でも覚えていますが、それだけの価値はありました。



私に大きな影響を与えた次の本は、ドラゴンブックでした。 彼女は「コンパイラ：原則、技術、ツール」です-A.アホ、R。セティ。 C ++の高度なタスクを備えたHerbert Shilddが先行していました。 ポイントが集まったのはこの場所でした。

Schildtのおかげで、言語のパーサーとインタープリターを書くことを学びました。 そして、ドラゴンブックは、私に自分のC ++コンパイラを書くように促しました。

学校または独学



しかし、IDEを超えて現実に戻ります。 その時までに、私は実生活からますます離れて仮想に没頭していましたが、すべて同じ年齢であり、一般に受け入れられている規範により、私は大学に行くことを余儀なくされました。 それは本当の拷問でした。 私はこの機関で何をしているのか、なぜこの情報を聞いているのかを完全には理解していませんでした。 私の頭には完全に異なる優先事項がありました。 Visual Studio 6.0を学習し、WinApiとDelphi 6を試してみます。



素晴らしいサイトfirststeps.ru。これにより、誰もが小さな一歩を踏み出すことに喜びを感じることができましたが、全体像は理解できませんでした。 たとえば、同じテクノロジーのMFCまたはActiveX。



大学はどうですか？ 時間の無駄だった。 一般的に、研究のトピックに触れると、私は不十分に勉強しました。 私は6年生までは優秀な生徒でしたが、その後3年生が飛ぶようになり、8年生から9年生まではしばしばクラスを飛ばしました。



したがって、大学に来てからも、熱意はほとんどありませんでした。

すべてに加えて、私は悪質な性格を持っていて、とても謙虚でした。 この新しい雑多なグループは明らかに信頼できませんでした。 そしてそれは相互的なものでした。 そのため、あらゆる種類のm笑が待つのに時間がかかりませんでした。 私はそれを我慢できず、教室で正真正銘の犯罪者の一人を襲うまで、長い間耐えました。 はい、だから彼は机のために飛び立ちました。 父のおかげで、子供の頃から戦うことを教えてくれたので、強い欲求があれば、物理的な力を使うことができました。 しかし、これはめったに起こりませんでした。私はしばしばm笑に耐え、最高沸点を待ちました。

ちなみに、犯人は、起こっていたことに大いに驚きましたが、それでもなお優れていると感じて、私を復帰の戦いに呼びました。 すでに学校の後ろの荒れ地に。



学校の場合のように、これらは子供の拳ではありませんでした。 鼻の骨折と血の多い高貴なマハハがいました。 この男はまた、ti病なダースからではなく、巧みに計量されたフックとアッパーカットでした。 誰もが生き続け、それ以来、この「プログラマーのための大学」で誰も私をいじめたことがありません。 私はそこに行きたいと思っており、すぐに完全に姿を消しました。 したがって、私はそこに入るのをやめ、両親からの脅威は私に影響を与えませんでした。 奇跡によって、彼らは学校の10年生のために大学に留学したと考え、私には11年生に行く権利がありました。



すべては大丈夫ですが、11年生は最高の大学ではありませんでした。 私は母国の学校に戻り、1年生から勉強していた知り合いの友達に会い、母国ですべてがうまくいくことを望みました。 ただ一つ注意点がありました。男は、私がジュニアクラスで友達だった男の子よりもシリーズの盗賊に似ていました。 誰もが筋肉を得るためにロッキングチェアに行きました。 竹のように見えました。 ランキーで非常に薄い。 もちろん、そのような雄牛の同級生は、私を左に1つねじることができます。



それで、それは時間とともに起こり始めました。 それから私の戦闘スキルは効果がありませんでした。 すでに非常に異なる体重カテゴリーが私とかつてのネイティブクラスの残りの少年たちにありました。 また、私の思考の特徴は自分自身を感じさせました。



木に思いを広めることなく、私も学校を去りました。 私が快適だと感じたのは、部屋のドアが閉まっているコンピューターモニターの前でした。 それは理にかなっており、直感的に私は正しいことをしていると感じました。 そして、学校に行くのは無用の運動であり、そこではm笑に耐えることさえあります。それは毎日ますます洗練されてきました...それで十分です。



クラスでの別の対立の後、タイトルの役割で私と一緒に、私は学校を去り、もうそこに行きませんでした。
