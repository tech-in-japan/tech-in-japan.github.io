人。 Anders Halesberg-Turbo Pascal、Delphi、およびC＃の作成者



「私は膨大な数の有用なプログラムを書きました。 かなり役に立たない。」



何か新しいものを作成する必要がある人もいます。 このニーズが完全に実現すると、芸術作品、科学的発見、画期的なテクノロジーが登場します。 そのような人は、ターボパスカル、デルファイ、C＃の作成者であるアンダースヘイレスバーグです。



多くの大学生にとって、Turbo Pascalはプログラミングを導入する最初の言語です。 かつて、彼はソフトウェア開発における一種の突破口でした。 理由もなく、「人々に」パスカル、つまりターボパスカルに行きませんでした。



しかし、Halesbergはそこで止まりませんでした。 彼の手に負えない新しい作成の必要性のおかげで、DelphiとC＃が登場しました。 したがって、多くの人にとって、AndersはIT業界の「スーパーヒーロー」の1人です。



最初のステップ



Anders Hejlsbergは1960年12月にデンマーク最大の都市、コペンハーゲンで生まれました。コペンハーゲンは首都でもあります。



ハレスバーグは、リンビー工科大学で工学の学位を取得したかったが、それを完了できませんでした。





デンマーク工科大学は、コペンハーゲン北部のリンビー・トルベックにある工科大学および研究所です。
若い男は子供の頃からプログラミングに情熱を傾けていました。 大学で、AndersはNascomマシン用のソフトウェアを作成しました。



「1979年に、デンマークにコンピューター会社PolyDataを設立しました。 これは、パソコンが存在しなかった時代です。 コンピューターシステムを販売し、ソフトウェアを作成しました。 アセンブラー、逆アセンブラー、小さなオペレーティングシステム、Microsoft ROM-Basic5のいくつかの拡張機能を作成しました。



「私の最初の大きなプロジェクトは、PascalコンパイラとROM-Basicを置き換えることができるエディタでした」 と Halesberg 氏は回想します。



ターボパスカル



1980年、Halesbergが20歳のとき、彼の情熱はPascal言語の手書きコンパイラに変わりました。 元々はNascom-2のBlue Label Software Pascalでした。 その後、コンパイラをCP / Mおよび当時人気のあったオペレーティングシステムであるMS DOSに移植しました。 その結果、コンパイラはCompas Pascalと呼ばれ、後にPolyPascalと名付けられました。







1983年、ボーランドを設立したばかりの人たちとチームを組みました。 Pascalコンパイラのライセンスを取得し、そこに独自のエディタを追加しました。
Borlandは、現在Turbo Pascalとして知られているIDEをリリースしました。

私は彼らがおかしいと思ったのを覚えています：これらの人たちは49ドル、95ドルで新製品を売っていましたが、500ドルかかりました！ しかし、すぐに私が間違っていたことが判明しました。TurboPascalは非常に人気がありました。 当初は想像もできなかった数のコピーを販売しました。
ボーランドは1995年までこのバージョンのIDEをサポートしていました。



1987年、財政問題を解決するためにアンダースはカリフォルニア（米国）に移り、そこでボーランドのチーフエンジニアになりました。



デルファイ



1996年、Halesbergと彼のチームは、Delphiと呼ばれる開発環境をリリースしました。 同じチームによって開発されたObject Pascalは、基本プログラミング言語として使用されました。



ボーランドがObject Pascalで作業を開始したとき、その先祖（かつてWirthによって作成されたPascal）はかなり貧弱な状態でした。 多くの機能が改訂および改訂され、多くが追加され、ゼロから作成されました。 そして最終的に、変更は非常に巨大であることが判明したため、開発者は新しいプログラミング言語を作成しました。





新しいIDE Delphiは、古代ギリシャの都市Delphiに敬意を表してその名前を付けました。 名前が選ばれた理由は次のとおりです。Delphiの都市は、知恵の神の名前とアポロの芸術のパトロンに関連付けられています。 伝説によると、アポロの主要な聖域はこの都市にありました。 彼のシビル女司祭は、彼らの運命を知りたいと願う人々に預言を発した。
「以前のBorland製品を思い出すと、Turbo Pascalは非常に成功したことに注意してください。統合開発環境を備えた製品は、プログラマの生産性を大幅に向上させました。 ただし、Turbo Pascal for Windowsの最初のバージョンでは、Windowsアプリケーションの開発者が直面した最も重大な問題はまだ解決されておらず、これらの問題を解決するためにすべきことはすぐに明らかになりました。 Windowsアプリケーションの作成は、ユーザーインターフェイスの設計から始まることになっており、これが新製品の重要なアイデアでした。



Delphiをリリースしたとき、それはコンパイルされたプログラミング言語に基づいた最初の迅速なアプリケーション開発ツールでしたが、当時存在していたVisual Basicはpコードのみを作成でき、本質的にインタープリターに基づいていました。 Visual Basicの後続バージョンのマシンコードでコンパイラの出現を刺激したのは、Delphiの作成だったと言えます」 と Halesberg 氏は言います。



新しい開発環境は喜んで開始され、Windowsオペレーティングシステムを実行しているマシンで十分に迅速に機能しました。 Microsoftはこの事実と才能あるエンジニアHalesbergを無視できませんでした。



毎年、Delphiの新しいバージョンが登場しましたが、5回目以降は頻度がなくなりました。 Delphiの新しいバージョンの開発と並行して、Linux用の新しいDelphiのようなプログラミング言語が開発されていることが判明しました。 新製品はKylixと呼ばれます。



Delphiは、.NETプラットフォーム用のC＃言語の概念の作成に大きな影響を与えました。 その要素と概念的なソリューションの多くはC＃に含まれています。 Anders HalesbergはまだMicrosoftに魅了されていたため、これは予想されることでした。



MicrosoftとC＃







1996年、AndersはMicrosoftで働き始めました。以前の場所で受け取った20万ドルの代わりに、Bill Gatesは250万ドルを提供しました。 そこで、彼は最初にJ ++やFoundation Classesなどのプロジェクトで働きました。 しかし、その後、彼はC＃言語開発チームを率いました。 彼のリーダーシップの下で、Scott WiltamutやPeter Goldなどのプログラマーが働きました。

私は言語のチーフアーキテクトでした。 チームは4人で構成されていました。 私たちは週に3回、2〜3時間会いました。
マイクロソフトは、新しいソフトウェア製品のリリースで新しい千年紀を祝うことにしました。 2000年までに、同社はメッセージングとデータの分野で新しいコンポーネントテクノロジーとソリューションの工業用バージョンを準備し、インターネットアプリケーション（COM +、ASP +、ADO +、SOAP、Biztalk Framework）を作成しました。 これらの革新をサポートするために、Microsoftは.NETプラットフォームというアプリケーション開発ツールキットをリリースしました。 また、彼女は「新しい屋根の下」でいくつかのプログラミング言語を組み合わせました。



.NETプラットフォームのもう1つの革新は、アクティブサーバーページASP.NET（アクティブサーバーページ）の技術でした。 その助けにより、データベースと対話するWebアプリケーションを比較的迅速に開発することができました。



C＃プログラミング言語は、ASP.NET専用に作成されました。 ASP.NET自体は完全にC＃で記述されています。







「私の意見では、C＃と他のプログラミング言語、特にJavaの主な違いは、可能な限りC ++に近づけようとしたことです」とAnders Halesberg氏は説明しました。

C＃言語を作成したとき、いくつかの目標がありました。 まず、C / C ++ファミリーから最初のコンポーネント指向言語を作成します。 5年または10年前にアプリケーションがどのように開発されたかを思い出すと、多くの開発者が、オンデマンドでのアプリケーションの起動、特定のタスクの実行、およびその停止を整理するために特別な環境を作成したことさえわかります。



Web現象とクライアントサーバーアーキテクチャの出現により、アプリケーションの性質が変化しました。 現在、特定のプロセスを実行するコンポーネントのセットが作成されることがよくあります。中間レベルのアプリケーションのビジネスオブジェクト、データベースサーバーのストアドプロシージャ、および現在アプリケーションと呼ばれているコンポーネントの全体です。



さらに、開発者がソフトウェアを設計および作成する方法を念頭に置いていました。 アプリケーション（HTMLページやビジネスオブジェクトを含む）を設計するための最新のアプローチでは、通常、コンポーネントまたはオブジェクトのプロパティ、イベント、メソッドの概念、およびプロパティインスペクターを使用して変更します。



したがって、コンポーネント指向プログラミングおよび業界全体では、コンポーネントの概念のサポートをプログラミング言語に統合することが非常に重要です。 これは、C＃を作成する主要な目標の1つでした。



また、C ++のより生産的なバージョンを作成することも目標でした。 開発者はその力とほぼ無限の可能性のためにこの言語を愛していることが知られていますが、C ++の問題は、その力が1％の時間使用され、99％の時間がどの言語設計を適用するかを理解するために費やされることです特定の問題を解決します。 開発者の生産性を高めるために、C ++の簡易バージョンを作成することにしました。


エピスタリーのジャンル



Anders Halesbergが書いた数少ない本の1つはC＃です。 この本は、Microsoft C＃開発チームの一員である他の専門家と共同で作成されました。 この本は、開発者の一種の「聖書」であり、MSDNと他のC＃に関する本の両方を簡単に置き換えることができる、とHalesbergの忠実な愛好家は信じている。



この本は4回再版されました。 第4版（2012年）以来、この本には、ジョンスキート、ジョセフアルバハリ、ビルワーグナー、クリスチャンナイゲル、エリックリッパーなどの有名なプログラミング「グル」によって書かれた広範な解説が含まれています。



受賞歴

2007年、彼と彼のチームは 、C＃言語を作成したことに対して賞（優秀な技術的成果）を受賞しました。

次は何ですか



2012年、Halesbergは新しいTypeScriptプロジェクトを発表しました。このプロジェクトは、Microsoftの支援の下で開発されています。





TypeScriptはJavaScriptと下位互換性があり、JavaScriptにコンパイルされます。 実際、コンパイル後、TypeScriptプログラムは最新のブラウザーで実行したり、Node.jsサーバープラットフォームと組み合わせて使用​​したりできます。

私はコンピューターをやっていたので、抽象化のレベルをますます高くしてきただけです。 すでにマシンコードからアセンブラ、次にC、次にC ++、そして.NETやJavaなどのコードの実行を制御する環境に移行しています。 これまでのところ、このリストを継続できないという兆候は見られません。 私の意見では、命令型プログラミングよりも抽象化のレベルを上げることができるかと思います。
