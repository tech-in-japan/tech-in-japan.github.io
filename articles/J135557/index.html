第2世代ファームウェアNDMS v2.00上のZyXEL Keeneticルーターの機能

しばらく前に、NDMS 2.0と呼ばれるファームウェアのベータ版がKeenetic Internet Centerのダウンロードセクションの ZyXEL Webサイトに登場しました。 それを運転するための招待は、通常の運動学の登録された興味のある所有者によっても受け取られました。 注釈によると、これは次のアップデートではなく、ソフトウェアプラットフォームの第2世代であり、前のプラットフォームとは根本的に異なり、上級ユーザー向けです。 主張の中で最も興味深いもの（第一世代のマイクロプログラムと比較して）：

  • 複数の外部IPアドレスをサポートするフル機能のアドレス変換メカニズム（フル機能NAT）。 ローカルネットワーク上の宛先ポートを選択する機能を備えたポート転送。 ネットワークインターフェイスの役割の再割り当て。
  • インターネットにアクセスし、仮想プライベートネットワーク（VPN）に接続するための複数のPPPトンネルのサポート。
  • 統合イーサネットスイッチのポートにいくつかの追加の物理WANインターフェイスを割り当てる機能。
  • 3G / 4G USBモデムおよびオプションで割り当てられたWANインターフェイスを介したインターネット接続の予約。
  • プロフェッショナルなコマンドライン設定インターフェイス（CLIのようなシスコ）。 すべての機能は、コマンドラインから設定し、テキストファイルとして保存できます。
  • 「オンデマンド」でのバッチファームウェアアセンブリ。 デバイスの機能は、ユーザーがデバイスのWebインターフェースで選択し、自動的に更新されます。
ファームウェアに接続されたコマンドラインを操作するためのコマンドラインドラフトのマニュアルは、本当に「刺激」を与えます-純粋なコマンドライン、個々の記事のトピック。 ここでは、新しいファームウェアがWebインターフェースのレベルで何を誰に準備しているのか（今のところマニュアルはありません）、また、「単純な主婦」によるインストールが推奨されない理由を把握しようとします。



そのため、新しいファームウェアはZyXEL Webサイトからダウンロードされ、Keneticに正常にインストールされました（微妙な違いがある場合がありますが、特別なトピックでメーカーのサポートを通じて解決できます。この場合は重要ではありません）。 お互いを知りましょう。 Web Configuratorは、ユーザー名/パスワードadmin / 1234で192.168.1.1でも開きます。 すぐに、新しいWebインターフェースは以前のバージョンとほとんど関係がないことがわかります。







彼は私に二重の印象を与えました：一方で、彼はよりきれいになりましたが、他方では、私はそれをずっと長く習得しました。 （先を見て、Webインターフェースのスキン構成を、コンポーネントのリストの好みに合わせて2つのオプションから選択できることを追加できます。）



NDMS 2.0は、第1世代のファームウェアと同様に、多くのタイプの接続をサポートします。各接続には、インターネットセクションに個別のタブがあります（IPoE、PPPoE、PPTP、L2TP、802.1X、3G、Yota、Wi-Fiクライアント）。 V1.00とは異なり、ここでは一度に複数の接続を任意の組み合わせで構成できます。 接続は、物理インターフェイス（イーサネットポート、USBモデム、Wi-Fiインターフェイス）、または作成済みの論理インターフェイス（たとえば、あるPPTPトンネルを別のPPTPトンネルに「ネスト」できます）に接続されます。 各インターネット接続には優先順位が割り当てられます。これにより、想像力とニーズに応じて、どちらをメイン接続、どちらを基本接続、セカンダリ接続、またはバックアップ接続にするかを決定できます。







デフォルト設定では、インターネット接続「ブロードバンド接続（ISP）」は1つだけであり、WANポートに関連付けられています。 ケーブルがWANポートに接続されたときにプロバイダーのネットワークからアドレスを自動的に受信するDHCPクライアントを作成しました（もちろん、プロバイダーがこのアドレスを発行する場合）。 ホームネットワークの場合、デフォルトでは、ルーターとWi-FiアクセスポイントのすべてのLANポートが設計されています。 これらは1つの論理インターフェイス（ホームVLAN）に結合され、その上でDHCPサーバーが起動され、192.168.1.1 / 24ネットワークからアドレスを配布します。







PPTP、およびL2TPまたはPPPoEを介してインターネットサービスプロバイダーに接続するには、2つの（！）インターフェイスを構成する必要があります。1つはプロバイダーのネットワーク（IPoEタブ）への接続用、もう1つはインターネット自体用です。 プロバイダーのローカルネットワークのアドレスが自動的に発行される場合、最初のインターフェイスは既に構成されていると見なすことができます。これは、上記の定義済みのブロードバンド接続（ISP）インターフェイスです。 プロバイダーから静的アドレスが提供された場合は、ブロードバンド接続（ISP）インターフェイスの設定を変更し、[DNS]タブで指定されたDNSサーバーを指定することを忘れないでください。 次に、PPTPタブ（L2TPまたはPPPoE）を開きます。







ここでは、大量のルーターに典型的な通常の設定に加えて、「接続」および「インターネットへのアクセスに使用」という項目が表示されました。 前者はPPTP接続（L2TPまたはPPPoE）を確立する接続をルーターに伝えるために必要であり、この場合はブロードバンド接続（ISP）であり、2番目はそれ自身を表しています：このPPTP接続（L2TPまたはPPPoEを使用する場合） ）インターネットにアクセスするには、チェックボックスをオンにします。次に、ルーターのルーティングテーブルで接続が確立されると、デフォルトルートがこの接続のゲートウェイになります。 設定を適用し、プロバイダーからケーブルのWANポートに接続すると、デバイスはインターネットに接続します。 802.1xを使用する必要がある場合（そのようなプロバイダーもあります）、さらに簡単に有効になります。別の接続としては表示されませんが、指定されたインターフェースで「稼働します」。



原則として、PPTP経由でインターネットにアクセスできる機能するルーターを既に受け取っています。 しかし、もちろん、この庭全体（ファームウェアとそれについての記事）は、そのようなありふれたもののためではありません。 次に、USBモデムを介したインターネット接続がどのように増加するかを見てみましょう。 Yotaモデムは、最初のファームウェアと同様に、USBポートに接続すると自動的に認識されます。動作に必要なものはすべて事前に構成されています。 人気のあるSamsung SWC-U200 iotovモデムをルーターに挿入します（アカウントにお金があるかどうかは重要ではありません）。 接続はすぐに通過し、モデムは青色のインジケーターで点滅し、ステータスページのYotaインターフェースが緑色に点灯し、接続パラメーターがYotaタブに表示されます。 ご覧のとおり、SLEEPモデムの状態は、デフォルトで動作を継続するPPTP接続よりも優先度が低いためです。







私の知る限り、これはNDMS v2.00の非常に重要な機能です。 通常のインターネットへのPPTP接続は正常に機能していますが、システムはすぐにYotaとの接続を確立しますが、データ転送には使用しません。 デフォルトでは、Yotaインターフェースの優先度はISPの優先度、特にPPTPよりも低いため、接続は予約されます。 典型的な優先順位の割り当ては次のとおりです。ISP-700、PPTP / L2TP / PPPoE-1000、Yota-400。これらの優先順位はどういう意味ですか？ そして、ここに何があります。 何らかの理由でPPTPセッションがクラッシュして回復しない場合、特別なことは何も起こりません。システムは継続的にそれを復元しようとしますが、プロバイダーのネットワークへの物理接続の優先度が高く、動作しているため、Yotaに切り替えません。 ただし、ケーブルをWANポートから引き抜くと、システムはすぐにiotaインターネットに切り替わります（切り替えても1つのpingが失われることはありません）。 PPTPセッションは、プロバイダーのネットワーク上でスケジュールされた作業だけで高い確率で脱落する可能性があるため、Yota経由の接続の優先度をたとえば900に設定するのが合理的です。その後、PPTPトンネルが落ちた場合、いわばモバイルインターネットに切り替えます PPTPを介した接続が有効になったことを彼女が発見するとすぐに、ヨタは保護区に移されます。 残念ながら、現在のファームウェアでは定期的なpingによるインターネット接続の正確なチェックはありませんが、将来のバージョンでは開発者がこの機能を追加することを期待しましょう。 結局のところ、接続を確保し、インターネットへの中断のないアクセスを確保することは、これがNDMS v2.00の主な機能であり、このファームウェアをKeeneticルーターの全ラインに拡張することを約束すると噂されています。



ちなみに、インターフェースを無効にするには（つまり、システムがインターフェースの動作を停止し、まったく応答しないようにするには）、インターフェースパラメータの[インターフェースを有効にする]チェックボックスを削除するだけで十分です。 有効にするには、同じことを逆の順序で行う必要があります。 パラメータは消去されず、「準備完了」のままです。



たとえば、ADSLを介して別のバックアップ接続を追加できない理由はありません。 最も一般的なバージョンでは、プロバイダーはDHCPを介してアドレスも発行します（静的アドレス指定の場合、難しくはありませんが、もう少し詳しく説明します）。 バックアップADSLリンクを作成するには、イーサネットポートを備えたADSLモデムが必要です。Kineticsで設定されたIPサブネットからモデムを制御するためのIPアドレスを削除することが望ましいです。 しかし、モデムはブリッジモードで動作するため、これは致命的ではありません。 モデムにADSL接続を設定してブリッジモードにした後、ルーターの空いているLANポート（現在の例ではLAN4ポート）に接続します。 次に、インターネットセンターの設定で、別のIPoEインターフェイスを作成し、LAN4にバインドします。

ルールは、インターネットインターフェイスとローカルインターフェイス間でのみ機能します。 「入力」インターフェイス、つまりインターネットからの呼び出しが来るインターフェイスを指定するルールを追加します（PPTPがあります）。 次に、外部宛先ポートの番号とそのタイプ（TCPまたはUDP）を指定します。 その後、内部サーバーのIPアドレスを書き込み、必要に応じて、要求をブロードキャストするポートを指定します。 同様に、ポート転送はバックアップ接続用に構成されます（もちろん、必要な場合）。 ルールを保存したら、必要なポートがルーターのファイアウォールを通過してローカルネットワークに到達できるようにする必要があります。 これは[ファイアウォール]タブで行います。







ここでは、デバイス全体または各インターフェイスに対して個別にルールを作成できます。 ルールがスピンするインターフェイスを決定する場合は、すぐに選択する必要があります。



一般に、判明したように、設定にそれほど複雑なものはありません。ファームウェアのロジックを理解するだけです。

1.物理インターフェイスと他の論理インターフェイスの両方にバインドできる論理インターフェイスがあります。

2.インターフェースは「インターネット用」にすることができます。使用するゲートウェイはシステム全体のデフォルトルートになります。 「インターネット用ではない」場合があります-このようなインターフェイスは、たとえば、リモートネットワークに接続するために使用できます。

3.各インターネットインターフェイスには優先順位があり、それに応じてインターネットチャネルのバックアップアルゴリズムがそれらを整理します。



また、NDMS v2.00ファームウェアには、これまで他のルーターやネットワークデバイスでは見られなかった興味深い機能があり、概して将来的に機能することも追加します。 これは、コンポーネント単位のファームウェアアセンブリの最初に宣言されたサポートです。 つまり、デバイスまたはZyXELのWebサイトでは、設計上、初期セットアップとインターネットアクセスに十分な基本バージョンのファームウェアが提供されています。 インターネットに接続したら、コンポーネントメニューに移動し、特定のモジュールをインストールまたは削除して、自分用のファームウェアを作成します。 たとえば、3Gモデムのサポートを削除できますが、Yotaサポートはそのままにして、FTPおよびCIFSを追加しながら、送信およびWiFiクライアントを削除できます。 特定のプロバイダーにアクセスするために必要なタイプの認証のみを残し、残りをすべて削除してデバイスメモリを解放することを後悔することなく論理的にすることができます。 現時点では、ファームウェアのすべてのコンポーネントを安全に残すことができ、まだ十分なスペースがあることを予約しますが、開発者は、DLNA、SIPクライアント、その他のグッズなどの「厚い」コンポーネントのサポートを約束します。 次に、ファームウェアを微調整することは理にかなっています。
