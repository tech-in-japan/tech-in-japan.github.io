DevOpsの展開方法：Visual Studio Team Servicesを使用した計画とプロジェクト管理





みなさんこんにちは！ Vorlon.JSチームによる一連の記事「How We Deployed DevOps」を公​​開しています。

Vorlon.JSはnode.jsベースのツールで、Web開発者が、特にモバイルシステムや組み込みシステムで、Webアプリケーションをリモートでテスト、制御、デバッグする便利な方法を提供します。 MSDNのブログで、チームはVorlon.JSの作業を整理する際のDevOpsプラクティスの段階的な実装と、日常のタスクを解決するためのツールの選択について詳しく説明しました。 Vorlon.JSはオープンソースプロジェクトです。
マイクロソフトは、DevOpsプラクティスの実装に使用できる多数のツールを提供しています。このシリーズでは、Visual Studio Team Services、Visual Studio Code、およびMicrosoft Azureとの連携について詳しく説明します。 もちろん、Microsoft Azureにアプリケーションをデプロイする場合を含め、マイクロソフト以外の多くのツールを使用する可能性についても触れます。



サイクルコンテンツ：

  • Visual Studio Team Servicesを使用したプロジェクトの計画と管理
  • GitHubおよびVisual Studio Team Servicesビルドシステムとの継続的な統合
  • Microsoft AzureおよびAzure Resource Managerを使用したASAフレームワーク
  • Visual Studio Team Servicesのリリース管理
  • Microsoft Azure App ServiceとWeb Appsを使用した実稼働環境でのテスト


また、Visual Studio Team Servicesを使用した計画とプロジェクト管理に関する最初の記事で、今すぐ慣れることができます。



すべてのプロジェクト参加者間の情報交換はコラボレーションにとって非常に重要であり、そのような交換を確立する最良の方法は同じツールを使用することです。



Visual Studio Team Services（VSTS）には、ユーザーストーリーの作成からエラーの報告、各プロジェクトタスクのタイミングの追跡まで、すべての段階でプロジェクトを計画するためのツールが含まれています。 これらのシステムは、開発の技術的タスクのみを目的としていますが、プロジェクトのすべての参加者が使用することができます（おそらく使用する必要があります）。



Vorlon.JSでのDevOpsプラクティスの実装に関する作業を整理するために、VSTSを選択しました。 しかし、プロジェクトはオープンソースであるため、GitHubの問題管理メカニズムを使用して、開発の問題を解決し、エラーを追跡しました。



Visual Studio Team Servicesの使用を開始する



VSTSアカウントをまだお持ちでない場合は、 このページで作成できます 。 接続したら、新しいチームプロジェクトを作成できます。



次の2つの決定を行う必要があります。

  • 使用するプロセステンプレート：スクラム、アジャイル、CMMI
  • 選択するバージョン管理システム：Team Foundation Version ControlまたはGIT






スクラムとGITプロセステンプレートを選択しました。 Vorlon.JSのソースコードはGitHubでホストされているため、バージョン管理メカニズムは重要ではありません。



プロジェクトを作成すると、システムはメインページの情報パネルにリダイレクトします。ここで、プロジェクトに関する有用な情報（最近のアセンブリの結果、開いている作業項目またはエラーの数など）を含むタイルを固定できます。







Visual Studio Team Servicesにはかんばんボードが含まれており、列間でアイテムをドラッグアンドドロップするだけで、ユーザーストーリー、タスク、エラーを簡単に管理したり、他のタスクを解決したりできます。 彼女はあなたがプロジェクトを計画し、チームとやり取りするのを手伝います。 すべての列は、要件とルールに合わせて柔軟にカスタマイズできます。







チームワーク



バックログアイテムとスプリントを作成する前に、チームをセットアップする必要がある場合があります。

これを行うには、チームプロジェクトのホーム画面で[チームメンバー]タイルをクリックするだけです。



新しいウィンドウが開き、メールアドレスを指定して新しいメンバーを追加できます。







スプリント、作業項目、およびチーム機能のカスタマイズ



すべてのチームメンバーを割り当てた後、スプリント、作業項目、およびVisual Studio Team Servicesの他のオブジェクトの操作方法をカスタマイズできます。



ダッシュボードのWORKセクションに移動します。 バックログビューで、歯車のアイコンをクリックします。 新しいウィンドウが開き、カードの内容、情報パネルに表示される列、およびチームの就業日などのその他のパラメーターを設定できます。







左側のスプリントをクリックすると、リンクが表示されます。 それに従って、このスプリントの日付を調整するか、名前を変更します。











その後、日付が各スプリントの右上隅に表示されます。







[キャパシティ]タブで、各チームメンバーがプロジェクトに専念できる時間数を設定し、非稼働日を指定できます。







この情報は、作業のスケジュールとプロジェクト計画のレポートに使用されます。



作業項目（ユーザーストーリー、タスクなど）の作成と、参加者とスプリントへの割り当て



これで、スプリントとチームをセットアップした後、ワークアイテム（ユーザーストーリーやタスクなど）を作成して、チームメンバーに割り当てることができます。



VSTSポータルでは、これをいくつかの方法で行うことができます。 最初の方法は、左側の[バックログアイテム]テキストをクリックして、クイック作成機能を使用することです。







追加されたアイテムは、スプリント間でドラッグできます。 追加後、行をダブルクリックして詳細情報を含むウィンドウを開き、ワークアイテムをチームメンバーに割り当てるか、追加情報を提供します。







ボードモードに切り替えて、+ NEWボタンを押すこともできます。







非標準リクエストの作成、チームプロジェクトの情報パネルでの修正



Visual Studio Team Servicesを使用すると、ワークアイテムのカスタムリクエストを作成し、メインページのダッシュボードに添付して変更を追跡できます。 たとえば、現在のスプリントで記録されたエラーの数を返すクエリを作成できます。



[WORK]タブに移動し、左側の[クエリ]項目を選択します。 ご覧のとおり、VSTSにはいくつかのクエリが既に作成されています。







新しいクエリを作成するには（たとえば、プロジェクト内のアクティブなエラーの数を返す）、[新規]ボタンをクリックします。







作成したクエリをグループ全体で使用できるようにするには、[クエリ]セクションから[共有クエリ]にドラッグします。 その後、ホーム情報パネルに戻り、+ボタンをクリックして、表示されるウィンドウで[クエリ結果]を選択できます。







新しいガジェットをクリックしてカスタマイズします。 タイルの名前を指定し、使用するクエリと表示されるフィールドを選択できます。







[保存]ボタンをクリックします。 すべて準備完了です！ グループプロジェクトのセットアップが完了しました！ これで、完了する必要があるすべての作業項目を宣言し、実行中のタスクに応じてダッシュボードを構成できます。



次の投稿では、Visual Studio Team ServicesとGitHubのソースコードリポジトリ間の継続的な統合を構成する方法について説明します。
