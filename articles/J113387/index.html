HPC、HA、およびSSIクラスターのみを必要とする人々への朗報、ついに

良い知らせがあります 今日、私はKerrighedの父親に彼らの作品をdebianizeするように説得したようです。



これは私たちにとって、普通の人々にとって何を意味するのでしょうか？ Ubuntuまたは他のDebianのようなLinuxを搭載したコンピューターをお持ちですか？ それをコンピューター1と呼びましょう。 次のような普通のことができます



apt-get install kerrighed-kernel...







まあ、おそらく構成に数分を費やす必要があります。 次に、Ubuntuを再起動すると、grubメニューに新しく作成されたカーネルが表示されます。 1つの異常なプロパティを持つ通常のUbuntuを選択して取得します。「 SBL with SDR 」と呼びましょう...



「SSI with DRBL」とはどのような獣ですか？




OK、同じネットワークセグメントに自宅またはオフィスにコンピューターがありますか？ -それをコンピューター2と呼びましょう。 Windowsが少なくとも何もインストールされていなくても、何がインストールされていても問題ありません。ドライブはとにかく触れません。 要するに、あなたは彼のBIOSに入ります。 そこで「ネットワーク/イーサネット/ PXEから起動」を見つけて公開します-近年のほとんどのコンピューターにはこのオプションがあります。 再起動します。



コンピューター＃2は、コンピューター＃1を「ドナー」として使用して、ネットワーク経由でUbuntuの起動を開始します。 ダウンロードの最後に、コンピューター1と2が単一のLinuxシステムになります。 これはクラスターです。 このようなクラスターシステムはSSI（シングルシステムイメージ）と呼ばれ、DRBLはそのようなクラスターのノードがハードディスクからではなくネットワークを介してロードされたことを意味します。



たとえば、オフィスに25のクアッドコアコンピューターがある場合、ロード後に100コアのLinuxが表示されます：o）



私はこの喜びを約5年前にRedhat + OpenSSIに基づいて使用しました。 すべてが問題なければ、新しいUbuntu + Kerrighedバンドルが表示されます
