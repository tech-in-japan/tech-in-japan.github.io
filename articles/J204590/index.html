会う：IntelliJ IDEA 13

2013年には何がありましたか



2013年に私たちにとって最も幸せなニュースの1つは、Google I / Oでした。Googleは、Android用の新しいアプリケーション開発環境であるAndroid Studioのプレビューバージョンを発表しました。 Android Studioは、IntelliJ IDEA Community Editionに基づいてJetBrainsと共同で開発されています。 IntelliJ IDEA 13は最初のメジャーリリースであり、このコラボレーションの結果として生まれた新しいものが含まれています。



この春、新しいJava 8をサポートしました。



たとえば、IntelliJ IDEAコラボレーションプラグインをTeamCityで更新し、IntelliJプラットフォームを作成した経験を使用して新製品を開発し続けるなど、ツールの1つを他のツールと思慮深く統合するという考えに引き続き取り組んでいます。 そのため、2013年9月、 新しい製品であるUpSourceを発表しました。これは、ナビゲーション、使用方法の検索、コードの変更の表示、コードの分析など、ツールに固有の機能を備えたブラウザーでコードを操作するサーバープラットフォームです。



そして今日...わずか5分前...パパム！ IntelliJ IDEA 13をリリースしました。











IntelliJ IDEAは、初心者からエンタープライズでマルチトンアプリケーションを作成するバイソンに至るまで、あらゆる開発者が最先端の技術を使用できる環境であり続けます。 IntelliJ IDEAは、Oracleコーポレートプラットフォームの最新バージョンであるJava EE 7をサポートし、Glassfish 4、Wildfly 8、Tomcat 8などのアプリケーションサーバーの新しいバージョンのサポートを追加します。



最も興味深い新機能



Java EE 7の完全サポート


Java EE 7のサポートに関しては、IntelliJ IDEAの新しいバージョンは、CDI 1.1、JSF 2.2、JAX-RS 2.0およびバッチ処理用のコードの作成を支援しています。 ビデオシリーズで改善されたJava EEサポートの詳細をご覧ください。



















さらに、13番目のバージョンでは、Javaだけでなく、あらゆるWebプロジェクトの開発に役立つコンポーネントであるRESTクライアントを改善しました。 これで認証が可能になり、リクエスト履歴が保存されます：







春


何よりも、IDEは@Configurationを使用して定義されたマッピングされていないコンテキストを自動的に検出し、それらをプロジェクト設定に追加することを提案します。







Springで作業するための新しいMVCツールウィンドウは、プロジェクトで定義されたコントローラーメソッド間をより簡単にナビゲートするのに役立ち、対応するマッピングされたURL、メソッドおよびパス変数をその中に表示できますおよびビンのドキュメント。 また、多数のBeanを含むプロジェクトのIDEパフォーマンスも改善しました。



グラドル


Gradleサポートが大幅に再設計されました。まず、依存関係とプラグインのオートコンプリート、コード生成が追加されました。 Gradle DSLのクイックドキュメントツールチップが登場し、IntelliJ IDEAがWebファセットを自動的に作成し、 WARアーティファクト構成をインポートするようになりました。 さらに、テストやexecなどのGradleタスクのデバッグが提供され、Gradle 1.8のおかげで、Gradleプロジェクトのインポートが加速されます。







バージョン管理システム


IDEA 13はSubversion 1.8のサポートを追加します。 さらに、IDEAにGitとMercurialのグラフィカルな履歴ログをすばやく表示するように指示しました。使用するのがより便利になりました。ブランチの表示がよりコンパクトになり、1つのプロジェクトで複数のリポジトリを操作しやすくなりました。







タスク管理プラグインを使用する人は、IDEの新機能に感謝します。バグトラッカーのタスクを切り替えると、IDE自体がVCSのブランチを切り替え、タスクを閉じると、コミットとマージが提案されます。



コミットダイアログボックスがより便利になりました。 GitHubプルリクエストは、IDEから直接作成できるようになりました。



拡張されたScalaサポートとSBT統合


SBTをビルドツールとして選択した人には喜ぶべき理由があります。SBTサポートがIntelliJ IDEA 13に組み込まれました。最新のScalaサポートプラグインには、SBTプロジェクトのインポート、自動プロジェクト同期、.sbtファイルの構文強調表示が含まれます。



データベースを操作する


新しいIntelliJ IDEAでは、新しいインターフェイスでデータベースとの作業を構成するのがさらに便利になりました。







以前と同様に、データベースの種類を選択するだけで、IDEが正しいJDBCドライバーを自動的にダウンロードしてインストールします。



さらに、構造ビュー、および外部キーと主キーのナビゲーションと表示がテーブルエディターに表示されました。 これらの機能は、当社のWebサイトの新機能セクションで説明されています 。



また、便利なもの-クエリフィルターも追加しました。 これを使用すると、テーブルまたはSQLクエリの結果を表示するときに必要なものをフィルタリングできます。また、列で並べ替えることにより、データベースにクエリを送信せずにIDEでテーブルを直接並べ替えることができます。







一部のタスクはほとんど無作法です。たとえば、H2またはSQLiteファイルをデータベースウィンドウにドラッグアンドドロップするだけでデータソースを構成できます。



さらにプライベートな改善があります。たとえば、Android用に開発する際に、デバイスとエミュレーターの両方でSQLiteを直接操作することがサポートされます（adb経由、デバイスのルートは不要です）。 データベースはユーザーに対して透過的にコンピューターに引き出され、通常のデータベースと同様にローカルコピーを操作できます。その後、コンテキストメニューから[ アップロード ]を呼び出して、デバイスにアップロードできます。 Habréのデータベースサポートプラグインで、ベースの操作について最近書きました。



IDEインターフェース全般


ユーザーがIDEをより快適に使用できるようにするために、インターフェイスを改善しました：デフォルトでは、IDEはメインツールバーとツールウィンドウボタンを非表示にします（後者は、左下の小さなウィンドウをクリックするか、スイッチャーと最近使用したファイルから使用できます）。







Linuxを含むすべてのオペレーティングシステムでフルスクリーンビューを使用できるようになりました。スクロールバーのエラーにカーソルを合わせると、エラーまたは警告の原因となったコードを含むポップアップウィンドウが表示されます。



Shiftをすばやくダブルタップすると、新しいSearch Everywhere機能が表示され、クラス、ファイル、アクション、またはIDE設定をすばやく見つけることができます。







待望の別の変更：IntelliJ IDEA 13は、WindowsとLinux、Community EditionとUltimateの両方で利用可能な新しい明るいLook＆Feelを追加しました。CommunityEditionではデフォルトでインストールされ、Linuxではデフォルトになりますそして究極の。







新しいターミナル


新しいコマンドラインインターフェイスでは、IDEを離れることなく、任意のローカルシステムからコマンドを実行できます。 さらに、SSHを介してリモートシステムを操作します。







素敵なささいなこと


ささいなことのいくつかは、最近の投稿（ こことここ ）で既に発表されていますが、まだ多くはまだ発表されていません。 いくつか言及すると、デバッガーの改善、Groovyのサポート、パフォーマンス、Web開発者向けの新しいツール、CloudFoundry 2.0、OpenShift、およびHerokuのサポートです。



スピーカーとスクリーンキャスト専用


IntelliJ IDEAにプレゼンテーションモードが追加されました。 ビュー経由で呼び出されます| プレゼンテーションモードを開始します 。 プレゼンテーションモードでは、コードが表示されたエディターウィンドウのみが表示され、コードが大画面で見やすくなるように拡大フォントが使用されます。



ホットキーについて言えば、ニコライチャシュニコフのおかげで、 プレゼンテーションアシスタントプラグインがIntelliJプラットフォーム全体に登場しました。このプラグインは、画面の下部に押したキーボードショートカットを大きく表示します。 これは、プレゼンテーションやスクリーンキャストを作成するすべての人にとって不可欠なものです。 JetBrainsリポジトリからすぐに配置できます！
