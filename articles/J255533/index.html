Javaスマートカードを使用してソフトウェアを保護する。 第3章IDEのインストールと構成





このシリーズの記事では、ソフトウェアを保護するためのJavaスマートカード（電子キーの安価なアナログ）の使用について説明します。 サイクルはいくつかの章に分かれています。



記事の情報を読んで理解するには、次のスキルが必要です。

  • Windowsソフトウェア開発の基礎（DelphiやVisual Basicなどの視覚環境でのプログラミングスキルで十分です）
  • 暗号化の分野に関する基本的な知識（暗号、対称、非対称アルゴリズム、初期化ベクトル、CBCなど。必須の読み取りにはBruce Schneier Applied Cryptographyをお勧めします）。
  • 少なくともリモートで構文（Java、C ++、C＃、PHPなど）のJavaを連想させる、あらゆる言語の基本的なプログラミングスキル


サイクルの目標は、読者にJavaマップを紹介することです（ロシア語での使用に関する文献はほとんどありません）。 このサイクルは、「Javaマップに基づくソフトウェア保護の開発のためのガイドライン」のステータスや「Javaマップへのガイド」のタイトルを主張していません。



サイクルの構成：



  • 第1章Javaマップ。 一般的な情報。

1.必要なソフトウェアのインストール



Javacard SDK 2.2.1を完全にサポートする通常のIDEに出くわしませんでした。 おそらく、IBMがNXPに売却したJCOP Toolsと、NXPはどこかで失ったかもしれません。 しかし、このキットを見つけたとしても、有料です。 それを使用するには、NDAの署名が必要だと噂されています。 有料のSDKがいくつかあり、そのうちCyber​​Flex SDKでしか作業していません。 好きじゃなかった。



NetBeans 6.9+には、非常に強力なJavacard Toolsプラグインが含まれています。 ただし、残念ながら、このプラグインは、Javacard SDK 3.0 Connectedと互換性のあるカードの開発のみをサポートしています。 しかし、市場にはそのようなカードはほとんどありません。 ただし、Java Card API 3.xの機能を使用せずにこの環境でアプレットを開発し、通常のSDK 2.2.1でアプレットのソースコードを単純にコンパイル/変換することはありません。



開発のために、私は長い間Java開発者向けの通常のEclipse IDEを使用しましたが、SDKから必要なユーティリティを組み込むことで少し頭に浮かびました。 そのため、Eclipse WebサイトからJava開発者向けのEclipse IDEの最新バージョンをダウンロードしてください。



ダウンロードしたアーカイブをどこかに（Program Filesでは望ましくない）ハードドライブにダウンロードします。 通常、eclipse-javaフォルダーのドライブのルートに配置します。 以降、フォルダ名にロシア語の文字とスペースを使用することは望ましくありません。



次に、すべてのバージョンのJava SDKが必要です。 通常、JavaSE（Java Standard Edition SDK）、JDKと呼ばれます。 OracleがSunを買収したため、リンクは変更される可能性がありますが、これが発生した場合はGoogleがお手伝いします。 これまでのところ、ここからダウンロードできます。 インストーラーが装備されているため、通常どおりにインストールします。



また、 Java Card Development Kit 2.2.1も必要です。 また、通常はハードドライブのルートに配置します。



次に、システムを少し構成する必要があります。 キーボードでWindows + Breakを同時に押すか、単に[マイコンピュータ]のプロパティを開きます。 [詳細設定]タブをクリックし、[環境変数]ボタンをクリックします。 以下をシステム変数に追加する必要があります。



JC_HOME = C：\ java_card_kit-2_2_1 -Java Card Development Kitを解凍したパス

JAVA_HOME = c：\ j2sdk1.4.1は、Java SDKがインストールされているパスです。 Program Filesにインストールする場合は、短い8.3パス「C：\ Progra〜1 \ ...」を使用することをお勧めします。



ここで、値「;％JC_HOME％\ bin;％JAVA_HOME％\ bin;」をPATH変数の最後に追加する必要があります。 。 また、変数％..％は展開してPATHパスに完全に指定する方が適切です。



Java Card SDK（サブディレクトリのサンプル）があるフォルダーにあるbuild_samples.batファイルを実行することにより、インストールを確認できます。 ただし、1.3または1.4以外のJava Developer Kitを使用する場合は、少し調整する必要があります。 build_samples.batファイルを開き、パラメーター「-target 1.2 -source 1.2」を「set JCFLAGS 」行に追加します（もちろん引用符なし）。 カードのファイルコンバーターは、互換性のあるコンパイラーによってファイルがコンパイルされることを期待しているため、これが必要です。



build_samplesが起動して正しく実行されると、 ％JC_HOME％\ samples \ classes \ com \ sun \ javacard \ samples \ HelloWorld \ javacard \フォルダーに、 HelloWorld.cap、HelloWorld.exp、HelloWorld.jcaファイルが表示されます。 HelloWorld.capファイルは、マップにすぐにアップロードする準備ができています。



2. Eclipseを構成する



Eclipseを起動します。 最初の起動時に、ワークスペースの目的の場所について尋ねられます。 通常、デフォルトの場所をそのまま使用します。



新しいJavaプロジェクトを作成します。 ファイル->新規->プロジェクト-> Java-> Javaプロジェクト。 彼に何かを呼んでください。 必要に応じて、ワークスペースの外側に配置できます。 残りの設定はそのままにしておくことができます。 次へをクリックします。 [ライブラリ]タブに移動し、そこにあるすべてのものを削除します。 [外部JARの追加]ボタンをクリックし、 ％JC_HOME％\ lib \ api.jarファイル（％JC_HOME％=解凍したJava Card SDK内）を選択します。 完了をクリックします。



左側に[パッケージエクスプローラー]ウィンドウが表示されます。 srcフォルダーを右クリックして、「新規」->「パッケージ」を選択します。 テストアプレットを含むパッケージを作成する必要があります。 好きな名前を付けてください。 たとえば、testpackage。 パッケージエクスプローラーでパッケージアイコンを右クリックします。 [新規]-> [クラス]を選択します。 スーパークラス（アプレットのクラスを継承するクラス）として、「javacard.framework.Applet」を選択する必要があります。 main（）（スーパークラスのコンストラクタ、継承された抽象メソッド）を除く、Eclipseが提供するすべてのスタブの作成に注意してください。 アプレットクラスの名前を入力します。 たとえば、TestApplet。 完了をクリックします。



ご覧のとおり、Eclipseはアプレットスタブを生成しました。これは、以降の章で完全に機能するアプレットになります。



ここで、アプレットをコンパイルして* .capファイルに変換するプロセスを自動化するファイルを作成することにより、作業を少し楽にする必要があります。 これを行うには、2つのファイルを作成する必要があります。 1つはコンパイルおよび変換とアプレット用で、もう1つはマップへのダウンロード用です。 ただし、必要に応じて、すべてを1つのファイルに結合できます。 パッケージにmytestappletという名前を付け、アプレットにMyTestAppletという名前を付けたことに注目してください。



したがって、build.bat： 



@echo off set APPLET_HOME=C:\Work\JavaCard\MyTestApplet set PACKAGE_NAME=mytestapplet set APPLET_NAME=MyTestApplet del /q %APPLET_HOME%\bin\%PACKAGE_NAME%\*.class del /q %APPLET_HOME%\bin\%PACKAGE_NAME%\javacard\*.* %JAVA_HOME%\bin\javac -g -d %APPLET_HOME%\bin -classpath %JC_HOME%\samples\classes\;%JC_HOME%\lib\api.jar;%JC_HOME%\lib\installer.jar -source 1.2 -target 1.2 %APPLET_HOME%\src\ %JC_HOME%\bin\converter -config %APPLET_HOME%\%APPLET_NAME%.opt pause




ここではコメントなしですべてが明確になっていると思います。 また、コンバータが情報を読み取るサンプルのMyTestApplet.optファイル（最後から2番目の行* .bat）： 



 -out CAP JCA -exportpath E:\java_card_kit-2_2_1\api_export_files;D:\java_card_kit-2_2_1\lib\api.jar -classdir C:\Work\JavaCard\MyTestApplet\bin\ -d C:\Work\JavaCard\MyTestApplet\bin\ -verbose -applet 0xa0:0x46:0x52:0x41:0x43:0x54:0x41:0x4c:0x0:0x1 mytestapplet.MyTestApplet mytestapplet 0xa0:0x46:0x52:0x41:0x43:0x54:0x41:0x4c:0x0 1.0




-appletディレクティブ（アプレットのAIDを示す）と最後の行（AIDとパッケージバージョン）に注意してください。 *操作後の.capファイルは、フォルダーC：\ Work \ JavaCard \ MyTestApplet \ bin \にあります。



このファイルは、「実行」->「外部ツール」->「外部ツール構成」を介してEclipseメニューに追加できます。 ここではすべてが非常に簡単で、説明を必要としないと思います。
