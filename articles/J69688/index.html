Web開発者とのオンライン会議

毎週、有名なWeb開発者や他の有名なWebワーカーとオンライン会議を開催しています 。 9月の小さなレポートをお見せしたいと思います。



セルゲイ・チクヨノク



直近では、Artemy Lebedev StudioのテクニカルコンサルタントであるSergei Chikuenk（ 発表 ）との会議を開催しました。 セルゲイが詳細な回答をした興味深い質問がたくさんありました。 それらのいくつかを次に示します。



エフゲニー・ナシロフ ：こんにちは、セルゲイ！ あなたが参加した最も興味深いプロジェクトについて教えてください。

Sergey Chikuyonok ：私の最も興味深く、好きなプロジェクトはParity98です。 1ページのさまざまなアニメーション（サイトの一部には残念ながら表示されなかったものもあります）がたくさんあるのではなく、コンセプトが1行のサイトであることを覚えています。 これが主な困難でした-テキスト行にブロックを挿入し、それらをスムーズに引き離すことさえ必要でした...（ 全文を読む ）



Inko ：ページの上部にアピールとリンク（ ie6update.comなど ）を含む行を表示することで、IE6（新しいものへのアップグレード）を取り除く手助けをするよう人々に促します。 セルゲイ、そのような取り組みに対するあなたの態度はどうですか？ サイトに載せる価値はありますか、それともナンセンスですか:-)？

Sergei Chikuyonok ：そのようなラインに対して私は何もしていません-方法は良くて正しいです、邪魔にならないように、セッションごとにこのバーを3回しか表示しません。 主なことは、IE6でサイトが正常に機能することです。 フリルや美しさはありませんが、基本的な機能とコンテンツは利用できるはずです。 私が本当に反対しているのは...（ 続きを読む ）



Valdemar Mozhaykin ：セルゲイ、HTML5サイトの作成は今正当化されていますか？ または、古いブラウザーが市場から追い出され、このテクノロジーをサポートするブラウザーのみが残るまで待つ必要がありますか？

Sergei Chikuyonok ：これまでのところ、<div id = "footer">の代わりに<footer>を記述する理由はありません。特に10回変更される可能性があることを考慮してください。



Leonid Nazarov ：レイアウトデザイナーが効率的にレイアウトを作成できるように、デザイナーに何をする必要がありますか（またはその逆は必要ありませんか）？ どのファイル、説明を添付する必要がありますか？

Sergey Chikuyonok ：デザイナー向けに、次のヒントを提供できます。

1.幅が1000ピクセルのレイアウトを作成すると、サイトをストレッチする割合をカウントしやすくなります。

2.レイアウト全体を同じ幅の列に分割し、ページ要素をそれらに合わせます。 列の数が余りなしで100を分割することが望ましい（たとえば、2、4、5、10）-グリッド上の異なるブロックの要素を簡単に揃えることができます（たとえば、コンテンツの画像をヘッダーのロゴと揃える）が、これはすでにより依存していますエンコーダーのスキルから。 また、複数列のサイトのレイアウトに関する記事を書きます。 （ 6その他のヒント ）



ほとんどの答えは非常に膨大であるため、保存する必要がありました。 ここで会議全体を読んでください 。



ダミール・ハリロフ



セルゲイに続いて、GreenPRのCEOであるDamir Khalilovがスピーカーを務めました。 Damirは、ソーシャルネットワークでのマーケティングとインターネットでの個人ブランドの作成に関する質問に答えました。



Dmitry Zvyagin ：ロシアでのTwitterの人気の低さは、他の国と比較してどうしてですか？

Damir Khalilov ：私たちはすべてが始まったばかりだと思います。 ここ数ヶ月のロシアのTwiの異常な成長率はこれを裏付けています。 もちろん、VKontakteはTwitterを追い抜く可能性は低いが、LJやLIと競合することは可能だろう。



Stas Gladkov ：こんにちはDamir！ コンテストはソーシャルネットワークのマーケティングツールだと思いますか。 コンテストを作成するとき、どの点に注意することをお勧めしますか？ コンテストをバイラルにする方法は？ 競争からトラフィックを得る方法は？ 成功したコンテストの例を挙げてください。 ありがとう

Damir Khalilov ：コンテスト、フラッシュモブ、インタラクティブプロモーションは非常に効果的なSMMツールです。 原則として、ユーザーはこのタイプの相互作用に非常に興味があります。ゲームのメカニズムが関係しているためです。これは、Anton Popovが書いているゲームマーケティングと同じです。

コンテストを作成するとき、...に最大限の注意を払うことをお勧めします...（ 続きを読む ）



Dmitry Zvyagin ：Twitterを使って何らかの収入を現金で得るのは現実的ですか？ もしそうなら、なぜ今誰もそれを使用していないのですか？

Damir Khalilov ：本当に。 獲得には3つの主なオプションがあります。

1. Twiを介して会社のサービスまたは独自のサービスを宣伝します。 ここではすべてが明確だと思います-主なことは名声を収益化する能力です。

2. Twiを通じて何かを販売します。 それはあなたの製品またはパートナーの製品である可能性があります...（ 全体を読む ）



質問に対するこれらおよび他の回答は、 ここで見つけることができます 。



イゴール・シャピロ



今日、ユーザーからの質問には、Conduitのシニア開発者であるIgor Shapiroが回答しています。 Conduitは、イスラエルに拠点を置く、ブラウザツールバーの世界的な大手プロバイダーです。 システムは24×7モードで200,000のパブリッシャーと70,000,000のユーザーにサービスを提供します。 このようなシステムは、スケーラビリティ、信頼性、効率の問題に緊急に直面しています。

スケーラブルなシステムの設計と開発についてイゴールに質問できます。







この会議は本日、ウェブサイトでモスクワ時間の約15:00に開催されます。
