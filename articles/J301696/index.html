シスコソリューションに基づいた自動受付の実装





ほとんどすべての会社に電話をかけるとき、私たちは通常、私たちがどこを通り抜けて、いくつかの行動を実行するよう申し出たことを思い出させる、心地よいまたはあまり声ではない声を聞きます。 ヒットします...自動受付をヒットします。 自動受付/ AAは通常、 対話型音声応答/ IVRの最も単純なケースです。これにより、内線番号のダイヤル、音声メッセージまたはFAXの送信、会社への電話時に秘書との連絡が可能になります。



この記事では、シスコのソリューションに基づいて自動応答を実装するためのオプションを理解することをお勧めします。 少なくとも4つあります。シスコユニファイドコミュニケーションソリューションを実装する場合、この機能を正確に実装する対象を尋ねることがよくあります。 それぞれの長所と短所を見てみましょう。



1. TCLおよびVXMLでの既製のスクリプトの開発または使用



このソリューションは完全にソフトウェアであり、機能はCisco IOSオペレーティングシステムに組み込まれており、Unified Communications（UC）機能セットがアクティブになっているルータでサポートされています。 シスコ機器用のIVRスクリプトの例はたくさんあります。 たとえば、ハブにはTCLとVXMLのオプションがあります。



このオプションの利点は次のとおりです。

  • UC機能を備えたCiscoルータ自体を除き、追加のハードウェアは必要ありません。
  • 幅広いサポート対象プラットフォーム：1861、2800、3800、2900、3900、4000、AS5000。
  • プログラミングしているので、分岐、サブメニュー、複数の言語バージョン、稼働時間/非稼働時間など、必要なスクリプト操作アルゴリズムを実装できます。 音声認識および合成システム（ASR、TTS）などとの統合をサポートしています。
  • 他のすべてのオプションよりもVXMLを使用する場合のグリーティングでサポートされるコーデックのより大きなセット。 また、誰も通常使用しないもの（同じアプリケーション内で異なる音声コーデックを使用して録音されたグリーティングの組み合わせ）もサポートします。
  • ゲートウェイとそのリソースへの直接アクセス-たとえば、通話中にFAXトーンをキャッチし、すぐにFAXに切り替えることができます。発信通話を使用すると、通話に応答した人物（もちろん、人または留守番電話）を（ある程度の確率で）理解できます。
  • VXMLは、VoIPソリューション（Audiocodes、Huawei、D-Link、Digium）の多くの開発者によってサポートされているオープンスタンダードです。
  • TCLに基づいて記述された既製の非常に機能的なシスコのスクリプトがあります。これは、基本的な自動コール配信（B-ACD）および自動アテンダント（AA）サービスです。










その操作のアルゴリズムがあなたに合っている場合、私たちはそれを取り、設定し、使用します。



このオプションの欠点は次のとおりです。

  • TCLを使用する場合、プログラミングスキルとCisco Voice Gateway API（VGAPI）の知識が必要です。 VXMLを使用する場合、プログラミングトピックを入力するためのしきい値ははるかに低く、特にXMLにすでに精通している人にとっては、Cisco VGAPIの知識は必要ありません。
  • Cisco IOSは、TCLアプリケーションによって処理されるコールにトランスコーダを使用しないため、アプリケーションの入力と出力の両方で同じ音声コーデックを使用する必要があります。
  • 比較的弱いスケーラビリティ-7（2800シリーズ）から360（AS5400XM）同時セッション（呼び出し）まで。
  • フォールトトレランスなし。
  • Cisco IOS音声XMLブラウザ機能は、同時コールの数によってライセンスされます（価格* FL-VXML-1の1セッション= 150ドル、FL-VXML-12の12セッション= 1440ドル）。 FL-VXMLライセンスの種類は、使用権、つまり 購入時に、シリアル番号と製品アクティベーションコードなしの使用権に関する文書を受け取ります（「名誉ベース」）。
*記事のすべての価格は、Cisco GPLから取られています



このオプションは、同時コールの数が少ない場合に適しており、PBX（Ciscoだけでなく）と統合できます。



2. Cisco Unity Express（CUE）を使用する



このソリューションは、Cisco 1861、2800、2900、3800、3900シリーズルータのモジュール形式（SM-SRE-700-K9など）で提供されます。 IVRスクリプトは、Cisco Unified Communications Express Editorを使用して作成されます。 ソリューションは非常に機能的であり、スクリプトと作業のロジックを作成するプロセスは、UCCXに基づく古いソリューションと強く重複しています。これについては、以下で説明します。 自動応答はスタンドアロンモードでも動作できるため、Cisco Unity ExpressはCisco Unified Communications ManagerおよびCisco Unified Communications Manager Expressと統合されます。



このオプションの利点は次のとおりです。

  • 便利なWEB管理インターフェイスCisco Unity Express-管理。
  • 豊富な機能。
  • スケジュール、週末、休日の便利な仕事。







  • Cisco Unified Communications ExpressエディターでのIVRスクリプトの開発は非常に簡単で、必要なブロックをドラッグアンドドロップしてそのプロパティを設定するビジュアルプログラミングに似ています。







  • より簡単なものもあります-基本アクションをサポートするスクリプト用のEditor Express Webエディター：







  • さまざまな言語の組み込みシステムフレーズと、既製の自動応答スクリプト（aa.aef）があります。

  • Cisco Unity Expressはボイスメールサーバであるため、CUEの導入により、自動受付に加えて、ルータでボイスメールの機能を利用できます。


このオプションの欠点は次のとおりです。

  • 弱いスケーラビリティ。 セッションの最大数には厳密な制限があります（ISM-SRE-300-K9の場合は10、SM-SRE-700の場合は32-SM-SRE-910）。
  • グリーティングでは、G.711 u-lawコーデックのみがサポートされます。
  • フォールトトレランスなし。
  • インストール、構成、統合が必要です。
  • モジュールのコストと音声ポートのライセンス（同時通話数）：

    • モジュール自体：たとえば、ISM-SRE-300-K9 = 1000ドルには、2つの音声ポートFL-CUE-NR-PORT-2の非ポータブルライセンスが含まれています。
    • 5個のメールボックスのライセンスFL-CUE-MBX-5 = 100ドル。
    • 追加の音声ポート2個のライセンス。 -FL-CUE-PORT-2 = 500ドル。
    • IVR音声ポートのライセンスFL-CUE-IVR-2 = 1400ドル（メール、FAXの送信、外部データベースへのアクセス、CUE IVRスクリプト内でのVXMLスクリプトの実行など、高度なIVR機能を使用する場合にのみ必要） 。）。


このオプションは、Cisco Unified Communications Manager ExpressをPBXとして使用する企業、またはCisco Unified Communications Manager PBXがセントラルオフィスに展開され、Cisco Unified Survivable Remote Site Telephony（SRST）によってブランチフォールトトレランスが提供される企業のブランチに最適です。 このオプションは、TCL / VXMLの場合と同様に、スタンドアロンの自動応答として電話交換と統合できます。



3. Cisco Unity Connection（CUC）を使用する



このソリューションはボイスメールサーバーであり、別の仮想マシンにインストールされます。 2つのCUC仮想マシンは、フェールオーバークラスターに結合されます。 組み込みのCUCツールを使用すると、自動応答のロジックを作成できます。 組み込みの呼び出しハンドラが使用されます-ハンドラ。



このオプションの利点は次のとおりです。

  • 高いスケーラビリティとフォールトトレランス-単一のサーバーで最大250の同時セッション（呼び出し）、アクティブ/アクティブクラスターで最大500。
  • サポートされているかなり幅広いコーデック（G.711 a / u-law、G.722、G.729、iLBC）、および組み込みのトランスコーディング機能。 CUCでのトランスコーディングは常に発生することに注意してください。
  • スケジュール、週末、休日の便利な仕事。
  • ロシア語をサポートする組み込みASRおよびTTS。
  • Cisco Unity Connectionはボイスメールサーバであるため、CUCの導入により、自動受付に加えて、IPテレフォニーシステムでボイスメールの機能を利用できます。
  • サードパーティ（シスコ以外）のPBXと統合する機能。


このオプションの欠点は次のとおりです。

  • 弱い機能。 組み込みの標準構造（ハンドラー）を使用して自動応答を作成するため、実装できる機能は限られています。
  • 組み込みのフレーズ（たとえば、「通話中の相手」）を自分のフレーズに置き換えることはできません。
  • インストール、構成、統合が必要です。
  • ボイスメール（メールボックス）の最初のユーザーのライセンスのコストUNITYCN11-STD-USR 78.75 $


このソリューションは、同時呼び出しの数と重要なフォールトトレランスに高い負荷を必要とする企業に適しています。



4.コンタクトセンターに基づく（Cisco Unified Contact Center ExpressまたはCisco Unified Contact Center Enterprise）



最も機能的で柔軟なソリューション。 UCCXは、CUCと同様に、別の仮想マシンにインストールされます。 2つの仮想マシンを組み合わせてフェールオーバークラスターにすることができます。 自動応答の場合、IP IVR機能を設定するだけで十分です。 Cisco Unified CCX Editorを使用してIVRスクリプトを作成します。 UCCXはCisco Unified Communications Managerと統合します。 セッションの最大数は、仮想サーバーの容量（仮想マシンテンプレート）と購入したライセンスの数によって制限されます。



このオプションの利点は次のとおりです。

  • 高いスケーラビリティと弾力性。
  • 最も一般的に使用されるコーデックG.711 a / u-law、G.729のサポート。
  • 最も豊富な機能。
  • WEB管理インターフェイス-Cisco Unified CCX管理。
  • Cisco Unified CCX EditorでのIVRスクリプトの開発は比較的単純で、視覚的なプログラミングに似ています。







  • さまざまな言語の組み込みシステムフレーズと、既製の自動応答スクリプト（aa.aef）があります。


このオプションの欠点は次のとおりです。

  • インストール、構成、統合が必要です。
  • ソリューションコスト：コンタクトセンターの5人のエージェント（オペレーター）と100個の音声ポート（同時通話）の最小プロモーションキット：1044.75ドル


UCCXは、高度なコンタクトセンターテクノロジーをサポートする本格的なコンタクトセンターです。したがって、自動アテンダントを作成するためだけに使用します。大砲からスズメを撃ちます。 このソリューションは、機能、パフォーマンス、および信頼性に対する要求が高い企業に適しています。



自動応答を整理するためのさまざまなオプションを検討しました。 選択は、使用可能なソリューションまたは実装されているソリューション（CUCMe、CUCM、BE6Kまたはサードパーティの交換）、および機能、パフォーマンス、および信頼性の要件に大きく依存します。



おわりに



最も簡単なオプションは、TCL / VXMLに基づくIVRであり、最も機能的で生産性の高いものは、Cisco Unified Contact Center Express / Enterpriseに基づきます。



Cisco Unity Connectionソリューションは、必ずしも見過ごされがちではありません。 通常、Cisco Unified Communications製品を購入すると、Cisco Unity Connectionに購入したライセンスが付属します（たとえば、UWLライセンスとBusiness Editionバンドルを購入した場合）。 この場合、自動応答を実装するために、追加する必要はありません。



Cisco Unity Expressソリューションを使用すると、ルータからのコールを処理する負荷を軽減できます（TCL / VXMLユースケースと比較）。IOSのスクリプトソリューションと専用サーバーのソリューションの中間の位置を占めます。 非常にシンプルで便利なため、次のように、会社のIT部門を単独で使用して自動応答を編成する問題を解決するのに適しています。 基本的なアクションをサポートするスクリプト用のシンプルな組み込みWebエディターがあります。



自動応答機能を実装するためのサードパーティソリューションとCisco IPテレフォニーシステムを統合するためのオプションがありますが、これはまったく異なる話です...
