主要なマイクロソフトプログラマーがテキストエディターで作業する

過去20年にわたって、MicrosoftはGUIを促進するために他の誰よりも多くのことを行い、Visual BasicおよびVisual Studioのツールは世界中の何百万人ものプログラマーに提供しています。 ただし、Microsoftのスペシャリスト自身は、多くの場合、プレーンテキストエディターでのコーディングを好みます。



これは 、プログラミングの将来に関するセクションの一部として、最近のPDC会議で議論されました。 「プログラムで絵を描くのは好きですか？ おそらくそうです」と、Microsoftで宣言型プログラミング言語とツールの作成に取り組んでいるDon Box氏は言います。 「しかし、あなたが私のようなプログラミングで育ったなら、あなたはそれをテキスト形式でやります。」 そして、私たちがそれを失いつつあることは私にとって危険なようです。」



「グラフィカルプログラミング環境は、役に立たないときには便利であり、必要なときには不便です」と、Microsoftの主要な開発者でありPowerShellの作成者であるジェフリースノーバーは付け加えました。 -画面上に5つのものがある場合、テキストによる手段を扱うことができますが、500ある場合、グラフィカルインターフェイスが役立ちますが、この場合は完全に機能しません。 ズームインおよびズームアウトすると、コンテキスト全体が失われます。」



ビジュアルプログラミングは習得が容易で生産性の向上に役立ちますが、「誤解を招きやすい」ともマイクロソフトの技術コンサルタントであるバトラーランプソンは述べています。 「たとえば、誰もUMLダイアグラムの意味を教えてくれません。」 バトラーランプソンは、1992年のチューリング賞受賞者であり、9つのプログラミング言語の共著者です 。



Jeffrey Snoverは、プログラミングが非常に抽象的なレベルに達し、Xboxゲームコントローラを使用してコードをすぐに記述できると冗談を言いました。 そのような新世代のコントローラー（Project Natal）は体の動きを認識します。つまり、画面の前で踊ることができます。



C ++ / CLI言語の主要な開発者である別のプログラミングの第一人者であるHerb Sutterは、10年後にテキストエディターでのプログラミングが流行に戻ると予測しています。パフォーマンスの限界に達すると、「最適化が再び非常に魅力的になる」からです。
