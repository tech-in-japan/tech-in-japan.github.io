Calculate Linux 11.9のデスクトップセットアップ

Calculate Linux 11.9ディストリビューションの新しいバージョンでは、ユーザープロファイルの設定方法が変更されています。 この記事では、Linuxデスクトップの計算の例を使用してプロセスを説明し、主要なポイントを強調します。









ユーティリティパックの計算



Pythonで書かれた計算ユーティリティは、インストールだけでなくシステム構成にもすべての作業の負担を果たします。 「calculate-desktop」パッケージは、ユーザープロファイルの設定を担当します。 他のユーティリティとは異なり、システムファイルだけでなくユーザーファイル（ホームディレクトリ内）も構成できます。 例：



# cl-desktop alex

* alex ...







カスタマイズはユーティリティテンプレートを使用して行われますが、その詳細については以下をご覧ください。

ユーティリティテンプレートの計算



数年前、Calculate Next Webサイトの新しいエンジンを作成するプロジェクトの一環として、マークアップ付きのテキストをXML形式に、またはその逆に変換するための多くのメソッドが作成されました。 エンジンが追加されなかったという事実にもかかわらず、多くの開発が他のプロジェクト、特にCalculate Utilities 2に含まれていました。



ほとんどの場合、構成ファイルのいくつかのパラメーターを変更するだけで、サービスまたはアプリケーションが要件に従って機能することが知られています。 このルールは、Apache、Bind、Compiz、dhcp、Dovecot、kde、ldap、openrc、plasma、postfix、procmail、proftpd、samba、squid、xml_gconf、xml_gconf_treeなど、最も一般的な構成ファイル形式で理解されるテンプレートの動作に基づいています。 、xml_xfce、xml_xfcepanel。 他の形式の場合は、テキスト生、バイナリビン、およびパッチ形式を使用できます。



テンプレートは、システムのテンプレートと組み合わせることができる単なる設定ファイルではありません。 条件式、関数、そしてもちろん変数を含めることができます。 結合のプロセスでは、テンプレートと構成ファイルがXML形式に変換され、次に結合されます。その後、XMLは構成ファイルの指定された形式に再び変換され、システムにコピーされます。



例1. kwallet信頼アプリケーションの設定、kwallertcファイル：

# Calculate format=kde path=~/.kde4/share/config

しばらくの間、KDEベースのアプリケーションはローカライズを使用し始めました。 すべての可能な言語を設定でロードしないように、システムの言語に応じてチェックおよび調整します。



例2. XFCEの下部パネルにあるファイルbrowser.desktopのデフォルトのブラウザーアイコンへのリンクを作成します。

# Calculate exists(/usr/share/applications/calculate-browser.desktop)!= link=/usr/share/applications/calculate-browser.desktop name=10000000030.desktop symbolic force







この例では、10000000030.desktopという名前で、calculate-browser.desktopファイルへのシンボリックリンクが作成され、ファイルがチェックされます。 同じ名前のファイルがある場合、そのファイルは上書きされます。



テンプレートストレージパス



Calculateユーティリティのパスがあまり関係ない場合は、テンプレートのレイアウトが重要な役割を果たします。その例では、独自のテンプレートを作成できるためです。 便宜上、Calculate 2.2テンプレートはすべて1つのパッケージ「calculate-templates」に含まれています。 インストール中に、パッケージの内容はディレクトリ/ usr / share / calculate / templatesに展開されます。 ここでは、ユーザープロファイル設定を見つけることができます。 テンプレート内のディレクトリ名は固定されておらず、そのプロパティはディレクトリ内の「.calculate_directory」ファイルで設定されます。 ディレクトリには一般的なテンプレートルールがあります。 calculate-desktopパッケージが機能するテンプレートは、ディレクトリ「/ usr / share / calculate / templates / desktop」にあります。 このディレクトリのプロパティを記述する.calculate_directoryファイルには、デスクトップディレクトリテンプレートのヘッダーが含まれています。



# Calculate append=skip cl_name==calculate-desktop&&cl_ver>=2.2.1







これは、テンプレートがパッケージに属していることを明確に示し、ユーティリティのバージョンがチェックされます。 ユーザープログラムの設定は、ディレクトリ「/ usr / share / calculate / templates / desktop / desktop」に保存されます。 たとえば、ディレクトリ「60 -mc-4.7」には、Midnight Commander設定テンプレートが含まれています。



カスタマイズイベント



Calculate Linuxは、Portageをパッケージ管理システムとして使用します。 この非常に柔軟なシステムにより、サードパーティの開発を組み込む自由度が高まります。 プロファイルユーティリティは、profile.bashrcスクリプトを使用して、システムプロファイルを介してemergeパッケージマネージャーと統合します。 したがって、パッケージのインストール時に、Calculateユーティリティを使用してシステム構成が呼び出されます。 Calculate-desktopは、インストールされたパッケージのテンプレートを探し、必要に応じて設定を行います。 設定がユーザープロファイルに限定されない場合、パッケージはシステムファイルを変更できます。 Portageはこれらのファイルを独自のものとみなします。 ハッシュ付きの新規および変更されたファイルへのパスは、インストールされたパッケージのローカルデータベースに残ります。



パッケージのインストールに加えて、プログラムが構成されている他の多くのイベントがあります。 たとえば、kdm / gdmを使用してセッションを開始するときに、ユーザーがディレクトリを持たない場合、セッションが作成および構成されます。 これらのすべてのイベントは、テンプレートに条件式が組み込まれている特殊な変数で説明されています。



カスタムパッケージ情報



標準のユーティリティメソッドを使用するユーザープロファイル設定テンプレートは、設定をファイル「〜/ .calculate / ini.env」に保存します。 Sambaのようなファイルには4つの主要セクションが含まれます。「main」-一般設定、「ver」-構成済みプログラムのバージョン、「update」-インストールされた各プログラムの更新を無効にするフラグ、「resource」-標準ユーザーディレクトリへのパス。 セクションの数に制限はありません。たとえば、当社では、「office」セクションを使用して特定の設定を考慮しています。 ドメインユーザーセッションを開始すると、テンプレートでプロファイルが構成されます。 通常、ソフトウェアの更新時またはアプリケーション設定の修正時に変更を加える場合、設定が必要です。



ini.envを使用すると、すべてのアプリケーションと単一のアプリケーションの両方の更新を有効または無効にできます。 たとえば、プログラムの初期設定を復元できます。



Calculate Linux 11.9テンプレートの操作の変更



99％の場合、ログイン時にユーザープロファイルを設定する必要はありませんでした。 Calculate Linux 11.9では、ローカルユーザーとドメインユーザーの間に線が引かれました。 以前のユーティリティが、最初の起動前にプログラムを設定するためにシステムで作業しているユーザーのみのプロファイルを変更できる場合、すべてのローカルユーザーのプロファイルが設定されます。 これにより、セッションに入るときに更新を削除でき、最大5秒節約できました。 変更の場合、変更されたファイルはパッケージのインストール中に表示されます。



新しいISOイメージをインストールしてシステムを更新しても、問題は発生しません。 最初の起動時に、更新ユーティリティが起動します。 たとえば、KDE ​​4.6からバージョン4.7に切り替えると、テンプレート「/usr/share/calculate/templates/desktop/desktop/20-kde-4.6-4.7/.kde4/share/config/nepomukserverrc」で変更される構成ファイル「nepomukserverrc」は1つだけです。次の内容で：

# Calculate format=kde



[Basic Settings]

Start Nepomuk=true







KDE 4.7より前は、Nepomukデーモンが存在しなくても気にしませんでした。
