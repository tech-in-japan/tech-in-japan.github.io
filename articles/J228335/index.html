Eucalyptus 4.0 Cloudのインストール

Eucalyptus 4.0のインストールの経験についてお話したいと思います。 これは、EucalyptusのAWS互換クラウドシステムの最新バージョンです。 多数の変更と改善とともに、同社はデモのインストール方法を変更しました。 ISOから、Chefレシピを使用してインストールに進むことが決定されました。 これにより、すぐにテストできるソリューションのインストールシステムを削減し、ソリューションを単一のコマンドラインに拡張できました。





AWS互換システムを完全にテストするために必要なのは、/ varに100GBの空きディスク容量がある最小のCentOS 6.5インストールだけです。

インストールするには、コマンドラインでのみ入力する必要があります：

bash <(curl -Ls eucalyptus.com/install)





システムはいくつかの簡単な質問をし、コーヒーを飲みに行きます。 15分後（インターネット接続の速度に応じて）、すぐに開始できる1つのダウンロードされたイメージを備えた既製のクラウドシステムが得られます。

これを行うには、キーペアを作成します。

euca-create-keypair -f test test





そして、仮想マシンを起動します。

euca-run-instances `euca-describe-images | grep default | awk {'print $2'}` -k test







現在、Eucalyptusは、インストール用の次のOSを備えた仮想マシンを提供しています。CentOS6.5、CentOS 7.0、Fedora 20、Ubuntu 12.04、Debian 7、openSUSE 13。

以下を呼び出すことで、最も簡単なインストール方法を使用しました。

bash <(curl -Ls eucalyptus.com/install-emis)





必要なOCをインストールしました。

前回は、新しいハードウェアを追加せずに起動できる仮想マシンの数を増やすために、システムを構成する方法をすでに書きました。 Eucalyptus 3.4の設定は、Eucalyptus 4.0と互換性があります。

より多くの計算能力が必要な場合は、次を呼び出して新しいノードを追加できます。

bash <(curl -Ls eucalyptus.com/install-nc)





CentOS 6.5がインストールされ、/ varに少なくとも100GBの空きディスク容量がある別のマシン上。 その後、以下を呼び出して、既にインストールされているシステムに登録する必要があります。

euca_conf --register-nodes <ip >







AWS互換クラウドシステムを完全にテストするのに必要なことはこれだけです。
