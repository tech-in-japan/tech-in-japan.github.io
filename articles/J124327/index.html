1.なぜeval = evilなのですか？



1. Evalはプログラムの通常のロジックに違反し、コードを生成します。

2.現在のコンテキストでコードを実行し、それを変更できます-攻撃の潜在的な場所。 コンテキストは、次のトリックを実行することで変更できます。 

(1,eval)(code);


ここで何が起こるか：evalは、呼び出し方法に依存する同じ関数です。 この場合、コンテキストをグローバルに変更します。



3. evalによって生成されたコードはデバッグが困難です。

4. Evalの完了には非常に長い時間がかかります。

16. JavaScriptのGOTO

JavaScriptにはタグがありますが、gotoはありません（予約語のみがあります）。 ラベルは他のすべての言語と同じように見えますが、動作が異なります。 ラベルはブロックを選択します。 ブロック内に他のマークされたブロックがある場合があります。 このブロック内にあるbreakとcontinueを使用してラベルに移動できます。



例： 

 loop1: for (var a = 0; a < 10; a++) { if (a == 4) { break loop1; //  4  } alert('a = ' + a); loop2: for (var b = 0; b < 10; ++b) { if (b == 3) { continue loop2; //  3 } if (b == 6) { continue loop1; //   loop1 'finished'    } alert('b = ' + b); } alert('finished') } block1: { alert('hello'); //  'hello' break block1; alert('world'); //     } goto block1; // Parse error - goto
