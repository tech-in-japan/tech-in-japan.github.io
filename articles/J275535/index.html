Ubuntuでapcupsd + Zabbixを使用してippon Winner 3000無停電電源装置を監視する

この記事を書くきっかけは何ですか？ 「ippon winner 3000 monitor」というクエリをインターネットで検索します。 インターネットは通常NUTの設定に関する記事を出しました。 このプロジェクトは本当に興味深いもので、多くの機能があります。 しかし、小さなものが1つあります！ USB経由のippon winner 3000ではサポートされていません。 COMポートでは、すべてがうまく機能します。 USB接続の設定を見ましたが、実際に検証せずにコピーして貼り付けるように思えます。 たぶん私は間違っています。 少なくとも私はこれを機能させるために半日殺したが、無駄だった。



私は代替プロジェクトapcupsdを見つけました。すべてが小さな設定で箱から出されました。 ipponの勝者3000 UPSを1台のコンピューターに接続し、そのタスクはUSB経由で監視を担当し、Zabbixもそのコンピューター上にあります。 Zabbixの詳細については説明しませんが、別の記事に値するものであり、1つの記事に値するものではない、とだけ言います。 プロセスを書くことには意味がありません;そのような記事はたくさんあります。 既製の設定を持ち込むだけです。 パッケージ自体をインストールします。

$sudo apt-get install apcupsd

簡単なスクリプトを使用して実行します

apcups-start.sh 
 #!/bin/bash [ -n "`pgrep apcupsd`" ] && killall apcupsd ; sleep 5 ; UPS_DAEMON="/sbin/apcupsd" for CONF_FILE in /etc/apcupsd/apcupsd*conf do $UPS_DAEMON -f $CONF_FILE echo $UPS_DAEMON -f $CONF_FILE done

スクリプトの目的を簡単に説明します。

これでapcupsdの設定が完了しました。 要するに、プロセスは次のようになります。デーモンはUPSデバイスをポーリングし、状態が変化したときに適切なスクリプトを呼び出し、ステータスファイルを生成します。 現在、これらすべてをZabbixにバインドする必要があります。



私の意見では、唯一の論理的な方法は、 外部チェックを使用してヘッドにデータを入力することです。



これを行うには、ステータスファイルを解析し、指定されたパラメーターの値を取得するスクリプトを使用します。 私の場合、エージェントとzabbixサーバーの両方が同じコンピューターに配置されます。 エージェントはポート10050でサーバー10051をリッスンします。外部検証ファイルが置かれているディレクトリを見つける必要があります。 これを行うには、サーバー構成ファイル/etc/zabbix/zabbix_server.confで文字列ExternalScripts = / usr / lib / zabbix / externalscriptsを探します。 このディレクトリにチェックファイルを配置します。 bashスクリプトであっても、ファイル名にドットを使用しないでください。 最初に、次のようなスクリプトを作成しました。



apcupsd-bashスクリプト 
 #!/bin/bash if [[ $1 == ippon? ]] && [ -f "/var/log/apcupsd/apcupsd-$1.status" ] ; then case $2 in BCHARGE) cat /var/log/apcupsd/apcupsd-$1.status | grep 'BCHARGE' | awk '{print $3}' ;; OUTPUTV) cat /var/log/apcupsd/apcupsd-$1.status | grep 'OUTPUTV' | awk '{print $3}' ;; LOADPCT) cat /var/log/apcupsd/apcupsd-$1.status | grep 'LOADPCT' | awk '{print $3}' ;; DWAKE) cat /var/log/apcupsd/apcupsd-$1.status | grep 'DWAKE' | awk '{print $3}' ;; STATFLAG) cat /var/log/apcupsd/apcupsd-$1.status | grep 'STATFLAG' | awk '{print $3}' ;; STATUS) cat /var/log/apcupsd/apcupsd-$1.status | grep 'STATUS' | awk '{print $3}' ;; XOFFBATT) cat /var/log/apcupsd/apcupsd-$1.status | grep 'XOFFBATT' | awk '{print $3}' ;; TONBATT) cat /var/log/apcupsd/apcupsd-$1.status | grep 'TONBATT' | awk '{print $3}' ;; TIMELEFT) cat /var/log/apcupsd/apcupsd-$1.status | grep 'TIMELEFT' | awk '{print $3}' ;; esac fi




/etc/zabbix/zabbix_agentd.dディレクトリで、内容を含むuserparameter_apcupsd.confファイルを作成します



UserParameter=apcupsd[*],/usr/lib/zabbix/externalscripts/apcupsd $1 $2







エージェントの再起動：

$sudo service zabbix-server restart







ただし、オフサイトのZabbixには警告があります



外部チェックを乱用しないでください！ Zabbixシステムのパフォーマンスが大幅に低下する可能性があります。





そして、私は管理者であるだけでなく、おしっこでもあるので、 Cの外部チェックを書き直すことにしました。 そして、誰がこのパフォーマンスを達成したかを知っています。

内容を含むapcups.cファイルを作成します。

apcupsd.c 
 #include <stdio.h> #include <string.h> #include <unistd.h> char ValueParam[30]; char *GetValue(char *NameVar, char *FileName) { FILE *fconfig=fopen(FileName,"r"); char fileline[255]; unsigned char FLAGDDOT=0,FLAGSPACEB=0; memset(ValueParam,0,sizeof(ValueParam)); while(!feof(fconfig)) { fgets (fileline, 255, fconfig); if(strncmp(fileline,NameVar,strlen(NameVar))==0) { for(int i=0;i<strlen(fileline);i++) { if(fileline[i]!=':' && FLAGDDOT==0) continue; if(FLAGDDOT==0) { FLAGDDOT=1; continue; } if(fileline[i]==' ' && FLAGSPACEB==0) continue; if(fileline[i]==' ' && FLAGSPACEB==1) break; if(FLAGSPACEB==0) FLAGSPACEB=1; ValueParam[strlen(ValueParam)]=fileline[i]; ValueParam[strlen(ValueParam)+1]=0; } if(strlen(ValueParam)>0) break; } } fclose(fconfig); return ValueParam; } void main(int argc, char* argv[]) { //argv[1] - hostname argv[2] - param char fname[255]; //      "ippon"    2  if((argc==3) && (strncmp(argv[1],"ippon",5)==0) ) { memset(fname,0,sizeof(fname)); strcpy(fname,"/var/log/apcupsd/apcupsd-"); strcat(fname,argv[1]); strcat(fname,".status"); //      // /var/log/apcupsd/apcupsd-ipponX.status if(access( fname, F_OK ) != -1) { puts(GetValue(argv[2],fname)); } } return; }




次のようにコンパイルします。

gcc apcupsd.c -o apcupsd -std=c99





コンパイルしたファイルを/ usr / lib / zabbix / externalscriptsに配置します。



残っているのは、Zabbixのテンプレートをインポートすることだけです。



普遍的なことをするという目標を自分自身に設定するのではなく、逆に、必要な最小化を試みました。

テンプレートに存在します （ パラメーターの説明は上記を参照 ）：



グラフの場合、4つのパラメーターのみを作成しました。



バッテリーの充電、電流負荷、出力電圧、およびバッテリーの残り寿命。



apcupsdデーモンのスクリプト自体がメールに通知を送信するため、トリガーには開始する理由がありません。 インターネットを備えた電話は常にあなたと一緒です。 それだけです、あなたの質問に答えてうれしいです。



さらに数日後、私はあなたと共有する非常に興味深いものを見つけました。 おそらくこれは、何日間も創造的な苦痛からあなたを救うでしょう。



apcupsdプロジェクトに関しては、それは本質的に狭心で、主にAPC UPSの下で研ぎ澄まされています。 これを使用すると、USB経由で接続したときに、USB + COMを組み合わせたデバイスが使用される可能性が高くなります。 私の場合、それは一本勝者3000でした。マイナスのうち、バッテリーの電力と温度は表示されません。 明らかに、これらのパラメーターはAPCにはありません。



NUTプロジェクトに関しては、より普遍的であり、それを使用すると、まれなUPSを拾う可能性が高くなります。 USB + COMを組み合わせた接続で最も重要なマイナスは、COMでのみサポートされます！
