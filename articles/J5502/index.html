アドビは、Vistaとの互換性のためのパッチをリリースしません

アドビのWebサイトで公開された声明によると、ホイストユーザーは、アドビからの新しいソフトウェアバージョンのリリースを待つ必要があります。



同時に、 ソースによると、互換性の問題があります。 たとえば、Adobe Photoshop CS2では各起動時に登録が必要です。 Dreamweaver 8は、ファイルの閲覧で作業しているときにクラッシュする場合があります。



すべてのアドビ製品がこの運命に苦しんでいるわけではありません。 今年の前半には、Adobe Acrobat 8​​、およびそれに応じてReader 8の無料パッチがリリースされ、Photoshop ElementsおよびAdobe Premiere Elementsプラグインのパッチはすでにリリースされています。 新しいバージョンのリリースを待っている製品のみが修正されません。 今年の春と夏に、新しいAdobe Photoshop、InDesign、Dreamweaver、Flash、After Effectsがリリースされます。



アドビとマイクロソフトの間のこの「緊張」は、 XML Paper Specificationドキュメントを作成するためのアドビの組み込みホイストソフトウェアに対する不満が原因である可能性が最も高い。 アドビは、このようなソフトウェアとOS の供給は欧州の独占禁止法に違反していると考えています。



また、アドビシステムズに加えてIBM、RealNetworks、Red Hat、およびSun Microsystemsを含む欧州規格委員会は、MicrosoftのXAML（Extensible Application Markup Language）（XPSはその「サブセット」）がHTMLの直接の競争相手であると考えています。プラットフォーム固有のMSテクノロジーに向けて市場を転換することができます。



XAMLは、Windows Vistaインターフェイスの構築に基づいて、 .Net Framework 3.0ライブラリで積極的に使用されていることを思い出させてください。
