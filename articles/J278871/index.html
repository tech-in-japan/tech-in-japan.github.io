集約では、2つの方法でトランザクションを操作できます-単純または簡単です。



明確な方法は、額のトランザクションを使用することです。 これを行うには、（特に）最後の引数を渡します。 データベースで機能するすべての標準メソッドは、この表記法をサポートしています。 



 class Post extends Record { author = ::new Relation(this, 'created', {direction: -1}).only //    only,      .      ,    . async static createAndAssign(text, user) { const transaction = this.connection.transaction() const post = await new this({text}).save(transaction) await post.author(user, transaction) await transaction.commit() return post } //  ,  -     async static createAndAssign(text, user) { const transaction = this.connection.transaction() try { const post = await new this({text}).save(transaction) await post.author(user, transaction) await transaction.commit() return post } catch (e) { await transaction.rollback() throw e } } }


接続オブジェクト（クラスとクラスインスタンスの両方で使用可能）は、接続、トランザクション、またはサブトランザクションです。 3つのエンティティすべてが同じインターフェイスを提供し、内部のわずかな違いがあるため、人生の使用に違いはありません。 connection.transaction（）を呼び出すと、接続はトランザクションを返します。トランザクションはサブトランザクションであり、サブトランザクションは別のサブトランザクションです。



内部の違いは次のとおりです。接続のコミットおよびロールバックメソッドはエラーをスローします。トランザクションが期待どおりに動作する場合、サブトランザクションの場合-コミットは何もせず、ロールバックは親トランザクションをロールバックします。



これは、たとえばRecord＃save（）のように、一部のメソッドがトランザクションを生成し、最後に終了するために行われます。 このようなメソッドがトランザクションのフレームワーク内で正しく機能するために、サブトランザクションの無限のネストが実装されています。



2番目の方法-シンプル-では、デコレータが使用されます： 



 import {Record, acceptsTransaction} from 'agregate' class Post extends Record { @acceptsTransaction async static create(text) { return await new this({text}).save(this.connection) } }


コードは次のようになります。 



 import {Record, acceptsTransaction} from 'agregate' class Post extends Record { async static create(text, transaction) { //Queryable -  ,    Connection, Transaction, SubTransaction if (transaction instanceof Queryable) this.connection = transaction try { const result = await new this({text}).save(this.connection) if (transaction) await transaction.commit() return result } catch (e) { if (transaction) await transaction.rollback() throw e } } }


上記の例のように、デコレータを直接使用して構成できます。 構成の場合、現在使用可能なフラグは1つだけです-forceは、トランザクションを強制的に作成します-トランザクションが転送されない場合、それを作成します。 次のように使用する必要があります： @acceptsTransaction({force: true}) ...







注意してください-現在、this.connectionはトランザクションになっています。 関数が完了すると、プロパティは以前の状態に戻りますが、トランザクションを渡すことを心配せずにクラスの他のメソッドを呼び出すことができます。 この魔法はこれでのみ機能します（予測可能です）。



トランザクションは順番に処理されるため、つまり あるオブジェクトが完了するまで、別のオブジェクトが開始されるまで、オブジェクトは複製されないため、このデコレータで静的メソッドをラップすると、誤ってトランザクションが失敗する可能性があることに注意してください。 クラスインスタンスの場合、これは、JSを正しく操作する場合、そのスコープ内にあり、他の実行スレッド（promise、asyncなど）から同時にアクセスできないという事実のため、怖いことではありません。オブジェクトにアクセスできません。
