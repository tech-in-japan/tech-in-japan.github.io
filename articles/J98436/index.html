「NetApp？ これは誰ですか？」-投稿のタイトルを読んだ後、ロシアの普通のIT専門家が言うかもしれません。

もちろん、会社とその製品として、NetAppはHPやMicrosoftなどよりも普通のhabraiserにはまだ馴染みがありませんが、それでもHabréに登場する時が来ました。 さらに、彼女は面白いことを伝えるものを持っていると確信しています。 NetAppは常に「コミュニティ指向」の企業であり、ユーザーを愛し、高く評価しています。ロシア、たとえばアメリカの故郷のように、ユーザーフレンドリーになることを心から望みます。 ここでこの手順を試してみましょう。

不思議なことに、1993年以来、すべてのNetAppストレージシステムは、Hitzと彼のスタートアップパートナーによって発明された特別な内部ファイルシステムの概念に基づいており、現在はNetAppの幹部の1人であるJames Lauによって発明されました。 WAFL （Write Anywhere File Layout）と呼ばれるこの「ファイルシステム」には、これまでのところ、市場に固有のすべてのNetAppストレージシステムの機能がベースになっています（これは、適切に設計されたアーキテクチャの意味です！）そして、過去17年間で概念的には、時代遅れではなく、逆に、今日使用されている重要なアイデアが今日転送され、たとえばZFSなどの最も高度なファイルシステムに実装されています。
