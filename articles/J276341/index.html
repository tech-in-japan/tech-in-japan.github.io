この製品の詳細をお伝えします。



Microsoft Azure Stack（MAS）は、ハイブリッドクラウドプラットフォームです。 データセンターのリソースを使用したり、MASをMicrosoft Azure（以下、Azureと呼びます）のサービスやMicrosoftパートナーの他のサービスプロバイダーと統合したり、パブリッククラウドの「無限の」リソースにアクセスしたり、データセンターのリソースを利用したりできます。



仮想マシン（以降、VMと呼びます）、仮想ネットワーク、ブロブ/テーブルストレージなどのサービスを使用できます。 使い慣れた（ほぼ）Azureインターフェイスを使用して、エンドユーザーにサービスを提供します。



Azure用に作成されたアプリケーションは、MASでも機能します。 AzureプラットフォームおよびMASの開発には、Azure SDKが使用されます。 Azure Application Templates、Role Based Access Control、Azure Active Directory、Azure APIにアクセスできます。 さらに、Linux、Java、Node.js、PHPなどのオープンソースソリューションを使用できます。



インターネット上には十分な水があると思います。この製品のマーケティング資料はたくさんあります。



MAS展開アーキテクチャTP1。



1. ADVM -Microsoft Azure StackのActive Directory、DNS、およびDHCPの役割を持つVM。

2. ACSVM -Azure Consistent Storageサービスの役割を持つVM。 これらのサービスはService Fabricを使用します。

3. MuxVM -Microsoftソフトウェアロードバランサーおよびネットワーク多重化サービスの役割を持つVM。

4. NCVM -Microsoftソフトウェア拒否ネットワーキングの主要コンポーネントであるMicrosoftネットワークコントローラーコンポーネントの役割を持つVM。 このサービスはService Fabricを使用します。

5. NATVM -Microsoftネットワークアドレス変換の役割を持つVM。 Azure Stackのインターネットコンポーネントを提供できます。

6. xRPVM -Microsoft Azure Stackの主要な役割（ コンピューター 、ネットワーク、およびストレージプロバイダー）を持つVM。

7. SQLVM -ACSおよびxRPサービスで使用されるSQLの役割を持つVM。

8. PortalVM-コントロールプレーン（Azure Resource Manager）、Azureポータルサービス、および他のポータルの役割を持つVM 。

9. ClientVM-開発用のVM（PowerShell、Visual Studioなど）。

サービス、サブスクリプション、プラン、オファー。



1. サービス -ここでは、SQL、Sharepoint、VMなど、すべてが明確になっています。

2. プラン -サービスのグループ。 プランを作成し、テナントがプランをサブスクライブし、プランにあるサービスを使用します。 サービス（VM、RAM、コアなどの数）にクォータを設定できます。 クォータは、テナントの地域によって異なる場合があります。 基本プランと追加プランがあり、基本プランは常にオファーに含まれています。追加プランは自由に購入できます。 たとえば、追加のプランには基本プランと同じプランが含まれる場合がありますが、割り当てが異なります。

3. 提案 -計画のグループ。 テナントが「購入」オファー。

4. サブスクリプション -テナントオファーの購入時に作成されます。 サブスクリプションに含めることができるプランは1つだけです。 サブスクリプションは、プラン/サービスへのアクセスを定義します。



Azureリソースマネージャー

リソースを論理グループに結合することでリソースを操作する方法。たとえば、仮想マシンは、ネットワーク\データ\ VMなどの複数のリソースで構成されます。 リソース管理の簡素化、リソースライフサイクル管理の簡素化、リソースへの権利の分配、作成など。 リソースグループ（以降RG）によって使用されます。



Azure Resource Managerの主な利点：

1. RGのすべてのリソースを個別にではなくグループとして管理できます。

2.テンプレートからリソースを1回、2回、10回作成しても同じ結果になります。

3.リソーステンプレートで依存関係を指定して、それらを正しく作成できます。

4. RGに権利を発行できます。権利はRGのすべてのリソースに適用されます。

5.タグを使用して、リソースを論理的に整理できます。

6.特定のRGまたはタグの請求書の詳細を表示できます。



リソースグループ

リソースをグループ化するための論理コンテナー。 すべての展開リソースとリソースの一部のみの両方を含めることができます。

1. 1つのRGのすべてのリソースは、単一のエンティティとして管理するため、同じライフサイクルを持つ必要があります。

2.各リソースは1つのRGにのみ存在できます。

3.リソースは、いつでもRGに追加または削除できます。

4.さまざまな地域のリソースが含まれる場合があります。

5.役割の割り当てに使用されます。

6.リソースは、別のRGのリソース（たとえば、あるRGのWebサーバー、別のRGのデータベース）に関連付けることができます。



Azure Resource Managerテンプレート

アプリケーション/サービスを展開するためのテンプレート。 JSON形式で作成されました。 宣言的なアプローチを使用し、リソースグループの展開、依存関係の追跡、パラメータと変数のサポートを可能にします。 テンプレートは、Azureおよび/またはMASでの展開に使用できます。 したがって、データセンターでテンプレートを作成してテストすることで、問題なくAzureに転送できます（逆の場合も同様です）。



これはすべて、Microsoft Azureと完全に一致しています。 これは、Microsoft Azure StackとMicrosoft Azureの両方に当てはまります。



主要プロバイダー：

1. 計算-VMの作成に必要です。 VM拡張機能のインストールを許可して、VM管理を容易にします

2. ストレージ -BLOB、テーブル、およびアカウントサービスを作成できます。 ブロブは情報を保存するために必要であり、ブロブはブロック、ページ、および追加です。1つ目はファイルとバックアップに使用され、2つ目はvhdに、3つ目はログに使用されます。 テーブルサービスはMicrosoftのNoSQL実装であり、このソリューションにはスキーマがありません。 各レコードを識別するには、パーティションキーと行キーを使用します。これにより、テーブル内のレコードを一意に識別できます。

3. ネットワーク -ソフトウェア定義のネットワーキングおよびネットワーク機能仮想化サービスを提供します。 Microsoftが約束したように、このプロバイダーはAzureから移行しました。 ロードバランサー、パブリックIPアドレス、ネットワークセキュリティグループなどを作成できます。
