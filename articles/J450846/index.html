まず、19H2の夏のビルドで、Windows 10は、独自のサブシステム「Linux for Windows」（WSL-Windows Subsystem Linux）の2018年10月22日のバージョン4.19に基づいた本格的なLinuxカーネルを提供します。



2つ目：将来のエンタープライズビルドで、安楽死させたMicrosoft EdgeのChromium生まれ変わりで、IEのコアが統合されます。

Windows 10上のLinuxカーネル



Windows 10に本格的なLinuxカーネルを追加することは、Microsoftのコミュニティとの系統的な作業を継続する論理的なステップです。 以前は、LinuxカーネルのエミュレーションのみがWindowsで利用可能でした。 システム内のLinuxカーネルと並行して、同社はWindows Terminalのリリースを発表しました。これは、PowerShell、Cmd、そして実際にはWSLパッケージ内のLinuxカーネルへの集中アクセスのための新しいWindowsアプリケーションです。



会社の代表者は、4.19はサブシステムのカーネルの最新バージョンではないと主張しています。 将来、WSLは更新され、Linuxカーネルの安定バージョンに対応する予定です。これも重要です。 さらに、システムにフルカーネルを導入すると、エミュレーションの必要性がなくなり、Windowsで作業する開発者の生産性と快適性が向上します。 さらに、WSLはオープンソースコードに基づいています。つまり、開発者はサブシステムに必要な変更を加えて、カーネルをフォークすることができます。



これは、開発コミュニティへの最初の主要なステップではありません。 以前、同社はBashをシステムに統合しましたが 、これは非常に保守的なMicrosoftにとって非常に深刻なステップと言えます。



Microsoftの行動は驚くべきものですが、もはや衝撃的ではありません。SatyaNadellaの管理下で、同社は実際にオープンソースコミュニティに参入し、過去3〜4年間のMicrosoftの行動から明らかなように、開発者と企業セグメントとの直接的な連携にますます注力しています。



Microsoft'aのモンスター、またはChromium EdgeコアInternet Explorerの理由



ぞっとする人は誰でもInternet Explorerを思い出し、古いレイアウトデザイナーも同時に泣きます。 MicrosoftがSpartanとIEの放棄を発表したとき、世界中でシャンパンが開かれ始めましたが、現実は私たちが思っていたより悪いことが判明しました。 古いバージョンのIEでのみ動作するように作成されたソリューション、ポータル、およびサイトがまだあります。 ほとんどの場合、これらのソリューションは閉じたエンタープライズソリューションに存在します。 むかしむかし、マイクロソフトは、世界中の企業がIEを完全に放棄したために内部製品とソリューションを更新することを強制するか、「間違った方向に」戦い始めることを選択しました。 IEの完全な拒否はMicrosoftの数十億の損失と評判の損害で脅かされたため、巨人はブラウザの互換性をInternet Explorerで動作するように適合されたページと年々ドラッグする必要がありました。



EdgeHTMLエンジンを大々的に放棄し、Windows 10の「デフォルトブラウザ」の開発をChromiumエンジンに移行した後でも、MicrosoftはInternet Explorerとしての恐ろしい遺産から逃れることはできません。



そのため、特にWindows 10のエンタープライズビルドの場合、同社は「驚き」を備えたバージョンのChromium Edgeを出荷します。これはEdge自体に組み込まれたIEエンジンです。 このような奇妙なソリューションは、新しいブラウザと、かつてInternet Explorerに無作法に釘付けにされていた古い企業ソリューションとの互換性を保証します。



新しいChromium EdgeでInternet Explorerに適合したページを開くと、ブラウザーは自動的に「IE互換モード」をアクティブにし、Internet Explorer 11のそれに対応するディスプレイを接続します。ページの横にある特別なIEアイコンで互換モードで起動されているかどうかを確認できますブラウザページのアドレスバーで。







マイクロソフト自体は、古いWebポータルを使用している企業は、新しいChromium Edgeに切り替えて、IE +他のブラウザの使用を完全に放棄すべきだと主張しています。 もちろん、技術専門家なら誰でも、古いエンジンを新製品に組み込むこのようなステップは冗長であると言うでしょうが、実際、Microsoftは現在、顔を救おうとしており、少し慈善活動を行っています。 問題は、巨人がビジネスにノーと言うことができ、最終的にロバを撃つことができるかどうかです。 まだ解決していません。
