RapidMinerの概要

現時点では、分析システムを必要とする多くの企業がありますが、ほとんどの場合、このソフトウェアの高コストと過度の複雑さにより、単純で有名なExcelを優先して独自の分析システムを構築するという考えを捨てざるを得ません。 また、従業員のトレーニング、高価なストレージシステムの保守などの追加費用 そして、ここでオープンソースのソリューションが助けになります-それらの多くはありませんが、非常に価値のあるソフトウェアがあり、その1つはRapidMinerです。 RapidMiner（以降、単に「鉱夫」と呼びます）は、データマイニング用に作成されたツールで、鉱夫（アナリスト）は仕事中にプログラミングを行うべきではないという主な考えを持っています。 同時に、ご存知のように、マイニングにはデータが必要であるため、さまざまなソース（データベース、ファイルなど）から情報を取得および処理するための幅広いタスクを解決するかなり優れた演算子セットが提供されていました。 ETLの完全なツール。



マイナー自体に加えて、マイナープロセス（スケジュールを含む）を保存および実行するためのリポジトリとして使用できるRapidMinerサーバー（以前のバージョン6までのRapidAnalytics）、ユーザー間のデータソースへの接続の「手探り」、 Webサービスとしてのマイナープロセス。



残念なことに、バージョン6では、マイナーの作成者は、このソフトウェアの販売で収益を得ることを決定し、ライセンスをAGPLからBusiness Sourceに変更しました 。 それにもかかわらず、AGPLのバージョン5および私たちは自由に制限なしで使用できます。 したがって、記事で検討されます。 また、6番目のバージョンでは、多くの新しい演算子と関数はなく（おそらく最も興味深いのはクラウドサポートです）、ほとんどのタスクにはRapidMiner 5コミュニティで十分です。



設置



少し前まで、RapidMiner 5をダウンロードするためのリンクが公式サイトから削除されていたため、githubの公式プロジェクトで取得したソースコードからRMを収集します。



リポジトリからRapidMinerを構築するには、次のものが必要です。

  • インストール済みのJavaおよびJDK
  • アパッチアリ
  • Gitクライアント


コンソールに行き、マイナーを置きたいディレクトリに行き、リポジトリをクローンしましょう 

git clone https://github.com/rapidminer/rapidminer-5.git


次のステップでは、プロジェクトを収集します 

 ant build ant release.makePlatformIndependent


今マイナーを実行します 

 .\scripts\RapidMinerGUI.bat


それぞれLinux用 
 ./scripts/RapidMinerGUI.sh


右の写真のように、ウィンドウが開きます。 [新しいプロセス]をクリックして先に進みます。



基本的な概念



サンプルを使用してRapidMinerを操作する基本原則を見る前に、その基本概念を少し紹介します。



プロセス



データ分析/処理に必要なタスクを実行するために、所定の順序で相互接続されたオペレーターのセット。







オペレーター



プロセスの論理ユニット。 オペレーターはデータに対していくつかのアクションを実行し、入出力（いわゆる「ポート」）を持ち、データは入力に送られ、オペレーターによって処理されたデータは出力に送られます。 したがって、データ処理チェーンを実行できます。たとえば、データベースからの顧客トランザクションを検討し、最大のものを見つけ、ドルに変換して結果を出すことができます。 同時に、チェーンを並列化することができます-たとえば、あるデータベースではさまざまなデータベースからトランザクションを読み取り、別のデータベースでは顧客データを検索し、結合して結果を取得します（それらを同時に並行して実行することも可能です！）



プログラムインターフェイスでは、[演算子]タブは[演算子]タブに対応します。階層では、機能タブはグループ化されています。 演算子を使用するには、演算子をクリックしてプロセスのワークスペースに転送する必要があります。



リポジトリ



RMプロセス用のストレージスペース。 ローカルでもリモート（RapidMinerサーバー）でもかまいません。サーバー側でプロセスを実行したり、データベースプロセス/接続にマルチユーザーアクセスしたり、スケジュールに従ってプロセスを実行したり、Webサービスとしてデータを送信したりできます。



RMへのリポジトリコントリビューションでは、サンプル、DB、およびローカルリポジトリのみがここに表示されます。 最初に、名前から既に明らかなように、一連のプロセス-例、DB-マイナーで使用可能な現在のデータベース接続（ツール->データベース接続の管理で定義）およびローカルリポジトリ、コンピューターに独自のプロセスを保存する場所です。



プロセスコンテキスト



コンテキストは、次の3つのセクションを表示できる[コンテキスト]タブに対応しています。



  • プロセス入力 - プロセスの入力に送信されるデータ。 ここで、リポジトリ内のデータへのパスを指定できます。
  • プロセス出力 -リポジトリ内のパスが示され、プロセスの結果が保存されます。
  • マクロは、プロセス内のどこからでも使用できるグローバル変数です。 値として文字列または数字のみを使用できます。


プロセス入力とプロセス出力は 、碑文inpとresを使用して、プロセスの境界に沿ったプロセス内の円で示されることに注意してください。 入力からのデータを使用または保存するには、対応する円を演算子の入力/出力に接続する必要があります。
