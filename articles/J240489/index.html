実際には、4の倍数である任意のボリュームのノートブックに特定のシーケンスで個別のページをレイアウトするというアイデアでした。注意深く見ると、1つのノートブックシートには3つまたは5つではなく正確に4つのページが含まれており、ページはシート上で特定の順序で配置されています、私は後でそれを終えましたが、今のところそれはそのままでした、すなわち（48ページのノートブック用）：



ページペア 中毒
1-48 m =（1）n =（48）
2-47 m =（1 + 1）n =（48-1）
3-46 m =（1 + 2）n =（48-2）
... ...
24-25 m =（1 + 23）n =（48-23）


これは、まず、本をiページに分割し、次に各kページのj個のノートブックに分割し、次に各ノートブックをシートに配置できるページのペアに分割して、単一のノートブックファイルにまとめることができることを意味します。 PDFニックネーム。 かろうじて明確な...



一般的に、依存関係は単純で、シートの最初のページはa + b、最後はkbです。aはノートブックのシリアル番号で、0 <a <j、bはペアページのシリアル番号、0 <b <k / 2。 つまり これはforループ（（b = 0; b <k / 2; b ++））で、これらすべてのページを処理する必要があります。



このノートブックをforノートブック（（a = 0; a <j; a ++））を介して実行する別のサイクルに詰め込みます 。

問題



私はこのソリューションが本当に好きではありません： 



  gs -q -sDEVICE = bmpmono -r2x2 -sOutputFile = $ TMP_DIR /％d.bmp -dNOPAUSE -dBATCH * pdf＃PDFからページを低解像度で抽出
 FILE_COUNT = `ls -l $ TMP_DIR / * bmp |  grep ^-|  wc -l`＃ディレクトリ内のファイル数をカウントすることにより、ドキュメント内のページ数をカウントします




klirichekに感謝します。現在は次のようになっています。 
  FILE_COUNT = `pdfinfo * pdf |  awk '/ Pages / {print $ 2}' '




したがって、PDFニックネームのページ数を展開せずに調べる方法についてのヒントに感謝します。
