CSEDaysに参加する理由や詳細を確認するにはどうすればよいですか

すべての人に良い一日を！



多くの皆さんと同様に、私はコンピューターサイエンスのプログラミングと研究を行っています。 そして一年ほど前、普通の講義で大学で得た知識では不十分であることに気づきました。 また、スマートブックを読むだけでは、特定のトピックの全体像が必ずしも得られるとは限りません。 問題とその解決方法をより具体的に説明しようとします。



大学では、OOPに関する講義で、テストの基本について学びました。 JavaでJUnitを使用してみました。 しかし、ほとんどの場合、プログラム自体ではなく、テストでエラーを修正しました。 さらに、頭の中の問題について共通の理解はないと感じられました-結局のところ、テストはJUnitテストに限定されません。 この分野の専門家の講義を聞くか、単に知り合いになって専門家や開発者と話をする必要がありました。 しかし、これをすべて実装する方法は？



この問題の論理的な解決策は、特別なイベントに参加することでした。 おそらくいくつかの学校に出席したり、会議に出席したりします。 しかし、どれですか？ シニア仲間と相談した後、 CSEDays学校を選択することにしました。 これにはいくつかの理由がありました。 まず、2010年秋のテーマである「ソフトウェアの信頼性」は、私のニーズに完全に合致していました。 第二に、講師は非常に印象的でした-科学の候補者とソフトウェア開発の分野の専門家。 また、SKB Kontur、Yandex、Microsoft Researchなどのロシアの大企業やセンターが学校をサポートしていることも魅力的でした。 だから、学校は注目に値する。 第三に、これらはすべてエカテリンブルク市の近くで新鮮な空気の中で起こった！



イベントの将来の参加者には、テストプロジェクトDOM API Testingが提供されました。 したがって、あなたは手を試して、イベント自体で結果を議論することができます。



イベントは2010年11月に開催されました！ 驚くべきことに、この学校は、トピックに関する質の高い講義と、国内のさまざまな地域の業界の専門家の両方を組み合わせることができました。 ソフトウェアの信頼性について多くのことを学びました-テストの静的メソッド、Yandexの負荷テスト、MSSQLのテストなど。 SKB KonturのLeo Plinerが組織した高負荷の情報システムのリスクに焦点を当てた「ラウンドテーブル」を覚えています。ブレーンストーミングと思考帽子の使用に対する興味深いアプローチです。 しかし、最も重要なことは、学校全体で居心地の良い雰囲気が作られ、人々がコミュニケーションできるようになったことです。 1つのことに従事しているさまざまな大学（さらにはさまざまな都市）の学生と教師は、非公式の環境でコミュニケーションを取り、その都市または大学でのコンピューターサイエンスイベントについて話す機会がありました。 非公式の雰囲気を維持するために、さまざまなコンテストやゲームが開催されました（ボウリング、ChKG、マフィアなど）。 CSEDaysは、テストの分野で新しい知識を与えてくれただけでなく、コンピューターサイエンスの他の多くの分野にも興味を持ちました。



学校に与えられたDOM APIテストプロジェクトが学校自体にほとんど触れられていないことは少しがっかりでしたが、このトピックは開発できました。



次のCSEDays学校は2011年春に開催されました。また、市内で開催されたという事実にもかかわらず、その組織のレベルは非常に高かったです。 都市の「永遠のbus騒」と新鮮な空気の欠如に加えて、目に見えるマイナスはありませんでした。 春学期のテーマは「情報セキュリティと暗号化」です。 繰り返しになりますが、講師とそのレポートは最高レベルでした。 この学校は、RuCTF 2011ファイナルに合わせて開催され、RuCTFファイナリストは自動的に学校のメンバーになりましたが、他の参加者はCTF運動に精通する機会がありました。



このようなイベントでは、さまざまな企業や大学の多くの専門家と自由にコミュニケーションをとれる可能性があるため、世界的な知識交換が行われます。 ITコミュニティがどのように団結しているかを見ることができます。これは喜ばしいことです。



なぜ私はこれすべてですか？ あなたがITの学生、若い開発者、または教師であり、さらに成長するための新しい知識がどこで得られるかまだわからない場合、この学校はあなたにぴったりです！ 個人的に、このような学校を2つ訪問したので、今年の11月に予定されている3番目のテーマである「コンピューターネットワークとマルチエージェントシステム」を宣言したいです。 テストタスクは既に投稿されているので、先に進みます。 結局のところ、完了したタスクは、助成金を取得し、無料で学校に参加する可能性を高めます！ 新しい顔を見て、IT専門家と新しい知り合いになることができてとても嬉しいです！
