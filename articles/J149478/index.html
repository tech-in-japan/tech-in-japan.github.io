MODX Cloud：最初のレビュー

かなり意外なことに、彼らはベータ版のMODX Cloudへの招待状を発行しました。 これは、前例のない機能を備えたMODX Revolutionの広く宣伝されている開発です。 しかし、実際には、クラウド上の多数のサイト用の単一のコントロールパネルを備えた高度な共有ホスティングです。



将来的には、これにより分散するのではなく、サイトを集中的に更新し、バックアップして、クライアントとの作業を楽しむことができるようになります。



クラウドはクローズドベータの状態ですが、私は何とか見てみました-そして、ここにレビューがあります（投稿にはクリック可能な写真がたくさんあります）。



制御盤



クラウドコントロールパネルは、 dashboard.modxcloud.comにあります。



ログインウィンドウは、 MODX Revolutionを非常に思い出させますが、これは偶然ではありません。 クラウドはRevoのみで機能しますが、 Evolutionサポートはそうではありません 。



MODXが採用されているように、コントロールパネルは非常にきれいに見えます。 ExtJS 4.0.7で記述されているため、「遅い」ExtJSからの拒否はありません。 すべてがスマートかつスムーズに機能します。



メインメニューは3つの項目で構成されています。

ホーム 雲 金庫




ホームはメインセクションです。 ログイン後にアクセスできます。 最近のイベント、選択したクラウド、ニュースのログが含まれています。



クラウド -クラウドを管理するためのセクション。 このシステムの用語では、クラウドは1つのサイトです。 すぐに彼についてtest1.bezumkinru.modxcloud.comをすぐに作成しました。



Vault-ストレージ。 とても興味深いことです。 すべてのクラウドのすべてのスナップショットとバックアップを以下に示します。 2つの違いは、スナップショットが新しいクラウドの作成に使用できるイメージを取得することです。 また、バックアップはリカバリ専用です。



ユーザープロフィール、通知、クレジットカード情報もあります。





クラウド





したがって、1つのクラウドはMODX Revolutionの1つのインストールです。 超高速インストールの後、いくつかのプロパティを含むクラウドが表示されます。

  • c0067.paas3.tx.modxcloud.comなどのクラウド名のサービスが提供されます 。
  • 通常名、 test1.bezumkinru.modxcloud.comと入力します 。 まだ名前を割り当てることはできません-クラウドはまだベータ版です。
  • サーバーへのSSH / SFTPアクセス。
  • スナップショットとバックアップ。 時刻表に気づかなかった。
  • モニタリング また、まだ初期段階です。 グラフは明らかに美しさのためのものです-単位も時間もありません。
  • MODXインストールアップグレード


ご覧のとおり、今すぐサイトの快適な作業を整理するのに十分な機会があります。 クラウドは他のアカウントに転送することさえできます！ 。





サーバー



サーバーには、SSH / SFTP経由のアクセスが許可されます。 FTPはありません。Apache2にも気付きませんでした-Nginx 1.2.2は、おそらくphp5-fpmと連動して動作します。 一般的に、私は保守派を喜ばせるものは何もありません、すべてがこのノートの精神に見えます。

サーバー自体は、 Debian Squeeze上で回転しているようです。uname -aは次のように生成します。 

/paas/c0067/home $ uname -a Linux paas3.tx.modxcloud.com 2.6.32-5-amd64 #1 SMP Sun May 6 04:00:17 UTC 2012 x86_64 GNU/Linux


しっかりと閉じて、見るものは何もありません。 ユーザーのルートは彼のディレクトリであり、上記のアクセス権はありません。 真夜中の司令官もいません-ごめんなさい。



完全な不在を支持して、Apache2は次のように述べています。 

 /paas/c0067/home $ cat ./nginx_rewrites.conf location / { try_files $uri $uri/ @modx-rewrite; }/


つまり、書き換えルールはNginx用であり、.htaccessは必要ありません。 プラグインではなく、Webサーバー経由でマスターをリダイレクトしてください！



サイト



絶対に通常のMODX Revolution 2.2.4、機能なし。 何よりも、速度に興味があったので、いくつかのパッケージ、典型的なレイアウトをインストールし、 loadimpact.comを運転しました







サイトが空であること、負荷や複雑なスニペットがないことは明らかです。 しかし、これは良いことです-プラットフォーム自体が遅くなることはありません-そして、それはプログラマ次第です。



管理パネルでは、すべてがスマートであり、ブレーキはありません。



おわりに



これは一見です。 2番目の、より注意深い方法では、完全に動作するプロジェクトを調達し、バックアップ、発掘、負荷などを加えて、1か月または2か月間そこにそれを運転する必要があります。 クラウドが稼働状態で開かれた後になると思います-すぐに移動するサイトがいくつかあります。



プラットフォームの可能性は膨大です。 CMS / CMFがホスティングプラットフォームに成長したことを思い出すことさえできません。 彼はプロジェクトの主力者であるため、誰も革命を投げるつもりはありません。



ただし、実際には、MODX Cloudは派手な共有ホスティングです。 リソースではなく、ロジックによって。 専用サーバーやVPS / VDSを置き換えるものではありません。再起動したり、サーバーに独自のソフトウェアを配置したり（とにかく）、Nginxを微調整したりすることはできません。

しかし、名刺サイト、ショップ、ブログ、個人ページを立ち上げるのは簡単で簡単です！



このソリューションは、スピードと利便性を求めて曲がったホスティングをぶらぶらしている膨大な数の未経験のMODX Revolutionユーザーの頭痛を和らげると確信しています。
