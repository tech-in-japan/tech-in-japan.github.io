ベストホスティングノート：パート1

つい最近まで、私のWebプロジェクトはすべて独自のサーバーに実装されていました。 そして、このため、私はホスティングに対処する必要がありませんでした。 しかし、すべてが終わり、2008年の終わりに、私は突然ホスティングといくつかのドメインを購入しなければならないと決めました。



この考えは突然来たわけではありませんでしたが、数ヶ月間吸い込まれました。 最適なオプションを推定するために、考えられるすべてのオプションを検討しようとしました。 さらに、私は自分で処方を試みましたが、なぜホスティングが必要なのですか。 最終的には、選択が行われ、ホスティングが先年支払われ、ドメインが購入されました。



このメモを使用して、ホスティングエクスペリエンスに関する小さなシリーズを（終了せずに）開きたいと思います。 最初のメモには、ホスティング、そのような選択が行われた理由、選択したホストが私に与えたものについての抽象的な考えが含まれます。

ホスティングへの情熱



ホスティングを購入する最初の衝動は、まだ動作中のサーバーに基づいているHabrRedactorのオープンに関連して現れました（手に届くとすぐに転送されます）。 一般的な知恵によれば、一時的なものほど永続的なものはないため、これらの衝動はすぐに消えました。 Habra Editorが起動し、機能し、更新され、機能し続け、1日最大120人が訪れ、これで十分です。 ホスティングは、どういうわけか興味のある分野から外れました。



個人的なホスティングに向けた2番目のより深刻なプッシュは、 habradigestプロジェクトの作業中に発生しました。 一時的な解決策が既にあり、その場で素晴らしい友人のおかげで、プロジェクトはホスティングとそのドメインの両方を受け取りました。 プロジェクトは開発中でしたが、ほとんどはコンテンツの面でした。 サイトは元の状態のままでした。 そして、通常のサイトを作るときが来たとき、ホスティングのアイデアが私の興味のある分野の表面に再び浮上しました。 なんで？



問題は、現在のホスティングhabradigest.ruがphpのみをサポートしていることです。 そして、私は、ここにいる一部の人が知っているように、Microsoft .NETプラットフォームのプログラマーです（PHPの愛好家も最後まで読むことに興味があります）。 したがって、私は最終的に成熟し、11月の終わり頃に、ホスティングの不明確なニーズが大きな欲求で形になりました。 残っているのは、ホストを見つけることだけです。



このような異なるホスティング



それで、私はホスティングから何を得たかったのですか？ 以下に、検索の指針となったニーズを示します。

  • MS SQL Server 2008の可用性。
  • IIS7;
  • .NET Framework 3.5;
  • ファイルおよびデータベース用の最大または無制限のスペース。
  • 無制限のトラフィック;
  • 一般に、可能な限り少ない制限。
  • 外国人のホスト。


最後の要件について以下に説明しますが、今のところは、検索結果に精通することを提案します。

  • www.discountasp.net -1ギガバイトのスペース、1か月あたり80ギガバイトのトラフィックを提供します。 SQL 2008など、別途購入する必要がある追加機能を備えた非常に泥だらけのスキームは、個別の機能として提供されます。 glyいサイト。 一般的に、印象は悪かった。
  • www.bizhostnet.com -20 MBのデータベースのような、 おかしなお金のためにいくつかの軽薄な数字を提供します。 すぐに流されました。
  • www.webstrikesolutions.com-ギガバイトのスペースと20ギガバイトのトラフィック。 サイトはデータベースとIISのバージョンに関する詳細を見つけることができませんでしたが、20ギガでは不十分であることが明らかになりました。
  • serverintellect.com-ワイルド価格;
  • 3essentials.com -2005 SQL Server、すぐに適合しませんでした。 さらに、提供される価格と機能に関する他の多くのコメントがあります。


すべてが悪いように思えますが、インターネット上には別のホスティングサービスがあり、それは単なる夢の実現でした。 彼の名前はMochaHostです。



奇妙な名前の夢のホスト



それでは、MochaHostは何を提供しますか？ 誰もが計画を立ててページで彼に行き、以下を確認することをお勧めします。

  • 無制限の場所;
  • 無制限のトラフィック;
  • データベースの数に制限はありません。
  • FTPアカウントとメールの数に制限はありません。
  • 5000サイト;
  • MSSQL Server 2008;
  • IIS7;
  • .NET 3.5
  • Plesk 9（これも良い）;
  • ホスティングを1年間購入すると、生涯ドメインが付与されます。
  • 定期的な割引とクーポンシステム。
  • 低コスト。


彼らが言うように、コメントは不要です。 PHPユーザーは、Linuxホスティングが悪くないことも確認できます。 ロシアの耳MochaHostの名前が不愉快なホスティング事業者は、1本の小さな指ですべての競合相手を作ります。



ここで、友人との議論の1つで、これらの無制限のすべてが通常のマーケティングであり、それ以上ではないという視点があったと言わなければなりません。 はい、そうです。 共有ホスティングで5000サイトをホストしたり、テラバイトでトラフィックを駆動したりすることはできません。 しかし、教えてください、あなたの多くはGmailを少なくとも30％使用していますか？ 私は2％しか使用しません。 また、7ギガバイトのフレームワークに自信を持っているからこそ、Gmailを使い続けます。 データベースの20メガバイトとは何ですか？ すべてのリクエストのロギングを開始した場合、データベースがサイズによって不条理なフレームワークで停止せず、20ギガバイトに静かに成長することを確認したいと思います。 トラフィックに制限がないことを確認したいと考えています。また、たとえ到達が困難であっても、制限についても考えたくはありません。 ちなみに、そうではありません。3週間もたたないうちに、「空の」ドメインで5つのギグのトラフィックに追いついたからです。 私は制限について何も知りたくありません、そして無制限のポリシーは私を幸せにします。 無制限の長生き！



なぜロシアではないのですか？



はい、ロシアのホスティングに留まらなかった理由を伝えると約束しました。 これには多くの理由がありますが、いくつかに焦点を当てます。 最初に、たとえばr01.ruなど、ホスティング事業者が提供するものを見てください。 24,000ルーブル（！）の場合、2000rで海外に受け取ったよりも少ない金額で提供されます。 比較なし。



第二に、国内のホスティング事業者の技術面と速度については絶対にわかりません。 国の居住者であり、システム管理者やビジネスマンを含む私の国の市民とのコミュニケーションの経験がありますが、ハイテクビジネスの進歩に追われる者はいないと確信しています。 したがって、.NET 4.0がサーバーに届くと、いつでもそれを受け取りません。 ここで重要なのは、ホスティング事業者が悪いということではなく、単にasp.netホスティングがわが国で不十分に分散されていることであり、誰もそれにあまり注意を払っていません（asp.netプロジェクトを当社のサーバーでホストするのが習慣であるという意見があります）。 まあ、価格については、私は再び何も言わないでしょう。



第三：サポート。 もちろんすみませんが、ホスティング会社のサポートについて聞いたばかりではありませんが、私は絶え間ない失礼、電話のピンポン、あなたが行き来するとき、無責任、そしてさまざまな会社のサポート部門と通信するときの無礼に遭遇します。 具体例を次に示します。「ブレイク」のある有名なストーリー。 はい、クライアントは非常識です。直接「フレーム」と言いましょう。 しかし、だれかが個人的な会話を出し、一般的な考慮を支持しなければならないのはどのような権利でしたか？ すべてがそのように賢いですか？ 売り手に誓うことはありませんか？ 次のそのような「スマート」サポートワーカーに、bashorgでの彼との会話のログを投稿してほしくありません。 ホスティングについてはうんざりすることはありません。私は普通のユーザーとしてだまされますが、インターネット全体が私を笑わせず、自分とホストのみに関係することを確認したいと思います。



失礼、過度の自尊心、誇り、人々への無礼はすべて、私たちのサービス部門に関するものです。 これがすべてなくなると、私は最初にホスティング業者に行きますが、今のところ、残念ながら外国のホスティングを選択しました（そして、サポートに失望していません！詳細については後で）。
