ubuntuの不具合と欠点

こんにちは。 実際、タイトルはすでに以下で説明する内容を説明しています。 しかし、私はまだいくつかの単語を追加したいと思います。 まず、特定の数のエラーがubuntu自体の不具合ではなく、GNOMの欠陥、または個々のアプリケーションであることを理解しています。 しかし、今日は、プログラムがどの言語で書かれているか、システムコンポーネントがどのように相互作用するかを気にしない単純なユーザーの代わりに書くことにしました。 第二に、私は専用のubuntuユーザーです。 実際、私には3台のコンピューターがあり、誰もがこのLinuxを持っています。 したがって、私は自分の好きなソフトウェアの欠点について書く権利があると信じています。 ここから始めます。



1.キーボードレイアウト。 バグは修正されました。



この不具合は、3台のうち1台のマシンで発生しました。 実際、gnomeのキーボード設定は保存されないため、コンピューターの電源を入れるたびにロシア語のレイアウトを再インストールする必要があります。

xorg.confにスイッチ設定を直接入力することで処理されます。 しかし、その後、キーボードレイアウトインジケーターは現在の言語の表示を完全に拒否します。



更新した

Habrauser maniakによると、ディストリビューションを更新すると、このバグはすでに修正されています。 キーボードのgnomeの設定（〜/ .gconf / desktop / gnome /周辺機器/キーボード）を削除し、xorg.confでシャーマニズムを削除するだけです（作成した場合）。



2.ノーチラスのパスのグリッチ。



非常に興味深い問題。 説明するのも少し難しい。 コンピューターにはいくつかのフォルダーがあり、そこには数百のファイルがあり、サブフォルダーもあります。 したがって、サブフォルダーに移動すると、現在のディレクトリのすべてのファイルが表示され、[戻る]をクリックすると、このような素晴らしい結果が表示されます。





その後、オウムガイは一般的にどこにも行きたがらず、働くことを拒否します。



3.マウントされたデバイスが使用できない場合、gnomeが固くハングします



私にとって個人的に長い間知られているグリッチはグリッチであり、ひどく迷惑です。 ボールが利用できない場合、たとえばサーバーからマウントされた場合、gnomeは単純に動作を停止します。 そして、もちろん、誰もそこにあるファイルを開きません。 たとえば、テキストエディタで作業したり、まったく別のデバイスでドキュメントを開きたいだけです。



4. pythonで書かれたアプリケーションの解決。



私のコンピューターの1つは、いわばEeePCです。 もちろん、ubuntuも存在します。 また、800×600未満の解像度では、この非常に不快なグリッチが観察されます。 pythonで書かれたすべてのプログラム（ExaileやCCSMなど）は、低画面解像度での正常な表示を完全に拒否します。 コントロールをアクティブにすると、文字通りディスプレイのさまざまなコーナーにジャンプします。

処置：Pythonで作成されたプログラムを使用しないでください。 彼はもっと賢いものを思いつきませんでした。

ところで、なぜこれらのプログラムでこれが起こるのか、誰にもわかりませんか？



5.メニュー項目「終了...」を呼び出すとブレーキがかかります。 治療が見つかりました!!!



このグリッチは1台のコンピューターでのみ確認されており、私のコンピューターで発生しています。 どうやら、私は信じられないほど幸運でしたが、ログアウトダイアログの表示には3分かかります。 これは明らかに過剰です。

彼は、基本的なコマンドを備えたパネルメニューにこのような素敵な小さなボタンを作成することで治療しました。



更新した

ユーザー@Kroopsのプロンプトに従って、セッションで、電源管理サービスを有効にする必要があります。これは、同じforum.ubuntu.ruの悪意のある仲間がラップトップにのみ必要だと考えていたためです。



6.欠点：それらはメニューからdisplayconfig-gtkを突然投げました。



なんで？ わかりません。 代わりに、少なくとも何かを構成できるアプリケーションは、愚かなgnome-display-propertiesを配置します。これは、画面のリフレッシュレートを正しく判断することさえできません。

もちろん、displayconfig-gtkはまだここにあります。 コンソールに名前を入力するか、任意のコマンドを起動するためのメニューから名前を入力することもできますが、それについてはまだわかりませんよね？



7.一部のプログラムは、メインメニューに正しく登録されていません。



たとえば、AptonCDを取り上げます。 「管理」メニューでインストールした後、スタートボタンはコマンド/ usr / bin / aptoncdで誇示されます。 そのような打ち上げ後はどうなりますか？ このプログラムは、パッケージなどを扱うことが不可能であることを誓います。 コマンドはgksudo / usr / bin / aptoncdのようになります。 このような欠陥については、普通の人はLinuxに切り替えません。



8.視覚効果設定の「ユーザー設定」項目が消えました。



まあ、私はすべてのcompiz-fuzionユーザーがこの問題を認識していると思います。 顔の劣化。 不快です。 「標準エフェクト」に切り替えてから設定に戻すことはできません。



9.ノーチラスの誤って切断されたフラッシュドライブの死体。



コンピューターの後ろに座っていた仲間が、スティックを安全に無効にする必要があることを知らないことがあります。 など ちなみに、このグリッチはランダムに発生し、一部のデバイス、特に携帯電話の特権であり、ubuntuは適切に切断されていないかのように削除しません。 その結果、nautilusシェルでハングするマウントされたデバイスの束を取得します。



10.メニュー項目「ネットワークサーバー」を超えてファイルをドラッグしてグリッチします。



任意のフォルダーを開き、デスクトップからファイルを取得して、このフォルダーへのドラッグを開始します。 そのとき、「ネットワークサーバー」アイテムに配置されると、ubuntuはLAN全体を表示する必要があると考えます。 「ネットワーク：///」にコピーします。 誰がそこにコピーしたいのですか？

11. PulseAudio。



彼の前では、すべてが順調で、草がより緑で、太陽がより明るく輝いていました。 誰かがどこかでデフォルトでこのサウンドサブシステムがアクティブ化されていないと書いているようですが、事実は表面上です。 Ubuntuをインストールした後、ビデオが遅くなり、音が噛みついていました。 sudo apt-get remove pulseaudio purgeコマンドによって修正されました。 しかし、Audacityは正常に動作しなくなりました。 さて、各サウンドデバイスを見て、数分間考えた後、彼はそれを使用できないと喜んで発表します。 Googleは私の問題について、それは助けにならなかった=（

この問題は記事の準備中に明らかにされ、写真が撮られ、それだけです。 スクリーンショットはそれぞれ、添付しないでください=）
