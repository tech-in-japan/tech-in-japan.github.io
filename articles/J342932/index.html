HPE ProLiant for Microsoft Azure Stack：完全に制御可能なAzureクラウドの一部

9月、Hewlett Packard EnterpriseとMicrosoftは、データセンターでMicrosoft Azureクラウドの一部をホストできるソフトウェアおよびハードウェアソリューションであるMicrosoft Azure Stack用のHPE ProLiantの可用性を発表しました。 この新しい製品が、クラウドソリューションの柔軟性とローカルインフラストラクチャの信頼性を組み合わせたITへのハイブリッドアプローチの実装にどのように役立つかを理解しています。







HPE ProLiant for Azure Stackを使用すると、組織はアプリケーション開発を合理化し、優先リソースにアプリケーションを展開し、ストレージとデータのセキュリティ要件を満たし、独自のインフラストラクチャを活用できます。



HPE ProLiantは、ハードウェアリソース、統合およびサポートサービス、個々の部門または組織全体向けのソリューションコンサルティング、およびリースから支払いまでのさまざまな支払いモデルに関する多くのオプションを選択できる、現時点で最も柔軟なAzure Stackプラットフォームを提供します使用します。



複合施設には何がありますか？



初期段階では、ハードウェアとソフトウェアの複合体の構成には、4〜12の生産ノード（DL 380 Gen9、2018年初頭にGen10サーバーのサポートも予定されています）、DL 360管理サーバー、2つの生産的なスイッチ、および制御ネットワーク用の1つのスイッチが含まれます。 オプションには、ラック、KVM、HPE無停電電源装置などが含まれます。







DL380ノードは、ハイパーコンバージドシステムを構成します。 各ノードには以下が含まれます。



  • Azure Stack VMマネージャー
  • 生産的なVM
  • Microsoft Storage Spaces Directに基づく仮想ストレージ（ちなみに、パートナーのMellanoxとMicronによるこのソリューションのパフォーマンステスト結果を参照）
  • Azure Stack仮想ネットワーク


柔軟なオプション



Microsoft Azure Stack用HPE ProLiantの特徴の1つは、さまざまな構成の幅広い選択です。



  • 4〜12ノードの構成-単一のノードを追加することができ、必要なボリュームのシステムを取得できます。 さらなる拡張（12ノード以上）-新しい「リージョン」またはサーバーグループを追加します。
  • さまざまなワークロード用のプロセッサ。
  • メモリとストレージのさまざまなオプション、シェルフの選択、サードパーティのネットワークスイッチと電源オプション、既存の環境へのシームレスな統合を提供します。






運営管理



Azure Stackを利用して、HPE ProLiant for Azure Stackでの開発とテストを自動化することで、管理の自動化オプションも利用できます。



Microsoft Azure Stack向けのHPE ProLiantハードウェアは、HPE OneViewを使用して、ソフトウェアの更新を簡単に管理し、システムステータスを追跡します。 また、HPEソリューションに基づく現在のインフラストラクチャへの複合体の統合を簡素化します。

OneViewを使用すると、ほぼすべてのHPEサーバー、多数のストレージモデル、ネットワークを1つのインターフェイスから管理できます。 管理はインフラストラクチャパターンに基づいています。 サードパーティの自動化および制御ツールへの統合が利用可能です。



サポートとサービス



システムの完全な組み立ては、スイッチの構成、ファームウェアとソフトウェアの更新、設定の検証など、工場で実行されます。 オンサイトの追加検証は、インストール、システム統合、および顧客へのターンキー配信中に実行されます。



問題が発生したかどうか（ソフトウェアまたはハードウェア）に関係なく、単一のログインウィンドウで、HPEとMicrosoftが共同でサポートを提供します。 顧客は、グローバルデータベースと専門家のコミュニティにアクセスできます。 開発中のAzure Stackの機能（DevOpsのコンテナーやツールなど）を使用して、Azure Stackを現在のインフラストラクチャに統合するための高度なトレーニングとコンサルティングも利用できます。

また、ベルビューとジュネーブには、HPE + Microsoft Azure Stackの2つの専門知識センターがあります。 これらのセンターでは、ビジネスおよび技術計画、テストテクノロジーのユースケース、モデリングなど、リモートを含むHPEおよびマイクロソフトのスペシャリストに連絡できます。



HPEは、Microsoftの同様のAzure支払いモデルに合わせて、消費時にAzure Stackのインフラストラクチャ支払いを提供することもできます。 HPE ProLiant for Microsoft Azure Stackは、定義済みのしきい値を持つリソース消費メトリックに基づいて支払いを提供するHPE Flexible Capacityオファリングと組み合わせることができます。 HPE ProLiant for Microsoft Azure Stackソリューションを購入するには、リースなどを含むさまざまなオプションがあります。



ソリューションの詳細については、ウェビナーエントリをご覧ください。特に、Azure Stackの機能、構成オプション、およびサイジングについて詳しく説明しています。 また、HPEおよびMicrosoftからのニュースをすばやく受信するには、Telegramのチャンネルに参加してください： @hpedigitizeおよび@microsoftrus 。
