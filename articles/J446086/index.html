Linuxの歴史。 パートIII：新しい市場と古い「敵」

90年代にリリースされた最初の商用Linuxディストリビューションを思い出しました。 さて、2000年代の時代です。このOSが消費者市場を征服し始めた時代です。

消費者へのステップ



10〜20年前、PCオペレーティングシステム市場におけるLinuxのシェアは非常に低く 、2％未満でした。 この理由の1つは、使い慣れたデスクトップ環境の欠如でした。 KDEとGNOMEの出現により状況は変わり始めました。 依然として最も人気のあるLinuxデスクトップ環境です。 これらの2つの製品は、消費者市場向けの流通の最初のシグナルの1つになりました。 そして彼は待たされませんでした。



2004年、Debianアーキテクチャ上に構築されたUbuntu 4.10のリリースが行われました。 このプロジェクトは、起業家であり開発者でもあるマーク・シャトルワースのおかげです。



マークは、Debianで働いていましたが、理想的なOSのアイデアに合わないいくつかの欠点を見つけました。 シャトルワースは、更新サイクルへのアプローチを変更し、製品の可用性に焦点を合わせる必要があると考えていました。 最終的に、彼と数人の同僚はCanonicalを設立し、ディストリビューションの開発を開始することにしました。



オープンソースコミュニティは、安定したアップデートリリーススケジュールで年に2回、新しいシステムを暖かく歓迎しました。 OSの登場から3年後、OS は最も人気のあるデスクトップLinuxディストリビューションになりました。 ユーザーは 、インストールとセキュリティを簡単にするためにUbuntuを選択しました 。 長年にわたって、便利で理解しやすいLinux製品を作成するというMark Shuttleworthのアイデアが実現したと言えます。



オープンソースソフトウェアの歴史でよくあることですが、Ubuntuの最初のバージョンがリリースされた直後に、改善された代替案のアイデアが浮上しました。 2006年、Linux Mintディストリビューションの作成者であるClement Lefebvreは、まさにこのアイデアに導かれました。 OSの最初のバージョンはKubuntuに基づいていましたが、v2.0で既に開発者はUbuntuに切り替えました。 その瞬間から、Mintは前任者からますます多くのアイデアを引き出し、2年後、更新サイクルはついにUbuntuリリースに「拘束」されました。



Linux Mintの機能の1つは、マルチメディアファイルを再生するためのプラグインやコーデックを含む独自のソフトウェアのサポートです。 このような動きはLinuxディストリビューションでは一般的ではありませんでした-開発コミュニティはオープンモデルを堅持しました。 しかし、この新しいアプローチはMintの成功を保証しました-Linuxユーザーはついに使い慣れたWindowsアプリケーションで作業することができました。 過去8年間、Mintはカーネルの歴史の中で最も人気のある3つのLinuxディストリビューションの1つです。


PC市場の戦い



2000年代半ばのUbuntuの人気と利便性は、デルによって認識されました。 1998年に、同社は製品ライン全体でLinuxを提供する最初の主要メーカーになりました。 ただし、デルのオープンソースOSサポートプログラムは数年しか続きませんでした。 閉鎖の間接的な理由は「需要不足」 と呼ばれていました 。



2007年に、UbuntuベースのPCとラップトップのリリースでプロジェクトが再開されました。 マイケルデル自身が配布キットを使用し、その便利さを高く評価したという事実が、この点で重要な役割を果たしました。 彼はCanonicalに連絡し、同社はDellコンピューター用のソフトウェアの準備を開始しました。 Ubuntuがプリロードされたコンピューターは、米国、英国、フランス、ドイツで販売されました。



この信号は、Asusによって応答されました。 同社は2007 Asus Eeeネットブックを発売しました。 その主な機能は、プリインストールされたLinuxシステムです。 オープンソースOSを選択したことにより、メーカーはデバイスの価格を大幅に引き下げ、コミュニティの注目を集めることができました。



ただし、コミュニティの反応は、Asus-Xandrosが選択した分布のために混合されました。 OSは同名の会社の商用製品であり、フリーソフトウェアの原則に完全には準拠していませんでした。 すぐに、Windows XPベースの最初のAsus Eeeがリリースされました。 マイクロソフトは、メーカーのライセンス料を大幅に削減し、新しいニッチ市場での地位を確保しました。 その後、Linuxは新しい市場でシェアを失い始めました 。



しかし、崩壊は起こりませんでした。 これは、2009年にGoogleからChrome OSがリリースされたことが一因です。 新しいOSは、別のLinuxデバイスであるChromebookの基盤になりました。 Linus Torvaldsはかつて、このラップトップのリリースがPC上のLinuxの将来のターニングポイントになる可能性があると言っていました。 4年後、これらのデバイスの売上は米国のApple Macの売上を上回りました。 ただし、これについては次の記事で説明します。

モバイル時代



多数のモバイルオペレーティングシステムがLinuxに基づいて構築されました。 たとえば、2005年にリリースされたNokiaのMaemo OSはDebianに基づいており、GNOMEからほとんどのGUI要素とライブラリを借りていました。 2004年、歴史上最初のモバイルオペレーティングシステムの1つであるPalm OSの開発者は、OSをLinuxカーネルに移行しました。 このイニシアチブから、Access Linux Platform（ALP）とPalm webOSの2つの独立したプロジェクトが生まれました。 最初のものは配布を受け取りませんでしたが 、webOSに基づいてPalmや他のメーカーからいくつかのモバイルデバイスが出てきました。



当時のLinuxコミュニティにとってより重要なイベントは、まったく異なるニッチで開催されました。 Android Inc.は2003年にカリフォルニアで設立されました。 打ち上げの1年後、チームは投資家を探し始めました。 同社が最初に戦略を構築した基礎となった製品は、デジタルカメラのオペレーティングシステムでした。 このニッチで成功していないため、チームは焦点を変更しました。 そのため、当時のモバイルOSセグメントのリーダーだったSymbianとMicrosoft Windows MobileのライバルであるAndroidシステムのアイデアが現れました。 2005年のこの方向での最初の開発により、同社はGoogleに買収されました。



すでに2006年に、ITの巨人は、通信会社に新しいOSを実行するデバイスの最初のプロトタイプを導入しました 。 それはQWERTYキーボードとタッチスクリーンのない携帯電話でした。 2007年、メディアは GoogleのモバイルOSがLinux上で動作することを知り 、同社はすでにモバイルオペレーターだけでなく、スマートフォンメーカーとも交渉中です。 リリースはすぐに行われる予定でした。



しかし同じ年に、最初のiPhoneが市場に参入したため、Googleはタッチスクリーンを電話機に緊急に追加し、リリース日を延期する必要がありました。 最後に、2008年、GoogleはT-Mobileとともに、最初のAndroidスマートフォンであるT-Mobile G1またはHTC Dreamを発表しました。 当初のG1の需要は予測を上回り 、そのリリースは世界で最も人気のあるモバイルOSの歴史の始まりを示しました。 数十のAndroidバージョンと数千のスマートフォンが、Linuxをこのニッチのリーダーにしています。



生態系開発



Linuxコミュニティでの新しいディストリビューションのリリースに加えて、2000年代には他の重要なイベントが開催されました。



長い間、カーネル開発者はコード管理システムを使用していなかったため、Linuxなどの大規模プロジェクトをサポートすることは困難でした。 したがって、2002年にLinus TorvaldsはBitKeeperに切り替えることを決定しました。 ソフトウェアは商用製品でしたが、Linux開発者には無料で使用する機会が与えられました。 それにもかかわらず、コミュニティは議論の余地があるSCMシステムを選択する決定を見つけました 。



2003年、BitKeeperは代替案を考案しました。 Linus Torvaldsは、Gitと呼ばれる彼自身のコア開発プロジェクトを紹介しました。 今日、Gitは開発環境で最も要求されているバージョン管理ツールです。


オープンプロジェクトのエコシステムも発展していました。 イベント、非営利製品の製造、ライセンス活動-これらの活動を中心に、90年代初頭からますます多くの組織が作成されています。 そのうちの1つは、後にLinux Foundationコンソーシアムになりました。



組織のプロトタイプは、非営利組織LIに基づいて作成されました。 2000年に、LIの管理に問題が生じ、会社はOpen Source Development Labs（OSDL）に改名されました。 2003年に、Linus Torvaldsは公式にOSDLのメンバーになりました 。 コンソーシアムには、HP、Intel、IBMなどの企業が含まれます。 4年後、OSDLは別の非営利団体であるFree Standards Groupとチームを組みました。 この提携により、Linux Foundationが設立されました。LinuxFoundationは、まだオープンソースプロジェクトの資金調達とその標準化に関与しています。

古い紛争と新しい地平線



新しい時代はLinuxに新たな挑戦をもたらしました。 2003年、Unixを所有していたSCO Group社は 、Linuxおよびその他のディストリビューションが著作権を侵害する元のOSのコードを悪用したと述べました。 SCOによって開始された船舶はいずれも申請者に有利になりませんでした。



これと並行して、Microsoft は Get the Facts広告キャンペーンを開始し、LinuxとWindowsのサーバーバージョンをオープンに比較しました（もちろん後者を支持しました）。 マイクロソフトの活動は、おそらく企業部門でのLinuxのシェアの拡大に関連していたと思われます。 2004年、UnixとLinux はこの市場のほぼ3分の1を占め 、IT大手の地位を深刻に脅かしました。



新しい広告キャンペーンの一環として、企業はWindows Server 2003の総所有コストがLinuxの総所有コストよりも低いと主張しました。 オープンソースコミュニティは、 あらゆる点でこれらの事実に反論しました 。 仮想紛争の規模は、Microsoft が 2007年にキャンペーンを停止するまで、数年にわたって拡大しました。



しかし、これらすべてが、Microsoftがオープンソース企業との仕事を確立することを妨げるものではありませんでした。 2006年、SUSE Linux Enterprise Serverディストリビューションの権利の所有者であるNovellは、Microsoftとのパートナーシップを発表しました。 同様に重要なのは、13年前のこのステップが、Microsoftのクラウド戦略に間接的に影響を与える可能性があることです。 SUSE Linux Enterprise Serverはその後、企業のクラウドプラットフォームで利用可能な最初のディストリビューションの1つになりました。
