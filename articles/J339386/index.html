Slack、xgboost、GPUのオープンデータサイエンスの作成者

オープンデータサイエンス（ODS）コミュニティは、オープンマシンラーニングコース（ OpenML ）のHabréで既に知られています。 今日は、作成者とODSの歴史、人々、および最も人気のある機械学習方法（Kaggleおよび業界のプロジェクトによる）についてお話します。 興味深い事実と技術的な専門知識については、猫の下でお願いします。









アレクセイナテキンナテキン 。 機械学習とデータ分析に関連する多数のプロジェクトの創設者。 東ヨーロッパ最大のデータサイエンティストのオンラインコミュニティであるOpen Data Scienceの独裁者およびコーディネーター。



- どのようにしてコミュニティを作成するというアイデアを思いついたのですか？ どのようにすべてが始まりましたか？ なぜ選択がSlackにかかったのか、そして誰が起源に立ち向かったのか？



Alexei Natekin： 2014年、DM Labs（ DM：Data Mining ）の無料教育プログラムの3番目のセットが終了しました。 私たちはすでに資料を完成させており、プロジェクトに協力することを望んでいました。 これは楽しいタスクの最初の繰り返しでした-ポルノタグの分析、ソーシャルネットワークの不況の判定、Dotaのゲームデータ。 すぐに、プロジェクトをオープンにして、コースの参加者だけでなく、より身動きの取れない人を巻き込むというコンセプトがありました。



VKontakteと自作のWordPressでの集合的なファームチャットルームを実験した結果、これらすべてが通常の作業に適していないことがわかりました。 幸いなことに、すでに2015年の初めにSlackは徐々に名声を高め始め、プロジェクトとコミュニケーションのプラットフォームとして使用しました。 それは非常に便利であることが判明し、ほぼすぐに私たちはそれに立ち寄りました。



その瞬間、私たちはこれらすべてが美しいイデオロギーに包まれなければならないことについてあまり考えず、単にこのビジネスをオープンデータサイエンスと呼びました。 このフレーズは1つの会議（いわゆるオープンデータサイエンス会議）でしか見つかりませんでした。オープンサイエンスとDS（ データサイエンス ）の共同教育の組み合わせ（他の人に教え、自分で何かを学ぶため）は、良い基本的なオプションでした。したかった。 それでも、オープンデータとオープンサイエンスはすでにそこにあり、ミッシングリンクを発明して実装するためだけに残っていました。







私たちはプロジェクトチャットルームとしてスタートしました。このチャットルームでは、DSの議論に関連する多くの専門家や技術チャンネルがすぐに登場しました。 ロシアのローカルスタックオーバーフローに焦点が移ったと言えます。 プロジェクトは独自の生活を送り、活動は一般的な議論に変わりました。



幸いなことに、私たちは主要なDS関連の分野と技術に関する多くの専門知識を迅速に集め、人々の最初の骨格はODSで形成されました。 いつでも、興味のある方向に何かを尋ねることができ、これに精通している人から良いアドバイスを得ることができます。



当時、現在の形のプロのDSコミュニティは、もしあれば、一方向の定期的な会議の聴衆を代表するか、特定の場所（大学生など）に閉じて強く結びついていることが判明しました。 。



私たちはすぐに団結と団結に賛成しました。そのため、ミタップやさまざまなグループとの統合を開始しました。モスクワ独立DS Meetup、MLトレーニング（サンクトペテルブルクのSNAハッカソン後のODSの前に始まりました）、R、ビッグデータ、そしてディープハックのミタップ、ディープラーニングMeetup、Mail.ruのDS Meetupなど。



興味深い事実：冬の集会の1つで、この会議で支払いが切れたことを知り、次のビッグデータスパマーがそれを手に入れられないように、失われたMDSM Mail.ruアカウントの支払いのために突然爆発しました-支払いはまだ私のカードから落ちています:)



歴史的に、私たちがODSを共同で構築した最も活発な人たちもオーガナイザーでした。 したがって、私たちはスピーカー、PR、組織の問題で互いに助け合っただけでなく、私たちの側ではすぐに新しいイベントや形式を思いつき、実施し始めました。 その中には、東ヨーロッパおよびCISで最大のDS会議としてのDataFest、およびビジネスにおけるDSの真のメリットに関するコンテンツで非常に貴重な一連のイベントとしてのDSケースクラブ、および独自の、しかし人々に愛されている形式のデータ朝食があります。



そしてもちろん、私たちは企業と協力しました。たとえば、Yandexで一連のデータ＆サイエンスイベントを作成し、Sberbank-Sberbank Data Science Journeyを作成しました。 最近の推定によると、CIS全体で20を超える定期的なイベントが蓄積されています。



拡張は間もなく始まりました-私たちは成果を共有し、他の都市や国でイベントとDSの開発を開始しました：最初にモスクワとピーター、次にエカテリンブルク、ノボシビルスク、ニジニノヴゴロドとカリーニングラード、次にウクライナとキエフ、リボフとオデッサ、ミンスクとベラルーシCISにはさらに多くの新しい都市があります。



現在、管理チームには4か国から35人が参加しています。 米国、ドイツ、ノルウェー、イスラエルでは会議に活発な参加者が集まっていますが、私たちはまだグローバル化に取り組んでいます。 そのため、Slackには20のタイムゾーンから7.5千人がおり、そのうち週に1回は3千人以上が訪れています。 したがって、グローバルな可能性があります。



アナログと競合他社はありますか？ 私たちが成長した形で、世界に少なくとも1つのアナログがあれば、それは非常にクールです。私たちは彼と協力します。 残念ながら、DS / MLのリーダーと見なされているアメリカでは、私たちに似た精神はなく、登場する可能性も低いです。



Mitapasには有料のマーケティングゴミが散らばっており、地域コミュニティは大学や企業と非常に強固に結びついています（Googleの個別のハングアウト、Amazonの個別のハングアウト、Facebookの個別のハングアウトなど）。 Machine Learning Meetupでは、適切な機械学習に従事している人を見つけることは困難です。原則として、100人の場合、10人は見つかりません。非常に興味のある人、見物人、伝道者、PR人です。 しかし、関連する会議では、それは本当に世界レベルであり、ランダムな人はほとんどいません。



昨年のNIPSカンファレンスの直後に作成されたAI Researchers slackは、9か月で1,000人を集め、最初の週にIan Goodfellowが見られましたが、本質的に死んでいます。 週に約3万件のメッセージを作成し、合計で約150万件のメッセージがあります。DSブログプラットフォームのようなKDnuggetsがあります。 DSコミュニティの最大のコミュニティはKaggleに住んでいると言えます（メッセージの数が多くても驚くことはありません）。 しかし、会場、イベント、およびトレーニングなどのイニシアチブを組み合わせたアナログはまだありません。



-「Run xgboost」というフレーズがミームになったので、xgboostとは何ですか。なぜスタックする必要があるのでしょうか。



Alexey Natekin： Xgboostは、勾配ブースティングアルゴリズムの具体的な実装です。 アルゴリズム自体は、Yandex検索エンジン、Mail.ru、Microsoft、Yahooなどの実稼働環境と、Kaggleなどの競技会場の両方で、非常に強力な汎用モデルとして広く使用されています。 Xgboostは、第一に、非常に効率的に記述されたコードであり、モデルをより速く、より生産的に学習できます。 次に、xgboostに追加のバンと正規化が追加され、ツリー自体がより安定します。



Xgboost自体は、詐欺なしで使用するのに十分です。 Kaggleとその外側では、それは棚から取り出される簡単なソリューションになり、彼らは通常非常に良い結果をもたらすことを知っています。 ここにあります：もちろん、無料の昼食はありません （ 注 、 これは無料の昼食定理への参照です）が、xgboostは次のタスクで試してみる価値があります。



ただし、Kaggleの主なタスクは、ランキングテーブルで最高の結果を取得することです。 繰り返し、この最高の結果は、小数点以下3桁、4桁、5桁を争う何十人もの人々に降りかかった。 スタッキングアルゴリズムとマルチレベルアンサンブルを使用して、このようなパンくずを正確にこすり取る方法が長い間知られています。



1つのモデルではなく、数十ダースを非常に慎重かつ有能にトレーニングし、その予測よりも数ダース多くトレーニングする場合、この方法でもう少し精度を削ることができます。 これの代償は多くの計算コストであり、彼らの正しい心の誰も実際には繰り返さないだろうが、Kaggleでは誰も健全な心と現実的な仕事を約束しなかった。



したがって、勝者、つまり最初の場所に勝った人の決定は、多くの場合、積み重ねられたxgboostの複数の層で構成されていました。 Xgboost-それは強力で高速だからです。 オーバーラン-あなたが最初の場所を取る必要があるため。



「stack xgboosts」というフレーズは、本質的に、Kaggleコンテストの無意味で容赦のない本質のm笑であり、その多くは、実用的、解決策の観点から、ブルートフォースとひどい方法で解決できます。 しかし、それにもかかわらず、Kaggleで勝ちました。



-彼らは、xgboostが実際のアプリケーションで優れた結果を示すと言います。 xgboostが競合他社よりもはるかに強い場合の例を挙げていただけますか？ そして、何らかの合理化がありますか、なぜこれがそのようなデータで起こっているのですか？



Alexey Natekin：競争相手と見なされるものに依存します。 一般に、勾配ブーストは、上記のように、詐欺対策/スパム対策、金融会社のあらゆる種類の予測、および大企業の検索エンジンなど、それらすべてを収集するのが難しいほど多数のアプリケーションの裏にあります。 まあ、Kaggleタスクは、プロダクションでのすべてのおもちゃの遊びにもかかわらず、データがあまりまばらではなく、写真ではないほとんどの競技会では、通常、勾配ブースティング+ある種のアンサンブルによって勝ちます。特に、精度を上げる必要がある場合はそうです。



ブースティングが競合他社よりも桁違いに強力であると言うことは不可能です。なぜなら、一部のモデルが実際のデータに対してどのような結果をもたらしたかは、ビジネスアプリケーションで誰も知らないからです。 また、ブースト自体の結果は、最も近い競合他社であるランダムフォレスト、ニューロン、またはSVMを超えることはありません。



ブースティングは実装の観点から最もよく機能し、非常に柔軟な方法のファミリであるため、特定のタスクに合わせてブーストを調整することは大したことではありません。 そして合理化について-正確に仕事を後押しする理由とトリックは何ですか-私はいくつかの私の（ 1、2 ）チュートリアルと1つをお勧めできます-Alexander Dyakonov。



-Smartdataconfに関するレポートの発表で、人々はxgboostからGPUでニューラルネットワークと同じ加速を期待していることに注意してください。 ニューラルネットワークがこのようなパフォーマンスの向上を得る理由を直感的に説明できますか？



Alexey Natekin：通常の日付は、知的労働の悪魔主義者- スタハノバイトをどのように考えていますか？ ニューラルネットワークがあり、クールなハードウェアと、数千の特別に最適化されたコアを備えたトップエンドのビデオカードを作成しました。 また、Nvidia自身は次のように述べています。私たちのハードウェアは、それが何を意味していても、AIを進化させ、加速させます。 したがって、GPUをより広範なタスクで使用できるという非現実的な期待。



ニューラルネットワークでは、トレーニングと予測の両方で実行される操作の大部分は行列の乗算です。 今では、これはテンソルを使った作業であると言った方が正直ですが、本質は同じです。多くのルーチン行列演算が必要です。



低コストと低消費電力でより多くの処理コアがあるため、GPUはこれに最適です。 そして、機能の拡張セットの欠如は安全に無視できます。



しかし、このため、GPUで配列を並べ替えて再帰的な計算を行うことは難しくなりますが、畳み込みを計算して行列を乗算するのは非常に安価です。 これは緊急の問題を提起します：GPUで決定木の学習を開始し、実際に広く使用されている場合はどうなりますか？ そして、このアイデアは設計上どのくらい意味がありますか？



-そして、なぜxgboostやその他の機械学習アルゴリズムの場合にこれが起こらないのか、GPUをトレーニングに使用するには何が必要なのか？



Alexey Natekin： GPUは、GPUに適応したアルゴリズムに最適です。 すでに書いたように、GPUでインデックスを作成しませんか？ ニューラルネットワークに加えて、他の機械学習タスクでGPUをより広く適用するには、GPUのアルゴリズムを効果的に実装する必要があります。 または、既存のバージョンのCPUアルゴリズムに触発されて、それらに基づいて効率的なGPUコンピューティングに適した新しいアルゴリズムを作成する可能性が高くなります。 または、Intel phiの広範なリリースについて待ちます。これについては、さまざまな伝説があります。 しかし、これはもはやGPUではなく、まったく別の話です。



-最後に、ハードウェアに関する質問：機械学習用のGPUカードを購入する際に注意すべきパラメーターは何ですか？ トップKaggleの人々は現在何を使用していますか？



Alexey Natekin：実際、最適な価格、速度、11ギガバイトのメモリを備えているため、ほとんどの場合1080Tiを購入しています。 Titan Xと1080Tiのうち、後者が選択されます。 それでも同じように、データセットは長い間メモリに保存されていないため、メモリが多いほど、処理に費やすことができます。 しかし、一般的に、誰もが次世代のカードで何が起こるかに興味を持っています。登場するとすぐに、非常に迅速に購入する必要があります。
