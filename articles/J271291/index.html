3D Touchの紹介

iPhone 6sおよびiPhone 6s Plusのリリースに伴い、Appleは電話と対話するまったく新しい方法であるタッチジェスチャを導入しました。 ご存じのように、この機能はApple Watch、MacBook、およびForce Touchと呼ばれるMacBook Proですでに利用可能です。 これは、文字通り、ユーザーインターフェイスに新しい概念を追加しました。



なぜForce TouchがiPhone用の3D​​ Touchに改名されたのか疑問に思っているなら、あなたはあなただけではありません。 クレイグフェデリギ （名前にも明らかに困惑していた）がこの新しい機会を導入した直後に、多くの憶測が浮上しました。 Force Touchという名前に合わなかったものは何ですか？ スターウォーズのジョークが多すぎますか？



しかし、違いがあります！ 明らかに、Force Touchは強い圧力しか認識できませんが、 3D Touch機能はより感度が高く、押した強さに基づいていくつかのレベルのタップを強調表示できます。







変更は取るに足らないように見えるかもしれませんが、開発者はiPhoneでより正確に測定することができます。 たとえば、このGravityアプリは、Force Touchを使用してiPhoneをデジタルスケールに変換します。 不明な理由でAppleによって拒否されましたが、そのアイデアは素晴らしいです。 したがって、3D Touchがどのように機能するかを示すために、同様のアプリケーションを作成してみましょう。



さあ始めましょう



開始するには、作成したこのプロジェクトテンプレートをダウンロードします 。 実際、これは、 シングルビューiPhoneアプリケーションテンプレートに基づく空の作成アプリケーションです。 ホストされたプロジェクト（UILabelsとUIImage）を作成し、 IBOutletsをViewController.swiftにフックしました 。







アプリケーションの設計は非常に単純です。2つのタグを持つ1つのコントローラーがあります。1つはタイトルで、もう1つはラベルで、iPhone画面への圧力の割合を示します。



さあ始めましょう！ iPhone 6sおよび6s Plusでは、UITouchオブジェクトには、 forceおよびmaximumPossibleForceと呼ばれるCGFloatなどの2つの新しいプロパティがあります 。 Forceはタッチの強さを表し、1.0は平均タッチを意味します。 MaximumPossibleForceは、タッチする最大力を示します。



ユーザーがセンサーをクリックするたびにtouchesBeganメソッドが呼び出され、ユーザーが画面上で指を動かすとtouchesMovedが呼び出され、状況に応じてtouchedCancelledまたはTouchesEndedが呼び出されます。 私たちの目的では、 touchesMovedメソッドが必要な唯一の方法です。 TouchesMovedには、タッチ（タッチ）とイベント（イベント）の2つのパラメーターがあります。 タッチは、UITouchオブジェクトのセット（さまざまなオブジェクトの順序なしセット）です。 タッチするUITouchオブジェクトは1つだけである必要がありますが、十分な注意を払っていない可能性があるため、touches.first（タッチセットの最初のUITouchオブジェクト）が追加のバインディングでnilであるかどうかを確認することを強くお勧めします。 次のメソッドをViewController.swiftに貼り付けます。 

override func touchesMoved(touches: Set<UITouch>, withEvent event: UIEvent?) { if let touch = touches.first { if #available(iOS 9.0, *) { if traitCollection.forceTouchCapability == UIForceTouchCapability.Available { // 3D Touch capable } } } }


ifステートメントを使用して、デバイスが3D Touchをサポートしているかどうかを確認します。 このプロジェクトを楽しみながら行う場合、この部分はオプションです。 ただし、アプリケーションをApp Storeで公開する場合、iPhone 6などの古いデバイスは3D Touchをサポートしていないため、チェックを実行することが前提条件です。



デバイスがiOS 9.0以降で実行されているかどうかも確認したことに注意してください。 これは、Swift 2.0で導入された新しい#available構文で行います。 Swift 2.0の新機能について詳しく知りたい場合は、この記事を読むことをお勧めします。 繰り返しますが、アプリケーションのターゲットプラットフォームが9.0以上の場合、このチェックはオプションです。



押圧力の割合を取得するには、接触力を最大力（つまりtouch.maximumPossibleForce）で除算するだけです。これは、可能な最大の押圧力です。 次に、ラベルテキストを更新します。 次のようにメソッドを更新できます。 

 override func touchesMoved(touches: Set<UITouch>, withEvent event: UIEvent?) { if let touch = touches.first { if #available(iOS 9.0, *) { if traitCollection.forceTouchCapability == UIForceTouchCapability.Available { // 3D Touch capable let force = touch.force/touch.maximumPossibleForce forceLabel.text = "\(force)% force" } } } }


iPhone 6s / 6s Plusでアプリケーションを実行すると、画面をクリックしたときに圧力の割合が表示されます。 ただし、スケールを作成しているため、iPhoneで計量したグラム数を追加することをお勧めします。 Ryan MacLeadによると、センサーの最大重量は約385gです。 したがって、 最大可能力は385グラム（約3.8ニュートン）に相当します。 簡単な計算により、％力をグラムに変換できます。 必要なのは、押圧力のパーセンテージに385を掛けるだけです。385グラム以上のオブジェクトの場合、ラベルを「385+グラム」に変更するだけです。



次のコードスニペットを使用してメソッドを更新します。 

 override func touchesMoved(touches: Set<UITouch>, withEvent event: UIEvent?) { if let touch = touches.first { if #available(iOS 9.0, *) { if traitCollection.forceTouchCapability == UIForceTouchCapability.Available { if touch.force >= touch.maximumPossibleForce { forceLabel.text = "385+ grams" } else { let force = touch.force/touch.maximumPossibleForce let grams = force * 385 let roundGrams = Int(grams) forceLabel.text = "\(roundGrams) grams" } } } } }


かっこいい！ デジタルスケールアプリケーションを作成しました。







現時点では、アイテムを削除したり、画面へのタッチを停止した後、アプリケーションの重量がゼロになることはありません。 touchesEndedメソッドを実装して、ラベルをリセットできます。 

 override func touchesEnded(touches: Set<UITouch>, withEvent event: UIEvent?) { forceLabel.text = "0 gram" }




メイン画面のクイックアクション



3D Touchのもう1つの優れた用途は、ホーム画面での迅速なアクションです。 クイックアクションは、アプリケーションディレクトリに直接移動するためのメニューをユーザーに提供します。 プログラムアイコンを強く押すと、メニューが表示されます。 3D Touchテクノロジーの出現により、 TwitterやInstagramなどのアプリやその他のアプリがこの新機能の使用方法を示してきました。







作成したアプリケーションにクイックアクションを追加して、白ではなく青の背景でアプリケーションを開きます。 クイックアクションを追加するには、プロジェクトのinfo.plistファイルを開きます（プロジェクトナビゲーターで[スケール]ワークスペースをクリックし、[ターゲットをスケール]を選択して[情報]タブに移動します）。 ファイルに「UIApplicationShortcutItems」タイプを追加し、配列（Array）を選択します。 配列の各要素は、1つのクイックアクションのプロパティを含む辞書です。

  • UIApplicationShortcutItemType （ 必須 ）：クイックアクションを識別する文字列。 この文字列は、アプリケーション固有の一意である必要があることに注意してください。 一意の文字列を作成するには、識別子またはその他のトリックを使用してプレフィックスを追加することをお勧めします。
  • UIApplicationShortcutItemTitle （ 必須 ）：文字列。ユーザーに表示されるクイックアクションのタイトルを表します。 例は、「最後に撮影した写真を表示する」です。
  • UIApplicationShortcutItemSubtitle （ オプション ）：クイックアクションの名前の文字列。 例は「昨日撮影した最後の写真」です。 クイックアクションのアイコンを追加する場合、Appleのシステムアイコンまたはカスタムアイコンの2つのオプションがあります。
  • UIApplicationShortcutItemIconType （ オプション ）：クイックアクションの横に表示されるシステムアイコンを示す文字列。
  • UIApplicationShortcutItemIconFile （ オプション ）：クイックアクションの横に表示されるカスタムアイコンを示す文字列。
  • UIApplicationShortcutItemUserInfo （ オプション ）：クイックアクションで実行したい追加情報を含む辞書。


配列では、クイックアクション「OpenBlue」を設定するための4つの要素を定義します。 これで、 info.plistは次のようになります。







「 com.appcoda.Scale 」または使用する他の識別子の代わりに「 $（PRODUCT_BUNDLE_IDENTIFIER） 」を使用したことに注意してください。 これはセキュリティ上の理由によるものです。何らかの理由でバンドルIDを「一般」に変更すると、プロジェクト全体が影響を受け、識別子はどこでも変更されます。 それ以外の場合は、どこでも手動で変更する必要があります。 info.plistファイルで、 Bundle Identifierキーが同じアプローチを使用していることがわかります。 ' $（PRODUCT_BUNDLE_IDENTIFIER） 'は、プロジェクトの識別子へのパスを説明しています。



最後に行うことは、ユーザーが起動したときにクイックアクションを実装することです。 主な作業は、 performActionForShortcutItemメソッドのAppDelegate.swiftで処理されます。 クイックアクションがアクティブになると、メソッドが呼び出されます。 したがって、クイックアクションを処理するには、このメソッドを実装する必要があります。 

 func application(application: UIApplication, performActionForShortcutItem shortcutItem: UIApplicationShortcutItem, completionHandler: (Bool) -> Void) { // Handle quick actions completionHandler(handleQuickAction(shortcutItem)) }


予想どおり、ハンドラーを呼び出して、高速アクションの成功/失敗に応じてnull値を渡します。 ここでは、アクションを処理するための別個のhandleQuickAction関数を作成します。 クイックアクションのいくつかのケースを表す優れた方法は、「 UIApplicationShortcutItemType 」を生の値として使用して列挙を使用することです。 列挙を宣言し、下に示すようにhandleQuickActionメソッドを実装して、クイックアクションでアプリケーションを起動したときに背景色を青に設定します。 

 enum Shortcut: String { case openBlue = "OpenBlue" } func handleQuickAction(shortcutItem: UIApplicationShortcutItem) -> Bool { var quickActionHandled = false let type = shortcutItem.type.componentsSeparatedByString(".").last! if let shortcutType = Shortcut.init(rawValue: type) { switch shortcutType { case .openBlue: self.window?.backgroundColor = UIColor(red: 151.0/255.0, green: 187.0/255.0, blue: 255.0/255.0, alpha: 1.0) quickActionHandled = true } } return quickActionHandled }


とても簡単です。 アプリケーションを起動してクイックアクションを実行すると、背景が青に変わります。







忘れてはならないものは何ですか？



考慮すべきことが1つあります。 起動シーケンスに関しては、起動するアプリケーションとクイックアクションを使用して再開するアプリケーションには違いがあります。 アプリケーションがいつ起動するかを知っているので、 willFinishLaunchingWithOptionsメソッドが呼び出され、その後にanddidFinishLaunchingWithOptionsメソッドが続きます。 ただし、アプリケーションがクイックアクションによって再開される場合、 performActionForShortcutItemメソッドの実行のみを引き起こします。







didFinishLaunchingWithOptionsメソッドを見ると、背景色を白に設定するコード行があることがわかります。 これは、アプリケーションがアプリケーションアイコンを介して正常に起動するときに使用されます。 

 func application(application: UIApplication, didFinishLaunchingWithOptions launchOptions: [NSObject: AnyObject]?) -> Bool { // Override point for customization after application launch. self.window?.backgroundColor = UIColor.whiteColor() return true }


これが問題の原因です。クイックアクションを使用してアプリケーションを起動すると、 willFinish 、 didFinishが呼び出され 、次にperformActionForShortcutItemが呼び出されます。 したがって、最初に背景の白色を設定してから、青色に変更します。 明らかに、ユーザーがクイックアクションを使用してアプリケーションを起動するときに、背景色を白に設定することは望ましくありません。



この問題を解決するには、 didFinishLaunchingWithOptionsメソッドをチェックインする必要があります。 

 func application(application: UIApplication, didFinishLaunchingWithOptions launchOptions: [NSObject: AnyObject]?) -> Bool { print("didFinishLaunchingWithOptions called") var isLaunchedFromQuickAction = false // Check if it's launched from Quick Action if let shortcutItem = launchOptions?[UIApplicationLaunchOptionsShortcutItemKey] as? UIApplicationShortcutItem { isLaunchedFromQuickAction = true // Handle the sortcutItem handleQuickAction(shortcutItem) } else { self.window?.backgroundColor = UIColor.whiteColor() } // Return false if the app was launched from a shortcut, so performAction... will not be called. return !isLaunchedFromQuickAction }


アプリケーションがクイックアクションを介して実行されているかどうかを判断するには、 UIApplicationLaunchOptionsShortcutItemKey起動ソフトキーを確認します 。 UIApplicationShortcutItemオブジェクトは、起動ソフトキーの値として使用できます。 クイックアクションを使用してアプリケーションを起動した場合、 handleQuickActionを呼び出して背景を青に変更します。



didFinishLaunchingWithOptionsでクイックアクションを既に処理したため、 performActionForShortcutItemを呼び出してhandleQuickActionを再度実行する必要はありません。 したがって、falseを返し、performActionForShortcutItemメソッドを呼び出さないようにシステムに指示します。



以上です！ これで、アプリケーションをテストできます。 クイックアクションは正常に機能するはずです。



まとめ



3D Touchは、便利で楽しい機能をアプリケーションに追加する素晴らしい方法です。 ただし、すべてのデバイスが3D Touchをサポートしているわけではないことに注意してください。ただし、これは将来変更される可能性があります。
