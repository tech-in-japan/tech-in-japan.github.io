メールクライアントの電話番号

それは簡単なタスクのように思えます：手紙に電話番号を表示します。 何が複雑なのでしょうか？ クラシック 誰も単一の標準に到達することはできません。







その日の質問：



ネイティブを使用しますか、奇妙ですか、開発者に連絡しますか？ 私自身は混乱しているので、それには答えません。



番号を取る：

8（800）888-88-88



そして、私たちは倒錯します



オプション番号1：ネイティブ性 



a { color:#0077cc !important } a[href^=tel] { color:#000000 !important; text-decoration: none; border-bottom:#000000 1px dotted; } p { color:#000000; } <p>  <b>8 (800) 888-88-88</b></p>


何が得られますか：



-人気の郵便板には電話番号が表示されます。 mail.ruでさえ、以前はそうでしたが、番号を通話用のリンクに変換しません。

-Androidはまだ正直にテストしていません。

-Gmailアプリケーションは、通話のリンクを変換しません。 リンクは変更されません。

-iOSアプリケーション：Mail、Outlook、Mailbox、Spark、およびRamblerは、数値を変換し、[href ^ = tel]のスタイルを選択することで、必要に応じて擬似リンクの下で数値をスタイル設定します。

-iOSアプリケーション：Google Inbox、Yandex、Mail.ru、My.comは、通話の番号を変換し、リンクをデフォルトの嘔吐物の青にします

-Yahoo iOSアプリは、通話の番号を変換し、＃0077ccにリンクをペイントします。 つまり、CSSからデザインを取得しますが、リンクが電話番号であることを理解していません。

-すべてのOSXデスクトップアプリケーションは、擬似リンクの下に数字を様式化します。

-デスクトップWindowsクライアントはまだテストしていませんが、90％はリンク変換に関与しないと確信しています。



オプション番号2：奇跡 



 a { color:#0077cc !important } a[href^=tel] { color:#000000 !important; text-decoration: none; border-bottom:#000000 1px dotted; } p { color:#000000; } <p>  <b><a href="tel:+78008888888" style="color:#000000 !important; text-decoration:none; border-bottom:#000000 1px dotted;">8 (800) 888-88-88</a></b></p>


何が得られますか：



-Vambboards：Gmail、Google Inbox、Yandex、Yahoo-すべて順調です。 呼び出し擬似リンクの作業。

-Webmords：Outlook.comおよびMail.ru-リンクは様式化されていますが、鳴りません。 hrefパラメーターが空です。

-Webmord Rambler-リンクは様式化されています。 hrefパラメーターでは、次の真珠は「/ m / badurl」です。

-すべてのiOSアプリケーション。 パーフェクト。 Mail.ruおよびMy.comを除く。 彼らはまだ青い嘔吐リンクを設定します。 幸いなことに、彼は電話しています。

-すべてのOSXデスクトップアプリケーションは、擬似リンクで番号をスタイルします。

-WindowsデスクトップアプリケーションとAndroidアプリケーションはまだテストされていません。



オプション番号3：開発者をノックする



-Yandex、モバイルアプリケーションでメディアクエリのサポートを既に入力しています。 これについて壊滅的なことは何もありません。

-Mail.ru、これらはMy.comでもあります。メディアクエリのサポートを入力し、電話番号のハードフォーマットを削除してください。 Webmordのリンク切れの問題を確認してください。 モバイルアプリケーションの不器用なテキスト区切りを修正します。 転送アルゴリズムはやや落ち込んでいます。 このようなナンセンスなメールアプリケーションは1つもありません。

-Ramblerのすべては明らかです。 彼らは来年、新しいウェブ銃口を約束しました。 アプリケーションはまだ未加工ですが、それに関する苦情はほとんどありません。

-ヤフー。 新しいアプリケーションを展開しました。 苦情はありません。 [href ^ = tel]セレクターが理解しない限り。 そして、誰がここで私を聞くことができますか？

-マイクロソフト。 原則として、私はOutluk開発部門に感謝しています。なぜなら、私の人生のセックスは、彼らのおかげで何倍も正確になったからです。 しかし、ここには虚空への懇願があります。 わかりました、デスクトップの見通し。 誰もがこれに同意しました。 しかし、すべてのウェブモールに同じパーサーを配置することの問題は何ですか？ Outlook.com、Office365、Live.com。 すべてをOutlook.comに変換できないのはなぜですか？ ところで、電話リンクの状態を修正する必要があります。



誰も気にしないことは明らかです。 チケットは処理されません。 そして、処理されると、顔のないサポートエージェントから「ありがとう、行動します」のような購読解除を受け取ります。 握手の理論を試してみましょうか？ どうして？
