Favicon.icoまたはブラウザのアドレスバーのアイコン

私はすぐに、この記事が「最小の」カテゴリに属していることに気付きます。 私は彼女を許します、突然それは誰かにとって重宝します。



ファビコン、それは何で、何のためですか？ Favicon.icoは、Win32 OSのショートカットファイルであり、それが存在するサイトのページへのアイコンリンクです。 ブラウザの選択したリンク、アドレスバー、およびマルチウィンドウブラウザのブックマークに表示されます。 ページに固定する方法は？ アイコンを描画するには、任意のグラフィックエディターを使用する必要がありますが、その後、受信したグラフィックファイルを.ico形式に変換するか、AWIcons Pro、IconWorkshop、MicroAngeloなどの専用エディターで最初に描画する必要があります。 ファイル拡張子を.icoに変更することは、お勧めできません。 一部のブラウザはそのようなファイルを受け入れず、アイコンを表示しません。 Photoshopでの作業がさらに慣れている場合は、 ここで .ico形式の画像を操作するためのプラグインをダウンロードできます。 プラグインを機能させるには、ファイルを解凍し、PhotoshopルートディレクトリのPlug-Ins \ File Formatsフォルダーに保存する必要があります。



したがって、正しい方法で.ico形式に変換されたアイコンがあります。 さらにポイント。



1.アイコンの名前はfavicon.icoである必要があります。 古いブラウザの中には、サイトのショートカットとして認識できないものがあります。

2.アイコンのドキュメントへの接続は、次のコードによって実行されます。 コンテンツタイプを画像/ xアイコンとして指定し、ドキュメントへのリンクをショートカットアイコンとして指定する必要があります。 これはw3c仕様に従って必須であるという事実に加えて、コンテンツのタイプを指定せずに一部のブラウザーの機能に再び出くわす可能性があります。

3. 16x16から48x48の範囲で選択できるアイコンのサイズ。アイコンが大きいほど、ユーザーがデスクトップのサイトにショートカットを配置する場合に、アイコンがきれいになります。 しかし！ 私たちは「We All Love Internet Explorer」と呼ばれる現実に住んでいます。 IE 6.0以下では16x16ピクセルより大きいアイコンは単に無視されることを説明します。 ただし、サイズと内容が異なる複数の画像を1つの.icoファイルにパックすることは可能です。

4.ファイルをサイトアイコンと共に配置する場所 信頼性を高めるために、サイトのルートディレクトリに配置してください。



すくい。



1.上記のように、IE lt 6.0は16x16ピクセルを超えるアイコンを認識しません。

2. IE 6.0以下では、お気に入りにページを追加してリロードした後にのみアイコンが表示されます。 まあ、MSはすべてを再起動するのが好きです;-)。

3. IE 6.0以前およびそれ以降が表示されない場合は、ブラウザのキャッシュをクリアしてください。

4.それでもアイコンが本来あるべき場所に表示されない場合は、サーバーがブラウザにアイコンを送信して表示し、ファイルとして転送せずに保存することを提案することを確認する必要があります。 これは、ブラウザにWebサイトのアドレスとアイコンの名前（ yoursite.com/favicon.icoなど）を入力するだけで確認できます（もちろん、アイコンがルートディレクトリにある場合）。 2番目の場合、サーバーは.icoファイルを表示するように構成されていません。 この状況を修正するには、サイトのルートに.htaccessファイルを作成し、次のAddType image / x-icon .ico行を追加する必要があります。
