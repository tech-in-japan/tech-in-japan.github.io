YII2を使用したプログラミング：はじめに



翻訳者から。

人生はそんなものです。どういうわけか私はJava Springに関する一連の記事を始めました。コミュニティは、選択は最新の情報ではないと報告しました。 人生は筋金入りのC ++プログラミングに私を投げ込みましたが、私の魂はまだウェブを必要としているので、自由な時間に、あらゆるホスティングが食べることができ、同時に開発されたアプリケーションとOOPの複雑さがあまり影響を受けないテクノロジーを研究することにしましたPHP

翻訳は逐語的なふりではありませんが、重大なエラーがある場合は、コメントでお知らせください。

「YIIとは」と尋ねる場合は、以前のチュートリアル「YIIフレームワークの紹介」を読んでください。YIIの利点について説明し、2014年10月12日のフレームワークの第2バージョンの革新についても説明しています。

このレッスンでは、YII2のインストール、環境のセットアップ、古典的なHello、World！アプリケーションの作成、GitHubでのホスティングとデプロイのためのリモート環境のセットアップについて説明します。



YII2をインストールする



ドキュメント


詳細な手順はこちらで確認できますが、Mac OS Xの環境の簡単で一般的なインストールとセットアップについて説明します。 フレームワークには、詳細なドキュメントとクラスの説明があり、 YIIに関するすべての詳細が含まれています。

翻訳者から。 私は、順番に、アップルデバイスを持っていないし、Ubuntu（VMWare Playerの下で実行）の下ですべてのアクションを実行します。



Composerをインストールする


フレームワークを快適に使用するには、人気のあるPHP拡張マネージャーであるComposerを使用することをお勧めします。 まだインストールされていない場合は、次を実行します。 

curl -s http://getcomposer.org/installer | php mv composer.phar /usr/local/bin/composer




YII2をインストールする


Composerを使用してYII2をインストールします。 インストールするには、 GitHubにアカウントが必要です。アカウントがない場合は、登録してください。

最初のプロジェクトを「hello」と呼びましょう。 

 cd ~/Sites composer global require "fxp/composer-asset-plugin:1.0.0-beta2" composer create-project --prefer-dist yiisoft/yii2-app-basic hello


YII2は2つのタイプの開発アプリケーションをサポートします：シンプルと高度。 このレッスンでは、デフォルトでインストールされる単純なアプリケーションを使用します。 高度なアプリケーションテンプレートは、WordPressと同様に、フロントエンド、バックエンド、およびコンソールをサポートします。これは、cronの管理パネルおよびタスクです。



ローカル開発環境をインストールして構成する


OS Xフリーウェア用のオープンソースMAMPを開発に使用しています。 MAMPが実行され、そのWebサーバーとデータベースサーバーが実行されていることを確認してください。 次に、サイトを含むディレクトリへのシンボリックリンクを作成します。 

 cd /Applications/MAMP/htdocs ln -s ~/Sites/hello/ /Applications/MAMP/htdocs/hello


ここで、すべてを正しく構成してからlocalhost ：8888 / hello / webを開くと、 Bootstrap 3.xを備えたシンプルなYII2アプリケーションテンプレートが表示されます。



このページは、以前はネットワーク上で確実にその位置を占めていた最新バージョンのBootstrapがプリインストールされていないYII 1.1で開発していた開発者を驚かせるでしょう。



YIIアプリケーションアーキテクチャ


フレームワークの重要な機能の1つは、 MVCパターンのサポートです。 これにより、より構造化されたコードを作成して、理解しやすくしたり、セキュリティやコードの自動生成など、PHPで実装するのが難しい多くのことを簡素化できます。 また、コードの維持と再利用にも役立ちます。

YIIアプリケーションでは、すべてのトラフィックは単一の/web/index.phpファイルに向けられます。 このファイルはYII、依存関係をロードし、アプリケーションを起動します。 

 <?php // comment out the following two lines when deployed to production defined('YII_DEBUG') or define('YII_DEBUG', true); defined('YII_ENV') or define('YII_ENV', 'dev'); require(__DIR__ . '/../vendor/autoload.php'); require(__DIR__ . '/../vendor/yiisoft/yii2/Yii.php'); $config = require(__DIR__ . '/../config/web.php'); (new yii\web\Application($config))->run();


デフォルトでは、これは/controllers/SiteController.phpファイルのIndexメソッドです 

 public function actionIndex() { return $this->render('index'); }


通常、コントローラーには多くのメソッドがありますが、コードの量は最小限です。 コントローラは、アクセス、ロードモデル、およびビューを制御します。 主な意味の負荷はモデルにあり、ビューにはパターンのみが表示されます。

デフォルトでは、YIIは/views/layouts/main.phpを/views/site/index.phpのビューとして表示します。 main.phpは、ツールバーとコンテンツを含むhtmlおよびcssページ構造をサポートします。 main.php内では、コンテンツは次のように呼び出されます。 

 <div class="container"> <?= Breadcrumbs::widget([ 'links' => isset($this->params['breadcrumbs']) ? $this->params['breadcrumbs'] : [], ]) ?> <?= $content ?> </div>


$コンテンツ変数は、静的htmlページを表す/views/site/index.phpで生成されたコードに置き換えられます。



アプリケーション「Hello world！」を作成しています



美しいURLを含める


まず、mod_rewriteでいくつかのきれいなアドレスを有効にしましょう。 ホームページで[About]ボタンをクリックすると、次のようなアドレスが表示されます： localhost ：8888 / hello / web / index.php？R = site％2Fabout。 次に、このフォームをローカルホストの 8888 / hello / web / site / aboutに移動します。

configディレクトリには、Webアプリケーション、コンソールアプリケーション設定、データベース設定など、環境全体の設定が含まれています。 ファイル/config/web.phpを次のように変更します。 

 'components' => [ //... 'urlManager' => [ 'showScriptName' => false, 'enablePrettyUrl' => true ], //... 'request' => [


次に、index.phpが保存されている/ webディレクトリに.htaccsessファイルを作成し、次の内容を含める必要があります。 

 RewriteEngine on # If a directory or a file exists, use it directly RewriteCond %{REQUEST_FILENAME} !-f RewriteCond %{REQUEST_FILENAME} !-d # Otherwise forward it to index.php RewriteRule . index.php


mod_rewriteがMAMPで有効になっていることを確認してください。問題がある場合は、 stackoverflowのブランチを確認してください。

ブラウザーでlocalhost ：8888 / hello / web / site / aboutに移動します。これで、単純なYIIアプリケーションのAboutページが表示されます。



アクションを作成する


次のステップでは、メッセージを繰り返すために、Sayという名前のコントローラーのアクションを作成します。 /controllers/SiteController.phpファイルに次を追加します。 

 public function actionSay($target = 'World') { return $this->render('say', ['target' => $target]); }


この関数は、urlアドレスでターゲットパラメータを探して画面に表示します。 存在しない場合は、単に「ワールド」が表示されます。 YIIフレームワークでは、アクションはactionNameEkshenaのプレフィックスが付いたメソッドとして実装されます。 SiteControllerクラスでは、パブリック関数actionSayを使用してSayアクションを作成しました。

renderメソッドの標準的な動作は、views / ControllerID / ViewName.phpでファイルを検索することですので、/ views / site /にsay.phpファイルを作成しましょう。 

 <?php use yii\helpers\Html; ?> <h1>Hello <?= Html::encode($target) ?></h1> <p>Welcome to your Yii2 demonstration application.</p>


このテンプレートは、文字列「Hello + $（target）」を出力します。ここで、$（target）はurlに渡される変数です。 セキュリティを確保するために、それは直接行に落ちませんが、最初に悪意のあるコードがないかチェックされます。

それでは、GitHubにアプリケーションをアップロードして、運用サーバーで実行してみましょう。

GitHubにリポジトリを作成する


基本的なYIIテンプレートをインストールしたら、 GitHubにリポジトリを作成します。



注：Yit .gitignoreをGitHub（https://github.com/yiisoft/yii2/blob/master/.gitignore）からローカルの〜/ Sites / hello / .gitignoreフォルダーにコピーして貼り付けるのは、後で手動で削除するよりもはるかに簡単です。 どちらのオプションも有効ですが。

GitHub Mac OS Xアプリケーション（https://mac.github.com/）を使用していますが、コマンドラインを使用してリポジトリを初期化することもできます。 たとえば、次のように（「youraccount」をGitHubの名前に置き換えます。 

 cd ~/Sites/hello git init git commit -m "first commit of hello world for yii2" git remote add origin git@github.com:youraccount/hello.git git push -u origin master


翻訳者から。 Bitbucketをリポジトリとして使用することを好みます。これにより、プライベートリポジトリを無料で使用できるようになります。

サーバーでApacheを構成する

サーバーをまだ構成していない場合は、Ubuntu（http://jeffreifman.com/digitalocean）でサーバーを構成する手順を確認することをお勧めします。 IPアドレスがある場合は、サーバーのDNSを構成します。 

 yourdomain.com A 192.161.234.17


IPまたはドメイン名を使用して、ssh経由でサーバーにリモートでログインします 

 ssh root@yourserver.com


構成を更新し、Gitをインストールし、Apacheのmod_rewriteとPHPのmcryptを有効にします。 

 sudo apt-get update sudo apt-get upgrade sudo a2enmod rewrite php5enmod mcrypt


GitHubへのリモートサーバーアクセスのセットアップ

次に、GitHubを使用してプルリクエストを設定します。 Git Firstをインストールする 

 sudo apt-get install git


次に、wwwのグループを作成し、データとGitHubユーザーを追加して、それらの権限を構成します。 

 adduser github addgroup www adduser github www adduser www-data www chown -R :www /var/www chmod +s -R /var/www chmod -vR g+w /var/www/


次に、リポジトリと同期するキーを作成します。 

 su github ssh-keygen -t rsa -C "yourgithubaccount@youremail.com" exit cat /home/github/.ssh/id_rsa.pub

Apache用のサイト構成の作成

構成ファイルを作成します。 

 nano /etc/apache2/sites-available/hello.conf


ドメイン名を次のように変更します。 

 <VirtualHost *:80> ServerName yourdomain.com # Set document root to be "basic/web" DocumentRoot "/var/www/hello/web" <Directory "/var/www/hello/web"> # use mod_rewrite for pretty URL support RewriteEngine on # If a directory or a file exists, use the request directly RewriteCond %{REQUEST_FILENAME} !-f RewriteCond %{REQUEST_FILENAME} !-d # Otherwise forward the request to index.php RewriteRule . index.php </Directory> </VirtualHost>


サイトを有効にします。 

 a2ensite hello.conf a2dissite 000-default.conf service apache2 reload
