行を更新します。 Google Chrome 7 Stableに会う







そのため、加速リリースカレンダーによると、GoogleはWebブラウザーの7番目のバージョンをStableチャンネルに移行しました。 そして、残念なことに、このアップデートは個人的にはあまり喜ばしいことではありません。 もちろん、開発者はブラウザを積極的に見て、毎日のビルドだけでなく、1時間ごとのビルドをリリースし、機能を変更および追加しましたが、大きな変更は加えませんでした。 Googleは数字を追いかけ、各バグ修正に新しいメジャー番号を付けているようですが、私の意見では、拡張機能の発売以来何も革命的なものが追加されていないため、変更は6.1、または4.5でも引っ張っています。 オペレーティングシステムとブラウザのハイブリッドを待っているので、8番目のブランチに特に期待しています。 はい、同じChromium OS（わずかに切り捨てられたサイズ）。最近静かにRelease Candidateステージに移行しました。



したがって、第7バージョンの変更点：

1. 数百のバグが修正されました。

2. 新しいHTML5解析アルゴリズム。 HTML5仕様が出現する前は、ブラウザがエラーのあるマークアップを処理する方法に関する標準はありませんでした。 WebKitに組み込まれた新しいアルゴリズム（Safariの将来のバージョンでは同じアルゴリズムが使用されることを意味します）が登場し、Firefox 4でも使用されるようになり、ブラウザーはエラーページを等しく表示します。 したがって、ページとブラウザ間のより良い互換性が提供されるだけでなく、ボーナスとして、SVGとMathMLをHTMLに埋め込む機能も提供されます。 SVGを埋め込むには、HTMLページにタグを追加するだけで、SVGの全機能を使用できます。

3. ファイルAPI 。

4. 入力タグを使用した「ディレクトリのアップロード」。 これで、いくつかのサービスにファイルをアップロードできますが、ディレクトリ全体にファイルがあります。 新しいタグのおかげで、サービスはフォルダーの内容を読み取り、それらを自分自身にダウンロードできるようになります。

5. Cookieの保存例外を指定する機能。



したがって、Google ChromeにはOperaのサイトに欠けている設定の完全な類似物があります。 特定のリソースが、サイトで多数ブロックされている非常に匿名化されたファイルを保存することを禁止できます。

6. 多くの脆弱性を解決します 。



Ubuntu Linux 10.10、Intel Core 2 Duo 2.26 GHz、2Gb RAMでいくつかのテストを実施します。

1）。 Google Chrome 7は依然としてHTML5サポートのトーンを設定しており、残りのブラウザーははるかに遅れています。



2）。 V8ベンチマークでは、Chromeはずっと前に進んでおり、高い結果を示しています。 7番目のバージョンは例外になりませんでした。





一般的に、最初と最後のサポートを約束しましたが、WebGL /ハードウェアアクセラレーション/印刷プレビューのサポートはまだ受けていませんが、ほとんどの場合はFAILよりも勝っています。 どうやら、グーゲルはザガシュニクにあるものを見せることを決めたが、今のところそれを広げないことに決めた。 また、プラグインを起動するためのビルトインブロッカーが含まれていなかったため、ユーザーが起動するまでページのFlash / Silverlightコンテンツの起動がブロックされていました。
