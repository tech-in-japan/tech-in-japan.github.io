できないことと比較してください：安価なホスティングとVMwareスタック上のクラウド

この投稿を書くことにした理由は、ホスティングディスカウンターサービスと企業クラウドプロバイダーサービスの大きな違いを理解していないためです。 もちろん、論理的な質問は次のとおりです。「クラウドサービスはなぜ高価なのですか？」 クラウドサーバーは、価格で「99ルーブル/月」の条件付きホスティングに「負け」ますが、「タンジェリンのスライスのリンゴ」ではなく「リンゴとリンゴ」を比較する必要があります。 これとそれの両方を試すことができるのは良いことです。最終的には、「自分の好み」に合ったものを理解しました。







ビジネスとしてのホスティングは比較的簡単です。 作成と開発にはいくつかのオプションがあります。 大規模なホスティング会社の再販業者になるか、専用サーバーをレンタルするか、独自の機器を購入してデータセンターに配置することができます。 また、Webインターフェースを介してWebサーバー、データベースサーバー、メールサーバー、その他の関連プログラムなどのソフトウェアを管理できるWebホスティングコントロールパネルを選択、インストール、および構成する必要があります。 技術サポートの問題を解決し、マーケティングを開始し、顧客を見つけることは依然として残っています。 そして、それは特別なものではないようですが、条件付きの「99ルーブル/月」という競争価格を満たす必要があります。つまり、節約する必要があります。



ホスティングディスカウンターとビジネスリスクは何を節約しますか？



低コストの航空会社は、安価で問題の多い機器を使用します。 高価なサーバーとストレージを購入し、同時に競争力のある価格を満たす方法は？ Habrahabrでは、主要なプロバイダーが古いディスクを新しいディスクに交換し、同じ古​​いディスクを新しいディスクとしてインストールする要求を拒否した場合について読むことができます 。 ディスクは「落下」し、データが失われます。 原則として、誰もが自分の企業データの価値を独立して評価できます。これは、原則として、節約された金額を少なくとも1桁上回っています。



バックアップにより、問題を部分的に解決できます。 しかし、すべての安価なホスティングプロバイダーがこのサービスを提供しているわけではありませんが、データのコピーを作成することを提供する場合、作業データが保存されているディスクと同じディスクになる可能性があります。 ディスクがクラッシュした場合、データとバックアップファイルは失われます。



ホスティングの2番目の重要なコンポーネントは、テクニカルサポートです。 24時間のサポートサービスには、資格のある従業員の雇用と多額の給与基金が必要です。 ホスティングオーナーが自分でホスティングをサポートすると決定した場合はどうなりますか？ また、スリープする必要があるため、24 * 7モードはキャンセルされます。 問題は間違ったタイミングで発生し、顧客は待たされて損失を被ることになります。



この市場セグメントでのホスティングプロバイダーの選択は、無料試用期間がないため複雑です。 クライアントは最初にサービスを試してから支払いをすることができません。 誰かが幸運でホスティングに問題はないかもしれませんが、低コストの航空会社は100％の前払いを求めているため、最初にリスクを負う必要があります。 失敗した場合、返金プログラムは金銭的損失を部分的にカバーしますが、費やした時間はカバーしません。



ビジネスには、多くのホスティング割引業者が満たすことができないニーズがあります。 ホスティングを選択するときは、確認する必要があります



  • ホスティングは、ロシアからの法人の支払いおよび締め切り書類の請求書を提供しますか。
  • プレインストールされたオペレーティングシステムの選択肢があり、独自のOSを個別にインストールすることは可能ですか？
  • 追加のIPアドレスが提供されるかどうか。
  • 近隣への攻撃は一般的なダウンタイムにつながる可能性があるため、ITインフラストラクチャに対するDDoS攻撃に対する保護はありますか。


したがって、安価なホスティングには保証がありません。 これは、すべてが何かまたは他人の費用によるものであるという経済システムの配置方法です。 顧客が「99ルーブル/月」だけを支払う場合、商業活動の量でサービスを提供する方法はありません。 そのため、機器、技術サポートを節約し、狭いソフトウェア機能に限定する必要があります。



このサービスが価格と満足するニーズの理想的な比率になるプロジェクトがあります。 ただし、より高い信頼性、セキュリティ、および柔軟性が必要な場合、非常に高い価格で、より高い消費者価値/価格比を持つクラウド製品があります。 もちろん、このようなサービスの比較は、安価なホスティングではなく、同様のサービスで行われます。



クラウドプロバイダーのビジネスがどのように機能するかをよりよく理解するために、SDDCの概念を検討することから始めます。



SDDCまたはSoftware Defined Data CenterのFAQとは



ソフトウェア定義をロシア語に翻訳すると、「ソフトウェア定義」または「ソフトウェア構成可能」という用語が得られます。 ただし、「ハードウェア」から「ソフトウェア」へのフォーカスシフトの傾向は、最初のオプションをより適切に説明するため、主なオプションとして使用します。



そのため、Software Defined Data CenterまたはSDDCはソフトウェア定義のデータセンターです。 このコンセプトは、2012年にStephen Herrodによって最初に提案されました。当時、彼はVMwareのCTOを務めていました。



SDDCコンセプトのこのような遅れた出現は、IT業界で行われたプロセスに関連しています。 1950年代には、ハードウェアのコストは、ハードウェアへの追加のみと考えられていたプログラムのコストよりも計り知れないほど高かった。 つまり、最初のタスクはハードウェアソリューションを開発することであり、セカンダリはソフトウェアです。







徐々に、コモディティ化のプロセス、またはより普遍的で互換性のあるコンピューターの作成への移行により、プログラムはどのコンピューター機器でも使用できることが理解されるようになりました。 その結果、80歳までにソフトウェア開発は独立した業界になり、ソフトウェアのコストはまず「ハードウェア」の開発コストに近づき、次に場合によってはそれを超え始めます。 これにより、ソフトウェアを優先してハードウェアの優位性が解消されます。 特殊な機能はすべてソフトウェアに転送され、ハードウェアはより汎用的になります。 データセンターシステムのすべての要素に及ぶプロセスにより、ソフトウェア定義のデータセンターまたはSDDCを作成するための前提条件が作成されます。



SDDCへの道を開いた最初のイベントは、ソフトウェア定義ネットワーク（SDN）の概念と、それ以降のソフトウェア定義ストレージシステム（Software Defined Storage、SDS）の理由でした。 物理サーバーに基づいて必要な特性を備えた仮想マシンをエミュレートできるサーバー仮想化テクノロジーと併せて、ネットワークおよびストレージ仮想化により、現在SDDCと呼ばれるテクノロジースタックを作成することができました。



このようなインフラストラクチャの編成により、柔軟性と容易なスケーラビリティを必要とするクラウドテクノロジーの開発が促進されました。 クラウドは、インターネットを介したリモートアクセスを備えたさまざまなITサービスおよびリソース（XaaS）を提供するモデルとして、静的ハードウェアを参照せずにコンピューティングリソースの展開、再構成、および監視を自動化するテクノロジーを必要としていました。 この機会は、ソフトウェアデファインドデータセンターのテクノロジースタックに含まれるソフトウェアによって与えられました。



VMwareスタックおよび安全なデータセンター



企業のクラウドプロバイダーであるCloud4Yは、VMwareの独自ソフトウェアのスタックを使用してクラウドのソフトウェア定義インフラストラクチャ（Software Defined Infrastucture、SDI）を作成するため、例として使用します。 ソフトウェアの高コストにもかかわらず、エンタープライズクライアントに必要な信頼性と安定性を提供します。



プラットフォームの中心にあるVMware vSphereは、多くの専門家によると、エンタープライズインフラストラクチャの業界標準です。 vSphereには、ESXiホストとストレージ管理サーバー、いわゆるvCenter Serverが含まれています。 ESXiホストは、それぞれ独自のサーバー上で仮想マシンをエミュレートするVMware ESXiハイパーバイザーを備えた物理サーバーです。 vCenter Serverは、ESXiホスト自体を「クラッシュ」し、グループ内の各ホストのすべての機能を管理できるだけでなく、バ​​ランシング（DRS）、ライブマイグレーション（vMotion）、クイックリカバリのオプションを備えたクラスターを作成できる「コマンドポイント」ですフォールトトレランスを向上させるためのスタンバイホスト（HA）上の仮想マシン。



また、クラスターを構築するには、ストレージシステムを接続する必要があります。 Software Defined Storage（SDS ）データ用のVMwareソリューションも利用可能です-Virtual VolumesおよびVirtual SAN。



特別なクラスターファイルシステムでは、クラスター内のすべてのESXiホストが読み取りおよび書き込み可能な仮想マシンファイルを含むセクションが保存されます。 1か所にストレージがあり、仮想マシンが物理プラットフォームから独立しているため、 HA、DRS、FT、vMotionで迅速な移動と復旧が実現します 。



最小限のSDDC実装については、ソフトウェア定義ネットワーク（ Software Defined NetworkまたはSDN ）向けのVMwareソリューションに言及する必要があります。 VMware NSXは、ソフトウェアレベルでネットワークモデルを再現します。 NSXネットワークは、論理スイッチ、ルーター、ファイアウォール、ロードバランサー、VPNなどの論理ネットワーク要素とサービスのライブラリです。 vSphereプラットフォームの一部であるこのソリューションのおかげで、プロバイダーの管理者とクラウドのテナントは、ネットワーク上の仮想マシンを任意のトポロジに迅速に統合できます。



クライアントはVMware vCenter Serverを使用しません。 プロバイダーは、クラスターと物理機器の管理を担当します。 顧客は、 すでに説明した便利なVMware vCloud Directorセルフサービスポータルを使用して、仮想データセンターを大幅に制御できます。 クライアント用のSDDCまたはvCODAの作成は、1つのvCloud Directorパネルからできるだけ早く行われ、必要な特性とオペレーティングシステムを備えた必要な数の仮想マシンを作成してから、それらを内部ネットワークまたはハイブリッドクラウドに結合できます。







vCloud DirectorとvCenterは、自動化と管理を担当するSDDCスタックの一部です。これが主な理由で、このようなソリューションが人気を集めており、多くのプロセスを自動化して仮想マシン群を管理する単調なルーチンからITプロフェッショナルを解放しています。







また、多くの人は、情報セキュリティツールを割り当てる必要があると考えています。情報セキュリティツールは、仮想化して、サービス（ ソフトウェア定義セキュリティ、SDsec ）の形式で提供する必要があります。 将来的には、これについて別の記事を作成しますが、ここでは定義を示して要約します。



SDDCは、プロセッサ、ストレージシステム、ネットワーク機器、情報セキュリティ、自動化および制御ツールなどのインフラストラクチャコンポーネントで構成されるソフトウェアを使用して、データセンター全体を定義する統合された抽象化レベルであり、サービスの形で消費者に提供される仮想リソースの共通プールが提供されますXaaS）。






VMwareテクノロジースタックを使用してCloud4YのSDIクラウドを実装すると、物理サーバーをレンタルするか、独自のサーバーを購入することにより、従来のデータセンターを使用して必要な顧客インフラストラクチャオプションを実装する場合と比較して、サービス品質（QoS）を改善し、コストを削減できます。 営業費用を優先する資本コストから顧客を拒否し、時間単位で請求できる可能性がある実際に使用されたリソースのみを支払うことにより、収益性が達成されます。 Cloud4Yは、SLAで修正された月あたり99.982％の可用性レベルを確保するための機器のコモディティ化にもかかわらず、信頼できる機器（サーバー（HPE）、ネットワーク機器（Cisco、ジュニパー）、オールフラッシュストレージ）を使用します。 装置は、通信チャネルの複製を備えた光リングで接続されたTIER IIIデータセンターのネットワークに配置されています。







すべてのリソースは専用です。 システムのCPUとRAMに許容可能な最小のパフォーマンスインジケーターを設定します。 vCPUあたりのMIPSの数は少なくとも2,900です。これにより、指定されたプロセッサ速度が保証されます。 また、物理RAMの「オーバーサブスクライブ」は許可されていません。RAMスワップは0％です。 つまり、ゲストOSに表示される仮想マシンの作成中に割り当てられる構成済み仮想RAMは、仮想マシンがいつでも利用できる100％割り当てられた物理メモリです。 実際には、このアプローチは、オペレーティングシステムがRAM内のブロックにアクセスし、迅速な応答を期待している場合にクラウドサーバーのパフォーマンスの低下を回避しますが、これらのブロックは他のクライアントの負荷のためにハードディスクに読み書きされ、速度が大幅に低下します。



VPNを備えた保証された5 Mbpsの無制限チャネルと、最大1000 Mbpsに拡張する機能が無料で提供されます。
