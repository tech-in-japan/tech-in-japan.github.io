NXP-Freescaleの組み込み電子機器向けのKinetisマイクロコントローラ。 自分で発見する



時々、何かを変える必要性も私をカバーします。 そして、ほとんどの場合、私は自分の設計でマイクロコントローラーファミリーを変更します。 そして、私はこれだけではありません。 毎年、少なくとも50％の開発者が次のプロジェクトを実行するプロセッサを変更しています。 今回は、Kinetisファミリーを試してみることにしました。



Kinetisは、最大240 MHzのコア周波数を備えた、MMUなしのARM Cortex-M0 / M4 / M7プロセッサコアに基づく低消費電力の32ビットマイクロコントローラのファミリです。 MMUがないことは、KinetisがWindows、QNX、Android、iOSなどのオペレーティングシステムを対象としていないことを意味します。 これらのマイクロコントローラは、心臓インプラントから電気自動車のコントローラまで、さまざまな組み込みデバイスの一部としてハードリアルタイムシステムコードを実行するように設計されています。 Kinetisで実行するLinuxビルドがありますが。



Kinetisはどこで使用できますか？



フリースケールは多くのアプリケーションをリストできます。 しかし、私にとっては、産業用コントローラー、タッチ制御なしのベクトルを備えた周波数コンバーター、サイリスターレギュレーター、電圧コンバーター、ネットワークアナライザー、エレベーター、エレベーター、コンベヤー、その他多くのドライブを備えた制御システム用のプログラマブルロジックコントローラーを特定しています。 Kinetisの有望なトピックは、モノのインターネットとスマートセンサーです。 そして最後に、マルチコプターや他のロボット工学で試してみるといいでしょう。



Kinetisの興味深い点は何ですか？



Kinetisに最も近い競合他社は、 STM32F4 、 LPC4000 、 XMC4000 、 SAM4Sおよびいくつかのあまり知られていないファミリーのマイクロコントローラーです。 しばらくの間、競合他社もARM-Cortex-M7コアを使用したソリューションを提供してきました。 一般的に、彼らも静止していません。



2014年の組み込み市場調査によると、世界で最も人気のある32ビットファミリはSTM32です。 したがって、STM32にはないKinetisの機能にさらに重点が置かれます。



次の図では、MK70FN1M0VMJ15チップのKinetisファミリのシニアメンバーのブロック図を示しました。 これは、Kinetisの特徴を強調するために、フリースケールの説明で慣習的に描かれているものよりも少し詳細です。 括弧内には、会社の説明で使用されるブロックの名前が示されています。





しかし、Kinetisの主な利点：



-命名法の幅 。 命名法の面では、KinetisはSTM32に劣らず、さらに優れています。 Kinetisには、ARMコア上に世界最小のチップ（2.0 x 1.6 mm）と256ピンを備えた高度なMK70チップがあり、STMとは異なりDDRAMを接続できます。



-厳しい動作条件への優れた適応性 。 このファミリには、最大5.5 Vの供給電圧のチップがあります。特に電磁干渉に対する耐性が向上した車載アプリケーション向けのチップがあります。 さらに、ファミリ全体のマイクロコントローラの周辺には、ノイズ耐性を高めるために多くのハードウェアソリューションが実装されています。 障害に対する耐性は以下を提供します。

-最大5つの独立したクロック、そのうちの3つは外部水晶振動子にあり、

-ウォッチドッグタイマー用の独立したジェネレータ、

すべてのデジタル入力での2種類のハードウェアフィルター、

-さまざまな周辺機器へのアクセス、タスク権限を管理する機能、

-さまざまなイベントに対する不正アクセス検出器、

-個別のウォッチドッグタイマーによる外部デバイスの特別なリセット信号、

-電源をオフにする前に、いくつかの周辺機器をスナップして後続の変更から保護する機能

バススイッチの柔軟な管理と優先順位付け

など



-広範なプログラミングとデバッグのメカニズム 。 デバッグの柔軟性と利便性は、ARM Cortex Mコアに基づくすべてのマイクロコントローラーの共通の美点です。前述の調査が示すように、作業中のすべてのツールの開発者はデバッグツールに最も注意を払います。 そして、STM32の人気が高いとしても、STM32ベースのデバッグボードにすぐに統合されるこれらのチップのJTAG / SWDデバッガーの可用性に起因すると思います。 Kinetisには、特別なシリアルSPI互換プログラミングインターフェイスであるEzPortもあります。 EzPortを使用すると、JTAGアダプターを使用せずにチップを高速でプログラムでき、マルチプロセッサーシステムでのチップの再プログラミングも簡単になります。 LPCやSTM32のような隠しダウンローダーや隠しコードはありません。



-大量のアプリケーションライブラリ 。 フリースケールのアプリケーションライブラリは、その徹底性とオープン性によって区別されます。 ZigBeeプロトコルを使用するためのライブラリは何ですか？ これはおそらくソースで利用可能な最高のZigBeeスタック実装です。 次に、もちろん、エンジン制御ライブラリが登場します。 フリースケールは現在、Kinetisでのエンジン管理システムの開発のために、更新および拡張されたソフトウェアパッケージをリリースする予定です。 グラフィカルユーザーインターフェイスeGUIを構築するためのライブラリ、タッチパネルとキーボードを開発するためのライブラリ、医療通信プロトコルをサポートするためのライブラリ、セキュリティ標準IEC 60730-1：2010をサポートするためのライブラリ、ハードウェアアクセラレータの暗号化機能のために最適化されたライブラリ、クラウドサービスに接続するためのプロトコルのライブラリがありますなど また、以下にリストされているKinetisのリアルタイムオペレーティングシステムに付属するソフトウェアは含まれていません。 それとは別に、さまざまなブートローダー、および開発者にとって非常に役立つFreeMasterツールについて言及する価値があります。FreeMasterツールは、ロガーとコマンド制御の機能を備えたマイクロコントローラーに組み込まれたオシロスコープモジュールの実装です。



-効果的な無料の開発ツールの可用性 。 無料の開発ツールがなければ、このKinetisの記事はほとんど意味がありません。 はい、Kinetisには、EclipseプロジェクトとKinetis Design Studio IDEと呼ばれるGCCコンパイラに基づく非常に開発された無料の開発環境があります。 この強力な環境には、高品質のソースコードエディター、コンパイラ、基本ライブラリ、マイクロコントローラー周辺グラフィカルコンフィギュレーター（STM32マイクロコントローラー用のSTM32CubeMXのアナログ）、テンプレートプロジェクトとソーステキストの自動ジェネレーター、すべての周辺機器用のドライバーのフルセットを備えたSDK、さまざまなベースのシンボリックデバッガープログラマーが含まれますオペレーティングシステムとリアルタイムオペレーティングシステムを使用するプロジェクトジェネレーターをデバッグするためのプラグインを備えたJTAG / SWDアダプター。 Kinetis Design Studioで商用のKeilまたはIAR IDEを使用している場合は、プロジェクトとソーステキストをこれらのIDEに自動的にエクスポートし、Kinetis Design Studioで同時にサポートできます。

-幅広い市販のリアルタイムオペレーティングシステム 。 リアルタイムオペレーティングシステムは、それなしでは現代の組み込み電子機器が考えられないものです。 信頼性を損なうことなく、1つのデバイスに膨大な数の機能を統合し、その競争力を確保することができるのは、それらです。 ここで、Kinetisには、私が魅力的な議論と呼ぶものがあります-MQXオープンソースリアルタイムオペレーティングシステム 。 実際、キネティスに対する私の関心は彼女から始まりました。 しかし、ほとんどの開発者はFreeRTOSを好みます。 FreeRTOSには、Kinetis Design Studioでプロジェクトを生成する機能もあります。 さらに、よく知られているオペレーティングシステムuCOS-IIおよびuCOS-IIIのプロジェクトを生成できます。 すでに述べたように、Kinetis用のLinux OSビルドがあります。 mbed.orgプロジェクトサイトから、Kinetis用のmbed OSオープンソースオペレーティングシステムをダウンロードできます。 そしてもちろん、Kinetisに適合した少なくとも12個の商用オペレーティングシステムを見つけることができます。



MQXについて簡単に説明します。



このリアルタイムオペレーティングシステム（以降RTOS）の最初の機能は、Kinetisファミリ用の異常に設計されたドライバーレイヤーです。 ほとんどすべてのインターフェイス用のドライバーがあり、通常、ポーリングの使用と割り込みの使用の2つの実装があります。 2番目の機能は、このRTOSのマルチプロセッシングです。 多くのマイクロコントローラで実行されるタスクの透過的な相互作用のためのプロトコルスタックとルーティングが含まれています。 この機能は、複雑なロボットシステムに非常に関連しています。 3番目の機能は、このRTOSに付属する高度なミドルウェアです。
