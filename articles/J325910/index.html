初心者向けのログの収集と分析

これがどれほど明白に見えても、ITインフラストラクチャとログの監視をどのように編成するかという問題は非常によく発生します。 そのため、パートナーの1人に経験を共有し、Log Analyticsサービスについて話すよう依頼しました。 著者に発言権を与えます。







この記事では、 Log Analytics Azureについて説明します。 これは、Microsoftの非常によく知られているログ分析およびインフラストラクチャ監視ソリューションです。 System Center Operations Managerのこのような独特のクラウドアナログ。 多くのベンダーがcな名前でサービスを呼び出すという伝統に反して、マイクロソフトはそれを際立たせ、製品に完全にシンプルで理解しやすい名前を付けました。



このサービスには、有料版と無料版の2つのバージョンがあります。 支払いなしで、1日あたり500 MBを超えるログを分析し、7日間を超えて保存することはできません。有料版ではこれらの制限が削除され、ログのギガバイトごとに1か月あたり143ルーブルが請求されます。



このサービスはクラウドに配置されているため、サーバーパーツのインストールと構成に関して、消費者から何も必要としません。



このサービスでできることを簡単に分析しましょう。 まず、 Microsoft Azureのアカウントが必要です。 さらに、 Azure Marketplaceでは、このサービスを[ 監視+管理]セクションで簡単に見つけることができます。







作成して、どのように機能し、何ができるかを見てみましょう。



監視マシンをWindows、Linux、Azureリソース（当然）、System Center、およびWindowsテレメトリーに接続できます。 一般的に、すべてがシンプルで、マシンにエージェントを置いて楽しんでいます。







WindowsおよびLinux、IIS、Syslogのログとカウンターからデータを取得し、カスタムログなどを作成できます。 一般的に、これまでのところすべてが美しいです。



コンピューターグループを作成し、Active Directory、WSUS、およびSystem Center Configuration Managerからインポートできます。





インフラストラクチャの問題のレポートと通知を受信するように、または自動化の目的でアラートを設定できます。 便利で、シンプルで上品です。
