ミュージシャンの観点から見たWindows 7

Windows 7 RC（7100）は長い間ネットワークに登場し、多くのレビューがありましたが、ミュージシャンとして私はそれらから何も得ることができなかったので、長い間オンまたはオフにすることをoffしましたか？ 必要なソフトウェアはそこで動作しますか？



システム。



テストは、特別な機器（ミディキーボードを除く）を使用せずに、「裸の」コンピューターで実行されました。

マット ボード： Gigabyte GA-965P-S3（rev。3.3）。

プロセッサー： Intel Core 2 Duo 1.8 Ghz。

サウンド：ビルトインRealtek ALC883。

OS： Windows 7 RC（ビルド7100）ロシア語版。

Midiキーボード： M-Audio KeyRig 49。



テスト中は、キャスパー、デーモンツール、uTorrent、KMPlayer、およびデフォルトで有効になっているすべてのサービスとプロセスもアクティブでした。



使用したソフトウェアから：

  1. Ableton Live 8。
  2. Ableton Live 7。
  3. Propellerheadsの理由4。
  4. Adobe Audition 3。
  5. Kontakt 2 + Prominy Clean Guitars（VST）。
  6. Toontrack ezDrummer（VST）。
  7. ギターリグ3（VST）。
  8. Amplitube Metal（VST）。
  9. Amplitube Fender（VST）。


上記から、すべてが機能します。



一般的に。



一般的に、「セブン」が決定します。 私はインターフェースが本当に好きで、速度と安定性も最高です。 ほぼ1週間、Windowsは決して「崩壊」せず、母親にレポートを送信することさえ要求しませんでした。 彼女が想定した唯一のことは、プログラムの一部が「おそらく正しくインストールされていない」ことでしたが、プログラムが正しくインストールされ、意見の相違はなかったと彼女は知ることができました。



長所



  1. ドライバーの手間はかかりません。 Windowsはそれ自体ですべてを見つけましたが、MIDIキーボードを検索する必要はありませんでした。 すべてが箱から出して動作します。
  2. たとえば、Vistaで組み込みの（または不器用な外部）サウンドカードを使用しているときに、弱い遅延を観察せず、まったく何も観察しなかった場合は、Asio4ALLドライバーは必要ありませんでした。 ミディキーボードも強打で動作します。 Reason 4、Live 7、およびLive 8でテスト済み。
  3. Reasonのグリッチは、Reasonが何にも接続できなかったときに消えました。 以前はパッチで処理されていました。


短所



  1. Live 8では、Amplitubeのプラグインが1つも動作しません。愚かにも「見え」ません。 おそらくこの不具合はWindowsではなく、8番目のLiveにあります。
  2. Kontakt 2は奇妙に動作しますが、再び8番目のLiveでのみ動作します。 奇妙なことは、「バンク」がゲーム中に完全にロードされるという事実にもかかわらず、いくつかのサウンドが再生されないということです。
  3. Guitar Rig 3にいくつかのバンクをロードすると、ホストプログラムがフリーズします。 繰り返しますが、Live 8。


結果。



では、何がありますか？ 多くのプラスがあり、短所は非常に疑わしいです、私はこれがWindowsに起因することを非常に疑います。 Windows 7をまだ信じておらず、Release Candidateという言葉が怖い人のために、もう少し議論することができます。ここに、私のマシンで使用した音楽以外のソフトウェアのリストを示します。 すべてがバタンと動作します：
