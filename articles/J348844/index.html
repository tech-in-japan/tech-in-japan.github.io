新機能と重要な変更を含む3CX v15.5 Update 3リリース



先日、3CX v15.5 Update 3をリリースすると同時に、新しいライセンスを導入しました-4〜16の同時コールからの3CX Standard（仮に最大50人の従業員の組織）。 これらのライセンスは、PBXエディションと呼ばれる以前の無料の3CXエディションのすべてのユーザーに完全に無料で発行されます。



新しい無料の標準ライセンスは、無制限の数のSIPトランク（外部回線）と内線番号（ユーザー）、Web会議への25人の同時参加者、およびGoogle Cloudでの無料ホスティングを提供します！ 現在のすべてのPBX Editionライセンス（更新のアクティブなサブスクリプション、つまり有効期限が切れていないもの）は、標準の年間ライセンスである16の同時通話に自動的に変換されます。



組み込み3CX WebMeeting Web会議システムの同時参加者の数も増加しました。標準では最大25人、PROでは最大100人、エンタープライズでは最大250人です。



Grandstreamの携帯電話（現在のところベータ版）の公式サポートを開始しています： GXP16XX 、 GXP17XX 、およびGXP21XX 。 以下の機能が実装されています。



  • SBCユーティリティおよびRPSサーバーを介した、ローカルネットワークでの自動構成（PnPおよび手動構成）

  • 証明書を暗号化しましょう

  • UaCSTAプロトコルをサポート

  • アプリケーションからの自動ダイヤル（CTIテクノロジー）

  • 3CX認定ファームウェアアップデート







前回のUpdate 3 Betaレビューで他のアップデート機能について説明しましたが 、もう一度繰り返しましょう。



  • Zoho CRMのサポート

  • 独自のCRMシステムのサポート

  • 新しいホスティング管理者アカウント

  • 音声会議を作成および実施するためのインターフェースの改善

  • Khomp Voice Gatewayのサポート

v15.5 Update 3へのアップデート

  • 3CXサーバーインターフェイスで静的IPアドレスを設定する

  • 利用可能なすべてのWindowsアップデートをインストールします。

  • ウイルス対策を一時的に無効にするか、スキャンからディレクトリを除外します。

Grandstream電話の「サポート」とはどういう意味ですか？



サポートとは、基本的なSIP通信に加えて、3CXが高度なデバイス管理をサポートすることを意味します。



  • BLF-GXP1610 / 15/20/25インジケータにはこれらがありませんが、残りは3CXインターフェイスから中央で設定されます。

  • 音声コーデックの優先度管理-3CXから集中的に

  • 電話ディスプレイの言語-3CXインターフェースで一元的に設定

  • タイムゾーン、日付と時刻の形式-3CXインターフェースから集中的に

  • 電話のWebインターフェイスにアクセスするためのパスワード-中央で生成され、電話で設定-優れたセキュリティが保証されています！

  • デフォルトのコール転送方法（フォロー、非同伴）-3CXインターフェースで一元的に設定

  • 呼び出しメロディ-3CXインターフェースで一元的に設定

  • 電源LEDの動作、ディスプレイのバックライト、スクリーンセーバーのタイムアウト-3CXインターフェースで一元的に設定

  • LLDPサポートとVLAN構成-3CXインターフェースで集中的に公開

  • 電話ディスプレイの会社ロゴは中央にロードされますが、GXP17XXおよびGXP21XXシリーズのみです。

  • UDP / TCP / TLSシグナリングとSRTP音声暗号化をサポート



Grandstream音声ゲートウェイ（FXO、FXS）は公式にサポートされますか？



ただし、おそらく将来的には、これは3CXだけでなく、Grandstreamの代表者にも依存します。 現時点では何も約束することはできません。



TLS out-of-the boxはどういう意味ですか？



これは、3CXからFQDNを使用する場合、セキュリティ証明書がすでにシステムにインストールされていることを意味します。 TLSを介したSIP通信用に電話機を手動で設定する必要があります。 ただし、Update 4を待機する準備ができている場合、TLSの自動作業はIP電話の自動構成テンプレートに既に実装されています（現時点では、電話ベンダーのファームウェアの準備が整うのを待っています）。







標準バージョンの重要な制限は何ですか？



標準バージョンの3CXサーバー名のFQDNは、3CXによってのみ発行されます。 独自のドメインは許可されていません。 ただし、Update 3へのアップグレード時に、ユーザーが既に自分のFQDNを使用していた場合は、引き続き使用します。 ただし、ユーザーがライセンスキーからこのFQDNを「ほどく」とすぐに、将来的には3CXからのFQDNのみを使用できるようになります。



3CXからのみ電子メール通知を送信するためのSMTPサーバー。 ただし、Update 3へのアップグレード時にユーザーが既に独自のSMTPを使用していた場合、ユーザーは引き続きそれを使用します。 しかし、ユーザーが自分のSMTPをSMTP 3CXに変更するとすぐに、対応する警告が表示され、その後戻ることができなくなります。



したがって、最大限の柔軟性と独立性が必要な場合は、Proエディションを使用してください。



3CXのコピーをホストすることで稼いだパートナーにはどうすればよいですか？



3CXのホスティングとドメイン名を販売したいパートナー向けの主な推奨事項、つまり 一種の「ブランディング」を提供する-最も柔軟で機能的なエディションとして、プロ（またはエンタープライズ）の提案に焦点を当てます。 また、標準ライセンスの年間サブスクリプションの今後の販売（ 8コールと16コールのバージョンは最初の1年間のみ無料 ）のために、 3CXをダウンロードする特別なリンクを提供し、その中のパートナーIDを示します。 この場合、無料のライセンスがすぐにパートナーに割り当てられ、ポイントが付与されます！



現在の標準商用ライセンスはどうなりますか？



絶対に何もありません、すべての条件は同じままです。 FQDNおよびSMTPサーバーに加えて（上記のようにこれらのサービスを3CXからのサービスに変更することにした場合のみ）。



未払いの年間ライセンスはどうなりますか？



現在、エディションおよび容量の年間ライセンスは「終了」していません。 そのようなライセンスの有効期限が切れた後の最初の30日間に、3CXマーケティング部門と（そうすることを望んでいます！）3CXパートナーがユーザーに更新の必要性を通知します。 更新が行われない場合、31日目に、すべての年間ライセンスが4つの同時通話の標準年間ライセンスに自動的に変わります。 ライセンスの有効期限は12/31/2018に設定されています。



現在の無料ライセンスはどのくらい正確に置き換えられますか？



更新のサブスクリプションがまだアクティブな場合（キーのアクティベーション後1年目は無料ライセンスでアクティブになります）、PBXエディションライセンスは16の標準年間ライセンスに「変わります」。 同時に、サブスクリプションの終了日は変更されません。 サブスクリプションの終了後（および未払いの場合）、ライセンスは年間標準4に「変わります」。 電話して無料であり続けます。



更新プログラムのサブスクリプションの有効期限がすでに切れている場合、PBXエディションのライセンスは年間の標準4ライセンスに「変わります」。 電話して無料であり続けます。 サブスクリプションの終了日は2018年12月31日に設定されます。



例：古い無料と新しい無料の3CXエディションの比較



「古い」無料ライセンス（8SC PBXエディション）。

ライセンスキーをアクティブにした後、8つの同時コール、1つのSIPトランク、および更新の1年間のサブスクリプションを受け取りました。 1年（95ユーロ）後にサブスクリプションを更新しなかった場合でも、同じ8つのコールはありましたが、3CXからFQDN、SMTP、およびWebMeetingを切断しました。



「新しい」無料ライセンス（16SC標準）。

ライセンスキーを有効にすると、SIPトランクの制限なしで（ およびProエディションの機能を除くその他の制限なしで）16の同時通話を取得できます。 更新のサブスクリプションを更新しない場合、システムは4つの同時通話に「減少」しますが、3CXからの25人の参加者のFQDN、SMTP、WebMeetingを含めて無料のままです。
