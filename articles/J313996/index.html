Windowsユーザー向けのGitHub







プロジェクトがディスクのみに保存されている場合、ディスクが故障するとトラブルになります。 定期的なバックアップでさえ、常にあなたを救うことができるとは限りません。



一部の開発者はプロジェクトで非常に多くのことを実行できるため、彼ら自身がショックを受けます。 そして、何が行われたか、どこが難しいかを思い出します。 それはまだ迷惑です。



バージョン管理システムは、これらの問題を回避するのに役立ちます。 必要に応じて、変更を復元またはロールバックできます。 編集を表示して確認またはキャンセルします。 さて、バージョン管理システムのないチームワークは考えられません。



お互いに知らない場合は、Gitと呼ばれるバージョン管理システムを少し紹介します。 猫の下に、Visual StudioでGitHubを使用する方法の説明があります。



現在の拡張機能は、Visual StudioのGitHub拡張機能と呼ばれます。 Visual Studio 2015以降に適しています。 vsixは、 githubページまたはVisual Studioギャラリーからダウンロードできます。



Visual Studioのインストール時に拡張機能をインストールできます。









続行する前に、いくつかの用語を学ぶ必要があります。 それらを知っている場合は、下にスクロールします。



プッシュ -ローカルリポジトリからリモートリポジトリに変更を送信します（この場合、GitHubに配置されます）。



フェッチ -比較および可能な後続のマージのためにリモートリポジトリから変更を取得します。



Mergeは合併です。 現在のリポジトリによって別のリポジトリに加えられた変更を適用します。 2つのリポジトリを結合するようなものです。



プルは、フェッチとマージの組み合わせです。 最初に、変更のリストがリモートリポジトリから取得され、次に変更が現在のリポジトリに適用されます。



つまり、あなた以外の誰かがGitHubリポジトリを操作して変更を加えた場合、2つのアクションを連続して実行できます：フェッチ、そしてマージ。 または、すぐにプルを実行できます。 その後、ローカルリポジトリに完全な変更が表示されます。



Visual Studio用GitHub拡張機能をインストールすると、チームエクスプローラーパネルは次のようになります。









チームエクスプローラーパネルが非表示の場合は、[表示]メニューから表示できます。 GitHubに接続することで（[接続...]をクリックし、パスワード付きのユーザー名を入力して）、GitHubリポジトリを複製するか、新しいリポジトリを作成できます（[複製]ボタンと[作成]ボタン）。









クローンを作成すると、アクセスできるリポジトリのリストが表示されます：









リポジトリを作成するときに、名前、説明を入力し、ライセンスの選択に従って、コードの使用を許可します。









この場合、Gitにはさまざまなタイプのプロジェクトのプリセットが含まれます。 したがって、この.gitignore形式のファイルは、バージョン管理システムから除外するディレクトリとファイルを指定することを目的としています。



cut索好きな目からカツレツリポジトリを非常にうまく隠したい場合は、プライベートとしてマークできます。 ただし、これには有料のサブスクリプションが必要です。



GitHubは、 学生向けの特別オファー、 Student Developer Packを提供しています。これには、特に無制限の数のプライベートリポジトリが含まれています。



リポジトリを作成したら、プロジェクトを作成する必要があります。 個人的には、私は反対の方を好みます。最初にプロジェクトを作成し、それをGitに追加します。 プロジェクトを作成するときに、Gitリポジトリを作成することもできます。 これを行うには、チェックボックスをオンにします。









プロジェクトの作成時にこのボックスをチェックせず、VSでプロジェクトを開くだけの場合、[ファイル管理]メニューの[ソース管理に追加]項目が使用可能になります。









クリックすると、プロジェクトがGitバージョン管理システムに追加され、プロジェクトフォルダー内にローカルの.gitフォルダーが作成されます。 チームエクスプローラーでは、次のようになります。









チームエクスプローラーとソリューションエクスプローラーを切り替えると、プロジェクトにいくつかの変更を加えることができます。 変更後、コミットできます-一種の復旧ポイント。 これを行うには、チームエクスプローラーに戻ります。チームエクスプローラーのメニューにはボタンがあり、家が描かれています。 それをクリックすると、メインメニューが表示されます。

Git用のGithubデスクトップおよびPowerShell環境



Github Desktopは完全に独立したユーティリティであり、Visual Studioとは一切接続されていません。 こちらからダウンロードできます。









このユーティリティは、MacおよびWindowsユーザーが利用できます。 Git Shellコマンドラインがインストールされます。 実際、これはGitと統合するための一連のスクリプトを備えたPowerShellです。 GitのPowerShell環境と呼ばれます。 posh-gitの略。



posh-gitプロジェクトのGitHubページには、 gitのposhコマンドラインを手動で設定する方法に関する簡単な説明があります。



私はユーティリティ自体のインターフェースを考慮せず、それを操作しません。 複雑ではなく、自分で対処できると思います。 コマンドラインで少し遊んでみましょう。 GUIとは異なり、コマンドラインは通常、はるかに多くの機能を提供します。 ただし、基本的なチームのみを検討します。



現在の構成を表示し、Gitが存在することを確認するには、次のコマンドを実行できます。 



git config –list


リポジトリを複製するには、 git cloneコマンドを実行します。 例： 



 git clone https://github.com/programmersommer/Barcode_Scanner_UWP.git BarcodeScanner


このコマンドを実行すると、プロジェクトのあるフォルダーが現在のディレクトリに表示されます。 http：//およびhttps：//に加えて、SSHおよびgit：//プロトコルもサポートされています。 cdコマンドを使用してプロジェクトフォルダーに移動すると（cd BarcodeScannerの例の場合）、コマンドラインが変換されます。









PowerShellステータスバーにテキストposh〜gitが表示されます。これは、GitのPowerShell環境にいることを意味します。 git statusコマンドを実行して、ローカルリポジトリを同期する必要があるかどうかを確認できます。 答えはこれかもしれません：









最も人気のあるコマンドは、VS拡張インターフェースのフレームワークで既に検討したコマンドです： git fetch 、 git merge 、 git push 。 ディレクトリに移動する場合（ディレクトリのPortableGit_xxxという名前は、多少異なる場合があります）：

次に、コマンドをエミュレートする多くの実行可能ファイルを見つけます。 すでに述べたように、gitヘルプを使用できますが、例としていくつかのコマンドを試してみましょう。



たとえば、プロジェクトディレクトリに新しいファイルが表示される場合、git statusコマンドは次を表示します。









そのため、 git add index.htmlコマンドでファイルを追加する必要があります。 git commitを使用して変更を確認する必要があります。 このコマンドは、デフォルトでインストールされるテキストエディターを開きます。 最初の行で行われた変更を説明するテキストを入力する必要があります。 行を＃文字で開始すると、コメントになります。 コメントは下の行に残すことができます。 コミットを説明するテキストを残さないと、コミットは発生しません。 -mオプションを使用すると、コマンドラインでコミットテキストをすぐに指定できます。 例： git commit –m“ File index.html added”









これで、 git pushを使用してGitHubリポジトリに変更を送信できます。 これがリポジトリの場合。 リポジトリのフォーク/コピー-フォークを作成することにより、外部リポジトリを自分にコピーできます。 いくつかの変更を行った後、プルリクエストを作成して、元のリポジトリの作成者に変更を提供できます。
