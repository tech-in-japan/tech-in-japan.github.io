Jelastic + Luna = Mini Github

2か月半前に 、アプリケーションについて書きました 。

この間、私は数十の問題をクローズし、機能を追加し、 Yeah 、 kekekeksのユーザーの願いを実行し、修正しました 。

要するに、追加されたもの：

  • ソースと差分のフローティングスクロール用
  • 差分フローティングヘッダーの場合
  • バイナリファイルの認識が向上（ICU4J）
  • GitのHTTPサポート
  • メールアラート（Apacheキャメル）
  • 更新されたドキュメント
  • GUIの小さな変更とバグ修正


そして今、私のアプリケーションの使用方法について。 http理由で見た...



設計：



  1. ログイン/ jelastic.comで登録
  2. 私たちは自分のための環境を作ります。 SSLとMongoが必要です。残りはあなたの好みです（jetty 6とtomcat 7で試しました）。 セッション複製を有効にしないでください（HAスライダー）。
  3. 何も落ちていない場合は、お茶を飲みに行きます。 それ以外の場合は、jelasticサポートに進みます。
  4. 環境を作成すると、Mongoのアドレスとパスワードが記載されたメールが届きます。
  5. 前から住所に行きます。 ポイント、ログイン。 データベースとそのためのユーザーを作成します 。 
  6. $ git clone http://luna-tool.jelastic.com/den/luna.git $ cd luna $ ./sbt package
  7. 組み立てられた戦争を開きます（target / scala-2.9.1 / luna-tool_2.9.1-1.0.warにあります）。 そこにWEB-INF / classes / props / default.propsがあります。 次の設定を追加しますdb.host（Webインターフェースの電子メールから取得）、dp.port、db.name（上記で作成したデータベースの名前）、db.userおよびdb.password（db.nameのユーザーとパスワード）。 その他の設定はオプションです 。 このファイルがASCIIエンコードされていることを確認してください。
  8. jelastic、diploimに戦争を仕掛けます。 喜ぶ。
  9. 生きている例。
