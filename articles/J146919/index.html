GNU / LinuxとMicrosoft Windowsの統合

最近公開されたMac OS Xのトピックから、Windowsユーザーの目を通して、非常にユーザーフレンドリーな方法で別のオペレーティングシステムのアプリケーションを起動できるParallels Desktopプログラムについて学びました。 そして、私はこのアイデアがとても気に入ったので、VirtualBoxでも同じことをしようと決めました。 （コメントで思い出したように、この機能は他の多くの仮想化プログラム-VMWareなどでも利用可能です）



少しグーグル、私は必要な機能のほとんどがすでにこの素晴らしい仮想マシンにあることを発見したので、経験豊富なVirtualBoxユーザーにとって、このトピックは少しキャプテンに思えるかもしれません。



「アンダーザカット」に従うことを他の皆に提案します↓







手順1. Oracle VirtualBoxとMicrosoft Windows XPのインストール



VirtualBoxから始めましょう。 始めるには、GNU / Linuxディストリビューションのリポジトリで探すことを強くお勧めします。 これはフリーソフトウェアであるため、彼に会う確率は非常に高いです。



VirtualBoxがなかった場合は、公式ウェブサイトwww.virtualbox.org/wiki/Downloadsからダウンロードします。



インストール後、グラフィカルシェルを実行し、新しい仮想マシンを作成します。







新しい仮想マシンを作成するためのウィザードが開きます。 OS「Microsoft Windows XP」のタイプを指定します-これにより、VirtualBoxはこのシステムの最小パラメーターを自動的に設定できます。 当然、それからそれらを変更できます！







次に、新しい仮想ディスクを作成するか、既存の（仮想）ディスクを使用するかを尋ねられます。 新しいものを作成します。 作成するディスク（「固定」または「動的」）の質問に答えます。

  • ダイナミックディスクの場合、初期サイズを指定します。これは、必要に応じて増加できます。 減らすことはできないことに留意してください。 また、それはそれよりも少し遅いと考えられています
  • サイズを指定し、変更できない固定ディスク 。 あなたはおそらくそれを減らすことも増やすこともできないでしょう。 私はこのオプションを選択しましたが、ロジックは私が間違ったことを教えてくれます。


UPD： Vumik がコメントで示唆しているように、OSをインストールする前に、仮想マシンのプロパティでいくつかのコアを指定することをお勧め します。



次に、Microsoft Windowsシステムがインストールされたディスクをコンピューターに挿入し、仮想マシンを起動し、ディスクを "フィード"して、そこから再起動します。



選択したMicrosoft Windows XPのインストールは非常に簡単で、ほとんどのGNU / Linuxディストリビューションをインストールするよりも複雑ではありません。



インストールの特定の段階で、ライセンスキーが求められます。







ディスクを取り出した箱に書かれているはずです。



インターネットからisoイメージをダウンロードしたが、まだキーがない場合は、このフィールドを空白のままにします。 この場合、Microsoftはシステムを30日間無料で使用することを提案します。 指定された期間の後、Windowsを購入するか削除する必要があります。



ステップ2.フォントのスムージング



残念ながら、個人的には、フォントスムージングが有効になるまでWindowsの構成を続行できません。 Windowsでは、ClearTypeテクノロジーがこれを担当しますが、これを構成する必要があります。



これを行うには、次の手順を実行します。

  • [スタート]メニューから、[ コントロールパネル ] 、[ 外観とテーマ ] 、[ 画面 ]の順に選択します。
  • [ 外観 ]タブで、[ 効果 ]ボタンをクリックします。
  • [ 次の方法でスクリーンフォントを平滑化する]チェックボックスをオンにし、リストからClearTypeを選択します。


「OK」をクリックすることを忘れないでください。







ClearTypeの有効化の詳細については、Microsoftの公式Webサイトsupport.microsoft.com/kb/306527/enをご覧ください 。



ステップ3.リソースを追加する



デフォルトでは、VirtualBoxはWindowsに、現代のすべてのGNU / Linuxディストリビューションが機能するわけではない、あまりにも馬鹿げた機能を提供します。 そのため、「気にしない人の数」の原則に従って、仮想マシンのプロパティにアクセスしてリソースを提供します。 考慮すべきことが2つあります。

  • 「気にしない人」は、デフォルトでVirtualBoxが提供するリソースを減らすべきではありません。
  • 指定しすぎると、メインOSの速度が低下し、その結果、仮想マシンも遅くなる可能性があります。 したがって、「中間点」を見つける必要があります。


ステップ4.「ゲストOS追加機能」のインストール



現在、Windowsは通常のPC上で実行されると単純に信じています。 まあ、または仮想マシンで起動したことを知っていますが、丁寧さからは表示されません。



何らかの方法で、Oracleから追加のソフトウェアをインストールする必要があります。これにより、2つのシステムの統合を開始できます。



インストールは思ったよりずっと簡単です。 ホスト+ D（通常は「ホスト」=「Ctrl」）を押すか、仮想マシンのメニューで「デバイス」→「ゲストOS追加機能のインストール」をクリックします。



インストールウィザードが表示され、必要なプログラムをインストールできます。





注：「Windows」がいかにフォントを滑らかにするかをご覧ください:-)



次に、Windowsを再起動する必要があります（[スタート]→[シャットダウン]→[再起動]）。



ステップ5.「統合モードの表示」



ディスプレイの統合から始めましょう。 あるウィンドウを別のウィンドウの下に「ドラッグ」できるように、メインシステムの画面に仮想マシンの画面を押し付けるのは非常にcです。 これを行うには、Host + Dまたは「View」→「Display Integration Mode」をクリックします。



VirtualBoxは同時にウィンドウを全画面に拡大し、Windowsデスクトップを削除します。 次のようなものが得られるはずです。







また、以下に新しいパネル「Windowsタスクバー」があります。







ステップ6.共通クリップボード



「ゲストOS追加機能」をインストールしたとき、クリップボードとゲストOS（Windows）を組み合わせて「機能」を獲得しているはずです。 単独では機能しない場合は、仮想マシンのプロパティで対応する設定が有効になっているかどうかを確認します。







これで、テキストまたは画像をGNU / Linuxにコピーして、Windowsに貼り付けることができます。 またはその逆。



ステップ7.パブリックフォルダー



ここで、ちょっとしたトリックを行います。ドキュメントフォルダーを仮想マシンに「共有フォルダー」として接続し、これがドキュメントフォルダーであることをWindowsに示します。



したがって、たとえば、PhotoShopとGIMPで同時に画像を操作できます。 さらに、GIMPはGNU / Linuxで、Adobe PhotoshopはWindowsで起動します。



これを行うには、Windows仮想マシンを停止し、そのプロパティに移動して、ドキュメントで独自のフォルダーを指定する必要があります。







再びWindows仮想マシンを起動し、「Windowsコマンドプロンプト」プログラムを開き（ 「スタート」 → 「すべてのプログラム」 → 「アクセサリ」 → 「コマンドプロンプト」 ）、次のコマンドを入力します。 



net use x: \\vboxsvr\






新しいデバイス（「ディスク」）が表示されます-「Documents on` vboxsvr`（X :)」。GNU / Linuxからのすべてのドキュメントが含まれます。







次に、「マイドキュメント」を右クリックして、新しい宛先パスを指定します。







[OK]をクリックした後、Windowsはすべてのドキュメントをこの[新しい]フォルダーに移動することを提案します。既に作成できている場合は同意します:-)



おわりに



このような簡単な方法で、GNU / LinuxとMicrosoft Windowsの2つのシステムで作業を組み合わせることができます。 Apple MacOSなど、他のシステムのインストールを試みることもできます（それらは違法だと言うだけです） 。 この場合にのみ、かなり強力なコンピューターを用意し、さまざまなOSのパネルとウィンドウを配置する方法を理解する必要があります（たとえば、MacOSとWindowsを異なる仮想デスクトップに配置することができます）。



他の何か



「すべてが明確であるため、なぜWindows設定をこのように詳細に説明するのか」などの質問を予測し、私は答えます。このトピックは、Microsoft Windowsで作業できないGNU / Linuxユーザーを対象としています。 実際、Windowsユーザーの場合、GNU / Linuxの指示もCaptain Obviousによって書かれたように見えます！
