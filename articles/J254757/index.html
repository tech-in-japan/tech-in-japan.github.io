私たちは人々のように住所を書きます

ユーザーフレンドリーなインターフェースの利点は、私には全く理解できません。



はい、彼は美しいです、彼と一緒に仕事をするのは簡単です。 しかし、物質的な利益を計算する方法は？



5年前、空港で合成音声が鳴り、各単語の後に引き裂かれたイントネーションで到着と出発を発表しました。 場合によっては、話し言葉の速度でさえ言葉ごとに異なっていました。



そして数週間前、私は出発の新しい発表を聞いた。 彼らは生涯この役割のために勉強したプロのアナウンサーによって完全に記録されたという感じがありました。 インターフェースはより良くなり、より自然になりました。

これからもっと飛び始めましたか？ ほとんどない。

空港間で好みがありますか？ これはそんなに重要なポイントだとは思いません。

しかし、何らかの理由で、人々は広告を自然な外観にするために時間とお金を費やしてきました。 そして、それはとてもクールです！



私は、まるで生きている人があなたと話しているかのように、自動化されたシステムによる情報の自然な提示が好きです。 これらは、製品全体に対する態度を形成する小さなディテールであるように思えます。

そして、この態度が前向きであれば、あなたは将来選ばれ、友人に推薦されるでしょう。



この記事では、あたかも人が書いたように住所を1行で表示する方法を説明します。 記事を読むことで利益が上がるとは断言できません。

しかし、できるだけ多くのシステムでアドレスを美しく書きたいと思っています。



以下で説明するすべてのルールは、私たち自身が開発しました。 それらのほとんどを使用するか、すぐにdadata.ruに接続します 。



1年半前、私たちは人間のやり方で住所を書く方法を考えました。 Googleは住所を簡素化する方法について回答を提供しなかったため、言語学者とのコミュニケーション、テストの実施、実験を開始しました。



私たちは、人々がそれを書く方法で書かれた、最小限の長さで理解可能なアドレスを形成するためのルールを開発しようとしました。



マテリエル



アドレスを自動生成するには、アドレスのソースが必要です。 KLADRまたはFIASについて知っている場合は、この項目をスキップしてください。 残りについては、ロシアでは住所が階層的に形成されていることを伝えます。

  1. 各アドレスには件名があります。 ロシアには85の科目があります。 例：モスクワ地方、コミ共和国、モスクワ。

    行政区分によると、モスクワはモスクワ地方に位置せず、独立した都市地域です。

    ヒムキはモスクワではなくモスクワ地方にあります。 しかし、ゼレノグラードは反対に、モスクワにいます。
  2. 主題には地域と都市があります。

    件名に地区のない都市は都市地区です。 たとえば、ノボシビルスクは「ノボシビルスク地方」という主題に直接従属しています。

    FIASには、都市および町のセンターのステータスの兆候があります。 この情報は、1行で住所を作成する際に、今後役立つものになります。
  3. 地域には都市と町があります。
  4. 都市には通りと集落があります。 FIASによると、そこにはまだ追加の領域があります。通常、これらはガレージ建設協同組合、庭と非営利のパートナーシップなどです。
  5. 多くの場合、住所には都市のエリアを示します。たとえば、「モスクワ、アレクセーエフスキー地区」、「ノボシビルスク、」などです。 残念ながら、FIASとKLADRには都市内エリアは存在しないため、住所の自動構築には使用しません。
  6. 集落には、通り、追加の領土（第4項と同じ）、家屋があります。 通りではなく、単に村にある家があります。




通常の語順を使用します



まず、文字列全体を検討します。 ロシアでは、行政部門の最高レベルから最低レベルまで住所を書くことに慣れています。 通常の要素の順序は次のとおりです。

  1. 件名;
  2. 地区;
  3. 市
  4. 和解;
  5. 通り;
  6. ホーム;
  7. 家の拡張（建物、建物、階段、床、アパート、オフィス、..）。


たとえば、「モスクワ地方、ムィティシ地区、ムィティシ市、ウリヤノフスカヤ通り、ビル3」



さらに、各レベル内の語順は重要です。 たとえば、「サンクトペテルブルク市」ではなく「サンクトペテルブルク市」、「アルタイ準州」ではなく「アルタイ準州」と書くのが正しいでしょう。 順序は、タイプごとに独自の方法で決定されます。

  • 地域、地域、地域、自律地域：最初に名前、次にタイプ。
  • 共和国、都市：最初のタイプ、次に名前。
  • 都市の要素：エリア、通り、広場など 要素の順序は、「名前タイプ」または「タイプ名」のいずれかです。 名前の終わりと出現頻度に応じて、各タイプに独自の順序を使用します。 たとえば、「モスクワ、アビエーションレーン」および「モスクワ、アレクサンドルネフスキーレーン」と記述します。


例外を忘れないでください：

  • FIAS / KLADRによると、名前に「自律okrug」が含まれているため、タイプなしで「ハンティ・マンシースク自治Okrug-Ugra」を記述します。 例：「ハンティ・マンシースク自治管区-ウグラ、ハンティ・マンシースク、アンナ・コンコヴァ通り、1」
  • チュヴァシ共和国には素晴らしいタイプのチュヴァシアがあります（つまり、地域の名前は「チュヴァシ共和国-」で、タイプは「チュヴァシア」です）。名前の後に「チュヴァシ共和国-チュヴァシア、チェボクサル、ウシャコヴァ海軍大将2」と表記するか、完全に省略します名前の末尾のダッシュを削除します：「チュヴァシュ共和国、チェボクサル、ウシャコフ海軍大将、2」


FIASからのアドレス：

ロシア、タタールスタン共和国、カザン市、カシモフ兄弟通り、64


要素を正しい順序に並べ替えると、

ロシア、タタールスタン共和国、カザン市、カシモフ兄弟通り、64


読みやすくなりました。 しかし、あなたはより良く、より短くすることができます。



フォーマットを使用してタイプと名前を分離する



アドレスを読みやすくするのに役立つ簡単なことは、大文字を使用して名前とタイプを視覚的に分離することです。



すべてのタイプは小文字で、すべての名前は大文字で書きます。 ご覧のとおり、2番目のオプションの方が高速です。



宛先：

ペトロパブロフスク・カムチャツキー市、カムチャッカ準州、ベーリングストリート、ビル90、アパートメント61


後：

ペトロパブロフスク・カムチャツキー市、カムチャッカ準州、ベーリング街、90、アパート61

型の短縮形について独自のルールを作成する場合は、次の2つの点に注意してください。

  • 型を減らすとき、アドレス行内の一意性を失うことを恐れることはできません。 すべてのタイプは名前の隣にあり、人は略語の意味を簡単に判断できます。 たとえば、説明のない「タタールスタン共和国、dナベレジヌイェモルクバシ、d 45」のアドレスでは、最初のdは村であり、2番目のdは家であることが明らかです。
  • ただし、削減は明確である必要があります。 たとえば、「Moscow p Turchaninov」というアドレスの代わりに、「p」は脇道、大通り、または広場のいずれかになりますが、「Moscow、per Turchaninov」と書く方が適切です。


場合によっては、人々は「Big」などの言葉を省略します。 たとえば、「B。 オルディンカ。」 しかし、名前の一意性を失う可能性があるため、これを自動的に行うのは恐ろしいことです。



タイプの略語の前のアドレス：

ロシア、タタールスタン共和国、カザン市、カシモフ兄弟通り、64。


後の住所：

ロシア、タタールスタン共和国、カザン、カシモフ・ブラザーズ通り、64。




削除できるものはすべて削除します



理想的なアドレスへの次のステップは、冗長な情報の削除です。

住所が通信の配信に使用される場合、小包と手紙のソートを簡素化するために、冗長な情報を残すのが最適です。

住所が人が場所を特定するためだけに必要な場合、または都市の宅配便配達が想定される場合、冗長な情報を削除すると住所がより自然で読みやすくなります。



次のルールを提供します。

  1. ロシアの場合は国を削除します。 パリの村のようなまれなケースは認識に影響を与えません：「チェリャビンスク地域、ナガバク地区、パリの村」。
  2. 都市のタイプを削除します。 都市の名前は1つの地区内で一意であるため、この情報は重要ではありません。 「Mr. Voronezh」、「Voronezh」は残りました。
  3. 都市部から地域を削除します（これらは地域で最大の都市です。私はそれらについて上記で書きました）。 これらの都市はロシアで有名でユニークです。 たとえば、「Novosibirsk Region Novosibirsk」ではなく、単に「Novosibirsk」と書きます。
  4. 地区の中心付近にある地区を削除します（FIASとKLADRにサインがあります）。 彼らはユニークで、地域内でかなりよく知られています。 たとえば、「モジャイスク、モジャイスク地区、モスクワ地域」ではなく、「モジャイスク、モスクワ地域」と書きます。
  5. コンテキストから明らかな場合は、都市を削除します。 エカテリンブルクでのコンサートのポスター？ 各サイトのアドレスに「エカテリンブルグ」という言葉を書くのをやめます。


私が記事を書いているときに、「ストリート」という言葉を削除するというアイデアが浮上しました。 このタイプが指定されていない場合、デフォルトとして認識されるように思えます。 「モスクワ、トヴェルスカヤ」という実験を行ってみましょう。 ストリートについてどう思いましたか？ しかし、トヴェルスカヤ広場があります。 ただし、これはまだサポートされていない仮定であり、これはご自身の責任とリスクで行ってください。 このテーマに関するコメントを聞くのは面白いでしょう。



明らかなエンティティを削除すると、例のアドレスはより読みやすい形式になります。

それは：

ロシア、タタールスタン共和国、カザン、カシモフ・ブラザーズ通り、64。


次のようになりました：

カザン、カシモフ兄弟、d 64


または、コンテキストが許可する場合：

カシモフ兄弟、d 64


家の近くのタイプは削除しません。通常、住所にはアパートや建物の表示も含まれているため、一部のタイプの不在は奇妙に思えます。 しかし、これは好みの問題です。

カシモフ・ブラザーズ、64、ビル1、apt。1


または

カシモフ兄弟、d 64、建物1、apt。1


住所に家が1つしかない場合、「家」という単語がなければ、住所は良く見えるだけです。

64カシモフ兄弟




住所を2行に分けます



私たちの住所はすでに良くて美しいので、インターフェースに表示できます。

書式設定の助けを借りてそれをさらに簡素化する方法が1つあります。2行に分割します。1行目は都市、地区、主題をこの順序で記述し、2行目は集落、通り、その他すべてを記述します。



FIASからの元のアドレス：

ロシア、タタールスタン共和国、カザン市、カシモフ兄弟通り、64。


このように書きましょう：

カザン、

64カシモフ兄弟




それだけです 情報が役に立つことを望み、自然言語でプログラムを「話す」方法を教えてください。
