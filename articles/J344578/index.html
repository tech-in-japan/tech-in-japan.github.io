イーサリアムブロックチェーンリンケビーテストネットワークでの固体契約実験





この記事は、EthereumブロックチェーンのSolidityスマートコントラクト開発環境をすばやくセットアップするのに役立つ小さな実用的なガイドです。 最初の契約を公開し、Rinkebyテストブロックチェーンに保存して、契約のメソッドを呼び出す方法を学習します。 これは、分散型DApp（分散型アプリケーション）を作成するための最初のステップになります。



この記事のトピックに関する本、記事、マニュアルは豊富にありますが、新規参入者が契約の発行とそれらの使用を開始することは非常に困難です。 本、マニュアル、記事について何かをしようとすると、例がうまくいかず、チームが理解できないエラーを返すことがよくあります。 この記事では、イーサリアムの研究での経験を反映して、開発の第1段階を多少簡略化しようとします。



このトピックに飛び込むときは、Microsoft Windows環境でMistアプリケーション（ブラウザー）を使用し、Ubuntu環境ではEthereumノードのGethコマンドラインインターフェイスを使用しました。 この記事では、Gethでの作業と、Node.jsからコントラクトメソッドを呼び出す方法について少し説明します。



感謝の意を込めて、Ethereumブロックチェーン、Solidity言語での契約の開発、DAppアプリケーションに関するさらなる記事に関するコメントと提案を受け入れます。

Ethereumは、スマートコントラクトを開発し、いわゆる分散型DAppアプリケーションを作成するための分散型プラットフォームです。 Ethereum Webサイトはhttps://www.ethereum.org/にあります 。 ここには、あなたにとって役立つ主なリソースへのリンクがあります。



たとえば、プラットフォームのドキュメントはhttp://www.ethdocs.org/en/latest/で公開されています 。



Solidity言語ガイドはこちらです。 ロシア語への翻訳がありますが、元の文書より遅れている場合があります。



資料のプレゼンテーションの過程で、特定のリソースへのリンクを作成します。



Ubuntu仮想マシンへのイーサリアムノードのインストール



契約を公開し、イーサリアムと連携する場合、通常のコンピューターユーザーはMicrosoft WindowsのビジュアルインターフェイスでMistブラウザーを使用する方が簡単です。 ただし、何が起きているかを理解するソフトウェア開発者（ソフトウェア）は、コマンドラインを処理する方が便利です。



Goで作成されたEthereumプロトコルの3つのオリジナル実装の1つであるGo Ethereumをインストールします（C ++およびPythonの実装はまだあります）。



Go Ethereumは、Ubuntu、Arch Linux、およびFreeBSDサーバープラットフォームにインストールできます。 Ubuntuを選んだのは、Debianに似ているためで、オンラインストアサービスでDebianを使用しています。 実験のために、VMWare Workstationを実行している仮想マシンにUbuntu 17.10 OSをインストールしました。



Ubuntuをインストールした直後に、SSH経由で接続する機能を追加する必要があります。 さらに、Vimエディターをインストールしました。 



sudo apt-get install ssh sudo apt-get install vim


次に、Go Ethereumの安定版リリースをインストールする必要があります。 



 sudo add-apt-repository -y ppa:ethereum/ethereum sudo apt-get update sudo apt-get install ethereum


このプロセス全体については、 ここで説明します 。



ここで、Rinkebyテストネットワークのフルノードをインストールする必要があります。 さまざまなタイプのノードの説明は、 https ： //www.rinkeby.io/#gethにあります 。 完全なノードには、この命令のブロックが必要です。



まず、次の場所からrinkeby.jsonファイルをホームディレクトリにコピーします： https ://www.rinkeby.io/rinkeby.json



次に、最初のコンソールウィンドウを開き、ホームディレクトリにrinkebyフォルダーを作成して、コマンドを実行します。 



 geth --datadir=/home/frolov/rinkeby init rinkeby.json


このコマンドは、初期化中に一度だけ実行されます。 ここでは、ノードデータは/ home / frolov / rinkebyディレクトリに格納され、rinkeby.jsonファイルは/ home / frolov /ディレクトリに格納されると想定しています。 もちろん、別のディレクトリの場所を選択できます。



初期化が完了したら、同じウィンドウで次のパラメーターを使用してgethを実行します（shop2youの代わりに、ethstatsパラメーターに一意の名前を指定します） 



 geth --networkid=4 --datadir=/home/frolov/rinkeby --rpc --rpcaddr "0.0.0.0" --rpcapi "admin,debug,miner,shh,txpool,personal,eth,net,web3" --ethstats='shop2you:Respect my authoritah!@stats.rinkeby.io' --bootnodes=enode://a24ac7c5484ef4ed0c5eb2d36620ba4e4aa13b8c84684e1b4aab0cebea2ae45cb4d375b77eab56516d34bfbd3c1a833fc51296ff084b770b94fb9028c4d25ccf@52.169.42.101:30303


gethコマンドのパラメーターの説明は、 https：//github.com/ethereum/go-ethereum/wiki/Command-Line-Optionsにあります 。 次のコマンドを使用して、gethパラメーターに関するヘルプを表示することもできます。 



 geth --help


--networkid = 4パラメーターは、R​​inkebyテストネットワークを使用していることを示します。 --datadirパラメーターを使用して、ホストデータベースとキーを保存するディレクトリの場所を指定します。



--rpcオプションは、HTTP-RPCサーバーを有効にします。 このサーバーを介して、2番目のコンソールウィンドウからノードに接続します。 --rpcaddrパラメーターを使用して、サーバーがホストのすべてのネットワークインターフェイスで実行されることを示します（デフォルトでは、localhostのみ）。 --rpcapiパラメーターを使用して、HTTP-RPCインターフェースを介して使用できるAPIを指定します。



--ethstatsパラメーターは、ethstatsサービスレポートのURLを指定し、nodename：secret @ host：portの形式で指定されます。 最後に、-bootnodesパラメーターはP2PダウンロードのURLを設定します（P2Pディスカバリブートストラップのコンマ区切りのenode URL）。



そこで、最初のコンソールウィンドウでRinkebyネットワークのGo Ethereumノードを起動しました。 2番目のコンソールウィンドウで、次のコマンドを使用してこのノードに接続します。 



 geth --networkid=4 --datadir=/home/frolov/rinkeby attach ipc://home/frolov/rinkeby/geth.ipc


接続すると、gethユーティリティのプロンプトが表示されます。 次のコマンドを入力します。 



 > web3.version { api: "0.20.1", ethereum: "0x3f", network: "4", node: "Geth/v1.7.3-stable-4bb3c89d/linux-amd64/go1.9.1", whisper: undefined, getEthereum: function(callback), getNetwork: function(callback), getNode: function(callback), getWhisper: function(callback) }


このコマンドは、契約で使用するWeb3ソフトウェアインターフェイスのバージョン、gethバージョン、およびRinkebyのネットワーク番号（4）を表示します。

アカウントを追加する



新しいサイトをインストールした直後に、そのサイトにアカウントを追加する必要があります（既に他のサイトにアカウントがある場合は、新しいサイトに転送できます）。



アカウントを追加するには、gethプロンプトでpersonal.newAccount（）コマンドを入力します。 



 > personal.newAccount() Passphrase: Repeat passphrase: "0x7d9006e7f24bd6d90dd8cc63764ab0b92b77d9b3"


ここで、パスワードを入力する必要があります。 テストされていない実際のイーサリアムネットワークを使用する場合は、特にパスワードの保存に注意する必要があります。 暗号化されたドライブであっても、コンピューターにパスワードを書き込まないことをお勧めします。 事実、ウイルスまたはトロイの木馬は暗号化されたディスクからパスワードを盗むことができるからです。 ディスクを操作すると、その内容が利用可能になります。 キーボードからパスワードを入力するときにパスワードを傍受できるキーロガーもあります。



パスワードは秘密鍵を保護します。 攻撃者がコンピュータからパスワードと秘密鍵を盗むことができる場合、攻撃者はこのアカウントが持つすべての手段を所有します。 ところで、パスワードを回復する方法はないため、パスワードを失うことはできません。



パスワードを入力すると、アカウントアドレスがコンソールに表示されます。この場合、「0x7d9006e7f24bd6d90dd8cc63764ab0b92b77d9b3」です（別のアドレスになります）。 作成されたアカウントのアドレスを記録する必要がないことに注意してください。 それらはeth.accountsコマンドを使用して表示できます。 



 > eth.accounts ["0x7d9006e7f24bd6d90dd8cc63764ab0b92b77d9b3"]


この方法で複数のアカウントを追加できます。 この場合、コマンドはすべてのアカウントの識別子の配列を出力します。 



 > eth.accounts ["0xa15862b34abfc4b423fe52f153c95d83f606cc97", "0x7d9006e7f24bd6d90dd8cc63764ab0b92b77d9b3", "0x957f5272eadf4e284aade9b2cb2a41f475936d07"]


アカウントごとに秘密鍵が作成されます。 この場合、すべてのキーはディレクトリ/ home / frolov / rinkeby / keystoreにあります。 特にテストネットワークではなくメインのアカウントに関連する場合は、これらのキーを安全な場所に保管することをお勧めします。



チャージアップアカウント



コントラクトを公開し、そのメソッドを呼び出すことができるようにするには、ウォレットにエーテル（エーテル）を補充する必要があります。 現在の残高は、次のコマンドで確認できます。 



 > web3.fromWei( eth.getBalance(eth.coinbase) ) 0


このコマンドには、アカウント識別子を割り当てることもできます。 



 > web3.fromWei( eth.getBalance("0x7d9006e7f24bd6d90dd8cc63764ab0b92b77d9b3") ) 10.428648202959703389


Rinkebyテストネットワークでウォレットを補充するには、実際の紙幣や暗号通貨は必要ありません。 サイトhttps://faucet.rinkeby.io/を使用します 。 また、Google +、Facebook、またはTwitterアカウントも必要です。



0x7d9006e7f24bd6d90dd8cc63764ab0b92b77d9b3など、アカウントのアドレスを含むリストされたソーシャルネットワークの1つに公開します（別のアドレスになります）。



次に、 イーサ リアムアドレスを含むソーシャルネットワークURLフィールドの公開アドレスをコピーし、リスト「私にイーサ」から値の1つを選択します。



ここでは、8時間ごとに3つのエーテル、毎日7.5のエーテル、または3日ごとに18.75のエーテルを受け取ることができます。 3エーテルで十分ですので、最初のオプションを選択できます。



すべてを正しければ、しばらくして「エーテル労働者」があなたのアカウントに行きます。 もちろん、これらの資金はRinkebyテストネットワークでのみ使用できます。



出版契約



すべてをインストールして確認し、アカウントを補充したので、契約の公開を開始できます。 非常にシンプルなHelloSolコントラクトを使用します。 Solidityのソースコードは次のとおりです。 



 pragma solidity ^0.4.10; contract HelloSol { string savedString; uint savedValue; function setString( string newString ) public { savedString = newString; } function getString() public constant returns( string ) { return savedString; } function setValue( uint newValue ) public { savedValue = newValue; } function getValue() public constant returns( uint ) { return savedValue; } }


このコントラクトにより、テキスト文字列と数値を保存できます。 setStringメソッドとgetStringメソッドを使用すると、それぞれ文字列を読み書きできます。 同様のsetValueおよびgetValueメソッドが数値に対して提供されています。



3つのテストアカウントを作成しました。 



 > eth.accounts ["0xa15862b34abfc4b423fe52f153c95d83f606cc97", "0x7d9006e7f24bd6d90dd8cc63764ab0b92b77d9b3", "0x957f5272eadf4e284aade9b2cb2a41f475936d07"]


契約を公開するには、アドレス0x957f5272eadf4e284aade9b2cb2a41f475936d07の最後のアドレスを使用します。 このアドレスは次のように参照できます。 



 > web3.eth.accounts[2] "0x957f5272eadf4e284aade9b2cb2a41f475936d07"


アカウントアカウント[2]に資金があることを確認します。 



 > web3.fromWei( eth.getBalance(web3.eth.accounts[2])); 1.49157544347657552


契約を公開する前に（およびデータを変更する契約メソッドを呼び出す前に）、アカウントのロックを解除する必要があります。 これは、personal.unlockAccountメソッドを使用して実行できます。 



 > personal.unlockAccount(eth.accounts[2]) Unlock account 0x957f5272eadf4e284aade9b2cb2a41f475936d07 Passphrase: true


アカウントの作成時に指定したパスワードの入力を求められます。



契約編集



次に、契約のいわゆるアプリケーションバイナリインターフェイス（ABI）を作成し、Solidityで記述された契約のソーステキストをコンパイルする必要があります。



ABIインターフェースは、契約とイーサリアムシステムとの相互作用の仕様、またはより簡単に言えば、契約メソッドのパラメーターと戻り値の仕様です。 詳細についてはこちらをご覧ください 。



Solidityで書かれた契約のソーステキストをコンパイルした結果、Ethereumネットワークのすべてのノードで仮想マシンを実行するプログラムのバイナリコードを取得する必要があります。 私たちの場合、これはRinkebyテストネットワークのみになります。



プログラムのABIインターフェイスとバイナリコードは、 https：//remix.ethereum.orgから入手できるIDE Remixブラウザツールを使用して取得するのが最も簡単です。 Remixの使用ガイドはこちらにあります 。



ブラウザでRemix IDEを開き、次の図に示すように、HelloSolコントラクトのコードをソースウィンドウにコピーします。







デフォルトでは、コードは次のようにすぐにコンパイルされます 自動コンパイルがチェックされます。 ABIとバイナリを取得するには、[ 詳細 ]ボタンをクリックします。 表示されるウィンドウでweb3Deployブロックを見つけます。







[ 値をクリップボードにコピー ]ボタンを使用してここからコードブロックをコピーし、このテキストをテキストエディター、たとえばVimエディターに貼り付けます。

これを行う前に、web3.eth.accounts [0]の行を編集し、web3.eth.accounts [2]の行に置き換えました。 これは、資金があるweb3.eth.accounts [2]アカウントから公開するために行われました。 アカウントを1つだけ作成し、エーテルで補充した場合は、Remixからコピーしたテキストをプロンプトでgethプログラムのコンソールウィンドウに貼り付けるだけで、何も編集する必要はありません。

コントラクトメソッドの呼び出し



次に、最も興味深いもの-契約との相互作用を扱います。 ここで説明するJavaScript Web3 APIを使用して、gethコンソールからメソッドを呼び出します 。 これはWeb3 0.2xxの安定版のドキュメントであり、http://web3js.readthedocs.io/en/1.0/index.htmlで説明されているバージョン1.0はまだ実装されていません。



まず、Gethコンソールプロンプトで、次の行を入力します。 



 var HelloSolContract = web3.eth.contract([{"constant":true,"inputs":[],"name":"getValue","outputs":[{"name":"","type":"uint256"}],"payable":false,"stateMutability":"view","type":"function"},{"constant":false,"inputs":[{"name":"newValue","type":"uint256"}],"name":"setValue","outputs":[],"payable":false,"stateMutability":"nonpayable","type":"function"},{"constant":false,"inputs":[{"name":"newString","type":"string"}],"name":"setString","outputs":[],"payable":false,"stateMutability":"nonpayable","type":"function"},{"constant":true,"inputs":[],"name":"getString","outputs":[{"name":"","type":"string"}],"payable":false,"stateMutability":"view","type":"function"}]);


ここでは、ABIに基づいて、契約のオブジェクトが作成されます。 atメソッドを呼び出し、コントラクトを公開するステップで取得したコントラクトアドレス0x11c63c5ebc2c6851111d881cb58c213c609c92d4をパラメーターとしてこのメ​​ソッドに渡します。 



 var HelloSol = HelloSolContract.at("0x11c63c5ebc2c6851111d881cb58c213c609c92d4");


これで、コントラクトメソッドを呼び出すことができます。 予想どおり、getValueメソッドは次のようにnull値を返します。 契約ベースにはまだ値を保存していません。 



 > HelloSol.getValue() 0


setValueメソッドを呼び出す前に、unlockAccountメソッドを使用してアカウントをロック解除する必要があります。 



 > personal.unlockAccount(eth.accounts[2]) Unlock account 0x957f5272eadf4e284aade9b2cb2a41f475936d07 Passphrase: true


覚えている場合、以前に契約を公開する前にアカウントのブロックを解除しました。



アカウントのロックが解除されたら、setValueメソッドを呼び出します。 



 > HelloSol.setValue(777, {from: "0x957f5272eadf4e284aade9b2cb2a41f475936d07"}) "0x019131872a3a834a1aaad7a810977eaaa1f9131344b9fb96e57d21a3f08c1363"


最初のパラメーターとして保存する値をメソッドに渡し、2番目としてこのメ​​ソッドが呼び出されるアカウントのアドレスを渡すことに注意してください。



次に、getValueメソッドを使用して値の取得を試みます。

solバッチコンパイラのインストール



IDEのリミックスを使用してデバッグするには、小さなコントラクトが非常に便利です。 ただし、代替手段があります-コマンドラインで実行できるsolcバッチコンパイラ。 このコンパイラのドキュメントはこちらです。



次のコマンドを使用してsolcをインストールしました。 



 sudo snap install solc


インストール後、コンパイラのバージョンを確認できます。 



 solc --version solc, the solidity compiler commandline interface Version: 0.4.18+commit.9cf6e910.Linux.g++


HelloSolコントラクトをコンパイルして、ABIとバイナリを含むファイルを取得しましょう。 コントラクトのソースコードをHelloSol.solファイルに書き込み、次のコマンドを実行します。 



 solc --bin --abi HelloSol.sol -o build --overwrite


実行後、現在のディレクトリのビルドサブディレクトリにHelloSol.abiとHelloSol.binの2つのファイルが作成されます。 最初のものにはABIが含まれ、2番目にはコンパイルされたコントラクトのバイナリコードが含まれます。



コントラクトをデプロイするには、Remixのコードを使用し、コンパイル結果をHelloSol.abiファイルとHelloSol.binファイルの内容に置き換えます。 したがって、これらのファイルの内容の行内に改行がないことを確認する必要があります。
