Oleg Dokukaはjug.msk.ruの事後開発について

2018年12月13日、モスクワのJava開発者のjug.msk.ruコミュニティの会議で、 Oleg DokukはJavaプログラミング言語の 事後開発の機能について話しました。







スピーカーについて



OlegはJBreak 、 Joker 、 JEEConf 、 GeeCON 、 Voxxed Daysなどの会議で積極的に講演しています。 Reactorプロジェクトの貢献者であり、 Reactive Programming with Spring 5の共著者でもあります。



時系列での彼の報告のいくつか：

  • 「Spring 5で反応する」（ JEEConf 2017 ：ビデオパート1およびパート2 ）
  • 「Spring 5およびProject Reactorでリアクティブにリファクタリングする」（ Eat Dog＃11 2017 ： ビデオ ）
  • 「Spring 5およびReactor 3を使用した暗号通貨取引プラットフォームの構築」（ JBreak 2018 ： プレゼンテーション 、 ビデオ ）
  • 「2b or！2b with react Spring」（ JEEConf 2018 ： ビデオ ）
  • 「Spring 5でリアクティブにリファクタリング」（ Riga Dev Days 2018 ： ビデオ ）
  • Spring 5とReactor 3でリアクティブにリファクタリング（ GeeCON Prague 2018 ： ビデオ ）
  • 「Reactive Serverを50分で構築する方法」（ Voxxed Days Minsk 2018 ： ビデオ ）
  • 「Reactive Hardcore：パブリッシャーの構築方法<？>」（ Joker 2018 ： プレゼンテーション ）


追加リンク： Twitter 、 GitHub 、 Facebook 。



レポートについて



Andrey Kogunが会議を開始し、今後のJava会議とjug.msk.ruの6周年を想起します。jug.msk.ruの最初の会議は 2012年12月13日に開催されました。







最初のレポートは、 ジョーカー会議でオレグが発表したレポートの拡張版でした。 利用可能な時間が長くなると、 パブリッシャーを構築する方法をより詳細に伝えることができ、ストーリーが進むにつれて視聴者の質問に答えることができました。 レポートは、 GitHubで利用可能なプロジェクトによって示されました。







第2部では、データベースへの事後アクセスの現状に関する別のレポートが提示されました。 とりわけ、競合するADBAおよびR2DBCテクノロジーが言及されました。 このレポートには例のデモンストレーションも伴っていました。







休憩中および会議後、講演者と話し、質問をして興味のあるトピックについて話し合うことができました。 報告の間に、多数の質問が出され、オレグはそれに答えられました。







最初と2番目のレポートのプレゼンテーション。 ビデオはすぐに表示されます（以下のリンクを参照）。
