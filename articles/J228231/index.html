モスクワのBeelineネットワークでHD-Voiceをオンにしました。 以下は、前後のサウンドレコーディングと技術的な詳細です。



HD-voiceモードは、AMR-WBテクノロジー （Adaptive Multi Rate Wadeband）-可変速度の適応ブロードバンドコーディングに基づいています。このコーデックはG.722.22規格としても知られています。 理論的には、人は20Hz〜20kHzの範囲の音を聞くことができますが、実際には、はるかに狭い範囲で音を送信するのに十分です。 AMR-WBで音声をエンコードする場合、50Hz〜7kHzの周波数範囲が使用されますが、これは完全な音声伝送には絶対に十分です。 たとえば、AMRコーデックは300-3 kHzの範囲を使用します。 この方法は、2Gおよび3Gネットワ​​ークで使用できます。



簡単に言えば、人間の声はより自然に聞こえるようになります。これは、インフラストラクチャの制限によってその必要性が決定されていたかなり狭い帯域に「絞る」のをやめるためです。 さらに、HDコーディングでは、外部ノイズが音声明瞭度に与える影響が少なくなります。



どのようにしてHD-Voiceに到達し、どのように機能しますか？



人は、特定の帯域の音の周波数を聞くことができます。通常は、20Hz〜20kHzに相当します。 圧縮せずに無線通信チャネルを介してスペクトル全体を送信することは困難であるため、帯域カットなしの音声は有線システムでのみ送信されます。 音声の圧縮には、音声コーデックまたはボコーダーが使用されます。 それらの副作用は、音声メッセージの周波数帯域を制限することによる送信音声の歪みです。



モバイルネットワークで使用された最初の音声コーデックは「フルレート」-FRです。 FRと同時に、ハーフレートコーデックも導入されました。その唯一の目的は、同時にサービスを提供する加入者の数を増やすことであり、その名前が示すように、無線チャネルでFRコーデックの半分の帯域を占有します。 ただし、FRへのトランスコーディング後の音声品質はそれほど高くなく、一部の国（たとえば、スピーチ速度が高く、多くの高周波音が聞こえるアラブ諸国）では、まったく受け入れられませんでした。 そのため、FRコーデックが再設計され、拡張フルレートが表示され、ビットレートが低くなり、音声品質が大幅に向上しました。



モバイルネットワークの普及に伴い、音声サービスの品質に対する要件が増大し始め、基地局からの無線信号のレベルが低いため、音声品質の低下に問題がありました。 この問題を解決するために、分析と圧縮に同じ周波数範囲を使用する「適応型マルチレート」という新しいコーデックが開発されましたが、アルゴリズムは、受信信号の品質に応じてコーデックのビットレートが動的に変化するように実装されました。 これにより、良好な無線条件で優れた音声品質を提供し、悪い条件で-FR / EFRコーデックがまったく機能しなかった場合でも会話を続けることが可能になりました。



すべての3Gネットワ​​ークは、AMRコーデックまたはその後の実装（AMR-WBなど）を使用してのみ機能します。 ただし、EFR / HRコーデックのみをサポートする電話を使用しているサブスクライバーが多数残っているため、2Gネットワ​​ークはすべてのコーデック（FR / EFR / HR / AMR FR / AMR HR）で直ちに動作し続け、あらゆる電話に音声通信を提供します。



現在、モバイルネットワークの音声品質要件は拡大を続けており、AMR-WBテクノロジーを使用する時期です。 上記のすべてのコーデックは、圧縮に最大200Hz-3.4kHzの周波数範囲を使用し、AMR-WBはエンコードに50Hz-7kHzの音声メッセージ帯域幅を使用します。 AMR-WBを使用すると、2倍の大きな周波数帯域を送信できるため、送信される音声の高周波数と低周波数の品質と彩度が向上します。

認識の問題



人間のスピーチは、「C」と「F」の音がラジオや音声スペクトルの狭い帯域で区別されにくいようなものです。 ペア「T」と「P」、「M」と「H」も被害を受けます。 それゆえ、「私はそれを理解しているように、私はレセプション？ コールサインの最も重要な部分に「p」を割り当てるなど、物事はここから成長します。たとえば、コールサインを混同してはならないスナイパーは、おそらく「3/3」になります。ストライプ。



また、HD-Voice音声認識も増えています。 これは、個人的なコミュニケーションにとって重要であり、たとえば銀行での音声による追加認証の人気が高まっているためです。
