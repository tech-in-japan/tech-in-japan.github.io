Windowsで壊れたVDSサーバーを修正する方法





コンテスト 「レベル80管理者」は終了しました。 参加者のタスクは、「壊れた」VDSサーバーを修正することでした。 受賞者の皆さん、おめでとうございます。 主な賞品はミハイル・ゴッホのサラトフに贈られます。 勝者は2時間以内にタスクを完了しました。これは素晴らしい結果です。

賞品を獲得しなかったすべての参加者は、生涯にわたってRUVDSからVDSサーバーの使用に対して10％の割引を受けます。 コンテストへの登録時に指定された電子メールで個人通知が送信されます。



ソリューションを投稿します。



それでは始めましょう！

RDPクライアントを介して標準的な方法でサーバーに接続しようとしています。 出てこない おそらくこれはコンテストのアイデアの1つでしょうか？

アカウントの緊急モードで接続されています。 接続済み！



adminプロファイルがロードされなかったため、非GUIモードが使用される可能性が高いことがわかります。

資格情報を入力します。 私たちの前に暗い画面があります。 通常、非GUIモードではcmd.exeがすぐに開きますが、現時点では何らかの理由でユーティリティが起動されていません。

ほとんどの場合、現在のユーザーに実行権限がないか、実行可能ファイル自体がありません。



ボタンをクリックしてctrl + alt + delコマンドを送信し、タスクマネージャーを開きます。 どういうわけか、コマンドラインインタープリターを実行する必要があります。

[ファイル]> [新しいタスク...]は、この目的に最適です。 cmd.exeファイルに必要な権限を与えようとします。そのために、コマンドを実行します。

cacls "c：\ windows \ system32 \ cmd.exe" / G管理者：R（現在のユーザーでicaclsを実行することは許可されていないため、caclsを使用します）



ここで、新しいタスクを介してcmd.exeを実行しようとします...判明しました！

次に、オペレーティングシステムをGUIモードで実行するために必要なシステムコンポーネントをインストールする必要があります。

私たちは実施します：

cacls "C：\ Windows \ System32 \ WindowsPowerShell \ v1.0 \ powershell.exe" / G管理者：R



次に、powershellを実行して、コマンドを実行します。

Add-WindowsFeature Server-Gui-Shell、Server-Gui-Mgmt-Infra



インストールが完了するのを待って、再起動します。 これで通常のGUIモードになりました。とても便利です！

ポート3389への着信接続を許可するルールが無効になっていることに気付いたため、RDPを介してサーバーに接続できませんでした。 オンにして、RDP経由で接続します。

IISマネージャーをインストールして開きます。



[アプリケーションプール]セクションに移動し、ruvdsコンテストプールが無効になっていることを確認します。 電源を入れて...

次に、サイトのルートディレクトリに移動し、web.configファイルを開きます。



次のものが含まれます。 

<?xml version="1.0" encoding="UTF-8"?> <configuration> <system.webServer> <security> <requestFiltering> <verbs allowUnlisted="true"> <add verb="GET" allowed="false" /> </verbs> </requestFiltering> </security> <staticContent> <remove fileExtension=".png" /> </staticContent> <defaultDocument enabled="false" /> </system.webServer> </configuration> <add verb="GET" allowed="false" /> <!--        http GET- --> <remove fileExtension=".png" /> <!--        *.png --> <defaultDocument enabled="false" /> <!-- -       -->


できるだけ早くこのファイルを削除する必要があると結論付けています。

削除...



すべて準備ができているようです...アドレスYOUR_VPS_IPをオープンしようとしています。 接続を確立できません。 ファイアウォールの設定を調べる必要があるようです。

World Wide Webサービス（HTTPトラフィックイン）ルールが見つかり、パブリックプロファイルではこのルールが機能しないことがわかります。

ルールをアクティブにします。



ページを読み込もうとします-ダウンロードは開始されましたが、非常に遅いです...サイトの設定を調べる必要があるかもしれません。

サイトの制限設定に移動します。

[帯域幅の使用を制限する]と1024の値の横にチェックマークがあります。これらの設定では、ページの高速読み込みはできません。 私たちはきれいに...
