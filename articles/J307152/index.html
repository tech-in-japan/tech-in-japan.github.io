スマートフォンにSailfish OSをインストールします（例としてNexus 5を使用）





この記事では、Sailfish OSをスマートフォンにインストールする方法と、このプロセスのすべての詳細を説明します。



私はホヤから非常に最初の電話を持っていて、それが本当に好きでした。 しかし、時間は彼をspareしみませんでした：彼のSIMカードスロットが壊れて、彼はオフラインのスマートフォンに変わりました。 しばらくAndroidを使用していましたが、戻って、非公式のファームウェアの方向に目を向け始めました。 最初のJollaの携帯電話は今日、道徳的に時代遅れで、嫌なカメラがあり、鉄はもはや関係がなく、画面は低解像度です。 私のMoto G 2スマートフォンは横になっていて、その上にSFOSをインストールしようとしましたが、準備ができていませんでした（カメラが機能せず、Bluetoothが部分的に機能し、ビデオの再生やその他の多くの欠点がありません）ポーランドの作者は何とかして急いで仕上げることができませんでした。 それで、私はeBayに数週間座って、実験に使用されたNexus 5電話を見つけました。 今日は、AndroidスマートフォンにSailfishOSをインストールするプロセスのすべての落とし穴について説明しようとします。



この記事では：

  1. ポートについて少し
  2. ブートローダーのロックを解除
  3. MultiROMブートローダーを配置しました
  4. CyanogenMod（CM）とSailfishOS（SFOS）を配置します

  6. アプリケーションのインストール方法は？

ポートについて少し



そのため、愛好家によってさまざまなデバイスに移植された非公式のファームウェア（以下「 ポート 」と呼びます ）は、さまざまな電話モデルに存在します。 しかし、彼らは公式とどう違うのですか？ まず、理解する必要があります。 それらはあなたや私などの普通の人々によって書かれていること そして、これは、おそらく何かが機能しない可能性があり、何かが利用できない可能性があり、何かが更新の到着で壊れる可能性があることを意味します。 また、すべてのサポートはファームウェアの作成者に完全に依存しており、Jolla自体は概してこれとは関係ありません。 第二に、鉄の永遠の問題。 ポートはCyanogenModの上に置かれ、ドライバーを最初から作成しないようにしますが、それでもすべてが正しく機能するとは限りません。 また、CM自体は大量のバッテリー電力を消費します。最初のYollaが1回の充電から4日間私と一緒に住んでいた場合、Nexusはすでに最大で1つまたは2つを消費します。



それで、どこから始めますか？ 開始するには、リンクにある既製のポートのマトリックスをご覧ください。



wiki.merproject.org/wiki/Adaptations/libhybris



ポートがすでに書き込まれている電話、ステータス、利用可能なバージョンが表示されます。 アクティビティテーブルのおかげで、 Nexus 5を実験用に使用することにしました。これは、ファームウェアが最も安定したものの1つだからです。 スマートフォンのポートで既に行われていることと、まだ機能していない機能のリストを確認してください。 通常、インストール手順は同じですが、多くの場合、スマートフォンの特定のバージョンに固有の追加の手順がいくつかあります。 たとえば、ネクサスの説明 。



そして、あなたが強い意志があり、袖をまくり上げて、任意のスマートフォンモデルに独立してポートを作成する準備ができている場合は、ここから手順を開始し、公式チャットにもサポートがあります。 ヨラの非常にフレンドリーな従業員は、アドバイスを喜んで手伝います。



まあ、精神的に準備ができていれば、始めることができます。 コンピューターにAndroid SDKがあることを確認します（ adbおよびfastbootユーティリティが必要です）。



ブートローダーのロックを解除



最初にMotoG2のネイティブブートローダーを使用してみましたが、それは悪いアイデアであることが判明しました。 十分に実績のあるMultiROMがあり、さらに、必要に応じて複数のOSをインストールできます。 それをインストールするには、まずスマートフォンに付属のネイティブブートローダーのロックを解除する必要があります。 （まだAndroidの）電話をコンピューターに接続し、開発者モードに入り、USBデバッグを有効にして、コマンドラインに書き込みます。 



$ adb reboot bootloader




電話がfastbootモードで起動するとき、最初にコンピューターに接続された電話が表示されていることを確認する必要があります。 



  $ adb devices




そして、実際には、ブートローダーのロックを解除します： 



  $ sudo fastboot oem unlock




デバイス自体は、同意する必要がある警告メッセージを表示します（ただし、保証は失われることに注意してください！）。







MultiROMブートローダーを配置しました



まず、 twrp.meにアクセスして、携帯電話用の最新バージョンのブートローダーをダウンロードします（たとえば、Nexusの場合）。 次に、コマンドラインから次のように記述します。 



  $ sudo fastboot flash recovery ~/Downloads/twrp-3.0.2-0-hammerhead.img target reported max download size of 1073741824 bytes sending 'recovery' (14524 KB)... OKAY [ 0.665s] writing 'recovery'... OKAY [ 1.129s] finished. total time: 1.794s


CyanogenMod（CM）とSFOSを配置します



インストールを開始する前に、OSのイメージをダウンロードする必要があります。 CM11スナップショットをダウンロードします（ポートはこのバージョン用に作成されているため、11が重要です）。 Nexusの場合、 この画像を撮りました 。 もちろん、お使いの携帯電話用にポート自体をダウンロードしてください。 リンクは通常、wikiまたはフォーラムにあります。 Nexusの場合、これはそのような画像です。 両方のファイルを電話にアップロードします。 



  $ adb push ~/Downloads/cm-11-20141008-SNAPSHOT-M11-hammerhead.zip /sdcard/ 6565 KB/s (373041464 bytes in 55.489s) $ adb push ~/Downloads/sailfishos-hammerhead-release-1.1.9.28-beta0-201510022255.zip /sdcard/ 6555 KB/s (348973612 bytes in 51.983s)




Nexus5の小さな発言
執筆時点で、ビデオは最新バージョンでは壊れていました。 著者は最新のスナップショットをダウンロードすることを提案しています。これはより安定していますが、まだテスト中です。 彼の指示とアセンブリイメージへのリンクはここにあります 。 しかし、このアドバイスは、将来、すぐに時代遅れになる可能性があります。 公式ドキュメントを確認してください。





これで、リカバリモードで起動してブートローダーに入ることができます。 ここでは、電話のクリーニング、SMのインストール、SFOSのインストールの3つのことを行う必要があります。 電話の電源をオフにしてから、音量キーを押したままにして、電源を入れてブートメニューに入ります。







リカバリモードを選択し、MultiROMブートローダーに移動します。



次に、 内部ストレージを除くすべてを消去して消去します。







次に、 Installに進み、SMを見つけてインストールします。 （ 残念ながら、写真ではCM13のイメージが間違っていますが、CM11が必要です ）







最後に、同じ方法でSailfishOSをインストールします。







最後に、電話を再起動します。その後、完全に異なるスマートフォンで既に起動するはずです。 システムの最初の起動には非常に長い時間がかかる可能性があるため、心配しないでください。これは正常な動作です。 約5分かかりました。



OSバージョンを2.0に更新します



ポートは古いバージョンのSailfishOSに基づいているため、この手順はNexusにのみ必要ですが、他の電話ではこれは便利なアプローチです。 この手順では、「無線」で更新を構成する必要があります。 これを行うために、善良な人々は、実行する必要がある小さなスクリプトを作成しました。



SSH経由で電話に接続します


SailfishOSは主にLinuxであるため、最初はコンソールを使用できます。 また、接続できるSSHサーバー。 コンソールはスマートフォン自体で開くことができます。デフォルトでは使用可能ですが、電話画面で指を突くのはあまり便利ではありません。 このオプションは、手元にラップトップがない場合の緊急事態の可能性が高くなります。







2番目のオプションは、SSHを介してコンピューターから電話に接続することです。これははるかに便利です。 これは、通常の人間のキーボードとマウスを使用してコンピューターから多くの操作を実行できるため、日常の作業で非常に役立ちます。

そのため、まずはWiFiに接続する必要があります。 次に、[設定]> [開発モード]を開きます。このモードでは、ルートパスワードを設定でき、グリッド内で接続できる既製のIPアドレスを確認できます。







指定されたIPアドレスのユーザーnemoの下に移動します。







次に、電話のどこかで、 このコンテンツを使用してシェルスクリプトを作成します。



準備ができたupdate.shスクリプト 
 RELEASE=2.0.1.11 devel-su sed -i -e 's#^adaptation=.*$#adaptation=http://repo.merproject.org/obs/nemo:/testing:/hw:/lge:/hammerhead/sailfish_latest_armv7hl/#' \ /usr/share/ssu/repos.ini ssu dr adaptation0 ssu release $RELEASE ssu dr extra ssu dr mw ssu ar mw http://repo.merproject.org/obs/home:/RealJohnGalt:/testing:/common/sailfish_latest_armv7hl/ ssu er mw ssu ar dhd http://repo.merproject.org/obs/nemo:/testing:/hw:/lge:/hammerhead/sailfish_latest_armv7hl/ ssu lr # ^ double-triple check that no -two- enabled repos point to the same URL # You may have many of OpenRepos enabled. It's recommended to disable them, even # though version --dup will do its best-effort to isolate repositories: ssu lr | grep openrepos devel-su zypper clean -a devel-su zypper ref -f version --dup # if above fails, try again version --dup sync reboot

アプリケーションのインストール方法は？



さて、私たちは裸の携帯電話を手に持って、たくさんのソフトウェアを置くために手がむずむずしています。 その方法を学びましょう。 4つの方法があります。



1.ホヤストア


公式ストア。 ここで説明することはあまりありません。入って、「インストール」をクリックしてください。 追加できることの1つは、一部のポートでJollaストアが利用できない場合があることです。







2）倉庫


Linuxディストリビューションと同様に、プログラムはさまざまなソース（リポジトリ）からインストールできます。 必要なリポジトリを追加するだけで、そこからすべてのパッケージがインストール可能になります。 したがって、公式アプリストアの最も重要な類似物-サイトopenrepos.netがあります。 彼は倉庫と呼ばれる小さな電話アプリを持っています。







とてもシンプルに見えます：







最も簡単なインストール方法は、携帯電話のブラウザからこのサイトにアクセスし、 rpmパッケージをダウンロードして、ダウンロードしたファイルをタップしてインストールすることです（すべてのダウンロードは[設定]> [転送]で表示できます。ベアシステムでは、疑わしいソースからのプログラムのインストールを許可する必要があります。この機能は[設定]> [信頼できないソフトウェア]で有効にできます）。







このストアのアプリケーションを見ると、まず「リポジトリを有効にする」ボタンが表示され、作成者のリポジトリを携帯電話に追加すると、「インストール」ボタンが使用可能になります。 したがって、openrepos.netから任意のアプリケーションをインストールできます。











3）RPM


前のステップですでに気づいたように、プログラムまたはパッケージは、ブラウザー（上記のように信頼できないソースからのインストールを許可）から、またはコンソールから直接インストールできます。 コンソールからWarehouseをインストールする前の手順は、次のようになります。



curl -o warehouse.rpm https://openrepos.net/sites/default/files/packages/1/harbour-warehouse-0.3-22.armv7hl.rpm

devel-su rpm -i warehouse.rpm









4）pkcon、zypper


それでは、リポジトリの「手動」管理について詳しく見ていきましょう。 それらを管理するために、 ssuコマンドを使用できます。



インストールされているssu lrリポジトリのリスト：







同様に、いくつかのリポジトリを一時的に無効にします。 

  ssu dr home




新しいリポジトリを追加します。 

  ssu addrepo ...




pkconまたはzypperを使用してプログラムをインストールできます。 たとえば、 gPodder （ポッドキャストアプリケーション）をインストールしてみましょう。



まず、リポジトリsailfish.openrepos.net/thp/personal-main.repoを追加します（ 著者のページにあります ）。 



  [nemo@Nexus5 ~]$ ssu addrepo openrepos-thp https://sailfish.openrepos.net/thp/personal/main [nemo@Nexus5 ~]$ devel-su pkcon refresh Password: .... Refreshing software list




次に、パッケージの名前を見つけます。



[nemo@Nexus5 ~]$ zypper search gpodder

Loading repository data...

実際、zypperを使用してプログラム自体をインストールします。

  [nemo@Nexus5 ~]$ devel-su pkcon install python Password: ....   [nemo@Nexus5 ~]$ devel-su pkcon install jolla-xt9 jolla-xt9-cp


初心者向けのトップ10プログラム



この記事はSFOSの経験がない人を対象としているため、最初にセットアップする必要がある最も興味深い（私の個人的な意見では）プログラムを簡単にリストします。





1） 灯台 。

現在のプロセスの非常に便利なビューアーであり、リアルタイムでCPUの負荷、メモリおよびその他の電話リソースの使用状況も表示します。 いくつかのプロセスをネイルしたり、バックグラウンドで実行しているプロセスを確認する必要がある場合に役立ちます。





2） Meecast 。

プログラムは天気を表示するだけです。 実際、このプログラムは（当然！）公式のJollaアプリケーションストアで長い間インストールのトップにありました。 彼女に加えて、トップには電話自体に付属するヨラ自身のプログラムのみがあります。 彼女は天気を表示することしかできませんが、彼女は非常に美しくしています。 また、サポートの観点から見ると、このプログラムは長期にわたるものです。 著者はこれまでにさまざまな更新を送信しています。





3） gPodder

ポッドキャスト用のアプリ。 このプログラムは、伝説のMaemoから移植されました。 不完全ですが、現時点では最高です。





4） サイログラム

Telegramのクライアント。 非常に優れたアプリケーションです。毎日自分で使用しています。 ほとんどのネイティブTelegram機能がサポートされています。





5） OwnKeepass

KeePassXパスワードマネージャーのクライアント。 パスワードデータベースをクラウド内のどこかに保持し、そのキーをオフラインにし、パスワードを頭の中に入れておくことは非常に便利です。 使いやすいKeepassXの両方のバージョンをサポートします。





6） FileCase

ファイルマネージャー。 また、Dropboxなどのクラウドストレージでの作業もサポートしています。 どの電話でもかけがえのないアシスタント。





7） 状況

プログラムはMeegoから渡されました。 いくつかの条件（時間、場所、利用可能なネットワーク）に応じた自動モード切り替え、または作成者が「状況」と呼んでいる。 たとえば、夜間に24-00から7-00の静かなプロファイル（ネットワークとサウンドをオフにする）を有効にするとします。 Workという名前のWi-Fiグリッドが使用可能になったら、仕事中です。Workプロファイルをアクティブにする必要があります（グリッドに接続し、音を静かにします）。 バッテリーの充電が10％未満の場合、省電力モードが有効になります。 まあなど。





8） Tweetian

MiigoとSymbianの世界からやってきた伝説のtwitterクライアント。



9）カード

SFOSのカードでは、Androidのようにすべてがすばらしいとは限りません。 すべての地図は、Open Street Mapsまたはこちらに基づいています。 ここでは、Yollaのメインカード（非常に単純にMapsと呼ばれます）はここに基づいており、すべてのJolla電話にインストールされています。





次のコマンドを使用して自分でインストールできます。 

 devel-su pkcon install sailfish-maps




しかし、コミュニティから他のオプションがあります。 それらの中で最も有名なものは貧しいマップです。
