職場でプリンターをインストールするためのシステムの作成方法





親愛なる％ユーザー％こんにちは。 今日は、ユーザーが自分で必要なプリンターをインストールできるようにするためのユーザーシステムの作成方法について説明します。 私が働いている場所について少しお話します。 事前に、ブラウザーからプリンターをインストールできるWindows Server 2012 R2の役割コンポーネントについて知っていたと言います。 しかし、私は自分の自転車を発明したかったので、ここでの制限はおそらく想像上のものでしょう。 猫の下でおもしろくなりました。 辛抱強くない人のために、スクリーンショットは記事の最後にあります。

すべてが見た目ほど難しくない



システム管理者にWindows Server 2012 R2に別のサーバーを割り当てるように依頼しました。



プリンタサーバーへ



  • プリンターサーバーの役割をインストールしました
  • プリンターのインストールスクリプトを保存するスクリプト$フォルダーを作成しました
  • インストール済みのXAMPP
  • サーバーを再起動した場合にWebサイトがクラッシュしないように、Apacheとmysqlを自動ロードするサービスとして追加しました


プリンターの場合：



  • 番号の付いた手が手に届くすべてのプリンターに貼り付けて、プリンターサーバーにインストールしました。 シリアル番号は、プリンターの識別子です。 彼がどこにいても、ジュバーは常にシリアル番号で彼を見つけることができます。 例：「HP LaserJet 600 Printer＃id％filial％」、％filial％はブランチの名前、＃idはプリンターの一意のシリアル番号です。 シリアル番号の付いた紙が印刷され、プリンターに接着されます。 ユーザーはプリンタの名前を知る必要さえありません。 数はサイトにあるものと一致しました-そうです。
  • 私は支店のマネージャーに同意し、支店で知っている人を集めてほしいと頼み、プリンターの名前、IPアドレスを送って、シリアル番号を貼り付けました。 すべてのプリンターを一度に1つずつプリンターサーバーに追加しました。


プログラミングを開始



Webサイトは、メモ帳++エディター、Google、ブラウザーのみを使用して、自己記述MVCエンジンを記述してPHP上に構築されます（はい-ハードコアのみ）。 私はフレームワークの知識がなく、怠inessを研究し、2k17でjavascriptフレームワークを使用しますか？また、プリンターには多すぎますか？



このシステムから必要な目標を設定します。



  • ユーザーにとって最もシンプルで直感的な
  • 管理者アカウントを作成する機能
  • 管理者にプリンターを追加/削除する機能
  • 管理者にブランチを追加/削除する機能
  • ユーザーの参照情報の存在
  • 言語の変更
  • プリンターを1つではなくいくつかの方法（スクリプト）をインストールする機能

ルートフォルダーで、index.phpファイルを作成し、コードを追加します。 



<?php //  session_start(); //   spl_autoload_register(function ($class){ include './app/controllers/' . $class . '.php'; }); //  .        $controller = new Controller($_GET, $_POST); ?>


configsフォルダーで、database.phpファイルを作成します。 



 <?php define("HOST", "localhost"); define("USER", "root"); define("PASSWORD", "OUR_PASSWORD"); define("DATABASE", "printer"); ?>


phpmyadminでブラウザーに移動し、utf8_general_ciエンコードを使用してプリンターデータベースを作成します。 ブランチ、プリンター、ユーザーという名前の3つのテーブルを作成します。

表からわかるように、branchidは（id値を取得することにより）プリンターが属するブランチを決定します。



idが1、ブランチ名が「none」、値が「none」の画像を使用してエントリを作成します。 このエントリは、プリンタをアンインストールされたブランチに設定するために必要です。 最初のエントリはハードコーディングされており、juzverejsや管理パネルのようにブランチのリストには表示されません。 削除できません。



インターネットで見つけたDatabaseクラス。 configsフォルダーにあるdatabase.phpファイルから接続するための値を使用して、データベースに接続する機能と、データベースにクエリを実行する機能があります。



İnfoクラスは 、データベースからデータを取得、変更、削除するために使用されます。 このクラスを使用して、システム内のすべての操作が実行されます。



1つの重要なクラスはController.phpクラスです。 名前が示すように、このクラスはindex.phpから__construct（$ get、$ post）でGETおよびPOSTリクエストを受け入れるコントローラーです。 特定のリクエストを受け取った後、彼は目的のリクエストに応じてページの断片を単一に収集し、 Info.phpから必要な機能を実行します。



Viewsクラスは、ページの一部をロードするクラスです。 各ページには、Viewsフォルダー内のphpファイルに対応する独自の名前があります。 たとえば、メインページを表示するには、次のように記述する必要があります。 



 $views = new Views; $views->addView('header', 'header.php'); $views->addView('menu', 'menu.php'); $views->addView('dashboard', 'dashboard.php'); $views->addView('footer', 'footer.php');


同じクラスで、現在のユーザー言語が決定され、ローカリゼーションがiniファイルからロードされ、ページのまさにその部分で既に使用されている$ lang変数が作成されます。 メインページ（ dashboard.php ）のサンプルコード： 



 <div class="dashboard"> <div class="container"> <h1><?php echo $lang['DASHBOARD_HEADER']; ?></h1> <p><?php echo $lang['DASHBOARD_TEXT']; ?></p> <br> <br> <p><a class="btn btn-primary btn-lg" href="index.php?branches" role="button"><?php echo $lang['DASHBOARD_INSTALL']; ?> »</a></p> <img class="dashboard-img" src="app/views/img/printer_icon.png"> </div> </div>


viewsフォルダーには、css、javascriptファイルもあります。 BootstrapをCSSスタイルの基礎として使用しました。



Langクラスは、ユーザーの現在の言語を定義します。 最初に、彼はクッキー「cookie_lang」を取得しようとします。 検出されない場合は、デフォルトの言語をロシア語に設定します。 getLangArray（）関数は、目的のiniファイルをロードし、「キー」->「値」という形式の配列を返します。 この関数は、ビュークラスで使用されます。



私たちが得たもの



ユーザーはメインページに移動します。 彼がどこに着いたかという説明と、すべてのブランチのリストを開くボタンがあります。 ユーザーがブランチを選択すると、プリンターのリストが開きます。 プリンターはグリッドに表示され、写真と名前が表示されます。 ユーザーがプリンターを選択すると、選択されたプリンターのページが開き、大きな「インストール」ボタンがあります（プリンターに関する情報、そのİPアドレスもあります）。 このボタンをクリックすると、batファイルがダウンロードされ、プリンターサーバーのscripts $フォルダーで共有されているvbsスクリプトが開きます。 問題は、vbsファイルをダウンロードすると、ダウンロードする代わりに、ブラウザがこのファイルを新しいタブで開くことです。 したがって、私はとても倒錯しなければなりませんでした。 ボールフォルダーのVbsファイルも、バットファイルと共にフォルダーにソートされます。 たとえば、フォルダ構造は次のとおりです。



プリンター＃id

  • プリンター＃id.bat
  • プリンター＃id.vbs


#idはプリンタのシリアル番号です。



管理パネルから新しいプリンターを追加すると、必須フィールドに入力され、目的のバットファイルがボールから選択されます。 セットアップファイル、プリンターの写真がアップロードフォルダーにアップロードされます。 管理パネルにより、最大3つの異なるインストールファイルをアップロードできます。 ダウンロードされるファイルの数と「インストール」ボタン。 この場合、プリンタをインストールする代替方法はありませんでした。 VBSスクリプトで十分であることが判明しました。



VBSスクリプト



プリンターをインストールするVbsスクリプトは次のようになりました。 



 printerName = "\\prnserver01\HP LaserJet 600 printer1 branch1" Set WshNetwork = CreateObject("WScript.Network") WshNetwork.AddWindowsPrinterConnection printerName WSHNetwork.SetDefaultPrinter printerName
