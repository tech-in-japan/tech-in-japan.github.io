My CircleでGitHubおよびStackOverflow参加者の評価を入力します

2週間前、My Circleで、HabrとToasterの参加者の評価を紹介しました。これについては前回の出版物で説明しました 。 そして本日、GitHubとStackOverflowの2つのコミュニティに参加率が追加されたことをお知らせいたします。



ユーザープロファイルでは、次のようになります。







候補者を検索するとき、または求人を検索するとき-このように：







2つの強力な国際ITコミュニティを追加することで、開発者はプロファイルをより適切に公開でき、潜在的な雇用者とより効果的に対話できます。 そして、一般的に、私たちの意見では、これはクールです。おそらく、ITコミュニティに参加するすべてのメリットを、一度に1つの場所で1つの形式で一度に見ることができるでしょう。



評価を表示するには、各コミュニティのアカウントをマイサークルのプロフィールにリンクする必要があります。 これは、プロファイルのオフィスの「キーマスター」で行われます



次に、GitHubとStackOverflowから出力するデータとその意味を説明します。



そのため、GitHubまたはStackOverflow参加者のブロックは、ユーザーが少なくとも1つのリポジトリまたは1つの回答を持っている場合、ユーザーの「My Circle」のプロファイルに表示されます。 参加者のブロックから、このコミュニティのユーザープロフィールに移動し、どのリポジトリとどの言語でサポートされているか、どのトピック、どのように正確に回答するかを詳細に確認できます。



Github



GitHubブロックでは、ユーザーリポジトリの数とそれらへの貢献の数を示します。 ユーザーが投稿したすべてのリポジトリが考慮されます。これらは、独自のリポジトリまたはフォークされたリポジトリ、および組織に属するリポジトリです。 GitHubでの投稿と呼ばれるものとして投稿を呼び出しました 。



次に、ユーザーがリポジトリに貢献した10のプログラミング言語を表示します。 先は、より多くのリポジトリで使用されている言語です。 各言語は、ユーザーがこの言語を使用するすべてのリポジトリのリストへのリンクです。



Stackoverflow



StackOverflowブロックでは、このコミュニティのユーザーの質問と回答の数を示しています。



次に、ユーザーが最大の貢献をした10のトピックを表示し、最大の貢献から最小の貢献へと並べ替えます。 ユーザーがStackOverflowコミュニティのこのトピックで金、銀、または銅のバッジを獲得した場合、対応する色のアスタリスクでこれを示します。 アスタリスクにカーソルを合わせると、このトピックに関するユーザー統計が表示されます。 SOのバッジの詳細をご覧ください （タグバッジのセクションをご覧ください）。 トピックごとに、そのトピックに関するユーザー応答のリストにアクセスして表示できます。





リマインダー



フィルターから適切な項目「ITコミュニティのメンバー」を選択して専門家を検索すると、ITコミュニティへの参加に関する情報を持っているすべてのユーザーをMy Circleデータベースで表示できます。



ITコミュニティのメンバーのブロックは、 「キーマスター」セクションの「マイサークル」のプロファイルのオフィスで接続および切断されます。
