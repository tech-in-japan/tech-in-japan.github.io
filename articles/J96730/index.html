MonoDevelop 2.4の新しいバージョン



Monoの開発は異なる可能性があります。誰かが耐えられないのは、この技術がコミュニティの力を引き出すだけであり、より緊急なタスクに向けられるためです。 それどころか、C＃言語を使用して、Microsoft Windows、GNU Linux、Apple MacOS Xの3つの最も人気のあるプラットフォーム用のアプリケーションを作成することがプロジェクトによって可能になったことに満足しています。

おそらく、人々はこのプラットフォームを使用し、そのツールを使用します。 主なツールの1つ-MonoDevelopは新しいバージョンのリリースを発表しました。

環境自体はMicrosoft Visual Studioを視覚的にコピーし、Monoおよび.Netアプリケーションの開発環境としてだけでなく、.slnプロジェクトファイルを完全に理解しているため、MS Windowsから移行する開発者の（初めての）代替MSVSとしても興味深い場合があります

IDE公式リリース



変更の完全なリストはここにありますが、要するに：

  • 外観、新しいダイアログ、スケーリングなどの開発環境の改善
  • MS WindowsおよびApple MacOS Xコンソールのサポート
  • エディターの改善（文字の強調表示、自動補完、ツールチップなどの際の認識の改善）
  • デバッガの多くの改善を宣言しました
  • 更新された開発環境ツール、特に新しいHEXエディター
  • MacOS Xとの統合の改善
  • 6言語（日本語と中国語を含む）への翻訳を追加


WCFとの互換性の改善が宣言されていますが、私が理解しているように、完全なサポートはまだありません。

環境のインストールと操作にはMono 2.6.1およびGTK＃2.12.8が必要です



近い将来、開発者は次のことを行います。

  • gitとの統合（GitSharp経由）
  • Androidプラットフォームのサポート
  • コードテンプレート
  • C＃4の基本サポート




今後のプロジェクトの発展を祈っています。
