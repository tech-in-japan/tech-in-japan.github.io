Amazon EC2からMail.ru Infraへ：Cloud VPSのテスト（Linux）

最近、国内企業はアマゾン、グーグル、マイクロソフトの3大クラウドプロバイダーと競合していますが、当社の市場には重要なプレーヤーはほとんどいません。 基本的に、これらは比較的小規模なプロジェクト、従来のホスティング事業者（大規模なものを含む）、および/または従来のサービスパッケージに加えてクラウドを提供する商用データセンターの運営者です。



Mail.ru Cloud Solutions（ MCS ）の出現により状況は変わり始めました。ロシアのプロバイダーは、顧客に仮想インフラストラクチャ、オブジェクトおよびブロックストレージ、および他の多くのマイクロサービスを提供します。







この一連の記事では、主要な外国の記事（Amazon、Google、Microsoft）に加えて、ロシアの著名なMail.ruのライバル、Selectel、Servers.ru、およびRostelecomを取り上げますが、利用可能なサービスの範囲の点では間違いなくMail.ruに負けています-そのような豊富なオファー地元のプレイヤーのどれがしません。



まず、LinuxでクラウドVPSを評価し、Mail.ru InfraとAmazon EC2、Google Compute Engine、Microsoft Azure VM、および前述の国内企業のサービス（Selectel、Rostelecom、およびServers.comのローカル支店であるServers.ruと同じオファーを比較します） 。



登録、支払い、サポートのニュアンス



国内サービスはプリペイド方式で作業することを好みます。最初に個人口座の残高を補充し、次にサービスを取得します。 これは、ロシアの個人から借金を集めるという非常に困難な法執行機関によるものです。 そして、他の会社からお金を回収することは難しいでしょう。



大規模な西洋のサービスは後払いを受け入れており、アカウントがハッキングされた場合は、非常に短時間でスペースに「飛び込む」ことができます。 奇妙なケースがあります。ある男がAWSアカウントでハイジャックされ、クラウド仮想サーバーを使用するために12キロバケツの借金を元気に「叩き」ました （この「侵入」を救ってください）。 会社は彼ら自身の自由意志の負債を許されたが、回復することができた。



ちなみに、ここからは登録に問題があります-新しいAWSアカウント（GoogleとMicrosoftにはほぼ同じものがあります）には、銀行カードを添付する個人が必要です。 ロシアの銀行をお持ちの場合、アカウントは「一時停止」されている可能性があります。 同時に、その作業は自動的に復元されません-テクニカルサポートを作成する必要があります。 そこで、彼らはそれに応じて、多くの証拠を提示するように求められます-そして、ほとんど職場の特性から健康診断に合格する証明書 ：







企業にとって、クラウドプロバイダーがロシアの法人を持たず、当社の市場に特に関心がない場合、外国の管轄地でのサービスの支払いも容易ではありません。



さて、最後のポイント：大規模な国際クラウドサービスのサービスの価格は硬貨に結びついており、不安定なルーブル為替レートの状況では痛みを伴う場合があります。 理論的には、ルーブルで支払うことができますが、それでも為替レートで、通常は自宅で-銀行のレートでドルでより収益性が高くなります（これは練習からです）。 国内サービスを選択する別の理由は、選択するものがあるためです。



価格



もちろん、最も興味深いのは価格です。 Amazon EC2は、コンピューティングリソース、ストレージ、専用IPアドレス、および場合によってはデータ転送に対して料金を支払います。 支払いオプションには、オンデマンドインスタンス、予約済みインスタンス、スポットインスタンスの3つがあります。 Amazon EC2価格表はこちらにあります 。 たとえば、米国オハイオ州のデータセンターでは、4プロセッサの仮想マシンのコストは11.26ルーブルです。 50.66ルーブルまで （0.2ドルから0.9ドル）1時間あたり+税（Amazon価格の価格は税抜きで示されています）。 オハイオ州データセンターのEBS SSD（gp2）ボリューム用の1 GBの専用ストレージは5.63ルーブルです。 （$ 0.10）/月ですが、 他のオプションがあります 。

国内のプロバイダーに移りましょう。 関税計算機を使用することをお勧めします。個々の収益性の高い価格表を作成する方が簡単です。 Mail.ruのMail.ru Infraは 、毎分支払いで毎月の請求書を発行し、プロバイダーは年間契約で大幅な割引を提供します。 構成の価格はVATで示されます：1 GBのメモリ= 190ルーブル/月、1 CPU = 510ルーブル/月、1 GBのディスク= 7ルーブル/月。 一般的な4プロセッサ構成では、1か月あたり5430ルーブル、つまり1時間あたり7.54ルーブルかかります。



Servers.ruでは、 既製の構成のセットから選択できます 。価格は月ごとおよび時間ごとです。また、ディスクと一定量のトラフィックが価格リストに含まれています-オーバーランが課金されます。 通貨は異なるため、プロバイダーのデータセンターは異なる国にありますが、すぐにルーブルに変換されます。 モスクワのデータセンターの4プロセッサ構成では、1か月あたり3600または4800ルーブルのコストがかかります（ロシアでは、4プロセッサ構成は2つしかありません）-1時間あたり6ルーブルまたは8ルーブルです。



Selectelのメインサイトには、仮想プライベートクラウド（VPC）サービスがあります。 仮想マシンはコンストラクターを使用してアセンブルされます。必要なvCPUの数、メモリの量、ディスクのサイズとタイプ、IPアドレスの数などを指定できます。価格はすべてVATとルーブルです。 4プロセッサマシンは、1か月あたり3千以上から7000ルーブル、または1時間あたりそれぞれ約4.17および9.72+に上昇します。



Rostelecomは、ビジネス（個人と協力しない）に全国クラウドプラットフォームへのアクセスを提供します-変更のために、価格表もあります（価格は1日あたり、サービスの量に応じて毎日請求されます）。



ご覧のように、国内のプロバイダーの価格は一般に、西側の3つのプロバイダーの価格よりも低くなっています。 ロシアは明確な価格を嫌うため、これはあまり明白ではありませんが、テスト中にAmazonまたはMicrosoftのサービスに3ドルかかると、個人のアカウントに100ルーブルを落とすだけで十分でした。 さらに、当社のサービス（Mail.ruなど）は無料のインスタンスを提供します。 Amazonにもそれらがありますが、その使用には制限があるため、簡単に使用することはできません。



国内サービスでは、仮想マシンのハードウェア構成の選択が比較的少なくなっています。 インフラは楽しい例外であり、その理由は理解できます。Mail.ruはかなり大きな会社であり、この市場に真剣に照準を合わせており、十分な機会があります。

そして、ほとんどすべてのプロバイダーがそのようなデータセンターを使用しているため（これが最も一般的なレベルです）、マーケターは誘惑されます。 実際、これは神話であり、実用的な準備は理論的には計り知れません。 TIERレベルは、データセンターのエンジニアリングサブシステムのトポロジと、上位の認証段階のビジネスプロセス（運用、インシデント対応、その他）に関連しています。 この観点から、IT負荷は単に「電力を消費し、熱を発生させるため、除去する必要があります」。 この意味で、サーバーと別の電気ヒーター（ジョーク）の間に違いはありません。

通信チャネルの確認



テストのために、サードパーティのロシアのデータセンターから別の仮想マシンを取得します。これをクライアントマシンとして使用します。 まず、大小のパケットのpingなどを通じてチャネルの帯域幅を確認します。エンティティを生成しないように、すべてのプロバイダーから1つの構成を取得します。 まず、100個の小さな（64バイト）パケットと100個の大きな（1キロバイト）パケットをpingします。 パケットは負けません（0％の損失）が、応答時間（表）はわずかに異なります。 国内のサービスは予想どおり少ないです。







次に、各仮想マシンでファイルをダウンロードするためのテストがあります。



$ wget -O / dev / null- http://ftp.corbina.ru/pub/100Mb

$ wget -O / dev / null- http://cachefly.cachefly.net/100mb.test

$ wget -O / dev / null- http://speedtest.london.linode.com/100MB-london.bin







ここで、Iperfを使用して速度を確認しましょう。このプログラムはクライアントサーバーの原理で動作します。 すべての実験用ロシア語（実験の純度のため）に関連する外部ホストで、サーバーを起動します。



$ iperf -s -p 9999







次に、テスト対象の各仮想マシンからコマンドを実行します（テストには60秒かかります）。



$ iperf -c server_ip -p 9999 -t 60











もちろん、これは詳細なテストではなく、1日のさまざまな時間に実行する必要があるなど、多数の反復を行う必要がありますが、そのようなプリミティブバージョンでも、RuNetの国内サービスがよりアクセスしやすい（遅延が少なく、帯域幅が大きい）ことは明らかです。 これは私たちのサービスの貯金箱でプラスです。 深刻なロシアのプロバイダーは真っ向からやります-違いは統計誤差の範囲内です。

ドライブを比較する



シーケンシャルな読み取りと書き込みの簡単なタスクを備えた個別の仮想マシンでドライブをテストします。 Servers.ruとMail.ruにはHDDの構成がありませんでしたが、Mail.ruにはceph分散ストレージのドライブがあります。 順次読み取り速度の観点では、SSDとほぼ同じであり、順次書き込み速度では高速HDDのようなものです。







これに対して、従来のソリッドステートドライブでは、プレミアムオプション（IOPSの支払いなど）が考慮されていませんでした。



結論



一般的に、仮想クラウドサーバーMail.ruおよびその他の国内プロバイダーは、欧米の競合他社のオファーに劣りません。 もちろん、主要なグローバルプレーヤーはより多様な構成を提供できますが、その価格は高く、登録と支払いが問題を引き起こす可能性があります（特にあなたが法人であり、プロバイダーがロシアの法人を持たない場合）。



もう1つの重要なポイント：ロシアのプロバイダーの中には西部にデータセンターを持っているものもありますが、わが国の外国人3大にはデータセンターがありません。 これは、インターネットのロシアセグメントからのリソースの可用性に最適な影響を与えず（応答時間の増加）、ロシアの個人データの処理と保存に関して所有者に問題を引き起こす可能性があります-現在の法律では、サーバーが国内にあることが必要です そして現在、多くの企業がPDを保存および処理しています。たとえば、従業員や顧客のデータです。
