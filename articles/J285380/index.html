ITでのキャリア

キャリアの成功の主な基準は何ですか？ 十分な経験がない場合、希望の仕事を得る方法は？ この記事では、ITスペシャリストのキャリアに関するこれらの質問やその他の質問に答えようとしました。







最初に私のキャリアについて





現在、私はByndyuSoftのオーナー兼テクニカルディレクターであり 、 SUSUの情報学科で教師として働いており、一般的な名前のEffective Programmingの下でトレーニングを行っています。 リストの形で職業を紹介します。

  • IT会社B＆Zの創設者
  • リード開発者、fuse8
  • カラー印刷部門の責任者、Almaz印刷会社
  • タクシー運転手
  • システム管理者、WPマーク
  • キャッシャー、ピャテロチカ
  • ウェイター、アメリカのいくつかのレストラン
  • ローダー、トラクターのスペアパーツ倉庫




私はこのすべての経験に感謝しており、一部の職業が他の職業よりも重要だと言うことはできません。 私はさまざまな企業、さまざまな分野に目を向け、今ではITに落ち着きました。



№0主なものについて簡単に





私が教えられた主なことと私自身の経験から理解したことは、3つの単語の1つのフレーズで言うことができます：



関係決定



新しい接続の確立に関心があり、古い接続をサポートする人は、非常に有利な状態になります。 そして、これはITのキャリアだけでなく、一般的にあなたが行うすべてに当てはまります。



逆もまた同様です。関係をばらまき、他の人よりも自分自身を優先し、偏見や細心の注意を払って行動し、もちろん自分自身をダルタニャンと考え、他のすべてをそのように考えないでください。 悪評は急速に広まり、友人、同僚、親relativeの輪は無限ではないことを理解する必要があります。



私はこのトピックについて何度か公に話し、聴衆にこの観点に投票するように頼みました。 私の観察によると、この位置をサポートしているのは5人に1人だけです。 残りは部分的または完全に同意しません。 興味深いことに、子音の20％のうち、ユニットは実際にこの方向に移動します。



しかし、生活の中で3語の原則を適用しない人にとっては、次の2つのトピックに焦点を当てる価値があります。



№1就職





そもそも、私は自分自身の経験から、または友人や同僚の成功した経験から、これらすべての推奨事項を採用したと言いたいと思います。 ほとんどのメモは、ITだけでなく、あらゆる分野のキャリアに適しています。



経験を得るには？





就職について考えるときに誰もが興味を持っている質問は、「別の分野で勉強/仕事をしている/他のプラットフォームで仕事をしている場合、どこで仕事の経験を得ることができるのか？」 雇用主は、高い確率ですぐにタスクを解決できる専門家を雇いたいので、これは合理的な質問です。



必要な経験を得るためのいくつかの方法を次に示します。

  1. オープンソースプロジェクトへの参加

    私の友人はしばしばオープンソースプロジェクトに参加しています。 プロジェクトを作成したか、既存のプロジェクトをサポートしました。 これらのプロジェクトでは、ユーザーからのフィードバック、チームワーク、バージョンのリリース、および計画がありました。

  2. オープンソースプロジェクトコードを表示

    かつて、NHibernate、NAnt、および他のいくつかのCodePlexプロジェクトの開発を追跡しました。 開発者が行ったコミットを見ました。 すでに記述されたコードとこのコードの単体テストを理解しました。 開発者がアプリケーションを構築するための原則を理解しようとしました。 彼は、彼らが彼のプログラムでしたようにした。

  3. コースワークと卒業証書

    勉強しているのであれば、プロジェクトをより複雑で興味深いものにする素晴らしい機会があります。 5年間、ソフトウェア開発を楽しむことができます。 さまざまなアプローチを試して、プログラムを数回書き換えることができます。 大学で勉強することは、自己啓発のためのすべてを持っているユニークな時間です。

  4. フリーランス

    プログラマの多くは、友人、親relative、またはfree-lance.ruのようなサイトを通じて少なくとも1つのプロジェクトを行っていると思います。 フリーランスで作業することで、さまざまな言語を試したり、小さなプロジェクトを作成したり、既存の言語を変更したりできます。 少額の費用で、低リスクのプロジェクトを作成することで、必要な分野で経験を積むことができます。 現在、すべてのプログラミング言語とプラットフォームはフリーランスで表されているため、この機能は無視できません。

  5. 企業でのインターンシップ

    現在、多くの企業が初心者デベロッパーをインターンシップに迎えています。 インターンシップに参加するだけで、Microsoft、Intelなどの大企業で働くことができます。 もちろん、インターンシップでは、重要で大規模なプロジェクトに参加することはできませんが、経験豊富な開発者とチャットしたり、おそらく一緒に作業したりできます。 繰り返しますが、インターンシップの結果によると、彼らは仕事に招待することもできます。 弊社は学生向けのインターンシップも行っていますが、場所は限られています。





説明する人を見つける





振り返ってみると、予想外の観察をしました。 設計パターンの研究をしてください。 私はたくさんの本を読み、これらのトピックに関するビデオを見、記事を読みました。 しかし、2つの要因がなければ、この研究はすべて無に終わっていただろう。 まず、これを実際のプロジェクトに適用する必要がありました。 第二に、相談できる人、明確に説明できる人を見つけました。 そして、すべてのトピックについて。 TDD、アジャイル、DDDなどです。



あなたに説明できる人を探してください。 それらを見つけるためのいくつかの提案があります：

  1. 記事や本を書く人に手紙を書く

    公共の記事や本を書く人は、あなたからのフィードバックを喜んで受け取ります。 記事に質問や説明を書いてください。答えてくれます。

  2. 会議に行く

    現在、ITの分野では、ますます多くの会議や会議が増えています。 チェリャビンスクにのみ、 .NETconf 、 SUNETA 、 beerconfなどがあります。 モスクワまたはサンクトペテルブルクに住んでいる場合、1か月あたりのIT会議の密度は非常に高くなります。 会議に到着したら、必ず名刺を持参してください。彼らの対談者にthemすることをためらわないでください。 ラップトップを持って、コードの問題に関する関連トピックについて話しているスピーカーに尋ねてください。

  3. 先生に聞く

    繰り返しますが、学生は最高の立場にあります。 生徒には、質問に答える特別な人、つまり教師がいます。 今、私は大学で教えていますが、この特権を最大限に活用しているのはごく一部の学生だけであることに驚いています。 なんらかの理由で、ほとんどの学生は質問と明確化を恥ずかしく思います。





メンターを探すべきかどうかまだわからない場合は、有名なフレーズを引用します。「尋ねると、5分間馬鹿になります。聞かないと、一生です。」



開いて





履歴書が届くと、私が最初に行うことは、Googleで応募者の名前を入力し、彼の記事、記事へのコメント、フォーラムでの質問と回答を読み、ソーシャルネットワークのプロフィールとスピーチのビデオを見ることです。 私は私だけではないと思うので、あなたはあなたの世評について考えるべきです。 応募者の第一印象を形成するには、1つの優れた記事でも1つの優れたパフォーマンスでも十分です。



ポジティブを重視





通常、笑顔、バランス、建設的な人が好きです。 そしてその逆、悲しみ、怒り、悲観主義者はネガティブなイメージを好まないか、残しません。 ですから、将来の雇用主とコミュニケーションをとるときは、最初のグループに参加してみてください。 これは明らかなように思えますが、開発者がつまずいたトピックが1つあります。 これが神聖な戦士のテーマです。 同僚からどれくらいの頻度で聞いたことがありますか。



  • 私はWindows / Linux / Macが嫌い...
  • .NET / Java / C ++が嫌い...
  • IE / FireFox / Chromeが嫌い...




人生は常に私たちの考えよりも広いです。 プラットフォームまたはテクノロジーに対するあなたの憎しみを少なくとも中立的な立場で表現させてください。 冗談を言わないために極端に急がないでください：「あなたは猫が好きではありませんか？ 調理方法がわからないだけです。」



なぜこの仕事が必要なのですか？





HRマネージャー、プロジェクトマネージャー、または主任開発者は、仕事を探す動機付けとなった動機を理解する必要があります。 マネージャーの頭の中でネオン文字で単語が点滅します。良い候補者はやる気のある候補者です。 インタビューでは、候補者の目標について常に尋ねます。 目標の概要は1年前、理想的には5〜10年である必要があります。



誰もが目標とは何かを理解しているわけではありません。 目標の例を次に示します。

  1. 私はたくさんのお金が欲しい
  2. 旅行したいですか
  3. みんなに愛してほしい
  4. 3年後、私は100人の開発者の数と月額10万ルーブルのプロジェクトに参加したい




最初の3つの目標は計り知れないため、そうではありません。 順番に分析します。 候補者がお金を望んでいる場合、それは非常に正常です。 しかし、お金だけでは目標になり得ません。 何かが必要です。 候補者にお金がある理由は、インタビュー中に判明しました。



潜在的なプロジェクトマネージャーがインタビューのために私に来たとき、私は対話の後で拒否することに気付きました：

  • なぜプロジェクトを管理するようになったのですか？
  • みんなに私を愛してほしい。




「誰もが愛した」とはどういう意味ですか？ この「誰もが愛した」はいつ来るのでしょうか？ 誰もが私を愛し、目標が達成されたと言えるのはいつですか？



後者の目標は測定可能であるため、最も受け入れやすいと考えています。 3年後にはこれが来ないかもしれないことは明らかですが、バーが設定されて目が燃えたら、成長するあらゆる機会があります。 理解できる動機を持つ候補者は大歓迎です。



履歴書を作成するには？





雇用主が最初に目にするのは履歴書です。 要約は、ケースのみに記載する必要があります。 履歴書を表示した後、雇用主は質問に対する回答を知る必要があります。



  1. 何が欲しい？

    目標を明示的に示すと、最初の文を直接入力できます。 次に、すでに達成したことを理解する必要があります。

  2. 以前の経験を説明してください

    参加したプロジェクト、これらのプロジェクトへの個人的な貢献、責任と結果。





それは履歴書に何があるべきかについてでした。 そして今、履歴書を作成する方法は？ これらの小さなトリックはあなたを助けるはずです：

  1. 準備ができた履歴書を見つける

    現在市場に出ている履歴書をできるだけ多く収集して分析します。 これは、あなたが行きたいエリアのアイデアを提供します。

  2. 空室から要件を収集する

    さらに強力なツールで、必要な空席をできるだけ多く集めます。 候補者の要件を分析し、不足しているもの、履歴書を書くときに探すべきものをすぐに理解します。





就職面接





インタビューの心理的な要素については話さず、ほんのいくつかのヒントを紹介します。

  1. 履歴書を準備する

    はい、すでに履歴書を提出しています。 それでも、別のコピーをご持参ください。 面接後、名刺がある場合は履歴書を残し、それと一緒にすることができます。

  2. あなたの目標について教えてください。

    達成したいことについてお気軽にご相談ください。 ヒントやラウンドトリップなしで、目標について明確に教えてください。

  3. あなたの業績について教えてください。

    これらはすべて履歴書にすでに記載されていますが、自分の業績について話すときの目がきらきらと光るので、面接官は無関心になりません。

  4. インタビュー会社

    あなたは自分自身を見せるためだけでなく、面接に来ました。 あなたが職を提供された場合、あなたはこの会社で働きたいかどうかを確認する必要があります。 会社、会社の目標、計画について質問します。 あなたは多くの興味深いことを学ぶでしょう、そしてこれらの質問はあなたが週に来た他の十数人の候補者からあなたを際立たせます。





No. 2キャリアを作る





成長ベクトル





開発者の作業には、日々のコーディング以上のものが含まれます。 ソフトウェア開発に携わっているすべての人には、少なくとも10の開発分野があります。



  • ソフトウェア要件
  • ソフトウェア設計
  • ソフトウェア構築
  • ソフトウェアテスト
  • ソフトウェアメンテナンス
  • ソフトウェア構成管理
  • ソフトウェアエンジニアリング管理
  • ソフトウェアエンジニアリングプロセス
  • ソフトウェアエンジニアリングツールと方法
  • ソフトウェア品質




このトピックの詳細については、M。Connellの著書Professional Software Developmentをご覧ください。



自己啓発





求められているスペシャリストであり続けたい場合は、ITの世界を認識し、常に改善する必要があります。 陳腐に聞こえるかもしれませんが、自己開発に多くの時間を費やす必要があります。 私が役立ついくつかのポイントを強調しました：



  1. 本を読む

    私は、オンラインストアで注文する紙の本が好きです。 私は月に1〜2本ほど読んでいます。 それはもっと起こりますが、これは標準です。

  2. 記事を読む

    GoogleリーダーのブログとLJのリストは常に更新および更新されています。 記事を読むと、ITの世界のイベントやトレンドを最新の状態に保つことができます。

  3. 記事を書く

    記事を書くと、自分の頭の中で考えを合理化できます。 宣伝したくない場合は、閉じたブログまたはテキストエディターで記事を書く必要があります。

  4. 会議への出席

    私はたくさん話して、面白い知り合いを作るのが好きです。 私にとっての会議は両方の素晴らしい情報源です。

  5. 成長するために他に何が必要かを同僚から調べます

    これは、「髪の毛で自分を引っ張る」ための最も効果的で心理的に挑戦的な方法です。 何が欠けているのか、どうすれば良くなるのかを同僚に尋ねてください。 興味を持ってください-あなたは多くの興味深いことを学びます。





最初に開発者、次にマネージャーですか？





開発者は、キャリアを構築するときに、まず開発者として働き、次にプロジェクトマネージャーになると考えます。 そのような人に会ったことがありますか？ 私自身は最初に開発者として働いてから、プロジェクトマネージャーになりました。 つまり このようなスキームは機能しますが、初心者にはこの方法では明らかではない落とし穴があります。



  1. マネージャー！=開発者

    優れた開発者は優れたマネージャーにはなりません。

  2. 異なる本、異なるアプローチ

    ITプロジェクトを管理することに決めたとき、ライブラリとRSSフィードを完全に変更しました。 開発者からの移行は約2年続きましたが、今でも100％のリーダーだとは言えません。単に十分な知識と経験がないからです。 今、私にとって最も興味深いのは、心理学、プロジェクト管理、方法論、リスク管理、人事管理、要件管理などです。

  3. 責任の異なる境界

    組織の階層について話す場合、プロジェクトマネージャーと開発者は同じレベルにいます。 違いは責任範囲のみです。

  4. 誰になりたいですか？

    繰り返しますが、目標を明確に定義することをお勧めします。 1〜2年で誰に会いますか？ これに応じて、書籍、会議、RSSフィードを選択する必要があります。





技術またはアプローチと原則？





傾向があります：初心者の開発者はテクノロジー（プログラミング言語、プラットフォーム）に多くの注意を払います。 数年にわたる商用プロジェクトでの作業の後、開発アプローチと原則にさらに多くの注意が払われています。 「銀の弾丸」が一度もなかったこと、そしてプログラミング言語の選択はプラットフォームのように適用されたタスクに依存することが明らかになります。



この章を読んだほとんどの開発者は、このような優先順位の変化にも気付いたと思います。 結局のところ、技術は非常に急速に変化し、適切なアプローチは時間とともに蓄積されます。 ほとんどの時間をアプローチとベストプラクティスの研究に投資することが実用的です。



私は、「 初心者プログラマーに知っておくべきこと 」の原則を学ぶ道に着手した開発者にアドバイスする本のコレクションを持っています。



私の祖父の言葉でキャリアについての章を締めくくりたいと思います。彼は「30年まであなたは自分のキャリアのために働いていました。30年後、あなたのキャリアはあなたのために働いてきました。」



おわりに





自分に役立つものを抽出してくれたら嬉しいです。 プロとキャリアの成長に成功！
