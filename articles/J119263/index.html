Flash Player 10.3をリリースし、インキュベータープロジェクトを更新（Molehill 3Dを使用）

10.3の新機能：

- 音響エコーキャンセル 。 非常に待望の機能、FULL_DUPLEX、HALF_DUPLEXなどのサポート。私のテストによると、すべてが正常に動作し、プロセッサの負荷はほとんどありません。



- メディア測定 。 分析へのリアルタイムアクセスを提供します-ビデオの配信方法、視聴者、再生頻度。



-ネイティブコントロールパネル、MacOSの自動更新、フラッシュCookieのクリアなど、ブラウザのプライバシーとの統合。



Android、Linux、MacOS、およびWindowsには32ビットバージョンのみがあり、64ビットはFP11（3D）に暫定的にのみ対応しています。







現在、次の機能がプレビュー2で実行されています。



-Molehill 3D API 。 この技術については、すでにHabrで多くのことが言われています。



-立方ベジェ曲線。 ネイティブ曲線。



-非同期ビットマップのデコード。 プレーヤーの別のストリームで画像をデコードするこの機能は、すでにAir 2.6にあります。



-大きなビットマップのサポート。 以前のビットメイトのサイズが制限されていた場合（2880x2880）、サイズは無制限になりました（まあ、利用可能なメモリはプラグイン用だと思います）。



-JPEG-XR、ATF 2番目の形式は、テクスチャのネイティブadobe形式です。



-LZMA圧縮 。 deflateとzlibに加えて。 さらに、フラッシュドライブ自体にlzmaをパックできるようになり、サイズが縮小されます。



-G.711コーデック 。 電話などでは、PCMAとPCMUがサポートされています。



- カメラ用のH.264 / AVC SW 。 以前は、カメラからのストリームはSorenson VP6でのみエンコードされていました。



-ネイティブJSON。 ネイティブXMLに加えて。



- イベントの一時停止/スリープ/再開 。 クリップのあるタブがアクティブであり、デバイス（デスクトップ|モバイル）が現在Flashクリップで動作していることを確認するため。



-GCのヒント。 System.pauseForGCIfCollectionImminentを使用して、負荷の高い操作中にメモリを駆除しないようにGCに指示できます。 後で、GCヒントAPIが拡張されます。



-TLS（Transport Layer Security）ソケット、およびソケット進行イベント。



-安全なランダムバイトジェネレータ。



WebWorkersのサポートと新しいデータ型が将来のバージョンで期待されています：floatとfloat4。 また、開発者はWebMサポートを有効にしたいと考えていますが、期限を設定していません。VP8ハードウェアアクセラレーションの問題による疑いがあります。
