グループポリシーを使用したMicrosoft Word設定の集中展開

多くの企業では、ほとんどのユーザーにとって、ほとんどの従業員が使用するメイン（そして唯一の）製品はMicrosoft Wordテストエディターであり、Microsoft Officeソフトウェア製品の主要コンポーネントとして含まれています。 原則として、最初にインストールした後、この製品は最適に構成され、ほとんどの人が設定を変更せずに使用します。 このソフトウェア製品を詳しく調べて、少なくとも[ Wordオプション ]ダイアログボックスを開くと、Microsoft Wordには約200種類の設定が含まれており、ニーズに合わせてこの製品を直接調整できることがわかります。

家庭環境またはワークグループの一部である複数のコンピューターにこの製品のいくつかのパラメーターを設定するタスクに直面している場合、すべての設定を並べ替えて手動で1つまたは別のパラメーターを変更することはそれほど難しくないと思います。 しかし、異なるフロアの複数の部門にいる数十人または数百人のユーザーの特定の設定を変更する必要がある場合、この職業は大まかに言えば、長い時間を浪費します。

多くの人が知っているように、システム全体の設定と一部のソフトウェア製品の設定を集中的に配布するには、グループポリシーの機能を使用することをお勧めします。 この記事では、架空のバイオ医薬品会社のWindows Server 2008 R2ドメインモードで動作するActive DirectoryドメインのメンバーであるWindows 7オペレーティングシステムを実行しているコンピューターに、グループポリシーを使用してMicrosoft Word 2010設定を配布することに焦点を当てます。

Microsoft Wordの管理用テンプレートはどこにありますか？



最も可能性が高いのは、Microsoft Wordなどの製品の設定の集中展開の初期段階で、この質問が最初に発生することです。 もちろん、このような管理用テンプレートは、クライアントコンピューターのローカルグループポリシーエディタースナップイン、またはドメインコントローラーのグループポリシー管理スナップインから呼び出されるグループポリシー管理エディタースナップインにはありません。 このような管理用テンプレートは、Microsoft Officeソフトウェア製品のインストール時に最初は提供されないため、そのようなテンプレートを自分で作成するか、かなり時間がかかるか、そのようなテンプレートのセットをどこかからダウンロードしてインフラストラクチャにインポートする必要があります。

当然のことながら、既存の管理用テンプレートをインポートすることにより、数千行を書く手間が省け、時間を大幅に節約できるため、2番目のオプションの方が優先度が高くなります。 当然、マイクロソフトはこの機会を提供します。 Microsoft Office製品の管理用テンプレートのセットをダウンロードするには、 次のリンクをクリックして、環境にインストールされているMicrosoft Office製品の管理用テンプレートのセットをダウンロードする必要があります。

管理用テンプレートを含むファイルがロードされた後、必要なテンプレートをインポートする前に、それを解凍する必要があります。 これを行うには、ダウンロードしたファイルをローカライズして実行します。 表示される[ Microsoft Office 2010管理用テンプレート]ウィンドウで、使用許諾契約の条項に同意し、[ 続行 ]ボタンをクリックします。 表示される[ フォルダーの参照 ]ダイアログボックスで、ADMXファイルとADMLファイルを最初にコピーするフォルダーを選択します。 ほんの数秒で、ファイルは指定したフォルダーにコピーされます。

次の段階で、 グループポリシー管理エディタースナップインを開いたときに必要な管理用テンプレートをすぐに見つけることができるように、管理用テンプレートファイルを特定のフォルダーにコピーする必要があります。 ワークグループの一部であるコンピューターでグループポリシーを使用してMicrosoft Word設定を管理する場合は、言語に依存しない.admxファイルとADML言語リソースファイルを％SystemRoot％\ PolicyDefinitionsフォルダーにコピーする必要があります 。

Active DirectoryドメインのメンバーであるコンピューターでMicrosoft Wordを構成する場合、最初に中央リポジトリー、つまり、 ％SYSVOL％にある別のフォルダーを作成する必要があります。このフォルダーには、必要なすべてのADMXおよびADMLファイルが含まれます。 中央リポジトリを作成する手順は、グループポリシーの操作に関する以前の記事の 1つで詳しく説明されています。 次の例では、ドメインのメンバーであるコンピューターへのMicrosoft Word設定の展開について説明します。

多くの不要な管理用テンプレートがグループポリシー管理エディターのスナップインに表示されないようにするため、この段階では、同じ言語の言語リソースフォルダーのファイル「word14.admx」と「word14.adml」のみを中央リポジトリにコピーすることをお勧めしますクライアントコンピューターにインストールされているMicrosoft Office製品。 上記のすべての手順を完了したら、グループポリシーを使用してMS Word設定を展開するプロセスに直接進むことができます。
