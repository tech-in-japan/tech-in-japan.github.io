Lytroの仕組み、または別のレビュー



Lytroは、Lytro（スタートアップRen Ng -Stanford卒業生）によって製造された、焦点のないプレノプティックカメラです。 カメラはライトフィールドの方法で機能します。ライトフィールドの方法で、動作原理について説明します（トラフィックをカットします！）。

lytroからの写真がある写真はクリック可能です（ただし、最近lytroのサイトでは、嘘が発生します-おそらく今は落ちます）。



カメラ

以前のレビューで説明した外観については 、Lytro Webサイトで完全な仕様を見つけることができます 。 カメラ自体は中国製で、USB経由で接続して充電します。





ディスプレイはタッチセンシティブで、通常の解像度です。ここでは、カメラが充電中であることを示しています。





操作の原理は簡単です。上でズーム（ゴムバンドで指をスライドさせる必要があります）、上でシャッターボタン。 ズームは24-105mmに似ています。

RAW



RAWファイル自体は、マトリックスのデータです。 マトリックスのベイヤーフィルターがあり、その前にマイクロリーブがあります。





処理中

最も興味深いのは処理です。 このプロセスはRen Ngの作品に詳しく説明されています。





最初に、RAWファイルは従来の方法を使用してデモザイクされます。 その後、ライトフィールドは、「ハニカム」（マイクロレンズ）が整数ピクセルになるような状態になります。そのため、写真を少し回転させて拡大縮小する必要があります（これは、骨の折れる調整が必要な最も難しい段階です。まだできませんでした） ）

その後、ライトフィールドは特定のパラメーターと統合され、出力は異なる焦点距離を持つ一連の画像になります。 ソフトウェア自体が焦点を合わせる場所と取得する必要がある画像の数を選択することに注意してください。これを設定する方法はありません。

結果のファイルはパッケージ化され、インターネットにアップロードされます。 ユーザーがファイルセクションの1つをクリックすると（「フォーカス」）、アプリケーションはマップで画像を調べて表示します。 これは深度マップですが、非常に粗く、このような解像度での実用的な目的のために、ほとんどの場合、小さすぎるように見えます。



後処理

さて、Photoshopで何かを修正できる場合。 ここでも可能ですが、予約が必要です。

なぜなら lytro形式が記述されているため、lfpファイルのすべての部分をエクスポートできるスクリプトを簡単に作成できます。 ワークフローは次のとおりです。

  1. lfpを解析します。
  2. 画像の編集。 はい、Ivan Govnovは2回以上クリーニングする必要がありますが、コンテナ内の写真と同じくらいです。
  3. lfpを元に戻します。 ここでは、TableOfContentsセクションでの更新を含むすべてのsha1ハッシュを計算し、それからハッシュを再計算する必要があることを考慮する必要があります。
  4. Lytroアプリケーションを起動します。Lytroアプリケーションは、このファイルをクリックすると、変更されたことを認識し、プレビューを更新します。

ライトフィールドとプレノプティック写真について

クラシックカメラでは、レンズのさまざまなポイントからマトリックスの1つのポイントに入射する光が統合され、そのポイントでのピクセル輝度の値として保存されます。 プレノプティックカメラでは、マイクロレンズの配列がマトリックスの前にあります。





各マイクロレンズは、入射した光線をマトリックスの異なる部分に分配します。





f / 2アパーチャはランダムではありません。 マトリックスを最も経済的に使用できます。 少なくすると、マトリックス上に未使用のピクセルがあり、それ以上の場合-投影された画像は交差しますが、これは受け入れられません。





空間のライトフィールドは、5次元空間の関数によって記述されます。







ご覧のとおり、ライトフィールドは、空間内の各ポイントと、このポイントからのビームの各方向のエネルギー輝度（放射輝度）を表します。

プレノプティックカメラは、一連のカメラの特殊なケースとして表すことができます。マイクロレンズがカメラの役割を果たします。

写真を取得するには、カメラからのデータを統合する必要があります（lytroの場合は、マイクロレンズの領域からのデータ）。





ウィンドウ内のポイントの選択方法に応じて、2つの変換があります。



オブザーバーバイアスは、マイクロレンズ内の特定のポイントの選択です。







デジタルリフォーカス-各マイクロレンズの領域からの累積ポイントの方法の選択。 マイクロレンズの下にはピクセルがほとんどないため、リトロの場合、リフォーカスの可能性は非常に限られています。







定理
PxPピクセルが各マイクロレンズに対応するf / Nアパーチャを備えたプレノプティックカメラを使用すると、f /（N * P）アパーチャを備えた従来のカメラと同等の鮮明な画像を得ることができます。 Lytroの場合、f / 14前後です。
同じ定理の別の解釈：

定理で説明されているプレノプティックカメラを使用すると、絞りf /（N * P）のある古典的なカメラの被写界深度のどこにでも焦点が合った被写界深度f / Nの画像を取得できます。




処理の難しさ
処理時には、ライトフィールドのビジュアライザーの記述を複雑にするいくつかの障害があります。

  1. マイクロレンズは不均一です。 アルゴリズムを調整する必要があります。
  2. ケラレの影響を排除する必要があります。 しかし、わずか50ピクセルのマイクロレンズでは、この操作によって大きなエラーが発生する可能性があることが明らかになりました。
  3. デッドピクセル。 他のカメラと同様に、それらもここにあります。 深度マップを損なわないように、統計的に決定する必要があります。
  4. パフォーマンス。 4Dフーリエ変換を使用して、周波数領域で計算を実行する必要があります。
