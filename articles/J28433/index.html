Opera-9.51をリリース

Opera Desktop Teamによると、Opera 9.51ブラウザの更新バージョンが本日リリースされました。



変更点のリスト：





ユーザーインターフェース

-Operaの外観を変更しました

-タブドラッグ機能の改善

-Opera 9.2xで更新する際に発生する検索リソースのリストに関する問題を修正

-印刷またはプレビュー時のブラウザーの安定性の向上

-オペラでマウスボタンを使用してナビゲーションを有効/無効にするオプションが追加されました：config（ユーザー設定-マウスフリップを有効にする）

-キャッシュが無効になっている場合、テキスト入力フィールドは空のままです

-ページからの画像の保存がダウンロードウィンドウに表示されなくなりました



メール/ニュース

-サブスクリプションを開始するとニュースフィードが表示されます

-ドロップダウンリストを使用してメールの表示モードを切り替えるときの不安定な動作の問題

-新しいメッセージを受信する際のグループ化されたメッセージスレッドの開示を追加

-多くのフォルダにある1つのアカウントのメッセージを表示するエラーを修正



スクリプトの表示と操作

-カスタムJavaScript使用時の安定性の向上

-履歴内を移動するときにスタイルシートがロードされるようになりました

-コンテキストメニューを使用してOpera Dragonflyを閉じた後、window.close（）関数は動作可能なままです。



安全性

-RAMのランダムデータへの関数アクセスエラーを修正

-任意のコードを実行するために使用できるバグを修正しました。 エラーの詳細は後で発表されます。

-HTTPからHTTPSに切り替えるときのセキュリティステータスの切り替えを修正

-証​​明書の処理OCSPおよびCRLの機能に関連するバグを修正



その他

-Yahoo!の安定性の向上 メール

-TinyMCE 2.1.xエディターが正しく動作するようになりました

-チャット要素の印刷の改善

-チャットへの再接続の改善

-deviantart.comでのメニュー作業

-フルテキストエディターの不要な行末を修正



Winodws固有の変更リスト

-ブートウィンドウが開いているときに発生したメモリリークを修正しました

-開いたページ（URL）のアドレスの前にコマンドラインパラメーターを指定する必要があります



GNU / Linux固有の変更のリスト

-64ビットGNU / Linuxシステム用のOperaインストールパッケージを追加

-Qt4のアセンブリで不正にページを閉じる問題を修正

-プラグイン構成変更の保存の改善



Mac OSX固有の変更ログ

-クイックスタートウィンドウの「現在のページを使用」機能の改善
