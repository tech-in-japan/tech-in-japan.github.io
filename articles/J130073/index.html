Chromeリモートデスクトップ経由のリモートアクセス

Googleは、Chrome Remotingテクノロジーを実証するために、 Chrome Remote Desktop Extension（ベータ）をリリースしました。これにより、オペレーティングシステム（Windows、Linux、Mac、Chromebook）に関係なく、Chromeブラウザーを介して別のコンピューターにリモートアクセスできます。







アクセスは、1回限りのセッションでのみ有効なワンタイム認証コードを介して行われます。



この技術は、ITサポートサービスやユーザー間の直接的な相互支援に使用できる可能性があります。 （永続的な認証コードを使用して）自分のコンピューターにリモートアクセスするための機能は、少し後に表示されます。 拡張機能の現在のベータ版は、技術を実証し、ユーザーからフィードバックを収集するために作成されました。



Chrome Remotingの詳細は、chromium.orgグループメーリングリストに投稿されています。 開発者の1人は、このプロトコルはいくつかのGoogleテクノロジーに基づいていると書いています。

  1. 下位レベルでは、オープンlibjingleライブラリを使用してP2P接続が確立されます。これは、UDP、TCP、またはGoogleリレーを介したトランスポートをサポートします。
  2. この場合、PseudoTcpが使用されます-信頼できる接続を確保するためのlibjingleの実装。
  3. SSL接続がセッションを介して確立されます。
  4. Protobuf （プロトコルバッファ）は、構造化データとフレーム同期に使用されます。
  5. グラフィックはVP8コーデックによって圧縮されます。
