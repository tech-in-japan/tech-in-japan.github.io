博士 Web：サーバーを実行できません

これは、博士への公然の訴えです。 Web そして、どのようにアナリスト博士の話 Web'aは、私のサービスの500以上のクライアントのサイトをホストするサーバーを根拠なくかつ断固としてブロックしました。



簡単な紹介：私は、情報起業家向けの専門サイトを作成するための小規模サービスの所有者です。 簡単に言えば、これは起業家や個人向けのミニサイトを販売するオンラインデザイナーです。



4月27日に、クライアントから、配管会社に関する彼の名刺サイトがDr.アンチウイルスによってブロックされたというメッセージを受け取りました。 「非推奨サイト」としてのWeb。



テクニカルサポートとのやり取りのテキストがレターに添付されました（リクエストYHZJ-0382）：



リクエスト：こんにちは！ 2年目、私はDr.Webの登録ユーザーです。 ウイルス対策は常に喜んでいた。 しかし、今日、私は自分のサイトにアクセスしようとしました***メッセージが表示されました：SpIDer Gateによって非推奨サイトとしてURLがブロックされました。 私は自分のサイトを作り、自分ですべての変更を個人的に行っています。 有害なものは何もありません。 とても驚きました。 サイトがブロックされている理由を教えてください。



回答： IPアドレスはDr.Webのリストに含まれていますが推奨されていません***そして、私たちのアナリストはそれをaccから削除することを考えていません。 ブラックリスト。 疑わしいコンテンツを含む多くのリソースに加えて、新しい「ジャンクパッケージ」を作成するサービス***があります。



まあすごい！ 私のサービスはアナリストの注目を集め、非常に「客観的」かつ「合理的に」「ジャンクパッケージを作成するためのサービス」と呼ばれました。 そして罰として、アナリストはIPアドレス全体をブロックすることにしました。これには、500以上のクライアントサイトと、サービスとは関係のない他のサイト（私のブログなど）が含まれています。



この問題を解決するためにテクニカルサポートに電話します。 偽のブロックサイトの状況について説明します。 これに対して、「これはアナリストの質問です。 彼らは電話でコミュニケーションを取りません。サポートするチケットを書きます。」



サポートするチケットを作成しています（リクエスト＃3348329）：



リクエスト：こんにちは。 私は、インターネット上で迅速にページを作成するサービスの所有者です。クライアントは、あなたが彼のサイトをブロックし、同時に私のサーバーのIP全体をブロックしていると不平を言っています。



IP全体をどのような基準でブロックし、「ブロックを解除することは可能だとは思わないのですか？」

なぜコンテンツは「疑わしい」と思いますか？ そして、なぜこれがブロックする正当な理由だと判断したのですか？



人々はそこで「名刺」やビジネスサイトなどを作成します。 正直なユーザーはあなたのブロッキングのために損失を被ります。



これまでのところ、作成されたサイトのいずれかが本当に悪意があることは知りません。



さらに、このサービスは完全に公式かつ合法的に機能します。 IPのロックを解除する方法は？



回答：リクエストは分析されました。 この応答は偽ではありません。 指定したWebサイトは、Doctor Webスペシャリストによる訪問にはお勧めできません。また、そのWebサイトへのリンクはDr.Webウイルスデータベースから削除されません。



こんにちはキャップ！ 私はすでに、アナリストが私のサーバーを好きではないことに気付きました。 しかし、良識のためにチケットから少なくとも1つの質問に答えるかもしれません。



私は秘書に電話して状況を説明します。サポートは電話で問題を解決するのではなく、チケットシステムも完全に無視します。 私はチケット番号に名前を付けて待つように頼まれました...そして-サポートするために2番目のリクエストを書いてください、多分今は物事が動くでしょう。 リクエスト＃3348697を素直に書く：



リクエスト：残念ながら、以前のチケット番号3348329では、1つの質問に答えていませんでした。



それはあなたのアナリストを喜ばせなかったよりも、どのサイトが訪問に推奨されていないのかを答えてください。



サイトへのアクセスを許可するには、何をする必要がありますか？ -もちろん、これは最も重要な質問です。



正直なユーザーが正直なサイトを使用することを禁止するのはなぜですか？



さまざまなプロジェクトのサイトが配置されているのに、なぜIP全体をブロックしたのですか？



回答：リクエストは分析されました。 この応答は偽ではありません。 指定したWebサイトは、Doctor Webスペシャリストによる訪問にはお勧めできません。また、そのWebサイトへのリンクはDr.Webウイルスデータベースから削除されません。



素晴らしい。 繰り返しになりますが、私は秘書に電話し、技術サポートの完全な無視について話しています。 テクニカルサポートで質問を解決できない場合は、局長宛の手紙を書く必要があるという説明を受けます。 当然、私はすぐにそれを書いて送ります。 メッセージの内容は、リクエストの内容とほぼ同じです。



会社の従業員にも同じ質問があります。



-IP全体をブロックする正確な理由は何ですか？

-アナリストはどのような特定のコンテンツに不快感を覚えましたか？

-IPのロックを解除する方法、このために何をする必要がありますか？



ストーリー全体で強いresりを引き起こすもの：



-IPアドレス全体がブラックリストに登録されました。



彼らはにきびを見た-彼らは頭を切り落とした。 どういうわけか判明した。 すべてのクライアントが「白くてふわふわ」であるとは限りません。 しかし、たとえそれらの中に率直な詐欺師がいたとしても（これは証明されていません）-指を指差して目的を持って罰してみませんか？ このために正直なユーザーが苦しむのはなぜですか？



では、なぜ、たとえばUCOZサービスなどのブラックリストサーバーではないのでしょうか。 実際、その助けにより、「疑わしい」、さらには明らかに違法なコンテンツでさえ多くのサイトが作られました-例えば、WAREZサイト。 差別があります。



-彼らは彼らの決定に客観的な正当化を与えませんでした。



「疑わしいコンテンツ」-正確には何ですか、正確にはどこですか。 「疑わしさ」の基準は何ですか？ 会社にはそのような決定を下すための明確な内部ルールがないかのように、ある種の堅実な主観性があります。



-テクニカルサポートの質問を完全に無視します。



私は、仕事が砂糖ではないアナリストや他の従業員を人間的に理解できます。 そして、エラーと複雑なケースがあります。



しかし、500を超えるサイトで会社の製品に問題がある場合、少なくとも1つの丁寧な質問に答えることはできますか？



-総官僚制。



テクニカルサポート部門とアナリスト部門の間に関係がないとは思わない。 そして、そのような会社には、この問題をより迅速に解決するのに役立つ中間管理職がいないと思います。 いずれにせよ、私の要求はすべてのレベルで無視されました。



その結果、私は現在、この問題を建設的に、そして最も重要なこととして、迅速に解決する方法を検討しています。 これまでのところ、あなたが見るように、私はこの投稿を書き、博士の反応を見ることにしました。 Web そして、私はこの状況についてコメントし、この不快な状況から抜け出すためのヒントを提供するすべての人に非常に感謝します。
