Symfohub = symfony + github

symfonyフレームワークは、ユーザープラグインに常に多くの注意を払っているため、大好きです。 これらは常にフレームワークアーキテクチャの一部であり、公式Webサイトにはすべての可能なプラグインのカタログを含むセクションがあり、待望のSymfony2にはバンドルを含む同様のディレクトリがあることが最近発表されました。



それにもかかわらず、不快に思われることがありました。 多くのダミープラグインがあり、条件付きの分類で十分であり、プラグインの明確なグラデーションはありませんでした。たとえば、古いプラグインはより多くの人が使用し、すべてが投票したため、古いプラグインは新しいバギーよりも高くなります。



そのため、Symfonyに関連するすべてのリポジトリを収集するGitHubのメッシュプロジェクトであるLupineDreamerでSymfohubを作成しました。 SymfohubはGitHubからすべてのsymfonyリポジトリに関するデータを収集し、タグでカタログ化し、特定のプラグインまたはバンドルの使用に関するユーザーレビューを収集します。







symfohubでできることの短いリスト：

実際に機能します：

  • Twitter統合（例： symfohub.com/repo/lichess ）
  • タグと要件（リポジトリの所有者が設定）
  • ユーザー評価。 GitHubから得られるWatchers /フォークは確かに優れていますが、プロジェクトの実際の有用性を評価するのに常に役立つとは限りません。
  • railsplugins.orgに類似したユーザー安定性評価は 、アサーションと呼ばれます。 各ユーザーは、このコードが自分のために機能するかどうかを判断できます。 このようなレビューは、他の開発者が低品質のプラグイン/バンドルを使用するのを防ぎます。

  • www.symfony-project.org/pluginsにあったように、基準によってリポジトリをフィルタリングします。 バンドルについては、このセクションはまだあまり有用ではありません（シンフォニー2とドクトリンはどこにでもありますが、シンフォニー2.1の登場により、2を歌いました。
  • READMEファイルのコードの強調表示symfohub.com/repo/DbFinderPlugin/documentation
  • そしてもちろん、登録せずにレビューを残すことができるように、githubを介した認証。




Symfohubは初期の成長段階にあり、現在では約200のリポジトリがあります。 上で述べたように、symfohubは独自の方法でユーザーがフィードバックを残すのを助け、他のユーザーが他の人のレーキにつまずかないようにします。 つまり、symfonyコミュニティ内で特定の社会的機能を実行します。



現在、このプロジェクトはねじれが解消されており、残念ながら猫が泣いたときのレビューです。 そのため、コミュニティにリクエストがあります。Symfonyを使用している場合は、 Symfohubにアクセスして 、他の開発者に推奨したいプラグイン（バンドル）に評価を付けます。 それらを使用する際に落とし穴がある場合は、それらについて説明してください。感謝します。 GitHubにsymfonyコードを含むリポジトリがある場合は、それを追加してください（既に存在する場合もあります）。



symfohubプロジェクトは非営利で、楽しみとコミュニティのために作られています。 アイデアや提案がある場合-声を出してください。 忘れないでください。symfohubデータベースは常に更新されています。
