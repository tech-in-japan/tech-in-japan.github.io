すてきなPR、またはHyper-VのVDSでparking.ruから失望した方法

Parking.ruとの協力の年がどのように過ぎたかを一般の人々に伝えたいと思います。

残念ながら、その結果、プロバイダーを変更し、市のデータセンターの1つにサーバーをインストールすることで共有ホスティングを一般的に放棄しました。



さらに本文には多くの建設的な否定性があるでしょう、私は各項目に答える準備ができています。



それで、私はそれがどのように始まったのかを教えます。



2009年3月、私たちの若い会社は、最初のWebプロジェクトのホスティングプロバイダーを選択するという問題に直面しました。 たまたまASP.NET MVC + MS-SQL 2008で開発しているため、Windowsプラットフォームでのホスティングが必要です。 もちろん、検索が開始されたのは、主に考慮されたプロバイダーが推奨されているか、少なくともマイクロソフトと何らかのパートナーシップを結んでいるためです。 そして、Parking.ruが地平線に現れました。ハブには非常に多くの記事があるため、試してみることにしました。



始めるために、私たちは簡単な関税に接続しました、私たちは多くを必要としませんでした。 彼らがHyper-Vとの通信を開始するまで、通常の「Windows Server 2008でのホスティング」に落ち着きました。 すべてがすぐにうまくいき、私たちのサイトは美しく新しい場所に上昇しました。 DNSを制限なしに管理できることに満足しました。



だから私たちはしばらく住んでいた。 しかし、プロジェクトは開発中であり、完全なサーバー管理を望んでおり、許容負荷を超えたときにサイトへのアクセスをブロックすることに遭遇し始めました（ああ、この場所でホストを変更した方が良いでしょう...）。



「サービス」セクションを見て、「Hyper-Vプラットフォーム上の専用サーバー」のすばらしい選択を確認します。 解決しました！ どうぞ！ 当然のことながら、我々は低関税計画に立ち止まった。 ここで、無料の接続のプロモーションがすでに行われたこと、またMicrosoft Windows Web Server 2008のライセンスを購入する必要がないことを嬉しく思います（Parking.ruとMicrosoftとのパートナーシップのおかげです）。



新しい仮想サーバーはすぐに獲得しました。 構成、サイトの移動、起動。 やった！

サーバーは正常に動作しており、コンソールへのリモートアクセスは問題なく、支払いもなしにすべてのようです。 しかし、幸福は長続きしませんでした。 後に判明したように、年末に向けて、Parking.ruはHyper-Vクラスターの構造を変更し始め、非常に面白い結果をもたらしました。



小さな余談。 Parking.ruでホストしたサイトは、契約関係にある他の企業のサービスへのアクセス、および業務の中断を提供します-主に私たちと私たちの顧客関係に打撃を与えます。



だから私たちは主なもの、すなわち問題に着きました。 顧客からの苦情が届き始め、サイトが応答しなくなることがありました。 私たちはサーバーに行きます-彼は電源を切ることによって彼がオフにされたことを誓います。 うーん...まあ、それは起こります。 私たちはサポートに書き込みます、彼らははい、マイナーな技術的な問題があり、すべてが解決され、二度と起こらないと言います。 はい、行きましょう。



さらにもっと。 顧客からの苦情がますます頻繁に現れ始めました。 時々彼らはサイトの突然の顕著な減速、時には完全な接続拒否について不平を言いました。 私たちは把握します...再び電源を切ります。 繰り返しますが、私たちはサポートを書きます、再び標準的な答え、彼らは突然何かが壊れたと言います、すみません、それは二度と起こらないでしょう。



時間が経ちました... inりが大きくなりました。 そして、突然、新しい問題！ 私たちは小さなチーム全員で微笑み、眉をひっくり返して...愛するParking.ruからのそのような手紙を見ました： 

あなたは自分のアカウントを適時に補充しませんでした。 アカウントのすべての料金プランは、支払いが受領されるまでブロックされます。
契約番号：xxxxx
ログイン：yyyy
個人アカウントのステータス：-56,852.55ルーブル。


ええと...もちろん、私たちは皆理解しています、それは起こります。 関税計画が実際にブロックされたことは残念です。もちろん、私たちはまだ朝寝ていて、すでに仕事をしている早朝にお客様からこのことを学びました。 後で判明したように、それは請求システムの障害でした。 まあ、少なくともサーバーは生きていて、単に利用できませんでした。 修正し、お金を返し、稼いだ。



さて、もう少し余談があります。 主にプロバイダーのクライアントに必要なものは何ですか？ 私たちの意見では、信頼性やその他の明白なことは別として、手頃な価格のサポートサービスがあります。 さらに、文字通りの意味で、電話で入手できます。 Parking.ruにはこのようなサービスは一切ありません。すべてのサポートはメールでのみ提供されます。 また、問題が発生した場合は、サポートチームに手紙を書き、誰かが応答するのを待つだけです。 それまでの間、あなたはいつでもお気に入りの顧客とチャットし、「サイトが機能しない理由」というトピックについてさまざまな言い訳や言い訳を考案することができます。 一般的に、それは理解できます。あなたは自分で何かをすることができますが、...私たちのクライアントはこれまでのところ、人々を理解し、「いつ機能するのか」という論理的な質問に答えたいだけです。 そして、彼らは何に答えるべきですか？ すぐに。 答える必要はありません サポートサービスからのメールによる応答を待っています。



このような楽しい気分で、私たちは一年で最も快適な月になりました-12月まで。 一方では、新年の会議、報告会、クライアントとの借金の回収の準備をすることができます。他方では、お気に入りのParking.ru会社は、誰もが少しでも手に入らないようにHyper-Vをもう一度「セットアップ」することにしました。 この作業の結果、日中に5台のサーバーが再起動します。 20〜30分間毎回、彼はまったく答えず、その後起床し、ベースのエラーをチェックし、すべてが機能し始めました。 この作業は非常に遅いモードで開始されることがあり、サーバーが宇宙の新しい惑星を探しているように見えました。 もちろん、サポートサービスに手紙を書くことを余儀なくされるたびに、回答を待ち、その後それを読んで、すべてが修復されたことを喜んだ。



サーバーに予期しない技術的な問題がありました。 ご不便をおかけして申し訳ございません。



ある時点で、私たちは不思議に思った-それは私たちがとても不運だというだけですか？ フォーラムを検索すると答えが得られました-いいえ、私たちだけではありません。 私たちはさらに幸運です すべてのサーバークラッシュは結果なしで発生しました（MS-SQLのおかげで、期待どおりに動作します）。 他の「影響を受けた」彼はParking.ruで奇妙な問題を抱えていましたが、データを失っていました。 テクニカルサービスのリーダーの1人の電子メールを見つけました。 はい、クラスターで作業しているため、いくつかの困難があります。 しかし、すみませんが、これらの問題がすべて勤務時間中に発生するのはなぜですか？ なぜこれらすべてがHyper-Vマシンの崩壊につながるのですか？



年が終わりました。 私たちはそれにうんざりしています。 仮想ホスティングを忘れて、通常のエントリーレベルのラックサーバーを購入し、それを私たちの都市のデータセンターの1つに置くことにしました。 それが行われました。 当然、パフォーマンスの問題はすべてすぐに解消されました。 数年前から働いていたデータセンターの従業員がマシンの力でプレイされる可能性はほとんどありません。 だから今それはすべて私たちに依存しています。 そして、最良の部分は、トラフィックの支払いを考慮に入れたサーバーのホスティングのコストであり、共有ホスティングを選択する場合の同等のサービスのコストとほぼ同じです（もちろん、サーバー自体のコストを除く）。



そして最後の仕上げ。 新しいサーバーをセットアップし、DNSを転送するなど、すでに1月に、Parking.ruのHyper-Vサーバーが再び「強制終了」されました。 長くはなく、合計で約40分で、すでに慣れています。 しかし、最後の釘を打つという適切な事例がありました...サポートサービスの効率： 

 01/18/2010 17:18 support@parking.ruに連絡してください
 01/19/2010 10:26サポートサービスの回答：「現時点でサーバーの操作性を確認してください。」


うらやましい反応率。 たった17時間。 サーバー自体が非常に速く起動したことに感謝します。そうでなければ、クライアントと非常に不快な会話をする必要があります。



一言で言えば、彼らのプロジェクトのために通常のホスティングを購入するための大きな資金を持っていない同僚は、注意してください。 サポートチームを確認してください。 サポートが電子メールのみで提供される場合に起こること-私は上記で概説しました。



「そのような小銭のために何が欲しいですか？」と言う人にすぐに答えたいです：私たちがParking.ruから最も高価なホスティングオプションを取ったとしても、彼らは私たちにサポート電話を与えるか、定期的なサーバークラッシュを防ぐと思いますいくつかの仕事をするために？ 私はそうは思いません。



Hyper-Vテクノロジに反対するものは何もありませんが、まったく逆です。 しかし、そのような試用は長い間、私たちの実験への欲求を落胆させました。 おそらく、あまり重要ではないプロジェクトでそれに戻り、もちろん、Parking.ruはもはや足元にありません。
