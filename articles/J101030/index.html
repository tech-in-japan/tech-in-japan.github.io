Internet Explorerプラットフォームプレビュー4.概要

IE9開発チームは本日、IE9 Platform Previewの新しいバージョンのリリースを発表しました。







ここからダウンロードできますie.microsoft.com/testdrive （バージョン番号9.0.7916.6000）。



Platform Previewは開発者向けの特別なバージョンであることを思い出させてください。 レンダリングエンジンとjavascriptエンジン（+開発者ツール）のみが含まれ、従来のUIブラウザーは含まれません。



IE9 PP4のリリースに続いて、次の行はベータ版です。



IE9 PP4の新機能



要するに、速度の向上、標準のサポートの改善、バグ修正、新しいテストとサンプル。



Javascript


基本的に、新しいバージョンではJavaScriptエンジン自体（Chakra）に大きな変更はありませんでしたが、それでも1つの根本的なシフトが発生しています。



新しいバージョンでは、JSエンジンがブラウザに統合されているため、1つのアプリケーションで情報の中間転送と2つのDOMの調整を削除できます。







実際、IE9は、ブラウザーとスクリプトエンジンが共通のECMAScript5ベースのDOMを共有する最初のブラウザーです。 つまり、すべてのDOMオブジェクトが本格的なES5オブジェクトになりました。 DOMバインディングは、W3C WebIDL標準に基づいています。



ブラウザでDOMおよびES5サポートを確認してください 。





そしてもちろん、これらすべてがパフォーマンスに影響することはありません。





標準テスト


新しい標準サポートテスト


IE Testing Centerのテストケースのコレクションには、519の新しいテストが追加され、合計数は2138に増えました。



すべてのテストが公開されており、W3Cワーキンググループと共同で開発されていることを思い出させてください。 W3C標準のテストは、適切なワークグループに送信できます 。また、IE9でのES5サポートについては、 Microsoft Connectで送信できます 。



ACID3


ACID3のすべてのあいまいさが、標準のサポートを検証するための一連のテストであるため、新しい結果は喜ばせざるを得ません。





故障テスト：

  • テスト75に失敗しました：オブジェクトはこのプロパティまたはメソッドをサポートしていません
  • テスト76が失敗しました： '0'が期待されていましたが、 '100'が取得されました-svgアニメーション後のanimVal値が正しくありません。
  • テスト77の失敗：実装されていません
  • テスト78が失敗しました： '90'が予期されていましたが、 '1.9237771034240722'を取得しました-getRotationOfChar（0）が失敗しました。
  • テスト79の失敗：実装されていません
欠けている点は、SVGフォントとSMILアニメーションのサポートに関連しています。 これに関する詳細なコメントはIEチームブログで見つけることができます。 要するに、私たちはワーキンググループで両方の分野の弱いサポートに注目しているという事実について話しているが、Web Open Font Format（WOFF）およびCSS3との統合の文脈で積極的に議論されているいくつかの矛盾がある。



新しいデモ


IE9PP4の新しいバージョンとともに、新しいデモ例がプロジェクトサイトに登場し、 HTML5、DOM、およびE5の操作方法、およびSVGとCanvasのサポートを示しました。
