GRUB2ブートローダーを備えたシステムでのWindows 7/8 / 8.1 / 10の手動インストール

確かに、Linux OSとWindows OSのほぼすべてのユーザー、そして多くの理由で両方のシステムをインストールしているユーザーを正確に意味し、時々疑問が生じました。「可能性はあります。これらのシステムを任意の順序でインストールできますか？ ブートローダーの設定を損なうことなく？結局のところ、最初にWindowsをインストールし、次にLinuxをインストールする場合、すべてが正常です-Linuxブートローダー（たとえばGRUB）は通常bootmgrを選択します。 逆に並べると、悲しいかな。 GPT + EFIを使用しても。 EFIの場合、MBRを書き換える危険はありませんが、個人的には気に入らない点が1つあります。EFIモードのWindowsインストーラーはコードでディスクセクターを上書きしませんが、ブートローダーへのパスをスリップすることでNVRAMを再定義します。 そのため、GBUB機能を引き続き復元する必要があります。 通常のインストーラーの設定には、ブートローダーインストールプロセスをカスタマイズするオプションはありません。 しかし、標準のインストーラーを使用しない場合はどうでしょうか？！ まあ、またはほとんど使用しない...











そして、そのような方法があります。 そして、「7」から始まるディストリビューションにあるWindowsインストールテクノロジーに基づいています。 この方法は、システムのMBR + BIOSの場合と同様に、EFI + GPTの場合のように、Windowsバージョン7/8 / 8.1 / 10の場合に機能します。 HDDセクションとVHDの両方にシステムをインストールできます。 同時に、ブートローダー構成のBCDリポジトリのセットアッププログラムを実行するには、一般的なインストーラー機能が必要です。 インストールディスクの代わりに、WinPEベースのブータブルメディアを使用できます。 bcdedit.exeおよびBootICEユーティリティに* nixの代替がある場合、システムを展開するプロセス全体は、一般的にLinux環境で実行できます。







HDDにGRUB2ブートローダーがインストールされ、NTFSでフォーマットされた空きパーティションがあるコンピューターがあるとします。 Windows 7 Ultimate SP1をインストールします。 これには以下が必要です。



  1. Windowsインストールディスク
  2. 純粋なフラッシュドライブ>> 4 Gb
  3. 対応するビット深度の7-zipアーカイバ
  4. BootICEユーティリティ、望ましいが、完全にオプション、Windowsディストリビューションに含まれているbcdedit.exeで取得できます。




1.起動可能なUSBフラッシュドライブを作成する





実際、7つはDVDからEFIモードでインストールすることを望まないということです。 それはそれを望んでいません。 さらに、インストールメディアには追加のユーティリティが必要であり、そこに置く必要があり、フラッシュドライブの場合はDVDの場合よりもいくらか簡単です。



Windowsでインストールイメージをマウントします。たとえば、 / home / $ USER / win7-setup.isoという名前でインストールしてください。



$ mkdir ~/iso

$ sudo mount -t udf ~/install/win7-setup.iso ~/iso









USBフラッシュドライブをUSBポートに挿入します。 彼女がデバイス名/ dev / sdbを取得したとします。 MBR形式でマークアップしましょう



$ sudo fdisk /dev/sdb









フラッシュドライブに1つのパーティションを作成しましょう。 次に、FATでフォーマットします

$ sudo mkfs.vfat /dev/sdb1









Windowsディストリビューションの内容をUSBフラッシュドライブにコピーする

$ mkdir /tmp/flash

$ sudo mount /dev/sdb1 /tmp/flash -o uid=1000,gid=100,fmask=113,dmask=002

$ cp -rv ~/iso/* /tmp/flash/









EFIローダーの構成をより高いレベルにコピーします



$ cp -rv ~/tmp/flash/efi/microsoft/* ~/tmp/flash/efi/









次に、EFI Windowsブートローダーが必要です。 ディストリビューションから削除する必要があります。 これは、 extra / p7zipリポジトリにある7-zipアーカイバーを使用して簡単に実行できます 。 Windows配布キットのすべてのコンテンツは、インストールDVDのsource / install.wimファイルにあるwim形式のイメージに圧縮されています 。 通常、この画像の構造は次のとおりです







1〜4の形式の名前を持つディレクトリには、Windowsの異なるエディションがあります。 いずれかを選択するには、[1] .xmlという名前のファイル内を調べる必要があります。 / tmpに解凍します



$ cd /tmp

$ 7z e ~/iso/sources/install.wim \[1\].xml









テキストエディタではひどいように見えますが、特別なxmlエディタを持っていなかったため、ブラウザの助けを借りて見てください。 Windowsの各エディションは、フォームの構造によって記述されます







EDITIONIDノードはリビジョンを指します。 INDEXパラメーターは、目的のエディションが存在する電話番号です。 したがって、Ultimateが必要です。これはインデックス4です。したがって、そこからブートローダーを選択します



$ 7z e ~/iso/sources/install.wim 4/Windows/Boot/EFI/bootmgfw.efi









Okie、結果のファイルをUSBフラッシュドライブに転送します



$ mv bootmgfwm.efi /tmp/flash/efi/boot/bootx64.efi









注文してください。Windowsをインストールするための起動可能なUSBフラッシュドライブができました。 次に、インストールに必要なユーティリティ（vindosバージョンの7-zipアーカイバとBootICEユーティリティ）を追加します。 公式サイトからアーカイバをダウンロードし、USBフラッシュドライブでutilsディレクトリに解凍します



$ mkdir -p /tmp/flash/utils/7-Zip/

$ cd /tmp/flash/utils/7-Zip/

$ 7z x ~/downloads/7z1602-x64.exe









そこにブーティスも入れます



$ cp ~/install/BOOTICEx64.exe /tmp/flash/utils









すべて、USBフラッシュドライブをアンマウントします

$ cd ~

$ sudo umount /tmp/flash/









再起動します。 EFIを備えたシステムがあり、リムーバブルドライブが起動優先​​順位にある場合、コンピューターはWindowsインストールで自動的に起動します。 同じGRUBメニューが表示される場合は、問題ありません。「C」を押して、コンソールでexitコマンドを入力します。 USBフラッシュドライブからブートを選択するEFIメニューが表示されます。 システムにBIOSが搭載されている場合、USBフラッシュドライブにMBRをセットアップする必要がないため、少し複雑になります。 その後、GRUBコンソールでlsコマンドを実行します-すべてのディスクとパーティションのリストが表示されます。 フラッシュドライブを見つけると、（hdX、msdosY）の形式で表示されます。X、Yはそれぞれディスクとパーティションの番号です。 今、私たちはコマンドコンソールで運転します



grub> set root=hdX,Y

grub> ntldr /bootmgr

grub> boot









Windowsインストーラーのインストールが開始されます。 ロードされますが、必要ありません。すべてを手で行います。 何が何であるかを明確にするために、いくつかの理論を考慮する必要があります



2. WIMイメージとCapricious Sysprep TemperからのWindowsの展開について





7から始まり、Windowsのインストールは次の手順で構成されます。



  1. 機器初期化システムのアーキテクチャに従ってディスクをパーティション分割します。 BIOSの場合-インストーラーはMBRでディスクをパーティション分割します。 アクティブなEFIがある場合、マークアップはGPTになります。
  2. 選択したパーティションへのinstall.wimイメージの解凍
  3. ブートローダーのインストールと再起動
  4. 機器の初期化。 再起動
  5. ローカリゼーション、キー入力、システム設定の完了




これが標準的な方法です。 すでにディスクにマークを付けています。 また、ブートローダーの自動インストールは必要ありません-手動でインストールします。 ディスクのパーティション分割は、次の要件を満たす必要があります



  • ディスクにEFIがあるシステムが、タイプ0xEFのFATパーティションに存在する必要がある場合。 EFIブートローダーがインストールされたLinuxがすでにある場合、このセクションはすでに存在します。
  • システムにBIOSが搭載されている場合、ブートローダーが配置されているパーティション（これはNTFSパーティションです） がアクティブになっている必要があります 。 これは非常に重要です！ なぜ、以下で説明します。




したがって、システムを開梱する前に、上記の要件が満たされていることを確認する必要があります。 システムファイル全体を解凍してブートローダーをインストールすると、システムが再起動します。 再起動後、Sysprepユーティリティは、特にハードウェアのセットアップとシステム構成の確認を開始します。 したがって、EFIモードでSysprepがEFI / microsof / bootパスに沿ったEFIパーティションでブートローダーとその構成を見つけられず、BIOSシステムで\ Bootおよびbootmgrディレクトリが非アクティブパーティションにある場合、Sysprepはsysprep一般化エラーで失敗します、および\ Windows \ Panther \ sysperr.logにあるログは次のようなものを吐き出します。



2015-03-11 16:55:42, Error [0x0e00b3] TOOL Sysprep_Specialize_Bcd: There was an error opening the system store. Status=[0xC0000098]

2015-03-11 16:55:42, Error [0x0f0082] SYSPRP LaunchDll:Failure occurred while executing 'c:\Windows\System32\spbcd.dll,Sysprep_Specialize_Bcd', returned error code 1006

2015-03-11 16:55:42, Error [0x060435] IBS Callback_Specialize: An error occurred while either deciding if we need to specialize or while specializing; dwRet = 0x3ee









タンバリンとのダンスはもう役に立ちません。 Windowsでは、ローダーがブートパーティションと期間にある必要があります。 BIOSの場合、WindowsがアクティブになっているNTFSパーティションを作成し、ブートローダーを同じ場所に配置します。 とにかくGRUBを使用してロードしますが、どのパーティションがアクティブになるかは関係ありません。 EFIの場合、ローダーはEFIパーティションのGRUBの隣に配置する必要があります。
