0.メソッドの名前の構造



サーバーメソッドのリストを取得しました。次に、それらがどのように構築され、何に責任があるかを確認します。

メソッドの名前は非常に単純に構築され、次のようになります。

％primary_class_name％。％class_name％。％method_name％。



しかし、tryton開発者は、主に％module_name％。％Class_name％のようなクラス名を再定義しますが、頻度は低くなります％module_name％。％Class_name％。％Logical_class％。 例：model.res.user.createおよびmodel.stock.shipment.in.create。



合計で、メソッドの名前は次のとおりです。

％main_class_name％.. module_name％。％class_name％。％boolean_class％。％method_name％。

1.メインクラス



Trytonには5つの主要なクラスがあります。

  • model-データベースと通信するためのORMクラス。
  • report -odtやpdfなど、さまざまな形式でレポートを生成するためのクラス。
  • ウィザード -一連のステップと状態を記述する有限状態マシンを実装するクラス。
  • common-データベースおよび匿名メソッドを操作するためのクラス。
  • system-システムサーバーコールを操作するためのクラス。




一般的な方法とシステム方法については、前の部分で説明しました。 レポートとウィザードの方法については個別の記事がありますが、このパートではモデルについて検討します。

2.1。 Resモジュール ユーザーとグループを操作します。

メソッド名 パラメータ 説明

2.2。 モジュールir。 追加モジュールのインストール


irモジュールの機能は屋根よりも高いため、この部分では追加モジュールのインストールに関する機能のみを考慮します。



追加するにはインストールします。 モジュールをダウンロードし、trytond / modulesで解凍し、解凍したフォルダーの名前をモジュール名に変更する必要があります。 公式ウェブサイトで、彼らはPyPiからダウンロードすることを提案します。解凍後、フォルダーtrytond_％module_name％-％module_version％の名前を変更して、％module_name％のみを残します。 ソースを調べて、setup.pyの1つで、trytonモジュールリポジトリへのリンクを見つけました。ここからダウンロードする方がはるかに便利です。 展開すると、％module_name％-％module_revision％フォルダーが取得されますが、%% module_name％のみが残ります。



以下は、モジュールを操作するためのAPIです。 私はgtkクライアントを通してモジュールをインストールすることをお勧めしますが。 また、APIはモジュールを削除するメソッドを見つけましたが、gtkクライアントからモジュールを削除することはできません。開発者のircチャンネルには、モジュールの削除は非常に複雑で重要な手順であると書かれています。 そのため、モジュールの取り外しには細心の注意を払ってください。



メソッド名 パラメータ 説明
