Virtual Private Cloudサービスの開始





Virtual Private Cloud（VPC）サービスについてはすでに書いています。 今日、私たちはあなたの多くが待ち望んでいたニュースをようやく報告しています。新しいサービスのベータテストが終了し、商用運用を開始しています。 この記事では、新しいクラウドを使用するためのオプションと、その使用に対して支払われる料金について詳しく説明します。



なぜ必要なのか：ユースケース



前の出版物で既に述べたように、仮想プライベートクラウドサービスは企業クライアントにとって特に興味深いものになります。 当社のサービスを使用して、彼らは以下の利点を得ることができます：



  • ハードウェアコストの大幅な削減。 リースされたクラウドリソースに基づいて、数百のプロジェクトを展開できます。これは、実際のハードウェアを使用する場合に比べてはるかに安価です。高価な機器を購入したり、サービスを提供するスタッフを追加する必要はありません。
  • コスト計画の容易さ。 クラウドコンピューティングリソースのレンタルは、柔軟で透過的なスキームに従って課金されます。これにより、コストを正確に計算し、VPCが提供する節約を予測できます。
  • インフラストラクチャの展開速度。 VPCでは、必要な特性を備えた仮想インフラストラクチャを数分でセットアップできます。 適切に構成されたクラウドから、たとえば、新しいオフィスのITインフラストラクチャを簡単に展開できます。 また、スケーリングに深刻な問題はありません。必要に応じて、いつでも新しいリソースを購入できます。


企業の目的で仮想プライベートクラウドを使用するための最も一般的なシナリオをさらに詳しく見てみましょう。



部門/プロジェクトごとに個別のインフラストラクチャを構築



どの組織にも、原則として、相互接続されていない複数の部門またはプロジェクトがあります。 この場合にパブリッククラウドを使用する場合、インフラストラクチャを単一のアカウントにバインドする必要があります（セキュリティの観点から非常に疑わしい）か、多くのアカウントを登録する必要があり、これも多くの不便を引き起こします。



仮想プライベートクラウドを使用すると、説明されているすべての問題を回避できます。リソースの使用に制限されたクォータで、組織単位ごとに個別のプロジェクトが作成されます。 異なる部門のユーザーは、お互いのリソースにアクセスできなくなります。



組織の通常の従業員は、厳密に定義されたリソースをホストする厳密に定義された仮想マシンにのみアクセスできます。



管理者権限を持つ他のユーザーは、仮想マシン、ディスク、およびネットワークを管理するすべての機能を実装する外部コントロールパネルを使用してプロジェクトにアクセスできますが、リソースを購入する可能性はありません。



既存のインフラストラクチャをクラウドに移行する



多くの組織は、独自の仮想インフラストラクチャのサポートが高価になり始める状況に対処する必要があります：冗長電源とインターネット接続の組織、追加の専門家の関与、機器の購入、障害のあるコンポーネントの交換が必要です...クラウドへ。 しかし、これには多くの場合困難が伴います。すべてのソフトウェアを最初から再インストールする必要があります。 さらに複雑なことは、クラウドサービスプロバイダーが多くの制限を課すことが多く、そのためにアーキテクチャに大きな変更を加える必要があることです。



特に、独自のインフラストラクチャを私たちに移転したい企業のお客様のために、移行を促進するためにいくつかの対策を講じました。



  • クラウドには、仮想マシンの既製のイメージをインポートする機能があります（すべての一般的な形式がサポートされています）。
  • VPCのネットワークトポロジは任意に設定できます。 ローカルエリアネットワーク（または複数のローカルエリアネットワーク）をパブリックネットワークと共に使用した場合、これを簡単に実装できます。
  • 当社の仮想化システムにより、任意のOSを実行している仮想マシンを実行できます。たとえば、SolarisやFreeBSDなどが突然必要になった場合、問題なくインストールできます。


顧客向けのSaaS



何らかの理由で自社のインフラストラクチャをクラウドに移行しない会社もありますが、請負業者や顧客向けのクラウドリソースの提供を組織することを気にしません。 当社の新しいクラウドは、このユースケースに最適です。このクラウドでは、仮想ジョブを作成し、世界中のどこからでも企業情報リソースへのアクセスを整理できます。



ピーク処理



あらゆる組織の活動では、ITインフラストラクチャが最大の負荷にさらされている季節的なピークが観察されます（金融組織では、これらはレポートの準備と配信の期間、営利企業では-新年など）。 このような期間には、追加のITリソースが必要になる場合がありますが、年に2〜3週間の追加のハードウェアとソフトウェアを取得することはほとんど正当化されません。 仮想リソースをレンタルする方がはるかに簡単で安価です。仮想リソースも簡単に拡張できます。



新しいプロジェクトで実行する



多くの場合、組織が新しいプロジェクトを立ち上げ、どれだけのリソースを消費するかが明確ではありません。 もちろん、新しいプロジェクトを実行するために高価な鉄を購入することは不合理です。 しかし、既存のインフラストラクチャに基づいて実験を行うことも安全ではありません。負荷が予期せず急激に増加するリスクが常にあります。



このような状況では、クラウドコンピューティングリソースをレンタルするのが最善のソリューションです。 VPCに基づいて、ストレステストと新しいプロジェクトの実行のための実験サイトを作成できます。



クラウドでプロジェクトを実行した結果に基づいて、プロジェクトが消費するリソースの量を評価し、さらなる開発のための戦略を構築できます。



開発プロセスとソフトウェアテストの最適化



新しいクラウドは、ソフトウェアテストのプラットフォームとしても使用できます。 テスト環境が製品の実際の動作条件に近い場合、最終バージョンでエラーが発生する確率は大幅に低下します。

クラウドに基づいて、必要なパラメーターを使用してテスト環境を非常に迅速に作成できます。 テストにVPCを使用する利点には、次のものがあります。

  • 分離。 各製品は個別の開発チームによって実行されるため、個別のテスト環境が必要です。 VPCでは、テストに必要な環境で複数のプロジェクトを作成し、各チームに独自のプールへのアクセスを許可できます。 分離のおかげで、労働生産性が大幅に向上します。プログラマーが互いに干渉することはなく、システム管理者は開発者に不便を与えることなくコンポーネントの作業をデバッグできます。
  • 展開速度。 Heat Orchestrationモジュールを使用すると、あらゆる特性を持つテスト環境をすばやく作成できます。 何らかの理由でテストプールが破損した場合でも、復元することはできず、既存のテンプレートに基づいて新しいプールを展開するだけです。


テストに仮想マシンを使用すると、テスト環境の展開とテスト環境の構成の時間とコストが削減されます。



VPCに基づいて、開発チームのメンバーとユーザーの両方の製品への一般的なアクセスを整理することもできます。これにより、エラーを見つけて修正するプロセスをスピードアップできます。



再販



サービス「Virtual Private Cloud」は、インターネット起業家を始めるための素晴らしい機会を開きます：ウェブスタジオ、共有ホスティングのプロバイダー、ゲームサービスなど。 レンタルしたリソースに基づいて独自のプロジェクトを構築し、それを顧客に再販できます。

特に再販業者向けに、外部コントロールパネルをカスタマイズする機能を提供しています。 インフラストラクチャに基づいて仮想マシンが実行されていることをクライアントが認識しないように、すべてを構成できます。



関税



一般原則



サービス「Virtual Private Cloud」を使用して、自由に使用できるリソースの量を支払います。 リソースが予約された後、それらの支払いは1時間ごとに残高から差し引かれます。 リソースが使用されているかどうかに関係なく課金されます。 支払いを計算するとき、1時間あたりの各リソースの最大インジケータが考慮されます。 たとえば、1時間以内にプロジェクトクォータを数回変更すると、支払いの計算は最大値に基づいて計算されます。



各プロジェクトのコントロールパネルには、「クォータの最適化」ボタンがあります-クリックすると、消費されたリソースに応じてプロジェクトクォータを設定できます。 これにより、実際に使用されていないリソースに対して支払う必要がなくなります。



システムリソース：支払いの機能



プロセッサコア



プロセッサコアはいくつでもレンタルできます。 カーネル全体のみがレンタル用に提供され、最大100％の負荷がかかります。 予期しないリソース不足を避けるため、必要なプロセッサー時間は各ホストで予約されています。 プロセッサコアの数を変更すると、マシンを別のホストに移行できます。



記憶



プロジェクトごとに、任意の量のRAMをメガバイト単位で要求できます。 仮想マシンの作成時にメモリが完全にバックアップされます。



ディスク



クラウドでは、基本と高速の2種類のドライブをレンタルできます。

パフォーマンスのベーシックディスクは、エンタープライズクラスのSATAディスクに対応しています。 頻繁に情報を読み書きする必要のない作業を行うプロジェクトでの使用に適しています。



Fast DiskはSSDベースのソリューションです。 応答時間と速度は、ベースディスクよりも数倍改善されています。



画像保存



リポジトリには、既製の仮想マシンのイメージを操作する機能があると既に述べました。 すべての画像は特別なストレージに配置されます。 その使用は、ベーシックディスクの使用と同じスキームに従って課金されます。



フローティングIP



フローティングIPアドレスについては、以前の出版物で既に説明しました。仮想マシンの内部では、すべてのポートがプライベート（「グレー」）アドレスでリッスンされ、マシンの外部はパブリック（「ホワイト」）アドレスでアクセスできます。 このテクノロジーは、ワークロードの軽いプロジェクトに適しています。 疑いの余地のない利点は柔軟性です。フローティングIPアドレスは常に1つのマシンから切り離し、別のマシンに接続できます。



パブリックサブネット



Virtual Private Cloudサービスのユーザーに、別のVLANにある5つのパブリック（「ホワイト」）アドレスのサブネットを提供できます。 インターネットへのアクセスを提供するこの方法は、長年にわたってテストされており、より信頼性が高いようです。 最初に、5つのアドレスのサブネットを提供します。



Windowsライセンス



クラウドでOC Windowsを実行する仮想マシンを作成する予定の場合、ライセンスを別途購入する必要はありません。レンタル用に提供しています。 レンタル価格には定期的な更新が含まれます。 Windowsライセンスは、1時間ごとに固定料金で請求されます。



ネットワークトラフィック



各アカウントに対して、毎月3テラバイトの外部トラフィックが無料で提供されます。 予備的な見積もりによると、これは中小規模のほとんどのプロジェクトに十分なはずです。



設定された制限を超えるとすぐに、入金と出金の両方のトラフィックの支払いのために、1時間ごとに残高から残高が引き落とされます。



コストを最適化するために、トラフィックを事前に優遇料金で購入できます。 この提案は、主に大量のデータを扱う顧客にとって重要です。



VPCバランス



新しいプロジェクトを作成するか、既存のプロジェクトのクォータを変更すると、コントロールパネルの特別なカウンターに、残高で利用可能な資金で十分な期間が表示されます。 これにより、残高の不足によるすべてのサーバーの予期しないシャットダウンを回避できます。
