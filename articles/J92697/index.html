Lubuntu 10.04リリース

知らない人のために、 LubuntuはUbuntuとLXDEに基づいたLinuxディストリビューションです。

その主な機能は、コンピューターリソースの要件が低く、同時に、グラフィックインターフェイスがすばやく機能することです。 Lubuntuは、主に脆弱で低パフォーマンスのコンピューター（ネットブック、古いデスクトップ、古いラップトップ）に焦点を当てています。

開発者の保証によると、LubuntuはPentium-II / Celeron CPUと128MB以下のRAM（64 MBでの実行でも）を備えたコンピューターで非常によく機能します。

Lubuntuは、Ubuntuに似たLinuxディストリビューションをインストールしたいが、GnomeやKDEのような大食いのDEに多くのリソースをあきらめたくない場合にも役立ちます。

デスクトップ環境



Lubuntuは、デスクトップ環境としてLXDEを使用しています（そのため、Lubuntu = LXDE + Ubuntuという名前です）。

LXDE（Lightweight X11 Desktop Environmen）は非常に軽量で高速なDEであり、Xubuntuで使用されているXfceよりも高速です。

LXDEは、GTK +ライブラリを使用してCで記述されたオープンソースプロジェクト（GPL / LGPL）です。

LXDEは軽量であるため、グラフィック環境は弱いハードウェアでも非常に迅速に動作します。



プロジェクトの歴史

LXDEプロジェクトは2006年に軽量のグラフィカルファイルマネージャーPCManFMの開発から始まりました。これは、台湾のプログラマーであるHong Jen Yee（ニックネームPCMan）によって開発されました。 その後、他の開発者がプロ​​ジェクトに参加し、他のモジュールがデスクトップ環境用に作成され、Openboxウィンドウマネージャーと一緒にLXDEデスクトップ環境に統合されました。



LXDE、LubuntuおよびCanonical

2009年2月、Mark ShuttleworthはLXDEプロジェクトをUbuntuコミュニティに、独立したプロジェクトとして招待しました。タスクは、高速で軽量であるが完全に機能するインターフェースを備えたUbuntuおよびLXDEベースのディストリビューションを作成することです。 Launchpadプロジェクトは、Lubuntuという名前とubuntuのようなロゴを受け取りました。 プロジェクトページは wiki.ubuntu.comにもあります。



しかし、これらすべてにもかかわらず、 LubuntuはまだCanonicalの公式ディストリビューションではなく、Ubuntuファミリーのディストリビューション（現在Ubuntu、Kubuntu、Xubuntu、Edubuntuを含む）の一部ではありません。 つまり Canonical Ltd. Lubuntuディストリビューションの開発、サポート、および配布には関与しません。 Lubuntuプロジェクトは、LXDE開発者が独自に主導しています。

Wikiページの著者によって示されているLubuntuプロジェクトの主な目標の中で、Canonicalの認知度を獲得し、Ubuntuファミリーのディストリビューションに正式に参加するだけの価値があります。 これはまだ起こっていないので、明らかに、CanonicalはLubuntuがまだこれに対応していないと信じています。



2009年8月、Lubuntuを使用した最初の公開LiveCDがリリースされました。 それ以来、Lubuntuチームは定期的にアルファ/ベータビルドをリリースしました（通常、対応するUbuntuビルドから数日遅れています）。 そして最後に、Lubuntu 10.04の最終ビルドの時が来ました。



簡単なFAQ



Q： x86-64アーキテクチャ用のLubuntu ISOイメージのビルドはありますか？

A：これまでのところ、ありません。 ベータビルドは、x86（i686）でのみ収集されます。



Q： Lubuntuの最新バージョンは何ですか？

A：この記事の執筆時点では、後者はLubuntu 10.04の最終的な「安定ベータ」です（2010年4月30日）。



Q：これがベータ版の場合、Lubuntu 10.04のリリースはありますか？

A：いいえ。 実際、「安定ベータ」はバージョン10.04のリリースです。 Lubuntu 10.04の最終ビルド。 LubuntuがUbuntuディストリビューションファミリでCanonicalによって公式に採用されるまで、リリースをリリースしないという基本的な決定を著者が下しただけです。 そのため、LubuntuバージョンはUbuntuの最終リリースに基づいていますが、リリースではなく、安定したベータ版と呼ばれています。



Q： Lubuntu 10.04ディストリビューションのステータスはLTS（長期サポート）ですか？

A：いいえ。 Canonicalの長期サポートは、Lubuntuがまだ適用されていない公式のCanonicalディストリビューションにのみ適用されます。

私の一般的な印象と結論



禁欲的な角度のインターフェースはあまり気にしませんでしたが、それは素朴ですが、少なくとも弱いコンピューターではかなり許容できます。 私は、グラフィカルインターフェイスのハイライトと虹の色合いを備えたヘビー級の拳wの支持者ではないので、このインターフェイスの外観にかなり我慢することができます（さらに、私はこれに対して心の準備ができていました）。 このデスクトップの見た目、特にそのパネルとフォントは、視覚的な美学やその他の美の愛好家を混乱させると思いますが。



しかし、Lubuntuについて私が本当に気に入らなかったのは、機能性とカスタマイズの欠如でした。 ほとんどの場合、機能は使い慣れたアプリケーションをインストールすることで解決されますが、環境自体のカスタマイズ性をどうすればよいかわかりません。 つまり 不便を引き起こす迷惑な小さなものがたくさんあり、それらをどのように構成するかは明確ではありません。 たとえば、グラフィカルインターフェイスを介してキーボードスイッチングキー（入力言語）を再割り当てする方法が見つかりませんでした。



LXDEは、ある種の不完全性、不完全性、または何かの印象を残しました。 つまり インターフェイスを操作したとき、何かが欠けていて、何かが構成されていなかったという考えを忘れていました。 好きではないものを定式化することさえ難しい場合がありますが、何かが間違っています。 これは純粋に私の習慣に対する主観的な不満かもしれませんが、わかりません...



彼と数日間働いた後、少なくとも現在の状態ではLubuntuは必要ないと判断しました。

アジアの開発者が製品のさらなる開発を成功させることを心から願っています。 どういうわけか（1年ほど後）、多分私はLubuntuをもう一度見ます。 それまでの間、ありがとうございます。

合計で、3日間のLubuntu 10.04の検査と使用の後、私はため息をつき、システムを破壊しました。 さらに、私はそれを望み、非常に感銘を受けることを期待して、研究目的専用に設定しました。



一般的に、私はあなたについて知りませんが、Canonicalと私はまだLubuntuを私たちのディストリビューションにする準備ができていません;）



おわりに



私はLubuntuに私の意見を押し付けたくありません。Lubuntuのインストールを思いとどまらせたくありません。 それどころか、私もそれをインストール（またはLiveCDで試してみて）して、それを使って作業した印象を共有することは非常に興味深いでしょう。 低電力ネットブックや古代のデスクトップでの使用の実践について聞くことは特に興味深いでしょう（対応する低電力アイロンは見つかりませんでした）。 彼が彼について言っているように、彼が本当にとても速くて本当にそこにいるかどうかに興味があります。



ところで、LXDEをLubuntuと一緒にインストールする必要はありません。他のLinuxディストリビューション（Ubuntu、Debian、Mandriva、Fedora、openSUSEなど）にねじ込むこともできます。

LXDEを備えたUbuntuライクなディストリビューションが必要な場合は、すぐに幅広いプログラムがすでに用意されているので、Lucid Lynxリリースに基づくLinux Mint LXDEビルドが終了するのを待つことをお勧めします。 Linux Mint （またはロシア語の適応版 ）では、通常、システムをインストールしてすぐにすべての設備を使用したい一般ユーザーの観点からますます完成し、構成されます。
