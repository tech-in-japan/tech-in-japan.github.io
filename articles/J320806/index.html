エンタープライズデータセンターのOracle Cloud Machine

昨年、オラクルは、サービスとしてのハードウェア、サービスとしてのハードウェア、HaaSとして分類できる新しい製品であるOracle Cloud Machineを発表しました。 この記事では、提案の概念、一般的な構成、およびそれらを企業のITインフラストラクチャで使用する方法について説明します。







クラウドマシンのコンセプト



このコンセプトによれば、オラクルはハードウェアを顧客のデータセンターに移動し、価格設定はオラクルのパブリッククラウドでの提供と同様です。



このようなオファーは、顧客のローカルデータセンターでのOracleハードウェアの存在を拡大することを目的とした大胆なステップと見なすことができ、以前のオファーに対応するソフトウェアとサービスに対する支払いが行われます。



現在、288、576、1080のプロセッサコアを搭載したこのファミリの3つのマシンが利用可能です。 すべてのマシンは、X5アーキテクチャーのIntel Broadwellプロセッサーを使用しています。 2016年後半には、これらのオファーがヨーロッパ市場で利用可能になりました。



ただし、潜在的な顧客は、これが数十または数百ドルから始まる従来のプリペイドサービスではないことを理解する必要があります。1つのコンピューティングコアの支払いは月額75ドルです。



Oracleの要件は、最低3年間の顧客コミットメントであり、基本的な開始時の支払いは21.6千ドルです。同時に、クライアントはIaaSコンポーネント（ハードウェア、ストレージデバイス、ネットワーク）を受け取ります。 クライアントが追加のPaaSサービス（Java Cloud Servicesやデータベースなど）を受け取りたい場合、追加料金が必要になります。



顧客が受ける利点は、エンタープライズファイアウォールの背後にある、管理され、外部でサポートされるハードウェアシステムです。 この利点のコンポーネントの1つは、クラウドサービスとオンプレミスサービスのシームレスな統合です。つまり、顧客はオプションでクラウドでソリューションを開発し、オンプレミスで展開することができます。



また、OracleがSPARCアーキテクチャに基づいた独自のプロセッサではなく、Intelプロセッサに基づいたシステムを提供していることも興味深いです。 ただし、同社は潜在的な顧客に選択肢を残しており、OracleパブリッククラウドでSPARCベースのOracle Cloud Machineを提供することもあります。



したがって、新しいOracle Cloud Machineは、適切なIT予算でパブリッククラウドのオープン性を回避する理由と、サプライヤ企業としてのOracleへの欲求を持つ企業の専門市場を対象としたソリューションと考えることができます。



OCMの一般的な構成と使用



Oracle Public Cloud Machine（OPCM）は、Oracle Corporationが所有および運営するIaaS（Infrastructure-as-a-Service）ソリューションです。 このIaaSサービスは、顧客のデータセンターで「柔軟な」コンピューティングを提供し、独自のサイトで作業し、ファイアウォールで保護されるように設計されています。



オラクルは、OPCMハードウェアとローカルまたはリモートのゲートウェイサーバーを顧客定義のデータセンターに物理的に提供しています。

次に、OracleはGatewayサーバー、OPCM、ソフトウェアをインストールして構成し、サービスとして利用できるようにします。



このサービスは、モデル288、モデル、モデル1080の3つの構成で実装できます。オプションとして、モデル288からモデル576へ、およびモデル576からモデル1080へのアップグレードが可能です。



モデル番号は、使用されるOCPU（Oracle Cloud Processor Unit）の量に対応しています。 すべてのモデルには、OCPU B860xxコントロールユニットも含まれています。



各OCPUの処理能力は、ハイパースレッディングテクノロジーを備えたIntel Xeonプロセッサの1つの物理コアに相当します。 各OCPUは2つの命令実行スレッドに対応しています。



サービスのアクティベーション、その使用と支払い



-Oracleが特定のOPCM構成をインストールして構成し、ユーザー名とパスワードを提供した後、顧客はOPCMサービスの使用を開始します。



-OPCMサービスには、クラウドホスティングおよび配信ポリシーのドキュメントで説明されているように、クラウドのインストール、構成、パッチ適用、アップグレード、監視、および管理が含まれます。



-OPCMのサービス期間は36か月で、顧客がユーザーのログインとパスワードを受け取った日から始まります（注文で特に指定されていない限り）。



サービス期間の終了から60日後、同じOPCMハードウェアに対する新しいOPCMサービスオーダーがない場合、OracleはOPCMのディスク、フラッシュカード、ストレージコンテナからすべてのデータを消去します。 Oracleは、クライアントデータセンターからOPMCハードウェアとゲートウェイも解体します。



Oracle Public Cloud Machine ZS3



Oracle Public Cloud Machine ZS3は、ファイアウォールで保護された情報スペース内で、顧客のサイトに必要な量のデータストレージをサービスとして提供するように設計されています。 このオファーには、モデル268とモデル536の2つの構成があります。



月額料金で、お客様は選択したOPCM ZS3構成でストレージシステム全体（冗長性、キャッシュ記録、制御データを除く）を使用できます。



Oracle Public Cloud Machine ZS3 Model 268には、268 TVストレージシステム（データの冗長性、キャッシュの書き込み、管理データを含む）が含まれています。 ユーザーは120 TBを使用できます。



Oracle Public Cloud Machine ZS3 Storage Expansion Model 536には、同様の536 TVストレージシステムが含まれています。 ユーザー240 TVで利用可能。

OCMを使用した練習



過去数年間で、企業や組織の情報システムによって処理される情報量が急激に増加しました。 これは、「データ爆発」とそれに対応するデータ操作用のクラウドサービスの増加によるものです。 Gartnerは、2016年にグローバルクラウドサービス市場を2,040億ドルと評価しました



ただし、多くの企業のコンピューティングワークロードは、顧客のサイトにあるオンプレミスのデータセンターで処理され、ファイアウォールによる外部の影響から保護されています。 アナリストの推定は大きく異なりますが、企業データのオンプレミス処理が世界中で最大75％に達すると主張する人もいます。





企業のワークロードの4分の3は、まだ内部データセンターにあります。 アナリストによると、これは「このような企業を業界の底辺に引き寄せる」。



IDCのレポート「クラウド市場の展望：クラウド購入者のセグメント化」では、企業の18％がクラウドサービスを1〜2の比較的小さな企業ワークロードにのみ使用しており、回答者の20％がまだ展開の計画段階にあると述べています。



長時間実行されるワークロードの移行とオンプレミスワークロードシステムに深く組み込まれている問題と複雑さに加えて、多くの企業は、パブリッククラウドの重要なビジネスアプリケーションとデータにセキュリティの問題を抱えています。



オラクルの公式ウェブサイトのブログ「Oracle Cloud Machine：Oracle Cloud At Your Datacenter」で、彼女のプリンシパルプロダクトマーケティングディレクターのエドガー・ハーレンは、このような状況がコンセプトを構築し、Oracle Cloudハードウェアおよびソフトウェアコンプレックスを作成するきっかけとなったと述べています機械



Harenは、「柔軟で効率的なクラウドサービスを必要とするが、独自のデータセンターからこれらのサービスの高レベルのセキュリティと管理を行う顧客向けの新しい破壊的なソリューション」と定義しています。



企業のIT市場には、コンピューティングへのさまざまなアプローチを必要とする企業や組織のセグメントがかなりあります。 現在、このような各セグメントには、ビジネス要件および規制要件によりパブリッククラウドを利用できない多数の企業が含まれています。



その結果、彼らは急速に老朽化する技術に目を向けます。これには、外部の影響から十分に保護された、すべての（またはほぼすべての）コンピューティングパワーのサイトへのインストールが含まれます。 多くの場合、このようなクライアントは、ヘルスケア、金融、政府で働いています。





会社が提供するOracle Cloud Machineの3つのオプション。



Oracle Cloud Machineは、企業ファイアウォールの背後にあるPaaSおよびIaaS機能を提供します。 これにより、ワークロードとそのモビリティレベルを管理する方法を選択し、パブリッククラウドとオンプレミスコンピューティングパワーの間でバランスをとることができます。



その結果、ビジネスは自社のデータセンターでのクラウドイノベーションの恩恵を受け、開発者はアプリケーションを迅速に構築、テスト、展開する機会を得ます。 OCMはサブスクリプションモデルに従って支払われるため、これにより、クライアントはパブリッククラウドとオンプレミスの両方で単一のサプライヤーの責任を負うことになります。



要約すると、企業クライアントにOCMを使用する主な利点を簡単に説明できます。



1. Oracle Cloudは、企業ファイアウォールの背後にあるサービスとして動作します。



2. Oracle Cloudと同じPaaSおよびIaaSソフトウェアと同じ更新が使用されます。



3. Oracle Cloudと同じ費用対効果の高いサブスクリプション価格設定モデルが使用されます。



4.ハードウェアとソフトウェアの複合体全体が、法規制、法律、ビジネス、プライバシーの要件に準拠しています。



5.クライアントは、重要なビジネスシステムに対する制御を保持します。 独自のファイアウォール、ロードバランサー、VPNハードウェアなどを使用できます。

多くのお客様は、クラウドへの移行がもたらす可能性のある結果を検討し、多くの場合、オンプレミスプラットフォームと検討中のクラウドプラットフォームとの間に十分な互換性がないことに気付きました。 Oracle Cloud Machineはこれらの問題を解決します。





Oracle Cloud Machineに関する一般的な顧客の問題とソリューション。



顧客がハイブリッドクラウドモデルの使用を開始しようとしているか、すでに自分のサイトの外部で実行されるワークロードの一部を持っているかどうかにかかわらず、Oracle Cloud Machineは、データベース、ソフトウェア、およびアプリケーションのオンプレミスおよびクラウド展開の互換性を保証する独自のソリューションを提供します。
