Siemens電話からLinuxへのファイルの転送-SieFS





まえがき



約1週間前、電話専用に使用していた2台目の電話を紛失しました。 もちろん、彼と一緒に、SIMカード。 今日だけそれが復元され、余分な電話から古いほこりの多いシーメンスCX70だけが見つかりました。 最初は、彼は電源を入れませんでしたが、彼の連絡先を掃除し、夜の充電は彼を復活させました。 そして、疑問が生じました-2005年のリリースのこの奇跡をどのように接続するのですか？ WifiもBluetoothもありません。IrDaとコードを接続するためのプラグしかありません。 多くの時間が経過し、赤外線ポート用のアダプターが見つかりませんでしたが、シーメンスのデバイスを接続するコードが見つかりました。 そして、インストールの問題を思い出しました-デバイスのドライバー、ソフトウェア、そして転送中の不​​便。 現在、ArchLinuxとWindows 8があり、これらのドライバーはほとんど巻き上げられません。 幸いなことに、SieFSパッケージを発見しました。





インストールと使用



SieFSの最新バージョンは2005年にリリースされました。 また、著者のサイトのシステム要件から判断すると、カーネルバージョン2.4。*、マウント2.11以降、およびFUSE 2以降が必要です。 必要なコアにもかかわらず、すべてが調整されています。 ちなみに 、私はsiefsパッケージをaurに隠し、新しいシステムで動作するようにエラーを修正したので、ArchLinuxユーザーに必要なコマンドは1つだけです。 

yaourt -S siefs


次に、-t siefsオプションを使用してマウントします。 デバイスは通常/ dev / ttyUSB0です。 結論dmesgによって正確に決定できます。 このようなものを尾

USB 3-1：pl2303コンバーターがttyUSB0に接続されました

マウントの例： 

 sudo mount -t siefs /dev/ttyUSB0 /mnt


他のディストリビューションの場合、より多くの手順を実行する必要があります。 libfuse-devをインストールします。build-essentialはディストリビューションに依存します。 ソースをダウンロードし、解凍し、変更を加えて構成し、ビルドしてインストールします。 

 wget http://chaos.allsiemens.com/download/siefs-0.5.tar.gz tar -xvf siefs-0.5.tar.gz cd siefs-0.5/ LDFLAGS="-ldl -lrt" ./configure make


さらにインストールを行いますが、debディストリビューションcheckinstallのユーザーには、パッケージを作成して個別にインストールすることをお勧めします（他の人にアナログを見つけることをお勧めします）。 念のため、 rghost.ru / 39061415 （ Yandexディスクのミラー ）-パッチ。 次のようにします-patch -p1 -i ../siefs.patch cd siefs-0.5 /の後、configureの前。 （システム修正ポストを確認してください。）



おわりに



ボーナスとして、レコーダーからの録音をWAV形式に変換できます。 

 vmo2wav [-o output_file.wav] input_file.vmo


また、IrDaを見つけた場合は、irda-toolsまたはirda-utilsをインストールする必要があります。使用しているパッケージを確認してください。

次に、デバイスを作成します。 

 mknod /dev/ircomm0 c 161 0 mknod /dev/ircomm1 c 161 1 mknod /dev/irlpt0 c 161 17 mknod /dev/irlpt1 c 161 17


アダプターを接続し、電話でIrDaをオンにして、次の操作を行います。 

 irattach irda0 -s mount -t siefs /dev/ircomm0 /mnt


何かが機能しない場合は、トラブルシューティングのためのirdadumpとirdapingがあります。



また、ArchLinux i686用に組み立てたパッケージとミラーも提供します 。

古いデバイスを忘れたり捨てたりしないでください。それらはあなたにとって有用かもしれません。 たとえば、私はシーメンスでプログラムや写真の古いソースを見つけましたが、品質はあまり良くありませんでした。 さらに、バッテリーの寿命が非常に長いことを自分で発見しました。1日を超えないすべての新しいデバイスに慣れているので、ここで過去からの驚きを感じました。
