UnrealEngineの紹介。 パート2

最後の部分では、UnrealEngineでのゲームがどのようなものかを学び、ジオメトリを構築し、アクターを配置する方法を学びました。 UnrealEngineとUnrealRuntimeでのゲームを完了すると、風景、あらゆる種類のトリガー、武器、その他の便利なものなど、非常に多くの標準的なアクターが存在します。 適切に使用すれば、ゲームのさまざまな興味深いレベルを作成できますが、これは創造性の完全な自由を与えるものではありません。 ゲームには標準のスタート、勝ち負けの標準ルールがあり、追加のコントロールボタンを入力することさえ不可能です。 ここで、UnrealScriptに慣れるときが来ました。 あなたがUnrealScriptの第一人者である場合は、すぐに予約をしてください。そうすれば、おそらく興味がなくなるでしょう。 残りは猫の下で歓迎します。



言語の基礎



UnrealScriptはオブジェクト指向言語であり、構文的にはJavaに似ています。 この言語は、多重継承インターフェースをサポートしていません（第3バージョンからのサポート）。 演算子のオーバーロードはサポートしていますが、メソッドはサポートしていません。 構文全体を書き直してください。理由はわかりません。 プログラミング言語に精通していることを願っています。 したがって、主なポイントについて説明します。



データ型


UnrealScriptには、次の単純なデータタイプが含まれています。

  • bool-タイプはtrueまたはfalseです
  • byte-値の範囲が0〜255の符号なし整数
  • int-値の範囲が-2 147 483 648〜2 147 483 647の符号付き32ビット整数
  • float -32ビット浮動小数点数
  • string -unciode文字のストリング
  • enum-列挙型。名前付き定数のセットを含みます。最初の定数は0、2番目の定数などです。
  • name-何かの名前を示す型、例えば、状態、静的なミックス、またはクラスの名前


参照データ型も存在します。

  • オブジェクト - オブジェクトへのリンク、つまりリンク、つまり、いくつかの変数が同時に同じオブジェクトを指すことができます
  • Actor -Actorクラスのオブジェクトへの参照、またはそれから継承された
  • クラス -リンクは、それから継承されたクラスのタイプを示します


複合タイプ：

  • 構造体 -固定数のフィールドで構成される構造
  • 配列 -静的配列

状態と潜在機能


前述のすべてがすべてのOOP言語で見つかったが、状態（状態）がUnrealScriptの機能である場合。 ゲームの仕組みでは、多くのことが状態に基づいて構築されています。 たとえば、あなたのキャラクターはいくつかの状態にあります：立っている、座っている、うそをついています 各状態では、たとえば、移動速度、ジャンプする能力など、いくつかのオプションが異なります。 これを実装するためのオプションがいくつかあります。たとえば、ジャンプチームがプレーヤーの現在の状態をチェックして、それができるかどうかを確認してから、ジャンプするか、単に全力で立つかを決定します。 UnrealScriptは別のオプションを提供します。 クラスの状態を説明し、これらの状態では、プレーヤーがこの状態にある場合に通常の関数の代わりに呼び出される関数を説明します。 状態は次のように説明されます。 

 [modifiers] state statename [extends parentstate] { ... };


modifiers



修飾子、今回は2つのみ：

  • Auto-クラスのインスタンスは、作成されると自動的にこの状態になります。
  • シミュレート -レプリケーション中にクライアントで状態を実行できることを示します
statename



州名

parentstate



別の状態を拡張することを示します。

非表示のテキスト 
 class example extends actors; function test(){ log("No state"); } state StateOne { function test(){ log("In state one"); } } state StateTwo { function test(){ log("In state Two"); } } state StateThree { function test(){ log("In state Three"); } }

その結果、オブジェクトがStateOne状態になると、コンソールにGoing into StateOne行が表示されます。

条件は非常に柔軟なものですが、いつ、何が起こるか混乱しないように慎重に作業する必要があります。



また、UnrealScriptにはそのような概念があります-潜在的な機能です。 潜在関数は、完了するまでに一定の再生時間を要する関数です。 たとえば、一定時間コマンドの実行を一時停止するスリープ機能。 潜在関数は、ステート内でのみ呼び出すことができます。 これまでのところ、状態が役に立つとは考えられませんが、特に継承されることを考慮して、状態を覚えておく必要があります。



スクリプトを使用する



ご覧のとおり、UnrealScript言語は他のOOPに似ており、自然なものを超えるものはありません。 多くのニュアンスがありますが、一般的には特別なものは何もありません。彼らはそれを恐れていません。 スクリプトの作成とコンパイルの方法を考えてみましょう。そのうちの1つはUnrealEdを使用する方法が2つありますが、これは最も便利なオプションではないため、 uccを使用してこれを行う方法を学習します。



パッケージとは何か


UnrealEngineは、すべてのリソース（テクスチャ、モデル、サウンド、音楽（常にではない）、スクリプトなど）をパッケージに保存します。 タイプに応じてパッケージの拡張子はutx / umx / usx / uと異なる場合がありますが、これらはすべて同じパッケージです-便宜上、分離されています。 最初のインシデントはこれから発生し、同じ名前の2つのパッケージを作成することはありませんが、拡張子が異なるため、エンジンは通常、そのうちの1つだけを認識します。 2番目のポイントも表示されます-パッケージには何でも保存できます（つまり、サウンド、テクスチャ、およびスクリプトが同時に存在する可能性があります）。

階層と一般的に使用されるクラス



UnrealEngineのクラス階層について説明します。 ルートクラスはObjectクラスであり、すべてのメインクラスはすでにそこから継承されています。 オブジェクト-エンジンの基本構造といくつかのシステムメソッドの説明が含まれています。 ゲームパッケージを再構築しないでください。ほとんどのパッケージにはネイティブコードへのリンクが含まれているため、収集できません。 2番目に重要なのは、アクタークラスです。 レベルに追加できるすべてのオブジェクトの基本的な機能（プロパティ、移動、物理、複製など）について説明します。 レベルに追加される他のすべてのオブジェクトは、このクラスから継承されます。 以下に、遭遇する主なクラスの目的を書きます。



俳優


他のすべてのゲームオブジェクトの基本クラス。 物理学、照明、ネットワークの役割などの種類を説明します。クラスは少なくとも斜めに勉強する必要があります。 PreBeginPlayおよびPostBeginPlay関数を宣言します。これらの関数は、初期化のためにすべてのサブクラスで引き続き使用されます。



Gameinfo


GameInfo-クラスはゲームのルール、その開始と終了を担当し、プレーヤーの制御クラスを設定し、ゲームとの接続と切断も制御します。 PreLogin 、 Loging 、 PostLogin 、 Logoutで探索する価値のある機能。 A HUDType変数-HUDaクラスを設定します。 PlayerControllerClassName-クラスPlayerControllerを設定します。 bDelayedStart-ゲームをすぐに開始するか、ある条件の後に開始するか。



ポーン


Pawn（ポーン）-ボット、NPC、プレイヤーなど、ゲームに直接参加する俳優を記述するクラス。 クラスには多くの機能が記述されていますが、ほとんどすべてが3Dアクションに適用できます。 新しいキャラクターを作成する場合、このクラスから継承します。 カメラクラスPawnまたはクラスPlayerControllerを表示する責任者を示す変数bSpecialCalcViewには特に注意を払います。 true

プレイヤーコントローラー
