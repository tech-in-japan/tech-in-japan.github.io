UbuntuへのOracleのインストールに関するいくつかの言葉

少し前まで、Javaアプリケーションの1つのモジュールの移植性を確認し、java.lang.Process APIを介してOracle SQL Loaderを呼び出すために、Linux用の小さな仮想マシンを迅速に作成する必要がありました。



ご存じのとおり、Oracle SQL LoaderはOracle Instant Clientパッケージに含まれていないため、このタスクでは、サーバーがインストールされているマシンから必要なライブラリを手動でコピーするか、この仮想マシンに完全なOracleサーバーをインストールする必要があります。



UbuntuでのOracle Enterprise Editionのインストールは、5分間のタスクではありません。 完全かつ正確（著者によれば-私自身も同様の指示を多数試しましたが、とにかく完全にエラーなしで配信することはできませんでした）指示はこちらです-www.excession.org.uk/blog/installing-oracle-on-ubuntu-karmic- 64-bit.html Windowsで同じOracle EEをインストールするよりもhemoの量と複雑さを比較してください。OUI（Oracle Universal Installer）を起動し、その指示に従います。 要するに、Joel Spolskyの精神に基づく、Linux環境での流れるような抽象化の優れた例です（ユーザーフレンドリーなディストリビューションのように見えますが）。



ただし、Express（XE）バージョンの場合は、まだ人間による解決策があります。







ここにある英語のオリジナルはwww.varyonic.com/2010/01/installing-oracle-xe-on-ubuntuです。そして、最初はEnterprise Editionをインストールしようとした可能性が高いため、すぐには見つかりませんでした。



ロシア語への私の無料翻訳：



Oracle XEは公式にx86システムでのみ使用できますが、タンバリン、運、誰かの母親の助けを借りてx64にインストールすることは可能です。 この自然な驚きには、512MBのメモリと1GBのスワップが必要です。



apt-getを介してUbuntuにインストールするには、/ etc / apt / sources.listファイルに次の行を追加します。



deb oss.oracle.com/debian unstable main non-free







そして、ルートの下で実行します（つまり、sudo ..、現在のユーザーのパスワードを尋ね、ルートの下でコマンドを実行します）：



wget oss.oracle.com/el4/RPM-GPG-KEY-oracle -O- | sudo apt-key add -

apt-get update

apt-get install oracle-xe







libaioおよびbcライブラリはリポジトリ自体で利用でき、依存関係に応じて自動的に調整されるため、手動でインストールする必要はなくなりました。



これが誰かに役立つことを願っています。
