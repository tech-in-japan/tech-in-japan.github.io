ホームメールサーバーに関するおとぎ話

むかしむかしなWebプログラマーがいて、どういうわけかサーバーを自宅に置くことにしました。 はい、それは単純ではありませんが、すべてがそこにいる人のようになります：Webサーバーとメールサーバーの両方。 このプログラマの幸福に、既存のQNAP 109 Pro IIアイアンホーム用に既製のメールサーバーを備えた新しいファームウェアがリリースされました。

Vanguardから永続的なIPアドレスがあり、ドメイン名* .spb.ruが長い間登録されており、 FirstVDSからのDNSサーバー上の対応するレコードが登録され、メールサーバーがインストールされ、構成されています。 プログラマーの喜びは際限がありませんでした。メールはすぐに家に落ち、すぐに家から送られました。



しかし、これでおとぎ話の楽しい部分は終わりました。



数日後、一部の無料メールシステムはホームメールサーバーからのメールを受け付けないことが判明しました。 Ramblerのテクニカルサポートは、ホームサーバーが192.168.XXX.XXXのように見え、そのような貧弱なサーバーを処理しないと説明しました。 XMailサーバーのWebインターフェースの標準設定には、ウェルカム値に適したものはありませんでした。 プログラマーはインターネットを詳しく調べて、XMailサーバーのHALOコマンドの値を変更する方法を見つけました。サーバー構成に新しいHeloDomain変数を追加する必要があることがわかりました。

これで、メールサーバーが表示されるようになり、* .spb.ruゾーンからドメイン名が提供されましたが、メールはRamblerおよびMail.ruのシステムによってまだ拒否されました。 Ramblerの丁寧な技術サポートは、不運なプログラマーにメールサーバーがメールを送信しようとしているサーバーのPTRレコードをチェックし、この値がサーバーのドメイン名と一致しない場合、メールが受け入れられないことを再度説明しました この場合、PTRレコードはpppXXX-XXX-XXX-XXX.pppoe.avangarddsl.ruを指しています。これは、サーバーのグリーティング値に対応していないだけでなく、動的IPアドレスの使用も想定しています。

プログラマは、ドメインのDNSレコードに正しいPTRレコードを追加して待機しました。 数日が経過しましたが、何も変わらず、PTRレコードは同じように見えました。 VDSテクニカルサポートと話し、ISPmanager Lightコントロールパネルで実験した後、ホスティングのPTRレコードは、このホスティングによって発行されたIPアドレスに対してのみ変更できることが判明しました。



何をすべきか、他のDNSホスティングを探しているのでしょうか？ 私たちの貧しいプログラマーはそれをやった。 彼は登録とテストを行いました：無料のXnameとEverydns 、さらにはwebnames.ruで 1年間支払いました 。 しかし、すべて無駄でした-PTRレコードへのアクセスはどこにもありませんでした。 最後に、プログラマが* .spb.ruドメイン名を登録したRelcomでのみ、IP所有者のみがPTRレコードを変更できることを明確に説明しました。 彼のインターネットプロバイダー。

アバンギャルドの技術サポートは、PTRを変更するサービスを提供しなかったと述べました。 SOAドメインpppXXX-XXX-XXX-XXX.pppoe.avangarddsl.ruからのdnsmaster @のメールが返されました。



親愛なるhabrasociety、誰かが自宅で本格的なメールサーバーをホストする経験を持ち、あなたの経験を共有できますか？ そして、おとぎ話がハッピーエンドで終わるように、おとぎ話の主人公に何をアドバイスできますか？ RFC 1912の条項2.1に違反していると不平を言うことはできますか。Intenetプロバイダーは、クライアントの要求に応じてPTRレコードを変更する必要がありますか。 または、別のプロバイダーを探すことはできますか？ テクニカルサポートから、 Web PlusとGenaralNetはPTRレコードを変更できると答えましたが、これらのプロバイダーのインターネットの品質はまだ不明です。



PS原則として、今では本格的なホームメールサーバーは実際には必要ありません。メールを受信し、VDSホスティングのメールサーバーを介してのみ送信できます。 しかし、しばらくして、退職後は、ホスティングの費用を払わないことが非常に魅力的な選択肢だと思います。
