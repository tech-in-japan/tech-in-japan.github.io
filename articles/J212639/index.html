Intel ONSオープンプラットフォームに基づくソフトウェア定義ネットワーク（SDN）

この記事では、SDNなどの概念を思い出して、その実装がIntel ONSプラットフォームにどのように見えるかについて説明します。



最初に、たくさんのテキストを読んで写真を見るのが面倒な人のためのライフハック：記事の最後に、10Gスイッチを取得するための条件を示します。 「食事なしで、つまり無料で」と有名な​​漫画全体でフクロウは言いました。



SDNについては多くの資料が既に書かれているため、主なアイデアはネットワーク機器の管理とデータ転送の管理を分離することであることを簡単に思い出してください。







このアプローチの利点：

  • 直接ネットワークプログラミング：制御レベルを分離すると、アーキテクチャを直接作成できます。
  • 柔軟性：管理者は、ネットワークのルールを「オンザフライ」で変更して、変化する要件に適応できます。
  • 管理の集中化：すべてのネットワーク管理は、SDNコントローラー内の1つの場所に（論理的に）集中しています。 アプリケーションの場合、ネットワークは単一の論理スイッチのように見えます。
  • ソフトウェア構成：SNDでは、社内開発を含む自動化ツールを使用して、セキュリティルールの構成、管理、設定、ネットワークリソースの最適化を短時間で行うことができます。








この概念は多くの論争を引き起こしますが、私たちは方向性が有望であると考えています。



昨年、Intelは、買収に基づいた一連のスイッチソリューションを発表しました。

Fulcrum Microsystems、Intel Ethernet FM6000。







行列のアイデア



マトリックスは、従来のスイッチングからSDNに簡単に移行できるハイブリッドスイッチとして位置付けられています。 主な機能は、スレッドをOpenFlowハンドラーまたは従来のパケットハンドラーに向ける機能です。 Intel FM6000のプロセッサは非常に柔軟性が高く、管理者が独自のプロトコルを作成したい場合でも、あらゆる標準で作業することができます-それは問題ではありません。 この柔軟性の基礎は、ハンドラーの構造です。







ハンドラーのTCAM / RAM / MUX構造



このようなマトリックスに基づいたスイッチは、従来のネットワークとOpenFlowで記録的な低遅延と高性能を同時に実現できます。







リファレンスデザイン









シリーズのモデル



このような素晴らしいマトリックスを使用するために、IntelはWind Riverのソリューション-Open Network Softwareを導入しました。







ONS構造



機能：

  • 低レベルの機能にアクセスできるモジュール式のオープンで拡張可能なアーキテクチャ。
  • オープンAPIおよびRPC呼び出しとのシステムインターフェース。
  • XML-RPCベースの管理ソフトウェアインターフェイス。
  • XMLおよびデータベーススキーマを使用したオブジェクト指向設計。
  • スイッチに新しいアプリケーションを埋め込むためのアプリケーション開発キット。
  • モジュールを介したネイティブプロトコルのサポートと拡張。
