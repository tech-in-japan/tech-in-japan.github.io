LKML.orgの「クラッシュ」：故障の原因-古いホームサーバー

The Linux Foundationのレポートによると、Git時代の始まり（2005年以降）から、1,500社の15,000人のプログラマーがLinuxカーネルの開発に参加しました。 現在、開発者は毎日約7500行のコードをシステムに追加しており、主要なカーネルアップデートは2〜3か月ごとにリリースされています。



ただし、すべての変更は何らかの形で管理する必要があります。 これを行うために、 Linux Kernel Mailing List （LKML）があります。これは、アナウンスを配布し、Linuxカーネルの変更について議論するために使用される電子メール配布リストです。



したがって、先週、LKMLホスティングリソースの1つであるLKML.orgが利用できないという情報がネットワーク上に現れたとき、コミュニティで何が起こったかについて活発な議論が始まりました。 今日は状況を理解しようとします。





/ Flickr / Sam weng / cc



理由は何ですか



最近の出来事に照らして、サイトの失敗は脆弱性メルトダウンとスペクターに関連しているという意見がコミュニティに現れました。 しかし、リソースの「落下」の本当の理由は、オランダのJasper Spaansの自宅での停電で、そこではサイト用のサーバーがありました。 その結果、サーバーはリブートし（「明かりを灯した」とき）、ユーザー名とパスワードを要求しました（少なくともJasper の最初の考えでした）。 ジャスパーは不在だったため、識別データを入力する人はいませんでした。彼のツイッターで報告されているように、彼はリモートで接続できませんでした。



悲しい部分は、そのマシンにはパスフレーズを渡すためのリモートsshアクセスを備えたinitrdがありますが（Javaベースのkvmの悪口のため）、外部から盗聴者に連絡することはできません。 ただし、vps + cryptopsは、このハードウェアが死んだときのためのものです。





-ジャスパースパーンズ（@spaans） 2018年1月10日


コミュニティの対応



2018年にこのような人気のあるリソースが1台のホームコンピューターに配置されていたことは驚くべきことでした。 このイベントはソーシャルネットワークで活発に議論されました。誰かが皮肉なコメントを出し、誰かがサーバー障害をMeltdownバグに結び付けようとし 、誰かが助けを提供しました。



たとえば、NLNOGの創設者であるJob Snijdersは、LKML.orgサービス用の仮想マシンをJasperに提供したり、自宅で使用する無停電電源装置を送信したりできることに注意しました。



ホスティング会社のLiquid Webも、 ボランティアで支援し、Jasperにサービスを提供しました。 誰かが Ilon Maskにタグを付けて 、Tesla Powerwallリチウムイオン電池の供給を手伝うことを提案しました。
