AWS Snowmobile：ペタバイト単位のデータをクラウドに転送... Amazon Trucks





昨年、Amazonは、容量の大きいストレージデバイスを使用してデータを迅速に物理的に転送するように設計された新しいAWS Snowmobileサービスを導入しました。 このサービスは、このディスクに書き込まれたデータの暗号化と保護のためのアプリケーションがプリインストールされた80テラバイトのデバイスをクライアントに送信する機能を提供します。 記録が完了すると、クライアントはディスクをAmazonに送り返し、この会社の従業員はすべての情報を「クラウド」に送ります。



かつて私たちはこのサービスの価格設定に興味があり、一般的にはかなり必要だと考えています（もちろん、特定の場合）。 現在、AmazonはAWS Snowmobileと呼ばれる別の同様のサービスを導入しています。 これは冗談ではありません-AWS Snowmobileは幅3メートル、長さ13メートルの強力なトラックです。 トラックのボディは耐破壊性があり、非常に耐久性があります。 内部-大容量のデータストレージデバイスのシステム。 実際、トラック全体は車輪の「外部ドライブ」です。 ケーブルと高速スイッチを介して、毎秒1テラビットの速度でデータを送信できます。





一度に、このような車両の1台で最大100ペタバイトのデータを転送できます。 トラックに情報を完全に記入するには、1週間から10日間かかります。 受信したすべての情報をAmazonクラウドに転送するには、同様の時間が必要です。 トラックの消費電力は350 kWであるため、エネルギーインフラストラクチャは、このような負荷に耐えるのに十分な近代的でなければなりません。 適切な容量がない場合、Amazonはお客様に独自の発電機を提供できます。



このような車は犯罪者にとって特に興味深いことは明らかです。なぜなら、そのようなハイテク車とその充填には数百万米ドルがかかるからです。 そのため、Amazonは必要に応じて警備員から護衛を提供し、会社は情報の過負荷時にデータセンターの投稿に警備員を提供することもできます。



各トラックには、GPS追跡、ビデオ監視、24時間体制の作業、高度なアラームシステムが装備されています。 データは、 AWS Key Management Service （KMS）に保存されている256ビット暗号化キーを使用して暗号化されます。







Amazonの経営陣は、このプロジェクトがクラウド（ Amazon S3またはAmazon Glacier ）に情報を迅速に転送する必要がある顧客のニーズを満たすために作成されたと発表しました。 したがって、アマゾンウェブサービスのCEOであるAndy Jassco氏によると、同社にはストレージ容量がエクサバイトに達する顧客がいます。 これらすべてをクラウドに転送すると、クライアントは10 Gbps接続で26年以上かかります。 トラックはそのような情報を約6か月で送り出します。 AWS Snowmobileの代表者は、トラックでデータを転送する方が、ネットワーク上で同量の情報を送信するよりも安価だと言います。



残念ながら、サービスの正確なコストは呼び出されません。 料金ページには 、各注文が個別に評価されることのみが記載されています。 これまでのところ、このサービスは米国の一部の州でのみ利用可能です。
