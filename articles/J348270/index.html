金融市場のもう一つの現実

「私たちの仕事は、金融市場のプレーヤーに対するHoloLensの可能性と可能性を研究することでした」とdxFeedの関係者は語りました。 しかし、抵抗することはできませんでした。







免責事項 ：この記事は、Unity上でHoloLensを開発する方法を教えることを意図したものではありません。 このためのこの記事があります。 以下に、金融で複合現実を使用するための優れたシナリオを示します。


著者に発言権を与えます。



dxFeedは、グローバルデータ交換プロバイダーであり、市場データ分析サービスの開発者です。 金融では、市場データは、取引プラットフォーム（たとえば、取引所）によって報告される金融商品の価格と取引に関するデータです。 市場データにより、トレーダーと投資家は最新の価格を見つけ、株式、債券商品、デリバティブ、通貨などの商品の過去の傾向を確認できます。



ほとんどの分析端末のエンドユーザーは、営業日中に複数のモニターの前に座って、チャートを変更する相場を注意深く調べ、履歴データでさまざまな取引戦略をテストし、市場データの膨大なフローを分析し、同時に受け入れる前にニュースチャネルを監視する人々金融資産を売買する決定。







サービスを多様化し、膨大な情報の視覚化の品質を向上させるために、dxFeedはHoloLensに基づいたデータ分析アプリケーションを開発しました。



金融業界の複合現実



今日の金融市場と取引は、自動化とアルゴリズムの程度に大きく依存しています。 人は膨大な情報の流れを分析して処理する必要があります。



金融および分析プラットフォームを複合現実環境に移行し、ユーザー（トレーダーおよび投資家）がより深いレベルで情報を操作できるようにしました。



以前はフラットなグラフィックが膨大になり、モニターのサイズは問題になりませんでした。360度のビューにより、無制限の数の画面を開くことができました。

私たちの観察によると、世界中の金融機関の個人トレーダー、投資家、コンサルタントはこれらの技術に最大の関心を示しています。彼らにとって、場所に関係なく協力する可能性は非常に有用です。 銀行、投資ファンド、ヘッジファンド、FXブローカー、取引所も、MR技術を習得して、競合他社を追い越し、仮想オフィスでのコンサルティングや3Dでのポートフォリオの追跡などの顧客サービスを提供しています。



スクリプトについて話しましょう



2016年の初めに、最初のHoloLens開発キットを受け取ってすぐに、dxFeed Holographic Solution金融アプリケーションの開発を開始しました。



まず、世界の取引所から受け取った独自のデータを視覚化し、技術的な実装のためにMicrosoftの推奨事項に焦点を当てました。Unity、HoloLensジェスチャー認識技術、Cortanaを使用した音声認識を使用します。









この機能は、ホログラフィックオブジェクトを操作する機能と同様に、多くのシナリオで非常に役立ちます。



  • コンサルタントは、投資家を仮想会議室に招待して、価格の上下がポートフォリオにどのように影響するかを示し、可能な解決策を示すことができます。
  • 証券取引所で特定の会社の株式を監視する金融アナリストは、大量のデータを受信して​​視覚化し、過去の株価を分析し、受信した情報をホログラフィック形式でクライアントに即座に送信できます。
  • 取引を学んでいる学生や新入社員は、AR / MRテクノロジーを使用して、指標を研究し、履歴データに基づいてパターンを分析し、市場活動を再現できます。 また、彼らがさらなる相互作用のために金融市場に飛び込む絶好の機会です。


実装例



アプリケーションの動作原理と使用されているテクノロジーのリストをおおよそ理解するために、特定の例を検討します。 最も単純なウィジェットの1つ、たとえば、ティックチャート（ティックチャート）を表示するとします。 このようなグラフは、完了した個々のトランザクションの時間、価格、量、および最高の需給価格の回廊（買/売）をユーザーに示します。



チャートのデータは、TimeイベントとSalesイベントのデータストリームの形式で提供され、dxFeedは取引所から直接受信します。 データ処理には、 dxFeed C＃APIを使用します 。 このライブラリは本質的にdxFeed C APIのラッパーであり、これはデスクトップWindows用に開発され、Windowsランタイムツールの代わりにWinSock2を使用してインターネットにアクセスすることに注意してください。



そのため、設定する特定の文字（MSFTなど）のTime and Salesデータストリームを接続しました。









ウィジェット自体は、コンポーネントのセットを持つGameObject



すぎません。 ウィジェット側で受信したイベントの処理は、いくつかの段階で行われます。



  • 表示データの現在の範囲、軸上のラベル、表示ポイントのセットなどの計算。
  • 表示されたポイントによるジオメトリ（三角形）のスケーリングと生成。 この操作には、大量の日常的な計算作業が含まれます。これにより、計算シェーダーを介して実行することが可能かつ適切になります。 つまり、ここでは、ポイントをスケーリングし、そこから特定の太さで実線を生成し、グラフの境界に沿ってカットし、美のためにz軸に太さを追加するシェーダーを記述する必要がありました。 この計算は、すべてのフレームではなく、データの更新ごとに1回だけ実行できることに注意してください。
  • プロセッサ上のジオメトリの残りの部分の生成：テキスト文字の位置、水平線と垂直線、グラフの境界など。 最終的に、これらはすべて1つのメッシュにマージされます。
  • UIストリームの最終操作：計算シェーダーの出力（ ComputeBuffer



    ）およびその他の変更を考慮して、 MaterialPropertyBlocks



    を変更します。 子のUnity UIウィジェットオブジェクトの場所。








当然、これはイベントコースのオプションの1つにすぎません-また、ユーザーがウィジェットをクリックした後（HoloLensのAir Tap）または単に見た（Gaze）後にウィジェットを更新する必要がある場合もあります。 ウィジェットのサイズを変更する必要がある場合があります-たとえば、同じユーザーのリクエストで。 一部のウィジェットには、所定の場所にあるグラフのセクションのミニチュア画像を含むポップアップパネルがあります。 または、ユーザーが現在の記号（金融商品）のリストを表示して、ユーザーが現在の記号を変更したり、別のセクションにデータを表示する子ウィジェットを作成したりする必要がある場合があります。



グラフ、タブレット、その他の視覚化を備えたウィジェットに加えて、アプリケーションはニュースビデオを表示できます。 Unityエンジン自体はビデオストリームを表示できないため、特定のサードパーティバックエンド、この場合はMicrosoft Media Foundationを選択して使用する必要がありました。 彼らは彼のためにネイティブプラグインを作りました。



また、別の実験として、1つのワークスペースへのアクセスを少数のポイントで共有する可能性が開発されていることにも注意してください。 これには、 Mixed Reality ToolkitからのトランスポートとVuforiaに基づくコンピュータービジョンソフトウェアが使用されます。 ウィジェットレベルでは、これは、その内部状態（位置、サイズ、現在表示されているシンボル）の一部を他のデバイスと交換できるように、1つまたは別のシリアル化可能なデータ構造に個別に保存する必要があることを意味します。



dxFeed Holographic Solutionが表示できる他のウィジェットの例：



  1. 市場の深さ：プレートの形の古典的なガラス。 リストから1つの文字が表示されます。ユーザーは、ウィジェット上のボタンを使用してこの文字を変更できます。
  2. ウォッチリスト：いくつかの金融商品のさまざまなパラメーター、たとえば、最新の相場、価格、最後の取引の量を含む表。
  3. ボラティリティサーフェス：オプション自体のボラティリティのストライクと有効期限への依存。
  4. クラスター化の概要：ヒストグラム形式の各シンボルの最終取引価格と最終価格の差。







  5. Book Trace 3D：株式市場の州の表面。 軸：時間、価格、累積供給/需要。









Vista dxFeedホログラフィックソリューション



取引およびデータ分析でのMRテクノロジーの使用には、金融機関にとって多くの利点があります。 たとえば、企業はオフィスの複数のモニターをコンパクトなワイヤレスMRメガネに置き換えることができます。これにより、コンピューター、スマートフォン、タブレットの画面サイズの制限がユーザーから排除されます。



企業は「どこでも職場」の概念を実装できます。360度の視野で、ユーザーはあらゆる表面で、さらには空中で作業できます。 ホログラフィック技術により、ユーザーは生産性を改善し、市場データの表示を改善できます。これは、財務上の意思決定をタイムリーに採用する際の重要な要素です。
