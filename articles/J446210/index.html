ADAM-3600-多機能産業用コントローラー









ADAM-3600コントローラは、産業施設のリモート制御、データ収集、監視、およびプロセス自動化のための汎用デバイスです。

同時に、オブジェクト（ USO ）、データ収集および送信デバイス（ USPD ）、およびプログラマブルロジックコントローラー（ PLC ）を備えた通信デバイスとしても機能します。

I / Oポート用の拡張モジュールと、無線インターフェイスWi-Fi、4G / 3G / GPRS、ZigBee、GPS用のモジュールがあります。



デバイスは、RealTime LinuxオペレーティングシステムとCortex A8プロセッサで実行されます。 ModBus、DNP3、MQTT、HTTP REST APIのオープンな業界プロトコルをサポートします。 SCADAシステムに統合できます。 また、 Microsoft Azure IoT Hub 、 Amazon AWS IoTなどのIoTクラウドシステムへの統合もサポートしています。



この記事では、デバイスの主な機能を分析し、 産業用制御システムの構築にコントローラーを使用するシナリオを理解します 。



技術仕様



プロセッサー： Cortex A8 600MHz

RAM： DDR3L 256MB

USB 2.0ホスト： 1

VGA： 1

シリアルポート： 1x RS232 / 485、2x RS485

イーサネットポート： 2

デジタル入力： 8（40まで拡張可能）

デジタル出力： 4（20まで拡張可能）

アナログ入力： 8（24まで拡張可能）

アナログ出力： 0（16に拡張）

mPCIeスロット：ハーフサイズx 1、フルサイズx 1

I / O拡張スロット： 4

ワイヤレスインターフェイス： Wi-Fi、3G、4G GPRS、ZigBee（オプション）

自己給電リアルタイムクロック：はい

SDカード：はい

温度範囲： -40〜70°C

電源： 8〜30V



ポート拡張モジュール



I / Oピンの数は、顧客のニーズに応じて拡張できます。 最大4つのインターフェース拡張モジュールをADAM-3600コントローラーにインストールできます。 モジュールには、デジタル入力/出力、アナログ入力/出力、熱電対（TC）および抵抗センサー（RTD）の入力の6種類があります。







フロントパネルに取り付けられたインターフェース拡張モジュール



無線インターフェース



mPCIeポート、 Zigbee、Wi-Fi、3G / 4G / GPRSの存在により、GPSワイヤレスインターフェイスをコントローラーに簡単に追加できます。 2つのmPCIeモジュールを同時にインストールできます。1つはフルサイズ、もう1つはハーフサイズです。 Huawei、Quectelなどが製造するブランドのアドバンテックモジュールとサードパーティモジュールの両方がサポートされています。





アンテナのタイプに応じて、適切なコネクタが取り付けられます：SMAまたはRP-SMA。 アンテナ出力は、標準のU.FLコネクタを介してボードに接続されます。



ネットワーク機能



ADAM-3600には、スイッチモードで使用できる、またはWANとLANに分割できる2つの独立したイーサネットポートがボード上にあります。 これは、デイジーチェーントポロジで複数のコントローラーを直列に接続するのに便利で、100メートルを超えるラインに追加のスイッチをインストールする必要はありません。









2つのLANポートにより、デバイスを直列に接続できます



フェールオーバーモードとマルチWAN



リモートの重要なインフラストラクチャ施設の場合、通信の損失は大きな損失を招く可能性があります。 したがって、通信の信頼性を高めるために、 ADAM-3600にはフェールオーバーモードのチャネル冗長機能があります。 メインチャネルで通信が失われた場合、コントローラーは、たとえば有線インターネットから3Gへのバックアップに自動的に切り替わります。 チャネルの優先順位と接続を確認するためのアドレスは、EdgeLink Studioで構成されます。









チャネルの優先順位とフェールオーバーモードの設定



VPNサポート



OpenVPN 2.4の現在のバージョンのサポートにより、企業の仮想ネットワークにデバイスを統合できます。 UDPおよびTCPトンネル、証明書認証、およびAES-256-CBCなどの高度な暗号化アルゴリズムが利用可能です。





EdgeLink StudioでOpenVPNトンネルを構成する



デバイスハードウェアID



フロントパネルのDIPスイッチを使用して、簡単な初期構成を設定できます。 NodeIDは、デバイス識別子と開始IPアドレスを設定します。 これにより、予備設定なしで多数のデバイスをインストールでき、さらに設定がネットワーク経由でリモートで実行されます。









デバイスIDはジャンパーを使用して設定できます



動作モード



ADAM-3600コントローラーの主な機能は、その汎用性です。 データ取得デバイス（テレメトリの記録および送信）、オブジェクトを備えた通信デバイス（コントロールセンターからの制御コマンドの送信）、およびプログラマブルコントローラー（自動制御）の両方を使用できます。

各ユースケースには独自の開発環境があります。



データ収集ユニットとリモート端末



ADAM-3600は、従来のデータ収集および送信デバイス（DTD）として動作できます。 追加機能の1つは、送信前のデータ処理です​​。 たとえば、標準偏差または生の電流センサーまたは抵抗センサーの読み取り値を計算し、完成したデータをさまざまなデータベース（MS SQL Serverなど）の適切な形式に変換できます。 インターネット接続が利用できない場合は、SDカードでデータを保存できます。 接続が復元された後、データは失われることなくサーバーに転送されます。







MQTTプロトコルを使用すると、データ収集をMicrosoft Azure IoT Hub、Amazon AWS IoTなどのクラウドサービスに直接整理できます。



デバイスはEdgeLink Studioグラフィカル環境で構成されます。これにより、プログラムコードを作成せずに、データ収集モードまたはリモートターミナルでデバイスを構成できます。 これにより、実装を大幅に簡素化できます。



EdgeLink Studioの手頃な価格のクラウドプロバイダー



プログラマブルロジックコントローラー



ADAM-3600ファームウェアは、 ハードリアルタイムオペレーティングシステムの特別なバージョンであるRT Linuxに基づいています。 同時に、プラットフォームは完全にオープンであり、開発者はシステムにルートアクセスできます。 これにより、システムプログラミング言語を使用して非標準ソリューションを実装できます。 アドバンテックは、 C / C ++言語用のSDKと 、Linux用のプロジェクトを構築するための開発環境およびツールチェーンを提供します 。



使用シナリオ：油井のリアルタイム監視



Webサイトのトレーニング資料セクションで、製品の使用シナリオをwww.advantech.ru/resources/で公開しています。 ADAM-3600コントローラーを使用する一般的なシナリオの1つは、掘削ステーションとポンプステーションの監視と制御です。 アドバンテックは、自動プロセス制御システム、特にWebインターフェイスを備えたSCADAシステムであるAdvantech WebAccessを構築するためのあらゆるソリューションを提供しています。



システムの基本的なタスク：



  • 油温と「スリーブ」、油圧、ポンプユニットの流量、その他のパラメータの監視。 データベースの読み取りと保存も同様です。
  • 有線（RS-232 / 485、LAN）および無線バス（ZigBee）を介したセンサーの接続
  • センサーとリレーをデジタルおよびアナログコントローラーインターフェイスに直接接続する
  • 周辺機器センサーの読み取り値の収集と、石油生産および機器の動作に関する統計。 初期処理後、データはSCADAサーバーに送信されます。







油井監視システムのアーキテクチャ



おわりに



モジュール設計のおかげで、 ADAM-3600コントローラーは顧客のニーズに基づいてインターフェイスポートとワイヤレスプロトコルを選択できるため、1つのデバイスで多数のタスクを実行できます。 クイックセットアップ機能が提供されているため、多数のデバイスを簡単にインストールおよびインベントリできます。 -40〜70°Cの動作温度範囲により、過酷な条件でコントローラーを使用できます。 業界標準と最新のIoTプロトコルのサポートにより、 ADAM-3600をあらゆる産業システムに統合できます。
