GoogleはMySQLを開発します



MySQL ABは、2009年まで同名のデータベースの開発計画を導入しています。 開発者によると、 2009年には、このDBMSの第7バージョンの登場が期待できます。 さらに、同社は、インターネット大手のGoogleがMySQLの開発に協力しており、バージョン7ではMountain Viewプログラマーによって作成された多くのコードを見ることができると報告しています。



現在、MySQLはインターネット上で最も一般的なデータベースです。 テクニカルサポートを必要としないユーザーの場合、MySQLは無料です。 過去1年半から2年、MySQL ABはOracle、IBM、Microsoftが伝統的に支配していた企業セクターでの開発を積極的に推進してきました。



MySQLの副社長であるDavid Axmark氏は、2007年前半にGoogleとMySQLがコントリビューターライセンス契約に署名しました。これにより、MySQLはDBMSの他の企業によって作成されたコードを含みます。



公式には、Googleは会社内でMySQLがどこでどのように使用されているかについての情報を提供しませんが、非公式データによると、Googleは世界最大のこの製品のユーザーの1つと見なされ、MySQLの最大数千のコピーが会社のインフラストラクチャで動作します。



現在、Googleはデータベースレプリケーション、情報監視、データ取得などの内部ニーズに合わせて修正バージョンを使用して、MySQLのソフトウェアチューニングにすでに積極的に取り組んでいます。Google-MySQLの合意により、 MySQL 6のリリース。2008年末にリリースが予定されています。



David Axmarkによると、Googleはこれらの開発の公開からいくつかのメリットも受けます。 まず、公式のバグ修正がリリースされます。次に、これらの開発は幅広いDBMSユーザーによって開発されます。



MySQL 6のもう1つのハイライトは、新しいFalconストレージエンジンです。これは将来のバージョンで標準になる予定です。 Falconは、Oracleが買収したInnoDBを置き換えるものです。



Falconは、高速災害復旧、ロールバックセグメント、およびフルテキストインデックス作成をサポートします。 パフォーマンスは、InnoDBメカニズムよりも低くないと宣言されています。



MySQLのコミュニティ開発担当副社長Kai Arnaudが指摘したように、コードを分析し、システム開発プログラム全体に準拠していることを確認し、配布に含めるために、来年GoogleとMySQLが必要になりますシームレスなモノリシックシステム。 「この作業には大きな可能性がありますが、まだやることがたくさんあります」とアルノは言います。



さらに、GoogleはオープンLinux OSのサポーターであることが知られているため、MySQL用のGoogleの開発はすべて実際にはLinux指向です。 MySQLは15のオペレーティングシステムに対応しているため、コードも移植する必要があります。



また、Axmarkは、今日、MySQL 6.0の新しい関数のリストがすでに承認されているため、すべてのGoogle開発がバージョン6.1に登場し、2009年の初めに登場すると述べています。



MySQL 7に関しては、Googleコードもここに存在しますが、このバージョンは、セキュリティだけでなく、企業の使用とこれから生じる機能にほぼ完全に焦点を合わせます。 特に、MySQL 7では、ユーザーグループとロールの概念が表示されます（おおよそOracle DBMSとの類推）。 DBMSカーネルレベルでは、データの暗号化が表示されます。



近い将来、MySQL ABは本番用にバージョン5.1をリリースする予定です。このバージョンでは、データの保存とインデックス付けのメカニズムが若干修正され、データベースのパフォーマンスが向上します。
