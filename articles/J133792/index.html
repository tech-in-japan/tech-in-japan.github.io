音節を満たす

簡単な説明

Syllableはもともと、GNU GPLのもとで家庭用コンピューターや小さなオープンソースオフィスで使用するための、フレンドリーなUnixライクなグラフィックシステムとして設計されました。 Linuxのように見えますが、独自のカーネル、ライブラリ、ユーザーインターフェイスを備えた完全に独立したシステムです。 オープンなモジュール式で安定したコアを使用します。 ソースは、2000年3月に公開された「絶滅した」AtheOSにつながります。 AtheOSは、AmigaOSとBeOSの概念を取り入れた有望なプロジェクトでした。 残念ながら、AtheOSの開発はすぐに停止し、その後完全に終了しました。 しかし、この有望なOSの運命に無関心ではない人々がいました。そして2002年、AtheOSのソースコードから新しいOSが構築され、AtheOSの伝統である音節が続けられました。 音節は、POSIX標準と99％（おそらく既に100％）の互換性があり、Webの閲覧（ABrowseブラウザー）、メールの操作（Whisperメールクライアントなど）にデスクトップコンピューターに固有のさまざまな機能を実行する多くの組み込みアプリケーションがあります）、マルチメディア再生（ColdFishおよびMedia Player）など。 創業者はクリスチャン・ファン・デル・フリートであり、ヴァンダースとして知られています。 現在、次のような多くの人々が開発に積極的に取り組んでいます。

それらはすべてSPT（Syllable Project Team）のメンバーです。 SyllableのファイルシステムはAtheOS（AFS）から継承され、64ビットでジャーナリングされており、非常に優れています。



システムの起動

いくつかのスクリーンショットを見て 、プロジェクトru.syllable.org/pages/index.htmlのロシアの公式サイトでさまざまな情報を調べた後、 私はこのOSをよりよく知り、Syllableが何であり、開発者が言うほど良いかどうかを知ることにしました。 ロシア語版をアップロードしました。 イメージはLiveCDバージョンであり、このようなシステムのテストに非常に適しているため、インストールせずにディスクからオペレーティングシステムを直接起動できます。 画像自体は約151MBであり、それは私を幸せにしました、そこにはたくさんのプログラムが詰め込まれているという希望がありました。 256MBのRAMを搭載したqemuを介してイメージを起動します。



$ qemu -m -localtime -cdrom Syllable-0.6.4-LiveCD-1.1-RU.iso -boot d







この目的のために、Oracle VirtualBox（Win32、Linux、MacOS用）、VirtualPC（Win32）、VMWare（専用）などの他の仮想マシンエミュレータを使用できます。 ただし、イメージをディスクに書き込み、そこからブートする方が良いでしょう。 システム要件は最小限です-233MHzプロセッサ、64MB RAMおよびVESAビデオカード。 USBキーボードでは機能しません。 さあ、行こう！

私の意見では、このシステムの大きなプラスはデスクトップとOSの統一性ですが、端末も利用可能です。 MacやWindowsに似ているため、アプリケーションは常に同じ「イン」に見え、最小限の依存関係と高速のアプリケーション実行を実現します。 開発者は、ターミナル、バスケット、ホームディレクトリへのリンク、「ドライブ」ディレクトリ（これは「マイコンピュータ」に似ています）、htmlドキュメントへの壊れたリンクなど、7つのアイコンをデスクトップに配置しました。 上部にはロゴボタンのあるドックがあります-オレンジ色の円の青い文字S、実行中のプログラムのアイコンはこのロゴの近くに表示されます（ちなみに、 Windows 7のようにアプリケーションはタスクバーに最小化されます -署名のないアイコンだけが表示されます。あなたは言うことはありません-2007年）。 これは、プログラムと設定を起動するメインメニュー、およびログアウトボタンです。 私の意見では、コンテキストメニューはそれほど悪くはありません。デスクトップでRMBをクリックし、それに応じてフォルダーを作成するだけです。 少なくともデスクトップを更新するために開発者がそこに入れなかった理由がわかりません（3番目のgnomeのように）この便利な機能をトリミングするとき、ひどく怒ります。 デスクトップ要素のコンテキストメニューは、「ヘルプで開く...」、「情報...」、「名前を変更する」、「ゴミ箱に移動する」、「削除する...」のように改善されています。 さて、そこで止まらないで、さらに見てみましょう。 軸はロシア語にかなりうまく翻訳されており、それは本当に驚き、私を喜ばせました。 ウクライナ語ではなく、クールです。 習慣ではありませんが、新しい未知の軸で起動する最初のアプリケーションはコンソールです。 ここで、ターミナルは私の通常のATermであり、Linuxでvteを使用することがあります。 そこで最初に入力するのはコマンド 'uname -a'で、これに対する答えが得られます。

ソフトウェア

Syllableが誇る主なものは、プログラムのインストールのしやすさです。 必要なのは、プログラムとともにアーカイブを/ Applicationディレクトリに解凍することだけです。 Mac OS Xにも同様のものがあります。プログラムをアンインストールします-フォルダーを削除します。 そして最も重要なのは、依存関係とライブラリに煩わされないことです！ まあ、何が良いでしょうか？ しかし、これはネイティブアプリケーションでのみです。 移植されたプログラムは、/ usrに配置してから、pkgmanager -a [directory]で登録することをお勧めします。 pkgmanager -r [directory]コマンドを使用して、移植されたアプリケーションを削除できます。 アプリケーションの移植は比較的簡単で、Syllable自体はC / C ++プログラミングインターフェイスとELF実行可能ファイル形式を使用します。 したがって、DosBox、PHP、QEmuなど、一部のオープンソースアプリケーションは既にコンパイルされています。 近い将来、GIMP、OpenOffice、ゲームなどのプログラム（NexuizやQuake 3 Arenaを音節で表示できること）を非常に嬉しく思います。 実績のあるgccは、ソースコードからアプリケーションをコンパイルするために使用されます。
