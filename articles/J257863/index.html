KeeneticでIPv6 6to4を構成した方法

私のプロバイダーであるRostelecomがIPv6を希望していないか、まだ提供していないことがありました。 しかし、私は本当に何かを試してみたいです。 すぐに予約します。通常の操作では、静的IPv4が必要です。



ZyXEL Keeneticルーターのユーザーは、トンネルブローカーを介して構成できることを知っています。 そして、すでに試して調整した人は、ブローカーのネットワークに依存し、ロシアのユーザーを本当に満足させない仕事とpingの速度に不満を抱いています。



だから、あらゆる種類の無料ブローカーをテストした後、私はこのベンチャーを辞めたが、問題の勉強をやめなかった。 それで6to4に出会いました。 もちろん、すべてが面白いのですが、家の各デバイスでのセットアップは少し面倒です。 さて、質問を勉強してから、どこに来ましたか？ IPv6 6to4は、ブローカーが提供するのと同じIPv6 6in4です。 残された唯一のことは、これらすべてをZUKHELに供給しようとすることです。



上記サイトの「システム要件」セクションを確認することを忘れないでください。 IPv4アドレス192.88.99.1にアクセスできるかどうかを確認します（たとえば、pingを使用）。 このアドレスが応答しない場合、ISPに深刻な「間違った」設定がされている可能性が高く、この場合、6to4は自動ゲートウェイ選択では機能しません。



電卓とメモ帳を使わないために、 6to4.ru Webサイトを使用して、静的IPv4のIPv6アドレスを見つけます。 右上隅の「6to4接頭辞は」という単語の後に書かれています 。



サイトは変更を加え、計算機を追加しました。 IPv4を入力し、[ 設定を取得]をクリックしてIPv6を取得します。



これが静的な理由です。これをすべて動的アドレスに設定した場合、IPを変更するたびにルーターの設定に戻ってアドレスを変更する必要があります。



アドレスを学習したら、今度はルーターに行きます。



[IPv6設定]ページの[ サーバーアドレス]フィールドに、アドレス192.88.99.1を入力します。 [ IPv6アドレス]フィールドで 、サイトで「カウントする」アドレスをプレフィックスなしで指定しますが、末尾に1を追加します。 なぜなら これがルーターのアドレスになります。 [ IPv6ルーティングアドレス]フィールドで、プレフィックスと共にアドレスを指定します。







すべてが構成されているように見えますが、そこにはありませんでした。 DNSサーバーに問題があります。

なぜなら ネームサーバーが不明な場合、サイトを開きません。

私たちの愛するGoogleのDNSサーバーを使用します：2001：4860：4860 :: 8888および2001：4860：4860 :: 8844。

各マシンを手動でポイントしますか？ また、面白くない。



Webフェイスでは、これらのアドレスを指定することはできません、構文に誓います。



ZyXELのWebサイトを調べたところ、ターミナルからコマンドを使用してDNSサーバーを設定できることがわかりました。 しかし、私は最も抵抗の少ない方法を採用し、同じWebを介してstartup-config構成ファイルをダウンロードし、編集してからダウンロードし直しました。



ファイルのほぼ最後の行、 ipv6 local-prefix defaultの後に、次の行を追加しました。



ipv6ネームサーバー2001：4860：4860 :: 8888

ipv6ネームサーバー2001：4860：4860 :: 8844



構成ファイルをダウンロードした後、ルーターを再起動し、スマートフォンを含むすべてのマシンでIPv6を取得します。



現在、トンネルブローカーに依存することなく、IPv6を介した高速インターネットと小さなpingを利用できます。
