ユーザーはリソースによって制限されるべきですか？

ITの量-管理者から「ユーザーには太りすぎて、トラフィック、メールボックスのサイズ、ファイル共有、サイトをブロック、好みに合わせて変更する」 そして、私の質問とまったく同じ量が発生します。あなたのビジネスは何ですか？



私たちはIT技術者であり、クールストレージシステム、サーバーファーム、メールサーバーを管理し、このすべてのカタバシアを遠くから見ることを忘れましょう。 商業構造を検討してください。



1.あなたの会社は何をしていますか？



トイレットペーパー、ズボン、さらには航空機のエンジンを作ることを忘れてください。



正解：彼女はお金を稼ぐ。 さらに、生産されたお金が消費されたお金よりも多くなるようにします。









2.お金を稼ぐのは誰ですか？



そして、彼らは、非常に「愚かなジュズヴェレイ」によって制作されており、彼らは偉大で強力なテクノマグのキーを押すように求められています。 テクノメージは非常に大きいため、英語（ITの使用言語）の指示を読むことができませんが、ロシア語では既に大きすぎます。









3.彼らは何の助けでお金を稼ぐのですか？



お金は、クライアント、インターネット、会計システム、会計などと通信するためのメールサーバーを含むITサービスの使用を含みます。 お金を稼ぐ過程で、ITサービスはITリソースを消費します-生の容量（プロセッサ/ RAM /ストレージ）、ライセンス、チャネル。



4. ITインフラストラクチャとリソースの所有者は誰ですか？



ここでは、管理者がサーバー、ストレージシステム、およびスイッチを「自分のもの」と見なし始めているという事実に何度も出くわします。 少なくとも少しですが、それに応じてそれらに関係し始めます。



正解：彼らは会社（会社の所有者）に属し、お金を生み出すという彼らの仕事を果たさなければなりません。



5.そして今-事業部門の従業員がどれだけのリソースを消費できるかを誰が決定すべきですか



これは絶対にIT管理者であってはなりません。 これらのリソースがどのようにお金をもたらすのか、彼にはわからないからです。



リソースの消費はIT側で無制限にする必要があり、ITインフラストラクチャへのリソースの割り当てを決定する際のITの重みはほぼゼロです。 おそらく、消費の急激な増加がインフラストラクチャ全体を配置できる瞬間を除きます。



必要なのは、消費を制限する代わりに、月末に部門の責任者と財務アカウントに請求書を送信するだけの内部請求システムです。 誰にどのくらいのリソースを消費したかについてのディレクターに。 ルーブルで。 この2人が、特定の従業員が生み出す利益の割合と、この利益を生み出すために必要なリソースの量を理解しているからです。



もちろん、あなたは一度にいくつかの点に反対することができます。 それらを見てみましょう。









1.ソーシャルを制限しない場合。 ネットワークを使用すると、作業する代わりに、全員がVKontakteにのみ参加します。



たぶん。 しかし、これはあなたの責任と知識の領域ですか？ 従業員には、仕事の質を評価する上司がいます。 もちろん、上司がイヴァン・イワノビッチ・イワノフにソーシャルサービスへのアクセスを遮断するよう依頼するのかもしれません。 ネットワーク-しかし、これは彼の決定の領域です。 おそらくIBポリシーになりますが、これも管理者の決定ではありません。



2.彼らは無知であり、ITの何も理解していません。 また、メールの添付ファイルにある面白い写真やvidyushkiがメール用のストレージを完全に消去するとはまったく考えていません。



まあ、それはあなたが何であるか、brain明で理解しています。 再び興味深い点がありますが、財布への影響ほど迅速かつ効果的に人を教育するものはありません。 マーケティング部門全体がメールに費やされたスペースの90％を写真に費やしたため、ボーナスがなくなると、コンピューターリテラシー/常識が劇的に向上します。 そして最も重要なのは、ボーナスなしで残された部門の長が、ネイティブで理解可能な方法によって彼らの増加を刺激することです。 彼らの賞品はすべてストレージの拡張に充てられます。









3. ITの予算は限られているため、必要なリソースを確保することはできません。



すみませんが、これは技術的な手段では解決できない管理上の問題の典型的な例です。 古典的なモデルでの調達の予算計画/ビジネスケースの問題は、管理者ではなく総局レベルでの管理上の問題です。 さらに、請求の助けを借りて、予算の正当化の問題がはるかに簡単に解決されます。



リソース制限には別の側面があります。 あなたは会社で何が起こっているかを制御するのをやめます。 ユーザーは、新しいメールを受信できないため、実際に重要なメールを削除し始めます。 重要なドキュメントはクラウドに表示され、通信は企業のメールからYandexおよびMail.ruに移動します。 言い換えれば、「シャドウIT」の作成者であるのは盲目的な抑圧的な方法です。
