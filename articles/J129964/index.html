スタートメニューの進化



翻訳者のメモ：記事の無料のリテール（ネガティブな方法で）はすでにHabréalizarで発表されましたが、このトピックは、可能であれば感情を加えることなく、原文全体を翻訳する価値があると思われます。





[スタート]メニューを変更する理由と方法に関する一連の記事を共有したいと思います。 最初に、[スタート]メニューの歴史と進化、およびお客様から学んだ問題と傾向について説明します。 私たちは、すべてが動いている場所について話す前に、すべてがどこから来たのかを知ることが常に重要だと考えています。 次の記事では、新しいスタート画面の作成方法について説明し、その後、これらの出版物の議論が私たちをどこに導くかを見ていきます。



一部の人々にとって、Windowsの変更は破壊的なものになることを認識しており、したがって、これらの変更についてオープンな対話が必要です。 多くの人にとって、Windowsは人生の重要な部分です。つまり、ほとんどすべての変更が、「これをオフにする方法」などの悲痛な反応や、使い勝手が良くなったかどうかについての議論を引き起こします。





タッチコントロールに関する今日の議論は、マウスが新たな好奇心、生産性のキラー、またはユーザーインタラクションの革新であるかどうかについての1980年代の議論と恐ろしく似ています。 多くのコメントが、タッチコントロールに対するマウスの優位性を強調していることを知って、これを言います。 マウスはデスクトップパブリッシングプログラムが登場する前に登場しました。マウスと描画プログラムに加えて、マウスはどこでも特別に使用されることはありませんでした。 そして今、私たちは空港、ガソリンスタンド、映画館、レジ、そしてもちろん携帯電話など、どこでもタッチスクリーンに囲まれています。 タッチコントロールがまだユビキタスになっていない唯一の場所は、使用する最先端のデバイスです。 マウスの外観だけでなく、コンピューターのタッチコントロールの分野における革新は、オペレーティングシステム、新しいプログラム、新しいハードウェアのサポートなしでは実現しません。 マウスのように、時間の経過に伴うタッチコントロールは、ユーザーインタラクションのほぼすべての側面を補完しますが、それに代わるものではありません。 これへの移行は、Windows 8 Developer Previewのリリースから始まります。 それでは、Windowsとの基本的なユーザーインタラクションだけでなく、プログラムやハードウェアの分野でも、すべてがどのように進化するかについての対話を始めましょう。



主なこと、特にStartについて話すと、いくつかのよくある質問に気づきました。

-タスクマネージャーを起動せずにMetroアプリケーションを閉じることは可能ですか？

「はい、できますが、なぜこれを行う必要がないのかについても話したいと思います。」



-マウスがスタート画面のプログラムをより効果的にスクロールできるようにするために何かをするつもりですか？

-はい、この領域で改善を行い、ベータ版でさらに多くを表示します。



あなたの何人かは、彼らがプログラムのリストを扱うことはタスクバーよりも効果が低いと感じていると言った（そしてこれについては今後の記事でもっと話す）。 他にも質問があります。 主なことは、あなたが私たちが知っていることを理解することです。 最初に用語とその意味について合意すれば、いくつかのことを議論しやすくなります。 たとえば、Metroスタイルは、システムの任意の要素に適用できる設計言語です。 また、スタート画面はスタートメニュー（およびタスクバー、通知、ガジェット）の開発であり、「メトロシェル」ではありません。 別の例：Metroをオペレーティングシステムモードとは見なしません。 むしろ、WinRTを使用するプログラムの一般的な名前です。 まだ話すべきことがたくさんあります。 最初はプラットフォームとツールに焦点を当てていましたが、おそらくユーザーインタラクションの変更について十分に説明していませんでした。



[スタート]メニューはWindowsの最も目立つ部分の1つであるため、この領域のすべての変更を非常に真剣に受け止めています。 Windowsを取り巻く世界は、[スタート]メニューを初めて導入してから大きく変化しており、現在のダイナミックな世界に適した機能を引き続き提供したいと考えています。 [スタート]メニューの進化は、アプリケーションの起動、タスクの切り替え、システム通知、ガジェットなど、他の関連概念の開発と密接に関連しています。 これらすべての概念の背後にある歴史とそれらがたどった道は、現代のプログラムとコンピューターで作業する、より明確で、より強力で一貫した方法を作成する機会を与えてくれました。 新しいスタート画面は、これらの願望を体現しています。



したがって、Windows 8に触れる前に、過去を思い出して、スタートメニューについて知っていることを見てみましょう。



スタートメニューの簡単な歴史





[スタート]メニューの設計は1992年に始まりました。 メニューはWindows 95で初めて登場しました。メニューは、直立したコンピューターと15インチCRTモニターが職場を支配していた世界で発明されました。 World Wide Webはまだ実験段階であり、人々はソフトウェアを購入するために店に行くことを余儀なくされました。 すべてが現在のものとは完全に異なっていました。 メニューの基本的な目的は、仕事を始めた人々に明確にすることでした。 このメニューは、ショートカットを備えたフローティングウィンドウであったWindows 3.xのコンセプトである古代のプログラムマネージャーに取って代わり、このウィンドウはデスクトップ上で邪魔になり、他のプログラムのウィンドウの下で失われました。 [スタート]メニューは、タスクバーにしっかりと接続されているため、その位置が一定であることが特徴であり、プログラムとシステム機能への単一のポータルでした。 実際、メニューは、システムの腸内のどこかにある実行可能ファイルを探すことなく、プログラムを起動する最も速い方法でした。





Windows 95の[スタート]メニュー



Windows 95の[スタート]メニューを見ると、Windowsの将来のバージョンで徐々に改善された領域にすぐに気付くことができます。 たとえば、ポップアップウィンドウのアルファベット順のリストにあるプログラムの場所によって、ナビゲーションが遅くなるというレビューを聞きました。 これに対応して、最近使用したプログラムの一部がWindows XPに登場しました。 実際には、このセクションを構成する方法がなかったため、この変更により新しい問題が発生しました。 一部の人々は、このセクションでプログラムが選択された原則を理解するのに苦労しましたか？ 複雑なヒューリスティックアルゴリズムにより、最も頻繁に使用するプログラムが決定されました。 自分用に[スタート]メニューをカスタマイズする問題を解決するために、Windows XP（およびWindows VistaおよびWindows 7以降）にセクションが追加されました。このセクションでは、最も重要なプログラムを常に手元に置いておくことができます。 ただし、この機能はまだ不十分でした。 プログラムを固定して順序を変更することはできますが、プログラムをグループ化する方法はまだありません。これは、多くのプログラムがある場合に重要です。



インストールされているすべてのプログラムのリストを表示することもあまり便利ではありませんでした。 多くの場合、メニューは誤った動きのために最小化され、最初からやり直す必要がありました。 特に困難だったのは、トラックパッドを備えたラップトップの所有者と、あまり賢くない人です。 低解像度のモニターにすべてを表示することは依然として困難でした。 Windows Vistaでこれらの問題を解決するために、1つのウィンドウに完全に配置されたツリーメニューが表示されました。 このようなメニューでは、アクロバティックなマウスのエクササイズが少なくて済みました。 ただし、メニューにはスクロールバーが必要なため、インストールされているすべてのプログラムのリストはまだ混み合っているように見えました。 [スタート]メニューは既にいっぱいに見え始めています。





Windows 7にインストールされているすべてのプログラムのリスト



[スタート]メニューのもう1つの重要な進化上の変更点は、インスタント検索の登場です。 VistaおよびWindows 7では、メニューを開き、要求の入力を開始することが非常に簡単になりました。 プログラムやドキュメントを開くのに必要な時間が短縮されるため、多くの人がこの強力な作業方法を気に入っていることを知っています。 高度なコマンドを追加すると、「実行」ダイアログを起動する必要も減りました。 それにもかかわらず、私たちはまだ改善すべきことがあることを知っていました。 検索結果が多すぎて、さまざまな種類のデータ（文字、ファイル、コントロールパネルの設定など）を表示することが、固定サイズの列では必ずしも便利ではないことがありました。 Windows 8のスタート画面で検索がサポートされるかどうかを尋ねられた方もいらっしゃいます。 彼はすでにサポートしています。 入力を開始すると、タイプ（プログラム、ファイル、または設定）でフィルター処理できる検索結果がすぐに表示されます。 また、Windows 7と同様に、すべての検索機能をエクスプローラーで利用できます。



既知の問題とフィードバックに基づいて、長年にわたって[スタート]メニューとの対話を強化し続けました。 ただし、Windows 7のStartでさえ次の問題がまだあると考えています。

  • インストールされているプログラムのリストを表示しようとすると、メニューが混み合っているように見えます。 特に、使用可能な画面スペースの量を確認します。
  • 検索には、検索結果に追加情報を表示するためのスペースが割り当てられます。 これは特に大きな画面で顕著です。
  • メニューのカスタマイズは困難です。
  • アイコンとショートカットは静的であり、増加したピクセル数を使用しません。




もちろん、上記のリストは私たちが改善しているものの完全なセットではありません。 また、Startを残りのシステムと統合し、新しい作業シナリオを追加することも目指しています。 デザインの重要な部分は、時には一歩下がって、すべてを横から見る必要があることです。 これは、根本的に問題を再考し、製品の外観上の変更以上のものをもたらすのに役立ちます。 これは特に、[スタート]メニューのようなものに当てはまります。[スタート]メニューは、現在とはまったく異なる方法でコンピューターを使用したまったく異なる時期に生まれました。



問題の深さを認識することが重要です。 [スタート]メニューは、プログラムとドキュメントを開くためにほぼ排他的に使用されます（検索機能を除く）。 Windows 7の操作には、プログラムの切り替えとそれらのタスクバーへの固定、通知の受信、デスクトップ上のガジェットの操作も含まれます。 対話の過程で、Windows 8がWindows 8をどのように調和させるかについて話します。



スタートメニューはどのように使用されますか？





[スタート]メニューの履歴について簡単に説明したので、[スタート]メニューの実際の使用方法を見てみましょう。 メニューの使用が時間の経過とともにどのように変化したかを見ることに興味があると思います。 次の図は、Windowsの2つのバージョンでのメニューの使用法の変更を示しています。





Windows VistaおよびWindows 7の[スタート]メニューの使用に関する変更



Windows VistaとWindows 7では、[スタート]メニューの使用方法が驚くほど違います。いくつかの特別なフォルダー（メニューの右上にあるいわゆるオブジェクト）の使用頻度は50％以上減少しました。 同様に、Windows 7では、[スタート]メニューの固定されたオブジェクトの使用がVistaの2倍になりました。 また、インストールされているすべてのプログラムのリストと最近使用したプログラムのリストを使用する可能性ははるかに低くなります。 そして最後に、人々は一般に[スタート]メニューを開く頻度を減らしました（11％減）。 一見、11％は少量のように見えますが、何億人ものユーザーがWindowsインターフェースの普遍的に認識可能な要素を使用する可能性が低いことを考慮すると、すべてがより深刻に見えます。 私たちは少数派によって使用されているいくつかの隠された設定について話していません。 私たちは、人々がますます使用することのないWindowsの基本的な要素について話している。



では、なぜスタートメニューの使用方法が変わったのですか？ ヒントを次に示します。これは、Windows 7に表示された画面下部のパネルによるものです。



スタートパネル





タスクバーの進化は、スタートメニューに直接影響しました。 かつてメニューに隠されていたものが突然あなたの近くになりました。 最も明らかな改善点は、1997年の（Internet Explorer 4.0から更新プログラムをインストールした後の）クイック起動バーの外観と、Windows 7のタスクバーへのショートカットを固定する機能の追加です。



興味深い事実：元のバージョンのWindows XPのクイック起動バーは、最近使用したプログラムのリストと[スタート]メニューへのショートカットを固定する機能で十分だと考える人がいたため、デフォルトでオフになったことをご存知ですか？ 反対に十分な証拠を得て、心を変えました。 私たちは、最初の決定を下したデータが不十分であり、概して、顧客が何をしているのかを知らなかったという事実によって、いくぶん正当化されます。 このストーリーから、次の教訓を学びました。あなたにとって重要なプログラムを強調し、それらをすべて1か所で確認して、ワンクリックでそれらに到達できることが重要です。 また、誰かがあなたにとって重要なものを推測したり、重要なものと重要でないものを混ぜたりする必要はありません。



さて、すべてをまとめるために、人々がプログラムを修正する場所を見てみましょう。 次のグラフは、85％の人がタスクバーに3つまたは4つのプログラムを固定しているのに対し、同じ数のプログラムが[スタート]メニューに固定されている人は23％だけであることを示しています。 タスクバーと[スタート]メニューにはデフォルトで異なる固定プログラムが含まれていますが、ユーザーは必要に応じて両方を調整します。 ほとんどの人は、スタートメニューでプログラムを追うのではなく、タスクバーでプログラムを見たいと思っていることは明らかです。





[スタート]メニュー（上のグラフ）およびタスクバー（下のグラフ）に割り当てられたタスクの数。



また、Windows 7では、[スタート]メニューよりも頻繁にタスクバーを使用する上級ユーザーもいます。 Win +（nはタスクバーのアプリケーションアイコンのシリアル番号）などのホットキーを使用すると、アプリケーションをさらに高速に起動および切り替えることができます（これらのホットキーはWindows 8でも機能します）。 ITプロフェッショナルと面談すると、多くの場合、さまざまなアイコンで満たされたタスクバーが表示されます。 人々がコントロールパネルやタスクバーに似たものを固定するのを見ました。 これは再び実行を開始しないことです。 特にIE 9でサイトを固定できるようになった後、人々はますますタスクバーに何かを固定しています。 幸いなことに、タスクバーには十分なスペースがあります。 1024x768の解像度でも、22個の小さなアイコンが収まります。 また、ジャンプリストを使用すると、理論上、上記の画面解像度で220のファイル、フォルダー、およびサイトに簡単にアクセスできます。 つまり、デスクトップアプリケーションを使用したいだけの場合、タスクバーには、[スタート]メニューを表示せずに常に必要なすべてを統合して使用するのに十分なスペースが用意されています。



ところで、遷移リストについて話し始めたので、人々がExplorerをタスクバーにピン留めし、頻繁に使用されるフォルダーを含む遷移リストで埋めるのを見ました。 これにより、ドキュメントなどのフォルダーにすばやくアクセスできます。 当然のことながら、[スタート]メニューの[ドキュメント]ショートカットの使用頻度も減少しました。



要約すると、タスクバーは進化し、[スタート]メニューの多くの機能に取って代わりました。 タスクバーの[スタート]メニューに欠陥があり、このメニューは以前ほど重要ではないと言うこともできます。 インストールされているすべてのプログラムのリストを検索してアクセスすることは、引き続き[スタート]メニューのユニークな利点ですが、タスクバーを使用すると、毎日使用するアプリケーションにワンクリックでアクセスできます。 あなたとあなたの多くは、長年にわたり、デスクトップアプリケーションの起動と切り替えを行うための強力なツールにするために、レビューを押し進めてきました。 実際、Windows 7のタスクバーを「スタートバー」と呼ぶこともありました。これは、多くの人がメニューではなくバーから開始することが明らかだからです。
