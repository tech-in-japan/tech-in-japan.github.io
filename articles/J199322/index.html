カスペルスキーアンチウイルスアップデートがtcpip.sysファイルをブロックする

朝、友人がネットワークの問題に困惑しました。 今回判明したように、カスペルスキーはそれを手に入れました。 まだ知らない人のために公開しています。



2013年10月25日金曜日、カスペルスキーは、tcpip.sysシステムファイルの検出が誤って追加されたウイルス対策データベースをリリースしました。 このアンチウイルスデータベースのセットは、Windows 7 Service Pack 1（32ビット）を実行しているKaspersky Anti-Virus 6.0 for Windows Workstationでtcpip.sysファイルを悪意のあるものとして検出し、ファイルを検疫に配置し、このファイルに属するレジストリブランチを削除しました。



2013年10月25日の20:00（GMT +4：00）に、Kaspersky Labはウイルス対策データベースの固定セットをリリースしました。



カスペルスキーは、ウイルス対策データベースの更新を推奨し、ご不便をおかけして申し訳ございません。



また、この問題を解決するには、次のいずれかの方法を使用します。



方法1：コンピューターは再起動されず、ネットワークは機能しています。

ウイルス対策を更新します。 更新が適用されたことを確認します。

管理者権限でkaspersky_tcpip_fix.exeを実行します。

tcpip.sysファイルがC：\ Windows \ system32 \ driversに復元されたことを確認します。

ファイルが復元されない場合は、製品検疫から手動で復元します。

ネットワークが特別な設定（DHCPを介して配布されない）を使用する場合：

regextr.exe <SYSTEMレジストリブランチのバックアップコピーへのパス-通常、％windir％\ system32 \ config \ Regback \ SYSTEM> <エクスポートファイル名、たとえば、管理者権限でextract.reg>を実行します。

extract.regファイルを実行（適用）します

再起動します。



方法2：コンピューターを再起動しましたが、ネットワークは利用できません。

ウイルス対策保護を無効にします（再検出を除く）。

管理者権限でkaspersky_tcpip_fix.exeを実行します。

tcpip.sysファイルがC：\ Windows \ system32 \ driversに復元されたことを確認します。

ファイルが復元されない場合は、製品検疫から手動で復元します。

ネットワークが特別な設定（DHCPを介して配布されない）を使用する場合：

regextr.exe <SYSTEMレジストリブランチのバックアップへのパス-通常、％windir％\ system32 \ config \ Regback \ SYSTEM> <エクスポートファイル名、たとえば、extract.reg>を管理者権限で実行します。

extract.regファイルを実行（適用）します

再起動します。

ウイルス対策を更新します。 更新が適用されたことを確認します。

ウイルス対策保護を有効にします。
