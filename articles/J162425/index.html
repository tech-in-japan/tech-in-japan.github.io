Linuxカーネルから386SX / DXプロセッサーのサポートが削除されます

私は386SX-20を決して忘れません。そこではDoomが小さなウィンドウの中でも遅くなり、私の友人はクールな386DX-33を持っていました。 私は386DX-40を節約することができた1年間、すでにそのゲームはフルスクリーンになりました。



この頃、1991年10月に、陸軍から戻ってきたLinus Torvaldsは、新しいLinux OSの最初のバージョンのカーネルを完成させていました。 i486はまだ高価すぎるため、彼自身がこれに取り組んだのかもしれません。たとえば、コプロセッサーのない簡略化された486SXモデルは1991年5月に初期価格527ドルでリリースされました。



Intelは、2007年9月まで386DXプロセッサをリリースしましたが、その後、需要の低さから生産から撤退することを余儀なくされました。



2012年の終わりに、Linuxカーネルは古いコードを最適化してわずかにクリーンアップすることも決定しました。 Linus Torvaldsはプルリクエストを承認しました。それに応じて、i386仕様がカーネルから削除されます。 AMD / Cyrix / Intel 386DX / DXL / SL / SLC / SX、Cyrix / TI 486DLC / DLC2、UMC 486SX-SおよびNexGen Nx586プロセッサーで使用されます。 古いCPUのサポートを削除することにより、ソースコードを369行（56+ 425-）削減できます。



Linusはさらに最適化を進め、486SXをサポートするために必要な古いコプロセッサのエミュレーションでarch / x86 / math-emuブランチを削除することを提案しました。 しかし、同僚はこの提案を支持しませんでした。誰かが486SXクローンがまだ生産中であることに気づきました。



一般的に、Linusは、新しいコードを追加せずに、コードを削除することを申し出たとき、非常に満足していると繰り返し述べています。



いずれにしても、2か月半後にリリースされるLinux 3.8カーネルでは、「ターボ」ボタンを押しても、コンピューターで386SX / DXを実行できなくなります。
