Windows Azure回復サービス

多くの場合、Windows Azureクラウドプラットフォームを使用する場合、多くのお客様は、クラウドに保存されているデータまたはローカルサーバーでホストされているデータをバックアップするサービスが必要です。 SQL Azureデータベースを検討している場合、すべてがシンプルです-SQL Azure Data Sync。 しかし、たとえば、そのような機能が仮想マシンに必要な場合はどうでしょうか？ リレーショナルでないデータ、またはリレーショナルデータベースに保存されていないデータの場合



ほんの数か月前（というか、今年の4月）に、Windows Azureクラウドプラットフォームを開発しているマイクロソフトチームは、よく発生する同様の問題を解決するための別のサービスを発表しました。 彼の名前はRecovery Servicesです。 これにより、Windows Serverを実行しているマシンからのデータバックアップを保存できます。



このサービスの意味は非常に簡単です。 特別なプログラム（Windows Azureバックアップエージェント）がマシンにインストールされ、そのデータはクラウドに保存され、スケジュールに従って、選択されたデータを手動で同期するWindows Serverを実行する必要があります。 セクション全体のバックアップ、および個々のフォルダーとファイルのバックアップをサポートします。



比較



ご存知のように、Windows Azureが提供するストレージサービスに大量のデータ、たとえば同じバックアップを保存できます。 ただし、それにアクセスするには、Windows Azure SDKを使用する（ソリューションを作成している場合）か、サードパーティのソリューション（Azure Storage Backupなど）を使用する必要があります。 すぐに同様の機能を使用したいと思います。



Windows AzureリカバリサービスとWindows Azureストレージの機能を比較してみましょう。



*-ジオレプリケーション。 リージョン内でデータを複製する機能。 ヨーロッパなどの各地域には、西ヨーロッパと北ヨーロッパの2つのデータセンター（サブ地域）があります。 データはサブリージョン内で複製されます。つまり、西ヨーロッパのストレージ地域でホストされるデータは北ヨーロッパ地域で複製され、その逆も同様です。



表からわかるように、Windows Azureストレージサービスは、より多くのデータセンターでより安価に利用でき、ジオレプリケーション機能をサポートしています。 同時に、Recovery Servicesにはこれらすべての利点がありません。 ただし、Recovery Servicesでの作業は、トランザクションを通じて請求されることはありません。 さらに、Recovery Servicesのセットアップ全体がWindows Serverマシンへの追加ソフトウェアのインストールに限定されることを忘れないでください。ストレージにデータバックアップ機能を実装するには、独自のソフトウェアを開発するか、サードパーティソフトウェアを使用する必要があります。



次に、シンプルなタブレットを作成して、ストレージにバックアップを保存し、Recovery Servicesを使用するコストを比較しましょう。



*-給料のレベルは、キエフ市の調査による。 中央値 経験：1-2年。

**-Windows Azureリカバリサービスは現在「プレビュー」状態にあるため、価格が50％引き下げられました。



SQLレポートサービスに関する記事で、関連するテクノロジを提供するスペシャリストの費用を考慮していないことを非難されました。 今回は、この発言を考慮に入れようとしました。



ストレージサービスの「正味」コストの表からわかるように、Recovery Servicesははるかに安価であり、格納されるデータ量が多いほど、この差は大きくなります。 ただし、各ソリューションをサポートするためにそれぞれの専門家の給与を追加すると、Recovery Servicesの価格が安くなります。 同時に、格納されているデータの量が増えると、この差も解消されます。



したがって、どのサービスを使用するかについての決定は、特定の各タスクの要件に従って行う必要があります。
