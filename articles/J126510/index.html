TeamWox SaaSエージェント：SaaSファームを無料で高速に構築する方法

TeamWox SaaS Agentの新しいバージョンがリリースされました 。これにより、プロバイダーと独立サプライヤーの両方のために、何百ものTeamWoxグループウェアシステムのSaaSファームを無料で構築できます。







面白い？ あなたを猫に招待します。





ソフトウェア製品をインストールおよび保守する際にプロバイダーが直面するすべての問題を感じるために、SaaSサービスの提供を独自に開始することにしました。



最初に、コンピューターの準備と設定、ファイアウォールの設定、ドメインの登録、SSL証明書の割り当てと更新、クライアント用のTeamWoxサーバーのインストール、インストール後の可用性のテスト、ステータスの監視、TeamWoxのアンインストールなど、ホスティング事業者の仕事の魅力をすべて感じなければなりませんでした。



これらのすべての操作を手動で行い、そのたびに何度も繰り返した場合、ミスを犯す可能性があり、クライアントはマーケティング担当者が説明するほど単純ではないことを「推測」します。







当社が開発した他のソフトウェアと同様に、 TeamWoxホスティング管理ソリューションは、シンプルで効果的であり、管理者のために働き、必要な最小限の操作を実行し、危険とエラーについて警告する必要があります。 作業の結果はTeamWox SaaS Agentです。



MS Windows Server 2008を搭載したコンピューターを多数のTeamWoxグループウェアシステムのホームにするためには、1分で十分です。TeamWoxSaaS Agentインストーラー （1.5 MB）をダウンロードするのに 10秒、それをインストールしてWebページを開く50秒ログインとパスワードを入力します。



TeamWoxを使用するには、ユーザーがお気に入りの企業アプリケーションのアドレスを入力するWebブラウザーが必要です。 このようなアドレスをユーザーに提供するには、ドメインを選択する必要があります。



各アプリケーションに一意のドメインを提供することは、ホストとクライアントの両方にとって高価なオプションです。 普通の人が推測することさえないサービスに対しては誰も支払いを好みません。



シンプルでエレガントなオプションは、第3レベルドメインを使用してアプリケーションへのアクセスを提供することです。 そのようなドメインのサービスを引き継ぐことができます-クライアントは「そこに何かを割り当てる必要がある」ことすら知りません。



独自のSaaSファームを組織し、まさにそれを行いました。 TeamWox SaaS Agentは、HTTP APIを介して一般的なDNSサーバーとの統合を実装しました。 新しいアプリケーションのドメインが自動的に割り当てられます。設定でHTTP API DNSサーバーのアドレスと認証データを指定するだけで十分です。



セキュリティはSaaSの鍵です。 プロバイダーは、ユーザーデータストレージのセキュリティを担当します。 それらへのアクセスのセキュリティを確保することが彼の仕事です。 エンタープライズSaaSアプリケーションにオープンHTTPプロトコルを使用することは、無責任の極みです。 このため、TeamWoxへのアクセスは、安全なHTTPSプロトコルを介してのみ可能です。



HTTPSが機能するには、SSL証明書をサーバーにインストールする必要があります。これはWebブラウザーによって信頼されています。 Thawte、Comodoなどの企業から入手できます。



何百ものアプリケーションのSSL証明書を注文する前に、HTTPSプロトコルの1つの機能を覚えておく必要があります。1つのサーバーSSL証明書のみが1つのIPアドレス/ポートを提供できます。 この場合、各ファームコンピューターに数十個のIPアドレスを割り当てるか、ポートの形式の特殊な添加物でユーザーを苦しめる必要があります。



しかし、最初のアイデアも2番目のアイデアも熱狂的ではありません。 使用したソリューションは、ワイルドカードSSL証明書です。 1つのワイルドカードSSL証明書は、SaaSファームにインストールされているすべてのアプリケーションに使用されます。 ユーザーは自分で安全であり、TeamWoxを使用して苦痛を感じることはありません。



証明書ストアがどのように機能するかを知らなければ、初めてWindowsに証明書をインストールすることはできません。 このプロセスを単純化して理解しやすくするために、TeamWox SaaS Agentでサーバー証明書を処理するための便利でシンプルなインターフェイスを作成しました。



Webインターフェイスを介して、証明書をインストールおよび更新し、新しい証明書の要求を生成し、インストールされた証明書をファイルとしてダウンロードできます。 SaaSファームのテスト中に、コンソールユーティリティを検索せずに、一時的な自己署名ワイルドカードSSL証明書を生成できます。







基本的なルーチン操作が自動化された後、SaaSファームを編成するときが来ました。サーバーのセットを統合するためです。 独自のSaaSファームのセットアップに使用するソリューションは、非常にシンプルで効果的です。



各ファームサーバーは同等であり、TeamWoxホスティングという1つの機能のみを実行します。 SaaSファームに新しいコンピューターを含めるには、TeamWox SaaS Agentをインストールするだけです。 そして、これは上で書いたように、非常にシンプルで高速です。



ファーム内の複数のサーバーを整理するときに発生するタスク：

  • サーバー間の負荷分散。
  • 新しいアプリケーションをインストールするサーバーを選択します。
  • 複数のサーバー上のアプリケーションの制御。


それらを解決するために、TeamWox SaaS Agentの機能を拡張し、SaaSサーバーマネージャーに変えました。 ファーム内のすべてのサーバーのうち、マネージャーの機能が割り当てられたサーバーが1つ選択されました。



ファーム内のすべてのサーバーから情報を収集し、受信した統計に基づいて、TeamWoxの新しいコピーをインストールするサーバーを自動的に選択します。 これにより、正義の原則に従い、SaaSファーム内の各サーバーに均等に作業をロードできます。







SaaSファームの管理は1つの場所から行うと便利です。 TeamWox SaaS Agentのシンプルで直感的なインターフェースにより、SaaSファームの手綱を一般ユーザー、つまりマネージャーに転送することが可能になりました。



新しいアプリケーションは、技術専門家の注意をそらすことなく、わずか2〜3分でインストールできます。 また、インストールされている何百ものTeamWoxアプリケーションの作業を管理および制御するのも簡単です。



TeamWoxには、自己監視および監視ツールが組み込まれています-これにより、安定した動作が保証されます。 このデータを使用して、TeamWox SaaS Agentを使用すると、SaaSファーム全体の安定性を実現できます。 制御サーバーページですべてのアプリケーションの動作を監視することで、ユーザーが意識すべきではない危険からユーザーを保護できます。







TeamWox SaaS Agentを使用してSaaSファームを最初から整理することは、迅速かつ簡単です。 経験のあるホスティング事業者には既存のサーバー管理インフラストラクチャがあり、誰もそれを破ったり拒否したりすることはありません。 そして、自分自身を引用すると、SaaSファームの管理は1か所から行うと便利です。 ソフトウェアとソフトウェアプロバイダーの統合が必要でした。 そして経験が示唆するように、2つの独立した製品の統合は通常、苦痛を伴うプロセスです。



統合プロセスのすべての魅力を感じました。 実験では、Parallelsプロバイダーに人気のあるソフトウェアが選択されました。



作業の結果、明確で適切に構成されたTeamWox SaaS Agent HTTP APIと、APS標準のバージョン1.2に従って認定されたパッケージが作成されました。



Parallels Business AutomationまたはParallels Plesk Panelに基づくホスティングサービスを既に提供している場合、通常のユーザーインターフェイスを使用してTeamWoxアプリケーションのライフサイクルを管理します。



使い慣れた他のソフトウェアとの統合は、問題を引き起こすことはなく、HTTPおよびJSONを操作するツールを備えた一般的なプログラミング言語で実行できます。 たとえば、APSパッケージでは、スクリプトはPHPで記述されています。
