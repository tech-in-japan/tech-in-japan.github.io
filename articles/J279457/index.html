Googleスマートフォン検索ロボットの新しいユーザーエージェント

こんにちは、Habr！ Google検索は進化しており、関連ツールがそれに伴って開発されています。 近い将来、Googlebotユーザーエージェントが更新され、「モバイル」検索の方向性が決まります。 現在、useragentは次のようになっています。

現在のユーザーエージェント
Mozilla / 5.0（iPhone; Mac OS XのようなCPU iPhone OS 8_3）AppleWebKit / 600.1.4（KHTML、Geckoのような）バージョン/ 8.0モバイル/ 12F70 Safari / 600.1.4（互換性; Googlebot / 2.1; + http：// www .google.com / bot.html ）



2016年4月18日から、新しいGoogleスマートフォンユーザーエージェントが使用されます。

Mozilla / 5.0（Linux; Android 6.0.1; Nexus 5X Build / MMB29P）AppleWebKit / 537.36（KHTML、Geckoなど）Chrome / 41.0.2272.96 Mobile Safari / 537.36（互換性; Googlebot / 2.1; + http://www.google .com / bot.html ）













このような変更の理由は、検索ロボットの新しい機能です。 ロボットが大幅な更新を受け取るたびに、最新のテクノロジーを使用するページを処理する機能を備えています。 この場合、新しい行形式は、技術的にGooglebotがSafariではなくChromeブラウザーに近づいたことを意味します。



Web開発ツール



あなたのサイトが私たちの検索エンジンをどのように見るかを理解することは、あなたが思っているよりも簡単です。 ツール「Googlebotの仕組みを見る」は、このために特別に開発されました。 また、さまざまなブラウザでサイトを簡単に表示したいが、どこから始めればよいかわからない場合は、ツールを使用してサポート機能を確認し 、進歩的な改善の原則に従うことをお勧めします。 まあ、これは、「承認された標準を使用する」、「実験的な機能を使用しない」、「必要以上にエンティティを作成しない」などの明白なヒントを破棄した場合です。



予測によると、ユーザーエージェントの行を変更してもほとんどのサイトの処理には影響しませんが、サイトがこの行をチェックすると問題が発生する可能性があります。



まず、私たちの信念によれば、検索エンジンのコンテンツは、ユーザーが見るコンテンツと異なってはいけません。 そのため、検索エンジンを誤解させるためにユーザーエージェントに応じてコンテンツを確認および変更することは、Googlebotによって偽装の 1つと見なされます。 このサイトは、Googlebotを通常のブラウザとして認識する必要があります。



サイトでエラーが発生すると思われる場合は、Search Consoleでこのツール * （更新されたユーザーエージェント行を使用）を使用して確認するか、ブラウザーの開発ツールで行を変更します（Chromeブラウザーでモバイルデバイスのエミュレーションモードを使用するなど ） 。 ご質問がある場合は、ウェブマスター向けのヘルプフォーラムで質問するか、こちらのコメント欄で質問してください。



*-Googleウェブマスターアカウントとそれに関連付けられたサイトでのログインが必要です。



第二に、ほぼ1年前に、サイトのモバイルバージョンの多くの側面に影響を与える検索エンジンのメジャーアップデートについて話しました。 その発表を見逃した場合は、Hubrablogでそれについて読むことができます。 サイトで関連性がある場合は、1年にわたってすべてのウェブマスターがモバイル版のサポートを引き受けてくれることを願っています。 まあ、時間がないか、あなたが望むすべてをすることができなかった場合-その資料の多くのヒント、推奨事項、および有用なリンクはまだ関連しています。
