Root'aを使用せずにメモリカードにアプリケーションを強制的に再配置またはインストールする

このトピックは、Android OSのユーザーを対象としています。特に、デバイスの内部メモリが不足しているという差し迫った問題の解決策です。

そして最も重要なことは、この記事で説明されているアクションは、ルート権限の取得を必要としないことです。ルート権限はほとんどの一般ユーザーにとって怖い要因です。 興味をそそられるすべての人に、猫をお願いします。



おそらくGoogleの Android 軸のすべてのユーザーは、ステータスバーに通知が表示され、デバイスの内部メモリの枯渇を通知し、一時的なアプリケーションファイルを削除するか、アプリケーション自体を削除することを提案する問題を認識しています。 また、Androidバージョン2.2以降を使用している場合は、アプリケーションをカードに移動することを提案します。 しかし、問題は、すべてのアプリケーションが移動されるのではなく、作者がこれを祝福したアプリケーションだけが移動されることです。 すべてのアプリケーションが必要で、メモリが使い果たされ、作成者がマップへの移動を許可しなかった場合、どうしますか？ これについては後で説明します。



最初のステップ


HTC Syncをインストールします。 すべてをインストールすることはできません。HTCDriver Installerコンポーネントだけで十分です。



第二段階


Android SDKをインストールします 。 インストーラーを取ります







、たとえばシステムドライブのルートにインストールします：C：\ android-sdk-windows。



次に、新しいシステム変数を作成します。 これを行うには、[マイコンピューター]-[プロパティ]-[システムの詳細設定]に移動し、環境変数をクリックして、新しいシステム変数を作成します。 変数名：adb変数値：C：\ android-sdk-windows \ tools。



次に、Android-sdk-windowsフォルダーからSDKマネージャーを実行します。 プログラムは、必要なコンポーネントの可用性を確認し、不足しているコンポーネントのインストールを提案します。これを大胆に拒否します。 エラーが発生した場合は、[ツール]に移動し、[その他]に2つのチェックマークを付けます。 これでソフト​​ウェアの準備が完了し、最も興味深いものに移ります。



第三段階


[ SDカードに移動 ]ボタンを有効にします。 電話をPCにケーブルで接続し、同期モードを選択します。 PCでコマンドラインを実行し、次のように入力します。C：\ android-sdk-windows \ platform-tools \ adb shell press Enter、$アイコンが表示されます。



次に、次を入力します。pm setInstallLocation 2（登録が重要です！）。 応答として、Pm set InstallLocation 2行と$記号が表示されます。



その後、デバイスをPCから切断して再起動します。 再起動後、設定-アプリケーション-アプリケーション管理に移動し、アプリケーションを選択して、[SDカードに移動]ボタンがアクティブになっていることを確認します。 これで、デフォルトでは、すべてのアプリケーションがカードに配置されます。 ただし、「ディスクドライブ」モードで電話機をPCに接続すると、カードがマウント解除され、そのアプリケーションが一時停止することを忘れないでください。 そのため、バックグラウンドでハングしているアプリケーション（ウィジェット、ライブ壁紙、インスタントメッセンジャー）は、インストール後に携帯電話に移動することをお勧めします。



そしてもう1つ：念のため、pm SetInstallLocationコマンドの属性：

pm setInstallLocation 0-アプリケーションのインストール場所を自動選択します。

pm setInstallLocation 1-内部メモリへのインストール（「電話へ」）

pm setInstallLocation 2-外部メモリへのインストール（「カードへ」）



それがすべて、良い動きであり、あなたの注意をありがとう。
