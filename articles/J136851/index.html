Lenovo ThinkPadタブレットのルート

Lenovo ThinkPad Tabletでrootになる方法：

タブレット自体は、ハブとここですでに言及されています 。



以下に手順を示しますが、写真は一切ありません。



1.誰が最新のアップデートを持っていないか、すなわち タブレットソフトウェアのバージョンは... 0039_0089_ROWではなく、古いバージョンです。



Lenovo ThinkPadタブレットでルートを取得することに関する次のトピックをお読みください。

翻訳するのが面倒な人のための簡単な指示：



1.最新のADBドライバーがインストールされており、デバイスがUSBデバッグモードになっていることを確認します。

ドライバーをインストールし、デバッグモードを有効にする方法は、 ここに記載されています 。

2. ここから PCにルートエクスプロイトをダウンロードします。

3. Lenovo ThinkPad Tabletの電源がオンになっており、USBを介してPCに接続され、画面がロック解除されていることを確認します。

4. PCでアーカイブを解凍し、「run.bat」を実行します。 画面の指示に従い、インストールが完了するまでタブレットの何も押さないでください。



出来上がり！



私から：インストールプロセス中にセグメンテーションフォールトが発生しますが、バッチファイルが明示的に述べているように、それは何も悪いことではありません。

echo [*]注：セグメンテーション違反は発生するはずであり、無視する必要があります。



結局のところ、rootをインストールするだけでは十分ではありません。 引き続きシステムを更新しても、それを保存できる必要があります！

ルートのインストール後にソフトウェアをバージョン..0039_0089_ROWに更新すると、ルートが失われます。

これを防ぐには、 トピックを注意深く読んで、そこに書かれていることをすべて行ってください。

簡単な指示：



1.上記のエクスプロイトを使用して、ThinkPadタブレットにルートをインストールします。

2.スーパーユーザーアプリケーションがインストールされていることを確認します（ ダウンロードしてインストールします ）

3. Voodoo OTA RootKeeperアプリケーションをインストールします（ ダウンロードしてインストールします ）

4. RootKeeperを起動して、スーパーユーザーのバックアップを作成します。

5.同じRootKeeperで、「Temp。 un-root（バックアップを保持）”（一時的なルート解除）」を選択すると、ルートの復元オプションが表示されます。

6. [設定] / [タブレットについて] / [更新の確認]に移動して、更新を確認してインストールします... 0039_0089_ROW。 更新には約7分かかります。

7.アップグレード後、スーパーユーザーアプリケーションがインストールされていることを確認します。 そうでない場合は、Androidマーケットからインストールします。

8. RootKeeperを開き、[ルートの復元]を選択します。

9.おめでとうございます。 すべてが正しければ。 次に、最新の更新（いわゆるOTA2）と作業ルートがあります。



2.既に... 0039_0089_ROWに更新しているが、ルート化する時間がなかった場合、または忘れた場合...

...次に、Lenovoが穴を削除したため、上記のrootのインストール手順は機能しません。



しかし、それは重要ではありません、ルートをインストールする機会がまだあります！

私たちはこのトピックを注意深く読みましたが、これはすぐに良いです 。

最も読みにくい英語の場合：

1. download.lenovo.com/slates/think/tablet1からOLD更新をダウンロードします（ROWファームウェアを使用したTPTの場合、これはThinkPadTablet_A310_02_0024_0065_ROW.zipです ）

2. SDカードに書き込み、タブレットに入れます

3.設定で、USBデバッグを有効にします。[設定->アプリケーション-> Developmnet-> USBデバッグ]

4. ここからアーカイブをダウンロードして解凍します。



注意！

ここで小さな余談をする必要があります。 元のパッチは適切に機能しません。 本当にルートを取得できますが、エラーのために常にインストールすることはできません。

このトピック 、特に4ページを注意深く読み、ほぼ正しいファイルpwn-in.shがあるか、ここからアーカイブをダウンロードできます （修正されたpwn-in.shがあります）。 すべての修正は、 mv src destをcat src> destに置き換え、 mv /system/app/Superuser.apk / data / local / pwn-bakをcat /system/app/Superuser.apk>に正しく置き換えたことに要約されます。 pwn-bak / Superuser.apk

ルートを取得した後、単純にすべてのコマンドをadbシェルに手動で入力しました。



5. Lenovoデバイスドライバーをインストールします（まだ実行していない場合）

6.タブレットを接続します。

7.次の対象のリカバリモードで起動します。

-タブレットの電源を切り、電源を入れ、「音量を上げる」ボタンを1秒間に1〜2回数回押し始めます。

-白い碑文またはアンドロイドのアイコンが表示されたらすぐに、刈り取りを停止し、回復モードがロードされるのを待ちます

-音量ボタンを使用してメニューをナビゲートします。 アイテムを選択-電源ボタン

-すべてがうまくいけば、通常モードになります

-PCで、エクスプロイトが展開されているディレクトリに移動し、run.batを実行して指示に従います。



「Enterを押して、デバイスを再起動する」というスタイルの指示は簡単です。 正しく実行する必要があるだけです。



すべてのアクションの最後に、ソフトウェアの最新バージョン（いわゆるOTA2）のルートを取得します 。 今後、アップデートをインストールする前にルートキーパーを使用することを忘れないでください 。
