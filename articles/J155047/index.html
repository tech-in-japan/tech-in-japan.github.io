JollaのSailfish OSをどのように見たいですか？

Jollaが11月21日に新しいSailfishオペレーティングシステムのインターフェイスとSDKを導入することを発表した後、会社の管理に変更が加えられました。 CEOのJussi Hurmolaは新しいOSの開発に直接専念するためにポストを辞めました。Jollaの新しいCEOは、以前はMaemo＆MeeGoプロジェクトのチーフエンジニアの1人であったMark Dillonでした。 NokiaのS40 CoreSW Integrator、Symbian Build Manager / Nokiaのチームリーダー、Nokiaの構成管理管理者。







OS Sailfishの発表を待って、新しいOSの機能とインターフェイスに対するユーザーの希望を定式化しようとしました。 このリストは明らかに網羅的ではなく、おそらく主観的なものです。 コメントが大幅に補足または修正されることを願っています。







1.ユーザー自身がバックグラウンドで特定のアプリケーションの動作モードを割り当てることができるようにします。



2.ユーザーがオープンシステムを使用するかクローズドシステムを使用するかを決定するために（実際、これは既にMeeGo Harmattanで実装されています。この機能のみを保持したいです）。



3.デバイスが最大負荷モードで少なくとも20時間動作するようにするため（もちろん、合理的な範囲内で、デバイスの厚さは個人的には気にしません）。



4. OSが既存のすべてのOSのアプリケーションをサポートするために、たとえば、仮想マシンの原理に従って、ハードウェア部分ですでにこれが許可されています。 既に発表されているような、Android Playマーケットのアプリケーションのサポート。



5.動的再充電がオペレーティングシステムに実装されるように、つまり、スマートフォンがポケットに横たわっている間に変動から再充電されるようにします。



6.オペレーティングシステムインターフェイスがデバイスのhadvarnyeコントロールボタンを必要としないように（つまり、この意味で、N9に関して一定の連続性が維持されるように）。



7. OSがx86アーキテクチャをサポートするため、つまり、OSまたはその派生物を、ラップトップやデスクトップだけでなく、スマートフォンやタブレットにも同様にインストールできます。



8. OS Sailfishファームウェアは、さまざまなメーカーの多数の主力Androidデバイス向けにリリースされます。 一般的に、この方向のOS Sailfishの開発は、私の意見では、主なものの1つである必要があります。



9.インターフェースを本格的なマルチタスクだけでなく、同じデスクトップウィンドウで複数のアプリケーションを開くことを含むマルチウィンドウモードで実装するため。



10 1つ（または複数）のクラウドサービスとの深い統合をOSに実装するために、任意のデータを同期できます。
