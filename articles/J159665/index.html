IntelがARMチップ開発者を引き継ぐ-ZiiLabs

Intelは、さまざまなAndroidデバイスで使用されているZMS-40およびZMS-20マルチコアARMチップの開発者であるCreative Technologyの5000万ドルの子会社であるZiiLabsを買収するため、Creative Technologyとの契約を発表しました。タブレットCreative Zii Eggやその他のデバイスなどのメディアプレーヤー。



ZiiLabsは、2012年1月にZMS-40 100コアSoCチップを導入したことで知られています。 また、モバイルデバイス市場では、ZiiLabs GPUソリューションはImagination Technologiesが開発したPowerVRグラフィックチップと競合しています。



契約条件によると、5000万ドルのうち、ZiiLabsの固定資産とエンジニアリングリソースに約3000万ドルが支払われ、残りはZiiLabs GPUテクノロジーの特許とライセンスに支払われました。



最新のZiiLabs開発であるZMS-40 100コアSoCチップは、1.5 GHz 4コアARM Cortex-A9プロセッサ（Neonマルチメディアユニットを含む）と96個のシンプルで汎用性の低いStemCellコンピューティングコアのアレイを組み合わせています。

StemCellコアはエネルギー効率の高いSIMDアーキテクチャであり、32ビット浮動小数点計算のピークパフォーマンスは50 GFLOPSであり、そのコアはGPUとして機能し、ビデオ、画像、オーディオの処理、3Dおよび2Dグラフィックスの加速、その他のマルチメディアタスク（OpenGL ES 2.0およびOpenCL 1.1でサポート）-記事「ZiiLabsで導入された「4 + 96コア」ZMS-40プロセッサー」を参照してください 。



以前、Intelは、イマジネーションテクノロジーズが開発したPowerVRテクノロジーをデスクトッププロセッサの組み込みGPUとして使用し、Intel x86命令アーキテクチャに基づくディスクリートGPUアクセラレータのリリースを試みました。最近導入された60コアのIntel Xeon Phiコプロセッサー（ 「Intel Xeon Phi 5110Pコプロセッサー」を参照）により、1 TFLOPSの倍精度計算でピークパフォーマンスを達成します。



一般的に、今日、Intelは100コアを使用したいマルチコアプロセッサの開発に大きな関心を示しており、主にこれらのマルチコアプロセッサはサーバー市場に特化しています（Xeon Phiコプロセッサは今日、スーパーコンピューターはGPGPUを最も頻繁に使用しています） NVIDIA Tesla K20やAMD FirePro S10000などのアクセラレータ ）、Intel x86命令アーキテクチャがこれにあまり適していないことは明らかです。 おそらく、IntelがZiiLabsの革新的な開発と特許を利用することを決定した理由です。



IntelがZiiLabsのStemCellコアの配列を必要とした多くの質問があります。

1.インテルは、ZiiLabsテクノロジーを搭載した将来のマルチコアプロセッサ（ZMS-40など）でARMコアを使用しますか、それともIntel x86コアに置き換えますか？

2.一般に、従来のデスクトッププロセッサ向けのZiiLabs開発は統合GPUアクセラレータとして使用されますか、それともサーバー市場向けのGPGPUコンピューティング（Xeon Phiなど）用の超並列マルチコアプロセッサの開発のみに任せられますか？

3.または、 AMD Fusionアーキテクチャを作成するためのAMDのアプローチが正しいと判明した場合、Intelは最新のIntel Core iX + ZiiLabsコアアレイStemCellを使用する独自のAccelerated Processing Unit（APU）の開発を決定しましたか？ その結果、デスクトップコンピューターには、まもなくディスクリートGPUアクセラレーターが必要なくなるでしょう。

4.または、IntelはまだZiiLabsを使用して、NVidiaおよびAMDのディスクリートGPUと競合する独自のディスクリートGPUアクセラレータをリリースすることを決めているのでしょうか。
