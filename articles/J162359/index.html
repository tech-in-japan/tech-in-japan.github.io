「テスター」の給与の見直し

私たち、そして私たちはITを支配する人材紹介会社であり、私たちの投稿の1つを一般に「テスター」と呼ばれるテストエンジニアの給与調査に捧げることを長い間約束しています。 また、前回の出版物で、 システム管理者（Windows＆* nix） 、 Java / C ++ / C＃/ Delphi プログラマー 、およびWEBプログラマーの給与について話したことを思い出してください。







そのため、このレビューでは、次の役職の給与に焦点を当てます。
  • ジュニアテスター（手動および/または自動化）
  • テストエンジニア（マニュアル）
  • リードテストエンジニア（マニュアル）
  • テストエンジニア（自動化）
  • リードテストエンジニア（自動）
  • テストチームリーダー（手動および/または自動化）




研究パラメータ

研究対象：空室

調査対象都市：モスクワ、サンクトペテルブルク、ノボシビルスク、エカテリンブルグ、ニジニノヴゴロド。

出典： hh.ru 、 SuperJob.ru 、 ITmozg.ru 、 Job.ru 、 Rabota.ru 、 rabota66 （エカテリンブルグのみ）、 Rabota NGS （ノボシビルスクのみ）、

処理された空室の数は、プレートに示されています。

調査期間：2012年1月6日から2012年8月30日まで



調査方法

メインの求人ポータルで、空席/履歴書に示されている給与に関するデータを収集し、空席/履歴書の名前に従って投稿のディレクトリにレイアウトします。 空席/履歴書の名前が一般的または不正確である場合（たとえば、「テスター」または「テストエンジニア」）、空席/履歴書の説明を検討した後、専門家の評価に基づいて特定のポジションにそれを選択します。 この投稿では、 空席から取得したデータのみを使用します。



用語集

10パーセンタイル -統計上の最小値

中央値（50パーセンタイル） -平均市場賃金

90パーセンタイル -統計的最大値



N （10、50、90） パーセンタイルは、順序付き変分系列の開始からの距離の1 / nあたりの値です。 つまり、n番目のパーセンタイルは、ランク付けされた系列の値のn％がn番目のパーセンタイルの値よりも低い値を持ち、（100-n）％が大きいことを示します。



ジュニアテスター（手動および/または自動化）



ジュニアテスターのこのような少数の欠員、および彼らがほとんど完全に欠席している（4未満）地域では、私たちは、ほとんどの場合、入国要件が低い場合、雇用者がそのような専門家を非常に迅速に見つけるという事実によって説明します。 一方、若い専門家にとって、この種の欠員は非常に興味深いものです。





テストエンジニア（マニュアル）



テストエンジニアは、通常「ブラックボックス」でテストするテストケースとテスト計画を個別に作成できます。





リードテストエンジニア（マニュアル）



テストエンジニアに加えて、経験が豊富です。 おそらく、コードを読み取る機能は「ホワイトボックス」をテストしているのでしょう。





テストエンジニア（自動化）



彼らは既製の自動テストを使用し、それらを適応させることができます。 これらには1つ以上のプログラミング言語があり、スクリプト言語は仕事でよく使用されます。

テストチームリーダー（手動および/または自動化）





まとめると



テストエンジニアの需要は毎年増加しています。 これは、第一に、単にテストするだけでよい開発中のソフトウェアの複雑さが増していること、そして第二に、開発中のソフトウェアの品質に対するユーザー側の要件が高まっているためです。 5年前にテスターの職業が好奇心だった場合、今日、テスト部門または少なくとも「ミニセパレーター」を持たないソフトウェア会社を想像することは困難です。



IT企業に調査への参加を呼びかけます。



インターネットデータの調査に加えて、年に1回、 IT企業から提供されたデータに基づいて調査を行っています 。 この調査の一環として、労働市場の実際の賃金だけでなく、社会的パッケージの構成要素、さまざまな種類の福利厚生、報酬、スタッフの離職率、さまざまなプロバイダーの費用、大学とのやり取りなどを調査します。
