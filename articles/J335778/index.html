ハッカソンチームNavalnyに関するレポート

私は、2018年のロシアでの大統領選挙の候補者としてAlexei Navalnyを登録するために、キャンペーン本部のIT部門で働いています。 開発者の少人数のスタッフがいますが、リソースはすべての興味深いプロジェクトを実装し、新鮮なアイデアを生み出すのに十分ではありません。







そのため、7月26日に、キャンペーンとそのボランティアを支援する製品とツールを作成するハッカソンを発表しました。



ITプロジェクトの作成に関係するすべての人にプロジェクトのアイデアを提案するよう依頼し、それを持たない人は自分で提案しました。 2週間で、100を超えるアプリケーションが届きました。その大半は開発者からです。 さまざまな言語/テクノロジーに言及しているアプリケーションの数は次のとおりです。



パイソン-27

JavaScript-18

C / C ++ / C＃-17

Java-16

PHP-11

Swift / Objective-C-7

NodeJS-4

ルビー-3

Scala-3

行く-2

アセンブラー-1



メインスタックはPython（Django）とJavaScript（jQuery、React）であるため、このディストリビューションは私たちを満足させました。



Telegramでの一般的なチャットですべての参加者を事前に収集し、プロジェクトの議論とチームの検索を開始しました。 その結果、8月12日土曜日に57人がモスクワとサンクトペテルブルクの本社に来たとき、彼らのほとんど全員が誰とどのような仕事をするかを既に知っていました。 合計で19チームが判明し、2日以内にキャンペーンを支援する方法についてのアイデアを実現しました。



ハッカソン全体を通して、ITチームのリーダーと招待された専門家が参加者に助言し、最適なソリューションの選択を支援しました。 ファイナルでは、チームは彼らが何をしたかについて話し、プロトタイプを示し、 ju審員（キャンペーン本部のレオニード・ヴォルコフ 、FBKディレクターのロマン・ルバノフ 、FBKクリエイティブディレクターのエレナ ・マルス 、ITコンサルタントのヴラディスラフ・ズドルニコフとUXの専門家アレクシー・コピロフ ） 3つの受賞プロジェクト。 Bulk Liveアプリケーション、仮想コールセンター、Telegram-CECになりました 。







これらおよびその他の興味深いハッカソンプロジェクトの簡単な説明を次に示します。 githubへのリンクが含まれているため、プロジェクトに関心がある場合は、いつでも参加して参加者がプロジェクトを終了できるようにすることができます。



  • Bulk Liveアプリケーションを使用すると、YouTubeチャンネルと録音からのライブストリーミングを聴くことができます。 チームは、スタジオからRTMPストリームを受信する準備が整ったサーバーをセットアップし、ユーザーに配信しました。APIは、ブロードキャストメタデータとiOSアプリケーションを受信します。
  • バーチャルコールセンター （または、「クラウドダイヤラー」と呼ぶ）は、各ボランティアが有権者に電話をかけ、候補者に投票するようキャンペーンすることができるシステムです。 また、本部での会話のために提案されたスクリプトをユーザーに表示します。 これは、アスタリスクのコンピューターテレフォニーシステムに基づいています。
  • Telegram-CECは、Voice Association SMS-CECのプロジェクトの論理的な開発です。 テレグラムボットを介して、オブザーバーから予備選挙結果を収集し、違反の単一データベースを収集し、全国の選挙当日の状況をリアルタイムで地図上で監視することができます。





    navalny-observer.herokuapp.com



  • ボット検出器は、ソーシャルネットワークからのコメントから、作成者がOlginoの居住者である確率を判断します。 私たちは、インターネットの清潔さのための戦いで積極的にそれを使用する予定です。
  • ポッドラウンドマップは、各戸ごとの攪拌システムです。 各地域と地区のキャンペーン範囲を示し、ボランティアがサインアップして特定の入り口を迂回し、電報ボットを介して行われた作業について報告できるようにします。
  • Navalny Answerサイトでは、候補者からの質問に対するビデオの回答を検索したり、まだ質問されていない質問を追加したり、回答がまだ追加されていないビデオを追加したりできます。
  • Pulseアプリケーションでは、参加者は「 赤いボタン 」機能の実装を提案し、ユーザーの周りの小さな半径にいるすべてのサポーターに何かが起こっていること（良いことと不快なこと）を通知し、参加を呼びかけます。


私たちはハッカソン後もこれらのチームや他のチームと協力し続け、できるだけ早くそれらをリリースして広範囲に使用できるようにします。 参加し、すべてを整理するのを手伝ってくれた人たちに感謝するとともに、すでに開始されているプロジェクトと私たちのキャンペーンの将来のハッカソンに参加するよう皆を招待します。
