Google App Engine SDK 1.4.0がリリースされました









12月2日に、Google App Engine開発者がSDKの新しいバージョンをリリースしました。 誇張せずに、このリリースは革新と改善が豊富です。 あなたはカットの下で新機能と新バージョンの他の利点について知ることができます。







主な革新の中で：


1. チャネルAPI 。 WebSocketのネイティブサポートは、チャット、ゲームなどのリアルタイムWebアプリケーション、および高度なインタラクティブ機能を備えたアプリケーションを設計する場合に役立ちます。 Javascriptクライアントと透過的に動作し、Googleトークインフラストラクチャでスピンします。 利用可能なドキュメント



2. 常にオン 。 有料（月額9ドル）で、3つのインスタンスがアプリケーション専用に予約され、トラフィックがない場合でも継続的に動作します。 これにより、コールドスタート時の待ち時間が大幅に短縮されます。 ご注意 無料で、Task Queueを介して定期的なクエリを使用して、アプリケーションを「ウォームアップ」できます。 またはCronですが、インスタンスは1分でアンロードできます。コールドスタートはCPUにとって高価な操作です。







3. ウォームアップリクエスト 。 これで、新しいインスタンスの開始時に、ただしこのインスタンスによってトラフィックが直接処理される前に起動される特別なハンドラーを指定できます。 デフォルトでは、「常時オン」を購入した人に対して有効になっています。



4. バックグラウンドタスクのタイムアウトの増加 。 これで、タスクキューとCronは、以前のように最大30秒間ではなく、最大10分間タスクを処理できます。 これで、タスクを小さなタスクに分解する必要がなくなりました。



5. APIへのリクエスト/レスポンスの量の制限が増加しました 。

  • URLFetchは最大32MB（以前の1MB）の応答を返すことができます
  • Memcache get / setは、最大32MBのデータを処理できます。 1つのオブジェクトの制限はまだ1Mbです
  • Image APIは、最大32MBの情報（以前の1MB）を送受信できます。
  • Mail APIを介した添付ファイルのサイズが1 MBから10 MBに増加しました


6. メタデータクエリ 。 これで、リポジトリ内のエンティティタイプ、そのフィールド、ネームスペースに関するメタ情報を取得できます。 ドキュメント



7.アプリケーションをダウンロードした開発者は、 自分のバージョンのソースコードをダウンロードできます。 このオプションは無効にできますが、有効にすることはできません。 Java開発者はバイトコードをダウンロードしますが、驚くことはありません:)



ブログでは、High Replication Datastoreなどの新しいグッズの差し迫ったリリースについて言及しました。



変更の完全なリストは、 Python SDKリリースノート / Java SDKリリースノートに記載されています。 更新されたロードマップもご覧ください
