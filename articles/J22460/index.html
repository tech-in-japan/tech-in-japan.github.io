MySQL + Leopardの問題

昨日、ようやくLeopardにアップグレードし、必要なすべてのアップデートがインストールされ、今では仕事に就こうと考えていましたが、そのような問題に遭遇しました。 わずかに異なるPHP5アクティベーションパス（Tigerでは、httpd.confは/ private / etc / httpd /フォルダーにありましたが、現在は/ private / etc / apache2 /にあります ）、この問題はすぐに解決され、まったく問題ではありません（-：

しかし、MySQLはすでに実行されている（dmgからインストールされている）場合、予想よりもはるかに時間がかかり、phpはデータベースでの作業を完全に拒否し、 / tmp / mysql.sockが欠落しているという事実に動機付けられ、phpとmysqlが一緒に作業を拒否します。 インターネットを勉強して、この問題に遭遇したのは私だけではないことに気づきました。この問題を解決する方法はいくつかあります。そのうちの1つを次に示します。



判明したように、ソケットは/ private / tmpに移動し、必要だったのはphp.ini（/private/etc/php.ini）の行を補足することだけでした

mysql.default_socket =

そして

mysqli.default_socket =

前に

mysql.default_socket = /private/tmp/mysql.sock

そして

mysqli.default_socket = /private/tmp/mysql.sock

それに応じて。



おそらく他の誰かがこの問題に遭遇したか、または遭遇するでしょう。昨日私を助けてくれた解決策によってあなたが助けられることを願っています（-：



PS PHPバージョン5.2.4、MySQL-5.1.23
