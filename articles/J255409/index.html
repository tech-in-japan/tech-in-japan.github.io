Veeam Backup＆Replicationを使用した仮想化Microsoft Exchangeのバックアップと復元

Exchangeを仮想レールに切り替えたら、バックアップを開始し、必要に応じて復元します。 今日は、Veeam Backup＆Replication 8.0を使用してこれを行う方法を簡単に説明し、より詳細な情報のソースへのリンクを共有します。







バックアップジョブのセットアップ



アプリケーションの操作を考慮して正しいバックアップを作成するには（この場合、Exchangeについて説明します）、バックアップタスクで、対応する設定（アプリケーション対応処理）をアクティブにする必要があります。 次に、タスクを完了すると、一貫性のあるバックアップを作成するために必要な手順が続きます。 要するに、次のことが起こります。

  1. Veeamは、作業中のアプリケーションの存在について処理中の仮想マシンのゲストファイルシステムを分析し、VSS（またはアプリケーションとゲストOSを「フリーズ」する他の方法）を使用してバックアップする必要があるものを理解します。
  2. VSSが使用されない場合、Veeamは指定されたスクリプトを「フリーズ」する前に実行できます。
  3. 「フリーズ」が進行中です。
  4. 仮想マシンのスナップショットが作成されます。
  5. 霜取りが進行中です。
  6. 解凍後に実行する必要があるスクリプトが指定されている場合、Veeamはそれを起動します。
  7. バックアップデータが転送され、スナップショットがコミットされます。
  8. 最後に、ログの変換が実行されます（VSSを使用）。


一貫性のあるバックアップの作成については、 こちらまたはこちら （英語）または「仮想マシンをフリーズして一貫性のあるバックアップを取得する」を ご覧 ください 。



注：特定の条件下では、Microsoft VSS Persistent SnapshotのバックアップテクノロジーがExchangeバックアップに適用される場合がありますが、それ以外の場合はさらに適用されます。



そのため、今日、Microsoft VSSを使用してExchangeを処理するバックアップタスクを設定しています。

  1. タスクウィザードを起動して（たとえば、タスクの[ 編集]コマンドを選択して）[ ゲスト処理]ステップに到達したら、[ アプリケーション対応処理を有効にする]チェックボックスをオンにします。
  2. 次に、 ゲストOS資格情報セクションで、ゲストOSにアクセスするためのアカウントを指定します。







  3. [ アプリケーション ]をクリックし、リストでExchangeサーバーを選択し、[ 編集 ]をクリックして、アプリケーションとログの処理時に使用するオプションを指定します。
  4. まず、[ 全般 ]タブの[ アプリケーション ]セクションに移動します。







    アプリケーションの推奨オプションは、「 正常な処理が必要 」です。 選択すると、VSSの正常な操作に合わせてバックアッププロセスが実行されます。そうでない場合、バックアップは停止します。 （もちろん、 アプリケーション処理を試すを選択できますが、 失敗は無視します -ただし、VSSエラーの場合、そのようなソリューションはクラッシュ整合性のあるバックアップを作成するだけであり、これは私たちが目指しているものではありません。）

  5. [ トランザクションログ]セクションに移動します。 ここで、バックアップが正常に完了した場合のログ（トランザクションログ）の処理方法を指定する必要があります。 ロジックは次のとおりです。

    • このジョブでトランザクションログを処理するが選択されている場合、プロセスはVeeamによって作成されたバックアップに焦点を当てます：バックアップが成功した場合、ログの切り捨てはゲストファイルシステムで実行され、そうでない場合（ログは保存されます）。
    • また、[ コピーのみ実行]が選択されている場合、バックアップ中にVSS_BS_COPYメソッド（COPY ONLYフラグ）がスナップショットの作成に使用されます。 必要に応じて、ログは変更されずに残り、サードパーティのアプリケーション（Veeamではない）で処理できます。




[ OK]をクリックして設定を保存し、ウィザードの手順に戻って完了します。



インスタントVMリカバリ



Exchangeサーバーの災害復旧の場合、インスタントVM復旧を使用できます。これにより、文字通り数分で仮想マシンを稼働状態に復元できます。 これを行うには、Veeam vPowerテクノロジーが使用されます。これにより、圧縮および重複排除されたバックアップから仮想マシンを直接起動できます。 つまり、リポジトリからバックアップを削除し、本番ストレージなどにコピーする必要はありません-代わりに、VeeamはVeeamバックアップサーバー（通常はvPower NFSのサーバーとしても機能します）にディスクをマウントすることにより、リポジトリ内の仮想マシンファイルにアクセスします。 その後、たとえばソースマシンの修復中に、仮想マシン自体をオンにして使用を開始できます。これにより、エンドユーザーの作業を中断することがなくなります。 ローカルディスク上のスペースは、記録用の一時領域として使用されます。すべての変更は、マシンがバックアップから作業している間にそこに書き込まれます。



復元ウィザードは、[ 復元 ]メニューコマンドによって起動されます。 実行したいこと、つまりインスタントVMリカバリを選択します 。







回復ウィザードの手順を実行します。



  1. まず、復元する仮想マシンを選択します。
  2. 次に、回復するポイント （ Restore Point ）と回復モード（ Restore Mode ）を選択します。

    • 元のマシンが永久に失われたために元の場所に復元する場合は、 [ 元の場所に復元]を選択します。 その後、すぐに「理由の復元」ステップに進み、仮想マシンに必要なすべての設定（ホスト、リソースプール、フォルダー）がバックアップから正確に取得されます。



      重要！ 元の仮想マシンがインフラストラクチャにまだ存在する場合、このオプションは選択できません。



    • 仮想マシンを別の場所または異なる設定で復元する場合は、それぞれ、 新しい場所または異なる設定で復元を選択します （場所、ネットワーク設定、仮想ディスクの形式などを変更できます）。 このオプションを選択した場合、一連の手順を実行して必要な設定を入力する必要があります。


  3. 復元の理由のステップでは、リカバリを行う理由を示し（監査の場合に必要になる場合があります）、さらに2つのオプションを選択できる適用準備のステップに進みます： VMをネットワークに接続（仮想マシンをネットワークに接続）およびVMを自動的にパワーオン （仮想マシンを自動的にオンにします）。






したがって、リカバリが完了すると、仮想マシンは使用可能な状態になります。



注：異なる仮想化プラットフォーム（VMwareとHyper-V） の回復ウィザードUIは若干異なりますが、一般的な説明とオプションはほぼ同じです。



[ 次へ ]をクリックすると、 VeeamはNFSストレージ（VMwareの場合）または適切なドライバー（Hyper-Vの場合）を使用して指定された設定で回復操作を実行し、バックアップファイルから仮想マシンを直接起動します。

結論として、Storage vMotionまたはLive Migration（仮想化プラットフォームのメーカーによる）または独自のVeeam Quick Migration機能（VMwareのみ）を使用して、マシンを実稼働ストレージに移行します。

インスタントVMリカバリの詳細については、 こちらをご覧ください 。



Veeam Explorer for Microsoft Exchange：個々のオブジェクトの回復



インスタントVMリカバリは強力ですが、それを使用することは必ずしも便利ではありません。 たとえば、失われた1つの電子メールメッセージを回復する必要がある場合があります。このために、データベース全体を含むサーバー全体をバックアップから持ち上げないでください。 この状況では、 Veeam Explorer for Microsoft Exchangeツールが役に立ちます （Exchange 2010 SP1以降をサポート）。 これを使用すると、目的のオブジェクト（レター、カレンダー、またはタスク）のみを復元できます。 Habrのこの記事「仮想化されたアプリケーションオブジェクトのグラニュラー復元」で、ユニバーサルリカバリーとグラニュラーリカバリーの違いについて詳しく読むことができます 。



[ ホーム ]メニューから目的のプラットフォームの[ 復元]コマンドを選択し、[ アプリケーションアイテム ]を選択します 。 次に、オブジェクトを復元する特定のアプリケーションを示します。この例では、Microsoft Exchangeです。







Microsoft Exchange Item Level Restoreウィザードが起動します。 その中で、Exchange Serverと回復するポイント（復元ポイント）を選択します。 必要に応じて、回復が必要な理由を示します（Restore Reason）。 その後、Veeam Explorer for Microsoft Exchangeが自動的に開き、バックアップExchangeサーバーのメールボックスデータストア（.EDB）の内容が表示されます。

このツールを使用すると、メールボックスデータストアの内容を表示し、必要なレター、タスク、またはカレンダーのメモを検索できます。







目的のオブジェクトを見つけたら、それを右クリックするか、メニューコマンドを使用して、復元方法（メールによる送信、サーバーへの復元など）を正確に選択します。



Veeam Explorer for Microsoft Exchangeは、無料の無料版を含むVeeam Backup＆Replication 8のすべてのエディションで、「完全削除」（完全削除）オブジェクトとアーカイブメールボックス（オンラインアーカイブメールボックス）の復元をサポートしていることに注意してください。 Exchangeサーバーでの復旧は商用バージョンでサポートされていますが、無料の復旧は.MSGに保存し、.PSTにエクスポートし、添付ファイルとして電子メールで送信することで利用できます。
