Visual Studio Tools for Unityがリリースされました

こんにちは、Habr！



数日前、Unityの開発に便利なソフトウェアを使用することに慣れているすべてのUnity開発者にとって重要なリリースがありました。UnityのVisual Studio Toolsの新しいバージョンがリリースされました。 今回は2.2。



何らかの理由でStudioを使用しない人のためのLikbez：バージョン2.1以降のVSTUは、Unityをネイティブでサポートしているため、Visual Studioを国内のゲーム開発者としての快適な生活に可能な限り統合できます。







主な変更：



1）VSTUインストーラーは、統合プロセス中にUnityが必要とするすべての情報を最も正確に提供します。

2）OS Xおよび仮想マシンでのUnityゲームの正しいデバッグ。

3）Unity 4.6では、VSTUはUnityEngineとUnityEditorへの参照を正しく配置します。

4）C＃4のみがサポートされます（Unityで直接）。

5）Unityパッケージのバグを修正しました。これは、スクリプトをまったく使用しないプロジェクトに表示される可能性がありました。



ネットワークゲームを作成してNetworkBehaviorを使用するユーザー向けに、VSTU 2.2にネットワークメッセージのサポートが追加されました（下のスクリーンショットを参照）







VSTUはUnityでネイティブにサポートされているため、Visual Studio GalleryからVSTUをインストールし 、Unity PreferencesのExternal ToolsタブでVisual Studioをデフォルトエディターとして設定するだけです。 設定の詳細については、開発者のWebサイトをご覧ください。







その後、VSTUサポートが自動的に有効になります。 これは、[Unityについて]ウィンドウの下部で非常に簡単に確認できます。







重要な点：Unity 5.1以下では、VSTUパッケージをプロジェクトに再インポートする必要があり、Visual StudioソリューションはUnityVSプレフィックスで作成されます。 Unityのすべての最新バージョン（5.2以降）ではこれは必要ありませんが、VSTUを古いバージョンのUnityで使用した場合は、プロジェクトからUnityVSフォルダーを削除する必要があります。



変更の完全なリストはMSDNで見つけることができ、Visual StudioのさまざまなバージョンのVSTUをダウンロードするためのリンクは次の行にあります。
