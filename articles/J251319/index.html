本の第14章： '' Web Storage API ''
2つのストレージシステム



当初、ネットワークは静的な情報を表示する手段として考えられていました。 その後、データ処理を開始しました。最初にサーバーアプリケーションを使用し、次にユーザーコンピューターの低パフォーマンススクリプトと組み込みモジュールを使用しました。 それでも、プロセスの本質は変わっていません。データはサーバーで準備され、ユーザーのコンピューターに表示されただけです。 システムにはクライアントコンピューターのリソースを使用する能力がないため、ハードワークはほとんど常にサーバー側で実行されました。



HTML5仕様のおかげで、状況はバランスが取れています。 HTML5開発者は、モバイルデバイスの機能、クラウドコンピューティングの出現、プラグインを通じて長年にわたって実装されてきた技術と革新を標準化する必要性に焦点を当て、すべての機能を1つの仕様にまとめて、クライアント側で本格的なアプリケーションを実行できるようにしました。

どのアプリケーションでも最も一般的な機能の1つは、データを保存し、必要に応じてアクセスすることですが、Webアプリケーションにはこれを実行する効果的なメカニズムがありませんでした。 Cookieはクライアント側で情報のごく一部を保存するために使用されましたが、その性質上、短い行のみの書き込みをサポートするため、あらゆる状況から遠く離れて使用するのが便利です。



Web Storage APIは、本質的にCookieの開発における次のステップです。 このAPIを使用すると、ユーザーのハードドライブにデータを書き込み、デスクトップアプリケーションで行われるのと同じ方法でデータにアクセスできます。 このAPIでサポートされるデータの保存および取得プロセスは、1つのセッションでのみ情報を使用できる場合と、ユーザーが削除するまで情報を長期間保存する必要がある場合の2つの状況に適用できます。 したがって、開発者の生活を簡素化するために、このAPIは2つの部分、sessionStorageとlocalStorageに分割されました。



  • sessionStorage。 これは、単一ページセッションのデータのみを保持するストレージエンジンです。 実際、実際のセッションとは異なり、1つのブラウザウィンドウまたはタブのみがアクセスできる情報について話しているため、ウィンドウまたはタブが閉じるとすぐにこの情報は削除されます。 ウィンドウの内容が更新されたときや、ユーザーが同じWebサイトの新しいページを開いたときでも情報が保存されるため、仕様では「セッション」という用語を使用しています。
  • localStorage。 このメカニズムは、デスクトップアプリケーションのストレージシステムと同様に機能します。 データは永久に記録され、保存したアプリケーションはいつでもこの情報にアクセスできます。




両方のメカニズムは同じインターフェースを介して機能し、同じメソッドとプロパティを提供します。 さらに、両方のメカニズムはソース固有です。つまり、リポジトリ内の情報は、それを作成したWebサイトからのみ利用可能です。 各Webサイトには独自のストレージスペースがあり、使用されるメカニズムに応じて、ウィンドウが閉じられるか、クリアされないときにクリアされます。

このAPIには、一時データと永続データが明確に区別されているため、数行だけの一時保存を必要とする小さなアプリケーション（たとえば、オンラインストアのバスケットのコンテンツ）と、しばしば必要になる大量の複雑なアプリケーションの両方を設計するのがはるかに簡単になりますドキュメント全体を無期限に保存します。



sessionStorage



APIの最初の部分であるsessionStorageは、セッションCookieを置き換えます。 sessionStorageなどのCookieは、限られた時間だけデータを保存します。 ただし、Cookieがブラウザーにバインドされている場合、sessionStorageオブジェクトは特定のウィンドウまたはタブにバインドされます。 つまり、特定のセッション用に作成されたCookieは、少なくとも1つのブラウザーウィンドウが開くまでアクセスできますが、sessionStorageによって作成されたデータは、ウィンドウが閉じるまでしか使用できません（特定のウィンドウまたはタブのみがアクセスできます）。 ）



データストレージの実装



sessionStorageとlocalStorageの両方のシステムは同じインターフェースで動作するため、コードサンプルをテストしてAPIを試すには、1つのHTMLドキュメントとシンプルなフォームが必要です。



14.1。 ストレージAPIのHTMLドキュメント 



<!DOCTYPE html> <html lang="ru"> <head> <title>API -</title> <link rel="stylesheet" href="storage.css"> <script src="storage.js"></script> </head> <body> <section id="formbox"> <form name="form"> <label for="keyword">Keyword: </label><br> <input type="text" name="keyword" id="keyword"><br> <label for="text">Value: </label><br> <textarea name="text" id="text"></textarea><br> <input type="button" id="save" value="Save"> </form> </section> <section id="databox">   </section> </body> </html>




また、ページデザイン用の一連のシンプルなスタイルを作成しました。これにより、フォーム領域がデータを表示するフィールドから視覚的に分離されます。

データ作成



sessionStorageとlocalStorageの両方は、別々のレコードの形式でデータを保存します。 レコードはキーワードと値のペアであり、各値はリポジトリに配置される前に文字列に変換されます。 レコードは、作成、変更、削除できる名前と値を持つ変数として想像できます。



レコードを作成してストレージスペースから取得するために、このAPIでのみ使用される2つの新しいメソッドが使用されます。



  • setItem（キー、値）。 レコードを作成するには、このメソッドを呼び出す必要があります。 メソッドに引数として渡されたキーワードと値でレコードが作成されます。 リポジトリにそのようなキーワードのエントリがすでにある場合、新しい値がそれに割り当てられます。したがって、このメソッドを使用するとデータを変更できます。
  • getItem（キー）。 ストアから値を取得するには、このメソッドを呼び出して、目的のレコードのキーワードを渡す必要があります。 キーワードは、要素がsetItem（）メソッドを使用して作成されたときに宣言されたものと一致する必要があります。




14.3。 データの保存と取得 



 function initiate(){ var button = document.getElementById('save'); button.addEventListener('click', newitem); } function newitem(){ var keyword = document.getElementById('keyword').value; var value = document.getElementById('text').value; sessionStorage.setItem(keyword, value); show(keyword); } function show(keyword){ var databox = document.getElementById('databox'); var value = sessionStorage.getItem(keyword); databox.innerHTML = '<div>' + keyword + ' - ' + value + '</div>'; } addEventListener('load', initiate);




プロセスは非常に簡単です。 これらのメソッドはsessionStorageオブジェクトの一部であり、それらを呼び出すための構文は常に同じです：sessionStorage.method（）。 リスト14.3のコードでは、ユーザーがフォームの[保存]ボタンをクリックするたびにnewitem（）関数が実行されます。 この関数は、エントリを作成し、フォームから情報を追加してから、show（）関数を呼び出します。 この2番目の関数では、getItem（）メソッドとキーワード引数の値を使用してレコードが取得されます。 次に、レコードの内容がデータボックス要素に挿入され、画面に表示されます。

これらのメソッドに加えて、ストレージAPIは、レコードを作成したり、ストレージから取得したりする簡単な方法を提供します。 このメソッドでは、recordStorageオブジェクトをsessionStorageオブジェクトのプロパティとして使用して、レコード値を取得できます。 このメソッドには2つの構文オプションがあります。キーワードを格納する変数を角括弧で囲む（たとえば、sessionStorage [keyword] = value）か、ピリオドを使用してレコードにアクセスする（たとえば、sessionStorage.myitem = value）ことができます。



14.4。 簡単な方法でメモを操作する 



 function initiate(){ var button = document.getElementById('save'); button.addEventListener('click', newitem); } function newitem(){ var keyword = document.getElementById('keyword').value; var value = document.getElementById('text').value; sessionStorage[keyword] = value; show(keyword); } function show(keyword){ var databox = document.getElementById('databox'); var value = sessionStorage[keyword]; databox.innerHTML = '<div>' + keyword + ' - ' + value + '</div>'; } addEventListener('load', initiate);




データ削除



レコードを作成、読み取り、そしてもちろん、ストレージから削除できます。 最後のタスクを実行するには、2つの方法があります。



  • removeItem（キー）。 1つのエントリを削除します。 要素を識別するために、メソッドは、レコードの作成時にsetItem（）メソッドで使用されたキーワードを渡す必要があります。
  • クリア（）。 ストレージスペースをクリーンアップします。 その中のすべてのエントリが削除されます。
