Red HatがCoreOSを引き継ぐ

Red Hat は 、多くのhabrach居住者に馴染み、コンテナ断熱システムおよび関連ソフトウェアの市場で価値ある地位を占めているCoreOSの買収を発表しました。 発表された取引額は2億5000万ドルです。













DockerおよびRed Hat Atomicのソリューションと競合するCoreOSで開発中のプロジェクトの中で、 CoreOS Container Linuxの原子的に更新されたLinux環境（2014年にCoreOSという名前で登場しました）に言及できます。 Container Linuxは、OS上でコンテナーを実行するために必要な最小限のコンポーネントセットのみを持つロジックを導入しました。







さらに、CoreOS製品の中で：







  • Quay Container Image Registry（レジストリの高い信頼性と可用性を確保する機能を備えています）、
  • コンテナー管理ツールrkt （Dockerツールに代わる、より安全でポータブルでサーバーフレンドリーな代替として位置付けられています）、
  • 分散構成ストレージシステムetcd 。分離されたコンテナのサービスの実行に関する構成と情報を格納するための統一されたメカニズムであり、通常の充填が行われます。
  • テクトニックコンテナオーケストレーションプラットフォーム（Kubernetesベース）、
  • Torusと呼ばれるetcdを介して調整される分散ストレージシステム
  • Ignitionシステムの初期構成用のツール（初期ブート段階でシステムを構成するためのCloud-Initと同様のツールを提供）
  • フランネルネットワークコンフィギュレーター 、
  • dex認証サービス、
  • Clairコンテナーの充填における脆弱性を分析するためのシステム（コンテナーの定期的なチェックを可能にする興味深いソリューション。CVEデータベース（一般的な脆弱性と露出）およびRed Hat Enterprise Linux 、 Ubuntu、 Debianディストリビューションの脆弱性修正に関する情報がデータソースとして使用されます。


合計で、CoreOSは90以上のプロジェクトをサポートおよび開発しており、その多くはgithubで公開されています。 ほとんどのCoreOS製品はフリーソフトウェアとして進化してきており、不足しているソースは今後数か月以内に公開される予定です。 トランザクションが完了した後、Red Hat は CoreOSテクノロジーをOpenShiftおよびRed Hat Atomic製品に統合し、CoreOS開発に基づいた多数の新製品をユーザーに提供する予定です。 結合された製品を作成し、eBayやSalesforceを含む既存のCoreOSの顧客を移行する計画は、後日公開されます。







Tectonicプロジェクトは、OpenShiftおよびKubernetesプラットフォームに関連するRed Hat独自の開発と交差し、運用および管理の自動化の分野でOpenShift機能を拡張するために使用される予定です。 Quayレジストリは引き続きスタンドアロンコンポーネントとして使用され、関連する開発を使用してRed Hat Container CatalogとOpenShiftに統合されたレジストリツールを改善する予定です。







Container Linux、およびrktおよびetcdツールキットは、RHELの統合プラットフォーム管理ツールおよびコンテナー用ランタイムであるRed Hat Enterprise Linux Atomic Hostと交差するためです。 Red Hatは、Container Linuxテクノロジーのレビュー、統合、およびAtomicプロジェクトとの統合を行う統合開発チームを作成する予定です。 Container Linux、etcd、およびrktプロジェクトは、 CNCF （Cloud Native Computing Foundation）、 Open Container Initiative （OCI）標準、 Kubernetesプラットフォーム、およびランタイムcontainerdを開発している組織に寄付されました。
