信じられないほどのモバイルアプリストアと数字について

モバイルアプリケーションのオーディエンスサイズを知るために、アプリケーションストアにアップロードして美しいグラフィックを見るだけだと思いますか？ 控えめに言っても、そうではありません。







この記事では、さまざまなアプリケーションストアから提供されるダウンロードデータを分析し、実際のインストール数と比較します。





各アプリケーションストアは、独自のインストール統計情報と、多くの場合追加情報を提供します。 しかし、現代の世界では、これらのデータは十分ではありません。



私たちはいくつかのアプリケーションをリリースしましたが、その後、何か間違ったことをしていることに気付き始めました。 しかし、何が間違っていたかを正確に理解することはかなり困難でした。 さまざまなサイトからのわずかな数のインストールのみが表示され、ユーザーに関する情報は提供されませんでした。



消費者を理解するには、アプリケーションでの消費者の行動について可能な限り多くの情報を取得する必要があります。 このために、さまざまな分析サービスがあります。 Flurryを使用しました。これを使用して、アプリケーションの最初の起動を追跡し、ビルドを受け取ったストアを判別します。 次に、一般的な数値を収集し、インストールの起動係数（ Launch / Install ）を取得します。これにより、この点で最も効果的なストアがわかります。



いくつかの異なるアプリケーションからデータを収集した後、私たちは何を見ましたか？ かなり興味深い写真：







Google Playの場合、状況が驚くことではない場合-アプリケーションをインストールしたすべてのユーザーが少なくとも1回それを起動するわけではないことは理にかなっています-AmazonとOperaの場合、いくつかの異常が観察されます。



ほとんどの場合、Amazon開発者コンソールは実際のインストール数を過小評価しています。これは、1つを除くすべてのアプリケーションで発生しました。 そして、私たちの場合のOperaは、実際の起動数が一貫して少ないことを示しています。



これからどのような結論を導き出すことができますか？

  1. 使用する分析が間違っています
  2. 店舗データが部分的に間違っています
  3. 店舗ユーザーの中にはボットがあります
  4. さまざまな店舗の対象者の詳細は、アプリケーションの起動回数に影響します




Flurryは最大の分析ツールの1つであり、かなり信頼できるものとして確立されているため、最初のオプションは考えられません。 さらに、この場合、異常は再現できません。

2番目のオプションは最も可能性が高く、おそらく3番目のオプションと組み合わせて使用​​されるようです。 しかし、順序ではなく「そのように」アプリケーションをダウンロードするボットも奇妙に見えます。



Operaは一種の「アニメーター」であり、提出されたプロジェクトを多数のその他のストア（ Beeline 、 Blackberry 、 Nokia 、 Yandex.Storeなど）に配置することは注目に値します。 データの一部は、それらとの相互作用のレベルで誤って送信される可能性があります。



オーディエンスに関連する後者のオプションは現実的ではないようですが、Operaユーザーはインストールしたものを忘れてしまい、Amazonファンは空き時間にいくつかのデバイスを購入してテスターをプレイします。



インストールを起動に変換することに加えて、インストールの予想数を考慮する必要があります。 これは、高い変換と少ないインストール数を提供するプラットフォームは、非効率的な変換よりも悪い場合がありますが、インストール数が膨大になるためです。



私たちは実践から平均値を取り、知覚で理解できる合計数千の態度に丸めます。 より多くのインストールがOperaとGoogle Playをもたらします。 これ以上変化するものはないようで、明確なリーダーがいます。







しかし、ここで最近取得した係数を思い出して、インストールのリーダーであるOperaが30％近くも、実際にインストールの数が2番目に多いGoogle Playよりも17％少ないローンチを受け取ることを理解しています。 そして、どちらの場合でもAmazonはまったく考慮されるべきではありません。



これによる結論は簡単にすることができます-アプリケーションストアの数値を完全に信頼し、分析を使用し、ユーザーを知り、アプリケーションに遅れないようにしてください。



将来的には、モバイルゲームの分析と、さまざまなアプリケーションストアの仕様に関連するいくつかの側面についても詳しく説明する予定です。



この問題について意見を表明したり、経験を共有したりすることを決定した場合、私たちは喜んでいます。

さまざまなプラットフォームや市場でのインストールを分析したときに明らかになった異常や機能に遭遇した場合は、コメントを書いてください。
