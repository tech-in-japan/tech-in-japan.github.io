リアルタイムLinuxおよびReaperでの2つ以上のサウンドカードのオーディオミキサー

グループでプレイするときは、どこかでリハーサルする必要があります。 自宅で、手元にあるエフェクトを使って独自のミキサーを組み立ててみましょう。 そして手元には、内蔵の2コアオーディオカード、ESI Juli @およびC-Media CM8738を搭載した2コアコンピューターがあります。 録音用の合計6チャンネル。

2チャンネルで十分な場合、asio4all（またはネイティブドライバー）を使用してWindowsのReaperで2チャンネルを混合できるため、読み込めなくなります。問題なく再生されます。 すべての複雑さは、3枚のカードから1つの仮想マルチチャネルを作成するときに始まります。 asio4allを使用すると、（リハーサルでも）許容レベルの品質で6チャンネルを同時に記録することは機能せず（カードのクロックソースが異なるため、まれにスローダウンが発生するため）、このように困難な方法で進む必要がありました。



リアルタイムカーネルを備えたLinuxですべて実行します。 リアルタイムパッチは望ましいですが、必須ではありませんが、パッチがなければ、より長い遅延とそれほど安定していない場合を除いて機能します。 ディストリビューションはRemixOSではなくDebianになります。まず、何が起きているのか、何かをねじることができても何も壊れないかどうかを理解したい、第二にwineasioにパッチを当てる必要がある、そして第三に、remixosにはひどいデスクトップがあるからです。 Debianのインストール方法については説明しません。このトピックについてはすでに多くのことが書かれています。 したがって、デスクトップにDebian 6.0.3を新たにインストールしました。 デスクトップにはアイコンが表示されないことを除いて、ほとんど次のようになります。



最初に更新します（Debianがnetinstallでインストールされていない場合）。 ルートコンソール（[アプリケーション]-> [アクセサリ]-> [ルートターミナル]） 

#aptitude update && aptitude safe-upgrade


次に、すでにリアルタイムでコンパイルされたpengutronixカーネルをルートコンソールに配置します。 

 #wget -O /etc/apt/sources.list.d/pengutronix.list http://debian.pengutronix.de/debian/pengutronix.list #aptitude update #apt-get -q --allow-unauthenticated install pengutronix-archive-keyring #aptitude update #aptitude install linux-image-2.6-rt-osadl-686


再起動し、unameの表示を確認します。 

 root@debian:/home/vasiliy# uname -a Linux debian 2.6.33.7.2-rt30-1-686 #1 SMP PREEMPT RT Mon Jan 17 14:22:44 UTC 2011 i686 GNU/Linux root@debian:/home/vasiliy#


次に、オプションでupdate-notifierを削除します（または何らかの方法でオフにします）。 

 #aptitude purge update-notifier update-notifier-common


次に、必要なパッケージを配置します。 1つはJACKデーモンです。 オーディオアプリケーションが接続するのは彼にとってです。 そして彼は、ALSAを介してサウンドカードに接続します。 

 #aptitude install jackd2 qjackctl jackeq


Audacity、チェック用 

 #aptitude install audacity


wineasioを収集するパッケージ 

 #aptitude install build-essential libjack-jackd2-dev libwine-dev


そして、Reaperを実行するWine。 Wineはリポジトリから配布することも、rtパッチを使用して自分でコンパイルすることもできます。 wineが必要なストリームにリアルタイムの優先順位を与えることができるように、パッチが必要です。 つまり 通常のワインでは、アプリケーションの実行方法に関係なく、すべてのreaper-aフローはsched_other（すべてのワインがschedtoolを介して実行される場合はすべてsched_fifo）になり、パッチを使用すると、リッパーはどのフローが重要であるかを判断します（どのプロセスサウンド）そして、どのスレッドではないか（これはグラフィカルインターフェイスであり、その他すべてです）。 要するに、スケジューラの観点から見ると、スレッドはクラスと優先度を持つことができます。 クラスは、sched_fifo、sched_rr、sched_other（別名sched_normal）、sched_batch、およびその他のクラスです。 サウンドの面で興味深いものが2つあります。これらは普通（sched_other）とリアルタイム（sched_fifo）です。 sched_fifoクラスを使用して実行されるスレッドは、sched_otherクラスを使用するスレッドによって中断できません。 つまり sched_fifoの誰かが何かをするまで、すべてのsched_otherは失敗します。 ほとんどの場合、sched_fifo-streamsが何か、たとえば機器からの信号を待つのは良いことです。 この場合、サウンドストリーム（sched_fifo）は、着信オーディオバッファーがいっぱいになるまで待機し、バッファーがすばやく処理する準備ができているという信号が到着すると、結果を発信バッファーに入れて再度待機します。 そして、これらの休憩中に、グラフィカルインターフェイスなどが描画されます。

そのため、オプション1、通常のワインを入れます（望ましくありません）： 

 #aptitude install wine schedtool


オプション2、ワイン1.2.3をパッチで収集します（パッケージを収集してインストールする必要がある正しい方法で、これは正しいdebian-wayではないとすぐに言いますが、少し複雑です）： 

 #aptitude build-dep wine #exit $wget http://ibiblio.org/pub/linux/system/emulators/wine/wine-1.2.3.tar.bz2 #    winehq.com $tar xjvf wine-1.2.3.tar.bz2 # ,  $cd wine-1.2.3 $wget http://dl.dropbox.com/u/879835/wine-rt-101107.patch #,  .    http://wiki.cockos.com/wiki/index.php/Installing_and_configuring_Wine $patch -p1 -i wine-rt-101107.patch # $./configure --prefix=/usr && make depend && make # make depend ! $su #make install






Reaper、jackdなどをルートからではなく、ユーザーから実行するため、制限を調整する必要があります。そうしないと、オペレーティングシステムは、それほど多くのソフトウェアを起動できず、多くのメモリを消費し、リアルタイムの優先順位を取得できません。 /etc/security/limits.confを編集します 

 #gedit /etc/security/limits.conf


そしてそこに書きます（当然、vasiliyの代わりにシステムにログインする必要があります） 

 vasiliy - memlock unlimited vasiliy - rtprio 99 vasiliy - nice -20


制限を適用するには、ログインするか再起動する必要があります。 制限付きですべて問題ないことを確認します。

ALSAを選択し、Audacityで録音し、何が機能するかを見て、何が起こったのかを聞きます（ここでは指でジャックに触れただけです）。

興味深いことに、リニア入力に組み込まれたカードでさえ、かなり良い信号を出します。 唯一のものは、それが十分に強力でなければならないということです、そうでなければ、それはノイズを作ります。 IntelのHDAの[録画]タブである[キャプチャ]の音量レベルは、特に25％低いため、ノイズが最小になりました。 たとえば、エレキギターはそのようには書きませんが、同じギターをプリアンプ（私の場合はART Tube MP）に通すと、何も得られず、プログラムの歪みをオンにすることさえできます。



次に、JACKを構成します。 [アプリケーション]-> [オーディオとビデオ]-> [JACKコントロール]（またはqjackctlターミナル）からqjackctlを起動し、[パラメータ]をクリックします。



メインインターフェイスを選択します。ジュリ@があり、hwにあります：2（カードのリストと受け取った番号は、/ proc / asound / cardsにあります） すべてのカードがpciであっても、リブート後にサウンドカード番号が常に保存されるとは限りません。 リアルタイムモードをオンにし、バッファサイズを設定し、離散化し、jackdを実行し、 そして、Audacityを通して、何が機能するかを確認します（Audacityの設定では、JACKに切り替えることを忘れないでください）。 JACKを介して他のカードの録音（および音声出力）を確認することもお勧めします。

そして今、興味深い部分が始まります。 残りのオーディオインターフェイスをメインインターフェイスに接続します。 それでも、qjackctlを実行し、別のターミナルで起動して書き込みます。 

 $alsa_in -d hw:0 -j hda-intel -p 512 -n 2


hw：0は組み込みカード、hda-intelは単なるタグです。これなしでも可能です。512はバッファーのサイズ、2はその番号です。 バッファーのサイズと数は最小限の作業として選択されています（通常の作業はalsa_inが「delay = <anything>」を常に表示しないことを意味します）、私の場合は512（および128）のサンプルと2つのバッファーであり、悪くはありません。 残り（残り）の音は同じです。





すべてがjackeqを介して機能することを確認できます（アプリケーション->オーディオとビデオ-> JackEq、または私のようにjackeqターミナルで）。 入力ポート、出力ポートを選択し、チャンネルの音量、マスター音量を上げて、視聴します。 このステップでは、JACKおよびalsa_inバッファーのサイズを試すことができます（jackeqを再起動する必要があります）。





実際、ある種のシンプルなミキサーがすでに存在しています。 しかし、もっと多くのエフェクト、vstプラグインなどが必要です。 Ardourをインストールしたり、Windows VSTプラグインをサポートしてArdorを再構築したり、Mixbusをインストールしたりできます。 これはすべてDAW（デジタルオーディオワークステーション）であり、それらを介してリハーサルのためにミキシングすることはかなり可能です。 Reaperが好きだったのは、第一に、より馴染みがあり、第二に、優れた組み込みエフェクトがあり、第三に、便利なミキサーがあり、メインミキサーにエフェクトコントロールノブを配置でき、第四に、ミキサーにホットキーを設定できるからですほとんどすべての場合、5番目にVSTで動作します; 6番目に、Reaperは常に進化しているライブソフトウェアです。

reaper.fmを使用してダウンロードしてインストールすると、試用版を使用できます。 Reaper自体は起動するには時期尚早であり、jack-asioの束はありません。 wineasio（http://sourceforge.net/projects/wineasio/）をダウンロード、パッチ、コンパイル、インストールする必要があります。 安定版（0.9.0）をダウンロードするか、gitから開発版をダウンロードできます。 gitにあるバージョンは安定したバージョンとは少し異なりますが、32サンプルのJACKバッファーから開始されているため、ダウンロードする方が適切です。 

 git clone git://wineasio.git.sourceforge.net/gitroot/wineasio/wineasio


wineasioをビルドするには、Steinberg ASIO SDKに含まれているasio.hファイルが必要です。 残念ながら、Steinbergはこのファイルの配布を禁止しているため、SteinbergのWebサイト（http://www.steinberg.net/en/company/developer.html）からダウンロードし、解凍したwineasioのあるフォルダーに入れます。 ここで、asio.cにパッチを適用する必要があります。 

 /*    */ This->jack_input_ports = jack_get_ports(This->jack_client, NULL, NULL, JackPortIsPhysical | JackPortIsOutput); /*     : */ This->jack_input_ports = jack_get_ports(This->jack_client, NULL, NULL, JackPortIsOutput); /*     ,  */ This->jack_output_ports = jack_get_ports(This->jack_client, NULL, NULL, JackPortIsPhysical | JackPortIsInput); /*   */ This->jack_output_ports = jack_get_ports(This->jack_client, NULL, NULL, JackPortIsInput);




このパッチでは、JackPortIsPhysicalフラグを削除して、winesioがalsa_inで作成した仮想ポートを含むすべてのJACKポートにすぐに接続できるようにします。

コンパイル、インストール： 

 $make $su #make install #exit $wine regsvr32 wineasio.dll




Reaperの起動をテストし、wineasioが表示されていることを確認します。



wineasioはレジストリを介して構成されます。 

 $wine regedit




興味深い設定のうち、入力/出力の数のみがここにあります。現実の量にもっと似たものを置くことができます。 バッファーのサイズがJACKを決定するように、Fixed buffersizeを「1」のままにしておくことをお勧めします。

Reaperを再度起動します。パッチを適用したワインが存在するかどうかに応じて、起動コマンドは異なります。 

 $wine 'c:\program files\reaper\reaper.exe' #   ,   sched_other,  0 () $nice -n -20 wine 'c:\program files\reaper\reaper.exe' #    ,     $schedtool -F -p 10 -e wine 'C:\program files\reaper\reaper.exe' #  Reaper   realtime.       ,   . $env WINE_RT=15 WINE_SRV_RT=10 wine 'C:\program files\reaper\reaper.exe' #   , Reaper    ,    .


wineで最適に動作するように設定します。

「環境設定->バッファリング->同期にネイティブイベントを使用する」のチェックを外し、「環境設定->外観-> UIの更新」で「常に遅延」を選択します。

PS：

  • このバージョンのwine 1.0.1では、3番目のGuitar Rigのみを開始し、4番目と5番目はインストールされていません。 リハーサルと3回目で十分ですが。 GuitarRigを使用する場合は、より慎重に行う必要があります。プリセットをすばやく切り替えると、リッパーでクラッシュします。 私はワインの下でより安定して動作するいくつかの選択肢を見つけたいです。

    バージョン1.2.3では、Guitar Rig 5がインストールされており、正常に動作し、クラッシュしません。

  • wineasio 0.9.0では、バッファー256を使用したメインカードの遅延は約20ミリ秒でしたが、さらに約30ミリ秒でした（修正、 egorinskに感謝）。

    しかし、gitバージョンのwineasioおよびそれより小さなサイズでは、メインカード上でバッファサイズが32になり、1.4ミリ秒の遅延が発生し、alsa_inバッファ128、つまり 追加のカードでは6ミリ秒でした。

  • 4枚のカードでも動作します。EMU-0202USBを接続しました。

  • VSTフィードバックサプレッサーが見つかりませんでした。イコライザーを使用してホイッスル周波数を手動で押す必要があります。

  • リアルタイムカーネルにnvidiaドライバーを適切にインストールする方法がわかりません。

  • Reaperには、起動時にストリームを優先しない機能がありますが、再生/録音が開始されるとすぐに機能します。 つまり ミキサーでプロジェクトを作成する場合、それを開いた後、再生/停止を押す必要があります。 ストリームとは何か、クラスと優先順位を確認します 

     ps -emo pid,tid,class,rtprio,ni,pri,psr,pcpu,stat,wchan:14,comm




    reaper.exeには3つのFF（sched_fifo）ストリームがあり、1つは優先度5（これはジャック用に作成されたもので、この優先度はqjackctlで構成できます）、2つ（私は2つのコアがあります）、優先度20、スレッドの処理に影響し、Reaperは優先度THREAD_PRIORITY_HIGHESTで要求しました。





一般に、リアルタイムカーネル、リアルタイムパッチを適用したwine、gitを使用したwineasioバージョンを配置することを怠らないでください。 結果は非常に良いはずです。
