Gentoo Linuxを準備する





多くの人がGentooの多機能性を愛しています。 Portageを使用すると、任意のタスク用のツールを作成できます。 そして、秘書が決してFirefoxをコンパイルしないことを気にしないでください-ソフトウェアをインストールする機会を彼女に与えることを神は禁じています！ 何もする必要のないものをインストールすることで、配布キットよりも優れているものがあります！ 想像してみてください-5〜10分で、必要に応じて必要なソフトウェアのセットが組み込まれたシステムが得られます。



この記事では、Calculate 2.2ユーティリティを使用してGentooベースのディストリビューションのバージョンを作成するプロセスについて説明します。 その結果、CD / DVDまたはUSB-Flashに書き込むためのハイブリッドISOディスクが得られ、Gentooと完全に互換性があり、USB-HDD、USB-Flash、LVMとRAIDをサポートするHDD、ファイルシステムext4、ext3、ext2、reiserfsへのインストールをサポートします、btrfs、xfs、jfs、nilfs2またはfat32。



この記事は主に、すべてのGentoo愛好家、システム管理者、学校や大学のコンピューター教師、およびすべてのLinux愛好家を対象としています。 すべての操作は、コンソールからルートとして実行されます。



1.必要なもの



もちろん、すでにGentooがインストールされているか、Gentooベースのディストリビューションがあり、インターネットにアクセスするように設定されている必要があります。 Calculate Linux Desktop 11.9 KDE 64ビットから実行したすべての操作。



開始するには、calculate-assembleおよびcalculate-builderパッケージをインストールしてください。 Gentooでは、これに対してCalculateオーバーレイを有効にする必要があります。



emerge layman

layman -a calculate

emerge calculate-assemble calculate-builder







次に、構築するシステムを決定します。 基本的な画像の選択は次のとおりです。

  1. Gentoo Stage3-システムの組み立てを開始するための生活賃金。
  2. Calculate Scratch Server（CSS）は、カーネル、ドライバー、ローダー、Portage、Calculateユーティリティーを備えたGentoo Stage3と本質的に同じです。
  3. Linux Scratch（CLS）の計算-CSS + Xorg +ワイヤレス。
  4. KDE、Gnome、またはXFCEデスクトップ（CLD、CLDG、CLDX）またはCalculate Directory Server（CDS）を使用してLinuxデスクトップを計算します。




タスクに応じて、最新のGentoo Stage-archiveまたはCalculate Stage-imageをダウンロードして、/ var / calculate / remote / stagesおよび/ var / calculate / linuxディレクトリにそれぞれ配置します。



cd /var/calculate/remote/stages

wget mirror.yandex.ru/gentoo-distfiles/releases/x86/current-stage3/stage3-i686-20111018.tar.bz2







または



cd /var/calculate/linux

wget mirror.cnet.kz/calculate/CLDX/stages/i686/cldx-20111024-i686.iso







Stage image Calculate LinuxにはPortageツリーが含まれています。 実際、これはCalculateイメージとGentoo Stage4の主な違いの1つです。 Gentooステージからシステムを構築する場合、Portagesを/ var / compute / remote / snapshotsディレクトリにもアップロードする必要があります。



cd /var/calculate/remote/snapshots

wget mirror.yandex.ru/gentoo-distfiles/snapshots/portage-20111024.tar.bz2







ファイルのバージョンは常に更新されていることに注意してください。

この記事では、次の理由により32ビットCLDXを選択します。



  • ディストリビューションにはすでに必要なプログラムのほとんどが含まれています。
  • Gnome / KDEへのバインドなし;
  • XFCEにはいくつかの小さなパッケージが含まれており、必要に応じてすばやく削除できます。




2.組み立てのためにシステムを準備する



アセンブリのためにシステムを準備するには、cl-assembleユーティリティを使用します。 使用可能なプロファイルのいずれかを使用できます。 プロファイルでは、使用するデスクトップを考慮して、Calculate Linuxフラグが選択されます。 CLDはQt / KDE、CLDG for Gtk / Gnome、XFCE for Gtk向けに最適化されています。 CLDXプロファイルは、KDEまたはGnomeを使用したくない場合に理想的です。



「CLDX」をプロファイルとして指定すると、プログラムはその名前をより正確に入力するように求めます。







また、インストールするパーティションを指定する必要があります。 これは、ハードディスクパーティション（10〜15 GB）またはディレクトリのいずれかです。 ディレクトリを使用する場合、古いファイルの削除に追加の時間がかかります。 デフォルトでは、ユーティリティはGentoo Stage3イメージと新鮮なPortageを使用します：







32ビットCLDXイメージを取得することを示します。最新バージョンは、ここからmirror.cnet.kz/calculate/CLDX/stages/i686からダウンロードできます。 これを行うには、プロファイルを正しく指定する必要があり、64ビットシステムで作業している場合は、i686アーキテクチャを使用することを示します。



cl-assemble -p desktop/CLDX/x86/binary -d /dev/sda2 --source=CLDX --march=i686







私のマシンでは、すべての準備に10分もかかりませんでした。







3.システムを更新する



システムを構築するには、cl-makeユーティリティを使用します。 いくつかのパラメーターに注意してください。 Stage3でシステムを構築する場合-システム構築オプション "-m"（または "--make"）を使用し、Calculateイメージからシステムアップグレードオプション "-u"（または "--update"）を使用します。 独自のビデオドライバをコピーするためのオプション「-V」（または「--withvideo」）に注意してください。 インストールせずにコピーすると、GPLライセンスの違反を防ぎますが、ブートプロセス中にシステムにインストールする独自のビデオドライバーを選択することもできます。



cl-make -u -V

4.変更を行います



最後に最も興味深いものになりました。 変更できるもの：



  • パッケージ構成;
  • ビルドフラグ
  • プログラム設定;
  • ユーザーのデスクトップ設定。
  • トピックなど




ご覧のとおり、イメージはディレクトリ/ mnt / calculate-desktop-CLDX-x86-binary /に展開されています。 このパスを使用して変更を加えます。 スクリーンショットは、デプロイされたシステムで、いくつかの構成が行われたことを示しています。 実際、すべての構成テンプレートは、ディレクトリ/ usr / share / calculate / templates / assemble /で表示できます。 設定の競合が発生しないように変更を加えることが重要です。 どこかで独自の変更テンプレートを作成し、どこかで代替ファイルを使用できます。 ここでさらに詳しく考えてみましょう。



4.1。 ビルドフラグ


パッケージアセンブリ設定は、USEフラグによって制御されます。 依存関係をトリミングすることにより、いくつかの利点を得ることができます。



  • プログラムの実行を高速化する
  • システムのセキュリティを高める。
  • 不要な依存症を取り除きます。
  • ディスク容量の要件を減らします。




USE変数の内容を見てみましょう。 これを行うには、組み立てられたシステムで、次のコマンドを実行します。



linux32 chroot /mnt/calculate-desktop-CLDX-x86-binary /bin/bash -c "emerge --info"







linux32ユーティリティ経由でchroot呼び出しを使用したことに注意してください。 64ビットの下から32ビットシステムを構築しています。 それ以外の場合、linux32ユーティリティは必要ありません。



非常に多くのフラグがあります。 これらの詳細については、一般-/usr/portage/profiles/use.descおよび個別-/usr/portage/profiles/use.local.descの説明を参照してください。



すべてのパッケージのフラグを変更するには、make.confファイルがすぐに使用されます。 ビルドプロセス中に、/ etc / make.confファイルの内容が上書きされるため、変更するには、代替パス-/etc/portage/make.confを使用することをお勧めします。



例として、/ mnt / calculate-desktop-CLDX-x86-binary / etc / portage / make.confファイルに次の行を追加して、すべてのパッケージでipv6アドレス指定のサポートを除外しましょう。



USE="-ipv6"







何が起こるか見てみましょう。 今回は、「-u」ではなく「-U」フラグを使用して更新を実行します。 これにより、ポート同期の時間を節約できます。 'cl-make -U -V'を実行します。







ご注意 Xorgサーバーはパッケージの更新時に再構築されたため、最後にプログラムはモジュールを再構築します。 emerge.logファイルを分析して複数のアセンブリを連続して実行すると、プログラムは毎回Xorgサーバードライバーを再構築します。 これを防ぐには、構築中のシステムのディレクトリにあるvar / log / emerge.logファイルを削除します。 システムをイメージにパックすると、このファイルは削除されます。



4.2。 フラグとパッケージマスク


次のパスは、フラグとマスクを変更するために使用されます：etc / portage / package.use、etc / portage / package.keywords、etc / portage / package.mask、etc / portage / package.unmask。 Emerge 2.2パッケージマネージャーは依存関係をマスクできます。



Firefoxブラウザーを例として使用して、パッケージのマスキングを検討してください。 まず、ルートディレクトリを変更します。



linux32 chroot /mnt/calculate-desktop-CLDX-x86-binary

env-update && source /etc/profile







firefoxパッケージの正しい名前を定義し、利用可能なバージョンのリストを見つけます。



eix firefox

* www-client/firefox

Available versions: *3.6.12 3.6.20 ~3.6.21 ~3.6.22 ~7.0.1-r1







パッケージの安定バージョンは3.6.20です。 バージョン7.0.1-r1のマスクを解除します。







設定を更新するには、etc-updateの代わりにdispatch-confユーティリティを使用します。







ユーティリティはマスクとUSEフラグを更新します。 どちらの場合も、「u」を押して変更を受け入れます。







ここで、アセンブリに必要な他のパッケージの名前を確認して書き出すことができます。 最後に、すべての依存関係を確認するには、次を実行します。



emerge -p 1 2 ..







4.3。 パッケージを追加する


依存関係を除くすべてのインストール済みパッケージは、「world」ファイルに書き込まれます。 パッケージをインストールすると、依存ライブラリとともにパッケージがシステムにインストールされます。 パッケージでライブラリが不要になった場合、コマンド「emerge --depclean」でライブラリを削除します。 したがって、追加でインストールしたすべてのパッケージ（依存関係なし）を「world」ファイルに登録することが重要です。 パッケージを示すコマンド「emerge --unmerge」でファイルを再度削除するか、ファイルを編集して「emerge --depclean」を実行して手動で削除できます。



計算ユーティリティは、app-misc / Calculate-metaに基づいたメタパッケージを使用して、依存関係ツリーを構築します。 USEフラグは、依存関係ツリー全体を構築します。 ビルドプロセス中にチェック用のソフトウェアをインストールできるため、パックする前のvar / lib / portage / worldファイルにはこのメタパッケージのみが含まれます。



パッケージをディストリビューションに追加するために、Portageには素晴らしいツールがあります-セット。 セットの助けを借りて、プログラムのテーマ別コレクションを準備できます。 セットは添付ファイルをサポートしますが、これも非常に便利です。 パッケージのリストを編集したり、/ etc / portage / sets / custom @ファイルにあるカスタムセットを使用するには、次の行を入力します。



www-client/firefox







または、「web」などの別の名前で新しいセットを作成し、パッケージを書き込み、「custom」ファイルでリンクを指定します。



@web







したがって、セットにはネストされたセットとパケットの両方を含めることができます。



システム更新中に、セットのプログラムがインストールされます。 セットは、/ var / lib / portage / world_setsにあるワールドファイルに登録されます。



4.4。 パッケージの削除


ワールドファイルをクリアして「emerge --depclean」を実行すると、Stage3イメージが取得されます。 つまり まだ生活賃金があり、それは「システム」と呼ばれています。 コマンド「emerge system」を使用するか、set-「emerge system」を使用して更新できます。 Calculate-Metaメタパッケージは、選択したプロファイルに基づいて依存関係を構築します。 USEフラグcalculate_nowireless、calculate_nonetwork、calculate_noxfceなどを使用して、いくつかの依存関係を除外することができます。 フラグの完全なリストは、/ var / lib / layman / calculate / profiles / desc / calculate.descにあります。 たとえば、ワイヤレスデバイスのサポートをオフにしましょう。 これを行うには、ファイル/etc/portage/make.confを編集して書き込みます：



CALCULATE="nowireless"







5.変更を保存します



firefoxブラウザーを追加し、ワイヤレスデバイスのサポートを削除し、画像を更新しましょう。



cl-make -U -V







firefoxパッケージがインストールされ、不要な依存関係が削除されます。







5.新しい画像を作成する



イメージを作成するには、cl-imageユーティリティを使用します。



cl-image --live iso











「--live」オプションを使用すると、LiveCDまたはUSB-Flashから起動するときに数秒勝つことができます。 最新のCalculate Linux Stageイメージを使用している場合は、このオプションを使用します。



6.組み立て完了



必要に応じて、またはインストールの完了時に、次の方法でアセンブリを中断できます。



cl-make --break











7.まとめ



実験では、Calculate Linux Desktop XFCEディストリビューションを選択し、CLDXバイナリプロファイルを使用しました。 USEフラグ、コマンド 'emerge -uDNa world'を考慮に入れて、すべてのシステムアップデートをさらに実行します。 フラグが完全に一致する場合、パッケージはバイナリリポジトリからインストールされ、相違がある場合、パッケージはマシン上でアセンブルされます。



必要なソフトウェアのセットでイメージを取得しました。 説明した機能のすべてが使用したわけではありません。 たとえば、配布キットでは、ユーザーのデスクトップをカスタマイズするためのテンプレートを準備できます（ habrahabr.ru/blogs/linux/129658を参照）。
