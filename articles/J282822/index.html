Microsoft AzureのWindows Serverフェールオーバークラスター。 データウェアハウス

序文として。 私はすでにTechnetからの記事の翻訳を公開しており、それは非常に頻繁に私の友人に渡さなければなりませんでした。 私はロシア語版の資料へのリンクを提供できるのが好きでした。可能な限り、お気に入りの選択から記事を翻訳し続けます。 本日の記事は、Microsoft AzureクラウドでWindows Serverフェールオーバークラスターのデータウェアハウスを編成する方法に関するマリオルーの記事の翻訳です。



こんにちは、クラスターのファン。 私の名前はマリオルーで、高可用性ソリューションチームのマイクロソフトテクニカルサポートオフィサーです。 良いお知らせがあります-2015年4月以降、MicrosoftはAzureのIaaS仮想マシン用のWindows Serverフェールオーバークラスターのサポートを正式に発表しました。 この発表へのリンクは次のとおりです。



Microsoft Azure仮想マシンのMicrosoftサーバーソフトウェアサポート

https://support.microsoft.com/en-us/kb/2721672



フェールオーバークラスタリングも記載されています。 このリンクの情報は、Windowsクラスターの新しい改善がAzureに導入されると変更される場合があります。 新しいソリューションの展開に備えて、そこで情報が更新されているかどうかを確認してください。



一般的に、クラウド内の仮想マシン内のフェールオーバークラスタリングの機能は変わりません。これは同じ標準OS機能です。 すべての問題は、このマシンが展開されているインフラストラクチャ、つまりネットワークとデータウェアハウスに関連しています。 この記事はデータウェアハウスについてです。







今日は、オンプレミス環境とMicrosoft Azureパブリッククラウドにフェールオーバークラスターを展開する場合の違いについて説明します。 まず、Azureの仮想マシンはWindows Server 2008 R2、Windows Server 2012、またはWindows Server 2012 R2である必要があります。 更新プログラム2854082は、Windows Server 2008 R2および2012にインストールする必要があることに注意してください。



Azureでフェールオーバークラスターを作成する場合の主な問題は、SAN、SAS、またはiSCSIを使用してこれを実行できるローカル環境とは異なり、Azureが複数の仮想マシンに共有データストレージを使用する機能を提供しないことです。 これにより、AzureでのWindowsクラスターの使用が制限され、SQL Server AlwaysOn可用性グループが主なシナリオのままになります。この場合、共通のデータストアではなく、SQL DBMSレベルでの仮想マシン間のデータレプリケーションが使用されるためです。



現時点では、Azureでの共有ストレージの使用に関する制限を部分的に回避する方法に関するいくつかのオプションがあり、それにより利用可能なクラスター使用シナリオのリストを拡大しています。



オプション1：アプリケーションレベルのデータ複製





Azureのクラスタリングは、そのレベルでデータを複製できるアプリケーションに正常に使用できます。 実際、SQL Server AlwaysOn可用性グループが使用するのはこのオプションです。











オプション2：ソフトウェアを使用したデータボリュームの複製





個別のディスク間のデータ複製は、クラスター化されたアプリケーション自体ではなく、サードパーティのソフトウェアによって実行できます。 このようなソフトウェアの主なオプションの1つは、SIOS DataKeeper Cluster Editionです。 これはこの種の唯一のソフトウェアではなく、単なる例として示しています。



DataKeeper Cluster Edition：Windows Server環境のリアルタイムレプリケーション

http://us.sios.com/products/datakeeper-cluster/











オプション3：リモートiSCSIターゲットにExpressRouteを使用する





ExpressRouteは、Azureで利用可能な独自の機能の1つです。 Azureデータセンターとローカルインフラストラクチャの間に専用のデータチャネルを作成できます。 この機能を使用すると、高品質の通信が提供され、iSCSIを介してこのチャネルを介して接続されたドライブの可用性が保証されます。



このオプションを実装する製品の1つは、NetApp Private Storage（NPS）です。



Availability on Demand-NetApp Private Storageを使用したASR

http://channel9.msdn.com/Blogs/Windows-Azure/Availability-on-Demand-ASR-with-NetApp-Private-Storage











ExpressRouteの詳細については、この関数の公式説明を参照してください。



Expressoute

http://azure.microsoft.com/en-us/services/expressroute/



共有データウェアハウスを実装するための新しいオプションが利用可能になると、フォールトトレラントなWindowsクラスターを使用するための新しいシナリオが登場します。 新しい情報が利用可能になり次第、この記事を更新します。 「共有データウェアハウス」を作成したら、クラスターの基盤が整っていると想定できます。
