「ITインフラストラクチャ」：2017年の「クラウド」はどこへ向かうのか

クラウドテクノロジーは勢いを増しており、これはもちろんニュースではありません。 2016年、Googleの上級副社長であるダイアングリーンは、クラウド2.0の次の時代について語りました。 そして今年は、独自の特徴と開発の方向性を持つこの新しい時代の出現を目撃します。 このサイトのブログ で、過去1年間のクラウドテクノロジーについて話しました。 そして今日は、さまざまな専門家の意見を紹介し、最も興味深いトレンドと予測についてお話します。



/ Flickr / JDハンコック / CC-BY



マルチクラウド



HedvigのCEOであり、Apache Cassandraの作成者であるAvinash Lakshmanは、今年、多数の企業がロックインから離れ、マルチクラウドアプローチを適用し始めると考えています。 そのため、クラウドからより多くのメリットを引き出すことができます。 また、たとえば、AWSのみに結び付けられた顧客はますます少なくなります。



他の専門家は、いくつかのプロバイダーとプラットフォームの使用に対する、または単に新しいものへの移行に対する企業の態度の変化を期待しています。 クライアントは、これを簡単かつ自然に行う方法を学び、ビジネスの目標に合わせてプロバイダーを選択することができ、クラウドへの移行に伴う不安が大幅に軽減されます。



今日、企業はマルチクラウドの管理に特別なツールを使用していますが、既存のサービスは主に最大のプロバイダーをサポートしています。 West Gate Networksの社長であるAndrew Froehlichは、この状況の変化を予測しています。



別の問題も解決する必要があります。今日、たとえばVMwareなどの大規模で信頼できる企業のほとんどの製品は、この会社の他のテクノロジーを使用するクラウドインフラストラクチャに主に焦点を当てています。 近い将来、サービスはより普遍的になります。



マルチクラウドは、各企業が非常に個別の独自のニーズと要件を持っているという事実のために、一部人気があります。 しかし、これはまた、特定の業界の問題を解決することを目的とし、適切な分野で必要な技術の開発に貢献できる、より多くの「 業界 」クラウドの出現を引き起こします。



「万能のクラウド」システムは、テクノロジーの開発とクラウドソリューションの普及に大きな刺激を与えましたが、特に大規模なプロバイダーは、業界ごとに異なる顧客のニーズに迅速に対応できません。 したがって、この場合の対象を絞ったサービスの導入が唯一の可能なシナリオです。 IaaSプロバイダーの仕事が今日どのように組織されているかについては、 こちらとこちらをご覧ください 。



別の傾向は、マルチクラウドアプローチに関連しています-個々の独立したプロバイダーによって提供されるクラウドのパフォーマンスを監視するために、サービスとしてのクラウド（CMaaS）を管理する必要性。 そのようなサービスがプロバイダー自身によって提供されないことが重要です：彼らはそのようなサービスに彼らの仕事についてのデータを送信するだけです。 同時に、CMaaSはパブリッククラウドサービスとIaaSおよびPaaSサービスの両方との統合を提供します。



前進する



ガートナーのアナリスト、トーマス・ビットマンは 、企業はすでにハイブリッドクラウドへの切り替えを開始していますが、その割合はまだ小さすぎると述べています。 今日、全企業の約半数がプライベートクラウドのみを使用していますが、2017年末までに、すでに50％の企業がハイブリッドクラウドサービスを使用することになります。



ハイブリッドクラウドの人気が高まると、プロバイダーによるすべての義務の実装を監視し、プラットフォームの動作を監視するクラウドブローカーの出現につながります。 IaaSプロバイダーが今日保証できることの詳細については、 こちらをご覧ください 。 また、ブローカーはクラウドを管理および保護する最良の方法も決定します。 451 Researchのアナリストは、このようなサービスが2017年にクラウドの主なトレンドになると考えており、それらは社内で発生するだけでなく、別の構造としても形成されると考えています。



Forrester Researchによると 、企業はPaaSとクラウド管理の機能を組み合わせた、より経済的で安価なソリューションへの切り替えに関心を持っています。 おそらく、プライベートクラウドはハイパーコンバージドプラットフォーム上に構築されるでしょう。 専門家は、そのようなシステムが間違いなく注目を集めることを期待しています。 ハイパーコンバージェンスはこれまでのところ20％の統合しか提供できませんが、大きな開発の可能性があります（ ソースのポイント2を参照）。



Forresterのアナリストが予測する別の傾向は、クラウドプラットフォームの変化に関連しています。 リフトアンドシフトツールを使用しているお客様には、 移行が簡素化されます。 このようなサービスは今年、例えばパブリッククラウドからプライベートクラウドへ、またはその逆への移行を緩和し促進するため、非常に人気があります。



多くの専門家は、クラウドの作業に関連するサービスの数の増加を予測しています。つまり、理論的にはより多くの人々がデータにアクセスできることを意味します。 したがって、IBMとDellの独立した専門家であるKevin L. Jacksonは、企業がサイバーセキュリティに焦点を合わせると考えています。 これは、技術の標準化と自動化につながります。



セキュリティ対策は強化され続けます。 今年は、多数のセキュリティテストが実施され、安全なアクセスとデータ暗号化の分野における顧客のニーズの調査にも注意が向けられます。 ここでは、セキュリティに対するクライアントのさまざまなアプローチをどのように考慮するかについての経験について話しました。 その結果、企業はより安全に感じることができるようになり、その結果、クラウドテクノロジーの普及がさらに促進されます。



AIおよびデータセンターシステム



クラウドベンダーの優先事項は、機械学習技術と人工知能の開発のままです。 2016年には、3つの最大のプロバイダー（Google、Microsoft、Amazon）がこの分野の製品を発表しました。 そして彼らは止まらない。

たとえば、GoogleはAMD RadeonブランドでGPUを使用して、クラウドサービスLearning Cloud MachineおよびCompute Engineのパフォーマンスを向上させる予定です。 また、Rick's Cloudでは、人工知能システムの導入により、特定のアプリケーションの負荷または作業に合わせてリソースの形成と適応を自動化することができます。



IaaS市場の成長は、必然的にデータセンターの総数の増加につながります。 これは、ベンダーが顧客の増大する需要とニーズに対応しようとしているためです。 データセンターの大規模な建設のもう1つの理由は、地域のデータ保護およびストレージに関する法律の遵守です。 たとえば、2017年には、Googleは10の新しい地域にデータセンターを建設する予定です。 同社は、これによりレイテンシーと応答時間も短縮されることに注目しています。



専門家は、クラウド市場が完全に異なる方向に発展することを確信しているため、大規模なベンダーだけがその地位を強化することができます。 また、地域プロバイダーはクラウド2.0時代を活用し、機動性と顧客要求への迅速な対応を通じて企業と競争します。
