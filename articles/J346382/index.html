突然：マクロ付きのONLYOFFICEの新しいバージョン

重要なニュースがあります。マクロ付きのデスクトップの新しいバージョンがリリースされました。 公式ウェブサイトからダウンロードして、自分ですべてを試すことができます。



この記事では、使用しているマクロ、Microsoftマクロとの違い、およびそれらの操作方法について説明します。







マクロはどのように作成されますか？



マクロは、日常的な操作を自動化し、時間を大幅に節約できるスクリプトです。 Microsoft自身がこの概念を思いついたので、これらのスクリプトはMicrosoft Office向けであり、Windows上で実行されます。



「いつマクロを作成しますか？」という質問は非常に人気があります。 最初は、「決して。 しかし、プラグインでも同じことができます」



そのため、1年の間に、プラグインシステムは非常にクールになりました（さらに詳細に文書化されています ）。 現在、プラグインを使用して新しい機能を追加しています。 たとえば、シンボルテーブル：ドキュメントエディターでは特別なシンボルが必要であり、プラグインを介して追加することは何倍も高速です。 さらに、ドキュメント、スプレッドシート、プレゼンテーションを生成および処理するためのSDKがすでに用意されています-ONLYOFFICE Document Builder。



一般に、マクロに関しては、すべてがうまく機能していることがわかりました。 そして、それはかなりうまくいった。 Document Builderを取得し、プラグインにラップして、インターフェイスに転送します。 マクロを作成して実行できるようになりました！







これで、マクロはBuilderでできることをすべて実行できます。 そのためのドキュメントはここにあります 。 私たちはすでにマクロに関する別個のドキュメントを書いています。



私たちのマクロはマイクロソフトのものとは異なります



プラグインを見たなら、すぐにそれが何であるかを推測する必要があります。 Visual BasicではなくJavaScriptで記述します。



「なぜVisual Basicではありませんが、互換性はどうですか？！」



この質問を予想し、事前に回答してください。 まず、VBAはWindows専用であり、すべてのプラットフォームがここにあります。 Microsoftに何らかのスクリプト言語があれば、喜んでサポートします。 しかし、非常に古い言語のコンパイラを記述するのは100万年前であり、その結果、マクロ（MSと同じ）を取得することは夢のようなタスクではありません。 現時点では、エディターを改善することを好みます。 さらに、jsですべての準備が整いました。



これは素晴らしい理由です。



-すべてのプラットフォームで動作します。

-本当に非常にシンプル。

-スマートスクリプトを記述することにより、プラグインとしてもマクロとしても使用できます。 もちろん、もちろん、ドキュメントビルダーのスクリプトとして。



最後の点について：マクロは実際、プラグインの特殊なケースです。 単純にインストールされたプラグインは、すべてのドキュメントで機能し（つまり、エディターに添付されます）、マクロ-特定のドキュメントでのみ機能します（つまり、ファイルに添付されます）。



では、互換性はどうでしょうか？



はい、Microsoftのマクロは単純に開かれません。 同時に、多くの人が既に代替ライブラリーで実行したい完全なマクロライブラリを持っていることを理解しています。



今のところ、最も簡単なオプションを提供できます。VBAで書かれたマクロを少し書き換えます。 おそらくこれは頭痛の種に聞こえるかもしれませんが、実際にはまったく複雑ではありません。



以下は、いくつかのセルにデータを入力する例です（範囲の合計）。 1つ目はVisual Basicを使用したMSのバリアントで、2つ目は私たちのものです。



最初の例： 



Sub Example() Dim myRange Dim result Dim Run As Long  For Run = 1 To 3 Select Case Run Case 1 result = "=SUM(A1:A100)" Case 2 result = "=SUM(A1:A300)" Case 3 result = "=SUM(A1:A25)" End Select ActiveSheet.range("B" & Run) = result Next Run End Sub




第二： 



 (function() { for (let run = 1; run <= 3; run++) { var result = ""; switch (run) { case 1: result = "=SUM(A1:A100)"; break; case 2: result = "=SUM(A1:A300)"; break; case 3: result = "=SUM(A1:A25)"; break; default: break; } Api.GetActiveSheet().GetRange("B" + run).Value = result; } })();




ご覧のとおり、これは非常に簡単です。
