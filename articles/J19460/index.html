-そして、象はより長い幹を持っています。

「しかし、キリンの首はより急です。」

-この場合、これは重要ではありません！

たぶんそれはそのような戦いでは理にかなっていますが、私はそれを見つけませんでした。



すべてのソフトウェア開発者（幅広いユーザー向け）は、当然、システムの人気を目指して努力しています。 開発者の影響下であろうとなかろうと、特定のイデオロギーがしばしば発生します。 ユーザーを製品に結び付ける優れた方法は、ユーザーにイデオロギーを課すことです。 たとえば、Habrahabrはイデオロギーだけでなく、サブカルチャー（さまざまな「habraslovs」、「UFO」など）の類似性も課しています。 したがって、いわゆる「habrozavisimost」、「habro-breaking」などが表示されます。



イデオロギーに従うことは人を客観性から遠ざけるように思えます。 自分が人間であることを常に覚えておいてください。 これが主なものです。 使用しているOSまたは使用しているプレーヤーは、二次的なものであり、通常は一時的なものです。 まず、オペレーティングシステムはツールです。 ハンマーやaと同じツールですが、はるかに多様です。 各オペレーティングシステムには、独自の欠点と利点があります。 優れたインターフェースを持つものもあれば、潜在能力を隠すものもあります。 しかし、いずれにせよ、それらはツールのままなので、誰もが彼にとってより便利なものを選択します。それは解決されるタスクにより適したものです。



特定の製品を使用しているという事実は、それが最良であることを意味するものではありません。



私は、そのような紹介から第5条を開始することが重要であると判断しました。 これはLinuxの開発にはあまり役に立たないかもしれませんが、人生では役に立つように思えます。



重要なディレクトリとファイル



Ubuntuには多くの重要なファイルとフォルダーがあるため、1つの記事でそれらを考慮することはほとんどありません。そのため、私にとって最も重要だと思われるものを説明します。



/は、ファイルシステムのルートです。 このフォルダーは、システムの他のすべてのファイルおよびフォルダーの親（祖先、祖先、...）です。 Linuxで、単一のルートを持つ中央集中型ファイルシステム。 これは、各ディスクがシステムの個別のルートを表すWindowsとは大きく異なります。



〜-これはホームディレクトリ、またはそれの短い記録です。 完全なエントリの形式は、/ home /（ユーザー名）です。 このディレクトリには、個人用ドキュメント、個人用アプリケーション、特定のユーザー向けの多くのアプリケーションの構成が保存されます。



/ tmp-操作中にアプリケーションが一時データを保存する一時ファイルのフォルダー。 アクセス権を変更しないでください。変更すると、アプリケーションが正しく動作しない場合があります。



/ bin / sbin / usr / bin〜/ binは、アプリケーションが実行可能ファイルまたは実行可能ファイルを実行するためのスクリプトを保存するディレクトリです。 Windows実行可能ファイルは、主にEXE、COM、およびBATです。 Linuxでは、拡張子は重要ではありません。オペレーティングシステム自体がファイルのタイプを決定します。 したがって、Linuxでは、実行可能ファイルはバイナリファイルとさまざまな言語のコードを含むファイルの両方です。 これについては後で説明します。



/ media-外部メディアと非Unixファイルシステムのパーティションがこのフォルダーにマウントされます。 各ディスクは、このディレクトリ内の「フォルダ」になります。



/ var / log-ログ（ログファイル）はこのフォルダーとそのサブフォルダーに保存され、これを使用してさまざまな問題を診断できます。



/ etc-多くのアプリケーションの構成は、コンピューター全体のレベルでここに保存されます。



/etc/init.d-デーモンを起動/停止するためのスクリプトはここに保存されます（Windowsでは、「サービス」という単語が使用されました）。



/etc/rc.local-このスクリプトは起動時に実行されます。 たとえば、ここでインターネット接続コマンドを作成できます。



/etc/X11/xorg.confはX-Windowシステムの構成ファイルです（このシステムはLinuxウィンドウのレンダリングを制御します）。 ここでは、入力デバイス（キーボード、マウス）および出力（モニター）のさまざまなパラメーターを構成できます。
