2-3台のHDDがありますか？ Windowsおよびプログラムの読み込みと反応を高速化

起動時にプログラムのセット全体を搭載したコンピューターが4分間ロードを開始したとき、すべてを高速化する方法を考えました。

Windows XPとAster拡張機能があり、コンピューターで一緒に作業できるため、プログラムとサービスの一部が重複して読み込まれました。



3台目のハードドライブがあり、ハードドライブ上のパーティションをより便利なオプションに再配布するのに十分なスペースができたら、最初の3つのパーティションを異なるハードドライブに準備し、それらにシステムフォルダーを分散させることにしました。



3台のハードドライブを搭載したWindows XPの高速化方法：


1.「ドキュメントと設定」および「プログラムファイル」フォルダーを他のハードドライブの最初のセクションに転送します。 したがって、メインフォルダーからのブートディスクには、「Windows」フォルダーのみがあります。

2.「Windows」ディスク上の転送されたフォルダーへのファイルシステムリンク（ジャンクションリンク）を作成します。 詳細は以下になります。



Windows、Documents and Settings、およびProgram Filesフォルダが異なるハードドライブにあることがわかりました。



利点：
  • これらのフォルダーの独立した作業は、ダウンロード速度に大きく影響します。

    たとえば、ソフトウェアのリストを変更せずに、コンピューターのダウンロード速度を4分から2分に半分に減らしました。
  • デスクトップ上のアイコンを再描画するような効果は、Windows XPのすべてのユーザーになじみがあると思いますが、ほぼ完全に消えました。 つまり これは発生しないか、目立ちません。
  • 起動後のシステムも、アクションに対してはるかに速く応答し始めます。
  • これらのセクションを同時にまたは選択的にデフラグできます
  • WindowsおよびProgram Filesフォルダーは、通常、ディスクの断片化がはるかに少ないため、最適化する必要はほとんどありません。 結果は同じです-より高速。
  • リンクされたフォルダーフォルダーで同時にまたは選択的にウイルスをチェックできます。
  • 「ドキュメントと設定」フォルダーと「プログラムファイル」フォルダーのサイズを取得するのは簡単です。ディスクサイズから空きディスク領域のサイズを差し引くだけです。


短所：
  • すべてのアンチウイルスが正常に機能するわけではありません。



    Kaspersky 2010のみが、リンクフォルダーにインストールして実行を禁止する試みを定義しました。 その後、ドライブCに直接配置され、正常に機能しました。 これは、最終的にカスペルスキー製品に対する私の否定的な態度を変えたバージョンでした。 この状況での正しい作業に加えて、私は最終的に彼らの製品が迅速かつうまく機能し始めたのを見たからです。 ICQ経由のフィッシングリンクをブロックするまで。



    残りはリンクされた「プログラムファイル」に置かれ、何かが正しく機能しませんでした。 たとえば、EsetとDrwebはサービスを開始しませんでした。



    Esetを使用することはまったく不可能でした。Windowsディスクに直接インストールされていても、システムが「ハングアップ」しました。 つまり ファイルシステムリンクでのパフォーマンスが非常に低い。



    Drwebはもう少し優れています。 ドライブCにインストールした後、比較的正しく機能しましたが、そのインターフェイスは、私の意見では、非常に使いにくく、便利ではないため、長続きしませんでした。



    リンクされた「プログラムファイル」にインストールした場合、Comodoは正常に動作しませんでしたが、Windowsディスクにインストールした後は非常に正しく動作します。



    つまり この構成では、KasperskyおよびKomodoアンチウイルスを使用できます。 他の人をチェックしませんでした。 Eset製品を数年間使用しましたが、今でもComodoを使用しています。



  • ハードドライブの1つを単純に無効にする方法はありません。 今、彼らはたくさんあり、グループで働いています。 「ドキュメントと設定」フォルダーと「プログラムファイル」フォルダーでディスクを物理的に切断すると、Windowsは非常に「通常」ロードされ、作業ができることがわかりました。


昨年の夏、Windows XPをシャベルでシャベルしました。そのような構成で6か月間使用しましたが、ウイルス対策の問題を除いて特別なことはありませんでした。



Windows XPですべてを行う方法


あなたの行動の結果が重要な情報の損失になるかもしれないことを注意して理解してください。 そのため、それを受け取る前に、それを処理できるかどうかを考えてください。

  • ハードドライブにパーティションを準備します（ディスクの先頭が望ましい）。
  • Mark RussinovichからコマンドラインJunctionのユーティリティをダウンロードします。
  • WindowsPE、ERD Commander、または別のWindowsインストールなどの外部システムから起動します。
  • 「ドキュメントと設定」フォルダーと「プログラムファイル」フォルダーを目的のセクションに移動します。 さらに、これらのセクションのフォルダーの名前は任意です。
  • コンソール（cmd）を起動し、Junctionのあるフォルダーに移動して、次のようにします。


Program FilesフォルダーをWinXP Program Filesにリンクしました。これは、Windows 7 x64を並行して使用しているため、新しい年に切り替えました。 3つの要因が一致したため-4ギガバイトのRAMになり、AsterはWindows 7 x64で獲得し、新しい年が来ました。



Windows VistaおよびWindows 7ですべてを行う方法


同様に、一部の機能を除きます。



Windows 7では、Program FilesおよびProgram Files（x68）フォルダーを別のセクションにリンクすることで同じトリックを行いました。



しかし、彼はまだ内部フォルダーへのリンクでいっぱいであり、そこへのドキュメントと設定のリンクがあるので、ユーザーフォルダーでまだこれを行っていません。 後でスクリプトを使用して、ユーザーフォルダーを転送するオプションを検討します。これにより、多くのリンクを手動で再作成する必要がなくなります。

「ProgramData」フォルダを転送することもお勧めします。



Windows 7では、Program FilesおよびProgram Files（x86）フォルダーを単に削除することはできませんでした。 したがって、新しいセクションにコピーした後、古いセクションでは、リンクの作成を妨げないように、最初に名前を変更しました。

その後、彼はリンクを作成し、稼働中のシステムを起動しました。

その後、フォルダのプロパティで、彼は所有者を変更し、アカウントにしました。 これにより、ほぼすべてのファイルを削除できました。 残りのファイルは名前を変更でき、再起動後に削除できます。
