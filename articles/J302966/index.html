Intel Media SDK 2016 R2-新機能

メディアコンテンツのエンコードと再生のためのソフトウェアを開発するための一連のツールの新しいバージョンIntel Media SDK 2016がリリースされました 。 この更新には、いくつかの重要な変更が含まれています。



  • 第7世代Intel Coreプロセッサー（Kabylake）のサポートを追加。
  • Media RAW Acceleratorのパフォーマンスが向上し、柔軟性とパフォーマンスが向上しました。
  • ビデオ会議およびクラウドゲームサービス用にAVC / H.264をエンコードする際の新機能を追加。
  • 新しいVPPフィルターが追加され、既存のものが改善されました。
  • APIの新しいバージョンが導入され、メモリ管理とプラットフォーム要求機能が改善されました。
  • Windows Redstone Previewのサポートを実装しました。


カットの下には、Intel Media SDK 2016の現在の機能の簡単な概要があります。



HEVCおよびVP9の完全なハードウェアアクセラレーション 。

  • 10ビットHEVCエンコーダーおよびデコーダーをサポート
  • 8ビットおよび10ビットVP9デコーダーのサポート


AVCエンコーダー ：スライスサイズの制限とレポート機能を追加して、RTPプロトコルを使用するときに必要な低遅延を確保しました。 開発者はこの機能を使用して、ビデオストリームと通信チャネルの帯域幅をよりよく一致させることができます。



Media RAW Accelerator ：パフォーマンスを向上させるために、Media RAW Acceleratorはグラフィックスドライバーと共に配布されるため、現在アプリケーションに含める必要はありません。 16ビットARGB入力のサポートが追加されました。ガンマ補正、彩度収差、3DLUT、レジオメトリ補正フィルターを適用できます。 これで、アプリケーションの開発者は、図に示すように、デベイヤーを適用したイメージにフィルターを簡単に適用したり、処理パイプラインに追加することで独自の後処理ブロックを構成したりできます。







VPP拡張機能 ：開発者は、bt.601またはbt.709の色変換マトリックス、Videosignal Info関数でさまざまな範囲のビデオを使用できます。 第6世代のIntel Core（Skylake）プロセッサーから利用可能なScaler＆Format Converter（SFC）パイプラインを使用すると、Media SDK操作中にワークロードがレンダリングツールから削除され、ロックされなくなったときに新しいスケーリングモードを使用できます。 したがって、開発者は、処理中に他のロードのGPU実行ユニットをより正確に制御できます。



APIでの作業を容易にするための改善 ：プラットフォーム要求機能がAPI 1.19に追加されました。 新しいAPIを使用して、アプリケーションは実行するハードウェアプラットフォームを決定できるようになりました。 メモリパフォーマンスを向上させるために、その割り当ては3DおよびOpenCL構成用に最適化されています。
