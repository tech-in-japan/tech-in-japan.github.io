Cのクラスメソッドと通信します





よく知っているobjc_msgSend（）関数は、最初に呼び出されたメソッドを実装する関数を探しているため、この関数を見つけて自分で呼び出すことができます。



メソッド呼び出しをもう少し詳しく知ることができる小さなテストプログラムを書いてみましょう。 



 #import <Foundation/Foundation.h> #import <objc/runtime.h> @interface TestClass : NSObject - (void)someMethod; - (void)callSomeMethod; - (void)methodWithParam:(const char *)param; @end @implementation TestClass - (void)someMethod { NSLog(@"Hello from %p.%s!", self, _cmd); } - (void)callSomeMethod { NSLog(@"Hello from %p.%s!", self, _cmd); [self someMethod]; } - (void)methodWithParam:(const char *)param { NSLog(@"Hello from %p.%s! My parameter is: <%s>", self, _cmd, param); } @end int main(int argc, const char * argv[]) { TestClass * myObj = [[TestClass alloc] init]; [myObj someMethod]; [myObj callSomeMethod]; [myObj methodWithParam:"I'm a parameter"]; return 0; }




ドキュメントから、目的の関数を呼び出すときに、objc_msgSend（）が次の順序でパラメーターを渡すことがわかります。



  1. メソッドを呼び出したオブジェクトへのポインター
  2. メソッドを呼び出したセレクター
  3. メソッドに渡した残りの引数




これが、テストプログラムが次のように見える理由です。各メソッドで、ログselfと_cmdに出力します。これらには、それぞれ「self」とセレクターへのポインターがあります。

Objective-Cランタイムのメッセージングエンジン





Objective-Cランタイムにメッセージエンジンを実装する鍵は、コンパイラがクラスとオブジェクトをどのように表現するかです。



C ++言語で表現した場合、RAM内のオブジェクトは、クラスの各インスタンスだけでなく、各クラスに対しても作成されます。 つまり、基本クラスNSObjectを継承するクラスを記述し、このクラスの2つのインスタンスを作成することにより、実行時に作成した2つのオブジェクトとクラスの 1つのオブジェクトを取得します 。



このクラスのまさにオブジェクトには、親クラスのオブジェクトへのポインターと、ディスパッチテーブルと呼ばれるセレクターと関数アドレスの対応表が含まれています。 objc_msgSend（）関数は、渡されたセレクターのために呼び出す必要がある目的の関数を検索するのに、このテーブルを使用します。



NSObjectまたはNSProxyを継承する各クラスにはisaフィールドがあり、これはクラスオブジェクトへのポインターとまったく同じです。 オブジェクトのメソッドを呼び出すと、objc_msgSend（）関数はクラスオブジェクトへのisaポインターに従い、このメソッドを実装する関数のアドレスを探します。 そのような関数が見つからない場合は、親オブジェクトのクラスのオブジェクトに移動し、そこでその関数を探します。 これは、目的の機能が見つかるまで発生します。 NSObjectクラスのオブジェクトを含む関数がどこにも見つからなかった場合、既知の例外がスローされます。
