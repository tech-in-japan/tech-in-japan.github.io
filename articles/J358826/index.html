グループ



機能の基本的な考え方は単純です。グループ内の複数のグラフィック要素を結合し、グループを新しいシグネチャを持つ1つのグラフィック要素に折りたたんで展開する機会を与えることです。 ユースケースは、例えばそのようなものであり得る。 開発者は、他の人が書いた新しいコードを使用して作業を開始します。 分析の過程で、徐々に理解が深まり、これにより、たとえば10個のブロックで構成されるコードが作成されます。 次に、開発者はこれらのブロックをグループにまとめ、適切なテキストを提供します。 さらに分析すると、グループは画面上で最小化されたままになります。つまり、ダイアグラム上では、10個の要素ではなく、1つしか存在しないため、その後の分析が容易になります。 もちろん、グループはネストすることができ、モジュール内のグループの数に制限はありません。



初期開発時にグループの使用を禁止する人はいません。 プログラマーはグループを作成できます。これにより、後で一緒に作業する人のコード分析が容易になります。



グループは無制限のネストをサポートしているため、コードに3番目の次元の類似性を導入すると見なすことができます。 最初の2つは図で、3つ目は折りたたまれたグループで、内部を見ることができます。



グループのCML



チャートにグループを描くには、Codimensionがどこから始まり、どこで終わるかを知る必要があります。 コードに適切なマークを残すために、Codimension Markup Languageのコメントでマイクロマークアップ言語が使用されます。 



# cml 1 gb id=101 title="Module imports" import sys import logging import json # cml 1 ge id=101


最初のコメントにはgbコード（group beginという単語から）があり、グループの最初の要素の前にあります。 2番目のコメントにはgeコード（group endという単語から）があり、グループの最後の要素の後にあります。 Codimensionは、コメントが必須のid属性の値とペアになっていることを理解します。これは任意の文字列にすることができます。



明らかに、グループの形成には多くの制限があります。 グループには、1つの入力と1つの出力が必要です。 チャート上のグループの要素を選択する場合、すべての要素を隙間なく選択する必要があります。 if条件が選択されている場合、すべてのブランチも選択する必要があります。 ループの場合、同様の要件がelseコンストラクトに関連付けられています。elseコンストラクトが存在する場合、両方の部分を選択し、ループ自体とelseブロックを選択する必要があります。 同様に、try-except-else-finallyのコンポーネントを分解することはできません。 ユーザーインターフェイスでグループ化が許可されている場合、これらすべての制限が考慮されます。



さらに、グループの作業の一環として、Pythonパーサーの変更が必要でした。 技術の初期設計では、最終コメントを支持しないことが決定されました。 ただし、グループの場合、このような結論のコメントは絶対に必要です。 たとえば、グループはifブランチのいずれかのすべての要素を結合できます。 



 item = self.__processes[index] if item.widget: # cml 1 gb id=102 title="Script started" msg = item.widget.appendIDEMessage('Script started') item.procWrapper.startTime = msg.timestamp # cml 1 ge id=102


ここで、グループを終了するコメントはifブランチに属している必要があります。これは以前に提供されていません。



グループの場合、デフォルトの色を変更する機能も追加されました。 これを行うために、冒頭のcmlコメントで、背景、インク、およびストロークの色を設定するためのオプションの属性がサポートされています。



グループのグラフィックス



グループをレンダリングするには、折りたたまれたグループ、展開されたグループ、空のグループの3種類のプリミティブが必要です。



折り畳まれたグループの場合、そのようなプリミティブは、いくつかの要素がその背後に隠されていることがすぐに明らかになり、明らかにできることが望ましいです。 また、プリミティブのレンダリングは、選択されているかどうかに関係なく単純であることが望ましいです。 その結果、ダブルストロークの長方形が選択され、その中にグループヘッダーが配置されました。









崩壊したグループ



現時点では、選択したプリミティブに完全に満足することはできないため、将来、他のプリミティブに置き換えられる可能性があります。



開示されたグループはその境界を示す必要があり、さらにグループの見出しをすばやく確認できる必要があります。 境界線を「見やすく」するために、開いたグループの長方形が破線で描かれます。 開いたグループのフィールドを広くすることに特別な意味はないので、点線は別の要素である場合よりもプリミティブの近くに描かれます。









開示グループ



開いたグループの左上隅に、その「敏感な点」が描かれます。 マウスカーソルをその上に置くと、グループのアウトラインが実線で描画され、そのタイトルがグループの上に表示されます。









開示グループの「センシティブポイント」



必要に応じて、グループヘッダーに加えて、いくつかのコントロールボタンを追加できます。 これにより、最小限のマウスクリックでアクションを実行できます。



空のグループには何もありませんが、たとえば、見出しを含めることができます。 将来、グラフィカル表現の編集機能が表示されたら、新しい要素を空のグループに追加できます。 したがって、空のグループを描画する必要があります。 ストロークのある長方形が選択されます。









空のグループ



空のグループと折りたたまれたグループのプリミティブ間の類似性は明らかです。 したがって、折りたたまれたグループのプリミティブが変更された場合、空のグループに対して対応する変更を行う必要があります。



グループ管理は、コンテキストメニューから実行します。 グループを作成するには、必要なプリミティブを選択し、コンテキストメニューからグループ項目を選択します...グループヘッダーを入力すると、必要なCMLコメントがコードに追加され、チャートがすぐに再描画されます。 この場合、新しく作成されたグループは最小化された形式で描画されます。



グループを使用する他のすべての操作（開く、非表示、削除、タイトルの編集、色の変更）は、グループコンテキストメニューからの選択によって実行されます。
