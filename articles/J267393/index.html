アプリケーションとデスクトップの仮想化：古い問題の新しい見方

ご存知のように、私たちを襲った経済危機の状況では、最も強いものではなく、最も適したものが生き残り、さらに、サプライヤーはより多くの譲歩をする意思があり、顧客はさまざまな節約を探しています（そしてほとんどの場合成功裏に見つけます）。 アプリケーションとデスクトップの仮想化の分野も例外ではないため、この分野の主要なソリューションの長所と短所を今再検討したいと思います。





Citrix XenDesktop

Citrix XenDesktopは、アプリケーション仮想化の長年のリーダーであるXenAppの後継製品です。 一方では、Citrix XenDesktop \ XenApp 7.6の新しいバージョンは異なるアーキテクチャを持ちます-ワークステーションおよびMachine Creation Servicesサーバーの仮想デスクトップのイメージのための統合複製システムにより、XenApp 6.5よりも優れています（以前のProvisioning Servicesに加えて-イメージ配信仮想および物理サーバー用のネットワーク経由）。 一方、古いXenApp 6.5プラットフォームの一部の機能は現在利用できません。 たとえば、スケジュールに基づいてターミナルサーバーをオーバーロードし、コンテンツを公開します。 アプリケーションの公開は、6.5ほど柔軟ではありません。 現時点では、同社はこれらのギャップを埋めるために多くの努力を払っています。特にバージョン7.6では、セッションの事前起動とセッションの残時間が返されました。 Session Recording（別名Smart Auditor）はFeature Pack 1で返されましたが、VDIセッションを記録する機能はまだありません（ターミナルサーバーセッションのみ）。



Citrixは、拡張機能を有料で提供する追加の製品で拡大しています。 これらは、NetScaler（負荷分散、アプリケーションファイアウォール、Webアクセラレーション、セキュアアクセス）およびCloudBridge-WAN最適化（トラフィック圧縮、帯域幅の効率的な使用）です。



VMware Horizo​​n

VMWare Horizo​​nは最初はVDIソリューションであり、その主な利点はハイパーバイザーとの完全な統合によって達成されます。 したがって、VMWare Horizo​​n 6.2のすべての利点を得るには、vSphere 6.0および関連するアップデートをインストールする必要があります。 ターミナルファームからアプリケーションを公開する分野では、Citrixからのバックログを削減するために多くの努力を払っています。 たとえば、バージョン6.0ではアプリケーションの公開が、バージョン6.1ではUSBストレージデバイスのリダイレクト、バージョン6.2ではセッションへのクライアントデバイスドライブのリダイレクトとスマートロードバランシングが登場しました（ただし、ファームの各ターミナルサーバーでスクリプトを手動で構成する必要があります ）。



同時に、VMWare Horizo​​nにはVMware vSphere DesktopとvCenter Desktopが含まれており 、これらはVDIソリューションにのみ使用されます（つまり、サーバー仮想化のために追加コストを回避することはできません）。 さらに、ハイパーバイザーが既にある場合は、ハイパーバイザーから移行するか、ハイパーバイザーの隣に2番目のハイパーバイザーを実装する必要があります。これには、スタッフへの追加投資と管理コストの倍増が必要です。



どちらのソリューションにも3つのエディションがあります。

  • VMWare Horizo​​n-標準、高度、エンタープライズ。
  • Citrix XenDesktop-VDI、Enterprise、Platinum。


さまざまなライセンスモデル：

  • VMWare Horizo​​n-指定ユーザー、CCU


  1. 指定ユーザー-Viewインフラストラクチャに接続する一意のユーザーを考慮します。 ユーザーには、任意の数のデバイスから任意の数のVDI、RDS、またはアプリケーションデスクトップを実行する権利があります。
  2. 同時ユーザー-シングルユーザーデスクトップへの接続数が考慮されます。 競争力のあるユーザーが複数のシングルユーザーデスクトップを同時に起動した場合、各デスクトップのセッションは個別に考慮されます。 同時に、競争力のあるユーザーは、任意の数のデバイスから1つのライセンスで任意の数のRDSデスクトップまたはアプリケーションを実行する権利を持っています。


  • Citrix XenDesktop-ユーザー/デバイスごと、CCU


  1. ユーザー/デバイスごとは、変換可能なライセンスの一種です。 デフォルトライセンスは90日間ユーザーに関連付けられ、その後ユーザーは任意の数のデバイスから任意の数のVDI、RDS、またはアプリケーションデスクトップを実行する権利を持ちます。 複数のユーザーが同じデバイスからセッションを開始した場合、ユーザーごとの複数のライセンスが1つのデバイスライセンスに変わり、残りから1を引いたものがプールに返されます。
  2. 同時ユーザー-任意のデバイスから任意の数のVDI、RDS、またはアプリケーションデスクトップに接続するユーザーが考慮され、すべてのセッションが完了するか、切断状態に切り替わるとすぐに、ライセンスが解放されてプールに返されます。

選択の難しさ

多くのエディションとライセンスモデルにより、すでに困難なサプライヤが選択することは困難です。 さらに、いずれかのエディションのすべての機能が常に必要なわけではありません。 多くの場合、1つの機能のために、より高価なエディションを選択する必要があります。 同時に、メーカーは多くの場合、機能の比較のマトリックスを作成して、コンポーネントが価格に含まれる時期と追加料金が必要な時期がすぐにはわからないようにします。 たとえば、Citrixのこのマトリックスでは、NetScaler Gatewayアプライアンスが個別に支払われることがSmart AccessおよびSSL VPNに対して示されています。 ただし、NetScalerの負荷分散（オプション）およびNetScaler HDX Insightとの統合の場合、この情報は存在しません。 また、HDX Framehawk Technologyでは、 特定のバージョンのNetScalerが外部から機能する必要があるという事実は、その実装ガイドにのみ記載されています。



したがって、原則として、ソリューションのすべての可能性を100％実現して使用することは不可能です。 さらに、機能が多ければ多いほど実装が難しくなり、資格のある人材が必要になります。 一部のクライアントは、このような複雑なソリューションを自分で実装することはできず、予算のためにコンサルティングを招待できないか、このアイテムにもお金をかけたくありません。 したがって-決定に対する不満と投資の損失。



Parallels Remote Application Server

Parallels RASでできること まず、Microsoft RDSおよびVDIの機能を拡張し、Microsoft Hyper-V、VMWare vSphere、Citrix XenServerなどの主要なハイパーバイザーをサポートする包括的なソリューションです。 これは、ソリューションコンポーネントが管理コンソールから直接リモートでインストールされる単純な展開です。



次に、1つのターミナルサーバーからアプリケーションとデスクトップを公開できますが、Windows 2012R2では2つのセッションコレクション（少なくとも2つのサーバー）が必要です。 ドラッグアンドドロップまたは公開ウィザードを使用してアプリケーションを公開すると、シームレスモードでアプリケーションやデスクトップだけでなくコンテンツも公開できます。



Parallels RASは、リソースを1つの論理ユニット、つまり単一のライセンスサーバーを持つファームに結合します。 リソースが地理的に分散している場合、ファームでは、制御を委任できるサイトを使用できます。 Microsoft RDS（ホストの重量とセッション数）のバランスに加えて、高度な負荷分散（CPU、メモリ、セッション数）、およびそのまま使用できるユニバーサル印刷およびスキャン機能が使用されます（Windows 2012R2のTWAINには追加の構成が必要です）。

ソリューションに高可用性が組み込まれています。サイト内の最初のサーバー（マスター）が何らかの理由で故障した場合、次のサーバーがその機能（アプリケーションのバランスとリスト）を引き継ぎます。



Parallels Remote Application Serverソリューションには、SSLへのRDPトラフィックのトンネリングを可能にするSecure Client Gatewayソフトウェアゲートウェイ、およびSecure Client Gateway間の負荷分散を可能にする高可用性および負荷分散仮想デバイス（HALB）（2つのモードで動作-パス-thoughおよびSSLオフロード）。



HTML5と互換性のあるブラウザを介して、クライアントソフトウェアなしで公開リソースへのアクセスを提供できます。



クライアントデバイス管理機能を使用すると、Windowsベースのクライアントデバイスをシンクライアントに変えたり、独自のシャドウイングソリューションを使用してユーザーのワークステーションをリモートで制御したりできます。



さらに、11の言語、Windows、Linux、Mac、Chromebook、iOS、Android、Windows Phone、Raspberry Piのネイティブクライアント、管理者監査（設定監査）、監視およびレポート機能をサポートするインターフェイス。



この単一の標準バージョンには、柔軟なライセンス体系があります。

  • SPLAは、無制限の数のアプリケーションとデスクトップを起動するためのユーザーと彼のデバイスの1つの競争ライセンスです（ユーザーに2つのデバイスがある場合、同時使用には2つのライセンスが必要になります）。サブスクリプションの有効期間は1、3、6、または12か月に制限されています。 前払い方式のサービスプロバイダー向けに設計されています。
  • 永久CCUは、永久ユーザーライセンスであり、無制限の数のアプリケーションとデスクトップを起動するための1つのデバイスです（ユーザーが同時に2つのデバイスを使用する場合、2つのライセンスが必要です）。 次のバージョンにアップグレードする機能は、購入後1年間含まれます。サポートは別途購入してください。
