Total Commander 7.5：新機能

「新機能」は、このファイルマネージャーの次のバージョンの後に発行されたロシア語コミュニティのTotal Commanderの伝統的な記事です。 今回、私はhabra-peopleの記事に初めて慣れました。


TCの新しいバージョンは最初、最も物議を醸す感覚を引き起こしました。一方で、変更は非常にグローバルであり、他方では、グローバルは良いことを意味しません。 率直に言って、発表されたイノベーションからもっと期待していましたが、予想外に楽しいことが判明した未宣言のイノベーションで十分でした。

彼の同僚のスイス人プログラマーが今回何をしたかを理解してみましょう。





Unicode





間違いなく、Unicodeのサポートは重要かつ必要です。実装が遅すぎる場合は、決して大したことはありません。 ただし、すべてが定性的にサポートされており、ヘブライ語、カタカン語、またはその他の意味不明な名前のファイルは、TCおよびそのすべてのツールで正しく表示および処理されるようになりました。 また、ユニコードバージョンが非ユニコードの先祖の隣に存在する可能性のあるプラグインでさえ（拡張子にuプレフィックスを取得します）。 更新されたAPIのドキュメントはすでに入手可能です。実験を開始できます。

強化されたエンコードのサポートが標準ビューア（Lister）に登場しました。 前の3つ（ANSI、ASCII、1人のカスタムユーザー）の代わりに、システムでフォントが見つかったエンコーディングを選択できます。

これらすべてのために、TCは依然として一意でないアプリケーションのままです。たとえば、ローカライズファイルとメニューは、シングルバイトエンコードファイルに保存されたままです。 しかし、率直に言って、これはまったく重要ではありません。

ヒント ：ユニークで便利で快適な変更で何が悪いのでしょうか？ 以前にユニコード対応フォントを使用したことがある場合は何もありません。 しかし、非Unicodeフォントを愛する人は、Unicodeサポートとお気に入りのフォントのどちらかを選択する必要があります。 お気に入りのFixedsysのユニコードアナログを既に見つけました。嬉しいです。



内部協会





内部関連付けのメカニズムはクールです。 これはとてもクールです。 これは非現実的です。TCは独自の方法を使用してシステムの関連付けを無視できるようになりました。 プラスは明らかです。事前設定されたTCを使用すると、システム設定に関係なくすべてが同じように機能します。 Totalに加えて必要なすべてのソフトウェアをUSBフラッシュドライブに入れたい人の数を考えると、これは間違いなくクールだと言えます（TCベースのオペレーティングシステムに関するジョークのもう1つの理由があります）。 また、新しいシステムで古いTotalを実行するだけでも、この便利さが気に入っています。たとえば、テキストファイルをお気に入りのエディターにリンクすると、メモ帳は不要になります。 ちなみに、バインディングは拡張機能だけでなく、たとえばバックライトを設定するときと同じ方法で、ファイルのグループに対して実行できます。

今、主なことは、この設定を一度強制することです=）。



パンくず





パンくず（パンくず）-Windows Vista Explorerをナビゲートする方法-表示されたファイルパスの各レベルのダイナミックナビゲーションメニュー。 現在、TCにもこのメカニズムがあり、パネルタイトルの上にアドオンとして実装されています。



新しいナビゲーション機能自体の外観は楽しいものです。 特定の状況では、Breadcrumbを使用したディレクトリ内の移動は、他のどの方法よりも高速です（Explorerでは確実です）。

しかし、何らかの理由で、ギスラーは自分のブレッドクラムを作るだけでなく、少しの売春婦とブラックジャックを追加することを決めました。 悪魔はいつものように些細なことに住んでいます-そして、これらの些細な事は考慮されませんでした：メニューのナビゲーションはあまり便利ではありませんでした。 階層メニューを表示するには、マウスを何度もクリックするか、メニューの強調表示されたセクションにカーソルを合わせる必要があり、Breadcrumbの動作のロジックを理解するのは難しいことがわかりました。 そして多くの人にとって、彼は単に存在の事実に単純に干渉しました-パネルのタイトルをクリックすると、パンくずメニューが常に表示されました。

これはすべて、テスト担当者の間で不満の波を引き起こしました。ベータテストプロセスで、ブレッドクラムにさらに多くの設定が追加されたためです。 設定ファイルの直接編集によってのみ編集可能です。 彼らの助けを借りて、原則として、あなたはまだ行動を微調整することができますが、1つの重要な欠点は何によっても排除されません。 これはパンくずリストのサイズ、より具体的には高さです。 Windowsの対応物と比較してください-Explorerはマウスの照準を必要としません。

この最小化の理由を推測することができます。TCにはそれぞれ2つのパネルがあり、幅のパンくずの幅は2倍小さくなります。 高さは比例して減少します。 しかし、寸法をカスタマイズ可能にしてみませんか？ その場合でも、パネルに収まらないパスについては下劣なスクローラーがパネルに表示されますか？..もちろん、キーボードからのブレッドクラムコントロールにはオプションがあり、慣れれば、実際にはファイルパネルのナビゲーションを複製しますが、非常に便利です直接。

ただし、他の嫌いなイノベーションと同様に、パンくずは安全にオフにすることができます。 しかし、まだ-非常に興味深いアイデアの混乱の最も思慮深い実装ではありません。



FTPマネージャーとそのカタログ作成者





FTPアカウント管理ツールは、カタログ化機能を獲得しました。 フラットリストがツリー構造に置き換えられ、化合物を簡単にグループ化できるようになりました。 20を超えるものがあり、TCがFTPを操作するためのメインツールである場合、喜ばしいことに、FTPブックマークはお気に入りのブラウザのブックマークと同じように配置できます。

しかし、新しいFTPマネージャーが最初のベータリリースで登場した方法は、私をinさせました。 率直に言って未完成で、編集機能はほとんどありませんでした。 ディレクトリでは何もできません。名前を変更したり、移動したり並べ替えたりすることはできません。 削除のみできます。

悲惨な光景。

幸いなことに、私はこれについてinしただけでなく、ギスラーはまだヴォックス人口を聞いています。 ディレクトリの名前変更のリリースの結果、すでに利用可能なため、欠点の一部が除去されました。 しかし、運動-悲しいかな、いや。 どうやら人気の声はもっと大きくなっているはずです。



完全に安全です！





FTPの変更は、インターフェースだけでなく、それ以上の影響を受けました。 FTPからパスワードを安全に保存するという長年の問題は、明らかに完全に解決されました。 標準キーを使用したパスワード暗号化の古いバージョンに加えて、ユーザーが指定したパスワードを使用した暗号化の可能性があります。 このパスワードはどこにも保存されず、最初の要求で一度入力されます。 ユーザーアクティビティ中、TCはこのマスターパスワードを文字へのポインターの配列の形でメモリに保存し、各文字には乱数が散らばっています。これにより、サードパーティソフトウェアによる「パスワード回復」が事実上なくなります。 パラノイアは、追加のセキュリティ設定を構成できます。たとえば、TCは、時間が経過した後、ウィンドウが最小化されたとき、またはスクリーンセーバーが起動したときに、マスターパスワードを「忘れる」ことができます。

プラグインでマスターパスワードを使用できるようになりました。 原則として、開発者がこれを独立して実装するために必要な場合、開発者に干渉することはありませんが、既製のメカニズムを使用する方がより論理的で簡単です。



ツールバー





以前のバージョンでは、ツールバーはすでに根本的にやり直されていたため、XPおよび高度な表示機能のサポートを獲得しました。 これで終わりではありませんでした。現在、パネルの機能は論理的に改善されています。

まず、キーボードからもパネルにアクセスできます。 確かに、これはボタンへのホットキーの割り当てではなく、多くの人が望んでいたものであり、それに焦点を合わせる動きです。

次に、パネルの寸法が2行に制限されなくなりました。 もちろん、これは決して重大なことではありませんでしたが、TCインターフェイスの一部がサイズ変更されると単純に「消える」場合は非常に奇妙に見えました。

第三に、任意の場所でパネルを行に分割する機能を追加しました。 有用な画面スペースが失われるため、これが必要な理由は完全には明らかではありません。 ただし、使用しない場合、これは干渉しません。

4番目、そしてもちろん、主なもの：ツールバーのボタンは、メニューの形式で他のパネルを呼び出すことができます。 実際-TCMenuのようなユーティリティの組み込み（したがってより便利な）アナログ。 すべてがスムーズに、不要なホイッスルなしで行われました：ドロップダウンサブメニューなしで、1つのボタンで1つのメニューパネルへの呼び出しを切断できます（そして当然のことです-分岐メニューには、他の実装オプションがたくさんあります）。 おそらく、新しい機能を最初に構成しようとするときに問題が発生します。そのサポートが有効になっているチェックマークがすぐにはわかりません。 教えてください-このチェックマークは「メニューとして表示」と呼ばれ、ボタンがハングアップしてTCツールバーでファイルを呼び出すときにのみ表示されます。



ファイロフィチ





新しいTCバージョンごとに、ある種のファイル処理メカニズムが追加されていることに気付きました。これは誰もが長い間求めてきましたが、常に無視されてきました。 以前のバージョンでは、「オンザフライ」でダビングするときにファイル比較が追加され、BTMが改善されました。 これで-コピー時の自動動作を設定します。 操作を開始する前に、書き換えが必要な検出されたファイルに対するTCの反応を構成できます。 前と同じように、毎回ユーザーに確認を求めるか、TCが上書きするかどうかを決定するルールを指定できます。たとえば、新しいファイルのみをコピーするか、古いファイルの名前を自動的に変更します。

同時に、複数のレシーバーに一度にコピーすることが可能になりました-複数のディレクトリを選択して対応するオプションを有効にすると、コピーされたファイルがそれぞれのレシーバーに表示されます。 また、開始されたコピープロセスには、[一時停止]ボタンがあります（以前は、バックグラウンド操作のみを一時停止できました）。



そして、すべて





以下は、TCの各アップデートでワゴンと小さなトロリーである、あらゆる種類の楽しいものです。



  • Windows x64での作業時の互換性が改善されました。 最も顕著な明白な改善は、64ビットプログラム用の本格的なコンテキストメニューです。 これは、メニュー呼び出しをインターセプトするラッパー（TCディレクトリでTCMDX64.EXEファイルを検索-これです）を介して実装されます。
  • カスタムファイルコメントのサポートの強化。 Gislerは多くの必要な機能（たとえば、名前にUnicode文字を含むファイルのコメントのサポート）を追加しませんが、descript.ion著作権形式の一方的な変更が不可能であることを参照しながら、いくつかの改善が行われています。 このバージョンでは、最大コメント長が4 Kbに増加し、複数行のコメントがサポートされました。

    ヒント ：名前にUnicodeを含むコメントをdescript.ionファイルに手動で追加した場合、TCは通常それを表示します。 明らかに、制限は人為的です。
  • 個々のディレクトリツリーの改善。 以前のバージョンに登場した後、多くのユーザーが非常に曖昧に感じていたこの追加は、大幅に改善されました。 本格的なドラッグアンドドロップが登場し、メインパネルで使用可能なほとんどすべての機能がサポートされるようになりました。 これらの改善がこれらのツリーの使用を開始するのに十分であるかどうかはわかりません-それにもかかわらず、それらはエクスプローラからTCに移行するユーザー向けに設計されています。ご存知のように、私は長い間それらの1つではありません。
  • 新しい修飾キー：Altキー、Shiftキー、Ctrlキーと共に、キーボードショートカットにWinを追加できます。
  • クイックフィルター。 些細なことですが、それなしではどうすればよいか想像できません。 Ctrl + Sを押して、文字の入力を開始します-これらの文字を含む名前のファイルのみがパネルに残ります。 さらに、信じられないほど便利なことに、プラグインによってフィルターが拡張されることがわかりました。 テンプレート、正規表現などを使用してクイックフィルターを作成できるQuickSearch eXtendedプラグインが既にあります。
  • アーカイブが改善されました。zip暗号化とRARアンパッカーのマルチスレッド操作のサポートが組み込まれました（対応するバージョンのunrar.dllがある場合）。
  • Windows 7タスクバーの進行状況バーのサポート。
  • 最後になりましたが、 ロシア語のディストリビューション 。




もちろん、これはすべて改善と変更の完全なリストからはほど遠いです-修正のみをリストするには小さなサイズの数ページが必要です。 すべてを読みたい-いつものようにあなたを助けるHISTORY.TXT。 質問がある場合は、 RU.TCKBとロシアの Total Commander コミュニティが常に答えを手伝います。
