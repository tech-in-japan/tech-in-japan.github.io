パンの例であなたがITの世界にいる人を親relativeに説明する方法

こんにちは、Habr！ Mischa von Nachtigalの記事「あなたの家族に技術の仕事を説明する方法」の翻訳を紹介します。









ダイニングテーブルでIT専門家の本質を伝えるのに簡単に役立つメタファーのリスト。



さあ、行こう！



まえがき



あなたの会社はパン屋です。 ベーキングはアプリケーションです。 時々あなたのパン屋は人々にパンを直接販売します、すなわち 最終消費者（b2cアプリケーション）に提供されますが、他の会社（b2bアプリケーション）に販売し、製品を人々に提供することもあります。



役職



プロジェクトマネージャー



あなたの仕事は、ベーカリーまたは特定の製品の戦略を開発することです。製品にどのような味と特徴があり、会社が市場に投入する製品のタイプも同様です。 製品を開発するとき、あなたは消費者側を代表します。 フィードバックを新しいアイデアに変換するのに役立ち（たとえば、新しいシロップや新しい種類のベーキングを追加する）、製品が人気を博し、一定の品質レベルを満たすように決定するのにも役立ちます。 シェフを管理したり、パン屋を管理したりすることはありませんが、ビジネス展開の全般的な方向性を設定し、特にコーヒーショップの作成などの拡張オプションを提供します。 マスタークラス、音楽ショー、さまざまなイベントでのケータリングなどの実施



工業デザイナー



あなたはある種のシェフです。 あなたは顧客の好みを決定し、レシピを開発し、下された決定が本当に彼らがベーカリーに望むものであることを確認します。 しかし、お菓子を焼かないで、メニューやパッケージングから彼の好みに至るまで、どのように、どのような製品が買い手に届くのかを理解します。 シロップの味が正しいだけでなく、新しいタイプのベーキングだけでなく、既存のベーキングにも顧客が満足できるように、チームがその製品を正確に製造することを管理します。 新しいタイプの製品については、プロトタイプを開発しています。 彼らがどれだけ根を張り、ゲームがろうそくの価値があるかどうかを決定します。









エンジニア（開発者）



あなたはパン屋です。 あらゆる種類のペストリーを調理し、オーブンを使用する方法を知っています。 仕事では、スライス、チョッピング、フライ、ホイップなど、さまざまな調理方法を使用します。 パイからパン、マフィンからパンまで-どんな技術とステップが必要か、どの温度とツールがあなたの製品を素晴らしいものにするか知っています！ プロダクトマネージャーおよび工業デザイナーと密接に連携して、消費者が期待することを正確に実行します。 時々気づきます。 製品の構想や特性が製品の準備を複雑にする可能性があり、あなたとあなたのチームが状況から抜け出す方法を見つけようとしていること。 新製品に関しては、技術的な観点からそれが可能かどうかを判断し、問題を解決するために新しい技術を習得しようとします。



システム管理者



時間を無駄にしないために、理想的には大規模なオーブンで同時に焼く何百種類ものベーキングを備えた大規模なパン屋のネットワークを想像してください。 あなたの仕事は、すべてがスムーズに実行されるようにすることです。 製品のこの部分またはその部分は規則に従って焼き上げられ、顧客に販売できること、ベーカリー自体が過熱しないこと、および大量の作業による破壊の脅威がないこと。 また、あなたは安全にも責任があります-火災の安全性、必要に応じて消火し、常にベーカリーの効率を高めることを求めています。









製品マーケティング



メニューだけでなくショーケースにも取り組んでいます。 ペストリーの試飲と購入のために新しい消費者を引き付けるのはあなた次第です。



規約



API



オーブンダイヤル



クラウド



ペストリーを世界中に迅速に配達できる大規模なネットワークオーブン。









機械学習



オーブンは突然非常にスマートになり、ベーキングのタイプに応じて必要な温度を設定したり、プロセスのいくつかの段階（ローリング、生地の練り、準備の確認など）を支援するなど、独自の機能を実行できます。たとえば、機械学習はベーカリー自体にも役立ちます以前の購入やその他のデータから推奨事項を収集します。









ペアプログラミング



2人のパン屋が一緒にベーキングを準備し、互いに助け合い、プロセスの最後に達成された結果について話し合います。 通常、このような作業の結果は、より急勾配の製品と効果的なチームです。 また、時間の節約にもなります。 5回。
