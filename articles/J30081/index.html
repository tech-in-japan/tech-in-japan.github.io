GPRS（EDGE）対WAP、またはサイトのモバイルバージョンを作成するときに何を探すべきか

- ブログからのクロスポスト



最近、友人とバーに座って、彼からそのような状況を聞いた。 顧客は彼のサイトに来て、必死にモバイル版のサイトを必要としています。 しかし、実際には、人々はまったく路地を持たず、WAPまたはGPRSを必要とします（またはロシアで機能するテクノロジーの進化として-EDGE）。 モスクワには今何が必要ですか（それについて話しましょう、地域については何も言えません）、彼女にとって何が便利で、何が安いのですか？ 開発と保守が安価なのは何ですか？ 以下では、これらの質問にいくつかの答えを与えようとしますが、それらは深く主観的です-厳密に判断しないでください、私と議論する方が良いです;）





ウィキペディアを引用することから始めましょう。これは、私たちが話したいことのアイデアを与えてくれます。



WAP（ワイヤレスアプリケーションプロトコル）は、コンピューターやモデムに頼ることなく、携帯電話のみを使用してインターネットリソースにアクセスする手段です。 実際、インターネットからの情報が携帯電話のディスプレイに送信される方法を記述する技術標準です。



GPRS（一般パケット無線サービス-一般パケット無線通信）は、パケットデータ送信を実行するGSMモバイル通信技術のアドオンです。 GPRSを使用すると、携帯電話のユーザーは、GSMネットワーク上の他のデバイスや、インターネットなどの外部ネットワークとデータを交換できます。 GPRSには、オンラインで費やした時間ではなく、送信/受信した情報の量に応じた課金が含まれます。



ご覧のとおり、両方のテクノロジーは本質的に互いに異なっています。 WAP（ちなみに既に1995年に開発され、1998年に多かれ少なかれ広く実装された）が独立したプロトコルであり、GPRSがGSMモバイル通信技術のアドオンである場合。



それでは、ポイントを見ていきましょう。



課金 私のお気に入りのオペレーターを例に考えてみましょう-これはVimpelcom（Beeline）です。価格はモバイル市場全体でほぼ同じで、わずかな変動があることを考慮する価値があります。

  • WAP-受信または送信された情報の10KBあたり平均2ルーブル。
  • GPRS-1MBの受信または送信情報あたり平均6ルーブル。


ご覧のとおり、WAPはGPRSと比較して非常に高価です。 これは、テクノロジーをサポートするためのコストが原因であり、たとえ両方のテクノロジーがインターネットへのアクセスを提供する場合でも、それらは異なる方法と異なる電話モデルにアクセスを提供します。 ちなみに、おそらく数年前にWAPは送信された情報ではなく、セッションに対して請求されたことを誰かが覚えているでしょう:)



開発者向けの技術仕様。

WML（ワイヤレスマークアップ言語）は、WAP標準に従って携帯電話やその他のモバイルデバイスで使用するためのドキュメントマークアップ言語です。



例： <?xml version="1.0"?>

<!DOCTYPE wml PUBLIC "-//WAPFORUM//DTD WML 1.3//EN" "http://www.wapforum.org DTD/wml13.dtd">

<br/>

!



















ただし、今日、この言語のサポートはXHTML Mobile Profileに向けて消滅しています。 これは重要なステップです。WAPが死ぬか、GPRSの真の代替になるからです。 しかし、WAPの開発は失速し忘れられると確信しています。 なんで？ はい、その中に意味はありません。サイトの作成と保守には、親愛なる、遅い、追加の費用が必要ですが、それでも需要があります。 しかし、それは時間の問題です。



また、GPRSを使用して表示するように設計されたサイトは、通常のxHTML + CSSで構成できます。表面にあり、常識のある開発者に見えるニュアンスを観察するだけです。 さらに、開発者はモバイルデバイス用のスタイルを記述するだけでよいことがわかりますが、これは重要ではなく、すべて親サイト、その構造などに依存します。



膨大なファイルを操作します。 現代のビジネスペースには応答性が必要です。 それが人々が携帯電話からネットワークに登った理由です。 たとえば、今朝、地下鉄で働くために運転していたとき、銀行のウェブサイトから.docフォームをダウンロードし、それを開いて部分的に記入しました。 旅行がもっと長ければ、この同じWordファイルに記入してメールで送信します。 ワップで、私はそれを行うこともできますが、それは数倍高価で、数倍遅くなり、モスクワのメトロの中心部でも、メトロ、崖、デッドゾーンでカメの速度でファイルをダウンロードできるという事実ではありません。 GPRSに向けたもう1つの適切なプラス。



実際、あなたは長い間議論することができます、私はあなたに何かを納得させる仕事を自分自身に設定しませんでした、少なくとも私はすでに大量のファイルを充電して作業することで解決しました。 次のように要約できます。

  1. GPRSはWAPよりも安価です。
  2. GPRSはWAPよりも高速です。
  3. GPRSはWAPよりも多くの開発機会を提供します。
  4. サイトのWAPバージョンのサポートと開発はより高価です、私には思えます。
