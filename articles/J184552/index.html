DXperience 13.1-DevExpressの.NETコンポーネントの新しいバージョン

少し前に、.NET開発者向けのコントロールとコンポーネントのセットの新しいメジャーバージョン-DXperience 13.1のリリースを発表しました。 伝統的に、新しいバージョンは、WinForms、ASP.NET、WinRT、Silverlight、およびWPFプラットフォームの開発者に多くの新しい製品と改善をもたらします。



一般的な概念と設計について



Windows 8の設計は、多くのユーザーと開発者にアピールしました。 また、従来のデスクトップアプリケーションも引き続き必要です。 そのため、デスクトッププラットフォームの開発を続けていますが、同時にトレンドから離れることはありません。 その結果、タッチスクリーン、ナビゲーション、データ視覚化要素の使用に焦点を当てた新しいテーマ、コントロールが登場しました。 そして、ASP.NET、WinForms、WPF、Silverlightのいずれであっても、これらすべてを使い慣れたプラットフォームで使用できます。



新機能はかなり広まっているので、英語で完全に読みたくない場合は、catへようこそ。

Winforms



リリースには新しいアイテムが豊富に含まれていました。



スプレッドシート制御 アプリケーションで、式、セルの書式設定、XLS、XLSX、CSV、TXT形式のデータのインポート/エクスポートをサポートする本格的なMS Excelを実装できます。 また、スプレッドシートコントロールに必要なすべてのメニュー項目を備えたリボンメニューの既製の設定がなかったわけではありません。 一般的に、自分で見てください。



マップコントロールを使用すると、WinFormsアプリケーションでマップを表示できます。 コントロールには次の機能があります。

  • DirectXによるクイックマップレンダリング
  • BingおよびOpenStreetMapのマップをサポートします
  • エンドユーザー向けのナビゲーションバーがあります
  • Shapefile形式からの図形の読み込み、および図形を描画する機能をサポートします。
  • 他の素敵なささいなこと






PDFビューアーコントロール（ベータ）。 これで、アプリケーションから直接PDFファイルを見ることができます。PDFビューアーがエンドユーザーにインストールされているかどうかは関係ありません。 このコントロールは、スケーリング、スクロール、埋め込みフォント、ベクターグラフィックスなどをサポートしています。 リボンメニューの既製の設定も利用できます。



WinRTLiveTileManager 。 このコンポーネントは、WinFormsの既存のアプリケーションに簡単に統合され、 Live Tile Managerとの通信チャネルを確立します。 Live Tile Managerは、スタート画面とWinFormsアプリケーションの間を仲介するWinRTアプリケーションです。 Live Tile Managerは、スタート画面に、WinForms上のアプリケーションから受け取ったすべての必要な情報を含むタイルを表示します。 ドキュメントの詳細。







WinFormsの革新のリストは、別のかなり大きなトピックに十分です。 そのため、非常に簡単に言えば、アイコンのライブラリ、新しいエディター（ Tree-List Lookup 、 SparklineおよびPopup Gallery ）、タッチスクリーンでの使用に最適化されたコントロール、デザインタイムの改善などがあります。

詳細： WinFormsの新機能 。





ドキュメントサーバー



ドキュメントサーバー - ドキュメントの処理を自動化するための新しいライブラリ。 このライブラリは、次の部分で構成されています。

  • スプレッドシートドキュメントサーバー-スプレッドシートライブラリ
  • RichEdit Document Server-テキストドキュメントを操作するためのライブラリ
  • Snap Document Server-レポートライブラリ
  • 圧縮ライブラリ-暗号化されたものを含むZIPアーカイブを操作するためのライブラリ
  • バーコードライブラリ-あらゆる種類のバーコードを生成するためのライブラリ
  • 単位変換ライブラリ-さまざまな測定単位を変換するためのライブラリ

Silverlight



Silverlightについては忘れませんでした。

  • 新しいリリースでは、 グリッドコントロール用にバンドビューが作成されました。 これは、複数行の列ヘッダーと複数行の行を作成する機能です。







  • ビルトインツールチップ、マップセレクター、データベースにバインドする機能などが、 Map Controlに登場しました。 とりわけ、コントロールには、地理データに従って特定のエリアを自動的にペイントする機能が追加されました。







  • 特に、タッチスクリーンに優しいWindows 8のスタイルの新しいコントロールであるFlipView 、 PageViewおよびSlideViewに注目してください 。


さて、タッチスクリーンでのアプリケーションの使用について話すと、そのようなスクリーンに最適化されたテーマTouchlineDarkなしではできません。

WPF



Silverlight の新機能のリスト（まだ少し高い）を読んでいない場合でも、まだ読んでください。Silverlightの新機能はWPF製品ラインでも有効です。



さらに、多くのことがWPFに対してのみ行われましたが、それについては以下で詳しく説明します。 多くのイノベーションがあるので、それらをカテゴリーに分けます。

新しいコントロール：

  • プロパティグリッド -Visual Studioのプロパティウィンドウと同じです。







  • （Silverlightの上記リストに加えて）タッチスクリーンでの使用に最適化されたコントロール： DateEditおよびRange Control 。







新しいウィザードと設計時の改善：

  • チャートコントロールウィザードは、新しいチャートの作成と構成だけでなく、既存のチャートの編集にも役立つウィザードです。 さらに、このウィザードはアプリケーションに統合でき、エンドユーザーはダイアグラム自体をカスタマイズできます。







  • WPFデータソースウィザードは、3つのステップでデータを操作するコントロールのデータソースを構成できるウィザードです。
  • 足場ウィザード -さまざまなビュー（データ）およびデータモデル（データモデル）を生成するための面倒な作業をすべて行うウィザード。
  • インスタントレイアウトアシスタント -名前が示すとおり、数回クリックするだけで複雑なレイアウトを作成できます。


Youtubeチャンネルで実際に動作しているウィザードを見ることができます。

ASP.NET WebFormsおよびASP.NET MVC



新しいコントロールから始めましょう。

13.1では、 ImageGalleryコントロールを取得しました 。 指定されたディレクトリから画像をロードし、画像のサムネイルを自動的に作成し、タッチジェスチャをサポートし、画面の幅に適応できます。







ASP.NET MVCの拡張機能のラインはまだありません。 3つの新しいアイテムを追加しました。

  • ImageSlider







  • ファイルマネージャー







  • キャプチャ







改善点について：

  • ページのコンテンツをスクロールしながらロードするのが好きな場合（いわゆるエンドレススクロール、ソーシャルネットワーク、たとえばTwitterで非常に人気）、Webアプリケーションでも同じことができるようになりました。 次の製品は、無限スクロールをサポートしています。

    • グリッドビュー
    • データビュー
    • ニュースコントロール
    • ImageGallery






  • ImageSliderを改善しました 。 バイナリストアから画像を表示でき、テンプレートをサポートし、2つの新しいモード（スライドショーモードとナビゲーションバーモード）を受け取りました。
  • FormLayoutは 、データソースを直接操作できるようになりました。
  • アプリケーションをSharePoint 2013（このリリースでサポートします）でネイティブに見せたい場合、またはMS Office 2013と同じくらいクールにしたい場合は、新しいMetropolis Blueテーマを使用してください。

報告



レポートジェネレーター用に、新しいコントロールXRTableOfContentsを作成しました 。 このコントロールは目次を自動的に作成し、複数ページのレポートを作成するときに非常に役立ちます。







特定のビジネスプロセスのダイナミクスを明確かつ簡潔に示すために、 Sparklineコントロールを使用できるようになりました。







Visual Studioで独自のレポートデザインを作成する必要がなくなりました。 新しいレポートデザイナーにより、ユーザーは必要な形式でレポートを収集できます。



また、 Snapコントロールでは、ウィザードを使用してデータソースを追加する機能、コードを使用してレポートを作成するAPI、セルスタイルを編集する機能などを実装しました。

eXpressAppフレームワーク



新しいリリースでは、並行編集レコードの競合処理が改善されました。 ユーザーは、1つのオブジェクトのフィールドを同時に編集してから、変更をマージできるようになりました。







リストビューで複数のエントリを編集する機能が追加され、すぐに変更を保存します。 これで、ユーザーは各エントリを個別に保存する必要がなくなりました。 さらに、リストビューを閉じて、編集したレコードを保存できます。





XAFは、WinFormsのSplashScreenManagerコントロールとの統合をサポートしています。これにより、アプリケーションを起動するときに、美しく動的に更新された開始画面を（文字列で）簡単に整理および表示できます。







とりわけ、Entity Framework （eXpress Persistent Objects（XPO）ではない）のサポートも改善されました 。また、データ保護モジュールがさらに強力かつ柔軟になりました。開発者またはデザイナーのユーザーが指定した基準に従って、ビジネスエンティティの特定のフィールドを保護できるようになりました。







そして最後に、ユーザーのフィードバックと希望に基づいて、ビジネスフレームワークでその他の小さな改善を多数行いました。

詳細（英語）、および動作中の新機能を示すビデオは、ここで見つけることができます： XAFの新機能 。





CodeRush



新しいCodeRushリリースでは、XAMLサポートに焦点を当てました。 これはかつて私たちの弱点でした。 リソース、データ型、プロパティの分析を改善し、Visual Studioが無力な対象オブジェクトにユーザーが簡単にナビゲートできるようになりました。 優れた革新は、リソース、名前付きコントロールとバインダー、欠落しているXAML名前空間の自動宣言、マークアップにすばやくコントロールを挿入するためのグリッドとテンプレート（スタジオコードスニペットの高度なアナログ）に便利に行と列を追加/削除するためのヒントを備えたスタジオインテリジェンスの拡張です。 CodeRushのイデオロギー家であるMark Millerのブログで 、これらすべての詳細を読むことができます。



前回のリリースでは、デバッグの難しいタスクであるデバッグビジュアライザーで開発者にアシスタントをリリースしました。 13.1で、彼は新しいレベルに移動しました;革新の中で、特に注目に値します：

  • 切除運動の分析を追加
  • Silverlight 5でXAMLバインディングのサポートを追加
  • デバッグ中に発生したすべてのアクションに関する情報は、別のウィンドウに保存されます


新しいリリースの大規模プロジェクトのパフォーマンスの問題を解決するために、スマートタグの非同期メニューモードを実装しました。 これにより、別のスレッドで機能（大規模なプロジェクトでは高速ではない可能性がある）の可用性を確認できるため、ユーザーの作業のフリーズやスローダウンを回避できます。

改善と革新のより詳細なリストは、CodeRushの新機能で提供されています。
