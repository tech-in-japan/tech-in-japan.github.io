最新のブラウザと古いOperaの使いやすさの比較



この記事は、古いOperaの豊富な機能のいくつかの回想です。 しかし、その中に私は古いオペラに特有のものを集めたいです。 私はまだOpera 12をインストールしていますが、これはまだ使用していませんが、手で取り外せません。



私にとって、Old Operaはほとんどブラウザの標準でした。 それは非常に多くの素晴らしいものを実装しており、それは何らかの理由で現代のブラウザには存在しません。



1年ほど前、Opera は GoogleのChromiumフォーク（Blink）をベースにした新しいブラウザを作成していることを発表しました。



1年が経ちましたが、以前のユーザーエクスペリエンスから、OperaはどこまでChromeの哀れな弟でしたか。 比較的若いYandex.Browserでさえ、はるかに速く開発されており、独自の顔を持っています。 WebKitとBlink、Firefoxのブラウザの代表として、古いOperaとChromeを比較します。 そして、これは結局のところ、私の個人的な意見であることに注意してください。

3.ファイルをダウンロードして開く



古いOperaでは、ファイルをダウンロードするときにすぐに選択肢が表示されます-ファイルをダウンロードして、ファイルを開きます（それは重要です-それを開くアプリケーションの選択で）。 これは論理的です。 ファイルをダウンロードするとき、ほとんどの場合、ファイルを開くか保存するだけです。 これがプログラムのインストーラーである場合、ダウンロードにファイルの配布を保存する気はありません。単に実行し、プログラムをインストールするだけです。配布をブラウザのキャッシュに配置し、キャッシュが自動的にクリアされると消去されます。



[保存]を押すと、保存先が表示されます。 もちろん、この点は議論の余地があります。 デフォルトで設定されている同じ場所（ダウンロードなど）にファイルを保存する方が速くて便利です。保存場所を選択する方が便利です。 個人的には、後者は私に近いです。



最新のブラウザは何をしますか？



FirefoxはOperaに近い-アプリケーションを選択してファイルを開くか、場所を選択せず​​に保存することを提供します。 ブラボー、Firefox。





ChromeおよびWebKit / Blinkブラウザーは、単にファイルを保存します。 そこで、デフォルトのダウンロードで保存するように設定されています。 ファイルがダウンロードされると、ファイルを突くことができ、何らかの方法で開きます。





どこで開くかを選択することはできません。 このタイプに一致するアプリケーションで開くだけです。 もちろん、これは多くの場合に便利です。 しかし、標準プレーヤーではなく、別のプレーヤーでmp3を開きたいのですか？ 「このファイルを表示する」を突き、右ボタンで突き、どこで開くかを選択する必要があります。 この場合、一連の不要なアクションが実行されます。



Option OperaとFirefoxのほうがずっと便利だと思います。 シナリオに応じて、どこかでクリック数が増え、どこかでクリック数が減ります。 ただし、どちらの方法でも、すべてのオプションをほぼ同等にすばやく作成できます。 そして、WebKit / Blink-browsers-または同数のアクション、または多くのアクション。

8.タブを切り替える



CTRL +数字（MacではCMD +数字）キーボードショートカットは、ブラウザーでアカウントで既に開いているタブを切り替える必要があると誰かが考えています。 古いOpera（および現在はSafari）では、これらのキーボードショートカットにより、エクスプレスパネル（Safari-ブックマーク）からブックマークが開かれました。 個人的には、それは私にとって非常に便利に思えました。 彼はコンピューターに行き、すぐにキーボードだけで4-5タブを開きました-CTRL + T-> CTRL + 1、CTRL + T-> CTRL + 2、私が最も頻繁に訪れるサイトで、一部ではなく自分で設定したサイトブラウザのアルゴリズム。

しかし、もちろん、これは非常に物議を醸す問題であり、好みの問題です。ここで議論するのは無意味です。 しかし！ Operaのタブを切り替える際に、まだ素晴らしいことがありました-これはCTRL + Tabです-これは既に開いているタブを切り替えています。 古いOperaでのみ、OSのアプリケーションを切り替えるのとほぼ同じように機能しました。つまり、タブは順番に愚かではなく、最後に開いたものを考慮して切り替えられます。
