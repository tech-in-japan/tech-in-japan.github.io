ブラウザのVR / AR。 WebVR APIを使用して最初のアプリケーションにすばやく参加して作成する方法





仮想現実および拡張現実は積極的に開発されており、世界を占領する権利を主張し始めています。 さらに、これらの技術の用途は多様であり、ゲームだけに限定されません。 また、ブラウザーでVR / ARテクノロジーを操作するツールの出現により、関心はさらに高まっています。 WebVRと呼ばれるJavaScript APIを使用して、MVP（Minimum Viable Product）Webプロジェクトを実験して作成できます。 それが何であり、どのように動作するかを理解しましょう。 そして、この非常にWebVR APIなしで行うことは可能ですか？



この記事はガイダンスのみを目的としており、WebVR APIとWeb ARの実装に関する一連の記事の最初の記事です。 トピックが出てきたら、さまざまな手法とフレームワークを使用してより具体的な例を示しながら、引き続き開発を続けます。 私の個人的な経験を共有し、記事でWebVRを使用する基本的な原則、Web ARの作成方法、iOSを持っている場合の対処方法、および今日すべてを開発できるデバイスについて検討したいと思います。

なぜWebVR APIが必要なのですか？



WebVR APIは、デバイスを操作するためのソフトウェアインターフェイスです。 彼は3Dグラフィックスについて何も知りません。 グラフィックスの操作、シーンのレンダリング、光源のインストール、その他すべてはプログラマーの厳しい肩にかかっています。 WebVR APIを使用すると、デバイスへのアクセスを抽象化できます。 このAPIは、画像、デバイス、その機能、技術特性に関する情報を取得するためのツールを提供しますが、画像自体と3Dの世界は、HTML、CSS、WebGL、Canvasなどの既に馴染みのあるWebテクノロジーを使用して描画する必要があります...



WebVR APIは、作業のためのいくつかの基本的なオブジェクトを提供します。



  • ナビゲーター-デバイスのリストを取得し、アクティブを決定できます。
  • VRDisplay-ヘッドセットが頭を装着しているかどうか、フレームに関する情報、目を示します。
  • VRPose-デバイスの位置と向き、速度と方向に関する情報。
  • VREyeParameters-個々の目にビデオをレンダリングする方法に関する情報。
  • VRFrameData-別の目に投影するためのシーンフレームに関する情報。


オブジェクトの完全なリストは、次のソースで入手できます。



  • developer.mozilla.org/en-US/docs/Web/API/WebVR_API
  • msdn.microsoft.com/en-us/library/mt806281（v = vs. 85）.aspx

どうすれば世界を構築できますか？



VRの世界に参加するには、3Dグラフィックスの操作方法を学ぶ必要があります。 WebGLをゼロから学習する場合、これは困難な作業のようです。 しかし、最も簡単な方法は、Threejs、D3.jsなどの既製のライブラリ、またはVR（MozillaチームのA-Frameなど）を操作するための専用フレームワークを使用することです。 すでに、ReactのファンおよびFBの腸から出てくるすべてのもののためのReact VRのプレビューリリースがあります。 しかし、これは別の記事のトピックです。



タイプVR



A-Frameフレームワークを使用すると、最小限の労力でVRの世界を構築できます。 A-Frameは、仮想現実向けのWebアプリケーションおよびWebサイトを作成するためのHTMLフレームワークです。 ブラウザで結果を見ると、3D画像を表示するWebページのように見え、オブジェクトを操作したり操作したりできます。 全体像はWebGLを使用して描画され、フレームワークの主なタスクは、HTMLマークアップと同様に、フロントエンドレンダリング用に通常の方法で3次元の世界を作成できるシンプルなツールを提供することです。 フレームワーク自体はThreejsに基づいており、高レベルのアドインです。



A-Frameフレームワークは、一連の幾何プリミティブを介して3Dシーンを作成します。 基本的なコンポーネントがまだない場合は、より複雑なコンポーネントを追加したり、独自のコンポーネントを追加することもできます。 シーンオブジェクトには、位置、回転、スケーリングなどの一般的な幾何学的プロパティを使用できます。さらに、カメラと光源の位置を記述することができます。

Web ARはどうですか？



Web AR（拡張現実）-ブラウザで作成することもできます。 しかし、「WebAR API」は存在せず、Webテクノロジーに実装された拡張現実の単なる指定です。



技術的には、すべての作業はWebVRでの作業とまったく同じですが、さらにWebRTCを使用してWebカメラからビデオストリームを取得します。 さらに、フレームごとの処理のロジックを記述して、必要なオブジェクトを検索します。 そして、WebVRの場合のように、ビデオストリームの背景に対して3Dシーンをすでに描画しています。 そして、ARは必ずしもヘッドセットの存在を意味するわけではない、と言わなければなりません。 「Pockemon GO！」を思い出してください-これはARプロジェクトですが、VRヘルメットはありません。 したがって、ARプロジェクトを作成するために、VRヘルメットを用意する必要はありません。 しかし、同時に、VRとARの概念は、いくつかのパラメーターとテクノロジーで交差する可能性があります。

iOSデバイスで何をしますか？



WebVRについて話している場合、iOSデバイスではすべてがポリファイルを介して実装できます（またはWebVR APIを使用せずに、すべてを自分で記述し、加速度計やその他のセンサーを介して追跡アクションを実装し、自分で写真を打ち負かして2つのディスプレイをエミュレートします）。 ARについて話すと、WebRTCのサポートがないため、すべてが悪いです。 しかし、Argonjsのようなプロジェクトがあります。 これは、A-Frameベースのフレームワークと、もちろん特別なブラウザーで構成されるプロジェクトです。

ブラウザの本質は簡単です。2つの層があります。 1つの層はWebkitエンジンで、2番目の層（背景）はカメラからのビデオストリームの出力です。 ブラウザーには独自のAPIがあり、最初のバージョンのPhoneGapの動作に似ています（このプロジェクトの開発の夜明けにコマーシャルが2008年に試された場合、それは拡張JS APIを備えた特別なiPhoneブラウザーでした）。



同時に、ArgonjsはiOS用のVR開発にも使用できます。 より正確には、MVPを実行します。 JSでVR / AR向けのクロスプラットフォームアプリケーションを開発する必要がある場合は、React VRと組み合わせたReact Nativeの方向に目を向けるか、PhoneGapにすべてを詰め込むことができます。







重要！ AppStoreにはいくつかのブラウザーバージョンがあります。 現時点では、バージョン4をダウンロードする必要があります。ダウンロードしないと、フレームワークは起動しません。

ユースケースとアプリケーション



Tutu.ruについて話す場合、VR / ARを実際に使用するケースを使用するためのアイデアが既にあります。 たとえば、これらはToursプロジェクトの3Dギャラリーです。 少し洗練された360のギャラリーをVRヘルメットに適合させることができ、自宅にVRボックスを持っている人なら誰でも、プレゼンスの効果で将来のホテルとビーチを「見る」ことができます。 Trainプロジェクトの3Dギャラリーにも同じ手法を適用できます。このプロジェクトでは、車を内側から見ることができ、SapsanのVRヘルメットから場所を直接選択できます。 同様に、将来的には、都市間バスを利用できるようになります。 また、 エクスカーションプロジェクトでは、将来のエクスカーションのデモプレビューを行うこともできます。



別のケースは、 オフラインのオフィスでの観光客の娯楽です。 観光客が雑誌やパンフレットの代わりにツアーマネージャーに出向くのを待っている間に、Samsung GR VRヘルメットを入れて旅行カタログを再度作成することで、適切なツアーパッケージを選択できます。 ARテーマを開発したら、オフラインオフィスに顧客の娯楽のために情報スタンドやさまざまな面白いものを置くことができます。 古紙への反応が常に肯定的ではないためにクライアントがパンフレットをスキップできる場合、クライアントは新技術のプリズムを通して示される同じ広告を知覚し、満足することができます。
