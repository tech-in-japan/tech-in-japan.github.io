Olapサーバー-Oracle Essbase

Habréでは、少なくとも何らかの形で、 リアルタイムでの分析データ処理のタスクに関するマイクロソフトのソリューションの説明が提示されていますが、この問題の世界的リーダーであるOracle Essbaseについての言葉はありません。



Oracle EssBaseは、分析タスクの強力なサポートにより競合他社とは異なるOLAPサーバーであり、多くのアナリストのコンテキストで多変量分析を可能にし、会社の開発予測を作成し、レポート用のデータを準備します。 これは、 MOLAP （2種類の集計ストレージとブロックストレージ）、 ROLAPおよびHOLAP （MOLAP + ROLAP）のすべてのOLAPソリューションをサポートしています。



アーキテクチャは、次のように概略的に表すことができます。







次のコンポーネントで構成されます。



• Essbase Analyticsモジュール（ブロックストレージ）



このモジュールのAnalytic Servicesは、ブロックの概念に基づいてデータベースを作成します。これは、密方向のすべての可能なオプションで構成されるテーブルであり、これらのブロックは放電方向の交差点に配置されます。 このモジュールは、カスタムアプリケーションを使用した値の記録をサポートし、複雑な財務分析を目的としています。







のために設計

  • 反復計画、分布、複雑な分析（販売分析、収益性分析）に関連するアプリケーション向け
  • 多くの手続き計算と、あらゆるレベルの集計への変更の記録が必要な場合
  • 測定オーダ管理
  • データベースの全部または一部の計算
  • 複雑な計算を実行する


• エンタープライズ分析モジュール（ASO集合ストレージ）



このモジュールのAnalytic Servicesは、「集約された」データベースを作成し、それ自体にゼロレベルの要素を格納し、より高いレベルのすべての値を自動的に計算します。 数百万の可能性のある指標を使用して、非常に大きなデータ（極度に放出された）を分析するように設計されています。







のために設計

  • 大規模なスパースデータの場合
  • 標準集計ルールセット


• 分析管理サービス管理コンソール



これは、複数のデータベースサーバーの単一の管理ポイントをサポートするAnalytic Servicesデータベース管理者インターフェイスです。 このツールを使用すると、サーバーとアプリケーション、およびそれらのデータベースの両方を開発、保守、管理できます。







• 統合コンソール-Essbase Integration Studio （開発されていない、Essbase Integration Studioに先行）



Integration Serverは、リレーショナルデータソース、OLAPメタデータカタログ、Analytic Server（多次元データベースサーバー）間の関係を維持するマルチスレッドサーバーアーキテクチャです。







• Excel用Essbase SpreadSheatアドイン



このソフトウェアソリューションは、Microsoft ExcelでAD-HOCレポートを受信するように設計されており、多次元データベースに直接接続されています。 開発は停止されます。



• オフィス向けSmart View



Essbase SpreadSheetとは異なり、技術的ソリューションにおいて、Microsoft Officeソフトウェアパッケージ全体からデータにアクセスできます。 集中的に開発しています。



• EssCMD



コマンドラインインターフェイスは、アプリケーションの停止、ユーティリティユーティリティの起動、バックアップなどの管理タスク用に設計されています。



• MaxL DDL



管理タスク用のコマンドラインインターフェイス。



• Essbase API



これは、VB、C、またはJAVAから多次元データベースにアクセスできるソフトウェア開発ツールです。



•メンテナンスサービス-分析展開サービス



単一のエントリポイントを構築して、ユーザーアプリケーションと多次元データベース間の負荷分散とフォールトトレランスを確保できます。
