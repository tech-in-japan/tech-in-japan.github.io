コンストラクト2でモバイルゲームを作成します。経験の浅い人の道

この記事では、頭の中で考えを始めてから、アプリケーションを公開して広告を扱うまで、モバイルゲームをゼロから作成するプロセスについて説明します。 この記事は、このプロジェクトの作業を開始した時点で作者であったため、モバイルゲームの開発（およびプログラミング）の分野で経験や知識さえ持っていない人に適しています。



重要なポイント：

-このアプリケーションは、HTML5ゲームエンジンであるコンストラクト2を使用して開発されました。HTML5ゲームエンジンを使用すると、すべての一般的なモバイルプラットフォーム（だけでなく）にアプリケーションを移植できます。

-このような機能を実装するためのプログラミングスキルはまったく必要ありません。

-繰り返しますが、この記事では経験の浅い人の道について説明しているので、始めたいのであれば、この記事は役に立つかもしれません。

-アプリケーション作成の次の段階について説明します：開発、Androidモバイルプラットフォームへの移植、広告ネットワークとの統合、アプリケーションの公開、プロモーション（何らかの種類の無料アクティビティ）。

-これらの活動のすべての条件は、現時点では私の主な仕事で忙しくないという事実を考慮して示されているので、これに丸1日を費やすことができます。



スタート、アイデア、基本コンセプト



最初は、Construct 2ツール自体に関する記事を読んだ後、このプログラムの機能を調べることにしました。 Construct 2の機能を研究するには数週間かかりました。FreeEditionの無料版で十分です。 効果、レイヤーなどの数には制限がありますが、プログラムを研究するためにこれはあまり干渉しません。 無料版でモバイルプラットフォームに移植することはできなくなりました。



さまざまなスプライトの動作と物理エンジンを備えたサンプルは、プログラムに慣れるのに役立ち、同時にいくつかのアイデアのインスピレーションの源となりました。 スパイダーマンの原則に従って、私はそれらの1つで停止することに決めました-それはゲームを作ることに決められました。その主人公はロープの助けを借りて街を動き回ることができました。



GooglePlayをざっと見てみると、類似物は見つかりませんでした。これは開発開始の決定的な推進力となりました。 その後、既存のアプリケーションの調査で急いでいることが判明しました。 検索するとき、私はフレーズスパイダーマンとそのバリエーションのみに焦点を当てましたが、これは間違いでした。 後で、他のキーワードを使用した約12の類似のアプリケーションを見つけましたが、そのうちのいくつかは非常に人気がありました。



ゲームロジックのより詳細な構築により、基本原則から逸脱しないようにしました：ゲームは実装が簡単かつ高速でなければなりません-最初のcomに多くの時間を費やさず、後続のプロジェクトに必要な最小限の経験をすべて得るためです。 ゲームは本質的にランナーに似ていることを考慮して、私は数十の異なるレベルを構築し、1つを無限にするという形で頭痛なしでやりたいと思っていました。



ゲーム作成



さらに作業を進めるには、拡張個人ライセンスが必要でした。 それ以外の場合、モバイルプラットフォームへのエクスポートは利用できず、商用利用も禁止されています。



彼は1つの画面でスケッチを作成し、主人公の物理学に取り組み続けました。 その結果、次のことが起こりました。ヒーローが右に移動すると、ファサードとサイズが異なるランダムに生成された建物がランダムに生成されます。 建物をクリックすると、ヒーローと前述のタッチポイントを接続する条件付きロープが作成されます。 残りは、Construct 2に組み込まれた物理エンジンによって行われます。 建物のないスペースをクリックすると、条件付きロープが消え、ヒーローが接線に飛び出します。 地面に落ちたときのゲームの終わり。



その後、ヒーローが振った惨めなスティックは、ロープのより惨めな見た目（直列に接続された小さなスティックで構成される）に置き換えられました。



計画された運動の物理学を実装した後（つまり、構想されたゲームが実際にコンストラクト2を使用して実行できることを認識）、モバイルゲームに必要な最小限の詳細を紹介し始めました。

-通常のグラフィックス（無料のInkscapeをエディターとして使用しました。非常に素晴らしいベクターエディターです。初めて、そして喜んで作業しました）。

-メニュー画面。

-レベルの再起動、一時停止、アプリケーションの終了、その他すべてのボタン。

-ゲームに関する情報を含む画面。

-電話で最高の結果のアカウントとローカルストレージ。

-「コード」の最適化。

-ゲームプレイの改善、障害物の作成（ロープを引き裂く光とヒーローをノックするハト）;

-新しいレベルごとにゲームを複雑にします（ハトが速くなり、より一般的になり、ランタンが増え、建物の頻度が少なくなります）。

-異なる解像度のデバイスでのHUD要素の表示を修正しました。



大きな困難の1つは、ゲーム全体に1つの無限レベルを実装することでした。 問題は、レイアウトを無限にできないことです。 無限のランナーのようなゲームは、悪名高いフューチュラマシリーズから移動する宇宙船の原理に基づいて作られています。ヒーローは他のすべてに対して移動するのではなく、周囲のすべてのスペースがヒーローに対して移動し、ヒーローは静止します。 そのようなゲームでは、物理学自体は存在しないため、私の場合、実際には実現不可能でした。 その結果、私は最も抵抗の少ない経路を取りました-レイアウトの長さは1レベルと見なされ、それに到達すると、その動きに関するすべてのデータがヒーローから削除され、ヒーローとのスプライトが削除され、レイアウトの開始時に同じ動きの特性を持つ新しいスプライトが作成され、これが次のレベルと見なされます。



音楽を挿入し、freesound.orgから取得しました。 無料のものがたくさんありますが、ライセンスの種類に応じて支払う必要があります（合計3つあります）。 それらのいくつかについては、使用する前に著者に許可を求めるか、製品へのリンクを必ず含める必要があります。 ライセンス付きのサウンドを使用しました。ライセンスでは、何もする必要はなく、ダウンロードして使用するだけです。 その後、ほとんどのサウンドエフェクトを削除する必要がありました。ゲームの速度が大幅に低下し、この問題は簡単に解決できず、より良い時間まで延期する必要がありました。



上記のほとんどすべてが1週間半で行われました（私は思い出しますが、これは夕方ではなく、毎日10〜12時間、お茶/コーヒー/ダンスブレークで行いました）。 残りは後続の段階で完了しました。

輸出する



Construct 2で作成したゲームは、Android、iOS、Windows Phoneに移植できます。 これは、Facebook Games、Chrome Web Storeなどに加えて、現時点では興味のないものです。



もちろん、移植用のモバイルプラットフォームの選択は、私が日常生活で使用している携帯電話に従って行われたため、Androidであることが判明しました。



エクスポートにはいくつかの選択肢がありますが、Intel XDKを選択しました。 便利でわかりやすいXDK、変更のたびにアプリケーションを再インストールする必要なく、電話ですばやくテストできます。



しかし、私の場合、結果は非常に嘆かわしいことが判明しました-ゲームはひどく遅く、プレイは非常に悲しく、グラフィックスを描くことも望まれていませんでした。 私は弟のiPhoneで同じアプリケーションをテストしました-結果は巨大でした。



フォーラムの調査、パフォーマンスの問題の解決、「コード」の最大限の最適化と簡略化に多くの時間を費やし、このトピックに関して多くの記事を執筆しました。興味深い小さなことをたくさん発見しました。 たとえば、テキストによって常に変化する現在の結果（スコア）を表示すると、アプリケーションが非常にロードされるため、テキストの代わりにフォントを使用する必要があります。



その結果、物理的なBox2D Webエンジンを使用している場合にのみ、Intel XDKがこのように動作することが判明しました。 この問題を解決するために、彼らは別のBox2D asm.jsエンジンを作成しましたが、このエンジンを使用する私のアプリケーションは頑固に起動を拒否しました。 その結果、エクスポートツールとしてCocoonJSに切り替えました。 私が理解しているように、コンストラクト2はそれらとは特に協力していません（コンストラクタ自体では、CocoonJSはエクスポートツールのリストで非推奨のエクスポーターとして非表示になっています）。 ただし、物理学を使用する場合、これが唯一の有効な（正常に機能する）オプションです。 また、CocoonJSから、Construct 2のプラグインはまだ作成および更新されています。



別に、CocoonJSを介してエクスポートするときに広告を接続するには、プレミアムアカウントが必要です。 ただし、無料で入手するには、現在のプロジェクトの説明、将来の計画、プレミアムアカウントが必要な理由の根拠を記載したアプリケーションを作成する必要があります。 したがって、この問題に没頭することをお勧めします。アカウントを取得するには時間がかかります。 私の場合、約1日かかりました。



エクスポート方法の検索と調査、「コード」の最適化、電話でのテスト後のアプリケーションの改善、さまざまな可能な解像度への適応に約2週間かかりました。

私の記事がこの時間をいくらか節約することを願っています。



収益化



収益化の方法として、私は私の意見では最も単純な方法、つまり広告を使用することにしました。



私は広告をすぐにつなぐべきだと決めました。 ユーザーが以前に広告を表示していなかったものをインストールした後、無料のアプリケーションに更新を後でアップロードするのは危険です。



Android用のConstruct 2で作成され、CocoonJSを介してエクスポートされたゲームに広告を接続する唯一の方法は、Mopub + Admob広告ネットワークの束およびその他を使用することです。 MoPub広告は、CocoonJSにエクスポートするときにアプリに接続します。 ただし、この広告ネットワークは、広告（数が少なく、支払いも少ない）に加えて、各ネットワークのインプレッションあたりの最小コストを設定しながら、他のネットワーク間の仲介として機能できます。 したがって、1つのアプリケーションで複数の広告ネットワークの広告を表示でき、最も高価なものの1つが表示用に選択されます。 もちろん、AdmobはAndroidの広告ネットワークの中でも人気があります。



広告のテストでは、App Storeでの公開は不要であるため、アプリケーション開発の都合の良い段階で広告を埋め込み、テストできることをすぐにお伝えします。



広告表示形式の選択は、[戻る]または[再起動]ボタンを押したときにフルスクリーン広告で行われましたが、2分に1回しか表示されませんでした。 メニュー画面でバナーを使用しても意味がありません。長い間誰もいないからです。 動的なアーケードゲームの場合、レベル付きの画面上のバナーはさらに役に立ちません。



このスキームに従って広告を接続する場合、いくつかの機能に注意する必要があります。

1.広告をアプリケーションに統合することにより、テスト時の最初の数分/数時間を心配する必要はありません。 最初は、広告はまったく表示されませんが、まれにしか表示されません。広告が完全に読み込まれるまでに時間がかかります。

2. Admobは最も高いFill Rateの1つを持っていますが、MoPubをFillRateネットワークの仲介として使用するこのスキームの実装の場合、Admobは30〜80％に低下します。 この問題は存在し、それについて知られています。開発者はそれを修正することを約束しました。 ただし、これがいつ起こるかは不明です。



この理由から、私は待たずにこの状況から抜け出すことをかなり簡単な方法で決定しました。 明らかに、10秒ごとに誰もアプリケーションに広告を表示しません。 広告を表示するタイミング（特定のボタンを押すか、n分ごとに表示するか）を選択する方法に関係なく、2つのインプレッションの間にはかなりの時間がかかります。 2分としましょう。 構成2では、広告を接続するときに次のことを行います。AdsLoadTrigger変数を起動します。これは、広告がロードされたかどうかを示し、ロードされていない場合は、10〜20秒ごとに再試行します。 したがって、実際の広告掲載率が20％であれば、適切なタイミングで広告が表示される確率を100％に近づけることができます。



広告をつなぐのに多くの時間を費やしました。 多くの異なる広告ネットワークの分析、それぞれの登録と開発、アプリケーションへの広告の統合、テストと最適な場所と表示頻度の設定には多くの時間がかかります。 ただし、すぐに上記のパスに沿って進む場合は、数日で処理してください。

最も有望なGoogle Playでのプロモーションでは、強力な自信を持ってスタートすることが非常に重要です（セクションの先頭に到達するため）-代替プラットフォームを使用してユーザーでアプリケーションを実行し、基本的なコメントを削除し、分析用のデータを収集できます（もちろん、このような機会が考慮された場合）アプリケーション）。



ローカライズに関するいくつかの言葉：少なくともゲームの説明は、日本語、英語、韓国語、中国語の4つの言語に翻訳する必要があります。 したがって、あなたはあなたの製品のすべての潜在的なユーザーの半分以上に到達します。



もう1つの非常に重要な点は、製品の署名です。 他のサイトについてはわかりませんが、Google Playでは署名が必要です。 この操作を実行するときは、Java Development Kitバージョン1.6を使用することをお勧めします。 私は知らずに次のバージョン1.7を使用しました-その結果、Google Playからアプリケーションをインストールするときに、パッケージの署名が正しくないというメッセージを受け取りました。実際、ゲームの公開の最初の数分から、誰もインストールできませんでした。 それは不快です。 私が見つけた唯一の解決策は、JDK 1.6で既に署名された新しいapkをアップロードすることでした。したがって、このバージョンをすぐに使用することをお勧めします。

したがって、無料のレビューがプロモーションツールとして選択されました。 テーマサイトとブログのリストが選択され、ゲームの概要を含む提案が送信されました。 残念ながら、現時点では、これらのアクションの結果が何であるか、リクエストの何パーセントが成功したかは言えません。先日行われたアプリケーションの公開と同時にレビューが送信されました。
