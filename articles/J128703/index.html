コンストラクターパラメーターをMXML Flexコンパイラーに追加する

2007年4月26日。 アドビは、Flexフレームワークがオープンソースに翻訳されていることを表明しています。 このニュースは世界中のフラッシャーの間で拍手ofの嵐を引き起こし、これに基づいた根本的に新しいフレームワークが登場しますが、これはいくつかのプロジェクトを超えませんでした。



当時Flexを使用してプロジェクトを行っていましたが、プロジェクトが増えれば増えるほど、内部構造の微妙な違いが気に入らなくなりました。

Flexで作業した人は誰でも、その問題とMXMLで記述および生成されたコードの低品質を知っていると思うので、Flexコンパイラのデバイスをクリーンアップして把握するためにこのプロセスに介入することにしました。彼は、MXMLに砂糖を追加することから始め、Adobeの発案による他の機能の精神で、できる限り強力にしようとしました。 2年後、私はあなたと私の知識を共有することにしました。



まえがき



SVNリポジトリからFlexSDKプロジェクトをチェックアウトするプロセスの説明は行いません。 このプロセスの説明へのリンクを残します



この記事の執筆時点では、最新バージョンはFlex 4.5.1（4.5.1.21328）であったため、それに基づいてビルドする必要があります。



コンパイラを掘り始めたばかりの頃、私はJavaの十分な知識を持っていなかったので、常識を使わなければなりませんでした。すべての変更はNotepad ++で行われました。

お気に入りのJava IDEを使用することをお勧めします。 ブラックジャックと売春婦なしでは、 「後藤宣言」と「使用法を見つける」は非常に難しいからです。



中身は何ですか



残念ながら、1つの記事ではコンパイラ全体のデバイスをカバーするのに十分ではないため、この記事ではMXMLトランスレータというほんの一部についてのみ説明します。

それに関連するすべてのソースは、modules / compiler / src / java / flex2 / compiler / mxmlフォルダーにあります。







  • 奇妙なことに、アナライザーパッケージは、トークンの形式で解析されたXMLの分析を担当し、SyntaxAnalyzerが最も注目されます。 この段階では、コード生成は行われず、MXMLからの結果モデルの分析のみが行われます。
  • （この記事のフレームワーク内で）私たちにとって最も興味深いビルダーパッケージには、受信したトークンをAS3オブジェクトモデルに変換する一連のビルダーが含まれています。 この図は、パッケージコンポーネントを示しています。



    要するに：

    1. AbstractBuilder-他のすべてのユーザーの基本クラスには、受信したトークンからオブジェクトを生成するためのロジックが含まれています。
    2. XMLListBuilder 、 XMLBuilder 、 ArrayBuilder 、 VectorBuilder-同じクラスAS3のビルダー。
    3. InlineComponentBuilderは、「コンポーネント内のコンポーネント」、つまり <fx：Component> </ fx：Component>を記述するときに形成されるもの。
    4. PrimitiveBuilderは、String、Number、int、uint、Booleanなどの言語プリミティブを担当します。
    5. ModelBuilder 、 AnonymousObjectGraphBuilderおよびServiceRequestBuilderを使用すると、<fx：Model />などからオブジェクトを作成できます。
    6. ComponentBuilderは非常に重要なクラスです。 上記のカテゴリに適合しないMXMLタグからオブジェクトを作成するロジックが含まれています。 最も頻繁に使用されるのは、 フォーム<s：Button />、<s：List />、<someNamespace：SomeObject />の構造からオブジェクトを作成するのは彼です。
    7. DocumentBuilderは、たとえば<s：Application />などのルートタグです。
    8. HTTPServiceBuilder 、 RemoteObjectBuilderおよびWebServiceBuilder-対応するタグのビルダー。
  • domパッケージには、検出された各MXMLタグタイプのNode宣言が含まれています。 気にする人は、パッケージ構造を視覚的に表現した写真を投稿します。
  • genパッケージには、AS3コード生成用の速度テンプレート+このためのヘルパークラスが含まれています。 -keep-generated-actionscriptが指定されている場合にのみ使用され、そうでない場合は、AbstractSyntaxTreeですぐに生成が行われます。
  • langパッケージには、言語固有の定数のほか、属性、バインダーなどのハンドラーが含まれています。 StandardDefsに注意することをお勧めします。
  • Reflectパッケージには、Reflection APIと、それぞれのプライベート実装を含むTypeTableクラスが含まれています。
  • repパッケージは、MXMLドキュメントの各ノードの初期化子です。 構造：





理論から実践へ！



それ自体はそれほど多くの「理論」が判明することを期待していませんでしたが、それはあなたがどこで、何で、なぜ理解するのに大いに役立ちます。 そして、ここで、MXMLオブジェクトのコンストラクターパラメーターを指定する機能であるものに直接進みます。



AS3の多くのクラスには必須のコンストラクターパラメーターがあります。これらのクラスは、パラメーターなしのオブジェクトでラップするか、<fx：Script />タグで作成する必要があります。 ： 

<?xml version="1.0"?> <s:Application xmlns:fx="http://ns.adobe.com/mxml/2009" xmlns:s="library://ns.adobe.com/flex/spark"> <fx:Declarations> <Timer xmlns="flash.utils.*" new="1000, 1" /> </fx:Declarations> </s:Application>




新しいキーワードの選択は、AS3ではnewという名前のプロパティを作成できないためです。 予約されています。 newで渡されるものは、オブジェクトを初期化するときに角括弧で囲まれます。この場合、 

 new Timer(1000, 1);




検証に合格


バリデーターが新しい属性をスキップするには、同じ属性に追加する必要があります（たとえば、includeIn、excludeFromなど）。 テキスト検索により、この属性は最初にlang / StandardDefs.javaで宣言され、次にbuilder / ComponentBuilder.java（isSpecialAttributeメソッド）で検証されることが示されました



StandardDefsに追加 

 public static final String PROP_NEW = "new";


310行後： 

 public static final String PROP_ID = "id";




ComponentBuilderでも（isSpecialAttributeメソッド） 

 if (namespace.length() == 0 || namespace.equals(document.getLanguageNamespace())) { if (StandardDefs.PROP_ID.equals(localPart) || StandardDefs.PROP_NEW.equals(localPart) || StandardDefs.PROP_INCLUDE_STATES.equals(localPart) || StandardDefs.PROP_EXCLUDE_STATES.equals(localPart) || StandardDefs.PROP_ITEM_CREATION_POLICY.equals(localPart) || StandardDefs.PROP_ITEM_DESTRUCTION_POLICY.equals(localPart)) { isSpecial = true; } }


これで、バリデーターはMXMLタグの新しい属性を誓いません。



オブジェクトモデルの新しいプロパティを登録する


だから、私たちの属性はそうなりますが、どこかにその値を保存する必要がありますか？ このために、Model.javaがあります。

クラスにプロパティと2つのメソッドを追加することにより、rep / Model.javaのコンストラクタパラメータのゲッター/セッターを作成します。 

 protected String constructorParams; public void setConstructorParams(String value) { constructorParams = value; } public String getConstructorParams() { return constructorParams; }




AbstractBuilder.javaクラスのregisterModel（ノードノード、モデルモデル、ブール値topLevel）メソッドでこのプロパティを使用します（1617行目）。 

 protected void registerModel(Node node, Model model, boolean topLevel) { String id = (String)getLanguageAttributeValue(node, StandardDefs.PROP_ID); String newValue = (String)getLanguageAttributeValue(node, StandardDefs.PROP_NEW); // get the comment from the node and store in the model if(node.comment != null) { // if generate ast if false, lets not scan the tokens here because they will be scanned later in asc scanner. // we will go the velocity template route if(!mxmlConfiguration.getGenerateAbstractSyntaxTree()) { model.comment = node.comment; } else { model.comment = MxmlCommentUtil.commentToXmlComment(node.comment); } } if(newValue != null) { model.setConstructorParams(newValue); } registerModel(id, model, topLevel); }
