次に、コントローラーを作成する必要があります。 Springのコントローラーはアドレス変換（ブラウザーで表示されるもの）を実行し、ページ（この場合はhello.jsp）にリダイレクトします。



Ideaが基本的なコントローラーを作成したため、手動で作成する必要はありません。 念のため、彼のコードを提供し、そこで何が起こっているのかを説明します。 



 package com.springapp.controllers; import org.springframework.stereotype.Controller; import org.springframework.ui.Model; import org.springframework.ui.ModelMap; import org.springframework.web.bind.annotation.RequestMapping; import org.springframework.web.bind.annotation.RequestMethod; import org.springframework.web.bind.annotation.RequestParam; @Controller @RequestMapping("/") public class HelloController { @RequestMapping(method = RequestMethod.GET) public String printWelcome(ModelMap model) { model.addAttribute("message", "Hello world!"); return "hello"; } }




注釈から始めましょう。



@Controller



実際、これはコントローラーであると言います。 追加するものはありません



@RequestMapping("/")



-この注釈を使用して、このコントローラーのスコープ（この注釈がクラスに適用される場合）または特定の変換済みアドレス（メソッドの場合）を指定します。 また、このアドレスまたはそのアドレスがブロードキャストされる特定の要求を指定することもできます（メソッドパラメーター）。 この場合、このコントローラーはすべてのアドレスを（つまり、ブラウザーで）表示し、それらを目的のページに変換しようとするという事実について話します。 明確にするために、例を挙げます。



2つのアドレスmydomain.com/info、mydomain.com/aboutがあるとします。 "/"-"/ about"の代わりにRequestMappingで指定すると、コントローラーはアドレスmydomain.com/infoを変換しようとしません。



printWelcome（）メソッドの機能について。 このメソッドは、単にメッセージ「Hello World」をページに送信し、もちろん、ページ自体の名前を示します。 さて、ディスパッチャークラス（MvcConfig）で説明したビューアーの助けを借りて、この名前はプレフィックスとサフィックスを追加することでページへのフルパスに変換されます。
