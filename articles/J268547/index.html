仮想サーバー上のさまざまなタイプのディスクの管理

前に、 事前にインストールされたコントロールパネルを実装し、その場でライセンスを再注文する方法について説明しました 。 さらに、 APIの動作と独自のDNSマネージャーの開発についても書きました。 今日は、1つの仮想マシンでさまざまなタイプのディスクを操作した経験を簡単に分析します。

「オンザフライ」でディスクスペースを管理できる機能に取り組んで、SSDを追加注文する必要があることに気付きました。 これは、仮想サーバーがさまざまなタイプのディスクを使用することを意味します。



このアプローチは、階層型ストレージシステムでは絶対に論理的です。 最も「要求された」データは、より高速なドライブに配置できます。 これにより、データへのアクセス速度が向上し、エンドユーザーの観点からサービスの品質が向上します。



一方、低速のSATAを使用してバックアップを保存することで、多くのお金を節約できます。



どのように機能しますか？



現在、NetApp FAS6240 / FAS8040ソリューションとvCloud Directorを組み合わせて使用​​しているため、特定のディスクに個別のポリシーを適用できます（異なるタイプのディスクを管理する機能の一部として、ディスクの追加時に発生したVMware SDKでエラーが見つかりました）。



ディスクを追加するために必要なコマンドの例： 



VirtualDisk vDisk = new VirtualDisk(10240, BusType.SCSI, BusSubType.LSI_LOGIC_SAS); vDisk.GetItemResource().Address = null; vDisk.GetItemResource().AddressOnParent.Value = unit.ToString(); //unit number,   vDisk.GetItemResource().Parent.Value = bus.ToString(); //
