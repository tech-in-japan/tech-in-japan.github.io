3CX v16 Alpha 2と新しい年の計画

3CX v16 Alpha 2



まず、メリークリスマスと新年おめでとうございます。また、当社製品に対する信頼に心から感謝いたします。 2019年にあなたを失望させません！



クリスマスの直前に、非常に興味深い機能を備えた3CX v16 Alpha 2をリリースしました。 私たちのレビューを読んでお楽しみに-まだたくさんの興味深いことがあります！



WebクライアントBLFインジケーター







3CX v16 Alpha 2では、ユーザーは管理者を介さずにクライアントでBLFインジケーターを個別に構成できるようになりました。 これは、「パラメーター-BLFパラメーター」セクションで行われます。 Webクライアントで仮想ボタンを再割り当ておよび移動できます。IP電話を再設定すると、それらは電話のハードウェアBLFインジケータと同期されます。



Webクライアントは、新しいオペレーターコンソールも受信しました。これにより、キュー内の呼び出しを含む、システム内のすべての呼び出しを監視できます。 コンソールにはコールセンターマネージャーへの便利なリンクがあり、キューオペレーターを接続および切断できます。 オペレータのステータスとキューの個々の統計に関する情報は、コンソールで利用できます。 また、すべての呼び出しとキュー（キューマネージャーとして管理）の分析と統計の概要も提供します。



高度なCRM統合



コールレコードを新しいCRM連絡先として保存するために、CRMログのコール修正問題を解決するように繰り返し求められました。 そして、最終的にこれをサーバー側のCRM統合に実装しました。 Alpha 2は、サポートされているすべてのCRMシステムで通話データを記録する機能を追加します。



PBXインスタンスマネージャー3CX







3CX v16 Alpha 2には、パートナーがPBXクライアントインストールを透過的かつ安全に管理するために使用できる新しい3CXインスタンスマネージャーが導入されています（Linuxインストールのみがサポートされています）。



Managerの開発時には、開発者が使用するリソースのアクティブな監視と制御の内部テクノロジーが使用されました。 このテクノロジーが機能するためには、PBXネットワーク画面を構成する必要がないことに注意してください。 通常、Managerを使用すると、会社の常勤システム管理者が何らかの理由でこれを行うことができない場合に、3CXパートナーが正常に同行できるため、PBXの耐障害性が大幅に向上します。



データのプライバシーを重視しているため、どのデータがInstance Managerに転送されるかを常に確認できます。







監視を有効にするかどうか、および何を正確に監視できるかは、システム管理者または会社のマネージャーのみが決定します。



その他の更新



ボイスメール認識サービスを更新しました。これは、Microsoft Bing Speech Servicesの代わりに、より堅牢なGoogle Speech APIを使用するようになりました。 音声メッセージの最初の1分のみが認識されることに注意してください。 認識されたテキストは、電子メールでユーザーに送信されます。







新しいバージョンでは、nginx Webサーバー（3CXを使用）でHTTPS / 2がサポートされているため、パフォーマンスとセキュリティが大幅に向上しました。 これにより、前世代のアプリケーションおよびデバイスとの互換性が確保されます。 技術専門家は、HTTPS / 2がサーバーのパフォーマンスをどれだけ改善するかを知っています！



拡張インポートファイル（ユーザー）には、SIPトランクで指定されたDID行も含めることができます。 1人のCSVファイルから、ユーザー自身だけでなく、ユーザーに割り当てられているDID番号も「ドラッグアウト」できます。 これにより、新しい交換への構成転送が大幅に高速化されます。

3CX v16開発計画



コミュニティフォーラムでの3CXスタッフの投稿に基づいて、来年登場する新機能と改善点のリストを用意しました。 このリストは、正確かつ拘束力のあるものではなく、公式ではありません。 ただし、今後数か月の3CX開発計画をナビゲートすることができます。 便宜上、リストはシステムを構成する技術単位に分割されています。



統合、CRM、およびスクリプトインターフェイス



この方向で、最大の数の新しい機会が期待されます。



  • 特にOffice 365、Salesforce、その他の一般的なシステムとの強化されたCRM統合。

  • Call Flow Designerは、C＃、Powershell、およびPythonのスクリプトに置き換えられます。 また、CFDのインターフェイスに似たインターフェイスで、C＃でスクリプトを生成してから手動で編集することもできます。 エラーコードの操作、DTMF入力の要求、CRM REST APIへのデータ転送などが可能になります。 近い将来、このようなスクリプトの例をリリースします。 多くの複雑なまたは現在実行不可能なことは、5〜10行のコードで行われます。 新しい3CXスクリプトインターフェイスの完全な実装には数か月かかります。

  • スクリプトの開発とともに、IVRおよびCall Queue構成インターフェイスを拡張して、PBX管理者がプログラミングなしで呼び出しを処理するための複雑なロジックを構成できるようにします。

  • 同じことは、着信および発信の発信者IDを処理するとき、および着信および発信のルールで実装される予定です。 たとえば、追加の処理スクリプトを発信ルールに接続できます。



3CX Webクライアント



  • WindowsおよびMac用の3CXネイティブクライアントのすべての機能がWebクライアントに転送されます。 同時に、これらのクライアントは引き続き機能しますが、通常のSIPソフトフォンとして機能します。

  • Webクライアントインターフェイスは、新しい機能で拡張されます。 特に、1つの画面で異なるソースからの情報を組み合わせることにより、インターフェースをカスタマイズすることが可能になります。

  • Webクライアントの機能をさらに拡張するために、Chromeプラグインがリリースされる場合があります。

  • ユーザーの利便性のために、Windowsクライアントからのすべてのホットキーがサポートされます。



3CXセッションボーダーコントローラーユーティリティ



  • SBCの機能を大幅に拡張する予定です。 特に、3CXコンソール（SBC経由）からのリモート電話管理が利用可能になります。

  • SBCがインストールされているPCの電力に応じて、最大100〜150台のリモート電話のサポートが実装されます。



3CXライセンス



  • 8つの同時通話用の無料の生涯更新されたStandardエディションのリリースが予定されています。

  • 同時に、Standardで利用できるようになった一部の機能は、Proエディションに移行されます。 これは、古いユーザー（ライセンスキー）には影響しないことに注意することが重要です。 変更は新しい3CXライセンスでのみ表示されます。 3CX v16のリリース後に、完全な機能比較が公開されます。



Wordpressプラグイン、電話ファームウェア、レポート



  • Wordpressの3CXプラグインでは、ライブチャットがサイトに表示され、1人のオペレーターまたはグループがサービスを提供したり、音声通話に転送したりできます。

  • 3CXは、独自の追加テスト済みIP電話ファームウェアを引き続きリリースします。 最新のベンダーファームウェアに比べてわずかに遅れていますが、3CXでの作業を困難にする重大なエラーが含まれていないことが保証されています。

  • 特に、VoIP通話の品質に関する新しい有用なレポートが表示されます。
