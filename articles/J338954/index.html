NetApp ONTAP─すべてを棚に置く

NetApp ONTAP Cloudは、ソフトウェアサブスクリプションソリューションです。 NetApp ONTAPストレージは、クライアントデータ管理と強力なエンタープライズレベルのデータウェアハウスソフトウェアソリューションを組み合わせています。 過去1年間、ONTAPバージョン9および9.2について十分に詳細に説明されてきました。 ただし、テクノロジー自体に関する基本的な情報がある程度不足しています。特に新規ユーザーの場合はそうです。 彼らの簡単な説明はこの記事で与えられます。







Data ONTAPの意味？



技術的な詳細に移る前に、ONTAP自体の起源のいくつかのバージョンを検討したいと思います。 NetAppコミュニティでは、NetApp_Staffというニックネームの後ろに隠れている人が、それを笑い飛ばすことを選択しました。



「Data ONTAPは一種の頭字語です。 一部の頭字語はきれいです。 明らかに、目標は常に特定の単語を発明することであり、多くの場合、それらはほぼランダムに選択されます。 そのようなことがONTAPで発生しました。



頭字語自体は意味がありません。 さらに、Data ONTAPは...ビールに触発されました。 アイデアは、データが自由に流れるべきだということでした-タップで穴を開けた樽からのビールのように。



NetAppの私たちは、この名前が好きです。それは、人々がデータについてどのように考えているかを完全に説明しているからです。 必要なときにそこにいるはずです。」 これに加えて、「タップ時」は「すぐに使用または使用できる状態」を意味します。 手元に。」



この説明へのコメントで、知識のある人が金曜日にマグカップを1杯か2杯飲むのが会社のオフィスで習慣であることに気付きました。 しかし、真剣に、頭字語は「アプライアンス製品のオープンネットワークテクノロジー」の略であると書いた人もいます。



会社のシンボル（NTAP）がONTAPの一部であるという事実に関連するバージョンもありました。 まあ、それはありそうにない。 しかし、ティッカーについて言及したので、最初のリリース後の全期間のNetApp株価を見てみましょう。





NetAppよりも安定した会社を見つけるのは困難です。 2001年3月から現在まで、彼女は大きなショックを経験しませんでした。



ONTAP Cloudとは何ですか？



ONTAP CloudのCloud Storage Environment（CSE）は、次の機能を提供します。



-非破壊的な2ノードのサポートと高可用性により、予期しないクラウドストレージの障害に対処します。



-NFS、CIF、およびiSCSIをサポートする使用済みストレージシステムNASおよびSAN。



-データ重複排除、シンボリューム（シンプロビジョニング）、およびデータ圧縮のテクノロジーが使用されます。



-ゼロインパクトNetApp Snapshotテクノロジーは、追加のストレージリソースを必要とせずに、ほぼ瞬時にデータをバックアップします。



-NetApp SnapMirrorテクノロジーは、オンプレミスFASをONTAP Cloudにリンクすることにより、ハイブリッドクラウド環境を統合します。



-高度なデータ管理ツール（OnCommand Management Suite）とその監視（OnCommand Insight）は、OnCommand System Managerから管理されます。



-アプリケーションの可用性は、特別な「可用性ゾーン」（可用性ゾーン）を介してコピーすることにより保証されます。



ONTAP CloudがAWSコンピューティングおよびストレージリソースでどのように機能するか



ONTAP Cloud─Data ONTAP仮想マシンのバージョン。 さまざまなコンピューティングインスタンスでAmazon Elastic Cloud Compute（EC2）の上で実行するように設計されています。 詳細はこちらをご覧ください。 ロシア語のITの文献では、トレーシングペーパーがよく使用されます。 アマゾンウェブサービスでは、次のように定義されています。



Amazon EC2インスタンスタイプ 。 Amazon EC2は、さまざまなユースケースに最適化された多種多様なインスタンスタイプを提供します。 インスタンスタイプには、プロセッサ、メモリ、ストレージサブシステム、ネットワーク機能などのコンポーネントのさまざまな組み合わせが含まれます。 これらにより、クライアントが使用するアプリケーションに適切なリソースのセットを選択できます。 インスタンスの各タイプには1つ以上のサイズが含まれており、ターゲットワークロードの要件に応じてリソースをスケーリングできます。



ストレージは通常、AWS Elastic Block Storage（EBS）を使用します。 EBSストレージボリュームは、クラウドで実行されるData ONTAPストレージノードに相当するものを作成します。



ONTAP Cloudは、 AWS Marketplace *からOnCommand Cloud Managerを通じてデプロイされるAmazon Machine Instance（AMI）バリアントです。





AWS Marketplaceホームページフラグメント



AWSには、世界中の多くの地域にクラウドデータセンターがあります。 クライアントは、自分の要件に応じて最適なデータセンターと可用性ゾーンを自分で選択できます。 ONTAP Cloudは仮想であり、ある地域のデータセンターで動作し、必要に応じて別のリモートデータセンターに移動できます。 SnapMirrorテクノロジーを使用してデータをコピーする場合。 この場合、ONTAP Cloud環境内で管理されているデータが移動されます。



ONTAP Cloud vs. Ebs



EBSストレージは、標準サーバーのDASストレージと考えることができます。 実際、EBSはAmazon EC2サーバーに「生のストレージ容量」を提供します。



EBSには、DASが通常提供する機能よりも若干広い機能と機能がありますが、制限があり、データ複製ツールは含まれていません。



ストレージのスナップショットを提供します。 これらのスナップショットはストレージのパフォーマンスに影響を与え、顧客が支払うS3のストレージの完全で正確なコピーも作成します。



ONTAP Cloud Storageは、作業の基礎として未加工のEBSストレージを使用する管理ストレージ環境です。 ONTAP CloudはEBSストレージを引き継ぎ、Data ONTAPストレージOSを使用して管理し、実際のストレージに加えて、次の追加機能を提供します。



-いくつかのプロトコル（NFS、CIF、iSCSI）の使用。



-データ保護─スナップショットコピー、SnapMirror、SnapVault。



-ストレージ効率─データ重複排除、圧縮、シンプロビジョニング。 これらの機能により、購入する必要があるEBSストレージユニットの数を減らすことができます。



-管理─Unified Manager、System Manager、WFA、Insightなど



-FlexCloneテクノロジー。





FlexCloneテクノロジーの仕組み



NetApp Snapshotsはゼロコピー技術を使用しています。 これには、あるメモリ領域から別のメモリ領域にデータをコピーするタスクをプロセッサが実行しない操作が含まれます。 この用語は、中間バッファー間のコピー数を減らすのに役立つテクノロジーを説明するために使用されます。 詳細はこちらをご覧ください。



NetAppスナップショットはパフォーマンスに影響しません。 コピーは変更データのみを保存し、ストレージの消費を最小限にします。



ONTAP Cloudのデータ保護



ONTAP CloudにはSnapMirrorレプリケーションソフトウェアが含まれており、これによりData ONTAPプレースメント間のデータ移動が可能になります。 データは、1つの可用性ゾーンから別の可用性ゾーンに移動できます。



このために、クライアントはセカンダリアベイラビリティゾーンに他のData ONTAP環境を持つ必要があります。 このセカンダリロケーションは、別のONTAP Cloudプレースメントである場合も、NetAppプライベートストレージロケーションである場合もあります。



SnapMirrorを使用して、「データセンターからクラウドへ」、「クラウドからデータセンターへ」、「クラウド間」（または「クラウド地域間」）の指示など、Data ONTAPプラットフォーム間でデータを移動できます。



Oncommandクラウドマネージャー





NetApp Cloud ONTAPの仕組み





OnCommand Cloud Managerは、Cloud ONTAPおよびNetApp Private Storageでストレージ仮想マシンを展開および監視するためのシンプルなウィザードベースのインターフェースを提供します



OnCommand Cloud Managerは、ONTAP CloudおよびData ONTAPクラスタ化プラットフォームを含むハイブリッドクラウドストレージ管理を提供します。



各データストレージノードの構成、サポート、監視など、「データファブリック」の運用管理を簡素化します。仮想デバイスと実際の物理ハードウェアの両方です。



実際、OnCommand Cloud ManagerはONTAP Cloudの展開環境を実装します。これにより、マウスを数回クリックするだけでインストールが可能になり、各ケースに自動的にクラウドリソースが選択および割り当てられます。



ONTAP Cloudをデプロイすると、Cloud ManagerはAWS資格情報を使用して、クライアントのリクエストに一致するEC2仮想マシンとEBSストレージを選択します。



さまざまな種類のAWS認証情報には、パスワード、アクセスキー、多要素認証、キーペア、証明書が含まれます。 詳細はこちらをご覧ください。



ONTAP Cloudは、Cloud Managerを介してのみデプロイできます。 実行されるすべてのアクションを処理して、すべてのAWSリソース（コンピューティングとストレージの両方）が特定のONTAP Cloudデプロイメントに適切に割り当てられるようにします。 他の方法でONTAP Cloudをデプロイしようとすると、最初から障害が発生します。



Cloud Managerは、クラウドハイブリッド環境の制御の中心点であり、その場所に関係なくclustered ONTAP環境を管理できます。 AWSクラウドとクライアントのサイトのデータセンターのオンプレミスの両方で機能します。



ONTAP Cloudのバージョン間には主に2つの違いがあり、それらはソフトウェアの受け取り方法とインストール方法によって異なります。 OnCommand Cloud Managerは、AWS MarketplaceでAMIとして利用できます（AMIバージョンはEC2クラウド用にデプロイされています）か、NetAppサポートWebサイトからダウンロードできます。 NetAppのお客様はCloud Managerを無料で利用できます。



Cloud Managerは、Data ONTAPに基づくクラスター化されたプラットフォームで使用できます。 この環境の構成、管理、監視を提供しますが、ONTAP Cloudの展開が必要であり、ONTAP Cloudのみに適用される高度な機能（インストール、クラウドリソースの取得、クラウドで動作するネットワークのセットアップなど）が含まれています。 Cloud Managerは、ONTAP CloudとオンプレミスFASソリューション間のSnapMirror関係も管理します。



ONTAP Cloudの購入モデル



ONTAP Cloudは、AWS Marketplaceでいくつかの購入オプションで利用できます。 前払い（時間払い）で1時間ごとに支払うか、NetAppからサブスクリプションを購入できます。



毎時モデルによると、購入はAWS Marketplaceから直接行われ、ONTAP Cloudソフトウェア料金、サポート、AWSリソースで構成されます。 サブスクリプションモデルでは、顧客は6か月または1年ごとのサブスクリプションをNetApp営業担当者から購入し、Cloud Managerを介してサブスクリプションとサポートをアクティブにして、ONTAP Cloudにアクセスします。 このアプローチはBYOL（Bring Your Own License）と呼ばれます。 この場合のAmazonは、EC2およびEBSリソースの使用と稼働時間に対してのみ請求します。



ONTAP Cloudは、その作業が必要になるたびに起動できます。 1時間ごとのスケジュールでは、クライアントはONTAP Cloudでのリアルタイムの作業に対してのみ支払いを行い、必要がなければいつでも切断できます。



ただし、現在のデータを維持する必要がある場合、支払いは常に請求されます。それ以外の場合、データは失われます。 つまり、サブスクリプションモデルでは、クライアントは長期的なデータ管理に対して支払いますが、ONTAP Cloudの実行中に実際に使用されるAWSリソースに対してのみ支払います。 ユーザーが利用できるONTAP Cloudオプションは次のとおりです。





Cloud ONTAPサービス購入モデル



- 探検 。 比較的小さなリソースを必要とするアプリケーション向けに事前構成されたソリューションでData ONTAPのフルパワーを提供します。 最大2TBのストレージ容量を提供します。 EC2のM4.XLおよびM3.XLインスタンスに対応します。



- 標準 。 柔軟なパフォーマンスとリソースを備えたData ONTAPの全機能を提供します。 幅広いアプリケーションをサポートし、最大10 TBの容量があります。 EC2のM4.2XL、M3.2XL、およびR3.XLインスタンスに対応します。



- プレミアム 。 高リソースアプリケーション向けのData ONTAPのフルパワーを提供します。 最大368 TBの容量をサポートします。 EC2のR3.2XLインスタンスに対応します。



- 高可用性（HA） 。 HA構成では、ONTAP Cloudは最大360 TBをサポートします。



-BYOL 。 この構成では、ONTAP Cloudには高度なオプションがあり、最大360TBおよびすべてのタイプのEC2インスタンス（M3.XL、M3.2XL、R3.XL、R3.2XL、M4.XLおよびM4.2XL）をサポートしています。



ONTAP Cloud ExploreおよびStandardは、NetApp ONTAPのフルバージョンです。 購入には、Data ONTAP機能の主要かつ最も重要な機能の多くが含まれます（SnapShot Copies、FlexCloneボリューム、SnapMirrorデータ複製、データ重複排除など）。 OnCommand Cloud ManagerとNetApp Software Support Planも含まれています。



ONTAP Cloud High Availability（HA）─クラウド内の2ノードソリューションとして構成できるONTAP Cloudバージョン。



アプリケーションを機能させるために不滅の環境が必要な場合、ONTAP Cloud HAを使用すると、フェイルオーバーテクノロジーをサポートする2ノードソリューションを構築できます。 ノードの1つで計画外の障害が発生した場合、および計画されている場合（たとえば、システムを停止せずにアップグレードする場合）のストレージの操作をサポートします。
