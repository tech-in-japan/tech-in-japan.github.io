RAIDアレイのHGSTドライブ

こんにちは、友人と同僚。 今日は、RAIDアレイでHGSTドライブを使用する利点と、ソリューションがデータセンターの円滑な運用を保証し、金融および評判のリスクを最小限に抑える方法についてお話したいと思います。







エンタープライズユーザーは、信頼性の高いデータ保護、ダウンタイムの最小化、パフォーマンスの維持のために、RAIDアレイを備えたシステムを構成することを好みます。 そのため、クラス最高の信頼性、一貫したスループットと電力を提供するHGSTエンタープライズクラスのドライブを選択しています。 ただし、世界で最も信頼性の高いドライブであっても、データ損失のリスクを完全に排除することはできません。 RAIDシステムは、これらのリスクを最小限に抑える長い道のりを歩んできましたが、それでも、ディスク障害の結果として発生する復旧プロセスは依然として避けられません。 これに関して、多くの問題が発生します。時間の損失、パフォーマンス、およびデータセキュリティは、リビルドアシスト機能のおかげでHGSTが対処する予定のタスクです。



エラーの影響を受けないストレージテクノロジーはありません。 ドライブとフラッシュメモリが誤用される可能性があります。



HGSTは多くの診断機能を使用して、考えられるエラーを評価します。エラーに関する情報は、さらなる分析と考えられる修正アクションの特定のためにRAIDコントローラーに送信されます。

HGSTドライブは、エラー間の平均時間が2時間に等しいことから明らかなように、エラーの可能性を最小限に抑えます。 軽微なエラーは多くの場合、修正または再配布できるため、データ損失が発生しないか、最小限に抑えられます。 ただし、データがいっぱいのディスクには、ディスク障害として識別できる重大なエラーが表示される場合があります。 この場合、RAIDコントローラーは対策を講じる必要があります。ディスクエラーはスタンドアロン操作に転送され、アレイは機能が制限されたモードになり、長期的な誤動作による追加のデータ損失の可能性を減らします。 再インストールまたはホットスワップドライブでデータが再構築されている間、RAIDコントローラーは、XOR操作と呼ばれる回復プロセスを通じて、アレイ内の残りのディスクのパリティデータを使用してデータへのアクセスを提供し続けます。

そして、それは本当に機能します。 欠点は、ディスクの内容の復元に時間がかかる可能性があることです。 アレイが低速状態で実行されているという事実にもかかわらず、データを復元し、同時に要求に応答する必要があります。 パフォーマンスの観点から、RAID構成は共有ストレージとしてよく使用されます。 これは、回復による生産性の低下が多くの労働者の生産性に影響を及ぼす可能性があることを意味します。



RAIDを再構築します。 すべてのRAIDユーザーが、リカバリプロセス中の二次障害でデータ損失のリスクがどれほど高いかを理解しているわけではありません。 回復時間を短縮すると、このリスクを減らすことができます。



チューニング時間が増加する理由



アレイの容量がメガバイトとギガバイトで測定されたとき、RAIDアレイの再構築に必要な時間は許容できると考えられていました。 それでも、再構成に必要な時間を計算するための基本式を考えると、テラバイト配列に切り替えることの実用的な結果はすぐに顕著になります。



容量（Mb）/平均データレート（Mb / s）=チューニング時間（s）



平均データ転送速度が110 Mb / sの通常の3.5インチハードドライブで3 TBボリュームを再構築する場合、再構築には19,108秒（または5.3時間）かかります。 これはRAID 1アレイの典型的な状況です。これが怖くない場合は、ディスクの1つが切断されたときに、より複雑なアレイオプションが「機能制限モード」で機能することを覚えておいてください。 これにより、ボリュームのパフォーマンスが低下し、再構築時間がさらに長くなります。 実際には、RAID 5の再構築速度は約10Mb / sに簡単に低下する可能性があるため、このプロセスには数日かかり、特定のエンドユーザーサービスレベルの保証に従う必要があるほとんどのデータセンターでは受け入れられません。



ディスク容量が大きくなると、再構築にかかる時間とデータ損失のリスクが大きくなります。 たとえば、12 TBのディスクで構成されるRAID 5アレイの再構築時間推定プログラムを使用して、3 TBのディスクと5 TBのディスクで構成される1つのクイック自動バックアップイネーブルデバイスと40 Mb / sのまともな再構築速度を使用すると、データ損失の年間確率を確立できます5TB以上の容量のあるディスクアレイで54％高い。これは、主に再構築に約13時間必要であるためです*。 追加の冗長性を提供しないRAIDアレイで2番目のディスクに障害が発生した場合、ボリュームの内容は失われます。 RAID 6は、2番目のドライブ障害のリスクを減らすためによく使用されます。







*この例は、 www.memset.com / tools / raid- calculatorにある調整時間計算機を使用してモデル化されています



シミュレーション条件：平均データ転送速度40Mb / s、アレイ内の12個のディスク（パリティディスクおよび予備の高速自動切り替え用デバイスを含む）、5年間の保証、および7日以内にディスクを交換する可能性。



要約すると、ディスク容量とチューニング時間の間には直接的な関係があり、正確には容量が大きいほど、アレイ機能が制限されたモードでの動作条件下でチューニングプロセスにかかる時間が長くなることに注意してください。 この問題を解決するために、HGSTは最新のエンタープライズデバイスで、すべての業界標準を満たすRebuild Assistを提供しています。 アレイの再構築に必要な時間を短縮するだけでなく、データ損失のリスクを減らし、アレイの完全な操作性を迅速に復元します。
