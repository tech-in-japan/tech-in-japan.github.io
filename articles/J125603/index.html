予算のAndroidスマートフォンの第二の風

適度な量のRAM（私の場合は160）を備えた新しいスマートフォンの所有者は、すぐに仕事でブレーキを感じ始めます。 文字通りいくつかのプログラムをインストールしましたが、それらの間の切り替えにはすでに顕著な遅延が伴い、クイック設定の呼び出しはもはや「クイック」ではありません。



再起動後、すべてが飛んで、15分間の作業の後、アクティブなプログラムのセットは同じように見えますが、感情は鉄片が死にかけているようです。

-コマンドに対する反応が遅い。

-スムーズなスクロールではありません。

-小さなアプリケーションでも長時間起動。

-インターネットに接続した後、表示されるまで長い休止。

-そして、最もストレスの多い瞬間に、最も生産性が高く、タスクを素早く切り替える必要があるとき、スマートフォンはあなたに応答しなくなり、自分自身に入り、個人的なことについて非常に真剣に考えるふりをします。



カットの下では、ルート権限と2つの小さなプログラムを使用してスマートフォンに新しい命を吹き込む方法に関する個人的な経験が少しあります。





私の目標：

-頻繁に使用する3〜4個の小さなプログラム（カテゴリ：連絡先、メモ、タスク、設定、カレンダー）が常にRAMにあるようにします。 そのため、それらの呼び出しと切り替えは即座に行われます。

-携帯電話の状態の変化に奇妙なブレーキやハング（モバイルインターネットのオン/オフ、ロック解除、通話時間、SMS到着）が伴わないように。

-必要なプロセスの最小数が起動されるように（バッテリー寿命を節約するため）;

-システムを長時間操作した後でも、デスクトップとスクロールリスト間の移行が常にスムーズになるようにします。

-1つまたは2つの大きなアプリケーションを実行するのに十分なRAMが常にある（または、すぐに解放する機会があった）。





これが私を助けたものです

（Samsung GT-i5700 Galaxy Spica、160MB RAM、Android 2.1）：



1） ルート権限。 これにより、各デバイスには独自のストーリーがあることを理解しています。 ケースを探してください（model + os）。



2） Auto Memory Manager （root、無料、1.1M）/ ロシア語のレビュー -すべてのしきい値を最小に設定します （2、3、4、5、3、12など）。

これにより、次のことが可能になります。

-空きRAMで十分な場合は、必要なソフトウェアをアンロードしないでください。

-使用可能なRAMをより完全に使用して作業します。

-システムが他のアプリケーションの事前のアンロードに時間とリソースを浪費しないようになったため、新しいアプリケーションの起動を高速化できます。



3） 自動起動（root、$ 0.99、190k）/ ロシア語のレビューは最も重要なツールです。 この小さなプログラムのおかげで、システムでさまざまなイベントが発生したときに起動するさまざまなアプリケーションを離脱させることができました。 システムの起動時に起動されるプログラムのセットを100％制御できるのは彼女でした（多くの自動実行マネージャーを試しました）。

さまざまなイベントに対するすべてのプログラムのほぼすべての反応を解き、明らかに有用であると判断したもの、または害が見られなかったもののみを残しました。 設定を有効にするには再起動が必要なようですが、よくわかりません。

このハッキングの後、システムの起動が速くなり、接続後のインターネットが遅滞なく表示され、最も予期せぬ大胆なアプリケーション（Evernote、Android Marketなど）がバックグラウンドでランダムに実行されなくなり、起動されたすべての有用なアプリケーションがアンロードされました。

大量のRAMがある場合、システムのデフォルトの動作はほとんど正当化され有用です（ユーザーが最も必要とする可能性のあるバックグラウンドで実行するため）。しかし、私たちの予算の場合、この動作は単にシステムパフォーマンスを殺します。 これが修正したばかりです。



重要です

[戻る]ボタンの1つの機能 ：アプリケーションの操作時にナビゲーションチェーンに前の画面がない場合、またはこれが最初の画面である場合、[戻る]を押すとアプリケーションが閉じられます（実際、ステータスが空になり、メモリから削除する最初の候補になります。上記のシステムで設定されたしきい値は、非常に頻繁にアプリケーションがすぐにダウンロードされます）。

プログラムを常にメモリに残し、可能な限り高速に切り替える必要がある場合-常にホームボタンを押すだけでデスクトップに戻り、常にタスクスイッチャー（ホームを長押し）または特別なランチャー/スイッチャーで実行中のアプリケーションを切り替えます例： SwipePad



オプショナル

フライトを制御するには、空きメモリの量、実行中のプロセスを表示でき、不要なプロセスを完了してメモリを解放できる優れたタスクマネージャが役立ちます。

ホワイトリストを作成し、ウィジェットをタップしてこのリストにないものをすべて削除する機能を追加で備えているESタスクマネージャーからアプローチされました。 また、設定にチェックマークを付けて、スマートフォンをロックしたとき（画面がオフのとき）にそのようなクリーニングが自動的に行われるようにすることもできます。



UPD 08/08/11

また、有用なユーティリティFMR Memory Cleaner （無料、223k）を発見しました。これは、デバイスの長期動作中にメモリに蓄積されたガベージと対応するブレーキを取り除くのに役立ちます。 再起動と似たような動作をしますが、プログラム的にすべてのアプリケーションが過負荷になり、その後手動で起動する必要があるものもありますが、すべてがはるかに速く動作し始めます。

以前は、スマートフォンのオン/オフを切り替えるだけでその方法が役に立ちました。



ボーナス

その結果、常に実行されるプロセスが少なくなり、不要なアプリケーションのバックグラウンドでのアンロード/ダウンロードに不要な費用がないため、スマートフォンのバッテリー寿命は長くなります。 また、デバイスが望んでいることだけを実行し始めたときに、デバイスを快適に制御できるようになります。
