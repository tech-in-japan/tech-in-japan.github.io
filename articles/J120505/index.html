ウェブサイトの最適化。 技術的基盤。 パート2

前回の記事では、検索エンジン最適化の基本的な要件を満たすことにより、検索エンジンでWebリソースのプロモーションを成功させるために何をする必要があるかを説明しました。



今日、サイトを開発する際に観察しなければならない技術的特徴に注意を向けたいと思います。 それらに対してリソースを確認してください。 これは今日の私たちを助けるので、あなたを助けます。 簡単かつ便利に作業できるように、すべてをヒントのセットにまとめようとしました。



したがって、肉眼で見えるサイトの外部要因（構造、テキスト、デザイン）に加えて、検索エンジンによるインデックス作成を改善するためにサイトを適切に構成するには、サイトで行う必要のある多くの技術的手段があります。 最も重要なものをリストします。



メインホストの正確な定義。 ミラーの除外。



www.site.ruなど、サイトのメインアドレスを決定する必要があります。 この場合、WWWを使用せずにアドレスを要求すると、サーバーが301番目の応答を返し、サイトのメインアドレス（WWWを使用）にリダイレクトするようにサーバーを構成する必要があります。



また、メインページの同義語を検索インデックス（www.site.ru/index.phpおよび他の同様のもの）に含めることを除外する必要があります。なぜなら、完全なIDにもかかわらず、これらは検索エンジンにとって異なるページであり、最も重要であると考えているのは彼だけが知っているからです。

メインサイトのコンテンツを完全にコピーする他のドメインがないことを確認してください。 それぞれについて、robots.txtのディレクティブを使用する必要があります。次に例を示します。



User-Agent: *

Host: www.trubmaster.ru







有効なサーバー応答



サイトを宣伝するときに通常使用されるサーバーの主な回答は、301（ページが永住のために別のアドレスに移動しました）と404（そのようなアドレスを持つページはありません）です。



最初の回答は、メインホストを決定する場合（上記を参照）、およびアドレスを新しいページに変更したページからユーザーと検索ロボットをリダイレクトする場合に使用されます。 たとえば、サイトを再設計し、すべてのページが新しいアドレスを受信した場合、新しいアドレスを示す同じ301番目の応答コードを古いアドレスに与えることをお勧めします。 この場合、たとえば、ページが古いアドレスで獲得したリンクの重みの損失は最小限に抑えられます。



存在しないページには404を使用する必要があります。 このようなページでは、誤ったサイトではなく、サイトの他の重要なセクションにアクセスする提案を含むナビゲーションリンクを表示できます。 そのようなページの正解が存在することにより、検索エンジンのインデックスに意図的に存在しないページが含まれないことが保証されます。



正しいアドレス（URL）を設定する



各ページには固有のアドレス（URL）が必要です。 最新の検索エンジンアルゴリズムは、音訳でラテン語で入力されたロシア語を含む住所に含まれる単語を完全に認識し、検索結果でサイトをランク付けするときにも考慮します。 サイトの構造内のドキュメントの場所が明確に表示され、そのコンテンツが理解できるURLを作成します。 このようなアドレスは、人間が読み取れるURL（CNC）と呼ばれます。



不正なURLの例： www.site.ru/page.php?id=12313&brid=1536

適切なURLの例： www.site.ru/holodilniki/zanussi



Robots.txt



robots.txtファイルを使用して、サイトのセクションのインデックスを作成する必要のないディレクティブを検索ロボットに送信することができます。 なぜこれが重要なのですか？ 実際、特にサイト（オンラインストア）に製品カタログがある場合、一部のページは同じコンテンツを複製します。 これらは、ソート時に表示されるページであり、さまざまなフィルターが含まれます。 また、ページに表示されるさまざまなリストの要素の数を明確にした後、検索結果ページとインデックス作成からページを閉じることも必要です。



実際、そのような各ページには独自の一意のURLがあり、それらのページで生成されるコンテンツは、カタログのメインページに投稿された情報を複製します。 その結果、多数のカタログ要素を持つサイトでは、数百および数千の完全に不要なページがインデックスに追加され、関連するページを選択するときに検索ロボットを「混乱させる」だけで、サイトのすべてのページに関する情報を収集する作業が遅くなります。



最後に、robots.txtファイルでは、さまざまなサービスフォルダーおよびコンテンツ管理システムのフォルダーのインデックス作成を禁止する必要があります。



その結果、次のようなファイルが必要になります。



User-Agent: *

Host: www.site.ru

Sitemap: www.site.ru/sitemap.xml

Disallow: /katalog/tv/cost-sort/

Disallow: /katalog/tv/name-sort/

Disallow: /katalog/tv/50/

Disallow: /admin/

Disallow: /search/

検索エンジンがサイトに正しくインデックスを付けるのに役立つ別の非常に便利なツールは、sitemap.xmlファイルを作成することです。 このページには、サイトの各ページと、インデックス作成のための検索ロボットの推奨事項が含まれています。



SITEMAP要素の操作の原則の詳細な説明は、Yandexヘルプ： help.yandex.ru/webmaster/?id=1007070に登録されています。

このファイルを操作するためのいくつかのヒント：

  • ファイルへのリンク（複数ある場合はファイル）をrobots.txtに登録する必要があります。
  • ファイルに最終更新の実際の日付を含めることをお勧めします（たとえば、[保存]ボタンを押すことで、メインページとカタログページの場合、カタログを更新するという事実は、ページを更新するためのシグナルとしても役立ちます。また、ページの変更日を更新する必要もあります。発行は自動的に生成されます。たとえば、ニュースメッセージを追加したり、以前に入力した編集を行った後、ニュースフィードページが自動的に更新されます。
  • 実際の既存のページのみをファイルに含める必要があり、同じページの重複は、異なるアドレス（「技術」を含む）およびファイルの異なる部分の同じURLの重複の両方で除外する必要があります。


HTMLコード



HTMLは、Webページ用のハイパーテキストマークアップ言語です。 サイトコードはその上に「書き込まれ」、その結果、検索ロボットによって読み取られます。 したがって、このコードの品質とそれが伝えるものは、内部Webサイト最適化の非常に重要なポイントです。



HTMLコードも技術的基盤に起因する可能性がありますが、正しいHTMLコーディング自体の問題は非常に膨大で深刻であるため、個別に説明するのは理にかなっています。 ただし、多くのオプティマイザーはコードをあまり重視していないことに注意してください。 特に、これは、検索エンジン自体が、最も一貫性のない混chaとしたコードでさえ動作できるように検索ロボットを設定しようとするためです。さらに、人気のある標準CMS（Bitrix、Joomla、Drupalなど）のコードでも動作します。 ただし、実践が示すように、十分に開発されたHTMLコードを持つサイトは、長い間検索結果で最高のインジケータを持っています。



一般に、HTMLコードには次の要件を策定できます。

  • HTMLコンプライアンス
  • コンパクト-最小の「重量」を確保し、インデックス作成と読み込みの速度を向上させる
HTML標準への準拠により、コードの正しいインデックス付けと検索ロボットによるその復号化が保証されます。



コードのコンパクトさは、技術的に高度な設計（不要な「ベルとホイッスル」なし）とその適切なHTMLレイアウトによって保証されます。 残念ながら、適切で正しいレイアウトがまだ非常にまれです。 非常に有名で大規模なスタジオのみが、適切な品質レベルで作品を絶えずリリースしています。



この章では、独自のリソースのコードを評価できるコードに関するいくつかの一般的な推奨事項を示します。



レイアウトテクニック


サイトのHTMLコードは、DIVテクニックに従って作成する必要があります。

テーブルレイアウトの使用は許可されていません。 テーブルは、実際にはテーブルの設計、または特別な場合にのみ使用できます。



HTMLでコンテンツページを構築するための一般的な構造


以下は、典型的なサイトページのHTMLコードを構築する一般的な概要です。 このスキームを使用すると、DIVメソッドを使用してHTMLコードを作成するときに、いくつかのブロックで作業していると仮定した場合に、コード内のブロックの出現順序を確認する必要があります。 各ブロックは、特定のHTMLコードの「一部」です。

  • 帽子
  • メニュー（LI番号なしリストタグでフォーマット）
  • メインコンテンツブロック（H1見出しとPタグ内のテキスト+ LIリスト）
  • セカンダリコンテンツブロック（Pタグ+ LIリストにH2-H6ヘッダーとテキストを含む）
  • 追加のメニューと補助情報ブロック（広告、他のセクションの発表、他のリソースへのリンクのブロックなど）
  • サイトの地下（著作権、カウンター）
コードのコンパクトさ。 CSS Javascript カウンター


HTMLコードを記述するときは、コードの先頭にできるだけ近い直接テキストコンテンツの最大限のコンパクトさと近似を実現する必要があります。 コンパクトでないコードは、検索ロボットが本当に意味のあるテキストのインデックスを作成する前に、かなりの量の不要なコードを「食べる」という事実につながります。 その結果：

  • このページを保存するために、ロボットクォータが検索エンジンデータベースに割り当てられます。
  • ページでインデックス付けされたテキストの合計量から有用なキーワードの割合を減らします。
一般的な場合にコードをコンパクトにし、テキストブロックを最上部に出力する方法：

  • レイアウト方法DIVの使用。
  • HTMLコードの行と余分なスペースの間の不要な「ボイド」を防ぎます。
  • コードで直接さまざまなスタイルとクラスの使用を最小限に抑えるには、すべての基本要素と標準要素を個別のCSSファイルに記述する必要があります。
  • スタイルとクラスへの参照を使用する場合は、できるだけ短く、同時に理解可能な識別子名を使用してください。


必要に応じて、個別のファイルを取り出して接続する必要があります。

  • レイアウト要素のすべてのスタイル-CSSファイルの形式。
  • JavaScriptスクリプトはすべてJSファイルの形式です。
そして最後に、すべてのカウンターをNOINDEXタグで囲む必要があります。これにより、HTMLコードのこれらのフラグメントのインデックス作成が禁止されます。



HTMLコードのテキストのデザイン


サイト上のすべてのテキストは、条件付きでメインコンテンツ、メニューブロック、および補助テキストに分割できます（たとえば、広告リンクバナー、会社の著作権のある地下室ブロックなど）。 メインコンテンツと補助テキストのブロックのテキストのデザインのルールは同じです。 メニューブロックには独自の特性があります。

メニューブロックの設計


すべてのメニューブロックは、リストタグ（UL、LI）としてフォーマットする必要があります。 リストタグ内にリンクが配置されます。 メニュー項目を囲むDIVタグ内でさまざまなスタイルを定義することにより、スタイル上の区別を確保する必要があります。



:

DIV UL LI A /A /LI /UL /DIV







メインコンテンツと補助メニューブロックのテキストを作成する


A.テキストをフォーマットするとき、タグにテキストを直接配置することはお勧めできません。
  • DIV（例：DIVテキスト/ DIVはここに配置されます）
  • フォント-このタグはHTMLコードから完全に除外する必要があります
B.次のタグのいずれかをテキストの周囲に直接配置する必要があります（原則として、テーブルタグにリストタグをネストする可能性を除き、これらのタグはすべてネストされません）。

  • H1-H6-見出し用
  • P-ほとんどすべてのテキストのデザインのメインタグ
  • UL、OL、LI-リストの設計用
  • TH、TR、TD-表形式のセル用
  • PRE-事前にフォーマットされたテキストを表示するための追加タグ。例外的な場合に使用できます

ページメタの説明



サイトの各ページは特別なメタタグ（TITLE、DESCRIPTION、KEYWORDS）で記述できます。これにより、Webドキュメントの構造でより正確に識別できます。



TITLEタグは、サイト上のテキストに適用されるタグの中で最も重要なタグです。 明示的な形式では、TITLEに入力されたテキストは検索結果に表示され、一部のインターネットブラウザーではウィンドウタイトルとして表示されます。 TITLEタグには、ページのコンテンツをできるだけ正確に説明する簡潔なテキストを入力する必要があります。 ウェブサイトページのデフォルトのタイトル生成テンプレート：[現在のセクションの名前]。 [親セクションの名前]。 [一意の接尾辞]。



一意の接尾辞は、検索エンジンがドキュメントをインターネットドキュメントの配列全体で一意として強調表示できるようにするテキストです。 通常、会社の名前に加えて、主な活動の説明を含む2〜3語で構成されます。



DESCRIPTIONタグとKEYWORDSタグは、90年代後半から2000年代初頭にかけて影響力を失いましたが、その使用と適切な開発が引き続き推奨されます。



KEYWORDSタグには、ページのコンテンツを正確に説明するキーワードをリストする必要があります。 各ページに固有のセットを選択することをお勧めします。余分なものを書く必要はありません。 15〜20語に制限することをお勧めします。 また、プロモーションに最も必要と思われるフレーズをページからページに「ドラッグ」する必要はありません。このフレーズに最適なページのみを定義することをお勧めします。



DESCRIPTIONタグでは、ページのコンテンツに関する3〜5の関連文を記述する必要があります。 「魚製品のカタログ。 ベイトカテゴリの激怒のリスト。 釣りオンラインストアベストオンザライトフィッシング。 また、コンテンツに完全に一致する一意の説明を各ページに作成することも重要です。 また、このタグのコンテンツは、GoogleおよびYandexの検索結果でページの説明として表示されることがあるという事実を考慮する必要があります。



数百ページのサイトのメタ記述の開発は非常に退屈な作業であることに注意してください。 さらに、これらの説明を何らかの形で統一することが望ましい。 この場合、CMSはそのようなメタ記述のテンプレートを作成することが可能になります。 残念ながら、今日ではすべてのCMSでこれらのメタタグを簡単に編集できるわけではなく、テンプレートを作成できるのはごくわずかです。 そのため、サイトを作成する段階で、サードパーティの開発者を引き付ける場合は、通常のページに対してメタタグがどのように形成されるかをすぐに話し合うことをお勧めします。
