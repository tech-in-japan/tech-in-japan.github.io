Linux FoundationはGraphQLとCephの資金を確立しました-なぜ必要なのか、それらに何を期待するのか

非営利のコンソーシアムであるLinux Foundationは、 GraphQL FoundationとCeph Foundationという 2つの新しい基金の設立を一度に発表しました。 次に、それぞれについて詳しく説明します。





/写真ロデリック・アイメ CC BY-ND



新しい組織が必要な理由



Linux Foundationの主な目標は、 Linuxの促進と標準化です。 ただし、このコンソーシアムは、オープンソースエコシステム全体および関連製品も開発しています。 最近、GraphQLクエリ言語とCephオブジェクトストレージネットワークが追加されました。



GraphQLは、2012年にFacebookによって開発されました。 これは、データを要求する方法を説明する構文であり、サーバーとの対話に使用されます。 REST APIの代替として作成されました。 Cephに関しては 、2007年に登場し、7年後にRed Hatがプロジェクトを引き継ぎました。 当初、このソリューションはスーパーコンピューター向けに開発されましたが、クラウドインフラストラクチャの展開にCephが使用されるようになりました。 2017年のデータ（ PDF ）によると、CephはOpenStackのクラウドの3分の2を使用しています。



一般に、過去3年間で、GraphQLとCephは、ITの巨人から小さなスタートアップまで、さまざまな規模の企業で人気を博しています。 たとえば、Graphraユーザーには、Courstra、GitHub、The New York Times、Twitter、Pinterestなど、何百もの組織があります。 Cephのオブジェクトストレージネットワークは、クラウドプロバイダー（Rackspace、Linode）、大手金融機関（Bloomberg、Fidelity）、自動車メーカー（BMW）などで使用されています。



Linux Foundationはこれらのソリューションの可能性を認識し、それらをサポートすることを決定しました。 このコンソーシアムは、2つの新しいGraphQL FoundationとCeph Foundationを設立しました。



GraphQL Foundationが行うこと



GraphQLを中心とした技術エコシステムの開発を加速するために、別の基金が作成されました。 GitHub、Netflix、Shopify、Twitter、Pinterestなど、Linux Foundationと多数の企業が財政的および組織的なサポートを提供します。 このツールの作成者の1人であるリーバイロンは、 この基盤がGraphQLが業界標準になるのに役立つことを望んでいます。



これまでのところ、GraphQL財団は結成の過程にあります-基金の利害関係者の最終リストは後で承認されます。 その後、技術委員会とマーケティング部門の代表者の選択が開始されます。 このすべてを数か月以内に完了する予定です。 財団がGraphQLプロジェクトの作業を開始した後。 それらのリストはGitHubで入手できます 。



Ceph Foundationの機能



Ceph開発基金には、Canonical、Intel、Red Hat、SUSEを含む30人のIT巨人が含まれています。 基金の目標と目的は、GraphQL Foundationの場合と同じです。開発者を不要なタスクから解放し、製品の周りにエコシステムを形成し、開発を加速します。



Canonical、Amihan、Red Hat、ProphetStorなどのファンドメンバーは、彼らの個人的な経験と専門知識が、機械学習システムやアプリケーションコンテナを操作するための機能など、Cephに多くの改善をもたらしていることに気付いています。 Linux Foundationからのサポートは、ストレージネットワークのさらなる開発の「ドライバー」となります。



おそらく、この新しい基金は、大規模なITインフラストラクチャでテクノロジを使用するときに発生する問題の解消にも役立つでしょう。 たとえば、メモリ不足（OOM）エラーに関連する問題。 Cephコミュニティには、それらを解決するのに十分なリソースがありませんでした。 今、状況は変化しなければなりません。



期待すること



Linux Foundationは、過去に多くの基盤をすでに作成しています。 たとえば、コンテナのプロモーション用のCloud Native Computing Foundationや、映画およびアニメーション業界でオープンソースソフトウェアを宣伝するために設立されたAcademy Software Foundationなどです。 一般に、基金はすでに80以上のオープンプロジェクトをサポートしており、現在も積極的に開発中です。

しかし、これらすべてにより、Linux Foundationポートフォリオには比較的失敗したプロジェクトがあります。 たとえば、Tizenオペレーティングシステムの展望は、プラットフォーム自体の欠陥のためにITコミュニティからは 「あいまい」と見なされています。 悪いインターフェイスは特に強調表示されます。



一方、CephとGraphQLの場合、すべてがより良くなる可能性があります。 Linux Foundationメンバーの数は日々増加しており、基金はプロジェクトをサポートするために資本を増やしています。 GraphQL FoundationとCeph Foundationには、積極的な開発のあらゆる機会があります。
