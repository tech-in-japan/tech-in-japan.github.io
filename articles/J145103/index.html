新しいOSのリリース-新しい認定と試験のリリース

同僚、ようこそ！

ほんの数日前、新しいMicrosoft製品のリリースプレビューがダウンロード可能になりました： Windows 8およびWindows Server 2012。



私はすでに最新のクライアントOSに切り替えて、2番目のサーバーOSであるWindows Server 2012をインストールしました。以下のフレームから明らかなように、MetroインターフェースとTouchインターフェースの素晴らしさを味わっています。







新しいOSには多くの喜びがありますが、情報技術の専門家としての私にとって、新しいOSのリリースは、新しい認定とそれに対応する試験の迅速なリリースを意味します。





まず、比較的最近、ITプロフェッショナル向けのマイクロソフト製品の認定システムに変更があったことを思い出したいと思います。

最も関連する図を次の図に示します。







そして、MCITPとMCSEの間に混乱がないように、以下を明確にします。



MCSA / MCSEの最初のバージョンはかなり前であり、Microsoft認定システム管理者/エンジニアとして解読され、Windows NT4.0 / 2000/2003製品に属していました。 また、インターネット（IISサーバー）、メッセージング（Exchange）、セキュリティ（ISA）などの追加技術の専門化もありました。

DBMSスペシャリスト-MCDBA、Microsoft認定データベース管理者には別のステータスがありました。

HelpDeskスペシャリスト向けの別のステータスがWindows XPクライアントシステムに導入されました-MCDST、Microsoft Certified Desktop Support Techician。



次の進化は、認定システムの変革と、MCSAとMCSEのMCTSとMCITP、Microsoft Certified Technology SpecialistとMicrosoft Certified IT Professionalへの置き換えでした。

このアクションの特異性は、特定の製品やソリューションに合わせて認定がより専門的で、狭くなり、調整されるようになったことです。 このロジックに基づいて、MCSAおよびMCSEはMCITP：Server AdministratorおよびMCITP：Enterprise Administratorに置き換えられました。 MCDBAのステータスと理解度も考慮してください-MCITP：Database Administratorに置き換えられました。 クライアントオペレーティングシステムには、Windows Vistaのコンシューマーサポート技術者とエンタープライズサポート技術者の2つのMCITPステータスも表示されます。 Windows 7の場合、次のMCITPステータスがさらに後で表示されました。Windows7のEnterprise Desktop AdministratorおよびWindows 7のEnterprise Desktop Support Technician。



そして最後に、しばらくしてMCITPステータスがやり直されたMCSAおよびMCSEステータスによって補完され、略語の意味がMicrosoft Certified Solutions AssociateおよびMicrosoft Certified Solutions Expertに変更されました。 マイクロソフトからのメッセージで述べたように、クラウドインフラストラクチャの使用と開発はますます自然であると思われるため、新しい「古い」認定はクラウド時代に合わせて特別に設計されました。 しかし、MCTSとMCITPのステータスは、個々の製品と技術の専門分野として、去ることが約束されていました。 ただし、Windows 8およびWindows Server 2012の発表により、これらの製品の新しい認定が発表されました。 次に、Microsoftの最新のOSとテクノロジーに関する新しいコースと試験を詳しく見てみましょう。



新しいMCSAおよびMCSEトラック：何とどのように





MCSE：プライベートクラウド





まず、最初の新しいMCSE認定-プライベートクラウドについて詳しく説明したいと思います。







過去を少し思い出すと、文字通り4月の初めにSystem Center製品ファミリが完全に更新されました。現在、製品ファミリはすべての人にSystem Center 2012の単一バージョンを提供しています。

この製品バンドルは、新しいサーバーOS用に外向きに調整されていますが、現在はCTP状態のSP1のリリースでのみサポートされます。 したがって、System Center 2012のRTMバージョンで新しいWindows Server 2012に基づいて練習したい場合、成功しません。

System Center 2012 SP1のCTPバージョンは、こちらから入手できます 。



Microsoft製品に基づくプライベートクラウドのエキスパートになるには、次の要件を満たす必要があります。



  • MICTP：サーバー管理者またはMCITP：エンタープライズ管理者 （これらのステータスはMCSA：Windows Server 2008の要件に相当します ）
  • 試験70-246に合格：System Center 2012を使用したプライベートクラウドの監視と運用
  • 試験70-247に合格：System Center 2012を使用したプライベートクラウドの構成と展開 （2013年1月31日まで、試験70-659はこの要件にカウントされます）






70-659を含むすべての試験に合格したことに注意してください-クラウドステータスを取得するには70-246試験に合格しなければなりませんでした-ベータ試験番号71-246に合格しました（すべてのMicrosoft ベータ試験には番号が付けられています「71-xxx」）そして今、結果を待っています（笑）。



これらの試験246および247の準備をするには、System Center 2012のすべてのコンポーネントを展開し、System Center Orchestratorを使用して正しく動作するように構成する必要があります。

家族との3か月間の厳しい仕事を成功させるためには、最低限必要です。

ベータ試験と未発表のMCSA





ベータ試験とは：



他の製品と同様に、試験には最初にコンテンツを入力し、次にテストし、コンテンツの品質を確認し、評価尺度を設定する必要があります。 ベータ試験が提供されるのはこのためです-これらは公式リリース前の試験の予備バージョンです。 このような試験のプレフィックスは「071」です（「070」ではありません）。 ベータ試験を受けるには、SMEプロファイルに記入し、試験に合格することに関心のあるプロファイル内の製品を選択する必要があります（すべての製品を選択します）。

原則として、ベータ期間が始まる前に、試験に合格するためのプロモーションコードが記載された電子メールを受け取ります（はい、はい-ほとんどの場合無料です）。

もう1つの方法は、 Prometricポータルを監視し、コードで試験を見る方法です。71 -XXXの試験を参照してください。ここでは、下の図のようにベータ版です。







ご覧のとおり、Windows Server 2012のベータ試験が利用可能になりました-HTML5およびCSS3の71-410、71-480 、およびWindows 8 ... 71-687 -MCSAステータスを準備する試験：Windows 8。







MCSA：Windows 8のステータスに関する情報は見つかりませんでした。ちょうど、試験70-687がMCSAを受け取るための唯一の要件であるかどうかわからないからです。 ステータスの謎...

ただし、これらのベータ版すべてに登録しました（71-480を除き、私は開発者ではありません。すみません）。



これで、新しいステータスと試験のレビューを完了したいと思います-それは非常に表面的であり、その目的は近い将来に私たちを待っているもののアイデアを与えることでした。
