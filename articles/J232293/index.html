TizenとUbuntu-HelloWorldへの長い道のり





この投稿では、Ubuntu GNU / LinuxでのTizenの開発環境の展開と構成に関するガイダンスを提供します。

実践が示しているように、これは完全に簡単な作業ではありません。 詳細が必要な人-猫へようこそ。



私は長い間、tizen開発を計画していました。 すべての条件があります： RD-PQはアイドル状態にあり、時間が出現し、モバイルプラットフォーム（今はAndroid向けに書いています）から何か他のものを見たかったのです。



職場でも、自宅でも、どこでも、Lubuntuを使用しています。リソースを必要とせず、ほとんど設定を必要とせず、古き良きWindows 98に似ています。 一般的に迷惑ではありません。 そして、仕事に集中できるように思えますが、それから私はいじくり回さなければなりませんでした。 彼は、他の大仙の新入生への理解を促進するために、最初のアプリケーションの組み立てに至るまでの道筋のステップを概説しました。



1）Lubuntu-完全に標準、現在の14.04.1 LTS x86-64。 誰も知らない場合、彼らはここに横たわっています 。



2） Tizen SDKインストーラーをダウンロードし、 chmod + xを実行して、バイナリを実行します。 実行するには、 java （およびOpenJDKは適切ではありません）、 gettext 、 expect 、 make 、 libudev-devも必要です。



3）インストールが完了しました。 しかし、すべての楽しみはまだ始まったばかりです。





4）風車を使用して新しく設置した日食を開始すると、これが表示されます。



sdbはandroid adbのTaizenアナログで、非常に便利です。

確かに 、私たちが送られた場所にはログはありません （ 更新：ログフォルダにログがありますが、追加してくれたkrovattiのおかげで気づかなかった）が、Googleに尋ねると、問題は簡単に解決されます： 

sudo ln -sf /lib/i386-linux-gnu/libudev.so.1 /lib/i386-linux-gnu/libudev.so.0


5）IDEが起動します。 しかし、「フレンドリー」は、コンテンツのない白い「ようこそ」タブで迎えてくれます。 新しいプロジェクトを作成しようとすると、次のレーキに進みます。



要求された変数にfirefoxへのパスを書き込むこと自体は正当化されないという直感的な仮定であり、再びGoogleにアクセスする必要があります。 ここではすでにより興味深いものであり、いくつかの解決策がありますが、明確な指示はありません。 一言で言えば、Eclipseには組み込みブラウザーで動作するためにxulrunnerまたはwebkitが必要です。 Tizen SDKはかなり古いバージョンのEclipse-3.7.2に基づいているため、その時間に対応するxulrunner-1.9.xまたは3.6.x が必要です。 実際のバージョンは適合しません。xulrunner-31.0を試してみて実験的に発見しました。 さらに、ftp上のアセンブリでは、Mozillaにはx86_64アーキテクチャ用のxulrunner-1.9.2がありません。 2年前に夜の集会の荒野で発見されたのは良いことです。 ご自身の責任で、ダウンロードして解凍してください： 

  wget http://ftp.mozilla.org/pub/mozilla.org/xulrunner/nightly/2012/03/2012-03-29-03-32-28-mozilla-1.9.2/xulrunner-1.9.2.29pre.en-US.linux-x86_64.tar.bz2 tar -xjf xulrunner-1.9.2.29pre.en-US.linux-x86_64.tar.bz2


最後に、 eclipse.iniに次の行を記述します。 

 -Dorg.eclipse.swt.browser.DefaultType=mozilla -Dorg.eclipse.swt.browser.XULRunnerPath=/opt/xulrunner/1.9.2/xulrunner


6）IDEを再起動します。 この記事の最初の写真のように、このようなウェルカム画面が表示されたら、すべてが正しく行われ、作業を開始できます。



7）アセンブリに向けた最後のステップは、証明書の生成です。 これは標準アクションであり、ドキュメントで説明されています 。 [ウィンドウ]> [設定]> [Tizen SDK]> [セキュリティプロファイル]に移動し、アプリケーションの署名に使用するプロファイルと証明書を自分で作成する必要があります。



8）これで、開発環境が構成されました。 ボーナスとして、 こんにちは、 Tizenアプリテンプレートの世界は次のようになります。
