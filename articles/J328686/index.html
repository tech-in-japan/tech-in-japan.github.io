すべてのビジネスは科学研究所であり、あなたはその中の科学者です。



あなたが科学に没頭している科学者のチームであると想像してください。 あなたの仕事は、実験室と現場の両方で仮説の実行可能性を証明する一連の法則とモデルを実験的に導き出すことです。

適切なプロトタイプとその作成方法



デザイン思考とデザインスプリントのアプローチにおいて、方法論の非常に重要な部分はプロトタイピングの問題です。 消費者がこれを必要とするかどうかという主な質問に答える、高速でシンプル、中程度に汚いプロトタイプを作成するプロセス。







研究所での研究との類推に戻ると、製品のパッケージを作成せず、研究所でのテストで肯定的な結果が得られるまで、美しい名前を思い付くでしょう。







プロトタイプを作成するには、通常、デザイナーや開発者にアイデアを説明するのに費やす時間よりも少し長い時間を費やす必要があります。 しかし、最も重要なこと-あなたはあなたの潜在的な顧客から迅速なフィードバックを得る機会を得ます！ プロトタイプを使用すると、速度が低下し、速度が向上します。









Keynoteで数時間で作成されたプロトタイプアプリケーションの例







アプリケーションインターフェイスを開発する必要がある場合は、Keynote / PowerPointが100％適しています。 重要：Sketch / Photoshopやその他のプロのプロトタイピングツールを使用しないでください。数時間でプロトタイプを組み立てる代わりに、作成の段階で数日間ハングアップします。

長方形とKeynote / PowerPointで適切な色で塗りつぶす機能は、プロトタイプのクライアントに製品の主な価値を伝えるのに十分です。 私を信じて！







そして今、プロトタイプが組み立てられています。私は通常20〜30個の基調講演スライドを持っています。Invisionでクライアントのパス全体を収集するために残っています。 電話でプレゼンテーションを開くだけで、既にインタラクティブになっているため、これは必要ありません。 ただし、テストプロセスを高速でスケーラブルにしたい場合は、この手順が重要になります。







建設的なフィードバックを得るのに何人必要ですか（回答：5）



2000年代のWebデザインに関する一連の書籍の著者であるJacob Nielsenは、製品を表示する必要のある5人のユーザーに関する法律を経験的に制定しました 。 私自身の経験から、この法律が有効であることをお伝えします。 20人を見つけることができますが、5から得られる最も価値のある重要なフィードバックはノイズです。







ユーザーに尋ねるべき質問



プロトタイプの準備が整いました。5人が見つかったので、フィードバックを入手してください。

適切な質問をすると、役に立つ答えが得られます。 コピーしてあなたのタスクに適応させることで、以下で私たちが使用する質問のリストを取ることができます。

1.外部機関によるユーザビリティテスト



私はこのオプションを追加しました。多くの場合、このようなソリューションが最も簡単だと確信しているからです。 おそらく、アイトラッキングやその他の「魔法の」技術を使用しても、ユーザビリティテストを実施できる代理店を見つけます。 複数ページのドキュメントの場合、代理店に1000ドルを支払います。 おそらく誰かがこのドキュメント全体を読んで、そこに彼らの視点の確認を見つけたなら、あなたの製品の変更をプッシュするでしょう。 私はすでにユーザビリティエージェンシーの所有者と従業員からの憎しみを期待していますが、これは私の個人的な経験です。たぶん見た目が悪かったり、不運だったりします。







対処方法：

あなたのために仕事をする準備ができている数千の機関から選択してください。







長所：

とても簡単です。 十分な予算が必要であり、推奨事項が記載された美しいドキュメントを入手できます。







短所：







  1. デザイナーは、多くの推奨事項に同意しない可能性が高く、見つかったコメントを修正することに消極的です。
  2. 完全なユーザビリティテストは、ビジネスに変更を加えた後、少なくとも何かが変わることを意味しません。 そして、サービスのコストは非常に高いです。
  3. ほとんどの代理店はあなたのビジネスに飛び込むことはありません。いずれの場合も、A / Bテストと無駄な時間を介して推奨事項を確認する必要があります。 不必要な情報で頭を悩ませる別の「エキスパートアドバイザー」を見つけました。 間違った色のボタンを使用したか、ロゴが認識しにくいため、成功しなかった製品の例を聞いていませんでした。
  4. 推奨事項が収集されて美しく実行されるまで数週間から待機するため、貴重な時間が失われます。
  5. あなたはデザインを完成させなければならないかもしれませんし、代理店にそれを与える前にあなたのアイデアを開発することさえできます。これはお金と時間です。


推奨事項：

選択はあなた次第です。 私はそのような文書をどうするかわかりません、ただあなたの時間とお金を無駄にしないでください。







費用： 5万ルーブルから、推奨事項が記載された文書を数週間待つ。







2.オフィスの廊下テスト



フィードバックを得るための最速、安価で最も簡単な方法。 しかし、残念ながら、最も効果的ではありません。 この方法は、何もしないよりはましだと思います。



対処方法：

オフィスで5人の実験的な同僚を見つけます。 プロトタイプを見せて、テストプロセス中に同僚が遭遇したすべての問題を聞き、記録します。







長所：

おそらく最速のオプション。 廊下で5人の同僚を見つけて、彼らに質問をしてフィードバックをもらいましょう。 同僚を招待し、近くの会議室に閉じ込めて、Skypeを介してプロトタイプがどのようにテストされるかを観察しました。 あなたは間違いなくこのアプローチを好きになるでしょう;カメラを通して同僚をフォローすることは非常にエキサイティングです。







短所：

あなたの同僚の専門的な変形は、確かに回答の質に影響します。 同僚があなたの製品を試さず、賃金のみを支払ったとしても、彼はすでに会社について意見を持っています。これは、ほとんどの場合、平均的な内部視点につながります。 その結果、フィードバックは外部の候補者からのフィードバックよりもはるかに価値がありません。







回答のほとんどは、自社の既存の製品と比較することに限定されます。







推奨事項：

上記の問題を回避するには、プロトタイプを架空の会社の名前と呼び、他のカラーソリューションを使用して、これが現在のビジネスに関係していない製品であるか、自分で行っていることを説明します。 しかし、私のアドバイスはさらに読み、より代表的なオプションを選択することです。







費用：

無料 +インタビューの準備と実施に数時間。

5.「75ドルアプローチ」、スケーラブルかつ最速



最後に、最小限の関与でプロトタイプをテストできるプロセス。 以下に、このアプローチの使用を開始する方法の詳細な手順と、考えられる「落とし穴」の説明を示します。









外部候補者のプロトタイプテストプロセスの一部の例







対処方法：







  1. 回答者に関する質問とともに、Googleフォームで「ようこそアンケート」を作成します。 このステップでのタスクは、ユーザーに精通し、ユーザーと協力して、ユーザーがどのような経験を持っているかを調べることです。 ここと以下の質問は、潜在的な候補者を怖がらせないように、複雑さを増して構築されています。 「ありがとう」ページでこの調査を完了したら、Invisionでプロトタイプへのリンクを投稿します。 そのような調査の例 。
  2. Invisionプロトタイプから水平スクロールを削除することをお勧めします。そうしないと、何が起こっているのか理解せずにプロトタイプ全体が表示されるため、多くが左右にスワイプしてフィードバック全体が塗りつぶされます。
  3. プロトタイプの最終ステップとして、ユーザーを「最終」調査に転送します。 ここで質問の例をご覧ください 。
  4. Lookback.ioサービスのプロジェクトに最初の調査を「ラップ」します。 あなた自身が簡単にこれを行う方法を理解できると思います。 プラットフォームに関係なくユーザーを見つけやすくなります。したがって、iOSとAndroidの2つのプロジェクトを一度に実行することをお勧めします。 しかし、結果の純度のために、iOSで人を探すことをお勧めします。プラットフォームはその中で最もよく機能します。
  5. 残念ながら、 Lookback.ioはUserAgentによってユーザーをまだ2つのバージョンに分割できないため、 hyperurl.coサービスまたはその類似物を介してiOS / Androidの両方のリンクを1つに変えます。
