ソーシャルネットワークに住んでいるのは誰ですか？



PRISM、個人データ、およびその漏洩に関するスキャンダルがどのようにガタガタ鳴ったとしても、ソーシャルネットワークは自分自身についてのすべてを伝えるように呼びかけています。あなたが好きな子猫、友達と、そして朝からそれほど寝なかった理由です。

大多数のオンラインのアクティブな大衆の行動に関する百科事典全体は非常に近くにあり、私は常にそれを感じたかった。 一方で、このデータはパブリックドメインにあるように見えますが、それを取得して分析するのはそれほど簡単ではありません。すべてが構造化されておらず、断片化されています。 さらに、私の知る限り、機械分析に適したソーシャルネットワークデータセットは事実上ありません。 そしてロシアのために-そしてさらにそう。

選択の余地はありませんでしたが、ソーシャルネットワークVKontakte、Odnoklassniki、MoyMir、およびロシアのセグメントFacebookの単純なスパイダーを書くために夜に不吉に笑わなければなりませんでした。 人々が自分自身について語った情報のみが収集されました。 そして彼らはたくさん語った。



そのようなデータからなんとか取得することができたので、話は続きます。



どうして？



私は認めます、この研究は最初のものとはほど遠いです。 ソーシャルネットワーク（特にFacebookとVKontakte）は、何度も公然と研究されてきました。 そして、あなたの謙虚な召使いでさえ、VKontakteからの友人の完全なグラフを収集するために、 6つの握手についての記事を書きました 。

しかし、RuNetは単一のVKontakteに住んでいません。 私は他のソーシャルネットワークで何が起こっているのか、同じように人が住んでいることを見て、彼らの聴衆の違いを理解したかったのです。



データ収集



これは、夜中にビッグデータを収集する初めての経験ではありません。 そのため、速い5ハンドのペースで、4つのスパイダーがQt / C ++およびPythonで記述され、個々のソーシャルネットワークをゆっくりと歩き回り、データベースで出会ったすべてを書き留めました。

ソーシャルネットワークが異なれば、解析のアプローチも異なります。 OdnoklassnikiとFacebookで問題が発生しましたが、それらは判明したように、疑わしいボットを検出するためのかなり難しいシステムを持っています。 幸いなことに、これは主にスパマーを対象としており、この観点からのボットはピンク色でふわふわしており、非常に遅いコレクションではありますが、多少なりとも安定したセットアップを行うことができました。



分析



大量のデータを引き出すのは簡単で、わずか2か月の収集です。 しかし、パラノイアは地球上を歩いており、ほとんどの人にとって、ソーシャルネットワーク上のオープンなプロフィールは非常に貧弱に見えます。 ライオンの情報の共有は、友人だけが利用できます。 しかし、実際には、友人自身はほとんどの場合オープンです。

そして、それらに基づいて、非常に多くの興味深いことを計算できます。 たとえば、市、年齢、大学。 そして、はるかに。 シードについては、実際の年齢と友人の年齢の中央値のグラフを表示します。



ご想像のとおり、実際の年齢はほとんどの場合、友人の年齢の中央値と非常に関連しています。 だからあなたが妄想的であっても、あなたの友人は単に彼らの存在によってあなたについて多くを語るでしょう。



ストレージと分析のために、大企業のように、HBase / Hadoopを使用することにしました。 それは、スタイリッシュでファッショナブルな若者であり、さらに、そのようなテクノロジーのトレーニングの経験がありました。 その結果、収集したものから約50のパラメーターが計算されました（つまり、単一の形式に縮小されるか、社会的なつながりから分離されました）。 いいね。 次に、合計データセットから、各ソーシャルネットワークから100万人のユーザーのランダムサンプルを作成し、慎重に分析しました。 ユーザー数が異なるさまざまなソーシャルネットワークの視聴者を少なくともわずかに正規化するために、同様のフェイントが作成されました。

さらに、実際には、発見された最もおいしいもの。



年齢



手始めに、さまざまなソーシャルネットワークの年齢構造を知ることは興味深いでしょう。

年齢として、人が生年を示すことをためらわなかった場合は年齢そのもの、または学校/大学の卒業日に基づいた近似値が使用されました。 VKontakteとFacebookの正確な年齢はそれぞれユーザーの40％と20％に知られているため、このような操作はほとんどの場合必要でした。

結果は次のようになります。





おもしろい これを見ると、次の機能に間違いなく気付くことができます。

  • Odnoklassnikiは年配の聴衆に座っています。
  • 若い人は、ほぼ完全にVKontakteを使用しています。 14年の地域での奇妙な爆発は、プロファイルで選択できる最後の誕生年が1999年であるという事実によって説明されます。それは若い人たちがすべて選択するということです。 子どもの保護と14歳以上の行動の表示。
  • 人々は意識的な年齢でFacebookにアクセスしますが、18歳未満ではほとんど誰もいません。


性別や年齢の構造はどうですか？ 性別は、ソーシャルネットワーク上でそれを示すことができる場合はそのまま、または次のように名前に基づいて計算されました。 「アレクサンダー」という名前の人の大半が男性である場合、性別不明のすべてのアレクサンドロフを男性と見なします。 同様のアプローチは、ほとんどの名前で機能しますが、ZhenyaとSashaにはいくつかの問題があります。





素晴らしい。 私はいつもこれを疑っていました：

  • Odnoklassnikiでは、聴衆のほとんどが女性です。 政権がネットワークの名前をOdnoklassnikiに変更することをお勧めします。これは真実に近いでしょう。
  • 他のソーシャルネットワークでは、女性に対する一般的な優越感も感じられますが、それほど致命的ではありません。
  • 1976年以前の男性の数の一般的な失敗は、中年の男性が女性よりも早く、より頻繁に死亡するという悲しい事実によって説明できます。 この依存性は、このグループの人々の一般的な人口統計状況を反映していると思います。 男性の世話をします。


次に、ソーシャルネットワークでの個人のアクティビティの特定の抽象的な指標を、「アバターがいる」、「50歳以上の友達」、「最近オンラインになった」など、いくつかのルールのセットの形で計算しました。 そのような各ルールの運用については、いくつかの点が一般的なプロファイルの貯金箱に落ちました。 そのため、さまざまなソーシャルネットワークにこのインジケーターが分布しているように見えます。





驚くべきことに、VKontakteは、若くて活発なユーザーを単に駆り立て、そのヒューズは35歳までに消えます（または家族のチャンネルに敏感にリダイレクトされます）。 オドノクラシニキでは、主な活動は早くも30年から始まります。 しかし、この点でMoiMirとFacebookでは、状況はもっと嘆かわしいです-沼地があります。



冒とく



年齢と活動は確かに良いですが、非常に退屈です。 そして、眠らないようにするために、サンプルの各人について、彼の投稿で見つかったわいせつな単語の数が計算されました。 そのような単語の辞書を作成することは特に面白かったです。 縦座標軸に沿った次元-最後の10投稿の単語数。



明らかに、15歳の若者は非常に大胆であるため、ページ上でわいせつに誓うことができます。 個人的には、私は12年という早い時期に校庭で最初の「X * Y」を書きましたが、真実は匿名です。 わいせつな語彙の流出は、23日までコマーシャルを続けます。 そして、どうやら、真剣さが始まり、大人になる時間です。 非常に船長の声明ですが、今では少なくとも事実によって証明されています。

データ



分析のためにこのような素晴らしいデータセットを作成し、それを共有しないことは、人道に対する罪です。 したがって、パブリックドメインに配置することが決定されましたが、このデータセットに陥った人々の権利とプライバシーを損なわないようにするために：

  • プロフィールは匿名化されており、名前、姓、生年月日、大学またはプロフィールの住所はありません。 理論的には、一部のプロファイルは匿名化できますが、これは非常に困難です。
  • データセットはサンプルであり、そのボリュームは正確に400万のプロファイルであり、各ソーシャルネットワークで100万です。
  • この善の収集と分析は、個人データに関する法律を満たしているように思えます。 キーワード：公開データ（ユーザーが自分で投稿したもの）、匿名化（名前、生年月日などなし）、科学的非営利目的での使用。
  • データライセンス：Creative Commons Attribution Shake-alike Noncommercial（CC-BY-NC-SA）。
  • MySQLのデータをダンプします。 使用可能なすべてのフィールドの説明が含まれています。
  • データにバグがある可能性がありますが、それは確かです。 バグを見つけたら、プライベートメッセージで誇らしげにそれを私と共有できます。


データ 、7z、アーカイブ内の135Mb、アンパック形式の1Gbでアーカイブします。
