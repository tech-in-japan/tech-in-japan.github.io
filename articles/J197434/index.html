P2Pソーシャルネットワークとディアスポラの概念

P2P（ピアツーピア）は、各コンピューターがサーバーとクライアントの両方であるコンピューターネットワークを構築するためのテクノロジーです。このテクノロジーにより、共有リソース（ファイルなど）を使用することもできます。

集中型 （ネットワーク管理用の中央サーバーが1つ）、 ハイブリッド型 （そのようなサーバーが多数ある）、および純粋な P2Pネットワーク（サーバーがまったくない）があります。 コンピューター間の通信にはさまざまなプロトコルが使用されており、最近ではそれらを標準化する試みがあり、オープン性があります。JXTAおよびGnutellaは標準化されたプロトコルを開きます。 中央集中型およびハイブリッドネットワークでは、新しいコンピューターがP2Pネットワークに接続すると、サーバーに登録され、ネットワーク情報を受信します。 純粋なP2Pネットワークでは、コンピューターは以前にネットワークに接続されたコンピューターに接続し、そこからネットワークに関する情報を受け取ります。 さまざまなファイルを配布するように設計された従来のP2Pでは、各コンピューターはネットワークによって匿名のエンティティとして認識されます。 最近、P2Pテクノロジーに基づいたソーシャルネットワークを構築するための革新的なプロジェクトがいくつか登場しました。



例：

LifeSocia l-分散ソーシャルメディアプラットフォーム

PeerSoN-機密P2Pソーシャルネットワーク

Safebook —再び、機密のソーシャルネットワーク

ディアスポラ -機密性（個人の管理下）オープンソースのソーシャルネットワーク



これらのプロジェクトは一般原則、つまりP2Pアーキテクチャとデータ暗号化に基づいていますが、さまざまな方法で外観を動機付けています。

開発者LifeSocialは、 サーバーコンテンツの保存に注意を払っています （Facebookによると）

ユーザーはFacebookに年間0.5ドルの費用がかかります。Facebookユーザーの数が10億人を超えると、かなりの額になります）。



PeerSoNとSafebook は、データを完全に制御してユーザーを動機付けます。 （データは共有サーバーではなくユーザーのコンピューターに保存され、インターネットではオープン形式ではなく暗号化されて送信されるため、ユーザーはデータを完全に制御できます。）



これらのプロジェクトにはそれぞれ独自の特性があります。

たとえば、 LifeSocialでは、ユーザーはプログラムを使用して、インターネットにアクセスしなくても「友人」のサークルからプロファイルを表示できます。 ユーザーデータの99％のアクセシビリティを実現するために-このデータはネットワーク上の数千の他のコンピューターに配信され、そこで暗号化された形式で保存されます。 LifeSocialおよびSafebookは、検索サービスにもDHT（Distributed Hash Table）を使用しています。



PeerSoNは、分散型P2P検索サービスにOpenDHTを使用して、コンピューターがネットワーク上でお互いを検出できるようにします。 PeerSoNでの新しいユーザーの検索は、興味深い方法で構成されています。たとえば、古くからの友人の検索です。 PeerSoNの開発者が書いているように、「このような機能は、ユーザーのプライバシーを保護することを目的とするシステムで処理するのが困難です。」ユーザープロファイルは多くのファイルで構成されます。 各ファイルには、名前や誕生日など、何らかの種類のユーザープロファイルフィールドが含まれています。 ユーザーは各ファイルのアクセス権を決定することにより、他のユーザーに対するこの情報の可用性を決定します。 したがって、あるユーザーが「KSU」のリクエストで彼を見つけたい場合、「KSU」という名前のファイルが作成されるため、「KSU」で勉強したすべてのユーザーを見つけることができます。 このファイルは、GUID @ filenameのようになります（ネットワークユーザーのGUID idなど、電子メールからのハッシュがある場合があります。したがって、2つのファイル「14 @ KSU」および「123 @ KSU」が見つかった場合、 id = 14およびid = 123を調査したか、KSUと関係があります。



入れ子人形は、ユーザーデータが保存されるさまざまなレベルのコンピューターで構成されるユーザープロファイルの構造であり、中央にあるユーザーのコンピューターは、別のユーザーのマトリョーシカのあるレベルにある可能性があるため、ユーザープロファイルはSafebookに表示されます。 つまり 入れ子人形に含まれる任意のコンピューターからユーザーデータを取得できます。



Diasporaは、同じ原則に基づいた比較的新しいプロジェクトですが、最も重要な違いは、オープンソースであることです。 2010年9月15日に、プログラムのバージョン1が公開され、すぐに開発者はOAuthをサポートし、TwitterおよびFacebookと統合しました。



それにもかかわらず、今日のP2Pベースのソリューションには欠点があります。
