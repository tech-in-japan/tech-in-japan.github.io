対話のテキストCtrl + Alt + Delは、Steve Ballmerによって個人的に作成されました





多くの人は、古いバージョンのWindowsでの仮想マシンマネージャーウィンドウの外観を覚えています。 しかし、このメッセージの元のテキストがスティーブバルマーによって個人的に書かれたことを知っている人はほとんどいません。



マイクロソフトの開発者でありThe Old New Thingの著者であるRaymond Chenは、 この話を思い出しました 。



初めて、Ctrl + Alt + Delダイアログが80386カーネルのWindows 3.1オペレーティングシステムに実装されました。これは、標準モードおよび「拡張」モード（拡張モード）で動作することができました。 後者では、複数の16ビットアプリケーションを並行して実行し、それらの間でCPU時間を分散できました。



もう1つの違いは、特別な呼び出しです。この呼び出しにより、一部のアプリケーションからプロセッサ時間を取得できました。



最終的に、アドバンスモードの3番目の違いは、Ctrl + Alt + Delを押したときに画面に表示されたのと同じ「仮想マシンマネージャー」でした。 複数のアプリケーションの並列操作中に、そのうちの1つが時間内に応答を拒否した場合に表示されます。



マネージャーはハングしたプログラムの名前を表示し、プログラムを閉じる、コンピューターを再起動する、またはマネージャーを終了するという3つのオプションの選択肢を提供しました。



当時、Steve Ballmerはオペレーティングシステム部門（システム部門）を率いていました。 どういうわけか、彼はWindows開発者に行って、彼らがどのようになっているかを調べました。 「彼はCtrl + Alt + Delで新しい機能を見せられ、スティーブは思慮深くうなずき、次のように言いました。 間違っているように思えます。」



「オーケー、スティーブ、もっといい仕事ができたら、やりなさい」とプログラマーはマネージャーに提案した。 彼は挑戦を受け入れました。



数日後、バルマーはメッセージの新しいオプションを含む手紙を送りました。 



 アプリのタイトル

  このWindowsアプリケーションは、システムへの応答を停止しました。

   * ESCを押してキャンセルし、Windowsに戻ります。
   * Enterキーを押して、応答していないこのアプリケーションを閉じます。
     このアプリケーションに保存されていない情報は失われます。
   * CTRL + ALT + DELをもう一度押して、コンピューターを再起動します。 あなたは
     すべてのアプリケーションで保存されていない情報を失います。


このオプションは十分に受け入れられるため、テキストは1語ずつ変更されることなく本番環境に入りました。
