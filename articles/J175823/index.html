02-シンクライアントとシックサーバー

みなさん、素晴らしい一日を！ ブログの 最新号では、仮想化とは何か、そして仮想化が一般的にどのように機能するかについて話しました。 今日は、この素晴らしさがすべて回転する鉄についてお話します。







仮想化されたワークステーションは、人と対話できるハードウェアシステムがなければ機能しません。 神経インターフェースにはまだ到達していないので、もっとありふれた技術を検討します。



多くの人が「シンクライアント」という表現を聞いたことがありますか？ 2007年から2008年にかけて、シンクライアントはポジティブな感情よりもネガティブな感情を呼び起こしました。 本当に薄いものはうんざりするほどうまく機能し、うまく機能するものはそれぞれ通常のコンピューターとほとんどデザインが異なりませんし、それらからの利益もあまりありませんでした。



今日、「シンクライアント」のコンセプトはあなたの期待に応えます。 ほとんどの場合、これはホームルーターのサイズの箱であり、可動部品がなく、注意やメンテナンスが必要な複雑で高価な詰め物です。 仮想ワークステーションへの切り替えを決定した企業の観点から見たシンクライアント（一部のモデルのハードウェア機能を除く）の主な違いは、「ワークステーション」内にミニOSが存在することです。



業界には2つの主なアプローチがあります。 Windows XPまたはWindows 7に基づいて、特別なWindows Embedded （そのライセンスはデバイスの価格に含まれています）をインストールするか、いわゆる ゼロクライアント 。OS全体が数メガバイトであり、Linuxカーネルと、読み込み、DHCPを介したネットワーク設定の取得、およびゲストOSをクライアントに「投影」するサーバーへの接続を提供する一連のマイクロユーティリティとサービスです。



シンクライアントで使用される別のアプローチとして、 Smart Zeroがあります。 通常のゼロクライアントとは異なり-「ファームウェア」の量。 Windows Embeddedよりもかなり小さく、数十メガバイト必要ですが、より多くのユーティリティユーティリティとネットワーク診断および構成マネージャーが含まれています。 Smart Zeroは、ネットワークの展開を加速し、新しいジョブをインストールする独自のテクノロジーもサポートしていますが、ソフトウェアのすべての機能については別のトピックで詳しく説明します。 :)



ユーザーが取り組んでいるものは、多かれ少なかれ明確です。 サーバーと、データの保存や内部リソースの操作に使用される通常のサーバーの違いは何ですか？







基本的なハードウェアの違いはほとんどありません。 もちろん、サーバーが優れている（強力である）ほど、同時に接続できるクライアントを含めることができることを理解する必要があります。



「仮想マシン」用のサーバーの例として、HP ProLiant Gen8 DL160を検討してください。 SMBセグメントでターミナルサーバーとして選択されることが多いクラシックラックマウントサーバー。



内部では、いつものように、不自然なものはありません。通常のサーバーXeon、大量のRAM、サーバーOS用のディスクシステムです。 ターミナルサーバーのすべての機能はソフトウェア部分にあります。 そして、世界中のほとんどのオフィスでWindowsが事実上の標準になったので、Windowsサーバーに基づいたソフトウェアを検討します。



Windows Serverは、ターミナルサーバーのコンポーネントとターミナルクライアントのライセンスマネージャーをインストールし、それらの初期構成が実行されます。 ターミナルOSおよび初期セットアップのライセンスをアクティブ化した後、サーバーはそれ自体の寿命を保ち、原則として、それを維持する必要はありません：構成が完了すると、RPSまたはVDIを使用してすべてのユーザーのニーズに合わせて中断のない動作を保証します。



このようなサーバーには、信頼性の向上したデータストレージシステム、強力なネットワーク機器を追加することができます。また、「欠陥」コンピューターがあると考えずに12人以上の人々が作業できる信頼性の高い集中型ソリューションを取得できます。



別の興味深いケースがあります：VDIサーバーとして強力なワークステーションを使用します。







あなたがエンジニアまたはデザイナーであり、本当に強力な機器を必要とする重いパッケージで作業しているとします。 優れたマルチコアプロセッサ、32、64、または128ギガバイトのRAM、Nvidia QuadroまたはTeslaは、一般に、高度な技術の完全な詰め物であり、使用する方法を本当に知っています。







また、ワークステーションから遠く離れているため、プロジェクトを緊急に編集、修正、表示、または次のタスクを計算に加える必要がありました。 この場合、あなたの便利なデバイスからコンピューターに接続するだけです。コンピューターはサーバーになり、仮想デスクトップを操作します。まるで、膝の上に噛まれたリンゴのキログラムではなく、強力なワークステーションを持っているかのようです。



特定の実際的な例について、次のすべての投稿で、私たちが一般理論で終了しました。 :)ハブルのページでお会いしましょう。
