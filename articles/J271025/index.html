Spring 4 MVC、Hibernate：多対多の関連付け

この投稿では、Spring 4 MVC Webアプリケーションでテーブルを結合する例を使用して、Hibernateの多対多の関係を示します。 提出とやり取りで多対多の関係管理について説明します。 Webインターフェイスを使用して、作成、更新、削除、およびクエリの操作を実行します。



このアプリケーションはorg.springframework.core.convert.converter.Converter

ます。これは、データベース内の実際のサブジェクトにID 'アイテムを表示するのに役立ちます。

ステップ1：JOINテーブルを使用して多対多のスキーマを作成する







APP_USER ：ユーザーが含まれます。 ユーザーは複数のプロファイル[USER、ADMIN、DBA]を持つことができます。

USER_PROFILE ：ユーザープロファイルが含まれます。 プロファイルは複数のユーザーに関連付けることができます。

APP_USER_USER_PROFILE ：これは、APP_USERテーブルとUSER_PROFILEテーブルを多対多の関係に関連付ける結合テーブルです。



デモンストレーションの目的で、この例では、一方向インストール[ユーザープロファイルを持つユーザー]の多対多の関係について説明します。



テーブルの作成とデータの追加 
 create table APP_USER ( id BIGINT NOT NULL AUTO_INCREMENT, sso_id VARCHAR(30) NOT NULL, password VARCHAR(100) NOT NULL, first_name VARCHAR(30) NOT NULL, last_name VARCHAR(30) NOT NULL, email VARCHAR(30) NOT NULL, PRIMARY KEY (id), UNIQUE (sso_id) ); create table USER_PROFILE( id BIGINT NOT NULL AUTO_INCREMENT, type VARCHAR(30) NOT NULL, PRIMARY KEY (id), UNIQUE (type) ); CREATE TABLE APP_USER_USER_PROFILE ( user_id BIGINT NOT NULL, user_profile_id BIGINT NOT NULL, PRIMARY KEY (user_id, user_profile_id), CONSTRAINT FK_APP_USER FOREIGN KEY (user_id) REFERENCES APP_USER (id), CONSTRAINT FK_USER_PROFILE FOREIGN KEY (user_profile_id) REFERENCES USER_PROFILE (id) ); /* Populate USER_PROFILE Table */ INSERT INTO USER_PROFILE(type) VALUES ('USER'); INSERT INTO USER_PROFILE(type) VALUES ('ADMIN'); INSERT INTO USER_PROFILE(type) VALUES ('DBA'); commit;

@ManyToMany



は、ユーザーとユーザープロファイルの間に多対多の関係があることを示します。 ユーザーは複数のプロファイル[USER、ADMIN、DBA]を持つことができますが、プロファイルは複数のユーザーに属することができます。 @JoinTable



は、外部キー制約を使用して2つのテーブルをプライマリに結合する参照テーブルがあることを示しています。 この注釈は、主に所有関係の側で使用されます。 joinColumns



は所有側の列名（ユーザーID）を参照し、逆joinColumns



は関係の後ろの列（IDとユーザープロファイル）を参照します。 この結合テーブルの主キーは、USER_IDとUSER_PROFILE_IDの組み合わせです。



遅延読み込み



fetch = FetchType.LAZY



特に注意してfetch = FetchType.LAZY



。ここでは、 fetch = FetchType.LAZY



コレクションの「遅延」ロードについてHibernateに通知します。 この動作は基本的なものです。 このようなインストールでは、コレクションのダウンロード要求は、最初に対処されるとすぐに削除されます。 これは、グラフへのオブジェクトのすべての接続を取得するのを避けるための良い方法です。これは「高価な」操作です。 トランザクション/アクティブセッションでコレクションにアクセスしようとすると、Hibernateはそれらを受信するリクエストを分離します。



ただし、アクティブなセッションの外にいる場合（たとえば、セッションが閉じられている/トランザクションがない：JSPページにいる）、コレクションにアクセスしようとすると、 org.hibernate.LazyInitializationException – could not initialize proxy – no Session



。 これを回避するには、 Hibernate.initialize(user.getUserProfiles());



要求に従ってコレクションを初期化する必要がありますHibernate.initialize(user.getUserProfiles());



アクティブなセッションで。



また、カスケードを使用しないことに注意してください。 これは、ユーザープロファイルがユーザーに依存せず、個別に存在できるためです。



重要な注意事項 ：「多くの」関連付けの場合、コレクションでエンティティを渡すときにHibernateが監視するhashCode（）およびequals（）メソッドを常にオーバーライドします。
