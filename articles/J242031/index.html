Hetznerで仮想化ホストを整理する





Proxmox Virtual Environment（Proxmox VE）は、Debian GNU / Linuxに基づいたオープンソースの仮想化システムです。 オーストリアのインターネット基金が後援するオーストリアの会社Proxmox Server Solutions GmbHによって開発されました。



...そして、私は維持し、運用するための非常に快適なシステムを言わなければなりません。



私は約5年間、クライアント（ほとんどの場合、Webデザインスタジオ、またはスタンドアロンクライアント）と協力しています。 すべてのクライアントは、ドイツのホスティング会社ヘッツナーのサービスを使用しています。 提供するサービスの品質、サーバー内の鉄の品質については長い間話すことができますが、事実は変わりません。ルートサーバーセグメントでのHetznerの提供は、価格/品質比の点で競合を超えています。



参考：この記事の執筆時点では、 EX60サーバー（Intel Core i7-920、48 GB DDR3 RAM、2 x 2 TB SATA 3 Gb / s 7200 rpm HDD）の月額賃料は月額わずか59ユーロです。 同意する、かなり安価。



この記事では、ルートサーバーにproxmox仮想化システムをインストールするための簡単な方法を説明します。 2つ以上のサーバーのシステムをクラスター化するためのオプションについては説明しません（コミュニティに興味がある場合は、次の記事でこれを行うかもしれません）が、最も単純なオプションを作成します。



そのため、まず、サーバー自体が必要です。 EX60は、安価で生産性が高く 、1回限りのセットアップ料金などの不快な機能がないため、 EX60をお勧めします（他のモデルを注文する際は注意してください！）



サーバーを注文しました。 さて、我々はそれがインストールされるのを待っています、そして切望された手紙が郵便で到着するとき、我々はプロセスに直接進みます。 仮想化のために追加のIPアドレスが必要になることは言うに値します。 最大3個まで個別に購入することも、すぐにサブネットを購入することもできます（追加のFlexiPackサービスパッケージ、月額15ユーロ+サブネットレンタルコストが必要）。 これを行うには、サーバーコントロールパネルrobot.your-server.deに移動し、実際には、[IP]タブで注文します。 使用目的を必ず指定してください！







複数のIPアドレスを注文する場合は、好みに合わせて繰り返します。



次に、リカバリモードでサーバーをロードする必要があります。 これを行うには、[レスキュー]タブに移動し、[オペレーティングシステムLinux、アーキテクチャ64ビット]を選択して、[レスキューシステムのアクティブ化]をクリックします。







システムが提供するパスワードを書き留めます







サーバーを再起動します。







前に記録したパスワードでサーバーにログインし、次のコマンドを実行します。 

root@rescue ~ # passwd root@rescue ~ # installimage


つまり、パスワードをより使いやすいものに変更し、インストールプロセスを開始します。



開いたウィザードで、[ 仮想化]-> [Proxmox-Virtualization-on-Debian-Wheezy]を選択します



セットアップ構成ファイルエディターが開きます。 必要な変更をすべて加えて、不要なコメントを削除して、完全に提供します。

F10を押して変更を保存し、ディスク上のデータが失われるというひどい警告に同意します







そして、コーヒーを飲みに行きましょう。



一杯のコーヒーが飲まれ、心地よい暖かい波があなたの体を駆け巡るとき、おそらく私たちのサーバーのインストールはすでに完了しています。 あとは、コンソールに入力して再起動するだけです 

 root@rescue ~ # reboot




新しくインストールしたサーバーをロードし、アドレスにアクセスします： 

  https：// your_ip_address：8006


インストール前に変更したルートアカウントとパスワードを使用してログインします。 結局、変更しましたか？ そうでない場合は、hetzner =）が提供したものを使用します





設定を続けます。

ストレージを追加します。

データセンター->ストレージ->追加-> LVM





素晴らしい、ボリュームグループを接続しました。 少し忍耐、少し左=）



ネットワークをセットアップします。

データセンター-> Node1->ネットワーク

スクリーンショットに示されているビューを表示します。 ネットワークをブリッジとして設定する必要があることに注意してください！ 再起動するまで、設定を操作してお気軽に、まだ適用されません...





そして、右上隅にある[ 再起動 ]ボタンを使用して、実際にサーバーを再起動します。

コーヒーの後に残っているCookieを食べ、サーバーが起動し、動作する準備が整いました。

Proxmoxコントロールパネルに再度ログインし、右上の[ VMの作成 ]をクリックします 。

直観的なウィザードが起動しますが、これについては3つのことだけを言う価値があります。

  • 1. [ハードディスク]タブを混同しないでください
  • 2. [ネットワーク]タブの[MACアドレス]フィールドで、以前に注文したIPのMACアドレスを指定します
  • 3. [CD / DVD]タブで、画像を選択します。 ディレクトリ/ var / lib / vz / template / isoのハイパーバイザーにisoイメージを自分で置くことができます




実際には、注文したIPのMACを生成する方法。

サーバーコントロールパネルrobot.your-server.deに移動し、[IP]タブに移動して、[個別のMACアドレスを要求]をクリックします。 MACを取得し、仮想マシンの設定に移動し、DHCP経由でIPを取得します。



他に追加する価値のあるもの。

  • 仮想マシンコンソールには、[コンソール]ボタンをクリックして、Webインターフェイスからアクセスできます。
  • 仮想マシンにWindowsをインストールする場合、[ハードディスク]タブでSATAコントローラーを選択します。




みんなのようです。 コーヒーをもう1杯注ぎ、休憩に値します。
