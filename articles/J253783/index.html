新しいVeeam機能を使用してクラウドにバックアップする





Veeam Backup and Replicationの8番目のバージョンでは、Veeam Cloud Connectなどの便利な機能が導入されました。 バックアップを自分のサイトだけでなく、クラウドプロバイダーのリモートサイトに送信することもできます。 この投稿では、これがどのように機能するかを詳細に例とともに検証します。





短い背景といくつかの広告



8つ目のリリースを見越して、 IT-GRADは、 Veeam Backup and Replicationの8番目のバージョンで発表されたクラウドバックアップ機能を使用する目的で、 Veeam Cloud Providerゴールドステータスのステータスを受け取りました 。 その後、私たちはこの問題の組織面での作業を開始し、Veeam Backup＆Replication v8のリリースで、この問題に特に興味のある顧客と一緒にスタンドを展開してこの奇跡をテストすることを急いでいます。 この作業の結果を共有します。



スクリプト



VMwareベースの仮想化インフラストラクチャを持つクライアントは、Veeam Cloud Connectでクラウドバックアップテクノロジーを利用することを決定しました。 どこから始めますか？



まず 、Veeamクラウドサービスプロバイダーを決定する必要があります。Veeamのサイトは、仮想マシンの信頼できるバックアップストレージとして使用されます。 このようなプロバイダーには、Veeam Cloud Provider（VCP）の能力があります。これは、彼らの高度なプロフェッショナリズムを示し、達成された専門家レベルを強調しています。 同時に、クラウドへのバックアップを選択した顧客は、選択したクラウドプロバイダーからのサービスのサブスクリプションが必要になります。 適切なプロバイダーを見つけるには、リンクwww.veeam.com/en/find-a-veeam-cloud-provider.htmlを使用して、Veeamの公式Webサイトで検索エンジンを使用できます（注：現在、技術的な理由でロシアで検索することはできません。作品）。



次のステップ ：Veeam Backup＆Replication v8製品をクライアントサイドに展開します。 Veeam Cloud Connectは、Veeam Backup＆Replicationの第8バージョン以降でのみ利用可能です。



仮想インフラストラクチャサーバーに接続する



Veeam Backup＆Replication v8でバックアップおよび復元タスクを実行する前に、インフラストラクチャを計画および構成する必要があります。 もちろん、多くは仮想環境とバックアップ戦略自体に依存します。



  • Veeam Backup＆Replication v8を初めて起動し、バックアップインフラストラクチャウィンドウに入ります。






[ 管理対象サーバー]パネルで、リソースがバックアップ手順に関与するインフラストラクチャを選択する必要があります。 いくつかのオプションが可能です。



  • VMware vSphere — vCenter Server（4.x以降）またはvSphereハイパーバイザーの「スタンドアロン」（ESX / ESXi 4以降）を追加して、vSphereインフラストラクチャに接続します。
  • VMware vCloud Director -VMware vCloud Directorサーバー（5.x）を追加してvCloudインフラストラクチャに接続します。 vCloud Directorサーバーを登録すると、仮想マシン自体に加えてvApp設定を含むvCloud Directorメタデータをバックアップできるタスクを作成できます。
  • Microsoft Hyper-V — System Center Virtual Machine Manager（SCVMM）、Hyper-Vクラスター、およびスタンドアロンホスト（Windows Server 2008 R2 SP 1以降）を追加して、Hyper-Vインフラストラクチャに接続します。
  • Microsoft SMB3ストレージ -SMB3サーバークラスターまたは独立したSMB3サーバーを追加すると、SMB3仮想マシンで実行されているブロックへの変更の追跡を有効にできます。
  • Microsoft Windows Sever -Windows Server（Windows Server 2003以降）に接続する機能。
  • Linuxサーバー -Linuxサーバーを追加する機能は、バックアップサイトとしてのみ使用できます。


実際のオプションの1つであるvCenter Serverへの接続を検討してください。 これを行うには、VMware vSphereアイコンをクリックして、新しいVMwareサーバーウィザードを開始します。







[ 認証情報]ウィンドウで、vCenter Severの管理者権限を持つユーザーアカウントを入力します。







接続に成功すると、 Veeam Backup＆Replicationメインコンソールに追加機能が表示されます。これには、Veeamクラウドプロバイダーのリモートサイトを選択して接続する機能を備えたVeeam Cloud Connectが含まれます。







  • Veeam Backup＆Replicationコンソールからクラウドプロバイダーを検索できることに注意してください。 プロバイダーを既に決定している場合は、[サービスプロバイダーの追加]ボタンをクリックする必要があります。


IT GRADはVeeam Cloud Provider Goldのステータスを持っているため、これにより顧客は当社のリモートサイトを使用してクラウドバックアップ 、レプリケーション、高速データリカバリを実現できます。 IT-GRADの例を使用して、クラウドプラットフォームへの接続がどのように行われるかを確認します。



  • [サービスプロバイダーの追加]ボタンをクリックし、表示されるウィザードで、リモートサイトのIPアドレスまたはDNS名を指定します。 私たちの場合、これはデフォルトで使用されるveeamcc.it-grad.ru、ポート6180です。


注！ IPアドレスの値、リモートサイトのDNS名、および接続ポート番号は、クラウドプロバイダーによって報告されます。 ポート番号はデフォルト値と異なる場合があります。 複数の異なるプロバイダーに接続する予定がある場合は、[説明]フィールドに各接続の追加の説明を指定することをお勧めします。








  • 次のステップでは、資格情報（ログイン、パスワード）を指定します。通常、資格情報はクラウドソリューションプロバイダーによってクライアントに伝達されます。 この段階で、証明書も検証されます。






  • [ 次へ ]ボタンをクリックすると、サービスプロバイダーのサイトへの接続が確立されます。
  • [ リソース]ウィンドウに、指定したボリュームで利用可能なクラウドリポジトリが表示されます。
  • [ 概要]ウィンドウに概要情報が表示されます。 [完了]をクリックすると、ウィザードが完了します。


したがって、[サービスプロバイダー]フィールドには、接続されたクラウドプラットフォームが表示されます。 [ サービスプロバイダーの編集 ]ボタンを使用して、既存の接続を変更したり、[ サービスプロバイダーの削除 ]ボタンを使用して削除したりできます。







バックアップとレプリケーションパネルの使用



[バックアップとレプリケーション]パネルでは、選択したデータのスケジュールバックアップ、複製、既存のコピーのバックアップ、仮想マシンのコピー、ファイルのコピー、復元、既存のバックアップからのデータのインポート、フォールトトレランスプランを作成するタスクを作成できます。







バックアップジョブの作成



以前に発表したシナリオに戻って、クラウドサービスプロバイダーの接続されたプラットフォームにコピーを配置して、仮想マシンをバックアップする必要があります。 タスクを作成するには、バックアップジョブウィザードを実行する必要があります。







  • ウィザードウィンドウで、名前（この例では、VMのバックアップ）を設定します。 セマンティック名を設定して、このタスクが何を実行するかを明確にし、さらに追加するには説明を指定することをお勧めします。 例：タスクが意図されている理由、正確に何を実行するか。
  • 次に、ウィザードウィンドウで、バックアップする仮想マシンを選択することを提案します。 [追加]ボタンをクリックして、必要なオブジェクトを追加します。 次のようになります。






  • ウィザードでは、仮想マシンをバックアップする順序を指定することもできます。 この順序は、選択したオブジェクトの場所を変更することで変更できます。
  • 重要なポイントは、バックアップが行われるストレージ（リポジトリ）の選択です。 デフォルトでは、ウィザードは製品展開プロセス中に作成されるローカルVeeamリポジトリの使用を提案します。 この場合、サービスプロバイダーのクラウドプラットフォームを選択する価値があり、クラウドリポジトリとして表示されます。






  • ウィザードウィンドウでは、[詳細設定]ボタンを使用して、バックアップの種類、圧縮および重複排除パラメーター、ブロックサイズ、通知パラメーターを設定したり、暗号化、整合性チェック、その他の設定を有効にしたりできます。
  • 特に、バックアップのスケジュールを設定できます。


作成されたタスクは、手動で開始したり、停止、無効化、複製、削除、編集したりできます。 特定のスケジュールでは、設定で指定された時間条件に従ってタスクが開始されます。







まとめると、組み込みのクラウドコネクト機能を備えたVeeam Backup＆Restoreは、信頼できるクラウドプロバイダーをリモートサイトに接続し、クラウドへのバックアップを実行することを可能にする非常に理解しやすいツールであるように感じます。 企業のIaaSに関する最初のブログでは、企業クラウドとの連携をサポートする他の製品について書いています。 当社のWebサイトでは、仮想マシンのバックアップサービスのクラウドへの接続を注文できます。
