Joker 2016 Java Conference：より大きく、より強く、より興味深い

2016年 10月14〜15日にサンクトペテルブルクで開催されたJoker 2016カンファレンスは、おそらく史上最大のロシア語Javaカンファレンスになりました。 2日間、52のレポート、6つの並行トラック、40人以上のスピーカー、1000人以上の参加者。







会議の直前の期間は非常に波乱に富んでいた。



  • JavaポータルJUG.ruの起動 。
  • サンクトペテルブルクおよびモスクワ Javaユーザーグループの会議でのジョーカースピーカーによる多数のスピーチ。
  • 会議の準備に関するJUG.ruのインサイダーとのHabrahabrに関する興味深い記事（ スポンサー について、 オンラインブロードキャストの準備 について、管理コンポーネントについて ）;
  • シリーズ「Without Slides 」からのアレクセイ・フェドロフによる古典的な1対1のインタビュー（ Ruslan Cheremin 、 Barukh Sadogursky ）
  • 新しいフォーム-テーマ別インタビュー（ パフォーマンス 、 Java DevTools 、 Java Stream API 、 レガシーコード 、 OOP 、 テスト 、 パナマプロジェクト 、 ジグソープロジェクト 、 技術レポート 、 コンテナ化 、 JVM内部について ）;
  • Joker 2016のスポンサーとのインタビュー（ T-Systems 、 Luxoft 、 SEMrush 、 EPAM 、 Odnoklassniki 、 Sberbank-Technology ）。


私の意見では、上記の真の真珠は、 Baruch SadogurskyとVladimir Krasilshchik （どちらもJoker 2016のスピーカーです）へのインタビューでした。



過去数年と比較して、会議は成長しました-1.5倍以上の並列トラック（4つではなく6つ）。 ほぼ同等のいくつかのオプションから表示するレポートの選択はさらに難しくなりました。 多くの場合、お菓子屋の子供であるという感覚がありました-明るく、美味しく、選ぶのは難しいです。



この会議の新しい会場は、サンクトペテルブルク博覧会でマスターされました 。 美しい景色（ 1、2 ）と主催 者とのスピーカー 、彼らの存在で会議を飾ります。

Javaからアセンブリーへ：ウサギの穴

チャールズ・ナッター、レッドハット







チャールズは、 JRubyプロジェクトのリーダーの一人です。 レポートのタイトルは、 不思議の国のアリスの冒険 （ 不思議の国のアリス ） の最初の章のタイトルと一致しています。 プレゼンテーションでは、キャロルによるアリスについての両方の本の初版のジョン・テニエルによる古典的なイラストを使用しました。



WonderlandとJVMの類推は非常に成功しており、プレゼンテーションスタイルとレポートデザインも非常に独創的です。 面白い形式で、 Javaソースコードからネイティブコードへのパスが示されました-バイトコードへのコンパイル、バイトコードの解釈、ネイティブコードへのJITコンパイル。 さらに、 最終フィールド、 synchronized 、 volatile 、文字列のswitchステートメント 、ラムダ式の実装機能が開示されました。



JITコードのライフサイクル

Ivan Krylov、Azul Systems







前のレポートの論理的な続きは、 Just-in-time （ JIT ）コンパイルに関するIvan Krylovのレポートでした。 コード変換、プロファイル、および最適化解除のケースに加えて、5つのコンパイルレベル（それらの間の可能な遷移を含む）およびコンパイルのチューニングに使用可能な4つのAPIが言及されました。



興味深い話は、 Azul Zing仮想マシンに固有のテクノロジーであるReadyNowについてでした 。 静的なAhead-in-time （ AOT ）コンパイルに対するOracleのさらに大きな関心についても言及されています 。JEP295を参照してください。
