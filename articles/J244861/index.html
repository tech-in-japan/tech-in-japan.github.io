クイックスタートガイド：I2P経由のGitHub

「Githubは再びブロックされました」という出版物に触発されました。



githubのブロックに関するニュースは、松葉杖を作ることを考えさせてくれました。



何らかの理由で、すぐにI2Pの考えが浮かびました。



そして、それは本当に簡単であることが判明しました。



したがって、次の手順で：



1. I2Pをダウンロードしてインストールします。

2.開始後、I2Pネットワーク設定に移動します： 127.0.0.1 ：7657 / confignet;

3.「UDP設定」セクションで、UDPポート番号を見つけて、ルーターでリダイレクトします。

4. I2Pトンネルマネージャーに移動します： 127.0.0.1 ：7657 / i2ptunnelmgr;

5.チェック：I2P HTTPプロキシ（127.0.0.1-00-00444）およびI2P HTTPSプロキシ（127.0.0.1:4445）のトンネルがあるはずです。

6.ブラウザの1つ、たとえばMozillaで、プロキシ設定を指定します：HTTPプロキシ-127.0.0.1-00-00444およびSSLプロキシ-127.0.0.1:4445（Mozillaは、システムの設定とは異なるプロキシ設定を指定できるという点で便利です）。 したがって、githubのWebインターフェースにアクセスできます。

7. Gitコンソールに移動して、次のコマンドを実行します。 



git config --global http.proxy 127.0.0.1:4445 git config --global https.proxy 127.0.0.1:4445


できた これで、コミットとプッシュを行うことができます。



コンソールとTortoise Gitの両方で機能します。



唯一のものはかなり遅いです。
