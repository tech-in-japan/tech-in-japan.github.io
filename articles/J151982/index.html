OpenWRTで問題なくHTTP経由で何でも有効にする



「インターネットアウトレット」の製造経験に関する記事へのコメントで、 2つのコメントが私の注目を集めました。 そのようなことは本質的に役に立たないと主張したものは、私には不公平に思えました-例えば、あるトリッキーな小さなものをリモートで「リセット」する必要があったことを非常によく覚えており、私は一人ではないと思います。 しかし、たとえばTP-LINK TL-MR3020 + OpenWRTを使用することで、このような問題を解決できるという考えは簡単で安価です。 また、デバイス、プログラミング、およびマイクロコントローラーを分解せずに実行することを決定しました。要するに、ソリューションをできるだけアクセスしやすくするためです。 そして、ほとんど成功しました！



出力デバイスとして、通常のUSBキーボードを使用します。 スーパーマーケットでばかげた価格で嫌な品質のキーボードに出会ったとき、私はいつも疑問に思っていました-なぜそれが必要なのですか？ はい、何のために！ キーボードには3つのLEDがあり、このキーボードをサポートするデバイスからプログラムで制御できます。 LEDを制御回路に置き換えると、3つの負荷を制御できます。 おまけとして-入力デバイスとして使用するためのデバイスのさらなる改良の可能性。 もちろん、銃からスズメを撃つこと、マイクロコントローラー上で最も簡単なデバイスを組み立てる（または購入する）ことはできません。これは紛れもない利点（入力/出力の増加、通常の論理レベルなど）を持ちます。 しかし、「キーボード」アプローチには利点もあります。これは、プログラマーを持たないMKに不慣れな人々が利用できます。 今、あなたは今日、機能するデバイスを手に入れて作ることができます。 さらに、次の利点があります。

  • このソリューションは他のプラットフォームに簡単に移植できます。私はテストしていませんが、理論的には「ASUS」または「D-Link」に閉じ込められ、隅からインターネットを平和的に配布できます（USBがある場合）。
  • ルーターは本来の目的に使用できます。
  • ルーターを分解して回路を妨害する必要はありません（そして、TL-MR3020は、分解するのは簡単ではありません）。
  • プログラミング（小さなシェルスクリプトとしてのプログラムを除く）とコンパイルは必要ありません。
  • すべてのコンポーネントはコンピューターストアで購入できます（「LEDを点滅させる」だけの場合、実際の制御のために使用可能な無線コンポーネントをいくつか購入する必要があります）が、総コストは1,500ルーブル未満です。


OpenWRTを配置し、ネットワークインターフェイスを構成します


このステップでは、実際にルーターが必要です。 私は850rかかりました。

openwrt.orgのモデルページには、包括的な点滅手順が記載されており、トランクスナップショットを使用できます。 私自身、 ti病な態度でベータ版の姿勢調整12.09を行うことを提案します。 さらに、公開時点では、openwrt Webサイトでスナップショットは利用できず、AA12.09がついに公開されました。

インストールは、いくつかの基本操作で構成されます。
  1. PCのネットワーク接続を設定して、IPアドレスを自動的に取得し、ルーターをイーサネットケーブルで接続します（点滅後、WiFiインターフェースは無効になります）
  2. ルーター管理ページにログインします（デフォルトアドレスは192.168.0.254ユーザー：admin、パスワード：admin）
  3. [システムツール]> [ファームウェアアップグレード]タブで、以前にダウンロードしたファームウェアを選択し、[アップグレード]をクリックして、ダウンロードインジケーターが2回100％に達するまで待ちます。
  4. OpenWRTネットワークパラメータはデフォルトでTP-LINKのパラメータとは異なるため、インターフェイスを再初期化する最も簡単な方法は、イーサネットケーブルを切断して再接続することです。
  5. telnet（192.168.1.1-デフォルトでは「新鮮な」OpenWRTのアドレス）経由でルーターにアクセスし、passwdコマンドでパスワードを設定します-これでルーターはSSH経由でアクセスできます（telnet経由ではアクセスできません）。




追加のパッケージをインストールするには、インターネット経由でリポジトリにアクセスする必要があります。これには、ローカルネットワーククライアントとしての構成が必要です。

例：
192.168.1.xセグメントで静的IPを構成するには、次の手順を実行します。 

uci set network.lan.proto=static uci set network.lan.ipaddr=192.168.1.222 uci set network.lan.netmask=255.255.255.0 uci set network.lan.gateway=192.168.1.1 uci set network.lan.dns=8.8.8.8 uci changes uci commit /etc/init.d/network restart


ここで、192.168.1.1はネットワーク内のゲートウェイアドレスであり、192.168.1.222は未使用のIPであり、ボックスに割り当てられます。 uci changesコマンドは、行われたすべての変更を表示する機能を提供します。 インターフェースが正しく構成されていないデバイスは、外部からアクセスできないため、「ゾンビ」になるため、この可能性を無視しないようにしています。 すべて同じトラブルが発生した場合、OpenWRTは「リカバリモード」を提供します。ロード中、WPSボタンが点滅し始めたらすぐに押し続けます-MR3020はネットワークのデフォルト設定で起動します。

ネットワークでDHCPが実行されている場合、自動構成を設定できます。 

 uci set network.lan.proto=dhcp uci set network.lan.hostname=etherelay uci commit /etc/init.d/network restart


2行目はオプションですが、デバイスを見つけるのに非常に役立ちます。 ネットワークがサポートしていれば、IPではなくホスト名でデバイスにアクセスできます。 ネットワーク上のデバイスの検索を100％回避したい場合は、静的構成を使用します。

TP-LINKに組み込まれている組み込みのdhcpサーバーを無効にすることも役立ちます。 

 uci set dhcp.lan.ignore=1 uci commit /etc/init.d/dnsmasq restart


それで、ボックスをコンピューターから切断し（とにかく、sshセッションは最後のコマンドの後で既に終了しています）、ルーターまたはスイッチに接続できます。

キーボード（または別の入力デバイス）で発生するすべてについて、対応するファイルに表示される入力イベントが生成されます（この場合/ dev / input / event0ですが、これは他の入力デバイスが接続されていないという事実のための特殊なケースです）。 イベントの構造は、ヘッダーファイルinput.hで定義されています。 

  struct input_event { struct timeval time; __u16 type; __u16 code; __s32 value; };


typeは入力要素のタイプ（キーボードボタンまたはマウス/ジョイスティックの動きなど）を示し、codeは各タイプに固有の要素コード（たとえば、EV_KEYキーボードイベントのキー番号はここに送信されます）、および値はそれぞれ生成された効果と大きさ（これをサポートするデバイスの場合）。 たとえば、キーボードのQキーを押すと、次のようになります。

スキーム5.電気機械式リレー制御

ソリッドステートリレーは非常に高価であり、より強力なほど高価です。 従来のリレーは、より多くの「ルーブルあたりのアンペア」を与えますが、欠点があります。 その中には、オン状態での大きな電流消費があります。 図と同じモデルの3つのリレーを同時にオンにすると、USBから消費される電流はほぼ300 mAになります。 ポートをあまり「ミルク」したくない場合は、エミッター（図の破線）ではなく、独立したソースにエミッターを接続します。

スキーム4と5は、もちろん、「共通アノード」（ボードのアノードダイオードと「プラス」の間の抵抗をチェックします-0オームでなければなりません）のスキームに従ってLEDがオンになっているキーボードでのみ動作します。 これに煩わされたくない場合は、キーの代わりに低電力の光リレーを使用できます。 通常のLEDの代わりに点灯し、LEDごとに5〜25 mAの電流で動作します。つまり、99％の確率ですべてのキーボードオプションに適しています。 最大60 Vの電圧と最大300 mAの電流を切り替える必要がある場合は、リレーを取り外して、光リレーを直接使用できます。
