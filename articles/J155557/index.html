ポンプ用手押し車-D-Link DNS-325の機能をカスタマイズおよび拡張

NAS ネットワーク上で利用可能なファイルストレージ、バックアップを排出する場所、必要なデータを1か所に保存する機能。 しかし、それだけではありません！ 本日、DNS-325に新しい機能を提供します。 同じシリーズのデバイス（DNS-320、DNS-345、DNS-346など）の所有者も気分を害することはありません。 1ダース以上の写真とスクリーンショットの猫の下で、注意してください！





なぜこのNASなのか？ まず、それはかなりまともな特性を持っています：ほとんど静かな冷却、SATA2インターフェースを備えた2台のディスク（誰もが3台のディスクを必要とするわけではありません、1台のディスクでは十分ではありません）、1.2 GHzプロセッサーおよび256 RAM。 さらに、独自の実際のLinuxがインストールされているため、ほとんど何でも実行できます。 事前に警告したい-私はLinuxの経験があまりないので、どこかで用語を知っていたり、何かを忘れたり、追加のステップを追加したりしたら、喜んでお話しします！
0.ファームウェアの更新
まず、ファームウェアを更新することを強くお勧めします。幸いなことに、最新バージョンでは作業が大幅に安定し、FTPサーバーの作業など、多くのバグが削除されます。 更新は非常に簡単です-公式FTPからファームウェアをダウンロードしてください-ftp : //ftp.dlink.ru/pub/NAS/DNS-325/Firmware/ （最新バージョン1.03を書いている時点で、ファイルftp://ftp.dlink.ru/ pub / NAS / DNS-325 / Firmware / DNS-325_A1_FW_v1_03.rar ）、解凍（内部にtarファイルがあります）、システムのこのページに移動して更新します：




1.車輪について知ることが重要です
  1. ファイルシステムの形式はEXT3です。 オプションなし。 別のものがある場合は、再フォーマットする必要がありますが、NASで自分で行うことができます。
  2. USBネジとフラッシュドライブのファイルシステム形式は、EXT3とFAT / NTFSの両方です。
  3. Axisを制御するスクリプトは、最初のネジ（Volume1）に配置されています。NASを見ると、右側にあります。

5. fun_plugを設定します（NASへのSSHアクセス）
telnetおよびssh-clientsの前に注意してください-私はWindowsの下で平凡なパテを使用しますが、それらの多くは私が特定のものを推薦する勇気がありません。
  1. そのため、まずfun_plugファイル（ http://wolf-u.li/u/433 ）およびfun_plug.tgz（ http://inreto.de/dns323/fun-plug/0.7/arm/を最初のハードドライブにアップロードしますfun_plug.tgz ）
  2. NASを再起動します-これで、telnet経由で接続できます。 名前またはアドレスで接続-パスワードなし！







  3. ルートパスワードを設定する 
    usermod -s /ffp/bin/sh root mkdir -p /ffp/home/root/ sed -ie 's#:/home/root:#:/ffp/home/root:#g' /etc/passwd pwconv
    「usermod」：「usermod：no changes」または「pwconv」：「pwconv：/ etc / passwd-のモードを0600に変更できませんでした」というエラーが表示される場合は無視してください。 
     passwd
    新しいパスワードを入力してください。 
     login
    ルートログインとパスワードを入力します。応答として「メールなし」メッセージが表示されます。 パスワードを保存するには、次の手順が必要です。 
     wget http://wolf-u.li/u/172/ -O /ffp/sbin/store-passwd.sh store-passwd.sh
    SSHを有効にする 
     chmod a+x /ffp/start/sshd.sh sh /ffp/start/sshd.sh start
    そして今-SSHを介して別のウィンドウで接続し、キーを保存するという質問に肯定的に答え、すべてが順調に進んだ場合-最初のウィンドウでtelnetサービスの自動開始を無効にします。 
     chmod -x /ffp/start/telnetd.sh
    準備が完了しました-これで、システムに対する完全なアクセスと完全な制御が可能になりました。 突然fun_plugを突然削除する必要がある場合、
    1. ルートfun_plugのファイルを削除します
    2. 再起動
    3. ffpディレクトリを削除します

7.Midnight Commanderのインストール（ファイルマネージャー）
なぜ必要なのですか？ アレイ内のファイルとフォルダーに対して操作を実行します。 私はそれをブラウザインターフェースよりも好きです、それはより速く動作します、あなたはバックグラウンドでそれを実行し、ログアウトすることができ、それはそれ自体にコピー/移動します。
  1. 最初は、ロシア語の文字、色、カーソルキー（2つ、または3つを選択）で常に問題がありましたが、次のアルゴリズムが開発されました。 実際に、インストールします： 
     ipkg install pcre glib e2fsprogs slang libpng mc
  2. /ffp/etc/profile.d/フォルダーに次の内容のmc.sh（または何でも）と呼ばれるスクリプトを作成します。 
     #!/ffp/bin/sh export TERM="xterm" export TERMINFO="/ffp/share/terminfo" export LANG="en_US.utf8"
  3. コマンドを実行する 
     chmod a+x /ffp/etc/profile.d/mc.sh
    その後、移転します。
  4. これで、コマンドでコマンダーを実行できます 
     mc
    また、ロシア語のファイル名と通常のキー管理の両方がサポートされます。 ロシア語の名前にまだ問題がある場合は、このメニューに移動するとすべてが正常になります。











8.インストール画面（バックグラウンドでコマンドを実行）
画面は、背景で機能する「画面」を作成できる非常に便利なユーティリティです。 全体の魅力は、接続が中断されるか、パテを閉じると、ユーザーのセッションがバックグラウンドに残ることです。 たとえば、何かをダウンロードしたり、フォルダをディスクからディスクに移動したりする必要がある場合に適しています。 接続し、画面を起動し、コマンドを発行し、切断します。NASは、ガタガタと音を立てて指示に従います。
  1. インストールする 
     ipkg install screen
  2. /ffp/etc/profile.d/フォルダーと次の内容にscreen.sh（または何でも）と呼ばれるスクリプトを作成します。 
     #!/ffp/bin/sh alias screen="screen -U"
    これは、UTF8モードで画面を自動的に起動するために必要です。 ログインしましょう。


今-それを使用する方法。

screen-スタート画面

screen -R-最後に実行中の画面に戻ります。 複数の画面がある場合-番号で始まるリストが発行されます-screen -R [number]コマンドは特定の画面に移動します。

ctrl + a D-画面を閉じずに終了します（ウィンドウからフック解除します）

ctrl + ak-現在のウィンドウを閉じます

9. Transmissionのインストール（トレントダウンロード）
NASにはトレント用のロッカーが組み込まれていますが、その機能の品質は理想とはほど遠いため、そのような機能が必要な場合は別のデバイスをインストールすることが非常に望ましいです。
  1. コマンドを実行する 
     slacker -i
    トランスミッションとカールを選択し、「OK」をクリックします。
  2. 次のコマンドを実行します 
     chown -R nobody /mnt/HD/HD_a2/torrent mkdir /ffp/var/transmission mkdir /ffp/var/transmission/incomplete chown -R nobody /ffp/var/transmission
  3. ファイル/ffp/start/transmission.shを編集し、パスを次のように変更します。 
     TRANSMISSION_HOME=/ffp/var/transmission
  4. configの最初の起動と初期化のコマンドを実行します。 
     su nobody -c "transmission-daemon -f -g /ffp/var/transmission -w /mnt/HD/HD_a2/torrent -t -u admin -v password -a 127.0.0.1,192.168.*.*"
    adminはWebインターフェースを操作するためのユーザー名、 passwordはそのパスワード、 127.0.0.1,192.168。*です。*は、Webインターフェースに入ることができるIPのリストです。開始を待ってから、を押してシステムをシャットダウンします。 CTRL + C
  5. 次-カスタマイズ。 これを行うには、ファイル/ffp/var/transmission/settings.jsonを編集します

    私たちにとって最も興味深い行： 
     "incomplete-dir": "/ffp/var/transmission/incomplete",
    過負荷ファイルのディレクトリ 
     "incomplete-dir-enabled": true,
    未完成のファイルをこのディレクトリに保存する 
     "peer-port": 11000, "peer-port-random-high": 11999, "peer-port-random-low": 11001,
    ダウンロードおよびアップロード用のポート。 
     "rpc-port": 9091,
    Webインターフェースが使用可能になるポート。 たとえば、この場合、ネットワーク内のアドレスhttp://192.168.0.200:9091で動作します 
     "rpc-whitelist": "127.0.0.1,192.168.*.* "
    これらのIPでのみWebインターフェースにアクセスできるようにします 
     "rpc-whitelist-enabled": true,
    IPによるコントロールパネルへのアクセス制限を有効にします。
  6. インストールを完了します。 
     chown -R nobody /ffp/var/transmission chmod a+x /ffp/start/transmission.sh /ffp/start/transmission.sh start
さらに、ポートを破棄することを忘れないでください。DIR-615では、設定は次のようになります。

アップデート1 。 判明したように、メディアファイルの自動インデックス再作成を開始することはまだ可能です— upnpdbmgrが責任を負います。 これを行うには、タスク管理モードに切り替えます（このコマンドは、mcエディターを介して便利なタスク管理に必要です）。 
 EDITOR=mcedit crontab -e
最後に、この種のコマンドを最後に追加します（1時間ごと、5分ごとにベースのインデックスを再作成します） 
 5 * * * * /usr/local/modules/sbin/upnpdbmgr --mode=add [   ]
実際のコマンドの例： 
 5 * * * * /usr/local/modules/sbin/upnpdbmgr --mode=add /mnt/HD/HD_a2/torrent /mnt/HD/HD_a2/series /mnt/HD/HD_a2/movie
既に削除されたファイルからデータベースを削除するには、次のコマンドを使用します 
 /usr/local/modules/sbin/upnpdbmgr --mode=remove-all
