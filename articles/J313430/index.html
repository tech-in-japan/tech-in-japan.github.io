友人、こんにちは！

VMwareの例を次に示します。彼らはGigaOmとVMwareが所有するwhychooseview.comリソースによる研究に焦点を当てています。 そのため、これはマーケティングレポートまたは特別なアイデアであり、分析レポート（上記のリンクはクリック可能）で、VMwareは常に詳細な説明を「指さし」ます。



例：



「Citrixは、これほど幅広いサポートを提供していません。 真のDaaSオファリングを持たないという点で制限があり、Citrixは、オンプレミスおよびクラウド展開のための完全なハイブリッド展開オプションを提供しません。



そのようなもの.... （1つの「NOT ...」）これは、別の世界に関する超大国の哀れみに似ています。 何にも似ていませんか？

-CitrixはVMware SDDCをサポートするためにゼロから構築されたわけではありません

-Citrixは同じ幅のサポートを提供していません

-.......

Citrixは、テーブル比較の支持者です。



はい-有益、はい-明らかに、しかしこれは究極の真実ですか？ 真実はどこにあり、誰を選ぶべきか？ 2016年2月19日に、ガートナーは分析レポート 「デスクトップ仮想化の展開でCitrixとVMwareを選択する方法」をリリースしました。 かなりの時間が経過し、Citrix側とVMware側の両方からの評価は続きませんでした。



私の意見では、この分析レポートは、時間が経つにつれて、どの製品やベンダーにも傾いておらず、偏りに気づかれていませんでした（これは、二面的なWADAではありません ）。

最適なソリューションを選択するための基準を特定するために、このレビューの主要な点について詳しく説明します。



それでは始めましょう



処分の分析として、ガートナーはVDI市場でのCitrixの主導的地位に疑いの余地はありませんが、VMwareは追いついています。 （これは、x86サーバー仮想化インフラストラクチャ2016のマジッククアドラントレポートでのMSとVMwareの位置であり、Microsoftはキャッチアップの役割です）。 VMwareアナリティクスによると、彼らは自分自身をリーダーと考えています）。









さらに、このレポートでは、両方の決定の基礎となる長所または短所について説明します。これについては、「ほとんど、最も」というタイトルの各申請者に関して検討します。



ガートナーの専門家がレビューした両方の製品のプラスとマイナスのポイントに関する記事を分析します（ただし、すべてのポイントではなく、最も興味深いポイント）。



チャレンジャー1-Citrix：



+ Citrixの強みは、ネイティブまたはクラウド展開のアーキテクチャと、Citrix NetScalerとの統合によるアプリケーション配信の制御です。 CitrixよりもICAプロトコルをよく知っている人、そしてちなみにVMwareはこれらの目的のためにF5を使用します（ちなみに、F5はサードパーティのソリューションであり、NetScalerの直接の競争相手です）。



PS NetScalerはアプリケーション配信コントローラーの中でトップの位置を占めており、多くのコンポーネントのバランスをとる役割を果たします+セキュリティゲートウェイであるため、一部のHorizo​​nユーザーもそれを選択します。



+ Citrix XenApp / XenDesktopのテクノロジーパブリッシングアプリケーションは、市場に長い間存在している成熟したソリューションです（ちなみに、これはVMwareがMSに関してハイパーバイザーを支持していることを主張しているものです）。



+グラフィックシステムのパフォーマンスを向上させるために、グラフィックアダプターによるNvidia Grid vGPUテクノロジーのサポートがあります。 ここで、Citrixは再び先駆者です。



+ Citrix XenDesktopは、さまざまなハイパーバイザー（XenServer、Hyper-V、Acropolis、vSphere、さらにはすぐに使用できるさまざまなクラウド）に実装できます。



-Citrixは、実装と管理がより困難であると考えられています。 （まあ、ここでは確かに「ファン上」です）



-独自のクラウド、VDIはありません



-CitrixとNvidia vGPUの利点は、VMwareおよびHyper-Vよりも一般的ではないXenServerハイパーバイザーでのソリューションの実装に基づいています。



ここでは同意できない VMwareハイパーバイザーのバージョン6以降、Nvidia Grid機能はXenDesktopバンドルでも使用できます。



ドキュメントを読む：「VMware vSphere 6でNVIDIA GRIDグラフィックスボードを使用した、Citrix XenDesktopによるGPU仮想化」



さらに、3Dグラフィックスをサポートする必要があるが、高性能グラフィックスサブシステムを必要としないユーザーのために、XenServerハイパーバイザーと組み合わせてIntel Virtualization Graphics Technology（GVT-g）を使用できます。

「ハードウェアには依然として、予想以上の大きな投資が必要であり、コンピューティングPCの分散モデルに必要なコストよりもはるかに高くなっています。
