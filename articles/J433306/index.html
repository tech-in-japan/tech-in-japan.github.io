Visual Studio 2019

最近では、 Microsoft Connectで（）; 2018年 、Visual Studio 2019 Preview 1のリリースを発表しました。これは、次のVisual Studioの最初のPreviewバージョンです。 その中で、スタートアップの高速化とgitリポジトリに保存されたプロジェクトの操作、人工知能（AI）（Visual Studio IntelliCode）によるIntelliSenseの改善、Live Share統合による同僚とのコラボレーションの簡素化など、いくつかの重要な領域に焦点を当てました。 。 詳細はこちら！







仕事に集中する



すぐに、Visual Studio 2019が新しい開始ウィンドウで開きます。 ローカルリポジトリでも、GitHub、Azure ReposなどのオンラインGitリポジトリでも、最新のGitリポジトリでより便利に動作するように設計されています。







もちろん、既存のプロジェクトを開いたり、新しいプロジェクトを作成したりできます。 （ノベルティは、まもなくVisual Studio 2019 for Macでも利用可能になります。）





VS19 for WindowsおよびMac



また、本日、既存のクイック起動フィールドに代わるVisual Studio 2019の新しい検索機能を共有しています。 設定、コマンド、インストールオプションを検索できるようになりました。 新しい検索もよりインテリジェントです。 エラーを含む式の検索をサポートしています。







Visual Studio 2019は、コードの高速化に役立ちます。 このプレビューでは、コードの一貫性と保守性を実現する利便性に焦点を当てました。 LINQクエリでforループを変更したり、タプルを名前付き構造に変換してコードを良好な状態に維持しやすくするなど、新しいリファクタリング機能を追加しました。 新しいドキュメントヘルスインジケータとコードクリーニング機能を使用すると、ボタンをクリックするだけでエラーを簡単に特定して修正できます。







定期的なデバッグも簡単になりました。 パフォーマンスが改善され、デバッグプロセスがスムーズになります。 また、Autos、Locals、Watchに検索機能を追加し、オブジェクトと値の追跡を支援しました。 今後の記事では、「タイムトラベルデバッグ」機能、Azure Kubernetesサービス（AKS）および仮想マシンスケールセット（VMSS）の最新のスナップショットデバッガーなどのデバッガー領域の更新について詳しく説明します。アウトプロセスの64ビットデバッガーのおかげで、大規模なC ++プロジェクトをデバッグできます。







チームの共同作業を支援する



Visual Studio 2017で開始した作業に基づいて、AIインターフェイスを備えた状況依存IntelliSenseであるVisual Studio IntelliCodeを改善し、独自のコードリポジトリでトレーニングしてチームと結果を共有できるようにします。 IntelliCodeは、特定のケースごとに状況に応じた推奨事項を提供することで、必要なクリック数を削減します。 これについては、 こちらで詳しく説明しました 。







今年の初めに、 Visual Studio Live Shareを導入しました。このツールは、Visual StudioまたはVisual Studio Codeを使用して世界中のユーザーとリアルタイムでやり取りするのに役立ちます。 Live ShareはVisual Studio 2019にデフォルトでインストールされるため、同僚をすぐにコーディングに参加させてエラーを修正したり、迅速な変更を行ったりできます。







最後に、Visual Studio 2019では、プルリクエスト（PR）を使用するまったく新しい方法を提供します。これにより、IDEを離れることなく、チームからプルリクエストを確認、実行、さらにはデバッグすることができます。 今日、Azure Reposのコードをサポートしていますが、まもなくGitHubに拡張する予定です。 開始するには、Visual Studio MarketplaceからVisual Studio 拡張機能のプルリクエストをダウンロードできます。







.NET Core 3プレビュー1



また、.NET Core 3 Preview 1のリリースを発表しました。VisualStudio 2019は、あらゆるプラットフォーム向けの.NET Core 3アプリケーションの作成をサポートする製品になります。 もちろん、C ++でのクロスプラットフォーム開発、およびXamarinを使用したiOSおよびAndroid用の.NETモバイルアプリケーションの開発も引き続きサポートおよび改善します。
