Qtについて。 コンテナライブラリ

こんにちは、Habr！



今日は、おもしろくて便利な機能についてお話したいと思います。 彼女の名前はコンテナライブラリです。 これは1つではなく、便利な機能のグループ全体です。 そして、それらの目的は、要素のグループの編成と処理です。 面白いですね。 では、詳しく見てみましょう。猫へようこそ。



エントリー



ひとつだけ言いたいことがあります-この記事はプログラマーの初心者を対象としています。 続けます。


プログラミングで最も一般的なタスクの1つは、要素のグループの処理を整理することです。 プログラマーは彼女と一緒に多くの蒸気を取る。 結局のところ、すべてを棚に置く必要があります...しかし、Qtには素晴らしいヘルパーがあります-コンテナライブラリです。 このヘルパーはQtCoreモジュールにあります。したがって、追加のインクルードは必要ありません。 それでは、行きましょう！



QVectorベクトル



ベクトルは、通常の配列に非常によく似たデータ構造です。 ただし、ベクトルクラスを使用すると、単純な配列に比べていくつかの利点があります。 たとえば、ベクトル内の要素の数（サイズ）を確認したり、動的に拡張したりできます。 このコンテナは、他のタイプのコンテナよりも経済的です。 順次コンテナの最後に要素を追加するには、 push_back（）メソッドを使用できます。 ベクトルの要素には、インデックス演算子[]と反復子の両方を使用してアクセスできます。 例： 



QVector<int> vec; vec.push_back(10); //      vec.push_back(20); vec.push_back(30); qDebug() << vec;


推定されます-QVector（10,20,30） 。



QByteArrayバイト配列



このコンテナはQVectorに非常に似ていますが、違いはテンプレートクラスではなく、サイズが1バイトの要素のみを格納できることです。 中間データストレージが必要な場合は、タイプQByteArrayのオブジェクトを使用できます。 配列要素の数はコンストラクターで設定でき、[]演算子を使用して配列要素にアクセスできます。 



 QByteArray arr(3); arr[0] = arr[1] = 0xFF; arr[2] = 0x0;


QByteArrayクラスのオブジェクトのデータに圧縮操作と逆変換を適用することもできます。 これは、2つのグローバル関数qCompress（）およびqUncompress（）を使用して実現されます。 データを圧縮および解凍するだけです： 



 QByteArray a = "Some text..."; QByteArray aCompressed = qCompress(a); qDebug << qUncompress(aCompressed);


一部のテキスト...が表示されます。



QBitArrayビット配列



このクラスはビット（またはブール ）配列を管理します。 格納された各値は、余分なメモリを無駄にすることなく、1ビットのみを取ります。 この型は、 bool型の変数を多数格納するために使用されます。 

 QBitArray arr(3); arr[0] = arr[1] = true; arr[2] = false;




QListおよびQLinkedListリスト



リストは、関連するアイテムの順序付けられたセットであるデータ構造です。 ベクトルとキューに対するリストの利点は、これらの操作を実行するために最小限のポインターのみが変更されるため、任意の位置で要素を挿入および削除する方が効率的であることです。唯一の例外はリストの中央に要素を挿入することです。 しかし、欠点があります-リストはインデックスによってリストの特定の要素を検索するのにあまり適していないため、この目的にはベクターを使用するのが最適です。



リストはQListテンプレートクラスによって実装されます。 一般的に、このクラスは要素へのポインタの配列です。



リストを操作するための特定の操作：

  • move（） -要素をある位置から別の位置に移動します。
  • removeFirst（） -リストの最初の要素を削除します 。
  • removeLast（） -リスト内の最後のアイテムを削除します 。
  • swap（） -指定された位置でリストの2つの要素を交換します。
  • takeAt（） -指定された位置のアイテムを返し、リストから削除します。
  • takeFirst（） -最初の要素を返し、リストから削除します。
  • takeLast（） -最後のアイテムを返し、リストから削除します。
  • toSet（） -リストに含まれるデータを含むQSetコンテナを返します。
  • toStdList（） -リストの要素を含む標準STLリストstd :: Listを返します。
  • toVector（） -リストに含まれるデータを含むQVectorベクトルを返します。


要素の値を変更しない場合、効率上の理由から、[]演算子の使用は推奨されません。 代わりにat（）メソッドを使用します。これは、要素への定数参照を返すためです。



最も一般的な操作の1つは、リストを走査して、各リスト項目の値を順番に取得することです。 例： 

 QList<int> list; list << 10 << 20 << 30; QValueList<int>::iterator it = list.begin(); //         while (it != list.end()) { qDebug() << "Element:" << *it; ++it; }




コンソールには以下が表示されます。

要素：10

要素：20

要素：30



大きなリストを使用する場合や、要素を挿入する必要がある場合は、二重にリンクされたQLinkedListリストを使用する方が効率的です。 このコンテナーはQListよりも大食いですが、挿入または削除操作は、削除または挿入される要素の位置に関係なく、4つのポインターを再定義することになります。



qstackのスタック



スタックは、Last In First Out-last come、first goの原則に基づいて機能するデータ構造を実装します。 つまり、他のすべての要素の後に挿入された要素が最初にスタックから削除されます。



QStackクラスは、スタック構造の実装です。 このクラスはQVectorから継承されます 。 スタックにアイテムを置くプロセスは、通常pushと呼ばれ、そこから一番上のアイテムをポップすることは poping と呼ばれます。 プッシュ操作ごとにスタックサイズが1ずつ増加し、プッシュ操作ごとに1ずつ減少します。これらの操作では、 プッシュ（）およびポップ（）関数がQStackクラスで定義されます。 top（）メソッドは、トップ要素へのリンクを返します。 スタックの使用例： 

 QStack<QString> stk; stk.push("Era"); //        stk.push("Corvus Corax"); stk.push("Gathering"); while (!stk.empty()) { qDebug() << "Element:" << stk.pop(); }

QQueueキュー



キューは、原則に従って動作するデータ構造を実装します-先着順です。 QListから継承したQQueueクラスにキューを実装しました 。



次の例は、キューの使用例を示しています。 

 QQueue<QString> que; que.enqueue("Era"); que.enqueue("Corvus Corax"); que.enqueue("Gathering"); while (!que.empty()) { qDebug() << "Element:" << que.dequeue(); }

辞書QMap、QMultiMap



辞書は、原則として、日常生活で使用される普通の辞書に似ています。 キー値でインデックス付けされた同じタイプのアイテムを保存します。 辞書の利点は、特定のキーに関連付けられた値をすばやく取得できることです。 重複を許可する複数の辞書を除き、キーは一意でなければなりません。



このタイプのコンテナでは、要素を見つけるためのキーとともに入力します。キーは任意のタイプの値にすることができます。 QMapディクショナリの場合、同じキーを持つ2つの異なる要素が入力されないようにする必要があります。これらの要素の1つを抽出できないためです。 各キーは一意である必要があります。



QMapオブジェクトを作成するときは、そのサイズをコンストラクターに渡す必要があります。 このサイズは、他のコンテナクラスで通常行われているように、要素の最大数を制限するサイズではありませんが、位置の数を表します。 位置の数は、格納されると予想される要素の数よりも大きくする必要があります。そうしないと、辞書内の要素の検索が十分に効果的に実行されません。 この場合、要素の配置がより便利になるため、この値は素数のカテゴリに属する​​ことが望ましいです。



いくつかの方法：

  • iowerBound（） -指定されたキーを持つ最初の要素に戻ります。
  • toStdMap（） -辞書の要素を持つSTL辞書を返します。
  • upperBound（） -指定されたキーを持つ最後の要素に戻ります。


辞書項目にアクセスする最も一般的な方法の1つは、[]演算子でキーを使用することです。 ただし、キーと値はイテレータメソッドkey（）とvalue（）を介して取得できるため、これはなくてもかまいません。たとえば、 

 QMap<QString,QString> mapPhonebook; mapPhonebook["Piggy"] = "+380 93 785 11 11"; mapPhonebook["Kermit"] = "+7 85 123 65 56"; mapPhonebook["Gonzo"] = "+49 73 631 32 21"; QMap<QString,QString>::iterator it = mapPhonebook.begin(); for(;it != mapPhonebook.end(); ++it) { qDebug() << "Name:" << it.key() << "Phone:" << it.value(); }

名前：カーミット電話：+7 85123 65562 21

名前：貯金箱電話：+380 93 785 11 11



QHashおよびQMultiHashハッシュ





ハッシュの機能はQMapと非常に似ていますが、唯一の違いは、キーでソートする代わりに、このクラスがハッシュテーブルを使用することです。 これにより、 QMapよりもはるかに高速にキー値を検索できます。



QMapと同様に、要素が存在しないキーを指定すると要素が作成されるため、インデックス演算子[]を使用する場合は注意が必要です。 したがって、コンテナのcontains（）メソッドを使用して、キーにバインドされた要素の存在を常に確認することが重要です。



独自のクラスのオブジェクトをQHashに配置する場合は、クラスに比較演算子==と特殊関数qHash（）を実装する必要があります。 比較演算子の実装例は次のとおりです。 

 inline bool operator==(const MyClass& mc1, const MyClass& mc2) { return (mc1.firstName() == mc2.firstName() && mc1.secondName() == mc2.secondName() ); }




qHash（）関数は、ハッシュ内の各アイテムに対して一意でなければならない数値を返す必要があります。 例： 

 inline uint qHash(const MyClass& mc) { return qHash(mc.firstName()) ^ qHash(mc.secondName()); }




QMultiHashクラスはQHashから継承されます 。 同じキーで値を配置することができ、一般にQMultiMapに似ていますが、親クラスの詳細を考慮します。



多くのQSet



ドイツの数学者ゲオルク・カントールが言ったように、「多くは多く、私たちは精神的には1つとして意味します。」 これは、Tulipのコンテキストでは、要素を何らかの順序で書き込み、値を非常に迅速に表示し、ユニオン、インターセクション、差などのセットの特性である操作を実行する機能を提供するQSetコンテナーにすぎません。 前提条件は、キーが異なる必要があることです。 QSetコンテナは、データをすばやく取得するための順序なしリストとして使用できます。



セットで実行できる操作：結合、交差、差。



2つのセットを作成し、それらに要素を書き込みます。 

 QSet<QString> set1; QSet<QString> set2; set1 << "Lorem" << "Ipsum" << "Dolor"; set2 << "Sit" << "Amet" << "Lorem";




これら2つのセットを結合する操作を実行し、セットの要素が変更されないようにするために、中間のsetResultセットを導入します。 

 QSet<QString> setResult = set1; setResult.unite(set2); qDebug() << " = " << setResult.toList();




以下が画面に表示されます。



Union =（「Lorem」、「Ipsum」、「Dolor」、「Sit」、「Amet」）





同様に、 交差（）と差（ 減算（））が実行されます。
