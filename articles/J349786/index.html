Rustの簡単な歴史：StackOverflowによる趣味から最も人気のあるYPまで

Rustは、作成者が並行性、速度、セキュリティの3つのことに注意を払ったシステムプログラミング言語です。 また、Rust は若いプログラミング言語と見なされていますが、その最初の安定バージョンは2015年にリリースされましたが、10年以上にわたって開発されてきました。



今日は、過去を調べてRust言語の歴史を語り、開発期間中にRust言語の機能がどのように変化したかを示し、このPLを実践する具体例を示したいと思います。

個人プロジェクト（2006–2010）



それ自体から未来を救うことを目的とする過去の技術



-Rust Developer 、Graydon Hoare
これは、2016年のACM会議でのプレゼンテーション中にRustプロジェクト開発チームからSteve Klabnikが発言したGraidon Horの引用の1つです（プレゼンテーションのスライドはこちらにありますが、スライド間を移動するには、キーボードの矢印）。 これらの言葉は、Rustが高度な機能を備えた革新的な言語ではないという事実をよく反映しています。 「古い」言語の多くの作業メソッド（C ++を含む）が含まれているだけで、セキュリティが向上しています。



Graidon Chorus は 、2006年にRustの開発を開始しました。当時、彼はMozillaの従業員であり、空き時間にプロジェクトに関与していました。 グライドンによれば、舌は「単一障害点」を持たず、例外的な生存可能性を有する「分布生物」であるため、 さび科の真菌にちなんで命名されました。 このようなキノコはライフサイクルに5つの段階があり、すぐに芽が出ます。 合唱団は、これが安全性と速度に焦点を当てたPLの良い類推だと判断しました。



グライドンは言語に従事しているため、特定のルールを確立しました。 彼は 、そもそも言語のセマンティクスに注意を払うことが必要であり、構文の作業が最後であることを指摘しました。 したがって、Rustの初期の実装では、キーワードは5文字以下でした。言語は「短く」、log、ret、fnなどの演算子を使用していました。



たとえば、リリースされる最初のRustコードは次のようになりました。 



fn main() { log "hello, world"; } fn max(int x, int y) -> int { if (x > y) { ret x; } else { ret y; } }


Steve Klabnikが指摘しているように 、この制限は時間の経過とともに削除されました。たとえば、retがreturnに変わり、一部は完全に置き換えられたなど、一部のキーワードは「長く」なりました。 比較のために、現代の言語実装では、文字列「Hello world！」の出力は次のようになります。 



 fn main() { println!("Hello, world!"); }


また、進化の過程で、言語の概念とキーワードの一部が削除されました。 Graidonがこの言語で作業したとき、Rustはオブジェクト指向であり、オブジェクトを記述するためにobj演算子を使用しました。 現在、OOPはこの言語ではサポートされていませんが、Rustでは抽象化を使用してその概念の多くを実装できます。



Rustは、 パラメーターポリモーフィズム関数とも連携しました 。 一般的なプログラミングの概念は現在この言語で保持されています（このドキュメントでコードを見つけることができます）が、10年前、パラメーターの種類を示すために角かっこが使用されていました。 



 obj swap[T](tup(T,T) pair) -> tup(T,T) { ret tup(pair._1, pair._0); } fn main() { auto str_pair = tup("hi", "there"); auto int_pair = tup(10, 12); str_pair = swap[str](str_pair); int_pair = swap[int](int_pair); }


Gradonは、Rustで4年間独立して働きました。 この間、彼は計画された機能の約90％を実現することができました（その一部はかなり粗雑な実装でした）。 コード実行環境は70％完了しました。 この間、ChorusはOCamlコンパイラ用に合計38,000行のコードを作成しました。



Mozillaへの移行（2010–2012）



私は、言語に主要な機能があるべきだとは思いません。



明確で信頼性の高い一連のモジュールで構成され、相互に組み合わせて適切に機能する必要があります。



-Rust Developer 、Graydon Hoare
4年後、Graidon はプロトタイプをMozillaのマネージャーに見せることに決めました 。 同社は、C ++よりも単純なテクノロジーでブラウザースタックを再構築するツールを探していたため、プロジェクトに関心を示しました。 そのため、同社は（Graidonが率いる）チームを作成してRustで作業し、これがServoブラウザーエンジンの基礎になりました。



その後、Mozillaエンジンは 、シングルスレッドのコンテンツ処理スキームを備えていたため、マルチコアシステムで完全に動作できませんでした 。 たとえば、シングルスレッド関数は、ウィンドウのコンテンツの形成であり、JavaScriptを実行します。 Rustにより、レンダリングコードを並列に実行されるミニタスクに分割し、CPUリソースを節約することができました。



操作を並列化することで操作を高速化することに加えて、Rustはブラウザーのセキュリティを改善しました。 当時、FirefoxはC ++で実装されており、450万行のコードが含まれていました。 C ++は「正確な」プログラミング言語であり、細部にまで注意を払う必要があるため、プログラマのエラーが深刻な脆弱性につながる可能性があります。 Rustのタスクは、コンパイラを使用して人的要因の影響を減らすことでした。



2010年、言語開発者は以前使用していたOCamlコンパイラをRustで記述されたコンパイラに変更しました。 2011年にGraidonはコンパイラが正常に「アセンブル」できたことを示すメッセージを公開し、2012年にRustチームはコンパイラのアルファバージョンのリリースを発表しました。ドキュメントは完全ではなく、ビルド速度は理想からはほど遠いものの、ほとんどの言語機能とクロスコンパイルをサポートしました。



年型システム（2012–2014）



対象読者は「失望したC ++開発者」です。



-Rust Developer 、Graydon Hoare

この時点で、GraidonはRustでの作業から離れ、他のプロジェクトに切り替えました。 Steve Klabnikによると 、その後、管理システムはより「分散」されています。 連邦政府の構造が形成され、プロジェクトのさまざまな部分で導入された変更を別の開発者グループが担当しました。



チームは成長を続け、複雑なタイプのシステムに精通した人々が登場し始めました。 したがって、タイプシステムの積極的な開発が始まり、言語のより多くの側面がライブラリに提出されました。



たとえば、Rustの初期段階では、メモリセキュリティーを強化するためにGraidonが導入した「ガベージコレクター（GC）」が実装されました。 しかし、その後、開発者は、型システムを使用して同じレベルの信頼性を提供できるという結論に達し、GCは放棄されました。



この決定は、Rustで使用されるポインターシステムにも影響しました。 「ガベージコレクター」が削除される前、この言語には3つの主要なポインタがありました。



  • @（GCによって組み立てられたポインター）
  • 〜（一意のポインター）
  • ＆（ 借用ポインター ）


GCを削除した後、ポインターシステムは変更され、次の形式を取りました。



  • ボックス（一意のインデックス）
  • ＆（「リンク」と呼ばれる借用ポインター）


プログラミング言語の開発のこの段階では、型システムに関係のない多くの改善も行われました。 たとえば、CargoパッケージマネージャーとCrates.ioオンラインリポジトリで作業が進行中です。 開発者は、コンパイラ設定をカスタマイズする機能を追加し、他の言語からRustへの移行を簡素化しました。 新規ユーザーは、コンパイル中にこのフラグまたはそのフラグが必要な理由を理解する必要はありませんでした。 ところで、Steve Klabnikのプレゼンテーションで 、Cargoパッケージマネージャーのコマンドを見つけることもできます。



2012年から2014年までの期間は、Rustコミュニティが形になり始めた時期です。 C ++ユーザー、スクリプト言語のユーザー、機能プログラマーという3つの大きな「キャンプ」を形成しました。 彼らの専門知識は言語に影響を与えました-徐々に、関数型プログラミングと手続き型プログラミングのパラダイムを組み合わせ始めました。



2014年3月には、言語の重要な変更を表すために使用されるRFCプロセスも形成されました。 このソリューションはPython PEPのイメージと類似性に基づいて構築され、現在では3千件のコミットを行っています。 さらに、開発者によって行われた変更もRFCに反映されます。 Rustチームのルールによると、コミュニティとソリューションについて話し合わなければ、誰も大きな変更を加えることはできません。



リリース（2015）



何が起こるのかは確かではありません。



-Rust Developer 、Graydon Hoare

2015年の初めに、Rust 1.0 Alphaがリリースされました 。 言語のコアを安定化し、マクロシステムを開発し、最終的にisizeおよびusizeという名前を整数型intおよびuintに割り当てました。 同年の第2四半期の初めに、Rust 1.0 はベータ版になりました-この時点でcrates.ioリポジトリには1,700クレート（ コンパイル単位 ）があり、リポジトリからのダウンロード数は100万を超えました。



2015年5月、公式リリース-Rust 1.0 。 これが安定の始まりを示しました。 その時点から、行われたすべての変更には後方互換性が必要であり、これによりRustを実際のプロジェクトで使用できるようになりました。 彼は、ゲーム開発、Web開発、オペレーティングシステム開発などの分野でアプリケーションを見つけ始めました。



生産への移行（2016年5月）



言語がたった1つの点で優れている場合、これは失敗です



-Rust Developer 、Graydon Hoare

2015年、StackOverflowプラットフォームは開発者を対象に調査を実施しました。調査では、どのプログラミング言語を使用したか、どのプログラミング言語を知りたいかを尋ねられました。 その後、Rust は評価の3行目を取りました 。 しかし、1年後、彼は 1位になりました-ユーザーの79％が彼との仕事を続けたいという意向を表明しました。



Hacker Newsの住民の1人は、ドキュメントの透明性とシンプルさをこの言語の主な利点と呼びました 。 他のユーザーも、PLの機能の研究を支援する準備ができているRustコミュニティの開放性と友好性に注目しました。



ただし、多くの開発者は、そのセキュリティメカニズムのために、この言語での作業を継続することを決定します。 Redditユーザーの一人が言ったように、「Rustでのプログラミングは、保険と保護を備えたパルクールのようなものです。 奇妙に見えることもありますが、自分で何かを壊すことを恐れることなく、多くのトリックを行うことができます。



Rustの安定バージョンのリリース以来、本番環境でのフル使用期間が始まっています。 プロジェクトでRustを使用した最初の企業の1つは Mozillaです。 Linux版Firefox 45の「内部」の一部はRustで書き直され、Firefox 47以降、RustコードはWindowsバージョンにも存在します。 2016年10月に発表されたProject Quantumには、Servoコンポーネントも組み込まれています。



Rust は Dropboxでも使用されています-製品のコアはこの言語で書かれています。 Dropbox は 、新しいMagic Pocketクラウドストレージを作成し、Amazon S3から情報を転送しました 。 当初はGoで実装されていましたが、負荷が高い場合、メモリ消費量が多く、Goコードの動作の予測性が低いという問題がありました。 これらの問題を解決するために、Rustは部分的に関与しました。



昨年、Rustを使用してnpmでJavaScriptパッケージを処理し始めました。 Rustは、1日あたり3億5,000万パケットを実行するシステムの遅延を解消するのに役立ちました。 npmサポートスペシャリストのアシュリーウィリアムズは、ウクライナのRustFestでのRustの経験について話しました。 ここでビデオを見つけることができます。



また、BitfuryでRustを使用しています。 プログラミング言語は、 Exonumブロックチェーンフレームワークを実装しています。 RustFest 2017カンファレンスで初めてプラットフォームを紹介し、その機能を示し、暗号通貨作成サービスの作業を実演するワークショップを開催しました（Exonumで暗号通貨を作成する方法の簡単なガイドはこちらにあります ）。



Rustの実装は、スマートコントラクトを使用する際に役立ちました。 これにより、ExonumスマートコントラクトはEthereum（またはFabric）コントラクトよりも生産的になります。 プラットフォームコードは完全にオープンで、 GitHubのプロジェクトリポジトリにあります 。



Rustは、情報セキュリティの分野でもアプリケーションを見つけています。 Torのようなプロジェクトはすでにコードの一部をRustに移植しています。



一般に、105社が製品にRustを使用しています。 それらの完全なリスト（Bitfury Groupもマークされています）は、公式WebサイトのFriends of Rustページにあります。 また、Rustでソフトウェア製品を作成する企業の数は絶えず増え続けており、言語開発者は非常に満足しています。
