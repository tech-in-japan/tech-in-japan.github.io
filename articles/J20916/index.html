ブラウザーごとのwindow.openの動作の違い

1つのサイトを開発するときに、さまざまなブラウザーでwindow.open関数の動作を確認する必要がありました。 結果を作成してレイアウトすることにしました。誰がそれを必要とし、誰が追加するかがわかります。



テストには、FireFox 3ベータ3、Netscape Navigator 9、Internet Explorer 6、Opera 9、Konqueror 3.5を使用しました（トピックの最後に、Safariを個別にテストします）。 同時に、FireFox 3は2つの構成でテストされました：デフォルトとTabMixPlus（新しいウィンドウでポップアップを開く許可を持つ）。 ワークスペース：Debian上のKDE 3.5。

2.ウィンドウの寸法



新しいウィンドウの寸法を設定しました。

window.open（ 'http://www.w3.org/'、 'popup'、 'width = 640、height = 480'）;


FireFox 3 （TMPの有無にかかわらず）

640x480ピクセルの表示領域を持つウィンドウを開きました。 ウィンドウには、ステータスバーとアドレスバー（非アクティブ）が表示されます。

ステータスバー、アドレスバー、タイトルバー、ウィンドウの境界線により、ウィンドウ自体が指定されたサイズを超えています。 ウィンドウサイズは可変です。



Navigaror 9とKonqueror

640x480ピクセルの表示領域を持つウィンドウを開きました。 ウィンドウのタイトルと境界線により、ウィンドウ自体が指定されたサイズを超えています。

ウィンドウサイズは可変です。



Opera 9

640x480ピクセルの表示領域を持つウィンドウを開きました。 ウィンドウのタイトルと境界線により、ウィンドウ自体が指定されたサイズを超えています。

また、馬には小さなパネルがあり、押すとナビゲーションパネルに展開します。 これにより 、表示領域が縮小され 、ウィンドウサイズは変更されません。 ウィンドウのサイズは可変です（マウスを使用するユーザーを意味します）。



IE 6

640x480ピクセルの表示領域を持つウィンドウを開きました。 ウィンドウのタイトルと境界線により、ウィンドウ自体が指定されたサイズを超えています。 ウィンドウサイズはサイズ変更できません 。



* * *



今、私は明らかにモニターの解像度を超えるサイズにウィンドウを設定します。

window.open（ 'http://www.w3.org/'、 'popup'、 'width = 2000、height = 1200'）;


FireFox 3、Navigator 9、Konqueror

ウィンドウを開き、デスクトップ全体に拡大しました。 FF3では、ウィンドウにステータスバーとアドレスバー（非アクティブ）が表示されます。



Opera 9

新しいタブを開きました。



IE 6

彼は、すべてのパネル（タスク、メニューなど）を含む画面全体をカバーするウィンドウを開きました。 ウィンドウのタイトルは画面外です。



結論
  1. すべてのブラウザは、指定されたサイズをビューポートのサイズと見なします。
  2. 実際のウィンドウサイズは常に指定されたものよりも大きく、追加のパネルが存在するため、ブラウザごとに違いが異なります。
  3. Internet Explorer 6では、ウィンドウのサイズを手動で変更することはできません。 ところで、これはwindow.openの正しい動作です。 この関数の呼び出しで指定されていないすべてのパラメーターは、 "no"（サイズ変更可能を含む）と等しいと見なされる必要があります。 ユーザーの観点からは必ずしも便利ではありませんが。
  4. 開かれたウィンドウのサイズを監視する必要があります。 IE6の大きなサイズでは、ユーザーのデスクトップ全体をカバーできます。


3.ポジショニング



次の行は、ウィンドウの座標です。 これには2組のプロパティがあります-IEの 左と上 、NNのscreenXとscreenYです。

あるいは、すべてのブラウザーですべてのペアを試しました。



window.open（ 'http://www.w3.org/'、 'popup'、 'width = 320、height = 240、left = 50、top = 50'）;


FireFox 3 、Navigator 9、IE6、Konqueror

4つのブラウザはすべて、 左と上のプロパティを理解し、画面の左上隅を基準にしてウィンドウを配置しました。

この開始点にもかかわらず、top = 0のブラウザはいずれも、ウィンドウでメニューバーをブロックできませんでした。



Opera 9

leftとtopのプロパティも理解しましたが、タブバーに対してウィンドウを配置しました（カウントは下端から始まります）。



window.open（ 'http://www.w3.org/'、 'popup'、 'width = 320、height = 240、screenX = 50、screenY = 50'）;


Geckoブラウザー（FFおよびNN）のみがこれらのプロパティを理解し、OperaおよびIEはそれらを無視しました。



画面のサイズを明らかに超える位置を設定しようとした場合の興味深い動作。



window.open（ 'http://www.w3.org/'、 'popup'、 'width = 320、height = 240、left = 2000、top = 1000'）;


FireFox 3、Navigator 9、Opera

デスクトップの右下隅にあるウィンドウを開きました（Opera-そのワークスペース）。 ウィンドウ全体が表示されます。 パネルはブロックされません。



コンクレール

彼は、デスクトップの左上隅のウィンドウを覆いました。 パネルはブロックされません。



IE6

チェックに失敗しました。 IEは、プロセスを強制的に削除する前にウィンドウを開くのを停止しました。



結論

  1. ウィンドウを配置するには、 左と上のプロパティを使用します。 それらはすべてのbarausersによって理解されます。
  2. Operaは、画面に対してウィンドウを配置するのではなく、タブバーに対してウィンドウを配置します。
  3. ウィンドウがデスクトップに収まらない状況を回避することを試みることはよりよいです、なぜなら ブラウザの動作は異なります。
