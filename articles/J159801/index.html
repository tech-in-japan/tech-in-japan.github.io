Infoboxのクラウドサーバー

クラウドを実行したいホスティング事業者には2つの方法があります。 1つ目は、デザイン、ソフトウェアの作成、コーンの詰め込みのすべての結果をもたらす魅力的な世界に真っ向から飛び込むことです。 すでにそのようなケースを見てきました。 2つ目は、インフラストラクチャ上で有名なソフトウェアベンダーから既製のソリューションを起動することです。



2番目の方法を使用しました。 新しいParallels製品であるParallels Automation for Cloud Infrastructure（PACI）クラウドプラットフォームを選択しました。 いくつかの理由があります。 Parallelsは、ホスティングソフトウェア開発者のリーダーです。 もちろん、彼らは仮想化について多くのことを知っています。 さらに、サービスおよびインフラストラクチャコントロールパネルのシンプルさと利便性も気に入っています。







その結果、実装とテストに2か月しかかかりませんでした（自作のシステムでは不可能です）。 10月に、 非商用テスト用の新しいクラウドサービスを開始しました。これは2013年2月1日まで続きます。 今日では、ロシアで唯一のプラットフォームの実装です。



クラウドでは次のことができます。

  • コンテナーと分離された仮想マシンを作成する
  • WindowsとLinuxを使用する
  • ロードバランサーを接続する
  • APIを介してサーバーを管理する
  • サーバーのクローンを作成してそのイメージを使用する


一般的に、このサービスには、あらゆるタスクのためにクラウドを整理するためのすべてが備わっています。



仮想化 ちょうだい！



選択したプラットフォームの主な利点の1つは、クラウドで仮想マシンとコンテナー（Virtuozzo）の両方を使用できることです。



Linux（CentOS、Debian、Fedora）、Windowsなどのコンテナーを作成できます。 コンテナ仮想マシンは迅速にインストールされ（1〜5分）、ダウンタイム中のリソース消費が少なくなります。 同時に、コンテナにより配置密度を高めることができます。つまり、サービスのコストを削減できます。 コンテナ管理タスクの一部は、たとえばシステムカーネルの更新など、クライアントの責任の範囲外です。



クラウドの中心は、Parallels Server Bare Metalサーバーソフトウェアです。 完全に分離された仮想環境とコンテナーの両方を作成できるのは、このハイパーバイザーです。 たとえば、Microsoft Hyper-V Serverで実行されている仮想マシンに比べて、技術的な特性とパフォーマンスに劣りません。



その場でクラウドサーバーの設定を変更する



API呼び出しまたはコントロールパネルでサーバーを再起動せずに、仮想サーバーの基本パラメーターを変更できます。



  • コンテナ-メモリの量、コアの数、プロセッサのクロック速度、ディスク容量、ネットワークインターフェイス設定
  • 仮想マシン-メモリとディスク容量を増やし、ネットワークインターフェイス設定を変更します




また、既存のマシンを複製するか、このためにサーバーイメージを使用することにより、いつでも新しいマシンを作成できます。



自動サーバースケーリングは、API呼び出しを使用して今日実装できます。 次の投稿で、クラウドのAPIの機能について詳しく説明します。



サービス開発計画
