さて、ついに！ Windows 8.1へようこそ！

親愛なる同僚と友人の皆さん、こんにちは。

先週、10月18日金曜日に待望のイベントが開催されました-Windows 8.1 RTMが正式に導入されました（そして、 Windows 8.1 RTMとWindows Server 2012 R2およびSystem Center 2012 R2を組み合わせて-すべてを既にダウンロードできますので、興味のある製品をクリックする必要があります）。

これは、アップグレードの時間が間近に迫っていることを意味します-しかし、そのような深刻な一歩を踏み出す前に、ゲームがろうそくに値するかどうか、いわば8.0と比較した8.1の新機能を、公平なITプロの観点から見てみましょう。











Windows 8.1に影響を与えた変更は、主に、 BYOD（Bring Your Own Device） 、セキュリティの改善、モビリティの改善、インターフェースの分野の4つの主要な領域に関連しています。 Windows 8.1で指定されたトピックのコンテキストでどのような革新が登場したかを詳しく見てみましょう。



インターフェース





インターフェースに関しては、多くの変更があります。 注目に値する最初のことは、[スタート]ボタンの戻り値です。 ただし、ボタンの戻り値がイデオロギー自体の戻り値とおなじみのメニューの戻り値に等しいと想定しないでください-これは起こりませんでした。 [スタート]ボタンは、キーボードのWindowsボタンを複製します。つまり、スタート画面が表示されます。 気に入らなかった人や、少しスピードアップしたいという人には、スタート画面をバイパスして、デスクトップに直接ダウンロードする機会があります。



個人的に8で動揺したのは8.1ではなく、新しいWindows 8アプリケーションは最初のモニターでしか実行できなかったということです。つまり、複数のWindows 8アプリケーションを異なるモニターで実行することはできませんでした。 現在、この欠点はなくなり、複数のディスプレイでアプリケーションを実行できます。 さらに重要なのは、以前に第1モニターで最大2つのWindows 8アプリケーションを実行でき、実行中のアプリケーションのサイズ比であるウィンドウ自体が固定されていて、柔軟性がなかったことです。 1つの画面で最大4つのアプリケーションを実行できるようになりました。量は後者の解像度に依存します。





1番目の画面には多くのアプリケーションがあり、いくつかの画面が存在する場合があります-すべてがタッチセンシティブです！



機動性





モビリティに関しては、十分な改善もあります。 注目に値する最初のことは、Windows 8.1 RTとWindows 8.1 Pro / Entの両方のVPNアクセスでサポートされているプロトコルのリストの拡張です。 また、関心のあるアプリケーションを起動すると、VPN接続を自動的に呼び出して確立できます。



もう1つの興味深い点は、8.1デバイスにある場合、3Gチャネルへの一般的なアクセスを提供する可能性です。必要に応じて、友人や同僚にインターネットを配布できるようになりました。



また、OSレベルでの組み込み無線コンポーネントのサポートを追加しました-これにより、不要なデバイスを一時的にオフにできるようになりました-これにより、デバイスのバッテリー電力を節約できます-一方、より薄くて軽い外観を期待できますおよびスタンドアロンデバイス。



非常に興味深いイノベーションがワイヤレスの分野に登場しましたが、少し異なります：今では、ドライバーとアプリケーションをインストールせずに、Wi-Fiプリンターに接続して必要なドキュメントを印刷できます-すべてが非常にシンプルで便利です-有線でもドライバーでもない-要するに、おとぎ話！



また、Wi-Fiに加えて、 Miracast標準のサポートが登場しました。 しかし、これはすでに非常にクールですが、それほど単純ではありません。 Miracast-要するに-Wi-Fiを介した外部モニターまたはプロジェクターへの直接画像転送の標準-主なことは、ターゲットデバイスもMiracastをサポートしていることです。 私は個人的にそのようなデバイスを見たことはありません-これは私の個人的な観点からの最大の問題です-しかし、それは時間の問題です。特に、特にワイヤを取り除く傾向に照らして、アイデアは非常に成功しているので、それはそれです。



まず、コードレスのささいなことに-ネットワークプリンターをNFCタグでマークできるようになりました-プリンター設定に関する情報をタグ自体に含めることができます。 その結果、目的のプリンターに接続するには、NFCデバイスm 8.1を持ち込むだけで十分で、設定がすぐにデバイスに適用されます-前方に曲を（意味で、印刷付きで！）



安全性





ここで、セキュリティの分野の機能について少しお話しましょう。



セキュリティの分野で最初に注目する価値があるのは、デバイスの普及した自動暗号化の可能性です。 以前はWindows 8.1 RTとWindows Phone 8でのみ可能でしたが、現在ではすべてのバージョンのWindowsで可能になっています。 Windows 8.1 Pro / Enterpriseの場合、付録でBitLocker暗号化をアクティブ化できます。 通常の場合、Microsoftアカウント（Live ID）が必要です。このアカウントのデータを介して、バックグラウンドで自動暗号化が行われます。



指紋デバイスまたは指紋スキャナーとの相互作用の分野の改善にも注意を払う価値があります。

これで、Windowsのどこでもこの認証メカニズムを使用できるようになりました。 また、ドライバーモデルはより統一され、圧電接点とセンサーの両方のスキャナーをサポートしています-前者はセンサーに沿って指を配線する必要があり、後者はカメラの原理で動作し、最新のセンサー（ラップトップに組み込まれていない）は光学式であり、印刷物の「活気」を制御できるようにします-つまり それは生きている人のものですか、それはシリコンダミーではありませんか？ 現在、彼はこの奇跡のITパラノイアすべてを組み込みで使用できます。





指紋スキャナーを使用してVPNネットワークにアクセスします。



アプリケーションの狭い領域に対して、別の興味深い機能が登場しました-デバイスロック（デバイスロックダウン）。 デバイスで実行されているWindowsストアから1つのアプリケーションだけでデバイスを起動できます。 これは、さまざまなキオスク、お金を受け取り、サービスの支払いをするためのデバイス、および同様の性質の他のアプリケーションを実装するために必要になる場合があります。



ビルトインのWindows Defenderツールは、不審なウイルスに対処するように設計されていますが、ネットワーク監視機能を取得するか、またはこれに基づいてネットワーク動作を監視するようになりました。これに基づいて、アプリケーションの非特徴的な動作を判断できます。 IEはActiveXメカニズムを使用してバイナリスキャナーを起動し、実行される前にファイルの内容を分析します。 そしてそれは本当に便利です！



また、セキュリティの最大の利点は、ユーザーのデバイス上のデータをリモートで制御できることです。 多くの企業の従業員が個人データ、秘密データ、一般にさまざまなクラスの情報を扱うという事実を考慮すると、情報とデバイスをリモートで制御する能力は不要ではありません。 デバイスはリモートで暗号化でき、EASメカニズムまたはEAS + OMA-DMの組み合わせを使用してクリアできます。 このメカニズムには特別なインフラストラクチャが必要です。サーバーとアプリケーション、特にメールクライアントと組み合わせたExchange Serverはこのメカニズムを実装します。 アプリケーションはデータの正当性をチェックします-必要であればm-データを削除します。



BYOD-ロマンスがデバイスを歌う





Microsoftの戦略のトレンドと要素の1つであるBYODクラウドOSに関しては、無視できないイノベーションがたくさんあります。



私にとって最も注目すべきことは、Surface RTを自由に使える人にとって、デバイスを企業ネットワークに入力し、ドメインの設定とポリシーを適用し、データにアクセスできることです！ そして、これらすべて-ドメインに参加することなく。 8.0での待ち伏せは、デバイスが私たちに与えられたという事実に正確にありました-しかし、それらからの感覚は角のある男性のミルクのようなものです。 現在、Workplace Joinメカニズムのおかげで、ドメインに参加せずに企業ネットワークとデータにアクセスできますが、IT部門はこのデバイスのアクションと機能を非常に明確かつ明確に制御できます。



別の興味深い機能はワークフォルダーです。これは、どこからでもドキュメントを操作し、データセンターのサーバー上の特別なフォルダー（作業）と変更を同期できるツールです。つまり、さまざまなドキュメントの最新バージョンへの集中アクセスを取得する機会です。ユーザーの承認されたモバイルデバイス。 すべての変更は、データセンター内のサーバー上のフォルダーと自動的に同期されます-これは便利です。 これはSkyDriveであり、コイシポビースケールのみであると言えます-類推に頼る場合。 （気配りのある読者とユーモアのセンスに感謝-著者のコメント）





デバイス間で作業文書を同期するための作業フォルダーの設定。



MDM標準、モバイルデバイス管理、特にOMA-DM APIのサポートにより、BYODシナリオで使用するさまざまなプラットフォームでのモバイルデバイスのサポートだけでなく、この分野の他のソリューションとの統合についても話し合うことができます（たとえば、モバイルアイアンまたはエアウォッチ）。
