コールバックAPI上のPythonでのVK用Chatbot

チャットボットはすでに非常に一般的になっており、毎日すべてのインスタントメッセンジャーに表示されます。



この記事では、一連の単純なコマンドを使用してボットを作成する方法を段階的に見て、将来的にその機能を拡張する方法を見つけます。 この記事は、チャットボットを作成したことのないほとんどの初心者に役立ちます。



ボットを作成したいときは、VKontakteで使用可能なボットの例を検討し、その構造を最大限に単純化することを試みました。



ボットを作成するために、Python 3.5（3番目のpythonの他のバージョンがおそらく適切です）と追加のFlaskおよびVKライブラリを使用しました。 それらをインストールする必要があります。 Flaskのインストールに関するロシア語の記事が多数あります。 Pycharmをお持ちの場合は、おそらくPycharmと共にインストールされています。



API自体から始めましょう。 ボットには、グループメッセージに使用できるコールバックAPIを使用します。 まず、接続されたメッセージを含むVKontakteグループを作成するか、すでに持っている必要があります。



[ コミュニティ管理] →[ APIの操作]セクションで、コミュニティメッセージにアクセスできるキーを作成する必要があります。









コールバックを使用するには、APIからイベントのリクエストを受信し、それらを処理し、レスポンスリクエストを送信するWebサーバーが必要です。 つまり、送信されたリクエストにのみ応答し、送信する「ウェブサイト」を作成します。



pythonで記述しているため、使用できる最も簡単なものはpythonのホスティングです。 Pythonの無料ホスティングを使用しました。 そこで登録し、FlaskでPython 3.5用のアプリケーションを作成する必要があります（Webセクションで作成できます）。 初期ファイルが作成されます： 



# A very simple Flask Hello World app for you to get started with... from flask import Flask app = Flask(__name__) @app.route('/') def hello_world(): return 'Hello from Flask!'


現在ファイルにある唯一の機能は、登録時に指定されたアドレスでページを埋めることに責任があります。 username.pythonanywhere.comのブラウザ（ニックネームを使用）にアクセスすると、「Hello from Flask！」というテキストのみが表示されます。



将来的には、コードはブロックで表示され、ブロック全体の完了後にチェックすることが可能になり、そのプロセスでコードが環境によってエラーとしてマークされる可能性があります。 怖がらないで、ブロックを最後まで終わらせたほうがいいです。

サイトに送信されたリクエストを処理するには、ドキュメントの最後に次のコードを追加します。 



 @app.route('/', methods=['POST']) def processing(): return 'xxxxxxxx'


Xの代わりに、「サーバーが返す必要のある文字列」を代入します。 これは、コールバックAPIセクションのグループ管理にリストされています。



この機能により、グループに通知するためにサイトを接続できます。



これで作業を確認できます。 アプリケーションを再起動するだけです。 サイトが新しいデータで作業を開始できるようにファイルを変更して保存した後のホスティングでは、[Web]タブでそれをリロードする必要があります。 このコードを追加した後、VKontakteグループのサーバーアドレスバーに対応するアドレスusername.pythonanywhere.comを入力し、[確認]をクリックします。



サーバーアドレスが正常に接続されたことを示す緑色の通知が表示されます。



[確認]をクリックすると、VKontakteはサーバーに接続し、それが実際にグループ所有者に属していることを確認し、サーバーが要求に応じて確認コードを返すのを「待機」します。

ブロック5

記事の残りの部分では、必要だと思う1つの改善について説明します。 ただし、ボットはそれなしで動作します。



おおよそのコマンド認識



ユーザーが1つの文字を間違えた場合、最も類似したコマンドを念頭に置いている可能性が高いです。 したがって、ボットがまだ答えを出し、「私はあなたを理解していません」と言わなかったらいいでしょう。



おおよその認識のために、Damerau-Levenshtein距離を使用します。 文字を削除、挿入、置換、移動する操作が、ある行から別の行に移動できる回数を示しています。



この距離を見つけるためのアルゴリズムは、例えばウィキペディアで説明されています。



関数をmessageHandler.pyファイルに追加します。 



 def damerau_levenshtein_distance(s1, s2): d = {} lenstr1 = len(s1) lenstr2 = len(s2) for i in range(-1, lenstr1 + 1): d[(i, -1)] = i + 1 for j in range(-1, lenstr2 + 1): d[(-1, j)] = j + 1 for i in range(lenstr1): for j in range(lenstr2): if s1[i] == s2[j]: cost = 0 else: cost = 1 d[(i, j)] = min( d[(i - 1, j)] + 1, # deletion d[(i, j - 1)] + 1, # insertion d[(i - 1, j - 1)] + cost, # substitution ) if i and j and s1[i] == s2[j - 1] and s1[i - 1] == s2[j]: d[(i, j)] = min(d[(i, j)], d[i - 2, j - 2] + cost) # transposition return d[lenstr1 - 1, lenstr2 - 1]


この距離を見つけるためのアルゴリズムが実装されており、必要に応じて変更または改善できます。



これらの行によると、1つを別の値に変換するための操作の数が与えられます。 get_answerメソッドを変更します。 



 def get_answer(body): message = ",   .  '',    " attachment = '' distance = len(body) command = None key = '' for c in command_list: for k in c.keys: d = damerau_levenshtein_distance(body, k) if d < distance: distance = d command = c key = k if distance == 0: message, attachment = c.process() return message, attachment if distance < len(body)*0.4: message, attachment = command.process() message = '     "%s"\n\n' % key + message return message, attachment
