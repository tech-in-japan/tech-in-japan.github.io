Windows 10〜10。リリース＃1。 インストールの可視性と頻度を増やす方法

10のWindows 10シリーズの最初の記事へようこそ。 ユーザーインタラクションが始まる場所から、Windowsストアからシリーズを開始します。







Windows 10ユーザーがアプリケーションを見つけて起動できるように、今すぐ3つの演習を行うことをお勧めします。

  1. ストアの説明を更新して、アプリケーションがインストールされる可能性を高めます。
  2. Visual Studio Application Insightsと新しい使用状況レポートを使用して、アプリケーションの使用状況の追跡を開始します。
  3. 新しいストア機能を使用してキャンペーンの成功を追跡する方法を学びます。


今週1時間だけでも、Windows 10での表現を改善するだけでなく、アプリケーションの開発と改善に基づいて構築できる基本的な指標を自分で決定することもできます。





アプリケーションの説明を更新する



Windowsストアでアプリケーションを提示することから始めましょう。ユーザーは、あなたの提案を見て、ボタンをクリックしてWindowsアプリケーションをインストールし、チャンスを与えるかどうかを決定する時間を費やします。 これは、ユーザーと対話する最初の場所です。



潜在的なユーザーがアプリを見つけやすくする



ストアの検索結果に適切に表示されるためには、 キーワードガイドラインに従う必要があります。アプリケーションを正確に説明する最大8つのキーワードを使用できます。 FacebookやTwitterに関係ない場合は、それを持っているふりをしてあなたの言葉を無駄にしないでください。







また、ストアの説明の表示をWindowsのさまざまな要素に統合し、ユーザーが現在のタスクに関連するアプリケーションやゲームを見つけられるようにしました。 たとえば、Cortanaで検索すると、インストールされたアプリケーションと利用可能なドキュメントだけでなく、Windowsストアとインターネットからの関連する結果も表示されます。これにより、Windows 10のユーザー数を増やすことができます。アプリケーションの説明に使用するキーワード。







また、店舗の説明の一部としてプロモーション画像を提供することを強くお勧めします。 これらのイメージは、アプリケーションの選択（フィッシング）に選択されている場合、さまざまな場所でアプリケーションを宣伝するために使用されます。 そのような画像を提供しなかったとしても、アプリケーションがどんなに良いものであっても、プロモーションには入れられません。 Windows 10では、ストアチームがプロモーションキャンペーンで、特に「Spotlight」コレクションのメインイメージとして使用する新しい2400x1200pxイメージ形式を追加しました。 アプリケーションをプロモーションの対象とする場合は、今週に少し時間を割いて、特に2400x1200pxのサイズの広告画像を説明に追加します。



このトピックからそれほど離れていませんが、Windowsストアでのプロモーションの検討に含めるための残りの要件を確認してください。 ただし、ここで重要なのは、必要なすべての画像を追加し、プロモーションのすべての要件を満たすことは、プロモーションに参加することを保証するものではないことです。 ただし、コンプライアンス違反は、考慮されないことを意味します。



アプリケーションがインストールすることを確認してください



いいね！ ユーザーの注目を集めて、ストアでアプリケーションについて説明したとします。 彼らはそこで何を見ますか？



アプリケーションの説明ページは、潜在的なユーザーがアプリケーションについて最初に意見を述べ、インストールするかどうかを決定する最初の場所です。 ここでは、ソリューションを全世界に「販売」します。 簡単な例えは、本の表紙です。 あなたが提供しているものをほんの数語で視覚的に正確に伝えることができない場合、ユーザーが他の誰かにさらに行く可能性が高くなります。







驚くべきことに、ほとんどの開発者はストアでアプリケーションを記述することにほとんど注意を払っていません。 アプリケーションとの最初のミーティングに新しいユーザーを準備するには、いくつかの新しいヒントを適用して説明を修正します。

  • アプリケーションの説明では、最初の文でアプリケーションの利点を説明する必要があります。 ストアは通常、説明を切り取り、最初の数行のみを表示します（上記のリモートデスクトップアプリケーションのスクリーンショットを参照）。 ユーザーが詳細を読みたいと思うように、すべての重要な考えが「カットライン」に到達することを確認してください。
  • スクリーンショットは興味深く理解できるものでなければなりません-特に最初のスクリーンショット（タイトルのあるタイトル画面を使用しないでください）。 画像がアプリケーションまたはゲームを適切に説明していることを確認してください。 また、画像に説明テキストを追加して、ユーザーが見ているものをよりよく説明することもできます。 優れた画像を作成するためのツールが必要になる場合があります。無料のPaint.netやGIMPからAdobe Photoshop / Illustratorなどの有料製品まで、適切な製品を使用できます。
  • アプリケーションが多くの市場でローカライズされている場合、スクリーンショットと説明も翻訳されていることを確認してください。 （追加のヒントとして、既にWindowsアプリケーションがあり、新しい市場でローカライズしたい場合、 Dev Center Benefitsプログラムの一部として、Lionbridgeからローカリゼーションサービスの大幅な割引を受けることができます。）


広告画像がない場合と同様に、ストア内の膨大な数のアプリケーションは、十分な説明がないか、スクリーンショットが見出しフレームであるという事実により、プロモーションの機会を通り過ぎました。 これをアプリケーションで起こさないでください！



成功の測定を開始



ストアの説明ページを理解し、ユーザーにアプリケーションまたはゲームを試すよう説得したので、「販売」が成功したことを理解する方法を見てみましょう。 Windows Developer Centerで、最近いくつかの重要な更新を行い、レポート機能を改善しました。 この記事では、そのうちの2つについて説明します。 アプリケーション使用状況レポートと、近日中に利用可能になるキャンペーンを追跡する機能です。



使用状況レポートは、アプリケーションの使用方法と使用者の詳細を提供します。 今日ダッシュボードに移動してレポートを見ると、レポートが空であることがわかります。また、アプリケーションにVisual Studio Application Insights SDKをインストールするように求められます。 Application Insightsに慣れていない人のために、これは、Azureを使用して（無料で）情報を保存および提示するために、アプリケーションからテレメトリーを収集するためのサービスです。 Windows Developer Centerはこの情報にアクセスできるため、ユーザーがアプリケーションをどのように使用しているかを表示できます。



Visual Studio 2015または2013では、わずか10分でApplication Insightsをアプリケーションに追加できます。 開始するには、Windowsアプリケーション（Windows 8または10-デスクトップまたはスマートフォン用）を開き、以下を実行します。







  • Visual Studioでプロジェクトを右クリックし、「Application Insights Telemetry ...を追加」を選択します
  • Application Insightsは、Windowsにサービスを追加するのに役立つアシスタントを起動します。 このプロセスの一環として、データの保存先を尋ねられます（無料のAzureアカウントをまだお持ちでない場合は、作成する必要があります）。
  • アシスタントですべての手順を完了するとすぐに、アシスタントはテレメトリ収集のためにプロジェクトをセットアップします。








その後、Azureでアプリケーションを使用している人の数をほぼリアルタイムで確認できます。 収集されたテレメトリは、特定のアプリケーション機能がどのように使用されているか、何人、どのくらいの期間で測定できるので、アプリケーション、特に開発作業を指示する場所について多くを知ることができます。



アプリケーションをストアに公開し、Windows開発者の中心にある情報パネル（アカウント設定）でアプリケーションのテレメトリ収集設定を有効にするとすぐに、データが使用レポートにも表示されます 。



また、 TelemetryClientクラスを使用して、追加のデータ、たとえばページビュー（またはピボットコントロールとハブコントロールのタブを使用）、追跡するイベント、例外などを収集することもできます。 Application Insightsを使用して受け取る基本的なページテレメトリに加えて、例外追跡を使用して、ほぼリアルタイムでユーザーの問題について知ることができます（特に、次の更新プログラムを公開するときに役立ちます）。 以下は、例外情報を収集するためのサンプルコードです。 



try { // ... } catch (Exception ex) { var TelemetryClient telemetry = new TelemetryClient(); telemetry.TrackException(ex); }




分析の接続に慣れたら、関心のあるパラメータの測定を開始できます。 たとえば、次のコードは、タスクを完了するのにかかる時間を測定します。 



 var stopwatch = System.Diagnostics.Stopwatch.StartNew(); // ... perform the timed action ... stopwatch.Stop(); var metrics = new Dictionary <string, double> {{"processingTime", stopwatch.Elapsed.TotalMilliseconds}}; // Set up some properties: var properties = new Dictionary <string, string> {{"signalSource", currentSignalSource.Name}}; // Send the event: var TelemetryClient telemetry = new TelemetryClient(); telemetry.TrackEvent("SignalProcessed", properties, metrics);




個人用アプリケーション（Webサービスに大きく依存している）の場合、Webサービスへの呼び出しが低い評価につながる場合と、アプリケーションの問題が最も多い国を追跡することが有用であることがわかりました。 時間の経過とともに、サービス側でさまざまなアプローチ（サーバーエラーの収集、キャッシュなど）を試みましたが、ユーザーエンゲージメントへの影響（たとえば、アプリケーションに費やした時間による）とアプリケーションに対する全体的な満足度（改善点）を追跡するためにテレメトリも使用しました特定の地域でのランキング）。 これにより、アプリケーションの成功をより適切に測定できます。



昨年のCraig Brokschmidtによって作成されたエンジニアリングチームの記事で、テレメトリーの操作について詳しく読むことができます。 （執筆時点では、まだApplication Insightsを開発していたことを覚えておいてください。今日、サービスが広く利用可能になったため、サービスの操作がはるかに簡単になりました！）



自己宣伝を始めましょう



ユーザーの基本を理解したら、タップをオフにしてアプリケーションにトラフィックを誘導します。 これを計画するときは、アプリストアが開く次の機能に注意してください。

  • プロモーションキャンペーンを開始し、キャンペーントラッキングメカニズム（今月、開発者向けに展開）を使用して、どのユーザーが最も多くのユーザーを引き付けるかを理解します。
  • Microsoft Affiliateプログラムに参加すると、紹介リンクを通じてWindowsストアでの支払いから7％の手数料を受け取ることができます。
  • Windows Developer Centerを使用すると、アプリケーションを宣伝する広告キャンペーンを簡単に実行できます。




プロモーション成功分析



キャンペーントラッキング機能を利用することは、おそらく新しいユーザーがどこから来ているのかを理解する最も簡単な方法です。 この機能は、ユーザートラフィックの分析によく使用されるURLリダイレクトサービス（bit.lyなど）とほぼ同じように機能します。



  1. ストアのアプリケーションのアドレスの末尾に「キャンペーン」を追跡するためのIDを追加します（開発者センターのアプリケーション識別情報ページのアプリケーション制御セクションにあります）。 トラッキングIDは単純な文字列であり、ユーザーにとって意味のあるものであれば何でもかまいません。
  2. そのようなリンクの使用を開始し、IDの最後に、対応するパラメーターを次の形式で追加します。

    www.microsoft.com/store/apps/yourAppName/yourAppProductID?cid=yourCampaignID


    たとえば、Skype（製品ID 9wzdncrfj364）を宣伝し、異なるチャネルを使用する予定があるとします。 たとえば、次の3つのキャンペーンを実行できます。

    • ブログ： www.microsoft.com/store/apps/skype/9wzdncrfj364?cid=app_blog
    • Twitter： www.microsoft.com/store/apps/skype/9wzdncrfj364?cid=app_twitter
    • いくつかのレビューサイトのアプリケーションの概要www.microsoft.com/store/apps/skype/9wzdncrfj364?cid=rev_HotApps
  3. Windows Developer Centerの情報パネルのチャネルとコンバージョンに関するレポートで、結果を調べることができます（以下はサンプルレポートです）。





注 ：チャネルとコンバージョンに関するレポートは、今後数週間ですべてのデベロッパーが利用できるようになります。



別の興味深い調査オプションは、キャンペーンがユーザーへの良好なコンバージョンにつながったかどうかを理解することです。 このレポートにより、ユーザーがリンクをクリックしたときに使用したMicrosoftアカウントに基づいて、ユーザーがリンクをクリックしてから24時間以内にアプリケーションを購入（または組み込み購入）したかどうかを把握できます。



Windows 10でアプリケーションをUWPアプリケーションにアップグレードすると、 CurrentApp.getAppPurchaseCampaignIdAsync（）メソッドを介してキャンペーントラッキングの追加機能を利用できます。このメソッドは、アプリケーションの操作中に、ユーザーがキャンペーンから来たかどうか、どれ。 つまり、たとえば、さまざまなパートナーに追加のオファーを提供したり、特定のユーザーの特定のショッピングオプションを強調表示したりすることにより、ユーザーの到着チャネルに合わせてゲームまたはアプリケーションをカスタマイズできます。



アフィリエイトプログラムに参加する



ユーザーがどこから来たのかを簡単に理解できるようにすることに加えて、MicrosoftアフィリエイトプログラムをWindowsストアのコンテンツ（アプリケーション、ゲーム、音楽、ビデオなど）を含むように拡張するなど、ユーザーがストアにアクセスする新しい方法にも投資しています 。 。



このプログラムは、すべてのパートナーがWindowsストアのコンテンツの宣伝から手数料を獲得するのに役立ちます。 アフィリエイトプログラムインフラストラクチャは、キャンペーン追跡システムと同じように機能しますが、唯一の違いは、サードパーティプレーヤーがWindowsストアでアプリケーションへのトラフィックを作成し、紹介手数料を獲得できるようにすることです。 アフィリエイトは、24時間以内のすべての有料購入、14日以内の組み込み購入、および6か月間の更新されたサブスクリプションから毎月7％のコミッションを獲得します。



サイトやブログ（Twitterなど）でアプリケーションやゲームのレビューを行う場合、アフィリエイトプログラムに参加し、ユーザーにWindowsストアを紹介するときに適切なリンクを使用することをお勧めします。



広告でアプリを宣伝する



最後に、 アプリケーションの広告キャンペーンを作成する機能を追加することにより、より多くのユーザーを獲得しやすくし続けています 。 開発センターの「アプリの宣伝」セクションを使用して、2種類の広告を作成して実行できます。

  • 目的のデバイスファミリとアプリケーションカテゴリをターゲットにした、他のアプリケーション内の有料広告。
  • 他のアプリケーション内の無料の「 独自の 」広告-アプリケーションまたはゲームがまだインストールされていないデバイス用。


広告キャンペーンを追加したら、広告を通じて設定レポートを使用して、キャンペーンの動作（インプレッション、クリック、設定）を評価し、必要に応じてプロモーション戦略を調整できます。



今週：ストアでアプリケーションの説明を更新するのに1時間かかります



10のWindows 10シリーズの最初の記事をお楽しみください。 今後数週間のうちに、ユーザーがアプリケーションと対話する際の関与を高めるためにできることに集中します。



ただし、その前に、アプリケーションの説明を1時間更新することをお勧めします。

  1. アプリケーションの説明の更新に20分かかります
  2. スクリーンショットを更新し、2400x1200pxのプロモーション画像を追加して、さらに20分間お過ごしください。
  3. 最後に、アプリケーションへのリンク方法を確認し、 キャンペーンIDを使用してインストールソースに関する情報を収集します。


次回アプリケーションを更新するときは、Application Insightsを追加することを強くお勧めします。 ユーザーがアプリケーションでどのように作業するかを理解すると、テレメトリでほぼライブで見ることができ、特に例外やクラッシュの可能性が成功に不可欠な場合に定期的な更新を開始する場合はさらに興味深いです。



いつものように、コメントやTwitter @WindowsDevでフィードバックをお待ちしています。 今週、ハッシュタグ＃win10x10を使用して、テレメトリのヒントからアプリケーションのプロモーションに役立つレビューまで、今週Twitterでアプリケーションプロモーションのトピックについて議論します。
