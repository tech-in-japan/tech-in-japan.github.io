HP Flexible Thin Client-プラクティス





HP Flexible Thin Clientsシリーズは、さまざまな構成とパフォーマンスを備えた複数のシンクライアントを提供します。 シンクライアントは、HP ThinPro、HP Smart Zero Core、Windows Embeddedなどのいくつかのオペレーティングシステムを使用します。



次に、HP Smart Zero Core OSに基づいてシンクライアントをセットアップするプロセスを推奨事項とともに説明します。



パート1



メインモデルシリーズt5565 / t5565zおよびt5570 / t5570eは1つのデバイスであり、シンクライアントは基本OS、フラッシュドライブとRAMのサイズが異なります。 t510の出現後、以前のマーキングを放棄し、シンクライアントは部品番号によってのみ区別できます。



t520はその前身とは異なります。 AMDプロセッサがインストールされ、ラップトップのProbook / ElitBookラインと同様に電源にコネクタが使用され、PS / 2ポートはありません。



t5565 / t5570 / t510のドキュメントはここから 、t520のドキュメントはここからダウンロードできます 。



HP ThinProおよびHP Smart Zero CoreはUbuntu Linuxに基づいています。

バージョン5.0以降、HP ThinProとHP Smart Zero Coreが統合され、バージョン5.2がダウンロードできるようになりました。



Windows t5570 / t5570eは、対応するLinuxよりも高価ですが、必要に応じてHP ThinProまたはHP Smart Zero Coreをインストールできます。



t5565 / t5565zソフトウェアダウンロードページにアクセスすると、HP Smart Zero Core 4.4のダウンロードのみが求められます。 t510のバージョン5.2イメージを安全にダウンロードしてインストールできます。



Windowsはt5565 / t5565zにインストールできません。フラッシュドライブは小さいです。 フラッシュドライブの交換は、実際には経済的に実行可能ではなく、シンクライアントのコストを超えます。



パート2-HP Smart Zero Clientサービス



シンクライアントのフリートを管理するには、 HP Smart Zero Client Serviceを使用することをお勧めします 。



更新と設定を配布するメカニズムは非常に簡単です。シンクライアントはhttpサーバーにアクセスし、OSコンポーネントのプロファイルまたは更新をダウンロードします。 DHCPサーバーまたはブロードキャストを使用して、httpサーバーを検出します。







設置

こちらから19 mb配布キットをダウンロードできます。 インストールするには、Windows XP以上の「サーバー」が必要です。また、インターネットインフォメーションサービス（IIS）および.NET Framework 3.5もインストールする必要があります。



プロファイルエディターと自動Intellidgenceをインストールします。 Intellidgence Delivery Serviceは使用しません。







自動インテリジェンスのインストール中に、HP自動更新Webサイトが作成されます。







DHCPサーバーでDHCP自動更新への「リンク」を作成します;文字列パラメーター137がこれを担当します。







デフォルトのプロファイル

プロファイルエディタを起動して、デフォルトプロファイルを作成します。

「すべてのパラメータを表示」チェックマークの存在に注意してください。

すべてのオプションが緑色で強調表示されます。



HP Smart OSの使用を開始したばかりの場合：

1-イメージのバージョンを指定します。

2-プロトコル（RDP、Citrix、VMware View、WEBブラウザー）を指定します。

3-ホストを指定します。



個人プロフィール

個別のプロファイルを作成するには、ファイルをMACフォルダーに配置し、名前にシンクライアントのMACアドレスを使用します。







トラブルシューティング

更新と設定を配布するメカニズムは非常に簡単です。更新または新しいプロファイルをダウンロードしない場合は、httpサーバーのファイル許可を確認してください。 更新プログラムが読み込まれない理由を理解できませんでした。ファイルを読み取る権限がないことがわかりました。

プロファイルの推奨事項

1-ホストがWindows Audioサービスを実行している場合、ヘッドセットがない場合、シンクライアントは内部スピーカーを使用してサウンドを再生します。 サウンドをオフにすることをお勧めします。必要に応じて、ユーザーはオンにできます。

2-ユーザーは、間違ったパスワードを最初に入力した後にブロックされます。これは、SSL暗号化が使用されている場合に発生します（TLS 1.0）。 RDPセキュリティレイヤーを使用する場合、このようなブロッキングの問題はありません。



RDPセキュリティレイヤー暗号化を強制するには、 freerdpクライアントヘルプを使用します 。

/ sec：rdp引数を次のルート/ ConnectionType / freerdp / connections / {UUID} / ExtraArgsキーに追加します。



暗号化レベルの詳細については、 こちらをご覧ください 。



3-ホストサーバー証明書の検証を無効にすることができます。ルート/ ConnectionType / freerdp / connections / {UUID} / certificateCheckキーがこれを担当します。

4-時間を設定します。ルート/時間ブランチがこれを担当します。

5-管理者パスワードを設定します。これは、root / users / root / passwordキーが担当します。

6-VNCサーバーを構成します。ルート/ vncserverブランチがこれを担当します。

7-デフォルトドメインを指定します。root/ zero-login / defaultCredentials / domainがこれを担当します。

パート8-RDPを介したスマートカードリダイレクト



ページ18（マニュアル4.4）およびページ44（マニュアル5.2）

デフォルトでは、スマートカードは高レベルのリダイレクトを使用してリダイレクトされ、セッションや他のリモートアプリケーションへのログインに使用できるようになります。



このテクノロジでは、スマートカードリーダードライバーのドライバーをクライアントにインストールする必要があります。 既定では、CCIDおよびGemaltoドライバーがインストールされており、利用可能なスマートカードリーダーの大部分のサポートが追加されています。 追加のドライバーは、/ usr / lib / pkcs11 /に追加することでインストールできます。



彼らは箱から出して仕事をすることを約束しましたが、実際には、私のために何もうまくいきませんでした

長い間、ルート/ ConnectionType / freerdp / connections / {UUID} / ExtraArgsキーに/ smartcard： "*"引数を追加するまで、BIFIT iBank 2キートークンの転送を構成できませんでした。 キーは次の形式/秒を取得しました：rdp / smartcard： "*"。



トークンは銀行1番と連携を開始し、銀行2と連携したくありませんでした





診断

pcscdサービスはスマートカードの操作を担当し、バージョン1.7.4はSmartOS 4.4および5.2に含まれています。

トークンの動作を確認するには、次が必要です。

1-サービス/etc/init.d/pcscd stopを停止します

2-デバッグモードでサービスを開始しますpcscd -d -f



トークンを接続すると、ダウンロードのプロセスが表示されます。





何らかの理由で、トークンが定期的に読み込まれず、パターンを特定できませんでした。





ドライバーのインストール

バンクNo. 2およびNo. 3のトークンは、ドライバーがインストールされるまでシステムで定義されません。





ドライバーは、銀行のWebサイトまたはiBank2.RUポータルからダウンロードする必要があります。執筆時点では、バージョン1.08をダウンロードできました。



例：

sudo mount -o rw、再マウント/

sudo apt-get remove pcscd

sudo apt-get update

sudo iBank2Key-Driver-Linux-x86-1.08.sh


最初にpcscdサービスを削除しないと、トークンは検出されません。 また、安全のために、ディレクトリ/ usr / lib / pcsc /および/ usr / lib / pkcs11 /を削除しました。
