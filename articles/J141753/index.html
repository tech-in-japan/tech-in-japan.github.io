IBM Smart Cloud Invitation





何らかの形のクラウドサービス（SaaS、PaaS、IaaSなど）は、最終的には、ほとんどの場合、ネットワークの視聴者から好奇心をそそられることがなくなりました。 これは、大企業だけでなく、中規模および民間の起業家が、さまざまな条件でさまざまな対象者向けに、今日世界中で多数の企業によって提供されているさまざまなクラウドテクノロジーに細心の注意を払い始めた多くの理由の1つです。



しかし、おそらく変わらないことが1つあります。これは、クラウドで提供されるリソースとサービスの品質に対する組織の要件です。 クラウドにタスクを転送しようとする企業の中で、データは「地下」ではなく、おそらく他の国またはおそらく別の大陸のデータセンターに保存されるため、セキュリティ問題はおそらく最も差し迫った問題の1つです。 。 誰もがすぐにそのような一歩を踏み出す準備ができているわけではありませんが、時間が経つと、進歩のおかげで、暗い考えを疑う人を取り除くのに役立つテクノロジーが現れます中型および大型。 会社または組織が大きいほど、処理するデータが多くなります-このステートメントはより関連性があります。







手始めに、少し歴史。 IBMコンピューティングクラウドは、会社全体で最も強力で急成長している2つの分野、つまりメインフレーム生産と仮想化技術開発の自然な共生として現れました。 最初の仮想化実験は、オペレーティングシステムCP-40およびCP-67の仮想マシンの開発が始まった前世紀の60年代にIBMで始まりました。仮想イメージ間でメモリを共有できるソフトウェアをテストおよび開発するために使用されるハイパーバイザーまた、各ユーザーに一定量の仮想ディスク領域を割り当てます。 メインフレームを仮想マシンに分割することにより、IBMはプロセスパフォーマンスの顕著な向上とアプリケーション速度の向上を実現した最初の企業となり、それにより高価な機器の使用効率が向上しました。 1972年以来、IBMはメインフレーム用の仮想マシンテクノロジーを販売してきましたが、現在SmartCloud Foundationは、原則として最大の仮想化イニシアチブの1つです。



1990年2月以降、IBMはPOWERプロセッサーを搭載したサーバーの生産を開始しました。 プロセッサのこのラインは、いわゆるの侵入技術のために特別に作成されました 「ミッションクリティカルな」仮想化。 POWERシリーズのサーバーには、ユーザーとアプリケーション間でアクティブメモリを共有するPowerVMハイパーバイザーが含まれていました。 移行は、2007年5月に発表された第6世代のPOWERで後に導入されました。 次に期待される次のステップは、Smart Cloud Foundationに含まれるすべてのテクノロジーの標準化と自動化であり、スピードを向上させ、リソースの使用を削減しました。 仮想化、標準化、および自動化の分野でのこの作業の組み合わせにより、今日、IBMクラウドが企業環境で最も人気があるという事実に至りました（2011年4月現在の会社によると、Fortune 500企業の80％はIBM Smart Cloudを使用しています）。



実際、2007年以降、Smart Cloudの明るい発展に関するレポートを維持することができます。その直後、IBMはプログラムの主な目標と使命、つまり企業クライアント向けの高品質クラウドの構築とクラウドエコシステムで利用可能なすべてのサービスの提供を明確に形成しました。 2007年10月以降、IBMはGoogleと連携して、大学でクラウドテクノロジーを教えるプログラムを開始しました。さまざまな目的やタスクにクラウドインフラストラクチャを無料で提供することに加えて、2人の巨人が技術専門学生向けのカリキュラムを作成しました。







しかし、雲に直接戻りましょう。 最初に書いたこと、つまりデータをクラウドに保存することの重要性は、通常、さまざまなプラットフォームで実行されているサーバーとアプリケーションのいくつかのイメージを管理することによって企業によって決定されました。 これにより、自分自身をある程度保護できますが、同時に、クラウドテクノロジーの管理の複雑さとコストが大幅に増加します。 クラウドの観点から見ると、サービスを提供するためのモデルとして、これは非生産的なアプローチです。最初はクラウドテクノロジーが従来のホスティングモデルと比較してより効率的に作成されるためです。



少し前に、IBMは、すべてのSmart Cloud顧客に提供される2つの新しいサービスを発表しました。仮想サーバーと物理サーバーのデータアーカイブとリカバリです。 このサービスの本質はシンプルで理解しやすいものです。安全な環境でアプリケーションとそれに関連するすべてのデータを継続的に複製することにより、インフラストラクチャ全体で予期しない誤動作を検出した後、最短ですべての操作を再開できます。



IBM Smart CLoud Virtualized Server Recoveryは、サービスが連続モードでバックアップを実行するため、クラウドの信頼性と効率性を保証します。



IBM Smart Cloud Archiveは、検索および索引付けからデータ抽出およびeDiscovery（電子情報開示）までの幅広い機能を備えた、保管データの厳格な標準および機密要件に準拠するように設計された異なる種類のサービスです。



これら2つのサービスは現在、IBM Smart Cloudの「標準パッケージ」に含まれており、数百の企業が使用するクラウドの一部であるManaged Backupに含まれています。 今年7月19日から、すべてのSmart Cloudユーザーが2つの新しいサービスを利用できるようになります。
