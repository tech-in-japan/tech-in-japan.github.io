3CX v15の新しいCRM統合テクノロジー

CRMシステムとの3CXユニバーサル統合



今日は、おそらく現代のテレフォニーの最も差し迫った問題-サードパーティの情報システム、主にCRMシステムとの統合、を解決する方法についてお話します。 現在、基本的なCRM機能を提供する多くのクラウドテレフォニーサービスがあることに注意してください。 一方、多くのCRMメーカーは、基本的なテレフォニー機能をユーザーインターフェイスに直接統合しようとしています。 通信機能がすでに設計に組み込まれている既製の「ツーインワン」システムもあります。



要するに、これらのソリューションのメーカーはユーザーの要求を聞いており、正しい方向に向かっています。 一般的には、そうです。 ただし、ツーインワンアプローチにはいくつかの欠点があり、その中で最も重要なのは柔軟性が低いことです。 同意して、ユニファイドコミュニケーションと一流のCRMの機能システムを作成するには、まったく異なる能力が必要です。 その結果、ユーザーはコミュニケーション機能が限られているか、開発の見通しが不明確な役に立たない「クリッピング」CRMに直面しています。



CRMが元々テレフォニーと組み合わされたシステムでは、次のような多くの欠点もあります。



  • 多くの場合、ユーザーは自分の機器、特に必要なオペレーターからのSIPゲートウェイとトランクを接続できません。 CRMベンダーは、独自のSIPトランクに高い料金を提供しています。
  • このような統合システムでの電話の安定性は、システム開発の主な方向ではないため、しばしば不満を引き起こします。
  • 非常に限られたユーザー設定。通常、ステータスの最も単純な設定に縮小されます。 コールを処理するためのより複雑なルールを設定する機会はありません。
  • コールキューとコールセンター全体の機能は非常に限られています。 理由は同じです-これは主な優先事項ではありません。 一度に多くの優先事項があります。
  • 限られた電話のパフォーマンスとスケーラビリティ。
  • 分散およびマルチブランチネットワークでの限られた作業。
  • 特に機能に関して、物議を醸す価格/性能比。


おそらく、オールインワンソリューションでは、会社の選択肢が奪われる可能性があります。 コミュニケーションとCRMを1つのシステムに統合することにより、実際、あなたは永遠にそのシステムに執着したままであり、そのすべての欠点を我慢しなければなりません。 結局のところ、これらはビジネスの2つの重要なテクノロジーです。 人気のある知恵によると-すべての卵を1つのバスケットに入れないでください！ そしてもう1つ-前述したように、同じ製品で同じ品質の完全に異なる技術（テレフォニーとCRM）を開発することは非常に困難です。 会社が最高のVoIPおよびCRM開発者を1つのチームにまとめることができるとは考えられません。 そして、できれば、完成品のコストが気に入らないでしょう。 この「統一された」アプローチの結果として、コミュニケーションとCRMの両方の平均化された（または単純化された）機能が得られます。



バッグのスリーインワンコーヒードリンクとお好みの種類のおいしいれた天然コーヒーを、適切な量の良質の牛乳やその他の材料と比較してください。 認められた専門家からの柔軟なテレフォニーと、別に最適なCRMを用意する方が良いと思いませんか？



3CXはまさにそのようなアプローチを提供します。 安価で実績のあるユニファイドコミュニケーションプラットフォームを開発し、標準のAPIを使用して、あらゆる情報システムとの統合を可能にしました。



CRMを3CX v15と統合する場合は、まず、既にサポートされているCRMのリストに注意してください 。 3CX CRM統合がシステムに実装されていない場合、 3CX Visual Studioプラグインテンプレートを使用して、独自の統合プラグインをすばやく簡単に作成できます。 このテンプレートは、必要な統合コードをすべて生成します。



開発者がいない場合は、WebサイトでCRM統合を開発するためのリクエストフォームに記入してください。リソースとの統合を実装する可能性を検討します。



3CX Visual Studioプラグインテンプレートには、プラグインの開発に必要な認証の種類、Webサービスの種類、およびURLを示す構成ウィザードが含まれています。 これらのパラメーターを指定すると、セットアップウィザードは統合コードを生成します。 最小限の変更で統合コードを使用して、CRMに接続できます。



このテンプレートは、次の認証タイプをサポートしています。



  • 認証なし
  • 基本認証
  • OAuth2認証


次のAPI WebサービスCRMシステムがサポートされています。



  • REST
  • SOAP


プラグインを作成するプロセス全体は1営業日もかかりません！

  • 特別なフォーラムで統合と自動化の問題について議論できます。


3CX Phone System ProとFreshdesk CRMおよびNutshell CRMのカスタマーサポートとの統合



私たちは、ユーザーが無料でダウンロードして使用できる既成のCRM統合を可能な限り多く作成することに取り組んでいます。 すべての統合はオープンソースであり、必要な機能を追加することで変更できることに注意してください！



そして現在、新しい3CX Phone System Pro統合プラグインとFreshdesk CRMおよびNutshell CRMのカスタマーサポートを導入しています。

プラグインは次の機能を実装します。



  • Click to Call-CRMのクライアント番号でClick-to-Call呼び出し。 呼び出しは3CXクライアントまたはサポートされているIP電話によって行われます
  • 呼び出しポップアップ-着信時にCRMのクライアントカードが自動的に開きます
  • 発信者番号が見つからない場合、システムはCRMに新しい連絡先を作成します
  • コールジャーナル-すべてのコールはCRMコールログに記録されます


新しいプラグインは、3CX管理インターフェイスの更新セクションにあります。 統合はユーザーごとに個別に含まれます。







  • 3CXとFreshdeskを統合するには、 このガイドをご覧ください 。
  • 3CXとNutshellを正常に統合するには、 このガイドをご覧ください 。


また、すでに統合が実装されている他のCRMシステムのリストをよく理解することをお勧めします。



重要： CRM統合は、3CX Phone System Professionalエディションでのみ実装されています！
