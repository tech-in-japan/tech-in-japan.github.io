Hyper-VでOracle Solaris Express 11をインストールします

少し前に、Hyper-Vの下で古いバージョンから最新バージョンまでのSun Solaris x86の異なるバージョンを仮想化しようとしました。 残念ながら、当時は完全に成功していませんでした。 Solarisは正常にインストールされ、Hyper-Vで起動しましたが、エミュレートされたネットワークアダプターはゲストOSで機能しませんでした。 このような障害を持つことは不可能であった深刻なそのためのSolarisを使用することは明らかです。



日オラクル事業会社の買収に関連してのOracle Solaris改名されました。 コミュニティによるテストのために、無料で入手可能なSolaris Express 11がリリースされましたが、Oracleの新しいほうきがOSに組み込まれたドライバーの開発をより良く変えることを期待していました。 だから私はそれがHyper-Vの下のOracle Solaris Expressの11を感じるだろうかを確認することにしました。



急いでいる人にとっては、結果はかなり満足のいくものだったと言えます。 SolarisはHyper-Vの下でMicrosoftによって公式にサポートされておらず、そのための統合コンポーネントがないという事実にもかかわらず、安定して動作し、許容できるパフォーマンスで動作します。



さて、インストールの詳細、見事な美しさのスクリーンショット、テスト結果に興味がある人は、catに進んでください。





それでは始めましょう。 Solaris Express 11配布キットをダウンロードし 、新しい仮想マシンを作成し、配布キットを起動可能なDVDとして接続し、合成ネットワークアダプターを削除して、レガシーネットワークアダプターを追加します。







仮想マシンを起動し、ハードディスクのパーティション分割、ホスト名の設定、ルートパスワードの割り当てなど、すべての標準的な質問に回答します。 IPアドレスを取得するために、DHCPが選択されました。 ネットワークインターフェイスへの静的IPアドレスの割り当ても正常に機能します。 OSのインストール後にテストされました。







その後、インストールが開始され、約15〜20分かかります。 その後、OSが再起動し、grubプロンプトが表示されます。 ここでは何も設定する必要はありません。[Enter]をクリックして、OSの負荷を引き続き大胆に監視します。







読み込み後、ネットワークインターフェイスdnet0がエミュレーションモードで正常に動作しますが、100 Mbps以下の速度でネットワークを操作できることがわかります。 アイドルモードでは、CPU負荷は1％未満です。







また、静的IPアドレス指定とDHCPの間の切り替えもテストしました。 それから彼は、負荷テストネットワークはデータの10ギガバイトについてFTPを介して送信されました。 ゲスト・プロセッサ上のネットワーク負荷を介してデータ伝送モードで3％まで上昇させます。



起動後、OSはこのモードで4日間機能しました。 OSの動作にエラーや逸脱は認められませんでした。 ログにもエラーエントリはありません。 安定して動作し続けると想定できると思います。



ご覧のとおり、Hyper-V統合コンポーネントがない場合でも、Hyper-VでUnixを操作することはまったく難しくありません。



同時にそれは、ゲストOSが複数のプロセッサ、および2つまたは4つを与えることができることは注目に値します。 デュアルプロセッサ構成への移行の結果、システムの速度が低下し始めます。 それほどではありませんが、肉眼でも目立ちます。 4つのプロセッサを追加すると、ゲストOSが非常に強力になり、使用が不快になります。



これにより、現在、Hyper-VでSolaris Express 11を使用して高性能マルチプロセッサシステムを作成すると失敗すると結論付けることができます。 しかし、十分であってもシングルプロセッサpredastavlyaemogoのゲストOSは、非常に良好なパフォーマンスが得られました。 システムはトレーニングとテストに使用できることがわかりました。 このインストールで処理できるもう1つのタスクは、Windows Server 2008 R2または無料のHyper-V Server 2008 R2での古いバージョンのSolarisに基づくレガシーインフラストラクチャサービスの統合です。 これにより、SC VMMを使用したSC DPMベースのバックアップ、SC OpsMgrの監視、展開、および管理を仮想マシンに適用できます。
