Windowsのスタンプ：Vagrantを使用したHyper-VでのWindows仮想マシンの自動展開（パート1）

遅かれ早かれ、成長するIT企業はリソースを最適化するという課題に直面します。これは、システム管理の観点から、すべてのプロセスの最大限の自動化を必然的に伴うものです。 これは多くの理由で必要です。費やす時間を削減し、人的要因の影響を最小限に抑え、システム全体のスケーラビリティと信頼性のために増加します。



当社のWild Apricotは、ASPおよび.NETで実行され、MS SQL Serverをデータベースとして使用するソフトウェア（SaaS）を開発しています。 したがって、私たちが持っているインフラストラクチャの大部分は、Windows、主にServer 2012 R2を実行しています。 したがって、sysop部門は定期的に新しい車を調達する必要があり、場合によってはライブ環境とテスト環境の両方でファーム全体を展開することもあります。 現在、約20のハイパーバイザーがあり、それに応じて100を超えるさまざまなマシンがあり、その大半はWindows VMです。 現在は、それぞれが互いに分離することが望ましい12個のコンポーネントで構成されている、さらに6個以上の開発環境をインストールすることが計画されています。 このような問題を解決するために、プロセス全体の自動化を考えました。 全体のパスの説明を非常に詳細にしたため、記事が大きくなったため、3つの部分に分けることにしました。 このパートでは、一般的な問題に対処し、ツールの選択について説明し、さらなる作業のために環境を準備する方法を説明します。



なぜWindowsなのか、なぜHyper-Vなのか？



歴史的に、私たちが持っているすべての新しいVMは、Microsoft Hyper-Vハイパーバイザーを実行しています。
これにはいくつかの理由があります。
  • 近年、非常に多く開発されており、全体として、タスクに十分な機能を備えています。 最も重要なこととして、Vagrantは彼を人間的にサポートし始めました。これについては後で詳しく説明します。 また、多くのLinuxディストリビューションのサポートが登場しました。これにより、複数のOSが使用されている場合（他の誰かが間違っている場合）、Hyper-Vで異なるマシンの動物園をホストできます。
  • MicrosoftのサーバーOSとクライアントOSの両方に組み込まれているネイティブツール（MMCツールキットとPowershell）を使用して管理すると便利です。
  • Hyper-Vの役割はサーバーに追加するのが非常に簡単であり、一般的に、インストールは（たとえば、KVMまたはESXiの場合よりも）より広い範囲の管理者が利用できます。言語。 これにより、管理者だけでなく、開発者やテスター（多くの場合、目的のために環境を迅速に上げる必要がある）にも簡単にインストールでき、可能なアプリケーションの範囲が広がります。

    物理マシンから仮想マシンへの移行は、シンプルなユーティリティDisk2VHDの助けを借りて非常に簡単かつ高速です-スナップショットメカニズムを使用し、これにより、ライブ環境で実行されている稼働中のシステムのVHDイメージも作成できます。 インフラストラクチャを仮想化する際に非常に役立ちました。 同様に、VMWareホストを移行しました。

  • 必要に応じて、後でHyper-VからAzureクラウドプラットフォームに簡単に移行できますが、同じツールを使用して引き続き管理できます。

VagrantとChef、Chocolatey、Boxstarterを選ぶ理由



それで彼らは何をし、なぜ選ばれたのですか？

  • Chocolateyは、Windows用のパッケージマネージャーです。 もちろん、MicrosoftはWindows Server 10に独自の統合パッケージマネージャーを搭載することを発表しましたが、本番環境に実装する前はまだまだ先であり、現在のところ、chocolatiersを非常にうまく使用できます。 すべての依存関係とともに、必要なソフトウェアのインストールを自動化できます。 このために、NuGetパッケージが使用されます。これは、自分で作成できます。これは、独自のサーバーソフトウェアをインストールする必要がある場合に非常に便利です。
  • Vagrantを使用すると、ボックスサンプル仮想マシンを作成して使用し、1つ（または複数、または数十）の仮想マシンを自動的に展開し、必要なパラメーターを構成し、いわゆるプロビジョニング（必要なソフトウェアをインストールおよび構成するプロセス）を実装することもできます。 たとえば、IISとChef-clientをインストールでき、最新のWindows更新プログラムがインストールされ、フォルダーが（ファイル拡張子と隠しファイルが見えるように）構成され、ポートがファイアウォールで開かれ、ユーザーアカウントが設定されるなど、すべて直接参加せずに前述のシナリオに従って、必要に応じて自動再起動する管理者。

    類似物の中で、System Centerはすぐに思い浮かび、テンプレートを作成して、そこから複数の仮想マシンを展開することもできます。 ただし、多くの理由で、Vagrantを好みました。軽量で、重い中央サーバーを必要とせず、新しい仮想マシンのディスクは元のテンプレートのディスクに依存しません。 さらに、Vagrantは通常のコマンドラインから機能するため、GUIなしでハイパーバイザー上で簡単に使用できます。

    この記事には、さまざまな落とし穴へのかなりの参照が含まれます-最新バージョンが現在検討されていることをすぐに言いたいです-それぞれ後続のバージョンで1.7.2（私は願っています）多くの問題が修正され、不要な松葉杖が落ちます。 しかし、彼らはきっと新しいバグを追加してくれるので、一連の記事全体を作成することができます。
  • Boxstarterは、完成したマシンと発生したマシンの両方の構成を大幅に簡素化する便利なスクリプトのセットです。 Chocolateyと非常に密接に連携し、1つのコマンドで最終システムを構成し、Webサイトまたはネットワークフォルダーからスクリプトをダウンロードできます。 さらに、その下には、著者によるシェフ向けの既製のレシピがあります。
  • Chefは 、初期の初期化と最新の状態の維持の両方を含む、最も広い意味で構成を集中管理するために使用されます。 代わりに他のシステムを使用することもできますが、その普及率、強力なコミュニティ、および活発な開発のために私はそれに決めました。たとえば、最近、IISのコミュニティクックブックの大幅に改善された新しいバージョンが登場しました。 まあ、もちろん、それは彼が以前にこれを少し経験したという事実の影響を受けました。

Hyper-Vおよび管理ソフトウェアをインストールする


まず、ホスト（仮想マシンがデプロイされるシステム）を準備する必要があります。 それらがローカルテストに必要であり、すべてがあなたが座っているのと同じコンピューター上で回転する場合、すべてがシンプルで透過的です。 ただし、最終的に完全に異なる場所（ライブ環境のデータセンター、近隣のサーバー、または祖母のネットブック）で仮想コンピューターが必要になると、状況がよく発生します。 実際のところ、Vagrantスクリプトはすぐにローカルホストですべての操作を実行します。 原則として、だれも最初に自宅に展開し、設定してプレイし、それから遠いハイパーバイザーに移行することはありませんが、サーバーがあなたと同じネットワークセグメントにある場合でも、かなりの時間がかかることがわかります。 地球の反対側にあるデータセンターのVPNを介して展開する必要がある場合、インスタンスごとに1時間以上かかる可能性があり、不要なアクションを実行する必要がある場合があります：私の場合、エクスポートとインポート、さらに仮想スイッチが呼び出された場合のネットワークの再構成さまざまな方法で。

そのため、最適なソリューションとして頻繁に使用することが計画されているすべてのハイパーバイザーでVagrant Liftを選択しました。 これによりプロセスが高速化されます（1台のマシンでも-結局のところ、ボックスの重量は展開されたイメージのVHDファイルよりも大幅に少なくなります）が、サーバーに余分なソフトウェアが追加されます。 幸いなことに、ChocolateyとVagrantはGUIを必要としないため、無料のHyper-V Serverにも簡単にインストールできます。

このタスク全体はかなり簡単です。

  • Hyper-Vの役割をWindows 8またはServer 2012にインストールするには、再起動が必要であり、GUIを使用する場合はサーバーマネージャーを使用して（記事の最後にあるリンクを参照）、管理者権限を持つPowerShellを使用します：

    Install-WindowsFeature –名前Hyper-V -IncludeManagementTools
  • ショコラティエはPowershellから1つのコマンドで入力されます：

    iex（（new-object net.webclient）.DownloadString（ 'https://chocolatey.org/install.ps1'））
  • まあ、Vagrantは既にChocolateyから上昇しています：

    choco install vagrant
