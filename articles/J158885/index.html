WiFi：責任者：Windows 8またはIntel

Windows 8のリリース後、私は多くの人と同様に、Dell Vostroラップトップのライセンスを購入しました。

私はすぐにインターフェイスに慣れて、本当に気に入りました。 1つを除いて、プログラムに関連付けられていないフリーズが時々ありました。 戦争は生涯のためではなく、死のためでした。 長いテスト、多数のソリューションが見つかりました-何も助けませんでした。 マイクロソフトを支持する公式のアピールでさえ、解決策を与えませんでした。



ラップトップは次の3つの方法でハングします。

1）インターフェイスが機能するか、マウスがクロールし、ボタンのエッジが表示されます。 ただし、アクションを実行するとソフトウェアがフリーズします。

2）画面が起動しない-カバーを開くと黒が表示されます。

3）システムが休止状態または休止状態からブートするときにWindowsロゴを渡した後、システムはオプション2のようにフリーズします。



すべての場合において、ハードディスクインジケータは点滅をまったく停止します。



凍結が非常にわいせつになり、間違った時間にWindows 7にロールバックする準備ができたとき、そのような問題はなかったので...



しかし、最初に、一連の実験を行うことにしました。







毎回、フリーズするたびに、1つのデバイスを切断しました。



デュアルコアプロセッサエミュレーションを無効にし、通常のBluetooth（原則としては使用しませんでした）で終わることから、私は多くのことを試みました。

最後に推測したのは、IntelのWiFiでした。



4日間、飛行は正常です。 ハングアップしません。 休止状態およびスリープモードからの静かな終了。 ワイヤーで作業する...



インスピレーションはすぐに来ませんでした。 そして昨日、インテルのサイトで問題を調査するために登りました...



Windows 8はIntel ProSetをサポートしていません-WiFiプロファイルの作成、編集、削除。 これは便利です。 Windows 7から8にアップグレードする場合、誰もがこのアプリケーションを削除する必要があります。

恥ずかしくて迷惑ですが、大丈夫です...



しかし、後で問題が発生します... Intelは、ラップトップに組み込まれたWiFiカード用のWindows 8のドライバーをリリースしていません。

IntelのWebサイトでも、Windows 8に適していると思われるドライバーがある場合、オペレーティングシステムのリストからWindows 8を選択することはできません。



この問題の解決策は？ ラップトップを分解して、別のメーカーのカードを挿入するか、外部WiFiカードまたはワイヤーを使用します。

ラップトップが半静止状態になるのは残念ですが、これにより将来的にWindows 8で作業し、問題を解決するパッチを待つことができます。
