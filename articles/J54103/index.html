Linuxでパスワードをリセットする方法



Linuxでシステムパスワードをリセットするいくつかの方法を見てみましょう。 おそらく、rootユーザーがシステム内の任意のユーザーのパスワードを変更する権限を持っていることを知っています。 しかし、rootパスワードを忘れたらどうしますか？ これについては、記事で説明します。



パスワード（rootまたは他のユーザー）を回復するとき、異なるアプローチを必要とする2つの異なる状況が発生します。



GRUBはいつ使用できますか



GRUBブートローダーがインストールされており、選択した行をロードするためのパラメーターを編集するアクセス権がある場合、タスクは非常に簡単になります。 さらに推奨事項に従って、できるだけ早くルート権限を取得してください。 そして、あなたがルートになるとすぐに、あなたはすぐに何でもできます。

  • GRUBブートローダーウィンドウで、パスワードをリセットする必要があるLinuxの目的のバージョンの行を選択します
  • 編集するには「e」を押します。 カーネル行を選択します。 行の末尾に「単一」を追加します。 「b」を押してダウンロードします。 システムがrootパスワードの入力を続けている場合は、 init = / bin / bashを行末に追加し、もう一度bを押して起動します
















  • その後、ルートのプロンプトが表示されるか、ルートのある行を選択する必要があるリカバリメニューが表示されます。 passwd <username>を使用して、任意のユーザーのパスワードを変更します









  • 再起動コマンドを実行してシステムを再起動し、新しいパスワードでログインします


一部のディストリビューションでは、インストール中にリカバリモードが作成されることにも注意してください。 GRUBメニューにそのような項目がある場合、上記のすべてのアクションを実行する必要はありません。リカバリモードを選択し、その後のダイアログボックスでルートのプロンプトを選択するだけです。



GRUBを使用できない場合



何らかの理由でGRUBを使用していない場合は、ライブCDを使用してパスワードを回復できます（UbuntuライブCDを使用します。他のものを使用できます）。 以下の手順に従ってください。

  • ライブCDから起動
  • 「コンピューターに変更を加えずにUbuntuを試す」を選択します









  • システムの準備ができたら、ターミナルウィンドウを起動し、コマンドラインでいくつかのアクションを実行する準備をします。
  • ターミナルウィンドウで「 sudo fdisk -l」と入力します。 結果として、Linuxがインストールされているディスクのパーティションと、ハードディスク自体の名前、提示された例では、必要なセクションは/ dev / sda1と呼ばれます。 必要なセクションが正確にわかっている場合は、この手順をスキップできます。









  • 次に、Linuxパーティションをマウントする必要があります。 これを行うには、コマンドsudo mkdir / media / linx_partを使用して、マウントポイントとしてフォルダーを作成します。











  • sudo mount / dev / sda1 / media / linx_partコマンドでLinuxパーティションをマウントします











  • マウントされたパーティションのルートを変更します-sudo chroot / media / sda1
  • passwdを使用して、現在のパスワードを新しいパスワードに変更します。









  • rebootコマンドを使用してシステムを再起動します
