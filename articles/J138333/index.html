Java Day2012。印象

2月10日に行われたJava Dayについて少しお話ししたいと思います。 なぜこれほど遅れて公開するのかをすぐに説明します-最初はJava DayとRed Hatワークショップを比較することでした（ 次の投稿を読んでください）が、イベントは非常に異なることが判明したため、それらについて「混同」を書く意味はありませんでした。 実際、この投稿はそこで起こったことの説明ではなく（ これは既に公開されています ）、私が残した印象についてです。



すでにSun Tech Daysに行ったことがありますが、そのイベントはSunの「翼の下」で開催されたので、率直に言って、Oracleがどれだけ台無しになっているかを見に行きました。 :-)幸いなことに、私はそれほど失望する必要はありませんでしたが、Javaの新しい所有者の影響はまだ非常に顕著でした。



イベントは、ホリデイインホテルで開催されました。 理由はわかりませんが、何らかの理由で、大企業はホテルが会議に非常に成功していると考えています。 個人的には、私はそのような場所が過度に華やかであるのであまり好きではありませんが、これはオラクルの精神に完全に合致しているので、私は驚きません。 少し気味が悪い、しかし、すべては組織のオーバーヘッドなしで行った。 一般に、このイベントは「ドライブ」と企業の厳格さは（Sun Microsystemsの時代に比べて）少なくなります。



Java Day 2012の開催地を探すために長く歩き回る必要はありませんでした。



私が言ったように、組織計画には「ショール」はありませんでした-兆候があり、誰もが十分なスペースがあり、どこにも押しつぶされず、会議プログラムを配りました（驚くべきことに、主催者は時々訪問者はそれを必要としないと思います）、おいしいサンドイッチを与えましたコーヒー（これも忘れられがちで、一日中空腹で座ることはあまり幸せな見通しではありません）。 彼らはペン紙を配っていましたが、オラクルからではなくホテルからでした。 些細なことのように思えますが、すでに「そうではありません」。



これは、聴衆からの全体報告書のようなものでした。



たくさんの若い、熱狂的な人々の存在と、特に素晴らしいこと-多くのかわいい女の子に注目したいと思います。 若者の間でこのような人気を博しているJavaの見通しはそれほど悪くはありません。 ;-)十分に大きいホールはほぼ完全に埋められました。 もっと多くの人が来たいと思った。



まともなサイズの部屋はほぼ満員でした。



Oracleのプレゼンテーションから判断すると、今後数年間の主な「トレンド」は、開発中も使用中も、明らかにクラウドコンピューティングと呼ぶことができます。 これはまさに、最も完全な実施形態がプラットフォームの可能性を最大限に引き出すことができる領域だと思います。 ところで、ごく最近、Larry Ellison はクラウドコンピューティングについて話した人々を笑いました 。そして今、これがJava開発の主な方向です。 それが起こる方法です。 :-)



Javaの未来はクラウドコンピューティングにあります。



プレゼンテーションではコミュニティで頻繁に言葉がちらつき、競合他社の製品についても言及されていたにもかかわらず、講演者にはOracleの競合他社はいませんでした。 つまり、Java Dayではなく、 Oracle Java Dayでした。 そして、私は欠点について話し始めたので、主な部分にあった報告書が何らかの理由で英語で読まれたと言うことはできません。 私は英語のスピーチをあまり認識していないので、母国語のレポートを聞く方が便利だと認めています。 私だけがそこにいたかどうかはわかりません。



しかし、一般的に、繰り返しますが、イベントは成功しました。 みんな満足しました。 スピーカーとリスナーの両方が肯定的な感情を受けました。 多くの人がこのイベントから自分にとって有益なものを持ち出したと思います。つまり、何らかの理由で会議に出席しなかったということです。 主催者に感謝します。
