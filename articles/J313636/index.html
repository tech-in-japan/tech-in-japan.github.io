AD統合を備えたLinuxプリントサーバー

この記事では、ADに統合してLinuxベースのプリントサーバーを作成した経験を共有したいと考えました。 統合とは、LinuxサーバーをWindowsドメインに入れ、Sambaを介してCupsプリンター（プリンタードライバーを含む）を共有することです。 おそらくそれは不器用に置かれたかもしれませんが、これは次のように見えます-Windowsユーザー用のプリンターをインストールするには、「新しいプリンターのインストール」をクリックし、ADでプリンターのリストを表示し、目的のプリンターをクリックします-必要なすべてのドライバーがインストールされたプリンターが自動的にインストールされます 同時に、管理、アクセス、印刷のすべての権利はADから取得されます。

時刻同期



Linuxサーバーの時間は、ドメインコントローラーと同じである必要があります。そうでない場合、ドメインに移動できません。

これにはいくつかのオプションがあります。コントローラとプリントサーバーのドメインで同じntp時刻同期サーバーを指定するか、プリントサーバーのntpサーバーとしてコントローラーのIPドメインを指定します。 2番目のオプションを設定します。 



/etc/ntp.conf server 10.10.15.31 iburst


次のように同期を確認できます。 



 print-01:~ # ntpq -p remote refid st t when poll reach delay offset jitter ============================================================================== *10.10.15.31 85.236.191.80 3 u 888 1024 377 0.698 6.690 7.232

ここで説明します。

BrowseOrder allow、deny-システムがディレクティブの許可と禁止を検討する順序：許可されていないものはすべて禁止されます。

BrowseAllow all-利用可能なすべてのLANプリンターを表示します

BrowseAddress-プリンターへのアクセスが必要なすべてのサブネットを指定します

DefaultAuthType-認証タイプ。 デフォルトは基本です。


後者を犠牲にして。 ロゴに次のことに気付きました。 



 /var/log/cups/error_log cupsdAuthorize: No authentication data provided.


このトピックに関する2つの推奨事項を見つけました。



-Sambaでのプリンターの共有を完全に無効にします（特にプリントサーバーで非常に便利です）

-cupsd.confにこのオプションがある場合は、BasicをNoneに置き換えます（違いを感じませんでした）



cups.orgで、このオプションの値は2-BasicとNegotiate、後者はkerberosを使用した認証になります。

次に、管理パネルと構成ファイルへのアクセスを構成します。 ここで、サブネットを示すAllowディレクティブ（各セクション）またはプリンターを管理できる個別のIPアドレスを登録することもできます。 このディレクティブを追加しない場合-任意のLANサブネットから管理できます-すべて許可と同じです。

誰がサンバに煩わされたくないのか、コップではプロトコルipp（Internet Printing Protocol）を使用して印刷する可能性があります。 Windowsでは、プリンターは次のようにインストールされます。コントロールパネル→プリンター→プリンターのインストール→ネットワークプリンター→インターネット上のプリンターに接続し（Win7 / 8/10の場合は「名前で共有プリンターを選択」）、URLとしてフルパスを指定します。 



  http://10.10.15.11:631/printers/Kyocera_6525_PTO  http://Print-01:631/printers/Kyocera_6525_PTO










プリンターへのフルパスは、Webインターフェイスカップのブラウザーのアドレスバーからコピーできます。







この方法では、システムはプリンタードライバーを要求します。 事前にダウンロードして、インストール中にフィードする必要があります。



RAKE_№1： WinXPでは、SP2以降のサービスパックでippプロトコルがデフォルトで有効になっています。Windows7 / 8/10では、「インターネット印刷」コンポーネントが有効になっていない場合があります。



コントロールパネルからインストール→プログラムとコンポーネント-Windowsコンポーネントを有効または無効にします。 サーバーWindowsでは、このプロトコルはデフォルトで完全に無効になっています。 サーバーマネージャー→コンポーネント→コンポーネントを追加→インターネット経由でクライアントを印刷します。



私はこの問題に2日間苦しみました。 この方法でプリンターをインストールしようとすると、「Windowsがプリンターに接続できません」というエラーが表示されました。 同時に、カップとサンバのログには犯罪者はいません。 それはメガトレッシュでした。 tcpdumpとwiresharkを使用して、ネットワークインターフェースのフロー全体を分析することになりましたが、casが開きました。 問題はWindows側にありました。





パート2.ドライバーのインストール



cupsのプリンターがインストールされていると仮定して、Windowsのプリンタードライバーのコピーと登録を開始しましょう。



Windowsにインストールされているドライバーを手動でコピーできます-％WINDIR％\ system32 \ spool \ drivers \ W32X86および\ x64フォルダーにsambaドライバーバルーン-/ var / lib / samba / drivers / W32X86および./x64を使用して登録しますコンソールユーティリティrpcclientですが、これは非現実的なクエストであり、アクティビティは気弱な人向けではありません。



より簡単な方法で進めます。 ドメインのドメイン管理者アカウントでWindowsマシンにログインしています。 例としてWinXPを示します（以降、Win7での作業方法を説明します）。 エクスプローラーを開き、IPまたは名前でプリントサーバーのアドレスを入力するアドレスバーで、\\ Print-01 \または\\ 10.10.15.11 \に移動し、Printers and Faxesフォルダーに移動します。







プリンターを右クリック→プロパティ。







ドライバーのインストールの申し出では、いいえと言います。







「詳細」タブに移動して変更します。





ディスクからインストールし、ドライバーでフォルダーを指定します。 リストでプリンターを選択し、[OK]をクリックします。



重要 -最初は、システムが64ビットであっても、32ビットドライバーで父親を指定する必要があります！ 64ビットのwoodは後でインストールできます。





ドライバーは共有sambaフォルダーにコピーされています。





[アクセス]タブに移動し、[アクティブなDiectoryに追加]ボックスをオンにして、適用します。 64ビットドライバーが必要な場合は、[追加ドライバー]をクリックします





そして、ボックスx64→OKをチェックします。 システムは、ドライバーを含むフォルダーを要求します-同様にフォルダーにフィードします。





必要に応じて、[全般]タブで、ネットワークプリンターの名前を変更できます。 これらの名前は、エクスプローラでプリントサーバー\\ Print-01 \または\\ 10.10.15.11 \に切り替えると表示されます。





ADでは、プリンター名はカップで呼んだ名前と同じです。
