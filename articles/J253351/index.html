PHPでアスタリスクCLIを呼び出す実装例。 アスタリスク応答の構造化

PHPとAsterisk CLIの友情は松葉杖であるという誤解がウェブ上で広く見られます。 たぶんそうですが、たとえばCRMシステムとの統合を求める顧客からの要望に応じて、VOIPと関連付ける必要があります。ほとんどの場合はアスタリスクです。



CLIから複数のコマンドをすばやく簡単に結び付けて、Webアプリケーションに転送するための既成のデータ配列を取得する方法



実際、すべてが非常に簡単です。 ちょっとしたルーチン作業、ちょっとしたロジック、そして出来上がり。



データを取得するには、文書化されたアスタリスクの機能を使用して、外部からCLIにアクセスします。



資料を読む：



アスタリスク-rxは、外部からCLIコマンドを呼び出します。
-r：バックグラウンドで実行されているAsteriskサーバーに接続し、CLIコンソールにアクセスします

-x：-rオプションと組み合わせて、アスタリスクCLIコマンドを実行します



例：アスタリスク-rx "sip reload"-sipを再起動します。



コンソールでカラー出力を無効にします。
アスタリスクには別の便利なキーがあります。

-n：ANSIカラー出力のサポートを無効にします



しかし、カラー出力をオフにしようとしなかったので、うまくいきませんでした。手が曲がっているのがわかりました。



コマンドを実行するには、2つのPHP関数、exec（）およびpassthru（）を使用します。 



<?php function asterisk_cli_exec ($a) { exec("asterisk -rx '$a'", $b); return $b; } function asterisk_cli_passthru ($a) { passthru("asterisk -rx '$a'", $b); return $b; } ?>

その結果、情報を発行するための2つの異なるオプションを提供する2つの関数を取得しました。



exec（）-文字列の配列として。

passthru（）-結果の直接出力。



asterisk_cli_passthru（）関数は、ブラウザーから独自のCLIインターフェースを作成する場合にのみ有用です。 基本的に、直接的な結論は必要ありません。 Webアプリケーションの場合、配列が必要です。 この段階では、この「おいしい」はすぐに忘れられます。すでに完全に実装されており、これ以上必要なものはないからです。
