LinuxのSteam：クローズドベータの開始

今日、メールボックスの最初の幸運な人は、公式コミュニティで報告されたLinux用Steamクライアントのクローズドベータテストに招待されました。



「招待の最初の波が送信されました！

調査に合格したが、まだ手紙を受け取っていない場合は、絶望しないでください。 今後、多数の調査参加者を対象にベータ版を拡張します。



LinuxでSteamをリリースするのをサポートしてくれてありがとう、

Valve Linuxチーム。」



（* ValveのWebサイトに関する最近の調査では、Linuxおよびハードウェアに関する情報を入力できます）。







Steamはi686アーキテクチャ用のdebパッケージとしてリリースされました。これはmedia.steampowered.com/client/installer/steam.debからダウンロードできます。



また、Steamストアでは、 Linuxで利用可能なゲームの検索がありましたが、それほど多くはありません。基本的にこれは、以前はHumble Bundleで販売されていたものです。



Linuxのベータクライアントについて議論するための専用グループがあります。



招待を受けなかった場合はどうすればよいですか？




待って 待ちたくない場合、ArchLinuxには非公式のPKGBUILDがあります。これには、特に、招待を受けなかった場合に表示されるエラーメッセージのバイパスが含まれます。 メモの作成者のパフォーマンスはテストされていません。 とりわけ、ArchLinuxは既にsteamという名前のパッケージをAURで作成しています。 他のディストリビューションのユーザーは、以下を含むファイル/ usr / bin / steam-hackを作成しようとする場合があります。 



 ＃！/ bin / bash
 xdg-open steam：// open / friends /




それを使用してSteamを実行します。 パフォーマンスは保証されません。
