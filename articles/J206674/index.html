Lync ServerとのIP電話の統合

Lync Serverを企業に展開するとき、Lyncと連携できるIP電話の選択がかなり限られていることは、最も喜ばしい驚きではありません。 また、現在の状況が互換性のある機器の価格にどのように影響するかを推測するのは非常に簡単です。その大部分は1つのベンダーによって製造され、複数のブランドで販売されています。 ここでのポイントは、他のIP電話メーカーの怠evenでさえなく、Lync Server自体のターミナルクライアントデバイスに対する特定の技術要件です。



今日、1つの問題が少なくなっています。 Kinnex Media Gatewayソフトウェア製品の紹介。これにより、既存のタスクと予算に最適な通常のIP電話をLync Serverに接続できます。











説明

Kinnex Media Gatewayは、その名前が示すとおり、一方では本格的なクライアントとしてLync Serverと対話し、他方ではIP電話のレジストラおよびデータコンバーターとして対話するゲートウェイです。 これは、メディア処理エンジンKinnex VoIPエンジンに基づいています。 ちなみに、このエンジンでは、まだ公式にはリリースされていないが、人気が高まっているテクノロジー向けに別の製品をリリースする準備をしています。



サポートされているIP電話のユーザーは、企業テレフォニーのすべての基本機能（番号とSIP URIへの呼び出し、保留、転送、自動転送）にアクセスできます。 準備段階では、Lync自体には存在しないが、時には非常に役立つLyncの高度な機能および追加機能のサポート。 もちろん、ゲートウェイを介して接続されているIP電話でのみ機能します。



Kinnex Media Gatewayでは、互換性を確保する主な機能に加えて、多数の電話を管理するための自動化ツールを個別に取り上げますが、Lync Serverは大規模で分散した組織向けのソリューションです。 IP電話へ。 サポートされている各ハードウェアメーカーのソフトウェア更新プログラムを常に監視およびテストしています。 現在のファームウェアバージョンと構成ファイルテンプレートは、インストールの準備ができているパッケージに収集され、製品のWebサイトで入手できます。



展開

Kinnex Media Gatewayはソフトウェアゲートウェイです。つまり、Lync Serverの役割のいずれか、または専用マシンにインストールできます。 共同インストールに最適なのは、役割が別のサーバーに移動される場合、コンパクト展開、SBS（支店登録サーバー）および仲介サーバーのStandard Editionの役割です。











インストールおよび構成プロセス自体は非常に簡単です。 インストール自体に加えて、Lync Serverからユーザーを追加し、それらのPINコードを設定する必要があります。 認証を含む完全に自動化された接続が必要な場合は、機器のMACアドレスを入力し、ユーザーアカウントに関連付ける必要があります。 その後、開始するには、電話を職場のネットワークに接続するだけです。



メリット

Kinnex Media Gatewayを使用すると、企業にとって非常に重要な多くの利点が得られます。
  1. タスクに最適な機器に基づいて最適なインフラストラクチャを作成する機能。
  2. 機器購入の予算の最適化。
  3. Lyncでの作業だけでなく、使用できる機器の汎用性。
  4. IP電話の簡単なインストールとフリート管理。
  5. 最も適切なメーカーを選択することによる手頃な価格のアフターサービス。


接続できるもの

理論的には、すべてのIP電話はKinnex Media Gatewayを介してLyncに接続できますが、実際には、機器は相互作用プロトコルに考慮すべき独自の特性を持っています。 したがって、電話の重要な要件の1つである作業の安定性を確保するには、各モデルの適応とテストが必要です。



互換性のある機器のリストは、新しいモデルを統合してテストすると、このページで絶えず更新されます。 さらに、ユーザーの声に非常に注意深く耳を傾けています。つまり、誰もがお気に入りのモデルまたはお気に入りのIP電話メーカーの全範囲を統合するためのアプリケーションを提出する機会を持っています。
