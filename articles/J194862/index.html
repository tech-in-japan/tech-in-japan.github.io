台湾の会社Tibbo Technology Inc. Tibbo Project System （TPS）と呼ばれる新しい製品ラインを発表しました。 TPSは、独自の自動化デバイスを作成するためのモジュール式ハードウェアプラットフォームです 。 個々の要素ではなく、デバイスです。 プラットフォームとは、次の要素の組み合わせを意味します。マイクロコントローラがインストールされたマザーボード、入力/出力ボード、将来のデバイスのケース、作成したデバイスのファームウェアを作成できるソフトウェア。 さらに、注文の段階で、クライアント自身が必要な要素と配信するものを選択します。 しかし、まず最初に。







アイデア。



Tibbo Project Systemの重要なアイデアは数年前から存在しています。 ほとんどの自動化デバイスは、ユーザーが「変更」できない固定デバイスと、「ピース」から組み立てることができるモジュール式デバイス-隣り合ってインストールされるI / Oモジュールという2つの広いカテゴリに分類できることが確認されました。レール上。





最初のカテゴリの問題は、正しい仕様のデバイスを見つけることがほとんどできないということです。 ベンダーはこれを理解し、すべての基本的な仕様をカバーしようとし、目を見張るほどの数のモデル（構成）を提供します。 たとえば、命名法を開き、20行のテーブルがあります。 8つのリレーと4つの入力。 4つのリレーと8つの入力。 4つのリレー、4つの入力、シリアルポートなど。 しかし、6個のリレー、2個の絶縁入力、4個のオープンコレクタ出力、シリアルポート、温度および湿度センサー、および高電圧ソリッドステートリレーが必要な場合はどうでしょうか。 いや。 そのような構成はありません。



「複合」システムの問題は、通常、コストが高すぎることです。 彼の顔に真剣な表情で、主要な産業オートメーションサプライヤーの代表者は、デバイスが「数千ユーロ」の費用がかかることを、かなり正当にあなたに言うでしょう！ 優れた自動化は高価であり、クライアントはデバイスを使用するために大金を支払う準備をする必要があります。 そう？



公平に言えば、カスタマイズ可能で拡張可能なI / Oを備えた手頃なシステムがあると言わざるを得ませんが、通常は「未完成」に見えます-ただのボード、通常のケース、見苦しい配線などです。



しかし、優れた自動化は柔軟で、目に優しい、手頃な価格である必要があります。 Tibbo Project Systemを作成する際、Tibbo Technologyのエンジニアはすべてを実現しようとしました。 わかりやすくするために、単純な式を作成し、そのコンポーネントを分析します。



TPS = PCB + Tibbits（I / O）+ボックス



ボード（PCB）。





まず、デバイスのロジック全体が将来機能するプリント回路基板を選択します。 現時点では、2つのベースボードがあり、その違いはI / Oモジュールの座席数のみです（1つの場合は6つのサイトがあり、もう1つは14です）。 周波数88 MHzのチップ（PLL有効）がボードに既にインストールされており、フル機能のイーサネットインターフェイスが実装され、Wi-Fi拡張モジュール、外部ディスプレイとキーボードを接続するためのコネクタが準備され、ハードウェアリセットボタンとプログラマブルボタン（MD）がインストールされ、インストールされていますバックアップ電源用のコンデンサ、電源回路がはんだ付けされ、LEDと音声インジケータが配置されます。 Webサーバーは既にチップに実装されています。 実際、これらのボードはTibboが長い間製造してきたEM1000モジュールです。 今回、モジュールは裏返しになり、今ではこれはコンタクトパッド付きのプリント回路基板です。 将来的には、プログラムのデバッグはイーサネットインターフェースを介して直接実行されると言います。 したがって、「むき出しの」ボードを用意しておけば、すでにボードに接続してプログラミングを試すことができます。



ちび





Tibboビットは、シャーシに配置された小さなI / Oカードです。 Tibbitは、独立して（最もよく知られているマイクロコントローラーで動作する）、Tibbo Project Systemの一部として使用できます。 ピン間のピッチは2.54 mmです。これにより、ピンをブレッドボードに取り付けてテストするのが簡単になります。

Tibbitsは、実際に機能するボードとTibbitsコネクタの2つのサブタイプに分けられます。 現在、40種類以上のTibbitが実装されています。 これらは、ADC、DAC、PoE、バックアップ電源、RS232 / 485、さまざまなリレー、絶縁入力、IRトランシーバー、Tibbitsセンサー（温度、湿度、衝撃センサー）などです。 リストは常に拡大しています。 近い将来、GSM、ステッピングモーター制御などをリリースする予定です。

機能に応じて、Tibbitsはボード上の1つのコンタクトパッドまたはセクション全体（2つのパッド）を占有できます。 ボードへのインストールは基本です。 嬉しいことに、メーカーは振動保護プレートの取り付けを提供しました。このプレートは、輸送中や振動が増加した状態で使用中に飛び出すのを防ぐために、ボードにボードを「押し付け」ます。

したがって、デバイスのハードウェアはほぼ準備できました。 これで、プログラミング環境を安全に実行し、独自のファームウェアを作成できます。



箱





ユニットボリュームでエンクロージャを作成することは、受け入れがたい喜びです。 価格が高すぎます。 ボードの小規模生産があったとしても、エンクロージャーはデバイス開発者にとって依然として頭痛の種です。 しかし、施設でオープンボードを使用することは必ずしも便利ではなく、見た目も美しくなく、機器の故障のリスクにつながります。 したがって、システム全体をケースに入れておくといいでしょう。 すぐに言ってやった。 メーカーは、いくつかのタイプのユニバーサルTibbo Project Boxケースを提供しています。TPB2-小型マザーボード用に設計されたケース。 TPB2Lも同じですが、この場合はLCDディスプレイとタッチキーが取り付けられています。 TPB3-大型マザーボード用のハウジング。

ケースはいくつかのコンポーネントで構成されています。 ボードが取り付けられている底部カバー、側壁、デバイスの上部カバー。 Tibbitsコネクタは、デバイスの側壁と本体に正常に「適合」します。 特徴的な機能は、透明なプラスチックの電話窓（ディスプレイのないバージョン）がトップカバーに取り付けられ、その下に署名付きのステッカーを配置できることです。 エンクロージャー内で「どこから何のワイヤーを」と推測する必要がない場合に非常に便利です。

したがって、ボードが選択され、ビットが配置され、すべてがケースに正常に配置されます-デバイスのハードウェアの準備ができました。



IP68





1つの注意点。 すべてが美しいように見えますが、保護をどうする必要がありますか？ 多くの場合、施設は積極的な外部環境を提供します。 TPBベースケースは、このような条件には明らかに適していません。 この場合、以前のTibbo Technology DS10xx製品ラインのケースを使用できます。 これらのエンクロージャは、押し出しアルミニウムで作られており、外部保護カバーを使用する場合、IP68（完全な防塵および防湿）の保護等級があります。 Tibbitを接続し、ケースデータにインストールするための特別なボードが開発されました。 そして今、軟膏のハエ。 これらのカードにすべてのTibbitsをインストールできるわけではありません。 光センサーまたはIRトランシーバーが、デバイスの保護カバーの下でそれらのタスクに対処する可能性は低いことは論理的です。



オンラインコンフィギュレーター。





デバイスのコンポーネントを選択するプロセスを簡単かつ容易にするために、Tibbo Project Systemオンラインコンフィギュレーターが開発されました。 コンフィギュレータでは、プロジェクト（ローカルマシンに保存できます）で作業します。ここでは、プリント回路基板、利用可能なtibbitを選択し、便利なレイアウトを作成します。 コンフィギュレーターのロジックでは、ミスを犯すことはできません。その隣にボード上に配置する必要のあるビットを配置し、デバイス全体の消費電流を追跡します。 プロジェクトを作成したら、結果のシステムのコンポーネントをコンフィギュレータから直接注文できます。必要に応じて、既に組み立てられたデバイスを注文できます。 同時に、コンフィギュレーターはあなたに何も義務を負わせません。それがtibbit、ボード、または単なるケースの要素であるかどうかに関係なく、このシステムからいつでもコンポーネントを注文できます。

プロジェクトをローカルマシンに保存したら、プログラミングシェルでプロジェクトを開くことができます。環境は、デバイスの操作に必要なすべてのライブラリをソフトウェアプロジェクトに自動的にロードします。



プログラミング





デバイスをプログラムするために、TIDE独自の環境が使用されます（これは洗剤ではなく、TIDEはTibbo統合開発環境です）。 前述したように、プログラミングとデバッグはローカルエリアネットワーク（イーサネット）を介して直接行われます。 環境はユーザーに忠実で、便利であり、ブレークポイントやプログラムをステップ実行する機能など、必要な機能をすべて備えています。 Webインターフェイスはここでプログラムできます。

プログラミング言語はTibbo Basicです。 恐れてはいけません、これは多くの人が学校で見た古典的なBASICではありません。 これは、メーカーが提供する型指定されたオブジェクト指向イベントプログラミング言語です。 C言語にまだ慣れている人はがっかりしないでください。 コンパイラ「C」はすでに準備ができており、ドキュメントの作成段階は完了間近です。 近い将来、コンパイラのリリースを待っています。

ファームウェアの仕組みについて少し説明します。

チップには、ネットワークデバッグを可能にし、コントローラーの重要なパラメーターを監視するオペレーティングシステム（TiOS-Tibbo OS）が搭載されています。 ファームウェアは解釈モードで動作します（コンパイラーはプログラムをニーモニックコードに変換し、クリスタルに入力します）。

システムイベントが発生すると（ボタンを押す、データがシリアルポートで受信される、ライ​​ンステータス、システムタイマーなどが変更される）、このイベントのハンドラーが表示され、さらにロジックが実装されます。

一部のプロジェクトおよびライブラリのソースサイトは、パブリックドメインの製造元のWebサイトに公開されています。これにより、プログラマーはすぐに慣れて独自のアプリケーションを作成できます。



外部から管理する方法は？





まず、自分の手でデータ交換プロトコルを作成することにより、結果のデバイスを任意のシステムに統合できます。 また、製造元はデバイスのドライバーの開発を注文できます。 TKを作成すると、製造元がソフトウェア部分を実装します。 厳密に言えば、製造業者はソフトウェア全体の開発を行うことができます。

無料で配布されるAggreGate Agentライブラリを接続することにより、デバイスをTibbo TechnologyからAggreGateプラットフォームに接続できます。これはソフトウェア自動化プラットフォームです。 AggreGateには、すでに柔軟なSCADA / HMI構成、アクセス制御、ビルディングオートメーションなどがあります。これについては詳しく説明しません。このプラットフォームは別の記事に値します。

最後に、OPCサーバーのリリースが間もなく予定されており、サイトにインストールされているSCADAシステムにデバイスを接続できます。



まとめ...





スタイリッシュなデザインと手頃な価格で作られた機能的な自動化プラットフォームを取得します。 Tibbo Project Systemは、この分野で十分な経験がない場合でも、独自のデバイスを作成するためのシンプルなデザイナーとして使用できます。 柔軟性、機能性、拡張性を考えると、システムはシステムインテグレーターおよびオートメーション業界全体のプロフェッショナルなソリューションとして認識されます。 AggreGateプラットフォームとともに、ターンキーシステムを作成するために必要なすべての機能を利用できます。 監視（たとえば、電気メーターの測定値の監視）、オブジェクトのリモートコントロール、または独立した自動制御システムになります。

とりわけ、メーカーと公式ディストリビューターは、プラットフォームの使用に関する技術サポートサービス、設計作業に関するベンダーからの相談、保証、認定サービスセンターを提供します。 あなたは私たちにあなたの仕事について簡単に書くことができ、私たちはあなたがそれを解決するのを手伝います。
