実際にFacebook APIリクエストを高速化する5つの方法

Webアプリケーションの最も一般的なボトルネックがほとんどの場合、外部サーバーへのHTTP要求であることは周知の事実です。 そのため、API要求データのロード時間は、最も複雑なWebアプリケーションスクリプトのほとんどを実行するのに必要な時間よりもはるかに長くなります。



Facebook APIを使用しながら、クエリの最適化のためのいくつかのレシピを蓄積しました。スクリプトの速度を上げ、その数とリソース消費を削減する方法です。







この記事で概説したメソッドは、Facebook APIでのみ機能します。 しかし、APIを提供する他のサービスに適用できる可能性があることを除外しません。



はじめに



おそらく、ユーザーのデータと彼のFacebookソーシャルグラフで機能するWebアプリケーションのアイデアがあるでしょう。 そして、 ドキュメントを繰り返し開き、いくつかのスクリプトを作成することさえできました。 それを「誕生日プレゼント」アプリケーションとし、最初にすべきことは、ユーザーの友人とその誕生日のリストを取得することです。



すべてがシンプルです、あなたは思う：

  1. ユーザーを承認します（ユーザーIDとaccess_tokenを取得します。これは今後のリクエストで使用します）。
  2. 私は彼の友人のリストを取得します（出力では友人のIDと名前の配列があります）。
  3. 各ID について、ユーザーデータのリクエストを行います （ 誕生日データを取得するには、承認中にfriends_birthdayパーミッションをリクエストする必要があります）。


スクリプトはサーバー側で動作するため、ユーザーに表示する前にすべてのデータを取得する必要があります。 そして、ここであなたは最初で最も重要な問題に直面しています：APIへの1つのリクエストは1〜2秒かかります。 したがって、すべての友達のリストのみを取得するには、2〜4秒（友達の数による）が必要です。

最適化を始めましょう。



方法1：必須フィールドのみを読み取る



ユーザーデータ（Graph APIのユーザーオブジェクト）を要求すると、Facebookはデフォルトで、アクセスできるすべてのフィールドを渡します。 どのフィールドを返す必要があるかを示すことにより、冗長な情報を取り除くことができます。 これを行うために、要求はfieldsパラメータを使用します。このパラメータでは、必須フィールドはカンマで区切られています。 たとえば、この場合、リクエストは{user_id}？Fields = id、name、birthday

  1. [ アクセストークンの取得 ]をクリックし、 [ユーザーデータのアクセス許可]および[ フレンドデータのアクセス 許可]のすべてのチェックボックスをオンにして、 グラフAPIエクスプローラーで必要なアクセス許可を許可します。
  2. https：graph.facebook.com/の後に、ボックスに私を入力してください
  3. 前の出力をリクエストmeと比較しますか？Fields = id、name、birthday






fieldsパラメータを使用したクエリは、必要なデータのみを返すため、高速です。 小さいため、ネットワークを介してより高速に送信されます。 また、より高速に処理され、使用するメモリが少なくなります。



この最適化方法は、全体的なランタイムに実際には影響を与えませんが、決して忘れてはならない方法です。 各リクエストの必須フィールドのみを読んでください！



方法2：1つのクエリで複数のオブジェクトからデータを要求する



idsパラメーターを使用すると、受信する複数のオブジェクトを選択できます。 この場合、 fieldsパラメーターを使用して、オブジェクトの必須フィールドのみを制限できます（オブジェクトは同じタイプでなければなりません）。 たとえば、クエリ？Ids = 4,501012028を使用すると、2人のユーザーから一度にオープンデータを取得できます;）



カンマで区切って指定できるオブジェクトの数は、最大の長いURLによってのみ制限されます。 同時に、要求に対する応答のサイズも増加することを忘れないでください。 したがって、1つの要求でオブジェクトの数を制限することは合理的です。 たとえば、1番目と2番目の方法を使用すると、答えは次のようになります。

方法3：フィルタリングとページネーションを使用する



ページネーションは、情報を「ページ」に分解するだけです。 limitパラメーターを使用して、たった10人の友人に関する情報を受け取りました。 同じ部分の残りの情報を取得するために、 offsetパラメーターが存在します。 たとえば、最初の10人の友人： me / friends？Limit = 10＆offset = 0、2番目の10人の友人： me / friends？Limit = 10＆offset = 10など



次のようにスクリプトを最適化できます。

  1. サーバーへのAjaxリクエストを含むクライアント部分を作成します。
  2. 最初の10人の友人のデータを取得して表示します。
  3. データが表示されている間に、データの次の「部分」をロードします。


これで、情報を表示するために、リクエストのすべての部分が処理されるまで待つ必要がなくなりました。 同時に、最初の部分については、1回の要求で十分な量のデータを既に受け取ります。



1回のリクエストで（または一度にすべて）数十倍のデータを取得できる、より深刻なメソッドに移りましょう。



方法4：FQLクエリの構築（Facebookクエリ言語）



FQLを使用すると、データのクエリにSQLスタイルのインターフェイスを使用できます。 その助けにより、Graph APIを介して利用できないいくつかのクエリを実装するのが簡単になります。 要求フォームは次のとおりです。SELECT [フィールド] FROM [テーブル] WHERE [条件] ただし、FQLには多くの制限があります（SQLと比較する場合）。 したがって、たとえば、 FROMでは1つのテーブルのみを使用できます。 ただし、ネストされたクエリを使用できます。 FQLでは、論理演算子、 ORDER BYおよびLIMIT構文、およびその他の演算子を使用できます。



ユーザーの名前と生年月日を取得するには、ユーザーテーブルでFQLクエリを作成する必要があります。 たとえば、Graph API Explorerでは、これはクエリfql？Q = SELECT uid、name、birthday_date FROM user WHERE uid = me（）になります。 そして、友人のidのリストを取得するために、テーブルfriend fql？Q = SELECT uid2 FROM friend WHERE uid1 = me（）へのクエリを使用します。



サブクエリを使用して、2つのクエリを1つに結合してみましょう。 fql？Q = SELECT uid、name、birthday_date FROM user WHERE uid IN（SELECT uid2 FROM friend WHERE uid1 = me（）） 。 何十ものリクエストの代わりに、必要なすべての情報をたった1つで取得しました！

方法5：バッチリクエスト



Graph APIを使用すると、FQLクエリであっても、1つのクエリからのデータを処理できます。 バッチリクエストを使用すると、1つのリクエストでサーバーに送信できます-いくつかの異なるリクエストのパック。 唯一の制限は、1回のバッチ呼び出しで最大20リクエストです。 要求は、GET、POST、およびDELETEです。



たとえば、リクエストの「パッケージ」は次のとおりです。

  • リンク付きの新しいステータスを送信します。
  • 最新のステータスに関する情報を取得します（新規は考慮されません）。
  • 最後のステータスと同様にコメントを書きます。 


 $ batched_request = '[{"method"： "POST"、 "relative_url"： "me / feed"、 "body"： "メッセージ=新しい投稿は近日公開＆リンク= http：//habrahabr.ru/"}';
 $ batched_request。= '、{"method"： "GET"、 "name"： "get-post"、 "relative_url"： "me / feed？limit = 1"}';
 $ batched_request。= '、{"method"： "POST"、 "relative_url"： "{result = get-post：$。data.0.id} / likes"}} ;;
 $ batched_request。= '、{"method"： "POST"、 "relative_url"： "{result = get-post：$。data.0.id} / comments"、 "body"： "メッセージ=新しい自動コメント" } ';

 $ batched_request。= ']';

 $ url = 'https://graph.facebook.com/';
 $ param = array（）;
 $ param ['access_token'] = $ params ['access_token'];
 $ param ['batch'] = $ batched_request;

 $ ch = curl_init（）;
 curl_setopt（$ ch、CURLOPT_URL、$ url）;

 $ ret = curl_exec（$ ch）;

 //応答処理

 curl_close（$ ch）;
