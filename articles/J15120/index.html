休暇を乗り切る

私たちの地域の多くはリラックスするのが本当に好きではありません。 インターネットは飛躍的に発展しており、2週間の休暇が次のプロジェクトの開始時に「遅れ」となり、結果として金銭の損失になることがあります。

しかし、個人的に休暇は神聖だと確信しています=）私は海とビーチがあまり好きではありませんが、プールでくつろぎ、トルコのウイスキーを飲み、卓球をするのが好きです。 （トルコ=休暇のコンテキストで、すぐに予約する）

そして、別の問題が発生します。誰もいないときにプロジェクトをサポートする方法ですか？ トルコからインターネットカフェまでは選択肢ではありません。 そのため、私は何日もそこをうろうろしますが、休暇に行く価値はありますか？ もちろん、友人に出て行く方法を尋ねることですが、私たちは21世紀に住んでいます。

2週間かけてどのように出発し、basher.ruを開発のために残したのかを説明します。毎日更新しないと意味がありません...

basher.ruは、 bash.org.ru 、 quotes.dirty.ru 、 ibash.org.ru 、 lorquotes.ruなどのリソースのアグリゲーターであることを思い出させてください。 したがって、プロジェクトの存続期間中、更新がタイムリーであることは重要です。 特定の周期で引用符を収集する複雑なパーサーを作成しませんでした。 ただし、 bash.org.ruのメンバーはすぐにサーバーIPを禁止し、そこから引用符の収集が停止しました。 鼻に休暇があり、何かを考え、何らかの形で問題を解決する必要がありました。

解決策はすぐに見つかりました-ローカルマシンでプロキシを作成し、サーバー上のボットがプロキシを突破しています。

すぐに言ってやった。 必要な軸の3proxyをダウンロードして設定します。 私の設定は非常にシンプルに見えます：

nserver 192.168.1.1

nscache 65536

service

log c:\3p\bin\3proxy.log

logformat "L%C - %U [%d/%o/%Y:%H:%M:%S %z] ""%T"" %E %I %O %N/%R:%r"

auth iponly

deny all

allow * 217.65.2.216

allow * 192.168.*

proxy -a







最初にdnsサーバーとキャッシュの下の場所を示します。serviceコマンドはプロキシをサービスとしてインストールすることを示します（弱いWindows軸または1回限りの使用の場合、アプリケーションとして起動することもできます。nixの場合も同様）。 auth iponly-以下にリストされている特定のIPのみから開始します（それぞれ217.65.2.216サーバーアドレス、192.168。*-LAN）。 -aフラグは完全な匿名性について3proxyに通知し、X_FORWARDED_FORなどのヘッダーは送信されません。



以上です。 構成を書き込んだ後、3proxyをインストールすれば完了です。

2番目の問題は、動的IPです。 最初はマイナスのように見えました。なぜなら、数時間の間、魔法のサービスdyndns.comを思い出せなかったからです。 今、私はこれが大きなプラスであることを理解しています。 実際、ボットを取り除くために、「正直な」リソース所有者はストリーム全体を禁止する必要があります...

しかし、今はそれについてではありません。 インターネットは私にZyXelから魔法の装置を与えています。 驚いたことに、それはすでにdyndns.comサービスの操作方法を知っていて、それを操作するための詳細な指示と便利なインターフェースを持っていました。 dyndns.comは、第3レベルのドメインを動的IPに配布するサービスであることを思い出させてください。 登録は簡単で、ドメインを選択し、ソフトウェアをインストールします。各接続でIPが通知され、第3レベルのドメインがあります。 つまり ログインするとすぐに、dyndns.comはコンピューターblablabal.blogdns.netの場所を更新し、ネットワークユーザーはドメインによってコンピューターにアクセスできます。



そのため、私の場合、ソフトウェアをインストールする必要さえありませんでした。dyndns.comでdyndns.comにアクセスの詳細を入力し、内部IPでNATを構成しました（詳細と微妙な点はすべてロシア語のフォーラムにあります）。

以上です。 プロキシのアドレス（ドメイン）とポートをスクリプトにフィードするために残り、サーバーボットはプロキシ、つまり ローカルコンプを介して。 はい、この回路が機能するためには、コンピューターが常にネットワーク上にある必要があります。 しかし、実際には、最後にコンピューターの電源を切ったときのことは覚えていません。 極端な場合は、市場に行き、1000ルーブルの昔ながらの切り株を購入し、クローゼットに入れて、そこに光ファイバーを刺します。 しかし、ポイントではありません。

BIOSの忠実度を高めるため、毎日午前7時にコンピューターの「アラーム」を設定します。 あなたは決して知らない、電気は遮断され、穏やかな魂が残るだろう。



残っているのは、1つだけの問題を解決することでした。これをすべて平面にドラッグする方法です。
