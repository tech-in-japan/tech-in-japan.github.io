アマゾンウェブサービスインフラストラクチャ パート1



Amazonデータセンター内のサーバーの行



クラウドコンピューティングが新しいパラダイムとして出現し、最終的に別の分野になった後、Amazonはこの分野のリーダーになりました。 2006年のAmazon Web Servicesの開始（ほぼ10年前に考えてみてください！）小売業者は、市場シェア60億ドルで、市場で最大のプレーヤーになることができました。



時間が経つにつれて、Amazonのクラウドサービスは数万から数十万の顧客（現在は100万人以上）にサービスを提供し始めました。 したがって、サービスのアップタイムは非常に重要であり、わずかなダウンタイムでさえ、企業の顧客に非常に高いコストがかかる可能性があります。 それほど昔ではありませんでしたが、それでも失敗が起こりました。その結果、Netflix、Reddit、Tinder、IMdB、および他の多くのサービスが被害を受けました。 これらはすべて、米国バージニア州にあるデータセンターの障害が原因で発生しました。 今日、私たちは会社のインフラストラクチャ全体に精通し、おおよその地理と機能を説明します。



そのため、Amazonは現在、グローバルネットワークの少なくとも30のデータセンターを管理しており、別の10〜15がまもなく構築されるか、設計段階にあります。 残念ながら、同社はインフラストラクチャの完全なスキームを開示していませんが、間接データによると、専門家は、米国でのみ、会社のDCの総容量は約600 MWであると結論付けています。



ガートナーの専門家によると、アマゾンウェブサービスの計算能力は、他の14の「クラウド」プロバイダーの合計計算能力の5倍です。



秘密のベールを明らかにする



Amazonは、Amazon Web Servicesの登場以来、Google、Facebook、Microsoftなどの他の企業よりもインフラストラクチャに関する情報がはるかに少ないと報告しているため、データセンターについて話すことに非常に消極的です。 ただし、ここ数年で状況は少し変わりました。トップマネージャーはすでに会社のデータセンターについて話したいと思っています。



「アマゾンウェブサービスの物理インフラストラクチャについてよく質問されます。 私たちはこの分野についてあまり話したことがないので、ネットワークとデータセンターに関する秘密のベールを開きたいと思います。 。



このような会議の主な目的は、開発者がクラウドインフラストラクチャに関するAmazonの哲学を理解し、システムの稼働時間とその信頼性についてさらに学ぶことです。 そのため、インフラストラクチャ全体が11の領域に分割され、各領域にはデータセンターのクラスターが含まれています。 各地域にはいくつかのアベイラビリティーゾーンがあり、お客様はサービスを複製したり、ダウンタイムを回避したりすることができます。 確かに、最近のAWSインフラストラクチャの障害は、Amazonチームがここでもっと徹底的に作業できたことを示しています。



プラットフォーム投資は成長しています



アマゾンウェブサービスは、前年同期と比較して、前四半期に81％成長しました。 これは、インフラストラクチャ全体が同じ速度で成長していることを意味するものではありませんが、ここでは、Amazonが常にインフラストラクチャにサーバー、データウェアハウス、データセンターを追加していることを確認できます。



アマゾンのバイスプレジデント兼リードエンジニアであるジェームズ・ハミルトンは、次のように述べています。 これは非常に重要です。



Amazonのデータセンター開発戦略は現在、コスト削減の考えに基づいています。 ところで、アマゾンウェブサービスの開始以来、サービスの価格は49倍に低下しています。





ヴェルナー・フォーゲルス（写真：YouTube）



「サービスのコストを削減するために多くのことを行っています。 当社のビジネスにはわずかなマージンがありますが、現在のレベルを維持できることを嬉しく思います。 それでも、私たちは定期的にサービスのコストを削減しています」と、Werner Vogelsは状況についてコメントしました。



Amazonの戦略全体の基礎は、データセンターの最適なサイズを決定することです。 会社の代表者によると、ほとんどのAmazonデータセンターには5万から8万台のサーバーがあり、単一のデータセンター容量は25〜30 MWです。 会社は長い間この数字に行きました。



データセンターの大きさはどれくらいですか？



Amazonデータセンターの規模が大きくなると、単一のデータセンターの障害の価値も大きくなります。 会社の専門家によると、いわばデータセンターは失敗の要素です。 また、データセンターが大きいほど、障害の規模への影響が大きくなります。 そのため、同社は10万台を超えるサーバーの容量を持つデータセンターを作成せず、ほとんどのDCはより小さく、容量も小さくなっています。



2番目の質問が発生します-いくつのサーバーがAmazon Web Servicesをサポートしていますか？ 会社の副社長から提供された情報により、150万人という最低人数について話すことができます。 プラットフォームによって計算されるAWSサーバーの最大数は560万です。



Amazonは、Digital Realty TrustやCorporate Office Properties Trustなど、多くのデータセンター事業者の設立をリースしています。 過去に、同社は倉庫などの建物をリースし、それらをデータセンターに変換しました。 最近、Amazpnは、DCをゼロから作成することに焦点を当てて、戦略を変更することを決定しました。 オレゴンでは、同社は既製のモジュラーコンポーネントを使用して完全なデータセンターを構築しました。



アマゾンの利点は、独自のエネルギー変電所の作成です。 ここでは、運用費用の管理ではなく、作業の速度を確保する必要性がすでに明らかになっています。 節約は最小限ですが、はるかに速いペースでデータセンターを作成できます。



カスタムサーバーとデータストレージ



クラウドプラットフォームの開発の初期段階で、Amazonは最も有名なメーカーから機器を購入しました。 Amazonの主要な機器サプライヤはRackable Systemsでした。 2008年だけで、Amazonはこの会社から8,600万ドルで、1年前に5,600万ドルでSeversを注文しました。



しかし、インフラストラクチャの成長に伴い、同社はデータセンター用の独自のハードウェアの開発を開始しました。 これにより、Amazonはサーバー、データウェアハウジング、ネットワーク機器を微調整し、コストを削減しながらすべてのハードウェアのパフォーマンスを最適化できます。



「はい、独自のサーバーを作成しています。 既製のソリューションを購入することもできますが、それらは非常に高価であり、ニーズに合わせて最適化されていません。 したがって、私たちは独自の機器を作成します。 従来のプロセッサを高性能モードで使用できるようにするため、インテルと協力しました。 これにより、非常に特定の目的で使用するためにカスタマイズされたサーバータイプを作成することができました」とVogels氏は述べています。





画像：ジェームズハミルトン



EC2インスタンスでは、Haswellアーキテクチャを使用した22ナノメートルプロセステクノロジーを使用して作成されたXeon E5プロセッサに基づいて構築されたサーバーです。 さらに、ここのサーバーはさまざまな構成であり、さまざまなタスクを実行するように設計されています。



Amazonの専門家によると、同社は現在、特定のソフトウェアやサービスの操作のサポートなど、特定の範囲のタスクに最適な構成でサーバーを構築する方法を知っています。

光と雲の速度



「光速度の速度」は、Amazonのインフラストラクチャの設計において重要な役割を果たします。



「顧客が仕事をする最も一般的な方法は、特定のデータセンターでアプリケーションを起動することです。このオプションでは、99.9％のアップタイムで十分であるという点で、データセンターの信頼性を最大限に高めることができます。 ただし、信頼性の高いアプリケーションまたはサービスを作成する場合、その動作を保証するために2つのデータセンターが既に必要です。 さらに、データセンター間の距離は非常に長くなる可能性があり、信号経路は非常に長くなります。 したがって、特にデータセンターが互いに離れている場合、分散インフラストラクチャの作成は困難な作業になる可能性があります」とVogels氏は述べています。



この問題に対する答えは、アベイラビリティーゾーンです。これは、顧客がさまざまな隔離された場所でインスタンスを実行できるようにするリージョン内のデータセンターのクラスターであり、したがって障害点を回避します。 あるインスタンスに何かが発生した場合、アプリケーションは別のアベイラビリティーゾーンの別のインスタンスによってサポートされます。 各リージョンには2〜6のアベイラビリティーゾーンがあります。



同時に、同社はアベイラビリティーゾーンを相互に分離しましたが、ネットワーク上の信号遅延を最小限に抑えるのに十分な距離にありました。 会社の専門家によると、ゾーン間の信号遅延は通常1〜2ミリ秒です。 比較のために、ニューヨークからロサンゼルスへのデータの受け渡しの遅延は70ミリ秒です。



「アベイラビリティーゾーンを近くに配置することにしました。 しかし、それらは依然として異なる地理的地域に存在し、異なるエネルギーネットワークに接続されている必要があり、さらに、海抜の異なる高さにある必要があります」とハミルトンは言います。
