開発者のサイトからChrome拡張機能とアプリケーションをインストールする

先週、ブラウザ拡張機能のインストールに関するChromeのルールに違反するプログラムを特定するための新しいセキュリティ対策の導入を発表しました。



Chrome拡張機能をインストールするための標準メカニズムを使用することは、ユーザーだけでなく開発者にとっても有益です。 例として、開発者のサイトから拡張機能をインストールすることを検討してください。



このテクノロジーを使用すると、ChromeユーザーがサイトのUIから直接拡張機能またはアプリケーションをインストールできるようになります。 これはユーザーにとってわかりやすいだけでなく（ Chrome Web Storeページに移動し、そこでインストールを開始してから開発者のサイトに戻る必要はありません）、拡張機能やアプリケーションのダウンロード数を大幅に増やすことができます。



サイトからのインストールはアプリケーションの成功にどのように影響しますか？


  • Evernote Clearly拡張機能のインストールの15％とChromeでのEvernote Web Clipperのインストールの25％は、サイトからのインストールを通じて開始されます。
  • RovioがChromeバージョンの Angry BirdsをChromeウェブストアにリダイレクトする代わりにサイトからインストールした後、インストール数が10％増加しました。


サイトからのインストールの実装は非常に簡単です。


  1. Chromeウェブストアでアプリケーションへのリンクを提供します。
  2. クライアントブラウザーがアプリケーションに必要なすべての機能をサポートしているかどうかを確認する小さなスクリプトを作成します（WebGL、 Web Audio APIなどのサポート）。 Modernizrライブラリはこれに最適です。 Habréについては、すでに詳細に説明されていました。
  3. JSメソッドを呼び出して、インストールプロセスを初期化します。






ユーザーがアプリケーションをインストールすると、インストール確認ダイアログが表示されます（上の図を参照）。 Chromeウェブストアから直接インストール中に表示されるダイアログのように、アプリケーションが要求するすべての権限がここにリストされます。 さらに、ダイアログには、Chromeウェブストアのアプリケーションの評価とユーザー数が表示されるため、ユーザーはインストールするアプリケーションについて知ることができます。



ユーザーは「追加」ボタンをクリックするだけです。 これで完了です。サイトを離れる必要さえありません。
