Siege-Webサーバーのストレステスト用ユーティリティ

この資料が誰かに役立つことを願っています。



Siegeは、Webサーバーの負荷テストユーティリティです。 開発者ができるだけ実際のコードに近い条件でコードのリソース消費を確認できるようにするために作成されました。 Siegeは、サイトに一度にアクセスする複数のユーザーをシミュレートすることもできます。 これにより、サーバーを長時間「包囲状態」に保つことができます。 「攻囲」中に行われたリクエストの数は、ユーザーの合計数とサーバーへのアクセス数から計算されます。 たとえば、20人のユーザーが50回適用すると、合計1000件のリクエストが作成されます。 テスト後にプログラムによって表示される結果には、検証に費やされた時間、送信された情報の総量（ヘッダーを含む）、サーバーの平均応答時間、そのスループット、および応答がコード200で送信された要求の数が含まれます。これらのデータは毎回生成および発行されます検証。 以下で詳細に説明します。 Siegeには、回帰テストモード、インターネットシミュレーションモード、ブルートフォースモードの3つの主要な操作モデルがあります。 プログラムは、構成ファイルからリンクの一部を読み取り、それらに順番に（回帰テストモード）またはランダムに（インターネットをシミュレート）アクセスします。 または、ユーザーは、すべての呼び出しが行われる単一のアドレス（ブルートフォースモード）を指定できます。

起動オプション

Siegeの起動形式は次のとおりです。「siege options」。 プログラムは次のパラメーターを使用できます。



「-V」

「–バージョン」

現在のバージョンを表示します。



「-h」

「–ヘルプ」

ヘルプを表示します。



「-C」

'–Config'

現在の構成を表示します。 Siegeは設定を読み取り、その内容を表示します。 これらを変更するには、$ HOME / .siegercファイルを編集します。 そのようなファイルがない場合は、siege.configユーティリティを実行してファイルを生成する必要があります



「-v」

「–詳細」

詳細情報の出力。 このオプションを有効にすると、Siegeはサーバーへのすべての呼び出しに関する詳細情報を表示します。 これには、HTTPプロトコルのタイプ、応答コード、および呼び出しのアドレスが含まれます。 例：



HTTP / 1.1 200 OK：/cgi-bin/whoohoo.cgi?first=Homer&last=simpson



このオプションは、プログラムが広範囲のアドレスにアクセスする回帰テストまたはインターネットのシミュレーションの場合に特に便利です。



「-g URL」

「–URLを取得」

指定されたリンクにアクセスします。 サーバーからヘッダーを取得し、画面に表示します。 スポットテストに最適なツール。



'-c NUM'

'–同時= NUM​​'

シミュレートされたユーザーの数。 このオプションを使用すると、NUMで指定された同時ユーザー数でWebサーバーをテストできます。 実際、この数はコンピューターリソースによってのみ制限されますが、実際には、良好な結果を得るには、数百のユーザーセッションをシミュレートするだけで済みます。 どんな設定でも、実際の人の行動を真似しないことに注意してください。 少なくとも、彼らは各ページに留まり、情報を読んでいて、リンクを次々と整理しないからです。



「-i」

「–インターネット」

このオプションは、多くのリンクを含む構成ファイルで使用されます。 オンにすると、Siegeは呼び出しのアドレスをランダムに選択し、それらに要求を行います。 実際には、ユーザーにどのページをどの順序で送信するかを伝えることはできません。 彼らは常に異なる方法でこれを行います。 シミュレートされたインターネットモードは、この動作をエミュレートしようとします。 テスト中、ファイルの一部のアドレスはランダムに選択されるため、まったくアクセスできない場合があることに注意してください。



「-t NUMm」

'–時間= NUM​​m'

テストに合格するのにかかる時間。 「NUMm」という形式で示されます。NUMは時間単位の数、「m」はそれぞれ秒、分、時間のS、M、またはH修飾子です。 たとえば、1時間のテストを開始するには、-t3600S、-t60M、-t1Hの組み合わせを使用できます。 修飾子は大文字と小文字を区別しませんが、修飾子と数値の間にスペースを入れないでください。



'-fファイル'

'–ファイル= FILE'

リンクを含む構成ファイル（SIEGE_HOME / etc / urls.txt）。 このオプションを使用して、別のパスを指定できます。 例：



siege –file = serverb.txt



'-l'

「–ログ」

このオプションは、すべての情報をSIEGE_HOME / var / siege.logログファイルに書き込む必要があることをSiegeに伝えます。 新しいテストごとに、ファイルが追加されます。



'-mメッセージ'

'–Mark = MESSAGE'

このオプションを使用すると、ログファイル内のさまざまなテストに関するレコードを共有する式を指定できます。 次のように-lオプションを使用する必要はありません。 自動的にオンになります。 MESSAGEの式にスペースが含まれる場合は、必ず引用符で囲んでください。



「-d NUM」

'–遅延= NUM​​'

このオプションは、シミュレートされたユーザーがサーバーにアクセスする間の遅延を指定します。 遅延時間は、1から入力された数値まで計算されます。 アプリケーションのリソースを集中的に使用するセクションでテストする場合、遅延を1秒（-d1）に設定することが望ましいです。 デフォルトでは、遅延は1秒から3秒まで発生します。このオプションを使用すると、サーバーを要求波でカバーできます。



構成ファイル

バージョン2.00以降、Siegeは、頻繁に使用するコマンドを保存できる構成ファイルをサポートしています。 これは、ほぼ同じ設定の多数のテストで役立ちます。 このファイルは.seigercと呼ばれ、Siegeをインストールしたユーザーのホームディレクトリにあります。 このファイルがない場合（たとえば、プログラムをインストールしなかった場合）、siege.configユーティリティを使用して作成できます。 ファイル内には、コメント付きのさまざまなディレクティブがあります。 任意のテキストエディターを使用して編集できます。



受け渡し可能なURL形式

Siegeは次のリンク形式を理解しています。



[プロトコル：//] [server.domain.xxx] [：ポート] [/ディレクトリ/ファイル]



HTTPおよびHTTPSプロトコルを使用したアドレスのみがサポートされます。 指定する必要がある最小値はサーバー名です。 ドメイン内にいて、shemという名前のサーバーをテストしており、hostsファイルまたはローカルDNSに登録されている場合、コマンド

「Siege -u shemp」はアドレスshemp.domain.net / index.shtmlにアクセスします。 Siegeでhttpsサーバーを使用する場合は、追加のプロトコルも指定する必要があります。 したがって、「siege -u shemp 」 コマンドは、プログラムにshemp.yourdomain.net/index.shtmlでアドレス指定を強制します。



リンクファイル

回帰テストを開始するか、インターネットをシミュレートする前に、検証済みアドレスのリストをプログラムに渡す必要があります。 これを行うには、それらをSIEGE_HOME / etc / urls.txtファイルに配置します。 その中で、アドレスは1行に1つずつ配置する必要があります。



homer.whoohoo.com/howto.jsp

homer.whoohoo.com/index.jsp

homer.whoohoo.com/cgi-bin/hello.pl?first=bart&last=simpson

など



Siege-2.06以降では、POSTおよびGETディレクティブの存在がサポートされています。 GETディレクティブはデフォルトで使用され、オプションです。 ただし、POST要求の場合は、ディレクティブを指定する必要があります。 例：



homer.whoohoo.com/cgi-bin/hello.pl POST name = homer

homer.whoohoo.com/haha.jsp POST word = doh！＆scope = ALL



[-u URL | --url = URL]、このファイルからアドレスを取得します。 通常モードでは、Siegeはファイルの先頭から開始し、すべてのアドレスに徐々にアクセスします。 インターネットの模倣モードを選択した場合[-i | --internet]、アドレスはランダムにアクセスされます。 [-f FILE |オプション]を使用して、リンクを含むファイルへの別のパスを指定できます。 --file = FILE]。



変数

バージョン2.57のリリース以降、Siegeは.siegercファイルおよびリンク付きファイル（urls.txt）での変数の宣言をサポートしています。 Siege変数宣言の構文は、UNIXシェルの構文に似ています。 「varname = value」の形式で1行に1つ宣言されています

変数の名前は、$（）または$ {}構造内に配置されます。 これらを使用して、たとえば、チェックされたリンクの2つのプロトコルをすばやく切り替えることができます。



PROT = https：//

$（PROT）eos.joedog.org/siege/index.php

$（PROT）eos.joedog.org/wacky/index.php

$（PROT）eos.joedog.org/scout/index.php

$（PROT）eos.joedog.org/libping/index.php

$（PROT）eos.joedog.org/gunner/index.php



この例では、すべてのリンクのプロトコルを変更するために、1行だけを変更する必要があります。



ログファイル

ログオプション[-l /-log]をオンにしてSiegeを起動すると、プログラムはPREFIX / var / siege.logに表示されるすべての情報をログに記録します。PREFIXはSiegeインストールディレクトリです（INSTALLファイルを参照）。 プログラムによって標準モードで画面に表示されるものはすべて、ログファイルに書き込まれます。 記録中の情報はカンマで区切られ、他の形式にすばやくインポートできます。

さまざまなチェックの結果を分離するために、オプション-m“ text” /-mark =” text”があります。 スキャンを開始する前に、指定したメッセージをログファイルに保存して、結果をすばやく見つけられるようにします。 たとえば、httpおよびhttpsプロトコルを使用してリンクをテストする場合、「HTTPSテストの開始」などのマークをログに残すことができます。 -m / –markオプションを使用する場合、-l / –logオプションはオプションです。



スタディ統計

プログラムによって発行される統計には、合計テスト時間、送信データの合計数（ヘッダーを含む）、平均サーバー応答時間、そのスループット、サーバーがコード200で応答した呼び出しの数が含まれます。レポート自体は、リンカーンスタイン拷問スクリプトによって生成されます.pl。 Siegeレポートの例を次に示します。



ベン：$ siege -u shemp.whoohoo.com/Admin.jsp -d1 -r10 -c25

..Siege 2.65 2006/05/11 23:42:16

..25人の同時ユーザーを戦闘に備える。

サーバーは現在包囲されています...

トランザクション：250ヒット

経過時間：14.67秒

転送されたデータ：448000バイト

応答時間：0.43秒

トランザクションレート：17.04トランス/秒

スループット：30538.51バイト/秒

同時実行性：7.38

状態コード200：250

成功したトランザクション：250

失敗したトランザクション：0





ここに：

トランザクション-サーバーへの呼び出しの数。 この例では、この数は各ユーザー[-r10]を10ヒットした25人のユーザー[-c25]から計算され、合計で250人です。

経過時間-テストの合計期間。 サーバーへの最初の呼び出しから計算され、最後の要求に対する応答の受信で終了します。 この例では、テストに14.67秒かかりました。

転送されたデータ-シミュレートされたすべてのユーザーが送信したデータの合計量。 リクエストボディとそのヘッダーの両方が含まれます。

応答時間-サーバーがクライアントに応答した平均時間。

トランザクション率-サーバーが1秒間に処理するのに管理した呼び出しの平均数。 これは、リクエストの合計数を費やした時間で割ることによって取得されます。

スループット-サーバーからユーザーに毎秒送信されるデータの平均数。

同時実行性-サーバーが遅延なく応答する同時接続の数。

成功したトランザクション-サーバーが400未満のコードで応答したリクエストの数。



プラットフォーム

Siege Multi-Streamingは、次のプラットフォームで構築され、正常にテストされています。



* AIX（powerpc-ibm-aix4.2.1.0）。 Siegeは「IBM C for AIX」バージョン5コンパイラを使用してコンパイルされ、正常にテストされています。

* GNU / Linux（i [56] 86-pc-linux-gnu）。 Siegeは、gccを使用してSuSE GNU / Linuxで開発されました。

* HP-UX（hppa2.0w-hp-hpux11.00）Siegeは、「HP ANSI Cコンパイラ」とgccを使用してこのプラットフォームでコンパイルされました。

* Solaris（sparc-sun-solaris2 [678]）Siegeはこのプラットフォームを気に入っています。 通常、gccを使用してコンパイルされました。

* Microsoft Windows（pc-i686-cygwin）SiegeはCygwin Funkmanに移植されました。 1.5以降のSiegeのすべてのバージョンで問題なく動作します。
