VPS上のSQL Server Express 2008の問題

何も悪いことではありません。サイト（ASP.NET MVC + MS SQL）の準備ができていたので、それをホスティングに展開するときが来ました。 プラットフォームとして、InfoboxのHyper-Vに基づいた仮想サーバーが選択されました（ホスティングについての詳細-準備中）。 データベーススキームを備えたスクリプトは準備ができており、サーバー上で実行するためだけに残っていました。 そして、彼は表示されます-SQL Server Express 2008（Windows Server 2008 + SQL Express 2008のイメージが仮想サーバーにアップロードされました）。



問題



そして問題は、権利の欠如のために、私が基盤を作成できなかったことでした。 だから、ここでは、「浴槽から吐き出されたように」©。 ログインのリストにはBuiltin \ Users（素晴らしい名前！）とsaがありました。 最初の権利は公開されていましたが、もちろん私はsaからのパスワードを知りませんでした。



調査



SQL Server 2008をインストールするときに、適切な権限を持つBuiltin \ Administratorsグループ（sysadmin）が追加されなくなったことがわかりました。 「セキュリティ上の理由」のため。 私たちは別の解決策に関心があるので、この決定の適切性に関する議論を省略します-権利の問題。 Microsoftは（特別な場合のみ！Ahtung！3回クロスしてください！）-fフラグでサーバーを起動し、Administratorsグループのメンバーに必要な権限を付与します。 しかし、残念ながら、Expressでこれが許可されていないか、その日が悪かったので、成功しませんでした-私が得たのは、赤い十字の付いた空のポップアップだけでした。



解決策



とても簡単です。 データベースエンジンを再インストールし、インストール時に混合認証を選択し、saの下に入力して生活を楽しみます。 そして、なぜ私が突然このすべてを書いたのかを知りたいなら、猫の下を見てください。



サポート



当然、最初の反応はサポートのチケットを作成することでした。 彼らはすぐに、エンジニアに渡すと答えました。 エンジニアの対応も非常に迅速で、丁寧で無意味でした。「これはデフォルトではないため 、すべての権利があります。」 個人的にこれを検証するように頼んだとき、彼らは私が技術部門と相談する必要があると丁寧に答え、明日私に答えを与えます。 サポートに感謝し、Amazonで同じことを見つけたと言い、問題を解決し、寝ました。 午前中、リモートデスクトップのログインとパスワードの入力を求められましたが、何かを疑ったということではありません。 私はあなた自身の危険とリスクですべてを自分で再インストールする方が良いと判断しました。



私はサポートのレベルを評価することを約束しませんが、私はそのような状況が初めてだったのか、または私がそのような問題でそれらに対処した最初の人であることに驚きました。 見た目は、ほとんどありません。



ホスティング自体については、これまでのところ満足しています。 PROMO-512プランに従ってサーバーを選択しました。これらは、500 MHz、512 MBのRAM、25 GBのスペース、およびシェアウェアトラフィックが保証されています。 リモートデスクトップは高速で、サイト自体も高速です。 ロシア語版のWindows Serverは非常に面白く、「ウェブサイトを作成する」ことにまだ慣れることができません。



ここに物語があります。 偶然にも、誰かにとって役立つことが判明するかもしれません。
