PlayStation 2のプログラミング：コントローラーライブラリ-パート1



この記事を書き始めたとき、私はコントローラーの章がさらにリリースされることに気付きました。

これは、ジョイスティック自体、2番目のジョイスティック、バイブレーション、マルチタップ（4つのジョイスティックを1つのポートに接続するためのデバイス）です。

ジョイスティック自体には、それぞれ独自の情報を提供する2つのライブラリがあります。



私は2つのオリジナルDual Shock 2ジョイスティックを使用していますが、2つの非オリジナルジョイスティック（特別なデバイスまたは4人のマルチタップを介してコンピューターに接続するために使用）があります。







各コントローラーには、いわゆるターミナルコントローラーがあります。これは、コントローラーにアクセスするための実際のポートです。



コントローラを制御するために、ボタンからのデータを提供する特別なライブラリlibpadがあり、コントローラモードと振動を変更します。

「通常の」libmtapライブラリもあります。 どちらのライブラリもマルチタップで機能します。



ライブラリは、PS1およびPS2からの次のコントローラーをサポートします。

残りについては、途中で情報を補充します。



ポートとスロット



PS2では、ポートとスロットの概念は2つの異なるものです。

ポートはメインコントローラーを指します-PS2自体の最初と2番目のジョイスティック用です。

スロットは、ほとんどの場合、MultiTapがあるときに使用されます。 複数のジョイスティックを1つのポートに接続するには、マルチタップが必要です（最大-4）。 通常、ゲームではこれはジョイスティック1A、ジョイスティック1Bなどとして表示されます。

マルチタップがない場合は、原則として、ポート0、スロット0-最初のジョイスティック、ポート1、スロット0-2番目のジョイスティックを煩わせないでください。

マルチタップでは、2番目のポートにマルチタップが挿入され、スロットAでジョイスティックが接続されている場合、ポート1、スロット0を使用してアクセスする必要があります。

ジョイスティック



libpadを使用するには、ライブラリ「libpad.a」とヘッダーファイル「libpad.h」を接続する必要があります。

開始するには、アプリケーションの起動時に「sceSifLoadModule（）」関数を使用する必要があります。

「–lpad」パラメーターをメイクファイルに追加する必要があります。



コントローラーを使用するための基本手順：



scePadInit（）-ライブラリの初期化。 パラメーターは0です。



scePadPortOpen（）-コントローラーポートを開きます。 戻り値パラメータは、使用するポート、スロット、バッファです。 サイズは64バイトでなければなりません。 ビューを宣言します。

static u_long128 pad_dma_buf[scePadDmaBufferMax] __attribute__((aligned (64)));





scePadDmaBufferMaxは、宣言する必要のない定数です。

int関数は戻ります。これにより、ジョイスティックが挿入されていることを確認できます。挿入されていない場合、パラメーターは0になります。



さらに、ライブラリーは、コントローラーの電源を入れて開くと、コントローラーと自動的に通信し始めます。 さらに、2つの機能があります。



scePadRead（）-ボタンの読み取りに使用されます。

この関数には、ポートとスロット、および結果を保存するためのバッファーの3つのパラメーターがあります。

バッファは符号なし文字バッファです[32]。

関数は結果も返します。 ゼロに等しい場合、読み取りはフロッピーになります。

次に、バッファを読み取る必要がありますが、実際には簡単ではありません。



scePadGetState（）-これを使用して、コントローラーが落ち着いた状態にあるかどうかを確認できます。 ポートとスロットのパラメーターを受け入れます。

コントローラのステータスを返します。 ステータスがゼロ以上で、7以下である場合、何かが起こりました。



コントローラーを閉じるには、scePadPortClose（）関数を使用します。この関数では、ポートとスロットのパラメーターを使用し、scePadEnd（）の後にライブラリーをオフにします。 通常、これらの2つの関数は使用されません。



これらの機能はすべて、モードがオンの場合にのみアナログの読み取りにも使用されることを追加します。



最も一般的な例は次のようになります。



main(){<br> unsigned char rdata[32];<br> u_long128 pad_dma_buff[scePadDmaBufferMax] __attribute__((aligned(64)));<br> scePadInit();<br> scePadPortOpen(0, 0, pad_dma_buff);<br> while (1) {<br> scePadRead(0,0,rdata);<br> // <br> }<br> scePadPortClose(0,0);<br> scePadEnd();<br>} <br> * This source code was highlighted with Source Code Highlighter .







DualShockおよびタッチ感度





最初に、scePadSetMainMode（）関数でアナログモードを有効にする必要があります。 パラメーターのポート、スロット、1、および0を取ります。0または1の形式で結果を返します。つまり、それぞれオンまたはオフになります。



コントローラーが実際にDualShockであることを確認するには、scePadInfoPressMode（）関数を呼び出す必要があります。 ポートとスロットのパラメーターを受け取り、結果を0または1として返します。



モード自体の有効化は、scePadEnterPressMode（）関数によって行われます。 パラメータを受け取り、前の関数と同じように返します。



さらに、バッファのscePadRead（）関数を介して、さらに1バイトを受け取ります。このバイトには、ボタンを押す度合いが0〜255の値で表示されます。0は、ボタンが押されないことと同じです。



メモとして、最初のジョイスティックはときどき最大3回押す力を示すことがあります。

仕様によれば、アナログの動きの完全に通常の誤差はプラスまたはマイナス25です。



scePadExitPressMode（）関数を使用して、キー感度モードを終了できます。 ポートとスロットを受け入れます。 メニューに切り替える必要がある場合があります。



ユーザーが高感度モード中にアナログボタンを押すと、感度なしで通常のデジタルモードに切り替わります。 アナログボタンをもう一度押しても、ジョイスティックは感度モードに戻りません 。



振動





ジョイスティックの振動は非常に簡単であり、同時にそうではありません。

別のライブラリを接続するか、libpadに含まれる2つの関数のみを使用できます。



次回、別のバイブレーションライブラリについて説明します。すぐに2つの簡単な機能について説明します。これらはscePadSetActAlign（）およびscePadSetActDirect（）で、ポート、スロット、および6バイトのねじれを受け入れます。

最初のバイトでは、強いモーターまたは弱いモーターを示します。

2番目のバイトでは、弱いモーターは値0または1を取ります。これは、停止またはねじれを意味します。 強力なモーターは、2番目のバイトに1〜255または0の値を取ります。これは、ねじり力または停止を意味します。

scePadSetActDirect（）関数は、振動操作の完了を示すtrueまたはfalseを返します。 残りのバイトは、使用しない場合は0xffで埋めることをお勧めします。

その後、ユーザーのジョイスティックは、振動操作のために10個の現在のユニットを受け取ります。

現在のユニット数を20に増やすには、残りの値0x00を入力します。



重要！

各ポートは、現在のユニットの最大数である60に制限されていることを知っておく必要があります。60を超えると、コンソールがリブートするまでポートがブロックされます（内部リミッターがトリガーされます）。

したがって、マルチタップを使用しており、1A、1B、および1Cジョイスティックに20ユニットの電流を送信する場合、1Dジョイスティックにこれを行うことはできません。そうしないと、ポートをブロックできます。 したがって、電流供給を増やすことはお勧めしません。



また、これらの機能は長い振動には使用できないことも付け加えてください。



非同期関数





次の関数は非同期です。 彼らはあなたに結果を同時に返さないかもしれません。 この場合、関数scePadGetState（）またはscePadGetReqState（）を使用して実行結果を取得できます。



scePadSetMainMode-入力モードの切り替え。

scePadSetActAlign-振動機能;

scePadEnterPressMode-ボタン感度モードへのエントリ。

scePadExitPressMode-ボタンの感度モードを終了します。



終わりに近い





一般に、ジョイスティックが削除（オフ）されていても、原則として、アプリケーションはポートとスロットで引き続き動作します。 scePadStateDisconnectを返すscePadGetState（）を使用して、ジョイスティックが削除されたことをプログラムで確認します。

各フレームの前に、コントローラーのステータスを確認することをお勧めします。これにより、ジョイスティックが誤ってオフになる状況がなくなります。



また、DualShockまたはDualShock 2コントローラーのバージョンによっては、アナログの仕様が変更される可能性があるため、ジョイスティックの自動キャリブレーションに依存しないでください。 したがって、ほとんどの場合、アナログが127x127で停止しなかったという事実を修正する必要があります。
