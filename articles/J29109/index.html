Windows Server 2003ブラックサインイン画面

ある晴れた日、Windows Server 2003は葬儀（ゴシック？）の挨拶を発行し始めました。 こんな感じ



これはシステムの動作には影響しませんでしたが、パスワードの入力には問題がありました。 Runetには解決策がありませんでした。 しかし、英語のフォーラムの1つには、問題、原因、および解決策の正確な説明を含むトピックがありました。



結局のところ、現象の原因は... システムディスクの空き容量が不足しています。 不明な理由により、レジストリのリセットが発生します。 スペースを解放したら、設定を手動で戻す必要があります。



そして、設定は次のとおりです

[HKEY_USERS \ .DEFAULT \ Control Panel \ Colors]

"ActiveBorder" = "212 208 200"

"ActiveTitle" = "10 36 106"

"AppWorkSpace" = "128 128 128"

「背景」=「102 111 116」

"ButtonAlternateFace" = "181 181 181"

"ButtonDkShadow" = "64 64 64"

"ButtonFace" = "212 208 200"

"ButtonHilight" = "255 255 255"

"ButtonLight" = "212 208 200"

"ButtonShadow" = "128 128 128"

"ButtonText" = "0 0 0"

"GradientActiveTitle" = "166 202 240"

"GradientInactiveTitle" = "192 192 192"

"GrayText" = "128 128 128"

「ハイライト」=「10 36 106」

"HilightText" = "255 255 255"

"HotTrackingColor" = "0 0 128"

"InactiveBorder" = "212 208 200"

"InactiveTitle" = "128 128 128"

"InactiveTitleText" = "212 208 200"

"InfoText" = "0 0 0"

"InfoWindow" = "255 255 225"

「メニュー」=「212 208 200」

"MenuText" = "0 0 0"

「スクロールバー」=「212 208 200」

「タイトルテキスト」=「255 255 255」

「ウィンドウ」=「255 255 255」

"WindowFrame" = "0 0 0"

"WindowText" = "0 0 0"


修正後、ウェルカムウィンドウは標準のフォームに戻りました。



ホリバーを開始することは意味がありませんが、Microsoftのサーバー（！）OSのこの動作は非常に奇妙です。 スペースがいっぱいになると、レジストリの設定が上書きされるのはなぜですか、UFOだけが知っています。
