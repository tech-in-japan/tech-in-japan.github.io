オペレーティングシステムのインストールと設定（Fedora 25）



次にソフトウェアについて。 私はこれまたはそのLinuxディストリビューションを拒否しません。 歴史的に、私は主に開発とクライアントのインストールプラットフォームの両方にRed HatまたはCentosを使用しています。 そのため、選択はFedora 25で行われました。Centosほど保守的ではなく、最新のFedora 26とは異なり、多少安定しています。







ここからネットワークインストールに最適化されたISO分配を取ることができます 。 ラップトップの最終状態を理解するために、インストールと構成についてもう少し詳しく説明します。







手始めに、USBフラッシュドライブにISOイメージをスローする必要があります。 たとえば、他のLinuxコンピューターを使用する場合：







  • ISOファイルより大きいボリュームの空のUSBフラッシュドライブを挿入します。 その上にあるファイルシステムのタイプは重要ではありません;最も重要なことは、ボリュームに分割すべきではありません。

  • どのようにマウントされ、どのデバイスに対応するのかを見ていきます。 







    cat /proc/mounts


  • このデバイスをアンマウントします。そうしないと、必要なモードで使用できなくなります。 







     umount /dev/NNN


  • ルートモードに移動し、ISOファイルが保存されているディレクトリに移動します。
  • データのブロックコピーには標準ユーティリティを使用します。 



     dd bs=4M if=Fedora-Workstation-netinst-x86_64-25-1.3.iso of=/dev/NNN


BIOSで、USBブートオプションを選択し、インストーラーをロードします。 ネットワークにはカラフルなインストール手順があります（正式にはバージョン26ですが、バージョン25との違いは見当たりません）。 この指示に従って、以下を行う必要があります。







  • インターネットアクセスが機能するように、ケーブルが接続されているネットワークカードのすべてのパラメーターを設定します。IP、ゲートウェイ、DNS、アダプターのアクティブ化、コンピューター名の設定。
  • 時間設定ウィンドウで、正しいタイムゾーンを選択し、時間を設定します。 個人的には常にネットワーク経由で時刻同期を使用しています。
  • ディスクをパーティション分割します。 通常、デフォルトのレイアウトスキームを使用しますが、すべての作業データを/ workフォルダーに保存することを好むため、/ homeフォルダーのサイズを大幅に削減します
  • インストールプロセス中にパッケージが取得される最も近いミラーを選択します。


その後、「インストールの開始」をクリックします。 パッケージのダウンロードおよびインストール中に、rootパスワードを設定したり、システムにユーザーを追加したりすることができます。 インストールが完了したら、再起動します。その後、環境とソフトウェアを構成します。 この段階の主なツールはコンソールです。 当然、以降のすべての操作はルートの下から実行されます。







  • 新しくインストールしたシステムで最初に行うことは、システムを更新することです。 システムにはすでにdnfパッケージマネージャーがあります。 また、適切なdnfコマンドを呼び出す古き良きyumもあります。 私は習慣的にyumを使用しています。 バッチリポジトリはデフォルトですでに設定されているため、システム全体を更新するにはコマンドで十分です。 







     yum update


  • ディストリビューションに含まれていないソフトウェアで便利なリポジトリを追加します。 ソフトウェアを直接インストールするのではなく、リポジトリからインストールすると、新しいバージョンが表示されたときに（yum updateコマンドによって）自動的に更新されるという利点があります。


  1. rpmfusion-VLCメディアプレーヤー、コーデック、Steam用。 2つの小さなパッケージをインストールするだけで、リポジトリがアクティブになります 







     rpm -ivh http://download1.rpmfusion.org/free/fedora/rpmfusion-free-release-25.noarch.rpm rpm -ivh http://download1.rpmfusion.org/nonfree/fedora/rpmfusion-nonfree-release-25.noarch.rpm


  2. Google Chromeリポジトリ：完成したパッケージはありません。次の内容のgoogle-chrome.repoファイルを/etc/yum.repos.dフォルダーに手動で追加する必要があります。 







     [google-chrome] name=google-chrome baseurl=http://dl.google.com/linux/chrome/rpm/stable/x86_64 enabled=1 gpgcheck=1 gpgkey=https://dl.google.com/linux/linux_signing_key.pub


  3. GNOME3をシステムシェルとして使用する場合、その拡張機能をGoogle Chromeからインストールするには、region51-chrome-gnome-shellリポジトリのchrome-gnome-shellライブラリが必要です。 このリポジトリも手動でインストールされます-/etc/yum.repos.dフォルダーにあるため、ダウンロードする必要があります。 



     wget https://copr.fedorainfracloud.org/coprs/region51/chrome-gnome-shell/repo/fedora-25/region51-chrome-gnome-shell-fedora-25.repo


  • その後、1つのコマンドで、すべての汎用アプリケーションソフトウェアをインストールできます（当然、このリストは最初に必要なものの例に過ぎず、誰にも強制することはありません）。 



     yum install mc gnome-commander menulibre gnome-tweak-tool dconf-editor vlc gimp inkscape gthumb steam lshw lshw-gui brasero audacity thunderbird unzip unrar google-chrome-stable java

  • 標準リポジトリには、便利なデスクトップ拡張機能がいくつかあります。
