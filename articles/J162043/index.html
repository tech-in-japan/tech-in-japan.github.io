Robotron CM 1910-過去のコンピューター

VEB Kombinat Robotron





1969年、ドイツの民主共和国に「人民企業」VEB Kombinat Robotronが設立され、タイプライターやラジオからパーソナルコンピューターやメインフレームに至るまで、さまざまな機器が製造されました。 ドイツ民主共和国が生産したコンピューターは、ソ連を含むCMEA諸国に分散し、あらゆる種類の研究機関や設計局で働いていました。



そのような「ワーカホリック」についてお話したいと思います。







興味のある方は誰でも、habrakatへようこそ（ 怒っている猫にはたくさんの写真があります）。





このデバイスは、1988年春にライプツィヒフェアで正直な人々に初めて紹介され、Robotron A7150（別名SM1910、別名CM1910-ソ連版）という簡単な名前が付けられました。



VEB Kombinat Robotronが製造した以前のコンピューターとは異なり、CM1910モデルには20 MBのハードドライブ（はるかに少ない-40 MB）、512 KBのRAM、クロック周波数4.9 MHzの16ビットSiemens SAB8086-Pプロセッサ（Intel 8086マイクロプロセッサクローン）がありました）およびコプロセッサ8087。2台目のハードドライブをインストールする可能性も計画されていました。コンピュータには、無料の内部5インチベイと別のハードドライブ用のはんだ付けされていないインターフェイスがありました。



1988年のRobotron CM 1910の価格は56,873ドイツマルクでしたが、1年後には43,969マルクに下がり、1990年初頭にはこのコンピューターを36,000マルクで購入することができました。



ただし、Robotron CM 1910コンピューターは2番目のハードドライブを取得できず、他のVEB Kombinat Robotron計画は実現しませんでした。 1990年6月30日に、Kombinat Robotronは清算され、そのユニットは合資会社に転換され、その後も清算または売却されました。



企業はもうありませんが、コンピューターは残ります。



Robotron CM1910コンピューターの1つのコピーは、ゼロ年で科学研究所から以前の所有者に移行し、スクラップコレクターの手にあるという避けられない運命から彼を救いました。 しかし、数年が経ち、コンピューターは何百倍も高速になり、古いRobotronはスペースを取りすぎ始めました。 ええ、広告のように。



そして最後に、コンピューターが私のところに来ました。



外観



モニター







Robotron CM1910コンピューターには、黒と緑の12インチCRTモニターが装備されており、その外観は非常に標準的です。 ブラウン管スクリーンのすぐ下のモニターケースの前面には、電源インジケータライトと輝度調整ノブがあります。







ケースの背面には、電源スイッチ、電源ケーブルとヒューズのペアを接続するためのコネクタがあります。 ハウジングのカバーは金属でできており（X線放射からユーザーをよりよく遮蔽するためのようです）、通気孔があります。



キーボード







キーボードは標準のものとはまったく異なります。CTRLおよびESCボタンは非常に珍しい場所にあり、右のALTキーの代わりにCLボタンがあります。 ファンクションキーPF1〜PF12は、3つのボタンを右にシフトしました。現在、それらの一部は、通常のキーボードのINS、Home、およびPageUpキーが配置されている場所にあります。







LEDインジケータは2倍あります。通常3つはテンキーの上にあり、さらに3つはALT1、^ S、およびMOD 2キーの上にあります。おなじみのF1〜F3ボタンがあるはずの場所です。 キーボードコネクタ-オスDB9。 ケース素材-プラスチック。



本体







システムユニットの外観は、その当時の通常のIBM PCとほとんど変わりません。 Robotron CM1910コンピューターのケースは固体金属で作られており、時々わずかに黄色くなるフロントパネルはプラスチックで作られています。 ケースの前面には、2つの5.25インチ720Kbドライブ、リセットボタン、LEDインジケータHD、RUN、HALT、PWR ONがあります。 ハウジングの側壁には通気孔があります。







ほとんどすべてのインターフェイスコネクタと電源ボタンは、システムユニットの背面にあります。 キーボードとグラフィックタブレットコネクタは、便宜上、ケースの左側壁に表示されます。



システムユニットは非常に重いため、おおよその重量は22キログラムであることに注意してください。



システムユニット内



分解されたモニターとキーボードの写真を撮るかどうかをよく考えた結果、これらのデバイスの内部は重要ではないと判断されました。 実際、内部には高電圧の最も一般的なモノクロCRTモニターがあり、ケースの下にはソビエトのマイクロ回路を備えた完全に普通のキーボードがあります。



ただし、Robotron CM1910システムユニットの内部世界はIBM PCの詰め物とはまったく異なるため、特別な注意が必要です。



それでは始めましょう。







ケースの背面の両側にある4つのネジを外し、背面パネルを取り外します。 ハウジングの側壁を延長し、最後に上部カバーを取り外します。



側面のネジをさらに4つ外して、フロントパネルを取り外します。以前にケーブルを外していました。







フォアグラウンドには（左から右へ）：



2つの5.25” 720Kbドライブからのハードディスクドライブ





20 MBのMFMハードドライブRobotron K5504.20。





2つの電源。 大きな電源は、ハードドライブと+12 Vに沿ったドライブに電力を供給します。



小型のPSUの目的は私にはわかりませんが、すぐにファンになったと確信できます。





ちなみに、ファンには220ボルトの電圧が供給されており、実際にはノイズは発生しません。





下のドライブの下には、MMS16モジュール用の3つのスロットがあり、そのうち2つはハードディスクとドライブコントローラで占められています。 最初のスロットは無料で、鉄のプラグで閉じられています。





バックグラウンドには、スロット（7個）に取り付けられたMMS16モジュールと、+ 5 Vに沿ってモジュールボードとハードドライブに供給する3番目の400W電源ユニットがあります。





マザーボード自体が欠落しています。 コンピューターのすべての主要コンポーネント（電源を除く）は、MMS16バスを使用して相互接続されます。







各モジュールには、 ドキュメント内に独自の英数字の指定があり、次の機能を実行します。









ABG K7075は、CGA規格と互換性のあるビデオカードです。 これは、コネクタを使用して相互接続された2つのプリント回路基板のサンドイッチで形成されます。





KGS K7070-キーボードコントローラー（IFSS）およびグラフィックタブレット（V.24）。 コンピューターの左側にあるキーボードとグラフィックタブレットのコネクターは、ケース内のケーブルを使用してこのコントローラーに接続されます。





KES K5170-ハードドライブとフロッピーディスクのサブシステムRobotron CM1910。





OPS K3571-256 KBのRAM。 2個。





ZVE K2771は、Siemens SAB8086-Pプロセッサと8087コプロセッサがインストールされたモジュールで、セントロニクス、IFSS（電流ループ）インターフェイスコネクタ、およびケースのフロントパネルのボードを接続するコネクタも収容します。





ASP K8071-ポートV.24（Comポート）、IFSS（電流ループ）、IFSP（ラジアルパラレルインターフェイス、LPTと完全に互換性がない）のコントローラー。





AFS K5171-フロッピー磁気ディスク（HDD）上のドライブのコントローラー。





AFP K5172-ハードディスクコントローラー（HDD）。



Robotron CM1910は、セントロニクス、V.24、IFSP、IFSSの4種類のインターフェースをサポートしています。 Centronicsインターフェースはよく知られたパラレルポートであり、V.24はシリアルComポートですが、IFSPおよびIFSSインターフェースは特別な注意に値します。



IFSPインターフェースは、CMEA諸国で製造された多くのプリンターおよびコンピューターで使用されている、いわゆる放射状並列インターフェースです。 長い接続ケーブルを使用できます。 Centronicsは完全に互換性がありません。



IFSSインターフェース （電流ループ）-電流強度の測定値を使用して情報を送信するように設計されたインターフェース。 IFSSインターフェースは、長距離（最大数キロメートル）でデータを送信するために使用されました。 1990年代までソ連およびCMEA諸国で製造されたコンピューターで広く使用されていました。



コンピューターはセントロニクスとV.24のインターフェースを標準化していますが、接続ケーブルを変更せずにそれらを使用することはできません。



ご存じのとおり、IBM互換コンピューターの標準LPTポートには「マザー」タイプのコネクターがあり、標準（DB25とはいえ）COMポートには「オス」コネクターがあります。 Robotron CM1910コンピューターの場合、これらのコネクターは正反対に作られており、標準プリンターまたはマウス接続ケーブルの嵌合部分は物理的にそこに収まりません。



ただし、コネクタをはんだ付けした後、プリンターとマウスをRobotron CM1910コンピューターに接続する機会があります。



しかし、すべては理論的にのみ良好で滑らかですが、実際にはまったく逆になりました：



1.マウス





写真ソース： http : //www.robotrontechnik.de/



Robotron CM1910コンピュータ用のマウスは、通常のDB9「メス」ではなく、25ピンコネクタが必要です。 はい、そして「男性」。 ジーニアスの普通のげっ歯類しか入手できませんでしたが、これは原則として致命的ではありません。







プリンタから古いケーブルを取り出し、目的のコネクタのはんだ付けを解除し、信号の仕様を確認して、マウスを新しいテールにします。 DOS用のドライバーをダウンロードし、マウスを接続し、 gmouse.exeを実行すると...離陸しませんでした。



質問が生じました、Comポートはまったく機能しますか？ これは、 CheckIt診断プログラムを使用して簡単に検証できます。次のような簡単なテストスタブがあります。







はんだ付けし、プログラムを実行すると...テストは両方のV.24ポートに失敗しました。



この場合、シリアルインターフェイスの物理的な誤動作について自信を持って話すことはできないことに注意してください。 曲がっただけの可能性があります。 または、障害はRobotron CM1910コンピューターと通常のIBM PCとの不完全な互換性です-ポートアドレスと割り込みは標準のものと異なる場合があり、CheckItプログラムは信頼されるべきではありません（特にドライブを正しく識別しなかったため）。 しかし、このコンピューターで入手可能なドキュメントはドイツ語で書かれているため、この理論を確認または反論することはできませんでした。 私は後背地からのソビエトのパルチザンのレベルでこの言語を話しますが、自由時間がないため、後でOCRを使ってテキストをGoogle翻訳者に送りたくありませんでした。



2.プリンター



プリンターの状況はあいまいです。 理論的には、セントロニクスインターフェイスは標準化されていますが、Robotron CM1910には非標準のコネクタピン配列がある可能性があります。 これに関する情報がドキュメントに見つからなかったため、既存のEPSON LX-100マトリックスプリンターをRobotron CM1910コンピューターにねじ込むという考えを捨てなければなりませんでした。



鉄の部分が終わったら、ソフトウェアに進みます。



オペレーティングシステム



Robotron CM1910には、次のオペレーティングシステムをインストールできます。



DCP1700は、MS-DOS 3.2オペレーティングシステムのクローンです。 Robotron CM1910で最も一般的なOS。

SCP1700はCP / M86と互換性のあるオペレーティングシステムです。

MUTOS1700はUNIXクローンの1つです。

BOS1810は、マルチタスクのリアルタイムオペレーティングシステムです。



DOSで記述された一部のプログラムを使用すると、Robotron CM1910がIBM PCハードウェアと完全に互換性がないため、問題が頻繁に発生しました。 それでも、ドライバーを少し改良してコンピューターのマウスを接続すると、Microsoft Windows 3.0を起動することさえ可能になりました。



残念ながら、配布キットKarl-Marx-StädterWindow-Systemを見つけることができませんでした（これは、ドイツ民主共和国で最終決定されたMicrosoft Windows 3.0のバージョンが呼ばれたものです）。





写真ソース： http : //www.robotrontechnik.de/



Robotron CM1910コンピューターで最初に使用しなければならなかったプログラムの1つが、ハードディスク上にパーティションを作成およびフォーマットするように設計されたMWINCHユーティリティであったことは奇妙なことではありません。







問題は、このコンピューターには、磁気ヘッドの自動駐車機能のないRobotron K5504.20 MFMハードドライブがあることです。 このようなハードディスクでコンピューターをシャットダウンする前に、駐車コマンド（park、hdparkなど）を実行する必要があります。そうしないと、ハードディスクに記録された情報だけでなく、ハードディスク自体も損傷する可能性があります。



残念ながら、ハードドライブにはparkコマンドのようなものはなく、HDDパーキング手順をバイパスしてコンピューターの電源をオフにする必要がありましたが、すぐに支払いが必要でした-次の誤ったシャットダウンの後、オペレーティングシステムは読み込みを停止しました。



まあ、私は自分でそれを壊し、私は自分でそれを修復する必要があります。



これを行うには、Motherlandのビンから5.25インチドライブといくつかのフロッピーディスクを取得する必要がありました。また、Windows 98を搭載したコンピューターが必要でした。



ブータブルフロッピーディスク DCP 3.3のイメージと、このイメージを5インチフロッピーディスクに復元するTelediskプログラムをダウンロードします。



フロッピーディスクから起動し、 mwinch.exeを使用してパーティションを作成し、 format c：/ sコマンドでハードディスクをフォーマットします （/ sスイッチを使用して、システムファイルをディスクからハードディスクに転送するようにフォーマットユーティリティに指示します）。 完全に幸福にするために、autoexec.batおよびconfig.sysをフロッピーディスクからC：ドライブのルートにコピーし、残りのファイルをディレクトリ（たとえば、C：\ DCP-DOS）にコピーします。



オペレーティングシステムがインストールされます。 利益！



8086プロセッサの大きな可能性を評価するために、当時のいくつかのプログラムをインストールして実行するだけでした。
