ウェルカムスタートボタンがWindows 8.2に戻ります

2013年10月にリリースされたWindows 8.1では、ユーザーとMicrosoft自体の主な問題を解決できませんでした。 もちろん、非常に大きな変換が行われましたが、それらはすべてセンサー指向のタイルインターフェイスを対象としています。







通常のコンピューターにも何かがありましたが、ユーザーは以前の「絶滅」を実際には得ていませんでした。 Windows 9のリリース前に、誰もがWindows 8.2を待っているのは不思議ではありませんか？



このアップデートの最大の失望は、スタートボタンです。 いいえ、もちろん彼女は登場し、Windowsロゴがあってもすべて順調です。ただし、前のメニューは返されていません。 このボタンをクリックすると、同じタイルメニューが表示されます。 [スタート]ボタンを返すプログラムが新しい人気を博したのも不思議ではありませんか？ （例： pokki.com ）







現在、Windows 8.2は謎であり、その周辺には多くのうわさがあります：最初に彼らが出てくると言い、他の人はMicrosoftがWindows 8.1へのアップデートをリリースし、すぐにWindows 9をリリースすると信じています。 Windows 8.2に関する入手可能な情報を調べてみましょう。





Windows 8.2-新機能



Windows 8.2の新機能は何ですか？ MicrosoftがWindows 8.1に1年を費やしたと判断すると、8.2ははるかに少ないので、新しいものはそれほど多くないと言えるでしょう。 Windows 8.1で発生したように、Windows 8.2が更新されたカーネルを受け取る可能性は低いなど、根本的に新しいものはないと思います。Windows8は6.2カーネルを誇りますが、バージョン6.3に達しました。





コード名といえば。 特定のWindows「しきい値」の準備に関する情報があります。 これがWindows 8.2自体であるかどうかはまだ不明ですが、この更新プログラムはすぐにリリースされるはずです。 問題は、これがWindows 8.1の更新である可能性があり、その変更がそれほど劇的ではないことです。



前述のように、Windows 8.2では、非常に多くの人に愛されてきた[スタート]メニューが復活しました。 ボタンだけでなく、メニュー自体も表示されます。 タイルは、決して、そのようなコンピューターの古い[スタート]メニューの代替になり得ます。 いいえ、便利ですが、実際にはショートカットウィジェットを備えた独立したデスクトップです。 どのアプリケーションも起動できないデスクトップ-タイル張りのアプリケーションは全画面で開き、クラシックなアプリケーションはデスクトップと呼ばれます。



実際、マウスを搭載した通常のPCのスタート画面には利点はありません。 はい、それはタイルのいずれかに入るための単なるポインタですが、通常のよりコンパクトで邪魔にならない[スタート]メニューでは、これも実行できます。 同時に、メニューは開いているウィンドウにわずかに重なっており、画面全体を占めることはないため、予期せず開いた理由を忘れることができます。 一般に、ここでは何らかの種類の妥協が必要です。これは、「タイルとクラシック」の間のより有能な接続です。 おそらくWindows 8.2では、その開発者がこの決定を下すでしょう。



[スタート]ボタンと同じ名前のメニューに関連する別の変更は、システムの再起動とシャットダウンへの便利なアクセスです。 タブレットやスマートフォンでは、こうした種類のものはそれほど頻繁に使用しませんが、コンピューターでは「この野郎を蹴ってうまく動作させる」必要があります。 しかし、何らかの理由で、Microsoftは従来のPCを「提示」し、一定の稼働時間でこの「経験」をすることにしました。



一言で言えば、コンセプトは好きではありませんでした。 その結果、Windows 8に関する一般的な質問は「コンピューターの電源を切る方法」でした。 Windows 8.1は[スタート]ボタンを返し、このボタンでコンテキストメニューが返され、この非常に[スタート]でマウスの右ボタンで呼び出されました。 メニューはアイテムを表示し、オフにして再起動します。



「カムバック」のもう1つの候補は、Explorerのライブラリです。 この共通名の下で、Microsoftはフォルダーリンクをドキュメント、音楽、写真、ビデオと組み合わせます。 Windows 8エクスプローラーにはWindows 7のようなものがありましたが、8.1ではもうありませんでした。 一般的に、「基本に戻る」ことはここで起こります。







タイル化されたアプリケーションに関する多くの質問もあります。 Microsoftが「裏口から入る」ことを決定し、通常のコンピューターにタッチインターフェイスを実装しようとした理由は完全には明らかではありません。 数年前、彼女は反対のことをすることができなくなりました-「デスクトップ」OSとアプリケーションの原則はPDAに根を下ろさず、後にコミュニケーター/スマートフォンにも根づきませんでした。 それに基づいたWindows CEとWindows Mobileは失敗しました-消費者は、主に指の制御用に設計された便利なインターフェイスを備えたiPhoneを好みました。



では、なぜ同じアプローチが、鏡像のように機能するのでしょうか？ Apple自体はすべての問題を解決するわけではなく、古い習慣を壊さないように、また新しい習慣を開発するために、OS Xに要素を注意深く慎重に追加しています。



しかし、このような解決策は、過去15〜17年間、このような操作を別の方法で実行することに慣れていれば、直感的と言えるでしょうか。 要するに、Windows 8.2はこの問題を解決します。 途中で、この問題は他の「新しい」Windows 8.1設定でも解決されます。デスクトップへの即時ダウンロード、チャームパネルの無効化などです。 そして、すべてはそこにあるように見えますが、今のところ隠されているように見えます。



実際、Microsoftは現在、マルチウィンドウモードと、フルスクリーンモードで動作する多数のタイルアプリケーションを備えたクラシックデスクトップを提供しています。 実際、それぞれに個別のデスクトップがあります。 したがって、「明日のために100文字のテキストと2つの予測アイコンを表示する必要があるときに、27インチ画面全体に天気予報アプリケーションが必要なのはなぜですか」などの自然な質問が発生し始めます。



ほとんどの場合、これらは非常に顕著な重要な変更のすべてです。 しかし、他に改善できることは何ですか？ たとえば、Internet Explorer。 クラシックバージョンのバージョン10および11は、9番目のバージョンと変わりません。 最初のケースでは、スクロールバーが置き換えられ、2番目のケースでは、大幅に再設計された開発者ツールが追加されました。 しかし、インターフェイスでは何も「ねじれ」ませんでした。 彼は、一般に、まったく悪いことではありませんが、これは、より良いことをすることが不可能であることを意味しません。 そのため、Windows 8.2の新しい改良された顔を持つInternet Explorer 12を待っています。







そして最後が残っています。 多くの人は、すべてのMicrosoftシステムの統一について聞いています。 しかし、それにもかかわらず、これはWindows 9またはWindows 10のリリースでも発生する可能性が高いでしょう。しかし、アプリケーションストアの統合はそれほど複雑ではありません。 そのため、Windows 8.2はおそらくWindows Phone 8.1を備えた単一のWindowsストアを提供します。 理論的には、後者は2014年初頭にそのようなバージョンに成長します。



発売日



Windows Blue、そしてWindows 8.1は、2013年の初めに見ることができました-そして最初のビルドがリークしました。 あなたは、システムが春のどこかに十分にすぐにリリースされると思うでしょう。 しかし、10月を待たなければなりませんでした。 一方、2015年の初めには、Windows 8.2の2〜3か月後にリリースされる可能性が低いWindows 9のリリースを約束します。



一般に、Windows 8.2のリリース日は、「9」の少なくとも6か月前に行う必要があります。 さらに、予備情報は、より楽観的な予測を提供します-2014年1月。 その場合、この更新プログラムはWindows 8.1ほど多くの基本的な変更を行いません。 さらに、リークしたテストアセンブリの出現を期待する価値はほとんどありません。
