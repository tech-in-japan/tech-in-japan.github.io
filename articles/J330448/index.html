太陽。 海。 砂。 最後に、1年半の最初の休暇。 この冬、何かが燃え尽きました。 まったく予想外。 待望の開発を求めて、私は自分自身に困難を感じました。通常の静的レイアウトからインタラクティブなプロトタイプ作成に切り替えました。 だから私はこのAxureで冬の間中突っついて半春で、それから燃え尽きてandでた！ 「新しいスキル、新しいレバーがありますが、それを適用する場所はありません...」 そのような考えで、私は理解しました-それは休暇に行く時間です。 早ければ早いほど良い。 そうでなければ、私のオフィスでは、新しい家具、新しいモニター、そして新しい「マウス」マニピュレーターが必要になります...壊れたものの代わりに、明らかに！



ちなみに、Figmaを使用する場合は 、 既製の設計システムに注意することをお勧めします 。 フリーランサーが1か月あたりの注文を増やすのに役立ち、プログラマーは自分で美しいアプリケーションを作成できます。また、チーム作業用の既製の設計システムを使用して、チームが「スプリント」スプリントを迅速にリードします。



また、深刻なプロジェクトがある場合、当社のチームは、ベストプラクティスに基づいて組織内に設計システムを展開し、Figmaを使用して特定のタスクに合わせて調整する準備ができています。 Web /デスクトップ、および任意のモバイル。 また、React / React Nativeにも精通しています。 Tへの書き込み： @kamushken




私は悔い改め、少し装飾しました。 設備や家具を破壊するつもりはありませんでした。 長い間忘れられていた欲求不満を突然感じました。 フラストレーションをデザインする！ 小さな組織の人を想像してみてください。彼は問題を迅速に解決し、管理者は優れたアカウントを持ち、例外として遠隔地に行くことができました。常に40％の負荷がかかり、彼は肌から出ません。 そのような従業員は、調和と幸福のために他に何を必要としますか？



小規模な組織では、原則として、最新のツールは必要ありません。 あなたは仕事を迅速かつ順調にしていますか？ 少なくとも、Windowsペイントでは、後で実行します。 そして、調和と幸福のために、あなたは前進し、専門的に開発し、デザイナーのトレンドとツールの波に沿って泳ぐ必要があります。 そして、明らかに、私は西海岸から来る波の頂上にいたいです。 Figmaで全力を使い、Framerでアニメートし、React（！）をマスターし始めたいかもしれません。 全世界は、その方法をすでに知っているようです。 そして、あなたはまだです！ そして、世界にはこの知識とスキルを適用する場所があります。 そして、5〜7人の組織では、これは特に必要ではないことがわかります。 Taskuはすぐにotfotoshopilに接続され、脂肪に付着し、急成長しました。 数日後、すでにテスト中。 1週間後の運用。 設計仕様を指定する必要さえありません。誰もが知っています。 近年解決しました。



一方、世界は前進しています。 そして今、マイクロイテレーションが勢いを増し、設計システムが統合され、設計者がブラウザーで直接1つのプロトタイプを作成しています。 そして、あなたは全員座って、次のモックアップでタスクをすばやく閉じます。 小さなオフィスではすべてが時計のように機能するため...



そして、西洋の情報デザインの背景が私の脳に滴り落ちています。 ドリブル、ベハン、アップラップなど。 トレンドガイドでレイアウトを作成する方法がわかりませんか？ フォントペアを収集する方法がわかりませんか？ または、青紫のグラデーションを使用しませんか？ それとも、スケッチでケシを持っていないのですか？ 彼がいなければあなたはもはやデザイナーではなく、明日はすでに市場の背後にいるからです。 デザイナーがどれだけ監視し、読み、研究しなければならないかという恐怖。明日は物事に出かけません。 候補者の要件は日々増加しています。 誰も競争をキャンセルしませんでした。 ヒンズー教徒は常にリンクインであなたに追加しようとしており、缶詰の豆の3つの銀行のために働くように準備ができているようです。 そして、あなたより悪い結果をもたらすことは決してありません。



西側のクライアントは、出力にpsd / jpgを必要としなくなりました。 彼はすでに「いいえ、いいえ、まあ、あなたはSketchのソースが必要です。」と言っています。 一度-そしてすぐに落ちました。 本当の話。 スウェーデン人は、ドリブルを介して、私にとって便利になりましたが、インタラクティブなプロトタイプを作成するための新しいスキルで「撃つ」ことはしませんでした。

わずか1〜2か月後に、トップデザインスタジオの上野のクリエーターによる記事を読みます。彼は率直に言って、「列車はなくなっています。ドリブルでの撮影日は数年前に過ぎました」と言います。



次に、私の心を必死に興奮させた非常にインタラクティブなプロトタイプについて説明します。 当然、現代のデザイナーのように、私はしばらく前にPhotoshopを捨てて、遅れないように高度なツールを探しに行きました。 そして今、私はAxureに完全に精通しています。 1つのプログラムに同じポピーを購入しないでください。 これには、JS関数、変数、アニメーション機能、CSSイベント、および「コーディングなしのコーディング」が少し含まれています。 はい、たくさんあります。 ここで、それは顔の開発段階です-今、あなたは写真でタスクを閉じるだけでなく、直接コードを与えることができます。 デザイナーは、デザインとスタイルの両方、そして実質的にレイアウトを取得します。 コードが曲がっているのは残念ですが、有効です。 Akshurでデザインを作成すると、ブラウザに結果がすぐに表示されます。 オブジェクトから影を落とします-すぐにCSSで表示され、DOM要素を考慮してそれらの間のインデントをすぐに計算します。 夢！ 額に手をたたいて、彼は考えました：「そして、なぜこのフライは以前私を刺さなかったのですか...」



このような移行に、私が非常に理解したい開発がありますか？ どちらの側から見るか：小さな組織内で、開発者は「間違いなく優れている！」 彼らは、マシンコードからすぐに数えたりインデントしたりするのが簡単であり、スタイルをコピーすることができ、スタイルガイドのピペットで登る必要がないと言います。 しかし、大企業では、開発者は「このごちゃごちゃしたものはかき集めている。 スケッチファイルを使用すると、簡単に元に戻すことができます。 または最悪の場合はpsd。」



だから、一方で、大企業。 彼らにとってますます難しくなっています。 統合の問題が発生する場合があります。 設計システムをサポートするインフラストラクチャがない場合、開始する意味はありません。

または、このような現代的な方法で組み合わせる必要のある製品はそれほど多くありません。 ちなみに、クールなデザインシステムを作成することは、ほとんど成果です。 独自のスタイル、独自のアニメーションロジック、ルールなど。 そして、ケースを確実に切り裂いてください。 「私たちを知る」ために！ そして、出力は写真ではなくコードであるため、Akshuraはそれを引き出したでしょう。



しかし、そのような欲求を持つことは一つのことです。 そして、実装経験を持つことはすでに完全に異なっています。 そのような操縦のための経験やスペースはありませんか？ 求められていないデザイン、架空のケース、見事なアニメーションなど、自分でインタラクティブに座ってください。これについては、明らかに燃え尽きています。 発明したコンセプトのプロトタイプを作成し、デザインコミュニティで好きなものを収集し、誰かがこの曲線の有効なコードをダウンロードします。 しかし、実際の問題は解決されません。 特に所有しています。 たとえば、デザイナーはこの新しい知識をどこに持っていきますか...ツールに正しい賭けをしたことをどのように理解しますか？



しかし、サードパーティもあります-これはフリーランスです。 あなたが職場に落ち着いているなら、種のようなすべてのタスクをクリックし、実際にボスと良好な状態で、誰も気にしないと仮定します。 特に、小規模な組織で100％をロードするタスクがそれほど多くない場合。 そしてどういうわけか、あなたの個人的なクライアントは彼の問題であなたのところに来ます。



私たちは小さな顧客を連れて行き、平均はスタジオに行きますが、大きな顧客は州のそのような問題を解決します。 あなたはすぐに顧客の問題の本質を掘り下げ、すでにあなたの優位性を感じています。 なんで？ すごい要素が提供されているため、写真の代わりに、顧客はすぐに「ほぼ生産」を示すことができます。 インタラクティブにも。 そして、彼はすぐにブラウザでそれを開きます。 彼はマウスでそれを回すことができます、彼はプロトタイプのボタン/リンクを直接クリックできます。 主なことは、「このようなデザイン開発は現在トレンドになっています！」と発表することです。すべての費用でポイントを獲得する必要があります。 そして、彼は静止画を表示しますが、スケッチでそれを行うPetyaに「ジャンプ」します。 また、バイオレットオレンジはグラデーションを使用することで有名です。つまり、「クーラー」です。 そして最も重要なこととして、Petyaは絶対に明確なソースコードを提供します。 はい、「Akshura」には十分な弱点があります。出力はpng、あいまいなhtml / css / jsコード、またはrpファイル形式のソースです。 後者は、もちろん、すべての潜在的な顧客のわずか2.37％で使用できます。 このオプションは安全に削除できます。



そのため、いくつかの利点が迫っているように見えます。クライアントに完全な「詰め込み」で高忠実度のプロトタイプを提供します-すべてを微調整、クリック、移動などします。 静的レイアウトよりも間違いなく効果的です。 スケッチはほぼ遅れています！ しかし、疑問が生じます。クライアントはこれをどうするべきでしょうか？ クライアントは大きくありません。スタッフの強さから、2〜3人の開発者がいます。最初に質問する方が良いでしょう-「ちょっとレブザ、怖いコードをビジネスに取り入れますか？」

ひどいのは、コンテンツを調べると、スパン内に余分なdivのひどい品揃えがあり、これに不可解なスタイルも付加されるためです。 Axureがすべてのスタイルを外部ファイルに追加することを考えると、これは特に奇妙です。 しかし、html'ok内に多くの不可解なcss-garbageを残すことは軽daされません。



実際、ほとんどすべての開発者が「ここに来て、それ以上はない」と言ったのは、おそらく少し致命的なことでしょう。 ただし、それらを整理する必要はありません。すべてのインデント、スタイル、色、フォントサイズが理想的です。 ここに来て-あなたを抱きしめて！ 確かに、彼らはどのようにしてコードを有効なコードに導いた-推測しかできない。



「まあまあ」のシナリオがあります-開発者は、ソースコードが必要であると言うでしょう。 よくそのように動作していました。 結局、私は彼らと自由に暮らしているので、適応しなければなりません。 Akshurのソースはそれらを風化させません。Photoshopを開いた後、それを開く方法は怖くて嫌です。



はい、これはすべてデスクトップインターフェースに関するものであり、Akshurのモバイルではすべてがそうです。 いいえ、もちろん、実際のレイアウトで8dpグリッドを拡張し、最大限の権限で開始することができます。 つい最近でも、svgのインポートは、春の更新で正しく（最終的には）到着しました。 次に、レイアウトを絞り込んで権限を小さくします。 出口での不必要なジェスチャーと混乱の束：すべての画面のいくつかのバージョンとhtml。ただし、これ以上与えるものはありません。 そして、モバイル開発者はこれで何ができますか？ ツールの製造元が万能薬を思い付かない場合、市場の一部を失うような激しい痛み。



インタラクティブに戻りましょう。 Akshurの基本的なアニメーションは、どんなにhowられても、実際に受け入れられます。 ほんの少しのコンドミニアムで、他のツールとは異なります。 しかし、実際に結果を適用する方法は、まだ完全に理解不能です。 私は繰り返します：小さな会社の中では、先験的に誰もそれを必要としません。 まあ、インターフェースのアニメーションのトレンドにはいくらかの賭けがありますが、「結局のところ、すべてが私たちにとってうまくいく」と開発者は答えます。 そして、インターフェースをアニメーション化することは非常にエキサイティングであり、必要な場所でさえあることが判明しました。 あなたの想像では、動的オブジェクトの相互作用のための独自のルールを作成し、プロトタイプを作成するときにこれらのルールに従います。 これによって混乱が生じなければ、将来の製品の全体的な印象は間違いなく悪くなることはありません。 また、アニメーションが製品の使いやすさをどのように改善するかを理解すると、二重に興味深いものになります。 そして今、あなたは拳であなたを誇らしげに胸に打ち、あなたはインタラクションデザイナーであると主張しています。 興味深いことに、多くのITリクルーターはそのような職業を知っていますか？ プロダクトデザイナー、モバイルデザイナー、クリエイティブデザイナーもいますが、これはあなたが誰であるかを示す完全なリストではありません。 これは西部では特に当てはまります。西部では、言葉と行為の両方に明確に自分を位置付ける必要があります。



そして、ここで主な問題が発生します。あなたはクールで、インタラクティブなインターフェースを作成する方法を知っています。 ブラウザで作品を開き、魅力的なインタラクションに没頭できます。 しかし、実際に何が起こるのでしょうか？ すべてのタスクにより多くの時間を費やすだけです。 Photoshopではすべてが簡単でした-約2回で完了です。 そして今、あなたはあなた自身のために仕事を考えました：インタラクティブなプロトタイプをやってください。 この言葉の美しい組み合わせがあなたの市場価値を高めているということは決してありません。 実際、現在の職場では、これは特に必要ありません。 純粋にあなたのイニシアチブ、よくやった、Timurovets！



フレーマーについて聞いたことがありますか？ そこは便利で、モバイル第一、そしてモダンです。そして、もしよければ、少し時間をとってください。 そうしないと、エフェクトやアニメーションを実行できません。 新しいデザイナーツールが市場に出回る頻度から判断すると、デザイナー向けのコーディングはすぐに必須になります。 興味深いことに、コードに夢中になったら、私はデザイナーになりますか？ 2つの椅子に座るのは難しいです。 設計またはコーディングが得意です。 私はすでに何人かの読者のinりを見ています。 デザイナーはhtml / css / jsを知っているべきだと言ってはいけません。 Akshurの開発者が最終的にコードのエクスポートをクリーンで有効なものにすると、コーダーは自分たちの将来について心配し始めなければならないと思います。



いずれにせよ、私はこのようなツールに関する音節を終わらせます：毎年（そして時間とともに、そしてより頻繁に）新しい設計ツールが起動され、それを維持するために移行する必要がある場合、私は不安を感じます。 問題の解決よりもトレーニングに多くの時間を費やす必要があります。



それとも、大企業の波ですか？ そこで、すべてのこの新しい知識は需要があり、適用され、適切な報酬があります。 私の意識的なキャリアのほぼ全体において、私はデザイナーでした。 彼は必要だと考えたクライアントの問題を解決し、それから議論した。 複数のクールなグラフィックスペシャリストが力を合わせて1つの問題を解決するのはどうですか？ 誰が知っている 誰が試しましたか？ 私はいつも理解していました：チームプレイはクールで効果的です！
