14 ReSharperのナビゲーションおよび編集機能

ツールは才能を高める手段です。 彼らがより良く、あなたがそれらをより良くすればするほど、あなたはより多くのことができます。

アンディハントとデイブトーマス「実用的なプログラマ。 見習いからマスターへの旅 ''



免責事項 ：これはカスタムメイドの記事または完全に無料の記事ではありません（JetBrains、私は何も示唆していませんJ）。 以下は、 Visual Studioの一般的な拡張機能の機能（およびスタジオ自体の同様の機能）のみです。私は個人的に日常の活動で使用しています。



道具は曲がった手にとって万能薬ではなく、それ自体では、よく知られた四本足のぼろきれのような仕事を引き裂くことができる普通の専門家から普通の地獄のような悪魔を作ることはありません。 それらはあなた自身のタスクを解決するのではなく、必要な使用ディレクティブの追加、フォーマットの問題、コードナビゲーションなどのありふれた事柄について考えることなく、いくつかの日常的なことを肩から外して、主な事柄に集中する機会を与えるのに役立ちます。 多くのツール（特にReSharper）は、コード内のエラーの検出、コーディング標準の遵守、単体テストの実行など、その他の重要度の低いタスクの解決に役立ちます。 そして、これらの機能はすべてメガフィットそのものですが、今日は主にナビゲーションと検索機能に焦点を当てます。



説明の形式は次のとおりです。機能名、括弧内はVisual Studioスキーマ、次にIDEAスキーマのホットキーです。



例：タイプへ移動（Ctrl + T、Ctrl + N）。





1.シンボル補完のインポート（Shift + Alt +スペース、Ctrl + Alt +スペース）


誰もがファンタスティックなIntelliSense ReSharperを知っており、10番目のスタジオでは非常に優れています。 しかし、IntelliSenseの特性は、usingディレクティブを使用して既にインポートされた文字でのみ機能することです。 しかし、型名（またはその一部）は知っているが、それがどの名前空間にあるのかがよくわからない状況があります。 この場合、完全なタイプ名でドライブし、Alt + Enterを使用して目的のネームスペースを追加できます（セクション9. 自動 - ディレクティブ を使用した インポート を参照）。ただし、名前の一部しかわからない場合、この方法は機能しません。 とにかく、すべての文字を踏みつける必要がありますが、これは必ずしも便利ではありません。



標準のコード補完メカニズムとは異なり、 Import Symbol Completionは、「可視」タイプ間だけでなく、現在のアセンブリで使用可能なすべてのタイプ間でも入力語の対応を検索します。







2 ..ファイルとタイプの検索


少なくともいくつかのナビゲーション機能が、 Navigate Toコマンドの形式でVisual Studio 2010の構成に追加されました（Edit.NavigateToコマンド、詳細については、Scott GattreeのメモVisual Studioでの検索とナビゲーションのヒント/コツを参照してください）。 ただし、VS2010のこのコマンドの標準バージョンとは異なり、ReSharperには、タイプ検索、ファイル検索、および文字検索という1つのコマンドではなく、いくつかのコマンドがあります。 一方で、これにはより多くのホットキーを覚えておく必要がありますが、目的のクラスまたはファイルをすばやく見つけることができます。



すべての検索機能はCamelCaseの補完をサポートしているため、 tmcを入力すると、 T ext M essage C omand、 T est M ono C ompilerなどが見つかります。



• タイプに 移動 （Ctrl + T、Ctrl + N）-タイプ名で検索（クラス、インターフェイス、列挙、構造を検索）



• ファイルに移動 （Ctrl + Shift + T、Ctrl + Shift + N）-ファイル名で検索



• シンボルに 移動 （Shift + Alt + T、Ctrl + Shift + Alt + N）-VS2010からのナビゲートの類似、タイプ名またはこれらのタイプ内のメンバーによる検索。



ちなみに、3種類すべての検索（より正確には4つ、 Go to File Memberもあるため ）の間で、現在の検索を閉じずに切り替えることができます（つまり、突然タイプを探し始めた場合に検索バーを再度入力する必要はありませんが、 -soファイルを検索します）。



注

実際、VS2010からのナビゲートは、 Go to Symbol ReSharperチームに最も近いですが、 ナビゲート先 もファイル名を検索します。これにより、アーキテクチャ図やその他のファイルを含むファイルの形で予期せず大量の「ノイズ」が生じることがあります。明らかに興味がない。

3.ファイルメンバーに移動（Alt + \、Ctrl + F12）


Visual Studioの一部として、特定のファイル内のメンバー名に移動することができます（この機能はナビゲーションバー、 Windowコマンド。MoveToNavigationBarと呼ばれます）が、同様のReSharper機能の方が少し便利です。 まず、 Go to File Memberは、タイプまたはファイル検索と同じフィルタリングおよび検索機能をサポートしています（つまり、CamelCase検索をサポートしています）。 次に、[ファイルメンバーに移動]は、宣言の順序ではなく、わずかに異なる順序（コンストラクター、フィールド、プロパティ、メソッド）でメンバーのリストを表示します。これにより、コンストラクターへの移行が可能な限り便利になります。







4.ナビゲート（Alt +“ '”、Ctrl + Shift + G）


ReSharperは、宣言と実装への移行、基本クラスと子孫への移行など、さまざまなナビゲーション機能をサポートしています。 各チームには対応するホットキーがありますが、それらはdofigよりも少し大きいため、どうにかしてそれらをすべて覚えたくないでしょう。 まず、すべてのナビゲーション機能にアクセスできる1つのコマンドを覚えておいてください。これは、既に特定のアクションを選択できるNavigate Toというナビゲーションコマンドです。 （このコマンドをVisual Studioからのナビゲートと混同しないでください。VisualStudioからは、タイプとファイルの検索の説明で言及しました）。







ほとんどのナビゲーションコマンドはよく知られていますが、1つを個別に選びたいと思います。 これは実装に移動（Ctrl + F112、Ctrl + Shift + Alt + B）です。



現在、インターフェースに基づいたさまざまな疎結合アーキテクチャが非常に人気があります。 その結果、インターフェイス型変数を介してメソッドが呼び出されると、実際に何が呼び出されるかが完全に明確になりません。 この場合、メソッドの宣言ではなく「実装への移行」が非常に便利です。







5.「最新のファイル」をナビゲートする


1. [ VS ] 後方への移動 （ 表示 。NavigateBackward 、 Ctrl +-）、 前方への移動 （ 表示 。NavigateForward 、 Ctrl + Shift +-）



2. [ R ＃] Recent Files （ Ctrl +“、”、 Ctrl + E ） -最近表示されたファイルのリスト



3. [ R ＃] 最近の 編集 （ Ctrl + Shift +“、”、 Ctrl + Shift + Alt + Backspace ） -最近変更されたファイルのリスト



4. [R＃]ソリューションエクスプローラーでファイルを検索（Shift + Alt + L、Shift + Alt + L） -ソリューションエクスプローラーで現在のファイルの場所を表示



ソリューションを徹底的にナビゲートすると、多くの場合、以前のファイルに移動する必要があります。 もちろん、開いているタブに切り替えることもできますが、これが常に最も便利なナビゲーション方法とは限りません。



1つ以上のステップに戻る（または進む）ための最も簡単な方法は、標準のVisual Studioコマンド-後方/前方に移動することです。 ただし、一度に複数のレベルに戻る必要がある場合は、（ 最近の ファイル）リストを検索する方が簡単です（他の機能と同様に、最近のファイルは、キャメルケース検索など、目的のファイルを検索およびフィルターする機能をサポートしています）。







それとは別に、ソリューションエクスプローラーで現在のファイルの位置を表示する機能に注意したい（ソリューションエクスプローラーでファイル を 検索 ）。



6.コードを上下左右に移動します（Ctrl + Alt + Shift +上/下/左/右）


このコマンドを使用すると、クラスメンバーを構造化したり、メソッド内でコードのブロックを移動したりできます（たとえば、if / for etcブロック内でコードを追加または削除できます）。







7.次のメンバー/タグに移動（Alt +下）、前のメンバー/タグに移動（Alt +上）


これは、Ctrl + Up / Downを使用したWordの段落ナビゲーションのようなものです。この場合のみ、「段落」がクラスのメンバー（メソッド、プロパティ、イベント、フィールド）です。



8. コードの 生成 （ Alt + Insert ）


ReSharperは、コードを生成するためのさまざまな機能をサポートしています。そのほとんどはAlt + Insertで利用できます。 ソリューションエクスプローラーから、クラス/構造/列挙を生成するためのテンプレートを使用できます（独自のテンプレートを追加および変更できます）。







また、クラスから、コンストラクター、プロパティ、メソッドのオーバーロードなどを生成する別のコマンドを使用できます。







少なくとも、コンストラクター、オーバーロードメソッド、および比較メソッドの生成に注意を払う必要があります（比較メソッドは別のトピックであり、それらを正しく記述するのは少なくとも面倒です）。



9.ディレクティブを使用した自動インポート（Alt + Enter）


私が常に使用しているもう1つの便利な機能は、 usingディレクティブの自動インポートです。







上の図は、ReSharperとVisual Studioの両方の組み込みツールの機能を示しています（Visual Studioからの修正を含むコンテキストメニューは「スマートタグ」（View.ShowSmartTag）と呼ばれ、Alt + Shift + F10またはCtrl + "。"で使用できます）。 異なるホットキーを使用することに加えて、2つの実装の違いは、ReSharperは追加されたアセンブリだけでなく、ソリューションの他のすべてのアセンブリでもこれらのタイプを検索することです。これは、単体テストでプロジェクトで作業する場合など、非常に便利です一部のプロジェクトがまだ追加されていない場合。







10. モジュールに 依存 する コード を見つける


特定のアセンブリから具体的に何が使用されているのかを理解すると役立つ場合があります。 これらの目的のために、ReSharperは6日目に特定のアセンブリに依存するコードを検索する機能を追加しました。 ソリューションエクスプローラー->参照-> System.Xml.Linq->モジュールに依存するコードの検索。







11.行または選択範囲を複製する（Ctrl + D）


コピーと貼り付けは悪意があることは明らかですが、良い手の中にあります...一般に、場合によっては、フラグメントまたは現在の行を複製することが有用です。



ところで、ここで別の同様の機能（今回はReSharperではなく、スタジオの機会です）に言及する価値があります。これはよく使用します。選択せずに行全体を削除する- 現在の 行を カット または 削除 （Edit.LineCut、Edit .LineDelete、Ctrl + L、Shift + Del）。



12.パラメーター情報（Ctrl + Shift + Space、Ctrl + P）


メソッド呼び出しが既にコードに完全に記述されている場合、メソッドパラメーター（または追加のメソッドオーバーロード）に関する情報を調べる必要があることが非常に多くあります。 通常、この問題は、メソッドの開き括弧を削除してから再度駆動することで解決されます。 その結果、IntelliSenseが呼び出され、メソッドまたは目的のパラメーターに関する情報が表示されます。



しかし、もっと簡単なオプションがあります-Parameters Infoコマンドを使用します。 この関数は、Visual Studio（キーの組み合わせは同じ：Ctrl + Shift + Space）とReSharperの両方で使用できますが、唯一の違いはどちらのIntelliSenseが呼び出されるかです。







13. クイック ドキュメンテーション


ただし、Visual StudioとReSharperの両方で使用できる、使いやすさのわずかな違いがある別の機能。



[VS]クイック情報（Edit.QuickInfo、Ctrl + K、I）





[R＃]クイックドキュメント（Ctrl + Shift + F1、Ctrl + Q）





残念ながら、あなたが突然契約プログラミングが好きなら、これらのツールはどれも通常の前提条件と事後条件を示さず、むしろそれらはまったく表示しません。 この情報をVisual Studioからツールヒントに追加するCode Contracts Editor Extensionsというスタジオへの拡張機能がありますが、この拡張機能の安定性は非常に低くなっています。



14. ReSharper 名前空間 プロバイダー


標準的な命名のイディオムの1つは、プロジェクトの論理構造（名前空間）と物理構造（フォルダー）の調整です。 つまり、 パブリッククラスの名前はファイルの名前と一致する必要があり、 名前空間はファイルが配置されているフォルダーの構造と一致する必要があります。



ほとんどの場合、そのようなスキームに従うことは合理的であり、それはまさにReSharperアナライザーが考えていることであり、これに従わない場合、彼は文句を言い、再び考え、正しい名前空間を使用することを提案します。







ただし、常にこのような原則に従うことは困難ですが、ReSharperのこの機能を無効にすることは望ましくありません。 このジレンマは非常に簡単に解決されます。フォルダープロパティに移動し、 Namespace ProviderをFalseに設定するだけです。







おわりに


ReSharperは長い間、膨大な数の機能を備えたまともなモンスターであり、私が常に使用している機能の一部のみをここに示します。 これは、上記のすべての機能がすべての人にとって便利であることや、他の便利な機能がないことを意味するものではありません。 そのため、たとえば、リファクタリング、簡単な修正、コーディングスタイルの提供手段、単体テストの実行、テンプレートなどについては、意図的に触れませんでした。 Visual Studioに関しては、これは非常に多くの機能を備えた大きなモンスターです。 それらのいくつかは非常に便利で、いくつかはかなり平凡です（JetBrainsや同様の人も何らかの形で生計を立てる必要があります）。 スタジオ自体の便利な機能の中で、垂直編集の可能性に注意する価値があります（これについてはここでもう少し書きました）、コードの任意のセクションを隠す、強力な検索および置換ツールなどがあります。 しかし、これは別の投稿のトピックです。
