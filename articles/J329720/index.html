メールサービスのアクセス制限のバイパスについて

この投稿は、主にウクライナのユーザーを対象としています。



Yandex、Mail.ruなどを使用する場合、メールの送受信が要求されます。



そのため、 ウクライナですでによく知られている法律の採用により、ドメインをYandexに委任し、メールホスティングを使用するすべての企業で問題が発生しました。



指定されたリソースは、制裁リストの番号275にあることに注意してください 。



ウクライナの類似物（ukr.net、meta.ua、bigmir.netなど）は同様のサービスを提供しないため、問題はかなり深刻であることを認めなければなりません。



はい、グーグルがあります-しかし、アメリカの巨人は多くを望んでいますが、各ボックスに5ドル、そして30 GBの制限を回避したい場合は、金額は10ドルになります。 Yandexのこれらのパンはすべて無料でした。



頭に浮かぶ最初の解決策がVPNであることは明らかです。ただし、メールのみの場合、接続速度の必然的な低下に伴うVPNの追加費用は受け入れられません。



問題をかなり高速で解決する最適な方法が見つかりました。 彼について-以下。



ネットワークにはさまざまな無料のVPNクライアントがあります。 それらにはそれぞれ制限があり、支払われると削除されます。 ここからダウンロードできるPsiphonクライアントを見ていきます。



このクライアントの機能にはいくつかの動作モードがあり、そのうちの1つにはSOCKSプロキシの作成が含まれます。SOCKSプロキシを使用すると、メールプログラムからトラフィックをリダイレクトできますが、ブラウザのトラフィックには影響しません。



残念ながら、これは一部のサードパーティクライアント（Thunderbird、TheBat）でのみ実装できます。 組み込みのWindowsクライアント-Live MailおよびOutlookでは、システムに関する優れた接続設定を使用できません。したがって、ブラウザーとは別にトラフィックをリダイレクトすることはできません。



それでは、何をする必要がありますか？



  1. Psiphonプログラムをダウンロードしますが、実行しないでください。
  2. OSで組み込みクライアントを使用する場合


    次のデータを実行してレジストリにインポートします。

    Windows Registry Editor Version 5.00



    [HKEY_CURRENT_USER\Software\Psiphon3]

    "SkipBrowser"=dword:00000001

    "SkipProxySettings"=dword:00000000

    "SkipAutoConnect"=dword:00000000

    "SplitTunnel"=dword:00000000

    "DisableTimeouts"=dword:00000001

    "Transport"="VPN"

    "LocalHTTPProxyPort"=dword:00000000

    "LocalSOCKSProxyPort"=dword:00003039

    "SSHParentProxyHostname"=""

    "SSHParentProxyPort"=dword:00000000

    "SSHParentProxySkip"=dword:00000000

    "SystrayMinimize"=dword:00000001





    仕事に最適なサイフォン設定を含めようとしましたが、すべてのトラフィックがこのVPNを通過することに注意してください。 速度が低下する場合があります。 たとえば、ネットワークを「直接」操作する必要がある場合は、ストリーミングビデオを視聴したり、急流をダウンロードしたりする場合、クライアントのオンとオフを切り替える必要があります。


    ThunderbirdまたはTheBatを使用している場合
    次のデータを実行してレジストリにインポートします。

    Windows Registry Editor Version 5.00



    [HKEY_CURRENT_USER\SOFTWARE\Psiphon3]

    "SkipBrowser"=dword:00000001

    "SkipProxySettings"=dword:00000001

    "SkipAutoConnect"=dword:00000000

    "SplitTunnel"=dword:00000000

    "DisableTimeouts"=dword:00000001

    "Transport"="CoreTransport"

    "LocalHTTPProxyPort"=dword:00000000

    "LocalSOCKSProxyPort"=dword:00003039

    "SSHParentProxyHostname"=""

    "SSHParentProxyPort"=dword:00000000

    "SSHParentProxySkip"=dword:00000000

    "SystrayMinimize"=dword:00000001





    この場合、システムはポート12345でローカルSOCKSプロキシを作成します。したがって、メールクライアントを接続するためだけに残り、残りのトラフィックは直接送信されます。
  3. Psiphonを起動します。 デスクトップ上のプログラムにショートカットをドラッグすることも、ファイル自体をデスクトップに残すこともできます。
  4. OSで組み込みクライアントを使用する場合


    メールクライアントを起動して作業するだけです。 メールで作業する必要がない場合は、クライアントを閉じてPsiphonで接続を切断します。 まあその逆も。


    ThunderbirdまたはTheBatを使用している場合
    メールクライアントの設定で、ポート12345のローカルSOCKSプロキシを介した接続を指定します。Thunderbirdの場合、次のようになります。



    SOCKS4またはSOCKS5タイプは原則として基本的なものではありませんが、一部のTheBatユーザーから私に通知されたように、このプログラムではSOCKS5を許可なく指定する必要があります。

    その後、メールは正常に機能します。 原則として、ブロックされたリソースにすべてのメールがある場合、Psiphonを自動ロードに追加することもできます。





「主権者の意志を果たす熱意」がなければ、原則としてこれは終わったかもしれない。 事実は、「インターネットサービスプロバイダーがリソースへのネットワークアクセスのユーザーにサービスを提供することを禁止する」という条項（「リソース/サービスへのインターネットアクセスにアクセスするためのフェンスインターネットサービスプロバイダー」）が一部のプロバイダーによってあまりにも積極的に採用されたということです。 合計で、Yandexから複数のウクライナのドメインにレターを送信する場合、受信サーバーがYandex smtpからの接続を拒否するため、メールは配信されません。 ウクライナのsmtpがメールボックスへの接続を拒否し、ホスティングがYandexで行われていることを発見した場合（残念ながら、まれなケース）がありました。 最初のケースがパブリックsmtp- relaysを使用して戦うことができる場合（たとえば、 SMTP2GOを強くお勧めします ）、残念ながら2番目の問題は別のサイトに移動することなく克服できません。 同時に、私は強調します。このようなブロックは法律の本文にはまったく示されていませんが、単に「過度の熱心」またはウクライナのシステム管理者の非識字の結果です。
