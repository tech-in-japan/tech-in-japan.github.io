XKB：最愛の人のためにキーを再構成する

ある晴れた日は、Shiftキーを押して、 `の代わりに記号〜が表示されるようにします。

CapsLockキーを使用している間、Escに連絡するのはうんざりです。

ブラシを下に動かし、弱い小指でCtrl / Cmd / Winを押すのはうんざりです。

PgUp、PgDn、Home、Endに登るのにうんざりして、ページをスクロールダウンするだけで、毎回手を動かす。

タイプミスをするたびに退屈して、Backspaceボタンを探してキーボードを怒って見てください。







今日は、これらの苦痛を永遠に取り除く方法を学びます。



なぜこれだけなのですか？



少ないキーを押します。 よく押すキーに指を近づけないようにするため。 手の疲れを軽減します。 どこを覚えやすくするため。 結局、より速く印刷するために。



印刷用の多くのレイアウト（Dvorak、Colemakなど）は、キーのレイアウトを大幅に改善し、ストレッチを少なくする必要がある場所の近くに配置します。 しかし同時に、彼らは修飾子、スペシャルを完全に忘れます。 文字とコマンドキー（Enter、Backspace、Tab、Esc、Delete）。 そして、現状をどのように改善するかを考えます。



この記事では、最も単純な変更から、より面白く深いチューニングに移行します。 そして、良い映画の最後のように、私は読者に次に何をすべきかを考えさせ、すべてのために既製のレシピを与えることはしません。 最終的には、理想的で普遍的なソリューションはなく、キーボードをレイアウトする仲間もいません。



クリップルの最初のステップ



標準レイアウトは自分で変更する必要があります。 これを行うには多くの方法があります。 ここでは誰もエキゾチックなibus、Fcitx、SCIM、UIMを使用するとは思わないので、誰もが標準のXキーボード拡張機能であるXKBを使用すると仮定します。 今後のWaylandデスクトップでは、XKBがキーボードとXorgの責任を負う可能性が高いことに注意してください。 突然XWaylandで作業している向こう見ずな人がいる場合、私が説明したことがあなたに役立つなら、あなたの印象を共有してください。 Canonicalの代替方法であるWayland Mirについては何も言えませんが、キーボードインタラクションサブシステムを書き換えることはほとんどありません。



最初に頭に浮かぶのは、何らかのグラフィカルユーティリティを使用することです。 正直に言うと、この手順をスキップして、すぐにxmodmapを使用しました。 以下に、このアプローチの何が問題で、どのように正しく行うかを説明します。



Xmodmapは、たとえば、レイヤーの交換を許可します。 そして：、これは、行末に必須のセミコロンをもはや必要としない現代言語でプログラムする人にとって特に便利です。 JavaScriptでも、これは必要ありません。 そして、この簡単なコマンドは役立ちます： 

$ xmodmap -e "keycode 47 = colon semicolon Cyrillic_zhe Cyrillic_ZHE"


.xsessionまたは.xinitrcから実行する必要があります。 引用符で囲まれた最後の2つの引数は、交換中に文字wとfが失われないようにするのに役立ちます。



ただし、たとえば、これは〜と`を入れ替えます： 

 $ xmodmap -e "keycode 49 = asciitilde grave Cyrillic_io Cyrillic_IO"


レイアウト切り替え



Altを頻繁に使用する必要がありますか？ 突然ではないが、GimpやLibreOfficeなどのメニューが豊富なインターフェイスでのみこれを行う必要がある場合は、AltGrとして知られる右Altスイッチとして使用できます。 

 $ setxkbmap -layout us,ru -option grp:toggle


EscとCapsLock



多くの場合、Vimを使用するか、コンソールでviの組み合わせを使用し、フォーラムで少し誓う人は、CapsLockキーでEscapeを使用することを好みます。 

  $ xmodmap -e "clear lock" $ xmodmap -e "keycode 0x42 = Escape"


同じことが少し簡単にできます： 

  setxkbmap -option caps:escape


または、次のように、高音での対応の場合にCapsLockを古いエスケープの場所に保持します。 

  setxkbmap -option caps:swapescape


これらの不思議なパラメーターがどこから来たのかについては、少し後で確認します。



花



前にやったことが正しくなかったと言ったら、おそらく驚くでしょう。 setxkbmapはまだ何もありませんが、xmodmapはXKBの前に登場し、互換性のためにのみ存在します。 それだけでなく、彼はキーの半分について何も知らない、彼の仕事の構文と原理は不可解な以上であり、キーコードとキーシムを翻訳し、場合によっては元に戻します。 これを理解することは意味がありません。ある時点で死に、すべてのトリックを別のユーティリティの助けを借りて繰り返す必要があり、あなたは動揺し、すべてにもかかわらず、2本の人差し指で再び印刷されます。



そのため、すべてが必要なキーがたくさんあり、ホームライン（ASDF JKL;）にある指が全員に届かないことを理解し始めるまで、すばらしい瞬間まですべて機能します。

たとえば、Mod4修飾子であるWinという指定のキーをよく使用します。 キーボード上にあるのはあまり便利ではありません。つまり、左AltとCtrlの間です。



正しい方法



これらすべてをxkbcompで行うのは正しいことです。 ユーティリティの名前は、人間のように、その機能についてほとんど情報を提供しません。 伝えられるところでは、バイナリ形式のキーボードレイアウトのテキスト記述を翻訳します。 しかし実際には、それらもロードします。 助けを借りてのみ、既に行ったことを行う方法を理解してみましょう。



インターネット上の多くのユーザーは、/ usr / share / X11 / xkb / symbolsにあるファイル、つまりus、pc、ruを変更することを推奨しています。 これをしないでください。 これらは、システム全体に一度に作用するファイルです。 他のユーザー（突然持っている場合）、およびそこに何かをいじる場合、ユーザー名とパスワードを入力することさえ難しくなります。 変更したファイルをそこに置いてはならず、evdev.xmlに何かを書く必要はありません。



重要：突然何か間違ったことをした場合は、Xを再起動すると役立ちます。 より簡単な方法があります。パラメータなしでsetxkbmapコマンドを実行すると、ほとんどすべての設定がデフォルトのレイアウトにリセットされます。



まず、現在のレイアウト設定を確認する必要があります。すべての操作の前にこれを行う必要があります。 

 $ setxkbmap -layout us,ru -print


出力は次のようになります。 

 xkb_keymap { xkb_keycodes { include "evdev+aliases(qwerty)" }; xkb_types { include "complete" }; xkb_compat { include "complete" }; xkb_symbols { include "pc+us+ru:2+inet(evdev)" }; xkb_geometry { include "pc(pc104)" }; };


これらすべてをファイルに書き込み、XDGの指示に従って〜/ .config / xkb / myに配置します。



このファイルでは、xkb_symbols行に最も関心があり、残りは変更されません。 中括弧を展開し、この行を次のように変更します。 

 xkb_keymap { xkb_keycodes { include "evdev+aliases(qwerty)" }; xkb_types { include "complete" }; xkb_compat { include "complete" }; xkb_geometry { include "pc(pc104)" }; xkb_symbols "my" { include "pc+us+ru:2+inet(evdev)" key <AC10> { [ colon, semicolon ] }; key <TLDE> { [ asciitilde, grave ] }; }; };


さて、あなたの指を交差させて、この構成をロードしてください： 

 $ xkbcomp $HOME/.config/xkb/my $DISPLAY


やってみます。 文字writtenを書いたキーを押すと、記号が印刷されます：ではなく、Åを押すと、チルダ〜が印刷され、Shiftを押すと、すべてが逆になり、正しい軌道に乗っています。



Cyrillic_zhe Cyrillic_ZHEをすぐに気にする必要はありません。すべてがそのように機能することに注意してください。



続けましょう。 私たちは何をしましたか？ パラメータを指定してsetxkbmap -optionを実行しました。 彼が何を変えるか見てみましょう。 

 $ setxkbmap -print | grep symbols xkb_symbols { include "pc+us+ru:2+inet(evdev)" }; $ setxkbmap -option caps:escape $ setxkbmap -print | grep symbols xkb_symbols { include "pc+us+ru:2+inet(evdev)+capslock(escape)" };


さて、今ではいくつかのオプションを含めることでレイアウトが変化していることが明らかになったようです。

myファイルで、既存のインクルードの直後に、xkb_symbolsセクションに次の行を追加します。 

  include "capslock(escape)"


この構成ファイルの他の場所のセミコロンとは異なり、インクルード後のセミコロンは不要であることに注意してください。

よく見ると、これら2つの記録方法は同じであることがわかります。

これはどういう意味ですか？ レイアウトを読み込み、qwerty（xkb_keycodes）、104個のキー（xkb_geometry）、文字を再定義するとします。 同時に、シンボルセクションに、いくつかの構成ファイルpc、us、ruを含めます。 彼らはどこに横たわっていますか、彼らを見ることができますか？ 当然、これらはすべて/ usr / share / X11 / xkb / symbolsにあります。 特に私たちがすでにやったよりももっと野心的なことをするつもりなら、それらを見るのは便利です。

さて、CapsLockとEscapeを交換する必要があると判断するファイルはどこにありますか？ はい、あります。 次のようになります。 

 partial hidden modifier_keys xkb_symbols "swapescape" { key <CAPS> { [ Escape ] }; key <ESC> { [ Caps_Lock ] }; }; partial hidden modifier_keys xkb_symbols "escape" { key <CAPS> { [ Escape ] }; };


部分とは何ですか？ これは、別のレイアウトに含めることができるレイアウトの一部です。 一部の構成ファイルでは、寛大な著者がコメントを追加しました。 しかし、まったくそうではありません。



パーシャルを作成して、オンにしないのはなぜですか？ 残念なことに、私は説明に従ってこれをやろうとしたことを認める準備ができていますが、うまくいきませんでした。



理論的にそれを行う方法
レイアウトファイルでは、xkb_keymap全体を配置する必要はありません。xkb_symbolsセクションだけで十分です。 



  xkb_symbols "my" { include "pc+us+ru:2+inet(evdev)" key <AC10> { [ colon, semicolon ] }; key <TLDE> { [ asciitilde, grave ] }; };




ダウンロードは次のコマンドで実行する必要があります。 

 setxkbmap -I$HOME/.config/xkb my -print | xkbcomp - $DISPLAY


残念ながら、setxkbmapの出力は、1つの場合を除いて、どのパラメーターからも変更されません。 

 setxkbmap -I$HOME/.config/xkb -symbols my -print


しかし、残念ながら、この出力をxkbcompに転送する場合、エラーがスローされます。

ベリー類



CapsLockに逃げたくありません。 コントロールも必要です。

どうやって？ これはどのように可能ですか？ 魔法の時が来ました。



まず、CapsLockをクリックしたときに、これをxkb_symbolsセクションに追加してLCTLを取得します。 

  replace key <CAPS> { [ Control_L ] }; replace key <LCTL> { [ Caps_Lock ] };


2番目の行は、古い左Ctrlキーを押すという悪い習慣を取り除くために必要です。 これをわずかに和らげることができます： 

  replace key <LCTL> { [ VoidSymbol ] };


次に、 woodpeckerのeyeユーティリティxcapeが必要です。これにより、2番目のアクションを修飾子に割り当てることができます。 したがって、いくつかの英数字キーと組み合わせて押された場合、修飾子として機能し、単独で押されて離された場合、必要に応じて何らかの方法で動作します。 戦いに！ 

 $ xcape -t 1000 -e "Control_L=Escape"


xcapeはキツツキの目によって悪魔化されます。 また、興味深いパラメーター-t 1000に注目する価値があります。いいえ、彼はターミネーターを呼び出しません。左のCtrlを押したまま1000ミリ秒以上押し続けた場合、エスケープは機能しないと言います。 優柔不断のための素晴らしいオプション。



この方法の欠点の1つは、Escapeが押すことではなく、キーを放すことで機能することです。 この遅延は一部の人にとってはひどいものではありませんが、誰かにとっては重要です。 そして誰かが妥協します。



ボーナス



あらゆる種類の便利ですが、まれな文字については、 Compose Keyなどがあります。これは、画面に表示されるUnicode文字に対応する特定のシーケンスを押して放した後に押すことができます。 たとえば、一連のキーストローク「Compose oo」は°を表示します。 Compose cccpはを提供します 。 面白いですか 多くの記号があり、それらをキャプチャしやすいように組み合わせが選択されます。ほとんどの場合、押す順序は重要ではありません。たとえば、「Compose e =」と「Compose = e」は両方とも€を与えます。 標準的な組み合わせの完全なリストはここにあります。また、〜/ .XComposeファイルで独自の組み合わせを割り当てることもできます。

そして、正しいCtrlをComposeとして機能させるために、次の行をファイルに追加できます。 

  include "compose(rctrl)"


道で、道で



読者が実験したいように、設定全体を意図的に公開しません。 ここにいくつかのアイデアがあります。

-Fキーを押して、 HJKLを通常の矢印のように機能させます。

-左のShiftを固定する必要がなく、一度押して放すだけで十分であることを確認してから、たとえばキー5を押して、 ％記号を印刷します（ヒント：ラッチ）。

-1つのキーに2つの異なる修飾子を掛ける
