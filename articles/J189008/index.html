MikroTik-プロバイダーサポートなしの6in4またはIPv6

何らかの理由でIPv6テクノロジーを使用したかったのですが、プロバイダーがこのサービスを提供していません。 Googleは、プロバイダーのIPv4ネットワークを介してIPv6パケットを転送できる6in4メカニズムを提案しました。これは、 MikroTik RouterBoard 951g-2hndルーターでサポートされています 。











開始する



最初に必要なもの：

  • プロバイダーからの「ホワイト」永久IPv4アドレス（いわゆる専用IPアドレス）-「ホワイト」だが動的アドレスがある場合は、これを読む必要があります： Hurricane Electric IPv6 Tunnel-IPv4 Endpoint updater
  • 着信ICMPパケットはブロックされません（つまり、専用IPアドレスはインターネットからpingできます。一部のプロバイダーは着信ICMPパケットをブロックします）




トンネルブローカー





トンネルブローカーに登録します。 ハリケーンエレクトリックを使用しています



このアドレスにアクセスして登録します。 これは複雑なことではありません。 返信として、ユーザー名とパスワードが記載されたメールを受け取ります。 Webサイトでそれらを入力し、トンネルを作成します： 「ユーザー機能-通常のトンネルを作成」フィールドに、現在のIPアドレスの横に専用IPアドレスを入力します。この接続を設定する場所を登録すると、強調表示されたIPアドレスは現在のものと同じになります。 次に、最も近いトンネルサーバーを選択します。これにより、 Looking Glassが役立ちます。 作成します！



その結果、以下が得られます。





ここに、IPv4およびIPv6サーバーアドレスと、クライアントのアドレスがあります。 以下は、デバイスの/ 64ネットワークです。



「構成例」タブで、 MikroTikの設定を選択します。





Mikrotikを構成する





専用IPアドレスが49.45.157.133 、サーバーアドレスが236.63.85.135 、サーバーIPv6が2001：470： 2e3a ：5fe :: 1 、クライアントIPv6が2001：470： 2e3a ：5fe :: 2/64、2001 ：470であるとします： 2e3b ：5fe ::専用/ 64サブネットの1/64最初のアドレス2001：470：2e3b：5fe :: / 64



「設定例」セクションで提供された設定 ：

/interface 6to4 add comment="Hurricane Electric IPv6 Tunnel Broker" disabled=no local-address=49.45.157.133 mtu=1280 name=sit1 remote-address=236.63.85.135





/ipv6 route add disabled=no distance=1 dst-address=2000::/3 gateway=2001:470:2e3a:5fe::1 scope=30 target-scope=10





/ipv6 address add address=2001:470:2e3a:5fe::2/64 advertise=yes disabled=no eui-64=no interface=sit1







そして、BRIDGEインターフェースのIPv6アドレス：

/ipv6 address add address=2001:470:2e3b:5fe::1/64 advertise=yes disabled=no eui-64=no interface=BRIDGE







BRIDGEインターフェイスを使用して、内部イーサネットポートとワイヤレスインターフェイスを統合しました。IPv4もネットワーク上に分散されています。



コンピューターで、IPv6サポートを無効にしている場合は有効にします。 IPv6を介したインターネットアクセスが表示されます。 IPv6アドレスは、 ipconfig / allコマンドが表示されたときに、インターフェイス情報またはコンソールにも表示されるはずです



インターフェースの詳細








ipconfig / all








すべてが正常である場合、 ipv6.google.comは応答する必要があり、 テストも正常に合格します。



ping ipv6.google.com

テストの最後の部分が失敗し、9/10の場合、DNSサーバー（プロバイダーが提供）はIPv6インターネットにアクセスできません。 Google DNSを処方するだけで簡単に扱われます



GoogleパブリックDNS IPアドレス



GoogleパブリックDNS IPアドレス（IPv4）は次のとおりです。



8.8.8.8

8.8.4.4

GoogleパブリックDNS IPv6アドレスは次のとおりです。



2001：4860：4860 :: 8888

2001：4860：4860 :: 8844

いずれかの番号をプライマリまたはセカンダリDNSサーバーとして使用できます

両方の番号を指定できますが、1つの番号をプライマリとセカンダリの両方として指定しないでください。



IPv4またはIPv6接続、あるいはその両方にGoogleパブリックDNSアドレスを設定できます。




Windowsは定期的にIPv6アドレスを生成するため、一部のサイトではユーザーがIPアドレスを変更することを誓います。 切断は簡単です （再起動せずに）： 



 netsh interface ipv6 set global randomizeidentifiers=disabled store=active netsh interface ipv6 set global randomizeidentifiers=disabled store=persistent netsh interface ipv6 set privacy state=disabled store=active netsh interface ipv6 set privacy state=disabled store=persistent
