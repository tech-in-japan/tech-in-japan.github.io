Google Mail ChromeアプリをGoogle AppEngineのドメインのメールで動作させる

最近、Google Chromeの次の開発ビルドがChromeアプリのサポートを開始したというメモが現れました。 間違いなくこれは朗報です。デフォルトではGoogle Mailアプリケーションのみが標準のgmail.comメールでのみ機能し、設定はありません。 この問題は簡単に修正できます。



アプリケーションに対応するmanifest.jsonファイルを見つけます（ubuntuでは、/ opt / google / chrome / resources / gmail_appディレクトリにあります）。 このファイルで、 web_urlの値を標準の値からmail.google.com/a your_domain /に置き換える必要があります。

ドメイン内のメールへのリンクをクリックしたときに、ブラウザでアプリケーションのタブを開きたい場合は、行のすぐ後に
「URL」：[
行を追加
" *：//mail.google.com/a/ your_domain / "、


manifest.jsonのカットバージョンの下では、「前」と「後」。



編集前のmanifest.js ：



{
