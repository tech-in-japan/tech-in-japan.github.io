Mozilla Firefox 48ベータ版Webブラウザにはプロセスを分離する機能があります

Mozilla Firefox Webブラウザーの作成者は、ベータ版のWebブラウザー（Windows） の 48番目のバージョンをリリースしました。このバージョンでは、重要な機能がユーザーに期待されています。 Webブラウザには、デフォルトでプロセスを分割する機能がついに備わっています。 このようなイベントは、Firefoxユーザーにとって非常に重要です。これは、以前はWebブラウザーがすべての作業を1つのプロセスで行っていたためです。 このような背景に対して、彼は競合他社のMS Edge、Google Chrome、MS IE11と比べてかなりpale色に見えました。 さらに、この機能がないため、完全なサンドボックスを作成する機能が制限され、作業の安定性にも影響がありました。







確かに、タブからプロセスへの分離については述べていませんが、2つのプロセスに基づいてWebブラウザーを分離することを意味します。 1つ目はユーザーインターフェイスGUIの操作を担当し、2つ目はタブのコンテンツの再現を担当します。 Firefox でタブを分離するプロジェクトは Electrolysis（e10s） と呼ばれ 、そのドキュメントでは新しいブラウザーモードの機能について説明しています。 Ars Technicaによると、Firefoxは今年後半にタブプロセスを完全に分離する可能性があります。



上記の機能は、Firefox 47のベータ版のユーザーが既に利用できましたが、Webブラウザーで有効にするには、特別な設定を有効にする必要がありました。



Firefoxの将来のバージョンでは、Webコンテンツとは別のプロセスでブラウザーUIを実行します。 このアーキテクチャの最初のイテレーションでは、すべてのブラウザータブが同じプロセスで実行され、ブラウザーUIが別のプロセスで実行されます。 将来のイテレーションでは、複数のコンテンツプロセスが予定されています。 マルチプロセスFirefoxを提供するプロジェクトはElectrolysisと呼ばれ、e10と略されることもあります。


developer.mozilla.org/en-US/Firefox/Multiprocess_Firefox



Mozilla Firefoxはすでに別のプロセスを使用してWebブラウザーのプラグイン（plugin_container.exe）を実行していることを思い出してください。現在、ユーザーがアクセスしたWebページのコンテンツに同じルールが適用されます。







Webブラウザの新しいバージョンは、Mozilla ftpサーバーまたはベータセクションでダウンロードできます。 この記事を書いている時点では、新しいベータ版はftpサーバー経由でのみ利用可能でした。



Windows 10で最も安全なWebブラウザーは、Google ChromeとMS Edgeです。 どちらも64ビットベースでタブベースのプロセス分離を使用し、AppContainerに基づくプロセスのサンドボックスにも依存しています。 さらに、両方のブラウザーは、いわゆるメソッドを使用したLPEの悪用に対する特別な保護方法を使用します。 win32k.sysのロックダウン。 一部のセキュリティ機能は、Webブラウザーの機能をプロセスに分離することにより可能になりました。これにより、メインGUIプロセスを、サンドボックス化されたコンテンツで実行中のタブプロセスから分離できました。 最近、GoogleはWindows用のChromeダウンロードページも変更し、Webブラウザーのデフォルトのダウンロードリンクが64ビットバージョンにつながるようにしました。
