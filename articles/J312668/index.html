「こんにちは、シリ。 ヒーターをオンにします”-NooLiteベースのスマートホームとApple HomeKitの統合







私の最初の記事では、家族と私が長い間使ってきたテレグラムボットを介して、カントリーハウスの遠隔暖房制御システムが登場した背景について説明しました。







iOS 10のリリースで、AppleはHomeアプリをユーザーに導入しました-HomeKitを介したスマートホーム管理インターフェイスの実装 。 私はこのトピックに非常に興味があり、利用可能な資料を数晩研究して、この製品とシステムの統合を実装することにしました。 この記事では、インストールと構成のプロセスを詳細に説明し、その結果としてビデオを共有します。

はじめに



スマートホームの初期データと初期構成を理解するには、 最初の記事を読むことをお勧めします。

最初のタスクは、HomeKitのサポートにより、家庭用機器の自動化における既製の無料ソリューションを見つけることでした。 その後、私はすぐにオープンソース製品OpenHabに関するいくつかの記事を思い出しました。 ドキュメントと少しグーグルを読んだ後、HomeKitプロトコルサポートアドオンを見つけました。 現時点では、OpenHabの2番目のバージョンがリリースに向けて準備されており、アクティブなベータテストが進行中です。 その時点で利用可能な最後のバージョンはbeta4で 、これを使用することにしました。







統合プロセスは4つの段階に分けることができます。







  1. OpenHabをインストールして構成する
  2. NooLiteをOpenHabに接続する
  3. OpenHab用のHomeKitをインストールして構成する
  4. ホームアプリのセットアップ


その結果、次のスキームが判明したはずです。











OpenHabをインストールして構成する



OpenHab 2には非常に詳細なドキュメントがあります。メインプラットフォームに加えて、Raspberry Piへのインストールに関するチュートリアルもあります。 インストールに問題はなかったため、ここではすべての手順をコピーしません。







インストール後、OpenHab Webインターフェースはhttp：// <openhab_device_address>：8080のブラウザーで使用可能になりました。













すぐに利用可能：







  1. 基本UI、クラシックUI-OpenHabに接続されたデバイスのコントロールパネル
  2. REST API-実際にREST API
  3. Paper UI-OpenHab管理インターフェイスを使用して構成できます


これまでのところ、基本UIは空です。

















NooLiteをOpenHabに接続する



ドキュメントに基づいて、新しいデバイスをOpenHabに接続するには、次のことが必要でした。







  1. アイテムに追加
  2. OpenHabスマートホームコントロールパネル（基本UI、クラシックUI）に表示されるように、 サイトマップに追加します。 この手順は省略できますが、HomeKitの操​​作には適用されませんが、OpenHabがNooLiteを認識して正しく機能することを確認できます。
  3. 自動化または追加のイベント処理ロジックが必要な場合は、 ルールにルールを追加します


アイテム



項目では、最終的な管理対象デバイス、たとえばNooLiteパワーユニットや、OpenHabを操作するように「教える」ことについて説明します。 







itemtype itemname ["labeltext"] [<iconname>] [(group1, group2, ...)] [{bindingconfig}]


ここで：







  • itemtype-デバイスの種類（スイッチ、調光器など）
  • itemname-デバイス名
  • labeltext-テキスト表示。 たとえば、センサーの場合：「温度[％.1f°C]」、ブロックの場合は単に「ヒーター」
  • iconname-表示されるアイコン
  • group1-グループ
  • bindingconfig-このデバイスを管理する方法


ここで最も興味深いのはバインディングです。 管理対象デバイスとの対話が実装される方法。 OpenHabがNooLiteでどのように機能するかについての情報を少し探した後、既製のバージョンを見つけました。 ただし、 PCおよびRXシリーズ用のNooLite USBアダプターで動作するように作成されています。 私のシステムでは、電源ブロック管理がPR1132イーサネットゲートウェイを介して機能するため、他のオプションを探す必要がありました。







利用可能なバインディングを調べたところ、NooLiteとの通信を実装できるユニバーサルHttpバインディングとExecバインディングが見つかりました。







  1. HTTPバインディングを使用すると、イベントの発生時にネットワーク要求を実行できます（オン/オフ、センサーからの情報の受信）。 PR1132のAPIを使用して、OpenHabとNooLite間の直接通信を実装することができました。
  2. 実行バインディング-イベントでコマンドが実行されます。


その時点で、NooLiteへのリクエストの前処理/後処理に追加のロジックが必要になると考えたため、2番目のオプションを選択しました。

ルール



各RGBチャンネルの輝度値を取得するには追加のロジックが必要だったため、LEDストリップコントロールは1つのコマンドに収まりません。 したがって、ルールの状態変更の処理について説明しました。 







 # /etc/openhab2/rules/noolite.rules import org.openhab.core.library.types.* var HSBType hsbValue var String redValue var String greenValue var String blueValue rule "Set RGB value" when Item RGBLight changed then val hsbValue = RGBLight.state as HSBType val brightness = hsbValue.brightness.intValue val redValue = ((((hsbValue.red.intValue * 255) / 100) * brightness) / 100).toString val greenValue = ((((hsbValue.green.intValue * 255) / 100) * brightness) / 100).toString val blueValue = ((((hsbValue.blue.intValue *255) / 100) * brightness) / 100).toString var String cmd = "python noolite_cli.py -ch 3 -cmd 6 -fmt 3 -d0 " + redValue + " -d1 " + greenValue + " -d2 " + blueValue executeCommandLine(cmd) end


RGBLightの状態が変化し、各チャネルの値（0-255）を取得し、行python noolite_cli.py -ch 3 -cmd 6 -fmt 3 -d0 redValue -d1 greenValue -d2 blueValue



をpython noolite_cli.py -ch 3 -cmd 6 -fmt 3 -d0 redValue -d1 greenValue -d2 blueValue

リモートアクセスと自動化



私のローカルネットワークにはApple TVがあり、それ自体は追加設定なしで「ホームセンター」として識別されました。 後でわかったように、スマートセンターへのリモートアクセスと自動化設定（スケジュールされたアクション、ジオロケーションに基づくアクションなど）にはホームセンターが必要です。 ホームセンターには、3世代または4世代のApple TV（第3世代ではリモートアクセスのみが機能し、第4世代では自動化が必要）またはiOS 10を搭載したiPadを使用できます。

Apple TVで追加の設定を行わなかったことを嬉しく思いました。 あなたがする必要があるのはあなたのアカウントでiCloudにログインすることだけです。

おわりに



HomeKitとの統合により、Homeアプリケーションを介してiOSデバイスからスマートホームに便利な制御インターフェイスを追加できるようになり、その機能も拡張されました。







  • テレグラムボットの代わりに、自宅のすべてのデバイスを完全にリモート制御
  • 音声制御
  • に基づく自動化：時刻、ジオロケーション、センサー読み取り値（例：モーション検出）
  • アクセス権の差別化により、招待状（iOSデバイスとAppleアカウントが必要）を通じてユーザーをスマートホーム管理に便利に追加


ホームアプリケーション自体とホームオートメーション用のアプリケーションの詳細なレビューは、すでにこのトピックの範囲外であり、別の記事に値すると思います。 しかし、私にとって最も興味深いのはジオロケーションです。これに基づいて、興味深い自動化シナリオを実装できます。 たとえば、自宅のすべてのユーザーが離れると、どこでもライトをオフにできます。 ユーザーがさらに進んだ場合（都市に向けて）、地下のソケットや電気などの一部の消費者をオフにします（そこにポンプステーションがあります）。







この記事の作業により、カントリーハウスの制御システムを横から見て、近い将来に私が改良する場所に注意することができました：







  • システムを簡素化するために、PR1132イーサネットゲートウェイを使用し、HTTP APIを介してNooLiteブロックを制御する代わりに、RXおよびPCシリーズUSBアダプターを使用して、コマンドを直接発行できます。
  • 私の知る限り、近い将来、NooLiteはフィードバック付きの新しいUSBトランシーバーアダプターと電源ブロックをリリースする予定です。 それらを試してみるのは非常に興味深いでしょう。
