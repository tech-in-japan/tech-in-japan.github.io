PostgreSQL 8.3.0

PostgreSQLの非バグ修正バージョン（8.2）がリリースされてから1年余りが経過しました。公式ブログには既に「今週は8.3を見る！」 と書かれています 。



まあ、待っているようです。 サイトはまだ発表されていませんが（更新： 現在発表されています）、 新しいバージョン8.3.0のソース 、さらにはWindows用のバイナリもダウンロードできます 。



そして、それはスイングの価値があります。 リリースノートによると、PostgreSQL 8.3には多くの利点があります。

  • 古き良き全文検索tsearch2は現在、PostgreSQL言語の一部であり、contribモジュールではありません。
  • SQL / XML標準がサポートされ、XMLデータ型が登場しました。
  • 待望の列挙（データ型）が登場-ENUM;
  • 複合型の配列を作成できるようになりました。
  • UUIDデータ型が表示され（私自身：昨日使用したかっただけです...）、それに対応する関数が作成されました。
  • これで、ソート時に、NULL値を重要な値の前に表示するか後ろに表示するかを選択できます。
  • 現在のカーソルの値を変更したり、削除することもできます。
  • サーバー設定は、個々の機能ごとに変更できるようになりました。
  • ユーザーが作成したデータ型に修飾子を追加できるようになりました。
  • テーブル統計またはその構造が大幅に変更された場合、キャッシュされたクエリを自動的に再スケジュールできるようになりました。
  • ...一般に、ロギングと統計計算の両方が大幅に改善されました。
  • Windowsでの認証にSSPIを使用することが可能になりました。
  • Autovacuumはデフォルトで有効になり、パフォーマンスを改善するために複数のプロセスで同時に実行できるようになりました。
  • Windows版のPostgreSQLは、MS Visual C ++を使用して構築できるようになりました。




...そしてもちろん、かなりの数のバグ修正とパフォーマンスの最適化。 興味深いベンチマークを待っています:)
