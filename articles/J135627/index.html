AndroidのXMPP-SMSゲートウェイ





はじめに


この記事を書いた理由は、Androidシステム用のプログラムを作成する必要があるためです。このプログラムを使用して、商品またはサービスの注文が行われたオンラインストアの所有者にSMSメッセージの形式で注文データを送信できます。 以前、私はGSMモデムとC ++で書かれたプログラムを含むシステムを使用しました。これはATコマンドを使用してモデムとglooxライブラリと通信し、XMPPを介してメッセージを受信し、ウェブサイト側でxmpphpライブラリを使用して注文データを送信しました。 このスキームでは、注文受付システムが24時間動作するため、コンピューターを常にオンにしておく必要がありました。そのため、追加のエネルギー消費、夜間のファンからのノイズ、インターネット接続の常時監視が行われました。



この記事全体で作成するプログラムの主なタスクは、 XMPPプロトコルを使用して特定の形式のメッセージを受信し、受信したデータをSMSで転送することです。 開発環境は、 ADTプラグインと必要なSDKがインストールされたEclipseになります。 XMPPプロトコルを介した対話には、 Androidデバイス用のSMACKライブラリーが使用されます。



1. SMSを送信する


まず、アプリケーションのフレームワークを作成します。その後、必要な機能を構築します。 これを行うには、Eclipseで、次のデータを使用してAndroidプロジェクト（Ctrl + N-Android-Android Project）を作成します。







新しいプロジェクトを作成した後、SMSメッセージを送信するために必要な許可（Permission）をAndroidManifest.xmlファイルに追加します。 これを行うには、Eclipse開発環境でAndroidManifest.xmlファイルを開き、[Permissions]タブに移動して[Add ...]ボタンをクリックし、表示されるウィンドウで[Uses Permission]アイテムを選択し、[OK]ボタンをクリックして、解像度を選択し、検索して選択できるようになりますアイテムandroid.permission.SEND_SMSを選択し、アクションを保存します。 すべての操作後、[アクセス許可]タブは次のようになります。







ここで、たとえば、SMSメッセージを送信する最も簡単な方法を考えます。これは、通常のAndroidエミュレーターでテストできます。 これを行うには、Android Virtual Device Manager（ウィンドウ-AVD Manager）と次のパラメーターを使用して、2つの新しい仮想デバイスを作成します。







作成したプロジェクトのonCreateメソッドで 、次のコードを追加します。 



package ru.blagin.xmppsmsgate; import android.app.Activity; import android.os.Bundle; import android.telephony.SmsManager; public class XMPPSMSGateActivity extends Activity { @Override public void onCreate(Bundle savedInstanceState) { super.onCreate(savedInstanceState); setContentView(R.layout.main); SmsManager sms = SmsManager.getDefault(); sms.sendTextMessage("5556",null,"Text SMS",null,null); } }




次に、両方のAndroidエミュレーターを起動します。各エミュレーターにはSMSメッセージの送受信を確認するための独自の番号があります。エミュレーターの起動と初期化が完了したら、番号5554のエミュレーターでアプリケーションを実行する必要があります。アプリケーションの起動後、番号5556のエミュレーターがSMSメッセージを受信します。







SMSメッセージの送信は、AndroidシステムがSMSメッセージで必要なアクションを実行できるようにするSmsManagerクラスを使用して実行されました。 このクラスのオブジェクトを初期化するために、静的メソッドSmsManager.getDefault（）が使用されました。 SMSはsendTextMessageメソッドを使用して送信されます。メソッドパラメータは次のとおりです。



destinationAddress-メッセージの送信先の番号。

scAddress-メッセージが送信されるモバイルオペレーターのSMSセンター番号。このパラメーターの値がゼロの場合、デフォルトの番号が使用されます。

text -SMSメッセージテキスト。

sentIntent -null値でない場合、PendingIntentオブジェクトがこのパラメーターに渡され、メッセージの送信結果に関するメッセージを受信します。

deliveryIntent -null値でない場合、PendingIntentオブジェクトがこのパラメーターに渡され、メッセージ配信の結果に関するメッセージを受信します。


Androidエミュレーターはそのタスクを完全に処理しますが、いくつかの制限があります。たとえば、その上でメッセージ配信の結果を確認することはできません。このため、将来実行される実際のデバイスを使用する必要があります。 また、sendTextMessageメソッドを使用してメッセージを送信する場合、その長さは160文字を超えることはできません。 長いメッセージの場合は、sendMultipartTextMessageメソッドを使用する必要があります。このメソッドは、160文字未満のメッセージを送信することもできます。

上記の例では、SMSメッセージの送信と受信者への配信に関する通知を受信しないため、必要な処理を追加してアプリケーションの機能を拡張します。 これを行うには、アプリケーションにブロードキャストの意図の2つの受信機を登録する必要があります。これにより、必要な意図が処理され、対応するテキストメッセージが画面に表示されます。

2. XMPPプロトコルで作業するためのサービスを作成する


次に、XMPPを介して通信する機能をアプリケーションに追加する必要があります。 これらの目的のために、バックグラウンドで実行されるサービス（サービス）を作成します。 SMACKライ​​ブラリーを使用するサービスは、メッセージを受信して​​処理します。 さらに、ブロードキャストの意図を利用して、受信したメッセージからのデータは、SMSを介した表示およびその後の送信のために、アプリケーションのメインクラスに転送されます。



サービスをアプリケーションに追加するには、サービスの実装クラスを定義する必要があります。そのためには、Eclipse開発環境で、プロジェクトパッケージ名を右クリックし、表示されるメニューで[新規]および[クラス]を選択します。







表示されるウィンドウで、画像に示されている必要な項目を入力し、「完了」ボタンをクリックします。







これらのアクションの後、クラスの実装がプロジェクトに表示されます。AndroidManifest.xmlファイルにサービスクラスを登録する必要があります。そのため、Eclipse開発環境でファイルを開き、[アプリケーション]タブに移動し、[アプリケーションノード]セクションの[追加]ボタンをクリックし、表示されるウィンドウで、サービス項目を選択します[OK]ボタンをクリックします。 サービスのクラス名を指定する必要があると、すべての操作の後、[アプリケーション]タブは次のようになります。







次に、アプリケーションに別のアクセス許可を追加して、オンラインにできるようにします。これを行うには、上記と同じ手順に従ってSMSメッセージの送信を許可します。今回のみandroid.permission.INTERNETを選択します。

おわりに


この記事は入門記事であり、私自身の初心者プログラマーを対象としています。 作成されたアプリケーションには、多くの欠点と制限があります。たとえば、SMSメッセージの送信は、メインアプリケーションウィンドウがアクティブな場合にのみ実行されます。 インターネットアクセスがWI-FIのみを介して行われている場合、インターネットアクセスのチェックは行われません。デバイスがスリープモードになると、バッテリーを節約するためにWI-FIがオフになります。 この問題は、 Wi-Fi Keep Aliveアプリケーションを使用するか、アプリケーションに自分でこの機能を追加することで回避できます。 送受信されたメッセージのログはありません。
