OpenShift CloudにWordPressをインストールする

WPをクラウドにインストールするトピックに興味がある場合は、ここで実装オプションの1つを検討します。



OpenShiftクラウド（RedHatによる）にWPをインストールすることを検討します。これは、私が知っているすべてのオプションがあるためです。



同様の記事はすでに存在していましたが、私にとってはすべての質問に答えたわけではなく、すべての問題を解決したわけでもありませんでした。 したがって、ここで私のパスを説明します。



この説明は、読者がWebアプリケーションを操作するための基本的なスキルを持っていることを期待して書かれています。



1. WordPressを共有ホスティングにインストールします



2. GITを介したコードのロード（ただし、SFTPを使用することもできます）



始めましょう...



OpenShiftアカウントを登録する



登録フォームは非常にシンプルで、すぐにさらなるシンプルさと将来への希望を抱かせます。これらの期待は満たされていると思います。



指示に従って登録手続きを行います。





先に進みます...ログインして、コンソールで設定を確認します。



名前空間を指定する



まず、[設定]セクション、つまり名前空間から始めます。ここでは、アプリケーションのあるURLの部分を指定する必要があります。



図のアプリケーションURLがどのように見えるかをケースプレスに指示しました。



SSHキーを構成する



アプリケーションへの安全なアクセスを整理するために、SSHキーが必要です。



-SSH（安全なコマンドライン）



-SFTP（安全なファイルのアップロード）



-GIT（アプリケーションコードの同期）



次：



すでにSSHキーがあり、何をどのように行うかがわかっている場合は、 [設定]ページの[公開キー]フィールドで公開キーを手動でアップロードできます。

理解しておらず、最小限のトラブルが必要な場合は、ツールのクライアントパッケージをインストールするだけで（ OpenShift RHCクライアントツールのインストール ）、「rhc setup」コマンドを取得すると、システムはキー自体をチェックし、キーがない場合は作成し、サーバーにアップロードします。

結果は次のようになります。





必要な接続の数に応じて、キーを追加することもできます。 同僚がいるか、別のデバイスから接続する必要があるかもしれません。



最初のアプリケーションを作成します



基本的なカートリッジを入れて


ここでは、OpenShiftが「アプリケーション」という用語を正しく定義していることを止める価値があります。 CMS WordPress。 ロシアで採用されている概念とは対照的に、意味がぼやけた多くの用語が導入されています：Webサイト、Webアプリケーション、サービスなど。



行こう：

1.「 アプリケーションの作成 」ページに移動します

2.次のような多くの既製のテンプレートが提供されます。 WordPressの簡単なインストール。これは丁寧に拒否します。 簡単な方法を探していない



3. OpenShiftテンプレートを使用してWordPressのインストールを試すことができます。 シンプルで高速です。 しかし、私はそれが好きではなかった、なぜなら GitHubのリポジトリから英語のリサイクルWordPressがインストールされます 。これは、OpenShift機能への適応によるさまざまな変更により、標準のWordPressとは大きく異なります。 いくつかのプラグインをインストールした後、白い画面でエラーが発生しました。理由を理解できず、すべてを破壊しました。



4. PHP 5.3を選択します。 アプリケーションのデータを指定する必要があるフォームが開き、アプリケーションがPHP 5.3カートリッジに基づいていることを示します（はい、OpenShiftのコンポーネントはカートリッジと呼ばれます）。 ここで必要なのは、アプリケーションの名前を指定することだけです。 彼は名前testを与え、URLはtest-casepress.rhcloud.comでした。 それだけです 「アプリケーションの作成」をクリックします。 少し時間がかかります。





5.アプリケーションをインストールした後、詳細な手順が記載されたページに移動します。 ここでは、GITの指示に注意する価値があります。 一番下の行は、ここでのコードのダウンロードは、多くが慣れているFTPではなく、GITを介して行われるということです。 これは重要な機能です。 掘り下げることはできませんが。





6. 「マイアプリケーション」ページに移動して 、リストの最初のアプリケーションを確認します。





7.アプリケーションの名前をクリックすると、設定ページが表示されます。



8.ここで、このアプリケーションに別のカートリッジMySQLを追加します。 まったく同じ方法で、「カートリッジの追加」ボタンを押すだけです



WordPressをダウンロードする準備ができました。 次のステップに進みます。



ドメイン（URL）をカスタマイズする


appname-namespace.rhcloud.comのようなドメインが気に入らず、「mydomen.ru」タイプを追加する場合、それらは美しく理解しやすいため、エイリアスを追加する必要があります。







WordPressファイルをアップロードする


ホスティングの一般的な方法であるFTP経由ではなくGIT経由でコードをダウンロードしないことに注意してください。



GITをお持ちでない場合は、それを配置する時間です。 良い指示はこちらです。



別のGITクライアントを使用していますが、簡単にするために、このマニュアルでは標準のGIT GUIを使用します。 OpenShiftの手順で、コンソールモードでGITを操作する方法を読むことができます。



1. GIT GUIを起動し、「既存のリポジトリのクローン」コマンドを選択します



2.開始位置として、OpenShiftを使用してアプリケーションのGIT URLを選択します。これは、アプリケーションの設定ページ（項目7、基本カートリッジの配置）で確認できます。宛先ディレクトリとして、リポジトリのコピーを保存するディレクトリを指定します。 PC上の任意の便利なディレクトリを指定できます。 それだけです 「クローン」をクリックして、ファイルがダウンロードされるのを待ちます。



3.次の結果が表示されます（GITは落ち着いており、フォルダーにはアプリケーションのコンテンツが表示されます）。



4.とりあえず、これらのフォルダーの意味と意味を見逃し、phpフォルダーに注意してください。これは、アプリケーションの実行可能ファイルが置かれている場所です。 index.php、ここで、WordPressファイルをコピーします。 以下を取得する必要があります。



5.リポジトリの内容を変更しました。 GITを開いて[再読み込み]ボタンをクリックすると、次のように表示されます。



6.変更を受け入れます。 [すべて準備]をクリックし、変更に関するコメントを指定し、[保存]、[送信]の順にクリックしてから、送信フォームでもう一度[送信]をクリックします。 GITはサーバーへのファイルのアップロードを開始します。



7.データをサーバーリポジトリ（オリジン）にロードすると、パブリケーションスクリプトが自動的に実行されます。



WordPressのカスタマイズ


ここで、WordPressでの従来の作業との最初の違いが生じます。GIT以外の方法で結果なしにサーバー上のファイルを編集することはできません。



一方で、これは悲しみです-一方で、不必要なアクションにより、変更を簡素化し、エラーが発生した場合に簡単にロールバックでき、複数の人がアプリケーションで作業している場合は交換できません。



そのため、wp-config.phpファイルを作成するための標準のシンプルで使い慣れたダイアログは表示されません。 PCで変更し、GIT経由でサーバーにアップロードする必要があります。 そして、単に変更するだけでなく、OpenShift環境の機能を考慮して、少し注意が必要です。



本当にGITを学習したくない場合は、SFTPを使用して実行できます。 彼と一緒に仕事をする際には、 ビデオの説明（英語）とテキスト（ロシア語、マット付き）があります。 Windowsでは、FileZillaを使用してログインできました。 ただし、GITを考慮しなければならないことを理解する必要があります。何らかの方法で行われた変更がGITに反映されない場合、GITを介した次のダウンロードでは、すべての変更が洗い流されます。



行こう：

1. WordPressの新規インストールがある場合、フォルダーにwp-configファイルはありませんが、wp-config-sample.phpがあります。 コピーしてwp-config.phpに名前を変更します

2.新しいwp-config.phpを開き、キーパラメータを指定します：データベース名、データベースユーザー名、データベースパスワード、データベース接続アドレス。 それらを知る必要はありません、なぜなら それらは変数として示されます。 ここから素晴らしく完全にコピーできます。 次のようになりました。



3.次に、塩を手動で調整する必要があります。

3.1。 ファイルに記載されているように、このためにはapi.wordpress.org/secret-key/1.1/saltに移動する必要があります

3.2。 結果を適切なセクションwp-config.phpにコピーします

4.これで、wp-config.phpの変更が完了しました。 保存します。 閉じます。 GITに行きましょう...

5. GITは、wp-config.phpファイルの変更を検出し、受け入れ、コミット、プッシュすることを示す必要があります。

6.ここで、アプリケーションのページに移動すると、WordPressのインストールがサイト名などの設定で直ちに開始されることがわかります。 GITを使用してすべてを実行したため、接続の構成をデータベースに渡しました。



7.通常どおり、さらにインストールが行われます。



アプリブレーキ



取り付け後、ハードブレーキングに気付きました。 1つのページが非常に長い時間（30のデータベースクエリで1ページあたり1〜4秒）読み込まれますが、通常の共有ホスティングでは、同じ条件下でこの数値は0.2〜0.4秒です。



フォーラムでのコミュニケーションは、私がシベリアに住んでいるという議論に帰着しました。 そして彼らはアメリカにいます-彼らはより速く動作するデータベースを持つウェブサイトを持っているので



サーバーの解体は私のやり方ではありません。 シンプルなソリューションを探しています。



公式メールに連絡することで、アプリケーションのリソースを中レベル（デフォルトではSmallに設定）に増やすことに同意することができました。



Gears = Mediumを選択してアプリケーションを作成できるようになりました。 その後、サイトは通常のパラメーターに近づきました：1ページあたり0.3〜0.4秒。





機能とメモ



1.サイトを操作するスキームを選択する必要があります。



1.1。 従来型（コンソール、SFTP ...）



1.2。 GIT-WordPressで得られるすべての設備、プラグインの追加のしやすさ、SFTPを介して何かを追加、削除、または修正する機能を放棄する必要があります。 しかし、見返りに、履歴、ロールバックなどを追跡する機能を使用して、例外なくアプリケーションのすべての変更の完全な監査とバージョン管理を取得します。SFTPを選択した場合、サイト管理の原則は、 GIT、サイト全体を誤って破壊する可能性があります。



2.すべてのファイルを含むアプリケーションアーカイブを作成するのは非常に簡単です。



2.1。 これを行うには、ツールのパッケージが必要です（OpenShift RHCクライアントツールのインストール）



2.2。 さらに、D：ドライブにアーカイブを取得する場合は、次のコマンドを指定します。「rhc app snapshot save -a {appName} -fd：\ appname.tar.gz」。{appName}は指定したアプリケーションの名前です。アプリケーションを追加するためのページ（項目4、基本カートリッジを入れる）。



3. MySQL DBMSを操作するためのphpMyAdminカートリッジを自分で置くことができます。これは、アプリケーションのページのWebコンソールから簡単に実行できます。



4.すべてのインストール後に、.gitignoreファイルを「/ appname / php /」ディレクトリに追加し、「wp-content / uploads」を除外して、サイトファイルがGITの注意を引かないようにするとよいでしょう。



5.無料の関税があります。これは良いことです。 しかし、このレートでは、1500〜2000ルーブルの従来のホスティングと比較してプラグインがなくても、サイトの速度は著しく低下します（30のデータベースクエリで1〜4秒のページ読み込み）。 1年あたり（データベースへの30のクエリで0.2-0.4秒でページを読み込む）。 Medium Gearsへの移行はこの問題を部分的に解決しますが、現在はサポートサービスを通じてのみ行われており、将来的には有料オプションになります。



誰かが中規模モードに入らずに、より高速な作業でOpenShiftでDBMSを構成する方法を知っている場合、私は非常に感謝します。
