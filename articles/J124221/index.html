私のハードドライブ蘇生の経験

おそらく、遅かれ早かれこれはすべての人に起こります-一度、USBハードドライブを手に入れました。

急いでどこかで、彼は単にラップトップのUSBソケットからコードを抜きました（よくあることです）。

私は仕事に来て、HDDを接続しますが、空です...



何をすべきか！？ 私がしたことは読んでいます。





もちろんパニックに陥らないでください。



まず、主なルールを覚えておく必要があります。記憶媒体で行う操作が少ないほど、情報を復元する可能性が高くなります。 コピー、カット、削除する必要はありません。

メディアの「 病気 」のさまざまな症状があるかもしれません：いくつかのフォルダ/ファイル、コピーエラー、失われたシステムはメディアをフォーマットするように頼みます。

もちろん、各ケースには個別にアプローチする必要があるため、テスト用にいくつかの異なるツール/プログラムを用意することが重要です。

私は常にラップトップ、自宅や職場のPC、ポータブルHDDにメインソフトウェアが入ったフォルダーを保持しています。これらはいつか役に立つかもしれません。 もちろん、在庫のデータを回復するためのハードディスクとソフトウェアを操作するためのプログラムがいくつかありました。

たとえば、EASEUS Partition Master、Norton Partitionmagic、PartitionManager、Active File Recovery、Error Repair Professional、HDD Regenerator、HDDScan、Raxco PerfectDisk、R-Studio、Recover4allがありました。



以前は、 EASEUS Partition MasterとNorton Partitionmagicをクロールしました。 1つ目は、2つ目のほぼライトで無料のバージョンです。

プログラムはメディアの存在を示しましたが、ファイルシステムのタイプを判別できませんでした。



サンプルプログラムインターフェイス：





もちろん、最初に必要なことは、必要なファイルシステム内のメディアをフォーマットすることです。また、動作するハードドライブが再び得られるようですが、そこに格納されている情報は私にとって重要だったので、ハードドライブを復元するための急進的な方法としてフォーマットを残しました。

残念ながら、すべてのプログラムのテストのスクリーンショットはありませんが、最終的には、プログラムを整理してさまざまなフォーラムを1時間読んだ後、ネットワークから別の小さなユーティリティtestdiskをダウンロードしました 。

プログラムのインターフェースは英語ですが、少なくとも技術的な英語を少しでも知っている人にとっては難しくありません。



このプログラムでは、「突く」方法が選択され始めました。

1）ログを保持する必要があるかどうかを選択します。 必要ありません。





2）次に、テストするメディアを選択します。





3）パーティションテーブルのタイプを選択します。 私の場合、Intel。





4）次に、メディアで可能な操作を含むメニューに移動します。





4.1既存のパーティション構造の分析と失われたパーティションの検索：





4.2追加セクション-ファイルシステムのユーティリティ：





4.3ディスクジオメトリ：





4.4そして、私たちにとって重要なセクションは「 MBRコード 」です。 ブートセクターを上書きします。

「マスターブートレコード」-メインブートレコード、ハードディスク上の最初の物理セクター（サイズは512バイト）。





突っ込んだところ：

目的のドライブを選択-セクターをチェック-MBDを上書き-ブートセクターを上書き

同時に、一度再起動しました。 ハードディスクは、すべてのデータとともに元の動作状態に戻りました。

この事件の後、私はこのコンソールユーティリティを必要なソフトウェアとともに正当な理由でフォルダにコピーしました。

すぐに私の同僚がコンピューターを見るように頼みました-電力サージの後、コンピューターはロードを停止しました。



私の行動：


別のPCに接続します。

システムはディスクを認識しますが、入力しようとすると、ディスクはフォーマットされていないというメッセージが表示され、フォーマットするように提案されます。

削除または破損したファイルを回復するプログラムを試しました。

いくつかのプログラム（たとえば、R-Studio）はすべてのフォルダーとファイルを見つけましたが、それらを曲がって復元しました-単一のドキュメントは開かれませんでした。 同時に、プログラムはFATファイルシステムがディスク上にあることを明確に示しました。

最初に頭に浮かんだのは、おそらくディスクのブートセクター（ブートセクター）が破損していることです。

不良セクタをチェックして修正するために、HDD Regeneratorプログラムを一晩で終了しました。

朝、「新しいハードウェアまたはソフトウェアとの競合」のような碑文で「 死のブルースクリーン 」を発見しました。

ハードなものを削減し、マザーボードからバッテリーを引き出してBIOS設定を削除し、セーフモードでの通常の読み込みを含むあらゆる種類の操作を試みました。 何も助けませんでした。

すべては以前と同じです。システムは認識しますが、入力しません...

さらに、彼はusb-hddを復元したときと同様の一連の操作を行い、10分後に同僚のPCが起動しました。



中国のフラッシュドライブ


最近、中国から32 GBのフラッシュドライブを注文しましたが、何らかの欠陥があることがわかりました。 最初はフォルダーとファイルが消え、フォーマット後に一般に開くことができなくなりました。 私は数日間彼女と手品をしてきましたが、私はただ蘇らせられません。
