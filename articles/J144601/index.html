Red HatがJRuby開発者によるPaaS OpenShiftを強化

今月初め、 Red HatがPaaS OpenShiftのコードを開いたことがニュースで明らかになりました 。 このプロジェクトの開発をフォローしている人は、おそらくRed HatがOpenShiftに多くの注意を払っていることに気づいているでしょう。 新しい機能の追加（最近-Node.jsのサポートから）に加えて、既存の機能は常に改善されています。 後者には、JRubyの2人の主要開発者のRed Hatへの移行に関するニュースも含まれています。



Red Hatは、かつてSun Microsystemsで働いていた2人の主要なJRuby開発者であるCharles NutterとThomas Eneboを雇ったと言いました。 彼らは入社して、 Immutant 、 TorqueBox 、 JRuby 、 OpenJDK、およびJava Virtual Machine（JVM）の言語に関する作業を拡大しました。 また、Charles Nutter氏によると、OpenJDKの開発に積極的に参加する機会が、Red Hatに移行するという彼の決定に影響を与えました。



Red Hatは、Sunが開始したOpenJDKプロジェクトをサポートした最初の企業の1つです。 Red Hatは、Java Compatibility KitにアクセスするためにOpenJDKコミュニティTCKライセンス契約に署名し、引き続きプロジェクトに積極的に参加しています。



NutterとEneboは、Sunの後に仕事をし、独自のPaaSプラットフォームを開発するEngine YardからRed Hatに移行します。 これらの開発者は、JRubyの専門知識と優れたプログラミングスキルを尊敬しています。 Red Hatへの移行により、同社はクラウドコンピューティングに専念し、Linuxディストリビューションのメーカーだけにとどまらないことを明確に示しています。



Red Hatは、オープンクラウドとは何かについて独自の意見を持っています。 OpenShiftを使用すると、プログラミング言語、フレームワーク、ミドルウェアを選択できるようになります。これは、 ベンダーロックインが存在しないことを保証します。 JavaとJRubyの両方がOpenShiftでサポートされているため、Red Hatは新しい開発者の注目を集めるためにPaaSの開発と強化を続けたいと考えているようです。



2001年に作成されたJRubyプロジェクトは、RubyのオープンソースJava実装であり、その主なマネージャーはNutterとEneboに加えて、Ola BiniとNick Siegerです。 Sun Microsystemsは、JVMで動作するプログラミング言語の数を増やす計画が現れた2006年にEneboとNutterを採用しましたが、OracleがSunを引き継いだ後、JRubyの Oracle の将来の「不確実性」のために会社を辞めました。



2006年にSunチームに参加して、Nutter は次のように書いています。
主な目標は、JRubyに本当に必要な注意を向けることです。 SunはJVMでのRubyの可能性に気付いたため、JRubyを可能な限り完全、高速、安定にすることに焦点を当てます。 Rubyツールのサポートを引き続き提供していきます。 個人的な優先事項に関する限り、私は引き続きJRubyコミュニティを拡大し、JavaとRubyのコラボレーションを強化し、Sun Microsystems向けのRuby-on-JVMフルプラットフォーム戦略に取り組みます。


JRubyのバトンはSunからRed Hatに移行したようで、OpenJDKプロジェクトのおかげでJavaで実装されます。
