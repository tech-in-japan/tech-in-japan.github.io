オンプレミスと クラウドIaaS-長所と短所

クラウドの利点と欠点に関する多くの情報を見つけることができると思います。 多くの場合、クラウドテクノロジーのメリットに関する情報が、デメリットに関する情報よりも優先されます。



以下の情報は、決して最終的な真実であると主張するものではなく、完全ではありません。 しかし、あなたのサポートのおかげで、このリストが大幅に拡張され、多くの読者に役立つことを本当に願っています。 長所と短所は、主にInfrastructure as a Service（IaaS）に関連しています。



クラウドのメリット、オンプレミスのデメリット



クラウドに移行するとき、次のことを忘れることができます 。



必要性は、データセンターの物理的なITインフラストラクチャを管理します。



1.サーバーハードウェア、欠陥のあるコンポーネントの交換

2.バックアップ電源（USV）のソース。 データセンターの規模、ディーゼル発電機のメンテナンス、または例えばバッテリー交換に応じて

3.ネットワーク：ルーター、スイッチ、モジュール、ケーブル

4. SHD（ストレージ）、RAID-sなし、アグリゲートおよびディスク。 iSCSI、FibreChannel、またはInfiniBandに関する議論はありません。 オールフラッシュアレイへの完全な移行または部分的な移行ですか？

5.バックアップ（バックアップおよびデータストレージシステム）

6.情報セキュリティ



上記のアイロンは定期的に更新するだけでなく、リースまたは保証期間（3〜5年）後に交換する必要があります。 どのメーカーからハードウェアを注文するかについて議論する必要はありません。



7.コンピューターセンター冷却システム

8.消火システムと火災の早期発見

9.データセンターの物理的保護の組織

10.拡張オプション（新しいキャビネットの追加）、および関連する冷却および消火システムの変更



セキュリティの観点から、物理的であれ情報であれ、深刻なプロバイダーは平均的な企業を大きく上回っていることは注目に値します。



人件費



上記のすべてのカテゴリには独自の制御システムがあり、それには資格のあるスタッフが必要です。 新しい技術への移行の場合、従業員も訓練する必要があります。



クラウドは、主に既存のインフラストラクチャの交換または更新に関連するプロジェクトで外部リソースを引き付けるという点で、コストの削減に役立ちます。



課税に関しては、クラウドは魅力がないわけではありません。 機器の購入のための設備投資（CAPEX）は、より収益性の高い通常の運用費用（OPEX）に変換されます。



さて、最後に主要なこと、電気のコストについて忘れてください。



スケーラビリティ、自動化、革新



インフラストラクチャの急速な自動拡張または縮小の可能性は、クラウドテクノロジーの主な利点の1つと言えます。 本当に必要なものだけを使用して支払います（Pay-as-You-Go）。



クラウドサービスの主要プロバイダーは、追加サービスとして顧客に提供できるもの、またはプロバイダーのコスト削減に役立つものを常に探しています。



災害復旧



インフラストラクチャの中断のない運用を作成し、災害復旧を確保するという点で、クラウドには明確な利点があります。



クラウドの欠点、オンプレミスの利点



多くの場合、上記の利点は簡単に欠点に変わります。



そのようなデータセンターから始めましょう。 多かれ少なかれ中規模のビジネスには、すでに独自のデータセンターがあります。 そのため、消火システム、中断のない電源、および冷却装置を備えた適切な保護された部屋が既にあります。 データセンターには、適切な機器（サーバー、ストレージ、スイッチ、バックアップライブラリ）で満たされた十分なキャビネットが既にあります。



購入したサーバー機器を使用すると、システムを簡単にアップグレードでき、たとえばメモリを追加できます。 廃止されたサーバーを使用して、テスト環境を作成できます。



上記のすべての機器が同時に購入され、同一の保証期間があることを想像するのは困難です。その後、クラウドへの移行は十分に考慮された論理的なステップになります。



近年、プロセッサの電力消費からデータ転送速度が40 GB以上のネットワークやオールフラッシュアレイストレージに至るまで、多くのコンポーネントの効率が大幅に向上しています。 多くの場合、ストレージシステムを交換するだけで、データセンターの「負荷軽減」（スペースの節約、電力コストの削減）が効率的になります。 6Uを占有するPureStorageのAllure Flash-Arrayを使用して、NetApp製品で満たされた19A 42Uキャビネットをどうにかして取り除いた自分の経験からの話を思い出してください。



ローカルインフラストラクチャを放棄することは投資を失うことを意味するという主張には、生命に対する権利があります。



信頼と信頼性



私の意見では、プロバイダーに対する信頼は、クラウドテクノロジーを評価するための最も重要な基準です。



必ずしなければならない



  • データの整合性
  • 特別な状況の場合の損失のない高速回復
  • 現地法の完全な遵守
  • 従業員のプロ意識
  • 提供されるサービスの品質がはるかに高いか、少なくとも以前のサービスと同等であること


安全基準に準拠するためのさまざまな証明書の存在は、セキュリティ自体を保証するものではありません。



2つの地域のデータセンターのいずれかが失われた場合でも、プロバイダーはインフラストラクチャ全体の完全な機能を提供できると確信しています。 主なものは、特にビジネス活動の増加の期間における拡張性です。 これについてはわかりませんが、あなたは？



同僚の一人がかつて私に、ある地域のデータセンターが完全に失われる可能性は、道路で恐竜に会うようなものではないと言っていました。 そして、ニューヨークの住民の誰が、明日の午後、都市のシンボルの1つがもはや存在しないと信じるだろうか？ 大規模なデータセンターは間違いなく「不親切な人々」の目標になる可能性があります。 その損失は深刻な経済的損害を引き起こすだけでなく、広範囲に及ぶ結果をもたらす可能性があります。 クラウドとは、多くの企業が卵を1つの大きなバスケットに入れ、バスケットが決して落とされないことを願うときです。 卵のそれぞれは、不滅のコシュチェイの卵の一種です。 希釈することで、所有者の生存率を危険にさらします。



私の知る限り、米国の法律の改正により、プロバイダーの過失によるデータ盗難の場合、ユーザーごとに最大100,000ドルの損害賠償を請求できるようになりました。 この文脈では、サボタージュという言葉は新しい意味を帯び始めています。 結局のところ、人員の選択がどれほど慎重に行われようと、これらは人ですが、彼らは理想的ではなく、貪欲などの悪になりやすいです...



災害復旧ローカルインフラストラクチャ



災害復旧は、柔軟性とスケーラビリティとともに、クラウドを支持する主な論点の1つです。 クラウドテクノロジーを使用すると、メインデータセンターから数百キロメートルに位置するインフラストラクチャのコピーを非常に簡単に作成できます。 現在、バックアップシステムのような同様のソリューションがありますか？ あなたが長年彼らなしでいたならば、あなたは本当に彼らを必要としますか？



インターネット接続



クラウドテクノロジーの実装を成功させるための重要な条件の1つは、高速で信頼性の高いインターネット接続の可用性であり、速度と帯域幅の増加が必須です。 残念ながら、多くの人がこの要因を過小評価しており、プロジェクトの実装が完全に成功するわけではありません。



価格



正直に言って、クラウド内の仮想マシンはそれほど安くはありません。 （特にディザスタリカバリの点で）クラウドに間違いなくある追加の利点を削除すると、クラウドのVMのコストはローカルデータセンターのVMのコストを大幅に上回ります。



おわりに



オンプレミス-これは選択の自由があるとき、これはあなたのコスチュームがあなただけに合わせられ、あなたに完全に合うときです。 クラウドは、第一に、高品質の既製スーツの膨大な品揃えからもあなたに適したスーツを見つけようとする試みです。 クラウドプロバイダーは、特別な条件なしで、すべての顧客にまったく同じサービスを提供する義務を負います。



クラウドへの移行が、綿密で厳密な計画を必要とするかなり複雑なプロセスである場合、インフラストラクチャをクラウドからオンプレミスに戻すことは、さらに複雑で信じられないほど高価なタスクです。 クラウドへの完全な移行は、片道チケットです。



ところで、主要なクラウドサービスプロバイダーのマーケティングのレトリックがどのように変わったかに気づきましたか？ Cloud Firstのアイデアや要望は、Hybrid Cloud Firstにスムーズに変換されました...
