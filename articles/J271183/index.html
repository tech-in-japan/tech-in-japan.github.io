Chrome Dev Summit Live、2日目

こんにちは、Habr！ 今年のメインChromeイベントのオンラインブロードキャストの2日目： Google Chrome Dev Summit 2015がまもなく始まります！







夜の間に、地理学には何も起こりませんでした;パロアルトは、地球の反対側にあったので、そうです。 ライブ放送は最も都合の良い時間（モスクワ時間20:30から）には行われませんが、ライブで視聴できなくても、明日ビデオが利用可能になります。





初日-プログレッシブWebアプリ：2015年11月17〜18日



初日について
Chrome Dev Summitの初日は、最新の高速で便利なWebアプリケーションに捧げられました。 Google Chromeエンジニアは、高速でスムーズなダウンロードの作成、不安定な（モバイル）接続の状況での信頼性の高い操作、ユーザーとの適切なやり取りの経験を共有しました。 すべてのワークショップと講義は、YouTubeで録画として利用できます。




ビデオへのリンクがある初日プログラムについての投稿はこちらにあります 。



2日目-RAIL、パフォーマンス、および開発ツール：2015年11月18〜19日



Chrome Dev Summitの2日目は、 RAIL （Responce、Animation、Idle、Load）の概念、パフォーマンス、およびDevToolsに専念します。 優れた結果を達成するには、できることとできないことのリストだけでは不十分です。 RAILアプローチにより、アプリケーションを高速化できます。 開発者は、今日の特別なケースと実際の製品でのRAILの使用について説明します。 :)



注：放送時間はモスクワ（UTC + 3、RTZ + 2）です。





20 :30-21: 00 >> 2015年のDevTools：幅広い視聴者のために働く





2015年の初めに、Chrome DevTools開発チームは、製品の視聴者がWebプログラマーだけでなく、デザイナー、UI / UXスペシャリスト、フロントエンド開発者であるという事実について考えました。 これにより、サービス開発の見通しを再確認できました。 現在、DevToolsはより多くのユーザーの関心を考慮しています。 Paul Bakausが、DevToolsでの作業を大幅に簡素化および高速化する今後の変更について説明します。








21:00-21:15 >> レール入門





パフォーマンスを改善するためのヒントは、良いことよりも害になる場合があります。追加の説明、ルールの例外、予期しない矛盾により、誰もが混乱する可能性があります。



「DOMが遅い」や「CSSアニメーションを選ぶ」などのフレーズは派手な見出しには適していますが、実際、このようなカテゴリの推奨事項には多くの落とし穴があります。 Paul IrishとPaul Lewisがパフォーマンスについての考えを共有し、アプリケーションパフォーマンスに対するグローバルで全体的なアプローチがいかに重要であるか、ユーザーに製品を最適化する努力に気づき感謝する方法を示します。








21:15-21:45 >> RAILのプロジェクトへの適用性について





RAILは、パフォーマンスの問題に対するChromeチームのアプローチです。 Google Chromeでは機能しますが、アプリはどうですか？ RAILを「そのまま」使用することは可能ですか、それともファイルで変更する必要がありますか？ Paul LewisがRAILコンセプトのツアーを提供し、そのメリットを紹介します。




21:45-22:30 >>コーヒーブレイク









22:30-23:00 >> RAILでパフォーマンスを監視





RAILトピックの続きでは、Paul Irishが、サイトのWeb開発、デバッグ、最適化にDevToolsとRAILを適用する方法を説明します。








23:00-23:30 >> V8-床へのガス！





WebアプリケーションがJavaScriptのパフォーマンスに依存している場合、ブラウザまたは非効率的なコードのどちらが原因ですか？ Seth Tompsonが、Javascriptのパフォーマンス、Google ChromeのV8 JavScriptエンジンの最新アップデートについて説明します。 プレゼンテーションの第2部では、効果的なコードの作成に専念し、現在および将来の最新のブラウザーで最適化する方法を説明します。








23:30-00:00 >> RAILを使用した品質と生産性の変化の評価





あなたはDevToolsの第一人者になり、RAILコンセプトを使用してアプリケーションを再設計しましたが、次に何をしますか？ これらの変更は、ユーザーの何千もの異なるデバイス上のアプリケーションにどのように影響しましたか？ Ilya Grigorikは、アプリケーションのパフォーマンスを測定、評価、改善するのに役立つ既存のAPI、新しいAPI、計画されているAPIについて説明します。
