Chocolatey Magic：Windows用のapt-getとyum



今日は魔法についてお話します。 そして、この魔法はWindowsシステム管理者の人生を一変させます。



現在、少なくとも一度はLinux環境にソフトウェアをインストールしなかった人はますます少なくなっています。 信じられないほど簡単です：真夜中の司令官（mc）をRH環境（RedHat Enterprise、CentOS、Fedoraなど）にインストールするには、「マジック」コマンドがいくつか必要です。 



# yum install mc


yumパッケージマネージャーは、 mcの最新バージョンがインストールされていること、およびパッケージの依存関係（ある場合）を確実にインストールします。 しかし、Windowsを自由に使用でき、そのようなものが必要な場合はどうでしょうか。 右、Linuxに切り替えるか読んでください！



私の「後見人」の下には、WindowsとLinuxマシンの異種ネットワーク（より簡単には動物園）があり、約2年間、Chocolateyを使用してWinの下にソフトウェアをインストールしています。 Chocolatey（ chocolatey.org 、 github.com/chocolatey ）は、apt-getまたはyumに非常に似ていますが、Windows専用のパッケージ管理システムです。



開発者のコ​​ンテキストで既にチョコレートのテーマを扱っているハブラーで、今日、システム管理者の観点からこの素晴らしいツールを見てみたいと思います。 ChocolateyはNuGetテクノロジーに基づいて動作し（Windows用ソフトウェアの開発者が積極的に使用しています）、Chocolateyの主な機能は、パッケージにインストールファイル（setup.msi、setup.exeなど）がほとんど含まれていないことです 。 次のように機能します。パッケージには、インターネット上の適切な場所から必要なバージョンのインストールファイルをダウンロードしてインストールするpowershellのインストーラースクリプトが含まれており、簡単にインストールすることができます。



チョコレートのインストール



Chocolateyマジックを使用する前に、コアをインストールする必要があります。 これを行うには、コマンドプロンプトで実行します。 



 @powershell -NoProfile -ExecutionPolicy unrestricted -Command "iex ((new-object net.webclient).DownloadString('https://chocolatey.org/install.ps1'))" && SET PATH=%PATH%;%systemdrive%\chocolatey\bin


このコマンドは、メインのchocolatey.org/install.ps1インストールスクリプトをダウンロードして実行し、必要な環境変数を設定します。 必要なものがすべて揃ったので、パッケージ管理システムをテストしてNodepad ++をインストールしましょう。 次のコマンドを実行するだけで十分です。 



 cinst notepadplusplus

誰もがchocolatey.orgにパッケージを追加できるため、パッケージは毎日追加されます 。主なことは、要件を満たしていることです 。



パッケージ公開の要件
  • 違法なプログラムを公開しないでください 。 世界のほとんどの国で違法なプログラムは、Chocolatey.orgに掲載することも禁止されています。 これは、著作権を侵害するソフトウェア、海賊版ソフトウェア、およびクラックにも適用されます。 これは、著作権侵害に使用されるプログラムにも適用されることに注意してください。
  • 配布権のないチョコレートでプログラムをパックしないでください 。 ソフトウェア配布ルールを確認し、それらに違反しないでください。
  • ウイルスやその他の有害なプログラムを公開しないでください 。
  • 他の人に役立つプログラムのみを公開します。 パッケージがこのカテゴリに属さない場合は、公開しないでください。
  • スパイウェアやアドウェアを公開しないでください。 組み込みのアドウェアまたはスパイウェア、またはその他の無関係なプログラムが付属するプログラムは公開できません。 通常、インストーラキーを使用して、関係のないプログラムをすべてインストールから除外できます。 そのようなプログラムの例はPDFCreatorおよびCCleanerです。
  • 既に公開されているプログラムを公開しないでください。 Chocolatey.orgの検索を使用します。 既存のパッケージを改善したい場合は、パッケージをサポートしている人に連絡するか、そのリポジトリにプルリクエストを送信してください。
  • 既に独自のパッケージがある場合は、パッケージに他のプログラムを含めないでください 。 パッケージに特定のプログラムが必要な場合は、既存のパッケージを依存関係として含める必要があります。
  • 依存関係を複数のパッケージに分離します 。 パッケージをできるだけ多くのパッケージに分割してください。 たとえば、プログラムにはオプションのモジュールが付属しています。 モジュールを共通パッケージに含める代わりに、モジュール用の追加パッケージを作成します。 このアイデアは、軽量パッケージを作成でき、競合の可能性を最小限に抑えるという理由で、Linuxパッケージで長い間使用されてきました。

Puppetを使用してインフラストラクチャの構成を管理しているので、時間と手間を大幅に節約できます。 Puppetにはリソースの素晴らしいコンセプトと、コンパートメント内で抽象的に考えるのに役立つ宣言スタイルがあります。これは、「Windowsで実行するコマンドとLinuxで実行するコマンド」のレベルではなく、「どのプログラムをこのサーバーに置くか」というレベルで抽象的に考えるのに役立ちます「。 Puppetには、以下を実行できるChocolateyプロバイダーがあります。 



 package { "7zip" : ensure => installed, }


または 



 package { "notepadplusplus" : ensure => 1.0, }


他のすべてはPuppetとChocolateyによって処理されます。 私を信じて、これはmsiファイルからインストールするよりもはるかに便利です.msiファイルはまだどこかでホストする必要があり、バージョンを更新するときに（まだ実行する必要がある）、古いバージョンも保存され、何も壊れないことを確認してください。

Chocolateyを使用してワークステーションを管理する2つの方法を検討することをお勧めします。

ChocolateyGUIは、Chocolateyパッケージ管理システムのグラフィカルインターフェイスです。 リポジトリの現在の状態、およびローカルにインストールされたパッケージのステータスを確認する便利な方法。 何らかの理由で、私は以前のバージョンのシナプスまたは適性さえも思い出しました。 それは合理的にうまく機能します。 ところで、コマンドラインからインストールできます： 



 cinst ChocolateyGUI








Windowsポストインストール（WPI）
さらに進んで、 WPIインターフェイスを使用して、Chocolateyコマンドを実行するパッケージを簡単に選択できます。 WPIを使用すると、プログラムを簡単にカテゴリにグループ化したり、テンプレートやインストールキットを作成したりできます。

このソリューションは、USB-HDDをソースとして使用する絶対的な代替手段になるとは限りませんが、可能性のあるすべてのコンポーネントをChocolateyリポジトリの同じコンポーネントに置き換えると、フルセットのソフトウェア（Photoshop、Office、3D Max with Archicad）で画像（フォルダー）そこにありますか？）およびWPIシェル（および、Google Chrome、Notepad ++、Avastなどの「軽量」プログラムを配置するためのすべて）。



たとえば、一元化されたストレージなしで散らばったマシン群をサポートする受信管理者の場合、次のようなテンプレートのリストを用意すると便利です。

  • 事業内容
