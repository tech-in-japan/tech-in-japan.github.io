Node.jsでMySQLおよびMongoDBの管理パネルを作成します

ノードに「phpMyAdmin」（Web GUIの読み取り）が必要です



共通のDBMSを管理するためのユニバーサルWebインターフェースがないため、 Node.jsの開発が多少複雑になり、隣のインフラストラクチャで別のWebサーバーと別の言語をデプロイすることは気になりません。 別のサーバーまたは職場のコンピューターから接続してポートを開いてデータベースを管理するのは不便であり、セキュリティ上の考慮事項があります。 そのため、このようなツールをWebアプリケーションImpressのプラットフォームに含めることにしました。Impressは発表されましたが 、これについては少し書いており、オープンソースで一般向けに利用できます。 これは、Node.jsと組み合わせて使用​​されることが多いDBMS用のシンプルで便利な統合インターフェイスを実装し、迅速な展開（フォルダーをコピーするだけ）と環境からの独立を実現するという考え方です。 ベータ版はすでにMySQL 、 MongoDBをサポートしており、まもなくPostgreSQLとOracleにキューが届きます 。



DBMI機能



データベースを使用したすべての操作に対して、足場が使用されます。 データベース構造に関するメタデータの動的な受信またはDBMS構造のイントロスペクションに基づく、インターフェース、クエリ、およびすべてのデータ処理操作の構築。 以下の図に、インターフェースを示します。 ツリーは3レベルで、最初のレベルはデータベース接続（もちろん、2つのタイプです）、2番目のレベルはデータベース、3番目のレベルはコレクション（Mongoの場合）、テーブル（MySQLの場合）です。 ツリーのノードで、マウスの右ボタンでコンテキストメニューを呼び出すことができます。







機能は次のとおりです。

  • テーブルとコレクションを表示します。 これはMySQLの自然な編集方法ですが、MongoDBではグリッドを使用しました。列はコレクション内のすべてのレコードから収集されたフィールドです。このような編集はJSON、ツリー、プロパティインスペクターを編集するよりも便利だと思います。
  • フィールドでフィルターします。 グリッドの見出しにフィルターの行があります。MongoDBの場合は厳密な一致によってのみ機能し、MySQLの場合は完全な一致に加えて、フィルター条件の構文もあります。引用符なしで入力します。つまり 「5に等しくない」は「<> 5」、範囲「5..10」として書き込まれます-BETWEENに変換され、文字「*」のパターン（空のシーケンスでもSQL LIKE式では「％」に変換されます） 「？」 （SQL LIKE式で任意の文字を「_」に変換する必要があります）、たとえば、「* str？」 （このような文字はユーザーにとってより馴染みのあるものです）、セット「（1,2,4,10,11）」の1つの値がSQL IN式に変換されます。
  • レコードを挿入します。 これまでのところMySQLでのみ機能します。 グリッドの最後の空行が挿入に使用されます。1つのフィールドを編集した後、すでにINSERTを実行でき、レコードがすぐに追加され、そのIDがPRIMARY KEYの列に表示され、UPDATEを介してさらに編集が行われますが、「NOT NULL」制限に複数が必要な場合必要なフィールドがすべて入力された後にのみINSERTが発生します）。
  • テーブルとコレクション内のレコードの編集 （常にMongoで機能しますが、MySQLの場合、コンポジット主キー、つまりレコードIDがない場合にのみ、後でコンポジット主キー、コンポジットおよび非コンポジットのユニークインデックスがサポートされます。これが何も見つからなかったため、UPDATEステートメントのWHEREに、編集中のレコードのすべてのフィールドを配置します。
  • テーブルおよびコレクションからレコードを削除します 。 これはMongoで機能しますが、MySQLでは非コンポジットPRIMARY KEYでも機能します。
  • ログSQLステートメント。 当然、リレーショナルDBMSでのみサポートされます。 おそらく、MongoDBの場合、プロジェクトにコピーして貼り付けるためのコードをJavaScriptログに出力します。
  • データベースの作成と削除。 MySQLでも機能しますが、Mongoには機能があります。データベースは、少なくとも1つのコレクションをすぐに作成した場合にのみ作成されます。そうしないと、画面が更新されるとデータベースが消えます。これらはDBMS機能です。 。
  • データベースの名前を変更します。 Mongoのように機能しますが、MySQLではまだ機能しません。ご存じのように、RENAME DATABASEがMySQLに導入されて削除され、データベースの名前を変更するには新しい名前で新しいデータベースを作成し、そこにすべてのテーブルをコピーしてからユーザーとアクセス権をコピーする必要があるためですおそらく、他のいくつかのことを考慮する必要がありますが、今のところ到達していません。
  • コレクションとテーブルの作成。 これまでのところ、Mongoでのみ機能しますが、まもなくMySQLのテーブルデザイナーを追加します。
  • コレクションとテーブルを削除します。 MySQLとMongoの両方で機能しますが、テーブル間の接続（FOREIGN KEYS）はMySQLでの削除を防ぐことができます。この場合、削除は失敗し、テーブルはツリーに再表示されます。
  • コレクションとテーブルの名前を変更します。 MySQLとMongoの両方で機能します。


インストールとセットアップ



完全なセットアップ手順は次のとおりです（これはセキュリティのためですが、ほとんどの場合、大幅に削減されます）。

1.プロジェクトディレクトリを作成し、 npmリポジトリからImpressをインストールします。 

$ npm install impress


2.フォルダー/ node_modules / impress / examples / copyContentToProjectFolderからすべてのコンテンツをプロジェクトのルートディレクトリにコピーします。

3. config.jsファイルで、「databases」セクションで必要なデータベース（エンドポイントの文字列化）を指定します。

4. /sites/localhost/dbmi/access.jsファイルにアクセス設定があります 

 module.exports = { guests: true, //       logged: true, //       http: true, //     http https: true, //     https groups: [] //     ( ) }


5.パスワードで管理パネルを閉じるには、「node setup.js」コマンドを使用してユーザーデータベースを作成する必要があります。 そして、「node server.js」システムを起動します。 127.0.0.1に移動して登録します（右上の[アカウントの作成]）。 その後、/ sites / localhost / api / auth / register.jsonにaccess.jsファイルを追加し、guests = falseを設定することにより、登録機能を閉じることができます。

6. HTTPS経由でデータベースにアクセスする場合は、server.keyファイルとserver.cerファイルをプロジェクトのルートに配置し、config.jsのパラメーターservers.www.protocol = "https"を調整します。

7.さて、 127.0.0.1 / dbmiまたは127.0.0.1を通過し、左側のメニューで、例の危険性、アイテム「DB Management Interface」を選択します。



DBMI開発計画



現在、DBMIモジュール全体が70kbを占めています。その半分はcssとhtmlで構成され、約4分の1はサーバー側のjs、もう1つの四半期はクライアント側です。 何かを修正または追加したい場合、コードを理解することはそれほど重要ではないと思います。 また、サイズが小さいのは、（当社の）Impressプラットフォームが、URLのルーティング、テンプレートの処理、データベース、特にMySQL（ データアクセス 方法 、 メソッド）を操作するための膨大な数の補助機能など、ほとんどのインフラストラクチャ作業を行うためですイントロスペクション 、 クエリ生成メソッドなど）。 近い将来に追加する予定のもの：

  • 任意のSQLクエリの実行。 含む CREATE TABLE、DROP ...、ALTER ...など これらのクエリの履歴を保存し、将来クエリクエリを呼び出す。
  • ツリーの第4レベルの名前付きノードとしてグリッドにフィルターを保存します。 また、指定した名前で保存できるカスタムSQLクエリを記述することは、ツリーの4番目のレベルのようであり、その結果の操作も同じグリッド内にあります。
  • テーブル間のリレーションを介した移行 ：「親」テーブルまたは「子」への移行（1対多の関係）。
  • 足場と同様に、グリッドと同様に、レコードを編集および貼り付けるためのフォームの生成 。
  • IDではなく、 ディレクトリからの値の表示 。 また、それに応じて、ドロップダウンリストの使用を編集してディレクトリを表示する場合。
  • JSONメタ記述からテーブルを生成します 。/node_modules / impress / schemasの例を参照してください。 生成はすでに機能していますが、まだDBMIに接続されていないため、次を実行して生成の結果を確認できます。
