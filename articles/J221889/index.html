IBeaconプラットフォーム：アプリケーション構築のためのターンキーソリューション





前回の記事で、 iBeaconを選択して注文した経験を共有しました。これは、スマートフォンで検出できる小さなスタンドアロンのBluetooth LEビーコンです。 スマートフォン所有者の正確な位置（数センチメートルから数十メートル）を特定することに基づいて、モバイルアプリケーションを作成する新しい機会があります。



現時点での基礎は、ユーザーにとって価値のあるソリューションの検索です。 最も可能性の高いエポギーの専門家は、非接触型決済システムを提案しています。



テクノロジーを「いじって」、同時に開発に関与したくなかった。 これを行うために、プログラミングなしでiBeaconアプリケーションを構築できる既製のソリューションを見つけることにしました。



この記事で説明するいくつかのプラットフォームにアクセスしてテストしました。







屋内： http : //indoo.rs







屋内ナビゲーション用のプラットフォーム。 スペースのモデリング、ビーコンの配置、キャリブレーションなどのためのツールが付属しています。 デモアプリケーションでは、シミュレーション結果を確認できます。







LabWerk： http ://labwerk.com







主な製品は、mAppミュージアム向けのソリューションです。



博物館オブジェクトに関する記事を作成し、通知ルールを設定するためのCMS（コンテンツ管理システム）があります。 博物館の特定の地域に関連付けられた調査を作成することができます。 多言語がサポートされています。 訪問者についての統計が収集されます。



アプリケーションは、近づいたときに芸術品に関する記事を表示します。 訪問者の現在の位置を表示する博物館の地図があります。







ShopJoy： http ://shopjoy.se







このプラットフォームには、CMSと統合されたデモアプリケーションが含まれており、 App StoreおよびGoogle Playからダウンロードできます。 アプリケーションはスウェーデン語でオリジナルです。 英語を追加する予定です。 テストの過程で、彼らは私のためにロシア語のローカライズ版を集めました。







プッシュ通知を設定し、プッシュ通知用に設定されたWebページをアプリケーションに組み込まれたブラウザーに表示することができます。 プッシュ通知の場合、メッセージテキスト、応答半径を設定できます：即時（電話とビーコンの直接接触）、近距離（最大2メートル）、遠距離（2メートル以上）、キャンペーン期間。 また、アプリケーションへの登録時に指定された年齢、性別、ユーザーの興味によって広告をターゲティングすることもできます。







プラットフォームにアクセスするには、Estimoteの3つの灯台を含む160ユーロのShopJoyデモパックを購入する必要があります。



SmartBeacon： http ://smartbeacon.eu







その機能のプラットフォームは、ShopJoyに非常に似ています。 （UUIDを変更せずに）ビーコンを追加し、応答半径、プロモーションコンテンツ（画像、ビデオ、組み込みブラウザー）のいくつかの形式でプッシュ通知を構成できます。 CMSには統計の視覚化があります。 同社は、自社生産のビーコンも販売しており、その特徴は大容量バッテリーです。







LocalSocial： https ://www.mylocalsocial.com







私がテストしたすべてのプラットフォームの中で、LocalSocialには最も豊富な機能があります。



CMSを使用すると、地理的エリア、プロモーションおよびそれらのプロモーションを構成し、ビーコンを登録して特定の地域にリンクし、地域に入るときにプッシュ通知を構成できます。 ロイヤリティシステムも実装されているため、ロイヤリティポイントを獲得してプロモーションコードを発行できます。







デモアプリケーションは、CMSに登録されている地理的ゾーンとローカルゾーンを決定し、それらを対象としたグラフィック、テキスト、およびプロモーションコンテンツを表示します。



残念ながら、ビーコンへの応答の半径を調整する方法はありませんが、次のバージョンのいずれかにこの機能を実装することを約束します。 彼らは現在、新しい形式のプロモーションコンテンツにも取り組んでおり、組み込みのブラウザで特定のWebサイトを表示するように拡張しています。



Pushmote： https ://www.pushmote.com







このプラットフォームでは、ビーコンを登録し、プッシュ通知を構成し、特定のWebページまたは静的なHTMLコードでブラウザーを開くことができます。 チームは、Rich Mediaコンテンツを表示する応答範囲設定（ShopJoyと同様）を実装する予定です。







LightHouse： http : //lighthousebeacon.com.au







デモアプリケーションは、プラットフォーム開発者からベータ版にアクセスしてインストールできます。 独自のアプリケーションを構築するためのAPI / SDKが提供されています。



ここで見ることができるCMSのデモ版。 CMSを使用すると、アプリケーションを搭載した電話機が可視領域に表示されたときに、ビーコンを登録し、プッシュ通知を構成できます。 ゾーンへの訪問に関する統計が収集されます。







チームは主にCMSシステムの開発に重点を置いており、現在、携帯電話向けのデモアプリケーションは提供していません。 ただし、独自のアプリケーションをプラットフォームに統合するためのドキュメントがあります。

結論



既存のプラットフォームのいくつかを見て、自分にとって最も興味深い質問に答えることができませんでした。エンドユーザーにとっての本当の価値は何ですか？ なぜ彼は新しいアプリケーションをインストールするのですか？



また、小売、美術館、ナビゲーションなど、特定のビジネスセグメントで完成したと見なされる単一のソリューションを見つけることができませんでした。 すべてのプラットフォームは、さらに開発する必要のある基本的なデモ機能を提供します。



独自のCMSを開発する際のエネルギーを節約するために、リストされたプラットフォームの1つにAPI / SDKレベルで接続するオプションが機能している可能性があります。 ただし、特定のタスクがある場合、独自のアプリケーションを開発することは避けられません。
