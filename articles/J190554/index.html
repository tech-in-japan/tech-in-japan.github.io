AndroidでAircrack-ngとReaverを起動する

何もする必要がない場合、またはAndroidに空爆と刈り取り機をインストールする方法。

Linuxインストーラーを使用してAndroidにkali-linuxをインストールする必要があるかと考えましたが、気が変わって、電話にReaverとAircrackをインストールすることにしました。 しかし、その後、WiFiをモニターモードに転送する方法に関する質問が発生しました。







主にXDA-Developersに関するandroid'omでのreaver'aおよびaircrack'aの打ち上げに関するインターネット上の情報はほとんどありません。 XDAから読み取った内容に基づいて、WiFiをモニターモードに切り替えるには、スマートフォンでカスタムWiFiドライバーをコンパイル/インストールする必要があります。アンドロイド用にコンパイルされたreaverが見つからない限り。 しかし、aircrack'omには問題があり、私はそれを長時間探して退屈に探しましたが、armelの静的バージョンを見つけました。

フロアが完成したので、モニターモードを処理する必要があります。

繰り返しになりますが、これについてはXDAに関する記事がありましたが、カーネルにもカスタムドライバーをインストールする必要がありました。 私はそれが多すぎると思った、それなしで何らかの形でうまくいくと思った。 電話で端末エミュレーターを起動し、チョップし始めました。 その結果、ドライバーとアダプターの両方がモニターモードをサポートしていることがわかりましたが、アダプターをこのまさにモードに変えるのはそれほど簡単ではありませんでした。 しかし、良いグーグルは私がこの問題に対処するのを助けました。

これでほぼすべての準備が整いました。 airodump-ngを実行します。







奇跡、うまくいきます。

しかし、ハチミツの各樽には、軟膏にハエがあります。







上記のスクリーンショットから、reaver'omキーの検索速度はキーごとに7秒であることがわかりますが、これはもちろん残念です。 彼がルーターのパスワードを取得するまで待つのは不合理なので、私はしませんでした。

テストはSE Xperia Miniでしたが、より強力な最新の携帯電話では、プロセスがより速くなると思います。



指示：



突然誰かが便利になります。



警告！ 著者は、結果、あなたがすることすべてがあなた自身の危険とリスクで行うことに対して責任を負いません。



私がしたことはすべて、Android ICS 4.0.4で行いました（これが他のユーザーでも動作するかどうかはわかりませんが、4.xxで動作するはずです）。


必要なもの：


  1. Android'omを搭載した携帯電話（この場合、SE Xperia Mini、512 RAM、1 GHz、 Android ICS 4.0.4 ）;
  2. ルースは正しい。
  3. ターミナルエミュレータまたは同等のもの。
  4. Android用にコンパイルされたReaverとaircrack（ ここからダウンロードするか、独自にビルドできます）;
  5. ルーター（この場合、モデルのアカウントへのTP-Link-私は知りません）;
  6. ADBがインストールされたコンピューターは必要ありませんが、便利です。
  7. そして最も重要なこと-それらなしで、まっすぐな腕-方法はありません。


それでは始めましょう


次の2つのオプションがあります。

  1. ユーティリティをSDカードにコピーし、実行のために再マウントします。
  2. 記録用にシステムを再マウントし、そこにドロップします。


2番目の方法を選択しました。



次に、モニターモードでWiFiを開始する必要があります（WiFiモニターの実行中は、少し不安定になります）。

まず、モニターインスタンスを作成する必要があります。そのためには、ターミナルに移動して次のように記述します。 

su iw phy0 interface add mon0 type monitor


mon0は他の名前に置き換えることができます


その結果、次のような結果が得られます。







netcfgと入力すると、次のように表示されます。







wlan0とmon0はまさに必要なものです。

モニターが機能するには、電話設定でWiFiをオンにする必要があります


少し残った。



airodumpを起動する 


 netcfg mon0 up cd ////airodump ./airodump -i mon0








停止するには、Ctrl + Cを押す必要があります（音量を下げると、Androidターミナルエミュレーターで英語レイアウトのCが有効になります）。


リーバーを実行する 


 cd ////reaver ./reaver -i mon0 -b 11:22:33:44:55:66 -vv


11：22：33：44：55：66ご理解のとおり、必要なネットワークをBSSIDに置き換える必要があります






停止するには、Ctrl + Cを押す必要があります（音量を下げると、Androidターミナルエミュレーターで英語レイアウトのCが有効になります）。


あとがき


もちろん、検索はゆっくりと行われますが、おそらくより強力な新しい携帯電話では、処理が速くなります。

略奪者に加えて、airbase-ng、aircrack-ng、airdecap-ng、airdecloak-ng、aireplay-ng、airodump-ng、airserv-ng、airtun-ng、besside-ng、besside-ng-crawler、packetforgeがあります-ng、ウォッシュは重宝します。



私もあなたに警告したい...

  • ... reaverは少し不安定で、ネットワークに接続したくない場合があります。
  • ...モニターをオフにする必要があります。そうしないと、ネットワークへの接続に問題が発生します。 

     su netcfg mon0 down


  • ...電話を再起動した後、mon0モニターは削除され、再作成する必要があります。
