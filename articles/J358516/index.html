AutoCAD用ターミナルサーバー

私は長い間CADシステムのターミナルサーバーに興味があり、この記事はインターネットでの研究の結果です。 この記事では、特にAutoCADのターミナルサーバーについて説明します。



AutoCADは、RAM、ビデオカード、強力なプロセッサ、高速ディスクなど、すべてのハードウェアリソースを必要とする、かなりリソースを消費するアプリケーションです。 そしてすぐに、このアイデアは、自立型の強力なPCが最良の選択肢であることを示唆しています。 しかし、このアプリケーション用のターミナルサーバーを展開して、それを実行しようとしました。



したがって、クールなサーバーの構成：



  1. Supermicroプラットフォーム（既に扱いにくいが、それでもかなり速い）;
  2. 28個の物理コアと56個のハイパースレッディング対応コアを備えた2個のE5-2260 v4プロセッサー。
  3. ハイパーバイザー-ホストオペレーティングシステムCitrix Xenserver 7.0;
  4. ゲストオペレーティングシステムサーバー2016 Standart。
  5. 10,000 rpmの回転速度で、それぞれ1 TBの10-4 HDDのRAIDアレイ。
  6. イーサネット1 Gb / sek;
  7. 最も重要なコンポーネントは、2 Tesla m60 16 Gbビデオアクセラレータです。


実際、サーバーは、Windows 10で仮想マシンを作成し、Citrix Receiverを介してユーザーに配信するように設計されていました。 仮想マシンは存在しますが、私は多くのユーザー用に多くの仮想マシンを作成したくはありませんでしたが、1つ作成したかったのです。



私はcitrix受信機を介して配信するターミナルサーバーを作成する方法を探していました-サーバーOS用のVDAforHDX 3dProエージェントが見つかりませんでした。誰かが方法と内容を知っている場合は、コメントにリンクをドロップしてください。そのため、RDP経由の配信でサーバーを作成しました。



順番に始めましょう：



  1. Citrix Xendesctopで仮想マシンを作成する
  2. Server 2016 Standart仮想マシンのオペレーティングシステムのインストール
  3. 仮想マシンOSのアップデートをインストールします。
  4. 仮想マシンへのCitrixゲストサービスのインストール-その後、シャットダウンと再起動の項目が仮想マシンのコンテキストメニューに表示されます。これにより、Xenserverから仮想マシンを正しくシャットダウンおよび再起動できます（以前は強制シャットダウンと強制再起動のみでした）。
  5. パススルー方式を使用して、Teslam60ビデオカードを仮想マシンに転送します。






さらに、 記事によると、両方のドライバーをインストールしますが、仮想マシンにはグリッドドライバーしかインストールできないと思います。 それ以降のドライバーは、この記事からこれ以上何もしません。



次に、nvidiaコントロールパネルに移動して、3dパラメータでデフォルトのTesla m60ビデオカードを選択する必要があります。ちなみに、その後、デフォルトのビデオカードを選択する機能を失い、常に次のように表示されます。







次に、ターミナルサーバーの役割をインストールし、次のグループポリシーを有効にする必要があります。 コンピューターの構成-管理用テンプレート-Windowsコンポーネント-リモートデスクトップサービス-リモートデスクトップセッションホスト-リモートセッション環境-すべてのリモートデスクトップサービスセッションにデフォルトのハードウェアグラフィックアダプターを使用する



これを実現するのに役立つ記事は次のとおりです。リモートデスクトップサービスでのGPU使用率の測定。



ターミナルサーバーを作成しているため、ユーザーがアプリケーションの1つで作業する結果として一時ファイルが作成されるため、ファイルサブシステムの負荷を最小限に抑えるために、Ramディスクをインストールして一時ファイルをそこに転送できます-システム設定-追加システムパラメータ-[詳細設定]タブ-ボタン環境変数 ：

再起動後、サーバーは動作可能な状態になり、AutoCADを実行すると、 ハードウェアアクセラレーションがオンになっていることがわかります。



すべてのテストは、Nvidiaコントロールパネルの[ライセンス]セクションのGridVirtualAPPライセンスを使用して実行されました。 QuadroDataCenterWorkstationを使用するには、ライセンスサーバーをインストールし、適切なライセンスを購入する必要があります。 それらは私たちから購入されました-しかし、テストはGridVirtualaPPライセンスの下で実行されました。



それでは、テスト結果に移りましょう。 まず、すでに述べた記事によると、WEbGLのテストを含めました（例：http: //fishgl.comで起動）。



1つのクライアントで起動し、50〜70 fpsのうれしそうなビデオを見ました-ビデオカードは動作しています。

3時に起動し、fpsは25から60にジャンプし始めました-平均35-40です-品質は著しく落ち着いた小さな遅れが現れ始めました。



これは、1人と3人のユーザーのリソース消費の違いを示すGPU輻輳のグラフです-負荷が増加し始めます-これは、さらに2人のユーザーがieサイトにfishgl.comを含める時点です。







さらに3をオンにすると、すべてが悪化し、fpsは最大10になり、ビデオカードプロセッサは最大100％







結果の結果、ターミナルサーバーでwebglテクノロジを実行しない方がよいことに気付きました。一般的に、ローカルブラウザからインターネットへのアクセスを提供できる場合。 まず、ユーザーは10個のブラウザーで100個のタブを開くことができますが、ここでは確かにサーバーがそれに耐えることができません。 そして第二に、インターネットがなければ、ターミナルサーバーはより完全な状態になります。

これから、パススルー方式の1番目のTeslaM60が転送された仮想マシン上のターミナルサーバーは、最大15ユーザー（できればサーバーごとに約12-13）のrdpプロトコルを使用して自動カセットで動作するために使用でき、2dおよび3dで描画できると結論付けることができます（小）、ただしレンダリングなし 。



使用場所： AutoCADでの作業方法を学習する場合-たとえば、2dのみで描画するオートメーションエンジニアや技術者向けに、レンダリング（またはその他の高度な技術）を必要としない分野で。 ビルダー、メカニック、およびその他の非常に要求の厳しいAutoCADのパーソナリティは、文句を言い、激しく吐き出します。 しかし、あなたがそのようなサーバーを持ち、要求の厳しい人格が非常に古いコンピューターを持っている場合、彼らはレンダリングを使用せずにそのようなサーバーで描くことを喜んでいるでしょう。なぜなら、描画のレンダリングは規律のためのクレジットを渡す直前の最終段階であり、この瞬間は時間内に配布できるからです、たとえば、順番に実行します。 ただし、これには、ドラフトプロセスでコンピュータークラスを管理する適切な教師または実験助手が必要です。

ターミナルサーバーを容易にするもの：まず、インストール時間は、15台のコンピューターを再インストールし、更新プログラムをダウンロードして各auto-cadを適用するのではなく、インストールと構成の日です。 第二に、15台のコンピューターではなく、1台のサーバーの管理。



短所：まず、1台のサーバーが1つの障害ポイントであり、サーバーが停止し、トレーニングプロセスが停止しました。 したがって、2番目のそのようなサーバーが必要ですが、それは仮想サーバーであるため、2つのホストサーバーがあり、仮想マシンのバックアップが作成され、1つが失敗した場合、2番目のバックアップから仮想マシンをすぐに起動します。



第二に、多くのユーザーが一度にサーバーリソースを使用します。多くのことを起動する賢いユーザーがいる場合、リソースはほとんどなく、誰でも使用できます。 ただし、これはサーバー管理者と学生グループの教師の両方が監視する必要があります。



第三に、rdpはアフリカでもrdpです。 ネットワークに障害が発生し、動作することが不可能になり、rdpプロトコルの最適化により確かに使用品質が大幅に向上しますが、この技術の動作に関してはまだいくつかの不満があり、すべてのアプリケーションが正常に動作するわけではありません。

なぜrdpを選んだのか- ここでは 、AutoCADがこのプロトコルでの作業をサポートしていることを読んだためです。 しかし、Citrix Receiverを使用する方が良いですが、残念ながら、CADアプリケーションを備えたターミナルサーバーでCitrix XenDesktopを使用する方法に関するマニュアルは見つかりませんでした。見つかったマニュアルはすべて、インストール中のwin7、8または10の個々の仮想マシン専用ですVDAエージェントのタイプが示されています-HDX 3dproの場合、サーバーOSにはそのようなエージェントはなく、VDAのみがあります。 ビデオカードを搭載した仮想マシン用のVDAforHDX 3dProエージェント。 Citrix XenDesktopを使用して同じターミナルサーバーを作成する方法を知っている場合、およびサーバーOSのVDAforHDX 3dProの場所を知っている場合は、コメントに記入してください。
