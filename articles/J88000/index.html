UbuntaからUbuntaをインストールする（新しいバージョンの更新計画）

既にインストールされているubuntaから（コンピューターで作業している妻に干渉することなく）作業コンピューターにubuntuをインストールする必要がありました。 インストールは、FreeNXを介してラップトップからコンピューターに接続することにより実行されました。



そのため、ラップトップでソファに横たわり、FreeNXを介してコンピューターに接続して、次の操作を行います。



オプション1：



1）ubuntu-9.10-alternate-i386.isoをダウンロードします



2） sudo apt-get install kvm







3） sudo kvm -m 512 -cdrom ubuntu-9.10-alternate-i386.iso /dev/sdb



（ここで/ dev / sdbはOSをインストールする2番目のネジです）



4）その後、インストールが行われ、ブートローダーが/ dev / sdbにインストールされます。



5）VMWarePlayerで空の仮想マシンを作成し、それに物理ディスク/ dev / sdbを追加して、新しいubuntuを起動できます。



6）VMWareで快適に作業するために、WMVareツールをインストールできます。



7）これで、この新しい小さなubuntuを設定し、パッケージを再インストール（または不要なパッケージを削除）できます。



8）新しいUbuntuを/ dev / sdaのブートローダーに追加します。



オプション2：



1）ubuntu-9.10-alternate-i386.isoをダウンロードします



2）VMWarePlayerで空の仮想マシンを作成し、物理ディスク/ dev / sdbとイメージubuntu-9.10-alternate-i386.isoを追加します。



3）この仮想マシンを実行します。 ubuntaインストーラーが起動します。



4）ブートローダーは/ dev / sdbにインストールされます。



5）オプション番号1のパラグラフ6、7、8に従います。



これで、稼働中のコンピューターを再起動し、新しいOSを選択してEnterを押すことができます。



合計：

したがって、システムには2つのディスクがあります。 sdaでは、現在のオペレーティングシステム。 ディストリビューションの新しいバージョンが出てきて、それをsdbに置き、設定します。 一方、コンピューターはsdaを備えたシステムによって制御されます。 次に、最も難しいことは、sdbからsdaにイメージをコピーすることです。 そして、sdaにはまったく新しいシステムがあります。 sdbシステムのコピーを実験に使用して、ubuntaの新しいバージョンを待ってから、手順を繰り返します。



PS：sdcにある/ homeパーティション。
