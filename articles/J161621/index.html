Google Maps API v2-サービスの終了

本日、Google Maps APIのバージョン2のサービス期間の終了に関するGoogle Maps API Developer Relationsからの手紙を受け取りました。

カットの下-翻訳と反射





Google Maps JavaScript API v2キーに関連付けられたメールアドレスが次のドメインで使用されているため、ご連絡します。

°％domain_name％

ご承知のとおり、JavaScript Maps APIバージョン2の3年間の廃止期間は2013年5月で終了します。 この日付以降、APIはサポートされなくなり、機能が期待どおりに機能しなくなる可能性があります。 ある時点で、V2のカードが完全に機能しなくなる場合があります。

幸いなことに、Google Maps Javascript API v3はv2よりも信頼性が高く、機能も豊富であり、大多数のサイトではV3への移行に問題はありません。 したがって、移行ガイドを確認し、2013年5月までに移行することを強くお勧めします。





もちろん、2、3のマーカーとマップのみを使用するほとんどのサイトでは、移行の問題は、小さなマニュアルを読んで、数十行のコードを変更するだけです。

google apiを使用して多くのマーカーを配置し、マップを操作するための幅広い機能を備えたサイトに関しては、これは大きな問題になります。

私の経験が示すように、そのようなサイトは長い間書かれてきましたが、これは大きな問題です。 そのようなサイトを書き直し、その動作を理解するためだけに1か月以上を費やしました。 そのようなアーキテクチャの欠如、PHPクラス、フレームワークの使用、1つのファイルに含まれるhtml + php + javascriptのハッシュは単なる花です。 ベリーは5〜7年前の未使用のライブラリで使用されていました。もちろん、functions.phpの途中で直接実行可能なコードの行の形で「イースターエッグ」が使用されていました。

その結果、カードのすべての機能を書き直し、残りの古いコードとドッキングする必要がありました。



V3への移行は長くて苦痛なプロセスになる可能性があるため、V3への移行を遅らせないように、V2のサイトの所有者にアドバイスしたいと思います。

結局のところ、サイトが単にある時点で動作を停止することを誰も望んでいません。
