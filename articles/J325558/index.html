「Cloudbox」は、最新のIaaSサービスのプラットフォームとしてLenovoサーバ​​ーを選択しました

2016年12月、クライアントのプライベートクラウドとITインフラストラクチャを独自のプラットフォームおよびMicrosoftのWindows Azureプラットフォームで整理および展開することを専門とするMONTグループのメンバーであるOblakotekaは、IaaSサービス「Virtual Infrastructure on Windows Server2016。」









2012年以来、同社はMicrosoft Hyper-V 2012 R2に基づくIaaSプラットフォームを商用モードで展開して運用していることに注意してください-2つのクラスター（各データセンターに1つ）とN + 2モードの冗長コンピューティングノード。









Windows Server 2016プラットフォームの新しいIaaSサービスのハードウェアコンポーネントは、Lenovo x3650 M5サーバーとMellanoxネットワーク機器です。



Lenovoの高性能サーバーハードウェアは、Windows Server 2016ソフトウェアプラットフォームで使用するために特別にテストおよび準備されました。高性能を確保するために、ストレージスペースダイレクトテクノロジーに基づくフラッシュアレイを備えたハイパーコンバージド環境にプラットフォームを展開し、ストレージレイテンシを最小化しました。



これは、最新の仮想化プラットフォームであるMicrosoft Windows Server 2016でのロシアでのIaaSサービスの最初の実装の1つです。このサービスは、ロシアのデータセンターからのMicrosoft COSN Russiaイニシアチブの一部として提供され、仮想リソースのセキュリティとパフォーマンスを確保するユニークな機会を提供します。 これは、Microsoft Azureユーザーになじみのある使い慣れたWindows Azure Packインターフェースを通じて管理されます。









ハイパーコンバージドプラットフォーム：仮想マシンをホストするためのサーバークラスター。



特に、シールドVMテクノロジーは、プロバイダー管理者を含む仮想マシンの信頼性の高いデータ保護を提供します。 コンテナテクノロジーは、開発者のソフトウェアライフサイクル管理を大幅に簡素化します。 仮想マシンテンプレートとNano Serverはコストを削減し、最小限のリソースに多くのVMロールを配置できます。



また、新しいプラットフォームにはネットワークロードバランサーが組み込まれており、割り当てられたVM RAMとストレージの容量、それに関連する計算能力を動的に変更でき、他の多くのツールを提供します。 現在では、Windows Server 2016仮想化プラットフォームとゲスト仮想マシン上のWindows Server 2016の機能の両方を含む本格的なサービスです。







2Uパッケージの新しいエネルギー効率の高いLenovo System x3650 M5サーバーは、8コアAMDおよびIntelプロセッサー上に構築された強力でスケーラブルでコンパクトなプラットフォームであり、768 GBのRAMを持ち、SSDソリッドステートドライブを使用します。



このシステムの機能：ホットスワップ可能なディスクと電源、さまざまなディスクドライブオプション、9個のPCIe 3.0スロット、およびRAIDコントローラーの機能を拡張する機能。



これらのサーバーは、クラウドアプリケーション、パフォーマンスコンピューティング、メディア配信などのタスクを実行する必要がある分野で需要があります。 Lenovo System x3650 M5サーバーは、24時間365日の高いワークロード向けに設計されており、ホスティングサービスの提供によく使用され、仮想化プラットフォーム、ターミナルサーバー、分散および高性能コンピューティング、データベースサーバー、アプリケーションとして使用されます。



サーバーは集中リソース管理をサポートし、強力な暗号化とデータ保護を提供します。 診断ツールを使用すると、エラーをすばやく見つけて修正できます。ホットスワップファンとドライブは、必要に応じて迅速に修復を実行するのに役立ちます。 ネットワークインフラストラクチャは、Mellanox ConnectX-4 LxデュアルポートネットワークアダプターおよびSN2100スイッチ（100GbEポート付き）上に構築されています。



帯域幅が1/10/25/40/50 Gb / sのイーサネットスイッチングソリューションを備えたConnectX-4 Lx ENネットワークコントローラーは、仮想化インフラストラクチャの課題に対応するように設計されており、さまざまなアプリケーションや市場でクラス最高のパフォーマンスを提供します。 これらのアダプターは、真のI / O分離と拡張性および効率性を兼ね備えており、クラウドインフラストラクチャ、データベース、およびストレージプラットフォーム向けの費用対効果の高いソリューションになっています。









ConnectX-4 Lx ENは、1、10、25、40、および50 GbEの速度をサポートし、1マイクロ秒未満の遅延と1秒あたり7,500万パケットのメッセージレートを提供します。 RDMA over Converged Ethernet、ステートレスイーサネットオフロードメカニズム、オーバーレイネットワーク、GPUDirectテクノロジーのハードウェアサポートが含まれます。



「Cloudbox」の代表者にデータの展開とプロジェクトの機能について話してもらい、顧客とパートナーに最先端のテクノロジーを提供することができました。



「Cloudbox」は、Windows Server 2016に基づいたIaaSサービスを提供するロシアの唯一のクラウドプロバイダーですか？



はい、そうです。 このサービスがロシアで開始された時点では、Windows Server 2016プラットフォームでIaaSサービスを構築した単一のプロバイダーはなく、一部のプロバイダーは仮想マシンのゲストとしてのみこのOSをサポートしていました。 私たちのデータによると、私たちはまだ半年以上前にそのような唯一のプロバイダーです。



Lenovoサーバ​​ーが仮想化プラットフォームとして選択された理由、構成 どういう意味で高いサービスパフォーマンスを達成しましたか？



2016年8月、Microsoftのロシア駐在員事務所と連携して、サービスを作成するというアイデアが生まれました。 最も重要な要件は、Storage Space Directハイパーコンバージドクラスターの展開およびシールドされたVMのサポートに関して、ベンダーがWindows Server 2016との互換性についてテストされたハードウェアシステムを持っていることです。



さらに、実際には、モスクワの倉庫から、非常に短い納期で機器を配送することになっていた。 レノボはこれらの要件をすべて満たし、プラットフォームに同様のソリューションを展開した経験があり、このタスクのためにサーバーの選択と構成に必要な支援を提供する準備ができていました。



Microsoftと一緒に、構成要件を定義しました。 ストレージサブシステムとして、フラッシュドライブのみを使用し、キャッシュにNVMeドライブを使用することが決定されました。 このアプローチにより、仮想ディスクアレイで優れたパフォーマンスを実現できました。



ソリューションのアーキテクチャは、Microsoftのロシア駐在員事務所と連携して「Cloudbox」の専門家によって開発されました。 プロジェクトの詳細を開発する中で、Lenovoからのマイクロソフトの推奨事項とソリューションが研究されました。



ネットワークインフラストラクチャの役割は何ですか？ メラノックス機器を選んだ理由は何ですか？ ネットワーク帯域幅はどのくらいですか？ SDNを実装する手段は何ですか？



ハイパーコンバージドソリューションにとって、ネットワークインフラストラクチャは不可欠な要素です。 すべての管理、データ交換、障害がネットワークを介して実行された後の回復、したがってその作業の品質に対する要件は非常に高いです。 S2D（ Storage Spaces Direct ）の場合、SMB3プロトコルのサポートが必要です。 Mellanox機器に最も完全に実装されています。



選択したソリューションにより、ノードごとに最大50 Gbit / sの速度でアクセス性の高いデータ伝送ネットワークを構築できました。 ソフトウェア定義ネットワーク（SDN）は、Windows Server 2016オペレーティングシステムを使用して構築されます。SDNコントローラー、バランサー、および他の関連する役割の役割を提供します。



Windows Server 2016を実行しているLenovoとMellanoxの高性能機器に基づいて、SDNv2を備えたハイパーコンバージド環境が実装されています。 プラットフォームのハイパーコンバージェンスとは正確には何ですか？ 「新世代」SDNの違いは何ですか？



ハイパーコンバージドアーキテクチャは、コンピューティング、ネットワーク、およびストレージリソースを1つのレイヤーに結合します。 Windows Server 2016では、ストレージ管理サービス（SDS）、ネットワーク（SDN）、およびハイパーバイザー（Hyper-V）を同じハードウェアにフェールオーバーモードで展開できます。 SDNの以前のバージョンとは異なり、新しいバージョンでは、ロードバランサー-LoadBalancer（SLB）などの重要なサービスが追加されました。



ハイパーコンバージドシステムには単一障害点がないため、水平スケーリングの機会は十分にあります。追加のVMのクラスターリソースとパフォーマンスを向上させ、ストレージの容量とパフォーマンスを向上させる必要がある場合は、クラスターに新しいノードを追加します。 SANの代わりに、通常のローカルエリアネットワーク（RDMAを使用したSDN）が使用されます。 したがって、高価なFC、FCoEソリューションを購入したり、iSCSIを構成したりする必要はありません。 新しいSDSおよびSDN機能により、Windows Server 2016はハイパーコンバージドクラウドソリューションを作成できます。



N2215 SAS / SATA HBAとストレージプール、PCI NVMe、SATA SSDカードを備えたサーバーストレージサブシステムについて教えてください。 どのように構築されますか？



Storage Spaces Direct（S2D）ストレージシステムは、キャッシュ用のNVMe SSDを備えたローカルドライブに基づいて構築されています。 S2Dの場合、ハードウェアの互換性要件に準拠することが重要です。 したがって、RAIDコントローラを使用してプールディスクを接続することはできません（HBAモードであっても）。 したがって、互換性のあるHBAのみが使用されます。



プールにはSSDドライブしかないため、ティアリングは意味がありません。 キャッシュに関するMicrosoftの推奨に従って、NVMeインターフェイスを備えたPCIドライブを使用します。 従来のSSDと比較して、書き込み操作を大幅に高速化します。



高いパフォーマンスを確保し、新しいオペレーティングシステムのすべての機能を考慮に入れるため、Storage Space Directテクノロジーに基づくフラッシュアレイを備えたハイパーコンバージド環境にプラットフォームが展開されています。 プロジェクトのこのコンポーネントについて教えてください。



今日、SDS（Software Defined Storage）テクノロジーであるソフトウェア構成ストレージは、ますます人気を集めています。 多くのベンダーが独自のSDSソリューション（VMWare VSAN、HPE / Nutanix、Cephなど）を提供しています。 Windows Server 2016では、MicrosoftはSDSのバージョンであるStorage Space Direct（S2D）を導入しました。 さまざまなタイプのローカルドライブを備えた通常のサーバー上に、アクセスしやすく効率的なストレージシステムを構築できます。



同時に、従来の古いストレージシステムのテクノロジー（キャッシング、ピアリング、ラックおよびデータセンターのレベルでの高可用性）を使用できます。 ハイパーコンバージド構成の場合、サーバーはWindowsのクラシッククラスターディスクとしてストレージシステムを「認識」します。



ストレージへのアクセスを外部コンシューマーに提供する必要がある場合は、iSCSI、SMB、NFSなどの従来のテクノロジーを使用できます。 仮想ディスクレベルでは、IOPSパラメーターを制御できます。上限と下限は、目的のデータブロックサイズに正規化されたIOPSの数によって決定できます。



Windows Server 2016のどの特定の機能がすでに需要がありますか、または顧客が要求できますか？ サービスプロバイダースタッフ（シールドされたVM）からのアクセスから顧客の機密データを保護するための見通しをどのように評価しますか？ 仮想マシンとアプリケーションの迅速な展開には、どのような新しいテクノロジーが関係していますか？



Windows Server 2016には多くの革新が含まれています。 Microsoft Windows Server 2016の最新のビルトインサーバー保護機能は、攻撃の防止と疑わしいアクティビティの検出を提供します。 多くのユーザーのセキュリティとデータセキュリティの問題は非常に重要です。 さらに、脅威は外部だけでなく、サービスプロバイダーの側から内部と見なされます。 これで、仮想マシン内のデータへのアクセスをプロバイダーの管理者に制限できます。 これは、このカテゴリの顧客がクラウドへの移行を決定するのに役立ちます。 シールドされたVMは、テナントVMを管理者の侵入者から保護する最も安全な方法です。



シールドされたVMに加えて、開発ユーザーはコンテナテクノロジーを使用できます。 グラフィックアクセラレータを使用すると、CADプログラム、アニメーションコントロールなどを快適に操作できます。 サードパーティの追加ソフトウェアを使用することなく。 管理者向けに、Nano Serverが登場しました。これは、Windows Server 2016で最も安全な実行モードであり、「組み込み」のロードバランサーです。



このプロジェクトの進捗について少し教えてください。 これはどのように作成されたのですか、どのような困難があり、何が解決でき、「将来のために」タスクのために何が脇に置かれましたか？



最大の問題は、ドキュメントと診断ツールの不足です。 ネットワークアダプタードライバーの構成における一見マイナーなエラーのために、トラブルシューティングに費やしたほとんどの時間。 クラスタサーバーがホストガーディアンサービス（シールドVMで使用）の要件を満たすために、Lenovoサポートチームと緊密に連携しました。 また、すべての機能を起動するために、SDNはマイクロソフトのスペシャリストと積極的に連絡を取りました。



現在、利用可能な情報の量は日々増加しており、ドキュメントは改善されています。 今日プロジェクトを取り上げると、もっと簡単になります。
