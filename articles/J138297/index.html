IPアドレスが範囲内にあるかどうかを判断する

はじめに


「 IPによる国の決定：アルゴリズムの速度のテスト 」という記事を読んだ後、IPアドレスの範囲でエントリを見つける別の興味深い方法を追加したいと思います。



方法は、範囲を両端[（-1、ip_range_begin）、（1、ip_range_end）]を持つセグメントとして表現し、このセグメント上のポイント（0、ip_address）を検索することです。



DB構造


IP範囲を保存するためのテストパターン構造： 



CREATE TABLE `country_ip` ( `ID` int(11) NOT NULL AUTO_INCREMENT, `IPv4_beg` bigint(11) NOT NULL, `IPv4_end` bigint(11) NOT NULL, `IPv4_GeoRange` linestring NOT NULL, `Country_ID` SMALLINT(4) NOT NULL, PRIMARY KEY (`ID`), SPATIAL KEY `sk_range` (`IPv4_GeoRange`), INDEX `IPv4_beg` (`IPv4_beg`), INDEX `IPv4_end` (`IPv4_end`) ) ENGINE=MyISAM DEFAULT CHARSET=utf8 PACK_KEYS=0




ここで、IPv4_begは範囲の始まり、IPv4_endは範囲の終わり、IPv4_GeoRangeはセグメント、Country_IDは国識別子です。



IPv4_GeoRange列を使用して、空間インデックスSPATIAL KEY `sk_range`（` IPv4_GeoRange`）を作成する必要があります。 この機能はMyISAMでのみ使用可能です。

テストは2種類のテストに分けられます。

  1. 同じIPアドレスでの10秒間のリクエストの数。
  2. ランダムなIPアドレスで1000件のリクエストを完了するのにかかる時間。




すべてのテストで、キャッシュは無効になっています（SQL_NO_CACHE）。
