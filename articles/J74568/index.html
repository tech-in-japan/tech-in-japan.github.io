Paint.NET v3.5リリース

最初のベータバージョンのリリースから1か月半後、Windowsで最高の無料グラフィックエディターのリリースをようやく祝うことができます。 それ以来、さらに3つのベータバージョンがリリースされ、プログラムの最適化、さまざまな欠点の修正、新しい機能の追加のために多くの作業が行われました。









革新と変化



  • ユーザーインターフェイスを更新し、アイコンと視覚スタイルを置き換えました。 AeroサポートがWindows 7およびVistaに追加されました。
  • .NET 3.5 SP1が使用され、必要になりました。これは、速度を目的とした改良が加えられたためです。
  • 新しい効果：ぼかし->表面ぼかしby Ed Harvey
  • 新しい効果：歪み->へこみ
  • 新しい効果：歪み->結晶化
  • パフォーマンスとメモリ使用量を最適化する
  • .NET Framework 3.5 SP1が必要になりました
  • ディスク容量の使用は、インストールファイル（インストール* .msiおよび診断ファイル* .pdb）のNTFSに組み込まれた圧縮のため、約12 MB削減されます。
  • * .pdnイメージの圧縮の改善
  • 効果ダイアログの応答時間を大幅に改善
  • 特にシングルプロセッサ（シングルコア）システムでは、画像がはるかに速く開きます。
  • ほとんどのシステムで、アプリケーションの起動速度が20％に向上しました
  • 複数の画像を開くと、メモリ使用量が大幅に削減されます
  • 画像処理の全体的な品質は、ズームすると拡大されます
  • 選択枠はアニメーション化されなくなりました。 「アリを走らせる」代わりに、排除効果のある滑らかで破線のフレーム（「XOR」、OR）があります。 これは、パフォーマンスにプラスの影響を与え、プロセッサの負荷を軽減しました（さらにラップトップのバッテリー寿命を延ばしました）
  • テキストツールのフォントシステムは完全に書き直されました。フォントの欠落やクラッシュの問題は過去のものです
  • Windows XPおよびVistaでは、テキストツールのフォント処理が改善されました（GDI +の代わりにGDIを使用）
  • Windows 7では、テキストツールに（GDIの代わりに）DirectWriteが使用されます。 これにより、品質と速度がさらに向上します。 Windows Vistaの場合、DirectX 11をインストールする必要があります。そうでない場合、GDIが使用されます
  • ツールバーのフォント選択メニューの使いやすさ、フォント処理の品質、およびパフォーマンスの大幅な向上
  • フォント選択メニューで、新しくインストールされたフォントを検出するためにアプリケーションを再起動する必要がなくなりました
  • インストールされた多数のフォントによるメモリ使用量と起動速度の改善
  • ズームインすると、非常に小さな選択領域のサイズ変更がはるかに簡単になりました。
  • 必要な依存関係は、インストール中に自動的にインストールされます（NET FrameworkやWindowsインストーラーなど）
  • 更新はバックグラウンドでダウンロードされ、アプリケーションの終了時に自動的にインストールされます。 以前のバージョンでは、更新中にPaint.netを使用できませんでした
  • ユーティリティメニューを追加しました。 アイテムはそこに移動しました-アップデート、言語、プラグインエラー
  • 「ユーティリティ」メニューに「フォント管理」項目を追加しました。 デフォルトのWindowsフォントマネージャーを起動します
  • プレビューで中マウスボタンをクリックしたときに画像を閉じる
  • 改善された「非焦点」効果
  • DirectDraw Surface（.DDS）では、自動生成されたMIPテクスチャのオーバーサンプリング（リサンプリング）アルゴリズムを選択できるようになりました
  • 新しいバージョンの要件-SSEをサポートするプロセッサー
  • ガウスぼかしエラーとアルファ値の処理を修正
  • ツールバーの単位の選択に関連するプログラムのクラッシュを修正しました
  • オーバーフローによって引き起こされるプログラムクラッシュが修正され、非常に大きなイメージ（64ビットのみ）で作業できなくなりました。
  • その他の小さな修正の束
  • ロシア語を追加
