クロコダイルスーツケースまたはワニ袋：Lokaliseに接続されたオンライン翻訳者の比較

Lokaliseユーザーは、サイトの翻訳者を雇って、自分のチームで、または自分だけで製品をローカライズすることを選択できます。 プロの翻訳者が不要で、言語の知識が十分にあるプロジェクトのローカライズプロセスを簡素化するために、Google、Yandex、Microsoft、SDLからLokaliseに組み込まれた一般的な機械翻訳システムを使用する機会をユーザーに提供します。 今日は、特定の例を使用してこれらのシステムを翻訳する方法についてお話します。







Google Machine Translate / Google Neural Translate



約6か月前、Google はロシア語を含むGoogle翻訳サービスのニューラルネットワークへの次の言語セットの接続を発表しました。 このイベントは、ロシア語を話すインターネット空間全体のランドマークとなっています。毎日、何千人もの人々がChromeに統合されたGoogle翻訳を使用するか、Google翻訳Webサイトにアクセスして外国語を母国語に翻訳しています。



Google翻訳では、従来のルールベースの機械翻訳機の多くの機能が引き続き保持されます。特定の単語を翻訳する場合、サービスは、最も受け入れられる主要なバージョンの単語翻訳に加えて、そのすべての意味をさまざまな品詞として表示します。 過去の翻訳出版物の 1つで、ニューラル機械翻訳とその仕組みについて既に説明しました。



厳密なルールに基づく統計的な翻訳とは異なり、ニューラル翻訳では、コンテキストに基づいて最適な翻訳を選択するために、複雑で必ずしも明らかではないベクトル比較システムが使用されます。 もちろん、常に成功するとは限りませんが、多くの場合、Google翻訳はそのタスクをうまく処理します。



Google翻訳の翻訳フィールドでのコンテキスト分析は、検証が非常に簡単です。 テストでは、アプリケーションの90％のインターフェースで使用できるいくつかの単語とフレーズを紹介し、検索大手の翻訳者が提供するものを確認します。







このタスクは非常に簡単なようですが、Google翻訳の知的側面は、「ホーム」という単語の翻訳という小さなニュアンスに表れています。 コンテキストに基づいて、翻訳者は「ホーム」オプションを提案しました。これは、ネットワークのオープンスペースでの使用という点で、ソフトウェアインターフェースに関して最も意味のあるものです。 さらに、「アプリの設定」や他の言葉の形でコンテキストを削除した場合、翻訳は変更されないままになります-「ホーム」。 「ホーム」および「ホーム」の形式の他のすべてのオプションは、翻訳形式の下にリスト形式で提供されます。







コンテキストを追加することで、「ホーム」を「ホーム」または「ホーム」として翻訳するという通常の意味で、Google翻訳「マシン」を取得できます。 Google翻訳は口実で友人であり、「自宅」を「自宅」として即座に翻訳します。



検索の巨人の翻訳者に、より難しい仕事、つまりセオドア・ドライザーの小説「シスター・ケリー」の最初の文を与えましょう。 オリジナルでは、次のようになります。



キャロライン・ミーバーがシカゴ行きの午後の電車に乗り込んだとき、彼女の総衣装は、小さなトランク、安価な模造ワニ皮のかばん、紙箱に入った小さな昼食、そして彼女のチケット、紙切れを入れた黄色い革のスナップ財布で構成されていましたヴァンビューレンストリートでの妹の住所と4ドルのお金で。


マークG.ヴォロソフ（1895-1941）のすでに古典的なソビエト訳では、この文は次のようになります。



カロライナ・ミーバーが午後にシカゴに向かう列車に乗ると、彼女の持ち物はすべて小さな胸、偽のワニ革で作られた安いスーツケース、朝食ボックス、列車のチケットがある黄色の革製の財布、ヴァンに住んでいた妹の住所が書かれた紙で構成されていました。ビューレン通り、4ドル。


Google翻訳がこのそれほど単純ではない文の翻訳をどのように処理したかを以下に示します。



キャロライン・ミーバーがシカゴの正午に座ったとき、彼女の全体的な服装は、小さな胸、ワニのための安価な模造バッグ、紙箱での小さな昼食、彼女のチケットを含む黄色の革の財布、ヴァンビューレン通りの妹の住所と4ドルで構成されていましたお金で。


客観的には、結果は車に最適です。 優先翻訳では、ネットワークは「train」という単語を失いましたが、オプションの1つに含まれています。 また、「模造ワニ皮のかばん」と「4ドルのお金」で問題が発生しました。 ネットワークの残りの部分は、理解に値するレベル以上の翻訳を実行しました。これには、文体的な処理のみが必要です。



Yandexトランスレーター



Yandexは、ロシア市場内のすべての分野で最高の地位を確立することを好み、禁止を回避することすら、FASは「検索No. 1」などと述べています。 この声明が会社の他の製品に当てはまるかどうかにかかわらず、Googleの兄-translate.yandex.ruに似たドメイン上にある組み込み翻訳者Yandexの仕事を確認することができます。



さらに、開発者自身によってYandexトランスレータについて多くのことが書かれています。 自社のWebサイトで、同社の開発者は、Yandexトランスレーターが統計的であることをすぐに認識します。 わかりやすくするために、上記の翻訳者についてのテキストに戻る必要はありません。要するに、統計的な翻訳では、最適なオプションを計算してすべての可能なオプションを列挙します。 また、Yandexトランスレータの動作をより明確に示すためにチームが描いたスライド全体を持ち込むこともできます。







このトランスレータは、自己学習型ニューラルネットワークに基づいていないため、最初はGoogle翻訳を失います。 ただし、Yandexチームには少なくとも1枚の切り札がありました。明らかに、製品の主な対象読者はロシア語を話すユーザーだったため、英語とロシア語のペアの翻訳者の仕事をできる限り正確にしようとすることができました。



したがって、Google翻訳を最初にチェックしたときと同じ単語セットを「Yandex」に翻訳者にフィードします。







その結果、予想される「オプション」ではなく「オプション」、「ホーム」ではなく「ホーム」という2つのミスがあります。 同時に、翻訳者は「メニュー」という単語と組み合わせてのみ、「ホーム」のコンテキストで翻訳「ホーム」を保持し、提供します。 他のバージョンの翻訳では、「ホーム」という単語は「ホーム」の変形であるため、インターネット視聴者のニーズに基づいてGoogle翻訳に簡単に組み込まれ、どこか下に完全に下げられます。 翻訳者は、「Home」、「Home」などの一般的なバージョンのみを提供します。



次に、Yandexの機械翻訳がDreiserの作業にどのように対処するかを見てみましょう。



キャロライン・ミーバーが夕方シカゴに列車に乗り込んだとき、彼女の全体的な服装は、小さなトランク、ワニ革のバッグの安価な模造品、紙箱に入った小さな昼食、チケットを含む黄色のスナップレザー財布、ヴァンビューレンの妹の住所からの紙片で構成されていました通り、そしてお金で4ドル。


複雑な構造の形で長距離にわたって、古い技術の問題はより顕著になります。 はい、数年前、Google翻訳はほぼ同じ方法で翻訳していましたが、今では彼らが言うように違いは「明白」です。



Yandexの翻訳者は、ロシア語を勉強している人々と同じ問題に直面しています。彼は文字通りに翻訳しすぎており、文脈が少なく、ケースを理解していません。 一般的に、ケースは外国人の痛みです。 私たちはネイティブスピーカーとして、子供の頃からケースを正しく使用するためのトレーニングを行ってきましたが、これがいかに難しいか理解していません。 Yandexの翻訳者のすべての間違いを整理することは理にかなっていないと思います。機械統計翻訳の性質は上記の例で非常に明確に示されています。 国内の発案者が対処した唯一のことは、通りの名前を記録する形式です。



はい、開発者は機械学習と辞書の両方を使用していると主張しています。 ただし、複雑なテキストを処理したり、コンテキストを理解したりするには、これで十分ではないことは明らかです。



Microsoft Translator-Bingトランスレーター



Lokaliseに接続しているオンライン翻訳者のリストの次は、Bing検索エンジンで使用されるMicrosoftの製品です。 ちなみに、Bingトランスレーターは、あなたがすぐに思うよりも世界中で人気があります：Windows Phoneのすべての公式アプリケーション（たとえば、Twitterアプリケーション）に組み込まれ、Edge for Windows 10の既定の検索と共に使用され、Skypeに「ねじ込まれ」ます。



Bingのもう1つの特徴は、ニューラルネットワークの使用です。 まず、ニューラルネットワークはBing検索に接続してよりインテリジェントにし、ページインデックステーブルを拡張しました。これはGoogleの半分です。 これは2015年の初めに起こりました。 その後、ニューラルネットワークはBingトランスレータに到達しました。 したがって、Bingの翻訳者はGoogle翻訳とほぼ同じ「ウェイトカテゴリ」に属していると言えます。



以前と同様に、Bing Translatorが興味のある簡単な単語セットをどのように処理するかを最初に確認します。







一般的に、Microsoftの翻訳者はYandexシステムよりも優れていましたが、同時にGoogle Translateよりもはるかに多くの自由を認めました。 私たちのつまずきの場合-「ホーム」という言葉-Bingは抜け出し、「ホーム」ボタンに関連付けられたオプション「ホーム」を提案しようとしましたが、Yandexの場合は「ホーム」および「ホーム」には関連付けられません。 一方、「ツール」を「手段」として物議を醸す翻訳にはいくつかの疑問が生じます。 同時に、Yandexの機械翻訳者はそれを「ツール」として明確に翻訳しました。 おそらくBingはあまりにも創造的であるため、Dreiserでチェックアウトする必要があります。



キャロライン・ミーバーがシカゴの午後の列車に乗車したとき、彼女の一般的な装備は、小さなトランク、ワニ付きの安い模造バッグ、紙箱に入った小さな昼食、チケットを含む黄色の革製バッグ、ヴァンビューレン通りにある妹の住所が記載された紙、4お金のドル。


Bing Translatorを守るために、3人の翻訳者のうち、「午後の電車」というフレーズの本質を正しく理解したのは彼だけだとすぐに言うことができます。 Yandexの翻訳者は、この午後の列車を夕方に作りました。Google翻訳は完全に「ホースのふりをし」、列車の存在自体を無視しました。 彼の兄弟のように、ビングは「ワニ付きの安価な模倣バッグ」につまずき、また「装備」と「トランク」で台無しになりました。



一般に、BingはYandexよりも少し優れていたと言えますが、彼はまだ複雑な構造をGoogle Translateに翻訳するには程遠いです。 Googleの翻訳者は明らかにはるかに人気があり、優れたリソースを持っているため、Bingニューラルネットワークのサンプリングとサンプルの欠如が影響する可能性があります。



SDL Free Translator



Lokaliseに接続された翻訳者のリストの最後は、無料のオンラインSDL翻訳者です。 一般に、SDLは翻訳およびローカリゼーションサービスです。 特定の金額については、選択した言語領域のドキュメントなど、プロの翻訳を注文できます。 ボーナスとして、SDLには独自のオンライン機械翻訳機があり、Lokaliseインターフェイスに接続されています。



アプリケーションのボタンに署名するために使用される一連の単語をSDLがどのように処理するかについては、既に確認します。







一見すると、SDLは4つすべての中で最も「機械的な」翻訳者であることがわかります。 コロケーションの場合、ソース言語の語順を保持します。 驚いたことに、長い間苦労してきたホームSDLは、Google翻訳と同じようにホームとして翻訳されました。 最も単純なフレーズの翻訳レベルでもスクリーンショットで見ることができるため、SDLについて詳しく説明することは困難です。



ここで、翻訳者がカロライナミーバーと彼女の「ワニ付き安いバッグ」をテストするよう手配します。



シカゴで午後にキャロライン・ミーバーが列車に乗り込んだとき、彼女のスーツは小さなSLグループ、ワニ皮の安価な模造品、粘土のポンド、紙箱の小さな昼食、そして彼女のチケットを含む黄色の革製の春の財布、妹の住所の紙片で構成されていますヴァンビューレンストリートで、現金で4ドル。


オンラインSDL翻訳者が処理した唯一のことは、「午後の列車」です。 ただし、翻訳の形式自体が「誰がどこで」の因果関係に違反し、最終的に文の意味を完全に変更します。以前の3つのケースでは、これは起こりませんでした。 文の途中から、なんらかの不条理が始まります。 この文の例は、SDLが矢印などの直接音節にのみ適合していることを示しています。実際の構造と文は、ミッドゼロの伝説的な「Promt」翻訳のレベルを提供します。



合計



Lokaliseに接続している4人のオンライン翻訳者のうち3人は、単音節のデザインとアメリカの古典の複雑な申し出の両方に多かれ少なかれ適切に対応しています。 今後の指標の集計によれば、Google翻訳は期待されていますが、常に完全に正しく機能するとは限りません。 MicrosoftのBingトランスレータは、予想以上に優れていることが判明しました。 どうやら、ニューラルネットワークの使用は非難することです。 いくつかの側面では、Bingの翻訳はGoogle翻訳よりも優れています。



Yandexの翻訳者には非常に注意する必要があります。 同社にはGoogleやMicrosoftのようなリソースがないことは明らかですが、ロシアの会社の翻訳者は彼らの背景に対して、特にゲームが彼のホームフィールドでプレイされたことを考えると、弱いようです-英語からロシア語に翻訳しました。 開発者がこの特定の最大値を表示できなかった理由、私たちは間違いなく、最も人気のある言語ペアであり、「ロシアでNo.1」に関するマーケティング担当者の声明を正当化することは謎です。



SDLは、2017年のいずれの場合でも、賢明なことを何も示さなかったと予想されます。 それは単なる機械翻訳者であり、あまりにも賢くも簡単でもありません。 おそらく、特定の目的に適しているか、特定の単語をソースの1つとしてダブルチェックするのに適していますが、それ以上ではありません。



要約すると、高品質の翻訳を取得するには、後で戻ってやり直したりする必要はなく、可能なすべてのツールを使用する必要があります。 もちろん、最良の選択肢は生きている人を引き付けることですが、リソースが限られている場合は、時間やお金で、Google、Yandex、さらにはBingに直面して「3人のファイナリスト」全員を軽daしないでください。
