VMwareがvCenter Server for Windowsの「終了」を発表

過去1か月間、VMwareはその武器庫からいくつかの製品のサポートを停止しました。 その1つは、vCenter Server for Windowsでした。 vCenter Server Appliance（vCSA）に置き換えられます。





/ Flickr / alexas_fotos / cc



VMware vCenter Server Appliance（vCSA）モジュールは、2011年のvSphere 5.0のリリースで導入されました。 当初、vSphere 3.0に登場したvCenter Server for Windowsの機能と拡張性が著しく劣っていました。 数年が経ち、LinuxベースのvCSAはその欠点を取り除き、機能を拡張しました。



その結果、vCSAモジュールは単一の展開環境になり、Windowsプラットフォームは影になりました。 バージョン6.5では、移行機能、強化されたデバイス管理、組み込みのバックアップとリカバリ、VMware Update Managerなどの機能を備えたvCSAが導入されました。



同社は以前、vCSAが将来のエコシステムの基盤になると述べました。 したがって、後者の決定は驚くことではありません。 このステップは、データセンターの管理とITライフサイクル管理を簡素化するVMwareの戦略に適合します。 この観点から、vSphere環境には1つの集中管理プラットフォームが必要です。 vCSAは、vCenter Serverスタック全体、Photon OS、およびデータベース（vPostgres）をサポートします。



一元化されたプラットフォームにより、VMwareは新しいソリューションを迅速に展開できます。 ユーザーにとって、これはより良い適応性と高可用性を意味します。 このモジュールは、顧客に単一の更新センターも提供します。 したがって、vCSAアナログのリリースに伴うメンテナンスとダウンタイムが最小限に抑えられます。



Linuxの専門家であるStuart Burns は 、vCSAへの移行を求めています。 彼は、VMwareソリューションをスケーラブルで費用対効果が高く、導入の準備が整っていると呼びます。 特別な移行ツールサービスを使用した移行プロセスについては、VMwareブログで詳しく説明されています。



コミュニティの意見



Windowsのクライアントのサポートを停止するというVMwareの決定に関する意見は分かれました。 トピックの議論のRedditユーザーは、Webクライアントに不満を表明しました。Webクライアントは、その機能のために一部の機能をサポートしていません。 一部のパネリストは、「vCSAの理論上の利点のほとんどは実質的な利点によってサポートされていません」と「Windowsバージョンの廃止は顧客ではなくVMware開発者のみに利益をもたらす」と支持しました。



一部のRedditの住民は、会社がサポートへの顧客依存度を高めるために努力していると考えています。 そして、すべてのvSphereユーザーがLinuxスキルを持っているわけではないという事実によって、彼らはこれを説明しています。



ただし、同じ議論の中で、Windowsプラットフォームを拒否することは、開発者が1つのソリューションに集中し、ユーザーの利便性を高めるために改善する試みであるという意見があります。 さらに、vSphereの次のメジャーアップデートの少なくとも1年が経過します。 これにより、VMwareはモジュールを整理し、詳細なドキュメントを準備することができます。 また、Linuxの経験がないユーザーにとっては、時間が必要な知識を与えてくれます。
