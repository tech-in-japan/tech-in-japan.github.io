FlexRAID View-複数のドライブを単一のアレイに結合

RAIDを作成せずに複数のディスクを単一のアレイに結合するWindows Home Serverの機能が本当に気に入りました。いつでもディスクを取り外したり追加したりすることができます。

しかし、残念なことに、WHSを搭載したマシンは購入から数か月後に私を置き去りにし、ホームファイルウォッシャーの役割はWindows Web Server 2008が回転しているルーターに切り替えました。



しばらくの間、私は複合ディスクと一緒に住んでいました-これも良いオプションですが、2つの欠点があります-アレイからディスクを削除することはできません。アレイ内のディスクを交換することはhemoです。 さて、ネジの1つが死んだ場合の情報の回復はあまり便利ではありません。

RAID5はWindowsを意味します-逆も同様です。 ディスクの交換は正常であり、情報の回復は原則として必要ありません。 しかし、拡張の可能性はなくなります。



したがって、デュアルテラバイトに切り替えることにしたとき、同時により便利なオプションを探し始めました。 そして見つかった-FlexRAID View。 それが何であるか、どのように機能するのか、なぜRAIDなのかなどについては詳しく説明しません。 私が望んでいたことを行う方法を簡単に書いてください：







そして、私は一般的に次のものが欲しかった：

  1. データディスクを1つの大きなボリュームアレイにまとめて、ファイルを手動で並べ替える必要がないようにします。これは、複数のディスクで発生します（フォルダにマウントした場合でも）。
  2. アレイを崩壊させることなく、いつでもディスクを追加/削除/交換することができます。
  3. ディスク上の情報が互いに独立しているため、ディスクの1つが死亡しても、他のディスクがアクセス不能になることはありません。 冗長性は必要ありません。本当に重要なものの安全性はバックアップによって達成されます。


一般に、JBODはより便利です。



もちろん、Linuxsoidsは、LVM、unionfs、mhddfsなどの巧妙な言葉をすぐに言い始めます...はい、私はそのような言葉を知っています。 しかし、ここではWindows用のソリューションが必要でした。自宅ではLinuxよりもそこに住むのが好きです。



ソリューションを探して、FlexRAIDプロジェクトに出会いました。 それ自体は、「データを保護するRAID」と考えられていました。これは、デバイスレベルで機能する従来のRAIDとは対照的です。 現時点ではまだ頭に浮かびませんが、リアルタイムでは機能せず、「スナップショット」の作成とそれらへのロールバックのみがサポートされています。



FlexRAID Viewは追加機能として提供されます。 説明によると-ちょうど私が必要なもの（そしてもう少し、しかし再び-私はもうそれを必要としません）。



設置

十分に詳細な手順がここにありますが 、やや冗長なため、簡単なバージョンを示します。

  1. FlexRAIDホストとコマンドラインクライアントをダウンロードします： www.openegg.org/forums/posts/list/308.page （登録が必要です）。
  2. 最初に最初のものをインストールし、次に2番目のものをインストールします （ ここの写真 ）。 ただし、写真なしで[次へ]をクリックできます。
  3. クライアントを起動し、 view installコマンドを入力します -「FlexRAID Disk Manager」がサービスのリストに表示されます
  4. FlexRAIDホストがインストールされているフォルダーで、ViewConfig.txtファイルを作成します。 最小コンテンツ：



    DRIVE=K

    UNIQUE=C:\DISKS\2TB1;C:\DISKS\2TB2;C:\DISKS\2TB3

    RESTRICT

    RESERVE=5GB

    THREAD=1

    REMOVABLE=false



    -K:\

    |-*C:\DISKS\2TB1

    |-*C:\DISKS\2TB2

    |-*C:\DISKS\2TB3







    DRIVE-ドライブ文字

    一意-マウントポイント

    制限-必要ではありません。ディスク領域を管理するためのルールがそこに規定されています。基本的な場合は必要ありません

    RESERVE-パーティションの予約スペース、平均ファイルサイズ

    THREAD-スレッドの数。 例では5が提案されましたが、現時点では1を超える量で問題が発生している可能性があります-データが破損しています。

    取り外し可能-ドライブを取り外し可能として表示

  5. 同じフォルダーにAutostart.txtを作成して、システムの起動時にFlexRAIDビューが開始されるようにします。 ミリ秒単位で開始する遅延時間は、ファイル自体に書き込まれます。 マージされたディスクの初期化の速度に応じて、2000以上を推奨します。 たとえば、ネットワークドライブを使用する場合は、さらに追加する価値があります。

  6. クライアントで、 view startコマンドを入力します


すべてが機能するはずです。 上記のスクリーンショットに示すように、システムはこれを通常のディスクと見なします。 実際には、ファイルシステムはディスクの1つに作成され、ファイルはそこに書き込まれます。 いっぱいになると、非表示の_flxr_フォルダーが次のドライブに作成されます。FlexRAIDビューを過ぎてドライブに移動すると、ファイルが配置されます。





オプショナル


  • 現在のバージョンのステータスは「ベータ」、つまり6か月の寿命です。 次に、アップグレードする必要があります。 RCは1年ライブで、リリースにはライフタイムがあります。
  • WebインターフェイスとGUIがありますが、ViewよりもRAID機能の管理に重点が置かれています。 しなかったからです
  • サービスは32ビットですが、64ビットウィンドウでは問題なく動作します。
  • Linux用のFlexRAIDバージョンはありますが、ビューはありません（必要ありません）。
