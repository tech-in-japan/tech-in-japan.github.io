プロジェクトに伴うサイトのリード収集の最適化

こんにちは、Habr！ 私たちの顧客は、カジュアルな訪問者を常連の読者に変える方法についてよく質問します。 あなたがウェブサイトを持っているなら、おそらく訪問者がソーシャルネットワーク上であなたをフォローし、RSSまたはあなたのニュースレターであなたを購読するようにそれを作る方法も考えたでしょう。 リードコレクションの最適化に関する一連の記事でこれらの質問に答えます。 本日の投稿では、プロジェクトに付随するサイト、つまり、会社、製品、または人々の協会に関連し、同時に販売されていないほとんどすべてのサイトについて説明します。 それで、あなたのウェブサイトの訪問者を模倣の歩行者から常連に変える方法は？





目標を達成した販売目標到達プロセス



訪問者をリードに変えるには、3つの段階があることを思い出してください。



  1. 見込み客の収集の最適化-言い換えれば、訪問者に連絡先情報、そして何よりもメールを残すよう促すテクニック。
  2. 販売ファネルの最適化は、販売サイトにとって最も重要な概念です。これには、粘り強い足で顧客データをキャプチャし、最後に、購入したい有用なものでデータを充実させるプロセスが含まれます。 当然、このプロセス中に、クライアントは多数の反復を行うことができ、オプティマイザーには2つのタスクがあります。反復の回数を減らすと「ジャンプ」する回数を減らします。
  3. 自動メーリングチェーン-x + 1、x + 7、x + 14の形式で文字を送信するための戦略。 x =登録日。 したがって、スマートマーケターにとって大きな産業であり、販売ファネルの重要なコンポーネントであるため、販売と密接に接触して考えなければなりません。


情報リソースは、オンラインコラボレーションという用語に近い概念であると言わなければなりません。 コンセプトは広範で、一方の極-Googleの1つの文書に対する平凡な協力、もう一方の極-には大企業向けのソリューションを備えたプロジェクト管理（プロジェクト管理）の無限の海があります。 しかし、本質は変わらず、初歩的なフェイスブックのままです。まとめ、情報を提供し、次に何かをすることです。 だから、あなたはあなたのサイトをさらに賢くするために何が必要ですか。



プロジェクトに付随するリソース



あなたの目標主な目標は、プロジェクトをサポートすることです。 それはいくつかのコンポーネントに分けることができます：



  1. あなたの名刺とあなたに連絡する機会。
  2. あなたのポートフォリオ
  3. 現状に関する情報
  4. 志を同じくする人々の連想と魅力
  5. 特定の活動に向けて共通の努力を向ける




あなたがすることができます：電子商取引に従事していない任意の機関。 これは、いわゆるレンガとモルタル会社です。 この用語の最良の説明は何ですか？ 「オフラインビジネス」と呼ぶことができます。 確かに、これは最良の翻訳ではありません。 これは「eコマースの反対」であると言えますが、これは最良の説明ではありません。 しかし、実際はこれとあれです。 これにはほとんど何でも含めることができますが、まず第一に：



  1. 学校、大学、トレーニング
  2. 中小企業サイト
  3. スタートアップのサイト、初期レベルのプロジェクト
  4. さまざまな組織および協会のサイト（以前は「インタレストサークル」と呼ばれていました）。


リードコレクションの最適化



広告（情報）または販売（販売）を通じて存在する他のサイトとは異なり、リードを「食べる」ことはありません。 ほとんどの場合、リソースはオフラインコミュニティから成長しているため、現時点では、「見知らぬ人」を拡大して自分に引き入れることにあまり興味がありません。 したがって、リードのコレクションを最適化することは、「反最適化」、つまりユーザーの輪を増やしたいという欲求ではなく、特定のフレームワーク内でそれを維持することです。



  1. 招待します。 一般的に言えば、それは強力なマーケティングツールであり、「禁断の果実は甘い」という原則に基づいて機能します。 同じRunetには、これを積極的に使用している多くのリソースがあります-彼らはすでにそれらに乗りたいと思っています。 私たちの仕事はこれにはありません。したがって、私たちは言葉を注ぎません。 リソースを対象とする問題についてオフラインで連絡するユーザーに登録用の特定のアクセスキーを配布します。 たとえば、同じ学校や大学で、学生やセミナーの参加者のコミュニティを作成できます。 したがって、オンライン（優れたエンジンに関するフォーラム）とオフラインの両方でコミュニケーションを促進できます。今後の集会やイベントの発表は、Webサイトに公開するか、手紙の形で送信できます。
  2. リソース自体に有用な情報を配置する-リソースにアクセスするには承認が必要です。 学校の例に戻りましょう。この場合、リソースにはプログラム、スケジュール、スキーム、地図、教科書、視覚教材が保存されます。 インターネット上でそれらをクロールすることは望ましくありません。 それどころか、生徒や教師にとっては無条件です。

    関連。 ただし、資料にアクセスするには、招待状を使用して承認を行う必要があります。
  3. 登録のしやすさ。 その関連性はどのリソースにも残ります。 もちろん、プロジェクトに付随するリソースの場合、ユーザーのモチベーションは高く、おそらく1〜2周の地獄と不適切に構成された承認プロセスを経ることさえあります。 しかし、以前の投稿から、あなたはすでにそれを美しくする方法を知っています。
  4. 信頼性の高いエントリ保護-たとえばキャプチャも。 誰もが善のために良い資源を必要としません、そしてそれは楽しみのためにのみ破壊することができます。 ただし、リソース自体の誕生とともに、リソースの保護を考慮する必要があります。


販売目標到達プロセスの最適化



オフラインビジネスはサイトを「バイパス」して販売しているため、このタイプのリソースにはまったく関係なく、サイトの目的は大まかに言って「パーティー」です。 したがって、「パーティー」を最適化する必要があります。 これを行うには、必ずコミュニケーションを刺激します。



  1. 必要かつ有用なコンテンツを共有し、絶えず更新して補充します。 あなたがしたことの歴史について書いて、プロの秘密と微妙さを教えてください（私たちのブログから例を挙げてください:)
  2. 議論と議論を刺激します。
  3. 今後の会議に関するページを作成し、後で写真を投稿し、会議の進行状況に関する意見を交換します。
  4. あなたのビジネスに関連する活動の販売と交換に専念するフォーラムスレッドを取得します。学校やトレーニングの教科書とメモ、写真家のサイトの装備、観光客やペイントボールファンのウェブサイトのカモフラージュ。
  5. 今後の変更について話し、ユーザーに変更のアイデアを要求します。 リソースを変換し、変更を報告します。


一言で言えば、あなたの公的生活の本格的な伴奏を行い、それを継続的に行う-ここにキーワードがあります。 リソースを離れたユーザーを再構築するために、リソースから「離脱」するのは簡単な作業ではありません。



自動メーリングチェーン



  1. 登録レターおよび関連するもの-投稿「サブスクリプションプロセスの整理方法」を参照してください。
  2. 2週間ごとまたは毎月-リソースの最新情報。 疲れているので、linkinやfacebookなどの文字を自動的に生成しないでください。 リソースのディレクター、コーディネーター、または出版社に代わってそのようなレビューレターを書いてください。
  3. 新しい会議への招待状を定期的に送信します。
  4. 貨物を手放さないでください。
  5. 興味をそそる-まったく同じ定期的なニュースを面白い形で作成することができます。
