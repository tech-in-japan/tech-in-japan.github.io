Mac OS X上のWindowsOS。Everyman's Choice。

私は2年前からMac OS Xをプライマリシステムとして使用しています。 あるWindowsプログラムが、ポピーソフトウェアの大群に類似物を見つけられなかったことがたまたまありました。 この間、Mac OS XでWindows XPを実行するためのさまざまなソリューションを試したので、私の経験を一般の人と共有したいと思います。



次のエミュレーター/仮想マシンについて説明します。

  • Apple BootCamp
  • CodeWeavers Crossover;
  • Sun VirtualBox
  • VMware Fusion
  • Parallels Desktop。


私たちはどこでもWindows XPの1つのバージョンをテストしました（スピードのために良い人によってキャストされました）。



私の日常業務では、 Xara Xtreme Pro （ベクターグラフィックエディター）を使用しています。 このプログラムの機能を拡張するには、 AutoHotkeyを保持する必要があります（ボタンの自動押し、マウスクリックなど）。 この束には独自のニュアンスがあります（WindowsとMacの両方のキーボードショートカットを再割り当てする機能が必要です）。



バーチャライザーの要件（ソリューションはこれらの基準で評価されます）：

  1. 起動および動作速度。
  2. バグレス;
  3. WindowsとMakosiですぐに便利な作業（協力）;
  4. ホットキーを再割り当てする機能。






アップルブートキャンプ



速度 ：読み込みと非常にスマートな動作を実現します（ネイティブWindowsは他に類を見ません）。

グリッチレス ：Windows自体がグリッチする可能性がある限りグリッチが発生します。

協力 ：なし。 これによると、このソリューションは多くのリソースを必要とし、仮想マシン（1CやSolidWorksなど）での実行には適さないゲームまたは特別なソフトウェアにのみ適しています。

ホットキー ：Windowsは排他モードで動作するため、問題はありません。





CodeWeavers Crossover



速度 ：良好な走行速度と開始速度。

グリッチレス ：非常にグリッチなプログラムです。エディターを起動しても、常にクラッシュし、エラーが発生します（他の多くのプログラムでは、グリッチが発生するかまったく起動しないかのいずれかです）。 クロスオーバーが非常に便利だったのは、Half-Lifeとその化身（OrangeBox）をプレイすることだけです。 ハーフゲームをプレイするときに失敗はありませんでした。 Crossover Gamesの使用済みバージョン。 半分に加えて、他のゲームにも適合する可能性があります。

協力 ：問題ありません。 プログラムは、ポピーソフトウェアと共にウィンドウモードで動作します。

ホットキー ：通常の日常業務が不可能なため、テストもされていません。





Sun Virtualbox



速度 ：仮想ボックスの下のウィンドウは驚くほど高速にロードされます（約5秒）。 アプリケーションはスマートに動作します。 プロセッサをロードすることにより、システムはVMWareに匹敵します。

グリッチレス ：最新バージョンでは、ソフトウェアでの作業時に安定性が現れ、古いグリッチはどこかで消えました:)

協力 ：個別のサンドボックスモードとディスプレイ統合（ウィンドウとMacウィンドウが混在）があります。

ホットキー ：AutoHotkeyは、他の仮想マシンで問題なく機能するボタンでいくつかのエラーを生成し、おならを拒否しました。 また、システム設定から仮想ボックスの標準ボタンを再割り当てすることはできませんでした（他のポップアッププログラムでは、これは1回または2回行われます）。



SUNによるInnotekの購入後、物事は仮想ボックスで上り坂になりました。 将来的には、これは良いプログラムです。





VMware Fusion



速度 ：ネイティブモードに匹敵する負荷。 速く動作します。

グリッチレス ：幼少期に、Fyuzhinはネットワークに不具合を抱えていましたが、長い間治っていました。 Vmvarはスイスアーミーハンマーのように非常に安定して動作します:)

協力 ：別のサンドボックスモードとポピーインターフェイス（Unity）との協力があります。

ホットキー ：迷惑な不具合があります。 ユニティモードでは、必要なAutoHotkeyプログラムが機能しません。 つまり、私は1つの共通ウィンドウでしか作業できません。 AutoHotkeyを必要としない人、これは障害ではありません。Unityを使用できます。



VMWareは私たちの希望とサポートです。 このプログラムは、私のMacbook Proの他に、MacbookおよびMacbook Airでも正常に使用されています（ライセンスコンサルタント+その下に問題なく住んでいます）。 使用することをお勧めします。





Parallels Desktop



速度 ： 高速でロードして実行します。

Glitchlessness ：MacBook AirでParallels Desktopの速度が低下することが判明しましたが、これは一時的なものであり、すぐに回復する（または既に回復している可能性がある）と思います。

協力 ：ああ、これは歌です。 このシステムは、ウィンドウ内とmakosyu（Coherence）の両方で動作します。 そして、彼はそれをとても上手にしています。 ちなみに、オプションがあります-Windowsトレイをバーのポピーメニューに配置する（Coherenceモード）。 AutuHotkeyはトレイにあるため、これは重要です。

キーボードショートカット ：これは別の曲です。 オートホットキー機能！ Makovsky progを背景に、オートホット作業のキーボードショートカット。 確かに、Makovskyホットキーはそれらをオーバーラップしますが、これは、makoshiシステム設定で標準キーを並列に再割り当てすることによって処理されます。 たとえば、WindowsではCommand-QとCommand-Wを使用しています。



2つの競合他社



マクロに切り替えたばかりのとき、パラレルは3番目のバージョンでした。 これはブレーキのバグであり、2つのコアすべての速度が低下し、システムがクラストにハングアップする可能性がありました。 私は類似点をscり、誰にも彼女に連絡することを勧めませんでした。 ただし、Parallels Desktop v.4は秋に登場し、これはまったく異なる製品です。 すべてのウィッシュリストが実装されました。 システムを最初からインストールする必要さえありませんでした。VMWareからコンテナーをインポートしました。 そのため、VMWareとParallelsの競争はユーザーの手にかかっています。



剛性の観点から、ParallelsはVMWareを数回行います。 VMWareがアイドル状態のプロセッサの15〜25％を消費すると、Parallelsは2〜3％を消費します。 これにより、Windowsに加えて、Ubuntu ServerとFreeBSDを並行して実験することができます。



ソフトウェアと作業ファイルの構成



必要なすべてのWindowsソフトウェアは（主にポータブルバージョンで）アセンブルされ、特定のディレクトリのmakosiに保存され、内部ネットワーク共有を介して仮想マシンに個別のボリュームとして接続されます。 このソリューションにより、仮想コンテナを簡単に変更し、異なる仮想マシンおよびエミュレーターで1つのソフトウェアセットを使用できます。 WindowsがインストールされたZIPコンテナの重量はわずか190メガバイトです（Windows XPディストリビューションの廃止のおかげです）。



すべての作業ドキュメントはmakoshiのネットワーク共有にも保存され、VMWareおよびParallelsの最新バージョンでは、デスクトップおよびドキュメントディレクトリを仮想マシンに対して透過的にすることができます。 つまり、makoshiとWindowsに1つのデスクトップとドキュメントがあります。
