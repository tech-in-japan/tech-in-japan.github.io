本日の記事では、 Microsoft Fujitsu Private Cloud Fast Trackと呼ばれる、富士通とマイクロソフトの共同プログラムについて説明します。 しかし、それについて話をする前に、今日のトピックに関連する仮想化とクラウドの概念を簡単に更新します。







仮想化



簡単に言えば、仮想化とは、同じ物理機器上でオペレーティングシステムの互いに独立した複数のインスタンスを並行して（同時に）起動できるため、OSは異なる「ハードウェア」で動作すると「考え」ます。 同時に、これらの各仮想マシンには物理機器へのバインドはありませんが、異なるオペレーティングシステムとインストールされたアプリケーションのセットがある場合があります。 これにより、仮想マシンを互いに独立して管理し、異なるハードウェア構成の異なる物理サーバー間で仮想マシンを移動できます。 したがって、仮想化により、さまざまな物理サーバーがより少ないハードウェアサーバーを担当する既存のサーバーの役割とサービスを統合したり、過負荷のサーバーから過負荷のサーバーに仮想マシンをすばやく移行してサーバー間で負荷を再分散することで、既存のサーバー群の使用効率を高めたりできます。



標準の仮想化スキームには、ハイパーバイザーとその中で実行される仮想マシンの使用が含まれます。 ハイパーバイザーは、物理サーバーまたはコンピューターにインストールされるオペレーティングシステムです。 これは、ホストのメインOSとしてのみ使用することを目的としたハイパーバイザー（Microsoft Hyper-V Server）、または本格的なOSのライセンスを持つハイパーバイザー（Hyper-Vを搭載したMicrosoft Windows Server）のいずれかです。 後者を購入すると、Windows Serverを実行している仮想マシンと、仮想マシンを管理するための便利なグラフィカルシェルを使用する機会が得られます。



会社のインフラストラクチャで仮想化を使用すると、サーバーの使用率を改善するという明らかな利点に加えて、サーバー自体（同じタスクに必要な数が少なくなります）、ソフトウェア、ネットワークインフラストラクチャ、消費電力のコスト、機器のスペースを大幅に節約できます必要なユニット、ラック、平方メートルが少なくなります。 同時に、インフラストラクチャ全体のフォールトトレランスとサーバー利用の効率が向上します。 仮想化を使用すると、ダウンロードを柔軟に管理し、必要なサービスと仮想マシンを物理サーバー間で簡単に移行できます。



雲



仮想化ですべてが明らかな場合、多くの未経験のクラウドは依然として困惑しています。 ほとんどすべてのインターネットユーザーが非常に長い間それらを使用していますが。 私たち全員によく知られているクラウドサービスの例は、メールサービス、ソーシャルネットワークとフォーラム、写真とファイルのホスティングサービスです。 では、クラウドとは正確には何ですか？



実際、上記の例では、クラウドはインターネットを使用して普遍的にアクセスできるサービス（アプリケーション）です。 このサービス提供モデルは通常、SaaS（サービスとしてのソフトウェア）と呼ばれます。

PaaS（サービスとしてのプラットフォーム-サービスとしてのプラットフォーム）およびIaaS（サービスとしてのインフラストラクチャ-サービスとしてのインフラストラクチャ）など、他のサービスモデルでも同様のアプローチが使用されます。



PaaSの場合、クラウドプロバイダーから提供されたプラットフォームにアプリケーションを展開する機会があります。たとえば、オペレーティングシステムや必要なデータベース（DBMS）が含まれる場合があります。



IaaSの場合、実際に必要なソフトウェアのセットを使用して独自のオペレーティングシステム（仮想マシン）を展開および実行する機会が得られます。 この場合のクラウドサービスオペレーターは、IaaSコンシューマーの物理インフラストラクチャとその仮想化ビューの管理を担当します。



サービスモデルに加えて、クラウドタイプは非常に重要です。 通常、プライベートクラウド、パブリッククラウド、およびハイブリッドクラウドが割り当てられます。 それらは次のように区別することができます：プライベートなものは、1つの組織、パブリック-一般の人々による使用を意図しています



Microsoft Fujitsuプライベートクラウドファーストトラック



主なポイントを思い出したので、今日の話の主なトピックに進むことができます。 Microsoft Fujitsu Private Cloud Fast Trackは、プライベートクラウドを構築するためのリファレンスアーキテクチャを開発するためのMicrosoftと富士通の共同プログラムです。 このプログラムの目的は、富士通の機器と技術、およびマイクロソフトのソフトウェアに基づいて、柔軟で簡単に拡張可能なさまざまなソリューションを作成することです。



これらのソリューションは、サーバーやストレージシステムなどの特別にテストされた富士通機器、それらのための特別な富士通ソフトウェアソリューション、Microsoft Hyper-VおよびMicrosoft System Center 2012テクノロジーを備えたMicrosoft Windows Server 2012オペレーティングシステムを使用します。



Microsoft Fujitsu Private Cloud Fast Trackの一部として実装されたMicrosoftのプライベートクラウドは、Infrastructure as a Service（IaaS）へのアプローチを示し、以下を提供します。



•サーバーリソース、ネットワーク、データウェアハウスの効果的な管理。

•セルフサービスポータル。ITサービスを迅速かつ簡単に提供できます。

•物理シャーシごとに数百台の仮想マシン、オンデマンドで拡張可能。

•ローカルリソースとリモートリソース（組織自体とサービスプロバイダーの両方）にデータを配置する。



当然、このようなソリューションは1台のサーバーに実装することはできず、データストレージシステムを備えた少なくとも2台のサーバーが必要です。このような場合、多くの場合、ブレードサーバー（ Fujitsu PRIMERGY BX400 ）または複数のコンピューティングノード（ Fujitsu PRIMERGY CX420 ）に基づくサーバーが使用されます。 これらのサーバーは通常、クラスターインボックス（クラスターインボックス）と見なされ、Microsoftと富士通の共同プログラムでハードウェア基盤を形成します。







ハードウェアプラットフォームのもう1つの重要なコンポーネントはストレージサーバーです。これを使用しないと、物理サーバー間で仮想マシンを迅速に移行することはできません。 この場合の優れたソリューションは、Fujitsu ETERNUS DXシリーズディスクストレージシステムです。エントリレベルのシステムから高負荷のハイエンドクラスシステムまで、あらゆるタスクに適したソリューションを見つけることができます。







富士通のMicrosoft Fujitsu Private Cloud Fast Track用の3番目のコンポーネントは、富士通が開発したソフトウェアです。



• ServerView Suite。MicrosoftSystem Centerを使用する場合により効率的な管理を提供します。

•Fujitsuセルフサービスポータル。その機能はServerView Resource Orchestratorによって提供され、ITサービスを簡単かつ迅速に管理できます。

•ストレージリソースの柔軟で便利な管理を可能にする、 ETERNUS SFストレージシステム用の特別なソフトウェア。



そのため、富士通には、クラウド用のクラスターシステムの3つの重要なコンポーネントであるサーバー、ストレージシステム、特殊なソフトウェアの複合体があります。 これらのコンポーネントをマイクロソフトのソフトウェアで補完すると、組織のインフラストラクチャにのみ接続できる既製の作業システムが得られます。



Microsoftプライベートクラウドテストドライブ



プライベートクラウドを作成するために、Microsoftは複数のコンピューティングノードを持つマルチノードシステムを使用する必要はありません。 多くの場合、標準的なアーキテクチャの通常のラックマウントサーバーで十分であり、さらに、使用が必要になることもあります。 たとえば、タスクやアプリケーションで特別なカードコプロセッサー（nVidia Tesla、Intel Xeon Phi）の使用が必要な場合、エネルギー、産業、金融、科学計算での高度な並列データ処理と複雑な数学計算での有効性がすでに証明されています。医学と生物学を含む。 このようなデバイスは、天気予報や映画業界の高品質なアニメーションの作成に広く使用されていました。



並列コンピューティングと仮想化の開発により、VDI（nVidia GRID）カードが登場しました。これには、それぞれフルサイズのPCIe x16スロットも必要であり、ブレードシステムにはインストールできません。つまり、従来のサーバーソリューションも必要になります。 頭字語VDIはVirtual Desktop Infrastructureの略で、そのようなVDIカードを使用すると、実際のディスクリートビデオアダプターで動作するように異なる消費者間でビデオカードリソースを分配できるため、待ち時間が短縮され、リモートディスプレイのパフォーマンスが向上します。



Microsoft Technology Centerでテストベンチを作成するために、2つのFujitsu PRIMERGY RX350サーバー （2つのIntel Xeon E5プロセッサー、128 GBのRAM、24のディスク）と3つのFujitsu PRIMERGY RX300サーバー（2つのIntel Xeon E5プロセッサー、128 GBのRAM、8つのディスク）を使用しました） Fujitsu Primergy RX350サーバーの独自性は、nVidia Tesla、Intel Xeon Phi、nVidia GRIDなど、最大12 GBのメモリを搭載できる最大2つのコンピューティングアクセラレータをインストールできることにあります。



このスタンドは、Microsoftのプライベートクラウドに顧客を慣れさせるために設計されています。 そして、あなたの会社が組織でそのようなソリューションを使用することに本当に興味があり、マイクロソフトの営業担当者に連絡する場合は、独自の特別なテストドライブをリクエストできます。 その間、会社の活動とタスク、および頻繁にマイクロソフトクラウドを使用する可能性が考慮され、スペシャリストはプライベートクラウドでの実務経験を得ます。
