中小企業のメールシステムの構成。 完璧なオプション

このトピックでは、最大20人の会社でメールシステムを編成した経験を共有したいと思います。 また、最適な組織についてアドバイスを求めます。





当社にはWindows Small Business Server 2003 R2 Premiumがあります。

このパッケージには、Exchange 2003、WSS 2.0（3.0にアップグレード）、SQL Server 2005、ISA Server 2004などのパッケージが既に含まれています。



現在、メール階層は次のように構成されています。

Exchangeは内部メール専用に構成されており、外部ドメインに関連付けられていません。 つまり 各従業員には内部メールボックスがあり、お互いにメールを送信したり、Windows Sharepoint Serviceからリマインダーを受け取ったり、ディレクターからタスクを受け取ったりできます。



外部ホスティング（ホスティングプロバイダーのサーバー上のpop / smtp経由）は、オフィスマネージャー（会社の一般的なメールボックス）とディレクターによって構成されます。

そして、従業員が外の世界に手紙を送る必要がある場合、彼はそれをオフィスのマネージャーに送ります、そして彼はすでにそれを外の世界に送ります。



すべてがうまくいきますが、いくつかの不快な瞬間があります。

まず、MS Outlook 2003は外部メールボックスで1回動作します。 現時点では、Outlook 2003は外部メールボックスからのメールの送受信を拒否しています。 私はまだ理由を理解していませんでした（サーバーの設定を変更しませんでした）、Outlook Expressで外部メールボックスを一時的に設定します（すべてが完全に送受信されます）：彼はヘッダーを受信できません：（...ホスティングフィルター）。



現時点では、Exchangeサーバーで外部メールを構成することをお勧めします。 次に、オフィスマネージャーの形で追加のリンクを使用して問題を解決します。 ただし、スパムの問題が発生し、サーバーが負担する必要があります。 着信トラフィック1,5rに対して支払います。 MBごと。



Yandexは中小企業向けにSpamoboronaを提供しており、1日あたり1024通のクリーンなメールを受信します（スパムは除去されません）。 そして重要なことに、スパムメッセージはYandexサーバーでチェックされます。 スパムトラフィックが発生しないことがわかりました。



はい、すべての従業員に個人の企業のメールボックスからインターネット上で書く機会を与える価値があるかどうかは明らかではありません。



同様の経験がある場合は、私見を共有または表明してください。
