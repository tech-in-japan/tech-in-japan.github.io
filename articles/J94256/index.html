Emacsスターターキット

Emacs 大規模で複雑なプログラム。 この定義は、彼に他に類を見ません。 Emaxは、人工知能MITの研究所で非常に長い時間前に作成されたもので、過去のLispハッカーの非常に独特な文化の遺伝子を保持しています。 それ以来、何十年にもわたって何千もの拡張機能で何度も対応、改善、補完、成長し続けてきました。 かつて強力なテキストエディタだったEmaxは、一種のユニバーサルプログラムになりました。テキストの表示や編集などのタスクに触れたとしても、その中では何でもできます。 普遍性を知っている人は、1つのプログラムで、さまざまなインターフェイスに再トレーニングすることなく何千ものことを実行できるため、手に入ります。 初めてemaxを起動した人は、このような汎用性と珍しいインターフェイスに怖がるでしょう。 emaxを使用し、そのアイデアを理解し、浸透させることを学ぶことは、難しく、時間がかかり、長いタスクです（彼らが言うように、穏やかな学習曲線）。



この種の初心者ユーザーへの別れの言葉は熱意を呼び起こす可能性は低く、むしろ反対ですが、...道は行く人によって圧倒され、ここで主なことは始めることです-多くの場合、emaxはおもちゃのようなものになります。





1.1セットアップ



emaxのチューニングは一種の装飾芸術であり、ユーザーにとっては祝福でもあり呪いでもあります。 対照的に

私の意見では、「箱から出してすぐに」すべてが完全に調整されているということが最大の魅力です。

その箱状の状態では、それが可能な限り良好で快適ではありません。 最初の始まりから、叙事詩が始まります-emaxerは、エディターをそれ自体のためにシャープにし、拡張機能をインストールし、その構成ファイルを開始し、他からのコードの断片を調べ、ネットワーク全体でビットごとに独自の設定を収集します-適切なコードベースは時間の経過とともに蓄積します そして、すべてがうまくいきますが、多かれ少なかれ使用可能な状態に調整するプロセスは非常に長いです-それは数ヶ月または数年かかる場合があります（実際、emaxの設定は停止しません）。

2 Emacsスターターキット



オリジナルのEmacsスターターキット （現在はgithub.comの「中央リポジトリ」）はPhil Hagelbergの個人設定ファイル（Phil Hagelberg別名Technomancy）であり、長年にわたってemaxを使用してコンパイルしました。 これで、キットはリポジトリの数百のフォークと、GitHubで彼を監視する数千のフォークになります。



スターターキットは、ユーザー設定によって簡単に拡張できるように、またユーザーマシン間で1つの構成を簡単に移植できるように設計されているため、複数のコンピューターの所有者が好むはずです。



操作には、GNU Emacsバージョン22以上が必要です。 XEmacsのようなフォークは機能しません。





2.1インストール



Emacs Starter Kitのインストールは簡単です。リポジトリをgithubから.emacs.d



ディレクトリに複製します（古い設定を保存し、 .emacs



を削除した後）： 



  git clone http://github.com/technomancy/emacs-starter-kit.git〜/ .emacs.d


出来上がり。 Emacsを起動します。



キットを有効にしてemaxを起動するには、リポジトリを他のディレクトリに複製し、コマンドでemaxを実行します。 



  emacs -q -l〜/ emacs-starter-kit / init.el

動的言語のユーザーにとってのさまざまな便利さ：JavaScript、Emacs Lisp、Clojure、Common Lisp、Ruby、Perl。 私自身は主にLispを使用しているため、Lispモードの改善点を紹介します。

2.3.8 GUIの無効化



スクロールバー、ツールバー、メニュー、ダイアログボックスはありません。 これはすべてありません。トラフィックを節約することが私たちのすべてです。 グラフィックコントロールにはマウスが必要です。マウスに到達するには、キーボードから手を離す必要があります。メニューの場合、目的のアイテムの検索に多くの時間を費やす必要があります。 これらのコントロールが無効になっているという事実から（ユーザーから低下することはありません）、無効にされた要素の機能はインターフェイスに複製されます。 編集中にメニューが使用される場合、これは現在のモードのメニューであり、モードラインバーで使用できます。 スクロールバーは、バッファとマウスホイールを移動するための標準キーに完全に置き換えられています。 ツールバーは単に必要ではありません。キー入力によってアクションを実行する方が簡単です。

正直なところ、Emacs Starter Kitのキーボードショートカットは、特に標準のキーボードショートカットとあまり変わらないので、あまり好きではありません。 そこで、いくつかの組み合わせを提供します（githubのstarter-kit-bindings.el

ただし、最初に、Emacsは最も古く、広く使用されているソフトウェアであり、元のキービングは現代のPC / Apple / ...キーボード用ではなく、20年前に死亡したLispマシンのキーボード用に設計されていることを思い出してください1985 Ti Explorer用のZmacsでは、現在使用されているのと同じ組み合わせを見つけることができます）。 思い出して行動してください。さもなければ、不運なemaxerは「最大指小指症候群」の犠牲者になる危険性があります。これは、最新のキーボードで1時間に数百回押すのが恐ろしいControlキーを積極的に使用するためです。 手を健康に保つには多くの方法があります。



  • 最も簡単な方法は、 Caps Lock and Controlを変更することです。 あなたがハッピーハッキングキーボードの幸せな所有者ではない場合-変更、考えさえしないでください。 私はUbuntuを使用していますが、変更しても問題ありません。システム設定でオプションを探してください。

  • 通常のキーボードを購入します。 通常のキーボードのうち、人間工学に基づいたMicrosoftキーボードとKinesisキーボードを除きます。

  • emaxのキーイングを変更します。 ほとんどのデフォルトは噴水ではありません。 しかし、特に熱心であってはなりません。emaxキービングは、多くの場所、主にbashとzshで使用されています。



だから、私は私の組み合わせのいくつかについてお話します：



Cm

Enterキーの代替。 この組み合わせは、すべてのキーがControlキーと一緒に押されるチェーンで終了する場合に特に便利です。たとえば、 キーを放してEnterに到達しないようにするには、たとえばControlを押さなくてもバッファーを切り替えます： C-Tab Cs Cs Cs Cm この組み合わせは、長時間altにストレッチする場合、 Mx ： Cx Cmの代わりとしても役立ちます。



Cw

ほとんどすべてのターミナルエミュレーターでは、このキーはカーソルの左側の単語を削除しますが、emaxでは何の効果もありません。 バインドして単語を削除し、強調表示されている場合は領域を切り取ります。これは非常に便利で、 Backspaceは実際には必要ありません。 さらに、 Mwは選択した領域をコピーします。



Cg

Escを忘れてください。 何か問題が発生した場合、これがユニバーサルキャンセルキーです。



Cq

C- /の交換。 編集をキャンセルします。 したがって、標準のウィンドウシステム（ Cz、Cx、Cc）の下にある一連の組み合わせは、Cq 、Cw、Mwの位置まで移動します。 （誰かがこのキーとCgを組み合わせる方法を教えてくれたら、とても感謝します）。



Cタブ

通常、私は多くのバッファを保持し、しばしばそれらを切り替えます。 これを行うには、 Cx b を両手で押します—信じられないほどの無駄。



S-Tab、CS-Tab

次/前のウィンドウに移動します。 速くて便利。 したがって、上下に近接して配置された3つのキーを使用して、バッファーを切り替えます。



Cx ck

Controlキーを放さずにバッファを削除する方がはるかに簡単です。そのため、複数のバッファを数倍速く削除できます。



Cs、Cr

これらのキーは、前方および後方の正規表現検索に関連付けられています。 Macintoshの著者の1人であるJeff Raskinが、キャノンキャットのテキストを操作するためにコンピューターインターフェースの中心にそのような検索を置いたことを覚えています。 これはもちろんキャノンキャットではありませんが、何もありません。

これらのキーは、バッファー内の検索だけでなく、idoモードでミニバッファー内の次/前のアイテムに移動するためにも使用されます。バッファーを切り替えるか、ファイルを検索して開くためです。 また、地域選択モードでも。



Sスペース

非常にスマートな自動補完。 この組み合わせを5回押すだけで十分な場合もあり、プログラムは奇跡的に自身を作成し​​ます。



M空間

言語を切り替えます。 簡単にアクセスできる2つの近接キーが最適なオプションです。



F3、F4、F4

マクロ記録の開始、マクロ記録の終了、マクロ呼び出し。
