FreeBSDでラップトップにウィンチェスターヘッドを駐車

Bolsheviksが長い間書いてきた問題は、 Ubuntuだけでなく、FreeBSDでも起こります。



要するに：



一部のラップトップ（または一部のハードドライブ）では、オペレーティングシステムは過度に積極的な省電力モードを使用します。 このモードでは、文字通り数秒ごとに、ハードドライブのヘッドが非常に頻繁に駐車されます。 ウィンチェスターでは駐車場の数に制限があり、駐車場が頻繁にこのリソースを使い果たしてしまう可能性があります。



問題を診断するには、smartctlユーティリティ（ポート内のsysutils / smartmontools）が必要です。



コマンドを実行します：



＃smartctl -A / dev / ad4 | grep Load_Cycle



（ad4をハードドライブの名前に変更します）



実行の結果はおよそ次のようになります。



193 Load_Cycle_Count 0x0012 097 097 000 Old_age常に-37606



この行の最後の数字は駐車場の数です。 数秒の休憩を入れてこのコマンドを数回実行します。 数が増えると、問題は表面にあります。



FreeBSDでは、この問題はataidleユーティリティ（ポート上のsysutils / ataidle）を使用して解決されます。



コマンドを実行します：



＃ataidle -P 254 / dev / ad4



（ad4をハードドライブの名前に変更します）



-Pパラメーターは、ハードドライブの電力を制御します。 値254は、最大パフォーマンスモードを意味します。 このモードでは、エネルギーは節約されず、ヘッドは待機しません。 興味のある方は、他のモードの値を見ることができます。



このコマンドがブート時に自動的に実行されるようにするには、次の行を/etc/rc.confファイルに追加します。



ataidle_enable = "YES"

ataidle_devices = "ad4"

ataidle_ad4 = "-P 254"
