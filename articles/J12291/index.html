トラフィック圧縮プログラムの概要：Traffic CompressorおよびToonel

トラフィック圧縮の概要。

プログラムを使用してトラフィックを圧縮すると、インターネットに費やすお金を節約でき、場合によっては、送信されるデータのストリームのサイズを小さくしてページの読み込みを高速化できます。

この種のプログラムの動作原理は簡単です。プログラムは、クライアントコンピューターにインストールされ、他のプログラムからの接続要求をインターセプトし、それらを特別なサーバーに送信します。 サーバーは、処理して応答を受信した後、要求されたデータを圧縮形式でクライアントコンピューターに送信し、クライアントプログラムはそれを解凍して、要求を行ったプログラムに渡します。

トラフィックの圧縮は25％〜80％に達します。

次のトラフィックは適切に圧縮されています：Webページ、スクリプト（JavaScript / VBScript）、CSSファイル、テキストファイル、POP3またはIMAP4プロトコル経由で受信した電子メールメッセージ、添付ファイルあり（なし）（圧縮された添付ファイル（ ZIP、MP3、JPEG、GIFなど）を25％から30％削減できます）、メッセージングプログラムのトラフィック：ICQ、Windows Messenger、AIM、Yahoo！ メッセンジャー、Jabber、Gadu-Gadu、IRC、Usenet（ニュース）トラフィック。

次のようなトラフィック：画像、Flashムービー、ActiveX、Javaアプレット、ZIP、EXE、MP3およびその他の圧縮ファイル、gzipを使用してWebサイト自体によって圧縮されたページ、プロトコル：FTP、HTTPS、SMTP、UDP、ファイルは圧縮できませんメッセージングプログラム、ストリーミングビデオおよびオーディオを通じて送信されるボイスメッセージ。



トラフィック圧縮を実装する特定のプログラム。

交通コンプレッサー

プログラムのウェブサイト： http : //www.tcompressor.ru/

Development Ltd. Traflabs。 プログラムをインストールしたら、サイトで無料登録を行う必要があります。 1台のアカウントを使用できるコンピューターは2台までです。 アカウントの種類には、有料と無料があります。

プログラムを無料で使用するには、サービスの使用に関するいくつかの制限があります。

-遅延のあるサーバーへの接続（遅延は現在サーバーに接続しているユーザーの数によって異なります）;

-TrafficCompressorを介して数メガバイトのトラフィック（5〜10 MB）を読み込んだ後、サーバーへの接続が切断される

（切断後、プログラムはサーバーに接続するためのキューに再び入ります）。

-着信トラフィックの節約は、有料サブスクリプションを使用する場合よりも20％〜25％低くなります。

有料モードは、これらの制限をすべて取り除きます。

長所：

1.トラフィック圧縮が可能なプログラムの自動検出。

2.プログラム設定のシンプルさと明確さ。

短所：

1.圧縮が可能なプロトコルの限定リスト。

2.このソリューションは、Windowsプラットフォームでのみ提供されます。

3.無料で使用すると、トラフィック圧縮の遅延アクティベーションが可能です。

個人的な意見：

2006年10月からこのプログラムを使用しています。 最初は無料で、2月に6か月間サブスクリプションを支払いました。 常に1 GB以上が節約されています（私がGPRSインターネットを使用しているにもかかわらず:)。 サービスの質は平均です。 インターネット接続が正常に行われ、データが送信されない場合があります。プログラムを無効にすると、すべてが動作し始めますが、それに応じて圧縮は行われません。 しばらくすると、サーバーが安定します。

IPでユーザーの国の定義があるサイトでプログラムを使用するとき、ロシアのユーザーではなく、私が決定するという事実を楽しませます。 特にGoogleは、Google UKへの切り替えを常に勧めています。

トラブルなしではありません。 コンプレッサートラフィックを使用する場合、一部のサイトは完全に機能しません。 特に、セキュリティの目的で（たとえばUcozホスティングとして）ユーザーのIPが常にチェックされる同じもの。

トゥーネル

プログラムのウェブサイト： http : //toonel.net

このプログラムは、toonel.netサイトのチームによってサポートされています（残念ながら、この会社に関する言及は見つかりませんでした）。 このプログラムは、javaで実装されています。 したがって、Javaをサポートするどのプラットフォームでも動作します。

このプログラムは完全に無料で個人的に使用できます。 使用の制限についてはどこにも言及されていませんが、私見では、すべてのプログラムサーバーがビジーの場合、圧縮は自然に機能しないと思います。

長所：

1.マルチプラットフォーム。 サポートされているWindows、Linux、Mac、Pocket PC、Symbian、J2MEのアルファ版が利用可能です。

2.プロキシを介して機能するプログラムを構成する機能。

3.安全な接続を介して送信されるトラフィックを圧縮することが可能です。

短所：

1.構成の複雑さ。

個人的な意見：

プログラムは可能であり、良いです。 しかし、トンネルを使用するためにすべてのプログラムを手動で構成する準備がまだできていません（特に一部のプログラムではこのような機会を提供していません）。
