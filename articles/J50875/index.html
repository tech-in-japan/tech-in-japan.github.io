Eee PCにGPRSをセットアップしてOpera Miniをインストールする

少し前のHabréでは、コンピューターにOpera Miniをインストールするための英語の手順へのリンクが点滅していました。 しかし、彼女に続いて、「追加掘り」が必要な瞬間に直面しました。 結果を達成したので、私はいつものように、将来のために自分自身のための短い指示を書き留めました。そのため、次回はグーグル情報ではなく、自分が何をしていたか思い出せず、単に自分のメモを読み直しました。 そこで、Eeebuntuを搭載したAsus EEE PC 901にOpera Miniをインストールすることにしました。



ただし、この方法は、Javaマシンのみが実行される場合、すべてのオペレーティングシステム（Windows、OSX）に適しています。







1.少しの歌詞



過去2年間、Fedoraは私のラップトップにありました。 最初はFedora Core 4、7、9、そして最後にFedora 10をホストするマスターホストで馴染みがありました。数日前、ロシアの耳にとても辛いアフリカの名前Eeebuntuを身に着けている見知らぬ人と私を変えたのはなぜですか？



2008年半ば以来、財布とタスク（C、J2ME、PHPなどでのプログラミング-外出先を含む）クラスのデバイス-ネットブックに最適なサイズを自分で発見しました。 しかし、Eeeを使用すると、システムをインストールした後、ネットワークカード、WiFiモジュールなどのドライバーをインストールするたびに、ファイルを取得して軸を終了する必要があります。Fedoraが最近インストールされ、「ドープ」されたことがわかります10突然、ネットワークカードの操作方法を忘れてしまいました（「ヒョードルの同僚」が「それは起こります。単に再インストールするだけで処理されます」）、私の忍耐が終わったことがわかりました。 だから私のEee PCにEeebuntuが登場しました。 問題はありませんでした-彼はすぐにネットワークカードとWi-Fiの両方を認識し、Fedorのように、変更できないキリル文字のレイアウトでユーザー名/パスワードを入力することで新しいユーザーを開始することを提案しませんでした...一般的に、私が必要とするすべてが初めて「スパーク」を獲得しましたファイルを取る必要はありませんでした。 さて、道路で作業するための私のソフトウェアが完全にコンパイルされ、新しい軸の下で機能した後、私はEeebuntuが長い時間であることに気付きました。



2. GPRSの構成



GPRS接続の場合、wvdialユーティリティを使用しました。



最初に、bluetoothサービスが実行されていることを確認します（/etc/init.d/bluetooth status）。 必要に応じて、ラップトップのBluetoothモードを不可視から他のデバイスから可視に変更します。 残念ながら、hciconfigユーティリティを使用してコンソールからこれを行う方法を理解できませんでした。Gnomeシステムトレイの「blue-tooth」アイコンを使用して、「Preferences」->「Visible and access to other device」を選択しました（「常に表示」）。



電話（Nokia E61電話用の手順があります）で、コマンド->通信-> Bluetoothを実行します。 [ペアリングされたデバイス]タブに移動します。 オプション->新しいデバイス設定->新しい検索。 検出されたデバイスのリストにラップトップを示し、電話の「選択」を押します。 「デバイスの自動接続を許可しますか？」という質問に対してはいと答えます。電話は、通信用の秘密のデジタルコードの入力を求めます。 「0000」や「1234」のような一般的なものを入力しないでください！ 一部の友人の電話番号をコードとして示すことをお勧めします。 コンピューターにコードの入力を求めるウィンドウが表示されます。 このコードを紹介します。 Bluetoothアイコンをもう一度クリックして、モードを「他のデバイスが接続できる」（「非表示」）に切り替えます。 「Bluetooth設定」ウィンドウを閉じます。



電話をラップトップで熟知したので、GPRS接続の構成に進みましょう。



/etc/wvdial.confに接続パラメーターを書き込みます。 私の場合（MTS、モスクワ）、彼らは次のようになりました。

モデム= / dev / rfcomm2

パスワード= mts
 Init2 = AT + CGDCONT = 1、「IP」、「internet.mts.ru」
 ISDN = 0
自動再接続=オン
愚かなモード=オフ
アイドル秒数= 0
自動DNS =オン

3. Javaをインストールする



英語のマニュアルでは、「サードパーティ」のJava実装の1つを使用することを提案しました。 私はモバイルを開発しています。通常のSunを活用するのは簡単です。 JDK（Java Development Kit）をインストールしました。 Java開発に関与していない場合は、「ボリュームの少ない」 JRE（Javaランタイム環境）をインストールできます。 私のネットブックは他のユーザーを予測しないため、OSをインストールするときに、4ギガバイトのSSD「ドライブ」を「/」としてマウントし、16ギガバイトのSSDキャビネットを「/ home」としてマウントします。 そこで、JDKを/home/yana/jdk1.6.0_11にインストールしました。 管理者の観点から見ると、これはカテゴリーレイマーですが、コンピューター上で正しく実行できます（ネットブックディスクを正しくパーティション分割するためのヒントに感謝します）が、私はそれが自分に合っている限りこれを実行しました。



4.「モバイルJava」のエミュレーター



マイクロエミュレータの公式ウェブサイトはmicroemu.orgです。 それから（ダウンロード->リリース） SourceForge.netにアクセスし 、そこから最新バージョンのマイクロエミュレータをダウンロードします。 ダウンロードしたアーカイブを解凍します。 次に、システム->コントロールセンター（設定）->メインメニュー->新しいアイテムの順に選択して、マイクロエミュレータを起動するメニュー項目を作成します。 開いたウィンドウで、新しいメニュー項目を構成し、次のように記述します。

タイプ：アプリケーション

名前：MicroEmulator

コマンド： <Javaディレクトリへのパス> / bin / java -jar <マイクロエミュレータが展開されたディレクトリへのパス> /microemulator.jar



私の場合、コマンドフィールドは次のようになります。 
  /home/yana/jdk1.6.0_11/bin/java -jar /home/yana/Distr/microemulator/microemulator-2.0.3/microemulator.jar




5. Operaを置く



Opera Miniファイルをダウンロードします。 作成したばかりのメニュー項目を使用して、マイクロエミュレータを起動します。 携帯電話の画像がコンピューター画面に表示されます。 しかし、仮想電話の小さな画面でウェブをサーフィンすることは、どういうわけか一般的ではありません。 マイクロエミュレータメニューで、次を選択します。オプション->デバイスの選択...->追加...マイクロエミュレータを解凍したディレクトリを選択し、その中にデバイスディレクトリを選択します。 ファイルのリストで、microemu-device-resizable.jarを指定します。 デバイスのリストで、「サイズ変更可能なデバイス-ロシア語キーボード」を選択し、「デフォルトとして設定」をクリックして、デバイスのリストのウィンドウを閉じます：「OK」。 次に、ウィンドウの右下隅にある「サイズ変更」ボタンをクリックします。 Asus Eee PC 901の画面では、幅1010、高さ493ピクセルを選択しました。 次に、MicroemulatorでOpera Miniを起動します：File-> Open JAD File ...、最近ダウンロードしたOpera Mini jadファイルを示します（Opera Miniをもっと速く選択できます：File：> Recent MIDlets ...）。 「Opera Mini」という行がエミュレータウィンドウに表示されます-ダブルクリックします。 Operaが実行中です！ 便宜上、最大のフォントはOperaメニューで設定できます：メニュー->ツール->オプション; フォントサイズ：特大。



6.マイクロエミュレーターの問題



私が最初に遭遇した問題は、テキストを入力できないことでした。 ログイン、パスワード、またはWebページ上の他のテキストを入力しようとして、私はわなに陥りました。左右の画面ボタン「OK」と「キャンセル」、プログラムウィンドウの隅、またはEsc'apeでEnterを押しずに助けにはならず、テキスト入力ウィンドウからウェブページに戻ることに成功しませんでした。 マイクロエミュレーターのドックには何も見つかりませんでした。 エミュレートされたデバイスをResizableデバイスからDefaultデバイスに切り替えることで、エミュレーターの裏をかこうとしました。 ネットブックの画面では、大きなブラウザウィンドウの代わりに、携帯電話の画像が小さな画面とボタンで表示されます-右/左のソフトウェアはマウスで完全にクリックされます。 しかし、デバイスを切り替えた後、エミュレーターは現在のアプリケーションをリセットします。 Tyke方式では、キーF1とF2が左右のソフトボタンに対応していることがわかりました。



2番目の問題：テキスト入力ウィンドウでは、ラテン文字で問題なく印刷できます。 しかし、キリル文字のアルファベットは時々（しばしば！）単に入力を停止します。 私が見つけた解決策は最も便利ではありませんが、非常に効果的です。以前にテキストエディタでテキストを印刷していたので、通常のコピー/貼り付けを使用してエミュレータに転送します。 不都合なことに-しかし、決してないよりはましです。



しかし、ファイルをダウンロードして、ブラウザからテキストまたはURLをコピーすることはできませんでした。
