Haiku R1 / Alpha2リリース

最初のアルファのリリースから8か月後、Haiku R1の2番目のアルファが予定どおりにリリースされました。 さまざまな形式のディストリビューションをここからダウンロードしてください 。





新機能：



  • 配布には、Webiktに基づく新しいWebiktブラウザーが含まれています。 まだあまり安定していないことに注意してください。
  • FreeBSD 8との互換性レイヤーを使用したWiFiサポート。ディストリビューションに含めることができないドライバーの部分は、install-wifi-firmwares.shを使用してインストールされます。 WEP暗号化をサポートするために、実験的なsetwepアプリケーションが引き続き使用されています（将来、より便利で使いやすいものに置き換えられます）。
  • MSI（Message Signaled Interrupts）のサポートを実装しました。 これにより、割り込み処理が改善されます。
  • Webページの部分的な読み込みを引き起こすネットワークの問題を修正しました。
  • ディスクがパーティション分割されると、DriveSetupは、アクティブパーティションからの後続のブートのために、MBRにベースブートローダーをインストールします。
  • ローカリゼーション効果が現れました。
  • ユーザーマニュアルおよびその他のドキュメントの翻訳が掲載されています。
  • 多数のロックが最適化されており、アセンブリ時間が大幅に短縮されています。
  • カーネルお​​よび仮想メモリ管理システムのコードは最適化されています。
  • フォントスムージングの改善
  • 関連ソフトウェアの多数の更新、bash 2.xから4.xへの最も顕著な移行。
  • PackageFSを介してパッケージ管理ツールを置き換えるための、最も単純なスクリプトであるinstalloptionalpackageを追加しました。 HaikuのOptionPackagesからパッケージの一部をインストールできます。
  • メモリを操作する戦略の変更により、USBフラッシュドライブとディスクを操作するパフォーマンスが大幅に向上しました。
  • ACPIはデフォルトで有効になっています。
  • Nvidiaチップセットに基づく組み込みサウンドカードのパフォーマンスの向上。
  • UDFファイルシステムを操作するためのモジュールが修正されました。
  • POSIXの実装における多数の改善。
  • API / ABIの変更：

    • BJoystick-BeOS互換性の問題を修正しました。
    • ABI GCC 4が変更され、それを使用するアプリケーションが再構築されました
    • ライブラリのさまざまなバージョンのサポートが登場しました。


  • GCC4ベースのHailkuは64ビットLinuxディストリビューション上でビルドできますが、GCC2バージョンではこれまでのところ、ビルドするには32ビットLinuxディストリビューションが必要です。

  • 新しいタイプのanybootイメージが作成されました 。これにより、Haikuイメージを両方のCDと直接USBスティックに書き込むことができます。 録音ソフトウェアはスマートであり、ディスクへの誤った録音につながることに注意してください。


これまで欠けていた機能：



  • WPA暗号化のサポートはなく、WEPのみ。
  • パッケージマネージャーはまだありません。
  • Haikuブートマンブートローダーは現在、1つのドライブからの起動のみをサポートしています。
  • USBキーボードは、OHCIコントローラーまたはUSB 2.0ハブに接続されており、カーネルデバッガー（KDL）では機能しません。 キーボードが機能するためには、キーボードをUHCIコントローラーのポートに直接接続する必要があります。
  • フォントオーバーレイはまだHaikuに実装されていません。 その結果、インストールの開始時にデフォルトのフォントでサポートされていない言語（たとえば、日本語）を選択した場合、この言語のテキストは読み取れません。 フォントオーバーレイが実装されるまで、インストーラーの起動時に英語を選択します


既知の問題：



  • 最初の起動時とLiveCDからの起動時のパフォーマンス。 LiveCDのインストールまたは起動後に初めて起動すると、いくつかのタスクがバックグラウンドで起動され、インストールが完了します。 これにより、パフォーマンスが低下します。 ハードドライブから起動する場合、これは問題を引き起こさず、タスクは迅速に完了します。 LiveCDの場合、CDの読み取りと検索の速度が遅いため、これはより顕著です。 CDは読み取りしかできないため、LiveCDモードでは、これらのタスクはブートごとに起動されます。
  • フォントのスムージングは​​、特許の有効期限により改善されましたが、まだ最適化されていません。 特許に関する不明確な状況のため、特許のコードは公式リリースでは無効になっています。 このコードには、FreeTypeで使用されるサブピクセルアンチエイリアスが含まれています。 すべての側面が明らかにされた後、サブピクセルスムージングが公式リリースに含まれます。

  • HaikuのACPIサポートはデフォルトで有効になっており、かなり安定していますが、それでも問題を引き起こす可能性があります。 ブートローダーメニューでACPIを無効にできます。

  • 高速ディスクから低速ディスク（USBフラッシュドライブなど）に大量のデータをコピーすると、スワップが発生する可能性があります。

  • 一部のコンピューターでは、Haikuブートローダーがフリーズする場合があります。
  • ローカリゼーション/内部化のサポートは実験的なステータスです。 一部のアプリケーションは部分的に翻訳されており、英語バージョンよりも長い可能性のある文字列に問題があります。 ローカライズの設定はまだ機能しません。
  • ユーザーマニュアルのウクライナ語翻訳を選択すると、イギリス国旗が表示されます。
