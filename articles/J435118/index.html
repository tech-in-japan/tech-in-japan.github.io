Save File Me-クライアント側で暗号化を行う無料のバックアップサービス

各管理者は、毎日のバックアップを作成することがどれほど重要かを知っています。

私にとって、「理想的な」バックアップサービスは次のとおりです。



  • 登録の欠如、郵送先住所への拘束など
  • プロジェクトごとに独立したアカウントを作成する機能
  • ファイル名を含むクライアント側の暗号化
  • 無制限の数の「バージョン」のファイルの保存
  • 暗号化を信頼できるオープンソースクライアント
  • 展開に便利なように、依存関係のない1つのバイナリファイル
  • 可能であれば無料で:)


私はそのようなサービスを作成することにしました、そして、私はこれを得ました：



クライアントをロードした後、必要なファイルの保存をすぐに開始できます。 このサービスは完全に匿名であり、登録は不要です。



この時点で、クライアントはアカウントとファイル暗号化のために一意のキーを作成します。 そして、ファイル自体は暗号化され、サーバーにアップロードされます。 

savefileme put < > Active subscription until 2021-01-20T17:08:16Z. put: < >: 8.1MiB (1.2MiB/s) done


ファイルの回復は、ダウンロードと同じくらい簡単です。 

 savefileme get < > get: < >: 2KiB (41.2KiB/s) done


stdinを介してデータを転送することにより、フォルダーをファイルとともに保存できます。 

 tar cz <> | savefileme put --stdin folder.tar.gz Active subscription until 2021-01-20T17:08:16Z. put: folder.tar.gz: 8.1MiB (1.1MiB/s) done


ダウンロード済みのファイルのリストを表示します。 

 savefileme ls total 1 id size updated datetime name (#files) 64 8539136 2019-01-21T14:13:05Z < > (< >)


デフォルトでは、クライアントはファイルバージョンを「折りたたみ」ますが、すべてのバージョンを表示できます。 

 savefileme ls -a total 6 id size updated datetime name 44 8539136 2019-01-19T07:49:53Z < > 45 8539136 2019-01-19T12:18:50Z < > 46 8539136 2019-01-20T16:17:42Z < > 47 8539136 2019-01-20T16:24:31Z < > 48 8539136 2019-01-20T16:31:25Z < > 49 8539136 2019-01-20T16:39:19Z < >


各バージョンには一意の識別子が割り当てられています。識別子を指定することで特定のバージョンを復元できます。 

 savefileme get --id 46 get: < >: 2KiB (38KiB/s) done


アカウントを別のサーバーにエクスポートするには、 account exportコマンドで初期化行をコピーするだけです： 

 savefileme account export savefileme account import --secret <  >


無料サービスを使用すると、ファイルを最大3日間保存でき、その後は自動的に削除されます。 通常、これは、毎日のバックアップが災害から回復する場合、または単にデータを別のサーバーに移行する場合に十分です。 年間サブスクリプションでは、ファイルの保存期間が月額0.99ドル（VATを含む）で最大30日間延長されます。



Save File Meサービスがあなただけでなく私にも役立つことを願っています 。
