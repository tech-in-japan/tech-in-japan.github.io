アプリケーションの概要

タブと複数のウィンドウ

お気づきかもしれませんが、アプリケーション内のナビゲーションはタブを使用して実行されます。 しかし、多くのタブには複数のウィンドウがあります。 遷移中に実行中のウィンドウをクリーニングしないと、ビューの階層によって許容値が増加し、アプリケーションがドロップするという事実に問題があります。 これを行うには、TabActivity（すべてのタブがあるメインアプリケーションウィンドウを継承する）から継承した独自のクラスを作成し、必要に応じてonBackPressed（）をオーバーライドするか、遷移時に自分でfinish（）を呼び出します。

また、1つのタブの外観のカスタマイズに問題がありました。 実際、これには複雑なことは何もありません。ビューを使用してレイアウトを作成し、タブに渡すことで好きなようにスタイルを設定します。

ListActivityから継承するかどうか 実際、唯一の違いは、ウィンドウデザインで他の要素とレイアウト自体が必要かどうかです。 はいの場合、継承とは関係ありません。継承していない場合、クラスとsetContentViewを呼び出す必要はありません。

また、リストについて重要なことを1つ覚えておく必要があります。 リストはビュー要素を繰り返し使用するため、慎重に作業する必要があります。 たとえば、ビューメソッドsetTag（）、getTag（）を使用して、任意の識別子をバインドできます。 上記で書いたように、カスタムアダプターの記述とgetView（）メソッドでconvertViewパラメーターの使用も慎重に検討する必要があります。

この設計オプションでは、リストの要素が切り捨てられ、すべてが表示されるわけではありません（私の意見では、Googleコードにはそのような問題があります）

この問題の解決策は、リスト内の要素の数でリストのサイズを設定することでした。次のコードを使用します。

//set ListView size by items count

public static void setListViewHeightBasedOnChildren(ListView listView) {

ListAdapter listAdapter = listView.getAdapter();

if (listAdapter == null) {

// pre-condition

return;

}



int totalHeight = 0;

for (int i = 0; i < listAdapter.getCount(); i++) {

View listItem = listAdapter.getView(i, null, listView);

listItem.measure(0, 0);

totalHeight += listItem.getMeasuredHeight();

}



ViewGroup.LayoutParams params = listView.getLayoutParams();

params.height = totalHeight + (listView.getDividerHeight() * (listAdapter.getCount() - 1));

listView.setLayoutParams(params);

}
