データセンターの仕組み：今日と明日

データセンターの未来は、クラウドソリューション、ハイパーコンバージドインフラストラクチャ、より強力な機器にあります。







データセンターは、企業が企業の機能に不可欠なアプリケーションや情報を保存するために使用する部屋です。したがって、改善および開発を行う際には、信頼性とセキュリティを長期間維持することに関するすべての詳細を熟考することが重要です。



データセンター内の機器



通常、多くの人々は、データセンターが巨大な一体型のメカニズムのように見えることを想像しますが、実際には、データセンターはルーター、スイッチ、セキュリティデバイス、ストレージシステム、サーバー、アプリケーション配信コントローラーなどの技術要素が混在しています。 これらすべてのコンポーネントは、ITが最も重要なシステムを保存および管理するために必要であり、これは会社の継続的な運用に不可欠です。 したがって、優先度の高いタスクは、信頼性、パフォーマンス、セキュリティ、およびデータセンターの継続的な改善です。



データセンターのインフラ



技術機器に加えて、データセンターには、ハードウェアとソフトウェアの継続的なメンテナンスのための施設の重要なインフラストラクチャも必要です。 これらには、電源サブシステム、無停電電源装置（UPS）、換気および冷却システム、バックアップジェネレーター、外部ネットワークオペレーターに接続するためのケーブルが含まれます。



データセンターのアーキテクチャ



ほとんどすべての大企業は、地理的に分散した複数のデータセンターを持っている可能性が高くなります。 いくつかの地域で。 したがって、これにより、組織は情報をサポートおよびバックアップする機会を増やすことができ、さらに自然災害または洪水、暴風雨、テロリストの脅威などの人為的災害に対する保護を提供できます。 データセンターアーキテクチャを作成する場合、ほとんど無限の可能性があるため、常に多くの矛盾が生じます。 主な質問の一部を次に示します。



  • ビジネスにはミラー化されたデータセンターが必要ですか？
  • 地理的分布の大きさはどのくらいですか？
  • 障害が発生した場合、回復するのにどれくらい時間がかかりますか？
  • 容量が増加した場合、どのくらいのスペースが必要ですか？
  • プライベートデータセンターを借りるか、共有コロケーション/ハーメチックゾーンを使用する必要がありますか？
  • 帯域幅と電力の要件は何ですか？
  • どのプロバイダーを選択しますか？
  • 保護は必要ですか？


これらの質問への回答は、いくつのデータセンターを構築する必要があるのか​​、どこで構築するのかについて適切な決定を下すのに役立ちます。 たとえば、マンハッタンの金融サービス会社は、停電によって数百万ドルかかる可能性があるため、継続的なビジネスが必要になる可能性があります。 おそらく、同社は、ニュージャージー州やコネチカット州など、互いに複製する2つのデータセンターの都市のすぐ近くに建設することを決定するでしょう。 作業は別のデータセンターを使用して実行されるため、データセンターの1つの作業は、会社の作業を中断することなく停止できます。



ただし、小規模な専門サービス会社は情報への即時アクセスを必要としない場合があるため、最適なソリューションは、オフィスに直接メインデータセンターを配置し、夜間にすべての支店から別のサイトに情報をバックアップすることです。 障害が発生した場合、情報の回復プロセスが開始されますが、市場で勝ち抜くために必要なデータが常にリアルタイムであるビジネスと比較して、そのような緊急性はなくなります。



多くの場合、データセンターは大企業とクラウドサービスプロバイダーのみに関連付けられていますが、実際には、どの企業も独自のデータセンターを持つことができます。 小規模企業の場合、データセンターは、たとえばオフィススペースにある小さな部屋にすることができます。



データセンターの標準



2005年にITリーダーがデータセンター、米国規格協会（ANSI）、および電気通信産業協会（TIA）のインフラストラクチャを簡単に判断できるようにするため、 4レベルの信頼性を提供するデータセンター。 承認された標準の公式Webサイトで、データセンターの各レベルの要件の完全な説明を見つけることができます。



未来の技術



すべてのIT業界とテクノロジーに加えて、データセンターも大幅な技術進歩の段階にあるため、将来のデータセンターは最新のデータセンターのようにはなりません。



中小企業は勢いを増しており、より動的で分散化されています。つまり、データセンターで使用されるテクノロジーは、より柔軟でスケーラブルになります。 また、サーバーの仮想化がますます一般的になり、ネットワークに新しい負荷がかかり、トラフィックの性質が変化します-伝統的に、南北トラフィック（ユーザーとデータセンター間のデータ交換）はデータセンターの特徴であり、最近ではより多くのデータ量がデータセンター内で送信されます。 東西のトラフィックが増加しています。



以下は、従来の厳格で静的なデータセンターを柔軟で操作可能なデータセンターに変換する主要なテクノロジーです。その結果、デジタル組織のあらゆる要求を満たすのに役立ちます。



パブリッククラウド



従来、企業は常にホスティングプロバイダーまたは外部管理のサービスを使用して、独自のデータセンターを構築してきました。 このアプローチにより、既存のデータセンターの所有構造と経済的要素を変更することができましたが、以前と同様に、技術ソリューションの実装と組織化に多くの時間が費やされました。 ただし、クラウドサービスの人気の高まりと、Amazon Web ServicesやMicrosoft Azureなどのクラウドプロバイダーが提供するIaaS（サービスとしてのインフラストラクチャ）モデルは、企業がマウスを数回クリックするだけでクラウド内の仮想データセンターを提供できるようになりました。 分析会社ZK Researchのデータは、80％以上の企業がハイブリッドアプローチの使用を計画していることを示しています。これは、パブリッククラウドとプライベートデータセンターの両方の使用を意味します。



ソフトウェア定義ネットワーク（SDN）



デジタルビジネスの柔軟性は、最も柔軟性の高いコンポーネントではなく、ネットワークに直接依存しています。 SDNは、これまで知られていなかったダイナミクスを新しいレベルに引き上げることができます。



ハイパーコヒーレントインフラストラクチャ（HCI）



データセンターの主要なタスクの1つは、サーバー、ストレージ、およびネットワークの正しい「ミックス」を作成して、高い要件を持つアプリケーションをサポートすることです。 インフラストラクチャを展開したら、アプリケーションを中断せずに迅速にスケーリングする方法を理解する必要があります。 HCIは、展開中に追加のコンポーネントを追加することで拡張可能な標準ハードウェアに基づいて展開しやすいデバイスを提供することにより、作業を大幅に簡素化します。 以前は、HCIは主にユーザーデスクトップの仮想化に使用されていましたが、最近ではユニファイドコミュニケーションやデータベースなどの他のビジネスアプリケーションにこのアプローチを適用し始めました。



コンテナ



アプリケーションを開発する場合、多くの場合、このアプリケーションを後で起動するインフラストラクチャを作成する間、作業を中断する必要があります。 これにより、組織はDevOpsモデルに移行できなくなります。 コンテナは、開発者がスタンドアロンシステムでアプリケーションとその依存関係を実行できるようにする、作業環境全体の仮想化手法です。 コンテナは非常に軽量で、すばやく作成および削除できるため、特定の条件下でアプリケーションをテストするのに最適です。



マイクロセグメンテーション



標準のデータセンターにはすべての基本的なセキュリティテクノロジーが備わっているため、トラフィックが南北方向に移動すると、すべての保護ツールを通過して作業が安全になります。 東西方向のトラフィックの増加は、トラフィックがファイアウォール、侵入防止システム、その他のセキュリティシステムをバイパスすることを意味し、これによりマルウェアを迅速に拡散できます。 マイクロセグメンテーションは、リソースが相互に隔離されているデータセンターにセーフゾーンを作成する方法です。そのため、どこかでギャップやリークが発生した場合でも、被害は最小限に抑えられます。 マイクロセグメンテーションは通常ソフトウェアで実行されるため、非常に機動性があります。



NVMe



すべての情報がすでに完全にデジタル化されている今日の世界では、非常に高速でデータを転送する必要があります。 Small Computer System Interface（SCSI）やAdvanced Technology Application（ATA）などの従来のストレージプロトコルは、数十年前から存在し、その能力の限界にあります。 NVMeは、システムとソリッドステートドライブ間の情報転送を加速するように設計されたデータストレージプロトコルであり、データ転送の速度を大幅に向上させます。



GPU



中央処理装置（CPU）は、ムーアの経験則が限界に達するまで、数十年にわたってデータセンターのインフラストラクチャに容量を供給してきました。 さらに、分析、機械学習、IoTなどの最新のワークロードでは、既存のプロセッサーの能力をはるかに超える新しいタイプのコンピューティングモデルが必要です。 以前はゲームにのみ使用されていたGPUは、多数のスレッドを並行して処理できるため、将来のデータセンターに最適であるため、根本的に異なる動作をします。



データセンターは、常にその規模に関係なく、あらゆるビジネスで重要な場所を占めています。 ただし、データセンターの展開方法とデータセンターで使用されるテクノロジは、大幅に変更されています。 理想的なデータセンター運用スキームを実装するには、世界がますます動的になっていることを忘れないでください。 IT部門の重要な変化に影響を与えるテクノロジーは、未来のテクノロジーです。
