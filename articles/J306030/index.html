Nokia 7210 SAS-Mのソフトウェアと初期構成の更新手順

この記事は、Nokia機器（以前のAlcatel-Lucent）7210 SAS-Mの初期試運転の手順を理解したい人を対象としています。 機器を操作する際の唯一の正しいアプローチは、ドキュメントを事前に読むことです。 しかし現実には、将来の仕事のために準備することなく、人に緊急の機器の打ち上げのタスクを与えることができます。 タイミングはオンで、ドキュメントはありません。チューニングは「すぐに」行われます。 残念ながら、これはまれな生活状況ではありませんが、ほとんどの場合、その結果は嘆かわしいです。









ネットワーク上での7210 SAS-Mの主要な統合の手順を理解するのに役立ちますが、特に説明されている手順は新しいソフトウェアバージョンで変更される可能性があるため、製造元からの公式文書を置き換えることができないことにすぐに気付きます。 7210を実行してリモートアクセスを提供するために必要な最小限の情報のみに制限する予定です。 それにもかかわらず、私はこの記事を興味を示す情報で簡単に補足します（または質問が範囲を超えている場合は新しい記事を書きます）。







Nokia 7750の主要な構成トピック（SR-7 | SR-12）に興味がある場合は、 対応する記事でそれをよく理解できます。







1. 7210 SAS-Mのソフトウェアアップデート



まず、ソフトウェア（TiMOS）を現在のバージョンに更新することをお勧めします。 5620SAM制御システムを使用する場合、サポートするTiMOSの最大バージョンを検討する必要があることにすぐに注意します。







1.1仕事の準備


仕事に必要なもの：







1）COMポートまたはusb-comアダプターを備えたラップトップ。

2）7210用のコンソールケーブル。Ciscoロールオーバーケーブルを使用しています。

3）ターミナルクライアント（Putty、SecureCRTまたは同等のもの）。

4）TFTPサーバー（tftp32または同等のもの）;

5）現在のTiMOSの存在（例としてTiMOS 7.0R12を使用しましたが、指示はすべてのバージョンに関連しています）。







1.2充填ソフトウェア


ソフトウェアをダウンロードするには、コンソールポートに接続し、次のようにセッションパラメータを設定する必要があります。

Data Bits 8

Stop Bits 1





Flow Control None

TiMOSがSAS-Mにすでにインストールされている場合は、ステップ1.3をスキップして、1.4に進みます。

4.便利なメモ



  • 「admin display-config」コマンドで構成を表示できます。 コンソールの場所からコマンド（any）を実行する必要がある場合は、最初に「/」を入力します。
  • infoコマンドを実行すると、場所のブランチに関連する構成を確認できます。 「デフォルト」パラメータ値も表示する必要がある場合は、「情報の詳細」を使用します
  • 「show chassis」および「show card」コマンドを使用して、シャーシの温度とFAN動作モードを確認できます。 ファンが全速モードで動作している場合、これはユニットの温度が上昇していることを示しています。
  • サイトのNokiaの温度要件：

    -7210 SAS-Mの場合、サイトの温度の維持を0〜50°Cの範囲で整理する必要があります。

    -7210 SAS-M 24F 2XFP ETRの場合、設置場所の温度は-40〜65°Cに維持する必要があります。
  • 「show chassis」コマンドを使用すると、電力とその範囲外に関する情報を取得できます。
  • 「show log log-id 100」および「show log log-id 99」コマンドを使用すると、それぞれデフォルトの重大エラーログとデフォルトのシステムログを表示できます。
  • show portコマンドは、ポートのステータスを確認します
  • show router route-tableコマンドはルーティングテーブルを反映します
  • ボーナスゲーム、または7210 SAS-MとTiMOS 2ndバージョン以下を入手した場合、アップデートは3回目のリリースまで実行されます。 また、MGMを介してソフトウェアポートがフラッディングしないことにも注意してください。このタスクでは、1/1/1または1/1/2ポートを使用する必要があります。
  • IOS（Cisco）：「ping 1.1.1.1サイズ1500 dfビット」、この場合1500バイトはIPおよびICMPヘッダーが付いているデータグラムのサイズです。
  • TiMOS（Nokia）：「ping 1.1.1.1サイズ1472」-1472バイトはICMPのデータフィールドのサイズを示します。 つまり、8バイトのICMPと20バイトのIPをさらに追加し、Ciscoと同じ1500バイトを取得します。
  • Nokiaのテクニカルサポートに連絡する場合は、コマンド「/ admin tech-support cf1：<ファイル名> .bin」で撮影された2つのテクニカルサポートファイルを、最低15分の時間差で提供する必要があります。 残念ながら、他の多くのベンダーとは異なり、これらのファイルは暗号化されており、ノキアのエンジニアのみがそれらを読むことができます。 問題やクラッシュが発生した場合は、まず最初のテクニカルサポートファイルを削除してから、問題の解決に進むことをお勧めします。
