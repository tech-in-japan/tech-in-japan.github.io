少し前に、Windows Server Technical Preview 2の新しいバージョンがリリースされました。これは、最新のサーバーオペレーティングシステムの将来のリリースの可能性をさらに明確に示しています。 ITインフラストラクチャの運用と開発に直接関係する人のために、仮想化に関連する技術革新について簡単なメモを用意しました。

しかし、初心者向けに、Nano Serverについて少し説明します。WindowsServer Technical Preview 2の新しいタイプのインストールで、2つの一般的なITタスクを解決できます。

  • 信頼性の高いITインフラストラクチャを構築します。厳密なSLAの実装を確保し、OSの稼働時間が長く、システム要件が低い
  • ITインフラストラクチャの保守と管理を簡素化および短縮することにより、運用コストを削減します。


マイクロソフトは、Nano Serverの基本的なインストールに必要なディスク容量は400 MB以下であり、Windows Serverのフルエディションをインストールする場合の約20分の1であると約束しています。 これらの約束により、Nano ServerはWindows Serverドライバーの完全なサポートを保持することに注意することが重要です。



Nano Serverは、WMIおよびCore PowerShellを介して管理されます。これには、.NETの欠如に関連するコマンドレットのセットと制限がありません。



Nano Serverは、仮想化サーバーと仮想マシンの両方に展開できます。 さらに、Windows ServerおよびHyper-Vコンテナーを作成する両方の方法がサポートされています。 Windows Server Containersは、Windows Serverのインスタンスでアプリケーションランタイムを分離するWindows Server TP2の新機能です。 Hyper-Vコンテナーは、仮想化環境でこれらの機能を拡張します。この場合、ホストOSまたは他のHyper-Vコンテナーの動作に対するゲストOSの影響を排除します。



仮想環境のセキュリティの強化

企業がインフラストラクチャをパブリッククラウドに移行することに抵抗を感じるのはなぜですか？ サービスプロバイダーへの信頼が最も重要な選択基準であり、次にデータセキュリティの問題が続くと考えています。



Microsoftはこの方向で取り組み、仮想マシンでデータ暗号化を使用できる新しいTrusted Platform Module（TMP）機能をHyper-Vに導入しました。 TPMでは、仮想マシンがパブリッククラウドにある場合でも、BitLockerを使用してデータを暗号化できます。



Windows Server TP2のHyper-Vは、暗号化され、システム管理者によるアクセスから保護されているデータであるシールド仮想マシンの作成をサポートしています。 この機能は、安全なサーバーグループの整合性を維持し、データ暗号化メカニズムを制御する新しいHost Guardianサービスサーバーロールに基づいています。 System Centerと組み合わせて、新しい興味深い自動化シナリオが表示されます。

  • 新しい保護された仮想マシンを作成し、既存の仮想マシンをホットな仮想マシン上で保護された仮想マシンに変換できます。
  • データセキュリティを確保しながら、保護された仮想マシンをプライベートクラウドまたはパブリッククラウドに移動し、外部管理者に制御を移すことができます。
  • Virtual Machine Managerから新しい安全な仮想マシンを作成できます。 同時に、VMMでは、作成時にバーチャルマシンテンプレートのイメージが信頼され、一貫性があるかどうかを追跡できます。
  • 外部管理者は、以前と同じ方法で安全な仮想化サーバー間で安全な仮想マシンを移行できます。
  • 外部管理者は、以前と同じ方法で保護された仮想マシンをバックアップおよび復元できます。
  • また、Linux仮想マシンの場合、オペレーティングシステムのロード時にカーネルソースコードの整合性をチェックするセキュアブート機能が登場しました。


仮想マシンの構成

Windows Server TP2は、多くの典型的な機能を更新しました。 これらを使用する場合、仮想マシンを再起動する必要はありません。

  • 動的メモリが構成されていない仮想マシンでのRAMの使用に関するデータのサイズ変更と表示。
  • 既存のディスクのHyper-Vレプリカの構成に影響を与えずに、新しい仮想ディスクのレプリケーションステータスを変更します。
  • ネットワークアダプターのホットアド/削除。
  • ロールバックを使用したWindows Server 2012からのHyper-Vクラスターのアップグレード。
  • GUIで仮想マシンのバージョンを表示します（Windows Server 2012ではバージョン5、Windows Server Technical Preview 2ではバージョン6.2）
  • Windows Updateを使用して統合サービスを更新します。


インフラストラクチャパフォーマンス管理

私は喜びを隠しません-最後に、Windows Server TP2では、ストレージ品質パラメーター（ストレージQoS）の管理が登場しました。 ある種の仮想マシンが突然ストレージシステムを「遅く」することを心配する必要はありません。



ストレージQoSを使用すると、スケーラブルなファイルストレージまたはCSVを共有する仮想マシンのグループにIOPSポリシーを設定できます。 ポリシーでは、仮想マシンの最小および最大IOPS値を設定できます。



新しい弾力性機能

Virtual Machine Storage ResiliencyおよびVirtual Machine Cluster Resiliencyの新機能を使用すると、ストレージシステムの予期しない障害から仮想マシンを保護できます。 およびcsv。



ストレージが使用できない期間中、Storage Resiliencyは仮想マシンを一時停止状態（一時停止）に保ちます。 この機能は、Windows Server 2012に既に存在する復元性のあるファイルハンドルに似ていますが、ストレージの復元性はより信頼性が高いため、 長期間のダウンタイムに対処できます（復元性のあるファイルハンドルは、30秒以下のダウンタイムの一時停止を提供できます）。



Virtual Machine Cluster Resiliency機能の主な目的は、クラスターノードが短時間切断された場合にクラスターノードのアンロードを回避することです。 代わりに、マシンは切断されたノードで動作し続け、4分以内に通信が再開した場合、クラスターは正常に動作し続けます。 ノードが「信頼できない」ステータスを受信した場合（たとえば、いくつかの定期的な障害が記録されます）、クラスター整合性の復元時に、仮想マシンは自動的に他のノードに移動され、問題は隔離されます。



共有仮想ディスクの機能強化

Windows Server 2012では、複数の仮想マシンで共有VHDX共有仮想ディスクを使用する機能が導入されました。 この機能により、一般的な仮想フェールオーバークラスターを迅速に実装できます。 この機能の欠点は、ウイルスサーバーで実行されるバックアップスクリプトをサポートしないことです。



Windows Server TP2はこの問題を修正しました-ホストベースの共有VHDXバックアップスクリプトが機能し、ホットディスクのサイズ変更が可能になりました。 これらの機能では新しいVHDS形式への変換が必要になることに注意してください。



おわりに

この記事では、Windows Server 2012と比較して大幅に改善されたWindows Server Technical Preview 2の新機能および変更された機能について説明しました。 Microsoftは自信を持ってハイパーバイザーとクラウドコンピューティングの軍拡競争のトーンを設定しています。 そして、数年前にインフラストラクチャでのHyper-Vの使用について疑問がある場合は、この選択につながる機会をもう一度検討する価値があります。



私のプロジェクトの経験から、ITで最新のソリューションを使用すると、ITインフラストラクチャの運用効率が大幅に向上することがわかります。 計画を変更するときは、常に数歩先の動きを計算する価値があります。
