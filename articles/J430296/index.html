あなたのアイデアをアプリにしてください。 サーバーレスアプリケーション-段階的な手順





2018年、サーバーレスは、一度も実行したことがない場合でも、アプリケーションバックエンドを作成する最速の方法です。 はい、無数のアプリケーションビルダー、MBaaSまたはBaaSについて知っていますが、サーバーレスは基本的なアプリケーションだけでなく、コンストラクターでは実行できないスケーラブルで複雑なバックエンドにも適していることを示したいと思います。



先日、ロシアで最初のサーバーレスクラウド 、 Swiftyプラットフォームに基づくRusonyx Serverlessを発表しました。 プラットフォームを使用して最初の3か月は無料なので、誰でもサーバーレスアプローチを試すことができます。



この記事では、認証、ユーザープロファイル、イメージストレージ、そして実際にはサーバーレスアプローチを使用したタスク管理を備えた簡単なToDoアプリケーションの作成方法を説明します。 これは当然Swiftyで行いますが、ここでのアプローチはすべてのサーバーレスソリューションでほぼ同じです。 完成したアプリケーションの例はここにあります 。 フロントエンドはvue.jsで記述され、組み込みのObject Storage（S3）で実行され、バックエンドはGoおよびPythonの関数で実行されます。



このガイドに合格するためにJavaScript、goshechki、およびpythonの知識は必要ありません。 アプリケーションのソースコードは、 ここで取得できます 。



ユーザー認証



最初の段階では、アプリケーションバックエンド用の関数自体を作成します。 まず、Swiftyアカウントが必要です 。 ここで作成できます 。 ダッシュボードにログインすると、この画面が表示されます。







アカウントを取得したら、関数自体の作成を開始できます。 Swiftyには、サインアップ、サインイン、ログアウトの基本操作と、ユーザープロファイルの作成、変更、取得、削除の機能を提供する認証サービス-認証が含まれています。 また、Facebookとの統合、および既に作成されたプロファイルをFacebookプロファイルにリンクする機能も備えています。 しかし、それらはまだ必要ありません。 たぶん後で。



認証サービスを作成します 。



  • Swifty-> Authentication Servicesを開きます。
  • [認証データベースの作成]をクリックし、todoappデータベースに名前を付けます。 この名前は引き続き使用しますが、必要に応じてベースに名前を付けることができます。


その結果、多くのものが作成されます。



  1. todoapp.base関数 -サインアップ、サインイン、ログアウトのユーザーを作成し、OAuth 2.0プロトコルを実装します。
  2. todoapp.fb関数-fbを介してユーザーを認証できます。
  3. todoapp.link関数 -すでに作成されたユーザーのアカウントをfbのアカウントにリンクします。
  4. todoapp.profiles関数-MongoDBでユーザープロファイルを作成、更新、削除します。
  5. Todoapp_mgo DB-ユーザーアカウントを保存するためのMongo。
  6. Todoapp_profiles-ユーザープロファイルを保存するためのMongo DB 。
  7. 認証ミドルウェア（AuthMW）は、API関数にアクセスするときに、todoapp.base関数が発行したJWTトークンをチェックすることでユーザー認証を検証できるプロキシです。 トークンがないか、正しくありません-APIへのリクエストは拒否されます。


関数の名前に「。」を使用して、それらをフォルダーに分けます。 したがって、todoapp.newfunctionという名前の新しい関数を作成すると、自動的にtodoappフォルダーに入り、newfunctionという名前でそこに表示されます。 関数のリストには、次のセットが含まれているはずです（図を参照）。







スキップできますが、よく読んでください



この段落は原則として省略できます。 認証サービスがどのように機能するかを理解し、Swiftyがどのように機能するかについてもう少し理解したい場合、そうではありません。 Goで記述されたtodoapp.base関数は、基本的な認証機能を提供しますが、アプリケーションのニーズを満たすためにその機能を拡張することを妨げるものはありません。 どのように変更しても、サインインとサインアウトに触れることなく、引き続き機能します。 この関数には、AuthMWという名前を保存する変数SWIFTY_AUTH_NAMEがあります。 この関数は、MongoDBとAuthMW自体へのアクセスも必要とします。これらは、関数のプロパティの[アクセス]タブに登録されています。 彼女は、関数にアクセスするために呼び出す必要があるリンクを持つREST APIトリガーも持っています。



todoapp.base関数は、ユーザーIDとパスワードを要求引数として渡すことを想定しています。 パスワードは暗号化されています。



そのようなクエリの例を次に示します。 



* Sign up: https://api.swifty.cloud:8686/call/012.../signup&userid=user@yourmail.com&password=xxxxxxxx * Sign in: https://api.swifty.cloud:8686/call/012.../signin&userid=user@yourmail.com&password=xxxxxxxx * Log out: https://api.swifty.cloud:8686/call/012.../leave&userid=user@yourmail.com


たとえば、サインインが成功した場合（関数が送信されたパスワードを正常に検証した場合）、JWTトークン付きのJSONを受け取ります。JWTトークンは、認証が有効になっている関数にアクセスするたびに使用する必要があります。 JWTトークンは、ベアラー認証スキームに基づいて作成されます。 OAuth 2.0とBearerの詳細については、 こちらをご覧ください 。



認証が成功しない場合、呼び出された関数は開始せず、リクエストはコード401を返します。



ユーザープロファイル管理



そのため、すべての関数には、関数を実行するために呼び出す必要があるリンクであるREST API URLがあります。 認証関数のこのリンクを取得するには、todoapp.base関数を開き、[トリガー]タブに移動し、REST APIのURLをコピーして、AUTH_URLとして保存します。 さらに、このリンクをアプリケーションのフロントエンドの構成ファイルに挿入する必要があります。







また、アプリケーションでユーザープロファイルを管理できるように、todoapp.profilesのAPI URLも必要です。 この関数を開き、「トリガー」タブに移動して、REST API URLをコピーし、PROFILE_URLとして保存します。



ユーザーアバター管理



このアプリケーションでは、ユーザーアバターをアップロードし、組み込みのオブジェクトストレージにファイルを保存する方法を示すこともできます。 ユーザーの写真は特別な機能を使用してアップロードされ、組み込みのオブジェクトストレージに保存されます。 画像へのアクセスは、関数または標準のS3 APIを使用して取得できます。アクセスキーは、UIの[オブジェクトストレージ]コントロールタブで取得できます。



画像管理機能を作成するには：



  1. タブ機能->新規機能->リポジトリから（テンプレート）に移動します。 すべての関数テンプレートは、パブリックgitリポジトリswifty.demoに保存されます。 このリポジトリはデフォルトで選択する必要があります。
  2. Avatar management（python）functionを選択し、Nextをクリックして、新しいtodoapp.avatar関数の名前を入力します。 作成をクリックします。
  3. 次に、[トリガー]タブに移動し、[トリガーの追加]をクリックして、REST API（URL）を選択します。 表示されるリンクをコピーして、PICTURE_URLとして保存します。


次に、Object Storageにバケットを作成して、ユーザーの写真を保存する必要があります。



  1. [オブジェクトストレージ]-> [バケットの作成]タブに移動します。 新しいバケットにtodoappimagesという名前を付けます。
  2. [機能]タブに移動-> todoapp.avatar->アクセス-> [追加]をクリックし、[オブジェクトストレージ]、新しく作成したtodoappimgaesバケットを選択して、[追加]をクリックします。


これで、関数は指定されたバケットにアクセスできます。 とてもシンプルで、関数内のバケットへのアクセスを登録する必要はありません。 指定する必要があるのは、環境変数を使用して画像を保存するバケットの関数のみです。



  1. タブ機能-> todoapp.avatar->変数に移動し、変数の作成をクリックします。
  2. 変数名-BUCKET_NAMEとその値-todoappimagesを入力します。


タスク管理



実際、タスク管理アプリケーションを作成していますが、最終的に対応する機能を作成するときが来ました。 タスクを作成し、完了済みとしてマークし、タスクを削除できるテンプレートがあります。



関数を作成します。



  1. タブ機能->新規機能->リポジトリから（テンプレート）に移動します。
  2. TODOアプリケーション関数（python）を選択し、[次へ]をクリックして、新しいtodoapp.tasks関数の名前を入力します。 作成をクリックします。
  3. 次に、[トリガー]タブに移動し、[トリガーの追加]をクリックして、REST API（URL）を選択します。 表示されるリンクをコピーして、TASKS_URLとして保存します。


次に、タスクを保存するデータベースが必要です。 最も簡単なオプションはMongoDBです。



  1. Mongo Database-> Create Databaseタブに移動し、todoapp_tasksというデータベースを作成します。
  2. [機能]タブに移動-> todoapp.tasks->アクセス-> [新しいデータベースを追加して追加]。


これで、関数はtodoapp_tasksデータベースにアクセスでき、swiftyライブラリを使用して関数からアクセスできます。次に例を示します。



db = swifty.MongoDatabase(os.getenv('TASKS_DB_NAME'))







環境変数をデータベース名で登録するだけです：



  1. タブ機能-> todoapp.tasks->変数に移動し、変数の作成をクリックします。
  2. 変数名-TASKS_DB_NAMEとその値-todoapp_tasksを入力します。


機能の認証を有効にする



ユーザー認証用のバックエンドと、アプリケーションのバックエンドを構成する機能自体がある場合は、認証を有効にして、承認されたユーザーのみがプロファイル、アバター、タスク管理機能にアクセスできるようにします。



特定の機能のトークン検証を有効にする方法：



  1. [関数]タブに移動し、関数todoapp.tasksおよびtodoapp.avatarを選択します。
  2. [認証の管理]をクリックしてtodoappサービスを選択し、[有効にする]をクリックします。


これで、関数todoapp.tasksおよびtodoapp.avatarは、todoapp.baseを使用して生成された正しいJWTトークンを持つユーザーに対してのみ実行されます。

アプリケーションを公開



アプリケーションのフロントエンドに対処しましょう。 フロントエンドはvue.jsで記述されており、設定ファイルに関数へのリンクを追加し、この更新された設定でアプリケーションを再構築するだけです。 ここではすべてがシンプルであり、vue.jsとJavaScriptの知識は必要ありません。



アプリケーションを再構築するには、node.jsをインストールする必要があります。 お持ちでない場合は、公式ガイドを使用してください。 Macをお持ちの場合は、 こちらに優れたガイドがあります 。 また、リポジトリーをコンピューターにプルするにはgitも必要です。 してください： 



 # git clone https://github.com/swiftycloud/swifty.todoapp


その後、 /swifty.todoapp/src



フォルダーに移動して、お気に入りのエディターでconfig.js



ファイルを開きます。 そこで変数を以前に保存したものに変更する必要があります。



export const AUTH_URL = "https://api.swifty.cloud/call/991..."

export const PROFILE_URL = "https://api.swifty.cloud/call/281..."

export const PICTURE_URL = "https://api.swifty.cloud/call/e6a..."

export const TASKS_URL = "https://api.swifty.cloud/call/4b1..."







FBに関連する変数はまだ必要ありません。



次に、アプリケーションを再構築する必要があります。 



 # npm run build


...

ビルドが完了しました。 distディレクトリーをデプロイする準備ができました。



アプリケーションをビルドする前に、ローカルでテストすることもできます。 



 # npm run serve


http：// localhost：8080のブラウザからアクセスします



Object Storageを使用して、アプリケーションの静的ファイルを保存します。 [オブジェクトストレージ]タブに移動し、todoappバケットを作成し、/ swifty.todoapp / dist /swifty.todoapp/dist/



フォルダーからファイルをアップロードし、フォルダー名を確認します（手動で作成する必要があります）。



最後のステップは、アプリケーションを公開することです。 [詳細]-> [HTTPサーバー設定]をクリックし、バケットのHTTPサーバーを有効にします。 表示されるリンクをコピーしてフォローします-これがあなたのアプリケーションです！







これで、アカウントを作成して実際に試すことができます。 ここで機能画面に移動すると、認証、プロファイルの読み込みなどのために機能が起動されたばかりであることがわかります。
