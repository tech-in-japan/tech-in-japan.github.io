VMware Playerを使用してUbuntuにWindows XPをインストールする

私は自分自身をUbuntuに設定しました。

しかし、Winが必要な問題がありました。 Linuxでは動作しない十分な動作プログラムがあります。

私はWineを使ってみましたが、それは馬鹿げた仕事であることがわかりました。 一部のプログラムは開始されず、一部のプログラムは開始されましたが、正常に機能しませんでした。



そこで、VMwareから始めることにしました。 ご存じのとおり、WMware Workstationは無料ではありません。

しかし、WMware Player-絶対に無料のソフトウェアがあります。

ただし、* .vmxイメージファイルが必要です。

どこで入手できますか？

抜け道がありました。 グーグルはそのような解決策を見つけました。

仮想マシンのイメージファイルを作成するには、 Qemuプログラムが使用されます。

ステージ1

実際にプログラムをダウンロードして、Windows XPにインストールしました。

その後、ターミナルに移動し、特定のパラメーターでプログラムを実行する必要があります。

qemu-img.exe create -f vmdk WindowsXPPro.vmdk 2G

Formating 'WindowsXPPro.vmdk', fmt=vmdk, size=2097152 kB







示されているボリュームは2ギガバイトです。これで十分です。 結果のファイルはWindowsXPPro.vmdkと呼ばれます



このファイルをUbuntuに転送し、WindowsXPPro.vmxという別のファイルを作成します。このファイルには次のように記述します。

config.version = "8"

virtualHW.version = "3"

ide0:0.present = "TRUE"

ide0:0.filename = "WindowsXPPro.vmdk"

memsize = "64"

MemAllowAutoScaleDown = "FALSE"

ide1:0.present = "TRUE"

ide1:0.fileName = "auto detect"

ide1:0.deviceType = "cdrom-raw"

ide1:0.autodetect = "TRUE"

floppy0.present = "FALSE"

ethernet0.present = "TRUE"

usb.present = "TRUE"

sound.present = "TRUE"

sound.virtualDev = "es1371"

displayName = "Windows XP Pro"

guestOS = "winXPPro"

nvram = "WindowsXPPro.nvram"

MemTrimRate = "-1"



ide0:0.redo = ""

ethernet0.addressType = "generated"

uuid.location = "56 4d 5c cc 3d 4a 43 29-55 89 5c 28 1e 7e 06 58"

uuid.bios = "56 4d 5c cc 3d 4a 43 29-55 89 5c 28 1e 7e 06 58"

ethernet0.generatedAddress = "00:0c:29:7e:06:58"

ethernet0.generatedAddressOffset = "0"



tools.syncTime = "TRUE"

ide1:0.startConnected = "TRUE"



uuid.action = "create"



checkpoint.vmState = ""







これらの2つのファイルは同じフォルダーになければなりません。



ステージ2

次に、XPでブートディスクを取り出し、CD-ROMに挿入します。 DOSおよびNCを備えたマルチブートディスクは適していません。 MS Windows PEを備えたライブディスクのみが適していると思います。

その後、VMwareを使用してWindowsXPPro.vmxファイルを実行します。

Windows XPのインストールが開始されました。
