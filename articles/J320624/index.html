KKM ATOLをAndroidStudioに接続します（FZ-54への更新）

こんにちは、Habr。 昨年の夏、KKM ATOLをAndroidStudioのプロジェクトに接続する必要がありました。 タスクを正常に完了した後、私はHabréに投稿を公開しました。これは、 自分の道を進む人たちの道を促進するためです。KKMATOLをAndroidStudioに接続します。



法律の更新（FZ-54）に照らして、更新されたドライバーがKKM ATOLに対して発行されました。これには、前述とは少し異なる接続方法が必要です。







カットの下に、私が正確に何をしたかが表示されます。 しかし、Android開発のプロはおそらく自分にとって興味深いものを見つけることはないだろうとすぐに言いますが、私のような初心者にとっては、開発パスが少し簡単になるでしょう。



更新されたドライバーを接続するために、次の手順が実行されました。



1）プロジェクトから古いモジュールを削除しました：ファイル-プロジェクト構造-モジュールセクションから古いモジュールを選択し、赤いマイナスをクリックします。

2）新しいモジュールを追加しました：ファイル-新規-インポートモジュール-フォルダーを探します（提供されたフォルダー内にTrade_drivers_drivers / android / jar）FptrLibResを追加し、プロジェクトに追加します。

3）次に、前の記事と同様に、jniLibs（パス：app / src / main）にarmeabiおよびarmeabi-v7aフォルダーを追加します。

4）libsフォルダーに追加（パス：app / src / main）-jarファイル：fptrlib.jar、fptrproxylib.jar、paycardlib.jar、usblib.jar jarファイルを右クリックして、「ライブラリーとして追加」を選択します";

5）gradleに変更を加えます（前回の記事で作成したテストプロジェクトの例を示します）。 



dependencies { compile fileTree(dir: 'libs', include: ['*.jar']) testCompile 'junit:junit:4.12' compile 'com.android.support:appcompat-v7:23.4.0' compile project(':fptrRes') compile files('src/main/libs/fptrproxylib.jar') compile files('src/main/libs/paycardlib.jar') compile files('src/main/libs/usblib.jar') compile files('src/main/libs/fptrlib.jar') }


6）マニフェストから行を削除します（以前のテストプロジェクトのように行があった場合）。 



  <activity android:name="com.atol.drivers.fptr.settings.SettingsActivity" android:label=" " android:windowSoftInputMode="adjustPan"> <intent-filter> <action android:name="android.intent.action.VIEW"/> </intent-filter> </activity> <activity android:name="com.atol.drivers.fptr.settings.DeviceListActivity" android:configChanges="orientation|keyboardHidden" android:label=" "/>


7）コードにエラーが表示されます。 修正するには、次を変更します。 



 fptr = new IFptr();  fptr = new Fptr();


また、「Receept」という単語を含むステートメントに対してエラーをスローします。 彼らは「領収書」に変更する必要があります。
