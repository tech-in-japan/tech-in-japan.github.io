ETegro Hyperion RS430 G4-ビッグデータに対応

ユビキタスな「クラウド」に加えて、サーバーの世界で最も人気があり議論されているトピックの1つは、ビッグデータ-さまざまなタイプの非常に大量の構造化されていないデータを扱う概念です。 ソフトウェアデータ処理の問題については触れませんが、私たちの意見では、実行に値するプラットフォームである新しいETegro Hyperion RS430 G4サーバーを示します。











それでは、ビッグデータを扱うには何が必要ですか？ もちろん、かなり強力なコンピューティングプラットフォームからの高度なアクセス性を備えたディスクスペース、大量のディスクスペース。 まあ、そしていつものように、これは簡単に事前設定でき、便利に維持でき、同時にほとんどの場合、可能な限り少ないラックスペースを使い果たすことが非常に望ましいです。



これをすべて実装するには、4Uサーバーを使用することをお勧めします。 超高密度レイアウトのおかげで、70台の3.5インチドライブとマザーボードを備えた2つの独立したプラットフォームを配置できました。各プラットフォームは35台のドライブにアクセスでき、さらに手術室に非常に適した2台の専用2.5インチドライブを備えています。システム。 当然、すべてのドライブはホットスワップ可能です。

さらに、リストで認識しやすい特定の数と特性を使用します。

2つのマザーボード。それぞれに次の機能があります。

  • Intel C602チップセット
  • 最大2つのIntel Xeon E5-2600プロセッサー
  • 4チャネル動作モードをサポートする最大512 GBのDDR3-1600 RAM
  • 3スロットPCI-Express x8バージョン3.0
  • SFP +スロットを備えた2つの10Gイーサネットポート
  • SAS RAID、SAS HBAおよび10Gイーサネット、ファイバーチャネル、InfiniBandアダプターをオプションとしてインストールする機能
  • 専用イーサネットポートを備えた統合IPMI 2.0およびKVM over IPによる監視と制御
  • Windows Server 2012ファミリのサーバーオペレーティングシステム、SuSE Linux Enterprise Server 11、およびRed Hat Enterprise Linux 6のサポート。


80以上のプラチナ規格に従って認定された、それぞれ1200 Wの電力を持つ4つの電源装置が、システム全体に電力を供給します。 モジュールはホットスワップをサポートし、2 + 2スキームで複製されます。



しかし、十分なテキスト、このハンサムな男を見てみましょう。











この怪物を見たときに最初に頭に浮かぶのは、ソビエトのプロセッサが48本の脚と2つのキャリングハンドルを持っているという事実についての古い冗談です。 確かに、RS430 G4には4つのペンがあり、その外観はその徹底性に対する敬意を呼び起こします。 ただし、サーバーを移動する必要があるとすぐに、それらが非常にすばやく必要な理由を理解する必要があります-完全な「梱包」では約115キログラムの重量があるため、ケースの端まで運ぶことができません。











上部パネルの約4分の3が印象的なカバーで覆われており、多数のハードドライブへのアクセスをブロックしています。 システム管理者の運命を多少楽にするために設計された、サーバー内の何が、どこに、どのように配置されているかについて、かなり印象的ではあるがまだ簡潔な説明が自由に配置されています。











写真のリズムは魅力的ですね。 2つのマザーボード間で均等に分割された14行の5ドライブ。 そして、どういうわけか自動的に70を4 TBに増やし始めます。これは現在のハードドライブの最大容量であり、ペタバイトの4分の1を超える結果になると思います...

平らな細長い列に配置されたハードディスクは、ホットスワップを整理するためのかなり単純なスレッドに配置されています。 そのシンプルさにもかかわらず、スライドはまだ金属製です。強度の問題ではなく、熱放散を提供するものであり、プラスチックはこのような配置密度ではるかに悪化します。











私たちには、バックパネルはそれほど興味深いものではないようです。結局のところ、このサーバーのコンピューティング部分の構成を評価できるのはそこからです。 上部に2つの格納式プラットフォームが快適に配置され、それぞれがフルウェイトデュアルプロセッササーバーであり、それぞれ1200 Wの容量を持つまったく同じ4つの電源が下部全体を占めました。 私たちは統一を熱心に支持しているので、電源はETegroロゴのあるプラットフォームを見たことがある人なら誰でも知っているでしょう。 当然、ハードドライブまたは電源ユニットから始まり、各コンピューティングプラットフォームで終わるモジュールは、分解することなくサーバーから取り外すことができます。そのような機器のメンテナンスは、できるだけシンプルで便利である必要があります。















前述のように、各モジュールは、独自のメモリ、プロセッサ、追加のコントローラを備えた完全なシステムです。 これらはすべて、1つのコネクタでディスクアレイに接続された1つのプルアウトプラットフォームに非常に高密度にパックされています。 すべてのサーバーのもう1つの特徴的な機能は、2.5インチハードドライブまたはSSDの個別のフットプリントであり、OSを配置してデータキャリアから物理的に分離することが非常に便利です。 この場合、各サーバープラットフォームにはこれらのシートが2つあることがわかります。RAID1は優れたものであり、1つのディスクの障害のためにこのような強力なソリューションがアイドル状態になることはほとんどありません。 もちろん、これらのディスクもレール上にあり、ホットスワップをサポートしています。これを回避できるのに、なぜモジュール全体を削除するのですか？











各モジュールの背面パネルは、それぞれが実際には本格的なサーバーであることを思い出させます。 ここと独自の本格的な従来のコントロールボタンのセット、2つのSFP + 10Gイーサネットポート、変更されていないD-SUB、サービスCOMおよびRJ-45。 もちろん、拡張スロット用のプラグもあります。2つは高さを抑えたボード用で、1つはフルサイズのもの用です。















内部に見えるのは、ボードと電源のモジュールと前面の背面の間の小さなスペースを覗くだけです。これは印象的なディスクの配列です。 また、別の配列があり、サーバーの機能にとっても重要です。 12の強力なファンが互いに強く押し付け、フロントパネルから背面に熱気を送り込んだのはここです。 ファンは通常よりも明らかに幅広です。 高回転速度により、強い圧力で強力な空気流を作り出すことができます。これは、このような密なレイアウトのサーバーに必要なだけです。
