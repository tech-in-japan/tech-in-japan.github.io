Clever Geek Handbook

有名な大学の卒業証書なしでマイクロソフト、アマゾン、またはツイッターにアクセスする方法

この記事は、仕事を探す準備をしている人を対象としており、スタンフォード大学のコンピュータサイエンスの卒業証書なしではトップ企業に入らないのではないかと心配するかもしれません。 彼らはおそらく、誰もあなたをFacebookやMicrosoftに連れて行かないだろうと言ったでしょう。 しかし、これは完全に可能であることを伝えたいと思います。 Twitterで自分の夢の仕事をどうやって成し遂げたかについての私の話です。









この記事の内容：



  • 私の伝記の一部
  • 世界のトップIT企業：Facebook、Google、Amazon、LinkedIn、Microsoft、Twitter、Pinterest、Snapchatなどへのインタビューに招待された経緯
  • プログラマーとしていくつかの仕事のオファーを受けた経緯
  • この経験から学んだ教訓


履歴書



私はアイビーリーグ大学を卒業していません。 アイダホ州の公立大学で2年間勉強した後、小さなカトリック大学に編入し、そこでコンピューターサイエンスを卒業しました。



面白いと思ったからといって、1年目にプログラミングを始めました。 それまでの私のコンピューターでの経験はすべて、任天堂のコンソールに対する中国の偽物で使い果たされ、それが死んだとしても、カートリッジを挿入するだけで十分でした。



自分を養うために、教育を受けるまで、私は一度にいくつかのアルバイトをしました-床を洗って、映画館で食べ物を売りました。 卒業後、私には仕事がありませんでした。 できる限りすべての主要IT企業に履歴書を送りました。運がよければ、電話インタビューへの招待状をいくつか受け取りました。



当時、私はそれがどんなものかさえ想像もしていませんでした-技術職へのインタビュー、さらにはその準備方法。 リンクリストまたはバイナリツリーとは何かを聞かれると思ってインタビューに来ました。



当然、これらすべてのインタビューで私は中退しました。



次のステップ



それから、私は自分のレベルが良いかどうかについてあまり考えませんでした。 私は速く捕まっていることを知っていました。 必要なのは、自分自身を証明する機会だけでした。



一般的な知恵によれば、ネットワークは捨てられ、より広くされる必要があります。 だから私はやった。



次の段階でやったことについて、私は特に誇りに思っています。 Craigslistから欠員を収集し、特定のキーワードを含む投稿を選択し、表に記載されているメールアドレスを入力する簡単なPythonスクリプトを作成しました。 最も独創的な決定ではありませんが、Craigslistに仕事を投稿する人々は驚くほど確実に投稿を呼び出しています。



ただし、Craigslistは、データをアップロードしようとする人があまり好きではありません。 制限を回避するために、VPNを介してスクリプトを実行し、数分ごとにプロセスを一時停止するタイマーを設定しました。 完全ではありませんが、そのようなスキームは機能しました。



最終的に、サンフランシスコ、ポートランド、スポコン、シアトルにある組織の約500の住所を収集しました。 関連性と日付で結果をフィルター処理し、パラメーターをさらに追加して結果を最適化し続けました。



判明したように、Craigslistからデータを収集し、自動的にレターを送信するボットがすでに市場にいくつかありました。 ほとんどの場合、これらは米国市場に参入しようとするオフショア企業でした。



私が頼ったトリックの1つは次のとおりでした。主題に関連するキーワードが含まれるように、文字のテキストを作成しました。 次に、空席のテキストに基づいて詳細を追加し、特定の要件に合わせて提案がシャープに見えるようにしました。 私が実施したクイックA / Bテストから判断すると、その後の回答の割合は大幅に増加し、2〜3％から10％になりました。



私が送った500通の手紙に約500通の返信が来ました。 この50人のうちわずかな割合で、電話インタビューを手配しました。 時間がなくなったため、500で停止しました。すでにどこかに行く必要がありました。 当時の優先事項は結果であり、報道の幅ではなかった。



幸いなことに、私はシアトルの新興企業でジュニア開発者として仕事に就きました。 彼らのオフィスはカークランドにあったので、インタビューを受けるために45分間バスに乗らなければなりませんでした。



私はそこで3年半働いて、Amazon AWS、EC2、DynamoDB、SQS、Dockerなど多くのことを学びました。 この間、私はプログラマーとして大きく成長しました。 モジュラーで保守しやすいコードを書き、設計について正しく推論し、人々と仕事をするときの問題を解決する方法を学びました。



Microsoft、Amazon、LinkedInから来た知識豊富な人々のグループと一緒に働いて、彼らの間で「スポンジ」の役割を演じようとしました。 私は彼らが私にくれたものをすべて吸収しました。 これが私のキャリアに大きな影響を与えたと確信しています。



スタートアップで働く



スタートアップでは、まれな例外を除き、DevOpsでのまれな攻撃を伴うバックエンドでのみ作業しました。 最初は、通常はそれほど大きくない機能を補完または変更する関数を作成しました。 しかし、コードベースに精通し、コード検証手順を経験する絶好の機会でした。



1年後、彼らはコードベースの断片全体を私に信頼し始め、それから一連の関数からサービスを作るように指示されました。 スタートアップは、サービス指向アーキテクチャへの切り替えを始めたばかりでした。 サイトのさまざまなコンポーネントをサービスに変え始めました。その間に、RESTful、認証、AWS、パブリッシャー/サブスクライバーパターン、分散システムなどについて多くのことを学びました。



最も興味深いのは、本や大学の授業中に何も学ばなかったことです。 いくつかの機能を実装するか、いくつかのボトルネックを解消する必要がありました。 そして、私はこのようなものです：まあ、理解しましょう！



1回または2回以上、分析麻痺に陥りました。過度に詳細な分析が問題の解決の進行を妨げていた状態です。 開発の機会のほとんどが生じたのは、これらの困難な瞬間でした。 私は、機能の範囲の定義、交渉、監視、アラートの設定、文書のコンパイルを改善しました。 このプロセスのすべてのステップで、マスターする必要がある新しい概念が発見されました。 この2、3年で、プログラミングスキルの面でも人間の資質の面でも、私はこれまで以上に成長してきました。



インタビューの準備



就職活動の最初のラウンドで苦しんだ後、私は次回面接の準備をすることを約束しました。



私は、自分が得意なもの、弱いもの、引き上げてよいものを書くことから準備を始めました。 すべてのスキルを、データ構造、アルゴリズム、システム設計の3つのカテゴリに分けました。



専門分野では、私は主にPHPで働き、大学ではC ++で書いたので、インタビューのためにそれほど大きくないものを選ぶことにしました。



これらの基準に基づいて、Pythonを選択しました。 これは学習するのに最適なオプションです。簡単に消化でき、多くのデフォルトのデータ構造をサポートし、ボード上ですばやくコードを書くことができます。 このような YouTubeのチュートリアルとドキュメントでPythonをマスターしました。 個人的にはPython 2.xが好きですが、Python 2.xとPython 3のどちらかを選択できます。



ところで、私がPythonを選んだ理由の1つは、非常に読みやすく、書くときにコンパクトになることです。 C ++とPythonを比較する簡単な例を次に示します。



C ++で降順でソートするためのプログラム： 



#include <bits/stdc++.h> using namespace std; int main() { int arr[] = {1,10,0,4,5} int n = size(arr)/sizeof(arr[0]); sort(arr, arr + n, greater<int>()); for (int i = 0; i < n; i++) { cout << arr[i] << " "; } return 0; }


同じPythonプログラムと比較してください： 



 a = [1,2,4,5,1000] a.sort(reverse=True) print a


インタビューを行う専門家から、他のことは同じであるため、言語を選択するときは簡潔さを重視する方が良いと聞きました。 割り当てられた45分の時間のうち、タスクの解決に実際に費やすことができる時間が長いほど、より良い結果が得られます。



ヒント：ボード上でコードをより速く書くために、かさばらない言語を選択してください。



トレーニングモードで



約1週間ほどで、LeetCode、HackerRank、Project Eulerの簡単なタスクを実行して、これらのサイトのインターフェイスに精通し、Pythonで書くことに慣れました。 この間、私はいくつかのプログラミング言語で自分のレベルをより良く評価することができました。 別の週は、できるだけ広い範囲をカバーし、可能な限り深く潜りながら、タスクの設計に取り組みました。



非常にエキサイティングでした。多くの場合、iOSアプリケーションを使用して、それらがどのように作成されるかを見ました。 Instagramをゼロから作成する方法を考えてみましょう（これはまさにFacebookで尋ねられた質問です）。



私のポートフォリオには、APIとサービス指向アーキテクチャに関する仕事があります。 したがって、私は喜んでこの機会に自分のバージョンのInstagramをどのように設計したかを伝えました。 サイドプロジェクトのおかげで、iOSでの開発経験もあるので、同時にコールバック、プルテクノロジー、長期投票に関する言葉を台無しにしました。



Instagramの自分のバージョンで見たい機能をリストアップすることから始めました：いいね、写真のアップロード、簡単なフィード。 関数の範囲を定義することで、これらのシナリオはすべて馴染みがあるため、非常に強固なAPIを構築できました。



次に、ユーザーがバックエンドと対話する方法と、データがそこに格納される方法を示すいくつかの高レベルの図を描きました。 私は必要最小限のコンポーネントを徐々に追加し、ボトルネックを積極的に検索しました。 要件が何であり、このテクノロジーまたはそのテクノロジーがどのように適合するかについて、合理的な仮定を立てました（私は強調します： 正当化 ）。 また、これと同じくらい重要なのは、どのテクノロジーもまったく適合しないことです。



たとえば、特定の種類のデータ（ヒント：スケール、開発速度、スキーマレビュー）を保存するためにMySQLではなくCassandraを使用する価値があるのはなぜですか？OAuthは単純な認証よりも優れており、データをキャッシュするときのRedisとMemcached、ストリーミングまたはバッチ処理などよりも優れています。



ここでは、さまざまなエリアを探索できます。通常、1時間ごとのセッションでは十分ではありません。 このような質問に適切に答えるためには、業界のさまざまな技術のトレードオフ、賛否両論の問題を徹底的に研究する必要があります。 このため、 HighScalabilityなどのサイトをお勧めします。



これを同僚との通常のブレーンストーミングセッションとして扱い、可能な限り幅広いアプローチと詳細な研究に努めてください。



これらのデザインセッションは、あなたが知っていることとどれだけうまくいくかを引き出すように設計されていること、そしてこれがあなた自身を良い光で示す機会であることに注意することが不可欠です。 元Facebookデザイナーがデザインタスクにアプローチする方法についてこのビデオを見ました。 彼の推薦はインタビューの間に私を大いに助けた。 私がそこから学んだ2つの主な考えは次のとおりです。自分自身のデザインについての会話の方向を定め、あなたができることを示すことです。



そこで、特定の言語のデータ構造（リンクリスト、ハッシュマップ、バイナリツリー、バイナリ検索ツリー、ヒープ、配列）、アルゴリズム（バイナリ検索、ハッシュ、動的プログラミング、並べ替え）、および構文とライブラリの観点からスキルを描きました。 （Pythonのラムダ関数、append、index）。 それから私は物事が最悪だったエリアを選択し、それに取り組み始めました。 これらはアルゴリズムであることが判明しました。

トップ企業へのインタビュー



経験は、控えめに言っても、刺激的です。 それらはまだジェットコースターです。



私は次のように時間を配分しました。履歴書の20％、企業と知り合うための20％、面接の準備のための60％。



履歴書の情報の更新に20％の時間を費やしましたが、3年以上も触れていません。 過去に行ったすべてのことを注意深く見て、複雑さのレベルに関係なく、最初から最後まで実行したプロジェクトを選択しました。



これには2つの理由がありました。 第一に、プロジェクトを終わらせるには、規律とリーダーシップの質、つまり雇用主と結び付けたいと思う正確な特性が必要です。



第二に、プロジェクトへのこの程度の関与は、私がそれについて非常に詳細かつ有益に話すことができることを意味します。 Twitterでのインタビューでは、この状況が重要であることが判明しました。そこでは、デザインのさまざまな側面だけでなく、彼らがこの方法で実装することに決めた理由についてもバイアスをかけられました。



さらに20％が情報を検索しました。 これにより、私は企業に十分な注意を払い、推奨のリクエストを送信したことを意味します。 推奨事項により、コールバックされる可能性が大幅に高まります。



私自身の経験から：スタートアップや中規模企業に約20通の「冷たい」手紙を送りましたが、そのうちのほんの数人が私に答えました。 しかし、従業員の1人が私に言葉をかけた企業のうち、大多数は1週間以内に私に手紙を書きました。 もちろん、これは特定の1つのケースにすぎませんが、それでも彼は何かについて語っています。



私は社交的ではなく、興味のある会社に私を推薦できる人はあまり知りませんでした。 この問題を解決するために、LinkedInに向かいました。 彼らの検索エンジンの助けを借りて、私は第1レベルと第2レベルのつながりを見つけました。 2番目のサークルのつながりは、友達のサークルからわずか1歩離れた人々です。 言い換えれば、あなたは彼らを保証できる共通の友人を持っています。











これは非常に重要です。なぜなら、特に現代市場の現実では、「冷たい」方法でインタビューを受けることは非常に、非常に難しいからです。 そのような場合、人々は注意することを好みます。 LinkedInは、情報収集フェーズに非常に役立つことが実証されています。



振り返ってみると、インタビューした企業の印象は次のとおりです。



  • Facebook / Googleはすべて非常に機械的です。 インタビューはパターンに従って行われ、個人的なつながりは感じませんでした。
  • Pinterestは最高のインタビュー体験ではありませんが、製品自体と会社は素晴らしいです。
  • Microsoft-チーム、特にマネージャーとそのリーダーが本当に好きでした。 質問は標準でしたが、アプローチは非常に個人的です。 私のお気に入りのポジションに対する真剣なライバル。 ただし、印象が異なる場合があります。Microsoftの各チームは独自の方法でインタビューを実施します。
  • Amazon-すべてが標準です。 このアプローチが好きな申請者の半分は、そうではありません。
  • Twitterは非常に面白く、パーソナライズされています。 インタビューはうまくいき、以前の経験と人間の資質に多くの注意が払われました。
  • Snapchatは、ロサンゼルスの高級オフィスであり、スタートアップに参加することを決めた大企業です。 その感覚は秘密のベールのようでした。
  • Lyft-私の家、素敵なオフィス、標準的なインタビューからそう遠くない。 彼らは強い印象を残しませんでした。


最初に私のお気に入りについて



Twitterへのインタビューは多くの点で難しいと思います。 しかし同時に、この経験は他の企業とのミーティングよりも個人的で興味深いものでした。



インタビューの最初の段階は、技術マネージャーと知り合うための電話での会話です。 その後、電話で1〜2回の技術面接が行われます。金額は結果によって異なります。 すべてが順調であれば、彼らはあなたが仕事に応募したオフィス、私の場合はシアトルのオフィスに飛ぶように誘います。 そこでは、それぞれ2人の従業員が参加する1時間15分の3つのセッションを通過します。



技術職の通常のスキームに従って、2つの電話インタビューが行われます。共同コード編集モードでタスクを解決します。



ただし、オフィスでのセッションは、フォーマルではなく、怖いものではありません。 以前のプロジェクトと、これまでに行ったすべてのことについて徹底的に質問されます。 特定のプロジェクトに参加したと宣言する場合は、そのプロジェクトについて詳しく話す準備をしてください。 ソリューションのソースとして、またはアイデアをテストするためにそれらを使用することをお勧めします。



私は彼らが私にプレッシャーをかけていると感じたことは一度もありません。 それどころか、従業員は非常に忠実であるように見えました。



そして今、他のみんなについて



Twitterと比較して、FacebookとGoogleのインタビューは私にとってより機械的なものに思えました。 彼らは1〜2回の電話インタビューと5〜6回のオフィスでのセッションで構成されていました。 各セッションでは、ボードにコードを書く必要がありましたが、ほぼ完璧なソリューションが必要でしたが、十分な時間が与えられました。



Facebookには2つのプログラミングセッションがありました。1つはデザイン用で、もう1つは個人の質を評価するためのものです。 一日の終わりに、追加の「シャドウ」セッションを行いましたが、その結果は全体のスコアに影響しませんでした。



Googleには5つのセッションがあり、すべてプログラミング用でした。 デザインにはまったく何もなかったし、私の過去のプロジェクトについて誰も話したことがなかった。 これが絶対に悪いと言っているわけではありません。 しかし、このようなアプローチはコンベヤー感覚を残し、開発者が自分の能力を適切に示すことができないように思えます。 一部の人にとっては、この形式が適しています-学生や試験のようなものです。



Pinterestでのインタビューは好きではありませんでした。 製品自体は興味深いようで、チームは非常に興味深いタスクに取り組んでいるようです 。 しかし、申請者としての私の経験は明らかに否定的でした。



Pinterestは3つのプログラミングセッションを開催します。1つはデザインです。 これら4つのうち、最後の1つは私を失望させました。 そして、ここに理由があります。



インタビュアーが遅れて来て、最初の数分間私の履歴書を読んだだけです。 その後、彼はボードにAPIを描き、彼がすべきことを簡単に説明し、どのように実装するかを尋ねました。 機能について議論し、ボード上でソリューションを書き始めました。 約5分後、私は振り向いて、彼が居眠りしているのを見ました！



クールじゃない。



インタビューの後、アンケートでこれについて書いたが、他の誰も私に連絡しなかった。



私はインタビューでどのような質問をされたのかについては詳しく説明しません。 代わりに、準備の過程で得たいくつかの有用な結論と推奨事項をリストします。



私が学んだこと



  • 履歴書を書くときは正直に言ってください。 ほとんどの企業はそこに何が書かれているかについて質問し、あなたが発明し始めたらすぐに理解します。 10の異なるプロジェクトに関する情報の10％を保有するよりも、1つのプロジェクトを100％把握する方が適切です。
  • 1ページの履歴書が望ましいです。 IT部門にとって、これは特に当てはまります。 暗黙のルールによれば、学位と多くの出版物を持っている人、または多数のプロジェクトに本当に深く関わっている人は、2ページを申請できます。 ところで、私の友人の1人がJobscanという会社を設立しました。この会社は履歴書を分析し、履歴書を改善するための具体的なアクションを提供します。 素晴らしいサービス、あなたは試すことができます。
  • 通信し、接続します。 プログラマー間の競争は非常に激しく、トップIT企業は1日に数千人の履歴書を処理しています。 従業員の推奨事項は、あなたが注目を集めるのに役立ちます。
  • 自分を売る方法を知ってください。 あなたに興味のある会社は、あなたがそれに興味を持っている理由を知りたいと思っています。 「仕事が必要で、何かに生きなければならない」というのは悪い答えです。 「インターネットでオプションを探して、あなたのサイトを見ました」-より良いが、まだそうではない。 良い答えは次のように聞こえます。「XでYに貢献できる興味深いプロジェクトがあると聞きました。過去のプロジェクトでの作業の過程で、Xに適用できるA、B、Cをマスターしました。何とか何とかだから。 （テンプレートとして使用しないでください。ここでは、パターンをキャッチする必要があります。会社に関する情報を収集し、伝記を参照して、お互いに似合う理由を示します。）


その他のヒント



技術面接は非常に複雑で、結果を予測するのが難しい場合があります。 しかし、最善の機会は準備ができている人たちです。

  • 事前に適切に準備を開始してください。 面接の準備が必要であることは誰もが知っていますが、それを正しく行う方法を誰もが知っているわけではありません。 他の価値あるスキルと同様に、良い結果を得るには、よく考えられたスキームに従って練習する必要があります。 よく考えられたスキームには、何らかのシステムが必要です。
  • 技術的なスキルを向上させるシステムを作成します。 さまざまな分野のスキルを10ポイントスケールで評価することから始め、最低の評価を獲得したスキルを引き上げ始めました。 私は、各概念を適切に解決するまで、各タイプの課題に数日を費やしました。 毎日Evernoteでメモを取りました。 プログラミングのトピックに関する録音用の特別なノートブックがありました。 そこで、ヒントとコツ、よくある間違いと誤解、さまざまなタイプのタスクを解決するためのアルゴリズムなどをもたらしました。
  • 学んだことをメモしておきます。 この目的でEvernoteとOneNoteの両方を使用しました。 OneNoteは厳密に技術的なものとコードを対象としたものであり、録音を好きなようにフォーマットできるという点が気に入っています。 Evernoteを推論とアイデアに使用しました。 上記では、アーキテクチャとシステム設計に関するメモを見ることができます。
  • これがほとんど役に立たないと思われる場合でも、可能なすべてを修正します。 私は非常に忘れっぽいので、コマンドラインのコマンドを含め、学んだことをすべて書き留めます。 また、私はプログラマーのブログをよく読みます。もし何かおもしろいことがあったら、すぐにEvernoteに入れました。 毎週または毎月の終わりに、メモを調べて整理します。 この習慣は、私のキャリアにとって非常に役立ちました。
  • リハーサルがあります。 これは非常に貴重な経験であり、皆さんにお勧めします。 私は友達に「インタビュー」をするように頼み、できるだけ頻繁にそのようなトレーニングを手配しようとしました。 連絡する人がいない場合は、テストインタビューを整理するサービスを提供するRefdashにアドバイスできます。 Google、Facebook、Microsoftなどの大規模IT企業の専門家を雇用しています。 これらの従業員は、プログラミングと設計のスキルを評価し、最も重要なこととして、作業の必要性に関する最終スコアと具体的な推奨事項を提示します。
  • 失敗には何の問題もありません。 仕事を見つける過程で、私は多くのインタビューを埋めました。 時々、それはあなたの日だけではないことが起こります。 それがすべて失敗に終わったとしても、これは世界の終わりではありません。 一般的に、企業は「はい」よりも「ノー」と答える可能性が高く、リスクは低くなります。 長い目で見れば、候補者を過小評価することは、過大評価するほど不採算ではありません。 最初の数回の失敗は間違いなく最も痛かった。 就職の最初の試みで、私は電話インタビューの段階で選別されました、そして、これは私自身に対する私の信頼を弱めました。 私は自分の能力を疑い始め、自分のスキルが現代の労働市場で求められていないことを恐れ始めました。 しかし、私は自分のために次の公式を思いつきました。10回失敗した場合、さらに10回試行します。 一度だけ成功すれば十分です。 この考え方は、挑戦し続ける自信と強さを与えてくれました。 ようやく最初の求人を受け取ったとき、すべてがずっと簡単になりました。
