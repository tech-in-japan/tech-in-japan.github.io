Tarantinkiオンラインゲームのアイデア：ユーザー+ ES対ユーザー

最近公開されたhabrの質問に基づいています。






みなさんこんにちは！

私の名前はデニスであり、オンラインゲームの開発と公開には能力がありません。 正直なところ、私は実質的にゲーム自体に時間を無駄にしません。 しかし、私が高い可能性を秘めた製品について考えたという考えは、私を去りません。 私はあなたに頑固に見えるかもしれませんが、私は本当にそう思う、私はそれを感じ、私はそれを信じています。



私のゲームのアイデアは、今では流行しているので、 User + ES vs Userという1つのフレーズに簡単に収まります 。

この場合のESは、 Akinatorで使用されるものと同様のエキスパートシステムです。







「20の質問」、「7の帽子」というタイプのゲーム、私はそれが何と呼ばれているのかわかりません。1人のユーザーが誰かを推測し、2番目のユーザーが閉じた質問（yes / no）を尋ねてこの誰かを推測しなければならないとき。 正しい答えが出された後、ユーザーはロールを切り替えます。 または、それらは同時に推測し、時間間隔を空けて順番に役割を変更します。



私の考えのチップは、エキスパートシステム（Akinatorなど）が推測ユーザーの側でもプレイするということです。 したがって、ユーザーにテキスト（音声）で質問するだけでなく、たとえばAkinatorの上位3つの質問から既成の質問を選択することで質問する機会を与えます。



1対1のゲームでは次のようになります。

最初のユーザーは、たとえばキャラクターを作成し、ゲームページのフォームに名前を入力して入力します。 2人目のユーザーの場合、ゲームが開始されます。 その前に2つの形式があります。



最初のフォーム：「イニシエーターのトップ3の質問」：

-あなたのキャラクターを知っていますか？

-あなたの女性キャラクターは？

-あなたのキャラクターはロシアに住んでいますか？



各質問はクリック可能です（ゲームの進行状況）。 質問の隣にはボタンがあります。これは片腕の盗賊です-クリックすると、質問が他の人に変わります。 たとえば、これらのオプションが気に入らない場合は。 たとえば、1クリックからゲーム通貨の1単位を引いたもの。 または、このボタンに他の機能を追加したり、インタラクティブにすることができます。 まったくメインの移動ボタンにしない場合。



2番目の形式：現在のゲームセッションの質問/回答履歴+ Akinatorによって提案されたパーセントで表される3（5）個のピース​​。以前の「上位3つの質問」のようにクリックすることもできます。 さらに、質問を自分で入力（たとえば）できるようにする必要があります。 選択したオプションをクリックすると、最初のユーザーに移動します。 まあ、すべてがそこにはっきりしているようです。 彼は答えます-はいまたはいいえ。



繰り返しますが、私はゲーム開発者ではありませんが、記録、推測時間、質問の最小数など、あらゆる種類のバイラル性とプレイヤー評価が思い浮かびます。 など さらに、プレイヤーがプレイしたばかりのゲームと対戦相手を評価する機会が必要だと思います。そこにある星、好きなもの、そのようなものを選択してください。



また、グループゲームのアイデアも大好きです。 1人のユーザーが推測を行い、グループが推測を行い、順番に質問する場合。 または、誰もがしばらくの間、古典的な「タランタン」を演奏しています。 しかし、それを考えるのは一般的に非常にクールであり、私はこれで完全な素人です。 収益化のオプションとゲーム通貨に関する私のフレーズについて-ここで、ご存じのように、私はこのテーマについても知りません。



アイデアが生まれた後に私が最初にしたことは、雨のタランティーヌのホスト、アレックス・デュバスにそれを説明することでした。 その後、彼はAkinatorの代表者に彼のアイデアを簡単に説明しました。 現時点では彼らからのフィードバックはありません。 少し待ってから、ゲームの説明とハブのアイデアを評価するリクエストを含む質問を作成しました その後、彼は「会社」セクションを開き、古いスカイプの連絡先からほこりを吹き飛ばし、よく知られている未知のゲーム開発者、パブリッシャー、そして私の謙虚な意見で役立つすべての人々に対する彼のアイデアの説明でスパムチャンネルを設定しました。 したがって、突然私のスパムが誰かに不便を感じた場合、私は心から謝罪します。 他の目標を追求しました。
