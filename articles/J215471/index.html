LinuxとWindows 8の出会いの暗い歴史

コミュニティへの挨拶。 私の悲惨な話を一つお話したいと思います。それを自分の中に保持するのは難しく、志を同じくする2人の友人のベストで泣くことはあまり助けにならなかったからです。



私の職業はITに最も直接関係しており（主な仕事はITエンジニアであり、Webプログラミングの「あちこち」）、ポータブルワークステーションを持っていると便利な場合があるため、固定コンピューターからラップトップに移行することにしました。 自由に使えるネットブックがありましたが、画面の対角線はサイトで作業している瞬間に私を悲しくさせました。また、その精神的特性とコンピューティング特性のために本当に輝いていませんでした。



最後の、おそらく、約7歳の、私が持っているメインで唯一のオペレーティングシステムはArch Linuxであり、それはどこにでもあります：稼働中のコンピューター、自宅、ネットブック、VPSです。 LinuxでのWeb開発のすべての利点を説明するつもりはありません。このシステムで作業できてうれしいとしか言​​えません。 そのため、ラップトップを購入するとき、配布キットを選択することに疑問の余地はありませんでした。



Windows 8はラップトップにプリインストールされていましたが、今回はディスク上のすべてを破棄するのではなく、両方のシステムの友達を作ることにしました。 それでも同じように、システムのデバイスの価格は引き上げられ、万が一のためにマイクロソフト製品を手元に置いておく必要はありません（ゲームをプレイしたり、レジャーで走ったりする場合、ラップトップはまだ弱くありません） ラップトップがGPTベースのディスクレイアウトを搭載したUEFIに搭載されていたという条件付きで、このタスクは原則として非常に簡単です。



私はこのトピックに関するマニュアルを読み、行動し始めました。 Windowsコンテンツを含むパーティションを100 GBに減らし、Linuxルートの下に別の100 GBパーティションを作成し、スワップの下のパーティションの後、残りのスペースを両方のシステムのデータのNTFSパーティションとして使用しました。 復旧セクションに手を加えなかった後。 Arch Linuxをインストールすると、すべてが問題なく開始され、両方のシステムがF12を押して起動しました。



Windows 8を見ることにしました。ダウンロードしました。 さて、Windows XPが一部のPCデバイスに最後にインストールされたとき、私はWindows VistaまたはWindows 7のいずれかで通常のユーザーとして働いていなかったことを予約します。 私はそれらの職務だけに精通しています。 私が見つけたラップトップにプリインストールされたソフトウェアのどの動物園を説明する価値もないと思います。 5分ごとに、プログラムの1つが何かを報告し、Aviraはセキュリティ上の欠陥があることを叫び、誰かが何らかの登録を激しく要求します。 など Aviraはメモリから適切にアンロードする方法を見つけられなかったので、サービスを終了するつもりでした。 一般に、私はまったく興味のないソフトウェア製品を削除し始めました。 この瞬間は、オペレーティングシステム自体を直接特徴付けるものではないことを理解しています。私は、すでに触れたり触れたりした雰囲気を大まかに説明しようとしています。



直接Windows 8については。もちろん、タイル。 ここでは、私はあまり理解していませんでしたが、ここでは誰も私のインターネットトラフィックを節約できません。ニュース、天気などが自動的にちらつきます。 まあ。 電源を入れると、「Admin」アカウントが開かれ、ラップトップへのログインに使用されていることに気付きました。 これがストアの従業員の仕事なのか、それともビルトイン管理者アカウントと呼ばれるのかはわかりません（そして常に管理者がいます）が、それはできません。 必要に応じて「admin」を使用して、より高度な権限を持たないユーザーの下で作業することをお勧めします。 新しいユーザーを作成するのに便利です。 そして、それは私にメールとパスワードを入力するように頼みます、彼らは言います、私はあなた、雲などとすべてを同期させます。 必要ですか？ システムは私よりもよく知っています。 新しいユーザーの作成で待機する必要がありました。 そして、私はまだラップトップの電源を切る方法を見つけていません。 テレパシーの力により、Win + Iホットキーを発見しました。これは、右側のパネルを呼び出します。このホットキーには、特にシャットダウンボタンがあります。 しかし、より直感的な方法があるかもしれません（ラップトップのハードウェアボタンを押す以外に）。



しかし、すべての塩は後で始まりました、私はちょうどそのようです、私がすぐに好きではなかったものをほんの少し説明しました。 もちろん、私が指摘した肯定的な点はありましたが、今日起こった事件の後、それらを完全に忘れました。 もう一度Windows 8で起動すると、幸運なことに私はWindows 8.1のアップデートを無料で入手できると報告してくれました。 うーん、どうして？ さらに、このバージョンでは、たとえば[スタート]ボタンがないなど、Windows 8の抜本的な革新が放棄されたと聞きました。 私は同意します。ギグを含む3のインストーラーがダウンロードを開始しました。 すでに深夜になっていたので、ダウンロードが完了するのを待たないことに決めました。私のインターネットはそれほど高速ではありません。 画面のバックライトを暗くして、寝ました。 朝、私はそこでLinuxが動いているように見えました。 ええ、明らかにダウンロードの終了後、自動インストールが開始され、その間に再起動が必要になりました。Linuxはデフォルトで起動するため、結果は次のとおりです。 ラップトップを再起動し、Windowsを選択します。 インストールプロセスが続行されます。 ちなみに、それはまったく直感的ではなく、関心は単に実行されており、「機器の準備中」などの現在のプロセスは一言で説明されています。 すべてのステップの後、Windows 8.1はようやく起動しましたが、初期セットアップウィザードのようなものが表面化しました。 そして、ここでまたおなじみの歌-私のメールアドレスとパスワードを入力するように頼まれました。 拒否することも、少なくともこの手順をスキップすることもできません。 どうして？ 私は自分のGMailアカウントを入力することを敢えてせず、このウィザードの標準ツールを使用してoutlook.comで新しいアカウントを開始し、チョークして背後にいるように頼みました。 ちなみに、まだSMSによる確認が必要であり、何らかの理由でアクティベーションコードが届きませんでした（最終的にはかなり長い遅延が発生しました）。



すべてのステップの後、再起動が再度必要でした。 その後、新しく開いたアカウント@ outlook.comのパスワードがラップトップアカウントのパスワードになりました。 いいね まあ、それだけです。 私はWindows 8.1の幸せな所有者です。 しかし、私は早く喜びました。 居心地の良いLinuxを再起動すると... Grubは未知のファイルシステムについて何か言います。 ありえない！ 最初は、このインストーラーがディスク全体を再パーティションすると思いました。 完全に落ち込んだ気分で、私はWindowsを再起動し、ディスク管理に入ります。 いいえ、セクションはあります、神に感謝します。 しかし！ Windowsパーティションの後に、何らかの理由でインストーラーが約300 MBのサイズの別のパーティションを作成しました。 彼がWindowsパーティションを押したのか、Linuxルートの最初から断片を「ビットオフ」したのかはわかりません（2番目の場合は悲しみですが、ちょっと調べてみるとまだ1番目のように見えます）ゲーム外。 なぜ彼は私にこれをしているのですか？



私の仮定が真実であり、ファイルシステムが/生きており、ブートローダーを再び動作するように設定していることが判明したとしても（これを行う方法はありません、起動するフラッシュドライブまたはディスクはありません）、私は完全にこのシステムに対処する必要はもうありません。 なぜ別のセクションが必要なのですか？ これがWindows回復環境の一部であることは明らかですが、認めなければならないのは、このように、警告なしにディスクを再パーティションすることです。 その結果、週末の仕事に関しては無能なままでした。 明日、ツールにアクセスしたら、必要なものをすべて外部メディアにコピーし、GPTマークアップを破棄し、UEFIをオフにし、古き良きMBRでディスクをパーティション分割し、お気に入りのArchをディスク全体に排他的にポンプアップします。 すべての知人以上。 しかし、私を説得してみてください。
