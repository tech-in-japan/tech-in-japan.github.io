GPSウィークカウンターのリセット





どうなるの？



2019年4月6日に、2番目のGPS週ロールオーバーカウンターがゼロにリセットされます。最初は1999年8月21日です。これは、GPS衛星によって送信されるナビゲーションデータフレームで、ナビゲーションメッセージには、GPSの週番号の下に10ビット値が割り当てられます（年間約52週間を考慮すると、1024週間は20年弱になります）。



これは消費者にどのように影響しますか？



このイベントが提供されるソフトウェアの受信者は、ナビゲーションフレームから現在のGPS週番号を正しくデコードし続け、消費者の場合、週番号は増加し続けます。つまり、ゼロ値は2048、1は2049などとして自動的に解釈されます。 しかし、受信者がこのために準備されていない場合、受信者は過去に戻ります（1999年または1980年）。



何ができる/すべきですか？



現在のファームウェアバージョンがナビゲーションレシーバーにインストールされていることを確認します。 ほとんどのメーカーはこの状況を認識しており、消費者が問題を起こさないようにソフトウェアを調整しています。



消費者に分類されるデータが依存する受信機またはソフトウェアの製造元に連絡してください。



消費者側に松葉杖を追加することも理にかなっています。たとえば、チェックを追加します。 



if ( week < 1023 ) week += 2048;
