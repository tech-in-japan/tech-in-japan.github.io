Visual Studio Code-OS Xのサンプルバージョンを使用したクロスプラットフォームコードエディターの進化





Visual Studio Codeは、30を超えるプログラミング言語とファイル形式をサポートするコードエディターであり、多くの追加の便利な機能を備えています。



ツールはこの春にリリースされましたが、以前はこのクロスプラットフォームエディターの基本機能について説明していました。 Windows、OS X、Linuxを実行しているコンピューターでVS Codeを使用できることを思い出させてください。



ただし、6か月間にわたってこのツールは定期的に更新され、ユーザーのフィードバックと要望に基づいて、機能、サポートされている言語のリストを拡張し、既存の機能を絶えず改善し、月1回更新され続けています。



本日は、OS Xを例として使用したVisual Studio Codeの次の機能の概要を紹介します。Unity3D、Visual Studio Online、GitHubとの統合、およびMacでのVS Codeのインストール、構成、快適な操作に関する詳細なビデオ資料です。



コンピューターにVisual Studio Codeをインストールするには、公式サイトcode.visualstudio.comから配布キットをダウンロードする必要があります。



現在サポートされている言語の完全なリスト：







Node.jsを使用してJavaScriptまたはTypeScriptアプリケーションを実行する場合、Visual Studio Codeはデバッガーをサポートします。

デバッガは、Monoで実行されるアプリケーションでもサポートされています。これらはASP.NET 5.0およびUnity3Dアプリケーションです。 したがって、VS Codeでは、IntelliSenseサポートからデバッガーまで、すべての機能にアクセスできる最新のWebアプリケーションを開発できます。



バージョン管理システム



Git統合


Visual Studio Codeは、ローカルおよびリモートのGitリポジトリをサポートしています。 Gitユーティリティをインストールしたら、Visual Studio Codeを再起動して、現在のワークスペースのGitを初期化する必要があります。







これからは、Gitをローカルで使用できます。 Gitウィンドウに切り替えると、変更されたすべてのファイルが表示され、その内容を以前のバージョンと比較できます。







VS Codeでは、さまざまなプロジェクトブランチで作業することもできます。







Visual Studio Online


Gitプラットフォームを完全に使用するには、サードパーティのプロバイダーが必要になる場合があります。 この記事では、GitHubとVisual Studio Onlineについて説明します。 オープンソースプロジェクトを公開する必要がある場合は、GitHubを使用しますが、コードを外部に公開する必要がある場合は、サブスクリプションを購入する必要があります。



Visual Studio Onlineはオープンソースプロジェクトをサポートしていませんが、クローズドプロジェクトを作成できます。チームが5人以下の開発者で構成されている場合、Visual Studio Onlineの使用は無料です。 同時に、バージョン管理システムはVisual Studio Onlineの機能の1つにすぎませんが、スケジューリングユーティリティ、かんばんホワイトボード、チームの仮想ルームなど、他の多くの機能を使用できます。したがって、VS CodeとVS Onlineは、ソフトウェア開発サイクルの完全なサポートを提供できます。



Visual Studio Onlineでの作業を開始するには、アカウントを登録する必要があります 。 その後、新しいプロジェクトの作成を開始できます。 プロジェクトとは、ドキュメント、ソリューションに含まれるすべてのプロジェクトのソースコードなどを保存できるVisual Studio Online内で作成されたワークスペース全体を指します。







すべての設定が完了したら、Gitリポジトリへのリンクを取得する必要があります。 これを行うには、作成したプロジェクトに移動し、[コード]タブを開きます。ここでリンクをコピーできます。



GitがXcodeコマンドラインユーティリティの一部としてインストールされた場合、必要なファイルはすべてMacにすでに存在しているので、コマンドgit config - global credentialを実行するだけです。 ヘルパー osxkeychain



Github


プロジェクトのメインページに移動し、コンテンツの複製に使用する必要があるリンクをコピーします。 ターミナルを使用して初期化プロセスを完了します。 これは、VS Codeからコンテキストメニューまたはコマンドパレットウィンドウを使用して実行できます。







Visual Studio Onlineと同様に、Gitは選択した名前に関連付けることができます： git config --global user.name xxxxx



そして、プロジェクトを選択したディレクトリにクローンできます： git clone https：// github.com/xxxxx/testVSCode.git



クローン作成チームは、ディスク上にすべてのソースファイルを含む新しいディレクトリを作成し、VS Codeを使用して、作成されたディレクトリを開きます。



Unity 3D



Unity3Dは人気のあるゲーム開発環境ですが、OS XではMonoDevelopはC＃コードとJavaScriptを編集する手段として使用されます。 C＃プロジェクトでプロセスを実演します。



Unity3Dを使用して、新しいプロジェクトを作成するか、C＃のファイルを含む既存のプロジェクトを開きます。 最初にすることは、デフォルトのエディターを変更することです。 これを行うには、 Unity-> PreferencesのExternal Toolsタブで、デフォルトでMonoDevelopを含むExternal Script Editorドロップダウンリストを見つけ、ApplicationsフォルダーでVisual Studio Codeを選択します。



その直後に、コードで任意のC＃ファイルを開くことができます。 ファイルのみが開いており、コードにはプロジェクトに関する情報が含まれていないため、多くの機能が無効になっていることに注意してください。



プロジェクトを開きましょう。 Unityのバージョンによっては、プロジェクトファイルがデフォルトで作成されない場合があるため、Open C＃Projectメニュー項目を使用できます。 このコマンドの結果、プロジェクトディレクトリでslnファイルとcsprojファイルを見つけることができます。







これらの形式はVisual Studio Codeでサポートされているため、ディレクトリを開くことができます。



Unityは、さまざまな段階でゲームをビルドするために使用されるいくつかのプロジェクトで動作しますが、そのうちの1つだけを開くには、ステータスバーを使用してプロジェクトの選択をクリックします。







Visual Studio Codeは、プロジェクトを選択できるポップアップウィンドウを表示します。 この場合、csharpサフィックスを持つプロジェクトが必要です。







Visual Studio Codeは、最新バージョンのmonoを必要とするOmniSharpコンパイラーを使用するため、エラーを回避するには、mono-project.comから最新バージョンのmonoをインストールしてください。すべてをインストールしたら、Visual Studio Codeを再起動し、C＃ファイルを再度開く必要があります。



これで、IntelliSenseシステムが正常に動作し、コードで引き続き作業できます。







デバッグ



Visual Studio Codeは、node.jsプロジェクトおよびモノラルのプロジェクトのデバッグをサポートしています。 UnityはMonoを使用するため、UnityとVisual Studio Codeを統合する例を使用して、デバッガーの機能をデモンストレーションしましょう。



コードでデバッグを有効にするには、launch.jsonファイルを作成し、そこでデバッガー設定を入力する必要があります。 [デバッグ]ウィンドウに移動し、[設定]ボタンをクリックしてlaunch.jsonの作成を有効にします。







作成したファイルからnode.jsを操作するためのすべてのコードを削除し、次を追加します。 



{ "version":"0.1.0", "configurations":[ { "name":"Unity", "type":"mono", "address":"localhost", "port":56060 } ] }




Unityプラグインを使用している場合、Write Launch Fileオプションがインストールされていることを確認する必要があります。







オプションが設定されている場合、Unityプレーヤーでアプリケーションを実行する必要があり、launch.jsonが自動的に作成されます。



デバッガーを開始するには、[デバッグ]ウィンドウに移動し、[開始]をクリックします。 その直後に、Unityに移動してプレーヤーでゲームを開始できます。 すべてが順調であれば、デバッガが機能していることがわかります。







デバッガーには、プロフェッショナルツールに固有の必要なすべての属性があります：ブレークポイント、コールスタック、ウォッチ、変数、デバッグアクションパネル、デバッグコンソール。



タスク



Visual Studio Codeのコンテキストでは、任意のコマンドラインコマンドを実行し、開発環境から直接結果を表示できます。 したがって、外部コンパイラ、デバッガ、テストツールなどを使用できます。



コマンドパレットを開いて[タスクの実行]を入力すると、コードがgruntの構成ファイル内のすべてのタスクを認識し、開始できることがわかります。 タスクの結果として、アルゴリズムに従って変換されたすべてのソースコードを含む新しいJavaScriptファイルを見ることができます。 同時に、結果が出力ウィンドウに表示され、問題を時間内に検出できます。



Visual Studio Codeは、出力ウィンドウからのデータを処理し、それをエラーリストに表示されるフォーマットされたメッセージに変換できる特別な拡張機能をサポートしています。



紺Azure



Visual Studio CodeとAzureの直接統合は提供されていません。 ただし、WebサイトをコードからAzureに簡単にデプロイできる方法があります。



これを行うには、Azureアカウントを作成し、コントロールパネルを開き、Webアプリテンプレートを使用して新しいWebアプリケーションを作成する必要があります。これにより、数秒以内に将来のサイトのホスティングスペースを作成してそこに配置できます。



インフラストラクチャが作成されたら、設定に移動できます。 このシナリオでは- 継続的な展開 ：







ここから、Azureがさまざまな種類のストレージからソリューションを展開する機能をサポートしていることは明らかです。 前述のGitHubおよびVisual Studio Onlineリポジトリとの統合により、コードとAzureをリンクできます。 したがって、Visual Studioのコードエディターインターフェイスから直接リモートコードの展開を実行し続けることができます。



おわりに



Visual Studio Codeを使用すると、さまざまなシナリオを実装でき、通常のコードエディターにとどまりません。 説明されているシナリオは、公式ドキュメントページで詳細に説明されている可能なシナリオの一部のみをカバーしています。



また、MacでVisual Studio Codeツールをインストール、構成、操作するための段階的な手順を含む、 ビデオ資料を共有できます。







JavaScript、TypeScript、ASP.NETの操作を含む一連のビデオの指示でこれらの各シナリオを詳細に説明するフルコースへのリンク 。
