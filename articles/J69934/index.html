おそらく、誰もがすでにインターネット検索の一般的な傾向に気付いているでしょう-ユーザーに検索結果ページですぐに多くの質問への答えを与えるために。 しかし、ここでは、十分な注意を払って、注意を払うことが重要です-既製の証明書を発行できる場所ではなく、必要なデータをページに散らかして、必要なものを決定すべきではない場所です。 このような区切りの方法と、プロンプトを出力に統合した経験を以下に説明します。





最近、短い伝記情報がPoisk@mail.ruのヒント集に追加されました 。 検索エンジンが特定の個人専用であると判断したクエリの場合、生成されたヘルプとポートレート（ある場合）が発行されます。 ところで、検索エンジンGoGo.Ruのロボットは、簡単な伝記やその他のヒントのためにデータの収集と整理に取り組んでいます。



そのようなヘルプを表示するのが適切なクエリを理解するために、また、その関連性がないためにイライラする場合、検索エンジンはユーザーの行動を分析します（それ自体で、ロボットは匿名の統計情報で動作します-プライバシーを犠牲にしません）。 分析は、次のスキームに従って実行されます。

  1. 「 Ivan Fedorovich Kruzenshtern 」というリクエストを行うとき、ユーザーは通常、人に関する情報を探すという仮説があるとします。
  2. それを確認するために、ユーザーがクリックして検索クエリを絞り込んだ検索結果のサイトが分析されます。
  3. 仮説が確認されると、より単純なクエリでのユーザーの行動が分析されます。 したがって、「Ivan」または「Ivan Fedorovich」を探している人は、おそらく他の人を意味していることがわかります。 クエリ「Fedorovich Kruzenshtern」の検索結果の方が優れていますが、統計によると、そのようなクエリを実行する人はいません。 しかし、「Kruzenshtern」または「Ivan Kruzenshtern」の要求での行動の分析から、ほとんどのユーザーが船員を探していることがわかります。
  4. その結果、証明書は「Ivan Fedorovich Kruzenshtern」と非常に適切な「Ivan Kruzenshtern」および「Kruzenshtern」の両方に表示されます。






伝記に加えて、人々が住所、機関の名前、およびさまざまな組織や企業を探している場合の参考文献を追加しました。



たとえば、論理的な決定は、住所を入力するときに、マップ上の対応するポイントにユーザーを送信することです。 このようなリンクを問題のすぐ上に配置しました。





レストラン、クラブ、映画館、および組織については、関連する検索結果の近くにある電話番号と住所などの参照情報を提供します。





ホテルの場合もほぼ同じことを行いますが、場所と連絡先の詳細に加えて、ロボットがサイトから抽出したホテルの「スターダム」に関する情報も提供します。





ご覧のとおり、住所（Map.Mail.Ruプロジェクトにある都市に関連する場合）もここのリンクで強調表示されます。



興味深いことに、これらのヒントの人気は次のように配布されました。 そもそも、予想通り、住所が付いたリクエストは一番上にあり、より目立ちます。ほとんどの場合、地図上の建物へのリンクは、「Leningradsky Prospekt 47」のようなユーザーのリクエストに対する最良の応答です。 さらに、人気はまず第一に、施設の需要によって決まります。これらはレストラン、映画館、クラブ、ホテルの順です。
