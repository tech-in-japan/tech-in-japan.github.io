手遅れになる前にインターネットがIPFSを必要とする理由







IPFSはまだよく知られた技術になっていない;シリコンバレーでさえ、多くはまだそれを認識していないが、それについてのニュースはオープンソースコミュニティの口コミからすぐに分かれます 。 また、ファイル転送を改善し、インターネット上でのストリーミングを高速化するIPFSの潜在的な機能に多くの人が喜んでいます。



しかし、私の個人的な観点から見ると、IPFSは実際にはこれらの機能よりもはるかに重要です。 IPFSは、サイトが中央のソースサーバーを使用する必要性を排除します。したがって、これはおそらく、インターネットのアーキテクチャを内部の矛盾から外れる前に完全に変更する最良のチャンスです。



どのように、なぜですか？ この質問に答えるには、詳細に進む必要があります。



World Wide Webが遅く、壊れやすく、忘れっぽいのはなぜですか



IPFSは新しいp2pハイパーメディアプロトコルです。その目的は、現在WWWを支配しているHypertext Transfer Protocol（HTTP）を補完（または場合によっては置き換え）することです。 これはHTTPの問題です。今日、サイトにアクセスするとき、ブラウザはこのサイトが動作するサーバーと直接通信する必要があります。これらのサーバーが遠く離れていても、転送プロセスにはかなりの帯域幅が必要です。



ネットワーク間でピアツーピア契約が締結されるため、プロバイダーは費用を負担します。そのため、ネットワークからネットワークへデータを送信すると、プロバイダーに費用がかかり、チャネル帯域幅を消費します。 さらに悪いことに、HTTPファイルは、異なるコンピューターからファイルの一部を同時に受信するのではなく、1台のコンピューターからのみダウンロードされます。



そのため、ここで立ち往生しています。「ラストマイル」プロバイダー（少なくとも米国では）の略奪によって価格が引き続き上昇し、モバイルデバイスからの接続要求の増加が加速している、低速で高価なインターネットです。 低速で高価なだけでなく、信頼性もありません。 何らかの理由でHTTP転送の少なくとも1つのリンクが切断されると、転送が切断されます。 （そして、 Webページまたはメディアファイルの読み込みに長い時間がかかる場合、コンピューター間の接続の1つが原因である可能性が高いです。）



IPFSを通じてインターネットを作り直す



IPFS（InterPlanetary File System） 2013年にYahoo!によって買収された会社を設立し、昨年Y Combinatorを通じてProtocol Labsを設立しました。現在、IPFSプロジェクトに従事しており、過去20年間にわたって日常的に発生しているプロトコルを少し変更する予定です。



IPFSは、すべてのコンピューティングデバイスを1つの共通ファイルシステムに接続しようとするP2P分散ファイルシステムです。 そのため、いくつかの方法でHTTP機能を上回ることができます。 これらのうち、2つ（フアンが最近の会話で私に言ったように）が重要です。



「コンテンツのアドレス指定を使用しているため、コンテンツはソースサーバーから独立し、個別に長期間保存できます。 これは、コンテンツを消費者の非常に近くに保存し、配信できることを意味します。おそらく同じ部屋のコンピューターであってもです。 コンテンツのアドレス指定により、元のソースではなく、他の（信頼できない）ホストから受信したデータを確認することもできます。 そして、コンテンツがユーザーのデバイスにヒットすると、好きなだけキャッシュすることができます。



IPFSは、HTTPベースのインターネットを悩ますセキュリティ問題のいくつかにも対処します。コンテンツベースのアドレス指定（およびその後の検証）はIPFSベースのサイトを保護し、IPFSの配布はDDoS攻撃を不可能にします。 IPFSは、所有者によるサイトの閉鎖によって引き起こされる損害を軽減するのに役立ち、社会にとって重要なコンテンツをキャッシュし、非選択的に保存することもできます。



IPFSの主な利点は、コンテンツの分散配信です。これにより、インターネットへの不規則なアクセスの状況でもインターネットからのコンテンツにアクセスでき、オフラインキャッシュから取得することさえできます。 「中央のソースサーバー を持たないサイトとWebアプリケーションを作成します」とJuan氏は説明しました。 「それらは、分散型ビットコインネットワークのように配布できます。」 HTTPは単にこれを達成できないため、IPFSは、一流の相互接続を持たないネットワークの利点を提供します（たとえば、発展途上国や農村部）。



昨年2月にIPFSのアルファ版が登場したことにより、すでにIPFSの早期導入者（早期導入者）の間で多くの実験が行われました。 たとえば、9月8日、 Neocitiesホスティングは、コンテンツを保存するためにIPFSを使用する最初の大規模サイトになりました 。つまり、分散WWWの必要性に関するインターネットアーカイブの呼び出しに続きます。 現在、私たち全員がますます多くの新しいサイトを絶えず失い、作成者によって年々放棄され閉鎖されています。私たちの集合的なインターネットメモリのこの増大する危機では、永続的なWebへの小さな一歩でさえ私たちにとって重要です。



そして、大企業が所有するサイトはNeocitiesの例に従いますが、まだ完全にテストされていないプロトコルの実装を開始します-特にP2Pについての簡単な言及でさえも恐れることができますか？ そして、ここで私は記事の最後の段落に来ます。



IPFSが将来のオンラインビジネスにとって重要な理由



私の新しい本 （まだ出版されていない）で説明しているように、コンテンツの配信コストが実用性と経済的利益を上回るポイントにすぐに近づいています。 最大のインターネット企業はすでにコンテンツに対する需要に追いつくのに苦労しており、エンジニアの全軍がアカマイ、グーグル、アマゾンで単独でこのタスクに従事しています。



すぐに彼らの状況は悪化します。 今後10年間の安価なスマートフォンの迅速な販売のおかげで、新しい消費者の大陸全体がオンラインになります。 モノのインターネットもこの問題を悪化させることを約束します。何十億ものデバイスが独自のリクエストを持っているため、通信チャネルの急速に減少している機能に関連するからです。



私がマイクロ特異性と呼ぶ効果を本当に制限する必要があります。同じイベントに対する数十億人のインターネットユーザーの急速な「ウイルス」注意は、システム全体を絞め殺す恐れがあります。 （ネットワークの中断は、同じ種類の大災害、事故、または他の現象が微小特異性になると、理論的には生命を脅かす。）



Netflixは最近、大規模なP2Pストリーミングテクノロジーの研究に着手しました。これは、この規模の企業がよりスマートなコンテンツ配信方法を探している最初で最も有望な兆候です。 Netflix、YouTube、および私たちにとって貴重なすべての貴重なトラフィックプロバイダーの両方は、IPFSの登場とコンテンツ配信のコストと時間の大幅な削減によって変更されたインターネットに基づいて作成できます。



IPFSは、接続性を改善することで、インターネットを理想化で常に求めていたシステムにするのに役立ちますが、現在のプロトコルでは実現できませんでした：世界中のすべての人（オフラインでも）を永続的かつ絶えず進化する表現で真に接続できます私たちは誰であり、私たちは何のために努力していますか。

Habrahabrはシリコンバレーでもないため、2015年10月の初めまでに、読者がIPFSについて十分な考えを持っていなかったと思われるかもしれません。 したがって、TechCrunchの記事の翻訳を終えたら、それに説明的な後書きを付け加えなければなりません。 そしてそれを質問と回答の形で包んで囲みます。



-IPFSとは何ですか？



-コンテンツアドレッシングを備えた分散ファイルシステム。



-IPFSシステムはどのような意味で配布されていますか？



-P2Pファイル共有に関して。 ファイルをダウンロードした後、ユーザーは他のダウンロードのソースになります。



-IPFSシステムはどのような意味でファイルシステムですか？



-簡単に言うと、オペレーティングシステムがUnixライクである場合、 FUSEを使用すると、ファイルシステムのアドレス「/ ipfs / file hash」にあるIPFSファイルを表示するようにIPFSに接続できます。



「IPFSはどのような意味でコンテンツに対応していますか？」



-ファイルのハッシュは、ファイルの内容のみに依存します。 ファイルの名前が異なるか、別のサブディレクトリにある場合、それは同じファイルです。 （これは、たとえば、ファイルの名前と相対位置によって異なるBTIH bittorrentハッシュと比較すると有利です。）同じファイルが異なる名前で、または異なるサブディレクトリの一部として配布される場合、配布は自動的にマージされ、労力、トラフィック、ディスク容量。



-IPFSファイルシステムにサブディレクトリはありますか？



-あります; これらは、ファイルとサブディレクトリの名前（およびハッシュ）のリストです。 たとえば、アドレス「/ ipfs / directory_hash / file_name」を開くには、最初にディレクトリが（既知のハッシュによって）ダウンロードされ、次にハッシュはIPFSがファイルをダウンロードするディレクトリ内のファイルの名前によって決定されます。 各サブディレクトリ名は、このプロセスに1つのステップを追加します。



-IPFSホストは、どのホストに目的のファイルがあるかをどのようにして知るのですか？



-DHTによって使用されます（ Kademliaと同様ですが、 シビル攻撃に対する耐性がより高い）。 名前を取得、無効化、減算できる中央サーバーはありません（たとえば、torrent trackers capture and disableなど）。



-現在のWorld Wide Webのサイトは、分散IPFSファイルシステムにあるファイルにどのようにリンクできますか？



-サイトの作成者はIPFSを設定し、ファイルをIPFSに配置します。 次に、 IPFSファイルパスの前に「 https://ipfs.io 」と書き込み、たとえば「 https://ipfs.io/ipfs/QmcXx5mKDQAc7tCWLq84Hn7XFxWfBdZpvogJk3tNXQRFiv 」を取得します 。 サイト " https://ipfs.io "はIPFSからWWWへのゲートウェイのように機能するため、IPFSをサポートしていない読者はこのサイトからファイルを受信できます。



-IPFSサポートがインストールされている読者は？



-また、IPFSからのパーソナルゲートウェイがあります。 そのようなURLに「ipfs.io」の代わりに「127.0.0.1:8080」が自動的に置換されるFirefoxのアドオンまたはChromeの拡張機能を配置するだけです。



-ネットワークが停止した場合、IPFSはどのようにあなたを救いますか？



-最初からイラスト（およびその他のファイル）がWebサイトのアドレスではなくIPFSで公開されている場合、サイトがクラッシュしても、リーダーはIPFS Webサイトのキャッシュではなくキャッシュから既知のアドレスでファイルを受信できます隣人。 （また、ファイルはハッシュに対してチェックされるため、ネイバーはファイルを置き換えることができません。）



-サイトがトラフィックを節約するのにIPFSはどのように役立ちますか？



-最初からイラスト（およびその他のファイル）がWebサイトのアドレスではなくIPFSで公開された場合、マイクロ特異点が突然発生し、10億人の視聴者が非常にすばやくサイトにアクセスしたとしても、とにかくIPFS経由でサイトのほとんどがノックオフされます、そして十億の隣人へのキャッシュで、彼らは少し前に時間がありました。 したがって、サイトは横になりません。 （もちろん、このためには、まずIPFS modを視聴者に配布する必要があります。そうしないと、サイトではなく、たとえば「 https://ipfs.io 」ゲートが横になります）。



-ディスク損失が発生した場合、IPFSはサイトをどのように支援しますか？



-最初からイラストやその他の静的ファイルがWebサイトのアドレスではなくIPFSで公開されていた場合、IPFSを使用してサイトのリーダーのキャッシュから不足している写真（およびその他のファイル）をダウンロードすることができます。



-州検閲の場合、IPFSは読者をどのように支援しますか？



-最初からイラストやその他の静的ファイルがWebサイトのアドレスではなくIPFSで公開されていた場合、禁止されたイラスト（たとえば、 精霊の使用いの剣舞の暴走のイラスト）を使用して、ブロックをバイパスして他の禁止されたファイルを受け取ることができます（検閲が有効になる前にファイルを取得できたサイトリーダーのキャッシュから）。
