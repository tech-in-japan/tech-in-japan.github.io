Mac OS Xの未来に関する考察





今日、Mac OS Xはしばしば当たり前のことと考えられています。 もちろん、このオペレーティングシステムと使用するすべてのプログラムが大好きです。 しかし、振り返って、このOSの最近の新機能を思い出してみましょう。 1年半前、Mac OS X 10.6 Snow Leopardが導入されましたが、根本的なアップデートは提供されませんでした。 最後の重要なユーザーインターフェイスの改善は、Mac OS X 10.5 Leopardで注目されました。MacOS X 10.5 Leopardは、3年以上前に発売されました。



Appleから最近Mac OS Xに十分な注意が払われていない場合、その欠陥はおそらくSteve Jobsの新しいお気に入りであるiOSにあります。 覚えておいてください。Leopard市場の発売以来、モバイルオペレーティングシステムはいくつかの重要な更新を経て、それぞれがiPhone、iPad、iPod touchの機能を大幅に強化しました。



「Back to the Mac」と呼ばれる10月のカンファレンスで、Appleはついに2011年夏に出荷を開始する新しいMac OS X 10.7 Lionを発表しました。 システムのいくつかの機能を流に実演しましたが、この「レオ」がどうなるかはまだわかりません。SnowLeopardのような29ドルのランスルーバージョンや、Leopardのような129ドルの「大詰め」です。 しかし、1つの明らかな傾向について詳細を確認することができました。







Appleは、Mac OS Xの新世代を開発し、iOSシステムの重要な機能であるシンプルさに触発されました。 Mac OS Xがコマンドラインシステムよりも使いやすくなったように、iOSはMac OS Xや他のより強力で複雑なデスクトップOSとのシンプルさの対比でもありました。 Appleは、iOSの作成中に得た知識を活用して、Mac OS Xをより理解しやすく使いやすくすることを計画していると発表しました。



アプリケーションの問題



たとえば、オペレーティングシステムの最も基本的なタスクである、アプリケーションのインストールと実行を考えてみましょう。 経験豊富なMacユーザーは、目隠しをしていてもこれらのタスクを実行できますが、初心者にこのプロセスを説明しようとします。 圧縮ファイル、ディスクイメージ、インストーラアプリケーションの複雑なシステムでは、経験のある「マクロドライブ」でさえ混乱する可能性があります。



まず、ディストリビューションをダウンロードして、どこに保存されているかを確認する必要があります。 あなたがそれを見つけると、質問が起こります：それはインストールファイルですか、それともプログラムそのものですか？ アプリケーションをインストールした後、ドックにドラッグするか、ここから実行する必要がありますか？ そして、ディスクイメージを使用して何をする必要がありますか？ アンインストールはさらに悪くなります。 Mac OS Xは、このタスクのためのワンストップソリューションを提供していません。 プログラムアイコンをゴミ箱にドラッグするだけで十分な場合があり、削除される場合があります。 ただし、インストールにマルチステップインストーラーが必要なアプリケーションは、アンインストールするために特別なアプリケーションが必要になります。



これをすべてiOSと比較します。アプリケーションをインストールするのは1回の「タップ」だけです。 プログラムの削除も同様に簡単であり、この方法はすべての種類のアプリケーションに共通です。 インストールの容易さ（価格の手頃な価格を考慮しない場合）は、人々がiOSデバイス用のアプリケーションを積極的かつ積極的に購入する主な理由の1つです。 Mac OS Xにプログラムをインストールするプロセスの複雑さを恐れる人は、次のiPhoneまたはiPadの画面がアプリケーションアイコンでいっぱいになるまで「タップ」を続けることができます。



Appleは、消費者の要望に注意深く耳を傾けました。 そして、開く準備をしているMac App Storeは、コンピューターユーザーがiOSプログラムを購入する際のシンプルさを提供する必要があります：ワンクリックでの購入とインストール、ダウンロードとインストールプロセスに関する詳細情報、およびアプリケーションのインストールパスの明確な表示。 残念ながら、Appleは10月のイベントでプログラムをアンインストールするプロセスを示しませんでした。 ただし、確認してください。iOSと同じくらいシンプルで便利です。



ファインダーなしで検索



Mac OS Xドックは、Macでアプリケーションを起動するプロセスを改善する上ですでに大きな進歩を遂げています。 ただし、ユーザーがドックを越えなければならない場合、すべてが著しく悪化します。 Finderの検索ツールは、ドック上のシンプルな一連のアイコンから複雑化の方向への大きな一歩です。 Spotlightが提供する別のポイントアプローチがあります。 ただし、検索クエリに入力する必要があるという事実は、単純化のための戦いがすでに失われていることを示しています。







iOSは、ドックの概念を新しいレベルに引き上げました。 最も頻繁に使用されるアプリケーションを単にランク付けするのではなく、すべてのアプリケーションを一連のアイコンブロックに整理します。 もちろん、最悪の場合でも検索を使用できますが、iOSのFinderでは何も比較できません。



どうやら、現時点では、Appleは質問について考えています。MacOS XにはFinderが必要なのでしょうか？ Lionに登場したLaunchpad Launchpadは、iOSで使用されるものと同様のプログラムアイコンを作成する原理をMacに導入しました。 したがって、彼女はFinderの機能を、ドック上にない起動アプリケーションを見つけるためのツールとして引き受けました。 Macアプリケーションは、iTunesおよびiPhotoから継承したライブラリメタファーをますます使用するようになりましたが、ファイルシステムを介したファイルとの直接的なやり取りの必要性は徐々になくなりつつあります。



iOSの理想への道



また、オペレーティングシステムの開発ツールはOSに統合されたアプリケーションによってインストールされるため、オペレーティングシステムがアプリケーションの設計に影響を与えることが多いことにも注意してください。 Appleの新しいコースでは、Mac OS XアプリがiOSソフトウェア製品のようになります。



例：iOSアプリケーションは、デバイスの画面全体を占有します。 モバイルデバイスのコンパクトな画面サイズを考慮する場合、これは合理的な決定です。 これからは、Macの開発者も製品でフルスクリーンモードを使用することをお勧めします。 Apple自体は、グラフィックエディタiPhotoの新しいバージョンでこれがどのように機能するかをすでに実証しています。 Mac OS Xの将来のバージョンでは、フルスクリーンモードを終了することなく、実行中のアプリケーションを切り替える便利な方法が提供される予定です。 これは何に似ていますか？ そう、iOS 4のマルチタスク。







モバイルデバイスのメモリリソースが限られているため、iOSは複数のアプリケーションを同時に起動する機能をサポートするようになりました。 同時に、メモリからアプリケーションをアンロードし、バックグラウンドから終了して、アプリケーションを残した状態にするための一定の機能が必要です。 つまり、iOSアプリケーションには本格的な保存機能はありません。すべてのデータが自動的に保存されます。



デスクトップコンピューターはこのようなハードウェアの制限を受けませんが、AppleはMac OS Xアプリケーションも同じように動作することを決定しました。 システムの将来のバージョンには、データを自動的に保存し、アプリケーションの状態を復元するための組み込みサポートが含まれる予定です。 ドックがアプリケーションが動作していることを視覚的に表示しなくなる可能性さえあります。アプリケーションの状態が失われない場合、動作中のプログラムと動作していないプログラムの違いは問題になりません。



おわりに



Mac OS Xが進化し続ける多くの伝統的な領域があります：64ビットへの移行が完了し、フラッシュメモリのサポートが改善されます（新しい、より新しいファイルシステムの出現により）、3Dグラフィックスのパフォーマンスが十分に注目されます。



しかし、これらのすべての努力は、AppleがMac OS Xに関して取った新しいコースに従ってバックグラウンドに委ねられます。iOSでの経験から、同社は技術製品の販売の成功の秘theであるシンプルさを発見できたようです。 これは、私たちが知っていて愛しているMac OS Xが地球の表面から一掃されることを意味するものではありません。 それどころか、iOSで最も成功したアイデアを吸収したMac OS Xは、既知の欠点を取り除き、利便性、シンプルさ、効率性を優先して選択します。 そして、これらの品質はMacで常に重視されていませんか？
