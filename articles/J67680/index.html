10便利なWordpressフック



Wordpressのフックとその使用方法について。 www.wprecipes.comの所有者であるベルギーの若いWordpress専門家による記事。



はじめに



フックは、Wordpressの非常に便利な機能です。 ユーザー定義関数を既存の関数の「フック」に「フック」できるため、エンジンコアファイルを変更せずにWordpressの機能を変更できます。 この記事では、Wordpressの10の特に便利な、すぐに使えるフックと、そのソースコードの例と説明を紹介します。



フックとは？



特定の効果を得るには、Wordpressの動作方法をわずかに変更する必要があります。 開発者がカーネルファイルと呼んだファイルにいくつかの変更を加える必要があります— Wordpressが正しく機能するために必要です。

ただし、カーネルファイルを変更することは悪い考えです。これにより、ブログにセキュリティホールが生じる可能性があります。 また、エンジンを新しいバージョンにアップグレードするとすぐに、すべての変更が消えます。

ただし、拡張機能は依然として必要です。 これを行うために、開発者はプラグインAPIを発明しました。

フックは、プラグインを構築するための主要なブロックの1つです。 ほとんどすべてのプラグインはフックを使用してWordpressの機能を拡張します。



ブログでフックを使用する方法



プラグインを作成するまで、 functions.phpファイルにフックを作成する必要があります。 このファイルはwp-content / themes / yourthemeディレクトリーにあります （ / yourthemeは現在のテーマが置かれているディレクトリーです）。

カスタム関数をエンジンコア関数に接続する方法を示す例を次に示します。



  1. add_action （ 'publish_post' 、 'myCustomFunction' ） ;




この例では、ユーザー定義関数myCustomFunction（）をpublish_post（）カーネル関数に接続しました。 myCustomFunction（）関数は、 publish_post（）関数が実行されるたびに実行されます。

もちろん、 remove_action（）関数を使用してフックを削除することもできます。



  1. remove_action （ 'publish_post' 、 'myCustomFunction' ） ;




1. WordPressでの自動フォーマットの無効化



問題。


Wordpressのタイポグラファーがデフォルトで「直接」引用符を「曲線」に変え、投稿の書式設定にその他の小さな変更を加えることにお気づきでしょう。

これは、ブロガーが通常のコンテンツを投稿する場合に適しています（プレーンテキストと画像が暗示されています）。 しかし、後で議論するためにソースコードを投稿する人もおり、これらの「曲がった」引用のために、インタプリタまたはコンパイラが構文エラーに関するメッセージを与えると、彼らは非常に不幸になります。



解決策。

  5. $ピース = preg_split （ $ pattern_full 、 $ content 、 -1 、 PREG_SPLIT_DELIM_CAPTURE ） ;
  7. foreach （ $ piece as $ piece ） {
  8. if （ preg_match （ $ pattern_contents 、 $ piece 、 $ matches ） ）） {
  9. $ new_content 。= $は [ 1 ]と 一致し ます。
  10. } else {
  11. $ new_content 。= wptexturize （ wpautop （ $ piece ） ） ;
  12. }
  13. }
  15. $ new_contentを 返し ます 。
  16. }
  18. remove_filter （ 'the_content' 、 'wpautop' ） ;
  19. remove_filter （ 'the_content' 、 'wptexturize' ） ;
  21. add_filter （ 'the_content' 、 'my_formatter' 、 99 ） ;




これを行った後、投稿テキストが自動的にフォーマットされないように、 [raw]タグを使用できます。



[raw]This text will not be automatically formatted.[/raw]











コードの説明。


まず、正規表現に基づいて、投稿のコンテンツで[raw]タグを見つける関数を作成します。

次に、 my_formatter（）関数をthecontent（）関数にバインドします。 つまり 、関数はthecontent （）関数が呼び出されるたびに実行されます。

自動フォーマットを無効にするために、 remove_filter（）関数を使用しました 。

ファイルを保存すると、関数はbodyタグのユーザーのブラウザーに対応するCSSクラスを自動的に追加します。

Wordpressには、ユーザーが適切なブラウザーを使用する場合にtrueを返すグローバル変数があります。 ユーザーがGoogle Chromeブラウザーを使用している場合、 $ is_chrome変数はtrueになります 。 そのため、 browser_body_class（）関数を作成します。 その後、Wordpress関数body_class（）にアタッチします。

9.スケジュールイベントにフックとクロンを使用する



問題。


おそらく、Wordpressがスケジュールされたイベントを使用できることを既にご存知でしょう。 たとえば、事前に設定した特定の時間に投稿を公開できます。 フックとwp-cronを使用して、独自のイベントのスケジュールを簡単に設定できます。 次の例では、ブログに1時間に1回電子メールメッセージを送信するように強制します。



解決策。


次のコードをfunctions.phpファイルに貼り付けます。



  1. if （ ！ wp_next_scheduled （ 'my_task_hook' ） ）） {
  2. wp_schedule_event （ time （ ） 、 'hourly' 、 'my_task_hook' ） ;
  3. }
  5. add_action （ 'my_task_hook' 、 'my_task_function' ） ;
  7. function my_task_function （ ） {
  8. wp_mail （ 'you@yoursite.com' 、 '自動メール' 、 'こんにちは、これはWordPressから自動的にスケジュールされたメールです。' ） ;
  9. }




コードの説明。


もちろん、最初に行うことは、必要なアクションを実行する関数を作成することです。 この例では、この関数はmy_task_function（）と呼ばれ、指定された電子メールアドレスに電子メールを送信するだけです。

この関数の実行をスケジュールするには、 wp_schedule_event（）関数を使用します。 渡される最後の引数はフックなので、関数my_task_function（）をmy_task_hookに 「フック」します。
