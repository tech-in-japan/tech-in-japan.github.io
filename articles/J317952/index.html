Veeam Agent for Linux 1.0（無料、ワークステーション、サーバー版）リリース

長年にわたり、Veeamは仮想環境専用の製品をリリースしてきました。 しかし最近、「物理マシンに何かが表示されるのはいつか」という質問が頻繁に寄せられました。この方向でユーザーのリクエストと開発機会を分析した後、 Veeam Agent for Microsoft Windows （以前のVeeam Endpoint Backup FREE ）をリリースしました。 ところで、近い将来、バージョン2.0がリリースされ、多くの興味深い機能が提供されます。







本日、 Veeam Agent for Linuxの最初の公式バージョンをご紹介します！ Veeam Agent for Linuxは、ローカルまたはパブリッククラウドに展開されたLinuxベースの物理マシンを保護します。 このツールはスタンドアロンであり、Veeam Backup＆Replicationのインストールは必要ありません （ただし、Veeam Backup＆Replicationリポジトリをターゲットストレージとして使用できます）。













Veeam Agent for Linuxとは何ですか？



Veeam Agent for Linuxは、物理Linuxシステム（ワークステーション、サーバー、パブリッククラウド内のマシン）をバックアップするためのツールです。 一般的な説明は、Habréの投稿「 Veeam Agent for Linuxの知り合い」で見ることができます。 製品はイメージレベルでバックアップを作成し、データをVBKファイルに保存します（Veeam Backup＆ReplicationおよびVeeam Agent for Microsoft Windowsと同じ）。 Veeam Agent for Linuxは、独自のドライバーを使用してボリュームスナップショットを作成し、変更されたデータブロック（CBT）を追跡します。 このドライバーを使用すると、マシン全体を再スキャンすることなく、変更されたデータブロックに関する情報を取得できます。これは、増分バックアップに不可欠です。







Veeam Agent for Linuxのシステム要件：







  • CPU：x86 / x64
  • カーネル：2.6.32以降
  • OS：32および64ビットバージョン：

    • Debian 6-8
    • Ubuntu 10.04-16.10
    • CentOS / Oracle Linux / RHEL 6-7
    • Fedora 23-25
    • openSUSE 11.3-13.2
    • openSUSE Leap 42
    • SLES 11 SP1-12 SP2


ベータ版と比較して、インストールプロセスははるかに簡単になりました。パッケージを手動でインストールする必要がなくなりました。 インストールとアップグレードは、Veeam Repositoryから実行できます。 インストール手順の詳細については、リリースノートのドキュメントを参照してください。







これで、製品がインストールされました。 次は？ Veeam Agent for Linuxには、ステップバイステップのUIウィザードを使用する方法とコマンドラインを使用する方法の2つの操作モードがあります。







UI経由のバックアップ



ウィザードを開始するには、コマンドシェルを開き、次のコマンドを使用します。 







$ veeam


Veeam Agent for Microsoft Windowsをすでに使用している場合、バックアッププロセスはおなじみのものです。







  • バックアップモード：ボリュームレベルまたはファイルレベルのベアメタル。
  • バックアップリポジトリ：ローカルドライブ、ネットワークフォルダー、またはVeeam Backup＆Replicationリポジトリ（9.5 Update 1が必要）。
  • スケジュールおよび保持ポリシーの設定 。








コマンドライン経由のバックアップ



微調整が必​​要な場合、または独自のスクリプトを作成してソリューションのインストールを自動化し、バックアップタスクを実行する場合は、コマンドラインがあります。







次のコマンドで開始します。 







 # veeamconfig








バックアップタスクを作成するには、次のコマンドを使用します。 







 # veeamconfig job create


次に、目的のジョブパラメータを指定します。













スクリーンショットでわかるように、コマンドラインを使用して、UIウィザードで使用できないパラメーター（例えばcompressionLevelやblockSize）を構成できます。







Veeam Backup＆Replicationリポジトリにバックアップを保存することに決めましたか？ すべてのコピーは、 エージェントノードの下のVeeam Backup＆Replicationに表示されます。













回復



ファイルを復元するには、Veeam Agent for Linux UIを起動して、 Recover Filesコマンドを選択します。 バックアップの内容はシステム上にフォルダーとしてマウントされ、ファイルマネージャーを使用して必要なファイルを取得できます。







Veeam Agent for Linuxには、ベアメタル、ボリュームまたはファイルレベルのあらゆるレベルでデータを復元するのに役立つリカバリメディアも付属しています。













Veeam Agent for Linux Editions



Veeam Agent for Linuxには、 Free 、 Workstation 、 Serverの 3つのエディションがあります 。







無料



Free Editionは、家庭での使用および重要でない本番システムの保護に最適なソリューションです。 基本的な機能：ボリュームスナップショットの取得、CBTの使用、すべてのバックアップおよび復元モード（UIおよびコマンドライン経由）、Veeam Backup＆Replicationとの統合。







ワークステーション



Workstationエディションの主要な機能は、エンタープライズレベルのサポート24.7.365です（無料版の機能に加えて）。







サーバー



Server Editionは最も高度なバージョンです。 凍結前および解凍後のスクリプトを実行し、複数のバックアップジョブを構成する機能が含まれています。
