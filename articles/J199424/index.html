emsで定義された隣接メディアクエリは重複しています



一方の最大幅が他方の最小幅と一致する場合、一対のメディアクエリを「隣接」と呼びます。 天井から取った例を挙げましょう。 



 .container { color: black; background-color: white; }} @media (max-width: 40em) { .container { color: red; }} @media (min-width: 40em) { .container { background-color: red; }}


正確に40em幅のブラウザでこのサイトを開くと、「ラッキー」な人が訪問者にどのように表示されると思いますか？



オーバーラップは1ピクセルのみですが、そうです。 サイトの訪問者が突然このような画面の幅を見つけた場合、ページとデザインのレイアウトは、完全に読めないほどに歪む可能性があります。

ブレークポイントスライサーのアイデアは非常に単純です。 たとえば、400px、600px、800px、1050pxなどの短い間隔でブレークポイントを設定します。 モジュールは、これらのスペースに順番に番号を付けます。



  1. 0〜400
  2. 401-600
  3. 601-800
  4. 801-1050
  5. 1051 —∞


次に、Sassコードで、メディアクエリを挿入するのに必要な間隔を指定します。



  • at(2)



    -同等(min-width: 401px) and (max-width: 600px)



  • from(2)



    -同等(min-width: 401px)



  • to(2)



    -同等(max-width: 600px)



  • between(2, 4)



    - (min-width: 401px) and (max-width: 1050px)





これにより、範囲ごとにモジュールのスタイルを迅速かつ効率的に配布できます。 多数のギャップを使用することで、過剰な特異性に陥ることなく、すべてのクラスのデバイスをカバーできます。
