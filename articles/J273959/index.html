統合システムの世界





2010年代の初めから、企業のIT顧客の心に大きな変化があり、個々の情報システムごとにハードウェアとソフトウェアを購入する代わりに、多くの人々が戦略的に考え始めました：必要に応じてリソースをさまざまなITシステムに分散できるようにユニバーサルインフラストラクチャを構築します。





仮想化技術の成長とクラウドイデオロギーの普及が役割を果たしました。 そして、「クラウド」のat明期にはパブリックのみを意味していましたが、やがてプライベートデプロイメントモデルが策定および実装されました。 「 ボックス内のクラウド 」も、まもなく登場します 。 統合システム 、事前構成されたサーバーとネットワーク機器のコングロマリット、そのリリースはいくつかの大手ITメーカーによってマスターされました。



統合システムは、プライベートクラウドの作成コストを大幅に削減し、最も重要なこととして、このイベントの成功の予測可能性を劇的に向上させると同時に、以前の決定からの展開と移行時間を削減しました。 IBSは、もちろん、大規模で高度なシステムインテグレーターとして、そのようなシステムの実装を開始した最初の企業の1つです。 数年間、私たちは重要な経験を蓄積してきました。これにより、これらのソリューションを非常に深く広くマスターすることができました。 経験を積む過程で、彼らは気付かれずに熊手を踏んでいましたが、すべての問題に答え、時にはエンジニアリングとソフトウェアソリューションの恵み、そして多くの場合、西洋の開発者の体系的で粘り強いことに驚かなければなりませんでした。



そうでない場合は、誰ですか？





VMware、Cisco、EMC、NetApp、Oracleなどの巨人のハードウェアおよびソフトウェアソリューションは、もちろん素晴らしいものであり、世界中の何万人もの顧客によって撮影されていますが、国内のソリューションは、ほぼすべてのクラウド領域で成長および成熟しています：仮想化、ネットワークストレージ、オーケストレーション、管理、監視、および機器の組み立てが可能です。 「 ロシアの意思決定だけで同じ複合体を作ることは本当に不可能ですか？ 「- 制裁リスクが空のフレーズではないお客様に尋ねました。 「 コンテナ仮想化と洗練されたソフトウェア定義のストレージネットワークを使用して、これらの複合体をより生産的かつ安価にすることは本当に不可能ですか？ 「-IT市場の発展を追いかけ、キャビネットの有名なロゴだけに余分なお金を払いたくない人に尋ねました。



したがって、これらの質問を聞いて、「巨人の肩に立って」-グローバル市場に存在するほとんどすべての統合システムで多くのことを真剣に取り組んで-ロシアの「ビルディングブロック」から現代の統合システムを作成するというアイデアを得ました。 なぜ、ハードウェアとソフトウェアをこのような方法で組み合わせ、すべてがスムーズに機能し、かなり幅広いタスクを持つ幅広い顧客に適し、異なるサプライヤーの友人を作るために最適であるように最適な構成を選択するのは、 インテグレータークラスのタスクです「山で」迅速かつ最小限のコストで、お客様に迅速に納品できる完成品を生産するために。





基本的なSCALA-Rスキーム（およびその他の収束PAK）



分類：プラットフォーム、コンバージド、ハイパーコンバージド





当然の質問は、統合システムの世界で、そのようなソリューションはどのような場所を取ることができるのでしょうか？ ここで、おそらく、まさにまさに統合システムの世界の分類法を整理する価値があります。 クラウドサービスモデルのロジックに従って、IDCは統合システムの市場をプラットフォームシステムとインフラストラクチャに分割します。



プラットフォームシステム -DBMS、アプリケーションサーバー、開発環境などのプラットフォームソフトウェアリソースを提供します。このクラスには、1980年代以降に発行されたTeradataデータ処理および分析システム、Oracle Exa-complexes（Exadata、Exalogic、Exalytics ）、それほど前ではないが、Hadoopエコシステムからの既製のソリューションセットを備えた複合体が登場しました。 これらの製品はすべて非常に興味深いものであり、それらの多くはエンジニアリングの傑作として語ることができますが、これらは通常非常に専門的なソリューションであり、いずれの場合もかなり限られたクラスのタスクに対応し、ご想像のとおり、今日ではそうではありません（しかし、間違いなくこのトピックに戻ります！）



2番目のセグメント- インフラストラクチャコンプレックス -「すぐに使えるインフラストラクチャクラウド」により、メールサーバー、リモートワークステーションのインフラストラクチャ、アプリケーション情報システム、ファイルなど、最新の組織のすべてのITニーズを迅速かつ十分にシンプルかつ柔軟に展開および柔軟に管理できるインフラストラクチャを構築できますリポジトリ。 インフラストラクチャの複合体の中で、さらに2つのサブセグメントが区別されます： 収束とハイパー 収束 。 最初のビルディングブロックは、目的に応じて分割されます。それらの一部はコンピューティングノードであり、その他はストレージノードです（FlexPodやVBlockなど）。 ハイパーコンバージド-各ノードは、このサブセグメントNutanixとSimpliVityの最も顕著な代表であるストレージノードとコンピューティングノードの役割を兼ね備えています。



ハイパーコンバージドシステムは少し遅れて登場し、その市場シェアはまだ比較的小さいです。たとえば、2014年のIDCの推定によると、インフラストラクチャコンプレックス全体の世界的な配信量は60億ドルに達しましたが、同じ機関によると、ハイパーコンバージドシステムは373ドルで販売しましたしかし、このシェアは拡大しています：ハイパーコンバージェンスは、市場の他の市場よりも急速に成長しており、昨年の2倍以上の成長です！ おそらくこれにより、ガートナーのアナリストは、ハイパーコンバージドシステムセグメントを統合システム市場の完全に独立した部分と見なし、IDCのようにコンバージドとハイパーコンバージドを1つのセグメントに結合することはしません。



また、ハイパーコンバージェンスのサプライヤの多くは、事実上無制限のスケーラビリティについてかなり合理的に語っていますが、その主な消費者は中小企業に分類される組織であることがたまたまありました。 これには多くの説明があります。おそらく、専用の高性能データストレージネットワーク「大規模な顧客」の存在は、ロードされたトランザクションアプリケーションにとって重要だと考えられます。ハイパーコヒーレントシステムの主要メーカーは、おそらく比較的小規模で新興企業です（一方、大規模な組織は、 ITの大御所）、おそらく大規模な顧客の場合、ハイパーコンバージドのコンピューティングとストレージの容量の比率において操作が重要です 難しい。



単一ボックスでのハイパーコンバージェンスとコンバージェンス





したがって、ポジションの調整を理解し、ロシアの顧客にどのような複雑さを提供すべきかを理解しようとすると、1つのアプローチのみをカテゴリ別に選択することは意味がないという結論に達しました：ハイパーコンバージドソリューションは従来のSANを欠くことが多く、従来の収束ソリューションはソフトウェアの柔軟性を欠くためです定義されたストレージネットワークとスケーラビリティの容易さ、すべての機能を接続してみませんか？ それは言われました-いくつかのロシアのメーカーとのコンソーシアムに参加することで、短期間でそのようなソリューションを設計し、控えめな名前「SKALA-R」を受け取りました。



複合体のサーバー、スイッチ、SANノードは、ロシアの会社DEPO Computersによって組み立てられ、すべてのハードウェアコンポーネントを設計およびデバッグしました。



仮想化および管理プラットフォーム、および複合体のソフトウェア定義のストレージシステムは、Parallelsによって開発されました。 このソリューションの独自性は、従来のハイパーバイザー仮想マシンとコンテナーの両方を同時に実行し、このすべての荷物を単一のコントロールパネルから管理できることです。 ソフトウェア定義のストレージVirtuozzo Storage（旧名-Parallels PStorageとしても知られています）を使用すると、各サーバーを単一のハイパーコンバージドストレージネットワークのノードに変えることができます。 従来のストレージネットワークの「棚」には、サンクトペテルブルクの会社Raidixのソフトウェアが請求されています。 複合体の監視は、ロシアの開発者NaumenとIHD Monitoringのソフトウェアソリューションによって提供されます。



若い方から古い方へ：3分の1、5分の1、7分の1





当社の最年少の複合施設であるSKALA-R 300シリーズは 、高性能ストレージネットワークなしで完全にハイパーコンバージェンスとして配信できます。 このオプションは、いわゆる「 中小企業 」に属する顧客を対象としています（これは、グローバルな規模で、国内の現実から見ると、中規模の組織と呼ばれています）。 複合体のコストは700万ルーブルから始まります。2つのプロセッサソケットを備えたサーバーを搭載し、それぞれに2つのマウントユニットの高さがあるクラシックな水平設計です（NutanixとSimpliVityのノードも同じ設計です）。 最小構成は4サーバーノードであり、ノード数に上限はありません。したがって、このシリーズの「最年少」は条件付きと見なすことができ、小規模に開始する能力に関連付けられます。







500シリーズのSCALA-Rには、 1台のマウントユニットの高さのコンピューティングユニットが装備されており、300番目のシリーズのサーバーにディスクを取り付ける場所が24ある場合、「5分の1」のデザインにはそのうち8つしかありません。および4つのSAN連隊-700万ルーブルから始まります。 「ジュニアシリーズ」の一般的なタスクは、何らかの意味で普遍的な「広範なITサービス」です。VDI 、企業メールサービス、グループ作業プラットフォーム、ITサービスシステム、SCALA-R 500シリーズなどでは可能です。トランザクションおよび分析の負荷が大きいシステムをアップロードする必要があります。







SKALA-R 700シリーズ -ビル内の強力な4プロセッサユニットの複合体で、高さは2マウントユニットです。 追加のハードウェアおよびソフトウェアモジュールであるAcronisバックアップシステムも、通常このシリーズに付属しています。 このようなマシンは、たとえば、データベースサーバーが1つのノード上に「モノリス」で数十個のプロセッサコアを必要とする状況で、ビジネスに不可欠な負荷を委ねることができます。







したがって、顧客は、多くの特定のケースで需要があるクラシックSANを使用する機会がありますが、当然ながら、ハイパーコンバージドコアとこの若いクラスのソリューションのすべての有益な特性を維持しながら 、構造全体に統合管理ツールを使用します。



合計





それでも、ロシア人にとってこの複合施設の主なものは、その国内起源です。 批評家たちは、「鉄」の成分が東南アジアで生産されているという事実に注意を向けることを遅らせることはありませんが、現在、他の選択肢はありません。 ハードウェア全体がロシアで設計、注文、および組み立てられることがはるかに重要であり、最も重要なもの-複合体のソフトウェアの心臓部-は完全にロシアで作成、開発、保守されます。 そして、この非常に基本的な問題では、妥協はありませんし、信じるべきではありません：非ロシア起源の仮想化および管理プラットフォームでは、フリーソフトウェアに基づいて組み立てられたとしても、 ロシアの複合体、および遭遇した組織を構築することはできません直接の制裁により、それを暗示する必要すらありません。 今年の終わりの時点で、そのような完全にロシアの統合された複合体は他にありません、そして、この意味で、SCALA-Rはユニークです。



もちろん、統合システムのグローバルな世界では、SCALA-Rは新しい言葉です。 ハイパーコンバージドおよびコンバージドインフラストラクチャ、コンテナー、ハイパーバイザーの仮想化を組み合わせることで、複合体はこの急速に成長する市場に新しい意味を生み出し、ロシアのビジネスだけでなくグローバルな精神にも貢献していると確信しています:-)
