ルート 始まり

レール内のルートは非常に重要です。 しかし、当面は、それらに注意を払うことさえできません。 特に、scaffoldコマンドを使用する場合は、すべて自動的に処方されます。 しかし、ある時点で、非標準のルートを作成する必要があります。 これは、プロジェクトのconfigフォルダーにあるroutes.rbファイルにアクセスするときが来たことを意味します。



ルートとは



ルートは、サイト上のルート（パス、URL）のシステムです。 ルートのおかげで、ユーザーにとって美しく明確なリンクを作成できます。 mysite.ru/articles/2008/november/13のようなリンクを入力すると、2008年11月13日のすべての記事が表示され、リンクmysite.ru/shop/shoesを使用すると、店舗から靴カタログが表示されます。 これにより、サイトのディレクトリ構造は変わりません。 ファイル自体の場所に触れることなく、いつでもルートを変更できます。 しかし、これがすべて機能するためには、ルートを構成する必要があります。



練習する



シャーマンになるテストプロジェクトを作成しましょう。 （これが初めての場合は、コメントで、レールのインストールとアプリケーションの作成のプロセスについて説明できます）。



rails routes

cd routes







わかった プロジェクトが作成され、作業フォルダーに入りました。 すぐにルートを急襲します：



rake routes







このコマンドは、2行の標準ルートを提供します。

/:controller/:action/:id

/:controller/:action/:id.:format







これは、これらの2つのルールによってURLが解析されることを意味します。

：コントローラーはコントローラー=）です。 これはMVCコンポーネントであり、ほとんどの場合、ビュー（HTML）とモデル（データベースなど）の間の仲介として機能します。 さらに明確になりますが、ほとんどの場合、すでにそれが何であるかを知っています。

：アクションは、コントローラーの呼び出されたメソッドです。 通常、コントローラーには多くのメソッドがあります。

：id-idの作成の禁止を明示的に指定しない場合、デフォルトですべてのモデル（データベーステーブル）がidフィールドで作成されます。 したがって、モデルの要素にはidがあります。 また、特定のモデル要素を削除/編集/何かする場合は、このIDをコントローラーに渡す必要があります。

-データベースを操作するための一連のメソッド（コントローラー）

 <hr> <%= link_to ' ', :action => 'new' %><br /> <%= link_to ' ', '/magazines/new' %><br /> <%= link_to ' ', new_magazine_path %><br /> <%= link_to ' ', new_magazine_url %><br />




最も正しいのは、最後の2つの方法を使用することです。 違いは、urlは完全なリンク（http：// localhost：3000 / magazines / new）を返し、pathのみがパス（/ magazines / new）を返すことです。 名前付きルートを使用する方が良いのはなぜですか？ 名前付きルートは、このルートを使用するすべてのリンクを変更することによって変更される変数です。 手でパスを書くことはまったくお勧めできませんが、書く場合はaction => 'new'（多くの場合、すべての場合に十分な名前付きルートがないため、このオプションは非常に一般的です）



テーブルの2列目はクエリメソッドです。 まったく同じリンクですが、メソッドが異なると、コントローラーのメソッドも異なります。 たとえば、同じアプリ/ビュー/雑誌/index.html.erbで：

<%= link_to 'Show', magazine %>

<%= link_to 'Destroy', magazine, :method => :delete %>





最初のリンクでは、デフォルトのGET要求が実行され（link：method =>：getでリンクを省略できます）、2番目ではmethod：deleteが送信されます。 また、どちらの場合も雑誌のリンクは同じままです。



3番目の列は、実際にはHTMLで取得するリンクです。 最後の列は、コントローラーとメソッドへのリンクの対応です。 すでに上で述べたように、任意のリンクは、コントローラ、アクション（および場合によってはメソッド）のペアとして表すことができます。



<％= link_to 'マガジンから別のコントローラーへのリンク'、：controller => 'blogs' ,: action => 'show' ,: id => '1' ,: method => 'GET'％>

したがって、インデックス1のブログへのリンクを取得します（ここでは、メソッドgetを指定できませんでした）。

<％= link_to 'コントローラーのネイティブメソッドへのリンク'、：action => 'show' ,: id => '1'％>

インデックス1の記事へのリンク。この場合、コントローラーとHTTPメソッドを指定する必要はありません。GETがデフォルトで実行され、指定しない場合はコントローラーが同じように実行されるためです。

IIレベル



実際にルートを作成したら、自分の手で最初の名前付きルートを作成しました。

名前付きルート「 localhost ：3000 / zhurnal」を作成しましょう。 ブルジョアの「雑誌」が欲しいのではなく、「シュルナル」が欲しい！



map.zhurnal '/klevi_zhurnal/:id', :controller => 'magazines', :id => nil







そこで、名前付きのzhurnalルートを作成しました。そのURLはlocalhost ：3000 / klevi_zhurnalのようになり、マガジンコントローラーからコンテンツを受け取ります。 localhost ：3000 / klevi_zhurnal / 1のような記事を今すぐ読み込もうとすると、それを破ります。 ルートにいくつかの変更を加えましょう。



map.zhurnal '/klevi_zhurnal/:action/:id', :controller => 'magazines', :action => 'index', :id => nil







これはすべてどういう意味ですか：

-/ klevi_zhurnal /という形式のURLが解決されます：controller => 'magazines' ,: action => 'index' ,: id => 'nil'-つまり、インデックスページ（index.html.erb）を取得します

-/ klevi_zhurnal / 1は、アクション '1'が存在しないというエラーを吐き出します（ルートで引数を渡すシーケンスを見てください）

-/ klevi_zhurnal / showはidが指定されていないと言います

-/ klevi_zhurnal / show / 1-ID = 1の記事を提供します（確かに存在する場合）

-/ klevi_zhurnal / edit / 1-この記事の編集フォームを提供します



確かに、リンク自体は少し重くなりました。

<％= link_to 'すべての記事'、magazines_path％>の代わりに、<％= link_to 'すべての記事'、zhurnal_path％>があります。

<％= link_to 'Article number 1'、magazine_path（ '1'）％>の代わりに<％= link_to 'Article number 1'、zhurnal_path（：action => 'show' ,: id => '1'）％>



ルートの理解を深めるために、複数形/単数形システムが導入されていることに注意してください。

すべての記事magazines_pathを表示し、

個別の記事を表示：magazine_path。

くそー-正しい言葉ではありません=）。 Articleと呼ばれるものがすべてある場合：

インデックス=> articles_path、show => article_path（：id）



それでは、新しいメソッドを作成しましょう。

アプリ/コントローラー/ magazines_controller.rbを開きます

そこにメソッドを追加します



def random

offset = rand(Magazine.count :all)

@magazine = Magazine.find(:first, :offset => offset)

render :action => 'show'

end







このメソッドは、ランダムな記事を返すだけです。 呼び出してみましょう： localhost ：3000 / magazines / random

エラーが発生します-それには私たちからのIDが必要です。 なんで？ 標準ルートは次の形式のルートを意味するため：controller /：action /：id。

ルールに従ってルートを呼び出してみましょう：

ローカルホスト ：3000 / magazines / random / 1230492

このIDのレコードは存在しませんが、機能します！ メソッドではIDをまったく使用しないため、そこにどのようなナンセンスを書くかは重要ではありません。

それでも、 localhostという形式の正しいルートを作成してみましょう：3000 / magazines / random /

これにはオプションがあります：collection => {：action =>：HTTP_method}

私たち：アクションは：ランダム、メソッドは：get

私たちは得る



map.resources :magazines, :collection => { :random => :get }







今ではすべてが機能します！ =）
