ページコンテンツをスムーズに変更する

IEで読者が一時的にnoを開いた場合、統計に基づいて、疑わしいのですが、ページを切り替えるとおそらくスムーズな効果に気づいたでしょう。



この手法はIEでのみ機能するという事実にもかかわらず、このような移行がどのように行われるかを知る価値があります。





このような効果を一時的に達成するために、次のメタタグは挿入されません。 

  <meta http-equiv = "Page-Exit" content = "progid：DXImageTransform.Microsoft.Fade（duration = .5）" />


一般的に、エフェクトをトリガーできるイベントには、ページを開くこととページを離れることの2つがあります。

対応するメタタグは次のようになります。 

 <meta http-equiv = "Page-Enter" content = conversion filter>
 <meta http-equiv = "Page-Exit" content = conversion filter>


教育プログラム

  1. Page-Enter



    、ページを開くときに適用するフィルターを決定します。
  2. Page-Exit



    は、ページを離れるときに適用するフィルターを決定します。
  3. _



    は、パラメータを持つフィルターの1つです。 フィルタの完全なリストはmsdnにあります。
  4. (



    );





    このようなメタタグはIEでのみ機能します。

    Page-Enter



    とPage-Exit



    両方を挿入する必要はありません。 両方できますが、できるのは1つだけです。 何を達成したいかによります。

    外部cssファイルへのリンクは、メタタグの後に定義する必要があります。 通常、メタタグは開始タグの直後に挿入するのが最適です。
