Lifehacks NaviHaka

最初のNaviaddressハッカソンの開催から学んだ教訓。 そして、私たちが一番気に入ったプロジェクト。



10月9日に、Naviaddress APIを使用する新しい方法を見つけるために設計されたオンラインハッカソンである最初のNaviHackの受賞者を表彰しました。 これまでのところ、オンラインハッカソンはオフライン競技よりも少ない頻度で行われているため、私たちは組織の経験を足跡をたどり、Naviaddressから賞を受賞した最も印象的なプロジェクトについて簡単に話したい人々と共有することにしました。







ヒント1：オンラインハッカソンを試す



原則として、従来のハッカソンは短いオフライン形式で開催されます。2〜3日で、参加者は自分のアイデアをある種のクラフトに変えなければなりません。 オンラインハッカソンはさらに長くすることができます：QTUM 2018ブロックチェーンプラットフォームのグローバルバーチャルハッカソンは1か月半続きました。 プロジェクトを完了するために、NaviHackメンバーに10日間を与えました。 2つの週末が当たるように、特に日付が選択されました。 オフラインセミナーの開催に関するこれまでの経験によれば、人々は火曜日と木曜日のイベントが最も得意であるため、木曜日にオープニングイベントを、火曜日にクロージングイベントを計画しました。 長いハッカソン期間は、アイデアの開発の質と深さにプラスの効果をもたらしました。 中間チェックポイントの存在により、チームは距離全体にわたって動機と効果を維持できました。 その結果、26の実用的なアイデアを得ましたが、その多くには本格的なビジネスケースになる可能性があります。



オンラインハッカソンには、高い効率に加えて、もう1つの重要な利点があります。それは安価です。 NaviHackの費用は、参加者数が同程度のオフラインハッカソンの約3倍でした。これにより、賞金プールを増やすことができました。 ハッカソンの組織化に関するほぼすべての作業は、私たち自身が行いました。 彼らはお金を、テレグラムとVKチャンネルでのコンテキスト広告とプロモーションを通じて参加者を引き付けるためだけでなく、ホールを借りるのにも費やしました（そして、あなたはそれを拒否することができます、下記参照）。 最も効果的なものの1つは、ハッカソンの発表をロシアのハッカーグループに配置することでした。



ヒント2：オフラインミーティングに群衆を期待しない



NaviHackのオフラインオープニングセレモニーでの参加は75人の参加者によって確認され、15人だけが来ました。「ボトル」で私たちが借りた広々としたホールは非常に大きいことがわかりました。 オンラインのハッカソンは、問題を解決するために非居住者および外国人プログラマーを引き付けます（ロシア、モルドバ、ウクライナ、イスラエル、キプロス、カザフスタン、ベラルーシの人々がNaviHackに参加しました）。 これは、オフラインミーティングを完全に放棄する必要があるという意味ではありません。ハッカソンに重みが加わり、優秀な専門家と非公式にコミュニケーションできるようになります。 最後に、オープニングセレモニーの経験を考慮し、当社のオフィスでNaviHackの受賞者を表彰しました。



ヒント3：オンラインハッカソンでは、プラットフォームが役立ちます



ハッカソンの以前の経験から、電報や他のメッセンジャーは多くのチームの仕事の組織を完全に管理していないことがわかります。 そのため、NaviHackは、ハッカソンを実施するためのロシアのプラットフォームであるACTUMを試すことにしました。 彼女のおかげで、たとえばチームの無料メンバーの検索など、組織の問題を解決するのが簡単になりました。



NaviHackのカスタムフォーマットは、ACTUMの同僚にとって多くの新しい課題をもたらしたため、現在、システムは活発に開発されています。 次のハッカソンでは、このプラットフォームとの協力を継続する予定です。



ヒント4：賞金プールで第1レベルと第2レベルの両方の問題を解決する



一か所にたくさんの明るい頭を集めたので、彼らの知識を十分に活用しないことは犯罪者です。 受賞者への4つの賞品（1、2、3位および審査員の共感賞品）に加えて、NaviHackでは参加者に対して2つの個別の賞品が作成され、APIの最終決定に役立ちました。



ハッカソンが始まる前に、Telegramでテクニカルサポートチャットを開始し、コメントを書いている人を監視しました。 「カート」に加えて、連中はサイトとVKontakteグループで直接私たちに手紙を書いた。 ある参加者は、サービスに対してDDOS攻撃を仕掛けました。 Naviaddressテクニカルサポートチームは、参加者が発見したバグに迅速に対応し、ハッカソンの主催者はリアルタイムで質問に回答しました。 API Naviaddressについて大きなフィードバックを受け取りました。この作業の利点を過大評価することはほとんどありません。



ヒント5：審査の透明性と独立性を確保する



オンラインハッカソンの主催者は、評価の客観性に大きな注意を払わなければなりません。 NaviHackの一環として、9人の独立した審査員を招待してこの問題を解決しました。 それらのそれぞれは、一般的かつ詳細な基準システムに基づいてマークを付けます。 審査員は互いに連絡を取り合っておらず、評価について一緒に議論しませんでした。 例外は特別な審査員賞（明白な理由による）でしたが、それは数字で確認されました：特別な賞を受賞したRxProjectチームのプロジェクトは合計ポイントで4位でした。



ヒント6：200％の参加者意識を提供する



オンラインハッカソンの場合、チームと主催者が物理的に異なる場所にいる場合、情報は冗長である必要があります。 Navihack.ruウェブサイト、Telegramチャット、VKontakteグループ-ハッカソンの準備のため、くしゃみを記録するために最も便利なプラットフォームを使用しました。



ハッカソンはパートナーとのオープンなコミュニケーションの最初の経験であったため、developer.naviaddress.com開発者サポート用の特別なサイトを作成し、APIのさまざまな使用例を投稿しました。 多くの参加者にとって、これは開発における深刻な助けであることが判明しました。



すべての変更とニュースは、主催者だけでなく参加者も記録する必要があります。 たとえば、最終チェックポイントで、各チームに参加者の構成を明確にするように依頼しました。 長いハッカソンでは、一部のプログラマーがチームを去り、他のプログラマーが来ます。チーム（またはそのキャプテン）が労働参加のシェアを分配しなかった場合、勝利の場合、お金のシェアリングに関連する競合が生じる可能性があります。 NaviHack中の私たちの「情報要求」は、そのような先例を避けるのに役立ちました。 ボーナスは課税対象であるため、主催者は税金を源泉徴収することを大文字で随所に確認しました。 苦情はありませんでした。

1か所



チーム： RAFiKi

プロジェクトエリア：ビジネスプロセスマネージャー

問題：広告主が屋外広告を配置するのが難しい

ソリューション： Naviaddressテクノロジに基づいたビルボードサービスは、Yandex.Traffic分析の集約を使用して、各ビルボードの可視性を評価します。 広告主は、広告プラットフォームの需要のグラフと、これらの広告板を毎日通過するターゲットオーディエンスのプロファイルを確認します。 このサービスにより、最適な広告面を合理的に選択し、広告代理店に予約することができます。



2場所



チーム： ExDev

プロジェクトエリア：ビジネスプロセスマネージャー

問題：マーチャンダイザーを制御する必要がありますが、既存のシステムはこれを非効率的に実行します。 マーチャンダイザー自体は、多くの場合、店を見つけるのが困難です。 建物の異なる階にある中庭に設置することができ、マーチャンダイザーの入り口はほとんど常に公式です。

ソリューション： ExDev-モスクワとハリコフの統合チーム-Naviaddressプラットフォームとそのラストマイル機能を使用して、スーパーバイザーがアウトレットのデータベースを作成し、マーチャンダイザーによる訪問を制御できるようにしました。 NaviMerchサービスにより、マーチャンダイザーは販売時点でチェックインし、スーパーバイザーへのルートを確認し、必要な情報をすべて送信することができます。



3場所



チーム： Traffox

プロジェクト範囲：管理会社への苦情

問題：居住者は事故や故障の申請の実行を管理しておらず、管理会社はそれらを排除するためにリアルタイムを考慮しておらず、監督当局の要件に対応する時間がないため、結果として罰金を受け取ります。

ソリューション： Naviaddressプラットフォームを使用して構築された管理会社のワークステーションは、居住者のアプリケーションを処理し、アプリケーションの作業時間を監視し、naviaddressで場所を追跡します。 75％の精度の機械学習は、次の共同事故と故障の外観と場所を予測します。



特別審査員賞



コマンド： RxProject

プロジェクト範囲：配達

問題：オンライン注文の受信者が自宅または職場で宅配便業者を待つことは必ずしも便利ではなく、宅配業者が工業地帯やその他の混乱する場所で道を見つけることは困難です。

解決策： AnyPlaceサービスでは、中間地点（駐車場や高速道路の脇など）に配送できます。 Naviaddressを使用すると、クライアントと宅配便業者は注文を転送するポイントを簡単に見つけることができ、手渡しのトランザクションに追加のセキュリティを提供します。



米国を驚かせたプロジェクト



Iv'Anチームは旅行者向けのNaviGuideモバイルアプリを開発しました。これにより、観光客は街の景色を通るすべてのルートを見ることができます。 各ルートは、一連のアトラクション、その説明、音声ガイド、およびそれらに到達する方法で構成されています。



チーム101は、イベントボットを作成しました。このイベントボットでは、主催者がすばやくイベントを作成し、参加者を登録し、チケットのために参加者からお金を集め、外国人に住所を伝え、ゲストに行く場所を説明します。



CryptoAnchorsチームは、製品の生産と出荷後に製品の独創性を確保する方法を見つけました。 元の秘密キーと署名機能を使用して製品に簡単なIoTセンサーを接続すると、ロジスティックの各段階で製品の動きを追跡できます。 Naviaddressプラットフォームは、配送サイクルの任意の時点で製品の座標を取得するのに役立ちます。



しかし、これらの7つだけでなく、43の参加チームのそれぞれ、ほぼ100人の参加者のそれぞれが、私たち自身の製品に何か新しいものを見せてくれました。 APIの改善に役立った人もいれば、貴重な質問をした人もいれば、関連する市場のアイデアで頭を吹き飛ばした人もいました。 主催者が受け取ったフィードバックの観点から、NaviHackは投資されたお金の支払いを繰り返しました。 これは、オンライン形式で見た主な利点の1つです。 したがって、ハッカソンに数百万ルーブルを費やす準備ができている場合は、オンラインでそれを改善し、賞金プールを増やして、より多くのクールなチームを引き付けてください。 私たちはそれをやりました-あなたにも同じことを望みます。
