物理LinuxサーバーのMicrosoft Hyper-Vハイパーバイザーの仮想環境への移行

物理的なLinuxサーバーをMicrosoft Hyper-Vハイパーバイザーの仮想環境に移行するスキームでは、すべてがあいまいですが、これについて明確に話そうとしない限り、困難です。



「Hyper-VのLinux P2V変換への緩慢な試み」（Ivashentseva Andrey、 http: //bit.ly/hzaV6L）の続き。 そのような移行のためのインターネット上の単一のソリューションはありません;主なオプションは長い間知られています：
  1. Linux-> Vmware Converter 4-> Vmware ESX-> VMDK2VHD-> Hyper-V-> Install LinuxIC
  2. 代替ソリューション：

    1. PlateSpin®Migrate （以前はNovellが所有していたInvirtus Enterprise VM Converter）、
    2. Quest®vConverter （以前のVizioncore ）、
    3. Citrix XenConvert （ただし、ここではVmwareへの中間移行なしでは実行できません）。


  3. LinuxをHyper-Vに直接インストールし、Linuxを使用してデータと構成を移行することも非常に興味深いです。
  4. おそらく他に何かがあります。誰かがこの知識を共有してくれたら嬉しいです。


Hyper VおよびSystem Center Virtual Machine ManagerはLinux P2Vを移行できませんが、Linux P2Vを正常に仮想化することを妨げることはありません; Linux-LinuxIC用のhyper-v統合ツールがあります。

私が解決しようとしていた問題は、Oracleデータベースが実行されている古い機器を解放することでした。それらはすべてFedoraで実行され、使用するHyper-Vハイパーバイザーに仮想マシンを転送しました。



私はリストの有名なパスナンバー1に従いました。 移行は次の手順で構成されます。Hyper-VおよびSystem Center Virtual Machine Managerと共にWindows Server 2008が既にインストールされていることを前提としています。

  1. WindowsマシンにVmware Converterをインストールします。
  2. VMware ESXiをインストールします。 SSHアクセスを構成します。
  3. ESXiでの移行プロセス。
  4. Winscpを使用して、SSH経由でESXiからHyper-V ServerにVMDKをコピーします。
  5. vmwareファイルを含むフォルダーをSCVMMライブラリに追加し、V2Vを変換します（SCVMMウィザードを使用してVMDKをVHDに変換）。
  6. LinuxサーバーをHyper-Vで確認して実行します。
  7. LinuxICコンポーネントの統合。


Vmware Converterは、Vmwareへの移行中にのみ必要です。高速イーサネットアダプターがあれば便利です。 移行を開始し、ルートアクセスでデータを指定し、[ソースの詳細を表示]をクリックすると、Linuxサーバーの移行をすぐに試行できます。









この瞬間から、移行にはVmware ESXiが必要であり、無料でダウンロードして使用できることがわかります。 vSphere Clientを介して管理するための60日間の期間もあります。 仮想化ハードウェアをサポートするプロセッサを搭載した最初のPCを搭載しました。 Vmware ESXiのインストールを説明する意味はありません。問題を引き起こすものはありません（next-next -next）。 念のため、インストールガイド（http://bit.ly/frcqdD）へのリンクを示します。 考慮すべき唯一のことは、移行したサーバーのデータ用に十分な空き領域がハードディスクにあることです。 スクリーンショットに示されているように、リモートSSHアクセスも設定する必要があります。







Wizard Vmware Converterがp2vの移行を開始し、サーバーの1つから140 Gbのデータが届くまで約1時間かかりました。





次の手順では、たとえば、Winscpをダウンロードして、移行の完了後にSSH経由で必要なファイルをダウンロードします。







System Center Virtual Machine Managerで、Vmwareのイメージ、SSHを介してVmware ESXiから取得したすべてのファイルを含むフォルダーを追加します。





すべてを追加した後、V2VをSCVMMに移行するタスクを作成します。







繰り返しになりますが、Hyper-Vホストのみのサーバーにマシンを展開し、SCVMMは別のマシンに展開していたため、プロセスには約1時間半かかりました。







最終的に、Hyper-Vの下でLinux Fedoraの下で動作するマシンを手に入れました







Hyper-Vツールを統合する必要はありませんでした。マシンはスリープし、必要なときに1時間待機します。

さらに、無料のツールVMDK2VHD（ http://bit.ly/i2IuaT ）を使用すると、SCVMMなしでも実行できると言えます。
