Meac Veracity-新しい分散バージョン管理システム



こんにちは、好奇心％盛な％username％ ！



数か月前に、別の有望なバージョン管理システムであるVeracityを偶然見つけました。これについては、週末に掘り下げるために、今日お話ししたいと思います。 Veracityが1年以上開発されているという事実にもかかわらず、Habréの名前はコメントで数回しか言及されていませんでした。 カットの下に、Veracityの簡単な説明と、それに関するより詳細な情報を取得できるリンクがあります。



したがって、 Veracity （英語の「veracity」、「reliability」、「accuracy」）は、SourceGear社のApache 2.0のライセンスの下で開発されたGitに類似した分散バージョン管理システムです。



Gitとの主な違い

  1. ローカルリビジョン番号の可用性。 Veritityは、Gitと同様に、暗号化ハッシュ関数を使用して個々のファイルをバージョン管理します。 ただし、ローカルリポジトリでは、Subversionで発生するように、各コミットは一意のリビジョン番号を取得します。 リビジョン番号とハッシュは互いに同等に使用でき、多くの場合コロンで示されます。
  2. 複数のハッシュ関数のサポート。 Gitがその作業でSHA-1関数を使用していることは誰もが知っています。 Veracityでは、 SHA-1 、 SHA-2、またはSkeinから選択できます。
  3. ファイルの正式な名前変更。 ファイルの名前を変更するときのGitは実際にファイルを削除してから、同じ内容と新しい名前でファイルを作成します。 Veracityは、既存のファイルの名前を正確に変更します。
  4. ファイルロックのサポート。 Subversionのような集中型バージョン管理システムは、 ファイルのロック （svn lock \ svn unlock）と変更のマージ （svn merge）の2つの作業スキームをサポートしています 。 Gitおよび他の分散バージョン管理システムは、作業で中央リポジトリを使用することがよくありますが、ファイルロックメカニズムがありません。 正確さはこの欠陥を修正します。 ファイルをロックするには、ネットワーク経由でリモートリポジトリにアクセスする必要があることが理解されています。
  5. スタンプのサポート。 Veracityには、スタンプと呼ばれるまったく新しい機能が含まれています。 スタンプは、開発者が任意のコミットに添付できる任意のテキストからのマークです。 一意の名前を持つ各コミットに一致するタグとは異なり、同じスタンプで任意の数のコミットをマークできます。 たとえば、継続的インテグレーションシステムを使用する場合、一連のテストにエラーなしで合格したコミットに「合格したテスト」スタンプを付けることができます。
  6. 分散リポジトリデータベース。 多くのバージョン管理システムは、作業ディレクトリ内にリポジトリデータを保存します。 たとえば、Gitには、バイナリオブジェクト、ツリー、コミットオブジェクト、インデックス、構成、フックなどを格納する.gitフォルダーがあります。 Veracityは、リポジトリに関するほとんどのサービス情報を特別なデータベースの作業ディレクトリの外部に保存します。 これにより、たとえば、1つのリポジトリに対して複数の作業ディレクトリを一度に作成できます。
  7. 組み込みのJavascriptインタープリター。 Veracityは、JavaScriptをメインのスクリプト言語として使用します。 たとえば、フックをJavascript関数として作成できます。 Veracityを使用してWebアプリケーションのバックエンドを作成するために使用できる組み込みのJavaScript APIもあります。 JSONを使用してシリアル化されたデータを保存することは論理的です。
  8. リポジトリを表示するための組み込みWebアプリケーション。 1つのコマンドで、リポジトリの内容、変更履歴、プロジェクトビルドステータス、またはバグトラッカーを表示できるアプリケーションを使用して、組み込みWebサーバーを起動できます。


設置

インストール手順は最も一般的であり、2つの可能な選択肢があります。

  1. ソースコードからビルドします。 システムは、他のバージョン管理システム（Git、Subversion、Fossilなど）と同様にCで記述されているため、依存関係とcmakeやmakeなどのユーティリティのソースコードのインストールを処理する必要があります。 これらのことを好む人のために、ここにLinuxとWindowsのビルド手順があります 。
  2. バイナリパッケージからインストールします。 Veracityの選択を迅速に開始するには、ダウンロードページでダウンロードできるバイナリパッケージを使用する方がはるかに簡単です 。 Debian互換システム（* .deb：x86、x64）およびWindows（* .msi：x86、x64）用のバイナリパッケージがあります。


メインチーム

Veracityを使用するには、短いvvコマンドが使用されます。 以下は、基本的なVeracity操作とサンプルコマンドのリストです。

チーム 説明 例
vv add ファイルをバージョン管理下に置きます。 Gitとは異なり、Veracityにはステージングの概念がありません。 ファイルへのすべての変更がコミットされます。 
$vv add filename1 filename2 ~/thisdir
vv addremove バージョン管理下に新しいファイルを追加し、バージョン管理から既存のファイルを削除します 
 $vv addremove dirname1
vvブランチ 現在のブランチの一覧表示またはブランチの変更 
 $vv branch $vv branch add devel
vv猫 バージョン管理下にあるファイルの内容を一覧表示する 
 $vv cat filename
vvチェックアウト ローカルリポジトリの新しい作業コピーを作成します（複数ある場合があります） 
 $vv checkout
vv clone 既存のリポジトリの新しいコピーを作成します。 次に、新しく作成されたリポジトリから、必要な数の作業コピーを作成できます。 git cloneと同様に機能します。 Veracityは、リモートリポジトリを操作するためのHTTPプロトコルのみをサポートします。 
 $vv clone http://example.com/repos/reponame reponame
vvコメント コミットの作成時に指定したコメントに加えて、コミットに別のコメントを追加します。 コマンドを呼び出すたびに、コメント付きの新しい行が追加されます。 古い行は保存されます。 
 $vv comment --rev=123 --message='A new comment'
vv commit コミットを作成する 
 $vv commit --message='Commit message'
vv config Veracityを再構成します。 特定のオプションの値を設定するだけでなく、設定をJSONドキュメントにインポートおよびエクスポートしたり、設定をデフォルト値にリセットしたりすることもできます。 
 $vv config set whoami/username vania-pooh
vv diff 現在のリビジョンと指定されたリビジョンの違いを出力します。 
 $vv diff --rev=3 filename
vv diffmerge diffと同じですが、比較のためにサードパーティのDiffMergeプログラムを使用します。これは個別にインストールする必要があります。そうしないと、コマンドはエラーをスローします。 
 $vv diffmerge --rev=3 filename
vvエクスポート バージョン管理下にあるすべてのファイルを別のディレクトリにコピーします。 
 $vv export reponame ~/copyhere
vv高速エクスポート リポジトリの完全な状態（ファイルの内容+メタデータ）を1つのテキストファイルに保存できます。 このファイルは、 vv fast-importコマンドを使用して別のコンピューターにインポートできます。 
 $vv fast-export reponame ~/backupname.fi
vv高速インポート vv fast-importコマンドで作成されたファイルから新しいリポジトリを初期化します。 
 $vv fast-import --hash=SHA2/512 ~/backupname.fi
vvヘッド リポジトリのアクティブな各ブランチの最新のコミットの説明を表示します（ブランチはvv branch closeを使用して閉じることができます）。 
 $vv heads
vvヘルプ Veracityコマンドの統合ヘルプシステム。 
 $vv help branch new
vv履歴 以前のコミットのリストを表示できます。 git logと同じです。 
 $vv history
vv着信 リモートサーバーからダウンロードされる変更を示します。 
 $vv incoming
vv init 新しいローカルリポジトリを作成します。 リポジトリの名前と動作させるディレクトリを指定する必要があります。 後で他の作業ディレクトリを追加できます。 
 $vv init reponame dirname
vvの葉 「葉」のリスト、つまり ブランチでの最後のコミット。 
 $vv leaves
vvロック 編集のためにファイルをブロックします。 
 $vv lock filename
vvロック 編集のためにロックされているファイルのリストを表示します。 
 $vv locks
vv merge 指定されたコミットに対する変更を、作業ディレクトリ内のファイルの現在の状態に注ぎます。 
 $vv merge -r 37939b07309af8232c44048ca0a1633c982b7506
vv move 指定したファイルまたはディレクトリを新しい場所に移動します。 
 $vv move filename ~/newdir
vv発信 現在または指定されたリモートリポジトリにダウンロードされる変更のリストを表示します。 
 $vv outgoing
vv親 現在変更されているファイルの親コミットのリストを表示します。 
 $vv parents
vv pull リモートリポジトリから変更を取得します。 git pullと同じです。 
 $vv pull # ,          vv update
vv push 変更をリモートリポジトリに送信します。 git pushと同じです。 
 $vv push # ,
vv remove バージョン管理からファイルを削除して消去します。 
 $vv remove ~/repo/filename
vv rename ファイルの名前を変更します。 
 $vv rename ~/repo/oldfile ~/repo/newfile
vvレポ リポジトリを作成、変更、削除し、リストします。 
 $vv repo info
vv resolve ファイルマージの競合を解決します。 
 $vv resolve ~/filename #
vv復帰 現在の変更をロールバックします。 
 $vv revert ~/filename1 ~/filename2
vv serve Veracity組み込みリポジトリ管理アプリケーションを起動できます。 
 $vv serve --port=8080 --public
vvスタンプ 指定したコミットにスタンプを追加および削除できます。 
 $vv stamp add approved --rev=3
vvステータス 現在の作業ディレクトリ内のファイルのステータスを表示します。 
 $vv status
vvタグ 一意のテキスト名をコミットに割り当てます。 
 $vv tag add tagname --rev=3
vvロック解除 vv lockでロックされたファイルのロックを解除します。 
 $vv unlock filename
vvアップデート リポジトリから作業ディレクトリの状態を更新します（ vv pullの実行後に適用されます）。 
 $vv update
vvユーザー リポジトリを使用してユーザーアカウントを管理できます。 
 $vv user create vania-pooh
vvバージョン 現在インストールされているVeracityのバージョンを表示します。 
 $vv version
vv whoami 現在のユーザーアカウントを表示または設定します。 
 $vv whoami anotherUser
vv zip リポジトリの内容をzipアーカイブに保存します。 
 $vv zip ~/backup/archive.zip

ファイルの場所

前述のように、Veracityは作業ディレクトリファイルとは別にリポジトリメタデータを保存します。 これにより、複数の作業ディレクトリを作成できます。 Veracityのドキュメントには、いくつかの異なるストレージエンジンがサポートされていると記載されていますが、 FS3エンジンはデフォルトで使用されます。 このエンジンは、データベースクエリを格納するSQLite3データベースとテキストファイルを使用します。 データベースの内容を表示できるようにするために、Firefox用のSQLite Manager拡張機能をインストールできます。 デフォルトでは、すべてのリポジトリデータは〜/ .sgcloset /ディレクトリに保存されます。 さらに、各作業コピーには、リポジトリ名およびその他のメタデータへのリンクを含む隠し.sgdrawerディレクトリがあります。 リポジトリファイルを無視する場合は、以下に示すように、それらを.sgignoresまたは.vvignoresファイルに入れます。 Veracityには現在、 vvとvscriptの 2つの実行可能ファイルしかありません 。 1つ目はバージョン管理コマンドの実行を担当し、2つ目はJavascriptで記述されたスクリプトを実行できるようにします。
