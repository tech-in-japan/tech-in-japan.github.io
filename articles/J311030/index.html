「最近BMWのオンライン販売システムを開始しました」：T-Systemsの業務におけるJavaと会議の役割について

IT企業がJavaをどの程度積極的に使用しているかを理解するにはどうすればよいですか？ 兆候の1つ：会社がJava会議を後援し、そこでスピーカーを派遣する場合、それはアクティブです。 したがって、 Joker 2016を見越して、Javaとカンファレンスへの参加についてスポンサーに質問することにしました。 最初の投稿はT-Systemsについてです。







会社の複数の従業員に質問しました。T-SystemsでのJava開発の個人的な経験について、何人かは会社全体についてです。 対話者は次のとおりです。



  • キリルチェルニン-HRビジネスパートナー
  • ドミトリーアレクサンドロフ -一流のエキスパートプログラマー（スピーカージョーカー）
  • Vyacheslav Kruglov -Java開発者、チームリーダー（スピーカージョーカー）
  • Maria Zernova-内部コミュニケーションのスペシャリスト






キリル・チェルニン



-ジョーカーに参加できる理由は何ですか？



-2つの問題を解決します。 一方では、これは当社の従業員と他社の従業員との間の経験の交換です。 専門家のコミュニティは、新しいアイデアと新鮮な外観の点で非常に重要なポイントです。 一方、私たちはプロのコミュニティで雇用者として自分自身を宣伝し、100以上のプロジェクトを持つ大企業にとって、これはさらなる成功のために基本的に重要です。



-あなたの会社でJavaはどのような役割を果たしますか。これらの場合、Javaが優先されるのはなぜですか？



-これは、多数のプロジェクトの主要な開発言語です。 プロジェクトの大部分は分散されており、チームの一部はTシステムの別の部門（たとえば、ブラジル、スロバキア、ドイツ）にあります。 また、Javaは、複雑な企業のマルチモジュールシステムを作成するための強力なツールであるため、好みがありました。



-現在、どんな面白いJavaプロジェクトがありますか？



-たくさんあります。 当社の主力プロジェクトの1つは、スイス鉄道向けです。 私たちは、スイスのすべてのタイプの交通機関のチケットの販売管理の統一システムをゼロから開発し、200社以上の運送会社を統合しています。 親会社であるDeutsche Telekomでは、多数のプロジェクトが行われています。たとえば、企業の顧客向けのポータルであるビジネスポータルGK-Portaleなどです。



私たちは最近、オンライン販売システムであるBMWのプロジェクトを立ち上げました。 これは、BMWドイツとの緊密な協力の下、ロシアのオフィスでゼロから行う新しいプロジェクトです。



ドミトリー・アレクサンドロフ



-あなたはJava EEとSpringの両方に興味があります-T-Systemsでよく仕事をしているのはどれですか？



-両方のプラットフォームで多くのプロジェクトがあります。 私は両方のテクノロジーにほぼ同時に取り組みました。 具体的には、現在Java EEプロジェクトに取り組んでおり、このプラットフォームが提供する標準化が本当に気に入っています。 同じコードが追加の構成なしで複数のJava EE 7サーバーで実行されます。 私の意見では、これはJava EEの重要な利点の1つです。 多くの場合、Springは素晴らしいアイデアの「インキュベーター」であり、Java EEで標準化されます。



-Java EEの将来と、Oracleが今戦うことを約束している「停滞」についてどう思いますか？ T-Systemsでの仕事にどの程度の影響があり、約束のEE 8およびEE 9になりますか？



-はい、最近多くの議論が行われ、パニックの気分さえあります。 私は個人的に、近い将来EEが死ぬことはないだろうと信じており、さらに、それはうまく発展するでしょう。 あまりにも多くのビジネスがそれに結びついています。 Oracleに加えて、このテクノロジーの周りには強力なコミュニティがあり、非常に活発であることに留意してください。 JavaOneでは、新しいロードマップが示されました。 新しい期限に遅れないようにしたいです。 Java EEの新しいバージョンがプロジェクトに与える影響については、プラスになると思います。 私の現在のプロジェクトでは、EE 7の多くの革新を非常に積極的に導入しました。新しいプロジェクトでもEE 8と同じになると確信しています。



-ブルガリアでjPrime会議を開催しています。 他の人の会議に参加し、自分の会議を開催した経験はどのように関係していますか？



-はい、スピーカーの目、主催者の目、視聴者の目で「テクノロジー会議とは何か」を見る機会があることがわかりました。 これは確かに素晴らしいことです。このようにして、すべての関係者にとって会議をさらに改善できるからです。 jPrimeカンファレンスは、非常に若かったものの、すでにこの地域で最大規模になりました。 5人のオーガナイザーのグループはすべてプログラマーであり、プロのイベントオーガナイザーはいません。 したがって、私たちはゼロから多くを学ぶ必要があり、他の会議に参加することは多くの助けになりました。



-これに加えて、あなたはブルガリアのJavaユーザーグループのリーダーでもあります-コミュニティの活動について教えてください。



-私たちのコミュニティのタスクは非常に単純です。可能な限りJavaを知りたいのです。 さらに野心的なタスクを設定しました-この言語とプラットフォーム全体の開発に直接参加することです。 登場したばかりのイノベーション（MVC 1.0など）、またはまだ開発および研究中のイノベーション（Valhallaプロジェクトなど）に関するワークショップを行っています。 このために、私たちはお互いとのコミュニケーションに加えて、海外からも含めてこの分野の最大の専門家を招待するよう常に努力しています。 さらに、私たちは定期的にそれをしようとします。 そして、会議や会議は、私たちのアイデアを実現するための素晴らしい方法です。



-ジョーカーで何を話しますか？



-私のレポートは「JBatch、または最大のデータではない」と呼ばれます 。 比較的新しいJSR-352についてお話します。そのタスクは、Javaでデータ処理をバッチ（またはバッチ）処理するためのアプローチを標準化および最適化することです。 この言語は、バッチ処理のタスクが非常に一般的であるエンタープライズでしっかりと確立されており、驚くべきことに、最近まで標準化されたソリューションがありました。



ヴャチェスラフ・クルグロフ



-T-Systemsでは、あなたはJava開発者であり、チームリーダーです。そして、どのチームを率いていますか、何に取り組んでいますか？



-私たちのプロジェクトはSBBと呼ばれ、スイスの公共交通機関のチケットを販売するためのソフトウェアを開発しています。 チームは約40人で、ロシアですべての開発を行っています。 顧客は要件のみを提供します。 実際、私たちは製品チームです。



-昨年のジョーカー、Hibernateを使用したさまざまなデータベースの操作に関するレポートがありました-T-Systemsはどのデータベースで操作できますか？



-プロジェクトに大きく依存します。 多くのプロジェクトがあり、独自の要件があります。 もちろん、最も一般的なのはOracleです。 MySQLに適合。 一部のプロジェクトでは、NoSQLのMariaDBを使用しています。



-最近、 「JPA / Hibernateを使用する5つの理由」という素材が登場しましたが、あなたの経験では、それらを使用する主な理由は何ですか？



-私もこの記事を読みましたが、なかでも「開発の生産性を高める」という項目が気に入っていました。 実際、Hibernateを使用すると、日常的なリクエストを心配することなく、必要なコードを非常に迅速に作成できます。



- 「Hibernateは私のキャリアを台無しにしました」という反対のテキストが現れました。彼はどのようにしてプロジェクトを悪化させ、良くはしなかったのですか。



-私が働いていたプロジェクトでは、これは役に立ちませんでした。Hibernateは役に立ちました。 はい、多くの問題があり、その解決には時間がかかりましたが、全体的には安心しました。 しかし、これが必要ではないタスクがあることは間違いありません。 これは非常に深刻なフレームワークであり、多くの機能を備えており、どこかでは必要ありません。



-ジョーカーユニバーシティデイとJavaスクール形式の両方で若者に教える経験がありますか？また、そのようなトレーニング中に自分自身を形成しますか？ レポートは、彼ら自身が何かを学んだことを明らかにしましたか？



「はい、もちろん。」 素材の準備と検索の過程で、仕事では見つけられない非常に興味深い点がありました。 後で作業で使用できるものを含めます。



そして、「非技術的」計画でも勉強します。 Jokerの準備において、JUG.ruグループからのトレーニングは非常に有用であることが判明しました。レポートの維持と再構築に関する知識を構築するのに役立ちました。 これは、Javaスクールを含め、最終的には大いに役立ちました。



マリア・ゼルノバ



-会社はさまざまな通信方法を使用しています-会議は他の人とどのように違いますか？



リモートコミュニケーションには、会社、業績、および顧客の利点に関する標準的な言い回しが多く含まれています。 イベントでのライブコミュニケーションにより、訪問者はさらに興味深い情報を入手できます。 チームを募集する特定のプロジェクトについて質問したり、詳細について代表者と話すことができます。



-ジョーカーはサンクトペテルブルクで開催されています-これはTシステムズの都市の1つですか、それとも会場は重要ですか？



サンクトペテルブルクのオフィスは、ロシアの当社の中央オフィスであり、ほとんどのチームがここにいます。 Javaプロジェクトは主にここで行われます。たとえば、サンクトペテルブルクでは、親会社のDeutsche Telekomの内部システムや、自動車産業の顧客向けのプロジェクト（非常に大規模なプロジェクトBMW Online Sales Master（OSM））を開発しています。 これはBMWの3番目のプロジェクトで、2016年にソフトウェア開発センターで開始されました。



したがって、原則として、サンクトペテルブルクで開催される会議に直接参加します。 しかし、もちろん、これは私たちのスピーカーを制限しません-彼らは開催都市に関係なく関連する会議で話します。 そして、私たちはさまざまな都市の従業員を招待し、私たちは移動を支援する準備ができています。



-スピーカーの報告後、会社は通常どのような反応を受け取りますか？



通常、講演者のスピーチの後、ブースの訪問者の流れは、私たちの会社、プロジェクトについての詳細を知りたいと思って、大きく増加します。 そして、問題はレポート自体のトピックではなく、プレゼンテーションのために会社について形成される一般的な肯定的な印象にあります。 これは、キャリアを始めたばかりの人々のためのイベントでのJavaスクールとテストスクールのリーダーのレポートの後に特に顕著です。彼らはパンフレットからではなく、「内部から」学習機会について学びます。 一方、経験豊富な専門家のレポートにより、複雑な設計上の問題について話し合い、見つかった解決策を共有できます。



-答えてくれてありがとう、10月にドミトリーの新しいレポートを待ちます。 それまでの間、前のジョーカーのT-Systemsスピーカーの報告を思い出すことができます。DmitryはNashornについて、VyacheslavはHibernateについてです。
