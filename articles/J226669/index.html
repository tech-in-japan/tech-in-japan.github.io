クロスプラットフォームのファイルマネージャー？ それは現実です





良い時間、市民 本日、サードパーティの開発者向けに、新しいオープンソースプロジェクトであるクロスプラットフォームの2パネルファイルマネージャーがオープンしました。 この記事は、おそらく背景から始めます。 有名なファイルマネージャーであるWindows用のTotal Commanderがあります。これは、豊富な機能を備えた他の製品と比較して、Win32 APIに基づく強力なプラグインシステム、詳細設定、およびマルチウィンドウモードです。 他のオペレーティングシステムの場合、同等の高さのファイルマネージャーはまだ存在しません。 そして、存在するものは長期間開発されないか、プラグインをサポートしません。 そして、すべてが順調でしたが、蓄積されたジャンクからDebianをきれいにすることに決めました。 クリーニングプロセスは、Midnight Commanderを備えたターミナルの多数のウィンドウとタブ、さまざまなエディターとビューアーの多数のタブとウィンドウ、FileRollerアーカイバ用のグラフィカルシェルなどで構成され始めました。 git initを作成し、MonoDevelopを起動しました...



そのため、ほとんどの場合とは異なり、クロスプラットフォームの新しい非営利のスタートアップ、別の2パネルファイルマネージャーがありました。 その名前は長い間選ばれていましたが、File Commanderに立ち寄ることに決めました（放棄されたFileCommanderと混同しないでください）。 略語で他の*司令官と混同されない、シンプルでわかりやすい名前。 これは、以前にHabréで作成されたXWTツールキットを使用してC＃で作成されています。 .Net / MonoとXWTを使用すると、完全なクロスプラットフォーム（Windows、Linux、Mac OS）が提供されます。 実際、FCの重要な機能は元の機能ではなく、使用されているオペレーティングシステムからのすべてのプラグインとの独立性です。



このプロジェクトは、他の問題からの暇なときに私だけの努力によって開発されており、現時点では150件以上のコミットが蓄積されています。 そして今、1年の開発の後、プロジェクトはサードパーティの開発者に開かれています。



特徴



FCは、従来の2パネルのファイルマネージャーであり、通常はTotal Commanderなどに似ています。 ただし、FARまたはMidnight Commanderなどの比較的完全なコマンドラインがあります。 非ローカルファイルシステムを操作するために、FSプラグインがサポートされています。 FSプラグインはコマンドライン操作も提供します。ファイルマネージャーを離れることなく、端末とファイルの両方を同時にSSHで操作することについてどのように感じるかを考えてください。







このプログラムには、標準キーF3およびF4によって呼び出される組み込みのビューアーエディター（VE）が組み込まれています（編集はF3によって無効にされます）。 VEは、さまざまな種類のファイルを操作するためのプラグインをサポートしています。 VEチップはコマンドライン（Vimに似ています）であり、プラグインの機能によってのみ制限された、幅広いファイル操作をキーボードで実行できるようにします。 残念ながら、現時点では、プラグインの選択は大きくありません。 fsplugins.confおよびfcveplugins.confファイルを介してプラグインを接続することは可能ですが、別個のプラグインはまだありません。組み込みのプラグインのみ-「ローカルFS」（ファイル：//）および「シンプルテキスト」（* .txt）。 プログラムインターフェイスは世界の任意の言語に翻訳することができ（「ローカライザー」はまだ初期段階ですが、それに触れています）、外観はCSS形式のテーマを使用して「XWTの機能によって制限される」「それ自体」に調整されます。



建築



FCは、モノリシックモジュラーアーキテクチャ上に構築されています。 つまり、モノリシックカーネルfcmd.exeがあります。これは、ユーザーインターフェイスを提供し、特別なレイヤーpluginner.dllを使用して、プラグインから機能の切り替え可能な部分を読み込みます。 プラグインは、ファイルシステムの操作、ファイルの表示と編集を担当します;将来的には、アーカイブプラグインとユーザーインターフェイスを変更するプラグインが計画されます（Firefoxのように）。 プラグインはpluginner.IPluginを継承するクラスであり、DLLファイルに保存されるか、fcmd.exe（組み込みプラグイン）に埋め込まれます。 FCだけでなく、プラグインでも使用できる関数とクラスは、プラグインで作成されます。 たとえば、「ファイルリスト」ウィジェットFileListPanelはVEプラグインに埋め込むことができ、複数のファイルを一度に操作するために使用できます。 UIライブラリーとして、Mono開発者による公式のXWTツールキットが使用されます。 デザインテーマは、通常のCSSファイルであり、別のプロジェクト（および私の制作プロジェクト）、µCSSライブラリによって処理されます。 ディスク選択パネルはブックマークパネルと組み合わされていますが、シンプルなブックマークメニューはどこからも削除されていません。 ディスクブックマークバーのディスクのリストは、動的ブックマーク（ブックマークメニューに追加できる）「ディスクのリスト」として実装されています。 これにより、グラフィカルモードでの短縮ダイヤルリストファイルの編集が容易になり、これはXML形式のブックマークリストファイルの名前です。



現時点で提供されている機能は、不十分で非常に生々しく、バグが多いように思われるかもしれません。 これは、私が十分な時間と経験を持っていないという事実によるものです（あなたの謙虚な使用人はアマチュアです。私は何もすることがないときにコードを書きます）。 さらに、2013年12月まで、FCはWinFormsで記述されていました。その後、ほぼ完成してから数か月間XWTに記述されました。 しかし、これらはすべて言い訳です。主なことは、アイデアが徐々に現実に翻訳されているということです！



近い将来の計画



  • FileListPanelを書き換えます ディレクトリ内のすべてのファイルを一度に明示的に表示するのではなく、部分的にロードする必要があります。
  • リスト内のファイルの並べ替え、列のドラッグアンドドロップ、ファイルのドラッグアンドドロップを行います。
  • 次のように、複数行のテキストフィールドを持つものを考え出します。 GTKでは機能しません（XWTバグ）。
  • クイック検索を念頭に置いてください（XWTの最新のイノベーションを使用してください）。
  • 構文の強調表示、オートコンプリート、16進出力、および印刷を使用して、VE PlainTextモジュールのフルテキスト編集を行います。
  • FTP、WebDAV（Yandex.Disk）、SSH用のFSプラグインを作成します。
  • ディレクトリツリーを作成します。
  • グラフィック差分を作成します。
  • アイコンを作成します。
  • バルクファイルの名前変更の機能を作成します。
