そのため、EYの分析によると、今後数年間で最大の成長は、コンテンツとビデオに関連するサービスにかかっています。 事業者は、コンテンツの収益化の経路、つまりゲーム、ビデオ、音楽などを積極的に追求する必要があります。同時に、主要資産であるインフラストラクチャをより効果的に使用する必要があります。 これは、OTTプロバイダーとの競争における彼らの「切り札」です。



今日、ユーザーは目的のサービスを待たずに、「今ここで」高品質でそれを手に入れたいと考えています。 簡単な例を考えてみましょう。ユーザーがYouTubeからビデオを見たいと思ったとします。 オペレータは、視聴中に適切なネットワークリソース（より高い帯域幅）を即座に提供できます（理想的にはそうすべきです）。 これは、インフラストラクチャの所有者であるオペレーターのみが行うことができます。もちろん、OTTプロバイダーはこれを利用できません。もちろん、オペレーターとの合意がない限りです。 ところで、このような契約の締結は、電気通信のための追加収入を受け取るための別のレシピです。



ただし、上記の簡単なシナリオでも実装するには、ネットワークインフラストラクチャとその上で実行されているサービスプラットフォームが適応性があり、オープンであることが必要です。



インフラストラクチャの変革...



従来の通信ネットワークは、分とバイトを販売するために作成されたもので、特にパラメーターのカスタマイズや個々のレベルでのサービス品質の確保を伴う新しいサービスの提供にはあまり適していません。 ただし、事業者が新しい条件で機能するようにインフラストラクチャを最適化するのに役立つテクノロジとソリューションがすでに市場に登場しています。 まず、ソフトウェア定義ネットワーク技術（SDN）とネットワーク機能仮想化（NFV）について話します。



SDNの概念には、ネットワークノードの構成やルーティングフローなど、このプロセスの管理機能からトラフィックを送信する機能を分離することが含まれます。 これにより、ベアメタル製品やホワイトボックス製品などの比較的安価な「非インテリジェント」スイッチを使用できるため、継続的に増加するトラフィックをサポートするためにネットワークをアップグレードするオペレーターのコストが大幅に削減されます。



しかし、さらに重要なことは、スイッチおよびその他のネットワークハードウェアの動作を制御するコントローラーにより、ネットワーク全体を迅速に再構成して、上位のアプリケーションまたはサービスの特定の要件を満たすことができることです。 上記の例を参照すると、SDNネットワークにより、ビデオクリップを高品質で視聴するための追加の帯域をユーザーにすばやく割り当てることができます。 または、トップマネージャーのビデオ会議セッションを実施するためのリソースを割り当てるとします。 そして、これはすでに企業顧客にとって非常に重要です。



NFVテクノロジーにより、オープンなアプリケーションプログラミングインターフェイス（API）を介してネットワークと対話するソフトウェアモジュール（仮想マシンなど）を使用して、さまざまなネットワーク機能とサービスを実装できます。



以前の場合、トラフィックフローの詳細分析（DPI）のために、特別な（そして非常に高価な）ハードウェアソフトウェアプローブをインストールする必要があり、SDN / NFVネットワークに適切な（DPI機能を実装する）ソフトウェアをインストールするだけで十分だとしましょう。 NFVツールによって実装されたネットワーク機能/サービスをチェーンでリンクして、特定の企業顧客や個々のユーザーにパーソナライズされたサービスを提供できます。



一般に、ネットワーク開発のコストを削減し、新しいサービスを導入する時間を短縮することにより、SDN / NFVテクノロジーは、大衆市場および企業市場で根本的に新しいサービスを作成できます。 これらの技術の導入により、市場に新製品を投入する準備時間が短縮されます。 競争の激化に直面して、市場投入までの最短時間は厳密に必要です。オファーの範囲を迅速に拡大する必要があります。 また、新製品が購入者を見つけた場合、品質を磨く価値があります。また、提案が失敗した場合は、すぐにそれを取り除く必要があります。 また、SDN / NFVネットワークでは、特定のクライアントに対して、特定のサービスに対して定義された、サービスの品質の制御されたレベルを設定できることも重要です。







ペルミのストリートアート



ロシア市場にはかなり成熟したSDNおよびNFVソリューションが多数あります。これには、応用コンピューターネットワーク研究センター（CPPCS）やBrain4Netなどの国内企業の開発が含まれます。 それでも、これらの技術の導入は初期レベルであり、ほとんどの場合、実験的なネットワークセグメントの段階です。



制約要因の中には、さまざまな開発者からのソリューションの互換性に関する問題や、継承されたインフラストラクチャとの新しい（SDN / NFV）ソリューションの統合に関する問題があります。 さらに、新しいソリューションを実装および保守できる有資格のスペシャリストが不足しています。 最後に、プロジェクト内の作業の調整の問題があります。多くのオペレーターにとって、通信ネットワークのIT部門と技術部門は別々のユニットです。



Rostelecomは、SDN / NFVの分野でさらに多くの活動を実施しています。 同社は、2020年までSDNとNFVに基づいてネットワークインフラストラクチャをアップグレードする予定です。Vimpelcomは、ロシアのネットワークの断片に対するテストプロジェクトの一部としてSDN / NFVを導入し、顧客に追加の容量を自動的に計算、計画、提供します。 多くの仮想ネットワーク機能（vIMS、vPCRFなど）、およびvEPCソリューションの概念をテストして実装しました。 同様の作業が他のオペレーターによって実行されます。



...およびその使用効率を向上させる



インフラストラクチャについて言えば、通信事業者にとって別の重要な方向性は、ネットワークの開発と保守のコストを削減したいという要望です。 たとえば、MegaFonとVimpelcomは、ネットワークインフラストラクチャの一部（アンテナマスト構造）を別のビジネスユニットに割り当てています。 さらに、両社はロシアの複数の事業体でLTEインフラストラクチャを構築しています。



MVNE（モバイル仮想ネットワークイネーブラー）プロジェクトの実装は、携帯電話会社がインフラストラクチャを提供し、それに基づいて仮想オペレーター（MVNO）がそのブランドでサービスを販売できる場合、インフラストラクチャの使用効率を高めるために使用されます。 このモデルは、仮想オペレータからインフラストラクチャの資本コストを排除し、たとえば、以前の加入者の不満など、何らかの理由で大規模なプレーヤーが仕事をすることができなかった視聴者にアクセスできるようにします。



Tele2はMVNOプロジェクトで非常に活発です。 特に、この事業者は、ロステレコム、Sberbank、および他の多くの企業とそのようなプロジェクトをすでに開始しています。 最近、バルセロナで開催されたMobile World Congress（MWC 2018）の枠組みで、中国最大のモバイルオペレーターの1つであるChina Mobileは、来年ロシアで仮想オペレーターを作成する計画を発表しました。



モノのインターネット-IoT / M2M









潜在的な成長ポイントに戻ると、上記のEY予測からわかるように、最も急成長しているものの1つは、モノのインターネット（IoT）セグメントであり、年間約18％の成長です。



IKS-Consultingの予測によると、2020年までに、ロシアのM2M / IoT市場は2,600万枚のSIMカードに成長し、そのうち20％未満のデバイスがB2Cセグメント（主に家電セグメント、「スマートホーム」車）。 また、ほとんどのスマートデバイスは、業界や企業部門、たとえば、ナビゲーションシステムを積極的に使用する輸送会社に必要とされます。



ビデオとは異なり、M2M / IoTソリューションは大量のトラフィックを生成しません。 ここでは他の特性がより重要です。低トラフィック遅延、エンドデバイスの長いバッテリ寿命など。これらは、物ではなく人々にサービスを提供するように設計された従来のセルラー通信技術を提供できません。 IoTアプリケーションを効果的にサポートするには、新しい無線技術とネットワークアーキテクチャが必要です。 5Gコンセプトは、これらの問題を解決することを目的としています。 そのため、LTEの開発の一環として、さまざまなモード、特にeMTCとNB-IoTが開発されています。



ロシアの携帯電話会社は、これらの新しい技術を積極的にテストしています。 ただし、インフラストラクチャコンポーネントに加えて、IoT市場で成功するためには、エンドデバイス、IoTソフトウェアプラットフォーム、その他のプレーヤーの開発者でエコシステムを構築することが重要です。 このためには、インフラストラクチャのオープン性が重要です。これは、とりわけ、上記のSDN / NFVソリューションによって保証されます。

もう1つの分野はモバイルファイナンスです。独自の銀行の開発、ペイメントカード、マイクロローンなどの発行です。メガフォンは、モバイル番号とアカウントに関連付けられた独自のペイメントカードを発売しました。 MTSは独自の銀行を開発しています。 VimpelComはBeeline MasterCardカードを発行します。これにより、世界中での購入代金の支払い、ボーナスの蓄積、通信サービス代金の支払い、パートナー製品の割引の受け取り、手数料なしでの資金の補充と送金が可能になります。
