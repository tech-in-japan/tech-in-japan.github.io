アマゾン地上局の賃貸-衛星からのデータの受信と処理

12月上旬、Amazonは、衛星からの情報を受信および処理する地上局のネットワークの展開を発表しました。 大規模な衛星事業者は、多くの場合、独自の衛星ネットワークを管理するために独自の地上局を構築および運用しています。 これは費用のかかる作業であり、安価なカブサットの急増する所有者にとってコストは耐えられません。











よく知られているAmazon Elastic Compute Cloud（EC2）サービスは、従量課金コンピューティングパワーの費用対効果の高い使用で知られています。 AWS Ground Stationは、衛星地上局でも同じことを行います。 独自の地上局を構築したり、長期リース契約を締結したりする代わりに、AWS地上局を必要に応じて毎分使用し、支払いが行われるようになりました。



通常、クブサットの所有者は自分の衛星へのアクセスが制限されており、ほとんどの場合、彼の唯一の地上局は1日に1回数分で衛星に連絡しますが、現在は緊急の地上局にアクセスして衛星が観測する特別なイベントを処理することができます：深刻な気象災害、自然災害、またはよりポジティブなもの、例えばスポーツイベント。



衛星から定期的にデータを受信したり、衛星経由でコンテンツを配信したりするために地上局にアクセスする必要がある場合は、局を事前に予約して、さらに少ない料金で支払うことができます。



Amazonは2018年に一対のステーションで開始し、2019年半ばまでに12ステーションを展開する予定です。 各地上局は特定のAWSリージョンに関連付けられています。 ステーションが受信したアナログ信号はデータストリームに変換され、Amazon EC2インスタンスに送信され、そこで信号がデータに処理されます。 デジタル表示されたデータは、処理、分析、保存、および送信できます。



  • ストリーミング-データストリームをキャプチャ、処理、および保存するためのAmazon Kinesis Data Streams。
  • 処理-画像分析のためのAmazon Rekognition。 機械学習モデルを作成、トレーニング、デプロイするためのAmazon SageMaker。
  • 分析/レポート-処理されたデータを保存するためのAmazon Redshift。 データクエリ用のAmazon AthenaおよびAmazon QuickSight。
  • ストレージ-データストレージ用のAmazon Simple Storage Service（S3）、およびデータアーカイブと長期ストレージ用のAmazon Glacier。


したがって、地上局から始まり、処理、保管、レポート、配信に至るまでの衛星との作業は、柔軟なインフラストラクチャを使用して、従量制で実行できるようになりました。



AWS Ground Stationの使用方法



AWS Ground Stationを使用するには 、サテライト所有者がサテライトNORAD ID、FCCライセンス情報、およびAWSアカウントIDを持っている必要があります。



開始するには、Ground Station Consoleを開き、[今すぐ連絡先を予約]をクリックします。











最初のステップでは、「連絡先」を予約する必要があります-選択した地上局にデータを送信するために衛星が最適な位置にくる時間を計画するためです。











ステータス（利用可能、スケジュール済み、完了）および時間間隔でフィルタリングできます。











連絡先リストで、要件を満たす連絡先を1つ選択し、[連絡先の予約]をクリックして予約する必要があります。











次のページで、予約をクリックして連絡先を確認する必要があります。











その後、連絡先リストをフィルタリングして、今後のすべての予約を表示できます。











バックアップ後、予約された連絡時間の少なくとも15分前に、Amazon EC2インスタンスがステーションと同じリージョンで動作していることを確認する必要があります。 信号処理を担当するAmazon EC2インスタンスは、Elastic Network Interface（ENI）に接続し、DataDefenderを使用してデータ転送を制御し、qRadioなどのソフトウェアモデムにデータをルーティングしてデータをデジタル化します。



明らかに、Amazonは商用キューブサットに重点を置いており、その所有者はAWS Ground Stationの料金を支払うことができます。 ファン、特に学生チームは、衛星地上局のネットワークを作成するために設計された無料のオープンソースソフトウェアプラットフォームである、無料のアマチュア無線ネットワークSatNOGSの支援と独自の地上局に依拠する必要があります。 学生衛星CubeSailの最近の話は、このような地上局のネットワークに対する需要を示しており、Amazonはこの市場に初めて参入しました。 需要と負荷の増加に伴い、Amazonは追加のステーションとアンテナの構築を計画しています。
