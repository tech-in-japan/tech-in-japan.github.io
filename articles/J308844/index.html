Veeamが新しいソリューションと機能を発表

本日の投稿は、既存の製品の新しいバージョンのレビューと、Veeamが近い将来に幅広い視聴者に提供するソリューションに焦点を当てています。 したがって、今日のプログラムでは：



  • Veeam Backup＆Replication v 9.5の新機能
  • Veeam Agent for Windows（旧Veeam Endpoint Backup）およびVeeam Agent for Linux
  • Veeam Availability Console
  • 新機能：Veeam Backup for Office 365
  • New：Veeam Availability Orchestrator
  • ハイブリッドクラウド向けVeeam Availability Platform
  • そして、Veeam Backup＆Replication v10について少し


詳細については、猫へようこそ。















Veeam Backup＆Replication v 9.5の新機能



2016年の第4四半期には、主力製品の新しいバージョンが予定されています。 今回、エンジニアはソリューションのパフォーマンスとリソースの効率的な使用に特別な注意を払ったため、ほとんどの新製品はUIに表示されず、「内部」に残ります。 ただし、ユーザーは確かに感謝します。



そのため、バージョン9.5は、Windows 2016とともに今年の終わりにリリースが予定されているMicrosoft Resilient File System（ReFS）3.0をサポートします。ブロックの高速クローン作成技術により、実際のバックアップが高速化されます。 バックアップストレージがReFSを使用している場合、Veeam Backup＆Replicationはバックアップ操作を実行するためにはるかに高速になります。 たとえば、1つの完全バックアップと6つの増分バックアップがあり、それらから1つの合成完全バックアップを取得するとします。 増分バックアップファイルに保存された変更済みデータブロックを挿入することにより、最後の完全バックアップから作成されます（詳細については、 この投稿を参照してください）。 この操作は、必要なすべてのデータブロックが上書きされる新しい完全バックアップファイルを再作成することによって実行されます。



したがって、ストレージシステムでのI / O操作の数に関しては、バックアップを結合する操作では、結合されたすべてのバックアップからすべてのデータを読み取り（比較およびコピーのため）、データディスクに1つの新しい合成完全バックアップを書き込む必要があります。 ReFSでは、ReFSレベルで目的のブロックへのファイルポインターを再インストールするだけで、変更されたブロックを挿入できます。フルバックアップファイルを再作成してデータを上書きする必要はありません。 このプロセスでは、増分のみが分析され、更新が必要なブロックのフルコピーファイル内で、増分内のブロックへのポインターがリセットされます。 ブロック自体のデータはプロセスの影響を受けません。 その結果、読み取り/書き込み操作の回数が減少し（NTFSと比較して最大60％）、バックアップウィンドウが数倍も短縮されるため、プロセスが大幅に加速されます。









Veeam Agentsラインの新機能（以前のEndpoint Backup）

  • Linuxプラットフォーム-Veeam Agent for Linux （バージョン1.0、ベータ＃2が利用可能になりました）


新しいバージョンでは、Veeam Agent for Windows（2016年12月にリリース予定）およびVeeam Agent for Linux（2016年11月にリリース予定）に名前が変更されます。 したがって、無料バージョン（無料）、サーバーバージョン（サーバー）、ワークステーション用バージョン（ワークステーション）の3種類のライセンスが利用できる別の製品ラインが形成されます。 それらに関するより詳細な情報は、以下の表に示されています。







一言で言えば-何が面白いでしょう：



  • ローカルキャッシュ-何らかの理由でリポジトリへのバックアップが不可能な場合、Veeam Agentはデータをローカルキャッシュに配置し、後でリポジトリへのアクセスが復元されたときにその内容を同期します。 これは、リポジトリの切断または誤動作の場合のタスクの「失敗」に関する問題の解決に役立つはずです。
  • クラウド接続のサポート-クラウドリポジトリへのバックアップ（ ワークステーションおよびサーバーライセンスの場合）。

Veeam Availability Console



大規模インフラストラクチャおよびサービスプロバイダーのIT管理者の作業を促進するために、Veeamは新しいコンソールであるVeeam Availability Consoleを提供しています。 彼女は、管理者にインフラストラクチャに展開されているすべてのVeeamソリューションの全体像を提供することができ、前述のバックアップ、レプリカ、およびVeeamエージェントを管理する便利なツールになるとともに、便利な監視および課金機能を提供することを約束します。



たとえば、大規模なインフラストラクチャでの構成プロセス（必要な設定を含むXMLファイルをエージェントに発行する）は、時間がかかり、時間がかかる場合があります。 したがって、エージェントのコピーを作成して事前構成し、たとえば、販売スタッフのすべてのラップトップにインストールするという考え方は、非常に合理的です。 Veeam Agentを物理マシンのバックアップに展開するには、CSVファイル、Active Directoryグループポリシーを使用するか、IPアドレス範囲をスキャンします。 ここに、部門ごとにリポジトリをカスタマイズする機能を追加すると、写真がさらに魅力的になります。 これらの機能はすべて、新しいコンソールのリリースで期待されています。





Veeam Backup for Office 365



今年の終わりまでに、新しい製品Veeam Backup for Office 365のリリースも計画されています。最初のバージョンでは、Exchange Onlineからオフラインリポジトリにメールがバックアップされます。 多くの組織では、メールはすでにクラウドからローカルストレージにアーカイブされています。また、Veeamは、メールボックス所有者のカレンダーや連絡先リストなどのオブジェクトをきめ細かく復元する機能も追加します。



Office 365は、Exchange Webサービスを介してアクセスされます。 メールをアーカイブする組織を選択します。 次にバックアップタスクを作成します。このタスクでは、アーカイブするメールボックス（すべてまたは特定のメールボックス）、データをアーカイブに保存する期間、およびタスクを実行するスケジュールに従って指定します。 リポジトリの内容は、既におなじみのVeeam Explorer for Microsoft Exchange（ソリューションに含まれています）を使用して表示し、必要なオブジェクトを選択して、サポートされている方法（メールで送信、PSTにエクスポート、元の場所に復元）を使用して復元できます。









Veeam Availability Orchestrator



オーケストラは何をすべきですか？ もちろん、さまざまなパフォーマーの作品を整理します。 また、Veeam Availability Orchestratorは、テストと文書化を行い、災害復旧インフラストラクチャ計画のためにこれをすべて行います。 つまり、災害復旧（DR）の責任者は次のことができるようになります。



  • バックアップとレプリカを使用するための処方計画を作成します。たとえば、レプリカとの間で切り替えるポイントを規定します。 異なるサイトでレプリカの動的グループを作成して、それらを重要なアプリケーションの自動災害復旧に使用することが可能になります。
  • DR計画テストの自動化-テストを通じて新しいバックアップとレプリカを実行し、適切なタイミングで確実に切り替えられるようにします。
  • DR文書に処方計画とテスト結果を含めます。 これにより、最新のデータに基づいて定期的にDR文書を更新できるため、文書作成の責任者の多くの頭痛を和らげることができます。 これは、あらゆる種類の規制法のDRコンプライアンスを確保する人々にとって大きな助けになります。


Veeam Availability Orchestratorのベータリリースは2016年9月に予定されており、2017年第1四半期にリリースされます。



ハイブリッドクラウドデータの可用性-ハイブリッドクラウドのVeeam可用性プラットフォーム



この開発の方向が、新しいソリューションの開発と既存のソリューションの統合の基礎となるのです（下の図を参照）。 統合には、サービスプロバイダーのプライベートクラウドと、Microsoft、Amazon、Googleなどのパブリッククラウドの両方が含まれます。 目標は、ユーザーがローカル、リモート、またはクラウドのリソースとインフラストラクチャのどちらで作業しているかに関係なく、ユーザーのデータの可用性を確保することです。
