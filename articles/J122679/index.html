マイクロソフトプレナリーレポート







レポートの主なトピックは、ユーザーのコンピューターにマルチコアCPUとGPUで構成される巨大な計算能力が登場した、コンピューティングの進化と現在の開発状況でした。 マイクロソフトは、開発の未来は異種リソースの効率的な使用にあると確信しています。 同時に、Sutter氏によると、マルチコアCPU、GPGPU、クラウドコンピューティングなど、さまざまなトレンドを個別に考慮する必要はありません。 唯一の傾向があります-異種並列コンピューティング、つまり、その種類に関係なく、プラットフォームで使用可能なすべてのリソースの消費：電話、タブレット、ラップトップ、コンピューター、またはクラウド。







Sutterは、ハードウェアの現状と現在のプログラミング言語とモデルについて多くのことを語りました。







その後、新しいC ++ AMP異種アプリケーション開発プラットフォームについて大きな発表が行われました。 私はすでにこの発表について熱い追及で書いています。 要するに、C ++ AMPは、利用可能なすべてのハードウェア機能（APU、CPU、GPU）を均一に使用できる異種環境で、C ++言語の完全な仕事の機会を創出するマイクロソフトのオープンな取り組みです。



C ++ AMPは、C ++言語（関数、データ型）の小規模な拡張セットであり、C ++開発者が特別な技術を長時間研究することなく、並列異種プログラミングの能力を使用できるようにします。 C ++ AMPプラットフォームは、新しいC ++標準に基づいており、ラムダ式を使用します。 Microsoft Visual Studio開発環境の次のバージョンに含まれる予定です。







全体報告書は、C ++ AMPの実用的な応用を実証しました。 デモンストレーションのために、粒子同士の相互作用のシミュレーションを使用しました。 C ++ AMPを使用すると、優れた結果が実証されたため、ディスクリートグラフィックスカードを搭載した通常のラップトップでは、数十ギガフロップスの計算能力が得られました。 2枚のビデオカードを搭載したより強力なデスクトップシステムにより、約815ギガフロップスという非常に大きなパワーで同じコードを取得できました。











壮大なデモンストレーションの後、全体講演では、新しいC ++ AMPプラットフォームについての講演が続けられました。 ライブラリを作成する主な理由は次のとおりです。
  • アプリケーションアプリケーションを作成するためのより一般的な言語としてのC ++（Cではなく）。
  • ヘテロジニアスコンピューティングが主流となり、すべての人がアクセスできるようになります。
  • 入力を容易にするための最小限の言語拡張。
  • 移植性。1つのコードが、起動されたプラットフォームで利用可能なすべてのリソースを消費します。
  • 将来の目標であるこのプラットフォームは、将来の開発者がクラウドの何億ものGPUコンピューティングユニットを利用できるようになるときに、未来とクラウドコンピューティングを念頭に置いて設計されています。
  • オープン仕様。




さらに、新しいプラットフォームは次世代のVisual Studioの一部となり、プラットフォームはオープン仕様でリリースされ、Microsoftはこのプラットフォームを他のベンダーに実装する努力をするという発表が行われました。



このプレナリーレポートのもう1つの主要な発表は、Visual Studioの次のバージョンでのGPUコードの編集、デバッグ、およびプロファイリングツールの発表でした。







これで、Microsoftの全体報告書は終わりです。 プレス向けには、Q＆Aセッションが続き、ハーブサッターが特定の質問に答えました。 C ++ AMPプラットフォームの実装、または.NETプラットフォームのマネージコードとの類似性について興味がある質問をしました。 彼は、この点に関して会社がこれまで発表することは何もないという回答を受け取りました。 また、マネージコードでC ++ AMPを使用すると、たとえばP / Invokeを介して、アンマネージコードと同様に使用できます。



最も関心のある開発者向けに、Microsoftは秘密保持契約に署名することでアクセスできる新しい開発ツールのプライベートデモを開催しました。 このデモンストレーションでは、誰もが新しいツールを実際に試して、フィードバックを残し、その印象をマイクロソフトの開発者と共有することができました。
