AMP：電子メールの未来



ロシアの開発者がecwid.ruからAMPテクノロジーを使用して作成した動的な電子メール







AMPプロジェクトは、ネットワーク上のユーザーエクスペリエンスを改善するために考案されました。これは、Web上で発生した電子メールを操作することを意味します。 私たちのほとんどにとって、電子メール機能は最初からあまり変わっていません（明らかに、ほとんどの人はこの外観を見つけられませんでした）。 AMPの本質は速度とセキュリティを確保することであるため、電子メール用のAMPを開発しないことは不可能でした。 メール内のJavaScriptは良いアイデアのように思えますが、AMPフレームワークのおかげで、ユーザーはデータのセキュリティを心配することなく、リアルタイムで電子メールを操作できます。







「どう？」 -聞いた？ カットの下で答える







AMPのおかげで、ユーザーはメールとやり取りするまったく新しい機会を得ることができます。招待状への返信、アンケートへの記入、カタログの閲覧、手紙の本文へのコメントの直接投稿が可能になりました。 さらに、今では手紙の助けを借りて、トピック内の新しいコメントか最新の推奨事項かにかかわらず、更新を追跡できます。







最大のメールサービスのメールのAMPテクノロジー



メールのAMPテクノロジーは、Gmail、Mail.Ru、Yahoo！など、世界最大のメールサービスを使用し始めました。 メールとOutlook.com。 この形式のサポートを開始すると、送信者はAMPメールへのリーチを拡大します。 Mail.ruには独自のエディターもありますが、それについては後で詳しく説明します。







当初、開発者向けのバージョンは最もアクティブなユーザーのみが使用できましたが、現在ではすべての受信者と関心のある送信者が電子メールのAMPテクノロジーを利用できます。 Googleの公式 ブログでいくつかの良い例を見つけることができます。 さらに、追加情報は他のメールサービスのページにあります。







  • Mail.ru
  • Yahoo!メール
  • Outlook.com
  • また、 EcwidとOYO Roomsのブログでも。


ユーザーに高品質のメールを提供するよう努めており、メールのAMPテクノロジーをサポートする最初のサービスの1つになりました。 これにより、高速で効率的な電子メールの条件を作成できます。MarcelBecker、Yahoo！プロダクトマネジメントディレクター メール

メールサービスプロバイダーであり、メールのAMPテクノロジーサポートを追加する場合は、 こちらの詳細をご覧ください 。







今すぐAMPレターを作成する



AMPレターを作成するためのテストサイトは複数あります。 すでに述べたように、Mail.ruはPlaygroundを提供します。 さて、amp.devのAMPプロジェクト自体のテストサイトについて詳しく説明します。 マークアップを編集し、変更をリアルタイムで追跡できます。 サポートされている関数と構文の詳細については、 リンクを参照してください 。 しかし、このプロセスには複雑なものはありません。 







<!doctype html> <html amp4email> <head> <meta charset="utf-8"> <script async src="https://cdn.ampproject.org/v0.js"></script> <style amp4email-boilerplate>body{visibility:hidden}</style> </head> <body> Hello, AMP4EMAIL world. </body> </html>


ご覧のとおり、これは完全に通常のHTMLですが、 amp4emailを htmlタグに追加する必要があります。もちろん、AMPライブラリ自体も忘れないでください- <script async src = " https://cdn.ampproject.org/v0.js "> </ script> 。 そして、すべてが簡単です。 AMPに既に精通している場合は、 amp-imgやamp-carouselなど、お気に入りのタグをすべて満たします。 そして、このフレームワークがあなたにとって新しいか、あなたの知識が錆びているなら、あなたはここにいます。







「しかし、AMPメールの約束された塩はすべて動的であり、どこにあるのでしょうか？」 -お願いします。 amp-bindでは、他の場所で！ ここではすべてのドキュメントを書き直すことはしませんが、質問がある場合はコメントをお待ちしています！







サードパーティの会社が電子メールの送信に従事している場合、AMPをサポートしていることを確認する必要があります。 次のサービスは、AMP仕様を既にサポートしているか、このテクノロジーの実装に積極的に取り組んでいます。







  • SparkPost-メールサービスおよび分析プラットフォーム
  • Litmus-メールのマーケティングと設計のためのツールのセット
  • Twilio Sendgridは、トランザクションメールやマーケティングメールを送信するためのプラットフォームであり、80,000以上の企業で使用されています。
  • Amazon SESおよびAmazon Pinpoint-クラウドベースのEメールおよび分析サービス
  • そして、この技術をサポートするために働いている他のサービス。 詳細については、メールサービスにお問い合わせください。


AMPメールは、 新しいMIME標準のおかげで、現在のメールエコシステムと互換性があります。 したがって、メールクライアントがAMP文字をまだサポートしていない場合、そのような文字はHTML形式で表示されます。 仕様の詳細と手順については、 こちらをご覧ください 。 ユーザーと開発者の両方からのフィードバックを本当に楽しみにしています。







新しいAMPワーキンググループ



ユーザーの利益のためにAMPが成長し続けるには、あなたの助けが必要です。 そのため、 AMP管理モデルのフレームワーク内で、送信者およびメールサービスからフィードバックを受け取るためのチャネルとして、電子メールのAMPテクノロジを開発するための特別なワーキンググループが作成されました。 このグループには2つのタスクがあります。







  1. 郵便サービス間のAMPメールに関する知識の交換を促進し、さまざまな電子メールサービスの互換性を確保する。
  2. 技術を開発および改善するために、ユーザーおよびコミュニティのフィードバックを送信者およびメールサービスに送信するためのチャネルを編成します。


メールサービスを代表する、メールを送信する、またはAMPテクノロジー開発チームに参加したい場合は、 ampproject / wg — amp4emailリポジトリページで問題の議論に参加するか、Slackの＃wg — amp4emailチャネルに参加できます 。







オープンソースソフトウェア、複数のベンダーからのサポート、段階的な移行戦略、全員に耳を傾けるオープンワークグループは、誰もがアクセスできる新世代の電子メールを作成するのに役立ちます。
