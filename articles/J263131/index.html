ClojureのWebアプリケーション。 パート2

こんにちは、親愛なるユーザーとHabrの訪問者。 Clojure Webアプリケーションの最初の記事では、 Clojure Webプロジェクトを構築するための基本的なツールとライブラリについて検討しました。 すなわち、ラインニンゲン、リング、コンポジュレ、モンガー、バディ、セルマー。 ここでは、実際のアプリケーションに焦点を当てます。



自己開発用に構成されている場合は、記事の読み取りをバイパスするコードを理解することにより、ページの最後でGithubへのプロジェクトリンクを求めます。



はじめに


それでは、順番に始めましょう。 もっと面白くするために、私は多かれ少なかれ記事の焦点を当てることにしました。 今日は、Clojureで簡単なWebアプリケーションを作成します。これにより、メモが管理されます。 Leiningen、Clojure、MongoDBが既にインストールされていると想定しています（有効にすることを忘れないでください）。 コンテンツの大部分は、コード内のコメントに直接含まれており、便宜上、ネタバレの中に隠されています。





IDE


ClojureにはさまざまなエディターとIDEがありますが、この記事では長所と短所を説明せず、一度も時間がありません。 好みはそれぞれ異なりますが、使用するものはあなた次第です。 私はClojureScriptで書かれており、それに完全に満足しているLightTableを使用します。多くのモジュールがあります。ClojureとClojureScriptで開発を開始するために必要なものはすべて揃っており、ParEditモジュールがあります。 replによってプロジェクトにリモートまたはローカルで接続するために、何も構成する必要はありません。 LightTableのreplとの相互作用は非常に独特で、私の主観的な意見では非常に便利です-関数を呼び出して、ライブモード（たとえば、Emacsや他のすべてのIDE）で別のウィンドウに結果を表示したり、コードで同じことを直接実行したり、変換したりできます式の最初または最後のブラケットにカーソルを置き、cmd + Enter（MacOS）を押します。LightTableがrepl接続を作成してこの式をコンパイルした後、コンパイルされた関数の名前または下の変数行を入力し、その結果をコードで直接表示するだけです 。





バックエンド



プロジェクト


まず、プロジェクトを作成しましょう： $ lein new compojure notes



これで、アプリケーションが空白のカタログができました。 そこに行き、project.cljファイルをエディターで開きましょう。 使用するライブラリへの依存関係を追加する必要があります。



project.clj 
(defproject notes "0.1.0-SNAPSHOT" :description " " :min-lein-version "2.0.0" :dependencies [; -,  Clojure   ;   [org.clojure/clojure "1.6.0"] ;   GET  POST  [compojure "1.3.1"] ;  (middleware)   ;  [ring/ring-defaults "0.1.5"] ;  [selmer "0.8.2"] ;  Monger [com.novemberain/monger "2.0.1"] ;    [clojure.joda-time "0.6.0"]] ;  -     ;  ,     ; Ring'     - Jetty :plugins [[lein-ring "0.8.13"]] ;    Ring  ;   app  ;        :ring {:handler notes.handler/app} :profiles {:dev {:dependencies [[javax.servlet/servlet-api "2.5"] [ring-mock "0.1.5"]]}})






プロジェクトを開始すると、Leiningenは指定されたすべての依存関係を自動的にインストールします。必要なのは、それらをproject.cljで指定し、インターネットに接続することだけです。 次に、アプリケーションのサーバー側の作成に進みましょう。 表示機能を配置し、データベース、ハンドラー、およびルートを別々のファイルに保存して、名前の競合や頭痛の種の不便がないようにする方が論理的ですが、これはあなたが好きなことです。





ハンドラー（メインハンドラー）


handler.cljから始めましょう。Leinは既に作成してくれており、 / src / notesディレクトリにあります（Clojureコードもすべて含まれています）。 これはアプリケーションの重要な部分です。アプリケーション変数と、HTTPの基本的なミドルウェア（要求および応答ラッパー）を含むapp変数が含まれています。 アプリケーションのルートを名前空間に追加すると、次のコードが取得されます。



handler.clj 
 (ns notes.handler (:require ;   [notes.routes :refer [notes-routes]] ;   middleware [ring.middleware.defaults :refer [wrap-defaults site-defaults]])) ;    middleware (def app (wrap-defaults notes-routes site-defaults))








ルート


次に、routes.cljファイルを作成し、その中にルートを配置します。これは、指定されたURIでGETおよびPOSTメソッドによってプロンプトが表示されると、フォームハンドラーの機能を呼び出し、ページを表示します。 アプリケーションのこの部分では、Compojure APIを使用します。 気になる人のためにコードの膨大な部分をすぐに謝罪しますが、彼らの仕事の論理を理解しやすくするために、コメントをたくさん追加しました：

DB（MongoDBとの相互作用）


アプリケーションの非常に興味深い部分であるMongerライブラリは、その構築に役立ちます。 db.cljファイルを作成し、MongoDBと対話するための関数を保存します。 もちろん、ルートとコントローラーでMonger関数を直接呼び出すこともできますが、このために、デバッグ、拡張、多数の重複コードで報復を受け、最終的なコード行数を増やします。 Mongerでは、DSLクエリ（リレーショナルDBMS用の優れたsqlcormaライブラリがあります）を使用してMongoDBにクエリを作成することもできます。これは複雑なクエリには非常に便利ですが、この記事では説明しません。 db.cljに関数を追加しましょう：

ビュー（HTMLテンプレートの表示）


views.cljファイルには、HTMLテンプレートを表示してデータを送信する関数を配置します。 Djangoテンプレートのデータ表現システムに触発されたSelmerライブラリは、この問題に役立ちます。 Selmerでは、テンプレート自体のデータ（タグ）を処理するためのフィルター（関数）を追加することもでき、ユーザー自身に柔軟な設定を提供します。 ページ表示関数を書きましょう：

HTMLテンプレート


アプリケーションはほぼ準備ができており、データを表示するHTMLファイルを作成します。 / resourcesディレクトリは、静的ファイルをホストするために必要です。 .jarまたは.warファイルでアプリケーションをコンパイルした後でも、その中のファイルを置き換えることができます。 次回の記事では、CSSテーブルを追加します。 それまでの間、HTMLファイルを配置するテンプレートディレクトリを作成します。







まず、すべてのテンプレートの基本ファイルを作成します。これには、すべてのページのメインマークアップと、残りのセクションのマークアップが配置されるコンテンツブロックが含まれます。 始めましょう：



base.html 
 <!DOCTYPE html> <html> <head> <META http-equiv="Content-Type" content="text/html; charset=UTF-8"> <title>{{title}}</title> </head> <body> <ul> <li> {% ifequal page "index.html" %} <strong> </strong> {% else %} <a href="/"> </a> {% endifequal %} </li> <li> {% ifequal page "create.html" %} <strong> </strong> {% else %} <a href="/create"> </a> {% endifequal %} </li> </ul> {% block content %} {% endblock %} </body> </html>








次に、メモを作成するためのフォームでテンプレートを作成します。 その中に、アプリケーションのすべてのフォームと同様に、view.cljで作成したタグ{％csrf-field％}を追加する必要があります。追加しないと、フォームを送信すると、エラーInvalid anti-forgery tokenが表示されます。 続行：



create.html 
 {% extends "base.html" %} {% block content %} <h1> </h1> <form action="POST"> {% csrf-field %} <p> <label></label><br> <input type="text" name="title" placeholder=""> </p> <p> <label></label><br> <textarea name="text"></textarea> </p> <input type="submit" value=""> </form> {% endblock %}








すでにルート、ビュー、およびノー​​ト編集エディターがあります。これらすべてのフォームでテンプレートを作成しましょう。



edit.html 
 {% extends "base.html" %} {% block content %} <h1> </h1> <form method="POST"> {% csrf-field %} <input type="hidden" name="id" value="{{note._id}}"> <p> <label></label><br> <input type="text" name="title" value="{{note.title}}"> </p> <p> <label></label><br> <textarea name="text">{{note.text}}</textarea> </p> <input type="submit" value=""> </form> {% endblock %}








次に、ノートを表示するためのテンプレートを作成します。小さなタグ内の注意深いリーダーは、データの奇妙な表現を見ることになります。これは、view.cljで作成されたフィルター以外のものです。 Selmerでフィルターを呼び出すには、通常{{variable | filter}}と記述します。 次に、コード自体：



note.html 
 {% extends "base.html" %} {% block content %} <h1>{{note.title}}</h1> <small>{{note.created|format-datetime}}</small> <p>{{note.text}}</p> {% endblock %}








そして最後に、メモのリストが表示されるメインページ：



note.html 
 {% extends "base.html" %} {% block content %} <h1></h1> {% if notes %} <ul> {% for note in notes %} <li> <h4><a href="/note/{{note._id}}">{{note.title}}</a></h4> <small>{{note.created|format-datetime}}</small> <hr> <a href="/edit/{{note._id}}"></a> | <a href="/delete/{{note._id}}"></a> </li> {% endfor %} </ul> {% else %} <strong>  </strong> {% endif %} {% endblock %}

これでアプリケーションの準備が整いました。もちろん、非常に重要なステップを見逃しました-replで関数をテストし、結果を表示しますが、次の記事では詳細に説明します。



実行しましょう： $ lein ring server



このコマンドは、lein runと同様に、すべての依存関係をインストールし、localhost：3000のコアJetty Webサーバーでアプリケーションを起動します。 これまでのところ、アプリケーションを.jarまたは.warファイルにコンパイルしたり、lein runで実行したりすることはできません。
