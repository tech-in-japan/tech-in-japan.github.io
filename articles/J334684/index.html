ジュニパーノードスライシングとユニバーサルシャーシ

階段の真上にあるものを見ることは、電気通信業界のどのトレンドが固まり、将来の発展の基盤になり、どのトレンドが跡形もなく消えていくかを理解するのと同じくらい困難です。 少し前まで、ジュニパーネットワークスはCloud-Grade Networking設計の一部として、製品の新しい仮想化メカニズムの登場を発表し、ルーティングおよびスイッチングアプリケーション用のユニバーサルシャーシの概念も導入しました。 ニュースは、実用化の観点から重要であり、将来について幸運を語る機会として興味深いものです。



まず、マーケティング担当者、次にエンジニアが、最新のルーターをマルチサービスと呼び始めました。 1つのシャーシがピアツーピアジョイントでルーティングタスクを解決し、ブロードバンドサブスクライバーを終了し、MPLSトランジットトラフィックを送信し、データセンターオーバーレイからパケットをプルし、さらにアドレスおよびプロトコル変換を行うことができるのは素晴らしいことです。 設備の汎用性と削減は、Capex / Opexのカウントが得意なスーツを着た人にも、ラックスペースと必要なUPS容量を計算し、夜間にアラートを監視し、在庫を取り、この機器を設定または保守する人にも、 1つのエンティティの管理が10より簡単だからです。 すべては問題ありませんが、関数の数が増えると、それらの間に内部矛盾が現れます。 理想的なソフトウェアではないため、夕食を消し、夕食を調理し、冬用タイヤを保管するマルチアプライアンスを想像してください。夕食はおいしいものではない場合があります。少量の粉末洗剤が誤って材料に侵入し、オーブンの近くのためにタイヤが変形する可能性があるためです。 私の実務では、クライアント、ルーターを使用しましたが、それらはライフサイクルを通じてモノリスに変わりました。 「これが私たちのすべて」であるため、それを分解することはもはやできません。また、非理想的なソフトウェアのために1つの機能が別の機能の実行を妨げるため、操作がますます難しくなっています。



IT部門の同僚は、同様の問題を長い間解決してきました。サーバー管理者は仮想化とコンテナーをますます使用し、開発者はエンティティを積極的に分解し、マイクロサービスを作成しています。 オデッサの友人が言ったように、最近まで、ネットワークエンジニアは仮想化の余裕しかありませんでした-貧しい人々のための仮想化は、個別のルーティングテーブル、管理コンテキスト、または論理ルーターの形で。 少なくともネットワークオペレーティングシステムのプロセスレベルでタスクを共有するため、これにより安定性と利便性が向上しますが、モノリシックの問題はグローバルに解決されません。 最後に、ジュニパーネットワークスのサービスプロバイダー製品マーケティング管理部長のDonyel Jones-Williams氏は、次のように述べています。ハイパーバイザーがサーバーに対して行ったのと同じことをルーターに対して行っています。 私の意見では、Junis Networksのいわゆる仮想化システムであるJunos Node Slicingは、REではなく別のサーバーでオペレーティングシステムのインスタンスを実行することを意味するため、さらに興味深い結果になりました。







図からわかるように、GNF（ゲストネットワーク機能）-Junosのインスタンスは、独立した仮想マシンとして実行され、FPCスロットに配置されたリソースのセットのみを自由に使用できます。 各GNFはFPCセットとともに外部から認識され、完全に独立したルーターとして機能しますが、GNF間のトラフィックはファクトリーを介して送信できます。 このアプローチは、ルーターの機能間の内部矛盾を排除し、システム全体の安定性を高め、さらに、組織単位間で管理制御ドメインを分離する「自然な」能力を備えています。 別のイラストをお見せしましょう。彼らは非常に美しく描き始めました。







PTX10008、PTX10016、QFX10008、QFX10016、およびMXのインターフェースカードを使用できるユニバーサルシャーシの差し迫った外観は、ジュニパーネットワークスのプレスリリースで発表されたもので、1つだけ質問があります。 ジュニパーネットワークスは、すでに統一の分野でいくつかの基盤を持っています。たとえば、MX240-MX960ラインカードコンセントレータの一部は、トップMX2010およびMX2020で長時間使用できます。 現在のタスクはより深刻に見えます。1つのシャーシで、ネットワークのすべてのセグメント（データセンター、コア、ボーダー）の作業の特徴を考慮して設計されたカードを組み合わせる必要があるためです。 これがどのように行われるかはまだ発表されていませんが、Donyel Jones-Williamsによると、新しいプラットフォームはすでに大規模な顧客のネットワークでテスト運用されています。



それでは、ジュニパーネットワークスをさらに近づけようとしている未来は何でしょうか？ 製造業者は機器だけでなく、管理の方向性と運用の柔軟性にも重点を置いています。 さまざまな見積もりによると、最終製品のコストはハードウェアの詰め込みによってわずか40％しか決定されません。ソフトウェア、統合ツール、および自動化の開発とほぼ同じコストです。 Doc Brownが述べたように、将来はまだ書かれていません。
