GPUコンピューティング用のMicrosoftの新しいC ++ AMPテクノロジー

新しいブログ投稿によると、Microsoft DirectXプラットフォーム上に構築されたC ++ Accelerated Massive Parallelism（C ++ AMP）テクノロジー。 マイクロソフトは、このテクノロジを次のVisual C ++コンパイラの一部とし、コード名がVisual Studio vNextであるVisual Studioの次のバージョンに完全に統合する予定です（Visual Studio 2012という名前で発売される可能性が高い）。



Microsoftは本日、AMD Fusion Developer SummitでC ++ AMPを発表しました。 この会議で、 ハーブサッターは、GPUでC ++機能を使用するための技術を作成するマイクロソフトの取り組みについて聴衆に語りました。 目標は、開発者がプロ​​グラムを再コンパイルして、CPUとGPUプロセッサ間でコンピューティングを分散できるようにすることです。 マイクロソフトの最終目標は、C ++開発者がマルチコアシステムとクラウドシステムに同じモデルを使用できるようにすることです。



Microsoftは、C ++ AMP仕様を名前のない標準化組織に送信して、テクノロジがどのコンパイラでも実装できる「オープンな仕様」になるように計画しています。 このテクノロジーをWindowsおよびWindows以外のプラットフォームのコンパイラーで使用可能にする計画があります。
