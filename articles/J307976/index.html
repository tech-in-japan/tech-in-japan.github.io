Linuxで壊れたVPSサーバーを修正する方法





Ruvdsでは、ホスティング上で、オペレーティングシステムCentos、Debian、 Ubuntuサーバーを備えたLinux上のサーバーをレンタルする可能性を開始しました。



そのため、「Linuxで壊れたVPSサーバーを修正する方法」コンテストを発表しています。 前回の優勝者は3時間で管理したため、コンテストに参加してみてください。 今回は、タスクを少し複雑にしました。



コンテスト規約

アドレスhttp：// your_vps_ipにアクセスするときに、この目的のために準備されたサイトが表示されるように、nginx Webサーバーを構成する必要があります。



必要条件

このサイトのファイルは、ディレクトリ/ home / administrator / contest_filesに配置されています。

現在のパーティション/ dev / sda3を縮小して作成された、サイズが1 GB以上の別のパーティション（xfsファイルシステム）に移動する必要があります。



何をすべきかは禁止されています

  • ネットワーク経由でOSブートを使用する
  • OSの現在のバージョンのインストール時に使用されなかった、またはサーバー上になかった新しいファイルをサーバーにアップロードする
  • 以前にインストールされていないパッケージはインストールできません。
  • ファイアウォールを無効にすることはできません
  • 他のパーティション（/ dev / sda1、/ dev / sda2）のサイズを縮小することはできません
  • ファイルシステムを損傷することはできません
  • サーバーを他の目的に使用する




始め方

「Linuxのコンペティション」とマークされたsupport@ruvds.comにリクエストを送信し、 ruvds.comでの登録に使用したメールを示します。



タスクが完了したとみなされるとき

このサイトはhttp：// your_vps_ipで利用でき、問題を解決するためのアクションに関する詳細なレポートを提供しています。



勝者は、タスクに費やされた時間、つまり競合サーバーの作成に関するレターと、問題の解決策に関する参加者からのレターの差によって決まります。
