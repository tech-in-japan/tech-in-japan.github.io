SAP Cloud Platformでブロックチェーンからアプリケーションを起動する

SAP Cloud Platformは、PaaSまたはPlatform as a Serviceモデルで動作するクラウドプラットフォームです。 これにより、企業は既存のSAP製品の独自のアプリケーション、拡張機能、アドオンを開発したり、クラウドまたはオンプレミスソリューション間の統合を通じて企業内にハイブリッドITランドスケープを作成したりできます。



昨年の秋、SAPクラウドプラットフォームをロシアのデータセンターに移動し、国内の顧客とユーザー向けのプラットフォームの開発と使用を促進しました。



この記事では、年金基金向けのパイロットブロックチェーンアプリケーションだけでなく、SAP Cloud Platformでブロックチェーンアプリケーションをすばやく起動する方法について説明します。





SAP Cloud Platformのアーキテクチャ



SAP Cloud Platformが解決する2つの主なタスクは、ITソリューションの開発と運用（ホスティング、監視、スケーリングなど）です。



SAP Cloud Platformのサービスには、いくつかのカテゴリがあります。



-データ処理とストレージのサービス：インメモリ、SQL、NoSQL、ビッグデータサービス

-アプリケーション開発のためのサービス：Java、XS /​​ JS（SAP HANAのプログラミング用のSAP内部言語）、およびいわゆるBYOLモデル（「独自の言語を持ち込む」-プログラミング言語（PHP、Pythonなど）の使用）

-ビジネスアプリケーションを作成するための既製の技術サービスのセット：ブロックチェーンサービス、統合バス、モノのインターネットなど

-既製のビジネスサービスのセット：ショッピングバスケット、ロイヤルティプログラムなど



既製のサービスを使用すると、企業はビジネスアプリケーションをすばやく開発したり、他の企業から既製のソリューションを購入して、さらに統合したりできます。



SAP Cloud Platform Blockchain Servicesは、既製のプラットフォームサービスの最後のカテゴリに属します。







どのタスクを解決しますか：



  1. SAP Cloud Platformに基づくさまざまなブロックチェーンテクノロジーの迅速な展開。 マルチチェーンおよびHyperledger Fabricテクノロジーがサービスで利用可能になりました。 マルチチェーンは、ビットコインの原理に基づいて開発されたテクノロジーですが、プライベートまたは企業のブロックチェーンネットワークの作成に使用されます。



    Hyperledger Fabricは、招待によりアクセスできるモジュラーブロックチェーンネットワークを作成するためのテクノロジーです。 Hyperledger Fabricの機能には、スマートコントラクトのサポートがあります。
  2. SAP Cloud Platform内には、ノードの管理、トランザクションの追跡、スマートコントラクトの操作などのサービスを含む、選択したブロックチェーンテクノロジーとの作業を促進するサービスもあります。
  3. 他のSAPビジネスソリューションとの統合をサポートします。
  4. 追加サービス-分析、アプリケーション管理など。


SAP Cloud Platform Blockchain Servicesの主要コンポーネント：



  1. SCPのブロックチェーンサービス（Cloud Foundry内で動作）
  2. マルチチェーンノードとHyperledger Fabricノードのプロビジョニング-ノードの管理用
  3. ブロックチェーンコックピット-ブロックチェーンコントロールパネル
  4. Blockchain Explorer-トランザクション情報にすばやくアクセスするためのWebアプリケーション
  5. REST APIを介したチェーンコードの展開と呼び出し-アプリケーション機能にすばやくアクセスするための追加インターフェイス
  6. ブロックチェーンの証明書とキーのセキュアストア-ブロックチェーンに関連付けられた証明書とノードのセキュアストア


以下は、Hyperledgerを使用して作成されたアプリケーションのアーキテクチャの例です。







Hyperledgerは、参加者が個別に受け取る非パブリックブロックチェーンネットワークを実装するためのプラットフォームです。



たとえば、参加者は、Hyperledgerに基づくブロックチェーンソリューションの使用に同意するコンソーシアムの一部です。 すべての参加者には、分散データベースの完全なコピーを含む完全なノードがあります。 したがって、SAP Cloud Platformを含むすべてのサーバーおよびサイトで完全なノードをホストできます。 実際、クライアントはこのノード上に追加のアプリケーションをデプロイできます。これにより、Hyperledgerノードの動作を保証するさまざまな分析と動作が提供されます。



内部から見たSCPブロックチェーンサービス：







アプリケーションの最初の画面には、アプリケーションの説明、さまざまなサービス計画の選択に関する情報、およびサービスの使用に関する参照データが含まれています。





次の画面は、選択した料金に応じて、サービスの異なるインスタンスの作成です。 各インスタンスには、インタラクティブなコントロールパネルがあります。







ブロックチェーンサービスコントロールパネルのある画面。 現在のステータス、トランザクション、ブロック、およびその他の関連する詳細に関する情報が含まれています。







スマートコントラクト画面



このインターフェイスを使用して、Hyperledgerノードにスマートコントラクトを展開し、テストできます。 スマートコントラクトを操作するためのさまざまな方法（呼び出しとクエリ）があります。 Invokeメソッドは、本質的にブロックチェーンネットワークを変更することを伴います-特定のトランザクションをブロックチェーンに書き込みます。 Queryメソッドは、ブロックチェーンネットワーク全体を変更することなく、基本的に特定のローカルノードでトランザクションを実行する情報を読み取る方法です。



ブロックチェーンアプリケーションの作成自体は珍しいことではありません。分散レジストリに追加のサービスを追加すると、興味深いシナリオが現れ始めます。 以下の記事では、そのような例の1つ、個人年金資本管理を紹介します。



ブロックチェーンアプリケーションシナリオ：個人退職資本管理



SAPには、ビジネスプロセスでブロックチェーンテクノロジーを使用するための約30のシナリオがあります。 それらの1つは、SAP CISチームと実行開発会社のパートナーによって発明され、ロシアでパイロットとして実装されました。



年金にはいくつかの部分があります-強制、自発的、年金資本（政府の計画で）。 いくつかの計画によると、年金資本の拠出金の一部は0〜6パーセントであり、従業員はこの値を独自に決定できます。



従来のアプローチでは、何らかの方式の仲介者がこのスキームに必ず存在します-州の年金基金、州の銀行、または中央集権化されたレジストリを担当する他の組織。



しかし、問題があります-年金基金業界は非常に競争が激しく、公正かつ不公平な競争があります。 これは、信頼できない環境の典型的な例です。



これらの条件の下で、このシナリオの実装に平等に参加するブロックチェーンネットワークを構築するというアイデアが生まれました。







スクリプトの主な開発者は実行でした。 SAPの同僚とともに、ビジネスプロセスを分解し、ブロックチェーンを使用して解決できる部分を強調表示しようとしました。



政府の計画によれば、従業員は、給与の一定の割合を年金基金に支払う予定であることを雇用主に知らせることができます。 同時に、彼はいつでも警告を出さずにこの割合を変更することを決定できます。



この場合、当然、雇用主は年金基金にいくら支払われるべきかについての情報を受け取るべきです-しかし、彼はどの特定の年金基金にお金が入るかを知らないはずです。



このため、識別に問題があります。これは、法人に部分的なアクセスまたはゼロ知識（ゼロ知識証明）の情報を提供することで解決できます。 この問題は、サービス、テクノロジー、およびSAP Cloud Platformに基づくプラットフォームの組み合わせにより解決されます。



このプロセスの他の参加者を識別することも必要です。 それらの中には：



-銀行-誰に、どのくらいの金額を移転するかを理解しなければなりません。

-年金基金-ブロックチェーンノードの所有者であり、領収書に関する情報をアカウントに入力できますが、情報を偽造することはできません。

-本人-彼はいつでも別の年金基金に収入を移すことができます。 また、彼はいつでも貯蓄に関する情報にアクセスできなければなりません。



以下に、パイロットシステムの例を示します。







最初の画面はユーザー認証です。 SNILSまたはStateサービスを使用して許可を実装することができます。







次の画面は、分析機能を備えた個人アカウントです。



ブロックチェーン技術は基本的にデータベースであり、追加の分析、グラフ、またはその他の「装飾」はありません。 したがって、このパイロットでは、フロントエンド全体がSAP Cloud Platformのサービスを使用して実装されます。 このプラットフォームには、とりわけ、次のような外部システムとの統合の広範な機会が含まれます。 ブロックチェーンネットワーク。 このプロジェクトでは、イーサリアムネットワークがブロックチェーンとして使用されました。





取引明細画面



各請求には、独自のブロックチェーントランザクションIDとディスパッチIDがあります。 各IDはクリック可能で、技術的な詳細が含まれています。



システムには、トランザクションに関する軽いステータスもあります。 黄色はトランザクションが進行中であることを意味し、緑色はネットワークのすべてのノードによって確認されていることを意味します（そして変更するのは非常に困難です）。 その結果、すべての料金が表示され、かなり確実に保存されます。





関税変更画面



ユーザーは、一定の期間に1回、料金を変更する機会を得ます。 そのためには、多数の当局に通知する必要があり、約1か月かかります。 システムでは、1クリックで関税が変更されます。





年金基金変更画面



現在のプロセスを変更する方法の別の例は、年金基金を変更するプロセスの変更です。 ブロックチェーンベースのシステムでは、これは即座に行われます。古いNPFはドキュメントを遅らせることも、新しいNPFが自宅でドキュメントの処理を延期することもできません。



誰がこのシナリオを実践する最終顧客になることができますか？ おそらく、これは民間の年金基金または雇用主のコンソーシアムです。 分散レジストリの分散技術を使用しているため、このシステムへの単一の関心を決定することはできません。



開発自体について少し。 ブロックチェーンモデルの展開には数日しかかかりませんでした。 主な時間は、ユーザーインターフェイス、承認および分析システムの開発または追加、年金基金に関する情報およびその他の詳細の追加に費やされました。





スキーム：ブロックチェーンアプリケーションに関連付けることができる他のサービス



現在の状態のこのソリューションは革新的なSAPシナリオにすぎないことを理解することが重要です。 将来的には、現在使用されている金額と、新しいビジネスプロセスに移行するときに節約できる金額を計算するためのビジネスモデルを使用して、ビジネスケースを作成できます。 開発中の私たちのタスクは、このシナリオが実際に機能することを技術的に実装する可能性を示すことでした。 さらに、すべては年金改革と政府の行動の詳細に依存します。
