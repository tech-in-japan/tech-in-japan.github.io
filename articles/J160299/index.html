自分用のモバイルローミング。 モバイルオペレーターからアスタリスク+ドングルへのGSMゲートを作成する

出張や休暇で国際電話や仮想プロバイダーを使用することはあまりありませんが、めったにこれを気にせず、費用を気にせずにローミングを使用することがあります。

現地のオペレーターのSIMカードを購入するのにそれほど時間はかからず、Wi-Fiがほぼどこにでもある国には行きません。

たまたまプログラムでアスタリスクPBXを知ったので、次の旅行の前に、自分のSIMカードをアスタリスクの入ったトランクとして貼り付け、背後にある雪に覆われたモスクワに立ち、自分にしがみつくのがいかにクールかを考えましたインターネット上の顧客。 インターネットがある場合は自分で電話したくはありません。自分の番号に電話をかけます。これは多くの連絡先を持つ人にとって重要です（全員に通知せず、半分は忘れます）-SIMカード付きの携帯電話は実際に家にあります地域。

sip.confを作成します。 それだけです



extensions.confを開く-これは通話プランであり、通話の転送先と転送方法の説明です。



必要なのは、2つのコンテキストを記述することです。[ドングル着信]-モデムに着信するコールの処理と[デフォルト]-ソフトウェアから着信するコールの処理。 電話。

簡単なものから始めます。

[デフォルト]

exten => _7X。、1、Dial（Dongle / dongle0 / holdother：+ $ {FILTER（0-9、$ {EXTEN}）}）

exten => _ + 7X。、1、Dial（Dongle / dongle0 / holdother：+ $ {FILTER（0-9、$ {EXTEN}）}）

exten => _8X。、1、Dial（Dongle / dongle0 / holdother：+ 7 $ {FILTER（0-9、$ {EXTEN：1}）}）

exten => _007X。、1、Dial（Dongle / dongle0 / holdother：+ 7 $ {FILTER（0-9、$ {EXTEN：3}）}）

exten => h、1、ハングアップ（）




これらは実際には同じことを行う4行であり、宛先電話が7桁のパターンに一致する場合は1行目、+ 7桁の場合は2行目、3番目の8桁および4 007桁のみです（多くのソフト電話では、「+」を「00」として送信するオプションがあります）

各行は同じことを行います-呼び出しを+ 7桁の形式でドングル0チャネル（モデム）に転送します。

さて、最後の行は「電話を切る」イベントで呼び出され、電話を切るだけです。 しかし、あなたはそれを書くことさえできません。これは形式的なことです。



これがどのように機能するかを理解しやすくするために、meアカウントのアスタリスクに登録された電話からの着信コールは、[me]のコンテキストオプションで示されるように[default]コンテキストで処理されます。 [デフォルト]のコンテキストでは、宛先番号がテンプレートに対してチェックされ、テンプレートのいずれかに該当する場合、アクション「Dial（Dongle ...）」が実行されます。 ちなみに、宛先番号がテンプレートに該当しない場合、どこにルーティングされているかわからないため、コールはドロップされます。興味がある場合は、+ 4 ...などの場所にコールしてみてください。 また、電話詐欺から身を守る方法はありませんが、彼らが何らかの形で通り抜けてもウガンダに電話することはありません。
