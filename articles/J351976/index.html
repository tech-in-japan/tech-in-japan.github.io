FastReport.Mono。 パート1：デモWebレポートを実行する

FastReport.Monoの公式ドキュメントは古く、どのバージョンのDebianディストリビューションに属しているのかさえ明確ではない（6以下であると想定できます）ため、デモ用のWebインターフェースをインストールするための多かれ少なかれ正しい説明を書くことにしましたレポート。



この問題は、FastReport.Netが使用されていたプロジェクトの1つをWindows + OracleからLinux + PostgreSQLに移行する見通しが迫っていたために発生しました。 さらに、Dockerコンテナにマイクロサービスの形ですべてを実装する可能性に興味がありました。 悲しいかな、「紙の上では滑らかでしたが、渓谷については忘れていました」：私は、多かれ少なかれ詳細に説明し、問題の可能な解決策を提案しようとする多くのニュアンスに直面しなければなりませんでした。



その過程で、プロジェクトwikiでリマインダーメモを自分で作成し、それからすべてを体系化しようとしました。 その結果、私は多くのテキストを得ました、私は暫定的に出版物を3つの部分に分割しようとします：



  • デモレポートの実行
  • DockerコンテナでFastReportを使用する
  • DB接続


NB以下のテキスト全体は、アクションのステップバイステップガイドではなく、問題の可能な解決策です（複数の方法があります）。 したがって、すべての資料をすぐに読んでから、適切なインストール方法を選択することをお勧めします。



はじめに



したがって、最初に注意したいのは、パッケージ名と依存関係の問題です。 Debian 8（Jessie）を使用する予定がある場合は、 MonoプロジェクトリポジトリからMonoをインストールするか、ソースからビルドする必要がある場合に備えてください。 公式のJessieリポジトリには、libapache2-mod-monoなどのいくつかのパッケージがありません。 私はこれを始めなかったので、すべての調査はDebian 9（ストレッチ）で行われました。 ASP.Netサンプル（asp.net-examples）を含むパッケージをインストールすることは強くお勧めしません。 インストールプロセス中に、すべての仮想ホストの均一なMono設定を含むApache構成にインクルードが追加され、ユニバーサルエイリアス/サンプルも追加されます。



2番目：公式開発者ブログの出版物からApache仮想ホスト構成ファイルを使用する場合、Apache 2.2用に書かれており、Apache 2.4ではまったく機能しないことに注意してください。 問題は、バージョン2.4で登場したディレクトリ権限の設定を変更することにあります。古い設定では、[ディレクトリ]セクションに[すべて許可]はありません。 さらに、モノラル設定のいくつかの変更が可能です。 デフォルト値に対応しますが、何かが適切でない場合があります。 原則として、デモプロジェクトを開始するために、ほとんどすべてを省略できます。 そしてもちろん、選択した構成に応じて、環境変数またはファイルとディレクトリへのアクセス権を設定します。



3番目の非常に興味深い点は、適切な操作のために、ディレクトリ構造を変更する必要があることです。例から愚かなコピー＆ペースト行を行うのではなく、 別の投稿を検討しました （著者が2つのエントリを1つに結合しなかった理由は謎です）。 または、環境変数MONO_PATHを設定して、FastReport。*。DLLファイルと正しいweb.configを含むディレクトリを検索します。



4番目の、そして最も物議を醸す、興味深い瞬間-FastReport.Webを機能させるために、Xが必要になる場合があります。 たとえば、バーコードデモプロジェクトにはSystem.Windows.Formsが必要です。 また、GUI経由でログインしたユーザーに代わってサーバーを起動するときに特別な問題がなければ、ApacheまたはDockerコンテナーでSSHを介してプロジェクトを起動する場合は、環境変数を構成する必要があり、コンテナーの場合はXサーバーも「転送」する外から。



5番目のポイントでは、いくつかの一般的なDBMSのコネクタを組み立てるプロセスを検討します。 これは、FastReport（WindowsとMonoの両方のバージョン）では、DBMSを使用するためのプラグインがすぐに使用できるライブラリではなく、ソーステキストの形式で提供されるためです。 手動で収集する必要があります。



しかし、まず最初に。 システムはすでにインストールされており、完全に空ではないことを前提としています。お気に入りのテキストエディタとアーカイバがあります。



初期インストール



公式マニュアルを信じているなら、インストールにそれほど時間はかからないはずです;すべてがaptをスムーズに通過するはずです。 さあ、始めましょう。 Mono、XSP Webサーバー、Apache Webサーバー、およびApacheを介してASP.Netアプリケーションを実行できるモジュールをインストールします。 



apt-get install mono-complete mono-xsp apache2 libapache2-mod-mono


次に、デモプロジェクトをダウンロードして解凍します。 以下では、/ var / www以外のパスを具体的に使用します。 まず第一に、これにより、構成またはプロジェクト自体のエラーを見つけることができます。 次に、サードパーティのソフトウェア製品の/ optへの配置は、このディレクトリの目的に対応しています。 



 mkdir –p /opt/fastreport/htdocs #  wget https://www.fastreport.ru/public_download/frmono_demo.zip -O /tmp/frmono_demo.zip # , ,  unzip /tmp/frmono_demo.zip –d /tmp/frmono.demo cp –rp /tmp/frmono.demo/Demos/C#/Web/* /opt/fastreport/htdocs cd /opt/fastreport/htdocs xsp




図1. XSPの開始



XSPはAPS.Net用の軽量Webサーバーであり、C＃で記述されており、単純なアプリケーションをすばやく簡単にテストできます。 経験豊富なドキュメンテーションとアドバイスを信じているなら、戦闘環境での使用は推奨されません。 デフォルトでは、XSPはポート9000およびすべてのネットワークアダプター（IP 0.0.0.0）でリッスンします。 この手順は、原則として少なくとも何かが機能するかどうかを示します。 何らかの例外が発生すると想定できます。 残念ながら、レポートを生成するためのこのような一見興味深いツールの作成者は、デモ資料をインストールする便利さを考慮していませんでした。





図 2.考えられるエラーメッセージ（写真はクリック可能）



この問題は、MonoがFastReport.Web.dllライブラリを見つけることができないという事実によるものです。 さらに、ファイルがhtdocsにコピーされても、エラーが発生します。 私は.NETの専門家ではなく、Monoの専門家でもありません。 そして、私はすぐに紹介記事で著者のブログの投稿へのリンクを読みませんでした（ところで、質問があります。たとえば、ディレクトリではなくファイル名が太字で強調表示されているのはなぜですか？）。 これによれば、頭を砕かなければならなかった。 最初の可能な解決策は、DLLを現在のディレクトリにコピーし、環境変数MONO_PATHを設定することです。 



 cp /tmp/frmono.demo/FastReport.*.dll ./ export MONO_PATH=./


しかし、ここでも失敗が待っています。 ほとんどの場合、名前空間またはタイプFastReport.Utilsが見つからないという例外を受け取りました。 この問題は解決できます-web.configにFastReport.Mono.dllを登録するだけです。





図 3.コンパイルエラー-Utilsの種類または名前空間が見つかりません（写真をクリックできます）



依存関係を追加します-web.configセクションで、次の行を記述します。 



 <add assembly="FastReport.Mono, Version=2017.1.8.0, Culture=neutral, PublicKeyToken=db7e5ce63278458c"/>


バージョン番号とトークンは異なる場合がありますが、現時点では関連しています。 次のように、dllからトークンを取得できます。 



 sn -T FastReport.Mono.dll


次のようにバージョン番号を確認します。 



 monodis --assembly FastReport.Mono.dll | grep "Version"


私たちのすべての苦痛の結果は、デモレポートを含むページの多かれ少なかれ正気な表示になるはずです。

System.Windows.Formsを使用する



依存関係リストのサンプルWebレポートには、System.Windows.Formsが含まれています。 web.configを見ると、次の行に注意を払っていると思います。 



 <add assembly="System.Windows.Forms, Version=2.0.0.0, Culture=neutral, PublicKeyToken=B77A5C561934E089"/>


スクリーンショット4またはコンソールに表示される問題が発生した場合、「プロトコルが指定されていません」というメッセージが表示される場合、次のいずれかのオプションがあります。



  • Xorgがインストールされていないか実行されていない
  • Xサーバーが実行中です。XSPをルートとして実行しようとしています
  • Xサーバーが実行されていますが、sshセッションでXSPを実行しようとしています
  • Xサーバーが実行されている、Apacheを使用している、www-dataユーザーに必要な権限がない
  • Dockerで使用


最初のケースでは、Xをインストール/実行する必要があります。 オプション2と3は基本的に類似しており、環境変数DISPLAY 、および場合によってはXAUTHORITYを設定することで問題が解決します。 唯一の違いは、OSが最初にグラフィカル環境でインストールされた場合、ユーザーはXAUTHORITY変数が設定されており、〜/ .Xauthorityファイルを見ますが、ルートはそうではないことです。 原則として、XSPを起動する前に次のことを行うだけで十分です。 



 export DISPLAY=:0


ルートの場合、次を追加する必要があります。 



 export XAUTHORITY=/var/run/lightdm/root/:0


実際、スーパーユーザーに代わって何かを実行するのは悪い考えですが、本当にしたい場合は...さらに、XAUTHORITY変数を設定することは、www-dataユーザーがXサーバーを使用できるようにApacheを設定するときに役立ちます（Debianのインストール中に関連しますグラフィック環境パッケージが選択されました）。 OSが最初に最小バージョンでインストールされた場合、すべてがわずかに異なります。 Xorgは、ディスプレイマネージャーなしでインストールできます。 またはまったくインストールされていません。



FastReport.Monoをコンテナで「ラップ」する必要があり、「1サービス-1コンテナ」の原則を遵守することに決めた場合、Xサーバーでコンテナを作成し、コンテナ間でソケットを「転送」する必要があります。 ビデオドライバーとしてダミーディスプレイドライバーを使用する必要があります。 この構成については、マニュアルの以下のいずれかの部分で説明します。 



 apt-get install xserver-xorg-video-dummy


Linux / Unixユーザーの代替オプションは、ホストOS（tcpまたはunixソケット）からXサーバーを転送することです。 MacOSでこの機能を使用するには、 XQuartzをインストールする必要があります。



Apacheを構成する



原則として、Apache 2.4とmod_monoのセットアップには特別なトリックは必要ありません。 突然、私のアドバイスに従わずにasp.net-examplesパッケージをインストールした場合は、mod_monoヘルスチェックの次の行にコメントすることをお勧めします。

/etc/apache2/mods-available/mod_mono.conf： 



 Include /etc/mono-server4/mono-server4-hosts.conf


そうしないと、エイリアス/サンプルがすべての仮想ホストに釘付けになり、ASP.Netの使用例が開かれます。 さらに、Apacheはログを誓うことができ、再起動するとmod_monoがすでにロードされています。 これは、2つのモジュールmod_monoとmod_mono_auto（アプリケーションの自動デプロイメント用）がロードされるという事実によるものです。 mod_mono_autoは安全に無効にできます： 



 a2dismod mod_mono_auto && service apache2 restart


公式ドキュメントの仮想ホスト構成ファイルは、設定が冗長であり、場所によっては関係ありません。 最低限必要な構成は、ネタバレの下で見ることができます。



001-mono.conf 
 Listen 8081 <VirtualHost *:8081> DocumentRoot "/opt/fastreport/htdocs" <IfModule mod_mono.c> MonoUnixSocket FrSite /tmp/.mod_mono_server MonoServerPath FrSite /usr/bin/mod-mono-server4 #       htdocs/bin -      #      /opt/fastreport/htdocs     #    FastReport.*.dll MonoPath FrSite /usr/lib/mono/4.5:/usr/lib:/usr/lib/mono/4.0:/opt/fastreport/htdocs AddMonoApplications FrSite "/:/opt/fastreport/htdocs" MonoAutoApplication Disabled MonoDocumentRootDir /opt/fastreport/htdocs MonoDebug false MonoSetEnv FrSite DISPLAY=:0;HOME=/opt/fastreport AddHandler mono .aspx .ascx .asax .ashx .config .cs .asmx .axd </IfModule> <Directory "/opt/fastreport/htdocs"> Require all granted Options Indexes FollowSymLinks MultiViews AllowOverride All <IfModule mod_mono.c> SetHandler mono MonoSetServerAlias FrSite DirectoryIndex Default.aspx </IfModule> </Directory> #     bin   dll #      <Directory "/opt/fastreport/htdocs/bin"> Require all denied </Directory> </VirtualHost>


実際、お気に入りのテキストエディターを使用してすべてを/etc/apache2/sites-available/001-mono.confに入れ、サイトをオンにしてApacheを再起動します。 



 a2ensite 001-mono && service apache2 restart


構成全体を解析することは意味がありませんが、1行に詳しく説明します。 



 MonoSetEnv FrSite DISPLAY=:0;HOME=/opt/fastreport


これは非常に高貴な松葉杖で、一度に2つの問題を解決できます。 System.Windows.Formsに特化したセクションでそれらの1つを既に説明しました-アプリケーションはDISPLAYを指定する必要があります。 2番目のポイントは、アプリケーションのコンパイルおよび実行中に、ディレクトリ〜/ .localおよび〜/ .monoが使用され、Apacheを実行するユーザーwww-dataがホームディレクトリ/ var / wwwを持っているという事実に関連しています。 システム全体を散らかす価値はないと考えました。 原則として、機能します。 もちろん、この特定の仮想ホストのユーザーとグループを変更することもできますが、それはうまくいきませんでした。 私が理解しているように、mod_monoはApacheユーザーからサーバーを起動し、mpm_itkを使用しようとしても何も機能しませんでした。 Apacheプロセスは、MonixサーバーのUNIXソケットにアクセスできません。 FastCGIを使用して問題を解決できる可能性はありますが、試していないので、私は本当のふりをしていません。



最後の仕上げは、www-dataユーザーにXサーバーへのアクセスを許可することです。 そしてここで、スーパーユーザー用にXAUTHORITYをインストールする機能を使用できます。 



 export XAUTHORITY=/var/run/lightdm/root/:0 export DISPLAY=:0 xhost + local:www-data service apache2 restart


xhostが呼び出されたときに行われた変更は、シャットダウン/リブートまでしかアクティブにならないことに注意してください。 lightdmを使用する場合、この構造をスクリプトでラップし、/ etc / lightdm / lightdm.confのdisplay-setup-script = / path / to / script.shの行を追加して自動起動に設定するのが理にかなっています。 または、/ etc / X11 / xinit / xinitrcを変更できます。
