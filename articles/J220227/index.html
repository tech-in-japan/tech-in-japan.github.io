ipを拒否するものは何ですか？

アクセスリストのほとんどの実装は、「許可されていないものはすべて禁止されている」動作を意味します。 設計中に特定の方向でこれまたはこのタイプのトラフィックを事前に予測するという事実を考慮した合理的なアプローチ：サブスクライバーまたはピアツーピアパートナーが接続している場合、このインターフェイス上の他のIPからのデータはなく、インターネット接続がある場合、プライベートアドレスはどこから来たのですか？ それとも、これらすべてが無駄なのでしょうか？ たぶん、偽のトラフィックはなく、ACLの無条件禁止はリソースの転送にすぎません。 結局、オペレーターは顧客自身にアドレスを提供し、ピアツーピアパートナーとアップストリームプロバイダーはコミュニケーションの兄弟であり、状況の複雑さと繊細さを理解する必要があります。 残念ながら、これはまったく当てはまりません。

YaC2013で開催されたDaOS円卓会議の参加者は、既存のすべての推奨事項について、誰もネットワークのセキュリティに対処しようとしていないと非常に不満を述べました。 つまり、少なくともIPスプーフィングに対処するために、まず（通信事業者と）自分で始めなければなりません。

deny ip any any



from deny ip any any

インターネットのint3に何が期待できるかはまだわかりません 。 アクセスリストは以前と同じですが、既知のアドレスからだけでなく、すべてのアドレスからアクセスできるようになりました。
