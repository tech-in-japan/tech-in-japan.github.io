Cisco、NetApp、およびVeeamを使用したデータセンターリファレンスアーキテクチャの例

現在、IT部門は、サイト、データベース、電話などに不具合があるため、会社のデータセンターの円滑な運用を確保する必要があります。 経済的および評判上の損失をもたらす可能性があります。 したがって、データセンターの近代化の重要な部分は、標準化された、再現可能で信頼性の高いリファレンスアーキテクチャの構築です。 リファレンスアーキテクチャは、一方ではビジネスの特定のニーズを満たし、他方では非常に信頼性の高いシステムを構築するための基本原則を満たしている技術的特性と展開アプローチを定義します。 参照モデルには、コンポーネントの構成と構造の説明、およびデータセンター全体の最適な運用を保証する製品のインストールと構成のベストプラクティスが含まれています。 このモデルは、IT部門のさまざまなプロジェクトに取り組む際の便利なテンプレートとして機能します。



データセンターの参照モデルに基づいてITインフラストラクチャの設計を開発するには、まず、このインフラストラクチャの要件と期待を明確に明確にする必要があります。 次に、使用する予定のコンポーネントを詳細に説明する必要があります。 ITには、適切なパフォーマンス、クラッシュ保護、高データ可用性を実現する設計が必要です。 例として、NetApp、Cisco、およびVeeamのソリューションを含むデータセンターインフラストラクチャモデルを考えます。







図1 Cisco、NetApp、およびVeeamに基づくデータセンターアーキテクチャモデル



このモデルは、最適な構成を作成、テスト、および検証するために、すべてのメーカーの緊密な相互作用を提供します。 3つのテクノロジー（ネットワーク、データストレージ、アクセシビリティ）の組み合わせにより、ビジネスとITの要件を満たすアーキテクチャを設計する方法を見てみましょう。 図1は、データセンターをベースに構築する方法を示しています。 すべての監視およびデータ保護タスクのリソース割り当ての管理と制御は、Veeam用のCisco Applianceによって実行されます（図の＃1）。 Cisco Backup Applianceは、バックアップ、レプリケーション、クラウド、仮想環境の監視など、あらゆる種類のタスクの管理に役立ちます。



Veeamラボでのテスト中、このようなインフラストラクチャは3x60ホストで1.1 GB / sの帯域幅を示し、イーサネット接続で10ギガビット/ sに達しました。 同時に、サーバーノードでプロセッサとRAMのリソースの60％以上がまだ利用可能でした。 これらのテストの結果に基づいて、プロセッサ（および40ギガビット/秒で動作するイーサネット接続、およびそれに応じて構成されたストレージ）の全負荷で、3x60サーバーのスループットが2.5 GB /秒を超えると予想できます。 図では、テスト結果と期待値が図2に示されています。





*-C32600での最適な構成



図2。 データをテストします。



最も重要なアプリケーションの場合、Cisco / NetApp FlexPod（上記の図1の＃2）は、必要なコンピューティングリソースとネットワークリソース、およびストレージリソースを提供します。 FlexPodを使用したバックアップとレプリケーションでは、VeeamはNetApp DATA ONTAPハードウェアスナップショットとの統合を使用します（図3を参照）。



これにより、VMwareや実稼働ストレージリソースの使用が最小限に抑えられます。







図3 NetApp DATA ONTAPハードウェアスナップショット統合



バックアップまたはレプリケーションジョブ中に、VeeamはVMwareスナップショットを作成してから、NetAppにハードウェアスナップショットを作成します。 ハードウェアスナップショットには一貫した状態のVMが含まれているため、VMwareスナップショットはすぐに削除されます。 スナップショットのコミット後、VMwareはVeeamがソースとしてSnapMirror / SnapVault更新をバックアップNetAppストレージ（図1の＃3）に切り替えるか、プライマリストレージから直接バックアップを開始できます。 次に、NetAppハードウェアスナップショットがCisco Backup Applianceにマウントされ、ジョブがバックアップまたは複製されます。 ジョブが完了すると、ハードウェアスナップショットは自動的に削除されます。



NetAppとの統合（ Habréの投稿を参照 ）を使用して、ONTAPスナップショットを管理し、複数のONTAPシステム間のSnapMirrorおよびSnapVault接続を制御することもできます（図1の＃4）。



それほど重要ではないアプリケーション（VDIインフラストラクチャ、ブランチオフィスネットワーク、テスト環境など）では、Cisco HyperFlexシステムを使用して必要な制御とパフォーマンスを提供できます（図1の＃5）。 Veeam用のCisco Backup Applianceは、Cisco HyperFlexシステム（図1の＃6、および図4）からのバックアップ、およびHyperFlexとFlexPodシステム間での仮想マシンの複製（図1の＃7）に使用されます。



Cisco Backup Applianceは、あらゆるサイトに柔軟なバックアップおよびデータ転送機能を提供します。 さらに、レプリカデータを使用して、Veeam SureReplicaを使用して回復可能性を自動的に確認できます。







図4 データ送信の構成



どの業界で仕事をするかはそれほど重要ではありません-ビジネスは急速に変化し続け、ITインフラストラクチャを新しいレベルの適応性と信頼性にする必要があります-さもなければ、データ損失、アプリケーションやサービスの障害により利益の損失という形でビジネス損失を引き起こすリスクがあります。 そのため、信頼性、フォールトトレランス、および最終パフォーマンスの観点から、データセンターを構築するための汎用参照モデルを作成することが重要であると考えています。
