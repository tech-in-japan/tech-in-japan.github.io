Windows Mobile 6.5.3または私たちを待っている、デバイスでのベータレビュー

今日、Windows Mobile 6.5.3-28205上のToshiba Portege G900のファームウェアを更新しましたが、携帯電話を失うことはありませんでした。抑制感と不完全さはなくなりました。



順番に行きましょう。 Windows Mobile 6.5から6.5.3バージョンで発生した変更のレビューのような何かをしたいと思います。 これらの変更は最終的なものではなく、何か他のものがあるかもしれないことを強調したいと思います。

それでは、始めましょう：







これが最初の画面です。最初に嬉しかったのは、予想よりも速く表示されたことです。 20での関心の主観的評価によると、少なくとも今のところ、すべてはブレーキやハングなしで動きます。 下部パネルが半透明になりました。







スタートメニューはあまり変更されていませんが、アイコンが少し変更され、非常に便利なものが表示されているようです-ところで、最初の起動時に、それは最初の画面にもあり、それを見ると、そこから自分自身を削除することを提供します。







ネットワークに接続してMyPhoneをダウンロードした後、連絡先を復元し、プログラムを追加することにしました。同時に、Marketplaceが機能しているかどうかを確認し、同時にダウンロードしてインストールする必要がありました。2段階でインストールされ、起動するたびに更新プログラムをチェックしました。

インストールされたMobile Office 2010ベータ版をダウンロードできるようになりました。 同時に、私はBingをインストールしました-彼はロシア語を理解することを拒否しましたが、 音声を認識できる検索、地図、ルート、場所です。







オフィスがインストールされ、再起動を求められましたが、現在はSharePoint Workspace Mobile 2010（ベータ版）があり、docxとxlsxを使用する機能がありますが、後者はエラーを理由に正しく開くことを拒否しました。







オペレーティングシステム自体に戻ると、SMS通信はもう少し美しく見え始め、連絡先は変更され、それぞれの情報が大きく表示されるようになり、各タイプの番号には独自のアイコンがあり、連絡先の編集は少し驚きました-背景の奇妙な灰色のセルは不完全を示唆していますところで、実際に気づいたのは、メールグループとメッセージグループを作成できるようになったことです。







これらは、私がこれまでに発見したすべての変更です。

要約すると、新しいファームウェアに満足し、すべてが（今のところ）十分に高速で動作します。オフィスが新しいフォーマットをサポートするようになったことはうれしいです。Marketplaceが機能していることは非常にうれしいです。 そこにはさらにプログラムがありますが、これは時間の問題だと思います。 セルの動作時間がわずかに短縮されたように思えました。おそらく私が非常に積極的に使用したためです...

それだけです 読んでくれてありがとう。
