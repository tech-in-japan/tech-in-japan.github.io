テクニカルライター：あなたは私です-私はあなたです

それでは、テクニカルライターは何ができますか？ 従来、テクニカルライターは次のものの作成に関与しています。



  • ソフトウェア製品の使用を支援する「オンラインヘルプ」。このようなヘルプの例は、「ヘルプ」ボタンを押すとプログラムに表示されるすべてのものです。
  • ソフトウェア製品を組み立てるためのドキュメント（このドキュメントの価値は、特定のプログラムを組み立てる必要がある会社での最初の日に理解します。このため、既存のマニュアルは現実とは関係ありませんが、オフィスを歩き回って全員に質問する必要があります。おなじみの写真？）;
  • 会社の従業員向けのトレーニング資料。
  • 企業のWebページまたは特定の企業製品のコンテンツ-この基準によれば、コンテンツマネージャーはテクニカルライターでもあります。
  • システムメッセージまたはドキュメントを別の言語に翻訳する際の適切な用語と外国語の類似語の選択。
  • 会社のスタッフへの指示（英語の指示の例： http : //www.envision.ca/templates/resources.asp?ID = 111 ）;
  • ビジネスへの投資を誘致する潜在的な投資家の要求に答えます。




しかし、それだけではありません。 彼らの能力には、イラスト、グラフ、チャートの選択、ビデオ素材の準備、マルチメディア製品、ウェブサイトも含まれます。 一般に、すべてはエンドユーザーにこのソフトウェア製品の適用方法を説明することであり、マーケティング部門の領域に入ることではありません。「製品を購入する-最高です！」。

よりエキゾチックで型破りなアプローチがあります。 たとえば、システムメッセージとプログラムエラーレポートの編集にテクニカルライターを関与させたり、開発の過程でグラフィカルインターフェイスの使いやすさを評価したり、



とにかく、この職業の現代名は「技術コミュニケーター」であり、これは必ずしも書かれているとは限らないあらゆる方法による情報の提出と交換を提供します。
