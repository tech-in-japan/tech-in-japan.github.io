Windows 8 Release PreviewでのSkypeの問題

最近リリースされたWindows 8にSkypeをインストールした後、Release Previewで予期しない問題が発生しました。Skypeの2番目のインスタンスの起動中、ログイン画面にしっかりとハングし、プロセスの強制終了のみが役立ちました。 Skypeのビジネスバージョンと通常バージョンの両方が機能しませんでした。 システムの速度も大幅に低下しました。 ログイン中に、Skypeはログイン中にCPUの最大30％を消費し、この状態のままになります。











再起動後、Skypeは1つのコピーでもログインを完全に停止し、ログイン画面でフリーズします。 インターネットでは、AppDataフォルダーを消去するか、古いバージョンのSkypeを使用することをお勧めしました。



AppDataのクリーニングは、Skypeの活性化に本当に役立ちましたが、不当なCPU消費の問題は解決しませんでした。 さらに、しばらくして、Skypeの2つ目のコピーが開始され、一緒に50％以上をすでに食べました！







Skypeはこの問題の存在を確認し、解決に取り組んでいます。 ただし、エラーを修正する期限は指定されていません。



また、ユーザーは、Skypeの古いバージョンにはこのような問題がないことを発見しました。特に、Skype 4.2は、Windows 8リリースプレビューでの通常のCPU消費によって区別されます。 複数のインスタンスで実行する場合。







Windows 8 Release PreviewでSkypeを使用する場合は注意してください。 CPUの消費量が多い場合は、Skypeチームがこのエラーを修正するための更新プログラムをリリースするまで、Skypeの古いバージョンにロールバックすることをお勧めします。
