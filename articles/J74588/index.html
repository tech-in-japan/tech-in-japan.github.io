Ubuntu 9.10のNetworkManagerを使用したDSL接続の問題を修正

Ubuntu 9.10のリリースで、作成されたDSL接続を介してNetwork Managerからインターネットにアクセスする際に問題が発生したことは周知の事実です。

もちろん、このバグにスコアを付けてpppoeconfを使用することもできますが、私はその点について語りました。



そのため、集中的な検索、試行錯誤の後、Karmic KoalaでのNetworkManagerの通常の操作（DSL接続の作成と接続）では、更新するだけでよいことがわかりました（今日-バージョン0.8 *に）。



これはすべて非常に簡単に行われます。



1.まだインターネット接続が必要なので、現時点ではpppoeconfを使用します。

sudo pppoeconf





2. /etc/apt/sources.listに行を追加します

deb ppa.launchpad.net/network-manager/trunk/ubuntu karmic main

deb-src ppa.launchpad.net/network-manager/trunk/ubuntu karmic main





3.キーをインポートし、パッケージのリストを更新します

sudo apt-key adv --keyserver keyserver.ubuntu.com --recv-keys BC8EBFE8

sudo apt-get update





4. NetworkManagerの更新

sudo apt-get install network-manager





5. / etc / network / interfacesから余分な行を削除します。つまり、

auto lo

iface lo inet loopback





6.システムを再起動し、新しいDSL接続を追加し（起動時に自動接続チェックボックスをオンにしました）、楽しんでください（-：
