LaTeXのダイレクトフォントで数式にギリシャ文字を作成する方法

ご存知のように、LaTeXの式では、ギリシャ文字と積分は斜めになっています。 公式の旧ソビエト文学では、ギリシャ文字と積分が直接であり、一部の人々は、手紙、学位論文、要約およびその他の文書では、ギリシャ文字も直接フォントであるべきだと要求しています。

以下は、これをLaTeXで実装できるパッケージです。 このパッケージは、論文の作業の過程で作成しました。公式のギリシャ文字は直接フォントにするように頼まれました。



これは、LaTeXで書かれた式がパッケージを接続した場合にどのようになるかを示す図です。





誰かが突然これを必要とした場合は、猫の下を参照してください。







そのような機能を実装するLaTeXの完成パッケージは見つかりませんでした。 解決策を見つけるとmathptmパッケージが使用されましたが、このパッケージでは式はWord'eのように見え始めます。 ラテン文字は、通常と太字の中間になります。 また、\セクション、\サブセクションヘッダーの太字スタイルは失われます。



一般的に、私は自分の自転車を作らなければなりませんでした。

私のパッケージの基礎となるアイデアはここで借りています 。 このリンクは、文書の前文でギリシャ文字を再定義する方法を説明しています。



すべてが3つのパッケージの形式で設計されています。



  • rumathgrk-ギリシャ文字を直接定義します

    たとえば、\ alphaup-直接の文字 "alpha" \ betaup-直接の文字 "beta"など。

  • rumathgrk1-標準のギリシャ文字（\アルファ、\ベータなど）をオーバーライドして、まっすぐにする
  • fixint-積分が直線になるように再定義します。 fixintオプションを持つdisserクラスも同じことを行うことに注意してください。




使用するには、これらのパッケージのいずれかをプリアンブルの任意の組み合わせで接続します。 fixintパッケージを使用するには、amsmathパッケージを接続する必要があります。 パッケージは、一般的な論文のスタイルシートと互換性があります。



最小限のドキュメントの例を次に示します。 



\documentclass[a4paper,12pt]{article} \usepackage[koi8-r]{inputenc} \usepackage[russian]{babel} \usepackage{hyperref} \usepackage{mathtext} \usepackage{amsmath} \usepackage{fixint} %     \usepackage{rumathgrk1} %        \begin{document} %         \begin{equation} S(\omega)=F[s(t)]=\int\limits_{-\infty}^{+\infty}s(t)\mbox{e} ^{-j\omega t} dt \end{equation} \begin{equation} F=\frac{q_1 q_2}{4\pi \varepsilon_0 \varepsilon r^2} \end{equation} \end{document}




ドキュメント内の直接積分およびギリシャ文字が不要になった場合は、前文からrumathgrk1およびfixintパッケージを削除するだけです。



Githbaから既製のパッケージを収集できます。 ここから Linuxでは、gitがインストールされている場合、次のコマンドで実行できます。 

 git clone https://github.com/ra3xdh/rumathgrk




コンソールでLinuxおよびTeXliveにインストールするには、パッケージのあるディレクトリに移動して、コマンドを実行します。 



 make install




Windowsパッケージはテストされていませんが、動作するはずです。 MiKTeX用に手動でインストールします。 あらゆる目的での使用が許可されています。



結論として、私はLaTeXからの標準的な傾斜したギリシャ文字と積分を好むことに注意します。 また、一般的に、私たちの人々は傾斜した積分とギリシャ文字に寛容です。 通常、標準のLaTeXフォントは誰にでも適しています。
