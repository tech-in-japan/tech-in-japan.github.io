LTSP、Ubuntu。 シンクライアントの構成の問題

こんにちは、Habraコミュニティの皆さん。

私は大学で働いており、古いコンピューター室をターミナルシステムに移すことにしました。







本質は次のとおりです。コンピュータはブートディスケットから起動し、ロードするシステムのイメージをDHCPに要求し、DHCPはそれらをターミナルサーバーに送信し、そこからLinuxシステムのイメージをロードします。 ロードすると、自動ログインが実行され、RDPが開始されます。



#!bin/bash

var0=0

var1=1

while [ "$var0" != "$var1" ]

do



rdesktop -d sfmgus.local -f konika2003 -r usb=/media/usb -k ru-RU



wait



done



echo



exit 0



* This source code was highlighted with Source Code Highlighter .





このスクリプトの本質は、rdesktopとユーザーを実行して、「欠陥のあるハードウェア」を備えたコンピューターで作業しながら、通常のシステムパフォーマンスのWindows環境で作業することです。 rdesktopの起動ラインは無限ループにあるため、ユーザーはrdesktopが実行されているシステムにアクセスできません（少なくともそう思います）。



すべてうまくいきます。 ただし、問題はパラメーターが

-r usb=/media/usb



* This source code was highlighted with Source Code Highlighter .





動作しません、つまり 「欠陥のある」コンピューターにフラッシュドライブを接続すると、rdesktopにマウントされません。実際、rdesktopに表示されるすべてのハードウェアはサーバー上にあるためです。



このすべてをセットアップするとき、私は次のように導かれました：



フラッシュドライブの自動マウント



フラッシュドライブを自動的にマウントするには、次の手順を実行する必要があります。

/ opt / ltsp / i386 /に行き、そこにltspfsdをインストールします。ここから入手してください： pkg-ltsp.alioth.debian.org/debian/pool/main/l/ltspfs

chroot / opt / ltsp / i386 /

dpkg -i ltspfsd_0.4.3 + debian2_i386.deb

または、リポジトリから配置

aptitude install ltspfsd



ltspfs サーバーに pts-ltsp.alioth.debian.org/debian/pool/main/l/ltspfsをインストールします

dpkg -i ltspfs_0.4.3 + debian2_i386.deb

または、リポジトリから配置

aptitude install ltspfs



これはomsk.lug.ruのテキストからの引用です




一般に、タスクの本質は、ローカルマシンで動作するCD-ROM（Dvd）、フラッシュドライブを使用することです。これは決して実現できません。 誰かがこれを行っていた場合、どのように実装できるか（教えてください）教えてください。 または、少なくともマニュアルに鼻を突っ込んでください。 Google検索では何も得られません。
