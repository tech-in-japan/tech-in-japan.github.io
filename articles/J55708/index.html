gen_serverには何がありますか？



gen_serverは、クライアントサーバーアーキテクチャを実装するためのインターフェイスモジュールです。 このOTPモジュールを使用すると、「無料」で多くの特典が得られますが、これについては後で説明します。 また、シリーズの後半では、スーパーバイザーとエラーメッセージについて説明します。 そして、このモジュールはほとんど変わりません。



gen_serverはインターフェースであるため、いくつかのメソッドを実装するか、関数（コールバック）を返す必要があります。

  • init / 1-サーバーの初期化。
  • handle_call / 3-呼び出し要求を処理します。 呼び出し要求を送信したクライアントは、応答を受信するまでブロックされます。
  • handle_cast / 2-キャスト要求を処理します。 キャスト要求は、非同期であることを除いて、呼び出し要求と同じです。 クライアントはキャストリクエスト中も引き続き動作します。
  • handle_info / 2-一種の「すべてをキャッチ」メソッド。 サーバーがメッセージを受信し、これが呼び出しでもキャストでもない場合、ここに来ます。 このようなメッセージの例は、サーバーが別のプロセスに接続（リンク）されている場合のEXITプロセスメッセージです。
  • terminate / 2-サーバーが停止すると呼び出されます。 ここで、出発前に必要な操作を行うことができます。
  • code_change / 3-サーバーがリアルタイムで更新されるときに呼び出されます。 このメソッドにスタブを配置する必要があります。 この方法については、今後の記事で詳しく説明します。

Erlybankサーバーの初期化



サーバーは、gen_server：start_linkメソッドで開始します。このメソッドは、サーバーのinitメソッドを呼び出します。 その中で、サーバーの状態（データ）を初期化する必要があります。 状態（データ）は、アトム、値のリスト、関数など、何でもかまいません！ この状態（データ）は、各コールバックメソッドでサーバーに送信されます。 私たちの場合、すべてのアカウントとその残高値のリストを保持したいと思います。 さらに、名前でアカウントを検索したいと思います。 このために、キーと値のペアをメモリに保存するdictモジュールを使用します。



オブジェクト指向の心への注意： OOPから来た場合、インスタンス変数のサーバー状態を取得できます。 各コールバックメソッドでは、これらの変数を使用できますが、変更することもできます。



したがって、initメソッドの最終形式は次のとおりです。 



  init（_Args）->
   {ok、dict：new（）}。


本当に簡単です！ したがって、Erlybankサーバーの場合、initが返す期待値の1つは{ok, State}



です。 状態（データ）としてokと空の連想配列を返すだけです。 また、引数をinit（まだ空の配列、start_linkから覚えています）に渡さないため、引数の前に「_」を付けてこれを示します。



電話するかキャストしますか？ ここに質問があります



サーバーのほとんどを開発する前に、callメソッドとcastメソッドの違いをもう一度簡単に確認します。



呼び出しは、クライアントをブロックするメソッドです。 これは、クライアントがサーバーにメッセージを送信するとき、続行する前に応答を待つことを意味します。 たとえば、特定の口座の残高を要求するなど、答えが必要な場合に呼び出しを使用します。



キャストは、 非ブロッキングまたは非同期メソッドです。 これは、クライアントがサーバーにメッセージを送信するとき、サーバーが応答するのを待たずに、さらに機能し続けることを意味します。 Erlangは、プロセスに送信されるすべてのメッセージが宛先に到達することを保証するようになりました。したがって、サーバー応答が明らかに必要ない場合は、キャストを使用する必要があります。 つまり、メッセージが到着したことを確認するためだけに電話をかける必要はありません。Erlangにおまかせください。

最初に、モジュールの外部から呼び出してアカウントを作成するAPIメソッドを作成します。 



  %% ------------------------------------------------ --------------------
 %%関数：create_account（名前）-> ok

 create_account（名前）->
   gen_server：cast（？SERVER、{create、Name}）。


start_linkで?SERVER



として登録したサーバーにキャスト要求を送信します。 要求は、テーパー{create, Name}



です。 キャストを使用すると、「ok」がすぐに返されます。これは、関数によっても返されます。



次に、このキャストを処理するサーバーのコールバックメソッドを記述する必要があります。 



  handle_cast（{作成、名前}、状態）->
   {noreply、dict：store（Name、0、State）};
 handle_cast（_Msg、State）->
   {noreply、州}。


ご覧のとおり、handle_castの別の定義を追加して、別のリクエストを処理しました。 次に、現在のアカウントの残高を表示する値0の配列に保存します。 handle_castは{noreply, State}



返します{noreply, State}

また、最後に「キャッチオール」機能を追加したことに注意してください。 これはここだけでなく、一般的なすべての機能言語でも実行する必要があります。 このメソッドは、メッセージを静かに飲み込むためにのみ使用できます:)、または必要な場合は例外をスローする必要があります。 私たちの場合、私はそれを静かに処理します。
