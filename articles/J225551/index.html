更新されたHTTP / 1.1

意外なことに、最初の公開から15年後、HTTP / 1.1仕様が更新されました

たくさんの更新があります、私はdofigaとさえ言うでしょう。 多くの明確なテキストを追加し、仕様を6 RFC（以前は2でした）に分割し、308の新しいステータスを追加し、X-Forwarded-Forを標準化しました（現在は単なるフォワードです）。



変更の不完全なリスト：

  • 新しいステータス308はPermanent Redirectですが、同じデータを送信しています。 つまり 要求は以前のようにGETに変更されません。
  • X-Forwarded-ForとX-Forwarded-Protoを置き換える新しいForwardヘッダー
  • サーバーへの2つの接続に関する制限を削除しました
  • HTTP 0.9サポートを削除
  • デフォルトでISO-8859-1エンコーディングを削除
  • Content-MD5ヘッダーを削除
  • POSTリクエストでコンテンツ範囲を無効にする
  • コード204、404、405、414、501のキャッシングを追加
  • コード301および302のドキュメントが変更され、メソッドをPOSTからGETにリダイレクトできるようになりました。 つまり すべてのブラウザの現在の実装を文書化しました。
  • リファラーの送信の禁止と、リファラーが送信されない場合の区別が追加されます。 次に、Refererを送信する必要があります。about：リファラーがない場合は空白です。
  • 場所がハッシュタグリンクにリダイレクトできるようになりました。




新しいRFC：

  • RFC 7230：メッセージの構文とルーティング
  • RFC 7231：セマンティクスとコンテンツ
  • RFC 7232：条件付きリクエスト
  • RFC 7233：範囲リクエスト
  • RFC 7234：キャッシング
  • RFC 7235：認証
  • RFC 7236：認証スキームの登録
  • RFC 7237：メソッド登録
  • RFC 7238：308ステータスコード
  • RFC 7239：転送されたHTTP拡張
