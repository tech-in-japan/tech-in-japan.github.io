テスターに​​必要なツールは何ですか？



脳は、テスターだけでなく、おそらくあらゆる職業にとって最も重要なツールです。 特にUIをテストする場合は、目も役立ちます。 もちろん、耳：アプリケーションを起動すると、コンピューターが滑走路の飛行機のようにハミングすることがあります。 鼻、時にはアプリケーションから臭いがするので。 足も必要です。時にはあなたに何かが起こり、いくつかの質問が生じ、誰かに走り、何かを尋ね、誰かとコミュニケーションをとる必要があります。 手は、特に手動テスターに​​とっても便利です。すべてを自動化できるわけではありません（そして、私たちは手で自動テストを記述します）。 そして、もちろん、テストはあなたの情熱でなければならないので、心！



これについては、原則として、すでに記事を終えることができ、最も重要なツールはすべてリストされています。 しかし、冗談はさておき、手動テスターであろうと自動プログラマーであろうと、アナリストやプログラマーであろうと、あなたの立場に関係なく、私たちがより効果的になるのに役立つツールについて話しましょう。

セキュリティのテスト



パフォーマンステストだけでなくセキュリティテストも多くの人がバイパスしますが、最も簡単なテストはそれほど難しくありません。 ただし、 XSS Me / SQL Inject Meと呼ばれる、Webアプリケーションで最も一般的な脆弱性の名前のFirefox専用のプラグインがあります。 それらはブラックボックスの原則に基づいて動作します。つまり、自分でプラグインを配置し、ページ上のテキストフィールドを見つけて、そこにあるデータベースにあるスクリプトを貼り付けます。 このデータベースは単なるXMLであり、データで拡張できます。 出力は、特定のテストシナリオが合格、未合格、特定のテストデータが機能しなかった、どのシンボルが問題を引き起こしたかなど、既成のレポートです。 同じシンボルを手に取り、リソースでそれを試して、何がうまくいかなかったかを確認することができます。 ここで、これらのプラグインがHTTPステータスに応じて不十分または十分に色付けされるという事実に注目します。 したがって、200のステータスは大丈夫で、400〜500を取得した場合は緑、302が発生した場合は赤、という答えが緑になります。 レポートでは、これらは赤血球になります。これは、これが予想されたページへのリダイレクトであり、外部ではないことを手動で確認できます。



また、リクエストを傍受したり、ゼロから独自に作成したりできるツールもあります。 私はよくFiddlerまたはCharlesを使用します（彼に関する別の投稿、 habr.com / company / redmadrobot / blog / 269109もありました ）-リクエストを追跡し、変更されたパラメーターで再利用します。 これらのテストはファイルにダウンロードして自動化チームに渡すか、後で簡単に再利用できます。 Fiddlerを使用して、多数の同一のリクエストをすばやく送信することもできます。Shift+ Rホットキーを使用すると、選択したリクエストを何度でも繰り返すことができます。



ブラウザーに類似したものがあります- 改ざんデータプラグイン（FirefoxとChromeの両方で利用可能）：それほど機能的ではないため、長いネットワーク操作などをエミュレートすることはできませんが、最も単純な要求はそれをインターセプトして送信することですそれらをオンザフライで、そこにもできます。 Windowsで作業している場合、私はあなたの代わりにFiddlerを好むという事実に注意を喚起したいと思います。それは常に無料です。 Macでは、Fiddlerはまあまあ機能します。 公式にはサポートされていますが、問題があります。Charlesを使用する方が良いでしょう。公式には30日間しか無料で使用できないことに注意してください。
