世界で最高のプログラミング言語を見つけた方法。 パート1

プログラミング言語のトピックについてトロールをひどく挑発し続けるので、さらに世界で（現時点で）最高のプログラミング言語（絶対に最高、つまり留保なし）に名前を付けます。残りのすべての人は、ほぼ3年前の「プログラミング言語の選択について」という私の投稿に精通している必要があります。そこで書かれたものはすべて関連性があり、繰り返したくありません。



読みましたか？ 次は、世界で最も優れたプログラミング言語についてです。 はい、はい、3日前でもユニコーンが存在するとは思いませんでした-結局、各言語は何らかの形で吸い込まれ、それぞれ独自の方法で吸い込まれます（いや、いつでもどこでも吸うものがありますが、私たちはそれらについて語りません） エンジニアとして、言語の問題では、ガベージコレクションと手動のメモリ管理から始めて、さまざまなトレードオフなしでは不可能であることを完全に理解しましたが、Rust はこの問題でスティックに座らずに魚を食べようとします。 私たちは汎用言語について話していますが、それらはすべて何らかの方法で自分自身を位置づけ、それらが良いアイデア、原則、目標に焦点を当て、そのような原則と矛盾するすべてのものを可能な限り最大限に残しますが、効果的ではありません/または快適ではありません。 一言で言えば、非現実的です。 ただし、不一致を組み合わせた言語はです。







過去に書いたように、私の人生でプログラミング言語を選択する問題は、約7年前にJavaがもはやケーキではないことに気付いた成人期にのみ現れました。 次に、生産性と、今後10年間Scalaをメインオフィス言語にする素朴な欲求を求めて、Odersky et al。による未発表の本の早期アクセス（いくつかの章）を購入しました。 だから私は、Scalaが面接に来る候補者の履歴書や頭に登場する何年も前に自分でScalaを発見しました。 繰り返しになりますが、IDEを10年以上注文している会社にいたので、その中にたくさんの言語の納屋と真珠を見たので、著者に言語の歴史を紹介する前にキャプテンに少し考えるように頼みます-あなたは本当に誰だと思いますか7年間言語を探して見つけることができません。キャプテンの話を知りません。



近年、言語を選択する問題が悪化しているだけです。Xoredで新しい技術革新を開始する必要があり、あらゆる種類のマジックを実行できる「洗練された」技術のアイデアがありますが、私は長い間何も始める準備ができていませんでした。 質問に対する答えは、なぜ私を驚かせます。インタビューで同じような答えを受け取っていたら、子供の本の男にすぐにマイナスを書いていたからです。 答えは本当にばかげて子供っぽいように聞こえます。 私は、どのプログラミング言語に驚くべき技術のスタックを見たいかを知りませんでした 。 唯一の言い訳：私は本当に長い道のりを目指しています-これは単一のプロジェクトではありません。これは私の人生で最高の年を投資したいものであり、問​​題の価格は私にとって非常に高いです。



乗り物を選ぶ際に長くて難しい旅をすることは、間違いを犯すのは危険です。 そして、ここ数年、私は何らかの理由で拒否されることのないプログラミング言語（プラットフォーム）を探していました（待っていましたか？）。 私の計画にかなり壮大なタスクがあるため、このようなプログラミング言語のさまざまな機能要件と非機能要件を提案しました。



必要条件





ブラウザに高性能なコードが必要であり、コードをunikernel / cloud osにプッシュできる機能が必要です。 大量の情報を処理する高性能なサーバー側が必要です。 iOSとAndroidが必要です（当然、高性能でもあります）。 そして私はこれをすべてhemoなしでプログラムしたいです。 コード生成では動作せず、狭いゲートsun.misc.Unsafeを介してメモリで直接動作せず、Fortranで数学者のような大きな配列をプログラムしません。



非常に低レベルのコード（ベアメタルに近い）を作成し、ネイティブC / C ++に匹敵するパフォーマンスを実現したいと考えています。 必要なときにGCが必要であり、不要なときにGCは必要ありません。 オブジェクト指向スタイルで、機能的に書きたいです。 操作、HOF、およびクロージャーをオーバーライドしたい。 ジェネリックが欲しいのですが、将来のコンセプトを実装するためのテンプレートやマクロが欲しいかもしれません。 メモリおよび同時実行モデルから始めて、これらすべてが言語でどのように実装されているかを理解し、これらのことを思い通りに管理できるようにしたいと思っています。



すべてを低レベルにしたいし、Scalaに匹敵するすべてのパワーを高レベルにしたい。 私は脊髄で素早く簡単にコードを書きたい（コンパイラーに考えてもらいたい）。コンパイラーではなく（ScalaやC ++でプログラミングするときのように）。 プログラマーとして生産的になり、たとえばJavaやJavaScriptよりも何倍も速くプログラムを書きたいです。



これは、ウィッシュリストの頭からの素早いダンプです（意図的に書き留めたことはありません）。 しかし、私はすべてを一度に欲しいのは明らかです。そして、私は最初にすべてを一度に言ったのです 。 そのような言語はありません-ビーチから行きます 。 しかし、彼はそうです。



クロスプラットフォームでベアメタルに近い




JavaScriptでコンパイルするクロスプラットフォームが必要です。 そして、JavaScriptでのコンパイルだけでなく、ブラウザでの高性能コード 。 UIとクライアントサーバーインタラクションをプログラムする機能を提供する言語のすべての機能に加えて、クライアントで深刻な数値計算を行う必要があり、1秒間に100万または2つのタプルを粉砕することが非常に望ましい要件です。 クライアントコードに本格的なP2Pネットワークなどが含まれている可能性があります。 要するに、ブラウザ側のコードの要件は、サーバー側のソリューションの要件と同じです。



大量のデータに対して高性能な何かを書くこと。これはJavaとJavaScriptで可能です。 このようなことは、オフヒープデータ構造を積極的に使用することで解決されます。これには、生成された大量のコード（これに起因するall核）またはプログラミングの悲惨な外観が含まれます-あなたは技術的にJavaでプログラミングしていますが、これはお母さんのJavaではありませんが、悲惨です言語サブセット。 ブラウザの世界では-型付き配列とasm.jsを使用した同じストーリー（あらゆる種類のNaCl / PNaClなどを保持します-興味はありません）。



理論的には、scala- scalajs 、go- gopherjs 、およびほとんどのJVMベースのもの（ Fantom 、 Kotlin 、 Ceylon 、私のお気に入り）をJavaScriptでコンパイルし、 オフヒープをエレガントに使用できることを条件に、タスクに多くのものを使用できますバイトコードにコンパイルされたときのデータ、およびそれに応じて、JavaScriptにコンパイルされた場合は型付き配列。



実際、私はこの道をたどりましたが、記事のトピックに戻ると、これは言語に対する道です。 私は言語、またはむしろJVMプラットフォームに対してプログラムしました。これは解決される問題と本質的に直交しており、問題を解決するにはプラットフォームが提供するすべてのものが必要ですが、別の方法でそれを自分で行います。 オフヒープメモリ管理、値型の独自のシステムが必要であり、これらすべてに加えて効率的に動作するには、コード生成（コンパイル時および/または実行時）が必要です。 手短に言えば、疑問が生じます。これらの言語とプラットフォーム（JVM、ブラウザ）が必要なのは、それらが役に立たず、干渉するだけなのにどうしてですか？

ブラウザー、asm.jsおよびemscripten




ブラウザでは、emscriptenは驚異的です。 emscriptenを知らない人のために-例えばQuake3でプレイできます 。移植されたゲーム（など）のリストはこちらです。 私の知る限り、Unityとハイエンドゲームエンジンは、この方法でブラウザーに自分自身をドラッグしようとしています。 通常、Unityはemscriptenを使用してJavaScript（asm.js）での.NET（Mono）ランタイムを含むすべてのトラブルをコンパイルしますが、もちろんこれはあなたが自分の手で書くJavaScriptのようなものではありません-そのようなJavaScript-はコンパイルの結果にすぎません。 JavaScriptのプログラム（設計どおり）がasm.js（大量のデータ処理と大量のデータを意味する）に比べてどこかに近くなるのではないかという深刻な疑念があります。そうであれば、asm.jsを生成します（それに応じて、別のメモリモデル）必要に応じて。
