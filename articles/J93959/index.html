問題



世界のさまざまな国（「ネイティブ」の国/電話のオペレーターを除く）のユーザーは、体系的に同じ問題に遭遇しました。電話はGPRS / EDGE / 3Gモードで動作しない可能性があります。 データ送信はWi-Fi経由でのみ行われました。



同じ運命が私に降りかかった。 MTS.BYネットワーク（および他のローカルネットワーク）でも、SMSを呼び出して送信することしかできませんでした（ホームWi-Fiのおかげで、上で書いたように、palm.comでプロファイルを作成してアクティブにしました）。



さらに、バージョンwebOS 1.3以降、手動でアクセスパラメーター（APN、ユーザー名、パスなど）を指定できる項目が設定に表示されたという事実にもかかわらず、デバイスはそれらの保存を拒否し、警告が発行されました。 バグは、最新バージョン1.4.xに残っています。



フォーラムPreCentral.net （英語を母国語とする最高のオーナーコミュニティ）で実験してしばらくしてから解決策を見つけました。 デバイスには、電話が認識しているすべてのネットワークの説明とそれらへのアクセスパラメータが記載された特別なファイルがあります。 したがって、ご想像のとおり、ベラルーシのGSMオペレーターに関する言及はありませんでした。 さらに、このファイルの内容は、デバイスが対象とする市場によって異なる場合があります。 明らかな決定は、必要なネットワークを追加してこのファイルを編集することでした。



解決策



このため（そして実際、PCで電話を操作するため）、 WebOSQuickInstallをインストールする必要があります（Javaの最新バージョンが必要です）。 キーボードから引用符なしで「upupdowndownleftrightrightleftrightbastart」という魔法の組み合わせを入力して、デバイスを開発モード（Androidのルートの類似物）に転送します。







PreをPCにJust Chargeモードで接続します。 WebOSQuickInstallを実行し、メニューから[ファイルの受信]を選択し、必要なファイル/usr/lib/luna/CarrierNetworkSettings.db3へのパスを指定して、ローカルに保存します。



SQLite Database BrowserまたはFirefox SQLite manager拡張を使用してファイルを編集できます。 念のため元のファイルを保存することをお勧めします。



SQLiteデータベースブラウザーを開き、ドロップダウンメニューの[データの参照]タブで、テーブル..._ CarrierNetworkSettingsを選択します。







リストから必要のない演算子を編集することにしました（上の表の最初の演算子を選択しました）。



編集の本質は次のとおりです。

  • 列mcc （モバイル国コード）に国の番号（ベラルーシ-257、ロシア-250、ウクライナ-255）を示します。
  • 列mnc （モバイルネットワークコード）-モバイルネットワークコード（ベラルーシ：velcom-1、MTS-2、寿命:)-4）;
  • qOperatorShortNameおよびqOperatorLongNameを修正できます。
  • 重要なフィールド： gsmNumSimPDP （値-1）、 umtsNumSimPDP （値-1）。




MMS設定もそこに入力されます。 これには、フィールドmmsApnId 、 mmscWapGateway 、 mmscUrl 、 mmscPortNumberが必要です。 オペレーターの設定に従って入力する必要があります。 mmsApnIdフィールドにはシリアル番号が含まれており、その下にある2番目のテーブル（以下で説明します）にMMS送信のGPRS設定が表示されます。







また、左側のidフィールドを見つけて、その中の13桁の数字を覚えておくか書き留める必要があります。



テーブルのドロップダウンメニュー..._ DataConnectionSettingsに移動します 。



そこで、idフィールドに上記で保存された番号を見つけます。 これは数回発生します（必要な設定の数と数によって異なります）。 写真を説明します。







  • Apnid-アクセスポイントのシリアル番号（任意ですが、リスト全体で一意です。MMSが機能するには、値がテーブルのmmsApnIdフィールドと一致する必要があります..._ CarrierNetworkSettings）;
  • SupportedService-設定が指定されているサービスのタイプ（1-GPRS、2-WAP、4-MMS、5-テザリング）。
  • regionName-通常、アクセスポイントの任意の名前。




編集済み。 保存しました。 電話に再び入力します。 [ファイルの送信]メニューを選択し、変更されたデータベースのファイルを選択します。 初めて/ usr / lib / luna /を指定して、[デバイスに送信]をクリックします。 次に、データベースファイルの選択を繰り返しますが、/ var / luna / data /をパスとして指定する必要があります。



電話を再起動すると、すべてが機能するはずです。



私はそのような困難に直面しなければなりませんでした。 同意します。デバイスを選ぶのが好きな人にとっては、最も楽しいことではありません。 しかし、再び、GPRS-インターネットの欠如に関する問題は選択的です。
