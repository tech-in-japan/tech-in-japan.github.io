Linuxを理解するのが面倒な人のためのスクリプト

多くのLinuxアプリケーションが存在する可能性があります。 特に月額1ドルからVPSをレンタルできるようになったとき。 Webサイトホスティングの標準的な使用に加えて、ゲームのサーバー（CS：GO、Terraria、Minecraft）、プロキシサーバー、VPNサーバーとして使用されます。 暗号通貨マイニングの下。 バックアップバックアップストレージ用。 家の急流ロッキングチェアの下。 また、テスト、開発、およびさまざまな実験にも使用できます。 Linuxが人気を博したのは、LinuxベースのVPSが利用できることであり、さまざまな用途が考えられます。 しかし、Linuxの使い方を知っている人よりもLinuxを使いたい人の方が多いです。 そして、多くの場合、人々がLinux管理を使用できないようにするのはLinux管理に関する弱い知識です。 まあ、あるいはそのような人々の生活を複雑にしている-彼らはマニュアル、フォーラム、「初心者向けの記事」を選ぶのに何時間も費やさなければならない。



はい、私自身は、すでに学んだことに関して左へ、または右への一歩を踏み出す必要があるたびに、専門のフォーラムに登るのにうんざりしています。 だからこそ、私はすべての一般的なことを1つのスクリプトにまとめて、自分ですべてを実行できる使いやすいインターフェイスを備えているのです。 すべてが小さく始まった。 スクリプトは、必要なソフトウェアのインストールを自動化しました。 しかし、半年間、すでに100 Kbを超える非常に深刻なユーティリティになっており、その中で約2.5千行のコードが使用されています。 スクリプトはこれまでにレイアウトされたことがありません。 私と数人の同志が個人的な目的でのみ使用します。 それを一般に公開する時が来ました。 彼は多くの人々のために多くの時間を節約できると確信しています。

OSを操作する



このセクションには、ルートパスワードの変更、タイムゾーンの設定、OSの更新、リポジトリの追加、一般的なアプリケーション（midinght司令官など）のインストールなど、いくつかの小さなユーティリティが含まれています。



  • アンチウイルス 。 はい、Linuxにはウイルスはありません。 しかし、サイトを通じて攻撃者にすべてのファイルへのアクセスを許可するバックドアがあります。 これらのバックドアを介して、スパマーはサーバーを使用してスパムメールを送信できます。 かつて私はこのVPSを禁止されました。 したがって、アンチウイルスのインストールを提供しました。 理解する必要はありません。 彼はすべてを更新します。 さらに、ウイルス対策の構文を理解しなくても、スクリプトを使用して、ディスク全体または特定のフォルダー（サイトがホストされている場所など）を確認できます。



  • ファイアウォール 誰もがファイアウォールルール（iptables）を構成する方法を知っているわけではありません。 また、一部のリースVPSでは、インストール済みのサービスが利用できないように、事前に構成および有効化されています。 そして、設定を掘り下げてポートを開く必要があります。 また、一般的に、ファイアウォールの設定が正しくない場合は、原則として、SSHを介してサーバーに接続できず、OSを再インストールするか、Webターミナルを介してサーバーに登る必要がありますが、この機能はそうではありません。すべてのホストで。 私のスクリプトを使用すると、このような問題は解消され、ファイアウォールは「アシスタント」を介して設定されます。「アシスタント」では、簡単な質問に答えるだけで済みます。



  • タスクスケジューラ（cron） 。 ウイルスのチェック、ログのクリーニング、バックアップのアップロードなど、いくつかの典型的なアクションを定期的に実行する必要がある場合があります。 Linuxには、スケジュールされたタスクを実行できるスケジューラがあることを多くの人が知っていますが、誰もがそれを使用する方法を知っているわけではありません。 私のスクリプトでは、起動間隔を選択することで、スケジューラを設定、有効化、無効化し、タスクを追加できます。


ホスティングコントロールパネルのインストール



多くの場合、Linuxはホスティング専用に使用されますが、Apache、Nginx、PHP、MySQL、メールデーモンなどのすべてのサービスを自分で設定することは初心者にとって簡単ではありません。 ほとんどの場合、何らかのコントロールパネルをインストールすることを好みます。 しかし、それでも最初に何らかの方法でインストールする必要があります。 私のスクリプトでは、5つの無料のサイトコントロールパネル（Vesta CP、Webuzo、CWP、ZPanel、Ajenti）と有料のISPmanager（おそらくロシアで最も一般的なパネル）を収集しました。 各パネル、システム要件についての簡単な説明があります。 必要なパネルを選択すると、スクリプトはそれを公式サイト（最新バージョン）からダウンロードしてインストールします。



VPNを使用する



最近、他の地域のユーザーの利益を得るために、多くがVPNを開始しました。 たとえば、RosKomNadzorの禁止を回避したり、アメリカやヨーロッパの社内サービスを使用したりするなど。 多くの人はこのために既製のVPNサービスを購入します。 ただし、適切な国でVPSを購入し、独自のVPNを作成する方がはるかに安価です。 しかし、誰もがそれを設定する方法を知っているわけではありません。 このスクリプトを使用すると、質問に答えるだけで済みます。 必要なソフトウェアはすべて単独でインストールされ、必要なルールはファイアウォールに書き込まれます。 アクセスを許可されているユーザーを表示、追加、削除できます。 さらに、スクリプトは、使用しているOSを分析し、このOSの特定の機能を考慮してすべてを実行します。



プロキシを使用する



VPNの代わりにプロキシを使用する可能性が高いものもあります。 プロキシは、VPNとは異なり、独自の専用IPアドレス（NATの背後にある）を持たないサーバーで使用でき、そのようなサーバーは数倍安くなります（2ドルで購入できるため） 年 ）。 プロキシサーバーをLinuxに上げることも、初心者にとって簡単な作業ではありません。 ただし、このスクリプトはすべてを行います。 そして、多くの設定があります。 インストールアシスタントは、プロキシが必要なポート（または標準ポートを提供）を尋ね、ログイン/パスワードによる認証が必要かどうかを尋ねます（または、アドレスとポートを知っている人に許可します）。 このスクリプトでは、サードパーティのトラフィックを（Proxifierなどのプログラムを介して）プロキシに送り込み、必要に応じて構成を構成したいユーザーのニーズも考慮されます。 もちろん、彼自身がファイアウォール（iptables）で必要なすべてのルールを作成します。 プロキシの設定はこれまでになく簡単になりました。 手を設定に登る必要はまったくありません。



ファイルとプログラムを操作する



このセクションでは、目的のプログラム（パッケージ）をインストールまたはアンインストールできます。 さらに、最も便利なのは正しい削除です。 実際には、プログラムをインストールするとき、その操作に必要ないくつかの関連パッケージがインストールされることが非常に多くあります。 多くの場合、これらの追加パッケージのサイズは、インストールした実際のプログラムのサイズを超えています。 ただし、プログラムを削除すると、プログラム自体のみが削除され、プログラム（およびプログラムのみ）が動作するために必要なすべてのパッケージが残り、スペースを占有し続けます。 私のスクリプトでは、プログラムを完全に削除すること、およびインストールされたすべての追加ソフトウェアを提供しました。 スクリプトでは、単にプログラムの名前を指定するだけで、スクリプトはそれ自体ですべてを実行します。



システムクリーニング



ホスティングにサーバーを積極的に使用する場合、多くの場合、サイトへの膨大な量のアクセスログが蓄積されます。 このすべてのゴミの量は、ギガバイトに達することがあります。 誰もがそれをどこでどのように削除するかを知っているわけではありません。 このセクションでは、これらのログを消去できます。 ApacheとNginxのログは、ユーザー全体と特定のユーザーの両方で削除されます。 さらに、このセクションから古いインストールパッケージを削除できます。これは、ソフトウェアのインストール後もデフォルトでディスクに残り、引き続きスペースを占有します。





これについては、実際のところ、今のところ。 もちろん、計画にはさらに追加する必要があります。 たとえば、さまざまなサーバーコンポーネントのパフォーマンスを評価するためのベンチマーク。 他のOSなどのサポートを拡張します。 私の能力と時間を最大限に活用して、実際にこれを行います。 しかし、すでに今では、初心者が要求する非常に多くの機能を実行し、彼らの生活を大幅に簡素化することができます。



さて、実際、非常に重要な質問です。 また、どのLinuxディストリビューションですべて動作しますか？ CentOSのすべてのバージョン。 また、Red Hat Enterprise Linuxディストリビューションの他のすべてのディストリビューション（Scientific Linuxなど）。 なぜRHELなのか？ まあ、私は自分でスクリプトを書きましたが、CentOSは私に近いです。 さて、このスクリプトの対象読者（初心者）について話す場合、原則として、どのディストリビューションで作業するかは関係ありません。 結局のところ、通常、彼らはそれらのどれにも等しく不十分に気づいています。 また、RHELは非常に優れた安定したリソースを必要とせず、最も重要なこととして、ほとんどすべてのVPSホストに存在します。 同時に、CentOSの任意のバージョン（5、6、7）、およびビット深度を選択できます。 すべてのアクションについて、スクリプトは配布パッケージのバージョンを分析し、この特定のバージョンの詳細を考慮します。 私は通常、64ビットのビット容量を持つCentOS 6.xディストリビューション（可能な限り最小）を選択します。



まあ、最も重要なこと。 このスクリプトを実際に入手する方法は？ ここでは、ソースコード全体を見ることができます。 また、プロジェクトはgithubに投稿されます。 リンク： github.com/Brizovsky/Breeze-Easy-Shell



使用する必要があるのは、[name] .shファイルを作成し、その中にコンテンツを入れることだけです。 次に、コマンド「sh [name] .sh」を実行します。 サーバーとコンピューターの両方で作成し、サーバーにコピーできます。 別のインストールオプションがあります-githubから直接サーバーで圧縮します。 個人的に、私は次のようにします： 

cd /root/
wget https://raw.githubusercontent.com/Brizovsky/Breeze-Easy-Shell/master/breeze.sh -r -N -nd
cat >> /root/.bashrc <<END
alias breeze='cd /root/
sh breeze.sh'
END
exit
