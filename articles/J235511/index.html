一緒に楽しもう：python + flask + google app engine

背景：私は、Habrahabrの多くの人と同様、音楽を聴くのが大好きです。 ほとんどの場合、VKontakteでこれを行います。 既に作成されたプレイリストは面倒な場合があります-新しいものが必要です。 しかし、それは既にそこにあるものとあまり変わらないように。 このため、音楽を聴く機会を提供するすべてのサービスに関する推奨事項があります。 率直に言って、VKに関する推奨事項は私にとって恐ろしいものです。 たぶん誰かがそれを好むかもしれませんが、そこでの私の選択はブラウザを閉じたいと思うようにします（もちろん私見）。 一般に、私は自分でこの状況を修正し、VKontakteおよびLast.fm APIを使用して独自の推奨事項を実装することにしました。 私はGoogle App Engineについて多くのことを聞いて読んだが、一度も使ったことがないので、このプラットフォームに参加することにしました。



ここでは、APIまたは推奨事項を選択するアルゴリズムとのやり取りの複雑さについては説明しません。 それについて-多分、次の記事で。 この記事では、PythonとFlaskを使用して、Google App Engineプラットフォームでアプリケーションを作成、構成、デプロイするプロセスのみを説明します。

タイプミスや不正確さについては、午後にお知らせください。 ありがとう



それでは始めましょう。 appengine.google.comにアクセスします。 ここで、まだ認証されていない場合は、Googleアカウントのユーザー名/パスワードを入力するよう求められます。



認証に成功すると、「アプリケーションの作成」ボタン（またはアプリケーションがある場合はアプリケーションのリスト）が表示されます。 私の知る限り、追加の投資をせずに、Googleは最大25個のアプリケーションを作成することができます（間違っている場合は正しい）。 他のクォータのリストは、 リンクをクリックして見つけることができます。



ボタンをクリックすると、アプリケーション識別子（まだビジーではない）と名前、および認証オプションを選択できるフォームが表示されます。 利用規約に同意し、ボタンを押して最初のアプリケーションを作成します。 作成された、ある種の喜び。 アプリケーションはhttp：// [identifier] .appspot.comで利用可能になりました。ここで、 identifierは一意のアプリケーションIDです。 アプリケーションは使用可能ですが、もちろん動作しません-動作するものは何もありません。 状況を修正してください。



必要に応じて、 PyCharmをダウンロードします。 エディターのバージョンの選択には若干の微妙な違いがあります。ProfessionalEditionであるPyCharmは、IDEに統合された展開ツールで表現されるGoogle App Engineの組み込みサポートを備えています。 PyCharm CEにはそのようなサポートがないため、コンソールから展開する必要があります。



PyCharmでプロジェクトを作成します。 Professional Editionを使用している場合、プロジェクトを作成するときに、「Google App Engineプロジェクト」タイプを選択できます。 この場合、アプリケーションの識別子（上記の識別子）とApp Engine SDKへのパスを指定する必要があるため、事前にダウンロードしてください。 サポートされているプログラミング言語のSDKをここから入手してください 。 PyCharmは、既に完成したapp.yaml構成ファイルと動作するmain.pyスクリプトを使用してプロジェクトを作成します。



実際、メインスクリプトは次のとおりです。

main.py 
import webapp2 class MainHandler(webapp2.RequestHandler): def get(self): self.response.write('Hello world!') app = webapp2.WSGIApplication([ ('/', MainHandler) ], debug=True)


この例は、Python自体のGoogle App Engine マニュアルとJetBrains Webサイトの両方で提供されています。 スクリプトには注目すべき点はありません-標準のHelloWorldです。 webapp2は、Google App Engineと互換性があり、非常に使いやすい軽量なフレームワークであることを知らない人にだけ言います（私が使用した限り）。



構成ファイル：

app.yaml 
 application: pygask version: 1 runtime: python27 api_version: 1 threadsafe: yes handlers: - url: /favicon\.ico static_files: favicon.ico upload: favicon\.ico - url: .* script: main.app libraries: - name: webapp2 version: "2.5.2"


設定を分析しましょう：

  • application-アプリケーション識別子（「pygask」という単語を混同しないようにしてください-これはpython + gae + flaskの省略形です:)）;
  • ハンドラー-URLパターンのリストとそれらの処理方法の説明。 ハンドラーディレクティブでは、静的ハンドラーとスクリプトハンドラーの2種類のハンドラーを使用できます。 この場合、静的はfaviconローダーで表され、スクリプトはmain.appアプリケーションを起動するためのメインスクリプトです（注意！ .py



    拡張子は必要ありません）。
  • ライブラリ -アプリケーションが機能するために必要なライブラリ。 この場合、webapp2。 Google App Engine上で実行されるPython 2.7は、すぐに使用できるライブラリをサポートしています。 完全なリストはここにあります 。


Google App Engineアプリケーションの設定の詳細な説明はこちらです。



一般に、最初のアプリケーションは動作をテストする準備ができています。 PyCharmでプロジェクトを実行します。 ローカルマシンでは、 127.0.0.1 ：8080で利用できます（プロジェクト構成で何も変更していない場合）。 開くと、「Hello world！」と表示されます。すべてが機能しています。



PyCharmを使用していないのに、プロジェクトを開始したい場合はどうしますか？

Linuxの方法：ターミナルで、次の形式でコマンドを実行します。 

 <path to python interpeter> <path to SDK>/dev_appserver.py --host 127.0.0.1 <path to project>
説明のために、コンソールでpygaskを起動するために使用するコマンドを指定します（SDKとプロジェクトフォルダーの両方が/var/www/



）。 

 andymitrich@pc:~$ python /var/www/google_appengine/dev_appserver.py --host 127.0.0.1 /var/www/pygask/
ターミナルで似たようなものを入力して127.0.0.1:8080を開き、できれば「Hello world！」を参照してください。



Windowsユーザーに関しては、すべてが少し異なります。 SDKがダウンロードされてコンピューターにインストールされ、Google App Engineランチャーがプロジェクトの起動に使用されます。 Mac OSについては何も言えません-誰かがコメントであなたに言うかもしれません。



残念ながら、webapp2フレームワークは必要なものではありません。 したがって、フラスコを接続しましょう。



フラスコはデフォルトで動作するライブラリのリストに表示されないため、まず、その接続に注意する必要があります。 Google App Engineは、使用するモジュールを個別に構成する機能をサポートしています。 これはappengine_config.pyというファイルで行われます。 ここで彼についての詳細。 フレームワークを接続するために使用します。 これを行うには、pipを使用して、フラスコを内部ディレクトリにインストールし、 libと呼びます。 手動で行う 

 andymitrich@pc:~$ pip install -t /var/www/pygask/lib/ flask
または、 requirements.txtファイルを使用します 
 andymitrich@pc:~$ pip install -r /var/www/pygask/requirements.txt -t /var/www/pygask/lib/
-t



オプションに注意してください-パッケージをインストールするパスが含まれています。



Flaskがインストールされ、使用できます-設定とメインスクリプトを少し書き換えます：

  1. app.yamlアプリケーション設定から、 libraryディレクティブを使用してセクションを削除できます-不要です。
  2. メインスクリプトは次のようになります。

    main.py 
     from flask import Flask app = Flask(__name__) @app.route('/') def home(): return 'Hello world!'


アプリケーションがインストールされたフラスコを取得するために、 appengine_config.pyファイルでその場所を示します。

appengine_config.py 
 import site import os.path site.addsitedir(os.path.join(os.path.dirname(__file__), 'lib'))


私たちは始めようとします-私たちはすべて同じ「Hello world！」を見ます。



プロジェクトを多かれ少なかれ見栄えのするものにします。アプリケーションパッケージ、静的ファイルとテンプレートのフォルダーを追加し、ハンドラーとビューを別のファイルから削除します。 通常のフラスコプロジェクトを構築しているかのように動作します。 すべての操作の後、プロジェクトツリーは次のようになります。



さらに、任意の方向で開発できます。



元のファイルがどのように変更されたかに注目します。

  • 現在、1行で構成されているメインスクリプト

    main.py 
     import application
  • プロジェクト構成ファイル

    app.yaml 
     application: pygask version: 1 runtime: python27 api_version: 1 threadsafe: yes handlers: - url: /favicon\.ico static_files: favicon.ico upload: favicon\.ico - url: .* script: main.application.app


構成ファイルで、メインスクリプトへのパスが変更されました-アプリケーションパッケージの表示が追加されました。



表示された「Hello world！」は初めてだとしましょう。 appspot.comにデプロイしましょう。 実際、ここでも複雑なことは何もありません。 ただし、開始する前に、重要なことを1つ言いたいと思います。開発プロセスには、サーバーには不要な多くの作業ファイルがあります。 デプロイメント中にそれらがそこに到達しないようにするために、アプリケーション構成のskip_filesディレクティブで、レイアウトすべきではないすべてのものを指定できます。 詳細



あなたが幸せなPyCharmユーザーであれば、「最も難しい」操作を行う必要があります：ツール-> Google App Engine-> App Engineアプリのアップロード...次に、これが最初の展開である場合、あなたに合った認証方法を選択します（メール/パスワードのペアを理解できませんでした-なぜこのデータは適合しなかったので、OAuth2を使用しましたが、気にしませんでした）、できればhttp：// [identifier] .appspot.com （この場合はpygask.appspot.com ）にアクセスして、作業の結果を確認します。



何らかの理由で上記のIDEを使用しない場合、レシピは1行になります（Linuxの方法）。 

 <path to python interpeter> <path to SDK>/appcfg.py <path to project>
したがって、私は使用します： 

 andymitrich@pc:~$ python /var/www/google_appengine/appcfg.py --oauth2 /var/www/pygask/


--oauth2



パラメーターに注意して--oauth2



を介した承認に必要です。 指定しないでください-メール/パスワードの入力を求められます。 チームが正常に完了した場合、プロジェクトは適切なアドレスで利用できます。
