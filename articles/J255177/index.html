CentOS 7（1503）が利用可能になりました。 新機能とアップグレード方法

最近、 CentOS 7.1と呼ばれていたCentOS 7（1503）がすべてのユーザーに利用可能になりました。







更新するには、CentOS 7でサーバーに接続し、ターミナルで次のコマンドを入力します。 



yum clean all && yum update


この記事では、主要なOSアップデートの新機能について説明します。



大きな変更



  • 自動エラー通知ツール（ABRT）は、bugs.centos.orgに直接問題を報告できるようになりました
  • Parallel NFSの拡張サポート
  • 再起動なしの更新や新しいMCSロックなど、OSカーネルの多くの改善
  • 改善されたロードプランナー
  • 提供される新しいハードウェアサポート
  • LVMキャッシュが完全にサポートされるようになりました
  • Ceph Distributed Failover File Systemブロックデバイスをマウントできるようになりました
  • OpenJDK 1.8.0の完全サポート
  • 時計の安定性の向上
  • OpenSSH、Docker、Network Manager、およびThunderbirdの更新バージョン
  • 更新されたネットワークデバイスドライバー
  • プレビューによるBtrfs、OverlayFS、およびCisco VICドライバーファイルシステムの追加方法


いくつかの興味深い革新を見てみましょう。



Intel Broadwellのサポート



第5世代Intelプロセッサー（Broadwell）のサポートが追加されました。 turbostatユーティリティ（カーネルの一部–ツール）は、新しいプロセッサをサポートするように更新されました。 また、 microcode_ctlパッケージのIntelプロセッサーのマイクロコードが更新されました 。



LVMキャッシング



LVMキャッシングが完全にサポートされるようになりました。 この機能により、大きくて遅いデバイスをキャッシュするために、小さい高速I / Oデバイスで論理ボリュームを作成できます。



InfoboxCloudは 、すべての仮想サーバーに対してクラウドインフラストラクチャレベルでエンタープライズSSDキャッシングを既に実装していますが、まだ専用サーバーを使用している場合は、この機能が役立ちます。



オーバーレイ



OverlayFSファイルシステムを使用すると、ユーザーは1つのファイルシステムを別のファイルシステムの上で操作できます。 変更は上部のファイルシステムに書き込まれ、下部のファイルシステムは変更されません。 これにより、コンテナなどのファイルシステム内のイメージを共有したり、イメージが読み取り専用（DVD-ROMなど）の場合に複数のユーザーが共有したりできます。



現在のプレビューには制限があります。

  • サポートされていない場合は、ext4、xfs、gfs2ファイルシステムを使用することをお勧めします
  • SELinuxはサポートされていないため、OverlayFSを使用するには無効にする必要があります。


Parallel NFSの拡張サポート



パラレルNFS（pNFS）はNFS v.4.1標準の一部であり、顧客がストレージデバイスに直接かつ並行してアクセスできるようにします。 pNFSアーキテクチャにより、NFSサーバーのスケーラビリティとパフォーマンスが向上します。



pNFSは、ファイル、オブジェクト、ブロックの3つの異なるストレージレイヤーを定義します。 以前は、ファイルレイヤーのみがサポートされていました。 このバージョンから、ブロックとオブジェクトが完全にサポートされます。



将来、新しいストレージレイヤーがサポートされる予定です。



OSカーネルの改善



Cephブロックデバイスのサポート


libceph.koおよびrbd.koモジュールがカーネルに追加されました。 これらのRBDカーネルモジュールにより、LinuxホストはCephブロックデバイスをディレクトリにマウントし、標準のext4またはXFSファイルシステムでフォーマットできる通常のドライブとして認識できます。 CephFSモジュール（ceph.ko）は、このアップデートではまだサポートされていません。



Kpatch：再起動せずにOSカーネルを更新


CentOS 7（1503）では、技術プレビューとしてkpatchが導入されています。 このテクノロジーにより、再起動せずにOSカーネルを更新できます。 kpatchユーティリティを使用すると、ユーザーはカーネルパッチを管理できます。



ロードバランサーのパフォーマンスを改善する


以前は、ロードバランシングスケジューラコードは、すべての空きCPUの間でバランスを取りました。 これで、負荷を分散できる場合にのみ、分散が実行されます。 パフォーマンスにプラスの影響を与えるスケジューラーの最適化も実行されています。



新しいMCSロック


Cent OS 7（1503）は、 スピンロックの過負荷を軽減し、より効率的にする新しいMCSロックのロックメカニズムを導入しています。



多くのセキュリティ問題が修正され、他のOSの改善が導入されました。 変更点の完全なリストはここにあります 。



更新されます。 この改善により、 サーバーの安全性、高速性、安定性が向上することを願っています 。



使用成功！
