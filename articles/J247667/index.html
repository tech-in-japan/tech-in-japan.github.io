はじめに



このガイドの目的は、 Unity3dでレースゲームを作成する方法を示すことです。 3Dモデル、スクリプト、およびコンポーネントから車を作成します。 既製のシーンをプレイして学習できる本格的なレーシングシミュレータープロジェクトを提供します。 また、このガイドを使用して練習できる車を除いて、ゲームの準備が整ったシーンも提供します。

パート1：車の組み立て



Unity3d 4.6のエラーを修正 したプロジェクトフォルダーまたはプロジェクトへの代替リンクを含むアーカイブをダウンロードします 。 「 CompleteScene 」というシーンを開くことから始めます。 このシーンにはチューニングされた車があります。 再生ボタン（再生）を押して再生してみてください。 これにより、これから行うことの最終結果が表示されます。







再生が終了したら、 「TheTrack」というシーンを開きます。 このシーンには、レーシングシミュレーターに必要なすべてのものが含まれていますが、最も重要な部分は車です。



次に、車のモデルをシーンにドラッグします。 プロジェクトビューでは、 Models / Car / catamountにあります。



車をどこにドラッグしたかに応じて、おそらくその位置をより適切な位置に変更する必要があります。

インスペクターで車の位置を（860、102.3、878）に変更し、 Y回転（回転）を130に設定すると思います。



チュートリアルの残りの部分では、このゲームオブジェクトは「車」と呼ばれるため、ここで名前を変更できます。








これは単なる3D車モデルです。 インスペクターを見ると、車の「ボディ」、車の窓、車輪など、多くの子オブジェクトが含まれていることがわかります。 また、車のコライダーに使用する2つの単純なグリッドも含まれています。 心配しないでください、それらは現在表示されていません。これを変更します。 車がどのように構築され、 階層内でどのように相互に関連しているかを研究する時間を費やしてください。



いくつかの理由から、自動車用に独自のレイヤーを作成しました。 ゲームオブジェクト「Car」を見つけ、インスペクターに移動して、ドロップダウンメニューのレイヤーで「Car」を選択します。 コンテキストメニューで、[ はい、子オブジェクトを変更（はい、子を変更 ）]をクリックして適用します。変更は車の階層内のすべてのオブジェクトに適用されます。



Carゲームオブジェクトには、デフォルトでインストールされるアニメーションコンポーネントがあります。 これは使用するものではないので、先に進み、 Animation Componentの右側にある小さなホイールをクリックして「Remove Component」を選択し、少しクリーンアップしましょう。



衝突を追加する







次に、シーンがアクティブなときにステージから落ちないように、車の衝突コンポーネントを設定します。 車の実際のグリッドに基づいて複雑なグリッドをコライダーとして使用する代わりに、車の上部と下部に対応するはるかに単純な2つのグリッドを作成しました。 パフォーマンスを改善するために、これらのグリッドを衝突モデルとして使用します。



• 「車」オブジェクトの子として配置されているCollider_Bottomゲームオブジェクトをクリックします。

• コンポーネント/物理メニューに移動し、ゲームのオブジェクトに追加するMeshColliderをクリックします。

•新しく追加されたMeshColliderコンポーネントのドロップダウンセレクターで[ マテリアル]をクリックし、 物理マテリアルを選択します。

•2つの正方形にチェックマークがあることを確認します（チェックボックス） 'Smooth Sphere Collisions'および'Convex'

•コライダーは表示されないため、ゲームオブジェクトからMeshRendererおよびMeshFilterコンポーネントを削除します（コンポーネントの右側にある小さな歯車をクリックし、[ コンポーネントの削除]を選択します。

• Collider_Topゲームオブジェクトについて上記と同じことを行います。

自動車部品の追加



それでは、車（ 「車」ゲームオブジェクト）の適切な操作に必要なコンポーネントの追加を始めましょう。







まず、物理シミュレーションを使用して世界の車の位置を制御する方法が必要です。これには組み込みのRigidbodyコンポーネントが理想的です。 「車」ゲームオブジェクトを選択し、[ コンポーネント ]メニューに移動して、[ 物理 ] / [ RigidBody ]を選択します。 プレハブ接続が失われたことを通知するUnity3dメッセージを確認し、ゲームオブジェクトのコンポーネントの1つとしてRigidbodyコンポーネントがどのように接続されるかを確認します。



•車の重量は明らかに1キログラムを超えているため、 リジッドボディの質量を1500kgなどのより現実的なものに変更することから始めましょう。

•次に、 angularDragプロパティを変更する機能があります。これらは、 Rigidbodyの速度と回転を遅くする力です。 スクリプトを使用して車のドラッグを制御するため、 angleDragプロパティを0に設定します 。

• 「scripts / JavaScripts」フォルダーでCar.jsスクリプトを見つけて、 「Car」ゲームオブジェクトにドラッグします。 このスクリプトは自動車の「エンジン」です 。このマニュアルの最後で、スクリプトの詳細を説明することに焦点を当てます。



コンポーネントスクリプト「Car.js」には、自動車のセットアップを開始するときに説明するさまざまな変数があります。コードを学習するときに、詳細を説明します。 ここで、トラックで車を運転できるようにするために必要なものを設定したいだけです。







まず、 車のスクリプトは車の車輪について知っている必要があります。 インスペクターでは、 前輪（Front Wheels）と後輪（Rear Wheels）が表示されます。名前の左にある小さな矢印をクリックすると、両方が展開できることに注意してください。



•前輪と後輪のサイズを2に設定し 、2つの前輪と2つの後輪のためのスペースを残します。

• インスペクターでWheelFL、WheelFR、WheelRL、WheelRRを展開します。 それぞれにブレーキディスク（DiscBrake）が子供であり、各ブレーキディスク（DiscBrake）に子供がいることがわかります。

• DiscBrakeFLとDiscBrakeFRを、スクリプト"Car"の （ 前輪）前輪の 2つの空きスロットにドラッグアンドドロップし、DiscBrakeRLとDiscBrakeRRを（後輪）後輪のスロットにドラッグアンドドロップします。

•質問するかもしれません-なぜディスクブレーキがホイールにかかるのですか？ そして、説明は簡単です。親ゲームオブジェクトとしてのディスクブレーキなので、ホイールとしてディスクをインストールするとタイヤが含まれます。



ブロブシャドウの追加







指向性ライトは、プロジェクトで影が許可されている場合、車の道路上に良好なブロブシャドウを作成します。 また、この画像に示すように、車の下に影を追加します。



これを行うには、車の真下の道路に車の形に似たブロブシャドウを投影するプロジェクターを使用します。 プロジェクタは、実際のプロジェクタのように機能するコンポーネントに組み込まれています。 転送するテクスチャを指定し、プロジェクターの設定と（ターゲットまでの距離）に基づいて、影はプロジェクターの光の経路にあるオブジェクトのテクスチャに基づきます。



•階層で、空のゲームオブジェクトを作成し、それを「車」の車にドラッグして、 「車」のゲームオブジェクトの子にします。

• Blobシャドウプロジェクターに名前を付ける

•プロジェクターコンポーネント（プロジェクター）をこのプロジェクターのBlobシャドウに追加します（コンポーネント->レンダラー->プロジェクター）

•プロジェクタをニアクリッププレーン= 0.1、ファークリッププレーン= 50、視野= 30に設定します。

• Blob_shadowマテリアルをマテリアルスロットに割り当てます。

•[ レイヤーを無視]で [ すべて]を選択し、 [ 道路]レイヤーのチェックを外します。その結果、プロジェクターは道路にブロブシャドウを投影します。

• BlobShadowController.jsスクリプトを追加します（プロジェクトウィンドウのScripts / CSharpScriptsフォルダー内）



プロジェクターのコンポーネント（ Projector ）の位置と回転は、非常に単純なBlobShadowController.jsスクリプトのフレームごとに更新されます。 つまり、車の10メートル上にあり、車の回転に基づいて回転座標を取得します。 道路上の車のブロブシャドウを別のものに見せたい場合は、スクリプトを見て値を変更してみてください。



タイヤマーク







車の運転プロセスをさらに楽しくするもののいくつかは、車をトラックの角で滑らせたり、高速でUターンすることです。 車輪が道路上を滑る効果を高めるために、車が滑るときに表面にタイヤ跡を追加します。



そのため、シーン内のすべてのタイヤトラックを制御する既存のスクリプトを追加する場合、タイヤトラックを確立します。 このコントローラーは、タイヤトラックを表すグリッドを作成します。 各ホイールは、その位置と、スリップするかどうかを知っています。 この情報を使用して、 スキッドマークコントローラーに関連してスキッド マークを設定する世界のポイントを登録します。 複数のホイールからのタイヤトラックが混同されないように、各ホイールが前のスキッドマークポイントをトラッキングできるようにします。



プレハブを作成しました。このため、ステージにドラッグするだけです。



• SkidmarksプレハブをPrefabs-> VFX-> Skidmarksからシーンにドラッグアンドドロップします。



作成した各頂点の色を調整して、タイヤトラックの強度を設定します。 強度は、スキッドの強さによって決まります。 この効果を作成するために、ブレーキトレースをレンダリングするときに各頂点のアルファを使用するカスタマイズされたシェーダーを使用します。 車のように、あなたが構築しようとしている車にタイヤトラックをより良くフィットさせるために変更できることがいくつかあります。



最大タイヤマーク



Max Marks変数は、いつでもシーンに存在できるタイヤトラックの位置の数を決定します。 すべての車輪は同じタイヤトラックを使用するため、この数値はすべての車輪で計算されます。 タイヤトラックが最大数に達すると、最初の位置が上書きされます。 この数を増やすと、シーン内のタイヤトラックのすべての部分の全長も長くなります。



タイヤ幅



変数（ Mark Width ）は、タイヤトラックの幅を決定します。 作成された車両の車輪に合うように調整する必要があります。 これが大型トラックの場合、必要な非常に幅の広いスキッドマークを作成しようとします。スピードレコードを設定するために極薄タイヤを使用した「超音速車」の場合、非常に薄いスキッドマークがより現実的に見えます。



地上変位



タイヤトラックグリッドが作成されると、車輪から計算されたドットを使用して作成され、道路上にスキッドマークが配置されます。 これらのポイントは、ほとんどの場合、ホイールスリップ中に表面上で直線になります。 2つのグリッドがすぐ上にあると、ゲーム内でちらつきが発生する可能性があります。



これを防ぐために、 Ground Offset変数が導入されています。 メッシュが作成され、オフセットの全長がサーフェスに垂直な方向に上がります。 そのため、ちらつきがあったり、タイヤの跡が表面と交差する場合は、オフセットを増やしてみてください。 タイヤトラックが地球の表面に残るようにするため、シェーダーがタイヤトラックに使用され、オフセットが作成され、別のレンダリングラインに割り当てられます（レンダリング） 。



最小距離



SkidmarkControllerに新しいポイントを追加するときは、最後のスキッドマーク（タイヤトラック）からの最小距離（最小距離）でなければなりません。 これにより、非常に小さな領域にタイヤマークを生産的に追加できます。 この値を大きくしすぎないでください。ただし、これによりタイヤマークが大きくなりすぎる可能性があります。さらに、距離の増加は車輪の後ろにあるように見える場合があります。 タイヤマークが車輪の後ろに表示されます。 値を小さくすると、タイヤトラックがある程度滑らかになり、タイヤトラックがホイールに「近い」という錯覚が生じます。



テクスチャー



トラックの車輪のタイヤの質感は、 フォーミュラ1（式1）の車輪のタイヤの質感と比較して大きく異なります。 タイヤマーク（スキッドマーク）の テクスチャが表面に作成されます。 非常に異なるホイールで異なるタイプの車を作成した場合、 タイヤマークをホイールに合わせるには、このテクスチャを変更する必要があります。
