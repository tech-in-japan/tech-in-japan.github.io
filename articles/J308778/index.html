Windows Server 2012、2008、および2003：手頃な価格のシステムパフォーマンステスト

8月29日月曜日、お客様からの多数のリクエストに応えて、 UltraVDS仮想サーバーにインストールするレガシーオペレーティングシステムを選択する機能を追加しました。 過去の世代のオペレーティングシステムに関心がある理由は完全に異なります。習慣の強さ、ソフトウェア開発者の特定の要件、またはリソース消費の削減です。



仮想サーバーリソースの消費量が少ないOSを選択する理由は非常に理解しやすいものです。最新のオペレーティングシステムを「フィード」するためにお金を払っても、その高度な機能は使用されません。



この記事では、リースされたVDSサーバーに古いOSをインストールした場合に、追加のリソースをどれだけ取得できるか（そしてそれがまったく可能かどうか）を検討することにしました。



このテストには64ビットオペレーティングシステムが含まれます。



  1. Windows Server 2012 R2
  2. Windows Server 2008 R2 SP1
  3. Windows Server 2003 R2 SP2


これらのすべてのオペレーティングシステムを、2つのCPUコア、2 GBのRAM、40 GBのHDD構成の「プレミアム」料金プランの同一の仮想サーバーにインストールしました。



PerformanceTestソフトウェアを使用して、プロセッサパフォーマンスを測定する5つのテストをそれぞれ1時間間隔で実施しました。

表は、浮動小数点計算（CPU浮動小数点演算）のパフォーマンスの全体的な結果を示しています。 ソフトウェア設定のテスト期間：非常に長い。 ユーザーエクスペリエンスの最適な結果を確保するために、プロセスの優先順位は変更されませんでした。









結果から、サーバーを作成した直後に、クライアントタスクで使用可能なシステムパフォーマンスがWindows Server 2003以降でWindows Server 2012以上で最も低いことがわかります。これは、インストール後、Win 2012およびWin 2008モジュールのさらなる作業を最適化しますが、10〜15分後にリソースが解放され、クライアントが完全に使用できるようになります。 この効果により、新しいOSは作業にかなり多くのリソースを必要とするように見えますが、テストが示したように、これは真実ではありません。

テスト結果によると、 リソースを節約するという原則に基づいて古いオペレーティングシステムを選択しないでください-OSのすべてのバージョンで利用できるパフォーマンスはほぼ同じです 。 これに加えて、より新しいバージョンでは、 VPSサーバーのセキュリティ、信頼性、機能が向上します 。
