Windows 9では、チャームパネルが消え、仮想デスクトップが表示されます





Microsoftは、Windowsの次のバージョンでWindows 8のアイデアからさらに後退している、 と The Verge は書いています 。 マイクロソフトは、古い[スタート]メニューと独自のプログラムウィンドウの復帰に加えて、検索、スタート画面、デバイス、および設定にアクセスするために使用されるチャームパネルを削除する予定です。 チャームはタッチデバイスでは便利でしたが、マウスとキーボードパネルの実装は厄介で、ユーザーがカーソルを画面の隅に移動して慎重に開く必要がありました。



Windows 9と呼ばれることが予想される最新のWindows Thresholdビルドには、チャームパネルがなくなりました。 MetroスタイルのWindows 8アプリケーションは、いくつかのチャーム要素を含むタイトルバーを受け取ります。 この変更は、MicrosoftがWindows 8アプリケーションをより柔軟にし、従来のアプリケーションと同じようにデスクトップ上の別々のウィンドウで動作できるようにするという事実によるものと思われます。 これにより、これらのアプリケーションのマウスとキーボードのコントロールが大幅に改善されるはずです。マイクロソフトは、Windows 8でさまざまな更新を行って常に修正を試みてきました。



Microsoftは仮想デスクトップもWindows 9に追加しています。UbuntuとOS Xはこの機能を長年サポートしており、Windowsユーザーはサードパーティの代替手段を使用する必要がありました。 仮想デスクトップにより、Windows 9のユーザーは個別のアクティブデスクトップを作成し、タスクバーのボタンを使用してそれらを切り替えることができます。 これは、Microsoftがデスクトップユーザーに注意を向けることを決定したことのもう1つの兆候です。



Windows 9のプレリリースバージョンは今年後半に利用可能になる予定です。これにより、企業や経験豊富なユーザーは、2015年春に予定されているリリース前に新機能をテストできます。 マイクロソフトはまた、Windows PhoneとWindows RTを組み合わせたバージョンを準備しています。これは、内部的に「Windowsモバイル」と呼ばれ、デスクトップを削除します。
