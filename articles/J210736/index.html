Windows 7でのUSB HIDデバイス管理

この記事では、 AVRマイクロコントローラー上の自家製USB HIDデバイスと、 Windows 7 x64オペレーティングシステムを搭載したコンピューターを接続して、データを交換し、マイクロコントローラーのポートを管理する方法について順を追って説明します。 サンプルアプリケーションは、マイクロコントローラポートのUSBポートを介して制御されます（インジケータLEDが接続されています）。 LEDのステータスのステータスを読み取ることもできます-オフまたはオンです。 このトピックは初心者を対象としているため、プログラミングの専門家への大きな要望は、より便利なケースのために、 腐った卵と腐ったトマトを皮肉なコメントで保存することです。



使用したソフトウェア



1 マイクロコントローラーの場合、Objective DevelopmentのV-USBライブラリ[1]およびAtmelのAtmel Studio 6 [2] IDE。 また、WinAVRツールチェーン[3]をダウンロードしてインストールして、マイクロコントローラーファームウェアをコンパイルする必要があります（専門家にとっては、Atmel Studioの一部であるツールチェーンで対応できるため、これは必要ありません）。

2 。 Windowsプログラム（ホストソフトウェア）を作成するために、LibUsbDotNet [4] Travis RobinsonライブラリとMicrosoft Visual Studio C＃2010 [5] IDEが使用されました。



Visual Studio 2010を除くすべてのソフトウェアは無料ですが、Visual Studio C＃2010 Expressは期間限定で無料で使用できます。 すべてのアクションは、オペレーティングシステムWindows 7 x64の環境で実行されましたが、Windowsファミリの他のオペレーティングシステム（Windows XP以降）は間違いなく適切です。

Atmel Studio 6およびV-USBライブラリを使用したマイクロコントローラーファームウェアの作成



Atmel Studio 6（以降、単にAS6）で新しいプロジェクトを作成します。 AS6がマイクロコントローラーの選択を提案する場合、 Atmega32Aではなく文字AなしのAtmega32を選択します（Atmega32Aはボード上にありますが）-これは重要です。WinAVRツールチェーンは違いを認識せず、Atmega32のみを認識します。 これらのマイクロコントローラーの内部設計は同一であるため、私たちにとって違いはありませんが、AS6には違いがあります。



ここで、コンパイラを正しく構成する必要があります。 AS6のトップメニューで[ ツール ]をクリックし、[ オプション ]をクリックすると、次のウィンドウが表示されます。







リストの左側で、 ツールチェーンを選択します 。 フレーバーのリストが右側に表示されます。 この言葉で、Atmelは使用するツール（ツールチェーン）の可能なオプションをエンコードしました。 



.      Native,     (Default).  Native -   GCC      ,         .    Atmel,       AS6.    ,       ,       V-USB (   USB HID     USB)    .  ,         WinAVR,      .


WinAVRツールチェーンをフレーバーリストに追加するには、[ フレーバーの追加 ]ボタンをクリックします。次のウィンドウが表示されます。







このウィンドウの一番上の行に、WinAVRコンパイラーの名前（任意）を入力し、一番下の行に、ツールチェーンコンパイラーがインストールされている絶対パス（\ binフォルダー）を入力し、[ 追加 ] をクリックします。 スクリーンショットに示すように、追加されたコンパイラがFlavorsリストに表示されます。







新しく追加したWinAVRコンパイラをマウスで選択し、[ デフォルトとして設定 ]ボタン（デフォルトのツールバーにする）をクリックし、[OK]をクリックします。 この手順の後、AS6はWinAVRコンパイラを使用します。



プロジェクトのプロパティを設定します。このため、ソリューションエクスプローラーのカーソルを使用して左クリックでプロジェクト名を選択し、 Alt + F7 （メニュープロジェクト->プロパティ）を押すと、設定のウィンドウが表示されます。







次の設定を行います。

  • AVR / GNU Cコンパイラ->シンボルセクションで、行F_CPU = 12000000ULを-Dフィールドに追加します-これはマイクロコントローラー12 MHzの周波数に対応します（このクオーツはAVR-USB-MEGA16ブレッドボードにインストールされます）。
  • AVR / GNU Assemler-> GeneralセクションのAssembler flagフィールドで、 -DF_CPU = 12000000ULを追加します。
  • AVR / GNU Cコンパイラ->最適化セクションで、 最適化レベルフィールドのサイズを最適化（-Os）にする必要があります。


さらに、非常に重要な点-下の図に示すように、ウィンドウの左側のリストで、[ 詳細設定]セクションを選択します。







[ ツールチェーンフレーバー]ドロップダウンリストで、追加されたWinAVRコンパイラを選択し、AS6がプロジェクトのコンパイル時に使用するようにします。 これでAS6のセットアップが完了しました。



次に、プロジェクト[6]のソースコードファイルを作成したプロジェクトに追加します-ファームウェア\ VUSBフォルダー、VUSB.c、usbdrv.c、usbdrvasm.Sおよびoddebug.cファイルを参照してください。 ASS6プロジェクトは、V-USBライブラリの1つの例に基づいています。hid-custom-rqは、もともとコマンドラインからmakeユーティリティを使用してコンパイルされました。 V-USBライブラリに基づいて、他の多くのコード例を見つけることができます-主にUSB HIDデバイス（マウス、キーボード、入力および出力デバイス）がありますが、USB CDCデバイス（仮想COMポート）もあります。 自分でプロジェクトを作成するのが面倒な場合は、AS6でVUSB.atslnプロジェクトファイルを開くだけで、必要な設定がすべて行われ、必要なファイルがすべて追加されます。



別のブレッドボードを使用する場合は、 usbconfig.hファイルを正しく構成する必要があります。 これはV-USBライブラリの構成ファイルであり、多くの設定とパラメーター（VID、PID、マイクロコントローラーレッグ、記述子の値、およびその他の設定）を設定します。 すべての設定の詳細な説明は、このファイルのコメントに記載されています。 主な注意点は、USD D +およびD-信号（マクロ定義USB_CFG_IOPORTNAME、USB_CFG_DMINUS_BIT、USB_CFG_DPLUS_BIT）に使用されるマイクロコントローラーピンの目的です。これらのレッグには特別な要件が課されます。 アーカイブ[6]の設定ファイルusbconfig.hは、AVR-USB-MEGA16ブレッドボードの脚のレイアウト用であり、動作が保証されています。 プログラムはLEDで点滅します。LEDはブレッドボード上にあり、ポートBの0レッグに接続されています。

フォームロードイベントハンドラーを作成します。 プログラムの起動時にLibUsbDotNetクラスのインスタンスが初期化され、それを通じてUSBデバイスと交換されるようにする必要があります。 交換を開始する前に、複数のUSB HIDデバイスをコンピューターに接続でき、各デバイスに個別に連絡できる必要があるため、特にデバイスへのアクセスを開く必要があります。 USBデバイスを識別するために、すべてのUSBデバイスが持つ特別な識別子が使用されます。これらはVIDとPIDです。 



 .              -       USB   VID  PID,     .     USB-,   ,   VID/PID,    ,           .


VIDはメーカーID（ベンダーID）、PIDはデバイスID（製品ID）です。 USBデバイスのVID：0x 16C0 、PID：0x 05DF 、これらの値は、AS6マイクロコントローラープロジェクトの構成ファイルusbconfig.h （このファイルについては既に説明しました）で示されています。 ホストソフトウェアがUSBデバイスにアクセスするには、 usbconfig.hファイルに示されているように、同じパラメーターVID：0x16c0、PID：0x05dfでMyUsbFinderオブジェクトを初期化する必要があります。 これを行うには、Form1クラスのグローバル変数定義領域に次のコードを追加します。 



  public static UsbDevice MyUsbDevice; public static UsbDeviceFinder MyUsbFinder = new UsbDeviceFinder(0x16c0, 0x05df);


どのUSBデバイスを使用するかを決定したら、それに接続できます。これは、プログラムを起動する（フォームウィンドウを開く）ときに便利です。 これを行うには、プログラムのメインフォームを選択し、プロパティエディターで、ロードイベントハンドラーForm1_Loadを作成します。 ハンドラーの本文に、次のコードを入力します。 



 private void Form1_Load(object sender, EventArgs e) { MyUsbDevice = UsbDevice.OpenUsbDevice(MyUsbFinder); if (MyUsbDevice != null) { label2.Text = "  !"; } else label2.Text = "   !"; }

マイクロコントローラーファームウェアでUSBパケットをデコードする方法



マイクロコントローラー側でのデータの受信と処理は、 usbFunctionSetup関数で実行されます（ファームウェアAS6プロジェクトのメインモジュールVUSB.cにあります）。 関数は次のとおりです。 



 usbMsgLen_t usbFunctionSetup(uchar data[8]) { usbRequest_t *rq = (void *)data; if((rq->bmRequestType & USBRQ_TYPE_MASK) == USBRQ_TYPE_VENDOR){ DBG1(0x50, &rq->bRequest, 1); /*  :    */ if(rq->bRequest == CUSTOM_RQ_SET_STATUS){ if(rq->wValue.bytes[0] & 1){ /*  LED */ LED_PORT_OUTPUT |= _BV(LED_BIT); }else{ /*  LED */ LED_PORT_OUTPUT &= ~_BV(LED_BIT); } }else if(rq->bRequest == CUSTOM_RQ_GET_STATUS){ static uchar dataBuffer[1]; /*       usbFunctionSetup */ dataBuffer[0] = ((LED_PORT_OUTPUT & _BV(LED_BIT)) != 0); usbMsgPtr = dataBuffer; /*  ,    */ return 1; /*    1  */ } }else{ /*   USBRQ_HID_GET_REPORT  USBRQ_HID_SET_REPORT  , *     .        , *        . */ } return 0; /* default   :      */ }


USB HIDデバイスは最もシンプルで、エンドポイント0（デフォルトのコントロールエンドポイント）を通過するコントロール転送にのみ応答します。 要求のタイプ（フィールドbRequest）によって、データ送信の方向がデコードされます。 CUSTOM_RQ_SET_STATUSの場合、これはマイクロコントローラー向けのデータです。 データはデコードされ、マイクロコントローラーはそこに置かれたコマンドを実行します。 この場合、LEDのステータスは、最初に受信したデータバイトでエンコードされます。下位ビットに1がある場合、LEDがオンになり、ゼロの場合、消灯します。 CRequestOM_RQ_GET_STATUSがbRequestフィールドで受け入れられた場合、バッファはLEDの現在のステータスで満たされ、バッファデータがホストに返送されます。 すべてが非常に単純であり、必要に応じて、コードの動作をニーズに合わせて簡単に変更できます。
