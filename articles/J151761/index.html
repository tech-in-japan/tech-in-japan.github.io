MMSおよびITキャンプ向けのITデモの準備-パート1

同僚の皆さん、ご挨拶！



今日、最新のサーバーOSであるWindows Server 2012のITキャンプの準備に関する感想と、このイベントのMMSが開催される前日についてお話したいと思います。 イベントは9月26〜27日に開催されます。つまり、来週の木曜日にRIA Novostiサイトで開催され、Microsoftの新しいOSやその他の新しいチップの詳細をすべて直接会って学ぶことができます。

しかし、イベントを成功させるには（観客にとって興味深く有益なことを意味します）、準備段階で多くの努力をする必要があります。 カットの下で、これがどのように起こるかについて、より詳細に。









ラボ、ラボ、ラボ





おそらく、スピーカーの報告（おいしい食べ物は別として）に加えて、参加者にとって最も興味深い（最も有用ではないにしても）のは、常にライブデモと実験室作業です。 前者がデプロイされたソリューションとその範囲を明確に示している場合、後者は製品またはテクノロジーのセットアップと操作の経験を個人的に得ることができます。

これらは、これらのイベントで私の責任範囲に分類されたものです。 デモスタンドの説明と、デモスタンドで見たいものから始めましょう。



デモスタンドは常に立つ





一貫性を保ち、MMS初日である9月26日のデモンストレーションに関するストーリーから始めます。

Microsoftの代表者とHPの同僚の共同の取り組みによるデモンストレーションのために（偶然にも、非常に興味深い鉄片を私たちに提供してくれました-私から個人的に感謝しています-高品質で、強力で、希少な機器をいじることは常に非常にクールで興味深いです） Windows Server 2012 、 System Center 2012 SP1 Beta 、およびWindows Azureに基づいて、展開されたプライベートクラウド、さらにはハイブリッドクラウドをデモンストレーションすることが興味深いとの決定。 また、このプラットフォームでは、イベントの2日目（9月27日）にラボを展開する予定です。



デモンストレーションのために、c3000バスケット、64 GBのRAMと各2 CPUを備えたBL460 G8ブレード4枚、ブレードあたり300 GB 15K SASディスクのペアを割り当てました。 また、バスケットには2 TBのディスクスペースを備えたP4000ストレージがあります。 System Center 2012を新しいWindows Server 2012と組み合わせてデモンストレーションするためのまさにそれです。ここにあるのは、私の夢（家）の美しさ（バスケット）です）。 ハードドライブの表示は非常に興味深いです-RAID 0のディスク構成の場合、ドライブイジェクトボタンのLEDバックライトが点灯し、ドライブを取り外す必要がないことを警告しますが、RAID 1の場合、通常のボタンとの違いは見つかりません。 申し訳ありませんが、機械的なロックはありません...







バスケットには10​​ Gb / sの内部ネットワークもあり、ブレード間で非常に効率的なデータ交換が可能です。WindowsServer 2012のデモンストレーションラボを備えた150 Gb VMがブレードから500 Mb / sの速度でブレードに飛ぶ様子を見るのは本当にうれしいですデータパケットのサイズを最適化することを忘れてください（ジャンボフレーム）。 また、Windows Server 2012がSMB 3.0ボールに基づいてHA構成を作成できることを思い出すと、ここで多くのおいしいものが待っています...しかし、私たち自身の楽しみを奪って試してみませんか。



プライベートクラウドまたはジャストファーストトラック





いずれにせよ、WS2012ラボを備えた仮想マシンはホストにデプロイされ、すべてが作動します！

残っている唯一のことは、System Center 2012に基づいたプライベートクラウドを展開することです。これは、今後数日で行うことです。

車輪を再発明しないために、Microsoftは鉄のパートナーとともに、 Private Cloud Fast Trackイニシアチブを開発しました。

基本的に、これはハードウェアのハードウェアとソフトウェアの選択であり、Windows ServerおよびSystem Centerと統合するためのハードウェアであり、ベンダーによって検証されています。 つまり、言い換えれば、これはニーズに合わせて拡張できるプライベートクラウド用のプラットフォームを備えたバンドルです。

特に今後は、HPだけでなく、さまざまなベンダーのプラットフォームに関する詳細な説明と議論を含む多くの記事とビデオ資料が期待されるため、これに焦点を当てません。 それまでの間、私は実際にHPのプライベートクラウドFast Trackで働いており、HPサーバーハードウェアの消防専門家であるEvgeny Marchenkoがこれを手伝ってくれました。 私たちと一緒の写真は少し低くなっています（私がEugeneからどれだけの強さを吸い取ったかを見ることができます-ここで私はIT吸血鬼です）。







そのため、最も興味深いのは、クラウドとHPの機能との統合がちょうど先に進んでいるということです。これについては、また別の機会に話します。
