お知らせ：Veeam Backup＆Replication 9.5の新機能（およびそれ以降）

Veeamは、今四半期にVeeam Backup＆Replication 9.5のUpdate 3をリリースし、10周年記念エディションをリリースする予定です。 これは、仮想環境でデータを保護するための当社の有名なフラッグシップ製品ですが、物理マシン向けのソリューションも最近人気を集めています。MicrosoftWindows用のVeeam AgentとLinux用のVeeam Agentです。



当然、単一のコンソールを使用して仮想マシンと物理マシンのバックアップを使用したいユーザーの数が増えています。 したがって、当社のエンジニアは、この機会を遅滞なくVeeam Backup＆Replicationに実装することを決定しました。今日はそれについてお話しします。 詳細については、猫へようこそ。







外観から始めましょう。 Veeam Backup＆Replicationは仮想マシンと物理マシンの両方で機能するようになったため 、以前はVMビューにあったInventoryセクションを別のビューに移動することにしました。 Veeam Backup＆ReplicationまたはVeeam Agentのいずれであっても、Veeamによって保護されているすべてのマシンが表示されます。







論理コンテナ保護グループ



コンソールを介して保護を管理および構成する機能を提供するには、適切な論理コンテナ（いわゆる保護グループ （保護グループ））を作成する必要があり、必要なオブジェクトがそこに配置されます。



まず、保護を設定するオブジェクトのタイプを理解します。 Create Defense GroupウィザードのTypeステップには、いくつかのオプションがあります。



  1. 個々のコンピューターをコンテナー（グループ）に配置する場合は、グループにタイプ「 個々のコンピューター」を割り当てます。 IPアドレスまたはドメイン名を指定することでコンピューターを追加できる静的な結合（その場で変更が反映されない）を取得します。 これらのグループは、Active Directoryドメインが構成されていない小規模インフラストラクチャに推奨されます。
  2. Microsoft Active Directoryオブジェクトを配置する動的グループが必要な場合は、Microsoft Active Directoryオブジェクトのタイプをグループに割り当てます 。 これは動的タイプのコンテナであり、オブジェクトは、Active Directory OUやセキュリティグループなどに含まれると自動的に分類されます。 Active Directoryで動作する任意のサイズのインフラストラクチャにこれらのグループを使用することをお勧めします。
  3. かなり大規模なインフラストラクチャがあるが、Active Directoryを使用していない場合、またはCMDBとの統合を構成している場合は、CSVファイルからインポートされたコンピューターのリスト（CSVファイルからのコンピューター）で更新できる保護グループを使用することをお勧めします 。






ほとんどの場合、組織はActive Directoryオブジェクトにグループを使用する可能性が高いため、最初にグループについて詳しく説明します。



Microsoft Active Directoryオブジェクト保護グループタイプが選択された場合、Veeam Backup＆Replicationサーバーは、ホームドメインのActive Directory構造に関する情報を自動的に要求して表示します。



注：デフォルト設定を変更して、たとえば、一般的にサブドメインまたは別のドメインを選択領域として表示できます。 選択領域から個々のオブジェクトを除外することもできます。たとえば、先月中にオフになったコンピューターアカウントを除外したり、OU全体を除外したりできます。



したがって、ADオブジェクトがどの選択領域から表示されるかを示しました。 Veeamは、これらのオブジェクトにVeeam Agent実行可能モジュールをインストールしますが、実際にはバックアップされます。 もちろん、エージェントをインストールするには、Veeam Backup Serviceアカウントに適切な権限が必要です。



次に、選択領域をスキャンする頻度を指定する必要があります。このエンティティは動的です。つまり、次のスキャンで新しいオブジェクトが検出されるとすぐに、保護グループに自動的に含まれます。



任意のタイプのグループのスキャンオプション（ オプションステップ）で、新たに検出されたオブジェクトにVeeam Agentを自動的に配置するかどうかを選択できます-必要に応じて、 [バックアップエージェントを自動的にインストールする（推奨）]チェックボックスをオンにします。







バックアップ



コンピューターを適切にグループ化したら、エージェントを使用してコンピューターのバックアップを設定します。 これを行うには、目的の保護グループを選択し、そのための新しいタスクを作成します（または既存の保護グループに含めます）。







職種



処理するマシンの種類に応じて、タスクのタイプを選択します。 可能性があります：



  • 作業ページ-オプションワークステーションを選択します
  • サーバー-もちろんサーバーを選択します
  • そして-ユーザーオーディエンスの多数のリクエストによると！ - フェールオーバークラスター






ジョブ制御モード



同じステップで、バックアップを担当するエージェントが機能するモードを決定する必要があります。 保護するマシンのタイプに応じて、両方または2つのオプションのいずれかを使用できます。



  • Veeamバックアップサーバーからの集中管理（バックアップサーバーで管理 ）。

    このモードが選択された場合、Veeam Agentの軽量バージョンがマシンにインストールされます-GUIなし、ローカル構成データベースなしで。 すべての設定は、Veeamバックアップサーバーで行う必要があります。



    このバージョンでは、サーバーは、タスクのリモート開始と停止、統計の表示など、すべての作業を制御します。



    このモードは、インフラストラクチャに常に接続されているバックアップマシンに便利です。 これは常にバックアップクラスタに割り当てられ、バックアップサーバーの選択オプションにもなります。
  • タスク制御はエージェント自体によって実行されます（ エージェントによって管理されます ）。

    このモードを選択すると、Veeam Agentのフルバージョンがマシンにインストールされます。

    すべてのスケジュール設定とタスク管理は、エージェント自体の側で実行されます。

    このモードはバックアップマシンにとって便利です。バックアップマシンでは、良好なチャネルを介してインフラストラクチャへの常時接続を常に確保できるとは限りませんが、RPO要件を満たす必要があります。 これは常にバックアップワークステーションに割り当てられ、バックアップサーバーのオプションにもなります。


実行中のアプリケーションに基づいてバックアップを作成する



ゲスト処理のステップで、マシンで実行中のアプリケーションを考慮してバックアップを実行するように指定できます（アプリケーションの一貫性）。 次のアプリケーションがサポートされています。



  • Always-Onグループのサポートを含むSQL Server
  • オラクル
  • DAGサポートを含むExchange
  • Active Directory
  • SharePoint


これらのアプリケーションのオブジェクトをバックアップから復元するには、Veeam Explorersラインのツールを使用できます。



SQLおよびOracleデータベースの場合、トランザクションログのバックアップがサポートされます-選択した時点での回復の可能性。
