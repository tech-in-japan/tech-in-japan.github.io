インターネットはグローバルな情報ネットワークです。 インターネットは完全にコンテンツで構成されており、Webの「バージョン」の違いは、コンテンツの編成/表示方法の違いにすぎません。



Web 1.0は、単純な消費者であるユーザーの完全な無形性を暗示しています。

この単純な事実を認識して、多くの大企業（特にBBC、IBM、Google、Oracle）は、今後5〜10年間のインターネット開発の主な方向として、「スマート」情報処理に基づくセマンティックWeb 3.0システムを提案しました。 その本質は、メタデータとメタベースに基づいて、ビデオやデジタル画像での検索など、コンテンツに関する情報を検索するのはユーザーではなく、マシン自体であるということです。 Web 3.0は、インターネットの開発の最も差し迫った問題、つまり情報のゴミから情報を分離する意味のある情報の検索を解決するはずです。 1つの解決策は、ソースの評価に応じて、著者のソース別に情報をランク付けすることです。 また、評価自体は自動的に行われるべきではなく、ネットワーク上でのアクティビティに対してのみ発生しますが、他のインターネットユーザーによって決定されます。 他のより一般的なソリューションは、主にメタデータを中心に構築されています。

セマンティックWebは、Ser Timothy Berners-Leahの空想だけでなく、大企業や政府機関でも使用されますが、これらは多くの努力を必要とする特別な開発です。 たとえば、インテリジェンスでは、セマンティックテクノロジーを使用して、テロとの闘いにおける人々の関係を特定します。 テロリストに関するすべての情報は、標準データベース上に構築された3レベルのRDFシステムに保存されます。 このデータを処理する複雑なプロセスを管理するには、システムに22 TBのRAMが必要であり、これは不安定であり、障害発生後にシステムを復元するのに1週間かかります。



世界中のネットワークで私たちを待っているテクノロジーを見てみましょう。



RDF、OWL Web Ontology Language、SPARQLなどのテクノロジーと、XMLのような関連物に精通しているかもしれません。 この一連の文字により、開発者はデータをセマンティック構造に整理できます。



最も基本的な言語はもちろんRDFです。 これは、すべてのデータをトリプレットで記述します：subject-predicate-object。 たとえば、RDF用語の「空は青」という文は、次のように表すことができます：件名-「空」、述語-「色がある」、オブジェクト-「青」。



Web 3.0のもう1つのクジラは、Web Ontology Language（OWL）です。 OWLは、XMLベースのインターネットオントロジー言語です。 OWL Webオントロジー言語は、Webドキュメントおよびアプリケーションに共通のクラスおよびクラス間の関係を記述するために使用できる言語を提供するように設計されています。 この言語は、オブジェクトプロパティデータモデルでの現実の表現に基づいています。 この言語の各説明要素には、wwwアドレスURIが割り当てられます。

FreeBaseは、WikiPedia、Musicbranzなどの自由に表現されたデータベースへのリンクを作成するユーザーのコミュニティです。 リンクされたデータは、メタ属性、カテゴリ、およびフォークソムニーに従ってWikiスタイルで追加されます。 データベースには現在、約20,000の映画、350,000の音楽アルバム、35万人のデータ、合計873のカテゴリがあります。 データベース内の各エントリについて議論したり、関係を確認したり、編集したりできます。 プロジェクトはアルファテスト中です。したがって、データベースへのアクセスは招待者のみです。



DBpediaは、ウィキペディアから構造化された情報を抽出し、他のソースへのリンクを作成するコミュニティでもあります。 このプロジェクトでは、RDF（11言語で647.348のリンクを持つ930万のドキュメント）を使用しているため、このデータに対してSPARQLクエリを実行できます（SPARQL-セマンティックWebの標準言語クエリ）。 さまざまなプログラミング言語の実装がすでにいくつか存在します。 現時点では、Musicbranz、Geonames、YAGOデータベースへのリンクがリンクされており、約6万人のデータが含まれています。 データベースには、映画、音楽、政治、科学などのセクションが含まれ、常に補充されています。



Friend of a Friend（FOAF）は、機械可読なホームページとソーシャルネットワークのモデルを作成するプロジェクトです。 プロジェクトの中心は、誰かに関するステートメントで使用される特定の表現、たとえば名前、性別、その他の特性を定義する仕様です。 このデータを参照するには、友人の一意のプロパティを含む識別子が使用されます（たとえば、電子メールアドレス、Jabber ID、またはホームページ、ウェブログのURIからのSHA1合計）。

ODFを使用して定義され、容易に拡張できるように設計されたRDFに基づいて、FOAFを使用すると、異なるコンピューター環境間でデータを配布できます。



Semanticly Linked Online Communitis（SIOC） -ブログの投稿、フォーラムのコメント、記事、およびその他の投稿をトピックまたはキーワードの単一スレッドにリンクする方法。 最近、この技術はW3Cによって後援されています。



WordNetは、英語の語彙ベースです。 データベースには、名詞、動詞、形容詞、副詞が含まれ、これらは認知同義語（シンセット）のセットにグループ化されます。 Synsetsは、投機的なセマンティックおよび語彙の関係を通じて密接に関係しています。 結果として意味のある関連する単語と概念のネットワークは、サイトに表示されるナビゲーターで見つけることができます。 すべてのデータベースデータは、SPARQLマシンクエリを使用して自由にアクセスできます。



Geonamesは、地理的な名前を関連するRDFデータにマッピングするためのプロジェクトです。 言い換えれば、地理情報はRDF標準に従ってサイトに表示されます。
