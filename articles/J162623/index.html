大きく、長く、切り取られていない（または「自宅で大量のテキストを作成する方法」）

こんにちは、Habrの住民。



確かにあなたの多くはそのような状況に出くわしました-ある時点で大量のテキストが必要になります（たとえば、ページを作り、ブロックの「ゴム」をチェックしている）



そして、そのようなテキストを取得する方法は、3つの大きなグループに分けることができます。

  1. 最寄りのサイト\マニュアルなどからコピーしたテキスト全体を貼り付けます。
  2. 最大数の身体部分を使用して、キーボードのすべてのキーをすばやく押す。
  3. Ctrl + A-> Ctrl + C-> Ctrl + V-> Ctrl + V


最初の2つの方法は、それぞれの十分な自律性のために考慮されません（さらに、ほとんどすべて、いくつかの例外を除き、2番目の方法を使用して、時間が経つにつれて「コンピューターオペレーター」としての経験を積んで、無駄になり、最初または3番目の方法に完全に置き換えられます）



3番目の方法を検討し、その使用がいかに正当化されるか、および時間/労働の観点から最適な結果を達成するためにそれを近代化する方法を見つけます。



はい、私はあなたに警告する必要があります、この記事は科学に近い性質のものでさえありません、そして、それで説明されるすべては正しいことをすることへの脳の不本意と率直なナンセンスに従事したい願望の結果です。







驚いたと同時に、これらの行をまだ読んでいることを嬉しく思います！



はじめに、有効性の計算方法について説明します。計算自体は表の中で低くなります。

したがって、 Ctrl + A 、 Ctrl + CまたはCtrl + Vを1回押すだけで1ポイント... 1ポイントかかるとしましょう。そう、そうです。 このポイントは次のように取得されます：0.5ポイント-別の位置に手を移動するため（ AからCおよびVに指を移動するため）、および別の0.5ポイント-組み合わせ自体を押すために。



つまり、標準と最愛の組み合わせCtrl + A-> Ctrl + C-> Ctrl + V-> Ctrl + Vには1回の反復で3.5ポイントの努力が必要です。 同時に、ソーステキストが2倍に増加します。 視覚的な説明のために、ファイナルテーブルの一部を示します。

1回の反復の「コスト」は4ポイントに増加しましたが、効率は大幅に向上しました。 28ポイントの最初の組み合わせを使うと、256倍になり、2番目の組み合わせではこの値が2187倍に増えました！ さらに、後続の反復ごとに、増加の差係数が3分の1増加します。 いいね！ 可能ですか？...



まだ可能ですか？ -この質問は今夜私を苦しめました！ 頭の中で最初の2つの組み合わせを計算し、変更が妥当であることを確認することは難しくありませんでした。 しかし、組み合わせの増加が回帰につながる瞬間を見つけることができませんでした。頭の各組み合わせの各反復をカウントします。 できれば、この異端をすべて読む必要はありません:)



最適な組み合わせを非常に迅速に見つけることができましたが、増加率の明らかな減少に気付い​​た場合にのみ停止できました。 このためには、10の組み合わせで十分でした。 そして、ここに「効率」の最も顕著な減少があります-48回の努力を費やした10番目の組み合わせでは、1,771,561倍の増加が得られます。 酸味ではなく、ほぼ200万！ 8番目の組み合わせがなければ、すべてがうまくいき、49ポイントが4,782,969を獲得します。



「ちょっと待ってください」とあなたは言います。「10回目の組み合わせでは48ポイントしか使わず、ほぼ200万ポイントを獲得しました。結果は500万近くですが、ポイントはすでに49です！」

はい、しかしリラックスできます-10番目の組み合わせに余分なポイントを追加するのはCtrl + Vを 2回押すだけです（その時点で1回押すと161 051だけ結果が増えることを思い出してください。つまり、2回のクリックは32万を少し超えます。保存しません）



さらに、本当に完全な試合が必要な場合、同じ10番目の組み合わせと48ポイント（1,771,561）も6番目の組み合わせとの競争に耐えられず、48ポイントのスコアで5,764,801が増加します。 私にとっては、このデータはコピーと貼り付けをすばやく行う方法を理解するのに十分すぎるほどでした:)
