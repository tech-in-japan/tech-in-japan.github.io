紳士的なWeb開発者キット。 私のバージョン。



今日、書き方と書き方（しゃれを意図）についてもう一度書く代わりに、書き方と使用するユーティリティ、すなわち作業を簡単にするソフトウェアについて話すことを提案します。

誰もがソフトウェアを使用する上で独自の好みと習慣を持っているので、私は自分自身から始めて、それらをいくつかのカテゴリーに条件付きで分割し、あなたは続けます。 誰が何で動作するかを見るのは常に興味深いです。 始めましょう。

XML / XHTML / PHPエディター：

Editplus-非常に優れた柔軟なテキストエディター。 構文の強調表示、カスタマイズ可能なキーボードショートカット、カスタムテンプレート、多くの言語のサポート、「軽さ」、低リソース消費などの一連の設定。 1つの大きなマイナスは無料ではありません。 30ドルはそれほどではありませんが。

Notepad ++ -以前の競合製品とは異なり、無料です。 追加機能はプラグインを介してねじ込まれます。 セッションを保存します。 プログラムを閉じるときにすべてのファイルが開かれ、次の起動時に再び開かれるという意味で。 東洋言語（アラビア語、ヘブライ語、タイ語）をサポートします。

Eclipse 、 Zend Studio 、 Aptanaなどの開発者向けのヘビー級プラットフォームは、それぞれ別の投稿に値するので説明しません。 言及するだけです。 :)

CSSエディター：

疑いの余地のないリーダーは、もちろん、HomeSiteを作成した非ロシア名のNick Bradburyを持つネットワークWeb開発者の英語セグメントで有名なTop Style Pro 、そしてFeedDemon RSSリーダー（すでに存在する多作のプログラマー）です。

Top Style Proはただ操縦するだけです。 ヒント、検証（ロシア語ではどうですか？）、プレビュー、パレット、プロジェクトの保存など。 無料ではありません。 80ブカソイドの価格。

無料のLiteバージョンもありますが、ファイルを検索したり、ファイルを保存した後にアクションを元に戻すなどの機能が主に不足しているため、期待できません。 したがって、上記のエディターのいずれかを使用する方が簡単です。

Javascript：

これは論点です。 原則として、テキストエディターはそれを認識して強調表示しますが、それ以上は何もしません。 デバッグにはFirebugを使用し、JSでの作業を多少なりとも完全に実装しているエディターの中で、前述のAptanaを知っています。 あなたがより良い何かを知っているなら、コメントしてください。

ブラウザ：



このユーティリティを使用して同じマシンで実行されているInternet Explorer 6およびInternet Explorer 7。 そしてもちろん、ツールバーで 。

持っている必要がある拡張機能のセットを持つFirefox 、私は言及できませんでした。 私はいつもそれを補って、それから前の患者のために修正を加えます。

さて、Opera-好奇心for盛です。 もちろん、 ツールバーを使用します。 この素晴らしい（いいえ、本当の）ブラウザーのWebサイト最適化は確かに正しいことですが、私の意見では絶望的です。 しかし、もちろんそれはすべて顧客次第です。

グラフィックス 、 Photoshop、およびFlashの統治では、それらに焦点を合わせません。

バージョン管理には、シェルに統合されてスマートに動作する、小さいながらも自己完結型のユーティリティであるTortoiseSVNを使用することを好みます。

ファイル管理は紛れもなくTotal Commanderです。 私はこれ以上良いものを見ていません。 このプラグインの助けを借りて、私もそこに空白を焼きます。

すべては一般的な用語のようです。 私はここにホリワールに少しいますが、私を責めないでください。 上記の問題についてのあなたの意見は非常に興味深いものです。

しかし、もちろん結果は重要です。 sayingにもあるように、「終わりは手段を正当化する」。
