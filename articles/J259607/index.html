MMOをテストする際の開発者の間違い

この記事では、ゲーマーの観点から、「バリケードの反対側」からMMOをテストする方法について説明します。 アルファテスト、PTA、MBTをより効果的にする方法についてお話したいと思います。 そして、なぜベータテスターがバグレポートを書くのをやめて、プロジェクトに「反対」するのかについて。



まえがき



私の人生の中で、CBTとMBTで約30のゲームに参加しました。 そして、私が自分のために作った最も重要な結論は、ゲーム開発者は頭に落ちたゲーマーのすべての機能を呼び出すことに熱心な数千人という形でそれらの人的資源を完全に管理することができないということです。 そして、それがこの資料が書かれた理由です。 ゲーマーから、画面のその側に座っている開発者まで。 テスターから、プレーヤーの努力によってテストプロセスを適切に編成する方法を理解していない企業まで。



この記事は4つのセクションに分かれており、口語的なスタイルで書かれています。 アルファテスト、PTA / MBT、およびリリース後にテスターを募集する際の開発者の間違いを整理します。 多くの前奏曲や熟考のない詳細を求めている人は、各セクションと結論の最後の段落にすぐに戻ることができます。



I.アルファテスト



近年、World of Tanks（2009）、WarThunder（2011）、Lost Sector（2013）、Armored Warfare（2014）など、アルファテストにプレイヤーを招待することは良い趣味の兆候になりました。 これは悪い考えであり、ATはゲーマーではなくプロのQAエンジニアによって実行されるべきだと私は主張します。



なんで？ ATにプレイヤーがいることの主な結果は、あなたの会社の評判の低下です 。 明らかに、この論文は当惑させる可能性があります。 以下の説明：



1.プレイヤーは、バグをテストして銀の大皿にすべての間違いをもたらすためではなく、プロジェクトの中で他のプレイヤーより早く来ますが、エンターテイメントと喜びの部分を最初に取得します。 間違いなく、彼らがプレイを妨げる重大なバグを見つけた場合、彼らはそれについて報告します。 しかし、一般的に、ゲーマーはプレイしたい（そして再びプレイしたい！）、そして次のテストビルドの次の不具合は、実際にはわずらわしいだけで、理解できない。 そして、人々はポジティブなものよりもネガティブなものをずっと簡単に覚えているので、プレイヤーの一部のこのリリースでは、あなたが入力した新しい機能ではなく、すべてのミスを覚えています。 そして、それはあなたのオンラインゲームの批評家の最初の軍隊になります。



2. 実際には、次のようになります （たとえば、次のパッチに対するプレイヤーの反応）。

初心者 ：やった、新しいパッチ！ 新機能!!! これは素晴らしいです!!! フォーラムの開発者が好きです。

ベータテスター ：ああ、はい...新しいパッチ-新しい不具合。 まだプレイしません。 それらがロールバックし、修正され、頭に浮かぶまで待ちます。 私の神経は私にとって大切です。 それまでの間、私は競合他社のゲームをプレイします。

アルファテスター ：えーと、もし世界中でそれを入手しなければ。 0.4.5のロールアップ時にサーバーがクラッシュしたことを覚えていますか？ 私たちはLJに行き、開発者が再び失敗し、すべて間違ったことをしたと書いていますが、PTAでこのカードのバランスを再調整する必要があることを伝えました。



もちろん、これは仮説的な対話ですが、その本質は明らかだと思います。 さまざまなタイミングでWoTに参加したプレーヤーの実際の対話を行うこともできますが、これはこのリソースのルールに矛盾します。 特に、in辱やわいせつに関連するアイテム。 開発者を検閲するCBTシュニクを1人も覚えていないからです。



はい、アルファテスターとCBT / OBは他のカテゴリのプレーヤーよりもはるかに少ないです。 しかし、ほとんどの場合、すべての氏族/ギルド/企業でリーダーシップの地位を占めるのは彼らです（必要に応じて下線を引きます）。 少なくとも最初は。 そしてそれに応じて、最も経験豊富なプレイヤーとして、彼らは一族の枠組みの中でプロジェクト全体について意見を形成します。



もちろん、プロジェクトが「撮影」されると、時間の経過とともに最初の氏族が「ぼやけ」、一般的な情報ノイズでは、これまで不満だった最初のテスターの声が聞こえなくなります。 しかし、彼らが言うように、堆積物は残ります...



小さな説明の要約 ：いいえ、テスタープレイヤーが悪だと言っているのではありません。 AT / CBT / MBTとQAのエンジニアに来たプレイヤーは、まったく異なる目標を持っているため、ゲームの問題に対する反応が異なります。 したがって、ゲームが多くのバグであるときにプレイヤーをATに招待するという考えは、非常に悪い考えです。



II。 PTA＆MBT



理論上 （私の理解では）、CBTおよびMBTの主な目標は、ゲーマーの努力によって、リリース前に動的に更新されるゲームの機能をテストすることです。 そして、ここから2番目の目標、つまりゲームの概念に対するCAの反応と、開発者が導入する（または導入を計画している）さまざまな機能をテストします。 そして副作用として、開発会社の従業員による内部テストのコストを最小限に抑えます。



実際には、PTAとMBTは、どちらかと言えば疑わしい情報の生成者になります（例として、Google Shkolotyブロックは、装甲戦の存在について最初に聞いたことがあり、ゲームを天国に上げましたが、ゲームはリリースされていますが、すでに泥で散水しています）出てこなかった）。 また、テストの効率の観点から、結果は非常に物議を醸しています。 言葉遣いを正確に言えば-開発者は、何かをしたいテスターからプレイヤーを落胆させるためにあらゆることをしています。 そして、本当に質の高いフィードバックを提供できる数千人の聴衆と仕事をすることができなかったこの壊滅的な能力は、CBT / MBTの組織の典型的な間違いとその治療方法（プレイヤーの観点から）を書き留めました。



それでは、行きましょう：



1.テスト情報の欠如


一般的に、私個人の意見では、ほとんどのPTA / MBTは「それをテストし、私は何を知らない」というモットーの下で開催されます。 したがって、昼夜を問わずゲームに参加し、フォーラムでバグや落書きメッセージを見つけるテスタープレーヤーの特定の過活動グループがあるとします（たとえば、軍事史ゲームの場合、これらはお気に入りの機器の歴史的なパフォーマンス特性を守る「リベッター」です） ）



ここでの主な問題は、開発者もコミュニティマネージャーも、プロジェクトの利益のためにこのターゲットグループのエネルギーを建設的な方向に向けることができないことです。 「完全に」という言葉から。 真剣に。 テストの推奨目標はどこですか？ 開発者がプレーヤーによってテストされることを望む大まかな計画はどこにありますか？ いや 上記のどれも実際には見つかりません。 したがって、誰もが何をしているのかをテストします-1つはYouTubeでモデルを浸透させてビデオを作成し、2つ目はフォーラムで、さまざまなソースからのさまざまな口径の銃の浸透に関する要約データでサインを見ます



この状況の特別な魅力は、多くの場合、「歴史コンサルタント」が会社の空室にかかっていることですが、同時に、開発者はhistoria.ruまたはtsushima.ruでプロの歴史家のゲームのフォーラムでほとんどわいせつを送信します「100％リアルなゲームでは、歴史主義はバランスをとるために犠牲にされます。」 学童がGoogleの翻訳記事に基づいて行うWikiについて、私は一般的に黙っています。



パッチラップトップの開発者がテストで変更を書き込まないと、「バランスを最適化するために一部のマシンの特性が変更された」シリーズから個別にフリーズします。 では、何が変わったのかさえ知らずに、イノベーションをどのようにテストできるのでしょうか？ または、回帰テストを実施しますか？ 皆さん、情報提供者、開発者から毎回情報を得るために、猛烈なダニが必要ですか？ 私は特に強調します-これはPTA / MBTについてであり、絶対にすべてのパフォーマンス特性がバランスの過程にあります（そして私たちテスターはこれをよく知っています）。



2.適切なフィードバックの欠如


上記のすべてにもかかわらず、特定のVasya Pupkin（実際にはブラックボックステストに従事しているとは思わない）が、フォーラムで最初のバグレポートをテストして書く必要があると直感的に推測したと想像してみてください！



それに応じて：

-まったく応答がありませんでした。

-トピックは閉じられ、説明なしでアーカイブに転送されました。

-基本的な感謝がなくても、「バグは既知で、作業中です」という回答を得ました。

-などなど...



ヴァセンコの反応はどうなりますか？ -そうです！ Vasyaはこれ以上バグレポートを作成しません。



3.負の反応


ゲーマー（テスターも含む）は、まず第一に消費者です。 決して忘れないでください。 外交官や企業の広報担当者ではありません。 したがって、これはバグ報告とともに、冷たい水が開発者に注がれているときの完全に正常な作業状況です。 時々スロップがあっても（当然かどうか-これは別の問題です）。 なぜこれをしているのですか？ さらに、PTA / MBTに関する私の知り合いの約3分の1は、バグレポートの作成を停止し、激しい非難を受けました。



PTA / MBTの段階でのゲームフォーラムの一部として、私は通常、バグレポートによって裏付けられた、プレーヤーに対する合理的な批判に対する反応の2つのオプションを観察します。

-懲罰的かつ抑圧的、禁止まで（モデレーターはバンハンマーを備えています）;

-告発（開発者が来て、彼らに恩知らずな野郎だとプレイヤーを詐欺し始めました-開発者はここで夜眠れず、1週間妻とセックスをしていませんでした。



その結果、さらに多くのテスターがプロジェクトに敵対的になり、テストを終了します。 これはせいぜいです。 そして最悪の場合-ゲームポータルについて不平を言うために実行され、競合他社に行きます。



4.疑わしい報酬


そして、待望のMBTがやってきました。 私たちのヒーローである7人のバグレポートを持つVasya Pupkinと、PTA（実話）の最終日に登録して何もしなかったDima KozlovがTim-Speakに座っています。 そして、両方ともCBTに参加したことに対して同じ報酬を持っています-たとえば、特定の戦車です。 このような普遍的な不正行為の後、Vassenkaは少なくとも1つのバグレポートを書くと思いますか？ 正しく考える-いいえ。 そして今、資格のあるテスターの別のグループが忘却に沈んでいます。



一般的に、結論はすでに明らかであるように思えます。 しかし、これをPMの例で別の開発者に説明すると、誤解の空白の壁に出くわします（FBとVKによれば、これはゲームの乙女の外ではありますが、ほとんどの場合、これで十分です）。



だから、簡単な結論


1.プレイヤーにベータテストで何かを実際にテストしてもらいたい場合は、通常のQAのように扱います。テスト計画を提供し、これらのテストの結果を消化するためのフォーラムでインフラストラクチャを作成します。 何を、いつ、どのように、なぜテストする価値があるのか​​を説明します。 理想的には、全員に郵送します。 要するに、ベータテストの基準を引き上げてください！ エキストラやテスターに​​小学生が必要ですか？

2.ベータテスターに​​フィードバックを提供します。最も重要なのは、タイムリーな（！）コミュニケーションです。 バグレポートに関する適切なフィードバックは、ベータテスターとDG会社との間で引用された長期的な協力の鍵です。

3.プレイヤーの批判に適切に対応し、間違いを認めることを学びます。 気分を害したプレイヤーは、フォーラムで「禁止」された行だけでなく、急流（実話）で配布された500テラバイトのゲームでもあります。 また、仲間のメンバーの棚にあるギフトボックスにあるゲームのライセンスコピーは1つではありません。

4.覚えておいてください-CBT / MBTのすべての参加者に均等に授与することはできません。 報酬は、プロジェクトへのプレイヤーの貢献に比例する必要があります！ そうでなければ、本当にテストした人にとっては、プロジェクトの寿命が終わるまで何でもしたいという欲求を完全に落胆させます。



III。 リリース後



リリース後も何も変わりません。プレーヤーは新しいコンテンツを望み、開発しました。2人の恋人のコンテンツと島の小さな家のために自分のペニーを開発しました。 そのため、パッチやアドオンはパン職人の手から熱いケーキのように出てきて、虫はウサギのように増え続けます。 唯一の違いは、ベータテスターがスーパーテスターに​​置き換えられていることです。 そして、ここからドラマ「不条理」のマレゾンバレエの次の行為が始まります。



誰も理解していない場合、私はスーパーテスターの募集基準について話している。 WoTを思い出して、実生活でITで働いており、53〜55％の勝率を持つ戦術将校の申請が拒否され、43％で友人の息子を統計で連れて行ったとき、私たちは写真に非常に驚きました。 それは、私たちとは異なり、彼が1日12時間プレイできるからです。 ちなみに、1年後に戦車のアカウントを既に削除した当時、彼はまだ最初のバグを発見していませんでした。



そして、これは戦車の問題だけではありません。 何らかの理由で、彼らはスーパーテストのためにshkolotaを入力することを好むどこでも、24時間プレイする準備ができています。 私は中央アジアに対して何もしていません-他の誰かについては経験と断片を埋めなければなりません。 しかし、彼らにテストを依頼するのですか？ 皆さん、本気ですか？ どのテスターがより便利になりますか-「piso-pisol ololo-私はUFOドライバーです」というスタイルで16時間プレイするティーンエイジャーから、またはビジネスを行うためにテストに来る大人から1時間プレイするからですか？



それは私にとってほとんどすべてであり、古典が言ったように、「シェヘラザードは黙っていて、許可されたスピーチから許可されていない思考に移行しています。」



IV。 スマートな考え



実際、ゲームテストのゲーミフィケーションという賢いアイデアは1つしかありません。 プレイヤーにゲームをテストしてもらいたい場合-楽しんでください！ トッププレイヤーをランク付けしていますか？ -サイトにトップテスターの評価を追加します。 くしゃみのメダルと実績を出しますか？ -バグを見つけてプロジェクトを改善するのに役立つすべての人に、独自の成果をもたらします。 オタク向けの週末の経験は2回ありますか？ -テスターの増加率を入力します。 誰もがこの恩恵を受けるでしょう。
