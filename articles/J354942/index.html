クロスランゲージソフトウェア開発

挑戦する



ある言語でプログラムを開発している場合は、すぐに他のプログラミング言語でソースを入手してください...私はC＃.NETで書いていますが、最近ますますJavaと統合する必要があります。 1つの解決策は、対話用のWebサービスを設計することですが、これではなく、そうではありません。 JavaにはC＃コンバーターがあるように見えますが、実際のプロジェクトでは（試してみることができた）実際には機能しませんが、「hello world」では完全に機能します。 プロジェクト全体をゼロからJavaに書き換えることは非現実的です-6年以上にわたって積極的に開発され（Pullenti-自然言語処理）、C＃で必要です。 昨年、このコンバーターを動員して作成し、今年はC＃からPythonへのコンバーターを作成する必要がありました。



JavaでC＃コンバーターを作成した経験



そのため、Visual Studioのプロジェクト（ソリューション）の入り口で、Javaのクラスを取得する必要があります。 もちろん、すべてのタイプのプロジェクトをサポートできるわけではありませんが、タイプ「クラスライブラリ」と「コンソールアプリケーション」はかなりサポートされています。 プロジェクトは、System.Drawingなどのグラフィカルライブラリを使用しないでください。Windows機能やDllImportは使用しないでください。使用するライブラリには、Javaの類似物が必要です。 幸いなことに、そのような類似物は、私が使用した機能（XML、GZipアーカイブなどを扱う）で見つかりました。 アルゴリズムを少し調整して、いくつかの細かい詳細を放棄する必要がありました。 言語レベルでは、C＃とJavaは近いですが、Javaは少し貧弱です-これはそれから欠けているものです：



  • プロパティ{get; セット; }-関数を操作する必要があります。
  • 構造体-構造体がないため、最初は単純にJavaクラスを使用してモデル化しましたが、アルゴリズムを書き直して完全に破棄しました。
  • デリゲート-存在しませんが、1つの機能を持つインターフェイスによってモデル化されます。
  • event-対応するデリゲートのインターフェイスを実装するクラスの要素のArrayList配列によってモデル化されます。
  • メソッドのoutおよびref引数-1つのValueフィールドを持つクラスによってモデル化されます。
  • 内部-この修飾子は、アセンブリの概念がないため、Javaではまったく意味がありません。したがって、単純にpublicに置き換えられます。
  • new Class（）{assignments}-これらの割り当てを行う静的関数を作成する必要があります。
  • 列挙型-C＃で要素を使用したビット単位の操作が可能な場合、Javaではこれが禁止されており、コンバーターは列挙型をJavaクラスに変換する必要があります。


しかし、これは最も難しいことではありませんでした-アナログを探す必要がある膨大な数のシステム関数の問題。 いくつかの類似体が見つからなかったため、生成されたJavaソースに生成中に追加される独自のライブラリを作成して、Javaで記述する必要がありました。 たとえば、C＃には入出力操作用の基本クラスStreamがあり、Javaには2つの別個のクラスInputStreamとOutputStreamがあるため、それらに対するラッパーが必要です。



Cコードから一部のフラグメントを除外する必要がある場合、またはJava用に少し変更する必要がある場合は、プリプロセッサディレクティブ#if JAVA ...＃else ... #endifが使用されました。

そのため、コンバーターに新しい機能を徐々に追加し、C＃のソースコードを調整すると、Javaで生成された1,500以上のユニットテストが、C＃のプロトタイプのように正しく機能し始めたという結論に達しました。



.NETとJavaの速度の比較



今から楽しい部分です。 時間のかかるアルゴリズムがあり（主に文字列で動作します）、C＃とJavaでの実装はまったく同じです。 .NETの謝罪者として、私は常にJavaの有効性を疑っていました。 しかし、それは間違っていることが判明しました-C＃はJava（WindowsでEclipseから起動）よりも15〜20％だけ高速であり、期待どおりに数倍高速ではありません。 Pythonの状況はどうですか？ 読んでください。



PythonでC＃コンバーターを作成した経験



JavaよりもPythonでの変換がさらに簡単であることが判明しました。 基礎の準備は整っていましたが、ジェネレーターを新しい言語で修正し、使用されている関数の類似物を見つける（書き込む）だけです。



Pythonに欠けているものや不便なもの：



  • for（init、cond、incr）-完全な形式ではなく、範囲にいくつかの特別なケースがありますが、基本的にはwhileをモデル化する必要があります。
  • スイッチ-存在しない、モデル化されたif-then-else-s;
  • インターフェース-ありませんが、これは空のメソッドを持つクラスです。


他のクラスの非常に不便なインポートシステムであり、場合によっては循環依存と一般的なインポートの不可能性につながります。 これらの依存関係を追跡し、クラスのファイル全体ではなく、メソッドのレベルでインポートを実行する必要がありました。 これらおよび他の困難にもかかわらず、これらの1,500ユニットテストの正しい開発を達成することが可能でした。



.NETとPython Speedの比較



多くの人がPythonの速度が遅いことに不満を言っていますが、どれくらい遅いのでしょうか？ そして、それは低いですか？ Javaと同じアルゴリズム（より正確には、同じユニットテスト）で比較するときが来ました。 準備はいい？ そのため、Pythonは.NETの30倍遅く実行されます（EclipseでPyDevの下で実行されました）。



その理由は次のとおりだと思います。 PythonにはValueTypeのような単純な型がなく（私が理解しているように）、通常の数値でさえ本格的なオブジェクトです（実際には.NETでのボクシングのように）。 そして、文字列要素は文字ではなく、1文字の文字列でもあります！ つまり、文字列[i]はcharではなく、.NETのstring.Substring（i、1）に似ています。



クロスランゲージ開発技術



個人的な経験に基づいて、1つの言語とプラットフォームで開発し、他の言語とプラットフォームに自動的に移行することができると言います。 私は.NET Framework 4.0で基本プロジェクトを開発していますが、機能的には同等のものをいつでも持っています：



  • Javaソース（コンバーター）;
  • Pythonソース（コンバーター）;
  • .NET Core上のプロジェクト（別個のcsprojファイルですが、ソースコードは同じです）。
  • .NET Framework 2.0上のプロジェクト（コンバーター。これは1人の顧客に必要です）。


私は、Visual Studioの便利な機能の1つを共有します。これは、何らかの理由で、プロのC＃開発者でも使用されていません。 そして、これは、たとえばJava用の開発ツールにはありません。 つまり、実行中にコードを変更して現在の位置を直接移動する機能、およびすべての変数の状態が保持され、変更を考慮して実行を継続できます。 これにより、特に望ましい状況を再現するのに時間がかかる場合に、デバッグが劇的に容易になります。 たとえば、Visual Studioが例外の場所でプログラムを一時停止するモードをオンにし（または、むしろこのモードをオフにすることはありません）、長期処理を開始し、数分または数時間後にこの状況が発生します。ほとんどの場合、NullReferenceです。 コードを修正し、プログラムを再起動する代わりに、実行ポイントを目的のステートメントに戻し、さらに実行を開始します。 そして一般的に、実行中に頻繁に適切な場所に複雑なアルゴリズムを実装します。 取引所取引のシステムを作成したとき、初期化と必要なアルゴリズムブランチへのアクセスという面倒な手順のコンテキストで生活を一桁も楽にしたことを覚えています。 Visual Studioでは、この機能はIDEとコンパイラの緊密な統合によって実現されています。 しかし、テクノロジーに戻りましょう。



制限は明らかです-ライブラリとコンソールアプリケーションのみ。 カスタムGUIまたはWebアプリケーションを備えたエンドユーザーアプリケーションは、ほとんど非現実的です。 また、ライブラリを開発するときは、普遍性の要件によって課される制限を遵守する必要があります。 何かを拒否し、何かを書き直さなければなりませんが、結果は努力する価値があります！
