GENERATE ,,,10 ; 1 Amount office staff ADVANCE 4,0.8 ; 2 Time to reach the cooler DRINK QUEUE STAND ; 3 Queue to the cooler. input SEIZE COOLER ; 4 Verification of employment cooler DEPART STAND ; 5 Queue to the cooler. output ADVANCE 2,0.8 ; 6 Filling the cup with boiling water RELEASE COOLER ; 7 leave the cooler ADVANCE 60,10 ; 8 Time for tea, time to work, time to get back to the cooler TRANSFER ,DRINK ; 9 Go back to the cooler GENERATE 480 ; 10 Working day (minutes) TERMINATE 1 ; 11 Minus one minute START 1 ; 12 Start from the first minute




私は壊れた英語でコメントを残したので、私は確かにモデルの機能について詳細にポイントを見ます。

1行目-GENERATE ,,, 10-GENERATEブロック自体が着信要件（この場合はオフィスの従業員）の数を設定し、フォームのオペレーターの記録、、、 10は最初の3つのパラメーターが欠落していることを示し、4番目のパラメーターは最大着信要件の数。 GENERATEブロック。 他のパラメーターで設定することもできますが、これについては少し後で説明します。

ご想像のとおり、「;」 -コメントフィールドを区切ります。 コードの設計構造についてもう少し詳しく説明します。コードは左から右に読み取られます。タブで区切るのが望ましいいくつかのブロックがあります。 左端のブロックはラベル（他のプログラミング言語のラベルに似ています）、ブロック自体（生成など）、通常はパラメーター、コメントです。 むかしむかし、これらのブロックを表形式で分けるように教えられましたが、これは美学の問題です。 1スペースでもすべてが始まるのに十分なようです。 ところで、すべてが大きな文字で書かれている場合は、そのように書く方が良いです。そうしないと、実行エラーが飛び出す可能性があります。

行2-ADVANCE 4.0.8; 彼はプロセスの時間的特性を担当しているため、この場合により理解しやすくなります。これは、従業員が4分±0.4分間クーラーに行くことを意味します。

したがって、最初の数字は費やされた時間（反復）の単位を意味し、2番目の数字は偏差の半分または上下を意味し、書籍は偏差値がランダムに生成されると言っていますが、システムを再起動するたびに同じ値を与えます（おそらくわからない）古き良きパスカルからのランダム化の類似物がありますが、私はそれを見つけませんでした）。 ADVANCEブロックには他の値を設定することもできますが、それについては後で詳しく説明します。

行3-ドリンクキュースタンド; -ここで最初のラベルDRINKに会い、続いてQUEUEブロックが続きます。このブロックはこのキューにキューを作成します。クーラーに行く人全員に喉を濡らします。 STANDは、キューの識別子です。

行4-SEIZE COOLER-SEIZEブロックは、デバイスが空いているかどうかを判断します。この場合、デバイス識別子はCOOLER（まあ、または単にクーラー）です。

5行目DEPART STAND-最後のブロックで肯定的な結果が得られた場合（クーラーフリー）、次のDEPARTブロックがトリガーされ、ユーザーがSTANDキューから確実に離れます。

Line 6-ADVANCE 2.0.8ラインを離れた人は、マグカップに約2分間沸騰した水を注ぎます（さらに、クーラーに並んでいる人とチャットする時間もあります）

7行目-クーラーのリリース、「ウェット」ビジネスを行った人は、次の被災者のためにクーラーデバイスをリリースします。

8行目-ADVANCE 60.10-このブロックは時間を考慮しています。 水を飲んだ人が力を集めて再びクーラーに来るために努力します（少なくとも、これがシミュレートされることを願っています）。

9行目-転送、ドリンク-「すべてが以前と同じように繰り返されます」A.ブロックは、プログラミングに精通していれば言うでしょう。 このステートメントは、DRINKラベルへの移行を提供します。

10行目-GENERATE 480は、プログラミングの観点から、最後のブロックのラベルに移動した後、以下のすべてが意味をなさないように見えますが、そうではありません（正直なところ、私は本当に深く入りませんでしたが、経験とエラーに対して、このピースは前の8ブロックのコンテキストの少し外側で認識されます）。 主なことは、このユニットがモデル全体の実行時間、つまり480分（8時間）を設定することです。

11行目-TERMINATE 1-この側は、各反復カウンターから1を引きます。 このブロックがないと、無限ループに陥ることがあります。

12行目-START 1-サイクルの開始位置をプログラムに指示します。最初の1分から開始します。
