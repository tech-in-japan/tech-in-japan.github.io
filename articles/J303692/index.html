IT監視のためのOracle Cloudサービス



オラクルの新しいクラウドベースのIT監視サービスは、ビジネスアプリケーションのパフォーマンスの効果的なリアルタイム監視を提供し、Webページの開き方、エラーの発生、原因の多くの質問に答えます。 また、監視プラットフォームを使用すると、ログファイルの分析を通じてさまざまな問題の原因を効果的に見つけ、ITランドスケープのリソース計画を提供できます：データベースおよびアプリケーションサーバーの負荷を測定し、CPU、I / O、およびメモリリソースの実際の使用を評価および予測します。







2016年1月、オラクルは、アプリケーションパフォーマンスモニタリング、ログ分析、IT分析など、管理クラウドという一般名で新しいサービスのグループを導入しました。 まず、ITインフラストラクチャメンテナンスサービス、アプリケーション管理者、ITアーキテクト、ITマネージャーを対象としています。 もちろん、ユーザーには疑問があります。ITインフラストラクチャを監視および管理するための単一のプラットフォームであるEnterprise Managerについてはどうでしょうか。 事実、Enterprise Managerはもちろん非常に強力なシステムであり、データベース、サーバー、ファームウェアなどの個々のコンポーネントを監視する上で不可欠なアシスタントです。 しかし、多くのコンポーネント、多くのデータベース、さまざまな物理サーバーや仮想サーバーにインストールされた多くのアプリケーションサーバーで構成されるビジネスアプリケーションで問題を特定する場合、特殊なツールが必要です。 ところで、Oracle Management CloudツールはOracle Public Cloudで動作するが、オンプレミスのEnterprise Managerと密接に統合されていることを理解することが重要です。



Oracle Management Cloudツールの主な機能：エンドユーザーおよびインフラストラクチャ・コンポーネント・レベルでのWebアプリケーションの監視、IT組織の容量とリソースの計画の支援、メトリックとイベントの分析、ログ・データの集計、異常の自動検出およびインシデントの根本原因。 監視ツールの初期データは、ログファイル、ユーザー生成データ、マシンデータ、トレースファイル、パフォーマンスメトリック、EMリポジトリからの診断データ、トレースデータ、監査ログなどです（図1）。





Oracle Management Cloudの重要なデータソースは、Enterprise Managerリポジトリのデータです（図2）。





アプリケーションパフォーマンスモニタリング（APM）



Application Performance Monitoringサービス（図3）は、Oracle WebLogicまたはApache Tomcatプラットフォームで構築されたWebアプリケーションを監視するように設計されています。 APMのタスクは、さまざまなレベルでのWebアプリケーションの「プロファイリング」です。エンドユーザーからインフラストラクチャまで、インシデントの根本原因の発見を支援し、運用サービスと開発者に単一のインターフェイスを提供します。 Application Performance Monitoringサービスを使用すると、Webアプリケーションのエンドユーザーエクスペリエンスを判断できます。 同時に、APMを使用すると、ユーザーレベルからアプリケーションパフォーマンスのコンテキストトップダウン分析を実行し、アプリケーションモジュールの相互作用を視覚化し、必要に応じて「下位」レベルに切り替えて、関連するインフラストラクチャの問題を分析できます。





運用サービスと開発者は、アプリケーションパフォーマンスモニタリングの担当者に、コードとSQ​​Lクエリのレベルまでの実際のパフォーマンス診断のための単一の情報源を持っています。 開発者は貴重な診断データにアクセスし、サービス部門と協力します-彼らはお互いを補完し、これは迅速なトラブルシューティングを提供し、メンテナンスの品質（DevOps）が改善されます。



Application Performance Monitoringサービスは、コードインスツルメンテーションの方法に基づいています（たとえば、Oracle DBMSで同様のテクノロジーが使用されます）。 特別なエージェントは、アプリケーションコードとWebページを計測し、パフォーマンスを測定し、セーフモードでプロキシサーバーを介してパブリッククラウドにメトリックを送信します。



ログ分析



Log Analyticsツール（図4）は、アプリケーションパフォーマンス監視と連携して機能します。 ログファイルを直接操作するのはこのサービスです。雑誌のデータの検索、収集、保存、雑誌の情報の分析と調査、および視覚化に従事しています。





Log Analyticsツールの利点は、インフラストラクチャコンポーネントやログファイル形式に依存しないことです。 すぐに使えるサービスは、データベースログ、Enterprise Manager、FMW、Webサーバー、クラスターソフトウェア、Linuxオペレーティングシステム、スケジューラーなど、約30のログファイル形式をサポートしています。非標準のログファイルをサポートするには、簡単なパーサーを作成する必要があります。正規表現ベース



Log Analyticsサービスは、マシン生成の分析アルゴリズムを使用して、たとえば過去24時間にわたってログファイルに蓄積された数千の通常のメッセージから、重要なエラーメッセージを特定できます。これは、インシデントの根本原因である可能性のあるエラーです 問題が検出されると、「1分間のコンテキストを表示」メニューが表示されます。つまり、エラーメッセージの1分前と1分後にシステムコンテキストが表示されます。



Log Analyticsサービスのインフラストラクチャ全体は、パブリッククラウドにあります。 ユーザーには、便利なWebインターフェース、さまざまなタイプの図を通じてログファイルに記録された主要なイベントを視覚化する標準情報パネルが提供されます。また、独自の情報パネルであるウィジェットを作成することもできます。



Log Analyticsは、リアルタイムでログファイルからデータを収集し、高性能ビッグデータプラットフォーム上のパブリッククラウドにインデックスを付けて保存します。これにより、膨大な量のログデータ、並べ替え、検索、グループ、およびマシン分析を効率的に処理できます。



最後に、Log AnalyticsはApplication Performance Monitoringとシームレスに統合されます。 [ログへのドリルダウン]ボタンを使用すると、コードレベルから特定の関連インフラストラクチャ要素のレベルまで「失敗」できます。



IT分析



ビジネスには、現在のITランドスケープの機能、キャパシティプランニング、問題領域の特定などを理解するためのツールが必要です。既存のシステムは、データベースやミドルウェアなどのITランドスケープの個々のコンポーネントに焦点を当てているため、これらの問題を解決できません。また、一定の期間、アプリケーション、コストセンターのシステムなどのコンテキストで、データセンターの規模でシステムがどのように機能するかを確立することはできません。



したがって、Oracle Management Cloudサービススイートのもう1つの重要な部分はIT分析サービスです（図5）。 その範囲は、現在のITランドスケープの分析です。 彼が実行するタスクは、リソース使用の分析、体系的なパフォーマンス問題の認識、およびリソース計画です。 ITリソースの分析の目的は、問題領域の特定、さまざまな期間のワークロードの比較、さまざまなディメンションによる最大リソース消費の特定、リソースの最適化、および負荷予測による成長の計画です。





このサービスでは、プロセスリソース、I / O、メモリ、データストレージボリューム、ミドルウェアパフォーマンス、ガベージコレクターパフォーマンスなど、さまざまな基準に従ってパフォーマンス分析を実行することもできます。 データの視覚化機能により、さまざまな基準で検索を簡素化し、主要コンポーネントごとにKPIを視覚化して現在の使用状況と傾向を分析し、情報パネルも使用できます。 例：過去3か月間のビジネスクリティカルなデータベースバージョン12.1.0.2による合計CPU使用率のグラフを表示します。



上記のビジネスチャンスを使用した結果、監視と計画へのプロアクティブなアプローチ（不十分または過負荷のノードを特定することによるリソースの使用の改善）、運用コスト（問題の分析と特定のコスト）の削減、およびITの転換の支援（プロアクティブなアプローチと戦略的計画への移行） ）

Oracle Enterprise ManagerがOracle Management Cloudの代替ではないように、Oracle Management CloudはEnterprise Managerの代替ではないことを再度強調します。 Oracle Management CloudサービスとローカルにインストールされたEnterprise Managerは連携して機能します。 Oracle Management Cloudエージェントによって収集されたEnterprise Managerリポジトリおよび様々なメトリックのデータの統合および視覚化は、非常に興味深い機会を提供します。問題を解決するためのプロアクティブなアプローチ、タイムリーなビジネス指向の監視、開発者と運用サービス（DevOps）間の改善された相互作用です。



Oracle Management Cloud-サブスクリプションによって提供されるパブリッククラウド内のサービスのセット。これは、顧客が監視ツール用の高価で複雑なインフラストラクチャを追加投資し、維持する必要がないことを意味します。 必要なのは、エージェントをインストールし、クラウドへのデータ転送を構成することだけです。 Oracle Management Cloudサービスは、ビジネスデータまたは個人データをデータベースからクラウドに収集または転送しないことを強調する必要があります！ 診断情報、メトリック、およびログデータのみが収集および分析され、必要に応じてクラウドに送信する前に「マスク」できます。



Oracle Management Cloudは、この記事で説明されているアプリケーション・パフォーマンス監視、ログ分析、IT分析サービスに加えて、新しいサービスが積極的に開発されています。 そして最も重要なのは、開発者や運用サービスなどのさまざまなITサービスチームが共通の言語を見つけて効率的に作業できることです。
