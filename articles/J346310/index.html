スモールビジネスがWindowsからLinuxへの切り替えを真剣に考えている場合の対処方法

まえがき 私の話が、Windowsを削除せずにWindowsからLinuxに切り替えるために、遅滞なく初心者または優柔不断なユーザーに役立つことを願っています。



スタート。 運命の意志により、私はシステム管理者であり 、サービス部門の小規模企業（10台未満のマシン）のenikeyschikです。 経営陣は1年以上Linuxに切り替えることを考えてきましたが、最初の実験（Fedoraの選択）は完全に成功したとは言えませんでした。プログラムや機器を操作するインターフェースは好きではありませんでした。 もちろん、将来、間違いを認識しました。悪いインターフェイスを選択し、詳細を説明せずに最小限のドライバーをインストールしただけです。



しかし、経営陣は悲しみながらも失望することはなく、優れたPythonプログラマー（彼に対する最大限の敬意）との別の会話の後、インターフェイスを備えたすべての稼働中のマシンにLinux Mintをインストールしようとすると、私は困惑しました。

Mate（正確にはlinuxmint-18.1-mate-64bit）とWindowsとの並行。 このオプションは、通常のWinインターフェースに最も似ているように思われ、その時点でOSバージョンが最新でした。



このすべての主な目標は、フリーソフトウェアを入手することでした（企業のWIN10では、1万から2万の領域に何かを置く必要がありましたが、正確には覚えていません）。 同社は評判を高く評価しており、ライセンスのないソフトウェアを使用したくありませんでした。 同時に、システムの外観を維持し、Linuxへの移行を可能な限り快適にすることが非常に重要でした（これにより、レビューには多くの個人設定がありますが、これについては後ほど説明します）。 Windowsを放棄する2番目に重要な理由は、ロシアに対する米国の極めて非友好的な政策でした。 "）。



まあ、目標が設定されています。 実装を開始しました。 先に進んで、このテーマに関するHabréの記事全体を作成することにした理由を説明します。このトピックは、国内のほとんどの小規模企業に関連していますが、大規模なリソースではあまりカバーされておらず、そのリソースには多くの資料がありますが、平均的なユーザー向けの賢明な教育プログラムを見つけるためにサービス部門の小企業-それは困難であることが判明しました。



当初、私は自分自身のために指示を作成しただけだったので、将来はすべての設定を忘れることはありません。

ですから、私は長年の仕事の成果を共有せざるを得ません。 最後に、ITから遠く離れた友人が突然WindowsからLinuxに切り替える方法を尋ねた場合、この記事を簡単に投げることができます。



1. rufusプログラムを使用して、ダウンロードしたLinuxイメージをUSBフラッシュドライブに書き込みます。



2.既存のハードドライブを2つの部分に分割して、Linux用のパーティションを選択します。

Windows /コントローラー/管理/コンピューターの管理/ディスクの管理



3.再起動し、BIOSをオンにし、フラッシュドライブからの起動を有効にします（読み込みに問題がある可能性があります。UEFI/レガシーモードを選択する必要があります。



4.インストール時に、「サードパーティソフトウェアをインストールする」チェックボックスをクリックすると、おそらく「Windowsの隣にインストール」と表示されます。 「別のオプション」をクリックします。 次に、リストの中で特にLinux用に残したセクションを探します。ボリュームや空であるという事実から簡単に見つけることができます。 それを削除し、2つの新しいパーティションを作成します-スワップパーティション（たとえば4GBを入れます。通常はRAMと同じ量をインストールしますが、多くは有害でもあります-これは論点です）。 ext4（これはWindowsのntfsに似たファイルシステムです）と "/"のマウントポイントに残りのスペースを割り当てます。 重要です。 誤ってefiセクションを削除しないでください。 復元は高速ではありません。 これらすべてをLinuxで実行できるプログラムはgpartedです。



5.安全性に影響を与える小さなニュアンス。 インストール中にシステムの動作をデバッグするときに、コンピューターに信頼できる人しかいない場合、TeamViewerを介してリモートで再起動および構成できるようにする場合は、「システムに自動的にログインする」チェックボックスを残します。 また、vryatliの単純なユーザーは、TeamViewerにWindowsとの類推によってブート時にシステムに強制的にログインさせることができます。 リモートの再起動と管理のための永続的なソリューションが必要な場合は、sshを構成する必要があります。



6.おそらくLinuxのインストールのトピックではありませんが、一般的には、再インストールを行う人への良いアドバイスです。ハードドライブをssdに変更すると、システムと多くのプログラムの読み込みが大幅に高速化されます。



7.再起動後、grubメニューにオペレーティングシステムのリストが表示されます。 オペレーティングシステムのリストと順序を編集するには、grub-customizerを使用して、以下を設定します。 



sudo add-apt-repository ppa:danielrichter2007/grub-customizer
sudo apt-get update
sudo apt-get install grub-customizer
