（*） Perlの哲学は、難しい問題を解決する能力を維持しながら、簡単な問題を簡単に解決できるというものです。 これらは私たちが日々解決するタスクです。 あなたがやろうとしていることを言う前に予備の言葉をたくさん言う必要はありません。 Perlは伝統的にほとんどすべての* nixディストリビューションの配信に含まれているため、特に簡単に起動できます。 実際、オペレーティングシステムをインストールした後、Perlは動作する準備ができています（もちろん* nixについて話している）。



（*） Perlの新しいバージョンを開発するときは、下位互換性の維持に特別な注意が払われます。 Perlでどれだけ書かれているかを考えると、これは非常に重要です。



（*） CPAN （ maven 、 ivy ）/ Java、 RubyGems 、 npm /Node.JSのようなPerlのモジュール配布ネットワーク。 布の自己組み立て、あらゆる場面に対応する既製のソリューションの倉庫。 現時点では、言語の中で最大の1つ（ 121,000以上のモジュール）。 CPANのおかげで、多くの複雑なタスクが適切なモジュールの簡単な検索によって解決されます。 ただし、mavenとの比較には多少の負担がかかり、CPANは依存関係管理の点ではるかに簡単です。 事前の設定マジックは不要で、Debian / Ubuntuのaptに似ています。



（*） Perlは、テキストプログラムを解釈する前に中間表現にコンパイルします。 これにより、まれではありますが、 コンパイル段階でプログラムのエラーを検出できます。 まず、構文構造の正確性の確認、サブプログラムの名前と引数の正確性の確認（プロトタイプがある場合のみ）、クラスのフィールドの呼び出しの正確性の確認が含まれます。 もちろん、これはC ++コンパイラではありませんが、動的言語の場合はまったく悪くありません。 興味深いことですが、C ++コンパイラーでも何かを残すことができるため、Perlコンパイラーは数学演算への誤った呼び出しを検出できます（例えば、負の数からのルートはコンパイルエラーを引き起こします）。 以下に例を示します。 



 #!/usr/bin/perl -w use strict; my Dog $aDog = new Dog; # :   Dog $aDog->{wings} = '?'; #  (. ):     feed $aDog; # :     sqrt -2; # :




Perlは常にその原則に従い、これらのチェックのほとんどをオフにするのに費用はかかりませんが、それについては後で詳しく説明します。



（*）関数プロトタイプを使用すると、C ++の前方宣言に似たサブルーチンとその引数のリストを事前宣言できます。プロトタイプがある場合、コンパイラーはサブルーチンが正しく呼び出されることを確認し、サブルーチンを呼び出すときにペアの角かっこも必要としません。 



 sub feed($$); feed 'Sharik', 'Fruit'; # Ok feed 'Sharik'; # :    feed 'Sharik', 'Apple', 'Fruit'; #:   sub feed($$) { print shift, ' eat ', shift, "\n"; }




（*）$ @％ 。 Perlは、目的に応じて変数を分離します： ' $ 'スカラー（数値、行、リンク）、 ' @ '配列（スカラーのシーケンス）、 ' ％ 'ハッシュ（スカラーの連想配列）。 ご覧のとおり、古典的なプリミティブ型はスカラーで表されています。 ただし、これらはコンテナから厳密に分離されているため、コンパイラは追加のチェックを実行でき、面倒から解放されます。 Perlはその型の入力を許可していません（OOPの場合、譲歩が導入されています-組み込み型のいずれもクラスに加味できます）。 このような明確な分離により、同じ領域で、すべてのタイプの競合しない変数を持つことができます。 



 #!/usr/bin/perl -w use strict; my $solarSystem = 'Mercury Venus Earth Mars Jupiter Saturn Uranus Neptune'; my @solarSystem = qw(Mercury Venus Earth Mars Jupiter Saturn Uranus Neptune); my %solarSystem = (haveLife => 1, havePerl => 1); sub solarSystem() { print 'Mercury Venus Earth Mars Jupiter Saturn Uranus Neptune'; }




Perlの各操作は、実行コンテキストに関連して分離できません。 Perlは、空（コンテキストなし）、スカラー、およびリストの3種類のコンテキストを理解します（たとえば、スカラー変数に値を割り当てると、スカラーコンテキストを形成し、配列に割り当てるか、リストコンテキストをハッシュします）。 Perlルーチンは実行のコンテキストを決定することができ、したがって、異なるコンテキストで異なる動作をすることができ、創造性の余地を与えます。



この言語にはポインター演算はありませんが、リンクがあります。 サブプログラムへの引数は参照によって渡されますが、代入演算子は変数自体の値をコピーします（スカラーだけでなくコンテナもコピーされます）。 したがって、値によってサブルーチンに引数を渡すことは簡単に実装されます。 さらに、Perlで書く古典的なサブルーチンは、値渡し、つまり 引数に副作用を導入しません。 



 #!/usr/bin/perl -w use strict; sub upper { my $value = $_[0]; #    $value $value = uc $value; #    $_[0] = $value; #      } my $bigLetters = 'big-letters'; upper $bigLetters; print $bigLetters, "\n"; #  BIG-LETTERS




（*） オペレーターのオーバーロード （まれに、必要な場合）。 スペース、コンマ以外のほとんどすべてをオーバーロードできます。おそらく、誰かがコメントをオーバーロードできると聞いたことはありません。 定数（数字、文字、単語）をオーバーロードできる唯一の言語（私に知られている）。 これは、たとえば、定数として入力されたすべての数値を、複素数を表す同等のオブジェクトに置き換えるために使用できます。 または、無制限の値の数字を使用します。 



 #  BigMath.pm package BigMath; use overload; use Math::BigInt; sub import { overload::constant integer => sub { Math::BigInt->new(shift) }; } 1; #  BigMath.pl { use BigMath; my $bigInt = 2123; #  BigInt print $bigInt->as_bin(); # 0b100001001011 } { my $int = 2123; print $int->as_bin(); #   as_bin() }




Perlを使用すると、標準言語ルーチンをオーバーロードできます。



（*） 正規表現は Perlに深く統合されています。 通常の正規表現パターンに加えて、Perlはその拡張機能をサポートしています。 たとえば、Perlでは、検索パターンおよび置換パターンで直接Perlコードを使用できます。 



 my $math = '100 + 200 = 300'; $math =~ s/(\d+)/$1 * 2/ge; #       2 print $math, "\n"; # 200 + 400 = 600




正規表現はUnicodeプロパティをサポートし、独自の文字プロパティを入力することもできます。 複雑なパターン（\ s、\ wなど）には独自の省略形を入力できます。当然、Perlの正規表現は、貪欲な検索と貪欲でない検索の両方をサポートします。



（*）サブルーチンにreturnキーワードがない場合、Perlは最後に評価された式を返します（これは現在流行しているようです）。 さらに、サブルーチンを呼び出すときに、何をしているのか理解している場合は括弧を省略できます。



（*） Perlのブロック。 ブロックは中括弧{...}のペアで、ブロックを受け入れるための while 、 if 、などの構成体ですが、ブロックは単独で存在できます。 最も興味深いのは、ブロックがnext 、 last 、およびredo演算子をサポートしていることです。 つまり エントリは非常に合法です： 



 { print "Yeah\n"; last; print "Unreachable code\n"; } { print "Forever loop\n"; redo; }




さらに、サブルーチンは引数としてブロックを取ることができます。これにより、構文構造を使用して言語を豊富に拡張できます。 これにより、美しい、自己文書化されたプログラムをPerlで作成することができます。もちろん、想像力が必要です。 



 use strict; #sub transaction(&$); #sub update($$$); #sub set(+); #sub where(+); #sub rollback(&); #sub throw($$); transaction { update 'user', set { name => 'caiiiycuk' }, where { id => '1' }; } rollback { throw 500, "rollback exception"; }




Perlはtry / catch / finally ブロックをサポートしていませんが、ブロックルーチンを受信することで簡単に実装できます。 正確にif / elseがブロックを受け入れるため、if / else if / elseの構文はPerlに存在できないため、 if / elsif / elseが代わりに使用されます。 同様に、 switch / caseはありません。簡単に実装されます。



（*） whileループはcontinueセクションを定義できます。next 、 last、 またはredoが呼び出されたかどうかに関係なく実行されます（つまり、常に実行されます）。 



 while (1) { next; } continue { print "Yet another loop done \n"; }




（*）変数の3つのスコープ：グローバル、レキシカル、およびローカル。 ローカルスコープは、変数の値を現在のブロックおよびこのブロックからのすべてのサブルーチン呼び出しに拡張し、特定のブロックのグローバル変数の値をオーバーライドするのに役立ちます。 ローカルスコープは実行時に決定され、コンパイル段階でレキシカルに決定されます（これはブロック内で最も一般的なスコープです）。 



 use strict; our $a = 'global scope'; #   sub printLocal { local $a = 'local scope'; #   (    ) printGlobal(); } sub printGlobal { print $a, "\n"; } printGlobal(); # 'global scope' printLocal(); # 'local scope' printGlobal(); # 'global scope'




（*） Perlはクロージャーを実装します。匿名サブルーチンを宣言するための構文は、他のいくつかの言語で起こるように、通常の宣言と同じです。



（*） 2002年以来、Perlは定数サブプログラムを計算し、その呼び出し場所を計算された定数（ constexpr ）の値に置き換えるメカニズムをサポートしています。



（*）便利なファイル検証演算子（ -rwxoRWXOezsfdlpSbctugkTBMAC ）。 



 use strict; if (-e '/tmp' and -d '/tmp') { print '/tmp is directory', "\n"; }




（*）演算子<> 。 いくつかの機能を実行します。 まず、現在のディレクトリ内のテンプレートでファイルを検索するために使用できます（ところで、検索方法は変更できます。デフォルトではglobが使用されます）。 



 print join ',', <*.pl>, "\n"; #   *.pl




第二に、ファイル記述子からデータを読み取るための演算子です。 さらに、スカラーコンテキストでは、データを1行ずつ読み取り、リストコンテキストでは、ファイルの行を完全に配列に読み取ります。 



 open F, '<somefile' or die; my @random = <F>; #      close F;




そして最後に、空の<>演算子は、プログラム起動の引数で渡された名前のファイルから読み取ります。 1つの大きなファイルで作業している場合のように、明示的にこれを行いません。 プログラムが引数なしで起動された場合 、 STDINから読み取ります 。 ファイルを開いたり閉じたりすることを心配する必要がないので、これは|を通して結合されるパイプラインプログラムの開発に非常に便利です。 この演算子を適用すると、 catコマンドの類似物が1行で実装されます。 



 print while (<>);




（*）便利なデフォルト変数$ _ 。 ほとんどの標準Perlルーチンと演算子は、それで動作できます。関数の入力引数が省略されている場合は、その使用をお勧めします。 これにより、より簡潔なコードを記述できる場合があります。 



 use strict; while (<>) { print if (s/^print\s*(.*)/$1/); } #  while ($_ = <>) { print $_ if ($_ =~ s/^print\s*(.*)/$1/); }




（*） Perlでは、割り当て結果は左辺値です。 したがって、Perlでは、割り当て中に値を変更できます。 



 ($new = $old) =~ s/bad/good/g; ## chomp($answer = <STDIN>); ## if ( ($k, $v) = $string =~ m/(\w+)=(\w+)/ ) { print "key: $k value: $v", "\n"; }




（*）文字列に適用されるインクリメント演算子は、文字セットから次の順列を返します。 



 my $a = 'aa'; print ++$a, "\n"; #  ab




（*）ハッシュと配列を任意の深さに自動的に拡張します。 つまり まだ追加されていないキーでハッシュにアクセスすると、この要素が自動的に作成されます。 Perlは、作成する要素のタイプを推測します。たとえば、匿名ハッシュ、匿名配列、またはスカラーを作成できます。 



 my $hash = {}; $hash->{first}->{second}->{third} = 5; print $hash->{first}->{second}->{third}, "\n"; # 5




興味深い機能は、配列内の負のインデックスの動作です。 負のインデックスは、配列の末尾から要素を選択します。



（*）文字列と引用符。 Perlが1つの割り当てで複数行入力をサポートしているのは素晴らしいことです。 



 my $doc1 = 'There's More Than One Way To Do It -- 2012'; my $doc2 =<<"_DOC2_"; There's More Than One Way To Do It -- 2012 _DOC2_ print<<"_DOC3_"; There's More Than One Way To Do It -- 2012 _DOC3_




演算子= <<を使用すると、割り当てと同じ行で複数行のドキュメントの操作を簡単に実行できます。 また、ドキュメントを関数に転送するためにこの演算子を使用すると便利です。 



 use strict; sub printAll(@) { print join '', @_; } (my $quote =<<'__QUOTE__')=~s/^\s+//gm; The Road goes ever on and on, down from the door where it began. __QUOTE__ printAll(<<'__GOOD__',<<'__BAD__', $quote); There's More Than One Way To Do It __GOOD__ TIMTOWTDI __BAD__




Perlは、文字列を入力する2つの方法（解釈ありと解釈なし）を区別します。 単一引用符'を使用すると、文字\ nは改行に変換されなくても、文字列はそのまま（変更なしで）変数に割り当てられます。 一方、二重引用符"を使用すると、文字列が解釈されます-Perlは変数名を検出して値で置き換え、非常に巧妙に行います。文字列から関数を直接呼び出すこともできます。 



 use strict; my $car = { brend => 'Lada', name => 'Kalina', drive => sub { "WEEEEEE!\n"; } }; #  Do you want to drive on $car->{brend} $car->{name}?\n print 'Do you want to drive on $car->{brend} $car->{name}?\n', "\n"; #  Do you want to drive on Lada Kalina? print "Do you want to drive on $car->{brend} $car->{name}?\n"; #  WEEEEEE! print "@{[ $car->{drive}->() ]}"; #  print "@{[ exit ]}";




ただし、Perlには、文字列と正規表現をエスケープする文字を選択する権利があります。 



 print q {   ! }; (my $text = "ab c") =~ s{\s}{}g;




Perlのもう1つの非常に便利な機能は、文字列エスケープ文字として `を使用するときの文字列の動作です。 この場合、Perlはシェルでこの行を実行し、コマンドの結果を返します。 



 print `uptime`; #    print << `EOC`; #   pwd ls -l EOC




（*） Perlは興味深いサブルーチン属性メカニズムをサポートしています。 Javaからの注釈で類推できます。 属性は、コロンを使用してサブルーチン宣言で指定されます。 任意の数のマルチユーザー属性を宣言できます。 組み込み属性methodおよびlvalueもあります。 すばらしい左辺値属性により、サブルーチンは左辺値として操作に参加できます。 そのため、たとえば、同じルーチンでgetメソッドとsetメソッドを実装できます。 



 my $_name = '?'; sub name(;$): lvalue { $_name = shift || $_name } name = 'caiiiycuk'; print name(), "\n";




（*）顕著なAUTOLOADメカニズムにより、 宣言されていないルーチンへの呼び出しを処理できます。これは、プログラムを部分的に動的にロードするためによく使用されます。 Perlがサブルーチン宣言を見つけられない場合、関数シグネチャでAUTOLOAD呼び出しを呼び出し、失敗した場合のみエラーをスローすることを意味します。 たとえば、 AUTOLOADを使用すると、それ自体が（ シェルモジュールのように）Perlステートメントであるかのように、システム関数をより明確に呼び出すことができます。 



 use strict; sub AUTOLOAD { my $program = our $AUTOLOAD; $program =~ s/.*:://; system($program, @_); } sub uptime(); sub echo(@); uptime; echo "AUTOLOAD";




（*） tie-スカラー、配列、ハッシュの内部実装を置き換えるメカニズム。 このメカニズムを使用すると、配列またはハッシュをデータベースに直接関連付けることができ、データベースへの読み書きがデータベースに自動的に表示されるようになります。 この技術の別の可能なアプリケーションは、リソースの割り当てです。



（*） Perlは、スレッドモデルとプロセスモデルの2つのマルチスレッドモデルをサポートしています。 プロセスモデルには、並列データ処理用の親プロセスのデータの完全なコピーを含む新しいシステムプロセスの作成が含まれます。 これは、 forkコマンドを使用して行われますが、Windowsではプロセスモデルが完全にサポートされていますが、この並列化モデルは* nixでより一般的です。 bare fork関数を使用するのはあまり便利ではありません。CPANにはforkを簡単に使用できるモジュールがいっぱいです。たとえば、コードは次のようになります。 



 use strict; use Proc::Fork; run_fork { child { print "Process 1\n" while 1; } parent { print "Process 2\n" while 1; } }




プロセスモデルとは対照的に、スレッドモデルは際立っており、これらのストリームはすべてのデータリソースへの共有アクセスを持ち、そのようなストリームはJavaストリームに似ています。 スレッドサポートモジュールはThreadと呼ばれ、明確なコードを書くのに役立つ非常に便利な非同期関数を提供します。 



 use Thread 'async'; my $thread1 = async { sleep 5; return 1; }; my $thread2 = async { sleep 2; return 2; }; print $thread1->join(), "\n"; print $thread2->join(), "\n";
