ロシアの個人顧客をGoogleが排除

昨日、Googleから次のような手紙を受け取りました。

こんにちは



Google Cloud Platform、Firebase、またはAPIを使用しているため、このメッセージを受け取りました。 アカウントに影響するGoogle Cloudサービスの運用の変更についてお知らせします。



2016年12月7日以降、Google Cloud Platform、Firebase、およびAPIサービスは、商業目的でのみロシアで使用できます。 この点に関して、2016年12月7日に、アカウントのタイプは個人ではなく法人に変更されます。



アカウントの種類「法人」は、商業目的で使用することを意味します。 このようなアカウントの所有者は、営利団体、アフィリエイト、個人起業家、またはパートナーシップです。



アカウントの支払いにはVATは含まれません。ロシアで設定された税率で税額を個別に計算する必要があります。



Google Cloud Platform、Firebase、およびAPIサービスを非営利目的で使用する場合、2016年12月7日までにそれらの使用を拒否する必要があります。これを行うには、すべてのプロジェクトの支払いを無効にします。 変更の詳細については、ヘルプセンターをご覧ください。



ご質問がある場合は、Google Cloudサポートまたはアカウントマネージャーにお問い合わせください。




私にとって、Google Container Engineのユーザーとして、これはまず第一に「さあ、プロジェクトを一手に引き受けて失敗する」ということです。



また、GoogleのAPI（たとえば、私にとって興味深いGoogle Maps APIやYouTube API）を使用する場合は、少なくともIPを発行する必要があることがわかりました。 どういうわけかこれは奇妙です、多分私は何かを誤解した、誰かが説明できますか？..また、私は急いで私のドッカー/ kubernetesコンテナ用の新しいクラウドを探します...
