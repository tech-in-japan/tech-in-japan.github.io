InfoboxCloudのAzure Packインフラストラクチャクラウドで利用可能な最新のWindows Serverバージョン1709

Windows Serverバージョン1709がすべてのユーザーのAzure Packインフラストラクチャクラウドで利用可能になったことをお知らせいたします。









この楽しいイベントを記念して、2017年末までAzure Packインフラストラクチャクラウドの新規ユーザーに対して25％の割引を提供します。



登録して25％の割引を受ける方法



見積もりにアクセス、相談、またはリクエストするには、フォームに記入してください 。 コメントフィールドで割引を受けるには、「イノベーションのために25％の割引が必要」と指定します。



クラウドの説明はこちらからダウンロードできます 。



クラウドインフラストラクチャのコストは、公式の計算機で計算できます。 Windows Serverのライセンス、ライセンスコアの数、ソフトウェアアシュアランス（Windows Server 1709を取得できない場合）の支払いについて心配する必要はありません-Datacenterエディションの現在ライセンスされているオペレーティングシステムは、追加料金なしでクラウドで既に利用可能です。



必要なサーバーは事前に作成され、ウェルカムレターのデータを使用して接続します。 それでも、自分でサーバーを作成する場合は、ネットワークまたはサーバーの作成に関する マニュアルまたは記事を使用できます 。







最小限のクラウドリソースにも適しています


Windows Serverバージョン1709は、ASP.NET Core 2およびSQL Server 2017のサイトを含むクラウド内の最新のアプリケーションおよびサービスを、信頼できるデータウェアハウスとして、またコンテナーを起動するために積極的に使用されると予想されます。



1つのvCore、2 GBのRAM、25 GBのストレージ、1つのIPv4、2つのvLAN、2つのサイト間VPNを備えた可能な限り最小のクラウドサブスクリプションでも、Windows Serverバージョン1709を使用できます。



新しいWindows Serverリリースサイクル



Windows Serverバージョン1709は、絶えず変化する世界で現代のソフトウェアシステムに必要な新しい機能を最も迅速に実装するために、6か月ごとに新しいバージョン（半年チャネル、SAC）をリリースするサイクルを持つMicrosoftの最初のOSです。 このOSの各バージョンのサポート期間は18か月です。 Windows Server 1709は、クラウドおよびソフトウェアアシュアランスサブスクリプションでのみ利用可能です。 OSの新しいバージョンを購入する必要はありません。クラウドおよびソフトウェア保証では、顧客はサブスクリプションの料金を支払い、現在のOSを使用します。



Windows Server 2016は、Long-Term Servicing Channel、LTSCの最新の現在のOSに遅れをとっており、2〜3年ごとに新しいバージョンをリリースするサイクルと、5年間の定期サポートと5年間のサポート期間があります。

このリリースサイクルでは、OSにオプションでグラフィカルインターフェイスをインストールできます。



クイックリリースサイクルを使用する場合、タイトルには「バージョン」という単語が使用されます。 したがって、Windows Serverバージョン1709、Windows Server 2016と言うのは正しいことです。



Windows Serverの新機能



変更の完全なリストはTechnetで入手できます。



クラウドユーザーに関連するいくつかの改善：



  • Server Coreコンテナが占めるスペースは60％少なくなりました。
  • Nano Server Containers-80％少ないスペース（以前は390 MB、現在は80 MB）
  • Hyper-V分離を備えたLinuxコンテナーのサポートが追加されました。


Nano Serverはもはやホストになれません-現在、このOSはコンテナでの使用のみを目的としています。



Webインターフェイスを介してサーバーを管理するには、前の記事で確認したProject Honoluluを使用できます。

最大のパフォーマンスとセキュリティを実現するためのグラフィカルインターフェイスを備えたユースケースはありません。



Windows Serverバージョン1709では、クライアントとサーバーSMB1はプレインストールされなくなり、SMB2以降のゲスト認証はデフォルトで無効になっています。



Essentialsエディションは利用できなくなりました。StandardとDatacenterのみです（クラウドにはすべてのDatacenter Windows Serverエディションがあります）。







これで、クラスターを再構築せずにドメインを変更できます。これは、最初にサンドボックスを構築し、構成してから運用環境に切り替えるときに非常に便利です。



クラスタをグループ（クラスタセット）にグループ化して、管理と使用を簡素化できるようになりました。 クラスター集約には、異なる計算ノード、ストレージノード、またはハイパーコンバージドクラスターを持つ疎結合クラスターが含まれる場合があります。 これにより、クラウドプロバイダーは、たとえば、HDD、SSD、NVME、ストレージクラスメモリ（SCM）、さまざまな種類のCPU、GPUなどのさまざまなクラスター間でVMを自動的に移行でき、クラスターサイズの制限を取り除き、クラウドを数千のホストサーバーにスケールし、ゾーンを構成できます可用性（グローバルAzureと同様）。



最新のファイルシステムとデータ重複排除のどちらかを選択する必要がなくなりました。ReFSでもサポートされており、ディスク容量を最大95％節約できます。



リモートデスクトップサービスをAzure ADと統合できるようになりました。統合により、ユーザーは単一ポリシー、多要素ログイン、他のSaaSアプリケーションとの統合認証を使用できます。



コンテナ化の機能強化


Dockerルーティングメッシュのサポート-Swarmクラスターの外部でサービスポートを公開する機能。



Windows上でLinuxコンテナを実行する機能 。 Linuxコンテナーを実行するには、 LinuxKitプロジェクトを使用します。



SMBをディスクとしてコンテナに接続する機能 。 Windows Serverバージョン1709では、SMBボールをドライブとして指定し、コンテナ内のドライブ文字を使用できます。



おわりに



Azure Packインフラストラクチャクラウドへの新しいテクノロジの迅速な実装が、プロジェクトの成長を加速させ、競合他社をリードすることを願っています。 新しいOSは非常に高速で効率的であるため、クラウド内で使用するリソースを少なくし、費用を抑えることができます。
