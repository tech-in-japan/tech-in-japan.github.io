Operaブラウザの大きな歴史





当時、一部のユーザーだけがグローバルネットワークの広大さをサーフィンしていたとき、一般の人々は数年後に「クラブへようこそ！」と言われることを疑いませんでした。 明るい未来を信じている企業は、彼らをすぐに大きなウェブに引き込もうとしました。



そのような会社の1つがノルウェーのテレノールでした（1995年まで-Televerket）。 従業員のStephenson von TechnerとGeirIvarsøyは、特別なODAデータ収集および処理システムを開発しました。 しかし、1993年に彼らは新しい仕事に切り替えました-会社の個人的なウェブサイトの作成。 その間、それは大胆で革新的なソリューションでした。



その過程で、彼らは当時最も人気のあったインターネットブラウザーMosaicを積極的に使用しました。 すぐにTechnerとIvarsøyは、彼がWebサーフィンに適していないと結論付けました。 開発チームはイニシアチブを取り、新しいブラウザを作成しました。これにより、当時最も差し迫った問題がいくつか解決されました。



•ページおよびドキュメントの正しい表示。



•Webからの情報取得の高速化。



•仕事のしやすさと利便性。



•プログラムおよびインストールパッケージのサイズが小さい。



•最小リソース要件。



•大量使用。



通信会社のリーダーシップはかなり先見の明があり、専門ではなく非常に有望なプロジェクトを承認しました。



MultiTorgからOpera Softwareへ



Operaブラウザのゼロバージョンは、1993年に登場したMultiTorgと見なすことができます。 このブラウザは、大量配布用ではなく、Telenorのニーズに合わせて設計されたものです。



ただし、MultiTorgはOperaの優れた基盤となっています。 ブラウザーのソースコードは、サードパーティのAPIを使用せずにC ++で記述されていたため、システム要件は当時のマシンでも最小限でした。



MultiTorg Opera 1.0は1995年に導入され、その後、専門家はすばらしい将来を予測しました。







1995年8月30日、お世辞のレビューに励まされて、Jon Stephenson von Techner（左）とGeirIvarsøy（右）はTelenorを離れ、独自のOpera Software ASを作成しました。 彼らの移行は、独占的な開発権を持っていたため、それほど困難なく行われました。 新会社の壁の中でリリースされた最初のリリースは、信頼できる専門家に「涙」を与えるために与えられました。



リリース



1996年に、次のリリースが登場しました。 Opera 2.1はシェアウェアとして登場しました。 ユーザーはプログラムの機能を無料で評価するために90日間の猶予がありました。 プログラムをさらに使用するために、一定の金額を支払うことが提案されました。



このような提案は敵意に満ちたものではありませんでした。ブラウザーはWebページをすばやくロードするという素晴らしい仕事をしたためです。これは当時の成功の主な兆候でした。

前世紀には、サイトの読み込みの問題がインターネット業界全体の基礎でした。



ブラウザの3番目のバージョンは、リソースに対してより信頼性が高く、要求が厳しいものでした。 ハードドライブ上の386および6 MBのPCで、ブラウザのすべての機能を自由に使用できました。 さらに、Opera 3は複数のウィンドウで動作する機能、オートコンプリートフォーム、TSLセキュリティプロトコルのサポートを備えています。



4番目のバージョンでは、設定システムが改善され、電子メールクライアントが登場し、最も重要なこととして、クロスプラットフォームが実装されました。



2000年に、同社はOpera 5.0をリリースしました。 IRCクライアントのサポートがブラウザに登場しました。



もう1つの革新は、マウスジェスチャーです。 その間、これは完全に新しい技術でした。 5番目のバージョンから、ロシア語のサポートが追加されました。 シェアウェアライセンスの条件は、バナー広告の必須表示に置き換えられました。 ブラウザは完全に無料になりました。



6回目のリリースが登場する頃には、OperaはすでにMicrosoft Internet Explorerの価値あるライバルと見なされていました。 第6バージョンの主な革新には、Unicodeのサポートとユーザーパネルの変更がありました。



2003年から2004年にかけて、Microsoftとの対立がピークに達しました。 ちょうどそのとき、Operaの7番目のバージョンが新しいPrestoエンジンを搭載し、CSS、クライアントスクリプト、DOMインターフェイスのサポートを改善しました。 さらに、動的レイヤーとマスターパスワードのサポートがブラウザーに追加されました。







2004年8月、Opera 7.6の限定アルファテストが開始されました。 実行可能なWeb標準のリストが拡張され、音声制御とVoiceXMLのサポートが追加されました。



マイクロソフトとの競合



それはすべて、2001年にMicrosoftがOperaを含むサードパーティブラウザで作業しているユーザーのMSN.comへのアクセスをブロックしたときに始まりました。 サイトへのアクセスは、Internet Explorerのユーザーのみに許可されていました。 反トラスト組織からの怒りの波の後、マイクロソフトはこの制限を解除しました。



2003年に、MSN.comは、Operaユーザーのサイトコンテンツの表示がゆがむように設定されました。まるで古いバージョンのInternet Explorerで動作しているかのようです。 他のブラウザは、最新バージョンのInternet Explorerのページの表示に対応する適切な画像を受け取りました。 古いマークアップコードにより、いくつかの重要なページ要素が30ピクセル左にシフトしました。 この理由はブラウザのエラーにあるという印象がありました。



Opera開発者はマイクロソフトに助けを求めましたが、返事はありませんでした。 その後、彼らは企業を不正行為で公に非難したが、Microsoftは悪意のあるすべての申し立てを拒否した。



これに対応して、Opera Softwareはビルド7.02 Borg Editionをリリースしました。これには通常、MSN.comを除くすべてのサイトが表示されます。

その後、Microsoftはサーバー上で必要な変更を行い、Operaの7番目のバージョンはInternet Explorerの最新バージョンと同じ方法でページを表示し始めました。







2004年11月、Opera SoftwareはMicrosoftオフィスにメールを送信し、Operaの所有者がHotmailメールサービスで不完全なJavaScriptファイルを受け取ったことを訴えました。 これは、ユーザーが不要な通信のカートを空にできないという事実につながりました。 これに続いて、Opera SoftwareはMicrosoftに通常のメールを送信しました。



ただし、企業はこれらの要求に応答せず、Hotmailサービスを修正しませんでした。



夜明け



2005年4月19日に、Opera 8.0がリリースされました。これは、今日まで、このブラウザーのすべてのバージョンのベストと見なされています。新しいカーネル、改良されたJavaScriptサポート、トレントからのダウンロード機能、音声制御。



バージョン8.0では、SVG 1.1 Tinyのサポートが導入されています。 Operaは、このグラフィック形式と互換性のある最初のブラウザです。 プレゼンテーションモード（オペラショー）もありました。このモードでは、プログラムインターフェイス要素を表示せずにHTMLまたはXMLドキュメントを全画面表示できます。



Opera 8.5のリリース（2005年9月20日）に伴い、Opera Softwareはブラウザが完全に無料で広告なしになると発表しました。 革新の中には、セキュリティ修正と、ユーザーが誤って描画したページを修正する機能があります。







第9バージョンでは、開発者はセキュリティ、信頼性、プライバシーに多大な注意を払いました。 そのため、Opera 9.0では、フィッシング対策保護が強化され、カーネルが再設計され、安定性が向上し、危険な脆弱性が排除されました。

Opera 9.xは13の脆弱性を特定しました。 13人の開発者全員がなんとか排除できました。

保護されたページにアクセスすると、OperaはSSL 3またはTLSの信頼性の高いプロトコルを使用して、送受信された情報をエンコードします。セキュリティの有効化に関する情報は、ブラウザのアドレスバーに表示されます。 ユーザーはそこに表示されるボタンをクリックして、ページが不正であるか、フィッシング用に作成されているかどうかを確認できます。 ユーザーはフィッシングのページの自動チェックをオンにできますが、デフォルトではこのオプションは無効です。



また、Arche Twistグループは、匿名のインターネットサーフィン用に設計されたOperaTorソフトウェアバンドルをリリースしました。 Operaブラウザ、Tor匿名ネットワーククライアント、Polipo仮想プロキシサーバーが含まれます。 HTTPおよびHTTPSプロトコルでの匿名化された作業。


人気の低下



10番目のバージョンでは、新しいブラウザデザイン、視覚的なタブ、Opera Turboが登場しました。 11番目のバージョンでは、拡張機能のサポートが実装されました。 12番目のバージョンでは、カメラ技術が改善されています。 さらに、Opera 12はPrestoエンジンに実装された最新バージョンです。



さらに、人気の低下が観察され始めました。 世界は「高速インターネット」の時代に突入し、ページの読み込み速度と全ページを表示する機能（Operaの切り札）は決定的な役割を果たしなくなりました。





13番目のバージョンから、ブラウザー開発者はPrestoからChromiumに切り替え、その後、従業員のスタッフを大幅に削減しました。



それ以降のバージョンはすべて、「無段階」のChromiumに基づいてすでにリリースされています。 一般的な考えによると、Operaはその時以来そのアイデンティティを失っています。



フラスコにはまだ火薬があります



2016年9月、OperaはビルトインVPN機能を備えたデスクトップブラウザーをリリースしました 。 これは視聴者の成長に貢献しました。ブラウザは200万人の新しいユーザーを受け入れました。これは会社にとって絶対的な記録でした。



「人々がインターネットが実際にどのように機能するかを知っていれば、私は彼ら全員がVPNを使用すると確信しています。 ブラウザベースのVPNを無料で使いやすくしたことで、VPNがユーザーにとって不可欠な機能になることを願っています」とOperaの上級副社長であるChristian Kolondra氏は述べています。



ブラウザの開発者によると、新しいバージョンを作成する際の最も差し迫った問題は、ネットワーク上の個人情報の保護でした。 そのため、無料の非ロギングVPNをブラウザに組み込み、サードパーティのVPNのインストールと構成に関連する不必要な費用や困難からユーザーを節約しています。







ビルトインVPNサービスの関連性を確認するために、開発者は 1万人のOperaユーザーを対象としたグローバル調査を実施しました。







中国のパートナー



7月、Qihoo 360が率いる中国企業のコンソーシアムがOperaを6億ドルで買収し、ブランド名を含むその他のいくつかの資産を取得しました。 Qihoo 360は、オンラインモバイルアプリストアとウイルス対策ソフトウェアで中国で知られています。 コンソーシアムには、Kunlunおよび投資ファンドのGolden BrickおよびYonglianも含まれます。







今年初め、ノルウェーの会社は同じコンソーシアムに12億ドルですべての資産を売却しようとしましたが、取引は行われませんでした。 新しい契約によると、コンソーシアムは会社の大部分を所有しています。有名なOperaブラウザのモバイルおよびデスクトップバージョンであり、モバイル市場の10％を占めています。



同社には、Opera Apps＆GamesとOpera TVの2つのエコシステムが残っています。 1,669人のスタッフのうち、560人の従業員が残ります。



Opera TV部門は、年間6億1,600万ドルの収益をもたらしました。 Operaによると、これは全収益の4分の3を占めています。 そうかもしれませんが、Operaブラウザは同社の主力製品でした。





OperaがChromiumエンジンに切り替えた後、このブラウザには未来がないという意見がありました。 ただし、まだ開発中です。 開発の方向性は変わったかもしれませんが、その歴史は続いています。
