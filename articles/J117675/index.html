Visual Studioの高速化、パートIII。 IDEパフォーマンス

最後の記事を書いた後、ソリューションのコンパイルの高速化とは関係のない情報がまだあることに気付きましたが、コードの作業中にVisual Studio自体のパフォーマンスをわずかに向上させることができました。 そして、私たちはまだかなりの時間コードに取り組んでいるので、高速で動作するIDEでこれを行う方がはるかに快適です。 残念ながら、この記事にはグラフがありません。メニューやウィンドウを開くのにどれだけ速くなったかを測定するのは非常に難しいからです。 それでも、誰かがこれらのヒントで価値あるものを見つけて、時間を節約できることを願っています。





Visual Studioランチャーのショートカットに追加/ nosplash

正直なところ、起動時間はそれほど短縮されませんが、心理的にはスタジオの読み込みが速くなっているように見えます。



F1ボタンを無効にする

これは「環境->キーボード」で行います。 とにかくこのボタンは使わないでしょ？ そして、誤ってクリックすると、VSのバージョンに応じて、新しいウィンドウが開き始めるか、ブラウザーが起動します。これには非常に長い時間がかかります。



アニメーションをオフにします

「環境->一般->環境ツールのアニメーション化」。 アニメーションを美人のファンに任せましょう。 しかし、私たちは厳しいプログラマであり、何も必要ありません。



スタートページを無効にする

スタジオを開始するには時間がかかります（特に、あらゆる種類のニュースやヒントのためにインターネットをクロールする場合）。 「環境->起動」で無効になっています。



ソリューションエクスプローラーでアクティブアイテムの追跡をオフにする

本当に使いますか？ そうでない場合は、オフにします。 プロジェクトとソリューションで無効化



さまざまな美しさを無効にする

ここで詳細に書かれています。 「環境->一般->視覚体験」で無効化



冗長なプラグインを削除する

各プラグインは、スタジオのパフォーマンスにとってマイナスです。 余分なものがある場合は削除します。



IntelliSenseを無効にする

IntelliSenseを使用していて、この機能の機能と動作に完全に満足している場合、あなたはまれな人です。 おそらく、 ReSharperやVisual Assist XなどのVisual Studioプラグインに慣れる必要があります。 彼らは価格に見合った価値があり、IntelliSenseに取って代わり、他の便利なことをたくさんやっています。 これらのプラグインのいずれかを使用している場合、IntelliSenseを無効にすることは理にかなっています。 これを行うための定期的な方法はないため、シャーマンになります。

  • Visual Studioを閉じます。
  • ソリューションのすべての.ncbファイルを削除します。
  • ファイルCを削除または名前変更します：\ Program Files \ Microsoft Visual Studio 9.0 \ VC \ vcpackages \ feacp.dll（「9.0」の部分はVisual Studioのバージョンによって異なります）。


更新 ： Paulの Habraiser は 、設定によってIntelliSenseがVS 2010を無効にすることを提案しています。



仕事に来る前のプロジェクトのスタジオの立ち上げとコンパイル

私の作業マシンでは、毎朝、私が仕事に来る1時間前に、Visual Studioを閉じ、リポジトリからコードを更新してStudioを起動し、その中のソリューションを新しい方法でコンパイルするスクリプトが実行されます。 作業を始めると、オープンIDEが表示され、ソリューションの現在の状態（コンパイル済み\ no）と更新手順ログがすぐに表示されます。これにより、昨日の夕方から何が変更されたかがわかります。 これにより、毎日10分節約できます。
