毎年恒例のプラットフォーム2010カンファレンスは昨日作業を終了しました。 2日間の忙しい仕事の後、結果の一部を要約できます。



2日目には、Visual Studio、C＃、ASP.NET 4.0、Silverlight 3、およびWindows Mobile 6.5に関するレポートに参加しました。 それらについてさらに詳しく説明したいと思います。





Windows Mobile 6.5は、モバイルアプリケーションで魅力的で機能的なインターフェイスを作成するための強力なプラットフォームです。



講演では、UIフレームワーク、対話型アプリケーションを作成するためのGesture API、およびWindows Mobile 6.5用のウィジェットを作成する方法について説明しました。



ウィジェットは、HTML、CSS、およびJavaScriptを使用して開発された小さなアプリケーションで、たとえば為替レートの変換、天気の表示などのウィジェットなど、いくつかのロジックを実装しています。



ウィジェットは次のファイルで構成されている必要があります。

*ロジックを含むHTMLファイル

* XML形式のマニフェストファイル

*ウィジェットのアイコン（そうでない場合は、標準のアイコンが使用されます）



次に、* .zipアーカイブを作成し、権限の名前を* .widgetに変更する必要があります。 その後、Windows Mobile 6.5を実行しているデバイスにインストールできます。 詳細については、techdays.ruのビデオレポートをご覧になるか、 こちらをご覧ください 。

最新のWindowsアプリケーションの開発-Windows Presentation Foundation 4.0



レポートは、最新のWPF 4.0テクノロジーに当てられました。 WPFで記述されたMicrosoft自身のプロジェクト（Expression行、Visual Studio、プレーヤーなど）に少し重点が置かれました。 Visual Studioの改善されたWPFエディター、ExpressionはTFSで作業する機会を得ました。



新しいバージョンでは低解像度でのフォントレンダリングが改善され、新しいWPFはWindows 7のすべての新機能もサポートします。しかし、マルチタッチサポートが最も好きでした（これに関するビデオはchannel9で見ることができます）。 このために多くのコードを書く必要がないという事実が好きでした。

Silverlight 3.包括的なプラットフォームの概要



レポートでは、Mikhail Chernomordikovが、ブラウザ以外での作業、Smooth Streamingテクノロジー、3Dでの作業、ビジネスアプリケーションの作成の可能性について述べました。 しかし、このレポートのほとんどは、特にFireFox、Opera、Ghrome、SafariがSilverlightと友達になった多くのデモで記憶されていました。 4つのブラウザーが原則に基づいて構築されました。上部に2つ、下部に2つあり、それぞれに同じSilverlightアプリケーションが含まれ、その内部でボールがランダムに実行されました。 冗談は、あるブラウザで境界線の外に飛んだバルーンが、前のブラウザから飛んだ場所から別のブラウザに現れたことです。 したがって、4つのブラウザーすべてが1つのウィンドウとして機能するという錯覚が生まれました。 これは、Silverlightアプリケーションを互いに通信する機能のおかげで可能になりました。



他のテクノロジーと比較したSilverlightのパフォーマンスは、 http：//bubblemark.com/で入手できます。
