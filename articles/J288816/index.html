ウィキリークスに対するDDoS攻撃は10 Gb / sに上昇し、Amazonはまだ対処中です

昨夜、Wikileaksサイトでの2回目の DDoS攻撃のパワーは10 Gb / sを超えました 。 このサイトは、Amazon Web Services、米国およびアイルランドのサーバーでホストされていますが、これまでのところ、これらのサーバーは負荷にうまく対処しています。



DDoS攻撃の責任は、特定のアメリカの「ハクティビスト」 th3j35t3r （The Jester）によってすでに想定されていました。 彼のTwitterアカウントで、彼はWikileaksが「私たちの兵士や他の資産の生命を脅かす」と言ってこれを説明しています。 ジェスターのブログでは、「テロリスト、共感者、仲介者、助手、専制政権、その他の悪者の通信路に損害を与えるために、インターネットで最善を尽くしている」と述べています。



日曜日に、約25万件の秘密の外交文書がウィキリークスで公開されました。 サイトが大規模なDDoS攻撃を開始する数時間前に、今日まで続いています。 Arbor Networksの専門家は 、スウェーデンのサーバーが負荷に対処しなくなると、Wikileaks DNSサイトがスウェーデンのAS8473ホスティングからアイルランド（当時は米国）の分散ホスティングに変更されたことに注目しました 。







WHOISベースによると、 これはAmazon Webサービスホスティングであることがわかります。 最新の外交文書を直接含むcablegate.wikileaks.org Webサイトは、シアトルのデータセンターAmazon Web Services（ 204.236.131.131 ）とフランスの会社Octopuce（ 91.194.60.90 ）のIPアドレスを提供します。 Wikileaks.orgホームページは、 Amazon （ 184.72.37.90 ）およびIrish（ 46.51.171.90 ）Amazonサーバーを参照しています。



影響力のある米国の政治家がこのサイトの所有者をテロリストと同一視し、スパイ行為法に従ってそれらを判断するように促すという事実を考えると、アメリカのホスティングの選択はあまり合理的ではないようです。 アメリカ人はこれを非常に上手く行うことができます。スウェーデン当局による逮捕後にジュリアンアサンジを引き渡しました。スウェーデン当局は、セクハラの場合にすでにアサンジを逮捕するためにインターポールに申請しました。



同時に、専門家によると、Amazonは米国当局による法的訴追に直面する可能性は低い。 弁護士によると、重要な事実は、Amazonサーバーが配信を開始する前に情報が公開されていたということであり、これにより、会社は課金からある程度保護されます。 さらに、米国からの情報の普及により、言論の自由を保護する米国憲法修正第1条の効果を米国に拡大することができます。



Amazonホスティングは、ほとんどすべてのDDoS攻撃に耐えることができるため、サーバーの安定性を保証します。 少なくとも今では、サイトのパフォーマンスは高いレベルのままです。
