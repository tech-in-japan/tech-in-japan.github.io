ホームメディアセンターをどのようにまとめるか。 死後

最近、私の厳格な同志であるLenovo z61mラップトップがモニターを燃やしました。 熟考すると、私はそれが長くて高価な修理になることに気付き、新しいラップトップを買いました。 彼は古いラップトップをホームメディアセンターに適合させることに決めました。そこにはスクリーンが必要でなく、他のすべてが機能していたからです。



そのため、当初、Windows 7 Ultimateを実行するLenovo z61mラップトップ、Phillipsブラウン管テレビ、そしてそれを便利にしたいという要望がありました。 後で判明したように、最初の2つのコンポーネントはあまり適切ではありませんでしたが、後でさらに詳しく説明します。 投資を必要としないため、コンテンツをひねるプログラムを選択し始めたという事実から始めました。 Microsoft Media Center（MMC）とXBMCを試しました 。



MMCは、音楽ファイルのカバー、ビデオファイルのサムネイルを美しく表示し、Microsoft Media PlayerおよびMedia Player Classic-Home Cinemaで使用したコンテンツとまったく同じコンテンツを再生しました。 また、Microsoftのリモートコントロールを購入する予定であったことを買収しました。これは、ほとんどの場合、互換性の問題が最小限になることを意味します。



XBMCは、多数のスキンがあり、黒の色調のデフォルトスキンが元々MMCの青い背景よりも好きだったという事実を気に入っていました。 XBMCの豊富な設定、プラグイン、比較的頻繁な更新、および天気を表示するための組み込みプラグインもこのプログラムに関心を集めました。







この段階で、ソフトウェアには少なくとも2つの選択肢があると思い、鉄を取り上げました。 ラップトップには赤外線ポートがあり、それとテレビまたは古いDVDプレーヤーからのリモートコントロールを使用しようとする考えが生まれました。 さまざまなサイトで多数の指示を読み直し 、 WinLircなどのさまざまなプログラムをインストールして、Windowsでリモートコントロールを使用できるようにしました。 何もうまくいきませんでした。 ソフトウェアはハードに動作せず、Windowsでレジストリを編集し、ラップトップでドライバー、BIOS設定を変更しました-何も助けませんでした。 失敗した試行に約1日費やした後、受信機とリモートコントロールの周波数範囲が異なる可能性が高いため、このようなスキーム（ビルトインIRレシーバーと任意のリモートコントロール）は機能しないという結論に達しました。 ところで、彼らはフォーラムでそのような問題の可能性について警告しました。 特別に設計されたリモコンの購入に約600ルーブルを費やすという強い意志がありました。 :-)



私が最初にしたことは、以前にそのようなリモートがあったので、Omsk Nyxに行きました。 私が到着すると、すべてのリモコンが販売され、新しい配送は予定されていなかったことが判明しました。 どうやら動きの遅い商品。 その後、私は「Vist」に行きました。そこでは、リモートコントロールがなく、「DNS」には存在せず、今後も同じ成功を収めると言われました。 ほぼ必死、私は「リズム」に行きました-乾杯、そこに！ 価格は約360ルーブルでした（正確には覚えていません）。 リモートコントロールは、ある種のnonameです（写真をクリックできます）。



数日後、友人が「リズム」に行ったと言って、彼らはリモコンを持っていなかったし、持っていなかったと言ったが、「DNS」で彼は約900ルーブルでコンソールを買った。 そのため、販売コンサルタントが何かを言うときは信頼してください。



レシーバーをラップトップに接続すると、売り手が約束したように、すべてがすぐに機能しました。 ドライバーもプログラムもありません。 Microsoft Media Centerを起動しました-すべてが機能し、すべてがスクロールします。 やった？ 種類はありません。 :-(リモコンの範囲は、従来のテレビの範囲よりわずかに低いことが判明しました。そして最も重要なことには、リモコンには私にとって不要なボタンがあります：

  • 閉じる-ALT-F4のように機能します。つまり、プレーヤーを閉じます。
  • 電源-ラップトップの電源を切った後、リモコンから電源を入れることができませんでした
  • デスクトップ-WIN-Dのように機能します。つまり、デスクトップを表示します
  • Media Center-Microsoft Media Centerを起動します




リモートコントロールを試している間、私自身が誤ってデスクトップとメディアセンターのボタンを数回押したため、他の家族もボタンを押してしまい、「すべてが失われました。お父さんに電話して修正する必要があります」という状況になります。 小さな子供がいる場合の電源ボタンは、ラップトップを常に手に入れ、内部のボタンを押してスリープモードを解除することを保証します。 リモートの意味は失われます。 閉じるボタンも私の目的にはまったく不要です。 Media Centerボタンは、Microsoft Media Centerのみを起動します！ 代替プログラムを使用する場合、理解できます。



リモコンのボタンを再構成したり、ボタンの一部をブロックしたりする試みはすべて失敗しました。 レジストリ内のキーを変更し、ボタンを再割り当てするためのプログラムをインストールし、代替ドライバーをインストールしようとしましたが、何の目的もありません。 すぐに使えるように動作します。 何も変更することは不可能です。 確かに、私はいくつかの代替ソリューションを見つけました。

  • 電源ボタンを押して、Windowsの設定でスリープ状態にできないようにしました
  • 家族がデスクトップボタンをクリックした場合の対処方法を説明しました（もう一度クリックします）
  • [閉じる]をクリックした場合の対処方法を説明しました（矢印を使用してプログラムのショートカットを選択し、再起動します）
  • WindowsからMicrosoft Media Centerコンポーネントを削除しました（それまでに、使用しないことに決めていました）


その結果、条件付きで受け入れられるリモートコントロールが機能することが判明しました。



ソフトウェアの選択に戻ります。 次の理由により、Microsoft Media Centerを終了しました。

  • ゆっくり始まり、ゆっくり走る
  • ビデオファイルを表示した後、要約ウィンドウ（ロシア語で何と呼ばれているか覚えていません）が表示されます。このウィンドウでは、ファイルを削除したり、ディスクに書き込んだりできます。 このウィンドウは、インターネット上のフォーラム（ グリーンボタンフォーラムは非常に優れています）から判断すると、決してオフにはなりません。 個人的には、その存在は既に大きな不便であり、ファイルで何かをする能力はさらに大きくなります。 家に小さな子供がいることを思い出します。
  • 全体的なインターフェースの複雑さ（主観的）




プラグインでMMCがどれほど優れているかはわかりませんが、たとえばXBMCほど優れていると思います。



その後、 Media Portalを試しました。 しかし、どういうわけか私はすぐにそれが好きではなかった、それは彼がMMCやXBMCよりも悪いようでした。



XBMCを構成して使用することにしました。 一般に、すべてがうまくいきましたが、欠点もあります：

  • 一部のビデオがplay音で再生される
  • ビデオファイルを連続して再生できません。 つまり、漫画を開始した場合、それが終了した後、リスト内の次の漫画は再生を開始しません。 XBMCフォーラムから判断すると、この機会は5年以上にわたって要求されてきましたが、開発者はそうしたくありません。 フォルダーの再生（フォルダー内のすべての漫画が連続して再生される）を提供またはクリックするか、プレイリストを作成します。 開発者はこれで十分だと考えています。 人々がそのような決定に満足していない理由を説明しようとするすべての試みは、誤解の壁に打ちのめされます。
  • インターフェイスには、削除できない詳細があります。 たとえば、ファイルが外部ドライブにある場合は、最初にドライブを選択し、次にドライブ上のフォルダー、次にファイルを選択する必要があります。 ビデオファイルの配置を個別に指定した場合、ドライブを選択する理由-わかりません。 また、名前でファイルを検索する場合など、必要のない場所に移動することもできます（リモートコントロールを使用してこのモードを終了する方法は明確ではありません）
  • ファイル表示モード（リスト、サムネイルなど）の切り替えを禁止できませんでした
  • シリーズに関する情報の自動取得の構成に失敗しました。 どうやら、どういうわけか特別な方法でファイルをレイアウトする必要がありました-私はそれを理解できませんでした。 ちなみに、シリーズに関する情報は英語のみですが、これはもちろんプログラムのせいではありません。
  • kinopoisk.ruから映画に関する情報を受け取るプラグインは、常に正しく機能するとは限りません。 たとえば、ゾーリン大佐に関する3部作では、偶然としてアルパチーノと「ゴッドファーザー」を選びました。




XBMCでこのような比較的成功を収めた後、私はBoxeeも試してみることにしました 。 全体として、Boxeeは優れています。 インターフェースはもう少し思慮深く、主観的には、ナビゲーションがもう少し論理的になります。 しかし、オンライン動画などへの明らかなバイアス 私が使用する予定がなかったものは、インターフェイスとナビゲーションの喜びを完全に平準化しました。 XBMCに戻りました。



私はそれで十分であると判断しました、それはすべてを作動させる時でした。 彼はすべてをテレビにつなぎ、...恐ろしくなりました。 すべてはコンピューターで見栄えが良かったが、低解像度のテレビではフォントが小さく（構成を編集すると拡大できます）、画像が幅に収まらず（ラップトップでは16：9、テレビでは4：3）、斜めのストライプが走ります画面に。 あなたは許可を把握しようとすることができますが、私はそれを脇に置き、ヒントを取り上げました。 古いS-Video-> Tulipケーブルを介してラップトップをテレビに接続したため、新しいS-Video-> S-Videoケーブルを購入することにしました。 私はそれを買ったが、助けにはならなかった。 問題はケーブルにあるのではなく、ラップトップの電源がヒントを与えているという事実にあることが判明しました。 電源がオフになっていて、ラップトップがバッテリーで動作している場合、干渉はありませんでした。 そして、なぜ私はすぐにこれに気付かなかったのですか？



電源のコストは約2000〜2500千なので、その意味を変えるだけでは十分ではないと判断し、ホームシアターでの実験はこれで終わりです。 外部ドライブがUSB経由で接続されている古いBBK DVDプレーヤーに戻りました。 私はテレビを変えた後、二度目の試みに希望を残していません。



結論：

  • HD対応またはFullHDテレビがなければ、起動しないでください。
  • 開始した場合は、電源からの干渉がないことを確認してください。 液晶テレビやプラズマテレビの場合、これは関係ないと思います。
  • リモートは非常に慎重に選択する必要があります。 ボタンのセットとそれらの相対的な位置を確認する必要があります。 ボタンのサイズが変わったり、奇妙に配置されたりすることがあります。 たとえば、私が購入したリモコンでは、巻き戻しボタンは再生ボタンに対して不便な位置にあります（同じ行にある必要があります）が、スキップボタンは停止ボタンと同じ行にある必要があります。
  • ある程度のソフトウェアには欠点があります。 ほとんどの場合、すべてをセットアップ/変更しても機能しないため、あなたにとってより重要なものと、より少ないものを選択する必要があります。
