Visual Studio 2013を使用したクラウド内のWebアプリケーションのリモートデバッグ





Windows Azure SDK 2.2ツールのリリースにより、クラウドアプリケーションおよびWindows Azureサービスの開発者は、クラウドから直接リモートでコードをデバッグできる優れたVisual Studio 2013機能拡張を受け取りました。 リモートアプリケーションのデバッグは、クラウドサービスの役割とWebサイト（Windows Azure Webサイト）の両方で利用できます。



開発者が新しいリモートデバッグ機能を使用してWindows Azureアプリケーションを開発する方法を見てみましょう。 まず、Windows Azure SDK for .NET 2.2がインストールされたVisual Studio 2013 RTMが必要です。 Visual Studio 2013は既にWindows Azureサブスクリプションにバインドされている必要があります。



同じ名前のWindows Azure Webサイトに発行するRemoteDebugSampleプロジェクトがあるとします。 Windows Azureでプロジェクトを公開する際のリモートデバッグでは、デバッグでプロジェクト構成を設定する必要がある場合があります（デフォルトではリリースが推奨されます）。







Visual Studio 2013のコードの任意の部分にブレークポイントを設定します。







[Webサイト]タブのサーバーエクスプローラーに移動し、Webサイトを選択します。 コンテキストメニューを右クリックし、[デバッガーのアタッチ]を選択します。







数秒で、リモートWebサイトの設定を更新し、ブラウザーで新しいアプリケーションセッションを開始した後、クラウドでローカルに実行されているかのようにアプリケーションのデバッグを開始できます。







それだけです！ 同様に、クラウドサービスのWebロールとWorkerロールのデバッグが実行されます。 クラウドサービスに公開するときは、必ず「すべてのロールのリモートデバッガーを有効にする」オプションを設定してください。







開発をお楽しみください！
