Roskomnadzorは、ほぼ1500万のIPアドレスをAmazonでブロックします

IT専門家のミハイルクリマレフは 、Roskomnadzorが2つのAmazon Cloud Web Services（AWS）サブネットへのアクセスをブロックする実験に着手していることを示す文書を公開しています 。



文書自体はネタバレの下












ロックは、トラック運転手の間で非常に人気のあるZelloサービス（プッシュツートークラジオを模倣）に向けられています。 2017年3月に、プラトンシステムに対する抗議がZelloを通じて調整され、その年の4月にすでに、Roskomnadzorはサービスで使用されるIPアドレスへのアクセスの制限を開始しました 。 ブロックの公式な理由は、Zelloがユーザーの通信および個人データを保存することを拒否し、さらにこの情報を法執行機関に提供することです。



それ以来、ILVは毎日何十ものZelloアドレスをブロックしています。 サービスは、自動的に新しいアドレスに「移動」します。 Roskomnadzorがこの戦いに負けていることが明らかになりました。 Amazonには数百万のIPアドレスがあるという事実にもかかわらず（IPv6アドレスを考慮しない場合）、 4,000を超えるアドレスがブロックされます。 この猫とマウスのゲームは何年も続きます。 さて、Roskomnadzorは「正方形を打つ」ことに決めました。 部門の専門家は、Zelloサービスの「効果的な劣化」につながる2つのサブネットを特定し、アクセスの制限に関してプロバイダーに推奨事項を発行しました。



少なくとも2人の通信事業者がこれらの手紙の受領をすでに確認しています。



36個のサブネット（そのうち2/11個のサブネット）をブロックし、合計で約1500万のIPアドレスをブロックする予定です。



これらはすべて、Amazonクラウドも使用する他の多くのサービスに影響を与える可能性があります。 たとえば、Bitrix24 CRMシステムの運用が中断され、今年2月にインフラストラクチャが移管された可能性があります。



更新：Roskomnadzorプレスサービスからの応答：







更新2： AmazonはZelloにIPアドレスの終了を要求しました 。 サービスは別のプラットフォームに移動し、ILVは何千ものZello関連アドレスをレジストリから削除しました。
