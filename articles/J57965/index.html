Mac OS X 10.5でpdo_mysqlをPHPに配置しました

Mac OS Leopard上のApacheとPHPはデフォルトでインストールされます。 私なしでphpを含めるための十分な指示があります。



ただし、Appleユーザーは、何らかの理由で、PDO経由でmysqlを操作するためのドライバーを標準パッケージに含めなかったという問題が残っています。このため、PDO経由でmysqlに接続しようとすると、「 ドライバーが見つかりません 」というエラーが発生します。



私が見たハウツーのほとんどは、このモジュールを手動で構築することになります。

しかし、Mac OSはあなたにとってGentooではありません。 アセンブリの方法にはさまざまな困難が発生する可能性があります。たとえば、モジュールの不在と非互換性は、手動で解決する必要があるため、このオプションを最後に検討しました。







そして当然です。 結局のところ、 すべてがすでに盗まれており 、 すべてを簡単に解決できます。 Mysql、Apache、およびPHPを含む優れたMAMPパッケージがあります。 XAMPPまたはDenwerのそのような類似物。 MAMPには2つのバージョンがあり、そのうちの1つは無料です。



以降のすべての手順には、管理者権限が必要です。



パッケージを入れます。 すべてのpdo_mysqlをコピーします*フォルダーのファイル

/アプリケーション/ MAMP / bin / php5 / lib / php / extensions / no-debug-non-zts-20050922
フォルダーへ

/ usr / lib / php / extensions / no-debug-non-zts-20060613 /




最後のフォルダーの名前の番号は異なる場合があります。 PHP4の場合、すべて同じ方法で実行する必要があります。

必要に応じて、pdo_pgsql。* PostrgeSQLにもそれぞれコピーされます。



その後、phpモジュール自体を次の場所からコピーして置き換えます。

/アプリケーション/ MAMP / bin / php5 / bin / php
で

/ usr / bin / php
