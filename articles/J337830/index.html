ServiceNowインシデントソリューションがSlackに登場

SlackはFrontiers ConferenceでServiceNowとの統合を発表しました。 Slackによると、このタイプの相互作用はユーザーの間で最も予想されていました。 そのうちのいくつかは、オープンAPIを介してServiceNowをSlackに統合しました。 今では、メッセンジャーでインシデントに関する情報を直接「公式に」受け取る機会が現れています。





/ Flickr / Giorgio Minguzzi / CC



ServiceNowは、ITサービス管理を自動化するためのガートナーの最高のシステムです。 たとえば、2014年のオリンピック開催の数年前、ServiceNowプラットフォームはITSMプロセスを管理するための主要なツールでした。 ユーザーは電話と電子メールで問題を報告し、その後、アプリケーションは指示によって配布され、専門家によって処理されました。



そして今、ServiceNowのインシデント処理がメッセンジャーにやって来ました。 Slackを使用する企業は、チャットで直接追跡できるようになります。 問題が解決すると、通知がSlackに送信されます。



したがって、ServiceNowブログが書いているように、従業員は常にイベントに気づき、気が散ることが少なくなり、重要なインシデントについて議論しやすくなります。 Slackに組み込まれているServiceNow通知がどのように見えるかを確認できます 。



「ServiceNowからSlackへのインシデントの出現により、議論と解決のプロセスが迅速になり、チームメンバーは追加の視覚化とコンテキストを使用してより簡単に作業できるようになります」 と Slackの4月Underwood副社長は述べました 。


より快適な作業のためのその他のサービスとの統合は、ServiceNowポリシーの重要な部分です。 InGeniusとTownsend Securityが統合を発表したのは9月のみです。 また、Microsoft Teamsと「マージ」する予定です。 一般的に、執筆時点では、 ServiceNow Storeには 183個の統合があります 。



しかし、リストは公式ストアで終わりません。 既に述べたように、オープンAPIのおかげで、公式にはサポートされていないサービスを扱うことができます。 ZapierやIFTTTなど、「リンク」サービスの作業を自動化するための特別なソリューションもあります。 これらは、ZenKitマネージャーのEric Ebert が 、たとえばServiceNowをJIRAに統合するために使用することを勧めているものです。



開発者によると、ServiceNowをSlackチャットに埋め込むことを可能にするアップデートは、「今年の後半に」リリースされる予定です。 将来、統合がさらに開発され、機能が拡張されます。 そのため、SlackチャットからServiceNowを直接操作する機能を追加する予定です。



「Slackとのパートナーシップにより、ServiceNowユーザーはサービスを改善できます。リアルタイムで議論することにより、意思決定が迅速に行われます」 と ServiceNowのテクニカルディレクター、アランレインワンドは述べています。
