短所、利点、Operaで見逃していること

以前は、Operaは非常に軽量なブラウザであり、これらの人々の多くが気に入っていました。 今では、独自の電子メールクライアント、RSSリーダー、急流ロッキングチェア、その他多数の機能を備えた完全な組み合わせになっています。 すべてがすでにそこにあるので、幸福のために他に何が必要であるように思われるでしょうか？ しかし、私はそうは思いません。 私はこれのすべての欠点を説明しようとします、それはそうかもしれませんが、私はそれに欠けている素晴らしいブラウザであり、他のブラウザオプションと比較します。 もちろん、誰にも自分のアイデアを押し付けたくはありませんが、誰かに面白いと思われることを願っています。





短所



最初に、他のブラウザと比較するのではなく、いくつかの一般的な欠点を指摘したいと思います。



1.オーバーロードされた関数



Web ブラウザの主な機能に加えて、Opera ブラウザには次の機能が搭載されています。

  • Opera Mailメール/ニュースクライアント
  • アドレス帳
  • BitTorrentピアツーピアネットワーククライアント*
  • RSSアグリゲーター*
  • IRCクライアント
  • ウィジェットは、HTMLテクノロジに基づいたグラフィックモジュールであり、ブラウザウィンドウの外側で動作します。


Opera Link、ダウンロードマネージャー、ブックマークなどの機能により、ブラウザーの主な仕事をより楽しく、より速くすることができます。そのため、もちろんそれらの存在に異議を唱えることはできません。 ただし、これら6つの機能をキャンセルすると、Operaの起動速度が低下することがあり、パケットサイズは4〜5メガバイト減少します。 さらに、メールのWEBインターフェイスを備えた開いたタブは、メールクライアントを完全に置き換えることができます。 アドレス帳は、標準セットのほとんどすべてのOSにあります。 IRCクライアントは通常、ここで何が行われているのかを理解していません（個人的には）。 現在、ウィジェットが必要な理由が不明な形式のウィジェット。

BitTorrentクライアントとアグリゲーターにアスタリスクを付けました。これは、Operaユーザーの40〜50％がこれらの機能を使用していることを完全に認めているためです。

最良の場合、これらの6つの機能がボタンを押すだけでインストールされ、拡張機能と同じ方法で削除されれば、誰もが大丈夫でした。 そして最も重要なことは、電子メールクライアントがブラウザの拡張機能であるため、論理的であることです。

Fire Foxがユーザーが作成したプラグインのために拡張できなかったとしたら（ トートロジーでごめんなさい ）、それほど人気はなかったでしょう。 Operaはこれを理解し、ウィジェットを作成しましたが、FireFox拡張機能の機能と比較することはできません。 FireFox、Operaは無料の製品であるため、広告でかなりの収益を得ている場合でも、コミュニティのサポートは非​​常に重要です。 誰もが拡張機能を書くことでMozillのブラウザを改善できますが、Operaを支援するにはどうすればよいですか？ 実際の開発者にのみアイデアを提供します。これにより、これらの開発者に依存するようになりますが、それ自体は悪いことです。

メリット



以下では、「なぜOperaなのか」という質問に対する重要な議論であるOperaのいくつかの機能について説明しました。 私はこれらの可能性がすべてOperaでのみ利用可能であるとは決して主張しませんが、このブラウザではすべてがすぐに利用可能であり、使いやすく簡単です。



1.ブックマークの同期など

おそらく、この機能は、私が選択したブラウザーで最も重要です。 2台のコンピューターがあり、それぞれに2つのオペレーティングシステムがあるため、この機能は非常に重要です。 彼女のおかげで、他のブラウザからOperaに切り替えることをすでに考えているように思えます。 また、コンピューター上の携帯電話でブラウザーのブックマークを編集できるという事実は非常に快適です。つまり、興味深い記事を検索するときに、このページを携帯電話のブックマークに追加して、バスや地下鉄に乗っている間にそれを読むことができます。 この機能を提供してくれたOperaに感謝します！



2.高速

タブ間の迅速な移行、すべての設定の迅速な適用、ページの迅速なロードなどの小さな瞬間。 ユーザーを喜ばせてください。 ここでは、余分な言葉がなくても、すべてが明確です。



3.エクスプレスパネル（スピードダイヤル）

Operaはこの美しさを追加した最初のブラウザです。 同様の機能が拡張機能として利用可能なFirefoxとは異なり、このパネルは非常に高品質に実装されています。 現在、Safariを除くほとんどすべての最も人気のあるブラウザーは、サイト間のクイックナビゲーションのこの部分を使用しています。



4.作業Ctrl + Z

たぶん些細なことですが、私はそれが本当に好きで、私は頻繁にそれを使用し、これが他のブラウザでなぜ機能しないのか理解できません。



5.サイトコンテンツのブロック

Mozillaのブラウザでは、この機能はAdBlock +拡張機能を使用して実装されています。 ここにインストールするものは何もありません、そして、彼らが言うように、すべてがすぐに動作します。 はい、使い方は簡単です！ この機能を無視しないことをお勧めします。

1.「スマート」アドレス入力行

このフレーズとは、次のことを意味します：テキストの冒頭でユーザーがhttp、ftp、またはアドレスを英語以外で入力しない場合、Googleでこのテキストを検索して、必要な情報がGoogleが提供するサイトと一致する場合、自動的にこのサイトに移動します。 これはFireFoxに実装されており、非常に優れています。 オペラでは、テキストの前に文字「g」を書く必要がありますが、ロシア語でテキストを書く場合、レイアウトを切り替えて、文字gを追加し、レイアウトを再度切り替えてから、書きます。



2.質問なしのブックマーク

少し面倒なのは、「ブックマークの作成」アクションをクリックした後に表示されるダイアログです。 これは、私が本当に好きだったページのコンテンツから私をそらします。 さらに、これを次のように実装できます。ユーザーが既にブラウザのブックマークにあるページのブックマークを作成する場合、このブックマークを一番上に移動するだけです。 したがって、ユーザーは期待するものを取得し、不要な質問はありません。 また、ブックマークを配布する必要がある場合は、マネージャーごとに一度にすべてを実行します。これは、ブックマークごとに個別に行うよりもすべて高速です。



3.組み込みのTwitterクライアント

私は本当にこれを見逃している、あるいは、新しいツイートについてX分ごとに見たいという通知を本当に逃している。 実際、ここに書くことはもうありませんし、何もありません...それについて思い出してページを更新するとき、Twitterタブに行きたくありません。



4.ブラウザから個別にダウンロードする

私はこの機能を次のように見たいと思います：ブラウザが起動したら、今のままにしておくことをお勧めしますが、ダウンロードが続行し、ユーザーが「終了」をクリックした場合、ダウンロードの停止について警告せず、FireFoxのように別のダイアログでダウンロードします。 特に、組み込みのBitTorrentクライアントがある場合、大きなファイルをダウンロードできる場合。



5.簡素化されたインターフェース

トップパネルを多くのメニューと一緒に使用することはありませんが、エンコードを選択するか設定に移動する必要がある場合があります。 しかし、彼女はいつもタブのタブの名前から私をそらします。 上に書いたように、良いインターフェースの良い例：Google Chrome。 「気を散らす」や「干渉する」といったことは間接的な感覚であり、微妙でさえあることを理解していますが、これは悪いインターフェイスを使用することにすでに慣れているからです。 そして、プログラムを使用する一般的な印象は小さなレンガで構成されています。



6.すべてのOSダイアログ保存ファイルに共通

Gnome Operaではシステムダイアログを使用し、KDE3 / KDE4では独自のものですが、非常にシンプルで不便であり、Windowsでは正直に覚えていませんが、それも私のものです。 Gnomeだけでなく、すべてのシステムで同じですが、非常に便利なインターフェース（ブックマーク付き）、またはシステムダイアログが表示されます。
