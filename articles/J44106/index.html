Asus EeePC 904HD上のArch Linux。 パート1、インストール。

まず、この記事が登場した理由と対象者について説明します。

それはすべてUbuntu 8.10のリリースから始まりました。 私は彼を長い間待っていました。 そして彼は出てきました。 8.04からのアップグレード、そして...私の失望は際限がありませんでした。 私のEeePC 904のひどいブレーキは彼にとどまる機会を与えませんでした。 それは残念です、私はいつもこの製品が大好きでした。 そして、8.04にとどまることはすでにとても退屈でした...

さらに、私は長い間「管理された」配布を望んでいました。 だから、私の考えはArch Linuxに落ちました。



そして、ここで時間を犠牲にして後で楽しむ準備ができていて、「コンソール」が何であるかを知り、ubuntu_userよりも高いレベルで「コンソール」と通信する人々がいる場合、この記事はあなたのためです。

まあ、実験者（私のような:)）も満足します。







1.最初に、配布キットとメディアを準備します。



2つのインストールオプションがあります。シンプル（.imgファイルを使用）またはより複雑（.isoイメージを使用）です。

最初のオプションを検討してください。これははるかに高速で、プリインストールされたLinuxシステムを必要としないためです。

始めるには、たとえば、offsiteからarchlinux-2008.06-core-i686.imgの最新のディストリビューションをダウンロードします（ダウンロードおよびトレントリンク用のミラーがあります ） 。

コアバージョンをダウンロードする必要があります。ネットワークカードモジュールがないため、ftpバージョンは使用できません。



distribの重量は約300 MBなので、少なくとも512 MBのフラッシュドライブが必要です。

Windowsからシステムイメージをインストールするのは、 Flashnulユーティリティを使用すると最も便利です。

まず、USBフラッシュドライブが機能しているインデックスの下でflashnul -pコマンドを調べます。 例：



C:\>flashnul -p



Avaible [sic] physical drives:

0 size = 200048565760 (186 Gb)

1 size = 400088457216 (372 Gb)

2 size = 400088457216 (372 Gb)

3 size = 4060086272 (3872 Mb)







チームは、フラッシュドライブのインデックスが3であることを示しており、このインデックスを引き続き使用します。

コマンドC:\>flashnul 3 -L path/to/arch/usb.img



、イメージをUSBフラッシュドライブに「塗りつぶし」 C:\>flashnul 3 -L path/to/arch/usb.img







Linuxユーザーの場合、これはすべて、dd if = image.img of = / dev / sd [x]コマンドで行います。

/ dev / sd [x]はフラッシュドライブへのリンクです（注意！たとえば、/ dev / sdb1などのフラッシュドライブがある場合は、/ dev / sdb（インデックスなし）を記述する必要があります）。



ArchWikiで配布キットを使用したフラッシュドライブの準備の詳細をお読みください。



2.配布キットをインストールします。



これで、画像の準備が整いました。 降りる。

BIOSでフラッシュドライブからブートをセットアップします（ F2->Boot->Hard Disk Drives->1st Drive



フラッシュドライブを設定します）。 設定を保存して再起動します。

ロード時に、GRUBメニューがポップアップ表示されるようになりました。 最初のアイテム-Boot ArchLiveを起動します 。

ログインプロンプトで、「root」と入力します。 パスワードは要求されません。 次に、書かれているとおりに行います（ To begin installation, run /arch/setup



。つまり、コンソールで/ arch / setupと入力します ）。

ようこそウィンドウが表示されたら、[OK]をクリックします。 インストールする場所を尋ねる-最初のアイテム（CD-ROMまたはその他のソース）を選択します。 そして、インストールはポイントで始まりました：



1.ハードドライブを準備します。

自動的な方法（ Auto-Prepare



）でディスクを破壊します。 / dev / sdaを選択します。 次に、コンフィギュレーターは、/ boot、swap、/（ルートディレクトリ）、/ homeに割り当てるスペースを尋ねます。

このように壊しました-/ boot（64 MB）、swap（512 MB）、/（7500 MB）、/ home（すべての残りのスペース）。

FSとしてext3を選択します。 /および/家庭用に使用します。 データの完全な削除に同意します。



2.パッケージを選択します。

インストールするパッケージを選択します。 情報ウィンドウの後、コンフィギュレーターはパッケージをキャッシュに保存するように要求します。 もう重要ではないと思うので、「いいえ」と答えてください。 「base-devel」を選択してインストールします。 次に、coreおよびbase-develからのパッケージがパッケージのリストでマークされ、残りはあなたの選択です。 すべてを後で置くことができるので、私はさらに何も選択しませんでした。



3.パッケージをインストールします。

実際にパッケージのインストールが開始されます。 特別なことは何もありません。



4.システムを構成します。

初期設定。 彼は、hwdetectを使用して構成ファイル内の機器を調整するように求めます。「いいえ」をクリックします。 次に、必要に応じてファイルを編集します。 編集する内容を正確に説明しません。 ArchWikiで詳細を読んでください 。 ルートパスワードを忘れずに設定してください。



5.ブートローダーをインストールします。

ブートローダーのインストール。 誰にとっても便利です。 私にとって、GRUBとGRUBのみ;）

/dev/sda



インストールします。 次に、「いいえ」と答えます。



最後のポイントはExit Install



です。 コンソールで「 リブート 」と入力します。



それだけです！ パスの始まりが置かれました！
