これに至った経緯：現代のスマートフォンとその展望

スマートフォンとそのOSの歴史についての私の見解を共有したいと思います。 それについて書くアイデアは、Windows Phone 7/8の誇大広告、多くの相反するレビュー、レビューなどに触発されました。この記事は、この15年間でスマートフォンがどのように発展したか覚えていないため、まず若い人にとって興味深いものになります彼らが現れる瞬間から。



現在、Android、iOS、および死にかけているSymbianの3つのスマートフォンがスマートフォン市場を支配しています。 別のOSであるWindows Phoneの「大リーグ」に到達する可能性はすべてあります。 現在、マルチタッチタッチスクリーンの存在が一般的です。ネットワーク上では、AndroidとiOS（かつてはWindowsとLinux）のようなholivariタイプです。 「マイクロソフトが遅れた、すべてが失われた」などのトピックに関する多くのスラチャなど。この機会に、私は自分の見解を共有したいと思います。 しかし、最初に、歴史への遠足...



スマートフォンがスマートフォンと呼ばれ、なんらかのプログラムを置くことができ、Javaゲームがクールなトップであると考えられていた当時でさえ、写真の品質が今ではとても高いにもかかわらず、カメラは非常に高価なモデルにしかなかった当時を覚えています彼らはそれが可能だとさえ信じませんでした。



スマートフォンの動きの先駆者はノキアでした。 彼らは、1つのNokia 9000ケースで最初の実際のPDA + 電話の 1つを作成しました



ここでは、1996年にそのようなデバイスが技術思想の頂点でした。 適応OSを使用しました。 その後の製品では、有名で現在死にかけている（誰かが私に反対するかもしれませんが、それについては後で詳しく説明します）Symbian（2001）がすでに登場しています。 これは、最初の本格的で普及したモバイルOSでした。 この時点で、他の企業がモバイルOSとそれらに基づいた製品を作成しようとする試みが既にいくつかありますが、すべてが成功しているわけではありません。



その後、2003年にWindows Mobileが登場しました（速度が遅くバグがあり、修正されませんでした）。 Windows CEの以前のバージョンは広く使用されていません。



そして、これがSymbianの最初のバージョンです。





ちなみに、私の友人がそのような電話を持っていたら。 その遠い時間では、これは非現実的なものでした。

時間が経つにつれて、私自身がSiemens SX1を買収し、信じられないほど満足しています（多くの場合、側面のボタンは非常に愚かだと考えられていました）。 残念ながら、この携帯電話はあまり信頼できませんでした。





Windows Mobile 6.当時、実際には、唯一の一般的なタッチOS。 タッチデバイスの問題は抵抗スクリーンでした。 一定の圧力がかかっているため、内部は時間とともにスタイラスで「押し出され」ました。 だから私の最初のコミュニケーターE-TEN Glofiish M700は死にました：



私の古いシーマンとは異なり、彼はひどく遅くなりました。 それでも、両方のデバイスは非常にクールでした。 これが、SymbianとWindows Mobileがゆっくりと並んで歩いた方法です。 「アライグマ」の後、私は別の素晴らしいデバイスを試すことができました-Nokia E51：



そして、私はあなたに、この電話が私に最も合った電話をいつも使っていると言います。 その上にタッチスクリーンはありませんでしたが、快適で軽量で、素敵な金属製のボディで、彼はスピーカーを変更しなければなりませんでしたが、この電話を手に入れて申し訳ありませんでした。 これは、ノキアが本当にクールなことをした時でした。 RIM Blackberryスマートフォンはまだどこか遠くに存在していましたが、それを知っている人はほとんどいませんでした。これらはビジネスマン向けのデバイスであると一般に信じられていたため、この波は過ぎ去りました。 正直に言うと、このようなデバイスが私の人生で生きているのを見たことがありません。 そして、今まさにGoogle / Android Inc.の腸内で Androidは集中的に開発されていましたが、当時はタッチセンシティブであることさえ意図されていませんでした。



今回の特徴は、実際、携帯電話の世界では根本的に新しいものが登場しなかったことです。 すべては古い、大規模なスマートフォンの開発でしたが、非常に緩やかになりました。 そして、iPhoneのプレゼンテーションは大爆発しました。 電話はすぐにおもちゃと命名されましたが、2つの主な特徴的な機能がありました：すべてが美しくエレガントでした-外部デザインから美しいスクロールジェスチャーやアイコンまで。 そして、それはブレーキとグリッチなしで働きました。



彼はマルチタスクを持っていませんでしたが、多くの機能がありませんでしたが、このデバイスを使用できるシンプルさと便利さを世界はすぐに評価しました。 そして...そして、App Storeが登場しました。 初めて、電話へのアプリケーションのインストールが非常に簡単になりました（ほぼ同時にWindows Mobile Marketplaceが登場しましたが、進歩がなかったため、それを知っている人はほとんどいませんでした。



その瞬間、重要なことが起こりました。 世界が新しい世代のデバイスへの移行を開始した瞬間が来ました。 容量性の面ではなく（容量性スクリーン+マルチタッチを除く）、機能の面ではなく、利便性と操作のしやすさの面で新しいものでした。 実際、視覚的にも、iPhoneインターフェースは革命ではありませんでした。





美しく塗装されていましたが、概念的には違いはありませんでした。 アイコンの同じグリッド、同じメニュー。 目新しさは、タッチコントロール、ジェスチャーでした。 世界はすぐに、はるかに便利で信頼性が高く、最終的には安価なボタンであることに気付きました。



この時点で、実際には、スマートフォン/ OSの他のメーカーは、何かを変更する必要がある状況にありました。 Googleでは、Androidがまだリリースされていないため、同様のインターフェイスがなければ、このロボットは表示されずに死ぬことにすぐに気付き、インターフェイスの書き直しを始めました。 注目すべきことは、原則として、OS全体を書き換える必要はなく、そのほんの一部を書き換えるだけでした。 NokiaはSymbianの新しいタッチバージョンをリリースしましたが、これは通常のものとほとんど変わりませんでした...これが主な間違いでした。 プッシュボタン式スマートフォンの時代は終わりを告げ、アップルはジェスチャー制御の新しい時代を切り開きましたが、当時ノキアはそれを認識していなかったようです。 RIMはその小さなニッチでlo然としたままであり、Microsoftは、明らかに、Windows Mobileが死ぬ時であることにすぐに気付いたようです。 残念ながら、新しいトレンドに適応できるSymbianとは異なり、Windows Mobileには完全に新しいインターフェイスが必要でした。これは、古いアプリケーションとの完全な非互換性も意味していました。 そして、これは仕事のスピードは言うまでもありません。 Microsoftの問題は、代替品がなかったことです。



Windows Phoneの現在の状況は、Microsoftがすぐに新しいOSの作業を開始したことを示唆しています。 Windows Phone 7がWindows CEのコア上に構築されており、現在NTへの移行が8番目のバージョンで行われているという事実は、Microsoftが何か新しいものをリリースする必要があったことを示しています。 そして今、私たちは現代市場を形作った重要な瞬間に来ました。 当時、実際には、市場には4人のプレイヤーがいました。

-Apple iOS / iPhone

-Nokia Symbian

-Microsoft Windows Mobile

-リムブラックベリー

そして、Androidの世界を爆破する準備をします。

Appleはしっかりとその地位を確立し、業界に新しい標準を設定しました。

当時のシンビアンは、変化の風が吹いている場所をノキアが認識し、正しい方向に動き始めた場合、市場シェアと見通しを維持するあらゆる機会がありました...残念ながら、そうではありませんでした。 この決定はノキアの主な間違いであり、その瞬間からSymbian時代は衰退し始めました。 彼女は慣性のおかげでのみ浮かんでいました。

RIMは十分に速く反応しませんでした...そして市場を失いました。

マイクロソフトはまだ空いています。 Windows Mobileは劇的に倒産し、Windows Phoneは2010年末にしか登場できず、時間を失いました。

Googleは即座に反応しました...そして、半年後にAndroidの最初のバージョンがリリースされました。 はい、それはひどくバグがあり、機能が不十分でしたが、すべての主要市場のプレーヤーが見逃していたものがありました。 彼女はAppleが設定した例に従って...勝ちました。 iPhoneはHi-Endスマートフォンの事実上の標準になりましたが、Androidは予算デバイスに至るまで、他のすべてのニッチを徐々に占有しました。 Windows Phoneがそれほど成功しなかったのは、それが十分ではないからではなく、市場がすでに完全に占有されているからです。 市場を支配的なiOSとAndroidに分割し、他のすべてのシステムを消滅させるには、2年で十分でした。 別の同様のOSを作成しても意味がありません。 したがって、MeeGoおよび他のプロジェクトは失敗しました。それらが十分に開発されていなかったためではなく、小さなコミュニティだけが残っていました。 それらに切り替える価値のあるものは何もありませんでしたが、iOSとAndroidには何十万ものアプリケーションがあり、多くの場合は同じものでした。

Androidは一方ではiOSクローンの一種になり、他方では反対になりました。 これら2つのシステム間の対立の本質は、自由と信頼性です。



何が先に私たちを待っていますか？ Windows Phoneだけが、業界の巨人ではないにしても、少なくとも主要なプレーヤーであるとしても、3番目になる可能性が本当にあるように思えます。 なんで？ マイクロソフトの功績として、彼らは他の人が提供できないものを市場に提供することができました-新しい、本当に新しく、新鮮なインターフェース、そしてマイクロソフトが市場の約90％を占めるシステムのデスクトップバージョンとの最大限の統合...促進する。 AndroidとiOSに取って代わることはありませんが、これらの2つの要因のおかげで、生き残り、太陽の下でその位置を占めることができます。





特に、新しいMetro-Modernインターフェイスに言及したいと思います。 Symbianの登場以来、これはインターフェースの視覚化における最初の真の革新です。 そして、それは私たちに一度に2つの新しいことをもたらします：完全に新しい外観と新しい機能-アプリケーションのショートカットをウィジェットと組み合わせます。 快適で美しい-あなたが使いたいもの。 電話がデスクトップ向けの機能に向かうという傾向は、この点でマイクロソフトを助けるだけです。



Windows PhoneがiOSの大きなシェアを獲得することはまずありません-Appleファンのようにファンがいますが、Androidのシェアに打撃を与えるでしょう-これらのOS（WinとAndroid）上のデバイスの価格カテゴリは非常に広い範囲で交差しています。 どうしてですか？ Androidには独自の問題があり、多くの人がすでに悩んでいます（私を含む-私はこのOSに基づいて携帯電話とタブレットの両方のユーザーとして話します）-OS自体に反する自由、市場には多くの迷惑があります（Googleでもこれを実現し、 Google PlayでのAndroidマーケットのブランド変更による変更はほとんどありませんでしたが、DalvikVMについてはすでに沈黙しています。 私が開発を始めるまで、それがどれほど遅いかは分からなかった。 さらに、インターフェイスブレーキなど、ユーザーとして私を困らせるナンセンス。



モバイルOSの最終市場が形成されている歴史的な瞬間を目の当たりにしているように思えます。90年代にPCから形成されたもので、Windows、Linux、MacOS Xが基本的に残り、他のすべての人が死ななかった時代です。その後、裏庭に滞在しました。 同じプレーヤーが市場を共有するようになりますが、Linuxの代わりにAndroidモバイルクローンがあります。 市場がどのようにシェアするかを予測することは困難ですが、市場に他の人たちのための場所が残っていないという事実は事実です。 そして、これが理由です。

プロジェクトとして残っているプロジェクトの数はリストしません。 コメントでは、さらに多くのクールなOSを挙げますが、それに直面しましょう-現時点では、それらは市場に影響を与えません。 大手企業のどれも、他の何かを宣伝することに興味はありません。 そして、ユーザーは本当に未知のものを試したくありません。 Windows Phoneが最後の空き場所になりました。このシステムがクラッシュしない（そしてクラッシュしないと確信しています）場合、Windowsストア内のアプリケーションの数の増加によって証明されるように、他の誰も押し込めません。 ちなみに、これらの非常にアプリケーションの開発者にとって、システムが2つの巨大な他の市場（iOS、Android）の存在にもかかわらず引き付けられるという事実は注目に値します。 したがって、開発者はそれを信じています。 つまり、ユーザーは購入する価値のあるものを手に入れることができます。



その中核である成熟した、信頼性の高いiOS、新時代のWindows Phoneは、形成期間と特定のバグを乗り切る可能性がありますが、市場での足場を獲得し、すべての側面から無料のオープンソースのAndroidは、常に買い手を見つけて死ぬことはありませんフリーソフトウェアの愛好者のおかげです（もちろん、Googleがシステムの制御を厳しくする道を取りませんが、これは現在の状況ではかなり可能です）-モバイルOS市場の3つのエコシステムを統合する歴史的な瞬間を見ているようです。 残りはすべてそれらに切り替わるか、失われます。BlackBerryだけが小さなニッチを維持できますが、以前の栄光と偉大さを取り戻すことはありません...
