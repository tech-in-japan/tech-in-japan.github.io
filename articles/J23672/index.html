ドメインでキャッチオールを使用する際にスパムを排除する

ソース ： キャッチオールドメインスパムを回避する方法@ Lifehacker.com



一般的に言えば、これは翻訳になり、対応するセクションになることでした。 しかし、記事を少し書き直し、コメントのいくつかと私の考えをいくつか追加する方が便利でした。 したがって、「私は賢い」と書いています。 ただし、すべての感謝はGina Trapaniと未知のRay User GApps For Domainに送信する必要があります（はい、これは彼のネイティブアメリカンの名前です:)。




入門



そのため、すでに独自のドメインがあり、そのドメインにGoogle Apps （以降-GApps）をねじ込んだとします。 少なくともメール。 そして、Alexander-dog-domainと言うだけでなく、この名前の異なるスペルを心配しないために、サイトでの登録の利便性のために、catch-allをどのように使用しますか。 さらに、あなたは誰があなたのアドレスをスパマーに渡したかにも興味があります。



注 ：このシステムは、フィルターをサポートするクライアントとの連携に非常に簡単に適応できます。 必要なのは、ドメイン、 キャッチオール（wiki / en） 、およびフィルターをサポートするクライアント（Thunderbirdなど）だけです。


一般的に、サイトごとにメールアドレスを分ける方法はいくつかあります。 それぞれに独自のアドレスまたはメーリングリストを作成できます（メーリングリストの再構成が簡単で、数に制限がないため、2番目の方が優れています）。 マイナスは1つだけです。毎回、手でそれをしなければなりません。 または、キャッチオールを有効にして、サイトの登録フォーム（通常、amazon.com @ your-domain.tldのようなもの）に何かを入力するだけです。 少ない-キャッチオールは大量のスパムへの扉を開きます。 「+」表記も使用できます。 そして、すべての登録フォームが正しく行われ、プラス記号を受け入れた場合、彼女は素晴らしいでしょう。



方法があるように見えますが、いくつかは不快です。 そして、Ray GAppsユーザーが自分のシステムで登場しました...一番下の行は「秘密」の接尾辞です。



戦いに！



GAppsコントロールパネル（通常はwww.google.com/a your_domain ）に移動します。 新しいユーザーを作成します。 たとえば、スパムトラップ。 次に、コントロールパネルに戻り、メールオプション（https://www.google.com/a/cpanel/ your_domain / EmailSettings）を開きます。 そして、キャッチオール設定で、新しく作成されたユーザーのアドレスを入力します（spam-trap @ your_domain ）：





次に、このユーザーのメール（スパムトラップ）を入力します。 [設定]-> [フィルター]-> [新しいフィルターを作成]に移動します。 「To」フィールドに、ピリオドと文字および/または数字のシーケンスを入力します（たとえば、「 。 ThingySuffix42」 。個人的にはピリオドで区切られた2つの単純な単語の接尾辞を使用します）。 選択することは、覚えやすいシンプルなものです。 これは、このシステムの操作の鍵となる「秘密」の接尾辞になります。





[次のアクション]をクリックして、[アクションの選択]ページに移動します。 「手紙を転送」を選択し、メールを一元的に読みたいアドレスを入力します。 これらのメッセージにタグを付けてすぐにアーカイブすることもできます。これにより、後でエラーをキャッチしやすくなります。このフィルターに該当せず、GMailがスパムと見なさないすべてのメッセージが受信トレイに格納されます。





これで、メインのアドレスを指定したくない場合は、 必ずthat_suitable.secret_suffix@your_domainの形式のアドレスを指定してください 。 おそらく最良の選択は、 amazon.com.thingySuffix42 @ your_domainのようなアドレスを与えることでしょう。 したがって、誰があなたのアドレスをスパマーに渡したかを常に理解し、1つの簡単なフィルターでこのスパムを取り除くことができます（このフィルターはメインアカウントで作成できます）。



システム全体の唯一のマイナス点は、時々このアカウント（スパムトラップ）にログインし、メールのフィルターを作成する必要があることです。これは誤って転送に該当しません。 たとえば、メーリングリストは多くの場合、非標準のToフィールドを指定し、このフィルターによってスパムとして検出されます。



catch-allをマイナーエラーのあるアドレスのキャッチにも使用する場合、さまざまなスペルオプション、最も一般的なエラー、およびこれらのメッセージを転送するフィルターの作成を検討する価値があります。



このシステムが機能するための主な前提は、スパマーがアドレスを解析しないことです。 そしてそれが機能している間。 もしあなたが非常に注意深い妄想をしているなら、あなたが一貫して知っている、そして理解している、そしてまたあなたが知っている、そして理解しているある原則に従って、いくつかの原則に従って秘密の接尾辞の束を作成することを誰も気にしません:)
