Snow Leopard 10.6.5はexFATのサポートを開始しました

Snow Leopardのアップデートの1つは、あまり言及されていない革新をもたらしました-システムはexFATファイルシステムをサポートし始めました。

これは実際の応用において何をもたらすのでしょうか？



原則として、MacコンピューターとWindowsコンピューターの間で情報を交換するには、ディスクまたはフラッシュドライブをFAT32（ファイルサイズの制限が4ギガバイト）で、またはNTFSで分割する必要があります。 また、ほとんど快適ではありません-定期的に、サードパーティ製のドライバとシステムに統合されたドライバの両方が不快な不具合を起こします。



exFAT形式の革新は、これらの問題を簡単に解決する良い機会を提供します-exFATサポートは、Windows XP Service Pack 2および3、Windows Vista Service Pack 1、Windows Server 2008、Windows 7で利用可能です。



機能：
  • ファイルサイズ2 64バイト（16 exbibytes）の理論上の制限。
  • 最大クラスターサイズが2 25バイト（32メビバイト）に増加
  • ディスクの断片化を減らすことができる空き領域のビットマップを導入することにより、空き領域の割り当てを改善する

  • 1つのディレクトリ内のファイル数に制限はありません
  • 許可リストのサポートが導入されました
  • トランザクションサポート（オプション、デバイスでサポートされている必要があります）
ご覧のとおり、近い将来、上限によってあらゆる種類の質問が削除されます。 私は頭の中にメビバイトとエクスビバイトを想像していませんでしたが、 「640キロバイトのメモリですべての人に十分」だと確信しています :)



もちろん、exFATは商用システムに似ているため疑問が残りますが、私的使用の文脈では、これらのニュアンスは無視できると思います。
