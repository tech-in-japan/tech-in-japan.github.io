ExchangeからZimbraへのメールの転送

Microsoft ExchangeでZimbraに切り替える理由 Zimbraはオープンソースですので、従業員からサーバーに保存されるすべての情報を完全に制御できます。 Microsoft Exchangeを変更することはできません。また、Zimbraは、Open Source Editionで完全にブランド変更し、Network Editionでリマーケティングするように構成できます。 Exchangeを購入する場合、他のMicrosoftソフトウェアを購入する必要がある可能性が高く、Zimbraではこれを行う必要はありません。 Exchangeは、他のビジネスに不可欠なアプリケーションと統合することも非常に困難ですが、Zimbraにとって、これらのタスクは非常に簡単です。 最後に、組織の郵便料金について質問があります。 この記事では、ExchangeからZimbraにすばやく切り替える方法について説明します。





Microsoft Exchange 2003、2007、または2010からZimbra への移行を検討している場合、 ZCS移行ウィザードがすべてのデータの転送に役立ちます。 アカウント、メッセージ、カレンダー、アドレス帳を転送できます。 また、Microsoft ExchangeのキーワードはZimbra Collaborationタグに変換されます。 ZCS移行ウィザードは、エラー、警告付きのメモ、タスク、ログを除くすべてのデータをOutlookサーバーからインポートします。



移行中、ZCSは次のタスクを実行します。



  • ZCSサーバーにユーザーアカウントが存在しない場合は作成します。
  • 電子メール、添付ファイル、カレンダー、およびアドレス帳をインポートします。
  • ログファイルを作成して、エラーまたはエラー警告を解決します。


移行計画



Exchangeサーバーから移行する前に、新しいZCS電子メールプログラムをユーザーに通知し、新しい電子メールアクセス情報（ZCS URL、ユーザー名、パスワード）をユーザーに提供する計画を作成します。



サーバーの負荷が最小限の時間に移行を計画することをお勧めします。 移行後にExchangeから送信された電子メールは、ZCSメールボックスに保存されません。



ExchangeからZCSに移行するには、2つのオプションがあります。



1. ZCS移行ウィザードは、新しいアカウントの設定方法に関する一連のダイアログボックスで質問をします。 開始する前に、以下を決定する必要があります。



  • ユーザーアカウントの移行先のドメイン。
  • 個別のアカウントを選択するか、インポートするアカウントのグループを選択するか。


2. ZCS移行ウィザードの[ワンステップ移行]オプションを使用すると、生成されたXMLファイルを移行に必要なユーザーデータで使用できます。



移行 簡単な説明



移行を開始する前に、メールリレー情報をExchangeサーバーではなくZimbraサーバーに変更します。



アカウントの移行とメールのインポート



1.移行ウィザードをダウンロードします。



2. ZCS移行ウィザードを実行し、[次へ]をクリックして続行します。



3.「インポート先」ダイアログボックスが表示されます。 サーバーに接続して認証するには、次のように入力します。



  • ホスト名 これは、Zimbraサーバーのホスト名です。
  • 管理ポート デフォルトの管理ポートは7071です。
  • 安全な接続（SSL）を使用して安全な接続を確立する
  • サーバーポートへの接続。
  • ユーザー名とパスワード。 これは、Zimbraサーバーの管理者アカウントの名前とパスワードです。


「次へ」をクリックします。



4. [宛先ドメイン]ダイアログボックスで、アカウントが転送されるZimbraドメイン名を選択します。



5.インポートを続行するには、[MAPIログオン]ダイアログボックスで、ZCS移行ウィザードで使用するために作成したMAPIプロファイルを定義します。 [プロファイル選択の起動]をクリックして、既存のMAPIプロファイルの選択または新しいプロファイルの作成を開始します。



6. [ソースおよび宛先メールボックス]ウィンドウで、ExchangeからZimbraサーバーに移行するユーザーを選択します。



7. [送信元および宛先メールボックス]で追加されたアカウントを確認し、[次へ]をクリックします。



8. ZCS移行ウィザードは、これらのアカウントが存在するかどうかを確認します

Zimbraサーバー。



9.アカウントのサービスクラス（COS）を選択します。



10.アカウントの作成を待ちます。



11.転送する情報を選択します。



12.完了。
