I対openSUSE

簡単な紹介

この記事は長い間書かれてきましたが、現時点では関連性のないものよりも少し多く書かれています。 openSUSEに少し座った後、私はそれに執着して、最近までメインのディストリビューションがありました。



こんにちは、Habrahabrコミュニティの皆さん、こんにちは！ この記事では、openSUSEに戻りたいという私の印象についてお話したいと思います。



ご存じのように、openSUSEディストリビューションについては、このOSは非常にユーザーフレンドリーであり、そのすべての部分が相互によく適合しているという意見があります。 私はこの意見の一部に完全に同意します。 ビルドの品質に関して、openSUSEは、私の理解では、MacOSXに近いか、MacOSXと同等です。 他のどの配布キットも、私のような感情を引き起こしたことはありません。 しかし、使いやすさに関しては、ここで少し疑問に思う傾向があります。 詳細。



どのようにすべてが始まりましたか？



これが「シェル」への私の2番目のアプローチです。 最初は数年前で、成功したかのようでした。 ただ、当時は通常のインターネットアクセスがありませんでした。 そして、ATIビデオチップにいくつかの問題がありました。 そこで、より良い時期まで知り合いを延期し、AgiliaLinuxに置き換えました。どのバージョンを覚えていません。 そして最高の時が来たとき、私はその時点でMateデスクトップでLinux Mintを試すことにしました。 それで彼はそれ以来私と一緒に定着しました。 そして、彼はまだ立っていたでしょうが、昨日、私は我慢できなかった新しいものを見て、openSUSE 13.1をインストールしたかったのです。



設置



ご存じのように、Ubuntuとその兄弟/姉妹は、USBフラッシュドライブまたはハードドライブにあるisoイメージからダウンロードしてインストールできます。 openSUSEでも同じことができますか？ 試してみるべきだと思います。 それは機能しません、そしてそれは運命ではありません。 そして、私はしようとしません。



そのため、インターネットで簡単に検索すると、 このページに移動しました 。 考え直さずに、新しくダウンロードした配布キットのイメージをマウントし、そこからgrub.cfgファイルを取り出して、次の修正を行いました（太字）：



menuentry 'インストール' --class opensuse --class gnu-linux --class gnu --class os {

set gfxpayload = keep

ループバックループ/openSUSE-13.1-DVD-x86_64.iso

echo 'カーネルを読み込んでいます...'

linuxefi （ループ） / boot / x86_64 / loader / linux install = hd：////openSUSE-13.1-DVD-x86_64.iso

echo '初期RAMディスクを読み込んでいます...'

initrdefi （ループ） / boot / x86_64 / loader / initrd

}


このファイルをisoイメージと一緒にUSBフラッシュドライブのルートディレクトリに置き、名前をgrub-suse.cfgに変更しました。 以前、同じフラッシュドライブにgrub2-efiをインストールしましたが、実際に示したように、grub2が既にハードドライブにある場合、これは必要ありません。 それがそうであっても、私は再起動します。 grub2メニューが表示されます。 [Esc]を押すと、コマンドラインに移動して次のように記述できます。



ルート設定=（hd1,1）

insmod /path/to/grub2_modules/loopback.mod

configfile /grub-suse.cfg


メニューが表示されます。 最初のアイテムを選択すると、見よ、ダウンロードが開始されました！ それだけでなく、行ってください！ インストーラーは正常に起動し、システムも正常にインストールされました。 しばらくして、コンピューターが再起動され、新しくインストールされたシステムになりました。 更新アプレットは、システムが更新を望んでいることを示しました。 さて、それを更新しましょう。 更新の場合のみ、ターミナルでsudo zypper upコマンドを実行しました。 そして、タスクバーのアプレットには情報がありません。 更新後、Packmanを接続してコーデックをインストールし、これを正しく行う方法についていくつかの発見をしました。



システムの再紹介



上で書いたように、これは「シェル」への私の2番目のアプローチです。 したがって、すべてではないにしても、多くのことはすでによく知られているはずです。 原則として、そのとおりです。 しかし、すべてが期待したほどバラ色ではありませんでした。 これについて順番に。



まず、KDEの一般的な印象はなんとなく憂鬱です。 KDEが唯一の本格的なデスクトップであり、他のすべてが単なる哀れな工芸品であると思っていた時代がありました。 しかし、最初にAgiliaLinuxを使用し、次にLinux Mintを使用したことで、最初にGNOME2に慣れましたが、一般的にはMateに夢中になりました。 環境は非常にシンプルで高速であり、美しさと過剰さで過負荷になりません。 背景に対して、KDEは大きく、光沢があり、非常に重いもののように見えます。 AMDからドライバーをインストールするまで、プロセッサーのファンは定期的に回転し、停止しました。



上記の問題は、一般に問題ではありません。 これはただの習慣の問題です。 しかし、問題は少し後に現れ始めました。 ウィンドウの最小化やウィンドウの切り替えなどの単純なアクションを実行すると、画面が不適切なものに変わることがあります。 画像は三角形のダンプのように見えました。 テーブルの周りでマウスを動かすと、この三角形の束全体が動き、この混乱の中で何も見つけることができませんでした。 後続のsudo /etc/init.d/xdm restartコマンドを使用して最初のシステムコンソールに切り替えることのみが役立ちました。 このような問題に約5回直面したため、視覚効果をオフにしました。 今では問題はありませんが、そのようなものはありません、珍しいです。 まあ。 主なことは、すべてが機能し、グロス再起動なしであるということです。



その後、IPTVに問題がありました。 必要なコーデックはすべてありますが、表示されないだけです。 問題はファイアウォールにありました。 実際のところ、openSUSEではデフォルトでアクティブになっています。 そして、とりわけ、すべての放送を消します。 さて、/ var / log / firewallを開いて、最後の行を読み、その中のブロードキャストアドレスを見つけて、ユーザー定義のファイアウォールルールのリストに貼り付けます。 最終的には、すべてが機能します。



これまでのところ、これがすべての問題のようです。 ご覧のとおり、これらの問題は解決されています。 しかし、それらは初心者には簡単ではありません。 どこを掘るかわからないので、すぐにシステムにがっかりする可能性があります。 少なくともコーデックをインストールしてください。 ネットワークにはプロセスの多くの説明がありますが、「サプライヤの変更」について少なくともいくつかの場所で言われています。 最初は、サプライヤを変更してパッケージを更新するという提案により、システムを破壊することに対する疑念と恐れが生じました。 後で、コマンドラインを使用する前にzypper GUIにアクセスして、必要なバージョンを選択できることを発見しました。 この場合、依存パッケージも同じプロバイダーのパッケージに置き換えられます。



システムがインストールされます。 パッケージに問題はありません。 ビデオ、オーディオ、テレビ作品。 それは、楽しんで楽しんでいるようです！ しかし、いくつかの奇妙な感じは絶えずかじります。 Mint 17を待たずに、今すぐに16日目を返してopenSUSEを破壊したいという願いが常にあります。 同時に、何かが止まります。 さて、待って見てください-次に何をすべきかを決定してください...
