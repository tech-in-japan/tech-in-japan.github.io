今日のダイジェストでは、Habréのブログで公開された2017年の人気のある資料を思い出します：私たち自身の大学の開発とプロジェクト、ITMOへの応募者のための有用な資料、エッセイとプログラミングストーリー、自宅での開発方法を学びたい人のためのヒント、仕事についての大きなサイクル科学記事を読んだ（そして書いた）人のための高速な読み方のテクニックとヒント-これについては以下のセレクションで説明しています。

ITMO大学のプロジェクトと開発



  • ビッグデータと機械学習：医学の新たな機会 。 ビッグデータを使用した医療技術の市場のボリュームは、西とロシアの両方で成長しています。 この記事では、ITMO大学でどのような開発が行われているかについて説明します。特定のプロジェクト（サンクトペテルブルクでの救急車の作業のモデリング）から、計算生物医学や化学情報学などのトレーニングプログラムまで。


  • ITMO大学は、壊れないストレージシステムの開発を開始します 。 ITMO大学の科学者は、量子技術の問題に長い間関与しており、量子通信システムの開発の経験があります。 新しいイニシアチブの一環として、専門家のチームが壊れることのない効率的に管理されたデータセンターの技術に取り組んでいます-このプロジェクトにはグローバルな類似物はありません。


  • 第二世代クラウドコンピューティング：CLAVIREシステム 。 ITMO大学の科学者は、量子技術の問題に長い間関与しており、量子通信システムの開発の経験があります。 新しいイニシアチブの一環として、専門家のチームが壊れることのない効率的に管理されたデータセンターの技術に取り組んでいます-このプロジェクトにはグローバルな類似物はありません。


  • ITMO大学の科学者は遠距離で新しいエネルギー伝送システムを提案しました 。 ITMO大学は、誘電体共振器ディスクに基づいたワイヤレスエネルギー転送用のプロトタイプデバイスを開発しました。 テクノロジーの有効性は90％に達しますが、ソースとレシーバーの相対的な位置が変わった場合でも一貫しています。 開発は、外国のアナログと競合するだけでなく、多くのパラメーターでそれらを上回ります。


  • 「Know Yourself」：ITMO大学でのソーシャルメディアマイニングプロジェクト 。 ITMO大学の学生と従業員のどのプロジェクトがソーシャルネットワークからの情報を使用するかについて話します。潜在的なバイヤーのより正確なプロファイリング、ユーザーの行動の予測、さらには混雑した場所での緊張の病巣の認識と防止を可能にする感情の分析のためです。


役に立つ読書



  • 速読：動作するかどうか？ パート1 これは、速度読み取りサイクルの最初の資料です。 その中で、速読が読めない理由を説明し、速読者の顕著な成功を説明し、緊急モードでの難しい試験の準備が必要な場合、このテクニックに頼るべきではない理由を説明します。


  • 速読：動作するかどうか？ パート2：テクニックの分析 。 速読の指導に使用される最も一般的な手法のいくつかを検討します。 斜め読み、ノンストップリーディング、サブボーカリゼーションの抑制、RSVP-これらの方法のうち、実際に機能するのはどれで、どの方法が役に立たないか、生理学的にも不可能ですか？ サイクルの後半でこれについて。


  • 速読：動作するかどうか？ パート3：簡単なヒント 速読技術の習得に時間を費やしたくない人のために何をすべきか？ 「遅い読み取り」と同様に、読み取り効率を改善するための代替のトリックについて説明します。 そして、速読なしでは本当にできない状況について。


  • 科学論文の読み方：科学者からのアドバイス 。 記事を読むことはそれを理解することを意味しません。 特に科学出版に関しては。 この資料では、科学記事を選択して読む方法、この記事の構成内容を分析する方法、および時間をできる限り効率的にする方法に関する実践的なアドバイスを提供します。 また、科学者や科学雑誌の編集者、一般的に、この種の出版物を定期的に取り扱わなければならない人々から助言を与えてください。


  • 「さらに苦労せずに」：最短の科学記事 。 「科学論文を読むには時間がかかります」-このルールでさえ例外があります。 私たちは科学のパラドックスについて話します：単語の数が最小限である（そして単語がまったくない）記事です。 それらが存在する理由と科学コミュニティに与えるものについて-この資料で。

ITMO大学：入学およびトレーニングの取り組み



  • 大学への非古典的な入学 。 この資料では、イリヤ・グレボフの物語を共有しています。 IlyaはVkontakteの脆弱性を発見し、ITMO大学に興味を持つようになったHabréに投稿しました。 イリヤとの会話の中で、数学の試験の得点が不十分なため、学生はITMOの学生になる可能性を考慮しなかったことが判明しました。 最終的に大学入学の妨げにならなかった理由を説明し、他の才能ある志願者に自分を宣言する方法についてアドバイスします。


  • 大きなインタビュー：ITMO大学への入学方法 。 ITMO大学の入学委員会の副会長であるアンナヴェクリチへのインタビュー。 メガファカルティがどのように配置されているか、これは専門分野の選択、入学の機会を増やすことができるイニシアチブの決定に役立ちます-これらすべては私たちの資料にあります。


  • 入場キャンペーン2017：便利なリンクと資料 。 このダイジェストでは、ITMO大学への応募者に役立つ可能性のあるすべてのリソースを収集しました。 大学に関する資料、建物のバーチャルツアー、学部および大学院プログラムの応募者向けポータル、有用なイベントのカレンダー、応募者の質問と回答、および学生と卒業生の歴史。


  • 学問分野としてのブロックチェーン：ITMO University Master's program 。 ITMO大学の数理物理学および情報理論（MFiTI）のベース部門で開始された、新しい修士プログラム「数理物理学および情報理論」に関する資料。 大学院生が学ぶこと、ブロックチェーンに関連する実際的なプロジェクトでは、彼らは自分自身を証明することができます-これについては本文で詳しく説明します。


プログラミング：理論と実践



  • 永遠の学生：プログラミングが絶え間ない「研究」であるとき 。 プログラマーが常に新しいことを学ぶ必要があるのはなぜですか？ 開発者は空き時間をどのように費やしますか：どの言語が仕事で使用され、どの言語が自分のプロジェクトで使用されますか？ プログラミングが目標としてではなく、学習ツールとして使用されるのはいつですか？ 私たちの資料のこのすべてについて。


  • 卒業証書のないプログラマー：ITで「ホームトレーニング」を行えることとできないこと 。 自宅と大学でプログラミングを教えることの長所と短所について話します。このアプローチと他のアプローチの落とし穴を分析します。 また、大学での勉強が将来の開発者にとって本当に貴重な経験になる可能性についても話し合っています。


  • 「病気の治療」：自動プログラミング 。 ソフトウェアの問題の原因と、状況を修正する可能な方法-自動プログラミングについて話している。 それがどのように機能するのか、なぜ「おもちゃ」プロジェクトだけでなく適切なのかについて話します。


  • プログラムの設計方法：UMLから自動化アプローチまで 。 この資料は、ドキュメントの作成とプログラムロジックの開発の問題の解決に役立つ概念とアプローチに関するものです。 UMLが「離陸」しなかった理由と、オートマトンプログラミングがこの点に関してどのような観点を持っているかについて説明します。


  • MOOC「プログラミングとWebアプリケーション開発」：コースの著者との発表とインタビュー 。 オンラインコース「プログラミングとWebアプリケーション開発」の著者であり、ITMO大学のコンピューター教育技術学部の准教授であるDmitry Shtennikovのインタビュー。 コースの作成方法、対象者、およびコースの「実用」言語としてPythonが選択された理由のストーリー。


  • ITMO University Digest：データサイエンスに参加したい人のための資料 。 これは、（「純粋な」プログラミングに加えて）データの操作をマスターしたい人（理論上および実際）に役立つ材料の大規模な選択です。 ダイジェスト-トピックに関する有用な本、記事、ビデオコース。


  • 難解なプログラミング言語の5分間のガイド： 1および2 この一連の資料では、プログラミングの「非実用的な」側面である難解なPLについて説明します。 それらがなぜ必要なのか、誰がいつそれらを使用するのか（そして何の目的のために）、そして難解な言語をどのように分類できるのかについて話している。
