Androidプロセスとスレッド：AsyncTaskを正しく記述する

Androidについての話を続けます。 そして今回は、プロセスとスレッドに関する貴重な情報を共有したいと思います。これらの情報は、アプリケーションを記述する際のエラーや誤解を避けるために、よく理解され、常に手元にある必要があります。 記事の最後に、ボタンをクリックするだけで画像をImageViewにロードするAsyncTaskの実装例を示します。



まず、このガイドのこのトピックについて詳しくお読みいただけます-developer.android.com/guide/topics/fundamentals/processes-and-threads.html



Androidのプロセスとスレッドに関する注意



アプリケーションコンポーネントが起動し、アプリケーションで他のコンポーネントが実行されていない場合、Androidは単一の実行スレッドでアプリケーションの新しいプロセスを作成します。 デフォルトでは、1つのアプリケーションのすべてのコンポーネントは、「メイン」と呼ばれるスレッドの1つのプロセスで起動されます。 アプリケーションコンポーネントが起動され、特定のアプリケーションのプロセスが既に存在する場合（アプリケーションの一部のコンポーネントが存在する場合）、コンポーネントはこのプロセスで起動され、その実行フローを使用します。 アプリケーションのコンポーネントごとに異なるプロセスを設定することにより、この動作を変更できます。 さらに、任意のプロセスにスレッドを追加できます。



マニフェストファイルを使用して、コンポーネントの個別のプロセスを定義できます。 各コンポーネントタグ（アクティビティ、サービス、レシーバー、プロバイダー）は、android：process属性をサポートします。 この属性を使用すると、コンポーネントを実行するプロセスを指定できます。 異なるアプリケーションのコンポーネントが実行されるプロセスを指定することもできます。 この属性は、すべてのアプリケーションコンポーネントに特定のプロセスを指定できるようにするアプリケーションタグでもサポートされています。



Androidは、可能な限りアプリケーションプロセスをサポートしようとしますが、リソースが必要な場合は、重要な階層に従って古いプロセスが置き換えられます。



重要度の階層には5つのレベルがあります：（リストの最初のレベルのプロセスは最後に削除されます）



1.ユーザーが対話するプロセス（フォアグラウンドプロセス）

そのようなプロセスには、たとえば、次のものが含まれます。 ユーザーが対話するサービス（サービスインスタンス）。 startForeground（）メソッドによって起動されたサービス。 ライフサイクルのいずれかの方法を実行するサービス。 onReceive（）メソッドを実行するBroadcastReceiver。



2.目に見えるプロセス

パラグラフ1の条件が満たされていないが、ユーザーが画面に表示するものに影響するプロセス。 たとえば、アクティベーションのonPause（）メソッドが呼び出されます。



3.サービスプロセス

startService（）によって開始されたサービス



4.バックグラウンドプロセス

バックグラウンドで実行され、ユーザーには見えないプロセス。



5.空のプロセス



アプリケーションコンポーネントにはonLowMemory（）メソッドが存在しますが、onDestroy（）メソッドに100％依存できないのと同様に、このメソッドの呼び出しに依存することはできないので、データや設定を保存するロジックをメソッドに実装できますonStop（）、これは（彼らが保証する）正確に呼び出されます。



アプリケーションが起動すると、システムはこのアプリケーションの実行の「メイン」スレッドを作成します。これはUIスレッドとも呼ばれます。 このストリームは、ウィジェット（ボタン、リスト）が描画され、アプリケーションのイベントが処理されるため、非常に重要です。 システムは、コンポーネントのインスタンスごとに個別のスレッドを作成しません。 1つのプロセスで実行されているすべてのコンポーネントは、UIスレッドで作成されます。 Androidユーザーインターフェイスライブラリはスレッドセーフではないため、次の2つの重要なルールがあります。



1）UIスレッドをブロックしない

2）非UIスレッドからUIコンポーネントにアクセスしないでください



ここで、ネットワークからImageViewに画像をアップロードしてすぐに表示するというタスクがあったとします。 どうする？ 論理的には、次のような、すべての作業を実行する別のスレッドを作成します。 

public void onClick(View v) { new Thread(new Runnable() { public void run() { Bitmap b = loadImageFromNetwork("http://example.com/image.png"); mImageView.setImageBitmap(b); } }).start(); }


画像をロードする操作を別のストリームに移動したため、信じられないように見えます。 問題は、ルール番号2に違反したことです。 次の方法を使用して、この問題を修正できます。



Activity.runOnUiThread（実行可能）

View.post（実行可能）

View.postDelayed（実行可能、長い）



たとえば、最初のものを使用します。 

 public void onClick(View v) { new Thread(new Runnable() { public void run() { final Bitmap bitmap = loadImageFromNetwork("http://example.com/image.png"); mImageView.post(new Runnable() { public void run() { mImageView.setImageBitmap(bitmap); } }); } }).start(); }


現在、実装はスレッドセーフです。ネットワーク操作は別のスレッドで実行され、ImageViewはUIストリームからアクセスされます。

幸いなことに、これらの操作はHandlerクラスの継承を使用して必要なロジックを実装することで組み合わせることができますが、最善の解決策はAsyncTaskクラスを継承することです。



AsyncTaskを使用すると、非同期作業を実行し、ユーザーインターフェイスを更新できます。

アップグレードするには、onPostExecute（）メソッドを実装し、すべてのバックグラウンド作業をdoInBackground（）メソッドで囲みます。 独自のタスクを実装したら、execute（）メソッドで実行する必要があります。



画像の読み込みと表示のタスクを実装する、約束されたAsyncTaskの例を紹介します（注釈付きのオプションと標準ダイアログプロトコルの使用からの逸脱）。 

 @EActivity(R.layout.main) public class DownloadImageActivity extends Activity { ProgressDialog progress; @ViewById ImageView image; @ViewById Button runButt; String pictUrl = "http://www.bigfoto.com/sites/main/churfirsten_switzerland-xxx.JPG"; @BeforeCreate void getProgressDialog(){ progress = new ProgressDialog(this); progress.setMessage("Loading..."); } @Click(R.id.runButt) void runClick(){ new DownloadImageTask().execute(pictUrl); } class DownloadImageTask extends AsyncTask<String, Void, Bitmap> { @Override protected Bitmap doInBackground(String... params) { publishProgress(new Void[]{}); //or null String url = ""; if( params.length > 0 ){ url = params[0]; } InputStream input = null; try { URL urlConn = new URL(url); input = urlConn.openStream(); } catch (MalformedURLException e) { e.printStackTrace(); } catch (IOException e) { e.printStackTrace(); } return BitmapFactory.decodeStream(input); } @Override protected void onProgressUpdate(Void... values) { super.onProgressUpdate(values); progress.show(); } @Override protected void onPostExecute(Bitmap result) { super.onPostExecute(result); progress.dismiss(); image.setImageBitmap(result); } } }




次に、ダイアログの操作の観点から最も正しいオプションを検討します。 

 public class DownloadImageActivity extends Activity { ImageView image; Button runButt; final static String PICT_URL = "http://www.bigfoto.com/sites/main/churfirsten_switzerland-xxx.JPG"; final int PROGRESS_DLG_ID = 666; final static String DEBUG_TAG = "+++ImageDownloader+++"; @Override public void onCreate(Bundle savedInstanceState){ super.onCreate(savedInstanceState); setContentView(R.layout.main); loadViews(); } private void loadViews(){ runButt = (Button)findViewById(R.id.runButt); image =(ImageView)findViewById(R.id.image); } @Override protected Dialog onCreateDialog(int dialogId){ ProgressDialog progress = null; switch (dialogId) { case PROGRESS_DLG_ID: progress = new ProgressDialog(this); progress.setMessage("Loading..."); break; } return progress; } public void runButtonHandler(View button){ if(button.getId() == R.id.runButt) new DownloadImageTask().execute(PICT_URL); } class DownloadImageTask extends AsyncTask<String, Void, Bitmap> { @Override protected Bitmap doInBackground(String... params) { publishProgress(new Void[]{}); String url = ""; if( params.length > 0 ){ url = params[0]; } InputStream input = null; try { URL urlConn = new URL(url); input = urlConn.openStream(); } catch (MalformedURLException e) { Log.d(DEBUG_TAG,"Oops, Something wrong with URL..."); e.printStackTrace(); } catch (IOException e) { Log.d(DEBUG_TAG,"Oops, Something wrong with inpur stream..."); e.printStackTrace(); } return BitmapFactory.decodeStream(input); } @Override protected void onProgressUpdate(Void... values) { super.onProgressUpdate(values); showDialog(PROGRESS_DLG_ID); } @Override protected void onPostExecute(Bitmap result) { super.onPostExecute(result); dismissDialog(PROGRESS_DLG_ID); image.setImageBitmap(result); } } }


より多くのコードがありますが、ダイアログの操作には標準プロトコルを使用することをお勧めします。

また、初心者がこのコードを簡単に試せるように、すべての注釈を削除しました。



指定した値を持つ属性をボタンレイアウトに追加することを忘れないでください：android：onClick = "runButtonHandler"



さらに、公式ドキュメント（ Tyts ）および私の場合、preExecute（）は使用されませんが、タスクを開始する前にユーザーインターフェイスでいくつかのアクションを実行する必要がある場合は、このメソッドを使用してください。



AsyncTaskに渡されるパラメーター：

1.パラメーター（この場合、URL）。

2.進行状況（タスク変更の進行状況を指定する単位）。 私たちの場合、それは使用されません。

3.タスクの結果（この場合、ビットマップオブジェクト）



コードは非常に簡単です。すべてのバックグラウンド作業をdoInBackGround（）メソッドで行い、publishProgress（）メソッドを呼び出します。これにより、onProgressUpdate（）メソッドが呼び出されたときに画像がロードされるとProgressDialogがスピンします。 バックグラウンドの作業が完了した後、onPostExecute（）メソッドが呼び出され、doInBackGround（）メソッドの結果が転送されます。この場合、これはBitmapオブジェクトであり、そこでImageViewにロードします。

考慮すべきいくつかの重要な点に注意してください。



1）doInBackGround（）メソッドはバックグラウンドスレッドで実行されるため、このメソッド内ではUIストリームにアクセスできません。

2）onPostExecute（）およびonProgressUpdate（）メソッドはUIスレッドで実行されるため、UIコンポーネントに安全にアクセスできます。
