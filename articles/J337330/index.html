Clever Geek Handbook

データを操作するために選択するプログラミング言語は何ですか？





初心者のデータ科学者は、この科学をすばやく習得するのに役立つ多くのプログラミング言語の1つを選択する機会があります。



ただし、どのプログラミング言語がこの目的に最も適しているかを正確に伝える人はいません。 この分野の専門家としてのあなたの成功は多くの要因に依存し、今日はそれらを考慮しようとします。記事の最後で、データを扱うのに最適だと思うプログラミング言語に投票できます。



特異性



データサイエンスの分野に深く入り込むと、何度も何度も「車輪を再発明」しなければならないという事実に備えてください。 さらに、選択したプログラミング言語のさまざまなソフトウェアパッケージとモジュールを完全にマスターする必要があります。 このすべてをどれだけうまく学習できるかは、まず、選択した言語パックの主題指向のソフトウェアパッケージの可用性に依存します。



汎用性



一流のデータスペシャリストは、優れた包括的なプログラミングスキルと、計算および分析を実行する能力を備えています。 データサイエンスの分野での日々の仕事のほとんどは、ソースデータの検索と処理、またはデータの調整を目的としています。 残念ながら、これらの目的に役立つ機械学習用の新しいパッケージはありません。



有効性



ペースの速い商用データサイエンスの世界では、必要な仕事をすばやく獲得する多くの機会があります。 それにもかかわらず、データサイエンスの分野の急速な発展のおかげで、技術的な問題が絶えず伴い、そのような欠点を最小限に抑えることができるのは永続的な実践のみです。



性能



場合によっては、特に大量の重要なデータを扱う場合、コードのパフォーマンスを最適化することが非常に重要です。 ただし、コンパイルされた言語は通常、インタープリター言語よりもはるかに高速です。 同様に、静的に型付けされた言語は、動的に型付けされた言語よりもはるかにフォールトトレラントです。 したがって、唯一の妥協点はパフォーマンスの低下です。



ある程度まで、以下に示す各プログラミング言語には、2つのグループのそれぞれに1つのパラメーターがあります。普遍性-特異性。 パフォーマンスは便利です。



これらの基本原則を踏まえて、データサイエンスで使用される最も一般的なプログラミング言語のいくつかを見てみましょう。 以下のプログラミング言語に関するすべての情報は、私自身の観察と経験、および私の友人と同僚の経験に基づいています。



R













古いプログラミング言語Sの直接の子孫であるRは1995年にリリースされ、その後より洗練されました。 CやFortranなどの言語で記述されたこのプロジェクトは、R Foundation for Statistical Computingによってサポートされるようになりました。



ライセンス：



無料



利点：



  • 高品質のオープンソースドメイン固有パッケージの優れたセット。 Rには、想像することのできるほぼすべての定量的および統計的アプリケーション用のパッケージが用意されています。 これには、ニューラルネットワーク、非線形回帰、系統学、複雑な図の作成、グラフなどが含まれます。
  • 付属資料の基本的なインストールと一緒に、広範な組み込み関数とメソッドをインストールする機会が与えられます。 さらに、Rは行列代数データを完全に処理します。
  • データを視覚化する機能は、ggplot2などのさまざまなライブラリを使用する機能とともに重要な利点です。


短所：



  • 遅いパフォーマンス。 言うことはありません：Rは高速PLではありません。
  • 特異性。 Rは統計研究やデータサイエンスには最適ですが、一般的な目的のプログラミングにはあまり適していません。
  • その他の機能。 Rには、他の言語での作業に慣れているプログラマーを混乱させるいくつかの珍しい機能があります。1から始まるインデックス付け、いくつかの代入演算子の使用、型破りなデータ構造。


私たちの評決は最初の目的に理想的です。



Rは強力な言語であり、統計データとデータの視覚化を収集するための膨大なアプリケーションを特徴とし、オープンソースYPであるという事実により、開発者の間で多くのファンを引き付けることができます。 このプログラミング言語は当初の目的に対する有効性により、なんとか人気を博しました。



Python













1991年、Guido van RossumはPythonプログラミング言語を導入しました。 それ以来、この言語は非常に人気のある汎用言語になり、データの専門家のコミュニティで広く使用されています。 現在、メジャーバージョンはPython 3.6およびPython 2.7です。



ライセンス：



無料



利点：



  • Pythonは非常に人気があり、広く使用されている汎用プログラミング言語です。 特別に設計されたモジュールの広範なセットがあり、開発者によって広く使用されています。 多くのオンラインサービスは、PythonのAPIを提供します。
  • Pythonは簡単に習得できます。 エントリーのしきい値が低いため、プログラミングに携わる人にとって理想的な第一言語になります。
  • パンダ、scikit-learn、Tensorflowなどのソフトウェアパッケージにより、Pythonは最新の機械学習アプリケーションの信頼できるオプションになります。


短所：



  • 型安全。 Pythonは動的に型指定された言語です。つまり、Pythonを使用するときは注意が必要です。 型の不一致エラー（たとえば、整数を予期するメソッドに引数として文字列を渡す）は、時々発生する可能性があります。
  • たとえば、統計分析とデータ分析に特定の目標がある場合、R言語パッケージの広範なセットはPythonよりも優れています。 さらに、プログラミング言語にはPythonのより高速で安全な代替手段があります。


私たちの評決はあらゆる点で便利です



Pythonは、データサイエンスの目的に適したオプションであり、このステートメントは、この分野の初心者レベルと上級レベルの両方の作業に当てはまります。 データサイエンスのほとんどは、ETLプロセス（retrieve-transform-load）に集中しています。 この機能により、Pythonはそのような目的に最適なプログラミング言語になります。 GoogleのTensorflowなどのライブラリにより、Pythonは非常に興味深い機械学習言語になります。



SQL

img align = "center" src = " habrastorage.org/web/f7e/2cf/42d/f7e2cf42d60b4b8fa6f442504828fe57.png " />



SQL（「構造化照会言語」）は、リレーショナルデータベースを定義、管理、および照会します。 この言語は1974年に登場して以来、多くの修正が加えられていますが、その基本原則は変わっていません。



ライセンス：



無料と有料のオプションがあります。



メリット



  • クエリ、更新、およびリレーショナルデータベースの処理時に非常に効果的です。
  • 宣言構文により、SQLは非常に読みやすい言語になります。 18歳以上のユーザーからのSELECT名が何をすべきかについて不確実性はありません！
  • SQLはさまざまなアプリケーションで非常に頻繁に使用されるため、SQLに精通していると非常に役立ちます。 SQLAlchemyなどのモジュールは、他の言語とのSQL統合を簡素化します。


短所：



  • SQL構文は、命令型プログラミングに慣れている人にとっては非常に困難な作業のように思えるかもしれません。
  • PostgreSQL、SQLite、MariaDBなど、SQLにはさまざまなバリエーションがあります。 それらはすべてまったく異なるため、互換性について話すことはできません。


私たちの評決は時間にもかかわらず有効です



SQLは、高度な分析ツールよりもデータ処理言語としてより便利です。 それにもかかわらず、データサイエンスの分野における非常に多くのプロセスはETLに依存しており、SQLの寿命と有効性は、すべてのデータサイエンティストがそのようなPLを知っている必要があることを再度示しています。



Java











Javaは、Java仮想マシン（JVM）で実行される非常に人気のある汎用言語です。 プラットフォーム間のシームレスな移植性を提供する抽象的なコンピューティングシステムです。 現在、Oracleでサポートされています。



ライセンス：



8番目のバージョン-無料



利点：



  • 普遍性。 多くの最新のシステムとアプリケーションは、Java言語を使用して開発されています。 このようなPLの大きな利点は、データサイエンスの方法を既存のコードベースに直接統合できることです。
  • 厳密なタイピング。 型の安全性を提供することは、Javaにとって空のフレーズではありません。また、ビッグデータを扱うミッションクリティカルなアプリケーションを開発する場合、この機能はこれまで以上に重要になります。
  • Javaは、高性能でコンパイルされた汎用言語です。 これにより、効率的な製品ETLコードの作成、およびコンピューティングツールを使用した機械学習アルゴリズムに適しています。


短所：



  • Java言語の「冗長性」は、特別な分析を行い、より専門的な統計アプリケーションを開発するための最良の選択肢ではありません。
  • Javaには、Rなどの一部のドメイン固有言語と比較して、高度な統計的手法のための多数のライブラリがありません。


私たちの評決は、データサイエンスの分野での仕事に最適な言語のタイトルに対する真剣な候補です。



データサイエンスの分野で働くための言語としてJavaを学習することを支持する多くのことが言えます。 多くの企業は、既製のソフトウェア製品コードを独自のコードベースにシームレスに統合できることを高く評価しています。Javaのパフォーマンスとタイプセーフティは、紛れもない利点です。 ただし、このような言語の欠点には、他の言語で使用できる特定のパッケージのセットがないという事実が含まれます。 この欠点にもかかわらず、Javaは、特にRまたはPythonを既に知っている場合は特に注意する必要があるプログラミング言語です。



スカラ









JalaベースのScalaプログラミング言語は、2004年にMartin Oderskyによって開発されました。 これは、オブジェクト指向のアプローチと機能的なアプローチの両方を使用できるいくつかのパラダイムを持つ言語です。 さらに、Apache Sparkクラスターコンピューティング構造はScalaで記述されています。



ライセンス：



無料



利点：



  • ScalaとSparkを使用すると、高性能クラスターコンピューティングを操作する機会が得られます。 Scalaは、大量のデータを扱う人に最適です。
  • マルチパラダイマティック。 Scalaで作業するプログラマーには、オブジェクト指向プログラミングと機能プログラミングの両方のパラダイムが利用可能です。
  • ScalaはJavaバイトコードにコンパイルされ、JVM上で実行されます。 これにより、Java言語と対話できるようになり、Scalaは非常に強力な汎用言語になります。 また、データサイエンスの分野での作業にも適しています。


短所：



  • Scalaを使用する場合は、頭を痛めないように準備してください。 特別なScalaプラグインを使用して、sbtをダウンロードし、EclipseやIntelliJなどのIDEを構成することをお勧めします。
  • Scalaの構文と型システムは非常に複雑であると考えられています。 したがって、Pythonなどの動的言語を扱うことに慣れているプログラマーは苦労します。


私たちの評決は、ビッグデータを扱うのに理想的です。



クラスターコンピューティングを使用してビッグデータを処理する場合は、Scala + Sparkのペアが理想的なソリューションです。 さらに、すでにJavaやその他の静的に型付けされたプログラミング言語の経験がある場合は、Scalaのこれらの機能を高く評価するでしょう。 ただし、アプリケーションがすべてのScalaコンポーネントを追加することを正当化できる大量のデータと関係がない場合は、RやPythonなどの他の言語を使用するとパフォーマンスが向上する可能性があります。



ジュリア













わずか5年前にリリースされたジュリアは、計算手法の世界に感銘を受けました。 この言語は、金融業界を含むいくつかの大規模な組織がすぐに独自の目的で使用し始めたという事実により、このような人気を獲得しました。



ライセンス：



無料



利点：



  • Juliaは、コンパイル済みのJIT言語（ジャストインタイム）であり、良好なパフォーマンスを実現できます。 この言語は非常にシンプルで、Pythonなどのインタープリタ型言語の動的な型指定およびスクリプト作成の可能性を提供します。
  • Juliaは数値解析用に設計されたものであり、汎用プログラミング言語と見なすこともできます。
  • 読みやすさ。 この言語で作業する多くのプログラマは、この機能が最大の利点であると考えています。


短所：



  • 未熟。 Juliaはかなり新しい言語であるため、一部の開発者はパッケージの操作中に不安定になります。 ただし、言語の基本的な手段は安定していると見なされます。
  • 言語が未熟であることのもう1つの兆候は、ソフトウェアパッケージの数が限られていることと、開発者の間の少数のファンです。 確立されたRやPythonとは異なり、Juliaプログラミング言語には多数のソフトウェアパッケージがありません（現時点では）。


私たちの評決はまだ証明される言語です



はい、ジュリア言語の主な問題はその若さですが、それを責めることはできません。 ジュリアは最近作成されたばかりであるため、まだ主要なライバルであるPythonおよびRと競合することはできません。辛抱強く、この言語に細心の注意を払うべき多くの理由があることを理解できます。



MATLAB









MATLABは、数値計算用に認識されている言語であり、科学的目的と業界の両方で使用されています。 これは、1984年に設立されたMathWorksによって開発およびライセンス供与され、その主な目標はソフトウェアの商用化でした。



ライセンス：





価格は言語オプションによって異なります。



利点：



  • 数値計算用に設計されたMATLABは、信号処理、フーリエ変換、行列代数、画像処理などの複雑な数学的要件を伴う定量分析の使用に適しています。
  • データの視覚化。 MATLABには、多くの組み込みのグラフ作成機能とチャート作成機能があります。
  • MATLABは、物理学、工学、応用数学などの正確な科学の多くの学部コースでよく見られます。 したがって、これらの分野で広く使用されています。


短所：



  • 有料ライセンス。 選択したオプション（科学的、個人的、または会社の目標）に関係なく、高価なライセンスを購入する必要があります。 ヒント：無料の代替案であるOctaveに注意してください。
  • MATLABは、最高の汎用プログラミング言語ではありません。


私たちの評決は目標に最適な選択肢であり、重要な数学的計算を必要とします



科学的目的と産業目的の両方でさまざまな定量計算で広く使用されているため、MATLABはデータサイエンスの分野での応用にふさわしいオプションになりました。 日々の目標を達成するために集中的で高度な数学機能が必要な場合に便利です。これはまさにMATLABの設計目的です。



他の言語



データの専門家にとって興味深いかもしれない他の人気のあるPLがあります。 このセクションでは、簡単な概要を説明します。



C ++



多くの場合、C ++はデータサイエンスの分野では使用されません。 ただし、非常に高速なパフォーマンスと幅広い人気があります。 C ++がデータサイエンスの分野で人気を得ていない主な理由は、この目的のための非効率性です。



フォーラム参加者の一人が書いたように：

「何らかの特別な分析を行うためのコードを書いているとします。これは一度だけ実行される可能性が高いです。 ですから、10秒間動作するプログラムの作成に30分を費やすか、1分間動作するプログラムに10分を費やしますか？」



そして、この男は正しいです！ ただし、低レベルの機械学習アルゴリズムを実装するには、C ++が最適です。



私たちの評決は日常の仕事には最適ではありませんが、生産性に関しては...



Javascript



Node.jsプラットフォームは過去数年間にわたって積極的に開発されてきたという事実により、JavaScriptプログラミング言語はサーバー言語の機能をますます獲得しています。 ただし、データサイエンスと機械学習の分野でのその能力は今日では控えめです（それでも、brain.jsとsynaptic.jsを忘れないでください！）。 JavaScriptの欠点は次のとおりです。



  • 彼がゲームに参加するには早すぎます（Node.jsはわずか8歳です！）...
  • Node.jsプラットフォームは非常に高速ですが、JavaScriptを積極的に批判する人は常に存在します。


Node.jsの否定できない利点には、非同期I / O、人気の高まり、JavaScriptでコンパイルする言語が多数あるという事実があります。 そのため、近い将来、ETLを使用してリアルタイムで処理できる可能性のあるデータサイエンスの分野で役立つ有用なフレームワークを目にする可能性が十分にあります。 別の質問：その時に関連するのか...



私たちの判断-JavaScriptをデータサイエンスの分野で働くための適切な言語と見なすために、まだやるべきことがたくさんあります。



Perl



Perlは、汎用スクリプト言語としての汎用性から「プログラミング言語のスイスアーミーナイフ」として知られています。 Pythonと多くの共通点があり、動的に型指定されたスクリプト言語です。 しかし、彼はまだPythonがデータサイエンスの分野で持っている人気からはほど遠い。



バイオインフォマティクスなどの定量分析法が使用される分野での用途を考えると、これは少し驚くべきことです。 データサイエンスに関しては、Perlにはいくつかの欠点があります。Perlはこの分野ですぐに普及することはできず、構文は使いにくいと見なされます。 さらに、開発者の側からは、データサイエンスの分野で使用できるライブラリを作成する試みはありません。 そして、私たちが知っているように、多くの場合、すべてが適切なタイミングで適切なアクションによって決定されます。



私たちの評決は有用な汎用スクリプト言語ですが、その助けを借りれば、確かにデータの専門家としての仕事を得ることはありません...



ルビー



Rubyは別の動的に型付けされた、解釈された汎用言語です。 ただし、Pythonの場合のように、その作成者はデータサイエンスの分野での作業に適したものにすることを望んでいないようです。



これは奇妙に思えるかもしれませんが、上記のすべては、何らかの形で科学研究の分野でのPythonの支配的な地位と、この言語で書いている人々の肯定的なレビューに関連しています。 Pythonを選択する人が増えると、より多くのモジュールとフレームワークが開発され、Pythonを好むプログラマーが増えます。 SciRubyプロジェクトは、科学的コンピューティングの機能（行列代数など）をRubyに実装するために作成されました。 しかし、これらすべての努力にもかかわらず、Pythonは現時点でまだリーダーです。



私たちの評決はデータサイエンスにとって正確な選択ではありませんが、Rubyの知識が履歴書を傷つけることはありません。



おわりに



さて、ここにあり、データサイエンスの分野に最も近いプログラミング言語の短いガイドをレビューしました。 ここで重要な点は、あなたがもっと必要とするもの、つまり言語の特異性または普遍性、その利便性または有効性の理解です。



私の現在の仕事は主に既存のデータパイプラインとETLプロセスの開発に集中しているため、私は定期的にR、Python、およびSQLを使用しています。 これらの言語は、この作業の一般性と効率性の適切なバランスと、必要に応じてより高度なR統計パッケージを使用する機能を組み合わせています。
