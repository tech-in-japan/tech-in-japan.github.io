Pocketbook 611 E-Book Recovery

以前のエピソードの簡単なプロット：/「昨日見ました」彼らはE-Ink、モデル611 <....>で死んだPocketbookの電子書籍を残しました「このデッド状態はアップデートの失敗の結果ですか、それともデバイスのコンポーネントに問題がありますか」



そして今、新しいシリーズ：







フライパン、はんだごて、SKF、SD <-microSDアダプター、およびPocketbookプログラマー（彼を尊重）を使用してPocketbook 611 Obreeyを復元する方法-カットの下。



以下のステップの前の条件：







前の投稿から次のように、出版直後に、PocketbookのプログラマーであるMikhail Boykoが私に連絡しました（私自身のイニシアチブで！）。 マイケルはデバイスの復元に協力し、すぐに6つの重要な推奨事項を提示しました。これを使用して最初の結果が得られました。



1.可能であれば、バッテリーの電圧を確認します。 充電すると、4.0ボルト以上になるはずです。 3.6が悪い場合、3.2以下の場合は非常に悪いです。

良いヒント。 何もなかったことを確認するのは残念でした。 しかし、結局、バッテリーは何の関係もありませんでした



2.バッテリーを取り外してみてください。 数分待って、USBケーブルとバッテリーが切断されていることを確認し、電源コネクタの2本のリード線を金属物で短絡させます。 テスターがある場合は、残留電圧がないことを確認してください。 バッテリーを接続して、本の電源を入れてみてください。
残念ながら、バッテリーコネクターからパッドを外した段階で、後者はパッドとともにボードから落ちました...私は25Wの針はんだごてを手に入れて、はんだ付けし直さなければなりませんでした。 ただし、コネクタをはんだ付けする前に展開された本の主な闘争は、 彼女はネットワーク電源のみに接続され、美しく振る舞いました。 つまり バッテリー（その有無）は、Pocketbook 611のパフォーマンスに影響しません。 貴重な観察が行われました-バッテリーコネクタは、多数の切断接続用に設計されていません







3.本はどのくらい正確にオンになりませんか？ 緑色のライトは点灯しますか？ インジケータの両方が点灯しますか？ どのファームウェアがインストールされましたか？ この前にどのファームウェアがインストールされましたか？ 「非公式」ファームウェアとはどういう意味ですか？ 彼女はどこで手に入れましたか？ デバイスのシリアル番号は何ですか？
私はミハイルに知っていたすべての情報を説明し、更新された更新プロセスのスクリーン写真を補足しました



4.メモリイメージを作成します（Linuxマシンにアクセスできると仮定します）

〜＃dd if = / dev / $ YOUR_SD_CARD of = sd_card_dump && sync && sync

パーティションではなく、デバイスへのパスを必ず指定してください。 / dev / sde（数字なし）のようなものがあるはずです

信頼性のために、メモリ領域全体が保存されていることを確認する価値があります。次の方法で確認できます。

（私の場合、カードは/ dev / sdeとして定義されています）

mike @ mike-P55A-UD4：/ tmp $ cat / sys / block / sde / size

3862528

mike @ mike-P55A-UD4：/ tmp $ expr $（cat / sys / block / sde / size）\ * 512

1977614336

結果のファイルは、1977614336の番号と正確に等しくなります（私の場合）
私の場合（Fedora 17）、ddコマンドはsuを介して実行され、デバイスはmmcblk0（アダプターを介してラップトップカードリーダーに接続されたmicroSDカード）でした。 より詳細には、端末のすべてのアクションが一般的な回復ログに反映されます



5.可能であれば、カードの画像の最初の16メガバイトを送信してください。デバイスに書き込みを試み、そこで何が起こったかを確認します。

＃dd if = / dev / $ YOUR_SD_CARD of = sd_dump_16M bs = 1M count = 16 && sync && sync
このアイテムも問題を提起しませんでした;画像を含むアーカイブはすぐに研究のために送られました



6.デバイスからメモリイメージを取得し、（バックアップを作成した後）カードに書き込み、更新ボタンを押したままデバイスの電源を入れます。 画面は生命の兆候を示す必要があります。 この場合は、サイトから最新の公式ファームウェアをインストールしてみてください。

dl.dropbox.com/u/2632562/sd_dump.tar.bz2をダウンロードするためのリンクはこちらです

カードに画像を書き込むとき、このコマンドを使用します

＃dd if = sd_dump of = / dev / $ YOUR_SD_CARD bs = 512 seek = 2 && sync && sync

最初の2ブロックをスキップするように指定してください。パーティションテーブルとシリアル番号があり、保存する必要があります
内蔵メモリーカードに画像を記録し、それを本に戻した後、最も興味深いことが始まりました-それは生き返りました！ 「あるべき」、ジョイスティックの中央ボタンのバックライトが電源ボタンを最初に押すと点滅し、公式のPocketbook Webサイトから取得したバージョン15.8.0への更新が正常に機能しました



更新するのに3分もかかりませんでした。私の目の前にあるのがこの本のメインメニューです。











結論を出すためにすべての機能をテストするのに約30分かかりました-本にはまだ問題がありました。WiFi（更新、接続チェック）を使用しようとすると、さまざまな特殊効果でハングします。 最後まで描画されていない接続確立ウィンドウ、黒い画面、黒いウィンドウ...



しかし、次のミハイルへの手紙の後、この行動の理由が明らかになり、元の問題の理由が発見されました。

あなたが送ったカードの画像を見ました。 ユーザーは、613（611 plus）から611にファームウェアをインストールしました。プラットフォーム613と611はハードウェアが非常に異なりますが、外観はほとんど同じです。 613からのファームウェアがインストールされたファームウェア611、何らかの理由で613からのブートローダーがキャッチに気付かず、611からのハードウェアで動作できない611にインストールされました。送信したカードのイメージを変更し、デバイスに書き込む必要があります。 私が以前に送った画像（私は理解しているように、あなたはそれでなんとか始めた）、おそらく、あなたの本のコピーには全く適合しないでしょう。 波形はそこで異なり（画面のバッチごとに固有です）、おそらく、ハードウェアへの他のバインディングがいくつかあります。 1時間以内にカード画像を準備し、メールで送信します。



画像しか待つことができませんでした（この休憩中に、投稿のタイトル写真のインベントリを使用してバッテリーコネクタを復元しました）。



添付されているのは、カードの画像が変更されたアーカイブです。

1.チームでカードに画像を書き込む

＃dd if = sd_dump_16M_modif of = / dev / $ SD_CARD && sync && sync

2.カードをデバイスに挿入します

3.更新ボタンを押しながらデバイスの電源を入れます



すべてがうまくいった場合、画面は黒で2回更新され、上部に「ファームウェアの更新」が表示されます。



更新時には、使用可能な任意のファームウェアを使用できますが、できるだけ新しいものを使用することをお勧めします。




これは勝利への最後のステップのように思えますが、そうではありません！ この段階の後、microSDカードが恥知らずにグリッチを起こし始めました...最後まで書き留められていない更新の画像、更新しようとしたときのファイル読み取りエラー...これは、デバイスのインターフェースを介して内部メモリと外部SDカードをフォーマットすることによってのみ停止できました。 その後のみ、更新が成功しました。 一般に、問題は外付けSDカード自体にあるという感覚があります（Apacer 1Gb、キットに含まれているか、別途購入することはできません）



そして、公式ファームウェア15.8.0を再フォーマットして書き換えた後にのみ、更新は成功し、WiFiが開始されました
