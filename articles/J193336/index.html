オープンデータ：バックグラウンド



オープンデータにはいくつかの定義があります。 ウィキペディアで提供されているものがあります：私自身はロシア語の記事をもたらすために英語から翻訳しました。 政府のウェブサイトには、法律で定められている定義があります。 さらにいくつかの定義がありますが、ポイントは次のとおりです。 オープンデータとは、それを所有する組織（オーソリティ、オープンステートデータの場合）によって公開される情報であり、自由形式（つまり、負担のない無料ライセンス）および繰り返し自動処理に適した機械可読形式で提供されます。 データをオープンとして定義する基準がいくつかあります。 クリエイティブコモンズライセンスは、オープンデータのほぼ前提条件です。



原則として、オープンデータは新しい現象ではなく、さまざまな形で長い間存在しており、オープン性のイデオロギーは長年にわたって存在しています。 オープンソースと無料のライセンスは、5年前ではなく、10年前ではなく、はるか以前に登場しました。 特にその研究を行う科学界では、その結果のために、あらゆる方法でそれらを検証、検証、公開、および操作する能力が重要です。 研究は、原則として、排他的に-今私たちが呼ぶもの-機械可読な特別な形式です。



今日、世界中の先進国はさまざまな方法で開放性を求めています。 今年の6月、英国で開催されたG8会議で、ホストはOpen Data Charterへの署名を提案しました。 また、ロシアによって署名されました。 憲章に記載されている主な原則は、デフォルトデータの公開性、機械可読形式でのタイムリーな公開、透明性、および開発者がオープンデータに基づいてアプリケーションを作成する条件を確保する義務です。



「膨大な量のデータを蓄積することにより、当局や企業は、それらを簡単に見つけ、使用し、理解できるように、常にそれらを共有するとは限りません。 これらは逃した機会です。 私たちは、新しい時代の前兆となる転換点に来ました。 人々はオープンデータを使用してアイデアを生成し、世界をより良い場所にするサービスを作成できるようになります」と憲章は述べています。



現在、すべてのG8諸国は、犯罪率、企業の登記、土地取引に関する情報を開示する用意があることを宣言する必要があります。 もちろん、このビジネスのリーダーは、長年にわたってこれを行ってきた英国と米国です。 しかし今、ロシアを含む世界の多くの国々がオープンデータの公開を開始しています。



これは、あらゆる種類のテクノロジー企業の成長、およびデータの価値の成長、知識経済の成長、ビジネス全体が情報の自由に基づいているYandexなどの大企業の出現に影響されました。 インターネット上のすべてのサイトが有料であり、データを集約することが不可能な場合、オープンデータの問題は発生しない可能性があります。 パブリックドメイン情報の操作は、これに大きな影響を与えています。

オープンデータステータス



通常、データは3つの条件付き形式で存在します。



最初の。 手頃な価格で仕事に適しています。 つまり、州またはその所有者がマシンの可読性を確保します。 ここで入場のしきい値は最小限です-それらをいくつかのカードに入れて携帯電話に適用できます。 すべてがすぐに準備できます。



二番目。 状況はさらに悪くなります。原則として情報はありますが、さまざまなサイトから抽出する必要があります。 たとえば、州下院議員に関する情報は州下院のウェブサイトにありますが、ウェブページの形で抽出する必要があります。



地域別のモスクワ市の水質に関する情報は、モスボドカナルのウェブサイトにあります。 しかし、最初に通りに入り、次に家番号を入力しなければならない特別なサービスを通じて、その後、面積、汚染レベル、さまざまな指標の汚染レベルが与えられます。



このすべての情報を収集するために、活動家はさまざまなスクラップブックを作成します。これは、Webサイトから情報を削除し、それをいくつかのデータベースに変換するプログラムです。



三番目。 何らかの形の情報は原則として存在しますが、公共の場では利用できません。 一般に、私たちが行うことは、情報の透明性を達成することです。 私は今、自分自身だけでなく、ロシアで積極的にこれに従事している他の多くの活動家（商業会社を含む）について話し、情報の公開性を達成しようとしています：



  1. そのため、すでに機械可読形式で公開されているデータが作業に適していて便利であるため、エラーが最小限に抑えられます。
  2. そのため、現在機械可読ではないデータが変換されます。 それらが公開されている場合、それらを便利にしましょう。これが最も重要なことです。
  3. そのため、現在公開されていないものは公共スペースに表示されます。




このため、いわゆるOpen Data Councilが Open Governmentに登場しました。 州は、これに参加する準備ができていると述べており、いくつかの変更が法律、規制に加えられています。 原則として、データのオープン性を確保するために作業を開始し、それらを使用するために、制限はありません。

重要なタスクの1つは、オープンデータへの関心を高めることです。 このために、 Apps4Russiaが作成されました。これは、開発者にとって長く継続的な競争であり、州がこのトピックに興味を持つ前に行ったものです。 2011年には、7人が賞金プールを構成するために自分のお金を集め、最初のコンテストを開催しました。 彼に続いて、 非営利のパートナーシップ「情報文化」を作成し、今度は3回目のコンテストを開催しています。 その主なタスクは、開発者がオープンデータに目を向けるように動機付け、プロジェクトに使用できること、また使用すべきであることを理解させることです。



Apps4Russiaは、ソーシャルカードという素晴らしいプロジェクトに参加しました。 これは、携帯電話の座標によって、どの州の機関が近くにあるかを判断し、すぐに携帯電話を持ってきたアプリケーションです：DEZ、政府、警察署など。 これは、さまざまなサイトから収集され体系化されたオープンデータです。 最近、私たちは警察のデータに基づいて小さな競争を開催しました。 その枠組みの中で、地区役員を見つけるのに役立ついくつかのアプリケーションが登場しました。



今年のApps4Russiaでは、その技術で作成されたアプリケーションが競合するYandexノミネートがあります。 非常に具体的なアイデアがあります。Yandexは、オープンデータにも取り組んでいるサービス会社であり、開発者が製品の品質を向上させる多くの機会を生み出しています。 Yandex.Mapsで獲得したプロジェクトの数を測定することは困難ですが、非常に多くのプロジェクトの食品品質が確実に向上しました。 Yandex.Mapsだけでなく、Yandex.Search API 、 他のサービスのAPIも使用できます 。







一般的に受け入れられているAPIに加えて、Yandexには、言語を自由な形式で処理するために特別に設計されたテクノロジーもあります。 たとえば、しばらく前に、このために特別に設計された富田パーサーがオープンになりました。 Yandex.Newsなど、テキストの意味を理解するのに役立つのは彼です。



また、検索と病院のレジストリの助けを借りて、病院の検索エンジンを作成できます。 または、すべての検察官のサイトからデータを収集し、RSSにニュースを追加して、検察官または検察官に関心のある人向けのモバイルアプリケーションを作成します。 そしてそれを検察官自身に売ってください。



各データ配列から小さな断片を取り出して、何らかの方法で使用できます。 組織のレジストリにWebアドレスがある場合、ロボットを再起動してRSSフィードを収集し、モバイルアプリケーション「モスクワ市の最新ニュース」を作成できます。モスクワのすべての部門にRSSフィードがあります。 これはすべてYandexテクノロジーで実行できます。api.yandex.ruにアクセスするだけです。 今年、Apps4Russiaのアプリケーションは9月16日に終了しますが、拡張する可能性があります。



Yandexオープンデータハッカソン



9月14〜15日、モスクワで最初のYandex Hackathonが開催されます。 2日間2晩、開発者はオープンな政府データとYandexテクノロジーに基づいてアプリケーションを作成します。 5人までのチームでも参加できます。 また、既成のチームとして参加することも、その場で組織することもできます。



競技会で長い間考え、その後2時間で彼のために何かをすることができるなら、ハッカソンで速く考える必要があります。 原則として、あなたはそれに備えて来なければなりません。 したがって、何をするかを事前に考え、情報を探す場所を理解し、APIを学習してください。 もちろん、彼らはその場であなたを助けます：オープンデータとYandexテクノロジーに関するコンサルタントがいます。



私はもう一度あなたが販売する製品をすぐに作る必要はないことを強調したいと思います。 別の製品の一部にすることができます。 プロジェクトの特定の部分を定性的に実装しているという事実のために、あなたは自分自身を売ることができます。 そして、必ずしも雇用者ではありません-それは評判のための単なる仕事です。 ハッカソンでは、いくつかの情報とツールに基づいてクールなものを作成できることを示すことができます。



Apps4RussiaとYandex Hackathonの両方の主なタスクは、有用なものを作成できる多くの情報と技術があることを示すことです。
