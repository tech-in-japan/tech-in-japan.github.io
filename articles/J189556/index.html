Mac OS X用のBashkirレイアウトの作成

Bashkirなど、あまり一般的でない言語で記述する場合、オペレーティングシステムまたはデバイスを変更することは常に困難です。 そのため、人々はまだWindows XPに座っており、カスタムレイアウトはレジストリに曲がって書かれており、同様にBashkirフォントも書かれています。 結局のところ、曲がっていても動作します。 何年もの間、私はWindowsに座っていて、どういうわけか問題の存在さえ覚えていませんでしたが、Macに再シードすると、OS XにはデフォルトでBashkirレイアウトがないことがわかりました。 私はインターネットで習慣を探しに行きましたが、そこでも失望しました。 どうしてそんなに長い年月が過ぎたように思えますか？ 最終的には、かつてAndroidを使用していたので、私はすべてを自分で行うことにしました。



実際、Bashkirsに問題があるだけでなく、さらに多くの国では、新しいOSにレイアウトがなかったり、リリースが遅れたりしています。 ですから、私と同じように熱心な人たちを助けるために、作成プロセスを説明しようと思います。











バシキール文字



次に説明する内容をよりよく理解するために、まずBashkirのアルファベットを見てください。 全部で42文字あり、そのうち33文字はロシア語のアルファベットの文字+ 9文字です。 私の意見では、これは非常に多く、特に一部の手紙はロシア語からの借用のみに使用されるためです。 このような冗長性により、キーボードに快適にフィットすることはできません。 一般に、スターリン主義の影響を受けた旧ソ連の領土の多くの人々は同じ問題を抱えています。





レイアウトの作成に使用される原則



始める前に、疑いのある瞬間に構築して適用できる単純な原則を策定する必要があります。 私の場合、元の原則の一部は根を下ろさず、現実に直面して落ちたので、それらについて以下で説明します。 しかし、残ったもの：

  1. ほとんどの人は、Vistaで最初にリリースされたWindowsの標準レイアウト（これ以降、どこでもPCレイアウトと呼びます）に慣れています。 すべてが完璧であるとは限りません。最も快適ではありませんが、すでに定着しています。 したがって、作業ツールをできるだけ早く作成するために、私はそこから始めました。 そして、言語での文字の実際の使用頻度を考慮した元のレイアウトは、近い将来の課題です。

  2. 追加の（活版印刷）文字と英語レイアウトとの互換性。 物理キーボードで入力できる内容を見ると、非常に便利です。 ロシア語のレイアウトでは、多くのキャラクターが残りの原則に従って位置を見つけました（そして、それらの数はより少なくなります）。したがって、それに焦点を合わせないようにします（これは追加のキャラクターにのみ適用されます）。
  3. テンキーはオプション（Alt）の下にあります。 はい、一見、これは非常に疑わしい決定です。 しかし、文字はより頻繁に使用されるため、他の選択肢はありません。
  4. Bashkirレイアウトを使用すると、切り替えなくてもロシア語でテキストを入力できることが重要です。




設計



エディターでの作業を開始する前に、キーボードが存在する可能性のあるすべての状態をリストして描画する必要があります。 これらの状態は、Command（cmd）、Shift（⇧）、Control（ctrl）、Option（alt）、Caps Lockなどの修飾子のキーストロークに依存します。 Bashkirキーボードの場合、次の状態になりました。



  1. Bashkirアルファベット小文字+基本的な句読点-これはデフォルトの標準状態です。
  2. バシキール語の大文字+基本的な句読点-Caps Lock;
  3. アルファベットの大文字+基本的な句読点2- シフト。
  4. 英語qwerty小文字（コマンドとホットキー用、英語レイアウトと同一） -ommand;
  5. 英語のQWERTY大文字（英語のレイアウトと同じ） -Command + Shift;
  6. 追加の文字+数字- オプション。
  7. 追加文字2- Option + Shift;
  8. 追加の文字（英語のレイアウトと同じ）- コントロール。


正直なところ、仕事の過程で私は絶えず実験し、条件を変更、追加、削除しました。 たとえば、コマンドを押した状態でのみ数字を残そうとしましたが、テスト中は機能しませんでした。テキストエディターはテキストを入力せず、ユーザーからのコマンドを待ちました。 最終的に、シンプルで使い慣れたソリューションが勝ちました。



エディターで作業する



レイアウトを作成するために、 SILコミュニティによってリリースされた無料のUkeleleプログラムを使用しました。これは、たとえば優れた無料のフォントや言語ソフトウェアの開発で知られています。



ファイルは、YTsUKENの既存のロシア語レイアウトに基づいていました。 これを行うには2つの方法があります。まず-OS入力ソースとしてロシア語を選択し、プログラムで[ファイル ] -[現在の入力ソースから新規作成 ]（つまり、現在の入力ソースから新規）を開きます。 次に、 System Keybords / Cyrillic / Russian.keylayoutプログラムに付属のレイアウトファイルを開きます。



すべての作業はビジュアルエディターで行われ、簡単で便利です。 キーに文字を割り当てるには、仮想キーボードボタンをダブルクリックします。 修飾子をオンにしてレイアウト状態を編集するには、標準状態の代わりに修飾子キーを押すだけで、修飾子に対応する状態が表示されます。 常にキーを押したままにしないために、 メニュー-表示-スティッキー修飾子でスティッキングをオンにします。



作成されたすべての状態の概要は、command + shift + mを使用して開くことができます（上のスクリーンショットでは、番号は最初に設定した番号と一致しません）。 下部には、どの状態が標準であるかが個別に示されています（デフォルト設定）。 表では、各状態にどの修飾子が使用されているかがわかります。







自分で状態を作成する必要がある場合があります。 これを行うには、「+」をクリックして、使用する修飾子を選択します。 Shift、Option、およびControlキーについては、右または左のキーを個別に割り当てることができます。 私の条件は、ロシアのレイアウトの条件に似ていることが判明しました。







その後、使用する文字セットを選択する必要があります。

  • [ 空]を選択すると、文字のないきれいなレイアウトが開きます。
  • Standart、英語のQWERTYなど、いくつかの共通セットを開くことができます。
  • 既存のセットを選択する必要がある場合は、[ 既存のマップのコピー]を選択します。


Appleは、キーのレイアウトと標準が異なる12種類以上のキーボードをリリースしています。 したがって、常にレイアウトをテストする必要があります。 すべてのキーボードのリストがメニュー-表示-キーボードタイプで開きます。







ANSIタイプとISOタイプの違いに特に注意してください。2番目のタイプにはより多くのキーがあります。 たとえば、最近のApple Wireless ISOキーボードでは、「>」キーの代わりに「>」キーが数字の隣にあり、ユーザーがANSIキーボードを持っていると、letterという文字が突然消えてしまうという問題がありました。 さらに、それを「»」キーに転送すると、ISOに番号の列ではなく、4行目のIの隣になります。 その結果、両方のキーに文字theを残さなければなりませんでした。







レイアウトの中間バージョンの1つで、タワーでよく使用されている別のレイアウトとの互換性を追加するというアイデアが浮かびました。 しかし、テスト中に、私はそれが不便であることに気付きました。







最終的なレイアウトレイアウト



Macの物理キーボードの構成の違いにより、PCレイアウトとまったく同じBashkirアルファベットの追加文字を追加して最上行を保存することはできませんでした。 たとえば、「Ң」は間違いなく一般的に使用される記号ではないため、「Ң」は「いいえ」の場所に移動しました。 最初のレイアウトの作成者に、彼がどうやってそこにたどり着いたかを尋ねたいと思います。 コロンの代わりにコンマが使用されました。コンマは非常に頻繁に必要であり、物理キーボードにマークされているためです。 Shiftキーを押すと、セミコロンがセミコロンの代わりになり、ドットがコロンになり、覚えやすくなります。







OptionおよびOption + Shiftが押された状態で、タグを最も長く「再生」しました。 しかし、それらはいくつかのルールを理解するのに役立ちました。 Optionでは、上段のすべての数字と文字（物理キーボードに表示される活版印刷文字の一部）が英語レイアウトの標準状態から移動し、Option + ShiftはShiftで同じ状態から移動しました。 ニーモニックルールを可能な限り使用しようとしました。たとえば、数字はNキーにあり、キーのユーロ記号はラテンEのものです。







Commandを押したときの状態、およびCommand + Shiftは、予定どおり、Shift + Control-English Controlの標準英語レイアウトに完全に対応しています。 これは、キーボードとキーボードショートカットがシステムとアプリケーションで機能するために必要です。







アイコンを追加する



まず、グラフィックエディターの言語に対応するフラグの小さなコピー、16x12 px（またはフラグに描かれているものに応じて16x11 px）を作成し、PNG形式で保存する必要があります。







fasticmsプログラムを使用して、 これらすべてをMac .icnsアイコンのネイティブ形式に変換できます。 プログラムを起動すると、画面にキューブが表示され、pngファイルをドラッグして、開いたウィンドウにicnsとして保存します。







アイコンをレイアウトに添付するには、 メニュー-キーボード-アイコンファイルを添付に進みます。



詳細オプションの設定



レイアウトを公開する前に、さらにいくつかの手順を実行します。



システムに表示されるレイアウトの名前を追加する必要があります（メニュー—キーボード—キーボード名の設定）。 名前の例：ArmenianWesternQWERTY、セルビア語ラテン語、トルコ語QWERTY-PC。 Bashkir-PCのバージョンに名前を付けました。



次に、言語、スクリプト、地域、およびスクリプトオプションを設定する必要があります（メニュー—キーボード—キーボード名の設定）。 Bashkir言語では、キリル文字が使用されます。







レイアウトのバージョンを管理することを忘れないでください（メニュー—キーボード—バージョン情報の設定）。



さて、これでファイルを保存して使用できるようになりました。 keylayout形式のメインレイアウトファイルと共に、同じ名前のicnsのアイコンファイルがその横になければならないことに注意してください。







作成したBashkirレイアウトをダウンロードしてインストールできます 。

インストール手順はプロジェクトページにあります。
