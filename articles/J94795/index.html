プロジェクトの収益化：8 +何も獲得しない4つの方法

これは、明らかに、ロシアでのソフトウェアビジネスの構築に関する「ゼロから」の3番目の記事（ 1、2 ）です。 過去に、彼らは私たちの収益化モデルについて話をするように頼みました-私はあなたに言います（重点はモデル自体にあり、一般的なソフトウェア販売チャネルの構築にはありません）



要するに、収益化モデルの選択（私の個人的な強い信念による）は、プロジェクト/ビジネスごとに個別です。 そして、モデル自体は、時間とともに変化し、市場での新しいトレンド/パラダイムの出現とともに変化するはずです。



しかし、すべて順番に。



ゼロアプローチ：フリーウェア



年代順に見ると、メイン製品であるWEBO Site SpeedUpは、オープンソースとしてだけでなく、無料のソリューション（ Web Optimizer ）としても始まりました。 時間が経っても、オープンソースは消えませんでしたが、製品の有料版が登場しました。 なぜそう



それは根拠のある決定でした。 2009年4月から5月にかけて、最初の製品インターフェイスを作成するための資金を調達するための小規模な実験が行われました。 私たちは収集しました（一見適切なソリューションのように思えますが、もちろん、私たちが望むほど便利ではありませんでしたが、そのタスクを解決しました） 2,700ルーブル （月あたり）



これが十分ではないと思われる場合（そのような必要な製品の場合）、比較のために、クライアントの最適化に関する書籍の料金の数値を示します（もちろん、料金はどこにも昇格しませんでしたが、約2万から3万人が書籍をダウンロードしました集約）。 最初の本には約700ルーブル、2番目には1200ルーブルが来ました。



あなたがよく理解しているように、誰もお金のために真に高品質のソリューションを作成しません。 そのため、2009年の夏に、他の収益化方法の選択について最終決定が行われました（無料のオープンソースソリューションの支持者としてどれだけ悲しくても）。



収益化： 寄付 。

効果： ゼロに近い 。



最初のズーム：サブスクリプション



秋に、WEBO Softwareでビジネスプランが作成されました。このプランでは、ページビューの数に応じて、サブスクリプションで製品を配布することが計画されました（実際、もたらされる利益に直接比例します）。 つまり、たとえば、1か月あたり最大5000ページビュー-無料、5000から10000まで-1か月あたり10ルーブルなど。



課金の実装とこのスキームのサポートは技術的に複雑であるため、ほとんど即座に破棄されました（サイトビューの数を正しく計算する必要がありますが、最大のパフォーマンスを保証しますが、互いに多少異なります）。ターゲットユーザーに加えて、製品の支払いスキームは完全に廃止されました明らか：なぜ隣人は自分のサイトに100ルーブルを支払うのに、私は500ルーブルとほぼ同じ額を支払うのか



このスキームはやや簡素化され、年間サブスクリプション（製品のプレミアムバージョン）が年間2,000ルーブル（$ 100）の価格で登場しました。 原則として、あなたが1年間ストレッチする場合、量は非常に少なく（月あたり167ルーブル）、ほとんどのサイトでは費用を支払います。



収益化： 製品の年間サブスクリプション （a-laアンチウイルス）。

効果： 比較的良い 。



1.5近似：製品ライン



しかし、結局のところ、誰もが製品のフルバージョンに対してそのようなお金でさえ喜んで支払うわけではありませんでした。 そのため、プレミアムバージョンのリリース直後、Liteが登場しました。Liteには年間500ルーブル（$ 25）かかります（ バージョンの完全な比較 ）。 これは、同様のすべてのソリューションよりもはるかに優れており、単純なWebサイトアクセラレーションに最適です（現在、ベースアクセラレーションと呼ばれています）。



収益化： 製品ライン （異なる機能を備えたいくつかのバージョン）。

効果： 良い 。



近似2：製品のインストールとセットアップ



当初、製品自体は、サイトを高速化するためのサービスの拒否として考えられていました。 つまり アイデアは「ボタンをクリックする-サイトが加速する」ことでした。 アイデアを実装する過程で、私たちは膨大な数の困難に直面しました（たとえば、技術的なものから心理的なものに至るまで、私はしばしば信じられないような質問に遭遇します：どのように機能するのか、それは空からの別のチャージですか？）。 私たちはそれらのほとんどを克服することができましたが、ファイナルはまだ遠いです。 Webの絶え間ない発展とその複雑さを考慮に入れて、この非常にフィナーレに到達するための多くのアイデアがあります（はい）。今年末までにそれらを実装します。



しかし、中小企業のセクターへのアクセスにより（十分な訪問済みサイトや収益性の高いサイトを高速化する必要がある場合）、サイトを高速化する専門家がほとんどいないことが判明しました。 そのため、ソリューションを研究し、その機能を調査する準備ができている人はいません。 「サイトを高速化する」ために「合理的な金額」を要求する方がはるかに簡単です（箱入りの製品やサービスなど、どのように加速するかはそれほど重要ではありません）。 その結果、製品をインストール/設定するための一連のサービスの提供を開始し、ある時点からテクニカルサポートを提供しました 。 これは、「生地を切る」ことを望んでいたという事実とはまったく関係ありません。 実際、高品質のフィードバックがユーザーから提供されることはめったにないため、各ユーザーに（改善するのではなく）製品がどのように機能するかを説明する準備ができていません（これを行わない会社は知りません）プロモーションの一環として）。



収益化： 関連サービス （インストール/構成/製品サポート、関連コンサルティング）。

効果： 良い 。



アプローチ3：アドウェア



アドウェア（広告を表示するための製品を使用）のアイデアは、昨年3〜4回出現しました。 製品自体は、サイト自体と管理インターフェースの2つの面で機能するため、広告には2つのオプションがありましたが、最初に、製品はサイトを高速化します（広告があると速度が低下します）。 第二に、多くの場合、広告が既にいっぱいであるか、広告を表示できない場所（企業サイトまたはオンラインストア）にインストールされます。 したがって、サイト自体に広告を表示するオプションはこれまでのところあまりありません（それが何も起こらないという事実ではありません。これを可能な限り控えめに行う方法はまだわかっていません）。



2番目のオプション：管理インターフェースに広告を表示します。 いくつかの選択肢があり、私たちは（本当に信じたい）いくつかのスレッドを感じました。 現在（製品の最新およびベータ版）、製品のアフィリエイト（アフィリエイト）広告の表示が実装されています。 ここで、いくつかの重要なポイントを強調します。



1.広告はできるだけターゲットを絞る必要があります。 テクノロジーウェブ製品でシャンプーを宣伝することはできません。 より正確には、可能ですが、そのような広告からのコンバージョンは非常に低くなります。 ここでは、最大の「視聴者を獲得する」ことに焦点を当てます。 関連するサービスや製品を宣伝できると便利です（これから行うように、最初のツバメはloadmpact.comによるサイトフォールトトレランスチェックで起動されます

2.いかなる場合でも、広告は製品の機能を妨害するものではありません。 つまり ポップアップバナー、ストレッチマーク、メディア広告はほとんどありません。 最大、コンテキスト、およびソフトバナー（つまり、タスクは反対です。広告は目立つ必要がありますが、目立つことはありません）。



これまでのところ、この分野では結果はありませんが、このような製品の使用により、少なくとも一部の機能をフリーモードに移行できる可能性は十分にあります。



収益化： 製品のコンテキスト広告 。

効果： まだ明確ではありません 。

アプローチ5：SaaS



ユーザーからの多くのリクエストとソフトウェア配布の現在の傾向（ popodpiske.ruなどのサービスの出現を含む）を分析し、コンセプトを形成しました（まだ完全には実装されていませんが、最新のベータバージョンをインストールすると、すでに「感じる」ことができます） -Webパフォーマンスとフォールトトレランスに関連するいくつかの機能のプラットフォーム。



6月に、ユーザーがバランスを取る製品の柔軟なエディションの形でこのコンセプトをリリースする予定です（ロシア連邦の法律の観点からのこの決定の設計は、別の記事のトピックです）。製品機能の最適化に費やすことができます（たとえば、1日に1ルーブルデータを使用するには：URI）。 したがって、2つの問題を一度に解決します：柔軟な課金/請求機能（誰もが同じスプライトまたはHTMLキャッシングを必要とするわけではありません）とライセンス更新（残高に資金がある限り、それらは接続された機能に比例して費やされます）。



オープンベータテストの開始は後日発表されますが、製品をダウンロードし、システムステータス->更新->ベータ情報を表示）を最新のベータバージョンに更新できます 。



収益化： サブスクリプション、SaaS、柔軟な請求 。

効果： まだ明確ではありません 。
