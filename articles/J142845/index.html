基本レッスン04-オブジェクトの破棄





このレッスンでは、Destroyコマンドを使用してシーンからオブジェクトを削除する方法について説明します。



空のシーンを作成し、球体（ GameObject-> Create Other-> Sphere ）とキューブ（ GameObject-> Create Other-> Cube ）を追加します。 キューブを「ボックス」と呼びます。 次の図に示すようにオブジェクトを配置します。





C＃スクリプトを追加し（ プロジェクトビュー->作成-> C＃スクリプト ）、 Destroyerという名前を付けます。 既に述べたように、C＃スクリプトを作成するとき、Unityは、 Start（）およびUpdate（）メソッドを使用して、接続されたライブラリとメインクラス（スクリプトで使用）で構成されるフレームワークを作成します。 基本レッスン02（入力の基本）では、 Update（）メソッドが使用されました。これは、すべてのフレームと呼ばれます。 この場合、シーンをロードした直後に実行されるStart（）メソッドを使用します。 Destroy （）関数をStart（）メソッド本体に追加し、 そこにgameObjectを渡して、スクリプトがコンポーネントであるオブジェクトを破棄する必要があることを示します。 



//  Start()  ,      Start() { //        Destroy(gameObject); }




球体にスクリプトを追加します。 これを行うにはいくつかの方法があります。 スクリプトをプロジェクトビューからシーンビューの球体にドラッグします 。





または、 Hierarchyのオブジェクトの名前。





また、コンポーネントメニュー（ コンポーネント->スクリプト->デストロイヤー ）から、または選択したオブジェクトのインスペクタービューにスクリプトをドラッグするだけで、球体を選択してスクリプトを追加することもできます。





スコープを再度選択し、コンポーネント間にスクリプトが存在することを確認してください。





[ 再生]をクリックすると、シーンを読み込んだ直後に球体が消えることがわかります。





それでは、別のオブジェクトを破壊してみましょう。 これを行うには、 GameObjectクラスの静的メソッドFindが必要です。 Start（）メソッドのコードを次のコードに置き換えます。 

 //     Box     –   Destroy(GameObject.Find(“Box”));




翻訳の作者からの注意 ：2番目のケースでは静的関数を呼び出して値を渡すため、クラス名（ G ameObject-大文字）を記述し、最初のクラスではこのクラスのオブジェクト（ g ameObject-小さい文字）。 そうでない場合、コンパイラーはエラーをスローします。



error CS0176: Static member `UnityEngine.GameObject.Find(string)' cannot be accessed with an instance reference, qualify it with a type name instead







次のように翻訳されます：

「UnityEngine.GameObject.Findの静的メンバー（文字列）は、インスタンス（クラス）への参照によってアクセスできず、代わりに型名で定義（呼び出し）します。」



スクリプトコードへの変更を保存します。 スクリプトを球体から削除してキューブに追加する必要はありません。 どのオブジェクトにアタッチされていても、スクリプトはBoxという名前の（ステージ上の）オブジェクトを検索します。 Playを押して、球がシーンにどのように残り、立方体が消えるかを確認します。





オブジェクトをすぐにではなく、しばらくしてから破棄する必要がある場合はどうなりますか？ これは、値をDestroy関数の2番目のパラメーターに渡すことで実行できます。 

 //   Box  3     Destroy(GameObject.Find(“Box”), 3);




[ 再生]を押して、シーン全体が読み込まれた3秒後にキューブが消えることを確認します。

基本レッスン05-オブジェクト作成の実装





このレッスンでは、プレハブとインスタンス化コマンド（インスタンス化）を使用して、シーン内にオブジェクトをリアルタイム（実行時）に作成する方法を説明します。



シーンが既にロードされているときにオブジェクトをシーンに追加する場合は、スクリプトを使用するか、 Instantiate（）コマンドを使用する必要があります。



ベースレッスン03（プレハブ）からシーンをダウンロードするか、同じ新しいシーンを作成します。 シーンにはプレハブBouncyBoxがあります。





[ 再生]をクリックして、キューブがまだ落下し、サーフェスから押し出されていることを確認します。

次に、シーンからBouncyBoxインスタンスを削除します。





その後、空のオブジェクトを追加します（WindowsではGameObject-> Create EmptyまたはCtrl + Shift + N 、MacOSではCmd + Shift + Nを思い出してください）。 BouncyBoxインスタンスがあった場所と同じ場所に配置します 。





C＃スクリプトを作成し、 Createrを呼び出します。 スクリプトの編集を始めましょう。 GameObject型のオープン（ パブリック ）変数を取得して、 thePrefabと呼びましょう。 たとえば、 Inspector Viewを介して変数の値を渡す場合は、 public修飾子を指定する必要があります 。 その後、 Start（）関数の本体で、別のGameObject （ instanceという名前）を作成し、静的なInstantiate（）関数を使用して値で初期化します。 



 public GameObject thePrefab; void Start () { GameObject instance = new GameObject(); instance = Instantiate(thePrefab, transform.position, transform.rotation) as GameObject; }




Instantiate（）メソッドの詳細を検討してください。 

 Instantiate(GameObject original, Vector3 position, Quaternion rotation)


このメソッドは、指定された位置ベクトルと回転クォータニオンで元のオブジェクトを複製します。

タイプVector3は、通常の3コンポーネントベクトルです（ R 3のベクトルに似ています）。

クォータニオンタイプは、オブジェクトの回転を定義するクォータニオンです。



スクリプトを空のオブジェクト（ GameObjectという名前）に追加します。thePrefabは public修飾子で宣言されているため、 Inspectorビューで初期値を直接設定できることを思い出します （対応するコンポーネント、つまりこの場合、これはGameObjectの Creatorです ）。 プレハブを値が示されている場所にドラッグします（または、右側の円をクリックしてリストから選択します）。





Playを押して、ジャンプキューブがシーンに表示されていることを確認します。





しかし、キューブには初期の回転角がないため、単純に上下にジャンプします。 GameObjectを選択し、わずかな角度で回転させます。



Playをクリックすると、キューブが落下し、さまざまな角度で反発されることがわかります。

基本レッスン06-シンプルタイマー





このレッスンでは、ユニットを使用してスクリプトを使用する方法について説明します

Time.deltaTimeとfloat型の変数を使用して単純なタイマーを作成します。





前のレッスンのシーンを使用します（空のゲームオブジェクト、

プレハブインスタンスの生成、つまり BouncyBox ）。





C＃スクリプトを作成し、それをTimerと呼びます。 変数myTimerを追加し、次のコードを記述します。 



 public float myTimer = 5.0f; void Update() { //Time.delaTime -  (),       (read only) myTimer -= Time.deltaTime; if (myTimer <= 0.0f) { Debug.Log("GAME OVER"); } }




スクリプトを保存し、Unityに切り替えます。 GameObjectにスクリプトを追加します 。





思い出させてください myTimerは publicとして宣言されているため、 インスペクタービューで初期値を変更できます。



再生をクリックします 。 myTimerの値がゼロになるとすぐに、ステータスバーにGAME OVERという行が表示されます。 myTimer変数の値はドロップし続けます（これはInspector Viewで確認できます ）。





myTimer変数がゼロを下回らないようにするため、 Update（）関数にもう1つのブランチを追加します。その結果、次のコードを取得します。 

 void Update () { if(myTimer > 0){ myTimer -= Time.deltaTime; }else if(myTimer <= 0){ Debug.Log("GAME OVER"); } }




[ 再生]をクリックして、 myTimer変数の動作を監視します。 その値がゼロに十分に近づくと、変更が停止します。

基本レッスン07-モーションの基本





このレッスンでは、transform.Translate関数を使用してオブジェクトを移動する方法について説明します。



キューブ、カメラ、光源でシーンを作成します。





新しいC＃スクリプトを作成し、それをMoveと呼びます。

移動は時間的に連続している必要があるため、メインコードはUpdate（）メソッドに配置されます 。 変換オブジェクトのTranslate関数を呼び出し、 Vector3（0、0、1）をパラメーターとして渡します。 次のコードを取得します。 



 void Update () { transform.Translate(new Vector3(0.0f, 0.0f, 1.0f)); //      Oz }




コードでさらに詳しく調べます。 ここで、 transformは、オブジェクトにアタッチされたTransformクラスのオブジェクトです。 Translate（）メソッドは、オブジェクトをその長さに等しい距離だけベクトルに沿って移動します（並列転送）。



スクリプトを保存し、キューブに追加します。 [ 再生]をクリックして、キューブがOz軸に沿ってどのように飛んでいくかを確認します。







たとえば、ベクトルにTime.deltaTimeを掛けて、移動速度を下げましょう。 



 transform.Translate(new Vector3(0.0f, 0.0f, 1.0f) * Time.deltaTime);




[ 再生]をクリックし、キューブの動きが著しく遅くなることを確認します。



速度が固定ベクトルではなく変数によって設定されている場合、スクリプトをより便利に使用できます。 float型のパブリック変数を取得して、 speedと呼びましょう。 この変数を使用すると、 インスペクタービューで初期速度を設定できます （デフォルトでは5.0fに等しくなります ）。 



 public float speed = 5.0f; void Update () { transform.Translate(new Vector3(0.0f, 0.0f, speed) * Time.deltaTime); }




スクリプトを保存して、 Inspector Viewの MoveコンポーネントにSpeed変数が表示されていることを確認しましょう。





この変数の指定された初期値に従ってオブジェクトの速度がどのように変化するかを確認してください。 たとえば、 Speed = 3の場合 、オブジェクトの速度はそれほど速くありませんが、遅くもありません。



注 ：式Vector3（0.0f、0.0f、1.0f）には、短い（ただし、明らかに遅い場合がありますが、正しくない場合は修正してください）エントリVector3.forwardがあります。



ラッパーからのコードの一部： 

 public static Vector3 forward { get { return new Vector3(0.0f, 0.0f, 1.0f); } }

基本レッスン08-筋力運動の基本。





このレッスンでは、力の適用を使用して肉体（剛体）を移動する方法について説明します。



ステージ上にRigidbodyコンポーネントを持つオブジェクトがある場合、フォースアプリケーションを使用してオブジェクトのモーションを設定する必要があります（したがって、すべてのモーション計算を物理的なゲームエンジンに転送します）。 そうしないと、オブジェクトの物理と競合する可能性があります。 ベースレッスン03のシーンを引き続き使用しています。







C＃スクリプトを作成し、それをForceと呼びます。 Start（）メソッドで、 rigidbodyコンポーネント（オブジェクトに関連付けられている）のAddForce（）メソッドを呼び出し、 Vector3に適用された力ベクトル（0.0f、0.0f、power）を渡します。 クラスでは、名前がpowerでデフォルト値が500.0fの float型の変数を作成します。 



 public float power = 500.0f; void Start () { rigidbody.AddForce(new Vector3(0.0f, 0.0f, power)); }




スクリプトを保存し、オブジェクト（またはプレハブ）に追加します。 ここで、[ 再生 ]をクリックすると、 Oz軸に沿って適用される力の影響により、立方体がどのように倒れずに斜めになるかがわかります。
