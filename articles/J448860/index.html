Android Studio 3.4リリースノート（2019年4月）

Android Studio 3.4は、さまざまな新機能と改善が加えられたメジャーリリースです。









IntelliJ IDEA 2018.3.4



  • 2018.3.4リリースからのIntelliJ IDEAの機能強化で更新されたAndroid Studioコア


Android Gradleプラグイン3.4.0



  • Android Gradleプラグイン3.4.0の新機能については、 リリースノートを参照してください


新しいプロジェクト構造ダイアログ



  • 新しいプロジェクト構造ダイアログ（PSD）を使用すると、モジュール、ビルドオプション、署名構成、ビルド変数など、依存関係を簡単に更新し、プロジェクトのさまざまな側面を構成できます。
  • メニューで[ファイル]> [プロジェクト構造]を選択してPSDを開くことができます。 WindowsおよびLinuxではCtrl + Shift + Alt + S、またはCommand +を使用してPSDを開くこともできます。 （セミコロン）on macOS。 以下は、PSDの革新と更新の説明です。


変数



  • このセクションでは、ビルド変数を作成および管理できます。たとえば、プロジェクト全体に対して対応する依存関係のバージョンを指定します。









    • プロジェクトのgradleビルドスクリプトに既に存在するビルド変数の簡単な概要と編集
    • [プロジェクト構造]ダイアログからプロジェクトまたはモジュールレベルに新しいビルド変数を直接追加する
    • 注：既存のビルド構成ファイルが複雑なGroovyスクリプトを介して値を指定している場合、プロジェクト構造ダイアログを介してこれらの値を編集できない場合があります。 さらに、プロジェクト構造ダイアログを使用して、Kotlinで記述されたビルドファイルを編集することはできません。




モジュール



  • モジュールに存在するすべてのビルドオプションに適用されるプロパティを設定するか、モジュールセクションを使用してプロジェクトに新しいモジュールを追加します。 たとえば、ここでdefaultConfigプロパティを構成したり、署名構成を管理したりできます。


依存関係



  • 次の手順に従って、プロジェクトの同期中にGradleによって構築されたプロジェクトの依存関係グラフの各依存関係グラフを調べて表示します。







    1. PSDの左ペインで、「依存関係」を選択します
    2. [モジュール]パネルで、依存関係を表示するモジュールを選択します
    3. PSDの右側で、以下に示すように「解決された依存関係」パネルを開きます。





  • モジュールを選択し、[宣言された依存関係]セクションの[+]ボタンをクリックして、追加する依存関係の種類を選択することにより、プロジェクトにすばやく依存関係を見つけて追加することもできます。







  • 選択した依存関係のタイプに応じて、以下に示すようなダイアログが表示され、モジュールに依存関係を追加できます。











組み立てオプション



  • このPSDセクションでは、プロジェクトの各モジュールに対してビルドオプションとバリエーション（「製品フレーバー」）が作成および構成されます。 マニフェストテンプレート（「プレースホルダー」）、ProGuardファイルを追加し、署名キーなどを割り当てることができます。





ヒント



  • 以下に示すように、「提案」セクションでプロジェクトとビルドオプションの提案された依存関係の更新を確認できます。





新しい「リソースマネージャー」



  • 「リソースマネージャー」は、アプリケーションのリソースをインポート、作成、管理、および使用するための新しいツールです。 このツールは、[表示]> [ツールウィンドウ]> [リソースマネージャー]メニューから開くことができます。 リソースマネージャーでは、次のことができます。









    • リソースの視覚化：ドロアブル、色、レイアウトをプレビューして、適切なリソースをすばやく見つけることができます
    • 一括インポート：一度に複数のドローアブルアセットをインポートするには、それらを「リソースマネージャー」にドラッグするか、ドローアブルウィザードをインポートします。 ウィザードにアクセスするには、ウィンドウの左上隅にある「+」ボタンをクリックし、ドロップダウンリストから「Drawablesをインポート」を選択します
    • SVGファイルをVectorDrawablesに変換：Drawablesのインポートウィザードを使用して、SVG画像をVerctorDrawablesに変換できます。
    • アセットのドラッグアンドドロップ： 「リソースマネージャー」ウィンドウから、ドローアブルをレイアウトエディター（「レイアウトエディター」）のデザインおよびXMLウィジェットにドラッグできます。
    • ウィジェットの代替バージョン： 「ツール」ウィンドウでリソースをダブルクリックして、リソースの代替バージョンを使用できるようになりました。 作成したさまざまなバージョンとそれらに含まれる修飾子が表示されます。
    • タイルまたはリスト表示：異なるレイアウトを使用して、リソースの視覚化ウィンドウの表示を変更できます


  • 詳細については、 アプリリソースの管理マニュアルを参照してください。









APKのプロファイリングおよびデバッグ時のビルドIDの検証



  • APK内で共有.soライブラリの「デバッグシンボルファイル」を提供すると、Android Studioは「デバッグシンボルファイル」のビルドIDがAPK内の.soライブラリのビルドIDと一致することを確認します
  • ビルドIDを使用してAPKでネイティブライブラリを収集する場合、Android Studioは「シンボルファイル」のビルドIDとネイティブライブラリのビルドIDが一致するかどうかを確認し、一致しない場合は「シンボルファイル」を破棄します。 ビルドIDでビルドしていない場合、間違った「シンボルファイル」を指定すると、デバッグ時に問題が発生する可能性があります


R8はデフォルトで有効になっています



  • R8は、脱糖、圧縮、難読化、最適化、デキシングを1つのステップで組み合わせており、顕著なパフォーマンスの向上をもたらします。 R8はAndroid Gradleプラグイン3.3.0で導入され、プラグインバージョン3.4.0以降を使用するアプリケーションとAndroidライブラリプロジェクトの両方でデフォルトで有効になりました







  • 以下の画像は、R8より前のコンパイルプロセスの概要を示しています。









  • R8では、以下に示すように、脱糖、圧縮、難読化、最適化、デキシング（D8）が1ステップで実行されます









  • R8は、既存のProGuardルールと連携するように設計されているため、R8を活用するためにアクションを実行する必要はありません。 一方、このテクノロジーは、Androidプロジェクト専用に設計されたProGuardとは異なるため、ProGuardが削除しないコードを削除することで圧縮と最適化が行われる場合があります。 したがって、このまれな状況では、追加のルールを追加して、このコードを最終アセンブリに保存する必要があります。







  • R8の使用に問題がある場合は、 R8互換性FAQを読んで、既製のソリューションがあるかどうかを確認してください。 解決策がまだ文書化されていない場合は、バグを報告してください。 プロジェクトのgradle.propertiesファイルに次のいずれかの手順を追加することにより、R8を無効にできます。 





     <i>＃AndroidライブラリモジュールのみでR8を無効にする</ i>
 android.enableR8.libraries = false </ p>
 <p> <i>＃すべてのモジュールでR8を無効にする</ i>
 android.enableR8 = false


  • 注：指定されたタイプのアセンブリについて、アプリモジュールのbuild.gradleファイルでuseProguardにfalseを設定すると、Android Gradleプラグインは、プロジェクトのgradle.propertiesファイルでR8が無効になっているかどうかに関係なく、R8を使用してこのタイプのアセンブリのアプリケーションコードを圧縮します









ナビゲーションエディターがすべてのタイプの引数をサポートするようになりました



  • ナビゲーションコンポーネントでサポートされるすべてのタイプが、ナビゲーションエディターでサポートされるようになりました。 サポートされているタイプの詳細については、宛先間のデータの受け渡しを参照してください


レイアウトエディターの改善



  • レイアウトエディターの[属性]パネルは、必要な属性を構成するために展開できるセクションを含む1ページに簡略化されています。 属性パネルには、次の更新もあります。







    • 新しい宣言された属性セクションには、レイアウトファイルで指定された属性が一覧表示され、新しい属性をすばやく追加できます。
    • 「属性」パネルには、属性値がリソースである場合に入力される各属性の横にインジケーターも装備されるようになりました
    • エラーと警告のある属性が強調表示されるようになりました。 赤いハイライトはエラーを示し（たとえば、間違った配置値を使用する場合）、オレンジは警告を示す（たとえば、ハードコーディングされた値を使用する場合）




依存関係をすばやくインポートするための新しいアクションのヒント



  • コードで特定のJetpackクラスとFirebaseクラスの使用を開始する場合、新しいアクションのヒントは、必要なGradleライブラリの依存関係がまだ追加されていない場合、プロジェクトに追加することを推奨します。 たとえば、WorkManagerクラスを初めて参照する場合は、android.arch.work：work-runtime依存関係が必要です。アクションヒントを使用すると、次のようにワンクリックで追加できます。



  • 特に、 Jetpackは個別のパッケージで提供され（管理と更新が容易になります）、アクションショートカットを使用すると、使用するJetpackコンポーネントに必要な依存関係のみをすばやく追加できます

  • Android Q Betaを実行しているデバイスにアプリケーションをデプロイすると、プロファイリングが無効になります
  • データバインディングライブラリが使用される場合、LiveDataListener.onChanged（）はNullPointerExceptionで落ちる可能性があります。 この問題の修正はAndroid Studio 3.4.1に含まれており、 Android Studio 3.5の最新のプレビューバージョンで既に利用可能です（ バグ122066788を参照）
