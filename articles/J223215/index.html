デスクトップCRM：実用的な実装の外傷学

企業が一般的なビジネスの自動化、特にCRMの実装に関するアドバイスを求めると、まったく同じ症状がよく見られます。インフラストラクチャをゼロから作成する必要はなく、別のソフトウェアと交換する必要があります。 これは、特に古いバージョンであるにもかかわらず、ソフトウェアが適切であることがわかっているが、使用方法や使用方法がわからない場合に、しばしば困難な状況です。 クライアントの代表者を集めて、「Do it All」ボタンが作成されておらず、ソフトウェアが個人の責任ある作業を必要とすることを残念に説明する必要があります。





さまざまなCRMの実装で蓄積されたすべての経験を収集し、コーンからのトラブルを防ぐためにhabrozhitelamiと共有する準備ができています。 デスクトップソフトウェアについて主に説明します。これは、最も安全で、効率的で、柔軟性があるだけでなく、実装が難しいからです。 少し説明する価値があります。 分析目的のデスクトップソリューションは、いくつかの理由で選択されます。

  • ビジネス目的でシステムをインストール、トレーニング、ファイナライズ、および管理するための一連のアクションとしての「実装」の概念は、ご存じのとおり、ほとんどの場合サービス（SaaS）として提供されるクラウドには適用されません。

  • クライアントタスクのCRMの完全な微調整は 、デスクトップCRMでのみ可能です。

  • デスクトップCRMは商用データを保護するという点でより安全です

  • クラウドシステムが開発ツールを開発することはめったにありません。もちろん、これが完全なオープンソースであり、少なくともフルタイムプログラマの給与に相当な投資が必要な場合を除きます。



これらの要因に基づいて、分析のために3つの一般的なデスクトップCRMシステムを選択しました。

  • Terrasoft CRM-優れた機能を備えた最も有名なCRM

  • RegionSoft CRM-強力な機能を備えた有名なCRM

  • クライアントコミュニケーター（Clic） -有名なCRM、興味深いプラットフォームソリューション



多くのベンダーのソリューションの実装における幅広い経験を考えると、これらのシステムは中小企業向けのシステム市場のリーダーであると言えます。 さらに、分析の目的のために、それらは、独立した外部の洗練のための武器庫にユニークで興味深いソリューションがあるという点で適切です。



まず、実装プロセスは、クライアント企業（コンサルタントがすぐにその側で働く）、会社の従業員、およびベンダー（開発者）の管理を結び付けるビジネスであることを理解する必要があります。 したがって、潜在的なリスクはどの側からも発生する可能性があります。



クライアントのリスク


ビジネスニーズの誤解は、最初の主要な最も破壊的なリスクです。 自動化する前に、ビジネスを完全に分析し、長所と短所を評価し、何を変更するかを決定する必要があります。 たとえば、小さなプリントショップがあり、2つの荒廃したリソグラフに10人の営業担当者がいて、多くの顧客を引き寄せている場合、1つのCRMは役に立ちません。 まず、機器を購入し、生産を確立してから、CRMを介して顧客と社内の関係を構築する必要があります。 しかし、これは完全に無視されたケースです。

  • 生産とロジスティクスはありますか、それらはどのように編成されていますか。 ロジスティクスと生産を開発する場合、 KliK 、 RegionSoft CRM 、 Terrasoft CRMなどの強力なシステムに注意を払う必要があります 。 多くの企業では機能的に冗長ですが、クライアントがビジネスプロセスを最適に構築し、効果的に管理できるようにします。 最高の外国の競合他社に匹敵するパワーと機能セットの点で、彼らはロシア市場に合わせて調整されており、システムをファイナライズするコストの大幅な削減に役立ちます。
  • 顧客は誰ですか：b2cまたはb2b。 たとえば、倉庫、値札、割引カード、その他の小売機器を備えたチェーン店について話している場合、 RegionSoft CRMの方が適しています。 卸売業者またはビジネス（取引、銀行、広告代理店、生産など）を扱う会社の場合、リストされたベンダーのいずれかで適切なソリューションを見つけることができます。
  • システム管理者とプログラマーの専門性は、まったくありますか。 従業員のトレーニングはほぼすべてのベンダーによって支払われており、多くは有能な管理者が実行できるサービスを支払っています。 さらに、プログラミングに精通したスタッフがいる場合は、すべてのベンダーから提出されたフォームとレポートを完成させるためのツールを使用できます。 ただし、誰かにとっては、これはレポートを作成したり、カードにフィールドを追加したりする機会であり、誰かは完全な開発のための強力なソリューションを持っていることに注意する必要があります（ クリックとそのレポートデザイナー、 RegionSoft CRMおよびそのアプリケーションサーバー ）。
  • そのようなソフトウェアが会社にインストールされており、なぜそれがあなたに合わないのか。 たとえば、クライアントには巨大で複雑な分析CRMがありました。 従業員がレポートの海を受け入れず、恐ろしいマルチウィンドウインターフェイスで混乱したため、実装に多大なお金が必要でしたが、利益をもたらしませんでした。 その結果、クライアントはその目標のために運用CRMを選択しました。このCRMは、数年にわたって彼と彼の商用サービスに役立ってきました。 ソフトウェアの使用中に問題が発生した場合は、問題を分析し、従業員にインタビューして、新しい購入がお金の無駄にならないようにすることをお勧めします。
  • 専門性の低い会社の場合 、業界ソリューションを探すか、有名なベンダーに本格的なプロジェクトを注文してください。 あなたのために開発されたソフトウェアは、最も高価ですが、最適なソリューションになります。


クライアントの管理に関する2番目の一般的な問題は、 すべてを一度に迅速かつスムーズに獲得するという要件です。 これは不可能です。従業員はインターフェイスに慣れ、CRM構造を理解し、データを入力する必要があります。 その後のみ、すべての機能を完全に使用できるようになります。 新しいプログラムへの適応を拒否する経営陣は、混乱と無数のエラーに陥ります。



三番目の惨劇は貪欲です。 欲は、ほとんどすべてのプロジェクトの導入によって追求されます。 選択時に始まり、改善とトレーニングで終わります。 ここでは、理解し、少なくとも部分的に受け入れる必要があるルールのリストを単純にリストできます。

  • 価格はCRMを選択するための主要な基準ではありません。 主な基準は、バージョンと機能の新規性です。製品は開発者が放棄するのではなく、開発する必要があり、機能は最新の要件に準拠する必要があります。 たとえば、主要なドキュメントフォームのバージョンは関連性があり、インターフェイス-便利、倉庫、ロジスティクス-商用機器などをサポートする必要があります。...今日、すべての主要なCRMシステムがサポートする一連の機能が市場で開発されました。 ベンダーが基本的な機能を提供できない場合、開始価格が低いにもかかわらず、CRMに最大限の投資をすることを知っておく必要があります。 これは、企業がソフトウェアを成長させ、2回支払いを開始する状況です。


企業にCRMを実装する場合、多くの場合、ビジネスのニーズを満たすためにソフトウェアを「仕上げる」必要があります。 そして、多くの人々は、「経済」の理由で、その緊急の必要性を認識してさえ、最終的に決定することを拒否します。 これは、「カズベクの喫煙、試合の節約」という標準原則に相当します。 洗練の価格は、利便性、効率性、作業の迅速な開始の価格です。



改善には主に2つのケースがあります。独自の仕様（HoReCa、薬局、建設、イベントなど）を持つ企業向けと、大規模実装向けの小さな改善（レポート、モジュール、カード分野）です。 前者の場合、コストはかなりの額になりますが、その結果は最も効果的で適切な仕事のためのツールを会社に提供するはずです。 2番目のケースでは、これらは基本的な機能に基づいたクライアントの希望であり、改良はそれほど高価ではありません。 この状況では、フルタイムのプロのプログラマーの助けを借りてもCRMを改善できます。ベンダーは、レポート（ClicKレポートデザイナー、Terrasoft CRM、レポートデザイナー、および前述のApplication Server RegionSoft CRM）を操作するツールを提供するか、最も簡潔な締め切り。



これからさらに3つの原則が続きます。

  • ファイナライズが支払われます。 常に支払われます。 開発者がソーシャルネットワークのボタンを無料で「スナップ」する必要はありません。さらに、「フィールドとここでのアルゴリズムを少し変更するだけ」という要求は未回答のままです。 あなたの顧客はあなたに支払います、そして、この状況では、顧客はあなたに支払います。 改訂の費用を支払わなければならない場合は、目標を達成するために最も高品質で適切な納品が得られるように要件を整理してください。
  • トレーニングを節約しないでください -コストはそれほど高くなく、トレーニングを受けた従業員の仕事はずっと早く始まります。
  • アップグレードの支払い。 もちろん、すべてのベンダーは一定の制限内で無料のアップデートを提供しています。 ただし、新しいグローバルバージョンがリリースされ、それを適切な量でインストールするように提案された場合-技術革新と更新を研究すると、優れた技術サポートを備えた最新の安定したバージョンが得られます。


クライアントの従業員の側からのリスク


  • 新しいソフトウェアの妨害。 一見奇妙に聞こえますが、常に発見されています。 この現象の原因は心理学にまで遡ります。管理者は過剰な制御、目新しさ、トレーニングと理解に費やされる時間を恐れています。 この問題は簡単に解決できます。ほとんどすべてのベンダーがトレーニングを提供しており、多くのベンダーが優れたマニュアルと組み込みのヘルプを提供しています。 実装後、エキスパートレベルでシステムを理解し、他の人と知識を共有する準備ができている従業員がいることが重要です。 ITスペシャリストでも商人でもかまいませんが、理解を深めることが重要です。 トレーニングにかかる​​時間と費用を節約すると、CRMを使用する際に特定の効率性が失われます。
  • 古い顧客の部分的な損失。 この状況は、従業員が新しい顧客と新しい取引をCRMに紹介するときに発生しますが、古い顧客は引き続きファイルやメモ帳に保持されます。 アンロード（可能であれば）または手動で古いデータベースをCRMに追加することは、顧客を維持し、顧客との関係を追跡する方法であるだけでなく、システムの機能全体を学習しながら実践する大きな理由でもあります。


ベンダーからのリスク（開発者）


これらのリスクは通常、顧客の行動に関連しています。 したがって、実装プロセスでは、クライアントは開発者との対話の機能の一部を理解する必要があります。



  • 参照条件 -ランディングページの作成から包括的なビジネスの自動化まで、ITプロジェクトで最も苦しい場所だと確信しています。 それがどんなに悲しいものであっても、クライアントはその書き方を知らないことが非常に多く、ベンダーはクライアントが必要とするものを完全に知り理解することができません。 この状況では、解決策があります。技術的な「繁栄」を主張せずに、クライアントはすべての要件を人間の言葉で述べ、必要に応じてベンダーが要件を正式化する必要があります。 さらに、1つまたは別の機能が必要な目標を述べるようにしてください。ほとんどの場合、開発者は、必要な機能の説明からではなく、目標に基づいてシンプルで手頃なソリューションを提供します。

    参照条件はプロジェクト計画ではなく、あなた（顧客）がソフトウェア（CRM）に何を求めているかの完全な詳細な説明であるべきです。 ToRがより有能で詳細になればなるほど、実装はより速く、より成功します。
  • 改善。 改訂の過程で、プロジェクトの要件、およびその結果としての期限は、しばしば変更される可能性があります。 この状況では、両側が負けます。 このような問題を回避するためには、正しく作成された技術的なタスクから再び進む必要があります。 さらに、クライアント企業は、自社の目標の改善の必要性を正確に評価する必要があります。そうしないと、供給の多くの変更が大きなコストにつながります。


CRMの実装プロセスを3つの側面から検討した結果、次のように結論付けることができます。

  • 実装前に社内のプロセスを徹底的に分析する
  • あなたのビジネスとその目標に可能な限り適切なシステムを作ることを節約しないでください
  • ベンダーと緊密に連携します。この状況では、ベンダーは単なるパフォーマーではなく、システムを完全に知っており、最大限の効率でシステムを展開する準備ができている専門家です。
