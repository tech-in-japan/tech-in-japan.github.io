FireFox-異なるOS用の1つのプロファイル

それで、すべて同じように、Windows用のデータとプログラムの同期の問題に対処しました-フラッシュドライブでソフトウェア（主にPortableAppsのクラフト）を持ちます...女性にとっては面倒ではありません-女性は豚を買いました。 もちろん、私は女性ではありませんが、Windowsなしで自宅のマシンを離れ、Linux Gentooでの生活に完全に切り替えました。



スイッチャーの最初の問題は、新しいプラットフォーム用の使い慣れたソフトウェアの検索です。 私の場合、パスワードやFireFoxとThunderbirdを保存するためのKeePassクロスプラットフォームアプリケーションが最初にポートからインストールされました。



Windowsを完全に放棄することはできませんでした。Photoshopで作業するだけで、それについて何もできないからです。 Gimpでさえも代替手段ではありません。 したがって、私の古い作業コンピューターのWindowsには、Photoshopがまだインストールされており、Windowsフラッシュドライブが起動します。 これらのプログラムのベースを同期する方法は？ 最初に、デバイスをマウントするときにUSBフラッシュドライブからユーザープロファイルをコピーし、アンマウントするときにUSBフラッシュドライブにコピーしようとしました。 誰にとっても良いように思えますが、USBフラッシュドライブのマウントまたはマウント解除を愚かに忘れてしまうことがあり、同様の方法で同期すると、データの損失に悩まされます。 判明したように、より洗練された、しかしささいな解決策があります。





だから：

1.一般プロファイルの保存場所を決定します。 私の場合、これはフラッシュドライブです。同じコンピューターで2つのOSを使用している場合は、FATパーティション上のディレクトリにする必要があります。 なぜFATが理解できるのか。 念のため、LinuxでのFATパーティションへのアクセスは非常に簡単ですが、Windowsでext / reiserfsパーティションにアクセスするには、追加のジェスチャーが必要です。



2.アクティブなプロファイルを含むフォルダーを選択したディレクトリにコピーします。 デフォルトでは、Windowsでは、プロファイルフォルダーはC：/ Documents and Settings / user / Application Data / Mozilla / Firefox / Profiles / xxxxxxxx.default、およびLinuxでは/home/username/.mozilla/firefox/にあります。



3.特別なパラメーターでFireFoxを起動します。 Windowsでは、Win + R firefox.exe -profilemanager Linuxでは、firefox -profilemanager。





4.プロファイルを作成するオプションを選択し、表示されるウィンドウで名前を指定します。 主要な犬のブリーダーは、デフォルトのユーザープロファイルを呼び出すことをお勧めします、私は推奨事項に耳を傾けなかったし、天罰は私に落ちませんでした...





5. [*フォルダの選択] *をクリックし、手順2で作成したプロファイルのあるディレクトリへのパスを指定します。完了。





以上です。 ポータブルバージョンのユーザー -古いプロファイルを急いで削除しないでください！ 個人的には、希望のパラメーターでPortableFirefoxを起動できなかったため、Linuxでは/ FirefoxPortable / Data / profile /でUSBフラッシュドライブへのパスを直接指定しました



ちなみに、これはもちろんThunderbirdにも当てはまります。
