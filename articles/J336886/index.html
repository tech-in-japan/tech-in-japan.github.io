Rambler / Likeレポートの例を使用したブロック分析



2016年、RamblerはソーシャルメディアボタンのデザイナーRambler / Likeをリリースしました 。 Rambler / likesの例、「Block Analytics」の仕組みを見てみましょう。 しかし、最初に、いいね！ Rambler / Likeコンストラクターでは、さまざまなソーシャルネットワークを選択し、ボタンの外観とその動作をカスタマイズできます。最後に、ボタンをサイトに挿入するウィジェットコードを取得できます。 Likesを設計するとき、ウィジェットユーザーがインストールされたボタンの統計を表示できるようにしたいと考えました。 Rambler /トップ100で利用可能なユーザー行動に関するデータを収集するすべてのツールの中で、選択は「ブロック分析」にありました。これを使用すると、必要な詳細を備えた各ページのボタンのクリック可能性に関する情報を取得できます。



元のウィジェットのボタンには、ランブラー共有データ属性が既にマークされています。この属性は、トップ100カウンターで表示できます。 トップ100カウンターとソーシャルネットワークボタンがWebサイトにインストールされている場合、ボタンのデータは「行動/ランブラー/いいね」レポートで確認できます。



Likesウィジェットがインストールされたページがロードされると、 bl-イベントがTop-100カウンターに渡されます。このカウンターには、 rambler-share :: domain :: page_url :: social_networkなどのソーシャルネットワークボタンパラメーターが含まれます。 任意のボタンをクリックすると、 clイベントには同じパスが残ります。 その結果、Rambler / Likeレポートでは、サイトの各ページの各ソーシャルネットワークボタンのインプレッション、クリック、CTRが表示されます。



ボタンを使用してソーシャルネットワークへの送信試行の総数を確認できるように、クリックデータは次の各レベルで要約されます。 1つのカウンターが複数のドメインにある場合、ドメインごとの統計を表示できます。 上位レベルのインプレッションデータは、サブレベル間の最大値と、短縮パス（サブレベルなし）のチェーンのインプレッション値を合計して計算されます。 Rambler / Likeウィジェットの場合、ウィジェットは全体として機能するため、通常の操作ではボタンがない状況は不可能なので、これは関係ありません。



必要に応じて、レベルを手動でマークアップに追加して、より詳細なデータを収集できます。 たとえば、Rambler / Newsで行われているように、3つのLikesウィジェットがページの異なる部分にインストールされ、ブロックの場所を指定してトップレベルがマークアップに追加されます。 これにより、各ボタンウィジェットの統計を個別に表示し、それらを相互に比較できます。







今後、ブロックをマークするときに既存のページ構造を使用できることに注意してください。各divに対して、階層に対応するマークアップレベルを指定します。 さらに、この方法をさらに詳しく検討します。



マークアップの詳細



上記の例からわかるように、「ブロック分析」レポートを使用して、ページ上のさまざまな要素のクリック可能性を分析できます。 実際、これはサイトのすべてのページで管理するクリックマップです。どの要素が重要で、どのように階層的に並べられるかを選択し、CTRの詳細な統計を取得します。

もちろん、これは魔法ではありません。データ収集のために、少し回避する必要があります。対象のブロックをデータ属性でマークし、attributes_datasetパラメーターの値としてカウンター初期化コードに渡します。 このパラメーターを指定すると、Top-100 usability.jsカウンターモジュールがサイトにアップロードされ、データの収集が開始されます。 マークアップされたブロックで収集するためのコードは別のモジュールで接続されているため、Top-100カウンターの簡素化されたバージョンは可能な限り簡単です。



次に、データの収集方法について説明します。 ページがロードされると、カウンターは渡されたデータ属性を「リッスン」し、ロードされたマークアップされたブロックの表示をキャプチャします。 このような「リスニング」は、後でダウンロードされる要素の表示を収集するために、ページ表示ごとに数回発生します。 クリックは、訪問者の対応するアクションとともに直接記録されます。 「Block Analytics」レポートは階層構造です。最上位レベルには、送信されたデータ属性が表示され、各属性は最大8レベルのツリーに展開できます（深さは構成されたレイアウトによって異なります）。 いくつかのレベルは、複雑な階層構造を持つページに役立ちます。そのためには、特定のリンクとリンクを持つブロック全体の両方の有効性を測定する必要があります。 CTRは、各レベルでのインプレッションに対するクリックの比率として計算されます。 「Block Analytics」モジュールの接続を使用したカウンター初期化コードの例は少なくなりますが、ここではマークアップ自体について説明します。







紙の上でマークアップを設計することから始めてから、サイトで実装を進めることをお勧めします。 まず、データ収集の目標を特定する価値があります。 たとえば、サイトとそのページの有効性を評価するのに十分ではない情報は何ですか？ 評価のための十分なデータを取得するために、どの訪問者の行動を収集する必要があるかについての情報？ 収集されたデータはあなたの質問に答え、読みやすくする必要があります。そのため、最適な数の属性を選択し、必要なすべてのレベルに明確な名前を付けることが重要です。 このアプローチは、「すべてをマークアップしてから、それを把握します」という原則として、将来的には多くの質問とマークアップの変更につながります。 これにより、長期間にわたってレポートが読めなくなります。同じデータが異なるブランチで収集されます。 同じことは、後でレポートを操作するマネージャーではなく開発者がマークアップを作成するオプションにも当てはまります。 失敗した名前を選択する場合、数週間で1つまたは別のパラメーターで収集されたデータとそれらが必要な理由を思い出すことが困難になる可能性があります。

TL; DRまたは要約



Rambler / top-100は、インターフェイス要素のクリック可能性を分析するためのシンプルなツール、「ブロック分析」レポートを提供します。 追加のモジュールコールをTop-100カウンター初期化コードに追加し、必要なサイト要素をマークアップすることにより、統計を収集し、次の問題を解決できます。



  • ページまたはボタンのターゲットブロックのクリック可能性を評価し、それらのCTRを比較します。
  • 材料のさまざまな見出しの有効性を分析します。
  • 推奨事項のブロックのCTRとその中の特定の位置を調査する。
  • 同じブロックが異なるページでどのように機能するか、クリック可能性が時間とともにどのように変化するかを調べます。
  • A / Bまたは多変量テストで同じタイプの要素のクリック可能性を比較します。
