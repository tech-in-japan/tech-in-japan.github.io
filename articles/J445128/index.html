私はすぐにこのギャップを埋める素晴らしいアイデアを思いつきました。

既存のチュートリアルが悪い理由



既存のチュートリアルが悪いと思う理由を少し説明します。 Apexはプログラミング言語ではありません。非常に高度な開発ツールであり、MS Accessと呼ばれるイデオロギーに最も近いものです。 したがって、apexでの開発の大部分は、この環境のWebインターフェースでの作業で構成されています。ボタンをクリックする、リンクをたどる、フィールドにコンポーネント名を入力する、リストから値を選択するなど。 典型的なチュートリアルは何をしますか？ キャプション「クリックしてください」と同じ画面のスクリーンショットを示しています。 同時に、IDE自体は非常に便利で理解しやすく、ほとんどすべての要素に参照説明が用意されており、組み込みのヘルプがあります。 その結果、アプリケーションの作成方法をユーザーに説明する必要がある場合は、チュートリアルに「新しいアプリの作成」ボタンをクリックする方法を説明する大きなスクリーンショットが表示されます。 ここでは、オラクルの公式チュートリアルをご覧ください。













また、スクリーンショットを使用して、「クリック」とはマウス型マニピュレーターの左ボタンを1回クリックすることを説明します。

もちろん、このようなチュートリアルには有用な情報もありますが、そこにはほんのわずかです。 チュートリアルの80％はこのようなスクリーンショットで構成されており、私自身の経験から言えば、チュートリアルをまったく読んでいない人は、アプリケーションの作成方法、ページの追加方法、レポートの配置方法などを個別に見つけることができます。 そして、彼は頂点のいくつかの要素の多かれ少なかれ非自明な相互作用を整理する必要があり、この質問はチュートリアルでほとんどカバーされていないことがわかります。 類推してみましょう。プログラミングを学び、変数、配列、ループ、条件を研究し、配列をソートする必要があるフレームワークでタスクを取得しました。 配列のソートはどういうわけか非常に簡単ですが、「 大きな配列をすばやくソートする」ことでタスクが複雑になるとすぐに、知識が不十分であることがわかります。 グーグル、Stackoverflow、フォーラムを掘り、あなたはクイックソートアルゴリズムを見つけるでしょう。 もちろん、だれかが遅かれ早かれこのアルゴリズムを再発明しますが、より効果的な方法は、典型的なアルゴリズムの分析を含む本をすぐに読むことです。 そして、そのような頂点の「典型的なアルゴリズム」のセットは通常どこにも見つからず、科学的な突っ込み、検索エンジン、フォーラム、Stackoverflow（そして、このチュートリアル）の同じ方法が助けになります。

IDEの概要



IDEに入るとすぐに表示される開始画面：













主な要素は、4つの大きなクリック可能な画像です。







  1. 「アプリビルダー」：実際、これはアプリケーションを作成できる場所であり、最大99％の時間を費やします。
  2. 「SQLワークショップ」：Oracle DBMSを操作するための通常のIDEの代替。 ここで、データベースオブジェクト（テーブル、インデックス、パッケージなど）を表示し、SQLクエリを実行できます。 「通常の」IDE（Oracle SQL DeveloperやPL / SQL Developerなど）を介してこれらすべてを実行できる場合は、それらを介して実行してください。 それにもかかわらず、このセクションも役に立つかもしれません。私もそれについてお話します。
  3. 「チーム開発」：チーム開発のためのツール。 正直なところ、誰もが実際にそれらを使用するのを見たことがありません（私を含む）。 好奇心のために、私もそこを見ました。本当にしたいのですが、このセクションを完全に役に立たないとは言いません。
  4. 「アプリギャラリー」：デモアプリケーションのセクション。 彼が登場する頃には、私はすでに頂点をよく知っていて、ほとんどそこを見ていないが、初心者にとっては有用だと思う。 気取らないアプリケーションにはさまざまな種類があり、その例では、これらの機能やその他の機能の使用方法を学ぶことができます。


ページの一番上のメニューで、同じ4つのポイントが複製されます。







さらに、さらにいくつかの役に立たないことがあります。以下は、一定量の統計情報（最も頻繁に編集されるアプリケーションや最もアクティブな開発者など）です。 また、右側に列があり、いくつかの統計と追加のリンクもあります。







App BuilderとSQL Workshopをさらに詳しく見てみましょう。







アプリビルダー



このページは次のようになります。













このページの主な要素：前のページと同様の4つの大きなボタン、既存のアプリケーションのリスト、最近変更されたアプリケーションのリストと追加のアクションを含む右の列。 4つのボタンは次のとおりです。







  1. 「作成」：新しいアプリケーションを作成します。 同じ機能が少し下に複製されます（アプリケーションのリストを含むレポートの上の青い「作成」ボタン）。 ボタンは、アプリケーション作成ウィザードを起動します。
  2. 「インポート」：アプリケーションをインポートします。 Apexを使用すると、あるサーバーから別のサーバーにアプリケーションをテキストファイルの形式で転送できます（これは、本質的に、すべてのアプリケーション要素を作成するための関数を呼び出す長いPL / SQLスクリプトです）。
  3. 「ダッシュボード」：アプリケーション開発統計の少し役に立つコレクション。 誰が、いつ、どこで、いくつの変更が行われたか。 あなたの上司でさえ、そこを見ることには興味がないと思います。
  4. 「ワークスペースユーティリティ」：アプリケーションを管理するための追加ツールのセット。 正直なところ、私もそこを見ることはほとんどありません。


また、レポートパネルのこのような小さなボタンに注目してください。













彼女は、レポートのビューを上記のスクリーンショットにあるもの（アプリケーションの「写真」を含む）から表形式のレポートに切り替えます。













同様のレポートは、IDEの他のセクションにあります。 私は表形式のビューを好むので、さらにすべてのスクリーンショットが表示されます。







申し込みページ


アプリケーションを作成するか、既存のアプリケーションを開くと、次のページが表示されます（構造上の前のページと非常によく似ています）。













このページには非常に頻繁にアクセスしますが、ここには多くの便利な機能があります。 写真付きの大きなボタン：







  1. 「アプリケーションの実行」：このボタンは、アプリケーションを起動します。 別のウィンドウで開きます。 デフォルトでは、ブラウザタブではなくウィンドウ内にありますが、この動作は設定されています（詳細は後ほど）。
  2. 「サポートオブジェクト」：このセクションでは、アプリケーションのインストール、更新、およびアンインストールに関連する問題について説明します。 通常、それらはほとんど使用されません。
  3. 「共有コンポーネント」：これは、IDEの最初または2番目に重要なセクションです（もう1つはページエディターです）。 頂点で多くを開発する場合、このセクションの構造を「私たちの父」として学んだことにすぐに気付くでしょう。
  4. 「ユーティリティ」：（少数の）便利なヘルパー関数の別のセット。
  5. 「エクスポート/インポート」：アプリケーションまたはアプリケーションの一部をエクスポートまたはインポートできます。


スクリーンショットではオレンジ色のフレームで囲まれた2つの追加ボタン：







  1. 「アプリケーションのプロパティを編集」ボタン。 クリックすると、アプリケーションのプロパティ設定ページが表示されます。 このセクションに頻繁にアクセスすることはできませんが、多くのことが依存するアプリケーション設定を保存するため、非常に重要です。 さらに、どこかで「アプリケーションプロパティ」について言及する場合は、このセクションでそれらを探します。
  2. かつて、右ペインの便利な機能は「このアプリケーションを削除する」です。 頂点の多くの関数は異なる場所で複製されますが、この関数はここにしか存在せず、習慣なしで初めて見つけることは困難です。


アプリケーションのプロパティ


まず、多数の設定へのアクセスを整理することは興味深いです。













設定は4つのグループにグループ化され（上のフレームを参照）、各グループにはさらにいくつかのサブグループがあり（下のフレームを参照）、各グループには独自の名前があります。 同様に、設定は頂点の非常に多くの場所で整理されます。 将来的には、「アプリケーションのプロパティ」→「定義」→「名前」→「バージョン」のように、目的の設定へのパスを指定するだけです。







また、ページ上部のパンくずリストを見ると、「アプリケーション定義の編集」と表示されていますが、「アプリケーションプロパティ」に移動しているように見えます。 無視してください。







共有コンポーネント


共有コンポーネントがすべてです。 このセクションは、特定のアプリケーションコンポーネントへの道の中間点です。 ここで一般的なビューを表示します。これは、どこが何であるかを想像できるようにするためです。













ここにはナンセンスはほとんどありません。 各セクションが何らかの形で必要になる場合があります。 セクション名は一目瞭然で、追加することはほとんどありません。







  1. 「アプリケーションロジック」：アプリケーションロジックコントロール。
  2. 「セキュリティ」：ユーザーロールに応じて、アプリケーション全体および個々のコンポーネントのセキュリティとアクセス制御の問題。
  3. 「その他のコンポーネント」：これらは「その他すべてのコンポーネント」です。 良い意味では、このセクションは最後のはずですが、何らかの理由でここにあります。 また、「値の一覧」サブアイテムは、通常、アプリケーションロジックに転送します。
  4. 「ナビゲーション」：ナビゲーション用のアプリケーションコンポーネント。
  5. 「ユーザーインターフェイス」：アプリケーションの外観、スタイルなどをカスタマイズします。
  6. 「ファイル」：ファイルのセクション（たとえば、cssまたはjs）。 OSにアクセスできない場合は不可欠であり、アクセスがあっても便利です。
  7. 「データソース」：データソースを操作します（ロードおよびアンロード用）。
  8. 「レポート」：レポート、明らかに。
  9. 「グローバリゼーション」：アプリケーションを他の言語に翻訳するためのセクション。

ページデザイナー


Page Designer-ページエディター。 これはおそらくIDEの主要な場所です。 ここでは、ユーザーが直接操作するもの、つまりアプリケーションのページを作成します。 それは、コックピットのように、単なる機能の集まりです。 Page Designerにアクセスするには、アプリケーションページ上のページの名前のリンクをクリックする必要があります（トートロジーについては申し訳ありません）。 スクリーンショットでは、エディターの主要コンポーネントがカラフルなフレームで強調表示されています。













上部のボタンバー（赤枠内）：







  • ページ番号、上矢印と下矢印、および「実行」ボタンを入力するためのフィールド：アプリケーションのページをナビゲートします。 各ページには番号があり、ここで目的のページの番号を入力するか、矢印を使用して次/前に移動するか、左端のボタンをクリックして表示されるモーダルウィンドウのリストからページを選択します。
  • ロック付きのボタン：他の開発者による変更からページをブロックできます。
  • 元に戻す/やり直しボタン
  • 補助機能にアクセスできる2つのボタン。
  • 共有コンポーネントへのボタンリンク。 常に行き来する必要があるので便利です。 （ただし、私自身は、共有コンポーネントを別のタブに保持することを好みます。これはさらに高速です）。
  • 「保存」ボタン：ページの変更を保存します。
  • 「実行」ボタン：ページを「起動」します。つまり、ユーザーが後で見るように表示されます。


左側（オレンジ色のフレーム）には、ページコンポーネントのあるパネルがあります。 このパネルには4つのブックマークがあります。







  • 「レンダリング」：ページを構成するコンポーネント、レンダリング中に実行されるプロセスを示します。
  • 「動的アクション」：クライアントで実行されるアクション（javascriptイベントハンドラーなど）。
  • 「処理」：ページに関連付けられたサーバープロセス。
  • 「ページ共有コンポーネント」：ページで使用される共通コンポーネント（同じ共有コンポーネント）に直接移行します。


中央（緑色の枠内）-さまざまな異種情報を含むパネル。 パネルの上部には、次の5つのタブがあります。







  • 「レイアウト」：ページ上のコンポーネントのおおよその配置を示します。
  • 「コンポーネントビュー」：概して、これは過去の遺物です（バージョン5.0または5.1では、ページエディタはApexの4番目のバージョンと同様に見えました。彼の薄い影だけ）;
  • 「メッセージ」：エラーメッセージが表示されます（たとえば、一部の必須フィールドが入力されていないことを示すメッセージ）。
  • 「ページ検索」：ページを検索します。 何かを入力できるすべての場所（コード、メッセージテキスト、コンポーネント名など）で目的の文字列を検索します。
  • 「ヘルプ」：ヘルプ。 常にではありませんが、かなり賢明です。 左パネルでコンポーネントプロパティを選択し、その説明を確認できます。


中央のパネルには、ページにコンポーネントをすばやく追加するために使用できる別の追加パネルもあります。







右側（青枠内）にプロパティパネルがあります。 ここではすべてがシンプルです-現在選択されているコンポーネントのプロパティが表示されます。 コンポーネントのプロパティの一部は単純です：テキスト、数値、リストの値。 ただし、コンポーネントのより複雑なプロパティには、それらを操作する際の生活を大幅に促進する追加のインターフェイス機能があります。

大きなテキストであるプロパティ。 たとえば、PL / SQLまたはJavaScriptのコード。 次のようになります。













このフィールドにコードを直接記述するか、ページ上部のボタンをクリックすると、コードエディターでモーダルウィンドウが開き、構文の強調表示（SQL、JavaScript、CSS、HTMLの場合）、テキスト検索、その他のエディター機能が利用できます。







プロパティ-共有コンポーネント。 プロパティがアプリケーションコンポーネント（共有コンポーネント）の1つである場合、リストから特定のコンポーネントを選択し、リストの右側にあるボタンをクリックしてください。













[共有コンポーネント]セクションに移動して、選択したコンポーネントのプロパティに直接移動できます（たとえば、変更します）。







リストからプロパティ値を選択すると、右側に別の追加ボタンが表示される場合があります。













「クイックピック」と呼ばれ、多くのオプションがある場合に使用されます。 「クイックピック」は、オプションの小さなサブセットへのアクセスを少し速くします-原則として、最も一般的に使用されるオプションがあります。 ここで「頻繁に使用されるオプション」は、ユーザーによって使用されるのではなく、「一般的に」使用されます。 これは固定リストであり、時間の経過とともに変化することはありません。







SQLワークショップ



先ほど言ったように、SQL Workshopは通常のIDEの代わりになります（何らかの理由で使用できない場合）。













5つのセクションで構成されています。







  1. 「オブジェクトブラウザ」：データベースオブジェクト（テーブル、パッケージ、関数とプロシージャ、インデックス、シーケンスなど）のブラウザ。
  2. 「SQLコマンド」：SQLクエリまたはPL / SQLコードを実行できるセクション。
  3. 「SQLスクリプト」：ここでは、スクリプトを含むテキストファイルをダウンロードして実行できます。
  4. 「ユーティリティ」：データベースに関連するさまざまな追加機能：たとえば、グラフィカルなクエリビルダーがあります（ただし、それだけではありません）。
  5. 「RESTfulサービス」：RESTサービスを作成します。 また、ApexにはRESTがあります（長い間、すでにバージョン4.2でした）！


オブジェクトブラウザ








データベースオブジェクトを操作するために必要なものはすべてあります。作成、変更、削除、オブジェクトのプロパティの表示、テーブルへのデータ入力-このすべてのために、このセクションにはグラフィカルインターフェイスがあります。 ドロップダウンリストの左上で、オブジェクトのタイプを選択できます（スクリーンショットに表が表示されます）。右上で、オブジェクトを表示できるスキームを選択できます。 利用可能なスキームのリストでは、すべてのスキームが一度に表示されるわけではありません-これは正常です。 頂点にスキームを表示する方法は、管理に着いたときにお知らせします。

また、このセクションをより詳細に説明することにはあまり意味がありません。「テーブルに列を追加する」というフレーズの意味がわかれば、「列を追加」ボタンをクリックする必要がある時期と、次に期待できることを簡単に推測できます。







SQLコマンド


このセクションでは、SQLクエリとPL / SQLコードを実行できます。













能力の面では、このセクションは本格的な開発環境とほぼ同じです。 不足しているのは、コード補完と構文の強調表示だけです（そして、Object Browserには構文の強調表示があります）。 利用可能な機能：







  • パラメーターを含むSQL実行（パラメーター値を入力するための別のウィンドウが表示されます）;
  • dbms_outputを介してテキストを出力する機能を備えたPL / SQLの実行（出力は下のウィンドウに表示されます）;
  • コードの選択された部分のみの実行（マウスでテキストを選択し、次に「実行」）;
  • リクエストプランの表示。
  • リクエストを保存します。
