2017年のソリューションアーキテクチャ：STO EPAM、Eli Feldmanのインタビュー

最近、EPAMのCTOであるEli Feldmanがウクライナを訪問しました。 CTOのAdvanced Technologyで、Eliはコンピテンスセンター、建築分野、「ガレージ」に基づくイノベーションの開発（ 「ガレージ」またはガレージ -新しい技術を研究するためのハイテク研究所）の開発、および会議の準備を担当しています。開発者、および他の多くの興味深いイニシアチブ。 フェルドマン氏の訪問の一環として、ソリューションアーキテクト向けの興味深いプロジェクトと教育プログラムについて話し合いました。









大ve日には、在庫を取るのが習慣です。 2016年にEPAMで開催されたさまざまな業界の興味深い革新的なプロジェクトは何ですか？



イノベーションプログラムの一環として、仮想現実や拡張現実などの分野、特に新世代のユーザーインターフェイスを中心に、「ガレージ」で多くの取り組みを行っています。 また、ダイアログインターフェース、チャットボットなどの実際のインターフェースも使用します。 顧客の1人は有名な石油およびガス会社であり、石油生産のための鉱山の3D計画の準備など、多くのハイテクソリューションを作成しています。 このようなプロジェクトにより、石油生産プロセスをさらに1ステップ改善することができます。



現在、モノのインターネット（IoT）の分野で革新的なプロジェクトがかなり多数あります。 当社は、さまざまなビジネスイニシアチブを単一のエコシステム、いわゆるコネクテッドビジネスに変換するソリューションを開発しています。 IoTプロジェクトの一環として、小売業、接客業、チケット販売（Ticketmaster Entertainment、Inc.）などのさまざまな業界のクライアントと協力しています。



過去数ヶ月間、私たちは自動車産業に注力してきました。 私たちは、普通の車を未来のスマートモバイルデバイスに変えるためのまったく新しいコンセプトのプロトタイプを熟考し、作成します。

もちろん、例外なくすべてのプロジェクトには、ソリューションアーキテクト（SA）の参加が必要です。



現在、世界のソリューションアーキテクトの主な傾向は何ですか？



まず、ソフトウェアアーキテクチャの開発に対する新しいアプローチについて話します。 これはすべて、マイクロサービスと、アーキテクチャに対する他のモジュール式アプローチに関連しています。 これはバックエンドについて話しています。 フロントエンドテクノロジーに関しては、ユーザーがアプリケーションの最も重要な側面のみを使用できるように、アプリケーションの作成時にモジュール性の原則を使用しようとしています。



サーバーレスアーキテクチャ、ラムダアーキテクチャの非常に大きな傾向があり、ソリューションアーキテクチャの現代の領域をますます満たしています。 もちろん、これはPlatform as a Serviceの単なる側面であり、新しいプラットフォームをゼロから構築するのではなく、個々のブロックからアプリケーションを作成できます。



EPAMソリューションアーキテクトが興味を持つものは何ですか？



EPAMでは、多数の大規模で複雑なプロジェクトに取り組んでいます。 中央および東ヨーロッパについて言えば、建築家がEPAM以外でこの規模のプロジェクトに取り組むことができる可能性はほとんどありません。



建築家が成長して次のレベルに進むには、「正しい」プロジェクトの段階を経る必要があります。 私たちは、最先端の技術と技術的な製品開発だけでなく、「正しい」現代の顧客についても話し合っています。 私たちは、市場で勝ち、将来クライアントを引き付ける方法を考え、理解し、成功を収めたい一方で、目標を達成するためにEPAMと協力している顧客について話します。



あなたの理解で理想的なソリューションアーキテクトは何ですか？ そのようなことはありますか？



私の理解では、理想的なアーキテクトは、彼が完璧ではないことを理解し、次のより高度なレベルに到達するために絶えず勉強している人です。 この人は、知っているほど知識が少なくなることを理解しています。



EPAMには、ソリューションアーキテクトが現在どのような知識が関連しているかを理解するのに役立つ教育プロジェクトがありますか



一方では、それらの多くがありますが、他方では、それらが十分ではありません。 現在、ソリューションアーキテクチャに関する4つの主要なプログラムがあります： Architecture Excellence Initiative 、 Solution Architecture University 、 Solution Architecture School 、 Solution Architecture Mentoringです。



Architecture Excellence Initiativeは、2014年から実施されています。 このイニシアチブの目標は、建築家に新しい知識を毎週提供することです。



Solution Architecture Universityは、主にソリューションアーキテクト間のチームワークの提供に重点を置いています。 このプログラムにより、建築家は同じ言語を話すことができます。 まず第一に、それはアーキテクチャにあるべき一般的な用語と概念のセットであるため、私たちが実装しているものの観点とプロジェクトへの複数のアーキテクトの参加の観点の両方で効果的です。 一方、アーキテクトが特定のタスクのための革新的なソリューションを見つけることができるのはチームワークです。 Solution Architecture Universityは、自分で用語の単一の語彙を作成し、チームでの作業方法を学びたいレベル1および2のアーキテクト、および将来自分自身をソリューションアーキテクトと見なす人向けに設計されています。



2016年に開始した新しいプログラムは、 Solution Architecture Schoolと呼ばれます。 その主な目標は、建築家になれる、またはなりたいと願う社内の専門家を見つけ、教育、チームワーク、動機付け、メンタリングを通じてこのプロセスを加速することです。 ウクライナでの過去6か月間に、4つのソリューションアーキテクチャスクールが合格しました。 システムアーキテクチャ、品質アーキテクチャ、ドメイン固有アーキテクチャなど、他のアーキテクチャ分野向けに同様のプログラムを構築する方法についても検討しています。



また、2016年の最後の四半期に、 Solution Architecture Mentoringを再開しました。 この方向は、上級レベルの専門家を育成し、最高レベルのソリューションアーキテクチャであるCTOレベルに進む機会を提供するために基本的に重要であると考えています。







歴史的に、ウクライナのソリューションアーキテクトの数は少ないです。 EPAMでは、状況を修正するために何をしていますか？ EPAM内で従業員を教育し、それによってすべての目標が達成可能であることを証明します。



あなたは男に何をアドバイスすることができます-さらなる教育のためのソリューションアーキテクト。 おそらく、いくつかの重要な会議、書籍、ビデオチュートリアル、ブログ、ポッドキャストですか？



上記のすべて。 独学は非常に用途が広いはずです。

ポッドキャスト、私たちが提携している分析会社のウェビナー（ガートナーとフォレスター）、大手テクノロジー企業（グーグル、フェイスブック、ツイッター、Netflix）のウェビナーは確かに有用です。
