D-Link Dir-620に基づくアマチュアAPRS用のAPRXクライアント

すべての人に挨拶！ この問題をカバーするように私に頼んだ人に特別な感謝。

APRSに関するいくつかの記事を作成する予定です。

1）Windows、Linuxに基づいて固定クライアントをインストールします（ローカルサーバーを作成することもあります）。

2）ラジオ局の接続。

3）モバイルクライアントのインストール。



本日の記事（ペンテスト）では、Dir-620ルーターにライトクライアントをインストールすることを検討します...

APRSとは何ですか？



APRS （自動パケットレポートシステム）は、1992年にBob Bruning（WB4APR）によってプロトコルが登録されたパケット無線通信の特別なバージョンです。 現在では、世界中の7000以上のハムで使用されています。 このプロトコルを使用すると、物体の位置または測定された物理パラメーターに関する情報を、技術的なデバイスを使用して長距離で送信できます。 特別なソフトウェアを使用して、情報を視覚化して処理できます。



いくつかの情報はウィキペディアで見ることができ、詳細はロシア語フォーラムで読むことができます。



KeeneticでフラッシュされたD-Link Dir-620でのAPRXゲートウェイのインストールと簡単な構成を検討してください。

ルーターのアップグレード方法と必要なパッケージの収集方法については説明しません-これは既にHabréで何度も取り上げられています（以前の記事をご覧ください ）。



クライアントを起動するには、データの構成を編集する必要があります。

#vi /etc/aprx.conf







私の設定：

#

mycall *******

#

login *******

server rotate.ap**.n** 14580





#

pidfile /media/DISK_A1/system/var/run/aprx.pid

rflog /media/DISK_A1/system/var/log/aprx/aprx-rf.log

aprxlog /media/DISK_A1/system/var/log/aprx/aprx.log





# 20,

beaconmode aprsis

cycle-size 20m

beacon symbol "R&" lat "4925.57N" lon "03203.98E" comment "DE UR3CKR --- APRX on OPENWRT"









すべてのパラメーターのより詳細な構成は、作成者の Webサイトに描かれています。

以下を開始します。

#/usr/sbin/aprx -f /etc/aprx.conf
