更新プログラムのロールアップ6をインストールした後のSCVMM管理コンソールの問題

System Center 2012 R2 Virtual Machine Manager用の更新プログラムのロールアップ6をインストールした後、奇妙な問題が発見されました-仮想マシンを作成しようとしたとき、またはテンプレートまたはハードウェアプロファイルのプロパティを見るとき、SCVMM管理コンソール（VmmAdminUI.exeプロセス）がSystem.Reflection.TargetInvocationExceptionエラーでクラッシュし始めました。









ドイツ人はセルフサービスポータルsocial.technet.microsoft.com/Forums/ru-RU/eab3a316-1cd3-450b-963d-d97be94cf395/scvmm-2012-で同じ問題を抱えていたため、これはある種のローカリゼーションの問題のようです。 r2-ru-6-self-service-user-konsole-strzt-ab



ネイティブ言語フォルダーの名前を変更（または削除）することで解決します
C：\ Program Files \ Microsoft System Center 2012 R2 \ Virtual Machine Manager \ bin


たとえば、「ru_」の「ru」。 この名前変更後、コンソールは英語のインターフェースで起動し、すべてが正常に機能します。



itbandを使用している人たちが、System Center製品は常に英語でインストールする必要があると言っているのは無駄ではないようです。 残差の原則に従ってローカライズされているため
