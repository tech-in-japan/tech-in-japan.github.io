Microsoft USAでの練習（履歴書と電話インタビュー）







この記事を「マイクロソフトで働きたい」と言いたいのですが、このタイトルは完全に正直ではありません。特に何らかの理由で、私は現在マイクロソフトで働いていないからです。 それでも、練習（インターンシップ）と職場の検索に対するアプローチの違いは定性よりも定量的です。つまり、同じ手順を実行する必要があり、（可能な場合）より深い知識を示すだけです。 したがって、私が書いたことは、そこでそこで練習をしたい人と、マイクロソフトで開発者として働くことに興味がある人の両方にとって、等しく興味深いものになると思います。



最初に、1つの記事で、プロセスに関する個人的な印象と、Habrahabrの読者への実践的なアドバイスを組み合わせようとしましたが、それは何の役にも立ちませんでした。 そのため、記事自体で、得た経験のみを説明し、ネタバレの下での印象を取り除こうとしました。





パート1：一般的な事実、書類の準備、電話インタビュー。

オプションのプレリュード
私がどのように文書を提出し、マイクロソフトでの実践のためのインタビューの準備をしたかについての実際の話をする前に、いくつかの詳細を明確にする必要があります。

私は教育による小児科医です。 より正確には、私は2006年に彼だけでした。 ロシアの医科大学を卒業し、ドイツの理論的な大学院を卒業した後、ドイツでの医師の卒業証書が認められず、科学の仕事は何とか私にはまったく魅力的ではなかったので、職業の選択に再び直面しました。

ドイツは完全な官僚制の国です。 したがって、専門教育を受けずにあらゆる分野でここで働くことは、長年の経験があっても常にリスクです。 まず第一に、これは給料のレベルに関係します。資格に関係なく、指導的地位をとることが事実上不可能であるように、教育なしで特定の「天井」を超えることは困難です。 まあ、多かれ少なかれ大企業（Microsoftを含む）はしばしば卒業証書なしで候補者を考慮さえしません。

したがって、すべての長所と短所を比較検討した後、この分野で専門的な高等教育を受ける時期であると判断し、ハンブルク工科大学の最初の学期にサインアップしました。



勉強する


ドイツでは、すでにここで書いた高等教育システムは、10年以上も勉強できるように設計されていることに注意してください。 そのため、新入生にとってすでにかなり堅実な年齢にもかかわらず、原則として、私は他の学生の間であまり目立ちませんでした。 私のコースには、私よりも年上で私と同年齢の人々がおり、ここでも再訓練または再​​訓練を行っていました。

すべての科目がきめ細かく描かれ、各学期のスケジュールが学部長のオフィスでまとめられているロシアの医科大学とは異なり、ドイツの工科大学にはスケジュールがまったくありません。 「小さな卒業証書」（ヴォルディプロム）を取得し、卒業証書を取得するために合格しなければならない科目のリストがありますが、試験に合格し、科目に合格する手順は、実際には何の役割も果たしません（まれな例外を除く）。 すでに、habrのトレーニングについて詳しく説明しました



マイクロソフトの学生プログラム。


3年生（5学期）として、私は最初に誤ってMicrosoft Student Partnersプログラムに精通しました。

マイクロソフトの学生プログラムは国によって大きく異なるため、すべてのボーナスを詳細に説明するのは意味がありません。ロシアのプログラムの詳細については、公式Webサイトをご覧ください 。 一般的に、学生パートナーは、さまざまな種類のレポート、ワークショップ、自分が一番好きなエリアでの「ハッカソン」を開催し、書籍、試験バウチャー、および多数の個人ボーナス（会議への参加など）で独学する機会を得ます。

しかし、私の意見では、これはプログラムで最も重要なことではありません。 その主な利点は、マイクロソフトの従業員と直接連絡を取ることができることです。技術的なエバンジェリストやマネージャー、およびマイクロソフトテクノロジーに興味がある他の大学の学生と密接に連絡を取ることができます。 私の意見では、2つ目はさらに価値があります。なぜなら、「ピア」から多くのことを学ぶことができるからです（年齢ではなく、技術に対するステータスと関心の観点から）。 さらに、ドイツでは大学生に関してバイアスはありません（ここの学生はかなり長期的なステータスになる可能性があるため）：一部の学生パートナーは、活動に関してMicrosoft MVP（Most Valuable Professional）ステータスも持っています。 そして、あなたと一緒に勉強している人々が何を達成したかを見ると、それはあなた自身をさらに発展させ、取り組むための非常に動機付けになります。

個人的な例による動機付けに加えて、私にとって大きな動機付けとなったのは、アメリカのマイクロソフトで働くために勉強した後、多くの元「学生パートナー」が去ったという事実でした。 開発者にとって、これはマイクロソフトソフトウェア開発に携わる唯一の機会でした。ドイツではマイクロソフトが製品のサポートとコンサルティングにしか関与していないからです。 もちろん、米国での仕事には独自の特徴がありますが、多くの企業（Microsoftを含む）の主要な開発者がそこに集中しているという事実を否定することは困難です。

現在、Common Language Runtimeグループで働いている元学生パートナーの1人のアドバイスで、私はアメリカのマイクロソフトで最初に練習をすることを勧められ、練習の結果に基づいて、この仕事が私とこの国に適しているかどうかを判断しました。 したがって、私の戦略目標「マイクロソフトで働きたい」は、戦術的な「マイクロソフトで練習したい」に変わりました。





練習のための文書の準備と提出。



まず、米国のマイクロソフトでの業務は非常に権威があり、非常に大きな競争が行われていることを（何とか考えていなかった前に）知りました。 ほとんどの学生インターンは、Microsoftによって米国から直接募集されます。 外国人学生については、信頼できる情報源から受け取った「インサイダー」情報によると、毎年米国外からマイクロソフトに送られる約5,000件の履歴書から、約20〜30人が練習に招待されています。 つまり、選択は約100分の1になり、もちろんロシアの主要大学での競争に劣りますが、それでもかなり印象的です。

公式の情報源から-特に、 申請サイトの詳細な調査の後-学生のための米国での実践が2つの方法で可能であることがわかった：



マイクロソフトリサーチインターン

マイクロソフトビジネスインターン

このタイプの実践は、科学に従事する大学院生と「科学」インターンシップを希望する「贈与された」学生を対象としています。 主なことは、Microsoft Researchで既に働いてトピックについて話し合う可能性のある実践リーダーを見つけることです。 ほぼ一年中練習に申し込むことができます；申込書の提出期限はありません。 アプリケーションが成功した場合、練習はこの科学者が働くグループで開催されます。 研修生の仕事は、論文の主なトピックに直接関係してはいけません。

3つの可能な役割のいずれかでの5月から9月（この間隔から12週間）までの3か月間の練習：

  • テスター（SDE-T） 、自動テストの開発と作成
  • プログラムマネージャー（PM） 、プログラマーとテスターの作業の編成、仕様の作成、機能エンジニアリングの処理など。

  2. 履歴書に基づいて候補者を選択した後、候補者は電話インタビューを受け、成功した場合は、
  3. 研修生を必要としたグループの代表者との対面インタビュー。




最後のフルタイムインタビューが正常に完了した場合、候補者は練習に招待されました。 ヨーロッパからの候補者に対する対面インタビューは、ダブリン、ワルシャワ、ミュンヘンのマイクロソフトの「ヨーロッパオフィス」で行われます。 私が理解しているように、これらのオフィスでは、年に一度-2月末頃-3月初旬-インタビューを行い、インターンシップを無事に終えた人を招待する米国のMicrosoft社員が上陸します。



インタビュー-知っておくべきこと




マイクロソフトでのインタビューがどのように行われたかについての情報を見つけることは、結果の海に過ぎません。このテーマに関する別のウィキペディアの記事さえあることが判明しました。 インターネットから収集したすべての論文を提出すると、次のようなものが得られます。

  • 会社としてのマイクロソフトは、インタビューに対する並外れた態度で知られています。
  • 練習のためのインタビュー自体は、就職の面接とほとんど変わりません。 複数の従業員が実施し、候補者のコンセンサスを得る必要があります。 面接を採用する場合、部門の4〜5人の従業員が面接を行いますが、面接官のレベルはその後の面接ごとに増加します。つまり、通常のプログラマが面接を開始すると、部長またはアーキテクトが面接を終了します。 面接が「予定より早く」終了し、2回の面接の後、候補者が帰宅した場合が多くあります。
  • 初期のころ、インタビューは会社の創設者であるビル・ゲイツによって個人的に監督されていたため、インタビュー中にさまざまなパズルの質問（脳の体操）を頻繁に尋ねましたが、これは従業員が関与するタスクに常に直接関連するわけではなく、インタビュー対象者から何らかの学識と工夫が必要でした。
  • パズルの質問に加えて、よくあるタイプの質問は、Fermatの問題として知られる評価質問であり、その本質は特定のパラメーターを評価することであり、その正確な値はインタビュー対象者に知られていませんが、よく知られた事実といくつかの仮定に基づいて正確に推定できました（「ニューヨークには下水道マンホールがいくつありますか？」、「シカゴにはピアノチューナーがいくつありますか？」、「スクールバスにいくつのゴルフボールが収まりますか？」）。
  • 近年、Microsoftはそのような質問の実施を公式に放棄しました;そのような質問の可能性は、特に追加の質問として、まだ100％除外されていません。




ただし、面接を受けるには（電話でも）、最初の段階を経て-履歴書が必要です。



準備を再開します。




私の場合、履歴書の準備と校正のプロセスには約2か月かかり、最終バージョンまで約25回繰り返しました。 マイクロソフトの従業員数名、アメリカで働いている友人数名、アメリカで10年以上科学研究を書いた大学教授、ドイツ人の友人数名が私の履歴書を編集してくれました。 各版の後に、履歴書は非常に多く再描画されたため、以前のバージョンを編集した後に残った部分から実際に再コンパイルする必要がありました。 最終版はここで見ることができます 。



MicrosoftのWebサイトで練習を申し込む場合、「既存の履歴書をインポートできるが、それ自体がフォーマットと必要な情報を決定する「コンストラクタ」を使用する必要があるため、この作業はほとんど冗長だったとすぐに言わなければなりません。 。



多数の反復の結果として、私は米国での実践のための履歴書を書くための次の原則を得ました。

  • 写真に基づいて候補者について「偏った」決定を下すことができるため（たとえば、彼の肌の色、肥満、人種などを判断できる）、訴訟の根拠になる可能性があるため、写真を米国の履歴書に挿入することは強く推奨されません。
  • 同じ理由で、候補者の年齢と生年月日は履歴書に記載されていません。 性別もオプションですが、名前でよく理解できます。
  • 可能であれば、評価を控え、事実を説明するようにしてください。 大学で勉強する場合、実践または研究室の枠組みでどのテクノロジーを使用したかを示すと便利です。 会社での経験がある場合は、技術と習得したスキルを示すことも非常に良いことです。
  • テクノロジーを説明するときは、できるだけ詳しく説明するようにしてください。 たとえば、「データベースプログラミング」は、「MS SQL Server 2008 R2上のデータベースを操作するためのSilverlightクライアントアプリケーションの開発」よりもずっと悪いように思えます。
  • マイクロソフトの学生パートナーである場合は、履歴書に必ず明記してください。 公式に確認されていない（ただし、誰にも反論されていない）情報によると、そのような履歴書は常に電話インタビューの段階に達します。 私がこの段階に達した後に履歴書を送ったすべてのパートナーの友人。
  • あなたがどんなに英語が上手であっても、少なくとも2、3人のアメリカ人、または極端な場合、あなたよりも英語を上手に話す人々に校正のための履歴書を常に与えるべきです。 言語エラーがなくても（必ずしも文法的なエラーではない-いくつかのことは単に英語で「聞こえない」）、これはデザインのタイプミスや欠陥（ドット、コンマの欠落）を見つけるのに役立ちます。


履歴書は候補者が最初に除外するものであり、履歴書の段階で90％が除外されることを覚えておくことが常に重要です。 したがって、すべてが完全に揃っていることが非常に重要です。



電話インタビューの準備




履歴書に基づいて事前に選別された候補者の場合、次のステップは電話インタビューです。

インターネットからの準備と電話インタビューの準備のためのさまざまなマニュアルに基づいて、このインタビュー中にインタビュアーは以下の問題を解決します。

  • まず第一に、インタビューは英語力のレベルを評価することを目的としています。 もちろん、米国や英語圏以外で人生を生きてきた人から、彼が完璧に話すことを期待する人はいません。 ただし、自由なレベルの言語能力、自分の考えを明確かつ明確に述べ、インタビュアーの質問を理解し、答えを推論できる能力が必要です。 Programm Managerの役割については、SDEよりもうまく話す必要があると彼らは言いますが、実際にはこれを確認できませんでした。
  • 候補者の英語レベルが十分であれば、インタビュアーは対談者の全体的な妥当性を評価します。 ここでは、いわゆる「レッドフラッグ」に特別な注意が払われます。これは、候補者の（一部の）不十分さや偏見を示している場合があります。 これらの信号は次のとおりです。

    1. 彼らの役割の不適切な評価（過去のプロジェクトを説明する際のar慢または慢、「私が来てすべてをやるまで何もできなかった」、「私だけが知っていたので、すべては私だけにかかっていた」など） 、
    2. 完了したプロジェクトの本質を明確かつ明確に述べることができない（過度に細心の注意を払った（「ドライバーからの情報にアクセスするためにkernel32.dllでPInvokeを使用した」）か、過度の一般化（「Javaでデータベースを開発していた」））





候補者は、自分の活動と完了したプロジェクトについて多かれ少なかれ批判的な評価と、「あなたはどうしますか？」という質問に対する比較的正直な答えを持つことが期待されています。 同時に、答え自体はそれほど重要ではありません（たとえば、「まったく同じことをした」と冷静に答えることができます）が、その議論（「別のケースでは次の問題があると思うので...」）



どうやって手に入れたの？
12月上旬に履歴書を作成し、すでに述べたサイトcarreer.microsoft.comに送信した後、1月末までマイクロソフトから何も聞きませんでした。 ミュンヘンで2月に対面インタビューが行われることになっているという事実（これはインサイダー情報でした）を考えると、徐々に履歴書の運命について心配し始めました。 私は、拒否がまだ私に届いていないという事実によってのみ安心しました。

1月30日、ようやくアメリカの大学の採用担当者から手紙が届きました。これは私の履歴書が選考の第1段階を経て、30時間の電話インタビューの時間を選ぶことを申し出たことを祝福しました。 アメリカとドイツの時差を考えると、インタビューは午前11時から午前4時の間にあると想定されていました。 とにかく、インタビューが夜遅くなら眠れなかったので、11泊を選びました。



回答書で、採用担当者は面接の時間を私に確認し、面接者の名前も伝えました。 リクルーターは勤務先の会社を明示的に示していないにもかかわらず、Facebookでの短い検索はすぐに成功しましたが、グレーターシアトルエリアの名前と勤務地が明確に彼を特定しました。 なぜ私がこれをしたのかはわかりません-おそらく私自身が本質的に視覚的で、知らない人と電話で話す必要があるとき、非常に不安を感じるからでしょう。 私の対談者が写真でどのように見えるかを見たという事実は、すでに私を大いに安心させました。



また、手紙では、段落全体が赤で強調されており、インタビュー中に小さなプログラミングの問題を解決するように求められる可能性があり、電話でリクルーターにその決定を指示する準備ができているはずだと述べました。 コンピュータを使用できるかどうかについては、何も言われていません。



本「 Cracking the Coding Interview 」のアドバイスに従って（準備の詳細については次のパートで説明します）、履歴書を作成し、各行に履歴書（研究、仕事、練習）の項目を含む大きなテーブルを準備しました。採用担当者の質問に対する可能な回答を簡潔に（2〜3語で）説明しました。



最も可能性の高い6つの質問を選択しました。

  • プロジェクトまたは仕事に対する私の個人的な貢献は何でしたか。
  • 最も興味深く、重要で、面白く、有益であったものなど
  • プロジェクト、研究、または実践の結果として達成された結果。
  • プロジェクト中にどんな問題に遭遇しましたか？
  • 他のプロジェクト参加者とのコラボレーションはどうでしたか。
  • 私の意見では、私の現在の経験を考えると、私はどう違うでしょう。
