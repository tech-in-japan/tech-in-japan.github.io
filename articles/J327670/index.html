cAdvisor、InfluxDB、Grafanaを使用したDocker Swarmの監視



実行中のアプリケーションのステータスを監視するには、常に監視する必要があります。 また、アプリケーションがDocker Swarmなどの非常にスケーラブルな環境で実行される場合、非常にスケーラブルな監視ツールも必要です。 この記事では、このようなツールのチューニングについて説明します。







このプロセスでは、各ノードにcAdvisorエージェントをインストールして、ホストとコンテナーのメトリックを収集します。 メトリックはInfluxDBに保存されます 。 これらのメトリックに基づいてグラフを作成するには、 Grafanaを使用します 。 これらのツールはオープンソースであり、コンテナとして展開できます。







クラスターを構築するには、Docker Swarmモードを使用し、必要なサービスをスタックの形でデプロイします。 これにより、動的監視システムを編成できるようになり、新しいノードがswarmに追加されると自動的に監視を開始できます。 プロジェクトファイルはこちらにあります 。







ツールの概要



監視システムの選択は非常に広いです。 スタックを構築するには、コンテナで適切に機能するオープンソースサービスを使用します。 次に、スタックの構成について説明します。







cAdvisor



cAdvisorはホストとコンテナーのメトリックを収集します。 ホスト上の共有ボリュームおよびルートファイルシステムとして接続されたドッカーソケットを備えたドッカーイメージとしてインストールされます。 cAdvisorは、InfluxDB、Prometheusなどを含むいくつかのタイプの時系列データベースに収集されたメトリックを記録できます。収集されたデータからグラフが構築されるWebインターフェイスも備えています。







Influxdb



リアルタイムのメトリック、イベント、および分析用のスケーラブルなストレージ。

InfluxDBは、数値メトリックを保存し、それらにタグを割り当てるオープンソースの時系列データベースです。 このシステムは、保存されたデータの操作に使用できるSQLに似たクエリ言語を実装しています。 ホストまたはコンテナでタグを使用してイベントをフィルタリングします。







グラファナ



Grafanaは、Graphite、Elasticsearch、OpenTSDB、Prometheus、InfluxDBから取得したメトリックに基づいてツールバーとグラフを作成できる多機能のオープンソースシステムです。

Grafanaは、Graphite、Elasticsearch、OpenTSDB、Prometheus、そしてもちろんInfluxDBからデータを取得することでツールバーを作成できる一般的な視覚化ツールです。 4番目のバージョンから、クエリ結果に基づいてアラートを構成できるようになりました。 特定のホストとサービスのデータを表示できるツールバーを作成します。







Docker Swarmモード



Swarm Mode



、バージョン1.12.0以降、Dockerに登場しています。 複数のホストから簡単に群れを作成し、簡単に管理できます。 サービスとオーケストレーションを検出するための組み込みメカニズムを提供するために、Swarmモードはキーと値のストレージを実装しています。 ホストは、マネージャーまたは作業ノードとして機能できます。 一般に、マネージャーはオーケストレーション機能を担当し、コンテナーは作業ノードで実行されます。 これはデモインストールなので、マネージャーにInfluxDBとGrafanaを配置します。







Swarm Modeには、仮想ロードバランサーとして機能する興味深いルーティングメッシュ機能があります。 5つのノードで実行されるポート80でリッスンする10個のコンテナーがあるとします。 これらのコンテナのいずれかの80番目のポートにアクセスしようとすると、要求はそれらのいずれかに送信され、別のホストで実行されている場合もあります。 したがって、任意のノードのIPアドレスを公開することにより、10個のコンテナー間のクエリバランシングを自動的に有効にします。

Grafanaダッシュボード







Grafanaにインポートされたツールバーは、Swarmのホストとコンテナーを監視するように設計されています。 ホストレベルとその上で実行されているコンテナにドリルダウンできます。 2つの変数が必要になります。それらをGrafanaツールバーに追加するには、テンプレートを操作する機能が必要です。 InfluxDBと組み合わせたテンプレートの操作に関する詳細は、 このページにあります 。 ノードを選択するホストと、 コンテナを選択するコンテナの2つの変数があります。 これらの変数を表示するには、ツールバーページで[ 設定]を選択し、[ テンプレート ]をクリックします。







最初の変数-host-では、ノードとそのメトリックを選択できます。 cAdvisorがInfluxDBにメトリックを送信すると、フィルタリングに使用できるいくつかのタグがメトリックに追加されます。 cAdvisorインスタンスのホスト名を含むmachineというタグがあります。 この場合、swarmのホストIDと一致します。 タグ値を取得するにはshow tag values with key = "machine"

使用します。







2番目の変数-container-は、コンテナのレベルまでドリルダウンできます。 container_nameという名前のタグがあり、コンテナーの名前が予想どおりに含まれています。 また、ホストタグの値でメトリックをフィルタリングする必要があります。 リクエストは次のようになります： show tag values with key = "container_name" WHERE machine =~ /^$host$/

変数を設定したら、グラフで使用できるようになりました。 さらに、 メモリグラフィックスについても説明しますが、残りについても同じ原理で作業できます。 メモリ関連のデータはInfluxDBのmemory_usage行にあるため、クエリはSELECT "value" FROM "memory_usage"



ます。







ここで、 WHERE句にフィルターを追加する必要があります。 最初の条件は、 machineが ホスト変数の値と等しいことです： "machine" =~ /^$host$/



。 次の条件では、 container_nameはcontainer変数の値で始まる必要があります。 ここでは、 コンテナ変数を最初のポイントにフィルターしたため、「starts with」演算子を使用します： "container_name" =~ /^$container$*/



。 後者の条件は、grafanaツールバーで選択された時間間隔$ timeFilterに従ってイベントに時間制限を課します。 クエリは次のようになります。 







 SELECT "value" FROM "memory_usage" WHERE "container_name" =~ /^$container$*/ AND "machine" =~ /^$host$/ AND $timeFilter


ホストとコンテナごとに別々の行が必要なので、 machineタグとcontainer_nameタグの値に基づいてデータをグループ化する必要があります。 







 SELECT "value" FROM "memory_usage" WHERE "container_name" =~ /^$container$*/ AND "machine" =~ /^$host$/ AND $timeFilter GROUP BY "machine", "container_name"


このクエリのエイリアス： Memory {host: $tag_machine, container: $tag_container_name}
