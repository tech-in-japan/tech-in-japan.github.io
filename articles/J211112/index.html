モジュラーサーバーの順序：個別のモジュール提案からサーバーを「収集」する





注文が非常に便利になりました。その理由は次のとおりです。



1.ディスクタイプ、トラフィック量、サポートタイプなど、個々の要素から注文を「収集」できます。



  • サーバーを注文すると、次のOSが提供されます： Centos、Debian、Freebsd、UbuntuまたはWindows Server Standart 、およびISPmanagerまたはDirectadminコントロールパネル）。


  • スライダーを使用して、作業に必要なIP数量を2から30まで注文できます。






  • サーバーの記憶媒体のタイプを選択する可能性もあります。 1〜4 TBのSATA WD 、128〜512 GBの Samsung SSD 、または300〜600 GBのSeagate SASのいずれかです。 同時に、任意のサーバーに、上記の任意のタイプとボリュームの任意の数のメディアをインストールできます。 デフォルトでは、4つのメディアから選択できますが、さらに必要な場合はテクニカルサポートに連絡してください。






  • RAIDのタイプも選択できます： LSI 9207-8i、Adaptec 6405 + BBU 、 LSI MegaRAID 9271-4i + BBU、Adaptec 7805 + BBU 。




  • 注文を選択する際、お客様は必要なタイプ/ボリュームのトラフィックを注文できます：固定パケットサイズ（10〜100 TB）から、特定の帯域幅の無制限のチャネルトラフィック（30〜1000 Mbit / s）まで。






2.最終価格が依存する専用サーバーのリース期間の柔軟な調整（以下の詳細を参照）。



3.任意の量のRAM、ハードドライブおよびトラフィックの量を注文する機能を含む、仮想サーバーの同様の注文設定。



4.前と同様に、 高速アクティベーションを備えたサーバーを提供します！ サーバーがサプライヤから配信されるまで待つ必要はありません。すべてがすでに在庫にあります。 あなたが注文すると、私たちのチームがOSを投入し、数時間後に完全に処分して注文を受け取ります。
