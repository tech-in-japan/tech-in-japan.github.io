Veeam Agent for Linuxを使用したバックアップからの復元

バックアップを作成する最終的な目標は、障害が発生した場合にデータを回復する機能を提供することです。今日は、新しいVeeam Agent for Linuxがデータを処理する方法について簡単に説明します。 前の投稿で作成について説明した「実験ウサギ」と同じバックアップを取り、それからどのように回復できるかを確認します。 シムのために猫へようこそ。











ファイルレベルの回復



ご存知のとおり、Veeam Agent for LinuxはバックアップをNFSサーバーに正常に保存しました。 コマンドを入力して、使い慣れたUIを実行します

veeam





クロック、つまり、バックアップジョブの最後のセッションに関するデータを確認します。









コマンドのリストの下部に、コマンドR（ファイルの復元） -ファイルの復元が表示されました。 使用可能なバックアップに関する情報が表示されます：動作中のホスト、バックアップを作成したタスク、判明した復旧ポイントの数（この場合は1つ）、およびいつ。









Enterキーを 2回押すと、選択したバックアップがホストのファイルシステムの/ mnt / backupフォルダーにマウントされます。









なぜこの操作に限定することにしたのですか？ ユーザーは通常、ファイルを操作する上でユーザー自身の好みがあると考えていたため、車輪を再発明する必要はありません。 そのため、インストールが完了した後、使い慣れた方法を使用できます。たとえば、コマンドラインや人気のあるMidnight Commander （ mc ）などです。









ボリューム回復



次に、ボリューム全体の回復がどのように実行されるかを見てみましょう。 最初に、Veeam Recovery Media（Veeam Agent for Linuxソリューションのインストールパッケージとともにダウンロード）を使用してマシンを起動します。 ISOファイルの使用を開始します。



Veeam Recovery Mediaは、次の一連のコマンドでグラフィカルインターフェイスを開きます。









ここでは、 ボリュームの復元 （ボリュームの復元 ）、ファイルの復元 （ファイルの復元 ）、ネットワーク設定（ ネットワークの構成 ）、コマンドライン（コマンドラインへの切り替え ）のほか、再起動（ 再起動 ）とシャットダウン（ シャットダウン ）を行うことができます。

この例のように（推奨されるように）、バックアップがローカルマシンに保存されず、ネットワークストレージシステムに保存される場合、リカバリ手順を開始する前に、バックアップストレージの場所にアクセスできることを確認し、手順中に設定を構成しますネットワーク。 設定は手動で構成できます。 これを行うには：



  1. このメニューで、[ ネットワークの構成 ]項目を選択し、リストからストレージへの接続に使用するネットワークアダプターを選択してEnterを押します。



  2. [ アダプターの構成 ]ダイアログボックスで、[ 手動 ]を選択してEnterを押します。



  3. [ アダプターの設定 ]ダイアログボックスで 、必要なIPアドレス、サブネットマスク、デフォルトゲートウェイ、DNSサーバーを指定します。



  4. [ 適用]をクリックしてEnterを押します。





    DHCPサーバーを使用している場合、Veeam Agent for Linuxは必要な設定を自動的に行います-[ アダプターの構成 ]ダイアログで[ 自動]が選択されている場合。



  5. ボリュームの復元を続行します。コマンドのリストから適切な操作を選択します。これはボリュームの復元です。



  6. 次に、 [バックアップの場所の選択]ステップで、 バックアップの場所を指定する必要があります。 この例では、ボールを追加するオプションを選択する必要があります共有フォルダーを追加...



  7. 次に、 共有フォルダーのマウントのステップで、次のNFSがあることを示します。









  8. [ サーバー/ディレクトリ]フィールドに、バックアップファイルがあるネットワークボールの名前を入力します。 Veeam Agent for Linuxは、リカバリイメージのファイルシステムの/メディアフォルダーにマウントし、マウントされたボリュームの内容を表示します。 [ バックアップファイルの参照]ステップで、目的の復旧ポイントを選択してインポートできます。









    有用：バックアップがローカルデバイスの1つに保存されている場合、 [バックアップの場所の選択]ステップで、 [ ローカルディスクのマウント ]オプションを自然に選択します。 この場合、バックアップファイルが存在する複数のデバイスに対して、何度もマウントすることが可能になります。 これを行うには、 [バックアップ場所の選択]バックアップ場所の手順に戻り、 [ ローカルディスクのマウント ]オプションを再度選択します。



  9. [ バックアップ]ステップで、目的のバックアップとその中の復旧ポイントを選択します。



  10. 次に、 ディスクマッピングのステップで、本番（つまり、ローカルホスト- 現在のシステム ）およびバックアップにあるマシンのボリュームを確認できます。 Veeam Agent for Linuxは、選択したボリュームの詳細情報（パーティションの種類、ファイルシステム、マウントポイントの場所、ボリュームサイズなど）を表示し、使用可能なコマンドのリストも表示します。



    • ボリュームを復元 -バックアップからこのボリュームを復元します。



    • パーティションの削除 -ボリュームを復元する前にディスクのパーティションを再作成できます。 パーティションを削除した後、新しいパーティションを作成し、バックアップからボリュームをマッピングできます。



    • [LVMボリュームを復元するには] LVM物理ボリュームの作成-選択したパーティションに物理LVMボリュームを作成し、既存のボリュームグループ（ボリュームグループ、VG）に追加するか、新しいグループを作成します。 これにより、選択したVGにLVM論理ボリュームまたはボリュームグループが復元されます。



    • 閉じる -ダイアログを閉じて、別のボリュームを選択します。


    ここで、[ ボリュームの復元元]を選択してEnterを押します。



  11. [ 現在のシステム ]パネルの、選択したボリュームの反対側の[ 復元]フィールドに、 復元元のボリュームの名前が表示されます。









  12. 選択を確認します（注意してください、データはバックアップ内のデータによって上書きされるため、監視は高価になる可能性があります！）、 （復元を開始 ）をクリックします。



  13. 簡単な要約を見て、もう一度選択を確認し、進行状況を観察します。











プロセスを完了した後、Veeam Recovery Mediaの使用を終了します。



  1. Escキーを押して、メインメニューに戻ります。
  2. リカバリイメージのあるメディアを切断します。
  3. メインメニューで、[ 再起動 ]を選択してEnterを押します。
  4. OSの起動を待っています。


ここでは、一般的に、Veeam Agent for Linuxを使用してリカバリがどのように発生するかについての全体的なストーリーがあります。
