正規表現とは誰ですか？



簡単に言えば、これはテキストを検索するように設計されたプログラミング言語です。 非常にシンプルなプログラミング言語。 条件は実際には適用されず、サイクルと機能はありません。検索するテキストを記述する式は1つだけです。 しかし、これは非常に長い可能性がある唯一の式です:)。 一般的に、特にPythonで正規表現を正常に使用するには、いくつかのことを知る必要があります。 まず、すべての自尊心のある正規表現ライブラリは、これらの正規表現に独自の構文を使用します。 一般に、構文は似ていますが、詳細と追加機能は大きく異なる可能性があります。そのため、Pythonで正規表現を使用する前に、 公式ドキュメントの構文に慣れる必要があります 。

第二に、正規表現は言語自体の構文とユーザーデータを分離しません。 つまり、「Vasya」という単語を検索する場合、それを検索するための正規表現は「Vasya」のようになります。 プログラミング言語自体はこの行には存在せず、指定した行のみが存在するため、検索する必要があります。 しかし、「Vasya」という単語の後にコンマまたはセミコロンが続く場合、正規表現は必要で重要な詳細「Vasya、| Vasya;」に囲まれます。 ご覧のように、縦棒で表される「論理的または」言語の構成がここに追加されました。 同時に、設定した行は言語の構文から分離されていません。 これは重要で不快な結果につながります-言語の構文に存在する検索文字列に文字を指定する場合、その前に「\」を記述する必要があります。 したがって、ピリオドまたは疑問符が後に続く単語「Vasya」を検索する正規表現は、「Vasya \。| Vasya \？」のようになります。 ピリオドと疑問符の両方は、正規表現言語:(の構文で使用されます。

第三に、正規表現はデフォルトで貪欲です。 つまり、これが具体的に示されていない限り、正規表現を満たす最大長の文字列が見つかります。 たとえば、テキストで「name = value」という形式の文字列を検索し、正規表現「。+ =。+」を記述したい場合、テキスト「a = b」では「a = b」を返すことで正しく動作します。 しかし、テキスト「a = b、c = d」の場合、「a = b、c = d」、つまりテキスト全体が返されます。 正規表現のこの特性を常に覚えて、検索の結果としてライブラリが「戦争と平和」の半分を返す誘惑を持たないような方法でそれらを書くべきです。 たとえば、前の正規表現を少し変更する必要があります： "[^ =] + = [^ =] +"-このバージョンでは、記号「=」の前後のテキストに記号「=」が含まれていないことを考慮します。
