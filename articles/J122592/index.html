WordPress 3.2：新機能

WordPress 3.1と同様に、最新のWordPress 3.2の最新情報をお届けします。 WordPress 3.1のリリースが遅れた後、開発者の目標は、新しいバージョンのリリース速度を上げることでした。 Mark Jaquithはこれを個人的に管理して、チームの全員ができるだけ早く問題を解決することに集中できるようにします。 それでは、WordPress 3.2で何が面白いか見てみましょう。

記事の執筆





記事の執筆者が記事を書くこと以外に気を取られないという概念は、かなり前に生まれました。 WordPress 3.1では、デフォルトで追加フィールドやその他のフィールドを非表示にしているため、投稿を書くためのページが若干簡素化されています。 WordPress 3.2では、「Free write」モードを使用すると、インターフェイスをフルスクリーンにしてテキストを完全に降伏させることができます。 目標は、さらに簡単で落ち着いたインターフェースでした。







トゥエンティーイレブンのテーマ





WordPress 3.0には、Twenty Tenという新しいデフォルトテーマが付属していることを覚えていますか？ そのため、WordPress 3.2では、新しいデフォルトテーマのTwenty Elevenを発表する予定です。 これは、WPに付属する標準テーマを多様化するためであり、Kubrickで状況を繰り返すことはありません。 知らない人のために、キューブリックは長い間デフォルトのテーマでした。 新しいテーマは2色で表示されます。











新しいTwenty Elevenテーマには、配色と背景を変更できる新しい設定ページもあります。 （ええと、これは有料のトピックだと思うかもしれませんが、それは絶対に無料です!!!）-WordPress UIチームを尊重します。







新しいページテンプレートも作成されました。 デフォルトでは、サイドバー（サイドバー）のないテンプレートが使用されます。 サイドバー付きのバージョンと、スライダー付きのバージョンもあります。 ローカルコンピューターでテストするときにバグがあったため、まだスクリーンショットをアップロードしていません。



最も予想されるものの1つは、ヘッダー（ヘッダー）のTwenty Elevenテーマに画像を追加することでした。これにより、各ページにランダムな画像を表示できます。 多くの人が以前のバージョンでこれを望んでいた。 これは非常に便利な機能のようです。



自動更新





原則として、現在はこの機能に問題はありませんが、対処する必要がありました。 新しいバージョンでは、自動更新により、変更が発生したファイルのみがダウンロードおよび更新されます。 これにより、低コストのホスティングを使用するユーザーの自動更新が間違いなく高速化されます。



スピード





現在、WordPressはカーネルレベルでいくつかの古いテクノロジーをサポートしていますが、新しいバージョンではサポートされないため、より高速になります！ NacinはPHPの遅いロードで動作します。 管理パネルの読み込み速度の向上、およびより高速な更新のためのFTPの改善としての作業。



PHPおよびMySQLの新しい要件





WordPress 3.2にはPHP 5.2.4+およびMySQL 5が必要です。したがって、バージョン3.2にアップグレードする前に、ホスティングがこれらの標準をサポートしていることを確認してください。 WordPressの品質ホスティングは既にサポートしています。 HostGatorでホストすることをお勧めします 。 ホスティングがWPの新しいバージョンをサポートしているかどうかを知りたい場合は、 WordPress要件チェックプラグインを使用します。 このプラグインは、ホスティングが新しいバージョンに対応しているかどうかを通知します。 そうでない場合は、ホスティングの管理者に事前に依頼して新しいリリースの準備をするか、ホスティングの変更を検討してください。



ナイトリービルドをダウンロードすることで、これらすべてをテストできます。 革新は刺激的です。 計画によると、WordPress 3.2は2011年6月30日頃にリリースされる予定です。
