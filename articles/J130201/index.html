Androidエミュレーターを400％高速化する方法

過去数か月の間、Nuxeoコンテンツ管理プラットフォームの一部であるAndroid SDKに取り組んできました。 特に最近は、SDKの公式リリースのアプローチにより、多くの作業が行われました。 Android開発、特にテストとエミュレーションに関する実用的なヒントを共有したいと思います。 数日間の開発の後、Androidエミュレーターの操作は非常に遅いため、砂糖ではないことに気付きました。



そもそも起動に時間がかかりますが、状態スナップショットを使用して起動を高速化しても、特にデバッグモードでは、コードの実行は依然として低速です。



Qemuのパラメーターの微調整はあまり役に立ちませんでした。エミュレーターの速度は依然として遅いため、ほとんどの開発者はUSBで接続された実際のデバイスを使用することを好むでしょう。



いずれにせよ、私は解決策を見つけたかったのです。なぜなら、アプリケーションの結果をプロジェクターで表示したいので、アプリケーションの表示が遅くなることを望まなかったからです。



エミュレーションとシミュレーション



Appleのプラットフォーム向けのソフトウェアは開発していませんが、iPhoneシミュレーターにはAndroidエミュレーターのようなパフォーマンスの問題はないようです。



その理由の1つは、「実際のシミュレータ」として機能しないことです。 iPhoneシミュレーターで使用されるCPU命令は、シミュレーター（x86）を実行しているコンピューターの場合と同じです。



一方、Androidは、x86プロセッサの上に実際のARMアーキテクチャプロセッサをエミュレートすることを余儀なくされています。 これにより、パフォーマンスの面でかなり大きなオーバーヘッドが追加されます。



少なくともアプリケーションのデモを行うために、ARMをエミュレートする必要はありません。Androidとアプリケーションを実行する機能のみが必要です。



Android_x86



幸いなことに、Androidをx86プロセッサに移植するためのオープンソースプロジェクトがあります： http : //www.android-x86.org/ 。



プロジェクトにはダウンロード用の画像がいくつかあり、Android SDKの公式リリースのすべてに対応しているわけではありませんが、Android 2.3の起動可能な画像を提供してくれました。



VirtualBox内のAndroid_x86セットアップ



最初のステップ：Android_x86 ISOイメージをダウンロードします。 私はandroid-x86-2.3-RC1-eeepc.iso



を使用しました。これはhttp://www.android-x86.org/downloadからダウンロードできます 。



次のステップ：このイメージを実行できる仮想マシンを作成します。



VirtualBoxを使用しましたが、QEmuもうまく機能すると聞きました。



そのため、VirtualBoxでは、新しいマシンを作成する必要があります。



  • ターゲットOS：Linuxを選択
  • ターゲットOSバージョン：その他
  • 1GBのRAMと1つのCPUを使用しました（デフォルトで残りの設定を残しました）
  • ハードドライブの追加：VDIドライブ、動的サイズ、512 Mio
  • ダウンロードしたISOがロードされるCDROMを追加します




ブートメニューで、ハードドライブのインストールを選択します。







インストール中に、次のことを行う必要があります。



  • ハードドライブに新しいパーティションを作成します
  • ext3でフォーマットする
  • このセクションを選択してAndroidをインストールします




インストールが完了したら：



  • 仮想マシンをオフにします
  • ISOイメージを指すCDROMデバイスを削除します（VirtualBox設定ウィンドウで）




仮想マシンを起動します。これで、動作するAndroid x86イメージが作成されました。



ただし、Eee PC用にデフォルトで構成されているため、これはスマートフォンの画面用に設計されたアプリケーションのテストには理想的ではありません。



次に、電話画面の構成を変更する必要があります。



VMとVirtualBoxをオフにします。



最初のステップ：新しい解像度モードを作成します。 3つのモードを定義しました。 



 VBoxManage setextradata "Android2.3" "CustomVideoMode1" "320x480x16" VBoxManage setextradata "Android2.3" "CustomVideoMode2" "640x960x16" VBoxManage setextradata "Android2.3" "CustomVideoMode3" "480x720x16"




ここで、Android2.3は作成された仮想マシンの名前です。



新しいモードが発表されたので、それらを使用する必要があります。 これを行うには、カーネルパラメーターを変更する必要があります。



これを行うには、仮想マシンを再起動し、ブートメニューでデバッグモードを選択して、Androidがコマンドラインで起動するようにする必要があります。



仮想マシンを起動します。



起動したら、ブートパーティションを読み取り/書き込みで再マウントして、Grub構成を変更できるようにします。 



 mount -o remount,rw /mnt




その後、menu.lstファイルを編集できるようになります。 



 vi /mnt/grub/menu.lst




最初のメニュー項目（3行）を複製し、カーネルパラメーター（最初のエントリ「タイトル」と次の2行）を編集します。



デフォルトのオプションは次のとおりです。 



 quiet root = / dev / ram0 androidboot_hardware = eeepc acpi_sleep = s3_bios、s3_mode DPI = 240 SRC = / android-2.3-RC1




私が使用したオプションは次のとおりです。 



 quiet root = / dev / ram0 androidboot_hardware = generic_x86 acpi_sleep = s3_bios、s3_mode DPI = 240 UVESA_MODE = 480x720 SRC = / android-2.3-RC1




好きな名前を入力してください。



起動時に解像度を選択したい場合、これを行うことができます： 



 quiet root = / dev / ram0 androidboot_hardware = generic_x86 acpi_sleep = s3_bios、s3_mode vga = ask SRC = / android-2.3-RC1




menu.lst（：wq）を保存し、VirtualBoxメニューを使用して仮想マシンを再起動します。



これで、Android仮想マシンを起動できます。これは、電話のように見えます。



仮想マシンを使用するためのいくつかのヒント：



  • Android画面にマウスが表示されない場合は、VirtualBoxメニューを使用してマウス統合を無効にします（ホスト+ Iボタン）。
  • Windowsボタンは、Androidのホームボタンに対応しています。
  • EscはAndroidの戻るボタンに対応します
  • F2はメニューボタンに対応
  • F3は検索ボタンに対応
  • Alt + F1-コンソールモードに切り替えます
  • Alt + F7-GUIモードに切り替えます




Android、AVD、およびEclipse仮想マシンの接続



目標は、Eclispeの新しい仮想マシンを使用してアプリケーションをテストおよびデバッグする機能でした。 これが機能するためには、メインのオペレーティングシステム（Eclipseが実行されている）とゲストVMの間にネットワーク接続を構成するのは面倒です。 VirtualBoxのネットワーク設定には、いくつかの設定があります。



  • ブリッジモードを使用：動作するはずですが、メインOSで起動するにはDHCPが必要になる場合があります
  • ホストオンリーネットワーク：ホストマシンとゲストマシン間に仮想ネットワークを作成します。 これは使用する最も簡単なオプションです








ネットワークが構成されたら、仮想マシンを再起動し、コマンドラインモード（Alt + F1）を入力して、次を入力します。 



 netcfg




これにより、仮想マシンの現在のIPが表示されます。 通常、ゲストマシンの場合は192.168.56.101のようになり、メインマシンはアドレス192.168.56.1を取得します。



メインマシンでコマンドラインを開き、Androidのplatform-toolsディレクトリに移動します。 



 ./adb connect 192.168.56.101




これにより、仮想マシンが新しいデバイスとして登録されます。







これで、Eclipseからアプリケーションを直接起動およびデバッグできます。



ご覧のとおり、速度の増加は非常に顕著です。



  • VMの起動には30秒ではなく2秒かかります
  • デバッグモードであっても、アプリケーションは非常に迅速に起動します（ARMエミュレータに固有のブレーキはありません）




SDカードを追加



http://www.android-x86.org/documents/sdcardhowtoにあるドキュメントを使用します 。



ファイルをSDカードとして使用します





Androidコマンドラインから： 



 dd if=/dev/zero of=/data/sdcard.img bs=1024 count=65536 (64MB image) losetup /dev/block/loop7 /data/sdcard.img newfs_msdos /dev/block/loop7




次に、VMをデバッグモードで再起動し、読み取り+書き込みモードでパーティションをアンマウントし、menu.lstを編集して1つのカーネルパラメーターを追加します。 



  SDカード= /データ/ sdcard.img




別のセクションを使用します



これにはもう少し作業が必要です。



まず、VirtualBoxで新しいハードドライブを作成し、仮想マシンに接続する必要があります。



次に、デバッグモードでマシンを起動します。 fdiskを使用してパーティションを作成します。 セクションが作成されたら、フォーマットする必要があります。 



 newfs_msdos /dev/sdb1




次に、menu.lstを編集してパラメーターを追加します。 



  SDカード= sdb1




テスト環境としてAndroid_x86を使用した印象





使用する

唯一の顕著なグリッチ：ギャラリーアプリケーション（Cooliris）が壊れています。最後のナイトリービルドで試しました-少し良くなりましたが、まだ問題が残っています。



別の奇妙な点：約10％の場合、仮想マシンは起動せず、再起動する必要があります。 ダウンロードは非常に高速であるため、これは実際には気になりませんが、とにかく問題が何であるかを理解しようとします。
