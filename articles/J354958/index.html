図1年 設計ツールの長所と短所について主観的に





昨年、私はFigmaを使用しています。 おそらくあなたはすでにそれについて聞いたことがあるでしょう。 この記事では、個人的な観察に基づいた欠点と利点について説明します。 言われたことはすべて主観的です。 コンポーネントを操作するロジックが非常に粗雑だったときに、私はFigmaを使い始めました。 その年の間に、進化はほとんど気付かないほどに起こりました。 現在、コンポーネント内の要素は、より論理的かつ秩序立った方法で相互に作用しています。 しかし、まだ欠陥があります...



ちなみに、Figmaを使用する場合は 、 既製の設計システムに注意することをお勧めします 。 フリーランサーが1か月あたりの注文を増やすのに役立ち、プログラマーは自分で美しいアプリケーションを作成できます。また、チーム作業用の既製の設計システムを使用して、チームが「スプリント」スプリントをすばやくリードできます。



また、深刻なプロジェクトがある場合、当社のチームは、ベストプラクティスに基づいて組織内に設計システムを展開し、Figmaを使用して特定のタスクに合わせて調整する準備ができています。 Web /デスクトップ、および任意のモバイル。 また、React / React Nativeにも精通しています。 Tへの書き込み： @kamushken




進化は消費者の関与なしには不可能です。



figmaチームはユーザーの声を聞いています。つまり、Figmaが最終的に変わる可能性のある理想的なツールを信頼しているということです。 私の製品を例にとると、この数か月で環境の生産性がどれほど向上したかを知ることができました。



彼らは明らかにそれほど多くのコンポーネントを期待していなかった。



今年初めにFigmaで最初の設計システムを開発したとき、パフォーマンスの限界に直面しました。 「おっと、あまり多くのコンポーネントを期待していなかったようです」と私は考えました。もう少し待ってから、私の製品がすべてのテンプレートの原色をリセットするまで。 100以上のコンポーネントで使用されています。



文字通りパフォーマンスの改善を感じました



徐々に、私のシステムを開発する過程で、私はFigmaの多くのプロセスがより速く発生し始めることに気付き始めました。 私は常に多くのオブジェクトの数百万の小さなものを変更しなければならなかったので、それがより速く起こり始めた瞬間をすぐに記録しました。 それからさらに速く。 それからまた。 以前は理由がなかったかのように、明らかに何かを最適化しました。 figmaはクールな製品ですが、欠陥がないわけではありません。 私の考えでは、作業速度を低下させる3つの主なものをリストします。



グローバルな色の欠如



これまで、カスタマイズに柔軟な設計システムを得るために、各モジュールのコンポーネントとして色を投資する必要があります。 彼らはすでにそれに取り組んでいますが、それにもかかわらず。 グローバルカラーを導入すると、バルクアトミックシステムのパフォーマンスが大幅に向上すると考えています。 何百ものコンポーネントは、配色を変更するたびに再描画する必要がある余分なネストされた要素を取り除きます。



ローカルコンポーネントの検索なし



検索は、チームライブラリに公開された後にのみ使用可能になります。 これがチームに受け入れられるとしましょう。 しかし、自分の設計システムを自分のクライアントに使用したいフリーランサーには受け入れられません。 彼は新しいクライアント用にクローンを作成し、リクエストに応じて素早くカスタマイズします。 ところで、彼は検索も必要です。 そして、チームの機能は絶対に必要ありません。



500を超えるモジュール間で操作する場合、検索せずに「コンポーネント」タブ（Alt-2）での作業は非常に困難になります。




多数のコンポーネントがある場合のインスタンスでのハードワーク



これは、多数のコンポーネントがあり、それらに多くの同様の複製があるが、わずかな変更がある場合に重要になります。 たとえば、1ダースのカード、何百ものアイコン。 カード1、2、3、4 ...と呼ばれる場合、ドロップダウンリストで目的のカードモジュールをすばやく見つけることができません。 N.そして、コンポーネントの名前をその説明に変えることはあまり良い考えではありません。



正直なところ、これらの欠点であなたを怖がらせたくありません。 開発チームには巨大なTo Doリストがあり、すべての顧客が満足できないため、すべての人の希望を満たそうとすることを完全に理解しています。 結局のところ、会社は独自のビジネストラジェクトリに沿って移動することと、チャットの機能リクエストセクションを毎日更新する何百もの新しいリクエストの間でバランスを取る必要があります。 たくさんのプラスがありますが、個人的に私を喜ばせるものだけに注意しています。



コンポーネント



コンポーネントのまさにその原理は驚くべきものです。 たとえば、コンポーネント内の要素をシフトすると、数十または数百の下位でシフトします。 わずか3〜5アクションで大量編集。



スマートスティック



スナップは非常によく考えられており、敏感なので、遠く離れた場所で一度に多くのオブジェクトを操作できます。 あなたはそれらを動かし、動かします、そして彼らがグリッドに入ることを心配しません。



スマートハイライト



私は多くのオブジェクトをハイライトし、見逃すことを恐れていません。 選択はアクション履歴に保存されます。 また、選択をCtrl / Altキーを押しながら組み合わせると、多くのルーチンアクションが半自動モードになります。
