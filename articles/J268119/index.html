例としてModbusプロトコルを使用したロード可能なZabbixモジュールの作成

Zabbix 2.2のバージョンでも、システムの機能を新しいレベルで拡張できるロード可能なモジュールが追加されました。 「なぜこれが必要なのですか？」と、Zabbixからいつでも外部のスクリプトとプログラムを実行できるので、あなたは尋ねます。 もちろん、これは主に速度です。Zabbix自体のようなモジュールはCで記述されており、各調査で実行する必要がある外部プログラムとは異なり、できるだけ速く正しく動作します。 多くの人はコードを書く必要性に恐れるかもしれませんが、今日はすべてがそれほど複雑ではないことをお見せしたいと思います。



たとえば、Zabbixが広く使用されている産業オートメーションプロトコルであるModbusで動作するデバイスから情報を収集し、温度センサーの読み取り値を取得し、Mercury 230メーターから電気パラメーターを取得できるようにするモジュールを作成します。 share.zabbix.comポータル。ユーザーはZabbixでの経験を共有できます。







Modbusについて一言



Modbusは、産業オートメーションネットワーク（工場、ワークショップ、その他の産業施設、さまざまなエンジニアリングシステム-Modbusでつまずくことができるあらゆる場所）で使用される非常に一般的なプロトコルです。 このプロトコルは1979年にModiconによって開発され、その後数十年にわたって、その利点のおかげでModbusは膨大な数の異なるデバイスに実装されました。



シンプルさ

Modbusは非常にシンプルです。 これにより、最も簡単で安価な鉄片で実行および実装できます。 また、開発者がソリューションで使用することを妨げません。



イーサネットサポート

Modbusは、産業施設を含むイーサネットネットワークの普及に間に合うように準備され、現在ではシリアルラインRS-232、RS-485（Modbus RTU / ASCII）だけでなく、TCP / IP（Modbus TCP）でも動作します。 。



開放性

プロトコルのすべての仕様はオープンであり、誰に対してもロイヤリティを支払うことなく、意思決定に使用したいすべての人がアクセスできます。



次に、Modbusがどのように機能するかについて簡単に説明します。 デバイスのメモリ内のデータへのアクセスは、特別な論理テーブル（ ディスクリート入力、コイル、入力レジスタ、保持レジスタ）を介して実行されます。 最初の2つのデータはシングルビットの形式で、最後の2つのデータは16ビットワードの形式で表示されます。

テーブル データ型 アクセス
離散入力 1ビット 読み取り専用
コイル 1ビット 読み書き
入力レジスタ 16ビット 読み取り専用
保持レジスタ 16ビット 読み書き


最初の2つの表は、単純な離散値を扱うために使用できます：「オープンクローズ」、「事故ではなく事故」など。 また、 入力および保持レジスタは他のすべてに使用できます。 さらに、データが16ビットに収まらない場合の問題は、2つの隣接するレジスタを使用することで簡単に解決できます。 したがって、32ビットのfloatまたはlongで作業できます。

Modbusデータ要素は、すべてのテーブルで0〜65535のエンドツーエンドアドレッシングを通じてアクセスできます。



データへのアクセスは重複しない場合があり、各テーブルは個別のメモリブロックにつながります。







したがって、複数のテーブルを使用して同じデータにアクセスできます。







たとえば、16個のドライ接点を持つコントローラーがあり、現在どのように開いているか閉じているかを知りたい場合、 次に、 入力レジスタテーブルを使用して16個すべての連絡先の値を一度に取得するか、 入力ディスクリートテーブルを使用してすべての連絡先を順番に読み取ります。



Modbusデバイスがシリアルライン（RS-485）上にある場合、Modbusの各デバイスには固有のアドレス（スレーブID、1〜247）があり、問い合わせデバイスは常に1であり、マスターと呼ばれます。 Modbus TCPデバイスの場合、IPアドレスで十分です。

モジュールの作成



Zabbix用のモジュールを書く方法は？



UserParameterを介してスクリプトを実行する場合のように、最初にZabbix自体のデータ要素キーがどのように見えるかを考える必要があります。 これから始めましょう、私たちの場合、それはそのようなキーになります： 



modbus_read_registers[ <connection>, <slave_id>, <reg_to_read>, <modbus_function>, [<datatype>],[<endiannes>],[<first_reg>] ]


このキー内で、4つの必須パラメーターと3つのオプションパラメーターを渡します。



  • 接続 -接続パラメーター付きのIPアドレスまたはシリアルポート
  • slave_id -ModbusデバイスID
  • reg_to_read-データ付きの大切なレジスタ
  • modbus_function -Modbusテーブルを読み取るための関数の1つ、1- COILSを読み取るため
  • 2- ディスクリート入力の場合、3-保持レジスターの場合、および入力レジスターの場合は 4
  • datatype-レジスタから返されたデータのタイプ。 ビット、16ビット整数、32ビット整数、または32ビット浮動小数点。
  • endianness 、 first_reg -32ビット値のビット順序を正しく設定するため、および使用するアドレス指定を示すために必要です-Modbusデータモデル（最初の要素がアドレス1と最後のnを持つテーブル要素）またはPDUアドレス指定（0〜65535のアドレス指定） ）


それで、彼らは思いついたので、モジュールに直接行きましょう。 実際、Modbusレジスタからデータを収集するための関数を作成し、次の関数から特別なZabbixインターフェースを追加する必要があります。

2つの必須機能： 



 int zbx_module_api_version(void); int zbx_module_init(void);


また、3つのオプション： 



 ZBX_METRIC *zbx_module_item_list(void); void zbx_module_item_timeout(int timeout); int zbx_module_uninit(void);


しかし、 dummy.cの例から完全にコピーします。これは、Zabbixのドキュメントに記載されています。 優れた、時間とエネルギーの節約、前進します。



そして、問題は、コレクション関数自体をどのように書くかです。 ちなみに、次のように呼び出しましょう。 



 int zbx_modbus_read_registers(AGENT_REQUEST *request, AGENT_RESULT *result)


そして、ここでの作業は最小限に抑えられます。これは、Modbusプロトコルの実装を損なわず、ホイールを再発明しないために、Cライブラリを使用するためです 。



libmodbusインターフェースを使用して、Zabbixからの受信データを検証し、 libmodbusからポーリング関数を実行し 、受信した値をZabbixデータ型に変換して 返すだけです。 さて、またはエラーを返します。

もう一度、重要な点をいくつか強調します。



1）関数が正常に実行されると、結果はマクロを介して* resultに格納されます。

  • SET_UI64_RESULT-結果が整数の場合
  • SET_DBL_RESULT-結果が浮動小数点数の場合
  • SET_STR_RESULT-結果がCharの場合
  • SET_TEXT_RESULT-結果がテキストの場合
  • SET_LOG_RESULT-結果がログの場合


例： 



 SET_DBL_RESULT(result, modbus_get_float(temp_arr));


また、関数自体はSYSINFO_RET_OKを返す必要があります。



2）何か問題が発生した場合は、 SYSINFO_RET_FAILを返す必要があり、 *結果に、Zabbix Webインターフェースで確認できるエラーメッセージを入力します。 



 SET_MSG_RESULT(result, strdup("Check datatype provided."))






3）すべての受信パラメータを検証することを忘れないことが非常に重要です。 そうしないと、モジュールのクラッシュによりZabbixサーバーまたはエージェントがクラッシュします。



4）Zabbixは多くのポーリングプロセスを並行して実行します。 したがって、同時アクセスを許容しないリソース（この場合はファイルまたはシリアルポート）を使用する場合は、このアクセスを自分で制御する必要があります。 セマフォを使用してこれを行いましたが、これはタスクを多少複雑にしますが、幸いなことにこれは必ずしも必要ではありません。



5）外部のlibmodbusライブラリが使用されるため、Zabbixモジュールが使用されるシステムにインストールする必要があります。 



 wget http://libmodbus.org/releases/libmodbus-3.1.2.tar.gz tar zxvpf libmodbus-3.1.2.tar.gz cd libmodbus-3.1.2 ./configure make make install ldconfig

テンプレートを作成



まず、温度センサーを読み取るためにMB110-224.2A のデータ要素を作成します 。







キーはどこですか： 



 modbus_read_registers[{$MODBUS_PORT},{$MODBUS_SLAVE},10,3,f,1,0]


  • {$ MODBUS_PORT} -デバイスへの接続パラメーターによって上書きされるマクロ 。 たとえば、ホストレベルでは、{$ MODBUS_PORT}を/ dev / ttyUSB0 9600 N 1として定義します
  • {$ MODBUS_SLAVE}はslave_idを意味する別のユーザーマクロであり、 32に書き換えます
  • 10-温度値が存在するレジスタ
  • 3-読み取りに使用されるModbus関数、この場合は0x03 READ HOLDING REGISTERS
  • f-データ型、float
  • 1 -Ariesコントローラーはビッグエンディアンを使用して、同じレジスタに収まらない値をエンコードするため
  • 0 -PDUアドレッシングが使用されます
