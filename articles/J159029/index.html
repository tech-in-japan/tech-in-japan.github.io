Hulu、Pandoraなどを使用する最も簡単な方法 VPNなし

UPD ：
Guys、Tunlrの代わりに、PandoraとSpotifyのDNSプロキシを上げました。 当然のことながら、すべてのプラットフォームで機能し、実質的に設定を必要としません-habrahabr.ru/post/215889/ 。 ようこそ！




私のように、Pandoraラジオを聴くたびにVPNトンネルを州に接続することにうんざりしている場合は、音楽の好みを知っているか、hulu.comのリンクでビデオを見ると、2つのマジックアドレスが役立ちます（サイトで確認し、定期的に変更します）：



142.54.177.158



209.141.56.79



これらはtunlr.net WebサイトのDNSサーバーのアドレスであり、ヘッダーの トリックを使用 して 、ホスト国のIPのチェックを回避するのに役立ちます。



このサービスは、機能するサイトの特定のリストのみをサポートします 。 他のすべてのクエリの場合、通常の正直なDNSサーバーとして機能します。



ただし、 FAQページで報告されているように：



継続的にDNSに切り替えるべきではない理由：

これはTunlrインフラストラクチャに大きな負担をかけるため、ユーザーのDNSクエリへの応答を人為的に遅らせます。



あなたは実際の生活でこの遅れを感じないかもしれません（私は気づきませんでした）。 しかし、サービスと私たち自身の尊重から、ローカルキャッシングDNSは私たちを助けることができます。 たとえば、DD-WRTファームウェアルーターの幸運な所有者であれば、それに組み込まれているDNSMasqを使用できます。



次の行をサービス-DNSMasq-追加のDNSMasqオプションに追加し、ルーターを再起動します。
サーバー= / tunlr.net / 184.82.222.5

サーバー= / pandora.com / 184.82.222.5

サーバー= / hulu.com / 184.82.222.5

サーバー= / cbs.com / 184.82.222.5

サーバー= / abc.com / go.com / 184.82.222.5

サーバー= / mtv.com / mtvnservices.com / 184.82.222.5

サーバー= / nbc.com / nbcuni.com / 184.82.222.5

サーバー= / fox.com / theplatform.com / 184.82.222.5

サーバー= / tv.com / 184.82.222.5

サーバー= / pbs.com / 184.82.222.5







まあ、192.168.1.1（またはルーターが応答する別のアドレス）に切り替えることを忘れないでください。 私の知る限り、他のすべてのクエリにはGoogle 8.8.8.8を使用します。



または、マシンでローカルDNSを使用します（このブログの以前の投稿 ）。 私は、これをどのように行うことができるかをコメントで提案する専門家を招待します。
