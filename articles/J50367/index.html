高速シャットダウン\ Windows XPを有効にする

*サンドボックスから移動、招待してくれてありがとう:-)

こんにちは

正直に似たようなトピックを探してみましたが、見つかりませんでした。 今日、私が誤って学んだこと、そして少し怒りさえしたことについてお話したいと思います。 マイクロソフトには、驚くほど機能する便利なユーティリティがたくさんあることがわかりました！ Windowsに標準で含まれていない理由もわかりません。

1）Windows XPの高速アンロード。

このようなユーティリティは、Microsoft Webサイトにあります： UPHClean（ユーザープロファイルハイブクリーンアップサービス）

Microsoft.com/UPHClean

何らかの理由でダウンロードできなかった場合は、ここから次のことができます。

Filehoster.ru

UPHCleanサービスは、ユーザープロファイルのアップロード中にコンピューターを監視し、開いているリソースを閉じます。 正直なところ、英語ではそれは重要ではありませんが、私が理解しているように、通常、ユーザーをアンロードした後、Windowsはレジストリのスキャンと調整とそこでの処理を開始します。 。 このユーティリティはそれ自体をサービスとして登録し、常にメモリにハングアップします（1 mbから5 mbまで。それぞれに異なる方法があります）。起動中のサービスには非常に批判的ですが、このユーティリティは私を拒否できませんでした。 -）5〜7秒でオフになります。 25年前

2）Windows XPの高速読み込み。

このトピックはインターネット上で非常に多くのスペースが割り当てられていることを理解していますが、ここでは、ボタンを1つ押すだけで超高速で行う方法を説明しません。

Bootvisツール

Microsoft.com/BootVis 。

ここからダウンロード

率直に言って、このユーティリティ自体はロードを特に高速化するものではありませんが、マシンを起動する秒数と、ロード中に問題を引き起こすプロセス、サービス、ドライバーを正確に知ることができます。 ところで、このユーティリティが読み込みを最適化する程度を確認する簡単なトリックを行うことができます（結果は誰でも異なると思います）。 プログラムをダウンロードして、[Trace \ Next Boot]を選択し、[OK]をクリックしてから[Reboot]をクリックします。 マシンは再起動を開始します。Windowsの起動時に適切なユーザーをできるだけ早く選択するだけで済みます。--）ユーザーが1人だけなので、このような問題はありません。 次に、BootVisウィンドウが読み込まれるのを待ちます。これにより、何かが分析され、最終的に図のリストが表示されます。 いずれの図でも、フィニッシュポインターのある赤い縦線が特定の位置にあります。 これは、マシンが起動する時間です（もちろん、その数は実際よりも少なくなりますが、明らかに、このユーティリティはインストールしたスタートアッププログラムを考慮せず、システムプロセスのみを考慮します）。 いずれにせよ、この数値は、3Dマークの抽象的なパフォーマンスインジケーターと同様に調整する必要があります。 番号を覚えて、Trace / Optimize Systemを押してください。 再起動をクリックします。 次回の起動時に、すでにマシンが最適化されます。 再起動と、すべてがうまくいったというメッセージを待って、Trace \ Next Bootの最初に同じことを行います。 さて、同じように、2番目の数値が得られました。理論的には、これは最初の数値よりも小さいはずです。 そのため、マシンの起動がどれだけ速くなったかがわかりました。 個人的には、このユーティリティは最適化自体ではなく、正確に時間を測定するのに役立ちます。 たとえば、私は座っていないまま弟のコンピューターを最適化するために座っていました）最初は70秒で、レジストリ、サービス、その他のゴミを調べてから、35秒で読み込みが開始されました。 グラフを表示して分析すると、起動時に起動するデーモン、起動を遅くするプロセスを見つけることができます。 また、Trace \ Next Boot + Drivers Delaysも非常に便利な機能です。 どのドライバーがロードされているかが表示され、競合しているドライバーが赤で強調表示されます。 これはいくつかの問題を解決するのに役立ちます;-)幸運

注意！ このユーティリティを機能させるには、Windowsサービスの「タスクスケジューラ」が無効になっている場合、有効にする必要があります！

更新：最適化の過程で、Companionがコンピューターの起動が3秒間遅くなり始めた理由があるため、既に35秒以内に起動した場合は、実験しないでください。
