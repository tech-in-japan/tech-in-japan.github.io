ロシアの鉄道の例における何よりも安全なセキュリティ

多くの人がインターネットを介して鉄道チケットを購入するシステムを使用したと思います。 そして、私はそれを複数回使用しました。 もちろん、さまざまな奇妙な点がありました...しかし、全体的にシステムはクールです。 多くの人は、モスクワの時間に電子チケットが発券のために登場したことに気づいたでしょう。 ユースケースは次のとおりです。 インターネットサイトからチケットを購入し、バーコード付きのフォームを印刷して駅に行き、そこでこのフォームを鉄の脳に持ち込み、パスポートの詳細を入力してチケットを取得します。

1つありますが、このスキームでは常に混乱します。 パスポートデータの暗号強度。 それでも、私は安心しました-彼らは通りで私のフォームを見つけます、彼らはそこに私の名前を見るでしょう、どこで私のパスポート番号を手に入れることができます！



サイトでチケットを発行すると、この美しいサイトですぐに印刷できます。 称賛に値する熱意！ しかし、一つだけあります-FFは、用紙サイズのためにプリンタードライバーと口論するため、プリンターでドキュメントを印刷しません。 そうですね、OpenOfficeでフォームをコピーして印刷します。 これは私がそれをしているだけではない-多くの、習慣から、オフィススイートにコピーしてそこから印刷することができます。

そして、ここから楽しみが始まります！ この方法でフォームを印刷した後、私はそれを見つけました...私のパスポート番号！ つまり、このフォームを手に持っている人が受け取ります。 フォームでこれを見て、私は最初にロシアの鉄道について悪いことを考えてから、彼らのサイトを見て、コードで次のことを見ました：



< td >

< スパン クラス =スクリーン> PN1234567890 < / / スパン >

< スパン クラス =印刷>月***** 7890 < / / スパン >



< / td >





モードに応じて、パスポートデータ付きブロックまたはアスタリスクのいずれかが選択されます（構造番号を考慮して星の位置もクールに選択されます）。 最後に、実際にパスワードとして使用されるデータを非表示にする手段としてのCSSの使用を確認します。
