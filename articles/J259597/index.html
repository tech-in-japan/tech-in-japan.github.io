SharePoint Server 2016の新機能をプレビューする

5月初旬にシカゴで開催されたIgniteカンファレンスで、MicrosoftはSharePoint Server 2016の新機能と開発計画のビジョンを共有しました。いくつかのセッションで、会社の役員は開発を完了し、来年出荷を開始することを目指していることを公式に発表しました。



会議で提示された情報に従って、マイクロソフトは今年の第4四半期にSharePoint Server 2016のベータ版を発表します。 この製品は、2016年の第2四半期に顧客が完全に利用できるようになります。







ベータ版が以前にリリースされる可能性があります。 シニアテクニカルプロダクトマネージャーであり、SharePointのマイクロソフト認定マスターであるBill Bayer氏は、SharePoint Server 2016の一般公開は夏の終わりまでに可能になると述べました。

マイクロソフトは、ファイル、サイト、ポータルの分野でSharePointの機能に集中することを計画しています。 SharePointおよびOneDriveのマーケティングシニアディレクターであるSett Pattonは、ビジネスの準備ができたらクラウドに簡単に移行することを目標に、ハイブリッドアーキテクチャ（SharePoint ServerとOffice 365）を組織で利用できるようにする計画を強調しました。 新しいSharePointはよりソーシャルで、モバイルであり、新しい機械学習機能を備えています。



Pattonは、新しいOffice 365の機能はSharePointと互換性があると述べ、Microsoftは今後のリリースでそれらを導入する予定です。 そのような革新の顕著な例は、SharePoint Server 2016の次世代ポータルです。OfficeGraphエンタープライズ検索テクノロジに基づくMicrosoft Delveコンテンツ検索ツールを使用します。 既に、SharePoint Server 2016の機能に含まれているDelveポータルを含むSharePoint Server 2013のアップグレードをリリースする予定です。



企業部門、特にモバイル分野でのOneDrive for Businessの機能を拡張する作業も進行中です。 同様のSharePoint Server機能と同様に、ファイル共有オプションが拡張されます。



Bayerは、SharePoint Server 2016がFoundationバージョンの基盤としてクラウドサービスを持つことを強調しました。これは、自社製品に関するMicrosoftの現在の考え方を反映しています。 ハイブリッドクラウドソリューションの作成は、個別の情報ストレージ（クラウドまたはオンプレミス）の存在を意味しますが、これは見つけるのが非常に困難ですが、MicrosoftはSharePoint Server 2016のすべてを変更する予定です。クラウド検索の新しいサービスは、SharePoint Server 2013また、SharePoint Server 2016にも含まれます。この新製品により、組織はクラウドとオンプレミス展開オプションの検索インデックスを統合できます。



SharePointは従来、可変システムであり、その機能はユーザーの役割によって決定されるため、パッチのダウンロードは非常に不便でした。 この問題を解決するために、Microsoftは新しいインストールウィザードを使用して、SharePoint Server 2016に新しい役割ウィザードを追加します。 Bayerは、ハイブリッドインストールを完了するために15ページのPowerShellコマンドを他の人が破る必要はないと述べました。 SharePoint Server 2016では、パッチの問題は解決されます。 過去には、SharePointパッチのサイズはそれ自体と同等でした。 現在のインストールに必要な37個のパッチと言語パックの18個のパッチと比較して、Microsoftは将来、言語パックの一部としてさらに4個と1個を管理する予定です。 更新が少なくなるため、費やす時間が大幅に削減されます。



SharePoint Server 2016のシステム要件には、Windows Server 2012 R2またはWindows Server 10が含まれます。Microsoftは、SharePoint Server 2013をSharePoint Server 2016の起動パッドとすることを決定しました。スタンドアロンインストールのサポートの欠如。 また、単一サーバーファームへの展開には、SQL Server Expressは含まれません。 組織は、同じマシンにインストールされたデータベースサーバーと統合する必要があります。







他にも良いニュースがあります。 マイクロソフトは、コンテンツデータベースのサイズ、最大ファイルサイズ、およびその他の制限を拡大します。



SharePoint Server 2016で発表された多くの新機能：



  • SAMLとOAuthを使用したクラウドベースの認証モデル
  • データ転送中の暗号化には、ポート25だけでなく、他のデフォルト以外のポートも使用できます
  • 以前のように、サイトの作成は簡単で迅速であり、複雑な操作ではありません
  • Project ServerはSharePointコンテンツデータベースと統合されます
  • Microsoftは、ファイルを検索するための「長期リンク」を作成するため、ファイルの名前を変更したり移動したりしても、コンテンツの可用性には影響しません。
  • Microsoftは、ファイルをODF形式で保存する機能を追加します
