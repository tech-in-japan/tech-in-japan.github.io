ルーターのPPPoEにインストールされているログインパスワードを調べる

ルータのPPPoEにインストールされているログインパスワードを知る必要がある場合はどうすればよいですか？ 質問はhabrahabr.ru/qa/28217で尋ねられ、私は自然な好奇心から実験することにしました。 管理パネルにアクセスできないと仮定します。 ルーターモデルはそれほど重要ではありません（私の場合、D-Link DIR-300がテストのために使用されました）



猫の下での解決策



この問題は、優れたMikrotik RB951-2nルーターの助けを借りて解決しました（この会社のどのルーターでも実行できますが）。 コンソールに接続し、コマンドを実行します-



interface pppoe-server server add authentication=pap disabled=no interface=ether5 service-name=service1







このコマンドは、ルーターの5番目のポートでPPPoEサーバーを有効にします。



次に、PPPoEで拡張ログ出力を行います



system logging add topics=pppoe







次に、パスワードとMikrotikの5番目のポートを引き出す必要があるルーターポートにWANパッチコードを接続し、ログに移動します。 パスワードをプレーンテキストで表示します:)





PS：結局のところ、ロギングとPPPoEサーバーを削除することを忘れないでください





PS2：原則として、コンピューターにMikrotik PPPoEサーバーをインストールしなくても同じことができますが、私にとっては簡単でした:)しかし、この方法はMikrotikを必要とする唯一のマイナスです。



PS3：認証方法がCHAPまたはMSCHAPとして選択されている場合、この方法は機能しません。 ただし、テスト済みのルーター（Dlink DIR-300、Dlink DIR-320、Linksys WRT54gl、Tplink TL-WR941ND）では、デフォルトのPPPoE設定で、パスワードが静かに引き出されます。
