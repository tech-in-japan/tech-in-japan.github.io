アマチュアおよびプロフェッショナル向けのHTMLフラグメントの個人用ストレージ

個人のニーズに合わせてウィジェットを配置、選択、カスタマイズする個人用ページは、Webサービス開発の主なトレンドから長い間外れています。



一方では、個人ページサービスが存在し、忠実なユーザーを喜ばせます。最良の例はMy Yahooサービスです



一方、そのようなサービスは機能しなくなります。 iGoogleサービスの閉鎖は2012年夏に発表され、2013年11月1日に、このサービスのユーザーの個人ページはすべて完全に削除されました。



一般的に、このようなサービスの作成に対する投資家や起業家の関心はゼロであると言えます。 ユーザーに必要なものだけを1ページで提供することは良い考えのようで、非常に自然な困難に直面します。



  1. 新しいウィジェットを配置するプロセスの相対的な複雑さ：特別な場所に移動し、必要なものを選択し、お気に入りを個人用ページに配置する必要があります。
  2. 多数のウィジェットが提供されているため、多様性が欠如しています。
  3. 可能な機能の厳密なフレームワークで絞る-管理者によって承認されたもののみが可能です。
  4. 多数のウィジェット（数十、数百）を使用するのは難しく、管理するのはほとんど不可能です。


TheOnlyPage Webサービスは、 事前定義されたスキームに従って構築されたコンポーネントとしてのウィジェットの概念を再考し、すべての制限を削除して任意のhtmlコードを使用することを決定しました。



以前の出版物では、オンラインノートサービスの機能コンテンツについて説明しました。 TheOnlyPage Webサービスで使用されていると考えられていました 。メモを提示するための最も禁欲的なオプションは、テキストのみです。 純粋なメモの一種。



正反対は、htmlコードの任意のフラグメントとして表されるウィジェットを保存することです 。 すべてのWebデザインツールを使用できます。



マーケティング上の理由から、このようなhtml-fragmentsなどの平均的なユーザーを混乱させないために、 Picturesと呼ぶことにしました 。 任意のhtmlコードを使用して表示されるビジュアルオブジェクトは、確かに画像のみに限定されませんが 。



提案されたソリューションの詳細



前に、ブックマークサービスとTheOnlyPage Webサービスでの実装の成功した視覚的メタファーの選択について説明しました 。



当然、 TheOnlyPageサービスの写真は、 ブックマークやメモと同じテンプレートに従ってユーザーに表示されます。 上：検索バー。 左側のナビゲーションバー：テキストフラグメントとカラーマークによって最後に保存された選択。 中央の場所は、画像の表示/編集/作成の時間でソートされた画像のリストで占められています。

画像のタイトルをクリックすると、画像のタイトルが開き、その内容を見ることができます。





画像タイトルを含む行をダブルクリックすると、編集フォームが開きます。







画像を作成するには、 [ 新しい画像 ]ボタンをクリックし、[ コード]フィールドにhtmlコードを挿入し、[ 署名 ]フィールドに画像の署名を挿入して、[ 保存 ]ボタンをクリックします 。



ブラウザーが指定されたhtmlコードをどのように処理するかを確認するには、[ 表示 ]ボタンをクリックします。



したがって、htmlコーディングの専門家は、自分のために特別なソリューションを作成できます。 ファンは、インフォーマー、画像、またはビデオのコードをコピーアンドペーストするか、特別なブックマークレットを使用して他のサイトからhtmlフラグメントを収集する必要があります。



ブックマークレットの例



ブラウザのブックマークバーに特別なリンクをドラッグアンドドロップするだけで、新しいブックマーク、メモ、写真を作成するためのブックマークレットを設定できます。 ブックマークレットのインストールの詳細については、 オンラインドキュメントを参照してください 。



たとえば、 habrahabr.ruで最も人気のある出版物を読んで、この記事のビデオを保存したかったのです。



TheOnlyPageブックマークレットのリンクをクリックして、必要に応じて[ 画像 ]タブに移動します。 サービスは、ページ上のサイズと場所に基づいて、保存するオブジェクトを自動的に決定し、保存するのに最も顕著なものを提案します。







[ コード]フィールドには、保存されたビデオのhtmlコードと、メディアオブジェクトが取得されたページのアドレスが自動的に挿入されます。

[ 署名]フィールドでは、ページタイトルが置き換えられます。

[ 表示 ]ボタンをクリックすると、画像の外観を確認できます。 保存されたオブジェクトがTheOnlyPageに正しく表示されることを確認するには、保存された画像をプレビューする必要があります。







[ 作成 ]ボタンをクリックすると、新しい画像が作成され、通常のhtmlコードのフラグメントとして保存されます。



ブックマークレットを使用する場合、写真として保存するものを決定する3つの方法があります。

  1. ページ上で何かが強調表示されている場合、 選択したオブジェクトのhtmlコードが保存用に提供されます。
  2. ページで何も強調表示されておらず、ページがメディアオブジェクトのプレゼンテーションに特化した人気のあるサイト（ YouTubeビデオホスティングやスライドシェアプレゼンテーションホスティングなど）のいずれかに属している場合、 TheOnlyPageは対応するメディアオブジェクトのプレゼンテーション用のhtmlコードのフラグメントを自動的に構築します。
  3. ページ上で何も強調表示されておらず、このサイトの画像を作成するための特別なメカニズムがない場合、ページ上の主要な位置を占めるメディアオブジェクトのhtmlコードが使用されます。


ご覧のとおり、平均的なユーザーは、お気に入りのメディアフラグメントをプライベートコレクションに保存するタスクに対処します。 しかし、htmlマークアップを理解している専門家は、何が起こっているのかを認識してこれを行い、場合によっては、保存されたhtmlコードを自分の利益のために修正します。
