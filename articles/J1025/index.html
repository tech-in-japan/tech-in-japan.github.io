サイトで広告を販売するための新しいアプローチ。

一般的に、私はかつてDmitry Koterovの非常に興味深いライブラリを見つけました。これにより、頭痛なくデータをページに動的にロードできます。





私のサイトで広告のインプレッションを販売するのではなく、広告時間を販売するという考えに関するStatistics for Web 2.0の投稿に対するコメントを拡大したいと思います。



広告時間を販売するというアイデアは、私の頭のどこかで長い間非公式な形でしたが、今日では、あらゆるものとすべてに対する私の固有の矛盾の感覚のために、アイデアはより具体的になり、私の頭の中に直接入り込みました。



だから。 ここには何がありますか？ 勇敢な兵士アヤックスがここにあり、ここに広告があります。

これで賢明なことができるのでしょうか？ また、いくつかのスクリプトを作成することもできます.JSは、一定の時間が経過すると、単にコードの新しい部分と新しいタイムアウトについてPHPローダーに接続し、指定された場所にコードを貼り付けて、新しく設定されたタイムアウトまで操作を繰り返します。

「コードの一部」は、広告バナーの装飾コード（少なくともテキスト、少なくともgif、少なくともフラッシュ）です。 「新しいタイムアウト」は、JSの新しいタイムアウトになります。



したがって、広告時間の計算、広告の出力順序の決定、統計の取得に関するすべての作業は、PHPの肩にかかっています。 これは、広告の表示時間とスクロール数に関する健全な統計を整理できることを意味します。



広告主に「私たちのサイトはテレビチャンネルのようなものです。 この超大型は現代的です。 そして、私たちはあなたから余分なペニーを取りません。 広告を配信するのにちょうど100時間かかります。」
