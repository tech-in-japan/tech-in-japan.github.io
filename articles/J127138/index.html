2つの世界の団結

ですから、雲はクールでファッショナブルで、今は雲に夢中になっていない人は皆知っています。 すべての「老いも若きも」はすでに雲に登り、そこから哲学的啓示と市場予測を放送しています。 うん。 ここが私たちの明るい道です。整然と列になって行きましょう。



分析的思考の圧力の下で、外部の観察者は、ここで、最終的に、すべてのための、そしてすべてのための待望の万能薬が見つかったという印象を得るかもしれません。 そしてもちろん、マルウェア、スパム、ハッカーに対する保護に関するすべての問題を解決します。



そして、はい、また、MightとMainでクラウドを使用しています。 これは、多くの問題を解決するのに役立つ素晴らしい発見です。 Kaspersky Labは2006年にこのテクノロジーを初めて導入し、現在ではより多くの開発を行っています。 Kaspersky Security Networkは、既存の脅威に関する情報をリアルタイムで収集し、未知の脅威とそのソースを効果的に特定して、ユーザーを迅速に保護できる分散システムです。 しかし、実際のインジケーターで測定する場合、検出から更新までに平均でわずか40（秒）しか経過していないと言えば十分です。 KSNに感謝します。







明らかに、クラウドテクノロジーを使用しないセキュリティソリューションは、今日ではもはや関係ありません。 しかし、インターネットだけでなく多くの脅威が存在する場合、ベンダーはどのようにしてすべての保護をクラウドに移行できますか？



これまでのところ、ユーザーマシン上のすべての脅威の約30％の中和をKSNに委託しています。 残りは、コンピューター上のローカル保護プログラムによって直接分析および処理されます。 不要なプログラムアクティビティの監視、ヒューリスティックアナライザー、ハザードレーティング、Kaspersky Security Networkに蓄積された情報を使用するその他のプロアクティブテクノロジーなど、すべてのウイルス対策機能が含まれています。



ユージンカスペルスキーが言うように ：



「クラウドは自己完結型のテクノロジーではなく、自律的な保護と連携した成功したバッチです。」



はい、でもなぜですか？



「まず、 アクセシビリティ 。 クラウド保護は、コンピューターがオンラインのときにのみ機能します。 接続なし-保護なし。 しかし、脅威は残っています。 彼らは感染したフラッシュドライブやCD / DVDディスクを滑らせ、他のコンピューターやモバイルデバイスからデータをダウンロードしました-それがales kaputです。 そして、彼らは感染することができるので、あなたはクラウドに触れさえしません。



第二に、クラウド自体のセキュリティ 。 率直に言ってみましょう。クラウド自体が攻撃を受けないことを100％保証することはできません。 むしろ、反対。 そして、最近のPSNのハック（詳細はこちらとこちら ）は、これを確認するものです。 さらに、これはクラウドが攻撃者の主な標的になっていることを示しています。 1回のハックで一度に数千、数万のユーザーをカバーすると便利です。



結論-スタンドアロンエージェントがなければ、クラウドをハッキングするとコンピューターの絶対的な脆弱性につながります。 クラウド保護の実装に応じて、ハッキングは他の結果をもたらす可能性があります。 たとえば、ハッカーは保護されたコンピューター自体を制御できます。



最後に、自己防衛を忘れないでください。 強力な自己完結型エージェントがなければ、マルウェアはクラウドへのアクセスを簡単にブロックできます。



第三に、 保護の質 。 クラウド保護を使用したルートキット、ポリモーフィック、およびその他の複雑なマルウェアの検出と削除は、原則として困難または不可能です。 ここでは、オペレーティングシステムに深く統合されたコンピューター上のエージェントなしでは実行できません。 クラウド自体に既にアクセスできないため、クラウドからマイクロエージェントをスローすることはオプションではありません。



メッセージに戻ります。私は雲が大好きですが、このテクノロジーが自給自足であるというポジショニングと認識に同意しません。



ユーザーは、クラウド保護とオフライン保護の組み合わせが必要です。 そして、この状況が近い将来にどのように変化するかはわかりません。 まあ、私たちはインターネットアクセスで世界中の100％のカバレッジを達成しました。 少なくともエルブラスで、少なくとも南極で。 厚くて遍在するそのようなチャネル。 わかった とにかく、クラウドのセキュリティと複雑な脅威との戦いの問題が残っています。」
