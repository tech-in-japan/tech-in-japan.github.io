書籍「Internal Linux device」

ブライアン・ウォードによる本があります。それはすでに西洋でベストセラーになっています。 Linuxオペレーティングシステム、システム管理、低レベルのLinux機能を提供する深いメカニズムでの作業のすべての複雑さについて説明します。 この出版物のページでは、Linuxカーネルの操作とコンピューターネットワークの適切な操作の原則に関する基本的な知識を習得できます。 この本はまた、シェルスクリプトのプログラミングとC言語の処理の問題に対処し、情報セキュリティ、仮想化、およびその他のかけがえのないものを強調しています。



誰が本を読むべきか


Linuxオペレーティングシステムの設計に対する関心は、さまざまな理由による可能性があります。 Linuxのソフトウェア開発者と同様に、情報技術サービスの分野の専門家は、この本で、オペレーティングシステムを最適な方法で使用するために知っておく必要があるほぼすべてを見つけます。 多くの場合、システムを自分で調整しなければならない研究者と学生は、なぜすべてがこのように配置されており、そうでないのかについての実用的な説明を見つけるでしょう。 また、「芸能人」もいます。エンターテイメント、利益、またはその両方のためにコンピューターで時間を過ごすのが好きなユーザーです。 なぜ機能するのか、そうでないのかを知りたいですか？ 何かを変更するとどうなりますか？ その後、あなたは「芸能人」の数に属します。



前提条件


この本を読むためにプログラマーである必要はありません。 基本的なコンピューターユーザースキルのみが必要になります。グラフィカルインターフェイス（システムインターフェイスのインストールおよび構成時）をナビゲートする必要があり、ファイルとディレクトリ（フォルダー）の概念も必要です。 また、システム上およびオンラインで追加のドキュメントを探す準備をする必要があります。 上記のように、最も重要なことは、コンピューターを探索する意欲と欲求です。



本の読み方


技術的なトピックに関しては、必要な知識をすべて伝えることは簡単なことではありません。 一方で、人間の心は単に多数の新しい概念を同時に処理できないため、読者は過度の詳細に行き詰まり、本質をほとんど理解しません。 他方、詳細の欠如は、読者が主題の漠然とした考えのみを受け取り、さらなる資料を吸収する準備ができていないという事実につながります。



この本では、著者はプレゼンテーションを簡素化し、資料を構造化しました。 ほとんどの章では、さらなる作業に必要な重要な情報が最初に提供されます。 この章を読むと、その中に追加の資料があります。 これらの詳細をすぐに学ぶ必要がありますか？ ほとんどの場合、著者はそうではないと考えています。 勉強したばかりの資料に関連する多くの詳細を見て目が薄れ始めたら、ためらわずに次の章に進んでください。 他の重要なことがあなたを待っています。



この本はどのように機能しますか？


本の冒頭で、Linuxシステムの概要を説明し、その後、システムでのさらなる作業に必要なツールを使用した一連の実用的なタスクを提供します。 次に、システムが起動する通常の順序に従って、機器管理からネットワーク構成で終わる、システムの各部分を詳細に調べます。 そして最後に、稼働中のシステムの詳細の一部を把握し、重要なスキルを習得し、プログラマーが使用するツールについても理解します。



最初の章のほとんど（第2章を除く）にはLinuxカーネルが積極的に関係していますが、本を読み進めていくと、ユーザー空間で作業することになります。 著者が今話していることを理解していなくても、心配しないでください。説明は第1章にあります。資料は、システムの分布を参照せずに可能な限り提示されます。 システムのすべての亜種を説明するのは退屈なので、WardはDebian（Ubuntuを含む）とRHEL / Fedora / CentOSの2つの主要なディストリビューションファミリについて話そうとしました。 サーバーのバージョンとワークステーションのバージョンに重点が置かれています。 AndroidやOpenWRTなどの実装されたシステムも表示されますが、これらのプラットフォームの違いを見ることができます。



第2版​​の新機能


この本の初版は、主にLinuxのユーザー側を扱っています。 主な注意は、部品の配置とそれらを機能させる方法に向けられました。 当時、システムの多くの要素をインストールして正しく構成することは困難でした。



ソフトウェア開発者とLinuxディストリビューションの作成者の努力のおかげで、状況は変わりました。 ウォードは、アップデートを求めて第1版の資料を改訂しました。彼は、システムのロードプロセスと機器の管理方法に特別な注意を払い、古い資料（たとえば、印刷プロセスの詳細な説明）を削除して、各ディストリビューションでのLinuxカーネルの役割の議論を拡大しました。 あなたはおそらくあなたがそれについて知っているよりもはるかに頻繁にカーネルと対話し、著者はこれが起こる瞬間を具体的に指摘しました。



ウォードはまた、現代の読者の興味とニーズに合うように、本の資料のプレゼンテーションの順序を変更しました。 変更されていない唯一のものは、本のボリュームです。



著者は、すぐに始めるために必要な情報を提供したかったのです。 それらを学ぶことはいくらかの努力を必要としますが、ウォードはあなたがこの本を打つためにあなたを「重量挙げ者」にするつもりはありません。 ここで説明した最も重要な点を理解すると、詳細を見つけて理解することは難しくありません。



著者は、主にあなたの注意を集中させるために、初版にあった歴史的な詳細のいくつかを削除しました。 LinuxとUnixの歴史との関係に興味がある場合は、Peter Da Salus、The Daemon、Gnu、およびPenguin（Reed Media Services、2008）をご覧ください。私たちのソフトウェア。
