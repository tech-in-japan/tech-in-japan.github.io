Hyper-Vでのゲスト仮想化

それが起こったのは、アセンブリ10565でWindows Insider PreviewがHyper-Vでのゲスト仮想化の最初のプレビューを公開したことです。 このニュースは興味深いものですが、ここにあるようには見えませんでしたが、かなり奇妙です。







これが最初の兆候なので、いくつかの制限があります。



1.両方のハイパーバイザーがプレビューバージョン10565である必要があります。そうでない場合、起動しません。

2.ゲスト仮想化中、仮想マシンでは次の機能を使用できません。動的メモリ、オンザフライメモリ変更（Win10の新機能）、保存\復元、ライブマイグレーション。

3.ゲスト仮想化を有効にする場合、MACスプーフィングを有効にする必要があります。有効にしないと、ゲストゲストのネットワークが機能しません。

4.ゲスト仮想化を使用する機能を有効にする前に、仮想化ベースのセキュリティをオフにする必要があります。

5. Intelプロセッサーのみ（Intel-VTが必要）。

6.多くの（！）RAMが必要だと言われますが、後の記事でラップトップで4 GBで十分だと書いています;）



有効にする方法：



1.仮想マシンを作成し、Windowsをインストールして、オフにします。

2. スクリプトを実行します 。 スクリプト自体がすべての設定を行います（上記の制限を修正し、同意を求めます）。

3.仮想マシンをオンにし、ホストからpowershell Directを使用してHyper-Vをインストールします。

Invoke-Command -VMName "" -ScriptBlock { Enable-WindowsOptionalFeature -FeatureName Microsoft-Hyper-V -Online; Restart-Computer }
