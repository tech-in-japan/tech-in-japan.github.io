GmailからGoogle Appsにメールを転送する





Google Appsの基本パッケージを使用することに決め、その前にGmailを使用し、古い文字が必要な場合、 Google Email Uploaderがキリル文字で正しく動作せず、一般に移行および移行時に文字が失われるという事実にすぐに気付くでしょう。 Google Appsは無料版では使用できません。 もちろん、POPを介してすべてをダウンロードできますが、新しいメールで送信されたフォルダーは空になります。 おそらく試用版プレミアムパッケージをリクエストして移行ツールを使用できます（試用版で利用可能ですか？）、ただし、少なくとも10人のユーザーがいる場合、カードで500ドルをブロックする必要がありますが、お金はわずかですが、これは不必要な問題です。



カロチェ。 移行はimapsyncで素晴らしいです。 ポピーでは、 ポートから問題なくインストールされますが、他のシステムでも何らかの方法でインストールされます。 インストール後、 readmeとfaqを読む必要があります。 そこで、特定の再配置スクリプトへのリンクを見つけることができます。



いつものように、スクリプトはすぐに動作しませんでした。まず第一に、引用符とハイフンは同じではありませんが、主なものはスクリプトの不当な期待に関連するエラーです：「8033バイトを予期しましたが8238を受信しました」それを取る。



imapsync --host1 imap.gmail.com --port1 993 \

--user1 putin@gmail.com --reconnectretry1 5000 --passfile1 ./passfile1 --ssl1 \

--host2 imap.gmail.com --port2 993 \

--user2 medvedev@kremlin.ru --reconnectretry2 5000 --passfile2 ./passfile2 --ssl2 \

--syncinternaldates --split1 100 --split2 100 \

--authmech1 LOGIN --authmech2 LOGIN \

--allowsizemismatch \

--justfolders



imapsync --host1 imap.gmail.com --port1 993 \

--user1 putin@gmail.com --reconnectretry1 5000 --passfile1 ./passfile1 --ssl1 \

--host2 imap.gmail.com --port2 993 \

--user2 medvedev@kremlin.ru --reconnectretry2 5000 --passfile2 ./passfile2 --ssl2 \

--syncinternaldates --split1 100 --split2 100 \

--authmech1 LOGIN --authmech2 LOGIN \

--allowsizemismatch \

--regexmess 's/Delivered-To: putin\@gmail.com/Delivered-To: medvedev\@kremlin.ru/g' \

--regexmess 's/<putin\@gmail.com>/<medvedev\@kremlin.ru>/g'









--reconnectretry *および--allowsizemismatchの2つのパラメーターを追加する必要がありました。



最初の呼び出しでラベルが作成され、2番目の呼び出しで文字があふれます。 フィルタを転送するためだけに残りますが、そこにxmlをダウンロードして記入する必要があります-問題はありません。



彼らによると、パスワードはファイルに保存するのが最適です。そうしないと、パスワードを失う可能性があります。



著者の台本にも、主題のない手紙を「主題なし」と呼ぶようになった正規表現がありますが、誰にそれが必要か、そしてなぜ、私は理解しなかったので、私はそれを使用しませんでした。



大きなデバイスがあり、全員がGmailを使用している場合、バッチ転送用のスクリプトを作成するのは難しくありません。



ユーティリティは、ギガバイトのメールを5時間で自宅のコンピューターに転送しました。
