そうしないとデイブがあなたのところに来るからです。



いいえ、それは真実ではありません、デイブはとにかくあなたに来ます。 そして、レナートと他の多くの善良な人々。







（警告：テキストは、レナートの最新の功績の悲しい名声を記念して、悲観的でイロニックなスタイルで意図的に書かれました。このスタイルを本来維持できない場合は、モニターから無邪気な猫と妊娠中の男性を削除し、それ以上読みません。）



私たちは代替の現実に現れ続けており、今回はテストではありません。 より正確には、そうではありません。



Lennartは頑固かつ永続的にコーディングを行っており、systemd 38リリースを開始して既に同じMagazineを実装していますが、その安定性は疑わしいです。 41番目のバージョンは[テスト]にすでに登場しており、コンピューターに送信されています！



systemd v42は[追加]になります。 はい、そうです。



v41以降、重要なバグのほとんどが修正されましたが、バージョン42で修正される可能性のあるいくつかの迷惑な誤解がまだあります。実際に何が起こるかを想定すると、システムメンテナーのDave Reisnerは、選択せずに安定した余分なリポジトリに置きます。



これは、システムを使用しない人々にとって何を意味しますか？



  • なし。 一般的に。 あなたは以前のように生きることができ、誰もあなたの人生を邪魔することはありません。スキニーのぬるぬるした生き物は夜にあなたのパンツに忍び寄ることはありません。 あなた（あなたにとって最も重要な）の親relativeや動物は生き続ける可能性があります。
  • しかし、あなたは自分の魂に一滴の恐怖を抱くことができます。
  • おそらくこれ以上読むことはできません。




しかし、まだシステムを使用している場合はどうでしょうか？



  • 新機能！ systemd-cat、systemd-cgtop、systemd-journalctl。 機能とバグ修正の完全なリストが本当に必要な場合は、gitのNEWSファイルを読んでください。
  • 同じログはデフォルトで有効になるため、syslogを実行する必要はなくなりました（syslog-ng、rsyslog、any）。 同じジャーナルは、デフォルトで、/ run / systemd / journalに書き込みます（つまり、再起動するとバイナリログが所属する場所に移動します）。 突然ログをメモリに残したい場合は、/ var / log / journalを作成するだけで、同じJournalがそれを独自に検出し、マスターし始めます。 本当にsyslogから離れたい場合は、/ dev / logではなく、/ run / systemd / journal / socketに書き込むように彼に伝えてください。
  • systemadmは、別のリポジトリ（上流）に割り当てられました。 したがって、AURにはsystemd-ui-gitが登場しました。 彼が十分な票を獲得した場合、彼はキリスト教の司祭からの証明書を持つ白人として育ち、[コミュニティ]に送られます。
  • systemd-arch-unitsおよびsystemd-initscriptsで何も変更されません。 これらのパッケージは[コミュニティ]に残ります。 Manteinerは、これらのパッケージがなくなることを楽観的に望んでいます。 一部のアップストリームプロジェクトでは、その機能が停止しています。




デフォルトのシステムとしてsystemdをすぐにインストールする計画はありません。 しかし、一部の開発者によると、長期（5年間）でsystemdの本当の代替手段を見つけられません。 コア/エクストラの多くのパッケージはすでにsystemdをサポートしており、将来このディストリビューションは進歩するだけです。



そして、はい、それは代替現実ではありませんでした。 最も一般的。 紳士の皆さん、アップデートを待ってください。フォースがあなたと共にいることを願っています！
