Apple、オープンソースのZFSプロジェクトを閉鎖

Appleは本日、オープンソースのZFSプロジェクトの完了を発表しました。 メーリングリストとリポジトリも間もなく削除されます。



ZFS （Zettabyte File System）は、元はSun Microsystemsによって開発された高度なファイルシステムであり、AppleによってMac OS Xに移植され、2007年以降Mac OS Forgeページで利用可能なオープンソースプロジェクトとして提示されました。



噂によると、ZFSはMac OS X「Leopard」のデフォルトのファイルシステムになるはずでしたが、最終的には読み取り専用機能を備えた切り捨てられたバージョンの形でしか現れませんでした。



Appleは、開発されたSnow Leopard Server OSの初期の説明で明らかになったように、ZFSへの関心を維持し続けましたが、ZFSへのすべての参照はすぐにサイトから削除され、このファイルシステムのサポートはSnow Leopardに含まれず、 Snow Leopard Server。」



後に、いくつかのソースは、Snow LeopardのZFSサポートは、製品のライセンス問題のために行われなかったと示唆しました。 Sunは商用ライセンスCDDLの下でファイルシステムのコードを開くことを選択したことを思い出してください。これはAppleのビジネスに追加のリスクをもたらしました。 2009年4月、SunはOracle Corporationに買収されました。OracleCorporationは、独自の「クローン」ZFS（ Btrfs）を既に所有しており、最初の問題の運命を定めています。 さらに、ZFSには、 NetAppが所有する特許侵害に関して未解決の質問がいくつかあります。 これらはすべて、Appleの肩にかかる可能性があります。



Cupertinoの会社が昨日、ファイルシステムエンジニアとして仕事を投稿しました。 ただし、この説明には、既存のプロジェクトをサポートするための要件だけでなく、この分野の将来の技術の開発、サポート、および実装のみが含まれていました。 この事実は、会社がZFSの追求よりも独自のソリューションを好む可能性が高いことを示しています。
