GPRSは通話の障害ではありません

この記事は、オープンGPRS / EDGEセッション中に音声サービスを作成/受信する機能に関する質問、およびロシアの特定の「楕円形」オペレーターの問題に関する記事で表明されたこの問題に対するinりに触発されています。

TyMaHの瞬間を払拭するために、アクティブなGPRS / EDGEセッションで音声通話を受信/発信する機能を実現するために必要な技術的前提条件を検討します。





したがって、パケットデータ転送のアクティブセッション中に、SMSを送受信するために必要なものは何ですか。 実際、それほど多くは必要ありませんが、より具体的には：

  • SGSN 「オームとMSC / VLR 」オーム間のG sインターフェースの存在
  • ベースステーションコントローラー（ BSC ）サポートネットワーク操作モード[NOM] I
  • 特定のクラスのモバイル端末の存在。2つのサービス（音声およびパケットデータ）の同時使用を可能にします。


G sインターフェイス



したがって、最初の要件は、これまで知られていなかったインターフェイスの存在です。 これは、SGSNやMSC / VLRなどのネットワーク要素間のインターフェイスです。 インターフェースはオプションです。 一般的なGPRS / EDGEネットワークアーキテクチャではオプションであり、実際に次の手順を実行できます。

  • RA / LAの統合アップデート
  • 非GPRSアラート
  • MS情報お問い合わせ
  • MM情報
  • リセット（MSC / VLR <-> SGSN）。
  • MM情報
  • 非GSMシグナリングのトンネリング
  • MM情報
  • CAMEL Any Timeによる位置情報の取得

UMA / GAN（en）

UMA / GANテクノロジーの最も一般的な用途は、モバイル端末の使用です。これにより、ワイヤレスLANとWANネットワークの使用を「切り替える」ことができます。 LANとして、いわゆる ライセンスのない802.11テクノロジ、およびWANネットワークは、通常、GPRS / EDGE / UMTSアーキテクチャで表されます。 同時に、加入者の端末が無線LANネットワークのカバレッジエリアにある間、安全な接続を介して接続されます。信号品質が低下すると、端末はGPRS / EDGE / UMTSアーキテクチャに「切り替わります」。 UMA / GANアーキテクチャでは、ベースステーション（BTS）の概念の代わりに、物理的にWi-Fiアクセスポイントに非常に似ているGANC [GAN Controller]の概念が使用されます。 このコントローラは、GPRS / EDGEアーキテクチャのベースステーションコントローラに直接接続されています。





注意を喚起したいのですが、スイッチ（MSC）をベースステーションコントローラ（BSC）に接続するために使用されるサブシステムは、インターフェイスAのBSSAP（en） [ベースステーションシステムアプリケーションパーツ]サブシステムです。およびMSC / VLR'om-BSSAP +インターフェースG s 、つまり このサブシステムを使用すると、SGSNがスイッチ（MSC / VLR）の基地局コントローラー（BSC）になりすますことができるため、SGSNを介して加入者の音声サービスを実行/受信することができます。



上記のサブシステム（BSSAP +）は、SS7スタックの基礎レベルに基づいており、SSN7 / SS7の一般的なスキームの他のサブシステムとともに、右の図に示されています。



ネットワーク操作モード



NOMパラメータは、基地局コントローラ（BSC）の側で設定され、基本的に加入者のページング手順が「標準」GSMチャネルを通過する方法、またはパケットサブネットのページングチャネルがアクティブになる方法を決定します。



基本的なネットワーク操作モードがいくつかあります。

  • 私は

    このモードでは、パケット交換および回線交換加入者データのシグナリングは、GSMページングチャネルPCH [Paging CHannel]と、GPRSページングチャネル-PPCH [Packet Paging CHannel]の両方で実行できます。 これにより、パケットデータとともに音声サービスを同時に使用することができます。 さらに、PPCHを使用する場合、シグナリングは必然的にG sインターフェイスを介してSGSNを介してMSC / VLRに渡されます。これは、GSM / GPRS接続手順およびロケーション更新手順にも使用されます。

    加入者がGPRS / EDGEセッションを開いているときに音声サブネットをページングする必要がある場合、回線交換サブネットではないPACCH [Packet Associated Control CHannel]チャネルを介してモバイル端末に通知されます。アクセスできます。

    GPRS / EDGEセッションがアクティブ化されていない場合、ページングは​​「標準」GSM PCHまたはPPCHチャネルを通過します。 NOM Iは、G sインターフェースを有効にできる唯一のモードです。
  • NOM II

    これは3つのモードの中で最も単純であり、パケット交換回線と回線交換回線のオペレーターサブネット間に直接の信号リンクがないため、PPCHチャネルは使用できないと想定されています。 この場合、加入者はアクティブなGPRS / EDGEセッションで電話を受信/発信できません。
  • NOM III

    このモードでは、G sインターフェイスがアクティブになっていないと想定されるため、PCHチャネルを介して回線交換ページングを実行する必要があります。 また、このモードでは、一般的なGPRS制御チャネルが利用可能です-PCCCH [パケット共通制御チャネル]とそのサブチャネル：PPCH、PRACH [パケットランダムアクセスチャネル]、およびPAGCH [パケットアクセス許可チャネル]。これは、パケット部分のすべてのページングが「独自の」GPRSチャネルを通じて実装されます。

    このモードは、PCH GSMチャネルの負荷を軽減するために、NOM IIモードよりも優先される場合があります。


加入者端末に、オペレーターのネットワークで現在使用されているGPRSモードを通知するために、GSMブロードキャストBCCH [Broadcast Common Control Channel]が使用されます。



Netmonitorなどの特殊なアプリケーションを使用して、オペレーターが現在使用しているモードを確認できます。



モバイル端末クラス



音声サービスの作成/受信を可能にする2番目の重要なポイントは、端末クラスです。 以下は、モバイル端末の主要なクラスです。

  • クラスA

    クラス「A」では、GPRSを介してデータを受信/送信しながら音声トラフィックを送信する可能性があります。
  • クラスB

    GPRSクラス「B」の携帯電話は、GPRSサービスとGSMサービスを自動的に切り替えることができます。 GPRS接続がオンの場合、着信または発信SMSで-通話が終了した後、接続が一時的に中断されます-GPRS接続が自動的に再開します。 着信コールと発信コールでも発生します。
  • クラスc

    GSMとGPRSサービスの両方を使用することはできません。 たとえば、アクティブなGPRSセッションでは、インターネットに接続している場合、通話を受信したりSMSを送信したりすることはできません。


ユーザーが現在使用しているデバイスのほとんどはクラスBの「代表者」であり、したがって、G sインターフェイスがないと、音声サービスを受信/作成できません。 この状況では、あなたに連絡しようとしている加入者は、「加入者はネットワークカバレッジエリア外です...」という応答を受け取り、GPRS / EDGEセッションを終了するまで、音声サービス（SMSを含む）のゾーンから出ます。



まとめ



それで、この記事から何を見つけましたか？音声サービスとパケットデータ転送の同時操作のために、いくつかの条件が満たされなければなりません：

  1. 加入者のデバイスはクラスBより低くてはいけません。 クラスCでは、音声サービスとパケット送信を同時に使用できません
  2. オペレータはベースステーションコントローラでNOM Iモードを使用する必要があります
  3. G sインターフェイス*は、オペレータ側で設定する必要があります。これにより、音声サービスを作成/受信する機能を提供できます。

    *-オペレーターのすべての機器がG sインターフェースを実装する機能をサポートしているわけではない場合、そのような状況が発生する可能性があります（もちろん、純粋に仮説的に:)。 たとえば、異なるベンダーまたは異なるソフトウェアバージョン（または古いハードウェアの使用）が原因でオペレーターのすべての配電盤（MSC / VLR）がSGSNとMSC / VLR間の接続を提供できるわけではなく、不注意で国境にいることに気付くでしょう異なるスイッチのサービスエリア、次に、一方ではアクティブなGPRS / EDGEセッションを通過できるようになり、他方では（G sインターフェイスを編成できない場合）-いいえ。


これらの条件が満たされない場合、GPRS / EDGEセッションを開いている加入者に電話をかける（SMSを送信する/ CSD通信を確立する）ことはできません。
