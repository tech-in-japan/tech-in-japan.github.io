例としてWindows Embedded x64を使用した自動OSインストール

例付きの手順説明



誰でもWindowsオペレーティングシステムを手動でインストールおよび構成できます。 しかし、実稼働目的でこのタスクを数十回または数百回実行する必要がある場合、自動インストールが必要になります。



この記事では、オペレーティングシステムの「サイレント」自動インストールを作成し、構成して結果をテストする方法に関する基本的な手順とヒントについて説明します。 もちろん、ネットワークでは、そのようなタスクの個々のフェーズに関するかなり多数の記事とヒントを見つけることができますが、一般的な指示と質問への回答はどこから始めますか？ 何を準備する必要がありますか？ プロセスの主なステップは何ですか？ -残念ながら、珍しい。



したがって、これらの質問への回答は、このようなタスクを初めて実行する人によってここで見つけられます。



例として、 Windows Embedded Standard 7x64がオペレーティングシステムとして選択されています。



稼働中のマシンの要件：



インストールする必要があります：


  • Windows Embedded Standard 7 SP1（ツール）

    ツールキットはここからダウンロードできます （評価版）

    以前のバージョンのWindowsでは、 Windowsシステムイメージマネージャー （SIM）を使用して応答ファイルを編集しました。 Windows SIMは、 ここからダウンロードできるWindows自動インストールキット（AIK）の一部です 。 Windows Embedded Standard 7 Toolsのイメージ構成エディターインターフェイスは、 Windows SIMインターフェイスに非常に似ています。


利用可能でなければなりません：


  • Windowsをインストールするための元のISOイメージ（評価版をインストールする予定がない場合は、+ライセンスキー）。
  • ターゲットデバイスに必要なすべてのドライバー。
  • OSのインストール直後にシステムにインストールする予定のアプリケーション。


基本的なツール：


  • Windows Embedded Standard 7 SP1（ツール）\ Image Configuration Editor（ICE）
  • Windows Embedded Standard 7 SP1（ツール）\ Windows PEツールコマンドプロンプト
  • Symantec Ghost \ Ghostブートウィザード
開始する前に、これらのツールとWindowsのインストールのフェーズに精通することをお勧めします。



作業環境のフォルダー構造：


作業フォルダーを作成します。 たとえば、「Work」という名前のフォルダー。

作業ディレクトリにWindowsをインストールして、元のISOイメージの内容をコピーします。 結果は次のようになります。







Distribution Share（DS）フォルダーの2つのサブフォルダーに注意してください：「 $ OEM $ Folders 」および「 Out-of-Box Drivers 」。



「Work \ DS \ Out-of-Box Drivers \」フォルダを作成します。 これにより、ターゲットデバイスに必要なドライバーが保存されます。



「Work \ DS \ $ OEM $ Folders \（CustomFolder）\ $ OEM $ \ $$」というフォルダ構造を作成します。 追加のファイル、アプリケーション、レジストリファイル、構成、およびスクリプトはここに保存されます。



\ $ OEM $ \ $$フォルダーの内容は、％WINDIR％フォルダーに自動的にコピーされるため、Windowsインストールのどの段階でも使用できます。 $ OEM $フォルダーの詳細 。



自動インストールを作成する一連の手順：

手順1. AutoUnattend.xml応答ファイルを作成します。

ステップ2. OSの自動インストールプロセスにドライバーを追加します。

手順3.応答ファイル、スクリプト、およびコマンドを使用して、アプリケーションとシステム設定を追加します。

手順4.テストイメージを作成します。

手順5.テストマシンからイメージをキャプチャし、ターゲットデバイスに展開します。



これらのステップは、多くの詳細を含むサブアイテムに分割できます。 しかし、この記事では、タスクの実装の一般的なビジョンのみを提供する予定です。



1. AutoUnattend.xml応答ファイルを作成します。



インストールを完全に自動化するには、少なくともWindowsセットアップウィザードのすべての質問に回答する必要があります。 XML形式の応答ファイルには、これに必要なすべてのフィールドが含まれています。



Image Configuration Editor（ICE）を使用して、自動インストール応答ファイルを作成および編集できます。 もちろん、これには任意のテキストエディターを使用できますが、このタスクに焦点を当てたICEの便利さとGUIはありません。



ICEでは、インストールを完全に自動化するために、応答ファイルを作成し、機能、ドライバー、および必要なすべてのオプションを追加できます。



直感的なユーザーインターフェイスと高度な検索エンジンにより、応答ファイルを作成および編集するツールを選択する際にICEが優先されます。







ICEを開始するには：


  • Open Distribution Share（DS）：この場合、「Work \ DS」になります。
  • 新しい（空白の）応答ファイルを作成します。 コンポーネントとオプションを追加します。


特徴


Windowsの機能（以下、このテキストではロシア語で「コンポーネント」という語が使用されます）は、ターゲットデバイスの要件に応じて追加されます。 ICEは、必要最小限のコンポーネントを追加すると、必要なコンポーネントとオプションのコンポーネントを自動的に追加できます。



デバイスのディスク領域が限られている場合は、推定フットプリントインジケーターに注意してください。これは、インストール後にWindowsがディスク上で取る推定サイズです。







オプション


すべてのコンポーネントには1つ以上のプロパティがあります。 少なくとも、 標準インストールの質問に対する回答を決定する必要があります 。 デフォルトのインストール言語、Microsoftソフトウェアライセンス条項の確認、プロダクトキー、Windowsがインストールされるドライブおよびセクション、ローカルインストールです。



次の[Windowsへようこそ]設定を自動的に設定することもできます：言語と地域、ユーザー名、コンピューター名、デスクトップの壁紙、日付と時刻、現在の場所、ホームグループ、ワイヤレスネットワーク、管理者パスワード。

すべてのオプションは、Windowsインストールの1つまたは複数のフェーズで構成されます。 特定のオプションを構成する予定の特定のフェーズに注意してください。



ディスクのセットアップ


Windows（ Windows PE ）のインストールの最初のフェーズでは、物理ディスクパーティションの作成と削除を構成できます。 一連のオプションにより、既存のパーティションの削除、新しいパーティションの作成、タイプの指定、パーティションのフォーマット、文字とラベルの設定、Windowsをインストールするドライブの選択ができます。



DiskConfiguration / Disk / CreatePartitionsオプション用の新しいディスクパーティションを作成するには、コンテキストメニューで[ 新しいCreatePartitionの挿入 ]アクションを選択します。







表示されるCreatePartitionブランチで、次のように入力します。

  • 作成されたセクションのシリアル番号（ Order ）;
  • タイプ（ タイプ ）-プライマリ、EFI、拡張、論理、またはMSR。
  • 大きさ
  • そして、パーティションが固定サイズか拡張可能か（ Extend ）-trueまたはfalse。 Extend = trueの場合、Sizeオプションを設定する必要はありません。


DiskConfiguration / Disk / ModifyPartitionsオプションの作成されたパーティションに特性を追加するには、コンテキストメニューで「Insert New ModifyPartition」アクションを選択します。







表示されるModifyPartitionブランチで、次のように入力します。

  • シーケンス番号（ Order ）-CreatePartitionセクションのシリアル番号と一致する必要があります。
  • セクションラベル（ Label ）;
  • セクションレター（ Letter ）;
  • 形式-NTFSまたはFAT32;
  • セクションが固定サイズか拡張可能か（ Extend ）-trueまたはfalse。
  • セクションアクティビティ（ アクティブ ）-trueまたはfalse。
  • セクションタイプ（ TypeID ）。 たとえば、システム回復ツールの格納に使用されるシステムパーティションの場合、TypeIDは0x27に設定する必要があります。








ImageInstallオプションを使用すると、オペレーティングシステムをインストールするパーティションを選択できます 。







応答ファイルを確認して保存します


エラー応答ファイルを確認するには、 F5を使用します 。 警告とエラーがメッセージバーに表示されます。







必要なパッケージが見つからない場合は、キーボードショートカットCtrl + F5を使用して自動的に追加できます。

Ctrl + Shift + F5は、すべての必須パッケージとオプションパッケージを追加します。 このオプションを使用する場合、フットプリントを大幅に超える可能性があることに注意する必要があります。



応答ファイルにはAutounattend.xmlという名前を付け、作業ディレクトリのルートに保存する必要があります。



2. OSの自動インストールのプロセスにドライバーを追加します。



自動インストールでドライバーを有効にするには、事前に公式Webサイトからドライバーをダウンロードする必要があります。

  • ドライバーインストールファイルを含むフォルダーをOut-Of-Boxドライバーに追加します。
  • Windowsインストールフェーズのいずれかで、各ドライバーへのパスを追加します。 注 ：特定の段階で（特定の段階で）特定のドライバーをインストールする必要があります。




3.応答ファイル、スクリプト、およびコマンドを使用して、アプリケーションとシステム設定を追加します。



追加のファイルをインストールに追加するには、 $ OEM $フォルダーが使用されます。 $ OEM $ディレクトリとそのサブフォルダーは、Windowsインストールによって自動的に検出され、ハードドライブにコピーされます。

たとえば、「DS \ $ OEM $ Folders \\ $ OEM $ \ $$」からのすべてのファイルとフォルダーは、％WinDir％にコピーされます。 そこからスクリプトを実行し、アプリケーションをインストールできます。

$ OEM $フォルダを使用するには、次の手順を実行します。

  • 応答ファイルで、 「UseConfigurationSet」を「true」に設定します
  • ICEの[配布共有]パネルで、$ OEM $ Foldersを展開します。




  • CustomFolderを右クリックして、「Oemフォルダーパスの挿入」を選択します。




OSのインストール直後にアプリケーションをすぐにインストールし、設定を行い、レジストリエントリとファイルを追加するには、次の方法を使用できます。



  • 応答ファイル内のRunSynchronousおよびRunAsynchronousコマンド。
  • 応答ファイルのFirstLogonCommands 。 これらのコマンドは、システムへの最初のユーザーログオン時に自動的に実行されます。
  • SetupComplete.cmdは、OSインストールの終了時に自動的に開始します。 Windowsインストールは、 「％WINDIR％\ Setup \ Scripts」ディレクトリでSetupComlete.cmdという名前のファイルを検索するため、このファイルを「DS \ $ OEM $ Folders \（CustomFolder）\ $ OEM $ \ $$ \ Setup \ Scriptsディレクトリにコピーします。 」作業フォルダ内。


これらの方法にはそれぞれ長所と短所があります。 そして、コマンドの実行のコンテキストを必ず考慮してください。



OSインストール中のアプリケーションのサイレントインストールでは、適切なキーを使用します。InstallShieldインストールの場合は/ s 、 / silentまたは/ rをsetup.issファイルで、MSIパッケージの場合は/ qn 、InnoSetupの場合は/ verysilent、Wiseinstallerの場合は/ sなど ヘルプと/？を使用します アプリケーションをインストールするためのコマンドラインオプションに慣れる。



4.テストイメージを作成します。



テストイメージは、ISOイメージまたは起動可能なUSBとして作成できます。



WorkフォルダーがCドライブのルートにある場合に、oscdimgツール（％ProgramFiles％\ Windows Embedded Standard 7 \ Tools \ AMD64 \ oscdimg.exe）を使用してISOイメージを作成する例：



oscdimg -bC：\ work \ boot \ etfsboot.com -u2 -hmc：\ work c：\ work \ wes7x64_test.iso



テストのために、テスト中はインストールに標準ツールを含めて、必要なすべての構成（アクセサリ、Microsoft管理コンソールなど）を確認することをお勧めします。



5.テストマシンからイメージをキャプチャし、ターゲットデバイスに展開します。



テストマシンからイメージを作成してターゲットデバイスに適用する場合は、まずSYSPREPを使用してテストマシンを準備する必要があります。 これはMicrosoftの要件です。 それ以外の場合、マイクロソフトは製品サポートを提供しません。



Sysprepは、複製、監査、およびクライアントへの配信のためにMicrosoft Windowsインストールを準備します。

Sysprepは、ComputerSIDなどのシステム固有のデータをWindowsから削除します。 したがって、SYSPREPを使用したトレーニングは、イメージ作成の重要なステップです。



ImageXまたはSymantecGhostツールを備えたWindowsPEを使用してイメージを作成できます。

WindowsPEツールのコマンドプロンプトを使用して、wimファイルの自動作成で起動可能なWinPEイメージを作成できます。

Symantec Ghost \ Ghostブートウィザードを使用すると、自動化されたゴーストファイルの作成で起動可能なゴーストイメージを作成できます。



これらの各画像は、ターゲットデバイスへのさらなる配信に使用できます。



結果のイメージは、ライトタッチおよびゼロタッチ展開戦略で使用できます。
