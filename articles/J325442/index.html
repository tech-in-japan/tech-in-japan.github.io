「優れた外部制約は創造性の基礎です」：Sberbank Technologies、Java、およびNovosibirskのOleg Chirukhin

ノボシビルスクでJava会議を手配する場合、これはどこからでもすべての興味深いスピーカーを連れて行く必要があるという意味ではありません。市には独自の著名な人物がいます。 その中のOleg olegchir Chirukhinは最近のJavaハブポストで多くの意見を獲得しており、今後のJBreakで「DevOps：Now Java Does Not Slow Down」というタイトルのプレゼンテーションを行います 。



JBreakを見越して、私たちはOlegにさまざまなトピックについて尋ねました。彼が働いているSberbank-Technologiesから、プログラマにとってモスクワよりも優れたNovosibirsk、PHPよりもJavaが優れています。



-スベルテクであなたは正確に何をしますか？あなたはどのチームで働いていますか？あなたの役割は何ですか？



-私たちのチームは、BPMNからビジネスプロセスを起動するためのエンジンを開発しています。 このエンジンは、プラットフォームの技術コアの一部です。 私は、プロジェクトのDevOpsの作成を含む、時にはグローバルなものを含む、アーキテクチャに携わっています。



-銀行プラットフォームの開発は常に巨大な責任であり、Sberbankの場合は巨大な規模でもあります。 同時に、これらの2つの要因は、あなたがいかに正確に働くことができるかについて多くの制限を課す可能性があります。 これらの制限は干渉しませんか？



-優れた外部制約は創造性の基礎です。 クリエイティブな人がまったく制限を設定する必要がない場合、不定形で面白くない何かが生じます。 アーティストは視覚的手段の枠組みの中で作成し、条件、TK、予算などの枠組みの中で作成します。 クリーンで美しく、強力なソフトウェアソリューションが生まれるのは、制限と機会の接点です。



とにかく、制限は天井から取られるのではなく、最高のアナリスト、建築家などによって作成されます-これらの人々とチームで働き、彼らのベストプラクティスを使用することは物理的に快適です。



さらに、非常に大規模で野心的なプロジェクトがSbertechで行われています。 彼らの存在は、官僚的な問題があれば、うまく解決されていることを裏付けています。



-Sbertechまで驚くほど多様な実績があります。システム管理者からゲーム開発者、IDEでの作業、State Servicesまで。 このようなキャリアはジグザグのように見えます。各段階ですべてを新たに学ぶ必要がありますが、実際には他の分野で得られた経験はどれほど有用ですか？



-キャリアはジグザグのようです。 私はマーケティングディレクターではなく、プログラマーです。 プログラマーの観点から見ると、どこでもほぼ同じテクノロジーを使用しています。 最初は退屈に思えるかもしれませんが、実際には、数百万の合格したフレームワークを探索する代わりに、本当に新しくて面白いものに集中して時間を費やすことができます。



優れたコンピューターゲームとボードゲームで類似点を描くことができます。ルールの小さなセットは、多数のユニークなゲーム状況を作成し、さまざまな問題の解決に適しています。 これはゲームでは簡単ですが、現実の世界では簡単ではありません。



Javaは、そのような些細なことにすべてが考え抜かれている、まさに珍しいエコシステムです。 デザインパターンやSpringフレームワークのようなものを一度研究したことがあるので、ゲームや情報システムでも使用できます。 新しい仕事に来ると、通常、すべてが見慣れたものになります。



この点で、Sbertechではより困難でした。プラットフォームのほとんどは、Sbertechが独自に作成したものです。 一方、プラットフォームはよく知られたアイデアに基づいており、それらを非常にうまく実装しているので、とにかくすぐに知ることができます。
