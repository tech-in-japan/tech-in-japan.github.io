Zend Framework 3ロードマップを発表

Zend Framework 3のアイデアと提案の議論は長い間続いており 、最終的にZend はフレームワークの新しいバージョンを発表することにしました 。



Zend Framework 3はZF2の進化版であり、3つの主要なポイント-シンプルさ、コードの再利用、パフォーマンスに焦点を当てています。



優先プランには以下が含まれます：

  • フレームワークのコンポーネントを個別のバージョン管理されたプロジェクト（hello、Symfony！）に分離します。
  • PSR-7と一致するHTTPメッセージに特に重点が置かれます。
  • 既存のフルスタックMVCフレームワークは更新され、再利用と単純化のために新しい独立したコンポーネントに依存します。 ZF2 MVCプロジェクトは、最小限の労力で新しいバージョンに移行できます。このバージョンについては、移行ガイドを準備します。
  • MVCフレームワークスタック（ これが何であるかの例）のより軽い代替として、Ruby / Pythonの多くに馴染みのあるミドルウェアコンセプト、およびスリムフレームワークとStackPHP PHPフレームワークを使用します。
  • Apigilityをミドルウェアとして使用する機能。
  • PHPはバージョン5.5からサポートされますが、PHP 7の最適化。


そして今、主な驚き-精神的な作業とコーディングの深刻な部分が残されているため、ZF3は2015年の第3四半期にリリースされます。



その他の詳細-リリースの特定のタイムライン、特定の技術選択の開発および正当化の方向を含む-は、近い将来Zendブログで公開されます。
