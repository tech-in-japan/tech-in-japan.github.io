ntfsclone-他の人には手に入らない方法でイメージを作成します

最近、Windows 7を搭載したコンピューターに出会いましたが、Windows 7はインターネットとの通信を定期的に拒否しました。 詳細は舞台裏に残しますが、問題は明らかにハードウェアにあるのではないということだけを述べます。 誤動作を修正する必要があり、たまたま私はこの仮定の実装の最初の候補者でした。



システムを再インストールすることにより、すべての妨害を修正できます



私の管理者はあまりよくありません。ルーターを簡単に構成します。自発的な切断などの重要なことを修復するのは言うまでもありませんが、そのような場合は経験豊富で洗練された人はウイルスをチェックすることを推奨し、それが助けにならなければシステムを再インストールします。



コメントの怒りを予想して、余談を言いたい。 再インストールには、たとえば1日かかり、資格認定中の根本原因の検索は少なくとも1週間続きます。そのため、思慮深く体系的なアプローチはここでは選択肢ではありません。



そして、すべてがうまくいかない場合



そのため、確認し、その場合は再インストールします。 しかし、もちろん、コンピューターはその上で動作するために使用されます。もちろん、必要で重要なファイルの束があり、それを使用する従業員は当然手元にありません。 はい、そうであっても-私の苦い経験は、保存するファイルを尋ねられたとき、平均的なユーザーが通常「すべて！」と答え、穏やかに彼の日常生活に戻ることを示しています。 また、これらのファイルの一部は、誰も存在していなかった場所に必ず存在します。



セクションの完全なコピーを作成することをお勧めします



この問題の解決策は表面にあります-マシン上にあるすべてのファイルを保存する必要があります。 これは、直接コピーすることで実行できますが、今日のコンピューターではオフィスの標準で微視的であるため、通常はディスク全体の画像を取得する方が高速で便利です。 「はい、これはddrescueの作業です！」と思いました。



しかし、ディスクのより徹底的な研究は、私自身の妥当性を疑いました。 ウィンチェスターは、最高の犬の飼育者の推奨に従って、単一の論理ドライブに分割されました。一般的にはCとして知られています。160ギガバイトのうち22が占有されています。



80％が何も占有されていないディスクのコピーは、ntfscloneを実行することをお勧めします



もちろん、空きスペースをゼロで埋めて、ddrescueにスパースファイルにイメージを保存するように指示することもできますが、最初にこれらのゼロを書き込んでから（イメージを削除するときに）それを読み取る必要があります。 ディスクイメージを小さなフラッシュドライブにコピーする必要性が高い場合、このオプションは通常非常に機能しますが、同じアクロニスの背景に対しては愚かに見えます。



ディスクのパフォーマンスについては事実上疑いがないので、パーティションイメージを作成するために設計されたユーティリティのいくつかを試すのに適切な時期であると判断しましたが、ddやその多くの後継とは異なり、ファイルシステムの機能を考慮します つまり、デバイスのビットごとのコピーは作成しませんが、ファイル割り当てテーブルをスキャンして、実際にデータがある領域のみをコピーします。 ブート可能なUSBフラッシュドライブに直接ntfscloneが見つかりました。これは、ntfsprogsパッケージの一部であり、ご想像のとおり、ntfsファイルシステムで動作するように作成されています。



ntfscloneは、マウントできない形式でイメージファイルを作成できます



Googleとの短いセッションの後に期待したのは、ntfscloneの特定の形式で22ギガバイトのイメージを取得することだけでした。 このイメージをUSBフラッシュドライブに保存して、160ギガバイトすべてに穏やかな環境で展開し、それらにウイルス対策プログラムを設定することができます。



噂によると、ntfscloneを使用して作成されたイメージはimagemountユーティリティを使用してマウントできますが、フラッシュドライブにはなかったため、噂の確認は後まで延期する必要がありました。



ntfscloneは、通常のツールでマウントされた圧縮イメージを作成できます



結果は、私の予想をはるかに上回るものでした。 まず、特別な形式の使用は必要ではなく、含める必要のない特別な機能であることが判明しました。 第二に、通常のntfscloneイメージはスパースファイルとして作成されます。つまり、クローンパーティションで実際に使用されるのとまったく同じくらいのディスクスペースを占有します。



上記の2つの点から、ntfscloneによって作成されたイメージは標準のマウントユーティリティで非常に簡単にマウントでき、どこかに何かを拡張する必要はありません。つまり、22ギガバイトのアーカイブのコピーでウイルスをスキャンできます。彼に！



ntfscloneは、健全なメディアからコピーする優れた仕事をします



名前が示すように、プログラムはntfsでフォーマットされたパーティションでのみ動作するという事実にもかかわらず、そのユーティリティは過大評価されることはほとんどありません！ 80％のブランクディスクのバックアップコピーを無造作に作成するのに、これ以上優れたものはありません。 時間の節約は莫大であり、スペースの節約は計り知れませんし、副作用もありません！



もちろん、これは、動作中のデバイスのバックアップコピーを作成する場合に使用します。 ただし、デバイスが誤動作している場合でも、欠陥はファイルで占められていない領域、つまりコピー時にntfscloneが移動しない場所にある可能性があります。



ntfscloneは明確で使いやすい



ところで、この記事には単一の例はありません。 これは、プログラムの使用が本当にとても簡単だからです！ しかし、例がなければ、もちろん、それは不可能です。



ディスクイメージは1つのコマンドで取得されます。



ntfsclone -o <イメージを保存するファイル> <実行するイメージのセクション>

つまり、C：ドライブイメージをキャプチャするには、win_c.imgファイルに書き込む必要があります。

ntfsclone -o win_c.img / dev / sda1

そしてそれだけです！ このような機能がイメージファイルを配置するファイルシステムに存在する場合、スパースファイルが自動的に作成されます。



イメージは完全に通常の方法でマウントされます。



mount -o loop win_c.img win_c



主なことは、画像を作成するときにフォーマットを間違えないことです



注意してください！ インターネットでは、 -s引数を使用することが推奨されることがよくあります。これは--save-imageです。 使用すると、ntfsclone固有の形式でイメージが作成され、その結果、標準マウントでこのイメージをマウントできなくなります。



ntfsclone-ntfsでフォーマットされたパーティションのイメージを作成するための不可欠なツール



ddrescueと比較して、ntfscloneには2つの重要な利点があります。



  1. ntfscloneは、実際に使用されるディスクの断片のみをコピーし、無駄な荷物はコピーしません。
  2. --save-imageキーなしでntfscloneによって作成されたイメージは、標準のオペレーティングシステムツールを使用してマウントできます。


一般に、ntfscloneは人生を大いに促進し、状況が許せば使用します。
