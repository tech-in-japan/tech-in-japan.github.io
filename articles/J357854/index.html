VPSのダウンロード方法：Nextcloudクラウド



2017年以外：政府はサイトをスパイおよびブロックし、大企業はあなたの個人データを政府に提供するか、側に売ります。 すでにVPNサブスクリプションを持っていますが、ロシアではそのようなサービスは現在違法です。 VPSを海外でレンタルし、それを介してネットワークにアクセスしましたが、容量がアイドル状態です。



Nextcloudをインストールします！ スマートフォンの連絡先、カレンダー、タスクリスト、ファイルストレージ/エクスチェンジャーの同期が置き換えられ、PCでDropboxになります。 独立性を追加し、サーバーをロードします。



次は、サーバーのレンタル、サーバーおよびPC /スマートフォンへのNextcloudのインストールと構成、連絡先とカレンダーの同期などについての、長くはないが複雑な指示です。



免責事項



1.私は中国に住んでいます。ここでは、ブロッキングの問題はロシアよりも深刻です。 GoogleとDropboxでさえいつかはいつまでも切断されることがあると聞いたことがありますが、さまざまな理由で国内のサービスを使用したくありません。



2.私のサーバーでは、 Shadowsocksはロックをバイパスする手段として機能します。ロシアにとっては今では過剰ですが、中国では役立ちます。 最初にStreisandを入れますが、やり過ぎです。



3.私はプロのシステム管理者ではありません。 重要な情報があるサーバーでは、この命令を軽率に使用しないでください。 バックアップを作成します。



Nextcloudは、個人的な「クラウド」のオープンソースプロジェクトです。Dropboxの機能、連絡先、カレンダー、Googleのタスクのリストなどを組み合わせたものです。 つまり、Googleをオプトアウトして、コンピューターとスマートフォンからサーバーにデータを同期できます。



なぜこれが必要なのですか？



  • あなたが妄想を持っている場合。
  • 大企業を信用していない場合。
  • データを個人の管理下に置きたい場合。
  • あなたの国が機能しない場合、または突然機能しなくなる場合があります。
  • ホスティングに毎月料金を払っても構わない場合、または既に稼働中のサーバーがある場合。
  • もし...はい、あなたは決して知りません。


何が必要ですか？



  1. Ubuntu 17.04上のVPSサーバー



    DigitalOceanにVPSがあります-近くのドロップレットにあり、私のブログは回転しています。 紹介リンク（たとえば、私のブログ、またはインターネット上の他の場所）を探すと、登録のために、テストのために2、3か月に十分なものがスローされます。 しかし、サーバーの選択は完全にあなた次第です。
  2. ドメイン



    必ずしもではないが、非常に望ましい。 .RUゾーンでは、ドメインは1ペニー （年間199ルーブル） かかりますが、他のゾーンではあまり高くありません。 ドメインを登録し、サーバーIPアドレスをバインドします（レコードA）。
  3. SSH用ソフトウェア



    PuTTY、Puttygen、 Windows用ページェント 。 UbuntuまたはmacOSを使用している場合は、すべてを既に知っているか、必要はありません。


どうする？



サーバー上



  1. Apache、PHP、MySQLをインストールする
  2. 無料のLet's Encrypt SSL証明書を追加
  3. NextCloudをインストールして構成する


顧客について



  1. PCおよびAndroid上のクライアント
  2. Gmailから連絡先、カレンダー、タスクを転送する


サーバー



すでにサーバーをレンタルしていて、Putty経由でSSH経由でサーバーに接続できたとします。



開始する



パッケージのリストを更新する 



sudo apt-get update


アパッチ



Apache2 Webサーバーをインストールする 



 sudo apt-get install apache2


構成を編集する 



 sudo nano /etc/apache2/apache2.conf


ファイルの最後に、値を置き換えて2行追加します。 



 ServerName IP-- ServerName -


将来的に役立つheaders modをオンにします 



 sudo a2enmod headers


Letsencrypt



ドメインがある場合は、暗号化された接続の証明書をインストールします。 とても望ましい。



LetsEncrypt Certbotのリポジトリを追加 



 sudo add-apt-repository ppa:certbot/certbot


パッケージのリストを更新する 



 sudo apt-get update


Certbotをインストールする 



 sudo apt-get install python-certbot-apache


証明書を取得します 



 sudo certbot --apache -d -


質問に答えます：メールを提供し、条件に同意し、リダイレクトを選択するときに、オプション「2-リダイレクト」を選択します。



3か月ごとに証明書の自動更新を設定します。 



 sudo crontab -e


nanoを使用して編集を選択し、次の行を追加してファイルを保存します。 



 15 3 * * * /usr/bin/certbot renew --quiet


MySQL



MySQLデータベースをインストールする 



 sudo apt-get install mysql-server


データベースのルートパスワードを設定します。 覚えておくか書き留めてください。 次に、コマンドを実行してセキュリティを構成します 



 mysql_secure_installation


いくつかの質問に答えてください。 パスワードの複雑さを強制的に確認し、ルートパスワードを置き換えることを拒否できます（最初の2つの質問、「N」と答える）、残りの質問「Y」に答えます。



Php



PHP 7と必要なモジュールをインストールします 



 sudo apt-get install php libapache2-mod-php php-mcrypt php-mysql


php-cliを追加 



 sudo apt-get install php-cli


そして、php-apcuをキャッシュするためのモジュール 



 apt-get install php-apcu


構成ファイルを開いて編集します 



 sudo nano /etc/apache2/mods-enabled/dir.conf


拡張子が異なるインデックスファイルのリストで、index.phpが先頭にくるように書き換えます。



Apacheを再起動します 



 sudo systemctl restart apache2


ファイルのディレクトリ



NextCloudがデフォルトで実行する方法ではなく、サーバー上の別の場所に個人ファイル用のフォルダーを作成することをお勧めします。 これはセキュリティを強化するために必要です。



/ media /にディレクトリを作成します 



 sudo mkdir /media/nextcloud/


ディレクトリのアクセス許可を変更する 



 chown -R www-data:www-data /media/nextcloud/


Nextcloud



公式ウェブサイトからNexttarの最新バージョンを* .tar.bz2形式でダウンロードします。 これを行うには、ブラウザでリリースページを開き、新しいバージョンを見つけます。 この記事の執筆時点では、最新バージョンは12.0.2であるため、リンクは次のようになります。



download.nextcloud.com/server/releases/nextcloud-12.0.2.tar.bz2



クリップボードにコピーします。



PuTTYに戻ります。 一時ディレクトリに移動します 



 cd /tmp


NextCloudからアーカイブをダウンロードし、リンクを最新のものに置き換えます。 



 curl -LO https://download.nextcloud.com/server/releases/<span style="color: #ff0000;">nextcloud-12.0.2.tar.bz2</span>


アーカイブを解凍します 



 sudo tar -C /var/www -xvjf /tmp/nextcloud-12.0.2.tar.bz2


インストールするスクリプトを作成する 



 nano /tmp/nextcloud.sh


次のコードをそれに貼り付けます 



 #!/bin/bash ocpath='/var/www/nextcloud' htuser='www-data' htgroup='www-data' rootuser='root' printf "Creating possible missing Directories\n" mkdir -p $ocpath/data mkdir -p $ocpath/assets mkdir -p $ocpath/updater printf "chmod Files and Directories\n" find ${ocpath}/ -type f -print0 | xargs -0 chmod 0640 find ${ocpath}/ -type d -print0 | xargs -0 chmod 0750 chmod 755 ${ocpath} printf "chown Directories\n" chown -R ${rootuser}:${htgroup} ${ocpath}/ chown -R ${htuser}:${htgroup} ${ocpath}/apps/ chown -R ${htuser}:${htgroup} ${ocpath}/assets/ chown -R ${htuser}:${htgroup} ${ocpath}/config/ chown -R ${htuser}:${htgroup} ${ocpath}/data/ chown -R ${htuser}:${htgroup} ${ocpath}/themes/ chown -R ${htuser}:${htgroup} ${ocpath}/updater/ chmod +x ${ocpath}/occ printf "chmod/chown .htaccess\n" if [ -f ${ocpath}/.htaccess ] then chmod 0644 ${ocpath}/.htaccess chown ${rootuser}:${htgroup} ${ocpath}/.htaccess fi if [ -f ${ocpath}/data/.htaccess ] then chmod 0644 ${ocpath}/data/.htaccess chown ${rootuser}:${htgroup} ${ocpath}/data/.htaccess fi


スクリプトを実行する 



 sudo bash /tmp/nextcloud.sh


ApacheでNextCloud構成ファイルを作成する 



 sudo nano /etc/apache2/sites-available/nextcloud.conf


次のテキストを貼り付けます 



 Alias /nextcloud "/var/www/nextcloud/" <Directory /var/www/nextcloud/> Options +FollowSymlinks AllowOverride All <IfModule mod_dav.c> Dav off </IfModule> SetEnv HOME /var/www/nextcloud SetEnv HTTP_HOME /var/www/nextcloud </Directory> <VirtualHost *:443> ServerName <span style="color: #ff0000;">_</span> <IfModule mod_headers.c> Header always set Strict-Transport-Security "max-age=15552000; includeSubDomains"</IfModule> </VirtualHost>


構成ファイルをアクティブにします 



 sudo a2ensite nextcloud


詳細オプションを有効にする 



 sudo a2enmod rewrite sudo apt-get install php-bz2 php-curl php-gd php-imagick php-intl php-mbstring php-xml php-zip


Apacheを再起動します 



 sudo systemctl reload apache2


MySQLデータベースを作成する



MySQLに切り替えましょう。入力すると、前に作成したMySQLルートパスワードが必要になります。 



 mysql -u root -p


nextcloudという新しいデータベースを作成します 



 CREATE DATABASE nextcloud;


彼女のパラメータを設定しましょう。新しいパスワードを忘れずに挿入してください 



 GRANT ALL ON nextcloud.* to 'nextcloud'@'localhost' IDENTIFIED BY '____';


パラメーターをリセットしてMySQLを終了します 



 FLUSH PRIVILEGES exit




コンソールにいる間、トラブルのない更新のためにさらにいくつかのことを行います。



Nextcloudでディレクトリに必要な権限を設定します 

 chown -R www-data:www-data /var/www/nextcloud/




ディレクトリ/ var / www / nextcloud / assetを削除します



NextCloudを構成する



ブラウザーで、 ip_or_domain / nextcloudを開きます



すべてが正しく完了すると、NextCloudページと管理者アカウントを作成するための招待が表示されます。



最初の2つのフィールドは、 管理者アカウントのユーザー名とパスワードです



データフォルダー ：「ファイルのディレクトリ」手順で作成したディレクトリを指定します 。例では、/ media / nextcloudです。



データベースを構成する



「MySQLデータベースの作成」のステップで使用したパラメーターを指定する必要があります



最初の行はデータベースの名前です



2行目はデータベースのパスワードです（rootパスワードではありません！）



3行目-ユーザー



4行目は、データベースにアクセスするためのアドレスです。localhostおよびポート5342である必要があります

共有する



クラウドのみを使用しない場合は、ファイル配布の設定を設定します。 私たちにはプライベートな小さなサーバーがあり、数千の同時ダウンロードが深刻な負荷をかける可能性があります。 各パブリックリンクにダウンロードするためのパスワードを強制的に設定し、数日後にそれを非アクティブにする必要があります。







テーマ



ここでは、Nextcloudに個性を与えることができます。名前、署名、背景の画像を設定し、さらにデザインの色を変更することもできます。 それは思いのままに行われます。







暗号化



オンにしないでください 。 サーバーとディスクストレージの負荷は大幅に増加しますが、セキュリティに大きな影響はありません。暗号化キーは同じサーバーに保存され、攻撃者がアクセスすると、暗号化されたファイルを読み取ることさえできます。

Gmailから連絡先を転送する



これはコンピューターを介して実行できますが、連絡先のポートレートは保存されないため、スマートフォンですべてを実行します。 スマートフォンにGoogleの連絡先がない場合は、下にスクロールします。



必要になります



  1. Nextcloudのアカウント
  2. Androidスマートフォン
  3. 連絡先とタスクを同期するためのDAVdroidおよびOpenTaskアプリケーション
  4. ZenUI Dialerアプリケーション（または連絡先をコピーするためのその他のアプリケーション）


ステップバイステップ



DAVdroidをインストールする



  1. DAVdroid と OpenTask をインストールします 。



    FAV-Droidは無料なので、DAVdroidをインストールすることをお勧めします。 そしてGoogle Playでは、彼らは249ルーブルを要求します。
  2. DAVdroidを起動し、アカウントを作成します



    「+ 」 ボタンを押して 、「URLとユーザー名でログイン」を選択します

    フォームにNextcloud アドレスを入力してください 



     https://ip--.ru/nextcloud/remote.php/dav/


    また、 ユーザー名とパスワード 。



  3. アカウントの名前を作成します（例：user @ nextcloud）
  4. [ アカウントを作成]をクリックします
  5. DAVdroid は 、連絡先とカレンダーにアクセスするためのいくつかの許可を要求し 、それらを許可します。 （要求していない場合は、上からカーテンを下げて、要求があるかどうかを確認します）



  6. 上部の[ 連絡先]の横にあるチェックボックスをオンにし、下部にGoogleカレンダーの名前を表示します（ 連絡先の誕生日と個人情報があります ）。
  7. 上部の円形矢印アイコンをクリックして、最初の同期を開始します。

    （最初にエラーメッセージが表示される場合があります。もう一度やり直してください）


GoogleからDAVdroidに連絡先を転送する



  1. ZenUI Dialer をインストールする
  2. 起動して連絡先管理に移動します（「3ポイント」ボタン-> 連絡先の管理 ）
  3. 連絡先のコピーを選択
  4. 連絡先をコピーするには、Googleアカウントを選択します
  5. 連絡先をコピーして DAVdroidアドレス帳 を選択する場合
  6. 質問に対して、元の回答をOKの ままにしておき ます 。 そうしないと、連絡先がGoogleから削除されます 。
  7. コピーが完了するのを待ちます。
  8. DAVdroidを開き、連絡先を再度同期します。


同期が終了すると、すべての連絡先がWebインターフェースで利用可能になります：https：//your_domain/nextcloud/index.php/apps/contacts/







スマートフォンに連絡先がない場合



何らかの理由で携帯電話にGoogleの連絡先がなくても、PCのブラウザーで開くことができる場合、GoogleからDAVdroidに連絡先を転送する手順をスキップし、Googleの連絡先から連絡先をエクスポート（詳細->エクスポート）して、次に、Nextcloud Webインターフェースからファイルをインポートします。



カレンダー



スマートフォンの標準カレンダーアプリケーションに移動し、 カレンダーのリストに新しいNextcloudカレンダーが表示されていることを確認します。 イベントをそこに保存できるようになりました。イベントは独自のNextcloudサーバーと自動的に同期されます。 Googleはもう必要ありません。







Webバージョンのカレンダーは、ブラウザのyour_domain / nextcloud / index.php / apps / calendarから利用できます







タスクリスト



OpenTasksアプリケーション（メニューは単にTasksと呼ばれることもあります ）を実行すると、ローカルリスト（デバイス上のみ）と「クラウド」リストがあります。







タスクリストの別のクライアントはSimpleTask Nextcloudです。 機能は少なくなりますが、少し信頼性が高くなります。



ブラウザでは、タスクはyour_domain / nextcloud / index.php / apps / tasksで利用できます

Nextcloud言語を変更する方法（Windows）



何らかの理由で、開発者はアプリケーション設定を介して言語を変更することを提供していませんでした。デフォルトでは、システム設定を使用します。 私は中国語でWindowsを使用しています（これについては何もできません）が、アプリケーションをより使い慣れた言語で保持する方が便利です。 問題の解決策は次のとおりです。



  1. Nextcloudクライアントをオフにする
  2. Nextcloudをインストールしたフォルダーを開きます（デフォルトではc：\ Program Files（x86）\ Nextcloud \）
