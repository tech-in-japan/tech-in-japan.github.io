新しいITインフラストラクチャ管理とIBM Flex System Managerの機能

こんにちは！

システムのIBM Pure Systemsファミリーの説明を続けたいのですが、システムの一般的な説明を含む最初の記事はここにあります 。 このシステムを構成するコンポーネントをさらに詳しく分析してみましょう。 最も興味深い部分である制御システムから始めたいと思います。 モジュールは、監視、管理、さまざまな追加機能を担当します。



このソリューションの作成につながったニーズ：



•管理の困難につながるインフラストラクチャの増加。 サーバーとストレージシステムの数は毎日増加しています。 これらすべての「動物園」の管理は日々難しくなっています

•インフラストラクチャの柔軟性。 インフラストラクチャの増加は、使用されるリソースの量が常に明確であるとは限らず、無料のリソースを使用することが難しいという事実につながります。

•セキュリティの問題。 アクセスパスワードが異なるさまざまな機器用の多数の制御コンソール。 単一の会計ベースと単一のアクセス権の配布の欠如。

•リスクの低減。 ほとんどのプロジェクトは、人的要因（コンピテンシーの欠如、リソースの不足、エラーなど）により予定どおりに展開されません

•マルチプラットフォームソリューションの管理の課題。 x86プラットフォームだけでなく、RISCマシンもインフラストラクチャに存在するため、システム全体の管理が困難になります。

•プラットフォームをスケーリングする際の使いやすさを維持します。 システムスケーリングの現在の段階と次の段階の両方で、単一のシンプルな形式の制御を保証するソリューション。

•ほとんどの日常的な日常業務を自動化します。 巨大なインフラストラクチャ（100台以上の物理サーバー）の状況では、プロセスの自動化により多くの作業時間が節約されます。



さまざまなメーカーの制御システムが多数あり、無料の場合もあります。 なぜ新しいものを思い付くのですか？

IBM Pure Flexシステムは、ハードウェア（ストレージシステム、サーバー、スイッチ）だけでなく、ソフトウェア要素（ハイパーバイザーと仮想マシン管理）だけでなく、制御および監視システムの新しい要件を提唱する、すべてのコンポーネントの深い統合を備えた最初のシステムです。



IBM Pure Flexシステムはどのように問題を解決しますか？ IBM Flex System Manager管理モジュールがこれを担当します。

IBM Flex System Manager管理モジュールは物理ノードであり、シャーシ内の1つの設置場所を占有し、同時に最大16台のシャーシまたは448サーバーを管理できます。







IBM Flex System Managerの主な機能の簡単なリスト：



•追加情報（ファン、インストールされているサーバー、スイッチ、電源）を追加したシャーシのハードウェアコンポーネントのリアルタイム監視

•自動障害検出（多数のパラメーター（ディスク、プロセッサー、プロセッサー電源モジュールなど）の障害予測システム）および通知メカニズム（電子メール、SMS）およびトラブルシューティングアクションの設定（たとえば、システムはサービス内の要求を自動的に開くことができます）中央）

•システムの可用性と可用性に影響する変更の識別。 何らかのアクションが発生した場合のシステムの操作性とフォールトトレランスの決定。

•仮想マシンまたはその他のコンポーネントセットによるリソース使用レベルを表示します。 システム利用効率の総合評価

•物理サーバーと仮想サーバー、およびそれらのスイッチングパスの自動検出により、ネットワークインフラストラクチャを透過的に管理できます。

•ハードウェアコンポーネントの説明。 既存の機器のインベントリにより、将来の購入を予測し、リソースの使用を追跡することが容易になります。

•ハードウェアプロパティの説明

•機器のファームウェアバージョンの関連性。 すべての機器のファームウェアの決定の容易さとグループの更新。

•さまざまなメーカーのネットワークスイッチの管理。 つまり データセンターネットワークインフラストラクチャ全体の統合管理

•さまざまなKVM、IBM PowerVM、VMware、Microsoft、およびさまざまな仮想スイッチハイパーバイザーのサポート

•コンピューティングリソース管理環境（vCenterなど）との統合

•物理および仮想ストレージデバイスとのコラボレーション

•リソースプールの構成

•仮想サーバーを作成、編集、管理、および再構成する機能



オプションの機能セット：



•仮想デバイスを保存するための新しいリポジトリの作成

•外部の新しい仮想デバイスを既存のリポジトリにインポートする

•ワークロードを管理グループに統合するシステムリソースプールの作成



それはどのように見えますか：







IBM Pure Systemsファミリーのシステムには、専門知識が組み込まれています。 この検査は、事前に構成および構成されたテンプレートのシステムの使用に基づいています。 テンプレートには、コンピューティングリソース、ネットワークインフラストラクチャ、およびストレージ用のさまざまな種類があります。

これらすべてのタイプのテンプレートは、変更への反応速度の向上、新しいリソースの割り当て時の管理コストの削減、割り当てられたリソースの利点の加速、および新しいシステムの作成における既存の経験と知識の再利用を目的としています。

テンプレートは、IBMの専門家、独立したソフトウェア開発組織（SAP、Siemens、Diasoftなど）の専門家、および顧客組織内の専門家によって作成できます。







明らかに、多くの顧客は、他の組織によって複製または使用されていない独自のアプリケーションを持っています。 一方、企業内では、常に使用する必要があるため、顧客自身がテンプレートを開発できる場合にこのようなオプションが提供されていました。







テンプレートがオープンリポジトリにあり、将来使用できることが重要です。



IBM Flex System Managerの主な利点は、IBM Pure Flexシステム用の単一のコントロールセンターを作成し、内部および外部の物理リソースと仮想リソースの両方を（インポートメカニズムを介して）監視および管理することです。
