著作権デッドロック？

少なくとも彼の潜在意識の端で、彼の新しい独創的な創造物を夜に書いたか、単に彼の新しい投稿をHabréに展開した人は誰でも、著作権または著作権のアイデアです。 しかし、著作権がネットワークの発展の道に大きな重みを与えているという意見があります。 そのような概念を聞いたことがない人のために、著作権または異なる国の小さなバリエーションの著作権には、著者の独占的権利が含まれることを思い出させてください：
  • あらゆる種類のメディアでの著者の作品の複製と配布。
  • 翻訳や修正などの二次的著作物の作成。
  • 著者の作品の公演、展示、またはコミュニケーション。
これには基本的に著作権があります（何かにひどく間違えた場合、弁護士が私を修正します）。 これらの規則を詳しく調べると、財産権を普通のものに著作権作品に移そうとした結果として生じたことがわかります。 著作権作品が主にアイデアであるという事実を考慮に入れていませんが、それを控えめに言っても、彼らのキャリアに添付するのは無意味です。 その結果、すべての著作権法は、それら（これらの法律）を現実と調和させるように設計されたさまざまなカードにあふれています。 それは不条理のポイントになります：たとえば、ロシアの法律（他の方法ではわかりません）では、すべての注釈付きリンクを禁止するという一般的な知識を持ち込む権利がありますが、ここでは、このタイプの法律の実際的な利点に疑問を投げかけるいくつかの例外が与えられています。 この不条理な状況は、アイデアの仮想化の時代の到来、つまりメディアの必要性がなくなり、作品の修正と複製の事実を記録することがますます困難になるにつれて、さらに悪化しています。 したがって、明らかに既存の著作権を修正する必要があります。

プロ

著者としての私たちは著作権から何を望んでいますか？まず、もちろん、私たちは社会に創造者としての偉大さを認めてほしいのです。 それは、青銅でさえ、金ではなく、全ロシア展示センターではなく、ボブルイスクの近くにあるが、常に小さな記念博物館である、生体内の記念碑にしよう。 Web 2.0の現在の開発の根底にあるのは、この動機です。 どのくらい具体的に？ 著者によって異なります：ビールの箱からバハマ諸島まで。 この一般的に健全な目標を達成する際に、著者はさまざまな成功を収めながら解決しようとしている多くの問題にすぐに直面することに注意する必要があります。
  • 彼の独創的な仕事を広める問題。 さまざまな種類の将来の料金の大部分を、メーカー、流通ネットワークなどに解約する必要があります。
  • 作品からすべてのcimesを最大限に抽出する問題。 私たちは広告に費やし、作品を部分的に分配し、その周りの興奮をかき立て、各段階からすべてのジュースを絞り出すためのスキームを考え出す必要があります。 そのため、たとえば、ビデオ製品のメーカーが来ます。
  • 私たちの仕事を私たちの物質的な商品の流れを減らす試みから保護する問題。 コピー、複製、そしてもちろん弁護士に対するあらゆる種類の保護に費やしています。

コントラ

Webスペースでのアイデアの仮想化がこれらの問題のほとんどを無効にすることを確認するのは難しくありません。
  • Web 2.0テクノロジーの開発により、著者の市場への参入障壁は無視できるほど小さくなりました。ランキングテクノロジー、評価、および心理タイプと社会的嗜好によるユーザー分類の開発により、著作権のある作品を促進するためのこのモデルの有効性が高まります。 同時に、さまざまな仲介者の余地がほとんどなく、広告やPRのために、率直に言えば弱い仕事を大衆に押し付ける能力が大幅に低下しています。
  • 近い将来、コピーと配布に対する効果的な保護を作成する方法はありません。また、ピアツーピアテクノロジーと暗号化の開発は、これらのプロセスを合法的に抑制する試みを無効にします。 したがって、著者と彼のプロモーターは、Webで利益を絞るプロセスを管理する能力を失います。
これらの矛盾を解決する唯一の方法は、著作者と著作権製品の消費者の目標の間で妥協点を見つけることです。 ユーザーが著者に何を望むのか？：
  • 作成者は、自分のアイデア（作業）ができるだけ迅速かつ完全にWeb上に配信されるようにする必要があります。 著者がこれを行わず、アイデアが要求されている場合、他の人々がそれを広める努力は完全に正当化されます。
  • 著者は、すべての利害関係者に二次的著作物（翻訳など）を提供するか、この義務を直ちに他の人々に委任する必要があります。
  • 著者は、ユーザーとして、エラーを排除し、新しい機能、アイデアなどを導入するために作品を修正する機会を提供する必要があります。
特徴的に、これらの論文は、オープンソース、ピアツーピアネットワークおよびWikiにすでに実装されていますが、著者の物質的なニーズを完全に無視しています。

クラウスラ

明らかに、著者と消費者のニーズの間のこれらの矛盾は、Web 1.0でもWeb 2.0でも解決されません。 したがって、それらのソリューションは、次世代ネットワークモデルに基づいて実装する必要があります。 それらをWeb 3.0と呼びます。 これらの矛盾はWeb 3.0でどのように解決する必要がありますか？
  • Web 3.0により、著作者はロイヤリティを確実に受け取り、ユーザーフレンドリーなフォーマットで最小限のコストで作品をできるだけ早く配信できるようになります。
  • 著者は、一般的な情報に彼のアイデアのすべての機能（配布メカニズムと評価の保証）をもたらす権利を完全に譲渡します。
  • 著者がアイデアの一種の独占であることを考えると、Web 3.0はロイヤリティの最大レートを制限するメカニズムを提供する必要があります。
  • Web 3.0には、変更された作品を隔離し、オリジナルの作者にその使用からの報酬の一部を支払うメカニズムが必要です。
これらの要件は素晴らしいように見えますが、Web 3.0ネットワーク自体のフレームワーク内でのみですが、現代の技術レベルで実装される可能性があります。 Web 3.0デバイスのいくつかの機能は、それらからも見えてきます。
  • 分散ピアツーピアネットワークである必要があります。
  • 報酬を分配し、評価を維持するためのメカニズムは、オープンソースのロボットによって提供されるべきです。
  • 修正された作品を分離するメカニズムは、ロボットによって最も単純な場合に提供されるべきであり、より複雑な作品は、ネットワークユーザーの代表委員会の意見によって決定されるべきです（あなたは民主主義を望みました-私はそれらを持っています）。


いずれにせよ、著作権に関して提起された質問を解決することなくネットワークをさらに一貫して開発することは不可能であり、相互に適応するソリューションを最初に見つけた人は、プロジェクトの開発において決定的な利点を享受します。
