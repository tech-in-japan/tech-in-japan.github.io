FS1-Oracleの大きなスマートフラッシュドライブ

2001年7月、Larry Ellisonが所有するTaco Ventures投資会社は、中規模および大規模企業向けのネットワークストレージシステムのメーカーであるPillar Data Systemsの設立に投資しました。 同社は、Axiom製品ラインで市場に定着し、EMC Corporation、Hitachi Data Systems、およびネットワークストレージおよびストレージネットワーク機器のメーカーと直接競合しました。







市場での10年後、オラクルはPillarを買収し、知的財産を自社の産業機器に移転しました。 その結果、多目的ネットワークフラッシュストレージであるOracle FS1ストレージシステム（図1）ができました。 そして、これは別の記事に値するほどクールなことです。



システムの現在のバージョンはFS1-2（1つのクラスター、2つのコントローラー）です。 一般に、FS1システムは1〜8個のクラスター、つまり2〜16個のコントローラーで構成できます。 コントローラーは、システムにインストールされているドライブの数に応じて、その数を増やすという観点から水平方向と垂直方向の両方でスケーリングされます。



現在のバージョンFS1-2のコントローラーには、基本と高性能の2つのバージョンがあります。 両方のバージョンの各コントローラーペアは、4つのIntel Xeon E5-2620プロセッサー（24コア、2.0 GHzのクロック周波数）に基づいて構築され、4〜12個のポート（FC 16 Gb / s、NAS 10 Gb / sおよびiSCSI 10）を持つことができます。 Gbps HBAアダプターペアは、ユーザーの選択により、任意の組み合わせでインストールできます。 基本コントローラーには、4つの標準6 Gb / s SAS-2ポートがあり、オプションで8つあります。 高性能-12.データキャッシングに使用されるメモリに関しては、コントローラーの基本バージョンでは最大80 GB（64 GB DDR3 DRAMおよび16 GB DDR3 NVDIMM）、高性能-416 GB（それぞれ384 GBおよび32 GB）が可能です。



このような容量の組み合わせにより、システムは16コントローラーの高性能構成で最大200万IOPSのパフォーマンスと最大80 GB / sのデータ転送速度、および個別のライセンスを必要としないデータを操作するためのフルセットの機能、およびジャーナリングサポートによる同期および非同期レプリケーションがサポートされます時点レベルとアプリケーションサポートの両方でのCDP。



この素晴らしいハードウェアはすべて、ラック（すべてのコンポーネントの後）でなく、ラック（この場合、コンポーネントはラックに設置され、ケーブルで接続され、完全装備のすぐに使えるシステムの形でテストおよび出荷されます）ケーブルで接続され、ラックでテストされた後、ラックから取り外され、お客様自身のラックに設置するために個別のパッケージで配送されます。



I / Oのコストを最適化するために、オラクルが開発したQoS Plusメカニズムが使用されます。これは、ビジネスの優先順位を考慮してI / O操作のキューを管理し、論理ボリューム（LUN）レベルへの自動分割を使用する仮想化機能を備えたポリシーベースのシステムソフトウェアです単純なパターンに基づいています。 QoS Plusは、ストレージシステムの使用に関する詳細情報を収集し、特定のストレージレベルに転送する必要があるデータ要素を評価し、1つの入出力操作のコストを最適化するために、最も効率的なメディア（フラッシュドライブまたはハードディスク）にデータを自動的に転送します使用プロファイルと、ビジネスにおけるこのデータの重要性に基づきます。 今日、それは市場で最も効率的なシステムです。

FS1はインテリジェントなデータストレージシステムであり、さまざまなアプリケーションとその部品に最適な特別なプロファイルを使用します。 Oracle Databaseおよび主要なエンタープライズアプリケーション（Microsoftなどの他のメーカーのアプリケーションを含む）の一部のプロファイルは、事前に構成されています-約30種類のプロファイルのみです。 アプリケーションが最初に指向するドライブのレベルを選択し、サービスのレベルを決定できます。 システムにより、インデックスファイル、データベーステーブル、アーカイブログ、トランザクションログ、監査ファイル、一時ファイルなどのデータベースコンポーネントを分離できるため、リソースを割り当てるときに、データベースコンポーネントの詳細な分析なしでOracle Databaseのパフォーマンスが自動的に最適化されます。 ユーザーは、新しいプロファイルを作成し、既存のプロファイルを変更し、すべてのプロファイルを他のシステムにエクスポートして、データセンターによるリソースの割り当てを標準化できます。 システムでは、異なるプロファイルを優先度ごとにグループ化することができます。これにより、異なるシステム間のI / Oを区切ることができます。



FS1システムのもう1つの強力な機能は、ストレージドメイン、または同じ物理FS1システム内の仮想ストレージシステムです。 各ストレージドメインは、他のストレージドメインからデータを分離する「データコンテナ」です。 これにより、プライベートクラウドまたはパブリッククラウドの共有環境での独立性、規制への準拠、またはチャージバックモデルの実装が保証されます。 ストレージドメインは、任意の組み合わせのSSD \ HDDドライブのRAIDグループで構成されます。 データストレージドメインは、データを物理的に分離し、ストレージリソースを他のデータと共有する必要性を排除します。 これにより、頻繁に発生する「ノイズの多いネイバー」問題が解消されます。 FS1-2システムには最大64個のストレージドメインを割り当てることができます。 RAIDグループとディスククラスは簡単に再構成できるため、ストレージドメインのパフォーマンスを手動で最適化するか、目的のストレージプロファイルを各LUNに割り当ててこのタスクをシステム自体に提供することで最適化できます。 ストレージプロファイルの割り当ては、マウスを1回クリックするだけで行われるため、管理者は多大な時間コストと頻繁な日常業務から節約できます。



ストレージドメインを使用すると、異なる環境用にQoS Plusソフトウェアを個別に構成できますが、すべてのストレージドメインは引き続き同じ物理FS1システムに配置されます。 重要なアプリケーションに特定のドライブリソースを割り当てて、パフォーマンスを保護できます。 アーカイブの場合、1 TBあたりの単価が低い個々のドライブ、異なるタイプのドライブ、さらには異なるストレージドメインを選択できます。



ドライブに関しては、FS1ディスクシェルフに収まるソリッドステートデバイスとハードドライブの数とパラメーターに関する完全な情報を提供します。 簡単に説明します。400GBの容量と1.6 TBの容量の高性能SSD、300 GBと900 GBの容量の高速ディスク（スピンドル速度-10,000 rpm）のさまざまな組み合わせ、および4 TBの容量のハードドライブ（サポートされています） 7200 rpm。）。



その結果、システムは912 TBのフラッシュドライブと最大2.9 PBのディスク容量まで拡張できます。 容量と速度の観点から、FS1ドライブは、高性能フラッシュSSD、大容量フラッシュSSD、高性能ハードドライブ、および大容量ハードドライブに分かれています。 この分類は、特定のデータのストレージレベルを選択するときにQoS Plusによって使用されます。 したがって、LUNには最大4つの物理レベルが許可されます。 さらに、パフォーマンス要件に応じてRAIDレベルを変更するオンラインメカニズムを使用すると、各RAIDグループはRAID5 \ RAID10またはRAID6 \ RAID10の2つの状態になり、単一のボリュームを配置できる論理レベルは最大8になります。 OracleがFS1システム用に選択したデータ形式は640 KBのデータブロックを使用しますが、これは競合他社の数百倍です。たとえば、42 MBブロックや他のいくつかのメーカーのアレイのギガバイトブロックよりも小さいです。 小さなブロックは、データの転送と保存にはるかに効率的です。 640 Kのブロックサイズは、4 Kブロックを使用する他の多くのメーカーが提案するアプローチよりもはるかに効果的です（ディスクページサイズが64 Kで、記録用のフラッシュページサイズが128 Kであることを思い出してください）。
