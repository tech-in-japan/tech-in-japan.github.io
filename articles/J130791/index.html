やった！ Androidソースが再び利用可能に

数分前、android-building googleグループで、kernel.orgから新しいサーバーに移動するリポジトリに関するメッセージが表示されました。 Android 4.0のソースは、少し後で投稿することを約束します。 



こんにちは！

ご存知のように、他の多くのプロジェクトと同様に、Androidオープンソースプロジェクトは、最近のkernel.orgのダウンタイムの影響を受けました。 そのため、Gingerbreadのソースコードが再び利用可能になり、AOSP gitサーバーがオンラインに戻ったことをお知らせいたします。

 kernel.orgのダウンタイムが発生する前から、AOSPがkernel.orgのgitインフラストラクチャに負担をかけていることは明らかでした。 たとえば、Gingerbreadのソースリリースを行ったとき、AOSPによるロードにより、kernel.orgの一部が数日間使用できなくなりました。 これはkernel.orgのスタッフやコミュニティにとって公平ではないため、しばらくの間、Googleサーバーで独自のgitホスティングを準備してきました。

 kernel.orgがダウンタイムを経験したのと同じように仕上げていたため、GingerbreadソースがGoogleのサーバーで利用可能になりました。 したがって、git URLは変更されました。

新しいgitサーバーにアクセスする手順は次のとおりです。
リポジトリツールの最新バージョンを入手する必要があります。
 curl https://dl-ssl.google.com/dl/googlesource/git-repo/repo>〜/ bin / repo
新しいリポジトリを初期化する必要があります。
 repo init -u https://android.googlesource.com/platform/manifest -b android-2.3.7_r1
完全な手順はhttp://source.android.com/source/downloading.htmlにあります

注意すべき制限がいくつかあります。
私たちの優先事項は、メインのソースコードミラーをオンラインに戻すことです。そのため、現時点ではgitwebソースブラウジングとGerrit Code Reviewはまだ利用できません。
現在、AOSPのGerrit Code Reviewサイトを復活させることに取り組んでおり、近いうちに何かを言うことができることを望んでいます。
残念ながら、Gerrit Code Reviewが次の優先事項であるため、gitwebが戻ってくるまでに少し時間がかかる場合があります。
繰り返しますが、これらのサーバーには、古いAOSPサーバーの「gingerbread」および「master」ブランチのみが含まれます。 最近発表されたIce Cream Sandwichのソースは、デバイスで利用可能になり次第、リリースする予定です。
これらの新しいサーバーは非常に新しいため、予期しない問題が発生した場合は問題が発生する可能性があります。 しかし、私たちは彼らに注意を払っており、あらゆる問題に可能な限り迅速に対応します。

最後に、kernel.orgコミュニティとオレゴン州立大学のオープンソースラボのスタッフに、「ありがとう」という大きなメッセージを送りたいと思います。 彼らはAOSPソースコードミラーとGerrit Code Reviewをホストする素晴らしい仕事を3年近く行ってきました。 それらがなければ、AOSPが今日の私たちのいる場所ではないと言うのは安全です。

ありがとう、そして幸せなコーディング！

 -ダン
