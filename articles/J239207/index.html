Swiftでジェネリックを使用する

Swiftでのプログラミング方法を既に理解している場合は、おそらくSwift言語の基本と、クラスと構造の作成方法をすでに知っているでしょう。 しかし、Swiftはそれ以上の機能を備えています。 この記事の目的は、 ジェネリックと呼ばれる他のいくつかの言語ですでに普及しているSwiftの非常に強力な側面について話すことです。



型に関して安全なプログラミング言語を使用すると、これは、1つの型のみで動作するコードを記述するような一般的な問題ですが、別の型では非常に正確です。 たとえば、関数が2つの整数を追加するとします。 2つの浮動小数点数を追加する関数は非常によく似ていますが、実際には同じように見えます。



唯一の違いは、変数の値タイプです。



強く型付けされた言語では、個々の関数をaddInts、addFloats、addDoublesなどとして定義する必要があります。各関数には正しい引数があり、型を返します。



多くのプログラミング言語は、この問題の解決策を実装しています。 たとえば、C ++はテンプレートを使用します。 Swift、JavaとC＃の両方がジェネリックを使用している-したがって、このチュートリアルのトピックです！



一般化されたSwiftプログラミングに関するこの記事では、既に見たものを含め、プログラミング言語の既存のジェネリックの世界に飛び込みます。 次に、ユーザーの検索条件を追跡するカスタムユニバーサルデータ構造を使用して、Flickrで写真検索プログラムを作成します。



注： このSwift関数型プログラミングの記事は、Swiftの基本をすでに知っている人向けです。 Swiftの基本に慣れていない場合は、最初に他のSwiftチュートリアルをチェックすることをお勧めします。



ジェネリック医薬品の紹介



あなたはこれを知らないかもしれませんが、おそらくジェネリックがSwiftで動作しているのを見たことがあるでしょう。 配列と辞書は、ビジネスにおける一般的なセキュリティの典型的な例です。



Objective-C開発者は、同じコレクション内の異なるタイプのオブジェクトを含む配列と辞書に慣れています。 これにより柔軟性が向上しますが、APIから返される配列がストレージ用であることをご存知ですか？ 別の形式のドキュメントであるドキュメントまたは変数名を見ると確認できます。 ドキュメントの場合でも、実行時にコレクションのエラーを防ぐ方法はありません（エラーのないコード以外にはありません！）。



一方、Swiftには配列と辞書があります。 Ints配列にはIntsのみを含めることができ、たとえばStringを含めることはできません。 これは、コードを記述することでコードを登録できることを意味します。これにより、コンパイラは型チェックを実行できます。



たとえば、Objective-C UIKitでは、ユーザービューに基づいてタッチを処理するメソッドは次のようになります。 



- (void)touchesBegan:(NSSet *)touches withEvent:(UIEvent *)event;




ご存知のように、このメソッドのセットにはUITouchインスタンスのみが含まれていますが、それはドキュメントに記載されているためです。 しかし、オブジェクトが他の何かになることを妨げるものは何もありません。通常、セット内のタッチをUITouchインスタンスとして把握して、それらをUITouchオブジェクトとして効果的に扱う必要があります。



現時点では、Swiftには標準ライブラリで定義されているセットがありません。 ただし、セットではなく配列を使用した場合は、上記のメソッドを次のように記述できます。 



 func touchesBegan(touches: [UITouch]!, withEvent event: UIEvent!)




これは、touchs配列にUITouchインスタンスのみが含まれていることを示し、このメソッドを使用するコードが他のものを渡そうとすると、コンパイラーはエラーをスローします。 型がコンパイラによって設定/定義されるだけでなく、UITouchインスタンスの要素を計算する必要がなくなります！



一般に、ジェネリックはクラスのパラメーターとして型を提供します。 すべての配列は同じように機能し、値をリストに保存しますが、一般的な配列は値のタイプをパラメーター化します。 配列で使用するアルゴリズムは非固有ではないため、すべてのタイプの値を持つすべての配列でそれらを分離できます。



ジェネリックとその利点に関する基本的な情報が得られたので、それらを特定のシナリオに安全に適用できます。

順序付き辞書



アプリケーションはユーザーリクエストごとに画像をアップロードし、最新の検索が画面の上部にリストとして表示されます。



しかし、ユーザーが同じアイテムを2回検索するとどうなりますか？ アプリケーションが古い結果を新しいリストの一番上に移動し、新しい結果に置き換えればいいと思います。



データ構造に配列を使用して結果を返すことができますが、ジェネリックを調べるには、新しいコレクション、つまり順序​​付けられた辞書を作成する必要があります。



多くのプログラミング言語とフレームワーク（Swiftを含む）では、配列とは異なり、セットと辞書は順序を保証しません。 順序付けられた辞書は通常の辞書のように見えますが、特定の順序でキーが含まれています。 この機能を使用して検索結果を保存します。これにより、結果をすばやく検索したり、表の順序を維持したりできます。



慎重でない場合は、ユーザーデータ構造を作成して、順序付けられた辞書を処理できます。 しかし、あなたは先見の明があります！ 今後数年間、アプリケーションで使用できるものを作成したいと考えています。 ジェネリックは理想的なオプションです。



一次データ構造



[ファイル]、[新規]、[ファイル]、[iOS]、[ソース]、[スウィフトファイル]の順に選択して、新しいファイルを追加します。 [次へ]をクリックし、ファイルにOrderedDictionaryという名前を付けます。 最後に、作成をクリックします。



その結果、空のSwiftファイルが作成され、次のコードを追加する必要があります。 



 struct OrderedDictionary { }




これまでのところ、これは驚くことではありません。 オブジェクトは値のセマンティクスを持たなければならないため、構造体になります。



注 ： 要するに、意味のセマンティクスは、「パブリックリンク」ではなく「コピー/貼り付け」と言う珍しい方法です。 値のセマンティクスには多くの利点があります。たとえば、予期せずデータを変更する可能性のあるコードの別の部分を心配する必要はありません。 詳細については、第3章のレッスン「 クラスと構造」で Swiftを理解する方法を参照してください。



ここで、ジェネリックにする必要があるため、必要なタイプの値を含めることができます。 構造定義を次のように変更します。 



 struct OrderedDictionary<KeyType, ValueType>




山カッコ内の要素は、ジェネリック型パラメーターです。 KeyTypeとValueTypeはそれ自体は型ではなく、構造定義内の型の代わりに使用できるパラメーターになります。 理解できなければ、近い将来にすべてが明らかになるでしょう。



順序付き辞書を実装する最も簡単な方法は、配列と辞書の両方を維持することです。 辞書には、データ変換と配列キーが格納されます。



構造定義に次のコードを追加します。 



 typealias ArrayType = [KeyType] typealias DictionaryType = [KeyType: ValueType] var array = ArrayType() var dictionary = DictionaryType()




これは、説明した2つのプロパティと、既存のタイプに新しい名前を付ける2つのタイプエイリアスを示します。 ここでは、バックアップアレイとディクショナリのそれぞれに、アレイのエイリアスとディクショナリのタイプを指定します。 タイプエイリアスは、複雑なタイプを取得し、はるかに短い名前を付けるのに最適な方法です。



型の代わりに、構造定義のKeyTypeおよびValueType型パラメーターを使用できることに注意してください。 配列はKeyTypesの配列です。 もちろん、KeyTypeなどのタイプはありません。 代わりに、Swiftは、ジェネリック型のインスタンス化中に、あらゆるタイプの順序付き辞書ユーザーのようにそれを扱います。



この時点で、コンパイラエラーに気付くでしょう。



タイプ「Keytype」はプロトコル「Hashable」に準拠していません



これはあなたにとって驚きかもしれません。 辞書の実装を見てください。 



 struct Dictionary<KeyType: Hashable, ValueType>




これは、KeyTypeの後の「：Hashable」を除いて、OrderedDictionaryの定義に非常に似ています。 セミコロンの後のHashableは、KeyTypeに渡される型がHashableプロトコルに準拠する必要があることを示します。 これは、ディクショナリがその実装のためにキーをハッシュできる必要があるためです。



この方法でジェネリック型パラメーターを制限することは非常に一般的になりました。 たとえば、アプリケーションがこれらの値をどう処理する必要があるかに応じて、EquatableまたはPrintableプロトコルに適合するように値のタイプを制限することができます。



OrderedDictionary.swiftを開き、構造定義を次のものに置き換えます。 



 struct OrderedDictionary<KeyType: Hashable, ValueType>




これは、OrderedDictionaryのKeyTypeがHashableと一致する必要があることを示しています。 これは、KeyTypeがどのタイプになっても、メイン辞書のキーとして受け入れられることを意味します。



これで、ファイルはエラーなしでコンパイルされます！



キー、意味、ジャズ



値を追加できない場合の辞書の使用は何ですか？ OrderedDictionary.swiftを開き、次の関数を構造定義に追加します。 



 // 1 mutating func insert(value: ValueType, forKey key: KeyType, atIndex index: Int) -> ValueType? { var adjustedIndex = index // 2 let existingValue = self.dictionary[key] if existingValue != nil { // 3 let existingIndex = find(self.array, key)! // 4 if existingIndex < index { adjustedIndex-- } self.array.removeAtIndex(existingIndex) } // 5 self.array.insert(key, atIndex:adjustedIndex) self.dictionary[key] = value // 6 return existingValue }




これらはすべて、新しい情報に慣れ親しんでくれるでしょう。 それらを段階的に見てみましょう：



  1. 新しいオブジェクトの挿入に役立つメソッドinsert（_：forKey：atIndex）には、3つのパラメーターが必要です。特定のキーの値とインデックス。キーと値のペアを挿入できます。 ここには、これまで見たことがないかもしれないキーワードがあります：変化します。
  2. そのキーに既存の値が既に存在する場合は、既存の値を返す辞書インデクサーにキーを向けます。 この挿入メソッドは、ディクショナリのupdateValueと同じ動作を模倣するため、キーの既存の値を保持します。
  3. 既存の値がある場合、メソッドを使用する場合にのみ、そのキーの配列内のインデックスが検索されます。
  4. 既存のキーが挿入インデックスの前にある場合、既存のキーを削除する必要があるため、挿入インデックスを設定する必要があります。
  5. 必要に応じて、配列と辞書を更新する必要があります。
  6. 最後に、既存の値を返します。 関数が追加の値を返すため、既存の値が存在しない可能性があるため。




辞書に値を追加できるようになったので、値を削除するのはどうですか？



OrderedDictionaryの構造定義に次の関数を追加します。 



 // 1 mutating func removeAtIndex(index: Int) -> (KeyType, ValueType) { // 2 precondition(index < self.array.count, "Index out-of-bounds") // 3 let key = self.array.removeAtIndex(index) // 4 let value = self.dictionary.removeValueForKey(key)! // 5 return (key, value) }




コードをステップごとにもう一度見てみましょう。



  1. これは、構造の内部状態を変更する関数であり、したがって、そのように認識します。 removeAtIndexという名前は、Arrayのメソッドに対応しています。 必要に応じて、システムライブラリのAPIのミラーリングを検討することをお勧めします。 これにより、APIを使用している開発者がプラットフォーム上で気軽に操作できます。
  2. まず、インデックスをチェックして、配列内にあるかどうかを確認できます。 基本配列から許容範囲外のアイテムを削除しようとすると、ランタイムエラーがスローされるため、チェックはこれを少し前に検出します。 Claim-CでObjective-Cのステートメントを使用した可能性があります。 assertはSwiftでも利用可能ですが、前提条件が機能しなくなった場合にアプリケーションを終了できるように、現在、最終ビルドで前提条件が使用されています。
  3. 次に、配列から値を削除しながら、指定されたインデックスの配列からキーを取得します。
  4. 次に、そのキーの値を辞書から削除します。これにより、以前に存在していた値も返されます。 辞書にはこのキーの値が含まれていない可能性があるため、removeValueForKeyは追加のマテリアルを返します。 この場合、ディクショナリに追加できるメソッドは独自に作成した挿入（_：_：forKey：atIndex :)のみであるため、ディクショナリにはこのキーの値が含まれることがわかっています。 したがって、価値があることを知りながら、追加の資料をすぐに公開できます。
  5. 最後に、キーと値をタプルに返します。 これは、既存の値を返すremoveAtIndex配列とremoveValueForKey辞書の動作と一致しています。




値へのアクセス



これでディクショナリに書き込むことができますが、ディクショナリから読み取ることはできません-これはデータ構造には役に立ちません！ ここで、辞書から値を取得できるメソッドを追加する必要があります。



OrderedDictionary.swiftを開き、次のコードを定義構造に追加し、配列と変数の宣言の下で辞書をポイントします。 



 var count: Int { return self.array.count }




これは、通常そのようなデータ構造に必要なデータである、順序付けられた語彙の数の計算されたプロパティです。 配列内の番号は常に順序付けられた辞書の番号と一致するため、すべてが単純になります



次に、辞書の要素にアクセスする必要があります。 Swiftでは、次のようにインデックス構文を使用して辞書にアクセスします。 



 let dictionary = [1: "one", 2: "two"] let one = dictionary[1] // Subscript




これで構文に精通しましたが、おそらく配列と辞書に使用されていることを見ただけでしょう。 独自のクラスと構造をどのように使用しますか？ 幸い、Swiftを使用すると、カスタムクラスにインデックス動作を簡単に追加できます。



構造定義の下部に次のコードを追加します。 



 // 1 subscript(key: KeyType) -> ValueType? { // 2(a) get { // 3 return self.dictionary[key] } // 2(b) set { // 4 if let index = find(self.array, key) { } else { self.array.append(key) } // 5 self.dictionary[key] = newValue }




このコードの機能は次のとおりです。



  • これは、インデックスの動作を追加するのと同じように機能し、funcまたはvarの代わりに、indexキーワードが使用されます。 パラメーター（この場合はキー）は、角括弧内に表示されるオブジェクトを定義します。
  • 添え字には、計算されたプロパティと同様にアクセス方法を含めることができます。 （a）getおよび（b）setメソッドはそれぞれアクセス方法を定義することに注意してください。
  • getメソッドは簡単です。このキーの値について辞書を照会する必要があります。 辞書の添え字はすでに追加のマテリアルを返しているため、そのキーに値がないことを示すことができます。
  • setメソッドはより複雑です。 まず、キーが順序付けられた辞書に既に存在するかどうかを確認します。 存在しない場合は、アレイに追加する必要があります。 新しいキーは配列の最後に移動することをお勧めします。そのため、値を追加して配列に追加します。
  • 最後に、指定されたキーの辞書に新しい値を追加し、暗黙的に名前が付けられたnewValue変数を介して新しい値に渡します。




これで、通常の辞書であるかのように、順序付けされた辞書にインデックスを付けることができます。 特定のキーの値を取得できますが、配列のようにインデックスでアクセスするのはどうですか？ 順序付けられたディクショナリでどのように機能するかを見て、インデックスを介して要素にアクセスするのもいいでしょう。



クラスと構造体は、さまざまなタイプの引数に対して複数のインデックス定義を持つことができます。 構造定義の下部に次の関数を追加します。 



 subscript(index: Int) -> (KeyType, ValueType) { // 1 get { // 2 precondition(index < self.array.count, "Index out-of-bounds") // 3 let key = self.array[index] // 4 let value = self.dictionary[key]! // 5 return (key, value) } }




これは、パラメータタイプがIntになっていることを除いて、以前に追加した添え字に似ています。これは、これが配列のインデックスを参照するために使用するものだからです。 ただし、今回は、指定されたインデックスを格納するのはOrderedDictionaryであるため、結果の型はキーと値のタプルです。



このコードの仕組み：



  1. この添え字にはゲッターメソッドのみがあります。 順序付けされた辞書のサイズ範囲にあるインデックスを最初にチェックすることで、同様にセッターメソッドを実装できます。
  2. インデックスは配列内に存在する必要があります。これにより、順序付けされた辞書の長さが決まります。 前提条件を使用して、順序付けされた辞書の外部にアクセスしようとするプログラマーに警告します。
  3. キーは、配列から取得することで見つけることができます。
  4. 値は、特定のキーの辞書から取得することで見つけることができます。 繰り返しますが、拡張された追加マテリアルの使用に注意してください。知っているように、辞書には配列内のキーの値が含まれている必要があります。
  5. 最後に、キーと値を含むタプルを返します。




タスク：このインデックスのセッターを実装します。 前のインデックス定義のように、セットを追加してから完了します。



この時点で、KeyTypeがIntの場合はどうなるのか疑問に思うかもしれません。 ジェネリックスの利点は、Intを含むキーとしての任意のハッシュタイプの入力です。 この場合、インデックスはどのインデックスコードを使用するかをどのように知るのですか？



これは、コンパイラが何をすべきかを知るために、より多くの型情報をコンパイラに提供する必要がある場所です。 各インデックスには異なる戻り値の型があることに注意してください。 したがって、キー値のタプルを設定しようとしている場合、コンパイラは配列のような添え字を使用する必要があることを認識します。
