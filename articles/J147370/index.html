Windows AzureでSharePointを使用して仮想マシンテンプレートを作成する





ご存知のとおり、Windows Azureの新しいバージョンには仮想マシンを使用する機能があります。 含めて、SharePoint 2010を搭載したマシンでもかまいません。

既定では、システムの一覧にSharePoint 2010を実行しているコンピューターはありません。これを修正する方法を検討してください。



Windows Azureへのアクセスを取得する



現在、 90日間の無料トライアルが利用可能です。

また、MSDN、Microsoftパートナーネットワーク、またはBizSparkプログラムのメンバーである場合にもアクセスできます。 詳細はこちら 。



Windows Serverマシンを作成する



まず、 公式ガイドに従って仮想ネットワークを作成します。

次に、 ネットワークセクション（図1）に移動し、 Affinity Groupという名前を覚えておいてください。これは、仮想マシンを作成するときに役立ちます。



図 1. Windows Azureポータルのネットワークセクション



ポータルの下部にある[新規]ボタンをクリックします。 その後、仮想マシンとギャラリーから（図2）



図 2.ギャラリーから仮想マシンを作成する



次に、ギャラリーで、Windows Server 2008 R2を選択します（図3）



図 3.仮想マシンテンプレートの選択



テンプレートを選択したら、下部の矢印をクリックして次の手順に進みます。 仮想マシンのパラメーターを指定します（図4）



図 4.仮想マシンの設定



その後、次の手順に進み、仮想マシンモードを選択する必要があります（図5）。



図 5.仮想マシンモードの選択



タイプとして、スタンドアロン仮想マシンを選択し、DNS名、ストレージアカウント（作成済みの場合）、および仮想マシンが動作するネットワークを指定します。

最後のステップは、 可用性 セットを作成することです（図6）可用性セット-既存のマシンを複製する地理的に分散した仮想マシンのグループ。 デフォルト値のままにします。



図 6.可用性セット



新しく作成されたマシンは、ステータスがRunningになった後にのみ使用できます。



SharePoint 2010をインストールする



SharePoint 2010を詳細にインストールすることは、これはすべて既に行われているため、何度も繰り返します。

sysprepを使用してマシンを準備します。 そして、それを機能させるためには、2つの条件（KB 2728976による ）を覚えておく必要があります

  • SharePointをインストールした直後にSharePoint構成ウィザード（PSConfig）を実行しないでください。
  • SharePoint 2010をオフラインでインストールする必要はありません（データベースとSharePointを同じマシンにインストールする）。


SharePoint用Service Pack 1および必要なすべての更新プログラムをインストールすることもできます。



仮想マシンに基づいてイメージを作成する



前述のように、sysprepを使用します。sysprepの場合は、コマンドラインに入力する必要があります

%WINDIR%\system32\sysprep\sysprep.exe /oobe /generalize /shutdown





その後、Azure管理ポータルで、[ 仮想マシン]セクションに移動します。 その後、マシンが完全に停止するのを待って、[ キャプチャ ]をクリックします（図7）



図 7.仮想マシンのコントロールパネル



表示されるウィンドウ（図8）で、イメージの名前を指定し、sysprepが仮想マシンで実行されていたボックスをオンにします。



図 8. VMイメージの設定



仮想マシンを作成するためにテンプレートを使用します



テンプレートの準備ができたら、それに基づいて新しいマシンを作成できます。 作成する手順は上記の手順と似ていますが、違いが1つあります。テンプレートを選択するときは、 画像セクションに移動して、事前に準備したテンプレートを選択します（図9）。



図9 独自の仮想マシンテンプレートの選択



そのため、SharePoint 2010で仮想マシンテンプレートを作成するプロセスを検討しました。
