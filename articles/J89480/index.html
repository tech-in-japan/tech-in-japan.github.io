Gentoo Linuxで5年

古いノートパソコン



Gentoo Linuxを当時の新しいHP nx6125にインストールしてから今月が5年になります。 過去に、デスクトップハードウェアのサポートは根本的に改善されました。HPEliteBook 8440pに Gentooをインストールした最近の経験と比較すると、違いは非常に大きくなっています。 ディストリビューションのカーネルバージョンは2.6.15で、2.6.36になりました。デスクトップシステムでLinux OSを使用することは困難ではなく、非常に快適な体験でしたが、この違いに陥りました。



インストール後初めて、ACPI は完全に異常な状態でした。 途中で、3Dハードウェアアクセラレーションは開始されず、システムがハングしたり、そのように落ちたりしました。 ただし、 カーネルモード設定からのダウンロード、USBヘッドフォンを介したインターネット経由でのオーディオビデオコンテンツのダウンロード、xorg.confファイルの誤った設定など、さまざまな理由により、システムが頻繁にクラッシュし、非常に定期的に最後までクラッシュしました。 、 バグレポートを送信し 、Gentooの使用を続けました。





叙情的な余談



同じコンピューターでWindows XPを使用しましたが、使用頻度ははるかに低く、まったくクラッシュしませんでした。 これらの事実に関連して、私はいくつかの結論を出しました。 私が最後に出会った最初のことは、非常に嫌なATI Xpress 200mチップセットで、3Dハードウェアアクセラレーションを備えた安定したビデオドライバーがかなり遅く書かれていました 。 ユーザーと開発者のコ​​メントはこれを確認しました。 2つ目は、以前のバージョンとは異なり、Windows XPは非常に安定していることです。 Windows XP以降のデスクトップシステムで「LinuxはWindowsよりも安定している」という話は、私にとってはとてつもないことです。 それにもかかわらず、Windowsを読み込むたびに、私は明らかに奇妙で、不快で、なじみのない領域にいると感じます。 その理由は、Linuxを選択することは強制されていないが、実際的な考慮によって動機付けられていないからです。 この選択は、ほとんどの場合意識的に価値があります。 特定のITスキルと価値観のセットにより、Linuxに切り替えることができます。 そして、その中に多くの利点を見つけ、それに慣れるだけです。 これが私の3番目の結論です。



新しいラップトップ



Gentooは過去5年間で多くの成果を上げてきました。 Emergeはより高速になり、新機能で大きくなりました。 新しいパッケージマネージャーとブートスクリプトの新しいシステムが登場しましたが 、まだ安定したブランチに移動していません。 したがって、 HP EliteBook 8440pにGentooをインストールする方がはるかに簡単で、ある場所ではwpa_supplicant.conf構成ファイルのwpa_supplicantを介してWPA2でWiFiをセットアップすることで速度を落としました 。 ワイヤレスネットワークカードはiwlagnドライバーを使用します。 KDEのNetwork Managerユーティリティを使用して、イーサネットケーブル経由でインターネットに接続することでWiFiを構成できましたが、スポーツではないと考えました。 wextの代わりにnl80211ドライバーを使用する必要があることが判明しました 。



このために処方します。



modules=( "wpa_supplicant" )

wpa_supplicant_wlan0="-Dnl80211"

wpa_timeout_wlan0=15







特に、 / etc / conf.d / netファイルにあります。 一般に、 wpa_supplicantは 、ログが記録されてお粗末なログですが、すべてを構成すると、すべてを安全に忘れることができます。



さらに、新しいハードウェアへのインストールは迅速でエラーがありませんでした。 パッケージのアセンブリ速度が大幅に向上しました。巨大なLibreOffice / OpenOfficeでさえ5時間以内に、Chromiumで2時間以内に、残りの重量物を1時間以内に組み立てることができます。 参照用：



model name: Intel(R) Core(TM) i5 CPU M 540 @ 2.53GHz

bogomips: 5054.48

MemTotal: 3842972 kB







したがって、リリースとは独立した非常に柔軟なシステムがあり、あまりにも頻繁に、またはまれにしか出ず、ネットワーク接続などをわずかに不注意で破壊します。 各イノベーションを理解し、システムに適切に配置する時間があるときに、Gentooが採用している永続的なリリースを好む。 しかし、時には行き止まりに「スリップ」する必要があります。 たとえば、 xorg-serverを1.4から1.9に一気に更新したいのですが、1.5 InputDeviceで HAL、fdi xml形式のファイルに転送してからバージョン1.8に戻しました。



結論として



そして最後に、Gentushniksへのいくつかのヒント。

  • Gentoo開発者ブログを読んでください。 ほとんどの場合、プログラマーのメモがありますが、特定の話題の問題に関する詳細なガイドと、あちこちに散らばるレーキについての警告がある場合があります。
