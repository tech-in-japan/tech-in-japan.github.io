MapillaryサービスでGoProフレームごとの写真を使用する方法

過去6か月で、Mapillary Webサービスの人気が高まりました。50万枚以上の写真がアップロードされ、過去10日間で10万枚以上がアップロードされました。



一見、このケースに適しているのは通常のスマートフォンだけです。 しかし、GoProなどのアクションカメラを使用してMapillaryの写真を収集できるとしたらどうでしょうか。 自転車、ヘルメット、または車に搭載されていますか？ 結局のところ、これは可能です。



そして先週、私は短い旅行をして、TimeLapse機能を使用して30分で約1,700枚の写真を収集しました。 32ギガバイトのSDカードでは、3.4ギガバイトしか食べませんでした。 メモリカードがいっぱいになる前に、私はまだ4時間以上乗車し、約15万枚の写真を撮ることができました。



しかし、これは完全に些細な問題ではないため、これがどのように行われるかをより詳細に説明する必要があると思います。 主な問題は、GoProにGPSレシーバーが含まれていないことです。 地図サービスは、地図のどこに写真を表示するかをどのようにして知るのですか？ 私たち自身がこの情報を彼に渡さなければならないので、別のGPS受信機を使用する必要があります。 個人的には、HTC Oneスマートフォンを使用しました。これは、Mapillaryで通常の写真を撮るために使用したものと同じものです。



つまり、手順は次のとおりです。

  • さまざまな取り付け装置を使用して、カメラを自転車または車に取り付けます。
  • プログラムを実行してGPSトラックを記録します。 私はgeopaparazziプログラムを使用しましたが、同様に機能する他の多くのものがあります。
  • GoProをフレームごとの撮影に設定します。 設定を選択しました：5MP、中、狭い、1秒。
  • GoProを起動して乗車します。
  • 写真とGPXトラックをコンピューターにアップロードします。
  • gpx2exifを使用して、写真を地理座標に添付し、撮影時間をGPXトラックと同期します。
  • Mapillaryに写真をアップロードします。


難しいことではないようです。 先週の旅行を例に、これらのアクションについて詳しく説明します。







GoProをマツダRX8の左フロントフェンダーに取り付けて、道路の真ん中がよく見えるようにしました。







次に、車内のHTC Oneスマートフォンを特別なマウントに固定しました。 「 GoPro App 」プログラムを使用して、カメラに参加し、GoProが見ている地形のプレビューを表示しました。 これは便利ですが、実際にはそれほど必要ではありません。 記録が開始されると、プレビューが消えます。 GPS衛星信号がよく捕捉される場所にあるスマートフォンが車内にあることだけが必要です。 GoProアプリを使用する場合は、カメラの時間を調整するために少し時間をかけてください。デバイスの現在の時間の差はできるだけ小さくする必要があります。 後で見るように、トラックと写真を同期するには時間差を考慮する必要があります。 そして、あなたがしなければならない修正が少なければ少ないほど、作業を完了しやすくなります。







カメラの設定をもう一度確認しましょう。1コマ撮影のタイムラプス写真モードをオンにする必要があります。 これはGoProアプリで簡単に行えますが、GoPro自体で行うことができます。 携帯電話で撮影しているものに似たビューを得るために、5MB、中、狭を選択しました。 同等の焦点距離（35 mmカメラの場合）は約20 mmです。 GoProのデフォルト設定では、視界が広く、光学歪みが大きいため、今回の旅行ではこのモードを拒否しました。 Mapillaryの写真を広角4Kビデオストリームから撮影したとき、別の記事で別のシナリオを説明する予定です。 難しかったので、とりあえず話を飛ばしてください。



1秒あたり1フレームを選択した設定では、写真間の距離は40〜80 km / hの速度で10〜20メートルになります。 2秒ごとに1フレーム程度のMapillaryの推奨事項は自転車に適していますが、もちろん、自転車よりも速く動くことを計画しています！



GPSトラックを記録するためのアプリケーションを起動します。 この場合、 geopaparazziプログラムを使用しました 。 このアプリケーションには、記録を開始するボタンがあります。 このボタンを押して、提案されたファイル名を確認してトラックを保存しました。 わかった これですべて準備完了です。 撮影を開始して行くだけです！







転がると、カメラとGPS受信機での記録を停止します。 これで実際の作業が始まります！ マテリアルをMapillaryにアップロードする前に、ジオリファレンスを実行する必要があります。 Geopaparazziでは、トラックをGPXファイルにエクスポートし、メールで自分に送信しました。 GoProから、カードを取り出して写真をラップトップにコピーしました。



私が最初にやりたかったことは、私の旅行がどのように見えるかを見ることでした。 コマンドを実行しました： 



geotag -g väla_to_billesholm.gpx \ -o väla_to_billesholm.png \ -D 1 -s 2048x2048




その結果、パスマップ全体の高解像度の画像が得られます。 -Dオプションを使用して、マップ上のマーカー間の間隔を大きくすることに注意してください。 これは、ジオタグが、たとえば高速自転車に乗るような短いトラックに対してデフォルトで設定されているために必要です。 結果の画像は、旅行のキーポイントの時間と位置を示しています。 カメラとGPS受信機の時刻の一致を手動で確認し、それらの間のクロック誤差をより正確に決定できるいくつかの特徴的な場所でスケールを拡大する必要があります。 同期段階で、この違いを修正することができます。それを正しく判断することが重要です。



コマンドを実行しました： 



 geotag -R 20140505T12:39:00+02-20140505T12:41:00+02 \ -g väla_to_billesholm.gpx \ -o krop.png -D 0.1
この画像を生成しました：







橋の下に入ったところの写真を見つけて、時計を確認できます。







この写真のEXIFデータは、12：39：11に撮影されたことを示しています。 地図を見ると、12：39：14にトンネルを通過したことがわかります。 したがって、エラーは3秒です。 ジオリファレンスプロセスでこの値を使用できますが、最初に別の写真を確認することをお勧めします。







建物や交差点などのオブジェクトを識別できるため、Mörarpを通るパスのマップを生成しました。 停止した交差点（私の右折など）は使用しないでください。 移動中に近くにあるランドマークを探してください。 地図の上部の右側の最初の通路を探していたところ、12：46：10に撮影された写真が見つかりました。







私はそれらがどこにあるかを正確に知っているいくつかのランドマークに注意しました：中央の警告標識、交差点の白いマーキング、街路照明のポール。



チェックする別の興味深い方法も使用できます。Googleストリートビューの写真がある場所にいた場合は、2つの画像を比較できます。







Googleのストリートビューのショットには、道路標識、交差点のマーキング、木、電柱が表示されています。 しかし、フェンスはありません。クリスマスツリーはその場所で成長します。 Google車がこの場所を運転した後に建設が始まったことは明らかです。 Googleは、写真は2011年9月-約2.5年前-に撮影されたと言う。



GPXトラックのデータからMerarpを見ると、交差点が12:46:13に通過したことがわかります。これは、カメラが写真に記録した時間より3秒遅れています。 また、3秒のエラーがあります。 喜ばしいことに、エラーはトラック全体で同じであることがわかりました。 次のコマンドを使用して、1,500枚の写真すべてを続行して同期できます。 



 geotag -g väla_to_billesholm.gpx \ 20140505_TimeLapse/*JPG -t 3 -v




「-t -3」オプションを使用して時間オフセットを設定し、「-v」オプションを使用してプロセスを監視しました。 スクリプトはexif_fileコマンドラインプログラムのラッパーであるため、複数のプロセスが各ファイルの処理を開始します。 これには時間がかかりますが、最後にはすべての写真にGPXから取得した地理参照情報が含まれます。







写真にリンクする地理情報が完成したら、mapillary.comに写真をアップロードできます。 サイトを入力し、名前をクリックし、「画像をアップロード」を選択して、「ファイルを選択」ボタンをクリックします。 すべてのファイルを選択した後、ページを下にスクロールし、「アップロードの開始」ボタンをクリックします。 サイトの配色は、ダウンロードが開始されたかどうかが常に明確ではないようなものです。 ページを下にスクロールするだけで、各画像の下に赤いローディングインジケータが表示されます。







最後に、ダウンロードが完了したら、名前をクリックして、「マイアップロード」を選択します。 'トラックの新しい画像が表示されます。







Mapillaryで写真を表示するには、最新のアップロードをクリックしてください！







写真の完全な処理には時間がかかる場合があるため、すぐに利用できない場合でも心配しないでください。 それらに少し後で戻ります。



そして今-スローモーションビデオ


記載されている撮影方法のGoProでの設定は、「タイムラプス」-「スローモーション」と呼ばれる無駄ではありません。 写真から映画を作ることができます。 毎秒1フレームを撮影したため、毎秒25フレームの頻度でビデオを作成すると、25倍の加速が得られます。 これはすごい！ 私が得たものを見る：







このビデオは次のように作成されました::

  • すべての写真の名前を変更して、ファイル名に0000で始まる番号を付けます。foo-0000.jpegのような名前を付けました。 生活を楽にするために、必要な名前でハードリンクを作成するルビースクリプトを作成しました。 その後、ffmpegコマンドを使用してビデオをコンパイルできます。 



     ffmpeg -f image2 -i foo-%04d.jpeg \ -r 25 -s 1280x960 ../myvid.mp4


  • このコマンドは、7メガピクセル4：3形式の写真を960p HDムービーに圧縮します。
  • 次に、OpenShotビデオエディターを使用して、16：9にトリミングし、サウンドを追加し、カードを追加し、720p解像度で圧縮して中程度の品質でWebにすばやくダウンロードしました。




Gpx2exifインストール


この記事では、ジオタグチームが広く使用されています。 このコマンドは、ruby gem "gpx2exif"に含まれています。 この記事では、バージョン0.3.6で使用可能な機能を使用しました。 ただし、記事の公開時には、バージョン0.3.1が利用可能でした。 したがって、最新バージョンのインストール方法を説明します。



UbuntuにRubyをインストールする


最初に必要なのはRubyです。 インストールはオペレーティングシステムによって異なります。 Ubuntu 14.04とRVMを使用しています。ここでは、OSに適した手順を示します。 また、プラットフォームに適したアドバイスを得るためにruby-lang.orgとrvm.ioのサイトに行くことをお勧めします。 



 sudo apt-get install curl curl -sSL https://get.rvm.io | sudo bash -s stable --ruby sudo usermod -g rvm craig # logout and login to get RVM group source /etc/profile.d/rvm.sh


PNG画像を作成するプログラムの部分は、その作業にImageMagickを使用します。 Ubuntuの場合、これは最初にいくつかの依存関係をインストールする必要があることを意味します。 



 sudo apt-get install imagemagick imagemagick-doc libmagickwand-dev gem install rmagick # Requires libmagickwand-dev to compile




rubygems.orgからのインストール


rubyがインストールされたら、gemをインストールするだけです： 

 gem install gpx2exif


次に、gem listコマンドを使用してリストを出力し、結果としてインストールされたバージョンを見つけます。 0.3.5より古い場合は、以下の手順に従ってください。



githubを使用したインストール


gitをインストールしてからコマンドを実行します： 



 git clone https://github.com/craigtaverner/gpx2exif cd gpx2exif bundle install rake build gem install pkg/gpx2exif-0.3.6.gem


すべてがうまくいけば、Ruby Gemの最新バージョンをビルドしてインストールしました。
