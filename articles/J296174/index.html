クラウドコンピューティング市場はすべての期待を超えています（パート1）





シリコンバレーでは、スタートアップの急増も衰退し始めていますが、Amazon、Microsoft、Googleはいずれも過去4四半期で優れた利益を報告しており、クラウドコンピューティングに大きな賭けを行っています。



これらの会社は正確に何を売っていますか？ バイヤーは誰ですか？ そして、10年前にテクノロジー業界になかった企業であるAmazonが、すべての競合他社に勝っているのはなぜですか？



クラウドビジネスとの関係は次のとおりです。



なぜ誰もがクラウドコンピューティングを追いかけているのか



クラウドコンピューティングの分野で最も重要な概念は「大規模」です。



ウェブサイトとサービスをサポートするために、Amazon、Microsoft、Googleは幅広いコンピューティングインフラストラクチャを構築しました。 これらの企業のデータセンターは、他のほとんどの企業が独自に構築または保守できるサーバールームやデータセンターよりもはるかに大きく効率的です。



そして今、巨人はこれらの施設のいくつかを世界中の開発者や企業にリースしています。 開発者または会社は、クレジットカードで支払うだけで、無制限のコンピューティングパワーにアクセスできます。



これは、ソフトウェアがはるかに大規模で、少ない費用で、より高いレベルの可用性とパフォーマンスで動作できることを意味します。 また、データセンターのコンテンツについて心配する必要はありません。



一見、すべての主要なプレーヤーは、開発者向けに同じ基本的なサービスセットを提供しています。



Amazon、Google、Microsoft、および他のほとんどのクラウドプレーヤーは、2つの異なるクラウド「レイヤー」を提供します。



  • Infrastructure as a Service（「IaaS」）を使用すると、他の誰かのデータセンターで仮想サーバーとストレージシステムを構成できます。 これが最も基本的なレイヤーです。
  • Platform as a Service、つまり「PaaS」は、開発者がアプリケーションを構築し、使用するサーバーに関する心配を排除するのを容易にするツールとサービスのセットです。


（第3層の「サービスとしてのソフトウェア」、またはSaaSがあり、これはSalesforceやマーケティング製品、または競合するMicrosoft Office 365やGoogle At Work製品などのアプリケーションで動作します。これらはすべてクラウド製品ですが、通常は「クラウドコンピューティング」に関しては、人々はそれを意味しません。）



すべてのクラウドコンピューティングプロバイダーの主なアクション戦略は、これらのクラウドを独立した開発者と大企業の両方に適したものにすることです。



開発者は単一のアプリケーションを使用して開始できますが、ビジネスが成長するにつれてクラウドの必要性も高まります。



クラウドプラットフォームが受信するクライアントが多いほど、追加できるサーバーの数は多くなります。 サーバーが大きいほど、成長による節約額が大きくなるため、企業は、より信頼性の高いエンタープライズ固有の機能を低価格で顧客に提供できます。 価格が低く、製品が優れているほど、クラウドにアクセスしたい顧客とユーザーが増えます。



Amazonはこれを「仮想サイクル」と呼んでいます。



Amazonはクラウドサービスを最初に開発した...



...それがまさに競合他社よりも優位になっている理由です。



AmazonのWebサービスは最も人気があり、2006年にJeff Bezosが承認した実験にルーツを持ち、今年少なくとも70億ドルの収益を上げる企業の一員になりました。



「これは彼のドリームフィールドのスクリプトです」と、ガートナーのクラウドサービス研究担当副社長、エドアンダーソンは述べています。 「彼らは製品を作成しましたが、全員がそれに出くわしました。」



Wikibonの統計を使用するStatistaのこのチャートは、IaaSにおけるAmazonの優位性を示しています。







AWSのサービスが開始されたばかりのとき、顧客は主要インフラストラクチャのサービスのセットを提供されました：仮想サーバー用のElastic Compute Cloud（EC2）。 少し後に、データを保存するためにAmazon Simple Storage Service（またはS3）が追加されました。



当初、アマゾンウェブサービスは主に小規模な開発者がさまざまな部分をテストしたり、簡単なウェブサイトを管理したりするための安価な方法として使用されていました。



しかし、これらのクライアントの多く（Netflix、Airbnb、およびその後のSlackを含む）は、小規模で実験的なアプリケーションの管理からAmazon Web Servicesに移行し、AWSをペースの速いビジネスの中核にしました。



Forrester ResearchのチーフアナリストであるDave Bartolettiは、次のように述べています。



この傾向は、Amazonの好循環を開始しました。 最初の顧客がもたらした収益により、Amazonは企業向けにより多くの機能に投資し、スタートアップや実験を超えて、大規模なアプリケーションで機能するより良い品質のサービスを提供することができました。



現在、Comcast、Capital One、米国中央情報局などの大企業もAmazon Web Servicesの顧客であり、コンピューティングインフラストラクチャの少なくとも一部でサービスを利用しています。



Amazonの継続的なリーダーシップは重要な成功要因です。 Amazonの好循環は他の誰よりも長く続くため、同社は競合他社よりも幅広い機能と規模を提供できるという利点があります。

アマゾンウェブサービスは現在70億ドルのビジネスであり、2020年までに500億に成長すると予想されています。アマゾンウェブサービスは、市場の他の14人のプレーヤーを合わせただけのコンピューティングパワーを提供します（最近の推定ガートナー）。



マイクロソフトとその壮大な戦略



テクノロジー業界との激しい競争に直面して、巨人はすぐにAmazonレベルに到達すると思うかもしれません。



しかし、違います。 Microsoftは2000年代半ばからクラウドの概念に取り組んできましたが、2010年までは、Azure（AWSの主要な競合相手）の作成に世界を紹介しませんでした。



マイクロソフト（およびその他のエンタープライズソフトウェアの巨人）は、最初にクラウドを目新しさまたは流行と見なし、次に脅威と見なしました。 顧客がアマゾンウェブサービスに目を向けた場合、ほとんどの企業のデータセンターで使用されている数十億ドル規模の製品であるWindows ServerやSQL Serverなどのマイクロソフトソフトウェアの同じ数のコピーは必要ありません。



マイクロソフトはその後、その弱点を強みに変えることができました。 クラウドコンピューティングは現在、Microsoftの収益成長の最大の要因の1つになっています。ウォールストリートは喜んで受け入れます。



Azureが最初に起動されたとき、Windows Azureと呼ばれ、開発者がアプリケーションを簡単に構築できるようにするPlatform-as-a-Service（PaaS）レイヤーを提供していました。 2010年の発売以来、Azureは成長しており、Amazonを非常に人気のある低レベルのインフラストラクチャサービスを既に含んでいます。



Microsoftクラウドサービスビジネスユニットの責任者であるSatya Nadellaは、Azureをこれらの変革に導きました。 このケースでの彼の成功は、2014年に会社のCEOに任命された理由の一部でした。 現在、AzureはMicrosoftの最優先事項になり、Microsoftの最大の顧客の多くがすでに使用しているWindows Serverやその他のエンタープライズ製品との密接な関係に焦点を当てています。



マイクロソフトの主な利点は、テクノロジーではなく、そのノウハウと確立された顧客ベースにあります。



Microsoft Azureは、.NETランタイムで構築されたアプリケーション向けに最適化されています。これは、過去15年ほどWindowsでのプログラミングのマイクロソフト標準でした。 企業は、競合製品と比較して、WindowsアプリケーションをAzureクラウドに簡単に移行できます。



さらに、Microsoftの最大の顧客の多くは、いわゆる「エンタープライズ契約」、つまり、Microsoftソフトウェアの大幅な割引を会社から受ける契約を結んでいます。 マイクロソフトはこれらの契約を操作して、お客様がAzureを試用するインセンティブを持つようにすることができます。



ガートナーのアナリストであるアンダーソンは、次のように述べています。



テクノロジーについて話すと、Nadellaの出現により、Microsoftはその誇りを飲み込み、開発者が単に崇拝し、かつては以前のジェネラルと呼ばれていた無料のLinuxオペレーティングシステムなど、以前は潰そうとしていたAzureでそれらのテクノロジーのサポートを開始しましたMicrosoftディレクターのスティーブバルマーは、「がん」と「共産主義」です。



オープンソースコードに対するこの再発見された愛は、多くの開発者をマイクロソフトを愛するように導きました。



「Microsoftが成功した理由は、会社が開発者と共鳴したからです」とBartoletti氏は言います。



マイクロソフトはまだAmazonの背後にいます。 MicrosoftはAzureの統計情報を共有していませんが、AzureやOffice 365アプリスイートを含む同社のクラウドサービスからの総収益は、今年63億ドルに達するはずです。



Microsoftは、Azureが急速に成長していると主張しており、アナリストのGarthnerは、Azure Web Servicesよりも速く成長していると考えています。



最近、MicrosoftのCEOであるSatya Nadellaは、クラウドサービスのライバル関係をAmazonとMicrosoftの間の「シアトルレース」、および多くの小規模プレーヤーと呼びました。
