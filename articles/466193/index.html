The standard option for the auto-link and why it did not suit us



Mailhimp itself has excellent documentation on any functions, and specifically about this too: Share Your Blog Posts with Mailchimp . Translations into Russian can also be found, of varying degrees of adequacy.







Briefly, what you need to make an auto-link:







  1. Create a separate RSS feed for MailChimp (or even not create, but use a standard feed).
  2. Create a campaign, specify the source, configure the sending conditions.
  3. Create and select a template.
  4. Done. You are gorgeous!


What did not suit us is that it is simply impossible to add any of our own things.







There are basic data from the RSS feed that Mailhimp understands:







  • Basic information about the feed (title, link, description, date of formation or updating of the feed) - *|RSSFEED|*



    .
  • Link to a separate post - tag *|RSSFEED:URL|*



    , looks at the <link>



    inside the <item>



    (hereinafter).
  • The title of a single post is the tag *|RSSITEM:TITLE|*



    , looks at <title>



    .
  • Date of publication of a single post - tag *|RSSITEM:DATE|*



    , look at <pubDate>



    .
  • Text announcement of a separate post - tag *|RSSITEM:CONTENT|*



    , look at <description>



    , if this tag is not inside <item>



    - MailChimp looks at <content:encoded>



    . You can use HTML tags inside.

  • The full content of a single post is the tag *|RSSITEM:CONTENT_FULL|*



    , looks at <content:encoded>



    .
  • Image of a post, often a preview ( thumbnail in WordPress terms) - tag *|RSSITEM:IMAGE|*



    , looks at <media:content>



    and don‚Äôt forget about additional tags and attributes for media attachments : age markings, information about the file itself and so on.

  • Comment counter

  • Separate custom post cover , which you need to somehow reach out to;

Create RSS feeds



The first thing to do is create a separate RSS feed for MailChimp. There are already several similar feeds on the PromoPult blog: for Yandex.Zen and for Turbo pages .







You can create a new feed through add_feed () . Here is the code in the theme function file:







functions.php: 









 /* start       MailChimp */ add_action( 'init', 'customRSSforMC' ); function customRSSforMC() { add_feed( 'mchimp', 'customRSSforMCFunc' ); } function customRSSforMCFunc() { get_template_part( 'rss', 'mchimp' ); } /* end       MailChimp */


In line No. 7 get_template_part ( 'rss', 'mchimp' );



- indicates that the template for the feed lies in the root of the theme folder and is called rss-zen.php

After creating the file and adding the function, you need to go to the WordPress admin panel, the CNC settings: ‚ÄúConsole ‚Üí Settings ‚Üí Permalinks‚Äù and click the ‚ÄúSave Changes‚Äù button so that WordPress updates the /feed/URL

Separate line about RSSFEED and FEEDBLOCK



What is the difference between *|RSSFEED|*



and *|FEEDBLOCK|*?









The differences are easy to figure out:







  • *|FEEDBLOCK|*



    can be used inside any email and campaign templates. For example, if you want to talk about the latest posts in a periodic newsletter, use it. You can specify a link to any RSS source.
  • *|RSSFEED|*



    only works in auto campaigns and as a source sees what is specified in the campaign settings.
