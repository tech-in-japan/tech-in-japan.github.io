ステップ4-レイアウト




レイアウトプロセス自体は標準と変わりません。 確かに微妙な違いがあるので、それらについて話しましょう。 最初のそのようなニュアンスはメタタグです。 



<meta name="viewport" content="width=device-width, initial-scale=1, user-scalable=no" />




htmlコードのヘッダーにこの行がないと、アプリケーションは通常のサイトとして表示されます。 ブラウザはズームしますが、アプリケーションにリアリズムは追加されません。



デスクトップブラウザとは異なり、携帯電話ブラウザ（おそらくすべてではない）は、フォーカスが設定される要素にフレームを追加します。 Google Chromeでは、次のフィールドにデータを入力した時点で、焦点を合わせると同様のフレームがデフォルトで表示されます。 同様に扱われます。 



 input:focus { outline: 0 none; } textarea:focus { outline: 0 none; } .Button:focus { outline: 0 none; }




そして、最新のニュアンスは位置です：修正されました。 そして、普遍的な解決策がないため、これは本当に問題です。 すべてはモバイルブラウザー自体にかかっており、そのような機能をサポートしていない、またはサポートしているが、完全にはサポートしていません。 また、すべての場合に1つのソリューションでコントロールパネルを修正することもできません。 たとえば、バージョン1.1までのjquerymobileは、ブラウザーが位置をサポートしていない場合：固定、エミュレートされたスクロール、および固定要素の位置を動的に変更します。これは一般にリアリズムを与えず、「氷がない」ように見えます。



このリンクには、位置をサポートするモバイルブラウザの説明があります：修正済み

bradfrostweb.com/blog/mobile/fixed-position

また、位置の作業をエミュレートするJavascriptライブラリへのリンクもあります：固定およびスクロールプロセス。 残念ながら、それらのどれも満足のいくものとは言えません。



私の特定のケースでは、モバイルプラットフォームはAndroid 2.3としてリストされており、position：fixedをサポートしていますが、カスタムズームは機能しません。これはアプリケーションでは本質的に役に立ちません。 ビューポートヘッダーで指定する 



 <meta name="viewport" content="width=device-width, initial-scale=1, user-scalable=no" />




そして、スタイルを処方する 



  .Header { background-color: white; background-image: none; border: none; text-shadow: none; border-bottom: white solid 3px; font-weight: bold; position: fixed; width: 100%; height: 62px; top: 0; left: 0; z-index: 100 }
