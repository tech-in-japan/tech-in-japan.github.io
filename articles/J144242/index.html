Windows用のサイトを開発するためのWAMPプラットフォームの選択

私の青春の時点では、インターネットのロシア語セグメントで人気があったWAMP複合体は1つだけでした。 これは、ご想像のとおり、古き良きデンワーです。 彼は2003年に私のコンピューターにしっかりと落ち着きました。 他の選択肢は、当時存在していた場合、狭い円で知られており、あまり人気がありませんでした。



長い年月を経て、今日使用できるものを見てみましょう。 14のレビュー、テスト、比較をお勧めします！ Windows用のサイトを開発するために設計されたソフトウェアシステム。





1. AMPPS



最新バージョン：1.7

ロシア語：はい（部分的に）

ポータブル版：いいえ

ホームページ： http : //www.ampps.com/







基本キット：



  • Softaculous AMPPS 1.7
  • Apache 2.2.22
  • MySQL 5.5.22
  • PHP 5.3.10および5.2.17
  • PERL 5.10.1
  • Python 2.7.2
  • MongoDB 2.0.4


AMPPSは単なるWAMPアセンブリではなく、興行を離れることなく各アプリケーションをすばやくインストール、回転、選択できるアプリケーションのライブラリです。 この経済はすべて、綿密にカタログ化され、詳細に説明されています。



このアセンブリの間違いない利点は、多くの一般的なCMS、ブログプラットフォーム、Webフォーラム、掲示板、フォトギャラリーなどのサポートです。 200を超える利用可能なアプリケーションの合計！ パッケージに含まれるコンポーネントのセットも寿命に遅れることはありません。すべてのモジュールは非常に新鮮であり、注目に値するものとして、PythonやMongoDBなどのエキゾチックなものを見つけることができます。



マイナスのうち、私は部分的なロシア語の翻訳に注意したいと思います。これはscriptsディレクトリにのみ存在し、どこにでも存在しません。 Apacheを再起動した後、トレイメニューが機能しなくなり、Windowsを再起動しても解決しませんでした。 もう1つの欠点は、移植性がないことです。 私の意見では、AMPPSは、真面目な開発者よりも初心者がスクリプトを学ぶのに便利です。 私はこのソフトウェアパッケージで作業するのはあまり快適ではありませんでした;より良いオプションがあります。

テスト1-誤起動



テストの本質は、警告を受け取るか、すべてのインジケーターが緑色に変わり何も機能しないかどうかを確認するための、Apache構成への意図的な損傷です。 アセンブリを使用することには同意します。アセンブリを使用すると、誤解を招くような起動の成功を報告し、あなたをバカにしますが、それは十分な望みではありません。 サーバーが信号を送信しないか、密かに動作する場合、テストも失敗したと見なされます。



テストNo. 2-難しい経路



テストの本質は、複雑な名前のフォルダーにサーバーを意図的にインストールすることです。このテストでは、「C：\ test \ Mys $ ka！._-（）zlo {} +〜％＆2％Files」という名前のフォルダーを使用します。 PhpMyAdmin（すべてのアセンブリ内にあるため、便宜上使用されます）が開かない場合、テストはそれに応じて失敗します。 各人は個性であり、フォルダは最も予期しない名前と呼ばれることもあります。 テスト用に選択されたフォルダーの名前はWindowsで正しいため、正しく処理する必要があります。

非常に多くのソフトウェアシステムを調べてテストした結果、疑問が生じます。最終的に、日常業務で何を選択すべきでしょうか。 この質問に答えるのはそれほど簡単ではありません。各アセンブリは特定のタイプのタスクに適しています。



初心者には、最も単純で最も実績のあるWAMPキットとしてDenwerを使用することをお勧めします。 他のすべてのWeb開発者、およびDenwerの機能を欠いている新参者には、Open Serverの使用をお勧めします。



よく知られているスクリプト、フォーラム、フォトギャラリー、またはゲストブックをすぐに理解する必要があり、それをインストールして適切なプラットフォームを見つけることに煩わされたくない場合は、AMPPSを試すことができます。 もちろん、興味のあるスクリプトがこの開発でサポートされている場合に限ります。



誰かがローカルメールサーバー、Node.js、MongoDBなどのようなまれなモジュールを使用する必要がある場合、私たちのレビューの他の参加者の中から誰もが適切なオプションを選択できると思います。



PS著者の意見は、各アセンブリとの表面的なコミュニケーションに基づいており、作業中に明らかにされなかったプラットフォームの利点や隠れた欠点を反映していない場合があります。



PPS Linuxを搭載した仮想マシンを推奨し、他の選択肢を他に認めないコメンテーターに警告したい。 部分的には、サイトを実際のホスティングに移行するときに、ストリーム、fs権限、シンボリックリンク、コンソールなどの問題が発生する可能性があることに同意します。



ただし、反対側から見てください。 同様の問題は特殊なケースであり、まれにしか発生しません。 実際には、逆のことが当てはまります。Smartyエンジンのレイアウトデザイナーは、ストリーム、シンボリックリンク、およびコンソールについてまったく知る必要はありません。CMS開発者は、ダウンロードされたファイルに100年以上にわたって正しい書き込み権限を設定する方法を学びました。



その結果、私は言いたい-WAMPプラットフォームには存在する権利があり、ほとんどの場合、仮想マシンの使用は完全に不当です。
