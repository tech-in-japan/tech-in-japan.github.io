ダミーの取引システム

この記事では、取引システムの根底にあるいくつかのアイデアについてお話します。 プレゼンテーションは、可能な限り技術的および経済的な賢さなしに、可能な限りシンプルになります。



この分野の活動は英語圏で活発に開発されているため、多くの用語は、途中で意味を失わないように、翻訳なしで使用する方が簡単です。



この検討では、電子証券取引プロセスのコンピューター自動化の観点から、問題の技術的な側面に焦点を当てます。 （主に証券の一般的な経済理論から）関連する問題を明確にするには、他の情報源に目を向けるべきです。



したがって、まず、いくつかの簡単な定義。



取引とは、証券の取引です。 したがって、取引システム（プラットフォーム）は、さまざまな入札者（販売および購入者、およびさまざまなサービスを提供し、コミッションでライブを行う第三者）にこのプロセスの自動化を提供します。



電子商取引の原則の多くは、コンピューターがなかった以前のやり方を繰り返していることに注意してください。 しかし、今でも、完全なコンピューター化の時代には、売買には多くの手作業があり、さらに、売買の決定に直接関係していません。



セキュリティは取引の対象です（英語では「セキュリティ」と呼ばれます）。 証券には多くの種類があります（株式、債券、通貨など）。 コンピュータシステムの観点から見ると、すべてのセキュリティは単なる識別子であるため、その本質については説明しません（これについては、経済学に関する文献を参照する必要があります）。 たとえば、GOOG US Equityはニューヨーク証券取引所でのGoogleの株式、VOD LN Equityはロンドン証券取引所でのVodafoneの株式、CT2 Govtは米国政府の2年国債などです。 そのような識別子は数千あり、一般に、ほとんどの電子取引システムは、異なるプラットフォーム間の操作中の混乱を避けるために、これらの識別子の一意性を観察しようとします。



各システムは、会計を容易にするために証券を識別するための内部システムを持つことができます（通常は持っています）。



取引（または取引）は、証券の売買の事実です。 通常、トランザクションにはいくつかの基本的な特性があります。日付/時刻、セキュリティ識別子、ユニットあたりの量、およびセキュリティの購入/販売ユニット数です。 この情報の量は、通常ティック（またはフィード）と呼ばれます。 トレーダーが判断する「株式市場の状態」を説明するさまざまなレポート、チャート、グラフ、ティッカーなどの主な情報源である、さまざまな取引プラットフォームからリアルタイムで受信されるティックのフローです。いつ、どのくらい買うか。



ティックの到着率は非常に高い場合があります（1秒間に数百または数千）。 これが、取引システムにとって最も重要な指標の1つが速度である理由を説明しています。 入力データの処理が高速になるほど、トレーダーは市場の変化についてより速く学習し、より迅速に意思決定を行うことができます。 また、アルゴリズム取引については（後ほど説明します）-これは一般的に主要な指標の1つです。 この速度は、ソフトウェアの速度、情報チャネルの速度（多くの場合、インターネットなどの公衆ネットワークではなく専用回線を使用）、ハードウェアのパフォーマンスなど、多くの要因で構成されます。



システムのいくつかの問題が原因で、動作が非常に遅いため、ティックが開始し始める場合があります（システムにはそれらを処理する時間がありません）。 この場合、誤ったデータを表示しないように、一般化されたインジケーターの定期的な調整が必要です。



証券市場の主な参加者



セルサイド（この用語は翻訳せずに残す）は、再販に焦点を当てたものです。 トレーダーはセルサイドの典型的な代表者です。 通常、彼らには資産がありません。 彼らにはお金があります。 取引日の初めに、彼らは何かを購入し、日中に転売したり、他の何かを購入したりできます。 一日の終わりには、たとえ損失を出して売らなければならない場合でも、通常はすべてを売る傾向があります。 売り手は通常、たとえば予期しないニュースによって引き起こされる、毎日の速いレートの変化でお金を稼ぎます。



販売側の存在の経済的理由は、 処理システムの場合と同様に、量の問題です。 つまり、あなたはそれを手に入れることができず、お金がかかるので誰でも証券取引所に接続できます（預金、機器、必要に応じて認証など）。 小規模の顧客を遮断するために高価格の資格が与えられました（取引所は、1年に1株を購入する顧客をいじることに関心がありません）。 後者は、大規模な商社のサービスを使用する必要があり、すでに手数料を支払う必要があります。



バイサイドは「リッチ」な参加者です。 通常、これらはファンド、銀行、保険会社などです。 概して、これらは証券の実際の保有者です。 彼らは長い視点でそれらを大量に購入し、通常は価格が上昇する（または下落するとこれも起こります）と長い（たとえば数年）時間間隔で稼ぎます。



つまり、取引日中、売り手は証券を買い手側からあちこちに転売し、一日の終わりまでにはすべてが実際に買い手側に戻り、売り手側は日々の違いを持ちます。



もちろん、この分離は非常にarbitrary意的であり、多くの金融機関は売り手側と買い手側の両方です。 しかし、この違いを理解することは非常に重要です。



取引システムのいくつかの重要な部分を考慮してください。



モニタリング



このサブシステムは、トランザクションに関するさまざまな入力データ（独自および他のシステム、ニュースなど）を取得し、分析に便利な形式（テーブル、グラフ、統計）でこのすべての情報を提供します。 ニュースは非常に重要な部分であると言わざるを得ません。市場、特に株式市場と同じくらいモバイルであり、特にその完全なコンピュータ化のためにモバイルは即座に反応するからです。 たとえば、リーダーシップの変更など、企業のウェブサイトにプレスリリースが掲載された瞬間から、このプレスリリースによって市場での企業の証券価格の積極的な動きが始まるまで数秒かかります。 そして、より速くニュースを受け取った人、彼は反応し、その結果-稼ぐことができました。



ニュースシステムについて言及する価値があります。 それらはいくつかのレベルで構成されています。 これまたはその事実を受け取った後、最初のレベルで、ニュースはいくつかの単語の1つの文に圧縮され、すぐに公開されます。 これにより、ニュースプロバイダーはニュースを非常にすばやく（わずか数分で）リリースでき、消費者はニュース（たとえば、トレーダー）がニュースラインで新しい行を見て（段落全体を読まない）、本質を把握し、必要に応じてアクションを実行できます。 たとえば、GAZPROMはCEOを解任したか、Sberbankは破産を宣言しました。 さらに、ニュースは次のレベルに渡され、より詳細な説明が記載された段落または2つが既に記述されています。 ここから1、2時間は既に経過しています。 元のニュースが爆弾だった場合、多くの人がこの説明を待ちますが、そうでない場合、おそらく誰もそれを読まないでしょう。 さて、すでに第3レベルで詳細分析を書くことができ、たとえば、特定の市場、証券などへのニュースの影響に関する専門家の意見を含めることができます。



ポジション会計システムと取引履歴



このメカニズムにより、トレーダーは日々の活動を追跡できます。 実際、これは彼の行動の日記です。非常に多く購入し、何かに売りました。 このシステムの見た目には実用的で単純なことは、実際には誤りです。 ポジションと取引履歴を記録するシステムにより、トレーダーは、他の市場参加者と比較して、どのくらい、どれだけ売ったか、買ったのか、その価格は即座に確認できます。 ちなみに、多くのトランザクションはまだ電話でのみ実行されており、そのようなトランザクションはどこかに修正する必要があります。 そして重要なことに、このレベルでは、トレーダーの制限が制御されます。 たとえば、特定のトレーダーは、そのような金額を超えない日に取引することはできません、またはそのような証券の負債をそれ以上に借金することはできません、またはトレーダーがそのような金額を超える取引を入力した場合、上司の許可が要求され、など 制御システムは非常に高度なものにできます。



アルゴリズム取引もこのレベルで機能します。 市場からの情報に基づいて、個人ではなく特別なプログラム（または個人であっても、個々のトランザクションを管理せず、この作業を行う特別なスクリプトを起動する人）が売買を決定します。 プログラムはこれを非常に迅速に、人よりもはるかに速く実行できます。明白なロジックを使用すると、何かを売るとすぐに安くなり、購入すると価格が上がり、市場で価格を管理し、それによって非常に可能性の高い予測を行うことができます価格の動きについて、その結果、時間通りに（=収益性のある）売買する。



トランザクションシステム



通常、トレーダーの仕事は取引に同意することです。 電子的または電話で-それは重要ではありません。 トランザクションは通常、市場の閉鎖後に発生します。 そして、これがミドルオフィスの役割です。 この段階で、トレーダーのアクティビティの追加チェックも実行できます。



そこに何かを違法に取引したトレーダーに関するさまざまな話は、通常、トレーダーがチェックなしで取引を行うことができた、またはこれらのチェックを回避する方法を見つけたという事実に関連しています。



ここでのプロセスの自動化とは、通常、取引に関する情報を請負業者に電子的に送信し、請負業者から回答を受け取り、請負業者と物理的に決済を行う金融部門のデータを準備することです。



匿名取引



最初の取引方法は、トレーダーが特定のカウンターパーティ（たとえば、通常のパートナー）から直接購入（または販売）する場合です。



買い手と売り手に通知する別の方法は、匿名取引です。 この場合、買い手（または売り手）は、価格のみに関心があるため、取引相手が誰であるかは関係ありません。 このような匿名取引を自動化するための特別なメカニズムがあります。



匿名取引のための電子市場デバイス



電子市場は通常次のように配置されます。 市場の深さなどがあります。 翻訳の「市場の深さ」はあまり好きではないので、この用語は翻訳せずに残しておきます。 「市場の深さ」は、選択したセキュリティの市場における最高の需要と供給の表です。 次のようになります。 



 入札オファー
 --------- + ----------
 C1 x K1 |  Ts5 x K5
 --------- + ----------
 C2 x K2 |  Ts6 x K6
 --------- + ----------
 C3 x K3 |  Ts7 x K7
 --------- + ----------
 ...
 --------- + ----------
 C4 x K4 |  Ts8 x K8
 --------- + ----------


[入札]列は、人々が購入したいものの価格と量です（現在の需要リスト）。 この列は価格の降順で並べ替えられます： 



  C1> C2> C3> ...> C4


つまり、この列の物理的な意味は、潜在的な売り手の観点から特定の証券の最高（=最高）価格（および可能なボリューム）を表示することです。



セル「Ts1 x K1」がまだなく、入札列の上部に「Ts2 x K2」があるとします。 私は、市場参加者として、「私は、「T1」の価格で「K1」の金額で証券Nを買いたい」と言います。 私の価格は価格「C2」よりも高いため、私のアプリケーションは列の一番上にあります。



「オファー」（または「質問」と呼ばれることもあります）列は、人々が販売したいものの価格と量（現在のオファーのリスト）です。 この列は価格の昇順でソートされています。 



  C5 <C6 <C7 <... <C8


この列の物理的な意味は、潜在的な買い手に関して、特定の証券の最高（=最低）価格（およびボリューム）を表示することです。



その後、楽しみが始まります。 ご存知のように、需要は供給を満たし、逆も同様です。 したがって、この表は、需要と供給を比較するためのメカニズムです。



市場に来て、「Ts1」の価格と「K1」の金額で証券Nを購入したい場合、システムは証券Nの表「市場の深さ」を取得し、「オファー」の列を確認します。



一部の価格が私の価格と一致する場合（たとえば、「Ts6」）、システムは自動的にアプリケーションを満たし、「Ts1」（=「Ts6」）の価格で「K1」の量で適切な量の証券Nを販売します。 セル「Ts1 x K1」は、需要が満たされているため、需要列「入札」に表示されません。 要求された数量「K1」が「K6」に等しい場合、「Ts6 x K6」の行がオファー「Offer」の列から削除されます。 私の量「K1」が「K6」より少ない場合、行「C6 x K6」は列に残りますが、「K6」から「K1」が単純に減算されます。 私の数量「K1」が「K6」よりも大きい場合、「Ts6 x K6」という行が「Offer」列から削除され（このオファーは完全に満たされます）、「Bid」列に「Ts1 x K1」という行が表示されます。 「K1」から「K6」を引いた値、つまり「Ts1 x K1-K6」が表示されます。



同様に、販売中。 市場に来て、「証券Cの価格と金額K6で証券Nを販売したい」と言うと、システムは証券Nの表「市場の深さ」を取得し、「入札」列を調べます。



一部の価格が私の価格と一致する場合（たとえば、「Ts2」）、システムは自動的にアプリケーションを満たし、「Ts6」（=「Ts2」）の価格で「K6」の量で適切な量の証券Nを購入します。 セル「Ts6 x K6」は、私の申し出が満たされているため、需要列「オファー」には表示されません。 要求された数量「K6」が「K2」の場合、「C2 x K2」の行が「入札」文の列から削除されます。 「K6」の量が「K2」より少ない場合、列「C2 x K2」は列に残りますが、「K6」は「K2」から単純に減算されます。 私の量「K6」が「K2」よりも大きい場合、行「Ts2 x K2」が列「入札」から削除され（この要求は完全に満たされます）、列「Offer」に行「Ts6 x K6」が表示されます。 「K6」は「K2」を除いて表示されます。つまり、「C6 x K6-K2」です。



実際、Market Depthテーブルには、満たされていない需要と供給のリストが表示されます。 需要のためにペアの供給が見つかるとすぐに、またはその逆の場合、それらの間で取引が行われ、このペアがテーブルから削除されます（上記のように、完全な一致がなかった場合、取引の後に当事者の1人がマーケットデプステーブルに残ることがあります数量）。



通常、市場の深さには、需要と供給の平均10〜100の価格レベルが表示されます。 これ以上意味がありません。



さらに、この表の各セルは、価格でアプリケーションのリストで開くことができます。 これはどういう意味ですか？ たとえば、セル「Ts1 x K1」は価格集約インジケーターにすることができます。つまり、数量「K1」は唯一の買い手のオファーではありませんが、多く（複数の買い手は「Ts1」の価格で買いたい場合があります）販売中（価格は同じで、システムは一般的に匿名）です。デフォルトでは、この詳細は表示されない場合があります。



マーケットデプステーブルに従って需要と供給を一致させるメカニズムは匿名取引の基礎であり、いわゆるアルゴリズム取引はマーケットデプステーブルデータの分析に基づく自動取引ロジックのプログラミングに基づいて構築できます。



アルゴリズム取引の場合、システムはプログラムインターフェース（API）を提供します。これは、オプションとして、Luaなどのスクリプト言語であり、システムで市場深度テーブルを表示したり、売買注文を出すための一連の機能を備えています。 ロジックはスクリプト作成者によってプログラムされます。



興味深いことに、トレーダーはスクリプトを極秘に保管している場合があり、技術的な不具合（スクリプトが正常に機能しないなど）の場合でも、技術的な問題を解決する必要があるものの、技術的なサービスの提示を拒否します。
