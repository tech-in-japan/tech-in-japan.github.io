JPoint 2015 Javaカンファレンスのお知らせ

地球の情報フィールドがそれに依存していることを考慮してプログラムを作成してください。

ニキータ・ジグルダ



文字通り3か月後の4月20日、モスクワのラディソンスラビャンスカヤホテルのワールドトレードセンターで、3回目のJPoint Java会議を開催します。 最初のJPoint会議は、ベンダーから完全に独立しているという意味でロシアにとってユニークなイベントでした。 昨年の2番目は、かなりハードコアであるという事実によって区別されました。







JPoint 2015は何を準備していますか？ カットの下の詳細。



ジャンルの古典

主なものから始めます。 まず、JPoint 2015は引き続き独立した筋金入りの Java会議です。 個人的に（世界の認識のレベルで）、現在ITイベントで支配しているトレンドは、私には異質です：スタートアップ、20分間のレポート、ハッカソンなど。したがって、これは私たちについてではありません。 私たちは保守的であり、この筋金入りの保守主義では非常にユニークであることがわかります。 何が成功し、悪用されます。 ハッカソン、稲妻討論、カンファレンス「すべてをカバーする」、その他のさまざまな面白いことの周りにあるほど、私たちはより自信を持つようになります。 おしゃれじゃないですか？ さて、ファッションをクラシックに戻します



第二に、JPoint 2015は、私たちがこの言葉を理解するのに慣れている正確な意味での古典的な形式の専門家会議です。 詳細を見ると、3つまたは4つのストリームがあり、それぞれ6つのレポートがあります。 レポートの期間は40〜50分です。 日中は1時間の昼食です。 WTCでのディナーは高価であることに注意する価値があるため、ファンが節約できる特別な「ランチフリー」チケットのカテゴリを導入しました。







第三に、JPoint 2015はさらに大きくなります。 今年、ロシアや近隣諸国からロシア語を話す1,000人以上の Java専門家を集める予定です。 ブースエリアを大幅に拡大して、従業員を探している企業や展示会に参加するJava開発者向けの製品を販売している企業だけでなく、他の企業も引き付ける予定です。 特に、Highload ++の例に従って、専門家とスタンドを編成します。 このゾーンでは、誰もが最もホットなトピック（Java 9、パフォーマンス、Spring、ORM、Java EEなど）について講演者や他の専門家と話をすることができます。同じトピックで、予定どおり同時に別の部屋で開催される一連のラウンドテーブルを開催しますレポート。



スピーカー


JPoint 2014の最高のスピーカーを維持し、新しい顔をいくつか追加するように最善を尽くします。 現時点では、5つのレポートを発表しています。

  • Oracle Java Performance TeamのAlexey Shipilevがjava.lang.Stringの本質について語ります。 （ジョーカーからのこのレポートの英語版： tyts ）
  • 同じチームのSergey Kuksenkoが、ハードウェアカウンターに関する最新のレポートを作成します。これは、 ロシアの永遠の質問「何をすべきか」に答えることができるツールです。
  • OdnoklassnikiのAndrey Panginが 、デバッグツール（キーワード：JVMTI、動的アタッチメカニズム、Serviceability Agent、Instrumentation API）に関する彼の講演の改訂版を語ります。 これはジョーカーとの報告の繰り返しであるため、サンクトペテルブルクですでにこの報告を見た人は、良心を持って他の部屋に行くことができます。
  • 伝説、最初のJUG.ruのリーダーであり創設者であるJacob Sirotkinが 、JavaでのPL / SQL-hellの書き換えについて語る
  • タリンの会社ZeroTurnaround（JRebel、XRebel）の親友であるAnton Arkhipovが、バイトコードの変更について説明します。








特に、キエフのニコライ・アリメンコフはメッセージングとテストに関するトピックから選択し、 グレブ・スミルノフ （現在はタルトゥ！）、ホットスポットで他に何ができるかを考えています、 バルーク・サドグルスキーとエフゲニーイスラエルのボリソフは、Groovy Puzzlersの第2部を準備しています。これは、10月のジョーカーからのすばらしい報告の続きです 。





新人


講演者として真剣に考えている人のうち、クイックコレクションのマスターであるHigher Frequence Trading（Koloboke、OpenHFT）、有名なPeter Loriの同僚であるRoma Leventova、および最近カリーニングラードに移住したAcademgorodokの全員に知られているMax Ivanov 。 Romaはメモリ内のキーと値のストアについて話をする準備ができており、Maxはエンタープライズテクノロジーの回顧を行う予定です。



あなたの報告が必要です！


会議は生物です。 その発展と成長のために、新しいトピック、新しい顔、新しいアイデアを積極的に探しています。 JPoint 2015の重要な機能は、もちろんそのプログラムです。 同じDevoxx 、 RigaDevDay、そして私たちの親友であるJEEConfとは異なり、「近隣のトピックに関する」レポートを追加することで、Javascript、Android、またはBigDataに関するトピックがプログラムに表示されるのは、直接接続されている場合のみJavaで。 多くの皆さんがJavaから切り離されていてもこれらのトピックに興味があることを理解していますが、何ができるのでしょうか。



哲学の瞬間
この理由から、私はジグルダから引用文を作成しました（ああ、なんてことだ、私は本当にそれを書いたのですか？！） 。 Javaに焦点を当てたまま、この会議のための素晴らしいプログラムを作りたいと思っています。 上で書いたように、この道は多くの人にとって危険なようです。なぜなら私たちはトレンドに逆らっているためです。 トレンド-これは地球の情報フィールドであり、これが外部から私たちに圧力をかけるものです。 そして、彼らが私たちを押すとき、私たちは本当にそれが好きではありません。 そして、トレンドを打ち破り、根源に戻ることによって、このまさに情報分野を変えていきます。
