はじめに



Windows 8についての最初の噂は、2011年初頭に登場しました。 開発者が新しいプラットフォームがどのようなものになるのだろうと考え始めたとき、多くの憶測がインターネットをサーフィンしていた。 新しいプラットフォームは.NET Frameworkをサポートせず、C ++またはHTML 5とJavaScriptに完全に基づいており、既存のソフトウェアは動作しないといううわさがありました。 初期のリリースとスクリーンショットはTwitterに掲載されましたが、これは一般的な疑念に追加されただけです。 最後に、MicrosoftのWindows社長であるSteven Sinofskyは、2011年9月13日にWindows 8を世界に紹介しました。



私は、システムディストリビューションをダウンロードした最初のせっかちなプログラマーの一人でした。 仮想マシンにWindows 8をインストールすると、.NET Frameworkがサポートされ、すべてが正常に機能していることがすぐにわかりました。 新しいプラットフォームでは、Silverlightアプリケーションが機能しました。 C＃およびXAML言語は、「Metroスタイル」で新しいアプリケーションを開発するためのツールの中に存在していました（WindowsのRTMバージョンの「Metro」の名前は「Windows Store」に変更されました）。 カリフォルニアで開催されたWindows 8カンファレンスには出席できませんでしたが、レポートはすぐにインターネットに掲載され、毎晩、毎朝、いつも自由な時間にレポートを見ることができました。



Windows 8は、Windowsランタイムを提供します。Windowsランタイムは、Windowsを実行しているコンピューターでこれまで利用できた機能を備えたアプリケーションを作成するための新しいプラットフォームです。 私は一日中アプリケーションを構築してきましたが、SilverligntとWindows Presentation Foundation（WPF）を使用したC＃とXAMLの知識が新しいランタイムに完全に適用できることを賞賛しました。 このプラットフォームの新しいコンポーネントセットにより、ユーザーとのタッチインタラクションに焦点を当てた機能アプリケーションの開発がこれまで以上に簡単になります。 最後に、すぐに、私の出版社であるDesigning Silverlight Business Applicationsに連絡し、Windows 8に関する本を書きたいと言いました。



幸いなことに、私はマイクロソフトの早期学習プログラムに参加しました。 私が働いているコンサルティング会社のWintellectは、Windows 8用のアプリケーションの作成方法を学びたい新しい開発者向けのワークショップやセミナーを実施するように提案されました。最終リリースの一部。 画面上のオブジェクトのタッチコントロールの例を作成し、フォーマットされたコンテンツをアプリケーション間で転送し、ホーム画面で「ライブ」インタラクティブタイルを使用してすばやく表示する機能を使用している間、新しいシステムがますます気に入っていました。

この本の一部として、開発者がWindows 8用のアプリケーションを作成するのが好きなトップ10の理由に関する記事を書きました。

完全な記事はwww.informit.com/articles/article.aspx?p=1853667で見つけることができます。



詳細に触れない場合、これらは、私の意見では、新しいプラットフォームでの作業を楽しむことができる主な理由です。

• さまざまなプログラミング言語のサポート 。 Windows 8アプリケーションは、VB、C＃、C ++、およびXAMLで作成するか、HTML5およびJavaScriptを含む一連のテクノロジーを使用できます。

• XAML。 XAMLのパワーと柔軟性に精通し、SilverligntまたはWPFのアプリケーションの作成に以前に取り組んだことのある開発者は、XAMLを使用してWindows 8のアプリケーションで作業するときに自信を感じるでしょう。

• HTML5 。 マークアップ記述言語の1つとしてのHTML5の広範なサポートは、タブレットコンピューターのプログラミングに切り替えるWebプログラマーにとって魅力的です。タブレットコンピューターでは、ユーザーとの感覚的な相互作用に重点が置かれています。 この本は、主にC＃とXAMLの操作に取り組んでいますが。

• Windowsランタイム環境（WinRT）。 WinRTには、わずか数行のコードで複雑な問題を解決できる多くのコントロール、コンポーネント、クラス、およびメソッドが含まれています。

•契約。 「契約」のシステムは、アプリケーションとユーザー間のデータ共有および相互作用の新しいレベルの組織を表します。

• 非同期操作のサポート。 await



およびasync



コマンドのサポートにより、マルチスレッドコードの開発がこれまで以上に簡単になりました。

• 感覚的相互作用 。 Windows 8はタッチ入力に焦点を当てています。 タッチイベントと操作を処理するとき、そのような入力はデフォルトで利用可能なすべてのコントロールとAPIをサポートします。

• オプション。 コントラクトを介してパラメーターを設定することにより、開発者はアプリケーションをセットアップするための非常に一貫した使い慣れたメカニズムをエンドユーザーに提供できます。

• 移動プロファイル 。 クラウドを介して異なるコンピューター上のWindows 8の状態を同期するコードを作成することは、簡単で簡単になりました。 （1行のコードでファイルを共有できます）。

• アイコン 。 Windows 8には、一貫したアプリケーションコマンドインターフェイスを作成するために使用できる一連の定義済みアイコンが導入されています。



混乱を避けるために、この本では、Windows 8用に特別に作成された新しい種類のプログラムを「Windows 8のアプリケーション」と呼びます。 新しいアプリケーションを作成するために設計されたVisual Studio 2012テンプレートは、「Windowsストア」という名前でグループ化されます。 これらのアプリケーションはWindowsストアを介して配布できますが、ストアおよび従来のデスクトップアプリケーションを介して配布できます。 したがって、Visual Studio 2012テンプレートに言及する場合、または新しいタイプのアプリケーションを従来のデスクトップアプリケーションと比較する場合にのみ、「Windowsストア」という用語を使用します。 他のすべての場合、これらのアプリケーションへの参照は「Windows 8のアプリケーション」として見つかります。



これらの10の理由-これは、新しいプラットフォームの表面的なレビューにすぎません。 Windows 8は以前のリリースのWindowsとは異なるため、このオペレーティングシステムの認識は異なるはずです。 新しいインターフェイスに適応する必要があります。これは主にユーザーとの感覚的な相互作用に焦点を合わせていますが、マウスとキーボードを使用する機能もサポートしています。 システムのアンマネージコンポーネントを透過的に呼び出すコードを使用して、新しいコントロールセットやその他のオブジェクトを操作できます。 あなたが手にする本の主な目的は、新しい知識分野を習得する道にあなたを素早く導くことです。そうすることで、C＃とXAMLの以前の知識を使用して素晴らしいモダンアプリケーションの作成を開始できます。



この本は何についてですか



この本の目的は、C＃、XAML、Windowsランタイム、および.NETプラットフォームを使用してWindows 8アプリケーションを作成する方法を説明することです。 開発経験があると思います。 C＃とXAMLに関連する基本的なトピックを展開して、新しいアプリケーションの開発に特徴的な機能に焦点を当てようとしています。 Windows 8プラットフォームだけに関連するものではない、C＃またはXAMLを使用するより複雑な方法については、他の書籍、記事、またはインターネットリソースを参照して、必要に応じてこれらの手法を引き続き学習できるようにします。



既存のアプリケーションを新しいプラットフォームに移行したい経験豊富な開発者と、Windows 8アプリケーションをゼロから作成したい初心者の両方の、あらゆるレベルの開発者が、この本で必要なものすべてを見つけるでしょう。 ソフトウェアプロジェクトの作成からWindowsストアでの公開まで、アプリケーションのライフサイクル全体を網羅しています。



この本の使い方



この本は、C＃とXAMLを使用してWindows 8用のアプリケーションを作成する方法を示すことを目的としています。 各章は、最初にターゲットプラットフォームの基本的な機能を理解してから、知識に基づいてアプリケーションを作成できるように作成されています。 この本は、システムの機能を示すプログラムのテキストの例と、それらを使用するための推奨事項を提供します。 多くの章は以前のものに基づいています。 これにより、典型的なWindows 8アプリケーションを構成するすべてのコンポーネントについて徐々に話をすることができます。



すべての章は同様の方法で構成されています。 それらは、トピックの概要と、トピック内で利用可能なオプションの概要から始まります。 以下は、説明されているテクノロジーのアプリケーションを示すコード例とチュートリアルです。 コード例の詳細を説明します。この章の最後で要約を行い、最も重要な点に注意を払ってください。



あなたが持っている知識やスキルに関係なく、この本を最初から最後まで読んでいただければ幸いです。 各章を読むと、その資料をよりよく理解し始めていることに気付くでしょう。また、新しい概念がお互いを強化し、結び付けます。 本を完全に読んだ後は、特定の問題を明確にする必要があるときにいつでも個々の章を参照しながら、参考として使用できます。
