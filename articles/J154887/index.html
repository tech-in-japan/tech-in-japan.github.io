アスタリスクへの遅延コール

問題は、オフィスのPanasonic PBXと同じくらい古いものです。 彼女が、もちろん、まだそこにいるなら。

アスタリスクを実装する場合、経験豊富な労働者は常に、以前使用していたPBX機能の以前のセットを残したいと考えています。 そして、パナソニックのキラー機能の1つは「6」でした。これは遅延コールです。



「私は彼を呼んでいるが、彼は「忙しい」！ ドク、それから私は「6」ホップを振って電話を取り、彼がどのように彼を解放し、ATSkaが私に電話をかけるかを教えてくれます。 「奇跡の機械はできますか？」



ああ、そしてアスタリスクは箱から出して6を本当にすることができますか？ 次は、この質問に対する少数の物語を含む詳細な肯定的な答えです。







それで最近、アスタリスククックブックに出会いました 。 同僚と共有します。 彼と一緒に、私たちはすでに多くのレシピを私たち自身で長い間マスターしていましたが、本は小さく、それで何が新しくて便利ですか？



しかし、週末の後、同僚が不思議な笑みを浮かべて、「CCSS！」と言いました。



Asterisk 1.8では、開発者はCall Completion Supplementary Serviceを実装しました。 本をめくったとき、私は注意を払わなかった。 おそらく、ISDN用のいくつかのチップの実装はこの背後に隠れていると思いました。



ただし、この大きな名前の背後にはきちんとしたアーキテクチャがあり、このPDFで見ることができます（ISDNについては本当にあります:)。



そして、私は公式文書から例のアプリケーションに渡します 



  sip.conf [Mark] context=phone_calls cc_agent_policy=generic cc_monitor_policy=generic [Richard] context=phone_calls cc_agent_policy=generic cc_monitor_policy=generic  dialplan' [phone_calls] exten => 1000,1,Dial(SIP/Mark,20) exten => 1000,n,Hangup exten => 2000,1,Dial(SIP/Richard,20) exten => 2000,n,Hangup exten => 30,1,CallCompletionRequest exten => 30,n,Hangup exten => 31,1,CallCompletionCancel exten => 31,n,Hangup




そしてそれだけです！ ほとんど:)フォーラムであまり発明されていない自転車（ リンク ）



ただし、注意点が1つあります。 パナソニックでは、「ビジー」ビープ音が聞こえた後に「6」を押しましたが、電話を切る前に電話を切り、PBXからの呼び出しを待ち始めました。



アスタリスクの場合、状況は次のとおりです。電話をかけ、「話中」と聞き、電話を取り、話中の加入者（例では「30」）の呼び出し順序コードをダイヤルし、電話を切り、アスタリスクからの呼び出しを待ちました。 ダイヤルプランの例では、「忙しい加入者へのコールが注文されました」のような十分なメッセージがありません。



キャンセルコード（「31」の例）は、特に負担をかけることはないと思います。 ただし、忙しい加入者に電話するために注文をキャンセルすることはできます。



さらに、便利なオプションから、それはごちそうのためになります：

cc_max_agents-ごちそうを見る人の最大数

cc_max_monitors-フォローできる人の最大数

cc_recall_timer-サービスを注文するときにアスタリスクからあなたに電話するときの呼び出し時間



リアルタイムを使用する場合、一般に、データベースにフィールドを追加することですべてがきちんと解決されます。



CLIでCCSSを使用する現在の状態を監視するには：cc report status



ccss.conf構成ファイルで、追加の設定を指定できます。フォロワーの最大数max_cc_agents。 すべてのパラメーターと可能な値の説明があります。



また、電話に2番目の呼び出しを受け入れる機会がある場合、電話はビジーになりません（タスクの条件に応じて必要になります）。 チャネルをごちそうに制限するには、デバイスでコール制限を使用したり、コール待機を無効にしたりできます。



一般的に、ミカリーチはアスタリスクに6が含まれることに非常に満足し、拾い上げて、一度に防衛工場で電気自動車を切断した方法を伝えました。
