Windows 7 Betaリリースノート

以下は、「リリースノート：Windows 7のこのベータリリースでの重要な問題」というドキュメントの（キャップ​​と結論なしの）オフハンド翻訳です。 簡単に紹介するために、あなたに何が起こるかを考えてください:) <br />

それが重宝することを願っています。 不明な点がある場合（ネットワークの問題など）に誤りがある場合は、元の文書を参照してください-正しいバージョンに鼻を突っ込んでください。







アクセシビリティ機能



Microsoft®ナレーターは、コマンドライン情報を正しく読み取りません。 これは、サードパーティのスクリーンリーダーにも適用される場合があります。



これを修正するには、常に管理者権限でコマンドプロンプトを実行します。







Apple iTunes



このケースは、Windows 7の64ビットエディションにのみ適用されます。



Apple iTunesをダウンロードしてインストールしようとすると、64ビット版ではなく32ビット版のアプリケーションがインストールされます。 これは、iPhoneとの同期機能を妨害します。



これを回避するには、次のアドレスにある64ビットバージョンをインストールします。http： //go.microsoft.com/fwlink/？LinkID = 139389







バックアップ

  • 最新の復元ウィザードを使用して、Windows 7コンピューターでバックアップされたファイルをWindowsVista®を実行しているコンピューターに復元しようとすると、復元は失敗します。



    これを回避するには、バックアップファイルがあるディレクトリに、コンピューターと同じ名前のディレクトリにあるMediaid.binファイルを配置します。 Mediaid.binファイルを上位レベルに移動して、名前がコンピューターの名前と一致するディレクトリと同じレベルになり、その中にないようにします。

  • システムイメージのバックアップをリカバリポイントとして使用しようとすると、システムの復元はユーザーの操作への応答を停止します。



    これを回避するには、システムイメージのバックアップをリカバリポイントとして選択しないでください。 システムイメージのバックアップを使用していないことを確認するには、システムの復元ウィザードで「 タイプ 」列をチェックし、バックアップのタイプが「 バックアップ 」と異なることを確認します。



    これが既に発生している場合は、システムの復元を再起動し、システムバックアップではない別の復旧ポイントからの復元を試みます。

  • Windowsバックアップウィザードを使用してシステムイメージをバックアップする場合、このバックアップを使用して他のハードウェアに復元することはできません。 同様に、バックアップの作成後にディスクレイアウトを変更した場合、バックアップイメージからシステムを復元することもできません。



    これを回避するには、Windowsバックアップウィザードの代わりに、「 システムイメージの作成 」コントロールパネルの「バックアップと復元」（Windowsのエディションで利用可能な場合）を使用して、システムイメージのバックアップを作成します。

    これが既に行われている場合、最後のバックアップのために仮想ハードディスクをマウントし、それを使用してユーザーデータを取得しようとする以外に方法はありません。






ドライブ暗号化（BitLockerドライブ暗号化）



BitLockerドライブ暗号化は、exFAT（拡張ファイルアロケーションテーブル）ファイルシステムを使用するドライブではサポートされていません。 代わりに、FATおよびFAT32ドライブでBitLockerドライブ暗号化を使用できます。







直接アクセス



Windows 7の一部のエディションでは、DirectAccessの名前解決ポリシーテーブルを設定すると、コンピューターがユーザーの操作や繰り返しの再起動に反応しなくなる場合があります。



これが既に発生している場合、ユーザーはネットワークモードなしでセーフモードでコンピューターを再起動し、Regedit.exeを実行して指定されたレジストリキーの名前解決ポリシーテーブルを削除することにより、システムを復元できます。



HKLM \ソフトウェア\ポリシー\ Microsoft \ Windows NT \ DNSClient \ DnsPolicyConfig



その後、システム管理者はグループポリシーに必要な変更を加えて、システム設定がユーザーの変更と競合しないようにする必要があります。







Internet Explorer 8

  • Adobe®Reader®アドインを使用してブラウザーからPDFを直接印刷しようとすると、操作は失敗します。



    これを回避するには、PDFドキュメントをディスクに保存し、Adobe Readerアプリケーション、Adobe Acrobat、または別のPDFリーダーを使用して開いて印刷します。

  • ニュースフィード（RSSやAtomなど）は、バックグラウンドで正しく更新できません。



    修正するには、コマンドプロンプトを開き、次のコマンドを実行します。



    msfeedssync無効



    for / f "tokens = 3"％i in（ 'reg query hkcu \ software \ microsoft \ feeds / v synctask'）do schtasks / delete / tn "％i" （エラーが発生した場合は、無視して安全に続行できます）次のコマンドの実行）。



    reg delete hkcu \ software \ microsoft \ feeds / v synctask



    msfeedssyncを有効にする



    その後、次のコマンドを実行して、問題が修正されたことを確認できます。



    for / f "tokens = 3"％i in（ 'reg query hkcu \ software \ microsoft \ feeds / v synctask'）do schtasks / run / tn "％i"






言語パック



このベータリリースの一部のエディションでは、Windows Updateでダウンロード可能な言語パックが表示されません。



編集ボードにこの機能がある場合、現在変更する方法はありません。







モバイルブロードバンドソフトウェア



現在、モバイルデバイス用のすべてのソフトウェアが正しくインストールされているわけではなく、一部のデバイスは検出されないか、インターネットに接続されない場合があります。



このベータリリースを使用してインターネットに接続する機能、および他のデバイスとソフトウェアの既知の問題を解決する方法がサポートされているデバイスの現在のリストを表示するには、 http：//go.microsoft.com/fwlinkで入手できます。 /？LinkID = 139390 。







MP3ファイル



MP3ファイルを（手動または自動で）Windows Media PlayerライブラリまたはWindows Media Centerライブラリに追加した場合、またはこれらのファイルのメタデータがWindowsエクスプローラーを使用して変更された場合、ファイルの先頭から数秒のオーディオデータを削除できます。 この問題は、ファイルがインポートまたは変更される前にかなりのスペースを占有するサムネイルまたはその他のメタデータを含む場合に発生します。



これを回避するには、次の手順を実行します。



MP3ファイルを保護するには
  1. このベータリリースをインストールする前に、リムーバブルメディアや共有ネットワークリソースなど、オペレーティングシステムがアクセスできるすべてのMP3ファイルをバックアップします。
  2. すべてのMP3ファイルに読み取り専用属性が設定されていることを確認してください。 Windowsエクスプローラーでこれを確認するには、ファイルを見つけ、マウスの右ボタンをクリックして呼び出されるプロパティで、[ 全般 ]タブの[ 読み取り専用]チェックボックスをオンにします。
  3. 次に、Windows 7のこのベータリリースをインストールします。
  4. 記事を読んで、 http：//go.microsoft.com/fwlink/？LinkID = 139391で入手可能な更新をインストールします 。
  5. 更新プログラムのインストール後、MP3ファイルのレコード読み取り設定をお好みの設定に安全に変更できます。
更新プログラムをインストールしない場合は、段落2で説明されているように、オペレーティングシステムがアクセスできるすべてのMP3ファイルに読み取り専用属性を設定することにより、このような問題を回避できます。







電源オプション



Windows 7の一部のエディションでは、コンピューターがデフォルトでスリープモードの終了時にパスワードを要求するように構成されていない場合があります。



修正するには、 [電源オプション]コントロールパネルの[再開時にパスワードを要求する]設定を使用します。







ルーティングとリモートアクセス

  • Connection Managerでは64ビットプロファイルのみがサポートされています。 指定されたプロファイルは、32ビットオペレーティングシステムでは機能しません。 接続マネージャー管理キットを使用して、32番目のクライアントコンピューターに32番目のプロファイルを作成します。

  • 2番目のバージョンのインターネットキー交換を使用してVPN（仮想プライベートネットワーク）経由で接続しているユーザーは、コンピューターの再起動時に、あるネットワークから別のネットワークにVPN接続を簡単に切り替えることができません。 これを修正するには、ネットワークを切り替える前に、ユーザーはルーティングとリモートアクセス（RRAS）モビリティマネージャータスクを手動で実行する必要があります。

  • [すべてのネットワークの表示]ウィンドウに表示されるダイヤルアップマルチリンクVPN接続の状態が正確ではありません。 この状態を確認するには、指定されたウィンドウの代わりに、ネットワーク接続コントロールパネル（ncpa.cpl）を使用します。

  • パスワード認証プロトコル（PAP）、チャレンジハンドシェイク認証プロトコル（CHAP）またはMicrosoftチャレンジハンドシェイク認証プロトコルバージョン2（MS-CHAPv2）のユーザー名認証パスワードに国別アルファベット文字を使用する場合、Windows用ネットワークポリシーサーバー（NPS） Server®2008 R2は証明書を認証できません。 これを回避するには、PAP、CHAP、およびMS-CHAPv2 のユーザー名に国別アルファベットを使用しないでください。 （Windows Server 2008 R2にNPSを使用している場合、認証は機能します）。






データストレージデバイス



システムをこのベータリリースにアップグレードすると、iSCSIターゲットの「お気に入り」接続は保存されず、構成情報も保存されないため、iSCSIに接続されたドライブにはアクセスできなくなります。



これを回避するには、このベータリリースにアップグレードする前にiSCSI接続ドライブの構成を記述し、アップグレードプロセスの完了後にそれらを再追加します。 CHAPセキュリティ（チャレンジハンドシェイク認証プロトコル）との接続を使用する場合、CHAPキーを含むこれらの設定を書き留めます。



この場合は、iSCSIターゲットに再接続し、「お気に入り」接続としてマークします。







システムパーティション



システムは、システムパーティションのフォーマットを妨げません。 システムパーティションをフォーマットできないことを示すエラーが表示されますが、操作は続行され、データが失われ、コンピューターを起動できなくなります。



システムを操作するときは、システムパーティションをフォーマットするための手順を実行していないことを確認してください。 この場合は、Windows回復環境を使用してください。







ビデオドライバー



XDDMドライバーを使用するシステムは、ユーザーアクションへの応答を完全に停止する場合があります。 コンピューターはCTRL + ALT + DELETEに応答せず、電源をオフにして再起動する必要があります。これにより、保存されていないデータが失われる可能性があります。



このエラーを引き起こす可能性のあるケースには次のものがありますが、これらに限定されません。

  • Winsat.exeまたはツールを使用してコンピューターのパフォーマンスを評価および向上させる
  • マルチモニター構成の変更
  • DVDまたはその他のビデオコンテンツを再生する
これを回避するには、XDDMドライバーを使用しないで、適切なWDDMドライバーを見つけてください。 ビデオチップセットにWDDMドライバー（Intel 915など）がない場合は、WDDMドライバーをサポートする標準のVGAドライバーまたはビデオカードドライバーを使用します。



インストール中に問題が発生した場合は、無人インストールまたはXDDMドライバーのアンインストールを避け、インストールが完了するまでVGAドライバーを使用してください。







Windows Media Center

  • Windows Media Centerは、Protected Broadcast Driver Architecture（PBDA）ドライバーを見つけられないため、一部のISDB-TおよびISDB-Sチューナーを認識しません。



    相手先商標製造会社のコンピューターに同梱されているWindows Media Center互換チューナーのみを使用してください。

  • 統合サービスデジタル放送（ISDB）コンテンツは、Windows Media Extenderデバイスでは再生されません。



    現在、エラーの回避策はありません。

  • Windows Media Centerセッションを開始し、キーを押さないと、スクリーンセーバーは起動しません。



    これを回避するには、開始画面で任意の項目を選択します。

  • 録画をスケジュールしていて、Windows Media Centerが再びスリープモードになった場合、ライブTVを視聴しようとしたときにビデオオーバーレイエラーが発生したり、ファイルを保存して録画を削除しようとしたときに使用可能なディスク容量が不足するなど、いくつかの問題が発生する可能性がありますディスク容量が不足しているため。



    これを回避するには、Windows Media Centerの実行中または録画のスケジュール中に、電源オプションを使用してコンピューターの「スリープ」機能を無効にします。



    これが既に発生している場合は、コンピューターを再起動し、Windows Media Centerでテレビ放送を開始します。

  • Windows VistaからWindows 7にアップグレードした場合、Windows Media CenterからDVDを作成することはできません。



    状況を改善するには、Regedit.exeを使用して次のキーをレジストリに追加します。

    SonicMCEBurnEngine、カルチャー=ニュートラル、バージョン= 6.1.0.0、PublicKeyToken = 31bf3856ad364e35 "

  • このベータリリースの前にWindows Media Center TV Pack 2008を搭載したコンピューターまたはWindows 7の初期リリースを搭載したコンピューターをアップグレードした場合、Windows Media Centerのガイド検索を使用してタイトル、キーワード、俳優、またはディレクターを検索しても結果は返されません。



    状況を修正するには、タスクスケジューラを再構成します。

    5. タスクスケジューラを閉じ、数分待ってからガイド検索を使用します。

    これを回避するには、Windows Media Centerを使用する前に、指定された更新プログラムを（Windows Updateを使用するか、指定されたリンクから直接）インストールします。

Windows Mediaサービス2008



このベータリリースでは、Media Servicesストリーミングテクノロジーは使用できません。 Windows Mediaサービス用のリモートサーバー管理ツールが含まれています。



これらのテクノロジがネットワークインフラストラクチャの重要なコンポーネントである場合、Windows Vistaを実行しているコンピューターのオペレーティングシステムをアップグレードしないでください。







Windows Searchのインデックス作成



Windows Search（Windows 7、Windows Vista、またはWindows SearchがインストールされたWindows XP）を実行しているコンピューターを使用し、パスが128文字を超える特別なインデックスの場所を選択した場合、オペレーティングシステムをこのベータ版にアップグレードした後、インデックス作成は開始されませんリリース。 これは、検索の機能と一部のライブラリの外観に影響します。



これを回避するには、現在のオペレーティングシステムのアップデートとしてこのベータリリースをインストールする前に、インデックスをパスの長さが128文字未満の別の場所に移動します。



インストールが既に完了している場合は 、「Indexing Options」コントロールパネルのAdvanced Optionsの Restore Defaultsボタンを使用して、設定をリセットし、インデクサーを再起動します。







無線ネットワーク接続



一部のアダプターおよびプラットフォームでは、次の場合にワイヤレスアダプターがオフになり、接続自体が表示されない場合があります。

  • 休止状態およびスリープモードの後、コンピューターが引き続き動作する場合
  • コンピューターがワイヤレス接続を介して複数のビデオストリームを同時に処理する場合
  • Microsoft Virtual PCがコンピューターにインストールされている
コンピューターにMicrosoft Virtual PCがインストールされている場合、このパッケージをアンインストールすることにより、これを防ぐことができます。 さらに、これが発生した場合はすべて、コンピューターを再起動します。
