Windows 7のエラー

次のシナリオを検討してください。Windows7またはWindows Server 2008 R2を実行しているコンピューターがあるとします。 2テラバイトを超えるハードディスクをシステムに追加します。 このハードドライブ上の特定のボリュームにダンプファイルを保存するようにシステムを構成します。



そして、ここでは、Microsoftによって文書化された不快な「バグ」が予想されます。 アドレス指定の問題により、ダンプファイルはハードドライブのアドレス空間の外側に収まろうとしています。 オペレーティングシステムがクラッシュし、ダンプファイルが保存されます。その結果、指定されたハードドライブの1つ以上のボリュームが論理的に破損し、それらに関するすべての情報にアクセスできなくなります。



Hiberfile.sysファイルが保存されると、ディスクの論理構造がさらに損傷するという問題に対処するためのほぼ同じ状況が発生します（「ハイバネーション」モードが含まれている間）。 当然、オペレーティングシステムがインストールされているボリュームが破損している場合、コンピューターは読み込みを停止します。



間違いは間違いなく非常に深刻です。 このページで詳細を確認できます 。 オープンアクセスパッチはありませんが、必要な場合は、Microsoftサポートからダウンロードするように注文できます。
