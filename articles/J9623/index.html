2007年5月：専門家が話していたこと

レビューITニュースは、原則として、有名人の人生の出来事の記述に採用されたスキームに従って構築することができます。 星、「冒険」、そしてその関係に焦点を合わせています。 今日のケースでは、まず、GoogleとMicrosoftです。 スキャンダル、有名な訴訟、多くの場合同様の問題-著作権。 結婚と離婚、読み取り-ビジネス統合（M＆A）。 したがって、先月、全員が同じかなり日常的なシナリオに従って開発しました。 しかし、私の観点からは、今月のイベントと安全に呼ぶことができるイベントが1つありました。 これは、企業の新製品であるMicrosoft Surfaceの 5月のプレゼンテーションです。 つまり、このレビューでは彼に焦点を当てたいと思います。



秘密の革命、またはマイクロソフトの差し迫った死の噂は、いくぶん誇張されていることが判明しました



もちろん、どんな革命も常に秘密に準備されます。 彼らが郵便、電信、そして市場を手に入れて初めて、彼らは長年話し合ってきた革命が起こったことを発表します...専門家が現れました。 ただし、テクノロジーでは、特に新製品が防衛に関連していない場合、開発は公然の秘密です。 彼らは隅々で彼女について話し、すべての専門的な集まりで話し合います。 そして、結局のところ、彼らは原子爆弾を発明しなかった。 知られていませんが、おそらく情報技術の分野では、Microsoftの新しいプロジェクトはどんな爆弾よりも悪いでしょう。 私はあなたのことは知りませんが、多くの「肥満猫」に愛されていない人がそのような驚きを準備していることを聞いたことはありませんでした。 そして実際、ミラノプロジェクトの舞台への入り口（Microsoft自身がその発案者と呼んでいるように）は、多くの人にとってまったく予想外のものでした。

外部では、Microsoft Surface（ミラノ）は、大きなタッチスクリーンを備えた小さな机に似ています。 真のブレークスルーは、デバイスが人だけでなく、オブジェクトと独立して対話できることです。 主な用途は、店舗からレストランまでさまざまです。 製品の開始コストの見積もりは7〜1万ドルの範囲ですが、Bill Gatesは時間の経過とともにそれを1千ドルに引き下げることを約束しており、その後、彼によれば、ノベルティはすべての家に届きます。



ただし、完全なクーデターの秘密といえば、Microsoft Surfaceを作成するプロセスの緊密さだけでなく、それほど重要ではありません。 私は、何が起こった場合に起こり得るすべての結果の潜在的な未知に興味がありました。 言われたことを簡単に説明しようと思います。



Microsoft Surfaceの概念は、さまざまな方向への突破口を暗黙的に意味します（または意味せず、これが新しいプロジェクトに固有の主な陰謀です）。 そのうちのいくつかをご紹介します。







新しいユーザーインターフェース？ 20年以上前に起こったように、コンピューターシステムとエンドユーザー間の対話のラインインターフェイスがグラフィックに置き換えられたとき、グラフィックの対話方法がタッチシステムに置き換えられたことが判明する場合があります。



再び集団使用？ 集団使用とパーソナルコンピューターは、2つの互換性のない概念のように見えました。 ただし、Microsoft Surfaceで具体化されたアイデアの1つは、複数のユーザーのスマートテーブルを同時に操作できることです。



ハードウェアインターフェイスを構築する他の原則は？ それ以外の場合、たとえば、「テーブル」の表面に置かれたデジタルカメラとシステムの間で相互作用がどのように行われるかは完全にはわかりません。 もちろん、提示された製品については、Microsoft自身が開発したハードウェアにのみ一致すると想定しています。 しかし、これは、Microsoftがコンピューター周辺機器や家電製品のメーカーとどのように協力することを計画しているかについては、すでに完全に別個の、かなり具体的な質問です。



バーコードの使用の拡大？ 「テーブル」がデジタルインターフェースを持たないオブジェクト（たとえばワイン1杯）を認識するためには、このオブジェクトは取引で使用されるバーコードのようなものによって識別される必要があります。 ただし、「新しい」バーコードはアイテムのデザインに違反してはならず、一方では確実に接続され、他方では簡単に交換可能です（たとえば、グラスにワインのブランドを示すため）。



新しいソフトウェアアーキテクチャおよび/またはプログラミングテクノロジですか？ 実際、Microsoftがギャンブルハウス、エリートクラブ、スーパーマーケットに「テーブル」を使用することに限定しない場合、この問題は決定的なものの1つになることがあります。 実際、それはいくつかのサブ質問に分けられます：

•プログラムのアーキテクチャ編成の一般原則について

•「古い」プログラムとMicrosoft Surfaceソフトウェアの互換性について

•ソフトウェアインターフェイスについて

•ソフトウェアとハ​​ードウェアの統合の「密度」について。

•プログラミング方法とツールについて。



Microsoft Surfaceの運命だけでなく、家庭用電化製品およびコンピューターテクノロジーの将来も、これらすべての質問および多くの同様の質問に対する答えに大きく依存しています。 待って、Microsoftが次に何を言うのか、そしてもちろん、時間を...
