Goを探索する理由は何ですか？

みなさんこんにちは！ Keval Patelによる記事（グラフを含む）の翻訳を紹介します。なぜGoを学ぶ必要があるのですか。 この記事には、新参者が学ぶために役立つと思う多くの興味深い事実が含まれています。



PS：この記事は、私の意見では、ほとんどの記事と同様に、著者の経験と意見を反映していますが、これはあなたの意見と一致しない場合があります。

過去数年にわたって、新しいプログラミング言語であるGoまたはGolangが大きく成長しました。 同意して、新しいプログラミング言語のように開発者を狂わせるものは何もありません。 したがって、4〜5か月前にGoの学習を開始しました。これが、この新しい言語を学習する理由について説明できることです。



この記事では、「Hello world !!」の書き方を教えません。 これに関する他の多くの記事があります。 コンピューターのハードウェアとソフトウェアの現状と、Goのような新しい言語が必要な理由を明確にします。 結局のところ、問題がなければ、解決策は必要ありませんよね？



鉄の制限：



ムーアの法則は機能しません。



3.0 GHzのクロック周波数を備えた最初のPentium 4プロセッサは、 2004年にIntelによって導入されました。 今日、私のMacbook Pro 2016のクロック速度は2.9 GHzです。 したがって、約10年間、純粋なコンピューティング能力の進歩はほとんどありませんでした。 下のグラフでは、時間の経過に伴う計算能力の増加の比較グラフを見ることができます。







グラフから、シングルスレッドのパフォーマンスとプロセッサ周波数がほぼ10年間安定していることがわかります。 トランジスタを追加することが良い解決策だと思うなら、あなたは間違っています。 これは、より小さなスケールでオブジェクトの量子特性が現れるようになるためです（トンネリング、[ またはトンネル効果。-およそTransl。 ]など） 。 1ドルで追加できる額は急速に減少し始めています。



したがって、上記の問題を解決するには、



  • メーカーは、プロセッサにコアを追加し始めました。 現在、4コアプロセッサと8コアプロセッサが利用可能です。
  • ハイパースレッディングも開始されました。
  • パフォーマンスを向上させるために、プロセッサにキャッシュを追加しました。


ただし、上記のソリューションにも制限があります。 キャッシュには物理的な制限があるため、プロセッサにさらにキャッシュを追加することはできません。キャッシュが大きいほど遅くなります。 プロセッサにコアを追加することにも代償が伴います。 さらに、これを無期限に行うことはできません。 これらのマルチコアプロセッサはすべて、同時に複数のスレッドを実行でき、これにより、画像のマルチタスク処理が可能になります。 これについては後で説明します。



したがって、ハードウェアの改善に頼ることはできないため、この状況から抜け出す唯一の方法は、より効率的なソフトウェアを使用することです。 しかし、残念ながら、現代のプログラミング言語はそれほど効果的ではありません。



「最新のプロセッサは、ニトロで満たされた楽しい車のように見えます-四分の一マイルのレースでは素晴らしいです。 残念ながら、モンテカルロトラックのような最新のプログラミング言語は、twist余曲折に満ちています。」-David Ungar


Goにはゴルーチンがあります!!



前述のように、ハードウェアメーカーは、パフォーマンスを向上させるために、プロセッサにコアを追加しています。 すべてのデータセンターはこれらのプロセッサーで稼働しており、今後数年でコアの数が増えると予想されます。 さらに、最新のアプリケーションは多くのマイクロサービスを使用して、データベース接続、メッセージキュー、およびキャッシュをサポートしています。 したがって、私たちが開発するソフトウェアとプログラミング言語は、マルチタスク（ 同時実行性-およそTransl。 ）を簡単にサポートする必要があり、コア数の増加に合わせてスケーラブルでなければなりません。



しかし、ほとんどの最新のプログラミング言語（Java、Pythonなど）は、90年代のシングルスレッド開発環境に由来していました。 これらの言語のほとんどはマルチスレッドをサポートしていますが、実際の問題は同時実行、スレッドブロッキング、競合状態、およびデッドロックです。 これらのことから、上記の言語でマルチスレッドアプリケーションを作成することは困難です。



たとえば、Javaで新しいスレッドを作成すると、メモリの使用効率が低下します。 各スレッドはヒープから約1MBのメモリを消費するため（ 動的に割り当てられたメモリ。-約Transl。 ）、その結果、数千のスレッドを実行すると、メモリに大きな圧力がかかり、その不足によりアプリケーションがシャットダウンする可能性があります さらに、2つ以上のスレッドが互いに通信するようにしたい場合、これを行うのは非常に困難です。



一方、Goは、マルチコアプロセッサが既に利用可能であった2009年にリリースされました。 これが、Goがマルチタスクを念頭に置いて作成された理由です （ 同時実行性-およそTransl。 ）。 Goはスレッドの代わりにゴルーチンを使用します。 ヒープからほぼ2KBのメモリを消費します。 したがって、少なくとも何百万ものゴルーチンをいつでもスクロールできます。









ゴルーチンはどのように機能しますか？ 詳細： golangtutorials.blogspot.in/2011/06/goroutines.html



その他の利点：



  • ゴルーチンには、拡張可能なセグメント化されたスタックがあります。 これは、必要な場合にのみメモリを使用することを意味します。
  • ゴルーチンは、スレッドよりも起動時間が高速です。
  • ゴルーチンには、自身（チャネル）間の安全なデータ交換のための組み込みプリミティブが付属しています。
  • ゴルーチンは、データ構造を共有するときにミューテックスをブロックする必要性を回避します。
  • また、ゴルーチンとOSスレッドには1対1のマッピングがありません。 1つのゴルーチンは複数のスレッドで実行できます。 ゴルーチンは、少数のOSスレッドに結合（ 多重化-約Transl。 ）されます。


Rob Pikeの優れたパフォーマンスを見ることができますが、このトピックをより深く理解するために、 マルチタスクは並行処理ではありません 。


上記のすべての点により、GoはJava、C、C ++と同じようにマルチタスクに非常に強力であると同時に、Erlangのようにコードの美しさと自然さを保ちます。





Goには、両方の長所が組み込まれています。 書きやすく、マルチタスクで効果的



Goはハードウェアで直接実行されます。



Java / Pythonのような最新の言語の代わりにC / C ++を使用する最も重要な利点の1つは、そのパフォーマンスです。 C / C ++はコンパイルされているため、解釈されません。



プロセッサはバイナリコードのみを理解します。 原則として、Javaまたは他のJVMベースの言語でアプリケーションを作成する場合、プロジェクトをコンパイルするとき、人間が読み取れるコードからバイトコードにコンパイルされます。これは、JVMまたはOS上で実行される他の仮想マシンで理解できます。 実行されると、VMはこのバイトコードを解釈し、プロセッサが理解できるバイナリコードに変換します。





JVMベースのコードステージ



一方、仮想マシンを使用しないC / C ++があるため、上記の図から1つのステップを削除しながら、生産性を高めることができます。 人間が読めるコードをバイナリに直接コンパイルします。





しかし、これらの言語の変数をクリーンアップして配布するのはまだ苦痛です。 同時に、ほとんどのプログラミング言語は、ガベージコレクターと参照カウントアルゴリズムを使用してオブジェクトの配布と削除を処理します。



Goは両方の長所を吸収しました。 C / C ++などの低レベル言語のように、Goはコンパイルされた言語です。 これは、低レベル言語と同様のパフォーマンスを意味します。 また、オブジェクトを配布および削除するための独自のガベージコレクタもあります。 さらにmalloc（）およびfree（） ！ かっこいい！



Goで記述されたコードは保守が簡単です。



一つお話ししましょう。 Goには、他の言語のような狂った構文はありません-すべてが非常にきれいできれいです。



GoogleのGoogle開発者は、作成時にこれを念頭に置いていました。 Googleには巨大なコードベースがあり、数千人の開発者が作業しているという事実により、コードは他の開発者が理解しやすく、1つのコードが他のサイトに最小限の副作用をもたらすはずです。 これにより、コードの保守と変更が容易になります。



Goには、意図的にOOP言語の多くの機能が含まれていません。



  • クラスはありません。 各ものはパッケージのみに分割されます。 クラスの代わりに、Goは構造（ 構造体-およそTransl。 ）のみを使用します。
  • 継承をサポートしていません。 これにより、コードの変更が簡単になります。 Java / Pythonなどの他の言語では、ABCクラスがXYZクラスを継承し、同時にXYZクラスにいくつかの変更を加えると、これから継承されるすべてのクラスに追加の変更が加えられます。 さらに、継承がなければ、Goコードは理解しやすくなります（ コードセクションを学習するときに注意する必要があるスーパークラスがないため ）。
  • コンストラクターはありません。
  • 注釈なし。
  • ジェネリックはありません（ 汎用テンプレート-約Transl。 ）。
  • 例外はありません。


上記の変更により、Goは他の言語と区別されるため、他の言語でのプログラミングとは異なり、Goでのプログラミングが可能になります。 上記のポイントのいくつかが気に入らないかもしれません。 しかし、これは、これらの機能なしではアプリケーションを作成できないという意味ではありません。 あなたがしなければならないのは、2〜3行だけ書くことです。 ただし、その一方で、コードがよりわかりやすくなり、理解しやすくなります。





コードの読みやすさとパフォーマンス。



上記のグラフは、GoがC / C ++とほぼ同じくらい効率的であり、Ruby、Python、および他の言語と同じ単純な構文を持っていることを示しています。 これはwin-winオプションです（ win-win。-およそRem。 ）人にとってもコンピューターにとっても！



Goの構文は非常に安定していますが、 他の新しいSwift言語ではそれほど安定していません。 2012年のバージョン1.0の公開リリース以降、構文は変更されていません。 これにより、下位互換性が得られます。



GoはGoogleによってサポートされています。



  • これは技術的な利点ではないことを知っています。 ただし、GoはGoogleによって設計および管理されています。 Googleは世界最大のクラウドインフラストラクチャを持ち、現在も拡大しています。 GoogleはGoを作成して、スケーラビリティと効率性をサポートして独自の問題を解決しました。 独自のサーバーを作成するときに、同じ問題に直面する必要があります。
  • さらに、GoはAbode、BBC、IBM、Intel、 Mediumなどの企業でも使用されています。 （ソース： github.com/golang/go/wiki/GoUsers ）


結論：



  • Goは他のOOP言語とは異なりますが、依然として同じ獣です。 Goは、C / C ++と同じ高いパフォーマンス、Javaと同様の非常に効率的なマルチタスク処理、およびPython / Perlと同じコード記述の容易さを提供します。

  • Goを学習する予定がない場合は、ハードウェアの制限により、非常に効率的なコードを記述したいソフトウェア開発者にプレッシャーがかかると思います。 開発者はハードウェアを理解し、それに応じてプログラムを最適化する必要があります。 最適化されたソフトウェアは、安価で低速なデバイス（ IOTなど）で実行できます。これは、一般的にエンドユーザーエクスペリエンスに影響を与えます （ UX。-約Transl。 ）。
