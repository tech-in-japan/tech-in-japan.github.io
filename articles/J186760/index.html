WindowsのTP-LINK TL-WR741NDルーターのステップバイステップOpenWRTファームウェア

ルーターは、さまざまな理由でネイティブファームウェアを失います。 機能を追加したり、USBポートをハンダ付けしたり、ルーターの安定性を高めたいという要望。



私の場合、1〜5個のWiFiと1〜2個のイーサネットコンシューマを備えた最後の安定したネイティブファームウェア上のルーターは、週に1〜2回ハングアップしました。 これはそれほど怖いわけではありませんが、再起動するのにうんざりしています。 安定性が欲しかった。 長い検索とレビューを読んだ後、選択はOpenWRTに落ちました。 この記事には、試行錯誤をせずに順を追って説明する手順が既に記載されています。





ステップ0



ルーターのファームウェアを使用することを望みます。ステップの1つでインターネットにアクセスできなくなる可能性があることを理解してください。 したがって、この記事を完全に読み、必要なファイルをすべて事前にダウンロードし、その後ファームウェアを続行することを強くお勧めします。



ステップ1



最初に行うことは、最新のネイティブファームウェアをアップグレードしてインストールすることです。

ここでは、 TL-WR741NDの対応するバージョンの機器の最新ファームウェアを見つけることができます。

  • ルーターを裏返し、ケースのステッカーの案内に従って、その中のグランドのバージョンを確認します。
  • 最新のファームウェアでアーカイブをダウンロードします。
  • 通常の方法（システムツール->ファームウェアアップグレード）でアーカイブからファームウェアでルーターをフラッシュします。


ステップ2



次に、OpenWRTファームウェアバージョンを選択する必要があります。



モデルのIronバージョンと一致するファイルを正確に使用する必要があります。 このルールを無視しないでください！ ダウンロードできるファイルシステムオプションは、JFFS2とSquashFSの2つです。 レビューから判断すると、SquashFSファイルを選択する価値があります。 ファイル名の「 ファクトリー 」という語は、そのようなファイルをファクトリー（ネイティブ）ファームウェアの上にあるデバイスにアップロードする必要があることを意味します。 sysupgradeという単語を含むファイルは、OpenWRTの更新にのみ適用されます。 工場ファームウェアの後に注がれます。



SquashFSバージョン：



  • TL-WR741ND V1工場
  • TL-WR741ND V1 SysUpgrade


  • TL-WR741ND V2ファクトリー
  • TL-WR741ND V2 SysUpgrade


  • TL-WR741ND V4ファクトリー
  • TL-WR741ND V4 SysUpgrade




JFFS2バージョン：



  • TL-WR741ND V1工場
  • TL-WR741ND V1 SysUpgrade


  • TL-WR741ND V2ファクトリー
  • TL-WR741ND V2 SysUpgrade


  • TL-WR741ND V4ファクトリー
  • TL-WR741ND V4 SysUpgrade




工場ファームウェアをダウンロードし、通常の方法で縫います（システムツール->ファームウェアアップグレード）。



ステップ3またはそれはすべてなくなった



ファームウェアが正常に実行されると、Webインターフェースがオフになっている稼働中のルーターが取得されます。 したがって、ブラウザから設定にアクセスできなくなります。 その他のアクションは次のとおりです。



PuTTYをダウンロード

ダウンロードWinSCP



PuTTYを開き、 Telnetプロトコル、アドレス192.168.1.1、ポート23を選択して、「接続」をクリックします。



ルーターに入ります。 最初は、パスワードのないルートです。

次のコマンドを入力します。 

passwd root


そして2回、 rootユーザーの新しいパスワードを書きます 。 さらに書きます： 

 exit




WinSCPを開き、 SCPプロトコル、アドレス192.168.1.1ポート22 、ユーザー名： root 、以前に設定したパスワードを選択し、「ログイン」をクリックします。



あいまいな質問にはすべて答え、コンピューターとルーターにファイルがあるウィンドウが表示されます。 次に、ファームウェアの2番目の部分（sysupgradeの部分）を取得し、その名前をcode.binに変更して、/ tmpフォルダーのルーターにドロップします。 プログラムを閉じます。



PuTTYを開き、 SSHプロトコル、 root @ 192.168.1.1、ポート22を選択し、「接続」をクリックします。 ルートに対して以前に設定されたパスワードを入力します。 ルーターを入力し、ファームウェアのコマンドを入力します。 

 sysupgrade -n /tmp/code.bin


ルータがフラッシュして再起動するのを待っています。 私たちは電球に導かれています。 次に、手順を繰り返します。



PuTTYを開き、 Telnetプロトコル、アドレス192.168.1.1、ポート23を選択して、「接続」をクリックします。



ルーターに入ります。 最初は、パスワードのないルートです。

次のコマンドを入力します。 

 passwd root


そして2回、 rootユーザーの新しいパスワードを書きます 。



さらに書きます： 

 /etc/init.d/uhttpd enable /etc/init.d/uhttpd start


これらの2つのコマンドは、ルーターのWebベースのインターフェースを有効にして起動します。



ルーターの電源を切り、ルーターの電源を入れます。



192.168.1.1では、LuCI Webインターフェースを使用できます。 rootユーザーと、rootに対して以前に設定されたパスワードを使用します。



UPD。 長時間の操作の後、単一のグリッチが認識されます。 WANポートのMACアドレスを変更すると、1〜2日後に工場のアドレスにリセットされます。 それ以外の場合、ルーターは安定性の奇跡を示します。
