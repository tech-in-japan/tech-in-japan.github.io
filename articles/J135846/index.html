自宅の1Uサーバーのレシピ

この記事で説明するこの皿は非常に簡単ですが、多くのオンライン会議で準備の正確さと使用の適切性の両方について深刻な議論が繰り広げられます。



この料理は、デル、富士通、HP、IBM、およびサンの料理人による独自のレシピに従って、高品質の食材を使用した高価なレストランで調理できます。 はい、そのようなサーバーの価格は適切です。なぜなら、それは料理に対する支払いだけでなく、 保証サービスの支払いは、ちなみにかなり高いレベルです。



別のオプションがあります。品質はそれほど劣りませんが、ポケットにかかる圧力は小さくなります。 もちろん、Intel、Tyan、Supermicro、さらにはASUSなどの企業から、より少ないブランドのサーバーを購入することについて話している。 ここで、以前のバージョンとは異なり、製剤は多かれ少なかれ標準的であり、成分をアップグレードするとき、多くの場合、それを通常の非ブランドのものに置き換えることが可能です。



原則として、2番目のケースでサーバーを準備するのと同じ材料を半製品として購入し、最初に個々のコンポーネントの安価な（ただし信頼性が低い）アナログを使用して、それらから自分で機械を作ることができます。



もちろん、サーバーを自分で構築し、想像するのに十分なものから準備することができます。 しかし、多くの人にとって、想像力はより大きなサイズの2Uケースにかかっています（ウクライナでは、 CSV製品がよく使用されますが、 InPCはあまり使用されません ）。 これにはいくつかの理由があります。



まず、このようなパッケージでシステムを組み立てる方が安くなります。 2Uケースだけでコストが削減されるだけでなく、背面の80 mmファンで冷却される限り、電源もインストールできます。 また、プロセッサを冷却するには、バンドルされた「箱入り」クーラーまたは同様の安価なクーラーを使用できます。



第二に、冷却部品に問題はありません。 ほとんどのデスクトップマザーボードでは、メモリスロットは拡張スロットに垂直にボードに沿って配置されています。 つまり、このようなボードを1ユニットの高さのサーバーエンクロージャーに取り付けると、エアフロー全体がRAMモジュールによってブロックされますが、2ユニットエンクロージャーでは、空気循環のためにさらに5センチメートルの高さがあります。



第三に、ケースの高さを2倍にすると、複数の拡張カード（RAIDコントローラー、ネットワークカードなど）を一度に取り付けることができます。 また、2Uでは、入力/出力パネルのオーディオポートの高いブロックに関する問題は絶対にないことに注意する価値があります。これは、一部の単一ユニットのケースに単に収まらない場合があります（これは、ボードのモデルとケースのモデルによって異なります）。



成分



プロセッサー： Intel Core i7-2600-305ドル

クーラー：1U Active CoolJag DEN-7-35ドル（米国で注文）

マザーボード： Fujitsu D3062-B -150ドル

メモリー：4x 4096Mb Team Elite TED34096M1333C9-75ドル

ハードドライブ：2x 500Gb Western Digital WD5000AAKX -190ドル

ケース：1U Chenbro RM12700- $ 140

電源：250W シーズンSS-250SU- $ 55およびそのプレート-$ 3

オプション：2x SATAデータケーブル、2x 4pin-SATA電源アダプター、$ 5電源ケーブル。



料理



まず最初に、マザーボードのソケットにプロセッサを取り付けて修正する必要があります。 次に、ボードを裏返し、クーラーのベースプレート（バックプレート）を貼り付けてから、ボードを元に戻し、クーラー自体を取り付けます（ベースに塗布されたサーマルペーストから保護フィルムを取り外した後）、バックプレートナットにネジをねじ込みます。 メモリモジュールは、すぐに取り付けるか、ケースに「マザーボード」を取り付けた後に取り付けることができます。



次に、固定用のネジで電源装置をブラケットに接続し、このデザインをケースに取り付けます。 また、マザーボードを取り付ける前に、ケースに欠けている突起を固定する必要があります。 ちなみに、私の場合、プレート付きの電源とマザーボード付きのボンネットの両方がすでにインストールされています（このハードウェアは1つのサプライヤから購入されました）が、プロセッサ冷却システムをインストールするために後者を削除する必要がありました。



各ハードドライブは4本のネジでバスケットに取り付けられ、バスケット自体は1本でシャーシに取り付けられます。 この取り付け方法により、追加のバスケットがある場合は、ハードドライブをすばやく変更できます。 また、 メーカーのウェブサイトに示されているように、追加のバスケットが3台目のハードドライブのインストールに役立つ場合があります。



ハードドライブを取り付けたら、ケーブルの接続に進むことができます。 電源にはSATAコネクタがないため、1つの4ピンから2つの15ピンコネクタのアダプタを使用しました。 この場合、SATAライトアングルデータケーブルは機能しないことに注意してください。マザーボードにベンドのある側を取り付けると、隣接するSATAポートがブロックされ、既に取り付けられているハードドライブに回転コネクタを挿入することはほとんど不可能です。



マザーボードの電源のメインコネクタを20ピンバージョンで接続しました。これは、追加の4つの接点が役割を果たさず、空気の流れを確実に妨げるためです。 元々、これらのワイヤを完全にバイパスしてファンに対して反対側に配置することを計画していましたが、それらの長さは十分ではなく、対応する延長コードはどこにも販売されていませんでした。 はい。この段階では、4ピンプロセッサの電源コネクタを忘れずに接続する必要があります。



プロセッサー冷却ファンは、最も近い適切なソケットに接続する必要があります。 マザーボードには2つの空きファンコネクタしかないため、ケースに付属のアダプターを介してケースファンを接続しました。 フロントパネルのインジケータとボタンの接続図は、ボードの説明書に記載されています。 フロントUSBポート（必要な場合）は、他のボードと同じ方法で接続されます。



接続されたすべてのワイヤは、好みに応じて味付けすることができます。







食事を提供する前に、BIOSでいくつかの設定を行う必要があります。 POST中にエラーが発生する可能性があるにもかかわらずシステムを起動するには、 Boot-POST ErrorsオプションをDisabledに転送する必要があります。 また、 Disabledでは 、 Advanced-System Monitoring-Fan Controlオプションをインストールすることをお勧めします。これにより、クーラーのファンが常に最高速度で回転しますが、同時に「SMCTRL：Sensor＃ALERT」エラーが表示されます。 サーバーが機能するための重要な設定は、 電源障害回復です。 オプションを「 常にオン」に設定すると、緊急シャットダウン後にマシンが自動的に起動します（たとえば、データセンターの電源が失われたときや、クリーナーが誤ってしばらくソケットからプラグを抜いた場合）。



エネルギー値



サーバーを構築した後、Windows 7オペレーティングシステムをインストールして、負荷がかかっているプロセッサの温度を判断しました。 ボードに付属のディスクのイメージをUSBフラッシュドライブに記録し、サーバーに送りました。 ディスクユーティリティは、ボードのモデルを決定し、次のドライバーのインストールを提案しました。







並行して、SysGuard独自のプログラムがインストールされました。これにより、センサーの温度を監視し、ボードに接続されたファンを制御できます。







BIOSで自動ファン制御システムが有効になっている場合にのみ、このソフトウェアが機能することは注目に値します。



プロセッサの温度は室温22°Cで、サーモペーストKPT-8を使用して測定しました（2つのプロセッサにはストックサーマルインターフェイスが十分ではありません。KPT-8の方が1度効率的です）。 各モードのプロセッサの温度とは、RealTemp 3.60によって5分間測定された最もホットなプロセッサコアの温度を指します。 また、単純なBCLK乗算器ではx16にリセットされ、ロードされると、Celeronではx24に、Core i7ではx35に上昇したことも注目に値します。







ファンがオンの場合、プロセッサクーラーは温度を許容レベルに保ち、ストレステストで68℃を超えないようにします。 ファンをオフにすると、i7-2600の温度は着実に上昇し、WinRARの速度と信頼性のテストでも、WinRARはわいせつな値まで上昇しました（6分後、温度は88℃になりました）。 これは、このテストがこのプロセッサを50％しかロードしないという事実にもかかわらずです。



Celeron G530は、視覚的に比較するために、はるかに寒いです。 このプロセッサに90％の負荷をかけるアーカイバに組み込まれたテストに合格すると、ファンをオフにしても温度は40°Cを超えません。 10,000のタスクボリュームでLinXを5分間実行し、プロセッサからのすべてのジュースを絞って、50°C以上に加熱しないようにします。 ファンをオンにした状態で動作モードについて話す必要はありません。



より安価なプロセッサのトピックが提起されたので、低コストのマザーボードについて話す価値があります。 メモリスロットを横方向に並べたマザーボードについて話をすると、Biostarのマザーボードだけが思い浮かびます。 しかし、マザーボードの電源コネクタは、富士通よりもさらに失敗しています。 そのため、テスト用に撮影されたH61MLボードでは、Intel Core i7プロセッサはD3062-Bよりも数度熱くなります。



機能が少ないことに加えて、Biostarボードは経済的でもありません。







グラフでは、電源装置Ablecom SP262-1Sのテスト結果も確認できます。 このPSUは、以前は低コストのSupermicroサーバーとエンクロージャーにインストールされていました。 あなたが見ることができるように、それはまた効率で輝きません。 彼の庭のもう1つの石は、コンセントでのLtecコンデンサーの使用です。これは、控えめに言っても、疑わしい評判以上のものです。



栄養価



このセクションでは、マザーボードサーバーの構築に使用されるキーチップに焦点を当てます。



Fujitsu D3062-Bは、Intel Q67チップセットに基づいています。 このロジックセットは、Intel Active ManagementおよびIntel vProテクノロジーをサポートしています。これらの相互作用により、コンピューター（この場合はサーバー）をリモートで制御できます。 ネットワーク経由でサーバーをオン、オフ、または再起動できるだけでなく、ディスクからサーバーをロードしたり、KVMアクセスで管理したりすることもできます。 THG.ruには、これらのテクノロジーの機能を説明する優れた記事があります。



このような「バン」を取得するには、vProをサポートするプロセッサを使用する必要があります。 プロセッサのリストはここにあります 。 また、KVMは統合グラフィックアダプターを使用する場合にのみ機能することにも注意してください。



まず、BIOSボードで小さなシャーマンをプレイする必要があります。 拡張-AMT構成-MEBxモードを標準に設定する必要があります。 [ 電力-低電力ソフトオフ]オプションは[ 無効 ]に適切に転送されます 。これにより、オフになっているサーバーにアクセスできます。



その後、サーバーを再起動する必要があります。POSTを渡すときは、キーボードのCtrl + Pを押します。これにより、Management Engine BIOSメニューに移動できます。 そこで、標準の管理者パスワードを入力し、すぐに独自のパスワードを設定する必要があります。 パスワードは少なくとも8文字で構成され、少なくとも1つの小文字、1つの大文字、数字、特殊文字（ "、：を除く）を含む必要があります。したがって、パスワードとしてP @ ssw0rdがよく使用されます。



リモートコントロールを機能させるには、 MEBxメニューのIntel AMT Configurationセクションで、次の項目を構成する必要があります： Manageability Feature Selection-Enabled 、 SOL / IDER / KVMセクションですべての項目をEnabledに設定し、 User Consent-User Opt-inメニューをNoneに設定しますネットワーク設定-Intel MEネットワーク名設定-ホスト名 （例： server ）でマシン名を設定してください。 DHCPを使用する場合は、 [TCP / IP設定-有線LAN IPv4構成-DHCPモード ]メニューで[ 有効 ]を設定する必要があります。そうでない場合は、ネットワーク設定を手動で構成します。 このすべての後、[ ネットワークアクセスのアクティブ化 ]をクリックして、決定を確認する必要があります。



奇妙なことに、MEBxの設定はすべて、DELLのWebサイト（ こちらとこちら）でロシア語でよく描かれています 。



IPアドレス16992にアクセスするか、Radminプログラムを使用して、ブラウザーからiAMTおよびvPro機能を使用できます。 Real VNCなどのプログラムを通じてKVMを使用してサーバーを管理できます。



合計で、1000ドル未満で十分に充填された自己組み立てサーバーがあります。 利点には、有名ブランドのコンポーネントの使用、パフォーマンス、効率、見込み、KVMが含まれます。 もちろん、欠点は、通常のコンピューターストアでマザーボードとクーラーを購入できないことです。



一方、市販のBiostarのようなマザーボードには十分な機能がなく、安価なサーバーに収まります。 また、このボードにインストールされているプロセッサがCeleron G530よりもはるかに貪欲ではないことを除いて、アクティブな単一ユニットクーラーの購入はキャンセルされません。 いずれにせよ、私はすでに選択をしました。
