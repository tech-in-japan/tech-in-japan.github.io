Ubuntu上のEclipseでのARMコントローラーのプログラミング：実行方法

職場で、AT91SA7A3コントローラーのプログラミングに遭遇しました。これにはIARが使用されています。 しかし、私は2番目のシステムを維持するのが面倒なので、私のお気に入りのUbuntuでプログラミング、デバッグ、ファームウェアの可能性を提供する試みがなされました。 そして、わいせつなボキャブラリーを通して2週間の壁とのコミュニケーションの後（壁紙が動き始めたのと同じくらい）、この経済をすべてセットアップすることができました。 そして、おそらく、私だけではないので、これがどのように行われるかを書きます。



AT91SAM7A3-EKボード、Atmel SAM-ICE JTAGアダプター、Ubuntu 11.04を搭載したコンピューター、タバコ1パックとコーヒーマグ2杯。 素敵なキット。 行こう！



1）Eclipse Webサイト（http://www.eclipse.org）にアクセスし、システムのC / C ++開発者向けのEclipse IDEをダウンロードします。 これの最新バージョンはEclipse 3.7 Indigoです。

2）OpenOCDプロジェクトのWebサイト（http://developer.berlios.de/projects/openocd/）にアクセスし、OpenOCD 0.5.0ソースコードを出力します。

3）コンパイラーに行きます。 ARMGCCとは異なり、Sourcery G ++を選択しました。 IA32 GNU / Linux TARをダウンロードしてください。

4）オプション：これらはすべて排出されますが、煙を吐くことができます。

5）Eclipseアーカイブを必要なディレクトリに解凍します（/ home / user / armがあるため、eclipseフォルダーが表示されます）。

6）コンパイラアーカイブを目的のディレクトリに解凍します（/ home / user / armがある場合、arm-2011.03フォルダーが表示されるか、類似したものが表示されます。数字は異なる場合があります）。

7）Synapticに移動し、usb-devパッケージをインストールします。後で必要になります。

8）OpenOCDを使用してアーカイブを解凍します。ホームフォルダーに保存すると、パスが少し簡単になります。

9）ターミナルを実行し、OpenOCDソースフォルダー（解凍したフォルダー）にジャンプして、次のコマンドを記述します。



./configure --enable-jlink

sudo make

sudo make install



SAM-ICEのサポートを有効にするには、キー--enable-jlinkが必要です。このため、usb-devパッケージをインストールしました。

10）オプション：これがすべて行われている間、コーヒーを飲みに行くことができます。

11）コンソールでも次のように記述します：sudo gedit / etc / environment。

開いたファイルで、コンパイラへのパスを追加すると、次のようなものが得られます。

PATH = "/ usr / local / sbin：/ usr / local / bin：/ usr / sbin：/ usr / bin：/ sbin：/ bin：/ usr / games：/home/user/arm/arm-2011.03/bin 」

車を再起動し、（オプションで）喫煙します。

12）マシンを再起動し、フォルダーからEclipseを起動し、ワークスペースを選択して、ボックスをチェックして、クロールしないようにします。 「ヘルプ」->「新規ソフトウェアのインストール」に進み、「追加」ボタンをクリックして、名前Zylincdt、アドレスopensource.zylin.com/zylincdtを書き、「OK」をクリックし、Zylin Embedded CDTウィンドウで唯一のパッケージにチェックマークを付け、インストーラーの指示に従います。再起動します。 ここで新しいプロジェクトを作成します：ファイル->新規-> Cプロジェクトに移動し、Makefileプロジェクト->空のプロジェクト、そこにあるツールチェーン-他のツールチェーンを選択し、プロジェクトに名前を付けて終了をクリックします。 次に、一時的にEclipseを最小化します。

13）フォルダー/ usr / local / share / openocd / scripts / targetに移動し、そこにある名前のコントローラーを使用して構成ファイルを探します。 鉱山（AT91SAM7A3）はありませんでしたが、Googleですぐに見つかり、ハンドルを少し調整しました。 あれば、結構です。 SAM-ICEをコンピューターとボードに接続し、ボードに電源を投入します。 ターミナルを起動して、次のように記述します。



sudo openocd -f interface / jlink.cfg -c "jtag_rclk 30; gdb_flash_program enable; gdb_memory_map disable" -f target / at91sam7a3.cfg



注意！ 別のアダプターまたは別のボードがある場合は、jlink.cfgとat91sam7a3.cfgを必要なファイルに置き換えます。

OpenOCDが起動し、大量の情報が表示されるはずです。その中には、チップモデルがあります。 ターミナルウィンドウをオフにします。

14）AtmelのWebサイトにアクセスして、トライアルプロジェクトを探します。 やや曲がったシステムがあるので、すぐにAT91SAM7A3-EK ボードのプロジェクトへの直接リンクを提供します： www.atmel.com/dyn/resources / ...ing_started.zip 。 ダウンロードし、解凍し、解凍した場所を覚えておいてください。 Eclipseに戻り、プロジェクトを右クリックして、[インポート]を選択し、そこにGeneral-File Systemを選択します。 新しくダウンロードしたプロジェクトが解凍されたフォルダーを彼に指示し、必要なファイルを指定して（すべてを選択するだけで苦しむ必要はありません）、[完了]をクリックします。

15）プロジェクトのMakefileに移動し、CROSS_COMPILE = arm-elf-の行を見つけてCROSS_COMPILE = arm-none-eabi-Saveに置き換えます。Makefileを保存し、Project-> Build Projectに進み、すべてがコンパイルおよびコンパイルされる様子を確認します。 すべてが正しく行われた場合、出力は2つのファイル* .elfおよび* .binになります。

17）デバッガーを構成します。 Run-> Debug Configurationに移動し、Zylin Embedded Debug（Native）sram（デバッグ用）とZylin Embedded Debug（Native）フラッシュ（ファームウェア用）の2つの新しい構成を作成します。 タブ付きになりました。

メイン：C / C ++アプリケーションは、パスとファイル名を指定します。* .elfはデバッグに必要、* .bin-はファームウェアに必要です。

デバッガー：GDBデバッガー-デバッガーへのパスを指定します。この場合、home / user / arm / arm-2011.03 / bin / arm-none-eabi-gdbのようになります。

GDBコマンドファイル-ここでは、対応するgdbinitファイルへのパスを示します。

コマンド：[初期化コマンド]フィールドは空です。

[共通]タブ：[お気に入りメニューに表示]の[デバッグ]ボックスをオンにします。

[適用]をクリックし、OpenOCDが実行されていることを確認し、[デバッグ]をクリックします。 sram構成が実行されている場合、デバッグ情報を含むウィンドウが表示されます。 オプションで、変数、メモリ、逆アセンブリなどの概要を含むウィンドウを追加できます。

出来上がり、すべてが動作します。

ちょっとした追加。 ダウンロードしたプロジェクトを削除しないことをお勧めします-レジスタの操作を容易にするための定数を記述するヘッダーファイル、スタートアップファイル、リンカースクリプト（重要）、最も重要なことは、デバッガーのファイルにメモリがどのように割り当てられるかを説明するMakefileですプログラマー。 私にとっては、それらをそれぞれの新しいプロジェクトに転送し、Makefileを少し編集する方が良いでしょう。 そして人生は美しいでしょう。
