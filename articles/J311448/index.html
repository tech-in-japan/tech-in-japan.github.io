再結成後のNode.jsの展望-専門家の意見





昔々、PHP、Apache、MySQLをAJAXを介してJavaScriptと組み合わせることは、Web開発者にとって完璧な組み合わせでした。 この一連のツールで問題を解決できると思われました。 ただし、要件が増加し、ソースコードが目の前に成長し、負荷が増加し、通常のツールが対処しなくなりました。



専門家は 、古典的な「リクエスト/レスポンス」スキームが原因であると確信していました。 ページリクエストにより、Webサーバーはスクリプトを取得し、それを直線的に実行し、結果をクライアントブラウザーに返しました。 その後、次のリクエストの処理に進みます。



この状況から抜け出す方法を探して、明るい頭脳は、サーバーがJavaScriptでも記述できることを思い出しました。



これにより、サイトのサーバー部分とクライアント部分を同じ言語で開発できます。 2000年代の終わりまでには、通常の手続き型から機能的なスタイルと組み合わせたOOPまで、さまざまなパラダイムでコードを記述できるシンプルかつ柔軟な言語でした。 しかし、JavaScriptの別の機能が決定的な役割を果たしました-非同期性。



ここから 、PHPコードの連続実行の制限の例を示します 。 

$result = $db->fetchOne('SELECT user_name FROM user_accounts WHERE id = 1'); echo ' : ' . $result . ';';
最初の行には、id = 1のユーザー名を取得するためのデータベースへの単純なSQLクエリが含まれています。この時点で、スクリプトは停止し、データベースによって要求が処理され、結果が返されるまで次の行は実行されません$結果変数。 この例では、これらは1000分の1秒ですが、実際にはクエリははるかに複雑であり、データベースのサイズは多くの場合ギガバイトであり、同時に数千のクエリが存在する可能性があります。


非同期JSコードの実行： 

 db.query('SELECT user_name FROM user_accounts WHERE id = 1', function(err, res) { if (!err) sys.log(' : ' + res); }); sys.log(' ');
ここでも、データベースへのクエリが作成されますが、SQL式自体に加えて、リクエストはコールバック関数も渡します。 この関数は、データベースからの応答が到着すると正確に呼び出され、その瞬間までスクリプトは実行を停止しません。 そのため、サーバーサイドJavaScriptのどのバージョンでも、イベントとコールバック、つまりイベントハンドラーの概念が基礎になっています。 イベントのイベントで実行する必要があるアクションは、イベントハンドラーの関数内に記述されています。


このような考慮事項により、JS愛好家は独自のサーバーエンジンを作成するようになりました。



Node.js



9月2日に、Chromiumオープンブラウザの最初の公開リリースが正式に発表されました。 開発中に、ブラウザーベースのJavaScriptインタープリターであるV8も作成されました。 主な開発者はラース・バックでした。 エンジンで開発者が解決しなければならなかった主な問題は、パフォーマンスとスケーラビリティでした。



ブラウザが成功した後、明るい頭脳は、このような高速エンジンもサーバーで正常に使用できることに気付きました。



さらに実験を行った結果、Node.jsプロジェクトが登場しました。これは、エンジンに加えて、統合サーバー（HTTPおよびTCP / UDP / Unixソケット）とライブラリの基本セットを含む完全に独立したプラットフォームであり、ファイルとネットワークデバイスとの完全な非同期作業を提供します。



Node.jsは2009年にRyan Dahlによって開発されました。 彼は、フローに基づく並列処理の従来のモデルではなく、イベント指向システムに目を向けるべきだという結論に達しました。 このモデルが選ばれた理由は、そのシンプルさ、低いオーバーヘッド（「接続ごとに1つのスレッド」というイデオロギーと比較して）、および速度です。



数千の接続を非同期で処理できるHTTPサーバーを起動するには、数行のコードが必要です。 



 //   http var http = require('http'); //  web-    var server = http.createServer(function (req, res) { console.log('  '); //     http- res.writeHead(200, { 'Content-Type': 'text/plain; charset=UTF-8' }); res.end('Hello world!'); }); //  web- server.listen(2002, "127.0.0.1", function () { console.log('  http://127.0.0.1:2002/'); });


Node.jsの出現は、開発者の間で騒動を巻き起こしました。 プロジェクトの開発の初期段階ですでに、Netflix、Walmart、PayPal、Dow Jones、Grouponなどの企業がプロジェクトの使用を開始しました。



そして、彼らはいつまでも幸せに生きるだろうが...



2014年11月、Node.jsコミュニティで分裂が発生しました。 一部の参加者は、Node.jsの開発を監督するJoyentのポリシーに不満を抱いていました。



また、Joyentはコミュニティの意見を無視し、自身の利益のためだけに行動し、プロジェクトの制御を自身の手でのみ集中したと主張されました。 さらに、その時までに、Joyentはコードベースの安定性を確保することに絶対的な優先順位を与えていたため、新機能の統合が複雑になりました。 2013年の初め以来、重要なリリースはありません。最新の現在のブランチ0.10は、V8エンジンの古いバージョンに基づいていました。



Joyentからの分離のイニシアチブは、Node.jsの7人の主要開発者のうち5人によってサポートされていました。その中には、元プロジェクトリーダーであるIsaacSchlüterがいました。

共同プロジェクトのキュレーターはLinux Foundationで、Node.jsの開発のための効果的で独立したプラットフォームを形成しました。 Joyentは、プロジェクトの開発、サポート、資金調達に引き続き参加しましたが、コミュニティの普通のメンバーとして参加しました。 Joyentに加えて、Node.js Foundationの創設者には、IBM、Microsoft、PayPal、Fidelity、SAPなどの企業が含まれていました。
