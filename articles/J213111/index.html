日立コーポレートフラッシュソリューションの概要





今日、フラッシュメモリテクノロジーはすでに消費者向けデバイスの市場で確固たる地位を占めており、企業ソリューションの分野で急速に普及することを期待する理由はすべてあります。 主な問題は、組織が利用可能なさまざまなタイプのフラッシュドライブ、アプリケーション、企業に提供できるメリットと機会をどのようにナビゲートするかです。 この点で日立データシステムズが提供できるものについてのHabraobzor



日立は、フラッシュテクノロジーを顧客にとって大きな可能性があると考えており、ミッションクリティカルなエンタープライズレベルのアプリケーションでこのテクノロジーを使用するための新しいソリューションを作成しています。 日立は伝統を守り、長年にわたって各ストレージシステムにSSDオプションを提供してきました。最近、 日立の新しいフラッシュ管理テクノロジーに基づいた一連の統合フラッシュデバイスをリリースしました。



エントリレベルのエンタープライズストレージシステムHitachi Unified Storage VM（HUS VM）とフラッシュドライブFlash Module Drive（FMD）で構成されるフラッシュソリューションをご紹介します。

この提案には3つの主要なモデルが含まれ、フラッシュモジュールの数が異なるため、パフォーマンスが異なりますが、ソリューションはリソースを必要とするタスクのための十分なスケーラビリティとさらなる成長のための拡張をサポートします。



次の表に、さまざまなソリューションモデルのいくつかの特性を示します。







HUS VMプラットフォームとFMDモジュールに基づくフラッシュソリューション





HUS VMシステムは、最新のソリッドステートドライブを使用して最も要求の多いデータを保存することにより、大規模なストレージ容量と高性能の間で必要なバランスを提供できます。 ただし、SSDの使用にはいくつかの制限があり、その中で最も重要なのはデータ密度が低いことです。 この問題の解決策は革新的な製品でした。これは、フラッシュモジュールドライブと呼ばれる日立独自の開発の結果です。



各モジュールは、フラッシュメモリチップと、チップの動作を制御し、高度な機能を実装するために使用される専用の日立製コントローラを組み合わせています。 このようなコントローラーにより、次のような非常に便利な機能を実装することができました。

•アドレスの変換。 論理的および物理的なアドレス変換により、古いデータを非同期で削除できます。これにより、ピーク負荷時のプロセッサ負荷を削減できます。



•短いエントリ用のバッファ。 フラッシュモジュールのメモリセルは特定のサイズであるため、短いレコードは最初に特別なバッファに収集され、必要なサイズに達した場合にのみブロック全体として書き込まれます。 このメカニズムにより、FMDモジュールの容量を大幅に最適化できます。



•データの更新。 データの損失を避けるために、各モジュールは30日ごとに利用可能な情報を自動的に上書きします。



•リアルタイムでのデータの圧縮。 一定のストリーム「1」または「0」では、データ圧縮アルゴリズムがアクティブになります。これにより、最初の文字が1つだけ書き込まれ、その後、書き込まれる同一文字の数を示すポインターが置かれます。 このメカニズムにより、ストレージ容量の最大94％を節約できます。



•レコードのグローバルな配布。 FMDモジュールのパフォーマンスと寿命を延ばすために、コントローラーは、データがすべてのメモリ位置に均等に記録されることを個別に監視します。 これにより、フラッシュメモリセルの摩耗が大幅に減少し、システム全体のパフォーマンスが向上します。



HUS VMシステムの一部としてFMDモジュールを接続するには、専用のディスクシェルフが使用されます。 各シェルフの高さはわずか2Uで、1.6 TBおよび3.2 TBの容量で最大12個のFMDモジュールのインストールをサポートします。 HUS VMシステムは96 FMDに拡張されるため、300 TBを超えるフラッシュ容量を取得できます。



ディスクモジュールフラッシュモジュールドライブは、リソースを必要とする複雑なタスクのソリューションにうまく対処します。高負荷のデータベースのパフォーマンスの向上、分析サービスの効率的な運用、クラウドコンピューティングです。



単なるフラッシュ以上のもの



HUS VMシステムとFMDモジュールを一緒に使用すると、各ドライブの内部機能に加えて、本格的なストレージシステムのあらゆる機能が利用可能になります。 HUS VMプラットフォームは、 Hitachi Data Systemsのストレージシステム製品群の主要製品であり、高性能エンタープライズレベルプラットフォームのパワー、スケーラビリティ、および中間層システムからのデータへの統合アクセスを組み合わせています。 プラットフォームの独自の機能により、あらゆる種類のデータ用のユニバーサルストレージシステムを作成できます。



•外部アレイの仮想化。 HUS VMは、他のメーカーのストレージシステムのバーチャライザーとして機能できますが、データの移行は可能な限り最高の速度で最小限のリスクで行われます。 仮想化ボリュームは64 PBに拡大します。



•マルチレベルストレージシステム。 さまざまなタイプのディスクを備えたシステムでリソースを最適に使用するために、異なるストレージレベル間でのデータブロックの移行を保証するHitachi Dynamic Tiering（HDT）機能が使用されます。 これを使用すると、最も負荷の高い論理ボリューム（LUN）をより高速なディスクに自動的にまたは手動で移動し、最も要求の少ないボリュームを大規模なNL-SASディスクに移動できます。



•インスタントショット。 Hitachi Thin Imageエンジンは、論理ボリュームのスナップショットを提供し、Copy-on-Writeスナップショット機能の進化であり、Hitachi Dynamic Provisioning機能を使用して、論理ボリュームごとに最大1024個のスナップショットを作成できます。



•クローンデータ。 ローカルデータ保護とファイルリソースのクローン作成は、ユニークなJetCloneメカニズムによって提供されます。これは、ソースデータを実際にコピーすることなく、最も効率的なストレージメカニズムでオブジェクトのクローンを作成する機能です。 クローンを作成する際のコピー操作がないため、作成プロセスはほぼ瞬時に進みます。



•同期および非同期複製。 HUS VMプラットフォームは、同期および非同期のデータ複製のメカニズムを実装しています。 解決するタスク、容量、通信チャネルに基づいて、1つまたは別のレプリケーションメカニズムを使用して、耐障害ソリューションを設計し、地理的に分散したデータセンターを作成できます。



•ファイルアクセス。 HNASプラグインの助けを借りたプラットフォームは、CIFS、NFS、およびFTPプロトコルを介してデータへのアクセスを提供し、ブロックデータとファイルデータの両方のアクセスとストレージを提供するためのすべてのタスクを解決します。



ソリューションの一般的な特性



FMDフラッシュモジュールと組み合わせた本格的なHUS VMストレージシステムは、機能、スケーラビリティ、信頼性の幅広い機能を備えたソリューションです。

•応答時間が1ミリ秒未満。

•幅広いスケーラビリティ：300 TBを超えるフラッシュ容量、1,300 TBのSAS容量、4,600 TBのNL-SAS容量。

•各FMDドライブに専用コントローラー、独自のキャッシュメモリ、MLCメモリモジュールが存在し、全体的なパフォーマンスが向上します。

•TBの電力消費は、SSDドライブを使用する場合よりも40％少なくなります。

•エラー整合メカニズム（ECC）および記録されたすべてのデータを30日ごとに自動更新して、データの整合性を確保します。

•ファイバーチャネルとiSCSIを介して接続する機能。

•中断なしのファームウェアのアップグレードとハードウェアの変更。

•ホットスワップ可能な冗長コンポーネントへの自動フェールオーバーのアーキテクチャ。

•バッテリーとフラッシュドライブを使用したキャッシュメモリの不揮発性バックアップ。

•ハードウェアRAID 5、6、10のサポート。
