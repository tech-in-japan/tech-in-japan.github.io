コードベンチャー：人時間投資

昨日、新興企業に緑色の光を与え、この世界をより良い場所にし、光の中に現れるアプリケーションのコードの品質を制御する方法のアイデアに訪れました。

この記事を読むのに15分以上かかることはありませんが、注意深く読んでください。 読書は持ち去り（私や私の友人のように）、精神的な負荷を生み出しません（ 式もコードも技術もありません ）。 プロジェクトへの非金融投資についてお話します。

要するに 、私は自分の分野の専門家であり、自由な時間があります。 開始したいプロジェクトに投資するにはどうすればよいですか？





クラウドソーシング



このスキームについては詳しく説明しませんが、みんなの意見の例はオープンソースの世界です。

1つの例外を除き、この開発スキームはクローズドソースになる場合があります。 たとえば、やや歪んだ状況で-フリーランサーがプロジェクトに雇われたとき。

支払いの場合、ここには多くの利点があります。

  • 人は自分のビジネスに情熱を燃やし、自分の仕事に対して桁違いの責任を負い、短時間で仕事をします（仕事数/労働時間の観点から、週に数時間を費やし、速度が低下します）。 アイドル!!! ）は、コードを改善する方法を考えています。
  • はるかに低い支払いが必要です：

    • 輸送費なし
    • スケジュールは完全に無料です
    • 彼は自家製の食べ物を食べることができ、高価なカフェで散財しない
    • 彼が仕事を愛し、無料で働く準備ができているからです
  • オフィスにお金を払う必要はありません。オフィスは単に必要ではなく、誰もが自宅で働いています。


注：この記事では、マイナスについては説明しません。 彼らは怖いだけのように聞こえますが、ビジネスへの巧みなアプローチで、彼らは無頓着になります。 全体の問題は、それらが次のように定式化されていることです。 これを予測/制御しないと非常に悪いでしょう。 例：責任者を割り当てないと、コードの品質が低下し、学生が記述し、異なるモジュールが個別に動作します（ 残念ながら、ほとんどの問題は次のように定式化されます： コードの品質が低下します ） 。



クラウドファンディング



顧客獲得


多くの人が、ソフトウェア開発の世界におけるクラウドファンディングとは何かを知っていると思います。 知らない人のために、たくさんのゲームファン（ 100万人 ）が集まり、全員が予約注文として10ドルを提供します（ 開発に投資します ）。 その結果、開発者は数か月で予算が1,000万ドルの新しいゲームを提供します。

このスキームには明らかな利点があります：

  • プロジェクトからのすべての利益は開発者に送られ、投資家への低下ではありません（ 通常、彼らは彼に30-70％を与えます ）。 これは、会社のさらなる発展の鍵です。
  • 開発者は、開発に全額を費やすことを恐れません-破産することはありません。 実際、彼はすでにそれを費やしており、イベントは時間内に混在しています。
  • 人々が必要とするプログラムのみが開発されています。 人々は投資家（ そして製品の株式の非合法的な所有者 ）になるので、彼はどのような変更を行うかを決定する際に投票を受け取ります。 バランスを改善しますか？ パフォーマンス？ 新しい人種/ヒーロー/武器を追加しますか？ 新しい言語のサポート？ 多数派に関連する問題のみが開発されていますが、少数派の重大な間違いを修正する優先順位は常に高いため、それらは重大です。




最初の成功


ゲームなどのソフトウェア開発の場合、必要な量が収集されるとどうなりますか？ このお金は以下に行きます：

  • オフィスレンタル
  • 有料プログラマー
  • アーティスト/デザイナーの支払い
  • ミュージシャンを支払う
  • このゲームに必要な他の人（俳優など）への支払い
  • 広告（ 通常、数百万人があなたを知っているクラウドファンディングで稼いだ後は必要ありません ）


したがって、このすべてのお金はビジネスに費やされ、「枕」に不可抗力の場合には数千を残します。



私のアイデアは、これら2つの世界を結び付け、プラスのみを接続し、お互いのマイナスを吸収できる場所に接続します。

たとえば、キックスターターに十分なお金がない場合、プロジェクトは開始されません。 ただし、10人未満の場合、プロジェクトは引き続き開始されますが、少し後に完了します。



コードベンチャー



作成する理由


お金がすべて？


あなたはプログラマであり、まだゲームのファンです。 これがディアブロだとしましょう。 あなたは本当に次のバージョンのリリースが欲しい、あなたはあなたの10ドルを投資します。

同社はプログラマー、優秀なプログラマーを雇用していますが、必ずしもゲームのファンではありません。 1か月あたり2000ドルを受け取るという事実に基づいて、10ドルで1時間働きます（彼らは本当に良いプログラマーです）。

ゲームファンであるあなたが幸運にもこの会社に入ったとしましょう。 そして、それはあなたがどのくらい早くあなたのサイトを書くか、どれだけ早く新しいバージョンがリリースされるかに依存します。 10ドルでいくら稼げますか？ 時間？ 二人？ はい、私はあなたが無料で（または非常に安い）仕事をする準備ができていると確信しています。 残業、24時間体制で、可能であれば分割します（間違いなくそうします）。

会社が次の給与を支払うことができないと言って破産した場合、どうなりますか？ 2行目は、そこに滞在して無料で仕事をするか、オフィスの家賃を支払うために貯金を貸すこともできます 。 そして、残りはファンではなく？ そうです、彼らは去り、最初はお金のために働きました。



経験がない場合は？


ある人は素晴らしいアイデアを持っていました（ 要点はコブを奪うことです ）が、彼はソフトウェア開発の経験がまったくありません。 そしてさらにそう、人々の管理、この開発の調整。 （ または多分資金？ ）

製品をリリースする唯一の方法は、製品を外部委託することです。 さらに、そうすることで、彼は著作権を完全に失い、少額の補償を受けます。 または、議決権が1％未満の小株を受け取ります。 彼は開発の方向性に影響を与えることはできません。たとえ100の改善があったとしても、それらはすべて死んでいます。

ここで、資金調達なしで（ または最小限の、より真実に見える ）他の人のソフトウェアの開発を専門とする会社（ Code Venture ）を想像してください。

（ 正式に言えば、これは投資時間、労働と呼ばれます ）

この場合、すべてのプロジェクト参加者は例外なく、アイデアの著者（ Linus Torvalds ）を尊重し、もちろんユーザーのニーズを考慮して、さらに実装するものを決定するすべての権利を持ちます。 これは進歩の道のりで明らかなプラスです-明らかに死んだアイデアは生きる権利を得ます。



なぜうまくいくと思いますか？



主な仕事に加えて、私は常に始めました。 しかし、彼は決してグローバルなものを思いつきませんでした。 小さなプロジェクトですが、興味があるかもしれません。

まだ一般的ではなかった2009年のインターネットラジオ。 私たちは10人のバックボーンであり、3年間、一定のスケジュールと放送で働いていました。 誰もお金を受け取らず、誰もそれを計画しませんでした。 私たちは観客に感情を伝えて幸せでした。 （ 私の知る限りでは）世界で最初の非フォーマット無線。 私たちのエーテルは、すべてが海で泳ぐように多様でした。



そして、何が起こりますか？


私はプログラマーのコミュニティを作成して、やがてLinuxコミュニティに匹敵するようになる予定です。 完全な調整構造が利用可能であり、よく知られている/一般に受け入れられているものとは異なります。

投資が必要なシステムに新しいアイデアが現れると、例外なくすべての参加者が投票に参加します。 負荷、参加者の経験レベル、その他の必然的に客観的な判断に基づいて、ユニットのリーダーが優先されます（リーダーの評価には、詳細なコメントが必須です）。

いくつかの人が考えた（そして考えたかもしれない）ので、 私は結果として生じるプロジェクトから利益のペニーを充当するつもりはない 。 このコミュニティは、プロジェクトの隣に形成されるのは社会ではなく、社会で形成されるプロジェクトであることを除いて、オープンソースに非常に似ています。 オープンソースシステムのドナのような利益の広がり。



コード品質


経験のない燃えるような目と曲がった手を持つ多くの学生がそのようなプロジェクトに行くことは秘密ではありません。 それらのほとんどは、単純なコードを叫ぶか書く必要があります（ すべてではありません！ ）。 排気はゼロですが、アイデアの生成者です。 これは非常に重要です！

このチームが3〜4人の優秀なプログラマーとして働くには、10人の学生に対して1人で十分です。 （ 数値は天井からの数字であり、統計はありません 。 学生との共同作業の経験のみです）。 そして、各ショットの品質に関心のある優れたチームでは、個人のオフィスよりも速く、彼らは急速に上昇します。

しかし、社会が数年前から存在していると仮定します。 毎月新しいタスク、新しい球。 プロジェクト内で作成された数百のライブラリ、数百のモジュール。 これにより、サードパーティのライブラリを購入したり、独自のライブラリを開発したりすることなく、こうした問題をすばやく解決できます。

多数の目標により、数週間で複雑なソリューションを作成することができます（ もちろん、可能な場合 ）。そして、「頭脳」への献身と献身- それにより 、 理想的な ものになります（ かなりの仕事経験がある人がいる場合 ）

いずれにせよ、私はあなたが理解していると思います、問題は解決されており、それは私にとって壮大なようです。



次は？


  • これが成功すれば、アーティスト、アニメーター、ミュージシャン、弁護士、会計士、マネージャーなどがチームに登場することを願っています。 私たちは1つの大きな家族になり、最初から最後まですべての開発を引き受けることができます。 私たちはアウトソーシング会社になります。
  • アイデアを開発するための有利な条件（ アイデアから製品への一歩であり、サンプルや数百万は必要ありません ）が、私たちが新しい成果のインキュベーターになり、進歩がはるかに前進することを願っています。
  • 1つのコミュニティで多数のソフトウェアが開発される場合、コードはどこでも統一され、製品は相互に互換性があると思います。 これにより、1つのアプリケーションで何百ものプログラムをサポートし、フォーマットの変更に迅速に対応して脆弱性を修正できます。
  • 「新しいチップのリリース」から「すべてのモジュールでのサポートの実装」までの距離は最短時間に短縮され、サポート形式の交渉はより忠実になります。
