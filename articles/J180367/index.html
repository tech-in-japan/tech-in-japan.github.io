こんにちはHabr！



この投稿は、モバイル、デスクトップ、またはブラウザアプリケーションの開発中に時間を稼ぐ方法に興味がある人にとって興味深いものになります。 この情報は、アプリケーションがより機能的になり、何倍も速く世界に入るのに役立ちます。







（注意：猫の下には簡単なコードの例がたくさんあります。「自転車」を愛する人は読むことをお勧めしません。このサービスの人気が高まった後、サーバー側を書くときに日常的なタスクを押すことで地球温暖化の脅威が大幅に減少することが期待されます。）



Backendlessは、あらゆる種類のアプリケーションに既製のクラウドサーバーインフラストラクチャを提供するサービスとしてのバックエンドプラットフォームです。 これにより、開発者、新興企業、および大企業は、サーバーの開発を放棄し、アプリケーションの機能、プロモーション、およびユーザー（UXの向上）に集中することで、時間とお金を稼ぐことができます。 APIプラットフォームは、JavaScript、Android、iOS、Windows Phone、Flex / AIRのクライアント環境のネイティブSDKを介して利用できます。 すべてのAPIはRESTからも利用できます。



背景
90年代後半のパーソナルPCの最初の大量配布は、このような概念の作成を必要としませんでした。 そして、グローバルテクノロジークラスター全体のソフトウェアおよびハードウェアインフラストラクチャの開発から20年後（パーソナルステーションとサーバーステーションのパフォーマンス、安定した高速通信チャネル、ワイヤレスネットワークとモバイルネットワークのスループットを待つことはできません ）、クライアントサーバーアーキテクチャとソフトウェアモデルによるクラウドの普及が栄えました-likeサービス（soft-for-rent）。 パワフルでスマートなモバイルデバイスはすでにここにあります 。 企業の世界の「非効率性」が今日の最悪の罪であるように、将来、インターネットに接続されたデバイスは隠蔽の類似体として機能します（ところで、マトリックスもtooします）。すべてがリアルタイムであり、データネットワークに接続されたほぼすべてのデバイスにサーバーが必要ですよね？



そのようなプラットフォームが表示されるのはなぜですか？
  • 誰かが宇宙の優しさのエントロピーを高めたいと思っており、世界と良いプロジェクトを共有することにしました（オープンソースのオープンソース）
  • 前のポイントを実行した後、誰かが正常に機能するようにサポートすることでお金を稼ぎたい（商用のオープンソース）
  • 誰かが繰り返しのアクションを何度も繰り返すことにうんざりしているため、このキッチンをすべて自動化することにしました（繰り返しのコードはすべて自動化され、時間の経過とともに減価します）
  • コストの問題で、誰かの既製のサービスを使用する方が、書き直すよりも安価です（サービスとしてのソフトにより、誰でも最初から投資なしでエンタープライズレベルのソフトウェアにアクセスできます）。




アイデアは、既製のサーバーサービスがあり、それらのセットが一緒にあらゆるプロジェクトに必要なユニバーサルクロスプラットフォームバックエンドを追加するということです。 また、これにより、サーバーバックエンドの作成とサポートの頭痛の種を取り除くことができます。



試す理由の詳細
対処する必要を避ける

  • アプリケーションサーバー
  • データベース
  • クライアントサーバーライブラリ
  • 管理領域を書く
  • IPAを設計する
  • そしてホスティング。




しかし、 ホスティング、サーバー、および各アプリケーションまたは顧客をさらにサポートするオタクチームを心配する必要がない場合はどうしてですか。 誰もがアプリケーションを作成するのが大好きで、退屈なサーバーコードを選択してサーバー側を管理するのは退屈で退屈です。



理由：

BaaSの潜在的な業界標準

  • プロトタイプ
  • モバイルアプリ
  • クロスプラットフォームアプリケーション（スマートフォン、タブレット、PCのユーザーベース全体に同じプラットフォームを使用できます（ブラウザーおよびデスクトップアプリケーション経由）
  • エンタープライズアプリケーション（および既存のバックオフィスシステムと新しいモバイルアプリケーションの統合）。


誰のために：

開発者、スタートアップ、デジタルエージェンシーおよびコンテンツプロバイダー、システムインテグレーター、アプリケーションパブリッシャー、企業。



プラットフォーム機能：



ユーザー管理API-クロスプラットフォームでのアプリケーション登録とユーザーログイン、パスワード回復、アカウント更新を提供します。



説明：ユーザー管理サービスは、ユーザー登録とその許可という2つの主な機能を担当します。 ユーザーが存在するかどうかを確認したり、セッションを維持したり、さまざまなイベントのメールを送信したり、ユーザーのアクセス制御を作成したりするためのチェックを書く必要がなくなりました。 このタスクには多くのビジネスルールがあり、すべてのニーズを満たすためにユーザー管理を非常に柔軟に構成できます。



コード例：

Android 
{ Backendless.UserService.login( ”james.bond@mi6.co.uk", ”i.am.bond", asyncCallback ); }


iOS 
 { [backendless.userService login: @”james.bond@mi6.co.uk" password:@”i.am.bond" responder:responder]; }


Javascript 
 { Backendless.UserService.login( ”james.bond@mi6.co.uk", ”i.am.bond", responder ); }


データサービスAPI （リレーショナルデータ）-ユーザーデータのストレージを提供します。 アプリケーションは、データを保存、更新、削除し、検索を実行できます。



説明： データベースの知識がなくてもサービスを使用できます。 このプラットフォームでは、APIを使用して独自のテーブルとテーブル間の関係を作成できます。 クライアントでデータ構造を作成するだけで十分です; SDKメソッドが呼び出されると、サーバーは必要なテーブルを作成し、それらをリンクします。 または、Webコンソールを介して直感的なユーザーインターフェイスを使用して実行できます。



コード例：

Android 
 { Person person = new Person( "Bob", 35, "bobby@gmail.com"); Backendless.Persistence.of( Person ).save( person, asyncCallback ); }


iOS 
 { Person *person = [Person new]; person.name = @"Bob"; person.email = @"bobby@gmail.com"; id <IDataStore> dataStore = [backendless.persistenceService of: [Person class]]; [dataStore save:person responder:resp]; }


Javascript 
 { var person = new Person( "Bob", 35, "bobby@gmail.com"); Backendless.Persistence.of( Person ).save( person, responder ); }


パブリッシュ/サブスクライブメッセージングAPI （メッセージ）-クライアントアプリケーションは、パブリッシャーとサブスクライバーを作成することにより、リアルタイムでメッセージを交換できます。



説明： メッセージには、任意の（個別の）データを使用できます。 この機能は、データブロードキャストまたはp2p配信を必要とするゲーム、チャット、およびアプリケーションを開発するときに役立ちます。 プッシュ通知とアプリ内メッセージをモバイルユーザーに送信する。



アプリケーションの例：チャット、クライアントアプリケーションの同期、何かに関するユーザーへの通知。



コード例：

Android 
 { Backendless.Messaging .subscribe( ”myChannel", methodCallback, subscriptionCallback ); }


iOS 
 { subscription = [backendless.messagingService subscribe:@"myChannel" subscriptionResponder:responder subscriptionOptions:subscriptionOptions]; }


Javascript 
 { Backendless.Messaging .subscribe( "private_channel", methodResponder, subscriptionResponder ); }


プッシュ通知APIはメッセージのサブセクションですが、プッシュ通知のようなモバイルデバイスへのメッセージ配信のネイティブ統合を備えています。 iOS、Android、およびWindows Phoneでサポートされています。



コード例：

Android 
 { Backendless.Messaging .publish( "Hello!", new DeliveryOptions( PushBroadcastMask .ANDROID )); }


iOS 
 { PublishOptions *p = [PublishOptions new]; [p addHeader:@"Name" value:@"Anonymous"]; MessageStatus *res = [backendless.messagingService publish:@"myChannel" message:@"Hello!" publishOptions:p deliveryOptions:[DeliveryOptions deliveryOptionsForNotification:PUSHONLY]]; }


Javascript 
 { Backendless.Messaging .publish( "Hello!", new DeliveryOptions( PushBroadcastMask .ANDROID )); }


ジオロケーションAPI （ジオロケーション）-アプリケーションは、サーバー上の追加のメタデータで地理座標（ジオポイント）を登録し、その後、指定された半径または正方形内のメタデータによって他のポイントの検索クエリを実行できます。



日常生活におけるアプリケーションの可能な例：

  • 親-子供がいる場所の写真を自動的にリンクするサービス。
  • 配達サービスと宅配便-受取人のオブジェクトの写真。
  • 運転手-道路の状態と半径内の正確な天気。
  • 観光客-地域の観光スポット;
  • 自動旅行者-近くのレストラン/ホテルのリストとメニューと価格、無料の場所;
  • 友人が近くにいる場合、友人の場所を示す。
  • ユーザーは、カテゴリを示す投稿を「レストラン」（レストランのインテリアの写真を追加）またはロマンチックな場所（風景の場所の写真を追加）に作成します。 その後、他のユーザーは自分の周りのロマンチックな場所の表示のみをフィルタリングできます。
  • 指定された半径のユーザーの位置を知るアプリケーション。 ある種の状況またはイベントを報告し、この状況が特定の半径内に収まった場合、ユーザーは通知（交通渋滞または道路上の交通警官、インシデントに関する情報）を受け取ります。


コード例：

Android 
 { Backendless.Geo.getPoints( new BackendlessGeoQuery( "city", ”Kiev" ), asyncCallback ); }


iOS 
 { BackendlessGeoQuery *query = [BackendlessGeoQuery query]; [query metadata:[NSDictionary DictionaryWithObjectsAndKeys: @"Moscow", @"city", nil]]; [backendless.geoService getPoints:query responder:responder]; }


Javascript 
 { Backendless.Geo.getPoints( new BackendlessGeoQuery( "city", ”Dallas" ), responder ); }


Media Services API （メディアストリーム）-「オンデマンド再生」とライブストリーミングビデオおよびオーディオを提供するサービスのセット。 アプリケーションは、録画またはライブ放送用に、ビデオカメラとマイクからのストリームを公開できます。 記録されたメディア（ビデオまたはオーディオ）とライブストリームは、他のクライアントで再生できます。



説明：メディアサービスを使用すると、クライアントアプリケーションからサーバーにビデオまたはオーディオをストリーミングし、このストリームを他のデバイスで再生できます。 サービスファイルを使用してダウンロードしたメディアコンテンツを再生することもできます。



アプリケーション例：ビデオ会議、ビデオチャット、すべてのデバイスから同時に、またはすべてのデバイスへのライブビデオブロードキャスト（電話、タブレット、PCカメラから）、ビデオまたはサーバーへの直接録音、ライブオーディオブロードキャスト（ラジオ、音楽）



コード例：

iOS 
 { MediaPublishOptions *options = [MediapublishOptions recordStream:self.preview]; [backendless.mediaService publishStream:@”myVideoChannel" tube:@”Funny Dance" options:options responder:resp]; }


ファイルサービスAPI （コンテンツストレージ）-ファイルまたはデータブロックのダウンロード、共有、およびダウンロードをサポートします。 ファイル/データは、リレーショナルデータサービス、メッセージ、およびジオポイントからの永続データレコードにリンクできます。



説明：ファイル管理サービスにより、コンソールまたはクライアントSDKを介してファイルをアップロードおよびアクセスできます。



アプリケーションの例：コンテンツの保存と管理、ダウンロードしたビデオのストリーミング、アプリケーションを更新する必要のないコンテンツの更新。



コード例：

Android 
 { Backendless.Files.upload( file, path, asyncCallback ); }


iOS 
 { [backendless.fileService upload:path content:content responder:responder]; }


Javascript 
 { Backendless.Files.upload( fileList, ”myFolder"), responder ); }


このプラットフォームが他のプラットフォームより優れているのはなぜですか？



バージョン管理 -バージョン間で共通のデータ/テーブルを使用-アプリケーションを作成することにより、公式リリースを作成でき、現時点では同じアプリケーションの別のバージョンで動作します。



オーディオとビデオのライブストリーミング：ビデオチャット、ライブブロードキャスト、音楽、ラジオ、クリップなど、好きなものを接続できます。



メッセージのフィルタリング -サブトピックによるメッセージのフィルタリングは機能します。また、SQL形式のフィルター、いわゆるセレクターを設定することもできます。



Flex / Airクライアントがサポートされています-ActionScript用のSDKがあります。 フラッシュを使用すると、アプリケーションはどのデバイスでも同じように見え、タンバリンと踊ることはありません。



箱入りソリューション -社内のバックエンドレスを箱から取り出して入手できます。 大企業のクライアントは、自分用のサーバーにのみプラットフォームを展開できます。



柔軟な価格設定と「大胆な」無料プラン -フリーミアムには、API無制限の呼び出し、2 GBのディスク容量、200,000のパブリッシュ/サブスクライブメッセージ、200,000のプッシュ通知が含まれます。 支払いが必要な場合は、直接使用されるものに対してのみ。 便利な電卓を使用して、 ここでより詳細に表示および計算できます。 さらに、プラットフォームはベータ版ですが、制限は正式な規則です。



APIとWebコンソールは、シンプルさと利便性に最大限の注意が払われているため、開発者向けに慎重に研磨されています。



自動 - スケーラビリティ -プラットフォームはAmazonのインフラストラクチャ上にあり、負荷がかかった場合（所定の場所とメモリ使用量）に自動的にスケーリングします。 限界を超えると、リクエストの処理時に追加の仮想マシンが起動されます。



開始方法：FB認証を介して開発者のWebコンソールに直接登録するか、直接アクセスして、 KFORの例をダウンロードし、各KFORのキーを処方し、APIの呼び出しを記述するだけです。クラウドバックエンドは自由に使用できます。
