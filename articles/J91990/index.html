WordPressデータベースの8つの便利なトリック

WordPressに敬意を表します。 これは、ブログや完全なサイトを構築するための本当に素晴らしいシステムです。 素敵で論理的な管理パネル（特にバージョン2.7以降、おいしいもの）、かなりの数のかなり高品質のテンプレート...そして実際、すべてが非常に素晴らしく、比較的シンプルです。 このシステムに敬意を表し、ユーザーに役立つ何かをしたいと思います。



過去10年間で、MySQLはネット上で非常に人気が高まっています。 各WordPressブログは、すべての投稿、設定、コメントなどを保存するMySQLデータベースに基づいています。



プラグインやいわゆるハック（「コード挿入」を好む）でも問題を解決できますが、SQLクエリをphpMyAdminに入力するか、SSH経由でデータベースに直接入力する以外に方法がない場合があります。 それでは、WordPressデータベースの8つの便利なトリックを見てみましょう。



1.データベースのバックアップを作成する



問題 。 この記事のヒントはテスト済みですが、MySQLデータベースのバックアップを作成する前に実践するべきではありません（決してわかりません...）



解決策 。 データベースを手動でバックアップするには、次の簡単な手順に従います。



1.最初にphpMyAdminにログインし、そこでWordPressデータベースを選択する必要があります。



2.水平メニューにある「エクスポート」ボタンをクリックします。



3.データ圧縮方法（私は個人的にgzipを使用しています）を選択し、「実行」ボタンをクリックします。



4.バックアップをダウンロードするかどうかをブラウザが尋ねます。 もちろん、「はい」としっかり言って、コンピューターのどこかにファイルを保存します。



ご注意 特別なWP-DB-Backupプラグインを使用すると、WordPressデータベースのバックアップコピーを作成する方がはるかに便利です。 WordPressユーザーは、なんらかの理由でまだプラグインをインストールしていない場合、ためらうことなくこのプラグインを今すぐインストールできます。



2.レコードリビジョンの一括削除



問題 。 ポストリビジョンは、バージョン2.6以降の新しいWordPress機能です。 これは非常に便利であり、MySQLデータベースのサイズを増やすこともできます。 もちろん、管理者パネルから投稿リビジョンを手動で削除できます。 しかし、それは非常に長く退屈です。 より良い解決策があります。



解決策 。 そして、問題の解決策は非常に簡単です。私たちはバッチです。つまり、すべてが一気に行われ、単純なSQLクエリを使用して投稿リビジョンを削除します。 多数のエントリがある場合、結果は驚くべきものになります。 データベースが2回失われる可能性があります！



1. phpMyAdminにログインし、そこでWordPressデータベースを選択する必要があります。



2.次に、「SQL」ボタンをクリックします。 次のクエリを挿入する必要があるウィンドウが表示されます。



DELETE FROM wp_posts WHERE post_type = "revision";







3.それだけです！ レコードの数に応じて、貴重な時間を大幅に節約し、ベースをクリーンアップしました。



コードの説明 。 wp_postsテーブルにはpost_typeというフィールドがあります。 このフィールドには、 post 、 pageまたはrevisionなど、多くの意味があります 。 レコードのリビジョンを削除する場合は、コマンドを実行して、 リビジョン値があるpost_typeフィールドのwp_postsテーブル内のすべての値を削除します。 方法は次のとおりです。



3. 1秒で5000件のスパマーコメントを削除します



問題 。 実話：私の友人は最近彼自身のブログを開き、インターネット上で積極的にそれを促進し始めました。 数週間にわたる集中的な作業の後、彼はネットワークなしで休暇に数日間過ごしました。



彼が家に戻って自分のブログを見たとき、彼は管理を待っていた5000以上の投稿を見ました！ になる方法



解決策 。 幸いなことに、私の友人は、クソスパマーの問題について話してくれました。 私は彼に非常に有用なトリックを示したので、彼はすでに手動でスパムをチェックして除去するのに45分を費やしていました



1. phpMyAdminにログインし、そこでWordPressデータベースを選択します。



2. [SQL]ボタンをクリックします。 次のクエリを挿入する必要があるウィンドウが表示されます。



DELETE from wp_comments WHERE comment_approved = '0';







3.そして、スパマーに別れを告げます！ 清潔さと快適さをお楽しみください...



説明 。 wp_commentsテーブルには、ブール値（1または0）を格納するcomment_approvedフィールドが含まれています。 承認済みのコメントの値は1で、モデレートを待機しているコメントは0です。上記のコマンドは、承認されていないコメントを単に削除します。 すべてがシンプルです。



しかし、注意してください！ このソリューションは、無数のスパムコメントを自動的に削除するのに最適ですが、承認されていない通常のコメントも削除します。 Akismetのようなプラグインをまだ使用していない場合は、ブログのスパムを防ぐために始めましょう。



4.レコード属性を変更する方法



問題 。 WordPressをインストールすると、デフォルトの管理者アカウントが作成されます。 一部のブロガーは、このアカウントを使用して投稿を作成するというミスを犯し、それが何らかの形で顔の見えないことに気付くまで失敗します。



解決策 。 各エントリの作成者の属性を変更するには時間がかかります。 幸いなことに、SQLは私たちを助けることができます：



1. phpMyAdminにログインし、そこでWordPressデータベースを選択します。



2.最初に、正しいユーザーIDを決定する必要があります。 「SQL」ボタンをクリックしてください。 次のクエリを挿入する必要があるウィンドウが表示されます。



SELECT ID, display_name FROM wp_users;







3. phpMyAdminは、WordPressユーザーに関連付けられている「識別子」のリストを表示します。 ちなみに、NEW_AUTHOR_IDは最後に作成された作成者のID、OLD_AUTHOR_IDは元の管理者アカウントのIDです。



4.「識別子」NEW_AUTHOR_IDおよびOLD_AUTHOR_IDを特定したら、次のコマンドを実行します。



UPDATE wp_posts SET post_author=NEW_AUTHOR_ID WHERE post_author=OLD_AUTHOR_ID;







5.それだけです。 管理者アカウントにリンクされたすべてのエントリは、選択したユーザーのプロパティになります。



5.パスワードのリセット



問題 。 ブログを保護するために、多くの場合、7 * KoF5i8_などの強力なパスワードを選択します。 もちろんこれは称賛に値しますが、誰もが管理者がパスワードを忘れる方法について多くの話を聞いています:)



解決策 。 パスワードを忘れた場合、WordPressはパスワードをリセットするためのリンクをメールで送信できます。 ただし、WordPressデータベースで指定されている石鹸にアクセスできない場合、または問題が何らかの方法で解決できると思われる場合は、次の方法で「ハッキング」することができます。



1. phpMyAdminにログインし、そこでWordPressデータベースを選択して、SQLウィンドウを開きます。



2.次のコマンドを入力します（ユーザー名が「admin」であることを考慮して）：



UPDATE 'wp_users' SET 'user_pass' = MD5('PASSWORD') WHERE 'wp_users'.'user_login' = 'admin' LIMIT 1;







3.それですべてです。 パスワードは、「PASSWORD」とマークされた場所で指定したパスワードに正常に更新されます。



説明 。 ユーザーパスワードはwp_usersテーブルに格納されます 。 もちろん、MD5ハッシュは表示から保護するために使用されます。



SQLクエリ「UPDATE」を送信し、組み込みのMySQL関数MD5（）を使用してパスワードをMD5に変換し、更新しました。 「WHERE」を使用すると、管理者パスワードのみが更新されます。 同じリクエストですが、「WHERE」パラメータを使用せずに、データベース内のすべてのパスワードを更新します！



6.ドメイン名を変更する



問題 。 これはお勧めしませんが、ある時点でブログのドメイン名を変更し、同時にすべてのデータを保存したい場合があります。 WordPressはドメイン名をデータベースに保存するため、新しいドメインとWordPressブログをリンクするには、データベースをわずかに変更する必要があります。



解決策 。



1.ご想像のとおり、phpMyAdminにログインし、そこでWordPressデータベースを選択して、SQLウィンドウを開きます。



2. WordPress URLを変更するには、次のコマンドを実行します。



UPDATE wp_options SET option_value = replace(option_value, 'http://www.oldsite.com', 'http://www.newsite.com') WHERE option_name = 'home' OR option_name = 'siteurl';







3.次に、各エントリの相対URL（GUID）を置き換える必要があります。 次のコマンドでこれを行います。



UPDATE wp_posts SET guid = replace(guid, 'http://www.oldsite.com','http://www.newsite.com');







4.これでほぼ終わりです。 説得力のあるエンディングのためにwp_postsテーブル内の絶対URLを見つけて置き換えるだけです。



UPDATE wp_posts SET post_content = replace(post_content, 'http://www.oldsite.com', 'http://www.newsite.com');







5.しかし、これはすでに終わりです。 新しいURLを使用して、ブログの管理パネルにアクセスできます。



7.ブログのSQLクエリの数を表示します。



問題 。 ブログの読み込み時間を最適化しようとすると、データベースクエリの数を知ることが非常に役立ちます。 リクエストの数を減らすために最初にすることは、ページで発生するリクエストの数を調べることです。



解決策。 楽しい：phpMyAdminにアクセスする必要はありません:) footer.phpファイルを開いて編集し（トピックに間違いありません）、次のコード行を追加するだけです。



<?php if (is_user_logged_in()) { ?>

<?php echo get_num_queries(); ?> <?php timer_stop(1); ?> .

<?php } ?>







ファイルを保存して、ブログにアクセスしてください。 「地下室」には、WordPressデータベースへのクエリの数と、クエリの作成にかかった時間が表示されます。



ご注意 多くのWordPressユーザーがこの素晴らしい機会に気付いていないという印象を受けました。 get_num_queries（）関数は、ページのロード時に生成されたリクエストの数を返します。



ブログのゲストと検索ボットはこの情報を知る必要がないため、上記のコードはログインしたユーザーのみにリクエストの数を表示することに注意してください。 ただし、コードから条件付きif（is_user_logged_in（））ステートメントを削除するだけで、マッピングを公開できます。



8.データベースを復元する



問題 。 たとえば、ハックや更新の問題などの何らかの理由で、ブログのデータが失われたり、絶望的に破損したりすることがあります。 バックアップコピーがある場合（真実は正しいですか？）、それをWordPressデータベースにインポートする必要があります。 そして、すべてがうまくいきます。 最も可能性が高い。



解決策 。



1. phpMyAdminにログインし、そこでWordPressデータベースを選択します。



2.水平メニューの「インポート」ボタンをクリックします。



3. [参照]ボタンをクリックして、ディスクからデータベースの最新のコピーを選択します。



4. [実行]ボタンをクリックします（実行）。 すべてがうまくいき、神があなたの味方になったら、データベースは再び完全に機能します。
