オープンデータを使用して所得申告を解析する方法

2年目は、ロシア連邦の州の公開データに携わり、政府機関と協力して、データがどのように表示されるかについて興味深い話を始める時が来ました。 ただし、本日は、開発者にとってより身近な分野、つまり「宣言子」プロジェクトのデータ解析と、オープンデータがもたらす予期しない利点についてお話します。







「宣言者」は、公務員の収入と財産の宣言の絶えず更新されるデータベースです：代理人、役人、裁判官、地域および市当局の代表者、他の機関、国営企業および国有企業。 このプロジェクトは、メディア、公的統制に関与する活動家、および研究者の参照拠点として機能します。

ロシアでは、収入データは100万人以上によって公開されるべきです。



興味深い事実：収入宣言の配置に関するウェブサイトには統一されたルールがあり（特に、それらは常に「腐敗防止」セクションにあります）、ロシア連邦労働社会保護省がすべての責任を負っています。 宣言の大量配布は5月に行われます。 さらに、労働省は、情報を掲載する義務があるすべてのサイトを例外なく監視するのに1か月しかありません。 監視は手動で行われます。



宣言の発行に関連するいくつかの問題があります。



  • 各機関は独自のウェブサイトでこれを行います。
  • 宣言を公開するための単一の標準はありません。
  • 情報は公開後すぐに削除される場合があります。
  • 法律により公開されていますが、宣言はまだ機械可読形式で公開されていません。


マテリエル
連邦国家機関、ロシア連邦および組織の構成団体の州当局、および公開のためにこの情報をロシアのマスメディアに提供する特定のカテゴリの個人および家族のメンバーの収入、費用、財産および財産義務に関する情報を連邦政府機関の公式ウェブサイトに掲載する手順の条項2に従って、 2013年7月8日のロシア連邦大統領令第613号「腐敗との戦いの問題」 、連邦政府機関により承認 当局、ロシア連邦中央銀行、ロシア連邦年金基金、ロシア連邦社会保険基金、連邦強制医療保険基金、国営企業（企業）、連邦法に基づいて作成されたその他の組織は、過去1年間の収入に関する情報を公開する義務があります。



関心のある人は、労働省530nの順に詳細を読むことができます。 要するに 、「腐敗防止」セクションは、メインページから1クリックで配置し、ハードコードされた名前を持ついくつかの必須サブセクションを含む必要があります。財産および財産義務」。 政府職員の損益計算書が配置されるのは、.doc、.docx、.excel、.rtf（要件の15項）の形式を含む表形式のアクセス制限なしです。



同時に、ドキュメントで直接述べられているように、「ファイルのテキストを検索し、テキストのフラグメントをコピーできる必要があります」-これにより、パーサーの作成が可能になります。



宣言について



各公務員の損益計算書には、所有、使用、不動産に関する情報、車両に関する情報、収入および収入源に関する情報が含まれています。 宣言には、役人だけでなく、その家族のメンバーも記載されています。 運がよければ、要約ファイルは、労働省が所得情報の形式で提案したテンプレートに従って作成されます 。これにより、ユニバーサルパーサーの作成作業が大幅に簡素化されます。







最初の関心の対象は、教育科学省でした。 306人を対象とした2014年の連邦公務員の所得申告のためのトライアルパーサーは 、迅速かつかなり苦痛なく書かれました。 その後、私たちは通常、最も多くの研究者の要求を引き起こすもの、つまり大学の学長の収入に関する情報に進むことにしました。 そして、 2014年の文部科学省の下位機関の宣言の難しさが始まりました 。



簡単な分析により、このファイル内のレクターのみが272人であり、明らかに何らかの方法でグループ化する必要があることが示されました。 彼らは地域ごとにグループ化することにしました。 ただし、宣言は機関の位置と名前のみを示しています。







ほぼ300のユニークな高等教育機関は、手作業で検索することを望んでいませんでした。 これは、 「認可された教育プログラムの下で教育活動に従事する組織の登録簿」と呼ばれるロソブルナゾールのオープンデータが役に立ちました（ 更新：名前とリンクは2016年5 月24日に修正されました ）。 レジストリは非常に有益ですが、かなり複雑な形式であり、別の記事に値します。 Rosobrnadzorに敬意を表して、政府機関では伝統的なcsvではなくxml形式で提示され、構造にエラーが含まれていないため、予備処理なしでデータベースとして使用することができました。 （ 2016年5 月16日現在のリンクは一時的に機能していないため、レジストリはここで引き続き取得できます 。 更新：Rosobrnadzorは2016年5 月24日にリンクを修復しました ）。



タスクについて



そのため、最初のタスク：損益計算書からレクターのデータを抽出し、レクターの地域を決定し、フィラープラグイン用の特別な形式の xmlファイルを生成します。







プラグインは、サイトでのユーザーアクションを模倣し、フォームに自動的に入力します。 そして、テストプロセス中に判明したように、彼は一度に処理できるレコードの数に制限があります...



入力には、1,561人の役人の収入に関するデータを含む.docファイルがあります。 職員だけでなく家族のデータも必要なので、実際には3347人の情報を処理する必要があります。

そのため、xmlのXPathは非常に高速に検索され、部分的な一致を見つけることができますが、悲しいかな-文字の大文字小文字に完全に一致しています。 検索機能を無効にすると、問題は解消され、同時に速度も低下します。



最初のステップは、繰り返しの部分-機関のタイプ-を削除し、実際の名前のみを残すことでした。 そして、ここでの発見が待っていました：高等教育機関のFSBEI（およびFSBEI、FSBEI）に加えて、「インクルーシブ高等教育」、つまりVOのFSBEIの機関があるように見えましたか？ 「連邦政府の高等教育予算機関」に置き換えられました-違いを見つけることができますか？



その結果、正規表現で次のような構造が得られました。 



 orgname = Regex.Replace(orgname, @"(.*)((|)\s)((|)?)?", ""); orgname = Regex.Replace(orgname, @"([-]*\s|[-]*\s)?([-]*\s|[-]*\s)?(.*)?([-]*\s|[-]*\s)([-]*\s|[-]*\s)?([-]*\s|[-]*\s|[-]*\s|[-]*\s)([-]*\s|[-]*\s)?([-]*|[-]*)", ""); orgname = Regex.Replace(orgname, @"([-]*\s|[-]*\s)?([-]*\s|[-]*\s)?(.*)?([-]*\s|[-]*\s)", "");


さらに、ここでのチームの順序は非常に重要です。 さて、引用符との戦い： 



 if (orgname.Contains("<<")) orgname = Regex.Replace(orgname, @"(.*<<)(.+)(>>.*)", "$2"); if (orgname.Contains('«')) orgname = Regex.Replace(orgname, @"(.*«)(.+)(».*)", "$2"); if (orgname.Contains('“')) orgname = Regex.Replace(orgname, @"(.*“)(.+)(”.*)", "$2");


大きな問題は、誰かにちなんで名付けられた大学にあることが判明しました。



  • 「them。」と「name」の変形は、異なるレジスタにあります。
  • 名前自体は、任意のレジスタの任意のオプションで、任意の数のスペースを使用して（またはスペースなしで）書き込むことができます。たとえば、IM.I。KANTA、Ivan Fedorovにちなんで命名されました。 V.I. N.I.ロバチェフスキーにちなんで名付けられたウリヤノフ（レーニン）


スペルが予測できないため、「名前」を全体として削除する方が簡単でした。 



 orgname = Regex.Replace(orgname, "(.*)", ""); orgname = Regex.Replace(orgname, @"(\..*)", "");


幸いなことに、大学の名前はその独自性を失わなかった。

 Match tempmatch = Regex.Match(orgname, @"[-]{2,}"); tempname = orgname.Substring(tempmatch.Index, tempmatch.Length);
