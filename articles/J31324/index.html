CSSの未来：Eric Meyerのインタビュー

Eric Meyerは、HTMLおよびCSS標準の分野で有名な専門家であり、1993年以来Webテクノロジーの分野で働いています。 彼はカスケードスタイルシートなどの本を出版しています。 詳細ガイド「（O'Reilly）」、「CSS 2.0開発者ガイド」（Osborne / McGraw-Hill）、およびCSS（New Riders）のEric Meyer。 また、Web標準、CSS、Webデザインに関する多数の会議に参加しています。



彼は、Dreamweaver CS3 BibleとCSS Hacks and Filtersのベストセラー作家であるJoseph Loweryにインタビューされています。 彼は現在、WebAssistのマーケティング部門を運営しています。



ジョセフ・ローリー投稿

オリジナル： http : //adobe.com/newsletters/edge/december2007/articles/article5/index.html

翻訳者： Alexander Musaev



Joseph Lowery：このインタビューに備えて、1999年6月に書かれたCSSブラウザーサポートとこの技術の将来についての記事をレビューしました。 あなたの経験を疑う人はいませんが、そのような予測をどのように正確に行うことができますか？



Eric Meyer：元の計画によれば、CSSテクノロジーはモジュールに分割され、各モジュールは特定の段階で形成されました。 ここでの難点は、原則として、これらの時間間隔のそれぞれが大幅に禁止されていることです。 最も単純なモジュールでも、長い開発の歴史があります。



私はかつて革新の一部は地平線上にあるだけだと主張していましたが、今では実装にかなり近づいています。 たとえば、要素の透明度プロパティは、さまざまな方法ではありますが、多くのブラウザで既に実装されています。 一方、テキストを列に分割する可能性は、非常に遠い未来に残っています。



ブラウザーの開発もこの記事で説明されているようにほぼ常に進化しましたが、もちろん、Netscape 5の死とMozillaの台頭は予見できませんでした。 私はそのような予測が当時一般的に可能であったことを疑います。



JL：現在に戻りましょう。 Microsoft Internet Explorer 7の登場は、その開発の大きな飛躍でしたか？



EM：それはもっと大きなジャンプのようなものでした。 つまり、新しいバージョンでは、多くのIE 6の欠陥が修正されました（たとえば、パーサーエラー、レイアウトの問題など）。 さらに、多くの基本的なCSS 2機能（主にセレクターと厳密な配置）を追加しました。 IE 7では、このすべてが一度に表示され、これは私たち全員にとって朗報でした。



IE 6のようにIE 7の進化が定着するのか、このブラウザーが競合他社からの開発でますます遅れるのか、その機能の拡張が続くのか、非常に興味深いものになっています。 2番目のオプションが本当に欲しいです。



DL：私の知る限り、CSS 3は現在W3Cで議論および開発中です。 この標準からデザイナーが期待できる新機能とセレクターを一般的な用語で説明できますか？



EM：あなたは誤解しました。 彼らが現在議論し開発しているのは、個別のCSSモジュールであり、それぞれがテクノロジーの開発における特定の段階を指します。



CSS 2の場合のように、CSS 3標準が表示されない可能性があります。 そして、それが良いか悪いかはまだ最終的な決定には至っていませんが、最後の選択肢に傾倒しています。 このようなイベントの開発により、共通の目標が1つなくなります。



CSS 2の個々の部分について私たちがどう考えているかに関係なく、標準は同じままです。 これは議論の対象であり、話し合うことができるトピックです。 CSS 2の詳細は互いに依存しています。 モジュール性の条件では、この整合性は消滅します。 相互依存関係を理解するのははるかに困難です。



将来的に私たちを待っている新機能については、いくつかの新しいセレクターを引用できます。 特に:nth-child()



:last-of-type



など。 また表示されます:not()



では:not()



、有用ですが、その説明は今私を怖がらせます。 それらに加えて、私自身もRGBAカラー記述形式が大好きです。「A」は「アルファ」を意味し、HLSおよびHLSA形式（色相-明度-彩度モデルの場合-翻訳者のメモ）も同様です。 これらの機能をできるだけ早くブラウザに実装したいと考えています。



上記のすべてに加えて、幅広い機能を備えたエンコーディングサポートモジュールが計画されています。これは、Web上にASCIIアートが存在する可能性をまだ夢見ている人にとっては素晴らしいはずです。



明確に質問に答えるのが難しい主な理由は、CSS開発がブラウザ開発者に大きく依存しており、彼らが計画を明かす習慣がないことです。 ほとんどの場合、セレクターが最初に表示されます。 たとえば、Opera 9.5では、セレクターモジュールが現在存在する形式で完全にサポートされることを約束します。 ブラウザで最も可能性の高い技術革新は、ほとんどの部分が実装が簡単だと思う-これらは同じセレクター、ユニットなどです。



DL： CSS仕様は、今日大きな論争の問題のようです。 これの主な理由は何ですか？ 元W3Cメンバーとして、これらの論争の結果として考えられることについてどう思いますか？



EM： CSSのモジュール性のため、あなたの質問に答えるのは難しいです。 モジュールが異なれば、意見が一致しない独自の理由があります。 また、2004年にCSS開発チームを辞めたため、これを判断することも困難です。



私自身の経験に基づいて、CSSに追加するのに役立つもの、追加するものがすでに規定されているもの、開発者が実際に実装するものの間で主な矛盾が生じると想定できます。 かなりの混乱を伴う3つの戦線での戦いが判明しました。



現在起こっていることは、技術開発プロセスにとって絶対に正常なことですが、ウェブにとって良いことだとは言いません。



DL：仕様が推奨された後、ブラウザに新しい機能が表示される順序を予測できますか？ CSSハック時代の新しい夜明けが来ていますか？



EM：一部のブラウザには新機能が導入されていますが、他のブラウザには導入されていない場合でも、 プログレッシブ開発の原則を既に学習しています：高度な機能は、それらをサポートするブラウザのユーザーにプラスの効果をもたらすように使用する必要があります。 この場合、古いブラウザを使用しているユーザーに関連するユーザー特性の低下を回避する必要があります。 良い例は、生成されたコンテンツを使用して、URLリンクをテキストに挿入してから印刷することです（詳細については、「 A List Apart 」の記事「 CSSデザイン：印刷の準備 」を参照してください）。



CSSハックの数は増えると思いますが、これの主な理由は、これらのハックを見つけて使用することに対する人々の関心でしょう。 このプロセスは、モザイクを拾うようなものです。



DL： Webページ開発プログラムのCSSレンダリング機能は積極的に開発されています。 特に、Dreamweaverで。 これはデザイナーの仕事の質に影響すると思いますか？



EM：もちろんです。 私は多くの若いデザイナーと話をしましたが、彼らはすでにページレイアウトにテーブルを使用する理由を把握することさえできません。 彼らはちょうどそれを取得しません。 このような業界の変化は、Webページを作成するためのツールの機能とブラウザーの現在の開発方法によって大きく正当化されます（両方の要因も同様に重要です）。



DL：当社は最近、 Eric MeyerのCSS Sculptorと呼ばれるDreamweaver拡張機能を開発するためにあなたと提携しました。 あなたの名前がこの名前を有名にした後、あなたの人生は良くなりましたか？



EM：はい、カンファレンスで彼らは今、スタンディング・オベーションを迎えてくれました。 また、（誰にも言わないで）最近、このビジネスで稼いだお金を貯めるために地下に巨大なプールを作りました。 スクルージ・マクダックのイメージを具現化することは、私の大切な夢に私をもたらします。



他の重要な変更は発生していません。



JL：真剣に、CSS Sculptorの外観はDreamweaverユーザーに影響を与えたと思いますか？ そして、もしそうなら、どのように？



EM：これにより、CSSを使用したページマークアップの開発がスピードアップすることを願っています。 CSS Sculptorは、私が自分で使用するソフトウェア製品の1つです。 そして、これは彼の方向に大きな賛辞です。 私は、手動コーディングを好むWebデザイナーの古い学校に所属しています（この分野で14年の経験があります）。



Sculptorインターフェースの主な利点の1つは、ほとんどすべてのCSSプロパティが非常にコンパクトに表示されることです。さらに、ドキュメントの構造とそれに適用されるスタイルシートの直感的な表現を提供します。 これらすべてにより、デザイナーはCSSがWebページとどのように統合されるかをよく理解できます。 CSS Sculptorは、開発だけでなくトレーニングにも適したツールです。 そして、それは私を喜ばせます。



JL： DreamweaverでのCSSサポートの開発をどのように見たいですか？



EM：このサポートが現在のブラウザーバージョンの機能に十分である限り、私に適しています。



JL：始めたばかりのWebデザイナーへのアドバイスは何ですか？ 将来CSSを適用しなければならない人は？



EM：私ができる最も重要なアドバイスは、練習、練習、そしてもう一度練習することです。 これがなければ、読んだ本、記事、仕様の数に違いはありません。 これはすべて、実験によって得られた実際の経験に置き換わるものではありません。



目標を達成するためには、技術の研究に大きな注意を払うことも必要です。 プロのWebデザイナーになりたい人は、ブラシやペイントのアーティストと同じように、HTML、CSS、画像形式に精通している必要があります。 グラフィックレイアウトを取るプロのデザイナーは、それを実現することができるはずです。



デザイナーが常に作成できるはずの同じことを言うべきではありません。 彼らは単にこの能力を所有し、彼らが働く環境の性質を感じ、そしてDreamweaverやSculptorなどのツールから何が得られるかを理解しなければなりません。



JL：どの情報リソースをお勧めしますか？



EM：最も優れたCSSリソースの1つは、このテクノロジーについて説明しているwikiです。 css-discuss.incutio.orgの Webサイトは 、 専門メーリングリストの 5年間の作業中に蓄積された膨大な量の情報を収集しています 。 さまざまなCSSテクニックを使用することの長所と短所の説明、CSSを使用する複雑な側面の簡単な言語の説明、他のリソースへの大量のリンクなどがあります。 これは、CSSについて何かを知りたいときに私自身が頼る最初の情報源です。



ウィキに加えて、 PositionIsEverything.net 、 House of Style Webサイトの資料、 およびRuss Wickleyの短いガイドにも注意を払う必要があります。 複雑なセレクターをデコードするには、 SelectORacleをお勧めします 。 そしてもちろん、Googleのすべてを見通す目でリストを補完する価値があります。
