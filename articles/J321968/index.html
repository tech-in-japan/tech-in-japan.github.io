30分でTwitterボットを作成する方法

しかし、時間を待つ。 最初に、いくつかのアカウントを作成する必要があります-もちろん、まだ持っていない場合。



次のものが必要です。



  • Twitter
  • Cloud9 IDE
  • ヘロク

Heroku変数の設定



ボットがHeroku上にあるので、Twitterキーを保存するための環境変数を追加する必要があります。 実際には、それらが記録されている.envファイルは.gitignoreファイルに示されており、リストされたファイルをHerokuにアップロードしないようにコマンドがgitに送信されます。



したがって、将来GitHubにコードを配置する場合、キーが.envファイルとともに公開されることを心配する必要はありません。自動的に除外されます。



必要なのは、Herokuのアプリケーションのコンソールに移動し、[設定]セクションを選択して、.envファイルからTwitterキーを追加することだけです。 次に、「 構成変数の表示」ボタンをクリックして、適切な値を持つ変数を追加します。 



 CONSUMER_KEY CONSUMER_SECRET ACCESS_TOKEN ACCESS_TOKEN_SECRET


Herokuで変数を設定したら、現在のプロジェクトのconfig.jsファイルに移動します。 次の行を削除する必要があります。 



 require('dotenv').config();


これで、Herokuを再度デプロイする準備がすべて整いました。 コンソールのコマンドは次のようになります。 



 $ git add . $ git commit -m 'add environment variables' $ git push heroku master


次に、コマンドでHerokuのログを確認できます 



 $ heroku logs -t


完了：機能を永久に実行するボットがあります。 まあ、または検索条件を変更することを決定するまで。
