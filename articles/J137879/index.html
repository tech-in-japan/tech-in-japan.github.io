SaaSomではありません...

TeamLabは主にユーザー間のSaaSソリューションに関連付けられていますが、実際にはこれがポータルを展開する唯一の方法ではありません。 SaaSがまず第一に使いやすさである場合、プロジェクトの設立の瞬間からほぼ利用可能なオープンソースソリューションは、製品開発者であるあなたからの高度な独立性です。 ほとんどのポータル所有者はSaaSを好むため、すべての更新は主にこの特定のバージョンに関連しています。 これは同時に、私たちがオープンソースコードの愛好家を忘れてhammerるという意味ではありません。 本日、TeamLabのオープンソースバージョンの更新についてお伝えしたいと思います。



まず、6.0でカレンダーとCRMが追加されました。これは、SaaSでのリリース以来、多くのユーザーから非常に要求されています。 したがって、オンラインドキュメントエディタを除くすべてのサービスモジュールは、自己インストールが可能になりました。 さらに、インストールパッケージがsourceforgeに投稿され、TeamLabをWindowsサーバーに展開できるようになりました。









つまり、TeamLabの展開プロセスは、IISを構成し、データベースを作成し、構成を修正するインストールをダウンロードして起動することになりました。



インストール中に起こること



TeamLabServer.exeのインストール



-マシンにMicrosoft .NET Framework 3.5 SP1またはMySQL ODBC Connector 5.1がない場合、これらのコンポーネントをダウンロードしてインストールします。



-サービスおよびサイトに必要なファイルをインストールディレクトリにコピーします。



-チャットと全文検索のためのシステムサービスTeamLabSvcを作成します。



-MySQLサーバー上で必要なデータベースを形成し、IISでTeamlabポータルを作成すると同時に、すべてのデフォルトバインダーが構成されます。



TeamLabをローカルサーバーにインストールするためのシステム要件



  • プロセッサー：1.7GHz以上
  • RAM：1GB以上
  • 空きディスク容量：少なくとも100Mb
  • オペレーティングシステム：少なくともXP SP3、Microsoft Windows Server 2003およびMicrosoft Windows Server 2008を推奨するMicrosoft Windows
  • Microsoft .NET Framework 3.5 SP1
  • Microsoft IISバージョン5.1以降
  • MySql Serverバージョン5.1以降
  • ブラウザー：IE8以上、Firefox 6以上、Opera 10以上、Safari 5以上、Chrome 10以上




ローカルサーバーにTeamLabをインストールする方法の詳細については、ユーザーマニュアルを参照してください 。
