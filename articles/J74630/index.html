FSXで快適なゲームをプレイするためのシステム要件





FSXはコンピューターリソースを非常に要求します。 そして、どのコンピューターを使用していても、最終的にはすべてが「リアリズムまたはFPS」の妥協に帰着します。 計算は簡単です。追加するアドオンが多く、使用する航空機のモデルが優れているほど、FPSが少なくなります。 したがって、妥当なFPSで高品質で快適な画像を提供できる最小限の（私の意見では）システム要件をすぐに表明したいと思います。
  • OS ：Windows XP、Windows VistaまたはWindows 7（後者をお勧めします）
  • プロセッサー ：Intel Core Duo / Quad（またはそれ以上）、2 GHz以上。 GHzが高ければ高いほど、 FSXのFPSあたりのコア数は実際には効果がありません。 もちろん、AMDも引っ張りますが、私は彼らのプロセッサは初めてです。
  • メモリー ： 2GB以上 。 メモリー速度はFPSに影響を与えるため、頻度とタイミングに注意する方が適切です。
  • ハードディスク ： 少なくとも100 GBの空き容量がある最新のハードドライブ。
  • ビデオカード ：できればnVidiaから、 Radeonのユーザーは時々問題を抱えています。 GeForce 8800 GT以上。 また、SLIを実行したり、クールなマルチコアビデオカードを購入したりするよりも、GPU周波数とメモリを大きくする（256MB-最小、512MB-良好、1024MB-良好）ことをお勧めします。
  • ジョイスティック ： ゲームにはジョイスティックが必要です ！ どちらがあなたのビジネスか、私は個人的にLogitech Extreme 3D Proを使用していますが、非常に満足しています。 理想的には、 舵を取ることさえできます:)
たとえば、現時点で十分な構成を提供できます：Core Duo E6750（2.6 GHz）、2GB RAM（OCZ）、GeForce 8800GT、Hitachi 500GB。 この構成では、FSXの問題：地上では、空港、航空機、気象条件に応じて10〜40 FPSになります。 15から60 FPSの飛行中。 10-15 FPSを恐れていた人にとっては、これは非常にまれであり、20 FPS未満では合計飛行時間の2-5％しか見られないと付け加えることができます。 さらに、航空機シミュレータの15 FPSは、一部のNFSまたはGTAの15 FPSとはほど遠いです:)そして、グラフィックの品質は、記事の最後に表示されるスクリーンショットから推定できます。ゲームのインストールを開始できます。



FSXをインストールする





シミュレーターのインストールは、2つの段階で構成されます。
  1. Flight Simulator Xの基本バージョンのインストール（2枚のDVDで提供）。 インストール後、シミュレーターを実行し、ゲームに入り、終了して構成ファイルの作成を開始することをお勧めします。
  2. アクセラレーションアドオン（別のDVD）をインストールします。 これは非常に重要なステップです。 追加には、必要なサービスパックと多数の解決済みの問題が含まれます。 インストール後、コンピューターを再起動することをお勧めします。その後、忠実度のために、シミュレーターを1回実行します。

主なことは、LAモデルの多くのメーカーが要求するように、「一般」スライダーが最大になっていることです。ここで、シミュレーターを終了して、設定をファイルに保存します。 その後、もう一度実行して、設定が正常にロードされることを確認します。 次のように、シミュレータを再び閉じます ここで、メインFSX構成のいくつかの行を手動で修正する必要があります：fsx.cfg。 このファイルは、Windowsユーザー設定フォルダーにあります。 Windowsのバージョンに応じて、「C：\ Documents and Settings \ YOUR \ Application Data \ Microsoft \ FSX」または「C：\ Users \ YOUR \ AppData \ Roaming \ Microsoft \ FSX」ディレクトリ パスに煩わされないようにするには、起動時に「実行」コマンドを選択して、そこに行を入力します。



notepad％APPDATA％/ Microsoft / FSX / fsx.cfg



設定は、Windowsのすべてのバージョンのメモ帳で自動的に開きます。 念のため、ファイルのコピーを保存することを忘れないでください。突然ミスをした場所に保存してください。 変更する必要があります（パラメーターが既にファイルに存在する場合）または次のパラメーターを追加します。



[Display] <br/>TEXTURE_BANDWIDTH_MULT=70<br/> [GRAPHICS] <br/>TEXTURE_MAX_LOAD=4096<br/> [Main] <br/>DisablePreload=1<br/>FIBER_FRAME_TIME_FRACTION=0.33<br/> [TERRAIN] <br/>TERRAIN_MAX_AUTOGEN_TREES_PER_CELL=1200<br/>TERRAIN_MAX_AUTOGEN_BUILDINGS_PER_CELL=1000<br/> [BUFFERPOOLS] <br/>PoolSize=12000000<br/><br/>



みんなの意味を説明しましょう：
TEXTURE_BANDWIDTH_MULT テクスチャをメモリにロードする「速度」を制御します。 推奨値は70〜80（デフォルトは30）です。 カメラを回すときにシミュレーターの速度が低下する場合は、より大きな値（最大400）を設定してみてください。
TEXTURE_MAX_LOAD サポートされているテクスチャの最大サイズ。 このパラメーターは、R​​EXアドオンおよび（おそらく）他のアドオンに必要です。 4096が最大です。
DisablePreload シミュレーターは、起動するたびにデフォルトでフライトをロードしません。これにより、ゲームのダウンロード速度をわずかに上げることができます。
FIBER_FRAME_TIME_FRACTION このパラメーターは、シーンのロードとレンダリングに必要なプロセッサー時間の比率を決定します。 たとえば、この場合の0.33は、プロセッサ時間の約30％がシーンのロードに費やされ、残りの70％がレンダリングに費やされることを意味します。 領域0.05-0.66のサンプルの推奨値
TERRAIN_MAX_AUTOGEN_TREES_PER_CELL 風景の自動生成のためのツリーの数。 値を大きくすると、FPSが大幅に減少します。 最適な値の範囲は800から3000までさまざまです。
TERRAIN_MAX_AUTOGEN_BUILDINGS_PER_CELL 同様に、自動生成されたランドスケープ内の3D建物の数のみを決定します。
プールサイズ このパラメータは、そこから必要なテクスチャを「引き出す」ために、テクスチャメモリのバッファのサイズを決定します。 値を12000000以上（バイト単位）に設定すると、レンダリングの品質に大きく影響し、オブジェクトやテクスチャがよりスムーズに表示されます。 512 MB以上のメモリを搭載したビデオカードをお持ちの場合は、最適な値を探して再生してみてください。
変更を保存した後、シミュレーターを実行して、微調整された設定の影響を評価します。 私は常にオーストラリアのブリスベン空港（ICAOコード：YBBN）を使用して、グラフィックスとFPSの品質を測定しています。 いずれかの航空機を選択して、この空港に積み込みます（無料フライト->場所->変更->空港コード別-> YBBN）。

FSXのCPU時間を最大にするには、より高い優先度で実行できます。 これを行うには、.batファイルを作成し、そこに次の行を挿入し、fsx.exeファイルへのフルパスを置き換えます。



スタート/ HIGH J：\ゲーム\ FSX \ fsx.exe

シェーダーバージョンのセットアップ





シェーダーバージョンを構成するには、シミュレータのROOTディレクトリにある別のFSX構成ファイルdisplay.cfgを修正する必要があります。 これは「J：\ GAMES \ FSX」です。 しかし、最初に、Windowsに含まれているdxdiagユーティリティが必要です。 実行するには、「スタート->実行」で「dxdiag」と入力するだけです。 表示されるウィンドウで、「すべての情報を保存...」ボタンをクリックし、受信したdxdiag.txtファイルを任意のフォルダーに保存します。 次に、メモ帳で両方のファイル（dxdiag.txtおよびdisplay.cfg）を編集用に開き、次の形式でセクション[xxxx：yyyy]を作成します（まだない場合）。 GF 8800 GTカードの例が示されていますが、ATIカードでも同様に機能するはずです。 これらの操作の後、FPSと水レンダリングの品質がわずかに向上することがあります。
