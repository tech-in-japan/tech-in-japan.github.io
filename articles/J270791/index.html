DataTalks＃4：予測分析

10月10日に、4回目のDataTalksが開催されました。 今回の会議のトピックは予測分析であり、スピーチのビデオ録画をコミュニティと共有したいと思います。



予測分析が必要な理由

顧客の行動や実行したアクションの結果などの履歴データに基づいて、さまざまな将来のイベントを予測できます。 それにより、企業は最適な決定を下し、将来の行動と顧客の希望の予測を考慮することができます。

「何が起こるのか」という質問に答える方法：実践的なアドバイス/アンドレイ・ヤルモラ

Wargamingのデータサイエンスチームリード

Andreiはプロとしてのキャリアの中で、データ分析の知識を科学とビジネスの両方に応用することができました。 彼の報告書では、彼の経験を共有し、利害関係者によって正しく理解され、当然のことと考えられている特別な技術の助けを借りて、予測を作成する方法を話しました。

このレポートは、アナリストだけでなく、予測に基づいて意思決定を行う人にとっても興味深いものになります。





推奨システムを構築するために必要な最小限のツール/ Alexey Dyomin

Java Server Side Developer、InData Labs

より多くの企業が、パーソナライズされたユーザーインタラクションの重要性と価値を認識しています。 レポートでは、Alexeyは、最小限の時間コストでパーソナライズされた推奨事項を作成するためのサービスを構築できる簡単なソリューションの例をいくつか示しました。

このレポートは、「少なくとも何かから始めたいが、どのアプローチをとるのかわからない」などの質問に答える技術専門家にとって有用です。





予測分析とは何か、誰がそれを必要とするか/ Nadezhda Rucanova、Mikhail Avetisov

SAP CIS LLCの代表オフィスの副ディレクターであり、予測分析のSAP CIS LLCの第一人者

NadezhdaとMikhailのレポートから、機械学習の理論の開発における現在の成果について学びます。また、特定の例で予測分析がどのように機能するかを確認します。 すべての計算は、オンラインのSAPソフトウェアを使用して実行されました。





Hadoopプラットフォームに基づいたデータウェアハウスの構築/ Igor Nakhvat

データ統合エンジニア、Wargaming

イゴールは、ClouderaとOracleのテクノロジーに基づいたWargamingデータウェアハウスのアーキテクチャについて話しました。 ソースデータソースと、独自に開発されたソリューションやApache NiFiなどの専用ツールを使用した統合の経験の両方について話します。

このプレゼンテーションは、Hadoopスタックにすでに精通している技術者にとって興味深いものになります。





予測分析を使用してクライアントベースの価値を管理する/ Maxim Mozgovoy

WargamingのCRMディレクター

満足して満足している常連客は、多くの企業にとって幸福の基盤です。 ただし、顧客の流出が細心の注意を払われることはめったにありません。通常、収入が減少します。 そして、それに応じて、その増加について考えますが、ユーザーの保持とクライアントベースの管理については考えません。 これは、多くの企業が1人を数回引き付けることにリソースを費やすことを余儀なくされるという事実につながります。

マキシムはスピーチの中で、クライアントベースとは何か、その価値は何か、そしてクライアント戦略の基盤としてそれを使用する方法について話しました。 マキシムは、価値管理の問題を解決するためのKPIメトリックとシステムについても話しました。
