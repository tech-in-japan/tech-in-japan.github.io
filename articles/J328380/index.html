中国のDocker：利用統計と成功事例

先週の月曜日、4月下旬に開催されたDockerCon 2017カンファレンスからのビデオの利用可能性について知られるようになりました。 そこで行われた講演の 1つで、クラウドコンピューティング、IaaS、PaaS、SaaS、ビッグデータを専門とするAlibabaグループの一部門であるAlibaba Cloud （またはAliyun）の代表が、中国におけるDockerの統計と成功事例について話しました。



Dockerの実装に関するグローバル統計に関する最近の出版物を継続するこの記事は、今年4月のパブリッククラウドの1,000人以上のユーザーとDocker使用の既知のケースの調査に基づいて、Alibaba Groupレポートから最も興味深い事実を要約しています。 しかし、中国の情報技術の世界におけるDockerの「出現」の背景から始めましょう...







中国でのDockerの最初のステップ



アリババクラウドとのDockerのコラボレーションは昨年10月に発表され、全国のアリババクラウドサーバーからドッキングされたアプリケーションを配布するために、中国にDockerハブが設立されました。 しかし、この国でのDockerの産業応用はずっと早く始まりました。 このコラボレーションの発表時だけでなく、Docker Meetupsは中国の20の大都市で開催されたと報告されましたが、2015年の夏にはさらにDaoCloudの代表者がDockerCon SF 2015で地元のDockerコミュニティについて詳しく話しました。



国内で最も初期の大規模なDockerユーザーの1人はBaiduでした。Baiduは 、2013年の夏にDockerをバックエンドとしてBaidu App Engineという名前でPaaSを開発しました。 1万を超えるアプリケーションと2万を超えるコンテナ（ホストあたり400のコンテナ）が機能しました。



テクノロジースタックがDockerに依存している中国のITスタートアップ（2015年のレポートから）



2015年6月の時点で、プロジェクトの主要リポジトリ（docker / docker）の主要な貢献者100人のうち9人が中国の組織（特に活発なZ江大学を含む）の代表者であり、Dockerに捧げられたイベントの合計は3,000人以上でした。





中国Dockerイベント地理



中国でDockerを使用しているのは何人ですか？



ただし、Alibaba Cloudの最新データに戻ります。 4月の調査の結果は、 約3分の1の企業が Dockerコンテナーを使用している、または使用を開始していることを示しています 。



  • 33％はDockerに精通していない。
  • 32％は勉強中です。
  • 13％-開発およびテストで使用。
  • 12％-何らかの方法で使用を開始します。
  • 10％-実稼働で使用されます。


同時に、回答者の17％は今後6か月以内にDockerを使用する予定さえありませんが、おそらく18％を使用するでしょう。







Dockerには何が期待され、実装を妨げるものは何ですか？



とりわけ、Dockerはソフトウェア開発プロセスを加速したいと考えています -957人の回答者が回答しました。 人気の低い回答：



  • リソース使用率を改善します（684）。
  • アプリケーションをポータブルにします（501）。
  • マイクロサービスアーキテクチャのサポートを実装します（428）。
  • 制御と標準化を改善します（425）。
  • ハイブリッドクラウドサポートを提供して、単一のサービスプロバイダーへの依存を回避します（387）。


Dockerの適応に対する主な障害は、 知識の欠如と運用経験の欠如です 。 特に、Alibaba Groupの顧客の多くは、Dockerの使用に関するトレーニング資料が不足しており、インターネット上の出版物が古くなっているか、十分に正確でないことが多いと不満を述べています。 コンテナの使用に関する特定の問題（ネットワーク、永続的なデータストレージ、負荷の移行、セキュリティなど）に関する懸念はそれほど懸念されません。



Dockerの使用方法



オーケストレーションの手段の中で、明確なお気に入り（約400人の回答者）は彼ら自身の開発です。 さらに、このオプションを選択した多くの人は、自分のタスクに対して既製のソリューションを試してさえいないことに気付きました。 有名な製品の中で、Kubernetes（200人未満の回答者）とDocker Swarm（約150人）がリーダーであり、Mesosと他のオプションははるかに遅れています。



Dockerを実装するための3つの主要な成功事例の詳細を以下に示します。



1.小さい



Tmallは、中国最大のB2C（企業対消費者）eコマースプラットフォームです。 9年前にTa宝網（当初は（宝網モールと呼ばれていました）によって導入され、後に独立し、現在はアリババグループによって管理されています。



毎年、11月11日、中国は独身の日を祝います。これは、婚status状況だけでなく、世界最大の1日あたりのオンライン販売でもあります。 2016年の独身の日のTmallサイトとTa宝網サイトの売上高は178億米ドルでした。 Tmallの場合、プラットフォームの主要なビジネスシステムはすべてコンテナ化され、1秒あたり175千のトランザクションを処理しました。 これらのタスクのために作成されたDocker Swarmベースのクラスターは、3万以上のノードにサービスを提供し、世界最大の著者と呼ばれています。 TmallがDockerイメージに使用する配布システムは、最大7000の同時プルをサポートしていました。



2. Weibo



Sina Weiboは中国最大のマイクロブログサービス（1日あたり1億件の投稿）であり、国内で最も人気のあるサイトの1つです（ローカルインターネットユーザーの30％以上がこれを使用しています）。



メンテナンスの主な問題は、注目度の高いイベント（ニュース）または祝日の場合、出席者の急増（たとえば、3倍の増加）です。 Weiboインフラストラクチャで使用されるプライベートクラウドは、ベアメタルと仮想化（OpenStackとKVMに基づく）の両方で動作しますが、これらのリソースは限られています（および道路）。 この問題は、作成されたDCP（Docker Container Platform）のフレームワーク内のコンテナーに切り替えることで解決されました。アプリケーション管理を統合し、ハイブリッドクラウドを使用できるようにしました。 10分で）。



（Docker Container Platformの技術構造に関するいくつかの情報はこのプレゼンテーションにありますが、中国語用に準備してください。）



3. CITICグループ



中国最大の国有投資会社の1つは、さまざまな国に44の子会社を持っています。 彼女のエンジニアは、さまざまなクラウドサービスを統合するソフトウェアプラットフォーム（CITIC Industrial Cloud）の作成に忙しく、その機能は子会社や多数のパートナーや顧客によって使用されています。 その結果、CITICプラットフォームはさまざまなサービスへのアクセスを提供する仲介者として機能し、業界固有の金融アプリケーションの統合と配信のために、Dockerでのコンテナー化が優先されました。



Alibaba自体でDockerを使用するには？



Alibabaはニーズに合わせてコンテナを使用するだけでなく、クラウドサービスと統合されたサービスとして提供するための包括的なソリューションを準備しています。 アリババクラウドコンテナサービスは 、分散型アリババクラウドECSクラスターで実行されるコンテナ化されたアプリケーションを管理するように設計されています。 コンテナとクラウドサービスは宣言形式で記述され、その後、ユーザーはワンクリックで展開できます。 Docker SwarmおよびDocker Composeとの完全な互換性を宣言したAPIは、CI / CDシステムとの統合に使用できます。 不思議なことに、 Aliyun Container Serviceは無料で利用でき 、必要なクラウドリソース（ECS、ロードバランサーなど）を購入することで報われます。







さらなるDockerの生産経験により、Alibaba CloudはDocker Enterprise Editionと統合されたApsara Stack Agilityクラウドプラットフォームを作成し、すぐに使用できるIaaS（Infrastructure-as-a-Service）およびCaaS（containers-as-a-service）機能を中国企業に提供しました） 彼女の発表はDockerCon 2017で行われました。



おわりに



中国のDockerエンタープライズアプリケーションを支える原動力はアリババだけではありません。 たとえば、 Huaweiには 、DockerとKubernetesに基づいた独自のPaaS（FusionStage）があります。 Kubernetesに基づくCluster-as-a-Service（ClaaS）製品を備えたCaicloudや、DockerおよびKubernetesに基づくPaaSを作成するHarmonyCloudなど、コンテナを中心に構築されたインフラストラクチャに完全に専念する若い企業が成長しています。





KubeCon（CloudNativeCon）でのHuawei Cloudネイティブベストプラクティスプレゼンテーション



中国でのDockerの適応はまだ本当に大規模ではありませんが、今日利用可能な例は、生産におけるコンテナーの実行可能性だけでなく、ビジネス上の問題を解決するために使用することの本当の利点も確認します。
