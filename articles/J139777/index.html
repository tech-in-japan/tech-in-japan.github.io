アプリケーションのエネルギー効率の改善

ARMプロセッサアーキテクチャ用のWindows OSの構築



Microsoft Windowsオペレーティングシステムの注目すべき機能の1つは、そのアーキテクチャの柔軟性であり、技術の進歩とクライアントベースの段階的な拡張の間に実証されています。 このオペレーティングシステムの初期バージョンは、作業を効率化するソフトウェアを個別に使用する1人を対象としています。 現在では、幅広いハードウェアおよびソフトウェアテクノロジー、相互接続されたWindows製品、驚くほど柔軟なコンピューティングアプローチを使用するための基盤となっています。 Windows 8では、チップセットからユーザーエクスペリエンスで終わるWindowsの概念を再考し、この新しいWindowsのアイデアをARMプロセッサアーキテクチャに移植することが、この革新的なプロセスの重要な部分になりました。

Windows 8オペレーティングシステムは、世界中の幅広いユーザー向けに開発された製品です。 特に高いレベルのイノベーションを実装しようとした分野の1つは、物理的な能力に関係なく、すべてのユーザーがWindows 8のアクセシビリティ（主にMetroスタイルアプリケーションとのやり取りに関して）を確保するという問題です。 この記事では、「特殊」と呼ばれる機能に関連するエンジニアリング作業について説明します（ただし、後で説明するように、これらの機能の多くは非常に広く使用されており、すべてのユーザーの製品の使いやすさを高めています）。



任意の言語を使用する

Metroスタイルアプリの高性能HTML5コンテンツ



MetroスタイルアプリでWebコンテンツを表示する場合、Windows 8はInternetExplorer 10のパフォーマンスを最大限に活用します。MetroスタイルアプリのWebコンテンツは、常にハードウェアアクセラレーションを使用してJITコンパイルおよび表示されます。 他のプラットフォームは、アプリケーションでこのレベルのパフォーマンスを提供しません。 たとえば、Cocoa iOSアプリは、Safariで同じコンテンツを実行する場合と比較して、JavaScriptのパフォーマンスが（UIWebViewコントロールを使用して）大幅に低下します。 CocoaアプリケーションはJITコンパイルを利用せず、これらのアプリケーションは、ブラウザーがオペレーティングシステムで実行できるのと同じ方法でWebコンテンツを表示および使用できません。



Internet Explorer 10のXHRのCORSテクノロジー



Internet Explorer 10プラットフォームの4番目のバージョンは、XMLHttpRequest（XHR）のCORS（クロスオリジンリソース共有）テクノロジーのサポートのおかげで、異なるブラウザー間で一貫して動作するクロスサイトスクリプトの構築を簡素化します。 XHRのCORSテクノロジーにより、データのサイト共有が簡単かつ柔軟になります。 ほとんどの基本的なシナリオでは、CORSテクノロジーを使用すると、任意のサイトからアクセス可能なデータソースを作成できます。いくつかの小さな設定で、許可サイトのリストを制限し、データ変更のサポートを実装し、さらに認証を有効にすることもできます。 ほとんどの場合、CORSテクノロジーはサーバーへの参加を要求することにより、既存のサイトを保護します。



サブピクセルレンダリングとCSSオブジェクトモデル



Windows 8は、大きなデスクトップモニターからコンパクトタブレットまで、インターネット上のWebページを表示するためのデバイスの幅広い選択肢をユーザーに提供します。 このデバイスの全範囲へのコンプライアンスを確保するために、ブラウザは、画面のサイズとその要素のサイズに応じて、Webページのスケーリングとレイアウトをサポートする必要があります。 これらのシナリオをサポートするInternet Explorerのいくつかの機能については既に説明しました。 サブピクセルレンダリング（テキストとレイアウト）は、プラットフォームの基本的な技術の1つです。これは、Webページがどの表示スケールでも美しく、一貫して見えるためです。 この記事では、Internet Explorer 10で導入された、CSS-OMオブジェクトモデルを使用したサブピクセルレンダリングのサポートを改善できる変更について説明します。
