1.はじめに



システム管理者のタスクはさまざまです。 コンソールでの作業-ユーザーの作成、サーバーでのパッケージのテスト、インストール、構成、ログとトラフィックの表示、ネットワークとトンネルのセットアップ。 鉄の使用-機器とドキュメントのインストール、作業計画の作成、サービスの説明。



システム管理者には、自動化のための多数のスクリプトがあります。 通常、ホームフォルダーに保存され（そこだけでなく、CVSにも良い）、定期的に新しい機能で更新されます。 原則として、それらを書いた同じ人だけがそのようなスクリプトを使用できます（そうでなければ、結果は最も壊滅的です）。



したがって、自動化タスクには、管理、簡素化を容易にするために単純なGUIが必要になることがよくあります。 たとえば、トラフィック収集。 または、ボタンでバックアップ/リリースをロールバックする機能（誰かがSCMシステムを介してパッケージをロールバックする場合でも）。 または、コンソールをスパイせずにMysqlマスターを変更します（間違ったサーバー上のコマンドに誤って詰まったために、一定数のエラーが発生します）。



伝統1：本番環境でのBashスクリプトとダイアログ
私の過去の作品の1つでは、サーバーとシステムをインストールした後、サーバーでbashスクリプトを実行し、ダイアログを使用して開発者向けのエコシステムを構成できました。 このサーバーで具体的に設定したいもの、具体的にはチェックする機会がありました。 ただし、後でサーバーでパペットを使用して構成をロールバックし始めました。



伝統2：Netflowベースのトラフィック収集ソリューション
別のジョブでは、すべてのトラフィック情報がNetflowを使用してサーバーから収集され、bashスクリプトを使用してデータベースに保存されました。 別のスクリプトでは、データベースのコンテンツが毎月xlsファイルにアップロードされます。





最新のテクノロジーは、適切なGUIでサービスをすばやく作成するための多数のオプションを提供します。 クライアントとサーバーの相互作用の例を分析し、Jquery、Bootstrap、Pythonプログラミング言語、python-flaskライブラリのテクノロジーを使用して独自のREST APIサービスを作成します。 データをテキストファイルに保存します。



REST APIのクライアントは、javascriptコードを含む通常のhtmlページになります。



この記事は、ときどき小さな決定を下さなければならないシステム管理者を対象としています。 Linux Ubuntu 12.04オペレーティングシステムで作業します。 同じテクノロジーセットを他のOS（Windows、Mac、Freebsd）で使用できます。



2.技術について







RESTは一般に受け入れられている推奨事項のセットであり、サードパーティの開発者がクライアントアプリケーションやフロントエンドを作成するのに便利なバックエンドを構築できます。 将来を見据えて、これらの推奨事項から少し逸脱し、最初はべき等のGETメソッドを使用して新しい情報をサーバーに追加します。



注4：REST
推奨事項-推奨します。必要なボリュームではそれらを正確に無視できます。 しかし、フロッピーAPIを使用して大規模なサービスを考えている場合、RESTに従うことで、その周りの混乱の量を大幅に減らすことができます。 読むことをお勧めします（そこのべき等性についても）： habrahabr.ru/company/yandex/blog/265569 （HTTPプロトコルでの正しい操作に関する15の些細な事実）



注5：フロントエンド
フロントエンドは、ブラウザのユーザー側で実行されるものです。 原則として、これはある種のjavascriptコードです。 バックエンドは、サーバー（大まかに言うと、ポート80に応答するプログラム）上で実行されるものです。



ブートストラップは、ページのデザインを考えずに既製の要素を使用できるようにするHTMLスタイルとテンプレートのセットです。



Jqueryは、言語を拡張し、たとえばGETリクエストを生成するための既製の便利な関数を使用できるようにするjavascriptライブラリです。



Python-flask-数行のコードでWebサーバーを作成できるPython言語のライブラリ。

注8：ajaxとは何ですか。
むかしむかし、インターネットが小さく、Webページがモデムから30分間読み込まれたとき-ページでの操作は、完全な再描画を意味していました。 ただし、時間が経つにつれて速度が向上し、ユーザーがページを操作するパターンが変化しました。 Ajaxテクノロジーは、ページがバックエンドとの関係で一種の「シンクライアント」であることを意味します。 現在、ユーザーがボタンをクリックすると、リアルタイムでjs-codeが実行され、ブラウザーのページのDOMツリーが直接変更され、要素が追加または削除されます。 たとえば、「検索」ボタンをクリックした後のYandexの検索行は、ページ全体は再描画されず、サーバーから受信した情報-バックエンドは、現在のhtmlに直接追加されます。



4.1 htmlおよびstatics returnの構成


バックエンドでPythonコードが機能する場合、ブラウザではページをレンダリングするために別の言語（html）が使用されます。 そして、それが機能し始める前に、どういうわけかそれは与えられなければなりません。 したがって、server.pyに次の行を追加（変更）します。

/var/www/server.py 
 #!/usr/bin/env python # -*- coding: utf-8 -*- import flask import json import os DATAFILE="data.txt" ROOT='/var/www' app_test = flask.Flask(__name__, static_folder=ROOT) @app_test.route('/') @app_test.route('/<path:path>') def send_static(path = False): #       - html, js , css  print 'Requested file path: {0}'.format(path) if not path: return app_test.send_static_file('index.html') return app_test.send_static_file(path) … if __name__ == "__main__": app_test.run(host='0.0.0.0', port=80, debug=True)




ここにあります：

  • 変数ROOTでサーバーのルートディレクトリを定義しました
  • 彼らは、このディレクトリをルートとして使用するようにFlaskに指示しました。
  • send_static関数への2つのルートをハングさせました。urlで指定されていない場合、index.htmlを返すか、url（パス変数）で指定されたパスにあるファイルを返します。
  • サービスをポート80に移動しました（以前と同様に、bashスクリプトを実行するserver.pyを作成したとき）
