コンパイルしないでください：ボタンがボタンに収まるかどうかをオンザフライで確認するためのLokalise SDKのライブプレビュー

アプリケーションをローカライズするときに、「レイアウトに適用する」のではなく、アプリケーション自体のテキストオプションをすばやく把握できると想像してください。 かっこいい？ だから私たちはそう思った。 そのため、本日、iOSアプリケーション用の新しいLokalise SDKライブ編集モジュールを導入します。これにより、iOSでプロジェクトをローカライズするときに加えられた変更をリアルタイムで監視できます。







ローカリゼーションプロセスを数週間から数か月にわたって引きずり出して開発者を絶えず引っ張る余裕のない小規模なチームがどれだけ便利で時間をどれだけ節約できるかを以下で説明します。



以前の出版物では、ウェブサイトの要素の次元性や、ローカライズのニーズに合わせてデザインを調整する必要があり、レイアウトに300本の釘を付けないという落とし穴について説明しました。



残念ながら、このような「フェイント」はモバイル開発の分野では機能しません。モバイル開発の分野では、デスクトップコンピューターやラップトップのデバイス画面のモニターと比較して、アプリケーションの作成者が非常に限られていることがよくあります。 この問題は、スマートフォン向けのアプリケーションでは特に深刻です（タブレットでも何かを考えることができます）。



ここで、この単語またはその単語が割り当てられたスペースに収まるかどうかを確認するには、ローカライズを変更するだけでなく、毎回アプリケーションをコンパイルする必要があり、これには長い時間がかかります。 覚えているように、モバイル開発ではスペースが非常に限られており、テキストに割り当てられたスペースに「収まらない」場合は、操作全体を再度実行する必要があります。



iOS用SDKのアドオンであるLokalise SDK Live Edit Moduleを使用すると、UIの変更と、インターフェイスで使用される語句がどのように見えるかをほぼリアルタイムで監視できます。 ライブ編集モジュールの主な対象読者は、ローカライザーとQAエンジニアです。 ユーザーにはアクセスできません。 ビデオを使用してモジュールの原理を示すことが最も効果的です：





Lokalise SDKライブ編集モジュールのデモ



別に、エディターとデバイスセッションの同期はQRコードのスキャンを通じて行われ、物理的またはその他の特定の接続を必要としないことに注意してください。 インターネット接続のみで十分です。







ライブ編集モードでできること：



  • デバイスでアプリケーションナビゲーションを使用する
  • ローカライズエディターの対応するウィンドウですべてのアプリケーション行を確認します。
  • エディターで行われた変更をリアルタイムで確認します。
  • エディターからアプリケーションのローカライズの言語を切り替えます。
  • デバッグメニューを呼び出すには、デバイスを振るだけです。
  • ローカライズ段階（リリース、プレリリース、ローカル、デバッグ）を切り替えます。


また、近い将来、デバイス画面からスクリーンショットを取得するための組み込みメカニズムを追加する予定です。 ライブ編集モジュールのもう1つの強みは、アプリケーションに加えられた変更をユーザーにすぐに送信できることです。 修正プログラムおよびアプリケーションのローカリゼーションの期限切れの修正に最適なオプション。



この開発により、アプリケーション開発者がローカリゼーションプロセスを大幅に簡素化および高速化できるようになることを願っています。 さらに、ライブモードを使用すると、直接開発者の負担を軽減し、ローカリゼーションプロセスを、翻訳者、QAエンジニア、そして極端な場合はデザイナーなどに対処する必要のある人に完全に「移行」できます。 あなたが人間のオーケストラであり、一人で仕事をすることを好むなら、ライブモードは最良のソリューションを見つける過程で無限のコンパイルの時間を節約するのに役立ちます。



ライブモードで作業するには、 Lokalise iOS SDKバージョン0.6.0 以降をインストールする必要があります。 また、正しく動作するにはXcode 8.0以降が必要です。 Lokalise Live編集モジュールのインストールと構成に関する詳細情報は、開発ページにあります 。 SDKは、iOS 8.0以降を実行するデバイスでの動作をサポートします。つまり、Appleによる2017年の現在のすべてのデバイスを対象としています。
