これは、画像の読み込みの様子です。 



$image = cv\imread("images/faces.jpg");


比較のために、Pythonでは次のようになります。 



 image = cv2.imread("images/faces.jpg")


PHP（およびc ++）で画像を読み取る場合、情報はMatオブジェクト（マトリックス）に保存されます。 phpでは、そのアナログは多次元配列ですが、多次元配列とは異なり、このオブジェクトでは、すべての要素を数値で除算するなど、さまざまな迅速な操作が可能です。 pythonでは、画像を読み込むと、numpyオブジェクトが返されます。



注意レガシー！ imread（php、c ++、およびpyton）がRGB形式ではなくBGRで画像をロードするのは偶然のことです。 そのため、opencvの例では、変換手順BGR-> RGBを見ることができ、その逆も同様です。

dnnモジュールを使用して写真の顔を検索する



そのため、 opencvでは、 readNetFromCaffe関数を使用して、事前にトレーニングされたモデルをCaffeに読み込むことができます。 .prototxtファイルと.caffemodelファイルへのパスという2つのパラメーターが必要です。 prototxtファイルにはモデルの説明が含まれ、caffemodelにはモデルのトレーニング中に計算された重みが含まれます。



prototxtファイルの始まりの例を次に示します 。 



 input: "data" input_shape { dim: 1 dim: 3 dim: 300 dim: 300 }


ファイルのこの部分は、4次元の1x3x300x300マトリックスが入力されると予想されることを説明しています。 モデルの説明では、通常、この形式で予想される内容を記述しますが、ほとんどの場合、これは300x300サイズのRGBイメージ（3チャンネル）が入力されると予想されることを意味します。



関数imreadを使用して300x300 RGBイメージを読み込むと、300x300x3マトリックスが得られます。



opencvで300x300x3マトリックスを1x3x300x300にするために、 blobFromImage関数があります。

その後、 setInputメソッドを使用してネットワーク入力にblobを送信し、 forwardメソッドを呼び出すだけで、完成した結果が返されます。 



 $src = imread("images/faces.jpg"); $net = \CV\DNN\readNetFromCaffe('models/ssd/res10_300x300_ssd_deploy.prototxt', 'models/ssd/res10_300x300_ssd_iter_140000.caffemodel'); $blob = \CV\DNN\blobFromImage($src, $scalefactor = 1.0, $size = new Size(300, 300), $mean = new Scalar(104, 177, 123), $swapRB = true, $crop = false); $net->setInput($blob, ""); $result = $net->forward();


この場合、結果は1x1x200x7マトリックス、つまり 各7要素の200配列。 4人の写真で、ネットワークは200人の候補者を見つけました。 それぞれ[、、 $信頼、$ startX、$ startY、$ endX、$ endY]のようになります。 $信頼要素は、「信頼」に責任があります。 予測の確率が成功したこと、たとえば0.75。 次の要素は、面と長方形の座標を担当します。 この例では、50％を超える信頼度で3人しか検出されず、残りの197人の候補者の信頼度は15％未満です。

設置



dnnモジュールは、バージョン3.4でのみopencvに登場しました（それ以前はopencv-contribにありました）。



ubuntu 18.04では、opencvの最新バージョンは3.2です。 ソースからopencvをビルドするには約30分かかります。そのため、ubuntu 18.04（17.10で動作、サイズ25MB）のパッケージをコンパイルし、php 7.2（ubuntu 18.04）およびphp 7.1（ubuntu 17.10）（サイズ100KB）。



ppa：php-opencvを登録しましたが、まだ塗りつぶしをマスターしておらず、 パッケージをgithubにアップロードする以上の良いものは見つかりませんでした。 また、peclでアカウントを作成するリクエストを作成しましたが、数か月後に応答がありませんでした。



したがって、ubuntu 18.04でのインストールは次のようになります。 



 apt update && apt install -y wget && \ wget https://raw.githubusercontent.com/php-opencv/php-opencv-packages/master/opencv_3.4_amd64.deb && dpkg -i opencv_3.4_amd64.deb && rm opencv_3.4_amd64.deb && \ wget https://raw.githubusercontent.com/php-opencv/php-opencv-packages/master/php-opencv_7.2-3.4_amd64.deb && dpkg -i php-opencv_7.2-3.4_amd64.deb && rm php-opencv_7.2-3.4_amd64.deb && \ echo "extension=opencv.so" > /etc/php/7.2/cli/conf.d/opencv.ini
