コンテンツについて簡単に：

  • staticmodを使用すると、サーバーは静的データを返すことができます（この場合、これはコンパイルされた.jsファイルになります
  • ocsipersist-sqlite-永続データを操作できるようにします。これが必要な理由がわかります。
  • deflatemodは、予想どおり、送信時にデータを押します。 以下では、サイト設定のdeflateセクションを見ることができます。これはjsのみを押すことを示しています。 注：圧縮方式（gzipまたはdeflate）を選択することはできません。クライアントの期待に基づいて自動的に選択されます。
  • eliom.serverは、フレームワークのサーバー部分全体を提供します
  • ホストのhostfilterパラメーターで、すべてのドメインに応答すると言います
  • そして、サイトの空のパスを示して、サイトがドメインのルートにあると言います。 空の行を「yoba」などに置き換えると、すべての静的コンテンツと共にアドレスhostname / yobaにサイトが強制されます-明確な利益。同じドメインに複数のサイトを保持し、好きなようにシャッフルできます
  • / home / username / yobaは、コード（およびコンパイル済みのyoba.jsファイル）が格納されたディレクトリになり、_build / serverには、コンパイル済みのサーバーモジュールになります。 もちろん、理想的には、コンパイルしてからコードを別のフォルダーに転送する必要がありますが、コンパイルされたモジュールと同じフォルダーに静的コンテンツを保存しないでください。

 let get_count = let counter_store = Ocsipersist.open_store "counter_store" in let cthr = Ocsipersist.make_persistent counter_store "countpage" 0 in let mutex = Lwt_mutex.create () in (fun () -> cthr >>= (fun c -> Lwt_mutex.lock mutex >>= (fun () -> Ocsipersist.get c >>= (fun oldc -> let newc = oldc + 1 in Ocsipersist.set c newc >>= (fun () -> Lwt_mutex.unlock mutex; return newc) ) ) ) ) ;;


OCamlに慣れていない場合（最初にやったように）、私たちの機能がトリッキーであることに気付くでしょう。 Get_count自体は、ミューテックスとストレージオブジェクトを格納する「オブジェクト」です。 コードに「get_count」を記述すると、input（）を受け取り、すべての作業を実行する関数が返されます。 次に、関数に入ります。 すぐにトリッキーな演算子 ">> ="が表示されます。これは、最初の引数（スレッド）の結果を2番目の引数（新しいスレッドを作成する関数）の入力に渡す特別な演算子です。 正式には、オペレーターの署名は次のとおりです。 

 val (>>=) : 'at -> ('a -> 'bt) -> 'bt


そして、最後のreturn関数はスレッドの結果を返します。

ここで確認できる多数のJs.Opt.iterは、getElementById関数が必ずしも何も返さないという事実によるものです。 したがって、Js.Opt.iterは、結果が本当にそうである場合にのみ、結果に対してアクションを実行します。 したがって、探していたページ上の3つのオブジェクトには、4つのJs.Opt.iterが必要でした。 4-デフォルトでは、getElementById関数は最も一般的なプロパティのみを持つ要素型のオブジェクトを返すため。 そして、textareaのvalueプロパティに到達するために、オブジェクトをtextarea型にキャスト（Dom_html.CoerceTo）しようとしていますが、これは一般的な場合の結果を保証しません。
