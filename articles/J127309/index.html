KolibriOSイメージの作成について

Chaos Constructions 2011でKolibriOSが導入される可能性があるというニュースから始まりました。 イベントの4か月前にKolibriOS開発者フォーラムで公開されました。



ですから、2011年8月上旬に、このシステム（KolibriOS）、さまざまなソフトウェア、使用可能なファイルのサンプルを使用して、仮想マシンのイメージを作成するというアイデアを思いつきました。 実際、それは私がやったことです。



私にとって最初の疑問は、どのマシンでイメージを作成するかです。 最初はBochsエミュレータとQEMUエミュレータ（すでに扱った）を好みましたが、この作業のためにVMwareから仮想マシンを選択することにしました。 VMware Playerは、明確なインターフェイスを備えた非常に高度なものに見えました。







イメージを作成するには、バージョン3.0以上のVMware Playerが必要です。 仮想マシンのイメージを作成した後、1 GBの仮想ディスクが作成および接続されました。これで十分です。 しかし、その後、パーティションを作成してフォーマットする必要がありました。 そして、ここで問題が発生しました。 Hummingbirdには、このようなアクションを実行するユーティリティはまだありません。 DOSブートディスクイメージを適用しました。 そしてそこに-fdiskとフォーマットc：...原則として、ハードディスクイメージなしでHummingbirdsを実行できます-必要なのはディスクイメージだけですが、もっと深刻なものが欲しかったです。



仮想ディスクへの最初のHummingbirdsのインストールは、GRUB4DOSブートローダーを使用して行われました。 はい、残念ながら、私は再びDOSで奇跡のフロッピーディスクの助けに頼らなければなりませんでした。



後に、「ネイティブ」のmtldrがセカンダリブートローダーとして使用されました。 これを行うために、仮想ディスクはディスクとしてホストシステムに接続されました。 ブートローダーインストーラーを少し変更する必要があります-inst.asmソースコードで、2行のコメントを外して、ブートローダーをリムーバブルメディア（フラッシュドライブ）だけでなくハードドライブにもインストールできるようにします。 プログラムを再コンパイルした後、ブートローダーをインストールしました。 



; Uncomment following lines to allow hard drives ; cmp eax, 3 ; DRIVE_FIXED ; jz @f


Hummingbirdを起動するために必要なのは、ブートローダーをインストールし（ご覧のとおり、少なくとも2つのオプションがある）、ディスクイメージをディスクのルートにコピーすることです。



ディスケットのイメージとして、最後の「夜」ビルドを選択しました。 Hummingbird OS（0.7.7.0）の最新リリースは2010年2月に行われました。 また、SVNに変更を加えた後、ナイトリービルドが自動的に作成されます。



特に、ハミングバードOSは21世紀にあるため、フロッピーディスクイメージがそれと何の関係があるのか​​を理解していない可能性があります。 説明します。



ユーザーの観点から見ると、これは単なるシステムファイルのコンテナです。 ディスクイメージはFAT12ファイルシステムを使用します。 * nix（マウント）とWindows（Winimage、Total Commanderのプラグイン）の両方でイメージを操作できます。 単一のファイルを転送する方が簡単です。 開発者の観点から見ると、これは先祖KolibriOS（MenuetOSオペレーティングシステム）からの継承です。 それなしでは、ramdisk / rd / 1を作成することは困難ですが（可能です）。 RAMのディスクは、Hummingbird OSの機能の1つです。



仮想マシンのイメージを作成するプロセスの説明に戻りましょう。 Hummingbird OSの変更されていない夜のアセンブリは使用しませんでした。 まず、ハードドライブを持つという意味が失われます。 はい、ハミングバード用に存在するすべてのプログラムはフロッピーディスクイメージに収めることはできませんが、それらのほとんどは非常に可能です。 次に、仮想マシンイメージの目的はオペレーティングシステムに慣れることであるため、プログラムの場所をより視覚的にすることを試みました。 このために、ディレクトリ構造が変更され、可能であれば、プログラムがRAMディスクからハードドライブに転送されました。 さらに、リモートアプリケーションの接続が修正されました（たとえば、メインメニューから呼び出すことができるように）。







そのため、ramdiskに残しました。カーネル、ドライバー、フォント、動的にロードされるライブラリー、および絶対パスを使用してアクセスするプログラムです。



次のディレクトリがハードドライブに作成されました：デモ、開発、ドキュメント、教育、エミュレータ、ゲーム、メディア、ネットワーク、オフィス、その他、スキン、システム、壁紙。 アプリケーションおよびその他のファイルは、これらのディレクトリによってソートされ、アプリケーションごとに独自のディレクトリが作成されました。



ほとんどのゲームは51でした。これらは単純なおもちゃ（タグ、三目並べ、蛇、サッパー、ソリティア、テトリス、ライン、リバーシ、海戦、チェッカー）、および移植された運命です。







CBSのグラフィカルインターフェイスを備えたゲームに加えて、タグ、チェス、人工知能のエリザシミュレーターなど、いくつかのコンソールゲームがあることに注意してください。



他のカテゴリのアプリケーションは、代表者がそれほど豊富ではありません。



システムプログラムには、ファイルマネージャーKFAR、KFM、およびEolite（それぞれFAR、Total Commander、およびExplorerのアナログ）、情報ユーティリティCPUID（CPU-Zのアナログ）、Disptestモニターをテストするユーティリティ（Nokiaモニターテストのアナログ）、ハードドライブに関する情報を取得するユーティリティが含まれますHDD-Infoドライブ、画面上の拡大鏡、仮想キーボード、および単純なシェルコマンドインタープリター。



開発アプリケーションはDevelopディレクトリにあります。 これは、ボードデバッグボード、Mtdbgデバッガー、Heed hexエディター、動的ライブラリによってエクスポートされたcObj関数を表示するためのユーティリティ、FASMアセンブラー、KolibriOSのメイン開発ツール、およびその使用例（Brainfuck、Lua、Tinypy）のインタープリターです。







同じディレクトリに、ハミングバードOS用のプログラムの例、主に動的ライブラリのテストがあります。







エミュレータカタログには、DOSBoxエミュレータ、NES FCEUエミュレータ、ZX Spectrumコンピュータエミュレータ、およびB3-34プログラマブル計算機シミュレータが含まれています。







トレーニングソフトウェアは、教育フォルダーにあります。 これまでのところ、周期表、高度なキーボードシミュレーター、マップビューアー（SASPlanetプログラムのキャッシュを使用）という3つのプログラムしかありません。 後者については、カードのライセンスではすべてが明確ではないため、仮想マシンのイメージにはほんのわずかしかありません-参考のためです。







メディアディレクトリには、オーディオプレーヤーとビデオプレーヤー、グラフィックファイルとアニメーションの視聴者、さまざまな形式のフォントなどのマルチメディアプログラムが含まれています。 サウンドドライバーのないビデオプレーヤーの動作は非常に悪くなりますが、それでも試してみるために2つのビデオファイルが画像に含まれています。 zSeaイメージファイルビューアーに特に注意を払いたいです。 非常に高度な機能を備えていますが、システムの公式アセンブリには含まれていません。 ここでは、デフォルトの画像ファイルビューアとして設定します。



他のソフトウェアを使用する場合は、よく理解することをお勧めします。



画像には他に何がありますか？ たとえば、スキンと壁紙。 システムの外観は非常に簡単に変更できます。 しかし、画像の作業の過程で、私はまだ外観に問題がありました。 アイコンセット、壁紙、ウィンドウスキンは非常に多くありますが、互いにぴったり合う組み合わせを見つけるのは難しい作業であることがわかりました。







参加者は、MacOS Xインターフェイスをコピーするスキンを使用しているため（左と右の両方にボタンがあるボタンがあるため）、最初のオプションが好きではありませんでした。 プロジェクトの参加者は多数のオプションを提供しましたが、私はそれらのどれも好きではありませんでした。 したがって、このアセンブリの外観は標準に達していません。







ところで、システムの外観を変更するには、/ rd / 1 / autorun.datファイルを変更し、/ rd / 1 / dafault.sknという名前で目的のスキンを保存し、ramdiskイメージを/ hd0 / 1に保存する必要があります。 ramdiskの変更後（システムの再起動後に保存されるように）、rdsaveプログラムを使用して保存する必要があります。



システムのドキュメントはDocsディレクトリにあり、DocpakアプリケーションとアセンブリによってHTMLファイル（HTMLvプログラムを使用して表示）の形式で表示されます。



それで、画像の内容を調べたので、この画像の組み立ての結果は何であるかを言いたいと思います。

-KolibriOSを使用した仮想マシンのイメージは、このシステムに誰もが慣れ親しむために組み立てられました。

-KolibriOSのプログラムの数についてアイデアを得ました。 SVN、フォーラム、その他のリソースからそれらを取りました。

-ソフトウェアの弱点（システムのインストール、ディスクのパーティション分割およびフォーマット用のプログラムはありません）および設計（結局、一致するスキン、壁紙、アイコンはほとんどありません）

-貴重な経験を得た。



結論の代わりに。 残念ながら、Chaos Constructions-2011のイメージは忘れられましたが、これは主なものではありません。 公開前に、イメージがわずかに更新され、いくつかのエラーが修正されました（そして、おそらく新しいものが導入されました）。 「人々」からそれをダウンロードできます（98 Mb）： narod.ru/disk/36181467001/KolibriOS.zip.html
