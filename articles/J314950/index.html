「コードアワー」の過ごし方：カザフスタンの経験

1時間のコードは世界的なイニシアチブであり、学校の授業での1時間のプログラミング入門であり、コーディングの複雑さについての神話を払拭し、誰もが基礎を学べることを示すように設計されています。 昨年、当社は最初に全国レベルで2つの省庁とともにHour of Codeを開始および開催し、イニシアチブの一環として、約2,000の学校（これは実際には国内の4分の1である）から75,000人の学童を対象としました。



免責事項：この資料は、地域や都市のレベルで1時間のコードを費やすことを希望し、他のすべての人にとっては忙しい人にとって実用的です。







なぜこの資料を書いているのですか？ -Hour of Code 2016がすでに手元にあり、12月5日から11日までの週に開催されるため、準備の時間はすでにゼロであり、後で何かがおそらく無関係になることを意味します。 一方、「1か月間」は、昨年の準備を始めた時期とほぼ同じです。 「どうしたの？」という質問に答えるとき、私はまだ「こんなにたくさんのレッドブルを飲んだことはない」と正直に答えます。 ただし、すべてを順番に説明すると同時に、注意を払うべきことと絶対にすべきでないことについての個人的な推奨事項でお世辞を述べます。



前戯



2014年にHour of the Codeを知っていましたが、その後、最初の顧客の検索に専念し、Hour of Codeの従業員の子供たちのために過ごすという提案のある大企業にたどり着きました。 純粋にcode.orgの資料で実施しました 。 みんな気に入った。 彼らは7-8人でした。



フェーズ1：組織的および政治的



2015年、11月のどこかでコードアワーを思い出し、全国レベルのイベントをどのように作成するかを真剣に考えました。 そのようなスケールは、新しいフロンティアで等しくおびえ、楽しませた。 論理的な最初のステップとして、コードアワーが既に開催されているロシアの最も近い隣国としての経験を調査しました。 そして、彼らは、カバレッジを達成するために、管理リソースが必要であることに気付きました。それがなければ、タスクはすぐに耐えられなくなるからです。 管理リソースとはどういう意味ですか？ -これは、カザフスタン共和国の教育科学省から、授業の実施が推奨される学校への手紙です。 質問はすぐになりました-そのようなサポートを得る方法？ 私たちの場合、文字通り30分後に「私たちはどうですか？」-「素晴らしいアイデア、私たちに何が必要ですか？」という手紙に返信した隣の省の人に連絡することを勧められたので、私たちはとても幸運だったと思います。 今後、文部省と大臣自身の両方に連絡を取るのを助けてくれたのは彼だと言います。



結論1：あなたの側の国家の関与を加速する適切な連絡先を見つけます。 おそらくこれはそれほど単純ではないが、重要です。 カザフスタンには約8,000の学校があり、200万人近い子供が学校に通っています。 確かにそれらをすべて自分で引き付けることはできません。



結論2：お金ではなく、管理サポートを求めます。 たとえば、「学校への手紙」と「大臣とのビデオ」が必要であると答えたのは90％だけでした。これは、金銭に関する多くの承認を必要としないため、プロセスを大幅に加速しました。



フェーズ2：材料の適合



州の支援を受けて、第2段階に進みました。code.orgの資料を調整する必要がありました。 適応は主に3つの領域にありました。まず、サイト（code.org）から演習のレッスン全体を提供するのではなく、トピックのプレゼンテーション、短いディスカッション、演習自体を含むレッスン計画を作成します。 第二に、多くの資料をカザフ語に翻訳する必要がありました。 第三に、すべての学校がインターネットにアクセスできるわけではなく、どこにいても-コンピューターの数が十分でないか、インターネット接続が非常に遅いことがよくわかっているため、コンピューターなしで実行できるレッスン計画が必要でした。



驚きのこの部分では、正直なところ、それほど多くはなく、困難は準備時間が非常に短いためでした。 数分でスケジュールされたレッスン計画を含むツールキットを開発しました。 コンピューターを使用しないバージョンでは、参加者の1人がロボットの役割を果たした「ロボット」でゲームを選択しました。チームのメンバーが書いた指示に従って、カップから構造を組み立てる必要がありましたアルゴリズム。







さらに、教師向けにプレゼンテーションを行い、コードアワーとは何か、レッスンの準備が必要なものなどについて説明しました。 それで、彼らはそれをすべてカザフ語に翻訳しました。



結論3：優れた解決策は、教師自身がそこに行ってすべてをダウンロードできるため、すべての資料をサイトに配置することでしたが、レターに添付ファイルを追加する必要はありませんでした。



結論4：私の数はどこにでも示されていたため、教師の質問の数を非常に過小評価していました。 「証明書を取得しますか？」という質問に対して まったく同じ地獄がメールで発生していました-膨大な数の手紙が注がれていたため、迅速に処理することは非常に困難でした。 ここでの良い解決策は、コミュニケーションの負担を少なくとも少し軽減し、他のことをできるようにするボランティアを引き付けることです。



結論5：低い教育レベルに備える必要があります。 たとえば、サイトの言語を切り替える方法や、デフォルトで「簡単だ」と評価した他の操作を行う方法については、説明では詳しく説明していませんでした。 しかし、すべてのコンピューターサイエンスの教師がデジタルネイティブであるとは限らないため、「低レベル」の教材を準備して、誰もが明確に理解できるようにする方が良いでしょう。



結論6：何があっても-証明書が必要です！ 私たちにとって、それは驚きでしたが、紙切れ-価格で。 ほぼ3人に1人の先生が、イベント後に彼に証明書を与えるかどうかを尋ねました。 アイデアには証明書がなかったので、最終的には、名前を入力できる空の証明書（メールにPDFとして送信）を全員に与えましたが、自動化について真剣に考えました。



フェーズ3：学校での作業



学校との仕事の仕組みは次のとおりです。省がプロジェクトの責任者を任命し、プロジェクトの責任者が地域の責任者を任命しました（16人-地域の数に応じて）。 その後、16人全員が、Code Hourに関するコミュニケーションを複製するために、メールの学校長のリストが必要であると書きました。 私自身はまだ質問をしている-なぜ省庁自体ができないのか、しかし州の道筋は必ずしも線形ではないので、再確認することにした。



最初の課題はコレクションの事実でした。なぜなら、データ（xls、特定の列）を表示する非常に明確な形式を提供したにもかかわらず、WordからPDFまで、まったく異なる形式でリストを送信したためです。その後の作業が非常に複雑になりました。



ニュースレターの作成を開始したときに発生した2番目の問題：多くのアドレスのスペルが間違っていました（無実に見逃された@から始まり、現在は機能しなくなったドメインにバインドされていたアドレスで終わるだけです）。



多くの場合、メッセージの送信先のアドレスでさえ非アクティブであることがわかりました。 そして、監督は週に一度メールをチェックするので、単にあなたの手紙を見ることができませんでした。 または彼女をショーに連れて行った。 そして、それだけです。



結論7：ディレクター/情報学者とコミュニケーションをとるための最良の方法を見つけるために時間をかけてください。 そうでなければ、多くの手作業の準備をしてください。



フェーズ4：パートナーとパートナーの関与



学校で働くことは良いことですが、幅広い参加者の間でイニシアチブを推進したかったのです。ここでは、良いパートナーなしではできません。



誰と仕事をしたか-まず第一に、これらはcode.orgのメンバーです-誰が私たちがどんなデータを手に入れることができるか、どこから始めればよいかなどの質問に常にオープンに答えてくれました。 オフィシャルパートナーのセクションに私たちを追加するように依頼し、イベントの写真も送ってくれました。







さらに、Microsoftカザフスタンの支援を得ることができました。これは、財政的および衣服の観点から、参加者、ステッカー、ボール、その他の作品の証明書の印刷に支払うだけでなく、学校や孤児院への旅行に積極的に参加しました。







最後に、私たちは書店とエンターテイメント店の大規模ネットワークの1つである「Meloman」（一般的にモスクワの「共和国」の近くにあります）にも支えられました。 Code HourがAppleによってサポートされているというニュースを読んで、ほぼランダムに連中に会いました。これで、Code Hourを会社の店舗で渡すことができるようになります。 なぜこれを一緒にやらないのですか？ -私たちは思った。 そして彼らはやった。 :)







結論8：パートナーを事前に探す必要があります。 多くの場合、企業は大規模であり、計画期間が広い。 そのため、イベントの10日前に到着してお金やその他のものを求めても、あまり頼る必要はありません。



フェーズ5：マーケティングイニシアチブサポート



正直なところ-マーケティングに関するすべて、私たちは少し失敗しました。 まあ、それほど難しくはないかもしれませんが、もっと良くできたかもしれません。



資産のうち、サイトがありました。 しかし、マーケティングよりも機能的です。 サイトにはプロジェクトの説明があり、教師がすべての資料をダウンロードし、学生がそこに登録しました。



私は少し馬鹿なことをします。code.orgのポリシーでは、ユーザーに関するデータをほとんど収集しません。 そして、どの地域のどの教師が参加しているか、どの学校から、彼らが何歳か、生徒の性別を理解することが重要でした。 そのため、登録が当社のWebサイトに転送され、フォームに記入した後、参加者はcodorg Webサイトにリダイレクトされました。



しかし、続けましょう。 codorgの伝統では、キャンペーンの重要な要素がビデオであることを認識しました。 最初の最も人気のあるビデオのアプローチをコピーすることにしました。 これを行うために、地元の成功したIT起業家のリストを作成し、子どもたちとチームのメンバーで希釈しました。



私は起業家をよく知らなかったので、招待状の形式はFacebookに書き、回答を期待することでした。 私は驚くほど幸運で、多くの人が同意しました:)ここでの課題は、すべての参加者が同じ都市にいるわけではなく、別の参加者で撮影しなければならないということでした。 さらに、大臣の記録にも同意しましたが、その結果としての承認数は、撮影時間の承認から始まり、最終的なビデオに含まれるものの承認で終わる-屋根を通過しました。



2015年、私たちは実質的にメディアを引き付けませんでした。 なんで？ -単に十分な力、時間、集中力がなかった。 プレスリリースはありましたが、積極的なプロモーションはありませんでした。



結論9：ビデオを作成する-プロのオペレーターを雇う/見つける。 私たちはすべて自分たちでやってきて、多くの時間を費やしましたが、より速く、より良くなる可能性があります。 ビデオは基本的にあなたの名刺であり、良いはずです！



結論10：事前に報道機関について考えてください。 このトピックに関する記者会見を開き、より多くの利害関係者を集める価値があるかもしれません。



結論11：パートナーを引き付ける。 無償の原則に基づいて、このプロジェクトでPR機関と協力してみてください。 これはまさに、コードアワー後に彼らと話をしたときに英国の同僚が提案したアプローチです。 これにより、リソースを節約し、代理店が連絡先を作成して、最良の結果を得ることができます。



フェーズ6：ポストエフェクトとクロージャ



実行段階を逃しました。 しかし、これはその中にたった一つの偏りがあるからです-常に連絡を取り合うためです。 そうすれば、質問に答えることができます。



一般に、授業の一部が1週間ではなく2週間で実施された学校が多数あったため、Code Hourは時間内に延長されました。 さまざまな理由で-彼らは遅れて見つけた、すべてのクラスまたは何か他のもので解決する時間がありませんでした。



イニシアチブの後、すべての教師は2〜5枚の写真の形式でレポートを送信することになっています。 正直に言うと、結果から大いに驚きました。



-地域の人々は、首都の居住者よりもイベントを高く評価しました。活動が最も少なく、それをより重視しているからです。

-大人の教育者だけでなく、子供たちからも絶賛されました。 彼らは次に何をすべきか、どこで学ぶべきかを尋ねた。

-地域からのプログラミングの学校でのトレーニングのための多くのアプリケーションがあります。



結論12：次のステップについて考えます。 規範の1時間では、人生は終わりません。イベント後の次の論理的なステップとして、点灯した参加者に何を提供するかを理解することが重要です。



結論13：間違いを記録します。 何がうまく機能し、何を別の方法で行う必要があるか。 イベントの後、チーム全体が大きなリストを作成しました。このリストは現在、フォルダー内にあります。CodeHour 2016の準備を始めたとき、最初にそれを確認したのは私たちでした。



それだけです そして、今年のCode Hourで私たち全員に幸運を！
