HP EVAディスクアレイ-簡単！

P6000 EVAディスクアレイの新しいモデルのリリースを見越して、これらのアレイがどれほどシンプルで効果的かを示したいと思います。



HP P6000アレイのアプリケーションの従来の分野は、仮想化、Microsoft、Oracle、SAP、および広範なビジネスアプリケーションとの統合であり、現在もそうです。 P6000ラインは、P6300 EVAとP6500 EVAの2つのモデルで表されます。 これらのストレージシステムはSAS（Serial Attached SCSI）テクノロジーに基づいており、146 GBから3 TBまでのさまざまな容量の小型（SFF）および大型（LFF）フォームファクターディスクをサポートしています。



前世代のEnterprise Virtual Arrayアレイとは異なり、P6000 EVAは8 Gb / sファイバーチャネルポートと10 Gb / sまたは1 Gb / sイーサネットポートの両方を備えているため、iSCSIおよびFCoEプロトコルを使用してサーバーに接続できます。



なぜEVAなのか？



独立した有名なEdison Groupの調査によると、P6000アレイの管理は、同様のシステムと比較して、50％の時間がかかり、30％の複雑さが軽減されています。 これをお金に換算すると、平均節約額はアレイあたり年間約13,000ドルになります。



コマンドビューの概要



Command View-P6000アレイ管理ソフトウェア。 EVAアレイに慣れる時間がなかった場合は、ここから CV P6000シミュレーターをダウンロードすることをお勧めします。 この記事のすべてのアクションはシミュレーターで実行され、自宅で繰り返すことができます。



P6000では、コマンドラインインターフェイスまたはスクリプトを使用してアレイを管理できますが、日常のタスクではGUIインターフェイスを使用する方が効率的で簡単です。



シミュレーターをダウンロードするには、HPパスポートを登録して取得する必要があります（まだ持っていない場合）。



次に、compmgmt.mscコマンドを使用してコンピューター管理セクションにアクセスし、パスワードを持つ「CV」ユーザー、ローカルPCの「HP Storage Admins」グループを作成し、CVユーザーをHP Storage Adminsグループのメンバーにする必要があります。



最後に、「HP P6000 EVA Simulatorの開始」ショートカットを使用してシミュレーターを実行します。 すべてのスクリーンショットはクリック可能です。







新しいP6000ディスクアレイのリリースでは、Command Viewインターフェイスは劇的に変更されないため、ある世代から別の世代に切り替えても、従業員のトレーニングに時間とお金を費やす必要はありません。







さらに、アレイをサポートする新しいテクノロジーを含めることは非常に透過的に見えます。たとえば、新しいSSDディスクは、追加の製品情報へのリンクを備えた新しいアイコンで表されます。これらはすべて、ディスクシェルフの同じナビゲーションウィンドウ内とナビゲーションツリーの同じドロップダウンリスト内にあります したがって、新しい配列を操作するロジックは保持されています。



Edison Groupの調査の評価には、 プロビジョニング 、 データ保護 、 メンテナンス 、 トラブルシューティングのインジケーターが含まれていました。 このリストとそれに含まれるものをさらに詳しく考えてみましょう。



プロビジョニング -インストール後にすべてアレイで動作します：



  1. ディスクグループの作成
  2. ホストを追加する
  3. 仮想ディスク（LUN）の作成
  4. ホストへの仮想ディスクのプレゼンテーション。
  5. コンテナ作成


HP P6000 Command Viewソフトウェアには、無料のシンプロビジョニングオプションが含まれています。これにより、ディスクスペース要件、ハードドライブの数が削減され、サーバールームの電力および空調コストが削減されます。



このオプションをCommand Viewに追加するには、仮想ディスクを作成するときにドロップダウンリストのボタンを1つだけクリックする必要があります。







データ保護 -アレイのデータを保護するすべてのメカニズム。 これらのメカニズムには、アレイ内の複数のコントローラー、フォールトトレラントI / Oパス、リモートレプリケーション、スナップショット（読み取り専用コピー）およびクローン（書き込み可能コピー）の作成が含まれます。



HP EVAでは、各仮想ディスクに独自のスナップショットポリシーを設定できます。 これにより、リポジトリ全体の構成がより柔軟になります。







メンテナンスと構成の変更



配列の提供には、多数の操作が含まれます。 たとえば、ソフトウェアの更新、故障したディスクの交換、ディスクとディスクシェルフの追加、使用されている構成の変更、スナップショットの削除などです。



ソフトウェアの更新は、アレイ全体またはそのコンポーネントの動作を停止することなく一元的に行われます。 アレイディスクとシェルフもホットプラグされます。 コントローラーはアクティブ-アクティブモードで動作し、キャッシュミラーリングで複製されます。アレイには単一障害点がありません。



P6000でのスナップショットの削除は非常に簡単です。Vdiskドロップダウンメニューで、必要なスナップショットを選択します。これは、最初に割り当てられたホストから解放する必要があります。







監視とトラブルシューティング -アレイの状態とパフォーマンスを決定するアクション。



Command View管理ソフトウェアは監視と統計の収集を提供し、グラフィカルインターフェイスからアクセスできます。また、電子メールアラートを設定できます。 この機能（自動サポート）はデフォルトで有効になっています。







ディスクアレイで最も一般的なタスクのいくつかを検討してください。



1.ディスクグループとLUNを作成する







必要なアレイを選択し、ドロップダウンメニューで[ディスクグループ]-[ディスクグループの作成]を選択し、グループの名前、保護設定、およびディスクの数（8の倍数を推奨）を選択します。 アレイ内のボリュームが「高速」および「低速」ボリュームなしで同じ速度になる場合、アレイ内のすべてのディスクを1つのディスクグループに入力することをお勧めします。 したがって、アレイのディスクスペースはシステムのすべてのディスクにわたって仮想化され、最大のパフォーマンスが得られます。ディスクグループのIOPSは、グループ内のすべてのディスクのIOPSで構成されます。







ディスクグループを作成した後、ディスク、その場所、ファームウェアの更新に関する情報を確認できます。



次に、[仮想ディスク]セクションで、必要なパラメーターを使用してLUN（Create Vdisk）を作成します。



2. LUNホストへのプレゼンテーション







[ホスト-ホストの追加]セクションでホストを追加します。 必要な設定（システムが事前に準備したテストWWNを使用したビデオで）、システムのタイプを選択し、IPアドレスを入力します。







[プレゼンテーション]タブのディスクプロパティで、ホストにLUNを指定します（現在）。



3.アレイ間でボリュームレプリケーションを作成する







アレイ間で複製を作成する例を考えてみましょう。ソースはP6300アレイ、レシーバーはP6500です。



必要なパラメーターを使用して、LUNを各アレイに（シンプロビジョニングなしで）作成する必要があります。

ソースソースアレイでは、ディスクがデータレプリケーショングループに追加され、ターゲットアレイ、ソースボリューム、レプリケーションのタイプ（同期、非同期ベーシック、非同期拡張）、ターゲットLUNのフォールトトレランス、LUNへのホストアクセス、ログのログサイズがここに示されます。 レプリケーショングループを作成した後、アレイ間で完全なコピーが記録されるまで待機します。



4.ディスクグループへのディスクの追加







しばらくして、ディスクをディスクグループに追加する必要がありました。



ディスクをアレイに接続した後、Command Viewインターフェースに移動して、より多くのディスクスペースが必要なディスクグループを選択し、「ディスクの追加」ディスクを追加します。 アレイが再構築され、追加されたディスク容量が使用可能になります。



5.スナップショットの作成







[仮想ディスク]セクションで、スナップショットを作成する必要があるLUNに移動します。 名前（デフォルトでは作成日が取得されます）、フォールトトレランスレベル、スナップショットを含むコンテナー（必要な場合）を選択します。 ディスクが親になり、作成日を示すスナップショットが下のディレクトリに表示され、将来的にはホストに提示できます。
