Webスタジオの主な法的エラー

30年前でさえ、アーティストの作品には想像力と器用さだけでなく、身体的な費用も必要でした。 はい、デザイナーは彼の視力と姿勢に別れを告げることができました。 ランプの薄明かりの中で胎児の位置で摘み取って、アーティストはコンピューターの助けなしで彼の傑作を作成しました。 たとえば、ソビエトのアニメーターは、漫画「12か月」のすべてのキャラクターをすべての動きで手動でトレースしました。







「12か月」-S.Yaの戯曲に基づいたソビエト漫画のフレーム。 マーシャク、1942–43



時間がなくなっています。 コンピューターのサポートがありました。 現在、アーティストの作品にはコンピュータープログラム、想像力などの開発が必要です。 仕事は簡単にはなりませんでしたが、時代に遅れずについていきました。 完全に新しい職業-Webデザイナーが登場しました。 職業は新しいものですが、法律は同じです。 多くの場合、Webスタジオ自体はクリエイティブウェアハウスの人々によって運営されており、これらの遅延や紙に署名するだけではありません。 したがって、ウェブスタジオにはしばしば法的問題があります。 法律実務で最もよく見られるものを検討します。



最初の問題：顧客は支払いません



このような問題には多くの詳細と色合いがありますが、本質は常に同じです。 あなたは仕事の全部または一部を行いましたが、顧客はあなたを送り、きっぱりと支払いを拒否しました。 不快な状況、そしてあなたがラクダではないことを証明する方法は？ 悲しませたり、ささやいたりしないでください、問題は解決されます...



解決策：

  1. 明確な作業受け入れアルゴリズムを提供します。
  2. 試用前の申し立てに対して短い期間を設定します。
  3. 全額支払いが完了するまで、結果に対する排他的権利を保持することを忘れないでください。
  4. 契約を締結する前に、取引相手を確認してください。 ブランドは有名であり、その代わりに1日のスタンドがあります。


2番目の問題：顧客はすべてを100回やり直すことを強制します



最初の20件の編集は従順に行いますが、忍耐は終わります。 怒る？ 当然です。



解決策：



  1. サイトの設計に関するすべてを含め、作業を受け入れるための明確な手順を契約で詳細に指定する必要があります。
  2. 参照条件を詳細に書き留めます。 これは、プロジェクトの作業を開始する前に行う必要があります。 特定のタスクを設定し、すべてを顧客と調整します。 参照の条件では、プロジェクトが完了する期限を示すことを忘れないでください。


3番目の問題：顧客は、製品またはその部品に対する権利の可用性の確認を必要とします



排他的権利を譲渡するには、以下を行う必要があります。



  1. 雇用契約で言及される公式の仕事に関する規定の存在。
  2. 独占的権利の移転に対する報酬の額は、給与に含まれるべきではないことを考慮してください。 便宜上、知的活動の各結果の疎外に対して、3,000ルーブルが請求されることを規定できます（たとえば）。
  3. 従業員との排他的権利の譲渡に署名します。 報酬額の条件は、契約書および法律に表示される必要があります。
  4. 雇用契約で、従業員が権利を公式の作品だけでなく、すべての作業資料および原資料にも譲渡することを規定する。
  5. すべての開発、労働時間中に作成されたソースコードが雇用者に属することを雇用契約に登録します。 フリーランサーとは少し違いますが、それは別の話です...


解決策：

顧客がサイトの権利の確認を求めた場合、従業員との契約を提供することによってのみこれを行うことができます。これは、権利が雇用主に譲渡され、排他的権利の譲渡金額が個別に割り当てられることを示します。



あなたが創造的な人であれば、これはあなたが聖霊を食べていると信じる理由を顧客に与えるものではありません。 あなたは自分で見つけたゴミ箱にいません。 仕事のためにお金を受け取らなければなりません。 これは正常です。 顧客と協力する際の戒めは次のとおりです。



  • 著者との関係を正式化する（彼が誰であるかに関係なく、従業員、フリーランサー、請負業者）;
  • 開発されたドキュメントに従ってビジネスプロセスを構築します。
  • CRMを介してのみジョブ割り当てのリストを作成します。


4番目の問題：法人のリーダーシップが変わり、彼らはあなたに支払いを拒否します



Webサイトなどの製品を完全に開発しました。 レイアウトの準備が完了し、すべてが見栄えが良く、すべてがクリック可能になり、リリースの準備ができました。 決定的な瞬間が来ました-彼らはあなたに支払いをしなければなりませんが、「あなたは何も注文していませんでした」と「あなたは誰なのか、私たちはあなたを知りません」ことが判明しました。 LLCの新しい経営陣はWebデザイナーのサービスをまったく必要としません。サイトも必要ありません。それらはすべて70を超えており、木製計算機で動作します。





写真：time-news.net



解決策：

正式に他の人事異動があなたを傷つけないように、顧客と契約します。 あなたにとって何が重要ですか？



  1. 契約（少なくとも両当事者の印鑑で署名されたスキャン）
  2. 契約書に署名済み（事前に合意した価格）
  3. 仕事の受け入れの指示された行為
  4. 顧客の主導で契約が終了した場合、実際に提供されたサービスの代金を支払わなければならないという契約の表示。 罰金さえ予見できます。


覚えている：誰もがアーティストを怒らせることができます！ はい、そして創造的な人々は彼らが戦争に行かないように配置されています。 それは価値があるだろうが！ そして、戦争を避けるために、ビジネスプロセスを正しく構築することは、より良いことです。
