Google I / O 2016：Firebaseプラットフォーム開発

こんにちは、Habr！ 1年半前、 Firebase開発チームがGoogleのフレンドリーなチームに加わりました 。 それ以来、BaaS（サービスとしてのバックエンド）は、開発者が110から45万人に4倍以上に成長しました。 そして今回も、DFID（リアルタイムデータベース）、ユーザー認証、ホスティングといった同じサービスをすべて提供し、アプリケーション開発を容易にしました。 次の段階に進むため、Firebaseの主要な拡張機能をご紹介します。











Firebaseは、Android、iOS、およびモバイルWebアプリケーションを構築するためのユニバーサルプラットフォームになりつつあります。 開発の簡素化、ユーザーの獲得と収益化を目的とした新しいツール、および新しい分析製品があなたを待っています。



私たちは、当初Firebaseで最初に定められた原則を常に忠実に守ってきました。

  • 開発者はユーザーであり、ユーザーエクスペリエンスは非常に重要です。 使いやすさ、高品質のドキュメント、直感的なAPIにより、開発者は満足しています。

  • クロスプラットフォーム。 同様に重要なのは、iOS、Android、モバイルWebです。

  • 可能な場合は製品の統合。 Firebaseには、1つのSDK、1つのコンソール、および質問への回答、サポート、および必要なすべてのドキュメントを取得できる1つの場所があります。 どの機能をどの組み合わせで組み合わせるかを選択すると、可能な限りデータを使用して最も速く快適な作業を提供することがタスクになります。





Firebase分析



Firebase Analyticsは、モバイルアプリケーションを分析するための新しい無料ツールです。 一部、彼はGoogleアナリティクスから何かを継承しましたが、いくつかの重要な違いがあります。



まず、Firebase Analyticsはユーザーとイベントを回避します。 セッション、ページビュー、アプリケーション画面は優れていますが、Googleには既にあります。 私たちの新製品は、ユーザーがアプリケーションで何をするかに焦点を当てています。 また、ユーザーがどこから来たのかを正確に知ることができます。有料広告とユーザーを引き付けるためのチャネルを追跡すると、明確なグラフと表が得られます。



第二に、合理的な統合の原則についてすでに説明しました-ここでは、最大限に実装されています。 FAは、アプリケーション内のユーザーアクティビティに関する信頼できる情報を提供します。これにより、新しい機能-Audiencesを実装できます。 オーディエンスにより、同様のプロパティを持つ異なるユーザーグループを識別できます。 その後、単一のFirebase環境内で異なる視聴者の個別の情報を受け取ることができます。



メッセージング、ストレージ、構成で開発をスピードアップ



最近のアプリケーションでは、インタラクティブなインタラクションをある程度使用していることが非常に多いため、バックエンドサービスを拡大しています。 Googleクラウドメッセージング-クラウドからデバイスにプッシュ通知を配信するための最も人気のあるサービスは、Firebaseに統合され、FCM- Firebaseクラウドメッセージングと呼ばれるようになりました。 完全に無料で、制限がなく、完全に最適化されており、クライアントデバイスのバッテリーを節約し、iOS、Android、および最新のブラウザーで最適に動作します。 FCMの中心となるのはスケーラビリティです。今日では、1700億（！）を超えるメッセージが毎日、約20億の異なるデバイスに送信されています。



画像、動画を保存する機能を追加して、ファイルを転送する方法を学習するように長い間求められてきました-現在、Google Cloud Storageのおかげで、このような機会も提供しています。 Firebase Storageを使用すると、開発者はファイルを簡単かつ安全にダウンロードして保存できます。また、Googleの堅牢なプラットフォームと技術は、比類のないアクセシビリティ、Googleサービスとの統合、スケーラビリティを提供します。 また、Firebase StorageのクライアントSDKの高度なロジックにも注意を払ったため、不安定なインターネット接続でもサービスが正常に機能します。



Firebase Remote Configを使用すると、開発者はアプリケーションパッケージを更新せずにアプリケーション要素をその場で微調整および更新し、アプリケーションストアで利用可能になるのを待ってからユーザーで更新できます。 アプリケーションの特定の要素のオン/オフを切り替えたり、特定のユーザーオーディエンスに更新プログラムを配布したりできます。これらはすべて、数回のクリックで実行できます。



もちろん、既存の製品-BDVR、ホスティング、ユーザー認証については忘れていません。 ホスティングは、すべての開発者向けの新しいインターフェースと無料のサードパーティドメインサポートを受け取りました。 データベースは近い将来にメジャーアップデートを受信しますが、これまでのところ何も言えません。 待ってください、アップデートはあなたを失望させません。 最大の更新はFirebase Authenticationでした。ここでは、新しいSDKと再設計されたバックエンド構造だけが、セキュリティ、信頼性、およびスケーラビリティの改善に必要です。 内部では、Googleユーザーアカウントで動作する他のGoogle製品と同じテクノロジーが使用されています。 また、リンクされたアカウントと電子メールの所有権の証明を追加しました。これは長い間求められてきたものです。

アプリケーション機能の拡大



最も有用なアプリケーションでさえ、遅かれ早かれユーザーの関心の低下に直面します。 Firebaseには、顧客の注目を集めるのに役立つ5つの効果的なツールがあります。



Firebase Notificationsは、Firebase Cloud Messaging APIに基づいて構築された新しいインターフェイスであり、Firebase AnalyticsおよびAudiencesの機能によって補完されます。 これにより、ソースコードエディターに触れることなく、アプリケーションのユーザーにターゲットを絞った通知を送信できます。 新しいプロモーション、割引、個人的な条件の提供、新しい機会への人々の注意を引くことについて話しやすくなりました。



Firebase Dynamic Linksは、2方向のリンクの作業を同時に改善します。 まず、リンクが「信頼できる」ものになりました。 つまり、ユーザーがクリックしてGoogle Playから初めてアプリケーションをダウンロードした場合でも、アプリケーション内の特定の場所（特定のプロジェクト、イベント、会社など）につながります。 「アプリケーションをダウンロードしました。次は何ですか」などの質問や、リンクの繰り返しクリックはありません。 このような「歓迎」は、ユーザーがアプリケーションを操作するという点で優れた結果を示しました。



第二に、リンクは実行している環境に応じて動的に目的を変えることができます：異なるデバイス、ブラウザ、クライアント-高度な＆utm_sourceのようなもの。



Firebase Invitesを使用すると、紹介ネットワークを2つの方法で構築できます-顧客に新しいネットワークをもたらすことができます。 最小の労力、最大の結果。



Firebase App Indexing （以前のGoogle App Indexing）を使用すると、Googleユーザーはアプリケーションから直接結果を見つけることができます。 インストールせずにアプリケーションを実行するAndroid Nの新機能、加速インストール（ユーザーがすべてを必要とする場合）、および新しいダイナミックリンク機能を考えると、製品に本当に興味がある人でユーザーベースを定期的に補充します。
