W3C仕様の標準化について

今日、私は再びWeb標準のトピックを取り上げたいと思いますが、少し異なった方法です。 W3Cのすべての支持者が防衛から攻撃に移行する時が来たと思います。



誰かが彼が何について話しているのか尋ねますか？ 彼は何かを禁止するつもりですか？ いいえ、何も禁止されたくありません。ウェブソフトウェア開発者の義務を消費者に見せたいです。



そのため、インターネットの技術標準を開発および実装する組織であるWorld Wide Web Consortium（W3C）を知っていますが、今日は残念ながら勧告のステータスがあります。 製品ブラウザ市場の主要製品の開発者がその「スリーブレス」開発に言及しているのは、まさにこのステータスのためです。 Internet Explorerが当時のNetscape Navigatorを打ち負かし、Microsoftが実際に市場を独占できるようになったという事実は、現在の状況における重要な役割ではありませんでした。 そして、私は彼女を責めません。 これは市場関係であり、同社は直接の競合他社を打ち負かすことができました。



IEがNNに勝利した結果、今日は何ができましたか？ 90年代後半、Webにアクセスした人の大半は、ITに関連する最後の人ではありませんでしたが、情報に基づいてソフトウェアを選択できましたが、今日は怠laな人だけがWebにアクセスできませんでした。 このため、コンピューターを使用することは完全にオプションです。 したがって、顧客が会社でどのブラウザを使用しているのかを質問することは珍しくありません。「5台のコンピューター」の目を、誓わないでください。 =）Vasyaの小学生または特定の会社のナターシャの秘書の現代ユーザーにとって、インターネットの概念は、たとえこの名前を知らなくてもInternet Explorerと密接にリンクしています。 そして、この同じInternet Explorerは、ほとんどの場合IE6で、W3Cの推奨事項を理解する上で絶望的に時代遅れで恐ろしいものです。 IE7でさえ、状況を適切に修正しませんでした。



オペレーティングシステムにバンドルされているIEブラウザの配信に関してマイクロソフトの立場に反対することはありませんが、その品質については非常に否定的です。



先日、同名のライバルInternet Explorerの開発者であるノルウェーの会社Operaは、レドモンドのブルージャイアントに公然と反対し、Microsoftへの苦情の欧州委員会への提出を知らせるプレスリリースを発行しました。 Operaの開発者は、Microsoftに支配的なデスクトップPC市場の使用をやめて、Internet Explorerを強制し、Web標準に従うように強制したいと考えています。



Operaの要件の本質は、2つの主要なポイントに削減されます。

  • Microsoftは、Internet Explorerと共にオペレーティングシステムの提供を停止するか、代替開発者のブラウザのプレインストールバージョンを含めるか、またはその両方を行う必要があります。 Operaによると、これはユーザーに本当の選択肢を提供します。
  • ブラウザーを開発する場合、Microsoftは該当する業界標準に従う必要があります。 サイトの作成者が既存の標準ではなく、サイトが大きな市場シェアを持つブラウザーで表示される方法によって、他のブラウザーが無視されるという不快な状況がありました。




Media Playerの場合のように、MicrosoftがWindowsオペレーティングシステムを使用してディストリビューションからIEを削除することを強制することは、前例のないほど難しいと思います。 IEコンポーネントはOSの奥深くまで浸透し、そのアプリケーションの多くで使用されています。 代替ブラウザがWindowsにバンドルされている状況は、私にとって理想的ではないようです。



しかし、ノルウェーの開発者の要件の2番目のポイントは、1つの条件下で非常に実行可能なタスクのように思えます。 W3C仕様は、国際的な業界標準として認識される必要があります。 この場合、Microsoftだけでなく、Webブラウザーの開発者の誰も、受け入れられている標準に違反する製品を販売する権利を持ちません。



ウェブ開発者、特にクライエントの開発者は、これにより安reliefのため息をつくことができ、低品質のブラウザに高品質のコードを適合させるのではなく、コードの改善に時間を割くことができます。 ユーザーと開発者の両方が勝ちます。



クライアントサイド開発者の主な頭痛の種は、W3C仕様の以下の主要な誤って実装された仕様です。

  • XHTML
  • CSS
  • ドム




さらに、HTMLは長い間標準になっており、現在のすべてのブラウザでのHTMLの実装は満足のいくものではないという事実にもかかわらず、HTMLもこのリストに含める必要があります。

したがって、私の提案は、W3Cの一部の仕様に工業規格のステータスを付与することを要求する請願書を発行することになります。



だから。

  1. HTML 4.01現在のステータス-1999年12月24日のW3C勧告。
  2. XHTML 1.0の現状-2000年1月26日のW3C勧告。
  3. CSS2.1現在のステータス-2007年7月19日の推奨候補。 ここで、請願をサポートする署名を収集する会社は、少なくとも6か月から1年かかり、その時点で、この仕様が推奨ステータスを受け取る可能性が最も高いことに注意してください。
  4. DOMレベル3コアの現状-2004年4月7日のW3C勧告。




このようなキャンペーンの好結果により、ブラウザー開発者は、適合証明書を取得して市場で製品を発売するために、製品を高レベルに改善する必要があります。



現時点では、開発された標準に準拠するためのアプリケーションの認証プログラムはありませんが、キャンペーンが成功すれば、この問題は解決されると思います。



問題と調査結果の詳細な議論の後、私と私の会社は、請願書を作成し、それをサポートするウェブサイトを作成する必要性を決定します。
