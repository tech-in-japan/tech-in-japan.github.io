インストーラーの作成に関与したことはありません。 したがって、この問題は徹底的に「グーグル化」されました。とりわけ、Habrからもこのような記事に出会いました。 このタスクのためのツールの選択は非常に幅広く、プロプライエタリ製品とオープンソース製品の両方が含まれます。 ここに私が「感じた」もののリストがあります

1.インストール、構成、および単純な（しかし非常に堅実な）スクリプト





公式サイトからプログラムをダウンロードしてインストールするのは難しくないと思います。 Inno Setup Compilerを起動すると、このようなウィンドウが表示されます



Inno Setupは恐ろしく悲しいことに私たちに会います...





これは何ですか 実際、これは単純なスクリプトエディターであり、構文の強調表示、コンパイル、起動ボタンが装備されています。 彼らは、このウィンドウで、将来のインストーラーのロジックを定義するスクリプトテキストの入力を開始することを期待しています。 さて、時間を無駄にしません。



まず、必要な定数を定義します 

;------------------------------------------------------------------------------ ; ;     Inno Setup 5.5.5 ; (c) maisvendoo, 15.04.2015 ; ;------------------------------------------------------------------------------ ;------------------------------------------------------------------------------ ;    ;------------------------------------------------------------------------------ ;   #define Name "Miramishi Painter" ;   #define Version "0.0.1" ; - #define Publisher "Miramishi" ;    #define URL "http://www.miramishi.com" ;    #define ExeName "Miramishi.exe"




これらの行はスクリプトコードで頻繁に発生するため、Cのように#defineディレクティブを使用して定義します。



スクリプトの本文はセクションに分割され、各セクションには独自の機能的な目的があります。 必須の[セットアップ]セクションでは、インストーラーとアンインストーラーのグローバルパラメーターを設定します。 



 ;------------------------------------------------------------------------------ ;   ;------------------------------------------------------------------------------ [Setup] ;   , ;  Tools -> Generate GUID AppId={{F3E2EDB6-78E8-4539-9C8B-A78F059D8647} ;  ,    AppName={#Name} AppVersion={#Version} AppPublisher={#Publisher} AppPublisherURL={#URL} AppSupportURL={#URL} AppUpdatesURL={#URL} ;   - DefaultDirName={pf}\{#Name} ;     "" DefaultGroupName={#Name} ; ,     setup     OutputDir=E:\work\test-setup OutputBaseFileName=test-setup ;   SetupIconFile=E:\work\Mirami\Mirami\icon.ico ;   Compression=lzma SolidCompression=yes




AddIdオプション-Windowsレジストリにアプリケーションを登録するために使用される一意のアプリケーション識別子（GUID）に細心の注意を払っています。 「ランタンから」ではなく、中括弧を開き、メニューから[ツール]-> [GUIDの生成]を選択して（またはShift + Ctrl + Gホットキーを使用して）生成します。 次に、システムにインストールするアプリケーションの名前、そのバージョン、開発会社のデータ、開発者のサイトのアドレス、テクニカルサポートおよびアップデートを指定します。



インストーラーがインストール用に提供するデフォルトのパスは、 DefaultDirNameオプションによって決まります。 この場合、変数{pf}は、対応するビット深度のProgram Filesディレクトリへのパスです。 DefaultGroupNameオプションは、[スタート]メニューのプログラムグループの名前を決定します。 アプリケーションの名前を示すために、上で示した名前マクロを使用し、中括弧とグリッドでフレーミングしていることに注意してください。



いくつかのオプションOutputDirとOutputBaseFileNameは、コンパイルされた「セットアップ」とその名前（拡張子なし）が書き込まれるディレクトリを指定します。 さらに、 SetupIconFileオプションを使用してtest-setup.exeのアイコンを取得する場所を示します。



このセクションの最後のオプションは、圧縮アルゴリズム（LZMA）を定義し、すべてのファイルが個別に圧縮されるのではなく、同時に圧縮されることを示します（ SolidCompression ）。



優れたインストーラーは、複数の言語をサポートする必要があります。 オプションの[Languages]セクションを使用して、「セットアップ」にそれを含めます。 このセクションがない場合は、英語が使用されます。 



 ;------------------------------------------------------------------------------ ;      ;------------------------------------------------------------------------------ [Languages] Name: "english"; MessagesFile: "compiler:Default.isl"; LicenseFile: "License_ENG.txt" Name: "russian"; MessagesFile: "compiler:Languages\Russian.isl"; LicenseFile: "License_RUS.txt"




このセクションの各行は、インストール中に使用される言語の1つを定義します。 文字列の構文はこちら 



 < >: < >




セミコロンは、パラメーターの区切り文字として使用されます。 Nameパラメーターは、言語の「名前」であり、一般に受け入れられている2文字の略語（「en」、「ru」、「de」など）が許可されています。 MessagesFileパラメーターは、インストール中に表示されるメッセージのテンプレートを取得する場所をコンパイラーに指示します。 これらのテンプレートは、Inno Setupコンパイラディレクトリに格納され、 コンパイラディレクティブで報告されます。 英語の場合、Default.islテンプレートが適しています。ロシア語の場合は、言語\ Russian.isl



LicenseFileパラメーターは、対応する言語のライセンステキストでファイルへのパスを指定します。



通常、インストーラは、たとえば、デスクトップにショートカットを作成するかどうかを決定するために提供します。 このようなインストールオプションは、オプションの[タスク]セクションで決定されます。 



 ;------------------------------------------------------------------------------ ;  -  ,      ;------------------------------------------------------------------------------ [Tasks] ;      Name: "desktopicon"; Description: "{cm:CreateDesktopIcon}"; GroupDescription: "{cm:AdditionalIcons}"; Flags: unchecked




ここで、名前は操作の名前を設定します-「desktopicom」-デスクトップ上にアイコンを作成します。 説明 -ユーザーに表示されるオプションを含むチェックボックスの説明。 建設業 



 {cm:< >}




インストールの開始時に選択された言語に対応する標準メッセージテキストを設定します。 GroupDescriptionパラメーターは、オプションを含むチェックボックスのグループの見出しです。 Flagsパラメーターは、特定のアクションとコントロールの状態を設定します。この場合、「デスクトップにショートカットを作成する」チェックボックスをオフにする必要があることを示します。



明確にするために-結果は次のようになります






ここで、どのファイルをディストリビューションに含めるか、インストール中にどこに配置するかを示します。 必要なセクション[Files]はこれに使用されます。 



 ;------------------------------------------------------------------------------ ; ,       ;------------------------------------------------------------------------------ [Files] ;   Source: "E:\work\Mirami\Mirami\bin\Release\Miramishi.exe"; DestDir: "{app}"; Flags: ignoreversion ;   Source: "E:\work\Mirami\Mirami\bin\Release\*"; DestDir: "{app}"; Flags: ignoreversion recursesubdirs createallsubdirs




ここに



  • ソース -ソースファイルへのパス。 MS VSプロジェクトのリリースディレクトリには、プログラムが動作するために必要なものがすべて揃っています
  • DestDir-インストールディレクトリ。{app}変数には、インストーラウィンドウでユーザーが選択したパスが含まれます。
  • フラグ -さまざまなフラグ。 この例では、実行可能ファイルの場合：システムに既に存在する場合、実行可能モジュールを上書きするときにプログラムのバージョンを無視します（ignorevarsion）。 他のファイルとディレクトリについては、バージョンも無視し、すべてのサブディレクトリとソースファイルを再帰的に含め（recursesubdirs）、そうでない場合はサブディレクトリを作成します（createallsubdirs）




最後に、すべてを美しくするために、オプションで、プログラムメニューとデスクトップ上の配置用のアイコンを取得する場所をコンパイラに示します。 



 ;------------------------------------------------------------------------------ ;  ,      ;------------------------------------------------------------------------------ [Icons] Name: "{group}\{#Name}"; Filename: "{app}\{#ExeName}" Name: "{commondesktop}\{#Name}"; Filename: "{app}\{#ExeName}"; Tasks: desktopicon




ここで、[スタート]メニューのグループとデスクトップの場合、アイコンは実行可能モジュールから取得する必要があることを示します。 当然、アイコンを「コンパイル」する必要があります。そうしないと、必要な場所にWindowsコレクションの標準アイコンが表示されます。



だから、すべてが準備ができているようだ Ctrl + F9を押して、インストーラーのビルドを試みます。 構文エラーがない場合、ビルドプロセスが開始されます

2. .NET Frameworkの展開





私たちが何かを忘れたことに気づかなかった？ C＃で作成されたアプリケーションは、それが構築されたフレームワークがないと機能しません（システムにない場合）。 適切なフレームワークをインストールする必要がありますが、これには必要です



  1. フレームワークの配布キットを受け取る方法を決定します。 2つのオプションがあります-プログラムの配布パッケージと一緒に配信するか、必要に応じてインストールするか、ターゲットマシンにオンラインでダウンロードするかです。 フレームワークをディストリビューションに含めるオプションに焦点を当てます。 フレームワーク自体、たとえばバージョン4.0は、 Microsoftから無料で入手できます。
  2. ターゲットシステムでのこのフレームワークの有無を検出します。 これを行うには、システムレジストリの状態を分析するロジックを記述する必要があります。




チェックする必要のあるレジストリの値は、 Microsoftの公式ドキュメントに記載されています。記事では、簡単な絞り込みを行います。



表1 。 .NET Frameworkのインストール済みバージョンを確認するレジストリキー

インストーラーの任意のロジックを実装するために、Inno Setupには[Code]セクションがあります。 このセクション内では、Pascal言語でロジック関数を実装するコードが実装されています。 このセクションの内容を別のdotnet.pasファイルに入れ、メインスクリプトに#includeディレクティブを含めます 



 ;------------------------------------------------------------------------------ ;       ;------------------------------------------------------------------------------ [Code] #include "dotnet.pas"




[コード]セクションに直接コードを入力することもできます。 このセクション内ではPascal構文が使用され、スクリプトの主要部分で使用されるセミコロンではなく、「//」シーケンスがコメントの前にあることに注意してください。



システムに.NETの正しいバージョンがあるかどうかを判断する関数を作成します 



 //----------------------------------------------------------------------------- //     //----------------------------------------------------------------------------- function IsDotNetDetected(version: string; release: cardinal): boolean; var reg_key: string; //     success: boolean; //     .NET release45: cardinal; //     4.5.x key_value: cardinal; //      sub_key: string; begin success := false; reg_key := 'SOFTWARE\Microsoft\NET Framework Setup\NDP\'; //  3.0 if Pos('v3.0', version) = 1 then begin sub_key := 'v3.0'; reg_key := reg_key + sub_key; success := RegQueryDWordValue(HKLM, reg_key, 'InstallSuccess', key_value); success := success and (key_value = 1); end; //  3.5 if Pos('v3.5', version) = 1 then begin sub_key := 'v3.5'; reg_key := reg_key + sub_key; success := RegQueryDWordValue(HKLM, reg_key, 'Install', key_value); success := success and (key_value = 1); end; //  4.0   if Pos('v4.0 Client Profile', version) = 1 then begin sub_key := 'v4\Client'; reg_key := reg_key + sub_key; success := RegQueryDWordValue(HKLM, reg_key, 'Install', key_value); success := success and (key_value = 1); end; //  4.0   if Pos('v4.0 Full Profile', version) = 1 then begin sub_key := 'v4\Full'; reg_key := reg_key + sub_key; success := RegQueryDWordValue(HKLM, reg_key, 'Install', key_value); success := success and (key_value = 1); end; //  4.5 if Pos('v4.5', version) = 1 then begin sub_key := 'v4\Full'; reg_key := reg_key + sub_key; success := RegQueryDWordValue(HKLM, reg_key, 'Release', release45); success := success and (release45 >= release); end; result := success; end;




豊富なコードにもかかわらず、その操作のロジックは非常に単純です-バージョンパラメーターの値に応じて、RegQueryDWordValue（...）関数を使用して、対応するレジストリキーの値が読み取られ、必要な値と比較されます（表1を参照）。 バージョン4.5では、リリースパラメータにリリース番号を追加で渡します。
