「SAP Coder 2017」の結果：スマート冷蔵庫、採用担当者向けの仮想アシスタントなど







SAP Coder 2017は、オープンSAPクラウドプラットフォームの開発者向けの最初のCISオンラインコンペティションです。 トップ10の受賞者の中には、救急車派遣システム、ATMネットワークの現金回収管理、小売価格の最適化など、多くの興味深いプロジェクトがありました。 そしてju審員は商品の出荷を監視するサービスに非常に興味を持っていたので、ホールで「ほとんど買われた」というフレーズが聞こえました。



コンテストは4月3日から6月28日まで開催され、参加者はクラウドプラットフォームSAP Cloud Platformに基づいてビジネスアプリケーションのプロトタイプを作成するのに約1か月かかりました。 あなたはあなたの仕事を提供するか、3つの提案された分野の1つを取ることができます-大量採用、小売販売促進、または冷凍装置管理。



決勝では、3つの競技者が2つのカテゴリ（独立した開発者と開発会社）のそれぞれで勝利を目指して戦いました。 このイベントはモスクワ市にあるレクサスドームの領土で開催されました。



プラットフォームとイノベーションの副社長であるロルフ・シューマンが決勝に臨みました。 彼は審査員長を務めただけでなく、高度な技術のおかげで私たちの日常生活で起こっている変化についても話しました。 そのため、ロルフはスマートスニーカーで決勝に進出しました。 モバイルアプリケーションを使用して、彼は距離、歩数、運動のその他のパラメーターを見つけることができました。 モノのインターネットは、私たちが考えるよりも近いものです。









シューマン氏はまた、SAPスポーツプラットフォームを実証しました。これは、スポーツチームと選手を効果的に管理できるSAPクラウドプラットフォームに基づいたソリューションです。 彼は、ペナルティーに関する一連のデータに基づいて、システムがどのように打撃が打たれるゴールのセクターとその結果を外部条件に応じて予測できるかを示しました。 そして、誰もが予測分析の力をほぼ信じていたとき、彼は実際に予測不可能性が依然として人類の強い側面であることを実証しました。









SAP CISのエグゼクティブディレクターであるナタリアパルメノバは、ロシアのデータセンターでSAPクラウドプラットフォームを立ち上げる作業について話しました。 打ち上げは2017年秋に予定されています。







その後、別の外国人ゲストが講演-B.ブラウンのセールスおよびサービスのイノベーションディレクター、ソーレンイェンスラウインガー。 彼は、外科手術でSAPクラウドプラットフォームとモノのインターネット技術を使用する可能性を実証しました。 同社は、手術器具の状態と会計、手術用セットの形成を監視するための便利なアプリケーションを顧客向けに作成しました。









参加者は新しい技術について興味を持って聞きましたが、緊張は収まりませんでした。誰もが否定を待っていました。



独立した開発者の指名









だから、最初の3つのファイナリストの簡単なスピーチ-個々の参加者。 3位は、TNT開発チームの「バイパスと制御」プロジェクトであるIgor DochievとAlexander Petrochenkovです。 彼らは、機器や機器を検査するためのタスクの設定と会計のプロセスを自動化するモバイルデバイス用のアプリケーションを提案しました。 プロトタイプはSAP Cloud Platformに実装され、完全に運用可能な状態になりました。 しかし、審査員は他のファイナリストを好みました。 TNT開発者はiPad Proタブレットを賞品として受け取りました。















審査員と言えば。 競争は、ロルフ・シューマン、アンドレイ・ゴリャイノフ、デニス・サヴキンだけで評価されたわけではありません。 また、Eurosetの情報技術部長であるAlexander Talalykin、Kaspersky LabのAlexey Lafitskyも参加しました。 したがって、主要なIT専門家の真剣な見方の下では、競合他社にとって容易ではありませんでした。









2位は Oleg Demyaniukで、彼は「スマート冷蔵庫」のアイデアの実現を提案しました。 SAP CPに基づくビジネスソリューションを使用すると、冷蔵機器での商品の可用性を制御したり、新しい冷蔵庫を簡単に追加したり、記録を保管したりできます。 物理的なプロトタイプの実装前に、問題は到達しませんでした。おそらく、これによりオレグが勝つことはできませんでした。 賞品として、彼はSAPのトレーニングコースまたは150,000ルーブルを選択する機会を得ました。















個人競技会の勝者はベロゼロフ家でした。 IlyaとTatyanaは、仮想販売アシスタントV-Salerを導入しました。 これは、ストアで収集されたビッグデータの処理に基づいて、購入者に他に提供できるものに関する推奨事項を提供するクロスプラットフォームソリューションです。



「私はわずかに異なる分野の開発者です。 最近、私はこのコンテストへのリンクが送られてきたので、ウェブ開発をすることにしました。勉強を始めたところです。 それは面白くなり、良いモチベーションになりました。努力することと勉強することがあります」とイリア・ベロゼロフは言いました。









彼によると、ソリューションの主要な部分は、SAPクラウドのコンポーネントの1つである予測分析サービスです。 エンジンはSAPUI5フレームワークで記述されています。 システムをテストするためのデータを提供しました。これは、実店舗の非個人化された情報で、1200万件のレコードです。 Ilyaは、クラウドでの作業をデザイナーと比較しました。デザイナーでは、すべてのリンクを構成し、独自のアルゴリズムを追加する必要がありました。



男たちは非常に鮮やかなプレゼンテーションを行い、良い写真と面白い例を追加し、審査員はそれを高く評価しました。 結果-最初の場所とSAP TechEd（または300,000ルーブル）でのラスベガス旅行。









開発会社への指名



多くの開発会社のファイナリストも深刻な戦いを始めました。



3位はIBA ITパークに行きました。 開発者は「スマート冷蔵庫」を管理するタスクを選択し、冷蔵庫の積み込み、保管された商品の保存期間を考慮し、コンテンツの変更をリアルタイムで考慮できる本格的なシステムを作成しました。 このソリューションは、モノのインターネットサービス、RFIDタグ、HANAプラットフォームを使用して情報を保存および処理します。



確かに、競技者が冷蔵庫を持ってきて、RFIDタグ付きのケフィアとハーフベーコンのボトルが中に入ったらすぐにシステムに現れることを実演すると、これはju審員を感動させるでしょう。















2位はEPAM Systemsのミンスク支店の代表者でした。 開発者は、IACube（Yakub）というソリューションを紹介しました-リクルーターの仮想アシスタントです。 このアプリケーションは複数のサイトでインテリジェントな検索を実行し、経験、スキル、タスクに基づいて最も関連性の高い履歴書を選択します。 使用されるコンポーネントは、SAP HANA Text AnalyticsおよびText Miningです。 IACubeは、EPAM自体の担当者の選択にすぐに参加し始めるかもしれません。















最後に、 最初の場所は TeamIdeaプロジェクトによって獲得されました。これは、商品の出荷を管理し、輸送を監視し、機器を計画するためのmoTIonサービスです。 このソリューションは、クラウドアプリケーションの機能とSAP CPプラットフォーム上のモノのインターネットを使用します。



「競争に参加することにしたのはなぜですか？ その時までに、私たちはすでにこのプラットフォームで開発を行っていて、積極的に研究していましたが、いくつかの考えがありました。 チームEdeaのSAP EWM / TMヘッドであるPavel Malko氏は、他の競合企業に対する当社の強さのテストとして、現在の状況、移動した場所、および正しい方向に移動しているかどうかを確認する機会として、競争を取りました。



彼によると、それは本質的にオプションの仕事であり、そのためには追加の動機付けを見つけることが重要でした。 競争に参加する前に、ソリューションのロジックは20％以下で解決されました。 1か月間、彼女は実用的なプロトタイプに連れて行かれました。



moTIonサービスはju審員に非常に興味を持っていたため、多くの新興企業にとって「ほとんど買われた」神聖なフレーズさえ言われました。 そしてその結果-競争での勝利。





















報酬として、受賞した3社はすべて、SAP Cloud Platformへの年次アクセスと、SAP App CenterおよびSAP Forum 2018でソリューションを宣伝する機会を得ました。



特別ノミネート



コンテスト参加者の中には、すでに完成品を持っている人が多く、「SAP Coder」の一部として、それをSAP Cloud Platformに転送しました。 3人の出場者に注目しました。



これは、救急車の集中派遣システムをクラウドに移行したIPG会社です。









ATMのネットワークの現金と集金を管理するサービスを備えた会社「サービスモデル」。









また、価格を最適化し、流通ネットワークでKPIを管理するためのソリューションを備えたSAPRUN。 この決定について詳しく説明します。 興味深いことに、このアプリケーションのメインアルゴリズムはPythonで実装され、Cloud Foundryを介してクラウドソリューションに接続されていました。









「Foundryで機能するPythonの部分であるSAPUI5へのインターフェイス全体を書き直し、HANAと連携するように適合させました。また、データを適切な形式で配信するアプリケーションをHANAで作成しました。」



開発者によると、クラウドプラットフォームの既存の機能に関するドキュメントの量が不十分であることに関連して、転送に関する一時的な問題がいくつかありました。 「しかし、SAPからのサポートがあったので、SAPに直接質問し、同僚がサポートしてくれて、説明してくれました」と、SAPRUNエグゼクティブディレクターのRuslan Zaripov氏は述べています。



また、3社すべてがSAP Cloud Platformへの年次アクセスと、SAP App CenterおよびSAP Forum 2018でソリューションを促進する能力を獲得しました。









別の出場者は、特別審査員賞-OpenBanking APIプロジェクトに参加したEkaterina Lyapinaを受賞しました。 このソリューションは、銀行にPSD2ディレクティブを実装し、新しいルールを考慮して、銀行システムに接続するためのサービスを提供します。 作業の過程で、Catherineはコードを1行も記述しませんでした;製品は標準のSAPクラウドコンポーネント-API Managementに基づいて構築されました。これにより、サービスのアクセスポリシーを構成し、請求を管理できます。 ファイナリストの賞品はiPad Proタブレットでした。















コンテスト「SAP Coder」は、短時間でもプログラマーが高度なソリューションを作成できることを示しています。 これには、有望なアイデア、自発性、および幅広い機能を備えた最新のクラウドプラットフォーム（SAPクラウドプラットフォームなど）が必要です。
