IM +は本日、独自のクロスプラットフォームBeepマルチメディアメッセージングサービスを開始します





こんにちは、Habr！



たまたま、私はすでに人気のあるIM +クライアントのイノベーションに関するhabrapostを2回書いています。 しばらくして、さまざまなコミュニティサイト（xda-developers.comのAndroidバージョン、Blackberry-on blackberrys.ruなど）

次に、Android 4.0 ICSで新しいIM + c Beepをテストしました。



今日、このサービスの正式な立ち上げが行われました。habrakatの下で、私はそれと私の個人的な印象についていくつかの言葉を書きます。







まず第一に、サービスのアイデアは新しいものでもオリジナルでもないことに注意してください-IMクライアントはかなり前にファイルを共有することを学びました。



新しいサービスは、SMSとMMSの無料の代替手段として位置付けられています。実際、データ転送がWi-Fi経由ではなく、モバイルネットワーク経由の場合、トラフィックは依然としてユーザーによって支払われます。 幸いなことに、これは通常SMSやMMSよりも安価です。







マルチメディアコンテンツを交換するために、個別の連絡先リストを作成する必要はありません。BeepがインストールされたIM +を持っている（および電話番号がアドレス帳の電話番号と一致する）すべてのユーザーが自動的に表示されます。 個人的には、この方法はあまり好きではありません。アドレス帳に作業中の一時的な連絡先がたくさんある場合、Beepを使用しているすべてのユーザーが一般リストに追加されるからです。



一般的に既存のアナログと比較すると、ビープ音に関連するすべての機能が存在することがわかります。 アスタリスクは、タブレット向けに最適化されていないアプリケーションのバージョンを示します。







楽しい瞬間-BeepはLiteとProバージョンのIM +の両方で完全に無料です。



仕事では、サービスはそれ自体を示しました...通常:)写真やビデオは、他のユーザーに送信されたときにあまり速くアップロードされず、ダウンロードされたコンテンツへの定期的なリンクを受け取ります。 写真、ビデオ、音声メモ、または場所へのリンクを送信できます。 配信アラートは問題なく機能します。 プッシュ通知はすべてのプラットフォームでサポートされています。



Beepの利点は、すべてのプラットフォームで無料でアクセスできると考えられます。このサービスの登場により、デバイスにインストールされているプログラムのリストを減らすことが可能になります。



見通しについては、新しいサービスは困難な時期に視聴者を獲得する必要があります-「自立型」アプリケーション（WhatsAppやSkypeなど）に加えて、組み込みのコンテンツ交換ツール（iOS用のiMessage、Android用のGtalk \ Google+）と競合する必要があるためです



PS：IM + Beepをインストールするためのボーナスと動機として、新しいRovio Angry Birdsテーマ（iOSバージョン用）が含まれています:)



PPS：そして、新製品に関連して、IM +のProバージョンのコストは一時的に1.99ドル（すべてのプラットフォーム）に削減されます。
