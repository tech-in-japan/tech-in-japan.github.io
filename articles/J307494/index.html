VDS / VPS：ホスティングからクラウドまで

ホスティングの歴史は、インターネットの発展に関連しています。 90年代に、プロバイダーは大規模なサイトホスティングサービスを専門とするさまざまな国に登場し始めました。 通常、信頼性が高く手頃なLinuxディストリビューションが、ホスティングサーバーのオペレーティングシステムとして使用されていました。









80年代にIBMメインフレーム用に開発された仮想化テクノロジーは、他のプラットフォームで非常に有用であることが判明しました。 仮想化ソフトウェアが登場しました-Parallels Virtuozzo、OpenVZ、VMware ESX、Microsoft Hyper-V、XenSource、Xen、KVM、HyperVM、FreeVPS、FreeBSD Jail、VDSManager、Solaris Zones。



異なる仮想化方法を使用します。 VMWareは物理ハードウェアを仮想化し、SWsoft Virtuozzo（現在のParallels）はオペレーティングシステムレベルです。 OSのゲストインスタンス-仮想プライベートサーバー（VPS）-ベースOSの環境で動作します。 このテクノロジーを使用すると、WindowsまたはLinuxの同じ物理サーバー上で分離された仮想環境である仮想コンテナーでOSの複数のコピーを実行できます。



仮想化は、利用可能な物理リソースの使用を最大化する適切な方法であることが証明されています。 そのため、ホスティングサービスプロバイダーの注目を集めています。 現在、 調査によると、企業の76％がすでにサーバー仮想化を使用しています。



VDS / VPSとは何ですか？



このタイプのホスティングサービスは、仮想化ソフトウェアとともに進化してきました。 VDS （Virtual Dedicated Server）またはVPS （Virtual Private Server）という用語は、物理上で実行される仮想専用サーバーを意味し、「プレクラウド」時代にほぼ10年前に登場しましたが、今日のVDS / VPSはクラウドサービスと見なされ、通常は古典的な定義に従って。



実際、両方の用語は1つのことを意味します-仮想専用サーバー：複数の仮想サーバーが同じ物理サーバー上で動作しますが、1つの仮想サーバーの動作の障害は、隣接する仮想サーバーの機能に影響しません。 これらのサービスには違いがあると考える人もいますが。









各仮想サーバーは物理的な作業をエミュレートし、VPSのお客様は完全に自由にそれを受け取ります：OSとソフトウェアをインストールし、アプリケーションをインストールし、ファイルを操作し、通常のサーバーの所有者として他のアクションを実行する管理者権限を持っています。 可能性は、割り当てられたリソースによってのみ制限されます。 ロシアでは、2000年代初頭にVPSサービスが普及しました 。



VPSの代替-仮想および物理ホスティング。



共有ホスティング



VPSとは異なり、いわゆる仮想（共有）ホスティングは、限られた構成と管理機能を提供し、ユーザーには事前に準備されたサーバーの容量が割り当てられます。 割り当てられたリソースの変更、追加のソフトウェアのインストール、サーバー設定の変更はできません。 変更できるのはコンテンツのみです。



物理サーバーのリソースはユーザー間で共有されるため、ピーク負荷時にリソースが不足する可能性があります。 通常、共有ホスティングは、大量のリソースと高レベルのセキュリティを必要としない小規模なプロジェクトやタスクに適しています。

従来の仮想ホスティングとは異なり、仮想サーバーVPS / VDSは通常、DDoS攻撃、データバックアップに対する保護を提供します。各仮想サーバーには独自のIPアドレスがあり、必要に応じて、追加のIPアドレスを取得できます。複数のサイトをホストする場合は、同じIPアドレスを使用することがよくあります。



仮想専用サーバーには、仮想ホスティングに固有の制限はありません。 任意の数のサイト、データベース、ドメインゾーンなどを作成できます。物理リソースには制限がありますが、VPSはそれらを受け取ることが保証されますが、共有ホスティングにはそのような保証はありません。 すでに述べたように、VPSの所有者はシステムおよびルートユーザー権限へのフルアクセス権を持ち、任意のソフトウェアをインストールできます。共有ホスティングでは、プロバイダーのみがこれを実行できます。









ただし、仮想専用サーバーのレンタルコストは通常​​、共有ホスティングのコストよりも高くなりますが、同等の物理サーバーのレンタルよりも大幅に低くなります。



物理的なホスティング



ホスティングサービス（ホスティング業者）を提供する会社は、サーバーをネットワークに接続する信頼性と品質、技術サポート、メンテナンス、機器のアップグレード、データバックアップ、セキュリティなどを担当します。多くのユーザーはこのサービスに満足していますが、特別なソフトウェアをインストールして管理し、一般的にサイトの完全な所有者になる必要がありますか？



解決策は、プロバイダーのサイトにクライアントサーバーをインストールして部分的に管理またはホスティングするサーバー機器のレンタルです。 プロジェクトリソースに高い要件があり、一定の時間まで、1つのオプションがありました。専用物理サーバーへの配置：レンタル、専用（専用）、またはプロバイダーのサイトにインストールされた顧客の機器（コロケーション）。



モスクワでは、無停電電源装置のサポート、必要な機器の選択、冷却、監視、セキュリティシステムを備えたいくつかの大規模なデータセンターが建設されました。 このタイプのサービスは多くの大手プロバイダーによって提供されていますが、非常に高価です。



専用サーバーは、データセンター内の独立した物理サーバーです。 その所有者は、すべてのサーバーリソースにアクセスし、それを構成することができます。 彼は任意のソフトウェアをインストールでき、すべてのサーバーデバイスにアクセスでき、オペレーティングシステムを変更できます。 共有ホスティング中に、ホスティング業者がオペレーティングシステムとインストールされたソフトウェアの更新を担当する場合、専用サーバーを使用する場合、所有者の責任となります。



しかし、VDS / VPSに戻ります。 このタイプのサービスは長年にわたって活発に開発されており、サイトまたはプロジェクトが仮想ホスティングの可能性を超えているが、専用サーバーを必要としない顧客に人気があります。









ホスティング市場でのVPSサービスの登場により、多くのユーザーは、より手頃な価格でタスクに適したオプションを選択する機会を得ています。



VDS / VPSの機能と利点



サービスの優れた管理機能を備えた低コストは、VPSの利点です。 もう1つの利点は、そのシンプルなスケーラビリティです。コンピューティングリソース、メモリ、またはストレージ容量を簡単に追加できます。 これらは一般的なクラウドサービスのプロパティであり、今日のVPSは原則として、その特性上、IaaS（「サービスとしてのインフラストラクチャ」）サービスです。 一方、共通のストレージシステムと通信チャネルは、一部の顧客がリソースの使用を悪用して他の顧客に損害を与える可能性があることを意味します。



既に述べたように、このサービスの基礎はサーバーの仮想化です。 たとえば、OpenVZ（オペレーティングシステムレベル）やKVM（ハードウェア）など、いくつかのタイプの仮想化をVPSユーザーに提供できます。



仮想化テクノロジは、仮想サーバーを選択する際の重要な要素の1つです。ソフトウェア（オペレーティングシステムのカーネルレベルでの仮想化）またはハードウェア（ハイパーバイザーによって制御される完全分離）。 最初のケースでは、システムの1つの共通カーネルとプロバイダーによって選択されたOSタイプが使用されます。Linuxサーバーの場合は CentOS、Debian、Ubuntuなど、 Windows VDSの場合は同じバージョンのWindowsです。









すべての仮想化システムには長所と短所があります。 仮想化の損失は通常、リソース全体の5〜10％と推定されますが、正確に計算することは困難です。多くの要因に依存します。 仮想専用サーバーサポート環境の中で、長年のリーダーの1つはVirtuozzoテクノロジーです。



アプリケーションのVirtuozzoの場合、すべてが1台のマシンで動作するように見えるため、リソースを集中的に使用するタスクを実行できます。 仮想化による非生産的な損失はわずかであり、開発者によると、1〜2％を占めています。



OpenVZ、Virtuozzo、VDSmanager、ソフトウェア仮想化。 その利点は、速度、低コストです。 マイナス：リソースのあいまいな区切り、高負荷は近隣に影響を与える可能性があります。 他のVPSがアイドル状態の場合、アクティブに動作している仮想サーバーはすべての未使用容量を使用できますが、別のVPSをアクティブにすると、指定されたパラメーターに従ってリソースが「カット」されます。 VPSごとに、大幅に異なる場合があります。



さらに、1つのOSとメモリの使用のおかげで、OSレベルの仮想化はサーバー上で最高の密度（コンテナーの最大数-仮想サーバー）を提供します。



OpenVZは、物理サーバー上にオペレーティングシステム（仮想環境、VE）の分離されたコピーを作成します。 仮想サーバーは一般的なLinuxカーネルを使用するため、お客様が選択できるOSは限られています。 通常、Ubuntu、Debian、またはCentOSです。 このテクノロジーの特徴は、高性能、使いやすさ、サーバー管理です。



完全な仮想化（Xen、KVM、およびHyper-Vテクノロジー）は、各参加者が必要なOSをインストールできる場合にカーネルを物理的に分離することです。 より信頼できると考えられている 仮想サーバーは互いに完全に分離されているため、OSの選択肢が広がります。 短所：OSのインストールまたは再起動に時間がかかり、支払いが高くなります。



たとえば、KVM（カーネルベースの仮想マシン）-いわゆるハードウェア仮想化。 OSの独自のカーネルを使用して、物理サーバーの完全に分離された類似物を作成します。 このような各サーバーには、RAMおよびディスク容量、ネットワークカードの独自の領域が割り当てられます。 この場合、任意のOSをインストールできますが、通常、リソースのスケーリングは制限されており、顧客はCPUコアの数またはRAM容量を常に変更できるとは限りません。 はい、そのようなサービスはより高価です。



何を選ぶ？



VPSは、ウェブサイトのホスティング、ソフトウェア開発、テスト、コンテンツ管理システム（CMS（WordPress、Drupal、Joomla）など）のホスティングによく使用されます。特に、物理サーバーの購入やレンタルができない小規模企業です。









VPSは、Webサイトとアプリケーションのほとんどの所有者に価格と機能の合理的なバランスを提供するため、一般的なタイプのホスティングです。



VPSサービスは、今日多くのホスティング業者によって提供されています。 このタイプのサービスを使用して解決できる機会、価格特性、およびタスクは非常に多様です。 顧客にVPSサービスを提供する各企業は、異なる機能セットを提供します。 これらのサービスは、関連するクラウドサービス、管理およびサポートサービス、 DDoS攻撃に対する保護を含むセキュリティサービス、ソフトウェアライセンスによって補完できます。 価格は、サポートおよび管理サービスのパッケージによって大きく異なります。



物理的な専用サーバーは、仮想専用サーバーよりもはるかに高価ですが、提供されるリソースに加えて、VPSはほぼ劣っています。 唯一の例外は、機器への直接アクセスを必要とするアプリケーションの使用です。 VPSサーバーの作成は数分で完了し、プロセス自体を自動化できます。









仮想サーバーは、サポートなしで提供できます（このオプションは安価ですが、OS管理とアプリケーションの知識なしでは提供できません）またはサポート付きで提供できます（サーバーを個別に保守する必要がない場合）。 仮想サーバーの選択方法は？ これは、予算とニーズ、実装されるタスクとプロジェクトの性質によって決まります。



ロシア市場にはこのサービスの多くのオファーがあります。 OSP Dataが実施したプロバイダー企業の調査からわかるように、仮想サーバーは依然として企業クライアントにとって最も手頃なクラウドサービスです。これらは10プロバイダーごとに9プロバイダー（92.3％）のオファーのリストに載っています。 顧客の間での人気という点では、仮想サーバーは電子メールの次に2番目の位置を占めています。調査参加者の3分の1のみがこのサービスを使用しています-37.3％。



多くのホスティングプロバイダーは、 安定性 、ディスクサブシステムの速度（フラッシュアレイの使用を開始した）およびCPU、インターネット接続の信頼性に関する規定のパラメーターへの準拠に重点を置いて、Windowsを実行する仮想サーバーサービスを提供します。 コストを最適化したいという願望は、一般的にITアウトソーシング、特にVPSおよびIaaSサービスの人気を高めます。 J'son＆Partners Consultingによると、ロシアのIaaS市場のボリュームは、2014年には25億レベルであったにもかかわらず、2018年までに70億ルーブルまで成長し、平均年間成長率は約30％になります。



ロシアでは、VPSホスティングが数十または数百のホスティングプロバイダーによって提供されています。 確かに、 2016年4月に実施された Timeweb モニタリングの結果は、 1日あたり100を超える訪問を行うロシアのサイトの数が、2015年10月の301,128から2016年3月の270,287に減少したことを示しました。 ほとんどの場合、VPSはサイトのホストに使用されるため、これはマイナス要因です。



仮想化ハイパーバイザーの中で、 調査によると 、VMware ESXiがリード（57％）、Microsoft Hyper-V（19％）が2位、「その他の仮想化ツール」（9％）が3位、Parallels、Citrix XenおよびKVMが4位（各5％）。 数年前、VMwareのシェアは80％を超えました。 Hyper-Vはどのようにして競合他社を圧迫しましたか？ VPSに適用されるこの技術のいくつかの利点を考えてみましょう。



Microsoft Hyper-V：利点と機能



Hyper-V仮想化テクノロジは、スタンドアロンのMicrosoft Hyper-V Serverフリーウェアとして、またはWindows Server 2012の一部として利用できます。これは、メインのWindows Server 2012 OSとは別の64ビットハイパーバイザーに基づいています。各ゲストOSは、独自のカーネルお​​よびハイパーバイザーによって割り当てられた仮想リソース。これにより、同じ物理サーバー上で異なるオペレーティングシステムを実行できます。 このテクノロジーにより、サーバーネイバーのアクティビティに関係なく、宣言されたすべてのリソースをいつでもクライアントが使用できるようになります。 Hyper-Vホスティングと通常のVPSの主な違いは、ハードウェアリソースが他の仮想マシンから完全に分離されていることです。



管理者権限により、仮想サーバーの所有者は、物理サーバーと同様に、システムの構成、ユーザーの作成、ソフトウェアのインストールを行うことができます。 Hyper-V VPSは、管理と構成の点で「鉄」サーバーに完全に類似しています。 任意のOS、WindowsまたはLinuxをインストールし、必要に応じて構成できます。 同様に、追加のソフトウェアをインストールして構成し、RAMの量、ディスク容量、IPアドレスなどのハードウェアリソースを構成できます。 VPSホスティングとは異なり、Hyper-Vホスティングは、サイトまたはアプリケーションのリソースをスケーリングする機能を保証し、スケーリングはサービスを停止することなくオンザフライで行われます。



このような仮想サーバーの使用は、サイトのホスティングに限定されません。 データベースサーバー、ターミナルサーバー 、DNS、FTPなど、他のサービスを提供でき、リモートワークステーションとして使用できます。 Microsoft Hyper-V VPSを使用すると、Windows Server 2012のライセンスの購入を節約できます。 すでにホスティングにバンドルされています。 さらに、このオプションを使用すると、共同作業でのサードパーティソフトウェアのライセンスコストを削減できます。リモートサーバーで使用できるライセンスコピーを仮想サーバーに1つインストールできます。 最後に、ホスティング事業者は追加のMicrosoftソフトウェア （SQL Server、SharePointなど）のレンタルを提供します。

標準のサーバー管理コンソールを使用して仮想インフラストラクチャ全体を簡単に管理し、災害復旧、サーバーおよびアプリケーションの統合のためのツールを使用できます。 データをバックアップする機能により、24時間連続した運用が保証されます。



Hyper-Vは、Webホスティング（負荷の高いサイトを含む）およびアプリケーションホスティング、テストとソフトウェア開発、サーバー統合、および仮想デスクトップ展開（VDI）に適しています。 物理サーバーを購入したり、インフラストラクチャに設備投資したりすることなく、これらすべてを魅力的な価格で提供します。
