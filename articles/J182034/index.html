Stack Overflowは、すべての問題の公式ドキュメントを置き換えます

MicrosoftやGoogleなどのソフトウェア会社は、ソフトウェアインターフェイス、サービス、およびプラットフォームについて説明する何百万もの記事を開発者向けに公開しています。 ドキュメントの作成に多大な労力を費やしました。



同時に、開発者自身が公式文書をどんどん読み、専門サイトで情報を受け取る頻度が増えている、とNinlabs Researchの調査は示しています。 開発者がStack Overflowを含むプログラマ向けの専門サイトに費やす時間の約50％が、公式のドキュメントがあるポータルではまったく費やされていないことが判明しました。



一部の企業は、このクラウドソーシングの傾向にすでに気付いています。 たとえば、YouTube API APIのドキュメントに関する開発者のサポートは最近、特にGoogle APIからStack Overflowに完全に移行しました。特に、プログラマー向けの質問と回答のWebサイトに何千ものYouTube APIの質問が蓄積されていました。



状況は他の多くのトピックでも同様です。 検索エンジンで高度に専門化された質問を検索する場合、検索の最初の行には、MSDNではなくStack Overflowが存在する可能性があります。 この調査では、実際の検索クエリの後、Stack Overflowサイトは公式のドキュメントがあるサイトよりも2〜10倍頻繁にアクセスされることが示されました。



開発者は、実際には、ブログ投稿を公開して質問に答えることにより、さまざまなAPIの代替クラウドソーシングドキュメントを作成します。 Ninlabs Researchの調査によると、外部調整のないAndroid開発者は、Stack Overflowの議論でAPIクラスの88％を詳細に説明しました。 公式ドキュメントにAPIクラスの26％のみのコード例が存在する場合、Stack Overflowでは、受け入れられる回答の56％に含まれています。 一般に、すべての公式のAndroidドキュメントには1711個のコードフラグメントが含まれており、Stack Overflowには156 747個のコードフラグメントがあります。



Ninlabs Researchの調査の一環として、17人のAndroidアプリ開発者のサーフィンの歴史を分析しました。 平均11週間で、彼らは9234回Stack Overflowにアクセスし 、公式のドキュメントでdeveloper.android.comにアクセスしたのは2547回だけでした。
