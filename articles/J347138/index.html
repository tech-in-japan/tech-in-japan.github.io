Python 3およびPygameでゲームを作成する：パート1

多くの開発者は、ゲームを作成するためにソフトウェア開発に参加します。 誰もがプロのゲーム開発者になることができるわけではありませんが、誰もが興味のある（そして多分利益を得て）自分のゲームを作成できます。 この5部構成のチュートリアルでは、Python 3と優れたPyGameフレームワークを使用して2次元のシングルプレイヤーゲームを作成する方法を説明します。

ゲームプログラミングの簡単な紹介



ゲームの主なものは、画面上のピクセルの動きと放出されるノイズです。 ほとんどすべてのビデオゲームにはこれらの要素があります。 この記事では、多くのネットワークプログラミングを必要とするクライアントサーバーゲームやマルチプレイヤーゲームについては検討しません。



メインサイクル



ゲームのメインループが実行され、一定の間隔で画面が更新されます。 それらはフレームレートと呼ばれ、動きの滑らかさを決定します。 通常、ゲームは1秒間に30〜60回画面を更新します。 頻度が少ない場合は、画面上のオブジェクトが動いているように見えます。



メインループ内には、イベントの処理、ゲームの状態の更新、画面上の現在の状態の描画という3つの主要な操作があります。



イベント処理



ゲーム内のイベントは、ゲームコードの制御外で発生するすべてのもので構成されていますが、ゲームの実行に関連しています。 たとえば、ブレイクアウトでプレイヤーが左矢印キーを押した場合、ゲームはラケットを左に移動する必要があります。 標準イベントは、キーのキーストローク（および押下）、マウスの動き、マウスクリック（特にメニュー内）、およびタイマーイベント（たとえば、特殊効果の効果は10秒間続く）です。



ステータス更新



すべてのゲームの核心はその状態です。つまり、画面上で追跡および描画するすべてのものです。 ブレイクアウトの場合、状態には、すべてのレンガの位置、ボールの位置と速度、ラケットの位置、寿命とポイントが含まれます。



ゲームを制御できるようにする補助状態もあります。



  • メニューは表示されていますか？
  • ゲームオーバーですか？
  • プレーヤーは勝ちましたか？


描画



ゲームは、幾何学的な形状、画像、テキストのレンダリングなど、画面にその状態を表示する必要があります。



ゲーム物理学



ほとんどのゲームでは、物理的環境がシミュレートされます。 ブレイクアウトでは、ボールはオブジェクトに跳ね返り、固体物理学の非常に近似したシステムを持ちます（それを呼び出すことができる場合）。



より洗練されたゲームでは、より洗練された現実的な物理システム（特に3Dゲーム）を使用できます。 また、カードゲームなどの一部のゲームでは、物理学がほとんどなく、これは完全に正常であることに注意してください。



AI（人工知能）



多くのゲームで、コンピューターの敵と戦うか、敵が私たちを殺そうとしています。 多くの場合、彼らはあたかも知性を持っているかのようにゲームの世界で振る舞います。



たとえば、敵はプレイヤーを追いかけ、その場所を知っています。 ブレイクアウトにはAIはありません。 プレイヤーは寒くて硬いレンガと戦います。 ただし、ゲームのAIは多くの場合非常に単純であり、疑似合理的な結果を提供する単純な（または複雑な）ルールに従います。



音を鳴らす



サウンドの再現は、ゲームのもう1つの重要な側面です。 一般的に、サウンドにはバックグラウンドミュージックとサウンドエフェクトの2種類があります。 Breakoutでは、さまざまなイベント中に再生する効果音のみを実装します。



バックグラウンドミュージックは、常にバックグラウンドで再生される音楽です。 一部のゲームでは使用されず、一部のゲームでは各レベルで変更されます。



生活、ポイント、レベル



ほとんどのゲームでは、プレイヤーは一定数のライフを持​​ち、ゲームが終了するとゲームは終了します。 また、ゲームには、よくプレーしていることを理解したり、自己改善の動機付けをしたり、記録を友人に見せたりするためのメガネがあります。 多くのゲームには、完全に異なるレベルまたは難易度が徐々に高くなるレベルがあります。

クラスGameObject



GameObjectは、自分自身をレンダリングし、境界を保存し、移動する方法を知っているビジュアルオブジェクトです。 Pygameにも同様の役割を果たすSpriteクラスがありますが、このチュートリアルでは、すべてが低レベルで機能し、既成の魔法にあまり依存しない方法を示します。 GameObjectクラスは次のようになります。 



 from pygame.rect import Rect class GameObject: def __init__(self, x, y, w, h, speed=(0,0)): self.bounds = Rect(x, y, w, h) self.speed = speed @property def left(self): return self.bounds.left @property def right(self): return self.bounds.right @property def top(self): return self.bounds.top @property def bottom(self): return self.bounds.bottom @property def width(self): return self.bounds.width @property def height(self): return self.bounds.height @property def center(self): return self.bounds.center @property def centerx(self): return self.bounds.centerx @property def centery(self): return self.bounds.centery def draw(self, surface): pass def move(self, dx, dy): self.bounds = self.bounds.move(dx, dy) def update(self): if self.speed == [0, 0]: return self.move(*self.speed)

メソッドで現在の速度に応じてオブジェクトを移動します。 draw()



メソッドでは何も行いませんが、サブクラスでオーバーライドする必要があります。



ゲームクラス



Gameクラスはゲームの中核です。 メインループで実行されます。 多くの便利な機能があります。 メソッドごとにメソッドを分析しましょう。



__init__()



メソッドは、Pygame自体、フォントシステム、およびサウンドミキサーを初期化します。 すべてのPygameゲームがすべてのコンポーネントを使用するわけではないため、3つの異なる呼び出しが必要です。したがって、使用するサブシステムを制御し、適切なパラメーターで必要なサブシステムのみを初期化できます。 このメソッドは、背景画像、メインサーフェス（すべてが描画される）、および正しいフレームレートのゲームタイマーを作成します。



self.objects要素には、レンダリングおよび更新が必要なすべてのゲームオブジェクトが格納されます。 さまざまなハンドラーが、特定のイベントで実行する必要があるハンドラー関数のリストを管理します。 



 import pygame import sys from collections import defaultdict class Game: def __init__(self, caption, width, height, back_image_filename, frame_rate): self.background_image = \ pygame.image.load(back_image_filename) self.frame_rate = frame_rate self.game_over = False self.objects = [] pygame.mixer.pre_init(44100, 16, 2, 4096) pygame.init() pygame.font.init() self.surface = pygame.display.set_mode((width, height)) pygame.display.set_caption(caption) self.clock = pygame.time.Clock() self.keydown_handlers = defaultdict(list) self.keyup_handlers = defaultdict(list) self.mouse_handlers = []


update()



およびdraw()



メソッドは非常に簡単です。 すべての管理対象ゲームオブジェクトをバイパスし、対応するメソッドを呼び出します。 2つのオブジェクトが画面上で互いに重なり合っている場合、オブジェクトのリストの順序によってどちらが最初にレンダリングされ、残りの部分が部分的にまたは完全に重なります。 



  def update(self): for o in self.objects: o.update() def draw(self): for o in self.objects: o.draw(self.surface)


handle_events()



メソッドは、キーボードイベントやマウスイベントなど、Pygameによって生成されたイベントをリッスンします。 イベントごとに、対応するタイプのイベントを処理するすべてのハンドラー関数を呼び出します。 



  def handle_events(self): for event in pygame.event.get(): if event.type == pygame.QUIT: pygame.quit() sys.exit() elif event.type == pygame.KEYDOWN: for handler in self.keydown_handlers[event.key]: handler(event.key) elif event.type == pygame.KEYUP: for handler in self.keydown_handlers[event.key]: handler(event.key) elif event.type in (pygame.MOUSEBUTTONDOWN, pygame.MOUSEBUTTONUP, pygame.MOUSEMOTION): for handler in self.mouse_handlers: handler(event.type, event.pos)


最後に、 run()



メソッドがメインループを実行します。 game_over



要素がTrueになるまで実行されます。 各反復で、背景画像をレンダリングし、 handle_events()



、 update()



およびdraw()



メソッドを順番に呼び出します。



次に、画面を更新します。つまり、現在の反復でレンダリングされたすべてのコンテンツを物理ディスプレイに書き込みます。 最後にclock.tick()



ことですが、次の繰り返しがいつ呼び出されるかを制御するためにclock.tick()



メソッドを呼び出します。 



  def run(self): while not self.game_over: self.surface.blit(self.background_image, (0, 0)) self.handle_events() self.update() self.draw() pygame.display.update() self.clock.tick(self.frame_rate)
