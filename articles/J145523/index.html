Beamer-組版プレゼンテーション

LaTeXは、あらゆる種類のドキュメントのコンパイルだけでなく、高品質のプレゼンテーションの作成にも適しています。 beamerパッケージを使用すると、PowerPointまたはImpressのプレゼンテーションに決して劣らないプレゼンテーションを作成できます。







プレゼンテーショントピックス

したがって、プレゼンテーションを開始するには、ドキュメントのクラスを指定する必要があります。 
\documentclass[10pt,pdf,hyperref={unicode}]{beamer}
次に、ラテックスドキュメントのすべての通常のオプション（エンコード、転送、追加パッケージなど）を示します。



ビーマースライドの標準の縦横比は4：3です。したがって、他の縦横比（16：9など）でプロジェクターのプレゼンテーションを作成する場合、\ documentclassコマンドのオプションの引数でaspectratio = 169を指定する必要があります。



次に、ドキュメントのトピックを選択する必要があります。 これらのテーマはたくさんあります（リストはビーマーのテーマ/テーマディレクトリにあります。私はこのディレクトリ/ usr / share / texmf-dist / tex / latex / beamer / themes / theme /を持っています）。 これは、\ usetheme {topic}コマンドで実行されます。 例として、いくつかのトピックを調べました。

提案されたトピックは多数あります（ただし、この場合は独自のトピックを作成できます。詳細については後で説明します）。 特定のテーマの配色が気に入らないが、テーマ自体が好きな場合は、\ usecolortheme {dove}コマンドを使用して配色を変更できます。 ビーマーのインストールで使用可能な配色は、themes / colorディレクトリで表示できます。 たとえば、スライドをプリンターで印刷する場合、鳩の配色を使用できます。





カラーテーマに加えて、ビーマーでは「内部」テーマと「外部」テーマを選択できます。 \ useinnertheme {theme}コマンドを使用すると、一部の内部要素（リストマーカーなど）の外観を変更できます。\ useoutertheme {theme}コマンドを使用すると、外部プレゼンテーション要素（フッター）を表示できます。



プレゼンテーションで要素の順次強調表示を使用している場合は、次のように言っておく価値があります。



コマンド\ setbeamercolor {element} {bg = color1、fg = color2}を使用して個々の要素の色を強制的に指定することにより、ビーマーのさらに深い色設定を実現できます。 ここで、「element」は色を変更する要素の名前です（たとえば、「通常のテキスト」はプレーンテキストです）、「bg」は背景色、「fg」はテキストの色です。



カラースキームを調整した後、ヘッダーとフッターをさらに「想起させる」ことができます。 beamer / themes / outerディレクトリの内容を例として使用すると、ヘッダーまたはフッターに目的の外観を与えるために何をする必要があるかを理解できます。 著者のフルネームと組織のショートネームをフッターの引用符で囲みたい場合、2番目のフィールドはプレゼンテーションのフルネーム（長すぎない場合）、3番目は日付とスライド番号/スライド数です。前文に追加する必要がありますドキュメントは次のとおりです。 



 \makeatletter \defbeamertemplate*{footline}{my theme}{ \leavevmode% \hbox{% \begin{beamercolorbox}[wd=.3\paperwidth,ht=2.25ex,dp=1ex,center]{author in head/foot}% \usebeamerfont{author in head/foot}% \insertauthor~~\beamer@ifempty{\insertshortinstitute}{}{(\insertshortinstitute)} \end{beamercolorbox}% \begin{beamercolorbox}[wd=.5\paperwidth,ht=2.25ex,dp=1ex,center]{title in head/foot}% \usebeamerfont{title in head/foot}\inserttitle \end{beamercolorbox}% \begin{beamercolorbox}[wd=.2\paperwidth,ht=2.25ex,dp=1ex,right]{date in head/foot}% \usebeamerfont{date in head/foot}\insertshortdate{}\hspace*{2em} \insertframenumber{} / \inserttotalframenumber\hspace*{2ex} \end{beamercolorbox}}% } \makeatother

「色」パラメータは、すでに定義されているか（上のカラーボックスと同様）、ユーザーが指定することができます（複数の色の混合方法を含む）。 透明度N（N-パーセント）の色を使用するために、「N！ 色。」 「color1！N！」という形式のレコード color2”（100-N）色の部分” color1”とN部分の色” color2”を混合します。 \ averagebackgroundcolorコマンドは、現在の背景色を返します。



色に加えて、フォントを変更することもできます。 テーマ/フォント/ディレクトリには、プレゼンテーションで使用できる「フォントテーマ」が含まれています。 \ usefonttheme {font}コマンドを使用して、「フォントテーマ」を変更できます。 たとえば、デフォルトのフォント（セリフなし）を使用したいが、セリフ付きのセリフでは、次のコマンドを使用します。 



 \usefonttheme[onlymath]{serif}


スライドの背景を変更する場合は、\ usebackgroundtemplateコマンドを使用します。このコマンドの内部には、画像の配置などの詳細な指示があります。 たとえば、フレーム全体の背景に画像を配置できます。 



 \usebackgroundtemplate{\includegraphics[width=\paperwidth,height=\paperheight]{penguin.jpg}}






主な分野



次に、プレゼンテーションの一般的なフィールドに入力します。 これは、ドキュメントのプリアンブルでも実行できます。 フィールド値を設定するためのコマンドのほとんどにはオプションの引数があります-同じフィールドの短い指定（この短い指定は通常フッターに配置されるため、同様に入力するのが理にかなっています）。 一部のフィールドは標準ラテックスです：\ title（プレゼンテーションタイトル）、\ author-author、\ date-date; そして、いくつかは具体的です：\ Institute-組織、\ subtitle-小見出し、\ logo-組織のロゴ（一部のプレゼンテーションスタイルではロゴを独自に配置しますが、ロゴを持たないものもあります-これは覚えておく必要があります）。



さらに、プリアンブルでは、\ AtBeginSectionコマンドを指定できます。これにより、プレゼンテーションの各セクションの先頭に追加情報を表示できます。 たとえば、現在のセクションを強調表示してコンテンツを表示するには、次のように記述します。 



 \AtBeginSection{ \begin{frame} \frametitle{} \tableofcontents[currentsection] \end{frame} }


同様に、各サブセクションの先頭で実行されるコードは、\ AtBeginSubsectionコマンドによって決定されます。 両方のチームにオプションの引数があります-その内容は、コマンドで定義されたセクション/サブセクションの前にアスタリスクを付けて配置されます（つまり、目次に分類されず、番号付けされません）。



スライド



さらに、ドキュメントの本文では、プレゼンテーションフレームのフレーム化を開始します。 各フレームはフレームに囲まれています。 タイトルとコンテンツをプレゼンテーションの先頭に配置する場合、標準コマンド\ maketitleと\ tableofcontentsを別々のフレームとしてフォーマットできます。



各フレーム内の論理ブロックを選択するには、必要なパラメーターでブロック環境を使用できます-ブロックの名前（定理、補題およびサンプル環境はデフォルトで英語名を持っているため、代わりにブロック環境を使用するか、これらのブロックの名前を再定義できます）。 exampleblock環境は別の色で強調表示されており、必須パラメーターとしてブロックの名前もあります。 一部のテキストに特別な注意を払いたい場合は、\ alertまたは\ color {color}コマンド内にテキストを配置できます。



フレームに逐語的環境または\ verbコマンドが含まれている場合、オプションのcontainsverbatimパラメーターをフレーム環境に追加する必要があります



containsverbatimパラメーターに加えて、フレーム環境には次のオプションオプションがあります。
  • label = label-スライドラベル（\ refコマンドとハイパーリンク用）;
  • b、c、t-垂直方向の配置（上端、中央-デフォルト、下端）;
  • プレーン-レイアウトなしのスライド。
  • shrink = number-フレームのコンテンツが画面に収まらない場合、ビーマーは少なくとも「パーセント」の割合で拡大しようとします。
  • スクイーズは、多くのオブジェクトをスライド上に「スクイーズ」するもう1つの方法です。垂直方向のギャップが最小化され、
  • fragile-「壊れやすい」テキストがフレーム内に配置されていることをビーマーに伝えます（たとえば、同じ逐語的な環境。


デフォルトでは、スライドは1つの列に配置されますが、列環境を使用すると、スライドを複数の列に分割できます。 この環境内の各列は、\ column {width}コマンドで指定されます。 たとえば、スライドを2列に分割するには、次のようにします。 



 \begin{frame}\frametitle{ } \begin{columns} \column{0.5\textwidth}    \column{0.5\textwidth}    \end{columns} \end{frame}




オーバーレイ



多くの場合、あなたが徐々に提出したいフレーム上の情報-ナレーションとして、行ごとに。 つまり、1つのフレームのコンテンツがいくつかの連続したスライドに分割されます。 これにはオーバーレイが使用されます-同じフレーム内の新しい情報へのさまざまなタイプの遷移。 すぐに注意します：実際には、各スライドにはpdfファイルの新しいページのレイアウトが必要ですが、メタ情報に注意を引く「適切な」pdfビューアーを使用すると、これらのトランジションは表示されません（プレゼンテーションにアニメーションオーバーレイを挿入することもできます）。



同じフレーム内の異なるスライド上のブロックの可視性を制御するには、次のコマンドを使用します（すぐに注意します。スライドの番号は1から始まります）。
  • \ onslide <N> {text}-テキストはスライド番号Nに表示され、残りは非アクティブなテキストの色でマークされます。
  • \ only <N> {text}-テキストはスライドNでのみ表示され、他のスペースを占有しません。
  • \ visible <N> {text}-テキストはスライドNにのみ存在しますが、他では表示されませんが、スペースを占有します。
  • \ invisible <N> {text}-スライドNでは、テキストは表示されませんが、スペースを占有し、残りは存在します。
  • \ alt <N> {text 1} {text 2}-スライドNでは「text 1」が表示され、残りでは「text 2」が表示されます。
  • \時間<N> {before} {text} {after}-スライドNに「テキスト」、前のスライドに「前」、後続のスライドに「後」が表示されます。
  • \ uncover <N> {text}-\ onslideコマンドの逆。


これらのコマンドはすべて、1つの段落を含む文に対してのみ機能します。 この方法でいくつかの段落のブロックを選択する必要がある場合、同じ名前の環境内にそれらを囲む必要があります：onlyenv、altenv、visibleenv、uncoverenv、invisibleenv。



これらのブロックに加えて、アイテム化および環境ポイントの列挙を複数のスライドに分割できます。このため、\ itemコマンドの直後に、この情報が表示されるスライドの番号を示す仕様を挿入する必要があります。 他のスライドでは表示されませんが、スペースを占有しますが、これらの非表示アイテムを非アクティブテキストの色で表示したい場合は、プリアンブルで示す必要があります。 



 \setbeamercovered{transparent}


スライド番号の仕様自体は山括弧で囲まれ、<ab、cd、e->の形式をとります。スライド番号の範囲はハイフンでリストされ、番号はカンマで区切られます。 これらの仕様は、\ alertなどの強調表示コマンドで指定することもできます。



ブロックを1つずつ表示する別のオプションは、各ブロックの後に\ pauseコマンドを指定することです。 このコマンドは、テーブル行の最後でも、プレーンテキスト内でも機能します。 つまり、このコマンドは、フレームをスライドに分割するように指示します。前のスライドをすべて前のスライドに残し、その後のすべてを次のスライドに転送します。



アニメーションとサウンド



ドキュメントへのハイパーリンクを含むファイルを再生できるAdobe Readerまたは別のプログラムを使用してプレゼンテーションを表示する場合、プレゼンテーションにビデオまたはサウンドファイルを含めることができます。



ビデオを有効にするには、コマンドを使用する必要があります 



 \includemovie[poster,mouse=true]{width}{height}{file}


次のようなサウンドを有効にできます。 



 \sound[options]{poster}{file}


オプション：オプション：ループ、リピート、パリンドローム、ボーダー幅、showcontrols、externalviewer、inlinesound（その本質は名前から明らかです）。



「poster」パラメータは、オーディオまたはビデオファイルを開始するためにスライドに表示されるものを決定します。 空白のままにすると、ファイルは自動的に再生されます。



メディアファイルをプレゼンテーションに導入するには、マルチメディアパッケージを手動で接続する必要がある場合があることに注意してください（ビーマーが「単独で」接続しない場合）。



media9パッケージを接続し、それを使用してメディアファイルを埋め込むこともできます。



ビーマー自体を使用すると、pdfを使用してオーバーレイをアニメーション化できます。このため、フレームはN個のスライドに分割され、ユーザー定義変数（カウンターまたは長さ）の値が変更されます。 \ animateコマンドは、このアニメーションを範囲スライドに適用します。 変数の変更は、コマンド\ animatevalue {variable} {initial value} {final value}を使用して行われます。 つまり スライド番号範囲「範囲」のユーザー定義変数は、値「初期値」から「最終値」に均一に変化します。



オーバーレイのアニメーションのタイプを設定するには、\ transOVR <slides> [options]コマンドを使用します。ここで、OVRは次のとおりです。
  • blindshorizo​​ntal-スライドは縦縞によって「カット」されます。
  • blindsvertical-スライドは横縞で「カット」されます。
  • boxin-古いスライドは画面の中央のポイントに「引っ張られ」ます。
  • ボックスアウト-新しいスライドが中央のポイントから「引き伸ばされ」ます。
  • ディゾルブ-スライドはモザイクに置き換えられます。
  • グリッター-ワイプとディゾルブの混合物;
  • splitverticalinは上と下から「一緒に引き寄せられます」。
  • splitverticaloutは上下に「伸び」ます。
  • ワイプ-次のスライドは左側に「離れます」。
「スライド」は、スライド内の特定のシーケンスのどの特定のオプションがトランジションを適用するかを示すオプションのオプションです。 「オプション」-オプション：期間=秒-移行時間。 方向=度-遷移の方向。



\ animateコマンドを使用する場合、各スライドの継続時間を指定する必要があります。そのためには、\ transduration {duration}コマンドを使用する必要があります。



bimerツールを使用してアニメーションを使用する例を次に示します。 



 \newcount\ooo \newdimen\offset \begin{frame} \animate<2-10> \animatevalue<1-10>{\ooo}{100}{0} \animatevalue<1-10>{\offset}{0cm}{5cm} \begin{colormixin}{\the\ooo!averagebackgroundcolor} \frametitle{ }  \hskip\offset\alert{ !} \end{colormixin} \transduration{0.1} \end{frame}


ここでは、アニメーションを作成するために、テキストの不透明度を設定する「ooo」カウンターと、「fade」という単語に対する「and fly away！」というフレーズのオフセットを設定する「offset」サイズが使用されます。 10枚のスライドのうち9枚以上で、これらの変数の値が変化し、その結果、テキストの不透明度が0から100まで10刻みで変化し、「フライングアウェイ」フレーズのオフセットが0から5センチメートルまで5 mm刻みで変化します。



その他



なぜなら ビーマーは自動的にhyperrefパッケージを接続し、ハイパーリンクをドキュメントに挿入できます。 たとえば、プレゼンテーションの最後にオプションのスライドを挿入し、その中にハイパーリンクを作成します（「突然」と尋ねられます-そしてすぐに-スリーブからエース）。 さらに、スライド内のオブジェクト（式など）へのハイパーリンクを作成できます。 一般的な原則は次のとおりです：参照するオブジェクトにはラベル（オプションの引数label =フレームで囲まれたラベルまたはフレーム内のコマンド\ label {label}）が付けられ、ハイパーリンクはコマンド\ hyperlink {label} {link text}で実行されます。 テキストに加えて、「ボタン」を挿入できます：\ beamerbutton {text}。 フレームに複数のスライドが含まれる場合、特定のフレームスライドを参照できます。このため、\ labelコマンドとその引数の間の三角括弧で、目的のスライドの番号[a]を示す必要があります。 同様に、特定のフレームの特定のスライドの要素を参照する必要がある場合は、ラベルの直後の最初のパラメーター（「ラベル」）でスライド番号の指定を指定できます。 特定のコマンドでハイパーリンクを指定することもできます：\ beamergotobutton（テキストの前に矢印アイコンがあります）、\ beamerskipbutton（二重矢印）、\ beamerreturnbutton（左矢印）。



適切なプレゼンテーションの最後に、参考文献を投稿することをお勧めします。 これも可能です：書誌の使い慣れた環境でスライドに置くか、bibtexを使用します。



プロジェクターとスクリーンを使用する場合、スクリーンを使用してプロンプトを表示できます。 このために書く 



 \setbeameroption{second mode text on second screen=[]}


ここで、オプションの「location」パラメーターは、プロジェクターに対してスクリーンがどこにあるか（上、下、左、右。デフォルトは右）をビーマーに伝えます。 フレーム環境の「typeset second」オプションは、フレーム内にスライドがあり、その内容が2番目の画面に表示されることをビーマーに伝えます。 そこで表示する必要があるものは、「only：slide number」という仕様で宣言されています。 例えば 



 \setbeameroption{second mode text on second screen} \begin{frame}[typeset second]      . \only<second>{ -  } \only<second:0>{ - } \end{frame}


説明と翻訳は、一方のテキストともう一方のテキストの2つの画面に表示することもできます。 引数を垂直バーで区切った\ twoscreenコマンドを定義します。左側のすべてが左側のモニターに、右側に-右側にすべて表示されます。 



 \setbeameroption{second mode text on second screen} \makeatletter \def\twoscreen#1{\tw@scr#1\relax} \long\def\tw@scr#1|#2\relax{\alt<second>{#2}{#1}} \makeatother ... \begin{document} \begin{frame}[typeset second] \frametitle{\twoscreen{ |   -  }} \begin{itemize} \item<1-> \twoscreen{|  }. \item<2-> \twoscreen{|  }. \item<3-> \twoscreen{|  }. \end{itemize} \end{frame}


ここで説明しなかった他のすべては、ユーザーガイドに記載されています 。 そしてここから最新バージョンをダウンロードしてください 。
