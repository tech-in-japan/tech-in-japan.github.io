SnapProtect 10 SP2：新機能

この記事では、 SnapProtectソフトウェアに関するトピックであるNetApp FASシステムのバックアップアーキテクチャに続き、 SnapProtectの新機能に焦点を当てます。 SnapProtectソフトウェアは、高速スナップショットNetAppとテープレプリケーションを組み合わせて、ダウンタイムとデータ損失を削減します。 これは、インフラストラクチャ内の単一の管理、作成、カタログ化、アプリケーション一貫性のある管理コンソールスナップショットであり、ディスクからディスクへのストレージライフサイクル（D2D2T）を提供します。







このバックアップスキームは、NetApp FASシステムのバックアップパラダイムに従って実行されます 。メインデータとバックアップデータはストレージシステムにあり、バックアップはハードウェアアシスタントスナップショットに基づいて実行され、アプリケーションと対話して、アプリケーション整合性バックアップを作成します。



バージョンSP 2以降の新機能：



データベースのクローン作成（SQLおよびOracle）



  • 例：顧客は、テスト/開発操作を実行するために、アプリケーション整合性のあるSQL / Oracle データベースクローンを作成したい
  • NetApp FlexClone機能（およびライセンス）およびスナップショット機能を使用して、一貫したデータベースクローニングを実現
  • クローニングはSnapProtect Managerから行われます


Virtual Server Agent（VSA）のマルチスレッド



  • 例：顧客がVMwareバックアップをより高速に実行したい
  • 複数の仮想マシンのアプリケーション整合性のあるVMwareバックアップを同時に実行できるようになり、プロセスが高速化されました。
  • 同時バックアップの数を構成できます


遅延インデックス付け



  • 例：顧客には非常に長い時間インデックスが付けられた多くのファイルがあり、バックアップの速度に影響する可能性があります。 お客様は、バックアップスケジュールとインデックス作成を分離したいと考えています。
  • 遅延インデックスを使用すると、バックアップが完了した後、2番目のリモートストレージシステムで実行でき、操作が非常に迅速に完了します。
  • メインシステムの負荷を取り除き、バックアップ時間を短縮します
  • インデックス作成を実行する場所を選択することができます：プライマリシステムまたはバックアップシステム


NASライブビュー



  • 例：顧客はNASから個々のファイルを回復できることを望んでいますが、インデックスを作成する必要はありません
  • NAS Live Browseは、ファイルをカタログ化せずにNASから個々のファイルを回復する機能を提供します
  • プライマリシステムとバックアップシステムを選択して、リカバリを実行する場所を表示および選択できます。
  • 「スナップショットから参照」とも呼ばれます


選択的コピー



  • 例：顧客は、最初の完全な週次バックアップのみをバックアップしたいと考えています。 たとえば週が金曜日から始まる場合、最初の完全バックアップは金曜日またはその日以降に実行され、アーカイブ用に選択されます
  • 特定のルールに基づいて、アーカイブの実行に使用されるメインスナップショットを1つ選択できます。
  • 完全バックアップとアーカイブに使用されます（ミラーレプリカには適用されません）


長期保持



  • 例：顧客は、通常行われるよりも長い期間、月次バックアップの保存期間を延長するビジネスタスクを持っています
  • 拡張保持機能を使用すると、顧客は1つの完全な月次バックアップの保存期間を1年に延長できます。
  • 完全バックアップの最大3つの追加「延長」期間を指定できます。


VMwareデータベースの例外



  • 例：VMwareバックアップから特定のデータベースを除外できます
  • Datastoreの例外機能により、VMwareサブクライアントセクションの[プロパティ]メニューの[フィルター]タブを使用して一部のデータベースを除外できます。


SMTapeのブロックレベルの増分バックアップ（BLI）



  • 例：顧客は、増分バックアップを実行して、テープメディアへのコピー操作の効率を高めたいと考えています。
  • SnapProtectは、 DOTのBLI機能をサポートして、完全なSMTapeの増分バックアップを実行します。


Oracleのサポートの改善



  • 例1：顧客はLinuxおよび/またはUNIXでOracle データベースをバックアップしました。 そして、Windows上のOracleにも同じ機会を持ちたい
  • 例2：Oracleログファイルは、SnapProtectから管理できずに大きくなります
  • SnapProtectは、Windows上のOracleのバックアップをサポートし、すべてのプラットフォームでログファイルの切り捨てをサポートするようになりました
  • ファイルシステムASMとRAWデバイスの両方でホストしているデータベースを含む


Clustered Data ONTAP 8.2でのVserver対応（またはVSM対応）NDMPテープバックアップ



  • 例：顧客はData ONTAP 8.2を使用し、 NDMPを使用してテープデバイスにバックアップします。 お客様は、 NDMPバックアップに影響を与えることなく、クラスターノード間でボリュームを移動できることを望んでいます。
  • SnapProtectは、clustered Data ONTAP 8.2以降のVserver対応NDMPテープバックアップをサポートするようになりました
  • プロセスは完全に透過的です
