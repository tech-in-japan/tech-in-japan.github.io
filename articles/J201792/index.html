フェールオーバークラスターHyper-V 2012

マイクロソフトの仮想化に出くわした人の多くは、自分にとって新しいものを見つけられないことにすぐに気付きます。これは、Hyper-V 2012に基づいたクラスターを作成するための小さなガイドです。

だから私たちが持っているもの：

  • 外部SASポートと4ポートネットワークカードを備えた2台のサーバー
  • HP P2000 G3 SASディスクストレージ
  • 私の場合、LACPサポートを備えたスイッチ、またはスタック内の2つは、2つのCisco 2960Sのスタックです。


MSによると、クラスターのすべてのノードで同じSASコントローラーを同じドライバーとファームウェアで使用することが望ましいとされています。



以下も必要です。

  • ADドメイン
  • 管理用にWindows 8または8.1がインストールされているドメイン内のコンピューターまたはラップトップ
  • リモートサーバー管理ツール-Windows 8または8.1用のRSAT


コントロールPCの場合、Windows 7は動作しないことに注意することが重要です。 win 7の一部のRSAT機能は、2012ファミリのサーバーでは機能しません。



Hyper-V Server 2012をダウンロードすると、無料の登録が必要になります。何らかの方法で2つのサーバーに配置してください。 コアバージョンを使用していることを思い出させてください。 サーバーにGUIは必要ありません。



ネットワークを設定しました

サーバーコンソールで、コマンドラインに移動し、同じ名前のコマンドを使用してPowershellに移動し、4つのインターフェイスすべてを1つの論理インターフェイスに結合して、LACPマージモードを設定します。

New-NetLbfoTeam –Name Team –TeamMembers “ethernet 2”,“ethernet 3”,“ethernet 4”,“ethernet 5”





Set-NetLbfoTeam -Name Team –TeamingMode LACP





スイッチでは、必要なポートチャネルのポートとLACPサポートを組み合わせて、トランクを有効にします。

Switch(config-if)#switchport mode trunk





Switch(config-if)#channel-group 1 mode active





その後、サーバー上のネットワークアダプターのリストで、新しいチームインターフェイスがUpステータスに表示されます。 

 PS C:\Users\administrator> Get-NetAdapter Name InterfaceDescription ifIndex Status ---- -------------------- ------- ------ Team Microsoft Network Adapter Multiplexo... 24 Up Ethernet 5 HP Ethernet 1Gb 4-port 366i Adapter #4 16 Up Ethernet 3 HP Ethernet 1Gb 4-port 366i Adapter #2 14 Up Ethernet 4 HP Ethernet 1Gb 4-port 366i Adapter #3 15 Up Ethernet 2 HP Ethernet 1Gb 4-port 366i Adapter 13 Up


そして、タイミングのステータス： 

 PS C:\Users\administrator> Get-NetLbfoTeam Name : Team Members : {Ethernet 4, Ethernet 2, Ethernet 5, Ethernet 3} TeamNics : Team TeamingMode : Lacp LoadBalancingAlgorithm : TransportPorts Status : Up




さらに、擬似グラフィカルメニューを使用して、IP設定を割り当て、リモートコントロールを有効にし、ドメインに入ることができます。 再起動後、便利な職場を使用し、Windows 8 / 8.1を搭載したPC /ラップトップを使用してすべてをリモートで実行できます。

上記のリンクを使用してRSATをインストールし、コントロールパネル（プログラム）でHyper-Vコンポーネントを有効にします。すでにハイパーバイザーを制御できますが、クラスターが必要です。



クラスターを作成する

サーバーマネージャースナップインからサーバーを追加すると、サーバーに関するすべての情報を確認できます。そこからpowershellをサーバーに接続し、フェールオーバークラスターコンポーネントを追加します。

Install-WindowsFeature Failover-Clustering





Install-WindowsFeature RSAT-Clustering-PowerShell





その後、フォールトトレランスマネージャーのスナップインから、GUIまたはPSを介してクラスターを作成します。

New-Cluster –Name HV-Cluster –Node Srv01,Srv02 –StaticAddress 192.168.10.5







共有ストレージを構成する

ストレージ上で、2つのLUNaを割り当てます。1つは仮想マシン用、もう1つはクォーラム用（1GB）

ディスクマネージャーを介してサーバー上のこれらの2つのディスクを初期化し、GPTに変換してフォーマットします。文字を割り当てることはできません。

クラスタテストでこれらのディスクを受け入れるには、バスタイプをRAIDからSASに変更する必要があります（ HP Source ）。レジストリのBusTypeパラメーターをパスHKEY_LOCAL_MACHINE \ SYSTEM \ CurrentControlSet \ Services \ hpcissss2 \ Parameters \に沿って8からAに変更する必要があります

MicrosoftからMPIOをインストールすることも必要です。

Install-WindowsFeature Multipath-IO





mpiocplスナップインにSASサポートを追加します。ディスクマネージャーで再起動した後、サーバーが2本のSASケーブルで接続されている場合、LUNは2倍になりません。

これらの操作の後、フェールオーバーマネージャースナップインでディスクを追加できます。追加したディスクからクォーラムも作成します。



仮想スイッチを作成する

Hyper-Vスナップインを使用して、外部仮想スイッチを作成し、「管理OSがアダプターにアクセスできるようにする」チェックボックスを設定します。Vlanは設定しません。 PS経由で同じこと：

New-VMSwitch “Lan” –NetAdapterName “Team” –AllowManagementOS:$True





仮想マシン自体に、同じ仮想スイッチで異なるVLANを持つ複数のネットワークアダプターを追加できます。



それだけで、仮想マシンを作成して共有ストレージに配置できます。共有ドライブはフォルダーC：\ ClusterStorage \ Volume1 \にマウントされていることを思い出してください。
