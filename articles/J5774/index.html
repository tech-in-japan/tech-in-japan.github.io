オンラインアプリケーションはオフラインにジャンプします

今日、Zimbraは、オフラインZimbraデスクトップクライアントのアルファ版を導入しました。 これにより、インターネットに接続していなくても、ブラウザを介してメールプログラム、テキストエディタ、その他のZimbra Webサービスを操作できます。 同様の機能は、Firefox 3や他のいくつかのソフトウェアプラットフォームにも登場します。



Webサービスへのオフラインアクセスは、最近関心が高まっている分野です。 誰もが独自の方法で問題を解決しようとしています。 たとえば、Firefox 3ブラウザーの開発者は、ローカルの一時データストアを介してWebサイトでオフライン作業の機能を実装することを約束します 。 Scrybeなどの新しいスタートアップもこの分野で実験中です。 ご存知のように、最近、Adobeは同様の問題を解決するためにApolloプラットフォームもリリースしました 。 Web開発者は、Apolloおよび他の同様のソフトウェア製品を使用して、使い慣れたツール（Flash、Flex、HTML、Javascript）を使用して、ブラウザーの外でもオフラインでも動作するデスクトッププログラムを作成できます。



Web 2.0プラットフォームをオフラインスペースに移行する開発のほとんどは、まだ市場に参入する準備ができていません。 同じApolloはアルファ版のScrybeにのみ存在します-クローズドテストでは、Firefox 3も最終リリースからは程遠い状態です。 Zimbraはこれらの製品のリリースを待たないことを決定し（結局、同社には600万人の有料加入者がいる）、代わりにZimbraパッケージからメールプログラム、オーガナイザー、連絡先リストおよびその他のプログラムをオフラインで起動するための独自のソフトウェアクライアントを作成した。







Zimbraデスクトップクライアントは、Windows、Mac、Linuxのバージョンとクロスプラットフォームであり、さまざまなブラウザー（Firefox、IE、Safari）でも動作します。 Webアプリケーションとすべてのユーザーデータは、 Apache Derbyデータベースのクライアントコンピューターに保存されます。 インターネットに接続すると、データはリアルタイムで同期されます。 オフラインクライアントは、サーバーバージョンと同じAjaxインターフェイスを介して動作します。 クライアントの唯一の制限は、ブラウザでのドラッグアンドドロップのサポートがないことですが、同社は将来のリリースでこの機能を追加することを約束しています。



Zimbra Desktopのソースコードは、他のすべてのZimbraプログラムのソースコードと同様にパブリックドメインで公開されるため、サードパーティの開発者はこの技術の実装方法を詳細に研究する機会を得ます。
