1日での価格管理システムの開発。 EDocLibの例

「自分用に」システムを独立して構成する機能は、おそらくビジネスを管理するためのソリューションの選択に影響を与える重要なパラメーターの1つです。 しかし、すべてのシステムは、あなたがすべてを自分で行うことができ、そしてそれは、特定の会社のために必要なだけのようではありません。

カットの下-ビジネスの規模や調達スタッフの能力に関係なく、会社の価格を管理するためにeDocLibシステムを構成する方法の例。



まず、少し理論。 eDocLibはECMシステム、またはより単純なビジネスプロセス管理システムであり、ご存知のように非常に異なる場合があります。 誰かが別の国に子会社を管理する必要がある、と誰か - 1つのオンラインストア。 ソリューションの機能は非常に幅広く、ドキュメント管理からチームワークの組織化までです。

主な特徴は、その上の自己eDocLib設計マニュアル、説明、およびする機能です。 この例は、本質的に、特定のケースで必要なツールを簡単に調整できることを示しています。

かなり広い範囲の商品を定期的に購入する必要がある会社を想像してください。もちろん、サプライヤーからこれらの商品の価格を追跡し、価格を更新する必要があります。

まず、システムの基本ディレクトリに入力します。

「組織」セクションの「連絡先」ディレクトリに、各サプライヤのデータを入力します-会社名、連絡先、通信用電話。





次 - 「私たちの組織」のセクションを作成するために同じ場所。 彼女の場合、管理、調達、販売、特定の従業員とその連絡先の表示など、自分の会社の階層を作成できます。 構造は、もちろん、どのようなものでも構いません-必要に応じて。





「Document rubrics」ディレクトリでは、必要なすべての商品を紹介し、それらを別々のグループ（「消耗品」、「ノートブック」など）に配布します。 すぐに、アイテムグループごとに価格表を検索およびソートできるようになると言わなければなりません。





その後、私たちは、新しいディレクトリと名前を付け、「配信の利用規約」を作成します。 信用の機会と締め切りようにプリペイドとしなければならない - ここでは、データ供給のオプションがご紹介します。 これらのカテゴリのためにも、現在合った価格を検索し、選択するために行われます。





その後、システムを構成する必要があります。 これを行うには、コンフィギュレータで「Price List」という名前の新しいタイプのドキュメントを作成し、フォームエディタでその入力フォームを開発し、「Document Groups」ディレクトリで「Price Lists Equipment」グループを開きます。 ここでは、このグループのドキュメントでのアクションについて、電子メールで通知するように設定します。





例えば、2週間 - 一定の時間のために更新されていないこれらの価格表の色を単離することが可能です。 これを行うには、Color Filtersディレクトリで「期限切れ」フィルターを作成します。 その後、価格の送信先となる個人用フォルダー「Price Lists」を作成し、その中に検索フィルターを構成します-アイテム、番号、サプライヤーごとに必要なものをすべて想定します。 次に、これらのフィルターに「期限切れ」カラーフィルターを追加します。したがって、2週間以上更新されていないすべての価格がすぐに表示されます。





最後のステップ：一般的な検索クエリを作成して、主要な詳細の価格を選択し、個人用のリクエストを作成および編集した価格について通知します。 実際、それがすべてです。

製品範囲のグループ数、サプライヤーのリスト、および供給オプション-これらはすべて、会社のニーズに応じて拡張および変更できます。

結果は何ですか？ 各ベンダーのために一度、個別の価格表および価格表で提示命名法を選択しました。 全ての価格は、個人用フォルダ、「価格」に配置されています。

アップグレード後、新しいバージョンでは、最後の更新の日付、または変更する必要があるその他の詳細と価格リストを作成されます。 新しいバージョンはユーザーの個人用フォルダーにも送信されます。これと並行して、更新された価格へのリンクを含む通知が電子メールに送信されます。





この例では、アイテムごとに価格を選択する3つの方法があります。適切なフィルターを使用する、検索クエリを使用する、見出しで直接使用する。 検索は、検索クエリを使用して行われる場合、検索は命名上にある他の検索パラメータと組み合わせることができます。 左側で目的のアイテムを選択して見出し検索を行う場合、このグループのアイテム位置を含む価格を右側に表示できます。

eDocLibシステムには、他にも多くの機能があります。 詳細については、 www.eos.ru / eos_products / eos_eDocLibをご覧ください 。 また、現在開発中のシステムの使用例とデモサイトです。



システム要件は次のとおりです。 データベースサーバー：Microsoft SQL Server DBMS-OS Windows Server 2003/2008、Microsoft SQL Server 2000/2005/2008 DBMS（フルテキスト検索がインストールされているすべてのバリアント）で使用します。 ORACLE DBMS、ORACLE、Oracle 9i DBMS（バージョン9.2.0.4以降）またはOracle 10g（Oracle Textがインストールされている両方のバージョン）をサポートするOSを使用します。

アプリケーションサーバー：OS Windows 2003/2008 Server、Microsoft .NET Framework v3.5 SP1、Microsoft Internet Information Servicesバージョン6.0、7.0、対応するバージョンのOracle DBMSのクライアント部分（Oracle DBMSを使用する場合のみ）

データベースサーバーとアプリケーションサーバーは、同じサーバーで物理的にホストできます。 この場合、OS Windows Server 2003/2008でのみ動作することが可能です。

クライアントの一部：OSのWindows 2000 Professionalの/ Windows XP Professionalの/ Windows Vistaでは 、マイクロソフトのInternet Explorer 6.0、7.0。
