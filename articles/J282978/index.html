Microsoft DevCon 2016-コミュニティトラックスピーカーの最終波の紹介









こんにちは



毎年恒例のDevCon 2016 Developer Conferenceでスピーカーを発表し続けます！ 今年、DevConは参加者に新しいプログラム形式と、特別なパスパスオファーを含むさまざまなチケットタイプを提供します。 以前の出版物では、会議の内容とメイントラックのリスト（Windows、Office、Azure、スタートアップ、コミュニティトラック）を発表しました。 そして、コミュニティトラックスピーカーの最初の波を発表しました。 トラックのすべてのスピーカーを教えて紹介する時が来ました。各スピーカーはそれぞれの分野で疑いのない専門家であり、業界の専門家として認められています。



また、招待された専門家がITのさまざまな分野の代表者であり、そのレポートがクラウド、モノのインターネット、DevOps、テスト、柔軟な方法論、仮想現実など、まったく異なるトピックをカバーしていることも非常に嬉しく思います。









レポート：大規模プロジェクトでのスクラムプラクティスの実装



スクラムは、アジャイルソフトウェア開発方法論を実装するための最も一般的なアプローチの1つです。 彼には、従わなければならない特定の規則と法律があります。 一方、実践が示すように、すべての規則を順守することは常に可能とは限りません。 このレポートでは、スイスの銀行向けのソフトウェアを作成する際に開発チームと運用チームの例として方法論を使用して、SCRUMメカニズムを変更する必要がある方法と理由を説明します。



















レポート：ゲームの収益性を高める方法



このレポートは、何百万人もの幸せなプレーヤーによって確認されたモバイルアプリケーションからの利益を保証する方法の秘密を明らかにします。 スピーカーは、モバイルアプリケーションまたはゲームからの収入を大幅に増加させるさまざまなニュアンスとトリックについて話します。 また、グローバル市場で成功するための可能なステップの例もあります。



















レポート：開発者向けのコンテナとマイクロサービス



製品および開発チームのコンテナには何が役立ちますか？ すべてをマイクロサービスに書き換える必要がありますか？ どのテクノロジーを選択しますか？ レポートは、これらの質問に答えるのに役立つ現在の技術とパターンに関する情報を提供します。



















レポート：オープンソースのベンチマーク



講演者は数年にわたってマイクロオプティマイゼーションのトピックを扱っており、オープンソースライブラリBenchmarkDotNetのメンテナーです。 このライブラリは、特定の方法が機能する時間を測定する「正しい」ベンチマークを作成するのに役立ちます。 ベンチマークは非常に複雑ですが、同時に非常に興味深いトピックであり、残念ながら多くのプログラマーが理解しているわけではありません（少なくとも基本を知っていると非常に便利です）。 このレポートでは、最新の.NETベンチマークの文化を形成する上でのオープンソースコミュニティの役割について説明します。スピーカーは、一見すると思えるほどコードの数行の速度を評価するのが簡単ではない、いくつかの面白い例を示します。

















レポート：シンプルでスケーラブルなバックエンド



作成するアプリケーションが複雑になる理由は何ですか？ この哲学的な質問への答えは、知識と経験に基づいて、IT業界の専門家によって毎日試行されています。 これを支援するために、最良のエンジニアリングプラクティスと設計パターンが設計されています。 スピーチで、スピーカーは複雑な主題分野でバックエンドを作成した経験を共有し、NuClear Riverプロジェクトについて話します。これは、いくつかのビジネス問題の解決を大幅に簡素化できる読み取りモデルを構築するためのオープンソースツールです。























レポート：個人的な体験からAzure Active Directory APIセキュリティモデルを確認する



アプリケーションとは何ですか？統合とはどういう意味ですか？ このレポートでは、スピーカーはさまざまな側面を提示し、多くの質問に答えます。 同意フレームワーク、認証と承認、利用可能な認証プロトコルの概要、アクセス方法などが含まれます。

























レポート：30分でデータベースのパフォーマンスを破壊する方法



データベース管理、特にSQL Serverは、最も難しいプロセスではありません。 比較的小規模なデータベースでは、サーバーを通常の効率的な継続的な運用に構成するのに1営業日しかかかりません。 ただし、インターネット上の推奨事項のほとんどには、表面的に見ると「美しく」見える非常に有害なヒントが含まれていますが、実際にはパフォーマンスが大幅に低下します。 スピーカーは、実際の例を使用して、データベースサーバーを効率的に運用するための「良い点と悪い点」について話します。















レポート：どのようにファームを育てたか、またはモバイルデバイスでの自動化のテストにおける実際の経験



このレポートでは、スピーカーは開発者にとって最も重要であると同時に物議を醸すトピックの1つであるテストのトピックに触れます。 過去数年間、彼のチームはモバイルデバイス用のDevExtremeフレームワークに取り組んできました。このストーリーはさまざまなデバイスでのテストに専念します。 発見と不作為、敗北と勝利、そしてさまざまなお気に入りの熊手を踏んで、チームはまだ夢の自動テストを行うことができた方法を報告します。

















レポート：ドードーピザ：成長酵母



チームは、Azureに移行してスケーリングを開始するという課題に直面していました。 現在Dodo Pizza社は月に10％成長しており、速度は落ちません。 レポートでは、会社の創立者が、ビジネスに合わせてどのような変更を加えているか、Azureがどのように役立つかを説明します。



スピーカー：Gleb Lesnikov-システム管理者およびテクニカルサポート責任者、Dodo Pizza

















レポート：VR / ARでのユーザーインタラクション：進化か革命か？



新しいテクノロジーの出現により、UI / UX、視覚化、ユーザーインタラクションに新たなマイルストーンがもたらされます。 このレポートでは、参加者はスピーカーと一緒にすべてに飛び込み、開発の段階を検討します。 彼らはどのように変化し、現在利用できるものがどのように登場したかを学びます。 VR / ARが視覚化とUI / UXにもたらした傾向を明らかにし、開発の良い点と悪い点を検討します。 未来のインターフェースで何が待ち受けているか、いつ来るか想像してみてください。 また、バーチャルリアリティテクノロジーがユーザーインタラクションの世界に革命をもたらすかどうか、VR / ARメガネが「必須」デバイスになるかどうかを調べてみます。

















レポート：モノのインターネット、クラウド、モバイルアプリで醸造所を構築する方法



レポートでは、Azure、IoT、および分析の非常に有用な使用法、つまりクラウド醸造所管理システムの構築について検討します。 参加者は、家庭での穀物醸造の正しいプロセスに精通し、Azure IoT Hub、Stream Analytics、Windows 10 IoTを使用して自動化できるプロセスを見つけます。

















レポート：iOS、Mac、Android用のOfficeアドインのデバッグ



2015年に、OfficeアドインはiOSおよびOSXで部分的に利用可能になりました。 統合されたJava Script APIにもかかわらず、新しいプラットフォームの出現により、すべてのプラットフォームのソリューションをテストおよびデバッグする必要が生じました。 レポートの一部として、スピーカーは、Webアプリケーションのリモートデバッグを可能にするweinreおよびngrokツール、およびOffice用アドインのデバッグに使用する機能について説明します。

















レポート：プラットフォームコネクテッドカーを構築するためのMicrosoft.ServiceFab



オンラインストアで商品を購入するtwitterで制御されるコーヒーマシンや冷蔵庫を持っている人は誰も驚くことではありません。 コネクテッドカーソリューションを使用すると、何千人ものドライバーが車をリモートコントロールできます。また、自動車メーカーとディーラーは、さらに分析するために、自動車センサーからの信号とデータを診断、集計し、これまでにない方法で顧客の世話をすることができます。 このレポートは、Service Fabricを使用してコネクテッドカープラットフォームを構築する方法と、これを実現するために会社が使用するツールとテクノロジーに関するものです。

















レポート：Azure Functions-クラウド内のサービスとしてのソフトウェアロジック



Azure Functionsは、イベントに応じてオンデマンドコンピューティングを提供する機能を提供します。 このサービスは、市場をリードするPaaSプラットフォームAzure上に構築されています。 このレポート全体で、Azure Functionsが既存のWebJobsインフラストラクチャを拡張し、Azureサービス、SaaS製品、ローカルシステムの両方から発生するイベントに応じて簡単にコードを実行する方法を学習します。 また、Blob Storage、Table Storage、Azure Queues、Service Bus、Event Hubs、HTTPリクエスト、またはタイマーによって生成されたイベントに応答する直感的なブラウザーユーザーインターフェイスを使用して、JavaScriptおよびC＃で記述されたAzure Functionsコードを作成およびデプロイする方法についても説明します。



スピーカー：Marcin Borecki-Microsoft、Hedgehog FundのCloudExpert.pl













レポート：72時間で継続的インテグレーションを構築する実際的な例-社内でプロセスを構築した方法。



DevOpsは優れていますが、小さなステップでそれに行く必要があります。 会社に効果的な開発プロセスと品質管理を構築した経験がない場合は、たとえば、継続的インテグレーションのプロセスをセットアップするなどして、小規模に始める必要があります。 レポートでは、スピーカーは、実際に社内でCIをできるだけ早く実装する方法を示します。

















レポート：広大さを受け入れる、または1つのXAMLプロジェクトで複数のMVVMフレームワークを使用する方法



一部のXAML開発者は、プロジェクト間でお気に入りのMVVMフレームワークを使用します。 他の人は毎回新しいプロジェクトのために新しいMVVMフレームワークを選択します。 レポートでは、スピーカーはさらに進んで、同じプロジェクトで複数のMVVMフレームワークを同時に使用でき、それぞれが最適な問題を解決できることを共有します。 参加者は、古き良きPrismと流行に敏感なReactiveUIを組み合わせて、両方のフレームワークを活用する方法を学びます。

システムの速度と信頼性を知っていますか？ どのくらい効果的ですか？ パフォーマンスの問題をローカライズする方法は？ レポートでは、スピーカーは、Azureクラウドアプリケーションのパフォーマンスカウンターの収集と分析のコンテキストでこれらの質問に答えようとします。
