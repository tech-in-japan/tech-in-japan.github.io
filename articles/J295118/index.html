CRMのテレフォニーとその選択方法

現在、テレフォニーのないCRMシステムを想像することはほとんど不可能です。 もちろん、CRMでテレフォニーを使用しない企業もありますが、現在ではその数はますます少なくなっています。



テレフォニーがどのように機能するか、CRMに存在するオプション、それらがどのように異なるか、このオプションまたはそのオプションの長所と短所をクライアントに説明する必要があります。 私は常にCRMとテレフォニーの統合に携わっており、この記事では知識を一般化し、共有することを決定しました。



この記事は、以前に書いた記事「CRMシステムとは何か、それらを正しく選択する方法は？」の続きと見なす必要があります。



CRMのテレフォニーとは何ですか。



テレフォニーにより、双方向通信が行われ、発信コールがCRMシステムから直接記録され、CRMのクライアントからコールが受信されます。 ただし、CRMでのテレフォニーの利点はこれに限定されません。 テレフォニーは便利なだけでなく、番号やその他の不要なアクションをダイヤルすることなく、CRMシステムで電話をかけたり受けたりします。



コネクテッドテレフォニーは、営業部門の統計情報の最も重要なソースの1つでもあります。 一定の期間の通話時間、応答済み、未応答の通話数、着信、発信通話の数がわかれば、1人または別のセールスマネージャーがどの程度うまくいったか、どのくらいうまく処理できたかを知ることができます。



たとえば、トランザクション数とコール数の比率などのレポートがあります。 このようなレポートに基づいて、従業員の効果を理解できます。 もちろん、1人のマネージャーがいる場合-比較するものはありませんが、2人、3人、またはそれ以上のマネージャーがいる場合-これらの係数により、どちらのマネージャーがより効果的かを理解し、これらのデータに基づいて経営判断を下すことができます。



CRMシステムで提供される電話の種類。



使用しているCRMに関係なく、SAASシステムまたはスタンドアロン（クラウドまたはボックスソリューション）にかかわらず、CRMシステムへの2種類のテレフォニー接続があります。



  • CRMに統合されたテレフォニー
  • SIPテレフォニー。




最初の種類の接続はもちろんSIPテレフォニーでもありますが、SIPクライアントはWebブラウザに統合され、会話はコンピューターに接続されたヘッドセットを介して行われます。



2番目のケースでは、ブラウザから呼び出しを行わず、ブラウザから呼び出しを開始します。呼び出しと会話に対する応答は、専用の電話（いわゆるVoIP電話）またはSIPクライアント（Zoiperなど）で行われます。 MicroSIP（多数ありますが、すべてはリストしません）。



CRMに統合されたテレフォニー。



ブラウザを使用して、最初のタイプの接続をより詳細に分析します。 このタイプの接続により、テレフォニーサーバーはCRMシステム自体に組み込まれます。



このタイプの接続を使用して呼び出しを行う方法を検討してください。




  1. ユーザーはシステムに接続し、顧客カードを選択します。
  2. ユーザーがクライアントのカードにある通話アイコンをクリックします。
  3. 通話と、ヘッドフォンとマイクを使用したクライアントとの会話があります。
  4. 通話を終了します。 コールカードはその期間を記録し、さらなるアクション（コールの説明、取引の作成など）を提案します。


コールを受信する手順は同様に実行されます。




  1. 着信コールが到着すると、ブラウザにコールウィンドウが表示されます。
  2. ユーザーは回答アイコンをクリックします。
  3. ユーザーはマイクとヘッドフォンを使用して発信者と話します。
  4. 通話を終了します。 コールカードはその期間を記録し、さらなるアクション（コールの説明、取引の作成など）を提案します。


つまり、クライアントに連絡するには、ブラウザに接続し、ブラウザからデバイス（マイクとヘッドフォン）にアクセスする必要があります。



また、このオプションでは、コールの料金がCRMシステムまたはCRM内で接続されているシステムの料金で行われることを理解する必要があります。 つまり、SIPテレフォニーを選択することはできません。SIPテレフォニーは、自分にとって都合のよいもの、または好みのものです。 システム内でのみ番号を選択すると、設定はシステム内でも完全に行われます。 着信転送や通話を録音する機能などの重要な機能も、CRMシステム自体に依存します。 プラグインなどを接続することはできません。



SIPテレフォニー。



次に、VoIPテレフォニーを使用した2番目のタイプの接続について詳しく説明します。
この場合の呼び出しはどうですか？




  1. CRMユーザーが連絡先の呼び出しアイコンをクリックします。 CRMに呼び出しアイコンが表示されます。
  2. CRMは仮想ATCを呼び出します。
  3. 仮想ATCは、ユーザーの内線番号を呼び出します（プログラムまたはハードウェア電話にすることができます）。
  4. ユーザーは「電話を拾います」。
  5. ATSは内線番号から連絡先を呼び出します。
  6. 連絡先が電話を取り、接続が進行中です。 CRMでは、通話時間は通話ウィンドウで始まります。
  7. 通話は終了します。 CRMでは、コールカードがコールの時間を記録します。 原則として、コールの終了後、コールカードウィンドウでタスクを作成するか、このコールを説明するか、別のコールをスケジュールすることをお勧めします。


この接続オプションを使用した着信コールのスキームは次のとおりです。






  1. クライアントが市番号を呼び出してから内線番号をダイヤルするか、秘書がクライアントを内線番号に転送します。
  2. コールウィンドウがCRMシステムに表示され、CRMシステムの設定方法に応じて、誰がコールしているか（連絡先がデータベースにいる場合）およびその他の情報に関する情報が表示されます。 ユーザーが（プログラムまたは電話で）「電話を拾います」。 CRMでは、コールカウントが開始されます。
  3. クライアントとの会話があります。
  4. 会話を終了します。 CRMでは、コールカードがコールの時間を記録します。 システムに既に呼び出された連絡先がある場合、呼び出しの理由、その説明、およびその他の情報が記録されます。 連絡先がシステムによって定義されていない場合、連絡先を作成し、必要に応じて取引を作成することを提案します。必要に応じて、連絡先はすべてシステム設定に依存します。


この場合、接続はCRMシステム自体に依存しません。 ユーザーはCRMシステム（エンジニアや会計士など）の存在を疑うことさえありません。



長所と短所。



1つまたは別のテレフォニーオプションを選択する場合、通常、クライアントに各接続の長所と短所を引数として与えます。 この記事でそれらを紹介します。



CRM統合テレフォニーの「短所」：


  • ブラウザとデバイスへの依存。 ブラウザの設定またはヘッドセットに何かが起こった場合、電話をかけたり受けたりすることはできません。
  • CRMシステムの機能が非常に限られているため、同じアスタリスクと比較することはできません。 ここで、通話を内部加入者だけでなく他の人にも転送したい場合、通話録音、通話転送、特に複雑な通話転送などの機能は不可能です。
  • 関税のCRMテレフォニーへの依存。 他の何かを選択することはできません。CRMシステムが提供するものを使用します。
  • 既存の電話番号を接続することはできません。 たとえば、すでにSIP番号を持っている場合、接続することはできません。 CRM自体が提供する新しい番号を接続する必要があります。
  • このタイプのテレフォニーを使用すると、職場、ヘッドセットに縛られ、コンピューターとCRMへのアクセスが必要になります。


CRM統合テレフォニーの「長所」：


  • 接続は直接です。つまり、接続はすぐに行われ、より馴染みのある呼び出しパターンです。
  • クイック接続。 この場合、仮想PBXを構成する必要はありません。 番号と内線番号を登録するだけで、安全に発着信できます。




これらのオプションは、小規模企業に最適です。



ここで、CRMシステムと統合されていないテレフォニーの長所と短所に注目します。



SIPテレフォニーの「短所」：


  • チューニングの必要性、チューニングの相対的な複雑さ。 仮想テレフォニーサーバーの構成、各ハンドセットまたはデバイスの構成、パスワードの設定などが必要です。
  • 比較的高いコスト。 この接続オプションは、IPテレフォニーデバイスまたはそのためのプログラムを購入する必要があるため、最初の接続オプションよりも高価です。 多くの場合、サーバーを上げて料金を支払う必要があります。 ただし、取得コストに加えて所有コストがあり、料金が選択される可能性があるため、統合されていない電話の場合は少ないことを理解する必要があります。
  • インフラストラクチャのメンテナンス。 この項目には大きな費用がかからないことをすぐに言わなければなりません。 同じアスタリスク-最小限のサーバーと基本的にすべてのために、それは動作し、多くのメンテナンスを必要としません。


原則として、このバージョンのテレフォニーには短所はありません。



SIPテレフォニーの「長所」：


  • テレフォニーの幅広い選択（あらゆる味と色の機器が市場に出回っています）
  • ブラウザおよびCRMシステムからの独立。 ここでのCRMシステムは、IPテレフォニーからの情報の別の消費者として機能しますが、IPテレフォニーの呼び出し元ではありません。 CRMシステムからの独立、私は最も重要なプラスを検討します。
  • プロバイダーを選択する機能。 あなたが仕事をしたいプロバイダを選択することができ、あなたに合った関税を選択することができます。 このタイプのテレフォニーの一部のユーザーは、より安価なプロバイダーに応じて、異なるプロバイダーから異なる番号に電話をかけます。
  • 広範なテレフォニー設定。 着信の転送、特定の条件下での通話、録音、ボイスメール、さまざまなコーデックを設定できます。


SIPテレフォニーなどのこのタイプのテレフォニーは、大規模な営業部門を持つ企業に必要です。



これら2つのシステムのもう1つの非常に大きな違いについて説明します。



CRMに統合されたテレフォニーでは、CRMに登録されたユーザーのみが電話をかけることができます。 たとえば、サービス部門とアカウンティングが機能する内線番号が1つあり、このテレフォニーを使用して通話を完全に受信して発信する場合、サービス部門の従業員を含むすべてのテレフォニーユーザーをCRMシステムに登録する必要があります。会計士。 他に方法はありません。 一部の拡張機能を仮想番号に保持することはできません。また、CRMシステムに組み込まれたテレフォニーを使用する他の拡張機能を保持することはできません。



そのため、CRMシステムに登録されていないが電話を使用する従業員がいる場合、別の電話を使用するか、このユーザーをCRMシステムにドラッグする必要があります。これはあまり良くありません。



2番目のケースでは、SIPテレフォニーを使用して、内線番号の一部を使用してCRMで作業し、一部の番号を非CRM従業員、会計士、倉庫従業員などに残すことができます。



このアプローチまたはそのアプローチを実装するときに、どんな落とし穴があなたを待っています。



CRM統合テレフォニーの場合、特定の落とし穴はありません。 注意を払う価値がある唯一のことは、ブラウザとサウンドを送受信するデバイスとの互換性です。



SIPテレフォニーの場合、さまざまな状況が待っています。



IPテレフォニーとCRMシステムの間に、高いpingがないことが必要です。 理由を説明します。 あるプロジェクトで、私はそのようなひっかかりがありました。 CRMシステムは海外で使用され、アジアのサーバーのどこかに配置され、テレフォニーサーバーは北アイルランドに配置されました。 したがって、pingは非常に大きく、クライアントが呼び出したときに、コールカードがシステムにすぐに表示されませんでした。信号がSIPサーバーからCRMシステムサーバーに長時間送信されました。 カードをすぐに表示するには、PINGをできるだけ小さくする必要があります。



一部の企業では、セキュリティ上の理由からVPN接続を使用しています。 つまり、VPN経由でオンラインになり、VPNから呼び出します。 このような場合、テレフォニーサーバーもVPN内にあることに留意する必要があります。



テレフォニーを導入するコスト。



テレフォニーを選択するときに予想されるコストを検討してください。



CRM統合テレフォニーの最初のオプションですべてが明確な場合：ブラウザーで料金を確認し、電話番号を接続します-2番目のタイプの接続を選択するとき、すぐにそれを理解する必要があります：

  • トラフィックが通過し、仮想PBXがインストールされるVDSサーバーをレンタルする必要があります。
  • 電話番号を仮想PBXサーバーに接続するには、プロバイダーと電話番号を選択する必要があります。
  • サーバー自体のセットアップにかかるコストが想定されます-内線番号の設定、自動転送、VoIPテレフォニーで動作する電話自体の設定が必要です。
  • テレフォニーの維持にコストがかかる場合があります-これはそれほど大きな費用ではありません。一度修正すれば、リソースを使用するだけで済みます。


その他の事項。



従業員が携帯電話で電話に出たらどうなりますか？


私はよくこの質問をされます。 たとえば、従業員が旅行中に携帯電話から電話をかける必要がある場合はどうでしょうか？ 携帯電話番号は固定されていないため、現時点ではまだ不可能であると言わなければなりません。ただし、ロシアのモバイルオペレーターの1人が企業番号をSIP番号として記録することができます（携帯電話を呼び出しますが、SIPプログラムカードが開きます）。

ただし、このためには、情報がモバイルハンドセットからではなく、アクセスが与えられていない仮想電話交換機から送信される必要があります。



会社の電話を使用し、CRMと統合するための既成のソリューションがない場合はどうすればよいですか？


関係ありません テレフォニーシステムをCRMと統合できるかどうかをISPに問い合わせてください。 たとえば、会社用に特別なソフトウェアアダプターを作成し、プロバイダーの電話と顧客のCRMを接続するプロジェクトがありました。 ビッグスリーのプロバイダーであり、統合に関して特別な問題はありませんでした。



CRMシステムに存在する電話の種類、特定の電話の導入に伴うコストを選択する際に注意すべき点を明確に説明していただければ幸いです。
