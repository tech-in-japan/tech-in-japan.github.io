Inventarium.mobi-ユーザーを怒らせないでください

今日、私は製品inventarium.mobiを立ち上げました。その目的は、モバイルアプリケーション開発者が常にユーザーの問題を認識できるようにし、後者はユーザーの問題を聞いて満足することです。 要するに、それはGetsatisfaction ＆ UserVoiceですが、モバイルアプリとプライベートコミュニティ向けです。

そして、裸のブートストラップだけで製品を作成した方法を、たくさんではなく、皆さんと共有したいと思います。 また、そのようなシステムのトピックについても議論したいと思います。最も重要なことは、あなたの中で私の最初の、最も困難で、最も不機嫌な、しかし最も愛されている顧客を見つけることです！ 無料でSMSなし。



背景



製品に直接移行する前に、その背後にあるものについて少しお話ししたいと思います。

私の名前はRoman Nikitin（ facebook 、 twitter ）で、私はあなたのほとんどと同じように、オタク、技術者、プログラマー（主にC＃、asp.net）であり、優れたSFのファンです。

数年前、私は「スタートアップ」と呼ばれるものに遭遇し、プログラマーからすぐにテクニカルディレクターに昇格しました。チームリーダー、PM、スクラムマスターオールラウンドの機能を持ち、これから生じるすべての問題と結果があります。

その後、ePythiaプロジェクト（これはタスクの場所を考慮するコンテキストスケジューラです）に取り組み、少なくとも7つの地獄の輪を通りました。 しかし、今日はそれについてではありません。



このプロジェクトの作業では、 Lean Startup and Customer Developmentの手法を使用しようとしました。これにより、メイン製品はモバイルアプリケーションであるため、ユーザー、そして最も重要な「モバイル」との良好なフィードバックチャネルが必要であると考えるようになりました。

そして最終的に、この種の製品の要件を作成しました。



  1. モバイルアプリケーションのサポート-将来、異なるプラットフォームで
  2. 作業は常に外部リソースに切り替える必要なしにウィジェット内で行われ、アプリケーションコンテキストに簡単に戻ることができます（ユーザーの半分は戻らないという意見があります）
  3. 二重認証がないため、シングルサインオンでもあります（ユーザーの半分は、ユーザー名とパスワードを入力する要件を見て、すぐに何かを書くという考えを捨てましたが、すべてを聞きたかったと言います）
  4. 最低価格、理想的には初回は無料です（30日間の試用版ではなく、それ以上）




これらの要件すべてに100％満足できるシステムを見つけることは、不可能な作業であることが判明しました。 UserVoice ＆ Getsatisfactionが最も近かったが、マーケティング担当者とオフラインブランド（Tideなど）に焦点を合わせ、価格が59ドルから99ドルであり、無料版に必要な機能がなかったことに驚いた。



その結果、アプリケーションからメールにメッセージを送信することを平手打ちし、より良い時間まで質問を延期しました。 しかし、この考えはすでに私の頭に浮かびました。なぜこのようなものがまだないのか、一見するとシステムはそれほど複雑に見えないので、開発が難しいのでしょうか。



2012年2月に少し時間があり、別のHarvestが登場しました（西のStartupWeekendのような週末のスタートアップイベント）。その後、Mobile HackDayをフォローする予定でした（ああ、キャンセルされて申し訳ありません）。 そして、そのアイデアにはまだ悩まされていました。

数週間、私は開発者のすべての友人/幹部/およびモバイルアプリケーションに何らかの形で接続しているすべての人に積極的に尋ねましたが、この製品は興味深いですか？

批判、助言、関心のために参加してくれたすべての人に感謝します。 それは私が行動する必要があるという私の心を強めました。



そして、特定の条件で実験することにしました。

  • 一週間で完全に機能するプロトタイプ
  • ブートストラップ、つまり 投資の放棄と自己啓発
  • 最小限のチーム、またはむしろ可能な限り単独ですべてを行う


プログラミングから6か月離れた1日18時間のハードプログラミングは素晴らしいものです。

私はHarvestに時間を割いて 、短い提案をしましたが... ...開発者がHarvestにほとんど行かないことは知っていましたが、製品開発からかなり遠く離れた人々がいるとは思っていませんでした。 Copiny / UserVoice / Getsatisfactionに精通しているのは、聴衆のうちわずか3人だけでした。 しかし、おそらく、眠れない夜の後、私のピッチは技術的すぎて複雑でした:)

その後、残念ながらMobile HackDayがキャンセルされたため、対象ユーザーで製品をテストすることができませんでした。

その後、プロジェクトの約80％が完了しました。 残りの20％の改良には、今日までずっと時間がかかりました。 いつものように、この段階で最も難しいことは、すべてを書き直し、リファクタリングし、新しくリリースされたテクノロジーから何かを使用したいという欲求を抑えることです。 困難を抱えて、私はこれをすべて将来のタスクマネージャーに入力するように強制します:)



技術的な詳細



現代の技術は本当に奇跡を起こすことを可能にします。

私は長年にわたって.NetでWeb向けに開発してきたので、あらゆる種類のクールなRubyおよびPythonを試すのではなく、すでに慣れ親しんでいるものを使用することにしました。

それとは別に、最小限の労力でモバイルプラットフォームのリーチを最大化する方法についての質問が急増しました。 最も論理的な答えは、html5 +ネイティブラッパーでした。



そのため、プロジェクトは次のように分類できます。



  1. バックエンドのウェブサイト
  2. Webアプリケーション-コミュニティモデレーターおよび開発者向けのパネル
  3. 電話に埋め込むためのhtml5モバイルWebアプリケーション
  4. iOS、Android、Windows Phoneのシェル


テクノロジー別に分類すると、次のようになります。



  1. 3つのWebサイトはすべてasp.net MVC 4に基づいています
  2. モバイルサイトはjQuery Mobile + Cordova上に構築されています （元のPhoneGapは「戻る」ボタンを傍受するためにのみ使用されます）
  3. メインページとしてのオーチャードCMS
  4. もちろん、Android SDK（2.3）、Windows Phone SDK、iOS SDK


一般に、完全になじみのない技術のうち、モバイルSDKのみがありましたが、最小限のコードが計画されていました。



これらのうち、iOS SDKは現在開発中です。

そして、プロトタイピングの1週間で、Windows Phone用のほぼすべてのWebとSDKを、言語のように、自分に最も近いプラットフォームにすることができました。



また、私はオープンパスに進むことを決めました。すべてのSDKはソースコードの形式で提供され、githubに投稿されます。http：//github.com/inventarium-開発者の信頼性を高めることを願っています。



費用



プロジェクトの開始にあまりお金がなく、投資について考えたくありませんでしたが、プロジェクトをしなければならなかったので、無料でそれを行う方法を見つけなければなりませんでした。



一般的な考えに反して、「無料」は必ずしもphp、ruby、java、pythonなどのオープンソーステクノロジーではありません。 Microsoft開発者の陣営では、方法を見つけることもできます。 だから：



  • ツール= 0こする。

    Microsoft Bizsparkプログラムに登録し、すべてのソフトウェアとツールを完全に無料で入手します（製品のエクスプレスバージョンを使用することもできます）
  • ホスティング= 0こする。

    繰り返しになりますが、MSDNサブスクリプションまたはBizsparkは究極の割引を提供します- 詳細はこちら 。
  • デザイン〜1000r 、パパジョーンズのピザ2個とワイン1本の価格。

    この段階でのサイト設計はそもそも標準ではなく、標準的なテーマとミニマリズムの原則が最大限に使用されていました（新しいbasecamp.comサイトを見ると、私は負けていなかったことがわかります:)）。 アプリケーションの設計のために、お馴染みのデザイナーを訪ねて、ピザ2枚とワイン1本を購入しました。2時間一緒に、おおよそのレイアウトをスケッチしました。
  • ドメイン= 200摩擦。

    GoDaddyは常にあらゆる種類のプロモーションと販売を行っているため、ドメインは本当に安価です。
  • SSL証明書= 0こすります。

    startssl.comの記事のヒントのために、 syncerのユーザーをビールに招待したいと思います




合計で1200ルーブルの費用と多くの時間が費やされます。 悪くない！



製品



記事の冒頭で、このようなシステムに必要な主な基準を概説したことを思い出させてください。

それらに基づいて、Webウィジェット、コミュニティの公開ページ（Tide、Sorry）などを拒否しました。 そのうちのいくつかは遅かれ早かれ現れるかもしれません、そしてそれはあなた、私の将来の最愛の顧客に依存します。

また、競合他社の価格設定ポリシーを確認した後、この側面も簡素化することにしました。 複数レベルの関税を導入したくはありませんし、試用版アカウントも導入したくありません。 このシステムは、ユーザー数が10,000を超えるまで、必要に応じて完全に機能するように使用できます。その後、月額49ドルを支払う必要があります。

つまり 純水フリーミウム 。



私はクライアントでは主に若くて急成長しているスタートアップを見たいと思ったので、これらの数字とそのようなポリシーはまさに私たちが必要とするものです。 これにより、不足しているお金を使わずに、すぐに適切なツールを使用できます。

一般的に、金持ちになったら後で支払います:)

すべては教科書のようです-私はまず自分のために完璧な製品を作りました、そして今、私は他の人のためにそれを修正する準備ができています。

視覚的なデモンストレーションのために、豪華なインタラクティブデモページを作成しました： app.inventarium.mobi/demo



チーム



最初から、私はこの製品の作り方、使用方法、だれがほぼ使用するかを完全に理解していました。 したがって、この段階でさまざまなマーケティング担当者、マーケティング担当者、ラケット担当者、その他の広報担当者が脱落しました。デザイナーと共同設立者は信頼できません。本当に役に立つのは開発者だけですが、私の友人は全員忙しかったです。

アクセラレーター、インキュベーター、またはスタートアップコンテストに参加するためのタスクがある場合は、チームが必要です。 まだらにされていることが望ましい。 孤独は実際にはかかりません。

その他の場合-必要に応じて確認する必要があります。

その助けは本当に私にとって非常に貴重です-これはニコライ・ミハイロフスキーです。彼はこのプロジェクトで私のコンサルタント/アドバイザーを務めています（西側の場合はアドバイザーまたはメンター）。

まあ、私の将来の妻にも大きな感謝を申し上げたいと思います。この妻はいつも私に耳を傾けただけでなく、可能な限り助け、私のロシア語のテキストを編集し、英語への翻訳を助け、おいしい夕食を作り、一般的にすべてに耐えました:)



それから何？



そして、製品を開発する必要があります。 もちろん、私の頭の中には、近い将来の大まかな計画があります。

  • jQuery Mobileの拒否の可能性（まだ遅い）
  • テーマと視覚的なカスタマイズのサポート

  • 奪うことができます


これと何をどのような順序で実装するかは、私のクライアントのみに依存します。 それとも、私はまだあなたのニーズを完全に理解しておらず、何か他のものが必要ですか？

時間はわかります。
