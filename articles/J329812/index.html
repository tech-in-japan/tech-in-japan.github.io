      個々のシステムの興味深い機能：Rブローカー



      品質評価

      Rブローカーでは、「品質評価」と呼ばれる機能が気に入りました。 広告キャンペーンのこの評価を注文できます。 システムは、各フレーズの評価とカザフスタン共和国全体の一般評価を含むExcelファイルを発行します。 その結果、広告キャンペーンのデメリットを確認できます。 たとえば、見出し、広告テキスト、またはクイックリンクには、キーフレーズからの単語がほとんどなく、リンクは無関係であり、除外キーワードはありません。ランディングページの読み込みに時間がかかり、ランディングページの直帰率が高くなります。 フレーズごとに評価が与えられます（「悪い」から「優秀」まで）。 システムは、同じサイトからの関連するランディングページの独自のバージョンも提供します。 キャンペーンには、2つの評価が与えられます：個々のフレーズの重要性を考慮に入れ、考慮しない。 重要性の考慮とは、次のことを意味します。キャンペーンのクリック数と費用に占めるフレーズの割合が大きいほど、キャンペーン全体の評価に影響します。 これは正しいです。費用の大部分が2〜3個の低品質のフレーズに当てはまる場合、誰もクリックしない数百個の高品質の広告が状況を救います。









      Rブローカーの品質評価の欠点は、レポートのファイルに説明がほとんどないことです。これは、各列を意味します。 しかし、「よくある質問」のセクションで、より詳細な説明を見つけることができました。 ただし、レポートファイル内のすべての情報を取得したいので、自分で検索するのではありません。

      CPCまたはCPO / CPAによるR-maxの計算

      Rブローカーのもう1つの興味深い機能は、「CPCによるR-maxの計算」です。 多くの場合、この問題を解決する必要があります。 クリック単価（CPC）が所定の値に近いことが必要です。 クリック単価を正確に達成するために設定する入札単価 高い入札単価を設定すると、クリックの費用が高すぎます。 低い入札単価を設定すると、安いクリックが購入されますが、クリック数が少なすぎるため、トラフィックプランは実装されません。 Directは最適なレートに関する質問には答えませんが、「CPCによるR-maxの計算」はこの質問に答えます。 希望のCPCを提供する入札単価を選択できます。 システムは、この価格でクリックが何回受信されるかの予測も提供します。 クリック数が少なすぎる場合は、クリック単価を調整して、新しい入札単価を取得できます。









      同様に、希望の注文値（CPO）または選択した目標を達成するためのコスト（CPA）を提供する入札単価（R-max）を選択できます。 Rブローカーは、このレートで受け取る注文（または達成された目標）の数を予測で示します。

      CPCまたはCPA / CPOによる自動制御

      Rブローカーの推奨に従ってキャンペーンに入札することができます。 しかし、Directの実際の状況は日々変化しています。 したがって、Rブローカーの予測を常に調整してから、レートを変更する必要があります。 この単調な作業に時間を無駄にしないために、キャンペーンを自動実行することができます。 次に、RブローカーはYandex.Directのレートを調整して、割り当てられたCPCまたはCPA / CPO値を確認します。

      プロジェクト

      Rブローカーでキャンペーンを操作するための便利なツールはプロジェクトです。 複数のキャンペーンを1つのプロジェクトに結合し、これらのキャンペーンをすべて1つのキャンペーンに結合したかのように管理できます。 たとえば、セグメントのセットを作成し、プロジェクトの入札単価を設定できます。 一度に何百ものキャンペーンを管理する必要がある場合、これは非常に便利です。 同じルーチン操作を何百回も繰り返す必要はありません。

      個々の戦術

      Rブローカーでは、「特別配置の3番目の位置のみ」という形式の標準戦術を使用できます。 しかし、多くのコンテキスト広告の専門家にとって、これは十分ではありません。 より洗練された戦術が必要です。 たとえば、特別なプレースメントへのエントリの控除価格が特定のしきい値を超えた場合、ダイナミクスの1位に入るためにレートを設定する必要があります。 特別なプレースメントへの入場料が指定されたしきい値よりも低い場合、別の料金を選択する必要があります。 このような戦術は、Rブローカーで利用可能な個々の戦術を作成する機能を使用して作成できます。 戦術を作成するには、便利な「計算機」があります。 AND演算子とOR演算子、および計算機で利用可能なさまざまなレートを使用して、かなり複雑な条件を作成できます。 さまざまなポジションの入札と借方価格に加えて、戦術を作成するときに「現在の入札」、「過去の入札」、「現在の借方価格」、「過去の借方価格」などの要素を使用できます。









      セグメント

      1つのキャンペーンには、さまざまなインジケータを持つ多くのフレーズがあります。 順序値が低く、コンバージョン数が多い効果的なフレーズがあります。 頻繁にクリックされたが、何も注文しなかった明らかに効果のないフレーズがあります。 注文をもたらさなかった「疑わしい」フレーズがありますが、それらに関する統計はほとんど蓄積されていないため、それらに何を期待するかはわかりません。 効果的なものもあれば、そうでないものもあります。 明らかに、これらのフレーズのすべてに同じレートを割り当てるべきではありません。 したがって、異なるカテゴリのフレーズには異なる戦術を割り当てる必要があります。 これらのカテゴリの構成は毎日変化します。統計が蓄積されると、フレーズが1つのカテゴリから別のカテゴリに移動します。 ただし、Rブローカーでは、フレーズを手動で分類する必要はありません。 これを行うには、機能的な「セグメント」があります。 ユーザーは複数のセグメントを作成し、各セグメントの条件を調整できます（たとえば、過去90日間に1フレーズにつき3トランザクション以上、注文値が1000ルーブル未満）。各セグメントに対して、入札または戦術を設定できます。 各統計の更新後、Rブローカーはセグメントの状態を確認し、必要に応じてフレーズを別のセグメントに転送します。
