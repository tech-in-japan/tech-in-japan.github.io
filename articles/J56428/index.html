Turboは現在Operaメインコードにあります

本日、Operaの開発者はOpera 10の新しいアセンブリの提供を発表しました。このバージョンの主な革新：インストールファイルとプロファイルファイルの新しい場所、新しいスペルチェッカー、クラッシュログを生成するシステムも、Operaのメインコードに含まれるOpera Turboに移植されました。



まだ知らない人のために、Opera TurboはOpera 10の以前の実験アセンブリに登場した機能であり、その本質はページと画像を事前に圧縮することでページの読み込みを高速化することです。 このため、Opera Miniの操作に使用されるものと同じサーバーが使用されます。



このアセンブリの新機能は何ですか？

  • 新しい実験的なOpera Turboモード：自動。 このモードでは、Operaは接続の品質を監視し、接続が遅いと判断するとTurboがオンになります。 Operaが接続が十分速いと判断した場合、ターボモードはオフになります。 もちろん、このモードを使用するかどうかを選択できます。
  • 画像の読み込みに関するエラーを修正
  • Gmailのバグを修正
  • RSSにサブスクライブするためのチャネルの表示を修正
  • Linuxバージョンの最後の実験的ビルドで表示されたキリル文字の入力に関するバグを修正しました
  • 生産性の向上
  • 新しいスペルチェックエンジン。 現在、 hunspellがエンジンとして使用されています。これは、よりスマートで、さまざまな辞書との互換性が高いようです。
  • 現在、LinuxバージョンのOperaは、クラッシュの場合にクラッシュログを生成します。 開発者は、そのようなログを送信するよう求められます。 これにより、多くのエラーがすでに修正されていますが、Linuxバージョンでは安定性の向上が期待できます。
  • 統一するために、構成ファイルの名前といくつかのパスが変更されました。 現在、使用されているシステムに関係なく、すべての構成ファイルの名前は同じです。
新しい変更により、次のエラーが発生する場合があります。
  • すべてのサイトアイコンのキャッシュからの消失
  • クリーニング短縮ダイヤル
  • 広告ブロッカールールの喪失
  • スペルチェックが機能しない場合があります。 辞書ファイルを新しい場所に移動する必要があります。 Windowsの場合：C：\ Program Files \ Opera 10 \ locale \ ru \、Linuxの場合：/ usr / share / opera / locale / ru /（ruは辞書言語の名前です）。
また、mail.yandex.ruなどの多くのYandexサービスが、ターボモードのこのビルドでは機能しないこともわかりました。



注意！ 必ずOpera構成フォルダーをバックアップしてください
