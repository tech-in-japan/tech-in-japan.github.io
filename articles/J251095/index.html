Goの低速接続のシミュレーターを作成します

この記事では、Goで非常に複雑なことを行うのがどれほど簡単で、インターフェースがどれほど強力かを示したいと思います。 遅い接続をシミュレートすることです-しかし、iptablesのルールの形式で一般的なソリューションとは異なり、コード側でこれを実装します。これにより、たとえばテストで簡単に使用できます。



ここでは複雑なことは何もありませんが、わかりやすくするために、アスキーアニメーションを（アスキーネマサービスを使用して）記録しましたが、 参考になることを願っています。







インターフェース



インターフェイスは、Go型システムの特別な型であり、オブジェクトの動作を記述することができます。 メソッド（動作）が定義されている静的型は、これらのメソッドを記述するインターフェースを暗黙的に実装します。 最も有名な例は、標準のio.Readerライブラリのインターフェイスです。 

// Reader is the interface that wraps the basic Read method. // ... type Reader interface { Read(p []byte) (n int, err error) }


Read（[] byte）（int、error）メソッドを定義する構造は、io.Readerとして使用できます。



インターフェイスが他のライブラリで使用されている場合、最初はあまり価値がなく強力ではないように見える単純なアイデアは、まったく異なる外観になります。 これを実証するには、標準ライブラリとio.Readerが理想的な候補です。



コンソール出力



それでは、最も単純なReaderのアプリケーションから始めましょう-行をstdoutに出力します。 もちろん、このタスクではfmtパッケージの関数を使用する方が適切ですが、Readerの動作をデモンストレーションしたいと思います。 したがって、strings.Reader型（io.Readerを実装する）の変数を作成し、io.Copy（）関数（io.Readerでも機能する）を使用して、これをos.Stdoutにコピーします（これが実装します）。 io.Writer）。 

 package main import ( "io" "os" "strings" ) func main() { r := strings.NewReader("Not very long line...") io.Copy(os.Stdout, r) }




次に、コンポジションを使用して、独自のタイプのSlowReaderを作成します。これは、100ミリ秒の遅延で元のReaderから1文字を読み取るため、1秒あたり10バイトの速度を提供します。 

 // SlowReader reads 10 chars per second type SlowReader struct { r io.Reader } func (sr SlowReader) Read(p []byte) (int, error) { time.Sleep(100 * time.Millisecond) return sr.r.Read(p[:1]) }


p [：1]とは何ですか。説明する必要はありません。元のスライスの最初の1文字で構成される新しいスライスです。



あとは、strings.Readerを元のio.Readerとして使用し、遅いSlowReaderをio.Copy（）に渡すだけです。 シンプルさとクールさを同時にご覧ください。

（ascii-castは新しいウィンドウで開きます。Habrのjsスクリプトを埋め込むことはできません）





この単純なSlowReaderは、画面への出力だけでなく使用できるのではないかと疑い始めるはずです。 遅延などのパラメーターを追加することもできます。 さらに良いのは、SlowReaderを別のパッケージに入れて、以降の例で使いやすくすることです。 コードを少しとかしました。

 package slow import ( "io" "time" ) type SlowReader struct { delay time.Duration r io.Reader } func (sr SlowReader) Read(p []byte) (int, error) { time.Sleep(sr.delay) return sr.r.Read(p[:1]) } func NewReader(r io.Reader, bps int) io.Reader { delay := time.Second / time.Duration(bps) return SlowReader{ r: r, delay: delay, } }

ファイルから読み取る



そして今、ほとんど手間をかけずに、SlowReaderでファイルからの読み取りが遅いことを確認してください。 * os.File型の変数を受け取りました。これは、開いているファイルのハンドルを格納しますが、同時にio.Readerインターフェイスを実装しているため、strings.Readerを使用して以前と同じようにファイルを操作できます。

JSONをデコード



しかし、ファイルからの読み取りは簡単すぎます。 もう少し興味深い例を見てみましょう。標準ライブラリのJSONデコーダーです。 encoding / jsonパッケージは便宜上json.Unmarshal（）関数を提供しますが、json.Decoderを使用してio.Readerを操作することもできます-json形式のストリームデータをデシリアライズできます。



単純なjsonエンコードされた文字列を取得し、SlowReaderの助けを借りて「ゆっくり読み」ます。json.Decoderは、すべてのバイトが到着した後にのみ完成したオブジェクトを返します。 これを明確にするために、画面に読み取られた各文字の出力を関数slow.SlowReader.Read（）に追加します。 

 package main import ( "encoding/json" "fmt" "strings" "test/habr/slow" ) func main() { sr := strings.NewReader(`{"value": "some text", "id": 42}`) // encoded json r := slow.NewReader(sr, 5) dec := json.NewDecoder(r) type Sample struct { Value string `json:"value"` ID int64 `json:"id"` } var sample Sample err := dec.Decode(&sample) if err != nil { panic(err) } fmt.Println("Decoded JSON value:", sample) }

遅いHTTPクライアント



io.Readerが標準ライブラリのどこでも使用されていることに驚かないでください-どこからでも何でも読むことができるすべてのために。 ネットワークからの読み取りも例外ではありません-io.Readerはいくつかのレベルで使用され、このような単純な1行呼び出しhttp.Get（URL文字列）の内部に隠されています。



最初に、HTTP GET要求の標準コードを記述し、応答をコンソールに出力します。 

 package main import ( "io" "net/http" "os" ) func main() { resp, err := http.Get("http://golang.org") if err != nil { panic(err) } defer resp.Body.Close() io.Copy(os.Stdout, resp.Body) }




ネット/ http-libraryに慣れる時間がない人のために-いくつかの説明。 http.Get（）は、http.Clientタイプ用に実装されたGet（）メソッドのラッパーです-ただし、このラッパーは、DefaultClientと呼ばれる「ほとんどの場合に適した」既に初期化された変数を使用します。 実際、クライアントは、タイプTransport。のオブジェクトを使用したネットワークからの読み取りを含む、すべてのほこりっぽい作業をさらに行います。このオブジェクトは、タイプnet.Connの下位レベルのオブジェクトを使用します。 最初はわかりにくいかもしれませんが、実際には、単にライブラリのソースを読むだけで習得するのは非常に簡単です。他のほとんどの言語とは異なり、Goの標準ライブラリは、できるコードの例です。 Goを学び、そこから例を取り上げます。



先ほど、「io.Readerはいくつかのレベルで使用される」と述べましたが、これは事実です。たとえば、resp.Bodyもio.Readerですが、ブラウザの速度低下ではなく、接続速度の低下をシミュレートすることに関心があるため、次に、ネットワークから読み取るio.Readerを見つける必要があります。 そして、先を見ると、これはnet.Conn型の変数です。つまり、カスタムhttpクライアント用に再定義する必要があります。 埋め込みでこれを行うことができます： 

 type SlowConn struct { net.Conn // embedding r slow.SlowReader // in ascii-cast I use io.Reader here, but this one a bit better } // SlowConn is also io.Reader! func (sc SlowConn) Read(p []byte) (int, error) { return sc.r.Read(p) }




ここで最も難しいのは、標準ライブラリのnetおよびnet / httpパッケージをもう少し深く理解し、遅いio.Readerを使用してhttp.Clientを正しく作成することです。 しかし、複雑なことは何もありません。標準ライブラリのコードを調べると、スクリーンキャストにロジックが表示されることを望みます。



その結果、次のクライアントが取得されます（実際のコードの場合は、別の関数に入れて少しコームすることをお勧めしますが、概念実証の例では行います）。 

  client := http.Client{ Transport: &http.Transport{ Dial: func(network, address string) (net.Conn, error) { conn, err := net.Dial(network, address) if err != nil { return nil, err } return SlowConn{conn, slow.NewReader(conn, 100)}, nil }, }, }

おわりに



記事の冒頭で述べたように、インターフェイスは非常に強力なツールキットであり、プロパティと動作の型を分離するという考え方は非常に正しいものです。 しかし、実際には、インターフェースが異なるライブラリーで他の目的に実際に使用される場合、この力が発揮されます。 これにより、非常に異なる機能を組み合わせて、元の作者が疑ったこともないようなことに対して他人のコードを使用することができます。 また、標準のインターフェイスだけではありません。大規模なプロジェクトでは、インターフェイスは非常に高い柔軟性とモジュール性を提供します。
