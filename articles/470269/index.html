I feel that this time you really missed the news from our project! For seven months we have been working tirelessly so that you can download and test the next version of our operating system. Despite the announced operational schedule for generating releases every 3 months, the final preparation of release 0.4.12, which was traditionally conducted in a separate branch, this time took a little more than six months. The reason for such a long preparation process was the desire of our perfectionist and part-time release engineer, Joachim Hensee, to correct as many regressions as possible over the past few years. As a result, Joachim eliminated at least 33 regressions, which can be called an impressive result.









360 Extreme Explorer based on Chromium 69 in ReactOS 0.4.12. Screenshot sent by Andrey Shatalov



The most significant correction in version 0.4.12 was the elimination of a series of problems that led to distorted rendering of text on buttons in many different applications, such as iTunes and .NET-based programs (2.0 and 4.0). Now there really is a good deal with .NET applications!







Through the efforts of our design team, two new themes were added - Lunar in the XP style with a changed color scheme and Mizu in the style of new versions of Windows.







Support for aligning application windows with respect to the edges of the screen or expanding / minimizing when moving the window with the mouse in certain directions is activated.











A free driver has been added for the Intel e1000 network adapter, which is used in the default VirtualBox and VMware virtual network interfaces. Its development involved Victor Perevertkin and Mark Jensen.



Stanislav Motylkov added the ability to download drivers for MIDI ‚Äì instruments and manage them.



The oldest bug report (with a solid 15-year-old age), fixed in ReactOS 0.4.12, was the CORE-187 request to add support for local Dll override using files with the extension ".local". Local redefinition is necessary for the operation of many portable ‚Äì programs.



Among other things, it is worth remembering that now some problems in the implementation of network boot via the PXE protocol have been resolved. This launch method has become even more effective.







Rewritten code to protect applications from changing components that run in kernel space (ntoskrnl, win32k, drivers, etc.).



As usual, the synchronization with the Wine Staging code base version 4.0 was performed and the versions of third-party components were updated: BTRFS 1.1, UniATA 0.47, ACPICA 20190405, libpng 1.6.35, mbedTLS 2.7.10, mpg123 1.25.10, libxml2 2.9.9, libxslt 1.1.33 , libtiff 4.0.10.
