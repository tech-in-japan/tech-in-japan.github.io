Windows Server 2003からWindows Server 2012 R2への移行：Active Directory

Windows Server 2003のサポートの終了が近づいていることは秘密ではありません。 X日は2015年7月17日に予定されています。つまり、インフラストラクチャをより新しいバージョンのオペレーティングシステムに移行するための管理に費やす時間がますます少なくなっています。 Habréでは、サポートの終了について既にいくつかの発表を行っており、ジャンプスタートの教材に関するコースがMicrosoft Virtual Academyポータルで公開されています 。 ファイルサーバーの転送に関する記事の翻訳があります 。 この記事では、Active Directoryの移行について説明し、移行プロセスを支援するための段階的なアルゴリズムを提供します。





Windows Server 2003からWindows Server 2012 R2へのActive Directoryの移行は、移行プロセス中に対処する必要がある主要なタスクの1つです。

実際、Active Directoryの移植は面倒ではありません。 いくつかの手順を実行するだけで済み、これについては以下で詳しく説明します。

まず、Windows Server 2003がインストールされているドメインコントローラーで小規模な構成を実行し、既存のドメインとフォレストの機能レベルとしてWindows Server 2003が選択されていることを確認してください。

ドメインとフォレストの動作モードを変更するには、Active Directoryドメインと信頼スナップインを実行する必要があります。 ドメインの操作モードを変更するには、ドメインを右クリックし、フォレストの操作モードについては、[Active Directoryドメインと信頼関係]をクリックします。 [ドメインの機能レベルを上げる]と[フォレストの機能レベルを上げる]をそれぞれ選択します。





どちらの場合も、動作モードはWindows Server 2003に設定する必要があります。





次のステップは、Windows Server 2012 R2を実行する2番目のドメインコントローラーをネットワークに追加することです。 これを行うには、Windows Server 2012 R2を実行しているサーバーにActive Directoryドメインサービスの役割をインストールします。



インストール後、新しいドメインコントローラーを既存のドメインに追加します。 これを行うには、Enterprise Adminsグループに属し、適切な権限を持つアカウントを使用する必要があります。



このサーバーがDNSサーバーおよびグローバルカタログ（グローバルカタログ-GC）として機能するかどうかを指定する必要があります。



[追加オプション]画面で、既存のドメインコントローラーに複製するドメインコントローラーを指定します。 Windows Server 2003を実行しているドメインコントローラーを選択する必要があります。



ドメインをインストールするには、フォレスト、ドメイン、およびスキーマを準備する必要があります。 以前にこのためにadprepコマンドを実行する必要があった場合（およびドメイン構成を開始する前にこれを実行する必要がありました）、ADDS構成ウィザードがこのタスクを処理し、準備が自動的に行われます。



次に、インストールが完了するのを待って、コンピューターを再起動する必要があります。 その結果、Windows Server 2012 R2がインストールされたドメインコントローラーを受け取ります。

Active Directoryユーザーとコンピュータースナップインで、ネットワークに2つのドメインコントローラーがあることがわかります。





準備手順が完了したら、Active Directoryの移行に直接進むことができます。 Windows Server 2012 R2を実行しているドメインコントローラーで次の順序で必要なアクションを実行します。

  1. FSMO（フレキシブルシングルマスターオペレーション）ロールの移行
  2. Active Directoryドメインコントローラーの変更
  3. スキーママスターの変更
  4. グローバルカタログからWindows Server 2003ベースのドメインコントローラーを削除する


1. FSMO（フレキシブルシングルマスターオペレーション）の役割の移転



FSMOの役割を転送するには、 Active Directory ユーザー と コンピュータースナップインを開き、ドメインを右クリックして、表示されるサブメニューで[ 操作 マスター ]を選択します。



操作マスターの役割を移す必要があります。 これを行うには、再び表示されるウィンドウの各タブで[ 変更 ]ボタンをクリックし、2003サーバーから2012 R2を実行しているサーバーに役割を転送します。



転送操作を確認し、正常に完了するのを待ちます。 最後に、操作マスターの役割がWindows Server 2012 R2を実行しているサーバー上にあることを確認してください。





2. Active Directoryドメインコントローラーの変更



次に、Active Directoryドメインコントローラーの変更に進みます。 Active Directoryドメインと信頼関係コンソールを開き、フォレストを右クリックして、[ Active Directoryドメインコントローラーの変更 ]を選択します 。



新しいウィンドウで、[ このドメインコントローラー]または[AD LDSインスタンス]を選択し、Windows Server 2012 R2を実行しているサーバーを選択します。



ここでも、フォレストを右クリックし、アイテムOperations Masterを選択します。



[ 変更 ]をクリックして、ドメイン名前付け操作マスターの役割を転送します。





3.スキーママスタの変更



次に、スキーママスターの変更に進みます。 管理者権限でコマンドプロンプトを実行し、コマンドregsvr32 schmmgmt.dllを入力します



このコマンドを使用すると、動的DLLのプライマリ登録が実行されます。これは、Active Directoryスキーマスナップインに必要です。

コマンドが完了したら、コマンドラインを閉じてMMCコンソールを起動し、Active Directoryスキーマスナップ インを 追加できます（このためには、[ ファイル] > [ スナップショットの 追加 / 削除]を選択します ）。



同じMMCで、Active Directoryスキーマを右クリックし、[Active Directoryドメインコントローラーの変更]を選択します。 手順2で実行したアクションと同様に、新しいウィンドウで[ この ドメイン コントローラー] または[ AD LDS インスタンス]を選択し、Windows Server 2012 R2を実行しているサーバーを選択して[ OK ]をクリックします。 Active Directoryスキーマスナップインが接続されていないという警告が表示されます。 [ OK]をクリックして続行します。

ここでも、フォレストを右クリックして、アイテムOperations Masterを選択します。 新しいウィンドウでスキーママスターの役割を転送するには、[ 変更 ]をクリックします。

これで、MMCコンソールを閉じ、Active Directoryユーザーとコンピュータースナップインを開き、Windows Server 2012 R2を実行している新しいサーバーにデータが正常に複製されることを確認できます。 レプリケーションプロセスには時間がかかる場合があることに注意してください（すべて、レプリケートする必要があるActive Directoryオブジェクトの数に依存します）。



4. Windows Server 2003を実行しているドメインコントローラーをグローバルカタログ（グローバルカタログ）から削除する



Windows Server 2003を実行しているドメインコントローラーをグローバルカタログから削除することは残ります。 これを行うには、Active Directoryサイトとサービスを開き、[サイト]フォルダー、[Default-First-Site-Name]、[サーバー]の順に展開し、最後に両方のサーバーを展開します。



Windows Server 2003を実行している古いサーバーの[ NTDS 設定]を右クリックし、[ プロパティ ]を選択します 。 新しく表示されたウィンドウで、 グローバル カタログアイテムのチェックを外し、[ OK ]をクリックします。





Active Directoryユーザーとコンピューターでは、Windows Server 2003のドメインコントローラーはグローバルカタログではなくなりました。



FSMOの役割がWindows Server 2012 R2で実行されていることを確認するために残っています。 これを行うには、管理者権限で開いたコマンドラインで、 netdom query fsmoコマンドを実行します



これで、Active Directoryの移行が完了しました。 Windows Server 2003を実行しているコンピューターで、ドメインコントローラーからコンピューターを降格するために、 dcpromoを実行します（ところで、Windows Server 2012 R2にはdcpromoはありません ）。 その後、Active Directoryユーザーとコンピューターコンソールを見ると、Windows Server 2012 R2を実行しているドメインコントローラーが1つしか残っていないことがわかります。
