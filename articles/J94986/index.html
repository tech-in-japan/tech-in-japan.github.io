Nokia N900：ファームウェアの次の問題を克服する（PR 1.2）

Nokia N900を2台目の電話（1台目はiPhone）として長い間使用していたので、新しいファームウェアを見つけたとたんに、すぐにアップグレードすることにしました。 確かに、新しいPR 1.2ファームウェアは確かにはるかに優れています！ これは簡単な作業ではないことを知って、彼は日曜日の夕方に数時間取って、それらを完全に使いました。 しかし、まず最初に。



最初は航空でアップグレードすることにしましたが、「選択した場所にメモリが足りません」という不思議なメッセージを受け取りました。 私はすでにこれに対処する方法について書いたが、問題は今では難しい方法だけが機能するということであり、単純な方法は長い命を命じた。 彼らはそのようなcな男たちの抜け穴をカバーしました。



残念なことに、私の気質は多くの多くのチームを書くことを許していないので、私はまだ簡単なオプションを見つけることにしました。 Maemo.orgに行き、 Nokia Software Updaterの存在を思い出させました。 うん！ これが簡単な方法です！ ダウンロード、インストール、実行。 プログラムはN900を見つけ、更新プログラムを見つけ、インストールを提案し、6.3メガバイトをダウンロードした後、宣誓して停止しました。 たとえば、USBが間違っているか、インターネットが悪いか、サーバーに何らかの問題があるかもしれませんが、ダウンロードできません。もう一度お試しください。



私も20回試しました。 助けにはなりませんでした。 そして再起動し、ケーブルが引っ張られ、すべて無駄になりました。



Maemo Flasherアプレットをインストールしようとしましたが、ワイナリーがそれを使用することも、穏やかに置くことも困難です。 そして、そう、そして私が試した-それは点滅しません。 私が非難するのは私の曲がった手だけであり、経験豊富な人はほとんど問題がないことを強調します。



一方、西部のインターネットはフリースになり始め、6.3メガバイトのクラッシュを伴うバグがグローバルであることを発見しました。 そして、彼らは皆うめき、うめきますが、彼らは治すことができません。 Nokia自体のサーバーに何らかの不具合があります。



そして、私は考えました：この未完成の作物を取り、それをここで入手可能な本格的な画像に置き換えるとどうなりますか？ おそらく最も難しいのは、このスクラップがどこに隠れているかを見つけることでした。 しかし、見つかった。 住所は次のとおりです。



C：\ Documents and Settings \ All Users \ Application Data \ Nokia \ Nokia Service Layer \ A \ nsl_service_module_00001 \ vpls \ www.dsut.onl ine.nokia.com.oti.caresuite \ Products \ rx-51



そこでファイルを交換し、Nokia Software Updaterを再度起動しました。 -すべてが大丈夫だった。 特に深く掘り下げたわけではないので、時間がありませんでしたが、新しいファームウェアを使用すると、N900はタンバリンと踊らずにUSSDの要求を理解できることがわかりました。



このレシピがデバイス上の兄弟に役立つことを願っています。 ハッピーニューマエモ！ :)
