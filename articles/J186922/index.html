Windows Server 2012 R2の新機能

この投稿では、 Windows Server 2012 R2の最も興味深い新機能の概要を、もちろん現在利用可能な予備バージョンから始めたいと思います 。 各機会にいくつかの提案を行ってその意味を明確にし、実装の詳細は今後の出版物に残します。 したがって、主な目標は、新しいサーバーが何を興味深いものにするかについての一般的なアイデアを提供することであり、それから特定のタスクに最も適切なものを決定します。 一般的に、テキストはたくさんありますが、写真はまったくありません。



R2はマイナーなOSアップデートであるという事実にもかかわらず、多くの新機能があります。 何らかの方法でプレゼンテーションを合理化するために、これらの機能を3つのグループに分けました。Hyper-V、ネットワークスタック、およびストレージ管理サブシステムの変更です。 これは、Windows Server 2012のレビューコースで一度に行われた方法と類似しています。この分類は非常に条件付きですが、多くの機能が一度に複数のカテゴリに等しく帰属できるためです。 Hyper-Vから始めましょう。





Hyper-Vの新機能



仮想マシンの第2世代（世代2）



通常の仮想マシン（第1世代）に加えて、新しい（第2世代）の仮想マシンを作成できます。 多くの古いエミュレートされたデバイスがこれらのVMから削除されており、以下がサポートされています。

  • セキュアUEFIブート
  • 仮想SCSIハードディスクおよび仮想SCSI-DVDからの起動。
  • 合成ネットワークアダプターを使用したネットワークブート。


第2世代のVMのパフォーマンスは高く、特にVMにOSをロードおよびインストールする場合、違いは顕著です。 第2世代VMのゲストオペレーティングシステムは、Windows Server 2012、Windows Server 2012 R2、および64ビットバージョンのWindows 8およびWindows 8.1です。



VMBus経由のリモート接続（VMBus経由のリモートデスクトップ）



ゲストOSがWindows Server 2012 R2またはWindows 8.1であり、これらのOSでリモートデスクトップサービス（RDS）が実行されている場合、ネットワーク上だけでなくVMBusバスを介してゲストOSのRDSサービスに直接接続できます。 この接続は、Hyper-VコンソールでVMを単に開いたときに発生します。 同時に、ディスプレイ解像度の選択、オーディオサポート、クリップボードサポート、プリンターのリダイレクト、スマートカード、USBデバイスなど、RDPセッションのすべての利点が得られます。 VMがネットワークにまったく接続されていない可能性があることを強調します。



自動アクティベーション



仮想化ホストがWindows Server 2012 R2 Datacenterであり、ゲストOSがWindows Server 2012 R2のいずれかのエディションである場合、ホストOSが既にアクティブ化されていれば、そのようなゲストOSは自動的にアクティブ化されます。 さらに、ゲストOSのアクティベーションには、ネットワーク接続（インターネット、KMS、またはその他の場所への接続）は必要ありません。



動的な移行



  • まず、Windows Server 2012からWindows Server 2012 R2へのVMの移行の可能性。 何も共有しないことを含め、すべての可能なタイプのライブマイグレーションについて話し合っています。 この移行オプションを使用すると、VMの実行を停止しないため、インフラストラクチャをWindows Server 2012 R2に簡単に移行できます。 ただし、以前のバージョン、つまりWindows Server 2012 R2からWindows Server 2012への移行はサポートされていないことに注意する必要があります。
  • 第二に、Windows Server 2012 R2サーバー間の動的な移行はデフォルトで実行され、データの圧縮（無効化可能）により、ネットワークの負荷と移行時間が短縮されます。
  • 第三に、RDMAをサポートするネットワークアダプターがソースとレシーバーにインストールされている場合、これらのアダプターの機能を使用して、つまりSMBダイレクトテクノロジーを使用して動的移行を実行できます。これにより、圧縮よりも移行プロセスがさらに加速されます。


VHDXディスクをオンラインでサイズ変更する（VHDXサイズ変更）



Windows Server 2012 R2では、VMを停止せずに仮想ハードドライブのサイズを増減できます。 オンラインサイズ変更は、OSドライブを含むVHDXドライブでのみサポートされ、SCSIコントローラーにのみ接続されます。 この機能は、Hyper-VコンソールとPowerShellの両方で使用できます。



オンラインでVMをエクスポート/クローン（ライブVMエクスポート/クローン）



Windows Server 2012 R2では、VMをシャットダウンせずに完全なエクスポートを実行できます。つまり、実際にクローンを作成したり、実行中のVMの必要なスナップショットをエクスポートしたりできます。



ストレージのサービス品質（ストレージQoS）



外出先の各仮想VMハードドライブについて、1秒あたりの最大および最小I / O操作（IOPS）を設定できます。 Hyper-Vは、上位のドライブの帯域幅を制限し、ディスクアクティビティが指定された最小しきい値を下回った場合にアラートを生成します。 さらに、リソースメータリング機能が更新され、特定のVMまたはVMグループの統計が収集されます。 測定結果にIOPSインジケータが含まれるようになりました。



共有VHDXファイル（共有VHDX）



2つ以上の仮想マシンが1つの共通VHDX仮想ハードディスクを使用できます。 この機能を使用すると、ゲストクラスター、つまりノードがVMであるクラスターを構築できます。 VM内の共有VHDXは、共有SASドライブとして表示されます。 原則として、前にゲストクラスターを作成できます。 ただし、このためには、iSCSIやファイバーチャネルなどを使用して、VMをSANに明示的に接続する必要がありました。 ただし、このオプションは、理想的にはストレージシステムをVMレベルから完全に抽象化したい、つまりVMからストレージ実装機能を隠したいホスティング業者にとって完全に最適ではありません。 これで、このシナリオを簡単に実装できます。 次のゲストクラスターに個別のLUNまたはCSVボリュームを割り当てる必要はありません。 ゲストクラスターは、共有VHDXファイルに基づいて構築されます。このファイルは、物理フェールオーバークラスターのCSVボリューム上、またはスケールアウトファイルサーバーの共有フォルダー（ボール上）に配置できます。 フェールオーバークラスターまたはスケールアウトファイルサーバーのノードは、Windows Server 2012 R2を実行している必要があることに注意してください。 ゲストクラスターを構成する仮想マシンのOSは、Windows Server 2012 R2またはWindows Server 2012のいずれかです。後者の場合、VMの統合コンポーネントを更新する必要があります。



Hyper-Vレプリカ



VMレプリケーションのメカニズムにおける2つの有用な革新。

  1. 次に、VMのレプリカを複製できます。 たとえば、ビジネスに不可欠なVMを複製できる2番目のデータセンターはありません。 したがって、VMをプロバイダーのデータセンターに複製することにします。 プロバイダーには追加のサイトもあり、VMの復元力を高めるために、このVMのレプリカが2番目のサイトに複製されます。
  2. 現在、Windows Server 2012でサポートされている5分間のレプリケーション間隔に加えて、間隔を30秒または15分に設定できます。


さらに、任意の数のVMおよび異なるデータセンターでレプリケーションを構成できることを思い出してください。 Windows Azureには、VMのレプリカを集中管理（オーケストレーション）できる特別なサービスがあります。





ネットワーキングの新機能



仮想RSS（vRSS）



RSSテクノロジーは、ホストネットワークアダプターでサポートされている場合、ホストは、利用可能な物理プロセッサーのいくつかのコアで受信ネットワークトラフィックを処理できます。 ただし、VM内のトラフィックは引き続き単一の仮想プロセッサによって処理されます。 Windows Server 2012 R2では、vRSSのおかげで、ネットワークトラフィック処理をVMのさまざまな仮想プロセッサに分散させることが可能になりました。 これは、ホストがほとんどまたは1つのVMを実行するシナリオでは特に重要ですが、ネットワークフローを非常に集中的に処理します。 同様の状況が典型的です。たとえば、さまざまなゲートウェイ、Windows Serverベースの専用デバイスなどです。 vRSSを有効にするには、ホストにVMQ対応のネットワークアダプターが必要です。



NICチーミングのトラフィックバランシングの動的モード（NICチーミング動的モード）



Windows Server 2012に組み込まれたNICチーミングテクノロジーを使用すると、複数のネットワークアダプターをグループに集約して、フォールトトレランスを提供し、ネットワークトラフィックを分散できます。 バランスは2つのモードで可能です。Hyper-Vポート。ホストで実行されている仮想マシンがタイミンググループのネットワークアダプターにマップされている場合。 実際に特定のTCPまたはUDPポート（またはIPアドレス）へのトラフィックが特定のグループアダプター経由でルーティングされる場合のアドレスハッシュ。 Windows Server 2012 R2では、別のバランシングモードが登場しました-動的。 この場合、発信トラフィックはいわゆるフローレットに分割され、グループ内のすべてのアダプターに分散されます。 このモードを使用すると、既存のネットワークアダプター間でトラフィックをより均一に分散できます。 たとえば、1つのホストに4つのネットワークアダプターが1つのタイミングで結合され、3つのVMが実行されているとします。 バランシングモードのHyper-Vポートの場合、VMからの発信トラフィックは、動的モードの場合、4つすべてを使用して、3つのグループアダプターのみを使用して送信されます（各VMは1つのネットワークカードに「結び付けられます」）。



拡張アクセス制御リスト（拡張ACL）



Windows Server 2012では、Hyper-V拡張スイッチポートごとにACLを設定できます。これにより、特定のMACアドレスまたはIPアドレスへのトラフィックを、一方または両方で許可または拒否できます。 Windows Server 2012 R2では、ACL設定で、プロトコル、ポートを追加で指定し、たとえば高度なトラフィック分析などのステートフルフラグを設定することも可能になりました。



ネットワークトラフィックのリモート監視（リモートライブ監視）



WMIおよびETWの新機能により、ネットワークトラフィックをオンラインでリモートで監視したり、その後のオフライン分析のために情報を収集したりすることができます。 これを行うには、ワークステーション（次期バージョンのネットワークモニター、現在ベータ版）にMicrosoft Message Analyzerをインストールし、対象のホストまたはこのホスト上の特定のVMのトラフィックを示します。 ホストがWindows Server 2012 R2を想定していることは明らかです。前述のWMIおよびETW拡張は、このOSでのみ使用可能です。



ネットワーク仮想化の変更



ネットワーク仮想化（NV）テクノロジーの4つの最も重要な変更について説明します。

  1. NICチーミングとの統合。 現在、ネットワーク仮想化が使用されているホストでは、フェールオーバーとトラフィックバランシングの両方にNVでネットワークアダプタータイミングを使用できます。
  2. NVGREタスクオフロード。 NVGREをサポートするネットワークアダプターが市場に登場し始めており、NVトラフィックの処理に関連するタスクの一部をアダプターハードウェアレベルに転送できるようになっています。 当然、これは主にネットワーク操作のパフォーマンスを改善することを目的としています。
  3. Windows Server 2012のHyper-V拡張スイッチは、NVを使用する場合にのみCAアドレスを認識します。 対照的に、Windows Server 2012 R2では、スイッチはPAアドレスのレベルでも機能するため、NVに基づいて転送、フィルタリング、およびパケット検査のルールを構成できます。
  4. 組み込みゲートウェイ。 Windows Server 2012 R2は、NVの既製のゲートウェイとして機能し、ネットワーク仮想化を使用したVMと外部との接続を提供できるようになりました。 同時に、トラフィックルーティングオプションとサイト間トンネリングがサポートされています。


スイッチ管理



Open Management Infrastructure（OMI）をサポートするスイッチは、PowerShellで管理できます。 Windows Server 2012 R2には、スイッチに必要な設定を構成できる一連のコマンドレットが追加されています。VLAN、特定のポートなどを構成します。



IPアドレス管理（IPAM）



System Center Virtual Machine Managerと対話するためのモジュールにより、VMMとIPAM間の双方向の情報交換を構成し、IPAMの物理アドレススペースと仮想アドレススペースの両方にデータを蓄積できるようになりました。





ストレージ管理の新機能



iSCSIターゲットでのVHDXサポート



サーバーOSの組み込みコンポーネントであるiSCSIターゲットサーバーは、VHDX形式をサポートするようになりました。最大64 TBのサイズのファイルのサポート、オンラインでのVHDXのサイズの拡大/縮小など、VHDXがiSCSIターゲットのデフォルト形式になりました。 、VMMを使用してiSCSIターゲットを管理するための完全なSMI-Sサポートを実装しました。 トピックを少し避けて、System Center 2012 R2 Virtual Machine Managerコンソールから、SMI-Sサポートを使用してiSCSIストレージを構成できるだけでなく、Windowsが既にインストールされているホストからスケールアウトファイルサーバーを作成および構成することもできますServer 2012 R2およびベアメタル。



ファイルクラスターの機能強化



Windows Server 2012のファイルクラスターは、クラスター内のすべてのノードが共有フォルダー（クラスターボール）へのSMBクライアント接続を処理できる場合、アクティブ/アクティブモードで構成できます。 これがスケールアウトファイルサーバーです。 このモードは、CSVボリューム（クラスター共有ボリューム）が存在することを意味します。共有クラスターストレージに保存されている同じファイルの読み取り/書き込み要求は、異なる（アクティブな）クラスターノードから来る可能性があるためです。 CSVはこの機会を実現するだけです。 ただし、CSVボリュームごとに、メタデータの操作（ファイルの作成、名前変更、削除など）を担当する所有者が引き続き割り当てられます。 次のSMBクライアントがクラスターに接続し、ファイルを作成しようとすることを想像してください。 クライアントからのSMB接続は、特定のクラスターノード、たとえばNode3を処理します。 ただし、ファイルを作成するボリュームの所有者はNode2ノードです。 この場合、要求はネットワークを介してNode3からNode2にリダイレクト（SMBリダイレクト）され、後者がボリュームの所有者としてファイル作成操作を実行します。 そのため、Windows Server 2012では、特定のSMBクライアントのすべての接続が同じクラスターノード（特定の例ではNode3ノード）によって処理され、多数のリダイレクト操作が生成される可能性がありました。 Windows Server 2012 R2では、接続はサーバーごとではなく共有ごとに監視されます。 同じSMBクライアントが別のクラスターボールに接続する場合、この接続は別のクラスターノードによって処理される場合があります。 このアプローチにより、クラスターノード間のロードバランシングがより最適化されます。 さらに、クラスターに属するさまざまなCSVボリュームの所有権がクラスターノード間で自動的に分散されるようになり、クラスターの構成が変更されると（ノードの追加、ノードのフェールオーバーなど）、この分散が変更されます。



SMB帯域幅管理の帯域幅管理



Windows Server 2012 R2では、特定の種類のSMBトラフィックに1秒あたりのバイト数で表される制限を設定できます。 このような定義済みのタイプには、VirtualMachine（仮想マシンとSMBファイルサーバーにあるVHDXファイル間のトラフィック）、LiveMigration（SMBを使用する場合の動的移行トラフィック）、Default（その他のすべてのトラフィック）があります。 制限は、Set-SMBBandwidthLimitコマンドレットを使用して設定されます。



重複排除の機能強化



Windows Server 2012で導入された最初の重複排除は、スケールアウトファイルサーバーに接続されたCSVボリュームだけでなく、開いているVHD / VHDXファイルでもサポートされるようになりました。



ストレージスペースでの階層化ストレージのサポート（ストレージスペース-ストレージ階層化）



Windows Server 2012 R2で記憶域スペースプールを作成する場合、SSDとHDDを別々のレイヤーに結合できます。 生産性を向上させるために、OSは最も要求の多いデータをSSDレベルに自動的に配置し、管理者はこのレベルに配置する必要がある特定のファイルを明示的に指定できます。
