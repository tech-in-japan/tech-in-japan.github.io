全文検索を使用したMSSQL Expressのインストール、既存のデータベースの接続、およびTCP / IPでの作業

コンピューターの周りをタンバリンでジャンプしたことを思い出したので、私は一度とすべての行動についてメモを取ることにしました。 私は常に何かを忘れています...



  1. SQL Server 2005 Express Edition with Advanced Services SP2をダウンロードします （サービスパックなしの英語版を既に持っているので、それに集中します）
  2. インストール：
    1. [登録情報]ページで、[詳細な構成オプションを非表示にする]をオフにします。
    2. [機能の選択]ページで、次の追加オプションが必要です。
      • 複製
      • 全文検索
      • 接続コンポーネント
      • Management Studio Express
    3. 「インスタンス名」ページで、「デフォルトのインスタンス」を選択します
    4. 「サービスアカウント」ページで、サービス「SQLブラウザ」の起動を追加で設定します。
    5. 「認証モード」ページで、「混合モード」を選択し、ユーザーsaのパスワードを設定します
  3. セットアップ：
    SQL Server 2005セキュリティ構成を開始し、「サービスと接続のセキュリティ構成」ブランチ「MSSQLSERVER \データベースエンジン\リモート接続」をポイントします-「ローカルおよびリモート接続\ TCP / IPのみを使用する」を設定し、サーバーを再起動します。
  4. インポートDB：
    • コマンドラインを実行する
    • osql -S 127.0.0.1 -U sa -P 2.5



    • USE [master]

      GO

      CREATE DATABASE [] on

      ( filename = N' :\ \ \ \ \ \ .MDF' ),

      ( filename = N' :\ \ \ \ \ \ .LDF' )

      FOR ATTACH ;

      GO





    全文検索カタログを持っていることがわかりました。 修復プロセスはさまざまなソースから引き継がれたため、一見すると、それは狂気の混乱の混乱のように思えるかもしれません。 これはおそらく真実からそれほど遠くない。

    exec sp_fulltext_catalog ' ', 'drop'

    exec sp_fulltext_database 'enable'

    CREATE FULLTEXT CATALOG AS DEFAULT;

    CREATE FULLTEXT INDEX ON ( ) KEY INDEX PK_ ;

    GO





    これはすべて1回の実行で実行されたものであり、リスクを冒して検索の手法を理解しようとする欲求はありません。 それまで、少し:)しかし、これらの詐欺が適切に機能した後、全文検索を行っています。



    そしてそれは動作します。

    たとえば、メモ帳でファイルを開くことができます
    %windir%\system32\drivers\etc\hosts



    そして追加
    127.0.0.1 mysampledomain.net



    その後、mysampledomain.netに関してサーバーにログインできます。
