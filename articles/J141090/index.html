オタクまたはベビーモニターの家族の子供





私は常にハブラーを読みましたが、1月の休暇のために、 「オタクの家族の子供、または私たち自身のアプローチがある」という記事が通り過ぎました。 そして、たまたま偶然、検索エンジンで彼女に出会いました。 素晴らしい記事、著者にとって大きなプラスです！



そのような申し出がありました。「多くの人がベビーモニターを購入します-そのようなラジオ局は、音でオンになります。 だから、このベビーモニターをご覧ください。 ハブに座っているので、おそらくルーターがあります。つまり、最適なオプションはWebまたはIPカメラです。 赤外線を追加すると、暗闇の中で子供を見ることができます。 繰り返しますが、タブレットから時々接続して状況を監視できます。」

しかし、彼は正しいです！ ファイヤーボックス内の従来のベビーモニター！



イントロ、スキップできます



だから、問題の本質。 子供が小さいとき、彼は通常大人より多く眠ります。 このような期間中、ほとんどの親はドアベルと携帯電話の音を消します。 一部は、彼らが本当に赤ちゃんの夢を気にしているからです。一部は、少しエゴイスト（知っている、彼は理解します）と彼が寝る時間が長いほど落ち着くからです。



そして、そのような瞬間に最も重要なことは、常に近くにいることです。 子供の睡眠が敏感な場合、彼は定期的に半分眠っていることができ、動き始めてママやパパを探します。 彼が見つけた場合、彼はさらに眠ります。 そうでない場合は、睡眠の継続を忘れることができます。



このために、ラジオやビデオの看護師などのすばらしいデバイスを思いついたのです。 あなたは子供の部屋に送信ユニットを残し、あなたと一緒に受信ユニットを持っていきます。 別の部屋でビジネスを安全に行って、子供が元気になっていることを受信ユニットから聞くことができます。



私はすぐに乳母について考えました。すべてが整っていることを確認したいからです。 マイクが突然ベビーモニターを台無しにしてしまうことはありますか？ 最も簡単なことは、既製のキットを取り、購入することです。 そして、すべては問題ありませんが、これらのデバイスにはすべて、アクションの半径という大きな制限があります。 実際、小さなアパート内では、すべてが多かれ少なかれ機能します。 そして、隣人に行く必要がある場合（誕生日に招待されましたが、赤ちゃんは寝ています）？



ユーレカ！



私は、範囲の制限を取り除き、既存の機器を最大限に使用する別のソリューションを探し始めました。 将来、私は乳母を招待する必要がある場合に何が起こっているかを記録することを計画していました。 Debianがインストールされ、ディスプレイのない新しい東芝製ラップトップはありませんでした。 はい、はい... 5秒間だけ背を向け、子供はマウスと呼ばれる明るいものを引っ張ることにしました。

このラップトップにはSSH経由でのみアクセスできます。

私はiPadを持っていて、その上に写真を表示したいのです（私は告白します。Androidを自分で購入したはずですが、妻はiPadを望んでいました。



カメラを拾うために残っています。 最初はウェブカメラを使いたかった。 しかし、その後、ワイヤーを引っ張るのはあまり便利ではなく、Wi-Fi IPカメラを使用するのが簡単だと判断しました。



その結果、サイトの1つでEdimaxカメラを見つけました。

私のプロバイダーは外部IPを提供していません。 そのため、仮想マシンの1つへのVPNトンネルを作成して、インターネット経由で監視サーバーにリモートアクセスしたかったのです。



最初は、昔ながらの方法で、Motionを使用する予定でした。 ハブでのインストールエクスペリエンスを探し始め、Ivideonのコメントにつまずきました。 インターネット上のビデオ監視サービス。 私はそれらについて読み始めました。 Team ViewerやSkypeなどの宣伝。 非営利目的での使用はすべて無料です。 3分間ですべてを開始する方法に関するビデオの説明を見ました。





ウェブカメラを備えた2台目のラップトップで何とか繰り返しました。 私はすぐにiPadで自分を見ましたが、4秒遅れました。 サウンドを獲得し、以前に行った録音を視聴して獲得しました。



基本原則。 インターネット経由で表示するには、電子メールによる簡単な登録が必要です（私が理解しているように、インターネットなしのローカル表示もあります）。 登録後、電子メールで任意の数のカメラをアカウントに「添付」できます。 権利管理などがありますが、私はそれらを使用しませんでした。 最も興味深い機能、IMHOは、サイトにビデオを表示する機能です。 さらに、 Ivideonブログによると、Flashを搭載したデバイスでのブラウジングだけでなく、HTML5経由のAppleでのブラウジングもサポートしています。



問題は、私にはLinuxがあり、IvideonにはWindows用のソフトウェアがあることです。 まあ、ワインは私を助けると思います。 Linuxでwineの下にIvideonをインストールした人がいたら、Googleで検索し始め、 Linuxバージョンを持っていることが判明しました。



そして、1つではなく、2つのインストールオプションがあります。 1つ目は既製の仮想マシンのイメージで、ダウンロードして実用的なソリューションを入手できます。 どうやら、怠け者のため。 2番目は、APTリポジトリからパッケージをインストールすることです。



ここでは発言が必須です。 Linux用のIvideonは、Windows用とはまったく異なります。 デーモンとしてインストールされ、Webインターフェースを使用して構成されます。



指示を読み、コマンドをコピーして貼り付け、15分後にはタンバリンがなくてもすべてがうまくいきました！ 私はウェブインターフェースに入って、彼らがサイトに表示したものを見ました、女の子なしでのみ:)

何よりも、Webインターフェースには簡単なユーザーガイドがあります。







設置







そして今、順番に。 IvideonのWebサイトには、Linuxを実行しているネットトップの販売にパッケージが使用されていると書かれています。 したがって、インストール中のパッケージはApache構成を変更し、IvideonのWebインターフェイスがデフォルトサイトになります。 私にとってこれは邪魔ではありません。 そして、デスクトップを使用する人はこれに注意を払う必要があります。



Ivideonリポジトリを接続します。 



$ echo deb http://packages.ivideon.com/public/full_install stable non-free >/etc/apt/sources.list.d/ivideon.list $ wget -O - http://packages.ivideon.com/public/keys/ivideon.key | apt-key add -




パッケージのリストを更新してインストールします。 



 $ apt-get update $ apt-get install ivideon-server




それだけです できた これで、ブラウザー行にhttp://127.0.0.1/を入力して、IvideonサーバーWebインターフェースにアクセスできます。 私の場合、それはローカルホストではなく、ラップトップのアドレスでした。 デフォルトのパスワードは123456です。
