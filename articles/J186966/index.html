ReSharper 8

こんにちは、Habr！



おそらく、この投稿の内容を推測しているでしょう。 はい、はい、 Visual Studioで開発する際にコードのパフォーマンスと品質を改善するためのプラグインであるReSharper 8をご紹介します。 Microsoft Visual Studioパートナープログラムに参加したおかげで、ReSharper 8.0をMicrosoft Visual Studio 2013 Previewの最近発表されたバージョンに迅速に合わせることができました。 現在、この製品は2005、2008、2010、2012を含む最大5つのバージョンのVisual Studioをサポートしています。必要に応じて、Visual Studio 2013の最終リリースが行われると、ReSharperのサポートアップデートがリリースされます。







G8には、次の分野の改善と革新が含まれています。

  • コード補完;
  • ナビゲーション
  • XAMLサポート。
  • CSSサポート;
  • 新しいリファクタリング

  • 拡張機能を使用した作業の簡素化。


さらに、ReSharper 8はVisual Studioを超えています。ReSharperで使用可能な数百のコード検査機能を備えた無料のスタンドアロンツールと、コード内の重複を検出する機能を提供しています。 ReSharper Command Line Toolsと呼ばれる新製品は、Continuous Integrationサーバーまたはバージョン管理システムと統合できます。

ナビゲーション



ナビゲーションメカニズムを簡素化するために、 Go to Everythingという新しいメカニズムを作成しました。 このメカニズムは、すべてのファイル、型、および型要素を同時に検索するという考え方です。





また、ファイル内の特定の行に移動する機能も追加しました。このためには、その番号を書き込むだけです。





さらに、ReSharperは、 ジェネリック置換のナビゲーションメカニズム、つまり、ジェネリックパラメータが置き換えられた場所とタイプを見つける機能をサポートしています。





さらに、逆コンパイルされた型のソースコードナビゲーションでは、ソリューションエクスプローラーではなく、アセンブリエクスプローラーに移動します。 これを行うには、Shift + Alt + Lを押すか、適切なコンテキストメニュー項目を選択します。

範囲内で修正





スコープ内の修正または「特定の可視性ゾーン内の修正」は、特定のタイプの修正をコード内の特定の要素だけでなく、ファイル、プロジェクト、またはソリューション全体にも適用する機能です。





ファイルセットを作成するためのテンプレート





ReSharper 8では、Live Templatesメカニズムが拡張され、同じテンプレートから複数のファイルを一度に生成できるようになりました。 これらのテンプレートの作成は簡単です。このため、テンプレートエディターにファイルを追加するための2つのボタンが表示されました。





既存のテンプレートからファイルを追加するか、新しいテンプレートを作成できます。 この場合、エディターはファイルを編集するためにいくつかの部分に「分割」されます。





マルチファイルテンプレートのすべてのファイルは共通の変数を使用できます。これにより、たとえばアプリケーションのいくつかの異なるレイヤーの作成が大幅に簡素化されます。





ReSharperの建築ツール





私たちは、Visual Studioで大規模なソリューションを使用する他の多くの開発者と同様に、プロジェクトが相互にどのように依存しているかを確認したいと考えています。 残念ながら、 Visual Studio UltimateがビルドするReSharperソースコードの依存関係表現は理解するのが非常に困難です。





その結果、プロジェクトの依存関係を表示するための好みと詩を備えた独自のツールを作成することにしました。これにより、さまざまな時点でソリューションアーキテクチャの状態を保存および比較することもできます。



これはどのように機能しますか？ Visual Studioでソリューションを開いたら、メニューからReSharper | Architecture | Build Architecture Graphコマンドを選択します。





ReSharperはソリューションを分析し、すべてのプロジェクトの依存関係図を表示します。 通常、このためにプロジェクトを コンパイルする 必要はありません 。





依存関係グラフの要素の位置は、最適な表示を実現するように自動的に決定されます。 コードメトリックスの表示機能が有効になっている場合（デフォルトで有効になっている場合）、依存関係グラフを構築するときに、リンク分析が非同期に実行されます：ReSharperはソリューション全体を分析し、プロジェクトの接続性の度合い（リンクの数）を表示します。



これらはすべて別のスレッドで行われるため、ReSharperが仕事をしている間、コードの記述を続けることができます。



ビューには2種類のリンク矢印があります。 黒い矢印は他のプロジェクトへのアクティブなリンクを示し、 灰色の矢印はビルドを中断せずに削除できる未使用のリンクを示します。 ソリューションエクスプローラーでアーキテクチャ図のスケールを選択できます。たとえば、複数のプロジェクトを持つフォルダーのみを選択した場合、これらのプロジェクトのみが図に参加します。 適切なボックスをオンにするだけで、プレゼンテーションに含めるプロジェクトを（要素ごとに）個別に選択することもできます。





このチェックボックスまたはそのチェックボックスをオンにすると、ダイアグラム内の対応する要素を表示または非表示にできます。 これにより、左側のリストのアイテムが選択または除外された直後に、異なるサブシステム間の関係の表現の変化を確認できます。





プロジェクトをグループ化するためのオプションもいくつかあります： なし （プロジェクトをグループ化しない）、 ソリューションフォルダー （プロジェクトフォルダーによるグループ化）、およびファイル構造 （ファイルが物理的に配置されているフォルダーによるグループ化）。

拡張機能を使用して作業を簡素化する





Resharperには独自の拡張機能のエコシステムがあり、ReSharper 8ではこれらの拡張機能の検索エンジンを大幅に簡素化しました。 NuGetを使用して拡張ディレクトリを配置すると、ワンクリックでプラグインをダウンロードしてインストールできます。
