HUAWEIネットワーク機器構成（スイッチング、静的ルーティング）

HUAWEIは中国最大の電気通信会社の1つです。 1988年に設立されました。



HUAWEIは最近、エンタープライズレベルのネットワーク機器のロシア市場に参入しました。 総節約の傾向を考えると、当社はシスコ機器の価値のある代替品を選択するという質問を非常に鋭く提起しました。

この記事では、例としてQuidway 5300シリーズを使用して、HUAWEI機器のスイッチングおよびルーティングサービスを構成する基本的な側面を検討します。



グローバルチーム、運用モード、CLI CISCOとの類似点と相違点。



VRPオペレーティングシステムに基づいて構築されたHUAWEIハードウェアには、ciscoのようなコマンドラインインターフェイスがあります。 原則は同じままで、構文のみが変更されます。

HUAWEI CLI機器には2つのコマンドインターフェイスモードがあります。

  1. system-view-tsiskovsky設定モードconf tの類似物。 このモードでは、コマンドラインプロンプトは[Switch]のようになります。
  2. user-view-tsiskovskogo非特権モードの類似物。 招待モードは次のようになります：<Switch>。


シスコの非特権モードとは異なり、ユーザービューから多くの機能を使用できます。特に、実行中のプロセスをリセットできます。 Telnetおよびssh接続も、ユーザービューからのみ使用できます。

シスコシステムズからのいくつかの苦情の後、サービスワードの一部がVRP OSで置き換えられました。 CLIサービスワードの対応表を以下に示します。

主なチーム：


  • システムビュー-ユーザービューから特権モードのシステムビューへの移行。
  • 保存-現在の設定をデバイスの不揮発性メモリに保存します。
  • display current-configuration-現在の構成ファイルを表示します
  • 現在の構成設定XXXXを表示-セクションXXXX設定を表示します。
  • これを表示-現在のセクションの構成出力。
  • quit-現在のセクションを終了して親に戻ります。




VLANインターフェイスの設定、スイッチの物理ポートの動作モード



VLAN作成


vlanをエンティティとして作成するには、システムビューモードのスイッチでvlan XXXコマンドを実行します。XXXはvlan番号です。 

<Quidway>system-view Enter system view, return user view with Ctrl+Z. [Quidway]vlan [Quidway]vlan 1 [Quidway-vlan1]


VLANが作成されました。 また、descriptionコマンドでは、vlanの説明または名前を指定できます。 シスコとは異なり、名前はVLANの作成時に必須の属性ではありません。



作成されたVLANをローカルネットワーク内で転送するには、GVRPプロトコルが使用されます。 これは、システムビューモードでgvrpコマンドによってオンになります。 

 [Quidway]gvrp Info: GVRP has been enabled. [Quidway]


また、インターフェイスでgvrpを有効にする必要があります。 

 # interface GigabitEthernet0/0/23 port link-type trunk port trunk allow-pass vlan 100 to 200 gvrp #


Cisco VTP（vlan transfer protocol）との互換性はなく、互換性もありません。



VLANインターフェイスの作成。 


 <Quidway> <Quidway>system-view Enter system view, return user view with Ctrl+Z. [Quidway]int vlanif 1 [Quidway-Vlanif1]ip address 2.2.2.2 24 [Quidway-Vlanif1]display this # interface Vlanif1 ip address 2.2.2.2 255.255.255.0 # return [Quidway-Vlanif1]


シスコとは異なり、マスクは省略形で記述できます。 とても快適です。

コメントは不要だと思います。



ポートモード


実際、新しいものは何もありません。 メインポートには、アクセスモードとトランクモードの2つのモードがあります。

トランクモード

ポート設定： 

 <Quidway>system-view Enter system view, return user view with Ctrl+Z. [Quidway]int gi0/0/1 [Quidway-GigabitEthernet0/0/1]port link-type trunk [Quidway-GigabitEthernet0/0/1]port trunk allow-pass vlan 1 [Quidway-GigabitEthernet0/0/1]di th # interface GigabitEthernet0/0/1 port link-type trunk port trunk allow-pass vlan 100 900 # return


Ciscoスイッチとは異なり、デフォルトでは、すべてのVLANが無効になっており、ポートトランクallow-pass vlanコマンドを許可するように強制する必要があります。

次のコマンドを使用して、ポート上のタグなしネイティブVLANを有効にします。 

 [Quidway-GigabitEthernet0/0/1]port trunk pvid vlan 600




アクセスモード 



 [Quidway-GigabitEthernet0/0/1]port link-type access [Quidway-GigabitEthernet0/0/1]port default vlan 2 [Quidway-GigabitEthernet0/0/1]di th # interface GigabitEthernet0/0/1 port link-type access port default vlan 2 # return




eth-trunk設定 


 interface Vlanif100 ip address 1.1.1.2 255.255.255.252 # interface GigabitEthernet0/0/23 eth-trunk 1 # interface GigabitEthernet0/0/24 eth-trunk 1 # interface Eth-Trunk0 port link-type trunk port trunk allow-pass vlan 100 #




コメントは冗長です。



STPを構成する



STPテストでは、Cisco 2960とHUAWEI Quidway S5328C-EIスイッチが接続されました。

スイッチでSTPを有効にするには、システムビューモードでコマンドを入力する必要があります 

 [Quidway] stp enable


デフォルトでは、HUAWEIスイッチとCiscoスイッチの優先順位は32768です。

現在のポートステータス情報を表示します。 

 [Quidway]display stp brief MSTID Port Role STP State Protection 0 GigabitEthernet0/0/1 ALTE DISCARDING NONE 0 GigabitEthernet0/0/2 ROOT FORWARDING NONE




Ciscoスイッチの優先度が高いことが判明したため、ポートの1つがブロックされていることがわかります。

グローバルSTP情報を表示します。

HUAWEIスイッチの優先度を変更します。 最小にしましょう：4096。 

 [Quidway]stp priority 4096


ポートがロック解除されていることを見てみましょう。
