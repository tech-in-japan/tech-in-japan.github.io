ReactOSはWindows 2003の類似物として開発されており、バージョン5.2として報告しています。 インストールには少なくとも6.0が必要ですが、すぐにWindows 7を目指す方が良いと判断し、バージョンを6.1に変更する方法を探し始めました。 バージョン/ MajorVersion / MinorVersionによると、コードには、予期しない場所であっても、システム全体にいくつかの場所がありました。 5.2から6.1のすべてのペアを置き換えると、結果はゼロでした-OSインストーラーも起動しませんでした。 ブート時にfreeldrとntldrをさらに検索しました。4つの場所で、_WIN32_WINNT_WS03を_WIN32_WINNT_WIN7に置き換える必要がありました。 OSがインストールされ、これが6.1であることが通知されます。 わかった？ いや なんらかの理由で、そのような代替品で実際にインストールされるものは何もありません。VirtualBoxGuest Additions、Firefox、.NET 4.0、Explorerでさえ一度も失敗しません。 さて、必要なものすべてでISOを作成し、仮想マシンに接続する必要があります。 栄光のバイト、ドライブはまだ動作しており、インストーラーはそこから再び起動します。 やった！ 起動することもありますが、特定のOSコンポーネントが不足していることを訴えます。







1つ目は自動更新サービス、2つ目は信頼できるインストーラーです。 そのようなものはありません。実際、ReactOSにはWindowsの更新があり、信頼できるインストーラーはWindows 7と一緒に表示されるため、定義によって定義できません。 OK、 PR 355更新サービスのwusa.exeとwuauserv.dllのスタブを作成します。 信頼できるインストーラーは、Windows 7からレジストリキーと共に32ビットをコピーしただけです。 現在、インストーラーは必要なすべてのものの可用性を判断し、一滴の血で契約を締結するだけです。



ファイル検証プロセスが成功し、インストールが開始されます。 kernel32モジュールのLCMapStringEx関数が欠落しているというメッセージをすぐに受け取ります。







わかりました。コード内の関数を探しています。驚いたことに、関数は存在しますが、何らかの理由でエクスポートリストに追加されません（各DLLのルートにあるCMakeLists.txtの隣の仕様ファイル）。 ビルド/インストール/実行し、再び同様のエラー。 さて、スクリプトは知られています。 インストーラーがクリックされてからこの手順を5〜10回繰り返した後、インストーラーは、OSに無人インストールモードがあることをGoogleに知らせます。 これを行うには、ファイルboot \ bootdata \ bootcd \ unattend.infでUnattendSetupEnabled = yesを有効にします。 超そり！ それらは非常に速く進んでいるので、お茶を飲む時間はありません。インストール全体で2分間です。

これを行うには、ルートCMakeLists.txtで、次の条件0x502を0x600に置き換えます。 はい、脂肪を摂取する時間はありません。少なくとも6.0が出力されます。

このベンチャーからの成功への期待は当初はまあまあでしたが、全体としての考えは真実でした。
