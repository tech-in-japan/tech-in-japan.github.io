Cordova 5.1.1およびプラグインの更新

2015年6月10日 Cordova 5.1.1がリリースされました（以前のバージョンは5.0でした）。 この出版の遅れは、ロシア語の翻訳をすぐに完成させることができなかったという事実が原因です。 基本的に、このバージョンでの変更は、大幅な改善よりもマイナーな改善でした。 約1週間前、2015年6月22日。 メインプラグインのアップデートもリリースされました。



行われたことに興味がある人のために、カットをお願いします



主なイノベーションは次のとおりです。

  • Windows 10アプリケーションのサポート
  • 新しい要件コマンドが追加されました
  • cordova prepareのサポートの改善--browserify
  • プラットフォームの新しいパッチバージョンのリリースは、ツールの再リリースなしで可能になりました




そして今、もう少し詳細：



Windows 10アプリケーションのサポート


以前は、CordovaはWindows 8.0およびWindows 8.1のアプリケーションのみを開発できました。 現在、Windowsの新しいバージョンが間近に迫っているので、このリリースの準備をするツールの外観は非常に良好です。 Windows 10は新しいアプリケーション起動モードを導入し、Cordovaアプリケーションはセキュリティポリシーの変更の対象となります。 Windows 10の変更に関する詳細は、公式ドキュメントをご覧ください。



また、Windows 8は時代遅れのプラットフォームであると宣言されており、徐々にそのサポートは無効になります。



また、Windows 10アプリケーションでは、CSPを使用してリソースへのアクセスを制限できます。これにより、safeHTML制限で機能しなかった一部のJavascriptライブラリの実行が容易になります。



新しい要件コマンドが追加されました


プラットフォームをプロジェクトに追加する前に、コンピューターで満たされていない要件を確認できます。 以前は、 cordova platform add android



を呼び出して、開発のために不足しているものを復号化する必要がありました。 代わりに、今すぐcordova requirements android



を呼び出して、要件のリストを参照してください。



それは私には思えるので、初心者にとって非常に便利です。 また、チームが機能するためには、プラットフォームの最新バージョンを使用する必要があることに注意してください。 そのため、もちろん、それらを更新する必要があります。新しいプロジェクトに問題はないはずです。



cordova prepareのサポートの改善--browserify


これで、 cordova prepare --browserify



をサードパーティのプラグインと組み合わせて使用​​して、 cordova prepare --browserify



できます。 知らない人のために、 cordova prepare --browserify



使用すると、プラグインのすべてのJSファイルを1つのファイルに収集できるため、アプリケーションの起動時にこれらのファイルの動的な読み込みを回避できます。



プラットフォームの新しいパッチバージョンのリリースは、ツールの再リリースなしで可能になりました


この革新は、コマンドラインツールを更新する必要なく、パッチバージョン番号が変更されたプラットフォームのバージョンの配信が実行されることを意味します。 したがって、新しいアプリケーションはそれぞれ、プラットフォームのより更新されたバージョンを受け取ります。 これで、アプリケーションの作成時にインストールされるプラットフォームのバージョンはコマンドラインツールに厳密に接続され、セキュリティ更新プログラムが表示されたときに、プラットフォームの修正だけでなく、プラットフォームの保護されていないバージョンをインストールしないようにツールを更新する必要がありました。



プラグインについて





簡単に言えば、主な変更点



cordova-plugin-battery-status-ブラウザープラットフォームのサポートが追加されました。

cordova-plugin-file-transfer-さまざまなバグが修正されました



公式リリースには作品のリストが記載された巨大なシートがありますが、これは主にドキュメントの更新とプラグインの自動テストの最終化に関するものです。 最終開発者があまり面白くないこと。



Cordovaの最新バージョンの作業についてコメントがある場合は、 JIRAでチケットを作成できます。 翻訳を手伝いたい人はCrowdInに登録して翻訳を手伝ってください 。
