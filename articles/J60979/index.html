Weblogic 10.3 + Mysql

新しいIDE IDE JDeveloper 11gとそれに付属する統合アプリケーションサーバーWebLogic 10.3での長いシャーマニズムの後、ついにMySqlサポートを取得する方法を知りました。



しかし、問題の本質はこれです：上記の製品は悪名高いOracle社の発案であるため、デフォルトでアプリケーションサーバーがMySqlをサポートしていないことは驚くことではありません。 そして以来 インポートされたアプリケーションでは、すべてのEJBよりも少し多くがマッスルベースになり、これが実際の問題に変わりました。

ソリューションは非常にシンプルであることが判明しました（Linuxのソリューションプロセスについて説明しますが、Windowsの場合もほとんど同じです）。



1）サーバーが使用するドメインへのパスを見つける必要があります（以降、$ DOMAIN_DIRと呼びます）

2）$ DOMAIN_DIR / libに移動すると、サーバーがそこにある.jarアーカイブを取得するのはここからであるという素晴らしいreadmeがあります

3）MysqlのJDBCドライバーの最新バージョンをそこにコピーします（Webサイトから簡単に入手できます）

4）WLSを実行します。 これですべてが機能するはずです-）



このガイドが引き続き索引付けされ、誰かが多くの時間を無駄にしないようにすることを願っています）
