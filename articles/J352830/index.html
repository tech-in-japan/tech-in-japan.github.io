Herokuでのホスティング



Herokuは、2007年6月に登場した最初のクラウドプラットフォームの1つで、当初はRubyプログラミング言語のみをサポートしていましたが、現在サポートされている言語のリストには、Java、Node.js、Scala、Clojure、Go、PHP、そしてもちろんPythonも含まれています。







Webアプリケーションは、 git



バージョン管理ツールを使用してHerokuにデプロイされるため、アプリケーションはgitリポジトリに存在する必要があります。 Herokuは、アプリケーションを起動する方法について、アプリケーションのルートディレクトリでProcfileというファイルを探します。 Pythonプロジェクトの場合、Herokuには、インストールする必要のあるすべてのモジュール依存関係のリストを含むrequirements.txtファイルが必要です。 アプリケーションがgitを介してHerokuサーバーにアップロードされた後、基本的にすべてを実行しました。アプリケーションがネットワークに表示されるまで数秒待つだけです。 本当に簡単です。







Herokuでは、必要な計算能力とアプリケーションに必要な時間に基づいてサービスレベルを選択できます。ユーザーベースが拡大するにつれて、Herokuが「dyno」と呼ぶ計算ユニットを追加購入する必要があります。

エフェメラルファイルシステム



Herokuプラットフォームは、仮想プラットフォーム上で実行される一時ファイルシステムを実装するという点で、他の展開プラットフォームとは異なります。 これはどういう意味ですか？ つまり、Herokuはいつでも仮想サーバーをリセットできるため、サーバーはクリーンな状態に戻ります。 ファイルシステムに保存したデータが保存されるとは想定できません。実際、Herokuはサーバーを非常に頻繁に処理します。







これらの条件下で作業すると、いくつかのファイルを使用するアプリケーションで問題が発生します。







  • SQLiteデータベースエンジンは、デフォルトでディスクファイルにデータを書き込みます
  • アプリケーションのログはファイルシステムに書き込まれます
  • コンパイルされた言語翻訳リポジトリもローカルファイルに保存されます


次のセクションでは、これらの3つの領域について説明します。

Elasticsearchのホスティング



ElasticsearchはHerokuプロジェクトに追加できる多くのサービスの1つですが、Postgresとは異なり、Herokuが提供するサービスではなく、Herokuと連携してアドオンを提供するサードパーティです。 今日（これらの行を書くとき）、統合されたelasticsearchサービスの3つの異なるプロバイダーがあります。







Elasticsearchを設定する前に、Herokuでは、サードパーティのアドオンをインストールする前に、無料の階層内にいる場合でも、アカウントにリンクされたクレジットカードが必要であることに注意してください。 Herokuクレジットカードの詳細を提供しない場合は、このセクションをスキップしてください。 アプリケーションをデプロイできますが、検索機能は機能しません。







アドオンとして利用可能なElasticsearchオプションのうち、無料のスタータープランが付属するSearchBoxを試してみることにしました。 アカウントにSearchBoxを追加するには、Herokuへのログイン中に次のコマンドを実行する必要があります。 







 $ heroku addons:create searchbox:starter


このコマンドは、Elasticsearchサービスをデプロイし、サービスの接続URLをアプリケーションに関連付けられた環境変数SEARCHBOX_URL



します。 クレジットカードをHerokuアカウントに追加しないと、このコマンドは実行されないことを再度お知らせします。







第16章を思い出すと、私のアプリケーションはELASTICSEARCH_URL変数でElasticsearch接続URLを探しているので、この変数を追加し、SearchBoxによって割り当てられた接続URLに設定する必要があります。 







 $ heroku config:get SEARCHBOX_URL <your-elasticsearch-url> $ heroku config:set ELASTICSEARCH_URL=<your-elasticsearch-url>


ここでは、最初にHerokuにSEARCHBOX_URL



の値をSEARCHBOX_URL



するように依頼し、次に同じ値に設定されたELASTICSEARCH_URL



という名前の新しい環境変数を追加しました。
