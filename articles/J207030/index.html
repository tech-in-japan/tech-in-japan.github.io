Litecoinに飛び込むか、暗号通貨のマイニングを開始する方法





まえがき


暗号通貨が強まり、世界の認知度に近づいていることは誰にとっても秘密ではありません。 かつて、ビデオカードでビットコインをマイニングすることに従事していましたが、1年以上もの間、ASICマイナーの出現により「廃業」のままでした。 私は常にASICファームに反対していたため、自分自身で所有者に加わらないことに決めました。 理由に関するいくつかの言葉-ASICファームは高価で、普及しておらず非常に具体的であり、これによりマイニングや暗号通貨を入力するためのしきい値が大幅に増加し、これらのデバイスの選択的分散はエントロピーの低下に寄与します。 一部の人にとっては、これらの理由はそれほど重く思えないかもしれませんが、私にとっては決定的になっています。 少し前に、ビットコインフォークの1つが強まり始めており、ASICファームから保護されていることに気付きました。 それでも、インターネット上のほとんどの情報は古くなっており、最初の農場を稼働させるためにでこぼこの道を歩かなければならなかったので、私の経験をあなたと共有することにしました。



ステップ0-状況を評価し、暗号通貨の仕組みを理解する


特に私たちの収入が彼女の繁栄に依存している場合、暗号通貨愛好家として私たち全員が彼女の繁栄を願っています。 しかし、暗号通貨市場はまだ安定していないことを覚えておく価値があり、唯一の収入源として暗号通貨をどうするかは、関連するリスクのために価値がありません。 あなたの目標が簡単なお金であるなら、それを他の場所で探すほうが良いです。そうでなければ、コミュニティにようこそ！



では、マイニングとは何ですか？ マイニングは、基本的にネットワークの提供です。 コンピューターをネットワークに接続すると、多くのハッシュを計算する1つの大きな暗号ネットワークの一部になります。 ハッシュが計算されるたびに、計算を実行するマシンはブロックを検出する機会があります。 50LTCは、マイナーブロックを見つけると授与されます（他の通貨では、この数値は異なる場合があります）。 ブロックを見つけることは運次第ですが、システムは一定数のブロックに対してプログラムされており、ブロックを見つけるのに必要な平均時間を計算できます。 規制は、システム全体の速度に応じて柔軟に設定されるシステムの複雑さを使用して実行されます。 速度はハッシュ/秒で測定されます。 執筆時点では、1秒あたりのハッシュの実際の単位はkh / s（1秒あたりのキロキャッシュ）でした。

1000時間/秒= 1kh /秒

1000 kh / s = 1mh / s

1000 mh / s = 1gh / s



鉱夫は常に、1キロヘーあたりの最小電力消費で、できるだけ多くのkh / sを取得するよう努力する必要があります。



ステップ1-ファームを構築する






それでは、Litecoin（または別の暗号通貨）をマイニングすることにしたとしましょう。 もちろん、マイニングにはメインPCを使用できますが、通常、PCにはビデオカードが1つしかインストールされておらず、ビデオカードは常にATIではないため、ファームの構築から始めます。 さらに、執筆時点では、1枚のビデオカードを使用したマイニングが間近に迫っていて、すぐに収入が得られなくなる可能性があります。 ソフトウェアのセットアップに興味がある場合は、次の手順に進みます。



農場を組み立てるとき、ルールに基づいて鉄を選択する必要があります-ビデオカード用のすべて。 つまり、どの種類のプロセッサ、RAM、またはハードドライブを使用するかはそれほど重要ではありません。PCI-eスロットの数と適切な電源は重要です。



1200ワットのモジュラー電源をすでに見つけており、RAM、ある種のプロセッサ、16GBのフラッシュドライブを備えたマザーボードをすでに持っているとします。 最悪の考えは、このすべてをケースに隠すことです。原則として、農場では大量の熱が発生するため、それに応じて農場を冷却する必要があります。 非常に簡単な解決策は、木製ケースを作り上げるか、 既製の金属ケースを購入することです。



今最も重要なことは、ビデオカードの選択です。 選択する際には、 この表とこの計算機のガイドに従ってください。 あなたの目標はkh / sの数です。 彼についてはもう少し後で。 1000kh / sを発行するビデオカードよりも700kh / sを発行するビデオカードを2枚購入するほうが安い場合があり、見つけやすいお気に入りのカードとはほど遠いため、アクセシビリティも重要な要素であることに注意してください。



また、レイザー-マザーボードからビデオカードを作成するのに役立つループを取得する必要があります。







ライザーには2つのタイプがあります-追加の力がある場合とない場合です。 食料はあらゆる農場の主な惨事であり、3枚以上のグラフィックカードを使用している場合は、追加の力を持つレイザーを使用することをお勧めします。 そうしないと、ビデオカードがまったく起動しないか、マザーボードの電源トラックが過負荷になり、焼き付けられる危険があります。 非動力のレーサーは、こちらの指示に従って動力のレーサーに変換できます 。 ビデオカードを接続するときにCrossFireXを使用しないでください。これにより、結果が低下します。



ハードウェアを構成するためだけに残ります-UEFI / BIOSに移動して、次の設定を設定します。

  • 電源をオフにすると自動的にオンになります。
  • すべての余分な周辺機器を無効にします。 USB、ネットワークカード、IOMMUのみを残します。
  • USB-HDDからのブートをメインとして設定します。
  • キーボードとモニターの存在のチェックをオフにします。


残念ながら、UEFI / BIOS用の単一のインターフェイスはないため、これらの設定の一部はまったく存在しないか、非表示になっている可能性があるため、マザーボードの説明書を読む必要があります。



ステップ2-ソフトウェアのインストール


ファームはリモートコントロールを必要とし、ビデオカードを操作する必要があるデバイスであるため、Linuxは非常に優れたソリューションになります。 このガイドでは、Ubuntu 12.04の構成について説明します。



まず、インストールフラッシュドライブを準備します。このために、 x86またはx64バージョンのミニイメージをダウンロードします。



LinuxまたはOSXを使用している場合は、



sudo dd if = mini.iso of = / dev / sdX bs = 4M

ここで、mini.isoはダウンロードしたファイルへのパスであり、/ dev / sdXはフラッシュドライブです。



まだ Windowsを使用している場合は 、Unetbootinを使用します。



すべての準備が整ったら、USBフラッシュドライブをファーム（ハードドライブとして機能するように設計された別のUSBフラッシュドライブが既に存在する）に貼り付け、パッケージのリストが表示されるまで画面の指示に従い、Xubuntuデスクトップを選択します。 OpenSSHサーバーとインストールを完了します。 再起動後、SSH経由ですぐにファームに接続できるようになります。



この段階で、ファームをリモートでセットアップする場合、Filezillaを使用することは非常に便利ですが、すべてが既に機能している場合にのみモニターをオフにすることをお勧めします-個人的な経験から、これは良いアイデアであることが示唆されます。



最初のステップは、lightDMで自動ログインを有効にすることです-ターミナルを入力します 



sudo /usr/lib/lightdm/lightdm-set-defaults --autologin username




ユーザー名をユーザー名に置き換えます。

ステップ3-ソフトウェアを構成し、プールを選択する


そのため、ソフトウェアで大騒ぎを少し中断し、自分用のプールを選択する価値があります。 プールはあなたの「雇用主」であり、プールにあなたの農場を与え、彼はそれを使用するためにあなたに支払います。 ある程度安定した利益を得るためには、プールが必要です。プールがないと、非常に長い時間ブロック（50 LTCを与えるブロック）を検索でき、運が大きな役割を果たすからです。 ただし、プールは原則として強固な力を持っているため、収入を鉱夫の間で分割することでブロックを見つけることがよくあります。 多くのマイニングの達人は、長い間マイニングする場合、プールは重要ではないと考えていますが、これは完全に真実ではありません。 プールは何らかの方法で受け取った資金を分配する必要があり、すべてのプールが同じ方法でそれを行うわけではありません。 プールを選択するときは、自分に合った支払いシステムを選択する必要があります。 現在、支払いには主に2つのタイプがあります。



  • PPS- P ay P er S hare
  • PPLNS- P ay P er L ast N S hares




PPSは、プールが計算ごとに支払いを行うことを意味します。 あなたの収入は複雑さだけに依存し、運には依存しないので、魅力的です。 それにもかかわらず、そのような安定性のために支払う必要があります-この支払い方法のプールは大きな手数料を取り、さらにプールが突然不運になり始めると、プールが破産する可能性があります。

PPLNSは、次のブロックを見つける前に行われた最後のH計算に対してのみ支払うため、手数料を最小限に抑え、破産の可能性を0に減らします。 したがって、あなたは同時にプールの運と複雑さに依存します。



プールリスト



プールを決定してアカウントを開くと、セットアップを開始できます。 .shホームフォルダーにスクリプト（たとえば、start.sh）を作成し、そこに以下を記述します。 



 #!/bin/bash export DISPLAY=:0 export GPU_USE_SYNC_OBJECTS=1 export GPU_MAX_ALLOC_PERCENT=100 ./cgminer --scrypt -o stratum+tcp://poo.l:3333 -u username.worker -p pass




username.workerをユーザー名とワーカーに置き換え、パス-パスワードstratum + tcp：//poo.l：3333-プールのアドレスに渡す



私たちは始めます-そして、大丈夫、農場は働いています！



しかし、それだけではありません。 このような設定では、多くのkh / sを失う可能性があります。これは、微調整を適用することで取得できます。 これはシャーマニズムの領域からのものであるため、私たちはすでに知っているテーブルから設定を取得するか、インターネットで見つけることを初心者に勧めます。 自分自身が高度であると考える人のために、設定について少し助けてあげます。



-thread-concurrency-同時実行、つまりcgminerが計算しようとするハッシュの数。 この値を選択する必要があります。テーブルから取得するか、シェーダーの数を設定するのがさらに簡単です。



-シェーダー-スレッドの並行性を指定する代わりに、ビデオカードに含まれるシェーダーの数を知り、それらを指定する必要があります。cgminerは並行性自体を選択します。



-intencity（-I）-詰まるビデオメモリの量。 このパラメーターは 13以上で指定する必要があります。最大値は20です。値を大きくしすぎると、メモリが上書きされてエラーが発生するため、速度が低下する可能性があります。



-g-1つのGPU上のプロセスの数



-自動ファン-自動クーラー調整



-gpu-memclock-メモリ周波数。 非常に重要なのは、250の倍数を指定する必要があることです。Litecoinが構築されている暗号化暗号化アルゴリズムを使用する場合、この頻度が最も重要です。



-gpu-engine-コア周波数-ここではそれほど重要ではありません。 実際、メモリ周波数に対するこの周波数の「魔法の比率」が重要です。 そのような周波数を見つけると、パフォーマンスが大幅に向上します。



最後の2つのパラメーターを使用するのは危険です。 ビデオカードをオーバークロックするときは、常にそれらを焼く機会がありますので、何をしているのかが確実になるまで、これらのパラメータを使用しないでください。



リモートでマイナーを開くことができるようにするには、GNU Screenで実行するのに費用がかかります 



 screen -dmS miner start.sh




マイナーをオフにせずに脱出したい場合は、Ctrl + A + Dを押して戻り、コマンドを入力します 



 screen -r miner




これで、キーボードだけでなくモニターもオフにできます。
