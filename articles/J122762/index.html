オペレーティングシステムCP / M-80



CP / M-80は、Intel i8080プロセッサと互換性のあるZilog Z-80を搭載したマシン用に1970年代にDigital Researchによって開発されました。 CP / Mは、マイクロコンピューターの制御プログラムの略です。 接尾辞-80は、PCおよび8088/8086プロセッサにCP / M-86が登場したときにのみ、正式に名前に登場しました。 PCが急速に人気を博し始める前に、比較的低電力の個人用コンピューターのニッチは、CP / M-80を備えたマシンによって正確に占有されていました。



CP / M-80は、近代的な意味での最初のオペレーティングシステムの1つでした。 物理的な機器からプログラムを抽象化するような機能が登場したのはそのためであり、これがその（相対的な）人気の理由でした。 CP / M-80は、CCP（コンソールコマンドプロセッサ-プリミティブコマンドインタープリター）、BDOS（基本ディスクオペレーティングシステム-システムコールの実装）、およびBIOS（基本入出力システム-標準デバイスのドライバー：ディスク、コンソール、プリンター、シリアルインターフェイス）。 CP / Mを新しいマシンに移植する場合、BIOSのみを書き換える必要がありました。 CCPとBDOSはデバイスに依存しませんでした。 アプリケーションプログラムはBDOS機能、または必要に応じて直接BIOSを使用できますが、異なるマシンのディスクコントローラーを操作する知恵を知る必要はありませんでした。



もちろん、実際には、すべてがそれほどスムーズではありませんでした。 BDOS / BIOSの最大の欠落は、テレタイプ端末で機能する最も原始的なものを除いて、画面上の出力機能の標準化がないことです。 その結果、インストール中にスクリーンエディター（WordStarやTurbo Pascalなど）などの機能を含むプログラムが、開発者が知っている数十台のマシンから選択するか、リストにマシンがなかった場合は画面サイズとさまざまな制御コードを自分で入力しますカーソルなどを移動するために。まあ、これらはささいなことです。

繰り返しになりますが、このメモリカードはCP / Mを搭載したすべてのマシンで有効です。 したがって、多くの不確実性があります。 たとえば、ビデオメモリはありません（実際に存在するという事実ではありません-マシンが一般にテレタイプに接続されている可能性があります）が、内容が保証されているアドレスは0x0000-0x0100のみです。 システムコールを行うには、間違っていなければその番号をレジスタCに入れてから、CALL 0x0005命令を実行する必要があります。 0x0005のJMP命令は、BDOSの必要な場所に既に移動します。 また、このエリアには次のものがあります。

  • BIOSエントリポイント（アドレス0x0000）。 この関数はマシンを再起動しますが、簡単な計算で他のBIOS関数のアドレスを取得できます。
  • 現在のドライブ番号（A：= 0、B：= 1 ...）;
  • 現在の入出力フローの方向（たとえば、コンソールを印刷デバイスまたはシリアルポートに向けることができます）。
  • ロードされたプログラムに、BDOS関数を呼び出すための引数として適切なFCBデータ構造のコマンドラインパラメーターとして転送されるファイルの名前（2つ以下）。
  • 現在のプログラムの完全なコマンドライン（アドレス0x0080-0x00FF-127文字以内）。


TPA（ダウンロード可能なプログラムの領域）の容量も保証されていません。原則として、実装に応じて、10〜52 KBの範囲になります。



CP / M-80ファイルシステムはFATよりも大幅にシンプルですが、タスクに十分に対処できます。 ファイルは8.3という名前で保存されます。 作成/変更の日付は保存されません（CP / MのマシンでのRTC時間の可用性は保証されませんでした）。 ファイルサイズは、切り上げられた128バイトの精度で保存されます（ディスク上で占めるスペースの量は最大4 KBです）。 一般にディレクトリもありません-CP / Mはハードドライブで動作するように設計されていませんが、フロッピーディスクで生きることができます。 ただし、「ユーザー領域」と呼ばれる一種の代理がありました。 これらの領域には0〜15の番号が付けられ、それぞれに個別のファイルを保存できます。 彼らはコマンドUSER nを切り替えました。



コマンドについて話していたため、DIR、SAVE、ERA（ファイルの削除）、REN（ファイルの名前変更）、USERのようなコマンドはほとんどありませんでした。 （SAVEはかなり特殊な機能を実行しました-TPAの内容をディスクに保存します-たとえば、デバッガーを起動し、プログラムをロードし、それをスモークし、メモリに残っているバージョンを保存したい場合） ZCPR3インタープリター、はるかに洗練された。 しかし、私自身は試していません。



一般に、CP / MはDOSの初期バージョンを非常に連想させるものであり、そう、一般に、そして不思議ではありません-DOSはCP / Mのクローンとして開発されました。 CP / M-80には、多くの応用ソフトウェアがありました-オフィスプログラム、開発ツール、ユーティリティ。 しかし、PCの出現と急速な発展により、これらすべてはすぐに無関係になりました-DOSとの重要な競争をすることができなかったCP / M-86バージョンのように。

CP / MとSpectrumをクロスする方法



CP / MとSpectrumには、Z-80プロセッサで動作するという共通点が1つだけあります。 メモリカードを比較すると、ほとんど何も共通点が見られず、CP / MとSpectrumマシンで予想される周辺機器は大きく異なります。 それにもかかわらず、Quorumの作成者は、CP / Mと同時に動作できるマシンを作成でき、かなり普通のSpectrumのようになりました。 何を変更する必要がありましたか？







まず第一に、クォーラムでは、ROMが私たちに干渉します。 CP / MにはROMがありません（BIOSはROMに配置されている場合がありますが、BIOSはアドレス空間の最初ではなく最後に配置され、そのボリュームは16 Kバイトよりはるかに小さくなっています）。 そのため、クォーラムに0x00メモリ管理ポートが追加されました。これにより、Spectrum-Quorum ROMを0x0000-0x3FFFアドレスの追加の16Kバイトのメモリに切り替えることができます。 そのため、ちなみに、新しいQuorumモデルは「Quorum-48」ではなく「Quorum-64」と呼ばれます。実際には、CP / Mモード用に特別に16 KバイトのRAMが残っています。



（実際には、「Quorum-64」はまだ最年少モデルではありません。以前は「Quorum BK-02」または「Quorum BK-04」でした。これはSpectrumの単純なクローンであり、CP / Mを起動できません。 ）



ROMに加えて、ビデオメモリが干渉しているため、貴重なTPAスペースの真ん中に留まります。 したがって、ポート0x00の2番目の機能は、ビデオメモリアドレスをデフォルトの0x4000-0x5AFFから0xC000-0xDAFFに切り替えることです。 2番目のモードでは、ビデオメモリがBDOSとBIOSの間で圧縮されますが、クォーラムに適合しているため、これを知っているので、プログラムが知る必要はありません。



ビデオメモリといえば、Spectrumには、16色の256x192のビデオモードが1つしかありません。 （より正確には、256x192はモノクロ画像の解像度であり、32x24の解像度で色属性が重畳されます。したがって、各8x8ピクセル領域には2色しかありません。詳細は8x8ピクセルのフォントで表示されます）各32文字の24行-これはビデオメモリ構造に便利に重ねられていますが、ほとんどの実用的な目的のために率直に小さくなっています。 ほとんどの非適応CP / Mプログラムでは、通常、文字列に少なくとも80文字が必要です！



したがって、CP / Mクォーラムでテキストを表示するための4つのモードがあります-1行あたり42文字（6x8フォント）、1行あたり51文字（5x8フォント）、1行あたり64文字（4x8フォント）および1行あたり83文字（3x8フォント）。 デフォルトでは、ロード後、行ごとに42文字が使用されます。
