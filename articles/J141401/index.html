Windowsストアでの最初のウクライナアプリケーション

3月28日は、ウクライナの開発者がWindows 8- Euro Cup 2012向けに作成した最初のアプリケーションです。 このアプリケーションは、Windowsストアで最初のウクライナのアプリケーションです。 アプリケーションの開発者はDigital Cloud Technologiesでした。



私はこのアプリケーションの開発者の一人であり、その作成プロセス、リリース中に発生した問題について少しお話ししたいと思います。

チームは、 Sergey Lutai率いる5人の開発者で構成されていました。MVPSilverlightは、 Windows 8でのC＃開発に関する世界初の本とDenis Reznik MVP SQL Serverの共著者です。

開発者は、モスクワでMicrosoftが実施したコースを除き、プラットフォームの経験がありませんでした。 アプリケーションアーキテクチャはMVVMパターンに基づいています-これは非常に役立ちましたが、いくつかの深刻な問題も引き起こしました。



Windows 8 Developer Previewのバージョンでアプリケーションの開発を開始しました。

当時、VS 2011 Express、フォーラム、MSDNのみが利用可能でした。 その瞬間、情報不足の通知を喜んで受け取りました。「この資料はまだ利用できません。 このプレースホルダートピックは、後のリリースに含まれるドキュメントの例として提供されています。 この碑文は開拓者の感覚を植え付け、望ましい結果を達成するための抵抗できない欲求を加えました。 上記のすべてにより、試行錯誤によって直感的に行動し、開発プロセスが遅くなりました。 ちなみに、Expression Blendはhtml / jsプロジェクトのみをサポートし、スタジオのデザイナー機能は非常に遅く、時々ハングしました...



2月28日に、Windows 8 Consumer Previewの新しいバージョンがリリースされました。スタジオのパフォーマンスは著しく向上し、一般的に「呼吸が楽になりました」:)。 Windows 8の新しいバージョンでは、Metroアプリケーションの.NET APIに変更が加えられ、一部のクラスの名前が変更され、メソッドが変更されました...アプリケーションを新しいバージョンに移行する必要がありました。

さらなる開発の障害の1つは、「 致命的なエラー 」 エラーでした。 これは予測不可能な場所に落ち、その時点でいくつかのトピックが開発者フォーラムで開かれ、その解決策についての推測が可能になりました。 モデレーターは、次のアップデートでバグを修正することを約束しました。 私は彼女を追跡するために多くの努力をしなければなりませんでした。 私たちの場合、およびフォーラムで説明されているほとんどの場合、エラーは静的アプリケーションリソースの使用にありました。



すべての不便にもかかわらず、解決策を見つけました-WindowsストアおよびMSSWIT会議の開始時に起動するアプリケーションをデバッグおよび準備しました。 マイクロソフトの副社長であるヨーロッパ本部のWolfgang Ebermannは、会議の冒頭で彼のプレゼンテーションで私たちのアプリケーションを示しました。



新しい問題を解決したり、予期しない問題を理解したりするのはとても楽しかったです。 アプリケーションの安定バージョンがWindowsストアに追加されました。 アプリケーションの開発は継続されており、将来的には新しいバージョンのアプリケーションが提供されます。



最後に、API自体について少し説明します。 マイクロソフトのスタッフは本当に素晴らしい仕事をしました。 例として、従来のファイル選択シナリオまたはストリームの操作は簡単に記述でき、プラットフォームと統合できます。検索またはデータ共有の実装は、数行のコードで実装されます。 クラスとメソッドの機能は非常に直感的であるため、平均的なプログラマーは、各クラスの説明を読んで、MSDNでフォーラムを調べたり時間を無駄にしたりする必要はありません。 .NET FrameworkまたはXAML開発の新機能については、MSDNにこの機能またはその機能を実装するための多くの「ガイド」と、無数の例が含まれています。

個人的には、このプロジェクトに参加できてラッキーだったことを嬉しく思います。新しいAPIを使った素晴らしい経験でした。
