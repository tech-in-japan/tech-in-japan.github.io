1年間の失敗、またはAndroid用ゲームの開発経験

こんにちは、Habrasociety！



この記事では、モバイルプラットフォーム向けのゲーム開発における（最初ではない）経験、1つのゲームの作成の歴史、得られた結果についてお話したいと思います。 記事のタイトルから明らかなように、この経験は失敗とみなすことができますが、以下で説明する内容を確認する方法を次に示します。



背景


しばらく前、2年間、Windows Phone用のゲームを開発しました。 この期間中に6つのゲームがリリースされ、そのうち2つだけが有料+試用版であり、わずか数ドルで、残りは広告なしで無料でした。 XNA + C＃があったため、このプラットフォーム用のゲームを開発しました。私はこの束が本当に好きでしたが、その後、噂があり、XNAサポートが終了する現実がありました。 他のエンジンの検索が開始されましたが、中でも私はUnityが最も好きでした。 他のエンジンにエディター、C＃、クロスプラットフォームなどがある場合でも、小さなゲームに使用することの妥当性については議論したくないとすぐに言います。 当時、UnityはまだWindows Phoneをサポートしていませんでしたが、それは気にしませんでした。 少なくとも今のところ、このプラットフォーム用に開発したくありませんでした（ある友人が言ったように、なぜ死んだ馬に乗るのか）。 率直に言って、私のゲームは弱かったが、選択がなされた。 Windows Phone開発者のサブスクリプションは終了し、新しい一連の研究、チュートリアル、そして実際にはゲーム開発が始まりました。



ゲーム選択、開始


ご存知のように、難しいことから始めることはできません。 このため、トライアルゲームのアイデアが選ばれました。WindowsPhoneで既に実装を開始しました。つまり、パイロットブラザーズのスタイルのパズル（スクリーンショット）ですが、メカニズムが少し異なります。 私のゲームでは、オブジェクトをクリックすると、行全体と列全体ではなく、選択したオブジェクトと隣接オブジェクト（上、下、左、右）の状態のみが切り替わります。 スイッチング効果を視覚的に表示するために、インクルージョンまたはイグニッションを直接使用することにしましたが、何が燃えますか？ 星！ そのため、「シャイニングスター」という名前がすぐに選ばれました。 開発が始まりました。



予算がまったくなかったと言う価値があります。 したがって、プログラミング、グラフィックス、プロモーションなど、すべてが独立して行われることになっています。



開発


ここにはほとんど問題はありませんでした。 基礎を非常に迅速かつ簡単に作成したため、機能とメカニズムは非常に単純です。 私は、回転、スケーリング、スライドなどの形の異なる効果で苦しむ必要がありました。



回転-Unityにはオブジェクトを回転させるメソッドがあります。 しかし、正直に言うと、彼らは時々奇妙な仕事をします。なぜなら、一部はランダムに突っ込んで、自分に合ったオプションを探したからです。



レベル選択画面でスライドが実装されました。 レベルのあるアイコンの一部のみが画面に表示され（一部のサイズは画面のサイズによって異なります）、「左」および「右」ボタンで切り替えると残りが表示されます。



スケーリングは、私が理解するとすぐに非常に簡単でした。 基本的な考え方は、ゲームはすべての画面サイズで見栄えがするということです。 ネットワーク内のこのトピックに関するレッスンは多くありません。主にフォーラムを勉強しなければなりませんでした。 ちなみに、私はすぐにUnityの初心者ユーザーに英語のフォーラムに行くようにアドバイスしたいと思います。さまざまなトピックに関するほとんどのロシア語のフォーラムとは異なり、人々は常に助けようとします。 その結果、特定の画面サイズを基準として選択するオプションを思いついたので、携帯電話またはタブレットの実際の画面サイズに対する係数が考慮されます。 すべてのゲームオブジェクトは、この係数に応じてスケールと位置を変更します。 個人的に、私の係数は1ではありませんでした、なぜなら さまざまな種類のゲームオブジェクトは、さまざまな画面で異なって認識されるという事実のため、さまざまにスケーリングする必要がありました。 小さな画面上の一部のオブジェクトは標準サイズのままにすることができましたが、他のオブジェクトは最初のオブジェクトに比べてわずかに大きくする必要がありました。



レベル


ゲームの非常に最初のバージョンでは、20レベルを作りたいと思いました。 まあ、ゲームは面白いでしょう。 賢明なアドバイスによると、この数は（計画で）〜100に拡張されています。 もちろん、私は自分でレベルを発明し、それらが解決可能かどうかをチェックしたくありませんでした。 私はプログラマーなので、なぜ自分をmyself笑するのでしょう！ しばらくすると、ジェネレーターの準備が整い、指定されたサイズのレベルのすべての可能な組み合わせが提供され、レベルが解決されたかどうかがチェックされ、テキストファイルに出力されました。 これは、このゲームで私の最大の問題があった場所でした。 競技場のサイズ4 * 4でレベル数が数万単位で測定された場合、フィールド5 * 5の場合は既に数百万、フィールド6 * 6の場合は...わかりました。 はい、私はプログラマーです。ジェネレーターを作成し、レベルを解決するように彼に教えることはできますが、どのレベルが美しく、ゲームに参加するか、どのレベルを排除するかを選択することはできません。 したがって、これらの数百万のレベルすべてを手動で選択して表示する必要がありました。 私も最初は試してみましたが、アクションの規模を把握し（そして、フィールドを10 * 10までのサイズにしたかったのです）、気分が悪くなりました。



好むと好まざるとにかかわらず、私はさまざまなパターンの手動描画に戻る必要がありました。 これらのパターンはゲームに導入され、私は個人的に自分の携帯電話で決定しました（同時に、解答性テスト+レベルを解くことによる審美的な喜び）。 解決できないレベルが確定し、興味のないレベルは破棄されました。 そしてしばらくして、サイズとサイズ内の複雑さの順に並べられた120レベルの準備が整いました。



レベル作成の例
これがレベル作成プロセスの様子です。 グリッドはフィールド全体、星がある場所の黄色のセル、星がない白いセルをマークします。 左上隅には、ゲームで保存および処理される形式のレベルコードがあります。 右上隅にある明るい緑の数字は、そのような多くのステップでレベルを解決したことを意味し（未解決のレベルにはまだ赤い四角があります）、感嘆符はゲームに入るためにレベルが選択されていることを意味します。







番号120は偶然選ばれませんでした。 レベル選択画面のさまざまなサイズの電話機では、これらのレベルのさまざまな数のアイコンに対応できます。 120という数字で、私はアイコンをそれぞれに同じ数のアイコンを持つグループに分割することができました。これは重要です。 最初のページには12個のアイコンを、最後のページには10個のアイコンだけを表示したくありませんでした。



グラフィックス


私の意見では、私のゲームのグラフィックは完全に進化しました！ 当初、私はゲームを非常に高速にし、それらの20レベルで「何が」をリリースしたかったのです。 そして、この「あれ」は最初は本当に良さそうに見えましたが、他の人からいくつかのヒントを受け取った後、私は深く考え、すべてを描き直し始めました。 たとえば、Angry Birdsなど、さまざまな有名なゲームに基づいて、UIの外観、レベルアイコンが選択されたときのレベルアイコンの位置を考えました。

2つのボタンオプションがあるメインメニュー。 もちろん、最初の後に、2番目は美しいように見えました！ 最終オプションが作成されるまで。

最終オプション







TexturePackerはグラフィックレイアウトで私を大いに助けてくれました。 その無料の機能は私たちの目的には十分であり、アトラスを非常によく収集します。 私はいくつかのアトラスを用意しました-1つはボタン用、2つはゲームオブジェクト用（ロシア語と英語のローカライズ版）です。 確かに、Unityの古いバージョンでは、UV座標を使用して必要な部分をカットする必要がありましたが、これはやや不便です。 Unityの新しいバージョンでは、スプライトの助けを借りて、すべてをより簡単にすることができます。



音と音楽


ここに問題はありません。 サウンドを無料で購入または取得できるリソースは多数あります。 なぜなら 予算はありませんでした。これらのリソースで無料でダウンロードできる適切なサウンドとメロディをいくつか選択しました。 すべてが絶対に合法であり、盗難はありません（はい、私にとってこれは重要です）。

主なアイデアは、プレイヤーがバックグラウンドで静かな音楽で考えることができる穏やかな環境を作成することでした。 しかし、ゲームの最初の更新により状況が変わりました。設定で希望のメロディ（穏やかまたは速い）を選択する機能を備えたリズミカルな音楽が追加されました。



リリース、プロモーション、結果


Android専用のゲームをリリースする予定だったことに注意してください。 対応する電話はありますが、ゲームをテストできるiOSデバイスはありません。実際のテストなしではリリースしたくありません。 また、その理由は、Google Playへの登録は25ドルの1回限りの支払いであり、Appleは毎年99ドルを支払う必要があるためです。 おそらく、iOSでゲームをリリースしますが、少し後でリリースします。



リリース後すぐに、私は多くのAndroidフォーラムでゲームについて書き、すぐに表示された結果を追跡し始めましたが、これはほとんど停止しました。 その後、2つの更新プログラムをリリースしました。各更新プログラムはフォーラムで報告しました（少なくとも一部は更新されました）。 また、相互評価のバリエーションもありました（誰かに高い評価を与え、ゲームを賞賛するように求めます）。 しかし、これはエイリアン（常に少なくともある程度ではない）のゲームをスマートフォンにインストールすることを意味し、他の人の詐欺の可能性があります（見返りにあなたのゲームに感謝するという事実ではありません）、私はすぐにこのオプションを拒否しました。



現時点では、ダウンロード数は非常に少ないです。 何らかの失敗があると思いましたが、それは非常に多くのことです...これは、フォーラムでGoogle Playへのリンクだけでなく、apkファイル自体も投稿したという事実に一部起因し、統計によると、そのダウンロードは市場からのゲームのダウンロードを超えています。



ゲームは無料です。 収益化として、広告配信が選択されました。 開発プロセス中に、いくつかのモジュールを接続したかった-1つを表示することができない場合、2つ目を表示するなど。 最初のものは有名なChartboostで、RevMobとPlayHavenを接続しようとしましたが、いくつかのヒッチが出ました（率直に言って、ゲームで動作しませんでした。理由はまだわかりませんでした）。 。 ただし、上記（ダウンロード数と、ユーザーが広告をクリックするか、広告されたゲームを掲載したいという願望）を考慮すると、収入がないことがわかります。



結論


彼らが言うように、悪い結果は結果でもあります。 Unityでの開発の開始（Windows Phoneでの時間は考えていません）からリリースまで約14〜15か月が経過しました。 良い意味で、このようなゲームは1か月間のパートタイム雇用で書かれており、これでさえマージンです。 理由は時間の不足でした 。 いいえ、十分な時間はありました-夕方の数時間、週末の数時間は、長い間停止することなくゲームを作り続けるのに十分でした。 そして、私は十分な道徳的健康を持っていませんでした。 この期間中に個人的な問題がたくさんあり、ゲームを作りたいというアイデアがありましたが、私はコンピューターに座ってモニターをじっと見つめていましたが、インスピレーションがないことに気づき、まったく何も実現できず、それをオフにしました。 そして今年は過ぎ去りました。



負の点から始めましょう。 私は本当にゲームに多くの時間を費やしましたが、ここに私の制御を超えた状況があります。 ゲームは率直に失敗しました、それは私に収入をもたらしませんでした（まあ、数ドルは数えません）。 そして、この観点から、これは完全な失敗です。



しかし、おそらくもっとポジティブな点があります。 まず第一に、これは経験です！ このゲームでは、Unityの基本機能を1つの注意事項で学習しました。このゲームは、2Dがまだない古いバージョンで開発されました。 この新しいテクノロジーを使用して2Dゲームを開発できるようになりました。これは、ゲームからの私の成果のほとんどが不要になるためです。



このゲームで、グラフィックが本当に重要であることに気付きました！ 決して忘れないでください！ 最初の設計オプションを使用してゲームをリリースした場合、私は彼を恥じます。 今、私は美しいゲームをリリースしました。 はい、彼女は失敗しましたが、私はまだ彼女を誇りに思っています！ 予算があれば、優秀なデザイナーを雇うことができます。 そうでない場合は、結果が本当に価値があるまで、何度も何度もすべてを再描画します。



私がここで犯した唯一の大きな間違いは、ゲームのジャンルの選択だと思います。 おそらく、ほとんどのプレーヤーはアクションゲーム、アーケード、または何か他のものを愛していますが、このタイプのパズルはあまり好きではありません。 人々は私のゲームを見て、誰かがグラフィックスを好まないかもしれません、そして誰かがジャンルのためにそれを断ち切ります。 そして、私が開発する次のゲームについて話す場合、ジャンルを変更するだけで、残りはやります！
