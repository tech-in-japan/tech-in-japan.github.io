JelasticのGlassfishクラスタリング

ご存じのように、 GlassFishは、完全な産業用クラスタリングサポートと幅広い機能を備えた信頼性の高いJava EEアプリケーションサーバーです。

最近まで、GlassfishはJelastic Java PaaSで単純に別個のサーバーとして使用されていましたが、現在、高可用性（HA）を含むこのサーバーのすべての機能をサポートしています。



管理ドメインの概念に基づいたGlassFishの「ネイティブ」クラスターアーキテクチャを保存しました。 管理ドメインは、 DAS （ドメイン管理サーバー）によって制御されるクラスターとインスタンスで構成されます。



管理コンソールを使用して中央リポジトリを管理できます。 これは、Glassfishで利用可能なすべての機能をサポートする使いやすいGUIです。 DASはJavaドメインインスタンスを管理し、 GMS （グループ管理サービス）はクラスターとそのインスタンスに関する情報を提供します。







GlassFishでのセッションレプリケーション




クラスター内のインスタンスはペアに分割されます。 インスタンスの1つがクラッシュした場合、その上でリクエストが処理されたユーザーは、クラスターの別のインスタンスに自動的に転送されます。 「インスタンスパートナー」には既にフォールドセッションのすべてのセッションがあるため、エンドユーザーは変更に気付かないでしょう。 両方のインスタンスが失敗すると、ユーザーの要求は別のクラスターにリダイレクトされます。 JelasticはNGINXバランサーを使用してリクエストをリダイレクトします（他のアプリケーションサーバーの場合と同様）。





スケーリング


もちろん、スケーリングについては忘れませんでした。 サーバーの負荷が増加または減少する場合に備えて、水平スケーリングと垂直スケーリングの両方が提供されます。 これは、クラスターのサイズと数の両方が異なる可能性があることを意味します。 Jelasticは、このような高度なスケーラビリティシステムを提供する最初のPaaSです。



JelasticのGlassFishクラスタアーキテクチャは標準に非常に近いため、すべての機能を使用できます。
