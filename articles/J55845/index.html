Mobify-モバイルパブリッシングのためのサービス



Webへのモバイルアクセスは、毎月約20％という素晴らしいペースで成長しています。 ますます多くの消費者がスマートフォンを購入し、お気に入りのサイト（ソーシャルネットワーク、ニュースリソース、ブログ）にアクセスします。 問題は、サイトがモバイルアクセス用に最適化されていない場合、ユーザーが満足せず、戻ってこない可能性があることです。 Opera Mini / Safariなどのブラウザでさえ、サイトのインターフェースをhttp://m.facebook.comやhttp://m.espn.comなどの便利なモバイルポータルに大幅に移動させることはできません。 モバイル画面向けのWebサイト最適化は、Webデザイナーの特権である必要があります。



どうする .mobiなどの別のモバイルサイトを作成できます。これはSEOに悪影響を及ぼし、多くの場合、時間コストを正当化しません。 開発者にモバイルレンダリング用のモジュールの作成（および保守）を依頼できますが、これも非常に時間がかかります。 rssフィードを読むためのシンプルなページを作成できますが、これだけでは十分ではありません。検索が必要で、コメントが必要で、サイトのフルバージョンに存在する多くの機能が必要です。



「モバイルプロジェクション」を作成するためのサービスであるMobifyを紹介します。既存のサイトの軽量バージョンです。 Mobifyを使用すると、Webデザイナーは、ダウンロードまたはインストールすることなく、プロジェクトのモバイルバージョンを同じドメイン（m.site.com）にすばやく配置できます。



Mobifyの仕組み-http : //mobify.me



  1. mobify.meの WebアプリケーションでWebサイトを開きます
  2. モバイルユーザーに表示されるブロックをクリックします。 2〜3の基本パターンについてこのプロセスを繰り返します。
  3. CSSを調整して、美しく見えるようにします。 すべてのCSSクラスは変更されません。




それだけです。今では、DNS CNAMEポインターm.site.com => you.mobify.meを指定し、JavaScriptを入れてメインサイトの携帯電話を検出します。 完全に機能するモバイルバージョンがあります。10〜100倍高速で動作し、数千の異なる電話を認識し、ソースサイトと自動的に同期します。 Mobifyは、セッション、Cookie、およびPOST / GETリクエストをサポートしています。



Mobifyを使用するサイトの例：



  • http://m.alistapart.com
  • http://m.techvibes.com
  • http://spin.mobify.me




このプロジェクトは、ジャンゴ、GAE、EC2に基づく国際チームによってバンクーバーで開発されています。 最近Mobifyを立ち上げました-TechCrunchや私たちのような他の人が私たちについて書く前にあなたのフィードバックを聞きたいです！
