ServiceNowの開始方法：会社の簡単な歴史

Gartnerによると 、ServiceNowは2017年の最高のITSMソリューションです。 この記事では、会社の過去を調べ、サービスがどのように発展し、誰がその起源に立ち会ったかを見てみたいと思います。





/ Flickr / マークウォーカー / cc



2003：プラットフォームを作成する



ServiceNowはスキャンダルの結果として登場しました。2002年に、Peregrine Systemsは破産を宣言しました-証券詐欺に巻き込まれました。 フレッド・ラディはテクニカルディレクターでした。 彼はPeregrineの資産管理ソフトウェアの開発に13年を費やしました。 2003年10月に、Fredは手数料なしで会社を正式に退職しました。 お金の代わりに、フレッドは箱入りソフトウェアの使用経験を積み、ソフトウェアをもっと簡単にする時だと気づきました。



2003年12月、Fred Luddyはクラウドベースのビジネスプロセス自動化システムであるGlideを発表しました。 Glideはサービスとしての最初のプラットフォーム（PaaS）ですが、別の意見もあります。 Glidesoftのアイデアは浮上しており、後にService-nowに名前が変更され、その後ServiceNowに名前が変更されます。



会社の名に関連した多くの噂があります。 彼らはフレッドが飛行機でコードを書いたと言います。 接近する前に、彼はパイロットが航空機が目的地まで「計画」（英語滑空）すると発表したのを聞いた。 そして彼はその言葉が好きでした。 開発者は、GlideRecordまたはgrによってこの名前に精通しています。 API



今日会社のウェブサイトにアクセスするには、servicenow.comにアクセスします。 しかし、元のURLにはservice-nowが含まれ、中央にハイフンが付いています。



同社のブランドマネージャーはハイフンを削除することにしましたが、誰かが既にハイフンなしでドメインを所有していることが判明しました。 所有者は誤ってServiceNowの顧客の1人であることが判明しました。 交渉の後、ServiceNowは、ハイフンなしで既知のアドレスを取得しました。



2004：ソフトウェアは正常に機能するはず



フレッドは友人と一緒に、ITインフラストラクチャライブラリ（ITIL）とサービスデスクをサポートするアプリケーションを作成します。 チームのモットーはフレッドのフレーズでした 。「IT業界には、機能するツールが必要です。 そしてそれを彼女に渡します。」



2004年初頭、Fred Luddyのチームは投資家Paul Barberから930万ドルのスタートアップ資金を受け取りました 。 フレッドはクラウドを新製品のプラットフォームとして選択しました。 彼は、プログラムを簡単に構成および実装できるように、プログラムをアクセス可能かつシンプルにしたかったのです。 このため、チームはサービス指向アーキテクチャ（SOA）を使用することにしました。 その実装のシナリオは理想的でした。新会社では、すべてのビジネスプロセスがゼロから開発されたため、変更する必要はありません。



その結果、開発者は標準を定義し、Fredが夢見ていたJava製品を取得するためのアーキテクチャを作成しました。使いやすく、構成し、実装します。



2005：市場参入



市場に参入した後、チームは最初の10人の顧客にフィードバックと引き換えに無料でGlideを使用する機会を与えました。 開発者自身は 、後にServiceNowのクライアントとなったカフェRed Robin Gourmet Burgersで会いました。



9月に、WagerWorks はサービスの支払いを行う最初のServiceNow顧客になります。



2006年：起業家精神のためのSAAS



Glidesoftは、その名前をServiceNowに変更します。 企業のIT組織-Qualcomm、TIAA-CREF、Cricket Communications、MetLife、およびBurlington Coat Factory-ITサービスプロバイダーとしてService-now.comを選択してください。



フレッドは、IT部門とIT以外の小規模組織の両方でSaaSの需要があることにすぐに気付きました。 Service-now.comは、企業のIT顧客および起業家にSaaSを販売した最初の企業です。 中小企業向けのプロジェクトには、ブリトーを注文するサービス、醸造プロセスを管理するサービス、結婚式を組織するサービスがありました。



2007：100の顧客



3月、Service-now.com は投資を完済し 、最初の収入をもたらします。 また、4月には、最初のグローバルなKnowledge07ユーザー会議が開催され、100人のService-now.comの顧客が集まります。 10月、チームの規模は50人に増加しました。



2008：成功としての経済危機



年の初めに、フレッドはインタビューで次のように述べています。 私たちは経済危機の影響を受けず、それでお金を稼ぐことができました。」 6月、Service-now.comは3年連続で年間収益を2倍にします。



また、7月にロンドンで開催された最初の年次Service-now.com会議-英国エグゼクティブブリーフィングおよびユーザーグループ会議。 会議はヨーロッパからの顧客のために開催され、その数は絶えず増加していました。 11月、Red Robin Gourmet BurgersはService-now.comの200番目の顧客になりました。



2008年時点で、29か国の25,000人がこのプラットフォームを使用しています。 地理的拡大の一環として、ServiceNowはロシアのIT Guildなどの地元市場のパートナーと協力しています。



2009：新しいリリースと機能



ServiceNowは、年間約2つの新しいバージョンのプラットフォームをリリースしています。 以降のリリースは、世界の都市にアルファベット順に名前が付けられます。 しかし、このアプローチは2012年に初めて使用されました。最初のリリースはAspenでした。 これに先立ち、新しいバージョンの名前は、シーズンとリリース年で構成されていました：Winter 2009、Spring 2010など。



今年のイベントのうち、次のことに注意してください。



  • Winter 2009のリリースでは、企業ユーザー向けのサイトの機能が実装されています。
  • 2009年春には、セルフサービスITサービスが簡素化されます。
  • 年末に、会社のスタッフは150人に増えました。


2010年：200人の従業員と400人の顧客



2010年4月現在、400人の顧客と10万人のITプロフェッショナルがService-now.comを使用して、世界中の500万人のエンドユーザーにITサービスを提供しています。



2010年春には、ビジネスプロセスを自動化し、サービスとITコストのポートフォリオを管理するための新しいアプリケーションと、人事管理のための2つの新しいSaaSアプリケーションが導入されました。



インタビューで、フレッドは次のように述べています。「クラウドプラットフォーム上の1つのサービスポートフォリオでビジネスとITの運用を管理できると考えています。」



2011：管理の変更：フランクスルトマン



会議で、Luddy は次のように述べました 。「私は毎朝起きて、やりたいことはコードを書くことだけです。」



Luddyは当初、CEOとして働くかどうか自信がありませんでした。 彼はソフトウェア開発者としてより身近な立場を望んでいました。 彼は常に市場がどの方向に発展するかを感じていたため、新製品の開発は会社を経営することよりも彼を魅了しました。



かつて、Sequoia Capitalのパートナーであり、ServiceNowの取締役会メンバーであるDouglas Leoneが、ソフトウェアを開発したいくつかのシリコンバレー企業のツアーにLuddyを連れて行きました。 ツアーで、フレッドは会社のディレクターと会い、仕事の詳細について話しました。 これにより、彼はCEOのスキルがなく、それらを取得したくないことを理解するのに役立ちました。



2011年、ServiceNowは、Data Domainの元CEOであるFrank Slootmanを会社のトップとして発表しました。 フレッドはテクニカルディレクターを引き継ぎました。 リーダーシップの変化は、会社にとってもフレッドにとっても、人生の忙しい時期である深刻なテストでした。 フランクとフレッドの間の紛争は、会社の従業員によって長い間記憶されていました。 取締役会のポールバーバーのサービスバーメンバーは、ルディがほぼ毎日電話をかけ、こう言ったと言いました。 会社を辞めたい」



フレッドは何度も会社を売却するよう申し出られました。バイヤーはServiceNowのために何億ドルも払う用意ができていました。 フレッドはすべての提案に同意する準備ができていましたが、ポール・バーバーは次の言葉で毎回彼を説得しようとしました。 取締役会は、提案の収益性にかかわらず、全会一致で会社の売却を拒否しました。 ServiceNowはあまりにも優れた投資であり、販売できませんでした。



最終的に、LuddyはSlutmanへのポストを放棄することに同意しました。 ServiceNowは成長段階にあり、Frank Slutmanは大規模な組織で作業を構築できました。 5年間の会社経営で、Slutmanは年間収益を9,300万ドルから10億ドルに増やすことに成功しました。



2011年、Ovum Researchのコンサルティング会社は、 ServiceNowをITSM業界で2年で最も高い成長率を誇る最も革新的な開発者の1つとして認識しました。



2012-2017：Billionth Corporation



この5年間で、同社は非常に成功したIPOを実施し、 地理的にもスタッフの面でも大幅に拡大しました。 急成長しているテクノロジー企業のランキングで強化されました。 2009年から2014年までの5年間のServiceNowの収益成長率は2000％を超えました。 2015年、年間利益は最終的に10億ドルを超えました。



2016年には、ServiceNowは既にGlobal 2000企業の30％以上にサービスを提供し、Knowledge16カンファレンスは同時に開催されました。



2017年2月、ServiceNow は 、eBayの前社長であるJohn Donahoeを会社の社長に任命したことを発表しました。 その後、ServiceNowの株式は一時的に4.2％減少しました。



現在、経営陣は2020年までに年間40億ドルの売上高を達成することを目指しています。 同社の目標は、ライフサイクル全体を通して顧客が製品から利益を得られるようにすることです。 John Donahoeは、会社の企業文化において、顧客のニーズを他の何よりも重視しています。 したがって、クライアントとServiceNowの助けを借りてビジネスから受ける利益は、管理に最も関心があります。



ITSMソリューションを顧客に提供することは依然としてServiceNowの主な目標であり、機械学習やその他のツールを使用すると、日常的なタスクを自動化できます。 これにより、ビジネス開発により多くの時間を割くことができます。 作業が複雑になる必要がないためです。
