Sencha Ext JS 5の紹介





SenchaとExt JSチーム全体を代表して、本日Ext JS 5をリリースすることを発表できたことを誇りに思います。ExtJS 5は大きな前進であり、フィードバックと継続的なサポートに感謝します。 これは私たちの歴史上最大のベータであり、10万ダウンロードを超えました。 一緒になって、世界で最も先進的なマルチデバイスJavaScriptフレームワークを作成しました。 最高のExt JSフレームワークを作成するために、皆さんは計り知れない貢献をしました。



新機能



タッチサポート


Ext JS 5は、フレームワークの最も広範なアップデートです。 このリリースでは、単一のコードベースを使用して、タッチデバイスでデスクトップアプリケーションを実行する機能が提供されます。



アーキテクチャの改善


MVVMアーキテクチャと双方向データバインディングを使用して、アプリケーションの作成がこれまで以上に簡単になりました。 クライアント側では、サーバーアクセスの回数を減らすためにデータセッション管理を統合しました。 ルーティングを入力すると、アプリケーションにディープリンクを作成できます。 これらすべての改善により、アプリケーション開発はさらに効率的になります。



レスポンシブマークアップ


本日から、新しいレスポンシブレイアウトシステムのおかげで、アプリケーションは、向きの変更に関係なく、デスクトップとタブレットの両方で最適な外観を持つ機会があります。



コンポーネントの更新


アプリケーションが成長するにつれて、Senchaはコンポーネントライブラリを改善し続けます。 Ext JS 5を使用すると、ウィジェットをテーブルセルに追加して、ビッグデータの視覚化と分析を行うことができます。 財務機能を備えたグラフ作成用のタッチに最適化されたコンポーネントのセットも登場しました。

Ext JS 5の詳細







デスクトップとタッチスクリーン間の統一されたイベント


Ext JS 5は、タッチスクリーンを備えたデスクトップおよびデバイスで1つのコードを使用できるようにする最初のバージョンです。 Sencha Touchのジェスチャシステムを使用して、ピンチ、ズーム、スワイプ、タップ、長押しなどのジェスチャのサポートを導入しました。これにより、タッチスクリーンでデスクトップアプリケーションをほとんどまたはまったく変更せずに使用できます。 委任されたイベントシステムを使用して、タッチデバイスのイベントのマウスクリックをマップしました。



MVVMおよび双方向バインディング


Ext JS 4では、MVCアーキテクチャを導入しました。 また、Ext JS 5-MVVM（モデル-ビュー-ViewModel）では、双方向のデータバインディングを提供し、ビジネスロジックとビュー間の責任の共有を改善します。 MVVMを使用すると、ビューからモデルへ、またはその逆に変更を自動的に転送できます。 ビューとモデル間のこの活発な自動同期により、開発時間とエラーが削減されます。 計算される値とフィールドも追加し、同期前のオブジェクトのビューへのバインドを事前に変換できるようにしました。 新しいMVVMアーキテクチャが役立つことを願っています。







Ext JS 5のMVVMに加えて、ViewControllersを導入することでMVCサポートも改善しました。 ViewControllersは特定のビューを特に対象としており、オブジェクトの管理とアプリケーションの状態の復元に関して、従来のMVCと比較して不要なコストを削減します。 現在、すべてのビューイベントをリッスンする通常のコントローラーを回避することにより、アプリケーションはメモリ使用量を削減し、パフォーマンスを向上させることができます。



レスポンシブマークアップ


Ext JS 5は、デスクトップとタッチスクリーン間のアプリケーションとナビゲーションの外観を改善することを目的とした新しいレスポンシブマークアッププラグインを導入します。 向きが変わる（デバイスが回転する）場合、要素の位置または配置を変更する必要があるアプリケーションは、新しい構成システムを使用してこれを自動的に行うことができます。 



responsiveConfig: { landscape: { region: 'west' }, portrait: { region: 'north' } }


または 

 responsiveConfig: { 'width > 800': { region: 'west' }, 'width <= 800': { region: 'north' } }








動作を確認するには、ブラウザ画面のサイズを変更するか、タッチスクリーンを回してこの例を試してください。



ネプチューンタッチとクリスプタッチ


UIコンポーネントの豊富なコレクションがデスクトップとタッチスクリーンで同じように見えるようにする2つの新しいマルチデバイステーマがあります。 既存のNeptuneテーマと新しいCrispテーマのタッチバージョンでは、拡大された要素を使用して、タッチスクリーンでの作業に必要な大きなコンタクトスポットをサポートします。









デスクトップとタッチスクリーンでテーマのタッチバージョンを使用できますが、デスクトップにはNeptuneとCrisp、タッチスクリーンにはNeptune TouchとCrisp Touchを使用することをお勧めします。



テーブルの機能強化


グリッドは、Ext JSで最も人気のあるコンポーネントであり、バージョン5ではさらに強力になりました。 グリッドウィジェット（ Grid Widgets ）の導入により、新しい機能-Buffered Updatesと組み合わせることができ、リッチデータとリアルタイムの更新を視覚化してビッグデータ分析をサポートできるようになりました。







セッション管理


ExtJS 5の最もエキサイティングな機能の1つはセッション管理です。トランザクションセッションが追加されました。 Ext JS 4を使用してサーバーにレコードを保存した場合、それを実行した順序に注意する必要がありました。 変更された各レコードとストレージに対してsaveメソッドを呼び出したことを確認する必要がありました。 このすべての「簿記」により、データ一致エラーが発生しやすくなりました。



Ext JS 5では、これらのアクションを操作するための新しいクラス[Ext.data.Session]が導入されています。 セッションは、サーバー上で変更、作成、または削除する必要があるエントリを追跡します。 新しいレコードがこのトランザクションの他のレコードを正しく参照するように、これらの操作をソートすることもできます。 セッションを使用すると、レコードとその関連付けを編集し、すべての変更を収集してサーバーに送信することが容易になりました。 そして以来 セッションはすべての変更を保存します。トランザクションが正常に完了したかどうかに応じて、すぐに取得することも、単一の変更を取得することもできません。



ルーティング


Ext JS 5では、URLルートがコントローラーメソッドに変換されるときに、アプリケーションでディープリンクを作成できます。 ルーティングを使用すると、サーバー上の物理ファイルを必ずしも参照しないURLを受け入れるようにアプリケーションを構成できます。 ＃で始まるURLの部分（フラグメント識別子として使用）は、ページをリロードせずにブラウザー履歴スタックへの道を開きます。 ハッシュが変更されると、ブラウザは「hashchange」イベントを発生させます。これは、クライアントアプリケーションが単独で使用できます。



タッチに最適化されたグラフィックス


Ext JS 4は、SVGおよびVMLに基づくプラグインなしのグラフ作成システムを提供しました。 ただし、このシステムはモバイルデバイスでは最適ではなく、メモリが限られているため、Canvasは高速でSVGは低速であることがわかりました。 Ext JS 5では、Sencha Touchの同様のシステムに基づいて再構築しました。これは、少量のメモリを搭載したデバイスで指を操作するために最適化されています。 新しいチャートシステムには、ズームサポートを含む、財務チャート、多次元チャート、3Dチャートなどの機能が追加されました。 パフォーマンスが向上し、拡張性が高く、タッチスクリーンで優れた機能を備えています。







Kitchen Sinkなどのさまざまなタイプのグラフを操作するためのサンプルアプリケーションも作成しました。 Ext JS 4の既存のグラフィックスは引き続き個別に利用できるため、それらも使用できることに注意してください。 ただし、タブレットに焦点を合わせている場合は、新しいチャートシステムにアップグレードする必要があります。 Ext JS 5への移行の詳細な手順は、 ドキュメントに記載されています 。



Senchaコアパッケージ


Ext JS 5の最大の課題の1つは、Ext JSとSencha Touchコアを可能な限り近づけることでした。 クラスシステム、データ処理パッケージ、および関数/環境検出のコードを一貫して削減したため、このコードはExt JSとSencha Touchの両方で使用できます。 この共通プラットフォームを使用すると、フレームワークに関係なく、デスクトップ、タブレット、その他のタッチスクリーンで共有リソースを使用してアプリケーションをレンダリングできます。 現在、この共通コアに基づいたSencha Touchリリースに取り組んでいます。 お楽しみに。



手順とドキュメント


Ext JS 5では、新機能と改善のために大量の新しい命令を追加しただけでなく、命令システム自体を更新して使いやすくしました。 これらの手順は、Sencha Cmd、グラフィックス、アプリケーションアーキテクチャなど、新しいフレームワークへのアップグレードに関するすべてをカバーしています。



Ext JS 5のいくつかの重要な機能についてのみ言及しましたが、まだ多くの話題があります。 Ext JS 5の新機能マニュアルで、残りの新機能と改善点を探求します。



サポートされているブラウザーとプラットフォーム



おそらく、Ext JS 5で変更されていない母音の1つは、VIPクライアント（ 一流の市民として-およそTransl。 ）としてIE8を使い続けていることです。 Ext JS 5は、Safari 6以上、Firefox、IE8以上、Chrome、およびOpera 12以上をサポートしています。 モバイルプラットフォーム、iOS 6および7のSafari、Android 4.1+のChrome、およびWindows 8タッチスクリーン（Surfaceやその他のタッチスクリーンラップトップなど）-IE10 +がサポートされています。







Ext JS 5トレーニング



また、Sencha トレーニングプログラムも更新しました。 Ext JS 5トレーニングは6月に開始され、これには次のクラスが含まれます：Ext JS 5アップデート、Ext JS 5へのアップグレード、Ext Track to Ext JS 5。



Ext JS 5の価格



このリリースから2014年6月30日まで、Ext JS 5のライセンスは現在のExt JS 4.xの価格で利用できます。 これは現在の価格でExt JS 5を購入する絶好の機会です。 7月1日から、新しい価格が施行されます。 詳細については、 Ext JS 5製品ページにアクセスするか、お問い合わせください 。
