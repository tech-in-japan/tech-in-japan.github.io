Skype 4.0：無料

これは、Skype 4.0 for Windowsでの作業方法に関する一連のSkypeブログ投稿の2番目です。 最初の部分の翻訳はこちらです。



最初のステップ

2008年6月18日は、Skype 4.0 for Windowsが初めて一般公開された日であり、Skypeの完全に新しいパブリックベータテストプロセスが開始されました。 以前のバージョンに取り組んでいたとき、1つのベータ版を修正し、バグを修正し、1〜2か月後に「ゴールデン」バージョンをリリースしました。

今回は、エラーに関する情報を収集するだけでなく、インターフェースに関するフィードバックを得るためにもパブリックベータを使用することにしました。 さらに、経験豊富なユーザーだけでなく、このベータ版のテストに参加することを意図していました。そのユーザーは、その瞬間までベータテスターの大半を占めていました。 したがって、このブログでSkype 4.0 for Windowsの3つのベータ版すべてについて多くのことを伝えることに加えて、新しいユーザーや経験の少ないユーザーにも積極的に提供しました。

これらの新しいユーザーの多くにとって、電話をかけるためにコンピューターを使用するという考えは非常に珍しいように思われることを忘れないでください。以前に使用したことも、ユーザーベース全体をよりよく表しています。



営業日

最初のベータ版に関するレビューはまちまちでしたが、これが私たちにとって驚きだったとは言えません 。 アプリケーションウィンドウのデザインに加えられた変更は、多くの人にイライラを引き起こしました。これは、今では「コンパクトモード」と呼ばれる古いインターフェイスでユーザーが何を好むかを理解するために引き起こしたい反応です。 Skypeがあなたのフィードバックの収集と処理にオープンにアプローチしたのはこれが初めてです。 Skype製品戦略ディレクターのマイク・バートレットが最近のブログ投稿で次のように述べているように：

ユーザーが異なれば、レビューも大きく異なります。 たとえば、Skypeフォーラムでは、経験豊富なユーザーの多くが連絡を取り合っていますが、特にウィンドウサイズに関して多くの否定的な問題に直面しています。 ただし、私たちが実施した調査では反対の結果が示されています。回答者の70％以上が、古いインターフェースよりも最初のベータインターフェースを好むと答えています。

最初のベータ版がリリースされて以来、世論調査、バグ報告、および数千のブログ、フォーラム、ツイートなどを通じて、合計50,000件のレビューを受けています。 デザイナーと研究者で構成される私たちの経験チームは、あなたのコメントを研究するのに何時間も費やしました。

また、ユーザビリティチームは、Skypeユーザーと個人的に会い、Skypeに加えることができる変更と改善について話し合い、合計350を超えるこのようなセッションを世界中で行いました。



Spotlight 4.0

Skype 4.0に取り組むときは、優れた通話品質、ビデオ品質の向上、使いやすさの3つの主な目標に集中することにしました。 これらの優先度に基づいて、定量的評価を使用して特定のリクエストの優先度を決定し、レビューの操作方法に関する戦略的決定を行いました。

2番目のベータ版の準備作業は2008年の夏に開始され、10月1日にリリースされました。 新しいインターフェイスについて苦情を申し立てた人々の圧倒的多数は、「コンパクトモード」に切り替える機能の導入を満足させ、通知システムの改善により、着信コールと不在着信およびテキストメッセージを区別しやすくなりました。



改善

2番目のベータのリリース後、3番目のベータをより迅速に準備することができました。この機会を利用して、完全なメッセージ履歴（履歴）などのいくつかの欠落機能を追加しました。最終バージョン。

これらは、過去2年間にわたってSkypeによって独自に開発されており、革新へのコミットメントを示しています。 Skype 4.0 for Windowsの最終バージョンがダウンロード用にサーバーにアップロードされると（オリジナルはQuartetの前夜に公開されました）、これはプログラムの歴史の5年間で最も顕著なSkypeリリースになります。 しかし、これは開発が少なくとも1秒間停止するという意味ではありません。 すでに次のバージョンに取り組んでいます。
