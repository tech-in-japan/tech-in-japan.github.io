クラウドデータゲートウェイ：EMC CloudArrayおよびCloudBoost

今日、データストレージの分野では、ITサービスは2つの大きな課題に直面しています。それは、情報量の雪崩のような成長と、それをより長く保存する必要性です。 これらの2つの要因により、既存のデータセンター、スタッフ、およびIT予算が大幅に削減されます。 企業は、ローカルインフラストラクチャへの投資を維持し、同時に、それが何であれ、さらなる成長のための安全マージンを提供するのに役立つ革新的なソリューションを探すことができるだけです。 パブリッククラウドとプライベートクラウドのアクセスが容易になると、小規模企業でも、ボリュームの増加に直面してストレージ管理を簡素化および簡素化するスケーラブルなストレージオプションを幅広く購入できます。 ほとんどの企業では、パフォーマンス、セキュリティ、または複雑なアーキテクチャ要件を満たすことができるのはローカルデータセンターのみです。 ただし、アーカイブやバックアップなどの重要度の低いデータは、安価であると同時に、常にアクセス可能なパブリックまたはプライベートクラウドに転送できます。







EMC CloudArray-統合クラウドストレージ







データストレージタスクはますます難しくなっています。 データ量の爆発的な増加に加えて、ITマネージャは、政府規制の強化、パフォーマンス要件の増加、ITの消費化によって圧力を受けています。 誰もがすでに限られた予算内でITサービスに期待しています。



これに対応して、多くのITサービスはデータをクラウドに移行して、ビジネス要件を満たし、同時にお金を節約します。 ただし、クラウドストレージへの移行は、パフォーマンス、統合、セキュリティの点で事態を複雑にするだけです。 EMC統合クラウドストレージを使用すれば、これらすべてを恐れることなく行えます。



CLOUDARRAYレビュー



CloudArrayを使用すると、企業はローカルインフラストラクチャに稼働中のITシステムのみを残し、第2レベルのデータをパブリックまたはプライベートクラウドストレージに転送できます。 CloudArrayは、アプリケーションにSAN（iSCSI）および/またはNAS（NFS / CIFS / SMB / SMB2）ボリュームを提供します。 迅速なアクセスを提供するため、CloudArrayはすべてのデータをローカルにキャッシュし、パブリッククラウドまたはプライベートクラウドに自動的に複製します。 このアーキテクチャを使用すると、通常はクラウドストレージサービスと統合するときに発生するため、APIの統合と長いトレーニングに時間を費やす必要はありません。 最小限のダウンタイムと時間でクラウドへの既存のインフラストラクチャの接続を大幅に簡素化します。



CloudArrayは、SANおよびNASを介したオブジェクトストレージへのアクセスを提供するだけでなく、クラウドストレージの展開時に必然的に発生する幅広いタスクを解決します。 このデバイスは、クラウド間のセキュリティ、パフォーマンス、ネットワーク帯域幅、データのインポートと削減、データ保護と移行を提供します。



プロトコル管理



CloudArrayは、EMC ViPR、VMware vCloud Air、Rackspace、Amazon Web Services、Google Cloud、EMC Atmos、Openstackなど、20を超えるプライベートおよびパブリッククラウドストレージプラットフォームのAPIを直接サポートしています。 CloudArrayの外部インターフェイスは、クラウドベースのオブジェクト指向ストレージリソースを、iSCSI、NFS、CIFS、SMB、SMB2などの使い慣れたプロトコルを使用して既存のサーバーに接続できるローカルボリュームに変換します。 iSCSIが使用されている場合、CloudArrayはサーバーに未フォーマットのブロックボリュームを提供し、そこでそれらをマウントおよびフォーマットできます。 ファイルプロトコルを使用するシステムの場合、CloudArrayは標準のネットワークフォルダーを導入します。 CloudArrayインスタンスごとに、独自のプロトコル（SANまたはNAS）、サーバーが必要とするパフォーマンスと可用性のレベルを備えた多くの異なるボリュームを作成できます。



ローカルレベルのパフォーマンス



CloudArrayは、保存されているデータのタイプに一致する適切なレベルの可用性とパフォーマンスを提供する、独自のルールベースのキャッシング機能を提供します。 ローカルデバイスのキャッシュはパフォーマンスを提供すると同時に、クラウドに非同期的にデータを複製します。 キャッシュごとに、適切な構成を定義できます。 その後、ポリシーを割り当てて、現在のニーズに応じて必要な割合のデータを保存するようにします。 クラウドを使用してアクティブデータをリモートサイトに保存する場合は、データの100％がローカルとクラウドの両方に保存されるようにキャッシュを構成します。 データをアーカイブする必要がある場合は、1％増やします。 両方のオプションが必要な場合は、複数のキャッシュインスタンスを作成し、必要なパフォーマンスに応じてボリュームを移動します。



セキュリティと暗号化



CloudArrayは、動的暗号化と保存データの暗号化を使用して、情報のセキュリティを保証します。 クラウドに複製する前に、各ボリュームは256ビットAES暗号化を使用して暗号化されます。 さらに、CloudArrayはデータとメタデータを別々に暗号化し、それらの異なる暗号化キーを顧客にローカルに保存します。 これにより、ハッカーもクラウドストレージプロバイダーもデータへの不正アクセスを取得できなくなります。



コンパクトなスナップショット



CloudArrayのスナップショット機能は、データを長期にわたって保存するための高度な計画と機能を提供します。 これにより、組織はクラウド内のアーカイブデータに追加の保護を提供できます。 従来のフルバックアップと比較して、CloudArrayスナップショットは20分の1のスペースしか占有しません。 これにより、ストレージ要件とストレージコストが削減されるとともに、リモートクラウドサイトで継続的なデータ保護が提供されます。



帯域幅管理とデータ削減



多くのITサービスは、クラウドストレージに移行すると、ネットワークの全体的なパフォーマンスが低下することを恐れています。 CloudArrayのデータ削減テクノロジーと帯域幅の微調整機能により、ネットワーク負荷が大幅に削減されます。 さらに、CloudArrayはクラウドスペースを節約することでコストをさらに削減します。 特に、ネットワーク負荷が高い日と時間を考慮した週次スケジュールを作成できます。 したがって、ネットワークの可用性とデータ複製の待ち時間の最適なバランスを実現できます。



リカバリークラウドアレイ



CloudArrayは、障害が発生した場合にITサービスがデータにすばやくアクセスできるように、幅広い災害復旧ツールを提供します。 CloudArrayの仮想マシンとしての機能は、事故やハードウェア障害が発生した場合、管理者がシステムの新しいインスタンスを数分でどこにでも展開できることを意味します：予備の機器、補助サイト、またはクラウドですら。 管理者はCloudArrayポータルに仮想マシンをアップロードできます。 インストール中に、システムは管理者に、CloudArrayポータルに保存されている構成ファイルのバックアップコピーから既存の設定を復元するよう求めます。 数分以内に、アクセスできないCloudArray仮想マシンの完全に機能するレプリカが、すべてのボリューム、機能、設定とともに復元されます。 クラウドに復元するとき、オンプレミスとクラウドアプリケーションの両方とやり取りできる特別なCloudArrayイメージがデプロイされます。 CloudArrayをクラウドベースの災害復旧戦略の一部として展開する場合、企業は別個の災害復旧サイトを維持するためのコストを大幅に節約できます。



CloudArrayを使用すると、クラウドストレージはオンプレミスのように見え、機能します。 CloudArrayの幅広い機能により、顧客は低コスト、低メンテナンスのクラウドリソースで高性能とセキュリティの要件を満たすことができます。



EMC CloudBoost-クラウド内の長期バックアップストレージ向けの業界をリードするテクノロジー







CloudBoostは、クラウド機能を拡張するためのEMCソリューションです



長期バックアップストレージ用のプライベートクラウドおよびパブリッククラウドへの安全で自動かつ効率的なデータ転送を提供します。



既存のソリューションを簡単に使用して、柔軟でフォールトトレラントで水平方向にスケーラブルなクラウドストレージでデータを保護できます。



物理的および仮想的なフォームファクターがあります。



ほとんどのITサービスは、数年間定期的に作成されたバックアップを保持する必要があります。 長期保存を目的としたバックアップの必要性はまれにしか発生しませんが、IT部門は完全または増分リカバリのためにそれらをオンデマンドで取得できる必要があります。 同時に、IT専門家は、テープソリューションに関連する無数のリスクを排除し、長期保管に使用する際に発生する大きな運用コストを管理する必要があります。



長期保存用にバックアップする方法



CloudBoostを使用すると、EMC Data Protection Suiteユーザーは、テープの使用に伴うリスクとコストを回避できるだけでなく、コストを削減し、クラウドベースの応答性を実現できます。 これを行うには、テープ上のストレージシステムを、長期バックアップストレージ用のパブリック、プライベート、またはハイブリッドクラウドストレージシステムに置き換えるだけで十分です。 EMC NetWorker、CloudBoost、EMC Data Domain、およびEMC Elastic Cloud Storage（ECS）などのエンタープライズクラスのオブジェクトストレージを組み合わせることで、運用データと長期ストレージのデータをバックアップするためのすべての要件を満たすことができる、業界をリードする統合ソリューションを作成できます。 CloudBoostは、長期バックアップストレージのセキュリティ、効率、高性能、および費用対効果を提供します。



CLOUDBOOSTアーキテクチャ



CloudBoostは、仮想および物理の両方のフォームファクターで利用できます。 すべてのモデルの機能と利点は似ています。 すべてが、プライベートクラウドとパブリッククラウドへの安全で効率的、高性能、費用対効果の高い接続を提供します。 CloudBoostの各インスタンスは、6バイトのクラウドでデータを管理できます。 データがメタデータから分離されている階層化アーキテクチャは、広範なオブジェクトストアの組み込みサポートを提供します。 CloudBoostは、多くのプライベートクラウドプラットフォーム（EMC Elastic Cloud Storage（ECS）、EMC Atmos、およびOpenStack Swift）、およびパブリッククラウドプラットフォーム（AT＆T Synaptic、Amazon Web Services S3、Google Cloud Storage（Nearlineを含む）、Microsoft Azure）をサポートしています。







安全性



CloudBoostは、データがファイアウォールの外側に保存または送信される場合でも、エンタープライズクラスのセキュリティを提供します。 このソリューションは、各ファイルを多数の小さな「フラグメント」に分割し、そのような各フラグメントを独自の独立したAES-256キーを使用して暗号化します。 動的データと保存データは断片化および暗号化されたままであり、TLSプロトコルを使用してそれらを送信します。



リモートオフィスおよびブランチ向けの理想的なソリューション



WAN環境では、帯域幅、待ち時間、または信頼性が制限されているという問題がしばしば発生します。 このため、パブリックまたは中央に配置されたプライベートクラウドでデータストレージのクラウド層を使用することは実用的ではない場合があります。 CloudBoostは、グローバルネットワークのソース側のデータ重複排除、圧縮、および最適化を提供します。これにより、パフォーマンスがさらに向上し、リソース消費とネットワーク帯域幅とクラウド容量のコストが削減されます。 最大32 TBの容量を持つデータのローカルキャッシュにより、長期間保存されたデータのバックアップとリカバリを高速化できます。 同時に、グローバルネットワークが利用できない場合でも、ローカルネットワーク経由でキャッシュに格納されたデータへのアクセスを取得できます。



CLOUDBOOSTの利点



-クラウドは、長期保管の分野でコスト効率の高いテープの代替手段です。



-テープと比較して、データの損失または損傷のリスクが最小限。



-クラウドサポートにより、データ保護への投資をより効果的に活用できます。



-EMC NetWorker with CloudBoostは、受賞歴のあるデータ保護ソフトウェアと、業界で最も安全で費用対効果の高いクラウドサポートテクノロジーの組み合わせです。



-長期保存のために、バックアップデータをパブリックまたはプライベートクラウドストレージに安全かつ効率的に転送。



-テープの使用に関連するリスクを排除し、資本コストを削減します。



-オンライン生産性を備えた費用対効果の高いテープ。
