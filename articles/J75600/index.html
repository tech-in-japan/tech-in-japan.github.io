Skippy-手間をかけずに公開（つまり、Compiz / Beryl）

最近、私は自分自身に尋ねました-CompizのようにLinuxでExposeの類似物を作ることは可能ですか？



それはあなたができることが判明しました！ SkippyとExpocityの2つのオプションを見つけました。Skippyを試してみることにしましたが、Expocityには到達しませんでした。 完全に満足しました。







Skippyの説明、ファイル、スクリーンショットはここにあります：



thegraveyard.org/skippy.php

thegraveyard.org/files



Xサーバー拡張機能XDamage、XComposite、XFixesを使用するSkippy-XDのバージョンをお勧めします。これは、通常のバージョンよりもはるかに高速で優れた動作をします。



Debian Lennyにはパッケージがないため、ソースからコンパイルされました。プログラムのサイズが小さいため非常に高速でした。



# make

gcc -I/usr/X11R6/include `pkg-config xft xrender xcomposite xdamage xfixes --cflags` -g -pedantic -Wall -DXINERAMA -o skippy-xd skippy.c wm.c dlist.c mainwin.c clientwin.c layout.c focus.c config.c tooltip.c -L/usr/X11R6/lib -lX11 -lm `pkg-config xft xrender xcomposite xdamage xfixes --libs` -lXext -lXinerama



# make install

install -d /usr/local/bin

install -m 755 skippy-xd /usr/local/bin/skippy-xd







その後、標準構成ファイルをプロファイルにコピーし、skippyがウィンドウを作成するキーを変更するだけです。



# cp ./skippy-xd.rc-default /home/USER/.skippy-xd.rc



# grep keysym /home/USER/.skippy-xd.rc

# - keysym can be anything XStringToKeysym can handle

# (like F11, KP_Enter or implementation specific keysyms)

keysym = Scroll_Lock







正しいキー名は、xevユーティリティを使用して見つけることができます。



実際、これですべてです-skippy-xdを環境のスタートアップに入れて、結果を楽しむだけです！



喜ばしいこと-skippyは2台のモニターで正常に動作し、現在のモニターと一度に（Shift +ホットキー）の両方からウィンドウを構築できます。 XDバージョンの場合、ウィンドウビューモードでは、XDバージョンで発生したすべての変更が表示されることに注意してください。
