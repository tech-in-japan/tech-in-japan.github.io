System Center 2012を使用してプライベートクラウドを作成および管理するためのすべての主要なシナリオをカバーするような方法でプログラムを構築しようとしました。 まず、2日前にダウンロードできるベータ版の Windows Server "8"の技術レビューから始めましょう。 NICチーミング、クラスター構造のないライブマイグレーション、Hyper-Vレプリカなど、新しいHyper-Vの機能に焦点を当てます。



次に、簡単に、そして狂信なしで、Microsoftがプライベートクラウドによって意味すること、およびSystem Center 2012が言われたことの実装にどのように役立つかについて説明します。

  • VMM 2012がインフラストラクチャコンポーネント（サーバー、クラスター、ネットワーク、ストレージ）を管理する方法と、新しいノードのベアメタル展開がその助けを借りて実装される方法。
  • VMM 2012で上記の利用可能なデータセンターリソースをプライベートクラウドの形式でグループ化し、それらへのアクセスを区別する方法、さまざまなサプライヤー（Microsoft、VMware、Citrix）のハイパーバイザーとの連携方法。
  • Service Manager 2012の助けを借りて、データセンターリソースの割り当てとアプリケーションの起動のためにアプリケーションを整理および標準化する方法、ユーザーの外観、Orchestrator 2012がそのようなプロセスを自動化する方法。
  • VMM 2012がサービステンプレートを使用してアプリケーションを展開する方法、そのようなアプリケーションが更新される方法。
  • 最後に、Operations Manager 2012がプロアクティブな監視にどのように役立つか。ネットワーク機器から特定のアプリケーションまで。 OrchestratorおよびSystem Managerと連携して、問題を自動的に解決したり、インシデントを作成したりして、ITスペシャリストを引き付けることができます。
