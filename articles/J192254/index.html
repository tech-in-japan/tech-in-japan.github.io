このゲームとは何ですか？



私は10年前にShadow Worldsにつまずいたが、それは私を完全に捕らえた。 ひどいモデム接続で友達と遊んだり、ネズミやゾンビをびしょぬれにしたり、オークから逃げ出したり、チェストを開けたり、ボスの上で勝利したりして歓喜し、「タフなゲーム！」と叫びました。



彼女の何が特別なのか説明するのは難しい。 そのときでさえ、それはかなり原始的なグラフィックスを持ち、ほとんど変化がありませんでしたが、何かに惹かれました。 それはアヒルの症候群にあるかもしれない。



それ以来、たくさんの水が流れましたが、時々ゲームクライアントをダウンロードし、新しいアカウントを作成して疲れるまでプレイしました。その後、ネジからゲームの痕跡をすべて削除しました。 何年も変わっていないことを確信するたびに、同じ場所、同じモンスター、グローバルチャットで同じ宣誓、新しいことは何もありませんが、バランスは上下します。



1か月前、古いメールボックスに大量のスパムをすくい、SWチームからの手紙を見つけました。 最近、新しいカード、アイテム、その他の設備を備えた新しいサーバーが発売されたと述べました。 次回は懐かしさに突入するようになったトリガーが私の脳内でオフになった。

登録し、クライアントをダウンロードして、ゲームに参加しました。



すべての新しいプレーヤーが最初に目にするもの。




ダンジョンでネズミを殺し、墓地でゾンビを殺し、お金を集め、漁網を買うことができます。



再び若く感じ、十分なプレイをした、日曜日の夕方近く、私は好奇心を抱いていました。ゲームはどのようにサーバーと通信するのですか？ 実際、インターフェースの原始性（以下で説明します）と長年にわたる進歩の欠如から判断すると、使用されるプロトコルの単純さを期待できます。 Wiresharkを起動し、ログのおlogsをバイトから見ましたが、意味のあるものは何もありませんでした。 さて、内部から見ていきましょう。



IDA、DeDe、OllyDBGのゲームのメインehe-shnikであるMUDClient.exeを開きます。 IDAでは、Ollyではクールに、DeDeではデバッガーを見つけない方がいいので、ゲームはBorland C ++ Builderで書かれているためです。

すべてがバタンと行く、元はパックされておらず、トレッドによって上から保護されていません。



DeDeでは、奇妙なプロシージャ名が表示されます。これにより、多くの有用な情報を迅速に取得できる（保護なし）という自信が得られます。



Ollyには小さな問題があります。事実は、起動時にゲームが毎回更新のチェックを開始し、ダウンロードして再起動することです。





したがって、デバッガでMUDClient.exeを実行すると、すぐに終了し、代わりにrun.exeが起動し、管理者特権を要求することもあり、管理者パスワードを入力する必要があります。 わかった。



OllyDbg run.exeを開き、CreateProcessWにブレークポイントを設定します。 これを取得します。



マジックパラメーターはE8DA81A3FFE9B1です。 何らかのチェックサムなのか、単なる秘密なのかはわかりませんが、batファイルを作成して作成します 

start MUDClient.exe E8DA81A3FFE9B1


ゲームが開始され、管理者パスワードの入力は不要で、更新は試行されません。 勝利



これで、Ollyでコマンドライン引数E8DA81A3FFE9B1を指定できるようになり、ゲームは更新の確認が不要であると判断し、デバッグを安全に進めることができます。 そして、単にプレイしたいときは、このバットファイルを実行する方が便利です。 重要な更新を見逃すという事実を心配する価値はありません。すべて同じで、新しい実行可能ファイルはありません。ゲームは週に1回更新されます（このサーバーの機能）が、更新にはゲームの場所の新しいファイルのみが含まれます。
