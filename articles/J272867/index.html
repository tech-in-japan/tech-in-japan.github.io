Alcatel-Lucent 7210 SAS-Mのハイブリッドポート構成

最近、激しい競争の状況でわが国で事業を展開している事業は、最終的にクライアントにすべての結果をもたらしました。 「顧客重視」という言葉と、ロシアの資本家の言葉で新しくなった私たちの組織は、迂回しませんでした。 クライアントに最高品質と信頼性の高いサービスを提供することを任されました。 ネットワークの最適化とともに、重要な分野のバックアップチャネルを開発するプロジェクトが開かれました。



バックボーンチャネルの大部分はFOCLに基づいて長い間実装されており、これらのセクションのバックアップチャネルは、メインセクションで事故が発生した場合に、最愛のクライアントが何も感じないように適切な容量でなければなりません。 州の独占企業（ロシア鉄道、ガスプロムなど）から技術仕様を入手できないため、一部の地域で光ファイバーリンクを構築するのは単純に非現実的です。または、スペースがかかります。 RRLは、5 kmの距離で1Gbpsの容量のチャネルを提供します。



IP / MPLSバックボーンネットワークはAlcatel-Lucent機器（7750SRおよび7210 SAS-M）に実装されており、この機器はFRRテクノロジーとBFDプロトコルに基づいてトラフィックルートを切り替えることができます。 スイッチング時間は50ミリ秒未満であり、これはパケットデータサービスの継続的な操作性を維持するのに十分な時間です。 その結果、バックアップスキームは次の形式を取りました。







RRL NEC iPasolink E-BANDは完全に屋外設計であり、その管理は2つの方法で編成できます。



1.アウトバンド。 この場合、アンテナサポートのアクティブユニットから、個別のUTPケーブルを下げて、ハードウェアの一部の機器に含める必要があります（同じアルカテル7210でも可能です）。

2.インバンド。 制御チャネルは、トラフィックチャネルがある1つの物理環境で起動できます。 既存のファイバーを使用して上昇します。



方法2は、より安く、より信頼性が高く、使いやすいので、選択は中止されました。 そして、ここで私たちは次の問題に直面しています。 自動予約のアイデアを実現し、FRRおよびBFDテクノロジーを最大限に活用するには、RRLに向かうMBH1およびMBH2のポートを「ネットワーク」モードで構成する必要があります。これにより、MPLS Alcatel-Lucentネットワーク用のRRL制御トラフィックを処理できなくなります。また、そのために（トラフィック）ポートを「アクセス」モードで構成する必要があります。

使用されているTiMOS-B-5.0.R9 OSのバージョンでは、ハイブリッドポートの実装が許可されていないため、考えなければなりませんでした。 次のスキームが思い浮かびました：







MBH1スイッチの例に関する図へのいくつかの説明。 イーサネットフレーム上のRRL「ハング」タグ299を介して編成されたバットネットワークのインターフェイスすべてのMPLSチップはイーサネットフレーム内にカプセル化されるため、VLANサポート（IEEE 802.1Q）でL2レベルで動作するパケットRRLはこれらのフレームを切り替えて転送しますリモート端末は、無線リンクから光リンクにフレームを送信します。 同時に、別のタグVLAN 22を持つIN-BAND RRL制御チャネルは引き続き光リンクに切り替えられるため、2つの仮想ネットワークは同じ物理ポートMBH1 p1で終端されます。 ポートはアクセスモードで構成されます。 



port 1/1/1 description "1GE-1/1/1-to-NEC_E_BAND" ethernet mode access access exit encap-type dot1q mtu 9212 ssm no shutdown exit exit no shutdown exit


2つのサービス、MPLSのVPLSおよび管理NECのVPLSは、7210で構成されます。 各サービスにはサービスアクセスポイント（文字通りサービスアクセスポイント）があり、VLAN 299およびVLAN 22タグに基づいて、それぞれが独自のVPLSになります（VPLSは基本的にボード7210で編成された仮想スイッチです）。 sap 1/1/2を介したMPLSのVPLS：299は、MPLSフィールドを持つフレームを、インターフェイスinterface "if-MBH2"に関連付けられているp23スイッチポートに「渡し」ます。 ポート1/1/2はアクセスモードで構成され、ポート1/1/23は既にネットワークモードで構成されています。 



 port 1/1/2 description "1GE-1/1/2-to-1GE-1/1/23" ethernet mode access access exit encap-type dot1q mtu 9212 exit no shutdown exit port 1/1/23 description "1GE-1/1/23-to-1/1/2" ethernet encap-type dot1q network exit ssm no shutdown exit exit no shutdown exit


NEC RRL管理サービスは、同じスイッチ上の任意のsapから引き出すことができます。または、管理ゲートウェイへのスポークsdpサービストンネルを使用してMPLSネットワーク経由で起動できます。



MBH1の構成を追加します。



1.インターフェース： 



 router interface "if-MBH2" address 192.168.1.33/30 port 1/1/23:299 bfd 100 receive 100 multiplier 3 exit


この場合のBFDの設定は、RRLを介して組織されたネットワークのインターフェイスが特定の時間間隔後に相互に質問され、RRLを介した通信がない場合、バックアップルートへのトラフィック移行が開始されるように必要です。 この場合のFRRテクノロジーは正しく機能しません。 RRLリンクに違反すると（たとえば、氷山が古いチューブテレビのサイズのパイプから落下し、パス内のすべてを破損する場合があります）、スイッチのネットワークポートに接続されているポートがアップし、スイッチ間のリンクが切断されます。 しかし、これについては次の記事で詳しく説明します。



2. VPLSサービス（私の場合、epipeが構成されています）： 



  epipe 9024 customer 7 svc-sap-type any create description "Gibrid_Port_7210_MPLS" service-mtu 9194 service-name "Gibrid_Port_7210" sap 1/1/1:299 create description "MPLS_over_E-BAND" exit sap 1/1/2:299 create description "MPLS_to_7210_MBH1" exit no shutdown exit epipe 9025 customer 7 svc-sap-type any create description "Gibrid_Port_7210_NEC_MGMT" service-mtu 9160 sap 1/1/1:22 create description "NEC_MGMT_IN" exit sap 1/1/3:22 create description "NEC_MGMT_OUT" exit no shutdown exit


原則として、すべて。 コメントと批判を待っています。
