フラットアセンブラを使用したWindowsキーボードレイアウトアセンブラ/逆アセンブラ







おなじみのLinuxoidが私を非難し、彼らは、WindowsにはCaps Lock言語スイッチがなく、レイアウトも編集できないと言います。 私が見た、そして真実は、すべてのレイアウトがC：\ Windows \ System32 \ kbd * .dllファイルに含まれており、これを16進エディターで編集することは決して便利ではありません。







便利に到達する方法 Caps Lockレイアウトを切り替えるには、あらゆる種類の添付ファイル、Punto Switcherなどの重い添付ファイル、またはlswitchのような単純な添付ファイルを使用できます。 レイアウトの編集にはMSKLCがありますが、あまり機能的で不便ではなく、KbdEditやKLM32のようなアナログが支払われます。







そして、レイアウトDLLを収集するフラットアセンブラーでコードを書くことにしました。







レイアウトはkbdusx.dllとして取得され、システムでは「USA International」と呼ばれます。 キャラクターを追加するための組み合わせがたくさんあるという点で気に入っています。 彼女は、<`〜>、<6 ^>、<'">キーをいわゆるデッドキー、「デッドキー」に変えるという事実を好まなかった。別個のキーボードショートカットがない分音記号付きのラテン文字を入力できますが、プログラマとして、私はこのような頻繁に使用されるキーを「貼り付ける」ことは非常に不愉快でした。







今こそ、このフォーマットをきっぱりと扱う時です。 最初は物事がゆっくりと進み、その後、kbd.hとWindows Driver Kitのレイアウトの例を見つけました。







内部レイアウト



MicrosoftがレイアウトをDLL形式にすると決めたのは面白いことです。ただし、韓国語のkbdkor.dllと日本語のkbdjpn.dllを除き、メインレイアウトテーブルへのポインターを返すエクスポートされたKbdLayerDescriptor関数を除き、コードはまったくありません。

したがって、レイアウトファイルには、キーの名前、スキャンコードの仮想への変換（0x1C→VK_RETURN）、仮想コードの文字への変換、デッドキーの組み合わせ、「合字」のテーブルが含まれています。







どうやら、キーの名前が使用されることはめったにないので、レイアウトからキーをリストするプログラムをコンピューター上で見つけることができませんでした。







スキャンコードのテーブルを変更することにより、任意のキーを別のキーに再割り当てできます。 電源ボタン（電源、スリープ、ウェイクアップ）をすぐに再割り当てしたい場合、これも可能ですが、元の機能はキャンセルされません。 電源設定でオフにすることができます。







シンボルテーブルにはジュース自体があります。 キーストロークK、Shift + K、AltGr + K、Shift + AltGr + Kが何を行うか、Caps Lockが影響するか、Shift + Kと同じ文字列を使用するか、別の文字を使用するかを決定します。彼女のかな。 文字がすぐに印刷されるか、デッドキーキューに入力されるか、「合字」からの一連の文字が印刷されるか、何も起こらないかどうか。







AltGr-これは右Altの名前（レイアウトでKLLF_ALTGRフラグで有効になっている場合）で、Ctrl + Altキーの組み合わせを生成します。







デッドキーのリストには、3番目の文字に変換する必要がある文字のペアが含まれています。 この文字はすぐに印刷することも、リストをもう一度調べてさらに変換することもできます。 リストに文字のペアが見つからない場合は、そのまま印刷されます。 デッドキーのチェーンを使用すると、Compose Keyの動作をシミュレートできますが、Firefoxなどの一部のプログラムは、最初のキーを超える変換を認識しません。







基本的にマクロのセットである「合字」のリストを使用すると、キーに触れるだけで最大4つのWCHAR文字を入力できます。 実際、Windows 7では最大16文字が機能しますが、非常に不愉快な例外があります。Firefoxはこのようなレイアウトに遭遇すると完全にフリーズし、長い「合字」を含むレイアウトがシステミックである場合、実行が完全に停止します。







レイアウトを作成



まず、記事とプログラムの作成に適した、ロシア語と英語の2つの便利なレイアウトを作成したいと考えました。 Ilya Birmanの活版印刷のレイアウトは優れていますが、特に自分で行う場合はさらに改善される可能性があります。







それから、かなキーについて知りました。 かなは、日本語キーボードのCaps Lockのように切り替わるボタンです。 それから、英語とロシア語のレイアウトを1つにまとめ、カナキーでレイアウトを切り替えることにしました。 Caps Lockに再割り当てしました。 Caps Lockライトでかなの状態を表示する簡単なインジケータープログラムを作成しました。







このような組み合わせのレイアウトには、システム全体で同じレイアウトが使用され、プログラム間で切り替えが行われるようになりました。また、「非常にスマートな」プログラムは、テキストの編集やカーソルの移動時にレイアウトを切り替えることができなくなりました。







そして、1つの不快な驚き：何らかの理由で、Psi +は、カナを押してレイアウトを切り替えた後に入力された最初の文字を食べ始めました。







ただし、かなレイアウトと一緒にこのレイアウトをアーカイブに残しました。







次に、Caps Lockに別の一連のキャラクターを割り当てることができることを発見し、KanaをCaps Lockに置き換えました。 指標プログラムは不要になり、不快な驚きはなくなりました。 変更中に、1つの制限のみが見つかりました。CapsLockはKおよびShift + K行でのみ機能します。AltGr+ Kを掛けることはできません。







すべてのデッドキーを2行（Kanaの場合は4行）に収めようとして、順序を逆にするというアイデアに出会いました。AltGr+が文字で入力され、次に修飾子が入力されました。 これにより、以前よりも多くの文字を割り当てることができました。 私はそれを「アンデッドキー」、「デッドキーからの復活」と呼びました。







これで、完璧なレイアウトが完成しました。







kbdusru_undeadレイアウトの利点



  1. 1つのキーCaps Lockでロシア語と英語を切り替えます。
  2. Caps Lockインジケーターは現在の言語を示します。
  3. すべてのアプリケーションのグローバルレイアウト状態。
  4. 「非常に賢い」プログラムが自発的にレイアウトを変更できないこと。
  5. 分音記号やその他の文字を含む文字を入力する機能。
  6. 好みに合わせてレイアウトを簡単に編集する機能。

設置



システムファイルC：\ Windows \ System32 \ kbdru.dllをレイアウトに置き換えるだけです。あらゆる点で優れているからです。 この場合、Windows XPまたは2000では、C：\ Windows \ System32 \ dllcache \ kbdru.dllを削除することを忘れないでください。







しかし、Windowsのそのような強制アップグレードが気に入らない場合は、レイアウトをフォルダーC：\ Windows \ System32 \にコピーし、そのようなreg-fileを使用してシステムに登録できます。 







[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Keyboard Layouts\07430419] "Layout Text"="US+RU" "Layout Display Name"="United States-International + Russian + Extra" "Layout File"="kbdusru_undead.dll" "Layout Id"="00d0"


コードの最初の部分である0743はレイアウトを一意に識別するための任意の番号であり、0419はロシア語のコードであり、その下にレイアウトが表示されます。 非固有（ANSI）プログラムでのレイアウトの動作は、言語コードに依存します。 文字を入力するときに疑問符が表示される場合、これは誤って指定された言語コードの確実なサインです。







さらにもっと



強力なマクロ言語、美しくて素直な構文、あらゆる種類のリンカの役に立たないこと、コマンドライン上のパラメーターの混乱など、フラットアセンブラーでプログラムすることは私にとって喜びです。







そのため、アセンブル時にレイアウトDLLと出力のソースコードを読み取るプログラムを作成することにしました。 変だよね？ アセンブラを逆アセンブラとして使用します。 しかし、fasmの力はそれを可能にします。







その結果、システム全体で外部プログラムを使用せずにCaps Lock言語を切り替えることができるレイアウトと、Windowsキーボードレイアウトを便利に編集できるツールのセットを手に入れました。







すべてがhttps://github.com/grompe/kbdasmにパブリックドメインとして投稿されています。







64ビットWindowsを使用している場合は、アーカイブをダウンロードし、 make.batを解凍して実行してから、install.batを実行できます 。 32ビットの場合、最初にkbdusru_undead.asmファイルを修正する必要があります。







また、アーカイブにはget_scancodesプログラムがあります。 追加のキーを備えたトリッキーなキーボードがある場合は、プログラムを使用してスキャンコードを見つけ、レイアウトでこれらのキーを使用できます。
