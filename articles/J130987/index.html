FOGを使用してOSイメージを展開する



この記事では、OSイメージの作成と展開に役立つ無料のFOG製品を紹介します。 驚いたことに、私はもちろんこの興味深い製品に捧げられたロシア語のリソースに関する記事を見ませんでした。

私は、すべてのニュアンスを絶対的に記述し、すべてをペイントするタスクを自分で設定していないことに注意してください。 個人的な体験に代わるものはありません。 私はただカーテンを開き、多くのシステム管理者が正しい方向に進むのを助けたいです。残りはあなたの手にあります。





FOGは、Acronisの有料製品の代替です。 私はこの会社の製品に何も反対していませんが、今日は彼女についてではありません。 Windows ServerのRIS /展開サービスについても知っています。コメントに書いてはいけません。



簡略化すると、すべてがPXE / TFTPの束を介して機能します。



FOGには、画像管理に加えて、機器診断（memtest）用の優れたツールと、プリミティブインベントリツールが付属しています。

また、Windows XP、Windows 7オペレーティングシステム用に準備されたイメージの作成を検討します（FOGは他のOSをサポートしますが、他のOSには触れません）。

CentOS 5に賭けます。ストーリーをいくつかの部分に分けます。

1.インストールと構成


サイトから最新バージョンをダウンロードします（私の場合はfog_0.32.tar.gz）。

解凍します。 インストールの前に、installation.txtファイルとビデオの説明を読んでください（リンクを参照）。

仕事には、 次のものが必要です。

PHP 5.2.1 +、MySql 5 +、Apache 2+。 結果は、古典的なランプです。

CentOS 5の2番目と3番目に問題がない場合は、最初の公式に問題があります。 リポジトリはありません。pkgs.orgで簡単に見つけることができます（パッケージは公式ではありませんが、自己責任で使用してください）。 私が理解しているように、FOG自体はパッケージをインストールせず、システム内でのその存在のみを調べます。

具体的には、必要なパッケージのリストは./lib/redhat/config.shファイルにあります。 ところで、エラーがあります-CentOS 5にはphp-gettextパッケージはなく、php-php-gettextパッケージがあります。 ウイルススキャンも必要なかったので、clamav-updateをリストから削除しました-このインストールなしでは機能しません。 ところで、管理パネルやその他の設定をインストールするためのパスをすぐに編集できます。



インストールは非常に簡単です。ファイル./bin/ installfog.shを実行する必要があります。 同じファイルにMySQLデータベース設定（snmysqluser、snmysqlpass、snmysqlhost）が含まれています。 FOGデータベースへのアクセスは、他のコンピューター（イメージを展開するコンピューター）に対して開かれている必要があるという事実に注目します。

したがって、次のようなことをする必要があります



create database fog;

grant all privileges on fog.* to 'foguser'@'localhost' identified by '';

grant all privileges on fog.* to 'foguser'@'%' identified by '';









ただし、インストール後でもベースを作成できます。 データベースにアクセスするための設定、管理パネルへのパスなど、FOGはデフォルトでファイル/opt/fog/service/etc/config.phpから取得します。

通常モードとストレージノードモードでインストールできます。 最初のオプションは初めての使用に適していると思います（ビデオの説明で説明されています）。 DHCPサーバーとTFTPサーバーを異なるホストに配置していますが、インストーラーがこの可能性を考慮すると非常に便利です。

FOGが必要なものを見つけた後、インストールが行われます。 注意！ すでにTFTPサーバーが構成されている場合は、設定のバックアップを作成します（/tftpboot/pxelinux.cfg/default）-FOGはそれらを消去し、独自に設定します。 デフォルトでは、メニューでパスワードが使用されます。パスワードが不要な場合は、そこから削除できます。

3つのサービスが作成されます。

FOGImageReplicator、FOGMulticastManager、FOGScheduler。 したがって、chkconfig <サービス名>をオンにすることを忘れないでください。

私の場合、管理ファイルは/ var / www / html / fogに書き込まれました。 サービス自体とその設定は、デフォルトで/ opt / fogディレクトリに書き込まれます。

すべてをセットアップしたら、txi-serverを（xinetdを介して）実行し、FOGサービスが実行されていることを確認する必要があります。

次に、リンクに移動します。 

  http：//あなたのIP /霧/管理


デフォルトユーザー：

ユーザー名：fog

パスワード：パスワード



2. Windows 7用のイメージの作成


イメージの作成は、ハードウェアの独立性を意味します。 これを行うには、Windows 7に既にあるsysprepパッケージが必要です。

イメージを作成するとき、私は主に現在のリリースのコメントで見つけたマニュアルを使用しました。 それは単純で簡単であり、さらに複雑なことは何もありません。

主なポイントは次のとおりです。

ステップ1-Windowsのインストール

ステップ2-Sysprepの使用

ステップツリー-展開のためのPCのクリーニング

ステップ4-設定のアップロード



自分からいくつかの重要なポイントを追加します。

-ウイルス対策ソフトウェアをインストールすることはお勧めしません。イメージを展開した後、OSが起動しないという大きなリスクがあります（これは比較的無害なESET NOD32でも発生しました-他の製品でも同じことが言えます）

-デーモンツール、アルコール120％を元のイメージに明確に入れないでください-sysprepの問題のあるドライバーが含まれています。

-WAIKがなくても実行できます（サイズが大きいのでそうしました-ダウンロードするのが面倒でした）、展開されたイメージを再構成するときの自動化が少し少なくなります。 さらに、他の誰かのunattend.xmlをチートしてダウンロードすることもできます（たとえば、マニュアルから）。 次に、sysprepでスリップします。

-余分に追加しました。 イメージのドライバー。 これらは、driverpacks.netから取得できます。 それらを参照イメージに追加するには、Windows 7をインストールした後、ダウンロードしたドライバーファイルを解凍する必要があります。 次に、FARを使用して、すべての.infファイルを見つけ、検索結果を一時的なパネルに配置し、ファイルへのフルパスをクリップボードにコピーしました。 次に、彼は.batファイルを山積みしました。

pnputil -i -a <.infファイルの前でも>開始しました。 これを完全に自動化する方法は実際にはわかりませんでした（多くの場合、「ドライバーは署名されていません。署名済みドライバーのみをインストールすることをお勧めします」）。 誰もが知っている場合-これを克服する方法をコメントに書いてください。



3. Windows XP用のイメージを作成します。


私は4つの記事に基づいていました。

1） forum.sysadmins.su/index.php?showtopic=6924

2） www.answersthatwork.com/Download_Area/ATW_Library/WinXP_Professional/WinXP__3-Setup-How_to_SYSPREP_a_Windows_XP_PC_setup.pdf

3） remyservices.wordpress.com/2007/09/28/sysprep-in-depth-part-5-customizing-sysprepinf

4） remyservices.wordpress.com/2008/01/24/sysprep-in-depth-part-5-addition-1-customizing-sysprepinf

最初は一般的なアルゴリズムを説明し、2番目はステップバイステップの手順を写真で説明します（また、異なるタイプのIDE / SATAコントローラーを搭載したコンピューターにイメージを展開する際の主な問題を強調します）。

3番目と4番目は、sysprepの応答ファイルの説明です。

Windows XP SP3のsysprepはここからダウンロードできます

この場合のイメージ作成スキームは、Windows 7とは多少異なります。

Driverpacks.netには、OSイメージにドライバーを導入するための特別なプログラムがあります。 私が利用したのは彼女でした。 ここでの使用に関するメモをお読みください 。

ここでのニュアンスは、nLiteなどと一緒にこのプログラムを使用する順序にあります。

最初のリンクにコピーペーストを作成できるようにします。 



 1.システムをコンピューターにインストールします。  （他のコンピューターにインストールしない方が良いスキャナーなどのビデオカードドライバーとエキゾチックなハードウェア）。 必要な設定、変更、追加をすべて行います。  （MUI）ロシア語およびサービスパックをインストールします。
 2.必要なすべてのソフトウェアパッケージを配置します。 アクティベーションを必要としないディストリビューションを使用することをお勧めします。  SP2が統合されたWindows XP Corporate Editionを使用しましたが、SP1も可能です。 
 3.システムをクリーニングします。 ここで、何を残すか、何を破壊するかを自分で決定します。 
 3.1。  DLLキャッシュをクリアします。 これを行うには、次のコマンドを実行します。 
 「sfc / cachesize = 5」および「sfc / purgecache」 
 3.2。 バスケットを空にする
 3.3。 システム回復をオフにし、パーティションのルートからシステムボリューム情報フォルダーを削除します
 3.4。  Msn Messengerを削除します。 
 3.5。  windowsinfsysoc.infを修正してhideという単語を削除し、「プログラムのインストール/アンインストール-> Windowsコンポーネント」で、MSN Explorerと不要なものをすべて削除します。
 4.次に、クローン作成用の構成ファイルを作成します。 
 4.1。  Windows XP CDで、[cd]を探します：/サポート/ツール/ DEPLOY.CAB。 または、インターネット（現在のSP）でダウンロードします。 そこからすべてのファイルを抽出します。 
 4.2。  C：にSYSPREPディレクトリを作成し、DEPLOY.CABからすべてのファイルをそこにドロップします。
 4.3。  SETUPMGR.EXEを開始します。 ウィザードが起動します。 
 4.3.1。  [新しい応答ファイルの作成]、[Sysprepインストール]の順に選択します。 
 4.3.2。  Windowsのバージョンを慎重に選択してください。 
 4.3.3。 次のウィンドウで、[はい、インストールを完全に自動化します]をオンにします。 
 4.3.4。 目の前にウィンドウが開き、あらゆる種類の情報を入力するように求められます。  [プロダクトキーの提供]フィールドは非常に重要です。 次のフィールドは、システム管理者にとって重要です。 コンピューターという名前を書くと、すべてのクローンは同じ名前になりますが、これはグリッドではあまり良くありません。  [コンピューター名を自動的に生成する]を設定した場合、システムはコンピューターに任意の名前を付けます。 私の場合、このフィールドを見逃しました（名前を入力しないと次のメニューに移動できません！）、システムはコンピュータにどの名前を付けたいかを尋ねます。 このプログラムで最初に処方できる興味深いパラメーターがたくさんあります。 すべてのオプションを慎重に検討することをお勧めします。 
 4.3.5。 完了したら、これらすべてをsysprep.infファイルに保存し、ファイルをC：SYSPREPディレクトリに保存する必要があります。 
 5.この時点から、イメージがCDまたは別のパーティションに書き込まれるまで、次に行うことはすべて、システムを再起動せずに実行する必要があります。 
 5.1。  Windowsを別のコンピューターに移動するときの最大の問題は、ディスクコントローラードライバーです。 原則として、ほとんどすべてのボードが標準のIDEコントローラーをサポートしています。 したがって、標準のIDEドライバーをインストールする必要があります。  IDEコントローラーブランチのデバイスマネージャーで、コントローラーを変更（ドライバーを更新）し、Standart ideコントローラーに変更します...キャンセルリブートのオファーをキャンセルします。
 5.2。 ここで、レジスターからキーHKEY_LOCAL_MACHINE / SYSTEM / MountedDevices /を消去することにより、セクション署名の一致に関する問題を回避します。 
 6.ご存知のように、Windows XPはすべてのコンピューターを「Standart PC」、「ACPI PC」、「ACPI Uniprocessor PC」などのいくつかのタイプに分割します。  1回目と2回目の起動時に起動するには、次を実行する必要があります。
 6.1。  Halフォルダー内のすべてをc：/ windows / system32フォルダーにコピーします;そのようなファイルがある場合は、それ以降のバージョンを残すことをお勧めします。
 6.2。 また、boot.iniをわずかに調整する必要があります。 変更後、次のようになります。 
 [ブートローダー] 
タイムアウト= 10 
デフォルト=マルチ（0）ディスク（0）rdisk（0）パーティション（1）WINDOWS 
 [オペレーティングシステム] 
マルチ（0）ディスク（0）rdisk（0）パーティション（1）WINDOWS = "Microsoft Windows XP Professional" / fastdetect 
マルチ（0）ディスク（0）rdisk（0）パーティション（1）WINDOWS = "Standart PC" / fastdetect /hal=hal.dll 
マルチ（0）ディスク（0）rdisk（0）パーティション（1）WINDOWS = "ACPI PC" / fastdetect /hal=halacpi.dll 
マルチ（0）ディスク（0）rdisk（0）パーティション（1）WINDOWS = "ACPI Multiprocessor PC" / fastdetect / hal = Halmacpi.dll 
マルチ（0）ディスク（0）rdisk（0）パーティション（1）WINDOWS = "ACPI Uniprocessor PC" / fastdetect / hal = Halaacpi.dll

このメニューは、デバイスが検出されたときに、コンピューターで最初と2番目のブートに役立ちます。 メニューを選択したときにコンピューターが灰色の画面に落ちたりフリーズしたりする場合は、PCの種類を誤って選択しています。 続行するには、マシンを再起動して、選択操作を繰り返します。

 7.最終段階に進みます。  （START-EXECUTE）コマンドを実行します：c：/ sysprep / sysprep –pnp -mini。 表示されるウィンドウで、「シャットダウンモード」を「終了」に変更し、「再シール」をクリックします。 主なことは、再起動を防ぐことです。 そして、Boot.iniを今すぐ確認します。そこで、タイムアウトを修正する必要があります。10を置きます。
 8.これは完了したと言えます。 ディスクのイメージを作成するための最小のディテールのままです。
 9.新しいコンピューター（または新しいディスク）でこのイメージを開くとすぐに、ミニセットアップが開始されます。 新しい鉄の検索があります。 
 10.インストール後、Standart ideコントローラーをネイティブのものと交換することをお勧めします（ドライバーを更新し、マザーボードのCDに収録されているものを提供します）。
 11.ネットワーク設定を構成します（もちろん、ネットワークがある場合）
 12.数回再起動した後（3回目以降のmoymaで）、Microsoft Windows XP Professionalを読み込むときに、すべてが正常に実行され、デバイスが起動したら、デフォルトでMicrosoft Windows XP Professionalをインストールできます。

deploy.cabパッケージに含まれているsetupmgr.exe実行可能ファイルからダウンロードすると、このファイルを変更できます。

これらのオプションの意味は名前から明らかであり、残りのコメントは3番目のリンクにあります。

コメントを追加します。

-AdminPassword = *パスワードを設定しようとしたときに、展開プロセス中に「パスワードを変更できません」というエラーが表示されたため、このように設定しました。 したがって、パスワードを知らないと、AutoLogonは意味をなしませんが、最初の起動には便利です。

-ドメインがある場合は、すぐにコンピューターをドメインに含めることができます（4番目のリンクとJoinDomainオプションを参照）。ただし、コンピューター名は一意でなければならないことに注意してください（つまり、ComputerName = *）。 実際、ドメインがあり、このオプションを使用しなかった唯一の理由は、ドメインに参加する前に一意の意味のあるコンピューター名を自分で入力することでした（ああ、ドメインを離れずにWindowsのコンピューター名を変更できる場合！）

それだけです

-sysprepフォルダーは、ミニセットアップ後に削除されます。 複製されたシステムには存在しないため、理論的には、sysprep.infファイルにパスワードを明確な形式で書き込むことを恐れることはできません。



4.イメージの展開/作成はどうですか。




そのようなイメージがまだない場合は、イメージ管理セクションに進み、新しいイメージのプロトタイプを作成します。



最初に行うことは、ホストをFOGに登録することです。 これには特別なものがあります。 ネットワーク経由で起動するときのメニュー項目。

これを行った後、ホスト管理セクションに進み、そこでホストを見つけます（MACアドレスと登録作成時間で）。



目的のOSをインストールし、プロトタイプイメージを選択します。





その後、選択できます-このホストをイメージのベースとして使用するか、既存のイメージをこのホストにロールします。

これを行うには、基本タスクでホスト編集モードに入ります。

次は主な選択です（他のタスクがあります）-ロールアップ（展開）またはイメージを参照として使用（アップロード）します。



確認します。



ところで、コンピューターでWakeUpOnLANが有効になっている場合は、コンピューターに物理的にアクセスしなくてもFOGスケジューラーを使用できます。



非常に重要です！ イメージが展開されると、FOGはすべてのパーティションを削除し、独自のパーティションを作成します。 おそらく何らかの形でこの振る舞いを管理できます-誰かが知っていればコメントを書いてください。

タスク管理でスケジュールされたタスクを確認できます。



ホストにタスクが割り当てられている場合、ネットワークから起動すると、メニュー表示なしで実行されます。 さらに、開発者はフィードバックを整理しました-つまり 管理パネルから、現在のタスクの進行状況バーを確認できます。



結論として、ホストの動作を制御する（タスクを実行する）クライアントをWindowsにインストールすることは可能だと思います。 最後のリンクをご覧ください。
