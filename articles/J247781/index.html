Mikrotik-冷却装置を開いて確認する

こんにちは。 幸福。



記事「Mikrotik-長所と短所」を読んで、 あまり高価ではない新しいモデルの冷却ラジエーターに関する「驚き」の言及が見つからず、コメントを追加できない（まだ成熟していない）ので、別の記事として公開することにしましたサンドボックス。 この問題については、私の場合、箱から出してすぐにルータが完全に動作しなくなる可能性がありますが、これは少なくとも不快です。



骨の骨髄のハードウェアメーカーであるため、私は（電源を入れる前に）新たに入手したデバイスを覗く癖があります。 一般的な開発とアセンブリの品質管理の両方の目的で、残念ながら、著名なメーカーでさえ輝いていません。



3つのMikrotikデバイスは、この運命を逃れませんでした。 1つ、高価で大きく強力なMikrotik Cloud Core Router CCR1036-12G-4S、および2つの基本的に管理されるスイッチ、Cloud Cloud Router Switch CRS109-8G-1S-2HnD-IN。



私の驚きには限界がありませんでしたが、あらゆる点で見かけ上素晴らしいデバイスで、アセンブリのほぼ完全に致命的な「サプライズ」、つまり輸送中に落下した小さなチップの冷却ラジエーターが発見されたとき、慎重さの喜びは計り知れないほど大きくなりました。 彼らはただボードの要素の上に横たわっていました！ 最初に「開ける」ことなく食べ物を出したらどうなるか想像するのは怖いです。



これらのラジエーターをチップに取り付ける方法を詳細に研究したとき、私はさらに大きな驚きを経験しなければなりませんでした。 落ちる前に、彼らは両面の文房具のテープのように疑わしく見える透明な紙の薄い材料で接着されました。 素早い研究中の要素と基板の配線のはんだ付けの品質が非常に高い品質に思えたので、この事実は私を少し困惑させました。 ラジエーターでこのような「バグ」をどのように許可できますか？わかりません。



ためらうことなく彼は工場の「粘着テープ」を外し、代わりに有名なロシアのメーカーの熱伝導性接着剤を使用しました。 永遠ではないにせよ、私は長い間、チップからラジエーターが落ちたり滑ったりする問題を取り除いたと確信しています。



記載された状況から2か月が経過しました。 ルーターは、適切な負荷で障害なく動作します。 温度体制は安定しています。



この記事に記載されている情報が、Mikrotik製品を操作する専門家に役立つこと、そして少なくとも指定メーカーのルーターのラジエーターの滑りや落下による故障した部品を交換するために、サプライヤーへの旅行に時間を浪費しないことを本当に願っています。



結局のところ、お金を稼ぐことができ、神経が回復し、健康さえすでに買うことができます。 そして、時間はかけがえのない唯一のリソースです。 それの世話をします。
