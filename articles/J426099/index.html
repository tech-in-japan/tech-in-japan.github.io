こんにちは、私の名前はVadimです。私はtypegramプロジェクトを代表しています。 Typegramは、オープンソースのオンライン出版プラットフォームです。


これは私がYCスタートアップスクールで行うように訓練されたものです。 自分自身とプロジェクトを紹介することから始めます。



はい、私たち（または、私）は、奇妙なことに、同じシゾーの束が彼らのプロジェクトを見て、毎週彼らの成果と失敗について話すこの素晴らしい場所に来ました。 また、素晴らしいアプローチで成功したメンターからの有用なビデオの束もあります。 彼らは面白くて非典型的なことを伝えます。 私は私の第一印象とすべてがどうなるかについて繰り返しません。 過去数年のビデオへのリンクがあります。 学校を卒業するとすぐに（3週間以内に）、私は間違いなく詳細を書きます。 おもしろいことに移りましょう（私が話すのはgithubのパブリックドメインにあります）。

各ユーザーは、個人的なマスコット、モンスター、またはアバターを簡単に配置できます。 各アバターは一意であり、ユーザー名に対して生成されます。 ここで、たとえば、モンスターを描いたデザイナーの秋：









どのように機能しますか？ アドレスru.tgr.am/a/username.pngへの最初のリクエストで、サーバーはこの名前の画像があるかどうかを確認します。 ある場合-レンダリングします。 そうでない場合は、 github.com / recoilme / tgram / tree / master / data / monsterフォルダー内のパーツから画像を収集します。ボディパーツの画像があり、それらは1つの画像にまとめられます。 とても便利でした。 まず、すべてがサーバー上で1回発生します（javascriptブレーキなし）。 次に、 デザイナーは写真をフォルダーに入れて、出来上がります（実際、Photoshopのグリッドに1週間描いて口がひっくり返らず、クールになった）、タイプグラムをダウンロードして他の写真をフォルダーに入れると、すべてが自動的に動作します。 突然、モンスターのメディアライブラリに作業を追加したい場合は-よく。 以下は、基礎を形成したbindataから疎外されたコードです。 たとえば、コマンドラインからジェネレーターを試してみたい場合。

電報への自動公開



記事を電報で送信するスクリプトが登場しました。 これは非常に議論の余地のあることです。 電報では、材料の書式設定がすべて悪いのです。 その中に長い投稿を書くことは非常に不便です。 これで、投稿をtaiperamに書き込むことができ、チャンネルに関連付けられている場合、すべてが自動的に行われます。 それは魔法のように見えます。 しかし、実際的な利点はほとんどありません。 原則として、この地獄のようなコンバーターを通過するよりも、記事へのリンクを記載した平凡なアイライナーを書く方が面白いからです。 一般的に、そのような機能は、技術的に興味深い、実際にはほとんど価値がありません。 Githubの実装
