インフラストラクチャは「プライベートクラウド」でシンプルかつ簡単です

1か月前、Cisco、NetApp、およびMicrosoftは、Hyper-V Cloud FastTrackイニシアチブの一環として、共同クラウドプラットフォームの開発を発表しました。 このイニシアチブは、ハードウェアメーカーとマイクロソフトクラウドプラットフォームを統合して、そのようなソリューションを可能な限り迅速かつ簡単に構築することを目的としています。

NetAppとCiscoによるこのソリューションのコンポーネントを詳しく見てみましょう。 ソリューションは、柔軟な統合アーキテクチャ（統合アーキテクチャ）上に構築されています。 通常、Hyper-V Cloud Fast Trackソリューションの主要なコンポーネント（ハードウェアの観点から）は、次の要素です。

1）x64サーバーアーキテクチャ（複数の同一サーバーノードを選択し、それらをクラスターに結合することをお勧めします）

2）ストレージシステム（これらのシステムにディスクスペースの仮想化機能があることが望ましい）

3）IPおよびSANトラフィックを送信するための通信機器（それぞれIPトラフィックとSANトラフィック用の統合デバイスまたは個別のデバイスのいずれか）



特定のケースでは、回路は次のようになります。

1）サーバーCisco UCS-シスコは最近、サーバー市場で、このカテゴリのソリューションを積極的に発表しています。 これらのサーバーの特徴は、仮想プラットフォームと物理プラットフォームの両方での音声およびビデオデータ転送アプリケーションの最適化です。 また、Cisco VN-LinkやCisco Extended Memory Technologyなどのテクノロジーは、データ転送タスクを簡素化し、RAM容量を増やすことができるため、不必要なコストをかけずにシステムのスケーラビリティを高めることができます。

2）NetApp Unified Storage System-このコンポーネントはクラウドソリューションに完全に適合しており、ディスク仮想化テクノロジーを備えていると同時に、さまざまなインターフェイス（iSCSI、FC、FCoE）を介してアクセスを提供し、さまざまなファイルシステム（NFS、CIFS）をサポートします。 これは真に統合されたソリューションです。LinuxとWindowsの仮想マシンを同じストレージシステム内に、それぞれ独自のファイルシステムに簡単に配置できます。 重複排除テクノロジーのおかげで、クラウドインフラストラクチャはそれほど「厚く」ならず、一意のデータブロックのみが保存され、重複するデータブロックはすべて一意の元のブロックへのリンクに置き換えられます。

3）通信機器Cisco UCS FabricおよびCisco Nexus-テレコムセクターのリーダーが提供する統合された統合機器。 信頼性の高い耐障害性と動的なクラウドインフラストラクチャを提供するには、このチャネル上でiSCSIまたはFCoEを使用する10ギガビットイーサネットソリューションを使用する必要があります（NetAppは両方をサポートします）。 ユーザーデータを転送するには、ビジネスタスクと機能に応じて、帯域幅の狭いスイッチを使用することをお勧めします。



ソリューションのすべての主要なハードウェアコンポーネントをリストしました。次に、この山のような金属からクラウドプラットフォームを構成するものを見てみましょう。

1）Microsoft Hyper-V-仮想化の主要コンポーネントであるハイパーバイザーは、Microsoft Windows Server 2008 R2 SP1オペレーティングシステムの一部です。 クラウドプラットフォームの重要なコンポーネントであるクラスター化された仮想化プラットフォームを作成できます。

2）Microsoft System Centerファミリー-異種クラウドインフラストラクチャを管理、サポート、開発するための一連の製品。 多くの製品で構成されており、主要なものは次のとおりです。

a。 System Center Virtual Machine Manager-仮想マシン、仮想ネットワーク、クラウド環境を作成および管理するための製品。 SMI-Sプロトコルを使用して、製品のストレージシステムを管理できます。 VMware、Citrix、Microsoftなどのさまざまな仮想化プラットフォームをサポートしています。 また、BMC、iDRAC、SMASHプロトコルのサポートにより、Hyper-Vをベアサーバーに展開するプロセスを制御できます。 また、仮想化サーバープロセスの更新も担当しています。

b。 System Center Operations Managerは、パフォーマンスを最適化し、ITインフラストラクチャをサポートするための製品です。 SCVMMと統合すると、クラウドプラットフォームのPRO機能を使用できます。つまり、仮想マシンの負荷を追跡し、負荷に応じて仮想マシンの配置を最適化し、冗長サーバーノードを無効にして電力消費を管理できます。

c。 System Center Data Protection Managerは、ITインフラストラクチャコンポーネントのバックアップと復元を担当する製品です。 物理コンポーネントと仮想コンポーネントの両方で動作します。 すべてのMicrosoft製品（SQL、Exchange、SharePoint、Lync）の全体的なオンラインコピーをサポートします。 NetAppストレージとシームレスに統合されているため、瞬時に信頼性の高いバックアップを取得できます。

d。 System Center Service Managerは、他のSystem Centerコンポーネントと緊密に統合されるサービスデスクシステムであり、サービスデスクリクエストを通じてルーチンタスクを自動化できます。 ITILおよびMOFに従ってITサービスの作業を実装できます。

e。 System Center Orchestrator（Opalis）は、インフラストラクチャのフレームワーク内でITプロセスを構築および自動化する製品です。 ITサービスに存在するルーチンプロセスの自動化の問題（特定のイベントが発生したときにバックアップまたはリカバリを開始するなど）のソリューションとソリューションの統合のための重要なコンポーネントです。



これらのコンポーネントはすべて一緒にクラウドを形成します。 プライベートかパブリックかは、あなたとあなたのタスクに依存します：高品質、信頼性、柔軟性、高速のITインフラストラクチャを企業に提供したり、サードパーティユーザーに高品質のサービスを提供したり、IaaS、SaaSなどのさまざまなクラウドソリューションを実装したりできます。

ロシアでは、これまでのところ、クラウドプラットフォームの新規性により、そのようなソリューションはほとんどありません。 しかし、新規性にもかかわらず、一部の企業はすでに準備を進めており、今日そのようなソリューションを実証する準備ができています。 Mikrotestデモセンターで、CiscoおよびNetAppの機器に基づいてMicrosoftクラウドプラットフォームがどのように機能するかを確認できます。 同社はSystem Center 2012製品ファミリの製品に基づくプライベートクラウドを展開しており、この製品ファミリはまだベータ版ですが、2011年末には製品の最終バージョンが発表されます。 ただし、今日は、文字通り明日使用されるものを実際に試して検証する機会があります。
