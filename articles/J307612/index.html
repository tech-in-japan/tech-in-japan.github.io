起こった！







当社は、PostgreSQLドキュメントの現在のバージョンのロシア語への翻訳を公式に完了しました。この出版物では、それがどのようであったかをお伝えしたいと思います。 また、この目標を達成するために行ったパス（およびどのような方向性を試みたか）についても話したいと思いますが、これはおそらく別の記事のトピックです。



PostgreSQL自体のロシア語への翻訳は2001年に始まり、その後、postgresql 7.1のバージョンのみがリリースされ、postgresql自体では、開発者の努力にはメッセージをローカライズする機能のみが含まれていました（ こちらを参照）。 メッセージのロシア語への翻訳が、フランス語、ドイツ語、スウェーデン語、中国語、チェコ語への翻訳とともに、バージョン7.2に初めて含まれました。



庭にはエンコーディングKOI8-Rがありました。 Windows 2000、Linux 2.4.1、RHEL 2.1、Debian Woody。 pgadmin II。 Ubuntu？ いいえ、聞いていません。



サーバー7.2では、約30％のメッセージがロシア語に翻訳されました（3205から881メッセージ）。 サーバー7.2のドキュメントには約1000ページかかりました。 2005年1月-バージョン8.0では、約80％が翻訳されました（2888メッセージ）。 Windows 2003、Linux 2.6.10、RHEL 4、Debian Sarge、Ubuntu-すでに、いぼのあることを聞いたことがあります。



バージョン9.0では、翻訳の割合は64％（2765メッセージ）に減少しました。









最後に、2012年にバージョン9.2で追いつきました-100％（6388メッセージ）。 （バージョン9.5では、7675のメッセージが翻訳されています。翻訳付きのメッセージのサイズは約1.8 MBです。）
