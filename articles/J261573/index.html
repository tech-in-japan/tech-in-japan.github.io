コンソールからVeeam Backup＆Recovery Freeを起動します



記事の冒頭で参照することで、著者は、VB＆R FreeがStart-VBRZipを介して仮想マシンをバックアップできると言って、舌を滑らせました。 当時はそうではありませんでしたが、ここでは Veeamの従業員がそのような機能を実装することを約束しました。 そして4月末にそれが起こりました。

実際、多くの人が知っているように、現在、 Start-VBRZipは無料ライセンスで実行できますが、いくつかの制限があります。 主なものは完全バックアップのみであり、1つのタスクで1つの仮想マシンのみです。 ところで、GUIを使用して、複数のバックアップを同時に実行できますが、タスクは異なります。 怖くはない、と私は思いました、そして、 指示を読んだ後、最も単純なバージョンで、私はこのようなスクリプトを実装しました：



PoSHのスクリプト。 この例では、VMwareの場合。 



# Initialize Veeam Toolkit & "C:\Program Files\Veeam\Backup and Replication\Backup\Initialize-VeeamToolkit.ps1" # Set Virtual Machines list $vmNames = 'vm1.domain.lcl', 'vm2.domain.lcl', 'vm3.domain.lcl', 'vmX.domain.lcl' # Start backup job for each VM from list foreach ($i in $vmNames) { $viObject = Find-VBRViEntity -Server vc1.domain.lcl -Name $i Start-VBRZip -BackupRepository "VeeamBackupRepo02" -Entity $viObject -Compression 5 }


私が説明します：

$ vmNames-シリアルバックアップ用の仮想マシンのリスト。 書き込まれているようにスペースを含めるようにしてください 。そうしないと、Veeamはこれが1つのタスク内のマシンのリストであると見なし、正しく動作して送信します... 1つのタスクに1つのマシンしか存在できないことを示します。

vc1.domain.lclはvCenterアドレスです。 ホストに直接固執することもできますが、VB＆RはvSphereの無料バージョンでは動作できないことを覚えています。 最小-vSphere Essentials。 （完全に無料のESXi 5.0でVeeamを使用できることは知っていますが、ここでは何も約束しません）。
