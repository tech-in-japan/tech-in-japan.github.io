開発者向けの2日間のオンライン会議を10ドルで開催する方法は？





.NETプラットフォームとオープンソースプロジェクトを愛する開発者コミュニティのオンライン会議であるDotNetConf会議を終了しました。



すべての会議レポートは、 公式ウェブサイトのリンクからすでに入手可能です。



会議プラットフォーム



私たちのプラットフォームを会議が実行されているソフトウェアと呼ぶのは非常に面白いです。それはあまりにも「エンタープライズ」であり公式に聞こえます。 過去に、必要な費用を支払ったスポンサーとのaspConfおよびmvcConf会議を開催しました 。 スタジオであるチャンネル9のパワーを活用し、シアトルまたはLive Meetingを通じてビデオを放送しました。



しかし、今年は、できるだけシンプルで、より安価で、より分散された会議を開催したいと考えました。 どんなタイムゾーンからでもスピーカーを招待したかった。 安かった？ 約10ドル。 正確な請求書は後で計算されます。容量を増やし、イベントを開催し、受け取ったリソースを拒否したかっただけです。



ビデオ放送と画面共有



  • 今年は、Googleハングアウトとそのハングアウトオンエア機能を使用しています。 dotnetconf Googleアカウントは、プレゼンターをハングアウトに招待し、ブロードキャストを開始する前に「オンエア」機能をマークします。 次に、ハングアウトツールボックスを使用して、画像にグラフィックスとプレゼンターの名前を動的に追加します。 すべてのスピーカーが解像度を1280x768に設定し、ライブストリームを480pに減らすことができます。
  • 「放送開始」をクリックするとすぐに、YouTubeへのリンクが生成され、ライブストリーミングが可能になります。「放送終了」をクリックすると、録画されたビデオは数分でYouTubeアカウントで視聴できるようになります 。 ）「ブロードキャストで非表示」をクリックして空中に消えます。下のスクリーンショットで「左」になっているのがわかりますが、まだ表示されていませんが、必要な場合にのみ表示されます。ビデオはフルスクリーンに拡大されますが、これが必要です。
  • 重要な注意： 8時間連続で1つのハングアウトを使用する代わりに、発表者ごとに新しいハングアウトを開始および停止しました。 これは、YouTubeページで投稿が既に共有されていることを意味します。 YouTubeビデオは最初と最後からトリミングできるため、ビデオの処理に問題はありません。






データベース



サプライズ！ データベースは使用しませんでした。 それらは必要ありませんでした。 WebMatrixで記述されたASP.NET Web Pagesを使用して2ページのサイトを立ち上げました。 このサイトはWindows Azureクラウドで実行され、データセット（スピーカー、スケジュール、ビデオストリームの場所など）はほとんど変わらないため、このデータをXMLファイルに入れます。 これは確かにデータベースですが、貧しい人々のためのデータベースです。 不要なものにお金を払うのはなぜですか？



会議中に「データベース」をどのように更新しましたか？ あなたの意見を述べる準備をしてください。 データはDropBoxにあります。 （はい。SkyDriveまたは任意のURLに配置できますが、DropBoxを使用しました）。



Webアプリケーションは、DropBoxからリンクデータを取得してキャッシュします。 とてもうまくいきます。

SignalRを使用したリアルタイムのサイト更新



YouTubeレポートをブロードキャストするリンクがスピーカーごとに変わるため、会議の参加者が新しいURLを取得するためにブラウザーのページを更新する必要があるという問題がありました。 このシナリオには、考えられる多くの解決策があります。 メタによる更新、タイマーによる更新を指定できますが、これらの決定はすべてオンデマンドで実行されるわけではありません。 さらに、スピーカーの変更中にスクリーンセーバーを表示したかったのです。 私たちの1人が次のスピーカーを準備している間、視聴者はビデオスクリーンセーバーを受け取り、視聴していました。



この問題は、会議の前夜の午後10時ごろに発見されました。 ハビエルと私はSkypeでチャットし、次のハックをすることにしました。



誰もがSignalRクライアント（リアルタイムHTML / JSサーバー/クライアントインタラクションライブラリ- およそTransl。 ）を持っている場合はどうでしょうか？ その後、管理コンソールからプッシュすることにより、視聴者に次のビデオを直接送信できます。



これを説明させてください。 ビューアー（あなた）、管理者（I）およびサーバー（ホスト）がいます。 ビューアは、SignalRライブラリで次のJavaScriptコードとともにメインページを使用します。 



 $(function () { var youtube = $.connection.youTubeHub; $.connection.hub.logging = true; youtube.client.updateYouTube = function (message, password) { $("#youtube").attr("src", "http://www.youtube.com/embed/" + message + "?autoplay=1"); }; $.connection.hub.start(); $.connection.hub.disconnected(function () { setTimeout(function () { $.connection.hub.start(); }, 5000); }); });


視聴者は聞くか、むしろ、YouTubeビデオの短いコードを含むサーバーからのSignalRからのメッセージを期待します。 メッセージが到着すると、iFrameを置き換えます。 すべてがシンプルで、機能します。



以下は、期待する視聴者に新しいYouTubeビデオコードを送信する管理コンソールのマークアップです（ASP.NET WebページとWebMatrixのRazorを使用しているため、これはHTMLとJSの混合コードです）。 



 <div id="container"> <input type="text" id="videoId" name="videoId"> <input type="text" id="password" name="passsword" placeholder="password"> <button id="playerUpdate" name="playerUpdate">Update Player</button> </div> @section SignalR { <script> $(function () { var youtube = $.connection.youTubeHub; $.connection.hub.logging = true; $.connection.hub.start().done(function () { $('#playerUpdate').click(function () { youtube.server.update($('#videoId').val(), $('#password').val()); }); }); $.connection.hub.disconnected(function() { setTimeout(function() { $.connection.hub.start(); }, 5000); }); }); </script> }


短いコードとパスワードを送信します。 これはすべて非常に複雑なはずですよね？ Windows Azure Service Busを使用してクラウドで実行される強力なSignalRバックエンドはどのようなものですか？ コードが大きすぎてブログに表示できないはずですよね？ リラックス、友達。 



 public class YouTubeHub : Microsoft.AspNet.SignalR.Hub { public void update(string message, string password) { if (password.ToLowerInvariant() == "itisasecret") { Clients.All.updateYouTube(message); ConfContext.SetPlayerUrl(message); } } }


それは純粋主義者の悪夢であり、実用主義者の夢でもあります。 いずれにせよ、会議中にこのコードを使用し、機能しました。 セッション間で、事前に記録されたスプラッシュビデオへのリンクを送信し、次のレポートの前に彼のビデオへのリンクを送信しました。



さらに、DropBoxリンクをビデオストリームの現在のアドレスで更新しました。これにより、新しいビジターはビデオへのリンクで次のプッシュを待つのが間違っているため、新しいビジターは最新ビデオへの正しいリンクを受け取りました。







スケールアップはどうですか？ ファーム内に2台の車があったため、更新されたビデオリンクをプッシュしてスケーラブルなソリューションに適合させるためにSignalRが必要でした。 さらに10分かかりました。



Windows Azure Service Busを使用したSignalRスケーリング



SignalR 1.1ベータ版とAzure Service Busトピックを使用してスケーリングしたため、Windows AzureアカウントにService Busを追加しました。 アプリケーションの起動コードが変更され、UseServiceBus（）メソッドの呼び出しが追加されました。 



 string poo = "Endpoint=sb://dotnetconf-live-bus.servicebus.windows.net/;SharedSecretIssuer=owner;SharedSecretValue=g57totalsecrets="; GlobalHost.DependencyResolver.UseServiceBus(poo,"dotnetconf"); RouteTable.Routes.MapHubs();


SignalRは、Publish / SubscribeメカニズムにService Bus Topicsサブスクリプションを使用して、2つのWebサーバー間でプッシュ通知を送信するようになりました。 Web 1サーバーからプッシュ送信を行うと、このメッセージは、リアルタイムSignalR接続を介してWeb 1サーバーとWeb 2（またはWeb N）の両方に接続している全員に配信されます。







会議が終了するとすぐに、このService Busトピックを削除します 。 私は本当に数ダイムのアカウントを取得したくありません。 ;）以下は、 Windows Azureでのこのサービスの価格です。
