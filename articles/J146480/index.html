すぐにエミュレーションなしでWindowsアプリケーションを実行するのがいいとすぐに思いました。 多くの人がすでにエミュレーター（Qemu、Bochs、aDOSBoxなど）で古いバージョンのWindowsを実行しようとしたことがあると思います。

まあ、手にデバイスを持っていないので。 残ったのは、Android x86エミュレーターでアイデアを確認することだけでした。



仕事を始める



まず、Android x86 Atom System ImageをAndroid SDKに追加しました。 それを実行すると、信じられないほどのブレーキにぶつかりました。GPUをオンにしても問題は解決しませんでした。 作業が非常に遅いため、Androidをルート化することさえ望んでいません。

そこで、私はすぐにVirtualBoxの下にAndroid x86をインストールしました。 VMを初めてセットアップするときは、少なくとも7 GBの大きなHDDサイズを指定します。 Androidのインストールの最後の段階で、偽のSDカードの作成を拒否します。



次の問題は、 * .imgに Ubuntuがインストールされているイメージを見つけることでした。

一方、Ubuntu 12のイメージはVirtualBox * .vdiの下にありました

最初に思いついたのは、次のようなコマンドで* .vdiを* .imgに変換することでした

VBoxManage clonehd --format RAW ubuntu.vdi ubuntu.img





しかし、これは正しい方法ではありませんでした。 このようにして、パーティションテーブルを含むディスク全体のイメージが作成されます;それをマウントしないのはとても簡単です。



相談した後、正しいソリューションが見つかりました。 livecdを使用して仮想マシンをロードする必要があります。最初に、鉄道の* .vdi AndroidおよびUbuntuイメージを接続します。

コンソールで、livecdから起動します

sudo parted -l





Ubuntu x86のマウントされたイメージの名前を確認します。 これは通常sda1です



さらに自分で、右側の777のイメージにすべてのディレクトリとファイルを設定することをお勧めします。 これらの操作には時間がかかるという事実にもかかわらず。 これを行わないと、画像をAndroidにマウントするときに解決できない問題が発生する可能性があります。 私はそれらを持っています。

そこで、次のコマンドを使用します。

# find /media/ -type d -exec chmod 777 {} \;





# find /media/ -type f -exec chmod 777 {} \;







どちらか



chmod -R 0777 /media/







次に、入力コンソールで、マウントされたUbuntuイメージのブロックデバイスをフィードします。 出力用に、イメージファイルubuntu.imgを書き込みます。便宜上、Androidイメージのsdcard / ubuntuディレクトリにすぐに書き込みます。

これを行うには、次を実行します。

dd if=/dev/sda1 of=/media/Android-x86/android-4.0-RC1/data/media/ubuntu/ubuntu.img bs=1M





操作が速くないのを待っています。 Androidイメージに十分なスペースがあることを確認してください。



再度、受信したubuntu.imgイメージファイルに777の権利を設定します

まあ、彼らは一種のイメージを用意しました。 先に進みます。



取り付け



マウントは、おそらく最もストレスの多い段階です。

すべての処理が完了した後、ついにAndroidを起動します。

まず、次のものが必要です。

1. /sdcard/ubuntu/ubuntu.imgディレクトリにあるUbuntu x86イメージ

2. Android x86のルーティングされたアセンブリ。 通常、それらはすぐにフレーム化されます。

3. Androidターミナル

4. android-vnc-viewer

5.ルートエクスプローラー



開始するには、Androidでネットワークを選択します。

ネットワークアダプターとして、NATモード（または環境にDHCPサーバーがある場合はブリッジモード）でPCNET Fast IIIを選択します。



インターネットの正しい動作は、端末から設定できます。

ネットワークインターフェイスを一覧表示します。 それらの中には、例えばeth0があるはずです

IP

ルート権限を取得します。

す

DHCP経由でネットワーク設定を取得する

dhcpcd eth0

DNSサーバーを構成する

setprop net.dns1 8.8.8.8

これらの手順の後、仮想マシンのインターネットが表示されます。

adbを構成します。 VirtualBoxオプションの仮想マシンポート5555を、たとえばホストポート5556に転送します。



これで、adbを使用して仮想Androidに接続できます

adb connect 127.0.0.1 ∗ 556

ルートのシェルを取得する必要がある場合

adbシェル

ファイル共有とアプリケーションのインストールはコマンドによって行われます

adb pushまたはadb install

市場にとって、インターネットは私にとってうまくいきませんでした。 ただし、ブラウザおよびシステムではそうです。

解像度を確認する最も簡単な方法は、スクリーンショットを撮ることです。

ワインを入れる

まあ、すべては簡単です

sudo apt-get install wine







落とし穴についてはあまりない



エミュレーターの下で作業する必要がありましたが、おそらくエミュレーターには素晴らしいバイナリトランスレーターがありません。

そのため、NDKを使用して作成されたソフトウェアは起動せず、エラーが発生しました。

NDKなしで記述されているandroid-vnc-viewerが使用されているのはそのためです。 ただし、android-vnc-viewerはハードウェアキーボードを完全にはサポートしていないことに注意してください。 そして、本格的なアナログは機能しません。

NDKが使用されたかどうかの確認は非常に簡単です。 たとえば、 libディレクトリがある場合、WinRARで* .apkを開き、NDKを使用しました。 このディレクトリでは、アプリケーションがどのアーキテクチャ向けに構築されたかを確認できます。 ARMv6、v7以外の開発者はすぐにx86を追加すると思います。

エミュレータの下での速度を考えてみてください。 また、Ubuntuではかなり重いシェルを使用しました。

android-vnc-viewerでは、特定のモードでのマウスの右ボタンは、左ダブルクリックのように機能します。



ゲームのバイナリトランスレータが最初にどれだけ生産的になるかを確認したいと思います。
