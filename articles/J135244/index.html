モジュール設計から始めましょう





感情の自動認識のためのデバイスの「入力」には、モジュールにロードされたカラーまたは白黒の画像またはビデオストリームがあり、「出力」には、顔と顔の特徴の周りに記述された長方形の配列、見つかった顔の特定の感情の存在に関するテキストの結論があるはずです。



モジュールは、3つの主要なステージで構成されています。

1.顔や顔の特徴をより便利に認識するための画像の準備。

2. 開発された修正Viola-Jonesアルゴリズムによる顔および顔の特徴の認識。

3.検出された感情の確認。画面に関連情報を表示します。



開発環境とツールの選択





計画されたプログラムモジュールは、仮に「EmotionRecognitionTool v 0.1 by SkyNoName」と呼びます。これは、デモ用にViola-Jonesアルゴリズムを修正して実装し、 Microsoft Visual Studio©2010の C言語の統合開発環境 （IDE）または最新バージョンのコンピュータービジョンの最新OpenCVライブラリー（オープンソースコンピュータービジョンライブラリー）を使用するC ++（バージョン2.2以前をインストール済み）、およびtheo watson userのvideoInputバージョン0.1995を使用 。 Cmakeモジュールのクロスプラットフォームビルドシステムを使用して、 Windows 7 UltimateおよびUbuntu 10.10オペレーティングシステムでモジュールを開発しようとします 。

実際、資金の選択は決して制限されません。 クロスプラットフォームのQtツールキットや他のプログラミング言語を使用して、たとえばGDを使用してPHPで独自のアプリケーションを作成できます。 しかし、私は現時点で手元にあるものを選択しました...

まず、 公式アプリケーションのダウンロードページから Microsoft Visual Studio をダウンロード、 支払い（;）） 、インストールします 。

次に、OpenCVライブラリをダウンロードします 。 ところで、最近、多くの追加と変更、つまりバージョン2.3.1が追加された新しいライブラリリリースがリリースされましたが、それについては後で詳しく説明します。 最小限の開発では、 公式ダウンロードページで 2.3または2.2を選択できます 。

また、 Creative commonsライセンス（cc by-nd 3.0）で配布されるCmakeビルドシステムが必要です。 ダウンロード

インストール手順

win32のバージョン2.2の場合、これはインストール中に行われます。現在のユーザーのシステムPATHにOpenCVを追加する項目のチェックボックスがチェックされています。
