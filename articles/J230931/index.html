Nuleo-F401はなぜですか？





NuleoプラットフォームこれはDISCOVERYとArduinoプラットフォームのハイブリッドであり、ユーザーはほとんどすべての拡張機能とすべてのコードサンプルを簡単に使用できます。



Arduino このプラットフォームはMBED.ORGをサポートしており 、ARM用コンパイラのインストール、設定、使用における多くの初心者の頭痛を取り除きます。 Arduinoの既存の拡張機能のほとんどすべてがmbed.orgにインポートされています。 すでに広く普及し、多くの人に愛されているDISCOVERYプラットフォームとは対照的に、ボード上のプラットフォームには、たとえば加速度計、コンパスなどの周辺機器はありません。 ただし、Nucleoプラットフォームは、Arduinoプラットフォームと同じくらい簡単に作業できるように設計されています。 プラットフォームには、柔軟な電源スキーム（3.3V-5V-7V-12V）、STM32をプログラミングするためのビルトイン更新ST-LINK / V2.1プログラマー、ビルトイン仮想COMポート（デフォルトでUSARTに接続）、マイクロコントローラーのフラッシュメモリへのアクセスを提供するリムーバブルメディアのエミュレーション。



また、STM32F401REマイクロコントローラーの機能を自分で調べることもできます 。購入時には、パフォーマンスと大容量のメモリを備えた他のモデルとは異なり、最も古いモデルであったため、ST Nucleo-F401を選択しました

LED操作モード。


ボードには3つのLEDがあります

3色LED（緑、オレンジ、赤） LD1（COM）は、ST-LINK接続のステータスに関する情報を提供します。 デフォルトでは、 LD1の色は赤です。 LD1は緑色に変わり、PCとST-LINK / V2-1を接続するプロセス中に、次のことを示します。

  • 「赤くゆっくり点滅」/「オフ」-USB接続が確立されるまで電源をオンにしたとき
  • 「すばやく赤く点滅」/「オフ」-PCとSTLINK / V2-1（接続プロセス）間の最初の正しいデータ交換後
  • 「赤色LEDがオン」-PCとST-LINK / V2-1間の接続が確立されている場合
  • 「緑色に点灯：接続が正常に完了したか、操作が正常に完了しました
  • 「赤点滅」/「緑」-データ転送中（ファームウェアプロセス自体）
  • 「オレンジ点灯」：通信障害。




USER LD2 ：緑のユーザーLED。LEDはArduinoピンD13に接続され、マイクロコントローラーPA5 （ピン21）またはPB13 （ピン34）の入出力ピンに対応します。

  • 入力/出力（I / O）値が高い場合、LEDは点灯しています。
  • 入力/出力（I / O）値が低い場合、LEDはオフになります。




LD3 PWR ：赤いLEDは、マイクロコントローラーが搭載されたデバイスに電源が供給され、+ 5 Vが供給されていることを示します。



ボタン


B1 USER ：ユーザーボタンは、STM32マイクロコントローラーのPC13入出力（ピン2）に接続されています。

B2リセット ：このボタンはNRSTに接続し、STM32マイクロコントローラーをリセットするために使用されます。



注： シールドまたは拡張カードがNucleoの上部に接続されている場合など、必要に応じてボタン の 青と黒のプラスチックキャップを取り外すことができます。 これにより、ボタンへの圧力が回避されるため、偶発的に押し続ける可能性があります





ジャンパーJP6（IDD）


IDDとマークされたジャンパーJP6は、マイクロコントローラーの消費電流を測定するために使用されます。これを行うには、ジャンパーを取り外して電流計を接続します。

  • ジャンパーオン：STM32マイクロコントローラーの電源が入ります（デフォルト）。
  • ジャンパーOFF：STM32マイクロコントローラーの電流を測定するには、電流計を接続する必要があります。 電流計がない場合、STM32マイクロコントローラーには電源が​​供給されていません。




ビルトインST-LINK / V2-1


ST-LINK / V2-1インサーキットデバッガー/プログラマーは、STM32デバイスのSWDのみをサポートします。

ST-LINK / V2バージョンと比較した変更：

ST-LINK / V2-1でサポートされる新機能：

  • USBドライバーの番号が変更されました
  • USB経由の仮想COMポートインターフェース
  • USBストレージインターフェース
  • USBパワーマネージャー


ST-LINK / V2-1でサポートされていない機能：

  • SWIMインターフェース（STM8のプログラミングに必要）
  • サポートされる最小アプリケーション電圧は3 Vに制限されています


ビルトインST-LINK / V2-1の動作モード：

PCB分離


STM32 Nucleoプリント基板プラットフォームは、ST-LINKパーツとターゲットMCUパーツの2つの部分に分かれています。これらは、ボードのサイズを小さくするために分割できます。 この場合、残りのターゲットMCUパーツは、ST Morpho CN7のVIN、E5V、3.3VまたはArduino CN6のVIN、3.3のみで給電できます。 この場合でも、ST-LINKパーツを使用して、CN4 SWDとST Morphoのコネクター（ SWCLK CN7 ピン-15およびSWDIO CN7 ピン-13 ）間のワイヤを使用してメインマイクロコントローラーをプログラムできます。



ボードの電源モード。


電源は、USBケーブルを介してコンピューターから、または外部ソース（CN6またはCN7のVIN（7V-12V）、E5V（5V）または+ 3V3電源ピン）から供給されます。

ST-LINK / V2-1は、プログラマー自身が最大100 mAの電流を消費する一方で、USBを介したプラットフォームへの給電をサポートします。



これらの条件下で合計消費電流が300 mA（ST-LINK 100 mA消費を含む）を超えない限り、拡張カードを含むSTM32 Nucleoボード全体にST-LINK USBから電力を供給できます。赤色LD3 LEDが点灯し、マイクロコントローラーが動作しています 消費電流が多い場合は、プロジェクト全体または拡張カードのみに外部電源を使用する必要があります（共通グランドを忘れないでください）。



ボードにUSB（U5V）から電力を供給する場合、JP5のピン1と2の間にジャンパーをU5Vの位置に接続する必要があります。 JP1ジャンパーは、ボードがUSBで駆動され、U5Vの最大消費電流が100 mAを超えない場合にのみインストールできます（すべての種類の拡張カードまたはArduinoシールドを含む）。



Nucleoおよび拡張カードの消費電流がUSBの許容電流を超える場合、VIN（7V-12V）またはE5V（5V-5.5V）を外部電源として使用できます。 この状態でも、プログラミングまたはデバッグにUSB接続を使用できますが、最初にVINまたはE5Vを使用してボードを電源に接続してから、USBケーブルをPCに接続してください。



VINまたはE5Vから給電される場合、パワーオンアルゴリズムに従う必要があります。

  • JP5のピン2と3の間にジャンパーを接続します。
  • JP1が削除されていることを確認してください。
  • 外部電源をVINまたはE5Vに接続します。
  • 外部電源の電圧は、VINに対して7 V <VIN <12 V、またはE5Vの場合は5 Vです。
  • LD3がオンになっていることを確認してください。
  • PCをCN1 USBコネクタに接続します。


この順序が守られない場合、最初にVBUSを介してボードに電源を供給し、次にVINまたはE5Vを介して電源を供給できますが、コンピューターのUSBポートが誤動作したり、マイクロコントローラーのグリッチが発生したりする可能性があります（！

ボードにVINまたはE5Vから電力が供給される場合、ジャンパー構成は次のようになります。

  • JP2のピン2とピン3の間のジャンパー。
  • JP1でジャンパーが削除されました。
