最大可用性：Veeam Availability Suite 9.5

近年、膨大な数の異なるITサービスおよびビジネスアプリケーションが出現したため、処理される情報の量が指数関数的に増加しました。 これにより、企業はより効果的なデータアクセシビリティツールを探す必要があります。 拡張性の向上とあらゆるサイズの仮想環境のサポートにより、Veeam Availability Suite 9.5の新しいバージョンが提供されます。











製品の最新バージョンには、多くの新機能が含まれています。 Veeam Availability Suite 9.5の主要な機能の1つは、MicrosoftデータセンターテクノロジーであるWindow Server 2016およびHyper-V 2016、およびエンタープライズアプリケーションとの完全な統合です。 また、仮想マシンのバックアップとリカバリの速度の向上、Veeamバックアップをクラウドに復元する機能、Nimble Storageハードウェアスナップショットとの統合のサポートに注目する価値があります。



新機能



Veeam Availability Suite 9.5は、拡張された拡張性を提供し、あらゆる規模の仮想環境を任意の数の仮想マシンでサポートできます。 新しいバージョンでは、生産性の大幅な向上に注目できます。1秒あたりのI / O操作の数が2倍になりました。 バックアップ時間は5倍に短縮されました。 開発者は、プライマリストレージとバックアップストレージおよびvCenterサーバーの負荷を軽減することもできました。 ディスクアレイの操作、ストレージシステムとテープデバイスの重複排除のロジックの最適化により、仮想マシンのリカバリ速度が大幅に向上し、許容リカバリ時間のRTOが低下しました。 さらに、構成データベースのサイズが縮小され、ユーザーインターフェイスが改善されました。



完全統合



既に述べたように、バージョン9.5の主要な革新は、Microsoftデータセンター製品の最新バージョンに対する完全に拡張されたサポートです。 そのため、Veeam Availability SuiteをWindows Server 2016の最新バージョンと統合すると、Veeam Backup＆Replicationコンポーネントをインストールし、サポートされているハイパーバイザーで実行されているゲスト仮想マシンを保護できます。 PowerShell Directテクノロジーを使用したおかげで、Hyper-VホストからゲストVMに直接接続することも可能になりました。以前は、このオプションはVMwareハイパーバイザーでのみ利用可能でした。







Microsoft Hyper-V 2016との統合により、Windows Server 2016（TP5）用の新しいバックアップスキームおよび仮想マシンハードウェアの新しいバージョン7.1のサポートが有効になりました。 Hyper-Vは、変更された情報の柔軟な追跡機能である復元力のある変更追跡に基づいて、変更されたデータブロックを追跡するメカニズムも導入しました。 これにより、独自の特別なVeeamドライバーを使用せずに、増分バックアップとリカバリを高速化できます。 これにより、サードパーティのSMB3ストレージで実行されている仮想マシンのサポートも追加されました。



Veeam Availability Suiteは、Nano Serverにある仮想マシンのバックアップと復元を完全にサポートします。NanoServerは、リモート管理サービスを提供し、クラウドで作成されたアプリケーションを実行し、Hyper-Vホストとして機能するWindows Serverの特別バージョンです。 記憶域スペースダイレクトアーキテクチャもサポートされているため、データセンターのデータ量を削減し、セキュリティレベルを向上させることができます。







さらに、Active Directory、Exchange、SharePoint、SQL ServerなどのMicrosoft企業アプリケーションの最新バージョンの新機能のサポートに注目する価値があります。



すべて復元



Veeam Availability Suiteの新しいバージョンのもう1つの大きな改善点は、以前はスタンドアロンツールであったMicrosoft AzureにDirect Restoreが含まれたことです。 Windowsベースの仮想マシン、Azure用の物理サーバーまたは端末を迅速に復元して中断を最小限に抑え、職場からクラウドへのスケジュールされたワークロードの移行を可能にしてローカルリソースの制限を軽減します。 また、Azureに基づいて、パッチおよび重要な更新中のアプリケーションのデプロイに関連するリスクを軽減できるテスト環境を作成できます。 Azureにバックアップを保存する機能を実装しました。 顧客はクラウドストレージへのバックアップを即座に活用し、ローカルバックアップアーカイブをAzureに自動的に移動できます。 このソリューションは、企業顧客に費用対効果の高いAzureストレージとVeeamの個別リカバリ機能を提供します。 バックアップを直接復元できるということは、その柔軟でスケーラブルなクラウドリソースのおかげで、Azureをオンデマンドでクラウドデータセンターとして使用できることを意味します。



Microsoft MarketplaceのDirect Restore機能は、Azure Marketplaceで利用可能な無料のツールを使用してアクティブ化されます。 また、以前にVeeam Cloud Connectを使用したことがあるお客様は、Azureに保存されているため、バックアップコピーフェーズをスキップし、バージョン9.5にアップグレードした後、既存のAzureアカウントから直接復元できます。



ハードウェアのスナップショット



Veeam Availability Suite 9.5は、Nimble Storageハードウェアスナップショットと統合します。 バックアッププロセスを最適化し、プライマリストレージとバックアップストレージの両方でこれらのスナップショットから仮想マシンデータを読み取ることができます。 これにより、メインストレージシステムの負荷を大幅に削減できます。 Nimble Storageのスナップショットを作成するスケジュールを計画することにより、プロセスを自動化できます。 また、スナップショットを使用すると、仮想マシン全体と個々のファイルとアプリケーションの両方の必要な情報を簡単に回復できます。







さらに、ハードウェアスナップショットを使用すると、数回クリックするだけで作業環境のコピーを作成できます。 同様のテスト環境を使用して、問題の迅速なトラブルシューティング、テスト、学習を行うことができます。



その他の機能



バージョン9.5のその他の技術革新の中でも、VMware vCloud Directorのサポートの改善に注目する価値があります。 これにより、顧客はインフラストラクチャをサービス（IaaS）モデルとして実装できます。 これにより、インフラストラクチャの管理が簡素化され、効率が向上し、プライバシーが向上し、ユーザーはデータとアプリケーションの自己バックアップと回復の可能性を得ることができます。



ReFSファイルシステムとの強化された統合により、最小限のストレージ要件での高速クローン作成とフルバックアップのテクノロジーにより、操作の可用性が向上します。 これにより、大量のデータを処理する際の生産性を向上させることができます。 さらに、重要なビジネスアプリケーションのバックアップは、損傷から確実に保護されます。
