今後の準備は何ですか

またはWindows Phoneに期待すること



TechEdの初日、私はWindows Phoneを開発している人々、Cliff SimpkinsとLarry Liebermanと密接にコミュニケーションを取ることができました。 私たちは他の会議参加者と一緒に、プラットフォームの将来について最も刺激的で刺激的な質問をしようとしました。 もちろん、私たちはセンセーショナルな答えや刺激的な自白を受け取りませんでしたが、個人的には、開発者のキャンプに広がる雰囲気について良いアイデアを得ました。 この考えを共有したいと思います。



まず、 msteched.ruというサイトを引用して、読者が自分で検索する必要がないようにします。



ラリー・リーバーマン

プロダクトマネージャー、Microsoft



Larry Liebermanは、Windows Phoneアプリケーションプラットフォームの製品マネージャーです。 Lariは、開発者、ラボ、ビデオ、スピーチの録音など、Windows Phoneのアプリケーションとゲームの開発に関するあらゆる種類の資料の作成と公開を調整します。Lariは、新しいMicrosoftモバイルプラットフォームに関する貴重な情報源であり、喜んで共有しますTechEd Russiaカンファレンスでのこの情報。 Lariは1998年からMicrosoftに勤務しており、2003年からモバイルプラットフォーム向けのアプリケーション開発を専門としてきました。 Lariは、今後開催される世界的に有名なBUILD 2012カンファレンス（ 2012年 10月30日-11月2日）のコンテンツの作成に参加し、TechEd RussiaでもWindows Phoneに関する多数のレポートを作成します。 彼の参加のおかげで、BUILDに参加しなかった人は、ロシアのTechEd Russiaで、このMicrosoftモバイルプラットフォームでのWindows Phoneとソフトウェア開発に関する重要な情報を聞くことができます。









クリフ・シンプキンス

Windows Phone製品開発者リレーションシップマネージャー、Microsoft。



Cliff Simpkinsは、Windows Phoneチームのプロダクトマネージャーであり、開発者とのやり取りを担当しています。 彼は、開発者からのフィードバックと提案の整理、およびさまざまな研究に関して、Windows Phone開発プラットフォームの管理と開発を担当しています。 世界中のモバイル開発者やパートナーとの直接的なコミュニケーションとやり取りに加えて、CliffはWindows Phone User Voice Webサイト （ http://wpdev.uservoice.com/ ）の運営と、関連する製品や技術に関する早期の習熟プログラムを担当しています。 Windows Phone Windows Phoneチームに参加する前（Windows Phone 7を開発中）、Cliffはプロダクトマネージャーとして働き、Windows Communication Foundation（WCF）とWindows Workflow Foundation（WF）の計画を担当していました。 Cliffは非常に明るい講演者であり、TechEd Russia 2012でWindows Phoneプラットフォームに関する多くの重要な技術的問題を強調します。









ビジネスへ

人々は非常に敏感でオープンだと言わなければなりません。 彼らとコミュニケーションをとることは、彼らが彼らの仕事を愛する専門家であることを理解しています。



会話はかなり混oticとしたもので、トピックからトピックにジャンプしました。 質問を少しグループ化してみます。 会話は内部支払いから始まりました。Cliffは、開発されたすべての機能が最新リリースに含まれているわけではないことを正直に認めました。たとえば、アプリケーション全体のサブスクライブに向けた作業が行われました。 また、次のSDKリリースでアクセスできるかどうかについて、定期刊行物や同様のアプリケーションの配布センターとしてのiOSのニューススタンドアナログについても質問しました。 これに対して、ラリーは、唯一の質問はサーバー側でのサービスの可用性であり、これはSDKのリリースに関係していないことを指摘しました。 （考えてみると、出版物を発行するサービスと電話のハブだけが必要です。他のすべては現在のAPIで実装されています）



バックグラウンドタスクに関して多くの質問がありました。 彼らは、バックグラウンドロケーションタスクが4時間以内で機能する理由について尋ねました。 ラリーは、この決定はユーザーの関心事であり、ユーザーが4時間実行しなかったアプリケーションはおそらくバッテリーを浪費していると答えました。 開発者には、ユーザーがアプリケーションを起動してタイマーをリセットすることを思い出させる機会があります。 しかし、ラリーは、決定が間違っている可能性が高く、開発者からのフィードバックを待っていると指摘しました。 （私自身、iOSでバックグラウンドロケーションを操作している間、同じ4時間で4番目のiPhoneを放電したことに注意してください）。 また、特定の時間または特定の期間後にライブタイルを更新する必要がある場合のタスクについても説明しました。 条件付きで、スケジュールタイルと呼びました。 ラリーは、彼らはこの機能について多くのことを言われたが、これまでにより重要なタスクがあったと言いました。



また、プラットフォームを2つのブランチ（7.xと8.x）に分岐することに関する質問もありました。 ラリーは興味深いアイデアを表明しました（以前は考えられなかった方法です）：現在のデバイスを新しいコアに移行するには、OEMに既存のデバイスのドライバーを書き換えるように強制する必要があります。 プラットフォーム間でユーザーエクスペリエンスをできる限り共通にしたいと考えていますが、これまでのところ、新しい開始画面のみが公式に発表されています。 （その後、ロック画面にライブ壁紙が表示され、他のいくつかの機能が既に発表されているため）



ClifとLarryは、uservoiceのフィードバックについても同様に語りました。 多くの理由で、彼らはすぐに答えて「開発中」のようなステータスを維持することはできませんが、タスクは定期的にチェックされると誓いました。 また、公式のWindows Phone開発者ブログを通じて開発者と連絡を取り合うようにしています 。 一般的に、クリフとラリーは積極的にフィードバックを呼びかけました。「開発者のすべてのリクエストを喜んで実装します。」 さらに、ラリーは、そのような迷惑な男がWindows Phoneの改善を望んでいると言って、マネージャーに私のメールを送りました。 確かに、これまでのところ私は答えを受け取っていません。
