Wi-Fiの近未来：802.11acの現状

近い将来、Wi-Fiは次の3つの興味深い改善を手に入れる予定です。



  • WLANの速度と容量を増加させる手段としての802.11ac （別名「ギガビットWi-Fi」）
  • 802.11u仕様に基づくWi-Fi認定パスポイント （別名ホットスポット2.0）。ホットスポットの状況を大幅に改善し、セルラーネットワークを解放する方法として。
  • Wi-Fi Certified Voice Enterprise ：音声およびビデオ用の高性能ネットワークを構築して状況をクリーンアップする方法として。 現在、優れたインフラストラクチャを構築できますが、クライアントデバイスの厳密な標準がないため、ソリューションの全体的なパフォーマンスに問題が生じます。


それらのすべては、何らかの形ですでに実際の製品で利用可能であり、非常に近い将来に巨大になる可能性があります。 このセクションでは、802.11acについて説明します。 私はすでにこのテクノロジー全体のレビューを書いており、 パフォーマンスの比較も行ったため、適用される側面に焦点を当てています。 一般に、802.11acの標準としての採用は2014年初頭に予定されており、一部の企業が昨年の夏にチップセットを発表するのを止めませんでした。 この技術は、「波」と呼ばれるいくつかの段階で市場に参入します。

  • Wave 1を使用すると、「生の」データ転送速度（3x3で最大1300 Mbps、チャネル幅80 MHzの3 MIMOおよび新しい256-QAM変調）のギガビット障壁を突破できます。 一般に、同じ802.11nですが、より高速です。
  • Wave 2は、速度（4x4：4 MIMO / 160MHz〜= 3500Mbps）と容量-MU-MIMOの点でさらに改善されます。
  • 後続の波は、オプションと見なされる、または迅速な実装には複雑すぎる（タイプ8x8：8 MIMO）他のイノベーションをもたらす可能性があります。 または、まったく提供しない場合があります-802.11n規格で説明されている技術の多くは、実際の製品には実装されていません。


2013年5月現在のWave 1製品は、消費者レベルとビジネスレベルの両方のかなり多くのメーカーによってすでに発表されています： Aruba 、 Cisco 、 Motorola Solutions 、 Ubiquiti 、ASUS、D-Link、Linksys、NetGear、TP-Link およびall-allそれだけです。 それらのいくつかはまだ発表段階でのみ存在し、いくつかはすでに購入することができます。 しかし、それだけの価値はありますか？ このギガビットWi-Fiの長所と短所を見て、理解してみましょう。











転送速度の増加。 ここではすべてが明確です-ネットワークは高速で、誰もが幸せそうです。 カスタマーサポートが必要であることを忘れないでください。 Wave1機器で「ギガビットバリアを突破する」ために必要なものを見てみましょう。 遠くに行かないように、ウィキペディアのスピードプレートに目を向けます

  • チャネル幅が20MHzの場合、ストリームごとの最大速度は〜87Mbpsです。つまり、現在の制限が3ストリームの場合、ギガビットは表示されません。
  • チャネル幅が40MHzの場合、ストリームごとの最大速度は〜200Mbpsです。つまり、現在の制限が3ストリームの場合、ギガビットは表示されません。 しかし、802.11nと比較して、鉄を交換するだけで3分の1（600/450）加速しました。
  • 80MHzのチャネル幅では、ストリームごとの最大速度は〜433Mbpsであり、3つのストリームすべてを使用する必要があることを意味します。 これにより、カバレッジプランにかなり厳しい要件が課せられます：80MHzのチャネルを提供する必要があります（隣接するセルで重複しないので、少なくとも4つ必要ですが、どこにあるのでしょうか？）


80MHzチャネルとポイントの正しい配置であっても、ギガビット速度に依存するべきではありません。 ほとんどのコンパクトクライアントは、3つのストリームをサポートしません。 スマートフォンは通常、1ストリーム、タブレット-1-2スレッド、ラップトップ-2-3スレッドをサポートします。 これは、ネットワークを計画する際に覚えておくことが重要であり、顧客が単にサポートできない速度を追いかけることではありません。 ただし、Wave2とMU-MIMOを使用すると、低スレッドクライアントがより効率的に処理されます。 4SSのポイントが、1SSの4つのクライアント（それぞれダウンストリーム）または1 + 2 + 1などの組み合わせで3つのクライアントに同時に送信できるとしましょう。 これにより、通信時間の使用効率が大幅に向上し、ネットワーク容量が増加します。 以下について。



ラジオの感度が向上しました。 新しいチップセット（ポイントとクライアントの両方）の感度は優れているため、256-QAMが正常に動作する（および高速化する）ためにコーティングの密度を上げる必要はないと考えられています。 これは、MIMOにはまったく適用されないことに注意してください。また、超安価なクライアントデバイスのメーカーの品格に頼るべきではありません。



重要なニュアンスは、ポイントの有線接続でもあります。 802.11nポイントは、1つのGEリンクで十分な2つの無線（3x3：3 / 40Mhz）から最大900Mbpsを「圧縮」できます。 802.11ac Wave 1ポイントは、理論的には2x1300Mbps = 2.6Gbpsを絞り出すことができます。これに対して、3つのGEリンクでも十分ではありません（Wave2は一般に7Gbpsです）。 すべてを10GEにアップグレードしますか？ かなり高価になります。 幸いなことに（少なくともWave 1の場合）、これは必要ありません。 イーサネット帯域幅は、プロトコル自体のオーバーヘッド（「ネット」）を考慮して歴史的に測定され、Wi-Fi-なし（「グロス」）であることに留意することが重要です。 したがって、理論上の「生の」2600Mbpsを2つのGEリンクに圧縮することは非常に現実的です。 発表された802.11ac Wave 1ポイントのほとんどには2つのGEポートがあり、これで十分であると考えられます。 確かに、スイッチの追加ポートと2番目のケーブルの敷設のために追加料金を支払う必要があります（通常、最初のケーブルの敷設コストの約50％）。 繰り返しますが、BYOD用のネットワークを構築する場合（つまり、クライアントが1〜2スレッドをサポートする場合）、1ギガバイトで十分です。



ネットワーク容量の増加。 速度を上げることの有用な副作用は、同じ量のデータの転送にかかる時間が短くなることです:)したがって、半二重セルの容量が増加します。 アクセスポイントに多数の802.11n 1x1：1 / 40MHz（150Mbps）クライアントがあり、それぞれが10Mbpsの帯域幅を必要としているとします。 計算の便宜上、実効セルスループットは理論値の0.6 = 90Mbpsであると想定しています。 このように、半二重通信では、セルは9つのクライアントをサポートできます。 802.11acにアップグレードして、チャネル幅とストリーム数を保持すると、256-QAMによりセルで理論的な200Mbpsを取得でき、容量を200 * 0.6 / 10 = 12クライアントに増やすことができます。 単に鉄の交換により、30％増加しています。 ちなみに、セルの容量は、2x2：2または3x3：3ポイントがあるかどうかに依存しません。 1x1：1クライアントの半二重伝送では、残りのストリームは使用されません。 しかし、同じシナリオでWave2とMU-MIMOを使用すると、12個のクライアントを36個または48個に簡単に変換できます。8x8の場合：8-通常96！

  • 一方で：速度と容量の増加
  • 一方、速度が常に達成されるとは限らず、クライアントとポイントの交換が必要です（どちらもより高価になります）、有線部分のアップグレードが必要です。Wave2が終了すると、購入したものすべてを再度変更する必要があります。


要因を比較することにより、802.11acへの移行が有益な場合とそうでない場合を理解できます。





最初の要因は純粋に経済的です。 速度と容量の増加（私たちが把握したことを知った）が追加コスト（実際に見積もることができる）を本当に正当化する場合-移行は理にかなっています。



2番目の要因は戦略的です。 今すぐWave1を購入するか、Wave2を待つか？ 1か月前、Wave2を待つことはほとんど価値がないと思います。 機器は2015年にすでにリリースされる予定です。しかし、先週、Wave2 4x4：4 MU-MIMOチップセットが発表されました。

最初の802.11acチップセットの発表後、最初のWave1製品が消費者部門で6か月以内に登場し、企業部門では1年足らずで登場したことを考慮すると、来年のこの時点ですでにWave2を完全にテストする可能性があります。 したがって、緊急の必要がない場合は、Wave2に直接ジャンプする機会を検討する価値があります。

繰り返しますが、4x4：4 MIMOはまだ起動する必要があり、MU-MIMOは比較的低電力のクライアントにのみ意味があり、有線インフラストラクチャをアップグレードする必要があります。 ラップトップ用のネットワークを構築したい場合、または今すぐワイヤレスで取り外したい場合は、Wave1が最適です。
