UbuntuでSSHを介したログインを100倍高速化

長い間、Ubuntuサーバーへのリモートログインが非常に遅いことに悩まされていました（すべて2 ） この場合、非常に遅くなります-2-3秒。 それは完全に致命的ではなかったわけではありませんが、キーを介してログインするとき-すべてが即座に動作するようにしたい-結局、486SXがありません。



長時間のグーグルは、標準ソリューションのみにつながりました。

  1. / etc / ssh / sshd_configにUseDNS no-DNSが遅い場合にログインを高速化します。 私の場合、ローカルキャッシュDNSがあったため、すべてが高速でした。
  2. SSHクライアントでのIPv4の強制はあまり役に立たなかった、明らかに誰もが遅くなるわけではない
そして、 askubuntu.comで、賞金についての質問は、何も答えがなかったので悲しいことに何週間も遅れました...





私たちの祖父がUbuntuの本当に便利なタスク/ etc / motdに使用したファイルは非常に「創造的に」使用されていることがわかりましたその他）。 「update-motd --disable」で無効にします。



10番目のバージョンでは、それほど単純には機能しません。 motdの生成はPAMモジュールに転送されました。PAMモジュールはログインの瞬間にすぐに実行され、非常に貴重な2〜3秒の時間を消費します。 だから、メスを取る：



  1. /etc/pam.d/loginおよび/etc/pam.d/sshdファイルで、「session optional pam_motd.so」という行を削除します
  2. デフォルトでインストールされている有料モニタリングのコンポーネントを破壊します。

    aptitudeは、landscape-client landscape-commonを削除します




その後、最終的に好みに合わせて編集することができます/ etc / motd



/ etc / ssh / sshd_configで、まだ必要な場合は「PrintMotd yes」があるかどうかを確認します。



完了、サーバーへのログインはすぐになりました:-)
