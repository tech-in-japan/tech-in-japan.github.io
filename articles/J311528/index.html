3CX V15でのIP電話の自動調整の機能

3CX v15のリリース後、多くの同様の質問がIP電話の自動設定に関連するパートナーおよびユーザーから寄せられています。 それらは2つになります。







  • 自動チューニングファイルは作成されません。 ディスク上の対応するフォルダーにない
  • 自動調整ファイルには無関係な情報が含まれています。 管理者が3CXコンソールで行った変更は反映されません。


IP電話の自動調整ファイル



電話機の自動調整の概念に出会ったことがない場合は、説明してみましょう。自動調整を使用すると、3CXでのトラブルのない運用に推奨される最適なパラメータでユーザーのIP電話を自動的に構成できます。 正しいパラメーターに加えて、内部（内線）番号、認証名とパスワード、およびその他の個人ユーザーデータが電話に転送されます。 もちろん、自動調整ファイルは、サポートされているIP電話と3CXソフトウェアクライアントに対してのみ作成されます。



自動調整ファイルはプレーンテキストファイルです。 通常、ファイル名は、このファイルが生成されるIP電話のMACアドレスです。 ファイルは、3CXインターフェースで表示および変更できるテンプレートファイルに基づいて生成されます。 自動構成ファイルは静的であり、管理者が3CXインターフェースの拡張設定を変更した場合にのみ更新されます。たとえば、BLFボタンを定義したり、ユーザー名を変更したりします。



原則として、IP電話は、24時間ごとに1回、自動構成ファイルをアップロード（構成を「更新」）します。 待機して変更をすぐに適用したくない場合は、インターフェースで電話を選択し、[ 再プロビジョニング ]をクリックします。 3CXは、ファイルをすぐに受信するように電話に指示します。 電話が再起動する可能性があることに注意してください。









3CX v15の変更



3CXの新しいバージョンでは、自動構成フォルダー（WindowsのデフォルトパスはC：\ ProgramData \ 3CX \ Instance1 \ Data \ Http \ Interface \ provisioning \ <ランダム文字列> \ ）には、電話のファイルは表示されません！ 以前のバージョンからv15にアップグレードした場合、古い自動調整ファイルは残りますが、更新されません。 現在、3CXはオンデマンドで動的に電話機のファイルを生成します。 これにより、2つの問題が同時に解決されます。



  • ファイルはサーバーディスクに保存されないため、誰もアクセスできません。 たとえば、PBXがデータセンターにある場合、サーバーファイルシステムにアクセスできるのは誰であるかを知ることはできません。 これで、ユーザー設定（パスワードを含む）は3CX管理者のみが使用できます。
  • 生成された自動調整ファイルには、常に現在のデータが含まれます。 以前のバージョンでは、設定を変更したときにファイルがすぐに更新されなかったため、電話の艦隊を維持するのがさらに難しくなりました。


自動構成フォルダーの内容を以下に示します：一部の電話に必要ないくつかの一般的なファイル、ファームウェアファイル、ロゴ（電話としてダウンロード可能）、ユーザーテンプレート（つまり、3CX管理者によって何らかの目的で変更された電話テンプレート）および3CX企業電話帳サポートされているIP電話用。









電話の自動調整ファイルの内容を表示する方法は？



ファイルがローカルに保存されていないので、合理的な質問が発生します-それらのコンテンツをどのように見ることができますか（つまり、何が電話に直接ダウンロードされますか）？ 3CX v15では、[電話]セクションで電話を選択し、[ 構成 ]ボタンをクリックするだけです。 現在の設定でファイルの内容が表示されます。









注意してください！ 自動調整テンプレートからのデータは、システムサービスの開始時にキャッシュされます。 したがって、テンプレートに変更を加えた（つまり、カスタムテンプレートを作成した）場合は、 管理コンソールサービス（3CXMC01）を再起動します。



3CXテクニカルサポートは、変更されていない自動チューニングテンプレートを使用する構成にのみ適用されることに注意してください！



電話の自動調整の詳細については、 3CX管理者ガイドを参照してください。
