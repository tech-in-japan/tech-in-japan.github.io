Tiled2Unity：Tiled Map EditorからUnityへのエクスポート

ゲーム開発者向けのリソースに関する多くの出版物は、単純な2Dプラットフォーマーの作成に専念しています。 多くの場合、Unityはエンジンとして使用され、Tiled Map Editorはマップの作成に使用されます。 これら2つの環境の相互作用は、TMEファイルからUnityのプレハブを作成する無料のユーティリティTiled2Unityを使用して実行されます。



仕組みについては、プラットフォーマー「X-Drums」の開発例を紹介します。



次のスキームに従って移動します。



  1. Tiled2Unityエクスポートのセットアップ
  2. タイルマップエディターで衝突を構成する
  3. Tiled2Unityがサポートする設定を構成する
  4. タイルマップエディターの設定をカスタマイズする


  • レイヤー（オブジェクト）のユーザーパラメーター
  • オブジェクトのカスタムオプション


1.エクスポート設定



したがって、Unityにプロジェクトがあり、Tiled Map Editorで作成されたマップがあります。 Unityを起動し、 Tiled2Unity.unitypackageをインポートします 。







タイルマップエディターで、コマンド編集ウィンドウを呼び出します。







実行可能ファイルTiled2Unity.exeへのパスが登録されている新しいコマンドを追加します。 代わりに、％mapfile Tiledは、マップファイルへのパスを自動的に置き換えます。 このコマンドの最後の引数は、Tiled2Unityフォルダーへのパスです。これは、インポート後にプロジェクトに表示されるはずです。







コマンドを実行し、表示されるウィンドウで大きなボタンをクリックします。







エクスポートの結果として、Tiled2Unityフォルダーにあるカードのプレハブを取得します。 このプレハブをUnityのステージに配置します。 現在、TMEのマップとそのエクスポートを操作するとき、シーン上のプレハブとオブジェクトは、TMEの変更に従って変更されます。







そこで、TiledからUnityへのマップのエクスポートをセットアップし、ゲーム環境を作成しました。



それでは、マップ上にオブジェクトを設定しましょう。



2.衝突エディター



まず、ヒーローにとって障害となるゲーム空間の要素を定義します。 これらは、床、壁、天井のスポットライト、そしてヒーローがジャンプするプラットフォームとして機能するドラムです。



Tiledでコリジョンエディターを起動します。







ドラムの一部など、必要なタイルを選択し、ポリゴンを編集します。







同様に、必要な他のタイル（床、壁など）を編集します。 その結果、エクスポート時に、Tiled2Unityは自動的にPolygonCollider2Dコライダーを作成します。これはヒーローの障害物として機能します。











ただし、ヒーローが対話できる他のゲームオブジェクトはより複雑です。 シンバル、女性ファン、ノートなどの要素の動作は、コンポーネントとスクリプトを使用して制御されます。 このようなオブジェクトをエクスポートするには、Tiledで追加のパラメーターを指定し、Unityで処理する必要があります。



3. Tiled2Unityがサポートするパラメーター



ただし、Tiled2Unityが既にこれを行っているため、Unityで設定せずにTiledで使用できるパラメーターのリストがあります。 それらの中には：

主人公が飛び越すドラムスティックに必要なパラメーターを設定します。







Unityにエクスポートすると、次の結果が得られます。







レイヤーはOneWayPlatformパラメーターに設定され、下から透過可能なオブジェクトに使用されます。







4. Tiledのユーザーオプション



他のパラメータの処理を設定し、それらを「カスタム」と呼び、Unityでそのようなパラメータを持つオブジェクトをエクスポートします。 レイヤー（オブジェクト）とオブジェクトの両方に対して、タイルで設定できます。 場合によっては、最初または2番目のオプションが便利です。 これを実装する方法を検討してください。



4.1。 レイヤー（オブジェクト）のユーザーパラメーター



レイヤーのカスタムパラメーターを設定します。 このオプションは、同じタイプのオブジェクトに使用します。 私たちのゲームでは、これらはシンバルとノートです。 彼らの特徴は、彼らがヒーローと平等に相互作用し、お互いを区別する必要がないことです。



Tiledでオブジェクトのレイヤーを作成し、プレートを配置します。 次に、レイヤー全体にAddObjectパラメーターを設定します。







各プレートのパラメーターのフィールドは空のままです。







次に、Unityに移動します。ここでは、プレハブを作成し、ヒーローがジャンプしたときの動作を決定するスクリプトをそれにプレハブを添付します。ヒーローは落下して元の位置に戻ります。



Tiledから各オブジェクトのプレートをインポートするときに、プレハブのインスタンスを作成するコードを記述します。 これを行うには、Tiled2Unity.ICustomTiledImporterインターフェイスと[Tiled2Unity.CustomTiledImporter]属性を継承するクラスでCustomTiledImporterForHiHatsスクリプトを作成します。



このTiled2Unityスクリプトは、Tiledからインポートされた各レイヤーとオブジェクトを呼び出します。 そして、それぞれ、パラメーターの中に「Hi-hat」という値を持つAddObjectパラメーターを持つレイヤーまたはオブジェクトが処理されます。 



using System.Collections.Generic; using UnityEditor; using UnityEngine; [Tiled2Unity.CustomTiledImporter] class CustomTiledImporterForHiHats : Tiled2Unity.ICustomTiledImporter { public void HandleCustomProperties(GameObject gameObject, IDictionary<string, string> props) { if (!props.ContainsKey("AddObject")) return; if (props["AddObject"] == "Hi-hat") { //   string prefabPath = "Assets/8 - Prefabs/Hi-hat.prefab"; Object obj = AssetDatabase.LoadAssetAtPath(prefabPath, typeof(GameObject)); if (obj == null) return; //  ,     Tiled, //    //      List<GameObject> childrenToDestroy = new List<GameObject>(); //     foreach (Transform child in gameObject.transform) { if (child.name == "TileObject") { //     childrenToDestroy.Add(child.gameObject); //  ,      GameObject objInstance = (GameObject)GameObject.Instantiate(obj); objInstance.name = obj.name; objInstance.transform.parent = gameObject.transform; objInstance.transform.position = child.transform.position; } } //    childrenToDestroy.ForEach(child => GameObject.DestroyImmediate(child)); } } public void CustomizePrefab(GameObject prefab) { } }


スクリプトの準備が完了し、プレートをエクスポートできます。







そして、主人公は彼らと対話します。







ノートのエクスポートは、シンバルのエクスポートと同様に設定されます。 唯一の違いは、ゲームが色に応じて異なる値を持ついくつかのタイプのノートを持っていることです。 そのため、種類ごとのノートの種類ごとに、Unityでプレハブを作成し、Tiledでオブジェクトレイヤーを作成して、AddObjectパラメーターでノートの種類を示しました。 プレハブインスタンスを作成するインポートスクリプトは、シンバルスクリプトと同じです。



4.2。 オブジェクトのカスタムオプション



次に、同じタイプのオブジェクトをエクスポートする方法を検討します。ユーザーパラメータは、レイヤーではなくオブジェクト自体によって設定されます。 ファン、キューピッド、移動プラットフォームなどをこのようなオブジェクトとして分類します。 同じタイプにもかかわらず、速度、方向などの独自の個別のプロパティがあります。



たとえば、ファンは色が異なり、一定の距離で異なる速度で動きます。 彼らは瞬きをやめることができ、ヒーローと会うとき、彼らは選択するために特定の感情を出します。 これらのパラメーターは、オブジェクトごとにタイルで設定します。 レイヤーの「独自のパラメーター」はそれぞれ空のままです。







Unityでは、GirlControllerスクリプトが添付されている女性ファン用にプレハブが作成され、これがヒーローに対する行動と反応を決定します。 インポートはプレートのインポートに似ていますが、この場合はプレハブインスタンスを作成するだけでは不十分です。 GirlControllerスクリプトを使用して、Tiledで各ファンに個別に設定されたプロパティを設定する必要があります。 



 using System.Collections.Generic; using UnityEditor; using UnityEngine; [Tiled2Unity.CustomTiledImporter] class CustomTiledImporterForGirls : Tiled2Unity.ICustomTiledImporter { public void HandleCustomProperties(GameObject gameObject, IDictionary<string, string> props) { if (!props.ContainsKey("GirlColor")) return; //        Tiled  string prefabPath = "Assets/8 - Prefabs/Girls/Girl_" + props["GirlColor"] + ".prefab"; Object obj = AssetDatabase.LoadAssetAtPath(prefabPath, typeof(GameObject)); //  ,      GameObject objInstance = (GameObject)Object.Instantiate(obj); objInstance.name = obj.name; objInstance.transform.parent = gameObject.transform.parent; objInstance.transform.position = gameObject.transform.position; objInstance.transform.localScale = gameObject.transform.localScale; objInstance.transform.eulerAngles = gameObject.transform.eulerAngles; //  ,     //   Tiled     var girlController = objInstance.GetComponent<GirlController>(); if (props.ContainsKey("OffsetRight")) float.TryParse(props["OffsetRight"], out girlController.OffsetMax); if (props.ContainsKey("OffsetLeft")) float.TryParse(props["OffsetLeft"], out girlController.OffsetMin); if (props.ContainsKey("Velocity")) float.TryParse(props["Velocity"], out girlController.Velocity); if (props.ContainsKey("Stop")) float.TryParse(props["Stop"], out girlController.StopCount); if (props.ContainsKey("Fatal")) bool.TryParse(props["Fatal"], out girlController.Fatal); if (props.ContainsKey("HeartType")) { string heartPrefabPath = "Assets/8 - Prefabs/Girls/" + props["HeartType"] + ".prefab"; Object heartObj = AssetDatabase.LoadAssetAtPath(heartPrefabPath, typeof(GameObject)); girlController.heartPrefab = heartObj; } //    Object.DestroyImmediate(gameObject); } public void CustomizePrefab(GameObject prefab) { } }


Unityにエクスポートし、Tiledで指定されたパラメーターがGirlControllerスクリプト変数によって設定されたことを確認します。











他のグループのオブジェクトについて説明したスキームを繰り返し、Unityにエクスポートするときに、ゲームのほとんどのコンポーネントの構成済みプレハブを取得します。



説明したアプローチは、プラットフォーマーを作成するために使用されました。 この特定のゲームでは、Tiled Map EditorおよびUnityと連携して作業することにより、作成プロセスが加速され、TMEのレベルデザイナーとUnityのプログラマーの両方が互いに干渉することなく同時に作業できるようになりました。 私たちのようなプロジェクトでは、Tiled2Unityユーティリティは非常に便利であり、記事に基づいて、ゲーム開発で任意のオプションを使用できます。
