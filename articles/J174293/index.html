新しいVeeam Management Pack 6.0でSystem CenterとVMwareのギャップを埋める

System Centerを使用している場合は、おそらくITインフラストラクチャの中心でそれを見たいと思うでしょう。 インフラストラクチャ全体の中心。 いくつかの監視ツールを使用し、同様のことを専門家に何度もトレーニングするのは不便です。 このような状況では、特に仮想環境に関しては、適切なソリューションを見つけるのが難しい場合があります。



Windows Server 2012に含まれるHyper-Vの最新バージョンは重大な一歩を踏み出しましたが、VMwareは引き続き市場リーダーであり、正常に機能するインストールが多数あります。



System Centerはここにありますか？



System Centerには、さまざまなハイパーバイザーを操作するための多くの便利な機能が含まれています。OperationsManagerは仮想マシンとゲストOSおよびそれらのアプリケーションを監視でき、OrchestratorにはVMwareとの統合パッケージがあり、Virtual Machine Managerはさまざまなアクションを作成および移動できますVMware仮想マシン上。



それでも、System Centerには「空白」があります。SCOMは、VMwareを実行しているサーバーのハイパーバイザーとハードウェアを監視できません。 この場合、別の監視ツールを使用するか、System Centerへの投資を最大限に活用してください。 これは、Veeam Management Packが役立つものです。



Veeam MPとは何ですか？



Veeam MPは、System CenterのVMwareホストのヘルスステータスとヘルスパラメーターを表示する機能を追加するSCOM用の管理パックです。







長年にわたり、Veeam MPの主な機能は、VMwareのSCOMインフラストラクチャに高度な監視機能を提供することでした。ハードウェア、ストレージ、およびネットワーク接続リソースは、200種類のメトリックで監視されます。 このソリューションには、事前に設定されたアラート、観測されたメトリックの推奨最適値、およびほとんどの問題に対する簡単で迅速なソリューションのための広範なナレッジベースも付属しています。 これにより、OrchestratorまたはSCVMMから必要なコマンドを呼び出すために必要な情報が提供されます。 これにより、System Centerへの投資を節約し、増やすことができます。



しかし、それだけではありません！



4月2日に、Veeam MP 6.0が発表され、新しいレポート機能と意思決定のヒントが提供されます！ Veeam Management Packは、System Center 2012以前のバージョンで利用可能なすべてのレポート機能を使用して、仮想インフラストラクチャを分析するためのさらに多くの方法を提供します。



まず、新しいバージョンではSystem Centerに成長計画機能が追加されます。 現在の負荷の増加に基づいて、いつ別のサーバーを追加する必要がありますか？ ストレージリソースはいつ使い果たされますか？ サーバーのプロセッサまたはメモリリソースは、どのような条件下で完全に使用されますか？ これらは、Veeam MPがシンプルなダッシュボードとスマートレポートを提供して、ITインフラストラクチャの開発を予測するための質問です。



分析レポートとダッシュボードは、バージョン6.0のもう1つの新しい要素であり、インフラストラクチャの状態を完全に理解するのに役立ちます。 どの仮想マシンが最も問題を引き起こしますか？ 不当に多くのリソースが割り当てられたマシンはありますか？ 常に過負荷のVMがありますか？ ストレージシステムへのアクセスの遅延と、処理されるI / O操作の数はどのくらいですか？ コンピューターを使用するためのトップ10マシンのリストはどのように見えますか？ SCOMコンソールを離れることなく、これらすべてを学ぶことができます。



5.7に存在するものに加えて、ストレージシステムとコンピューティングリソースの新しいトポロジマップが登場しました。 Veeam MPを使用すると、インフラストラクチャを常に制御し、リソースの最適な使用を常に監視し、潜在的な問題を予測できます。
