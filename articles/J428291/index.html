CorutinをサポートするKotlin 1.3、Kotlin / Native betaなど

先日、Kotlin 1.3がリリースされ、関連する一連のライブラリ、アセンブリツール、およびトレーニング資料が提供されました。



Kotlinは、あらゆるプラットフォームであらゆる規模のアプリケーションを開発するためのツールです。 Kotlin 1.3では、コルーチンのおかげで、ノンブロッキング非同期コードの読み取りと書き込みが簡単で便利になりました。 スケーラビリティがはるかに手頃な価格になりました。 新しいリリースでは、Kotlinコードを直接マシンコードにコンパイルできるKotlin / Nativeのベータ版もリリースしました。 Kotlinマルチプラットフォーム機能は、サポートされているすべてのプラットフォームで機能するようになりました。これにより、たとえばAndroidとiOSのアプリケーションで同じビジネスロジックを使用できます。 共通ロジックは、サーバー、モバイルおよびWebクライアントにも使用できます。 さらに、日常的なタスクのほとんどを引き受ける、簡単に移植可能なマルチプラットフォームライブラリを作成できるようになりました。







近い将来、バージョン1.3の主な機能に関する一連のウェビナーを実施します 。 こちらからサインアップしてください 。



コミュニティとエコシステム



今年、Kotlinエコシステムは前例のない速度で成長しました。 2018年1月以降、約150万人のユーザーがKotlinコードを編集しました-昨年と比較して、その数は2倍以上になりました。 StackOverflowおよび公開Slackのナレッジベースも、印象的なペースで成長しています。 そして、Kotlinコミュニティがとても反応が良く、歓迎してくれることを非常に嬉しく思います。



コトリン周辺のエコシステムが成長し、発展する方法が大好きです。 Kotlinは、Androidプラットフォームでの公式サポートは言うまでもなく、 Google Cloud Platform 、 Spring Framework 、 Gradleとの友達です 。 オープンソースコミュニティは、 RxKotlin 、 mockito-kotlin 、 TornadoFX 、 Kodein、 ΛRROWなどの優れたライブラリを作成します。 SquareのOkioやLibreOfficeなどのプロジェクトは、すでにKotlinに移行しているか、移行を計画しています。 私たちは、誰もが自分のアイデアとプロジェクトでKotlinエコシステムに参加することを勧めます。



JetBrainsの従業員がKotlin 1.3に貢献しただけでなく、プルリクエスト、バグレポートを作成し、私たちとのあらゆる種類のフィードバックを維持した多くの人々も貢献しました。 Kotlinの開発にご協力いただきありがとうございます！



コルーチンは公式に安定しています



コルーチンは、理解しやすく、変更しやすい、ノンブロッキング非同期コードを記述するための最新の方法です。 この強力なツールは、計算負荷をバックグラウンドワーカー（ワーカー）にシフトすることから、複雑なネットワークプロトコルの実装に至るまで、さまざまな目的に使用できます。 このリリースに含まれるバージョン1.0のkotlinx.coroutinesライブラリは、構成、キャンセル、例外処理、ユーザーインターフェイスに関連するケースなど、あらゆる規模の非同期タスクを管理するための強固な基盤を提供します。



今日kotl.in/coroutinesで始めましょう！



Kotlin /ネイティブベータ



Kotlin / NativeはLLVMを使用して、iOS、Linux、Windows、Mac、さらにはWebAssemblyやSTM32などの組み込みシステムを含むさまざまなオペレーティングプラットフォームとプロセッサアーキテクチャ用のスタンドアロンバイナリ（仮想マシン不要）にKotlinソースコードをコンパイルします。 Kotlin / Nativeは、完全に自動化されたメモリ管理をサポートし、CおよびObjective-C（およびSwift）との対話を提供し、Core FoundationやPOSIXなどのAPIプラットフォーム、および開発者が必要とする可能性のあるネイティブライブラリを操作できます。



Kotlin / Nativeランタイムは、不変データの使用を推奨し、スレッド間で保護されていない可変状態を渡す試みを禁止します。 実際、Kotlin / Nativeの観点からは、スレッドは存在せず、実装の低レベルの側面として抽象化され、ワー​​カー（ワーカー）に置き換えられます-これは、並列処理を実装する安全で便利な方法です。



kotl.in/nativeで Kotlin / Native を知ることができます 。



マルチプラットフォームのプロジェクトとツール



Kotlinにはすべてのプラットフォームで作業するタスクが明示的に与えられていますが、それはより重要な目標、つまり異なるプラットフォームで同じコードを使用するための前提条件であると考えています。 JVM、Android、JavaScript、およびネイティブのサポートにより、Kotlinは最新のアプリケーションのコンポーネントと連携できます。 これにより、コードと既存の開発を再利用する貴重な機会が得られます。結局のところ、同じコードを2回以上実装するのではなく、本当に複雑な問題の解決に集中する方がはるかに優れています。 Kotlinのマルチプラットフォーム機能は実験的なままですが、バージョン1.3は大きな前進です。



Kotlin 1.3には、たとえば、 HTTP 、 シリアル化、 コルーチン管理など、すべての日常タスク用のマルチプラットフォームライブラリのセットが付属しています 。 このようなライブラリを使用することが、マルチプラットフォームコードを記述する最も簡単な方法です。 さらに、特定のプラットフォームの機能を単一のAPIで抽象化する独自のマルチプラットフォームライブラリを作成できます。



今日、さまざまなプラットフォームでコードの再利用を開始します： kotl.in/multiplatform 。



Kotlin / NativeおよびMultiplatform用のツール



Kotlin 1.3は、IntelliJ IDEA Community Edition、IntelliJ IDEA Ultimate、およびAndroid Studio開発環境で、Kotlin / Native開発ツールとマルチプラットフォームプロジェクトを提供します。 3つのIDEすべてで、エラーの強調表示、補完、ナビゲーション、リファクタリングなど、すべてのコード編集機能を使用できます。 将来的には、より複雑な機能の実装と商用ツールとの統合に取り組みます。



Ktor 1.0ベータ



Ktor-コルーチンに基づく完全な非同期性を備えたHTTPスタック全体を実装する相互接続アプリケーション用のフレームワークは、すでにベータ版であり、 ktor.ioで入手できます。



その他の改善



上記のすべてに加えて、このリリースでは他の多くの機能と改善点が登場しました。



  • パフォーマンスと型の安全性を向上させるためのインラインクラスの実験的サポート。
  • バイトおよびその他の低レベル操作のより便利な操作のための符号なし整数の実験的サポート。
  • 重複するコードを減らすために、whenステートメントでサブジェクトを指す。
  • ライブラリー呼び出しの静的分析を改善するための契約 。
  • 初心者開発者の利便性のための引数のないエントリポイント 。
  • 遅延計算を視覚化するためのシーケンスデバッガー 。
  • スクラッチファイル、REPL、およびスクリプトの機能強化により、スクリプトおよびコードスニペットの操作が容易になります。
  • 標準ライブラリでは、他の改善とバグ修正の中でも、Kotlin / Nativeサポートが登場し、Kotlin / JSサポートが改善され、すべてのプラットフォームでRandomが獲得されました。
  • Kotlin / JSのインクリメンタルコンパイルの実験的サポート。

アップグレード方法



いつものように、Kotlinはplay.kotl.inでオンラインで試すことができます。



  • Maven、Gradle、およびnpm：コンパイラーおよび標準ライブラリーにはバージョン番号1.3.0を使用します。 ドキュメントはこちらです。
  • IntelliJ IDEA：Kotlin 1.3はバージョン2018.3に含まれます。以前のリリースでは、Kotlinプラグインをバージョン1.3にインストールまたは更新するだけで十分です。
  • Android Studio：プラグインマネージャーを使用してプラグインをインストールまたは削除できます。
  • Eclipse：プラグインはMarketplaceからインストールできます。
  • コマンドラインのコンパイラは、 Githubのリリースページからダウンロードできます。
