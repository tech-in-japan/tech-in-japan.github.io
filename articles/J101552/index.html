無料の過去の商用ソフトウェア

オープンソースソフトウェアに基づく商用プロジェクトの多くのオープンソース支持者は否定的です。 彼らの目には、フリーソフトウェアをわずかに修正し、それを閉じて販売する会社は、オープンソースソフトウェアコミュニティの利益のための寄生虫です。 Linuxコミュニティの別の部分は、そのような活動に対して何もしませんが、同時に完全にオープンで無料のソリューションを使用することを好みます。

真実に近い視点は何ですか？ そして、誰がそのような独自のフリーソフトウェアを本当に必要としているのでしょうか？

最初に、オープンソースソフトウェアに基づいた既存のロシアの商用プロジェクトをレビューします。 このリストはおそらく網羅的なものではありませんが、オープンソースの商用化の実例がいくつか含まれています。



プロジェクト



Linux XP

TrustVers社は、FedoraベースのシステムをWindowsユーザーにとって普遍的で安全で習得しやすいと位置付けています。 ディストリビューションは完全に商用であり、無料版はありません。 製品に加えて、技術サポートとトレーニングが提供されます。

オープンソースのイデオロギーに関する厳しい声明により、開発者はLinuxコミュニティの怒りを被りました。 マイクロソフトの模倣、配布配布条件、デフォルトのルート作業などが批判されました。



ワイン@ Etersoft

この製品は無料のWineプロジェクトに基づいており、Linux上でWindowsベースのアプリケーションを実行するように設計されています。 サンクトペテルブルクの企業Etersoftで数年にわたって改善が行われたため、ロシアの組織（1C：エンタープライズ、コンサルタント、保証人、KOMPAS-3Dなど）の活動に必要なプログラムをLinuxで使用できるようになりました。 最も重要な改善は、セキュリティキー、デバイスドライバーのサポート、およびリソースとのコラボレーションの提供に関するものです。

WINE @ Etersoftには無料版と商用版の両方があります。 商用版のユーザーには、テクニカルサポートと無料アップデートが利用可能です。

Linuxコミュニティの一部は、無料版で最も重要な機能を提供していないことでEtersoftを批判しています。 また、Wineを介してWindowsアプリケーションの互換性を確保するために作業しているため、プログラムのLinuxバージョンの開発が遅くなると考える人もいますが、開発者は定期的にパッチをメインストリームに送信することでWineの開発に大きく貢献しています。



LinuxWizard

2007年に開始されたサンクトペテルブルクの会社LinuxWizardは、CentOSに基づいてサーバーソリューションと専用ワークステーションを開発しています。無料のバージョンはありません。

LW製品の主な利点は、自動インストールと個別設定です。 開発者は、ソフトウェアがそのまま使用できることを保証するだけでなく、ライセンスされた清潔さが検証中に疑念を引き起こさないことも保証します。 私がいた会社のセミナーでは、配布キットに含まれるすべてのプログラムが特許を取得しており、LWソフトウェアの使用の合法性に対するすべての責任を負っていると言われました。

さらに、LinuxWizardは、そのソリューションの利点として、顧客向け製品の運用技術サポートを個別に指摘しています。

この場合、無料配布の商用化は正当化できますか？ 一部のLinuxoidでは、これは疑わしいものです。 同時に、簡単なインストールと構成は、ITから遠く離れた人々の観点からも利点となります。

LinuxWizardは若くて成長中の企業であるため、その活動が一般ユーザーやオープンソースソフトウェアコミュニティにとってどのように役立つかは、時間が経てばわかります。



ウェルタイム

アスタリスクコンピュータテレフォニーアプリケーションは、追加機能を備えた無料および商用の両方のソリューションを作成するための非常に一般的な基盤です。

Welltime IP ATC商用製品は、会社の通信ネットワークを管理するために設計された多機能モジュラー設定を備えたロシアの開発製品です。 これは、オープンソースソフトウェアアプリケーションに基づいたプロジェクトが、先行製品を認識するのがそれほど容易ではない独立した製品に成長した場合です。

アスタリスクはIPテレフォニーシステムを作成するための優れた基盤と見なされますが、必要なツールの可用性と安定性の要件を満たす製品を入手するには、大幅な改善が必要です。 したがって、このような商業開発は一般的にITコミュニティに歓迎されています。



開発会社の側から

どのような営利会社でも、彼女の使命でどれほど美しい言葉が書かれたとしても、その主な目標は利益です。 これは正常です。

多くの企業は、フリーソフトウェアを宣伝することでお金を稼いでいます（さらに、商用製品だけでなく、完全に無料のディストリビューションとプログラム、および実装者とトレーニングセンターの開発者でもあります）。 自分の製品についてだけ話すと、興味を持つ人はほとんどいません。 一般にオープンソースソフトウェアを宣伝する必要があります。

多くの営利企業は、「ビジネスを支援する」準備ができており、無料プロジェクト用のパッチを提供しています。

同時に、開発自体が多くの企業がLinuxに移行する道を開いています。 Windowsアプリケーションとの互換性の問題（WINE @ Etersoftの場合）、実装の難しさ、ソフトウェアライセンスの確認の必要性（LinuxWizardの場合）など、多くの障害が取り除かれました。

したがって、このような企業の活動は、オープンソースコミュニティにとって価値のあるものではありません。



ユーザーごと

供給はどこからともなく現れるのではなく、需要に基づいていると教えられました。 では、ロシアでは誰が商用のフリーソフトウェアを要求できるのでしょうか？

会社が海賊版Windowsを合法化し、放棄したいと考えているが、Microsoftライセンスにお金がないと仮定します。 一般的にはすべてがLinuxに反するものではありませんが、本格的な実装は望ましくありません-それは難しく、長く、何が起こるかはわかりません。 次に、すべてが正常に機能し、頭が痛くないように、配布キットの暫定額（Microsoftが要求するものよりもはるかに少ない）を支払ってはどうですか？

このビジネスモデルは、ロシアのユーザーにとってはまだ珍しいものです。 会社のためにLinuxに切り替える主な理由は、ライセンス料を支払う必要がないことです。 そして、無料のソリューションがまったく無料でない場合、それはそうではありません。

同時に、消費者ユーザー心理学は、オープンソースソフトウェアに基づく商用ソリューションの開発者に有利に働きます。 私が会社に支払うので、彼女は彼女の製品に対して責任があります。 私はすぐに彼と仕事をすることができ、その月を終えることができないからです。 開発者が私のために時間を割くときではなく、技術サポートが必要なときに答えるという事実のために。 私が合法的にソフトウェアを使用しているという事実、そしてその場合-私に製品を販売した会社のすべての責任。 残念ながら、開発者は顧客にそのような理想的な条件を常に提供できるとは限りません。 しかし、製品が商業的であるという事実は、メーカーに多くの義務を課し、原則として、クライアントが彼の問題に取り残されないという保証を与えます。



RedHatを使用すると、複雑なITインフラストラクチャを備えた大企業にソリューションを実装できます。 そして、この場合の配布料金は絶対に正当であると見なされます。 RedHatの例を使用すると、商用化されたフリーソフトウェアに基づくビジネスモデルである西洋では、オープンソースの考え方に反するものではないことがわかります。 そして、Linuxコミュニティはとてもきれいです。RedHatは無料のFedorを開発しています。

ロシア語以外の例としては、商用のStarOfficeがあります。その開発により、OpenOfficeの開発を常にサポートできます。

いつかロシアの開発者は、作成されたソリューションとサービスと同じレベルの品質を達成すると信じたいと思います。 そして、彼らは利益を上げることとオープンソースビジネス全体に貢献することのバランスを見つけます。
