NetApp HCI─次世代のハイパーコンバージドデータシステム

データストレージシステムの開発とその管理は開発において長い道のりを歩んでおり、企業のITソリューションにとって重要です。 現在、最も先進的なのはハイパーコンバージドシステムであり、今日のレガシーシステムやレガシーシステムに比べて多くの利点があります。 安価で管理しやすく、スケーラブルであり、企業のニーズにリソースが正確に一致するようにします。







この記事では、データのストレージと処理に関連する従来の収束システムとハイパーコンバージドシステムの比較説明を行います。 同様のエンタープライズシステムおよびスケールアウトアーキテクチャのスケーリングオプションが考慮されます。 NetApp HCIデータを操作するための新世代のハイパーコンバージドシステムの説明と特性を示します。



統合および従来のシステム



統合システムは、自然な進歩の結果であり、従来のITインフラストラクチャからの脱却です。従来のITインフラストラクチャは、データの保存と処理のための別個の無関係な「バンカー」の作成に常に関連付けられてきました。



継承されたIT環境の場合、原則として、ストレージシステム、サーバー、およびネットワークサポート用に個別の管理グループ（専門家チーム）が作成されました。 たとえば、ストレージグループは、ストレージインフラストラクチャの購入、プロビジョニング、およびメンテナンスに関与していました。 また、ストレージハードウェアベンダーとの関係も維持しました。 サーバーグループとネットワークについても同じことが言えます。



統合システムは、これらのITインフラストラクチャコンポーネントのうちの2つ以上を事前に設計されたソリューションとして統合します。 このクラスの最適なソリューションは、対応するソフトウェアによって互いに密接に関連する3つのコンポーネントすべてを組み合わせます。



このソリューションの明らかな利点は、複雑なITインフラストラクチャ向けの比較的単純な設計です。 アイデアは、サポートのための単一のチームを作成することです。必要なすべてのコンポーネントをサポートするために単一の売り手に焦点を合わせるのと同じ方法です。



コンバージドおよびハイパーコンバージドシステム



ハイパーコンバージドシステム（HCS）は、まさに「収束」という概念を新しいレベルに引き上げます。 コンバージドシステムは通常、適切に機能するように設計された個別のコンポーネントで構成されています。 HCSは通常、追加のモジュールをシステムに組み込むことで拡張するように設計されたモジュラーソリューションです。 実際、コントローラーソフトウェアレイヤーにより、大規模データストレージシステムの「スケーリング」を実行します。





従来型およびハイパーコンバージドストレージシステムの典型的なアーキテクチャとその管理



ストレージデバイスを追加すると、全体的な容量とパフォーマンスが向上します。 ディスク、メモリ、またはプロセッサを拡張、追加する代わりに、必要なすべてのリソースを含む新しいスタンドアロンモジュールを追加するだけです。 HCSは単一のインターフェイスを介して管理されるため、単純化されたアーキテクチャに加えて、単純化された管理モデルが使用されます。



垂直および水平のスケーラビリティ



スケーラビリティとは、ユーザーのニーズに合わせてサイズ（またはボリューム）が変更された場合でも、一部のタイプのシステムが適切に機能し続ける能力のことです。 一部のコンテキストでは、スケーラビリティは、より大きいまたは小さいユーザー要求を満たす能力として理解されています。 ストレージのコンテキストでは、大容量の需要を満たすことについてよく話します。





垂直および水平のスケーラビリティの概略図



垂直スケーラビリティ（スケールイン）─物理システムに新しいリソースを追加することにより、既存のハードウェアまたはソフトウェアの機能を向上させます─たとえば、サーバーの計算能力を高めてサーバーを高速化します。 ストレージシステムの場合、これは、必要に応じて既存のシステムに新しいコントローラー、ディスク、および入力/出力モジュールを追加することを意味します。



水平スケールアウトとは、単一の論理ユニットとして機能するように多数の自律ユニットを接続することを意味します。 たとえば、水平スケーリングでは、地理的に離れたノードが多数存在する可能性があります。



スケールアウトストレージアーキテクチャ



スケールアウトアーキテクチャの概念によれば、必要に応じて、デバイスの新しいグループをほぼ無制限にシステムに追加できます。 各デバイス（またはノード）には、ある程度のデータストレージ容量があります。 次に、ディスクデバイスによってダイヤルされ、I / O帯域幅（入力/出力、I / O）のように、独自の計算能力を持つことができます。



これらのリソースを含めると、容量が増加するだけでなく、データを操作する生産性も向上します。 クラスター化されたノードが追加されると、システムの規模は拡大します。 このために、外部ネットワークを介して接続された特別なOSとストレージシステムを備えたx86サーバーがよく使用されます。



ユーザーはクラスターを単一システムとして管理し、グローバル名前空間または分散ファイルシステムのデータを管理します。 したがって、データの実際の物理的な場所を心配する必要はありません。



NetApp Enterprise-Scale HCI：次世代のハイパーコンバージドシステム



もちろん、場合によっては、特別でユニークなソリューション（カスタムメイドのストレージシステム、ネットワーク、サーバー）が最良の選択のままです。 ただし、他のオプション─「as-a-service」、統合インフラストラクチャ（CI）、ソフトウェア定義システム（SDS）がITインフラストラクチャ市場を迅速に獲得し、この動きは今後数年で支配的になります。



組織は複雑さを軽減し、ITの導入を迅速化する傾向があるため、CI市場は非常に急速に成長しています。 組織がすでに次世代のデータセンターの構築に向かっているため、ハイパーコンバージドインフラストラクチャ（HCI）プラットフォームは自然な開発の結果として登場しました。



2020年までに、ストレージ管理機能の70％が自動化され、インフラストラクチャプラットフォームに組み込まれることも予想されます。 NetApp HCIは、次世代のハイパーコンバージドインフラストラクチャであり、エンタープライズアプリケーション向けに設計された最初のHCIプラットフォームです。



第一世代のHCIソリューションは、比較的小規模のプロジェクトにより適していました。顧客は、かなりの数のアーキテクチャ上の制限があることに気付きました。 パフォーマンス、自動化、混合ワークロード、スケーリング、構成の柔軟性など、多くの側面を扱いました。



もちろん、これは、「俊敏性」、スケーリング、自動化、および予測可能性が必須要件である次世代の情報センターを構築する戦略と矛盾していました。



NetApp HCIの概要



NetApp HCIは、エンタープライズ規模の最初のハイパーコンバージドインフラストラクチャソリューションです。 このソリューションは、クラウドに似たインフラストラクチャ（ストレージリソース、コンピューティングおよびネットワーク）を、「軽快」でスケーラブルで管理しやすい標準4ノードユニットで提供します。



このソリューションは、 SolidFireフラッシュストレージシステムに基づいて開発されました。 VMware vCenter Plug-inによるシンプルな集中管理により、直感的なユーザーインターフェイスを使用してインフラストラクチャ全体を完全に制御できます。



NetApp ONTAP Selectとの統合により、既存のNetAppクライアントとデータセンターのアップグレードを希望するユーザーの両方に新しい範囲の展開オプションが提供されます。 NetApp HCIは、現在の世代のHCI製品の制限に4つの主要な方法で対処します。



保証されたパフォーマンス 。 今日、特殊なプラットフォームと優れた冗長性は受け入れられない選択肢です。 NetApp HCIは、各アプリケーションの「きめ細かい」制御を提供するソリューションであり、「ノイズの多い隣人を排除」します。 すべてのアプリケーションは、共通のプラットフォームにデプロイされます。 同時に、同社によれば、従来のパフォーマンスの問題の90％以上が解消されています。



柔軟性とスケーリング 。 前世代のHCIには固定リソースがあり、いくつかのホスト構成に限定されていました。 NetApp HCIは、独立したストレージおよびコンピューティングリソースを備えています。 その結果、NetApp HCIはあらゆる規模の構成に適しています。



自動化されたインフラストラクチャ 。 新しいNetApp Deployment Engine（NDE）ユーティリティにより、インフラストラクチャの展開に伴うほとんどの手動手順が不要になります。 VMware vCenter Plug-inは、管理をシンプルかつ直感的にします。 対応するAPIは、トップレベルの管理システムへの統合を可能にし、バックアップと災害復旧を提供します。 障害後のシステム回復時間は30分を超えません。



NetAppデータファブリック 。 初期のHCIプラットフォームでは、新しいリソースグループをITインフラストラクチャに導入する必要がありました。 これは明らかに、非効率的なアプローチです。 NetApp HCIは、「データファブリック」であるNetApp Data Fabricに統合されます。 これにより、データのモビリティ、可視性、保護が向上し、ローカル（オンプレミス）パブリッククラウドまたはハイブリッドクラウドでデータの可能性を最大限に活用できます。



NetApp Data Fabric





データファブリック展開モデル



NetApp HCIは、すぐに使用できるソリューションであり、すぐにデータファブリック環境で動作します。 したがって、ユーザーはパブリッククラウドまたはハイブリッドクラウドにあるすべてのデータにアクセスできます。



NetApp Data Fabricは、企業が互換性のないストレージリソースを使用し、オンプレミスとクラウドストレージ間で継続的なストリーミングデータ管理を提供できるようにするソフトウェア定義のデータ管理アプローチです。



NetApp Data Fabricを構成する製品とサービスは、顧客に自由を与えるように設計されています。 クラウドとの間でデータを迅速かつ効率的に転送し、必要に応じてクラウドデータを復元し、あるプロバイダーのクラウドから別のプロバイダーのクラウドに移動する必要があります。



NetApp Data Fabricの基盤は、Clustered Data ONTAPストレージオペレーティングシステムです。 データファブリックの一部として、NetAppはONTAP for Cloudの専用クラウドバージョンを開発しました。 企業全体のパブリッククラウド内に仮想NetAppストレージを作成します。



このプラットフォームを使用すると、内部のNetAppシステムに実装されているのと同じ方法でデータを保存できます。 継続性により、管理者は中間の変換を必要とせずに、データを必要な場所にいつでも移動できます。 これにより、実際には企業のデータセンターを拡張できます。 プロバイダーのパブリッククラウドが原因です。



NetAppは、2014年に毎年開催されるInsightカンファレンスでData Fabricコンセプトを初めて紹介しました。 NetAppによると、これは、多くの社内および社外のデータセンターに保存されているエンタープライズデータの統一されたビューを取得する顧客のニーズに応えたものです。 特に、Data Fabricを使用すると、企業はGoogle Cloud Platform、Amazon Simple Storage Service（S3）、Microsoft Azure、IBM SoftLayerのパブリッククラウドにある企業データに簡単にアクセスできます。



エンタープライズ要件への準拠



データセンターにおける最大の問題の1つは、現時点で必要なパフォーマンスの予測可能性です。 これは特に、「スプロール化」アプリケーションとそのワークロードに当てはまり、時には非常に激しい場合があります。



各企業は、同じITインフラストラクチャを使用して多数のエンタープライズアプリケーションを使用しています。 したがって、あるアプリケーションが別のアプリケーションの作業を妨げるという潜在的な危険が常にあります。



特に、たとえば仮想デスクトップインフラストラクチャ（仮想デスクトップインフラストラクチャ、VDI）やデータベースアプリケーションなどの重要なアプリケーションの場合、入力/出力メカニズムはまったく異なり、相互に影響する傾向があります。 HCI NetAppは、必要なパフォーマンスを常に提供することにより、予測不能性を排除します。



NetApp HCIは、小規模、中規模、大規模のストレージおよびコンピューティング構成で使用できます。 システムは1RU単位で拡張できます。 その結果、企業は必要なリソースを非常に正確に決定でき、未使用の冗長ハードウェアはありません。



各IT部門の主なタスクは、すべての一般的なタスクを自動化し、手動操作に関連するユーザーエラーのリスクを排除し、リソースを解放して優先度の高い複雑なビジネスタスクを解決することです。 NetApp Deployment Engine（NDE）により、ほとんどの手動インフラストラクチャ展開操作が不要になります。 同時に、vCenterプラグインにより、VMware仮想管理がシンプルかつ直感的になります。



最後に、APIセットにより、データストレージと処理サブシステムを高レベルの管理システムにシームレスに統合でき、バックアップと災害復旧も提供します。



NetApp HCIは、次のテクノロジーを統合およびサポートします 。



  • NetApp SolidFire Element OS─SolidFireストレージオペレーティングシステムでは、ストレージノードを迅速にスケーリングできます。
  • 直感的な展開エンジン─ストレージおよびコンピューティングリソース管理用のソフトウェアコンポーネントを展開および構築します。
  • 堅牢な監視エージェント─HCIリソースとストレージデバイスを監視し、vCenterおよびActive IQに情報を送信します。
  • NetApp SolidFire vCenter Plugin─データストレージを管理するための包括的な機能セットを提供します。
  • NetApp SolidFire Management Node─リモート管理サポートを備えたHCIを監視およびアップグレードするためのVM。
  • VMware ESXiおよびvCenter v6─仮想化およびソフトウェア管理用のホスト。
  • NetApp Data Fabric─SnapMirror、SnapCenter、ONTAP Selectファイルサービス、AltaVaultおよびStorageGRIDバックアップツールなどのSolidFire Element OSツールを使用した統合を提供します。






NetApp HCIの最小構成：フラッシュメモリ用の4つのモジュール、2つの計算モジュール、および追加モジュール用の2つの空のベイを備えた2つの構成（シャーシ）
