* nix-systemsでは大文字と小文字が区別されます。つまり、file.txtとFile.txtは異なるファイルです。 / uploadsと/ uploadsの両方のディレクトリも異なるディレクトリです。



さらに重要な違い：



  • その他のウィンスラッシュ-バックスラッシュ「\」。 * nix-スラッシュ「/」






  • 改行文字を獲得-“ \ r \ n-CRLF”; * nix-「\ n-CR」; mac-「\ r-LF」関連記事： en.wikipedia.org/wiki/Newline




PHP開発では、これらのクロスプラットフォームの問題を解消するために、コンソールの新しい行にPHP_EOLを使用し、正しいスラッシュにDIRECTORY_SEPARATORを使用することをお勧めします。

mysqldump —opt -u [uname] -p [pass] [dbname]> [backupfile.sql]

[uname]ユーザー名

[dbname]データベースの名前

[backupfile.sql]ダンプファイルを呼び出すときに（作成するフォルダーにない場合は、パスを指定することもできます）

[--opt]追加オプション

例： mysqldump -u root -p Tutorials> tut_backup.sql



  • add-drop-table-CREATE TABLEの前にダンプにDROP TABLEを追加します。
  • no-data-コンテンツのないテーブルのみをダンプします。
  • add-locks-LOCK TABLESおよびUNLOCK TABLESをダンプに追加します。




また、データベースが大きく、VPN接続が最速でない場合、次のコマンドを使用してダンプをすぐにアーカイブに圧縮できます。
