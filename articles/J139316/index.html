「Erlangは、フォールトトレラントな分散アプリケーションを構築するための言語です。 これは、実際に長い間成功裏に適用されてきた数少ない関数型言語の1つです。 たとえば、テレコム（エリクソンAXD-301スイッチ）、銀行システム、自動化システム、負荷の高いWebアプリケーション（Facebookチャット）などです。 同時に、Erlangはオープンソース製品であり、無料で配布されています。」- ロシア語を話すコミュニティErlangのサイトを教えてくれます。



この投稿は、Erlangプログラマーの聴衆を拡大することを目的としています。 その前に、関数型プログラミングを国内の目的に使用することはできないと考えていたなら、それはあなたの視野を広げる時です。 この投稿は、このフレームワークの作成者 （Evan Miller）を個人的に知っており、最近この言語のファンである私の友人のサポートで書かれました。



両方によると、このフレームワークに基づくWebアプリケーションの開発は、同じレール（Ruby on Rails）での開発よりも遅くありません。 いくつかの点でさらに高速です。 私は以前は関数型プログラミングに特に精通していませんでしたが、友人のプレッシャーのもとで試してみることにしました。 この体験は非常に成功し、最初のWebサイトを1年前よりも速く同じRuby on Railsで作成することができました。

興味のある人のために： インストール、チューニングツール、学習を含め、開発全体に約4〜5時間かかりました。

チコボス


Gitを使用せずにフレームワークをダウンロードできます。開発ページに移動して最新バージョンを取得するだけです。



Gitをまだお持ちでない場合はインストールします。

apt-get install git-core





ChicagoBossフレームワークのコピーを取得します。

git clone git://github.com/evanmiller/ChicagoBoss.git







リポジトリのディレクトリに移動します。

cd ChicagoBoss





コンパイルします：

./rebar get-deps clean compile







新しいプロジェクトを開始するには（フレームワークのディレクトリにいる必要があります）：

make app PROJECT=your_project_name_here





次に：

cd ../your_project_name_here







デバッグ（開発者）モードを起動するには：

./init-dev.sh





本番環境で実行するには：

./init.sh start







メッセージの移動が停止するまで待ちます。 エラーまたはクラッシュレポートがない場合は、localhost：8001を開くことができます

これで、新しいモデル、コントローラー、ビューを作成し、サーバーを再コンパイルして再起動せずに結果をすぐに確認できます。 場合によっては、たとえばルートを変更するときなど、再コンパイルが依然として必要です。

コンパイルは次のコマンドによって呼び出されます。

./rebar compile







ChicagoBossはデフォルトでMisultinのWebサーバーを使用し、アプリケーションの作成時にポート8001が構成に書き込まれます。必要に応じて、サーバーをMochiweb 、場合によってはCowboyに置き換えることができます。

サーバーが誤ってシャットダウンした後（たとえば、マシンがシャットダウン/リブートしたとき、データベースサーバーが起動したとき）は、まだオフのままです。 これは、特にロックファイルを作成することによって行われます。 サーバーを再起動するには、次を実行する必要があります。

1.ロックファイルを削除する

2.回復スクリプトを実行する



$ sudo rm /var/lib/mongodb/mongod.lock





$ sudo -u mongodb mongod -f /etc/mongodb.conf --repair

最初のプロジェクトの作成



すべてが配信され、すべてが機能します。 Webサイトの作成を開始できます。 ChicagoBossでディレクトリに移動し、コマンドを記述します。

make app PROJECT=addressbook







はい、プロジェクトの名前は有効なアトムでなければなりません。 アトムは名前付き定数です。 名前は小さな文字でのみ開始する必要があり、名前には文字、数字、アンダースコアのみを含めることができます。



フレームワークは、デフォルトでホームディレクトリに新しいディレクトリを作成します。 プロジェクト全体が内部にあります。プロジェクトを移動/転送/削除したい場合は、このディレクトリで十分に操作できます。 プロジェクトの新しく作成されたディレクトリに移動します

cd ../addressbook







すべてのコード/リソースはsrcディレクトリにあります。 一般的な構造：





Srcディレクトリ構造：



リソースファイル（javascript、css、メディアファイル）は/ priv / static /にあります。 スタイルシート接続は次のようになります。 

 <link rel="stylesheet" href="/static/css/style.css" type="text/css" charset="utf-8" />




boss.configを編集し、データベースに関するデータを追加します。

{db_host、 "localhost"}、

{db_port、27017}、

{db_adapter、mongodb}、

{db_database、アドレス帳}、

{db_write_mode、safe}、

{db_read_mode、master}、



これが私の設定（および環境）です。





初めてアドレス帳という単語を間違って書いたとき、多くの名前が付けられているため、後で正しい名前でアプリケーションを再作成する必要がありました。



次に、src / model /にaddress.erlファイルを作成します。 データベースにアクセスするために必要です。 私にとってはこのように見えます： 

 -module(address, [Id, Firstname, Lastname, Address1, Address2, City, State, Country, Active, CreationTime, ModifictionTime ]). -compile([export_all]).




addressbook_main_controller.erl（ appname_controllername_controller ）という名前のコントローラーを作成し、次のコードを内部に記述しましょう。 

 -module(addressbook_main_controller, [Req, SessionID]). -compile([export_all]). index('GET', [])->{ok, [{data, "Hello World"}]}.




インデックスは私たちの機能です

「->」以降はすべて関数の本体です。

この関数はGETリクエストに応答し、処理が成功すると、データ変数に文字列を返します。

OK-すべてがうまくいったことを意味します。 エラーを処理するために、エラーという単語が使用されます。 次のようになります。 

 index('GET', [])-> {ok, [{data, "Hello World"}]}; {error, Reason}-> Reason.




返された値をページに表示するには、ビューに{{data}}タグを追加する必要があります。 確認するには、最初のhtmlファイルを/ src / view / main /ディレクトリに作成します。 それをindex.htmlと呼びましょう： 

 <!DOCTYPE html> <html> <head> <meta charset="utf-8" /> <title>My addressbook</title> </head> <body> <form> <div id="main"> {{data}} </div> </form> </body> </html>




最後の仕上げは、ルートのセットアップです。 ファイル/priv/addressbook.routesを開き、新しいルートを追加します（デフォルトではルートがまだないため、すべての行がコメント化されています）。 

 {"/", [{controller, "main"}, {action, "index"}]}.


アプリケーションのコンパイル./rebar compile

./init-dev.shを実行します



ブラウザに移動して、localhost：8001を開くと、Hello Worldが表示されます。

ルートの変更を除く、以降のすべての変更は、サーバーを再コンパイルおよび再起動せずに適用されます。 その場で。



ルートについて：どのアドレスでどのコントローラーとメソッドを呼び出すかを厳密に示すことができます。 または、アドレスにフォーム/ [コントローラ名] / [アクション名]のフルパスを指定できます。 つまり この場合、mainという名前のコントローラーがあります。 createというコントローラーに新しいメソッドを追加する場合、localhost：8001 / main / createにアクセスして呼び出すことができます。 または、次のルートを指定します。 

 {"/create", [{controller, "main"}, {action, "create"}]}.


そして、呼び出しはローカルホストのようになります：8001 / create



DBを使用する



データベースへの接続はすでに行われています。 Webサーバーの実行中にコンソールでクエリを実行することもできます。 これを行うには、コンソールに移動し、コマンドを実行してアプリケーション./init-dev.shを起動し、Enterキーを押して、たとえば次のように新しい開いた行を入力します。 

 boss_db:find(address, []).




このコマンドは、アドレスモデルにバインドされているすべてのエントリを表示します。 現在、そこにはレコードがありません（またはデフォルトで空のレコードがあります）。

boss_dbと入力できます。Tabキーを押して、使用可能なコマンドを確認します。



[]-「パラメーター」を意味します。 括弧が空の場合、0個のパラメーターを渡します。 findコマンドの場合、firstname = "ivan"のエントリのリストを取得できます。 

 boss_db:find(address, [{firstname, "ivan"}]).


コンソールでは、データベースだけでなく、多くの便利なコマンドを実行できます。



CRUD



これで、モデルが機能する準備が整いました。コントローラに新しい関数を追加できます。 新しいエントリを追加する作成関数を追加します。 

 create('GET', [])->ok; create('POST', [])-> Firstname = Req:post_param("firstname"), Lastname = Req:post_param("lastname"), Address1 = Req:post_param("address1"), Address2 = Req:post_param("address2"), City = Req:post_param("city"), State = Req:post_param("state"), Country = Req:post_param("country"), Active = Req:post_param("active"), CreationTime = erlang:now(), ModificationTime = erlang:now(), NewAddress = address:new(id, Firstname, Lastname, Address1, Address2, City, State, Country, Active, CreationTime, ModificationTime), case NewAddress:save() of {ok, SavedAddress}-> {redirect, [{action, "index"}]}; {error, Reason}-> Reason end.

Id-したがって、変数を呼び出すことができます。

データベース内のすべてのレコードを表示する別の関数を追加します（より正確には、以前に記述されたインデックス関数を置き換えます）。

ChicagoBossは、 ErlyDTLライブラリを介してDjangoテンプレートシステムを使用します。 Djangoテンプレート言語 。



これで、アドレス/ main /および/ main / createにアクセスすると、データベースからの情報の入力と出力の2つのフォームを受け取ります。



レコードを削除（削除）するには、コントローラーの次の機能で十分です。 

 delete('GET', [Id])-> boss_db:delete(Id), {redirect, [{action, "index"}]}.


localhost：8001 / main / delete / [id]と呼ぶだけです。



レコードを表示（表示）するには、この関数をコントローラーに追加します。 

 show('GET', [Id])-> Address = boss_db:find(Id), {ok, [{address, Address}]}.




ビューファイルshow.htmlを作成します。
