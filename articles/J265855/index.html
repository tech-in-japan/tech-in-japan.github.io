クラスタからNetAppストレージシステムを離れる

NetApp FASストレージをクラスターに追加するのは非常に簡単です。

クラスタ相互接続ポートがスイッチに接続され、コマンドが実行されます。 

cluster setup




また、クラスターからノードを取得する方法は？





クラスターからノードを削除する



さまざまなケースで、クラスターからノードを削除する必要がある場合があります。 たとえば、FAS8020およびFAS2240 / 255xシステムで構成されるクラスターがあり、2番目のシステムを8020に接続するためにシェルフに変換する場合です。



これを行うには、以下を行う必要があります。

  • MDVシステムボリュームなど、切断されたノードにあるアグリゲートからボリュームを削除または移行する
  • 無効にするノードからLIFを削除または移行します。 SAN LIFは移行できません（結局、マルチパスを構成している可能性が高いため）またはオフラインで移行できます
  • 切断されたノードのポートを指す場合、LIFのホームポートを変更する
  • 手動で作成されたフェールオーバーグループから切断されたノードのポートを削除する
  • 手動で作成されたbroadcast-domain'ovから切断されたノードのポートを削除します
  • 切断されたノードのポートをportset'ovから削除するか、新しいportset'yを作成し、igroupの古いものを新しいものに置き換えてから、古いものを削除します。
  • 切断されたノードが配置されているグループの高可用性をオフにします。HAシステムの両方のノードを必ず削除してください（順番に）
  • 切断されたノードのユニットに監査ログ用のシステムボリュームがある場合、CIFS監査をオフにします
  • 必要に応じて、拡張モードから、イプシロンを削除されないノードに移動します 


 cluster1::> vserver audit delete -vserver vsm01 cluster1::> storage failover modify -node clA-01 -enabled false cluster1::> storage failover modify -node clA-02 -enabled false cluster1::*> set -privilege advanced cluster1::*> cluster show cluster1::*> cluster modify -node clA-02 -epsilon false cluster1::*> cluster modify -node clA-04 -epsilon true cluster1::*> cluster unjoin -node clA-02
