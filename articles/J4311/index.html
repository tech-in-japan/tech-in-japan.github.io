マイクロソフトは、ツールWeb 2.0の兵器庫を補充します

テクノロジWindows Presentation Foundation（以前のコード名はAvalon）は、次世代のWebアプリケーションを作成するための便利なプラットフォームとしてのマイクロソフトの切り札です。 XAML言語とともに、AtlasやLINQなど、Microsoftがリリースした他のWebツールを補完します。



ニュージーランドで開催されたTechEd 2006テクノロジーカンファレンスで、マイクロソフトの代表者は、次世代のWeb（Web vNext）の将来に対するビジョンについて話しました。 Web vNext 円卓会議の参加者には、Windows Live Developer PlatformのゼネラルマネージャーであるGeorge Mooreと、他の3人のMicrosoft開発者が含まれていました。



議論のトピックの1つは、新しいXAMLマークアップ言語でした。これは、Ajaxアプリケーションを開発するための便利なツールとして位置付けられています。 以前はAvalonというコードネームだったWindows Windows Presentation Foundation （WPF）を介してアプリケーションを安全に起動するために、新しいシステムと組み合わせて使用​​する必要があります 。 XAML / WPFバンドルは、Windows Vistaオペレーティングシステムに統合され、間違いなく非常に人気があります。



マイクロソフトの開発者は、WPFテクノロジーが非常に一般的になる理由を説明しました。 事実、Web 2.0アプリケーションを作成するには2つのアプローチがあります 。 最初の方法ではHTML転送とサーバー側の操作を行い、2番目の方法ではクライアント側のブラウザーで実行するインタラクティブなWebアプリケーションを作成します。 しかし、Microsoftの開発者によると、Javascriptを使用して2番目のタイプのアプリケーションを作成することは、メモリ制限やその他の問題のために非常に困難です。 そのため、Microsoftによると、開発者は、WPF、Flashなど、タスクを容易にする代替プラットフォームを間違いなく実験するでしょう。



マイクロソフトは、開発者に新しいXAMLおよびWPFテクノロジを迅速に紹介しようとします。 この目的のために、いつものように、予備的な技術リリース（コミュニティテクノロジープレビュー）が使用されます。 開発者がAtlasやLINQなどの他のMicrosoft Web 2.0テクノロジでXAMLとWPFをテストし、公式リリースの前にフィードバックを提供できることが重要です。



ラウンドテーブルでさえ、最近のインタラクティブなWebアプリケーションの作成をハックと比較できるという興味深いアイデアがありました。それは同じ重要なプロセスです。 マイクロソフトの担当者は、開発者とのコミュニケーションの結果から得られた統計に同意し、引用しました：Webアプリケーションの80％は簡単に作成でき、最後の20％はブラウザーの非互換性、バグなどのために非常に困難です。 そのため、 Dojoなどの人気のあるAjaxパッケージが登場しました。 もちろん、Microsoftは.NETの下ですべてを簡素化および標準化して、開発者の作業を楽にするつもりです。 これが、WPFなどのWeb 2.0アプリケーション作成ツールの目的です。
