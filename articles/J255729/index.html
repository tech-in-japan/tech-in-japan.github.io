Veeam Endpoint Backup Freeの概要-Windowsを実行しているラップトップおよびコンピューターの無料バックアップ製品

VeeamONカンファレンスでVeeam Endpoint Backup Freeソリューションが発表されてから6ヶ月が経過しました。 この間に、製品のベータ版をリリースして約2万人に配布し、プログラムの「小児疾患」を特定するために大規模なテストを実施しました。 さて、今日は、プログラムの最終バージョンをご紹介できることを嬉しく思います。



プログラムを使用して、コンピューター、そのボリューム（ディスク）、またはファイルとフォルダーのみをバックアップできます。 そして、システムクラッシュ、ハードドライブの誤動作、破損、またはファイルの誤った削除が発生した場合にデータを回復します。



製品が何であるかを見る時間をまだ見つけていない人のために、私はその主な機能とあらゆる種類の有用なささいなことの概要を準備しました。 猫へようこそ。









物理的な環境をバックアップするための無料のソリューションを作成することにした理由について、私は長い間退屈しません。 しかし、私たちではないにしても、誰が個人データの価値が現代世界でどのように成長したかを理解していますか？ なぜデータ保護について話すのか、私たちが働いている会社のサーバーだけを考えるべきですか？ さらに、Veeamは、単純なユーザーの生活をより便利にする無料のソリューションで常に知られていることを忘れないでください。 結局のところ、個人のラップトップデータをバックアップすることで生活が楽になることを否定することは困難です。



Veeam Endpoint Backup Freeをインストールします



それでは、製品の使用を開始しましょう。 最終バージョン（1.0.0.1954）はここからダウンロードできます （〜175Mb）。 インストールに関しては、大きな説明はありません。ウィザードの指示に従うだけで、数回クリックするだけでプロセスが開始されます。

その環境の製品要件のリストは次のとおりです。

  • Windows 7 SP1 +
  • Windows Server 2008R2 SP1 +
  • 2GB RAM +
  • 150MBのディスクスペース
  • .Net Framework 4.0
  • MS SQL Server 2012 Express Local DB Edition（SQLインスタンスがない場合、自動的に配信されます）


インストール中に、リカバリディスクを設定し、設定なしでバックアップを作成するように求められますが、この手順はスキップできます。





GUIの紹介



インストールが完了すると、プログラムはトレイアイコンで開始し、マウスをポイントすると最後のタスクに関する簡単な情報が表示されます。また、ここから数回クリックするだけでバックアップまたは復元ができます。





しかし、もちろん、もっと興味深いのはプログラムウィンドウ自体の内容です。 トレイから「コントロールパネル」を呼び出し、4つのタブを持つインターフェイス自体を確認します。





[設定]タブには、アクティビティとプログラムの種類の最小設定、CPUに負荷がかかっているときにバックアッププロセス自体の優先度と負荷を下げるオプション、および「スケジュールされた作業」と通知を無効にする機能が含まれます。 「サポート」は、GUIから直接サポートチームにバグまたは質問を送信するのに役立ちます。「更新」は、コードの最新バージョンがインストールされているか、更新が必要かどうかを示し、「ステータス」タブには、最新のバックアップセッションに関する簡単な情報が表示されます。 ところで、それらのいずれかをクリックして、詳細を確認できます。





次に、バックアップタスクスケジューラ（メインウィンドウの[ バックアップの構成]リンク）に進みましょう。 彼はすぐにバックアップモードに興味があります：マシン全体、個々のディスクまたは個々のフォルダー/ファイル。





その後、ウィザードは、ローカルディスク、NAS上のフォルダー、またはVeeam Backup＆Replicationリポジトリ（ネットワークで突然使用可能になった場合）の選択を提供します。 統合にはVeeam Backup＆Replication 8.0 Update 2が必要です。これはVeeam Backup＆Replicationのアップデートで、数週間後にリリースされ、vSphere 6を完全にサポートします。

Endpoint Backupに戻ると、Veeam Backup＆Replicationを使用してコピーをテープまたはクラウドに送信するユーザーにとって、この統合が役立つと言えます。 データを仮想ディスク（VMDK、VHD、VHDX）にエクスポートすることもできますが、P2V移行はプログラムの目的ではないことを理解する必要があります。





最後に、コンフィギュレーターはアクションプランの作成を提案します。 追加のオプションを有効にすると、さまざまなシステムイベントが発生したときにコピーが開始できることに注目してください：マシンがロックされたとき、システムがログオフしたとき、またはバックアップディスクが接続されたとき（テスト時に後者が特に好きでした）。 プログラム自体がディスクの接続（ディスクのみ）を判別し、すぐにそこにデータのコピーを作成します。

両親のコンピューター用にこのセットアップを行います:)





次に、バックアップチェーンについて少し説明します。 ここでは、Veeam Backup＆Replicationのように、 直接無限増分チェーンが使用されます。 また、保持期間は次のようにチェックされます。次のタスクの最後に、プログラムは過去N日間（ユーザー定義）の「成功」コピーをチェックし、この期間外に古いファイルを変換/削除するかどうかを決定します。 さらに、プログラムはカレンダーの日をカウントしませんが、バックアップが正常に受信された日のみをカウントします。 つまり、1か月間コンピューターの電源をオフにしてから再びオンにすると、Veeam Endpoint Backup Freeはすべてのポイントを削除しませんが、作業が中断されなかったことを「ふり」します。





データ復旧



このソリューションには多くの回復オプションがあります。 もちろん、すべては個別のファイルとボリュームから始まり、再起動や操作なしでオンザフライで回復します。 すべては、プログラムインターフェイスまたはメインメニューの特別な回復ウィザードを使用して実行できます。 個々のファイルを復元する場合、Veeamはバックアップを解凍することもせず、単にC：\ VeeamFLRにマウントし、ユーザーがエクスプローラーを使用して必要なファイルを見つけて復元できるようにします。





（スワップなどを使用して）ブートボリュームを復元するために、ブートリカバリイメージを使用した特別なリカバリ手順が提供されます。 これらの可能性を見てみましょう。
