Symantec Backup ExecでIntelligent Disaster Recoveryを使用する

Symantec Backup Execの一部であるIntelligent Disaster Recoveryコンポーネントと連携する機能についてお話したいと思います。このコンポーネントは、障害からの迅速な復旧を提供し、タイムリーなバックアップで「クリーンワーキングアイアン」状態から「すべて動作」状態にマシンを迅速に引き上げることができます。

この記事では、IDRの使用方法、発生した問題とそれらの解決方法、および私が知っている落とし穴について説明します。

注目を集める絵









IDRはどのように機能しますか？



通常、IDRは退屈なだけで機能します。必要なサーバーのタスクまたはバックアップポリシーが作成され、IDRディスクイメージが作成され、テストされ、安全な場所に置かれます。
必要に応じて、手順を復元します。
  • IDRディスクから起動

  • デバイスの初期化



  • 復旧ポイントの選択

  • データソース接続

  • 回復





手動リカバリオプションがあり、バックアップの場所に関する情報を含む.drファイルが利用できない場合に使用されます。 このオプションは、必要な情報がどのメディアにあるかを知る必要があるという点で自動オプションと異なり、回復するときに必要なメディアを手動で指定します。





リカバリ後、ディスクを取り出し、ディスクから起動して、復元されたシステムを取得します。 理想的です。 現実には、次の状況は人生を多様化します。



パーティショニングが変更されました
この場合、IDRは機能せず、すべてを手作業で行う必要があります。 サーバーにOSを配置し、Backup Execクライアントをインストールして、必要なクライアントを手動で復元する必要があります。 サーバー名は、事故前と同じように指定する必要があります。



IDRがデバイスドライバーを見つけられない、またはバックアップが表示されない
-バックアップはありますが、 愚かな IDR ソフトウェアがコントローラーまたはネットワークカードを認識しなかったため、回復できません。 このような場合、シマンテックは、必要なドライバーをUSBフラッシュドライブからロードすることを推奨します（スポイラーの下のスクリーンショット）。

スクリーンショット






ドライバは、シマンテックまたはマイクロソフトのいずれかである必要があります（いずれにせよ、シマンテックのサポートはそう主張します）。 上記のスクリーンショットの場合、そのようなドライバー（Emulexのアダプター）がなく、科学で知られているドライバーがシステムに認識されなかったため、回避策を探す必要がありました。すべてが働いた。



IDRを復元すると、ドライブ文字が混乱する




これは通常、オプティカルドライブを接続するときに発生し、接続時にドライブの一部はオンラインになりません。 ディスクマネージャーを実行し、手動でディスクをオンラインで変換することで処理されます。



私が知っている落とし穴もリストします。


  • バックアップを配置するディスクは、既存のファイルシステムのディレクトリにマウントする必要はありません。ドライブ文字を割り当てる必要があります。 外部RAIDがC：ドライブのディレクトリにマウントされていると、構成を回復できませんでした。
  • 各サーバー構成の変更後、IDRディスクイメージを再作成してテストする必要があります。できれば情報を回復してください。 そうでなければ、最も重要な瞬間に、回復の困難さの型にはまらない関係があなたを待っています。
  • バックアップタスクログを注意深く読み、Advanced File Open Optionと対応するエージェントを使用してください。そうしないと、バックアップ中にロックされたファイルやデータベースを復元する手間がかかりません。
  • IDRを使用すると、ローカルリソース（ディスクまたはテープ）、ネットワークドライブ、メディアサーバーから回復できます。 したがって、結論-異なるリソースに各サーバーの2つのコピーを保持し、コピーは対応するポリシーによって簡単に複製されます。
  • IDRはWindowsでのみ使用可能です。 Linuxを復元する必要がある場合-Netbackupなどが必要です。
  • IDRディスクを作成するときは、ロシア語版のオペレーティングシステムを使用しないでください。 英語を使用します。 Backup Exec 12.5以降でサポートされるロシア語
  • Windows 2003 Small Business Serverは、修正プログラム346104を使用してBackup Exec 12.5のみを自動的に復元します。古いBEがある場合は、手動で復元します。
  • 少なくとも復旧中は、バックアップがあるマシンのファイアウォールをオフにします。 これは、特にWindows XP SP2およびWindows Vista / 2008に当てはまります。
  • Windows 2008 x64でテープドライブまたはSATAテープライブラリを使用している場合は、データを手動で回復する準備をしてください。
