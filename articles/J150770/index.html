DS112 +の例で孤独に対処する方法

データストレージの場所が終わると、ユーザーは自宅と職場の両方で定期的に遭遇します。 新しいスマートフォンでも、PCでも、ネットワークドライブでもかまいません。 電話とコンピューターの場合、追加のメモリーカードまたはハードドライブを購入することで問題が解決する場合、特にシングルディスクの場合はNASについてはどうですか？ ハードドライブがクラッシュした場合にデータを失わない方法 そのため、シングルディスクNASの場所がなくなった場合、解決策は1つだけです-何らかの方法で「追加」します。 これは、外部ドライブ、フラッシュドライブ、またはDASドライブを使用して実行でき、代替オプションと比較してストレージの信頼性が向上しています。 すぐに2つの質問が表示されます。「友達を作る」ことができるのでしょうか。 また、通常の運用データ交換レートに対して高いまたは少なくとも十分なことについて話すことは可能ですか？



研究の対象として、 Synology DS112 +を使用します 。これは、 SynologyシリーズのシングルおよびデュアルディスクNASの中で最も生産性の高いものの1つです。 このドライブには最新のインターフェースがすべて装備されており、外部ストレージをネットワークに接続するためのNAS機能を最大限に理解することができます。



DS112 +の「ライフライン」として、 5ディスクCFI-B8253JDGGの使用を試みます。



多分誰かがそれはすでに多すぎると言うだろう-5ディスクストレージを1ディスクNASに接続するには！ 私たちは結論に急ぐことはせず、最終的にどのような解決策を得るかを見ません。



最初の質問の答えから始めましょう-デバイスは「友達を作る」のでしょうか。 答えは間違いなく「はい」です。 DAS CFIを異なるNAS Synologyモデルに接続しようとしましたが、同じ肯定的な結果が得られました。USB3.0とeSATAを介して接続した場合です。 今後、eSATA接続が速度の点で非常に望ましいことが判明したとしましょう。



それでは、実用的な実験室の部分に移りましょう。 2番目の質問に答えます。NAS、PC、DAS間の相互作用の速度はどれだけ「適切」と見なすことができますか。



まず、「品位」の基準点を取得する必要があります-これはもちろん、PCからのDASとNASの作業です。 両方のデバイスをコンピューターに直接接続し、PCからSynologyとCFIの間のデータ交換の速度を測定します。

PCとSynology PCとCFI
読書 記録 読書 記録
95.3 MB / s 71 MB /秒 220 MB /秒 110 MB /秒


これらの結果を覚えて、参照結果と考えてみましょう。



それでは、Synology NASと組み合わせてDASを使用するプラクティスに移りましょう。 同様のスキームが、さまざまなメーカーのどのNASでも機能すると付け加えます。 ただし、Synologyにはシングルディスクモデル上でもUSB 3.0ポートがあり、実験が可能な限り興味深いものになっています。



まず、CFIで、背面パネルのディップスイッチを使用して、RAID0ストレージモードを最速のRAIDモードに設定し、結果のアレイをNTFSでフォーマットします。 DASドライブをSynologyに接続し、FileStationアプリケーションに直接進みます。 ここでは、アレイはusbshare1-2（USB 3.0接続）という名前の共有フォルダーとして表示されます。











注：USB3.0とeSATAを介した同時DAS接続はできません。



このフォルダーには、共有フォルダーの設定で読み取り/書き込み権限を割り当てることができます。



このフォルダーをネットワークドライブとして接続し、ネットワーク上のDASフォルダーへのアクセス速度を測定します。

得られたデータから、最良の接続オプションはeSataであると結論付けることができます。



ストレージシステムに信頼性を「追加」して、RAID 5アレイモードをCFIに設定してみましょう。DASをNTFSでフォーマットして、通常のPCへの接続を簡素化し、速度を測定します。

USB 3.0 eSATA
PCでネットワーク経由で読む 22.5 MB \ s 42 MB \ s
PCからのネットワーク録画 10 MB \ s 52.7 MB \ s
CFIからSynologyへ 18.5 MB \ s 40 MB \ s
SynologyからCFIへ 6.6 MB \ s 52.7 MB \ s


この場合、eSataの方が高速です。



実験的に、速度の「品位」の問題に対する肯定的な回答を受け取りました。 Synology DS112 +に接続されたDASドライブへのアクセス速度は、ネットワーク上のNASサーバーの速度に匹敵します。 これは、Synology NASだけでなく、ホームシリーズとそれより古い他のメーカーのNASについてでもあります。 毎日の仕事に絶対に許容できると考えられるもの。



さらに、NASプラットフォームをより強力に使用してDASドライブをネットワークに接続すると、より高速なパフォーマンスが得られます。 残念ながら、USB 3.0を介した交換速度は、DASが接続されているホストデバイスのパフォーマンスに完全に依存しています。 したがって、得られた結果は、使用するプラットフォームの自然な制限を反映しています。



結論：



ネットワークドライブのボリュームを増やす必要がある場合、1台のディスクでも、外部DASドライブ（この場合は5ディスクCFI B8253JDGG）を接続するのが優れたソリューションです。 RAID5ストレージアレイモードを使用すると、システム全体に信頼性などの重要な要素が追加されます。これは、当初はシングルディスクNAS自体では利用できませんでした。



このソリューションのコストを見積もります。 5ディスクのNASのコストよりも安い価格で、DS112 +に近いパフォーマンスのほぼ6ディスクのストレージシステムが得られます。 DS112 +が代表的なPlusシリーズは、一連の生産的なデバイスであることに注意してください。 唯一の違いは、新しいストレージボリューム全体が1つであり、複数のフォルダーではないため、ユーザーの権利セットが1つだけであることです。 それにもかかわらず、多くの問題を解決するには、これで十分であり、「対面で」と言う節約です。



単一ディスクのNASでも1 + 4、1 + 5、1 + 8システムなどに変更できるという事実 外付けのRAIDストレージモジュールを接続するだけで、夢が実現することはないでしょうか。



DASは、NASの容量を拡張するだけでなく、ローカルバックアップの信頼できる外部容量にも優れたオプションであることに注意してください。 シングルディスクNASの場合-これは関係ありません。シンプルな外付けHDDで実行できます。 しかし、たとえば8ディスクのNASがある場合はどうでしょうか。 答えは、8ディスクDASを使用することです！



繰り返しになりますが、eSATAを介してデバイスを接続すると最高の速度が得られるため、可能であればこのインターフェイスを使用することをお勧めします。
