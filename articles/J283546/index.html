ジュニパースタッキングの利点

パート1.仮想シャーシテクノロジー



競合他社に対するジュニパーソリューションの主な利点の1つは、スタッキング機能です。 基本的なバーチャルシャーシから始まり、多くのデータセンターテクノロジで終わる、かなりの数のオプションがあります。



パート1.バーチャルシャーシ











基本的なバーチャルシャーシテクノロジの可能性と機能を考慮してください。これにより、最大10台のデバイス（モデルに応じて）EXシリーズとQFXシリーズを1つの論理デバイスに結合し、構成および監視することができます。



バーチャルシャーシテクノロジーをサポートするモデルから始めましょう。



•最大4台のデバイスのEX2200イーサネットスイッチ

•最大10デバイスのEX3300イーサネットスイッチ

•最大10台のデバイスのEX4200イーサネットスイッチ

•最大10台のデバイスのEX4300イーサネットスイッチ

•EX4550イーサネットスイッチ、最大10台のデバイス

•EX4600イーサネットスイッチ、最大10台のデバイス

•QFX5100最大10台のデバイスを切り替えます。



これらは、作成中のモデルであり、執筆時点で注文可能です。したがって、仮想シャーシで使用できます。



バーチャルシャーシテクノロジーは、スタンドアロンスイッチのスケーラビリティとコンパクトなフォームファクターと、従来のシャーシの高可用性、スループット、およびポート密度を兼ね備えています。 バーチャルシャーシを使用すると、シャーシデバイスの設置コストが高すぎる場合や、シャーシスイッチの設置が物理的に不可能な場所で、スイッチの効率的な展開とネットワークの可用性が実現します。



複数のデバイスがユニットとして管理されるため、バーチャルシャーシでは、グレースフルルーティングエンジンスイッチオーバー（GRES）テクノロジなど、シャーシスイッチで複数のルーティングエンジン（RE）を使用する場合と同じ冗長機能を使用できます。







仮想シャーシに接続するには、EX4500、EX4550、EX4200デバイス、および銅線ポートと光ポートにある専用インターフェイスを使用できます。 仮想シャーシへの接続に使用されるポートは、仮想シャーシポート（VCP）と呼ばれます。 たとえば、EX2200-24T-4G 24 10/100 / 1000Base-Tスイッチと4つのSFPでは、SFPまたは銅線ポートを使用して仮想シャーシに接続できます。 これにより、1つの仮想シャーシのメンバーを80 kmの距離で広げることができます。



混合バーチャルシャーシを使用すると、下図に示すように、同じバーチャルシャーシ内で異なるシリーズのスイッチや異なるラインのスイッチを使用できます。





混合バーチャルシャーシスイッチのスタッキングオプション



仮想シャーシには役割区分があります。2つのスイッチはRE0とRE1の役割を受け取り、残りはラインカードとして使用されます。



RE0は、仮想シャーシの残りのメンバーを管理することをタスクとするウィザードの役割を果たし、ルーティングテーブルを作成してそれらをラインカードで配布するのは彼が担当します。 また、構成ファイルも保存します。 RE1-RE0に障害が発生した場合のバックアップ。



以下の例を使用して、仮想シャーシ内のスイッチの役割の分散を検討してください。







ウィザードを選択するための基準：



-優先度が最も高いスイッチ。 優先順位は1〜255です。デフォルトでは、すべてのスイッチで128です。

-再起動前にマスターであったスイッチ。

-稼働時間が最も長いスイッチ（差は1分以上である必要があります）;.

-最小のMACアドレスを持つスイッチ。



仮想シャーシの各メンバーは、0から9までの独自のIDを取得します。このIDは、ウィザード（ID 0を持つ）から手動で構成できます（再起動後、番号が保存され、インターフェース定義でスロット番号として使用できます）。







IDがスイッチに割り当てられているという事実により、次の状況が発生する可能性があります：1つのスイッチに障害が発生してそれを変更すると、新しいメンバーは障害のあるスイッチの番号を受け取らず、次のメンバーは空いているか、仮想シャーシにまったく接続していない場合IDはすでに取得されています。



したがって、故障したスイッチを新しいものと交換するには、まず次のようにウィザードからそのIDを削除する必要があります。



{マスター：0}



user @ Switch-1>バーチャルシャーシリサイクルメンバID <member-id>のリクエスト



または、コマンドIDを使用してこれを行うことができます。



user @ host>リクエストバーチャルシャーシリメンバー番号<現在のメンバーID>新しいメンバーID <新しいメンバーID>

ソフトウェアの互換性



仮想シャーシのすべてのメンバーは、同じソフトウェアバージョンである必要があります。 仮想シャーシに新しいスイッチを追加すると、ウィザードはソフトウェアバージョンを確認します。ソフトウェアが異なる場合、新しいスイッチはIDを受け取りますが、仮想シャーシのアクティブなメンバーにはなりません。 これを行うには、次のコマンドを使用して新しいスイッチのソフトウェアを更新します。



システムソフトウェアの追加要求<member-id>再起動



このコマンドは、新しいスイッチのVCPを介してメインスイッチからイメージをダウンロードしてから再起動します。新しいスイッチはメインスイッチに直接接続されない場合があります。



自動ソフトウェア更新



自動ソフトウェア更新を設定することにより、新しい各スイッチがウィザード以外のバージョンに接続されると、ソフトウェアが自動的に更新されます。 バーチャルシャーシの自動sw更新パッケージ名<パッケージ名>を設定します



ノンストップソフトウェアアップグレード



ノンストップソフトウェアアップグレード（NSSU）テクノロジーにより、仮想シャーシのすべてのメンバーでソフトウェアの更新を最小限の損失で行うことができます。 このテクノロジーを正しく動作させるには、次のことが必要です。



-仮想シャーシのすべてのメンバーは、リングトポロジを使用して接続されました。

-マスターとバックアップは隣接していました。

-ラインカードは事前設定モードで設定されました。

-仮想シャーシのすべてのメンバーには、ソフトウェアの1つのバージョンが必要です。

-NSRとGRESを含める必要があります。 オプションで、request system snapshotコマンドを使用してNSBをアクティブ化できます。



NSSUプロセス：



-ソフトウェアのイメージをダウンロードします。 混合仮想シャーシを使用する場合、両方のイメージをダウンロードします。

-/ var / tmpフォルダーで推奨されるRE0にコピーします。

-RE0コマンドラインから、次のコマンドを使用してNSSUを起動します。

システムソフトウェアのノンストップアップグレード/var/tmp/package-name.tgzを要求する

または混合仮想シャーシの場合

システムソフトウェアノンストップアップグレードセット[/var/tmp/package-name.tgz /var/tmp/package-name.tgz]を要求します
