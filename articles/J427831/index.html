連邦税務局は国内のLinuxおよびOfficeにシステムを適合させるために33億ルーブルを費やす

ご存知のように、輸入代替プログラムの枠組みでは、政府部門は、高価なアメリカのWindowsおよびMS Officeを安価なロシアのカウンターパートに置き換えることが義務付けられています。 別の方法として、Bazalt SPOのAlt Linuxオペレーティングシステムと、New Cloud Technologies（NOT）のオフィスソフトウェアMyOfficeが提供されています。 しかし、ここには予想外の困難と予想外の費用があります。



連邦税務局（FTS）はTax-3自動化情報システム（AIS）をテストし、前述のロシアのソフトウェアとの互換性が低いことを発見しました。 テスト結果はレポートで公開されます （ コピーはKommersantの編集部にあります ）。



そのシステムを完成させるために、連邦税務局は33億ルーブルを要求しました。 州予算から。 「Tax-3」システムは.NETで記述されています。



10月19日、通信省は、連邦税務局の報告書を政府分析センター（AC）、研究所のVoskhod、ICT分野の輸入代替コンピテンシーセンターに、アレクセイ・ソコロフ副通信大臣との会談後に送信しました。 明らかに、ACはそのような資金を割り当ててシステムを完成させるか、WindowsとMS Officeにとどまるほうが収益性が高いかを決定します。 現在の為替レートでは、33億ルーブルは50.2百万ドルであり、ロシアの「アナログ」をインストールして維持するための追加コストを考えると、輸入ソフトウェアはさらに安くなる可能性があります。



ACは、センターへの手紙が「作業中に考慮され考慮されたが、直接の指示は含まれていない」ことを確認した。



Import Substitution Competence CenterのディレクターであるIlya Massukhは、1年半前のテスト結果に基づいてソフトウェア製品の品質を「間違って」判断するのは「動的に開発中」だからだと言いました。 それでも、部門別AISのロシアのソフトウェアへの適応に投資する必要があります。輸入代替のディレクターは確信しています。今では「外国のソフトウェア製品を非常に深く使用しているため、情報セキュリティの観点から危険になっています」。 そのような適応のコストは、情報システム自体のコストの10％を超えない、と彼は推定した。



「MyOffice」は2016年末にリリースされたバージョンでテストされ、過去2年間でコメントの約90％が削除されたと、Dmitry Komissarov総局長は述べていません。 彼は、My Officeにはロシア語の代替手段がないため、会社が集中調達に参加する権利がないことを強調しました。 このため、入札は外国の競合企業、主にマイクロソフトによって勝ち取られます。
