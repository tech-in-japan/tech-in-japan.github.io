Windows 7への移行時にWindows XP Modeを使用する

ご存知のように、仮想化は異なる場合があります。

Microsoftは、次の種類の仮想化を区別しています。



1.データセンターの仮想化。

この領域のソリューションは、Windows Server 2008 R2 Hyper-VまたはWindows Server 2008 R2 Hyper-Vサーバーの役割です。

2.クライアントの仮想化。

この分野のソリューションは、Windows Server 2008 R2のリモートデスクトップサービス、Microsoft Virtual Desktop Infrastructure、Microsoft Enterprise Desktop Virtualization、Microsoft Application Virtualization、Windows Virtual PS、そして最後にWindows XP Modeです。

3.クラウド仮想化。



なんで？ そして、いくつかの決定が不当に忘れられており、サーバー仮想化とVDI、おそらくRDSにすべての注意を集中しているという事実に。

たとえば、会社が企業の標準を変更し、標準がWindows XPではなくWindows 7になった場合、どうしますか？ 同時に、プログラマーはソフトウェアを書き換える時間がありませんでした。これは、古いオペレーティングシステム用に「シャープ」になっています。

もちろん、サーバーファームを購入し、VDIを実装し、仮想マシンをユーザーに配布できます-お金の問題です。 そして、それらがそれほど多くない場合はどうなりますか？



1つの解決策は、Windows XP Modeを使用することです。これにより、特定のソフトウェアがインストールされたWindows XPを実行するローカル仮想マシンを使用して、仮想マシンにインストールされていないかのようにWindows 7でこのアプリケーションを公開できます。

この場合、アクションのシーケンスは次のとおりです。ユーザーデータを保存し、Windows 7を再インストールし、Windows XP Modeをオンにして、実行中の仮想マシンにソフトウェアをインストールし、外部に公開します。 どれくらい？ そして

そして、そのようなマシンが100または200、または1000台ある場合はどうなりますか？ 口ひげとあごひげを放すことができます:)



わかりました、簡素化-Microsoft Deployment ToolkitがインストールされたMicrosoft System Center Configuration Managerがあります。 これらのツールは、ワークステーションとサーバーを管理するように設計されています。特に、新しいワークステーションでOSを集中的に使用したり、既存のステーションにOSを再インストールしたりできるオペレーティングシステム展開モジュールを強調します。 最近、これらの製品を共同で使用することで、新しいWindows 7 OSへのエンタープライズ移行プロセスを完全に自動化できます。



また、ソフトウェアアシュアランスを購入した企業には、追加のボーナスがあります-Microsoftの新製品-ソフトウェアアシュアランス用のP2V移行（Sysinternals Disk2VHDに基づく）。 Windows 7への自動移行プロセスにより、Windows XP以降を実行している既存の仮想マシンから仮想マシンを作成し、Windows XPモードで実行し、インストールされたアプリケーションとInternet ExplorerをWindows 7で公開できます（RemoteAppおよびVirtual PCメカニズムを使用）。

当然、XPモードでの公開アプリケーションの使用は一時的なソリューションです。これは、より多くのシステムリソースが消費され、すべてのOSがサポートされるわけではありませんが、少なくとも、Windows 7をサポートするソフトウェアの書き換えまたは新しいバージョンの購入を可能にする一定の時間バッファーがあるためです。



ソフトウェアアシュアランスのP2V移行のベータ版は、 https：//connect.microsoft.com/site14/Downloads/DownloadDetails.aspx？DownloadID = 30989で入手できます 。

当然、いくつかの制限があります。

-このような移行は、32ビットOS Windows XP SP3、Windows Vista Business、Windows Vista Enterprise SP1またはSP2、Windows 7をインストールした場合に可能です。

-インストールされたOSのライセンスはOEMではありません。

-ハードドライブは127 GB以下でなければなりません。



さらに、P2V移行のインストール中にインターネットアクセスがなかった場合は、次のソフトウェアをダウンロードして/ Toolsインストールフォルダーにコピーする必要があります。

-Disk2VHD.exe 。 P2Vハードディスク変換を実行するSysinternalsツール

-KB961742-v3.exe RemoteAppをサポートするためのWindows XP SP3用の更新

-Windows6.0-KB961741-x86.msu RemoteAppをサポートするWindows Vista SP1用の更新

-Windows6.1-KB958559-x64.msu Windows 7用の64ビットWindows Virtual PC

-Windows6.1-KB958559-x86.msu Windows 7用の32ビットWindows Virtual PC

-Windows6.1-KB977206-x64.msu Windows Virtual PC用の64ビットアップデート
