専用のSelectelサーバーへのオペレーティングシステムの自動インストール





前の記事で、専用サーバーの負荷の管理について既に説明し、さらに興味深いものを表示することを約束しました。 専用サーバーへのオペレーティングシステムの完全自動インストールを導入できることを嬉しく思います。



多くのお客様にとって、サーバーにOSをインストールするのは面倒で、あまり面白くないようです。 通常、エンジニアはこれを行い、増え続けるサーバーとクライアントにますます多くの時間を費やしました。 したがって、このプロセスを完全に自動化する一方で、クライアントにとって可能な限り柔軟でシンプルにすることが決定されました。



また、最近リリースされたCentOS 6.4をインストールする例について、記事の形式をわずかに多様化し、スクリーンキャスト（記事の最後）の形でプロセス全体を表示することも決定されました。



元のエントリは、ブログで公開されています。



おそらくOSをインストールする最も簡単な方法



サーバープロファイルから自然に開始します。サーバーのリストを開き、目的のサーバーを見つけてそのサーバーに移動します。 これをまだ行っていない場合は、サーバーにIPv4アドレスを割り当ててください。







次に、サーバーの再インストールページに移動して、インストールに必要なデータを指定します。

  • サーバーのホスト名。デフォルトでは、csNNNN形式の一意のサーバー番号が登録されます。
  • お客様の要件と個人的な好みに応じてOSディストリビューションを選択します。
  • OSアーキテクチャ（ビットネス）。特別な要件がない場合はx86_64をお勧めします。
  • OSバージョンは、最新リリースをお勧めします（Ubuntuの場合は最新のLTSリリース）。








ソフトウェアRAIDのタイプの選択については、さらに詳しく検討されます。 専用サーバーのほとんどには2台以上のハードドライブがありますが、ハードウェアRAIDコントローラーはありません。 したがって、データストレージの信頼性を高めるために、お客様はソフトウェアRAID1（ミラーリング）またはRAID10（ミラーリングとストライピングですが、このRAIDには4台以上のドライブが必要です）を使用することを強くお勧めします。 同じ理由で、RAID0オプションの追加（ストライピング）を拒否しました。使用すると、HDDの1つが故障すると、サーバーに保存されているすべての情報が失われるためです。 現在利用可能な合計：



  • RAIDを使用しない場合、サーバーで1つのドライブのみを使用するか、ハードウェアRAIDコントローラーを使用することをお勧めします。 インストールは最初のディスクで実行され、残りは未割り当てのままになります。
  • RAID1は最初の2つのディスクのアレイにアセンブルされ、残りのディスクは未割り当てのままになります。
  • RAID10は最初の4つのディスクを使用し、残りは未割り当てのままになります。




1ディスクのサーバーでRAID10を構築しようとしないでください。OSインストーラーはこのジョークを理解できない可能性が高いことに注意してください。 特に任意の構成サーバーを考慮すると、サーバーのすべての可能な組み合わせを考慮することはできません。 したがって、この問題については、慎重になります。



最後の、しかし非常に重要な段階は、ディスクレイアウトの選択です。 必要なパーティション：/ boot（1 GB）、/（少なくとも5 GB）、swap（少なくとも4 GB）。 セクションをいくつでも追加して削除できます。 セクションごとに、GB単位のサイズが示され、スペースを埋めるためにセクションの1つにマークを付ける必要があります。 ファイルシステムとして、デフォルトでext4を使用することをお勧めします。また、一部のオペレーティングシステムはFSの選択に特定の制限を課していることに注意してください。







すべての設定を選択して確認した後、「再インストールの開始」をクリックして、ダイアログボックスで確認します。 ここでも、再インストール時にすべてのディスクからすべてのデータが削除されることに注意してください。 4つのディスクがあるサーバーで「RAIDなし」パーティションを選択した場合でも、4つのディスクはすべて消去されます。 ほとんどのインストーラーは、パーティション分割中にディスク上のRAID / LVMメタデータの残りを見つけるときに困惑するため、これが必要です。







確認後すぐに、サーバーはIPMIを介して再起動し、ネットワーク経由で適切なOSインストーラーの読み込みを開始します。 OSのインストール中は再インストールページがブロックされ、代わりに対応するメッセージが表示されます。 サーバーが間違えたことがわかった場合は、「再インストールのキャンセル」ボタンをクリックして、いつでもインストールを中止できます。 この場合、サーバーはすぐに再起動し、ネットワークダウンロードは最初のハードドライブからの起動に切り替わります。 インストールプロセスは、同じページにあるリンクであるKVMコンソールを介して監視できます。







サーバーのインストールには、サーバーのパフォーマンスとOSディストリビューションに応じて5〜10分かかります。 インストール時間を短縮し、信頼性を高めるために、mirror.selectel.ruミラーが使用されます。 サーバーのインストール中に、たとえばインターネットアクセスが中断されたなどのエラーが発生した場合、最初にサーバーを再起動しようとすると、インストールが再開されます。 それでも解決しない場合は、チケットシステムを通じてテクニカルサポートに連絡してください。



これで、サーバーが起動し、さらに構成と操作を行う準備が整いました。







デフォルトでは、すべてのシステムは可能な限り低い構成でインストールされます。絶対に必要な場合を除き、システム設定は変更されません。 すべてのシステムで、sshサーバーのみが自動的にインストールされ、オンになります。 インストールパスワードは、インストール後にサーバーの再インストールページで表示でき、再インストールのたびに新しいパスワードが生成されます。 Ubuntu OSをインストールすると、rootユーザーは無効になり、ユーザー名「user」のユーザーが作成されます。他のすべてのシステムでは、ユーザーは「root」です。 また、サーバーのインストール後に自分でパスワードを変更し、新しいユーザーを作成して、そのユーザーのssh rootアクセスを無効にすることも常にお勧めします。

テクニカルFAQ





ドライブはどのようにマークアップされますか？ ドライブのパーティション分割では、LVMとmdraidを使用します。

  • RAIDなし

    • sda1 /ブート1GB;
    • LVM用のsda2 PV。
    • vg0内のLVとしての他のすべてのセクション。
  • RAID1

    • sda1、sdb1- md0 RAID1 /ブート1GB;
    • sda2、sdb2- md1 LVM用RAID1 PV。
    • vg0内の他のすべてのLVパーティション。
  • RAID10

    • sda1-sdd1- RAID1 /ブート1GB;
    • sda2-sdd2- LVM用RAID10 PV;
    • 他のすべてのセクションは、vg0内のLVとして。




なぜLVMなのか？ LVMを使用すると、パーティションの編集時に柔軟性が増し、パーティションのサイズ変更、マージ、追加、削除が比較的簡単になります。 また、LVMを使用すると、インストールパターンを統一し、ディスクをマークする際の問題を回避することができました（拡張パーティションとその番号付けを使用する必要はありませんでした）。



/ bootが分離されてとても大きいのはなぜですか？ ほとんどのオペレーティングシステムでは、ルートパーティションにLVMまたはRAIDを使用する場合、/ bootパーティションを個別に移動することをお勧めします。 1GBはそれほど大きくありません-500GBドライブの約0.2％です（サーバーで使用可能な最小の非SSDドライブ）。 同時に、これはinitrdまたはいくつかのdevカーネルを備えた数十の通常のカーネルを格納するのに十分であり、そのサイズは200MBを超えることができます。 / bootパーティションのサイズを変更するのは非常に面倒なので、欲張りにならないことにしました。



ディスクに空き領域を残したいのですが、「領域を埋める」のチェックを外すことはできません。 これはバグですか？ いいえ、これはバグではありません。ほとんどのインストーラーは、パーティションの1つをすべての空き領域に拡張する必要があります。 この問題の解決策として、ダミーパーティションを作成し、「スペースを埋める」チェックボックスをオンにして、システムのインストール後に削除することをお勧めします。



どのように機能しますか？ すべてのシステムは、ディストリビューションに応じて、preseed、kickstart、またはautoyastを使用して通常のインストーラーによってインストールされます。 現在利用可能：

  • Debian 6、7;
  • 12.04から13.04のUbuntu;
  • ブランチ5. *および6. *のCentOSの現在のバージョン。
  • ライセンスなしのRedHat Enterprise Linux 6.4（別途購入）。
  • OpenSUSE 12.3。
