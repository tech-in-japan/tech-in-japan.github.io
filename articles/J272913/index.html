詳細：AudioCodes Multimedia Gateway

こんにちは



この記事では、個別のラインのAudioCodes機器に関する個人向けおよび小規模企業向けのユニバーサルマルチメディアゲートウェイに関するオンライン講義と記事のセッションを開始します。 これは、SOHOと呼ばれる同じセグメントです。 この記事では、デバイスのラインの概要と情報、開発の歴史、機能の簡単な説明を紹介します。 ユニバーサルCPE AudioCodes MediaPack 26xシリーズ（MP26x）について話している。



このラインの開発の進化は8年以上前です。 それはすべて、MP20xシリーズのデバイスから始まりました。これは、単純な組み込みルーターを備えた単純なATA（アナログ電話アダプター）でした。 いくつかのモデルがありましたが、それらのほとんどはもう利用できません。 ただし、1つ、2つ、および4つのFXSポートを備えたMP201、MP202、MP204デバイスは、それぞれ大量に製造および販売されています。







これらのデバイスに基づいて、2010年に更新されたCPEライン-MP252が作成され、その後、スムーズかつ慎重にMP26xに置き換えられました。 現在、デバイスのMP26xラインは、個人からの加入者を接続するためのラインの主要なデバイスおよびCPEであり、主にキャリア市場向けに位置付けられています。 MP264機器は、すでに通常の小売チェーンを通じて販売されています。



AudioCodes MediaPack™シリーズ26x、272シリーズは、マルチサービスネットワーク向けの高度な多機能マルチメディアホームまたはオフィスゲートウェイです。 イーサネットネットワークおよびブロードバンドアクセスネットワークへの接続をサポートします。 ゲートウェイには、ADSL / ADSL2 + / VDSLをサポートする内蔵モデムがあり、WiFiとDECTの2種類のワイヤレスネットワークをサポートしています。 プラガブルDECTハンドセット（最大5ハンドセット）は、高品質のHD Voice over IP（高解像度VoIP）の送信をサポートします。また、電源バッテリーを使用できます。これにより、停電の場合にゲートウェイに3〜4時間の動作が提供されます。 USBポートの存在は、家庭やオフィスでの使用に追加の利点をもたらします。 MP-26x、MP-272デバイスは、トリプルプレイマルチサービスネットワークまたは小規模オフィスで作業するための真のオールインワンデバイスです。







既存のモデルの比較特性は、WANインターフェイス、LANインターフェイス、それらの速度、組み込みWi-Fiの可用性、DECT、その他の音声機能など、いくつかのパラメーターに従って作成されます。

デバイスの変更には、いくつかの必須コンポーネントが含まれます。

•4つのLANポート-セットトップボックス、デスクトップコンピューター、またはイーサネット接続を必要とするその他のさまざまなデバイスなど、あらゆる種類のIPデバイスを接続できます。

•4番目のLANポートは複合モードで動作します-構成に応じてLANまたはWANイーサネットのいずれか

•ビルトインWiFiアクセスポイント

•2つまたは3つのUSBポート-ファイルまたはプリントサーバーの作成、3G / 4Gモデムの接続

•2つのFXSポート-あらゆる種類のアナログ電話またはファックス機の接続用

•ADSL / VSDLポート（ビルトインADSL / ADSL2 + / VDSLモデム）



拡張モデルには、さらに次のものがあります。

•内蔵DECTベース（最大5台のハンドセットをサポート）

•携帯電話のアドレス帳と組み込みDECTベースの同期のためのBluetoothサポート



MP272モデルには、ADSL / ADSL2 + / VDSLモデムが組み込まれていませんが、代わりに専用のWAN GEポートがあります。

範囲と機能に関するより詳細な情報は、 ここにあるロシア語の簡単な説明にあります 。 私たちの公式ウェブサイトの特別なセクションを使用することもお勧めします： www.audiocodes.com/mp26x



それとは別に、MP26xを使用する場合の電話通信の品質について言及する価値があります。 ここでは、2つの側面について述べることが重要です。 まず、AMRコーデックを含む幅広いコーデックを使用する機能。 このコーデックはデバイスでサポートされており、その使用は品質において否定できない利点をもたらします。 次に、デバイスの機能には、革新的なABE（Artificial Bandwidth Extension）技術を使用する機能（AudioCodeの開発）が含まれます。 装置のこの機能は、狭帯域の音声信号から広帯域の音声信号を再現することです（音声信号の帯域幅は数学的方法によって拡張されます）。



これにより、音声感度が向上します。 加入者は、標準のアナログ回線に電話をかける場合でも、HD品質の音声を聞きます。 したがって、DECT加入者だけでなく、FXS加入者も高品質の音声を楽しむことができます。



次に、管理と診断の観点からデバイスの機能について少し話しましょう。 現在存在するほぼすべての管理機能がサポートされています。



1.構成および管理用の組み込みのRussified Webサーバー。 これは、デバイスを管理および構成するための最も手頃で最も一般的な方法です。

2.プロトコルTR-069、TR-104、TR-111のサポート。 たとえば、小規模なプロジェクトのACSサーバーとして無料のOpenACSを使用できます。

3. DHCPを使用した自動構成モードのサポート-オプション66、67

4.コマンドラインから設定する機能。 これは常に便利であるとは限らず、そのような管理方法について十分なスキルと知識を持っているユーザーにより多く適用されます。

5.構成ファイルを使用します。 さらに、構成ファイルはテキストファイルであることに注意してください（ただし、パスワードは暗号化された形式で保存されます）。 さらに、構成ファイルを暗号化し、暗号化された形式でデバイスにダウンロードできます。 CPEは、このようなファイルを独自に復号化します。

6. HTTPプロトコル（web、TR-69など）を使用するメカニズムを使用してデバイスにアクセスできない場合のリモートプロビジョニングの特別な機能のサポート。 これは、CPEがNATの背後または複数のNATの背後のユーザーネットワークの深い場所にあり、Webインターフェイスを介して管理できない場合に、デバイスの大量プロビジョニングのトピックにより関連しています。



診断とtrableshutingに関しては、次の点を強調しています。



1.トラブルシューティングと診断のための高度な機能-組み込みのtcpdump、syslog、およびパケット記録（プロセッサDSPから直接の情報）

2.音声品質とQoEを制御する組み込みエージェントBroadSoft PacketSmart。 このメカニズムは、PacketSmartソリューションを使用するオペレーターとその加入者に適用されます。

3.機能サポートLST（ローカルサポートとトラブルシューティング）-自動トラブルシューティングとユーザーアラート（呼び出しを試みたときのWebリクエストと音声アラートのインターセプト）-Russified。 この機能により、最も一般的な問題についてユーザーに通知できます。 たとえば、切断されたケーブルやプロキシサーバーでのSIP登録の不足について通知する場合。 接続の問題を探すために、どこでどのプライマリ情報を使用するかがすぐに明らかになります。
