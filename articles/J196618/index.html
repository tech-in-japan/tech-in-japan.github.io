クラウドアプリケーションの開発とテストの概要

2013年6月上旬、TechEdカンファレンス（北米）で、マイクロソフトテクニカルカウンシルメンバーであり、ALM製品開発のリードスペシャリストであるブライアンハリーは、.Net Framework 4.5.1と組み合わせてVisual Studio 2013およびTeam Foundation Server 2013を発表しました。 数週間後、サンフランシスコのBuild 2013は、これらの製品のプレリリースバージョンの提供を発表しました。 開発環境の多数の改善に加えて、Windows 8.1、オンプレミスおよびクラウド環境用のWebアプリケーションとサービスを含むWindowsベースのデバイス用のアプリケーションを作成する機能は、Webを簡素化するVisual StudioおよびASP.NETへの新しい追加を使用しますさまざまなブラウザーとデバイスの開発、プロジェクトポートフォリオの柔軟な管理、リアルタイムの対話、およびコードエディターから直接適切な情報へのアクセスを簡素化するVisual Studio 2013は、開発、診断のための完全なツールです 壱岐、クラウドでアプリケーションをデバッグします。





この記事は、Visual Studio 2013 Previewを使用してWindows Azureアプリケーションをゼロから作成およびデプロイし、クラウドでテストおよび管理することを初心者デベロッパーが理解できるシリーズの最初の記事です。 実際には、Visual Studio 2013 PreviewとWindows Azureのサブスクリプションが必要です。 MicrosoftダウンロードセンターからVisual Studio 2013 Previewの無料プレビューバージョンをインストールできます。 利用可能なエディションがいくつかありますが、Ultimateエディションが必要です。 インストール後、クラウドでアプリケーションを作成するためのプロジェクトテンプレートを確認できます。





図1



このテンプレートは最初は空です。 VS2013用のWindows Azure SDK 2.1のインストールが必要です。これは、VS2012および2010 SP1でも機能します。 このタイプの新しいプロジェクトを開くと、インストールが自動的に開始されます。 SQL Server Data Toolsなどの他の追加ツールと同様に、デフォルトのインストールはWeb Platform Installerを使用します。 クラウドアプリケーションの作成が行われる開発者のマシンがインターネットにアクセスできると想定するのは論理的です。 ただし、開発とテストがローカルで実行され、クラウドへの展開がユニット内のいくつかの専用マシンから実行される状況があります。 この場合、 指示に従ってWindows Azure SDKをローカルマシンに「手動で」インストールします（インストール手順を参照）。 システムの前提条件も「システム要件」セクションにリストされています。 WindowsAzureAuthoringTools-x64.msi、WindowsAzureEmulator-x64.exe、WindowsAzureTools.vs120.exe、およびその他のコンポーネントを個別にここからダウンロードできます。

Windows Azure SDKをインストールすると、プロジェクトテンプレート（図1）のGet Windows Azure SDKの碑文がWindows Azureクラウドサービスに変更されます。 作成したアプリケーションにASP.NET Webロールを追加します。

図12



アプリケーションが機能していることを確認した後、その配置に進み、クラウドで起動します。 以前、Windows AzureでWebロールを実行するには、Windows Azure管理ポータルに移動するクラウドサービスを作成する必要があります。左下隅の+アイコンをクリックし、表示されるメニューで[クラウドサービス]を選択します。 [カスタム作成]オプションを使用すると、構成ファイル（CSCFG）、パッケージ（CSPKG）、および証明書を指定することにより、クラウドサービスをすぐに公開できます。 Visual Studioプロジェクトからアプリケーションを公開するので、クイック作成オプションで十分です。 この場合、サービスの名前を考え出し、それが機能するデータセンターを設定するだけです。 原則として、データセンターは、同じデータセンター内で無料であるトラフィックのコストを最小限に抑えるために選択されます。 たとえば、アプリケーションが北ヨーロッパのデータセンターで作成されたSQL Azureデータベースで動作する場合、そこにサービスを配置するのも理にかなっています。

図14



「正直な」クラウドサービスとストレージアカウントを使用するために、エミュレータからVisual Studioでプロジェクトを再構成することは残ります。 これを行うには、Windows Azureプロジェクトのソリューションエクスプローラーで、ストレージサービスにアクセスするロールのプロパティを選択し、[設定]タブに移動します。 これで、接続文字列がローカルエミュレーターに設定されます：UseDevelopmentStorage = true。
