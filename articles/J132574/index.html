Red HatはEMC / VMwareでGlusterをターゲットにしています

Red Hatは、クラウドストレージのオープンソース開発者であるGlusterの買収を発表しました。 Glusterソフトウェアは、CloudFormsやOpenShiftなどのRed Hatクラウド製品のスタックを補完して、柔軟性とスケーラビリティを実現します。


Red Hatは「クラウドストレージ」Glusterを購入しています。 なんで？

少し前に、私はRed HatがVMwareに対して「クラウド」の面で戦争を宣言したという事実について書きました。



Glusterの類似の購入ニュースには多くの一般的な言葉があり、Red Hatがこの購入を行った理由とクラウドコンピューティング市場にどのように影響するかについての分析はほとんどありません。 私は自分で考えなければなりませんでした。 :-)さまざまな情報源から情報を収集しましたが、状況を少し明確にできることを願っています。



この購入はクラウド市場にどのように影響しますか？

Red HatがクラウドストレージのメーカーであるGlusterを購入した理由を理解するには、クラウドが必要な理由を理解する必要があります。 クラウドコンピューティングを導入する主な理由は、ITコストを削減することです。 そして、より正確に-それらを効果的に使用するために、すなわち コンピューティングリソースの全体的な低コストを実現します。 ただし、クラウドコンピューティングの低コストを実現するには、「クラウド」を構成するすべてのコンポーネントに低価格でアクセスできることが必要です。



そして今日、「クラウド」を作成するためのほとんどすべてが比較的安価に購入できます。結局のところ、それらは標準のx86サーバーから組み立てられ、多くの場合、ソフトウェアとしてオープンソースを使用します。 待って、「ほぼすべて」と言ったのですか？ 独自の「クラウド」を構築する場合、x86サーバー、ネットワーク機器を使用します。これは非常に安価です。 それから？ そして、処理されたデータを別の場所に保存する必要があることを理解します。 残念ながら、今日のストレージシステム（以下SHDと呼びます）だけでは、目に見えるお金がかかり、クラウドコンピューティングのメリットを十分に享受できない価格の「ボトルネック」です。



なぜこれが起こっているのですか？ はい、鉄の生産者も余分な利益を得たいからです。 また、ストレージ市場で最大のプレーヤーの1つであるEMCは、偶然にもVMwareの所有者です。 仮想化市場を支配しているのと同じ会社。 ただし、IBM、HP、およびSun / Oracleの両方にも、販売が必要な独自のストレージシステムがあります。 私たちが話している金額を評価できるように、ロシアのストレージ市場へのEMC、HP、IBMの配信の合計シェアは約80％です（ 出典 ）。 クラウドコンピューティングは、大規模なストレージシステム専用の機器を所有しているだけであるため、鉄の大規模なサプライヤーにとって有益です。 IDCレポートからの数字 ：
同時に、プライベート「クラウド」のストレージシステムに対する企業の費用のCAGRは28.9％になります。 2015年までに、パブリックおよびプライベートクラウドシステムのグローバルストレージコストは226億ドルに達します。



IDCの予測によれば、ロシアのストレージ市場はかなり速いペースで発展し続け、外部ストレージデバイスの金銭面でのセグメントは5年間でほぼ75％成長するでしょう。
参考までに、ロシアのストレージ市場におけるEMCのシェアは40％弱であり、EMCの「娘」であるVMwareは仮想化市場の約80％を占めています（「世界」であり、ロシアの数値は別に見つかりませんでした）。 EMCの担当者 は、以前の投稿 （価格の問題のみを取り上げた記事）の1つで 、サービス、ソフトウェア、ハードウェアのコストを個別に考慮するのではなく、まとめて考慮することを求めました。
ハードウェアのコストとは別に仮想化プラットフォームのみの価格を考慮すると、バンドル全体（バックアップ、セキュリティ、ネットワークインフラストラクチャなど）およびアプリケーションソフトウェアのライセンスのコストは実際的な意味を持ちません。
独自のクラウドを構築したい人は誰でも、最終的にどれくらいの費用がかかるかを計算することから始めます。 また、ソフトウェア、サーバー、ネットワーク機器ですべてが多かれ少なかれ明確であれば、ストレージではまったく逆です。 たとえば、「間違った」ソフトウェアプラットフォームを選択すると、価格が大幅に変更される可能性があります。 ;-)そして、今日、クラウドコンピューティングの大規模な使用を妨げる障害となっているのはストレージです。



Red Hatの担当者は、仮想化に費用をかけるべきではないと繰り返し述べています。 したがって、私の意見では、Glusterの購入は、クラウドストレージの最も高価な部分であるデータストレージのコストを削減する試みです。 そして、EMC / VMwareはそれを気に入らないでしょう。 ;-)



そして、Glusterとは何ですか？

Glusterは、 NASストレージ用に設計され、オープンソース（GPLv3）として配布されるファイルシステムGlusterFSで知られるようになりました。 GlusterFSは最大数ペタバイトの容量のアレイを処理できますが、アレイのコンポーネントは互いにかなり離れている場合があります-Infiniband RDMAまたはTCP / IPプロトコルを使用して組み合わされます。 Glusterは「水平方向にスケーラブルなオープンソースストレージソリューションの主要プロバイダー」と呼ばれています。 従業員の一人が美しく言いました：
私たちの目標は、ストレージの分野でRed Hatのようになることであり、Red Hatのストレージになりました。
Gluster製品はRed Hatのクラウド戦略に完全に適合し、CloudFormsおよびOpenShiftの重要な要素として機能しますが、Red Hatに非構造化データ管理市場に参入する機会も提供します。 Red HatはOpenStackプロジェクトに関与していませんが、このクラウドプラットフォームに対するGlusterのサポートは継続され、製品のソースコードはオープンなままであることに注意してください。 （ただし、Red Hatの製品コードは常にオープンであり、オープンのままです。）



しかし、グローバルファイルシステムについてはどうでしょうか。

Red Hatは、ファイルシステムとストレージの分野ですでに1つの開発、つまりグローバルファイルシステムを開発しています。 GFSはほとんどの場合、Red Hat Cluster Suiteの一部として実行され、複数のサーバーがSAN上の単一の共有ファイルシステムに対して同時に読み書きできるようにします。 名前に「グローバル」という言葉がありますが、このファイルシステムは主にフェールオーバークラスター（HA、高可用性）を対象としており、厳密に言えば、クラウドコンピューティングやそのような大規模な展開について誰も考えていなかった時期に開発されました。



ところで、GFSには興味深い話があります。 このFSの開発は、1995年にミネソタ大学で始まりました。 SGIのIRIX用に書かれ、1998年にLinuxに移植されました。 ほぼ同時期に、FSのソースコードが公開されました。 しかし、開発のシスティーナソフトウェアへの移行（〜2000）の後、GFSはプロプライエタリ製品として開発を開始しました。 独立した開発者はFSを分岐して新しいOpenGFSプロジェクトに名前を付けましたが、長続きしませんでした-Red Hatは2003年にSistinaを買収し、GPLライセンスの下ですべてのコードを再開しました。



また、この購入はRed Hat自体にとって何を意味しますか？

過去数年にわたって、Red Hatビジネスの焦点は変わりました。以前にオープンソリューションがビジネスに不可欠な別のサーバーで使用されていた場合、これらはデータセンター全体になります。 これは、クラウドコンピューティングに直接関連するサービスプロバイダーや企業がますます多くなるOVAの例で特に顕著です。



ところで、先日Red Hat がOpen Compute ProjectのRHEL認定を発表したという事実は、「データセンターのコスト削減」のベクトルを支持します。 大規模データセンターの効率的かつ経済的なスケーリングのためのアルゴリズムを開発するために、FacebookによってOpen Compute Projectが開始されたことを思い出させてください。



Red Hatには独自の仮想化（KVM）、IaaS（CloudForms）、およびPaaS（OpenShift）があり、Glusterの購入により、ストレージエリアの「ホワイトスポット」は閉鎖されます。 今日のオープンソースソフトウェアは、クラウドコンピューティングを構築するための事実上の標準になりつつあります。 最大規模のクラウドホスティングサービスはすべて、特にオープンソース製品で機能するため（コストを大幅に削減するのに役立ちます）、Red Hatがこの市場で大きなチャンスを得るのは驚くことではありません。



GFSの出現がミッションクリティカルなシステムでLinuxを使用するための重要なステップであったように、GlusterFSはデータセンターおよびクラウドコンピューティングにおけるLinuxのアプリケーションの新しいページを開くと信じたいと思います。 価値のある「産業用」の代替手段（当時はGFSなど）がないだけでなく、「クラウド」データストレージのコストを大幅に削減することもできます。
