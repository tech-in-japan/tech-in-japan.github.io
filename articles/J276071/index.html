OracleはJavaブラウザプラグインを廃止

2015年末までに、主要なブラウザの開発者は、組み込みプラグインのサポートを削除するか、そのような計画を発表しました。 たとえば、Mozilla は NPAPIプラグインのサポートを拒否します。これは、それらの機能のほとんどが現在標準のWebAPIを介して利用できるためです 。 Chromeは、昨年4月からNPAPIプラグインをブロックしています 。 Edgeブラウザはプラグインもサポートしていません。 したがって、すべての主要なブラウザでは、Flash、Silverlight、Javaなどの組み込みサポートはまもなく提供されなくなります。



この点に関して、Oracle は 、バージョンJDK 9以降、Javaブラウザプラグインのリリースを停止すると発表しました。 将来のJava SEリリースでは、Oracle JDKおよびJREから削除されます。



プラグインがブラウザでサポートされていない場合、なぜプラグインを開発するのですか？



Web開発者は、代替ソリューションに目を向け、Javaアプレットの使用を拒否することをお勧めします。たとえば、プラグインを必要としないJava Web Startテクノロジーを支持します。 移行オプションの詳細については、 リファレンスガイド（pdf）を参照してください 。



JDK 9の予備バージョンへのアクセスは、 http ： //jdk9.java.net/にあります。



長年にわたり、Javaプラグインはユーザーのコンピューターにマルウェアをインストールする主な方法の1つでした。 すぐに、この方法は適切ではなくなります。



JDK 9のリリースは、当初2016年9月22日に予定されていましたが、2017年3月23日に延期されました。
