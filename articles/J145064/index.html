GNU / Linux用のフリーでないゲーム：良いか悪いか？

デジタル制約管理（DRM）を備えた独自のデジタルゲームを配布する有名な会社であるValveは、最近これらのゲームをGNU / Linuxで配布することを発表しました。 これはどのような良い結果と悪い結果をもたらしますか？



GNU / Linuxの下で人気のあるプロプライエタリなソフトウェアが利用可能になると、システムの開発に貢献できると思います。 しかし、私たちの計画は、このシステムを「成功」させる以上のものです。 その目的は、ユーザーに自由をもたらすことです。 したがって、問題は、この変更がユーザーの自由にどのように影響するかです。



非フリーゲーム（他の非フリープログラムと同様）は、ユーザーの自由を否定するため、非倫理的です。 自由が必要な場合、条件の1つは、コンピューターにこれらのゲームがないことです。 これまでのところ、すべてが明確です。



ただし、これらのゲームを使用する場合は、Microsoft WindowsではなくGNU / Linuxでこれを行うことをお勧めします。 あなたは、少なくともWindowsがあなたの自由に与える害を避けます 。



したがって、直接的な実用的な意味では、この変更は有害で有益な場合があります。 彼女は、GNU / Linuxユーザーにこれらのゲームをインストールするように勧めることができ、ゲームユーザーにWindowsをGNU / Linuxに置き換えるように勧めることができました。 私の意見では、直接的な有益な効果は直接的な害よりも大きいでしょう。 しかし、まだ間接的な効果があります。これらのゲームを使用すると、コミュニティの人々に何が教えられますか？



これらのゲームを提供するプログラムに付属するGNU / Linuxディストリビューションは、ユーザーに自由に関するものではないことを教えます。 GNU / Linuxディストリビューションの専有ソフトウェアは、すでに自由の目標に反して機能しています。 これらのゲームをディストリビューションに追加すると、この効果が高まります。



自由を促進したい場合は、GNU / Linuxでのこれらのゲームの可用性について、私たちの大義を支えるものとして話さないように注意してください。 代わりに、 無料のPixel Cup無料ゲーム大会やFree Planet Game Societyの無料ゲームナイトについて人々に伝えることができます。
