切り取り/コピー/貼り付けおよびCtrl + Alt + Delコマンドの履歴

毎日、私たちはコンピューターの多くの機能を、どこから来たのかを少しも考えずに使用しています。 そのため、コンピューターの最も単純な機能でさえも敬意を払うために、少し歴史のレッスンを教えることは素晴らしいことです！



履歴「切り取り/コピー/貼り付け」



コピー、カット、ペーストを使用せずにコンピューターで作業することを想像できますか？ 彼らは私たちの日常生活の不可欠な部分になっているので、彼らなしでは生きることはほとんど不可能に思えます。



コンピュータが誕生する前から、「カットアンドペースト」という用語は、原稿を編集するときに文字通りシートから何かを切り取って別のページに貼り付けるときに広く使用されていました。 それは、コンピューターでのテキストの編集に関して「カットアンドペースト」という表現が使用され始めた1974年までで、ゼロックスのラリーテスラーがこれを行いました。



1981年、Appleは、グラフィカルコンピュータであるLisaのリリースでCut＆Pasteを普及させました。 このAppleはキーボードショートカットを標準化しましたが、切り取りは+ X、コピーはC、貼り付けはVでした。 マイクロソフトは後に、ほぼ同一のキーボードショートカットを使用して、これらのキーボードショートカットをWindowsに適合させました。



履歴Ctrl + Alt + Del



Ctrl + Alt + Delは、ダングリングアプリケーションを閉じたり、再起動したり、場合によってはシステムに入るためのよく知られたキーの組み合わせです。



誰がキーボードショートカットを導入しましたか。これは多くの人々の生活様式になっていますか 最初のIBM PCの設計者であるDavid Bradley。 彼は、コンピューターがクラッシュするたびに電源をオフにし、数秒待ってから電源をオンにしてブートプロセス全体を再度実行する必要にうんざりしていました。 最初、Bradleyはホットリブート用にCtrl + Alt + Escの組み合わせを開発しましたが、この組み合わせを片手で誤って押すことができることに気付きました。 EscapeキーをDeleteに置き換えて、リブートには両手が必要であることを確認しました。
