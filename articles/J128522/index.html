Jelastic PaaS-メジャーアップデート：水平スケーリング、Java 7、Jetty、HTTPS





Javaアプリケーション用のクラウドプラットフォームであるJelasticの大きな更新に関する素晴らしいニュースを急いで共有します。 最新リリースは本当に大きいです。 待望のが追加されました。

  • 水平スケーリング
  • Java 7サポート
  • 桟橋6
  • Https
  • トポロジー編集


水平スケーリングは、高負荷で最も一般的に使用されるソリューションの1つです。 実装では、Webサーバーの複数の並行コピーを非常に迅速に取得できます。 これで、サーバーをその場で追加および削除できます。 ただし、いずれにしても、アプリケーションは複数のサーバーで動作するように設計する必要があります。 垂直スケーリングとは異なり、水平スケーリングでは、アプリケーションが水平に拡大できることを「理解」する必要があります。 つまり 開発者はこれらの可能性を考慮する必要があります。



現在、開発者は、2種類のスケーリングを組み合わせて、アプリケーションの効率を最大限に高めるユニークな機会を得ています。 クラウドソリューションでのこのような機能の提供は、現在、世界規模でユニークです。 現在の競合他社には、このような機能はありません。



JDK 7をサポートします。 これで、新しいバージョンのJDKで多くのことを試すことができ、アプリケーションが新しいバージョンのJavaとどの程度互換性があるかを確認できます。 環境を再作成したり、再適用したりすることなく、コントロールパネルでJDKバージョンを切り替えることができることが重要です。



Jettyサポート 。 現時点では、Jetty 6のサポートを追加しました。これは、Tomcatに次いで2番目に人気のある「軽量」Javaサーバーです。 今後のリリースでは、Jetty 7とJetty 8が追加されます。



HTTPSサポートは 、最も要望の多い機能の1つです。 HTTPSは、マウスを1回クリックするだけで含まれます（ただし、他のすべてのものと同様）。 現在のリリースでは、* .jelastic.comに基づいて内部ドメインにHTTPSを提供しています。 将来のリリースでは、個人証明書をアップロードし、外部ドメインのHTTPSを有効にすることが可能になる予定です。



環境トポロジの編集 。 現在のリリース以降、Jelasticは以前に作成された環境の編集をサポートしています。 データベースをその場で追加または削除したり、アプリケーションサーバーの数を変更したり、バランサーを追加または削除したり、環境トポロジを編集するための基本的な操作をすべて実行したりできます。



+当然、構成エディターの改善、ログを全画面モードに展開する機能、ファイルのバッチアップロードなど、その他の小さくても便利な機能が多数追加されました。



次のリリースが計画されています

  • レプリケーションセッション
  • ビルドノード（maven、git-push-deploy）


JavaOne 2011への参加



10月2〜6日にサンフランシスコで開催されるJavaの世界最大のイベントであるJavaOne 2011に参加することを知りたいと思うかもしれません。 独自のデモスタンドがあります。 このイベントへの参加に成功した方がいらっしゃいましたら、喜んでご連絡し、現在の機会と将来の計画について詳しくお伝えします。



米国のデータセンターで開始



もう1つの良いニュースは、米国でパートナーを見つけ、すぐに米国の近代的なデータセンターの1つでJelasticを立ち上げることです。 協力の詳細は明らかにされていない。 これは、ソリューションを顧客に提供する2番目のホスティングパートナーです。 最初はヨーロッパです。 また、CISのホスティングプロバイダーとの交渉が進行中です。 データセンターの選択において最大限の柔軟性を提供するよう努めます。
