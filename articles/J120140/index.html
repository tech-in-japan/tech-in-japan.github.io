こんにちは、Habr！



良い一日！ こんにちは 私たちはロンドンからのbrief.lyスタートアップから来ています。 ウクライナの友人から、強力なHabrahabrコミュニティについて学び、私たちの裁量で革新的な製品を紹介することにしました。



彼があなたの手でどのように振る舞うか、あなたの意見は何ですか、あなたは何が好きで、何がそうではありませんか？ 別に何をしますか、そして何を誇りに思いますか？



インターメッツォ



インターネットアドレス削減サービスの数は、誰にも知られていない。 まさに千以上。 深刻な企業にはそれぞれ独自の短いドメインがあります。 Twitterは業界全体を生み出しました。 追加機能のみが異なります。 ただし、それらはすべて1つのURLを取得し、そのURLから別のURLを作成します。



背景



私たちは新しいことをすることにしました。 たくさんのリンクを取り、すべてをタブに入れて、1ページとして転送してみませんか？







1年前、プログラマーはname.ly/framesフレームテーマを開発しました。これにより、組み込みのインターネットブラウザーのように、多くのリソースをタブでダウンロードできます。 たとえば、有名なウクライナとロシアの政治学者であるヴィタリーポルニコフのサイトはすべて、sincere.ly / portnikov-およびportnikov.com/sitesに反映されています 。







name.lyの欠点は、サイトの複雑さと継続時間でした。 登録し、フレームテンプレートを選択し、各リンクを個別に入力する必要がありました。 約5分かかりました。



簡単に



私たちは多くの実験を行い、すべての巧妙な作業を数秒で完了する必要があるという結論に達しました。 したがって、name.lyに基づいて、brief.lyサービスが誕生しました-多くのサイトを即座にコンパイルするためです。



サイト編集







Brief.lyは次のように機能します。テキストボックスで、リンクのリストと各リンクのスペースを送信できます。 署名のないリンクを簡単にコピーして貼り付けることができます。brief.lyは、彼自身がドメイン名に基づいて碑文を提供し、コンパイル自体を作成する前にそれらを編集できるようにします。 追加のオプションでは、元のページ名をダウンロードすることができます。特定のリソースの速度に応じて、リンクごとに最大5秒ほどの時間がかかります。



すべてのサイトが同時に読み込まれ、人々とそのインターネットが過負荷にならないように 、タブの数を36に最適に制限しました。これは、コンパイル済みのリンクをコンパイルしてネストされたサブタブを作成することで回避できます。



ショートリンク



コンパイルの結果として、指定されたすべてのリンクを持つページが作成されます。 リンクが1つしかない場合、すべてが従来のショートカットのように機能します。 さらにリンクがある場合、対応するコンテンツは個別のタブにロードされます。



brief.lyのこのページに加えて、ユーザーは魅力的なドメインのリストからリンクを選択できます。短いコードが気に入らない場合は、新しいランダムコードを生成できます。







Brief.lyは、人々が自分自身を表現できるようにします。 マザープラットフォームname.lyは2008年に作成されました。 現在では、英語でフレーズとして読み取られるドメインのセットです。 「ブリーフフライ」-「簡潔に」、「名前のある」-「名前のある」、「誠実に」-「敬意を表する」、「多くの」-「多くの」、「2つのリンク」、「私にリンクする」、それは私です-「これは私です」、「もちろん」-「もちろん」、「他に何」-「他に何ですか？」、「どこに他」-「どこに他？」、「誰が他」-「誰他に何？」 在庫があり、同様の性質を持つ200を超えるドメイン。 これが冗談であるか真実であるかはまだ正確にはわかっていませんが、彼らはかつてジョージクルーニーとネスプレッソの一連の広告からインスピレーションを受けたと言います ： ネスプレッソ？What-El.se？



フレーズはそこで終わりません。 Name.lyには、アスタリスクと矢印の2つの短いドメインがあります。 たとえば、コーヒー広告サイトは、 http：//✯.ws/~3NXLまたはhttp：//➸.ws/~3NXLのリンクで動作します。



運営管理



「管理」の3番目の段階で電子メールとパスワードを入力すれば、登録せずに作成した編集物の統計を管理および表示できます。 これを行うには、既存のユーザー名を使用するか、単に新しい詳細を指定します。







管理パネルにすばやくアクセスするには、メインURLのスラッシュの後に@を追加するだけです。



統計



詳細な訪問統計は、ページ管理者のみが利用できます。 デフォルトでは、作成者のみがこのデータにアクセスできます。 必要に応じて、それらを公開できます。 Google Analyticsコードを追加することもできます。 統計をすばやく表示するには、メインURLのスラッシュの後に+を追加するだけです。

API



タブを作成するには、brief.lyサイト自体にアクセスする必要さえありません。APIを使用して、URLリクエストまたはJavaスクリプトを使用して必要なサイトをフレームに挿入できます。 詳細についてはbrief.ly/apiをご覧ください 。



あなたのサービスが私たちの機能を必要としているなら、私たちに書いてください、私たちは他のサイトと協力させていただきます。 Brief.ly/APIを使用すると、カスタマイズされたテンプレートを作成し、色、ヘッダー、アバターを変更できます。



リダイレクトと編集



他に注意する必要があるのは、リンクをリダイレクトする機能です。 ニュースレターを作成した後、いつでもどこでもリンクをリダイレクトしたいとします。



iPadとホームページ



アプリケーションbrief.lyユーザーは自分自身を考え出します。 1か月前、Facebookでタブを取得するためにbrief.ly iPadブラウザーを使用してくれたことに感謝しました。



多くの場所で多くのブラウザを使用する人は、brief.lyを使用して独自の統合ホームページを作成します。 1か所で変更され、変更はすぐにどこでも発生しました。



ニュアンス



すべてが素晴らしいですが、いくつかの欠点があります。 すべてのサイトがフレームによる挿入をサポートしているわけではありません。 人気のあるロシアのリソースの中で、このような制限はYandexとVKontakteによって課されています。 TwitterおよびLast.fmのページがフレームからポップアップします。 Brief.lyはすでにこれと「戦う」ことを学んでいます。 TwitterのプロファイルとFacebookのファンページは、独自のウィジェットを使用して変換されます。 他のフレームは単にフレームに挿入されず、直接リンクによって提供されます。



人気のあるサイトの中には、フレームからポップアップするこれらのダースがあり、サービスは常にそれらをキャッチします。 ただし、インターネットの99％はフレームで問題なく機能し、お気に入りのビデオや記事を送信するのに非常に便利です。



brief.lyでは、これは英語のインターフェイスにすぎませんが、ボルシチと同じくらい簡単です。 私たちは、改善とさらに興味深いアプリケーションを約束します。 しかし、これはおそらく別の記事のトピックです。
