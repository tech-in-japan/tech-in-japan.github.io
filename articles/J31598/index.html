37signals：Photoshopでスコアを付けた理由

これは、 37signalsの Jason Friedによる記事「 なぜPhotoshopをスキップするのか 」の翻訳です。







UIを開発するとき、通常はすぐに紙のスケッチからHTML / CSSに移行します。 Photoshopのレイアウト設計が欠けています。







これが私たちがそれを行う理由です：







  1. レイアウトはクリックできません。 これがおそらく静的レイアウトが私たちに向いていない最大の理由です。 それらは本物ではありません。 もちろん、紙のスケッチも本物ではありませんが、彼にはこれを期待していません。 画面にレイアウトが表示されます。 画面に表示されると、動作するはずです。 ただし、メニューを開くことはできず、フィールドのテキストを詰まらせることも、リンクをクリックすることもできません。 また、HTML / CSSを使用すると 、すべてが現実になります。
  2. Photoshopには多くのツールがあり、詳細を把握することはできません。 あなたがそれを使うとき、それはあなたが詳細をするためにあなたを引っ張る。 整列、正しい色、正しい形状は、最終的に重要ですが、今は役に立たないささいなことです。 最初は、詳細ではなく意味が重要です。 詳細は後で。
  3. Photoshopのテキストは、Web上のテキストとは異なります。 静的なPhotoshopレイアウトのテキストをすばやく変更することはできません。Photoshopを開き、テキストを変更し、ファイルを保存し、gif / png / jpgなどにエクスポートする必要があります。スケッチをオンラインにすると、 HTMLでできます。 代わりに、「5分...」と言う必要があります。 それでも、PhotoshopのフォントがHTMLのフォントと一致することはありません。 それは決してないようです。 改行も距離もありません。
  4. Photoshopは生産性ではなく生産に焦点を合わせています。 Photoshopでは、機能するものではなく、見た目を実行します。 外観のみを気にする場合は、制作に多くの時間を費やします。 HTML / CSSを使用すると、生産性を高めることができます。 変更するたびに、より現実的なものに向かって進みます。
  5. Photoshop-1つのジョブを2回実行します。 レイアウトに3日間を費やしましたが、それでは何ですか？ HTML / CSSを取り上げ、もう一度繰り返します。 時間の無駄。 HTML / CSSから始めて、再作成ではなく、反復に余分な時間を費やします。 HTML / CSSも遅い場合は、少し時間をかけて、より高速なHTML / CSS開発方法を学んでください。 この時間の無駄は空ではありません。
  6. Photoshopはコラボレーション用ではありません。 すでに述べましたが、もう一度言います。HTML/ CSSを使用すると、変更、保存、更新ができます。 だから私たちは協力関係を築いています。 「変更します。 更新。「数秒で。 「そのため、フロートがあります。左ではなく、右です。 更新。「数秒。 ツールの選択、移動、エクスポート、注ぐ、新しいファイル名はありません。 HTML / CSSは、ラピッドプロトタイピング用に単純に設計されています。 しかし、Photoshopはそうではありません。
  7. Photoshopは複雑です。 もちろん、彼と10年間働いた後、すべてが自然に機能しますが、私にとっては、いくつかのことはまだ複雑です。 ペンを使用すると、ツールバーにたどり着くよりも管理が簡単です。 ペンで何でも描くことができますが、Photoshopでは、テキストにはテキストツールを、シェイプにはシェイプツールを使用し、メニューからすべてを設定する必要があります。


Photoshopが悪い、またはお金に見合う価値がないとは言いたくありません。HTML/ CSSをすぐに実行することは、新しいものを作成する場合と既存のものを改善する場合の両方ではるかに役立つことがわかります。 HTML / CSSは、Photoshopが決して実現しない本物です。
