ICOの9分：ビジネス向けIPOの代替案の概要





元のDTIブログの記事を読んでください 。



今日は、暗号通貨を使用して企業の主要資本を引き付ける方法を検討します。 標準的なIPOよりもかなり多くの利点があります。これにより、より多くの資金を引き付けることができ、株式取引所に株式を配置する標準的な手順よりも10倍以上少ない費用でこれを利用できます。 この方法は、クラウドセール、またはICO（Inicial Coin Offering）と呼ばれます。



IPOの代わりにICO



概して、ICOは新規株式公開（IPO、新規株式公開）の類似物と見なすことができます。 違いは、IPOの間に投資家が実際の株式を受け取り、ICOの場合、いわゆる暗号通貨、本質的に株式ではないが、投資家が会社の利益の一部を受け取ることができる暗号トークンです。 さらに、IPOは通常、（ICOとは対照的に） 国内法によって規制されています。



そのため、たとえば、米国では株式の公募のために、その発行者（必ずしも合資会社として設立される会社）はSEC（証券取引委員会）に登録する必要があります。 登録プロセス自体は複雑で時間がかかります。



同時に、会社は十分な量の情報 （財務諸表を含む） を開示する必要があります。 同時に、各証券取引所は、発行者が上場手続き （取引所リストに証券を含める）を行うための独自の要件を確立します 。 そのため、取引所は、必要な株主数 、昨年の会社の収入の大きさ、または資産の価値などを決定できます。

ICOエントリープロセス



トークンの事前販売（ICO）のメカニズムは、トークンのタイプに直接依存します。 2つの主なタイプを区別できます。



  • アプリケーショントークン
  • 株式トークン


それらの違いは、主に会社のソースコードの有無です。



重要！ 会社は、そのニーズに応じて、トークンの組み合わせを独自に決定します。 アプリケーショントークンのみ、ストックトークンのみ、またはそれらの組み合わせにすることができます。 これはすべて、 ホワイトペーパーおよびICO条件（ https://goo.gl/YTHmRAなど ）で規定されています 。



会社にはソースコード>>アプリケーショントークンがあります



アプリケーショントークン/ Appcoins-ユーザーが分散ネットワークによって提供されるサービスを受けるために必要な通貨の形式。 彼らのカウンターパートはアトラクションを訪問するためのトークンであり 、投資家には権利はなく、アプリケーションを使用する能力しかありません。

トークントークンは、暗号交換の暗号通貨で自由に売買でき、対応するネットワークで値を作成することで獲得できます（報奨金プログラム）。



収益：マイニング/ニュースの購読/ソーシャルネットワークでのいいねやサブスクリプション/キャンペーン資料を他の言語に翻訳し、コンテンツのモデレート/公開。 同様のプログラムの例： https : //goo.gl/g9PNwJ



ICO中のチームの一連のアクション（アプリケーショントークン）：



  1. 暗号通貨投資家コミュニティ（ ビットコイントーク 、 Redditなど）でのホワイトペーパーのほか、プロジェクトに関する主要な技術情報：目的、ICO時間枠、チーム、サイト開発ロードマップ、プロジェクト機能、その他の詳細 ）
  2. 最初のトークンを生成する前の、今後のICOの発表とソースコードの公開。
  3. マイニングを使用したネットワーク展開とトークン生成。 ネットワークのアイデアと開発に対する報酬として、トークンの一部を創設者に予約することができます。
  4. ICOの広告とすべての人へのトークントークンの販売（会社は受け入れられている通貨を受け入れます：ドル/ユーロ/など、暗号通貨のみ、またはそれらの組み合わせのみが可能です）。
  5. さらに、チームは計画に従ってプロジェクトの開発に取り組みます：ネットワーク効果の作成、アプリケーションの作成、およびネットワークのサポート。 ネットワークが拡大するにつれて、トークンの需要が増加し、ユーザートークンのコストが増加します。


会社にはソースコードがありません>>ストックトークン



株式トークンは、 開発とネットワーク構築の資金調達に使用されます 。 会社のサービスにアクセスするために、それらは必要ありません。 実際、それらは会社の暗号共有と見なすことができます。 投資と引き換えに、株式トークンの所有者は、収入の割合またはネットワーク内の取引の手数料の一部の形で配当金を受け取ります。



多くの場合、共有トークンはDAOなどの組織の利害関係です（これらについては以下で説明します）。 報酬を受け取ることに加えて、DAOトークン保有者は企業の投資提案にも投票します。 投票数はトークンの賭け金に比例します。



ICO（ストックトークン）中のチームの一連のアクション：



  1. 暗号通貨投資家コミュニティ（ ビットコイントーク 、 Redditなど）でのホワイトペーパーのほか、プロジェクトに関する主要な技術情報：目的、ICO時間枠、チーム、サイト開発ロードマップ、プロジェクト機能、その他の詳細 ）
  2. ネットワークの創設者向けに予約された一定数の共有トークンを使用したスマートコントラクトの作成。
  3. 有料でネットワークを開発するプロバイダー会社の設立。
  4. 今後のICOの広告と宣伝、およびすべての人へのトークンシェアの販売。 収益から、支払いはプロバイダー会社に行われます。
  5. さらに、チームは計画に従ってプロジェクトの開発に取り組みます。ネットワークを拡大し、ネットワークを使用することに対する報酬を収集して配布します。


まだクレジットトークンがあります-ローン額からの利子収入と引き換えに、短期のネットワークローンと見なすことができます。 ICOプロセスでそれらを個別に使用するケースはまだありませんが 、前述のトークンのいずれかと一緒に使用されます。 クレジットトークン（ Steem Dollar （SD））を使用した最初のネットワークの1つはSteemitネットワークでした。



興味深い：一部の州では、政府の暗号通貨の発行の可能性を真剣に検討していますが、オーストラリアでも実装が開始されています。



DAO組織



上で書いたように、多くの場合、共有トークンはDAO組織の共有です。 これらの組織は、新しいビジネスモデルを表しています。ブロックチェーン上にデジタル形式で存在する法人との類似性、法定文書の代わりにコード、集中管理の欠如、投票を通じて株式トークンの所有者によって決定が行われるという点です。 このモデルは、中央管理会社なしで、すべての参加者の共通の貢献と責任を意味します。 このビジネスモデルは、インターネットと暗号通貨の組み合わせで機能します。



ビットコインとイーサリアムはこの分散モデルを初めて使用し、通貨/トランザクションのネットワークをロードするためにそれを使用しました。 現在、同じモデルが他のネットワーク（ Steem / Augur / Waves / Wings / Antshares / Golem /など）のダウンロードに使用されています。



すべての流行語を破棄すると、DAOは、人間の関与なしに組織のタスクを実行する一連の自己機能アプリケーションで構成されます。



作成方法 ：一部のプロジェクトは独自のブロックチェーンを作成しました（例-DAO Steem）。 その他は、デジタル通貨プロトコルの1つで作成されました（たとえば、ゴーレム、イーサリアムのオーガス）。



EthereumでDAO組織を作成する方法に関するチュートリアルを共有しています： https : //www.ethereum.org/dao#the-shareholder-association

ICOとクラウドセールの法的地位



ICOは、プロジェクトへの投資を誘致するための合法的かつ違法な方法とは言えません。 この方法で資金を調達する企業の法的地位、手順、要件は、現在、世界のどの国でも定義されていません。 彼らの行為についての非公式なルールはありません。それは、世界的に認められたリソースにあり、暗号コミュニティのほとんどが同意しています。



実際、暗号通貨の活動は経済の非公式部門で行われ、暗号通貨自体のステータスは法的な空白（いわゆる「グレーゾーン」）にあります。 さらに、ICOまたはクラウドセール中に生じる関係の法的性質を決定することは困難です。これは、それらを古典的な資金調達関係と呼ぶことは難しいからです。 明らかに、法律が技術の発展に追いついていない場合はまさにそうです。



現在、SECは、ブロックチェーンを転送エージェントとして規制することを可能にする法律の明確化を数多く進めています。

事例1



エーテルの事前販売（イーサリアムプラットフォームの暗号通貨）は、スイスで登録された非営利団体であるイーサリアム基金によって行われました。 基金の唯一の宣言された目標は、エーテルの売却による収益を管理し、イーサリアムのエコシステムを開発することです。 米国市民がSECの許可なしにエーテルを購入できるようにするため、基金は、イーサリアムプラットフォームで開発されたアプリケーションの運用に必要な「暗号燃料」の販売として、エーテルの予備販売（ETH）を設計しました。 このコードは、スイスに本拠を置くEthereum Switzerland GmbHによって開発されました。



事例2



もう1つの実験的なアプローチは、ブロックチェーンベースのエンターテイメントスタジオであるSingular-DTVです。 CODE-略語、2つの要素の組み合わせ：



CO（Central Organized）-Swiss GmbHの形式の管理コンポーネント

DE（分散エンティティ）-イーサリアムネットワークに基づく分散エコシステム。

GmbHは、DEを介したトークンの予備販売中に収集されたエーテルの消費について責任を負います。 このプロジェクトの目的は、メディアプロジェクトを作成して収益を集めることです。 CODEモデルは規制法および税法に準拠しており、トークン保有者を法的責任から保護すると想定されています。



事例3



トークンの発行とネットワークのサポートは、システムの作成者とは無関係に行われます。 トークンは、収益を受け取る公開鍵がないコンピューターアルゴリズムを介して発行されます。 システムの作成者は、完成したネットワークの最初のマイナーであるため、マイニングからトークンのシェアを受け取ります。 このアプローチは、Steemitネットワークとその作成者であるDelaware C Corpによって採用されています。
