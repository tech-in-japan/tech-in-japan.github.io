3つの小さな便利なユーティリティ

かなり頻繁に、Habrahabr（ ここなど） でも 、ブラウザに依存する何かをプログラミングするときにCtrl + S 、 Alt + Tab 、 F5の順に押すと疲労の不満があります 。 クリップボードからのテキストの複数回の貼り付けという 便利な問題も同様に重要です。 まあ、初心者向け-ボタンをクリックするだけで簡単に計算できます。

私とあなたによってテストされた簡単なソリューションのセットを提供します。



このソリューションは、3つの小さな、しかし驚くほどほとんど知られていない無料のユーティリティで構成されています。 （ 注意！これらのユーティリティは、私が知る限り、MS Windowsのみに存在します。コメントで他のOSの類似物を指定すると、他のユーザーに感謝すると思います。 ）



会う：

Softvoile Rubilnik -Ctrl + S→Alt + Tab→F5を押すことを自動化するための小さなユーティリティ





行動の原理はどこでも簡単です。 エディタプロセス、ブラウザプロセス（およびプログラムはこれを自動的に試行し、頻繁に推測します）、およびホットキーを選択します。 ここで、ホットキーを使用して、最初にエディターに切り替えます。 アクティブなエディターがある場合は、Ctrl + Sが押され、プログラムはブラウザーに切り替わり、Ctrl + F5が押されます。 1つのキーを 押すだけで、 3つのアクションを一度に実行できます。 また、特に実行中のプログラムが多数ある場合、1回のキーストロークでサードパーティのプログラムからエディターに戻ることも非常に便利です。

Ctrl + Sとチェック後の一時停止時間を設定するオプションもあります。Ctrl+ SとCtrl + F5を送信するか、プロセスを切り替えるだけです。 数回、これらの追加設定を使用する必要がありました。

原則として、プログラムは複雑ではありません。 アナログは、スクリプトまたはマクロを使用して（たとえば、AutoHotkeyなどを使用して）ひざの上で作成できますが、より機能的です。 しかし、私はこのプログラムがシンプルで邪魔にならないのが好きでした。



ArsClipは便利なクリップボードユーティリティです。





そのようなプログラムはたくさんあります（ GNU / Linuxでは、コメントが示唆するようにそれらを知りません （ 1、2 ）：Parcellite、Diodon、Klipper ）、私にとって重要なことは3つありました。 多くのプログラムでは、ホットキーを選択できます。 同様のプログラムの場合、私にとって重要な関数の実装はあまり便利ではありませんでした。 もちろん、それ以降は新しいバージョンがリリースされていますが、...アーキテクチャ、使用の考え方、プログラムが変更されることはほとんどありません。 （ アナログに興味のある方は、Clipdiary、Clipx、CLCLなど、他にもあります。 ）

かつて、Punto Switcherを使用したのはこれらの機能だけでしたが、何らかの理由でクリップボードの操作が不安定でした（トートロジーについて謝罪します）。 キーボードニンジャは、必要な設定でシステムをハングさせましたが、Orfo Switcherにはメモリリークが発生しました。 さらに、これらのプログラムの主な機能は、私にとって本当に必要ではありませんでした（実際、必要ありません）。



小規模ではあるが、プログラムのレビューではあるが、 NumLock Calculatorのような有用なものに言及せざるを得ない



画面に式を入力するための狭いストリップと細い境界線しかない場合、特に最小限の方法で、学生時代に不可欠なアシスタントであり、迅速な計算のために（だけでなく）。

普通の計算機を使って計算機に座っている人を見たことがありますか？ 私はよくです。 しかし、これを回避できるのは、そのようなプログラムの使用です。

残念ながら、NumLock Calculatorは開発を停止しました（ただし、ダウンロードすることはできます）。彼には多くの直接的なフォロワーがいません。 （従来の電卓プログラムと混同しないでください。） Calcuteは、迅速な計算に便利な小さなプログラムでもあります。 （ コメントから判断すると、Windows PowerShellは非常に人気があります。 ）

Mathad、MatLab、Mathematics、SpaceTime、RedCrabなどのモンスターや迅速な計算のためにプログラムを混同する必要がないことに注意してください。NumLockCalculatorとCalcuteは、高速で比較的小さくても頻繁に計算するために設計されています。

（ GNU / Linuxの場合：bcプログラム：extensions.bc作成者：Steffen Brinkmann。 コメントには組み込みのKRunnerも記載されています ）



私は3年以上前、かなり前にプログラムを選択しましたが、この分野では、私の知る限り、ほとんど変わっていません。これらのユーティリティの潜在的なユーザーを疎外できるのは英語のインターフェイスだけです。
