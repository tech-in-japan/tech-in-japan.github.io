July Hassle-VDS、SEO、One Startup

ケースのかゆみを安心させ、firstvds.ruでVDSを購入しました。

奇妙なことに、マトゥユットはわずか3日間です。

それが理由です=）



実際、1つの小さなクライアントサイトをホストする必要がありました。 そして、ホスティングのサポートを提供する同志（他のWebサイトを登録する）をいじらず、単純に自分自身の小さな一定のニーズ（誰かをホストする方法、perlスクリプトを実行する、実験するなど）のためにVDSを購入することを決めました。



登録し、VDSアクセラレーションプラン（400 MHzおよび96 RAM）を選択しました。WordPressサイトは1つだけで十分であり、残りは静的または小さなスクリプトで十分だと考えました。



彼は支払いさえした-半年間1,500ルーブル。 移転されたサイト。 そして、サイトが走り回って新しいサーバーで作業を始めてからわずか1時間で、VDS分離の計画を変更し、600 MHzと160 RAMを月に600ルーブルで提供しました。



トップを見ると、600 MHzでもWordPressに対応できず、spat、nginxの設定に行ったことがわかりました。これはパネルからすぐにインストールされているためです。 インストールし、Apacheを設定し、nginxを設定し、spatし、eAcceleratorを設定し、設定します-よりスマートに-しかしとにかく-ワードプレスは病気の象のように回転します-特に人々がそれに従う場合。 はい、ロードに時間がかかります。



よくやったのは、メモリを購入するか（鼻が離れているので本当にしたくない）、WordPressをKamradのホスティングにドラッグして戻すことだけです。 もちろん、ワードプレス自体を最適化するオプションが長い間あります。 まあ、または通常のキャッシュプラグインを作成するために、見通した人は誰も動作したくありませんでした。



とりわけ、ブログのURLを-馬鹿げたアーカイブ/ idの記事から読みやすいものに変更することは長い間可能です-/ eto_nazvanie_posta



次に、新しいYandexの更新を待ちます-リダイレクトを調整しましたが、まだ妨害があり、金庫が2か月間私を運転しないことは確かです。



しかし、ようやくSVN + Tracを構成し、小さなロボスタートアップの作成を開始する強みを見つけました。その主な目的は、ユニークなRSSフィードを作成し、これらのフィードをマビルやその他の小さなデバイスから読み取るための通常のサポートです。



誰かがそれを必要とするなら-私は記事を書くことができます-何か-「Apache2.2 + PHP5 + eAccelerator + Nginx + SVN + Tracを上げてクラッシュしないようにする方法」 私はそれがプロになることを疑います-私はそれを初めて入れました-すべてのホスティング業者が結局やった前に=）
