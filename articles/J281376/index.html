Citrixが仮想化市場で「新しいRed Hat」にならないのはなぜですか？ パート2

以前の投稿で、Citrixが仮想化市場で「新しいRed Hat」にならなかった理由について意見を述べると約束した。 私は約束を守ります-実際、この記事は、 以前に公開した翻訳に関するすばらしい解説になるでしょう。 そして、はい、あなたが私の議論に同意しない場合- 投稿後にアンケートで報告するか、コメントを書くか、私たちは議論します。 ;-)







ピーター・レバインはもちろん、IT企業の管理において非常に大規模で優れた経験があり、もちろん、彼は私よりもオープンソースでのビジネスについてよく知っているという事実から始めたいと思います。 しかしその一方で、おそらく「鐘楼から」の問題を見ることができます。 ちなみに、私がここで書いているすべてが本質的にピーター・レヴィンの言葉を確認しているように思えるなら、そうです。 ;-)





オープンソースのビジネスモデルについて。

Peterの重要な考え方は、オープンソースソフトウェアのテクニカルサポートを販売する「基本的なビジネスモデル」では、お金を稼ぐことができないということです。 確かに、オープンソースソフトウェアでお金を稼ぐ方法について読んだ場合、技術サポートのためのお金を得ることが主なアイデアになります。 （より広い意味で-「コード自体以外からお金を受け取る」ということで、少し後でこの論文に戻ります）。



しかし、「サポートモデル」が本当にオープンソース企業に固有のものかどうかを考えてみましょう。 いや エンタープライズ市場で営業しているIT企業には、「高度な技術サポート」などが必要です。 例を挙げないために、ここに1週間前のニュースがあります（後で必要になります）-NTTデータは、 ITサービス （Dell Services）を提供するDellの部門の購入を発表しました 。



Citrixは、エンタープライズソフトウェア市場の他のプレーヤーと同様に、 さまざまなITサービスを提供しています 。その中には、 XenServerテクニカルサポートがあります 。 オープンソースソフトウェア。



したがって、プロプライエタリなソフトウェア開発者は、ソフトウェアライセンスとそれに追加のサービスを販売するビジネスモデルを使用できるだけでなく、オープンソースソフトウェアに関連するサービスで収益を上げることもできます。 はい、あなたは正しく理解しました- プロプライエタリなソフトウェア開発者は、会社のOpenSourceと同様にテクニカルサポートやその他のサービスも販売していますが、ソフトウェアライセンスの販売からの主な収入に対する追加利益として。



Peter Levineは、Microsoft、Oracle、VMwareの収益はRed Hatの収益の何倍も高いと書いていますが、Red Hatサービスの販売から受け取った金額を、プロプライエタリソフトウェアの開発者が受け取ったすべてのお金（サービスだけでなく）と比較するのは正しいですか？ もちろん違います。



Red Hatはどれほど成功していますか？

しかし、Red HatとMicrosoft、Oracle、VMwareとの比較が正しくない場合、何と比較するのでしょうか？ たぶん、ITサービスのみを提供している会社では？ ITサービスのみを提供する大規模な国際企業を知っていますか？ 職業に最も近いのは、コンサルティングとシステム統合に従事している企業でしょう。 しかし、Red Hatに匹敵する「幅」はありません。



タスクを少し簡素化します。 大規模なIT企業の「サービス」部門を取り上げます。 もちろん、そのような比較は完全には正しくありません。なぜなら、 ソフトウェアおよびハードウェアのメーカーからサービスを販売することは、（メインではなく）単なる追加の方向であり、「トレーラ」によってメイン製品に販売されます。 また、ITサービスプロバイダーである場合、購入者はITサービス自体に価値を見出します。



上記で書いたDellサービスに関する非常に有用なニュース。 NTTデータはこのデルのユニットを30億5,000万ドルで購入し、2009年にデルが39億ドルで吸収した会社Perot Systemsに基づいて作成されました 。
州委員会が提示した文書を信じる場合、昨年のデルのサービス事業収益は5％減少し、Dellサービスのすべての分野で減少が記録されました。
したがって、Dellサービスのコストは数年間でほぼ4分の1（!!!）減少し、収益性は低下しました。 しかし、Perot Systemsは、米国のコンサルティングおよびITサービス市場のリーダーと見なされていました。 同意すると、デルは「鶏の産卵金の卵」を売らないだろう...？



Peter Levineは「Red Hatの成功はそれほど印象的ではない」と書いていますが、Dellサービスと比較すると、 資本金が135億ドル、売上高が20億ドルにのぼる Red Hatはそれほど悪くありません。



オープンソース企業が成功するかどうかは、ソフトウェアメーカーと比較するか、サービスを提供する企業と比較するかによって決まります。



Red Hatはソフトウェアメーカーではありません

他の「本物の」オープンソース企業（製品のすべてのソースコードを公開している）と同様に、 Red Hatはソフトウェアメーカーではありません 。 驚きましたか？ 私たちは、ビジネスモデルの有効性、つまり「コード自体以外からお金を受け取る」というフレーズに戻ります。



外部からオープンソース企業を見ると、すべてが論理的ではないように見えます。プロとしてリンゴの栽培に従事していると想像してください。 そして、消費者にとって価値があるのはあなたのりんごですが、あなたは無料で皆にそれらを与えます、そして、あなたはサービスと関連サービスでお金を稼ごうとしています。 たとえば、リンゴを提示された人の家にリンゴを輸送する場合。



そのようなモデルがソフトウェアライセンスの販売よりも収益性が低いことは驚くことではありません。



Red Hatの場合（XenSourceの場合よりも）すべてが少し異なります。元々は既存のオープンソースソフトウェアをパッケージ化し、同梱していました。 上記の例えに移り、「彼らは野生のリンゴの木からリンゴを集め、いくつかの木の世話をします。」 さまざまな推定によると、Red Hat製品に含まれる「独自のコード行」の割合は20％を超えていません。 しかし、このアプローチでも、 Red Hatはオープンソースビジネスの複雑さについて繰り返し語っています 。
これらすべてから、ホワイトハーストが数回滞在した別の非常に重要な結果があります。 これらの言葉で、彼はスピーチを始めました：「フリーソフトウェアを売ることは難しいです。」 同氏は、オープンソース企業にとっては、同種のプロプライエタリ企業よりも一定の収入を達成するのは10倍もはるかに難しいと述べた。
そしてまた：
Red Hatはいつ50億ドルに達すると計画しているのですか？ 彼は私に良い答えをくれました。 彼は、Red Hatが遅かれ早かれ50億ドルに達すると本当に信じていたが、それは他のコンピューター会社の「500億ドルの収益を置き換える」必要があると言った。




ところで、 $ 20億のRed Hatの収益について 、2011年にRed Hatでそれを書きました
次の5年間で売り上げを3倍にし、年間最大30億ドルを稼ぐという新しい目標を設定します。
ご覧のとおり、5年間で売上を3倍にすることはできませんでした。計画された結果の10億ドル（または33％）の「不足」は非常に大きな金額です。 比較のために、Citrixは2011年に当時 20億ドルの収益を上げていました 。 （2015年のCitrixの財務結果に関心がある人は、 こちらで見つけることができます。）



OpenSourceの開発のためのリソースはどこで入手できますか？

そしてここで、Peter Levineがメインの「弱いリンク」と呼んでいるものに到達します。
Red Hatモデルが機能しない理由は数多くありますが、このビジネスモデルの主な「弱点」は、長期的な開発に十分な投資ができないことです。


彼の記事では、この脆弱なリンクを取り除くいくつかの方法を提供していますが、それらはすべて、何らかの形で企業が「本物の」オープンソースから離れ、テクノロジーの一部を閉鎖せざるを得ないという事実につながります。 実際、プロプライエタリな製品の生産とライセンスの販売について話している。



正直なところ、私はこの考えがまったく好きではありません。 私個人の意見では、解決策は、オープンソース企業がプロプライエタリなソフトウェアのメーカーにならないようにすることです。プロプライエタリなソフトウェアのメーカーは、製品とリソースでオープンプラットフォームとフリーソフトウェアをサポートします。



だからこそ、Peter Levineが書いているプラ​​ットフォームのアイデアが本当に好きで、 Citrix Tech Exchangeで聞いたことがあります 。 Citrixはオープンソースソフトウェアと競合しませんが、その開発に投資し（Xen、OpenDaylight、Open vSwitchの場合）、 OpenStackおよびApache CloudStackの開発者と協力します。



では、なぜCitrixは仮想化市場で「新しいRed Hat」にならないのでしょうか？

この質問に対する簡単な答えは次のように聞こえます。その場合、Citrixは利益の一部を放棄し（他の専有会社に道を譲る）、ソフトウェア開発会社からITサービスを提供する会社に再割り当てする必要があるためです。 これは、ビジネスを完全に再設計して、収益性の低い、スケーラブルなビジネスモデルに変える必要があります。



ピーターは、R​​ed Hatが「新しいAmazon」（ Amazon Webサービスを意味する）になることを考える必要があると書いていますが、私はそれに同意しません。 Red HatはRed Hatのままであり、CitrixはCitrixのままである必要があると確信しています。 結局のところ、オープンソースコミュニティにはRed Hatだけでなく、Citrixも必要です。オープンソースソフトウェアと競合せず、オープンプラットフォームにサポートを提供し、オープンソースの開発に資金を提供するプロプライエタリ製品のメーカーです。
