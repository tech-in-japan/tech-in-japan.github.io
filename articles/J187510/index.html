データセンターはクラウドへ



開発の特定の段階で、すべての大企業は、独自のITインフラストラクチャを近代化するという課題に直面しています。 既存のインフラストラクチャの生産性がすでに十分でない場合、大企業のリーダーの多くは、独自の新しいデータセンターの構築を考えています。 データセンターに適した建物の検索、要員の選択、設計、データセンターの基本的要素と追加要素のインストール-これらすべてに多くの時間と材料リソースが必要です。 しかし、代替手段があります！



クラウドテクノロジーと仮想化の積極的な開発のおかげで、高度なスケーラビリティと信頼性を維持しながら、ITインフラストラクチャをアウトソーシングに完全に移行できるCloud Data Centerサービスを提供するプロバイダーがロシアに既に登場しています。 このようなサービスは安価ではありませんが、バランスの取れたアプローチにより、会社のリソースをより合理的に使用することができます。



クラウドデータセンターは本当に優れていますか、どのように宣伝されていますか？ 請求はお金の節約に役立ち、サーバー仮想化は高品質のクラウドインフラストラクチャを構築するのに十分なのは本当ですか？ これらに対する答えと、habrakatの下でのその他の多くの質問をしようとします。





誰のためですか？



サービスとしてのクラウドデータセンターは、主にソフトウェア開発に関係のない活動を行う大企業を対象としています。 中小規模の企業や、複雑でないタスクを解決するためにクラウドに「移動」したい個人にとっては、 Amazon Web Servicesに精通することは役立ちますが、大規模なプレーヤーにとっては、開発の特定の段階でのAmazonの能力は明らかに十分ではありません。



Amazon Web Servicesの機能は、Habréですでに説明されています。 大企業向けのロシアのクラウドプロバイダーの利点をいくつか紹介します（CROC Virtual Data Centerの例を使用）。

•Amazonテクニカルサポートは英語のみ。

•Amazonにはユーザー機器をクラウドに接続する機能はありませんが、ロシアのプロバイダーにはこのオプションがあります。

•Amazonはサービスでネットワーク仮想化を使用しません。CROC は OpenFlowプロトコルに基づくSDNソリューションを使用します。

•Amazon Elastic Compute Cloud（EC2）の標準構成では、情報は仮想化サーバーに直接配置されたインスタンスストレージドライブに保存され、セキュリティに影響します（サーバーがオンの場合、すべての有用な情報が飛びます）。 ただし、Amazon Elastic Block Storage（EBS）サービスを使用すると、隔離されたストレージサーバーにインストールされたハードドライブに情報を保存できるため、データセキュリティが大幅に向上します。 ただし、Amazonユーザーには、高信頼性のストレージエリアネットワーク（SAN）を使用する機能がありません（このオプションが提供される深刻なクラウドプロバイダーとは異なります）。



利益はいくらですか？





それでは、独自に構築するのではなく、クラウドデータセンターを選択することで何が得られますか？



まず、自分のニーズに合わせてデータセンターを構築するのにかかるコストについて話しましょう。 CROCの従業員は、モスクワのエンスージアストストリートにあるデータセンターの建設について詳細に説明しました（ 「データセンターの構築方法」 、 「フォトツアー」 、 「データセンターインフラストラクチャ」 ）。このデータセンターは一例です。



建物の選択と購入、建物の設計、変更、仕上げ、電力の供給、冷却システム、ポンプ場、エアコン、セキュリティシステム、消火システムの設置、サーバールームの設置は、建設段階でのみ行う必要があるものの完全なリストではありません。 建設自体のコストに加えて、システム管理者、ネットワーク管理者、仮想化グループ、会計士、調達スタッフ、およびデータセンターにサービスを提供する他の多くの労働者の作業に対して支払う必要があるという事実も考慮する必要があります。 さらに、ラックとサーバーを購入し、仮想化製品のライセンスを購入する必要があります。これらはすぐに思い浮かぶ費用です。



ご覧のとおり、設備投資が必要です。 もう1つの要因は、データセンターの装備を開始すると、ビジネスが将来どの程度の速さで発展するかについての概算しか得られないことです。 大量の機器を敷設することにより、インフラストラクチャに莫大なお金を費やすことで多くを逃すことができます。 ラックとサーバーのコストが高すぎると、サービスのサポートとメンテナンス、冷却システム、発電機、さらには非常に高価な機器に余分なコストがかかります。



サービスとしてクラウドデータセンターに滞在することで、データセンターの設計と建設に関連する資本コストを即座に放棄できます。 プロバイダーの費用は当然「クラウドデータセンター」サービスの料金に含まれますが、クラウドを選択した場合、費用は瞬時ではなく、数年間「分配」されます。 信頼できるクラウドプロバイダーを選択することにより、ITスペシャリストの経験豊富なチームを自動的に受け取り、幅広い複雑なタスクに対応できます。



仮想データセンターの初期構成を選択する際のエラーのコストは、物理データセンターの場合よりもはるかに低くなります。 秒単位または時間単位の請求により、使用するリソースの構成を調整することにより、クラウドをビジネス開発のペースおよびITニーズに同期させることができます。

サービスと仮想データセンター管理のリアルタイム自動化は、クラウドバージョンと物理データセンターを区別するもう1つの機能です。 適切なクラウドプロバイダーを選択すると、セルフサービスポータルを使用して、サーバーの作成、容量の増減、ディスクの追加、バックアップの作成、ファイアウォールの構成などをリモートで実行できます。 コントロールパネルの対応するスライダーをマウスで動かすことにより、使用するRAMの量を追加または削減し、オフィスを数回クリックするだけで、必要な値にプロセッサの電力を投入できます。 クラウドデータセンターの請求はすぐに変更に対応し、仮想システム構成に加えられた変更に応じて、1秒または1時間以内に（料金プロバイダーに応じて）サービスの料金を支払います。

クラウドデータセンターサービスに興味がある場合にクラウドプロバイダーを選択する基準の詳細については、 「クラウドプロバイダーと呼べるのは誰なのか、またはホスティング事業者の手口に負けないのか」という 2つのhabrastatsを読むことができます。 クラウドプロバイダーの選択 。 」



注目に値するもう1つのことは、データセンターの信頼性レベルです。 自分でデータセンターを構築する場合は、設計段階で既にデータセンターの信頼性とセキュリティに注意する必要があります。 「クラウドデータセンター」サービスを選択した場合、システムのセキュリティとフォールトトレランスは完全にプロバイダーにかかっています。 Habré には 、TIA 942標準のトピックとUptime Instituteのレベルによる分類に関する詳細な記事が既にありました。主なものを引用します。

非常に無礼な言い方をすれば、TIER Iは、故障が発生した場合、データセンターの運用が停止し、システムが修復されることを意味します。 TIER IIによると、データセンターは継続的に稼働していますが、必要に応じて停止し、重要なコンポーネントの修理や故障の修復を行います。 単一の障害が発生した場合、TIER IIIは停止することなく修復されるため、多くの事柄が複製されます（複数のノードが同時に壊れる極端な場合にのみ停止します）。 TIER IVは完全な複製を意味します。実際、2つのTIER 2番目のデータセンターが同じ建物内のラックの周りに構築されています。 平方メートルあたりのコストは、それぞれ15K、20K、25K、40Kドルですが、これはアメリカ向けです。 残念ながら、税関とロシアの仕様により、わが国では2〜3倍高価です。 そして、規模の経済が存在する本当に大きなデータセンターにのみ。

仮想化について少し



サーバーの仮想化に加えて、ストレージシステムの仮想化のためのソリューションは、技術開発のこの段階ですでに利用可能であることを既に述べました。



ストレージシステムの仮想化を使用すると、多数の物理ストレージシステムを単一の論理プールに結合できます。 つまり、データセンターで使用可能なすべてのストレージシステムを単一のユニットに結合し、たとえば800 TBのディスク容量として外部から見ることができます。 この場合、1つの仮想サーバーのディスクフラグメントは、プロバイダーの異なるストレージシステムに簡単に配置できます。



Cloud Data Centerサービスの一部として使用される仮想化ソフトウェアソリューションに加えて、クラウドプロバイダーを選択する場合、プロバイダーの兵器庫で利用可能なハードウェアにも関心があるでしょう。 ユニバーサルDell PowerEdgeサーバーに加えて、信頼性の高いサーバーラックに加えて、プロバイダーの選択を支持する議論は、すでにブログで書いた Dell EqualLogicディスクアレイと、新しいDell Compellent仮想ストレージシステムを使用することです。 Dell EqualLogicおよびCompellentのラインナップから、最新のストレージ仮想化ソリューションをすべてサポートし、非常に高いレベルでデータセキュリティを確保できるようにします。



そして最終的には...



•複数の大手クラウドプロバイダーが提供する仮想データセンターサービスは、収入が自社ソフトウェアの開発に直接関係しない大企業に適している場合があります。

•このソリューションを使用すると、資本コストを大幅に削減し、ITインフラストラクチャとそのメンテナンスのコストに関連するリスクを最小限に抑えることができます。

•クラウドプロバイダーとしてIT市場の主要なプレーヤーを選択した場合、非標準の機器を使用し、特定の仮想化機能を使用して会社固有の特定の問題を解決することもできます。

•大規模なクラウドプロバイダーと連携して得られるもう1つの便利なボーナスは、クラウドへの移行時のプロバイダーの従業員のオンライン支援と、いわゆる「ハイブリッド運用モード」のサポートです。 このモードでは、クラウドと物理ITインフラストラクチャを同時に操作し、それらを1つのネットワークに結合する機会があります。
