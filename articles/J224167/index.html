突然死症候群（別名突然死症候群、別名SDS）Samsung Galaxy Note 2およびSamsung Galaxy S3

みなさんこんにちは！

素晴らしいスマートフォンSamsung Galaxy Note 2とSamsung Galaxy S3のすべての所有者に、突然死の症候群について伝えたいと思います。







昨日、理由もなく、私のSamsung Galaxy Note 2が突然オフになりました。

電話をオンにし、グーグルで電話が死んでいることがわかりました。



突然死症候群（別名突然死症候群、別名SDS）による死亡は、次のデバイスの影響を受けます。



a）16GBのメモリを搭載している

b）eMMCチップVTU00Mを持っている

c）0xF1チップのファームウェアバージョンを持っている

d）OS Android 4.1.1をインストールしている。

お使いのデバイスがこのリスクにさらされているかどうかを確認するには、このアプリケーションをダウンロードしてください。

ルート権限が必要になりますが、テストには必要です。 必要なものはすべて、root権限がないように見えます。



スマートフォンにSDSの原因となるバグのあるチップがある場合、画面は次のようになります。







数週間前、友人のSamsung Galaxy S3のスマートフォンでも同様のことが起こりましたが、カスタムファームウェアやルート権限さえもインストールしていませんでした。 そして、ICSバージョンからJelly Beanにアップグレードして、Samsungの公式ファームウェアを常に使用していました。 サムスンは問題があることを公式に認識していませんが、サービスセンターに連絡した後、私たちはそのような問題に最初に対処したのではないことを目撃し、最近サムスンコールセンターへのそのような呼び出しが多くありました。



理由はありませんが、Galaxy S3とNote 2でボードが死んでしまいます。

サムスンはそれらを無料で変更する必要がありますが、オフィシャルサービスセンターは保証対象外の修理について言及しており、約8000rのボード交換を希望しています。



友人がマザーボードを交換したのはマザーボード上で、これも古いタイプのものでした。そのため、スマートフォンは数か月、またはそれ以前に再び死ぬ可能性があります。



工場の欠陥は明らかですが、サムスンのサービスセンターはデバイスの無料修理を拒否しています 。



注意して、SDSへの暴露についてペットをテストしてください。



同じ問題が発生し、ドナーの電話から自分でボードを再配置することにした場合、詳細な手順はすでにユーザーによって書かれています

投稿のvvzvlad 「 Galaxy Note IIを開く-ハードウェアについて 」
