会社の在庫：GLPIを更新する



そのため、当社ではGLPIバージョン0.72.4を長い間使用しています。 週末になって、私は最新バージョンにアップグレードすることに決め、多くの困難に遭遇しました。 すべての矛盾を解決した後、私は正しい行動方針を文書化し、一目では見えない多くの異なる点に注意を払うことにしました。



いくつかのニュアンス：

1.新しいバージョンをゼロからインストールし、バックアップを展開すると、巻き上げられたすべてのコンポーネントが消えます。 したがって、新規インストールではなく、更新を行う必要があります。

2.古いバージョンのコンポーネントでは、一部のフィールドを文字で開始することができました。たとえば、HDDサイズは2Gbです。 新しいものでは数字しか使用できないため、事前にすべてのコンポーネントで省略形を数値形式のみに変換する必要があり、Mbのみで別の略語を選択することはできません。

更新順序：



1.ファイル/フォルダーを保存します（添付ファイルはその中に保存されます。保存できるのは拡張子付きのフォルダーのみで、アンダースコアで始まるすべてのフォルダーは不要です）

2. / var / www /から古いglpiフォルダーを削除します

3.代わりに、新しいバージョンのglpiフォルダーを解凍します

4.添付ファイル付きのファイル/ *フォルダーを追加します

5. chown www-dataフォルダーにアクセス権を設定することを忘れないでください：www-data config files files / * inc

6.アドレスhttp：// server / glpiに移動します



7.言語を選択してください





8.ライセンス契約に同意します





9.更新を選択します





10.環境がチェックされます





11. MySQLへの接続のデータを入力します





12.データベースを選択してください





13.選択を確認します





14.承認ページに移動して、作業を確認します。



15. PDFへのエクスポート用にファイルを変更する操作を実行することを忘れないでください。
