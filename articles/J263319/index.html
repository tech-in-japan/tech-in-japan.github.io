タブの上のストリップをオフにし、Vivaldi 1.0.231.3の新しいアセンブリをタブバーにドラッグします。

みなさんこんにちは！



休日にもかかわらず、短縮された形式ではありますが、ブラウザでの作業を続けています。 Vivaldi 1.0.231.3の今日のビルドは、いくつかの新製品と修正されたバグのリストであなたを喜ばせます。



タブバーの機能強化



タブバーの空き領域を使用してブラウザウィンドウを移動できないことについて、ユーザーから多くの苦情が寄せられました。 まあ、私たちはあなたの希望にいつでも満足しています-今、私たちはそのような機会を追加しました。 ただし、同時に、別の一般的な機能を放棄する必要がありました。タブバーをマウスの中央ボタンでクリックして新しいタブを作成します。 しかし、これは一時的な不便です。 この便利な関数を返す方法を間違いなく思いつきます。



別の一般的な要求は、タブの上の狭いストリップをオフにすることでした。 タブバー全体が開いているページで占められている場合でも、ブラウザウィンドウを移動できるように設計されています。 以前のビルドでは、タブの上にこのストリップを無効にする機能を追加しましたが、これはタブのグループ化が拒否されたために発生しました。 本日、新しいソリューションをお知らせいたします。これらの設定を分割しました。上部のバーをオフにできるようになりましたが、タブをグループ化するオプションは引き続きあります。









アドレスバー



新しいアセンブリには、長く要望されていた別の機能も導入されています。 アドレスバーのドロップダウンリストにWebサイトアイコンの表示を追加しました。









しかし、開発プロセスは徐々に進行しており、すべての「ささいなこと」をすぐに修正できるわけではないため、アイコンは以前にアクセスしたサイトにのみ表示されることに注意する必要があります。 同じことがデフォルトのブックマークにも当てはまります。



空間ナビゲーションとその他のささいなこと



また、空間ナビゲーションの改善、Windows用インストーラーの作業の改善、キーボードショートカットの操作におけるいくつかのエラーの修正にも引き続き取り組みました。 記事の最後に変更点の完全なリストがあります。



以下のリンクを使用して、新しいアセンブリをダウンロードできます。



  • Windows 32ビット
  • Windows 64ビット （実験的）
  • Mac
  • Linux RPM 32ビット
  • Linux RPM 64ビット
  • Linux DEB 32ビット
  • Linux DEB 64ビット
