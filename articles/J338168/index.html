Windowsサーバーファイルの日曜大工バックアップ

ここでは、UNIX方式の最良の伝統で、最小限の数の外部モジュールを使用して、差分バックアップシステムを（ほとんど）すぐに使用できるようにする方法について説明します。

7za.exe \ 7zとUNIXに似たユーティリティpdate.exeを使用するため、時間が経つにつれて、ランプ* NIXのように作業するのが便利になり、bashの置き換えは「一般的な」BATになります。 背景と詳細-カットの下。



背景



すべては、作業コンピューターで個人のメールopen.exeによって開かれたファイルを開く、不注意で大胆不敵なユーザーに関する話から始まりました。 残念ながら、これは平凡なマルウェアではなく、完全に普通のエンコーダーであることが判明しました。 しかし、もちろん復元できないユーザーファイルに加えて、誰もがアクセスできる小さな、しかし重要なボールが1つヒットしました。



この暗号化されたわいせつを見て、私は毎日（これだけでなく）Windows Server 2003 SP2 x64の組み込みツールを使用してこの（これだけでなく）ボールのバックアップがあることを心から思い出しました。 しかし、このバックアップを調べてみたところ、Windows自体を使用したバックアップに関しては、すべてがバラ色ではないことに気付きました。 まず、完全バックアップが利用できないことが判明しました。つまり、コールドデータ（非常にまれにしか変更されないファイル）の回復は成功しそうにないことを意味します。 第二に、作成された増分バックアップからの復元は重要なタスクであることが判明しました。各ステップで、変更されたデータのみを復元でき、それ以上は復元できませんでした。 （完全バックアップが失われたため）少なくともすべての変更されたデータを復元するには、すべてのバックアップを順番に並べる必要があります。この場合、増分バックアップから予想したものとはまったく異なります。



皆さんの中には、バックアップのパフォーマンスを確認する必要があったと言う人もいるかもしれませんが、そうです。 しかし、業界で働いているあなた方は、管理者の時間がどこに行くことができるかを理解することができます-はい、彼らは最も多くのオンラインチケットオフィスです。



一生懸命考えて、Maxim Chirkov www.opennet.ru/dev/fsbackupによるfsbackupインクリメンタルコピーシステムを初めて知ったのを思い出しました-柔軟性、シンプルさ、多くの機能、オープンアーカイブストレージフォーマット（tar）。 システムが* NIX / Linuxの下で開発されたことは残念です。 GoogleはWindowsでの同様のシステムに関する私の質問にも答えませんでした。 私が見つけた最も有用なものは、 Habrovchanin antip0dの 短いガイドとバックアップ用のサンプルスクリプトです。 これは、スクリプトに使用した最後のリンクの素材でした。

まず、最新の安定バージョンをダウンロードします。 執筆時点では、これは16.04です。 バックアップを7zアーカイブに保存します。マルチスレッド、暗号化/マルチボリュームアーカイブのサポート、7zからの抽出速度はパッキング速度の10〜20倍です！

作業ロジック



変数の処理後、スクリプトはブロック：Mainを調べます。ここで、バックアップ操作ロジックが示されます-その場合、新しいバックアップを実行する必要があり、その場合-既存のベースアーカイブを更新します。 デフォルトでは、月の初めに新しいアーカイブが作成され、バックアップディレクトリのすべてのファイルが\ backup \ oldに移動されます。または、ベースアーカイブが存在しない場合。

すでにこの記事を書いている時点で、ベースアーカイブを更新する機能を追加する必要があることに気付きました-「月に1回の完全バックアップ+それへの差分バックアップ」の簡易バージョンは、最大250 GBまでのファイルエクスチェンジャーに使用することをお勧めします。 小さなファイルが優勢な550 GBのファイルホスティングサービスでは、バックアップ速度が不十分でした（約55時間）。 公平に言えば、これはパフォーマンスの信頼できる尺度としては機能しないと言われるべきです-バックアップ中に一部のファイルが利用できなかったことが判明し（hi chkdsk）、バックアップはディスクI / Oでもビジーなリモートサーバーのセクションにスタックされました 



:Main REM   ,       \  ,    . REM   -        IF NOT EXIST %baseArch% GOTO BaseArchive REM     + t    IF %dm% EQU 1 GOTO BaseArchive ELSE GOTO UpdateArchive REM     1   REM IF %dm% EQU 1 GOTO UpdateBase ELSE GOTO UpdateArchive REM    (2, 19, 36  ) REM IF NOT %wn%.%dw% EQU 02.5 GOTO UpdateArchive REM IF NOT %wn%.%dw% EQU 19.5 GOTO UpdateArchive REM IF NOT %wn%.%dw% EQU 36.5 GOTO UpdateArchive REM   ,   REM IF %dw% EQU 6 (GOTO UpdateBase) ELSE (GOTO UpdateArchive) REM     ,       . REM      ECHO Warning! No one condition matching, check :Main block of script >> %Log% GOTO End


最初の2つの条件はデフォルトで有効になっており、月1回の完全バックアップ+差分バックアップを記述しています。 残りの条件は代替です。つまり、一方をオンにした場合、もう一方をオフにするか、作業ロジックを好みに合わせて変更する必要があります。



変数



dm、dw、wn-それぞれ月の日、曜日、および週番号（数値で）。

verboseLevel- 「おしゃべり」モード。アーカイブの書き込み先などに関する情報を提供します。 スクリプト構造に大きな変更を加えるときに役立ちます。

tmpDirは、一時ファイルを保存する場所です。 デフォルトでは、7-Zipは古いベースアーカイブファイルと同じディレクトリに新しいベースアーカイブファイルをビルドします。 このキーを定義することにより、一時ベースアーカイブファイルが作成される作業ディレクトリを設定できます。 一時ベースアーカイブファイルが作成された後、元のファイルにコピーされます。 その後、一時ファイルが削除されます。



差分バックアップ



デフォルトでは、ベースアーカイブにないファイルと新しいファイルが差分バックアップに配置されます。 必要に応じて、この動作を変更できます。 キーの説明は、標準の7zヘルプファイルにあります。



このアプローチには利点があります（既に評価できています）-バックアップからの展開の速度。 完全復旧の場合、最後の（または他の）使用可能な差分バックアップを展開し、次に完全なアーカイブを展開して、既存のファイルの上書きを拒否します。



いくつかのコマンドラインオプション



-bsp2 -STDERRの進行状況を含む行を出力します。 STDOUT 7zはログにリダイレクトされます。もちろん、進捗はそこに書き込まれません。 このコマンドは、詳細については、STDERRに表示します。



-ssw-別のアプリケーションが書き込み用に開いているファイルをパックします。 このキーがインストールされていない場合、7-Zipはそのようなファイルをアーカイブに含めません。



-slpは非常に便利なオプションです。 ラージページモードでは、圧縮速度が向上します。 ただし、7-Zipはメモリ内の大きなページを配布している間、圧縮の開始時に一時停止があります。 7-Zipが大きなページに対応できない場合、通常の小さなページが配置されます。 さらに、7-Zipが大きなページを使用する場合、タスクマネージャーはプログラムによる実際のメモリ使用量を表示しません。 この機能はWindows 2003 / XP x64でのみ機能します。 システムの管理者権限も必要です。 この機能の推奨RAMサイズは3 GB以上です。 -slpモードを使用すると、7-Zipがメモリブロックを割り当てるときにWindowsシステムが数秒間フリーズする場合があります。 WindowsがRAMから7-Zipに大きなページを割り当てようとすると、Windowsはこの時点で他のタスクを中断する場合があります。 完全にシステムがフリーズしたように見えるかもしれませんが、その操作は復元され、配布が成功した場合、7-Zipはより速く動作します。 他のタスクを中断しない限り、 -slpモードを使用しないでください。 さらに、 -slpモードを使用して小さなデータセット（100 MB未満）を圧縮することは意味がありません。 ただし、大きな辞書でLZMAメソッドを使用して大きなデータセット（300 MB以上）を圧縮すると、 -slpモードで5％-10％の速度向上が得られます。

-mmt = on-マルチスレッドモードを設定します。 マルチプロセッサ/マルチコアシステムを使用している場合、このキーを使用して速度を上げることができます。 7-Zipは、LZMA / LZMA2圧縮とBZip2圧縮/解凍のみでマルチスレッドをサポートします。



-ms = off-ソリッドアーカイブの作成を無効にします。 この場合の圧縮品質はもちろん低下しますが、いくつかの非常に重要な利点があります-定期的にベースアーカイブのデータを更新して差分バックアップのサイズを削減できます。アーカイブが完了していないため、それ以上「ピンチ」する必要はありません。 非固体アーカイブは損傷に対してより耐性があり、アーカイブから抽出する時間ははるかに高速です。



シートを含める/除外する

デフォルトでは、2つのタイプのリストが定義されています-インクルードされたファイル/ディレクトリのリスト（ include_general.txt ）、および2つの除外リスト（ exclude_general.txt 、 exclude_regexp.txt ）。



包含リストはUNCパスもサポートしています。 ファイル/ディレクトリを例外に配置するには、パスが相対パスである必要があります。



たとえば、バックアップディレクトリがE：\ foo \ barで、サブフォルダーE：\ foo \ bar \ somefolderを除外する場合、exclude_general.txtにbar \ somefolderまたはbar \ somefolder \を追加する必要があります



末尾にスラッシュのないパスは、ファイルまたはディレクトリを参照できます。

Exclude_regexp.txtには、再帰的に表示されるregexpによって除外されたファイルが含まれます。 *-任意の文字のシーケンス、？ -任意のキャラクター。

7-Zipはシステムワイルドカードパーサーを使用しないため、7 Zipの「任意のファイル」は「*」であり、「*。*」は拡張子を持つファイルです。
