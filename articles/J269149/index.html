パート1. AからZまでのクロスプラットフォーム生産性のためのOfficeストア

今日、学校や大学から始めて、私たち全員が何らかの方法でソフトウェアを使用して作業の生産性を向上させています。 Microsoft Officeと呼ばれるアプリケーションパッケージを使用して、プレゼンテーションの作成、手紙の作成、ドキュメントの作成、計算の実行、オンライン会議の開催、その他の多くのタスクの解決に成功しています。



マイクロソフトは当時の要件を満たしており、WindowsやWindows Phoneなどのプラットフォームだけでなく、世界中に広く流通している他のメーカーのプラットフォームの両方でパッケージを広く利用可能にします。 現在、タブレット、ラップトップ、デスクトップコンピューター、さらにはスマートフォンでMicrosoft Officeアプリケーションとサービスを使用できます。







はい、新しいバージョンのOffice-Office 365には、Word、Excel、PowerPoint、Outlookなどの従来のアプリケーションだけでなく、SharePoint、Skypeなどのサービスも含まれているため、「アプリケーションとサービス」を作成したときに間違えられませんでした。 実装されたモビリティの概念により、作成する製品の品質を犠牲にすることなく、お気に入りのツールのオンラインバージョンを使用できます。 ただし、これはクライアント、つまりエンドユーザーからのビューです。



「そして、この問題に関してマイクロソフトは開発者に何を提供しますか？」とあなたは尋ねます。 数字を見てみましょう。 彼らは、この市場に目を向ける価値があるかどうかを明確に示すことができると思います。

だから、手始めに-市場の大きさ。







2014年-13億台のモバイルデバイス、2018年-1.9年 そして、これらのすべてのデバイスには、クロスプラットフォームのためにOfficeをインストールできます。 今日、10億人以上がオフィスアプリケーションを使用しています。 クラウドストレージに格納されている450ペタバイトを超える情報。 平均して、1人は1日2〜3時間Office Appsを使用します。



正当な疑問が生じます。「開発者として、これらの数十億すべてにアクセスできますか？」 もちろんそうです!!! 結局、これは新しいOffice 365モデルの作成者の前に設定されたタスクでした。誰もが拡張でき、アプリケーションから接続できるオープンプラットフォームを作成するためです。



このモデルの利点は何ですか？

  1. 安全性と信頼性。 アプリケーションコードは、明確なセキュリティバリアを備えたOfficeの外部で実行できます。 モバイル標準で明確に定義されたAPI。

  2. デバイスに優しい。 アプリケーションはタブレットと電話で動作します。 iPad、Android、およびWeb（Office Web Apps）用のクライアントをリリースしました。

  3. 現代の開発。 最新の開発ツールとテクノロジを使用して、Office用のアプリケーションを作成します。 OfficeおよびOffice 365用の堅牢なAPI。HTML5、JavaScript、Xamarin、Azure、Visual Studioなどの柔軟なツール。



プラットフォームに対するマイクロソフトの現在のビジョンは次のとおりです。

  • すべての種類のOffice拡張機能
  • どこからでもOffice 365サービスに接続
  • オープンプラットフォームを使用して新しいソリューションを作成する






歌詞を追加します。 今日、クライアントの視聴者のカバレッジを最大化するために、開発者は異なるプラットフォーム用に同じアプリケーションを書くことを余儀なくされています。 通常、これはWindows、iOS、Androidです。 モバイルの世界。 しかし、これらは異なるプログラミング言語、異なるIDE、異なるライブラリなどです。 開発者の作業を楽にするために、クロスプラットフォーム開発用のパッケージ（Xamarin、Apache Cordovaなど）が作成されます。 しかし、これは開発者にとっての追加の負担でもあります。新しいパッケージを研究し、変更を監視するためです。



Officeの開発はどうですか？ このパッケージはさまざまなプラットフォームのユーザーが使用するため、クロスプラットフォームです。Microsoftは、これらのプラットフォームで開発するユーザーがOffice用に開発できるようにすることに関心があります。 これには何がありますか？ これは、iOS、Android、Xamarin、Apache CordovaなどのSDKのセットです。 Officeのアプリケーションとアドインを作成する私たちは、この巨大なエコシステムの一部になり、知識を使用して、Officeアプリケーションを通じてソリューションを提供しないのはなぜですか。 言い換えれば、マイクロソフトはプラットフォームを提供し、あなたはスマートなソリューションを作成し、顧客はそれらをどこでも使用します！

ことわざにあるように： 一度書いて、どこでも実行

これはおそらく、ビジネスアプリケーション開発者の道を歩むことを決めた人に誰もが自問する質問です。

  1. Office製品のアドインを書く
  2. Officeのパワーを活用して、Office APIを介してアプリケーションを強化する

    たとえば、FedEx Package Trackerです。 たとえば、Amazonから、注文が発送されたこととFedEx追跡番号が記載されていることを通知するメールを受け取った場合、以前にOfficeストアからアドインをインストールしておくと、ブラウザーを開いてFedEx Webサイトでパッケージを探すのではなく、addを展開できます-パッケージトラッカーを使用して、パッケージのある場所とその状況（FedEx Webサイトからの情報）を直接確認します。







    別の、おそらく予想外の例は、Team Viewerです。 多くの人がコミュニケーションに使用しています。 このアドインは、ユーザーをサポートしたり、高齢の親が新しいオペレーティングシステムの複雑さに対処するのを支援したりする場合に特に役立ちます。 レターと作業からTeam Viewerセッションを開くだけです。

  2. タスクパネルアドイン -Word、Excel、Excel Online、PowerPoint、Projectのワークスペースの一部を占め、アクティブなドキュメントを操作する権限を持つパネル。







    Microsoftの翻訳者とウィキペディアは、このようなアドインの例です。 この記事の執筆時点では、指定された2つのアドインを含め、Officeストアからプレインストールしています。 これから、Wordを終了して、ブラウザでWikipediaまたはBing Translatorサイトを開くことなく、Wikipediaまたは選択したテキストの翻訳から情報を取得できます。 アドインは、Wordワークスペースの右側のウィンドウに表示され、必要なすべての情報を提供します。







    別の例は、PayPal for Excelです。 このアドインをインストールすることにより、新しいPayPal請求書を作成したり、既存の請求書を確認するためにPayPal Webサイトを開く必要がありません。 これはすべて、Excelで直接行うことができます。

  3. コンテンツアドイン -名前が示すように、コンテンツはドキュメントに含まれています。 これは、たとえば、マップ、グラフ、描画、および一般的に、Excel、Excel Online、PowerPointドキュメントの残りのコンテンツとやり取りし、ユーザーに追加機能を提供するすべてのものです。







    これは最も馴染みのあるアドインです。 探す必要のないExcelの例は、Bing MapsとPeople Graphです。 ストアからアドインをインストールし、データの一部を選択して、マップまたはグラフに反映を確認します。







おそらくお気づきのように、これらのアドインは、Word、Excel、Power PointなどのデスクトップバージョンとWebバージョンの両方で機能します。
