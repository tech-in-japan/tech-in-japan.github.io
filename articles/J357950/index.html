Djangoアプリケーション用のコンパクトサーバー

はじめに



多くの初心者のWeb開発者は、自分の作品をどこに投稿するかを考えています。 通常、これらの目的には、NIXのようなシステムを実行しているマシンが使用されます。 私の選択はRaspberry PIでした。

  • フルLinuxの実行
  • 長い間テーブルの上に横たわり、ほこりを集めます。


動的な外部IPアドレスを使用して、ネットワーク上で実行されているサーバーを構成する方法について説明します。 このようなソリューションは、大規模なプロジェクトの立ち上げには適していませんが、ポートフォリオと個人用アプリケーションのデモには非常に適しています。







必要になります



  1. Raspberry PIモデルB、B +、またはRaspberry PI 2（これらのモデルのボードにはイーサネットがあるため）とRaspbianがインストールされ、SSHサーバーがアクティブになっている。 設定については、 こちら 、 こちら、またはこちらをご覧ください 。 ラズベリーのラスピアンに加えて、多数の代替 ディストリビューションがあります。 ここでは、彼らが言うように、「あらゆる味と色のために」。
  2. 動作中のDjangoアプリケーション。
  3. DDNS対応ルーター。 DDNSはmalink自体で設定できるため、この項目はオプションです。


モデルB +のラズベリーを使用します。

準備する



Raspbian 7.8はraspberryにインストールされます。

まず、ssh経由で接続するためにネットワーク上のラズベリーを見つける必要があります。

nmap -sP 192.168.1.1/24 | grep raspberry











私の場合、ネットワークには2つのラズベリーがあり、そのうちの1つはIPアドレス192.168.1.100のラズベリーです。 一部のネットワークでは、nmapはネットワークデバイス名を表示しません。



この場合、プレフィックスB8：27：EBを持つMACアドレスでraspberry piを見つけることができます。

sudo nmap -sP -n 192.168.1.1/24 | grep -B 2 B8:27:EB









grepの-Bオプションは、印刷する前の行数を決定します。



ssh上のラズベリーに接続しています。

ssh pi@192.168.1.100









始めるために、 raspi-configを使用してラズベリーを1 GHzに加速します。

Pythonパッケージマネージャーをインストールする

sudo apt-get install python-pip







必要なパッケージのインストールに進みます。 私のWebアプリケーションはMySQL DBMSを使用しています。 FrontginとBackendは、それぞれnginxとgunicornです。

sudo apt-get install nginx gunicorn mysql-client mysql-server python-mysqldb









mysqlのインストール中に、ルートDBMSユーザーのデータを入力する必要があります。 python-mysqldbは、Djangoでモデルを操作するときに必要なドライバーです。 PythonリポジトリからDjangoをインストールします。

sudo pip install django







執筆時点で、ラズベリー1.2.1および0.14.5のリポジトリにあるnginxおよびgunicornの現在のバージョン。 ラズベリー5.5のMySQLバージョン。 Djangoを使用するには、 SciPyをインストールする必要があります 。

sudo apt-get install python-scipy







nginx 1.2.1は非推奨です。 新しいものはsourceからコンパイルできます。 新鮮なgunicornは、Pythonリポジトリからインストールできます。



サーバーのセットアップ



Webアプリケーションをラズベリーに配置します（たとえば、/ home / piに）。

動作する設定がある場合は、適切なディレクトリにコピーするだけです：

  • nginx / etc / nginx /サイト対応/
  • gunicorn /etc/gunicorn.d/の場合


nginxで複雑なことはありません。 gunicornの設定に注意を払いたいです。 

 CONFIG = { 'mode': 'wsgi', 'working_dir': '/home/pi/project', #'working_dir': '/home/pi/project/project', 'user': 'www-data', 'group': 'www-data', 'python': '/usr/bin/python', 'args': ( '--bind=127.0.0.1:8081', '--workers=5', # 5    '--graceful-timeout=60', '--timeout=60', #'--debug', #'wsgi:application', 'project.wsgi', ), }




working_dir（wsgy.pyファイルへのパス）が'/home/pi/project/project'



として指定され、argsで'wsgi:application'



として指定されている場合、Malinkaでワーカーが最初に起動し、理由を示すことなく死亡します（たとえば、Ubuntuの場合、 gunicornは両方のオプションで動作します）。



MySQLの移行



mysqldumpユーティリティを使用して、既存のデータベースをダンプできます。

mysqldump -u root -p dbname > dbname.sql







結果のファイルは、構造とデータベースに保存された情報を復元する一連のSQLステートメントで構成されます。



ラズベリーにデータベースを作成します。 mysqlシェルを起動します。

mysql -u root -p







新しいデータベースを追加します。 

 mysql> create database dbname character set utf8 collate utf8_general_ci; mysql> grant all privileges on dbname.* to someusr@localhost identified by 'somepassword';


ダンプからデータを復元します。 ダンプサイズが162 MBの場合、回復時間は約10分でした。

mysql -u root -p dbname < dbname.sql







データベースを外部ドライブに保存することをお勧めします。そうしないと、頻繁な書き込み操作のためにmicro SDカードがすぐに使用できなくなる可能性があります。 これを行う方法はここで読むことができます 。 mysql構成は、パス/etc/mysql/my.cnfにあります



確認する



nginxとgunicornを再起動します。 すべてが正しく構成されていれば、メインページを開くことができます。



ストレステストに合格します。 Apacheベンチマークをインストールします。

sudo apt-get install apache2-utils

DDNSセットアップ



ルーターまたはラズベリー自体でDDNSを構成できます。 数年前から使用していたため、 No IPを選択しました。 無料で使用することを検討してください。

ホスト登録
アカウントをお持ちの場合は承認を行い、それ以外の場合は登録します。 承認後、 ここに行きます 。



[AddHost]をクリックして、フォームに入力します。



以下でAddHostボタンをクリックします

ホストが追加されました。 ホスト名の右側に、ネットワークの外部IPアドレスが表示されます。





ルーターでDDNSを構成する


たとえば、padavanバージョン3.4.3.9-091のファームウェアを使用して、ASUS RT-N56UでDDNSを構成します。 ルーターのメニューページを開く

（例：192.168.1.1）。 WAN-> DDNS。



no-ip.comサービスを選択し、登録データと追加ホスト（technopark-test.ddns.net）を指定します。

残りのパラメーターは自由に設定します。



これで、外部IPアドレスを変更しても、アプリケーションはネットワーク上で使用可能のままになります。

ポート転送設定


ホストにアクセスするとき、MalinkaはWebアプリケーションを提供する必要があります。 ルーターは、ポートXから内部ポートYへの外部からの着信パケットの転送を行っています。ルーターメニューで、[WAN]> [ポート転送]に移動します。 外部80ポートをraspberry 80ポートにリダイレクトする必要があります。 新しいルールを追加して、変更を適用します。



これで、Malinkaはすべての着信パケットをポート80で処理します。 ブラウザのアドレスバーにIPなしで受信したホストを入力して確認します。



これで、インターネットユーザーがWebアプリケーションを使用できるようになりました。

ラズベリーでDDNSを構成します


raspberryにプライベートIPがある場合、このオプションは適切ではありません。これは、ローカルIPアドレスをNo IPサービスに転送するためです。 これは、ラズベリーLANのIPアドレスを見つける別の方法です。 DDNSクライアントをインストールします。

sudo apt-get install ddclient







インストール中に、サービスを選択する必要があります。 その他を選択し、dynupdate.no-ip.com、プロトコルdyndns2、ユーザー名、パスワード、インターフェース-eth0、ホスト名を入力します。



検証のために、IP更新間隔を60秒に設定しました。 ファイル/ etc / default / ddclientで、値daemon_interval = "60"を設定する必要があります。



デザート



私のラズベリーは長い間テーブルの上に横たわり 、tm1638とDHT11とともにほこりを集めて、部屋の温度と湿度の表示やその他の情報を表示していました。

それでも、私はジャンゴからGPIO Raspberry PIを制御しようとすることに興味がありました。 初期の仕事を組み合わせたシンプルなWebアプリケーションを開発しました。 DHT11で測定した温度と湿度、いくつかの有用な情報を確認し、8リレーモジュール（電化製品の制御に使用可能）を制御し、テキストをtm1638に送信できます。









GPIOを管理するには、ルート権限でサーバーを実行する必要があります。 これは潜在的な脆弱性です。

Webアプリケーションを完全に使用するには、サーバーがスーパーユーザー権限なしで動作し、httpsを構成し、アカウントを管理する機能を追加し、ログを維持し、管理対象デバイスへのアクセスを共有し、電化製品のスケジュールされた操作などを行います。
