趣味をスタートアップに発展させるべきですか？





何年も前、私はプログラミングに興味を持ち、自分用のアプリケーション（AutoCADのプラグイン）を書き始めました。 そして何年も経ち、疑問が生じます-スタートアップであなたの趣味を発展させるのか、それとも趣味としてさらに残すのですか？ カットの下のトピックに関する議論



このエントリは、1つのボトルに収められたストーリー、推論、および質問です。 したがって、示されている3つの部分に分割することが論理的です。



物語



2007年、22歳の老人として、故郷のデザイン研究所で職を得ました。 そしてその前に、私は建設業界で中等特別教育を受けました。 この分野で最高を獲得した後、それはポイントではありません。 AutoCAD製品をレビューした後、私はすぐにクールな機能に気付きました-それのためのプラグインを書く機能。 学校では、この問題に興味があり（おそらく他の皆と同じように-学校には基本がありました）、VBAを試し始めました。 その後、彼はすぐにAutoCADに近いもの-Lispに切り替えました。



1つの重要な点に注目する価値があります。放課後、オートメーションおよびコンピューターエンジニアリング学部の地元の技術大学に入学したかった（つまり、はい-プログラマーとして勉強したかった） 40のうち36のスコアが可能です。 そして、彼は祖父の足跡をたどりました-建設中。 おそらく、この苦い経験は、一般的にプログラマーとして勉強する意欲をそそりました（これは残念です）が、定期的に惹かれます



しかし、メインの話に戻ります-Lispの基本だけを学んだので、私は自分用にいくつかのプラグインを書きました。 そして、私は突然もっと欲しかった。 嘘をつかない-名声が欲しかった！ プラグインをまとめ、元の名前ModPlus（おそらくCADという言葉を名前に付けなかった数少ない人の1人）を思いつき、インターネットに登りました。 広がる。 無料で。



しかし、同じ考えが私の頭に常に座っていました（そしてまだ座っています）-それはただの趣味です！ そして、この考えにより、プラグインを長期間削除することができました。 最大は2年近くです。



同時に、キャリアの面で人生の変化がありました-2008年の国内の危機、いくつかの作品の変更、そのうちの1つはまったくキャリアに関連していませんでした。 これにはすべて、少額の給与が伴いました。 その結果、これらのスイングによりデザインの主流に戻り、プラグインを再び復活させました。 ちなみに、その頃にはすでにucozで作成された「スーパーサイト」がありました。



AutoCADの便利なプラグインは、同様の便利なプラグインと比較することはできず、定性的にデザインされた外観と作業中のあらゆる種類のダイナミックレンダリングと比較できないことに気付き始めました。 しかし、Lispはこれを行うことができません（これ以上）、新しい言語の検索が開始されました。 C ++を少し見て、言語が自分のレベルではないことに気付きました（より正確には、そのレベルではありません）。 残っているオプションは1つだけです-.Net。 そして、ここで言語の選択は生じませんでした-もちろんC＃。 さて、WPFについて知ってすぐに、WinFormsの使用に関する質問はありませんでした！



その結果、私の夜と週末の多くは私のプロジェクトに充てられました。 その多くは、私がC＃をより良く知っていることに比例して、絶えず書き換えられていました（そして書き換えられました）。 ある時点で、プラグインをモジュール化するというアイデアがありました。 一か所に集められた小さなプラグインの一種の「ストア」。 長い間、私はすでに実装が難しいと思っていたため、実装を引き受けることを「恐れていました」。コードを書き換えるという観点からすれば、それはとてつもない仕事でした。 それでも、私はそれを決めてやった。 外観については、もう少し上に書いたが、かなり人気のあるMahapps.Metroデザインライブラリをすべてのウィンドウにねじ込んだ。これはまだAutoCADには見られない。 並行して、ホスティングを取得し、Joomlaに通常のサイトを作成しました。 それでも何度か苦労した後、私はサイトのテンプレートを手に入れ、モダンに見えるようになりました。 そして最も重要なこと-プラグインのいくつかの部分（同じモジュール）を有料にしました。 ばかげた価格で真-最大100ルーブル。



最初の部分の終わりに、私はすでによりコンパクトに書き、本編を伝えようとしました。 そして、最も重要な（そして悲しい）結論は、過去10年間の人生です！

そして最近、私の趣味のおかげで、私はモスクワにオートメーション部門の国内最大の設計組織の1つに招待されました。 Revitのプラグインを作成します。 趣味は「すぐに支払われる」が、すぐではない



推論



今、私はかなりユニークな製品を手にしています。 類似および類似の製品とは異なり：



  1. 私は最も独創的なモダンなデザインを持っています
  2. 非常に高品質のサイトです（これらのプロジェクトのほとんどでは、サイトは単に惨めです）
  3. 考え抜かれ、よく開発された（もちろん完璧ではない）自動更新システム
  4. 機能提示のほぼ一意の原則（Revit専用の別の類似した原則があります。もちろん、外国の）
  5. すでに通常のユーザーがいます（Lispバージョン以降の一部）


そして、ここでは、趣味をスタートアップに変えるためにすべてがそこにあるようです。 しかし、いくつかのタールのスプーンがあります：



  1. 私は一人です。 私はこれをすべて自分でやっています。 同時に、他のエンコーダー、デコレーターなどを見つければ、プロジェクトがさらにクールになる可能性があることを明確に理解しています。 しかし、「誰かを信頼する」という点で自分自身を克服するのは難しいですが、長年にわたって一人でやった
  2. お金。 必要です！ 広告に投資する必要があります。 さまざまなささいなことに投資する（文字通り今日、私はロゴを開発するためにフリーランサーに前金を支払った）。 サイトでの作業に投資します（私はうろついているわけではありませんが、問題やタスクがあります）
  3. 年齢。 私はすでに32です。ここではおそらく問題を抱えています
  4. 恐怖。 スタートアップの主な問題は、うまくいかない場合はどうでしょうか？！
  5. そして最後はやる気です！ 彼女はしばしばここに欠けていて、私はすべてを地獄に落とすだけです！


質問



自分自身と私のことわざの読者の両方に質問したい-試す価値はありますか？

実際、私は自分自身に決してお金を払わないかもしれない何かに投資するだけです。 そして、あなたにはたくさんのお金が必要です。 そしてそれらの多くがあります。



誰かが今これを読んでいて、まったく同じ推論で頭を悩ませているのかもしれません。 私のような人は怖いけど
