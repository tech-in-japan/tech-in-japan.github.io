教育者はどのようにデジタル教育リソースを広めることができますか

こんにちは、Megamind！ この記事は、 「大富豪になりたい人：会社を設立せずに金持ちになる7つの方法」という出版物でsemen_grinshteinが説明した方法の1つに関する解説です。 あるサクセスストーリーでは、大学の教師と学校の教師として興味を持った億万長者の教師でした。







幼稚園の先生は、アイデアの販売から追加の収入を確保することができました。 彼女はインターネットを介して、他の教師に教育実践、つまりデジタル教育リソースを販売しています。



教師はどうすれば百万を作ることができますか？



まず、IT教師としての非常に明らかな収入源について説明します。仕事の主な場所は大学であり、パートタイムは学校の教師で、週末は家庭教師、時にはフリーランスのプログラミングです。 このリストは、教育リソースをお金で共有することで拡張できることがわかりました。 これは、教育者と購入者、教育者と販売者の両方にとって、誰にとっても有益である非常に良いアイデアです。 この記事では 、自分のサイトの会社を作成せずに教育リソースを配布する方法（著者にとって有益な方法）を検討しました。



世界で



教師PayTeachersは、上記の記事で幼稚園の先生から100万ドルを稼ぐのを助けました。 売り上げを確保するために、サービスは獲得した金額の40％でリソースの作成者から手数料を受け取り、取引ごとにさらに30セントを受け取ります。 年間支払いが59.95ドルの場合、プレミアムサブスクリプションを購入できます。これにより、手数料が売上の15％に減り、取引手数料が完全になくなります。



教師向けのオンラインマーケットは、2006年4月にニューヨークのPaul Edelmanという学校の教師によって作成され、ほぼ10年間TpTは180万人のリソースと360万人の教師のコミュニティを獲得しました。すでに1億7500万ドルを支払っています。



RuNetで



今日、ロシア語を話すオンラインコミュニティがいくつかあり、教師向けの取引プラットフォームが1つあります。教育資源の配布に関連する文部科学省の3つの公式プロジェクトもあります。

  1. 教育者のソーシャルネットワーク（ ポータル ）-すべての教育レベルの教師のために、ファイルはすべての形式でダウンロードできます。 教師は自分のミニサイトを作成し、開発をミニサイトに投稿し、90ルーブルで電子メディアに掲載された電子証明書と個人のWebサイトを作成した電子証明書を受け取ることができます。
  2. 教師相互支援コミュニティ（ ポータル ）-幼稚園の教師と学校の教師の場合、ファイルはms office形式またはアーカイブでダウンロードできます。 教師は自分の開発を公開し、200ルーブルで電子証明書を発行するか、250ルーブルで郵送でハードコピーで受け取るように注文できます。
  3. 教育リソースとレッスンプラン（ 取引プラットフォーム ）-すべての教育レベルの教師向けに、doc、ppt、xls、pdf、zip、rarなどの形式でファイルをダウンロードできます。著者はリソースを公開し、価格を割り当てることができます。 売り上げを確保するために、このサービスは著者から収入の15％の手数料を受け取りますが、年間1500ルーブルのプレミアムサブスクリプションを購入すると、手数料を売上の5％に引き下げることができます。 これは、TeachersPayTeachersのロシア語版です。
  4. デジタル教育リソース（ EC CORポータル ）の単一コレクション-学校の教師向けに、オフィスドキュメント形式、表、プレゼンテーションでファイルをダウンロードでき、インタラクティブなswfリソースがあります。 教師はリソースを公開できません;リソースプロバイダーの範囲は教育機関と商業組織に限定されています。
  5. 連邦教育情報センター（ FCIORポータル ）-一般教育および専門教育の教師向けに、ファイルを特別なOMS形式でダウンロードできます。リソースを公開することはできませんが、このプロジェクトを検討することは依然として興味深いです。
  6. 教育リソースへのアクセスの単一ウィンドウ（ ポータル ）-一般および専門教育の学生向けに、教育情報のあるサイトへのリンクが収集され、電子教科書をPDF形式でダウンロードできます。 登録する必要がある独自のリソースを追加できます。


教育省のポータルでリソースを公開した教師に対する金銭的報酬は提供されません。

教師のソーシャルネットワークは、メディアで出版の証明書を提供することにより、著者の意欲を高めます。

上記の例のいくつかについて詳しく説明します。

2.教師相互支援コミュニティ



「エカテリーナパシュコバの教育コミュニティ-PEDSOVET.SU」は2007年に設立され、2010年に電子メディアとして登録されました。 素材をサイトに送信することにより、著者はロイヤリティを必要とせずに無料で、編集者に商用または非商用目的で素材を使用する権利、特に作品を複製、公開、翻訳および処理する権利を譲渡します- ロシア連邦民法で。 （1270条およびその他）。 ポータルの新しく作成されたセクションには、これらのセクションを迅速に記入するためのインセンティブとして、無料の発行証明書の特別オファーがあります。



3.教育リソースとレッスン計画



ProgrammaRazvitiya.ruの教育者向けオンラインマーケットプレイスは、デジタル教育リソースを販売することで獲得できる場所です。 このサイトでは、各教師のストアへの個別のリンクとそのリソースが提供され、一般的なリソースカタログでも公開されます。 無料と有料の両方のリソースを配置できます。 このプロジェクトは、TeachersPayTeachersからこのビジネスモデルの成功を証明した後、RuNetに登場しました。 このサービスはまだ若く、最初の著者は2016年初めに登録されました。 教師は、専門分野の作業プログラム、授業計画、開発およびプレゼンテーションを授業およびその他の教育リソースに追加し、価格を割り当てます（0ルーブルが指定されている場合、リソースは無料です）。







販売した教育リソースのSberbankカードでの教師への支払いは、売り手のオフィスで指定された詳細で毎月行われます。 稼いだお金の転送は500ルーブルの蓄積の対象となり、当月に金額がこの閾値に達していない場合、この条件のチェックは来月実行されます。



4.デジタル教育リソースの単一コレクション



コレクションをリッチと呼ぶことはできません。教師に新しいリソースを公開する動機がまったく追跡されず、サイトからリソースを追加することさえできないためです。 リソース作成者は、リソースプロバイダーリストにリストされている組織です 。

ICTの教師として、私はいくつかのリソースを使用しました（それらはほとんどありません）。このポータルのプラスに注意できます。資料はロシアで採用された公式プログラムに対応しています。



ポータルのインターフェースと技術実装の品質は優れていますが、主催者はロシア連邦教育科学省です。 マイナスがあります-ポータルが利用できない場合があります。



5.連邦教育情報センター



FCIORは、 10のサプライヤー （2つの州立大学と8つの商業企業）の特定のサークルからのみ、独自の強制医療保険形式でリソースを提供します。 興味のある教師は、リソースを独自に公開することはできません。



オープンな教育用モジュラーマルチメディアシステム（OMS）
オープン教育モジュラーマルチメディアシステム（OMS）は、モジュラーアーキテクチャの電子教育リソースです。 電子トレーニングモジュールを使用するには、ポータルからOMSをダウンロードし、ローカルコンピューターの特別なソフトウェア（リソースプレーヤー）にインストールする必要があります。



リソースを表示するためのソフトウェアは、電子トレーニングモジュール（EUM）の保存、検索、選択、および再生を提供します。

  • OMS Player-EUMを再生するように設計されています。 プレーヤーは、ローカルストレージにあるモジュール間でEUMを検索する機能を提供します。 ローカルストレージは、ユーザーのコンピューター上の特別な専用フォルダーで、OMSのソフトウェアコンポーネントのインストール時に指定されるパスです。
  • OMSユーザーのオーガナイザー-FCIORの中央リポジトリからユーザーのローカルストレージへの電子コンピューターのロード、およびユーザーの職場での選択されたモジュールのストレージを提供します。 オーガナイザーを使用すると、ローカルストレージにある電子コンピューターのメタデータを再生および表示できます。




技術プラットフォームFCIOR
FCIORハードウェアコンプレックスは、IBMソリューションとテクノロジーに基づいています。



センターの計算能力は、高性能のIBMサーバーによって提供されます。 さまざまな種類の電子教育リソースの効果的なストレージは、オープンソースプロジェクトを使用して実行されます。 FCSIORソフトウェアアーキテクチャは、Apache Software Foundation（Tomcat、Lucene、Commons）プロジェクトを使用して、JavaServlet、Java Server Pagesテクノロジに実装されています。 MySQLは、アプリケーションシステムのDBMSとして使用されます。 アプリケーション開発-Java 2プラットフォーム、Enterprise Edition。





6.教育リソースへのアクセスの単一ウィンドウ



シングルウィンドウ情報システムに新しいリソースを追加することを提案できます-電子ライブラリに資料を配置するか、カタログのインターネットリソースにリンクします。



送信された文書は、あなたがその著者（共著者）であるか、著者の知識と同意を得て提供し、あなたの行動が2006年12月18日N230-の「ロシア連邦民法（パート4）」と矛盾しない場合、電子ライブラリに置くことができます。







電子図書館は、専門分野の作業プログラム、授業計画、授業の開発とプレゼンテーションなどを表す資料をホストしていません。



カタログには、インターネットで公開されている科学的および教育的性質のリソースへのリンクが含まれている場合があります。



おわりに



教師がRunetでデジタル教育リソースを広める十分な機会がありますが、これまでのところ、著者が独自の開発を公開するという物質的な動機は十分に開発されていません。 デジタル教育リソースの海外市場の10年の年齢と経験が示すように、独自のデジタル学習センターを販売することは有益です。 2016年まで、あなたは著作権で保護された資料をインターネット上で無料で公開ポータルまたはプライベートポータルでのみ世界に提供できました（後者は公開証明書を提供する有料サービスを提供します）。 最近、代替手段が登場しました-リソースでお金を稼ぐためですが、これの有効性について話すのは時期尚早です。 私の意見では、これは、教師が給料に加えて稼ぐことができる場合に、より新しい高品質の資料を公開する非常に効果的な動機です。
