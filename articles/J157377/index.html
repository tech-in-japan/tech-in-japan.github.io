スクールクラフト：Battlecruiser＆DeathCraft

ご注意 ノスタルジアを集中！ ここで説明するのは、何年も前の時代遅れです。 さらに、実装される前は時代遅れです。 この記事から、新しいことは何も学びません。

昔、 Embarcadero CodeGear Borland Delphi RAD Studio C ++ Builderがバージョン4であったとき、 Autodesk 3ds Max Discreet 3dsmax 3D Studio Maxがバージョン3であったとき、80 GBおよび200 MHzは冷蔵庫ではなくコンピュータで、インターネットは未知の神秘的な生き物、そのすべてのタッチは魔法でした...私は高校で勉強し、プログラミングを学びました。 また、シミュレートします。 そして、ささいなことに関するあらゆる種類の異なること。

疑わしいスクリーンショット：









お父さんは警官プログラマーでした（現在は警官警官です）。 そして、彼はコンピューターを購入しました-理論的にはアルバイトのためですが、ヘリコプター工場はどういうわけか急激に乾燥しすぎて、コンピューターはめったに使用されませんでした（その目的のため）。 私は何度か写真を見ました。あらゆる種類のボタン、金型、あらゆる種類のものが押された後、動作します（C ++ Builderでした）。 彼の目が光りました：それはクールです-あなたはボタンを描き、それからそれ自体が何かをします！ 彼らは私に型を描かせました。

ただし、問題が発生しました。ボタンは描かれていますが、何かを動作させたくないのです！ 突然、奇跡のプログラムでは、ボタンが1つの図面で移動しないので、奇妙な不可解なコードを書く必要があることがわかりました。 ビルダーに関する本を手に入れました。 予想外に、それを読む前に、少なくとも少しのCを知ることは非常に有用であることが判明しました...私はCに本を置きました。

奇妙なことが判明しました。CでBilderで動作しないプログラムに関する本（スキルで実行できることは明らかですが、この種のシャーマニズムについては学びませんでした）と、Prosの知識が必要なBilderの本（ここにクラスがあります）メソッド、ここはヘッダーファイルです...）。 しかし、何かが動き始めていました。 本からコードを転載することは、気弱な人のための活動ではありません（後でディスクがまだ本に依存していることがわかりました...）が、それは知識に対する渇望でした。 そして、最も興味深いのは、Bilderの本の最後に、グラフィックを扱うプログラムがあったことです。 スプライトが表示されました！ アニメーション付きの、切り抜かれたカラー写真！ （おそらく、私がどんな種類の本を持っているかは誰もが既に理解していたでしょう。

メソッドの本質：写真を撮り、将来の透明な領域を黒で塗り、その隣にマスクを置きます：オブジェクトは黒、背景は白です。 最初にマスクをSrcAndモードで表示し、次に画像をSrcPaintモードで表示します。 これがすべて消滅しないように、最初にすべてをメモリ内の画像に描画し、それをフォーム上のキャンバスに表示します。 魔法の秘密が明らかになりました！ そして離れて行く...

ゲームを作成する過程で、3Dモデリングを学びました。

おなじみのスクリーンショット？



Pentium 200 MHz、32 MB RAMで3D Studio Maxを使用したことがありますか？ プロセスは神経をよく訓練します。 スタジオは10〜15分しかロードしません...分単位で最も単純なモデルをレンダリングします...付属のサンプルから10分間、宇宙飛行の1フレームをレンダリングします... クールなものを描くことができます！

最初は、2つのロボットが互いに銃を突きつけて来るローラーセーバーがおもちゃに取り付けられていました。 由緒ある一般に公開する前に、ビデオを再レンダリングすることにしましたが、見つかりませんでした。 同じ時代の別のロボットのシーンを見つけました。 このロボットをスクリーンセーバーに表示します。

アニメーションの最後のフレーム、開始する時間です。



Pentium 200 MHz、32 MB RAMでC ++ Builderを使用したことがありますか？ もちろん、これは3D Studio Maxではありませんが、私のような基本的なプロジェクトのコンパイル（当時は完全に非基本的なように見えましたが）には1分ほどかかります。

古いプロジェクトを使用したC ++ Builder 6：



あなたは今までに...そしてそうではないが、古代のコンピューター上のSound Forgeはそれ自体で非常に賢く働いた。 現在、フランジャーやワウワウなどの奇妙な名前のエフェクトが、標準のサウンドレコーダーからのエコー、加速、減速、Uターンのエフェクトに追加されています。 通常、サウンドは互いにリンクできます。 かっこいいですね。 恐竜でさえ私の声で話します。

実行しましょう：



一部のユニットはStarCraftから引き出されます。 それから私はグラフィックがコーシャの方法によって引き出されることができることを知りませんでした、そして古き良きプリントスクリーンを使用しました。 MS Paintで編集しました。 冬の場所でスクリーン印刷を試みたので、ピクセルを見つけやすくなりました。 あなたは尋ねるかもしれません：MS Paintと3D Studio Max-それは奇妙なセットですか？ 奇妙な。 しかし、他の編集者についてどのように知りましたか？あなたのインターネットの編集者のことすら知りませんでした。 まあ、私は決定をグーグルで検索することはできませんでした。 私のおもちゃに彼のデータベースがあるお父さんは、足でもありません...

一部のユニットは3D Studio Maxで描画されます。 すべてのソフトウェアと同様に、それは「チェーホフ市場」（「ザトウクジラ」のカザン版）で撮影されました。 私の両親は私のために本を買った。 プログラムのチュートリアルは恐ろしく、ひどく、英語で、ソースコードが添付されていません（または、すでに次のバージョンにありましたか？..）。 何かが飛び出しました。 主にロボット。

ゲームにはAIがあります。 敵は建物を構築し（リストに従って、コマンドセンターに対して円状に配置）、収集ユニットを構築し（制限まで）、攻撃ユニットを構築し（制限まで）、プレイヤーを攻撃します（最小攻撃ユニットが設定されている場合）。 AIいいえ、一般に、しかし彼の仕事を知っていて、プレーヤーはそれを非常に効率的にします。

敵は、SCVをすべて装備して兵舎を建設し、増援を訓練します。



パスを見つけるためのアルゴリズムはありません。ユニットは最初の障害物に寄りかかっているので、斜めに送信する方が良いです。すべての障害物が長方形の場合、それらをバイパスできます。 ユニットには、ある時点で互いにくっつくという不快な機能があります。 プレイヤーのユニットの場合、スプレッドは設定されていますが、コンプは時々1つの殺すことができない山（こんにちは文明）で歩きます。

マウスコントロール：左クリックでユニットと建物を選択し、右クリックで移動して攻撃するコマンド。 エリアの選択はありませんが、Ctrlを使用して画面上に選択したタイプのすべてのユニットを選択できます。

キーボード制御：

  • 矢印：マップ内を移動します
  • H：選択した単位に従う
  • P：一時停止
  • 削除：ユニットを削除します
  • C + U + F12：ユニットを作成するためのチートコード。 次に、テンキーに2つの数字-ユニットコードと（オプションで）2桁-プレイヤーの数字を入力する必要があります。 たとえば、C + U + F12、0500は、マウスを使用してプレーヤーのロボットを作成するモードに切り替わります。 無効にするには、C + U + F12をもう一度押します。




ヒント：

  • 製油所を守れ！ まず第一に、敵はこの建物を破壊します。
  • 軍隊を構築する時間がなく、敵がすでに攻撃している場合、飛行機から投げられた恐竜があなたを救います。
