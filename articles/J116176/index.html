E5000-Exchange 2010大規模メールボックスのソリューション

以前の投稿で、 Microsoft Exchange Server 2010用のHP E5000メッセージングシステムの電子メールのターンキーソリューションの市場への期待される参入について既に書いています 。 このシステムは、最新のCeBITでHPによって公式に導入されましたが、現在、このシステムについて詳しく知る機会があります。







まず、Microsoft Exchange Server 2010の2つのイノベーションに言及する価値があります。これらは、E5000で積極的に使用されています。 第一に、メールボックスの最大サイズのこの増加は、7200 rpmの回転速度を備えた比較的安価な3.5インチシングル/ダブルテラバイトディスク上のメールサーバー上のストレージによるものです。 Microsoft Exchangeの以前のバージョンでは、7200 rpmドライブが遅すぎるため、このストレージオプションは使用されませんでしたが、メールサーバーの新しいバージョンでは、Microsoftが入出力を最適化したため、IOPS要件はテラバイトが提供できるレベルまで低下しましたホイール。



Microsoft Exchange Server 2010の2番目の主要な機能強化は、データベース可用性グループ（DAG）に統合されたExchangeサーバー間のレプリケーションによるメールサービスのフォールトトレランスの提供です。各サーバーは独自のDASディスクシステムにデータベースを格納します。 アクティブデータベースは1つのDAGサーバーで実行され、パッシブデータベースは2番目のサーバーで実行されます。 Exchange 2007では、クラスターが高可用性のために使用され、データベースは単一のSANディスクアレイに格納されていました。



E5000は、2つのWindows Server 2008 R2 Enterpriseライセンスで完全に展開可能な構成で顧客に提供されます。 さらに、HPには、システムの初期セットアップ、構成、および監視のためにMicrosoftで開発されたソフトウェアユーティリティが含まれていました。

たとえば、クイック展開ツールウィザードを使用すると、Exchangeの設定、ディスク（LUN、マウントポイントなど）の構成、DAGの作成と構成、メールボックスデータベース名の設定、データベースコピーの作成、サーバー間のレプリケーションの設定をすばやく行うことができます。 。 E5000とともに、Exchangeシステムマネージャーダッシュボードと呼ばれるサーバーマネージャープラグインがリリースされました。 このコンソールは、システム全体のステータス（Exchange自体（レプリケーション、データベースステータスなど）と機器の両方）を表示するのに非常に便利です。



お客様はマイクロソフトとボリュームライセンス契約を結んでいる可能性があるため、Exchange for E5000のサーバーライセンスとクライアントライセンスはマイクロソフトとは別に購入します。



HPは、最大サイズが1.2ギガバイトまたは2.5ギガバイトの500、1000、3000個のメールボックス用に5つのE5000モデルを生産しています。 システムの出荷時設定では、1つのメールボックスにつき1日あたり最大200件のメッセージが処理されます。 すべてのE5000モデルは、Xeon E5500プロセッサーを搭載した2つのProLiant BL460cブレード、DAG、MicrosoftによってExchange 2010 I / O用に最適化された新しいP1210mディスクコントローラーを備えたテラバイトSASディスクを備えたMDS600ディスクモジュールで構成されます。各ブレードにはExchangeの「役割」がありますメールボックス（MBX）、ハブトランスポート（HT）、およびクライアントアクセスサーバー（CAS）。



ブレードはMDS600とともに1つの3ユニットケースに取り付けられ、このモジュール（DAS）のディスクに直接接続され、2つのRAID-1アレイ（ブレードごとに1つ）に結合されます。



新しいモデルのE5300は、最大1.2 GBのサイズの500個のメールボックスに対応するように設計されており、12個のシングルテラバイトドライブが付属しています。 1000個のメールボックス用のE5500モデルは、最大1.2および2.5 GBのメールボックス用に16個のディスクを持つ2つの構成で提供されます（最初はシングルテラバイトディスクを使用し、2番目は2テラバイトを使用します）。 E5000シャーシにメールボックスを追加するときにこれらのモデルのディスクサブシステムの容量を拡張するには、12個のフルサイズSASハードドライブ用の最大4つの外部ディスクシェルフをSASポート経由で接続できます。 最後に、3,000個のメールボックス用の古いモデルE5700には、2つの拡張シェルフが標準装備されています。 ボックスの最大サイズ（1.2または2.5 GB）に応じて、その構成は1つまたは2つのテラバイトディスクを使用します。



Exchangeを災害から保護するために、異なるサイトにインストールされた2つのE5000を1つのDAGに結合できます。



Microsoft Exchange Server 2010用E5000メッセージングシステムを使用すると、3Uのみのシステムを使用して、大規模な組織で最適なサーバーおよびディスクサブシステム設定で新しいMicrosoft電子メールシステムを迅速に展開できるだけでなく、低メッセージストレージコストと電子メールサービスの高可用性も実現できます。
