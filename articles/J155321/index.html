zabbixでSMS通知を設定する

zabbixのSMS通知の問題はすでにHabréでカバーされており、SMSゲートウェイを使用して問題は解決されました。 Delphiを使用したSMSの送信についても言及しました。 USBモデムを使用してSMS通知を構成する方法を教えてください。

だから私たちが持っているもの：



1）かなり広範なITインフラストラクチャ



2）Fedora 14 x64で機能するZabbix 2.0.3モニタリングシステム（執筆時の現在のバージョン）



3）ウクライナ国立オペレーターKyivstarの3G USBモデムZTE MF100。











4）モデムに付属のプリペイドSIMカード。



また、モデム用に企業の3G SIMを購入しましたが、3GカードからSMSを呼び出して送信することは不可能であり、これは私たちには適していないため、動作しません。 。



必要なのインストール



まず、 マイキエフスターシステムにカードを登録しました。 モデムの口座の残りのお金を調べて、追加注文するのが便利です。 Fedoraですべてのアクティビティを実行していることを思い出させてください。 他のLinuxの場合、コマンドは異なる場合がありますが、どの置換があなたにとって簡単かを判断することを考えています。 また、必要なソースをインストールするオプションも検討しませんが、これも可能です。 既存の標準パッケージのインストールに限定します。 
sudo yum install usb_modeswitch #   USB- Huawei, T-Mobile, Vodafone, Option, ZTE, Novatel sudo yum install minicom #       /dev/ttyUSB* sudo yum install smstools #      .
モデムが表示されたかどうかを確認してください 
 ls -l /dev/ttyUSB*
結果は次のようになります 
 crw-rw----. 1 root dialout 188, 0 Oct 22 12:05 /dev/ttyUSB0 crw-rw----. 1 root dialout 188, 1 Oct 22 15:56 /dev/ttyUSB1 crw-rw----. 1 root dialout 188, 2 Oct 22 12:05 /dev/ttyUSB2
モデムは3つの仮想USBポートを作成しました。 私の場合、ttyUSB1とttyUSB2はATコマンドに使用できます。 モデムが5つ以上の仮想ポートを作成している場合、それらの数は3倍に減らすことができます。 これを行うには、cdromおよびフラッシュドライブモードを無効にします。 スイッチオフは、 コマンドAT + ZCDRUN = Eによって行われ、スイッチオンAT + ZCDRUN = Fによって行われます。 cd-romのみを無効にする場合は、 AT + ZCDRUN = 8です。 オンに戻したい場合-AT + ZCDRUN =9。ターミナルを起動します 
 minicom -D /dev/ttyUSB1
あいさつを見る 
 Welcome to minicom 2.5 OPTIONS: I18n Compiled on Feb 24 2011, 11:25:55. Port /dev/ttyUSB1 Press CTRL-A Z for help on special keys AT S7=45 S0=0 L1 V1 X4 &c1 E1 Q0 OK
CD-ROMをオフにする 
 >AT+ZCDRUN=8 Close autorun state result(0:FAIL 1:SUCCESS):1 OK
一般に、電話モデムの時代には、自尊心のある管理者にとって知識は必須でした。 これですべてが変更され、Googleを使用して忘れられたコマンドを思い出すことができます。 たとえば、良いヘルプ記事があります。モデムと話をしてみましょう。モデムの動作モードを見てみましょう 
 >AT+ZSNT? +ZSNT: 1,0,0 OK
私の場合-= 1,0,0-GSMネットワークのみの自動選択。 3G SIMカードではなく通常のSIMカードを使用しているため、このモードは完全に適しています。 次の動作モードオプションもあります。 
 AT+ZSNT=0,0,0 () -   AT+ZSNT=0,0,1   : GSM+WCDMA,  GSM AT+ZSNT=0,0,2   : GSM+WCDMA,  WCDMA AT+ZSNT=1,0,0   :  GSM AT+ZSNT=2,0,0   :  WCDMA AT+ZSNT=0,1,0   : GSM+WCDMA AT+ZSNT=1,1,0   :  GSM AT+ZSNT=2,1,0   :  WCDMA
ピン保護が設定されているかどうかを確認します。 
 >AT+CPIN? +CPIN: SIM PIN OK
PINを無効にする 
 >AT+CLCK="SC",0,"1111"
PINを有効にする 
 >AT+CLCK="SC",1,"1111"
オンになっている場合、作業を続行するには入力する必要があります 
 >AT+CPIN=”1234″ OK
もう一度確認してください 
 >AT+CPIN? +CPIN: READY OK
これでPINコードが受け入れられました。 信号レベルを見てみましょう 
 >AT+CSQ +CSQ: 21,99 OK
モジュールのIMEIバージョンを取得します 
 >AT+GSN 359518034903581 OK
ネットワークでの登録のタイプを見てみましょう 
 >AT+CREG? +CREG: 0,1 OK
0は、ネットワークの登録変更に関するメッセージがないことを意味します1-ネットワークに正常に登録されましたオペレータに関する情報が表示されます 
 >AT+COPS? +COPS: 0,0,"Kyivstar",0 OK
最初の0は、自動ネットワーク選択、名前の長いテスト形式（2番目の0）、およびオペレーターの名前自体-Kyivstarを意味します。インストール済みのSMS送信センター（サービスセンターアドレス）を確認しましょう 
 AT+CSCA? +CSCA: "+380672021111",145 OK
これはキエフスターの正しい番号です。メッセージを送信してみましょう。 メッセージを送信する前に、メッセージ形式を設定する必要があります。 これは、コマンドAT + CMGF = xで行われます。 xはメッセージ形式、0〜1.0はPDUモード、コマンドコード制御です。 HEXコードでのメッセージ出力。 デフォルトモードは1、テキストモードです。 コマンドはテキストです。 メッセージはテキスト形式で出力されます。最初のモードでは、メッセージは16進コードasciiまたはunicodeの形式で表示されます。これは非常に不便なモードです。SMSの送信は、 AT + CMGS = SMSの送信は、メッセージの形式によって異なります。 コマンドを入力すると、「>」のプロンプトが表示され、その後にメッセージテキストを入力できます。 ESCまたはCtrl-Z文字で終了します。 テキストモードを使用し、SCAを事前に設定してメッセージを送信します。 
 AT+CMGF=1 OK AT+CSCA= "+380672021111" OK AT+CMGS="+38067xxxxxxx" > test message > +CMGS: 110 OK
SMSが正常に到着しました。SMSをメモリに書き込むことで送信できます。 
 AT+CMGW="91234567"<CR>Sending text messages is easy.<Ctrl+z> AT+CMSS=3
ATチームで作業する場合、すべてがスムーズに進むとは限りません。 時々コマンドはエラーを伴って実行され、3桁のエラーコードを返します。 主なエラーコードのリストは次のとおりです。

smstoolsの構成



次に、SMSツールを構成します。 config /etc/smsd.comfを編集します。 コメント付きの既成の設定を用意します。 
 # ,   —    devices = GSM1 #    incoming=/var/spool/sms/incoming outgoing=/var/spool/sms/outgoing checked=/var/spool/sms/checked sent=/var/spool/sms/sent # ,    —   syslog,  = 1,   stdout logfile = /var/log/smsd/smsd.log #  loglevel = 5 #debug = 7 user = sms # ""  smart_logging = yes #,           "" .     checkhandler = /etc/smsd/check.sh #,        .     eventhandler = /etc/smsd/trsms.sh [GSM1] #  device = /dev/ttyUSB1 #- pin = 1111 #SCA smsc = 380672021111
smstoolの公式 Webサイトで、すべての構成パラメーターについて詳しく読むことができます。



スクリプトtrsms.sh



次に、送信されたSMSのログを書き込むスクリプト/etc/smsd/trsms.shを作成します。 さらに、すべてのキリル文字メッセージは、読み取り可能なエンコードでそれに分類されます。 スクリプトを実行する権限をsmstoolユーザーに付与し、/ var / log / smsd / sms.logログに書き込むことを忘れないでください 
 chown smstools:smstools /etc/smsd/trsms.sh chmod gu+x /etc/smsd/trsms.sh chmod gu+w /var/log/smsd/sms.log
スクリプト自体： 
 #!/bin/bash status="$1" file="$2" touch /etc/smsd/ok case "$1" in SENT) FILE=`mktemp /tmp/smsd_XXXXXX` head -5 $file | grep -e "^To: " -e "Alphabet: " -e "^Sent: " >> /var/log/smsd/sms.log if grep "Alphabet: Unicode" $file >/dev/null; then tail -n +6 $file | iconv -f UCS-2BE -t UTF-8 >> /var/log/smsd/sms.log else tail -n +5 $file >> /var/log/smsd/sms.log fi echo >> /var/log/smsd/sms.log echo "========================================" >> /var/log/smsd/sms.log echo >> /var/log/smsd/sms.log ;; esac
ログ部分の例： 
 ======================================== To: 38067XXXXXXX Alphabet: Unicode Sent: 12-10-23 21:07:29 test message ======================================== To: 38093XXXXXXX Alphabet: Unicode Sent: 12-10-19 12:16:28       ========================================


スクリプトcheck.sh



SMSToolsは、外部プログラムを使用してイベントを処理できます。スクリプトは、送信前にメッセージをチェックすることにより、正しいエンコードでロシア語SMSメッセージを送信するのに役立ちます（unicode2smsは、/ usr / binにある標準スクリプトパッケージからプロトタイプとして取得されました。 smstoolユーザーへのスクリプト 
 chown smstools:smstools /etc/smsd/check.sh chmod gu+x /etc/smsd/check.sh
スクリプト自体： 
 #!/bin/bash # checkhandler for SMS Tools 3 # autoconverts cyrillic messages to UCS-2BE # add checkhandler=/path/to/ucsautoconvert into global part of smsd.conf # written by lexy (lexy@mrlexy.ru), 2008 FILE=`mktemp /tmp/smsd_XXXXXX` if [ ! `grep '[--]' $1 > /dev/null` -o `grep 'Alphabet:\s*U' $1 > /dev/null` ] then exit 0 fi cat $1 | awk '{if(NF==0) {s=1} if(s==0 && NF>0 && $0!~/Alphabet:[ \t]*U/){print}}' > $FILE echo Alphabet: Unicode >> $FILE cat $1 | awk '{if(NF==0) {s=1} if(s==1){print}}' | iconv -t UCS-2BE >> $FILE mv $FILE $1 chmod 664 $1
これですべての準備が整いました。 
 >service smsd start Starting smsd: [ OK ]
開始したかどうかを確認します 
 > ps ax | grep smsd 1345 ? Ss 0:00 /usr/sbin/smsd 1347 ? S 0:00 /usr/sbin/smsd 2322 pts/2 S+ 0:00 grep --color=auto smsd

Zabbixのセットアップ



ZabbixがSMSを送信するために実行するスクリプトを作成します。 Zabbixでは、このようなスクリプトはAlertScriptsPath変数で定義されたディレクトリにあります。 
 cat /usr/local/etc/zabbix_server.conf | grep AlertScriptsPath ### Option: AlertScriptsPath AlertScriptsPath=/home/zabbix/bin/
スクリプト自体はsms_send.shと呼ばれ、ほぼ1行で構成されています。 
 #!/bin/bash to=$1 subject="$2" body="$3" smssend $to "$subject"
さらに設定は、Webインターフェースを介して行われます。 まず、管理->通知方法（管理メディアタイプ->）に移動し、新しい通知方法（メディアタイプ）を作成します。 スクリプト名はAlertScriptsPathディレクトリのスクリプトファイル名と同じである必要があります



さらにユーザー設定（[管理]-> [ユーザー]または[管理]-> [ユーザー]）で、通知方法（メディア）を追加します。



その後、アクション設定でこのメディアを使用できるようになります。







関税



1 SMSの標準価格= 0.5 UAH。 大量のSMSを使用すると、かなりの費用が発生する可能性があります。 オペレーターは、コストを削減するために何を提供していますか？



オプション1-プリペイド



次の2つのオプションがあります。
  1. 1週間あたり4 UAHの月額料金の「パッケージ30 SMS」。 1 SMS = 13.33コペックの価格であることがわかります。
  2. 1週間あたり6 UAHの月額料金の「60 SMSパッケージ」。 1 SMS = 10コペックの価格であることがわかります。
価格は良いです。 しかし、欠点もあります：
  • どちらの場合も、未使用のSMSは週末に「燃え尽き」ます。

  • My Kyivstarに定期的にログインしてサービスを注文する必要があります。そうしないと、最も重要な瞬間に予告なしに留まることができます。




オプション2-契約

次の3つのオプションがあります。
  1. 「SMSパッケージ-50％」、月額3 UAHの月額料金。 パッケージには500のSMSが割引価格で含まれています。 1 SMS = 25コペックの価格であることがわかります。
  2. 「SMSパッケージ-70％」、月額7 UAHの月額料金。 パッケージには500のSMSが割引価格で含まれています。 1 SMS = 15コペックの価格であることがわかります。
  3. 「パッケージ1000 SMS」、月額120 UAHの月額料金。 パッケージには、1000個の無料のSMSが含まれています。 1 sms = 12 kopecksの価格であることがわかります。
プリペイドパッケージよりも高価ですが、毎週アクティベートする必要はありません。 一度やった-そして忘れてしまった。

ところで、価格は競争力があります。 たとえば、組織のインターネットサービスsmscentre.kiev.uaは、15コペックの価格を提供しました。 加入者なしで、1か月あたりのSMS数が1〜25000の1つのSMSの場合。 ボードとパッケージの無料アクティベーションを使用しますが、モデムを使用する場合、インターネットからの独立性という、紛れもない利点が得られます。 結局のところ、重要なのは大量のSMSメッセージではなく、Zabbixからの通知です。 また、インターネットSMS配信サービスを使用する場合、たとえば、オフィスでの長時間の停電やプロバイダーでの事故の場合、重要なSMSを受信しないというリスクがあります。 どのパッケージを選択するかはあなた次第です。

SMSパッケージの詳細については、契約およびプリペイドについてをご覧ください。



PlaySMSセットアップ



このアプリケーションのインストールはオプションです。 このアプリケーションは、mysqlデータベースに保存されているSMSメッセージの大量メール送信に必要であり、httpd、mysql、php、php-pear、php-mysql、php-cli、phpmyadminの各パッケージが動作する必要があります。 しかし、Zabbixが機能するためにこれらのパッケージをすべてインストールしました。 PlaySMSの新しいユーザーとディレクトリを作成します。 PlaySMSの最新バージョンをダウンロードし、mysqlにデータベースを作成します。 
 adduser playsms mkdir -p /var/www/playsms mkdir -p /var/spool/playsms mkdir -p /var/log/playsms chown -R apache /var/www/playsms chown -R apache /var/spool/playsms chown -R apache /var/log/playsms chmod -a -G smstools playsms wget http://downloads.sourceforge.net/project/playsms/playsms/Version%200.9.7.1/playsms-0.9.7.1.tar.gz tar -zxvf playsms-0.9.7.1.tar.gz -C /usr/local/src cd /usr/local/src/playsms-0.9.7.1/web/ cp -rR * /var/www/playsms cp playsmsd* sendsmsd* /usr/local/bin/ chown -R apache /var/www/playsms mysqladmin -u root -p create playsms
作成されたデータベースにインポートします。 
 mysql -u root -p playsms < /usr/local/src/playsms-0.9.7.1/db/playsms.sql
設定をコピーし、その中のパスワード、データベースに接続するためのパスワードを編集した後（およびルートがない場合はログインします） 
 cp /var/www/playsms/config-dist.php /var/www/playsms/config.php vi /var/www/playsms/config.php $core_config['db']['pass'] = ' '; // database password
自動的に開始するようにプログラムをセットアップします。 
 cd /usr/local/src/playsms-0.9.7.1/bin cp playsmsd playsmsd.php playsmsd_start /usr/local/bin/ cp playsms /etc/default/
/etc/rc.d/rc.localで、次の行を追加します-/ usr / local / bin / playsmsd_start。 PlaySMSコントロールパネルは、htppで利用できます：//ip.address.your.server/playsms。 ログインadmin \ admin。

/ etc / default / playmsmsに正しいパスが指定されていることを確認してください 
 PLAYSMS_PATH=/var/www/html/playsms
