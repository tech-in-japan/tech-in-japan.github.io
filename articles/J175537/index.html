Visual Studio 2012 Update 2がリリースされました

以前に、Visual Studioチームが開発者ツールの準備プロセスと更新頻度を修正したことが報告されました。 修正に加えて、新しい機能が四半期ごとの更新に含まれるようになりました。 この記事では、Visual Studio Update 2に含まれる技術革新の一覧を示します。



Update 2に含まれる主な革新は、次の分野に影響を及ぼします。

  • 柔軟な計画-開発チームは、現在のニーズに適した柔軟な計画とプロジェクト管理の機会をさらに獲得しました。
  • 品質保証-テストスイートを管理する機能の強化。
  • ビジネス向けアプリケーション開発-開発者は、SharePoint、Web、Office、およびWindows用のビジネスアプリケーションをより効率的に作成できました。
  • プログラミング環境の改善-プログラミング環境での作業を容易にする新機能。


Visual Studio 2012用の個別のサービスパックはリリースされないことに注意してください。 バグ修正はすでにUpdate 2に含まれています。



柔軟な計画



プロジェクトデータの構造化とフィルタリングを容易にする作業項目のタグ。





組織またはチームに固有の新しい列を追加する機能を備えたカスタマイズ可能なかんばんボード。 提供されるかんばんボードのセットアップには、New、Approved、Commited、およびDoneの列が含まれていますが、チームが状態を示すために追加のフィールドを必要とする場合、これは設定を通じて簡単に行えます。





コードの表示と変更に関連するTFS Webインターフェイスの改善。





品質保証





Team Foundation Server Webインターフェイスを介してテストリストを管理します。 Microsoft Test Managerの機能に加えて、Webインターフェイスを使用してテストリストを管理し、テストリストのステップを編集できるようになりました。 これにより、たとえば、Test Professionalをインストールしなくても、テスター、開発者、マネージャーの間で既存のテストについて話し合い、テストの情報を更新できます。





Webインターフェースでのテスト計画の実装 。 TFSユーザーは、Microsoft Test Managerを使用せずにブラウザーのみを使用してテストリストを「実行」できるようになりました。





ハンドラーとユニットテスト用のJscript / HTML5インターフェイスの応答性プロファイラー。 ユーザーコードだけでなく、単体テスト、および前処理スクリプト、CSSコード、HTML5に関連するコンポーネントのパフォーマンスを評価できる機会。 レポートは、コールがどのように流れ、サブシステムとメインコード間で負荷が分散されるか、およびこれがアプリケーションの全体的なパフォーマンスと応答性にどのように影響するかを示します。





Windows Phoneアプリケーションの単体テスト。 これで、C＃でWindows Phone 8.0の単体テストを開発できます。 テストは、電話機またはエミュレータにユニットテストアプリケーションをインストールした後、エンドデバイスで実行されます。







ビジネスアプリケーション開発



HTMLテクノロジーを使用して、LightSwitchアプリケーションをWebサイトとして公開する機能。 LightSwitchは、データ構造の宣言宣言によりビジネスアプリケーションを迅速に作成するためのツールであり、データ入力や登録アプリケーションなどの基本的なシナリオでコーディングの労力を大幅に削減できます。 最近まで、LightSwitchアプリケーションはSilverlightのみで動作していました。 これで、HTML5テクノロジーを使用してアプリケーションを公開できます。 この機能により、開発者はLightSwitchアプリケーション起動プラットフォームを選択する際の柔軟性が高まります。





BlendでのWPF、Silverlight、およびSketchFlowのサポート。 現在Visual Studio 2012に含まれているBlendは、Windowsストア、Windows Phone、WPF、およびSilverlightのアプリケーション開発をサポートしています。 Update 2では、さまざまなバージョンのWPF（3.5 / 4.0 / 4.5）、Silverlight 4および5を使用できます。





SharePointユーザーインターフェイスをテストする機能15.インターフェイステスト（コード化されたUIテスト）の形式でアクションを記録および再生して、インターフェイスを制御し、検証を検証し、Sharepoint 2013アプリケーションのルーチンを繰り返す機能を追加しました。



プログラミングの機能強化



プレミアムフェイクおよびスタブテクノロジーのサポート。 Fakes and Stubsテクノロジーは、アプリケーションコードのより良いテストのために、単体テストの分離の問題を効果的に解決できます。 Visual Studio 2012 Ultimateのユーザーに高く評価されていたこれらの機能は、Premium Editionで利用できるようになりました。



「ブルー」テーマ。 環境の「ライト」および「ダーク」テーマに加えて、「ブルー」テーマが追加されました。これは、Visual Studio 2010で見ることができるテーマに非常に近いものです。





XAMLデザイナーのパフォーマンスが向上しました。 エラーが修正され、Visual Studio 2012のXAMLデザイナーのユーザーの多くの要望が考慮され、特に、大きなXAMLファイルと大きなプロジェクトを読み込む場合のパフォーマンスパラメーターが改善されました。



デバッガーとコードマップテクノロジーの統合。 コードマップを使用したコード依存関係の視覚化は、Update 1で登場しました。Update2のリリースでは、コードのデバッグ中にこのツールを使用するための追加シナリオが登場しました。





WebアプリケーションおよびASP.NETの分野における革新。 新機能には、単一ページのWebサイトテンプレート、Facebookアプリケーションテンプレート、およびWebサイトの公開のサポートが含まれます。 jQuery、Knockout、およびその他のパッケージのサポートされているバージョンが更新され、SignalRライブラリのサポートが含まれています。 IntelliSence拡張機能が追加されました。
