OpenOfficeには未来がありますか？

Apache Software FoundationはOpenOfficeを高レベルのプロジェクトにしましたが、これは本当に競争力を高めるのに十分ですか？ また、OpenOfficeは独立したオープンソースプロジェクトのままにしておくべきでしょうか？ 以下の記事でこれらの質問に答えようとします。



数日前、 Apache Software Foundation（ASF）は、 Apache OpenOfficeが Apache Incubatorの開発段階から抜け出し、トップレベルプロジェクト（TPL）になったことを発表しました。 これはもちろん素晴らしいことですが、特にLibreOfficeがほぼ同じコアを使用していることを考えると、OpenOfficeを本当に競争力のあるものにするのに十分ですか？ 「OpenOfficeの無料水泳へのリリースは、技術的および公共的な問題の両方において、プロジェクトの実行可能性を公式に認めたものです。 「Apache Path」とその方法（すべてのプロセスの完全な透明性を備えた各ソリューションの公開ディスカッションなど）により、プロジェクトは新しいボランティアを引き付けるだけでなく、Apache OpenOfficeの安定した開発を保証するプロジェクトマネージャー委員会全体を選出することができました」 OpenOffice社長のアンドレアペシェッティ。 正直なところ、この声明の論理はあまり明確ではありません。



いいえ、本当に、なぜ突然ですか？ はい、OpenOfficeは重要なオープンソースプロジェクトでしたが、ここでのキーワードは「だった」です。 90年代にStar Divisionによって作成されたとき（かつてはStar Officeと呼ばれていました）、それは本当に有望なオープンソースオフィススイートでした。 その後、 1999年にSunによってStar Divisionが発表された後、長年OpenOfficeに移行したことにより、このプロジェクトは最も重要なオフィススイートの1つになりました。



しかし、Sunはプロジェクトを無視し、2009年にOracleがSunを買収した後、以前の仕事にあまり満足していない元のOpenOfficeの開発者は、プロジェクトのソースコードを使用して新しい製品LibreOfficeを作成し始めました。 彼らは喜んでOracleと協力することに同意するだろう。OracleだけがLibreOfficeや、やがて明らかになったOpenOfficeとは何の関係も持ちたくなかった。 2011年5月末、OracleはOpenOfficeを放棄しました。



その間、LibreOfficeはこれまでになく良くなっています。 Ubuntuなどの主要なLinux開発者は、LibreOfficeを主要なオフィススイートとして認識しています。 IntelからFree Software Foundationまで、会社の友人もOfficeをサポートしていました。



IBMは当初OpenOfficeを全力でサポートしていましたが、現在では自社製品のSymphonyに切り替えています。 ただし、IBMソフトウェアアーキテクトのロブウィアは次のように述べています。「IBMはSymphonyに投資したリソースを取得し、Apache OpenOfficeに投資しました。 また、ハンブルクの開発チームを雇いました。彼は、SunでOpenOfficeに取り組んでおり、したがって、適切なコードを扱った10年の経験があります。 彼らは昨年10月からApacheプロジェクトに取り組んでおり、Symphonyの同僚も働いています。 このプロジェクトには、プログラマー、インターフェイス開発者、その他の専門家など、無形の多くのリソースを投資しています。」 実際、OpenOffice Apacheはまだ改善中ですが、ほとんどの改善はLibreOfficeデータベースによるものです。 しかし、なぜOpenOfficeで作業を続けるのでしょうか？ 全く分かりません



次のバージョンのOpenOfficeとLibreOfficeの計画をよく見ると、それらはまったく同じであることがわかります。最高のOffice 2007-2013、OpenXML形式のサポート、タブレットとクラウドのバージョンです。



一方、長年の抵抗の後、Microsoftはついに、Office 2013を介してファイルを読み取り、変更、保存する機能を備えたOpen Document Format（ODF）1.2のサポートを開始しました。 、OpenOfficeおよびLibreOfficeを完全にサポートしています。 Microsoftが成功すれば、LibreOfficeとOpenOfficeはエンドユーザーにとってより魅力的なものになるでしょう。



控えめな提案が1つあります。時間と労力の重複作業を無駄にする代わりに、ApacheはLibreOfficeの親組織であるDocument Foundationと合併し、単一のオフィススイートで共同作業を開始する必要があります。 OpenOfficeとLibreOfficeのすべての開発者とユーザーにとってより良いでしょうか？ そうなると思います。 あなたはどうですか？
