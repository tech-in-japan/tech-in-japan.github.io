Alien Swarmのマッピング機能

この記事では、 Alien Swarmのレベルを作成するときに注意する必要がある複雑さについて説明します。 この記事は、Source SDKのHammerを使用してマップを作成する方法を既に知っているか、このエディターでプリミティブオブジェクトを作成する方法を知っている人に適しています。



Alien Swarm SDKは、ライブラリ-> [ツール]メニューのSteamから無料でダウンロードできます。 ダウンロードしてインストールしたら、Hammer World Editorを起動して開始できます。



SDKは次の方法でインストールされます。

C：\ Program Files（x86）\ Steam \ SteamApps \ common \ alien swarm \

後続のすべてのパスは、このフォルダーまたは共通フォルダーに相対的です



サンプルマップはどこにありますか？



-SDKにはいくつかの例が付属しています：alien swarm \ sdk_content \ mapsrc

注意！ 公式マップのソースASI-Jac1-LandingBay_01など これらのエンティティの一部はインスタンスフォルダー内の個別のVMFファイルにあるため、すべてのエンティティは表示されません。

-プログラムGCFScapeおよびBSPSourceを使用して、任意のカードを逆コンパイルします。 そして、物事がそこで行われている方法を参照してください。

-公式キャンペーンを逆コンパイルして、最初に調査することをお勧めします。 Lana's Escape、Tears of Tarnor、City17、Paranoia、Reduction、Exterminationによるカスタムキャンペーンも非常に便利です。



便利な作業のために環境を構成する方法は？



アドオン\キャンペーン\マップには、ディスク上に別のフォルダーが必要です。 キャンペーンがBug Huntと呼ばれているとします。

-フォルダーDの作成：\ BugHunt

-サブフォルダーとファイルの基本構造を作成します：マップ、マテリアル、リソース...「alien swarm \ swarm \ addons \ ExampleAddon」を参照

注意！ デフォルトのファイル名とフォルダー名CustomCampaignCredits.txt、ExampleCampaign.txt、example_map_1.txtを使用しないでください。競合が発生します。 これらのファイルの名前をすぐにBughuntCredits.txt、リソース\キャンペーン\ bughunt.txt、リソース\概要\ bughut01.txtなどに変更します。 addoninfo.txtは名前を変更しません。

-alien swarm \ swarm \ gameinfo.txtというファイルを開き、フォルダーにパスを追加します。 

"SearchPaths" { "Game" "|gameinfo_path|." "Game" "swarm_base" "Game" "platform" "Game" "D:\BugHunt" }




リストの最後にパスを追加すると、パスのすべてのサブフォルダーとファイルが「alien swarm \ swarm」フォルダーにマウントされているようになります。 キャンペーンはゲーム内に表示され、テクスチャとサウンドはハンマーとゲーム内に表示されます。 キャンペーンをテストする場合、毎回VPKを作成する必要はありません。 ユーザーは、ファイル、テクスチャ、モデル、スクリプトでゲームフォルダーを汚染する必要がなくなりました。 すべてをフォルダに個別に保管します。



VMFファイルとは何ですか？



Hammerは、マップソースをVMF形式で保存します。 これは、すべてのエンティティ、ブラシなどの説明を含むテキストファイルです。 あなたのカードに。



VMFおよびBSPファイルとは何ですか？



Alien Swarmでは、すべてがキャンペーンとしてカウントされます。 カードが1枚でも、それはキャンペーンです。 絶滅など、キャンペーンに名前を付けます。 VMFファイルに「extermination01road」または単に「extermination01」などの名前を付けます。 快適であれば、ファイル名にアンダースコア「_」を追加できます。 この名前を使用すると、GoogleまたはYandexの検索エンジンを使用して、インターネットでキャンペーンを見つけてダウンロードできるプレーヤーを許可します。 実際、プレーヤーが専用サーバーのリストを見ると、キャンペーンではなく、BSPファイルの名前のみが表示されます。 そして、myfirrstlevel.bspのようなものがある場合、彼はインターネット上でそのようなキャンペーンを見つける可能性は低いです。 そして、「エイリアンの群れ駆除キャンペーン」の検索で見つける可能性があります。 また、検索エンジンにmoddb swarmarmory fpsbananaという単語を追加することをお勧めします。これらはSwormのアドオンが投稿される3つのサイトです。

注意！ コンパイル後、BSPファイルの名前を変更しないでください。ファイルは無効になります。



最も単純なレベルの\マップを作成する方法は？



あなたがする必要があります：

-目に見えるテクスチャでブラシを作成します。 これは、プレイヤーが歩く「地面」になります。

-ブラシの上部にinfo_player_startエンティティを作成します。 ウィキは、各カードに8つのそのようなエンティティを置くことをお勧めします。 ただし、ゲームのプレイヤーは最大5人です。

-すべてのサイズでマップ全体よりもはるかに大きい、テクスチャなしのブラシを作成します。 彼は地図を包み込み、光が世界を越えないようにします。

-このブラシを空にします。任意の2Dウィンドウで右クリックします。

-Ctrl + Tまたは「Tie to Entity」を右クリックして、クラス名のフィールドにfunc_brushと入力し、Enterを押します。 ブラシクラスをfunc_brushに変更しました。

-次に、Name：structure_sealおよびEnterフィールドに名前を付けます。



これらの手順に従うと、マップをF9にコンパイルしてゲームで実行できます。 カードに光源がない場合、カード内のすべてが100％明るく、完全に明るくなります。



Alien Swarmのカードには何が表示されますか？



2つのエンティティ：logic_autoおよびenv_tonemap_controller。 SDKに付属している既製のカードまたはサンプルレベルからコピーすることをお勧めします。 パラメーターをいじってみることができます。 それらはブルームの量に影響します。 これらのエンティティがカードに含まれていない場合、色は非常に明るく、目を引きます。



コンパイル方法



マップをコンパイルするには、対応するボタンをクリックして、通常モードからエキスパートモードにすぐに切り替えることをお勧めします。

ここには、さらにオプションとオプションがあります。 コンパイル後にゲームでカードが自動的に開かないという奇妙な動作を避けます。 マップの開発中に高速オプションを使用し、影と光の美しさを確認する必要がある場合にのみ、フルを使用します。 カードを解放するには、当然ながら[完全]オプションを使用する必要があります。

VPKとは何ですか？



VPKは、Valveによって作成されたファイル形式です。 Left 4 Dead、Alien Swarmなどのゲーム用にユーザーが作成したアドオン この形式のファイルによって配布されます。 Valve自身は、ゲームのメディアデータ（テクスチャとマテリアル、スクリプトなど）を保存するためにそれを使用します。

VPKは、内部にファイルとフォルダーがあるアーカイブと考えてください。 ゲームの開始時に、メモリ内のすべてのファイルをVPKファイルから「alien swarm \ swarm」フォルダーに抽出し、その中のすべてを上書きすると考えてください。 したがって、swarmフォルダーに既に存在する何らかの種類のファイルをVPKに配置すると、ゲームは元のファイルではなくファイルを使用します。 したがって、VPKファイルを使用してすべてのゲームファイルを変更できます。



VPKファイルをどうやって開くか？



GCFScapeプログラムを使用します。



VPKファイルを作成する方法は？



このファイルを作成するには、フォルダーをVPK.EXEのショートカットにドラッグします。 ファイル「alien swarm \ bin \ vpk.exe」のショートカットを作成します。 このショートカットにフォルダーをドラッグすると、VPKファイルがドラッグされたフォルダーの横に表示されます。
