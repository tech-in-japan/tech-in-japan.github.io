VMware Virtual Storage Applianceの概要

新しいvSphere 5機能に関する私の投稿に対するコメントで、同僚のOmnimodは Virtual Storage Appliance（VSA）について懐疑的でした 。 正直に言うと、私自身はこの新しい機能に非常に興味があったので、経験豊富な友人の懐疑論をどのように正当化するかを決めました。



それでは、VSAを簡単に見てみましょう。



VSAの主な目標は、ローカルのESXiホストを使用してSMB企業にパブリックNFSストレージの全機能を提供することです。 つまり、リモートオフィスでVSAを使用し、2〜3台のESXiホストを使用すると、NASやSANを購入する必要はありません。



これは、3つのホストにインストールされたときのVSAの外観です。







2ホストシナリオでは、vCenterは3番目のVSAノードをエミュレートします。これは、ノードの1つがクラッシュした場合にクォーラムの役割を果たします。 これにより、理論的には異なるノードで実行されている仮想マシンの2つのコピーを取得できるスプリットブレインの問題を回避でき、残念ながら相互の接触が失われます。







VSAでパブリックストレージを使用すると、HAおよびDRSクラスター、vMotion、Fault Toleranceの形で関連するすべてのボーナスを楽しむことができます。 はい、vSphere 5では、覚えているとおり、NFSデータストアでもFTがサポートされています。



データ保護は、ローカルドライブ上のRAID-10とVSAストレージクラスターのノード間のRAID-1を使用して提供されます。 VSAバージョン1.0では、これまで3つのホストがサポートされていますが、データセンター内の他のすべてのホストはVSAストレージを簡単に使用できます。 単一のvCenterで、使用できるVSAクラスターは1つだけです。



VSAのインストールを開始する前に、ディスク容量と適切な数のハードドライブに対する要求を慎重に評価する必要があります。 最終結果では、合計ディスク領域の使用可能な使用可能領域の25％を取得することに注意してください。 次のVSAリリースで、ローカルRAIDタイプを選択する機会が与えられれば、素晴らしいと思います。 ディスクシステムのパフォーマンスは必ずしも重要ではないため、場合によっては、オーバーヘッドを削減するためにローカルでRAID-5を使用することをお勧めします。 最新のデータによると、VSAは任意のローカルRAIDで動作しますが、サポートはRAID-10にのみ適用されます。 インストールと構成を完了すると、ディスクを追加したり、NFSデータストアのサイズを変更したりできなくなります。



インストール手順は非常に簡単です。 最初にVSA Managerをインストールします。VSAManagerは後でvSphere Clientにプラグインをインストールします。 vCenterに接続するときは、VSAウィザードを起動するだけで、VSA要件を満たすESXiホストのリストを決定できます。 その後、選択した各ホストに仮想マシンVSAがインストールされ、ウィザードがVSAノード間のvMotionを簡単かつ迅速に上げ、HAとDRSを有効にします。



各VSAノードに2つのディスクボリュームが作成されます。1つ目はNFSデータストアの下で使用され、2つ目は1つの隣接ノードからのNFSデータストアレプリカを格納するために使用されます。 原則として、最初の写真はこれをよく示しています。 VSAクラスターのノードにインストールされたVSA仮想マシンは、次の機能を実行します。

  • すべてのESXiホストへのNFSストレージのプレゼンテーション
  • NFSリポジトリのすべてのレプリカを同期します
  • このNFSストレージのメインコピーが置かれているホストがクラッシュした場合に、NFSデータストアレプリカの1つをオンラインモードに転送します。


インストールが完了すると、単一障害点のない、完全にフォールトトレラントでアクセス可能なストレージシステムが得られます。



VSAを使用すると、いくつかのレベルで一度に保護されます。

  • ローカルディスクの障害は、ESXiホストおよび仮想マシンの動作に影響しません
  • ESXiホストの障害は、NFSデータストアにある仮想マシンには影響しません。 VSAクラスターの別のノードは、障害が発生したNFSデータストアのレプリカをそのIPアドレスに即座に接続します。 故障したホストの仮想マシンは、HAを使用して自動的に再起動します。
  • さらに進んで、非常に重要な仮想マシンのフォールトトレランスを有効にすることができます。 この場合、1つのESXiホストの障害は、仮想マシンとそのユーザーに対して完全に透過的になります。


VMwareは、VSAとHAが適切に構成された仮想マシンの99.9％の可用性を正式に発表します。



残念ながら、技術的な詳細を見つけることはできませんでした。vSphere5のデモバージョンは、これまでのところエリートのみが利用できます。 そのため、私は財務面に切り替えて、小規模またはリモートオフィス向けのこのようなソリューションのおおよそのコストを計算しようとしました。 私はGoogleに与えた価格に依存しており、それは私の地域（イタリア）に適しています。 そのため、価格、割引の可能性、小規模オフィスでのvSphereのアイデアとは大きく異なる可能性があります。



理想的なリモートオフィスでは、HP Proliant DL380 G7サーバーが3台あり、それぞれに6x600GB 10Kディスクを配置します。 純粋な形では、10.8 TBが得られます。 使用可能なスペースのすべてのRAIDを考えると、それぞれ900 GBの3つのNFSストレージに2.7 TBがあります。 はい、これはディスク容量の大きな損失であるように思えます。 そのため、すぐにローカルRAID-5について考えました。

しかし一方で、完全に冗長なパブリックストレージの1ギガバイトの価格から始めましょう。



ここで、必要なディスクのコストが368ユーロであるため、18枚のディスクで6624ユーロになることがわかりました。 VSAの宣言値は5595ドルで、ヨーロッパでも同じ数字ですがユーロであると思います。 合計は6624 + 5995 = 12619ユーロです。 この数値を2700 GBで割ると、 ギガバイトあたり4.67ユーロになります。 高価なのか安いのかを評価するのは難しいので、競合するVSA製品の名前と会社のGBあたりのコストをコメントに残していただければ幸いです。



より安価な製品があり、一般にOpenfilerやFreeNASなどの無料の製品があると思います。 ただし、VSAに有利ないくつかの非常に深刻な議論をもたらしたいと思います。

  1. インストールプロセスが非常に簡単であるため、エラーのリスクが低くなります。 プロセス全体がすでに使い慣れたインターフェイスにあり、ウィザードはすべてが正しく選択され、すべてのステップが正しい順序で完了することを確認します。 個人的にはOpenfilerの経験しかなく、そのインターフェイスにすぐには慣れなかったと言えます。そのため、すべてを2回やり直す必要がありました。
  2. vCenterとの統合-すべての監視と制御が1か所に集中します。 そして確かに、VSA用の事前設定されたアラームがすでにあります。
  3. 1つのベンダーサポートサービス-HP Virtual ConnectとvSphereの両方が原因で問題が発生する可能性のあるケースがすでにいくつかありました。 両方のサポートサービスに連絡しなければならず、それが彼らの問題であることを証明しなければなりませんでした。 デフォルトでは、サポートサービスは他のすべての責任を単純に責めようとしました。 あなたも同様の問題に直面したと思います。 VSAの場合、すべてがはるかに単純です。常に1人の担当者がいます。 そして、Openfilerで実行されているiSCSIリポジトリが突然立ち往生したとき、私は基本的に何をすべきか全く知りませんでした。なぜなら、それを使った経験はすべてセットアップに限定されていたからです。 したがって、あるベンダーのサポートサービスは非常に大きな利点です。
  4. WindowsサーバーをiSCSIストレージまたは同じOpenfilerとして使用する場合は、ホストレベルのフォールトトレランスに注意する必要があります。つまり、同期レプリケーション、自動切り替えを提供します。 原則としてすべてが可能ですが、繰り返しますが、より多くの時間、知識、サポートが必要です。 VSAの場合、これはすべて含まれています。
  5. NFSのすべての利点（自動シンプロビジョニング、バックアップ用の仮想マシンファイルへの簡単なアクセス）を利用できます。
  6. 2011年第3四半期に、VMwareはVMware vSphere 5 Essentials PlusおよびvSphere Storage Applianceを7.995ドルで購入する期間限定のオファーを提示します。これにより、VMware vSphere Storage Applianceのコストの約40％が得られます。


現時点でのVSAの唯一の欠点は、仮想マシンを保護するためにディスクスペースの大幅な損失を考慮することですが、前述したように、ギガバイトのコストに注目する必要があります。費やす時間、バックアップシステム、サポートコストなどで構成されます。
