Microsoft DevOpsツール

エントリー


現在、ソフトウェアの開発では、DevOps手法の使用が非常に人気があります。 この記事では、DevOpsの基本的なアプローチについて説明し、これらのアプローチを.NETの世界に実装するのに役立つMicrosoftのツールについて説明します。



DevOpsとは何ですか？


DevOps-シンプルなアイデアを実装する一連のテクニック。タイトルに反映されています。 DevOpsという名前は、開発と運用という言葉の合併の結果であり、最も重要なことは、開発チームとソフトウェアの運用を担当するチーム（運用）の相互作用です。 DevOpsは、開発と運用の密接な相互作用により、ソフトウェア製品の新しいバージョンのリリースがより迅速に、より少ないエラーで可能になると主張しています。







DevOpsメソドロジは非常に新しいものです。2007年に登場し、ITプロフェッショナルのコミュニティで開発され、実用的な方向性が決定されました。



いつ、どのようにDevOpsを使用しますか？


DevOpsは、開発されたインフラストラクチャを備えた複雑なプロジェクトの開発に使用されます。 DevOpsアプローチの使用は、次の場合に役立ちます。

  • 複雑なインストールプロセスを必要とする洗練されたアプリケーションを開発します。
  • 多くのコンピューターの動作をサポートおよび保証する必要があります。
  • 頻繁なリリースが必要です。
  • アプリケーションのパフォーマンスの追跡と迅速なトラブルシューティングが重要です。




そのような状況では、すべてが時間に依存します。 本番環境でのアプリケーションのインストール時間を短縮し、インストールされたアプリケーションの操作に関するフィードバックを開発チームに迅速に配信することが重要です。







開発および運用では、自動化ツールを使用して、開発および運用チームの相互作用を促進します。 自動化できます：

  • リリース管理。
  • インストールされたアプリケーションを監視し、インストールされたアプリケーションの操作について開発チームにフィードバックを提供します。




次に、マイクロソフトが提供する自動化ツールについて説明します。



Microsoftリリース管理によるリリース管理（開発から運用まで）


複雑なアプリケーションには、複雑な展開プロセスが必要です。 このようなプロセスには、さまざまなコンピューターで実行される多くのステップが含まれる場合があります。 開発、品質管理、活用など、さまざまなチームの人々が展開に参加できます。 手動展開を実行すると、通信エラーと問題が発生します。



このような状況では、ソリューションは、展開プロセスに関与するすべてのユーザーに対して単一の自動化されたシステムを使用することです。 そして、Microsoftリリース管理はまさにそのようなシステムです。







Microsoftリリース管理機能


Microsoft Release Managementの主な機能は、運用環境での一連のテスト環境を通じてアプリケーションの展開を自動化することです。

システムの主なオブジェクトはリリーステンプレートです。これは、リリースを展開するために実行する必要がある手順の説明です。 リリーステンプレートは、WPFクライアントのグラフィカルインターフェイスを介して作成されます。 リリーステンプレートを作成するには、仮想マシンのセットを定義し、それぞれに展開アクションのセットを追加する必要があります。 これらのアクションセットは、仮想マシン間でコピーできます。







各環境へのインストールは承認または拒否できます。 各インストール手順に責任を割り当てることができます。







しかし、Microsoft Release Managementでできることはそれだけではありません。 システムは以下を提供します。

  • インストール手順の大規模なセット。 Microsoft Release Managementは、ファイル、レジストリ、IIS、SQLスクリプトの実行、Reporting Services、Windowsサービスのインストール、Windows Azure仮想マシンの操作、Microsoft Test Managerを使用したテストの実行ができます。
  • 構成ファイルを管理します。 構成ファイルをパラメーター化して、各環境に必要なパラメーター値に置き換えることができます。
  • インストールをロールバックする機能。 インストールをロールバックする一連のアクションを決定できます。 インストールプロセス中に障害が発生すると、ロールバックが自動的に開始されます。
  • TFSとの統合。 TFSでのビルドが完了すると、リリースを自動的に開始できます。




実行中のアプリケーションに関する情報の監視と提供（操作から開発）


次に、開発と運用の間の別のタイプの相互作用、つまり実行中のアプリケーションに関する情報の開発チームへの転送について説明します。 この情報には、エラーメッセージと、アプリケーションの操作に関するユーザーのフィードバックが含まれている場合があります。 マイクロソフトは、このようなプロセスを自動化するためのツールをいくつか提供しています。



Microsoft System Center 2012オペレーションマネージャー


Microsoft Operations Managerは、Microsoft System Center製品ラインの一部です。 Microsoft Operations Managerを使用すると、単一のコンソールから複数のアプリケーションを監視し、イベントに関する情報を開発者に簡単に送信できます。



このようにして、.NET、Webページ、およびWebサービスで開発されたWebアプリケーションの動作を監視できます。 また、Operations Managerの最新バージョンでは、Windowsサービスの監視が可能です。 クライアント側とサーバーの両方で、アプリケーションのパフォーマンスのクラッシュと問題を追跡できます。



ただし、これはOperations Managerの最も興味深いスキルではありません。 監視機能に加えて、アプリケーションにはTFSとの素晴らしい統合があります。 クラッシュとパフォーマンスの問題について収集した情報は、ワンクリックで簡単に開発者に送信できます。







イベント情報を開発者に送信すると、IntelliTraceファイルの形式でデバッグに必要なデータを含むワークアイテムがTFSに自動的に作成されます。







ワークアイテムで提供される情報を使用して、開発者はすぐに編集を開始できます。



Microsoft監視エージェント


Microsoft Operations Managerは、便利でありながら重量のあるソリューションです。 Microsoft Monitoring Agentにより、簡単な代替手段が提供されます。 このツールを使用すると、.NETで開発されたWebアプリケーションの動作をローカルで監視できます。







Microsoft Monitoring Agentは、ローカルにインストールされたアプリケーションでのみデータを収集します。収集されたデータはIntelliTraceファイルに保存され、開発者に手動で送信できます。 PowerShellスクリプトを使用して、Microsoft Monitoring Agentによって管理されます。



アプリケーションの洞察


Application Insightsは、新しい有望な監視ツールです。 Application Insightsがプレビューバージョンで利用可能になりました。 このアプリケーションは、Visual Studio Onlineの一部として利用可能です。 Application Insightsは以下を提供します。



  1. Webアプリケーションの可用性と応答時間を監視します。 これを行うには、アプリケーションのURLを指定するか、Visual Studio Webテストを提供するだけです。
  2. エラーとパフォーマンスの監視。 このケースは、前述のオプションに似ています。 Microsoft Monitoring Agentは、監視対象アプリケーションが実行されているコンピューターにインストールされ、Application Insightsと通信します。 その後、収集された情報はApplication Insightsで利用可能になります。 TFSとの統合はないため、IntelliTraceファイルを開発者に手動で送信する必要があります。
  3. ユーザーアクションの分析。 Application Insightsは、アプリケーションでイベントが発生する頻度（画面の読み込みなど）を追跡できます。 このオプションは、次の種類のアプリケーションで使用できます。


  • Windows Phone 8。
  • Windowsストア
  • Webサービス。
  • Webページ。








Microsoft Feedback Clientを使用して、アプリケーションのフィードバックを自動化する


開発者は、Microsoftフィードバッククライアントを使用して、実行中のアプリケーションに関するフィードバックをユーザーに要求し、受け取ることができます。 開発者は、Team Web Accessを通じてフィードバックのリクエストを作成し、電子メールで送信します。







アプリケーションユーザーは要求を受信し、ローカルにインストールされたMicrosoftフィードバッククライアントの助けを借りて、アプリケーションとの相互作用に関するフィードバックを書き込みます。 レビューには、ビデオ、スクリーンショット、音声、テキストコメントが含まれる場合があります。 作成されたフィードバックは、Microsoftフィードバッククライアントインターフェイスを介して開発者に送信されます。









受信したフィードバックはTFSに保存され、ワー​​クアイテムに添付できます。
