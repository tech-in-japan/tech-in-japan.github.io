リモートデバイスアクセス

すべての人に良い一日を！ RDA（リモートデバイスアクセス）について話したい

彼女のデバイスのいくつかを活用してください。 これは、開発者と、別の新製品を試してみたい人の両方に役立ちます。

猫に興味のある人は誰でも聞いてください。



サービスの開始方法：



まず、 nokiaフォーラムに登録する必要があります。

次に、入力したapu.ndhub.netにアクセスします。ブラウザーが要件を満たしている場合、このページが表示されます。







次に、必要なデバイスを選択し、必要な時間だけ予約して（最大予約時間は4時間、1日あたりの最大電話使用時間は8時間）、スタートボタンを押します。 ランチャーがコンピューターにダウンロードされます。ランチャーは、コンピューターがサービスを使用するための要件を満たしているかどうかを確認し、RDAプログラム自体と不足している要素をダウンロードします。 ダウンロードが完了すると、プログラムが開始され、次のような画面が表示されます。







すべてが機能します！



また、使用したいデバイスが現時点では利用できない可能性があります。その場合、後でデバイスを予約してこれを行うには、予約ページにアクセスしてください。





ページの上部にはタイムラインがあります。 赤でマークされたブロック-すでに予約された時間、灰色-空き時間、選択されたブロックは緑でマークされます。 時間内のそのようなブロックの1つは15分です。 現在利用できない赤色で強調表示されたデバイス。 他の人のセッションに隣接するブロックをマークしないでください。 セッションを予約するには、必要なブロックの数をマークします。



技術的な部分について詳しく説明します。



仕事には、次のものが必要です。



  • JavaScript™ブラウザー
  • Java Runtime Environment 5.0以降。
  • インターネット:)。




主な機能：



  • デバイスにアプリケーションをインストールして実行します。
  • デバイスとユーザーのコンピューター間のファイル交換。
  • デバイスを再起動します
  • これをサポートするデバイスで画面の向きを変更します。
  • デバイスから送信される画像の色深度、解像度を変更します。
  • デバイスからデバッグ情報を取得します。

ネットワーク：



  • デバイスは着信コールとメッセージを受信できます（番号が示されています）。
  • 発信通話、メッセージはブロックされます。
  • WLANおよびElisaインターネットポイントが利用可能です。
  • GPSはテストできません。




制限事項：



最後に、悲しいことですが、それほどではありません。

  • 音声サポートなし。
  • デバイスは常に利用できるとは限りません。
