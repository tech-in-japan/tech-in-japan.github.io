NieRを使用したゲームのUIデザイン：オートマトンを例として









こんにちは、私の名前は木島久義です。私はNieR：AutomataのUIおよびMechデザイナーです。 ゲームのUI（ユーザーインターフェイス）はしばしば見過ごされてしまうので、ファンがNieR：Automata開発日記を見つけて、私の仕事について尋ねたと聞いてうれしいです！



NieR：AutomataのディレクターであるYoko Taroがゲームに取り組むとき、彼は常に自分が望むものを正確に知っており、UIも例外ではありません。 NieR：AutomataのUIでの私の仕事と、ヨーコさんがくれた仕事についてお話したいと思います。





最初からやり直しましょう：UIアーティストが行うこと。 最近のゲームには、重要な情報を伝える多くのヘルスインジケータ、ダイアログボックス、さまざまなメニュー画面、ポップアップがあります。 それらはすべて、私と同じUIアーティストによって作成されています。



ユーザーインターフェイスを設計するとき、私のワークフローは次のようになります。



  1. UIのビジュアルデザインコンセプトを開発する
  2. ディレクターとゲームデザイナーの要件を考慮して、いくつかのメニュー画面と画面要素のデザインを作成します
  3. これらのUI要素のデータを準備し、それらをゲームに挿入できるようにプログラマに渡します
  4. ゲームのUIを確認し、行う必要がある変更と改善を見つけます。 既製の高品質UIが得られるまで繰り返します！


この投稿では、主に項目1と2 （メニュー画面の コンセプトとデザイン）についてお話したいと思います 。



UIコンセプトデザイン：SFとファンタジーのバランス



NieR：Automataの世界は空想科学に引き寄せられますが、シリーズの前のゲーム（NIER Replicant / Gestalt）にはファンタジーの美学がありました。 UI NieR：Automataコンセプトの開発では、サイエンスフィクションの要素が最初のゲームのファンタジースタイルの自然な拡張のように見えるようにしました。



仕事のために、視覚的なランドマークがいくつかありました。前のゲームのUI。 おなじみのサイエンスフィクション; 2Bのヒロインと、贅沢で枯れた世界の一般的なアイデア。 これらすべてのスタイルを組み合わせて、UI NieRの基本コンセプトを取得しました。オートマタ： 体系的で不毛であると同時に美しいデザインです。 その瞬間、このスタイルを伝える最良の方法は、装飾を避け、清潔でエレガントでフラットなデザインの作成に集中することだと確信していました。 しかし、私はすぐにこの考えに固執しすぎて、完成したデザインが乾燥したものになることがわかった。



コンセプトの「体系的でクリーンな」部分に、 「楽譜」の軽い動機を追加することにしました。 これは、デザインの「ファンタジー」オーラを強調するのに役立ちました。



「楽譜」の動機について話すとき、高音部記号に滑らかな線を追加することを意味しません。 音符、低音部記号、二重の特徴などの形式からインスピレーションを得て、音楽のフレーズを完成させました。 音楽的な要素を直接追加する代わりに、シンボリズムの助けを借りてそれらをほのめかしたかったのです。 （これは、ゲームの名前のコロンと、ポッドとの対話にも反映されます-ポッド-楽譜にしばしば表示される別のシンボル。）











メニュー画面の上部と下部の装飾は、最初のNIERのメニューの同様の境界線に基づいています。 同様の要素が「デジタル」デザインに「アナログ」感を追加します。 彼らのおかげで、UIコンセプトを開発する過程で、ファンタジーとサイエンスフィクションのバランスを管理できました。



UIコンセプトデザイン：平面から形状へ



このコンセプトは、ほとんどがフラットデザインに基づいています。 それをゲームの環境に少し良く一致させるために、弱いメッシュテクスチャを追加し、エッジで少し歪みと暗くしました。 このメニューのおかげで、まるで凸面スクリーンのモニターに表示されているように見えます。 完成したデザインが完成しました。









このGIFは、画面/レンズ効果を追加する前後のデザインの様子を示しています。



そして、この小さなスキューモフィックタッチの助けを借りて、シンプルなフラットデザインに基づいて、より物理的でリアルなものを作成しました。



*変形とは何ですか？ これは新しいものであり、それ自体の古いバージョンから設計要素を借用しています。 たとえば、2017年のディスケットとしての「名前を付けて保存」アイコン。



また、プレイヤーのフラットスクリーンでUIアニメーションを美しく滑らかにするために、わずかな変更を加えました。乾燥して無色ではなく、少し「ジューシー」に見えるはずです。 この滑らかさはやや弱めに表現されていますが、システムメニューを選択するときに画面上でオブジェクトがどのように移動するかを特に誇りに思っています。





UIコンセプトデザイン：ベージュを基礎として



陽子太郎から受け取った最初のUI仕様では、彼は「 美しく、温かみのあるベージュ 」であると想定されていました。



しかし、言うよりも簡単です。 最初は、冷たくて数値的なデジタル美学の助けを借りて「ウォームベージュ」の感覚を再現する方法を想像することさえできませんでした。



簡単に想像できるもの-シャープでコントラストの高い黒と白のデザイン-から始めて、徐々にそれをより柔らかいベージュにします。 徐々に、私は正しいように見える色になりました。 柔らかく、コントラストの低いベージュの色合いで、明確な配色を見つけて維持するのに多くの作業が必要でした。









私のオリジナルの配色は、最終的なデザインよりもはるかに対照的でした。

UIの配色は、ハッカーのミニゲーム、および世界地図の画面上の3Dモデルに反映されていました。 これにより、ゲームに均一な視覚スタイルを与えることができました。

メニュー画面の設計：利用可能な機能



NieR：Automataでの作業を通して、ヨーコさんは常に、メニュー管理は非常にシンプルで 、真面目なゲームをプレイしていない人でも学習できるようにするべきだと強調してきました。 「 あなたの祖母に彼に満足してもらいたい。」



メニューでは2つの基本的なルールが使用されました。



  • すべてのメニュー機能には、 ジョイスティックとO / Xボタンを使用してアクセスできます 。
  • 画面上のコントロールプロンプトは非常に簡潔である必要があります。通常は、上記の3つのコントロールのみを説明する必要があります。










L1やR1などのトリガー、および前面の他のボタンは、ゲームメニューで非常によく使用されます。 カテゴリ画面と他のナビゲーションを切り替えずに切り替える方法について、頭を悩ませる必要がありました。 単純な制御の要件は複雑なメニューデバイスと互換性がないように思われたため、理解しにくい試験制御オプションが作成されました。 ようこさんと私はよく落書きをします... つまり、彼らはメニューについて話し合い、私たち両方に合った妥協点に到達しようとしました。



その結果、多くのメニューを備えたゲームをプレイすることに慣れている人は、コントロールがスティックとO / Xボタンに限定されない場合、無意識のうちに快適に感じるという結論に達しました。 そのため、メニュー内を移動する場合はオプションであり、画面上のプロンプトではそれらについて何も言われませんが、L1、R1、および三角形/正方形ボタンを使用した隠れた素早い動きがあります。 （ゲームでそれらに気づきましたか？）



ゲームには次のホットボタンがあります。











プレーヤーは、システムメニューのカテゴリの画面間を移動し、ボタンL1とR1を使用して、武器とユニットのデータページをスクロールできます。











ユニットデータのリストから、ユニットの機器を変更し、「三角形」および「正方形」ボタンを使用してアニメーションの再生を制御できます。 右スティックを使用してモデルを回転することもできます。











マップ画面で、右スティックとL2 / R2を使用して、全画面マップにすばやく切り替えることができます。











システムメニューの階層内の任意の場所で[オプション]ボタンをクリックすると、すぐにゲームに戻ることができます。 任意のボタンを押して、メニューを閉じるアニメーションをスキップすることもできます。
