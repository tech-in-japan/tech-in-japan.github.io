パート1：シーン。



まず、平面、光源を作成し、「すべてがはっきりと見えるように」カメラを配置します（これは前のタスクの分析で説明します）。



次に、ターゲットを追加します。キューブを作成し、適切なターゲットのサイズに拡大します。 OX軸に対して垂直になるようにターゲットを移動します（軸の方向は、[シーン]タブの右上から表示できます）。 ターゲットに似たエディターで単純なテクスチャーをすぐにスケッチし（正方形にし）、プロジェクトパネルのマテリアルフォルダーに保存します（このフォルダーを作成した後）：







targetMaterialマテリアルをプロジェクトに再度追加し（Materialsフォルダー内）、新しく追加したテクスチャを（インスペクターパネル内の）Textureフィールドにドラッグします。 マテリアルをターゲットにドラッグします（階層パネルでそのオブジェクトを「ターゲット」と呼びましょう）。



カタパルトレバーの世話をしましょう。 キューブをシーンに追加し、plank1と呼び、「ボード内に」引き伸ばします。







今すぐplank1にコンポーネントヒンジジョイントを追加します（コンポーネント->物理学->ヒンジジョイント）-翻訳では「ヒンジ」を意味します。 上記のスクリーンショットでは、既に追加されています-オレンジ色のベクターが表示されています。 オブジェクトが回転する軸を示します。 現在、この方向はOX軸と一致しています。 確認するには、OXに対してボードをわずかに回転させ、シーンを開始します。



そのため、回転軸は私たちには適していません-結局、ボードを鋭く回転させると、そのエッジの1つからのボックスがターゲットの方向に飛ぶことが必要です。 Inspectorパネルで、plank1のHinge Jointコンポーネントを見つけ、Axisフィールドグループを見つけます。 これが回転軸です。 デフォルトでは、次の値が設定されています。

X：1

Y：0

Z：0



これは、OX軸の同方向性を示しています。 OZに平行な軸は私たちに適しています。 フィールドZを1に設定し、残りをゼロに設定します。 ステージ上のplank1の中心からのオレンジ色のベクトルの方向が逆になっていることに注意してください。







ほぼ完成したレバーで。 ここでの問題は、ロールダイスに十分な重点が置かれていないことです。 追加：キューブを追加し（plank2と呼びましょう）、圧縮し（下のスクリーンショットを参照）、plank1とわずかに交差するように移動して、グリップを形成します（これまでは仮想のみ）：











plank2 Rigidbodyコンポーネントを追加します（plank1には、ヒンジジョイントの追加時にこのコンポーネントがすでに自動的に追加されています）。 plank1とplank2が修正されたことを物理エンジンに「通知」しましょう。 plank1をFixed Jointコンポーネントに追加します（Component-> Physics-> Fixed Joint）。 InspectorパネルのConnected Bodyフィールドの値がNone（Rigidbody）になりました。 階層パネルからその場所plank2をドラッグします。



シーンを実行すると、レバーが一方の端に落ちる様子がわかります。 これはすでに喜ばしいことです。 シーンに立方体を追加し、その左（カメラに最も近い）端のレバーのすぐ上に配置します。 この場合、キューブのすべての次元を2倍に削減します。







Rigidbodyコンポーネントを追加します。 ここでシーンを開始すると、レバーを下げると、立方体が最後まで下がります（下がる）。 デザインは「ショット」の準備ができており、カウンターウェイトを準備するために残ります。カウンターウェイトが落ちると、新しく追加されたキューブが飛行します。



プレハブをプロジェクトに追加し、dropcubeと呼びます。 シーンに新しいキューブを作成し、階層パネルからプレハブにドラッグします。 Rigidbodyコンポーネントをプレハブに追加し、キューブをシーンから削除します。 プレハブの質量を30に設定しました（作成したシーンにオブジェクトを配置するため、この値により、起動されたキューブがターゲットに到達できるようになります）。

パート2：スクリプト。



スクリプトの部分を扱いましょう。



まず、キューブがヒットするとターゲットが消えます。 最後のタスク（LINK）からSelfdestroyerスクリプトを取得し（すべてのタスクに1つのプロジェクトを使用するため、新しいスクリプトを追加する必要もありませんでした-すべてが既にプロジェクトパネルにあります）、それをターゲットオブジェクトにアタッチします。 このスクリプトの完全なコードを次に示します（非常に小さい）： 

using UnityEngine; using System.Collections; public class Selfdestroyer : MonoBehaviour { void OnCollisionEnter(Collision obj) { Destroy(gameObject); } }




ここで、Boxdropper C＃スクリプトをプロジェクトに追加します。 彼はスペースバーをクリックして、アタッチ先のオブジェクトの位置にキューブを作成する必要があります（より正確には、インスタンスプレハブドロップキューブを作成します）。



キーストロークへの応答のみが必要なため、Updateメソッド、Startメソッド（自動生成された場合）は削除します。 計画を実装するには、少なくとも、プレハブをスクリプトに渡す必要があります。 タイプGameObjectのパブリック変数をクラスに追加します。 

 public GameObject cubePrefab;


更新方法を変更します。 

 void Update () { if (Input.GetButtonDown("Jump")) { GameObject.Instantiate(cubePrefab, transform.position, transform.rotation); } }




このコードに関する注意事項：



-まず、Input.GetButtonDown（ "Jump"）の代わりに、Input.GetKeyDown（KeyCode.Space）を使用できます。 2番目のオプションはスペースキーコードに厳密に結び付けられ、1番目のオプションはゲームコントロール設定でジャンプキー（デフォルトではスペース）の役割を果たします。 これらの設定は、プロジェクトをアセンブルし、結果の実行可能ファイルを実行することで確認できます（これが明確でない場合は、プロセスをより詳細にペイントできます）。



- 最初のタスクのスクリプトとは異なり、GameObject.Instantiateを呼び出すとき、関数の結果は必要ないため、どこにも保存されないことにも注意してください。 前回、オブジェクトを作成した後、追加で強制的に操作した場合、ここでは必要ありません。



-オーバーロードされたInstantiate関数がここで呼び出され、3つのパラメーターがありますが、オプションが1つあります：Instantiate（Object original）。 後者は、実際のパラメーターによって渡されたオブジェクトと同じ位置と回転パラメーター（3つのパラメーターを持つInstantiate呼び出しで、それぞれ2番目と3番目のパラメーター）を設定します。 これらのパラメーターは、スクリプトがアタッチされているオブジェクトの対応する値から取得されたものではないことに注意してください！



ここで、Boxdropperスクリプトをプロジェクトパネルから階層パネルのドロッパーオブジェクトにドラッグします。

まあ、タスクはほぼ完了しています。 シーンを開始してスペースバーを押すと、立方体が落下し、小さな立方体（すでにレバーアーム上にあった）がターゲットに飛び込みます。 言葉遣いとの小さな矛盾が1つ残っています。ターゲットを破壊するときに音がしません。



まず、サウンド自体、つまりある種のサウンドを持つファイルが必要です。 過去のプロジェクトの1つである爆発の音と関係のある最初の音を聞きました。 プロジェクトにサウンドを追加します（onDestroySoundという名前です）。



Unityのサウンドに関する少し叙情的な余談。 ここでの音はモノと3D（3Dサウンド）です。 実際、オーディオには1または2チャンネルが表示されます。 空間位置がかなり明確なエフェクト（爆発、衝撃音など）では、モノラルサウンドを使用することをお勧めします。実際の音量（プレーヤーが聞く）は、ソースからリスナーまでの距離に依存するためです（オーディオリスナーコンポーネントの存在によって決まります）。デフォルトでは、メインカメラオブジェクトに含まれています）。



サウンドが3Dであることが判明した場合（インスペクタパネルに「3Dサウンド」のチェックマークがあります）、絶望しないでください。 「3Dサウンド」項目のチェックを外し、同じパネルで「強制的にモノラルにする」を選択し、「適用」をクリックします。 次に、Selfdestroyerスクリプトを変更します。 再生するサウンドのパブリック変数が必要です。 

 public AudioClip onDestroy;


次に、OnCollisionEnterメソッドで、Destroyメソッドの呼び出しに加えて、サウンドの再生を追加します。 

 void OnCollisionEnter(Collision obj) { AudioSource.PlayClipAtPoint(onDestroy, transform.position); Destroy(gameObject); }




このようなAudioSource.PlayClipAtPointメソッドの呼び出しは、音源がこのスクリプトがアタッチされているオブジェクト（transform.position）の位置にあるかのようにonDestroyサウンドを再生します。



ステージまたは階層パネルでターゲットオブジェクトを選択し、サウンドファイル（onDestroySound）をプロジェクトからInspectorパネルの「On Destroy（Audio Clip）」フィールドにドラッグします。



注：任意のオブジェクトにオーディオソースコンポーネントを追加し、スクリプトから「オーディオ」と呼ぶことができます。 オブジェクトの場所で送信されたサウンドを再生する特別な方法もあります。 



 audio.PlayOneShot(AudioClip clip);




ただし、この場合、オブジェクトが破棄される直前にサウンドが再生され、audio.PlayOneShotを使用すると、削除後にサウンドの再生が停止します。 これは、彼の声が聞こえないことを意味します。 自分で確認できます（これも便利です）。

パート3.美学。



他に何がありますか？ しかし、1つの詳細が私を苦しめました：一度に複数のキューブを投げる（スペースバーを数回連続して押す）と、レバーを形成する2つの「ボード」（plank1とplank2）が互いに対してシフトし始めます。 たとえば、次のスクリーンショットで見ることができます（別の光源がシーンに追加されています）。







それから私は最も簡単な解決策を考えました：plank1とplank2を接続する必要があるのは一度だけなので、それがすでにその役割を果たした後にそれを削除してみませんか？ plank2を破壊するだけですか？ glyい。



ここでも、階層パネルでplank1を選択し、インスペクターパネルでFixed Jointコンポーネントを見つけます。 奇妙なフィールドに気付きます：



破壊力：無限大



必要なもの！ 私の場合、関節を破壊するのに必要な力の値は10になりました（ニュートンに望みます）。 「近づいた」とは、スローイングキューブがレバーの表面から離れた瞬間に、plank2がplank1から離れて飛行することを意味します。







判明したように、ヒンジジョイントも同じ問題の影響を受けます-plank1の奇妙な動作は、多数の落下キューブで再び観察されます。 同じ方法で問題を解決しましょう-Break Forceフィールドを30に設定します。その結果、最初の起動後にレバー構造全体が崩壊し、非常に「物理的」になります。



これは、最初の実行後のシーンの様子です。







PS美しさを高めるために、シーン内のすべてのオブジェクトに材料を追加できます。レバレッジ、すべてのキューブ（大小）、床。 これを行う方法については、記事の冒頭で説明します。
