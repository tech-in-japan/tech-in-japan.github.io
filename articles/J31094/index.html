Oracle Application Server 10gリリース3では、同じサーバー上でPHPおよびJ2EEアプリケーションを実行できます

OracleにはPHP d Application Server 10gリリースが含まれているため、J2EEおよびPHPアプリケーションに同じサーバーを使用できます。



PHPはデフォルトで有効になっています。 OracleのHTTPサーバーのルートは次のとおりです。



$ ORACLE_HOME / Apache / Apache / htdocs



このディレクトリ内の拡張子が.phpまたは.phtmlのファイルは、PHPを使用して処理されます。 拡張子が.phpsのファイルは、フォーマットされたソースコードとして表示されます。



Application Server（AS）バージョン10.1.3.0には、PHP 4.3.11が付属しています。 パッチAS 10.1.3.2は、PHP 5.1.2を追加します。 別のサーバーをインストールせずに異なるバージョンのPHPを使用する必要がある場合は、PHPリリースをコンパイルできます。



注：ASでのPHPバージョンの変更はサポートされていません（推奨されません）が、一部の環境では技術的には可能です。 エラーが発生した場合、サポートサービスは、PHPを再インストールしたかどうかを尋ねます。



技術的な問題は、Oracleライブラリにヘッダーファイルが含まれていないことです。 これは、Oracle Instant Clientを使用してPHPとOracleを接続することで回避できますが、ASはInstant Clientライブラリを使用しないため、これも問題です。 一方、同じエラーまたは予期しない動作が発生します。



記事の残りの部分では、PHPの再インストールに関する技術的な詳細を説明します。
