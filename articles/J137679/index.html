動作原理


1.コンピューターがWindowsで起動し、LinuxのデフォルトOSがブートローダー設定で変更されます

  • 管理者からの「タスク」がある場合、タスクファイル（パスワードで保護されたアーカイブ）は「タスク」ftpサーバー（管理者がアーカイブを配置する場所）からローカルディスクにコピーされます。
  • タスクを完了するためにコンピューターを起動した場合、ブートの非常に早い段階で、ネットワークアダプターが切断され、アーカイブが解凍され、start.exeが起動されてタスクが完了します。


2.コンピューターはLinuxで起動し、Windows OSのデフォルトOSはブートローダー設定で変更されます

  • 前回の起動時にコンピューターが管理者のタスクを実行していた場合、バックアップを作成し、最も関連性があるものとしてマークします。
  • コンピューターが最後の起動時にタスクを実行しなかった場合、最新のバックアップから回復し、プロファイル、「プログラムファイル」でセクションをマウントし、ウイルスをチェックします。 再起動します。




3段階の実装


1. Windows OSのインストールは、NLiteによって準備されたイメージから実行され、プロファイルフォルダーと「プログラムファイル」へのパスが示されます。

*インストールの前に、Windowsシステムパーティション用、プロファイル用、および「プログラムファイル」用の2つのパーティションを作成する必要があります。

2. Linux OSのインストール（最小限のビルドでDebian Lennyを使用しました）：

  • バックアップが保存されるディスク上に追加の（便宜上）セクションを作成することをお勧めします（ファイルシステムはWindowsでサポートされるべきではありません。これにより、単にイメージ/バックアップを変更することができなくなります）。
  • Grubは専用のパーティションにインストールすることをお勧めします（WindowsとLinuxの両方の構成を変更できることが重要です）。


3.スクリプトの追加：

  • LinuxのデフォルトOSをGrubの構成に変更するプログラムがWindowsスタートアップに追加されます（バットからBatchToExeを使用してexeを準備すると便利です。これにより、プログラミング知識がほとんどなく、機能が追加されます）。
  • 「スタートアップ」Linuxで、Grub構成を変更してデフォルトでWindowsを起動するスクリプトを追加します。
  • 管理者の裁量で他のサポートスクリプトの作成。




私の意見では、最も疑わしいのはFATパーティションへのGrubの配置です（結局、設定は不正に変更される可能性があります）。 事実、GrabにOSを条件付きで実行させた方法を覚えていないだけです。 ただし、habrozhiteliを別の方法で実装することは難しくありません。たとえば、Grub2を使用し、ホーンビーム自体にOSスイッチャーを実装する方が合理的です。
