Microsoft Web Platform Installer、またはIIS 7にPHPをインストールする方法

IIS7にPHPをインストールする必要が生じました。 以前は、IISバージョン5.1（XPにバンドルされている）でphpをインストールしなければならなかったことを覚えています-それが全体のポイントでした。 インストールするには、手で何かを台無しにしてから、ある種のライブラリが間違ったバージョンであることが判明したので、それを探す必要がありました。まあ、一般的に、良いマニュアルが見つかったのは良いことです。 今ではたくさんあります。 スクリーンキャストもあります。

現時点では、すべてがさらにシンプルでした。 Microsoft Web Platform Installer 2.0をダウンロードします （記述-約1メガバイト、現在ベータ版）。 そこで必要なコンポーネントを選択し、URL Rewrite、SQL Server 2008、ASP.NET MVC 1.0、さらにはあらゆる種類のブログエンジン、DotNetNukeなどのサイトデザイナーを選択できます。 以下は、新しいものから提供されるもののみのウィンドウです。



私はすべての楽しみを選んだ。 PHPを除くすべてが配信されました:)。 なぜダウンロードされなかったのかはわかりませんが、Web Platform Installerは詳細なエラーログを提供します。このログには、IISのphpをダウンロードするためのリンクが見つかりました。
