作業環境を評価する8つのクラウドログアナライザー

デバイスの数を増やし、資格要件を引き上げ、データ収集プロセスを最適化することは、あらゆるタイプのビジネスに必要です。 ログ分析は、情報環境で何が起こっているかを実際に把握することができます。 リアルタイムの例を次に示します。



  • パフォーマンス計画
  • 問題の早期発見
  • 現在の報告
  • 可用性管理


数メガバイトのログファイルがある場合、それらは手動で表示できますが、ログの量がギガバイトで計算される中規模および企業のビジネスでは、手動分析は悪夢に変わります。







インフラストラクチャを含むWebアプリケーションのログを監視および分析するソリューションを探している場合は、次のログツールを検討できます。 それらのほとんどには、わずかに切り捨てられた無料版があり、そこで作業を試みることができます。

Loggly



Logglyは最も人気のあるログ管理ソリューションの1つであり、以下を含むほぼすべてのソースからのデータの分析に適しています。



  • Webアプリケーション-Apache、Nginx、PHP、Node.js、.NET、Java、JavaScript
  • オペレーティングシステム-Linux、Windows


Logglyを使用すると、分析用のデータ（ログ）を収集し、有益なダッシュボードを作成して必要なインジケーターを監視できます。 もちろん、Logglyインターフェースからログを検索できます。







Logglyには強力な検索エンジンがあり、すべてのテキスト、1つのフィールド、ブール値に対してフィルターを構成および設定できます。 メール、PageDuty、Slack、HipChatなどをバインドして、アラートを受信することもできます。



Sumologic



SumoLogicでリアルタイム分析を取得して、ビジネスアプリケーションとインフラストラクチャを監視、分析、トラブルシューティング、および保護します。



SumoLogicには、ログをSumoLogicに転送するためにサーバーにインストールする必要があるコレクター/クラウドコレクターエージェントがあります。 あらゆる種類のログを変換して、設定された制限に達したときに監視および通知するために、リアルタイムで更新される、起こっていることの視覚的な画像を作成できます。







SumoLogicには、タスクを高速化し、問題を調査するための最も一般的なアプリケーションに基づいて、いくつかの事前構成されたアプリケーションがあります。



以下のサンプルアプリケーション：



  • mongoDB-一般的な状態追跡、クエリの最適化、セキュリティ監視。
  • AWS Lambda-プロアクティブなモニタリング、コストレビュー、詳細な調査。
  • AWS-パフォーマンスの問題の特定、アクティブなセキュリティ制御、高度な分析。
  • Salesforce-ユーザー、監査、データ比較の改善された適応。
  • トレンドマイクロ - 脆弱性分析、イベント履歴、攻撃の視覚化の改善。
  • Github-チームのパフォーマンスの視覚化、使用の基本的な指標の作成。
  • Docker-高度な分析、トラブルシューティング。
  • Linux -OSのトラブルシューティングと最適化、アクセス/検証およびセキュリティイベントに関するアラート、パフォーマンス指標の追跡。
  • Nginx-レビュー、訪問者の場所、 応答時間 、パフォーマンスのボトルネックの特定。
  • Apache-訪問者との対話、パフォーマンスの最適化、訪問者の視覚化。
  • IIS-包括的な情報、トラフィック統計、遅いページの検索。
  • MySQL-一般的な状態、キーエラー、クエリパフォーマンスの監視。


無料版では、1日に500メガバイトのデータが収集され、ソースタイプのデータを含む7日間のデータストレージが提供されます。



スプランク



Splunk Cloudは世界中の10のAWSリージョンで利用でき、あらゆる種類の情報を収集してインデックスを作成します。 Splunkは、物理デバイス、クラウドデバイス、または仮想デバイスで作成されたデータをすばやく簡単に分析および視覚化する方法です。







Splunkを使用すると、データを追跡して視覚化し、アラートのしきい値を設定できます。 無料版を見たことはありませんが、15日間5 GBのデータを分析および視覚化できる試用版があります。



ペーパートレイル



Papertrailは、syslog、テキスト、Apache、MySQL、Herokuアプリケーション、Windowsイベントログ、ルーター、ファイアウォール、RubyOnRailsなどからデータを収集できます。



ブラウザ、コマンドライン、またはAPIを使用してリアルタイムで検索し、電子メールまたはSlackでアラートを受信できます。







Logz.io



Logz.ioはELKの宿泊施設をサービスとして提供します。 ELK（Elastic Search、Logstash、Kibana）は、最も人気のあるオープンソース分析プラットフォームの1つです。 ログを処理するための柔軟なプラットフォームを探している場合は、Logz.ioが役立ちます。



Logz.ioは、ほぼすべての最新技術のデータにインデックスを付け、データの視覚化とアラートを使用した詳細な分析を提供します。







サポートされている技術：



  • Apache HTTP / Tomcat
  • Nginx
  • Hadoop
  • ヘロク
  • Node.js
  • mongoDB
  • postgreSQL
  • 窓
  • Linux
  • AWS


無料版では、1日あたり3 GBのデータを含む3日間のストレージが提供されます。



木材



リアルタイム追跡、最先端のフィルタリングシステム、クイック検索、およびTimberをアプリケーションまたはプラットフォームに統合する機能。







Node、Ruby、Elixirのライブラリが利用可能であり、以下を含む多くのプラットフォームをサポートしています。



  • Logstash
  • Zeit（現在）
  • AWSラムダ
  • Docker
  • Linux
  • ヘロク
  • クベルネテス


無料版から始めて、このプラットフォームを試すことができます。



Logentries



すぐにデータを取得して視覚化するために、 Logentriesにデータを送信します。 Logentriesユーザーインターフェイスでログを監視し、必要な値に関するアラートをすぐに受信できます。



アラートは、Slack、Hipchat、Pagerduty、またはWebhook経由で送信できます。 Logentriesは、視覚化のためにログを照会するためのREST APIを提供します。







無料版では、最大5 GBのデータを送信でき、有効期間は7日間です。



Logsene



Sematext Logseneは、ログを管理および分析するためのクラウドベースのELKスタックです。 Syslogなど、任意のソースから暗号化されたチャネルを介してログを送信できます。



AWSで実行され、SOC、SSAE、FISMA、DIACAP、HIPAAなどをサポートします。







デモを試して、その仕組みを理解してください。



これらのクラウドベースのログアナライザが、データの検索、分析、視覚化を支援し、IT分析を容易にすることを願っています。









LOOKING.HOUSE-このプロジェクトでは、40か国で150点以上の見た目ガラスを収集しました。 ホスト、ping、traceroute、mtrコマンドをすばやく実行できます。
