マスターまたは仕事？

みなさんこんにちは。 統計によると、ハブには学生であり、ソフトウェア開発の分野でキャリアを築きたいと考えている人がかなりいます。 したがって、私は仕事と奉仕の利点について少し議論し、またこれらのオプションのそれぞれを評価しようとする記事を書くことにしました。



だから、私たちはロシアで学部課程を終えています。 長年にわたり、数学的分析、線形代数、離散数学、数学的論理、確率論、統計の基礎に精通しています。 さらに、基本的なアルゴリズムとデータ構造、タッチエレクトロニクス/人工知能/データ処理/データベース/その他のプロファイルを学びました。 また、いくつかのコースプロジェクトを完了し、それによっていくつかの新しい「トレンド」テクノロジー（Node.JS、MV * javascriptフレームワーク、NoSQL、JavaEE / Spring、ASP.NET MVC、QT、Xamarinなど）を研究しました。 一般に、私たちは私たちにとって興味のある分野でイントロを得ました。



また、私たちの一部がすでに経験を積んでいることを忘れないでください。 少なくとも、全員に夏のインターンシップ/実践があり、大人の叔父がどのように働くかを見ることができました。



平均的な学生である私たちが卒業後誰であるかを要約します。 私たちは、彼らが専門分野の用語集を専門化し、実行した分野の普通の後輩だと思います。



最初の方法-マスター



行政の利点を理解しましょう。 マスターのプログラムは次のとおりです。

  • 高度;
  • より深刻な理論的知識。
  • 自分の何かに費やすことができる2年間（スタートアップ、本を読むなど）。
  • 一部の国への移民の簡素化（たとえば、カナダへ）;
  • 奉仕の助けを借りてヨーロッパ/アメリカに移動する機会。




高度。 ロシアでは長い間、学士号は「学部生」になると考えられています。 また、すぐにすべての空席（IT企業ではない）に「MDが必要」という項目が含まれ、「高等教育が必要」という項目は含まれないと考えられています。



以下の引数：

  • ロシアでは、すべてが急速に変化しています。




反対論：

  • プログラマーになりたいですか？ 現在、空席にある最大数は、高等教育の要件です。 誰も政権について書いていない。 興味のある方は、西部の空室を開くことができます。空室の99％で、要件はMDではなくBDです。 主なものは地殻ではなく、経験とスキルであることを忘れないでください。
  • 科学者に向かって発展するためだけにMDが必要であると信じられています。 MDはPhDへの入学要件です。




より深刻な理論的知識。 確かに、いくつかの大学では、あなたはうまくポンプすることができます。 これはサンクトペテルブルクのアカデミック大学、これはモスクワのYandexとABBYY Lingvoのプログラム、これはエカテリンブルクのKonturの修士プログラムです。 ただし、残念なことに、ほとんどの大学院では、さらにポンプを使用していません。 したがって、上記のプログラムに参加できる場合は、行動するだけでメリットがあります。 それ以外の場合は、この段落を短所に関連付けます。



あなた自身の何かに費やすことができる2年。 確かに、今回は持っています。 しかし、すべての卒業生が将来のCEO /共同設立者であるというのは本当ですか？ 学部在学中にプロジェクトを実行できなかったのはなぜですか？ おおまかに言って、このアイテムは非常に物議を醸すものであり、私たち一人ひとりに個別のものです。



一部の国への移住の簡素化。 このアイテムは、紛らわしいことではないが、政務のための貯金箱です。 移動したい場合は、このアイテムについて考えてください。 これまでのところ、BDは移動に十分ですが、次に何が起こるかは明らかではありません。



政権の助けを借りてヨーロッパ/アメリカに移動する機会。 繰り返しますが、アイテムは貯金のために貯金箱に行きます。 外国の行政に参加できるなら、考えるべきことは何もありません。



さて、行政のマイナス面について：

  • 週に40時間働く方法はありません（どちらかというと、非常に難しいでしょう）。
  • 人生の2年の無駄：MDを高く評価する人はいませんが、多くの正式なことをする必要があります（哲学のGOSy、GOSTに従った論文の説明、教育活動）。




週に40時間働く方法はありません。 多くは、仕事と勉強を組み合わせて奉行しています。 しかし、それはとても便利ですか？ 20時間働いても、すべての経験が得られるわけではありません。また、私たちが働くことができる会社のリストも小さくなっています。 40時間働いて、私たちは実際に学校に現れません：しかし、ポイントは入学（入学-勉強！）です。



人生の2年の無駄。 この項目は議論の余地があります。 どこに行くことができるかに大きく依存します。 このアイテムには、より深刻な理論的知識（政権の利点の1つ）と共通点があります。



オプションを要約するために-政務で勉強する。 あなたが本当に知識を引き出すことができるまともな場所に行けるなら、これは本当に良い選択肢です。 ボーナスとして、別の国への移動（この修士号がロシアにない場合）、または移住時のボーナス（この修士号がロシアにある場合）があります。 別のボーナスとして（小さなものではありますが）、クラストの改善があります。 それ以外の場合、マジストラシーは非常に疑わしい職業であり、キャリアの構築を開始することはできません（そして、マジストラシーの2年間で中間のランクに進みます）。



方法2-仕事



私たちは、学部研究の後の開発のさらなる段階として、すぐに仕事を選ぶ利点に進みます。 だから、 仕事は：

  • 2年でジュニアからミドルに成長する能力。
  • 給与;
  • 研究所に存在する正式なルーチンの欠如;




2年でジュニアからミドルに成長する能力。 この間、実践的なスキルを向上させます。 これにより、次のレベルに移動し、単純なタスク（通常は後輩に行く）の実行からより興味深いタスクに移動できます。



給料。 プログラマーにとって非常に疑わしい点。 誰がお金を望んでいる-この時点で長い間それを持っている（残念ながら、誰もフリーランスをキャンセルしませんでした）。



正式なルーチンの欠如。 運がよければ、多くのレポートを書く方法を忘れてしまう可能性が高くなります。さらに、GostやStandardsのような厄介な言葉も忘れてしまうでしょう。



仕事の短所：

  • MDなし;
  • 他の国への移住は複雑かもしれません。




MDなし。 この項目については、マジストラシーに関連するセクションですでに説明しました。 ここでは、誰もがMDとより急な地殻からプラスを受け取るかどうかを自分で決める必要があります。



他の国への移住は複雑になる可能性があります。 はい、おそらくこれは本当です。 たとえば、カナダに移住した場合、BDの前にMDを使用すると、競争で余分なポイントが得られます。



したがって、見てわかるように、仕事の長所と短所は、政務の長所と短所のほぼ反対です（2つの直交するオプションから選択する場合は論理的です）。 仕事をすれば、エンジニアとしてより速く成長できるようになりますが、高校（そして多くの場合、R＆D）科学でのキャリアは完全に台無しになります。 それは怖いですか-誰もが自分で決めます。



記事を要約します。 仕事と修士の両方の長所と短所は非常に明白です。 仕事は本当の経験です。 学習は地殻であり、おそらく理論的な知識です。 どちらが良い-あなただけ。



コメントであなたのコメントを聞くのは面白いでしょう。 あなたがリーダーである場合、学位を検討していますか、それとも学位を持つことが重要ですか？ あなたが従業員の場合、学位とは何かとよく聞かれますか、それともあなたの知識と経験だけで雇われましたか。
