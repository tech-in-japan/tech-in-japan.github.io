デスOS / 2：殺人かクロスボウか？

私の最初のOS / 2の記事は、主にOS / 2の失敗に個人的に貢献したことに対する謝罪で構成されていました。 しかし、私は自分のせいにすることはできません。 正直なところ、OS / 2の障害には何千もの理由があり、その大半は何もできませんでした。 ここにそれらのいくつかがあります。



むかしむかし、IBMは大規模なハードウェア-ドイツの大学の1つがBlue Giantコンピューターの冷却システムによって加熱されるほど大きなメインフレームで狂ったお金を稼いでいました。



「ミニコンピューター」でさえ部屋全体を占有することが多く、これによりIBMは顧客を完全に制御できました。 組織は、深刻なコンピューティングパワーを提供するにはかなりの金額が必要であるという事実を当然のことと考えていました。 IBMは、必要なだけの資金を要求できることを理解し、それを実行しました。 パソコンはこのモデルに適合しませんでした。





車、プリンター、ドライブ、人、猫、犬が一緒に住んでいたIBMの時代。





IBMの大きなハードウェアが危険にさらされていました。高度なPCがサーバーとしてBlue Giantに侵入していました。 メガプロバイダーは、自分のPCをミニコンピューターとメインフレームに変えることで対応することにしました。 これには、新しいオペレーティングシステムが必要でした。 DOSはなくなっているはずです。



IBMと一緒にOS / 2を開発したマイクロソフトは、その影響がデスクトップコンピューターに限定されていたため、市場の上位セグメントに移行するというアイデアが本当に好きでした。ビルゲイツは嘘をつかず、OS / 2への道にWindowsとDOSのステップを呼び出しました。 これにより、競合するDR-DOSとUNIXがゲームから除外されました。



それでは、OS / 2サーバーはどうなりましたか？



当時、x86サーバーは、IBM System / 36および38ミニコンピューター、さらには1980年代後半のOS / 2とほぼ同時に登場したAS / 400よりもはるかに注目を集めていました。



RAMとディスクをIntelベースのシステムに追加することができ、IBMからそれを購入したり、IBMプログラムを実行したり、IBM従業員に修理して他のシステムと統合したりする理由は1つではありませんでした。 これは良いだけでなく、避けられないことでもあるように思われるかもしれません。 しかし、非常に収益性の高い市場の85％を所有している場合、この必然性に良いものは見当たりません。



無数の企業が、IBM紙を使用してIBM SNA経由でIBMプリンターに接続されたIBMマシンでIBMソフトウェアを使用しています。 完全な愚か者だけがそのような影響を危険にさらす可能性がありますが、IBMは当初は破壊的で多様なPCの世界の制御を失い始めました。



IBMパーソナルコンピューターを作成するとき、私たち自身の生産の他のシステムと何らかの形で接続することは私には決して起こりませんでした。 非常に長い間、このためにサードパーティのハードウェアとプログラムを使用する必要がありました。 IBMは1ペニーのPCをすべて失いましたが、メインフレームに接続された各PCが端末になり、大型マシンのプロセッサーの負荷が増加することに気付きました。 これは、顧客がメインフレームを更新する必要があることを意味しました。 実験的なルールによると、各端末はメインフレームの売り上げをMIPの約半分増加させました。



私はこのデータの正確性について完全に確信しているわけではありませんが、PCにネットワーク機能を追加しないことが決定されたときに考慮されました。 IBMの影響力は非常に大きかったので、どの行動も賢明と見なされました。 もちろん、UnixとVMSはそうでしたが、多くの購入者はIBM製ではないという事実に恐れを抱いていました。



x86 IBMサーバーのリリースは、必然的にメインフレームからの高利益へのPCからの低利益へのシフトを伴いました。 とりわけ、これはOS / 2が一流のサーバーシステムになり得るが、この超大国は手遅れになるまで隠れ、IBMのx86サーバーの価格は非常に高かったことを意味しました。



IBM PC部門の従業員はサーバーの見通しを理解していましたが、戦術的な策略として、Microsoftサーバー用のOS / 2バージョンの開発を委託しました。 このようにして、彼らは、誇らしいが低収益のPC部門とは対照的に、深刻なお金を稼いだIBM部門の怒りを避けようとしました。



少なくともIBMが「ビジネスパーソン」に率いられている間は、他の決定を下すことはできそうにありませんが、主な機能は、自分の製品を理解しようとする根本的な不本意です。 競合他社は、個人的にテクノロジーの開発に参加した人々によって導かれ、誰もがビル・ゲイツを優秀なプログラマーと呼ぶことはまずありませんが、彼は経理部でキャリアを築くのではなく、コードを書くことから始めました。 この点で、彼はIBMのリーダーシップを大きく上回りました。
