Frontend Mixを使用したレポート：BEMおよびReactのWebサイトロード最適化および設計システム







フロントエンドの近くにいるすべての人に、8月のフロントエンドMIXラリーのレポートを見ることをお勧めします。 アルファ研究所の招待講演者であるYandex.MoneyとEpamは、モバイル最適化のニュアンスと、Npm v5、Yarn、またはpnpmの選択、およびBEMとReactの設計システム構築の秘密を共有しています。







カットの下に3つのビデオがあります。







＃1スマートフォンの時代にサイトの読み込みを高速化する方法



Yandex.Moneyの主要な開発者であるAndrey Melikhovは、スマートフォンでのサイトの読み込み速度を向上させる方法について語っています。









レポートでは、コンテンツを転送する最新の方法（HTTP / 2：プリロード、サーバープッシュ）、Chrome Dev Toolsでのロードのプロファイリング、JavaScriptの最適化について言及しています。







＃2 Alpha LaboratoriesがBEMとReactを使用して柔軟で拡張可能な設計システムを作成した方法



Vitaly Galakhovのレポートでは、Alpha LaboratoryでBEM方法論を使用した経験を紹介しています。 最初に彼らはBEMを試し、それを放棄し、その後、方法論が有用である場所を発見しました。









Vitaliyは、ソリューションの柔軟性、拡張性、および全体的な優雅さを失わずに、多くのチーム向けに設計システムを統合および拡張可能にする方法を説明します。







＃3 Npm v5、Yarn、またはpnpmの魅力的な選択



Epam開発者のMichael Bashurovは会議を閉じ、パッケージマネージャーの選択に関する意見を共有しました。 彼らは、「糸が必要」や「Npm v5がいつ登場したのか」などの神聖な問題と、pnpmのようなあまり人気のない製品を使用する利点について話し合いました。
