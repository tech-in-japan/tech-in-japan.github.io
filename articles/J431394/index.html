医療業界では、一部のコペンソースの人が（H）ealthcare（API）プロジェクト、またはHAPIを作成しました。これは、本質的に医療業界向けのオブジェクト指向のテキストメッセージングパーサーです。 残念ながら、彼らは「いつ止めるべきかわからない」症候群に苦しんでいるようでした 。



区切り文字列または固定形式文字列をテキストフィールド値のリストに単純に分割する汎用パーサーを実装する代わりに、 最新バージョンでは、それぞれ独自の高レベルデータ構造を持つ1205種類のパーサーを実装しています。 最高レベルの構造には、多数の下位構造があります。 各パーサーには、フィールドごとに1つ以上のアクセス方法があります。 フィールドは、単一のインスタンスまたはインスタンスのリストにすることができます。その場合、使用するアクセス方法をプログラムで決定する必要があります。



これは、約15,000のメソッド呼び出しを持つAPIです！ これらの開発者は何を考えていましたか？



たとえば、クラスEHC_E15_PAYMENT_REMITTANCE_DETAIL_INFOには、0個以上の製品サービスセクションが含まれる場合があります。 だから私はすぐにある種の配列やリストについて考え始めました。 したがって、このようなものの代わりに： 



  EHC_E15_PAYMENT_REMITTANCE_DETAIL_INFO情報= ...;
    リスト<EHC_E15_PRODUCT_SERVICE_SECTION> prodServices = info.getProductServices（）;
     //繰り返します


...次のいずれかを実行する必要があります。 



  //下部構造を取得します
     EHC_E15_PAYMENT_REMITTANCE_DETAIL_INFO情報= ...;
	
     //下位構造から組み込みの製品サービスを取得します

     // ...メッセージが1つしかないことが確実にわかっている場合：
     EHC_E15_PRODUCT_SERVICE_SECTION prodSvc = info.getPRODUCT_SERVICE_SECTION（）;
	
     // ...いくつあるかわからない場合：
     int n = infos.getPRODUCT_SERVICE_SECTIONReps（）;
     for（int i = 0; i <n; i ++）{
         EHC_E15_PRODUCT_SERVICE_SECTION prodSvc = info.getPRODUCT_SERVICE_SECTION（i）;
         //これを使用します
     }

     // ...または、それらを取得して反復することができます
    リスト<EHC_E15_PRODUCT_SERVICE_SECTION> allSvcs = info.getPRODUCT_SERVICE_SECTIONAll（）;


...そして、目的のメソッドを呼び出す必要があります。そうしないと、例外が発生する危険があります。 ただし、APIを介して1つのタスクを実行する方法が多数ある場合、APIを使用してコードでタスクを実行する方法は多数あり、必然的に問題が発生します。



あなたは言うことができます： 「さあ、すべてがそれほど悪くない」 ; 必要なものだけを使用してください。 しかし、これらのデータ構造の一部は10レベル以上の深さに埋め込まれており、それぞれに数十のサブ構造やフィールドがあり、それらすべてにいくつかのアクセス方法があることがわかります。 さらに、それらはすべて本当に長い名前を持っています。 そして、HAPI開発者はタイピングにうんざりしており、略語としてLA1、ILT、PCRなどのデータ構造の使用を開始したことに気づきます。



APIは便利になろうとします：解析を要求しているフィールドで予期されるものが見つからない場合、例外をスローし、自分で何がうまくいかなかったかを把握する必要があります。 もちろん、これは、データストリームで何が送信されているかを既に知っていることを意味します。



匿名はヘルスケアで働き、HAPIにラップされたライブラリをサポートしました。 彼は、1つの追加フィールドを単純に解析することで、タスクを完了するために定期的に（数週間かかった）タスクを与えられました。 APIドキュメントのボリュームをじっくりと時間をかけて、彼はインターフェイス全体の代わりとして、いくつかのsplit、substring、parseDate、parseIntを含む300行の1つのクラスから共通のパーサーを作成しました。



その後、新しいフィールドの追加には10分しかかかりませんでした。
