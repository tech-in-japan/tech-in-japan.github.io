Webアプリケーションはネイティブのものに置き換わりますか？

まとめ 近い将来、サイト/ Webアプリケーションは、ネイティブアプリケーションに取って代わるか、真剣に取って代わることができます。



アプリケーションは、HTML / CSS / JSでApp Store、Androidマーケット、その他から独立して作成できます。


これは、多くの独立した要因のおかげで可能になります。



1.非友好的な生態系政策





公式には、App StoreからのみiOSにアクセスできます



Appleのポリシーでは、収益の30％を差し引くことを義務付けられており、サブスクライバーに関する情報を共有しないことを決定したため、 The Financial Timesは App Storeを離れました 。 ウォールストリートジャーナルは以前、ストアでのサブスクリプションの販売を停止しました。



言うまでもありませんが、30％は非常に多く、絶えず変化する多数の制限と相まって、アプリケーションストアは開発者にとって最も楽しい現象ではありません。



App Storeが、プログラムを取得する他の機会とともに、特権が保証された、信頼できる、高品質で安全なアプリケーションのディレクトリである場合、そのようなサービスの支払いは正当化されます。



しかし、現在、これは開発者がアプリケーションをエンドユーザーに提供する唯一の公式な方法です。 実際、App Storeは、アプリケーションのインデックスを作成してユーザーに表示する検索エンジンに似ています。 しかし、検索エンジンは、App Storeとは異なり、このまさにインデックス作成に課金しません。



異議を唱えるiOS用の非公式の代替リポジトリCydiaがあるのも不思議ではありません。



2.生態系が多すぎる





KPCBによると、タブレットとスマートフォンの売上は、 2011年半ばのPCとラップトップの売上を上回りました 。



店舗内の問題に加えて、店舗全体をカバーする多数のアプリケーションバージョンを開発することは困難です。



アプリケーションを最大数のユーザーが利用できるようにするには、少なくとも次のバージョンを作成する必要があります。



  1. Apple iOS
  2. Google Android
  3. Windows Phone



    そして、あります：

  4. ブラックベリーOS

  6. サムスンバダ

  8. Nokia Maemo




8つのプラットフォームにはそれぞれ、対処する必要がある独自のニュアンスがあります。



WebOSは、多くの人に馴染みのあるHTML 5、CSS、およびJavaScriptが開発で積極的に使用されているという事実により、生活を簡素化します。 デバイスのハードウェアへのアクセスは、JavaScriptを使用して取得できます。 クールですが、一度に全員のためにそのようなコードを書く方が良いでしょう。



そして、現時点では、 PhoneGapが最適なソリューションかもしれません。 HTML、CSS、およびJavaScriptで行うと、出力はすべてのストアのユニバーサルコードになります。



非常によく、普遍性の問題は解決されましたが、 独立の問題は解決されていません。 あなたはまだオーナー企業との共通言語を見つけることができず、店から飛び出します。



3.サイトからスマートフォンおよびタブレットのハードウェアへのアクセス





W3Cが本格化



アプリケーションストアは互いに競合しますが、W3Cはウェブサイト/ウェブアプリケーションがデバイスのハードウェアコンポーネントにアクセスできるようにするオープンな標準と仕様を作成するために本格的です。 カメラから画像を取得したり、マイクから音声を取得したりできます。



たとえば、マイクから音を受け取るように見えます： 



navigator.getUserMedia({audio: true}, gotAudio); function gotAudio(stream) { // ... use 'stream' ... }




そして、私たちはビデオを取得します： 



  navigator.getUserMedia({video: true}, gotStream); function gotStream(stream) { // ... use 'stream' ... }




これは、多くの開発者にとってネイティブアプリケーションの主要なプラスであるデバイスのハードウェアへのアクセスが、サイト/ Webアプリケーションで利用できることを意味します。



4.モバイルデバイス向けの汎用かつ独立したソリューション



Boston Globe新聞とISOは、レスポンシブWebデザインテクノロジーを使用したアダプティブWebサイトを使用したモバイル訪問者のコンテンツアクセシビリティの問題に取り組んでいます。



このようなサイトは、モバイルデバイスとデスクトップの両方に最適に表示されます。 レスポンシブWebデザインのアプローチは、スマートフォンやタブレット向けのユニバーサルWebアプリケーションの作成にも使用できます。





モバイルデバイスからのボストングローブ



問題は、現時点では、適応型サイトからスマートフォンのカメラ、アドレス帳、SMSにアクセスできないことです。 まだ入手しないでください。



前の段落で説明したように、モバイルデバイスのハードウェアにアクセスするための標準が広く採用されることですべてが変化し、この作業はすでに進行中です。



この問題を解決するとき、別の問題が発生します。3Gの速度では、このような機能を備えたサイトを取得できない場合がありますが、これも解決されます。



5.非常に高速なLTEモバイルインターネット







LTE広告、商用利用可能な技術



次世代の高速モバイルネットワーク（マーケティング4G）の出現により、ダウンロードで326.4 Mbit / s、リターンで172.8 Mbit / sの理論速度を備え、ネイティブアプリケーションからサイトおよびWebアプリケーションへの重点がますますシフトしています。



2011年12月、ノボシビルスクで、ヨタはロシアで最初のLTEネットワークを立ち上げました。 そして、すでに2012年1月に、LTE用のモデムは小売店で販売されます。



ロシアの他の約12の都市でも、技術的にLTEを開始する準備ができています。 このプロセスが人為的に禁止されている理由とその説明は、別の記事に値します。



おわりに



現時点では、普遍的な独立したネイティブアプリケーションを作成することはできません。 これはすべてのデバイスで機能し、エコシステムによって課される制限に依存しません。



ハードウェアへのアクセスに関する仕様が一般的なデバイスに実装され、モバイルインターネットの速度が向上する場合、ソリューションはサイト/ Webアプリケーションである可能性があります。 複雑な機能を必要としない人、そしてこれは、たとえばメディア企業にとって、 レスポンシブWebデザインテクノロジーにはすでにソリューションがあります。
