Amazon EC2とAtlex Cloud VDC：パフォーマンスの比較

誰もがAWSのことを知っています。 Amazonクラウドは、一種の業界標準になったと言えます。 私たちも例外ではありません。 そこで、機能が近いAWSマシンと比較して、OpenStackベースのクラウドで仮想マシンがどのように見えるかを確認することにしました。







カットの下の詳細。



クラウドの「インスタンス」は、1か月間月額料金をすぐに支払う必要がないという点で、従来のVPSとは異なります。 このような仮想マシンはタクシーのように機能します-移動中はカウンターがカチカチと音を立てています。 スイッチを切ったマシンでもストレージ容量を使用するため、単純なものには一定の金額を支払う必要があります。 実際に消費されたコンピューティングリソースの支払いは、継続的に必要がない場合、はるかに安価で便利です。 このようなスキームにより、開発者とテスターは真剣にお金を節約でき、企業のITインフラストラクチャの所有者でさえ、たとえば営業時間後またはプロジェクトの終了時に不要なサーバーをオフにできます。 クラウドでは、負荷が増加した場合、リソースをすばやく増やす（プロセッサを1つまたは2つ追加し、RAMの量を増やす）か、リソースの必要性がなくなったときに不要なリソースを拒否できます。 確かな利点：ハードウェアまたはオペレーティングシステムのライセンスを購入する必要はありません。 サーバーはWebインターフェースを介して数分で立ち上がり、彼らが言うようにオンデマンドで支払われます。



一般的な質問



AmazonはAmazon Web Services（AWS）で市場を開拓しました。 その後、MicrosoftはAzureプラットフォームで到着し、Google Cloud Platformはすでにその背後に登場しました-このトリオは市場のかなりの部分を掌握しましたが、Amazon Elastic Compute Cloud（Amazon EC2）は長い間クラウド内のスケーラブルなコンピューティングリソースの代名詞でした。 3つの巨人に加えて、多くの国内および外国企業が同様のオファーを行っています。 したがって、4プロセッサ構成とEC2オファーを比較することは非常に自然でした。



ロシアのサービスにはいくつかの否定できない利点があることに注意してください。 彼らの価格はハードカレンシーに縛られていません、制裁戦争の別のラウンドのイベントであなたのデータへのアクセスを失うことはありません、そして最も重要なことには、私たちの会社は前払いの支払いスキームを使用していますが、請求書の毎月の支払いは法人にも利用可能です。 欧米の大手プロバイダーは、個人と後払いで取引することもあり、銀行カードをアカウントにリンクする必要があります。 悪党があなたの口座を盗む場合、深刻な借金が生じるかもしれません、 すでにそのような先例がありました 。



たとえば、AWSでの登録プロセス自体も非常に複雑です。ロシアの個人から借金を集めるためのスキームは実際にはうまく機能しないため、支払い方法の問題が原因でアカウントがすぐにブロックされる可能性が高くなります（カードから1ドルを差し引くことは妨げられません） ） その後、サポートサービスに連絡し、プロバイダーに追加情報（公共料金の支払いなど）を送信する必要があります。







各データセンターのリソースにクォータを割り当てるために、Amazonはサポートリクエストを作成して約1日待つ必要があります。 サポートを支払った場合、リクエストの速度は向上する可能性がありますが、この点は確認しませんでした。 基本無料プランでは、制限のリクエストと請求関連の問題の解決のみが可能です。

構成は完全に同一ではありませんが、Amazon EC2では既製のテンプレートを使用してのみインスタンスを作成でき、4つのvCPUでより近いバリアントを見つけることができませんでした。 テストプロセスでは8 GBのRAMを使用しなかったため、RAMの量の違いは結果に影響しませんでした。 Amazonにはヨーロッパにデータセンターがありますが、主にディスクサブシステムの計算能力とパフォーマンスを確認するため、仮想サーバーの場所も重要ではありません。



ping



最初に、グローバルネットワークのロシア語圏の両方の仮想マシンの可用性を確認します。実験の純度については、両方の実験的なサンクトペテルブルクに関連する外部データセンターで動作する別の仮想サーバーから確認します。 pingユーティリティを使用して、メッセージサイズが64バイトと1キロバイト（100パケットを送信）のホストの応答時間を調べます。 Atlex仮想マシンの応答時間は、それぞれ48.328ミリ秒と48.436ミリ秒でしたが、パケットの転送は損なわれていません（0％の損失）。 オハイオデータセンターのAmazon EC2マシンの場合-126.748および126.846。 実験の純度を高めるために、フランクフルトのAmazonデータセンターに別のマシンを作成しましたが、応答時間は46.005および46.1ミリ秒でした。 つまり、ここチェコ共和国とドイツのAWSでは、指標はほぼ同じです。



GeekBench 4



人気のあるパッケージは、暗号、整数、浮動小数点、メモリのいくつかのカテゴリに分けられた20以上のテストを実行します。 圧縮操作を使用し、JPEGで動作し、HTML、SQLiteを解析します。 これはWindows用の最も包括的なテストの1つですが、ディスクサブシステムをカバーしていません（このため、CristalDiskMarkを使用しました）。 仮想マシンに驚きはありませんでしたが、テストでは期待されるパフォーマンスが示されました。















Amazon EC2仮想マシンを使用すると、すべてが非常に興味深いものになりました。 このタイプのインスタンスのプロパティはBroadwellプロセッサを指定しますが、GeekBenchは頑固にHaswellを示しました。

CrystalDiskMark 6



この人気のあるベンチマークは、キュー深度1、8、および32で順次およびランダムの書き込み/読み取りテストを実行します。スクリーンショットの結果は、両方の構成のストレージでソリッドステートドライブを使用しているにもかかわらず、仮想マシンのディスクサブシステムがはるかに高速であることを表しています。 実際、しきい値はさらに高く、テストプログラムはそれに到達できませんでしたが、負荷分散も行われています。異なるクライアントは互いに干渉しません。

Microsoft SQLを使用したテスト



Microsoft SQL Server 2017のパフォーマンスを評価するために、業界標準の1つと考えられているTPC-Hテストを使用しました。これはDBMS開発者自身が積極的に使用しています。 このテストは、ビジネス指向の柔軟なクエリと並列データ変更のセットです。 大量のデータを分析し、高度な複雑さのクエリを実行し、重要なビジネス上の質問への回答を提供する意思決定支援システムの動作を示しています。 複雑すぎるため、TPC-Hのすべての仕様に準拠していませんでした。PDFには150ページを超える長いテキストがあります。 テストプロセスを簡素化するために、 データベース用のBenchmark Factoryの無料バージョンをダウンロードしました。 TPC-Hは、指定されたスケール係数パラメーターを使用して8つのテーブルを生成することを想定しています。 1に等しく、データベースサイズは1.54 GBでした（テーブルは1.18 GBと364.96 MBのインデックスを占有していました）。



このテストは、複雑さの度合いが異なる22のSQLクエリで構成され、4人の仮想ユーザーが3回円で実行します（合計264のSQLクエリ）。 Benchmark Factory for Databasesの最新バージョンでうれしい驚きは、テストデータを生成してデータベースにアップロードできることでした。 以前は、TPCからdbgenユーティリティのソースコードをダウンロードし、Visual StudioのWindowsでコンパイルする必要がありました。 このテストでも、すべてが驚くことなく行われました。

値札



Amazon EC2価格表はこちらにあります 。 クラウド仮想サーバーのタイプによって価格は非常に大きく異なる可能性があります-短期テストではオンデマンドのインスタンスを使用しましたが 、たとえば、1年間リソースをすぐに予約したり、Amazon EC2の無料のコンピューティングリソースを利用したりする準備ができていれば、他のオプションもあります Windowsを搭載した仮想クラウドサーバーt2.xlargeでは、ユーザーに1時間あたり0.2266ドル+税がかかります（価格表のすべての価格には税金と手数料は含まれていません）。 マシンが1か月間（〜720時間）動作する場合、ユーザーは約163.15ドルを税金なしで支払います-月の日数は異なるため、正確な金額を計算することはできません。 EC2インスタンスは1秒単位で課金され、最小使用期間は60秒です。 同様に、EBS SSD（gp2）ボリュームの専用ストレージは、60秒の最小使用期間で毎秒課金されます。 オハイオデータセンターの1 GBのストレージは月額0.10ドル（100 GBあたり10ドル）ですが、専用のIOPSを使用した支払いスキームもあります。

私たちの仮想サーバーは、1か月あたり4208ルーブル、または1時間あたり約5.56ルーブル（約0.1ドル）かかります。 テストは標準のIntel XeonプロセッサーE5-2620v4に基づいていましたが、最近、データベース用に高性能（3.8 GHz）の新しいプロセッサーを追加しました。
