まず第一に、ロシアでのQAとテストは非常に近いものであり、両者の違いはほとんどないため、これらの用語を一般化したことに注意してください。

この記事では、インタビュー対象者とインタビュー対象者の両方から、インタビュープロセスの一般的なポイントを共有したいと思います。 両側から一度に見てみましょう。 また、このプロセスの技術的な側面にも特別な重点を置きたいと思います。なぜなら、原則として、候補者を選択する上で重要だからです。 ここでこのトピックが気に入ったので、何かを追加してQAの側に持っていきたいです。

これらは個人的な観察にすぎず、レビュープロセスが双方にとってより理解しやすいものであるようにしたいのです。 どこでも独自の詳細がありますが、いくつかのポイントを強調することができます。 さらにすべては、初心者またはすでに経験のある専門家に帰することができますが、専門家に帰することはできません。





あなたは何をしましたか？



ほとんどの場合、最初の質問は次のフレーズになります：「今何をしていますか？」、およびオプションで、「なぜあなたは仕事を変えたいですか？」。 ここでは、通常の就業日を驚かせることなく、単に論文を発表することをお勧めします。 あなたの知識を「殺す」ことを試みないでください、そしてさらにandな略語でそうしてください。 反対側に座っている人は、現在の（以前の）会社で働くことができます。 作業中のプロセス、テスト計画の作成、テスト、検証などを教えてください。 製品の品質向上にどのような役割を果たしているのか、またプロセスをより効率的にするためにどの程度正確に役立ったのかを述べておくと特に役立ちます。



技術知識



すでに冒頭で説明したように、インタビューを通過するプロセスで重要な役割を果たすのはこの知識です。 そしてすぐに「聴衆からの声」：「もし人が実際に頭が良くて有能であるが、彼が向きを変えて何もしないなら許されなかったら？ 彼は良いです、彼はすべてを学びます！」 はい、もちろん、これは常に起こります。技術知識を評価するプロセスでは、さまざまな要因を考慮に入れることはできませんが、1つの簡単なルールに沿ってガイドする必要があります。 UDPトラフィックから、「はい、面白くありませんでした」と主張し、「警報」です



会社によって、現在のプロジェクト、位置、質問の領域は異なる場合がありますが、80％のケースで尋ねられている柱はまだ残っています。 だからここにある：



SQL



ここで必要な最小要件は、次の形式のリクエストです。

Select * from my_table where id=2;





はい、一部の人にとってはすべてナンセンスで、他の人にとっては文字のセットです。 ここで、1つのポイントだけを検討する価値があります。アクティブに使用するためにSQLが必要ない場合、JOINやCONCAT、または関数などを介してSQLを追跡しても意味がありません。 それから、むしろ「Google」というフレーズを聞きたいと思います。

+提案されたすべてのコマンド、名前、代替データベースの主な違いに正確に名前を付け、また、たとえそれを知らなくても議論を始めたとしても、答えについてあまり考えない場合は、ここになります。



Linux



したがって、それはさらに悪い。 さらに多くの質問があるかもしれませんが、残念ながらLinuxでは、ごく少数の「人」が自宅で働いています。そのため、よくある質問がいくつかあります。

コマンドの説明：

ls -la | grep *.doc >> 1.log





左から右へ。 それはひどく単純に見えるかもしれませんが、それにもかかわらず、「1つの記号が2つとはどう違うのか>>」というサブ質問は、人をup迷に導く可能性があります。

+チーム全体を説明し、より良い解決策を見つけます。 LinuxとWindowsオペレーティングシステムの主な違いを理解します。 特権、グループ、ファイル操作はすべて大きなプラスになります。 多くの人がABSをよく知っているわけではないので、ABSの知識は一般に超優れています。



ネットワーク



「すべての仕組み」プロセスの基本原則。 GET、POST、HTTP、FTP、SMTP-これらの恐ろしいカットはあなたを怖がらせるべきではありません。

+ DNS、ARP、NAT、DHCPの仕組みを理解する。 たとえば自宅で、「戦闘状況」で「何か」を調整する経験がある。 問題を恐れず、ドキュメントを使用できることを示しています。



プログラミング



ここで最も痛い質問。 プログラミングを熟知している人がQAで働くことは非常にまれです。 ここでは、プログラミング言語の基本的な知識が必要です。 サイクル、再帰、OOPの知識。 一般に、HTMLやXMLでさえ良いほど、これはコードを直接操作できることを示します。 QAが自動化に「関与」しているが、ほとんどがPython、JS、Javaの場合、構文の知識は非常に望ましく、独自のスクリプト/プログラムを作成した経験は大歓迎です。

+ OOPと関数型言語の違いに関する明確な知識。 自動テストの開発/サポートの経験。



テスト中



ここでは、残念ながら長い間、愚かに話すことができます。

エレベータ、ペン、飛行機のテストは、特にさまざまな方法、レベル、およびテストモデルを既に使用している場合には、完全に無意味です。 ここで私が出会った最高の例の1つは、ログインフォームのテストです。 タスク自体は次のとおりです。

ログイン、パスワード、チェックボックス、ボタンがあります：



実際、ここではすべてが簡単ですが、「耐久性」などについてシステムをテストする必要があります。最も重要なことは、両側からテストを説明することです。 ユーザーとしてのテスト-マウス、ブラウザー、キーボードがあります。 スクリプトを使用したテスト（特別なツール）-1000件以上のリクエストを連続して送信したり、非常に大きな値を入力したりするなど。ここで適用できるアプローチを理解できることが重要です。 また、「ボックス」をテストする場所もあることに注意してください。



英語



そのため、英語は国際標準として認められています。 この場合、雇用主とその特定のプロセスが役割を果たします。 英語の最良の尺度は、テキストの80〜90％をロシア語に翻訳し、たとえば、通常の就業日を英語で伝える能力だと思います。 したがって、ばかげたアクセントを描いて、各単語を5分間考えないでください。 できるだけシンプルで明確な、これは英語を話す能力（IMHO）です。



論理に関する質問



このような問題を解決することはあなたにとって大きなプラスになることを皆さんはご存知だと思います。 つまり、経験と知識のマスクが削除され、雇用主が本質的に必要とするまさにそのアルゴリズムが残ります。 私自身は、特に優れた技術面接の後、論理的なタスクでポイントを実際に見ていませんが、それでも...

一般的なタイプのタスクは次のとおりです。

  • 2枚のコインを裏返すために、そのうちの1枚がナットから落ちる確率。
  • モロニック！ 輸血タスク、学校へようこそ。
  • 幾何学的な問題。
  • 操作を実行するためのステップ数に関するタスク：ボール、バケツ、ボート。


ここで、最良の方法は思考の列を示し、あなたのすべての論理的なステップを表明することであることを覚えておく価値があります。



個人的な資質



趣味、興味、娯楽について、常に質問する価値があります。 特別なポイントは、プロとしても個人としても、成長と発展を望んでいる候補者の希望に注意することです。 言語とプログラミングの両方の言語を学びたいという願望について尋ねてください。 私の謙虚な意見である優れた専門家は、常に新しい技術を研究し、既存の知識を改善するよう努めるべきです。 同僚や仕事、キャリアや誠実さなど、あなたの選択を必要とする質問もあるかもしれません。 ここで私は正しい答えはないと思います...



「私たちに質問がありますか？」



ここではすべてが簡単です。 面接が良かったと思うなら、プロセス、見込み、チーム、ボーナス、出張の可能性について尋ねてください。 あまり質問しないでください。組織の公式ウェブサイトで必要な情報をすべて入手できます。 健康保険について尋ねると、「小さな」企業では多くの「落とし穴」を学ぶことになりますが、手遅れになります。 スケジュールについて尋ねてください。 それはあなたに合ったものであり、「残業」、どのくらいの頻度で、どのような理由で、どのように支払われるのかを尋ねるべきです。 面接がうまくいかなかったと確信している場合は、どの方向に進むべきか、どの分野の知識があなたにとって最も強くないか、この組織で何をする必要があるかを必ず尋ねてください。 後者が最も重要であると確信しています。なぜなら、これが前進に必要な推進力を与えるからです。



ここでの別の瞬間は、人間の、またはむしろ感情的な瞬間です。 そのため、たとえば、20〜30分間待機してから、謝罪せずに会話を開始するように「強制」することができます。 繰り返しますが、この会話には最初は否定的な「意味合い」があります。 また、彼らはあなたとあなたの知識の両方を直接in辱することができます。 これらの場合、その人のストレス耐性がチェックされます。なぜなら、この瞬間に、インタビューされている人の「マスクの下」を見ることができるからです。 私の経験では、そのような会社からできるだけ離れようとしました。 ここでは、最も有能な人材を雇用する必要があるのは彼女ですが、何らかの理由で彼らはしばしばそれを忘れているため、同社は魅力的な立場にありません。
