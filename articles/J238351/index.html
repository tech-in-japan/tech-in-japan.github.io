ヨタ禁止を回避して、電話からインターネットを配布する

導入する代わりに。



Samsung Note 3 LTE（N9005）を使用しています。 オペレーター、最近から、ヨタ。 このデバイスでのLTEはまだ正式にサポートされていないため、本日、電話でrootを作成し、LTEを手動でオンにすることにしました。



w3bsit3-dns.comリソースからの指示に従って受け取ったルート。 /system/etc/lteon_netlist.xmlファイルに次の行を追加することにより、LTEがアクティブになりました。

whiteLists numeric = "25011" ltestate = "PS / CSFB" operator = "Yota" country = "Russia" defaultMode = "GWL"




設定で所有者のアクセスポイントを設定します-LTE。

次に、デバイスを再起動し、設定で新しいLTEモードを選択できます。



ネクタイ



家に着くと、有線のインターネットはそこにあるように見えますが、ページは開きません（金曜日と土曜日のプロバイダーの通常の動作）。 そして、私はLTEを持っていることを思い出しました！ ポイントモードをオンにし、ブラウザを開いて、次のことを行いました。



大きな青い絵






どうする？



開発



不吉な青いウィンドウのボタンをクリックする前のpingの試行は失敗しました。 ポートが切断されないことを意味します。 そのため、トラフィックはOSIモデルの3（ネットワーク）レベルでカットされます。

ラップトップが電話を介して接続されている場合、ヨタはどのように電話パッケージとラップトップパッケージを区別しますか？ Wikiを開いて、パッケージの構成を確認しますか？ 材料の分析と研究中に、パッケージ間の違いが明らかになりました、それはTTLです！ TTL = 64のパケットは電話からYotaに到着し、TTL = 63のパケットはすでにラップトップから到着します（1 ttlは電話を通過します）！



クライマックス



電話がわだかされているので、/ system / etc / sysctl.confファイルにnet.ipv4.ip_default_ttl = 63を設定し、「モバイルデータ」をオフにしてオンにします。 私たちはラップトップでブラウザに行き、出来上がり！ すべてが開き、速度は低下しません！



あとがきの代わりに



よた、貪欲にならず、通常のインターネットを提供してください。



UPD： Android 4.3、4.4で動作しなかった場合：/data/data/com.android.providers.settings/databases/settings.db sqlliteエディターに移動し、値tether_dun_requiredを確認します。0である必要があります。
