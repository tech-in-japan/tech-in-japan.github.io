Ubuntu、MS Exchange 2007のメールクライアント

はじめに



Ubuntuを職場のメインOSとして使用して、MS Exchange 2007への接続の問題に遭遇しました。POPおよびIMAPプロトコルを無効にし、唯一の抜け穴はWebインターフェースであるOWAでした。 それでは、この問題の可能な解決策を検討してみましょう。 Ubuntu 11.04は、実験的なオペレーティングシステムとして使用されます。



進化交換



最も一般的なオプションの1つは、 evolution-exchangeパッケージを使用することです。これにより、Exchange OWAを介してMicrosoft Exchangeに接続できます。もちろん、この機能が有効になっている場合はWebインターフェイスです。 この接続方法の設定は非常に簡単です。進化では、メールアドレスを入力し、OWA URLを〜https：//owa.domaine.ru/owaの形式で入力し、ユーザー名を入力するよう求められます。 ユーザー名を入力するとき、1つの警告があります。進化は通常の方法でそれを理解しません-domaine / login 、ユーザー名は形式-login@domaine.localで入力する必要があります。 ログインしようとします：ドメインアカウントのパスワードを入力した後、 login@domaine.localのパスワードを入力する要求が表示され、 ログインのパスワード、つまりメールボックスの名前を入力する別の要求が表示されます。ここでの失敗は認証エラーです。 これはおそらく、evolution-exchangeがMS Exchange 2000/2003でうまく機能するためですが、Exchange 2007があります。



MAPIプロバイダー



次に、MAPI Providerまたはevolution-mapiパッケージの使用方法を検討します 。 Evolutionでアカウントを設定するときに、サーバーの種類-Exchange MAPIを設定します。 サーバー自体は、Outlookの構成時と同じ方法で指定する必要があります。 ユーザー名とドメインを入力する必要があり、認証と設定は完了です。 しかし、クライアントが起動すると、すべてが非常に悲しいことが判明します。進化は、名前のキリル文字を含むフォルダーを見ません。 Ubuntu 10.10の同様のセットアップでは、フォルダー名が疑問符付きで表示されていましたが、表示されていました。 しかし、連絡先リストとカレンダーを表示し、メールを送信することもできましたが、残念ながら、これでは十分ではありません。



DavMail



私の意見では、最も効率的な別の方法は、 DavMailを使用することです 。 これは、メールクライアントとMS Exchangeの間の層です。 DavMailはMS Exchangeに接続し、ローカルIMAPおよびPOPサーバーを作成します。 この場合、メールクライアントとしてMozilla Thunderbirdを使用しました。 DavMailを構成すると、OWAアドレスと接続ポートを指定することになります。 Thunderbirdでは、 localhost 、および対応するポートをSMTP、IMAP、またはPOPサーバーとして指定する必要があります。



カレンダーを使用するには、 Lightning拡張機能をインストールする必要があります。 カレンダーもDavMailを介して接続されます。以前の設定と同様に、 localhostをサーバーとして指定し、デフォルトのポートは1080です。私の場合、カレンダーは期限切れのセキュリティ証明書を呪われています。 Thunderbirdの設定を開き、 [詳細設定]-> [証明書]タブ-> [証明書の表示]-> [サーバー]タブに移動し、エラー時に以前に保存した証明書をインポートします。



次に、LDAPポート経由でアドレス帳を接続する必要がありますが、成功しませんでした。 連絡先を操作するには、 Inverse SOGo Connector拡張機能をインストールする必要があります。 次に、アドレス帳を開き、 ファイルメニュー->作成->リモートアドレス帳を開き、連絡先の名前とURLを入力します。 この場合、連絡先はCaldav HTTPプロトコルを介してDavMailに接続されるため、接続ポートはカレンダーを接続するときと同じ（1080）になります。その結果、URL文字列は次のようになります。 /連絡先 これでThunderbirdのセットアップが完了しました。 驚いたことに、ThunderbirdとDavMailのバンドルはMS Exchange 2007で完全に機能します。また、ThunderbirdとDavMailはクロスプラットフォーム製品であるため、Windowsのメールクライアントとして適合することに注意する必要があります。
