ダミーのためのプレイボーイ/ Vogue VNCモニター

こんにちは、紳士。



私の友人は、 この記事を読んだ後、Ubuntu 11.04のマシンを搭載したプレイボーイプレーヤーにVNCクライアントをインストールするための「写真」を含む手順を説明するように頼みました。 そして、多くの所有プレイヤーは、通常、用語の誤解やLinuxの使用スキルの不足にぶつかって使用できませんでした。

この資料で多くの疑問が消えることを願っています。



結果の写真：







私たちは何を持っています：
  • 実際にプレーヤー自体、私はプレイボーイからいた
  • きれいなUbuntu 11.04を搭載したコンピューター（サーバーがあり、vmware仮想マシンでは動作しませんでした）
  • 動作するVNCサーバーを備えたマシン（プレーヤーにLinuxをインストールするローカルサーバーに配置できます）
  • 基本的なコンソールスキル
1.最初に、ソフトウェアを準備します。 これを行うには、Ubuntuがインストールされているコンピューターの管理者権限が必要です。 アクションはUbuntuで実行されます。



2.アーカイブへのリンクがあるページの中央にあるサイトingenic.grindars.orgにアクセスします。 ページを読んでから、アーカイブをダウンロードしたディレクトリに移動して解凍します。



3. gitでコピーするためにコンピューターを準備します。このために、コンソールに移動して次のように記述します。

sudo apt-get install git





コンピューターは、sudoを初めて使用する前にパスワードを要求します。

インストール中に質問がある場合は、どこでも同意します（Yを入力してEnterキーを押します）。



4.ここで、git jzbootからコピーします。これは、コンソールで次のように記述します。

git clone git://git.whitequark.org/jzboot.git





その後、jzbootフォルダーがユーザーのホームフォルダーに表示されます。



5.次に、jzbootをコンパイルするためのコンポーネントをインストールします。このために、コンソールで順番に記述します。

sudo apt-get install libusb-1.0-0-dev

sudo apt-get install libreadline6 libreadline6-dev





同様に、Yボタンですべての質問に同意します。



6.次に、jzbootをコンパイルします。このために、コンソールで次のように記述します。

cd ~/jzboot

make -C src READLINE=1





その後、コンパイルプロセスが実行され、スクリーンショットのように新しいファイル（jzboot）が表示されます。





7.ここで、プロセスを簡単にするために、解凍したフォルダーをポイント2からユーザーのホームフォルダー（/ home / USERNAME /）に転送し、フォルダーの名前をlinに変更します。スクリーンショットのように、jzbootフォルダーの横にあります USERNAMEはユーザー名です。





8.プレーヤーを取り出し、ミニUSBケーブルを接続します。古いHDDのケーブルを使用しました。 コネクタに穴が開いています。



9.閉じた状態（プレーヤーがオフ）で、一番上のボタン（流行の唯一のボタン）を押し、ボタンを押したままにしてプレーヤーを開きます。



10.プレーヤーを閉じずに、コンピューターに接続します。 これで、プレーヤーは「ファームウェアモード」になりました。



11.ファームウェアプロセスを開始します。このために、コンソールに入力します。

cd ~/lin/

sudo ./linux-bootstrap.sh -b /home/USERNAME/jzboot/





すべてが正常であれば、スクリーンショットと同じ情報がコンソールに表示されます。



したがって、プロセスの完了を待っています。完了まで何も触れないでください。



12.主要部分は完成しました。これでプレーヤーはLinuxの最新バージョンの1つになります。



13.次に、VNCクライアントをプレーヤーに配置します。 しかし、最初に、コンソールに入力された次のコマンドを使用して、コンピューターのプレーヤーでインターネットを共有しましょう。

sysctl net.ipv4.ip_forward=1

iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE







14.これで、SSHを介してプレーヤーに接続されました。 すべてがうまくいった場合、プレーヤーはコンピューター自体に接続し、DHCPを使用してIPアドレスを割り当てる必要がありました。 ほとんど確実にアドレス192.168.20.1になります。 だから、コンソールで書く必要があります：

ssh root@192.168.20.1





これで、コンソールからプレーヤーに接続します。最初に接続するときに、このデバイスを信頼するかどうか（キーを受け入れるかどうか）を尋ね、もう一度同意し（はい）、Enterを押します。パスワードについて尋ねられたら、Enterを押します（パスワードは空です）。



15.ここで、VNCクライアントをプレーヤーに配置し、すべて同じ場所に順番に書き込みます。

opkg update

opkg install directvnc linux-fusion







16. VNCサーバーが構成されたので（ここでは考慮しません）、コンソールに順番に書き込みます。

modprobe fusion

directvnc 192.168.20.2:0 -p VNC





ここで、192.168.20.2はVNCサーバーのアドレスです。0は接続するモニターの番号です（これはモニターのないサーバーなので、0があります）。VNCパスワードはVNCパスワードそのものです。





それだけです 接続されていますが、 すでに書かれているように、実行中のVNCセッションの設定をさらに掘り下げる必要があります。x11vncに-scale 0.5を登録し、必要以上の許可を得ましたが、今のところこれで十分です。
