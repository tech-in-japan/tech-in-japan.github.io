未来は悪くも悪くもなく、私たちと同じです

  • ハイブリッドOS、ブラウザ、ソーシャルネットワーク、または「検索」を中心としたタンバリンとのダンス。 WebブラウザーはWeb 1.0（作成された）のタスクに完全に対応し、Web 2.0では既に曲がって見え、デザイナーやプログラマーにいくつかの困難をもたらします（「ブラウザー戦争」）が、Web 3.0はその品質でそれらを完全に破壊します以前にそれらを表示した場所（ Web 3.0はWebブラウザの終Brをもたらすでしょうか？ ）。 サイトのインターフェースは、ブラウザとオペレーティングシステムからますます借用します。 このような技術は、 jQueryのAeroWindowプラグインです。 Windows Aero 、 ContextMenuプラグイン 、 Mozilla Aurora のスタイルでウィンドウを作成することは広く普及します。 Windows、スタートパネル、デスクトップ、ショートカットの作成、作業面、履歴、ブックマークなどへのドラッグ-これらはすべてサイトで利用できます。 ドキュメントの編集、クラウドストレージ、プログラミング、デザイン、ビデオおよび写真の操作...ブラウザおよびOSのすべてまたはほぼすべての機能がWebに実装されます。 Web 3.0テクノロジー上に構築された別のサイトは、実際にはブラウザーとOSを置き換えます。 では、ブラウザを起動して1つのサイトだけにアクセスするのはなぜですか？ また、ブラウザサイトの概念、正確にはブラウザソーシャルネットワーク（google chrome-同様の概念ですが、Googleは検索エンジン、地図、または翻訳者にのみ訪問者の注意を引き付けることができますが、使用者を訓練することはできません最初は、Google +、そしてすでに-他のすべての人にとって、訪問者が「いいね」の代わりにg +を入力することに慣れることはできません）。 Facebookはまもなく（？）「検索」を開始します（ Facebook Delves Deeper Into Search ）。 ソーシャルネットワークで情報を検索するのは本当に難しいですか？ はい、それらはそのために作成されました-情報と人を検索することをより便利にするために。 「検索」は、ネットワーク外の情報を検索し、ウェブサイトやその他のネットワークの情報を監視するために必要です（実際、ソーシャルネットワーク、特に大規模で専門化されていないネットワークでは、情報の検索が難しくなります）。 このようにfacebookが検索の概念を理解していれば、facebookブラウザは世界で最も人気のあるブラウザになります。 そして、Googleにとってこれは不幸なことです（数年にわたる苦痛の後の哀れな死）。 マイクロソフトはこの分野で独自の決定を下すことができました-「ソーシャル」検索（ GoogleのSearch Plus Your World（ハンズオン）を打ち負かすためにNew BingがFacebookとTwitterを呼び出す ）と検索ソーシャルネットワーク（ So.cl ）の概念を実装する幅広いMicrosoftが再生を利用できるようになり、「検索」ソーシャルネットワークを立ち上げました 。 検索を制御するのは誰ですか-Webを制御します。 Web 1.0時代の最も人気のあるサイト（この記事では20世紀を取り上げません）は、yahoo.com、google.com、 yandex.ruでした 。 最も人気のあるWeb 2.0サイトはFacebook.comです。 すべてが期限内に関連しています。 以前は、サイトの検索は関連性がありました。 ソーシャルネットワークでの関連検索が可能になりました。 それでも、次の記事の1つを「検索」のトピックと、WebとOSの統合のトピックに当てたいと思います。 続けましょう。
  • サイトは消滅します-ソーシャルネットワークのみが残ります。 あなたにとってどのようにしたらいいのかわかりませんが、私にとっては1日として明確になりました。 今日、ネットワークテクノロジーを含む最も人気のあるサイト。 ブログはフォーラム、雑誌、新聞にひっそりと置き換わり、評判の良い出版物の記事でさえも、コメントできる場合にのみ読者の関心を引く。 Web 2.0 =ソーシャルネットワーク。 そして、この方程式に同意しない人は誰でも昨日生きています。 Web 3.0では、ソーシャルネットワークは消滅しますが、すぐには消滅しません。 議論したいですか？ 次のトピックの1つで、このアイデアを詳しく説明します。
  • マウスを殺します。 時々、コンピューターサイエンスの先生と知り合ったケースを思い出します。 最初のレッスンで、彼は私たちを情報学の2つの戒めに厳lyに捧げました。「ミシャは尻尾で引き裂かれることはできません...そして鍵の手はうめきません」。 私たちへのこの願いは、クラスによって非常に楽しかったです。 しかし、IT分野は郷noの場所ではありません。 マウスと標準キーボードはますますユーザーのニーズを満たします。 そして、悲しいことに、マウスを破壊しなければなりません。 Web 3.0サイトでは、まず、タッチパネル（画面自体はタブレット、スマートフォン、独立したタッチパネル、または2つのタッチパネル）を使用して制御に焦点を合わせます。 Web 3.0サイトでサーフィンをするためのラップトップの概念は、2つのタッチパネル（または1つのタッチパネルと従来のモニター）を備えたラップトップです。1つはスクリーンとして機能し、もう1つはコントロールパネルとして機能します。 パネルはカスタマイズ可能です。 キーボード、タッチパッドを呼び出して、パネルをミラー画面にし、ウィンドウをドラッグし、アイコンをパネルにドラッグして、パネルを通過できます。 そして、私は遠い未来について話しているのではありません。 私は、来年または2年に関わる技術と概念についてのみ書きます。 Touchéテクノロジーなどに言及しないのはなぜですか？ 同じ理由で。 一部のテクノロジーは時代を先取りし、人々のニーズを満たしています。 そして、そのような技術は、Web 4.0の概念の枠組み内ですでに関連性を獲得します。
  • フェンスに書かれているもの。 私たちは皆、フェンスや住宅（および非住宅）の家の壁に無料で真珠を寄付したいという若い才能の欲求を知っています。 これは、創造性にあまり自由がなかった「古代の芸術家」の症候群です。 そのような数少ない自由の1つは洞窟壁画です。 しかし、あなたは自分が細いパイプを通って強い圧力で水が流れることを知っています。 まあ、それは十分な話です。 そのため、Web環境では、「古代の芸術家」の症候群は、コメントに対する飽くなき渇望として現れます。 そして、神格化の頂点はウィキペディアです。 テキストを編集する機能を備えたサイト（ 「Truth in Numbers？」 ）。 しかし、次は何ですか？ Web 3.0は、ユーザーに任意のサイト（Web 3.0）でテキストを編集（追加、削除、マーカーでマーク、テキスト内にコメントを残し、テキストの後、エピグラフを挿入、名前を変更、エラーを修正、著者にメッセージを残す）機能を提供します。 当然、これらの修正や破壊的な攻撃は、テキストの作成者にのみ表示され、変更の実装を決定するか、スパムをブロックします。 ただし、一部の人にとって最も興味深い、おそらく最も楽しいのは、テキストの著者の希望に関係なく、すべての変更がそれらを行った人のために保存されるという事実です。 そのため、一部の人にとっては「フェンス」はきれいになり、他の人にとっては落書きになります。 しかし、それは同じ「フェンス」になります。
  • そして、あなたは同じものを持っていない、翼なしでのみ？ あなたはサイトのデザインが好きではありませんか？ 青にうんざりしていませんか？ （すべてのデザイナーが青に夢中になっているわけではないように思えますか？）あなたは鋭い角と直線に怒っていますか？ Web 3.0では、これは問題になりません。 すべてのサイトが表示される個別の統合デザインを設計するか、ページまたはサイトごとに個別のデザインを指定できます。 もちろん、このためにHTMLとCSSを知る必要はありません。 メッセージが左下隅にポップアップ表示されることに満足していない場合は、都合の良い場所を選択できます。 帽子が必要かどうか-それはあなた次第です！ あなたはすべてを黒で、または白に白で欲しいですか？））。 デザイナーは確かにそのような革新に喜んではないでしょう。 正直に言うと、Web 3.0の時代では、WebプログラミングとWebデザインの分野で労働市場が崩壊するのを待っています。 靴工場がこの都市に建設された場合、都市Nの職人の靴職人はどうなりますか？ 全員が労働者として雇われ、残りは再訓練されます。 このトピックについては、後で詳しく説明します。
  • 友人と同志。 インターフェースは、可能な限り使いやすくカスタマイズ可能です。 そして、これはすべて自動的に行われます。 これはコンテキストインターフェイスです。 Web 3.0は、あなたの選択、好み、興味を記憶します。 ニーズを考慮してプログラムが提供され、検索はニーズに合わせてカスタマイズされます。 すべての人々は異なっています。 1つはWeb環境でのナビゲートが簡単で、もう1つは難しいです。 設計変更機能が必要なものもあれば、まったく必要ないものもあります。 これがWeb 3.0インターフェースの重要なポイントです。それは、常にあなたが望んでいる方法です。 あなたが彼の能力を使用する準備ができていない場合、彼はあなたのためにすべてを行いますが、いつの間にか。
  • しかし、Web3Dはどうでしょうか？ これらの技術はすでにWeb 4.0の概念に属します。 同じことがProject Glass-Googleの未来のガラス、または「ターミネーターのように感じます」 、 未来のガラスです 。 これはすべて、Webの次の進化段階で私たちを待っています。 Webの進化のパラダイムは、新しい技術の合計ではなく、現代の技術を通じてユーザーが実現したニーズの合計で表現されます。 Webの進化は、ユーザーとインターネット技術の相互作用の進化です。 そして、最新のテクノロジーではなく、まったく新しいユーザーのニーズを実現できるのであれば、これは進化の新しいラウンドになります。 新しいテクノロジーを通じて「古い」ニーズを満たせば、これも進化ですが、既存のパラダイムの枠組み内です。 これについては後で詳しく説明します。
