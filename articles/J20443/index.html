怠WindowsなWindows管理者向けのスクリプト

しばらく前に管理者（怠け者）だった彼は、ルーチンを自動化し、さらに怠lazを許すいくつかのスクリプトを書きました。 私はもう管理に対処しておらず、スクリプトを広めました。誰かが役に立つかもしれません。



1.ローカルおよびADでのコンピューターの説明の自動インストール



会社にはコンピューター名の標準があり、COMPUTERxxxx（xxxxは数字）のようなものでした。ヘルプデスクによると、最初の診断とリモート接続用のコンピューターを人の名前で見つけるのは難しく、一般ユーザーがコンピューター名を決定する方法を見つけるのに長い時間がかかりました。 ソリューションはスクリプトの形で提供されました。





setcompdescr.zip



起動後、指定されたOUのすべてのコンピューターは完全に機能せず、ログインしているユーザーに応じて説明を設定します。



*スクリプトは、コンピューターの管理者権限とADのコンピューターの説明を変更するためのアクセス権を持つアカウントから実行する必要があります。

** Windows 2000では正常に動作しません。ログインしているユーザーがローカル管理者でない場合、Windows Vistaではテストされていません。



開始する前に、スクリプトを構成する必要があります。



2.「交換フォルダー」の自動クリーニング



会社の各部門には独自のフォルダーがあり、他の部門にはアクセスできませんでした。 部門間での情報交換（プレゼンテーション、テキスト、および音楽）のために、彼らは可能な限り倒錯しました。 共有用のフォルダと、情報がそこに存在しないようにするためのスクリプトを作成する必要がありました。



clear_exchange_v2s.zip



スクリプトロジック：作成/変更の日付から10日が経過したファイルは、交換フォルダー（_deletion_queue_）内の特別なフォルダーに配置され、その後7日後に「削除」フォルダーからファイルが完全に削除されます。



すべてがスクリプト内の変数によって構成されます。



3.「お気に入り」をリダイレクトする



マイクロソフトは、「お気に入り」以外のほぼすべてにリダイレクトを提供しています。 なぜ謎なのか。 会社には、すべてのデータがサーバーに保存されるという厳格なルールがありました。 もちろん、「選ばれた者」の喪失は致命的ではなく、不快です。 そのため、「お気に入り」をサーバー上のフォルダーにリダイレクトするスクリプトを作成しました。



redirect_favorites.zip



PS。 スクリプトは「自分用」に書かれているため、使用する前に、重要ではないものを設定して確認してください。
