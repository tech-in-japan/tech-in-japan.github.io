ElasticがX-Packソースコードを開きます

ご注意 翻訳者。 X-Packは、ELK製品専用の拡張機能です。







オープンなコミュニケーションからフリーソフトウェアまで、オープン性は常にElasticの精神に基づいています。

したがって、独自のX-Packコードを開きます。 X-Packには、セキュリティ、アラート、監視、レポート、分析、機械学習に関連する機能が含まれています。







なぜこれを行うのですか？



最初に、Elasticsearch、Kibana、Beats、LogstashといったElasticスタックを拡張する独自の機能セットとしてX-Packを作成しました。 監視などの一部の機能は無料でした。 たとえば、安全性と機械学習の一部が支払われました。







当社は、オープンソースと商業的利益の組み合わせの上に構築されています（詳細はShayの記事を参照）。 X-Packコードの発見は、開発を加速し、コミュニティの関与を高めるはずです。 誰もがコードを帰属、コメント、研究できます。







GitHubで何が変わりますか？



プライベートX-Packリポジトリのコードは、対応するパブリックリポジトリElasticsearch、Kibana、Beats、およびLogstashに移動されます。 Elasticsearch、Kibana、Beats、LogstashコードのApache 2.0ライセンスは変更しません。 そして決してしません。 各リポジトリに新しいX-Packディレクトリを作成し、Elastic EULAライセンスの下で配布します。 このライセンスにより、いくつかの派生的なアクションの属性を指定できます。







このようにして、異なるgithubリポジトリの一定の同期に関する問題を取り除きます。 また、アセンブリ、テスト、およびリリースを高速化します。 さらに、今では誰もが問題を作成して追跡できる1つの場所があります。













これはユーザビリティにどのように影響しますか？



バージョン6.3以降、すべての無料のX-Pack機能（監視、検索プロファイラー、Grokデバッガー、Elastic Maps Serviceのズームレベル、専用APM UIなど）は、Elasticsearch、Kibana、Beats、およびLogstashの標準アセンブリで提供されます。







電子メールによる登録、個別のインストール、クラスターの完全な再起動など、すべての制限を削除します。 これにより、開発の迅速な開始が簡素化され、追加機能をすぐに使用できるようになります。 その結果、テクノロジーのより効率的な使用が可能になります。







Apache 2.0ライセンスでオープンソースのみを使用する場合は、アセンブリの「-oss」バージョンを提供します。







OSSFL：生命のためのオープンソースソフトウェア



オープンソースを信じており、オープンソースへの貢献は変わらないでしょう。

多くの企業は、成長するにつれて閉鎖的になります。

しかし、このアプローチは私たちをもっとオープンにする自由な選択です。

オープンソースコミュニティとともにビジネスを発展させます。 これは、誰もが対等な立場で開発、貢献、テストできることを意味します。 「コミュニティ」または「エンタープライズ」バージョンはありません。







Apache 2.0ライセンスの下からコードを取得することはありません。 さらに開くだけです。







ユーザーにサポートされています



Elastic製品のダウンロード数は2億を超え、多くの人が当社製品を気に入っています。

パブリッククラウド、プライベートクラウド、独自のハードウェア、またはこれらすべての組み合わせであるかどうかにかかわらず、最大限の使いやすさを確保するために取り組んでいます。







私たちは、あなたが将来信頼できる素晴らしい技術を開発しようとします。







他に質問がありますか？



良いリクエストが大好きです。 翻訳者注：クエリ-検索クエリ、しゃれ

一般的な質問に対する回答を次に示します。







Elasticsearch、Kibana、Beats、Logstashはまだオープンソースですか？







はい 既存のApache 2.0コードのライセンスは変更しません。

X-Packコードは商用ライセンスでのみ開き、対応するリポジトリに追加します。







Apache 2.0ライセンス下にないコードがある場合、Elasticsearch、Kibana、Beats、およびLogstashリポジトリに密輸する方法を教えてください。







前と同じ。 寄稿者のライセンスとプロセス全体は変更されていません。

現在、X-Pack機能に起因するもののみが変更されています。

Elasticsearchを監視するために追加のチャートまたはディスプレイが必要ですか？ または、LogstashのGrokデバッガーの改善ですか？ これで、この機能の作成/改善に参加できます。







X-Packコードが現在開いている場合、これはそれが現在無料であることを意味しますか？







いや 監視、タイルマップ、Grokデバッガー、検索プロファイラーなど、多くのX-Pack機能は無料です。 一部のX-Pack機能は有料で、ゴールドまたはプラチナのサブスクリプションで購入したライセンスが必要です。







X-Packは現在オープンソースですか？







オープンソースライセンスは、Open Source Initiative（OSI）によって明確に定義されています。







バージョン6.3以降、X-PackコードはElastic Licenseの下で利用可能になります。







しかし、「オープンソース」の伝統的な意味ではありません。 Elastic LicenseはOSI定義を満たしていません。 ただし、X-Packコードとやり取りするためのモデルは、既存のgithubリポジトリでコードを学習し、問題を作成し、リクエストをプルする機能など、他のElasticオープンソース製品と同じです。
