Atlassian Jira Softwareでの一般的なスプリント

みなさんこんにちは！



この記事では、Atlassian Jira Softwareの共有スプリントについて説明します。



たとえば、 ここの公式ドキュメントを読んだ場合、一般的なスプリントのようなものはありません。



ここで最初に一般スプリントという用語に出くわしました 。 この記事では、一般的なスプリントとは何か、またそれらと一緒に暮らす方法について説明します。



共有スプリントはアトラシアンJira Softwareの重要な概念です。共有スプリントの知識は通常予期しないものであり、何らかの問題が発生したように見えるためです。 しかし、これはそうではありません。一般的なスプリントがどのように機能するかを知っていれば、それらを自分のニーズに使用できます。



この記事では、一般的なスプリントの外観と、通常のスプリントと区別する方法を説明します。 また、それらの機能についても説明します。



この記事のすべての例をJira Software CloudおよびJira Software Server 7.12.3で試しました。



一般的なスプリントとは何ですか？



一般的なスプリントは、複数のボードに表示されるスプリントです。



たとえば、次のようなボードのスクリーンショットがあります。



スクラムボード ：







SCRUM2ボード ：







SCRUMおよびSCRUM2ボードには、SCRUMスプリント3と呼ばれるスプリントがあることがわかります。このスプリントは2つのボードに表示されます。 それは一般的なスプリントを見ることを意味しますか？ いや Jira Softwareは、同じ名前の2つの異なるスプリントを持つことができます。



スプリントIDを確認する方法は？



共通のスプリントが目の前にあるのか通常のスプリントにあるのかを理解するには、これら2つのスプリントのIDを調べる必要があります。



次のようなスプリントIDを確認できます。







上の赤​​で強調表示されたボタンの上にマウスを置くと、sprintId = <number>で終わるURLが表示されます。 この番号はスプリントIDです。 この場合、2つのスプリントのIDは異なります。つまり、前に共通のスプリントではなく、2つの通常のスプリントがあります。



一般的なスプリント



次に、このスクリーンショットを見てみましょう。







スクリーンショットには、同じ名前の2つのスプリントがあるSCRUMボードがあります。 そして、赤で強調表示されたスプリントには、SCRUM2ボード上のスプリントと同じチケットが含まれています。 SCRUMとSCRUM2のボード上でこのスプリントのIDを確認すると、IDは一致します。つまり、共通のスプリントがあることを意味します。



なぜ2つのボードに同じスプリントがあるのですか？



ボードに表示されるチケットは、ボードに定義されているフィルターによって選択されます。 フィルタは、[ボード設定]-> [全般]に移動して表示できます。 ボード用のフィルターは次のようになります。



スクラムボード ：



project = SCRUM OR priority is not EMPTY ORDER BY Rank ASC







SCRUM2ボード ：



project = SCRUM2 ORDER BY Rank ASC







SCRUMのフィルターは、SCRUMプロジェクトからのチケットだけでなく、優先度が満たされているJiraインスタンス内のすべてのチケットも選択することがわかります。つまり、SCRUM2プロジェクトからチケットを選択します。 したがって、SCRUM2プロジェクトのチケットは、SCRUMボードとSCRUM2ボードの両方に表示されます。 したがって、チケットの1つでスプリントフィールドを埋めると、このスプリントは2つのボードに表示されます。



スプリントはボードから作成され、作成元のボードへのリンクが含まれています。 これを行うには、たとえばrest / agile / 1.0 / sprint / sprintIdを実行し、スプリントが作成されたボードを確認します。 この場合、次の結果が得られます。 



 {"id":5,"self":"https://alex.atlassian.net/rest/agile/1.0/sprint/5","state":"active","name":"SCRUM Sprint 3","startDate":"2018-09-23T06:33:01.266Z","endDate":"2018-10-07T06:33:00.000Z","originBoardId":3,"goal":""}


originBoardId = 3、これはSCRUM2です。 つまり、スプリントは元々SCRUM2ボードで作成され、スプリントからのチケットはSCRUM2ボードとSCRUMボードの両方にあるため、SCRUMボードに表示されました。



一般的なスプリントはどのように動作しますか？



スプリントが表示されているボードで一般的なスプリントに変更を加えた場合、その変更はすべてのボードで表示されます。



たとえば、SCRUMボードのスプリントの名前を変更すると、SCRUM2ボードの名前も変更されます。 SCRUMボードでスプリントを閉じると、SCRUMボードで閉じます。



ユーザーが通常遭遇するのは、この動作です。 彼らはスプリントで動作し、突然スプリントが閉じます。 どのチームもそれを閉じませんでした。 その結果、スプリントが別のボードから閉じられ、フィルターがこのプロジェクトからチケットを選択したことがわかりました。



一般的なスプリントはどのように使用できますか？



複数のチームがあり、各チームに独自のプロジェクトがあるとします。 各チームはボードを作成し、このボードで作業します。 すべてのチームのスプリントを見たい。 どうすればこれができますか？



  • すべてのボードを共有するように依頼できます。適切なボードに移動すると、スプリントが表示されます。
  • すべてのチームプロジェクトからチケットを選択するボードを作成できます。 この場合、すべてのスプリントに関する情報が1つのボードに表示されます。これにより、すべてのチームスプリントを表示するための労力が少なくなります。 しかし、ボード上のスプリントを使用したアクションは、チームボードでも同じ変更を伴うことを忘れてはなりません。
