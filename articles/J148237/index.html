Firefox 15はOpusフリーオーディオ形式をサポートしています

Firefox 15（ベータ版）はOpusフリーオーディオ形式をサポートしています。 このコーデックは最近、IETFによる標準として検討対象として受け入れられており 、他のコーディングおよび音声伝送標準よりも品質が優れています。







Opusには、高品質と低レイテンシの独自の組み合わせがあります。



既存のすべてのオーディオコーデックは、2つのグループに分けることができます。



汎用コーデック（高遅延、高品質）

  • MP3
  • AAC（すべてのタイプ）
  • ヴォービス


音声コーディング（低品質、小さな遅延）

  • G.729
  • AMR-NB
  • AMR-WB（G.722.2）
  • Speex
  • iSAC
  • iLBC
  • G.722.1（すべてのオプション）
  • G.719


上記のコーデックには、最高品質と低遅延の両方で、あらゆる種類のオーディオエンコーディングをサポートする柔軟性がありません。



テストで確認：

64 kbpsでは、 OpusはHE-AACやVorbisよりも優れたサウンドを実現します

64 kbpsでは、 Opusは96 kbps MP3のように聞こえます



Opusは、6 kbpsおよび256 kbpsで音楽を放送するのにも適していますが、広帯域では、品質の「聴力損失なし」の圧縮を提供します。 コーデックは、帯域幅条件の変化に応じて、異なるビットレートの圧縮に動的に切り替えることができます。



テストでは、Opusは音声圧縮に最適なコーデックであることが証明されました。つまり、高品質のVoIPだけでなく、オーディオブックやポッドキャストにも最適です。



Opusオーディオファイルは、他のオーディオファイルと同じ方法でWebページコードに統合されます：<audio>タグを介して。



<audio src = "ehren-paper_lights-64.opus"コントロール>



opusencユーティリティは、ファイルのエンコードに最適です（異なるプラットフォームのバイナリおよびソースコードで利用可能）。



FirefoxはOpusサポートを実装した最初のブラウザーですが、他のブラウザーには大きな疑問があります。たとえば、Chromium開発者はこれを行う予定はありません 。 Opusサポートは、gstreamer、libavcodec、foobar2000などのさまざまなメディアプレーヤーに実装されています。
