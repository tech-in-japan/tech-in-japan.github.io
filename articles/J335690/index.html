すぐに使用できるサービス：ServiceNowを「60秒で」構成します





今日は、ITサービス管理（ ITSM ）とこれらのタスクを自動化する特殊なソリューションについてお話します。 ITプロセスの編成、実行、制御に役立つシステムは多数ありますが、特定のITSMシステムであるServiceNowに焦点を当てます。 これは、当社独自のクラウドプラットフォームであるTechnoserv Cloudでお客様に提供するソリューションです。 この記事では、ITスペシャリストのスタッフ全員が関与することなく、ServiceNowを使用して会社全体のデスクトップサービスを迅速に構成する方法を説明します。









プロセス自動化システム



どの会社でも、ほとんどの作業は、常に同じ方法で実行され、実行者による即興演奏を必要としない日常業務を実行することです。 たとえば、保険会社は今週何をすべきかを月曜日の朝の総会で決定しません-保険を販売し、保険事件に関するメッセージを受け取り、それらを調査し、被害者に補償金を支払う必要があることを知っています。 彼らが何をするかを考える企業のカテゴリーがあります-これらはスタートアップです。 しかし、彼らはビジネスを作成する段階でのみ機能し、ビジネスが獲得されるとすぐにスタートアップが完了します。







合理的な会社は、従業員に問題を最善の方法で解決することを要求します。 これを実現するには、特定の問題を解決する最善の方法が従業員がどのように見えるかを正確に制御する必要があります。

いくつかの問題を解決した結果は社内または部門内に残り、一部は外に出て消費者に届きます。 最善の方法で問題を解決するということは、必要な結果がより多く得られ、結果自体がより良くなり、消費者がより満足し、結果を得るために費やされるリソースが少なくなることを意味します。







大声で申し訳ありませんが、これらのアイデアは一連のISO 9000標準の形で国際社会によって策定されました。これらの標準では、結果は製品と呼ばれ、製品を作成する一連のタスクはプロセスと呼ばれます。 管理者が従業員に確立されたプロセスを実行することを要求し、彼ら自身がこれらのプロセスを改善する方法を考える組織管理方法は、プロセスアプローチと呼ばれます。







プロセスアプローチは、最も尊敬されている管理専門家と密接に関連しています; E.デミングとP.ドラッカーに言及するだけで十分です。 プロセスの整理、実行、および制御を支援するコンピューターシステムは多数あります。 この記事では、当社のTechnoservクラウドプラットフォームでお客様に提供するServiceNowシステムの機能の一部を提供します。







ServiceNowを選ぶ理由



それでは、なぜServiceNowなのでしょうか？ Technoservクラウドからこの製品を顧客に提供することに決め、現在その機能について話しているのはなぜですか？ ServiceNowは現在、ソフトウェアSaaS（Software as a Service）の5つの主要なグローバルメーカーの1つであり、ガートナーのアナリストによると、ITSMソリューションの分野で主導的な地位を占めています。 メーカーは、ITサービスのライフサイクルを自動化し、プロジェクト、人材、リソース、リスクなどを管理するための単一のクラウドプラットフォームを提供しています。







ServiceNow製品の主な利点は、共通のユーザーインターフェイスと1つのデータモデルで構成される単一のクラウドプラットフォームに基づいて構築されていることです。 これは多くの統合されたモジュールではなく、他のベンダーから異なる時期に購入される可能性があります。 ServiceNowがサードパーティ製品を購入する場合、テクノロジーを購入し、プラットフォームでそれらを再実装します。 これにより、プラットフォームレベルでのアップグレード、更新、適応が可能になります。







より重要な製品の利点：







•スケーラビリティ、つまり、必要に応じてメインアプリケーションの操作を損なうことなく新しいユーザーを追加できる、小さなスケーリングステップ。

•ServiceNowはもともとクラウドプラットフォームであったため、クラウドの詳細に関連する多くの技術的な問題（セキュリティ、スプリットの可視性、統合、監査、ログなど）はそこで簡単に解決されます。

•設定が簡単でドキュメントが公開されているため、サポートに集中でき、顧客はシステムを設定してプロセスを自動化する機会が与えられます。 したがって、少数の人々が多数の顧客にサービスを提供できます。







ServiceNowはすでにインストールされており、あなたを待っています



ServiceNowはクラウドシステムです。すぐに使用を開始できます。 従来のシステムをインストールして起動するのに比べて、数週間節約でき、サーバーやその他の機器の設計、購入、インストールおよび構成に費やすことができます。 実際の接続よりもはるかに多くの時間が承認と法的エンティティ間の関係の避けられない署名に必要です。







したがって、システムを構成するには、いくつかの簡単な手順を実行する必要があります。







  1. クラウドサービスを接続します。
  2. ログイン用のユーザー名とパスワードを取得します。
  3. システムセットアップウィザード（図3）を実行します。これにより、ステップごとにシステムを構成できます。


データ統合およびダウンロードツール



プロセス管理システムでは、会社は最初にプロセスに関する情報ではなく、自分自身に関する情報を入力します。名前、組織構造、従業員のリストです。 ServiceNowでは、このステップに数分かかります。システムはActive Directoryまたは別のすぐに使用可能なLDAPソースと統合できます。









図1段階的なセットアップウィザード 接続設定







Active Directoryと統合されたシステムはユーザーを認識し、ユーザー名とパスワードを指定する必要はありません。Windowsに入るときにそれらを指定するだけで十分です。 統合するものが何もない場合、ServiceNowはファイルからユーザーと部門のリストをインポートできます。







プロセスを説明するためのグラフィックエディター



プロセスは、目標を達成するための一連のアクションです。 ServiceNowでは、プロセスアクションを実行するためのリスト、シーケンス、および条件がグラフィカルエディターで指定されます。 たとえば、雇用プロセス。









図2エディターでのワークフロー







ServiceNowプロセスの構成は、従来のプロセス制御システムとは異なり、プログラミングを必要としないため、ほとんど時間がかかりません。 上の図のプロセスは、10分未満でセットアップされます。 システムには予期しないことはありませんので、追加設定は必要ありません。 プロセスがエディターに描画されるとすぐに、エグゼキューターはそのプロセスで作業を開始できます。







プロセスアプローチ標準には、継続的なプロセス改善が必要です。 新しいバージョンのプロセスが古いバージョンで作成されたオブジェクトを処理できない場合、従来のプロセス制御システムは多くの場合、この要件に干渉します。 ServiceNowの特徴的な機能は、プロセスバージョンのサポートです。 プロセスで処理される各オブジェクトは、作成されたプロセスのバージョンを認識し、このバージョンで定義されたルールに従います。







事前構成済みプロセス



ServiceNowは、インシデント管理などの既知のプロセスを自動化するように既に構成されています。 明確な情報を提供するだけです。

ドットチェーン



上記の段落は、ServiceNowシステムがサイトビルダーに似ているという印象を与えます。「キューブ」のセットがあり、その一部のプロパティは「オン/オフ」ティックで構成され、左右のステップは不可能です。 この印象は間違っています。 ServiceNowシステムとキューブのセットにより、完全なプログラミングが可能になります。 言語は通常のJavaScriptであり、オブジェクトを操作する強力なメカニズムがあります。 ベンダーはそれをドットウォーキングと呼びます。技術的なソリューションに名前を付けるのが慣習であれば、「ドットのチェーン」が定着します。







ドットウォーキングは、関連するオブジェクトにアクセスするためのメカニズムです。 たとえば、サービスマネージャーの電話番号service.manager.phoneを取得できます。 なぜこれが必要なのでしょうか？ たとえば、高優先度のインシデント（incident.priority <= 2）が重要な（incident.ci.priority == 1）サーバー（incident.ci.type ===“ server”）に発生したことを彼に通知する場合。 そして、ここで（このことについてエンジニアに警告するために）インシデントは、ITに精通していない（incident.user.tech_level == 0）だけでなく、神経質な（incident.user.touchy == true）ユーザーからのものであることがわかります。 。
