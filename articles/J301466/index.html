C ++の世界でニュースを追跡する方法

C ++の知識を継続的に補充しようとして、集中できる情報を取得するためのいくつかのチャネルを自分で決定しました。 他の知識分野にも適用できるかもしれませんが、C ++に焦点を当てようとします。 それらについてもう少しお話ししましょう。



  1. 思慮深い読書 -基本的な本を読み、構造化された、有益な本。 これには長い集中力が必要であり、あなたは単に眠りに落ちて何も覚えていないため、疲労状態で読むことはできません。 電子リーダーをまだお持ちでない場合は、必ず購入してください。
  2. 読みやすい -動機、生産性、ソフトスキルに関する本のようなそれほど深刻でない文献を読む。 これはすべて、一部のプログラマーによって過小評価されている無駄です。 これらの本はそれほど集中する必要はありません。 はい、これは完全にC ++言語に関するものではありませんが、それは何かをする方法とそれを行う方法に関するものです。 寝る前によく読んでください。
  3. ポッドキャストは 、あるものから別のものに切り替える必要があるとき、または買い物中にうまく機能します。 彼らは予期しない何かを見るのに役立ちます。 彼らは、聞いた内容を聞いて理解する能力を訓練します。これは、有用なスキルになる場合があります（たとえば、会議に参加するため）。 英語が母国語でない場合は、用語と名前の発音を聞くことができます。 CppCastまたはSoftware Engineering Radioを聴くことをお勧めします -彼らのトピックが幅広く取り上げられていることに驚きました。 最初のエピソードは2006年にリリースされました。ポッドキャストの半分はソフトウェアエンジニアリングの歴史と半分の理論（パターン、問題、解決策など）で構成されています。
  4. ビデオと講義 -CppCon 、 CppNow 、 会議C ++ 、初心者向けACCU会議 。 驚異的な量の興味深いレポートがあります。 ポッドキャストとは異なり、これには視覚的な注意と集中力が必要ですが、結果としてより多くを学ぶことができます。 週に2〜5本のビデオを視聴することはかなり達成できると思います。

    ヒント ：オーディオ/ビデオの速度を上げる-脳はすぐにこの速度に慣れ、より短い時間でより多くを聞くことができます。
  5. ディスカッション、フォーラム、ツイッター -情報交換のこのチャネルは、最小限の集中力を必要とします。 ディスカッションやTwitterフィードをすばやくレビューして、面白いものを探すことができます。時間をつぶすだけです。 何かが見つかった場合は、後で読むためにリストに追加できます。 これには何が役立つのか-多種多様な人々、意見、アプローチ、タスク、ソリューション。 非常にランダムな情報で脳を破壊する良い方法。



    たとえば、C ++の世界の重要な開発者を追跡するためのCore C ++のリストです。 そして、Facebookにはいくつかのページとグループがあり、Jens Wellerの良いレビューもあります 。



    C ++に関する議論については、 Redditに関するセクションがあります（翻訳者のメモ： Habrahabrにもハブがあります ）。 そしてもちろん、C ++標準化委員会グループ
  6. ブログと記事 -通常、平均的な記事を読むのに5〜10分かかります。 より深い記事がある場合は、「いつ集中するか」を読むためにマークすることができます。 私の観察によると、週に5〜10の良い記事があり、それらすべてを読むことはそれほど難しくありません。




重要な注意事項





  1. インタレストグループ -対面のコミュニケーションに代わるものはありません。 そのような会議では、面白い人に会ったり、友達を作ることさえできます。 お住まいの地域でグループが見つからない場合は、作成する必要がありますか？ 私はそのようなグループを扱っており、特に問題はありませんでした。
  2. スポーツ -あなたに活力を与え、認知スキルの発達に貢献します。 はい、誰もがそれを言いますが、私はすでに定期的に練習を開始するのにどれだけ近いかについて「すでに」です。 明日から始めます、約束します。
  3. 教える -何かを学びたいなら、教え始めてください。 多くの可能性があります-同僚のためにプレゼンテーションをしたり、興味のあるグループでプレゼンテーションをしたり、ブログを書いたり、コースを教えることができます。
  4. 考える時間 -これもしばしば過小評価されています。 過去を振り返り、既に行われたことと計画を考え、そしてなぜあなたの計画が既に行われたセクションにないのかを考えることができるのは良いことです。
  5. 素晴らしいもののリスト -たとえば、 Awesome C / C ++やAwesome Modern C ++などがあります。




他の問題と同様に、C ++の世界を追跡する方法は、目的を達成する方法によって異なります。脳と身体の最適なトレーニング方法と能力を判断することが重要です。 情報を取得する上記の方法はすべて、さまざまなタイプのデータを提供するため、重要だと思われます。 少なくともそれらの存在を認識し、時々、この方法またはその方法を使用していない理由を自問する必要があります。



情報を取得する方法をこのように分類するだけで、毎日のタスクと長期トレーニングの両方を計画するのにすでに役立つことがわかりました。 たとえば、1時間の空き時間があれば、twitterやブログを読むべきではありません。 いくつかの記事または本の章を読む方が良いでしょう。 しかし、Twitterとフォーラムをチェックして、コーヒーを順番に待つことができます。 または、たとえば、C ++カンファレンスのビデオを長時間見ていなかった場合、今夜は1時間待たなければならないかもしれません。



また、現在のステータスを調整する価値があります。 私は時々読む時間がありますが、私は真剣な本にはあまりにも焦点が合っていません。 この場合、私はビデオを見たり、ポッドキャストを聴いたりします。 リラックスして元気いっぱいなら、ソーシャルネットワークで時間を無駄にしないでください。 すでに長い記事を読んでください。それはもう半年の間、「読み取り」セクションにほこりを集めてきました。今がその時です。 いつも、繰り返します、いつも（！）現在読んでいる本を手元に置いてください。



また、プログラミングが実践によって改善されていることを忘れないでください。 読み取りは、コンパイル、デバッグ、改善、破損、修復、テスト、測定する必要があります。そして、ある程度の確率で、議論されたことを理解したと言えるでしょう。 練習がなければ、読書は役に立たない。



家族と仕事も重要です。 また、十分な睡眠をとることも重要です。 とても重要です。 そして、コンピューターから休憩します。 上記のすべてに費やされる時間のおおよその見積もりは、1日あたり30〜40時間です。これで十分です。 そして、残りの時間は、あなたがやりたいことを何でもします。
