過去と現在



消費者コンピューターの時代は、コンピューター時代の夜明けに研究大学で生まれた巨大なマシンから始まるのではなく、70年代半ばから、コンピューターが定義上プログラマーではない、または直接仕事ではなかった人々の家にやって来たコンピュータ技術に関連するもの（例：作家、会計士、学童など）。 おそらく最初の人気のあるそのようなコンピューターはApple IIであり、それは完全に組み立てられた形式で最終消費者に供給されました。 ユーザーは、それをアセンブルする方法や、自分の（コンピューター）パフォーマンスのためにインストールする必要のあるソフトウェアを考える必要はありませんでした。



パーソナルコンピュータの開発のさらなる歴史は、生産性とディスクスペースの増加に向かっていました。 この電力の最適化は、コンピューターコンポーネントの小型化とともに常に行われています。 コンパクトさは、ラップトップやタブレットコンピューターなどのポータブルコンピューター（ラップトップ）を作成する可能性をもたらしました。 彼らの時代は80年代の始まりです。



システムユニットの小型化により、ベアボーンシステムとネットトップが登場しました。 さらに、後者は、モニターの背面に掛けることができるほど小さなスペースを占有します（最新のモニターでは、このための特別なマウントがあります）。 もちろん、この小型化は現在、これらのシステムの計算能力の低下につながっていますが、過去3〜5年でこの容量は平均的な消費者のすべてのニーズに応えるのに十分であり、ブラウザ、オフィス製品、HD品質のフィルムはそれほど多くありません。



パソコンリソースの平均的な消費者と他のすべての人を区別する価値があります。 平均的なユーザーは、最新のコンピューターゲームをプレイせず（特に最大設定で）、プログラムをコンパイルせず、画像や3次元シーンをレンダリングせず、プロのグラフィックソフトウェアプロセッサーを使用してグラフィックスを操作しません。 これらはすべて、平均を超えるニーズです。



コンピューティングの開発と並行して、電気通信が開発されています。 消費者インターネットの始まりは、メインデータ転送プロトコルが形成され、ドメインネームシステムと最初のブラウザが登場した90年代半ばと考えられます。 将来、消費者向けインターネットの開発は、ソーシャルネットワークと他のインターネットリソースへの統合（これらのネットワークの一部の機能を本来意図されていなかったリソースに移行する）の第1波の全面的なコミュニケーション（パーベイシブコミュニケーション）の開発に関連していました。 この重要な定義を覚えておいてください。



ただし、パーベイシブコミュニケーションという用語は、パーベイシブコンピューティング（詳細については後述）と同様、天井から取られたものではなく、最新の外国の科学記事から収集されたものです。 RuNetでは、現時点ではこのような言葉はほとんど発生しません。



並列コンピューティングの手法と通信ネットワークの開発は、スーパーコンピューター、コンピューティングクラスター、グリッドインフラストラクチャ、クラウドコンピューティングの作成に影響を与えています。 最初のものは考慮しません。 その容量は、平均的な消費者のニーズを満たすように設計されていません。 さらに、2番目、1番目、または他のコンピューティングデバイスに基づいて構築されるものもあります。 後者は、上記のすべてに技術的に実装できます。 新しいミレニアムの最初の10年の終わり近くに現れ始めたクラウドサービスにのみ興味があります。



クラウドの概念は、当然、消費のいくつかのパターン、すなわち、SaaS（サービスとしてのソフトウェア）、PaaS（サービスとしてのプラットフォーム）、IaaS（サービスとしてのインフラストラクチャ）、そして私たちにとってより価値のあるもの、WaaS（サービスとしての職場）を生み出しました。



ごく最近、別のモデルが登場しました-サービスとしてのプロセッサー時間（略語は覚えていません）。



これらのうち、Windows AzureとAmazon Web Servicesが最もよく知られています。 これで、自分のコンピューター、ローカルコンピューターにソフトウェアをインストールすることはできませんが、プロバイダーのWebサイト（Google Docsなど）で直接使用できます。 プログラムだけでなく、オペレーティングシステム（Google Chrome OS）もあります。
