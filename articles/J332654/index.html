1. sockeyeをインストールする



最初に必要なのは、sockeyeをインストールすることです。 DLAMIを使用すると、インストールプロセスは非常に簡単で、コマンドは1つだけです。 



sudo pip3 install sockeye --no-deps


ここで重要なのは、デフォルトではDLAMIのpipがPython 2を使用するため、pipだけでなくpip3を使用する必要があるということです。これは、sockeyeではサポートされていません。 また、依存関係はすべてインストールされているため、インストールする必要もありません。

トレーニング



sockeyeを使用すると、トレーニングプロセスは非常に簡単です。1つのコマンドを実行するだけで済みます。 



 python3 -m sockeye.train --source train.a --target train.b --validation-source train.a --validation-target train.b --output model


トレーニングと検証に同じデータを使用しないでください。 ただし、現時点のスクリプトでは、データを2つのグループに正確に分割していないため、より良い結果（主観的な評価について話している）が得られます。







最後の記事を読んでいると、sockeyeがあなたのワークアウトに適した設定を見つけようとしていることに気付くかもしれません：



  • 辞書サイズ;
  • ネットワーク設定。
  • など...


最適に近い構成は、より高速な（安価な）トレーニングにつながる可能性があるため、これはかなり良いです。 まだ、sockeyeがパラメーターを検索する方法と、このプロセスに費やされるリソースの量を正確に確認する必要があります。



また、sockeyeは、トレーニングプロセスを完了する価値がある時期を正確に判断します。 これは、モデルの品質が最後の8つのコントロールポイントの検証用データで改善されていない場合に発生します。

チャット...



このプロセスでは、トレーニングを停止する必要はありません。新しいウィンドウ（または、新しいSSH接続）を開き、トレーニングが行われている同じフォルダーに移動してコマンドを実行するだけです。 



 python3 -m sockeye.translate --models model --use-cpu --checkpoints 0005


注目したいいくつかの要素：



  • python3-当然;
  • model-学習プロセスがモデルを保存するフォルダーの名前。トレーニング中に指定された名前と一致する必要があります。
  • --use-cpu-これがないと、MXNetはGPUを使用しようとしますが、学習プロセスでまだ使用されているため、失敗する可能性が最も高くなります。
  • --checkpoints 0005-チェックポイントが保存されたときにコンソール出力から取得されたチェックポイント番号。


コマンドの実行後、sockeyeはSTDINから入力を読み取り、STDOUTに応答を出力します。 以下に例を示します。
