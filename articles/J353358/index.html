I / O 2018年モスクワ拡張：招待

春は庭にあります。つまり、技術の巨人の伝統的な会議の時が来ています。 そのようなイベントにはGoogle I / Oがあり、今年はサンフランシスコで5月8日から10日に開催されます。







Keynoteのブロードキャストを単独で見たくないが、すべてのニュースや新しいアイテムについて同僚とすぐに話し合う準備ができている場合は、 昨年のように、Avitoでの共同視聴にご招待します！ GDGモスクワと一緒に開催しているイベント。 会議は、モスクワ時間の19:30に5月8日に始まります。







それまでの間、私たちはこの瞬間を待っています。この投稿では、会議からの主な期待と新製品の発表についての推測を共有します。













ちなみに、10年前の2008年に行われた最初のI / Oの時以来、ほとんどの場合、新しい技術とソフトウェア製品の更新の発表があります。 新しいデバイスのプレゼンテーションは非常にまれでした（たとえば、2012年のNexus 7）が、今年はそうなる可能性があります。 なんで？ それを理解しましょう！







Android P









従来、Keynote I / Oの大部分は、Androidの将来のバージョンに割り当てられます。 3月に、Googleは最初のDeveloper Preview Android Pをリリースしました。アセンブリで公開された計画から判断すると、次の予備バージョンは5月上旬に、つまり、I / Oの数日間に提示されるはずです。













最終リリースのインターフェースへの変更に伴い、システムは多くの場合、必ずしも目立たないが重要な変更を受け取ることが予想されます。







  • バックグラウンドのアプリケーションはカメラとマイクを使用できません-プライバシー。
  • 通話を録音するとき、反対側の加入者は特別な音声信号によって通知されます。
  • スパムコールをブロックするための新機能。
  • Bluetooth HIDデバイスのサポートの改善
  • 網膜スキャナーのネイティブサポート。
  • iPhone Xでの「モノブロウ」のサポート。
  • 柔軟な画面と非標準のアスペクト比-2回起きないように、「その他の設計ソリューション」の将来をサポートします。


予想される革新の中には、非常に思慮深いものもあります。たとえば、家庭内暴力のためのホットライン番号への呼び出しの呼び出しログに隠すことです。







GoogleがOSの次のバージョンの名前を明らかにする可能性があります。 これまでのところ、たくさんのオプションがあります。ここにはパイナップル、ピナコラーダ、パンケーキ、パイ、ペプシもあります。 状況はOリリースよりも複雑です-文字Pのお菓子の選択ははるかに広いことが判明しました。







OSを着用









4年前に導入されたAndroid Wearは、3月からWear OSとして知られるようになりました。Googleはウェアラブルデバイス向けにオペレーティングシステムのブランド変更を決定しました。 名前から「Android」を削除することで、同社はWear OS製品がAndroidだけでなくiOSデバイスでも機能することをユーザーに説明したいと考えています。







ただし、現時点では、Googleのブログ投稿と1つの時計モデルのみのシステムの開発者ビルドのみに満足しています。 Android Wearと比較した変更点の中には、OLEDスクリーンを搭載した時計が電力を節約し、時計が充電器に接続されていない場合にアプリケーションがバックグラウンドで動作しないようにする暗いインターフェイステーマがあります。 Googleはウェアラブルデバイス市場でAppleに追いつきたいと考えているので、Google I / Oの将来のWear OSについて多くの詳細を学ぶ可能性が高いでしょう。







Googleアシスタント









Google Assistantは、17の新しい言語で話をし、ほぼ4ダースの国で提供される予定です。 少なくとも2月の会議で、Googleは2018年のアシスタントの開発計画を共有しました。













新しい市場の中にはロシアがあります。 アシスタントを通じて、同社はスマートスピーカーGoogle Homeを宣伝しようとしていますが、ロシアでは他のGoogleデバイスと同様に販売されていません。 こうしたことはすべて、検索巨人がデバイスストアで最終的にわが国に来るという意図を間接的に確認することができます。そのような噂もありました。 本当かどうかはわかりませんが、5月にI / Oで学びます。







Googleホーム









「鳥と言えば。」 最近、GoogleがHomeの新しい機能、いわゆる「ルーチン」に取り組んでいることがわかりました。 似たようなものには、すでに直接的な競争相手であるAmazon Echoがあります。 この革新により、アシスタントを使用して、別のコマンドで実行される一連のアクションを作成できます。 たとえば、条件付きの「おはよう」コマンドは条件付きライトをオンにし、条件付きサーモスタットは希望の温度を設定します。 これはすべてスマート世帯に関連しているため、国内の状況については、この特異性はコンベンションのレベルのままです。







ただし、1つの家でより深い消費者社会を構築する方法については、GoogleもI / Oについて話します。







4K画面の ChromeOSとChromebook



おそらくここに新しいデバイスの発表があるでしょう。 ChromeOSのソースコードには、解像度が4Kの特定のデバイス（コード名Atlas）への参照があります。 ただし、情報はほとんどありませんが、ラップトップはGoogle I / O 2018で発表される可能性があります。







拡張現実と仮想現実









過去数年間の傾向は、さまざまな眼鏡、ヘルメット、および目用のスロットを備えた段ボール箱の助けを借りて、自分自身の意識を欺くことです。 昨年のトレンドは拡張現実です。 すべては同じで、デバイス画面上のみです。 昨年、それは拡張現実であり、iOS 11とAndroid Oの両方の主要なマーケティング「チップ」の1つになりました。







開発者は、ARKit SDKとARCore SDKをそれぞれ使用してこの現実を実装することをお勧めします。 Googleが拡張現実のためにSDKを更新すると信じる理由があります。機能の面では、ARKitにすでに少し遅れているからです。







VRプレーンで新しいデバイスを待つ必要はありませんが、何かのために、GoogleはLytro VRカメラメーカーを購入しましたか？ おそらく、ベールはI / Oで開きます。







機械学習









テクノロジー企業のもう1つの傾向は、機械学習です。 過去に、GoogleのI / Oは、写真内のオブジェクトとコンテキストを識別するアプリケーションであるレンズを明らかにしました。 その後、LensはPixel 2と「計算写真」で独占的になりました。これは、写真を処理および強化するための機械学習の使用です。







したがって、たとえば、複数ではなく1つのカメラを使用して前景と背景を分離することがわかります。 おそらくこの波で、Googleフォトサービス、Google Cloud Platform、Androidの機械学習のすてきなものが更新されるでしょう。 また、2016年のテンソルプロセッサ（TPU）やPixel Visual Coreなど、新しい回路が成功するかもしれません。

Googleは機械学習市場で優れた地位にあり、そのアプリケーションには多くのシナリオがあります。 1つ確かなことは、トピックに触れることです。







フクシア









Googleが数年間静かに開発してきたオペレーティングシステムは、詳細に成長を続けています。2017年5月にユーザーインターフェイスを取得し、1か月前に機能が制限されたブラウザーバージョンがリリースされました。







Googleの公式な沈黙にもかかわらず、新しいOSへの関心は高まりつつあり、技術出版物はフクシアをAndroidの代替品と呼んでいます。 最後のI / Oで、Android開発の副社長であるDave Burkeが最初に謎のOSについて言及しましたが、それを「初期段階の実験プロジェクト」と定義しました。 このパイロットプロジェクトでのアクティビティの増加と最初の作業ビルドは、I / Oで通知される可能性のあるGoogleの準備を示唆する可能性があります。







驚きとあなたが絶対に期待すべきでないこと



上記で予測されたすべては、何らかの形で予測可能であり、既存の製品およびサービスを対象としています。 しかし、グーグルとグーグルは、誰もこれを待っていない瞬間に驚いた。







ただし、I / Oの更新を絶対に行う必要のない製品があります：ChromeOSソースコードのAtlasラップトップとの類推により、Pixel 3はAndroidコードで点灯します。新しいPixelがI / Oに表示される可能性は低いPixel 2のリリース、および同様のプレゼンテーションの伝統が定着したため、10月から11月にかけてより頻繁に表示されます。
