GNOME 3の改善

こんにちは、Habralyudi！







ディストリビューション（Russian Fedora）を更新した後、GNOME 3で多くの問題に遭遇しました。この記事では、収集したすべての方法を説明しようとします。 誰も気にしない、カットの下で！





クイック起動




これには2つのオプションを使用できます。 1つ目はAvant Window Navigatorです。 この場合、すべてが簡単です。 この種類のパネルが表示されます：



パネルの外観は簡単に変更できます。

2番目の方法： intgat.tigress.co.uk/rmy/extensions/index.html



次のリンクからアーカイブをダウンロードします： Extension 。 ホームフォルダーに解凍します。 ユーザーを再起動すると...クイック起動バーが表示されます:) GNOME 2にあったようなものが表示されます。







アプリケーションへのアクセスがより高速になりました。 スクリーンショットを見る：







デスクトップにファイルを配置する




これにはTweekツールを使用できます。 最初はこのユーティリティをインストールしていませんでしたが、yumを使用してインストールできます。 次に、開きます：Tweekツール->ファイルマネージャー->チェックマークを付けます（それだけです）。

そして、スクリーンショット-Tweak Tool：



その後、デスクトップにあったすべてのファイルが最終的にそこに表示されます。 また、デスクトップからコンテキストメニューを呼び出すこともできます。



さまざまな拡張機能




それらはすべてgnome-shell-extensionにあります。 サイトにインストールすると、すべてが非常に明確になります。

たとえば、System Monitor Extensionはいくつかのコマンドでインストールされます。



mkdir ~/git_projects

cd ~/git_projects

git clone git://github.com/paradoxxxzero/gnome-shell-system-monitor-applet.git

mkdir -p ~/.local/share/gnome-shell/extensions

cd ~/.local/share/gnome-shell/extensions

ln -s ~/git_projects/gnome-shell-system-monitor-applet/system-monitor@paradoxxx.zero.gmail.com

sudo cp ~/git_projects/gnome-shell-system-monitor-applet/org.gnome.shell.extensions.system-monitor.gschema.xml /usr/share/glib-2.0/schemas

cd /usr/share/glib-2.0/schemas

sudo glib-compile-schemas .







ただ一つの注意：私はGnome Shell Weather Extensionを持っていません。 インストール後、Gnomeシェルは起動を停止しました。 一般に、私は彼との問題をまだ解決していません。 それ以外はすべてうまくいきます！
