初心者向けのoDesk

なんで？

一般的に、この投稿のアイデアは、以前フリーランスの本格的な経験がなかった私が、oDeskを真剣にマスターしようと決めた瞬間に思い浮かびました。 はい、誰も知らない場合、oDeskは世界最大のフリーランスエクスチェンジの1つです。 だから、今年の7月でした。 その時までに、私はすでに半年間公式の仕事を持っていなかった、すべての副業は終わった、新しい深刻な注文はなかった、そしてoDeskは私にとって非常に有望な選択肢のようでした。 通常どおり、アカウントは1年前に「予約済み」で登録されていましたが、この間ずっとアイドル状態になっていたため、最初から始めなければなりませんでした。 同時に、どこで、少なくとも最短で、どのように、どこから始めるべきか、どこか（確かにHabréで！）

たぶん私はひどく見ていました。 しかし、このトピックで私に出くわしたすべてのことは、すぐに価格を曲げる必要はないという事実に帰着しました。小さなものから始める方が良いです。 おおよその見積もり：「1時間あたり10ドルから開始し、時間の経過とともに15ドルになります。」 私はカテゴリー的に15ドルまで成長したくありませんでしたが、時間があっても、もっともっと稼げると確信していました。 それに、膨大な数の質問が心配でした。 プロフィールの記入方法は？ どのプロジェクトに対応しますか？ カバーレターの作り方は？ 誰もがあなたを拒否するとき、あなたはどのようにしてこの最初の注文を得るのですか？

その瞬間、私はすべてが私のためにうまくいったなら、私は間違いなく私が見つけなかった初心者のために同じガイドを書くと決めました。



誰のため？

簡単です。プログラマ向けの記事です。 おそらく、私の経験の一部は、他の素晴らしい職業の代表者、たとえばデザイナーに役立つかもしれません。 より具体的には、この記事はWeb開発者向けですが、これはそれほど基本的なものではありません。 さらに重要なのは、 優れた開発者向けです。 これは重要です。なぜなら、高い確率で多くのことを短時間で学ばなければならないからです。 準備ができていない場合は、タブを閉じてください（はい、Habrokatを開いたのはそうではないことを理解しています）。



準備する

したがって、決定が行われ、アカウントが登録されます。 そもそもあなたの前に出てくる質問は次のとおりです。
専門分野は？
これは重要です。 すべての地域が同じように要求されるわけではなく、すべてが同じように支払われるわけではありません。 私の観察によると、2つの最も有望な方向があります。
  1. iPhoneの開発
  2. Web開発：Ruby on Rails / JavaScript
まだ試したことがないので、最初のことについて明確なことは言えません（しかし、間違いなく試してみて、興味があれば、経験を共有します）。ここでは、2番目のオプションのみを検討します。

Ruby / Railsを選ぶ理由 非常に簡単：Python / Djangoとは異なり-PHPとは異なり、多くの注文は-よく支払います。 この束をまだマスターしていない場合は、気軽に探索してください。 良いプラスは、フロントエンド開発の知識です。もしあなたが怠け者でなく、バ​​ックボーンを習得していれば、これは場合によってはスペシャリストにとって不可欠なものになります。 フロントエンドベンダーの場合は、少なくともサーバー開発の基本を学ぶと、価値が大幅に向上します。

そうそう：Macを買う。 私は本気です。
良い英語が必要ですか？
はい、少なくとも書面で：ビジネス通信は絶対に正確でなければなりません。 経口が望ましいのは 一部の顧客はスカイプ経由での通信を好むが、これはむしろ例外である。 つまり、言葉がまったくない場合、あなたはそれと一緒に生きることができ、潜在的に興味深いプロジェクトを失っているという事実に耐えるだけです。
テストは行われますか？
はい、間違いなく。 さらに、より良い。 もちろん、あなたの専門分野とまったく無関係なのは無意味ですが、間違いなく1つまたは2つに限定する価値はありません。 2つの理由があります。まず、プロジェクト内のプロジェクトのフィルターとしてさまざまなテストの通過がしばしば提案されます（それらに対応することは完全にオプションですが、これは常にプラスです）。 第二に、多くの場合、雇用主自身がプロジェクトに適した候補者を探しており、通常は対応するテストに合格した候補者を探しています。 当然、テスト結果が高ければ高いほど良いのですが、熱狂は役に立ちません。上位30％に入るだけで十分です。 完璧主義者向け-トップ10％。 最初の3行はマニアと詐欺師のためのもので、プロフィールの美しいアイコンに加えて、何も提供していません。
代理店に連絡する必要がありますか？
登録するとすぐに1つまたは2つの招待状が必ず届くため、この質問はすぐに発生します。 私の答えは、それだけの価値はありません。自分でやりましょう。 より自由になり、実際の使用が疑わしいサービスの収益の一部を提供する必要がなくなります。 ちなみに、「独立請負業者のみ」という条件のプロジェクトは、多くの場合見つかります。 反対のオプション-「代理店のみ」-が見つかりますが、はるかに少ないです。
プロフィールの記入方法、カバーレターの書き方
最も重要なルール：いかなる場合でも、oDeskが提供する例を書き留めてはいけません。 「過去5年間にわたって」という文字列で、文字通り2番目の履歴書が開始されます。 今、まったく同じプロファイルと文字に対する雇用者の反応を想像してください。これに、機関がこのテンプレートを悪用したいという事実を追加してください。 あなた自身の言葉で書いて、刻印された言葉遣いを避けてください。 プロファイルの専門化は、明示的に示すのが最適です（たとえば、Ruby on Rails開発者はWeb開発者よりも優れています）。 カバーレターでは、プロジェクトの詳細に注意してください。少なくとも数語で、説明を注意深く読んだことを明確にしてください。これにより、競合他社の無数のテンプレートレターよりも有利になります。 冗長性は役に立たない、ケースに書いてください。 テストタスクがプロジェクトにアタッチされている場合は、必ず完了してください。これにより、雇用される可能性が大幅に高まります。



評判

自由に使えるのは、知識、時間、評判の3つのリソースのみです。 最初の2つはすべて明確である必要があり、3つ目は停止する必要があります。 oDeskでは、評判はおなじみの5つ星評価システムです。 正式には、「5」は並外れたレベルの能力を意味するはずですが、実際にはすべてがいつものように異なり、優秀な専門家からの評価が4.5を下回ることはめったにありません。 プロジェクトを選択するときを含め、これを非常に慎重に実行してください。 顧客の評価ではなく、彼がパフォーマーにどのような評価を与えるかを見てください。これはあなたにとって非常に重要です。 その妥当性を疑う場合は、リスクを冒さないでください。 すでにプロセスに疑問がある場合は、プロジェクトを放棄してください。 通常、これは最初の週に発生します。この場合、作業時間をすべてキャンセルします。oDeskはそれを許可します。クライアントが何も支払っていない場合、彼は見積もりを出すことができません。



はじめに

超高収入の夢は、より良い時代まで去ります。評判のために初めて働くとき、そして実際、勤務時間の増加はカウンターになります。 議論の内容を明確にするために、oDeskで使用されるフリーランサーの「経験」の度合いを段階的に示します。
  • 0時間
  • 1〜99時間（1回限りの注文の場合、0時間と少なくとも1ドルの支払い）
  • 100〜499時間
  • 500-999時間
  • 1000時間以上
最初の注文
最も難しい順序が最初です。 1時間も作業もレビューも1回も行わない限り、彼らはあなたを無視し続けます。事前に準備してください。 この場合、あなたにとって最善の選択肢は小さな1回限りのタスクですが、彼らにとっては深刻な競争があります（奇妙なことに、深刻な長期プロジェクトよりももっと喜んでいる人がいます）ので、我慢してください。 日中に解決する必要のある緊急のタスクに出くわした場合は非常に良いです。この場合、顧客はアーティストを選ぶことに対する選択が少なくなり、しばしば最初に連絡を取る人を雇います。 どのレベルの支払いにも同意しますが、最も重要なこととして、長期契約にサインアップしないでください。

最初の100時間
そのため、最初の行が渡され、次のステップもそれほど遠くありません。 あなたの目標は、最初の100時間を解決することです。 原則として、ここでの戦略は最初の注文とほぼ同じです。あなたはまだ評判のために働いています。 大金を期待しないで、長期プロジェクトを購読しないでください（すぐにプロジェクトを辞めても構いません）。 ただし、注文の取得が簡単になったため、よりわかりやすく表示できるようになりました。 この期間中であっても、低価格セグメント（最大10ドル）での作業は特にお勧めしません。第一に、ほぼ間違いなくスラッシュジョブであり、同僚にインドヌードルコーダーがいます。第二に、契約の全履歴がプロファイルに残ります。 1時間あたり5ドルの仕事は単純に堅調に見えます。
100〜500時間
おめでとうございます、あなたは100時間のバーを越えて、 全く異なる体重カテゴリーに分類されました。 明確にするために、この比率は、登録されたフリーランサーの1秒ごとに少なくとも1時間働き、4分の1だけが100時間働いていました。 ちなみに、すべてを正しく行った場合（つまり、いくつかの専門テストに合格し、評判が4.5以上である場合）、その瞬間から、面接のために潜在的な雇用主から定期的に招待されます。 最低限の快適な支払いレベルを自分で決定するようにしてください-つまり、あなたが低賃金であると感じずにプロジェクトに取り組む準備ができているレベル（たとえば、1時間あたり20ドル）。 それ以下では妥協しないでください。そうしないと、ハッキングを開始します。悪いことは何もありません。

クライアントが交渉しようとするすべての試み、特に頑固な試みを疑うことは価値があります。 たとえ交渉を使い果たした後でも、あなたが快適なレベルの支払いを求めて交渉したとしても、彼はすでにクライアントに不快感を覚えているかもしれません。 この場合、文字通り、作業の10分ごとに正当化する要件を待ちます。もちろん、通信、環境のセットアップ、およびその他の「オーバーヘッドコスト」はほとんど確実に支払われません。
500時間以上
実際、私はまだこのマークに到達していません（記事を書いている時点で420時間働いています）が、最初の2つの場合のように劇的な変化は期待していません。 これはいずれにしても、改めて価格を改定する機会ではありますが。

次は？ 上限は非常に高く、トップの開発者は1時間あたり50〜60ドルを稼ぎますが、努力する必要があります。 チームを作成する必要がありますか？ 自分で決める、それは財政的な要求ではなく、あなたの傾向に依存します。 リーダーの役割に満足しているなら、確かにポイントがあります。 そうでない場合は、スキルを徐々に向上させ、価格を上げ続けます。

まあ幸運-あなたは間違いなくそれを必要とします。
