Picodiについて



Picodiは、幅広い国際的リーチと検索結果での高い地位を持つクーポンサイトから生じる消費者向けのショッピングプラットフォームです。



SEOの専門家は、おそらくクーポンサイトがSEOで最も競争力のあるニッチの1つであることを認識しています。 Picodiの目標は、そのニッチのリーダーに侵入するだけでなく、主導的地位を維持することでもありました。

「移行」戦略とその利点



移行の主な目標は、 マーケティング活動を統合し、メンテナンスコストを削減することでした。 これを踏まえたトラフィックの増加は、このような規模では誰も予想していなかった、楽しいボーナスであることが判明しました。 選択した戦略の主な利点は何ですか？



メンテナンスコストとワークロードの削減



ニッチの詳細に応じて、頻度を問わず、 各サイトは毎週または毎月これらのタスクに直面します 。

  • インバウンドリンクモニタリング
  • サイト構造の問題を監視する
  • サーバー監視
  • ブランドモニタリング
  • Google Search Consoleでのデータの監視
  • グラフィック画像
  • マーケティングキャンペーン


理論的には、これらの手段はすべて、金銭的な観点だけでなく、時間的にも非常に高価です。 そして、実践が示すように、ほとんどのプロジェクトでは、タスクの一部が完全に無人のままになります。 急成長している企業には通常、十分な時間やリソースがなく、これはすべて長い間脇に置かれています。



幸いなことに、ここで説明する戦略は、コストの削減だけでなく、何よりも成長の加速に焦点を当てています。



グローバルマーケティング戦略はローカル市場をサポートします



ローカルサイトと製品自体を扱うチームは、ローカルクライアントにより近くする必要があります。 理論的には。 通常、現実は異なって見えます。小さなチームは、会社の内部ポリシー、限られた予算、開発者、コンテンツ作成者、または他のリソースへの制限されたアクセスという深刻な問題に直面します。 単一の国際チームは、複数の市場で同時に事業を行う企業の典型的な困難に対する万能薬ではありませんが、原則として、そのニーズを「押し進める」ことは簡単です。



すべてのドメインを1つに統合すると、マーケティングの分野での成功をどのように高めることができますか？



ローカル成功はグローバル構造をサポートします



ロシアのPicodiチームが、世界中でユーザーが共有するクールなバイラルコンテンツを作成するとします。 Picodi.com Webサイトに公開され、数千のバックリンクを受け取ります。 このコンテンツと生成されるバックリンクは、ロシアのサイトだけでなく、Picodiの構造全体をサポートしています。



一般的なブランドを1つ導入すると、リンクプロファイルの作成プロセスが加速し、コンテンツマーケティングのコストが大幅に削減されます。



Picodiは何を提供しましたか？



現在、Picodiは単一のマーケティング戦略をリードできると同時に、 グローバル構造は個々の市場での成功からボーナスを受け取ることができます 。 たとえば、トルコのクールなコンテンツは、オーストラリアのサイトの位置を高めることもできます。



競合他社によって分類されたデータを収集するタスクの複雑さ



実際、この考えは見た目よりも単純です。 簡単に言えば、国際的なブランドのSEO戦略を分析することは、小規模なサイトよりもはるかに困難です。

1つのドメインが複数の国を対象としている場合、サイトの位置に影響する要因の数が非常に多いため、 競合他社はそれを分析して戦略をコピーすることはできません 。 そして、これのいくつかの証拠があります：

  • picodi.comリンクプロファイルは、多数のドメインリダイレクトのために分析が困難です。 ほとんどのツールでは、picodi.comのバックリンクの一部のみが表示されます。
  • picodi.comと比較して、あらゆる国のあらゆるドメインを分析してもまったく役に立ちません。
  • ローカルジオレベルでのpicodi.com/XXの位置は、サイトの構造全体の影響を常に受け​​ます。 これにより、競合他社はpicodi.comをローカル市場での直接の競合他社と見なすことができなくなります。

明らかに、このような大規模な構造の動きは常に大きなリスクを伴います。 すべての可能なトラブルを防ぐためにすべての措置が取られました。 また、 経験豊富な開発者 （Picodiが1人だけ）、効果的な先見の明のあるマネージャー（zukaszGwóźdź）、そしてもちろん、SEO機関の技術支援なしでは、この規模の手順を単独で実行することはできないことを理解する必要があります。



ドメイン移管プロセスの説明



そして、私たちはこの長い読み物の最も重要な部分に来ます。 SEOスペシャリストの間では、「 オルカ法 」は非常によく知られており、Googleペンギンアルゴリズムの影響下にあり、最初からやり直す必要があるサイト用に開発されました。 幸いなことに、Picodi.comの場合、フィルターや制裁は課されていません。 さらに、状況をより適切に制御するために、複数のドメインを1つにマージする前にリンクプロファイルと潜在的な技術的問題のクリーニングに集中できるように、一度に1つのドメインを移行するプロセスをストレッチすることにしました。





1.リンクの手動監査



移行の長期的な成功はそれに依存するため、最初のステップが重要です。 多くの場合、SEO機関はこの複雑で時間のかかるプロセスを過小評価しています。 すべてのリンクの監査なしの移行はスムーズに進むことができます。 多くの場合、ドメインのマージはプラスの効果をもたらしますが、実際にはSEO爆弾がサイトの下に植えられます。 さらに、このような複雑な構造とリンクのこのような多様なプロファイルを持つサイトから可能な制裁を削除するプロセスは、本当の悪夢です。



「ジャンク」リンクプロファイルを持つ単一のドメインは、サイト全体に悪影響を及ぼし、トラフィックを破壊する可能性があります。 このリスクは非常に現実的であり、SEOの分野では、移行の失敗後のサイトの転倒が複数回観察されました。 さらに多くのこのようなケースは、不適切なバックリンク管理に関連しています（ Expediaまたはハリファックスのサイトのケースを読むことを強くお勧めします）。 この点で、リンクを手動で分析し、不良リンクを回避することをお勧めします。 可能なすべてをチェックする価値があります。



単一ドメイン（すべてのローミングドメインにスケーラブル）の詳細なプロセスは次のとおりです。

  1. すべてのバックリンクに関する情報の収集（LinkResearchTools、Google Search Console、Ahrefs、Majestic、SearchMetrics、SEO SpyGlassなど）;
  2. LinkResearchToolsを介したバックリンクと投稿の重複排除-リンクデトックス。
  3. すべてのバックリンクの手動監査（最も時間のかかるプロセス）。
  4. Disavowファイルの作成（バックリンクの拒否）;
  5. DisavowファイルをGoogle Search Consoleにアップロードする（olddomain.com用）;
  6. Disavowファイルを拒否されたバックリンクのリストにアップロードします（newdomain.com用）。


ステップ3は非常に時間がかかります。 効果的な検証には、リンク後のリンクの手動分析が含まれます。 ここでさらに進んで、個々のリンクにその性質に関する情報（スクレーパー、ゴミ箱サイト、コメント、リンクディレクトリサイトなど）と1〜5のスケールのスコアをマークできます。



説明されている手順のステップ6は、ほとんどのSEO専門家が失敗するステップです。 ドメインolddomain.com→newdomain.comをリダイレクトすると、これら2つのドメインは1つになります。 つまり、olddomain.comからのリンクは、newdomain.comのGoogle Search Consoleに直接リンクとして表示されます。





2.フルクロール



リンク監査チームがリンクのクリーニングに取り組んだとき、SEO技術チームは、構造全体がクリーンで効率的であることを確認することに焦点を合わせました。

ほとんどの場合、タスクは、内部リダイレクト、404エラー、標準的な正確さなど、 ドメインの移動に悪影響を与える可能性がある典型的なSEOの問題を解決することでした 。たとえば、404エラーのあるページはすべてOnPage.orgを使用して簡単に特定できます。
