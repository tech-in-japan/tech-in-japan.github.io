Astrid Task Managerが閉じます。 代替案の比較

Android用の最も人気のあるタスクマネージャーの1つでの終了イベントは、Habrにはほとんど見えませんでした。 5月に、Yahooはこの製品を購入し、先週8月5日に終了したことについてユーザーにメールを送信しました（ 本文はこちら ）。

この点で、多くの代替サービスがAstridからタスクをインポートするための特別なツールを用意しています。 この記事では、この素晴らしいマネージャーの最も人気のある代替手段を確認します。



そのため、レター自体で提供されている代替手段は、Wrike、Wunderlist、Sandglaz、およびAny.DOです。 WrikeやSandglazなどの製品は、タスクよりもプロジェクト管理マネージャーの方が多いため、考慮しません。 さらに、GTasksは人気のタスクマネージャーでもあるため、GTasksを追加します。TickTickはこの分野の有能な初心者です。

アプリケーションインターフェイスがOSに一致するときに気に入っているため（Holoテーマが好きです）、その印象についても少し説明します。



Any.DO



61512の評価。

評価： 4.5。

マーケットリンク： https : //play.google.com/store/apps/details?id=com.anydo

ウェブサイト： http : //www.any.do/

Astridからインポート：アプリケーションをAndroidデバイスにインストールすると、それ自体がインポートを提供します。



UI要素が大幅に変更された青と白のインターフェイスを持つ興味深いアプリケーション。

インストール直後にインポートの問題が発生しました。 市場からインストールし、開いて登録します。 このアプリケーションでは、Astridからタスクをインポートできます。 同意し、しばらくして結果を確認します。 アプリケーションは、日付を示すブロックでタスクを分割し、「今日」ブロックには15個のタスクがありますが、今日Astridに割り当てられているタスクは2つだけです。 他のすべてのタスクも、理解できないロジックに沿って分散されていました。 また、期日を削除できませんでした。 彼らはそのような原則を持っているようです。

アプリケーションでは、タスクを削除する方法がまだ見つかりませんでした。 実行してから、一括ですべてを削除することができます。

マイナスはWebインターフェースの欠如ですが、サイトは間もなくそうなることを示しています。 これまでのところ、Google Chromeの拡張機能を備えたコンテンツである必要があります。

それで、Androidバージョンは何ができますか：

  • タスクを重要としてマークします。
  • Googleで名前でタスクを検索します。
  • 日付、時刻、繰り返しを設定します。 その後、日付は削除できません。
  • メモを添付します。
  • 共有します。


インポートは正しく機能しないか、独自の雰囲気があります。 個人的に、私は結果が好きではありませんでした。 機能の数により、明らかにAstridレベルに達しません。 私は多くの理由でインターフェースが好きではありませんでした：

  • モーダルウィンドウで名前を編集する。
  • アクションバーの設定ボタンをクリックすると、ある種の不気味な背景が表示されます。
  • ロシアのインターフェースによる多くのアーティファクト。 英語に切り替える方が良いです。
  • Google Nexus 4でタスクアクションアイコンがぼやけて見える。


一般的に、 評決 ：開発者が独自の方法で進んだか、アプリケーションがまだAstridレベルから非常に離れているかのどちらかです。



ワンダーリスト



評価： 21834。

評価： 4.4。

マーケットリンク： https : //play.google.com/store/apps/details?id=com.wunderkinder.wunderlistandroid

ウェブサイト： https : //www.wunderlist.com/

Astridからインポート： http : //astrid.wunderlist.com/



このアプリケーションは市場でも非常に人気があり、私が理解しているように、iOSに触発された独自のデザインを持っています（丸みを帯びたエッジ、凸面のタスクダイ） AndroidおよびWebアプリケーションは、同様のスタイルで作成されます。

ここでも、インポートには驚きがありました。 インポート手順の後、Webクライアントに移動すると、次の2つの画像のいずれかが表示されます。

  • 空のリスト。 はい、インポートが既に完了したという通知がメールで届きましたが、1つのタスクなしでは完全に空です。 この不具合を修正するには、新しいタスクを作成する必要があります。 その後、インポートされたタスクが魔法のように表示され始めます。
  • リストにはさらに2倍のタスクがあります。 タスクはインポート後に複製されますが、これは既知のバグです。


さて、ハンドルを持つ重複タスクを削除します。 並行して、Webインターフェースの遅れを感じます。 モバイルアプリケーションを初めてインストールしたとき、リスト全体が重複して表示されましたが、Webクライアントではインポート後にタスクの半分を削除しました。 今ではすでに修正されているようです。 Androidインターフェースも少しバグがあります。



同期は定期的に発生します。 ここでは、Astridのようにプッシュは表示されません。アプリケーションの重量は14 MBで、Any.DOの2倍です。

Androidバージョンでできること：

  • タスクをマークします。
  • 日付、繰り返し、時間のリマインダーを設定します。
  • タスクにサブタスクを追加します。 無料版では数量が制限されています。
  • 説明を追加します。
  • 共有します。


評決 ：Any.DOよりは良いが、アストリッドよりは悪い。



GTasks



投票： 28 928。

評価： 4.5。

マーケットリンク： https : //play.google.com/store/apps/details?id=org.dayup.gtask

Astridからのインポート：アプリケーションの起動後、アプリケーション自体がインポートを提案します。



Astridの前にこのアプリケーションを使用しました。 私はGoogle製品が大好きなので、Android向けGoogle Tasksのアプリケーションを探していました。 私はそれらの多くを試して、これに落ち着きました。 アプリケーションの設計は、標準のUIコンポーネントを使用して行われますが、これは本当に気に入っています。 すべてが美しく上品に見えます。

インポートにはいくつかの問題がありました。 日付は正常に設定されましたが、時間のリマインダーがなかったタスクの場合、正午に表示されました。

アプリケーションでできること：

  • 日付、時刻、繰り返しを設定します。
  • タスクを重要としてマークします。
  • 共有します。
  • タスクの説明とタイトルは1つのエンティティです。


アプリケーションには、Any.DOやWunderlistに比べてかなり多くの設定があります。

さて、今軟膏のフライ... Gmailで見ることができるGoogleタスクウェブクライアントインターフェースは、悲惨な以上です。 そこでリマインダー時間を設定することはできません。タスクに重要なマークを付けます。 一般的に、あなたの多くはあなた自身の目でこれを見ました。

幸いなことに、Webインターフェースを備えた次の製品と同期できます。

判定 ：通常のウェブクライアントがなければ、使用するのは便利ではありません。



ティックティック



投票： 73。

評価： 4.5。

マーケットリンク： play.google.com/store/apps/details?id=com.ticktick.task

Astridからインポート： https : //ticktick.com/about/astrid/welcomeまたはモバイルアプリケーションの設定から。



私は抵抗できませんでした:)。 記事の説明中にこのアプリケーションを見つけました。 このアプリケーションはGTasksによって作成されました。 どうやら、彼はグーグルへの依存をやめ、バックエンドをセットアップする時だと判断したようです。 アプリケーションはGTasksに似たスタイルで作成されますが、多くの違いがあります。

  • 左パネルが追加され、その上にリストがあります。
  • 設定はずっと少なくなります。 それらはほとんど最小限です。


何で私たちを喜ばせますか：

  • 日付、時刻を設定して、タスクを繰り返します。
  • 重要度を設定します。
  • タスクの説明は、GTasksと同じ方法で追加されます。
  • タスクは共有できます。


アプリケーションにはWebインターフェースがあります。 彼はベータ状態で、非常に平らでシンプルに見えます。 モバイルクライアントでの同期は、タスクの作成後すぐに開始されます。 一般的に、半自動化され、Wunderlistのようなキューはありません。

判定 ：モバイルアプリケーションは脆弱ですが、GTasksと組み合わせることは良いことです。
