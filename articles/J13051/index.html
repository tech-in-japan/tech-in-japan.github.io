フルスクリーンとウィンドウ

ほとんどの場合、 一度に1つのプログラムで作業します。 テキストエディタ、ファイルマネージャ、メールクライアント、ICQ、mp3プレーヤー、コマンドライン、ブラウザなど、どのプログラムでもかまいません。 このプログラムを使用するのが最も快適なのは、第一に何も邪魔されず、第二に最大の画面スペースが確保されている場合です。



これらの前提から、私たちは当然「フルスクリーン」モードで作業するようになります。各プログラムはフルスクリーンで開かれます。 また、プログラム間の切り替えの利便性のために、各プログラムは独自の仮想デスクトップ上で実行され、ホットキーAlt + Fx（Linuxのテキストコンソールなど）を使用して切り替えることができます。



したがって、Alt + F9のメールクライアント、Alt-F10のインスタントメッセンジャー、Alt-F11のブラウザーなどに即座に切り替えることができます。 これらのプログラムはそれぞれ最大のスペースを使用します（ほとんどの場合、ウィンドウタイトルにドラッグしたり、最小化したりする必要がないため、ウィンドウ装飾を無効にして実行します。また、WindowsのAlt-F4のキーボードのホットボタン）



その結果、私はいわゆる 「ウィンドウ」インターフェース-「ウィンドウ」はありません。ドラッグ、サイズ変更、最小化、またはAlt + Tabなどによる目的のウィンドウの検索は不要です。 -要するに、仕事の代わりにあらゆる種類のゴミに従事してください！



私の仕事を見てすべての私の友人も数年前にフルスクリーンに切り替え、それ以来、誰もウィンドウインターフェイスに戻るように描かれていません。 :)



だから私は考えている...またはいわゆる 「ウィンドウ」インターフェースは、人々の悪意のあるm笑であり、または私はこの人生で重要な何かを理解していません。 :)いいえ、ウィンドウインターフェイス（Gimpなど）のインターフェイスを持つプログラムがあることを理解しています。 しかし、私はそのようなプログラムを別の仮想デスクトップで実行し、再びフルスクリーンのアナログになりました。すべてのGimpウィンドウはAlt + F6を押すと利用可能になり、画面上にはそれら以外はありません。 そして、あらゆる種類の非同期アラート（ログで何かが見つかったり、mp3プレーヤーで曲が変更されたりなど） OSDを介して現在の仮想デスクトップにアクセスします。



ちなみに、いわゆる また、タスクバーを無効にします。これにより、1つの仮想デスクトップに常に1つのタスクしかないため、「タスクバー」が不要になるため、画面上の追加スペースが節約されます。 めったに使用されないプログラムのリストを含むメニュー（[スタート]メニューに類似）は、ホットなAlt-Escボタンで呼び出されます（マウスプレーヤーは、たとえば、画面の隅にマウスを移動することでメニュー呼び出しをカスタマイズできます）。 また、頻繁に使用するプログラムをメニューから起動する必要はありません：マシンの起動時にそれらはそれぞれ独自の仮想デスクトップに読み込まれ、常にそこにハングアップします（システムの起動直後にAlt + Fxを押すだけで適切なプログラムで作業を開始できます）。



一般的に、個人的には、そのようなインターフェースは理想的です-これは、インターフェースの存在そのものがまったく見えない場合にのみ当てはまります！



質問は、フルスクリーンでの操作を試みたすべての人に対するものです。ウィンドウインターフェイスに戻った場合、なぜですか？
