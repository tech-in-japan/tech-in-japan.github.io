aBeing One：モノのインターネットの中心

すべての製造業者は、「モノのインターネット」（IoT）という名前で「パイの一部を食い止めたい」と望んでおり、エイサーは無視するつもりはありません。 aBeing Oneは、IoTをよりリアルに近づけるために設計されたコンピューターです。 マーケターが「ユーザーは新しいネットトップが必要だ」と言ったために作られなかったという点で興味深い。 aBeing Oneは、AcerがクラウドにSDKとAPIを提供するモノのインターネット鉄ツールとして登場しました。







したがって、AcerはaBeing Oneコンピューターだけでなく、Acer Open Platform（あらゆる「スマート」デバイスのデータ処理と管理のためのプラットフォーム）も提供します。



中身は？



aBeing One（ レビューはGeektimesで行われました ）は、一見すると多くの異なる「ネットトップ」のように見えますが、魅力的なデザインで、かなり強力なフィリングを備えています。



  • プロセッサーおよびビデオ：Intel Core i3-5010U + Intel HD Graphics 5500
  • メモリー：1TBハードドライブ、4GB DDR3L RAM
  • インターフェース：イーサネット10/100 / 1000、802.11ac / a / b / g / n、BT4.0
  • 寸法（mm）：106.5（幅）x 106.5（深さ）x 155（高さ）


上面には、SDカード用のスロットと4つのアクティビティインジケータがあり、そのうちの1つは2色（ネットワークインターフェイス用）です。









背面には周辺機器用のコネクタがあります：



  • 食品用ソケット（BP-19V、3.42A）
  • 2 USB 2.0
  • 2 USB 3.0
  • ギガビットLAN
  • ディスプレイポート
  • HDMI
  • ヘッドセットジャック



また、背面には電源ボタンとケンジントンロックがあります。 すべてのコネクタの左側には換気グリルがあります（コンピューターにはアクティブな冷却が備わっています。寝室に置いてはいけません。ファンが完全に静かに聞こえます）。



ハウジングの底面は、ほぼ完全に換気グリルです。 電源インジケータは底部の前面にあります。



ちなみに、全体の表示は白色LED（ネットワークアクティビティインジケータを除く-2色：白/青）で行われ、白い光沢のあるケースと一緒に非常にスタイリッシュに見えます。



オペレーティングシステム：Ubuntu Linux 14.04.2 LTS（64ビットバージョン）。 電源ボタンとPSUコネクタの間に機械的なロックボタンがあります。



特徴



一方では、その特性において特に目立ったものはないように見えますが、他方では、「すべてが搭載されている」ため、システムはほとんどのタスクに対して非常に生産的です。



しかし、このコンピューターには、他のコンピューターと根本的に区別できるものもあります。 ロックボタンを押すと、ケースの上部を取り外すことができ、室内装飾が開きます。



aBeing Oneは、 さらに2台のハードドライブ（2.5 "SATA）のインストールをサポートします。これは、既にインストールされている1TBハードドライブに追加されます。ドライブのインストールはツールなしで行われます。適切なトレイにドライブを固定するだけです。



前に、上部にアクティビティインジケーターがあることを説明しました。そのうち3つはハードドライブ専用です。



追加のディスク用のトレイの1つの後ろに「ハッチ」もありますが、その背後には興味深いものは何も見つかりませんでした-マザーボードと無線通信ボード（アンテナが接続されている部分）が見えます。



メインハードドライブにアクセスするには、ケース全体を分解する必要があります（既にツールを使用しています）が、私はこれをしませんでした。



仕様ではMini PCI Expressスロットも主張していますが、ケースを分解せずにどこにあるかを言うことはできません。



なぜ必要なのですか？



もちろん、aBeing One AZ10は、モニターとキーボードをマウスで接続することにより、通常のデスクトップコンピューターとして使用できます。 ただし、これは「顕微鏡で釘を打つ」ことになります。これは作成されたものではありません。



aBeing Oneの本当の目的は、Acer Open Platformクラウドサービスとホームネットワークの間のブリッジとして機能することです。 やめて、なぜAcer Open Platformのような追加のエンティティが必要なのですか？ この質問は、さらに詳しく説明する価値があります。 事実、ごく少数の「スマート」システムは完全に自給自足で閉鎖的です。 明らかな例として、内部ネットワークから機能し、内部から制御されるプラントのシステムのみを引用できます。 そして、データは内部サーバーに保存されます。



ほとんどのシステムは通常、外部ネットワークへの何らかのアクセスを必要とします-それは外部制御のためのものである可能性があります：例えば、遠隔照明制御。 または、着信データを保存する場合：もちろん、単にハードデータにデータをドロップすることは可能ですが、多くのユーザーにすばやく検索したりアクセスしたりする必要がある場合はどうでしょうか？ クラウドとは異なり、1台のコンピューターにはこのための十分なリソースがありません。 ローカルネットワークで接続できない多くのシステムを同期するには、より多くの外部サーバーが必要です。



これらの目的のためには、AOPが必要です-それに基づいて、開発者はニーズと要望に応じて任意のシステムを構築できます。 必要です-10台または2台のIPカメラからのデータが集まり、クラウドを使用してどこからでも高速にこのデータにアクセスできるローカルビデオサーバーを作成できます。 同じクラウドに、デバイスまたはユーザーデータのバックアップコピーを保存できます。



カメラ付きのイチジク。 すべてのビデオ監視サーバーがこのタスクを実行できます。 ただし、通常のMySQLデータベースを備えたレコーダシステムがテラバイトに膨れ上がり、速度が低下し始めたときに、数千個のセンサー（温度センサーまたはGPSトラッカーの場合は関係ありません）が1秒ごとに生成される場合、ここではクラウドなしではできないことを理解しています。 そして、それはあまりビッグデータではありませんが、いくつかのサーバーの頭脳は明らかに十分ではありません。 そして、大丈夫、単に十分なメモリがないだろう、それはそれほど怖くない。 さらに悪いのは、データと分析を分析する機能を失うという事実です。このためには、迅速な検索のための適切に構築されたインデックスと、データ分析のための強力なマルチコアプロセッサが必要です。 クラウドテクノロジーが役立つのはこれらのタスクです。大量のデータに簡単にアクセスでき、さまざまなイベント間の相関関係を確認して、すべてを便利な形式で提示できます。



aBeing Oneは、よりスマートに使用することもできます-たとえば、スマートホームデバイスと外の世界との間のゲートウェイとして。 ボードにはすでにBT4があり、フィットネストラッカー、心拍数メーター、温度センサー、制御ランプなど、さまざまなBLEデバイスに接続できます。 USBとイーサネットを使用すると、Z波、ZigBee、RS485、Modbusなど、あらゆるトランスミッターとインターフェイスコンバーターを接続できます。



そして-行動の完全な自由。 コテージまたは工場全体の温度センサーからデータを収集しますが、少なくとも電話からの光と換気を制御しますが、少なくとも光センサーをまったく必要としないモーションセンサーのシステムを作成します。



AOPには詳細なドキュメントを含むSDKとAPIがあるため、上記のすべては特に簡単です。









残っているものは何ですか？



aBeing Oneは、モノのインターネットに偏った幅広いアプリケーションを備えた、かなり興味深いコンピューターです。 主な違いの1つは、Acer Open Platformの機能と機能へのアクセスです。 実際、このコンピューターは鉄と雲の間の「過渡的なリンク」です。 これを使用すると、すでに開発済みまたは開発中の製品をAcer Open Platformに簡単に接続してから、任意のユーザーデバイス、電話、タブレット、またはPCに接続できます。



たとえば、このようなオファーは、家庭用ビデオ監視用のWiFiカメラを開発している会社にアピールします。 彼らは「鉄」カメラを作りましたが、これらのカメラからのビデオを保存するための独自のクラウドストレージを作成することは彼らにとって困難です。 Acer Open Platformを活用して、ほぼ完全なクラウドビデオストレージインフラストラクチャを取得できます。



あなたが何らかの形でIoTに関連するデバイスを開発している会社の場合、AcerはaBeing Oneをテスト用に提供し、デバイス自体とプロジェクトで使用するAOPの両方の可能性を評価できるようにすることができます。 手紙 。 そしてもちろん、会社はシステムのあらゆるコンポーネントでの作業に関するフィードバックを喜んで受け取ります。
