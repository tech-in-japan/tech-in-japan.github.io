Greenheart Gamesのインタビュー。 キャラクターとインディー

（インタビュアー：Marina virtualtomato 、 All Correct Localizationのプロジェクトマネージャー）



ゲームが異なると主張する人はいません。 彼らはいくつかに多くのお金を費やし、他の人には多くの時間を費やします。 私たちの期待に応える人もいれば、後で棚にほこりを集める人もいます。 私たちだけが、販売の初日に待って、店を襲撃しています。 他の人はそれを楽しみにしていますが、彼らは来ており、彼らは皆なくなっています。 そして、すべてのゲーマーは、隅々で話をしなくても、彼が単に愛するゲームを見つけると思います。



インディーズゲームのGame Dev Tycoonから2013年の3晩が与えられました。 これは小さなゲーム開発スタジオシミュレータです。 私は喜んで合格しました、そしてそれを作った人に感謝したいと思います。 これがスタジオGreenheart Gamesのデビューであることが判明しました。 デビューはいつもとても刺激的です。 したがって、私は最初のゲームを今やっている人にとって、そのような話は、小さいながらも間違いなく成功するとはいえ、始められたことを完了する自信と強さを与えることができると思いました。 はい、私は私の好奇心を満足させます。



それで、グリーンハートに私の質問を書きました。 そして、彼らは彼らに答え、答えることに同意しました。 以下は、Greenheart Gamesの創設者、ディレクター、およびゲームデザイナーであるPatrick Klugとのインタビューです。



GDTについて



マリーナ：パトリック、あなたのゲーム体験について、そして自分のスタジオを設立する前にゲーム業界で働いた経験があったかどうかについて少し教えてもらえますか？



パトリック：私はほとんど子供の頃からゲーマーになりました。 CastleのようなASCIIゲームを楽しんで育ちました。 私の目の前で、ゲームは何度も進化してきました。 同時に、私の人生の10年以上をプロのソフトウェア開発に費やしてきましたが、Greenheart Gamesが作成される頃には、ゲーム業界での経験はまったくありませんでした。



M：他の人気ゲームであるGame Dev Storyを実際にコピーするゲームでキャリアを始めることにしたのはなぜですか？



P：ゲームコミュニティは2つのグループに明確に分かれています。1つは「盗作」、「複製」、「クローン」などのフレーズをすぐに投げ始めますが、もう1つは、ゲームで「インスピレーションを引き出す」盲目的にコピーするのと同じです。 実際、業界には多くのクローンゲームがありますが、Game Dev Tycoonはそれらの1つではありません。 GDTを作成するとき、私たちはGame Dev Storyに本当に触発されました（これについては、当社のWebサイト-About Usページで読むことができます）が、プロジェクトの仕組みはまったく異なります。 ゲームスタジオシミュレーターを作成し、特定の機能を含めないのは愚かなことです。 Game Dev Storyを思い出しても、実際、私たちのゲームはまったく異なる方法で動作します。 GDTを作成するとき、GDS虫眼鏡の下で勉強しませんでした。 すべてが正反対です-ゲームのコンセプトは私たちによってゼロから開発されました。 Game Dev Tycoonの作業全体を案内してくれたのは彼女でした。 プレイヤーはしばしば、このジャンルに対する私たちの認識が好きだと言います。 Game Dev Storyを愛するほとんどの人はGame Dev Tycoonを愛し、その逆も同様です。







M： Game Dev Storyからインスピレーションを受けて自分のスタジオを作成したのではないですか？



P：ゲーム開発者になろうと思ったのはGDSだったとは思いません。 はい、私は彼女としばらく時間を過ごしましたが、それは私がこの業界で働く可能性について考え始める前でした。 最初のゲームをゲームスタジオシミュレータにするという決定はすぐには得られませんでした。さまざまなアイデアを検討しました。その中には独創的なものもあれば、何かに触発されたものもあります。



M：さて、あなたはあなた自身のゲームが誰かをゲーム業界で働かせることができると思いますか？



P：はい。 ゲーム開発者シミュレーターを試してみると、実際に自分のゲームで作業を開始できる人もいると思います。 プレイヤーがこのステップをどのように決定するかを観察し、ゲームまたは完全に独立したプロジェクトの修正を作成し始めることを常に喜んでいます。



M：あなたのゲームはゲーム市場の現実をどの程度反映していますか？ プレイヤーがGame Dev Tycoonで成功し、すべてのリリースが成功した場合、彼はゲーム業界に精通しているので、実際のゲーム開発で同じ成功を収めることができると言えますか？



P： Game Dev Tycoonは主にゲームです。 それはゲームスタジオの絶対に現実的なシミュレーションとは言えないので、そこで達成された成功は実際にはそれほど意味がありません。 一方、私たちのプロジェクトは有利な点で、ゲーム開発のいくつかの興味深い側面を示しています。 たとえば、開発者の役割を持つプレーヤーに優先順位を正しく設定し、必要に応じて何かを犠牲にするメカニズムをゲームに導入しようとして、多くの努力をしました。 このアイデアの実装は非常に現実的であると考えられます（特定のジャンルの代表としてのGDTに固有のもの）。 現実の世界で完璧なものを作成しようとする場合、失敗は必然的にあなたを待っています。 このゲームでは、すべてがまったく同じです。 その中で、あなたは非常に難しい決断をしなければなりません。 おそらく、私が読んだ最も面白いコメントは次のとおりでした：「Game Dev Tycoonをプレイした後、私は今一つのことを知っています。ゲームの開発を始めるには、ポケットに7万ドルが必要です。」 7万は、ゲームの開始資本金のサイズです。 この数字は現実からそれほど遠くないと思います（もちろん、将来の開発者が住んでいる国に依拠することはもちろんです）。 初心者は常に作業量と単純なゲームを作成するのに必要な量を過小評価しています。



M：コンセプトから最初のバージョンのリリースまで、ゲームを開発するのにどれくらいかかりましたか？



P：私の兄と私は、プロジェクトのコンセプトをデザインドキュメントの形で約6か月間開発しました。 確かに、私たちは空き時間にのみこれを行いました。 その後、私はプロジェクトに1年以上にわたって密接に取り組みました。 ダニエルはまた、ゲームに多くの時間を費やし、わずかな機会に私を助けようとしました。 さらに、アニメーションとグラフィックスの作業を行うために、いくつかの専門家を追加雇用しました。 このゲームには私たちの多大な努力が必要でした。



海賊について



M：私たちが友人とゲームについて話し合ったとき、海賊はあなたの利益をすべて盗むので、彼らは通り抜けることは不可能だと言った。 当時、私はまだキャンペーンを実施していなかったので、ストーリーの中で「熱」がさらに始まると推測することしかできませんでした。 しかし、Greenheartのブログを読んで投稿したとき、海賊がゲーム開発シミュレーターをプレイし、海賊行為により破産した場合はどうなりますか？ 、それから私は問題が何であるかを理解しました。 海賊版を待たずに、海賊版を作成してトラッカーに投稿しました。 このバージョンのみが「キャッチ」付きでした。 公式バージョンとは異なり、「海賊版」バージョンの高レベルの著作権侵害はゲームの通過を妨げました。 そして、私の友人全員がトラッカーからゲームをダウンロードしただけで、私だけがSteamでそれを購入しました。 私は言わなければならない、あなたがしたことは非常に機知に富んでいる。 そして、これは私が今まで聞いた海賊行為との戦いの最良の例です。 しかし、ここで私に悩まされる質問があります。 これはあなたの最初のゲームです、あなたはまだ海賊の行動に大きく苦しむ時間を持っていません。 では、なぜ自分の立場をすぐに明確にしたのですか？ 海賊はいつこの方法であなたを連れてきましたか？



P： 「海賊版」のリリースの経験は私たちにとって実験でした。 まず、何人がそれを使用するのかを知りたかった。 そのようなバージョンを自分でリリースすることが唯一の方法であることに気付いた私たちは、海賊をプレイする誘惑に打ち勝つことができませんでした。



初日に得られた結果は、ブログに投稿された対応するメッセージで確認できます。 正直に言うと、発売日における海賊版の配布規模に衝撃を受けました。 メディアの次の嵐は、私たちを大いに驚かせました。 彼女には非常に感謝しています。まず、著作権侵害の問題にさらに注意を向け、次に、以前よりもはるかに多くの人々が私たちのゲームについて学びました。 同時に、彼らの多くは私たちのプロジェクトをサポートすることを決めました。



M：あなたの意見では、著作権侵害による被害を減らすことができる独立した開発者向けの戦略はありますか？



本当に効果的な戦略はほとんどないようです-ネットワークへの常時接続を必要とする純粋にマルチユーザーのゲームを開発する必要があります（そのサーバーは常にあなたの制御下にある必要があります）か、有料のフリーミアムモデルを使用します。 正直に言うと、最後の選択肢は私たちにとって非常に嫌なものです。 ゲームは、「ワンアームバンディット」で単に装飾するだけでなく、より多くの仮想コインを要求する必要があります。 Greenheart Gamesはこのようなモデルを使用しません。 一方、ネットワークへの永続的な接続を備えたネットワークプロジェクトは優れた代替手段のように見えますが、残念ながら、市場から普及するにつれて、従来のシングルユーザー/オフラインゲームは取って代わられ、個人的には精神的にはるかに近いものになります。



無料プレイ



M：つまり、Free-to-Playプロジェクトやゲーム内のマイクロトランザクションの広がりに対して非常に否定的な態度を持っていますか？



P：海賊の多くは、彼らの「活動」が業界に大きな影響を与えていることを理解しておらず、フリーミアムメカニクスで動作する有料ゲームが登場する主な理由です。 個人的に、私は心からこのようなゲームが嫌いです。 Greenheart Gamesはこのようなモデルに決して頼りませんが、収益性の観点から見ると、残念ながら非常に効果的です。



蒸気



M： Steamでの経験を教えてください。 簡単ですか？ 独立した開発者向けの手順を簡単に説明できますか？



P： Steamは常にいくつかの変更を受けているため、私たちの経験は、今日このシステムで作業する開発者が獲得した経験とはすでに異なります。 さらに、Valveの声明によると、同社はGreenlightメカニズム（プレーヤーがSteamディレクトリにゲームを追加するために投票できるようにする）を廃止する予定です。



私たち個人的には、Steamは大きな可能性を開いており、それらを開き続けています。なぜなら、実際には、ゲームを正直に購入する人がたくさんいるからです。



ローカリゼーションについて



M：ローカライズの仕事では、集団翻訳-クラウドソーシングを使用しました。 そして、私の知る限り、ローカライズをプロセスとしてかなり明確に考えていました。 私はあなたが持っていることが好きでした：

  • 翻訳管理システム-weblate.org;
  • 潜在的なボランティアの特定の要件：彼らはゲームをよく知り、ゲーマーであり、ゲームを理解し、それらを愛さなければなりません。
  • 翻訳者はネイティブスピーカーです。


ファン翻訳を決定する前に、他の選択肢を検討しましたか？ そして、最終的にクラウドソーシングをやめたのはなぜですか？



P：私自身はドイツ語を母国語としています。良い翻訳をすることの重要性（そして難しいこと）を理解しています。 業界での以前の経験から、仕事でお金を受け取るプロの翻訳者と仕事をすることは非常に難しいことを知っていました。 非常に頻繁に、彼らが取り組んでいる製品に対する彼らの熱意と知識は、ファンが示すものよりもはるかに劣っています。 それが、ボランティア労働者をプロジェクトに引き付けることを決めた理由です。 もちろん、このアプローチだけでは高品質の翻訳が保証されるわけではありません。多くの人が作業する場合、一貫性と一貫性を維持することは非常に困難です。 このような状況で高品質のテキストを取得するには、非常に強力な仲介者を使用する必要があります。 この種の品質管理システムに関する作業はまだ完了していません。



M：ローカライズに関して不明確な問題はありましたか？ あなたを困惑させた質問は？ たとえば、無料で専門家に相談する機会があった場合、それを使用しますか？



P：アラビア語のように、右から左に単語を書いたり読んだりする言語をまだ試しています。 さらに、出産に関しては英語の原本にはない一定の問題があります。 これらは、プレーヤーの性別に応じて性別を変更する必要があるテキストの断片を翻訳する場合に特に深刻です。 これらの場所を正確に特定することは困難であり、これまでのところ、この問題で特定の妥協に頼ることにしました。 このような問題を解決した経験について同僚から学べることを非常に嬉しく思います。 確かに、ある種の万能薬、ある種の完璧な解決策があるのではないかと疑っています。







M：ローカライズされたバージョンを持つことは、ファンへの敬意の表れだと思います。 そして、あなたは、世界中のゲーマーが英語を知っているかどうかに関係なく、あなたのゲームを楽しむことができるように、できる限りのことをした素晴らしい仲間です。 ローカライズがうんざりしない限り、私は常にロシア語版を好む。



ロシアではファン翻訳の伝統が非常に強力です。なぜなら、非常に長い間、私たちの国は公式なローカリゼーションという点で単純にバイパスされていたからです。小さな市場、高いレベルの著作権侵害です。 今、状況は良くなっています。 多くの人がロシアを重要な市場と見なし、ゲームをロシア語に翻訳しています。 また、小規模スタジオの場合、クラウドソーシングは解決策です。 しかし、最高のファンローカリゼーションでさえ、最終的に多くの欠陥を含んでいます。 最も一般的なものは、読み書きのできないロシア語、英語の構造の追跡、変数やタグを操作できないこと、さらに99％の翻訳者が女の子もゲームをプレイすることを忘れているため、ファンのローカリゼーションでは、ほとんどの場合男性プレイヤーへのアピールが使用されます（通常、英語では問題はありませんが、ロシア語ではすべてのゲームで対処する必要があります）。 たぶん私は気まぐれすぎるかもしれませんが、ゲームが私が男だと思ったときに私を悩ませます:-)



だから、ファンの翻訳がプレイヤーにローカライズの悪い味を植え付け、何が良いのか、何が悪いのかという彼らの考えを歪ませることは気になります。 最終的には、ゲーマーのリテラシーに影響します。 このように考えたことはありますか、これは本当に問題だと思いますか？ または、ゲームについて話すとき、それはそれほど重要ではありません。主なことは、ユーザーがゲーム内のタスクを理解していることですか？



P：はい、そのような問題が存在することに同意します。 私のお気に入りのゲームの1つは、Anno 1602（Annoシリーズの最初のゲーム）です。 私の祖国（オーストリア）で作成されたため、もともとドイツ語であったことを知っている人はごくわずかです。 おそらく、このゲームはドイツ語の第一級のテキストで私を喜ばせた最初のゲームだったでしょう。 Anno 1602は、ドイツ語の可能性を最大限に活用した鮮やかな表現と美しい文章でいっぱいです。 翻訳中のテキストの品質を同等にするために、世界クラスの翻訳者の作業が必要になります。



もちろん、優れた翻訳を作成するには多くの努力が必要です。 さらに、これには優れたチームが必要です。 プロの翻訳者のサービスを使用する際の問題は、間違いなくトップレベルのスペシャリスト（以前は本の翻訳に携わったことがある人）が必要なことです。 加えて（そして同様に重要）、そのような人々を見つけて、彼らの仕事を管理することは非常に困難です。 これまでのところ、私はボランティアサービスと厳格な品質管理を組み合わせることは良い妥協だと考えています。 同時に、高品質の翻訳には相当な労力と細心の注意が必要であることを理解しています。 最終的に、現在のロシア語版は妥協であり、改良が必要であることを十分に認識しています。



模倣例



M：現在、そのポリシーが最も印象的なスタジオはありますか？ 従うべき例はありますか？



P：あなたが何もする必要がない方法の例を挙げることができる非常に多くの会社があります。 私は、収益化されたゲームを介して、またはそれを介して作成することに集中するか、「早期アクセス」で無限のプロモーションを行う人々を意味します。 両方の十分な例があります。 一方、近年では、優れたプロジェクトに取り組んでいる多くの新人がいます。 それらの1つは、RimWorldの作成者であるTinan Sylvesterです。 まず、ゲームデザインに関する彼の本は真の傑作です（本を読むことを強くお勧めします）。 第二に、彼はキックスターターが招いた恒常的な「新しい約束」の魅力的な見通しを放棄しました。 シルベスターは、事前に数十の過剰に計画された目標を事前に設定しなかったため、開発の完全な自由が維持されました。 私はこのアプローチを本当に尊重し、ゲームが大成功することは間違いありません。



将来の計画について



M：現在のプロジェクトを開発する予定ですか、それとも次のゲームに取り組んでいますか？



P：できるだけ早く新しいプロジェクトの作業を開始したいのですが、同時にGame Dev Tycoonも無視されません。 ゲームの新機能や追加機能は登場しそうにありませんが、他の言語への翻訳は確実に改善されます。 さらに、ゲーム用のmodプレイヤーの素晴らしいコミュニティを支援するために最善を尽くします。
