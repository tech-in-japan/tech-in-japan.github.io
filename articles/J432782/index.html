Zimbra Collaboration Suiteのマルチサーバーインストール

Zimbra Collaboration Suiteを使用して1つのサーバー上に5〜6千を超えるメールボックスを作成することは推奨されていません。 この範囲を超えると、サーバーの品質が大幅に低下し、ユーザーへの可用性が低下します。 しかし、Zimbra Collaboration Suiteをサービスとしてアクセスする大企業またはSaaSプロバイダーがあり、10、20、30、または千人以上に電子メールを提供する必要がある場合はどうでしょうか。 特にZimbraのこのような場合、マルチサーバーインストールのサポートがあります。このインストールでは、メールボックスが異なるサーバー上にあり、高負荷下でも高い電子メールの可用性を維持できます。







もちろん、Zimbra Collaboration Suiteのマルチサーバーインストールは、単一サーバーへの通常のインストールとは大きく異なります。そのプロセスについては、前の記事で詳しく説明しています。 まず、Zimbra Collaboration Suiteがインストールされるサーバーインフラストラクチャのアーキテクチャを決定することをお勧めします。 このようなサーバーインフラストラクチャの構成は、運用に最適です。 メールボックスを保存するサーバーの数は任意であり、既に述べたように、サーバーへのアクセスがWebクライアントのみで実行される場合、サーバーごとに5から6千のアクティブなメールボックスと3から4千に基づいて計算されます。ユーザーがデスクトップクライアントで作業し、モバイルデバイスとメールを同期する場合、サーバーへのメールボックス。





Zimbra Collaboration Suiteのマルチサーバーインフラストラクチャを構築する1つの例



Zimbra LDAP



直接インストールを開始する前に、すべてのサーバーのシステム時間が同じであることを確認してください。 まず、Zimbraのマルチサーバーインストールでは、Zimbra LDAPを使用してサーバーをインストールおよび構成する必要があります。 必要に応じて、Zimbra LDAPを複数のサーバーにインストールできます。そのうちの1台はメインLDAPサーバーとして機能し、残りは負荷の一部を引き受けてメインサーバーの速度を上げる補助LDAPサーバーになります。 セカンダリLDAPサーバーの可用性は、Zimbraを使用したインフラストラクチャのセキュリティと復元力も強化します。



メインLDAPサーバーへのZimbraのインストール中、インストーラーはインストールするコンポーネントを選択するように促します。 この場合、 zimbra-ldapのみが必要です 。

Yを押すと、システムの変更が完了した後、テキストメニューが開きます 。このメニューでは、 共通設定項目に関心があります 。 それに行くと、基本設定のリストが表示されます：

アイテム4を選択すると、インストール中にランダムに生成されたZimbra LDAPにアクセスするためのパスワードを確認できます。 また、タイムゾーンを現在の場所に変更する必要があります。 LDAP管理者のパスワード、およびLDAPサーバーのアクセスポートとドメイン名を覚えておくか、書き留めておくことをお勧めします。 メールボックスとMTAを使用してサーバーをセットアップするときに、この情報が必要になります。



その後、再びメインメニューに戻り、 zimbra-ldapという2番目の項目を選択します。 ここでは、LDAPルート、LDAPレプリケーション、LDAP Postfix、LDAP Amavis、およびLDAP Nginxにアクセスするためにランダムに生成されたパスワードに興味があります。 LDAPレプリケーション、LDAP Postfix、LDAP Amavis、およびLDAP Nginxからのパスワードは、MTAおよび補助LDAPサーバーからのサーバーのさらなる設定に役立つため、覚えておくか書き留めることをお勧めします。 その後、変更を適用し、すべての設定をファイルに書き込むことに同意するだけです。 LDAPサーバーのセットアップが完了しました。



LDAPレプリカ



補助LDAPサーバーを構成する場合は、コマンド/ opt / zimbra / libexec / zmldapenablereplicaを使用して、メインLDAPサーバーでそれらのサポートをアクティブにする必要があります。 後で、補助LDAPサーバーを構成するときに、プライマリサーバーをオンにする必要があります。



セカンダリLDAPサーバーのインストールと構成は、プライマリサーバーのインストールと構成を繰り返します。 主な違いは、インストール後のセットアッププロセスです。 だから、あなたが必要になります：



  1. メインLDAPサーバーのアドレスをLDAPマスターホスト名として指定し、
  2. ポート入力フィールドで、メインLDAPサーバーで開いているポート番号を指定します。
  3. ランダムに生成されたLDAP管理パスワードを、メインLDAPサーバーにインストールされているパスワードに置き換えます
  4. [LDAP構成]サブメニューで、[ドメインの作成]フィールドに[パラメーターなし]を設定します。
  5. プライマリサーバーの構成時に設定されたLDAPレプリケーションパスワードを入力します。


その後、すべての変更を適用し、設定をファイルに保存します。

インストールに同意した後、システムの変更を許可し、インストールプロセスが完了するのを待って、サーバーのセットアッププロセスを開始します。 まず、 Common Configurationアイテムに移動して、LDAPサーバーアドレスとLDAPアクセスパスワードを指定する必要があります。さらに、サーバーに正しいタイムゾーンが設定されているかどうかを確認することをお勧めします。 また、LDAPアクセスパスワードを忘れた場合は、LDAPサーバーでコマンドzmlocalconfig -s zimbra_ldap_passwordを入力してパスワードを取得できます。



その後、アイテムのストア構成に進むことができます。 ここでは、サーバー管理者のパスワードを設定し、リポジトリのドメイン名を対応する番号に設定する必要があります。これにより、混乱が生じなくなります。 さらに、接続のタイプを選択する必要があります。 たとえば、HTTPまたはHTTPSを常に使用したり、HTTPSの強制使用を有効にしたり、認証にのみHTTPSを使用したりできます。 また、Zimbraメールボックスの構成中に、 SMTPホスト行で対応するMTAサーバーのアドレスを指定する必要があります。 その後、あなたの裁量でいくつかの設定を行うことができます。 設定の最後に、変更を適用し、すべての設定をファイルに保存する必要があります。



インストール後にSSHアクセスのパスワードを設定して、Postfixサーバーとキューのリモート管理を有効にすることをお勧めします。 また、補助LDAPサーバーを使用する場合は、構成が完了した後、MTAサーバーとメールボックスサーバーのldap_url値を変更して、正しく動作するようにする必要があります。 これを行うには、 zmcontrol stopコマンドを使用してZimbraを停止し、z mlocalconfig -e ldap_url =” ldap：//ldap-2.zimbra.com ldap：//ldap-1.zimbra.com”という形式のコマンドを入力する必要があります。最初に、すべての補助サーバーのアドレスを指定し、メインLDAPサーバーの最後に指定します。 MTAサーバーでは、設定が完了した後、Zimbraを再起動し、 / opt / zimbra / libexe / zmmtainitを実行してPostfix設定を上書きする必要があります。
