Parallels Virtuozzo VS Microsoft Hyper-V。 誰の仮想化がより正直ですか？

Habréで仮想化について多くのことを読み、プロバイダーのサイトを歩き回りましたが、残念ながらロシア市場で最も人気のある2つのシステムの明確な直接比較は見つかりませんでした。 したがって、私は自分でそれを書くことにしました。



そこで、2つのサーバー仮想化システムを比較してみましょう。新しい、比較的新しいHyper-Vで、多くのホスティング事業者-Virtuozzoで長い間使用されています。



この形式で非常に理解しやすいパートナーによって比較します。

パラメータ：

Virtuozzo（V）

Hyper-V（H）



1.仮想化のタイプ

V： オペレーティングシステムレベルでの仮想化。 このメソッドを使用すると、独立した保護領域（コンテナ）でメインOSのインスタンスを実行できます。

H：サーバーハードウェアの完全仮想化 。 仮想化は、メインOSとは別のハイパーバイザーに基づいています。 これは、サーバーの物理デバイスとゲストオペレーティングシステムでのプレゼンテーションの間の仲介として機能します。



2.サーバーリソースの使用。

V： Virtuozzoテクノロジーは、サーバーリソースの使用率と仮想化パフォーマンスの損失に関して非常に効率的です。 マシンごとに別のOSであるため、物理サーバーが使用するリソースの量をすばやく変更できます。

H： Hyper-V仮想サーバーは、プロセッサ、メモリ、ディスク、ネットワークデバイスと共に、通常の物理サーバーであるかのように動作します。



3.過剰販売の可能性。

V：基本的なOSツールはサーバー上で単一のコピーで実行されるため、ゲストOSが完全に独立していること、保証された最小リソースを確保することは事実上不可能であるため、ホスティング事業者が好む十分に大きな売れ行きを実行できます。

H：各仮想サーバーには、最小限のリソースが割り当てられています。 「隣接する」仮想サーバーの影響は最小限に抑えられ、実際には過剰販売の可能性がなくなります。 これは、クライアントにとって有益です。 彼は無料のリソースで明確に定義された最小の生産性と成長の機会を得ます。 しかし、このスキームはホストにとって必ずしも満足できるものではありません。なぜなら、 彼はクライアント間で物理サーバーを正直に共有することを余儀なくされています。



4.ライブ移行

V：完全にサポートされており、物理サーバー間で転送する際にVPSの中断のない動作を保証します

H： Windows Server 200 R2のリリースでは、ライブマイグレーションを完全にサポートしています



5.サポートされているOS

V： WindowsとLinuxの両方がベースとして機能できます。 ゲストとして、さまざまなLinuxディストリビューション。 窓

H： Windows Server 2008ハイパーバイザーのベースOSゲスト：Windows NT、2000、2003、2008、XP、Vista、Linux（Red HatおよびSUSE）、およびLinuxの他のバージョン、FreeBSD。



6.コントロールパネルの存在

V： Parallels独自のグラフィカルコントロールパネルと統合

H： Microsoft System Center Virtual Machine Manager（SC VMM）の独自のコントロールパネル。 物理サーバー（Hyper-Vマネージャー）としての特別な管理コンソールと、サードパーティ開発者（ DotNetPanel 、 パーキングキューブ ）からのソリューションがあります。



上記のすべてを要約すると、Hyper-Vは現在、物理サーバーと同様のパラメーターでより正直な仮想化を提供しており、それらの価格はすでにVirtuozzoの価格に非常に近いことを私は表明できます。
