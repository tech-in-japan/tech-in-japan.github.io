安価なフェールオーバークラスター？ 簡単！

ドライブへの共有アクセスを取得し、すべてを2Uフォームファクターにまとめる機能と相まって、低コストのクラスターソリューションになると、一見すると、3つの要件をすべて同時に満たすことはできそうにありません。 しかし、富士通はPRIMERGY CX420 S1クラスターでこれらすべての条件を満たすことができました。これは、中小規模ビジネス（SMB）でビジネスアプリケーションの継続性を確保するためのターンキーソリューションとして提供され、低コストのフォールトトレラントソリューションを選択できませんでした。









イデオロギー



SMBセクターの企業が通常、フォールトトレラントシステムを構築した経験を持つシステム管理者を雇うことができないことは周知の事実です。 さらに、多くの企業の活動は、ビジネスプロセスの継続性の確保に直接関係しています。 たとえば、オンラインストアサーバーを数分間ドロップすると、注文が失われ、アイドル時間は評判に影響し、1〜2日のアイドル時間はビジネスの魅力を大幅に低下させ、倒産につながる可能性があります。サーバー自体は、問題のサーバーで考慮されている冗長性の助けを借りて簡単に保護できます。



PRIMERGY CX420 S1は、SMBセグメントで使用するWindows Server 2012向けのフェイルセーフクラスターとして設計されており、Windows Serverに精通しており、特別なトレーニングを必要としないシステム管理者が使用できます。 したがって、エンジニアリングサービスは、時間と財務部門-その実装のコストを非常に正確に計画できます。 CX420クラスターのもう1つの機能は、別個の外部ストレージシステムを必要としない完全なソリューションであり、クラスター本体にインストールされた12台のドライブに両方のサーバーノードの共同アクセスを提供することです。



建築



Fujitsu PRIMERGY CX420 S1は、Microsoft Windows Server 2012を実行する2ノードのクラスターです。実際には、CX420 S1は、2つのサーバーノードがインストールされた2Uシャーシです。 各ノードはPRIMERGY CX272 S1サーバーであり、最新の（最大8コアの）Intel Xeon E5-2600プロセッサーのペア（E5-2603からE5-2680までのCPUがサポートされています）。 プロセッサは、メモリを使用した4チャネルデータ交換をサポートしています。合計で、各ノードは最大16 GBの容量を持つ16個のモジュールをインストールでき、ノードあたり最大256 GBのRAMを提供します。









2つの外部ギガビットネットワークポートに加えて、各サーバーにはサービスLANポートと3つのPCIeスロットがあり、そのうちの1つにはSASエクスパンダーがインストールされている必要があります。 CX272 S1サーバーには、最大2つの内部ブート可能SATAディスク（ホットスワップなし、ミラーリング機能付き）と、CX420シャーシのシャーシにある共有ドライブへのSAS接続があります。 ドライブ（最大12個）3.5インチのフォームファクターがありますが、3.5インチスロット用の特別なアダプターで同じ数の2.5インチドライブを使用することができます。



共有ドライブは、問題のクラスターの最初の一意のプロパティを提供します。 12のデュアルポートクラスタドライブはすべて、Windows Storage Spacesテクノロジを使用して単一のディスクスペースに結合されます。 2つのポートが存在すると、両方のノードから同時にスペースにアクセスできます。 必要に応じて、追加の2ポート外部JBODアレイを12個のSASディスクに接続することにより、このディスク容量を拡張できます。



ノードの高さは2Uで、互いに対して水平に配置されます。 シャーシ内のそれらの間には、ファンとホットスワップ可能な電源を含む中央コンパートメントがあります。 ノードは、ギガビットイーサネットチャネル（シャーシ内のバックプレーンに実装）によって相互接続されます。 このチャネルは、クラスターを同期するために使用されます（「ハートビート」）。 換気システムは、2つの独立した気流を両方のノードに生成し、中央ユニットから空気を排出します。 ノード自体はファンレスでパッケージレスであるため、偶然にもシステムのコストを削減できます。 クラスターの消費電力は1200ワット以下です。



したがって、クラスタ全体が標準の2Uラックに収まり、電源と冷却のための特別な条件を必要としません。



準備



高可用性には、ハードウェアとソフトウェアの冗長性が必要です。 Fujitsu PRIMERGY CX420 S1のハードウェア冗長性は、計算ノードの複製によって提供され、各計算ノードでホットスワップが可能です。 さらに、オンザフライ電源を交換できます。 前述のように、クラスタノードの同期は、バックプレーンにあるギガビットイーサネットチャネルを介して行われます。 シナリオに応じて、ノードの1つがオフに切り替えられると、ユーザーの観点から、動作の遅延なしに負荷全体が別のノードに転送されるか、フェールオーバーが短時間実行されて、パッシブバックアップノードの障害が発生したアクティブノードからタスクが開始されます。 さらに、フェールオーバーは、ネイティブアプリケーションと仮想環境で動作するアプリケーションの両方でサポートされています。 ボード上のSASアダプターは、R​​AIDの使用を必要としません。これは、共有ディスクスペースの作業がWindows 2012ストレージスペースによって編成されるためです。 このテクノロジーは、従来のRAIDシステムと比較して多くの利点を提供します。これは、単にRAIDグループディスク全体にデータを「分散」するよりも、NetapからWAFLに近いものを表します。 特に、シンプロビジョニングを提供します。クラスター内のディスクからさまざまなディスクの使用を微調整できます。 これは、異なるタイプのドライブが使用される場合に特に当てはまります。

まとめ



Fujitsu PRIMERGY CX420 S1の「正面」では、ユーザーは外部ストレージを使用せずに、共有ストレージを備えたコンパクト（2U）で低コストの高可用性クラスターを利用できます。 システムはWindows Server 2012で動作するように設計されているため、機器自体の節約に加えて、パートナーはメンテナンスを節約することができます。フォールトトレランスとクラスタリングの分野で特別な知識を必要としません。

同時に、Fujitsu PRIMERGY CX420 S1は、標準のサーバーラックで使用するために作られており、インフラストラクチャに追加の要件なしで（したがってコストなしで）従来の空冷用に設計されています。

よくある質問



このクラスターは、 富士通サーバークラスターPRIMERGY CX420 S1ウェビナーで少し前にパートナーに提示されたので、リスナーがロシアおよびCIS諸国のPRIMERGYサーバーの製品マーケティングマネージャーであるPavel Borochの従業員に尋ねた典型的な質問にすぐに答えたいと思いました。



1）代替オペレーティングシステム

Hyper-Vを使用した仮想化によってのみ、Windows Server 2012以外のオペレーティングシステムを使用できます。 仮想環境でも機能するプログラムは、フェールオーバー後に復元されます。



2）他の富士通サーバーとの違い

たとえば、900 GBの容量と複製機能を備えた12個のSASディスクをサポートする代替システムをコンパイルしました。







写真にあるように、2台のシングルサーバーPRIMERGY RX200 S7サーバーとETERNUS DX60 S2ディスクストレージを使用する場合、より多くのラックスペースを占有し、消費量が増え、最も重要なことは、富士通PRIMERGY CX420 S1の26,530ユーロではなく32,300ユーロの費用がかかることです。 これは、貧弱なサーバーとストレージシステムのセットを選択したことを意味するのではなく、CX420 S1が指定された目標を可能な限り正確に満たすだけです。標準インフラストラクチャ要件を備えたコンパクトな高可用性サーバーです。

上記の価格はヨーロッパの定価であることに注意してくださいロシアの費用を調べるには、パートナーに連絡する必要があります。



3）他の2Uクラスターとの違い

2つのサーバーと2Uフォームファクターの1つの接続ユニット-排他的なデザインのふりをしません。 ただし、たとえば、他のメーカーのサーバーを使用すると、このボリュームで高性能コンピューティング用のクラスターを組み立てることができますが、CX420は、障害発生時の組み込みの回復手順を使用して、信頼性の高いクラスターを構築するように設計されています。



4）開発の可能性

PRIMERGY CX420 S1は、マイクロソフトソフトウェア用に富士通が計画しているターンキーソリューションの1つです。 まもなく、Microsoft SQL ServerおよびSharepoint Server用の既製のソリューション（「アプライアンス」）も紹介します。
