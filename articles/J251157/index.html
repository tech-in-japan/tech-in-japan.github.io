HTMLとCSS





autoprefixerは、 カヌー統計に基づいてCSSプロパティに自動的にプレフィックスを付ける最も便利なプラグインの1つです。 AutoPrefixerは、Evil MartiansのPostCSSプロジェクトのフレームワーク内の多くのアドオンの1つにすぎない、と言うことが重要です。



grunt-browser-syncは、おそらくWeb開発者の生産性を向上させるという点で最も便利なツールです。 BrowserSyncは接続を作成し、その後クライアントまたはサーバーファイルだけでなく、すべてのデバイスのすべてのブラウザーでページを自動的に更新します。 さらに、スクロール位置とフォームの完成したデータを同期します。



grunt-html-buildはユニバーサルレイアウトアシスタントです。 プラグインは、ソースファイルを結合し、HTMLの完全なテンプレート/ブロックを作成できます。 それでも同様のタスクでは、 grunt-include-replaceは非常に優れています。



grunt-email-designは、すべてのCSSスタイルをインラインスタイルに変換し、ファイルへのすべてのパスを自動的に変更し、オプションで画像をCDNにアップロードしたり、手紙を電子メールに送信したりする方法を知っている、文字をタイプセットするための貴重なツールです。 別のgrunt-email-boilerplateもあります。



grunt-uncssとgrunt-ucssは、同時にCSSファイルを最適化するための2つの最適なソリューションです。 どちらのプラグインもHTMLコードを解析し、未使用のスタイルと複製されたスタイルをすべて検出します。 2番目のプロジェクトはOpera Softwareチームによるものです。



grunt-revizorは、CSSを縮小するだけでなく、HTML、CSS、およびJavaScriptファイル内のすべてのクラスの名前を短縮する非常にクールなコンプレッサーです。 今日、多くのCSSコンプレッサーとそれらの比較表 （ GitHub ）があります。 しかし最近、Gruntのない最高の速記ユーティリティを見ました。

=>

grunt-contrib-htmlminはシンプルなHTMLミニファイヤです。



grunt-penthouseとgrunt-criticalcss-プロジェクトのクリティカルパスを自動的に見つけます。 パフォーマンスに関する重要なポイント。これについては、 ここで詳しく説明します 。



grunt-csscomb -CSSの構造を高貴にします。 スタイルガイドを生成するための、 うっとうしいスタイルガイドがまだあります。

grunt-notify -Gruntを構築するときにシステムメッセージの形式でエラーを表示します。最も重要なことは、異なるオペレーティングシステムで機能することです。
