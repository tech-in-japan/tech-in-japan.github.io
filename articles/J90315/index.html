モノ、2台のMacとiPhone

まえがき



たまたま、ファームに2台のMacと1台のiPhoneがあります。 電話はいつも私と一緒にあります。私は自宅でデスクトップで仕事をし、外出先でラップトップを持ちます。 そして、Thingsタスクマネージャーが本当に好きでした。 使い始めてから、ラップトップにプログラムの2番目のコピーを置く必要があることに気付くまで、自分の存在に影を落とすものはありませんでした。 悲しみは、プログラムデータベースを他のインスタンスと同期するためのサポートの欠如でした。 開発者は、たとえば1Passwordプログラムのメーカーのようなストレージデバイスを気にしませんでした。 また、次のRadio-Tポッドキャストを聞いている間、開発者がプロ​​グラムをいくつかの同様のインスタンスと同期させる可能性を予見しなければ、何もすることはないという考えが表明されました...そして脳は沸騰し始めました。 そのような機能を探す必要がありました。



挑戦する



各デバイスには、iPhoneとモノのコピーの2つのMacがあります。 メーカーが実装する機能のフレームワーク内で、3つのデバイスすべての間でタスクをできるだけ簡単に同期できるようにする必要があります。



解決策



この資料に興味がある人は、既にThingsプログラムがインストールされていると思います。 そうでない場合は、インストールします。 その後、3つのことを行う必要があります。 プログラムがデータを保存するフォルダーを見つけ、スケジューラーの2番目のコピーがそれを見つけることができる公共の場所に移動し、その両方をデフォルトのストレージにするこの場所を指すようにします。 デフォルトでは、Thingsはデータを〜/ Library / Application Support / Cultured Code /フォルダーに保存します。 次に、必要なデータをDropboxフォルダー、USBフラッシュドライブ、またはネットワークドライブにコピーします。 さらに...そして、ここが最も興味深いものです。 スケジューラーのデフォルトのフォルダーの場所を変更するには、option / altキーを押しながらプログラムを実行します。 新しいウィンドウを作成するか、既存のデータベースを選択するように求めるウィンドウが表示されます。







その後、以前にコピーしたデータベースインスタンスを配置する場所を選択し、プログラムを実行します。 Thingsの2番目のインスタンスからも同じことを行います。 これで、あるコンピューター上のタスクのリストが別のコンピューター上のリストと異なることを心配する必要がなくなりました。 iPhoneは、プログラムの両方のインスタンスで問題なく同期されます。これは、1つのスケジューラが2番目のスケジューラと変わらないためです。 私にとって、Dropboxは非常に良い選択肢でした。コンピューターがどのネットワークにあるのか、またスレーブの前にプログラムデータベースを含むUSBフラッシュドライブを挿入したかどうかを忘れることができたからです。



PS誰かが以前に同様の問題に遭遇したことがあり、解決策が私のものと異なる場合、私は可能なオプションを知ってうれしく思います。
