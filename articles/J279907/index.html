NetApp ONTAP Cloud：AmazonおよびAzure

クラウドサービスの利点は、それ自体がサービスであることです。 機器がどこに住んでいるのか、誰がサービスするのかなど気にしません。 しかし、これは暗い側面でもあります。誰があなたのデータにアクセスできるのか、あなたのデータのコピーがいくつバックアップとバックアップデータセンターに保存されるのか、そしてどこに？ これらの問題はすべて、大企業や政府機関によって長らく推進されてきました。



これらの問題を解決することにより、多くの企業がクラウドサービスの使用を開始できます。 この問題を解決するために、クラウドストレージシステムONTAPクラウドが開発されました。組み込みの暗号化機能を備えたAmazonまたはAzureのパブリッククラウドにある仮想マシンです。 今、あなたはあなたとあなただけのデータを所有し管理しています。 詳細については、 NetAppの公式Webサイト 、および製品自体とOnCommand Cloud Managerに関する公式ドキュメントを参照してください。





管理とインターフェイスの点では、これはいくつかの制限がある通常のONTAPです。ONTAPCloudの双子の兄弟であるONTAP EdgeとONTAP SelectにはRAID-DPとFCPがないためです。 しかし、残りは本格的なストレージであり、クラウドに住んでいます。 生産的なFASシステムからクラウドにデータを複製して、DRサイトと開発環境を作成できます。 また、事故が発生した場合は、バックアップサイトに切り替えて、すべてまたは少なくとも最も重要なサービスを開始してください。



これは経済的な観点から非常に便利です。事故後、クラウド内の必要なリソースを並べ替えてサービスを開始するだけで、機器を購入して2番目のバックアップサイトで保守する必要はありません。



高可用性



ONTAP Cloudは、クラウド内でフォールトトレランスを提供するために2つのアプリケーションが異なるアベイラビリティゾーンに配置されている場合、シングルモードとHAモードの両方をサポートします。 ほとんどすべてのクラウドプロバイダーは、データセンターの高可用性、複数のパス、コンポーネントの複製などを顧客に保証していますが、 まともなクラウドプロバイダーはすべて、定期的なバックアップも強く推奨しています。 どういうわけか、それはお互いに適合しませんよね？ 事実、クラウド内でどのようにねじれても、事故が発生します。 そして、顧客が一時的にデータにアクセスできなかったり、一定期間データの全部または一部を失ったりするような事故があります。 この例はありますが、まさにこのため、クラウドプロバイダーはクラウドへのバックアップを推奨しています。 したがって、クラウド内のデータの高可用性を確保することは、非常に要求される機能です。



高可用性機能により、2つのHAノードを2つの異なるアベイラビリティーゾーンに配置することにより、クラウドでの高データ可用性が可能になります。これにより、データセンター全体の事故が発生しても生き残り、このデータを使用するアプリケーションのデータに自動的に切り替えてアクセスできるようになります。









クラウド内の災害復旧Webサイト



DRサイトの作業は、レプリカが実行されている場合にのみ必要であり、DRサイトに切り替える必要がある場合は、残りの時間、ONTAPをオフにして費用を節約できます。これは、パブリックAWS / Azureクラウドでは1時間ごとに支払いが行われるためです。 ONTAP for Cloudでスケジュールを構成して、予算を節約できます。



クラウド内およびクラウド間レプリケーション



SnapMirrorテクノロジーを使用すると、1つまたは複数のパブリックプロバイダー内の複数のONTAP for Cloud間だけでなく、ONTAP for CloudからONTAP Select 、FASおよびNPSにデータを複製できます。









クローニング



ONTAPのもう1つの便利な機能は、クラウドストレージのパフォーマンスに影響を与えずにデータを薄くクローン化する機能です。 これにより、サイズに関係なく、バックアップのテスト、開発、分析、検証、カタログ化などのタスクのデータの多くの複製をパブリッククラウドに数秒で展開できます。







クラウドマネージャー



このようなONTAP Cloud機能を管理できます。



  • FAS、 NPS 、 ONTAP Select 、ONTAP for Cloud間のレプリカ
  • ONTAP Cloudの暗号化キーを管理します
  • AmazonとAzureで消費されるコンピューティングリソースとストレージリソースの量を表示します
  • ストレージスペースの管理：ボリューム、アグリゲート、ムーン、ファイルボールを作成する
  • ONTAPバージョンの更新






暗号化



2つのタイプの保存データ暗号化がサポートされています：AWSまたはONTAP。 2番目のケースでは、暗号化キーを自宅に保存できます。これにより、ONTAPの起動時にKMIPプロトコルを使用して、データのセキュリティを強化し、クラウド内のデータのロックを解除できます。







NetApp Private Storage（NPS）



これはONTAP Cloud製品に関連するサービスですが、必ずしもそれに伴うものではありません。これはニアクラウド製品でもあるため、一言言いたいと思います。 はい、はい、クラウドについてです。 これは基本的にコロケーションサービス（ Equinixデータセンター内 ）であり、独自のまたは一時的にレンタルしたFASを「クラウドの隣」に配置できます。 「クラウドの近く」とは、クラウドプロバイダーがストレージに直接接続していることを意味します。ストレージを使用するアプリケーションは、クラウドで実行されています。 クラウドプロバイダーからは、コンピューティングサービスのみがレンタルおよび支払われます。 データ自体はクラウドではなくストレージシステムにあります。 あなたとあなただけがデータを所有し管理します。 また、このシステムではONTAPソフトウェア（オペレーティングシステム）が使用されているため、SnapMirrorまたはSnapVaultを使用して、ONTAPを使用して他のシステム/プラットフォームと「そこから」および「そこから」のデータを複製できます： ONTAP Select （VSA SDS）、 NPS 、FAS、AFF、別のONTAPクラウド。











Data Fabric Cloud Sync



Data FabricはNetAppの開発戦略およびビジョンであり、Cloud Syncはこの戦略の一部です。 Cloud Syncサービスは、ONTAP Cloud、およびONTAPソフトウェアに基づいた他のプラットフォームにも関連しています。 Data Fabric Cloud Syncを使用すると、Amazon S3でNFSファイルバルーンデータを同期できます。 このサービスは、Amazon内の分析処理に使用できるデータアーカイブとして機能します。 ONTAPソフトウェアを備えたプラットフォームのいずれかのNFSボールにあるデータは、オブジェクトに変換され、Amazon S3ストレージに送信されます。 さらに、Amazon S3のデータを分析アルゴリズムで処理して、レポート（MapReduce、RedShiftなど）を抽出できます。



このサービスの適用例と、分析処理が必要になる可能性のあるデータタイプを示します。



  • ヘルスケア：ゲノミクス、医薬品製造、患者記録、疾病分析;
  • 石油およびガス：耐震分析、機械分析、商品およびサービスの需要予測。
  • ファイナンス：取引実績、ローン、市場リスク管理、不正検出;
  • 自動車：行動分析、サプライチェーン管理システム、予測分析。








プロトコル



CIFS（SMB）v2 / 3、NFS 3/4 / 4.1、pNFS、iSCSIがサポートされています。











ONTAP Selectと同様に、 ONTAP Cloud製品には多くの便利な機能と統合があります。



免許



2つのライセンススキームがあります：Amazon / Azureのすべてのサービスに対して顧客が支払う場合。 2番目のスキームは、顧客がプロバイダーからコンピューティングとデータを保存する場所のみを購入し、ONTAP CloudソフトウェアライセンスをNetAppから購入する場合です。



結論



ONTAP Cloudを使用すると、クラウド内のデータを自分で所有および管理し、他のデータセンターや豊富なONTAP機能にデータを複製する機会を得ることができます。 バックアップサイトを維持する必要がないため、バックアップデータセンターの展開はクラウドで安価になります。 複製されたデータに基づいて、シンデータクローニング機能を使用して開発とテストを展開できます。 クラウドへのシンレプリケーションは、ネットワークチャネルの負荷を軽減し、インフラストラクチャをパブリッククラウドに移行する簡単な方法です。
