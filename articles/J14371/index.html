1週間の過ごし方



これは、ジャックが作成したマップです。 そして、貴重な情報を運んでいない日常生活以下。 いわゆる「スタートアップ平日」。



先週の月曜日、私は自尊心のあるツアーリソースには旅行マップが必要だという考えに追いついた。

火曜日、私は木曜日まで何でもするつもりで仕事に座った。金曜日には飲み物が欲しかったので、土曜日には誕生日があり、日曜日には「Desperate Housewives」の第nエピソードを見るつもりだった（冗談なし、ひどい感染）、月曜日または火曜日までに作成物に光を当てた方がよいでしょう-私の観察によると、RuNetで最もアクティブな日です。 その後、LiveJournalでのプレゼンテーション投稿は、人々にもっと感謝されるでしょう。



ソフトウェア部分は特に難しくありませんでした。 GDライブラリは、透明なpngを完全に重ね合わせます。 ここでは私には思えますが、他の解決策はありません-マップの基板を作成し、赤い国を掛けます。



火曜日に実験しました。 つまり、テクノロジーをチェックしました。 1日の終わりには、スクリプトの作成元であるレンガ（作業コードの断片と、これがどのように機能するかについての計画）で武装していました。 これはすべて、Drupalのモジュールとして登録されたユーザーのデータベースに国情報を保存するために作成されたものであり、原則として、Drupalのようなサイトで獲得できるはずであるという事実によって、少し複雑になりました。



水曜日はつらい日でした。 自動化できない作業-地図を200か国にカットし、

この国の混合物を左上隅からピクセル単位でスクリプトに書き込みます。 そこで、結果の国の幅と高さを追加します。 一般に、1日で習得しなかった場合、2番目の時間はないことに気付きました。あきらめます。 たぶん私は約14時間座っていましたが、結局、自動化はスローモーションの助けを借りてのみ手の動きを追跡できるようになりました。 しかし、国が終わった4日目の夜、私は幸せでした。 さらに、私は多くを学びました。 たとえば、世界のクソ島国は面白い旗でばかげており、何らかの理由で私はミクロネシアに住むことに決めました。



木曜日はフラグctrl-c ctrl-vで再び通過しました。 国のチェックマーク、データベースへの入力、データベースからの削除などを含むフォーム。ある種の愚かな問題が発生しました。 たとえば、Drupalのチェックボックスにchecked属性を設定する方法を知りませんでした。 Drupalのフォームは非常に面白く作成されます。次のような属性を持つフォームの配列が作成されます。

$ form ['euro'] ['countries2'] = array（

'#title' => t（ '国'）、

'#type' => 'チェックボックス'、

'#description' => t（ 'お好みの色を選択してください'）、

'#o​​ptions' =>配列（ 's001' => 'アルバニア'、 's002' => 'ホンジュラス'）、

'#value' =>配列（ 's001' =>“チェック済み”、 's002' => NULL）、

）;

そして、この配列はdrupal_get_form（）関数に渡され、それ自体がすべてを描画します。

一般に、木曜日の夕方までには何も準備ができていませんでした。



金曜日。 芝生を緑色に塗るスタハノフのペースで、私は台本を終了しました。飲酒しないか、機嫌が悪いかのどちらかの見通しが迫っていたからです。 そして、見よ、3時までにはすべてがうまくいった。 確かに、5人まで「コシャチカミ」と戦った後、気分が再び良くなりました（週に2回目）

まあ、金曜日の夜はうまくいった。
