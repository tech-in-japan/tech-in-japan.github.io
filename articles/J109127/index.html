ルーレットでのFacebookビデオチャット





Facebookが独自のアプリケーションを作成するための人気のあるプラットフォームになったことは周知の事実です。 思った。 Facebookにはどのようなアプリケーションがありませんか？

Facebookユーザーにはコミュニケーションの機会がたくさんありますが、友人とのビデオチャットはありません。 アプリケーションのアイデアではないものは何ですか？

1か月後、 Facebookのビデオチャットが登場しました 。



機能の説明

  • リストからFacebookの友人に電話する
  • 見知らぬ人に電話する


確かに、あなたには疑問があります。プライバシーと変態の問題をどのように解決しますか。

  • Facebookはこの問題を部分的に軽減します。 見知らぬ人に電話しても、彼のプロフィールIDを確認できます。
  • チャットにはステータス設定があり、見知らぬ人とチャットすることができます。オフにすると、見知らぬ人はあなたに電話をかけることができなくなります。


テクノロジー

  • クライアント側：Flash Player 10.1 +、RTFMP、Facebook API
  • サーバー側：Adobe Ciruss、PHP、MySQL、CentOS
Flash Playerは、アドビの最近の開発の1つであるRTMFP（ リアルタイムメディアフロープロトコル ）を使用して通信します。 これは、サーバー間でこのデータを渡す必要なく、ユーザー間でデータを転送するためのP2Pプロトコルです。 このプロトコルは、バイナリデータでも複雑な構造でも、Flashオブジェクトを操作できるという点でも興味深いものです。

UDPプロトコルはトランスポートとして使用され、情報の重複や損失を引き起こす可能性があります。 しかし、マルチメディアデータをブロードキャストするには、UDPが最適なオプションです。

次に、FlashはAdobe Cirussから一意の識別子を取得します （以前はStratusでした）。 これはアドビの腸内にあるサーバーであり、対話者を見つけて接続するのに役立ちます。 アドビは開発者に無料でアクセスできます。



今後の計画

  • Twitterの友人やGoogleの連絡先と接続する機能を追加します
  • 会話したユーザーの最後の5つの画像と、異常者に苦情を送信する機能を保存します。 チャットから永久に除外します。
