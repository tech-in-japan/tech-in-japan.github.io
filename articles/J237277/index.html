私の指導経験

こんにちは



この投稿では、私の指導経験についてお話します。 特に、私が犯した間違いとその修正方法について説明します。 トレーニングに使用した方法を説明するとともに、今年度の私の計画について話します。



まえがき 私とそれが起こった方法


私は小さな会社でプログラマーとして働いており、 実際には enikeyのシステム管理者です 。 2014年の初めに、私は遠い昔に会計士経済学の学位を取得して卒業し、会計士に科目「1C：Enterprise」を教えると申し出た専門学校からアプローチを受けました。 恐怖から5 セントルーブルの目で、私は同意しました。



プロセス


教え始める


私は生徒に何を教えるべきかについての情報を見て勉強することから教え始めました。 カリキュラムの中で、彼らは私にカリキュラムを提供し、生徒はその科目が属するカリキュラムの枠組みの中で知るべきであると説明しましたが、方法論の開発や以前の教師が使用した他の資料を求めたとき、彼らはいないと答えました。 結局のところ、この方向への教員の急速な交代により、資料は残っていませんでした。 これは、すべてのタスクと資料をゼロから検索する必要があることを意味するため、次のステップは2冊の本を購入することでした。

  • 1C：教育版。
  • 1Cのビジネスオペレーション：Bukhalterii 8（バージョン2.0）。 タスク、決定、結果。


これらの本に基づいて、クラスを構築し始めました。



指導


パート0



手始めに、私は厳しい訓練をペアで導入しました。 ペアの先頭にある電話は、誰も電話をかけなかった後、すべてを1番机に置きます。 すべての違反は、レッスンへの入学を求めることにより処罰されます。 アカデミックアフェアーズのディレクター。



その結果、学生は電話に気を取られず、学習プロセスに少しでも没頭しました。



パート1



私の始まりは私の主な間違いです。



クラスの構築は、理論=>実践=>題材に関する質問の原則に基づいて行われました。



理論は完全に講義として構築されました。 理論的な部分では、プログラムの作業の原則、どのドキュメントが使用されるか、ドキュメントが移動するときに何が起こるかについて学生に話しました。過去1年間に会計資料を学んだと完全に信じています（ジャーナルの評価は資料の同化について話しました）彼らは私が話していることを理解しています。



実際の部分では、彼らはPCに座って、ドキュメントのガイダンスの下で情報ベースに入り、それから、そのようなドキュメントを2〜3個自作しました。 知識が増えると、渡されたすべてのドキュメントを使用して、組み合わせタスクが与えられました。 ペアの終わりに、カバーされた素材についてコントロールの質問が行われ、学生から質問がありました（もしあれば）。



10組（すべての組の1/3）後、学生の知識のチェックテストを行いました。 テスト、定義の知識、状況問題の解決策が含まれていました。 テストの最大ポイント数は100で、そのうち50-満足、70-良好、90-優れています。



テストおよび状況タスクには、会計アカウントおよび投稿の知識に関するタスクが含まれていました。

エリア1Cからのみの定義。

結果は私に衝撃を与えました。

グループの94％がテストに失敗し、平均23ポイントを得ました。



まあ、私はショックを受けていますが、結論が出ています。



私の間違いは、学生が他の科目（アカウンティング）で教材を学んだという盲目的な信念でした。



結論と解決方法：

  • 会計の基礎に関する資料を理論部分に含める必要があります。
  • 前のレッスンの理論の同化をテストするために、クラスの始めに投票を含める必要があります。




パート2



さて、結論が出され、材料はペアに処理されます。 これで、トレーニングはスキームPoll => Theory => Practice => Passed materialに関する質問に従って行われます。



別の10組の後、再び知識のコントロールテスト。 条件は同じです。



結果：62％がテストに失敗しました-30に失敗した人の平均スコア。 25％は満足、13％は良いです。 名誉なし。



統制工事の分析では、学生は口座を理解し始めたが、彼らはまだ投稿を理解していないことが明らかになり、3番目に、非現金と現金の絶対的な誤解に気づきました。



状況的な課題では、学生は2つの操作を1つに関連付けることができますが、3つ以上の操作を関連付けるのは困難です。 すべてのタスクを検討した結果、トランザクションの理解不足が問題を口頭で解決する能力に影響することに気付きました。



定義により、結果はほぼ同じであり、多少なりともわかっていました。



エラー：



  • 多くの理論がありましたが、そこから学生は材料を覚えることなく単に書いただけであり、それを理解しようとはしませんでした。 投票中の回答については、カップルの前でノートを読んでから忘れました。
  • 彼らは法案を学んだが、私は彼らにそれらの使用法と投稿方法を教えなかった。
  • 彼は事業運営をナビゲートすることを学びませんでした。
  • 実際のタスクは、操作の原理を理解するのではなく、テンプレートによって解決されます。 パターンは記憶されません。




結論：



  • よりアクセスしやすく覚えやすい形式で理論資料を作成する必要があります。
  • 投稿を作成する原則、それが何から、どこから来たのか、どこから、そしてなぜ出たのかについての理解を生徒に伝えること。
  • プログラムでこれらの操作を確立するために使用される文書の投稿と説明の準備により、ビジネス操作の決定を増やします。
  • さまざまなビジネス上の問題を解決するためのテンプレートの頭の形成。




パート3



2回目のテストのおかげで新しいデータを受け取ったので、私は再び資料を作り直しました。 ペアプロセスは同じスキームに従って実施されました投票=>理論=>実践=>対象となる材料に関する質問ですが、現在は理論的および実践的な部分が変更されています。



理論的には、次の変更が発生しました。



  • 講義という形で、基本的な概念だけが発行され始めました。
  • それらに対する文書と投稿は、ビジネストランザクションのプロセスを示す図の形式で発行されました。
  • この家には、プログラム内の特定の文書に記入するために、YouTubeのビデオへのリンクが与えられました。




練習：

  • 生活から取った問題を解決する（たとえば、学生と彼の家族が企業である「パンの購入」。学生は、会計エントリと1Cで使用するドキュメントを使用してパンを購入するプロセスを記述する必要があります）
  • プログラムで問題を解決しますが、実用的な場合は、問題を解決するための行動方針を説明する必要があります。

コース終了前の3ペアについては、前に説明したすべてのトピックが上記の概略図に変換され、1つのプレゼンテーションが行われました。



結果。



コースの最後に、学生は会計と1Cの包括的な試験を受けました。学生は、まず会計エントリで状況の問題を解決し、次に1Cプログラムの必要な書類に記入し、教師の前で決定を守る必要がありました。



結果。 試験はグループの100％に合格し（もちろん2人が後悔し、満席にならないように引き出しました）、そのうち37％が良かったです。



結論



この経験から、私はいくつかの結論を出しました。



  • 人間の賃金のために、多くの教師はそれに続く情報の量を求めません。その結果、生徒はよりリラックスした行動をとります。
  • 生徒の学習を成功させるために、大量の情報を提供する必要はありません。その多くは忘れられますが、基本的な知識を提供する方がよいですが、最も記憶され論理的に相互接続されるような方法で-図と表;
  • 多くの記事では、学習を成功させるために必要なものについて説明しています。 携帯電話/タブレット/ラップトップを奪わなければ、学生に興味を持たせることは不可能です。
  • ビデオチュートリアルの問題解決の優れたデモンストレーションは非常にうまく機能します。
