AndroidでMACアドレスを自発的に変更する問題を解決する

人生の最初の記念日のために、彼らは息子が長年夢見ていたタブレットを与えました。 しかし、残念なことに、彼のMACアドレスは、デバイスをリブートするたびに、またはWi-Fiのオン/オフを切り替えるたびにランダムに変化します。 他のAndroidデバイスではこれを観察しませんでしたが、ここで最初に遭遇しました。 自宅ではルーターがMACアドレスのホワイトリストを介してインターネットを配信するように構成されているため、タブレットが放電されるたびに電源が切れるまでインターネットに接続されずに（子供を追跡できず、時間通りにペットを飼うことができませんでした） MACアドレスを入力してから、ルーターを再起動します。 当然、これは私たちに合うことができませんでした。



解決策を求めて、私はほとんどグーグルで検索しなければなりませんでした。基本的に、特定のトピックについて、私は答えられないままの助けを求めて悲鳴にぶつかったからです。 ブルジョアのフォーラムでは、「違法であり、世界中のインターネットに接続されたデバイスに問題を引き起こす可能性がある」ため、一般的にそのようなタブレットを店に渡すことを勧めました。製造されたデバイスでは、MACアドレスを動的に変更し、どこにも登録しない方が簡単です。



最終的に、問題を解決するいくつかの方法に出会いました。 最も正気はここに記述されています：

時間： http : //w3bsit3-dns.com/forum/index.php?showtopic=512198&st=560#entry27219089 2： http : //habrahabr.ru/sandbox/62871/ （この方法は私には合いませんでした-それにもかかわらず、Androidデバイス、非常に個別）および3つ： http : //www.china-devices.com/Thread-Fix-Wifi-BT-Mac-Adress-NvRAM-error （最も手頃な価格）。



要するに、一番下の行は、16進数の5番目のペアから始めて、ファイル/データ/ nvram / APCFG / APRDEB / WIFIに新しいMACアドレスを登録する必要があるということです。 これを行うには、16進エディタでコンピューターにドロップし、その場所に戻し、読み取り専用権限を設定するのが最も便利です。 後者は常に強調されるわけではありませんが、重要です。



私の前のタスクはMACアドレスを変更することではなく、その自発的な変更を防止することだけであったため、指定されたファイル（つまり、 cd /data/nvram/APCFG/APRDEB



およびchmod 440 WIFI



）の書き込み権限を削除するだけで十分でした。 出来上がり！ デバイスを再起動した後、MACアドレスは変わりません。



PSもちろん、大切なファイルにアクセスすることはできません。最初にデバイスをインストールし（これは別の曲です）、ルートから作業できる何らかのファイルマネージャーをインストールする必要があります。 多くはルートファイルエクスプローラーに助言しますが、Androidのターミナルエミュレーターで作業することを喜んでいます（最初にsu



コマンドを入力することを忘れないでください）。



PPS重要： システムのすべての変更は、ご自身の責任とリスクで行ってください！ この記事の著者も、上記のリンクの記事の著者も、デバイスをブリックに変える責任を負いません。

重要2： デバイスをルーティングすると、保証が無効になります。

重要3：WIFIファイルを引き続き編集する場合は、 最初にバックアップすることを忘れないでください。



PPPSはい、ほとんど忘れていました。MTK8312チップセット上のTurboPad 721タブレットとAndroid 4.2.2が搭載されています。
