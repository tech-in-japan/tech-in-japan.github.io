家庭および小規模オフィス向けのバックアップ

こんばんは、親愛なるhabrayuzery。



まえがき



職場では、プロジェクトのバックアップコピーを作成する必要がありました。そのため、サーバーをインストールすることが決定されました。 高価で高品質で安定したサーバーにはお金がかからないため、通常のデュアルコアコンピューターを使用する必要がありましたが、同時に2台の信頼性の高いHDDとSmartUPSを購入しました。 結局のところ、電圧降下または停電の場合、サーバーを正しくシャットダウンして、データをそのまま残したいのです。 私たちはハードウェアを決定しましたが、ソフトウェアについて少し説明しましたが、選択は無料のCobianバックアッププログラムに任せました。このプログラムは便利で直感的です。 そして今、すべての詳細について。



私が持っているもの



サーバー：Core 2 Duo、2GBのRAM、サーバー用のSmartUPS、およびバックアップストレージ用の2​​つのHDD

2クライアント（常時バックアップが必要な2ワークステーション）



ソフトウェア


サーバーはMicrosoft Windows XP ProfessionalおよびCobian Backupになります

Microsoft Windows XP、Microsoft Windows 7、およびCobian Backupクライアント



ソフトウェアのインストールと構成





プリセット


クライアントのユーザー名はUser1とUser2になり、サーバーのユーザー名はAdminになります

バックアップはローカルネットワーク上のサーバーに送信され、構成されます。 サーバーには、ip： 192.168.1.1 、最初のクライアント（ User1 ）ip： 192.168.1.2 、および2番目のクライアント（ User2 ）ip： 192.168.1.3があります。

ディスクC：\ -サーバーの最初のディスク。OSがインストールされ、バックアップコピーが保存されます。

ディスクD：\ -サーバー上の2番目のディスク、最初のディスクCからのバックアップミラーがありますC：\



サーバーのセットアップ


まず、次のようにC：\ドライブにフォルダーを作成します。

C：\ backups \ user1- 最初のワークステーションのプロジェクトがバックアップされるフォルダー

C：\ backups \ user2-2 番目のワークステーションからのプロジェクトがバックアップされるフォルダー



ローカルネットワーク上のサーバーにアクセスするには、これらのフォルダーへの共有アクセスを開きます。

  1. フォルダCに移動します。\
  2. バックアップフォルダーを右クリックし、[ プロパティ ]を選択します。
  3. [ アクセス ]タブに移動し、[ ネットワーク共有とセキュリティ]セクションで、[ 共有するために このフォルダーを 開く ] を選択し、ネットワーク経由でファイルを変更できるようにします
  4. 適用してOK




サーバーにCobianバックアップをインストールして構成する


このプログラムはCobian Backup 11（Gravity）の公式ウェブサイトからダウンロードできます。



インストールを進めましょう。

  1. ダウンロードしたプログラムを実行する
  2. 最初のダイアログで、 言語 - ロシア 語 、 次を選択します
  3. 契約を読み、 条件に同意することをマークし、 次へ
  4. デフォルトのまま、 次へ
  5. [サービスの設定]セクションで、 [ ローカルシステムアカウント ]で [ 実行 ]を選択し、 [ 次へ]をクリックします。インストールを続行するよう求められたら、[ はい]
  6. インストールする
  7. インストールが完了したら、[ 完了 ]をクリックします。




次に、プログラムを構成します。

  1. インストール後、プログラムは最小化モードで起動します。トレイ（右下隅の渦アイコン）でプログラムインターフェイスを開き、左マウスボタンでプログラムアイコンをダブルクリックする必要があります。
  2. 上部のプログラムタイトルのすぐ下にある[ タスク]を開き、[ 新しいタスク]を見つけて選択します
  3. [ 全般]セクションの[タスク 名 ： 2番目のHDDにバックアップする]をアクティブにし、 [ タイムスタンプ付きの個別コピーを作成する]をオフにします
  4. コピータイプ： 増分 、つまり、ファイルが変更された場合、バックアップが実行され、変更されたファイルが作成されます。ファイルが変更されていない場合、コピーは行われません
  5. [ファイル]セクションの[ ソース]で ディレクトリ を追加します： C：\ backups 、[ 宛先パス]で 2番目のディスクを指定しますD：\
  6. スケジュールは好きなように設定できます。 毎日 13:10に私に合うでしょう
  7. [ 詳細設定]セクションは、サーバーの最後の設定になります。反対側のチェックボックスをオンにします。[ 同期] 、 空のディレクトリをコピーせず 、1つのチェックマークを削除します。
  8. 同意すると、十字架でプログラムを閉じることができます




顧客設定


Cobianバックアップでは、アカウントがネットワーク上で動作するためにパスワードが必要です（1人のユーザーがいて、パスワードがないと仮定します）。そのため、クライアントに新しいユーザーを作成します。



パスワードを持つユーザーを作成するには、 ユーザーアカウントに移動します 。

  1. win + Rを同時に押して、表示されるダイアログで、 コントロールuserpasswords2を書き込み、[ OK ]をクリックします。
  2. [ ユーザー名とパスワードが必要]を有効にします （チェック）
  3. 追加をクリックします
  4. ユーザー： バックアップ 、その他のフィールドは空白のままにして、次へ
  5. パスワード： 1234 （より難しいパスワードを考えることができます）
  6. アクセスレベル： 基本アクセス（ユーザーグループ） 、完了
  7. ユーザーのリストで、パスワードのないユーザー（ User1またはUser2 ）を選択し、「 ユーザー名とパスワードの入力が必要」のチェックを外して、適用します
  8. [ 自動ログイン ]ダイアログが表示され、[OK]をクリックして[ ユーザーアカウント]を閉じます




ワークステーションにCobianバックアップをインストールして構成する


Cobian Backupのインストールプロセスは、ポイント5を除いてサーバーと同じです。

[サービスの設定]セクションで、 [ 通常のアカウントで実行 ]、 [アカウント： User1 （最初のクライアント用）]、[パスワード： 1234] 、[ 次へ ]を選択します 。



設定は、3つの点を除いてサーバーと同じになります。

「 一般」セクションの「 ジョブ名 ： サーバーへのバックアップ 」で 、 「タイムスタンプ付きの個別コピーを作成 」の選択を解除します 。



「 ファイル」 、「 ソース」セクションで必要なディレクトリを追加し、「 宛先パス 」で「 追加 」をクリックして「手動 ： \\ 192.168.1.1 \ backup \ user1 、 Accept 」を選択します。



毎日 13:00に スケジュール

この記事では、ネットワークポリシーとグループユーザーポリシーについては詳しく説明していません。これは、誰もが異なる方法で構成できるためです。

サーバーにはパスワードを設定しないでください。パスワードを設定しないと、記事に記載されていない「問題」が発生します。
