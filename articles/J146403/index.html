MultiWiiフライトコントローラー（ヘリコプター、飛行機、ヘリコプター）について

注意、記事は古くなっていますが、それでも教育目的に使用することができます

私は突然、さまざまな航空機（LA）を制御するためのオープンで無料のプロジェクトについて、もっと詳しく知りたいと思いました。 一般的に、最近まで、マルチローターシステム（ヘリコプター）、カメラサスペンションの安定化、およびベータ版のフライングウィングの安定化のみを目的として設計されていましたが、devファームウェアによると、飛行機とヘリコプターはすぐに正式に追加され、フライングウィングフライトの安定化はすでにリリースされていますベータテストから。



少しの背景



プロジェクトの開発の最初の段階では、センサーに大きな問題がありました-単に存在しないか、信じられないほど高価だったため、このフライトコントローラーの最初のセンサーは任天堂WiiのWii Motion Plus（WMP）でした。タイトルにWiiというプレフィックスを付けました。 しかし、時間が経つにつれて、センサーは安価になり、コンソールからコントローラーを使用してMultiWiiコントローラーを作成する人はほとんどいなくなりました。



構成部品



コントローラーの主な頭脳は、AVRプロセッサー、またはむしろ、さまざまなセンサーがI2Cポートにくっついている単純なArduinoボードです。 現時点では、飛行するにはArduino互換のボードとジャイロスコープが必要になります。たとえば、ITG3205は中国のWMPコントローラークローンから見た（これはスラングではありませんが、文字通り見た）ですが、このような構成ではヘリコプターを制御するのが非常に困難になります地面に平行（したがって、ドリフトを排除）する必要があります。 そしてここで、 加速度計は救助に来ます。そして、それは常に「底」、すなわち、どこを知っています 航空機を地面と平行に保つのに役立ちます。 したがって、静かで安定した飛行を行うには、Arduinoと2つの3軸センサー（ジャイロスコープと加速度計）が必要です。







さらに、磁力計、気圧計、GPS受信機などのセンサーを追加できます。

磁力計 （別名コンパス）は、航空機が地球の磁場の方向に対してどのように回転するかを認識し、デバイスの実際の機首の方向に関係なく、ヘリコプターがパイロットから「前方」コマンドに飛行する場合、機首方位、GPSナビゲーション、または初心者の気楽なモードで役立ちます「そしてコマンドのパイロットに」「戻る」。 磁力計は、電源、ワイヤ、モーター、および電磁波を発生させる可能性のあるすべての磁場の影響を非常に受けやすいため、航空機のフレームを設計するときは、このセンサーをすべての金属および磁化された場所からできるだけ離して配置する必要があります。

気圧計 （別名気圧センサー）は高度の維持に役立ちますが、高度の保持はセンサーの精度に直接依存することに注意してください。たとえば、BMP085（多くのフライトコントローラーにインストールされている）は+ -1メートル以下の精度を提供します。 最良の条件下では、航空機は1メートル以上の高さで飛行します。 より高度なMS5611センサーは、最大10cmの精度を実現できます。 気圧計は、プロペラからの周囲の圧力、風、または空気の流れの変化の影響を受ける可能性があるため、このセンサーを「フードの下」に置くか、フォームゴムをマイクに接着することをお勧めしますが、センサーの穴を塞がないようにすること（非常に重要です）そこに接着剤）。

GPS受信機 、便利であると同時に役に立たないもの。 一方では、位置を保持して出発点に戻る（家に帰る）などの良いものを含めるのに役立ちます。他方では、この点は非常に近似しており、受信機の精度、目に見える衛星の数、または気象条件に依存するため、女性のロッカールームの窓の近くにぶら下がることができます直径が約3〜10メートルの円内（および平坦化されたボールに適切な気圧計を追加した場合）。

執筆の時点で、ソナー（超音波距離計）と光学式変位センサー（マウスにインストールされている）を追加する試みがあります。 ソナーを使用すると、低高度（最大約3メートル）で高さ（約1 cm）を非常に正確に保持できますが、表面に大きく依存します。たとえば、背の高い草やマウンドは干渉を引き起こし、高さの急激な変化を引き起こします。 ヘリコプターは急激に跳ね返ります。 しかし、 光学センサーは、GPS受信機が位置をより正確に保持するのに役立ちます。

このプロジェクトには、さまざまな目的でサポートされている多数のセンサーがあり、リリースごとにその数が増えます。プロジェクトの開放性を考慮して、独自のセンサーを取り付けることができます。

既製のコントローラーボード



上記で書いたように、利用可能なセンサーとArduinoのセットからコントローラーを作成できますが、既にAVRチップと必要なセンサーのセットを備えた既製のショールを購入する方が簡単です（そして、多くの場合、より収益性が高くなります）。

最も人気のあるマザーボードはCrius MultiWiiで、LiteとStandart Editionの2つのバージョンがあります。最近、新しいバージョンが登場しました。便利なI2C入力と小さなサイズの存在のみが異なります。

Crius MultiWii Liteエディションには、ジャイロスコープと加速度計の2つのセンサーしか搭載されていません。 Standart Editionは、Liteで使用可能なセンサーに加えて、BMP085バロメーターとコンパスを備えています。 ボードの新しいバージョンには、UART GPSレシーバーであるGPSモジュールと、GPSとMultiWiiコントローラーの仲介役として機能するAVRチップ（Arduino）を接続するためのI2Cコネクターがあります（近い将来、メインブレインをアンロードするためにGPS機能を完全に使用します） 。

一部のオンラインストアでは、他の同様のコントローラーを見つけることができるため、2つまたは4つのセンサーと328pチップ（または1280/2560）を備えた何かに出くわすと、99％の確率でMultiWiiコントローラーになります。



Criusのピンと接続のニュアンスの説明


ヘリコプター に関する 私の 記事を既にお読みいただき、速度コントローラーとコントロールパネルのレシーバーがコントローラーに接続されていること、そしてカメラジンバル安定化コントロールのサーボを取り付けることができることをご理解ください。



写真は上記のデバイスを接続するためのコネクタを示しています。 ご覧のように、各コネクタは3つの接点を組み合わせています。S-信号、 + -電源（+ 5V）、 G-接地。 コネクタには、 A桁とD桁の形式の表記もあります-速度コントローラ、サーボ、および場合によっては無線制御受信機の追加のチャネルが接続されます（Arduinoの結論に対応）。 THR、ROLL、PITCH、YAW、MODEとマークされた入力は、レシーバーの対応する出力を接続するように設計されています。

各スピードコントローラー（ESC）から3本のワイヤーがあり、そのうちの1本は+ 5v（真ん中の赤またはオレンジ）ですが、ESCの1つからのみCriusボードに電力を供給する必要があります。レシーバーおよびサスペンションサーボ。 これを行うには、すべての（ボードに電力を供給するものを除く）スピードコントローラーのコネクターから中央のワイヤーを取り外し、それらをコンシューマーまたはアイソレートに接続する必要があります。

他の消費者に接続するには、コームから2つのコンタクトを切り離し、それらをはんだ付けします。





そして、速度コントローラからの電力線を消費者の電力線に接続し、熱収縮するように締めます。





赤いワイヤーを半分に折り、熱収縮で絶縁することで、不要なワイヤーでも同じことができます。





コントローラーにはワイヤーのセットが含まれています-レシーバーを接続するために使用する必要がありますが、信号線、1つの共通接点（グランド）を接続し、速度コントローラーの1つから電源線を接続するだけで十分です。



MultiWiiの設定のニュアンス



まず、結果のコントローラーのファームウェアを更新する価値があります。 公式の MultiWii リポジトリからファームウェアをダウンロードするだけです （その理由がわからない場合は、DEVバージョンをダウンロードしないでください）。ArduinoIDEにアップロードして、フライングマシンのパラメーターを構成し、選択しますconfig.hファイルに一連のセンサーを追加し、結果のスケッチをコントローラーにロードして、設定に328pチップが含まれるボードの1つを選択します（arduino UNOは除く）。

MultiWii 2.0以降、Criusコントローラはセンサーセットのリストに登場しています。

//#define CRIUS_LITE // Crius MultiWii Lite





//#define CRIUS_SE // Crius MultiWii SE





コントローラーのタイプに一致するいずれかのコメントを外します。

初心者にはいくつかの非常に一般的な問題があります（バージョン2.0に関連）。

  • hexocopterの場合、レシーバーは機能しません-config.hで#define A0_A1_PIN_HEX行のコメントを外す必要があり、追加のRUチャネルを使用するには、＃define RCAUXPIN8または#define RCAUXPIN12行のいずれかのコメントを外して、レシーバーチャネルをD8またはD12に接続します
  • すべてのモーターが同期して動作するわけではありません-速度コントローラーを調整する必要があります。そのためには、すべての信号とすべての共通（グランド）接点を接続し、レシーバーのスロットル（CH3）チャンネルに接続し、電源線の1つを接続してから調整する必要がありますスピードレギュレーターの指示に従いますが、通常はスロットルスティックを最大位置に設定し、スピードコントローラーに電源を投入する必要があります（ネジを外すのを忘れないことを望みますか？そうでない場合、非常に苦痛になります）、次にスロットルスティックを下側（最小）に下げます 横たわる
  • モーターが間違った方向に回転します-速度コントローラーからモーターへの2つの接点を変更する必要があります
  • モーターが作動しない-RU設定を確認してください。MultiWiiConfiguratorで、ハンドルの極端な位置が1000と2000の値に対応していることを確認してください。1500。そうでない場合は、それを修正し、速度コントローラーを再調整します




グラフィカルコンフィギュレーター



グラフィカルコンフィギュレータは処理中に記述され、クロスプラットフォームJavaアプリケーションであり、現在32ビットおよび64ビットアーキテクチャのLinux、Windows、およびMacOSオペレーティングシステム用にコンパイルされています。





外観は少し怖いですが、多くの場合、ここに登る必要はありません。 フライトコントローラーをコンピューターに挿入した後、コンフィギュレーターを起動します。 左上隅で、コントローラーポートを選択し、[スタート]ボタンを押します-グラフ上の曲線とその横の数字が実行されます（理想的には、すべてがスムーズになります）。





グラフの右側には、ヘリコプターの3次元モデルと、コンパスおよび傾斜インジケーターがあります。 デバイスが水平線に平行で、画像内のPITCHとROLLが傾いている場合は、コントローラーがこの位置を記憶し、常にそれを目指してCALIB_ACCボタンを押します。





グラフの上には、PIDコントローラーの設定、モードのアクティブ化、およびいくつかのステータスデータがあります。 READボタンを押して変更するために必要な値を確認するには、数字の付いたウィンドウをクリックしてマウスを動かし、保存します-WRITEを押します。 審査官の複数のPID値を一度に変更しないでください。変更すると、何が良くなったのか、悪くなったのかがわかりません。 ここで何をどのようにひねるのかを読むことができます 。
