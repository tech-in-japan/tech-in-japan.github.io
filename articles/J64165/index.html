App Engineで期待されること

  1. 大きなファイルを扱うためのサービス。
  2. 受信トレイのサポート
  3. XMPP API
  4. データセットのマッピング機能のサポート。
  5. データベースへのクエリのカーソル-1000レコードの制限を克服します。
  6. カスタムアプリケーションの例外的な状況（例外）に関する警告（例外）のシステム。
  7. ストレージシステムをダンプして復元します。

  1. 大きなファイルを扱うためのサービス。

    DataStoreを使用する場合、1メガバイトのエンティティのサイズには制限があります。 1つの解決策は、ファイルをユニットに分割し、それらを個別のエンティティとして保存することです。 このような操作用の別のライブラリがあります-google-file-service 、その説明はApp Engine Webサイトのレシピ （クックブック）にもあります。 ライブラリは、標準のApp Engineライブラリセットの一部ではありません。

    どうやら、Googleは同様のライブラリをユーザーに提供するか、明示的にパーツに分割することなく大きなファイルを保存できる新しいサービスを導入する予定です。
  2. 受信トレイのサポート

    App Engineには、HTTPおよびHTTPSプロトコルを介してのみ外部リソースを操作したり、Googleアカウント（Googleアカウント）を持つユーザーに代わって電子メールメッセージを送信したりできるサービスがあります。 受信トレイサービスの意味は次のとおりです。Googleアカウントを持つユーザーのメールボックスからメッセージを受信する機能を備えたライブラリがあります。
  3. XMPP API

    電子メールの送信と同様に、Googleアカウントを使用してXMPPメッセージを送信するためのライブラリがあります。
  4. データセットのマッピング機能のサポート

    ここで何を意味するのか分かりません。 たとえば、データストアからのデータをXMLにマッピングするということですか？ または、オブジェクト間のいくつかの新しいタイプの接続。
  5. データベースへのクエリのカーソル-1000レコードの制限を克服します。

    現在、データストアから要求されたときに返されるレコードの数には制限があります。 多数のレコードを処理するには、データストアに対して複数のリクエストを行う必要があります。

    この場合、Googleプログラマーは、他のデータベースでカーソルとして機能する新しいクラスを導入する可能性があります 。 1000レコードを超えるときにカーソルを繰り返し処理すると、次のレコードの要求が暗黙的に行われます。
  6. 警告システム

    キャッチされなかったすべての例外のダンプが記録され、管理パネルに表示されます。
  7. ストレージのダンプと復元

    データベースの現在の状態をダンプし、ホストしているApp Engineに保存したり、ローカルマシンにダウンロードしたりできます。 逆ダンプ回復機能も使用可能になります。
