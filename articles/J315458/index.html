ブラックフライデー-サービスの低下を避ける方法





すぐに、いわゆる「ブラックフライデー」、割引の日が来ます。何百万人もの買い物客が、巨大な割引でクールな製品を探してオンラインとオフラインの店を襲いました。 この日のバイヤーの流入は、何かを販売するほとんどすべての企業で注目されています。 ちなみに、オンラインストアが再び顧客の休日を開催するサイバーマンデーもあります。



トラフィックと顧客の数が増えているという事実は良いことです。 しかし、あなたは本当にブラックフライデーとサイバーマンデーの準備ができていますか？ より正確には、インフラストラクチャはこの2日間のイベントに備えていますか？ 実際、最近では実際の店舗の棚以外にこれらのサーバーを準備している企業は多くありません。 そして、適切に準備されていないと、最も重要な瞬間に失敗する可能性があります。



耐え難いほど苦痛にならないように、何をする必要がありますか？



1.昨年の顧客活動を評価することにより、機器/インフラストラクチャの機能を評価します。 必ずしもあなた自身の顧客ではありません-結局のところ、これはあなたにとって最初の「ブラックフライデー」かもしれません。 ネットワーク上のドキュメントと出版物を分析して見る必要があります。そのようなドキュメントと説明の多くは公開されています。 たとえば、1秒あたりのトランザクション数、トラフィックのレベルを評価し、データを自分用に調整します。 その後、必要に応じて、サーバーとラックを追加して、機器を準備する価値があります。



2. バックアップがあることを確認し 、予期しない機器障害が発生した場合に計画を分析します。 それでも問題が解決しない場合は、問題なくデータを回復できますか？ また、テストする価値があります。 これを行う方法については、過去の記事の1つで書きました。



3. ハードウェアおよびソフトウェアインフラストラクチャを確認します 。 この段階では、ハードドライブの正常性レベル、UPSの状態、およびバックアップエネルギーインフラストラクチャ（もしあれば）を評価する価値があります。 すべてがうまく動作するはずです。 そうしないと、機器またはデータベースが最も不適切な瞬間に失敗する可能性があります。



4. サーバーまたはそのグループは正常に動作し、データベースの動作を保証しますか？ これを確認するには、ストレステストを実施する必要があります。 また、Webアプリケーションが最適に構成されていることを確認してください。 SQL Serverとデータベースレプリケーションのクラスタリングは、機器の高可用性とフォールトトレランスを確保するための優れたソリューションです。







5. 統計システムを更新します 。 この段階では、アーカイブに使用されなくなった情報を送信する価値があります。 すべてが正しく行われている場合、このステップにより、クエリのパフォーマンスを含め、ソフトウェア部分のパフォーマンスが大幅に向上します。 インデックスの再編成中に、アーカイブデータをバックアップまたは他のデータベースに送信できます。



6. 現在の問題を解決します 。 問題がある場合は、たとえ小さなものであっても、時間「H」の前に解決する必要があります。 実際には、負荷が増加するまで問題は小さいということです。 交通量が大幅に増加すると、すべての問題がすべての見事な栄光に現れ、インフラストラクチャとサービスの崩壊につながる可能性があります。 データベースの速度を低下させるものをすべてクリーンアップします。



7.特定のレベルのトラフィックが予想される場合は、機器をさらに高いレートに調整します 。 訪問者が予想よりも多く来てくれると決めたらどうなるでしょう？ 申し訳ありませんが安全です。 もちろん、「耐久性」の複数のマージンは必ずしも必要ではありません。ここでは、状況を適切に評価する必要があります。 全体として、小売業者は通常、インターネット訪問者の10倍の成長について語っています。



8. 必要なすべてのアップデートとパッチ、特にセキュリティシステムに関連するものがインストールされ 、アップデートされていることを確認します 。 ユーザーデータは暗号化する必要があり、サーバーは攻撃の可能性に備えています。 はい、今日は善意のユーザーだけがさまざまな企業のサーバーに興味を持っているわけではありません。



一般的に、通信会社と小売業者にとってブラックフライデーは良い面と悪い面の両方があると言えます。 多くのバイヤーがいるという事実にもかかわらず、ブラックフライデーとサイバーマンデーの準備には多くの時間がかかり、多くのリソースを消費します。 十分に注意して準備しないと、問題が非常に大きくなる可能性があります。 場合によっては、小売業者は、必要なサービスを提供するサーバーの崩壊により、1分あたり5,000ドルの損失を報告しました。 誰もお金を失いたくないので、よく準備することをお勧めします。
