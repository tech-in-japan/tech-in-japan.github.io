NC 8.5のVisual Studio 2015で新しいプロジェクトを作成する方法



今回は、好奇心のために、より軽量のIDE Xamarin Studio（ MonoDevelop ）でプロジェクトを作成しましょう。



プロセス全体をネタバレの下に隠します。



Xamarin Studio IDEでNanoCAD 5.1のC＃プロジェクトを作成する方法
まず、新しいプロジェクトを作成し、C＃クラスライブラリを選択します。







彼に名前をつけて







次に、RMBでプロジェクトをクリックし、「パラメーター」ボタンを選択して、プロジェクトを構成します。







.Net 3.5バージョンを選択







次に、デバッグの便宜上、「F5」キーを押してNanocadの起動を設定します

（コードを変更するたびに、Nanocadを完全に再起動する必要があります）







次に、図の右側に表示されているライブラリが必要なライブラリへのリンクをロードします







「ローカルにコピー」項目からmapimgdライブラリのチェックを外すことを忘れないでください







実は、F5を押してすべてが開始することを確認するコードを記述することは実際には残っています。







ライブラリをロードするには、NanoCADコンソールでNetloadコマンドを入力し、dllを選択します（通常はプロジェクトフォルダーにあります。たとえば、C：\ Users \ b \ Dev \ habr \ XMTree \ XMTree \ bin \ Debug）



毎回これをしないために

アドレスCに移動します：\ ProgramData \ Nanosoft \ nanoCAD 5.1 \ DataRW（変更される場合があります）および次の内容のload.configファイルを検索または作成します 



<root> <list> <module path="C:\Users\...\bin\Debug\ .dll"/> </list> </root>


その後、プログラムの起動時にライブラリが自動的にロードされます。



Linuxでも同様の操作を実行できます。Linuxでは、NanoCAD 5.1を実行するにはWineを、プログラミングにはMonoDevelopが必要です。 前回の記事で詳しく説明しました 。



残念ながら、Wineで問題が発生し、NanoCADの色がどこかで壊れていたため、Linuxツリーは次のようになります。

次に、右側の「リンク」ボタンを右クリックして、NanoCADのすべてのバージョンのSDKでインストールしたライブラリにリンクを追加します。



このプロジェクトでは、mapimgdだけで十分です（NC 8.5ではmapibasetypesも必要です）。

「ローカルにコピー」プロパティのチェックを外すことを忘れないでください







次に、RMBでプロジェクトをクリックし、「プロパティ」ボタンを選択して、プロジェクトを設定します

デバッグの便宜上、「F5」キーを押してNanocadの起動を設定します







（コードを変更するたびに、Nanocadを完全に再起動する必要があります）



ライブラリをロードするには、NanoCADコンソールでNetloadコマンドを入力し、コンパイル済みのdllを選択します（通常はプロジェクトフォルダーにあります。たとえば、C：\ Users \ b \ Dev \ habr \ XMTree \ XMTree \ bin \ Debug）。

 <root> <list> <module path="C:\Users\...\bin\Debug\ .dll"/> </list> </root>


その後、プログラムの起動時にライブラリが自動的にロードされます。
