LinuxでCastle Wolfensteinに戻るの起動

少し前まで、伝説的なidsoftwareからの栄光のゲームReturn to Castle WolfensteinのコードはGPLライセンスの下で開かれました 。 ところで、これはそうです。





そして、物理学の試験に合格した後、少しリラックスしたかったのです。 RTCWをプレイすることにしました。 結局のところ、Linuxでゲームを実行することは現実的です。



（「Express Installation」という見出しの下には、説明のない一連のコマンド（Debian OSファミリ用）があります-多くのテキストを読みたくない人のために。この方法の完全な成功は保証されていません）。



注意！ この記事は、基本的なLinux端末スキルを前提としています。



誰かidsoftwareのTTimoが、LinuxでRTCW 1.41を実行する際のFAQを投稿しました。



打ち上げを成功させるには、次のものが必要です。

  • ゲームディスク
  • Linuxのインストールファイル
  • libstdc ++ 2.10-glibc2.2をインストール


順番に



ゲームディスク



城ウォルフェンシュタインに戻るゲームのディスクがあると仮定します。 それから、mp_pak0.pk3、mp_pak1.pk3、mp_pak2.pk3、pak0.pk3、sp_pak1.pk3、sp_pak2.pk3のリソースを持つファイルのみが必要です。



Linuxインストールファイル



ファイルはidsoftware ftpサーバーにあります 。 しかし、そこからの私の揺れは不安定でした。速度は増加または減少していました。 したがって、 念のため、 それをdropboxにアップロードしました 。 誰かが自分のパブリックストアにミラーリングするといいでしょう。



Libstdc ++ 2.10-glibc2.2ライブラリ



ライブラリのバージョン2.10は廃止されたと見なされるため、リポジトリから削除されます。 （この記事の執筆時点では、libstdc ++リポジトリバージョン6-4.3）。 しかし、古いバージョンは完全にグーグルです：

  • Ubuntu （ i386への直接リンク ）
  • Debian （ i386への直接リンク ）


その他の場合には、 確実な代替手段があります。



設置



必要な材料がすでに手元にある場合は、ゲームのインストールに直接進みましょう。

いくつかの問題



抽出に失敗しました



理由は、解凍が実行される/ tmpはゴムではないからだと彼らは言います。 （そして確かに）。 この問題を解決するには、キー--target / custom / temp / folderを使用してインストーラーを実行します 。ここで、/ custom / temp / folderは、一時ファイルの解凍に使用される任意のフォルダーです。 例：

Copy Source | Copy HTML



  1. sudo ./wolf-linux- 1.41b.x86.run --target / media / mydisk




音が出ない



次の行を実行可能ファイル/ usr / local / games / wolfenstein / wolfspに追加します。

  1. echo "wolfsp.x86 0 0 direct" | sudo tee / proc / asound / card0 / pcm0p / oss


前に

  1. exec ./wolfsp.x86 "$ @"




ゲームを開始するとき、パスワードを入力する必要があることに注意してください（したがって、質問は部分的に開かれています）。 端末ではなくメニューからゲームを開始する場合は、たとえば、sudoの代わりにgksudoを使用します。

それでも音が出ない場合は、oss-compactパッケージをインストールしてみてください。
