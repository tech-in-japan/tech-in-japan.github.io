アイスクリームキャンディ、またはアプリケーションに少量の材料を持ち込む方法

Androidの5番目のバージョンは、 ほぼ半年前にリリースされました 。 それにもかかわらず、市場のほとんどのアプリケーションは、Holoスタイルのままです。 新しいマテリアルスタイルはまだ平均的な開発者にとって厳しいものではないか、Android Lはまだ普通の開発者にしっかりと参入できていません。



それはともかく、新しいデザインパラダイムは「良い企業」によって積極的に推進されており、多少の矛盾はあるものの、かなり見栄えが良い。 そして、ますます多くの優れたウィザードが登場し、シンプルな開発者がモバイルUIの変化する世界の「ストリーム」にとどまるのを助けています。



最近のように、長い間更新されていないアプリケーションを新しいスキンでラップすることにした場合、このツールとライブラリのレビューによりN時間を節約できます。



3.5年以上にわたり、私はHabrの集合的な精神に関与してきました。 Android Development Kitを最初にダウンロードしてインストールしてからもう少し時間が経ちました。 貢献する時が来たと思います。



そのようなレビューはすぐに関連性を失うことを認識して、私はそれを書くことができませんでした。 これにはいくつかの主な理由があります。

  • 私のアプリケーションの新しいデザインに取り組んで、2か月でGithubへのリンクを少しずつ集めて、大量のGoogleページをシャベルにしました。
  • 夏がもうすぐ来るので、休暇の時間です。休暇中に開発者が何をしているのか（病欠など）を正確に想像できます。 だから今、私は、それが時間だと思う。
  • 現在インターネット上では、ほとんどのデザイン要素に対して、そうでない場合でも、すでに様式化を見つけることができます。


免責事項
もちろん、このレビューは完全なもの、絶対的なもの、または行動を促すものではありません。 おそらく私がこれを書いている間に、Googleはすべてのインターフェース要素に共通の互換性ライブラリをリリースしました。

さて、最後。 それにもかかわらず、私は（肉屋なしではなく）約6ヶ月前にAndroid Studioに切り替えました。 そして、この開発環境に恋をしました。 したがって、プロジェクトにライブラリを含めるすべての方法は、gradleに関連します。 Eclipseを使用すると、すべてが同時にシンプルで複雑になります。開発者は、このIDEでライブラリプロジェクトを接続する方法を知っていると思います。

NavigationDrawer







誰にもわかりませんが、この要素は本当に好きです。 さらに、彼はすでにAndroidの標準になり、ハンバーガーのアイコンにもなりました。 確かに、ここでGoogleは妨害なしではできませんでした：

  • 引き出しは、横に動く3本のストライプの形でActionBarDrawerToggleとともに現れました。
  • ActionBarDrawerToggleが変更されました（多くの人が「ハンバーガー」が矢印に変わって気に入った）。
  • Drawerのルールはツールバーの概念とともに変更されました。NavigationDrawerはアプリケーションのタイトルと重なります。 私の意見では、これは2つの奇妙な結果をもたらしました-ActionBarDrawerToggleアニメーションは今では見えません（そしてそれはまったく明確ではありません）、そしてスワイプまたは戻るボタンでメニューを閉じることができます。


すぐに予約します-私のアプリケーションでは、2番目のオプション（3番目から返されたオプション）をそのままにしておきます。



Drawer自体は、特にプロジェクトをゼロから作成する場合は、アプリケーションに簡単に統合できます。 ウィザードで適切なタイプのアプリケーションを選択するだけで、引き出し自体のアクティビティ、フラグメント、およびクラスを含むテンプレートを取得できます。 この瞬間は省略できると思います-操作は標準です。 古いアプリケーションを再編成しようとしている場合は、android-support-v4互換性ライブラリが必要になることに注意してください。



appcompat-v7ライブラリは、ActionBarDrawerToggleを新しい外観にするのに役立ちます（以前の外観と同様に、SDKのextrasディレクトリにあります）。



build.gradleファイルの依存関係ブロックの1行で、これらのライブラリをモジュールアセンブリに含めることができます。 



dependencies { compile 'com.android.support:appcompat-v7:22.0.0' }


22.0.0-ライブラリの現在のバージョンの番号（SDKマネージャーで覗くか、「+」を入力すると、開発環境は利用可能な最新バージョンを使用します。いわば、回避するために現在のバージョン番号を常に表示してください。 。）。



新しいガイドラインを（メニュー全画面で）定型化するには、少し汗をかかなければなりません。 まず、標準のActionBarをToolBarに置き換える必要があります。次に、アプリケーションスタイルに要素を追加します。 実用的なソリューションはStackoverflowにあります 。



メニューのテキストの色のインデントと16進コードを長時間設定したくない場合は、これを行うライブラリの使用方法をご覧ください。 記事に対するコメントは、よくあることですが、非常に価値がありますので、読むことをお勧めします。 いずれにせよ、あなたが決める。



NavigationDrawerの別のカスタマイズ
顧客が、開く際にメニューを暗くするのではなく、レイヤーをぼかしたいと思ったときに、 興味深いライブラリに出会いました。



指摘された困難のうち、コンテンツにスクロールリストがある場合、ライブラリは多くの場合そのステータスを追跡できず、メニューを開くと「古い」コンテンツのぼかしが表示されますが、リストは実際には異なります。 しかし、注意は間違いなく価値があります。



FloatingActionButton







別の新しいUI要素。 画面上のメインアクションの1つ（または複数）を強調表示し、ユーザーの注意を引くのに役立ちます。

残念ながら、この要素は（他の多くの場合と同様に）サポートライブラリに含まれていないため、この場合はサードパーティのソリューションを使用する必要があります。

それらのいくつかを選択し、それぞれについて、それらを使用するときに生じたいくつかの考えを書きます。

  • 親愛なるジョン・ホーガンの決定は、Habréで説明されています 。 私は2つの不快な瞬間を選び出します。
    1. プロジェクトにコードを含める必要があります。これにより、可読性が低下します。
    2. 開発者は消失/外観処理を残しました。
    一般に、あなたはただ取り、使用することができます。
  • このライブラリ。 「ネストされた」ボタンは非常にうまく実装されていますが、「致命的な欠陥」があります。ボタン自体の操作は実際には非常に限られており、クリックのみです。 もちろん、たとえばListViewをスクロールするときにボタンを自動的に削除/表示できる「リスナー」はありません。
  • これが解決策です。 私にとっては非常に良い実装です。 2つのサイズがあります（コンテンツを表示するとき、ミニボタンは補助アクションとして使用されることを思い出します）。これは、自動非表示のスクロール可能な要素で非常にオリジナルの「ドッキング」で、マークアップファイルによる柔軟な設定があります。 私がつまずいた唯一の問題は、このボタンを別のカスタムOnTouchListenerと友達にすることができないことでした。 ListViewでスワイプして閉じるのを使用するので、これは私にとって重要でした。
  • 私のプロジェクトに選ばれました。 シンプルでカスタマイズ可能なOnTouchと競合せず、2回のクリックで統合できます。
  • このライブラリは 、記事の準備中にすでに見つかっているため、テストされていません。 誰かがそれについての情報を提供できるなら、コメントを書いてください、そして、私はそれを記事に加えます。


ご覧のとおり、選択できるものはたくさんあります。

シークバー







このウィジェットに関連付けられている非常に楽しい思い出はありません。 実際、私のアプリケーションの古い画面レイアウトによると、横向きを使用する場合、 垂直の SeekBarsが必要でした。

聞かないでください。



したがって、Holoテーマでそのようなウィジェットを見つけた場合、Materialと通常のウィジェットを少し見つけました。 しかし、最終的には、画面を少しレイアウトし直さなければならなかったので、そのためにSeekBarに感謝しています。



現在、このアイテムにDiscreteSeekBarを使用しています。 ライブラリはまだ非常に若く、jcenterにも存在しないという事実にもかかわらず、私にはぴったりです。強力なカスタマイズ、リスナーの便利な実装、アニメーションなしのDrawableの使用。これにより、ブレーキなしで非常に迅速に作業できます。



さらに、このウィジェットのライブラリはほぼ唯一です。 MaterialRangeBarもありますが、単純なSeekBarを実装していないため、私には合いません。
