象とクジラのどちらが勝つか？ モスクワMySQLユーザーグループとPostgreSQLの合同会議。

PostgreSQLとMySQLは、インターネット上で最も一般的な2つのデータベース管理システムです。







MySQLは、Wikipedia、facebook、flickr、Digg、YouTubeなどの負荷の高いプロジェクトで、habrahabrで、DrupalとWordpressを実行しているすべてのサイトで使用されています。



Postgresは劣っていません。Skypeはそれなしでは機能しません。アメリカの人気ソーシャルネットワークmyyearbook.comおよびhi5.comで使用され、国内のMy Circle、smotri.com、およびWorld of Tesenはそれなしでは機能しません。

Drupalの最近のバージョンはPostgreSQLのサポートを宣言しており、Wordpressの代わりにMovable Typeを使用できます。 Postgresを使用する他の興味深いアプリケーションの中で、1Cを思い出すことができます：Enterprise 8およびその他の企業情報システム。



どのDBMSが優れていますか？ どのタスクのために？ 異なる環境で同じ問題をどのように解決しますか？



2009年1月22日、モスクワで、クラブの支援を得て、 ビジネススタイル.RUクラブ、ユニークなイベントが開催されます-モスクワMySQLユーザーグループとPostgreSQLの合同会議。



2チームの専門家（各システムのハッカー開発者、大規模プロジェクトの主要な技術専門家）が両方のシステムの長所と短所を説明し、ゲストからの質問に答えます。

  • コンスタンティン・オシポフ、Sun / MySQL
  • Victoria Reznichenko、Sun / MySQL

  • アンドレイ・スミルノフ、NetStream
会議の主催者は、Postgresmen社のNikolay Samokhvalovです。



この会議は、州立大学- ポクロフスキー大通りの経済学部の建物11 （Kurskaya / ChkalovskayaまたはKitay-Gorod地下鉄駅）のV-201講堂で開催されます。 入り口は、ポクロフスキー大通りからE棟を通ります。



開始は19-00です。
