Microsoft Mailアプリとsmtp経由の送信メールの破損



先日、私のディレクターは、彼の真新しいMicrosoft Surface RTタブレットがメールを送信したくない理由を見つけるように私に尋ねました。

グーグルは、「アダルト」Windows 8とタブレットの両方で、メールアプリがsmtpで動作しないという事実のみを確認しました。 解決策が見つかりませんでした。 マイクロソフトの従業員とチャットサポートを利用してチャットすることで、以前Googleで見つけたマイクロソフトフォーラムのトピックへのリンクを取得できました。





問題は次のように現れました。メールアプリから手紙を送信しようとすると、画面の右上隅に「user@mydomain.com is unavailable」というメッセージが表示されました。

Postfixメールサーバーログで、次のことがわかりました。



11月15日15:05:11 nanoweb postfix / smtpd [96503]：user.host.ru [ip.ip.ip.ip]から接続

11月15日15:05:11 nanoweb postfix / smtpd [96503]：user.host.ru [ip.ip.ip.ip]から確立された匿名TLS接続：暗号AES128-SHA（128/128ビット）を使用したTLSv1

11月15日15:05:11 nanoweb postfix / smtpd [96503]：user.host.ru [ip.ip.ip.ip]から切断します




MSフォーラムを再読した後、smtpとTLSに問題があり、ソフトウェアの更新を待つことだけが残っていることを認めたサッパーからのメッセージを見つけました。 実際、「use TLS」のチェックを外すと、メールアプリは最初にTLSを介して接続を試みます。サーバーバナーで要求されたTLSサポートを確認しますが、何らかの理由で自己署名証明書でTLSサーバーに接続できません。 SSLを使用する場合、同じことが起こるようです。



そして、「PostfixがTLSが無効になっている別のポートを持っているとしたらどうでしょう？」



マニュアルを掘り下げた後、master.cfに次の行が表示されました。



2567 inet n-n--smtpd

-o smtpd_sasl_auth_enable = yes

-o smtp_use_tls = no

-o smtpd_use_tls = no




タブレットのメールアプリで、ポート2567を送信用に設定し、「SSLを使用する」のチェックを外して、機能しました！





このソリューションは一時的なものとして提供され、メールサーバーの管理者を対象としています。

この問題は少なくとも2週間存在し、まだ修正されていません。
