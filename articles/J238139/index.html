快適なWeb​​開発：Parallels Desktop 10 + Vagrant

Web開発の世界は現在変化していますが、これは特に、開発者が作業環境のサポートと同僚との同期に多くの時間と労力を節約できる仮想マシン管理ツールであるVagrantによるものです。

今日は、新しいParallels Desktop 10の機能の1つである、開発者ユーザーから最も多くの質問とフィードバックを受けたVagrantとの統合についてお話します。 正直なところ、これはParallels Desktopの開発者自身にとって最もお気に入りの機能の1つです。

Web開発でVagrantが必要な理由がまだ分からない場合、またはVagrantの基盤としてMacでVirtualBox仮想マシンを使用している場合は、次のとおりです。









データベース、お気に入りのフレームワーク、nginx、redisなど、作業マシンにWeb開発環境をデプロイしようとしたことがある場合、おそらくセットアップがサイトの動作とは非常に異なるという事実に出くわしました生産中。 使用されているパッケージのバージョンは一致せず、一部はOSでまったく配信できないため、代替品を探す必要があります。



1つの方法は、仮想マシンの環境を上げることですが、これは希望どおりには機能しません。仮想サーバーを割り当てた場合とほぼ同じ方法で、すべてを手動で行う必要があります。



Parallelsでは、約1年前に社内開発チームが拡大し始めたときに同様の問題に直面しました。チーム内で積極的な共同作業を確立し、さらに実験する必要があることが明らかになりました。 そして、ここではすべてが非常に単純です。ある技術に「触れる」必要があり、できるだけ早くこれを行う余裕がない場合、毎日のタスクのプレッシャーの下では決してそれを行いません。 そして、LinuxとWindowsおよびMacの両方でサポートされているインフラストラクチャがあるという事実に関連した特異性がまだあります。 これについては詳しく説明しません。 ユースケースに興味がある人は誰でも、それに関するレポートからビデオを見ることができます。



ソリューションの検索は、 Vagrantにつながりました。 これは、コマンドラインインターフェイスを備えたUNIX向けユーティリティです。 操作の原理は非常に簡単です。必要な仮想マシン（プロセッサコアの数、メモリサイズ、ディスクサイズ、OSディストリビューションなど）を説明するテキストファイルがあります。 このファイルに基づいて、vagrant upコマンドを入力すると、新しい仮想マシンが作成され、使用可能な状態になります。 さらに進んで、必要に応じて「仮想マシン」を構成するスクリプトをこのファイルに添付できます。MySQL、Django、nginxなどをインストールします。 また、これらはすべて、構成時に自動的に行われます。 結果に満足していることを理解したら、これらのファイルをgitに入れて同僚に送信できます。 この簡単な方法で、標準化された作業環境が得られます。



Vagrantのもう1つの便利なプロパティは、「仮想マシン」内のコードで作業ディレクトリを「トス」できることです。 これにより、手動で同期する必要がなくなり、ファイルを保存するとすぐに仮想マシンに即座に表示され、ブラウザで変更をすぐに確認できます。



当時、VagrantはVirtualBoxでのみ機能していたこと、または元の著者のプレミアムプラグインの助けを借りてVMwareに接続できることを言及する価値があります。 私たちだけでなく、 市場ではすでに Mac上のどの仮想化ソリューションが最適と見なされるかを決定しているため、Parallels Desktopとの独自の統合を行うことが決定されました。 そして今、1年の開発の後、Vagrant用のオープンプラグインをリリースしましたが、その品質は公式の統合に劣りません。 Vagrantの作者は、もちろん、VMwareとの統合を販売することでお金を稼ぎたいので、Vagrantのメインページには無料ソリューションへのリンクは今のところありません。 また、一部のユーザーにとっては、Vagrantの一部の統合またはプラグインの存在もニュースになる可能性があるため、Parallels Desktopを閉じてVirtualboxを起動します。次に、この場合の非常に簡単な手順を説明します。



使用例



今少し練習。 MacでLinuxとDjangoを使用して作業環境を整える方法について説明します。



最初に、Vagrantが必要です。Vagrantは、 公式サイト 、 Parallels Desktop 10およびvagrant-parallelsプラグインからダウンロードでき、次のようにインストールできます。 



vagrant plugin install vagrant-parallels


準備済みのリポジトリをダウンロードします。 

 git clone https://github.com/Parallels/vagrant-django-example.git cd vagrant-django-example


そして、仮想マシンの準備を開始します。 

 vagrant up --provider=parallels


プロセスが完了すると、数分で仮想マシンを完全に使用できる状態になります。 sshを使用して内側に「ジャンプ」し、Djangoプロジェクトを作成するだけです。 



 vagrant ssh # (  ) django-admin.py startproject vagrant_django . nohup python manage.py runserver [::]:8000 >&/dev/null &


すべて、これで仮想マシンを終了できます。 



 exit


ブラウザでhttp：// localhost：8000を指定すると、サイトが表示されます。 そして、そのコードは、あなたがvagrantを実行した現在のフォルダーで利用可能になります。



環境コンストラクター



上記で行ったことは、誰かがソフトウェアの設定方法を説明したVagrantfileを書くことを意味します。 単に実験したい場合は、スーパーマーケットのように、仮想マシン構成をコンポーネントに「入力」できる優れたPuPHPetプロジェクトがあります。 PuPHPetはvagrant-parallelsをサポートしているため、必要な設定から数回クリックするだけです。 出力時に、このシステムは、必要な準備済みファイルすべてを含む小さなアーカイブをダウンロードすることを提案します。 上記のツール以外で作業する必要はありません。



Vagrantで他にできること



実際、Vagrantの適用範囲は開発環境に限定されません。 たとえば、自宅では、C ++およびObjective-Cで記述されたネイティブコンポーネントのアセンブリをテストするために使用します。 他にできることの例：



  • 珍しいソフトウェアを試す必要がありますか？ 浮浪者 ; インストール済み。 実験した; 放浪者は破壊する 。
  • 継続的インテグレーション：Jenkinsから仮想マシンを起動し（これについては、404FestカンファレンスでSamaraでレポートを作成します）、コンポーネントを組み立ててからマシンを折りたたみます
  • Webプロジェクトのテスト。コードへの新しいコミットに従って、Vagrantで準備された仮想マシンにサイトが自動的に送信されます。
  • デザイナーとQAエンジニア向けに、「サンドボックス」を取得する
  • そしてもちろん、ChefやPuppetなどの高度な構成管理ツールを使用している場合、Vagrantはレシピとマニフェストの変更を実行する唯一のスマートな方法です




なぜ10番目ですか？



実際に、すでにvagrant-parallelsを試した人は、Parallels Desktop 8および9のバージョンですでにそうしています。なぜ投稿で10番目について話しているのですか？ 彼女の前は、このプロジェクトはGitHubでの無料開発でした。 Parallels Desktop 10では、Vagrant Parallels Providerのサポートが正式に発表されただけでなく、メイン機能以外のほぼすべての機能をサポートしています。 これと共有フォルダーツールの使用、およびネットワークの微調整は、一般に、通常のシェルスクリプトではVMで実行できないことです。 最後に、待望のポートフォワーディング機能は、グラフィカルインターフェイスではなく、 'prlctl'ユーティリティを介して登場しました。 これにより、TCPおよびUDPを介して、Mac上の特定のポートから仮想マシンポートにデータを転送できます。 たとえば、仮想マシンでWebサーバーを起動し、ポピーでローカルlocalhost：8080を開いて、テスト済みのサイトにアクセスできます。 理論的には、細心の注意を払って、1024未満のポートを転送することもできます（ポートを開くプロセスはMacでrootとして実行されるため）。



VirtualBoxはどうですか？



彼らは、Parallels DesktopとVirtualBoxを同時に起動することはできず、何かがクラッシュしたことを私たちにたくさん書きました。 多くのユーザーにとってこれは議論ではありませんが、公平には、異なるテクノロジーに基づいた2つの仮想化システムはうまく機能しません。 Twitterでこのような問題を抱えている人を特に探し、浮浪者の類似を使用することを推奨するようになりました。 現在、この問題は解決されており、両方の製品を同時に起動できます。 Macのリソースで許可されている場合、戦闘機の1人が死亡することなく、シャットダウンせずに、すぐにParallelsにレポートを送信する必要なく、同時作業が行われます。



おわりに



明らかに、Vagrantの使用は簡単で高速です。 しかし、最も重要なことは、非常に迅速に、高い関心をもって、投資された投資とリソースを回収するツールです。 したがって、新しいフレームワーク、開発技術、またはツールの研究に時間を投資する選択肢があれば、間違いなくVagrantにアドバイスします。

さて、Parallels Desktopを既に使用している開発者にとって、開始は完全に簡単です。
