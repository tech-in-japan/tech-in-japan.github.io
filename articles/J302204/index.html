私たちは、ヘビと鳥（または、あなたが好きなサーカスと果物）を渡り続けます。 前回の記事では、通知を作成する方法を学びました。今度は、あまり人気のないウィンドウ、つまりデータ入力フォームとファイルナビゲーターに進みます。



データ入力フォーム



InputBoxの類似物を作成する最も単純な必要性から始まりました。テキストの入力フィールドを持つウィンドウです。 この「単純なニーズ」の結果として、UIフォームを作成するためのいくつかの便利なグッズと基本クラスが生まれました。

同じInputBox。 ここではすべてが簡単です-kivy.uix.textinput.TextInputが基礎として使用され、ポップアップウィンドウにハングアップしました。 この方法を適用します： 

XTextInput(title='  ?', text='   ?')

シンプル。 必要に応じて-1行で。 textパラメーターを使用して、入力フィールドの既定値を設定します。フィールドを空にする必要がある場合は、このパラメーターを除外します。



ところで、入力フィールドで「Enter」を押すと、ウィンドウは「OK」ボタンを押したと認識します。



入力したデータを取得する方法は？ 非常に簡単-次のウィンドウを閉じるハンドラーを作成します。

クラスXNotes



複数行テキストを入力するためのプロファイル内のみの同じXTextInput ： 

 XNotes(title='', text='\n ...\n ')


結果：





XNotesはXTextInputの子孫であり、 XTextInputについて上記で説明したものはすべて、 XNotesに対して有効です。

  • 入力フィールドで「Enter」を押すと、新しい行にジャンプします。
  • textパラメーターは入力として文字列を想定しています。 「\ n」を追加することで複数行になります。




XSliderクラス



kivy.uix.slider.Sliderを搭載したポップアップ。 この方法を使用します。 

 XSlider(title='', buttons=[''])


デフォルトでは、スライダーは[0（ 最小 ）、1（ 最大 ）]の範囲で移動し、現在の位置（ 値 ）は0.5になります。 必要に応じて、これらのパラメーターは簡単に他のものに置き換えられます。 

 XSlider(title=', dB', buttons=[''], min=-100, max=12, value=-10)


結果：









XFormの子孫と同様に、このクラスでは.get_value（）メソッドも使用できます。このメソッドは、ウィンドウクローズイベントハンドラーで使用できます。 さらに、このクラスを使用すると、on_changeイベントを使用して、スライダーの位置を即座に変更できます。 いつものように、イベントハンドラについて説明します。 

 def volume_change(instance, value): instance.title = ': %0.0f' % (value * 100) XSlider(title='', buttons=[''], on_change=volume_change)


ビデオデモは、on_changeイベントの操作のより「例証的な」例を示しています。 ここで、ハンドラーコードはポップアップのサイズを変更します。



XAuthorizationクラス



このクラスを使用すると、ユーザー認証用のシンプルなフォームを作成できます。 このフォームには、ログインを入力するためのフィールド、パスワードを入力するためのマスキング文字を含むフィールド、および「自動的にログインする」チェックボックスがあります。 この例： 

 XAuthorization()
空の認証フォームが作成されます。 次のパラメーターを使用してフィールドに入力します： login 、 passwordおよびautologin ： 

 XAuthorization(login='login', password='password', autologin=True)


結果：





フォームからデータを受信するために、ウィンドウクローズイベントハンドラーを作成します。 .get_value（）メソッドを使用して、対応するフィールドのコンテンツを取得します。 

 def my_callback(instance): if not instance.is_canceled(): login = instance.get_value('login') password = instance.get_value('password') autologin = instance.get_value('autologin') print('LOGIN: %s, PASS: %s, AUTO: %s' % (login, password, autologin)) XAuthorization(login='login', password='password', autologin=True, on_dismiss=my_callback)




XFormクラス



基本クラスについて知っておくべきことは何ですか？

  • 基本クラスには.get_value（id）メソッドがあり、指定されたidを持つ要素をフォーム上で検索し、その値を返します。 パラメーターはオプションです。指定しない場合、メソッドはidが指定されたフォームから最初の要素を取得し、その値を返します。 この実装は、要素値の取得をサポートしています： TextInput、Switch、CheckBox、Slider 。
  • クラスには抽象メソッド._get_form（）があり 、その結果はフォームのコンテンツになります。
  • このクラスは、デフォルトで次の一連のボタンを設定します：「OK」、「キャンセル」。


OK、フォームを書いてみてください：

そして結果を見てください：





.get_value（id）に加えて、クラスにはvaluesプロパティがあります 。 これは、キーがフォーム要素のIDであり、値がこの要素の値である辞書です。 このプロパティを使用して、ハンドラーを異なる方法で実装できます。

XFileOpenクラス



ファイルを選択するためのウィンドウを作成します。 例： 

 XFileOpen()


複数のファイルを選択する必要がありますか？ いくつか選択してください。 同時に、デフォルトのパスを示し、選択したファイルのリストを取得するハンドラーを説明します。 

 def my_callback(instance): if not instance.is_canceled(): print(u'Path: ' + instance.path) print(u'Selection: ' + str(instance.selection)) XFileOpen(multiselect=True, path=u'c:\python27', on_dismiss=my_callback)


クラスにはすでに空の選択肢をチェックするための実装が含まれており、ユーザーが少なくとも1つのファイルが選択されるまで「開く」ボタンをクリックしてウィンドウを閉じることはできません。 そのため、ハンドラーでは、「キャンセル」をクリックしたときにのみチェックを記述します。



そして、クラスのロシア語のローカライズを作成しようとしています。 ここのボタンの簡単な変更は不可欠です;タンバリンと少し踊らなければなりません：

XFileSaveクラス



保存したファイルの名前を入力するウィンドウ。 選択プロパティと複数選択プロパティを除き、同様の方法で使用されます。ここでは必要ありません。 代わりに、次を使用します。

  • filename-ファイル名の入力フィールドに関連付けられており、デフォルト値を設定するか、入力されたテキストを読み取ることができます。
  • .get_full_name（）は、ファイル名を含むパスを返すメソッドです。


例： 

 def my_callback(instance): if not instance.is_canceled(): #   print(u'Path: ' + instance.path) #    print(u'Filename: ' + instance.filename) #     print(u'Full name: ' + instance.get_full_name()) popup = XFileSave(filename='file_to_save.txt', on_dismiss=my_callback)


このクラスには、入力フィールドにテキストがないかどうかをチェックする実装も含まれているので、気にしません。

XFilePopupクラス



kivy.uix.filechooser.FileChooserに基づく -ファイルシステムをナビゲートするためのウィジェット。 それに加えて、ポップアップウィンドウには次の要素が含まれます。

  • ボタン「アイコン」-アイコン形式でコンテンツを表示するモードをオンにします。
  • ボタン「リスト」-リスト形式でコンテンツの表示モードを有効にします。
  • 「新規フォルダ」ボタン-新しいディレクトリを作成します。
  • 現在のディレクトリを表示するラベル。


FileChooserには便利なプロパティの大きなリストがありますが、 XFilePopupでは「必須」プロパティのみがレンダリングされます。

  • path-ウィンドウの作成時にコンテンツが表示されるディレクトリ。 デフォルトは「/」です。 ディレクトリを変更するときに値を変更します。
  • multiselect-複数の要素の選択を有効/無効にします。 デフォルトはFalseです。
  • dirselect-ディレクトリ選択を有効/無効にします; デフォルトはFalseです。 重要 ： dirselect = Falseの場合 、ディレクトリはワンクリックで開きます。 with dirselect = True-シングルクリックで選択したディレクトリをマークし、ダブルクリックで開きます（モバイルデバイスではあまり便利ではありません）。
  • selection-選択されたアイテムを含むリスト。
  • browser - FileChooserクラスのオブジェクト。他のプロパティを使用する機能を「出力」します。


ウィンドウを作成し、パスとボタンのセットを表示します（ XFilePopupには独自のボタンのセットはありません）。 そして、すぐにハンドラーを記述して結果を取得します。 

 def my_callback(instance): print(u'Path: ' + instance.path) print(u'Selection: ' + str(instance.selection)) XFilePopup(title=' ', buttons=['Select', 'Close'], path=u'c:/python27', on_dismiss=my_callback)


パスパラメータに注意しましょう-値はUnicode文字列として渡されます。 これは、非ASCII文字（windows-1251エンコード文字など）を含むファイル名とディレクトリを正しく表示するために必要です。



その結果、フォルダーを選択する機能なしで、1つのファイルのみを選択できるナビゲーターを取得します（XFileOpenのアナログ）。

XBaseクラス



クラス階層からわかるように、 XBaseはこの記事で説明したすべてのクラスの祖先です。 プロパティとメソッドのいくつかにすでに慣れています。ここでは、オブジェクト指向で送信される機能の完全なリストを示します。

  • auto_open-ブール値プロパティ。 Trueの場合、クラスのインスタンスを作成すると、ウィンドウが自動的に表示されます。 それ以外の場合、インスタンスにウィンドウを表示するには、 .open（）メソッドを呼び出す必要があります。 デフォルトはTrueです。
  • ボタン -ポップアップコントロールボタンのセットを定義するリストプロパティ。 デフォルトは[]です。
  • button_pressed-押されたコントロールボタンの名前を含む文字列プロパティ。 on_dismissイベントハンドラーで使用できます。
  • .is_canceled（）は、押されたボタンの名前がXBase.BUTTON_CANCELの場合にTrueを返すメソッドです。
  • ._get_body（）は抽象メソッドであり、その結果はウィンドウのコンテンツになります。


また、このクラスはデフォルト値をスーパークラスのいくつかのプロパティに設定します。

  • min_width = 300dp 、 min_height = 150dp 、 fit_to_window = True - XPopupのプロパティ、以下の詳細。
  • size_hint =（。6、.3）-Popupのプロパティは、ウィンドウの相対サイズを設定します。
  • auto_dismiss = False - Popupのプロパティ。ウィンドウ自体の外側のインターフェイスの任意の領域をクリックしてウィンドウを閉じることを禁止します。


ご注意 このクラスは、 Popupクラスのcontentプロパティを無視します 。



XPopupクラス



Popupの子孫であり、 XBaseのスーパークラスです。 モバイルデバイスでのウィンドウ表示修正の注意深いファイル処理用に設計されています。 その特性：

  • min_widthとmin_height-ウィンドウの最小の幅と高さ。デフォルト値はありません。 size_hintと連携して動作します。 一番下の行：指定されたmin_widthおよび/またはmin_heightに対して、 size_hint値が修正され、ウィンドウサイズが許容される最小値以上になります。
  • fit_to_window-ブール型プロパティ「ウィンドウに収まる」、デフォルトはFalseです。 サイズと連携して動作します 。 一番下の行：fit_to_window = Trueの場合、 サイズ値は、ポップアップウィンドウのサイズがアプリケーションウィンドウのサイズより大きくならないように調整されます。


ご注意 説明されているプロパティの適用は、ウィンドウが表示されているときにのみ発生します。
