AppleがiOSで新しいSan Franciscoフォントを思いついて導入したのはなぜですか？





Appleは、最初のiPhoneの発売以来、iOSのシステムフォントとしてHelveticaフォントセットを使用しています。 また、Mac OS Xでは、Yosemiteバージョン10.10からLucida Grandeに置き換えられました。 では、なぜAppleは今や世界のお気に入りのフォントを取り除くことにしたのでしょうか？



iOS 9は公式にすべての人が利用できるようになり、サンフランシスコと呼ばれる新しいフォントファミリが控えめにHelvetica Neueに取って代わりました。





Helvetica（左）とSan Francisco（右）



この時点で、それらはすでにApple Watchで使用されていました。 現在、サンフランシスコは、すべてのプラットフォーム製品（Apple Watch、iPhone、iPad、およびMac）の単一の標準フォントになっています。



Appleは、最初のiPhoneの発売以来、HelveticaファミリをiOSのシステムフォントとして使用しています。 また、Mac OS Xでは、Yosemiteバージョン10.10からLucida Grandeに置き換えられました。 では、なぜAppleは今や世界のお気に入りのフォントを取り除くことにしたのでしょうか？



小サイズ-Helvetica弱点



Helveticaは小さなテキストには適さないと考えられています。 HelveticaがMac OS X Yosemiteで以前のファミリを置き換えたとき、多くの設計者はこの置き換えが不適切であると感じました。





エリック・シュピーカーマンのブログの「Helvetica sucks」 。



Helveticaの読みやすさは次のとおりです。 小さなテキストを入力して「ぼかし」ます。 同時に、そのフラグメントの一部は「ミックス」されるため、コンテンツの解析が困難になります。 彼らは、Apple Watchの小さいテキストを読みやすくするためだけにAppleがSan Franciscoファミリーを開発したと言っています。







ただし、最新のスマートデバイスの解像度は印刷版の解像度を上回り、iPhoneのテキストはApple Watchのテキストほど小さくはありません。 なぜAppleはApple Watchだけでなく、iOSとMac OS Xも置き換えたのですか？



サンフランシスコ-多様


サンフランシスコのフォントには、読みやすい多くの機能があります。 実際、Apple Watch用のSan FranciscoバリアントとiOS / Mac用のバリアントは、2つの異なるフォントです。



「SF」と呼ばれるフォントファミリはiOS / Macに使用され、Apple Watchは「SF Compact」を使用します。 「o」や「e」などの丸い文字の例で違いに気付くことができます。 SF Compactの垂直線は、SFよりも平らになっています。







この違いにより、SF Compactを使用して入力したテキストでは、文字間のインデントが増加し、その結果、Apple Watchなどの小さなデバイスからテキストを読みやすくすることができます。



これに加えて、各ファミリーはさらに2つのサブファミリー「テキスト」と「ディスプレイ」に分割されます。 Appleはそれを「光学サイズ」と呼んでいます。 テキストサブファミリーは小さなテキスト用で、ディスプレイは大きなテキスト用です。





San Francisco Font Family



前述のように、Helveticaなどのグロテスクなフォント（または切り刻まれたもの）では、隣接する2つの文字「mix」、および「a」、「e」、「s」などの文字は、小さいテキストサイズで互いに非常に類似します。





DisplayサブファミリーとTextサブファミリーのフォントの文字間のインデントの比較





単一文字の例を使用したディスプレイおよびテキストサブファミリーのフォントの比較



この場合のテキストサブファミリーは、表示サブファミリーの文字に比べて文字間のインデントが増加するように設計されており、それらのギャップが拡大されて、小さいテキストサイズで読みやすくなっています。



サンフランシスコはダイナミックです



サンフランシスコの素晴らしい点の1つは、ヘッドセットが動的に最適化されることです。 表示されたテキストのサイズに応じて、表示とテキストが相互に置き換えられます。 ここでのしきい値は20ptです。







設計者とエンジニアは、ファミリーから適切なオプションを選択することを心配する必要はありません。 たとえば、システムフォントをUILabelに追加するだけです。 システム自体が、必要なヘッドセットを決定します。



ただし、サンフランシスコのフォントが本当に印象的なのは、コロンの表示方法です。 通常、他のフォントでは一番下の行のすぐ上に表示されるため、数字の間にある場合、コロンは垂直方向の中央に配置されません。 ただし、San Franciscoファミリーのフォントでは、このような配置は自動的に行われます。





コロンを自動的に中央に垂直に揃えます



サンフランシスコはデジタル時代から来ました



ご覧のとおり、San Franciscoフォントは、あらゆるサイズのあらゆるデバイスでテキストを簡単に読むことができるように設計されています。



彼らが交換したHelveticaは、デジタルデバイスがなかった1957年にスイスで作成されました。 それにもかかわらず、それはまだ多くの企業で企業フォントとして使用されており、間違いなく、将来的には優れたクラシックフォントとして使用されるでしょう。



対照的に、サンフランシスコは最新のフォントです。 彼のヘッドセットは、状況に応じて動的に変化します。 デジタル時代の一種の「ネイティブフォント」と呼ぶことができます。
