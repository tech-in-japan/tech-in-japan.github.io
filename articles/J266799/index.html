Siebel Email Marketing Serverのセットアップ

はじめに



Siebel CRMは、販売、マーケティング、サービスという3つの主要なプロセスグループ内で顧客と対話することを目的としています。 Siebelはデフォルトで、ほぼすべての主要な通信チャネル（電子メール、SMS、CTI、Web）をサポートしています。 この記事では、Siebelがメールチャネルを通じてマーケティングに提供する機会のみを取り上げます。



目的：マーケティングメールマーケティングのインフラストラクチャをセットアップします。 このインフラストラクチャでは、メッセージの配信不能に関する情報を収集し、メッセージを開き、メッセージ内のリンクをクリックする必要があります。 手紙は擬人化する必要があります。つまり、名前でクライアントにアピールする必要があります。 受信者リストは、キャンペーンの開始時に指定された基準に従って自動的に生成される必要があります。 特定の提案、特定のユーザーに関する特定の情報を示すことが可能であるべきです。



注：インターネット上でレターを送信するために使用されるSMTPプロトコルは、配信レポートおよび受信者のメッセージ（開封、削除、スパムとしてマーク付け）を意味しません。 幸いなことに、デフォルトでは配信の問題（バウンスされたメール）のレポートがあります。 問題が発生していない場合は、手紙を受け取ったことを確実に述べることができます。 手紙の開始を追跡する方法もあります。 残念ながら、この方法には制限があります。



Siebelで電子メールメーリングを構成するには、ソフトウェアモジュールの基本セット（Siebel Webサーバー、Siebelアプリケーションサーバー、Siebelゲートウェイ、Siebelデータベース）に加えて、追加のものをインストールする必要があります。



  1. SMTPリレーサーバー -電子メールを送信でき、認証なしでやり取りできるメールサーバー。
  2. Eメール送信デーモン（ESD） -テンプレートおよび特定のデータセットから文字を収集し、SMTPリレーサーバーに送信できるユーティリティ。
  3. バウンス処理デーモン（BHD）は、配信の問題（バウンスされた電子メール）のレポートを受信して​​解析できるユーティリティです。
  4. クリックスルーデーモン（CTD）は、HTTP要求（クリックスルー、リソース要求）を処理できるユーティリティです。 レターを開くことと、このレター内のリンクをクリックすることに関する情報を記録するために、これが必要になります。
  5. ネットワークゲートウェイ -一方ではインターネットからアクセス可能で、他方では会社の内部ネットワークに特定の要求を転送できるサーバー上のルーター。 バウンスされた電子メール（ポート25）をBHDを使用してサーバーに転送し、HTTP要求（ポート80）をCTDを使用してサーバーにブロードキャストするには、その構成が必要です。
  6. ビジネスインテリジェンス（BI）は、顧客がデータを選択するための複雑な基準を定義できるツールです。 インフラストラクチャ内で、BIには2つのタスクがあります。マーケティングキャンペーンのクライアントベースをセグメント化すること、レターの本文に置き換えるデータを含むリストを作成することです。 BIは、Siebelデータベースとリポジトリの両方からこれらの操作のデータを取得できます。








基本設定



この時点まで、Siebelのマーケティングキャンペーンがプロジェクトでまったく使用されていなかった場合、次のことを行う必要があります。

  1. 必要なサーバーコンポーネントを実行する

    docs.oracle.com/cd/E14004_01/books/MKTG_InstallAdmin/mktg_installadmin_installmktg4.html#wp186724
  2. マーケティングワークフローをアクティブ化する

    docs.oracle.com/cd/E14004_01/books/MKTG_InstallAdmin/mktg_installadmin_installmktg5.html#wp186830
  3. BIとの統合を構成します（既にインストールされ、動作していると仮定します）

    docs.oracle.com/cd/E14004_01/books/MKTG_InstallAdmin/mktg_installadmin_installmktg7.html#wp224347




Siebel Email Marketing Serverのコアコンポーネントをインストールして構成する



ESD、BHD、およびCTDは、まとめてSiebel Email Marketing Serverと呼ばれます。 これらのユーティリティの配布は、support.oracle.comの「パッチおよび更新」セクションからダウンロードできます。



注：執筆時点で、最新のビルドは629でした。

注：これらのコンポーネントが機能するには、JDK6がターゲットサーバーにインストールされている必要があります。



メール送信デーモン



ESDを構成するには、esd.propertiesファイルに次の変更を加える必要があります。

  1. 起動時に、ESDはSiebelにログインして、Siebel Marketing Serverを使用するためのライセンスを確認します。 これを行うために、彼はWebサービステクノロジーを使用しています。 

    esd.handler.webservice.url=http://sbl81srvr01/eai_enu/start.swe?SWEExtSource=WebService&SWEExtCmd=Execute&WSSOAP=1 esd.handler.webservice.username=ESD esd.handler.webservice.password=ESD


  2. ESDがレターを収集できるようにするには、受信者データとレターテンプレートを含むリストが必要です。 Siebelは、このデータをファイルとしてファイルシステムにアップロードします。ESDはこのファイルシステムにアクセスする必要があります。 

     esd.mountPoint=c:/Siebel/fs


  3. ESDは、手紙が送信されるメールサーバー（SMTPサーバー）の名前を知っている必要があります。 

     esd.mailServers=sbl81srvr01




ESDの実行はサービスとして最も簡単です。 Windowには、このための特別なインストーラーinstall-win32-service.batがあります。



注： ESDを開始する前に実行する必要がある別のアクション：esd-axis2.xmlファイルで、次の形式の行をコメントアウトします。 
 <parameter name="Transfer-Encoding">chunked</parameter>.


これは、support.oracle.comで説明されているエラーが原因です。



すべてが正しく構成されている場合、ログ（esd.log）で開始した後、次を確認できます。 

 2015-08-26 17:21:06,362 [Thread-1] INFO OnPremiseAppListener ESD is licensed.




次に、メール送信タスクを提供するために、ESDの場所をSiebelに伝える必要があります。 これを行うには、次のものが必要です。

  1. [管理]-[Webサービス]-> [送信Webサービス]画面に移動します。
  2. SendMailingServiceという名前のサービスを見つけます
  3. SendMailingという名前のサービスポートには、ESDが展開されているサーバーのアドレスを指定する必要があります




これで、Siebel ‒ ESDバンドルが機能し、SiebelがESDのタスクを設定して電子メールを送信できるようになります。



バウンス処理デーモン



BHDを構成するには、bhd.propertiesファイルに次の変更を加える必要があります。

  1. Siebel Web Serviceアクセス設定。これにより、BHDは起動時にライセンスをチェックし、受信したすべてのバウンス電子メールに関する情報を送信します。 

     BhdModule.handler.webservice.url = http://sbl81srvr01/eai_enu/start.swe?SWEExtSource=WebService&SWEExtCmd=Execute&WSSOAP=1 BhdModule.handler.webservice.username = BHD BhdModule.handler.webservice.password = BHD


  2. BHDがバウンスする電子メールをリッスンして待機するポート番号： 

     SMTPD.port=8096


    標準のポート25を使用しない方が良いです。管理者権限が必要です。 ポートを設定する最も簡単な方法は、8000〜8900の範囲です。





すべてが正しく構成されている場合、ログ（bhd.log）で開始した後、次のことがわかります。 

 2015-08-26 18:20:48,823 [main] INFO OnPremiseMain BHD is licensed. …. 2015-08-26 18:20:49,104 [main] INFO SMTPDaemon started; waiting for connections




クリックスルーデーモン



CTDを構成するには、ctd.propertiesファイルに変更を加える必要があります

  1. Siebel Webサービスにアクセスするためのパラメーター。CTDが起動時にライセンスをチェックし、受信者がレターを処理するときに生成されるすべてのHTTPリクエストに関する情報をSiebelに送信します（リンクをたどり、写真をリクエストします） 

     ctd.webservice.url = http://sbl81srvr01/eai_enu/start.swe?SWEExtSource=WebService&SWEExtCmd=Execute&WSSOAP=1 ctd.webservice.username = CTD ctd.webservice.password = CTD




このユーティリティは、install-win32-service.batスクリプトを使用してWindowsサービスとしてインストールすることもできます。



すべてが正しく構成されている場合、ログ（ctd.log）で開始した後、次を確認できます。 

 2015-08-26 18:31:49,365 [Thread-1] INFO OnPremiseAppListener CTD is licensed. … 2015-08-26 18:31:49,443 [Thread-1] INFO Registry Options configured for service: class com.siebel.sem.ctd.integrated.CTDRegisterWebService 2015-08-26 18:31:49,443 [Thread-1] INFO Registry Registering [urn:xml-initialize-ctd] with class [com.siebel.sem.ctd.integrated.CTDRegisterWebService]


注：フォルダーc：\ Siebel \ EMS \ ctd \ webapps \ ctd \ enu \には、受信者がニュースレターの購読を解除することにした場合に使用されるページテンプレートがあります。 これらは、会社のコーポレートアイデンティティに従って変更できます。



Siebelセットアップ



BHDおよびCTDを使用するには、Siebelレベルで構成する必要があります。 [管理]-[マーケティング][サーバー]画面で、Eメールマーケティングサーバーエントリと、BHDおよびCTDパラメーターについて、インターネットと企業のローカルネットワーク間のゲートウェイとして機能するサーバーの名前を見つける必要があります。



IPアドレスではなく、ドメイン名を指定することが重要です。 さらに、このドメイン名はインターネットから解決する必要があります。 これらの条件が満たされない場合、外部メールサービスはあなたの手紙をスパムとして解釈します。



ネットワークゲートウェイ



バウンスされた電子メールとHTTPリクエストが送信されるルーターでは、外部ネットワークインターフェイスから内部へのパケット転送を構成する必要があります。

  1. 外部インターフェイスのポート25に到着するパケットを、BHDが展開されているサーバー、リッスンするポートに転送します。
  2. 外部インターフェイスのポート80に到達するパケットは、CTDが展開されているサーバー、リッスンするポートに転送する必要があります。


外部インターフェイスでこれらのポートを使用することが重要です。そうしないと、外部メールサービスに問題が発生します。



テストニュースレター



すでに送信された手紙は返却できません。 したがって、送信する前に、事前に定義済みのテストアドレスにメールを送信する必要があります。 これを行うには、リスト管理を使用して固定リストを作成し、使用するマーケティングキャンペーンに添付します。

すべてが正しくセットアップされていることを確認するには、次のことを行う必要があります。

  1. リスト管理でテストメーリングリストを作成する
  2. テストオファー（オファー）を作成し、それに対する電子メール処理を定義します（処理）
  3. マーケティングキャンペーンを作成し、テストリストとメールアドレスに関連付けます
  4. このマーケティングキャンペーンを開始する
