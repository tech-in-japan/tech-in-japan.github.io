Citrix Synergy 2011：クラウドを通じて

5月25日から27日にかけて、次の年次Citrix Synergy 2011会議がサンフランシスコで開催され、仮想化およびクラウドコンピューティングテクノロジーの開発に専念しました。 live.citrixsynergy.comでリアルタイムで彼女をフォローできます。 今年のイベントの焦点は、デスクトップモデルからクラウドインフラストラクチャへのグローバルな移行を可能にするソリューションにありました。



ITシステムは、特に激しい競争と企業の従業員数の増加に直面して、より複雑になる傾向があることが知られています。 この開発には重大な欠点があります。システムが複雑になるほど、メンテナンスのための資金が必要になり、エラーの可能性が高くなります。 したがって、Citrixスペシャリストのスピーチの主なアイデアは、既存のITシステムを簡素化することでした。 Citrixの社長兼CEOであるマークテンプルトンによると、これはいつでもどこでもモバイルデバイスを介して作業できるための前提条件です。これは、今日の従業員が雇用主に対して頻繁に行う要件です。 シトリックスは、システムを簡素化するための普遍的なツールとして、ITインフラストラクチャに対する根本的に新しいアプローチに基づく独自の開発を提供します。パーソナル、プライベート、パブリックの3種類のクラウド（パーソナルクラウド+プライベートクラウド+パブリッククラウド）の組み合わせです。

マークテンプルトンによるライブパフォーマンスに興味がある人は、4部構成のビデオクリップ（英語）をお勧めします。

パーソナルクラウド



BYOS（独自のコンピューターを持ち込む、「独自のコンピューターを持ち込む」）の概念は、数年前にCitrixによって導入されました。 パーソナルクラウドの開発戦略に適合し、すべての企業アプリケーション、デスクトップ、データ、連絡先、サービスにアクセスするために独自のデバイスを使用する可能性を示唆しています。 コンセプトはBYO-3に変わりました。調査によると、平均的な従業員は、ラップトップ、スマートフォン、タブレットの3つのデバイスを同時に持つことができます。 すべてのリソースが配置される場所、データが保存される場所、およびこのプラットフォームまたはそのアプリケーションが作成されるプラットフォームについて考える必要はありません。 仕事の自由は、現代のビジネス条件によって決定されるルールです。 Citrixはパーソナルクラウド開発のためにどのような革新を提供しましたか？





昨年の会議に戻って、Intelと共同開発したCitrix XenClient（2009年に最初に発表された）が最初に発表されました。 ネットワークから切断されている場合でも、ラップトップおよびデスクトップコンピューターで仮想ワークステーションを使用できます。 つまり、XenClientを使用すると、職場で個人用ラップトップを使用でき、自分の機器以外に企業用のラップトップも持ち運ぶことはできません。 このスキームは非常に簡単に機能します。ハイパーバイザーはベアメタルにインストールされ、その上には既に仮想システムの標準イメージがあります。 世界中どこでも仕事ができます。 ネットワークへの常時アクセスを必要としないアプリケーションは正常に機能し、再接続後、すべてのデータがサーバーに自動的に転送されます。 従業員が出張に行って、ラップトップで仕事をしたとしましょう。 リモート接続では、データはサーバーと同期され、IT管理者はセキュリティを監視し、ソフトウェアの動作を監視します。 （システムまたはソフトウェアに）XenClientがインストールされているラップトップで何かが変更されると、データはVHD（仮想ハードディスク）形式の仮想マシンをサポートするCitrix Synchronizerに送信されます。 同時に、トラフィックを最小限に抑えるために、ブロック追跡テクノロジーとデータ圧縮が使用されます。 この製品はすぐに人気を博しました。 わずか1年で、同社のウェブサイトには10​​万件以上のプログラムがダウンロードされました。 XenClient 2ハイパーバイザーの新バージョンは、4500万台を超えるPCとラップトップ、および拡張グラフィックスサポートグラフィックスをサポートし、Intel Coreプロセッサーの第2世代のIntel HDグラフィックスと連携します。 FireProおよびRadeonビデオアダプターを含むAMDのディスクリートグラフィックステクノロジーもサポートされています。 一般に、このソリューションにはリモートPC制御テクノロジー（vPro）は必要ありません。これにより、VT-dのないラップトップを使用できますが、いくつかの制限があります（たとえば、VT-dがない場合、3Dカードを仮想マシンに転送できません）。 データセンターからXenClientを搭載したラップトップへの仮想マシンの配信に応答する更新されたコンポーネント。 プログラムの評価版は、会社のWebサイトからダウンロードできます。



極度のセキュリティ、分離（Extreme Desktop Isolation）およびパフォーマンス（Extreme Performance）の機能を含むXenClient XTのバージョンも導入されました。 Extreme Securityには、Intel Trusted Execution Technologyに基づくトラステッドブートテクノロジーが含まれており、XenClient構成がブートごとにチェックされることに注意してください。 さらに、1つのシステムで多数の分離された仮想PCを実行する機能が追加されました。







Citrix Receiver Universal Client Moduleも更新されました。 仮想デスクトップ、Windows、Web、SaaSアプリケーション、およびオンデマンドのコンテンツを、あらゆるデバイスのあらゆるユーザーに配信します。 クライアントは、PCおよびMac（1000モデル以上）、タブレット（37モデル）およびスマートフォン（149モデル）、および10クラスのシンクライアントを含む世界中の10億以上のデバイスにインストールできます。 広く使用されているオペレーティングシステムだけでなく、iOS、Android、webOS、Google ChromeOSなどの「新製品」でもサポートされています。 つまり、Citrix Receiverの存在により、膨大な数のデバイスをパーソナルクラウドに直接接続できます。 プライベートクラウドは、NetScaler Cloud Gatewayを介して利用可能になります（以下を参照）。 会議では、ReceiverはwebOSオペレーティングシステムを搭載したHP Webタブレット、Chrome OSプラットフォームおよびその他のデバイスで動作することが示されました。 以下は、iOSを実行しているiPhoneで実行されているCitrix Receiverソリューションのビデオです。

以前の記事の1つで、Citrix HDXテクノロジーについて説明しました（http://habrahabr.ru/company/citrix/blog/107761/）。 会議では、マルチメディアおよびクラウドアプリケーション向けの改善されたHDXソリューションを示しました。 ビデオ、オーディオ、3Dグラフィックスなどのマルチメディアサービスが豊富なアプリケーションを使用する場合、特にワイドエリアネットワークまたは外部クラウド環境から配信する場合、チャネルのパフォーマンス、精度、帯域幅を最適化します。 言い換えると、特に低速のインターネット接続を使用して自分のPCで作業している場合、ビデオを視聴したり、ビデオ会議システムに参加したりすることがより快適になります。 新しいバージョンはHDX RealTimeテクノロジーを最適化し、TCPおよびUDPプロトコルのトラフィックフロー間のリアルタイムQoS優先度を完全にサポートしました。これにより、VoIPおよびユニファイドコミュニケーションプログラムの比類なき双方向サウンドが実現します。 また、改良されたFlashリダイレクト、DirectXおよびGDI +グラフィックスへのリモートアクセス、および組み込みのHDX 3D ProテクノロジーとMicrosoft RemoteFX。 HDX WAN最適化は、最大300ミリ秒の遅延処理により、極端な距離でのフラッシュ配信を最適化します。





HD Facesを使用したGoToMeeting Webサービスを使用すると、高解像度のビデオ会議を整理できます。 このソリューションは、2004年に初めて作業を開始しました。 新しいCitrix Multistream HDテクノロジーは、1回のセッションで最大6人の参加者（通常モードで最大15人）を集め、最大1920 x 960ピクセルの最大解像度のビデオを視聴することに加えて、共通のデスクトップを見ることができます。 ベータテストは5月25日から無料で利用できます。

GoToManageの新しいiPadアプリを使用すると、問題を安全にトラブルシューティングして解決できるだけでなく、ユーザーのコンピューターシステムにリアルタイムでテクニカルサポートを提供できます。 会議では、アプリケーションが無料になると約束されました。 以下のビデオ（英語）：

プライベートクラウド



Citrixのプライベートクラウドコンセプトの急速な発展の見通しは、Kavizaとその汎用性のある中小企業向けソリューションVDI-in-a-Boxの買収により現実的になりました。 このテクノロジーの名前の1つは、その「自給自足」と使いやすさを表しています。このテクノロジーは、接続ブローカー、管理サーバー、ロードバランサーを必要としません。 すべてが1つの仮想モジュールに収まります。 VDI-in-a-Boxは、Citrix XenDesktopのビジネスデスクトップ仮想化ソリューションのラインを補完します。







デスクトップ仮想化ツールXenDesktop 5の更新バージョンには、IntelliCache機能が含まれています。 ストレージコストを最大50％削減し、デスクトップ仮想化の費用対効果をさらに高めることができます。

デスクトップ仮想化ソリューションの展開を簡素化するために、新しい無料のSuccess Acceleratorオンラインサービスも導入されました。 ユーザーは、1日24時間、週7日間、あらゆるプロジェクトに関するアドバイスを得ることができます。実際の実践からのヒントと具体例があります。



パブリッククラウド



データセンターの内部を改善し、パブリッククラウドを開発することにより、Citrixは2つの新しいソリューションを提供します。 Cloud Gatewayは、ユーザーがどこにいてもSaaSおよびWebアプリケーションを最も安全に配信します。 Cloud Gatewayには、認証済みユーザー向けのシングルサインオン機能が備わっています。 言い換えれば、この製品により、パーソナライズされた顧客クラウドが企業のデータセンターに接続できるようになります。 2番目のソリューションであるNetscaler Cloud Bridgeは、プロバイダーとの通信チャネルを暗号化し、アプリケーションをプライベートクラウドに転送し、プライベートクラウドとパブリッククラウド間の通信チャネルをサポートします。 NetScaler CloudBridgeを使用すると、アプリケーションをパブリッククラウドに持ち込んで、社内にデータを残すことができます。 物理および仮想の両方のソリューションは、x64ハードウェアプラットフォームで動作するように設計されています。

そして、すべてが「支障なく」機能するように、CitrixはOpenStackに基づいて開発され、クラウドインフラストラクチャの作成に使用されるOlympusプロジェクトを初めて発表しました。 これにより、スケーラブルで効率的なオープンソースのIaaSクラスクラウドを展開できます。 Olympusプロジェクトでは、Microsoft Hyper-VおよびVMware vSphereを使用して、パブリッククラウドとパーソナルクラウドの両方を作成できます。 世界最大のクラウドを展開するのと同じアーキテクチャ、アプローチ、テクノロジーを使用しています。

サンフランシスコでのイベントでCitrix社が提示したすべてのソリューションの主なアイデアは何ですか？ 比較的言えば、あなたはよりシンプルである必要があり、人々はあなたのために手を差し伸べるでしょう。 ほんの数個のソフトウェアおよびハードウェアソリューション、および3クラウド戦略により、データをデバイスからデバイスに移動させてセキュリティを危険にさらす従業員だけでなく、ITコストをひたすら評価する企業にとっても楽になります。 パーソナルクラウド、プライベートクラウド、パブリッククラウドの共生により、情報の輸送、保管、セキュリティのプロセスが簡素化されます。 そして、クラウド要件を満たすハイパーバイザーは、このビジネスの最初のヘルパーです。
