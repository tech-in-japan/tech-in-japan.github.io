リチャード・ストールマン・マニフェスト：コンピューティングの方法





FSF（Free Software Foundation）が無料の初期化プログラム（ libreboot ）と無料のオペレーティングシステム（ Trisquel GNU / Linux ）をインストールしたThinkpad X60コンピューターを使用しています。 これは、無料の初期化プログラムと無料のオペレーティングシステムを搭載した最初のコンピューターモデルです。 したがって、これはFSFが承認した最初のコンピューター製品です（ただし、Lenovoは販売していません）。



それ以前は、 Lemote Yeeloongを数年間使用していました。 当時は、無料のインストールプログラムと購入可能な無料のオペレーティングシステムを備えた唯一のラップトップでした。 しかし、無料のオペレーティングシステムでは販売されませんでした。



その前に、私はOLPC （子供ごとに1つのラップトップ）を数週間使用しました。 OLPCプロジェクトがWindowsをサポートすることを決定したため、私は停止しましたが、これは望ましくありません。 OLPCはWiFiに特別なファームウェアを使用するため、内部WiFiデバイスを使用できませんでした。 関係ありません、外部を使用しました。



結果は気になります。 OLPCでWindowsを使用している何百万人もの子どもたちを見ると予想していました。 代わりに、Intel ClassmateでWindowsを使用している何百万人もの子供たちがいます。



以前は無料のGNU / Linuxシステムでデバイスを完全に使用していましたが、非フリーのBIOSがありました。 8年間、私はこの問題を回避する方法を見つけようとしてきました。



この出版物は、オンラインの不動産データベースを開発する エジソンによってサポートされているだけでなく、産業用スケールおよびキャリブレーション機器用のソフトウェアを設計および開発しています 。







お気に入りのGNU / Linuxディストリビューションはありません。 私はすべての人、つまり100％フリーソフトウェアを実行する人に倫理的な配布をお勧めします。



私は倫理的な分配に関する好みを持たないことに決めました。 しかし、私は他の基準でそれらを判断することはできません。それらをすべて試しても、多くの作業が必要になります。



便宜上、主にテキストコンソールを使用します。 私の仕事のほとんどはテキスト編集であり、テキストコンソールでこれを行う方がはるかに効率的です。 テキストコンソールで、誤ってタッチした場合、タッチパッドで問題が発生することはありません。



グラフィカルインターフェイスが必要な場合はX11を使用します。 グラフィカル環境やウィンドウマネージャーに関する設定がありません。 グラフィカル環境の使用に対する私の関心は小さいため、それらを比較する時間を無駄にしたくありません。



これは倫理的な問題ではなく、個人的な好みです。 倫理的なレベルでは、フリーソフトウェアは無料のグラフィカルユーザーインターフェースを提供するべきだと思うので、GNUは3つのプロジェクトを開発に着手しました。 3番目のGNOMEは成功したため、4番目のプロジェクトは必要ありませんでした。



ほとんどはEmacsで編集していました。 Mx rmailとCx mを使用してEmacsメールを読んで送信しました。 他のメールクライアントプログラムの経験はありませんでした。 原則として、他の無料の電子メールクライアントについて知りたいと思いますが、これは第一に重要な問題ではなく、このための時間はありません。







Emacsを使用してこのサイトのページを編集しましたが、ボランティアが政治的および緊急のメモのセットアップを支援しました。 私はウェブサイトを維持する他の方法の経験がありませんでした。 原則として、他の方法についても喜んで学びますが、これは第一に重要なことではなく、他にもやることがあります。



このサイトは非常に保守が簡単です。 HTMLで手動でこれを行っているかのようにページを編集します。 HTMLの基本しか知らない。 詳細を知っている人はページの上部と下部を構成し、メインページのより複雑なレイアウトを作成します。 ボランティアは、メールで私から手紙を受け取った後、毎日政治メモを追加するのを手伝ってくれます。 cronの仕事は、2か月ごとに政治メモのあるページを「めくる」ことです。 フォトギャラリーは、 perlスクリプトを使用して生成されます 。 サイトの検索機能は、 このコードを使用して実行されます 。



これを手伝いたいですか？ gnu.orgのrmsに書き込みます。



これにより、デザインコンセプトが説明され、なぜ「ユーザーエクスペリエンス」のコンセプトがうんざりするのかがわかります。 したがって、私はstallman.orgがシンプルであることを望みます。「ユーザーエクスペリエンス」ではなく、特定の情報、ビュー、オプションを提供する場所でした。



Unix（ GNUシステム ）の無料の代替品を開発することに決めた後、私はUnixを使用しませんでした（すぐに）。 私はこの設計に固執することに決めました。なぜなら、それは移植性があり、非常に簡単だからです。 私はUnixのファンではありません。 私も彼女を批判しました。 しかし、彼女はモデルとして全体的に悪くはありません。



90年代半ばには、私の手がひどく痛いので、1日のほとんどを1本の指で書くことができました。 FSFは、その日の一部を私のために印刷するために人々を雇いました。 数年後、これはキーボードのハードキーによるものであることがわかりました。 剛性の低いキーボードに切り替えたところ、主な問題はなくなりました。

私はしばしば奇妙な家でインターネットに接続します。 もちろん、人は私が誰であるかを知っていますが、それは私を悩ませません。 しかし、私は彼らが見つけることができるデータベースに自分のアイデンティティを持つことに反対しています。 これを防ぐには、すべての場所でMACアドレスを変更します。



私はインターネットの使い方に非常に注意しています。

最も強力なプログラミング言語はLispです。 Lisp（またはその変形であるScheme）を知らない場合、プログラミング言語の力と優雅さについて話すとき、何を意味するのかわかりません。 いつか、あなたはLispを学び、他のプログラミング言語に欠けているものを理解するでしょう。



特殊なデータ型の定義に焦点を合わせている今日のほとんどの言語とは異なり、Lispは一般的ないくつかのデータ型を提供します。 特定のタイプを定義する代わりに、これらのタイプから構造を構築します。 したがって、このタイプのリストとそのタイプのリストを定義する方法を提案する代わりに、Lispには任意のデータタイプを含むことができる1つのリストタイプがあります。



他のプログラミング言語でこのタイプのリスト検索関数を定義できる場合や、このタイプのリストに使用できる一般的なリスト検索関数を定義する方法がある場合、Lispを使用すると、任意のリストを検索する関数を簡単に記述できます。そのような機能。



さらに、Lispの関数と式は、操作が非常に簡単な方法でデータとして表されます。



Lispシステムを起動すると、read-execute-printサイクルに入ります。 多くのプログラミング言語には、読むこと、実行すること、印刷することのようなものはありません。 大きな欠点は何ですか！



Lispは他の言語よりも理解しにくいです。 したがって、プログラミングを一度も勉強したことがない場合は、Lispから始めてください。 Emacsで編集する方法を学べば、Emacsエディター用の編集コマンドを書くことでLispを学ぶことができます。 Emacs Introduction to Programmingを使用して、Lispを学ぶことができます。 無料で 、FSFから印刷版を注文できます。











AbelsonとSassmanの「コンピュータープログラムの構造と解釈」からScheme（および多くの詳細なプログラミングのアイデア）を学ぶことができます 。 この本は現在無料で入手できますが、ハードコピーはそう考えていません。



アマゾンで本（または何か）を買わないでください！



私の好きなプログラミング言語はLispとCです。しかし、1992年以来、私は主にフリーソフトウェアアクティビティに取り組んできました。 その結果、Perl、Python、PHP、Rubyなどの新しい言語を学ぶ時間も機会もありませんでした。



Javaについての本を読んだところ、Cの非常にエレガントな開発であることがわかりました。 しかし、私はそれを使用したことはありません。 一度Javaでコードを書いたことがありますが、コード自体はCとLispでした（当時はJavaでした）。



ちなみに、C ++は見苦しい。

私は、プロプライエタリなソフトウェアのインストールを固く拒否したり、自分のコンピューターまたは作業する必要のあるコンピューターにインストールされているという事実を容認したりします。



利用可能な唯一のデバイスにプロプライエタリなソフトウェアが含まれている訪問や旅行にいる場合、それらの使用を拒否しません。 インターネット上の情報の検索などの簡単なタスクに使用します。 この制限された使用は、ライセンスされたソフトウェアへの同意を与えず、コンピューター上またはコピーの所有者の存在の犯人にならないので、拒否する理由はありません。 もちろん、地元の人たちにデバイスをフリーソフトウェアに転送する理由を説明しますが、これを説得する最良の方法ではないため、圧力をかけません。



さらに、私が使用しているキオスク、公衆電話、またはATMにどんなソフトウェアがあるかを心配する必要はありません。 私は彼らの所有者が彼ら自身の利益のためにそれらをフリーソフトウェアに移すことを望んでいますが、それまでそれらを使うことを拒否する必要はありません。 これらのデバイスとその所有者は私の個人データで何かできることを知っていますが、これはフリーソフトウェアを使用している場合でも発生する別の問題です。



主なことは、自分に関する情報を提供するアクティビティの種類を最小限にすることです。



これは、デバイスのまれな使用に関する私のポリシーです。 毎日1時間使用した場合、それはもうまれな使用ではなくなります-通常の使用です。 それから私は自分のソフトウェアについて心配し始め、それを追跡する義務を感じます。

さらに、コンピューターをセットアップするように求められた場合、または持ち込まれた場合、ソフトウェアのダウンロードに関して倫理的に責任があります。 私は自分のコンピューターであるかのように、フリーソフトウェアを主張します。



電子レンジやその他の家電製品に関して、ソフトウェアの更新がデバイスの使用の通常の部分ではない場合、これはもはやコンピューターではありません。 この場合、ユーザーは、デバイスを含むプロセッサ、デバイスのソフトウェア、または配置方法を考慮する必要はないと思います。 ただし、「ファームウェアアップグレード」オプションがある場合、これは、さまざまなソフトウェアのインストールがその使用の通常の一部であり、これがコンピューターであることを意味します。



Skype（または独自の相互運用不可能な通信プログラム）は、そのネットワーク効果のために特別なケースです。 Skypeを使用してSkypeを使用しているユーザーとチャットすることは、専用ソフトウェアの使用を奨励しています（常にこれを行うと、他者に専用ソフトウェアの使用を強制します）。 したがって、いかなる状況でもSkypeを使用しません（ 追加情報を参照）。



NetflixやSpotifyなどのストリーミングマルチメディアdisサービスには、ユーザーが自分のコンピューターを介してストリームデータのコピーを保存できないように設計されたデジタルメカニズム制限（ DRM ）を課す独自のクライアントプログラムが必要です。 破壊できないDRMは使用しないでください。DRMを解読できない場合は、これらのdisサービスを使用しないでください。



両方のストリーミングクライアントプログラムのもう1つの不正は、著作権法自体よりもユーザーを厳しく制限する不公平な契約（ライセンス契約）を課すことです。 私はこの状況に同意しません。あなたもそれらを拒否することを望みます。



これらのストリーミングサービスは、人々を社交的にするように設計された悪意のある技術です（コピーを所有していない限り、コピーを共有することはできません）。 それらと戦うことは最優先事項の一つです。
