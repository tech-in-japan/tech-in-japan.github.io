GNU / Linuxツールを使用したMS Windows XP / Server 2003 OSの簡単なクローン作成

注釈



ジャーナル化された作業結果を公開して、インストールされたオペレーティングシステム（以降、OS）Windows XP SP3の普遍的で実行可能なイメージを確保します。



ユーザーにとってサポート、アクティベーション、およびその他の重要でない瞬間に関する暴言にもかかわらず、この使い慣れたバージョンのウィンドウを間違いなく使用したい顧客のコンピューターにシステムをインストールするプロセスを高速化する必要がありました。



なぜこれが必要なのですか？






この資料とOSクローニングのトピックに関する一般的な記事の違いは何ですか？ 人生と私自身の前に置かれた制限。 それらをリストします。



1）OSがインストールされ、任意のサイズのパーティションで動作する必要があります。

2）OSは、ブートをサポートする任意のタイプのメディア（MBR *を装備）にインストールされ、適切にブートする必要があります。

3）OSは、ハードウェア依存レベル（HAL **）のさまざまなバージョンで動作する必要があります。

4）OSイメージは、メディアの転送、改造、バルクヘッドを加速するために、メディア上の最小スペースを占有する必要があります。

5）OSイメージには、最適な方法でインストールおよび構成された必要なソフトウェアのセット（「ターンキーシステム」オプション）を含める必要があります。

6）イメージの準備とその開発に関するすべての操作は、標準のGNU / Linux ***ツールを使用して実行する必要があります。 意味：OSクローン作成のために既存のソフトウェアの動作原理を分解します。

7）OSイメージのキャリアは、ネットワーク上のサーバー、USBドライブ（ソリッドステートまたはハードドライブ）、光学式またはハード磁気ディスクにすることができます。

8）OSイメージキャリアには、コンピューターソフトウェアの診断および修復ツールが装備されている必要があります。

9）結果の信頼性を低下させることなく、再現性を高めるために、OSクローニングのプロセスを可能な限りアクセス可能にすることが望ましい。

10）ddコマンドは確かに優れていますが、空のスペース、ゼロ、および現在のアクションの明確な結論の欠如に煩わされることに消極的です。 さらに、クローンをインストールするパーティションは任意である必要があります（段落1を参照）。



考慮外：



1）現在サポートされていないOSのインストールに関する法的問題。

2）現在公式に非アクティブなOSのアクティベーション。

3）実行されたアクションの実行可能性。 赤目のためではなく、私を送ってくれたユーザーの意志によってのみです。 ブロックマシン上で新鮮なフリーソフトウェアを使用することをコーシャに納得させようとした人たち。

5）ネットワーククローニング方法の詳細：今後検討しますが、現在はそのような必要性はありません。

在庫準備


繰り返しますが、すべてをUbuntu 12.04 OSの制御下で準備しました。



1）MS Windows XPのねじれたインストールイメージがないクリーンなイメージを取得します。これらは「MSDNイメージ」とも呼ばれます。 最新のサービスパックを実装することが望ましい。 入手したら、適切な光ディスクにロールします。 Braseroプログラムでこれを行いました。

2）任意のメディア（いわゆるLiveシステム）から起動する機能を備えた適切なGNU / Linuxディストリビューションをダウンロードします。 要件：さまざまなハードウェアの幅広いサポート、NTFS、FAT16、FAT32ファイルシステムのサポート。 最適な互換性を得るには、32ビットOSを使用してください。 繰り返し：ディストリビューションubuntu-12.04.5-dvd-i386を使用し、すべての点で満足しています。 トレント 、直接ダウンロードリンク 。

3）usb-creator-gtkプログラムを使用して、Ubuntuから抽出したLinuxをロールアップできます。すべてが直感的です。 NTFSパーティションが1つあり、容量が160 GBの別のクリーンなUSB-HDDにロールバックし、保存されたファイルとフォルダーに残りのすべてのスペースを割り当てました。 したがって、すぐにブートディスクと、さまざまな手段、クローン、ユーザーデータ、およびプログラムのライブラリを受け取りました。

3.1）次のコマンドを使用して、通常のフォーマット、USB-HDDのフォーマット、イメージの書き込みを誰も気にしませんでした。 



dd if=/--- of=/dev/sd{ } [ bs={      } ]
。

コマンドが潜行的であるため、ディスクインジケータの点滅が停止するまで待つことをお勧めします。

4）解凍されたアーカイブを新しいGrub4Dosアセンブリ（ ここからダウンロード ）とともに、生命を与えるLinuxのキャリアに追加します 。 ルートの近くに配置することをお勧めします。 なぜrude4doなのか？ はい。ネイティブのWindowsローダーの楽しさを学ばせたいとは思わないのですが、設定するのは無礼で快適です。 このアーカイブには、ブートメニュー管理ファイルmenu.lstが含まれています。ブートローダーを構成する場合は、任意のテキストエディターで実行してください。 詳細はこちら 。

5）同じメディアに、スペースが許せば、ユーザーが必要とするインストールソフトウェアパッケージを配置します。 もちろん、任意のディレクトリ内。

6）Windowsクローンとさまざまなマザーボード（または、電源管理方法）との最高の互換性を確保するには、ロードされたXPコアのリストを補足する必要があります。 これを行うには、Windows XPのインストールイメージから、i386 / Driver.cabアーカイブを解凍します。 次のdllを探します：hal、halaacpi、halacpi、halapic、halmacpi、halmps、halsp。 これらのファイルをLinuxブータブルメディアのディレクトリにコピーします。すぐに必要になります。

7）XPをターゲットコンピューターにロードする際の問題を回避するには、適切なNMZDコントローラーから最新のドライバーをダウンロードして解凍します。 コントローラーがマザーボードに組み込まれている場合は、UMのドライバーをダウンロードします。 いずれの場合でも、ターゲットコンピューターごとにドライバーを再インストールする必要があります。 同じヒーリングLinuxメディアにドライバーを保存します。

8）管理者！ 注意深く、正確に。 この多目的メディアにダンプしたものをすべてチェックし、ファイルパスを確認します。 知っておいてください：Linuxシェルでそれらを探す必要があるかもしれません。 ファイルをばらまかないで、ディレクトリに適切に配布してください。 そして最後に、Windowsのインストールイメージを同じメディアにコピーします。トラブルのストックは修復されません。



クローン、コマンド66を実行しましょう！






必要なものを準備しました。 参照コンピューターを手に、Windowsのディスクを手に取ります。 タスク：参照OS、extをインストールして構成します。 プログラム、クローンシステムの準備を完了します。



1）参照軸を参照コンプに設定します。 必要に応じて必要最小限のプログラムを追加して、それらを圧迫しないようにします。 Windowsとプログラムをアクティブ化することは今では価値がありません。ターゲットコンピューター上のエンドユーザーでこれを行う必要があります。

2）このOSで作業しています。 目的：クローン作成後に、ターゲットマザーボードの電源管理構成の選択を提供する。 ブータブルメディアをLinuxに接続します。NTFSでフォーマットしたと思いますか？ それから、％systemroot％\ system32 \ディレクトリにファイル{halaacpi、halacpi、halapic、halmacpi、halmps、halsp} .dllをコピーします。これは、メディア上の人里離れたフォルダーに事前に保存しました。 これらは、特定のマザーボードデバイスでOSをロードするためのマイクロカーネルです。 これらのカーネルを使用するために、Windowsブートローダー％systemdrive％\ boot.iniの構成ファイルを変更します。 ファイルをバックアップすることを忘れないでください！ 次のようなものが得られるはずです。 



 [boot loader] timeout=10 default=multi(0)disk(0)rdisk(0)partition(1)\WINDOWS [operating systems] multi(0)disk(0)rdisk(0)partition(1)\WINDOWS="Microsoft Windows XP Professional" /fastdetect multi(0)disk(0)rdisk(0)partition(1)\WINDOWS="Standart PC" /fastdetect /hal=halstd.dll multi(0)disk(0)rdisk(0)partition(1)\WINDOWS="ACPI PC" /fastdetect /hal=halacpi.dll multi(0)disk(0)rdisk(0)partition(1)\WINDOWS="ACPI Uniprocessor PC " /fastdetect /hal=halmacpi.dll C:\grldr=" GRUB4DOS"


わかりますか？ これは、XPインストーラーの隠された選択と同じもので、ディスクからインストールするときにF5を押すと使用できます。 最後の行は将来的に役立つでしょう。

3）ターゲットコンピューターの最新のハードドライブコントローラードライバーをダウンロードした場合は、参照XPに強制的にインストールします。 フィット！ ターゲットコンピューターのBIOSを使用する必要はなく、互換モード（IDE）を有効にします...

4）システム全体を確認します。 すべてが正しくセットアップされているかどうかを確認します。 再起動せずに、さらにアクションを実行する必要があります。

5）クローン作成のためにシステムを完全に準備します。 これを行うには、Linuxブータブルメディアに事前に保存したdeploy.cabアーカイブを探します。 アーカイバでそれを開き、SYSPREPフォルダーを％systemdrive％（ドライブC：の意見）のルートに解凍します。 これは、クローン作成のためにOSを準備するための自然なMicrosoftツールです。 クローン化されたOSの同じSID、同じアクティベーションデータ、その他すべての小さな問題を回避するために使用します。 これは、次のように詳細に行われます。



5.1）setupmgr.exeを実行します。 [新しい応答ファイルの作成、Sysprepインストール]を選択します。 Windowsのバージョンを慎重に選択してください。 はい、インストールを完全に自動化します。 必要に応じてフィールドに入力し、「次へ」をクリックします。 プログラムは、作業の結果をファイルC：\ SYSPREP \ SYSPREP.INFに保存します。

5.2）実行： 



 c:\sysprep\sysprep.exe -pnp -mini


シャットダウンモードを終了するように変更し、「再シール」をクリックして、プログラムの動作が完了するまで待ちます。

注意します。 次回のシステム起動時にインストールが開始されるため、再読み込みを許可しない方が良いでしょう。

6）ほとんどの場合、参照OSは作業を完了しています。 今こそ彼女のイメージを撮る時です。 これを行うには、生命を与えるメディアとLinuxを参照コンピューターに接続し、そこから起動します（XPの起動を許可しないでください！）。 もちろん、システムをインストールするのではなく、単にロードすることに興味があります。 Ubuntuの場合、グラフィカルインターフェイスとさまざまな便利なアプリケーションが用意されています。

7）今日、私たちは断固として迅速に行動します。 端末エミュレーター（ctrl + alt + t）を開き、一連のコマンドを実行します。 



 sudo su mkdir /mnt/sd{1,2}


-ルートの下に移動し、2つのメディアのマウントポイントを作成します。 ここでのタスク：参照OSがインストールされているドライブ、およびLinuxとファイルを備えた外部ドライブをマウントすること。 パーティションとディスクの必要な値をコマンドに代入することにより、これを実現します。 



 mount /dev/sd<. _1- > /mnt/sd1 mount /dev/sd<. _ > /mnt/sd2


マウントしたかどうかを確認します。 



 ls /mnt/sd1 ls /mnt/sd2


イメージを転送するためのディレクトリが用意されています。

8）tarコマンドを使用してWindowsクローンを起動します。 



 cd /mnt/sd1 tar -czvf /mnt/sd2/xp.tar.gz ./*


9）パッケージングプロセスを観察します。 Linuxメディアのルートにxp.tar.gzアーカイブを取得しました。サイズは約1.2 GBです。 これは、参照OSの既製のクローンです！

持続可能な結果が得られるまで、これらの手順を約15回実行する必要がありました。



今のタスク：トライアルターゲットコンピューターでイメージをロールするには、ブートローダーをインストールし、すべてが機能することを確認します。



1）命を与えるLinuxメディアをお持ちください。 その上にクローンがあり、それを置く手段があります。 ターゲットコンピュータに接続し、Linuxをロードします。 再度、ターミナルエミュレータを開いて作業を開始します。

2）ターゲットコンピュータのハードドライブにマークを付けましょう。 昔ながらのfdiskプログラムを使用しています。 注意！ 宛先ドライブに何か貴重なものがある場合は、信頼できるバックアップメディアにコピーしてください。 その後、コマンドを実行します。 



 sudo su mkdir /mnt/sd{1,2} mount /dev/sd<  ,  > /mnt/sd1 ls /mnt/sd1 mount /dev/sd<  ,   > /mnt/sd2 ls /mnt/sd2


そこで、ドライブを正しく識別したかどうかを確認しました。 すべてが正しく、すべてが指示どおりにマウントされているとします。 その後、さらに先へ進みます（ハリネズミ＃、もちろん入力しないでください、これはコメントです）： 



 umount /dev/sd<  ,   > fdisk /dev/sd<  > #   o  #  n  #  : ,  1, ,  ( ) #    NTFS t  1  //  7  //=NTFS #   ( -) a  1  #        ( !) w


Fdiskは正常に完了するはずです。 それ以外の場合は、文句を言って、Webで解決策を探してください。 多くの場合、管理者はパーティションテーブルを変更する前にメディアをアンマウントするのを忘れます。これにより、メディアが上書きされなくなります。 Umountが役立ちます。

3）すぐに受信したパーティションをNTFSでフォーマットし、マウントします： 



 mkfs.ntfs -f /dev/sd<  ,  > mount /dev/sd<  ,  > /mnt/sd2 ls /mnt/sd2


最後のコマンドの出力は空でなければなりません。

4）参照OSのクローンをこのセクションに解凍します。 



 cd /mnt/sd1 tar -xzvf xp.tar.gz -C /mnt/sd2


私たちは座って、コーヒーを一気に速く飲みます！

5）最後の主要なタッチ：ブートローダーのインストール。 新鮮なgrub4dosアセンブリの解凍されたアーカイブをLinuxメディアに入れたことを思い出させてください。 



 cd /mnt/sd1/    ls #    ,   : ./bootlace.com /dev/sd<  > #      grldr   menu.lst       : cp grldr menu.lst /mnt/sd2


6）ターゲットコンピューターの電源をオフにします。これを行うことができます。 



 shutdown -h now
