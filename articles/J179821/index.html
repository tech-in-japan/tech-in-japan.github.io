CentOS 6.4 + ReiserFS

Karachunは、Gentooと、emerge -uDpv worldの最後の起動の先史時代の日付を持つ古いサーバーで、古いMaxtorにやって来ました。 Gentooはどういうわけか、再度インストール-コンパイルしたくありませんでした。 やがて燃え尽きました。 安定性、信頼性、更新プログラムのインストールの速度（サーバーが最初の鮮度とはほど遠い）、依存関係の問題がないことを望んでいました。 一般に、怠lazは最適化エンジンです。



最初は、FreeBSDを使い続けて、スチームバスを浴びたくなかった。 しかし、サーバーは単純ではありませんが、開発者にとってより正確には、さまざまなLinuxバイナリがサーバー上で実行され（ソースなし）、アーキテクチャは32ビットです。 したがって、制限-Linuxディストリビューション、32ビット、古いハードウェアの幅広いサポート、古いライブラリの利用可能性、新しいテクノロジー、グラフィックスの消失、そのままでReiserF それとは別に、/ homeパーティションとこのReiserFSのネジがあります（ところで、3.6）。 サービスの-Samba、DNS、DHCP、TFTP。



どうでしたか


新しい流行は、サーバーOSの役割に関するLinuxディストリビューションの幅広い選択を示唆しており、それらの最初のものはUbuntu Server 12.04によってテストされました。 インストール-優れた、迅速な更新プログラムのインストール。 / etc / fstabに必要な行を書き込んで再起動した後、/ homeのあるパーティションが離陸しませんでした...ディスクにそのようなパーティションがないことを示す画面上のログを実行しました（単独で存在します）。 2番目のディスク（ReiserFSを含む1つのパーティション）-であり、正常にマウントされました。 そして、どこ/自宅で-いいえ。 はい、ドライブは自然にMHDDおよびfsck.reiserfsによってテストされます。 また、fdisk -lはパーティションがないことを示していますが、fdiskを介してディスクを開くと、パーティションがあり、パーティションテーブルを記録します（wコマンド）。出力はすべてで、パーティションは表示されマウントされています。 再起動します-パーティションはありません、fdisk、パーティションがあります...はい、ショックです。 Ubuntuサーバーには適していません。



したがって、話のトピックに戻ると、CentOSは、サーバーの管理下でのサーバー管理の経験と、制御下での妨害の解決の経験から選ばれました。

新しいCentOS 6.4イメージをインストールした後（詳細は省略）、デフォルトではReiserFSが見知らぬ人であることが判明しました。 良いGoogleは、このトピックに関する英語のリソースへのリンクを慎重に発行しました。これには、このトピックに関する他の英語のリソースへのリンクが含まれています



目標を達成するためのアクションの若い戦闘機リストの短期コース ：



別のカーネル-centOS-plusカーネルとreiserfs-utilsユーティリティのセットをインストールする必要があります。

しかし、最初に、コア。 CentOS-Plusリポジトリが含まれています。 これについての情報はまとめて、要するに：

/etc/yum.repos.d/CentOS-Base.repoを編集し、ブロックします： 



[centosplus] ...... enabled=1 ........ includepkgs=kernel* jfsutils reiserfs-utils




カーネルの更新： 



 yum install kernel




次のように書かなければなりません： 



 ….. kernel-2.6.32-358.6.1.el6.centos.plus.i686 …...




ええ、正しいコア。 grub.conf（menu.lst）で、起動時にデフォルトとして登録されているかどうかを確認できます。



Reiserfs-utilsユーティリティ-コマンドによる標準リポジトリ： 



 yum search reiser




欠落： 

 …. Error: Nothing to do #




判明したように、必要なreiserfs-utilsユーティリティはELRepoリポジトリにあります。 接続します。

公開鍵をインポートします。 



 rpm --import http://elrepo.org/RPM-GPG-KEY-elrepo.org




そしてリポジトリ自体： 



 rpm -Uvh http://elrepo.org/elrepo-release-6-5.el6.elrepo.noarch.rpm




今： 



 yum search reiser




必要なものを生成します： 



 …. ===== N/S Matched: reiser ===== kmod-reiserfs.i686 : reiserfs kernel module(s) reiserfs-utils.i686 : Tools for creating, repairing and debugging ReiserFS filesystems. #




私たちは置きます： 



 yum install reiserfs-utils




私は/ etc / fstabを自分のニーズに合わせてルールし、再起動します：
