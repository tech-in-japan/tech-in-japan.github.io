.about [写真のカタログ化と処理のプロセスの編成]

最もおかしくて最も悲惨なことは、多くの画像閲覧者が自分の写真をどのように、どこに置くかを自分で決めることです。 また、それらをソートする方法。 実際、多くの人にとってこれは万能薬ですが、私にとってはそうではありません。 この事実は、Windows用のFastStone Image Viewerの類似物の形で、簡単で迅速な写真ビューアーを見つけるための障害となっています。 私自身は、写真のコレクションをカタログ化する方法を決定しました。写真の処理の最初から最後までの全プロセスを快適に行えるようにしました。 これに先立ち、私はあらゆる種類の実験の長く骨の折れる道を歩きました。

それでは、プロセス全体をどのように整理したかについてお話します。 一部の人にとっては面倒すぎるかもしれませんが、プロセス全体を最適化および自動化しようとしたため、これは一見しただけです。

Mac OS X 10.6.4に移行した後、それを把握し、いくつかのプログラムの代替品を見つけて、このオペレーティングシステムでプロセスの自動化を構成するのに時間がかかりました。 したがって、ここではMac OS X 10.6.4について説明しますが、プロセス全体は実質的にWindowsのプロセスと変わりません。

そのため、RAWでのみ撮影し、ファイルを転送するときはカードリーダーのみを使用します。 すべての写真が保存されているディスクに写真フォルダーがあります。 Photosフォルダーの構造は2つのキャンプに分かれています。これらはrawフォルダーとjpgフォルダーで、対応するファイルがあります。

1.カメラから新しい写真をインポートするために、Canon EOS Utilityユーティリティを採用しました。このユーティリティは、カードのファイルを/ Photos / rawフォルダーに安全に追加し、日付でファイルを並べ替え、2010_07_31の形式のディレクトリを作成します。

2.すべて問題ありません。AdobeBridgeを開き、フォルダーの名前を変更して、プロットコンポーネントを追加します。 つまり 撮影の説明を追加して、何が、誰が、いつ撮影されたかが多かれ少なかれ明確になるようにします。 これで、ディレクトリのビューは「2010_07_31 City」になります。 以前は、ディレクトリ名でアンダースコアの代わりにドットを使用して年、月、日を区切っていましたが、Adobeとrsync同期プログラムでの処理が正しくないため、ドットを破棄する必要がありました。

3.鮮明で鮮明な写真の選択が始まります。 写真とフルサイズのプレビューの生成が100％増加すると表示されます。 写真に評価をつけました。 1〜5つ星。 少なくともある程度の価値があるすべての写真は、私の評価でマークされています。 ディスク領域を占有しないように、未評価のままの写真は強調表示され、容赦なく削除されます。

4.カードを選択し、すべてをAdobe Camera Rawに送信します。そこで、設定、切り抜き、その他のパラメーターを調整します。

5.すべてのカードが設定を受け取ったら、それらをすべて選択し、RAWファイルがある同じディレクトリにjpg形式で保存します。

6.最も気に入った写真は、Adobe Bridgeでいくつかの色でハイライトし、Adobe Photoshopでそれぞれ個別に処理します。

すべてのファイルは変換および処理されますが、jpgイメージがrawおよびxmpファイルと同じフォルダーにある場合、混乱することが判明し、作業結果を確認して実際に写真を楽しむのは不便です。 そのため、写真アーカイブを2つの部分に分割することにしました。 1つ目は原材料（別名生）で、2つ目はすでに処理された材料であり、作業の結果として実証できます（jpg）。 次のステップでは、拡張子がjpgとjpegのすべてのファイルをrawディレクトリからjpgディレクトリに転送しますが、これらのディレクトリの内部構造は保持されます。

7. Windowsでは、nnBackupを使用してディレクトリ間でファイルを同期および転送し、Mac OS Xでは、このようなプログラムの類似物を探す必要がありました。 検索は短命であり、単純なユーティリティrsyncが状況を保存しました。

次の2つのコマンドがあります。最初のコマンドはjpgファイルとjpegファイルを転送してソースの場所から削除し、2番目のコマンドはjpgフォルダーを別のメディアにバックアップします。

rsync -vaHx --numeric-ids --recursive --remove-source-files --progress --exclude-from = / Photos / jpg.ex / Photos / raw / / Photos / jpg /

rsync -vaHx --progress --numeric-ids --delete --delete-excluded /写真/ jpg / /バックアップ/写真/ jpg /


さらに、例外を含むjpg.exファイルは/ Photosフォルダーにあります。 なぜなら このプログラムには、使用するファイルを指定するだけの機能はないため、少しcheしました。 ファイルの内容は単純です：

+ * .jpg

+ * .jpeg

-*。*




以上です。 これら2つのコマンドをcronに入れると、このプロセスは自動的に、目に見えないように行われます。 （以前は手動で同期を開始する必要がありました）。

出力で、私は得る：

/写真

---> / raw

------> 2010_07_11キャンドル

------> 2010_07_31市

------> /など



---> / jpg

------> 2010_07_11キャンドル

------> 2010_07_31市

------> /など





プロセス全体を説明するのがそれほど難しくないことを願っています。 誰かに質問がある場合は、質問することができます。
