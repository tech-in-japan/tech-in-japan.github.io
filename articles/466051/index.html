Automate profiling Java meetup 10/09

We invite you to an open meeting on September 10. We automate the profiling of Java, which will be held at the Raiffeisenbank site in Nagatino.



Together we will figure out how to test and optimize the performance of microservices, and we will also learn how to get ourselves out of the routine of checking profilers for JVM.







Vyacheslav Smirnov, Raiffeisenbank

Java 8 profiling experience



While testing and optimizing the performance of microservices, I formed approaches to JDK profiling. In the report I will tell about the experience of using tools: SJK, Java Fligth Recorder, Java Mission Control (Oracle Java 8), Telegraf, Influxdb, Grafana. And about some errors in libraries: org.json4s, alpakka.



Alexey Ragozin, Independent Consultant

JVM Profiling and Diagnostics Automation



What shall we talk about:



  • Diagnostic information available in the JVM for automatic collection (JMX, FlightRecorder, etc.). How to get it and how it is useful;
  • Analysis of information in automatic mode. How to move from an array of source data to useful integral metrics of the quality of the application in terms of performance.


Doors open for guests at 18:30, the start of reports at 19:00

Participation in the event is free, but by prior registration .
