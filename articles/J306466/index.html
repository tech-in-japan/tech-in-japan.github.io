OpenVZの新しいバージョン





昨春、Virtuozzoがなぜオープン開発モデルに切り替わるのか、そして商用製品のソースコードを開くポイントは何か について話しました。 昨日、オープンコンテナ仮想化システムOpenVZのコードベースと商用製品Virtuozzo（Parallels Cloud Server）の合併の結果として形成されたOpenVZ 7.0の最終バージョンのリリースを発表しました 。 OpenVZの新しいリリースには多くの変更が含まれており、それらについて順に説明します。





テクノロジーではなく、製品です！





OpenVZプロジェクトが登場したとき、コンテナ仮想化用のパッチとコンテナ管理用ユーティリティを備えたLinuxカーネルがこのプロジェクトの一部として利用可能になりました。 ユーザーがコンテナ仮想化システムを構築できるコンポーネントを提供しました。 OpenVZ 7.0では、テクノロジーは提供していませんが、Linuxディストリビューションという形で完全なソリューションを提供しています。 新しいバージョンの使用を開始するために必要なことは、インストールイメージをダウンロードし、インストーラーのすべての手順を実行することだけです。 KVM / QEMUに基づいたコンテナーとハイパーバイザーの両方の仮想化の可能性を備えたサーバーになります。 有料ユーザーの場合、再インストールせずにライセンスを購入することでインストールできる追加機能が利用できます。



その他の統合オプション





市場は仮想化のための製品とソリューションで飽和状態にあり、単一の製品が完璧ではなく、すべての可能な機能を提供しているわけではありません。 これを実現するために、OpenVZ統合の機能を拡張しました。 OpenVZ 7は商用Virtuozzoのアーキテクチャを継承しているため、Parallels APIを使用できるようになりました。 さらに、別個のvirtuozzoドライバーを使用してLibVirtと統合することが可能になりました。 このドライバーを使用すると、標準ユーティリティvirsh、virt-install、 GUIシェルvirt-manager 、およびLibVirtに統合されたその他のアプリケーションを使用して、コンテナーとOpenVZ仮想マシンを管理できます。



以前は商用Virtuozzoでのみ利用可能な新機能





メモリ管理メカニズムは完全に再設計され、このメカニズムの第4世代が新しいバージョンで導入されました。 新しい実装では、Linuxカーネルとvcmmdサービスによって提供されるメモリcgroupsサブシステムを使用します。



OpenVZの以前のバージョンでは、コンテナテンプレートはかなり不便でした。OpenVZWebサイト上のディストリビューションの現在のバージョンのテンプレートを更新する必要があり、OpenVZユーザーはサーバー上のこれらのテンプレートを更新することを忘れないでください。 OpenVZ 7では、コンテナにEZテンプレートを使用する機能が追加されました。これにより、OpenVZを備えたサーバー上のテンプレートの管理が容易になります。 管理のために、 vzpkgユーティリティを使用することをお勧めします 。



OpenVZの以前のバージョンの機能はプロジェクトwikiに文書化されており、その記事はコミュニティの人々によって書かれました。 このアプローチには欠点がありました-情報の構造が不十分で、記事の関連性が失われ、更新されなかった場合もありました。 新しいバージョンでは、ドキュメント部門が作成したWebサイトdocs.openvz.orgで完全なドキュメントを入手できます。 さらに、OpenVZのドキュメントに変更を加える可能性は依然として存在します。ソースコードはopenvz-docsリポジトリで入手できます。



その他の技術的変更から：





Linuxカーネルは、最新のRed Hatカーネルバージョン-RHEL 7 （バージョンは3.10+カーネルに対応）に基づいています。 RHEL5、RHEL6コアと比較してパッチサイズが大幅に削減されました。これは、メインのLinuxカーネルに既に含まれている標準テクノロジー（メモリcgroup、CRIU、NFS仮想化）の積極的な使用により達成されました。 コンテナのライブマイグレーションは、 vzkernelの以前のバージョンで実装されていたプロセスのフリーズ/アンフリーコードを使用する代わりに、 CRIUおよびP.Haulツールを使用して実装されるようになりました。



不快なニュースもあります。 これまで、OpenVZの以前のバージョンと商用Virtuozzo製品では、vzctlユーティリティは独立して開発されました。 OpenVZ / Virtuozzo 7.0では、商用製品のバージョンを残すことが決定されたため、vzctlの互換性が壊れていました。 コンテナと仮想マシンを管理するには、prlctlユーティリティを使用することをお勧めします。 新しいユーティリティの使用を開始するには、一般的なコマンドの構文で「チートシート」を使用できます。 将来のバージョンでは、vzctlユーティリティを放棄し、メインユーティリティとしてprlctlを使用する予定です。



OpenVZ 7.0をインストールするためのインストールイメージは、OpenVZサーバーまたはプロジェクトミラーのいずれかからダウンロードできます。
