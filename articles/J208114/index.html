大Year日のタンバリンとのダンス、または古い車のセカンドライフ

こんにちは 今日は、大how日を過ごした記事のトピックの一部としてお話しします。 伝えるだけでなく、写真も見せます。 とても刺激的なアクションでした。 まあ、それはすべてそのように始まった。 ホームミニサーバーをインストールして構成した後、私の母は自分用のコンピューターを望んでいました。 のように、私は悪いです。 インターネットで情報を読みたい、テキストを入力したい、など。 私は考えて考えたと思った。



かつて、私はシンクライアントに関する記事を1つ読みました。 Intel i486、pentium I、IIなど、通常の操作で使用できる古いマシンをセットアップするにはどうすればよいですか。ビデオの視聴、インターネットのサーフィンなど。 素晴らしいですね。 しかし、それは本当です。 あなたはそのようなシャーマニズムをすることができます。 これを行うには、強力なサーバーとクライアントが必要です（私の場合、Intel Pentium MMX 200 RAM 64Mb 2MBビデオでHDDなし）。







一般的に、それは新年の前でした。 このデザインは、お母さんにとってお正月の良い贈り物になるでしょう。 私は長い間、それを延期しないことに決めました。 コンピューターの場合、新しいキーボードとマウスという意味のないものを購入する必要がありました。 笑ってはいけませんが、キーボードはATケースでは使用できなくなりました。 まれなボードでは、これはDIN-5コネクタです。 古いテープレコーダーのプラグのように見えます。 マウスはps / 2に接続できます。マザーボードにはこのための特別なピンがあります。



そして、私もキーボードps / 2を買いました。 そして、なんとかAT軍団と友達になりました。 これを行うには、アダプターを束ねる必要がありました。 古いアイドルATキーボードからコードを取り出し、アイドルマザーボードからps / 2コネクタを切断しました。 30分後、次のことが起こりました。







それだけです！ これで、古いコンピューターで光学式マウスと新しいキーボードを使用できます。 部品はすべて揃っていました。 この奇跡のためにサーバーをセットアップする秘跡を実行することが残った。 このジャンクが機能するためには、いわゆるLTSPサーバーをインストールするだけで済みました。 興味のある方はウィキペディアのページen.wikipedia.org/wiki/LTSPでこのプロジェクトについて読むことができます



私のサーバーのオペレーティングシステムはもちろんGNU / Debian Linuxです。 7番目のバージョン（安定版リリース）。 ユリ・ガガーリンが言ったように行きましょう。 最初は、サーバー側を念頭に置く必要がありました。 sshを介したマシンへのリモートアクセスを使用して、LTSPの奇跡が機能するために必要なパッケージをインストールしました。 

apt-get install ltsp-server-standalone


DHCPサーバーやTFTPなど、必要なすべてのパッケージがプルアップされました。 はい、1つで構成する必要があります。 すぐに使えるものはありません。 条件付きファイルでこのファイルを終了する必要があります。 そして今、約束されたタンバリンとのダンスはなくなりました。 これは、「赤い目」でよくあることです。 これらの踊りは大New日中ずっと続きました。 だから退屈していませんでした。 私は夜中もテレビを見ませんでした。



良いニュースが待っていました。 TFTPは設定する必要はなく、独立して巻き上げられます。 DHCPの基本設定を行う必要があります。 ネットワークアドレスをマシンに配布し、TFTPを介してネットワーク経由でダウンロードします。 /etc/ltsp/dhcpd.confに構成ファイルが見つかりました。 私のファイルは次のようになります。 



 authoritative; subnet 192.168.0.0 netmask 255.255.255.0 { range 192.168.0.1 192.168.0.250; #      option domain-name "paulo737.pp.ua"; #     option log-servers 192.168.0.1; # ip    option domain-name-servers 192.168.0.1; #   option broadcast-address 192.168.0.255; #   option routers 192.168.0.1; #   next-server 192.168.0.1; # get-lease-hostnames true; option subnet-mask 255.255.255.0; #   option root-path "/home/ltsp/i386"; #    LTSP- filename "/ltsp/i386/pxelinux.0"; #      }


ファイル/ etc / default / isc-dhcp-serverでさらにいくつかの変更を行う必要があります 

 DHCP_CONF=/etc/ltsp/dhcpd.conf INTERFACES="eth1"


ここですべてが明確になるはずです。 DHCPサーバーの構成ファイルの場所を示し、eht1インターフェイスでローカルネットワークからの要求をリッスンします。 これらの簡単な手順の後、dhcpサーバーを再起動する必要があります 

 /etc/init.d/isc-dhcp-server restart


ltspサーバーのファイルシステムのインポートを構成するために残ります。 / etc / exportsファイルに行を追加します 

 /home/ltsp/i386 *(ro,no_root_squash,async)


顧客向けの作業環境を作成できます。 

 ltsp-build-client --base /home/ltsp


これは、シンクライアントがネットワーク経由でダウンロードされる調達システムです。 このシステムを構成するには、chroot環境に入る必要があります。 

 ltsp-chroot


そして、それは行きました... apt-get installと必要なパッケージ。

nfsサーバーを再起動すると完了です。



今、問題は小さなものに任されています。 ネットワーク経由で起動するクライアントコンピューターを準備する必要があります。 これは、次のサイトrom-o-matic.net/gpxe/gpxe-1.0.1/contrib/rom-o-maticに役立ちます。 ダウンロードするイメージを準備します。 ISOイメージを選択しました。 ディスクに記録した後、CD / DVDドライブを使用してロードします。



そして...ああ奇跡！ 古いコンピューターでは、次のものが表示されます。 忘れた人のために思い出させてください。 ハードディスクと64 MB（64メガバイト）のRAMしかありません。 ギガバイトと混同しないでください。
