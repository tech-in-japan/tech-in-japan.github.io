写真アーカイブの作業カタログの構成



アーカイブ自体にゴミや混乱がある場合、バックアップユーティリティは状況を保存しません。 したがって、写真のアーカイブを整然と保つために必要な最初のアクションは、正しいルブリケーションとカタログ化です。



ほとんどのデジタルカメラ（快適な例外はペンタックスSLRなど）は、「100_FUJI」や「103NIKON」などの情報のない名前でフォルダーに画像を書き込みます。 フォルダをカードまたはカメラから写真アーカイブの作業ディレクトリにコピーするとき、30分後に[CAMERA_THAT IS IMAGE_WHERE DISCONTINUED]という形式に名前を変更することを後悔しません。 たとえば、出力では、フォルダの名前が不明瞭な「330_FUJI」から完全に意味のある「FUJI-X20_Corporate_November_2010」に変わる場合があります。 そのような情報を失うことは、すでにはるかに困難であることを認めなければなりません。



ちなみに、すべてのコンピューターで作業ファイルをコピーするためのフォルダーは「ワークフロー」と呼ばれます。



ファイル自体の名前についても同じことが言えます。 私のファイルは、エディタで未処理と処理に分けられます。 最初のタイプのファイルが「1010394.NEF」などと呼ばれる場合、処理済みファイルの名前は常に[WHAT IS PICTURE_DxB]に変更します。Dは長さ、Bは結果ファイルのピクセル単位の高さです。 最終的な名前が「Bridge_through_re__Kwai_900x675.jpg」のように見えるとしましょう。 したがって、特定のファイルを処理していたかどうかは常によくわかります。 処理されたファイルは、ソースディレクトリと同じフォルダ内のワークフローディレクトリに保存されます。 必要に応じて、これらのファイルを、Workflowの同じベースディレクトリに保存されている「To To print」および「To Publication」というタイトルの特別なフォルダーに移動します。



時々、私は手動でアーカイブします（バックアップではありません！）ワークフローフォルダーの内容。 この場合、いじり終わったフォルダーは、カメラからコピーした後に付けられた名前を保存したまま、別の場所（別の物理ディスクに！）にコピーされます。 これらのフォルダーは、[Archive_YEAR]という名前のディレクトリに保存されます-たとえば、「Archive_2013」。



したがって、私は常に自分が写真アーカイブのどこにいるかを知っています。







バックアップ、アーカイブ、データセキュリティの計画



最初に、バックアップとアーカイブの違いを簡単に説明します。 アーカイブとは、安全な場所にデータを保存することです。 ただし、バックアップとは、 定期的な更新と迅速な回復機能を備えた安全な場所にデータを保存することです。



写真を含む電子データをアーカイブするには、それらをどこかにコピーするだけで十分です。バックアップは、より複雑なデータ処理を意味します。定期的なスケジュールされたコピー、整合性のチェック、データの安全性、必要に応じて回復の正確性。



適切な場所への1つのコピーによる完全バックアップの場合、おそらく、できません。 Mac OS XのiPhotoや一般的な7-Zipアーカイバなどの組み込み機能を備えたもの、またはEASEUS ToDo Backup 、 Handy Backup 、Symantec Backup Execなどの専用バックアッププログラムのいずれかを使用することは理にかなっています。



はい、私は知っています、ほとんどの人は私を理解しません。 通常、バックアッププログラムはライセンスされており、自宅にLinuxがあります。 さて、bashを使用すると、バックアップに必要なすべてのスクリプトを自分で記述できます。また、 BaculaやrsyncなどのLinuxのネイティブオープンソース製品は、この機能によってまったく問題ありません。 ここで使用するのは、原則ではなく味の選択です。 私は個人的に、3,000ドルでDSLRを購入し、それに対応する価格の交換可能な光学機器を購入した後、ライセンスソフトウェアを50〜100ドル節約するだけで、非常に原則的な生活を送ることができると考えています。 しかし、再び、私は完全にオープンソースであり、フリーソフトウェアの機能を過小評価したくありません！ 写真のアーカイブですべてが適切に機能した場合に限り、100個の花を咲かせましょう。
