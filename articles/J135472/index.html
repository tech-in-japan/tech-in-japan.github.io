X5000-平均的な企業向けのコンパクトなNAS

現在提供されているほとんどのNASストレージシステムは、Linux / Unixの特別に最適化されたファイル/サービスバージョンで実行されます。 このストレージシステムがWindowsコンピューターのみで構成されるローカルネットワークにインストールされている場合、NASのこの機能は深刻なマイナスになります。 多くの場合、NASがActive Directoryディレクトリサービスと適切に通信するには特別な設定が必要です。 さらに、多くの場合、Windowsネットワークを提供するシステム管理者はLinux / Unix環境で作業したことがないため、このようなNASを十分に活用するには、新しいオペレーティングシステムを習得する必要があります。





HPは11月29日に、Windows Storage Server 2008 R2 Enterpise Editionを実行するX5000 G2ネットワークストレージシステムNASの第2世代を導入しました。 X5000 G2にはOSがプリインストールされており、Active Directoryとの統合を含め、このNASをWindowsネットワークに展開するのに1時間もかかりません（独立したTolly研究所でのテストでも確認されています）。



X5000システムの第1世代が3ユニットサーバーモジュールと3ユニットディスクアレイで構成されていた場合、X5000 G2は2つのハーフハイトHP BladeSystem c-Class 2ソケットブレードと1つの3ユニットシェルフにインストールされた16台のSASハードドライブを備えたディスクモジュールの設計を使用します（同じこの構成は、Microsoft Exchange Server 2010用HP E5000メッセージングシステムによってこの春にリリースされた電子メールソリューションにも使用されます 。 基本システムは32 TBまで拡張でき、SAS 15K rpmディスクを使用した構成では、1万人のユーザーにファイルサービスを簡単に提供できます。 中規模企業のファイルストレージのニーズを完全にカバーし、多くの大企業のファイルサーバーを統合するのに適しています。 さらに、大きなLFFまたは小さなSFFディスクを備えたD2000ファミリーのディスクシェルフを追加することでシステムを拡張し、合計容量を128 TBに増やすことができます。



機器の故障から保護するために、X5000 G2コントロールブレードはフェイルセーフアクティブ/アクティブクラスターに組み込まれ、さらに、システムのすべての主要コンポーネントが予約されています。 ブランチオフィスでの展開を容易にするために、頻繁に要求されるWindows Server Branch Cacheファイルと独自のiLOリモート管理テクノロジー（HP ProLiantサーバー所有者に馴染み深い）のキャッシュをサポートし、重複排除を使用して同じファイルのストレージを取り除きます。 X5000 G2のディスクリソースのスペースを節約するために、システム管理者は、個々のユーザーとグループにクォータを入力し、分類機能を使用して特定のファイルをディスクに書き込むことをブロックするルールを適用できます。 最後に、組み込みのスナップショットおよびレプリケーション機能により、ファイルが誤って削除または変更された場合に備えて、ファイルのコピーを作成できます。



X5000 G2にはすでにiSCSIターゲットライセンスが含まれているため、ファイルレベルだけでなく、データベース、アプリケーションサーバー、および仮想マシンを処理するときにブロックレベルでもデータを操作できます。 これらの機能はすべてX5000 G2の標準構成に含まれているため、このNASの所有者は追加ソフトウェアのライセンスに余分なお金をかける必要はありません。
