Google Chrome 6の期待とChromium 7の概要

超高速ブラウザが必要ですか？ 私も。 Webアプリケーションと完全なHTML5実装が必要ですか？ 私も。 しかし、待つ必要があります。 そして、いくつかの側面では、仕事は巨大であるため、長い間待ちます。



通常、ソフトウェア開発では、プログラマーがクールな新しいテクノロジーを追加することから注意をそらして、利用可能なすべての機能が完全に機能するようにする必要があります。 コードフリーズと呼ばれるこの瞬間は、Google Chromeの6番目の「マイルストーン」でした。





Googleはブラウザの絶え間ない更新を信じており、開発と広告の両方で積極的な作業を実証し、ラインを抑圧しています。 Chromeは、上位3つの主要なブラウザーにスマートに侵入し、Internet ExplorerとMozilla Firefoxと競合（まだそれほど深刻ではありません）し、Opera Software ASAとAppleのブラウザーをユーザー数で活発に凌ぎました。 そして、これはブラウザの不完全な2年間にすぎません。 （さまざまな情報源によると、Google Chrome は世界市場の7〜9％を占めています ）。



地下鉄に乗り遅れて、次の電車を待つことはありませんが、新しい電車が到着するまで待たなければなりません。 これは、Google Chromeの一部の機能で発生したことです。 Chrome 6と呼ばれる電車に乗る時間がなく、Chrome 7を待っているだけでした。



印刷サポートは、ウェブブラウザー市場に長く存在していた他のブラウザーと比較して、Google Chromeブラウザーの比較的弱い点の1つでしたが、Googleは競合他社に追いつき、アウトパフォームするために懸命に働きました。 また、選択したテキストフラグメントを印刷する実装が不十分であるという形で障害が発生し、Pwn2Ownコンテストでブラウザーをハッキングしようとして失敗したなどの成功がありました。



印刷の問題に関しては、Windowsユーザーには安心感があります。 開発者はバグを修正し、変更はChrome 6のリリースに含まれますが、MacおよびLinuxのユーザーは、印刷機能全体がやり直されるバージョン7を待つ必要があります。 印刷プレビュー機能への長いリクエストは、開発者に影響を及ぼし、機能が実装されます。これは非常に嬉しいことです。 新しいバージョンのリリース率を考えると、これは比較的すぐに行われます。 2010年秋冬。



Chrome 6を選ぶ理由

では、なぜGoogleはChromeのバージョンを「アップグレード」することにしたのですか？ 本当に新しいチップや機会はありませんが、「新しい列車を待つ残り」機能だけがありますか？ もちろんありますが、そのために多くのバージョンから庭をフェンスで囲む必要がありますか？ したがって、バージョン5とは異なり、Chrome 6は次のことを行います。

* 拡張機能を同期します。 これで、お気に入りのアドオンを検索する際に、拡張機能の広範で不十分な構造の公式ディレクトリに苦労して掘り下げる必要がなくなりました。 ブラウザーをロードし、同期を開始しました。 わあ！

* SVG形式の画像の出力を高速化します 。

* ブラウザの設定ボタンを組み合わせて、ページを操作します 。 最近のファッショントレンド。 Opera 10とMozilla Firefox 4はどちらも「ワンボタン」モデルを実践しています。 便利ですか？ みんなに決めてください。

* 閲覧履歴を同期します 。 一部のブラウザユーザーはパスワード同期機能を夢見ていますが、これは便利な機能だと思います。

*高品質、高解像度の画像のスケーリングを最適化します 。 マルチコアプロセッサのユーザーはこれに特に気づくでしょう。

*もちろん、V8 JavaScriptエンジンのパフォーマンスの標準的な向上、Web標準との互換性の改善、およびHTML5などの有望な開発。 根拠がないように、証拠を適用します：

Chrome 6には他に何が表示されませんか？

* タブバーを上から横に移動する機能には、 「終了」する時間がありませんでした。 Windowsで--enable-vertical-tabsオプションを使用して機能を実行できますが、開発者はこの機能がアクティブに開発中であり、したがって不安定になる可能性があると警告しています。

* HTML5でのフルスクリーンビデオ再生のサポート 。 Google Chromeはこれを行う方法を知りませんが、Googleはサードパーティのプラグイン（Adobe Flashなど）に依存することを明らかに望まないため、HTML5を使用したビデオの本格的な作業に非常にコミットしています。

* Webアプリケーションを高速化するためのNativeClientの実装。 もちろん、これがブラウザー専用に行われていると言うのは時期尚早です。ご存知のように、GoogleはChromium OSオペレーティングシステムに集中的に取り組んでいるからです。
