Google、Facebook、YouTube、およびその他のITモンスターの最初のバージョン（および独自のビジネスを開発するという観点からあなたに教えることができるもの）





「私たちが知っていて愛している最高のものは小さなものから始まりました」とジョエル・ガスコインは言いました 。



Buffer CEOのJoel Gascoigneは最近、小さな新しいプロジェクトを開始することの重要性について投稿しました。 彼は、他の誰かの苦労の結果である完成品を見ると、この時点でクリエイターがどれだけの時間を費やしたかを忘れることができることに注目しました：



「製品とブランドの進化プロセスが今日の製品の構築にどのように貢献し、企業にとって不可欠であるかを理解することは非常に困難です。」



ジョエルは、小さなプロジェクトを思い起こせば成功の可能性が高まると言います。 小さく始めて、プロジェクトを成長させ、大幅に成長させます。



「スタートアップを構築しようとさえしないでください。 これらは時期尚早です。 手始めに、あなたにとって面白そうなものを作りましょう。」-ポール・グラハム



IT業界の巨人が人生の初めにどれだけ単純かを知るために、タイムマシンに乗って過去を調べてみることをお勧めします。 設計では、会社の「舞台裏」で起こったことをすべて伝えることはできませんが、製品やサービスの開発を長期にわたって追跡するのに役立ちます。 さらに、各クリエイターは、自分自身が小さく始めた理由と、これがどのように成長して成功を収めたのかについての有用な情報を持っています。



それでは、これらの企業がいかに控えめなプロジェクトから強力なITビジネスの巨人へと進化したかを見ていきましょう。



Facebook-「長期にわたって働く」



「私は長期的に何かを構築するためにここにいます。 それ以外はすべて楽しいです。」-マーク・ザッカーバーグ



Zuckerbergには大きな目標がありました。彼はFacebookが世界中の人々の間のソーシャルコネクションを構築し、強化することを望んでいました。 そして、大学生だけを対象とする小さなソリューションですべてが始まったという事実にもかかわらず、Facebookは最大かつ最も人気のある（今日の）ソーシャルネットワークの1つに成長しました。 Facebookはこれまで成長を続けています。なぜなら、その作成者は長い間会社の構築に取り組んできたからです。



「これは感傷的に聞こえるかもしれませんが、特に社会的側面で人々の生活を改善したいと思います。そして、世界をよりオープンにすることは1日以上です。 10〜15年かかります。」



2004







今日、Facebookは当初よりも大きく複雑になっていますが、Zuckerbergは彼の考えに忠実です。



「Facebookの基本原則の多くは、人々が追加情報にアクセスし、それらの関係を強化することで、この世界がより良い場所になるという事実に基づいています。人々はお互いをよりよく理解し、思いやりを持つようになります。 私にとって、これは基本的な原則です。 困難な日には、私が退却する準備ができているとき、この信仰は私が前進するのに役立ちます。



2013







Google-1つの目標に焦点を当てる



「セルゲイと私がGoogleを設立したとき、会社がそのような規模に達し、今日の影響力を獲得できることを期待していましたが、想定していませんでした。 私たちは、人々が情報をより効率的に見つけられるようにすることに非常に興味がありました」とラリーペイジ。



1998







長年にわたり、Googleのデザインは、会社の活動が大幅に拡大し、マップ、YouTube、Gmail、ドライブなどの新しいサービスが追加されたにもかかわらず、かなり最小限にとどまっています。 そして今、あなたがGoogleホームページにアクセスしたとき、Googleがまだ一つのことに焦点を当てていることは明らかです：効果的な情報検索。



「Googleと言えば、できるだけ早くユーザーのニーズに応えようとしています」-ラリーペイジ。





2014







Tumblr-「最初に自分で作ろう」



Tumblrの創設者であるDavid Karpは、彼自身のニーズを満たすためにサービスを作成しました。 ブログはますます人気が高まっていますが、それはテキストを書くことに焦点を合わせています-デビッドが正しく指摘したように、誰もがやりたくないことです。



「あなたはこの巨大な白いフィールドを与えられます。そこに機知に富んだ見出しを入力する必要があります。それは人々に先に進んであなたの投稿を読むよう説得するでしょう。 投稿は、小さな画像を含むHTMLで正しくフォーマットされたテキストのいくつかの段落で構成される必要があります（配置を忘れないでください）。 そして、それは本当によく書かれ編集された資料でなければなりません」とデビッド・カープ。



2007年





今日、KarpはTumblrの成功を、「より多くのチップ」（写真やGIFを含む）を使用するという考えだけでなく、チームが自分が愛して使用するものの作成に注力しているという事実とも関連付けています。



「私にとって、Tumblrが今日まで何とか維持している重要な機能の1つは、私たちのチームが主にそれ自身のために作成し、毎日何時間も喜んでそれを使用する製品であるということです。 そして、これは製品に有益な効果をもたらすだけでなく、その主な目的を忘れず、コースから逸脱しないようにするのにも役立ちます。 そして、これは、私の意見では、元の概念を保持しています。」



2014







YouTube-「すべてを知っていると思わないで」



YouTubeチームは、以前の経験に基づいて人々を引き付ける方法に基づいて、デザインのテストと製品の作成に取り組んでいます。



「サービスをどのように開発すべきかについて、私たち自身のアイデアがたくさんありました。 PayPalとeBayについて言えば、YouTubeはオークションで製品にビデオを追加する素晴らしい方法であるように思えましたが、この方法で製品を使用する人はいないことがわかったため、対応する機能は追加しませんでした」 （チャド・ハーリー）。



2005年







YouTubeの共同設立者であるChad Hurleyは、スタートアップの創業者に、あなたのビジョンだけに基づいて大規模で複雑な製品を開発するための最初のステップをつかむのではなく、彼らの仮定をテストし、ユーザーの声に耳を傾けるようアドバイスしています。



製品の作成を開始するとき、すべてを知っていると思い込まないでください。 ユーザーの声に耳を傾け、適応します。





2014







Yahoo! -「成長に備える」



上記のほとんどの企業と同様、Yahoo！ 特定の問題を解決することを目的とした非常に小さなプロジェクトとしてその存在を開始しました：ヤフーの将来の創設者！ 後でアクセスしたいウェブサイトを見つけるのに苦労したため、ウェブページのカタログ化を開始しました。



「Jerry's Guide to the World Wide Webと名付けました。 世界中の人々が私たちが作成したデータベースを使い始めたため、瞬きする時間はありませんでした。」-ジェリー・ヤン。



1996







彼の個人的な経験に基づいて、共同設立者のジェリー・ヤングは、彼のビジネスを拡大する準備をするようアドバイスしています。



「ビジネスの成長に備える必要があるように思えます。 そして、ビジネスモデルがプロジェクトの高速スケーリングを提供できない場合、最終的には何ももたらされません。」





2014







Amazon-顧客を何よりも優先する



「当社の革新はすべてお客様から始まり、お客様のニーズに基づいて開発されます。 これが、私たちが何かを発明する方法の標準になります。」-Jeff Bezos



AmazonのクリエーターであるJeff Bezosは、顧客重視が最初からAmazonの競合他社の最高の優位性であると主張しています。



「18年間忠実であり、成功の理由である3つの重要なAmazon開発アイデアがありました。お客様から始めます。 作成します。 我慢してください。」





1999







オンライン書店から、Amazonはほとんどすべてを販売する巨大なオンラインサイトに成長しました。 しかし、信じられないほどの成長にもかかわらず、ベゾスは顧客が望むものに引き続き注力しています。



「朝のシャワーを浴びて、競合他社は、主要競合他社の1つをどのように上回るかを考えます。 これらの瞬間に、私たちはクライアントの目を通してそれを見て、新しい何かを作成する方法を考えています。」



2014







大企業が今日どのように小規模に始めたかについて、多くの例があります。 ジョエル・ガスコインは彼の記事で、デール・カーネギーの本「友人を作り、人々に影響を与える方法」という本を引用しました。この本は短い会話と一枚の紙から始まりました。



私が本当に気に入っているもう1つの例は、 Sunriseスタートアップです。これは毎日のメールニュースレターとして登場し、現在ではiPhoneの本格的なカレンダーになっています。
