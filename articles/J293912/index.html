サイトビルダーのアイデアをどのように変えたか

ウェブサイト作成ツールのアイデアがたどった道についてお話しします。その後、完成品を作り、ここにご案内します。



背景



インターネット上で自分自身または組織を紹介するサイトが必要でした。 ソーシャルネットワーク上のページは、彼のニーズを満たしていません。 同じ環境/視聴者が存在しない、「自分自身」の感覚がなく、広告が邪魔されている、または十分な多様な機能がない。 合理的な解決策は、専門家にウェブサイトを注文することですが、それは誰にも適していません。 それからすべてを自分でやりたいという願望があります。



オプションを検討してください。





タイプA



「いいえ、おばあちゃん、私です！」

遠く離れた王国のヴォフカ



人はウェブサイトを作成する意欲があります。 一般的な理由は、すべてを制御したいという欲求、知識への情熱、または問題のある技術的な静脈です。



この人は、サイトを作成するあらゆる手段を理解する準備ができています。 ほとんどの場合、手頃な価格のソリューションから始めて、好みに合わせてサイトビルダーを選択します。これは、マウス操作によってサイトが作成されるWebサービスです。 選択は自由時間と野心の量に依存します。



タイプB



「すきなし-耕作者ではなく、ハンマーなし-鍛冶屋ではありません。」

ことわざ



多くの時間と労力を必要としない人は、サイトを作りたいと思うでしょう。 コンテンツがアップロードされるWebサービスが彼に提示され、出力は最新のデバイスで適切に見える美しくモダンなサイトであると思われます。



彼はプロセスから喜びを得られないので、この人はプロセスにとどまりたくありません-あなたは短時間で結果を必要とします。



パターン



これ以上簡単なことはありません！ 両方のタイプのニーズは、テンプレートベースのサービスによって満たされます。



ユーザーは質問に答えて情報を入力します：「名前は何ですか？」、「サイトは何ですか？」、「連絡先の詳細？」、「販売するものは？」、「投稿する写真は？」。 データを入力すると、コンテンツ表示オプションのリストがロードされます-あなたが好きなテンプレート。 タイプBはこれに専念し、タイプAは追加の設定を処理します。



ただし、テンプレートには、固定構造で任意のデータを置き換えるという考えから生じる重大な欠点があります。 ユーザーは、コンテンツの詳細を考慮せずに、すべてのテンプレートを選択するように招待されています。これは、テンプレートがコンテンツを表示するのと同じように有益であるかのようです。 見出しが5つの単語のスローガンを暗示しており、10個ある場合、または3〜5個の項目のメニューが良さそうで、2個ある場合はどうすればよいですか



サイトは複雑すぎて、多数の多目的で多様なテンプレートを同時に作成できません。



パターンなし



テンプレートはコンテンツの機能を考慮していないため、テンプレートを放棄すると、ユーザーがページ、レイアウトを作成し、最初はコンテンツの詳細に基づいて必要に応じてそれらを埋めるサイトを作成するためのより意味のある方法を提供できます。



単純な順次ステップは素晴らしいアイデアです。1年前に決定し、ビジネスに取り掛かりました。この方法は主にタイプBにとって興味深いと思いました。



結果として生じるサービスをタイプBの数十の潜在的な顧客でテストした結果、型破りなアプローチでは作業が大幅に複雑になることがわかりました。 ユーザーは最初はすでに失われており、作成するページを理解していません。 その場合、コンテンツをページに配置する方法と順序を知りません。 タイプBには、最終的に何を取得したいのかについての全体像がないため、このオプションは彼に適していません。



タイプAには、テンプレートがないことによるメリットのみがあります。しばらく時間を費やした後、彼は自分のニーズに合わせてテンプレートを作成し、適応させることを学びます。



このような状況では、テンプレートを完全に拒否すると、対象読者の一部が拒否されることになるため、現代のデザイナーは混合ソリューションを提供します。 例： squarespace 、 ukit 、 blocs



私たちの方法



タイプBを支援するツールが必要だったため、コンセプトの変更が行われました。 このタイプは、広範な機能を備えたサイトビルダーを必要とせず、低いエントリしきい値が重要であるため、設定の数は劇的に減少しましたが、新しいデータタイプが登場しました。 重点は、サイトの視覚的なコンポーネントの構築から、入力の簡素化と外観の普遍的なソリューションの検索へとシフトしています。



そこで、 GitHub Pagesと静的サイトジェネレーターのおかげで、今では人気のある、古典的なブログのような個々のサイトを作成するための新しいサービスを得ました。



次の記事で、コーンとサービスの作業に関連する肯定的な結果について詳しく説明します。
