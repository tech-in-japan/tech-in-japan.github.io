Juniper MX80 + Cisco ASR1002 LACP Ether-channel

こんにちは このタスクは、ジュニパーMX80とCisco ASR1002という異なるメーカーの2つの鉄の間に4ギガビットポートのEtherチャネルを作成するように設定されました。 Cisco CatalystとASRの間で問題なく機能した場合は、ジュニパーをいじる必要がありました。





それはすべて簡単に始まります。 tsiskで、ポートチャネルインターフェイスを作成します。

interface Port-channel 1







必要なIPアドレスとパラメーターが設定されます。



次に、必要なインターフェイスで、入力します

channel-group 1 mode active







このルーターにはバランシング設定はありませんが、発信ASRトラフィックは正常にバランシングされているようです。 katalistには、src-dst-macの負荷分散が含まれています。



Tsiskaが完成したら、ジュニパーに移りましょう。



chassis {

aggregated-devices {

ethernet {

device-count 1;

}

}

network-services all-ethernet;

}









集約されたインターフェースの数でdevice-countを設定します。この場合、インターフェースは1つになります。



入力する必要なインターフェイスで

ge-1/0/0 {

gigether-options {

802.3ad ae0;

}

}







集約されたインターフェースを作成する

ae0 {

aggregated-ether-options {

minimum-links 1;

link-speed 1g;

lacp {

active;

periodic fast;

}

}

unit 0 {

family inet {

address 10.100.0.1/30;

}

}

}











リンクが上がると、pingが実行されます。 ジュニパーでバランスを取るためのドックが見つかりませんでした。チャネルにトラフィックを送信すると、トラフィックがツィスカからジャンパーにバランスが取れており、1つのチャネルですべてが1つのジャンパーからツィスクに移動していることがわかりました。



文書を掘り下げても何にもつながりませんでした。 ジュニパーに初めて精通し、「？」をクリックして鉄片の構成を調べました。 私は研磨セクションでバランスをとっていました（何らかの理由で誰もマニュアルでそれについて書いていません）



policy-options {

policy-statement balance {

then {

load-balance per-packet;

}

}

}









したがって、ジュニパーからtsiskへのトラフィックは、インターフェイスで分割され始めました。



私にとってそれは最も些細なことではなかったので、誰かが役に立つことを願っています。
