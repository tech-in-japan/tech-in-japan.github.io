これらのプロトタイプの何がそんなに重要ですか？



通常の世界でのプロトタイプの概念は非常に古いものです。 建物の建設の前には、プロジェクト（プロジェクト=プロトタイプ）、シリアルカーの生産-コンセプト（概念車=プロトタイプ）が先行しています。 それにもかかわらず、デジタルの世界では、長い間、高予算のプロジェクトでさえ、タイピングによって実際に開発され、一般的に中小企業向けの製品は請負業者に触発されました。



デジタル世界のプロトタイプは、将来の製品の視覚化であり、この製品のユーザー間で仮説をテストしたり、製品を作成することを決定した人々にアイデアを伝えたりするのに十分です。 プロトタイプを使用すると、迅速に、そして最も重要なこととして、安価なフィードバックを得ることができます。開発したインターフェイスはビジネス上の問題を解決し、ユーザーとやり取りするための使いやすく革新的な方法であり、問​​題とタスクは正しく理解されています。 そうでなければ、これはすべて製品の発売後に決定する必要があり、これは費やされた時間だけでなく、お金でもあります。









プロトタイププロトタイプ-不和



プロトタイプは、アイデアをすばやく視覚化する手段として、近年普及してきており、一般的にデジタル製品の品質にプラスの影響を与えています。



同時に、プロトタイピングツールを採用して以来、市場はプロトタイプがユーザビリティプロセスの一部であり、プロジェクト管理ではないことをまだ十分に理解していません。



過去数年にわたって、ポートフォリオに数十個のプロトタイプを持っている多くのデザイナーと話しましたが、多くの場合、彼らはプロトタイプの作成プロセスを完全に理解していないだけでなく、なぜ彼らがなぜ必要なのかを常に自信を持って説明できません。



その結果、実際のコンテンツのヒントがなく、グラフィック形式がコンテンツより優先されるプロトタイプを見つけることができます。 UI / UXデザインの急成長中のセグメントでは、本格的なデザインモックアップがプロトタイプと呼ばれ、その作成に数十時間を費やしました。



プロトタイピングツールの状況と市場の発展も、ビジュアルパーツの作業に影響します。 これはすべて、プロトタイプの構成と美しさに特別な注意を払う誘惑につながり、主なものである構造と内容を忘れます。

効果的な構造を形成し、コンテンツを準備することは、最も長く最も労働集約的なプロセスであり、実践が示すように、最終製品に最大限の影響を及ぼします。



プロトタイプは一連の作品の結果です



ユーザーのニーズに基づいて真に効果的なインターフェイスソリューションを作成し、特定のビジネス上の問題を解決する手順を検討してください。







リサーチ



研究は最も重要な段階であり、その後のすべての作業を成功させるための基盤です。 それにもかかわらず、「研究」の2つの方法が市場でまだ受け入れられています：「私たちにTKを送ってください」と「150の質問のために簡潔に記入してください」。



技術的なタスクもブリーフも、真に効果的なソリューションを作成するために必要なレベルの没入感を提供することはできません。しかし、クライアントがテンプレートの質問に答えるのにかなりの時間を費やすこともありますが、実際の意味はよく理解していません。



したがって、ビジネス研究の問題で最も効果的な方法は、クライアントとの個人的なコミュニケーションです。 これにより、特定の各ビジネスのプロセスと機能に可能な限り没頭し、結果として後続製品全体の基礎となる微妙なニュアンスを明らかにすることができます。



多くのUI / UXデザイナーは、ユーザーとのコミュニケーションが必要だとは考えていませんが、ユーザーの調査により、異なる観点から問題を調べ、因果関係を特定し、動機を理解し、恐怖を発見し、疑問を聞き、洞察を見つけることができます。



設計ソリューション開発



この段階で、研究結果は特定の製品要件に解釈されます。どのナビゲーションが必要か、どの画面が必要か、どのニーズに対応すべきかなどです。



研究を製品要件に変換するための多くのツールがあります。ユーザー中心設計、価値提案キャンバス、ジョブズトゥービーンなどです。しかし、それらはすべて同じ結果をもたらします-インターフェースの意識的な設計。



設計ソリューションの開発の結果は、紙のスケッチ-スケッチであり、後で詳細のプロトタイプになります。



内容



スケッチから詳細なプロトタイプに移行する前に、すべてのコンテンツを準備する必要があります。 テキストフィッシュを使用する誘惑には結果が伴います。そのようなプロトタイプは実際には推定できません。さらに、実際のコンテンツが表示されると、すでに開発された製品に適合しないことがわかります。 そして、コンテンツを指定された制限にカスタマイズする必要があります。



したがって、最初にWHAT（コンテンツ）、次にHOW（デザイン）。


試作機



設計ソリューションの研究開発からコンテンツの作成に至るまで、チームは最大限に関与し、ターゲットオーディエンスとビジネスタスクのニーズを考慮した、本当に新鮮なインターフェイスソリューションを提供することができます。 このようなプロトタイプは、ユーザーで簡単にテストしたり、製品の作成を決定した人にアイデアを伝えるために使用したりできます。



良いプロトタイプと悪いプロトタイプを区別する方法



ここに簡単なチェックリストがあります。これは、相互作用設計の分野の知識がなくても、プロトタイプ「ゼリーフィッシュ」を装って実際の作業の結果に陥っているかどうかを判断できます。



それで、ほんのいくつかのポイント：



  1. プロトタイプは、製品テーマに一致する実際のコンテンツのみを使用します
  2. プロトタイプがコメントやデザイナーの助けなしに「報告」することを完全に理解している
  3. 設計者は、ビジネスおよびユーザー調査の結果へのリンクを使用して、各フロントエンドソリューションを主張します。


最終話



ある会社は、開発予算が25,000ドルを超えると見積もられた革新的な製品を作成することを決定しました。開発者は既に参照条件の記述を完了し、開始する準備ができていました。 私はこの会社に知り合いがいましたが、偶然にもこのプロジェクトの問題でした。 私は彼らにコンサルテーションを提供しました。その間、プロトタイプを作成して実際のユーザーでテストするまで、共同で開発を延期することにしました。



しかし、ユーザーがそのような製品を必要とせず、よりシンプルで、そして最も重要なことには、既存のソリューションを使用する方が効率的であることが判明したとき、すべてが研究プロセスですでに決定されました。



このストーリーは、一方では実際の製品を開発する前にプロトタイピングの重要性を、他方ではブラインドインターフェース設計の前に必要なプロセスの重要性を完全に示しているため、優れています。
