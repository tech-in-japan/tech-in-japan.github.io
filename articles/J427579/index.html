社内でのiOSのアプリケーション配布（社内でのiOSアプリの配布）

（注意、カット中のトラフィック）

企業内でiOSアプリケーションを準備して配布することは、特にVisual Studioを使用してWindowsでアプリケーションを作成する場合、簡単な作業ではありません。インターネット上のほとんどのチュートリアルでは、Xcodeのみを使用してMacOSについて説明します。 しかし、Appleの発案者との数時間の戦いの後、私たちは不可能と思われることを達成することができました。つまり、キリンとサイを交差させて 、Xamarin IOSアプリケーションをXcodeアーカイブにアセンブルし、すぐにMacOSで、配布に必要なファイルを受け取って、最終的にへのリンクを作成しましたアプリケーションが配布されます。



はい、聞くのはそれほど難しくありません。 ただし、Appleデバイス用のアプリケーションの開発に関しては、すべてが数回理解不能で複雑になります。 そして、勝利を収めたが困難な勝利を収めた後、このチュートリアルを書いて歴史に足跡を残したかったのです。









前提条件：



1. Apple Enterpriseアカウントが必要です-年間299ドル。



1ステップ。 証明書を作成します。



1.最初に、Macでは、証明書を作成するリクエストを作成する必要があります。 これを行うには、たとえば検索を通じてキーチェーンアクセスを開きます。











2.画面の左上隅でキーチェーンアクセスを選択し、ポップアップメニューで[認証アシスタント]-> [認証局に証明書を要求する]を選択すると、対応するウィンドウが開きます。











3.表示されるウィンドウで、「ユーザー電子メールアドレス」フィールド（電子メール）と「一般名」フィールド（キーの名前）に入力します。 また、「ディスクに保存」オプションを選択して、要求ファイルをコンピューターに保存します。 [続行]ボタンを押します。







4.次に、要求ファイルの名前を指定し、ファイルを保存するパスを選択する必要があるウィンドウが表示されます。 必要な変更を加えて保存します。







5.保存に成功すると、次のウィンドウが表示されます。 「完了」をクリックします。







6.要求ファイルが保存場所（この例ではデスクトップ）に作成されたことを確認できます。 または、「キーチェーンアクセス」のキーリストで作成されたキーを確認できます。











7.次に、証明書を作成する必要があります。これは、 Apple Webサイトで開発者向けにアカウントにログインして行うことができます。















8.ログインに成功したら、「証明書、ID、およびプロファイル」に移動します。証明書ページでも「IOS、tvOS、watchOS」が選択されていることを確認する必要があります。











9.ページの[証明書]セクションで、[生産]を選択します。







10.ページで、「+」画像のあるボタンをクリックして証明書を作成します。 作成する証明書のタイプを選択する必要があるページが表示されます。











11.この例では、社内配布方法に関心があるため、証明書タイプとして「社内およびアドホック」を選択します。 [続行]ボタンをクリックした後：







12.証明書を作成するために次のページに進んだ後、証明書のMacOSへの要求を作成する方法について説明します。 前の段落でこのクエリを既に作成しました。 [続行]ボタンを押します。







13.次のステップでは、デスクトップで以前に作成したリクエストファイルをダウンロードする必要があります。 ダウンロードが成功したら、「続行」をクリックします。











14.その後、証明書が生成され、次のページでコンピューターにダウンロードできます。







15.この例では、証明書をデスクトップにダウンロードします。 サイトで作成された証明書を確認することもできます。











ご覧のとおり、最終的に証明書を正常に受け取りました。 次のステップは、アプリケーションIDを作成することです。



2ステップ。 アプリIDを作成します。



前の手順で、証明書を正常に作成しました。次に、Apps IDを作成する必要があります。 これを行うには、次のものが必要です。



1. 開発者向けApple Webサイトのアカウントで、最初に「証明書、ID、プロファイル」に移動します。証明書ページでも「IOS、tvOS、watchOS」が選択されていることを確認する必要があります。











2.ページの次の[識別子]セクションで、[アプリID]を選択する必要があります。











3.ページで、「+」画像のあるボタンをクリックして、アプリIDを作成します。 作成されたIDの設定を選択する必要があるページが表示されます。 ID設定はアプリケーションごとに異なります。唯一重要なことは、サフィックスの[アプリID]列で明示的なアプリIDを選択する必要があることです。











4.アプリIDを作成すると、サイトで確認できます。







2つのステップの結果、証明書を正常に受け取り、アプリIDを作成しました。 次に、作成した証明書を使用してプロビジョニングプロファイルを作成する必要があります。 これにより、次のステップ「3ステップ。 プロビジョニングプロファイルの作成。」



3ステップ。 プロビジョニングプロファイルの作成。



前の手順で、証明書を正常に作成しました。次に、それを使用してプロビジョニングプロファイルを作成する必要があります。 これを行うには、次のものが必要です。



1. 開発者向けApple Webサイトのアカウントで、最初に「証明書、ID、プロファイル」に移動します。証明書ページでも「IOS、tvOS、watchOS」が選択されていることを確認する必要があります。











2.次に、ページの[プロビジョニングプロファイル]セクションで、[配布]を選択します。











3.ページで、「+」画像の付いたボタンをクリックして、プロビジョニングプロファイルを作成します。 作成するプロファイルのタイプを選択する必要があるページが表示されます。







4.この例では、それぞれ社内での配布方法に関心があります。プロファイルのタイプ「In House」を選択し、「Continue」ボタンをクリックします。







5.次のページで、手順2で以前に作成したアプリIDを選択します。







6. [続行]ボタンをクリックした後、証明書の選択に進み、1ステップで証明書を作成しました。 次に、「続行」ボタンをクリックします。







7.次のページで、プロファイルを生成する前に、フィールドにプロファイル名を入力し、データを確認する必要があります。







8.プロファイルが生成され、ダウンロードできるようになった後：







9.この例では、プロビジョニングプロファイルをデスクトップにダウンロードします。 また、サイトで作成されたプロビジョニングプロファイルを確認し、アクティブであることを確認できます。











3つのステップの結果に従って、プロビジョニングプロファイルが正常に作成されました。



4ステップ。 Windows上のVisual Studioのアプリケーションに基づいてXcodeアーカイブ（.xcarchive）を作成し、.ipaおよび.plistファイルを作成する



前の手順はApple（Mac）のコンピューターで実行された後、すぐにMacでVisual Studio 2017 for Windowsで.xcarchiveを作成する方法を説明します。



1.これを行うには、Macに接続されるVisual StudioのXamarinアプリケーションが必要です。







2.ソリューションでは、IOSプロジェクトを右クリックして選択する必要があります。 表示されるメニューで、「プロパティ」を選択します。 開いたウィンドウで、「ios bundle setting」を選択します。 次に、「バンドル設定」-「手動プロビジョニング」で選択し、「手動プロビジョニング」列の下で、前の手順で作成した証明書とプロファイルを選択します。











3. IOSプロジェクトで、Info.plistファイルを選択し、「バンドルID」が目的のアプリIDと一致することを確認する必要があります。











4.その後、Visual Studioで（管理者として）「VS 2017の開発者コマンドプロンプト」で開発者コマンドラインを開き、iosプロジェクトのあるディレクトリに移動するか、コマンドの作成時に完全なアドレスを指定します。 このコマンドは、Visual StudioからMac上に.xcarchiveアーカイブを作成します。 アーカイブ自体には、配布する必要がある.ipaおよび.plistファイルが含まれていないため、アーカイブを生成した後、それらを作成する必要があります。 アーカイブの作成方法の詳細については、 こちらをご覧ください 。



コマンド： msbuild /p:Configuration=Release /p:ServerAddress=10.211.55.2 /p:ServerUser=xamUser /p:Platform=iPhone /p:ArchiveOnBuild=true /t:"Build" MyProject.csproj















5. Macでコマンドが正常に実行された後、アーカイブが作成されているはずです。 Xcodeを実行し、その中で「Windows」を選択し、このメニューで「Organazer」を選択する必要があります。 そこで、「アーカイブ」セクションに、作成された.xcarchiveアーカイブが表示されます。















6.ここで、作成されたアーカイブに基づいて、.ipaおよび.plistファイルを作成する必要があります。



彼らの助けを借りて、たとえば社内でAppStoreをバイパスしてアプリケーションを配布できます。 次に、「アプリの配布」ボタンをクリックする必要があります。 表示されるメニューで、「エンタープライズ」を選択し、「次へ」ボタンをクリックします。







7.次に、配布可能なデバイスを選択し、「無線インストール用のマニフェストを含める」を選択して、ブラウザからアプリケーションをダウンロードできるようにする必要があります。







8.次のウィンドウで、「名前」-アプリケーションの名前を指定します。 「App URL」-.ipaファイルへのパス、「Display Image URL」-アイコン57x57へのパス、「Full Size Image URL」-アイコン512x512へのパス。



.ipaファイルと.plistファイルが配置されているサーバーは暗号化されていることが重要です。つまり、httpsは必須です。 この例では、 dropboxサービスを使用します。 ドロップボックスサービスを使用する場合、 知っておくことが重要です。パブリックリンクを介したファイルへの正しいパスは、生成されたリンクに示されているように「https://www.dropbox.com/」ではなく、「https://dl.dropboxusercontent.com/ 」







9.次のステップでは、作成された証明書とプロビジョニングプロファイルを選択する必要があります。







10.正常にアセンブルされたアプリケーションを確認したら、アプリケーションと共にフォルダーを保存する場所を選択する必要があります。その後、配布します。











11.保存後、デスクトップにフォルダーが作成されました。 4つの.plistファイルが生成され、通常は1つの.ipaが生成されるときに、下のスクリーンショットでフォルダーの内容を確認できますが、テストアプリケーションではこれは少し異なりますが、この場合は、アプリケーションの名前のみを含む名前のファイルに興味があります。 4つの.plistファイルについては、「manifest.plist」ファイルが必要です。 アプリケーションをインストールするには、インストール前のプロパティを説明するplistが必要です。 エンタープライズディストリビューションの詳細と、manifest.plistの外観は次のとおりです 。















したがって、この手順では、Visual Studio 2017で作成されたアプリケーションの.ipaファイルと.plistファイルを正常に作成し、社内配布に使用します。



5ステップ。 アプリケーション配布



前の手順では、配布用のアプリケーションを準備しました。 この手順では、リンクを含む単純なhtmlファイルを作成し、ローカルIISに配置します。これは、例を簡単にするために行われますが、リンクの場所は重要ではありません。 非サンプルリンク

アプリケーションファイルのように従業員がアクセスできるように、独自のサイトに配置することができ、独自のサーバーに配置する必要があります。 ただし、この例では、前述のように、Dropboxサービスを使用しました。



1.最初に、ファイル（アイコン、.ipaファイル、manifest.plist）をdropboxに配置し、リンクで利用可能にする必要があります。







2. htmlファイルを作成した後、次の内容： 



 <html> <body> <a href="itms-services://?action=download-manifest&url=https://www.yoursite.ru/dirname/manifest.plist"> Install iOS in-house App</a> </body> </html>


3.次に、このhtmlファイルをローカルIIS（またはサイト）にアップロードします。モバイルデバイスからこのリンクをクリックすると、アプリケーションをインストールするように求められます。 アプリケーションをインストールした後、ユーザーはデバイス設定で証明書の信頼を確認する必要があります。 設定→一般→デバイス管理→「エンタープライズ名」。ユーザーのみがアプリケーションを開くことができます。















まとめ



5つの手順の結果に従って、証明書とプロビジョニングプロファイルを作成し、Visual Studio 2017からXcodeアーカイブを収集し、配布用の.ipaおよび.plistファイルを受け取り、リンクをクリックしてデバイスにアプリケーションを正常にインストールし、AppStoreをバイパスしました。
