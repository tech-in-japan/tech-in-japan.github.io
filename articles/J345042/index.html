IP PBX Zeon。 Bitrix24統合の構成

先週、PBX について話しました 。 さまざまなCRMとの統合があると述べました。 今日、私はCRM Bitrix24についてお話したいと思います-接続方法とその仕組み。











私たちは電気通信事業者、インテグレーター、開発者です。 弊社の武器には、仮想ハイブ PBX、ZeonボックスIP-PBX 、IP-SIMカード（FMCサービス）、ロシア全土の番号などのサービスが含まれます。



友人たち、私たちはサービスがお互いに積極的に相互作用し始めて、単一のデジタル環境を形成する素晴らしい時代に生きています。 しかし、私たちは道の始まりにいます。 5年後には、APIを持たない単一のサービスはなくなると確信しています。



Bitrix24は、会社の業務を整理するための既製の便利なサービスです。 誰がBitrixを知らない-誰もがBitrixを知っている！ これらの人は途方もない仕事をしました。 彼らは素晴らしい製品を作成しました。 そして、私たちにとって重要なこと-彼らは素晴らしいAPIを作りました。 （Bitrix24 APIは大きいため、テレフォニーとCRMのコンテキストで言及します）。



だから、私たちの前にどんな機会が開かれていますか：



  • クライアントカード（着信通話と発信通話、さらには通話の転送用）

  • リダの自動作成

  • Click-to-Call（click2callとも呼ばれます）

  • Leadへの呼び出しの接続（Bitrixでは、オブジェクトに接続すると言います。これはLeadだけではないためです）

  • Bitrix24インターフェースで会話の録音とリスニングを接続する

  • 責任者に電話する



どのように機能しますか？



新しいクライアントがあなたの会社に電話をかけると、その電話はキューに入れられます（誰もが愛するのと同じアスタリスクのキューで、戦略とポーズは異なります）。 この時点で、アスタリスクはイベントを生成し（ZeonのIP-PBXの電話コアはアスタリスクです）、ハンドラーはそれをキャッチし、不足している情報を追加して、B24（Bitrix24の略）に送信します。 新しい呼び出しがB24に登録され、一意の識別子が返されます。 後続のすべてのイベントには、この識別子が付随します。



このように見えます：電話がオペレーターに届いた-カードが浮上しました。 また、コールが1人のオペレーターに着信したか、複数のオペレーターに一度に着信したかには依存しません。全員にカードがあります。



私は言わなければならない、1つの困難がありました。 カードを表示するのは物語の半分にすぎません。 通話がさらに進む場合、カードを非表示にする必要があります。 そして、ここで犬は大暴れしました。 B24には誰もがカードを隠すことができる方法はなく、松葉杖を切る必要がありました（B24の同僚がこれに注意を払い、この欠陥を修正することを願っています）。



コールがクリックされると、すべてが簡単になり、B24はZeon側のスクリプトにリクエストを送信し、送信者の信頼性を検証（キーを確認）してコールを生成します。 ハンドセットが鳴り、オペレーターがそれを拾います-クライアントとの接続があります。 この場合、着信コールと同様に、カードがポップアップします。



設定方法は？



1つの発信webhookと1つの着信webhookを作成するだけで十分です 。 IP PBX統合をアクティブにします。 他に何もする必要はありません-カードがポップアップし、リードが作成されます。



Webhookを追加する



Webhookは、httpプロトコルを使用した、イベント発生時の外部システムへの呼び出しです。







着信Webhookを追加する場合、統合機能を決定する権限を設定する必要があります。 フックに名前を付ける必要があります。 フックが1つしかない場合は、デフォルト名のままにします。







フックを保存すると、重要な情報が表示されます：



1.秘密鍵とポータルへのリンクが必要です。







それらはIP-PBXのBitrix24との統合セクションに駆動される必要があります。







2. 「Click-to-Click」のキーが必要です。これは発信フックから取得されます。 作成するとき、それをトリガーするイベントを決定する必要があります。 [アプリケーションから通話を開始する]を選択します。







もう1つ小さなニュアンスがあります。 フックをクリックしてPBXに移動するには、B24の設定でもう1つの流行を設定する必要があります。 テレフォニー設定に移動します。







「発信番号」リストで「アプリケーション」を選択します。







3.サーバー上のスクリプトアドレスは、次のように定義されています。サーバーの割り当て中に提供されます）;







4.保存後、発信フック用のキーが表示されます（Click-to-Click呼び出しの場合）。







合計



  • 任意の戦略（「すべて同時に」、「最も早い」、「ランダム」）で任意のキューに呼び出しを送信でき、戦略の選択が制限される他の交換とは異なり、参加者が電話をかけるとカードがポップアップします（この制限はアスタリスクのキューはブラックボックスであり、そこからWebhookを作成するのは難しいためです）;

  • コールは1つのキューに到着し、しばらくそこに留まり、誰も受け入れない場合は、コールを離れて別のキューに移動するか、音声メニューに進みます。

  • 「正しい」戦略「最も早い」-通話時間の長いオペレーターが電話を受信します。 そして、彼が逃した場合、呼び出しはさらに次のオペレーターに行きます。 なぜこれに焦点を当てるのか-通常のアスタリスクでは、この機能は次のように機能します：長時間通話していない参加者がいて、電話に出られなかった場合、通話は無限に戻ります。 これは通常、キューの参加者の1人が一時停止するのを忘れた場合、またはシステムを離れずに昼食のために出発した場合、KV全体のサービス拒否につながります。

  • レコードはmp3に変換され、B24への呼び出しに添付されます。

  • 不明な番号から電話がかかってきた場合、リードが作成されます。



Zeonは、「プライベートクラウド」で、クラウドサービスの価格で、または「ボックス」の形でレンタルできることに注意してください。 Zeonは、CloudとBoxed Bitrix24の両方でBitrix24と連携します。
