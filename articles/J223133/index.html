AMD A4-5000 / 11.6 "/ 1366x768 / gloss / 4Gb / 500Gb / Wi-Fi / Bluetooth / DOS /〜11-14 tr



このネットブックを見つけてすぐに魅了されたので、このレビューを書くことにしました。



ラップトップコンピューターに対する一連の必須要件は、結局のところ非常に厳しいものでした。



  1. Matrix 11.6 "-13.3"でさえ、私にとってはすでに多すぎます。
  2. 最小16Gb DDR3-3L 1600（ 12800 ）。
  3. 電源を搭載した場合の重量は最大1.5 kg。 -それは非常に快適なので、必見です。
  4. 4コアCPU 、できればAMD Kabini \ Kaveri 。

    Intel Core i5-i7が「気に入らない」わけではありません。

    しかし、私は働くのにそんなに必要はありません。

    ところで、はい-私は約15年間AMDのファンです。

    革新者としてインテルが好きです。

    しかし、競争は無料でなければなりません。

  5. レガシー BIOSモードの存在。

    マイクロソフトが「嫌い」というわけではありません。

    ところで、はい-私は今10年間Linuxを持っています。

    したがって、 UEFIセキュアブートまたは他の制限の存在は、私を非常に怒らせます。

    私はまだ仕事をする必要があり、それが遅くなる/落ちた理由が限られているなどを理解していません。

  6. 通常のグラフィックスサブシステム。職場ではフルHDモニターに接続しているため。 したがって、高解像度で何かを見たいという欲求も時々発生します。 はい、テレビの自宅にはHDMIがあります。

  7. フルキーボード。 長い間キーを見ていません。 しかし、私の古いエイサーで発明して完全な矢印とキーHome、End、PgDn、PgUpから私を救った男-彼は明らかに私を病気にしたかった。 これ以上！ なぜなら、私は2年間でこのスカラーに慣れることができなかったからです。





MSI S12からの私の期待は完全に正当化されたので、それを間違いなく共有したかったのです。突然誰かが重宝します。 もちろん、「 適切なネットブック 」の見方と、このコピーがまさに「それ」である理由についての写真、YouTubeビデオ、動揺、挑発、その他の情報も含まれます。

私も欲しいです：



  • 堅いマトリックスカバー 。 その前には、Acer Aspire One AO725がありました。 閉鎖中、特に開放中にそれを「圧迫」するのを見るのはとても怖かったです。

  • LANコネクタと3つのUSBポートの存在。 それはかつて2でした-それは十分ではありませんでした。 4つありましたが、必須ではありませんでした。この場合、ハブを購入するのはすでに簡単だったからです。 少なくとも1つにUSB 3.0を搭載することが望ましいです。 WiFiは「フィールド」環境にな​​い場合があるため、LANが必要です。

  • 迅速に分解および組み立てする機能。 これは私にとって重要です。なぜなら、私は中にあるものを見るのが好きで、好きではないものを変えるのが好きだからです。 保証のために、私は通常、購入から数時間後に「デバイスを装着」します。 したがって、デバイスは高価ではありません。それでも、突然壊れた場合は申し訳ありません。 そして、私はささいなことで動揺したくありません。 見通しが不明確なサービスセンターからのこの退屈な期待がなければ、自分で修正できる方がいいです。
  • バッテリーなしで数時間「生きる」能力。 10個は必要ありません。職場にコンセントがあり、家もあります。 通常の時間バッテリーを心配しないように。 仕事の途中でカフェに座ってコーヒーを飲み、到着時に再び荷を積まないようにブナをオフにします。 または、アダプターを接続せずにテレビで映画を見ます。 または、顧客との会議でプレゼンテーションを表示します。 3時間、もう必要ありません。 この場合、満充電の時間は放電時間の2〜3倍であることが望ましい。

  • マイク入力、時々音楽を書くこともあります。

「物理学者」はまだ全体像を示していないので、個人的な印象に移る必要があります。

<気持ち>

  • マトリックスは 、IPSではありませんが、見た目は非常にきれいですが、ピクセル化はまったくありません。 視野角も非常に均一であり、半回転で座ってもまったく変化はなく、非常に「見える」。 しかし、光沢はわずかにまぶしいですが、これは正常です。

Intel CPUを搭載したラップトップと比較すると、これらは一般に異なるレベルのソリューションであるため、何も意味がありません。 Linuxが必要な理由を理解し、Charles Petzoldの「コード」を読んでその仕組みを理解している人なら誰でも、Linux用の4つのKabiniコアは、モバイルIntel for Microsoftの2つのコアと比較して完全に受け入れられるソリューションであることは明らかです。

私は長い間走り回ったが、アルミニウム製のカバーと2番目のネジが勝った。





さまざまな推定によると、2つのメモリチャネルからのIntelの「ゲイン」は、通常の使用で10〜15％増加します。 しかし-これは理論上です。 実際には、Intelの予算互換モデルには、ここで使用されている12800ではなく、DDR3 10600があります。 ところで、AMDのメモリコントローラーは、必要に応じて、スラットの低周波数と互換性があります。 つまり、同等の製品価格でのこの「利益」は、非常に疑わしいものになります。 また、AMD A4-5000はチャネルごとに2つのスロットを使用しますが、Intelモバイルは同じ条件で2つのチャネルを使用します。 1つまたは2つのバス上の「すべて」のメモリをアドレス指定する「より良い」ものは何ですか？ 私の意見では、単純な方が良いと思います。



また、Intel互換のソリューションでは、最大8 Gbになります。 これ以上必要ないように思われることは明らかです。 必要なのは、メモリが必要なタスクです。 これは、たとえば、Node.JSの下のIn Memory DBです。 SSDがあることに気付くかもしれませんが、より多くのSWAPを実行できます。 しかし、信じられないかもしれませんが、そうではありません。



個人的には、コストで比較できるインテルの「予算」ソリューションは「憂鬱」に見えます。 ここのグラフィックは2倍以上です。 一般に、このラップトップにWindowsを置くことは間違いだと思います。 また、タッチスクリーンでの修正は、たとえばSoft for Touchを開発している場合など、理解できるソリューションです。 ちなみにタッチは10本の指があります。



2つ目のネジを取り付ける機能も、私にとって大きなプラスです。 まだSSDを持っていない人のために、予算のmSATAを購入して工場のネジの横に挿入することができます。 このマシンのSSDは2倍に増加します。ソフトウェアの開発などを行う場合は物理的に必要です。 重い計算。 マット。 もちろん、ビデオコンプの編集などのタスクは実行されませんが、NetBeansはこのあたりでかなり受け入れられています。



BIOSにはPCIバスマルチプライヤもあります。個別に説明します...実際、市場のコメントが正しく述べているように、特にLinuxを使用している場合、WiFiコントローラーはここで悪いです。 RTL8723aeチップ上に構築されています。 Linuxでは、それは「ビダ」です。なぜなら 開始されません 。 Windowsでは、すべてが「Right Ok」になります。 しかし、Debian \ Ubuntu \ Mintの下では、このカードはすべての最新カーネルで問題なく検出および検出されますが、「ポイントブランク」自体は単一のネットワークを認識せず、「隠された」ネットワークに接続しません。 システムで実行されているドライバーを「無効にする」ことも、さまざまなフォーラムにあるソースコードからドライバーを「取得する」こともできなかったため、数時間で別のWiFiモジュールをインストールすることにしました。



最初はWiFiホイッスルTP-Link TL-WIN725Nを購入しました。 ところで、素晴らしいデバイス。 コンパクトで、802.11n上のネットワークは、コンポーネントの微視的なサイズにも関わらず、述べたように保持されます（分解して見ました）。 その中のチップもRealTekですが、ドライバーが集まってgoogleからこの2番目のリンクにアクセスしました 。 欠点は、カーネルを更新するときに、再度&&をインストールする必要があるということです。これは正常ですが、私には向いていません。



なぜなら AMD C-60ストーンで既に古いAcer Aspire One AO725を交換するためにS12を使用し、LinuxモジュールのほとんどのAtherosチップのドライバーが「火花」を耕すので、そこからWiFiモジュールを抜くだけにしました。 しかし、それは始まりませんでした...少し考えてから、BIOSにアクセスし、バスの乗数を32クロックサイクルから128クロックサイクルに変更しました。 出来上がり:)



WiFiを扱うことが可能であることを確認しましたが、それでも古いラップトップを離れずに、単純な検索とここからのデータに基づいて、WiFiをINTEL 2230BNHMWモジュールに置き換えることが決定されました。



ところで、S12は非常に簡単に理解します。 バッテリーと交換用カバーHDDおよびSO-DIMMを取り外した後、すべてのネジを「円形」に外し、底部カバーを取り外します。 歯車は厳密に下にあり、側面などはありません。便利です。 ラッチはきつくありませんが、ギターを「こじ開ける」には非常に役立ちます。 注意してください サウンド出力は底部カバーを取り外すのに少し「邪魔」になりますので、わずかに壁に押し込むだけです。 ループのうち、ボードではなく、取り外し可能な底部に接続されたUSBに接続するループのみを「傾ける」必要があります。 はい、CPUクーラーとタービン用のヒートパイプがあります:)



店で分解しました 本当に5分です。 すぐに、32の乗数で立ち上がりました。WiFiの発明者による素晴らしいWiFi 300NとBlueTooth 4.0。 感覚、それはまるでウサイン・ボルトを百メートルで渡ったかのようでした。 IntelのWiFiと、AMDの技術が結集され、技術が到達しました:)その後の「ホーム」テストでも問題は明らかになりませんでした。



すぐにサーマルセメントを交換しましたが、勝つことはできませんでした。明らかに、チューブの熱容量とタービンの性能はまだ残っています。 ファンは、つまり静かです。 彼はまったく聞いていないというわけではありませんが、彼は気にしません。 Intelベースの人々の基準では、もちろん石はぬるいでしょう。 負荷54-57がなく、負荷がファンをオンにするだけで、60を超えない場合。しかし、これはAMDにとっては非常に正常なことです。 Linuxでは、「高温」温度は70度と見なされ、シャットダウンは100度になると言われます。AMDの場合、これは問題ありません。恐れることはありません。 私の古いC-60は69-85の範囲で動作し、93でオフになりました（Acerがクーラーを大幅に節約したため、デフォルトのウィンドウでは数回ありました）。 ここでは、クーラーはかなり、すべてがOKです。 さらに、 仕様lm-sensorsは、最新のカーネルではすべてが問題ないことを示唆しています。 だから、おそらくミント17（6月、ある種）を待つか、Ubuntu 14.10を試すだけです。



はい、4ギガバイトのメモリカードを追加し、 SSDを挿入しました。 当然、なぜなら 私にはLinuxがあり、「再インストール」の話はありませんでした。すべてが以前のように機能し、1桁速くなりました。



主観的な感覚によると、私はMacBook Air 11をほぼ手に入れました。重量は1.1でなく1.3 kgであり、それほど「スタイリッシュ」ではありません。しかし、その後は通常のキーボードを使用し、コンピューターを5分で分解および組み立てできます。また、Airの2倍のメモリを搭載できますが、Intelはありません。



Airよりも良くするために残っています！これを行うには、1366x768のストックマトリックスをフルHDに置き換えます。マトリックスは、ケースが分解されるよりもさらに速く削除されます。カメラの領域にあるメディエーター装飾パネルを手に取り、輪になってください。心配する必要はありません。ラッチはさらに簡単です。マトリックスは4つの歯車の上にあり、ループはそれだけにつながります。モデルのステッカー：M116XW05-V1FH、SN：FE104B116CZ01407。重要なE204356がまだあります。モデルで検索すると、韓国語を学ぶように指示されます。 Chromeが老婦人を翻訳できるのは良いことです。正気なものが見つからなかったので、互換性のあるフルHDを探し始めました。突く方法によって、私はpanelookウェブサイトで116XW05と互換性があるものを探す必要があることに気付きました。モデルn116hse-ea1およびTOP-FHD-116L-FLR-Sを見つけました。これらのモデルがN-skにあるのは奇跡です。私は店に行きましたが、ループの接触パッドが合いませんでした。ここでも、30ピンLVDSですが、すでに数ミリメートルのトラックがあります。どうしたらいいかわかりません。見つけたら登録を解除します。
