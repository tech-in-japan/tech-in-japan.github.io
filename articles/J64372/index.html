SharePoint Server 2010の機能とラスベガスカンファレンスの概要

今日、SharePoint Server 2010のいくつかの新機能の説明は、幅広いユーザーが利用できるようになりました。



開発者が追求する目標は、新しいプラットフォームで、専門システムの展開にリソースを費やすことなく、ビジネスに必要なソリューションを迅速に作成できることです。 設計時には、現代のユーザーは電話、ブラウザ、および通常のアプリケーションの両方を積極的に使用しているため、必要な情報にどこからでもアクセスできる必要があるという事実から始めました。



デモを見て意見を表明することをお勧めします。製品は差し迫った問題の解決に役立ちますか？







機能のハイライトを使用して、ビデオを最も興味深い機能の説明にすばやく巻き戻すことができます。



もちろん、すべての機会は個別の記事に値するものであり、それらは後で表示されると確信しており、最も興味深いものだけを見ていきます。



新しいインターフェース



メニューの機能が多すぎてSharePointに到達したため、Office 2007クライアントアプリケーションで始まったインターフェイスの変化の波。 ユーザーはOfficeラインとSharePointアプリケーションを同じように認識し、追加のトレーニングなしで、必要なブックマークを開くことで必要な機能を直感的に見つけます。



Officeアプリケーションの概念は曖昧になりつつありますが、結局、WordやExcelなどのオフィスアプリケーションはWebバージョンを取得し 、ユーザーはブラウザーを使用してのみSharePointサイトでドキュメントを直接編集できるようになります。



簡素化されたコンテンツ編集



インターフェイスでのそのような単純化されたアプローチの例は、コンテンツを編集することです。これは、編集タブに切り替えるだけで、同じページで直接実行できます。



Internet Explorerだけでなく



驚くべきことに、これらはすべてInternet Explorerだけでなく、SafariとFireFoxでも機能します。 スクリーンショットは、新しいインターフェイス、編集機能、FireFoxを示しています。これらはすべて機能します。







Silverlight



SharePointのWebアプリケーションの制限が、Silverlight機能で正常に満たされることは長い間明らかでした。 このチュートリアルの10番目のモジュールの内容をチェックして、このアプローチがどれほど広く使用できるかを確認してください。 しかし、そのようなアプリケーションの構築はあまり便利ではありませんでした。 これで、Silverlight Webパーツが表示され、Silverlightアプリケーションを埋め込むために何もプログラムする必要がなくなりました。







Web上のVisio



Visioスキームを動的にできることを知っていますか？ たとえば、間取り図に従業員の職場の場所を表示する場合。 さて、またはワークフローを視覚化するために、たとえばここで説明します 。 以前はVisioでのみこれらのライブ図を表示することが可能でしたが、SharePoint Webサイトを開いて特定のプロセスのステータスを視覚的に確認できるようになりました。







SharePointワークスペース



これで、サイトリストをSharePoint Workspace（新しい名前はMicrosoft Office Groove）にリンクするだけで、Webアプリケーションはオフラインスクリプトのサポートを開始します。 電車での移動中または顧客との会議中にシステムの情報を変更し、オンラインに移行して、すべてがSharePointサイトと同期されました。 基本的に、SharePoint for SharePoint for SharePointはOutlook for Exchangeと同じです。

SharePoint Designerは大きく成長しました。 これにより、より複雑なワークフローを作成したり、外部データを操作することもできます。

ビジネス接続サービス（以前のビジネスデータカタログ）



これは本当にクールなものです。 BDCを使用した以前の外部データをSharePointに発行して読み取り、リスト、プロファイルで使用、またはそれらを検索できる場合（ 知識のない人の説明 ）、データも変更できるようになりました！ 実際、SharePointは、データベースおよび外部システムへの本格的かつ柔軟なインターフェイスになりつつあります。



また、SharePointプラットフォームはスケーラブルであり、拡張および統合のためのプログラムインターフェイスが既にあり、認証およびアクセス制御モデルが実装されているため、ページレンダリングおよび管理インターフェイスがあります-これを自分で行う必要はありません-Webアプリケーションは本当に強力であり、実行されますより簡単です。



エンティティの表示と編集に十分な標準機能がありませんか？ SharePoint DesignerまたはVisual Studio 2010で自分を拡張します。







LINQとクライアントオブジェクトモデル

次のステップ？



  1. システム要件を考慮し、64ビットアーキテクチャへのアップグレードを計画します。 SharePoint 2010はその上でのみ動作します。 これは、SQLサーバーにも適用されます。 64ビットSQL Server 2008または64ビットSQL Server 2005が必要になります。
  2. Windows SharePoint Service 3.0 用のService Pack 2とSharePoint Server 2007 用のService Pack 2をインストールし、 Service Pack 2に含まれているアップグレードチェッカーツールを実行します 。
  3. 組織にInternet Explorer 6が残っている場合、おそらく正しく動作しないため、ブラウザーをInternet Explorer 7、Internet Explorer 8にアップグレードするか、FireFox 3.xを使用する必要があります。
  4. 管理者向けのベストプラクティスと開発者向けのコーディングのベストプラクティスに従うことは理にかなっています。
  5. 開発ソリューションにSharePointソリューション形式 （WSPファイル）をまだ使用していない場合は、次のように開始します。 Sharepoint 2010では、この形式が標準になります。
  6. Visual Studio 2008を使用してSharePoint Server 2007を開発できる場合、SharePoint 2010にはVisual Studio 2010が必要です。したがって、スタジオのベータ版に注意してください。
  7. 言語統合クエリ（LINQ） 、 エンティティモデル、およびSilverlightに関するTechDaysレポートへのリンクを提供します（まだ知らない場合）。 SharePointの開発に対する新しいアプローチをすばやく学習したい場合は、これらのテクノロジーが役立ちます。
  8. さて、次のベータテストのニュースを待つだけでなく、最も重要なステップは会議に参加することです
