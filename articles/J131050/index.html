ヘッツナー。 RZ12およびRZ15で失敗する

UPD

2012年2月14日のすべての愛好家の日には、サポートするネットワーク機器に障害が発生しました。 現在、ロボットパネルへのアクセスは無効になっていますが、RZ14の20台のサーバーのうち1台でまだ停止しています





昨日、12-47 MSKで、RZ12およびRZ15ヘッツナーゾーンで電源障害が発生し、その結果、これらのゾーンのすべてのサーバーが再起動されました。 これまでに多数の申請者がいたためにLARAにアクセスすることは不可能であり、これらのセンターのサポートスタッフは最大化されました。

幸運を祈ります。そして、あなたのサービスがこの規模の大失敗の後に自分自身で開始することを確認してください。

LinuxおよびWindows Insiderのスタートアップをセットアップする手順



辛抱強いRZ12からの写真





窓

1） technet.microsoft.com/en-us/sysinternals/bb963905.aspxをダウンロードします

2）ユーザーに代わって自動ログインを実行して許可します。

3）必要なアプリケーションを標準のスタートアップに追加します（スタート->スタートアップ）。



Linux

a）不正確だが一般的に使用される方法：

/etc/rc.local

このファイルのすべてのコマンドは、システムの再起動後に起動されます。



b）メソッドはきちんとしている：

サービスが正しくインストールされている場合、chkconfig --add $ service_nameは、ブートレベル/etc/rcX.dで実行するエイリアスを追加します

当然、これの前にchkconfig自体をインストールする必要があります。 Debianファミリでは、RHELとは異なり、apt-get install chkconfigをインストールする必要があります。
