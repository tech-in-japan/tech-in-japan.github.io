高DPIのモニター用のLinux GUI構成

最近では、特にラップトップで、150 dbi以上の高解像度のモニターがたくさんあります。 そしてすべてがひどく小さく見えますが。

これは、モニターの解像度が96..100 dpiであるとシステムが判断するためです。

フォントサイズを大きくするだけでこれを解決するのは間違っています。 他のすべてのインターフェイス要素は小さいです。 グラフィカルシェル自体がこれを行う必要があります。



この問題を解決するために、私は非常に長い時間インターネットを探し回って、小さなメモを収集しなければなりませんでした。 一緒に見ることができる場所はありませんでした。



以下に何かを修正する場所のリストを示します。モニターのすべてのパラメーターを示します-13.3 "2560x1600で、これは226 dpiを提供します。FujistuU904ラップトップでは262 dpiしか表示されません。



モニターパラメーター：xorg.confのモニターパラメーターに以下を追加します。

オプション「DPI」「226 x 226」

または

DisplaySize 286 179＃は同じですが、画面サイズはミリメートルで設定されます

xorg.confがなく、モニターが1つしかない場合は、これをデフォルトのモニターに設定できます。 これを行うために、次の内容のxorg.conf.d / 90-monitor.confファイルがX config configディレクトリに作成されます。 

セクション「モニター」
    識別子「<デフォルトモニター>」
    オプション「DPI」「226 x 226」
終了セクション




X起動キーで、-dpi 226を追加します。kdmでは、これはServerArgsLocalパラメーターの行です



/ etc / X11 / Xresourseまたは〜/ .Xresourseのフォントの場合：

Xft.dpi 96は226に置き換えられます



クレームを再開した後、これが有効になったことを確認できます。

-Xログでは、/ var / log / Xorg.0.logのような行である必要があります

インテル（0）：クロック：268.5 MHz画像サイズ：286 x 179 mm

インテル（0）：（226、226）に設定されたDPI

-xdpyinfo | grep -B2解像度

画面＃0：

寸法：2560x1600ピクセル（287x179ミリメートル）

解像度：227 x 227ドット/インチ

-xrdb -query | grep dpi

Xft.dpi：226



しかし、Xパラメーターを考慮しないプログラムやGUIがたくさんあります。 したがって、継続します。



KDEの場合：システム設定=>基本的な外観設定=>フォント=>別のDPIを使用：226。

ただし、アイコンは個別に増やす必要があります。システム設定=>基本的な外観設定=>アイコン、詳細タブ。



Gnomeの場合：gconfで、/ desktop / gnome / font_rendering / dpiで書き込みます（Gnomeを使用していないため、ここで何か他に必要なのかわかりません）。



Firefoxの場合：約：config：layout.css.PixelsPerPx = 2.26、layout.css.dpi = 226。

しかし、多くの場合、ブラウザは完全に準拠する必要がないため、画面に収まるようにlayout.css.PixelsPerPx = 2を注文しました。 firefoxは100 dpiで最も正確に画像に一致しますが、ビットマップ画像の奇妙なスケーリングを使用します。 まず、内部で、100 dpiにあるはずのイメージを受け取り、次に2倍に拡大します。 このため、写真は高解像度であってもせっけんです。



Operaの場合：200％のマスターベーションを設定するだけで十分です-オペラはこれを非常に正しく行います。



Chromeの場合：//フラグ/「高解像度モード」パラメーターがありますが、Linuxの場合は使用できません。 したがって、200％と「非常に大きな」フォントサイズを配置するだけです。 残念ながら、これはフラッシュには影響しません。

しかし、120％に設定し、それを使用して高詳細画像のマップ（maps.yandex.ru、maps.google.ru）を表示します。



Konquerorはまだ高dpiを理解しておらず、同じエンジンを使用するakregatorは追加の指示なしにすべてを正しく表示しますが、正しくスケーリングしません。



システムがdpiを正しく認識していることを確認するには、OpenOfficeなどのオフィスプログラムでA4ページを開き、100％ズームで用紙を添付することをお勧めします。 すべてが正常に機能する場合、サイズは一致します。

また、OpenOfficeの設定では、アイコンのサイズを「大」に設定する価値があります。



すべての設定後、ハードコードされたビットマップ画像と拡大フォントの競合がありました。

これはkdmとOperaで見ました。作業はできますが、見た目はあまりよくありません。文字の端が碑文に割り当てられたフィールドに出てきます。 そのため、Xresourseで226を175に置き換えました（一般に、メニューやその他のラベルのdpiサイズを読み取るオペラでは、htmlページの表示に同じdpiを使用しないのは奇妙です）。



PSもともとここでは自分だけを対象としていましたが、今では多くの人がそのような質問をしていることがわかりました。 建設的な追加があれば嬉しいです。
