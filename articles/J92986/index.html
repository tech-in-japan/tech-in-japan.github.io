例としてAcer Timeline 3810TGを使用してUbuntuで個別のグラフィックカードを無効にする

すべての人に良い一日を！

最近Habrを訪れて、Acer Timelineラップトップのレビューを2つも見つけました。 私自身は、このラインの13.3インチモデルの自慢の所有者であり、このトピックの開発を促しました。事実、ラップトップを購入してから2日目にインストールしたUbuntuを使用する方が便利だということです。エイサーは、「箱から出してすぐに」絶対にすべてが機能するものを熱望しています。この場合、すべてがバラ色ではありませんでした。



問題の本質



多くの点でこの作業時間はWindows 7ソフトウェアによって達成されるため、宣言された8時間の作業を忘れなければならなかったとすぐに言わなければなりません。 しかし、ここで私はケースの温度に警戒していました。 Ubuntuでは、ラップトップはWindowsの2倍、または3倍もウォームアップされました。 理由を調べてみたところ、Ubuntuはビデオカードを「外出先」で切り替えたり、ビデオカードの1つを無効にしたりすることはできません。 つまり これらのラップトップに組み込まれた両方のビデオカード（および2つのビデオカードを搭載した多くのラップトップ）は同時にオンになり、膨大なエネルギーを無駄に消費します。 私はubuntuの下でゲームをするつもりはなかったので、システムからATIから不要なビデオカードを削除することにしました。 関連するフォーラムを読んだ後、Lenovoラップトップ用に書かれた「lenovo_acpi」というスクリプトに似た問題が見つかりました。 このスクリプトは、変更を1つも加えずに私の問題を解決しました。つまり、Timeline-sの他の所有者にとって役立つことがあります。



インストールに移りましょう



まず、 このリンクからrpmパッケージをダウンロードする必要があります。 Ubuntuで使用するには、dkmsとalienをインストールする必要があります。 インストール：



sudo apt-get install dkms alien







次に、以前にダウンロードしたパッケージがあるディレクトリで、次のコマンドを実行します。



sudo alien --script lenovo_acpi-0.1.0-1DKMS.noarch.rpm

sudo dpkg -i lenovo-acpi_0.1.0-2_all.deb







次のようにモジュールを確認できます。



sudo modprobe lenovo_acpi







システムが起動するたびにモジュールをロードする場合は、/ etc / modulesの新しい行から引用符なしで 'lenovo_acpi'を追加します。

また、BIOS設定で「切り替え可能な」グラフィックモードを設定する必要があることにも言及する価値があります。また、この方法はUbuntu 10.04でのみ個人的にテストされたことにも注意してください。



あとがき



このモジュールをインストールした後、powertopはより快適な数字を表示し始めました-省電力モードでwi-fiをオンにした状態で約5時間作業しました。 バッテリーの電力を節約する他の方法があります。 これは 、ubuntuforumのこのスレッドで読むことができます。
