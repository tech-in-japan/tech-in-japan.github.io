ONTAP 9.2：新機能

この記事では、 バージョン9.0のリリースから9か月後にリリースされたONTAP 9.2ファームウェアバージョンに登場したイノベーションを確認します。 いつものように、NetAppはデータストレージシステムの多くの新機能に満足しています。それらを分析しましょう。





ONTAP 9.1以降のリリースは、6か月ごとに定期的にリリースされるようになりました。 以前のリリースは、信じられないほど多数の新機能に対応するために遅延されていましたが、現在は各リリースでそれらの数が少なくなりますが、より定期的にリリースされ、他の機能の出力を遅延させません。 長期および短期のサポートがあるリリースがあります。 正しく理解してください。両方のリリースは完全に本番環境に対応しており、一般に利用可能であり、テスト、デバッグ、および機能します。 ONTAPリリースのサポート期間を延長するために、いくつかの新しいシステムで7年以上のサポートを購入できるようになったため、Ubuntuなどで発生するように、長いサポートのあるリリースと短いサポートのあるリリースをリリースするために妥協が行われました。



簡素化に向けた継続的な動き





NetAppは、サンドイッチを作るトースターと同じくらい簡単にシステムをリリースし始めていました。 その後、サンドイッチは成長し、オーブンだけに収まり始めました。現在、ONTAPは本格的なキッチン全体であり、どこかに、時には隠れて見つけにくい多くの機能が組み込まれています。 かつてONTAPは非常にシンプルなOSでしたが、現在では非常に洗練されており、非常に多くの機能を備えているため、NetAppは単純化に向けてこの絶え間ない動きを続け、クールで洗練された使いやすいものにします。



  • 互換性のあるノードを追加することによるクラスター拡張をGUIから実行できるようになり、そこから新しいノードの自動検出が利用可能になりました
  • 自動スイッチレスクラスター検出-クラスターが2つ以上のノードで構成されているかどうかを自動的に判断します。手動で切り替える必要はありません。
  • detect-switchless-clusterオプションは、9.2システムに新規および更新されたシステムでデフォルトで有効になっています
  • OnCommand System Managerガイド付きセットアップ（GUI）-最初に、ストレージコンソールに接続し、ノードIPアドレスを設定する必要があります。さらに、初期インターフェイス構成をグラフィカルインターフェイスから実行できるようになりました
  • 更新されたダッシュボード（ONTAP 9.1以降）
  • 初期セットアップ用のクラスター構成のエクスポートとインポート-通常のテキストエディターまたはExcelで編集できるcsvテンプレートからクラスター設定をアンロードおよびロードできます。


高度なQoS



バランス配置（QoS）-LUNを作成するとき、システムはそれを配置する場所を提供します。



QoS最小、この機能はスループットフロア（TP）とも呼ばれ、以前に利用可能であったQoS最大（スループット上限）機能と連携して機能します。 スループットフロアは、IOPSパフォーマンスが以前に設定した値を下回らないようにします。 この機能がiSCSI / FCPプロトコルで機能するには、クラスターに少なくとも1つのAFFノードが必要です。



最小QoS​​は、グループパフォーマンスポリシーで設定されているバージョン9.2で登場しました。 現在の負荷でシステムがどれだけ配信できるかは、ストレージシステムに組み込まれているヘッドルームモニタリングから動的に取得され、最適なものから選択されます。 最適なのは、IOPS / Latensiに基づいたシステム負荷グラフの部分であり、線形に実行されます。 グラフの線形セクションが始まった後、負荷が増加すると、IOPSの数が増えると、IOPSが線形に増加し、ラテンシが不均衡に幾何学的に増加します。 したがって、スループットフロアは、必要に応じて月に最小値が与えられ、まだ最小値に達していない月のパフォーマンスの一部を取得します。 言い換えれば、他のすべての衛星について、「一時的または動的な最大値」が設定されます。これは、スループットの天井よりも小さく、スループットの床よりも大きく、影響を受ける最小値に設定された最小値が与えられます。



クロスボリュームの献身



インライン集約-SSDでの広範囲の重複排除（またはクロスボリューム重複排除）。 この機能は、ONTAP 9.2のすべてのSSDユニットでデフォルトで有効になっています。 はい、もちろん「重複排除のグローバルクラスター」が必要でしたが、完全性に制限はなく、現在のところ集約レベルでのみ機能しますが、それでも現在のONTAP SSDは非常に大きいため、グローバル重複排除の競合他社のシステム全体よりも大きくなっています。 したがって、重複除去アグリゲートは、効率の点で他のオールフラッシュアレイ（AFA）システムと競合するのに十分です。 オールフラッシュFAS（AFF）システムの最大集約サイズが800TiB（以前は400）に増加



バージョン9.2では、コマンドラインで新しいコマンド「storage aggregate show-efficiency」を使用できます。これにより、クローン、圧縮、重複除去など、個々のテクノロジーに対する節約効果を表示できます。



階層化またはFabricPool



FabricPoolは、DataFabric戦略の一部として実装されるテクノロジーです。これにより、すべてのNetApp製品の統合が拡大し、データモビリティの可能性が拡大します。 NetAppは、新しい機能を作成するとき、他の機能を確認せず、「チェックのために」機能を追加しようとしません。 代わりに、非常によく考えられ、消費者から需要がある技術がリリースされます。 これは、新しい階層化機能FabricPoolにも適用されます。 SSDに対する需要の高まりに伴い、インストールの数とこのテクノロジーへの関心が高まっているため、NetAppはFabricPool機能をリリースしました。 FabricPoolは、コールドデータを集約SSD（AFF、FAS、ONTAPクラウドシステム）からコールドオブジェクトレベル（Amazon S3クラウドまたはNetApp StorageGRIDオブジェクトストレージ ）に移行するためのテクノロジーです。



Object StorageでのFabricPoolデータ階層化の実装は、2つのステージで構成されます。 9.2では、スナップショットのみが使用可能になります。 そして次の段階は、ボリュームのアクティブファイルシステムからコールドレベルにコールドデータをシフトする機能です。 FlexProup、MetroCluster、ONTAP Select、SnapLockなどはFabricPoolではサポートされていません。



Amzon S3を搭載したFAS / AFFシステムのテラバイトライセンス。 ライセンスを追加すると、すぐに10TBが追加されます。 その後、テラバイトを追加できます。

FabricPool機能がFAS / AFFストレージシステムで動作するには、Amazon S3を使用する場合にのみライセンスが必要です。 NetApp StorageGRIDの場合、FabricPoolは動作するためのライセンスを必要としません。 Amazonに存在するONTAPクラウドシステムの場合、FabricPoolを有効にするためのONTAPからのテラバイトライセンスもありません。AWSS3の実際のコールドコンシューマ領域のみをAmazonに直接支払います 。







ボリューム暗号化をGUIから有効化できるようになりました。 NetApp Volume Encryptionを使用して、SnapLockボリュームを暗号化できるようになりました。 FlexGroupは、NVE暗号化でも機能するようになりました（FGの作成時のみ）。



オンライン証明書ステータスプロトコル（OCSP）-この機能は、TLS over LDAPなどのTLS暗号化を使用するアプリケーションに使用されます。 9.2では、パスワードの推測を防ぐために、失敗したSSH接続の最大試行回数を設定できるようになりました。 iSCSIエンドポイントの分離-ストレージへのログインを許可するIPアドレスの範囲を指定できます。



ADP for AFF：パーティション（ RD2 ）上に構築された集合体を、最初のシェルフ（またはヘッド）だけでなく、次のシェルフ（最大48ディスク）のディスクにまで拡張する機能。



ADP ルートデータパーティショニングは 、ハイブリッドシステムFAS80XX / FAS8200 / FAS9000でもサポートされるようになりました。これにより、ルートユニット用の専用ディスクを省くことができるため、ディスクをもう少し合理的に使用できます。



パーティションを操作するためのブートメニューに新しい項目（オプション9）が表示されました（ONTAP 9で表示されます）。



  • ディスクのパーティション化とオーバーパーティティングを削除します。
  • 構成をクリアし、パーティションを使用してすべてのディスクを再初期化します。
  • 構成をクリアし、パーティション化されていないディスク全体ですべてのディスクを再初期化します。


OnCommand Unified Manager 7.2には、すぐにOnCommand Performance Managerが含まれるようになりました。
