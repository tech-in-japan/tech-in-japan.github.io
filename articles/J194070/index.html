ZeroNights 2013での仮想化の過去、現在、未来









友人、今年の会議の主要なトピックの1つは仮想化です。 最終的に基調講演者の一人を決定しました。 これは、長年このトピックに取り組んできたラファル・ヴォイチュクです。



知らない人のために：Rafal Wojtczuk-メガクールスペシャル！ 彼はInvisible Things Labsの元従業員であり、Joanna Rutkowskaで長い間働いており、「Xen Hypervisor Subversionsの検出と防止」、「Xen 0wning Trilogy：code and demos」、「Attacking Intel Trusted Execution Technology」などの研究に参加しました、「時間の節約で9を節約：複数のオペレーティングシステムの脆弱性の事例」など。 最近、彼は高度なIntelセキュリティ技術、特にTXTとVTdを研究しました。 また、低レベルのパッケージ再アセンブリライブラリであるlibnidsの作成者でもあります。 一般的に、これはサンドボックスと仮想化システムがどのように機能し、どのように機能するかを直接知っている人です。 彼はクラウドに関するマーケティングのでたらめを駆り立てるつもりはありませんが、仮想化が実際にどのように機能するかを説明します。



また、産業用制御システムの安全性についても詳しく説明します。 特に、ZeroNights 2013では、デジタルセキュリティリサーチセンターが、ICSの安全性に関する2つの最新の研究「HART（in）セキュリティ」と「AVRおよびMSPマイクロチップの動作」の結果を初めて紹介します。



したがって、 メインプログラムでのレポート：



1.「仮想化サンドボックスによるワークステーションの保護：過去、現在、未来」、スピーカーRafal Wojtczuk（ポーランド）

レポートの説明
ブラウザーなどの現代の大規模なアプリケーションは非常に複雑であるため、脆弱性なしでリリースすることを望むものはありません。 そのため、多くの開発者は代替アプローチを使用します-侵害されたアプリケーションを残りのオペレーティングシステムから隔離するように設計されたコンテナで実行します。 このようなコンテナは、アプリケーションレベルでのサンドボックスの使用やOS全体の仮想化を通じて作成できます。 問題は、この分離方法がどれほど安全で信頼できるかです。

このレポートでは、両方の分離方法の長所と短所をまとめて比較します。 具体例について説明します：Sandboxie、Google Chrome、Qubes OS、Bromium v​​Sentry。 これらの決定の進化を見て、将来それらを待っているものを予測しようとします。


2.「JSMVCOMFG-JavaScript MVCおよびテンプレートフレームワークの詳細」、スピーカーMario Heiderich（ドイツ）

レポートの説明
通常のクラシックWebアプリケーションを開発できます。 サーバー、データベース、少しのHTML、少しのJavaScriptを知っています。 たぶん、あなたの祖母がこれらを設計しました。 また、美しくてヒップで、軽くてモダンで、柔軟でスケーラブルなクライアントWebアプリケーションを開発できます。 時々それらの後ろにサーバー、時にはデータベースがありますが、すべての新しい機能のほとんどが耕しています-「モデル-プレゼンテーション-コントローラー」の原則に基づいて構築されたJavaScriptフレームワーク。

Angular、Ember、CanJS、Knockout、Handlebars、Underscore ...これらは有名なファイターの名前ではなく、ブラウザーで多くのことを直接処理する魔法の能力により、より生産的で生産的な現代のJSフレームワークの名前です。 より多くの人々がファッショナブルであり、同僚に従って、これらのフレームワークをインストールして大成功を収めています。 警備員の厳しい表情でこの牧歌をキャストする時ですよね？

このレポートでは、これらすべてのフレームワークがどのように機能するか、コアの安全性、このフル機能の宝庫から注がれるセキュリティ問題を学習します。 著者はDOMを十分に研究して、数十層の抽象化でDOMを充実させましたか？ あるいは、新しいインジェクションの脆弱性と最悪のプログラミング慣行で溢れているJavaScriptの地獄への扉を開いたのでしょうか？ このレポートの後、理解できます。 あなたはすべてを理解します...

4.「タイミング分析」は、Roman Korkikyan（スイス）が実施します。

レポートの説明
このワークショップでは、実行時間を測定することにより、DESおよびAESソフトウェア実装の秘密鍵を計算します。 この暗号解析手法は、タイミング解析と呼ばれます。 これは、サイドチャネル攻撃を分析する最も簡単な方法の1つです。 この方法を所有すると、暗号化アルゴリズムの攻撃のより複雑な方法を理解するのに役立ちます。これには、電磁放射分析（電磁分析）、光子放射の分析（微分フォトニック放射分析）、および消費電力の分析（微分電力分析）が含まれます。

ワークショップの主要トピック：

-暗号の実装のプログラム実行時間は何に依存しますか？

-どの場合、暗号の時間を使用してキーを計算できますか？

-キー計算は実際にどのように行われますか？

レポートの説明
Blackhat USA 2013カンファレンス（https://media.blackhat.com/us-13/US-13-Forristal-Android-One-Root-to-Own-Them-All-Slides.pdf）のJeff Forristalは、デジタルをバイパスする方法を紹介しましたAndroidアプリケーションの署名と、アプリケーションのファイルの一部を独自のものに置き換える。 この脆弱性は、アプリケーションデータの取得と変更、特に保存されたログインとパスワードの読み取り（rootなしで、したがってスマートフォンの保証を失うことなく）、ゲームの保存の変更（rootなしのゲームでの不正行為）、システム権限の取得（ifシステムアプリケーションの置き換え）、およびそれらの助けを借りて、ルート権限（「ro.kernel.qemu = 1 \ r \ n」行をファイル/data/local.propに追加することにより）。 ただし、提案された操作方法にはいくつかの制限があり、場合によっては重大な問題を引き起こす可能性があります。 この脆弱性を悪用する新しい方法が発見され、デジタル署名プログラムの検証をバイパスする機会が増えました。 さらに、Bluebox Security Scannerは、この脆弱性を悪用するapkファイルを検出しません。 新しい操作方法がGoogleセキュリティチームに報告されました。 バグ8219321のパッチもこの脆弱性を解決することが判明しましたが、Google Playマーケットのマルウェアフィルターは更新されました。 1日間の脆弱性を悪用する新しい方法が提示されます。これまでにレポートは提出されていません。
