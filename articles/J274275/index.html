DevExpress .NET、HTML5 / JSおよびVCL v2015.2コンポーネントのリリース

みなさんこんにちは！



2015年は完成に近づいており、ロシアのすべての開発者（そして、全世界、本当に！）をお祈りします-来年はきっとあったでしょう！



しかし、ここでは、たとえば、今年、.NET、HTML / JavaScript、およびVCLのコンポーネントの新しいバージョンをリリースすることができました。作業中に製品を使用して開発を監視する場合は、catへようこそ。







テキストよりもビデオを好む人のために、すべての主要な新製品の概要を記載した短いビデオを用意しました。







バージョン15.2の新機能：



WinFormsおよびWPFのダイアグラムコンポーネント



WinFormsおよびWPFの新しい図コンポーネントを使用すると、階層図とフローチャートを使用して複雑な情報を視覚的に表示できます。 これらのコンポーネントの主な機能：

  • 基本的な要素とフローチャートおよびSDLダイアグラムの要素の両方を含む、110を超えるさまざまな形状。
  • Visioにあるものと同様のチャートテーマ：Office、Linear、Integral、Daybreak、Parallel、Sequence、Lines。 また、独自のテーマを作成する機能もあります。
  • コネクタ（直線、曲線、および長方形）
  • ジャンクション矢印の15のプリセットスタイル
  • A *アルゴリズムを使用したコネクタの自動パス構築
  • 自動配置図形
  • エンドユーザーチャートエディター
  • ダイアグラムを保存/ロードする機能








WinForms-グリッドコンポーネントに基づいたレポートの生成



このリリースには、WinFormsコンポーネントGridControlに基づいてレポートを作成する新しい機能があります。 これで、わずか数行のコードで、Grid Controlに基づいて作成されたReport Designerを呼び出し、そのデータを使用してレポートを作成できます。







WPF-TreeMapコンポーネント



WPFの新しいTreeMapコンポーネントを使用すると、表形式または階層構造のデータを入れ子になった長方形として視覚化できます。そのサイズは表示される値によって異なります。 このコントロールには、次の組み込み機能が付属しています。

  • 要素のサイズと位置を計算するためのいくつかのアルゴリズム：「Slice and Dice」、「Squarified」、「Striped」
  • フラット（表）および階層構造のデータを視覚化する機能。
  • 次の方法のいずれかを使用して要素の色を自動的に設定する機能：要素の値に応じて、グラデーションを使用する要素ごと、グラデーションを使用するグループごと、パレットごと。
  • 対話性：ホバー上の要素を強調表示し、クリックしてプログラムで要素を選択する機能。
  • TreeMapの高度にカスタマイズ可能な外観。
  • カスタムツールチップ。








WPF-カメラコンポーネント



WPF用の新しいカメラコンポーネントを使用すると、Webカメラからのビデオストリームを表示し、それを使用して写真を撮ることができます。 組み込みの設定メカニズムを使用すると、さまざまなカメラパラメーター（明るさ、コントラストなど）を変更したり、複数のデバイスが一度に利用可能な場合に目的のカメラを選択したりできます。







WinFormsとWPF-Office 2016のテーマ



また、このリリースでは、WinFormsおよびWPFアプリケーション用の新しいテーマがあります：Office 2016 Black、Office 2016 White、およびOffice 2016 Colorful。







ASP.NET-CardViewおよびQuery Builderコンポーネント



DevExpress ASP.NET CardView Controlに新しい機能が追加されました。これは、今年初めにCTPとして導入され、現在公式リリースが提供されています。

  • バッチ編集
  • 条件付き書式
  • PDF、XLS、XLSX、RTFへのエクスポート
  • 選択したレコードをエクスポートする
  • 条件付き書式付きエクスポート
  • カードヘッダーパネル
  • 総計
  • 無限のページング
  • 日付範囲フィルタリング
  • トリミングされたセルテキストにツールチップでドットを表示
  • Design-Time Card Layout Editor（カードレイアウトの作成と変更を簡単にする特別な開発時エディター）








新しいDevExpress ASP.NETクエリビルダーコントロール-SQLクエリを視覚的に作成し、SELECTステートメントを含むテキスト文字列を返すことができます。



機能：

  • データベーススキーマを自動的に取得し、ユーザーインターフェイスに表示します。
  • WHERE、ORDER BY、およびGROUP BYクエリ句の視覚的な編集。
  • 外部キーに基づいたテーブル間通信の自動解決。
  • クエリ結果をプレビューします。








Excelからのデータバインディング-すべての.NETプラットフォーム



このリリースでは、Microsoft ExcelまたはCSVファイルを多くの.NETコンポーネントのデータソースとして使用できる新しいExcelDataSourceコンポーネントを作成しました。







HTML5フォームレイアウトウィジェット



バージョン15.2の新しいウィジェットはフォームレイアウトウィジェットで、データソースから現在のレコードを編集できます。 このウィジェットは、データ自体のフィールドに従って、開発者の裁量で任意の順序で値の入力フィールドを配置できます。 値の入力フィールドをグループ化し、セクションに分割し、列に分割できます。つまり、フォームにマークアップを付けることができます。 特定の指定された検証規則に従って、これらのフィールドにユーザーが入力した値を確認することもできます。







HTML5グリッド-フォームの編集



非常に多くの場合、テーブルのデータを表示するときは、新しい行に入力するか、既存の行を編集する必要があります。 このリリースでは、開発者が任意のシナリオに合わせてカスタマイズできる特別な入力フォームのおかげで、はるかに便利になりました。 さまざまなタイプのデータ、リスト、日付、グループ化フィールド、入力された値の確認を含むさまざまな数のフィールドとセクション-この形式では、ユーザーに必要なテンプレートを作成できます。







HTML5スケジューラ-タイムラインビューとモバイルUI



HTML5スケジューラの場合、このリリースにはデータ入力フォームとイベントのツールチップの両方をカスタマイズする機能があります。 また、モバイルデバイスとの互換性にも注意を払い、小さな画面サイズやタッチイベントに対応するようにユーザーインターフェイスを調整しました。 新しいタイムラインビューを使用してイベントを水平方向に配置できるようになりました。このビューは、日、週、および週の3つの期間にまたがっています。







ダッシュボード-メモリ内の大量のデータで速度と使いやすさが向上



このバージョンでは、メモリ内のデータを計算するコアが完全に書き直されました。 私たちのテストでは 、一般的なデータ分析操作（グループ化、並べ替え、集計）の生産性が、バージョン15.1と比較して最大50倍向上し、メモリ消費が20倍以上減少しました。 この結果は、辞書を使用してデータを圧縮し、ハッシュテーブルを使用して集計し、複数のスレッドで計算を実行することで達成されました。 また、ボクシングによりデータをロードする際のメモリオーバーランも排除しました。



それでもデータが多すぎる場合は、デザイナーで自動更新とデータの読み込みを無効にすることが可能になりました。 これにより、ダッシュボードの設計中に変更ごとにデータの再計算が終了するのを待つことがなくなります。



ダッシュボード-新しいアイテム-散布図



新しいタイプのグラフが登場しました。散布図を使用すると、いくつかの数値間の相関の有無を表示できます。







ダッシュボード-条件付き書式設定-棒グラフ



テーブルおよびピボットテーブル（GridおよびPivotGrid）の条件付き書式設定には、ヒストグラム（バー）が追加され、条件に応じて個々のバーの色を指定できます。 また、すべてのテーブルについて、集計値を管理する機能が登場しました。







eXpressAppフレームワーク（XAF）



追加のモジュールのコレクションは、Webアプリケーションのラスターマップまたはベクターマップにビジネスオブジェクトを表示するように設計されたMapsモジュールの前面に追加されました。 このモジュールは、 dxMapおよびdxVectorMapウィジェットに基づいています 。







レポートモジュールには、レポートを表示するためのHTML5ドキュメントビューアーコントロールの統合、XMLのマークアップの保存によるレンダリングの高速化、複雑なタイプのパラメーターの複数選択のサポートなど、多くの改善が加えられました。







前回のリリースでは、タッチデバイス用に最適化された新しいXAF Webテーマのかなり粗雑なプレビューを示しました。 Web UIを改善するために多くの作業を行っており、「戦闘」アプリケーションで新しいXAFスタイルを使用できるようになりました。







バッチデータ変更、 マスター/詳細モード、および適応列レイアウトの組み込みサポートがあるため、Webアプリケーションユーザーはグリッド内のデータをより便利に操作できるようになりました。







検証ルールがクライアント側で機能するようになりました（ルールの計算にサーバーからの追加データが必要ない場合）。 したがって、ユーザーは入力フォーカスを変更した直後に警告を表示し、変更の最終保存中には警告を表示しません。







詳細フォームのフィールドの場所は、デザイナーを開かずに、ビジネスオブジェクトのコードで直接設定できるようになりました。 新しいDetailViewLayout属性は、このためのものです。



VCL-Officeナビゲーションバー



このリリースでは、NavBarコンポーネントと統合してOutlook 2013ユーザーインターフェイスを完全に再作成できるようにする新しいOfficeナビゲーションバーナビゲーションコンポーネントが用意されています。







VCL-スパークラインエディターとスパークラインルックアップ



スパークラインは、視覚情報を圧縮形式で表示するために使用されます。これにより、傾向をすばやく確認し、異なるセルのデータを比較できます。 このリリースでは、VCLプラットフォームに2つの新しいコンポーネント、スパークラインエディターとスパークラインルックアップを追加しました。これらは2つの異なるシナリオで使用できます：個別に（データソースの有無にかかわらず）または他のコンポーネント（グリッド、ツリーリスト、ピボットグリッドまたは垂直グリッド。
