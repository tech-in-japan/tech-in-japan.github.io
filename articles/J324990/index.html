SDNの利点：例としてHuawei Cloud FabricでVMware vSphereを使用する





今日は、近年、クラウドテクノロジー開発、リソース仮想化、ソフトウェア定義ネットワークの構築のトピックが一般的になった理由についてお話します。



そして、クラウドコンピューティングやあらゆるものの仮想化に関する「情報ノイズ」の背後に隠されている本当のメリットの具体例と、活発に開発されているテクノロジーの波に乗っているときに得られるメリットを理解しようとします。



面白いですね。 そして、私たちは何について話しているのですか？



質問は合理的です。 ここでは、頭字語SDN（英語版Software Defined Networkingから）で知られるソフトウェア定義ネットワークの構築について説明します。



要するに、SDNはネットワーク仮想化の一形態です。



これらのネットワークの主な機能は、制御レベルがデータ送信デバイスから分離され、ソフトウェアで実装されていることです。



ソフトウェア定義ネットワークに関する3つのことを知ることが重要です。



  • データ転送プロセスとデータ管理プロセスは分離されています。
  • ネットワーク管理は統合ソフトウェアを使用して集中化されています
  • 物理ネットワークリソースの仮想化が必要です。


SDNが注目を集めているのはなぜですか？ アプリケーションの領域を見ると、質問は自動的に消えます。



  • インフラストラクチャクラウドサービス。特に、自動モードで消費者に仮想リソースをすばやく割り当てる必要がある場合。
  • データセンター;
  • モノのインターネット（IoT）。


理論は明確です。 そして、利益はどこにありますか？



利益-節約。



第一に、クラウドテクノロジーの使用と同様に、資本コストの節約：サーバーおよびネットワーク機器の購入、継続的な作業プロセスの確保など。



第二に、データセンターに追加のサービスとアプリケーションを展開する時間を短縮し、追加の仮想ネットワークリソースを割り当てます。



第三に、ネットワークメンテナンス（ソフトウェアコントローラーの管理の集中化と自動化による）および新しいアプリケーションの同じ展開のための人件費の削減。



そして最後に、動的制御と、コンピューティング能力の需要の増加に対するほぼ瞬時の反応により、ネットワークリソースの使用効率が向上します。



実際、これらは近年のSDN市場の活発な成長の理由である主要な側面です。



紙の上ではすべてが滑らかです。 使用例はありますか？



それらなしでどこに！



例として、Huawei Cloud Fabricプラットフォームを使用して、データセンターのソフトウェア定義ネットワークを構築することを検討してください。



より正確には、VMware vSphere仮想化プラットフォームとの統合。 Huawei Cloud FabricとVMware vSphereの統合は、新しいサービスとアプリケーションをデータセンターに展開するために必要な時間と労力を大幅に削減する能力の鮮やかなデモンストレーションです。



Huawei Cloud Fabric？ これは何ですか



SDNデータセンターのHuawei Cloud Fabricプラットフォームは、Huawei CloudEngineシリーズのスイッチ上に構築されています。 スイッチのCloudEngineファミリの多くの顕著な特徴の1つは、Open vSwitch Database（OVSDB）プロトコルを使用して、Network Virtualization Controller（VMware NSX）を介してVMware環境と直接統合する機能です。



このため、ネットワークインフラストラクチャの管理、計画、および構成のプロセスは、vSphere独自のツールを使用して実行できます。 さらに、この接続により、ネットワークインフラストラクチャハードウェアプラットフォームがコンピューティングインフラストラクチャと緊密に連携し、仮想マシンを作成、移動、または削除する際のコンピューティングおよびネットワークリソースの接続が非常に簡単になります。









OVSDBを使用したNSX VMware vSphereコントローラーとHuawei CloudEngineスイッチングプラットフォーム間の統合図



このレベルの統合と自動化により、VMware仮想化プラットフォームを使用する環境での新しいサービスの展開、リソースの最適化、および災害復旧に関連する時間、リソース、労力、運用コストが劇的に削減されます。



Huawei Cloud Fabricは、「セルフサービス」モデルのレベルまで、市場に新しいサービスを導入するプロセスを大幅に短縮し、プロセスを簡素化します。 これにより、データセンターのITリソースは「コストセンター」として機能しなくなり、その役割をデジタル時代の「ビジネスアクセラレータ」に変える可能性があります。



そして、どのように統合しますか？ 説明書が添付されていますか？



私たちはすべてについて考えました。



Cloud Fabricとの統合は、vSphere Network Virtualization APIを介してVMware vSphereをオペレーティング環境に展開することにより実装されます。 仮想ネットワークセグメントと特定の仮想マシンのステータス、プロファイル、関連付け、およびリアルタイムでの変更に関する情報は、vSphere SDNコントローラーによってCloudEngineスイッチに直接送信されます。



APIレベルでのCloudEngineとvSphereファミリーのデータセンタースイッチの統合により、IT管理者は、既存のVMware環境を変更することなく、VMwareクラウドとの作業を自動化するための一貫したシンプルで高品質のサポートを得ることができます。



それは簡単に見えます...そして、どのように便利ですか？



Cloud FabricとVMware vSphereの統合から生じる機能は、次の側面で大きな利点を提供します。



  • サービスとアプリケーションの展開を簡素化する
  • ネットワークリソースの構成を作成および最適化するために必要な時間を短縮します。
  • 運用、管理、およびロジスティックのオーバーヘッドのレベルの低下。
  • 展開および構成エラーの削減。










サービス展開サイクル



上の図は、新しいサービスまたはアプリケーションが展開されたときに実行される典型的なネットワーク操作手順を示しています;従来の自動化されていないネットワークでは、これらの操作は主に手動モードで実行されます。 このような手動の繰り返し手順では、合計展開時間とエラー率は人的要因によって決まります。 vSphereとの統合により達成される自動化は、SDNの強力な機能に依存しており、このサイクルを大幅にスピードアップし、エラーの可能性を低減できます。



さらに、VMware vSphereとの統合には、次の直接的な利点があります。



  • 運用担当者向けの短縮された「トレーニングカーブ」：ネットワーク仮想化とvSphereコントローラーを使用する場合、運用のすべての側面は互いに整合性があり、VMwareコンセプトのフレームワーク内にあります。
  • コンピューティング、ネットワーク、およびストレージリソースの管理は「単一のコントロールパネル」に関連付けられています。すべてのデータセンターリソースの調整と「オーケストレーション」は、単一のプラットフォームであるVMware vSphereで集中的に実行されます。
  • 既存のVMware vSphereおよびNSX Controllerインフラストラクチャに基づいて、柔軟でスケーラブルなハードウェア最適化SDNソリューションを提供することにより、既存の投資の効果を最大化します。


具体的な数値を見てみましょう。時間コストの削減により、データセンターの運用コストを大幅に削減できます。



以下の表は、一般的に受け入れられている（最良の場合）作業サイクルの典型例を示しています。これは、年に数千回ではないにしても、数百回発生する可能性があります。 ネットワークの規模、会社の規模、および年間に展開されるサービスとアプリケーションの数に応じて、実際の節約は大幅に大きくなります。









新しいサービスの導入速度に関しては、手動モードでは通常の準備期間は1日から2週間かかると言えます。



業界平均の3日間のサービスの手動展開時間を背景に、Huawei Cloud FabricやVMware vSphereなどのSDNシステムのほぼリアルタイムのパフォーマンスは、これまで利用できなかった即時セルフサービス機能を提供します。



新しいサービスおよびサービスを迅速に構成、展開、および提供する機能により、あらゆる業界の企業ITサービスは、ユーザーエクスペリエンス、顧客との対話、ビジネスインテリジェンス、運用、および物流の分野で競争上の優位性を生み出すことができます。



1 ... 2 ... 3 ...販売！



ネットワークインフラストラクチャをVMware vSphereオーケストレーターと統合し、単一システムを介してクラウドサービスを展開する目標は、エンドユーザーのセルフサービスモデルを実装することです。



このモデルにより、企業は新しいアプリケーションの起動や災害復旧の整理にかかるコストと時間を削減できます。 また、プロセスを自動化および簡素化する戦略により、ITスタッフは日常的な構成のタスクを取り除き、得られた時間をより効率的に使用できます。



さらに、SDNプラットフォームとVMware vSphereの統合は、ネットワークとコンピューティング機能のギャップを埋めるのに役立ちます。これは、エンタープライズ市場でプライベートクラウドを普及させる重要な要因です。
