なぜすべてがこんなに遅くなったのですか？！ Unityで開発するためのハードウェアの選択

こんにちは Unityでゲームとアセットを作成しています。 初心者の開発者から、どこから始めればよいか、コンピューターがどれほど強力になるかを尋ねられることがあります。 個人的に、私のハードウェアはかなり古いです。 そして、開発をスピードアップするために、私はついにコンピューターをアップグレードすることにしました。 遠い2011年以来、私のPCの心臓部はIntel Core i5 2500Kプロセッサーですが、それは廃止する時です。 しかし、どちらが良いですか-高いクロック速度または多数のコアの存在？ 高価なプロセッサを購入する必要がありますか、それとも節約できますか？ 開発にはどのようなグラフィックカードが必要ですか？ カットの下で理解します！







「要するに、どちらが良いですか？」



最もせっかちな人はすぐに最後までスキップできます）



Unityはいつ愚かですか？







Unityでの作業には多くの時間を要するいくつかの操作があります。 通常、そのような場合、私はお茶に行くか、猫と遊ぶ。 時々、Duolingoでいくつかのレッスンを完了することさえできます。



  1. プロジェクトファイルの処理。 Unityは、プロジェクト内のすべてのファイルのハッシュを計算し、メタファイルを作成し、独自のライブラリ（ライブラリフォルダー）を構築します。 Unityは、オーディオファイルの処理に特に時間がかかります。 バージョン管理システムにライブラリを保存するユーザーはいないため、プロジェクトを長時間同期していない場合は、数分待つことになります。 さらに、たとえばAndroidからiOSにプラットフォームを変更する場合、このプロセスを繰り返す必要があります。
  2. 光の焙煎（ベーキング）。 それはすべて、照明の複雑さに依存します。 私はモバイルゲームを作っているので、実質的にはベーキングを使いません。
  3. プロジェクトの組み立て。 それは、選択したプラットフォームとプロジェクト自体に大きく依存します。 WebGLは一般に長年にわたって収集することができます（場合によっては、実際に古くなり、あごひげまで成長することもあります）。 ソースコードをコンパイルするのではなく、完成したゲームを取得することを意味します（ほとんど瞬時に発生します）。


2500Kについて一言



Intel Core i5 2500Kは2011年にリリースされ、Sandy Bridgeライン（32ナノメートルの製造プロセス）に属します。 周波数3.30 GHz（ターボブーストで3.70 GHz）、4コアおよび4スレッド、6 Mbキャッシュ。 適切な値札と接頭辞Kがなければ、特別なことは何もありません。良いモボ（私はp67チップセットを持っています）では、それは地獄のように動きます（すべてカバーの下にはんだの付いたSandy Bridgeがあります）。 彼は私のために4.7 Ghzの周波数で7年間働いていました。 それは2018年であり、このプロセッサは依然としてフリーマーケットで約5,000ルーブルの予算セグメントで最高と考えられています。 一般に、開発中に特別な不便さは感じません。そのようなプロセッサで十分です。 なぜi7ではないのですか？ はい、2倍の費用がかかりますが、私は否定しません）



ビデオカード



ビデオカードは、開発には直接関係ありません。 プロジェクトの起動とテストにのみ必要です。 「クールなグラフォン」を使用してPCゲームを作成する場合は、優れたグラフィックカードに投資する必要があります。 私はモバイルゲームを作成しているので、GTX 1080の店に走る必要はありません（おそらく鉱山労働者はすでに購入しています）。 そこで、古いGTX 460をそのまま使用することにしました。しかし、2枚のビデオカードでテストを実施します。



CPUの選択



Ryzen AMDのリリースにより、プロセッサ市場に戻りました。 したがって、選択は次のいずれかになります。



  • Intel Core i5
  • Intel Core i7
  • ライゼン5
  • ライゼン7
  • 過去の世代のXeonマルチコアサーバープロセッサ（安価であるため）


i5以下のすべて、および古いAMDプロセッサについては、考慮しません（インディー開発者とdoshirakiに関するすべてのジョークにもかかわらず）。



最新のCore i5は、私のi5 2500Kの論理的な代替品です。 この7年間、Intelは競争をせず、「Tik-Tak-Tak」のコンセプトに従って、プロセッサを平均50％向上させました。 そのため、新しいプロセッサを購入した後、Unityのバカが1.5倍少なくなったとしても、私は満足します。



Core i7-ボイヤーのための石で、i5の2倍の費用がかかります。 お金のためにハイパースレッディングが必要ですか？ さまざまなベンチマークによると、ハイパースレッディングは最大30％のパフォーマンス向上を提供します。



Ryzen 5およびRyzen 7は、新しい6コアおよび8コアプロセッサです。 インテルよりもコアは多いが、周波数は少ない。



16XXおよび26XXシリーズのXeonサーバープロセッサには、6〜10コア（E5 2680 V2）があります。 中国のサーバーから撤去され、100ドルから150ドルの価格でAliExpressで販売されているため、検討する価値があります。 特に予算が限られている場合。 マイナス-あなたは100ドルと強力なクーラーで中国のマザーボードを購入する必要があります。 利点-安価なサーバーDDR3（登録）。



Ryzenのリリース後、Intelはすぐに事業計画を調整し、6コアプロセッサをリリースしました。 ゲーム用のプロセッサが必要な場合は、間違いなくIntelを選択する必要があります。 ほとんどのゲームテストでは、i7 7700Kなどの4コアIntelプロセッサーでさえRyzenよりも優れています。 さらに、空冷時でも最大5 Ghzを追跡します。



しかし、私は別のケースがあります-私はゲームを作ります。 私のコーヒーブレイクは8コアで半分の長さになりますか？



残念ながら、実験を行うためにこれらのプロセッサをすべて購入する方法はありません。 しかし、手元にはi5 2500Kと6つのコアを備えたサーバーXeon E5 1660があります（ベンチマークでは非常に近いため、Ryzen 5の「アナログ」と見なします）。 Unityでの開発に最適なものを判断するだけで十分です。



したがって、テストします。



  • オーバークロックi5 2500K（4C / 4T）〜80 $
  • 少しオーバークロック（華南では特に加速しません）E5 1660（6C / 12T）〜130 $


42/42/42/42/42/0/0の最大乗数がE5 1660に適用されたことに注意してください。 これは、4つ以上のコアをロードすると、周波数がベース値の3600 Mhzに低下することを意味します（ブランドボードでは最大4800まで駆動されます）。 i5 2500Kは、どのような負荷でも最大周波数4700 Mhzで動作できます。



比較のために、これらのプロセッサーと新しいプロセッサーのベンチマーク結果を購入することを検討しました（すべてオーバークロック）。







したがって、新しいプロセッサを選択すると、このデータを使用して速度の向上を評価できます。



テストスタンド



  • 新しくインストールされたWindows 10（迷惑メールで散らかっていない）
  • 最も安い中国のSSD
  • GT 210およびGTX 460グラフィックスカード
  • Project Tap Tap Builder（AndroidおよびiOS用のモバイルゲーム）、アセットの合計サイズは約500 Mb


試験方法



このテクニックはストレッチと呼ばれます。 ストップウォッチを使用して、次の操作を開始します。



  • Windows 10の読み込み（ユニットがオンになってからデスクトップが表示されるまで）
  • 削除されたライブラリフォルダーの再作成（プロジェクトを開いた瞬間からエディターウィンドウが表示されるまで）
  • Android用ゲームをビルドします（ビルドボタンを押した瞬間からAPKが表示されるまで）
  • 並行して、AIDA64のCPU負荷を調べます。


Windows 10をダウンロードする



  • E5 1660は21秒でシステムを起動します
  • i5 2500Kは26秒でシステムを起動します


OSのロードはほとんどの場合、SSDからRAMにファイルをコピーするだけなので、どちらの場合もシステムは非常に高速に起動します（お茶を入れる時間はありません）。 数秒の差は、マザーボードがオンになった時間（異なるソケット、異なるBIOS）に起因します。



削除されたライブラリフォルダーの再作成



リモートフォルダは、遠くにあるものではありません。 Unityがプロジェクトを再インポートできるように削除します（再インポート機能は信頼できません）。



i5 2500Kは最初に戦闘に参加します。







結果は5:43です。 プロセッサは、ほとんどの場合4700 Mhzの最大周波数で動作しますが、100％をロードしません。



次に、E5 1660キュー：







結果は5:53です。 プロセッサは負担をかけずに動作し、周波数はたまに最大4200 Mhzに達します。 平均プロセッサ負荷は約12％です。



これらの結果は何について話しているのですか？ どちらの場合も5分は非常に長い時間です。 i5 2500Kは最大周波数をリセットしませんが、100％ロードされていません。 E5 1660はすべてのコアを使用しようとするため、周波数を3600 Mhzに減らします（ただし、平均負荷は非常に低くなります）。



最初のスレッドでは十分なスレッドではなく、2番目の周波数では不十分であるため、両方のプロセッサが同様に悪い結果を示すと想定されます。



Android用のプロジェクトを構築する



i5 2500Kが再び起動します。







結果は2:50です。 最大周波数で動作し、コア負荷が100％に達することもあります。 それらがもっとあれば、結果は改善されていただろう。



その後、E5 1600：







結果は2:50で、まったく同じです！ 周波数は同じ-3600 Mhzですが、コアの負荷が100％に達することはありません。 ただし、プロジェクトをインポートする場合よりも負荷は確実に高くなります。



結論は、前のテストとまったく同じように作成できます。 さらに、i5は天井に置かれていました。



グラフィックカードを変更する



これまでのところ、GTX 460が機能しましたが、このテストでは、これをGT 210に置き換えます。 彼女よりも悪いのは彼女の不在だけです。



E5 1660のプロジェクトをインポートします。







結果は5:38です。 予想どおり、操作の速度は変更されていません（違いはエラーと、Windowsの操作などの外部要因によるものです）。



RAM



12 GbのRAMがあり、Unityで作業するときの使用量は50％を超えません。 したがって、Unityでの開発には8 Gb RAMで十分です。 並行して、PhotoshopとYouTubeを引き続き実行できます。



結論



1.快適な開発のためには、高いプロセッサクロック速度と多数のスレッドの両方が必要です。 Intel i5プロセッサの4スレッドでは明らかに十分ではありません。 予算に基づいて、ゲーマーが本当に好む4コアi7 7700Kまたは6コアi5 8600Kは、どちらも約250〜300ドルです。 お金があれば、6コアのi7 8700Kを400ドルで購入できます。 Ryzen 5 1600XとRyzen 7 1800Xは、より少ないお金で十分な代替品となりますが、最大周波数でIntelを失うことになります-4 Ghz対Intelの5 Ghz（もちろん、オーバークロック時）。 従来のSandy Bridge +プラットフォームを使用するのは、i5、i7、サーバー側Xeon 16XXおよび26XXのいずれであっても、限られた予算でのみ意味があります。



2.最もクールなグラフィックカードは必ずしも必要ではありません。 特にモバイルゲームを作成する場合。 だから、一番上のビデオカードは鉱夫に任せてください。 私のGTX 460を交換するために、Dark Souls 2がもう一度通過できるように、130ドルでGTX 1050を選択しました。



3.さらに、Unityはアクティブにファイルを操作しているため、SSDの使用をお勧めします。 予算が限られている場合は、8または16 Gbの最も安価なSSDを使用してプロジェクトを保存し、そこにUnityと必要なすべてのSDKをインストールすることもできます。
