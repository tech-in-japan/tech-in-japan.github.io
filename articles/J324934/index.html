PHPの配列



最初から始めましょう。



PHPには配列があります。 PHPの配列は連想的です。つまり、ペア（キー、値）を格納します。キーはintまたはstringである必要があり、値はどのタイプでもかまいません。



例： 



$arr = ['foo' => 'bar', 'baz' => 42, 'arr' => [1, 2, 3]];


キーと値は、記号「=>」で区切られています。 キーは「インデックス」と呼ばれることもありますが、PHPではこれらは同等の用語です。



PHPの配列では、かなり完全な一連の操作が定義されています。 



 //    $arr['new'] = 'some value'; //      $arr[] = 'another value'; //      echo $arr['foo']; echo $arr[$bar]; //     unset($arr['foo']); // ""  [$foo, $bar, $baz] = $arr;


配列を操作するための多くの関数もあります- 数十、数百！



ただし、おそらくPHPの配列の最も重要なプロパティは、配列のすべての要素を順番に処理し、すべてのキーと値のペアを順番に取得する機能です。



配列の反復



配列をトラバースするプロセスは「反復」（または「無効化」）と呼ばれ（ところで、各ステップ、個々のキーと値のペアを取得することも「反復」です）、したがって配列自体は反復可能です（「反復」）本質。



反復プロセスの最も単純な例は、もちろん、foreachステートメントによって実装されるジョイントループです。 



 foreach ($arr as $key=>$val) { echo $key . '=>' . $val; echo "\n"; }


ループヘッダーのキーと値を区切る同じ「=>」記号に注意してください。



しかし、PHPはどのように理解しますか-ループの特定のステップで配列のどの要素を取るか？ 次に取るのはどれですか？ そしていつ停止するのか？



この質問に答えるには、各配列に存在するいわゆる「内部ポインター」の存在に注意する必要があります。 この不可視のポインタは「現在の」要素を指し、さらに1ステップ移動できます-次の要素に移動するか、最初の要素に再度リセットします。



PHPの内部ポインターを直接操作するには、例を使用して最も簡単に学習できる関数があります。 



 $arr = [1, 2, 3]; //   ,      reset($arr); // key()    ,     ,  null         while ( null !== ($key = key($arr)) ) { // current()    ,      echo $key . '=>' . current($arr); echo "\n"; // next()         next($arr); }


与えられたコード例が実際に以前に使用されたforeachループと同等であり、foreachがreset（）、key（）、current（）、next（）関数の構文シュガーのようなものであることは簡単にわかります（そしてend（）およびprev（ ）-逆順で検索を整理します）。



このステートメントは、PHP 7より前は真実でしたが、現在はそうではありません-foreachループは、reset（）、next（）、およびその他の反復関数と同じ内部ポインターを使用して停止したため、位置の変更を停止しました。



小計



したがって、PHPの配列の繰り返しがどのように機能するかを要約するには：



  • 各配列には、内部ポインターが関連付けられています。
  • 配列の先頭（または末尾）にリセットできます
  • 次の（前の）要素に移動できます。
  • 最後に到達したかどうかを確認できます-ポインターが配列から出ていますか？
  • そして、現在の要素のキーと値を取得できます（ポインタが指す）


このようなデバイスを使用すると、ループ内で配列（要素を列挙）の繰り返しを整理できます。 ただし、foreachループは似ていますが、reset（）、key（）、current（）などの関数と同じ内部ポインターでは機能せず、独自のローカルループで機能することを理解することが重要です。サイクルのために。



オブジェクトを反復処理する



配列のようなオブジェクトは、反復可能なエンティティです。 オブジェクトは、このコンテキストで表示されるプロパティによってクロールされ、プロパティ名はキーとして機能します。 



 class Foo { public $first = 1; public $second = 2; protected $third = 3; public function iterate() { foreach ($this as $key => $value) { echo $key . '=>' . $value; echo "\n"; } } } $foo = new Foo; foreach ($foo as $key => $value) { echo $key . '=>' . $value; echo "\n"; } /*   first=>1 second=>2 */ $foo->iterate(); /*   first=>1 second=>2 third=>3 */


ただし、そのような反復は、その目に見える特性に従って、多くの場合完全に役に立たない。 最も一般的な例は、内部の安全なストアに値のセットを格納するオブジェクトです。 たとえば、次のように： 



 class Storage { protected $storage = []; public function set($key, $val) { $this->storage[$key] = $val; } public function get($key) { return $this->storage[$key]; } }


パブリックプロパティを持たないこのようなオブジェクトの反復を整理する方法は？ そして、ある種のカスタム非標準アルゴリズムを使用して一般的に反復を整理する方法は？



イテレータインターフェイス



独自の反復アルゴリズムを実装するために、PHP（またはSPL）は、5つのメソッドで構成される特別なIteratorインターフェースを提供します。 



 //       public function current(); //       public function key(); //    ""    public function next(): void; //    ""    public function rewind(): void; //    -      ? public function valid(): bool


クラスはこれらのメソッドを実装する必要があり、実装されたアルゴリズムに従ってforeachループを使用してこのクラスのオブジェクトを反復する機会が得られます。



ここでイテレータインターフェイスメソッドの説明で言及されているNB "ポインタ"は、実際に存在する配列の内部ポインタとは対照的に、純粋な抽象化です。 この抽象化の実装方法はユーザーによって異なります。結果のみが重要です。たとえば、rewind（）およびcurrent（）メソッドの連続呼び出しは、最初の要素の値を返す必要があります。



Iteratorインターフェースを実装する最も簡単な例 
 class Example implements Iterator { protected $storage = []; public function set($key, $val) { $this->storage[$key] = $val; } public function get($key) { return $this->storage[$key]; } public function current() { return current($this->storage); } public function key() { return key($this->storage); } public function next(): void { next($this->storage); } public function rewind(): void { reset($this->storage); } public function valid(): bool { return null !== key($this->storage); } } $test = new Example; $test->set('foo', 'bar'); $test->set('baz', 42); foreach ($test as $key => $val) { echo $key . '=>' . $val; echo "\n"; }




TraversableおよびIteratorAggregate



厳密に言えば、Traversableインターフェイスを使用すると、foreachを使用して反復処理を行うことができ、Iteratorがその後継です。 Traversableの機能は、直接実装できない（一種の「抽象インターフェース」）ため、アプリケーションでIteratorインターフェースまたはその「弟」IteratorAggregateを使用する必要があります。 彼について話しましょう。



SPLには、イテレーターオブジェクトで他のエンティティ（配列など）をラップできるいくつかの組み込みイテレータークラスが含まれています。 



 $iterator = new ArrayIterator([1, 2, 3]); foreach ($iterator as $key => $val) { // ... }


このような既製のイテレータラッパーのリストは非常に大きく、DirectoryIterator（指定されたディレクトリ内のファイルのリストを反復処理）、RecursiveArrayIterator（ネストされた配列の再帰的トラバース）、FilterIterator（不要な値の拒否によるトラバース）、およびその他の多数の有用なクラスが含まれます 。



既製のイテレーターとIteratorAggregateインターフェースを使用すると、独自のイテレータークラスの作成を大幅に簡素化できます。 したがって、上記のネタバレの下にある非常に長いクラスは、次のようなものに減らすことができます。 



 class Example implements IteratorAggregate { protected $storage = []; public function set($key, $val) { $this->storage[$key] = $val; } public function get($key) { return $this->storage[$key]; } public function getIterator(): Traversable { return new ArrayIterator($this->storage); } }


-結果は、Iteratorインターフェースの実装と同じになります。



ジェネレーターはどうですか？



もちろんです。 foreachでそれらを使用します！ 



 class Generator implements Iterator


ただし、ジェネレーターは別の記事のトピックです。 それまでの間、ジェネレーターのメカニズムに魔法は何もないと言うだけで十分です。同じIteratorインターフェースが反復に使用されます。 1つの「but」を除いて、反復が既に開始されている場合、ジェネレーターを「最初に巻き戻す」ことはできません。rewind（）メソッドを呼び出すと例外がスローされます。



反復可能タイプ



PHP 7.1より前には、奇妙な状況がありました。 一方には、IteratorまたはIteratorAggregateを介してTraversableを実装する反復可能なオブジェクトがありました。 同じメカニズムを使用するのと同じ側にジェネレーターがありました。 また、反対側には配列があり、オブジェクトの可視プロパティに対する「ネイティブ」反復があります。 実際、同じ動作をするが共通点のない2種類の反復可能なエンティティがありました。



7.1では、最終的にこの不整合が解消され、別の「疑似型」（またはむしろカスタム型）が「反復可能」になりました。



PHPでtype演算子が表示されるのをいつか待つと、iterableの定義は次のように記述できます。 



 type iterable = array | Traversable;


この型は、配列とすべてのTraversable子孫を組み合わせ、foreachを使用して反復できる値の型を示します。 



 function doSomething(iterable $it) { foreach ($it as $key=>$val) { // do something } }


そして、それは何ですか？



このタイプチャートは次のとおりです。 



 iterable ---> array --> Traversable ---> Iterator --> IteratorAggregate --> Generator


表示可能なプロパティに応じてネイティブな反復を許可するオブジェクト（「オブジェクトだけ」タイプ）は、反復可能なタイプにまだ含まれていないことに注意してください。 しかし、そのようなオブジェクトを反復処理することの実用的な価値はそれほど大きくないため、動揺する理由はありません...
