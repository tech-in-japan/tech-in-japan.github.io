Debian 7 wheezyにownCloud 6をインストールした経験





こんにちは、habravchane！ 今日は、ownCloudパーソナルクラウドストレージとそのインストールについてお話したいと思います。



Dropbox、Google Drive、SkyDrive、Mail.Ru Cloudなどのサービスを使用することが一般的になりました。特定のサーバーの明らかな欠点に注意を払うことなく、毎日サービスを使用しています。 1つのサービスにはスペースがほとんどなく、もう1つにはローカライズがなく、3番目には信頼性の低いストレージ、4番目の曲線にはインターフェースなどがありますが、これらのサービスにはすべて共通の重大な欠点があります-データは常に管理していないサーバーに保存されます。 あなたにとって大きな価値のないデータをホストするためにサービスを使用する必要がある場合、これらのサービスは完全に十分なはずです。 コンテンツ保護、完全な機密性、セキュリティ、無制限の空き領域、グループで作業するマルチユーザーアクセスを整理する機能、バージョン管理、同期、ドキュメント、テーブル、画像、ビデオ、オーディオ、世界中のどこからでもメモを表示する機能が必要な場合それらをダウンロードする必要があります-ownCloudはあなたが必要とするものです。



ownCloudを詳しく見て、インストールしてみましょう。



OwnCloudは、文字通り「独自のクラウド」と解釈されます。 これは、コンピューターにダウンロードして、後でホスティングまたは独自のサーバーにインストールできるオープンソースのCMSです。 機能面では、ownCloudはクラウドストレージの世界のトップ企業に遅れをとらず、一部を超えています。







シンプルで簡潔なインターフェイスは、事前インストールされたモジュールと公式Webサイトに表示される独自のAppストアの助けを借りて、さらに拡張可能な優れた機能を隠します。



「すぐに使える」ownCloudの機会：





  • ドラッグアンドドロップマルチスレッドドラッグアンドドロップデータの読み込み
  • 特定のユーザー、グループ、またはインターネットのファイル許可の編集
  • RSSフィードを接続して表示する
  • 画像、ビデオ、オフィス文書を見る
  • 同期カレンダー、タスクマネージャー、連絡先マネージャー、Webブックマークマネージャー
  • ユーザーアクティビティの追跡
  • 安全な接続、データ暗号化、LDAP \ WebDAV認証
  • 外部ストレージのサポート：Google Drive、Dropbox、FTP、SMB、WebDAV、Amazon S3、IRODS、OpenStack Object Storage、およびローカル
  • 現在のすべてのプラットフォームに完全に同期するクライアントアプリケーション




これはownCloudの機能の短いリストですが、その利点はすでに明らかになっています。 必要に応じて、メールクライアント、トレントクライアント、ビデオチャット、本格的なオフィス、VirusTotalのファイルチェックなどを追加できます。



理論から実践に移ります。



Debian 7 wheezyを実行しているサーバーにownCloudをインストールする



1.基本ソフトウェアのインストール




始める前に、システムが最新であることを確認しましょう。

ターミナルを開き、次を入力します。 

sudo apt-get update && sudo apt-get upgrade


これら2つの単純なコマンドは、サーバー上のすべてのプログラムを更新します。インターネットとサーバーの速度によっては、プロセスに時間がかかる場合があります。



これで、ownCloudが機能するために必要なソフトウェアのインストールを開始できます。 

 sudo apt-get install apache2 php5 php5-common php5-gd php5-intl php5-mcrypt php5-cli php5-ldap php5-sqlite curl libcurl3 libcurl4-openssl-dev php5-curl php-apc php5-mysql ffmpeg


これで、サーバーにownCloudを迅速にデプロイするために必要なすべてが揃ったので、ステップ4に進むか、フルインストールを続行できます。



2.MySQLのインストール




MySQLをインストールします。 

 apt-get install mysql-client mysql-server


MySQLデータベースのパスワードを作成して先に進みます。



重要：パスワードはスーパーユーザーのパスワードと一致してはなりません。



次に、セキュリティに少し注意して、コンソールに入ります。 

 mysql_secure_installation


「rootパスワードを変更する」以外のすべての質問（MySQLパスワードを変更したくない場合）は、Yによって肯定的に回答されます。



MySQL管理コンソールに入り、パスワードを入力します（表示されません）： 

  mysql -u root -p


データベースを作成します。 

  create database _;


できればデータベースのような名前でユーザーを作成します。 

  create user '_'@'localhost' IDENTIFIED BY '_';


重要：パスワードはスーパーユーザーのパスワードと一致してはなりません。



作成されたユーザーに作成されたデータベースへの完全な権限を転送します。 

  GRANT ALL ON _.* to '_'@'localhost' ;


終了します： 

  quit




3.サーバーで時刻を同期するためにntpをインストールする




ntpクライアントをインストールします。 

  apt-get install ntp ntpdate


次に、時刻同期サーバーを最も近いサーバーに変更します。 

  sudo gedit /etc/ntp.conf


開いたファイルには次のものがあります。

サーバー0.debian.pool.ntp.org iburst dynamic

サーバー1.debian.pool.ntp.org iburst dynamic

サーバー2.debian.pool.ntp.org iburst dynamic

サーバー3.debian.pool.ntp.org iburst dynamic


最も近いnptサーバーのアドレスに変更します。次に例を示します。

サーバー0.ru.pool.ntp.org iburst dynamic

サーバー1.ru.pool.ntp.org iburst dynamic

サーバー2.ru.pool.ntp.org iburst dynamic

サーバー3.ru.pool.ntp.org iburst dynamic


エディターを保存して終了します。



正しい時間を確認するには、ターミナルに入力するだけです： 

  ntpdate 0.ru.pool.ntp.org


サーバーの時刻がntpサーバーから受信した時刻と一致しない場合は、次のように入力します。 

  ntpdate -bs 0.ru.pool.ntp.org




4.構成




ファイル/ etc / apache2 / sites-enabled / 000-defaultを編集します。 

  sudo gedit /etc/apache2/sites-enabled/000-default


「<Directory / var / www />」セクションに行があります。

AllowOverrideなし


変更先：

すべてを許可


ファイルを保存して終了します。



Apacheを再起動します。 

  service apache2 restart


php.iniの編集： 

  sudo gedit /etc/php5/apache2/php.ini


次の行を見つけます。 

 mysql.default_socket = mysql.cache_size = memory_limit = post_max_size = upload_max_filesize =


変更先： 

 mysql.default_socket=/var/run/mysqld/mysqld.sock mysql.cache_size = 4000 upload_max_filesize = 1024M //          post_max_size = 1024M //     upload_max_filesize memory_limit = 1024M //     post_max_size


ファイルを保存して終了します。



Apacheを再起動します： 

  service apache2 restart

ownCloud 6を書いている時点で、ownCloudの最新バージョンをダウンロードしてください。

1.アプリセクションでのアプリケーションの誤ったインストール




アプリケーションをオンにした後、Webサイトが正しく動作しなくなった場合、3つの解決策が見つかりました。

1.完全な再インストール。

2. phpmyadminでowncloud用に作成されたデータベースを削除し、ファイルを削除します。 

 rm -f \var\www\owncloud\config\config.php


3.おそらく最も快適なオプションです。問題が始まったアプリケーションを含むフォルダーを\ var \ www \ owncloud \ appsフォルダーから他のフォルダーに移動し、サイトが機能していることを確認し、フォルダーを元の場所に戻し、修正に触れないでください。
