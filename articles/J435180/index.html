サンプルを解析するためにデータベースを使用するのではなく、情報のソースはAWS S3のプレーンテキストファイルになります。



  • したがって、AWS S3にはヘッダー付きのテキストファイルがあり、何らかのプロセスがそれに情報を書き込むとします。
  • クラウドAPIを作成します。これは、渡されたパラメーターを使用してJSON応答をGETリクエストとして返します。
  • 同時に、タスクの複雑さと、結果としてリソースの処理能力に対する要件の増加に応じて、心配する必要はありません。 サービスは完全に自動拡張可能です。 これは、管理、サーバーの割り当て、管理が不要で、コードをアップロードして実行するだけであることを意味します。


開発中のシステムのアーキテクチャ







使用されるアマゾンウェブサービスコンポーネント：



  • Amazon S3-オブジェクトストレージ。これにより、事実上無制限の量の情報を保存できます。
  • AWS Identity and Access Management（IAM） -AWSのサービスとリソースへのアクセスを安全に制御する機能を提供するサービス。 IAMを使用して、AWSユーザーとグループを作成、管理、管理し、アクセス許可を使用してAWSリソースへのアクセスを許可または拒否できます。
  • AWS Lambdaは、サーバーをバックアップおよび構成せずにコードを実行できるサービスです。 すべての計算能力は、コールごとに自動的にスケーリングされます。 料金は、機能リクエストの数とその期間に基づいて請求されます。 コードが実行される時間。

    無料アクセスのレベル（無料利用枠）は、無料で40万Gb-sの月あたり100万のリクエストを意味します。 サポートされている言語：Node.js、Java、C＃、Go、Python、Ruby、PowerShell
    。 Pythonを使用します。



    • boto3ライブラリは、さまざまなAmazonサービスとやり取りできるPython用AWS SDKです。
  • Amazon API Gatewayは、あらゆる規模のAPIを作成、公開、保守、監視、および保護するために設計された完全に管理された開発者サービスです。 デバッグ、改良、テストのために同じAPI（ステージ）の複数のバージョンを使用する機能に加えて、このサービスではAWS Lambdaを使用してサーバーレスREST APIを作成できます。 Lambdaは、非常にアクセスしやすいコンピューティングインフラストラクチャでコードを実行するため、サーバーの配布、スケーリング、管理の必要がなくなります。

    HTTP / REST APIの無料利用枠には、12か月間、1か月あたり100万回のAPI呼び出しが含まれます

AWS Lambdaのロールを作成する



  1. AWS管理コンソールで、AWS IAMサービスに移動し、[ロール]タブに移動して、[ロールの作成]ボタンをクリックします。



    新しい役割を追加する


  2. 現在作成しているロールは、AWS LambdaによってAWS S3から情報を読み取るために使用されます。 したがって、次のステップでは、「信頼されたタイプの選択」->「AWSサービス」および「このロールを使用するサービスの選択」->「Lambda」を選択し、「次へ：許可」ボタンをクリックします



    Lambdaサービスの役割


  3. 次に、新しく作成されたロールで使用されるAWSリソースのアクセスポリシーを設定する必要があります。 なぜなら ポリシーのリストは非常に印象的で、ポリシーのフィルターを使用して、「S3」を示しています。 その結果、S3サービスに関連してフィルター処理されたリストを取得します。 「AmazonS3ReadOnlyAccess」ポリシーの反対側のチェックボックスに注意し、「次へ：タグ」ボタンをクリックします。



    役割のポリシー


  4. ステップ（タグの追加（オプション））はオプションですが、必要に応じてロールのタグを指定できます。 これは行わず、次のステップであるプレビューに進みます。 ここで、ロール名を設定する必要があります-「ForLambdaS3-ReadOnly」、説明を追加し、「ロールの作成」ボタンをクリックします。



    タイトルの役割




すべて、役割が作成されており、今後の作業で使用できます。
