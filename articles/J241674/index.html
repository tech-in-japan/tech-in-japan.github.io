CppCon 2014カンファレンスペーパー

最近、C ++開発者の最大のカンファレンスの1つであるCppCon 2014がワシントン州ベルビューで開催されました。 5日間、Microsoft、Google、Dropbox、Citrix、Embarcadero、Ubisoft、言語標準開発者、C ++コンパイラの作成者、およびオープンソースコミュニティのメンバーなどの企業の主要なプログラマーがレポートを発表し、言語の将来について意見を交換し、新しいアイデアを提供しました。 以下に、私が最も気に入ったビデオのセレクションと、自分自身からのコメントをいくつか紹介します。 C ++ 11 \ 14標準の普及を考慮して、会議全体がC ++の復活の精神で飽和状態になっていることに注意してください。人々は、C ++の選択が彼らにとって正しいと判明した理由、C＃\ Java \ Objective-CからCへの移行方法を説明します++後悔しなかったなど。

Dropboxのスピーカーは、C ++でクロスプラットフォームモバイルアプリケーションを開発する方法について話します。

Dropboxには、従来のモバイルアプリ（Android用のJavaコードとiO用のObjetive-C）がありました。 しかし、時間の経過とともに、開発チームは同じことを異なる言語で2回書くことにうんざりし、C ++で共通のコードベースを作成する必要があるという結論に達しました。 時間が経つにつれて、アプリケーションのアーキテクチャ（MVC、MVVMなど）に関係なく、実際には、ビューを除くすべてのコードをC ++でレンダリングできることが判明しました。 実際、C ++では、データモデル、コントローラー、ビジネスロジック、ネットワークを操作するための補助ライブラリ、データベース、解析などを実装できます。 JavaおよびObjective-Cに残っているのは、プラットフォーム固有のビューにボタン\リスト\ラベルを描画することだけです。 そして、このアプローチは、すべてを2回書くよりもはるかに実用的です。

Rachel ChengとMichael VanLoonがBoostライブラリについて話します。Boostライブラリは、新しいC ++標準への影響、古いコンパイラの状況が原因で立ち往生している人々の新機能への「橋渡し」についてです。 レポートの前半は、C ++ 11およびBoostのスマートポインターとRAIIに当てられています。これは、まだ知識のない人にとってトピックを理解する良い機会です。 2番目の部分では、列挙、コンテナの初期化、正規表現、ストリーム、ネットワークの操作など、Boostの機能の一部について説明します。 結果は、Boostライブラリーに対するいくぶん理想化されたodeでしたが、ライブラリーは本当に優れているので、なぜそうではありません。
