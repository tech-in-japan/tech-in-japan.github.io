Flash'aをiPhone / iPadに転送する





iPhoneやiPadで彫刻をひねりたくないフラッシャーはどれですか？

簡単なように思えますが、手順にはまだ多くの「落とし穴」が含まれています。 多くの人は、アップルのフラッシュアプ​​リケーションの速度が遅くなるのを見て幸せではありませんでした...しかし、ネットワーク上で最も完全なFlash2iOSになる前に、すべてがうまくいきます！



そのため、 Adobe Flash CS5.5で良い結果を出します。 若いバージョンでも可能ですが、ここでは保証なしです。



美しいものでFLAファイルを開きます 。



[ ファイル ] -> [ 公開設定]を選択します 。

プレーヤーをAIR for iOSに切り替える







次に、 ファイル -> AIR for iOS設定に移動します

最初のタブを自分で処理し、[ 展開 ]タブに移動します。



Apple Developer証明書の所有者である場合、証明書を挿入します。そうでない場合、 「トレーニング」証明書をダウンロードします（アプリケーションはジェイルブレイク対応デバイスでのみ動作します）。 2つの必要なファイルを置き換え、パスワード1234を入力します。







すべての設定が完了したら、[ 公開 ]をクリックします。 運がよければ、スライダーを数分間観察した後、IPAファイルが正常に作成されたことを示すメッセージがポップアップ表示されます。

しかし、この段階の多くの人々は楽しい問題を抱え始めています。



Windowsでは、メッセージがポップアップ表示されます。



Error occurred during initialization of VM

Could not reserve enough space for object heap

Could not create the Java virtual machine







今夜書かれたJavaのラッパーでのみ扱われます（ダウンロードしてインストールします。キリル文字のパスには馴染みがないため、「マイドキュメント」からIPAを公開しないことをお勧めします）。



OSXでは、次のように表示されます



Unexpected or unknown element or attribute in the

application descriptor file. ...







ここで笑います。 「失敗した」XMLファイルをテキストエディターで開き、バージョンタグ全体（<version> 1.0 </ version>）を削除する必要があるたびに-処理を記述するのを忘れていたためです！



さらに、それはすでに簡単です。 結果のIPAファイルをiTunesライブラリに追加し、デバイスと同期し、そのアイコン（何か、置いていない場合は白）を見つけて、楽しい光景をお楽しみください。



残念ながら、ハードウェアアクセラレーション設定を使用したすべてのダンスは、まだ目に見える視覚効果を与えていません。 Adobeがこの面で昼夜を問わず機能することを期待しましょう。



誰かがトピックに関する結果を得るのにさらに困難がある場合-コメントで議論しましょう、投稿に資料を追加しましょう。



更新：ところで、何らかの理由で元のFLAがない場合（たとえば、InDesignからインタラクティブレイアウトをエクスポートする場合）、既製のSWFファイルをIPAに「ラップ」することができます。 これは、 ADT Helper Toolを使用して実行できます。



また、このユーティリティはAdobeパッケージ全体を必要とせず、動作するのにAIR SDKのみが必要です。AIRSDKはここから無料でダウンロードできます-www.adobe.com/special/products/air/sdk
