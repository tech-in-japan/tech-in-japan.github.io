詳細： ハンドルはAutoCADセッション間で永続的であるため、後で図面の更新に使用する必要のある外部ファイルに図面情報をエクスポートする必要がある場合、オブジェクトにアクセスする最良の方法です。 データベース内のオブジェクトのObjectIdは、データベースがメモリにロードされている間のみ存在します。 データベースが閉じられると、オブジェクトに割り当てられたオブジェクトIDは存在しなくなり、次回データベースを開いたときに異なる可能性があります。



データベースを操作する



通常、データベースの操作はトランザクションを使用して行われます。 オブジェクトがデータベースから要求され、変更され、コミットトランザクションが保存されます。

トランザクション中、3つのForRead、ForWrite、ForNotifyモードのいずれかでデータベースからオブジェクトが要求されます。

最初の2つの目的は明らかであり、3番目は何らかの形でイベントのメカニズムに使用されます。

ForWriteモードでは、autocadはトランザクション内の変更を破棄できる追加のオブジェクトを作成します。

「ForRead」として開かれたオブジェクトを変更する場合、そのUpgradeOpen（）メソッドが呼び出されます。

既に変更モードで開いているオブジェクトでこのメソッドを呼び出すと、メソッドは例外をスローします。



ObjectIdによってPolylineオブジェクトを取得する例

更新：段階的なデバッグの問題を解決しました：

3.エディターでコマンドを送信する
一部のアクションは、Autocad .NET APIに実装されていないか、コマンドラインで実行する方がはるかに簡単です。

コマンドを実行する最も簡単な方法は、ZoomExtentsの例である関数を実行することです。 

 var doc = Application.DocumentManager.MdiActiveDocument; doc.SendStringToExecute("_zoom _e ",false,false,true);


AutoCADでは、スペースはEnterに相当します。そのため、末尾のスペースなしでコマンドを送信すると、_eがeditor'eに入力され、さらに入力を待つことになります。



ただし、この方法は常に使用できるとは限りません。 SendStringToExecuteコマンドの原則は、コマンドによって呼び出された関数が完了した後にのみコマンドが送信されるということです。 したがって、最初にこのコマンドを呼び出してから、たとえば、ユーザーが図内のオブジェクトを選択するように要求すると、2行が選択関数「_zoom」、「_ e」に送信され、誤ったオブジェクトとして認識されます。

すぐに実行されるアナログを探す必要があります。 この場合： 

 object acad = Application.AcadApplication; acad.GetType().InvokeMember("ZoomExtents", BindingFlags.DeclaredOnly | BindingFlags.Public | BindingFlags.Instance | BindingFlags.InvokeMethod, null, acad, null);
