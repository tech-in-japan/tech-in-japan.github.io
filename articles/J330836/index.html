テクノロジーと非IT：S / 4HANAが卸売で使用される方法と理由





現代の店舗には複雑なインフラがあるとすでに言っています 。 しかし、卸売取引に携わる企業についても同じことが言えます。複雑で高度なITソリューションも使用します。



HabréのSAP S / 4HANAの開発者は、システムがビッグデータとどのように機能するかについて既に話しました 。 今日は、この製品が卸売業界でどのように導入されているか、どのような間違いがあり、ビジネスが達成できる結果について話します。



S / 4HANAが必要な理由



S / 4HANAは、SAPプラットフォームおよびプレインストールされたSAPソフトウェアに基づくソリューション用に最適化された機器を含むソフトウェアおよびハードウェアの複合体です。 ビジネス上の意思決定を行うための分析情報を迅速に取得するためのシステムが必要です。 以下では、使用方法について簡単な言語で説明します。



多くの場合、卸売取引では、データ分析の状況は次のとおりです。管理者が特定の問題に関心がある場合、部下に報告を求めます。 次に、同僚は、場合によっては異なる都市の支部でも連絡を取り、必要な情報を手動で検索し始めます。



情報はさまざまなシステムからダウンロードされ、手動で補足され、通常のドキュメント（通常はExcel）にまとめられます。 次に、会社のリーダーがこのレポートを分析し、いくつかの重要なビジネス上の決定を下します。 プロセス全体は1日から数週間かかることがあります。



ここでより効果的なアプローチは、S / 4HANAのようなシステムを使用することです。これにより、マネージャーは重要なメトリックを設定し、コンピューター画面、モバイルデバイスでリアルタイムに変更を確認し、さまざまなレポートを即座に生成できます。







システムの実装方法



ビジネスの成長に伴い、さらなる開発にはビジネスプロセスの変更と自動化の強化が必要になる状況が発生する場合があります。そうしないと、重要な決定を下すのに十分なデータが得られなかったり、効率が悪いために計画を遂行できなくなったりします。



企業はこれを単独で処理できますが、自動化システムを選択して卸売会社に実装する問題を解決するのはかなり困難です。 そのため、 SAPソフトウェアソリューションの実装に特化した部門全体を持つPilotなどの専門企業は、通常、このようなプロジェクトに関与しています。



外部のアナリストはビジネスを研究し、どのビジネスプロセスと自動化の方法について推奨を行い、これをどのツールで行うべきかを決定します。 このため、既存のプロセスは、現在動作している形式で詳細に説明されています。 次に、プロセスが最終的にどのように見えるべきかを正確に理解する必要があります。 これは、問題が最も頻繁に発生する場所です。







エラーと解決策



S / 4HANAのような複雑なシステムを常に導入しても、望ましい結果が得られるわけではありません。 多くの場合、これの理由は、会社のトップと普通の従業員が犯した間違いです。 以下にそれらのいくつかをリストし、それらを回避する方法について説明します。



間違い1：システムの誤用



経営幹部は、ビジネスソリューションの品質を向上させるのではなく、会社にとって重要だと考えるタスクを実行するためにS / 4HANAを実装したい場合があります。 さらに、実際にはこれはまったくそうではないかもしれません。 その結果、パイロットの専門家が実施した実装プロジェクトの1つで発生した状況と同様の状況が発生する場合があります。



顧客企業の経営陣は、進行中のプロセスでユーザーの日常業務に使用される約50のレポートを作成することを決定しました。 同時に、開発と改良に割り当てられたプロジェクト予算をほぼ2倍にする必要がありました。 標準レポートの使用に関するコンサルタントの意見は考慮されていません。



1年間の運用後、ほとんどのレポートは使用されなかったことが判明しました。これは、プログラムの起動に関する統計から明らかでした。 彼女の分析によると、ユーザーがシステムに慣れた後は、標準レポートで十分でした。 そのため、新しいものの開発と実装のためのお金が無駄になりました。







間違い2：奇跡を待つ



実装におけるもう1つの極端な点は、新しいシステムだけでビジネスのすべての問題を解決できるという事実に依存することです。 これは特効薬ではありません。S/ 4HANAを使用すると、重要な情報を取得して、重要な決定をすでに行うことができます。 しかし、システム自体はリーダーのためにこの決定を生成しません。



その結果、会社がS / 4HANAを実装し、経営陣が作業結果の改善を待っており、このテーマについて美しいプレスリリースが発行されていることが判明する場合がありますが、主要なユーザーはシステムを使用する意味がわかりません。 次の内容のフレーズを聞くこともあります。「当社のプレスリリースでは、計画された時間と予算で新しいシステムを非常にうまく実装できたと述べていますが、ビジネス自体の正確な変更点はわかりません。」



新しいソフトウェアではなく、特定の企業のタスクに適した管理システムを実装することが重要です。プロのコンサルタントを集めずに1つを選択するのは簡単ではありません。



間違い3：不十分なユーザー情報



ほとんどの場合、人々は変化をあまり好きではありません。 特に、1つのビジネスプロセスと使用しているプログラムから新しい、なじみのないものに再学習して移行する必要がある場合。 これは普通の従業員とマネージャーの両方に適用されます。 さらに、後者の場合、このような人々は実際の権限を持ち、ビジネスプロセスを新しいソリューションに変更せずに、同じS / 4HANAを古い、最適ではないが馴染みのあるプロセスに再フォーマットするか、プログラムにレポートの作成を強制するため、これはさらに危険です古い形式で。



SAPなどの大規模ベンダーの製品を実装すると、通常のビジネスプロセスを文字通り壊す可能性があり、これは常に困難な段階です。 しかし、最終的に、同社はベストプラクティスと国際的な経験に基づいたソリューションを手に入れました。結局、世界中の何万もの企業がそのようなERPシステムを使用しており、開発者は幅広い経験を持っています。



少なくとも自動化の度合いが上がると、しばしば削減につながるため、ライン担当者の経験も非常に正当化されます。 しかし、これはビジネスの発展に伴う避けられないプロセスであるため、「マシンの反乱」について話す必要はありません。



実装プロジェクトの準備段階でのチームの主なタスクは、ユーザーに会社の新しいビジネスプロセスを説明し、これらのプロセスで各チームプレーヤーが果たす役割と手順を正確に明確にすることです。 システムのテスト段階であっても従業員をトレーニングプロセスに参加させると、新しいビジネスプロセスがどのように機能するかについての知識をすぐに伝達できます。 また、ユーザーが新しいツールの導入が彼の仕事とさらなる昇進にどのように役立つかを見た場合、これは学習の追加の動機になります。



期待できる結果：いくつかの数字





結論として、実生活の小売企業を例に、SAPのERPシステムを実装することで達成できる結果について話しましょう。



新しいポイントを開く速度を上げる



私たちの経験から、SAP ERPシステムの実装により、成長している小売チェーンが新しいポイントを開く速度を大幅に向上できることが示唆されています。 そのため、1年（2013年）で最大のオンラインハイパーマーケットの1つは、自社のサイバーマーケットの数を29に、アウトレットを130に増やし、ロシアで最も急成長しているオンラインストアになりました。



また、同国北西部の「自宅」にあるオフラインストアの小売チェーンは、オープンする店舗の数をほぼ50％増やすことができました。 同様に、ウドムルチアの同様の会社は、新しい店舗の開店時間を80％短縮することができました。



より速いビジネスプロセス



Udmurtiaの同じ小売ネットワークでは、新しいシステムの導入により、倉庫で商品を処理する時間が20％短縮されました。 メトロポリタンチェーンの電気店でも結果が得られ、親組織と営業所間のデータ交換の効率が向上しました。 アルコール飲料の大手小売業者は、基本的な業務を20％加速しました。



レポートの最適化



上記のERPシステムを使用する電気店のチェーンは、商業、財務、会計情報の信頼性を30％向上させました。 アルコール製品の卸売に関与する別のお客様の管理レポートを作成するためのスケジュールは同じ数字だけ短縮され、コンビニエンスストアチェーンではこの時間を20％短縮できました。 別の同様のネットワークにより、レポート期間の締め切り期間が20規制日に短縮されました。



結論：誰がそれを必要とするか



実際、当社の実際の経験から、すべての企業がS / 4HANAを実装すべきではないことが示唆されています。 まず、中小企業にとってそれほど負担をかける必要はありません。Excelで十分な場合もあります。 大企業の場合、そのような製品の導入にはある程度の準備が必要です。



そのため、実装プロジェクトの開始前に、アナリストは顧客のビジネスと、会社のすべてが現在どのように機能しているかを詳細に調査します。 S / 4HANAの実装が最大の効果をもたらすことができるように、顧客が何をする必要があるかについての推奨事項を受け取る状況が考えられます。その後、彼は最初に準備作業を実行する必要があります。
