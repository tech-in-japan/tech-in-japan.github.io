BootCampを通じてMacBook Air 2013にWindowsがインストールされないのはなぜですか？

簡単なケースが1週間どのように引っ張られたかの物語。

しかし、それは興味深く有益なものであることが判明しました。







2011年にMac Airを購入しました。 Mac OSでは、タイピングとグーグルでそれを見つけました。 Windowsでのいくつかの問題を解決するために、VirtualBoxとVMware Fusionを使用しました。 Fusionは私に完全に適合しました。 NTFSで記録するために、Paragonからドライバーを購入しました。 私は自分がMacの専門家だとは考えられません。 しかし、私はラップトップ自体が好きです。



かつて、親友からWindowsをMacに搭載するように頼まれました。 彼らは仮想マシンが好きではなく、OS Xを習得できませんでした。 融合していません。 そして私には彼らを非難する権利はありません。



よく読んで、BootCampを使用してMacをWindowsラップトップに変える絶好の機会があると聞きました。 30分間でこれを行う方法については、インターネット上に十分なステップバイステップの説明があります。 人が嘘をついているとは言えませんが、少し違ったやり方をしました。



調査の過程で、かなりの量のユーザーがBootCampに問題があることが判明しました。 自分で答えを見つけて、グーグルで検索して、1か所に集めました。自分を忘れないように、他の人のために時間を節約するためです。



BootCampは非常に簡単に機能します。 現在のディスクパーティションからピースが切り取られ、Windowsがインストールされる新しいパーティションが作成されます（BootCampという名前）。 したがって、Windowsは仮想化なしでハードウェア上で直接動作し、高速化を意味します。 Mac OSは消えませんが、それに戻るためにはコンピューターを再起動する必要があります。 BootCampのメリットは、OS Xでこのプロセスを準備することです。その後、Windowsを自分でインストールする必要があります。



Windowsは、BootCampなしで個別にインストールできます。

しかし、正当な理由なしにこれをしないでください。 以下にその理由を説明します。



最初に、MacBookAir 6.1 11-inch 2013に出会いました。

最新のMacBook Airs。

6.1はモデル番号です。 彼の知識はいくつかの場合に役立ちます。



以下は、システム要件とラップトップのモデルを見つける方法に関する情報です。

support.apple.com/kb/HT5634?viewlocale=en_US



MacBook Air 6.1 11-inch 2013では、Windows 7.8 64ビットのみがBootCampを介してインストールできることに注意してください。 他のOSが提供されない理由は説明されていません。

BootCampウィザードは以下を提供します：

  1. Windows 7以降のインストールディスクを作成します。
  2. 最新のWindowsサポートソフトウェアをダウンロードします。
  3. Windows 7をインストールします。






多くの人がWindows 8を好まないのは秘密ではありません。ユーザーにとっては、Windows 7は馴染み深いものであり、Windows 7のインストールを提案してくれてうれしかったです。



インストールを開始するには、BootCampにクリーンフラッシュドライブとWindows 7のISOイメージが必要です。

多くの場合、手に「12 in 1のISOイメージ」があり、BootCampが好きではないという問題があります。

問題はディスクのライセンスされた清潔さではなく、その後のシステムブートに必要なファイルのイメージが存在しないことにあることがわかりました。



ISOディスクイメージの\ efi \ boot \イメージ内のディレクトリを確認します。

bootx64.efiファイルが必要です。

正直に言うと、ISOのディレクトリは異なる方法で呼び出される場合がありますが、BootCampは次のようになります。



なぜEFIが必要なのですか？



しかし、Macには通常のBIOSがないためです。

BIOSは廃止されました。 ダウンロードが遅くなり、2 TBを超えるパーティションを使用できなくなります。

Appleがそれを取り除いたのは論理的です。 MacBook Air 2013は、UEFI 2.0と互換性があり、OS XとWindowsの両方のEFIブートをサポートする最初のMacです。 サポートだけでなく、すでに機能しています。 MacにはOS X 64ビットEFIが搭載されています。



それが、64ビットWindowsのみが必要な理由です。



UEFIをインストールするためのマイクロソフトの要件：

technet.microsoft.com/en-us/library/hh824898.aspx



それだけでなく、BootCampを開始する前に、Macを更新することを強くお勧めします。 他のアップデートにはEFIおよびSMCのアップデートが含まれる場合があり、これらはインストールおよび将来の作業に役立ちます。



もちろん、私はこれをすべて知りませんでした。 フラッシュドライブを挿入し、ISO Windows 7-64ビットへのパスを示し、BootCampファイルのダウンロード（約500MB）をほぼ1時間待った後、Windowsパーティション用のスペースを割り当てるよう提案されました。



パーティションマネージャがメインパーティションを2つの部分に分割することにどれほど迅速かつ明確になったかに非常に満足しています。 Windowsの世界では、別のプログラムが必要で、多くの時間を費やし、ディスク上のデータを怖がっていました。

パーティションを作成した直後に、コンピューターが再起動し、フラッシュドライブ自体から起動し始めました。 Windowsの一般的なインストールが開始されました。 プロセスは非常にスムーズに進み、すぐにコンピューターの名前を入力するように求められました。 しかし、これを行うには失敗しました。



キーボードもトラックパッドもUSBマウスもUSBキーボードも機能しませんでした。



Windows 7にはUSB 3用のドライバーはありません。MAcBookAirには、最も怪しいUSB 2ポートはありません。

Windows 8のWAIKに基づいて作成されたWinPEフラッシュドライブから正常に起動し、キーボードとトラックパッドが機能しました。 インストールされたシステム上でドライバーを置き換えてレジストリを修正する試みは失敗しました。



肘は近かったが...



ディスクレイアウトダイアログでドライバーを再インストールし、手のひらをオフにしようとしました。 Shift + F10キーの組み合わせは非常に便利です。 彼女はコマンドラインのあるウィンドウを開き、そこでコマンドを実行することもできます。

Windows 7の状況は絶望的ではありませんが、残念ながら、このシステムは簡単にインストールできませんでした。



解決策があります：

Discussions.apple.com/thread/5742492?tstart=75



別のフラッシュドライブとMicrosoftの追加ソフトウェアが必要になります。

Windows PEを既に構築している人は、 DISMについて知っています。 残りは浸透し、指示に明確に従う必要があります。 自動化は行われず、すべてをペンで行う必要があります。



このオプションが気に入らない場合、他にもあります：

  1. EFIを忘れて、Windows 7をレガシモードで使用します。
  2. Windows 8をインストールします。


Windowsをインストールしようとしたときに、BootCampから再起動した直後にキーボードとトラックパッドがあることに驚きました。



どんな奇跡と魔法？

一度、再起動後に、起動可能なUSBフラッシュドライブのようにハードドライブを慎重に接続したままにせず、ダウンロードは外部ドライブのみから開始し、インストール画面の代わりに黒い画面しか表示されませんでした。

ミスを認識し、フラッシュドライブから再起動すると、すでに最初の段階ではキーボードやトラックパッドがなかったので、再び驚きました。



このすべてが私を少し魅了し、素晴らしい理由を見つけました。

BIOSをエミュレートし、有効なトラックパッドとキーボードを使用してWindows 7のインストールを開始するレガシーモードを設定できるコマンドがあります。



Mac OSターミナルで実行する必要があります。



diskutilリスト

sudo bless --device / dev / disk0s4 --setBoot --legacy --nextonly



--nextonlyは、モードの使用が1回だけであることを示します！ これがBootCampの仕組みです。 神に感謝しますが、それは容易になっていませんが、すべてが明確です。



視覚的には、MacのBIOSモードはEFIブートとは異なり、灰色の画面がしばらく表示された後、点滅するカーソルが黒い背景に表示されます。



希望する人は、私のyoutubeビデオでダウンロード速度の違いを推定できます。



再起動時にMacが出す音を聞くのにうんざりしている場合は、StartNinjaを配置できます。

www.allvu.com/index.php/products/startninja.html

このプログラムは、Macの起動音をオンまたはオフにすることができます。 無料



Windowsで使用されているUEFI /レガシーブートモードを確認するために、小さな「 detectefi 」プログラムがあります。

ソース：

xcat.svn.sourceforge.net/svnroot/xcat/xcat-core/trunk/xCAT-server/share/xcat/netboot/windows/detectefi.cpp

コードは非常にシンプルで、文字通り3行です。 実行可能ファイルを見つけるか、自分でビルドすることができます。



Windows 7のインストールに関する問題は、私だけでなく発生します。 そして、人々は個人的な経験をAppleサポートサイトで共有します。 ほとんどのヒントは、すみませんが、サイトの読者と同じアマチュアです。 アップル社の代表者は回答に見つかりません。



だれかが5番目のBootCampではなく4番目のBootCampを使用してWin7をインストールすることができたという餌食になりました。 当然、良いことは何もありませんでした。

別の方法で試しましたが、Windowsの一部のデバイスはまだ不明のままです。

VEN_14E4およびDEV_1570を備えたAppleカメラはどこにも存在しません！



さまざまな種類のドライバー探偵の広告に屈しないでください。 最良のケースでは、彼らはあなたにあらゆる種類のバーを提供します。 そして、彼らにはドライバーがいません。 Appleにはドライバーがあります！



そのため、BootCampなしでのインストールははるかに困難です！



公式のBootCampページに戻ると、MacBook Air 2013にインストールするには、BootCampの作業中に作成されたキットを使用する必要があることが明確に示されています。

support.apple.com/kb/HT5634?viewlocale=en_US



実行可能ファイルBootCamp.exeは5番目のバージョンのままですが、フラッシュドライブを作成する過程で、BootCamp5にはないモデルのドライバーがダウンロードされ、無料でダウンロードできます。

その結果、Windows 8を使用し、指示に従って厳密に動作する場合、Windowsは非常にスムーズにインストールされ、すべてが機能します。



Bootcamp.exeは、キーボードとトラックパッドの設定を制御します。 Macのファンクションキーは、Windowsの場合と同様に機能するように構成できます。Fnキーと組み合わせて、追加の機能を呼び出すことができます。



起動時にBootCamp.exeがRun autorunキーのAppleKeybMngrの値をチェックするのは面白いです。 レジストリに値がない場合、BootCampは何もしません。



2番目のMacbook Airは13インチでした。 モデル6.2 Windows 8がインストールされ、すべてのデバイスが検出されましたが、数秒ごとにスキャンの不快な歪みがありました。 歪みは非常に不快であるため、コンピュータの使用は完全に不可能でした。

苦しみ、最終的にIntelドライバーの代わりにMicrosoftから互換性のあるドライバーを選択しました。 その上で問題は解決しました。
