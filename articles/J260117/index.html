VKontakteは、AppleがiOS用のアプリケーション更新の拒否を送信したため、脆弱性をクローズしました

5月中旬以降、iOS用のモバイルアプリケーションの開発者は、Appleの要求に応じてVKontakteを介した認証をオフにするか、「松葉杖」を使用する-プルリクエストを強制されました。 Appleモデレーターの背後にある理由は簡単でした：

モデレーターは、vKontakteを介してApp Storeのアプリストア選択ガイド（ App Storeレビューガイドライン ）の条項22.4へのリンクを含む承認画面のスクリーンショットを添付することにより、更新を拒否しました。







承認は、アプリケーションが使用する唯一のvKontakte API関数です。 同様の失敗が、WWDC-2015の1か月前にさまざまなiOS開発者に起こり始めました。WWDC-2015では、Apple Musicのストリーミングプラットフォームが発表されました。これは会社の完全に論理的かつ合法的なステップです。



GitHub での問題の議論は 28日前に始まりました。



AIN.ua出版のジャーナリストのリクエストの後、ソーシャルネットワークはすぐに問題に対応しました。 文字通り、 出版の 15分後、VKontakteのプレスサービスGeorgy Lobushkinの責任者は、同社が対策を講じ、問題を修正したことを発表しました。 質問は議題から完全に削除され、翌日、いくつかのチームのiOS開発者によって確認されました。 VKontakteのプロダクトマネージャーであるMikhail Boldov氏は、GitHubのスレッドディスカッションスレッドで、SDKを更新する必要はありません 。
