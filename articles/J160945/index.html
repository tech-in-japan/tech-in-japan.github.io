デフォルトでUbuntuで暗号化された（TrueCrypt）Windowsをインストールする

あなたはする必要があります：あなたはパスワードなしでWindowsの内容を見ることができないように、同じマシンにWindowsとUbuntuを同時に置きます。 CD-ROMなし、USBのみ。

Ubuntuの仮想マシンをさまざまな理由で使用するのは好きではありませんでした。主な理由はパフォーマンスの重大な問題です。

解決済み：デフォルトでUbuntuがロードされ、Windowsがブートパーティションに配置されるように、2つのOS（WindowsとUbuntu）を並列に配置します。



最初に準備する必要があります。 必要なもの：十分なサイズのフラッシュドライブ2台。

Ubuntuを1つに配置します。

Windows用USB Ubuntuの作成手順

ユニバーサルUSBインストーラーLinuxのダウンロード

Ubuntuイメージをダウンロードする

いくつかのボタンを押して（ISOへのパスを指定して開始）、これでフラッシュドライブの準備ができました。



Windowsを別のものに置きます：

Windowsの指示

コンピューターのすべてのドライバーを同じUSBフラッシュドライブ（後で検索しないように）とTrueCrypt暗号化プログラム（無料！オープンソース！）にコピーしますダウンロード



つまり、TrueCryptプログラムを使用してWindowsパーティションを暗号化します。 Linuxブートローダーを最初にロード（grub）し、必要に応じてTrueCryptブートローダーをキャプチャし、システムの暗号化されたパーティションからWindowsをロードするスキームを取得したいと思います。 この場合、暗号化はオンザフライで実行されます。つまり、いつでもディスク上のデータは安全に保存されます。



今、私たちは武装し、完全に準備されています。 次に、以下の手順を印刷するか、タブレット/ラップトップを持って行ってください。



手順1.ディスクのパーティション分割


UbuntuからUSBフラッシュドライブから起動します（Ubuntuモードを試してください）。

GPartedプログラムを起動します（名前を入力するプログラムで）

目的のドライブを選択し、msdosスタイルで新しいパーティションテーブル（ 説明 ）を作成します。



このようにディスクを分割します： 

 / dev / sda1-将来のWindows用のパーティション
タイプ：主要党員
ファイルシステム：NTFS
サイズ：少なくとも30Gb（Windowsシステムパーティション。150Gbを配ることをお勧めします。暗号化する必要があるすべてのWindowsプログラムもそこにあるためです）

 / dev / sda2 Ubuntuブートパーティション（そこでマウント/ブートします）
タイプ：主要党員
ファイルシステム：ext2
サイズ：512mb

 / dev / sda3 LBA1-セクションコンテナーの合計
タイプ：論理ブロックアドレッシング
サイズ：ここで無制限はあなたの公共システムになります

	 / dev / sda4 SWAP
	タイプ：LBA1内のセクション（拡張パティション）
	ファイルシステム：SWAP
	サイズ：RAMのサイズと同じ（たとえば16Gb）

	 / dev / sda5 Ubuntuメインパーティション（/）
	タイプ：LBA1内のセクション（拡張パティション）
	ファイルシステム：ext4
	サイズ：ここで無制限は、パブリックUbuntuシステムとそのルートになります。
	
	 / dev / sda6 2つのシステム間でファイルを交換するためのオプションのパーティション-このパーティションは暗号化されず、両方のシステムがアクセスできます。
	タイプ：LBA1内のセクション（拡張パティション）
	ファイルシステム：NTFS
	サイズ：オプション。


必要に応じて、他のセクションが存在する場合があります（特にUbuntuのインストール時には、何かを個別に強調表示することができます）

私たちは何をしましたか？

  • Windowsのメインパーティションを最初のパーティションとして選択したので、問題なく起動し、ディスク自体を分割するような愚かな100mbパーティションを作成する必要はありません。
  • 別のブートパーティションを割り当てました。これは非常に重要です。なぜなら、Windows暗号化の直後に、タンバリンと踊らずにUbuntuシステムにアクセスするのが便利だからです。




ステップ2. Ubuntuをインストールする


Ubuntuモードを試して起動し、デスクトップからウィザードを使用する

Ubuntuインストールセクションを選択し、それに応じて上記の分割に設定します。 

  /ブート（/ dev / sda2）
 スワップ（/ dev / sda4）
  /（/ dev / sda5）


Ubuntuを再起動します（インストール済み）

次のコマンドを作成します。 
sudo grub-install /dev/sda2 --force


詳細はこちら： habrahabr.ru/post/150209

そこで、Grubを/ブートパーティションに配置します



ステップ3. Windowsをインストールする


別のフラッシュドライブから起動し、最初のセクションにWindowsを配置します。

TrueCryptをインストールします（ちなみに、ローカリゼーションはwww.truecrypt.org/localizationsにあります。ダウンロードしてプログラムフォルダーに解凍します）

システムでセクションを暗号化します（プロセスは簡単です。重要な点のみを示します）。
  • 隠されたシステムのない基本的な暗号化
  • 暗号化領域：「Windowsシステムパーティションの暗号化」
  • 「マルチブート」があることを示します
  • 暗号化を続行し、回復ディスク（回復ディスクの.ISOイメージ）を取得します
  • 受信した.ISOをUSBフラッシュドライブに保存します。これは非常に重要です。 それなしでは、多くのさらなるステップは不可能です！


必要な回数再起動し、暗号化されたシステムを取得します。



原則として、この段階で落ち着くことは可能です。なぜなら、すでに次のことができるからです。

  • メインのTruecryptブートローダーを備えた暗号化システム。Escキーを押すと、GRUB2ブートローダーに制御が移ります。
  • 適切なセクションのUbuntu。




次の手順は、順序を変更し、デフォルトでUbuntu暗号化システムではなく、Windows暗号化システムをセカンダリにするためにのみ必要です。



ステップ4. GRUB2を構成し、MBRに配置する


Ubuntuをダウンロードする

ここで、保存されたISOの操作方法をGRUB2に教える必要があります。 これは次のように行われます。

結果のISOを/ bootにコピーし、名前を/boot/truecrypt.isoに変更します

コンソールを開き、godmodeモードに切り替えます。 

 sudo -i


次に、このために/ bootにmemdiskユーティリティを配置する必要があります。 

 apt-get install syslinux cp /usr/lib/syslinux/memdisk /boot


GRUB2を構成する 

 nano /etc/grub.d/40_truecrypt


次の内容で/etc/grub.d/40_truecryptファイルを作成します。 

 #!/bin/sh exec tail -n +3 $0 # TrueCrypt menuentry "Microsoft Windows" { insmod part_msdos insmod ext2 set root='(hd0,msdos2)' linux16 ($root)/memdisk iso raw initrd16 ($root)/truecrypt.iso }


ここでmsdos2は最初のディスクの2番目のパーティションです（/ dev / sda2、はい、番号は同じです、msdosはmsdosにあるテーブルタイプがあるため、ここの構文です）

ギャップに注意してください-重要です。 ファイルにグループと所有者を実行する権限を設定します。 

 chmod og+x /etc/grub.d/40_truecrypt


したがって、すべての設定が完了し、実行可能ファイルがコピーされ、収集する必要があります。 

 update-grub


それをMBRに書き込みます。 

 grub-install /dev/sda


次に、再起動してプロセスに従います。

MBRにはGRUB2ブートローダーがあり、必要に応じてTrueCryptリカバリディスクイメージからTrueCryptブートローダーを起動できます。

パスワードを入力してもシステムに入ることができないと思われる唯一の理由は、

インストール中にGRUB2がTrueCryptキーを損傷しましたが、それらを回復することは難しくありません。

TrueCryptブートローダーで、F8を押して適切なアイテムを選択します。



ただし、TrueCryptブートローダーを上書きするとき、GRUBはすべてのフラグとトレースを完全に消去しなかったため、TrueCryptを介してロードすると、おそらく次のメッセージが表示されます。

隠しOSを作成しているようです。 これは正しいですか（y / n）


隠しシステムを作成するように招待すると、Yキーを押しても何も起こりません。Nキーを押すと、すべてが正常にロードされます。

この迷惑なメッセージを打ち負かすには、MBR内のTrueCryptのすべてのトレースを手動でクリーンアップする必要があります。

Ubuntuを起動して、次のコマンドを作成します。

  1. TrueCryptからテールがあることを確認してください 

     dd if=/dev/sda count=8 bs=1 skip=6


    「TrueCryp」を出力する必要があります

  2. MBRのTrueCryptマーカーを消去します。 

     echo 'grub<3tc' | dd of=/dev/sda count=8 bs=1 seek=6


    ここでは、単に/ dev / sdaと記述します-ディスクのMBRを修正するためです。 まさにそのような変位がどこから来たのか、誰かが知っていれば、私は見つけられませんでした、私はコメントを求めます。



メニュー項目を非表示にする必要がある場合（注意をまったく引かないように）、ここに説明があります

考えられる問題


  • 暗号化後に、追加の手順が取得されたかどうかに関係なくWindowsシステムが起動することを常に保証したい場合は、Ubuntuフラッシュドライブから起動し、これを行うことができます。



    バックアップを作成します。 

      dd if=/dev/sda of=~/truecrypt.mbr count=1 bs=512 dd if=/dev/sda of=~/truecrypt.backup count=8 bs=32256




    回復の場合のバックアップからの復元： 

      sudo dd if=~/truecrypt.mbr of=/dev/sda count=1 bs=512 sudo dd if=~/truecrypt.backup of=/dev/sda count=8 bs=32256
  • すべてを正しく行い、UbuntuをインストールしてからWindowsを暗号化した後、Ubuntuにアクセスすることはできません（ちなみに、何か間違っていると思われます）。Grub2ブートローダーを復元するには、次の手順を実行する必要があります。

    1. Ubuntuフラッシュドライブから起動する（Ubuntuモードを試す）
    2. コンソールで、次を入力します。 

       sudo -i mount /dev/sda5 /mnt mount /dev/sda2 /mnt/boot


      ブートローダーをMBRに配置した場合（ステップ4のみ） 

       grub-install --root-directory=/mnt /dev/sda


      または、まだ2番目のシステムとしてUbuntuを使用する場合は、 

       grub-install --root-directory=/mnt /dev/sda2 --force


      詳細： Grub2回復マニュアル





  • Grub2ブートローダーは、メインパーティション自体である/ dev / sda2にインストールされますが、TrueCryptブートローダーでEscを押した後、何らかの理由でアクセスできません。 TrueCryptローダーは、ブートフラグのあるパーティションのみを認識するという情報があります。 したがって、/ dev / sda2セクションにこのようなフラグを付ける必要があります。 Gpartedを開き、目的のセクションを見つけて右クリックし、フラグがあり、ブートフラグを選択します。その後、保存する必要さえありません-実行して再起動し、チェックするようです。
