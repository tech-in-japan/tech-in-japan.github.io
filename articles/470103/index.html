Creating a simple clicker game from scratch

In the past few years, games of such a genre as ‚Äúclickers‚Äù are gaining great popularity. I myself am very interested in playing them, but it is no less interesting to create a game - a clicker myself. Thanks to an engine like Unity, creating a masterpiece will not be difficult even for a simple beginner. Let's get down to business.



What should be done?



  1. Think about why and what should we click on?
  2. What should happen when we click?
  3. What should happen if we don‚Äôt click?


As I answered these questions:



  1. We must click anywhere on the screen.
  2. When clicked, the object should change.
  3. If ignored - a loss.


What will the game be about?



The player needs to choose either the sword or the shield in time, depending on what will fall from above. That is, the sword falls - we select the sword, the shield falls - we select the shield. If you make the wrong choice, the player simply loses. In words, everything is quite simple, but in implementation it is even simpler.



Where do I get a beginner to take a beautiful sprite and background? Answer: draw yourself.



There is nothing complicated, as we draw in Adobe Illustrator. Even if you do not have experience in drawing, then again we are not upset, as I will show you everything in detail.



What we need:



  1. Sword
  2. Shield
  3. The background
  4. Branch with leaves


That‚Äôs enough.



The technique is simple and I will show with the example of a sword:



  1. Draw a contour
  2. Fill with flowers
  3. Draw the shadows






And that's it, our sword is ready!



We also do it with others. Do not be alarmed if at the beginning one of your creations will seem to you far from ideal, because together they will look like a Van Gogh painting. Joke. Or maybe not.

We can say that the art lessons are over and we finally go to Unity.



First of all, let's throw our sprites, and drag the sword and shield into the hierarchy window, since we will need to create prefabs from them and not only.







  1. Add Rigidbody 2D and set the value to ‚Äú0‚Äù in the Gravity Scale field so that the shield stands in one place.
  2. Add Circle Collider 2D and check the Is Trigger checkbox to track actions.


With the prefab, the same actions, only you need to set the value ‚Äú1‚Äù in the Gravity Scale field so that the shield falls down. And drag it to the Prefabs folder. With the sword the same action. Done!



Next, decorate the game with sprites drawn earlier. The first thing we throw is the background.



  1. Place branches on top
  2. Add a log below
  3. Place a bridge in the middle






The main design secret is this: we will place branches on top and bottom, and on the ladder we will reduce the transparency value. Why do this?



The fact is that in this way we create a certain illusion of space. It will seem to the player that there is a front object, a back one and a horizon. And this is awesome!



Now create a button that will occupy the entire screen. To create a button in the hierarchy, click on UI -> Button. Further:



  1. Click on Color.
  2. Set the value "0" in the field A, which is responsible for the transparency of our button.
  3. We make sure that the button has become what we wanted.






In the same way we create a text to show our current score and record.

What is going on in these methods?



Start (). It starts every time we start the game or restart the scene. Everything that is written inside this method is performed only once. Therefore, we write there: an account equal to zero, the launch of coroutines (what is responsible for creating prefabs), the value of time.



Update (). Run once per frame. It is convenient to write down here that is constantly updated. In our case: an indicator of score and record, a random value n on which depends on what is created - a sword or shield.



BtnClick (). You may ask why this method is ‚Äúpublic‚Äù? We need it to be reflected in Unity itself and we can spell out the word ‚ÄúClick‚Äù in order to track which button we click on, although there is only one button, it will not hurt anyone to know a little more.







Here we write what should happen after pressing the button. When pressed, it should change from sword to shield and vice versa, set the time value to ‚Äú1‚Äù, since when losing it is reset to zero.



OnTriggerEnter2D (). Checks for collision of an object. If the tags of both objects match, then the score is increased by one, and the prefab is destroyed. If the tags are different, then a loss - time stops and the score is reset.



It remains to add these same tags. And here it is very simple:



  1. In the Inspector window, click on Tag.
  2. In the drop-down list at the very bottom, click on Add Tag.
  3. Click on the plus sign and name the tag as we want.






After creating, also click on Tag and select the desired one.



It remains to transfer the scripts to the necessary objects and fill in some fields. Create an empty object and add the ‚ÄúMain Script‚Äù script, and add ‚ÄúPlayer Script‚Äù to our sword and shield.
