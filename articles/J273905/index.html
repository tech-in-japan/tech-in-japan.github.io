DotNext 2015 in Moscow：境界を押し広げる

DotNextカンファレンスは、ロシアの.NETプログラマーの間ではもう紹介する必要がなくなった：ほぼ2年前にサンクトペテルブルクに登場してから、首都を征服し 、多くの参加者を集めました。 しかし、最近モスクワで開催されたDotNext 2015 Moscowで「それは何ですか」と説明する必要がない場合、新しい質問が発生します。昨年とはどう違うのですか？



この質問に対する答えは、過去の会議についての詳細な話にあります。











ラディソンスラブの有名なホールですべてが起こりました。 スタンドがロビーにある企業の中に、カスペルスキーが登場しました。 彼女は注目を集めて他の誰よりも独創的でした。カスペル​​スキーのスタンドでは、サムスンギアVRヘルメットのゲームをテストすることができましたが、そこから1か月も経っていないため、斬新さを試してみたい人が不足していませんでした。







しかし、人々がビジネスでこの会議に来るので、他のスタンドも怠けていませんでした。 DevExpressですべてが真剣に受け止められたとしましょう。彼らはバーチャルリアリティでそれを誘惑しませんでした。また、会社の名前は外部からの人々に語りませんが、DotNext訪問者にとってその製品は重要であるため、積極的にアプローチしました







Futuriceブースでは、ヘルシンキ、ベルリン、ミュンヘン、ロンドン、ストックホルムでの雇用の可能性について話すことができます。同社のオフィスはヨーロッパのさまざまな都市にあります。 DoturNextで定期的に講演しているFuturiceのビジネスディレクターであるMikhail Samarinは、会議が会社の有能な人材を見つける機会を彼に惹きつけているという事実を隠していません-彼はスタンドでだけでなく、ステージからもこれについて話しました。







そして、常に開発者の注目を集めていたJetBrainsブースは、今回も予想外に見えました。まさに会議の日に、会社がブランドを変更したので、親しみのあるロゴの代わりにブースに新しいロゴがありました。







昨年と同様、基調講演は英語で行われ、再び.NETの世界で最も有名なDino Espositoが基調講演を行いました。今回は、ドメイン駆動設計についての会話が行われました。 彼の表現方法は変わっていません。「10年以上前に2004年に出版されたEric Evansの本のおかげで、DDDについて話し始めました。 プログラミングの世界でこれが何を意味するのか理解していますか？ これはジュラ紀の本です。」 画像はクリスマス帽子で補完されました。



Espositoのスピーチには、ジョージオーウェルのアニマルファームへの言及と、ラスベガスの会議後、「妻が仕事のために来たことを妻に証明しなければならなかった」という話が含まれていました。私は自分のコードがエレガントであるという事実をもっと大切にしています」-しかし、同時に彼はメイントピックを忘れていませんでした。 また、彼のスピーチは、ドメイン駆動設計アプローチを盲目的にwasめたものでもありませんでした。







基調講演の後、全員が異なる部屋に散らばっていました。 同じMikhail SamarinがUniversal Windows Platformの登場の意味を説明しましたが、JetBrainsのDmitry Nesterukは、ほとんどすべての手動操作を自動化するボットの作成の基本について話しました。



最近立ち上げられたUWPを導入する必要があり、ボットの場合は何も変わらないように見えるかもしれませんが、Dmitryは彼らをより関連性のある新鮮な状況の例を示しました：SlackやTelegramのようなインスタントメッセンジャーの離陸は、ダイアログモードで役立つボットの全盛期をもたらしました。 また、ボットは直感的なトピックのように見えますが（スパムボットのアクションに遭遇した人は誰でも）、多くの新しいことを伝えることができます。たとえば、ボットがタスクの一部を単独で行うハイブリッドアプローチのアイデアは明らかではありませんが、特定の段階で、政府の手綱を生きたユーザーに移します（完全な自動化が必要な場合でも、不可能な場合は部分的であるよりも優れています）。







次の時間枠は特別になりました：Dino Espositoがドメインモデルについて話している間、別の部屋で、彼の本「Pro .NET Performance」で.NETの世界で知られるSasha Goldsteinがベクトル化について話しました。以前はDotNextでは発生しなかった英語でした。 ただし、これは、当時ロシア語の話が苦手な人が聞くものが何もなかったことを意味するものではありません。たとえば、 DevExpressの NikExpress Namestnikovが Xamarin.Formsについて話しました。これにより、モバイルポーティングは3つの主要なプラットフォームすべて（iOS、Android、Windows）に簡単にインターフェースを移植できます。 この技術の出現からわずか1年半が経過したため、最後のモスクワDotNextでは、長期の個人的な経験に基づいて誰も単にそれについて話すことはできませんでした。ここで、ニキータは、1年間のアクティブな作業の結果をまとめました-サービスが引き起こされた瞬間について正直に話す不便。



たとえば、iOSの単一行と複数行のテキスト入力フィールドが区別される場合、Androidでは単一のエンティティであり、その結果、普遍性を求めるXamarin.FormsはiOSの単一行にアクセスできません。 Nikitaによると、Xamarin.Formsの開発者は、マイナーアップデートに重要な変更を加え、それらについて明確に通知することすら好まないということです。 長所と短所についての正直な話は、出席者がサービスが具体的にどのようにプロジェクトに合っているかを理解するのに役立つものです。







昼食後、メインホールのレポートは本物のショーになりました。2人のスピーカーがいて、ステージ上で強力な対立が繰り広げられました。 ローマン・ベロフ（JetBrains）は、彼が「フォースのダークサイドに切り替えた」ことをすぐに正直に認めました。.NETは直接彼に馴染みがありますが、彼はJavaプラットフォームを好みます（恐らく、何百人ものアフィリエイトに直面すると言うのが恐ろしいです）。 反対に、 ドミトリー・イワノフは「明るい側」にいました。



一緒に、彼らは2つの世界の比較に関するレポート全体を作成しました-そして、彼らはすべてのスライドが「暗い」と「明るい」半分に半分になり、彼らの服でさえこれに対応するように細心の注意を払いました。 そのようなプログラマーのホリバーで通常行われるように、一方の当事者が他方の当事者に対して明白な優位性はありませんでした-しかし、彼らの対立の過程で、視聴者は両方について何かを学ぶことができました。 観客が友好的に笑った瞬間もありました：「そして、このオペレーターは「私は母に誓います！」と呼ばれています！」







その日の3番目のブロックには、その日の1番目のスピーカーが含まれています。JetBrainsのHadi HaririがHTTP / 2について話しました。 質問に興味がない場合は、「インターネットがない場合でもインターネットが高速である場合、プロトコルの新しいバージョンは非常に重要」と思われるかもしれませんが、レポートはすぐに説明しました。問題はスループットではなく、質問は待ち時間です。 巨大なファイルをすばやくダウンロードできますが、多数の小さなファイル（現在HTTPで作業しているときに処理しています）の場合はそうではありません。 各小さなファイルのダウンロードの開始の前に遅延がある場合、それは絶えず感じられます。 HTTP / 2は、この問題を解決し、HTTP 1.1の一部としてトリックに頼る開発者の生活を簡素化するように設計されています。 ただし、新しいバージョンのプロトコルの場合には、いくつかのトリックと困難があり、それらについても議論されました。 最後に、ハディはトピックをより深く理解したいすべての人、イリヤ・グリゴリックの本「高性能ブラウザネットワーキング」に推奨しました。



そして、この日2回目、サーシャ・ゴールドスタインは話をしました-今回はエラー処理の自動化について話をしました。 もちろん、彼が指摘したように、数百のクラッシュダンプを手動で解析できますが、これには多くの作業時間が必要であり、同時に多くの機械的な「コンピューターのような」作業が必要です。したがって、この作業をコンピューターに委ねることは理にかなっています。 レポートはこれを行う方法に専念しました。







最後から2番目のスロットには、トピックだけでなくスピーカーについても注目すべき2つのレポートが同時にありました。 「ビッグデータ処理と機械学習でのF＃の関数型プログラミング」は、機械学習がどれだけ積極的に宣言されているかという理由だけで意味があり、MicrosoftのDmitry Soshnikovがそれについて話しました。 .NET、会議は費用がかかりませんでした。







また、メインホールでは、 ミハイルシェルバコフ（Cezurity）が.NETで作業するときにWinDbgを使用することについて話しました。 彼のspbdotnet.orgは多くのプログラマーをコミュニティに変え、人々が互いに知り合い、経験を交換するため、サンクトペテルブルクの.NETコミュニティーでよく知られています。 しかし、他の都市に住んでいる彼のプロジェクトは、オンライン放送のおかげで役に立つこともあります。 ミハイルを認識するのは難しくありませんでした-彼のTシャツはそれ自身を代弁しました。







最後に、彼は「銀の弾丸症候群」についてのハディ・ハリリによる基調講演を閉じました-少なくともマイクロサービスで、少なくとも同じドメイン駆動設計で、あらゆる技術やアプローチに対して万能薬を宣言する傾向があります（ちなみに、ディノは興味を持って聞きました）。 表現にためらうことなく、ハディは「これが最終的にすべてを修正する」と彼らが次々と言ったことを思い出し、その後、新しい困難の突破口が生じました。 また、開発者が「Netflixが使い始めましたか？ Netflixとそのプロジェクトの仕様がまったく異なるという事実を本当に考えずに、私たちはそこにいるでしょう。」



聴衆の反応によると、彼らはまた、イノベーションが大好きで、「問題を完全に解決する」ことを望んでいるIT界の過剰な印象に対処しなければならないことは明らかでした。無限になる傾向があります。 Hadiのスライドはこれについて皮肉な引用で埋められていました-「ウェブ開発者のための飲酒ゲーム：google "word.jsを考えてください。その名前のライブラリを見つけたら-それを飲んでください」。



同時に、プレゼンテーションで明らかになったように、「私は新しいことは何も試さない」と言うことも選択肢ではありません。人々が新製品に期待しすぎているからといって、進歩が起こっていることをキャンセルしません。 イベントに慎重にアプローチし、大きな言葉ではなく、本当のメリットを評価しようとすることが残っています。







結果は何ですか？ DotNextは、前回の出来事（多くのハードコア、多くのJetBrains、Dino Espositoのヘッドライナー）、および他の何か：英語での多くのパフォーマンス（最大のものを含む：オープンとクローズ）、3ホールは、昨年の4つのコミュニティからのより多くのレポートではなく、要約できます-会議は発展しており、それとともにロシアの.NETコミュニティ全体が発展しています。



古典を言い換えると、外国のスターが招待された場合、誰かがそれを必要としていることを意味します。 英語のスピーチの需要は、ロシアの関連会社間のトピックの所有権のレベル、およびこのトピックへの関心（外国語では、人々は通常、彼らが本当に興味を持っているものに精通します）、および恋人同士で自分自身を締め出すことをためらうことの両方を示しています：人々は最高の報告を聞きたいです可能であり、残りは特定です。
