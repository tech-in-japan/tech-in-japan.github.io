マスターユアンマー、プログラミング本



コンピューターの世界にはデータしかありません。 データの読み取り、データの変更、新しいデータの作成ができます-しかし、データ以外には何もありません。 すべてのデータはビットの長いシーケンスとして保存されるため、互いに類似しています。



ビットは2つの状態を持つエンティティであり、通常は0と1として記述されます。 コンピューターでは、高電荷と低電荷、強い信号と弱い信号、またはCDの表面の光沢のあるマットな領域の形で生きています。 各情報は、ゼロと1のシーケンス、つまりビットとして表すことができます。



たとえば、数字の13です。10桁で構成される10進数システムの代わりに、2桁のバイナリシステムがあります。 数値の各位置の値は、右から左に移動すると倍になります。 番号13を構成するビットとその重み： 



0 0 0 0 1 1 0 1 128 64 32 16 8 4 2 1




2進数00001101、つまり8 + 4 + 1、つまり13です。



数量。



たくさんのビットを想像してください。 典型的な最新のコンピューターは、300億ビット以上をRAMに保存します。 通常、永続メモリ（ハードディスク）は、数桁大きくなります。





迷子にならずに作業するために、情報の単位を表す断片に分割できます。 JavaScriptでは、これらの単位は数量と呼ばれます。 それらの値はすべてビットで構成されていますが、異なる役割を果たします。 各数量には、その役割を決定するタイプがあります。 合計で6つの主なタイプがあります：数値、文字列、ブール値、オブジェクト、関数、および不定値です。



値を作成するには、その名前を指定する必要があります。 これは便利です。 建築材料を収集したり、それらを支払う必要はありません。 電話するだけです-op-pa、完了です。 それらは空から作成されたものではありません-各値はどこかに保存されており、膨大な数を使用したい場合、ビットが不足する可能性があります。 幸いなことに、これはすべてが同時に必要な場合のみです。 数量が不要な場合は、数量が解消され、使用されるビットが新しい数量の建築材料として処理に送られます。



この章では、JavaScriptプログラムのアトム—単純なタイプの変数とそれらに適用される演算子を紹介します。



数字



数値型の値、これは驚きです-数値。 JavaScriptプログラムでは、次のように記述されます 



 13




プログラムでこのエントリを使用すると、コンピューターのメモリに13という数字を表す一連のビットが表示されます。



JavaScriptは、固定ビット数（64）を使用して数値を格納します。 64ビットを使用して表現できる数量の数は制限されています。つまり、数自体も制限されています。 N桁の10進数の場合、書き込むことができる数字の数は10のN乗です。同様に、64ビットは2の64乗の数を表現できます。 これはかなりたくさんあります。



以前は、コンピューターのメモリが少なく、8ビットまたは16ビットのグループが数値の保存に使用されていました。 このような小さな数の最大数を誤って超えることは簡単でした。つまり、このビットのセットに収まらない数を使用しました。 今日、コンピューターには多くのメモリがあり、64ビットの断片を使用できます。そのため、天文学的な数字を扱う場合にのみこのことを心配する必要があります。



確かに、2 ^ 64未満のすべての数値がJavaScriptの数値に適合するわけではありません。 負の数値もこれらのビットに格納されます。したがって、1ビットに数値の符号が格納されます。 さらに、分数を保存できる必要があります。 これを行うには、ビットの一部を使用して小数点の位置を格納します。 数値の実際の最大値は約10 ^ 15であり、一般的にはまだほとんどです。



分数はドットを使用して記録されます。 



 9.81




非常に大きいまたは小さい数は、科学表記法で文字「e」（指数）とそれに続く度で書かれています。 



 2.998e8




これは2.998×10 ^ 8 = 299800000です。



10 ^ 15未満の整数（整数とも呼ばれる）を使用した計算は、正確であることが保証されています。 通常、分数の計算は行われません。 数π（pi）を有限の桁数で正確に表すことができないように、64ビットしかない場合、多くの小数部を表すことはできません。 悪いですが、非常に特定のケースでは邪魔になります。 これを念頭に置いて、分数を近似値として扱うことが重要です。



算術



数値を使用して実行できる主なことは、算術計算です。 加算と乗算は2つの数値を使用し、3番目の数値を与えます。 JavaScriptでこれを記述する方法： 



 100 + 4 * 11




記号+および*は演算子と呼ばれます。 最初は加算、2番目は乗算です。 2つの量の間に演算子を配置し、式の値を取得します。



また、この例では、「4と100を加算してから結果に11を掛ける」ことがわかりますか、それとも乗算が最初に実行されますか？ ご想像のとおり、乗算が最初に行われます。 しかし、数学のように、これは括弧を使用して変更できます。 



 (100 + 4) * 11




演算子-は減算に使用され、/は除算に使用されます



括弧なしで演算子を使用する場合、実行される順序は優先順位によって決まります。 *および/演算子は、優先順位が等しい+および-演算子よりも高い優先順位を持っています。 優先順位が同じ演算子を計算する場合、左から右に計算されます。 



 1 - 2 + 1


（1-2）+ 1として計算



優先順位について心配する必要はまだありません。 疑わしい場合は、括弧を使用してください。



すぐに認識されない別の演算子があります。 ％記号は、残りを取得するために使用されます。 X％YはXをYで除算した余りです。314％100は14を、144％12は0を返します。演算子の優先順位は乗算と除算と同じです。 「モジュロ除算」と呼ばれますが、より正確には「剰余あり」です。

ブール値



多くの場合、「はい」と「いいえ」、「オン」と「オフ」など、2つの可能性のいずれかを単純に示す値が必要です。 これを行うために、JavaScriptにはブール値型があり、 true



とfalse



（trueとfalse）の2つの値しかありません。



比較



ブール値を取得する1つの方法： 



 console.log(3 > 2) // → true console.log(3 < 2) // → false




記号<and>は、従来より少なくなることを意味します。 これらは二項演算子です。 それらを使用した結果、不等式が真であるかどうかを示すブール値を取得します。



文字列は同じ方法で比較できます。 



 console.log("" < "") // → true




文字列はアルファベット順に比較されます。大文字の文字は常に小文字の文字よりも「小さく」なります。 比較はUnicode標準に基づいています。 この規格は、あらゆる言語のほぼすべてのキャラクターに番号を割り当てます。 文字列を比較するとき、JavaScriptは文字を左から右に調べ、これらの文字の番号コードを比較します。



他の同様の演算子は、> =（以上）、<=（以下）、==（等しい）、! =（等しくない）です。 



 console.log("" != "") // → true




JavaScriptには、それ自体とは異なる値が1つだけありますNaN



（「not a number」）。 



 console.log(NaN == NaN) // → false




NaNは無意味な計算の結果であるため、他の無意味な計算の結果とは異なります。



ブール値自体で実行できる操作があります。 JavaScriptは、次の3つの論理演算子をサポートしています。



&&演算子は論理的な「and」です。 バイナリであり、適用される両方の量も真である場合にのみ、結果は真になります。 



 console.log(true && false) // → false console.log(true && true) // → true




演算子|| -論理的な「または」。 値の1つがtrueの場合、trueを返します。 



 console.log(false || true) // → true console.log(false || false) // → false




「いいえ」は感嘆符「！」を使用して書かれています。 これは、指定された値を逆にする単項演算子です。 ！trueはfalse 、! falseはtrueです。



論理演算子と算術演算子を使用する場合、括弧がいつ必要かは必ずしも明確ではありません。 実際には、これを知ることでこれを処理できます|| 優先順位は誰よりも低く、次に&&、次に比較演算子、そして他の全員が優先されます。 この順序は、次のような式で、括弧を最小限に使用できるように選択されました。 



 1 + 1 == 2 && 10 * 10 > 50




最後の論理演算子は単項でも二項でもありません-三重です。 疑問符とコロンを使用して記述されています。 



 console.log(true ? 1 : 2); // → 1 console.log(false ? 1 : 2); // → 2




これは、疑問符の左側の値がコロンで区切られた2つの値のいずれかを選択する条件演算子です。 左側の値がtrueの場合、最初の値を選択します。 falseの場合、2番目。



未定義の値



意味のある値がないことを示すために使用される2つの特別な値、nullとundefinedがあります。 単独では、情報を伝達しません。



値を返さないステートメントの多くは、単に何かを返すためにundefinedを返します。 未定義とnullの違いは、偶然に言語に現れ、通常は問題ではありません。



自動型変換



JavaScriptを使用すると、場合によっては非常に奇妙なプログラムを実行できます。 例： 



 console.log(8 * null) // → 0 console.log("5" - 1) // → 4 console.log("5" + 1) // → 51 console.log("" * 2) // → NaN console.log(false == 0) // → true




「間違った」タイプの値に演算子が適用されると、JavaScriptは、常に期待を満たしていない一連のルールを使用して、値を静かに目的のタイプに変換します。 これは強制と呼ばれます。 最初の式では、nullは0に変わり、「5」は5になります（文字列から数値へ）。 ただし、3番目の式では、+は文字列を連結（連結）します。そのため、1は「1」に変換されます（数値から文字列に）。



明白ではないものが数値に変わると（たとえば、「5」または未定義）、値NaNが返されます。 NaNを使用した後続の算術演算は、再びNaNを受け取ります。 この値を取得したら、ランダムタイプ変換が発生した場所を探します。



同じタイプの値を==と比較する場合、それらが同じである場合に真になるべきだと簡単に予測できます（NaNの場合を除く）。 ただし、タイプが異なる場合、JavaScriptは比較のために複雑でわかりにくい一連のルールを使用します。 通常、彼は一方の数量をもう一方の数量に​​変換しようとします。 ステートメントの片側でnullまたはundefinedが発生した場合、両側がnullまたはundefinedであればtrueを返します。 



 console.log(null == undefined); // → true console.log(null == 0); // → false




最後の例は、便利なトリックを示しています。 値にnullまたはundefinedの代わりに実際の値があるかどうかを確認する必要がある場合は、==または！=でnullと比較するだけです。



しかし、何かを正確な値と比較する必要がある場合はどうでしょうか？ 型をブール値に変換する規則では、0、NaN、および空の文字列「」は偽と見なされ、他のすべては真と見なされます。 したがって、0 == falseおよび“” == falseです。 自動型変換が必要ない場合は、さらに2つの演算子===および！==を使用できます。 最初は2つの値が完全に同一であることを検証し、2番目は2番目の値が完全に同一であることを検証します。 そして、比較“” === falseはfalseを返します。



予期しない結果につながる可能性のある予期しない型変換を防ぐために、3文字の比較演算子を使用することをお勧めします。 比較する値のタイプが一致することが確実な場合は、短い演算子を安全に使用できます。



論理演算子の短い計算



論理演算子&&および|| さまざまな種類の数量を非常に奇妙な方法で処理します。 次に何をすべきかを理解するために、演算子の左側の値をブール値に変換しますが、演算子とこの変換の結果に応じて、左側または右側から元の値を返します。



たとえば、|| trueに変換できる場合、左側の値を返します。それ以外の場合、右側を返します。 



 console.log(null || "user") // → user console.log("Karl" || "user") // → Karl




そのような演算子|| デフォルト値へのロールバックとして使用できます。 左側に空の値を返すことができる式を指定すると、右側の値がこのケースの代替として機能します。



&&演算子は同様に機能しますが、逆も同様です。 左側の値がfalseに変換されると、その値が返され、そうでない場合は右側の値が返されます。



もう1つの重要な特性は、右側の式が必要な場合にのみ計算されることです。 真の場合|| Xが何に等しいかは関係ありませんが、それが何らかのひどい表現であっても。 結果は常に真であり、Xは計算されません。 False && Xも機能します-Xは単に無視されます。 これはショート計算と呼ばれます。



条件ステートメントも同じように機能します。 最初の式は常に計算され、2番目と3番目の値から-結果として選択された式のみが計算されます。
