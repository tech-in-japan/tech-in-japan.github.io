Citrix NetScaler ADCの周辺-パート2：CitrixとMicrosoft

Citrixは長い間、さまざまな方法でMicrosoftと協力してきました。 これらの分野の1つは、Skype for Business（Lync）、Exchange、SQL Server、SharePointなどのWebテクノロジーを使用するMicrosoftソリューションのバランスをとり、最適化するためのNetscaler ADCアプリケーション配信コントローラーの使用です。







以前、マイクロソフトはフォールトトレランスにNLBテクノロジーの使用を推奨していましたが、このテクノロジーの多くの制限により、同社はCitrix NetScalerを含む推奨アーキテクチャにサードパーティのアプリケーション配信コントローラーを追加しました。



マイクロソフトの各ソリューションには、独自のアプローチが必要です。 Citrix NetScalerは、ほぼすべてのMicrosoft製品にフォールトトレランスとセキュリティを提供できます。 Citrix NetScaler内のどのテクノロジーを一部の製品に使用できるかを見てみましょう。 この可能性の説明は最も完全ではなく、主な機能についてのみ説明することを目的としていることに注意してください。 機能はさまざまな組み合わせで使用できます。



バランス調整



Microsoft Sharepoint



この場合、NetScalerはSharepointサーバーのバランスを取ります。 Web Application Firewallは、特定のWeb攻撃や企業情報漏洩から企業ポータルを保護します。 SSL操作をオフロードすると、サーバーインフラストラクチャの負荷が大幅に軽減されます。 フロントエンドの最適化とキャッシュは、ユーザーへのコンテンツの配信を大幅に加速できます（特にモバイルネットワークで顕著です）。



Microsoft Exchange



異なるサーバーとSSLオフロード間で負荷分散を使用することを提案しますが、要求の種類に応じてトラフィック分散を使用することも可能です（L7コンテンツスイッチング）。これにより、異なる役割を持つCASサーバーにトラフィックを再分散できます。 NetScalerでの事前認証により、ブルートフォース攻撃（NetScalerには保護メカニズムが組み込まれています）から身を守るだけでなく、多要素認証を使用してExchangeにアクセスできます。



NetScalerは、Skype for Business、Office 365、Direct Accessと連携して、ライフサイクルを完了したTMG製品を置き換える追加機能を提供します。



Citrix製品のインフラストラクチャ用にCitrix NetScalerを適切に構成するには、これらの製品や他の多くの製品とNetScalerの統合について詳しく説明している適切な実装ガイドを読むことをお勧めします。 これらのドキュメントは、Citrixの公式Webサイトで入手できます。



さらに、これらの製品の実装を大幅にスピードアップできるカスタマイズテンプレートがあります。



トラフィックの最適化



これは、CitrixがMicrosoftアプリケーションのパフォーマンスを改善できる唯一の方法ではないことを言わなければなりません。 たとえば、Citrix XenDesktopにはRealTime Optimization Packコンポーネントが含まれているため、Microsoft Skype for Business（Lync）は仮想環境で正常に動作できます。 このサポートがなければ、LyncクライアントはMicrosoft Lync Serverを介してオーディオおよびビデオトラフィックを送信します。 RealTime Optimization Packでは、2つのクライアントがローカルネットワーク内でポイントツーポイントモードで音声とビデオを交換し、セッション管理情報のみをデータセンターに送信できます。 これは現在、Microsoftによって公式にサポートされている仮想環境でMicrosoft Lyncのパフォーマンスを向上させる唯一の方法です。







仮想化



MicrosoftとCitrixのコラボレーションのもう1つの分野は、デスクトップとアプリケーションの仮想化です。 かつて、CitrixはMicrosoft OS / 2オペレーティングシステム、次にMicrosoft Windows NT用のターミナルサービスを開発しました。 それ以来、CitrixはMicrosoftと協力して、Microsoftの仮想化および端末サポートソリューションに常に新しい機能を追加しています。 このコラボレーションの最新の結果は、Microsoftが独自のクラウドベースのAzure RemoteAppソリューションを放棄したことです。これにより、Azureのリモートデスクトップサービスに基づくオンプレミスのMicrosoft RemoteAppプログラムが提供され、Citrix XenDesktopリモートデスクトップソリューションの使用が推奨されます。 さらに、Microsoftライセンスポリシー（SPLA）は、クラウドからクライアントOSを「提供」することを実際に禁止し、リモートユーザーの場所でサーバーデスクトップのみを使用できるようにします。 マイクロソフトがサポートを発表したクラウドインフラストラクチャからクライアントデスクトップを提供する唯一の方法は、Windows 10仮想マシンを使用することです。Windows10仮想マシンは、Microsoft Azureクラウドに展開し、Citrix XenDesktopを使用して配信する必要があります。



機動性



CitrixとMicrosoftのコラボレーションの3番目の領域は、モバイルアプリ管理です。 Citrix XenMobileインフラストラクチャにより、モバイルデバイス用の企業アプリケーションをいわゆるセキュリティコンテナーに「ラップ」でき、ビデオカメラ、外部リンク、クリップボードなどの使用などの関連ポリシーを禁止できます。 したがって、モバイルデバイスでは、個人アプリケーションと企業アプリケーションの区別が確立されます。



実際、Citrix XenMobileはもちろんMicrosoftだけでなく動作します。 Citrix XenMobileインフラストラクチャのさまざまなメーカーのモバイルアプリケーションのサポートリストには、Android向けAdobe Reader、AndroidおよびiOS向けExact CRM、IBM Connections AndroidおよびiOS、iOS用MindLink、Microsoft Office 365、Microsoft SharePointなどの人気のあるプログラムを含む、多数のプログラムが含まれていますモバイルクライアントおよびその他多数。



クラウドストレージ



最後に、CitrixはMicrosoft Azureを使用してCitrix ShareFileを実行できるようにします。 これは企業の「クラウドストレージ」であり、DropboxやYandexドライブなどのソリューションとは異なり、企業ディレクトリと連携し、ファイルリソースリンクの有効期間、ダウンロード数、ダウンロード権、アクセスパスワードなどのポリシーを設定できます。など 企業は、クラウドインフラストラクチャだけでなく、顧客のデータセンターに展開されたローカルストレージを使用して、さまざまなハイブリッドShareFile使用スキームを構築できます。 Citrix ShareFileでは、ユーザーが所有するさまざまなデバイス（職場のコンピューター、ラップトップ、自宅のPCなど）間でデータを複製することもできます。 このソリューションは、Microsoft Azureに加えて、Amazon Web Servicesもサポートし、ローカルのMicrosoft SharePointインストール、ネットワークディレクトリ、およびOffice365クラウドドキュメントをリソースの共有リストに接続できます。 セキュリティを強化するために、CitrixはICAPを使用してIRM機能とDLPサポートを追加しました。



* * *



上記から、開発者としてのCitrixがソリューションの機能を大幅に拡張し、CiscoやMicrosoftなどの他のメーカーが提供する機能を複製できると結論付けるのは簡単です。



そして、なぜそうなのかは簡単にわかります。 たとえば、CitrixはNetscaler ADCの機能を本格的なファイアウォールに拡張することに関心がありません。ファイアウォール市場で本格的な競争を行う代わりに、そのリーダーとの統合が容易です。 シスコは、Cisco ACIのベストバランサーとしてCitrix Netscaler ADCを推奨し、Microsoftネットワークプラットフォームのマイクロソフトを推奨しています。 偽りの謙虚さなしに言ってください-そうです。
