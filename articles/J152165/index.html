Windows Server 2012のサーバーマネージャー：新機能



さまざまな著者が、Windows Server 2012の新機能について多くのことを書いています。 サーバーマネージャーの更新はspareしみませんでした。 マイケルウーティとドンジョーンズが話し合い、イノベーションについて説明します。 興味のある方は猫を歓迎します。



したがって、Windows Server 2012 Server Managerの革新のリスト。

  1. 新しいインターフェース。 新しいサーバーマネージャーで最初に気付くのは、新しいインターフェイスです。 Windows Server 2012のインストール時には、グラフィカルインターフェイスが使用されます。サーバーコアモードとは対照的に、サーバーマネージャーは、システムの起動後に最初に表示されます。 サーバーマネージャーの以前のバージョンは、役割と機能のパネルとともに、Metroインターフェイスに置き換えられました。
  2. コントロールパネル-Server 2012 Server Managerは、コントロールパネルからすぐに開きます。 コントロールパネルは、非サーバーコアモードのWindows Server 2012の最初のエントリポイントです。 ウェルカムウィンドウには、クイックスタート、新機能、詳細の3つのブロックがすぐに含まれます。 「クイックスタート」ブロックには、「このローカルサーバーの構成」、「役割と機能の追加」など、環境を管理するために実行する必要がある手順のリストが表示されます。 追加のオプションは、コントロールパネル、管理、ツールの表示、およびヘルプの上部に表示されます。
  3. ローカルサーバーの管理 –Server 2012では、実行しているローカルサーバーを管理できます。 [このローカルサーバーの構成]オプションを使用すると、コンピューター名、ドメイン名、ファイアウォールステータス、リモートデスクトップ、リモートコントロール、NICチーミングなど、ローカルコンピューターの重要な設定のほとんどを変更できます。 [役割と機能の追加]を選択すると、Hyper-VやActive Directoryドメインサーバーなどのローカルサーバーのサーバーの役割、またはBitLockerドライブ暗号化やフェールオーバークラスタリングなどの機能を追加できます。
  4. マルチサーバー管理 -Windows Serverの以前のバージョンのサーバーマネージャーとは異なり、Server 2012サーバーマネージャーでは、複数のリモートWindows Serverシステムを一度に簡単に管理できます。 [管理する他のサーバーを追加]をクリックし、AD、DNS、またはIPアドレスでホストできるネットワークに他のコンピューターを追加します。 追加後、リモートサーバーはパネル「すべてのサーバー」（すべてのサーバー）に表示されます。
  5. サーバーグループ -サーバーをリモートで管理する機能に加えて、Server 2012 Server Managerではサーバーグループを管理することもできます。 グループで実行するアクションは、このグループのすべてのサーバーに表示されます。 そのため、コントロールパネルの[サーバーグループの作成]をクリックして名前を指定し、グループに含まれるサーバーを選択することで、複数のサーバーを管理するグループを作成できます。
  6. イベントログ -サーバーマネージャーを使用すると、ローカルサーバーとリモートサーバーの両方のイベントログにアクセスできます。 ローカルサーバーまたはすべてのサーバーの表示モードの場合、ローカルサーバーとリモートサーバーの両方の表示イベントを切り替えることができます。ナビゲーションパネルで上記の表示モードを切り替えるだけです。 イベントをクリックすると、詳細な情報が表示されます。
  7. サービス -新しいサーバーマネージャーは、管理対象のローカルサーバーおよびリモートサーバー上のサービスを管理する機能も追加します。 ローカルサーバーまたはすべてのサーバー表示モードの場合は、サーバーマネージャーの[サービス]セクションの[イベント]セクションまでスクロールします。 結果をフィルタリングできます。 サービスのコンテキストメニューを使用すると、サービスの開始、停止、再起動、および作業の一時停止と再開を行うことができます。
  8. ベストプラクティスアナライザー -サーバーマネージャーのもう1つの革新は、「ベストプラクティスアナライザー」（BPA）を実行する機能です。 [タスク]ドロップダウンメニューをクリックし、BPAを起動して、ローカルサーバーまたはリモートサーバーをスキャンします。
  9. パフォーマンス -また、ローカルまたはリモートサーバーを選択した場合は、[パフォーマンス]セクションの[BPA]までスクロールします。 [タスク]メニューでは、追跡するパフォーマンスフレームワークを選択できます。 サーバー名のコンテキストメニューを呼び出して、パフォーマンス統計の収集を開始または停止します。
  10. 管理ツール -[スタート]メニューが表示されなくなったため、Server 2012では管理機能への簡単なアクセスを実装する必要がありました。 上部の「ツール」オプションが提供します。 [ツール]メニューには、以前の管理メニューで表示するのに使用したのと同じように見える管理オプションのリストが表示されます。 これらの管理オプションには、iSCSIイニシエーター、ODBCデータソース、リソースモニター、サービス、およびタスクスケジューラが含まれます。
