Io言語：メッセージシステム

今日は、由緒あるsemkaが始めた一連の記事を続けます。 メッセージについて話す。



Ioには関数呼び出しはありませんが、メッセージを送信しています。 メッセージには引数を含めることができますが（関数の引数とほぼ同じ）、メッセージ引数は送信前に実行されません。

メッセージの実行方法

まず、メッセージ引数が実行されないことに注意してください。 Database connect(blah-blah)



が書き込まれた場合、接続が送信される前にblah-blahは実行されませんが、メッセージ「connect（blah-blah）」の一部として送信されます（はい、言葉で）。



オブジェクトがメッセージを受信すると、このメッセージの名前を持つスロットを検索します（この例ではconnect



）。 スロットが見つからない場合（通常）、その検索はすべてのオブジェクトプロトタイプとそのプロトタイプで再帰的に実行されます。 目的のスロットがどこにも見つからない場合、 forward



スロット検索が開始されます（Rubyのmethod_missing



に類似）。 適切なスロットが見つかるとすぐに、その値がアクティブになります （アクティブ化）。 （注：見つかったスロットがどの特定のオブジェクトにあるか、Object contextWithSlot（slotName）が通知します。）



アクティベーション

通常の値の場合、アクティベーションは何もせず、その値を返します。 したがって、数値、文字列、または他の多くのオブジェクトを格納する通常のスロットのアクティブ化は、getSlot（ slotName ）を介して値を取得することと変わりません。 ただし、 起動 可能としてマークされているオブジェクトは、 activateメソッドを呼び出します。 デフォルトでは、ブロック（ブロックとメソッド）とCFunction（関数へのリンク）の2つのオブジェクトのみがアクティブになります。 他のすべてのオブジェクトは、setIsActivatable（true）を使用してアクティブ化できます。



getSlot（slotName）メソッドが呼び出されると、スロットのアクティブ化は行われません（もちろん、getSlotスロット自体はアクティブになりますが、 slotNameはもう存在しません）。 したがって、メソッドまたはそのままのブロックを呼び出さずに取得する場合は、getSlot（methodName）を使用します。



1つの落とし穴はアクティベーションに関連しており、これについては記事の最後で説明します。



あー そして、引数はいつ実行されますか？

メッセージが送信され、スロットが見つかり、アクティブになりました。 しかし、メッセージ引数はいつ、どのようなコンテキストで計算されますか？



例を考えてみましょう：

withoutArgs := method() # nil

withoutArgs("Hello!" println)







引数「Hello！」Printlnが実行されなかったため、このコードは何も出力しません。

withArgs := method(a, b, c, list(a,b,c)) #

withArgs("Hello!" println) # list("Hello!", nil, nil)







このコードは最初の引数を実行し、Hello！を表示します。 欠落している引数はnilです。



別の例：

withTwoArgs := method(a, b, nil)

withTwoArgs(1 print, 2 print, 3 print, 4 print)







このコードは12を出力しますが、1234は出力しません。



最も独創的な人は、宣言された引数のみが実行されていることをすでに認識しています。 最も面白いのは、それらがどのように実行されるか、どこでどこで汚れた手を動かすことができるかです。 そして、なぜこれがすべて必要なのか。



イントロスペクションメソッドの呼び出し

Ioにはオブジェクト、プロトタイプ、スロット、メッセージのみがあることをまだ覚えていますか？ このリストにはグローバル変数もローカル変数もありません。



Ioはスロットをアクティブにする（つまり、メソッドを呼び出す）と、特別なLocalsオブジェクトを作成します。 これは多くの重要な機能を備えた非常に面白いオブジェクトです。 まず、このオブジェクトのプロトタイプはself、つまりです。 メッセージの送信先のオブジェクトへのポインター。 したがって、受信者オブジェクトに干渉することなくローカルスロット（ローカル変数とも呼ばれます）を作成し、このオブジェクトのすべてのスロットにアクセスできます。 次に、このオブジェクトには少なくとも3つのスロットがあります：self（受信者オブジェクトを指す）、call（呼び出しに関する多くの情報を含む）、およびupdateSlot。 後者は、ローカルではなく受信者オブジェクトでスロットを更新するという点で、通常のupdateSlotとは異なります。 これは、self a = bではなく、a = bと書くためだけに行われます。

    ローカル変数に戻る

    呼び出し送信者や呼び出しメッセージなどの強力な武器を使用して、メソッド呼び出しと引数の実行をIo言語で記述することができます。 実際、メソッド（a、b、nil）と言って、ローカルにスロット「a」と「b」を作成し、呼び出し送信者のコンテキストで対応する引数が実行された後に受信した値でそれらを埋めたいと言います。 宿題：組み込みメソッドと同じ方法でメソッドを作成し、宣言および渡された引数に対してdoInContextを使用して必要なすべての操作を行うmethod2メソッドを記述します。
