BeelineからUbuntuへの3GモデムZTE MF 170

1か月前、私はZTE MF 170 3Gモデムを購入しました。



残念ながら、この奇跡のテクノロジーをUbuntuで機能させる方法は、ネットワーク上に既製のソリューションがありませんでした。 したがって、私は自分自身を抜け出す方法を探す必要がありました。



最初の問題。 モデムはCD-ROMドライブとして定義され、Windowsプログラムとfireをダウンロードしようとします。 当然、そこからは何も出ず、ドライブモードのままです。 他のPchelainモデムと同様に、これはusb_modeswitchプログラムで処理できます。 モデムは実際にはモデムとして定義されており、接続を確立しますが、モデムを接続するたびに行う必要があり、接続後に2番目の問題が発生します。



2番目の問題。 接続が確立された後、数分後に接続が切断されます。 ネイティブプログラムは、モデムを動作させるために、2分ごとに、私たちには知られていないATコマンドを送信します。



合計2つの問題であり、1つの賢明なソリューションではありません。



gsmforum.ruで、ファームウェア内の正確に2つのファイル、AUTORUN.FLGとNODOWNLOAD.FLGがこれら2つの欠陥の原因であることがわかりました。 1つ目は起動時にモデムをドライブとして起動する役割を果たし、2つ目は数分後に切断する役割を果たします。 それらを叩く必要があります。



gsmforum.ruでは、Orthodoxオペレーティングシステムを搭載したコンピューターに少なくともOrthodox QPSTプログラムをインストールし、このプログラムを使用してファイルを破壊することが提案されました。



しかし、Tru-Linuxユーザーとして、Windowsを搭載したコンピューターと友達を探すべきではありません。Ubuntで同じ操作を行う必要があります。 ここで、 ATコマンドを使用してこれを実行できることを読みました。

AT + ZCDRUN = 8 \ r \ n-AUTORUN.FLGを削除するには、

AT + ZCDRUN = E \ r \ n-NODOWNLOAD.FLGを削除するには、



これを行うには、モデムをモデムとして定義し、/ dev / ttyUSB1またはそのようなものの上に立つ必要があります。 usb_modeswitchを1回使用するか、簡単に実行できます。 モデムがドライブとして検出されたら、デスクトップ上のアイコンを右クリックして、取り出しを選択します。



次に、ATコマンドを使用してこれらのファイルを削除します。



$ echo -e "AT + ZCDRUN = 8 \ r \ n"> / dev / ttyUSB1-ファイルAUTORUN.FLGを削除します

（そして突然必要な場合：AT + ZCDRUN = 9 \ r \ n-AUTORUN.FLGファイルを復元します）



$ echo -e "AT + ZCDRUN = E \ r \ n"> / dev / ttyUSB1-ファイルNODOWNLOAD.FLGを削除します

（AT + ZCDRUN = F \ r \ n-NODOWNLOAD.FLGを復元します）



それだけです ここでモデムを引き出して再接続し、青く光る（3Gをキャッチすることを意味する）か緑（GPRS）が点灯するまで待ちます。 これで、接続マネージャーを介してBeeline 3G接続を作成し、3Gインターネットを楽しむことができます。



PS唯一のものはwvdialを介して接続されていません。 （wvdialがATM0L0DT * 99＃を通過しようとするとすぐに、お気に入りのZTE MF 170が安全に切断されます:)理由がわかりません。
