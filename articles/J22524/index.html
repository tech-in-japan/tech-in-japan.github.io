Dojo Toolkit 1.1がリリースされました何が新しくておいしいですか？



すべての読者への挨拶。 そして、何が起こったのかは、多くの人（または少なくとも誰か）が待ち望んでいたことでした。 はい、 ついに AJAX WebアプリケーションDojo Toolkitを開発するための最も強力で本当に「Web 2ゼロ」ライブラリのリリースバージョンをリリースしました。 dojoパッケージに限定する場合とdojoxとdijitがある場合を除き、すべての機能の少なくとも10％を使用する通常のサイトはほとんど想像できないため、アプリケーション開発用です。バージョン0.4.3のアクティブな開発中（そう、そのような古さや化石ですが、ある時点では、最新バージョンよりもさらに優れているか簡単です）、Dojoの視覚的な部分はあまりうまくいきませんでした。 それにも関わらず、 ExtJSは数桁高いものであり、今のところ玉座から転覆させる代替手段はありません。



しかし、私たちのビジネスに戻りましょう。 開発とアーキテクチャの変更というかなり長い進化の道をたどったDojoは、最終的に安定した開発段階を開始しました。この段階では、プロジェクトの技術プラットフォームとしてだけでなく、それを完全に使用することもすでに可能です。 バージョン1.1では多くの変更と改善が行われていますが、このリリースをツールとして検討することをお勧めします。 そこで、平均的な開発者にとって新しくて興味深いものは何ですか？ 考えてみてください。



Dojoパッケージ 。 多くの一般的な改善、APIとドキュメントの更新、CSS /テンプレートの変更とバグ修正、以前のバージョンから残余と依存関係を削除するための改善とコードの変更（アーキテクチャとAPIは数回変更されましたが、一部のモジュールは以前のバージョンがほとんどないままでした変更（新しいAPI構造をサポートしないことを含む）。 多言語アプリケーションを作成するためのテンプレートと言語リソースの操作が改善されました。 多くのCSSスタイルが変更されているため、dijitを使用してユーザーインターフェイスを構築した場合は、コードの変更を確認する必要があります。



0.4.xブランチの非常に初期のリリースと最新の1.1の組み合わせまで、Dojoの複数のバージョンを1ページで処理するという興味深い機会が（私が理解しているように）同時にありました。 もちろん、これは非常に特殊な機能であり、多くの機能と落とし穴がありますが、プロジェクトの根底にあるアーキテクチャと概念の柔軟性と独立性を間接的に証明しています。 この機能の詳細については、Dojo bookセクションをご覧ください。



ライブラリーをロードするプロセスが少し変更されました-最初からではなく、メインページのロード後にDojoがロードするように構成を指定できるようになりました。 以前は、複雑なアプリケーションの場合、さまざまなダウンロードインジケータとマスクを設定する必要がありましたが、基本的にはWebアプリケーションはセッション全体で1回しか読み込まれないため、ダウンロードを待つ方が良いでしょう。 しかし、ここに何かがあり、Dojoダウンロードのリソース消費はしばしば非常に深刻でした。少なくとも私は1年近く見続け、DojoとExtJSの束を使用する大規模なオンラインプロジェクトを開発しました。 これを最適化できるようになったため、責任を持って設計にアプローチし、使用するモジュールの数を必要最小限に減らすと、パフォーマンスが大幅に向上するか、少なくとも個々のコンポーネントをロードするプロセスがそれほど目立たなくなります。



サーバーにデータを転送するための組み込みコンポーネントdojo.xhrは、ネイティブXMLHttpRequestの抽象化として機能し、大幅に更新されて機能が拡張されています。 現在、GET / POST以外のメソッド（PUT / DELETEなど）の実装がありますが、必要なクエリを自分で実装してモジュールを拡張することもできます。 最も人気があり興味深いリクエストは、指定されたページの更新を確認するHEADであるように思われます。 常に最も強力で機能的なDojoであったのはこのモジュールでしたが、今ではさらに拡張されています（最初のバージョンではより多くの代替トランスポートがありましたが、ネイティブオブジェクト、iframe、およびスクリプトタグのみが存在することを覚えています）。 奇妙なことに、XMLHttpRequestオブジェクトを介して送信されたリクエストにX-Requested-Withヘッダーが最終的に正しく挿入されるようになりました。 .ru-結局のところ、コミュニティはどこにあったのか、とにかく、リリースを待つ必要がありました。



ブラウザーのキャッシュを使用した作業が改善され、組み込みのテンプレートを使用するときにも使用できるようになりました（またはその逆、無効）。



DOMノードの属性を操作する力は積極的に変更され、CSSスタイルの操作は組み込みのdojo.styleクラスを通じて簡素化されました。これは開発者のおかげです。 dojo.queryモジュールも改良されました。これにより、DOMツリーの個々のノードとブランチ全体を簡単な方法で照会することが可能になりました。



ほとんどのWebライブラリに固有の機能であるアニメーションモジュールは大幅に変更されており、非常に多くの変更と改善があります。アニメーションアルゴリズムとテクニックの一部は、おそらくさまざまなエフェクトと動的アプリケーションの開発を簡素化するためにFlashから転送されたと言われています。 生産性とレンダリングを向上させることを約束します（最小期間が1秒から350ミリ秒に短縮されたため、むしろ、再描画の最小「ティック」についてお話します）。 これには、 dojo.fxなど、さまざまなエフェクトとグラフィックスを担当する他のモジュールの変更も含まれます。



そして最後に、Web業界の新しいトレンドに従って、DojoはAdobe AIRプラットフォームのすべての機能のサポートを受けました。これは、特にDojoに既に存在する機会（そのほとんどが実験的および革新的なDojoxモジュール）。



Dijitパッケージ 。 このパッケージには、ユーザーインターフェイスとさまざまなフォームを作成するための視覚要素とウィジェットが含まれています。 このリリースでは、主にパッケージに化粧品の変更が行われ、新しいテーマといくつかのウィジェットが追加されました-SimpleTextArea 、拡張不可のシンプルテキストフィールド、複数の要素を一度に選択できる複数選択ウィジェット。 新しいタイプのレイアウトが登場しました-BorderContainerは、SplitContainer / LayoutContainerの廃止されたオプションに代わるもので、 ツリーを操作するためのモジュール（ Tree ）でした。今、これはそのようであってはなりません。



Dojoxパッケージ 。 ここでは、以前はほとんど実装できなかった（または実装できたはずの、しかしそれ自体が非常に複雑で自明ではないように思えた）主要な革新的なモジュールと機能が集められています。 変更点の中で、DTLモジュールの変更点を強調する価値があります。テンプレートで、for / if制御構造を使用してウィジェットやその他の要素を宣言できます。 Web描画ツールの例で使用されている拡張アニメーションおよびエフェクトモジュールdojox.gfx （AJAX / VML / SVG上のベクター「ドロワー」をブラウザーで直接使用し、マルチユーザーの作業も可能）が更新されました。 さまざまなグラフとチャートdojox.chartを描画するモジュールも更新されました 。これは、多くの場合、サーバー実装またはGoogleに関するWebサービスのいずれかを置き換えるのに十分な機能です。



暗号化およびコーディングモジュール（base64、light base64、ascii85アルゴリズム）は1つに統合され、次のバージョンでは別の暗号化モジュールが削除されます。 ところで、情報をアーカイブするためのモジュールが登場し、これまでに2つのアルゴリズム、LZWとSplayをサポートしています。



多くの人にとって嬉しいことに、さまざまなプログラミング言語のコードブロック内で構文を強調表示するための組み込みの強調表示モジュールが登場しました。 開発者や特化したCMSに焦点を当てたプロジェクトを作成することが可能になりました。コードを使用して記事を作成すると、目立つようになります。 はい、さらに多くのアプリケーションがありますが、以前はサードパーティソリューションを適用する必要がありました。多くの場合、サーバーソリューションのみでしたが、現在は組み込みの機能です。



ストアモジュールも更新されました。これにより、FlickrやPicasaなどのリモートサービスを含む、データを保存および表示するさまざまな方法から抽象化し、統一することができます。 さらに、Atom、OPML、XML、HtmpTableなどのデータを操作できます。



個人的には、最も予想され、必要なモジュールを強調します- ストレージ 、最終的にバージョン1.1に完全に書き直され、クライアントに情報を保存する利用可能な手段で透過的な作業をサポートします-プラグインFlashオブジェクトとAdobeAIRストレージの両方（ファイルと保護されている）、Google GearsおよびHTML 5ストレージ。 これは最も強力で必要な機能であり、多くの開発者が本当に期待していたものであり、他のすべてのライブラリの中で実際の競争相手はいません。



Dojoxパッケージには、さらに多くの記事を書くことができる、そして必要とする多くのユニークなモジュールがまだあります。 記事の最後では、1つのモジュールのみを強調します。これは、以前のリリースを思い出す限り、今しか登場していませんが、公式のプレスリリースでは言及していません。 これは分析モジュール（ dojox.analytics ）であり、これにより、ページ上のすべてのアクション（動きやその他のユーザーアクションの追跡など）の「追跡」を柔軟に構成できます。 すべてのアクションに関する情報（および非アクティブ）がサーバーに送信され、ユーザーの行動を分析し、アプリケーションでの彼の作業または非アクティブを検出し、ユーザーがWeb上で何をどのように行うかを正確に知る必要があるユーザビリティテストシステムおよびその他のソリューションを作成するために使用できます-ページ。



それだけです。少なくとも、この強力で美しいライブラリーに関心を示し始めることを願っています。 その場合は、 http：//dojotoolkit.orgにアクセスして、機能、例、およびドキュメントに精通し 、配布パッケージをダウンロードして、 ...
