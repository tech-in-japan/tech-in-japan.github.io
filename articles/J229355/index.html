dd-wrtを使用してユーザーからルーターを保護する

私が勤務を辞めるプロバイダーは、契約の締結期間中、無料でルーターを加入者に配ります。 ルーターを発行する際、メカニックはユーザー名、パスワード、IPを入力し、wifiを設定することでルーターをフラッシュします。 そしてすべてがうまくいきますが、特に賢いユーザーはデバイスを設定したい人に出くわし、ルーターには工場設定へのリセットボタンもあります。その後、加入者がオフィスに行くか、メカニックがデバイスを再設定するためにサブスクライバに行きました。 忍耐は完全で、最後のストローはTL-WR841Nであり、プロバイダーはこれを大量に購入しました。

それだけでなく、リセットペダルはくぼんでいない（むしろ、ねじれたケーブルで押すことができ、加入者のデバイスが時々現れるごみ）ので、メーカーは1つのボタンでWPSとRESETを組み合わせました。これは、素晴らしいソリューションです。



別の痛みを伴う瞬間がありました-ネイティブTP-Linkファームウェアでは、ルーターは切断後に接続を常に復元しませんでした。

もちろん、このソリューションにはパスワードを変更できないなどの欠点もありますが、これに我慢できます。

それでは...「治療」を開始します。



Linuxはあまり好きではありませんが、FreeBSDで必要なツールをすべてオンザフライで入手することはできませんでした。また、Linuxのファームウェアソースを選択するための準備段階でフラッシュドライブを試したくありませんでした。 

# uname -a Linux debian64 3.2.0-4-amd64 #1 SMP Debian 3.2.60-1+deb7u1 x86_64 GNU/Linux


重要なニュアンスはx86_64ですが 、それについては後で説明します。

最終的なタスクは、ユーザーデータを含むデフォルトルーターにファームウェアを入力することです。ユーザーデータは、工場出荷時の設定にリセットすると復元されます。 途中で、切断後に再接続する問題を解決します。 後者の問題はdd-wrtでは発生せず、ソースコードの存在が特に魅力的です。

開始するには、リビジョンのファームウェアを選択するだけです。 ルーターに注ぎ、テスト設定を行い、保存場所を探します。 少し 

 root@DD-WRT:~# grep -rn test00 /etc /etc/defaults.bin:160: ppp_usernametest001 root@DD-WRT:~#


編集する/etc/defaults.binファイルに何が保存されているかを確認します。 それからデフォルト設定がひきつります。 少しグーグルは、16進エディターで編集オプションを見つけますが、ソースコードを持っていることは、16進エディターで歪める方法ではありません。 次の3つの方法があります。

-ソースからすべてのファームウェアを組み立てます。

-ソースのみdefaults.binからコンパイルし、元のファームウェアを分解して置き換えます。

-自分でdefaults.binを作成し、元のファームウェアで再度置き換えます。

すべてのオプションが実行可能であり、3番目のパスに沿って進みますが、最初の2つにも触れます。 ソースからファームウェアを作成するには、プロセッサアーキテクチャ（この場合はMIPS）に合わせてアセンブルする必要があり、クロスコンパイルでhemo核に自動的に到達することを覚えておく必要があります。 このためにdd-wrtサイトにはツールチェーンが用意されていますが、それらへのリンクはすべて「404 Not Found」と答えています。 彼らは何とかそれらを見つけることができました-ftp.dd-wrt.com/toolchains/toolchains.tar.xz

注意 ：すべてのツールチェーンはx86_64で構築されているため、最初に実験用にx86_64システムを設定する方が良い

クロスコンパイルについては説明しません。このトピックはwiki dd-wrtで説明されています。



しかし、私たち自身の方法で行こう。 ファームウェアを分解することから始めます。

ファームウェア変更キットのダウンロード 

 # git clone https://code.google.com/p/firmware-mod-kit/


ファームウェアを分解します 

 # firmware-mod-kit/extract-firmware.sh tl-wr841nd-webflash.bin ./working_directory/


では、defaults.binの準備に取り掛かりましょう。

dd-wrtのソースを入手する 

 # svn co svn://svn.dd-wrt.com/DD-WRT


実際、すべてのソースコードから必要なファイルは2つだけです 

 # find ./src/router/services/ -name defaults.c ./src/router/services/sysinit/defaults.c ./src/router/services/bin/defaults.c


1つ目は、埋められたsrouter_defaults構造を含み、2つ目はそれをdefaults.binにドライブします。
