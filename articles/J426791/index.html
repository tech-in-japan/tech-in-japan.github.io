SAP Enterprise Architecture Designerを使用して会社でITアーキテクチャを構築する方法

SAP Data Management Suiteに関する一連の資料のこの記事では、社内でITアーキテクチャを構築するためのアプリケーション、SAP Enterprise Architecture Designerについて説明します。



サイクルの以前の記事：

企業でビッグデータを扱うための複合体としてのSAP Data Management Suite

SAP HANA Data Management SuiteのデータベースとしてのSAP HANA機能



SAP Enterprise Architecture Designer（EA Designer）サービスを使用すると、情報ランドスケープ、機能要件、ビジネスプロセス、データ構造とフロー、およびビジネス内の他の成果物を記述、モデル化、分析できます。











EA Designerは、企業内で本格的な企業アーキテクチャを作成および維持するためのツールです。 EA Designerを使用すると、会社のITアーキテクチャをすばやく説明し、簡単に視覚化できます。



すべてのモデルは単一のリポジトリに配置されるため、共通の企業アーキテクチャを作成するプロセスでのコラボレーションが簡素化されます。



このサービスは、異なるモデルと図のオブジェクト間の通信もサポートします。 影響と起源の分析により、これらの関係を視覚化および追跡して、オブジェクト間の関係およびチェーン内のオブジェクトの1つに対する変更の影響を理解することができます。







EA Designerは、次のチャートタイプの作成と編集をサポートしています。



ビジネスプロセス ビジネスプロセス図を使用すると、ビジネスプロセスを識別、説明、および分解できます。 BPMN 2.0の2つのバリアントがサポートされています。



  • BPMN 2.0記述的。 一般にビジネスユーザーが使用します。 ビジネスプロセスの設計と分析に適したBPMN 2.0オブジェクトのサブセットが含まれています。
  • BPMN 2.0実行可能ファイル。 通常、プロセス開発者が使用し、すべての標準BPMN 2.0オブジェクトが含まれ、すべての技術的詳細を考慮してビジネスプロセスをシミュレートできます。 これにより、このモデルを使用して、BPMシステム（SAP Process Orchestrationなど）でプロセスをさらに実装することができます。 SAP BPMまたはEclipse BPMN2 Modelerのモデルのリバースエンジニアリングも可能です。








概念データモデル 情報システムの概念構造を開発および分析し、表現すべき主要なオブジェクト、その属性、およびそれらの間の関係を特定するのに役立ちます。



物理データモデル データベースの構造の開発と分析に役立ちます。 サポートされているデータベースをリエンジニアリングできます。 物理モデルに基づいたSAP HANAでのデータベース構造の直接生成がサポートされています。



データフロー図 データウェアハウスとその変換間のデータ転送の設計と分析を支援します。 データフローをリバースエンジニアリングして、SAP Web IDEで作成できます。







コーポレートアーキテクチャ（エンタープライズアーキテクチャ図） 。 組織、その機能とプロセス、それらをサポートするアプリケーションとシステム、およびそれらが実装されている物理アーキテクチャの分析と文書化を支援します。







NoSQLダイアグラム（NoSQLドキュメントスキーマ、JSON） 。 システムで使用されるJSON構造の開発、分析、文書化に役立ちます。 JSONスキーマをリバースエンジニアリング（またはJSONドキュメントから継承）して、JSONスキーマファイルを作成できます。



要件リスト ビジネス目標、機能要件などの階層リストを表示します。



モデルからの情報に基づいて、さまざまな一般的な形式でレポートをすばやく作成し、関係者に配布できます。



EA Designerは、オンプレミスとクラウド展開オプションの両方で利用できます。 ユーザーはブラウザを介してアプリケーションを操作します。
