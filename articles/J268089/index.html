時間です。 Oracle Database 12へのアップグレード

なぜ時間ですか？



時間はあっという間に過ぎ、Oracle Database 11.2のユーザーにとっては、以前のバージョンは言うまでもなく、アップグレードの時が来ました。 2015年1月31日に、Oracle Database 11.2のプレミアサポート期間が終了しました。 これは、たとえば、ハードウェアサーバーのアップグレードを計画している場合、新しいバージョンのオペレーティングシステムをインストールする必要があることを意味します。 以前のバージョンのドライバーは使用できなくなり、新しいオペレーティングシステムはバージョン11.2の認定を受けなくなりました。







Oracle Databaseの様々なバージョンの詳細なサポート情報については、Oracle Lifetime Support Policyホワイトペーパーを参照してください。 たとえば、バージョン11.1の拡張サポートが完了しました（図1）。 今日の最も人気のあるバージョン11.2に関しては、拡張サポートの最初の年が彼女のために終了し、来年1月から技術サポートのコストの漸進的な増加が、最初の10％で、その後の年で-20％であり、これは別の経済的インセンティブです企業はバージョン12cへのアップグレードを検討しています。



データベースの新しいバージョンへの移行の準備では、「更新」と「移行」という用語の違いを理解することが重要です。



データベースのアップグレードは、以前のオペレーティング環境のフレームワーク内での新しいバージョンへの移行です。プラットフォームとオペレーティングシステムは変更されず、データはサーバー間で「移動」せず、データベースのバージョンのみが変更されます。 実際には、更新はディクショナリとOracleメタデータを更新する手順で構成され、データ自体は変更も移動もされません。 この場合、データベースのサイズは重要ではなく、更新速度には影響せず、データベース内のオブジェクトの数のみが重要です。 つまり、更新によりすべてが明確になり、データベースバージョンのみが変更されます。この場合、データベースのバージョン12にアップグレードしますが、プラットフォームとオペレーティングシステムは変更されません。



Oracle Databaseのどのバージョンをバージョン12cにアップグレードするかが重要です。 バージョン10以降、Data Pumpテクノロジーが利用可能になりました。 古いバージョンからアップグレードするには、エクスポートとそれに続くデータインポートで構成されるプロセスが提供され、追加のディスク領域が必要です。 この方法は、小規模なデータベースでよく使用されます。 Oracle Database Upgrade Assistantと呼ばれる特別なアップグレードツールを使用することをお勧めします。OracleDatabaseディストリビューションキットには無料で付属しています。データベースのさまざまなバージョンへの適用性を図に示します。 2.バージョン12以降、手動更新オプションも利用可能になりました。これは、Perlの特別なスクリプトとして提供されています。



Recovery Managerユーティリティに追加された追加機能を使用すると、アップグレード中のダウンタイムを最小限に抑えることができます。トランスポータブル表スペースを使用してデータベースにファイルを転送し、Recovery Managerの増分コピーを適用します。 このテクノロジーは、RMAN増分バックアップリカバリと呼ばれます。 シンプルが受け入れられない場合、オンザフライで更新を提供するゴールデンゲートと呼ばれる特別なツールを使用する必要がありますが、これはソフトウェアライセンシング、レプリケーションメカニズムの準備および適応の観点から最も高価な方法です。



移行とは、新しいオペレーティング環境、つまり新しいサーバー、新しいオペレーティングシステムへの移行です。 もちろん、この場合、データベースの物理的なサイズが最も重要です。 多くの場合、アップグレードおよび移行タスクは共同で実行されます。たとえば、Oracle Databaseの新しいバージョンに同時にアップグレードすると同時に、機器を変更します。



移行のためにデータベースを移行する最も信頼できる方法は、古くて優れたエクスポート/インポートです。 エクスポートファイルはプラットフォームに依存しないため、あるファームウェアプラットフォームの古いバージョンのデータベースからアップロードし、別のファームウェアプラットフォームの新しいバージョンのデータベースにインポートできます。 バージョン10.2以降、表スペースを転送するためのテクノロジーが使用可能になりました。 残念ながら、RMANの増分バックアップリカバリテクノロジーは、すべてのハードウェアおよびソフトウェアプラットフォームに適用できるわけではありません。 ゼロダウンタイムが必要な場合は、Golden Gateテクノロジーを使用できます。



更新および移行方法の詳細については、この記事の最後に記載されているドキュメントをご覧ください。



パッチと認証



データベースの新しいバージョンへの移行中にハードウェアとオペレーティングシステムを変更する場合、このソフトウェアとハ​​ードウェアプラットフォームがOracle Database 12cに対して認定されているかどうかを確認する必要があります。 テクニカルサポートサイトsupport.oracle.com（[認証]タブ）でファームウェアプラットフォームの認証に関する情報を参照してください。 ディストリビューションの検索場所については説明しません。現在のすべてのサービスパックを入手することを忘れないことが重要です。 オラクルは、重大なバグを修正する大規模な累積サービスパックを四半期ごとにリリースしています。 現時点で推奨されるバージョン12.1.0.2のパッチは、PSU Update 4およびOJVM PSU Update 4（Oracle Java Virtual Machine、組み込みJavaマシン用のアップデート）です。



現在バージョン11.2.0.3または11.2.0.4を使用しており、特定のエラーを修正するために特定のパッチを既にインストールしている場合、これらのエラーがバージョン12で修正されていることを確認する必要があります。バージョン11では、バージョン12の累積更新で特定のエラーが修正されたかどうかを確認し、修正されていない場合は、適切な「パッチ」を取得してこのエラーを修正します。 そのようなパッチがまだリリースされていない場合は、ファームウェアプラットフォームのテクニカルサポートサービスに対応する要求を行う必要があります。



バージョン12.1.0.2から、OracleはExadata Engineering SystemのOracle Database In-Memoryパッチと呼ばれる特別なパッチをリリースしました。 Oracle Database In-Memoryテクノロジーの更新が含まれており、その名前にもかかわらず、Exadataだけでなく通常のディストリビューションにも適しています。OracleDatabase In-Memoryテクノロジーを使用する場合はインストールすることをお勧めします。



重要な更新は、ロシアの時間測定の機能に関するものです。これは、ご存じのとおり、タイムゾーンで時間を計算する原理が数年の間隔で2回変更されたという事実によるものです。 したがって、ロシアのタイムゾーンではこの状況を考慮した特別なタイムゾーン更新プログラムをインストールすることが非常に重要です。 Javaストアドプロシージャとタイムゾーン付きの日付や時刻などのJava変数を積極的に使用している場合は、Java仮想マシンのタイムゾーンで時刻を修正するための特別なパッチが必要です。



新しい更新プログラムをインストールするには、忘れずにOPatchユーティリティを更新してください。 Oracle Databaseバージョン12では、新しいDMBS_QOPatchパッケージが導入されています。



ソースデータベースの準備



アップグレードを開始する前に、ソースデータベースを準備する必要があります。 そのような準備の「チェックリスト」を図に示します。 3。



アップグレードする前にゴミ箱を空にしてください。 Oracle 12c以降、preupgrade_fixups.sqlスクリプトを使用して実行されます。 データベースの最小負荷期間中に自動的に設定することにより、少なくとも週に1回ごみ箱を空にすることをお勧めします。



エラーのあるオブジェクトを確認してください。 SYSおよびSYSTEMスキームにエラーのあるオブジェクトが存在しないようにする必要があります。 このチェックを実行するには、utlrp.sqlスクリプトを使用して更新または移行する前に、無効状態のオブジェクトをコンパイルする必要があります。 これで解決しない場合は、アップグレードを続行しないでください。 SYSスキームとSYSTEMスキームに同じ名前のオブジェクトがあるかどうかを確認する価値があります 。これは次のクエリで実行できます。 



select OBJECT_NAME, OBJECT_TYPE from DBA_OBJECTS where (OBJECT_NAME,OBJECT_TYPE) in select OBJECT_NAME, OBJECT_TYPE from DBA_OBJECTS where OWNER=SYS') and OWNER='SYSTEM' and OBJECT_NAME not in ('AQ$_SCHEDULES_PRIMARY', 'AQ$_SCHEDULES','DBMS_REPCAT_AUTH');


また、イベントを含むすべての廃止され、文書化されていないパラメーターを削除します。 SAP ERPやOracle EBSなどの大規模で複雑なアプリケーションを使用している場合は、それらのアプリケーションのデータベースの更新に関する付属のドキュメントを必ず確認してください。 Oracle会社自体のデータベースを更新する例があります。文書化されていないパラメーターがすべて削除されると、更新速度が7倍になりました。



すばらしいユーティリティHealth Checkがあります-辞書の整合性をチェックできるスクリプトです。 このスクリプトは、Oracle8i、Oracle9i、Oracle 10g、およびOracle 11gの既知の問題をチェックします。 パフォーマンスを高めるために、複数の並列チャネルで実行することも、個々のデータファイルまたは表スペースのみで実行することもできます。 ヘルスチェックはデータベースで実行され、すべてのコンポーネントに関する情報をSQL +コンソールに表示します。 辞書の不整合が検出された場合、スクリプトはこの情報を出力します。辞書は整合状態でなければならないため、更新する前にこの問題を解決する必要があります。 このユーティリティは、テクニカルサポートサイトからダウンロードできます。



また、バージョン12では、新しい特別なアップグレード前スクリプトpreupgrd.sqlが登場しました。 更新前にチェックを実行し、古いバージョンのデータベースで実行し、バージョン12への移行に準拠しているかどうか古いデータベースの環境をチェックします。



更新速度を上げるには、最新の統計情報を用意することをお勧めします。まず、Oracle DBMSのメタデータに関する統計情報を入手することをお勧めします。 辞書統計は、パッケージDBMS_STATS、GATHER_DICTIONARY_STATSを呼び出すことによって収集されます。 更新が関連性があるとみなされる前に収集された統計は1日以内です。



データベース更新



これで、更新手順自体を開始できます。 Oracle Database 12cへのアップグレードの期間は、主にデータベース内のコンポーネント、オプション、およびオブジェクトの数に依存します。バージョン12では多くの新しいテーブルが導入され、ベーステーブルの構造が再編成されたためです。 それほどではないが、更新の期間はシステムのパフォーマンスに依存します。 中央処理装置の周波数と数、およびI / Oサブシステムの速度。 バージョン12では、辞書は並行して（デフォルトでは4つのストリームで）更新されます。 残念ながら、最大並列度は8に制限されています。 ディクショナリには、並行して更新できる最大8つの独立したコンポーネントがあるため、最大8つのスレッドを作成できます。 おそらく将来のバージョンでは、並列処理の度合いが向上するでしょう。 ただし、すでにドキュメントでは、辞書の更新速度が40％増加すると約束されており、顧客の経験から、より高速な処理も可能であることが示されています。



Oracle Database 12cへのアップグレードは、Oracle DBMSディストリビューションの一部であり、追加のライセンスを必要としないDatabase Upgrade Assistantの管理を簡素化します。 共通のEnterprise Manager Cloud Controlコンソールから呼び出すことができます;これは、同じ構成のデータベースの大規模な更新を行う必要がある場合に便利です。 クラスタデータベースをアップグレードする必要がある場合は、Enterprise Manager Cloud Controlコンソールを使用できます。



更新完了



アップグレードを完了するための一連の手順を完了するには、utlu121s.sqlという特別なスクリプトまたはDatabase Upgrade Assistantユーティリティを使用する必要があります。 スクリプトは、データベースコンポーネントのリストを表示します-各コンポーネントステータスのステータスは有効である必要があります。そうでない場合、更新は失敗しました。 別途、タイムゾーンの特別な更新（タイムゾーンを決定するためのバイナリパッチと辞書を変更する特別なスクリプト）のインストール結果を確認する必要があります。
