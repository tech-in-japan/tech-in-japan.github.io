パート2.スマートフォンを車に接続する方法-練習しましょう

理論に精通し（ 第1部を参照）、コストと、スマートフォンを車に接続することで期待すべきでないことを既に想像した後、練習に進むことができます。





まず、自動診断を成功させるには、3つのデバイスを用意することが重要です。

1.車

2.アダプター

3.モバイルデバイス（ラップトップ、PC、スマートフォン）



車



だから、あなたは車を持っており、それが可能かどうか、可能であれば、あなたの車に接続するのに何が役立つかを知りたい。

診断スキャナー（コントロールユニット）への接続を主な機能とする診断コネクター（DLC）の存在は、適切なアダプターを使用したスマートフォンからなど、外部から車のオンボードコンピューターに接続する機能を自動的に想定します。 しかし、多くの異なるDLCコネクタと独自のプロトコルの前の部分で述べたように、特にほとんどの現代の車はこの規格に準拠しているため、 OBD-II規格のみを考慮します。



それで、最初の質問：私の車はOBD-II規格に適合していますか？



前提条件は次のとおりです。

  • 1996年以降、この車は米国市場（OBD-II）向けに生産されました。
  • 自動車はEU市場（EOBD）用に生産され、2001年以降はガソリン、2004年以降はディーゼル
  • 車は2004年以降に日本市場向けに生産された（JOBD）


車がCIS、アジア、アフリカなど他の市場向けに製造された場合、OBD-IIサポートの明確な日付と保証を確立することはより困難です。 また、一部のメーカーは以前のモデルの車の標準をサポートできることを覚えておく必要があります。



次のステップは、車のボンネットの下で適切なステッカーを見つけることです。

ラベルには、標準への準拠に関する情報が含まれている必要があり、OBD-II、EOBD、またはJOBDの略語が含まれている場合があります。 見た目が異なる場合がありますが、可能なオプションの一部を次に示します。



前のステップでの検索の成功に関係なく、 診断コネクタ自体を見つける時が来ました。 マシンがOBD-IIに準拠している場合、コネクタが確実に見つかり、さらに調査するために接続できます。 互換性にまだ自信がない場合は、コネクタの存在と形状がDLCの謎を解決するための追加のキーとして機能します。



図のような16ピン台形診断コネクタを探しています。 DLCコネクタは、SAE J1962標準に準拠する必要があります。 これらの規格によれば、車の特定の中心位置を占めるにはDLCコネクタが必要です。 ハンドルから16インチ（40 cm）以内でなければなりません。 メーカーは、EPAが指定した8つの場所のいずれかにDLCを配置できます。 各コネクタピンには独自の目的があります。



規格に従って、診断コネクタはステアリングコラムの近く、ほとんどの場合、魚雷の左、右、またはステアリングコラムの真下に配置する必要があります。 ただし、エキゾチックなケースがあります。

魚雷の前面のパネルの後ろ、運転席のドアの近くにあるヒンジ付きの引き出しの後ろなど、合計8つの場所があります。コネクタを見つけるための矢印とヒントの写真で説明しようとしました。



1.運転席側、インストルメントパネルの下、ステアリングコラムの下の領域：Skoda Octavia Tour、2008、Volkswagen Transporter 2002

2.運転席側、インストルメントパネルの下、運転席ドアとステアリングコラムの間の領域-Ford Focus C-Max、2006; フォードフォーカス（米国）2001、フィアットリネア、2008; スバル、インプレッサWRX、2005年。



3.運転席側のインストルメントパネルの下、ステアリングコラムとセンターコンソールの間。 また、コネクタはドライバー側からコンソール自体に配置できます。 2009ホンダシビック



4.運転席側のダッシュボード領域で、ステアリングコラムと1996年のフォルクスワーゲンパサートセンターコンソールの間...



5.運転席側のダッシュボードエリアの運転席ドアと2001 BWM X5センターコンソールの間。



6.垂直面上の中央コンソール（ラジオ、加熱コントロールパネルなど）。 診断ソケットへのアクセスは、灰皿、コインボックス、カップホルダーなどで閉じることができます。 ホンダアコード（米国）1997



7.助手席側では、1996年以降のホンダシャトルのセンターコンソールで...

可能な検索結果：



  • コネクタは見つかりましたが、写真に示されているもののようには見えません：長方形/正方形/円形、16の接点がないなど。 接続の可能性は十分に小さく、OBD-II配線への接点の対応が判明し、アダプターをはんだ付け（購入）できる場合でも、プロトコルが完全に互換性を持つ可能性はわずかです。 この場合、信号および論理プロトコルとそれらの「方言」の互換性に関して十分に準備ができている場合にのみ、継続する価値があります。
  • コネクタがあり、OBD-IIの説明と完全に一致しています。 接続が成功する可能性が大幅に増加します。 さらに、同様のモデルの自動車所有者が経験を共有している自動車フォーラムで質問することができます。




アダプター





次の項目はアダプターの選択です。アダプターは、コンピューター/スマートフォンへの接続のタイプ（USB、Bluetooth、Wi-Fi）と回路、そして結果としてコマンドインターフェイスの両方で異なる場合があります。 そのため、OBD-IIコマンドは統一されていますが、OBD-IIはいくつかの異なるシグナリングプロトコルを組み合わせているため、完全な均一性は得られません。



  • まず、さまざまな物理回線（連絡先）、K-Line、L-Line ...を使用して情報を送信できます。
  • 第二に、伝送周波数は4.8から500 kbaudまで大きく異なる場合があります。
  • 第三に、補助情報の形成に関するルール：ヘッダーとパケットの終わりが異なる場合があります。


アダプター自体は、接続のタイプとパラメーターを自動またはソフトウェアで制御します。 スマートフォンまたはコンピューターは、有線または無線接続を介してアダプターに接続し、高レベルのコマンドを受け取ります。 対応するコマンドの実行には、アダプター自体の設定を変更するか、アダプター設定に従ってOBD-IIコマンドを車のオンボードコンピューターに中継することが含まれます。



最新のアダプターの大部分は、 ELM327マイクロコントローラー 、以前のELM327の海賊版、またはSTN1110マイクロコントローラー（コマンドレベルでELM327と互換性があります）に基づいています。 前述のように、ワイヤレス（Bluetooth、Wi-Fi Direct）または有線（USB、RS232）を使用して、コンピューターまたはスマートフォンと通信できます。 いずれの場合も、上記のインターフェースを介したシリアル接続について説明しています。 特定のタイプのインターフェースの選択は、コンピューターまたはスマートフォンでこのインターフェースをサポートする能力に依存します。





モバイル機器

（ラップトップ、PC、スマートフォンなど）



デバイスのタイプを順番に検討してください。

コンピューター ラップトップでは、原則としてすべてのインターフェイスがあり、選択は価格、接続の容易さ、他のデバイスでアダプタを使用する能力、およびこのタイプのインターフェイスのソフトウェアサポートによって決まります。



iPhoneとiPad 標準APIを使用したBluetooth接続へのアクセスは制限されており、必要なタイプのアダプターとの接続を確立できないため、使用可能な接続オプションは（特殊なケーブルまたは認定Bluetoothアダプターの形式のエキゾチック以外は）Wi-Fi Direct接続のみです。



Android 最も一般的なオプションは、Bluetoothアダプターです。 Wi-Fi Directは、「ルート化された」デバイスまたはAndroid 4.xのバージョンのいずれかでサポートできます。また、一部のデバイスでは、Wi-Fi Directのハードウェアサポートに微妙な違いがある場合があります。 モバイルデバイスがUSBホストポートをサポートしている場合、接続オプションとUSBアダプターがあります。



Windows CE、Windows Mobile、Windows Phone。 Windows CEおよびWindows Mobileデバイスは、それぞれ低レベルのハードウェアインターフェイスに関して非常にオープンです。デバイスに物理インターフェイスがあれば、それに接続できます。 最も一般的に使用されるBluetoothアダプター。

バージョン7.xのWindows Phoneデバイスには、標準APIレベルでBluetoothおよびWi-Fiに必要なアクセス権がありません。 これらのデバイスでアダプターとの対話を実装することには問題があります。

Windows Phone 8.xは、Bluetoothアダプターの操作に必要なレベルのサポートを受けました。



ブラックベリー。 Bluetoothアダプターとの相互運用が可能です。



Symbian Bluetoothアダプターとの相互運用が可能です。



上記から結論を出すことができます： Bluetoothアダプターは 、サポートされているデバイスの点でより普遍的ですが、Appleデバイスから接続する必要がある場合は、Wi-Fi Directアダプターを探す必要があります。



あまり理解されていないインターフェースをボレる。 現在、多くの人はマイクロコントローラ（ELM327、ELM327クローン、STN1110）の選択の問題に興味があるかもしれません。価格はデバイスごとに5〜7ドルから100ドル以上のオーダーで変動する可能性があるからです。



ELM327は、 ELM Electronicsのオリジナルの開発および製品です。 実際のバージョン1.3a、1.4bおよび新しい2.0。 オリジナルのELM327に基づくソリューションのコストは、通常、マイクロコントローラー自体のコストが15〜25ドルであるため、約100ドルです。



ELM327クローン -ほとんどの場合、ELM327の初期の「保護されていない」バリアントの1つをコピーする中国のマイクロコントローラー。 元のコントローラー1.3a、1.4b、または存在しない1.4a、1.5、1.5a、v2.1のバージョンとして指定できます。 このようなアダプターのコストは、BluetoothおよびUSBソリューションでは約5ドルから、Wi-Fiでは15ドルからです。



STN1110はOBD Solutionsによる開発です。 ELM327と互換性のある独立した開発チーム。 いくつかの高度な機能があります。特に、OBD-II以外のコマンドを使用する場合、優れた機能（ELM327に比べて）の可用性を宣言しました。 データの読み取り速度も、ELM327バージョン1.xと比較して大幅に高速（2〜4倍）です。 バージョンELM327 2.0と比較すると、この機会はまだ現れていません。 そのようなアダプターのコスト（ OBDLink MXが 1つしかないことを知っています）は150ドルです。
