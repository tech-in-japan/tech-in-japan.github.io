スケーラビリティと復元力TelecityGroup Dynamic Data Center IaaS

クラウドサービスへの関心を背景に、多くのデータセンターオペレーターがサイトにクラウドプラットフォームを展開し、「クラウド」を大衆に積極的に宣伝し始めました。 TelecityGroup Finlandも例外ではありませんでした。 Dynamic Data Centerと呼ばれるIaaSソリューションの概要をご紹介します。





ダイナミックデータセンターソリューション図





TelecityGroup Finlandでは、彼らは「クラウド」について直接知っています。 最初のクラウドプラットフォームは2008年に導入され、この間にクラウドインフラストラクチャを構築するためのさまざまなハードウェアリソースとオプションがテストされました。 今年の初めに、プラットフォームは近代化され、顧客に提供されるIaaSソリューションはDynamic Data Center（略してDDC）と呼ばれました。 なぜDynamic Data Centerなのか、と尋ねますか？ まず、スケーラビリティと柔軟な容量拡張の可能性を反映したいと考えました。 さらに、さまざまなクラウドプロバイダーが提供するサービスには、「仮想サーバー」、「IaaS、PaaS、SaaS」、「クラウド」などの用語が数多くありますが、これはマーケティングの動きにすぎないこともあります。



DDCサービスと他のサービスの違いは何ですか？



主な「チップ」は、ソリューションの完全な耐障害性、ストレージシステムの卓越したパフォーマンス、追加サービスのパッケージ、さまざまなリソースアカウンティング、便利なセルフサービスポータルです。 しかし、まず最初に。



フェイルオーバーソリューション



物理的なサイトや通信チャネルから革新的なデータストレージシステムやブレードサーバーファームまで、あらゆるレベルで提供されます。



仮想インフラストラクチャが展開される物理プラットフォームは、プラットフォームの「充填」と同じくらい重要です。 データセンターが「立ち上がった」場合、重要なITシステムが作業の中断から失われることはありません。 DDCソリューションは、TelecityGroup HiomoとUspenskiの2つの独自のデータセンターに基づいて展開されています。 両方のデータセンターは、情報保護の信頼性、および潜在的なリスクを特定して排除するための一連の手段の可用性の追加確認として機能するISO 27001に従って認証されました。 顧客の要求に応じて、仮想マシンを追加のデータセンターに複製し、メインのデータセンターが利用できないときにそれらを起動することができます。 バックアップは、物理的に離れた3番目のSuvilahtiデータセンターに保存されます。



プラットフォームのハードウェアは、FC（ファイバーチャネル）テクノロジーを使用してSAN（ストレージエリアネットワーク）を介して接続されたIBM Flex Systemとストレージブレードで表されます。 このソリューションは、仮想化の世界的リーダーであるVMwareをベースに展開されています。 VMware High Availabilityのおかげで、一方の物理サーバーがクラッシュすると、VMはもう一方で自動的に再起動します（ VMwareの Webサイトで詳しく説明されています）。 さらに、各仮想マシンのイメージの毎日のバックアップコピーが作成され、そこから顧客の要求に応じて「落ちた」VMを復元できます。 さらに、エージェントベースのバックアップサービスを提供します。詳細は以下のとおりです。



ストレージに関しては、ディスクリソースのカテゴリによって異なります。 vStorage Silverカテゴリの場合、これはたとえばIBM DCS3700およびHitachiです。 ゴールドカテゴリ-IBM XIV Storage Systemの場合、これについて詳しく説明します。



その革新的なグリッドアーキテクチャにより、IBM XIV Storage Systemは比類のないパフォーマンスとスケーラビリティを提供します。 グリッドアーキテクチャは、同じタイプの多くのコンポーネントで構成され、共通の相互接続によって結合されます。 IBM XIVのディスクは「通信容器」として動作し、それによって均一な負荷を実現します。 すべてのIBM XIVコンポーネントはActive-Active N + 1用に予約されており、高レベルの信頼性と可用性を提供します。 コンポーネントに障害が発生した場合、最速の回復が可能です。



従来のストレージシステムとIBM XIVストレージシステムの比較



通信チャネルについて一言。 「外の世界」との通信は、冗長接続を使用してすべてのデータセンターを統合する当社独自の10 Gb / s光ファイバーネットワークによって提供されます。 通信チャネルの1つに障害が発生すると、別の通信チャネルに自動的に切り替わります。 デフォルトでは、100メガビット/秒の保証帯域幅が提供されますが、必要に応じて10ギガビット/秒に拡張できます。 大規模なお客様には、専用の通信チャネルの接続を手配できます。 TelecityGroupデータセンターは中立のアクセシビリティを備えていることを思い出させてください。 サイトに特定の通信プロバイダーが必要な場合は、いつでも歓迎します。 ダイナミックデータセンターソリューションは、フィンランドのデータセンターに基づいて展開されます→ロシアへの通信チャネルの遅​​延（他のヨーロッパ諸国と比較して）はごくわずかです。

追加サービスパッケージ



DDCサービスとともに、さまざまな追加サービスが提供されます。



•データバックアップ、2つのオプション：詳細およびシンプル



Commvalt Simpanaソフトウェアに基づく高度なオプションでは、セルフサービスポータルを使用して、バックアップスケジュールを自分で管理し、プロバイダーを介さずにデータをバックアップおよび復元できます。 シンプルオプションは、Veeam Backup＆Replicationに基づいて展開され、リモートサイトで7または30日間のストレージ期間でVMイメージをバックアップする機能を提供します。



•マネージドファイアウォールサービス

IPsecトンネリングのサポートと最大200 Mbpsのパフォーマンスを備えた基本レベルから高度レベルまでの3レベルのサービスプロビジョニング。

•Citrix Netscallerに基づく負荷分散

•ハイブリッドソリューション（物理+仮想インフラストラクチャ）

•専用通信チャネルの接続

•サーバー監視





さまざまなリソースアカウンティング



TelecityGroupは、常にクライアントへの個別のアプローチを目的としています。 仮想リソースのアカウンティングにさまざまなオプションを提供できることを嬉しく思います。また、お客様の現在のITインフラストラクチャを常に事前監査して、最適なオプションを選択します。



オプション1：コンピューティングリソースの割り当ての上限と下限は個別にネゴシエートされます。 下限は保証されており、上限と下限の間のリソースはオンデマンドで割り当てられ、消費時に支払われます。 これにより、支払いの柔軟性とスケーラビリティが提供されます。 ディスクリソースは消費時に支払われます。



オプション2：必要な仮想容量の量を予測できる顧客向けの固定月額料金のリソースのプール。



オプション3：時間単位の課金を伴うフルタイムの従量制モデル。



請求は自動化されており、まもなくセルフサービスポータルで支出資金の統計を直接表示できるようになります。 これで、サポートサービスから情報を取得できます。サポートサービスは、24時間体制で、週7日、お客様の関心の対象となります。 ところで、オプションに関係なく、前払いは必要ありません。



便利なセルフサービスポータル



セルフサービスポータルにより、顧客はクラウドに展開されたITリソースを完全に制御できます。





ポータル開始ページ



1つの仮想マシン（VM）からVMグループ（vApp）および仮想データセンター（vDC）全体に至るまで、仮想インフラストラクチャのすべてのレベルを管理することができます。 各vDC（たとえば、内部IT vDCおよび本番vDC）には、独自の請求と独自のSLAがあります。





4つのvAppで構成されるvDC



ポータルを通じて次のことができます。



•テンプレートの使用を含む、新しい仮想マシンの起動：ネイティブおよびTelecityGroupの両方のテンプレート。





•VMを一時停止してシャットダウンします。







•各vDCの物理サイトを選択します。

•サーバーの構成を変更し、複数のVM間でリソースを再分配します。

•vApp内でVMの起動シーケンスを設定します。



•OSをインストールします。

•ネットワーク構成の管理。







•ユーザー権限を管理します。







別のポータルでは、一定期間のコンピューティングリソースとディスクリソースの消費に関する統計情報を利用できます。 データは、1か月あたりの平均リソース消費量と時間単位のリソース消費量の両方で利用できます。

統計は1時間ごとに更新されます。
