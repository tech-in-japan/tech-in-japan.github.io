イルカ水族館のAndroid

あなたのポケットに収まるデバイスのすべての愛好家と、ポケットを7インチより広く持っている人を歓迎します。クロスプラットフォームグラフィックアプリケーション、つまりモバイルプラットフォーム（Androidスマートフォンとタブレット）で動作するアプリケーションのプログラミングの技術を取り上げます。 Windowsの場合（デスクトップPC、ラップトップ、ネットブック、タブレット）この場合、アプリケーションはグラフィカルであり、グラフィックはOpenGL（OGL）とモバイルバージョンのOpenGL ES（GLES）に基づいています。EmbarcaderoDelphi XE7を使用していますプラットフォームを使用していません FM（FireMonkey）、古き良き時代のように、私たちは自分たちですべてをゼロから書きます。







すぐに作業を半分に分割することを提案します。

-クロスプラットフォームのグラフィックライブラリを作成し、その使用例を検討し、実装の詳細をペイントし、質問に答えます。

-あなたの責任はそれほど大きくありません。さまざまなモバイルデバイスのデモテスト、実行に関するレポート、レビューと批判、新しいアイデアと新鮮な肉 、ライブラリとアプリケーション自体の新しいユーザーが必要です。



1.最初のデモ

Android mc.apk 3.8 MB yadi.sk/d/n1wDuX1RhC4QX

Windows mc.zip 1.9 MB yadi.sk/d/tBKQxaV7ff8KW

リポジトリ内のソース： sourceforge.net/p/delphioga/code/ci/default/tree

最初のバージョンでは、プログラムはトリッキーではありません-画面上に4096個の角のある色のボールを表示するだけで、「フリーカメラ」モードでそれらを飛び越えることができます。 RGBカラースペースの原理に従って球体をペイントしましたが、球体も回転します。 この例では、3Dプリミティブ（特に球体）を構築する方法と、レンダリングを最適化する方法（錐台カリング、オブジェクトの詳細化）を検討します。

しかし、私たちは小さな入門部分なしではできません。



2.取扱説明書、Androidスマートフォン/タブレット

1. mc.apkをインストールします。これはプログラムのインストールアーカイブです。 これを行う方法について質問がある場合は、いつでもお手伝いします。

2. apkをインストールすると、プログラムがインターネットにアクセスする必要があるという警告が表示される場合があります。 実際、プログラムはデバッグモードでコンパイルされているため、デバッガー（gdbserver）およびlogcatログコンソールが機能するためには、ローカルネットワークへのアクセスが必要になる可能性があります。 まだ必要なログ。 プログラム自体からは、ネットワークには一切アクセスせず、画面にグラフィックを描画するだけです。

3.インストール後、デスクトップ「mc」に新しいショートカットを追加します。 プログラムを実行します。 スクリーンセーバー-FMの光に感心します（デフォルトでは、Delphiモバイルプロジェクトのアイコンはまだ変更していません）。 アプリケーションの起動時間、約数秒でどれだけ持っているのか、そしてそのシェアに注意を払っていますか？ これをサンプルFMアプリケーションのダウンロード時間と比較することができます。これまでのところ、比較のほうが有利です。

4.ようこそ、ようこそ。 指とタッチスクリーンを使用して、視線の方向と動きの方向を変更します。 右上には、画面上のボタンがいくつかあります-前方と後方です。 手が2本ある場合は、移動とターンを同時に行うことができます。 これには、ある種の器用さ、本当に何かが必要であり、私は単純な生活を約束しませんでした。

5.別のアプリケーションに切り替えるか、タブレットを「スリープ状態」にすると（タブレットを持っているためモバイルデバイスをタブレットと呼びます）、プログラムはグラフィックスのレンダリングを停止します（GLESコンテキストとアプリケーションウィンドウ自体は削除されるため、Androidで実行する必要があります）、プログラム自体は停止しません仕事。 アプリケーションに切り替えるか、タブレットを「起動」すると、ウィンドウとコンテキストが再作成され、停止地点からの旅が続きます。 テストするときはこのニュアンスに注意を払う必要があります。一部のゲームは正しく「回復」する方法を知りませんが、できる必要があります。

6.タブレットを水平に、次に垂直に回します。 アプリケーションは、これら2つのモードのそれぞれで確認する必要があります。 もちろん、このようなトリックは、デバイスの表示方向が「自動」に設定されている場合にのみ機能します。

7.デモの終了は簡単です-「戻る」システムボタンをクリックします。

8.記事へのコメントに、使用したモバイルデバイスの種類（メーカー、モデル）を記入し、その特性、CPUとGPUの種類、OSバージョンを見つけようとします。 このデータをレポジトリのhardware.txtファイルに含めます。







3.操作説明書、Windows

1. zipアーカイブを解凍します。 exeファイルの量に驚くことはありません。デモはデバッグモードでコンパイルされ、exeリリースでは1.5 MBになります。 mc.exeを開始します。

Androidのようにネットワークの使用に関するシステム警告はありません。ネットワークはデモでは使用されないため、PCではデバッガーとログがローカルで動作します。

2.ようこそ、ようこそ。 視線の方向と動きの方向を変更するには、左マウスボタン（LMB）を押しながら画面を「ドラッグ」します。 スペースを移動するには、キーボードの[WASD]キーが機能します（矢印またはNumPadの矢印のいずれか）。 前後に移動するための画面上のボタンもあり、クリック可能です。 [F1]はヘルプを表示し、[ESC]はプログラムを終了できます。

3. 2番目のウィンドウ（プログラムが起動すると、コントロールパネルに2つのウィンドウが表示されます）に注意してください。これがデバッグコンソールです。 Android用の同様のコンソール（logcat）は、後で検討します。 ベンダーGPUの後に、レンダラー（ビデオカードの会社/タイプ）がOpen GLバージョンを書き込みます。 OpenGLのバージョンが3.0より低い場合、将来のライブラリ機能の一部が古いビデオカードで動作しないとだけ言ってみましょう。 ただし、アプリケーションには、このような機能の実装の2つのブランチを含めることができ、どのような場合でも機能します。

4.この例ではVSYNCがオフになっています。FPSは60 Hzをはるかに超えることがあり（可視オブジェクトから目を離してみてください）、GPUとCPUの負荷は最大です。 これは、フレームレンダリング速度をテストするために行います。VSYNCが有効になっている場合、負荷は正常になります。



4.画面へのデバッグ出力

左上のスクリーンショットには、いくつかの黄色の文字と数字があります。

FPSは、1秒あたりの平均フレーム数です。これは、描画速度を評価するための主なツールです。 VSYNCはAndroidで有効になり、FPSは画面のリフレッシュレート（60 Hz）によって制限されますが、デモでGPUの負荷を調整して、FPSがほとんどのモバイルデバイスで「沈む」ようにしました。 コントロールとアニメーションが適切である限り、それは怖くありません。 Windowsでは、VSYNCはオフで、FPSには制限がありません。

TDF -1フレームを描画する平均時間（ミリ秒単位）。実際はFPSの逆数。 「タイミング」を理解するために必要です-フレームを描画するために時間間隔が現れる順序。 FPS 60 Hzでは、この時間は約17 msです。

NFR-アプリケーションの開始から描画されたフレームの数。 最初の1秒間のFPS 60 Hzでは、この数値は60に達します。

T-アプリケーションの開始からのミリ秒単位の時間。アニメーション関数の引数として使用します。この例では、球体の回転はそれに依存しています。

POS -XYZ 3Dワールドのカメラ座標は、軸の始点で次のように配置されます。Xは右、Yは上、Zは私たちです。

AH、AV-水平および垂直に対するカメラの回転角度（度単位）。 ロール角については個別に話した方が良いでしょう。

Nは、ユニバース内のオブジェクトの総数です。

NV-全体的または部分的に非常に多くのオブジェクトが視界（錐台、カメラの視野の角錐台）に分類されます。

HD-非常に多くの近くのオブジェクトがわずかに優れたレンダリング品質で表示され、遠くのオブジェクトがより簡単に描画されます。

SCR-画面解像度。



5.開発プロセスはどのようなものですか？

USBケーブルを使用してWindowsでタブレットをPCに接続します（Wi-Fiも使用できます）。 まず、Android用の特定の機能のプラットフォーム固有のコードを記述し、タブレットでプログラムを起動します。 作業のこの部分には多くの忍耐が必要です。アプリケーションは比較的長時間タブレットでコンパイルおよび実行されます。 デバッグが可能です（ブレークポイント、ステップごとの実行、変数のステータスの表示）。 また、logcatでアプリケーションのデバッグ出力を使用します。リアルタイムでPCでログを直接監視できます。 タブレットで動作する場合、Windows用の機能コードのプラットフォーム固有バージョンを作成します。これは既に簡単です。 その後、楽しい部分が始まります-プログラムの主要部分をユニバーサルコードで記述し、同じ名前のプラットフォーム関数を呼び出し、PCでチェックし、たまにタブレットにアップロードするだけで、原則としてすべてがそこで機能します。



6.ボンネットの下

ソースコードを見てみましょう。 共通フォルダーには、ライブラリー自体があります。 Draw.pasファイルから始めましょう。

クロスプラットフォームグラフィックスは何に基づいていますか？ Windowsでは、OpenGL.pasモジュールを使用しますが、GLES for Androidに類似する機能（Androidapi.Egl.pas、Androidapi.Gles.pas、Androidapi.Gles2.pasモジュール）のみを使用します。 

uses Types,SysUtils, {$IFDEF Android} Androidapi.Egl, Androidapi.Gles, Androidapi.Gles2, {$ENDIF} {$IFDEF MSWINDOWS} Winapi.OpenGL, {$ENDIF} Vectors, Colors, Textures, DrawPolyhedron, DrawSphere;


OGLとGLESが同じ場合、構文にほとんど違いはなく、glOrthoのみをトリミングする必要がありました。 {$ IFDEF ANDROID}や{$ IFDEF MSWINDOWS}などの条件付きコンパイルディレクティブを使用して、プラットフォーム固有の実装について説明します。 

 {$IFDEF ANDROID} procedure glOrtho (left, right, bottom, top, zNear, zFar: GLfloat); begin glOrthof(left, right, bottom, top, zNear, zFar); end; {$ENDIF}


他に何が一致しませんか？ GLESにはglBegin / glEndがありません。したがって、GLESに関連付けられている関数ファミリ全体はありません。 怖くはありません。頂点の配列（法線、色、テクスチャ座標）へのポインタを一度に渡すOGL 2.0のスタイルで描画します。 たとえば、次のように単純な三角形を1つ描画できます（色は現在のもので、法線とテクスチャはありません）。 

 procedure Triangle(p1, p2, p3: TV3); overload; begin SetLength(ap3, 3); ap3[0] := p1; ap3[1] := p2; ap3[2] := p3; glVertexPointer(3, GL_FLOAT, 0, @ap3[0]); glDrawArrays(GL_TRIANGLES, 0, Length(ap3)); end;


3D図形の多くの三角形を描画するには、Triangleプロシージャを何度も呼び出さないでください。 すべての三角形のすべての頂点を1つの配列に追加し、glDrawArraysの1回の呼び出しでそれらを描画する必要があります。 この関数は、GL_TRIANGLESとGL_TRIANGLE_STRIPの両方で、現在のデモで各球体を描くために使用されます。 

 //     c  procedure DrawAVNT(avn: TAVertexN; met:GLenum; Position,Rotation,Scale: TV3); begin glPushMatrix; Transform(Position,Rotation,Scale); glVertexPointer(3, GL_FLOAT, SizeOf(TVertexN), @avn[0].V); glNormalPointer(GL_FLOAT, SizeOf(TVertexN), @avn[0].N); glDrawArrays(met,0,Length(avn));// GL_TRIANGLES glPopMatrix; end;
