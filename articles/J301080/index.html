Microsoftは、Windows 7用の2番目のサービスパックをリリースしました

それにもかかわらず、マイクロソフトはこの件のリリースを発表しましたが、ユーザーにそれを呼ばないように頼みます。 :) 2番目の更新パッケージは、ロールアップ更新または単に「累積更新パッケージ」と呼ばれ、本質的にはそうです。 更新セットはWindows 7 SP1のユーザー向けであり、必須ではなく、Windows Update Centerを通じて配布されることもありません。 サービスパックは、識別子KB3125574 （ Windows 7 SP1およびWindows Server 2008 R2 SP1用の便利なロールアップ更新）を受け取りました。







このサービスパックのリリースでMicrosoftが追求した主な目標は、Windows 7を使用している組織がロールアップパッケージ自体をオフラインのWindows 7ディストリビューションに統合できるため、最新の更新プログラムでOSをすばやくインストールできるようにすることです。サービスパックを公開するための形式を変更し、スタンドアロン配布の代わりに、Web経由でのみ使用できる既にかなり忘れられたMicrosoft Updateカタログメカニズムを使用して配布されるようになりました Internet Explorerブラウザー。



Windows 7にサービスパックをインストールするには、ユーザーはSP1をインストールし、 KB3020369を更新する必要があります（ 2015年4月Windows 7およびWindows Server 2008 R2のスタック更新サービス ）。 このパッケージには、SP1の最初から2016年4月までにリリースされたWindows 7のすべてのセキュリティおよびその他の更新が含まれています。WindowsServer 2008 R2のユーザーも同じ更新を利用できます。 以前、MicrosoftはWindows 7のService Pack 2のリリースを断念したことを思い出してください。



本日、Windows 7 SP1の新しい便利なロールアップを提供することをお知らせします。 この便利なロールアップパッケージは、 catalog.update.microsoft.com / v7 / site / Search.aspx？q = 3125574からダウンロードでき、Windows 7 SP1のリリース以降にリリースされたすべてのセキュリティおよびセキュリティ以外の修正が含まれています。この1つのアップデートをインストールすると、2016年4月以降にリリースされた新しいアップデートのみが必要になります。


アップデートの使用に関する詳細情報は、以下のリンクから入手できます。



blogs.technet.microsoft.com/windowsitpro/2016/05/17/simplifying-updates-for-windows-7-and-8-1

support.microsoft.com/en-us/kb/3125574



アップデート自体は、次のリンクからダウンロードできます。



catalog.update.microsoft.com/v7/site/Search.aspx?q=3125574



もう1つの注目すべき事実は、MicrosoftがWindows 7 SP1およびWindows 8.1（Windows Server 2008 R2 SP1、Windows Server 2012、およびWindows Server 2012 R2を含む）の非セキュリティ更新ポリシーを変更したことです。 以前に月に数回、部分的にリリースできる場合は、月に1回だけ厳密にリリースされ、単一の更新になります。 更新プログラムは引き続きWindows Updateから入手できます。
