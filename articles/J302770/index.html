SQL Server 2016の新機能について

データベースは、文字通りすべてのビジネスラインの健全性を保証する企業の心臓部です。 しかし、現代の現実ではデータベースはどの場所を占めていますか？ 結局、彼らは今日現れなかった。 それらは、テクノロジーの世界が今のようになるとは想像もできなかった時に作成されました。 従来のデータベースは、注文や支払いなどのトランザクションの記録と取得のみを目的としていました。 彼らの仕事は、小規模または中規模のローカルデータセンターでアプリケーションの信頼性と安全性を確保することでした。









当社の製品-SQL Server 2016-は、最新の技術トレンドの可能性を最大限に引き出すのに役立ちます。 ハイブリッドトランザクション処理、高度な分析と機械学習、モバイルビジネスインテリジェンス、データ統合、暗号化された要求処理、およびRAM内のトランザクションの操作をサポートします。 明らかに、これはこの種の唯一のリレーショナルデータベースであり、もともとはクラウド向けでした。 その機能のほとんどは、もともと世界中に散らばる22のデータセンターにあるAzureで開発およびテストされており、1日に数十億のリクエストがありました。 彼女はすでに実際に優れた結果を示しています。 多くのお客様がこのデータベースを正常に使用しています。

R言語を使用したデータベース内の分析



プログラミング言語Rは、データ分析および予測用のソフトウェアとして広く使用されています。 無料のオープンソースであり、大規模なコミュニティによってサポートされています。 数千のパッケージを使用して、さまざまな問題を解決できます。 あなたのサービスで-標準統計とデータ管理機能、グラフィカルインターフェース、高度な機械学習アルゴリズム。 R言語は、最新の分析手法を使用して革新を望む科学者や企業によって選択されます。



SQL Server R Servicesは、SQL Server 2016の新しいコンポーネントです。R言語、増え続けるパッケージセット、大規模なユーザーコミュニティを引き続き活用できますが、SQL Serverの機能と使い慣れたTransact-SQLインターフェイスにもアクセスできます。

  • Rサービスを使用すると、データベース内で分析を実行できます。 SQL Server 2016にデータを保存でき、アプリケーションはT-SQLストアドプロシージャを介してRスクリプトを呼び出します。 これにより、アプリケーションとR言語の統合が簡単になります.Rスクリプトは、データが保存されているのと同じマシンで実行されるため、セキュリティを心配する必要はありません：Rコードは、インターネットまたはSQL Serverで実行されている他のプロセスにアクセスできません。

  • Rサービスのユーザーは、 ScaleRアルゴリズムライブラリを使用できます。これは、Rで実行される日常のタスクと同等の機能を提供する一連の機能です。 これにより、一般的なオープンソースパッケージでは達成できないパフォーマンスが提供されます。 ScaleR APIは、Revolution Analyticsによって開発されました。 Microsoftがこの会社を買収した後、APIはSQL Serverに統合されました。 これらはクロスプラットフォームです。つまり、SQL Serverだけでなく、他のプラットフォームもサポートします。

  • SQL Serverは、統合と生産性向上のための既存の機能とメカニズムも提供します。 たとえば、R言語とともにColumnStoreインデックスを使用して、分析クエリを高速化することができます。 組み込みの管理メカニズムにより、Rランタイムに割り当てられたリソースを制御でき、SQL Server Integration Services（SSIS）は、ストアドプロシージャインターフェイスを介して、シームレスなETL統合とスケジュールされたタスクのスケジューリングを提供します。

  • Rサービスのもう1つの重要な機能は、ユーザーが使い慣れたツールで作業できることです。

    • データサイエンティストは、使い慣れたR IDEを使用できます。 彼らは、R Studio、最近発表されたVisual Studioツール用のR Tools 、または任意のIDEと連携できます。 ローカルマシンにデータを転送する代わりに、ScaleRツールを使用して、IDEから直接リモートで実行できます。 データベース内の分析にSQL Server 2016ツールを使用すると、大規模なデータセットの分析を高速化できます。

    • サーバー管理者のサービスでは、Rサービスの管理、リソースの制御、およびアクセス権の割り当てのためのSQL Server Management Studioソフトウェアに精通しています。

    • T-SQL APIは、アプリケーション開発者が利用できます。 R言語自体を習得することなく、Rからの評価、予測、グラフィックスを含むSQL Server Reporting ServicesレポートまたはPower BIダッシュボードを作成できます。

    • データエンジニアは、Rを既存のETLストリームと組み合わせて、SQL Server Integration Servicesを使用してメンテナンスタスクをスケジュールできます。


SQL Server Rサービスは、スケーラビリティ、産業環境のアプリケーションとの統合、および低コストを提供する独自のRベースのデータ分析ソリューションです。

PolyBaseの紹介



過去10年間で、情報技術の世界は認識を超えて変化しました。 モノのインターネット、非構造化データ処理技術、およびIT機器の低価格はすべて、真のビッグデータブームをもたらしました。 HadoopやHDFSなど、半構造化データの膨大な配列を処理できると同時に、高価な専用ハードウェアを購入する必要のないソリューションがあります。 これらすべてが利益のための新しいビジネスチャンスを切り開きましたが、「コインの裏返し」は、エンタープライズデータプラットフォームの複雑さの増大でした。 さらに、データソースははるかに多様化しています。 多くの場合、企業は複数の異なるデータセットを一度に処理する必要があります。SQLServerのリレーショナルデータとHDFSの非リレーショナルデータセットです。 アナリストが半構造化データと構造化データの分析を組み合わせる必要がある場合、最初にそれらをある媒体から別の媒体にコピーする必要があり、多くの時間と労力がかかります。



しかし、世界のすべてがより良く変化しています！ SQL Server 2016のPolyBaseソリューションは、リレーショナルデータおよび半構造化データの断片化の問題を取り除きます。 PolyBaseとT-SQLを使用すると、ユーザーはローカルのSQL Serverに保存されているかのようにHDFSデータにリクエストを送信でき、分析の新しい可能性が広がります。







使用例

PolyBaseは、SQL ServerとHadoopで3つのデータアクセスシナリオを提供します。

  • T-SQLを使用したHadoopおよびSQL Server 2016への1回限りのクエリ

  • SQL Server 2016のHadoopまたはAzure BLOBストレージからデータをインポートする

  • リクエストを同時に送信する機能を維持しながら、コールドリレーショナルデータをHadoopまたはAzure Blobストレージにエクスポートします。


データ要求をHadoopに送信する際、PolyBaseはこのプラットフォームのコンピューティングリソースを最も効率的に使用するよう努めています。 特に、PolyBaseは、最も効率的なクエリ処理を提供するMapReduceジョブを作成することにより、計算負荷をHadoopに転送することを決定できます。

Stretch Databaseテクノロジーによる無制限のデータストレージ



SQL Server 2016の一部であるStretch Databaseテクノロジーを使用すると、ビジネスサービスのレベルやエンタープライズストレージシステムの高コストに関する合意に違反することなく、必要な期間、任意の量のデータを保存できます。 コールドデータの保存に使用される従来のソリューションとは異なり、Stretch Databaseは、Azureの無尽蔵のクラウドリソースを使用してデータへの継続的なアクセスを提供し、ほとんどのアプリケーションを変更する必要もありません。 データベース管理者は、クラウド内の「コールド」データのストレージをアクティブ化するだけで済みます。



主なシナリオ

  • テーブル全体をクラウドに転送します。「コールド」データ用の既存の専用テーブルを完全に転送できます。 Order_detailsテーブルとOrder_details_historyテーブルがあり、最後のテーブルには最初のテーブルから転送された「コールド」データのみが含まれているとします。

  • 「コールド」行のラップ：同じテーブルに「ホット」データと「コールド」データの両方がある場合、「コールド」データのみをAzureに転送できます。 これを行うには、行を1つだけ指定します（通常、これは日付またはステータスによって決定されます）。SQLServerが転送を処理します。


メリット

Azure SQL Stretch Databaseを使用すると、条件に応じてAzureを使用できます。

  • エンタープライズクラスのストレージ-必要な量と時期を取得します。 自動バックアップとジオレプリケーションはデフォルトで有効になっています。

  • 必要に応じて、作業タスクの要件に応じてコンピューティングリソースとストレージスペースを拡張し、使用済みのボリュームに対してのみ支払います。

  • 組み込みのセキュリティメカニズムを使用して、オンプレミスのActive DirectoryとAzure Active Directoryを統合した顧客のアクセスを集中管理します。

  • SQL Server Management Studio、SQL Server Data Tools、T-SQL、PowerShellなどの既存の知識とツールを使用し、Azureポータルを通じて機能を拡張します。


Stretch Databaseを使用するために、ほとんどのアプリケーションはコードを変更する必要がないことを思い出してください。
