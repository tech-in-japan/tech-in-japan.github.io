CodeGearがRAD Studio 2007を発表

CodeGear製品（開発ツールが移管されたBorlandが所有する会社）のサポーターにとって重要なニュース： RAD Studio 2007 （コード名「Highlander」）のリリースが発表されました -Delphi for Win 32とDelphi forを組み合わせたオールインワン開発パッケージ.NETおよびC ++ Builder。



公式発表では、以前のバージョン-Borland Developer Studio 2006の一部であったC＃Builderについては何も言及していないことに注意してください。新しいバージョンに捧げられたプレゼンテーションの1つで、「IDEはもはやC＃の設計時サポートを提供していません」-これらの時代です、C＃Builderは別れを告げる必要があり、この製品に夢中になった少数の開発者はVSに移行せざるを得ないようです。



革新の中で、特に革新的なものは見つかりませんでした。最も重要なのは、.NET 2.0およびASP.NET 2.0のサポートです。ECOIV MDDフレームワークの新しいバージョンであるdbExpress 4（Delphi 2007で年初にデビュー）、組み込みビルドエンジンをMSBuild（また、Delphi 2007で以前に登場しました）主にC＃から借用したDelphi言語のいくつかの拡張、およびJavaで書かれたSteve Shaughnessyが開発した軽量のBlackfish SQL DBMS（いわゆるユニークなDBMS JDataStore / NDataStore C＃に変換され、.NETでも動作します ）



想定される可能性はありますが、新しいバージョンはすべてのCodeGear IDEを組み合わせた真の統合ソリューションにはなりません。たとえば、 Delphi for PHPは独立した製品であり、 Ruby IDEは今年の第3四半期にリリースされる予定です。



RAD Studioが.NET 2.0のサポートを導入したのは良いことです。VisualStudioで2年が経過しているのは悲しいことです。



その結果、Borland / CodeGearがMicrosoftに追いつくためのすべての努力にもかかわらず、状況はまったく逆です：競合企業の製品のリリースごとにギャップが広がり、一般的に新しいバージョンは戦闘損失に驚かされます（C＃Builderが生き埋めになっています）。 正確な統計情報は知りませんが、Delphi for .NETの支持者はそれほど多くないようで、主な収入はDelphi for Win32を使用する「古典的な」開発者にかかっています。

CodeGearがRAD Studioをエンタープライズアプリケーションを作成するためのツールとして宣伝することで、Java / .NET（特に.NETに関してはもちろんVS）が特に強力で、誰もが強化されているという特定の成功を収めることはまずありません。 たとえば、CodeGearでのECOのようなものは、実に非常に興味深いものです。 たぶん、何らかの独自の市場戦略を考え出し、モンスターとの競争をやめることは理にかなっていますか？ たとえば、ECOを分離し、Visual Studioの付属物として販売しますか？ または、VCL for the Web（以前のIntraWeb）に注目し、このアプローチに基づいてWebアプリケーションを開発するための軽量ツールを提供しますか？ またはKylixを復活させますか？ まあ、少なくとも価格を下げても害はありません-特に新しいバージョンが頻繁に出てくることを考えると、彼らは宇宙のように見えます（たとえば、Professionalバージョンは$ 1.2Kに加えて年間サポート料金$ 350）。長年更新せずに特定のバージョンでDelphiに海賊版のままでした。とにかく、新しいバージョンに根本的な改善がなければ、どうしてもっと払うのでしょうか。



ちなみに、別の会社を隔離している最中のボーランドの代表者が、機能性と品質の真の突破口となるのはハイランダーだと言ったことを覚えています。 しかし、品質を判断するのが早すぎる場合、製品はまだリリースされておらず、発表されているだけなので、機能に関する質問があります。 たとえば、なぜ発表でVCL for Win32のUnicodeサポートについて言及されていないのに、「ネイティブ」のDelphic 64ビットコンパイラについて何も言わないのですか？ そして、これに加えて、約束が聞かれた他の多くの約束された機能については何も言われていません。「少しだけ持って、これはすべてハイランダーにあります。」



私は個人的に、リリースバージョンに固有の制限を理解していません。たとえば、Professionalバージョンでは、512 MB以下のサイズのBlackfish SQLデータベースを作成でき、1（！）ユーザーおよび1プロセッサーをサポートし、VCL for the Webを使用するアプリケーションをサポートします（注）： 5（！）接続で最大。 1人だけがログインできるようにDBMSを必要とするのは誰ですか？また、最大5人のユーザーにアクセスできるサイトが必要な理由は何ですか？！ :)

狂気の頂点-Enterprise Blackfish SQLのバージョンでは、1プロセッサに制限されていますが、データベースファイルサイズは2 GBで、最大5（！）ユーザーです。 データベースに接続できるのは5人だけの「エンタープライズ」とはどのようなものですか？！

制限を説明するのは簡単ですが、追加料金で制限が解除されることはどこでも控えめに述べられています。 しかし、私の意見では、これはやり過ぎです。たとえば、無料のMS SQL Server 2005 Express Edition（さらに完全に無料で、スタジオを購入する必要はありません）は4 GBのデータベースサイズによって制限されます。 RAD Studio 2007のエンタープライズバージョンの「ブラックフィッシュ」は、プロフェッショナルバージョンの8倍です。 さらに、SQL Server ExpressにはBlackfishと比較して1つの追加の制限があります-1 GBを超える使用済みメモリがなく、リモート接続に制限はありません（SQL Server ExpressはローカルDBMSであるという一般的な考えに反します）。



Delphiプログラマーの需要が低下しているのは悲しいことです。資格のあるDelphi開発者のためのまともな仕事を見つけることができれば、西側では非常に困難です。

すべてがそれほど悲惨なわけではありませんが、たとえばdelphiplus.orgには、ロシアではすべての学校のコンピューターに、Delphiがカテゴリ「開発ツール」に含まれるライセンスソフトウェアが装備されるという情報（9月2日のニュースエントリ）があります。 ロシアは、Delphiの人気を復活させるのに役立つかもしれません。あるいは、海外での頭脳流出を減らすためのロシア政府の秘密の計画の実施かもしれません。 :)



興味深い観察：秋にリリースされるバージョンは2007インデックスを取得し、過去数年間、ボーランドは秋にフラッグシップ製品をリリースしましたが、バージョン番号は翌年に対応しています。 ボーランド製品のバージョン番号は従来よりもマイクロソフトのバージョン番号が1高いという事実にもかかわらず、この瞬間はRAD Studioの主要なライバルであるMicrosoft Visual Studioのバージョン番号2008の差し迫ったリリースに照らして特に興味深く見えます。 RAD Studio 2008のリリースはすぐに予想されます。特にCodeGearにはMicrosoftの機能に追いつくための作業がありますか？



バージョン番号の変更されたポリシーには別の説明がありますが、原則として、Borland / CodeGear開発者ツールはMicrosoft製品よりも頻繁に更新されます。バージョン番号を使用するのが不合理な場合は、単に十分ではありません。 :)



著者から：私は自分の投稿のテキストを読み直し、「健康のために始められた-平和のために終わりました」というカテゴリーから得られたものを見る。 しかし、ここでは、どこにも行けないようです-私は、新しいリリースに関する情報を収集し、公式資料を読み、プレゼンテーション、ブロック内のメモ、およびプレゼンテーションを見ることに楽観的でした-しかし、私が収集した情報が多いほど、失望しました。 HighlanderはRAD Studioの最も期待されているバージョンです。Borlandによって開発されたのではなく、CodeGearによって、Borland製品のサポーターが奇跡を期待したのはこのリリースからでした。 しかし、彼は本当に望んでいましたが、奇跡は起こらなかったようです。 いずれにせよ、健全な批判は賛成のみです。おそらくCodeGearの従業員はこの投稿を読んでこの問題について意見を述べるでしょう。SergeyOrlikが私のブログにコメントを残したことは覚えていますが、私は知りません-彼は現在BorlandまたはCodeGearにいます。



Vyacheslav Koldovsky、 Developers.Org.Ua
