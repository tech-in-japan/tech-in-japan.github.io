箱入りPBXを支持する5つの引数

クラウドソリューションの魅力にもかかわらず、お客様がBoxを選択する状況は数多くあります。 今日は、企業が「鉄」ソリューションを選択する5つの理由を検討し、箱入り電話交換機が提供する可能性を説明します。









→ 拡大表示



ブログで何度も書いたクラウド製品に加えて、 iptelefonはZeonソフトウェアIP-PBXも提供しています。 このボックスソリューションにより、既製のPBXを任意のサーバーまたは仮想マシンにインストールして、可能な限り幅広い機能を利用したり、既製のソフトウェアおよびハードウェアソリューション（つまり、既に機能する「ボックス」）を購入したりできます。



今日は、クラウド電話交換を作成することが不可能または非効率になるいくつかの状況に触れます。 同時に、Zeon IP-PBXなどのボックスIPソリューションは、優れた代替手段になりつつあります。



すべてのテレフォニー、チャネル、ルートを個別に管理したい



クラウドベースのPBXの柔軟性にもかかわらず、コールがどのノードを通過するか、トラフィックがどのノードを通過するかを正確に決定することはできません-通常、これらのタスクにはいくつかの選択肢がありますが、それ以上はありません。 サーバーで実行するPBXを選択すると、すべての通話のルーティングを手動で決定できます。 箱入り製品を使用すると、たとえば、従業員の携帯電話やIPソフトフォンに通話を転送する場合など、音声トラフィックを転送するためのルールを設定できます...



自動電話交換ゼオンの動作原理









E1ストリームが届きます



E1チャネル（30行のデジタルストリーム）がオフィスまたは本番サイトに来て、単一の最新のPBXを介してそれらを使用したい場合、ローカルソリューションの展開がより効率的になります。 この場合、回線は単一のリソースプールに接続され、指定した規則に従ってIPテレフォニーとともに使用されます。 Zeon IP PBXは、どの音声ゲートウェイよりも柔軟性があり、セットアップは簡単です。



会社のインターネットアクセスが大幅に制限されている場合



情報漏えいが容認できない企業があり、ほとんどの電話は内部的なものです。 同様の状況は、ファイアウォールでトラフィックをフィルタリングするためのルールが非常に厳しい機密企業で発生します。 この場合、外部クラウドを忘れることができますが、ネットワーク内に安全な通信サーバーをインストールし、暗号化されたチャネルを介して音声情報を送信する許可を得ることができます。 Zeon IP PBXでは、これはそのまま使用できます。



CRMシステムとの特別な統合が必要です



最新のクラウドPBXのほとんどには、一般的なCRMシステムとの統合モジュールが既にあります。 たとえば、当社のHiveは、amoCRM、クライアント、その他のシステムと連携して動作します。 ただし、仮想PBXを別のCRMと統合する場合は、非標準モジュールをクラウドソリューションに接続することなく、サーバーで直接これを行う方がはるかに簡単です。 ZeonはすでにBitrix24、amoCRM、Clientix、retailCRM、Yclientsと統合されています。CRMについては、マネージャーに確認してください。



大規模なコンタクトセンターを作成する予定です。



最後に、計画に大規模なコンタクトセンターの作成が含まれる場合、機能が豊富であるため、ボックス化された製品でこれを実現できます。 Zeon IP-PBXには、詳細レポートの作成、キューの操作、トリガーシステム（障害の場合にPBXステータスの自動切り替えが可能）、監視ツール、不在着信の通知、自動ダイヤラー、クライアントカード（ミニCRM）、準備完了のためのコンポーネントが既にあります仮想オペレータワークステーション、一時停止、盗聴、発散、会話への導入、権利の制限など。



IP PBX Zeon



上記のすべてのケースで、ZeonボックスIP-PBXの方がうまく機能します。 サーバーまたは仮想マシンにインストールできます。 または、データセンターにPBXを配置できます。



Zeon IP-PBXの一部として、コールセンター、オートダイヤル、統計、連絡先、ビジネスチャット、会議、人気のあるCRMシステムのいずれかとの既製の統合などのモジュールを使用したり、オープンAPIを介して独自のソフトウェアを接続したりすることもできます。
