Microsoft Azureの仮想マシンの信頼性の向上

7月の大きな更新では 、Microsoft Azureに関する重要なニュースをもう少し見逃してしまいました。











最近の発表によると、仮想化ホストのパッチメカニズムの改善とインプレース移行の使用により、単一の仮想マシンのダウンタイムが短縮されました。

現在、ほとんどの仮想マシンは、ローカルの一時ディスクとメモリの状態を保持しながら、最大30秒間だけ一時停止するAzureインプレース仮想マシンの移行の恩恵を受けています。




既にMicrosoftクラウドサービスを使用している人にとって、 99.95％SLAが 可用性セットに統合された仮想マシンに対してのみ保証されることは秘密ではありません。 単一の仮想マシンにはこのような保証はありません。また、ホストでの作業が原因でサーバーが再起動する場合があります。



Azureの使用を検討しているお客様にとって、この不快な瞬間を頻繁に祝わなければならないので、Microsoftがこれに取り組み続けていることを嬉しく思います。 単一の仮想マシンの信頼性を改善し続ける意欲もあります。

2016年後半に予定されているプラ​​ットフォームアップデートにより、ホストオペレーティングシステム上のほとんどのパッチで仮想マシンを再起動する必要がなくなります。


論理的な次のステップは、スケジュールされた作業中だけでなく、ホストで問題が発生した場合のマシンの移行です。



Microsoft Azureの単一の仮想マシンにSLAを導入することを引き続き楽しみにしています。
