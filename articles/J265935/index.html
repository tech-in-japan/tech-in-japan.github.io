osxまたはlinuxでアクセスウィンドウを迅速かつ無料にする方法

開発者の間では、メインの作業コンピューターがmacまたはlinuxであることは珍しくありませんが、Windowsにアクセスする必要がある場合があります：特定のプログラムを実行し、Internet Explorerでレイアウトを確認し、cores drawの試用版を.svgに変換します。



この問題を解決する「古典的な」方法は、人気のある仮想マシンの1つをインストールし、90日間の試用版のWindowsをダウンロードして、仮想マシンにインストールすることです。 しかし、この方法には大きなマイナス点が1つあります。開発者は通常、「ああ、ここにはWindowsが必要だ！」という瞬間に、そのような仮想マシンを手で引き裂く必要があることを思い出します。 また、ウィンドウはすぐに設定されません。 そして、特徴的な90日間のトライアルは、最も不適切な瞬間に終了する傾向があります。 リアームしても。 最近、Microsoftは開発者の会議に参加し、目的のバージョンのWindowsにすばやく無料でアクセスできる興味深いサービスを作成しました。



このサービスは元々Web開発者向けに作成されたため、新しいMicrosoft Edgeを含むさまざまなバージョンのブラウザでサイトとアプリケーションをすばやくテストできます。 サービスはdev.modern.ie/tools/vmsで利用可能です



このサービスを使用すると、3つの仮想マシン（Parallels、virtualbox、またはwmvare）のいずれかに完全に構​​成された目的のInternet Explorerを使用して、目的のバージョンのWindowsのイメージをダウンロードできます。 WindowsのXPから10までのバージョンが利用可能で、ダウンロードしたイメージがすぐに開始されるという大きな利点があります。どこにも何もインストールする必要はありません。 高速インターネットを使用している場合、1ギガバイトのWindows XPが数分でダウンロードされ、さらに1分で仮想マシンにイメージが接続され、「ああ、Windowsが必要です！」の3分後に実行中のWindowsがあります。



これがosx、virtualbox仮想マシン（テクニカルエバンジェリストがそうすべきだと言ったため）およびwindows xp（クライアントが奇妙だから）の例でどのように機能するかを説明します。 最初に、仮想マシンの目的のバージョンのイメージでzipアーカイブをダウンロードします。











oracle virtualboxをインストールして実行します。 公式サイトから、自家製の樽を使用できます： 



sudo brew cask install virtualbox


[ファイル/アプライアンスのインポート]を選択し、ダウンロードしたzipアーカイブから解凍された.ovaファイルを指定します。 アプライアンスは、ハードドライブの内容だけでなく設定も含むvirtualboxのエクスポートされた仮想マシンイメージです。 エクスポートプロセス中に、Microsoftのユーザーが構成したものが気に入らない場合は、仮想マシンの任意の名前を指定し、使用可能なメモリの量と残りの設定を変更できます。











インポートされたアプライアンスは、仮想マシンのvirtualboxリストに表示され、起動する準備ができています。 数秒待って-そして、ここでは、Windowsです。 すばやく簡単に：











興味深い点は、Microsoft自身が最初の起動の前に仮想マシンのスナップショットを取ることを推奨していることです。これにより、試用期間が終了した後に仮想マシンが復元されます。 これが彼らのライセンスポリシーにどのように適合するかはわかりませんが、事実は残っています。 どうやら、Internet Explorerでデバッグ目的のために-それは可能です。
