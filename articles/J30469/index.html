nVidia 8/9シリーズカードのUbuntu 2Dグラフィックスの高速化

0）ベータ版ドライバーの配布をダウンロードします



投稿の執筆時点では、最後の投稿（6月17日）

www.nvidia.com/object/linux_display_ia32_177.13.html







ビデオ用のドライバーを初めてインストールしますか？ パッケージを忘れずにインストールしてください。

カーネルモジュールをコンパイルするために必要になります。



linux-headers-`uname -r`

xserver-org-dev

build-essential







1）公式のnvidiaドライバーを削除する



-file / etc / modules、ファイルの最後に「nvidia」を追加

-ファイル/ etc / default / linux-restricted-modules-common。まだ存在しない場合は「nv」を追加します



2）以前のドライバーをインストールしたものに応じて、実行します



sudo envy --uninstall-all

sudo dpkg -P envy

sudo apt-get remove --purge nvidia*

sudo rm /lib/restricted-modules/.nvidia*

sudo nvidia-installer --uninstall







nvidia Webサイトからダウンロードしたディストリビューションから以前のドライバーをインストールしたため、最後のコマンドで十分でした。



/etc/X11/xorg.confを編集し、次のオプションを「デバイス」セクションに追加します。



Option "PixmapCacheSize" "300000"

Option "OnDemandVBlankInterrupts" "True"







3）新しいドライバーをインストールする



別のコンソールに切り替えます（CTRL + ALT + F1）

gdm / kdm（/etc/init.d/gdm stop）を停止します



NVIDIA-Linux-x86-177.13-pkg1.runドライバーのルートインストールファイルを実行し、実行するためのアクセス許可を設定します（+ x）​​。

構成ファイルを更新する要求を拒否します。

再起動します。



4）追加の最適化



システムの起動にコマンドを追加します（グラフィカル）。

gnomeでは-セッションマネージャーで実行できます。



nvidia-settings -a InitialPixmapPlacement=2 -a GlyphCache=1







合計



2Dアプリケーションのパフォーマンスが大幅に向上しました！

compizを使用したgnomeでは、4つ目のKDEを試しませんでした。以前のドライバーと比較して、すべてが速く実行されます。

しかし、3D速度は低下しました（glxgearsによる判断）。 私にとって重要ではありません。
