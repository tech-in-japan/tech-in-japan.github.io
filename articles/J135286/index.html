Microsoft Lyncがモバイルプラットフォーム、クライアントレビュー、サーバーセットアップを開始





親愛なるKhabrosocommunityよろしくお願いします。

11月、マイクロソフトは、モバイルクライアントの接続を可能にする、待望のLync Server 2010エンタープライズコミュニケーションサーバーアップデートをリリースしました。 ただし、その時点では、ネイティブのWindows Phoneであっても、どのプラットフォームでも単一のモバイルクライアントはありませんでした。 初期のプレスリリースから、MicrosoftはWP7、Android、iOS、Symbian、およびBlackBerry用のLyncクライアントの開発を開始することを明確にしました。 幸いなことに、悲しむことはそれほど長くはありませんでした。







先週、LyncクライアントはWindows Phone 7（Mango）およびAndroidで登場し、今週はiOS（iPhone、iPad）で登場しました。クライアントは先日Symbianを起動し、1月にBlackBerryで登場する予定です。 さまざまなプラットフォームのクライアントの機能はさまざまです。比較プレートを次に示します。







奇妙なことに、競合するすべてのプラットフォームの中で、Androidだけが最も洗練された機能を備えています。



すべてのプラットフォーム向けのLync Mobileの主な機能：

-同僚の存在に関する情報、最適な通信方法の選択-インスタントメッセージ、電子メール、または電話の使用。

-会議を整理する簡単な方法。

-企業の音声通信でコールを発信および受信するための単一のインターフェース

-VPNを必要とせず、TLSを介したデータ送信の保護。



通話の整理：

Lync連絡先間の音声通信は、コールバックを使用してPSTNゲートウェイを介して行われます。 このメソッドによる呼び出しの編成は、肯定的な観点と否定的な観点の両方から解釈できます。



+通話はVoIPを経由しないため、インターネットのカバレッジ（Wi-Fi、3G）に依存せず、ローミングのインターネットは音声よりも高価なことがよくあります。 コールバックを使用すると、通話の価格は気にせず、会社はすべての費用を負担しますが、無料の着信通話が必要です。 この方法を使用すると、ビジネスの主な利点が達成されます-常に連絡を取り合う

-PSTNゲートウェイを使用して通話を整理するための、より複雑で高価なスキーム。 通話のコストを削減するために、SIPプロバイダーを使用できますが、それらのほとんどはLync Serverと互換性がありません。 この問題を解決するには、アスタリスクを使用できます（これを行う方法に興味がある場合は、記事を書くことができます）



モバイルクライアントに関する批判のトピックを継続：

-WP7やiOSとは異なり、Androidのプッシュテクノロジーの欠如

-WP7のタイトルの色は常に青で、携帯電話のテーマを変更しても色は変わりません

-タブレットの不十分なクライアント最適化

-Androidでクリップボードが機能しない

-ビデオ通話をしたり、ファイルを転送する機能はありません

Lync Server 2010モビリティサービスセットアップガイド



コンポーネントをインストールする前に、最新の累積更新Lync Server 2010 Mobility Serviceをインストールする必要があります 。 リスト全体から、LyncServerUpdateInstaller.exeを選択し、フロントエンドサーバーとエッジサーバーにインストールします。

また、インストールする前に、トポロジを変更する必要があります-モビリティサービスの内部ポートと外部ポートを追加します。 これを行うには、Lync Server管理シェルを開き、次のコマンドを入力します。



Set-CsWebServer –Identity –McxSipPrimaryListeningPort 5086

Set-CsWebServer –ID –McxSipExternalListeningPort 5087

Enable-CsTopology –詳細



管理シェルを使用して、不足しているサービスのコンポーネントをインストールします。



インポートモジュールServerManager

Add-WindowsFeature Webサーバー、Web-Dyn-Compression



Lync Server 2010 Mobility Serviceの更新プログラムをダウンロードしてインストールする

Bootstrapper.exeがシャットダウンすると、Lync Mobilityサービスの準備が整います。 内部および外部で使用するために証明書を更新し、FQDNサーバーをDNSサーバーに追加するためにのみ残ります。

内部ゾーンの場合、フロントエンドサーバーを指す新しいAレコードlyncdiscoverinternal.sipdomainを作成し、外部DNSサーバー上で、フロントエンドの外部fqdnを指すレコードlyncdiscoverを作成する必要があります。

内部証明書を更新するには、[Lync Serverシステムのインストールまたは更新]セクションのLync Server展開ウィザードを使用して、ステップ3を選択します。証明書要求ウィザードを使用して、lyncdiscoverInternal.sipdomainおよびlyncdiscover.sipdomainという名前を含む新しい証明書を発行します。

外部証明書として、既存のワイルドカード（* .sipdomain）を使用するか、CN lyncdiscover.sipdomainを使用するか、SANフィールドにこの名前を含む新しい証明書を注文できます。



Test-CsMcxP2PIMコマンドレットを使用して、新しいサービスの正常性を確認できます。
