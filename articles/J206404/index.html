初級レベルで心の中でどのくらい、どれくらい早く考えますか？

友人、こんにちは。 開発に挑戦するために技術教育を受けることにしました。 どれだけ成功するか、まったく成功するかどうかはわかりませんが、試してみないとわからないでしょうが、それはそれについてではありません。



最近、私の友人の一人が「大規模な国際企業」でインタビューを受け、インタビューのそのような特徴について話しました。 彼は質問をされ、彼に答え始めました、そして、答えの間、彼は中断され、「84％の12％」などの計算をすばやく実行するように求められました。 m / s-何km / h？」など、同時に、彼は答えた主なアイデアから逸脱するべきではありませんでした。 その結果、友人はそこに何か答えましたが、自分にとっては深刻な問題だと思いました。私はゆっくりと思い、「RAM」の予備も非常に小さいので、私はどんな割合でも私は心を決めるつもりはないので、問題なくはっきりと見えるように書き留める必要があります。



しかし、私が伝えたかったのはそれだけではありません。 会話の過程で、友人と私は非常に基本的なレベルで異なる考え方をしていることが判明しました。 単純な算術のレベル。 彼がどのように数えたのかを正確に覚えていないので、私が話していることを理解できるように、私自身の例を説明します。



たとえば、最も近い便利な番号から開始すると思います（条件付きで参照番号を呼び出します）。 どういう意味ですか？ 個人的には、いくつかの数字は他の数字より簡単だと考えられています。 たとえば、7 + 5は7 + 5よりも簡単だと考えられるため、式7 + 5の値を考慮すると、条件付きで5を3と2に分割し、7に3を追加し、10（参照番号）を取得し、10に10を追加します2、12を取得。



別の例：96から74を追加する必要があると想像してください。この操作で最も近い便利な数は100です。96= 100-4であるため、100を74に追加して174を取得し、174から4を減算し、 170を取得します。



77 + 66など、そのような最も近い番号がない式は、数字でカウントします。 つまり、77に最初に6ダースを追加して137を取得し、次に137に6ユニットを追加して143を取得します（ちなみに、最初の例のように6で7を追加し、最も近い参照番号で操作7 + 6を表します、つまり、数は14で、7 + 6を7 + 7として計算し（6 = 7-1であることがわかっている）、1を減算する方が簡単です）



この「研究」の過程で、私の頭の中を数えるシステムは、ある程度不合理に思えるという結論に達しました。 「RAM」をロードする追加の補助計算が必要です。これにより、式がもう少し複雑な場合は、中間計算を頭に入れられないため、書き留める必要があります。 たとえば、84に12を掛けてから、カンマを2文字左に移動して、84の12％を探します。 このために、私の考えでは、2番目の行を数えて最初の行を忘れて、エラーでこれらの12％をすぐに解決する列を想像します。



そして今、私の質問はあなたのためです。 小学校レベルでどう思いますか？ どのくらいの速さだと思いますか？ 口頭のアカウントはどうですか？ そして、彼らがうまくやっている人は、アルゴリズムを共有してください、あなたはどのようにあなたの心の中でこれらの計算を実行しますか？



さて、すでに哲学的な質問があります：クイックカウントのスキルを大幅に開発することは可能だと思いますか、それが存在する、または存在しないという考え方であり、トレーニングが結果をもたらす場合、それは取るに足らないものですか？ そして原則として、一般的に、プログラマーが頭の中ですぐに数えることができる/できないのはどの程度重要だと思いますか？
