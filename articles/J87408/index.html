ホットキーctrl + shift + [key]およびctrl + shiftによる言語の切り替え（解決済み）

問題：切り替えレイアウトをctrl + shiftに設定すると、ctrl + shift +フォームのホットキーが機能しなくなります。

最近（わずか1か月前）イリヤムラビョフが、この誤解を修正するパッチを作成したことが判明しました。 一番下の行は、パッチが押されたのではなく、ctrl + shiftボタンを離したことでトリガーされた後に言語を切り替えることです。



このトピックは、4年前のランチャーとfredestoptop.orgのバグに関するものです 。 パッチ自体。



bubuntovodovのための小さな指示：

ソースと今後の.debのフォルダーを作成します。

$ mkdir xorg-server

cd xorg-server



ソースを取得します

apt-get source xorg-server



アセンブリに必要なパッケージをインストールします

sudo apt-get build-dep xorg-server



パッチをダウンロードする

wget bugs.freedesktop.org/attachment.cgi?id=33142



パッチをインストールする

パッチ<attachment.cgi \？id \ = 33142

パッチを当てるファイル：xorg-server-1.6.4 / xkb / xkbActions.c



収集（別のフォルダーがある場合があります。ubuntu9.10で行いました）

cd xorg-server-1.6.4 /

debuild -us -uc

cd ...

dpkg -i * .deb
