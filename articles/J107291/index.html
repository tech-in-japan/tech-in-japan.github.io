シンプルなJavascriptベンチマーク

JavaScriptコードのパフォーマンスをどのように測定しますか？

それはこのように私に起こった：
  1. index.htmlファイルを作成します。
  2. どこかからHTMLテンプレートをコピーします。
  3. テストされたコードが存在するスクリプトタグを追加します。
  4. 配列の動作を測定する必要がある場合は、既成のランダム配列ジェネレーターを作成するか、作成してください。
最も悲しいことは、テスト用に以前に作成されたindex.htmlファイルが手元にないことです。テスト環境を準備する操作全体は、最初の段落から始まります。

私にとって、これは最後の日です。



Simple JS Benchmarkerは、シンプルなJavaScriptコードのパフォーマンス測定ツールです。



彼と仕事をする方法は？
  1. テストに配列が必要な場合は、データ型と要素数を示します。
  2. テストしたコードの反復回数を設定します。
  3. たとえば、DOMでの作業をテストしている場合、HTMLをコピーします。 「反復ごとに」オプションを指定することもできます。そうすると、HTMLが反復ごとに更新されます。 すべてのマークアップは、 コンテキストフレームの右側に「存続」します。
  4. 次に、テストしたJavascriptコードを貼り付けます。 実際には、2つのパラメータードキュメント （コンテキスト内のドキュメントオブジェクト）と配列 （生成された配列、または未定義）を持つ反復関数の本体になります 。 これは関数内でコンテキストのフレームを参照します。
出力は次のとおりです。
  • 最長のコード実行時間
  • 最短;
  • すべてのコールの平均。
  • 合計時間。
将来のために：
  • HTMLテキストの代わりに、インターネット上のページへのURLを示します。
  • JSフレームワークを接続します。
  • テストを保存して、他のブラウザーで再生します。
