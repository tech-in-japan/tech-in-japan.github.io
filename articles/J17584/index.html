Hakon Wium Lieが請願に関してcss3.infoに回答

Hakon Wium Lieは、ccs3.infoのいくつかの質問に対して、2つの要件を持つ指示された請願について回答しています。

1.「Windowsと連動してIEの配信を停止する、および/または代替ブラウザを提供する」

2.「MicrosoftをWeb開発者コミュニティが採用した基本的なオープンスタンダードに準拠させる」





Microsoftに「Web開発者コミュニティが受け入れている基本的でオープンな標準に従う」ようにしようとしています。 どのような基準を意味しましたか？ CSS 2.1？ CSS3？ HTML 4？ HTML 5？ もう？ あなたの意見でこれらの基準が法的に適格である理由は何ですか？



Microsoftが選択した標準を完全にサポートしていれば、Webははるかに優れた場所になります。 たとえば、Microsoftは1998年に次のように述べています。

「MicrosoftにはW3Cと協力してHTMLとCSSをサポートするための専門委員会があります。 HTML4の最初の商用実装があり、CSSの一般的な部分の最初の実装があり、CSS2の完成に多大な労力を費やしています。 W3Cの推奨事項（HTML、CSS、およびDOM）のサポートに引き続き取り組んでいます。

明らかに、MicrosoftはW3C HTML、CSS、およびDOMの推奨事項をサポートするという約束についてフォローアップしていない。 したがって、Operaまたは欧州連合は標準を設定する必要はありません-マイクロソフトはすでにこれを行っています”



IEとWindowsの配信について苦情を申し立てましたが、なぜAppleについても同じことをしなかったのですか？ MSはあなたにとって簡単なターゲットだと思います。 マイクロソフトは非難され、彼らは独占権を持っていますが、アップルは違うのですか？

いいえ、それは大きな違いです。 マイクロソフトは独占企業ですが、アップルはそうではありません。 裁判所は決定を見つけて承認します。 独占者は、ある地域での地位を使って別の地域の独占者になることはできません。 MicrosoftはWindows独占を使用してIE独占を達成しました



なぜ今嘆願書を送ったのですか？

まず、ブラウザは今まで以上に重要になっています。 これは単なるドキュメントビューアーではなく、Webアプリケーションが実行されるプラットフォームです。 そのため、ユーザーがブラウザ間で実際に選択できることが非常に重要です。 それが請願の目的です。品質の競争に基づく自然な選択です。

第二に、欧州司法裁判所は9月に判決を下しましたが、これは私たちの請願に関連しています。 これは、Windowsでメディアプレーヤーを課す場合です。 マイクロソフトは控訴なしで負けました。 ブラウザーがメディアプレーヤーに似ているのは、それらがより重要であることを考慮に入れない場合です。 重要な先例が発生しました。
