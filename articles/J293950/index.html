フリーランサー管理-ヒント

私は頻繁にプロジェクト管理に対処する必要があり、その実行者はフリーランサーです。 その過程で、そのような管理の改善について考えが浮かびます。



私の経験に基づいてまとめたアドバイスと、私が常に使用しているアドバイスを共有したいと思います。

この資料は、フリーランサー、顧客、およびプロジェクトマネージャーに役立ちます。







伝統的に、私はすべてのアドバイスが物議を醸し、主観的であり、あなたの重要な評価と処理を必要としていることに注意します。 コメントや提案を歓迎します。



パフォーマーの調整と検索の段階：



  1. 明確な技術タスクを作成し、フリーランサーのコメントに従って、注文から注文へと継続的に追加および改善します。 Skype /メールではなく、個別のドキュメントに技術的なタスクを作成します。
  2. フリーランサーについて否定的なレビューがあるかもしれません。 彼と協力するかどうかを事前に決めてください。 それでもあなたがプロジェクトについて議論することを志願したなら、あなたの頭からフリーランサーのすべての否定的な認識を取り除き、肯定的で実りある協力にのみ同調してください。 そうでなければ、交渉は失敗します-あなたのフレーズは皮肉、疑念、独裁で通過します。
  3. プロジェクトの将来の受け入れに、彼自身がそのような命令を実行した人を引き付けるようにしてください。 会話やTKでこれに言及してください。 フリーランサーは、問題をより深くより完全に解決するためのプロジェクトと方法を検討します。 結局のところ、彼は失敗した決定が簡単に識別できることを理解しています。
  4. 友人や知人による新しいフリーランサーを継続的に検索します。 多数のフリーランサーの存在のみが、アーティストを必要とする瞬間に働く手の可用性を少なくともある程度保証することができます。 事前にフリーランサーに手配しておくと便利です。




コミュニケーション：



  1. フリーランサーは部下ではなく、IP、つまりビジネスパートナーです。 この態度は生産的なコラボレーションを促進します。
  2. なぜなら フリーランサーはビジネスパートナーであり、それに応じて-同様に彼と通信します。 彼の意見に興味を持ち、アドバイス、フィードバック、コメントを求めてください。
  3. 多くの場合、フリーランサーを名前で呼び、人々とのコミュニケーションの基本原則を使用します（Dale Carnegieによる古典的な本を参照）。 フリーランサーと友達になりましょう。
  4. 難しい言葉を和らげるために最善を尽くしてください。 ソフトシノニムを探します。 たとえば、「プロジェクトコメント」というフレーズを「プロジェクトコメント」に置き換えます。
  5. ディスカッション中に長時間気を散らす必要がある場合は、必ずそれについて書いてください。 「抜け出す」と書くのは簡単です。 「突然姿​​を消す」習慣がないことをフリーランサーに示します。
  6. 「一時的に姿を消した」、「どこに消えたのかは不明」、「たとえ職場で撃たれただけなのか」という隠phorでさえあるとみなされるものをすべて排除します。




作業プロセス：



  1. フリーランサーを最新の状態に保ちます。 たとえ彼が尋ねなくても、しばしば退会するプロセスで、いつ仕事が受け入れられるか、どれだけ待つべきかなど。 自分の期限を明確に設定し、約束を守ります。 フリーランサーも同様に答えます。
  2. 問題や遅延を説明するときは正直に言ってください。 しかし、あなたの会社の台所を掘り下げたり、長い話を書いたりしないでください。 このキッチンはフリーランサーにとって面白くない。
  3. 支払いを遅らせることはありません。 承認が完了したらすぐに支払います。 ボードの遅延の理由は、「プロセスの受け入れ」、「修正あり」などのようなものです。 「彼らはすぐに私に送金します」、「お金を受け取ろうとしています」-フリーランサーはこれらのニュアンスに興味がありません。 極端な場合は、「1日1時間以内に翻訳します」などと言います。
  4. 成功した受け入れの一部については、フリーランサーがそれについてあなたに尋ねない場合でも、金額の一部を支払います。 彼を楽しませてください。
  5. 仕事の終わりには常にフリーランサーに好きなものと嫌いなものを尋ねます。 フィードバックを受け取り、感謝し、管理スキルを向上させます。
  6. フリーランス交換のフリーランサー/ linkedInでフリーランサーにレビューを書いてもらい、プロジェクトへの彼の参加について言及してください。
  7. 作業の最後に、常に技術仕様を調整してください。 チェックする必要があるキーポイントのチェックリストを作成します。 したがって、管理を継続的に改善し、開発コストを削減できます。




すぐに、サイトを受け入れるために使用するチェックリストを公開する予定です。

また、典型的な技術的なタスク。 これらの資料に基づいてドキュメントを生成できるようになると、時間を大幅に節約できると思います。



このヒント集に対するコメント、批判、評価を楽しみにしています。
