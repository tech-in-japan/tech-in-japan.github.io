SQL Server 2008の遅延

Microsoftは、SQL Server 2008のリリース日が第3四半期に延期されることを発表しました。 2008年、DBMSの開発が予想外に引き継がれたため。 最初は、2月末にWindows Server 2008、Visual Studio 2008、およびSQL Server 2008の共同リリースを実装する予定でしたが、今ではプログラマーには時間がありません。



最終リリースの代わりに、フル機能のプレビューバージョンが2008年2月27日にリリースされます。 その後、II四半期に。 2008年のリリース候補は、III四半期にのみ表示されます。 2008-公式バージョン。



SQL Server 2008は非常に期待されている製品です。 技術プレビューのダウンロード数はすでに10万を超えています。 新機能のリストには 、スケーラビリティの改善、特定のユーザー/リソースガバナープロセスのリソース予約システム、空間データタイプ（空間）のサポート、圧縮モジュール、データベースおよび個々のテーブルのアクセス許可を配布するためのライブラリ、画像、音声、ビデオ、その他の種類のマルチメディアのデジタル形式を含む、構造化および半構造化データのサポート。



遅延は、SQL Server 2005の2〜3年後にSQL Server 2008をリリースするというMicrosoftの計画とよく一致していることに注意してください。そのバージョンには同様の問題があったためです。リリース日が数回延期され、非常に重要でした。



ただし、すべての遅延にもかかわらず、SQL Server 2005は成功した製品になりました。 市場アナリストによると、Microsoftデータベース部門の収益は、競合企業であるOracleとIBMの類似部門の収益よりも急速に成長しています。 一般に、この部門は、Microsoftで最も収益性が高く急成長している部門の1つになっています。
