あなたのWindowsはとても古いですか？

Habréの最新記事の白熱した議論が示すように、多くの上級ユーザーやあまり知られていないユーザーは「時代遅れの」Windows 7を好み、Windows 10に切り替えることを急いでいません。ありふれた習慣と、マイクロソフトマーケティング担当者のリードに従うことを望まない。



この記事では、開発者が仕事の責任に対する高品質のソリューションのためにWindows 7およびそれ以前のバージョンでさえ必要とする場合を検討します。 Windows 7のシェアが企業部門で依然として非常に大きいことは周知の事実であり、Microsoftが2023年までサポートを延長したため、近い将来に何かが劇的に変化する可能性はほとんどありません。 また、Server 2003および2008には、古いものの非常に必要なソフトウェアを実行する非常に古いシステムがあり、誰も「動作しないでください」という原則に基づいて移植する必要がないという事実は言うまでもありません。 ただし、「古い」オペレーティングシステムの更新は、予想よりはるかに高速で簡単です。



新しい開発をテストしたり、既存の開発機能を補完したりする必要があるとき、テストオペレーティングシステムをインストールして操作性をテストするという疑問が生じます。これは実際の作業システムで実行するのに最適なアイデアではありません。 そして、実際のイメージまたはインストールディスクがある場合でも、2011年モデルのWindows 7 SP 1は、過去10年間に何百ものセキュリティと安定性の更新を受けたユーザーのコンピューター上のWindows 7とはほど遠いことがわかりました。



一部のWindows Server 2008またはWindows 7 SP1で通常のツールを使用して更新プログラムをインストールしようとすると、更新システムが機能していないことがわかり、更新プログラムが無限に検索されます。 古くなったOSサービススタックを含むなど、これが起こっていることを多くの人が知っています。



オプションとして、仮想マシンをインターネットに接続せずに、Windows 7を現在の状態に完全にアップグレードするための最大限にシンプルな方法をできるだけ早く考えてみましょう。すべてのアクションは、Windows Server 2008 R2 SP1、Datacenter、およびEnterpriseで同様です。 適切なビットサイズのWindows 7 SP1をクリーンインストールする必要があります。これは、MSDNのイメージまたはお気に入りの仮想化システムまたは実際のマシンのインストールDVDから作成されます。 この段階でのキー入力とアクティベーションはスキップして、後で実行できます。

zashniksに公式画像がなく、ファイル共有サービスを使用してそれらをダウンロードする場合は、Microsoft Webサイトでこの画像に使用できるものとファイルチェックサムを確認することを忘れないでください。たとえば、Total Commanderの試用版を使用してこれを行うことができます。


OS自体を正常にインストールした後、microsoft.comから、対応するビット深度のOS用の更新プログラムを3つだけダウンロードする必要があります。 まず、 KB3177467 Windows 7 SP1およびWindows Server 2008 R2 SP1のサービススタックの更新-2018年10月のサービススタックの更新が必要です。 SSUをインストールした後、更新プログラムのロールアップKB3125574をインストールして再起動する必要があります。 インストールが失敗した場合は、KB3020369をプレインストールする必要があります。



最後の手順は、毎月のロールアップパッケージをインストールすることです。KB4462923の現在のバージョンはこちらから入手できます 。 将来、累積更新プログラムの最新バージョンは、月ごとのロールアップ更新ウィンドウ7の11月のクエリを使用して検索エンジンに配置されます。11月は更新パッケージの月です。 ヨーロッパでは、毎月第2水曜日から新しいパッケージが利用可能になりますが、以前のパッケージのインストールは原則として不要です。



したがって、これら3つの更新プログラムをインストールした後、Windows 7は、現時点ではすべての安定性とセキュリティの修正プログラムで完全に更新されます。 予備ダウンロードのサービスパックを除くすべてのアクションは、インターネットに接続しなくても実行できます。 さらに、サービススタックを更新すると、Windows Updateを介して問題なく迅速に新しい更新を受信できます。 OfficeやDot NET Frameworkなどの他のMicrosoftソフトウェアの更新は個別に行われることに注意してください。

妄想のごく一部として。 検索バーにWindows Software Quality Improvement Programが見つかった場合、MicrosoftはWindows 7のさらなる開発を計画していないため、この項目を無効にすることをお勧めします。おそらく、diagtrackサービスも削除する必要があります。
