1.1 4種類の引用符


まず第一に、あなたはすでに知っているかもしれません。 一部のプログラミング言語では、一重引用符と二重引用符は異なるものを表します。 Pythonでは、両方のオプションを使用できます（ただし、文字列は同じタイプの引用符で開始および終了する必要があります）。 Pythonには、さらに2種類の引用符があります： '' '（トリプルシングル）と "" "（トリプルダブル）。したがって、エスケープを心配する前に、いくつかのレベルの引用符を使用できます。たとえば、次のコードは正しいです。 

print """     ,   : ''' : " : '   '"'''"""


1.2さまざまなオブジェクトの真実性


一部のプログラミング言語とは異なり、Pythonでは、オブジェクトは空の場合のみ偽と見なされます。 これは、文字列、タプル、または辞書の長さをチェックする必要がないことを意味します-論理式としてチェックするだけです。



0も偽であり、残りの数が真であると予測するのは簡単です。



たとえば、次の式は同等です。 この場合、my_objectは文字列ですが、異なるタイプが存在する可能性があります（ifブロックの条件に対応する変更がある）。 

 my_object = 'Test' # True example # my_object = '' # False example if len(my_object) > 0: print 'my_object  ' if len(my_object): # 0   False print 'my_object  ' if my_object != '': print 'my_object  ' if my_object: #     False print 'my_object  '


したがって、オブジェクトが空であるかどうかだけに関心がある場合は、オブジェクトの長さをチェックする必要はありません。



1.3サブストリングの確認


これは小さく、かなり明白なヒントですが、私はPythonを1年間学習してから初めてそれについて学びました。 「item in list」または「item not in list」を使用して、目的のアイテムがタプル、リスト、辞書に含まれているかどうかを確認できることを知っておく必要があります。 これが文字列で機能するとは想像できませんでした。 私はいつもこのようなものを書いた： 

 string = 'Hi there' # True example # string = 'Good bye' # False example if string.find('Hi') != -1: print 'Success!'


このコードはかなり厄介です。 「文字列のサブストリングの場合」はまったく同じように機能します。 

 string = 'Hi there' # True example # string = 'Good bye' # False example if 'Hi' in string: print 'Success!'


よりシンプルで明確。 99％の人には明白かもしれませんが、私が知る前にそれについて知りたいと思います。



1.4美しいリスト出力


printを使用したリストの通常の形式は、あまり便利ではありません。 もちろん、リストが何であるかは明確になりますが、ほとんどの場合、ユーザーは各要素を引用符で囲みたくありません。 文字列の結合方法を使用した簡単な解決策があります。 

 recent_presidents = [' ', ' ', ' '] print '   %s.' % ', '.join(recent_presidents) #  "    ,  ,  ."


joinメソッドは、リストを文字列に変換し、各要素を文字列として扱います。 セパレータは、結合が呼び出された行です。 最後の要素の後に区切り文字を挿入しないのは十分賢明です。



追加ボーナス：結合は線形時間で実行されます。 forループでリストアイテムを折りたたんで文字列を作成することはありません。見苦しいだけでなく、 2次的な時間がかかります。



1.5整数除算と浮動小数点除算


整数を整数で除算すると、デフォルトで結果は整数に切り捨てられます。 たとえば、5/2は2を返します。



これを修正するには2つの方法があります。 最初の最も簡単な方法は、数値の1つをfloat型に変換することです。 定数の場合、数字の1つに「.0」を追加するだけです。5.0/ 2は2.5を返します。 float（5）/ 2コンストラクトを使用することもできます。



2番目の方法は、よりクリーンなコードを提供しますが、プログラムがこの重要な変更から壊れないようにする必要があります。 「from __future__ import division」を呼び出した後、Pythonは常に除算の結果としてfloatを返します。 整数除算が必要な場合は、演算子//：5 // 2を使用すると常に2が返されます。 

 5/2 #  2 5.0/2 #  2.5 float(5)/2 #  2.5 5//2 #  2 from __future__ import division 5/2 #  2.5 5.0/2 #  2.5 float(5)/2 #  2.5 5//2 #  2


Pythonの次のバージョンの1つでは、この動作がデフォルトになります。 コードの互換性を維持する場合は、このインポートを使用していない場合でも、整数除算に//演算子を使用します。



1.6ラムダ関数


関数を引数として渡すか、短いが複雑な操作を数回行う必要がある場合があります。 通常の方法で関数を定義するか、ラムダ関数（単一の式の結果を返す小さな関数）を使用できます。 次の2つの定義はまったく同じです。 

 def add(a,b): return a+b add2 = lambda a,b: a+b


ラムダ関数の利点は、それが式であり、別の式の中で使用できることです。 以下は、リスト内の各アイテムに対して関数を呼び出し、結果のリストを返すmap関数を使用した例です。 （次の段落では、マップが実際に役に立たないことを示します。しかし、それは1行で良い例を示す機会を与えてくれます。） 

 squares = map(lambda a: a*a, [1,2,3,4,5]) #  squares = [1,4,9,16,25]


ラムダ関数がなければ、関数を個別に定義する必要があります。 1行のコードと1つの変数名を保存しました。

ラムダ関数の構文： ラムダ変数：式

変数 -コンマで区切られた引数のリスト。 キーワードは使用できません。 引数を括弧で囲む必要はありません。

expressionはインラインPython式です。 スコープにはローカル変数と引数が含まれます。 関数は、この式の結果を返します。

2.1リストジェネレーター


Pythonを十分に長く使用していれば、少なくとも「リスト内包表記」の概念について聞いたことがあるはずです。 これは、forループ、ifブロック、および割り当てを同じ行に収める方法です。

つまり、1つの式でリストを表示（マップ）およびフィルターできます。



2.1.1リスト表示


最も簡単な例から始めましょう。 リストのすべての要素を二乗する必要があるとします。 焼きたてのPythonプログラマーは、次のようなコードを書くことができます。 

 numbers = [1,2,3,4,5] squares = [] for number in numbers: squares.append(number*number) #  squares = [1,4,9,16,25]


あるリストを別のリストに「マッピング」しました。 これは、マップ関数を使用して行うこともできます。 

 numbers = [1,2,3,4,5] squares = map(lambda x: x*x, numbers) #  squares = [1,4,9,16,25]


このコードは間違いなく短い（3行ではなく1行）が、それでもstillい。 一見すると、map関数が何をするかを言うのは困難です（関数とリストを引数として取り、リストの各要素に関数を適用します）。 さらに、関数を決定することを余儀なくされ、それはかなり厄介に見えます。 より美しいパスがあった場合のみ...例えば、リストジェネレータ： 

 numbers = [1,2,3,4,5] squares = [number*number for number in numbers] #   squares = [1,4,9,16,25]


このコードはまったく同じことを行いますが、最初の例よりも短く、2番目の例よりも明確です。 人はコードが何をしているのかを簡単に判断できます。そのため、Pythonを知る必要さえありません。



2.1.2リストのフィルタリング


しかし、リストのフィルタリングに関心がある場合はどうでしょうか？ たとえば、4以上の要素を削除します（はい、例はあまり現実的ではありません。とにかく...）



初心者は次のように書きます。 

 numbers = [1,2,3,4,5] numbers_under_4 = [] for number in numbers: if number < 4: numbers_under_4.append(number) # numbers_under_4 = [1,4,9]


とても簡単ですよね？ しかし、コードは4行で、2つのレベルのインデントを含み、同時に些細なこともしています。 フィルター関数を使用してコードサイズを縮小できます。 

 numbers = [1,2,3,4,5] numbers_under_4 = filter(lambda x: x < 4, numbers) # numbers_under_4 = [1,2,3]


上記で説明したマップ関数のように、フィルターはコードを短縮しますが、見た目はかなりいです。 一体何が起こっているの？ mapと同様に、filterは関数とリストを取得します。 要素からの関数がtrueを返す場合、その要素は結果のリストに含まれます。 もちろん、リストジェネレーターでこれを行うことができます。 

 numbers = [1,2,3,4,5] numbers_under_4 = [number for number in numbers if number < 4] # numbers_under_4 = [1,2,3]


繰り返しますが、より短く、より明確で、より理解しやすいコードを取得しました。



2.1.3マップとフィルターの同時使用


これで、リストジェネレーターの全機能を使用できます。 マップとフィルターがあまりにも多くの時間を費やしていることをまだ納得していない場合は、私に同意していただければ幸いです。



リストを同時に表示およびフィルタリングしたいとします。 つまり、リスト項目の4より小さい正方形を表示したいのです。もう一度、初心者は次のように書きます。 

 numbers = [1,2,3,4,5] squares = [] for number in numbers: if number < 4: squares.append(number*number) # squares = [1,4,9]


悲しいかな、コードは右に伸び始めました。 たぶんそれはそれを簡素化しますか？ マップとフィルターを使用してみましょうが、私はそれについて悪い気持ちを持っています... 

 numbers = [1,2,3,4,5] squares = map(lambda x: x*x, filter(lambda x: x < 4, numbers)) # squares is = [1,4,9]


以前はマップとフィルターの読み取りが困難でしたが、現在は不可能です。 明らかに、これは良い考えではありません。 また、リストジェネレーターは状況を保存します。 

 numbers = [1,2,3,4,5] squares = [number*number for number in numbers if number < 4] # square = [1,4,9]


リストジェネレーターを使用した以前の例よりも少し長くなりましたが、私の意見では、かなり読みやすいです。 forループを使用したり、マップとフィルターを使用したりするよりも間違いなく優れています。



ご覧のように、リストジェネレーターは最初にフィルターをかけて表示します。 あなたが間違いなく反対を必要とするならば、それはより難しいでしょう。 どちらがより単純であるかに応じて、ネストされた世代、マップとフィルター、または通常のforループを使用する必要があります。 しかし、これはすでに記事の範囲を超えています。

リストジェネレーター構文：[条件の場合のリスト内の変数の要素]

リスト -反復可能な要素

変数（s） -forループに似た、現在のリスト項目に等しい1つまたは複数の変数

condition-インライン式：trueの場合、要素が結果に追加されます

element-結果が結果リストの要素として使用されるインライン式



2.1.4ジェネレーター式


リストジェネレーターには裏返しがあります。リスト全体がメモリ内にある必要があります。 これは、前の例のように小さなリストでは問題にならず、さらに数桁以上も問題になりません。 しかし、最終的には効果がなくなります。



ジェネレータ式はPython 2.4で登場しました。 すべてのPythonチップの中で、おそらくあまり注目されていません。 リストジェネレーターとの違いは、リスト全体をメモリにロードするのではなく 、「ジェネレーターオブジェクト」を作成し、各時点でリストの1つの要素のみがロードされることです。



もちろん、リストを何かに使用したい場合、これはあまり役に立ちません。 ただし、反復可能なオブジェクトが必要な場所（forループなど）に渡す場合は、ジェネレーター関数を使用する必要があります。



ジェネレータ式の構文はリストジェネレータと同じですが、角括弧の代わりに丸括弧が使用されます。 

 numbers = (1,2,3,4,5) #     ,      ;) squares_under_10 = (number*number for number in numbers if number*number < 10) # squares_under_10 - generator object,      ,   .next() for square in squares_under_10: print square, # '1 4 9'


これは、リストジェネレーターを使用するよりも効率的です。



そのため、大きなリストにはジェネレータ式を使用するのが理にかなっています。 リスト全体が他の目的に必要な場合は、任意のオプションを使用できます。 ただし、これに反論がない場合は、ジェネレータ式を使用するのが良い習慣です。 確かに、リストが小さい場合、加速を期待しないでください。



最後に、式ジェネレータを1つの括弧で囲むだけで十分であることに注意してください。 たとえば、1つの引数で関数を呼び出す場合、some_function（リスト内のアイテムのアイテム）と書くことができます。



2.1.5結論


これを言うつもりはありませんが、式ジェネレーターとリストジェネレーターでできることについて触れました。 ここでは、forおよびifのすべての機能を使用でき、繰り返し可能なオブジェクトであれば何でも操作できます。
