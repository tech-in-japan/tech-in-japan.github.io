iOSゲームをBlackBerry PlayBookに移植する

iOSゲームをBlackBerry PlayBookタブレットに移植することを考えたことがありますか？ Pop Cornyゲームを移植したので、このプロセスの詳細を共有したいと思います。



基本



おそらく、先ほどの私のように、PlayBookはBlackBerry電話に組み込まれているテクノロジーと一定の関係があると思います。 私は同じように考えていたので、長い間、ゲームを転送するときにこのタブレットを考慮していませんでした。 実際、PlayBookは、RIMがQNXオペレーティングシステムに基づいて作成した新しいプラットフォームに基づいています。 また、新しいBlackBerry 10スマートフォンでも使用されます。RIMは、Googleのように、開発者にJavaの使用を強制し、後日ネイティブ開発ツールのサポートを追加するのではないかという懸念にもかかわらず、現実ははるかに優れていることが判明しました。 カナダの会社は、タブレット用のソフトウェアを作成するための包括的なソリューションであるネイティブ開発環境 （NDK-Native Development Kit）を提供しています。 NDKがメイン開発環境の拡張機能であるAndroidとは異なり、ほとんどの場合、最小限の機能をサポートし、Javaへのアクセスが必要です。PlayBokの場合、Javaを見ずに「ネイティブ」アプリケーションを作成できます。 NDKは、画面や入力ツールの操作からゲーム内での購入まで、必要なものすべてに対応する完全なAPIを提供します。







RIMが提供する開発環境はQNX Momenticsと呼ばれ、Eclipseに基づいていますが、必要に応じてコマンドラインユーティリティでこれを行うことができます。 EclipseはC / C ++（デバッガー、プロファイラーなどを備えている）で開発するのが便利であるため、Eclipseをゆっくりと考えるにもかかわらず、Momenticsを選択しました。 さらに、コマンドラインを見逃し始める瞬間まで、どれだけの量で十分かを確認したかったのです。 驚くべきことに、Eclipseに出会って以来初めて、私は選ばれた道を後悔しなかった。



次に、ゲームアプリケーションをiOSからPlayBookに転送する主な側面を検討します。



データ入力



PlayBook OSでのデータ入力処理は非常に簡単で、iOSの処理方法に応じて、特別な問題は発生しません。 システムの他のイベントと同様に、タッチスクリーンイベントは、bps_get_event（）で受け取るbps_event_t構造の形式で表示されます。 このイベントは「画面領域」に属し、関数screen_get_event_property *（）のセットを使用してそのデータを要求します。 



int touch_state; int id; int position[2]; screen_get_event_property_iv(screen_event, SCREEN_PROPERTY_TYPE, &touch_state); screen_get_event_property_iv(screen_event, SCREEN_PROPERTY_TOUCH_ID, &id); screen_get_event_property_iv(screen_event, SCREEN_PROPERTY_SOURCE_POSITION, position);




上記のコード例は、画面のタッチに関する基本的な情報を取得するために必要なすべてです。 touch_stateには、SCREEN_EVENT_MTOUCH_TOUCH、SCREEN_EVENT_MTOUCH_MOVE、およびSCREEN_EVENT_MTOUCH_RELEASEのいくつかのタイプがあります。 これらは、iOSのイベント「タッチの開始/位置の変更/タッチの終了」と比較できますが、「キャンセル」イベントがないことに注意する価値があります。



さらに、イベントをポーリングしているので、必要なストリームでそれを取得し、iOSからAndroidにゲームを移植するときに個人的に問題が発生することはありません。



PlayBookには基本的なハードウェアキーがないため、それらを処理する必要はありません。



音



iOSと同様、PlayBook OSはOpenALをサポートしています。 このオープンライブラリを使用してサウンドを操作する場合、プログラムを転送するプロセスで何も変更する必要はありません。 タブレットOSは、必要に応じてALUT（OpenAL Utility Kit）ユーティリティセットもサポートしています。







グラフィックス



ご想像のとおり、PlayBookはOpenGL ES（バージョン1.1および2.0）を問題なくサポートしています。 NDKには、画面設定に関するすべてのニュアンスを処理するOpenGLを使用する簡単なアプリケーションの例が含まれています。 シンプルだが効果的なAPI設計のおかげで、この例は自給自足であり、詳細な検討は必要ありません。







タブレットの画面の解像度は1024x600ピクセルで、アスペクト比はほぼ16：9です。その結果、現在のiPhoneやiPadの画面よりも狭くなっています。 ゲームによっては、この解像度のユーザーインターフェイスを作り直すか、エッジの周りに黒いバーを追加する必要があります。 Pop Cornyの場合、1.33から1.50のアスペクト比を維持し、それに自動的に調整するインターフェイスを備えています。 そのため、PlayBookのバージョンのゲームでは、黒いバーを追加し、アスペクト比を1.7から1.50に減らしました。



画面のピクセル数は、iPhone 4およびiPad 2のピクセル数とほぼ同程度です。したがって、これらのデバイス用のグラフィック素材があれば、問題なくPlayBookに使用できます。 ゲームがiPad 3の画面解像度に最適化されている場合、ほとんどの場合、グラフィック要素のサイズを小さくして、ユーザーのタブレットの貴重なメモリを無駄にしないようにする必要があります。



内容



コンテンツのダウンロード-それは決して簡単なことではありません。Zipファイルを突っ込んだり、圧縮を無効にしたり、特定の拡張機能やアセット管理マネージャー（hello、Android！） アプリケーションは、基本的にzipアーカイブである.barファイルとして配布されます。 インストール中に、.barファイルの内容が解凍され、アプリケーションはPOSIX互換の方法でファイルシステムにアクセスできます。



.barファイルに含まれるファイル（テクスチャ、サウンドなど）をダウンロードする場合は、次の方法でそれらを見つけることができます。



./app/native/



ファイルを作成してアプリケーションデータを保存する場合は、次の場所に配置します。



./data/



簡単にはなりません。



記録と実績の表



OpenFeint（Geeとも呼ばれます）はPlayBookでは使用できません。これはもちろん、Appleが所有するGame Centerにも適用されます。 RIMは、 Scoreloopと呼ばれる独自のソリューションを提供します。これは、プレーヤープロファイル、ハイスコアテーブル、実績、および戦いをサポートします。



結論として



私にとって、PlayBookはこれまでに移植した中で最も簡単なプラットフォームです。 開発者に愛されている、すべてのオープンフレームワークをサポートするPOSIX互換プラットフォームを明らかにしたRIMエンジニアに敬意を表します。 他のほとんどのプラットフォームでは、実行可能ファイルに必要なすべてのライブラリをプロジェクトに追加する必要があります。 PlayBookには、libpng、libjpeg、freetypeなどがすでに含まれており、すぐに使用できます。 コンパイルに問題はありません。システムに既に存在するライブラリとの静的リンクにより、実行可能ファイルのサイズが増加します。



ただし、サードパーティのPlayBookサポートは、他のプラットフォームほど適切ではありません。 たとえば、iOSで統計を収集するには、優れたサービスflurry.comを使用しますが、RIMタブレットには「フレンドリー」ではありません。 このサービスは、Javaを介したBlackBerryのサポートを提供しますが、この言語をまったく扱いたくありません。



iOSやAndroidストアとは異なり、AppWorldでアプリを公開するための料金はかかりません。 移植の最後に、登録し、デジタル署名のキーを取得して、開発を公開します。 ただし、AppWorldはApple Storeの方法で認定プログラムを適用します。



結論として、私は結果が努力の価値があったことに注意したいと思います。 移植プロセスには2〜3日かかりました。 私はすべての独立した開発者にこのプラットフォームに注意を払うことを勧めます。 iOSとAndroidのストアにはアプリケーションがいっぱいあることも考慮する価値があります。そのため、PlayBookはゲームに気付く良い機会になるかもしれません。 出版の瞬間からこの記事の執筆まで、Pop Cornyは4日間「New」セクションのリストにありました。 あなたが独立した開発者であれば、あなたの製品ができるだけ長く顧客に見えることの重要性をよく知っています。



PS驚きのPlayBook OS



RIMタブレットにどれほど感銘を受けたかを伝えることはできません。 私は、彼女とビジネスをしたことがない開発者が2〜3日で自分のゲームを移植できるプラットフォームは素晴らしいと信じています。 しかし、さらに驚いたのは、PlayBookのPop Cornyの速度で、iPad 2よりも著しく高速でした。







言葉を確認するために、2つのタブレットで並行してゲームを開始するビデオを録画しました。 Pop Cornyの両方のバージョンは毎秒60フレームを生成するため、ゲームの違いに気付くことは困難です。 タブレットがほぼ同等のハードウェア機能（1 GHzの周波数で動作するデュアルコアCortex-A9プロセッサ）を備え、ゲームが最初にiOSでリリースされ、その後PlayBookに移植されたことを考慮する価値があります。 しかし、同時に、RIMデバイスはゲームの起動（テクスチャのロード、それらのアンパック、GPUの転送、サウンドのロードなど）を30％高速に実行できます！ これは素晴らしい成果だと思います。
