ソースコード→クラッシュ！

昨日、 Firefox 3.0.9で「ソースコードを表示」をクリックすると、FFはひどいきしみ音でクラッシュしました。 とても印象的でした。 私が最初に考えたのは、Mozillaがどこかでバグを作成したことです。



しかし、今朝、FFが開始されたとき、アドオン更新ウィンドウがポップアップ表示され、 HTML Validatorの更新を提案しました。



奇跡！ HTMLを表示するときにFFがクラッシュしなくなりました！



このアドオンが原因であることが判明しました。 公式Webサイトからのアドオン作成者による声明の翻訳は次のとおりです（ナレーションのスタイルは可能な限り保持されます）。



- 以前のバージョンでは、DOM APIのFIREFOX 3.0.9でバグが発生しました。

実際のソリューションはFirefox 3.0.10になりますが、さらに1か月間は利用できません。 実際、Firefox 3.0.9でソースコードを表示すると、クラッシュします。 Firefox 1.0以降に機能していたコード行の一部がFirefoxのクラッシュを引き起こします...このため、Firefox 3.0.9のバグを解決するために非常に高速な0.856をリリースする必要があるため、HTMLエラーによる行の強調表示がオフになりました！ 残念ながら、私はより良い解決策を持っていませんでした。

-Firefox 3.1 beta 3で動作します。

-Seamonkey 2 alpha 3で動作します
