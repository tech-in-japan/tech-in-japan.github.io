Windows Azure Update：ポイントツーサイト接続、および仮想マシンとネットワーク、クラウドサービス、Ruby SDKのリリースに対するその他の改善

開発者とITプロフェッショナルの同僚である私は、 Windows Azureクラウドプラットフォームの次の更新に関する詳細を急いで共有します。 4月26日、Scott Guthrieはブログで、仮想マシンとネットワーク、およびその他のクラウドサービスの操作に関連する新製品の次のバッチを発表しました。 さらに、この発表により、RubyアプリケーションをWindows Azureクラウドサービスと簡単に統合できる、待望の更新されたRuby SDKツールキットが提供されました。 以下に、これらの詳細な説明と、Scott Guthrieのイノベーションでは説明されていないその他の説明を示します。



ポイントツーサイト仮想ネットワーク接続



このプラットフォームアップデートの主な革新は、仮想ネットワークの新しい機能と安全に呼ばれ、ポイントツーサイトの対話を提供します。つまり、クライアント側にVPNデバイスを必要としません。



これはどういう意味ですか？ つまり、企業ネットワーク外の通常のPCからWindows Azureの仮想ネットワークに接続できるようになりました。 このような機会は、クラウドプラットフォームと仮想ネットワークを使用するための膨大な数の新しいシナリオを切り開きます。







Point-to-Site Virtual Private Networkingテクノロジーは、インフラストラクチャ設定を理解したり、企業ネットワークからクラウドへのVPNを構成するために会社のITスタッフを使用したりする必要がないため、クラウド開発者にとって特に役立ちます。 代わりに、開発者は組み込みのWindows VPNクライアントを使用してクラウドにトンネリングできます。 これらの機能はSecure Sockets Tunneling Protocol（ SSTP ）によってサポートされており、ファイアウォールやプロキシを簡単に克服しながら、完全なデータ転送セキュリティを確保します。



Windows Azure仮想ネットワークの新機能が特に役立つという別の実際のシナリオとケース：

  • 独自のVPN対応デバイスを持たない小規模企業（または大企業内の部門）および/またはそのようなデバイスを構成する機能は、 ポイントツーサイトVPN機能を使用してWindows Azureクラウド内の独自の容量に安全に接続できるようになりました。
  • あなたや会社の従業員は、ファイアウォールやプロキシで保護された企業環境で働いている場合でも、IT担当者を引き付ける必要なく、クラウド容量に簡単かつ迅速に安全に接続できます。 これは、特に顧客のネットワーク上やカフェなどの公共の場所で仕事をしている場合に非常に便利です。


Point-to-Site接続のセットアップは非常に簡単です。仮想ネットワークをセットアップする段階で、接続を構成することを望みます。







必要なデータをすべて入力し、ゲートウェイを起動して、ローカルインストール用のVPNクライアントパッケージをダウンロードします。 その後、仕事用コンピューターからWindows Azureの仮想ネットワークに接続できます。







Point-to-Site VPNを構成する方法の詳細については、この記事「管理ポータルでPoint-to-Site VPNを構成する 」を参照してください。



サイト間接続のソフトウェアVPNサポート



Windows Azure仮想ネットワークへの新しいタイプの接続の発表とともに、VPNデバイスを介して2つのネットワークを接続する以前のサイト間タイプの機能も改善されました。



以前、サイト間接続の実装のために、VPNサポート（およびその他の要件）を備えたルーターがCiscoやジュニパーなどの企業から必要であった場合、サイト間接続はソフトウェアVPNを介して簡単に構成できます。



この新機能は、ルーティングとリモートアクセスサービス（RRAS）プロトコルの機能を使用し、 PowerShellスクリプトを使用してWindows Server 2012で構成できます。



仮想マシンとネットワークのその他の革新



現在の更新では、Windows Azureで仮想マシンとネットワークを操作するためのいくつかの新しい機能が利用可能になりました。



ダイナミックDNSサポート



作業が簡素化され、仮想ネットワークでDNSサーバーを設定する際の制限が削除されました。 これで、仮想ネットワークとその内部のマシンを再作成することなく、仮想ネットワークのDNSサーバー設定を更新できます。 ネットワーク上の各マシンは、DNSの更新後に更新された設定を受け取ります。 この改善により、仮想ネットワークの更新がはるかに簡単になります。



リモートPowerShellおよびLinux SSH機能



これで、仮想マシンを作成するときに、その中でリモートPowerShell機能を有効にする必要があることを指定できます。これにより、RDPセッションを使用せずにサーバーを管理できます。







仮想マシンの作成段階でLinuxベースのサーバーを管理しやすくするために、管理者がパスワードを必要としないようにSSHキーをダウンロードする機能が追加されました。







SSHセッションのパスワード機能は保持されており、必要に応じて使用できます。



仮想マシンパターンギャラリーのSQL Server Web



Windows Azureインフラストラクチャサービスのグローバルな可用性へのアクセスにより、開発者とITスペシャリストは、仮想マシンを使用する時間を支払うだけで、サーバーライセンスを取得せずに仮想マシンでSQL Serverを使用できます。



発表の時点で、SQL Server StandardおよびEnterpriseのバージョンは、仮想マシンイメージのギャラリーで利用できました。 現在の更新では、最も安価なバージョンのSQL Server Webもギャラリーに追加されています。



クラウドサービス：既にホストされているロールのRDPを有効にする



Windows Azure PaaSプラットフォームのクラウドサービスを使用すると、ホストされている各ロールへのRDP接続を使用できます。 ただし、以前は、コード配置の段階でさえ、RDP機能を有効にする必要性を明確に示す必要がありました。 事前に有効化していない場合、すでに実行中のロールに対して有効化することはできませんでした。



現在の更新では、待望の機能が追加され、RDPを介して既に実行中のCloud Servicesロールのインスタンスにアクセスできるようになりました。







すべてのロールに対してこの機能を1回有効にすることで、RDPを介して各インスタンスに接続できます。



Ruby SDKの発表



Windows Azureチームは、.NET、Java、PHP、Node.js、Pythonなどのさまざまなプログラミング言語およびプラットフォーム用のオープンソースツールキットを長年にわたって提供してきました。



現在の更新では、ツールファミリに新しいRuby SDKが追加され、Rubyプログラミング言語のプロジェクトがサポートされています。 このツールセットは、Ruby ruby​​gemプログラマーに提供され、クラウドストレージ（ブロブ、テーブル、キュー）とサービスメッセージングバス（サービスバスキュー、トピック/サブスクリプション）の操作を簡素化します。 Windows Azure用のRuby SDKはAppFogで作成されました 。



SDKのソースコードはGitHubにあります 。 Ruby gem Azureはhttp://rubygems.org/gems/azure/にあります。 rubydoc.infoの Ruby用Windows Azure SDK ドキュメント 。



開発者向けに、Ruby 開発センターは公式のWindows Azure Webサイトで公開されており、クラウドでRuby開発を開始するために必要な資料とガイドを見つけることができます。 完全なリストは、ポータルwazurr.ruにあります。



外国のGitまたはMercurialリポジトリからサイトをホストする



Windows Azureのもう1つの小規模だが重要な革新は、Webサイトとバージョン管理システムGitおよびMercurialの統合に関するものです。 以前は、統合と継続的な展開のために、独自のパブリックまたはプライベートGitリポジトリー（ローカル、BitBucketまたはGitHub上）またはMercurial（BitBucketまたはCodePlex上）のみを使用できました 。



現在の更新では 、WebサイトサービスをサードパーティのサードパーティリポジトリGitまたはMercurial と統合する機能が追加されました 。







当然、この統合では、Windows Azureがリポジトリへのアクセスを必要とするため、各コミット後に自動的に投稿する機能が失われます。 ただし、開発者はWindows Azure管理ポータルの機能を使用して、リモートのサードパーティリポジトリを自分のWebサイトWebサイトと独立して同期できます。
