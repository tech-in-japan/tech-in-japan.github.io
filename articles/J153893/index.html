どこから始めますか？



少し考えて、私は必要なものを実現しました：

  • Vkページで新しい投稿をスキャンします
  • これらの同じ投稿で、グループへの参照を検索します（例：143）
  • グループの番号が投稿に見つかった場合、この投稿をこのグループの携帯電話に送信します


Debianにはホーム「サーバー」があり、これを誇らしげにGrindelServerと呼んでいます。それに基づいて、これをすべて実装するつもりでした。

コマンドラインからSMSを送信する方法を単に学ぶことから始めることにしました。 そのため、Huawei E1150の3Gモデムが3台ほどあり、そのうちの1台を使用することになりました。 Googleはこのトピックに関する資料を簡単に見つけました。 要するに、 モデムを「モデムのみ」モードに切り替えて （そのため、まだ偽のCDドライブとカードリーダーを持っている）、GnokiiまたはGammuプログラムを設定して動作させる必要がありました。

まず、モデムを使用する権限を設定する必要があります。これらはこの例でのみ必要なので、再起動後にリセットされるという事実を心配する必要はありません。

mysql経由でSMSを送信する


$ man gammu-smsd-tables



、このコードは同じphpMyAdminで実行できます 

 INSERT INTO outbox ( DestinationNumber, TextDecoded, CreatorID, Coding ) VALUES ( ' ', 'This is a SQL test message', 'Program', 'Default_No_Compression' );


原理は次のとおりです

  • DestinationNumberは、SMSが到達する番号です
  • TextDecoded-来るはずのテキスト
  • CreatorIDは、どのプロセスがこの投稿を作成したかを示す単なるタグです
  • コーディング-エンコード。SMSが英語の場合は「Default_No_Compression」が示され、ロシア語の場合は「Unicode_No_Compression」が示されます。


注意！ SMSに送信できるのは、ラテン文字160文字とキリル文字70文字だけです！ より多くのテキストを送信するために、「マルチパートSMS」があります。 Gammu-smsdはそれらの送信をサポートしています。プロセスは$ man gammu-smsd-tables

スクリプト解析


簡単に説明すると、スクリプトのアクションのシーケンスは次のとおりです。

  • プログラムによって最後に送信された投稿のIDを調べる
  • 最後の投稿以降に新しい投稿が表示されているかどうかを確認します
  • 新しい投稿のそれぞれで少なくとも1つのグループを見つけてください。
  • グループ番号が見つかった場合は、グループ番号でSMSを送信します（つまり、smsdデータベースの送信トレイテーブルに特定のレコードを送信します）


データベースについて説明する価値があります。 VsPostsおよびGroupsテーブルをsmsdデータベースに追加しました。 VkPostsでは、プログラムによって送信されたすべての投稿のIDが追加されるため、次回、それぞれ最新の送信された最後の投稿のIDを要求できます。 そしてグループには、学生番号とグループ番号があります。 数。
