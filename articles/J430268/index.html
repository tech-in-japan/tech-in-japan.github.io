サーバーレスはDevOpsを殺しますか？





iPhoneやFacebookの次の新生児殺人者について読む必要があるたびに、思わず笑顔を始めます。 長年にわたり、それらの多くがありましたが、物事はまだそこにあります。 したがって、次のサーバーレスアプリケーションコンピューティング（サーバーレス）の時代がDevOpsを「食べる」という考えが鍵となる別の出版物に出くわしたとき、懐疑論に制限はありませんでした。



公平に言えば、クラウドテクノロジーはあらゆる面で本当に「ベストを引き裂く」と言っておく価値があります。 多くの点で、この理由は、革新の効率と速度を向上させたいというビジネスの永続的な要望を背景にした高度な技術のシンプルさとアクセシビリティの結果として、ハードウェアの価格が急速に下がることです。 誰もが今、そしてできれば無料でそれを望んでいます。



この背景に対してサーバー側を構成する必要のないアプリケーションの作成と起動を伴うサーバーレスの概念の出現は、非常に理にかなっています。 Serverlessという名前の人気にもかかわらず、略語FaaS（サービスとしての機能）も見つかりました。 したがって、悪名高いサーバーレスに惑わされるべきではありません。 サーバーが完全に拒否されるわけではありません。 ご存知のように、「何かが消えた場合、どこかに到着しました。」 この場合、ハードウェアがAmazon、Microsoft、およびその他の業界モンスターの側に行き、サーバーキャビネットのサイズやDevOpsチームとの良好な関係に関係なく作成する機会を通常の開発者に提供するという事実について話している。







サーバーレスコンセプトの主な利点：



  • 製品の作成と展開の容易さ。
  • プロジェクトを迅速かつ簡単にスケーリングする機能。
  • バックエンドの高可用性とフォールトトレランス。
  • サーバーインフラストラクチャを管理する必要がないため、パフォーマンスをサポートするためのコストを削減できます。
  • アプリケーション開発の高速化。
  • インフラストラクチャコストとDevOpsを削減します（そして、Dockerは不要になります）。

DevOpsは死んでいて、長生きするDevOpsですか？



サーバーレスは自動化を大きく補完します。 AWS、Azure、IBM Cloud、またはGCPを使用する優れたDevOpsは、適切なサーバーレスソリューションを適用して、アプリケーションの管理性を向上させることができます。 しかし、環境が豊かなほど、より多くの人々が環境に適応できるようになります。 言い換えれば、DevOpsはもちろん死ぬことはありませんが、専門家の知識とスキルの要件は必然的に進化します。



さらに、サーバーレスの範囲はすでに非常に広くなっています。



  • ファイナンス
  • 小売
  • IoT
  • ソーシャルメディア
  • チャット
  • Uberのようなアプリ


サーバーレスでは、非常に異なるシナリオを実装できます。



  • アプリとサイトのバックエンド
  • データ処理（画像、ビデオ、ログ）
  • IoT（SmartCityを含む）
  • サーバーレスWebサイト
  • 自動化されたタスク（バックアップを含む）
  • クラウドレット
  • （データの前処理）
  • チャットボット
  • プログラミング学習環境






誰がここにいますか？



Amazon AWS、Azure、IBM Cloud、Google Cloud、Oracleなどの主要プレーヤーがすでにこの市場に参入しています。 膨大なリソースにより、ITの巨人は有望な分野で仕事をすることができ、傾向を把握し、開発において業界を大きく先取りしています。 さらに、ある程度サーバーレスを実装する多くのオープンソースプロジェクトがあります。 ロシアの風景はさらに興味深いものです。







今日はかなりカラフルな絵があります。 一方では、世界の他の場所と同様に、マイクロソフト、アマゾン、グーグルなどの市場リーダーからのソリューションが支配的であり、他方では、本当に興味深い代替案が生まれます。



これらには、 ロシアで最初のサーバーレスクラウドであるswifty.cloudに基づいたRusonyx サーバーレスクラウドが含まれます 。 これは、 テリーの急勾配でスケーラブルなバックエンドへの簡単な方法です。



swifty.cloudに基づくサーバーレスのRusonyxは、実際にはアプリケーション、サイト、チャットボットのバックエンド用のすぐに使えるプラットフォームですが、同時に従来のサービスとしてのバックエンドソリューションの機能に制限されません。 必要なサービスのほとんどが含まれています。



  • サーバーレス機能
  • SQLおよびnoSQLデータベース
  • オブジェクトストレージ
  • サービスとしての認証
  • Mac / Linux用の素敵なUI、API、CLI
  • 機能および完全なサービステンプレート


条件：

Rusonyxサーバーレスプラットフォームを使用して最初の3か月は無料なので、誰でもサーバーレスアプローチを試すことができます。



結論の代わりに



従来のDevOpsに関連付けられているスキルセットからさらに離れています。 テクノロジーは変化しており、それらとともに変化しなければなりません。 同時に、クラウドプロバイダーがすべてを行っても、製品品質管理機能はどこにも行きません。 自動化により、日常業務から解放され、長期間延期した可能性のあることを実行する機会が得られます。



swifty.cloudのメンバーが言うように、アイデアをアプリに取り入れましょう 。
