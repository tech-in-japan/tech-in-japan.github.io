JavaOne / Oracle Develop：一般的な印象

4月12〜13日、ロシア科学アカデミーでモスクワで初めて、JavaとOracle開発者の有名な会議-JavaOne / Oracle Developが開催されました。 私は両方の日を訪れましたが、私が参加したレポート、一般的な印象、さまざまな家庭の些細なことについて意見を共有したいと思います



1日目



1. HTML 5およびJava：新しい視野。 最初の報告は非常に弱かった。 HTML 5で既によく知られていることと、一般的にJavaとは何かについて話しました（Javaが何であるかを既に知っている人にとってはなぜですか？） 好きじゃなかった。

2.低遅延のアプリケーションの作成に関するドイツ銀行からの報告 。 全体的に良好なパフォーマンス。 JavaでのCスタイルの記述方法について:)

3. Oracle JDeveloperおよびOracle ADF：Oracle Fusionソフトウェア開発プラットフォーム。 彼らは、1行のコードを書くことなく、JDeveloperでかなり複雑なアプリケーションを非常に迅速に作成する方法をあらゆる種類のチップに示しました。 7分で、ゼロからの人々が、入力データ、さまざまなグラフの検証を行うCRUDアプリケーションを作成しました。 しかし、個人的な経験から言えば、生産においてすべてが同じように美しいと信じることは困難です。 節約された時間は、開発者が予測していなかった場所で松葉杖をカスタマイズして書くことに費やされます。

4. 問題を診断し、特定のJavaアプリケーションのニーズに合わせてHotSpot JVMでGCを構成します。 ガベージコレクターの主なモデル、いつ使用するのが適切で、どの状況でどのパラメーターを強化するかを調べました。 スライドを見つけることは非常に便利なリファレンスです。

5.動作中の言語指向プログラミング。 Jet Brainsでの方法。 すべてのレポートの私の個人的なトップ-最初の場所。 驚くほどクールなこと（手でも技術と呼ばない）-MPS。 革命か進化かはわかりませんが、時代の先を行っています。 要するに、DSL、あらゆる種類のマクロ構築、およびプログラミング言語の作成と拡張のための一種のシステムです。 私たちのオフィスでのセミナーで詳しくお話しします:)



2日目



6. JavaFX 2.0。 どういうわけか、最初はこのテクノロジーを信じていなかったので、このプラットフォームの将来について希望を抱かせてくれるという報告はありませんでした。 彼らは、2.0は実際にゼロから書き直されたと述べ、GWTのようなHTML + JavaScriptでのコンパイルが行われるようになりました。 それは、デスクトップソフトウェア（UI）のプラットフォームとして機能する可能性がありますが、ダムスウィングに取って代わります。

7. Java仮想マシンの応答が速いアプリケーションの開発とチューニング。 最初は、すべてがGCレポートと同じであり、行ったことに動揺していましたが、最後には、アプリケーションがうまく機能し、ガベージコレクションが安価になるようにコーディングしない方法を教えられました。

8. Oracle SQL Developer 3.0を使用したPL / SQL開発およびブロックテスト（著者の注意：ユニットテストをそのように翻訳しました）。 他に行く場所がなかったので、ここに行きましょう。 私はほとんど眠りそうになりました:)あまりおもしろいトピックではありません+女性の話し手の小声...漠然と覚えています。 視覚的なクエリ作成、クエリのテスト方法についての何か。

9. 互換性のあるAPIの開発-悪意のない天才。 2番目に興味深いレポートは、かなり簡単なことを説明していましたが、:) IPAで何ができるかを説明しました。 JDKのさまざまなバージョンの互換性をテストするチームでの経験の真髄。



一般的な印象。



一般的に、私はそれが本当に好きだった。 5点中4点。

何が好きでしたか：

  1. 一般的な雰囲気。
  2. 多数のジャビスト。GoogleDeveloper Dayでは不十分でした。
  3. 優秀な英語を話す有能で快適なスピーカー。 GDDでは、翻訳システムなしでは難しい場合がありましたが、翻訳システムさえ取りませんでした。
  4. コーディングの特定のことについて話した場合、IDEですぐに実演されたという意味で、非常に活発なレポートです。
  5. バックパックやその他の贈り物の形でのボーナス。
  6. アフターパティー、それは少し弱いですが、もっとビールが必要でした。さもなければ私たち全員が飲み、みんな逃げました:)


気に入らなかったもの：

  1. ケータリング 栄養に関する肯定的なレビューを聞いたことがありません:)コーヒーは特定の時間にのみ配られ、Javaロゴは一杯のコーヒーでした。 さらに、コーヒーはすぐに嫌でした。 昼食は別の問題です。 150グラムのサラダ、チーズとソーセージのバター、少量のスニッカーズと少量のジュース。 コメントは冗長です。 GDDでは、シックな食事がありました（ランチとバーベキュー、たくさんのサラダ、バター、パン、クッキー、ソーダ、挽きたてのコーヒー、お茶-いつでもすべて）、そこは無料でしたが、チケットはここで購入されましたもちろんお金ですが、フリーローダーを取り除くために）。
  2. 非常に良い部屋ではありません。 GDDでは、この点で非常にクールでした。1つの大きなホールに食べ物、チルアウト、1つの壁に沿って4つのドアがあり、すべて同じフロアにあります。 シンプルで明確。 ここでは、マップでさえ助けにはなりませんでした。 あなたが今どこにいるかを理解することは決して不可能でした（これは小さなホールに適用されます）。 壁のサインがない場合-私はほとんどヒットしなかっただろう

    一般に、少なくとも1つのレポートに対して。


次回私が絶対に行くとき、主催者が迷惑な欠陥を取り除くことを願っています！
