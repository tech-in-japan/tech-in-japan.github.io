Veeamは、物理マシンとサーバー用の無料のバックアップユーティリティを発表します

ラスベガスの罪の街で開催されたVeeam 3日間会議が終了してから1週間以上が経過し、同社の経営陣は将来の計画を共有し、仮想化されたデータの世界の現在の問題、ストレージ、処理、中断のないアクセシビリティについて議論し、テクニカルレポートを読み、業界のリーダーによる優れた会議には、他にも興味深いイベントが数多くありました。

しかし、とりわけ、新製品の発表を強調したいと思います-Veeam Endpoint Backup with the Free registry。 このアプリケーションの唯一のタスクは、物理マシンのバックアップです。







カットの下で、私は今後の新製品に関する現在入手可能なすべての情報を蓄積し、仮想環境の認識されたJedi Masterのバックアップが隣接エリアに行くことを計画した理由を考えようとしました。



そして多分あなたはすぐに反対します：

  • Veeamは、未来は仮想化にあり、鉄のサーバーは前世紀と基本であると常に主張してきたため、物理マシンのバックアップに対処することは決してありません。 それは嘘であることが判明しましたか？


しかし、彼らは嘘をつきませんでした。 それを「しゃれ」と呼びましょう。 名前で結ばれたエンドポイントという用語は、特定のユーザー（エンドユーザー）の作業マシンとして理解される必要があります。これには、良い意味で、作業ファイルとサーバー以外のアプリケーションのセットのみが含まれます。 Veeamは、将来の製品を、ITプロフェッショナルの個人ワークステーション、さらには個人、家庭、およびランプ、特に もちろん、エンタープライズ環境では、誰もその使用を禁止しませんが、常識的には、集中管理、エンドユーザーに対して完全に透過的な機能などを備えた専門製品を優先すべきです。



もちろん、違いはもっぱらイデオロギーですが、一般にサーバーとワークステーションの要件、特にそれらのバックアップについて議論することになると、これらの要件がそれらを2つの異なるキャンプに分割します。 たとえば、多くの問題が解決するため、クラスタリングの可能性について覚えておく必要があります。

しかし、ペトカ、ニュアンスがあります！ （c）

アプリケーションを個々のマシンにインストールし、設定を規定し、すべてが意図したとおりに機能することを確認する必要があります。 一元化された展開と管理はありません。 すべてが排他的に処理されます。

しかし、もちろん、あなたの裁量で誰もアプリケーションの使用を禁止しません。 退屈した鉄製サーバーが1つあり、インフラストラクチャ全体が既に仮想世界に移行されている場合は、おそらくEndpoint Backupを使用するのが最善のソリューションです。

  • 純粋に技術的な詳細については少しですか？


間違いなくそれらがあります！ 現在発表されているのは次のとおりです。

-Windows 7以降で動作します。 サーバープラットフォームはWindows Server 2008からサポートされます。残念ながら、すべての* nixは過去のものです。

-バックアップファイルは、すべてのVeeam製品に固有のVBK形式で保存されます。 これにより、バックアップ＆レプリケーションやEnterprise Managerなどの「アダルト製品」から操作できるようになります。 たとえば、クラウドストレージにバックアップを自動的に送信します。

-バックアップとデータリカバリは、3つの反復で可能です：個別のファイル、ディスク全体、マシン全体。





-好きな場所にバックアップを保存できます





-マシン全体をベアメタルに復元する場合、回復ディスクが作成されます。回復ディスクには、鉄のテスト、ブートキューの管理、ローカル管理者パスワードのリセットなど、小さくても価値のある設備が用意されます。
