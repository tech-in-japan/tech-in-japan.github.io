「このハーフチェアで、マスターガンベスは家具の新しいバッチを開始します。 1865年サンクトペテルブルク」（I. Ilf、E。Petrov）。

あなたは間違っていませんでした。これは、ホームルーターの代替ファームウェアの組み立てに関する別の記事です。これらは、いくつかの場所にある小さなローカルネットワークのホットオーバーロードです。 合理的な質問に、なぜそのような材料が答えであるかをすぐに書いてはいけません：アセンブリのために仮想マシンを準備することからプロセスを説明したいと思います（はい、それはVirtualBox上のUbuntuになります）彼は、プリンターを接続し、メディアサーバーやトレントクライアントなどのパッケージをインストールして構成し、サードパーティのクライアントをインストールせずにiPhone、iPad、Windows Phoneから接続できるようなStrongSwan構成の説明で終了しようとします。 同時に、セキュリティなどの重要なものと、夜間に青色のTL-WDR3600青色LEDをオフにするスクリプトのような便利なものの両方を無視したくない。 そして、私は現在表明されている多くの質問に関する私の知識の完全性については全く確信が持てません。

一言で言えば、この段階では多くの計画と混乱があります。 したがって、アドバイスや提案は重要です。 記事のテキストは間違いなく変更され、時間の経過とともに補足され、Habrを詰まらせないように投票結果に基づいて記事全体をドラフトコピーに隠すことができます。 興味のある方、アドバイスをしたい、または単にスリッパを作者に投げたいと思ったら-ようこそ。 さらにテキストに沿って、この有望な興味深い娯楽のアクションを詳細に説明しようとします。その本質は中世の日本人アーティストによって描かれました。

組立プロセスの本質（注意！8+）








「武器-検査のために！」



最初に、製造元のWebサイトで発表されたデバイスレビューの標準ファームウェアTL-WDR3600の機能について簡単に説明します 。

  • 同時接続-2.4 GHzの周波数で最大300 Mbit / s、5 GHzの周波数で最大300 Mbit / sの合計スループットで600 Mbit / s
  • 2つのUSB 2.0ポートにより、ユーザーはローカルネットワークまたはインターネット経由でプリンターとファイルを簡単に共有できます。
  • ギガビットポートにより、最大のデータ転送速度を実現できます
  • ハードウェアNATのおかげで、WANからLANへの最大帯域幅は800 Mbpsに達します
  • IPTVマルチキャストのIGMPプロキシおよびブリッジサポート
この素晴らしさの中で、恐ろしく見える独自のユーティリティによって実装されたプリンタの共有の「使いやすさ」について個別に言及したいと思います。 簡単に言えば、上記のほとんどはOpenWrtを使用して簡単に実装できます。 しかし、忘れないでください、私たちの目標はTP-LINKファームウェアと比較することではありません。 OpenWrtがどのように回避できるかを知りたいと思います。



Openwrtへの切り替えは、いくつかの方法で実行できます。 サポートされているデバイスの場合、完成したイメージをダウンロードし、工場出荷時のファームウェアから直接入力できます。 さらに、USBポートがある場合は、事前にフォーマット済みのUSBフラッシュドライブをルーターに接続し、いくつかのセクションに分割します。 フラッシュドライブからのブートが設定され、必要なパッケージがインストールされます。 sayingにもあるように、「マダム！ ファウストパトロンは簡単です。」

OpenWrt Buildrootを使用してソースから独自のイメージを構築できます。これにより、 フランケンシュタインのモンスターのようなファームウェアがビルドされ、複数のアドレスからソースコードの断片が収集されます。 多くの場合、プロセスは中断され、リリース日からより多くの時間が経過します。 それから、新たに造られたビクター・フランケンシュタインは、無限のパッチとシンボリックリンクの形の絆創膏の助けを借りて、彼のコーヒーマシンを奇跡のマシンにするはずです。 この方法は多くの可能性を開きます。たとえば、ファイル名に疑問符の代わりにBusyBoxにキリル文字を表示させることもできます。 しかし、同時に、ほとんどの場合、必要なパッケージをすべて自分で収集する必要があり、OpenWrtリポジトリのほとんどのパッケージは使用できなくなります。

私たちは妥協せずに、 Image Builderを使用します。これにより、プリコンパイルされたパッケージからのファームウェアの比較的簡単なアセンブリと、OpenWrtリポジトリからのパッケージとの互換性が約束されます。

「弾薬の流通業者に、シフトに3回の実弾を与えてください！」



  • Oracle VM VirtualBoxおよびOracle VM VirtualBox Extension Packをダウンロードしてインストールします。
  • お気に入りの使い慣れたLinuxディストリビューションのインストールディスクイメージをダウンロードします。 そうでない場合は、Ubuntu Linuxの現在のLTSバージョンをダウンロードできます。 この場合、 代替ダウンロードページからネットワークインストーラーをダウンロードすることをお勧めします。サイズは小さく、インストール中に必要なパッケージ（デスクトップ環境など）を選択できます。

    VirtualBoxで作業するとき、私のデフォルトのシェル-ゲストシステムにアドオンをインストールした後でも、 Unityはひどくブレーキをかけました。 私は、なぜそれを治すことができるかを理解しようとしませんでしたが、パッケージを選択する段階で、 Xfceに基づいてXubuntuデスクトップに注目しました。 ゲストシステムの容量に関しては、仮想マシンに4 GBを超えるRAMを割り当てない場合、i386が論理的な選択肢になると思います。
  • 仮想マシンを作成し、ダウンロードしたイメージと同じビット深度のUbuntu Linuxを選択したゲストオペレーティングシステムを使用します。1GBのRAMで起動に十分ですが、約20 GBのハードディスクでも十分です。 マシンを作成した後、NATの代わりにネットワークを再構成して、ネットワークブリッジを選択します。 WiFiアダプターの代わりに、ラップトップ用のネットワークカードがある場合は、それを選択する必要があります。 光学ドライブで、インストールディスクのイメージをマウントします。
  • ゲストオペレーティングシステムをインストールします。 この段階では、ディスクパーティションの段階では自動パーティションに同意せず、手動で2つのプライマリパーティションを作成することをお勧めします。 最初はスワップ用の2 GBで、残りはルートの下にあります。 このようなディスクパーティション分割により、必要に応じて、仮想ハードディスクを増やした後、ルートパーティションのサイズをすばやく変更できます。
  • 仮想マシンにオペレーティングシステムをインストールすると、VirtualBoxでいくつかの標準画面解像度が利用可能になります。 ほとんどの場合、大きな対角線のラップトップまたはモニターには適していません。 この誤解を修正し、ゲストシステム用のVirtualBoxアドオンをインストールします。 同時に、コンソールファイルマネージャーであるMidnight Commanderを配置します。

    まず、不足している画面解像度を追加します。 私の知る限り、最大16個のユーザー定義の画面解像度を追加できます。 以下の例では、「Test」は仮想マシンの名前であり、「1366x768x32」は追加された画面解像度です。 ホストシステムのコマンドラインを開き、VirtualBoxディレクトリに移動して、目的の画面解像度を追加します。 

    cd "C:\Program Files\Oracle\VirtualBox" VBoxManage setextradata "Test" "CustomVideoMode1" "1366x768x32"
  • ゲストシステムに移動します。 ターミナルエミュレータを開き、パッケージのリストを更新し、Midnight Commanderとカーネルモジュールのビルドに必要なパッケージdkmsおよびbuild-essentialをインストールします。 これからは、Ubuntuまたは別のDebianのようなシステムで作業しており、 aptitudeとnanoがすでにインストールされていると仮定します。 イメージからインストールすると、ネットワークインストーラーがインストールします。 そうでない場合は、例のコマンドを自分で調整してください。 

     sudo aptitude update && sudo aptitude upgrade sudo aptitude install dkms build-essential mc
    ゲストシステムのアドオンディスクイメージを仮想マシンに接続し、そこからアドオンインストールスクリプトを実行します。 

     cd /media/%username%/VBOXADDITIONS_4.3.20_96996/ sudo ./VBoxLinuxAdditions.run
    スクリプトが完了したら、ターミナルを閉じ、ゲストシステムのアドオンイメージを無効にして、仮想マシンを再起動します。 すべてがうまくいった場合、フルスクリーンモードに切り替えると、仮想マシンの画面解像度はモニターの通常の解像度に対応します。 ただし、これまでのところグラフィカルモードでのみ、コンソールに切り替えると、画面の中央の小さな部分を占めることがわかります。 これを修正するには、ブートローダーの設定を少し調整する必要があります。 端末エミュレーターを開き、ブートローダー設定を編集します。 

     sudo nano /etc/default/grub
    起動時にGrubメニューの表示を構成します。 

     #GRUB_HIDDEN_TIMEOUT=0
    Grubグラフィカルメニューの解像度を設定し、コンソールのブートローダーによって設定された解像度を保存します。 

     # The resolution used on graphical terminal # note that you can use only modes which your graphic card supports via VBE # you can see them in real GRUB with the command 'vbeinfo' GRUB_GFXMODE=1366x768 GRUB_GFXPAYLOAD_LINUX=keep
    設定を保存し、ブートローダー構成を更新して再起動します。 

     sudo update-grub && sudo reboot
    ブートローダーの設定を更新した後、再起動したときにメニューが表示されませんでした。 同じ場合は、仮想マシンをオフにしてから再度オンにします。 全画面モードで起動すると、ブートローダーメニューが全画面で表示されます。Ctrl+ Alt + F1で最初のコンソールに移動すると、画面全体が表示されます。 この場合、ログイン、パスワードを入力し、フォントを設定できます。 ここのすべてはもちろん個別ですが、標準的なフォントは通常小さいように思えます。 あなたも、これを行う： 

     sudo dpkg-reconfigure console-setup
    太字のフォントが好きなら-TerminusBoldを、そうでなければ-Terminusを選択してください。 サイズ-あなたのビジョンに応じて、いくつかを決定することができます。

    ささいな信頼であなたを怒らせることを恐れていますが、Ubuntuでは、Ctrl + Alt + F7でコンソールからグラフィックモードに戻ることができます。
  • Image Builderのバージョンをダウンロードして解凍します。 TL-WDR3600のファームウェアを構築しました。ar71xx汎用アーキテクチャを選択する必要があるためです。 デバイスのアーキテクチャを改良し、サポートされているデバイスの表で OpenWrtでサポートされているかどうかを確認できます 。 アーカイブをホームディレクトリに展開し、便宜上、名前をOpenWrt-ImageBuilderに短縮します。 OpenWrt-ImageBuilderに移動し、ファームウェアに追加するファイルのファイルディレクトリを作成します。
  • 正常に機能するファームウェアをビルドするには、スクリプトを少し調整する必要がありました。 問題の本質は、「make clean」コマンドは、アセンブリ中にダウンロードされたファイルが保存されているdlディレクトリを削除せずに削除することです。 それから、このエラー専用のチケットを見つけて、修正済みとマークしました。 動作させるには、ターミナルエミュレータを開いてMakefileを編集します。行161-162を見つけます。 

     clean: rm -rf $(TOPDIR)/tmp $(TOPDIR)/dl $(TARGET_DIR) $(BIN_DIR)
    そして、私たちはこのフォームに持ってきます： 

     clean: rm -rf $(TOPDIR)/tmp $(TOPDIR)/dl/* $(TARGET_DIR) $(BIN_DIR)
    ファイルを使用してこの小さな修正を行った後、テストアセンブリを実行できます。テストアセンブリは次のコマンドで実行されます。 

     make image PROFILE=WNDR3700 PACKAGES="pkg1 pkg2 pkg3 -pkg4 -pkg5 -pkg6" FILES=files/
    場所：WNDR3700-デバイスのプロファイル。 pkg1-ファームウェアに追加するパッケージ。 -pkg4-ファームウェアから削除されるパッケージ。 プロファイルのリストは、make infoコマンドの出力にあります。 OpenWrt WikiでImage Builderの操作について詳しく読んでください。 ビルドを開始する前に、リポジトリをrepositories.confファイルに追加する必要があります。 それをエディターで開き、次のフォームに持ってきます。 

     src/gz barrier_breaker_base http://downloads.openwrt.org/barrier_breaker/14.07/ar71xx/generic/packages/base src/gz barrier_breaker_luci http://downloads.openwrt.org/barrier_breaker/14.07/ar71xx/generic/packages/luci src/gz barrier_breaker_management http://downloads.openwrt.org/barrier_breaker/14.07/ar71xx/generic/packages/management src/gz barrier_breaker_oldpackages http://downloads.openwrt.org/barrier_breaker/14.07/ar71xx/generic/packages/oldpackages src/gz barrier_breaker_packages http://downloads.openwrt.org/barrier_breaker/14.07/ar71xx/generic/packages/packages src/gz barrier_breaker_routing http://downloads.openwrt.org/barrier_breaker/14.07/ar71xx/generic/packages/routing src/gz barrier_breaker_telephony http://downloads.openwrt.org/barrier_breaker/14.07/ar71xx/generic/packages/telephony ## This is the local package repository, do not remove! src imagebuilder file:packages
    公式のOpenwrtリポジトリを追加しました。 膨大な数の計画があり、ファームウェアコレクターの起動を容易にするために、OpenWrt-ImageBuilderディレクトリに小さなスクリプトを作成します。 

     #!/bin/bash # Build OpenWRT 14.07 for TP-LINK TL-WDR3500/3600/4300/4310/MW4350R echo "Prepare OpenWRT images for Tp-Link TL-WDR3500/3600/4300/4310/MW4350R \n" make image PROFILE="TLWDR4300" PACKAGES="luci openssh-sftp-server nano" FILES="files/"
    たとえば、スクリプトをtl-wdr3600.shとして保存し、その実行を有効にして、ファームウェアアセンブリを実行します。 

     chmod +x tl-wdr3600.sh ./tl-wdr3600.sh
    ビルドの最後に、いくつかのファイルがbinディレクトリに表示されます。 ファイル名のデバイスモデルにより、どれがファームウェアであるかを判断できます。 名前の工場という言葉は、そのようなファイルを工場のファームウェアを介してデバイスにアップロードできることを意味します。 このことについて、JFFS2とSquashFSの違いなどについて詳しく説明します。
