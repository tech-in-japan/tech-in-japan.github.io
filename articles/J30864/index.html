Firefox 2微調整ガイド

Firefox 2をさらに便利にしたい場合は、クリックします



About：Configの使用方法



Firefoxを微調整するには、 about：構成画面を使用します。 あなたの多くはそれを使用する方法を知っていると確信していますが、あなたが知らなくても、私はあなたに話します。

  1. Firefoxを起動します。
  2. アドレスバーに「 about：config 」と入力し、Enterキーを押します。 次のような画面が表示されます。

  3. そのようなキーが既に存在すると言う場合、「フィルター」フィールドにその名前を入力します。 キー値を変更するには、それをダブルクリックします。
  4. キーが存在しないと言えば、作成する必要があります。 キーには、文字列、ブール、整数のいくつかのタイプがあります。 画面上の任意の場所でマウスの右ボタンをクリックするだけで作成できます。

  5. 場合によっては、ブラウザを再起動して新しい設定を有効にする必要があります。 デフォルト値に戻すには、 about：configで目的のキーを右クリックし、「 リセット 」を選択する必要があります。

    まあ... それは調整する時間です！




タブスクロール設定



現在、多数のタブが開いているFirefox 2では、タブをスクロールするためのボタンが表示されます。 ただし、それぞれが独自の方法で「多数」を理解します。 誰かがそれらをスクロールするよりも多くの小さくて読めないタブを持っている方が便利です。

  • タイトル： browser.tabs.tabMinWidth
  • キーは既に存在し 、再作成する必要はありません。
  • 標準値： 100 （ピクセル）
  • 可能な値：

    • 値が100を超えると、できるだけ早くスクロールが表示されます。
    • 値が100未満の場合 、スクロールは後で表示されます。
    • 値0は、スクロールを無効にします。




赤いXを設定する（閉じるボタン）



閉じるボタンの場所は、さまざまなユーザーの間で多くの論争を引き起こします。 各タブの閉じるボタンが嫌いな人と、便利だと思う人の両方に会いました。 私はこのオプションが好きですが、そうでない場合は、これを簡単に変更できます：

  • タイトル： browser.tabs.closeButtons
  • キーは既に存在し 、再作成する必要はありません。
  • 標準値： 1
  • 可能な値：

    • 0-アクティブなタブにのみ閉じるボタンを表示します
    • 1-すべてのタブに閉じるボタンを表示します
    • 2-閉じるボタンをまったく表示しない
    • 3-タブバーの最後に唯一の閉じるボタンを表示する（Firefox 1.xの場合と同様）




すべてのタブに閉じるボタンを残すことにした場合、興味のある別のオプションがあります。 閉じるボタンが表示される最小のタブ長を示します。 このオプションは、現在アクティブではないタブにのみ適用されます。

  • タイトル： browser.tabs.tabClipWidth
  • キーは既に存在し 、再作成する必要はありません。
  • 標準値： 140 （ピクセル）
  • 可能な値：

    • 値が140を超えると、閉じるボタンが早く非表示になります。
    • 140未満の値は、後で閉じるボタンを非表示にします。
    • 値を0にすると、閉じるボタンがオフになります。






アドレスバーから「I'm Feeling Lucky」を検索する



Firefoxは、GoogleのI'm Feeling Lucky検索メソッドを使用しなくなりました。 現在、同様の方法が使用されていますが、別の方法は「名前で参照」です。 実際、違いに気付かないかもしれませんが、違いはあります。



要するに、「名前で検索」は、最初の結果がまさに探しているものであるとGoogleが確信している場合にのみ、サイトに移動します。 別のケースでは、検索結果のあるページに移動します。 「Microsoft」と入力すると、両方の方法の結果は同じになり、「Stanford」と入力すると、結果は異なります。



名前による検索を終了する場合は、何もする必要はありません。 「I'm Lucky」メソッドに戻りたい場合は、以下の情報が参考になります。

  • タイトル： keyword.URL
  • キーは既に存在し 、再作成する必要はありません。
  • デフォルト値： 名前で検索 （右ボタンを押してアドレスをコピー）
  • 可能な値：

    • 私は幸運です （右ボタンを押してアドレスをコピーします）
    • 他の検索アドレスを入力することもできます。


不適切なスクリプト



不適切なスクリプトに関する通知の受信にうんざりしていませんか？ Firefox 2では、スクリプトの実行に20秒が与えられ、この時間が経過するとエラーメッセージが表示されます。 このようなエラーが頻繁に発生する場合（脆弱なコンピューターでよく発生します）、スクリプトの継続時間を増やす必要があります。

  • 名前： dom.max_script_run_time
  • キーは既に存在し 、再作成する必要はありません。
  • 標準値： 20 （秒）
  • 可能な値：

    • 値が20を超えると、スクリプトの実行に時間がかかります。
    • 20未満の値は、スクリプトの実行にかかる時間が短くなります。




マイクロサマリ更新間隔（マイクロサマリ）



Microheadingsは非常に便利な機能になると約束されていますが、いくつかの改善が必要です。 Cyber​​Netでは、 3種類のマイクロヘディングを使用して、新しい投稿やコメントについてユーザーに通知します。 また、まれな更新のみのためにそれらを使用しない場合は、すぐに更新間隔を変更できます。

  • タイトル： browser.bookmarks.microsummary.updateInterval
  • キーが存在しないため 、作成する必要があります。 タイプ： 整数
  • デフォルト値： なし
  • 可能な値：

    • 更新間の分数。




ユーザーエージェントの変更



ブラウザーでサポートされていないというサイトを入力するときにメッセージが表示される場合、考えられる理由はFirefox 2を使用していることです。 この変更後、サイトはFirefoxの以前のバージョンで表示していると見なします。

  • タイトル： general.useragent.extra.firefox
  • キーは既に存在し 、再作成する必要はありません。
  • デフォルト値： Firefox / 2.0
  • 可能な値：

    • Firefox 1.5
    • Firefox 1.5.0.7
    • など どんな識別文字列でもかまいません。






「進む」ボタンを削除します（移動）



気付いていない場合は、アドレスパネルから新しい[転送]ボタンを削除できません。 はい、パネルからボタンを削除する標準的な方法に従いませんが、別の方法で削除できます。

  • タイトル： browser.urlbar.hideGoButton
  • キーは既に存在し 、再作成する必要はありません。
  • デフォルト値： false
  • 可能な値：

    • false- 「転送」ボタンを表示します
    • true- 「進む」ボタンを非表示にします






ポップアップJavaScriptウィンドウを表示する



Firefoxのデフォルト設定では、ポップアップウィンドウを開く方法を選択できます。 新しいウィンドウで開くか、タブで開くという2つのオプションがあります。 ただし、これらの設定はポップアップJavaScriptウィンドウには影響しませんが、これを変更して標準設定を適用できます。

  • タイトル： browser.link.open_newwindow.restriction
  • キーは既に存在し 、再作成する必要はありません。
  • 標準値： 2
  • 可能な値：

    • 0-他の新しいFirefoxウィンドウと同じ方法でJavaScriptウィンドウを開きます。 新しいタブで新しいウィンドウを開くように設定すると、JavaScriptウィンドウもこれらの設定に従うため、この値をお勧めします。
    • 1-すべてのポップアップJavaScriptウィンドウを新しいウィンドウで開くことができます。
    • 2 -JavaScriptで他の開く方法が設定されていない場合 、設定に従ってすべてのJavaScriptウィンドウを開きます。




すべてのテキストフィールドでスペルチェックを有効にする



Firefox 2の組み込みのスペルチェックは私のお気に入りの機能ですが、単一行のテキストフィールドではデフォルトで有効になっていません。 マウスの右ボタンを押して適切なメニュー項目を選択することで有効にするか、設定を変更してスペルチェッカーを1行のフィールドに設定できます。

  • タイトル： layout.spellcheckDefault
  • キーは既に存在し 、再作成する必要はありません。
  • 標準値： 1
  • 可能な値：

    • 0-スペルチェックを無効にします
    • 1-複数行のテキストフィールドにスペルチェックを含める
    • 2-すべてのテキストフィールドにスペルチェックを含める




接続の最大数を増やす



複数のファイルをダウンロードすると、Firefoxが一度に最大2つのファイルをダウンロードすることに驚くかもしれません。 一度にさらに多くのファイルをダウンロードする場合は、Firefox接続の数を変更します。

  • 名前： network.http.max-persistent-connections-per-server
  • キーは既に存在し 、再作成する必要はありません。
  • 標準値： 2
  • 可能な値：

    • 一度に複数のファイルをダウンロードするには、2より大きい値を入力します。




新しいタブでインライン検索結果を取得する



Firefoxの右上隅にある組み込みの検索フォームは、検索を大幅に簡素化します。 新しいタブで検索結果を取得し、簡単な変更を加えることができます。

  • タイトル： browser.search.openintab
  • キーは既に存在し 、再作成する必要はありません。
  • デフォルト値： false
  • 可能な値：

    • false-アクティブなタブで結果を開きます
    • true-結果を新しいタブで開きます




セッション回復をオフにする



組み込みのセッションリカバリは非常にうまく機能しますが、私はTab Mix Plusを使用し、セッションを少し良く復元します（たとえば、保護されたタブを記憶します）。 ビルトインリカバリをオフにしないと、複製が発生する可能性があり、各タブのコピーを2つ受け取ります。

  • タイトル： browser.sessionstore.enabled
  • キーが存在しないため 、作成する必要があります。 タイプ： ブール 。
  • デフォルト値： true
  • 可能な値：

    • false-組み込みのセッション回復を無効にします
    • true-組み込みのセッション回復を有効にします




最小化された物理メモリの使用量が少ない



そして最後に、古典的なCyber​​Netです。 この記事は2006年3月に作成され、Diggで4300票を獲得しました 。 それ以来、約20万人が彼女を訪れ、多くの人が探していたものを正確に受け取りました。



この設定は、ブラウザを最小化するとFirefoxを仮想メモリに転送します。 したがって、他のプログラムのために物理メモリを解放します。 Firefoxは、折りたたみ時の物理メモリの使用を約10 MBに削減し、展開すると必要なメモリを再び使用します。 これはあまり正しくないと考えている人もいますが、Operaなどの他のブラウザーはデフォルトでこれを実行します。

  • 名前： config.trim_on_minimize
  • キーが存在しないため 、作成する必要があります。 タイプ： ブール 。
  • デフォルト値： なし
  • 可能な値：

    • true -Firefoxを最小化すると仮想メモリに転送します。
