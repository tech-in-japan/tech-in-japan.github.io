モバイル広告2.0

モバイル2.0アプリケーション-モバイルユーザー向けに分類。 最初のアイデア：モバイルネットワークの加入者は、ネットワーク上に任意の広告を配置して、フィードバックを受け取ることができるはずです。 広告を掲載する可能性は安全である必要があります-加入者はどこにも登録してはならず、ソーシャルネットワークのメンバー、閉鎖クラブなどであってはなりません。



私たちは何をしましたか？ dvolnoy tm時代にあるはずなので、サービス全体が大きなマッシュアップです。 すべてが、用意されているAPIに基づいて、既製の部品から組み立てられます。





1.システムのユーザーはSMSで広告を掲載します。 アナウンスはGSMモデム（実際にはGSM電話とデータケーブル ）によって受信され、 SMSゲートウェイによって処理されます 。 したがって、広告は通常のSMSとして通常の番号に送信されます（プレミアム料金ではありません）。 広告を投稿するコストは、広告の作成者の割合でSMSを送信するコストです。



2.受け入れられた広告はどこかに配置する必要があります。 マッシュアップがあるため、サイトを作成することもできますし、完成したプラットフォームのオープンAPIを使用してそこに広告を配置することもできます。 今回のケースでは、 Livejournal.com APIを使用し、雑誌に広告を掲載しました。 同時に、広告表示の問題は解決されます-LiveJournalを使用している場合は友人のフィードで簡単に表示できます。または、この雑誌のRSSフィードを購読し、別の便利な方法で広告を表示できます。



3.広告を配置するのに登録は必要ありません。 SMSを専用の番号に送信するだけです。 フィードバックが広告を送信したサブスクライバーの数を示すことは適切ではありません-突然著者は反対します。 そのため、 音声SMSに基づく匿名通信システムを使用しました。 公開された各広告には自動的にコードが提供され、これを使用して作成者に音声メッセージを送信できます。 また、著者の電話はどこにも表示されません。



最終的にはどのように見えますか？ これはここです： http : //mobnews.livejournal.com使用に関するすべての指示は、ジャーナルの最初のメッセージに記載されています。 お知らせは、SMSを送信することで雑誌に掲載されます。各お知らせには、 音声SMSシステムを介して著者と通信するためのコードが自動的に提供されます。



当然、システムを複数の雑誌にトピックごとに簡単に増やすことができます。 Livejournal APIを他のAPIに置き換えるだけでなく。 たとえば、liveinternet.ruの管理（またはユーザー）がこのようなサービスを必要とする場合、それを開始できます。



別のポイント。 最近、Web 2.0の時代における従来のメディアの変革について多くの議論が行われました。 上記の安全なフィードバック広告は、変換の一例です。 結局のところ、新聞のネットワークからテキスト形式で再印刷できます。 また、同様の原則に基づいて、新聞のテキストにコミュニケーションの音声チャネルを追加することもできます。



当然、2.0の歴史全体にとって重要な瞬間は、通信環境へのオープンインターフェイスの存在です。 過激派の発言によると、Emerging Telecom会議での議論の1つで、現代の通信事業者は1つの大きなAPIである必要があります。 これまでのところ、これはもちろん、そうではありません。 このプロジェクトでは、テレコミュニケーションコンポーネントがAudiotelのCTI製品に基づいて実装されました。
