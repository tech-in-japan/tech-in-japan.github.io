人工夜明け

それはすべて1年前に始まりました。 2014年の新年の前に、活力はいくらか低下しました。 自己掘り出しのプロセスは、次の考えにつながりました。







「まあ、とても暗い、主よ？」 ©ラジオの日。



しかし、冬に夏に住んでいる人にとって、その考えは非常に自然です。



アイデア



時間を再配置できなかったので、照明を変更して感覚を変えようとすることにしました。 思考の流れはほぼ次のとおりでした。自然光は窓から自然にアパートに入り、光がなければ暗くなります。 窓の外の光を照らすことは困難ですが、窓の外の光であるような錯覚を作り出すことができます。 夜に窓を見ると、暗闇が見えます-これは変更できませんが、夜は人々が通常ライトをオンにしてカーテンを閉じます-それはとても明るく快適です。 そして、夜と昼のカーテン付き窓の違いは何ですか？ そして、光がカーテンを突き破るという事実。 したがって、実際には、アイデア自体が生まれました。窓とカーテンの間のスペースを強調する必要があります。



概念の一般的な概念チェックのために、白いLEDストリップが購入され、キッチンのコーニスに接着されました（これらのテープに接着層があったことが判明しました）。 もちろん、多くの点で心理学の要素があることは承知していますが、それは実際に機能しました。



12Vの白の5メートルテープに必要な2Aは、死んだポータブルディスクのPSUから取得されました。 ただし、コンピューターのPSUからの12Vが悪化することはありません（起動方法を知る必要があるだけで、無負荷で完全にオンにしないでください）。



残念なことに、最初の歴史的実験（ミキサーで撮影）の写真は3枚しか保存されていませんでしたが、一般的なアイデアを伝えていると思います。







はい... まあ、そして混乱は一般的にまあまあです...しかし！ 夕方であることを忘れないでください。 しかし、冬の朝に、そして夏に-それは非常に爽快です！



最初の幸運に触発されて、私は今、自分自身ではなく、自分の子供たちで実験を続けることにしました。 結局のところ、目覚まし時計で自分で起きるのは一つのことであり、不幸な子供を目覚めさせないときに目を覚ますのはまったく別のことです。 そのうちの1人がひさしであり、それが外で軽い場合は、ずっと前にテーブルに座ってレゴを静かに集めていたことを完全によく知っていました... 夜明けが必要です！ 本物。 そして、お父さんの電源が入った朝の7時ではありません（お父さんもまた眠りました）。 そしてすぐではなく、徐々に。 そして、ライトは白ではなく、赤オレンジ黄色でなければなりません。



もちろん、スケジュールされたRGBテープが必要です。 そして、それはマイクロコントローラーとリアルタイムクロックを意味します。 そして、成功したように見える自由時間は、AVRで何か役に立つことをしたいという既存の欲求と一致しました。



実装



タダム。 会いましょう-Voskhod1b！ 自然な夜明けをシミュレートするデバイス、優しい目覚まし時計、オーガナイザー、ナイトランプ、必要なセンサーを追加してコードを変更する場合に必要なものはすべて揃っています。 例：集合信号機、遅刻や休憩を示すオフィス時計、水族館や照明と音楽のデザインなどの照明など。

ナイトライト



スケジュールは、テキストインターフェイスを使用してプログラムされます。 利用可能なコマンドは、コマンド-hでヒントを要求することで取得できます。可能なスケジュールモードは、コマンド-haで見つけることができます。 UART経由またはBluetooth経由で直接接続できます。 色は3つの16進バイトでエンコードされます。 色の値を選択するには、インタラクティブモードを使用できます。 インタラクティブモードでは、RGBとHSVの両方の色成分を制御できます。 作業スケジュールはコントローラのEEPROMメモリに保存されます。 デバイスはいつでもオフにすることができます。 オンに切り替えた後の状態は、「オフになっていないかのように」です。

スキーム ：





MK ATmega168を使用しました。 ATmega328またはATmega88 / ATmega8は対応しますが、後者はコマンドが少し少なくなり、ヒントが短くなります。 私はチェックしませんでしたが、ATmega16が動作することを確信しています。 リアルタイムクロック-DS1307。 LEDはIRF540N MOSFETによって点灯します。 もちろん、このような強力なトランジスタは必要ありません。手元にありましたが、一般的で高価ではありません-残念ではありません。 電力に余裕があり、24Vテープで作業できます。電源に電流保護がある場合、テープの短絡はひどくありません。



メガに必要な5VとHC-05 bluetoothモジュールに必要な3.3Vは、リニア安定装置L7805とL78L33によって生成されます。 ブルートゥースモジュール（U5）はオプションで、抵抗器R11、R12、R13（5V MKをブルートゥースコントローラーの3.3Vとマッチング）はスタビライザーU4です。 回路はU2がなくても起動して動作しますが、もちろん、インタラクティブモードでのみ動作します。



プログラム。 アセンブラーで書きたくなかったので、おそらくプログラムがATtiny2313に適合しなかったので、あまり気にしませんでした。 マイクロコントローラのプログラミングにC ++を使用する可能性を評価することは興味深いものでした。 抽象インターフェースを使用して、MK（Atmel Studio 6.1）とWindows（Microsoft Visual Studio 2012）の両方用にコンパイルされたコードを作成することが可能であることが判明しました。 Windowsでは、プロジェクトはRTC、UART、EEPROM、およびPWMエミュレーションプログラムを使用して構築されます。 ウィンドウタイトルに現在のPWM値を表示する唯一のwindows-api関数SetConsoleTitleA（）を除外すると、Linuxでも問題なくコードがコンパイルされると確信しています。



UARTを使用するためのライブラリは、アプリケーションノートAVR303-割り込み時のバッファ付きハードウェアUARTから取得されます。 コードは、ATmega8 / 16、ATmegaX8、ATiny2313（USART2.pp、USART2.h）をサポートするようにわずかに変更されています。



I2CインターフェイスDS1307を使用するには、AVR315（twim.cpp、twim.h）に基づいてPavel Bobkov ChipEnable.Ruのライブラリを使用します （多くの人に感謝します）。



うーん、プロジェクトは「スレッド上で世界から」組み立てられていることがわかります。 しかし、なぜ驚かれることでしょう-ありのまま。 自慢できるのは、おそらく除算演算を使用しない高速変換HSV-> RGBの実装です（HSV2RGB.cpp、HSV2RGB.hを参照）。



UARTとI2Cの動作は割り込みによって行われるため、プロセッサが退屈することはありません：PWM-ソフトウェア。 つまり、ほとんどソフトウェア-8ビットタイマーに関連付けられています。 実験では、メインループの純粋にプログラムされたPWMは、最大周波数にもかかわらず、見た目がちらつきます。 理由は、目に見えるようになる小さな周波数変動に関連していると思われます。 私のPWMはソフトウェアですが、完全に単純ではありません。LEDは周期の3分の1だけシフトして点灯します。 たとえば、R = G = B = 25の場合、LEDのグロー期間はR [0.24]、G [85.109]、B [170.194]です。 私の気持ちによると、この操作モードでは、テープの見栄えが良くなります。



プリント基板は 、DipTraceプログラムで1つのレイヤーで離婚します。 PPのサイズは、購入したプラスチックボックスのサイズによって決まります。 トランジスタとスタビライザーを配置する必要がありましたが、便利なクランプソケットを拒否する必要がありました-十分なスペースがありませんでした。



電源ユニット -LEDストリップに電力を供給するのに適したユニット 。 通常の5メートルの12ボルトRGBテープ（1メートルあたり60個のLED）の場合、6A電源が必要です。 もちろん、コンピューターのPSUも適しています。 コンピューターからPSUを使用する場合、5Vおよび3.3Vを直接使用できるため、安定装置U3、U4は必要ありません。 コンピューターPSUには、サイズとファンという2つの欠点しかありません。



回路図、プリント回路基板の図面（LUTのバージョンと塗りつぶしのLUTのバージョンがあります）、ソースコード、ATMega8 / 88/168/328のファームウェア、およびWindowsエミュレーターが添付されています。

組立



彼らがそのような場合に言うように- 「しっかりと固定された患者は麻酔を必要としない」 「適切に組み立てられた回路は調整される必要がない」。 ただし、いくつかの組み立て順序を推奨できます。すべての抵抗器とコンデンサ、超小型回路用ソケット、安定器U3、U4を取り付けます。 電源を供給し、ピン7にMK 5V、3.3Vも配置されていることを確認します。 次に、トランジスタ、クォーツを取り付け、LEDストリップを接続し、ボードまたはプログラマーのいずれかでMKをプログラムし、所定の位置に配置します。 電源を入れた後、テープのLEDが赤から緑、青から赤に5秒間すべての色を通ります。 色を変更するプロセスに20秒かかる場合、「内部でクロックを8分割する」というフュージョンが誤ってインストールされている可能性があります。 次に、UART（9600/8データ/パリティなし/ 1ストップ）を介して接続し、インタラクティブモードで色を再生します。 DS1307をインストールし、tsコマンドで時間を設定し、時計が進むかどうかを確認します... 微妙な点があります：DS1307は常に起動するとは限りません。 パニックに陥らず、フラックスを洗い流し、ボードを乾燥させ、クォーツボディを地面に引っ張り、クォーツを交換するか、最悪の場合DS1307自体を試してください。 クロックマイクロチップは非常に感度が高く、実際には微小電流で動作します。わずかな精度ですべてが動作します。
