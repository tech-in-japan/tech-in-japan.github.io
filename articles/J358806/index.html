Red HatとMicrosoftは、Azureプラットフォームで最初の共同ソリューションRed Hat OpenShiftソリューションを開発します。











Red HatとMicrosoftは、エンタープライズアプリケーション開発者に、Microsoft Azureクラウドでコンテナー化されたアプリケーションを実行し、Microsoft Azureパブリッククラウドで最初のRedShareベースのOpenShiftオープンソースソリューションでオンプレミスを実行する機能を提供するパートナーシップの拡大を発表しました。

Azure上のRed Hat OpenShiftは、エンタープライズアプリケーション開発者のプラットフォームメンテナンスの必要性を完全に排除するという事実に加えて、ソリューションは次の利点も提供します。



  • 柔軟性-ハイブリッドクラウド環境用の単一のユニバーサルOpenShiftプラットフォームを使用して、データセンターとAzureクラウド間でアプリケーションを自由に移動できます
  • 速度-AzureプラットフォームとOpenShiftオンプレミスクラスターを統合する高速で安全なハイブリッドネットワーク
  • 生産性-効率的なアプリケーション開発のために、Azure Cosmos DB、Azure Machine Learning、Azure SQL DBなどのAzureサービスにアクセスします

OpenShift自動操作



CoreOSエンタープライズKubernetesソリューションであるTectonicは、いわゆる自動無線アップデートを使用して、大規模なKubernetesインフラストラクチャの高度な管理機能を提供します。これにより、Tectonicクラスターおよび基盤となるContainer Linuxホストのロールアウトアップグレードアップデートが大幅に簡素化されます。 Red Hatは、このテクノロジーをエンタープライズクラスのKubernetesプラットフォームであるRed Hat OpenShift Container Platformに自動操作として組み込む予定です。



自動更新は、おなじみのRed Hat OpenShift Container Platformユーザーエクスペリエンス、サポート、高度なアプリケーション開発機能とともにTectonicを利用します。 これにより、大規模なKubernetes構成の展開が大幅に簡素化され、日常的なタスクの大部分が自動化され、ITプロフェッショナルによる継続的な管理の必要性が減り、完全自動化生産のようなクラスターの運用方法が開かれます 同時に、プラットフォームの安定性や既存のITシステムのサポートなど、企業顧客にとって重要な要素は完全に保存されます。



オペレーターフレームワークイニシアチブ



CoreOSは、Kubernetesユーザー向けに複雑なステートフルアプリケーションのインスタンスを作成、構成、および管理するためにKubernetes APIを拡張する特別なアプリケーションコントローラーである、Operatorと呼ばれるKubernetesの新しいソフトウェアクラスの創設者です。 つまり、演算子を使用すると、Kubernetesアプリケーションの動作がアプリケーション自体にどのように構成されているかに関する「人間の知識」を組み込むことができ、展開と保守が大幅に簡素化されます。



Operatorsのコンセプトは、KubeCon Europe 2018カンファレンスで初めて発表されましたが、今日は、Open Partner Frameworkプロジェクトの一部として実装されています。帽子。 このイニシアチブの目標は、サードパーティベンダーがメッセージング、ビッグデータ、その他の分析サービスなどのハイブリッド環境でクラウドサービスを提供しやすくし、特定のクラウドプラットフォームに縛られることなく、サポートされるエンタープライズ展開モデルの範囲を拡大することです。 さらに、独立系ソフトウェアベンダー向けのRed Hatパートナープログラムも、Operator Frameworkに基づく自動化ソリューションの開発者を含むように拡張されます。 これらの取り組みの結果、Red Hat OpenShiftプラットフォームでのサードパーティサービスの展開と運用のための単一の普遍的なモデルとなり、市場への導入が加速され、Red Hat OpenShiftが運用可能なクラウド環境でそのようなサービスを使用できる可能性が提供されます。

Container Linuxは、コンテナ環境向けの非常に軽量なLinuxディストリビューションであり、「無線」で自動更新されるため、大規模なクラスターでの効率的な運用を実現します。 Container Linuxは、オープンな開発コミュニティを通じて進化し続け、高度な自動化を備えたコンテナホスト向けの無料で急速に進化するOSであり続けると同時に、Red Hat Enterprise LinuxおよびFedoraエコシステムのコンポーネントを使用することもできます。 さらに、企業のお客様には、Red Hat CoreOSと呼ばれる商用のバックアップオプションが提供されます。これは、最終的に、Atomic Hostをコンテナー指向のRed Hatオペレーティングシステムとして置き換えるものです。

Red HatがAMQストリームを導入：OpenShiftのApache Kafka



Apache KafkaをOpenShiftプラットフォームで実行できるようにするRed Hat AMQソリューションの新しい補充であるRed Hat AMQ StreamsのDeveloper Previewバージョンを紹介できることを嬉しく思います。

Kafkaテクノロジーを使用すると、アプリケーションで次のことができます。



  • データストリームの公開と購読
  • データストリームの保存
  • 利用可能になったデータを処理する


Kafkaテクノロジーを使用すると、これらすべてを迅速に実行できるだけでなく、水平スケーリングとフォールトトレランスも提供するため、Webサイトでのアクティビティの追跡、メトリックとログの集約、ストリームとイベントソースの処理、IoTなど、さまざまなシナリオに最適ですテレメトリー。 Red Hatによって開発されたAMQ Streamsは、顧客がRed Hat Enterprise LinuxおよびRed Hat OpenShift Container PlatformでApache Kafkaを実行できるようにします。



AMQ Streamsは、次のようなタスクを大幅に簡素化します。



  • ワンクリックまたはoc createコマンドを使用して、必要なサイズのKafkaクラスターをデプロイします

  • Kafkaトピックパーティションスケーリング
  • 負荷に応じたKafkaクラスターのスケーリング

Red Hatはコンテナーと自動化テクノロジーを使用して、DevOps手法の実装と組み合わせることで、新しいアプリケーションを迅速に開発および展開し、内部インフラストラクチャとサードパーティソリューションの統合を改善することができました。

●Red Hat Enterprise Linuxは、AVIATARの基盤として機能する柔軟で信頼性の高いエンタープライズクラスのLinuxディストリビューションです。

●Azureプラットフォーム上のRed Hat OpenShift ContainerPlatform。DevOps方法論の一部として、AVIATARプラットフォームの継続的な改善をサポートするように最適化されています。 Red Hat OpenShift Container Platformは、Lufthansa Technikインフラストラクチャコンポーネントの開発、デプロイ、起動、統合にも使用されます。

  • Red Hat JBoss Enterprise Application Platform（JBoss EAP）は、AVIATARプラットフォームを使用してエンドユーザーに提供されるエンタープライズアプリケーションシステムのバックエンドコアです。 標準化と使いやすさにより、Javaアプリケーション開発を加速します

Red Hatの新しいミッション：IT環境間の違いを消去する

ポール・コーミエ、レッドハット製品およびテクノロジー社長









近年、Red Hatは、急速に変化する市場環境と顧客のニーズにおける現代の企業のニーズを完全に満たすことができないため、物理サーバー、仮想マシン、プライベートおよびパブリッククラウド環境の4つのIT環境の互換性に特に重点を置いています。 。



これら4つの環境間でリソースと負荷が分散されるハイブリッドクラウドは、今日、デジタルトランスフォーメーションの基盤の1つです。 もう1つは整合性です。ITマネージャーは、エンタープライズアプリケーションとサービスが常にどこでも同じように動作することを保証する必要があります。 これはニュースとは程遠いものです。あなたはすでに私やRed Hatから、そして以前のRed Hat Summitでさえ、すべてを聞いています。



そして今、私は本当に新しいことを言います。エンドユーザーにとって、これらの4つの環境は意味がありません。 ハイブリッドクラウドは企業の標準になりつつあります。組織が取り組む環境や提供者に関係なく、組織が課題に対する解決策を模索しているためです。



私の意見では、今では企業のITシステムにとって、技術スタックの基本的なコンポーネントが自宅の基礎、電気配線、下水システムなどの「ユーティリティ」のカテゴリに入るときの転換点があります。 類推を続けると、家の建設中の材料の選択と行われた仕事の質は確かに重要ですが、日常生活ではこれのほとんどすべてが住宅所有者の目から隠されています。



Red Hatは、コアとなるハードウェア、サービス、外部サービスに関係なく、4つのIT環境すべてで安定性、整合性、信頼性を提供する標準プラットフォームを顧客に提供することを使命と考えています。
