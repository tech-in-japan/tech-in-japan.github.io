HDTVのパフォーマンスを最大にするためのネットブックのセットアップ

こんにちは、ハブロフスク！



ハブに関する主題に関する小さな記事がすでにありました。それは「HDビデオ再生の高速化」と呼ばれていました。 それで提案された方法でさえも助けにならない人のために、私はこのテキストを書きました-とりわけ、字幕の操作の問題を調べました。



私自身は上記の人々に属しています。最初のネットブック（Asus EeePc 701）の1つを使用しているため、プロセッサの公称周波数は600 MHzであり、810を超えると「追いかけません」-BSODになります。 さらに、チップセットにはIntel GMA 900グラフィックスカードが組み込まれており、HDビデオのハードウェアサポートはありません。 更新：2 GBまで購入したRAM。 目標は、ネットブックに次の特性を持つファイルを再生させることでした。

  • mp4コンテナー
  • 解像度1280x720（HDTV 720p）
  • H264 / AVCビデオ
  • サウンドAAC 160-250 Kbps
  • 平均合計ビットレート1.9 Mbps
  • 最大ビデオビットレート14 Mbps




他のHDビデオ（短いブレーキに耐えてさらに詳しく見る）とは異なり、私のタスクにはさらに困難があります。

1.字幕を表示する必要性。 同様のネットブックなどの限られたリソースでは、HDTVを「限界で」デコードするときに「標準」の方法で（VobSubを介して）字幕を出力すると、繊細な追加の負荷がかかります

2.出力字幕は、ビデオと同期していない状態で逃げてはなりません。



上記の特性が示されているテストファイルを取得し、コーデック、プレーヤー、および設定のさまざまな組み合わせを試し始めました。 オプションを並べ替えるのに約3日かかりました。 うまくいかなかったものすべてについて話すと、記事は数倍大きくなります:)したがって、前述の「HDビデオ再生の高速化」というテキストで示した例に限定します。



まず、テストビデオクリップの選択について。 これは通常のアニメ（映画と同じ解像度と品質で低ビットレートが特徴）の一部ですが、完全に不cru慎なスプラッシュスクリーンを使用します。各フレームは非常に詳細な画像で、細部が豊富で、これらの種類の画像が次々に表示されます。 「カメラ」の動きに加えて。 「ノイズの多い」画像の効果が判明します。これにより、デコード時に大きな負荷がかかります。 サウンドと字幕をビデオトラックと同期させずにコンピューターにこのフラグメントをスクロールさせると、指定されたビットレート（最大10 Mbps-保証付き）内のすべてのアニメでこのメソッドが機能することがわかりました。 ほとんどの場合、14 Mbpsの指定された最大ビデオビットレートは、このスクリーンセーバーに相当するか、最小で最も重い部分に相当します。



とりわけ、「すぐに使える」VobSubは字幕を表示したくなかったので、新しい変数を追加せずに、プレーヤーが提供する字幕を処理することにしました。 最初に「go」、次に「checkers」の面倒を見ることができます-VobSubのセットアップは次の「研究」のトピックになります:)



だから、仕事の結果：



Media Player Classic（HomeCinema mod）のオプションはすべて失敗しました。 破れたビデオ-組み込みのH264 / AVCデコーダーを使用し、ffdshowからffmpeg-mtを使用する場合、およびDivX h264 / AVCコーデックを使用する場合。 その結果、ビデオの速度が低下し、音が正常になります-音が前方に流れます。 フレームのスキップ、プレーヤーの優先度の変更など、さまざまな設定が試行されました。 ティックについては、コーデック設定とプレーヤーの両方で、ポストプロセッシングが（essno！）無効になったことに言及します。プレーヤーに加えて、リソースを消費するタスクは開始されませんでした。



CoreAVC 2.0をインストールすると状況は劇的に変化しました。はい、有料のコーデックなので、リンクを提供せず、読者にそれを入手する方法を任せます。 現在、彼のWebサイトの価格は10ドル以下です。これは、私のような弱いハードウェアでHDTVビデオを視聴する唯一の機会であるため、安価です。 そのため、ビデオは非常に滑らかになりました-少なくとも鋭い動きはありません。 実際、これは問題によって部分的にしか解決されていません-時々、ビデオの再生が著しく遅くなり、字幕が完全に予測不能に動作し、タイミングが絶えず失われます。 それはテキストをサウンド/ビデオに適合させているようです...そして小さなビットレートで高ビットレートで-そして再び非同期です。



そして、ストーリーの3番目の部分に移ります。一人のプレイヤーが問題を解決します。 これはよく知られているCrystal Playerであり、数年前に私が個人的に「ダウン」しました。 再び使用することはあまり望んでいませんでした-MPC HomeCinemaには誰もが満足するでしょう...しかし、Crystal Playerには重要な機会がありました-設定されたバッファサイズに従って、「これから」ビデオストリームをバッファにデコード 私は他のプレーヤーでそのような機会を見つけていません。 私の良心をクリアするために、最も人気のあるプレーヤー（BSPlayer、VLC、smplayerなど）のみを視聴し、それらのすべてがインストールされているわけではないと言わなければなりません。 あなたが突然他の誰かとそのような機会を見つけた場合-私に書いてください！ そのため、名目上、設定は「フレームバッファサイズ」と呼ばれ、最大値は「50％メモリ」です。



なぜ名目上？ 実際、この値を「メモリの10％」に設定すると、プレーヤーは最大93フレームをキャッシュします。 「メモリの50％」を設定すると、キャッシュされたフレームの数は（93x5 = 465フレーム？はげたわさび！）-97に増加します。TaskManagerは、システムによる合計180 MBプレーヤーと800 MBのメモリ消費を示します。 別の1.2 GBはアイドル状態です。 バッファーの下でプレーヤーに1.2 GBをすべて与えたので、24秒前にムービーをデコードする機会を得ることができます。これは、最もダイナミックなシーンに十分であるでしょう。



しかし、そうではないのはそうではありません。 いずれにせよ、97のプリコーディングされたフレームがビデオストリームを「スムーズ」にし、バッファをスムーズに（ただし、:(）は動的シーンの下で空になり、比較的静的なシーンでいっぱいになります。未解決の問題が1つあります。字幕。プレーヤーの設定「速度の問題を解決する」 >> 「キーフレームでスキップする」これはまさにこのオプションであり、他には何もありません。すべてのフレームがビットレートの高いセグメントのキーであるように感じるため、「フレームをスキップする」だけではほとんど結果が出ません。



だから、私は要約します。 適切なタイミングで目的のコーデックをオンにするためのOS設定を言うことはできません-誰もが自分のマシンでこの問題を個別に決定します。



CoreAVC設定：



「入力レベル」 、 「出力レベル」 、 「入力カラースペース 」はそれぞれ「TV」 、 「PC」 、 「自動検出」に設定されています。



ガイドされるガイドはここにあります -Nvidia CUDA Accelerationを使用するためのCoreAVCセットアップをセットアップします。 入力レベルでは「TV」はどのような状況でもあり、出力では「TV」はMPC HomeCinemaを使用する場合に設定する必要があると記載されています。 この場合、 「デコードチェーンの初期段階ではなく、MPC-HCがB＆Wレベルを適切なレベルに拡張できるようにします。」© 、MPC：HCでは、 「レンダリング設定」 >> 「出力範囲」を設定する必要があります > > "16-235" 。 このアイテムはMPCでは使用できません。これがないと、写真の「ジューシーさ」が著しく失われ、Crystalではそのような設定はまったくありません。したがって、 「出力レベル」-「PC」を選択します。



インターレース解除：なし（織り）

次に、Crystal Playerの設定。



まず、ビデオをCoreAVCで開きます。「 プレーヤーを右クリック」 >> 「プロパティ」でチェックし、ビデオストリームコンプレッサーはme [AVC1] CoreAVCビデオデコーダーで指定します。 私はオーディオに焦点を当てていませんでした、それはまったく決定されていませんが、動作します:)



したがって、主なガイドラインは、同じ右ボタンまたはホットキー「S」によって呼び出される統計ウィンドウである必要があります。



一番上の行では、 「1194x679 Overlay Flipping YV12」のようなものが望ましいはずです。最も重要なことは、 YV12が可能な限り高速なモードであることです。 YUY2も有効です。

大きなテキストボックスには、「コメントなし、最速モード」が必要です 。 そうでない場合は、このフィールドの指示に従い、設定で不要なものをすべて無効にします。 ここでは、ビデオバッファとオーディオバッファの現在の状態を示すフィールドも表示できます。これを常に使用して、ネットブックに現在のビットレートを「消化」する時間があるかどうかを判断します。



[ パーサー ]メニューの[ データの読み取り ]メニューで、最大データブロックを設定します。 残りの設定は特に重要ではありません。主なことは、設定を変更するときに常識を適用することです。



上記のすべてのアクションの結果：

  • Celeron M 0.8 GHzはブレーキなしで720p HDTVを最大10 Mbpsまで回転させます
  • CoreAVC画像の品質は、他のデコーダーよりも著しく劣ります（後処理を有効にした状態でチェックされます！）が、これはビデオ自体の解像度によって「わずかに」補正されます。一般的に、このハードウェアでは、 habrausersの1つであるスクリーンショットのコメントに明確に示されているCoreAVCは、後処理を有効にして、他のデコーダーと区別できない品質を提供します。 私はこの記事を「歴史のために」という記事に残します
  • 「遊泳」ビットレートは、Crystal Playerのバッファーによってわずかに補正されます
  • ビットレートを「消化できるよりも著しく高い」状態に上げると、フレームドロップが発生します。これは、わずかにちらつき、ビデオが少し破れ、ビデオがわずかに遅くなり、音が途切れて同期し始めます。 私はこの問題を解決しなかったし、これがどのように可能かについてのオプションを見ていません-ビデオの速度が変わったので、それに合わせて音を調整しなければなりません。 しかし、これは非常に大きなビットレートであり、鉄はそれ以上引き出せません！
  • これらすべての不明瞭さを伴う字幕は消えず 、適切なタイミングで画面に表示されます。 Alt + K、Alt + Lは、100ミリ秒単位で時間の字幕オフセットを調整します。これは非常に便利です。
