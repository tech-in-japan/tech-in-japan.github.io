モノのインターネットのアーキテクチャ標準

先日、どのIoTアーキテクチャが典型的かについての情報が必要でした（参照）。 このような情報は、「ハブ」または他のリソースですぐに見つけることができませんでした。 この方向で標準を開発する最初の試みはわずか2年前に行われ、作業はまだ「プロジェクト」段階にあることが判明しました。 この記事では、IoTリファレンスアーキテクチャのIEEEおよびISO / IEC標準の開発状況の「スライス」を紹介します。

なぜ標準が必要なのですか？



標準化の全体論争のトピックから始めましょう。

世界の技術の進歩は次の方向に向かっています。技術が十分に成熟したら、標準化する時です。これは、典型的なコンポーネントとインターフェースを備えたいわゆる「オープンシステム」への一歩です。 よく知られているGartner Hype Cycleを考慮すると、テクノロジーの大量実装の準備が整うずっと前に、標準化を積極的に実行できます。



技術的な（そしてそれだけではない）製品の全世界は標準によって決定されており、それなしではカオスは私たちすべてをカバーしていたでしょう。



IoTに関しては、セキュリティの重要なアプリケーションの分野（安全クリティカル、セキュリティクリティカル、ミッションクリティカルなど）を含む、大規模な州および企業の顧客と連携する場合、標準の使用も重要なプラットフォームです。

「美化」の標準化は、「パワーのダークサイド」について語るしかありません。



-過剰な数の標準の存在、明確なシステムの欠如、同じ分野の標準化に関与するさまざまな組織の存在。

-標準化オブジェクトの不完全で不均一なカバレッジ。

-「標準の戦争」、すなわち 標準化の一般的な問題を解決する上で、個々の企業の利益にロビー活動を行います。

-主に最も単純なオブジェクトと大量プロセスのみの規制。

-官僚的な手続きと標準の長い開発期間（平均で3〜5年）。これにより、保守主義と実用的なニーズに遅れが生じます。

それにもかかわらず、標準の世界は存在し、私たちはそれに生きています。



IEEE vs ISO / IEC



IEEE（電気電子技術者協会）とIEC（国際電気標準会議-国際電気標準会議、IEC）の2つの一般的に認められている組織があるため、標準の世界に直面している人は、IT分野ではこの世界が双極であることを知っています。

標準化のさまざまな分野では、IEEEとIECの相互の影響は異なりますが、「平均して」（例外があります）アメリカはIEEEを愛し、ヨーロッパはIECを愛しています。



一部の分野では、IECはISOと協力しています。たとえば、私たちが関心を持っている分野は、ISO / IEC JTC1（国際通信技術共同技術委員会）に参加しているIoTです。



もちろん、EN（ヨーロッパ標準）、ITU-T（国際電気通信連合電気通信標準化部門）、アメリカNIST（国立標準技術研究所）などを開発しているCENELECなど、他の国際標準化機関もあります。



IEEEがIoTで行うこと



IoTは、IEEEとISO / IEC JTC1の範囲にほぼ同時に入りました。 2014年7月、 IEEE P2413ワーキンググループが作成され、「モノのインターネット（IoT）のアーキテクチャフレームワークの標準」を開発するための最初の会議が開催されました。 ワーキンググループには、Cisco、Emerson、Hitachi、Honeywell、Huawei、Intel、Kaspersky Lab、Rockwell Automation、Schneider Electric、Siemens、STMicroelectronics、東芝、横河などの代表者が含まれていました。 マイクロソフトとグーグルは見られませんでした。



2014〜2016年 集会は年に4回開催されます。 グループのウェブサイトでの開発ステータスの詳細情報は提供されません。高レベルのプレゼンテーションのみがあります。



興味深い事実は、信頼性とセキュリティ（ディペンダビリティとセキュリティ）を確保するという問題に対するIEEE P2413ワーキンググループの注意深い態度です。





出所



したがって、IoTは、安全性にとって重要な（安全性が重要、セキュリティが重要、ミッションが重要など）既に述べたシステムの分野での使用が計画されています。 これは、産業オートメーションの標準「モンスター」の開発への参加によっても確認されます。



IoTでISO​​ / IEC JTC1が行うこと



「モノのインターネットに関するワーキンググループ（WG10）」ワーキンググループを作成する決定は、2014 ISO / IEC JTC1全体会議で行われました。 2015年1月にWG10の最初の会議が開催され、標準のISO / IEC 30141「モノのインターネット参照アーキテクチャ（IoT-RA）」の開発が決定され、それ以来、年に3回開催されました。 ISO / IEC JTC1ページには、初期IoTレポート（IoTリファレンスアーキテクチャ/フレームワークに関する調査レポート）と「IoT on WG10」の結果の一部が含まれています。 言及されたレポートについて詳しく見てみましょう。

わずかな事実から何が理解できますか？



まず、将来のIoT-RA規格の多くの規定は、次のような関連するすでに標準化された分野から借用されています。



-家庭用電子システム（ISO / IEC 14543シリーズの規格に記載されている家庭用電子システム）;

-メディア管理用のMPEG-Vアーキテクチャ（ISO / IEC 23005シリーズの規格に記載されているメディアコンテキストと制御）。

-センサーネットワークリファレンスアーキテクチャ（ISO / IEC 29182シリーズの規格で説明されているSNRA）。

最新のアーキテクチャに関しては、IoTアーキテクチャの標準プロトタイプであり、デバイス、ネットワーク、サービス、アプリケーションの4つの典型的なレベルを提供していることに注意する必要があります。



第二に、IoTのアーキテクチャ記述は、2012年以降にすでに存在しているITU-T（国際電気通信連合電気通信標準化セクター）標準、特にITU-T Y.2060「モノのインターネットの概要」から適応されます。

第三に、IoTコンポーネントの開発要件は、ITU-T Y.2066「モノのインターネットの共通要件」で提案されている分類法に従って構成されます。これには次の要件グループが含まれます。



-実装および操作性の要件。

-非機能要件

-アプリケーションサポート要件。

-サービス要件。

-通信要件。

-デバイス要件;

-データ管理要件。

-セキュリティおよびプライバシー保護の要件。



おわりに



近い将来、IoTの参照アーキテクチャ（参照アーキテクチャ）を記述する標準のリリースが期待されています。 この事実は、典型的な互換性のあるコンポーネントとインターフェースを備えたIoTを「オープンシステム」にさらに近づけます。



2014年以降、このような標準の開発は、 IEEEワーキンググループP2413およびISO / IECの「モノのインターネットに関するワーキンググループ（WG10）」と並行して行われています。 2つの標準があると、コンポーネントのサプライヤとシステムインテグレータに頭痛の種が追加されます。



これらの開発に関するオープンな情報はあまりありません。 既存の開発が、特にITU-Tから標準化されることは明らかです。これにより、リファレンスアーキテクチャには、デバイスレイヤー、ネットワークレイヤー、サービスレイヤー、アプリケーションレイヤーの4つの典型的なレベルが含まれます。
