Why NOT Use Svelte?



  1. If your project is already written on another framework and works reasonably well. No need to run and rewrite it.
  2. Even in point 1 there is a window of opportunity - you can rewrite some of the "heavy" components on Svelte. The project will become easier, and the components will work faster. You don‚Äôt have to rewrite everything.
  3. If you are writing an enterprise, Angular or Ember is probably the best choice. Svelte, just like React and Vue, is not always a good choice for this.
  4. If if you need first-class Typescript support.
  5. If you develop exclusively standard solutions and are used to assembling projects from off-the-shelf components.
