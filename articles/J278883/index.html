ITリソースの傘監視

現代のビジネスはITに依存しています。 ただし、現時点では、ITは個々のITリソースまたはITプロセスをサポートするだけでなく、少なくともITサービスの提供を保証し、ビジネスプロセスのITサポートの品質を最大化します。 IT管​​理への統合アプローチを必要とするのは、このようなサービスモデルです。 そのため、顧客は、単一の「コントロールパネル」ですべての情報システムとすべての機器をカバーする、一元化された傘監視への異種の使用にますます移行しています。







傘の監視により、企業はITの真の有効性と、企業全体のパフォーマンスに対する直接的な影響を特定できます。 これは、ITインフラストラクチャおよび情報システムのすべてのソースから情報を収集し、サービスリソースモデルを構築し、主要業績評価指標をこれらのモデルにリンクし、これらのモデルをほぼリアルタイムモードで視覚化することによって達成されます。 さまざまな監視ツールを使用する企業や組織は、ITインフラストラクチャの単一のアイデアを持っていないため、イベントがITサービスの品質に与える影響を把握していません。 Umbrellaモニタリングを使用すると、すべてを接続し、単一のコンソールから管理を整理できます。



ビジネス上の利点に関しては、傘監視システムの導入により、まず、機器および情報システムの動作の誤動作につながる可能性のある問題を特定する時間を短縮できます。 システム管理者は、さまざまな監視システムの複数の開いているウィンドウ間を移動する必要はありません。いくつかのスケジュールを削減し、特定の法律と関係を視覚的に評価しようとします。 アンブレラモニタリングソリューション自体は、このような分析情報を提供するだけでなく、ハードウェアおよびソフトウェアの動作におけるインシデントの根本原因を通知します。



さらに、統合された情報の提供により、リクエストが管理者から管理者へ、部門から部門へリダイレクトされることを回避できます。 このように、ITスタッフの作業時間の70〜90％が既存のインフラストラクチャのサポートに費やされているため、ビジネスはITスペシャリストを日常業務からより重要なタスクの解決に再配置できます。



会社のIT資産を包括的に監視するための最も開発された統合ソリューションの1つは、長年市場に出回っていてさまざまなオペレーティングバージョンを備えた「古典的な」HPE Operation Manager監視システムの開発における画期的なマイルストーンである、Hewlett Packard EnterpriseのHPE Operations Bridgeと呼ばれますシステム。 長年の人気と膨大な数の顧客にもかかわらず、HPE Operation Managerは、たとえばJavaコンソールなどのすでに時代遅れのテクノロジーを使用しています。 また、HPE Operations BridgeイベントコンソールであるHPE Operations Manager Iは、HTML5テクノロジー、新しいエージェント管理方法、データ視覚化への新しいアプローチなどに基づいています。 最も顕著な革新の1つは、ビッグデータのサポートです。 したがって、HPE Operation Managerの方法は、技術的に改善されただけでなく、外部システムと対話するための統合機能も強化されています。 HPE Operation Managerのユーザーは、新しいコンポーネントをインストールし、機能を徐々に新しい製品に移行することで、開発者の技術サポートをシームレスに使用してHPE Operations Bridgeに切り替えることができることに注意することが重要です。



HPE Operations Manager Iイベントコンソールに加えて、HPE Operations Bridgeには、Monitoring Automation多層アプリケーション監視サブシステム、Service Health Reporter履歴情報およびレポートシステム、TBECトポロジイベント相関メカニズム、HPEデータ収集エージェントなどの基本コンポーネントが含まれていますOperations AgentおよびHPE SiteScope Agentless Monitoring System。 以前にHPEが情報を収集してIT資産を監視するための多数の異なる製品を所有していた場合、今日それらは包括的な統合ソリューションとして提供されています。



HPE Operations Bridgeの機能により、機器のステータスを監視できるだけでなく、実際のユーザーの観点から、システムとアプリケーションの両方で使用されるアプリケーションのパフォーマンスと操作性を分析することもできます。 監視はマルチレベルモデルに基づいており、最上位レベルでは、ITがユーザーに提供するサービスの状態を反映するサービスリソースモデルが形成されます。



HPE Operations Bridgeの中心となるのは、モジュラーアーキテクチャとコレクターです。さまざまなシステムから情報を収集するデータコレクターです。 たとえば、クラウドオプティマイザーコレクターは、仮想インフラストラクチャの動作に関する情報を取得するために使用されます。 ストレージ操作コレクターは、ストレージシステムの操作に関するデータを収集します。 Network Node Manager Iコレクターは、ネットワークの動作に関する情報を取得する役割を果たし、実際には、コレクターコレクターから受信したデータのフローに基づいて、サービスリソースモデルが形成されます。 すべてのコレクターはオプションです。 必要に応じて、顧客は、サードパーティの製造元を含む外部監視システムをHPE Operations Bridgeに接続できます。この目的のために、開発者は多くのアダプターを提供します。 この場合、このシステムから取得した情報は、サービスリソースモデルの構築にも関与します。 サポートされているサードパーティシステムには、IBM、CA、BMC Software製品、オープンソースアプリケーションNagiosおよびZabbixがあります。 サポートされていないシステムを統合するには、BSMコネクタを使用できます。 HP Operations Bridgeテクニカルサポート契約を結んでいるお客様は、これらの統合サービスを無料で利用できます。



HPE Operations Bridgeはお客様のITインフラストラクチャにインストールされますが、お客様はロシアの多数のHewlett Packard Enterpriseパートナーが提供するクラウドベースのIT資産監視サービスまたはIT監視SaaSサービスを利用できます。



HPE Operations Bridgeには3つのエディションがあります。 Junior Expressは、HP Operations Manager IコンソールといくつかのデータコレクターであるPremiumエディションのインストールを提供し、ビッグデータに基づいてデータベースにメトリックと統計を蓄積し、Operation Bridge Reporterコンポーネントを使用して分析レポートに基づいて構築し、サービスを相関させる機能も提供しますリソースモデル。 Ultimate Editionには、上記のすべてに加えて、HPE Operations Orchestrator ITインフラストラクチャ管理システムとBusiness Value Dashboardビジネス視覚化システムが含まれています。
