Firefoxの高度なロケーションバー

Firefoxは、拡張機能の作成者がインターフェイスの任意の部分を変更して最も大胆なアイデアを実装できるため、その拡張性で有名です。 しかし、このような柔軟性には重大な欠点があります。不注意な操作を1回行うと、ブラウザーが簡単に破損する可能性があります。



Locationbar²はこれを完全に実証しています。 この拡張機能は、標準のアドレスバーにURLの目的のセグメントをクリックしてサイトの構造を数レベル上に移動する機能を追加します。この「マジック」動作は、接続タイプアイコンの左から右にマウスカーソルを移動した場合にのみアクティブになります（これを考慮することができます）ジェスチャとして）またはCtrl / Alt / Shiftを押したとき。 ただし、この便利な変更に加えて、Locationbar²はIDNドメインの操作性を損ないます。IDNドメインはエンコードされた形式で表示され、サブドメインは完全に破棄されます。 この拡張機能は2011年以降確定しておらず、やがてFirefoxの新しいバージョンとのわずかな競合があり、拡張機能の作成者に連絡していません。



分岐する時が来ました： Advanced Locationbarに会いましょう！



どうだった？



実際、私は2012年にIDNの問題を修正しましたが、それ以降、さらなる改善が長い箱の中のどこかにありました。 この期間中、この拡張機能の新しいフォークは、Locationbar³という名前で表示されました。名前を見ないで、Locationbar²のややライトなバージョンで、Firefoxの新しいバージョンと競合するオプションの機能を削除しました。 ただし、IDNの問題やその他の欠点は解決されていません。 このため、改良を軽量コードと組み合わせて、さらに作業を始めました。 確かに、これがオープンソースの力です！



どうしたの？



  • IDNサポートが追加され、このようなドメインは正しく表示され、サブドメインはトリミングされません
  • これで、ドメインまたはサブドメインに個別に切り替えることができます（たとえば、blog.mozilla.orgからmozilla.orgに直接移動できます）
  • カーソルがほんの少しの間アドレスバーから離れると、セグメント選択モードはリセットされないため、マウスカーソルをあまり慎重に移動することはできません。
  • Ctrl / Alt / Shiftを押すか、アイコンからマウスカーソルを移動してリンクに変換することをオフにすることができます
  • オプションは標準の拡張マネージャーに統合されています
  • 省略記号は不要な場合は表示されません（Locationbar²のエラー）
  • Firefox 32以降のサポートの改善（いくつかのバグ修正を含むLocationbar³に基づく）
  • 新しいアイコンと名前:)
  • BitBucketで利用可能なプロジェクトリポジトリ




次は？



私はすでにFirefoxでこのような快適なことに慣れています。他のブラウザではそのようなインターフェイスを邪魔することができないため、ブラウザの変更に耐えることができません。 そのため、この拡張機能を最新の状態に維持する必要があります:)あらゆる場面でこの拡張機能を組み合わせることはありません。 将来的には、ユーザーがセグメントを選択するときに、いくつかの小さな快適な機能とよりインテリジェントなジェスチャー処理のみが計画されます。



この拡張機能があなたの役に立つことを嬉しく思います。 ご清聴ありがとうございました。
