5年間、5つの教育プロジェクト：教師の主なものと歴史について簡単に





最近、MSTUと一緒に組織した最初の教育プロジェクトであるMail.Ru Technoparkによって 5年が経過しました。 N.E. バウマン。 彼の仕事の間に、プロジェクトは、 テクノスフィア Mail.Ru （VMKモスクワ州立大学）、 テクノトレックMail.Ru （MIPT）、 テクノポリスMail.Ru （SPbPU）、 テクノアトムMail.Ru （MEPhI）など、国内の他の最も強力な技術大学でも開始されました。 伝統的に1月25日に祝われていた学生の日を記念して、IT教育が始まった経緯、教育プロジェクトがどのように現れ、発展したか、今日の様子、そして提携大学のおかげで、これはどれもありません。 テクノパークは最初のものであり、数年ぶりのものでした。 ソリューションとアプローチが完成したのはテクノパークであったため、後で他のプロジェクトの基礎を形成したため、これについて詳しく説明します。 そして、先生に床を渡します。



テクノパーク



2011年11月、「先駆的」な教育プロジェクトの立ち上げを発表しました。 12月、最初の学生グループ（42人）が募集され、1月末までに、Mail.Ruグループのスタッフから教師チームが結成され、トレーニングプログラムが承認されました。 学生の3段階の選択が開発されました：最初に、オンラインテスト、次にフルタイムの専門面接と人事面接。 6か月後、最初のセッションが開催され、42人中31人が無事に終了し、2012年9月までにテクノパークの学生数は91人に増加しました。 確かに、そのうち56人のみがセッションを終了しました。



その1年後の2013年5月に、テクノパークはRAECによると最高の教育プロジェクトのリストに入りました。 9月までに、1つの場所の競争は5〜7人でした。 花崗岩科学は168人の学生によってかじられました。 16の分野で構成される新しいトレーニングプログラムが作成され、24人の教師が知識を共有しました。 同じ年の11月に、ノミネート「科学と教育」でRunet賞を受賞しました。 2年後、テクノパークで最初の卒業式が行われました。 最初のセット（42人）のうち、17人がフィニッシュラインに達し、そのうち14人の卒業生がMail.Ru Groupに勤務しました。 その時までに、テクノパークの講堂の223人の学生が19の分野で講義に参加しました。







次のフロンティアは3年間のテクノパークです。 222人の学生が訓練を受け、22の分野で講義を受けました。 この時点で、40人がすでにMail.Ru Groupでのインターンシップを完了し、採用されました。 MSTUのすべての学生向け。 バウマンの「テクノスタート」チャンピオンシップが開催されました。その中で、学生の生活を楽にするモバイルWebサービスのコンセプトを作成する必要がありました。 250人がチャンピオンシップに参加し、46のプロジェクトを発表しました。



テクノスフィア



2014年2月に、2番目の教育プロジェクトとしてテクノスフィアを立ち上げました。これはモスクワ州立大学の壁の中で展開されました。 ロモノソフ。 多くの点でテクノパークで得た経験を考慮しましたが、テクノスフィアは盲目的なコピーではありませんでした。



  • Technoparkプログラムは、負荷の高いシステムのWeb開発者およびアーキテクトをトレーニングするように設計されており、テクノスフィアではビッグデータの分野の専門家をトレーニングします。
  • Technosphereでは、2〜4コースの学生を受け入れます。
  • すべての教育プロジェクトの入学試験のスキームは同じであるという事実にもかかわらず（学生はオンラインテストを受験し、面接に合格します）、テクノスフィアでは高等数学の基礎知識に重点を置いています。
  • テクノスフィアでは、生徒と教師の間の非公式のミーティングが良い伝統になりました。そこでは、心から話し、興味のあるトピックについて尋ねることができます。 これらの会議をPergolasと呼びました。


テクノスフィアへの最初の募集では、コンテストは1会場あたり3人で、1年後には5人でした。初めて42人を募集しましたが、2学期に行くことができたのは10人だけでした。







当初、トレーニングプログラムは1年間設計され、6つの分野で構成されていました：大量のデータの知的処理のためのアルゴリズム、インターネットアプリケーションの開発でDBMSを使用する方法、C ++でのマルチスレッドプログラミング、テスト、情報検索、Hadoop。 しかし、1年後、プログラムを改訂し、隔年にした。 4学期の間、学生は10の分野を研究し、大量の実習を行います。 同時に、準備コース「アルゴリズムとデータ構造」が開発されました。



また、学生がMail.Ruグループで直面している実際のタスクを扱うラボを作成しました。 たとえば、分析アルゴリズムを改善し、特定のヒューリスティックを作成しようとします。 つまり、彼らは会社との定期的なインターンシップ中に行うのと同じことを行います。 2015年の秋以来、研究室は研究を実施し始めました。 たとえば、ニューラルネットワークを使用してさまざまなビジネス上の問題を解決する可能性が研究されており、さまざまなアルゴリズムの実装が作成されています。



テクノトラック



2015年9月、MIPTで3番目の教育プロジェクトTechnotrekが開始されました。 今回は実験をせず、すぐに12分野の2年間のプログラムを開発しました。 しかし、選択システムは2つの段階で作成されました。最初はオンラインテストで、その後は技術面接です。 テクノパークやテクノスフィアとは異なり、すべてのコースの学生はすぐにテクノトレックのコースを受講し始めました。



最初の2つのプロジェクトと同様に、Technotrekには独自の専門分野であるモバイル開発があります。







2016年の春に、入学テストに合格した他のすべてのMIPT学生向けに、「C ++プログラミング」と「iOSアプリケーション開発」の2つのオープンコースを開始しました。 現在、Technotrackには、「Linuxシステム管理」、「高負荷プロジェクトのDBMS」、「産業用プログラミングおよびデータ構造の紹介」、「Androidアプリケーション開発」の4つのオープンコースがあります。

非表示のテキスト
今日のセミナーでは、次の問題がありました。 



CREATE TABLE IF NOT EXISTS `mydb`.`forums` ( `id` INT UNSIGNED NOT NULL AUTO_INCREMENT, `name` VARCHAR(45) NOT NULL, `users_id` INT NOT NULL DEFAULT 0, PRIMARY KEY (`id`), INDEX `fk_forums_users1_idx` (`users_id` ASC), CONSTRAINT `fk_forums_users1` FOREIGN KEY (`users_id`) REFERENCES `mydb`.`users` (`id`) ON DELETE SET NULL ON UPDATE SET NULL) ENGINE = InnoDB; ERROR 1215 (HY000): Cannot add foreign key constraint


それに対処するには、以下を参照する必要があります。 



 mysql> SHOW ENGINE INNODB STATUS; ------------------------ LATEST FOREIGN KEY ERROR ------------------------ 2014-02-26 21:39:44 690 Error in foreign key constraint of table mydb/forums: FOREIGN KEY (`users_id`) REFERENCES `mydb`.`users` (`id`) ON DELETE SET NULL ON UPDATE SET NULL) ENGINE = InnoDB: You have defined a SET NULL condition though some of the columns are defined as NOT NULL.


実際、NOT NULLに対してSET NULLを実行することはできません。



NOT NULLがDEFAULT 0を設定し、イベントをNO ACTIONに変換するため、問題を突然解決しようとします。 



 CREATE TABLE IF NOT EXISTS `mydb`.`forums` ( `id` INT UNSIGNED NOT NULL AUTO_INCREMENT, `name` VARCHAR(45) NOT NULL, `users_id` INT NOT NULL DEFAULT 0, PRIMARY KEY (`id`), INDEX `fk_forums_users1_idx` (`users_id` ASC), CONSTRAINT `fk_forums_users1` FOREIGN KEY (`users_id`) REFERENCES `mydb`.`users` (`id`) ON DELETE NO ACTION ON UPDATE NO ACTION) ENGINE = InnoDB; ERROR 1215 (HY000): Cannot add foreign key constraint


また？ 今何？ 



 mysql> SHOW ENGINE INNODB STATUS; ------------------------ LATEST FOREIGN KEY ERROR ------------------------ 2014-02-26 21:46:06 690 Error in foreign key constraint of table mydb/forums: FOREIGN KEY (`users_id`) REFERENCES `mydb`.`users` (`id`) ON DELETE NO ACTION ON UPDATE NO ACTION) ENGINE = InnoDB: Cannot find an index in the referenced table where the referenced columns appear as the first columns, or column types in the table and the referenced table do not match for constraint. Note that the internal storage type of ENUM and SET changed in tables created with >= InnoDB-4.1.12, and such columns in old tables cannot be referenced by such columns in new tables. See http://dev.mysql.com/doc/refman/5.6/en/innodb-foreign-key-constraints.html for correct foreign key definition.


まあ、確かに！



user_id定義は、usersテーブルのidとは異なります 



 `users_id` INT NOT NULL DEFAULT 0


そして 



 `id` INT UNSIGNED NOT NULL AUTO_INCREMENT


すべてを揃えましょう： 



 CREATE TABLE IF NOT EXISTS `mydb`.`forums` ( `id` INT UNSIGNED NOT NULL AUTO_INCREMENT, `name` VARCHAR(45) NOT NULL, `users_id` INT UNSIGNED NOT NULL, PRIMARY KEY (`id`), INDEX `fk_forums_users1_idx` (`users_id` ASC), CONSTRAINT `fk_forums_users1` FOREIGN KEY (`users_id`) REFERENCES `mydb`.`users` (`id`) ON DELETE NO ACTION ON UPDATE NO ACTION) ENGINE = InnoDB;
