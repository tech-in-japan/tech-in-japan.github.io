開発ストーリー：Crystalux-Android向けのパズルの開発と宣伝のニュアンス

こんにちは、Habr！



1か月前、私の新しいゲームCrystaluxがGoogle Playでリリースされました。 この重要な出来事にまだ満足しているので、開発中に学んだ技術とマーケティングの微妙さを共有します。 まあ、もちろん、私の作成のための少しのPR、もちろん。







ゲームについて簡単に説明します。



Crystaluxはレベルチェーンパズルです。 ゲームプレイはユニークであると主張していませんが、非常に優れており、引き締めることができます。 六角形チップ上にある結晶のパターンを収集して、結晶が色で接続されるようにする必要があります。 詳細に説明する意味はありません。動画を見るか試してみるのが簡単です。



私は、エネルギー（支払いまたは待機）や各ギャップから課せられた社会性などの新しい「ゲーム」トレンドに否定的な態度をとるのは私だけではないと思います。 私が一人だと叫ぶ人たちをホールから出てください 私たちの多くがいることを知っています。 私の友人と私は小さくて貧しいですが、誇り高いインディーズなので、ゲームにはエネルギーがありません。また、有料の障壁や他のものはありません。 しかし、脳が本当に沸騰するレベルがあり、従来の広告、広告の無効化、ゲームのヒントの販売による収益化がまだあります。 社会性は最低限であり、ゲームが何らかの形で進歩するためだけです。 シンプルさとゲームへの没入を主に強調しています。

技術的なポイント



1）ゲームはAndEngineで作成されており、これは重要なミスの1つです。 いいえ、エンジンは悪くなく、学ぶことは非常に簡単ですが、現代のモバイル市場はクロスプラットフォームが必須であることを明確に示しています。 私たちがプロジェクトを始めたとき、私はそれについて考えませんでした、そして今、他のプラットフォームで実行するために、ゲームは書き直される必要があります。 iOSからAndroidに移植するための補助ツール（hello、 Apportable 、Marmalade Juice）がありますが、逆のプロセスのサービス/フレームワークは知りません。 ああ。 次のプロジェクトはUnityまたはCocos2d / Apportableのいずれかになります



2）ゲーム-最も貪欲なタイプのモバイルアプリケーションの1つであり、ゲームを開発するときにこれに遭遇する可能性があります。 ゲームの最初のバージョンでのユーザーの最も一般的な不満は、デバイスの加熱とバッテリーの消費でした。 プレイヤーの1人のリコールに特に感銘を受けました。そこで彼はゲームのメリットを描き、新しいレベルを要求し、最後に次のように説明しました。 コメントは冗長です。

加熱の問題は、さまざまな理由で発生する可能性があります-エンジンのアーキテクチャーの失敗からコード内の重い操作まで、普遍的なアドバイスをすることは困難です。 このゲームは動的ではなく、最大のリターンを必要としないため、SpriteBatchに切り替えて（最初はそれらを使用しなくてもかまいませんでした）、FPS制限を適用することで問題を部分的に解決しました。



3）面白いバグがAndEngineのSpriteBatchに関連しているため、最初は使用しませんでした。 XとYにscale = 1（つまり、元のサイズ）で画像を配置した場合、画像の座標を表示すると、何らかの理由でそれらがランダムに歪んで混乱することが判明しました。 このようなもの：







私はいくつかの夜を過ごし、異なるアプローチを試みましたが、それらはすべて同じ結果につながりました。 結局、私はただ吐いて原始的なハックをしましたが、驚くほどうまくいきました： 

// SpriteGroup HACK! if (displayWidthScale == 1.0f) displayWidthScale = 1.00001f; if (displayHeightScale == 1.0f) displayHeightScale = 1.00001f;


ですから、それが何であるか、なぜインターネット上でそれについて何も見つけられなかったのかはまだわかりません。



4）多くの広告ネットワークがあります;それらは異なったリターンを与えます。 同じネットワークは、異なるアプリケーションで異なる動作をする可能性があります。 一言で言えば、広告を試すことは間違いなくたくさんあります。 広告ネットワークを交換するときに更新するたびに悪いオプションなので、私はすぐにゲームを再構築せずに広告を管理する機能について考えました。 非常に優れたアプローチをここで説明しますが 、私はゲームに似たような、より単純なものを実装しました。 いくつかの広告ネットワーク（AdMob、Chartboost、Tapjoy）のSDKを組み込み、広告マネージャーのクラスを実装しました。 マネージャは、特定の時点でどのネットワークのどのバナー/フルスクリーンを表示するかを決定します。 そして、このプロセスを管理するために、各起動時（ただし1日に1回のみ）にゲームがサーバーから小さなjson-configをダウンロードします。 設定で設定を変更すると、1日で最大2回、選択したネットワークのバナーがプレーヤーの大部分に表示されます。 素晴らしい、管理しやすい-利益！ おそらく、後でこのコードを別のライブラリとして配置し、githubに投稿します。

現時点では、AdMobバナーとChartboostフルスクリーンバナーがゲーム内で回転しています。インジケーターは私に合っています。



5）Androidの世界は多くのアプリストアの世界であり、利益と視聴者を逃すのはばかげています。GooglePlayでの公開に限定されています。 ただし、各店舗には独自のサービスがあり、多くの場合、それを統合する必要があるため、開発段階でさまざまな店舗のビルドをより快適に組み立てることができます。 1つの定数を変更することで（もちろんいくつかのフレームワークで）必要に応じてゲームを再構成できるロジックを開発しましたが、まだテストする必要はありませんでした-これまでにGoogle Playでのみ公開しており、ゲームの開発に忙しいです。 このトピックの説明は、別の投稿に値するものであり、今後数週間で間違いなくやろうとします。 とりあえず、これは開発の最初から収益化の設計において考慮に入れなければならないことを簡単に言います。 たとえば、ゲーム内での購入が各ストアで利用可能になることを期待しないでください。つまり、収益化の代替方法について考える必要があります。



6）最後のヒントは完全に技術的なものではありません。 ゲームの開発を開始した場合は、自分でプレイすることを実行してください。 論争の的となっている問題についての自分との妥協の最小値、厳格な内部自己検閲の最大値。 機能が疑わしいですか？ 自分に合うように削除するか、少なくともリメイクします。 気に入らなければ、プレイヤーはそれを高く評価することを期待する必要はありません。 たとえば、結果に満足するまで、プロンプトのロジックを3〜4回変更しました。 一方、興味深いチップを追加するのに遅すぎることはなく、それらが役割を果たすことができます。 たとえば、レベルの最後のコメントは、文字通り最後の瞬間に追加されましたが、プレイヤーに好評を博し、ゲームのハイライトのようなものになりました。







プロモーション



私たちのプロモーション戦略は、「お金がありません！」という2つの言葉で説明されています。 ここからこれとダンス。



1）F2Pは惑星を歩いています。 まあ、または地球上ではなく、確かにモバイルゲームの世界のために。 直接販売では、a）非常に高品質のプロジェクトまたは傑出したプロジェクト、b）莫大なマーケティング予算の2つの条件下でのみ良好な収入が期待できます。 それ以外の場合、ほとんどの場合、ゲームを無料で提供し、ゲーム自体でプレーヤーに提供するものについてパズルを解いて、彼らがそれを購入する必要があります。 ゲーム自体の楽しみを壊さないことをお勧めします。 私たちのゲームも例外ではありませんでした。販売のヒントは、急に良いホット商品であることが判明しました。 さて、次の項目に進みます。







2）Google Playは1日に1回更新されます-憧れは緑色です。 FlurryとGoogle Analyticsには、強力な更新間隔もあります。 しかし、統計ページを毎分座って更新するのはどうでしょうか？ 私は友人とそのような喜びを奪うことができなかったので、単純なDBとファイルをPHPにアップロードし、それをホスティングにアップロードしました。 最初の起動時に、ゲームはサーバーに挨拶を送信します。国ごとに最も関連性の高いインストール番号（デバイスのロケールから取得）を賞賛する機会があります。 同様の何かを取得することを強くお勧めします-それは価値があります。

ちなみに、このシステムは予想外の発見をするのに役立ちました-リリース後間もなく、ゲームはウェブサイトやアプリケーションフォーラムを通じて中国とうまくいきました。現時点では、中国のプレイヤーは既にGoogle Playで追跡されていない約100,000のインストールを記録しています。



3）プロモーション予算が私たちの予算と似ている場合、解決方法は1つしかありません-お金を熱意に交換します。 ゲームやAndroidアプリケーションサイトに書き込み、レビューサイトでレビューを書いてもらい、フォーラムで視聴者と協力し、Google Playでレビューに返信することを忘れないでください。 重要なポイント-ソーシャルネットワークでの広告は非常に効果的です。 私はバナーネットワークを意味するのではなく、大きなテーマの公共の場所でのカスタム投稿です。 また、リーズナブルな価格でパブリッシュするときに良いリターンをもたらすリソースもいくつかあります-たとえば、w3bsit3-dns.com。

インターネットやハブには非常に多くのリンクやリソースリストがあるため、特定のリンクやリソースリストは提供しません。 グーグルにYandexに答えさせてください。



4）ご存知のように、Google Playは新しいものと更新されたもののリストの作成を長い間停止しています。つまり、ゲームを公開して宣伝しない場合、これは、そのような箱で満たされた巨大な倉庫に持ち込んで商品を入れたものと同等です。 一方で、上位の新しいものがあります。成長指標が良好で、1か月前に公開されたゲームやアプリケーションがあります。 このトップは、ゲームを動かしてプロモートする人に大きなスタートの利点を与えます。 あなたは本当にそこに到達しようとする必要があります。そのためには、リリース前にプロモーションプランを準備することをお勧めします。 公開から30日後、そこからゲームがクラッシュし、一般的なルールでプレイを開始します（多くの場合、通常はそのルールでプレイします）。

私たちのゲームは、ロシアのパズルのカテゴリで新しいものの上位5位から10位に急成長し、残りの月はロシアで1日あたり1000回以上の追加ダウンロードがありました。
