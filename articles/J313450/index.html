人。 C ++作成者Bjarne Straustrup、プログラミング言語を「決して愛したことはない」





C ++の作成者は、「人々に出かける」機会はほとんどありませんでした。貧しい家族、市内で最悪の学校、何かに対する顕著な傾向の欠如です。 彼は「華麗」でさえありませんでした。



これは自作と呼ぶことができる人です。 彼の主な目標は、何でも真の成功を達成することでした。



おそらく私たち全員が、それがプログラミングに「組み込まれた」ことは幸運だったでしょう。 C ++のレガシーのない現代のIT業界を想像するのは現在非常に難しいからです。



Bjarne Straustrupは、デンマークのオーフス市で1950年12月30日に生まれました。 これはユトランド半島の東海岸に位置する、デンマークで2番目に大きい（人口25万人）都市です。



彼の両親は「学者」ではないという事実にもかかわらず、彼は学校でよく勉強しました。 しかし、ビャルネが後に知ったように、それは市内で最悪の学校の一つでした。



子供の頃、将来のエンジニアはサッカーをしたり、自転車に乗ったりするのが大好きでした。 彼はスカウトでもありました。 当時のほとんどの少年たちと同じように、ストラウストルップは新聞と牛乳を配達して余分なお金を稼いだ。



卒業後、彼はまだコンピューター技術部門の地元の大学に在籍していました。 Bjarneは、彼が建築家、歴史家、社会学者になることができると認めました。 しかし、結局、彼は数学とコンピューターサイエンスに落ち着きました。 彼の決定は、数学の学校の先生から影響を受けました。数学の先生は、少年にこの科学への愛を植え付けました。



大学



彼は最初に大学の壁にあるコンピューターと初めて会った。 Algol-60は、Bjarneが学んだ最初のプログラミング言語でした。







オーフス大学は、世界のトップ100大学にありました。 恐怖のストラウストラップは、プログラミングの分野で仕事がなければ数学の教師として働かなければならないと考えました。 彼は、数学には実際的な応用が必要だと確信していました。



しかし、2年目に、彼はプログラミング、およびコンピューター技術全般が数学だけでなく、それほど多くないことを理解しました。 さらに、Bjarneは彼が思ったほど数学が得意ではなかったことが判明しました。



インタビューで、 Straustrupはプログラミング指導に関する彼の立場を概説しました。



私は教育をトレーニングコースよりも高く評価しています。 ほとんどの場合、高度に専門化されたコースの修了証明書を持っている人よりも、広い視野と深い基礎知識を持つ専門家を選びます。 一般的に、私の理想は学問的な知識と実践的なスキルの組み合わせです。


大学時代、ストラウストルップはオーフスのバロウズ社の小さなプログラムを書くことでパートタイムで働いていました。 それから彼は、商業開発は自分自身や友人のためのプログラムを書くこととは関係がなく、さらに大学での実験室作業の実施とは関係がないことに気付きました。 Bjarneは、ユーザーとのコミュニケーションの経験、コードのサポートと保守の経験を​​得ました。



学生は収入の一部を自分の授業料の支払いに費やしました。 彼の開発は、彼自身にとって有益であることに加えて、会社を新しいレベルに引き上げるのに役立ちました。バローズは後にスペリーと合併し、統合ソフトウェアおよびハードウェアソリューションのグローバルメーカーであるユニシスを設立しました。



さらに、学生は大学のニーズに合ったソフトウェアを開発しました。 1975年、ストラウストルップは卒業し、修士号を取得しました。



ケンブリッジ



トレーニングの終わりに向かって、Bjarneはマイクロプログラミングに興味を持つようになりました。 ケンブリッジでのインターンシップ後、関心が高まりました。 そのため、Straustrupはイギリスに移り、ケンブリッジ大学での教育を続けました。 マイクロプログラミングの分野では、研究に必要なすべての機器がありました。







さらに、新しい大学は彼が彼自身の要求を増やすのを助けました。 この大学では、学生と教師のレベルはオーフス大学よりもはるかに高かった。 これにより、Straustrupはより積極的に開発するようになりました。



ケンブリッジのコンピューティング研究所で、彼は分散システムの設計に従事していました。 1979年、ビャルンは博士号を取得しました。



同年、ストラウストルップと妻のマリアンはニュージャージー州（米国）に移り、ベルラボコンピューターリサーチセンターで働き始めました。 当時、同社はケンブリッジ大学と「友好的な」関係にありました。







C ++



Straustrupが会社で研究に従事していたとき、彼は分散コンピューティングをモデリングするためのいくつかのシミュレーションプログラムを作成する必要がありました。 SIMULA-67-オブジェクト指向言語-は、そのような問題を解決するための理想的なツールになる可能性がありますが、プログラムの実行速度が比較的遅いためではありません。 ところで、Bjarneはケンブリッジで論文を書いているときにこの言語を使用しました。そこで彼は分散システムのシミュレーターを開発しました。



これ以前、Straustrupはプログラミング言語の開発に興味がありませんでした。 彼はマイクロプログラミング、オペレーティングシステム、コンピューターのアーキテクチャ、仮想マシンの開発に興味がありました。 確かに、卒業する前でさえ、彼はなんとか20のプログラミング言語を学ぶことができました。 しかし、それにもかかわらず、Bjarneは、まだ言語にあまり興味を感じていないと主張しています。



しかし、Straustrupは新しいプログラミング言語で作業を開始することにしました。



「良い」コードと「速い」コードのどちらを書くかを選択する必要がある場合、ここで何かが間違っています。 「良い」コードは「高速」でなければならないため、Straustrup はインタビューで説明しています。



これが、元々Cとクラスと呼ばれていたC ++プログラミング言語の作成方法です。 「C ++」という名前は、Rick Maschittiによって造られました。 「++」はCのインクリメント演算子です。これは、C ++言語が単なるCを超えていることを示唆しているようです。



1983年、言語は大幅に変更されました。 Straustrupは、コードが機械語にコンパイルされるのではなく、Cプリプロセッサコマンドのセットに変換されるように言語を設計したため、適切なコンパイラを持っている何十万人ものCプログラマーにアクセスできました。



1984年、ベル研究所は再編成され、AT＆Tベル研究所と命名されました。 同社が最初にC ++を提供し、ほとんど無料だったのは大学でした。



1985年、Straustrupは、最も広く知られている書籍の1つであるThe C ++ Programming Languageを出版し、4版（1985、1991、1997、2000）を生き延び、19言語に翻訳されました。







トム・ダフィーは次のように書いています。 しかし、勤勉さと先見性のおかげで可能になった人は他にもたくさんいます。 1985年にベルラボが最初に大学に提案したC ++プログラミング言語は、確かに2番目のカテゴリに分類されます。



コミュニティ構築



言語はすぐに聴衆を見つけました。 1987年、200人がC ++会議に集まりました。 翌年、参加者の数は600人に増加し、90年代前半にはC ++の開発者の数は50万人に近づいていました。 これにより、この言語は配信速度の世界的リーダーとなっています。



1990年にStroustrupの次の本、The Annotated C ++ Reference Manualが出版され、後に博士の意見で卓越した技術ドキュメンテーションのスキルが授与されました。 ドブの日記。



さらに、フォーチュン誌によると、ストラウストルプは「十二人の若きアメリカ人科学者」の一人として認められました。



一方、StraustrupはAT＆T Bell Labsで働き続け、そこで大規模なプログラム研究ユニットを率い、言語の改善と標準の作成に積極的に関与しました。 ANSI / ISO C ++標準は1999年にリリースされました。



Straustrupは、1995年から2002年までBell Labsのプログラミング研究の責任者でした。







Bjarne Straustrup-C / C ++について



「C ++を思いつき、その初期定義を書き留め、最初の実装を完了しました。 C ++の設計基準を選択および策定し、その主要な機能を開発し、C ++標準化委員会で言語を拡張する提案の運命を担当し、著者を書きました。 -C ++言語はC言語に大きく依存しており、C言語はC ++言語のサブセットのままです（ただし、C ++では、C型システムのいくつかの重大な欠陥が排除されています）。 また、最も重要なシステムタスクを処理するのに十分な低レベルのCツールを保持しました。 C言語は、その前身であるBCPLに多くを負っています。



2003年のインタビューで、 StraustrupはCおよびC ++の互換性の問題に対する彼の態度について話しました。

CとC ++の非互換性の既存の問題は、これらの言語の歴史で起こりうる最悪のことだと思います。 理想的には、これらの言語を組み合わせる必要があります。 そして、これは両方の言語に変更が加えられた場合にのみ技術的に実行可能であるように思えます。 ただし、これらの変更を行うことに消極的であるため、言語は互いに離れ続け、ほとんどすべてのCおよびC ++開発者の生活を害しています。



私は人々に組合のプラスを見てもらいたいと思っています。 このトピックに関する一連の記事を書きました。 しかし、誰かがこれを防ぐことに興味を持っています。 加えて、私はコミュニティからあまり関心がありません。 おそらく大多数は、いつものように、すべてが「解決する」ことを望んでいます。


AT＆T Bell Labsの後



2002年以来、BjarneはテキサスA＆M大学の教授を務めています。 彼はこの業界で四半世紀近く働いていたので、学生に何か言いたいことがあります。



また、コンピューターサイエンスの客員教授としてコロンビア大学と協力しています。



2014年、Bjarneは世界最大の金融コングロマリットの1つであるMorgan Stanleyから仕事を提供されました。



Straustrupは、純粋な教育学の分野から実際の問題の解決に戻ることを決めたため、この会社と協力し始めました。 彼の豊富な経験は、モルガン・スタンレー銀行がパフォーマンス、信頼性、安全性に関連する問題を解決するのに役立つことが証明されました。



さらに、彼は彼の子供や孫により近くなるためにニューヨークに移りたかった。



Bjarne Straustrupは、2004年に「C ++言語の作成」という言葉で米国国立工学アカデミー（「国立工学アカデミー」）のメンバーに選出されました。 最初のコンピューター科学者として、2005年にアメリカの研究コミュニティから賞（「シグマXiからの科学的業績に対するウィリアムプロクター賞」）を受賞しました。 彼はまた、IEEEプログラムコミュニティ賞を受賞しました。「プログラムを作成するためのオブジェクト指向技術の開発と商業化を開始し、ビジネスと産業に大きな変化をもたらした」
