RosYamaは無料ライセンスの下でサイトのソースコードをオープンしました

1週間前、有名なRosYamaプロジェクトのコーディネーターは、BSDのようなライセンスの下でサイトのソースコード（PHPスクリプト）を開くことを発表しました。「発生するRosYamaのすべてのクローンを追跡することはできません。同胞、より自由にライセンスを選択することにしました。 誰でも無料でソースコードをダウンロードし、希望に応じて変更し、商用および非公開のものを含むプロジェクトで使用できます」と、サイトの開発を行ったGreensight Web Studioの代表者は言います。



Mercurialのリポジトリはhg.rosyama.ruにあります。 クローニング： 



  hg clone http://hg.rosyama.ru rosyama-local /


新しいバージョンへのアップグレード-hg pullコマンドを使用。 プロジェクトのサポートと更新は継続され、リポジトリはプル時にのみ開かれます。 バグトラッカーはtrac.rosyama.ruで動作します。



RosYamaの開発価格は約200,000ルーブルです。 プロジェクトのコストと公共の重要性を考慮すると、すべてのマイナス点にもかかわらず、オープンソースへの移行は歓迎されています。



そして今、マイナス点について。 批判の主な理由は、使用されているCMSであり、これはオープンソースとは関係ありません。 残念ながら、無料のコードの支持者のために、ここではBitrix CMSが使用されています。 開発者は、CMSの選択について次のように説明しています。 Greensightの予算の大部分は、企業クライアント向けのカスタムデザインとコンポーネントを備えたBitrixでのサイトの販売です。 「Bitrixは途方もなく無力な怪物の叙事詩です」と言う人々に同意しますが、私たち自身のCMSはありませんが、Bitrixはソーシャルネットワークを介して承認されているため、RosYamaにすべての基本的な機能を提供します」 （語彙とスペルは変更） 。



そのため、プロジェクトのクローンを作成するには、少なくとも最小のBitrixライセンスを購入する必要があります（2つの情報ブロックの最小ライセンスを除く）。



プロジェクトは、基本的な機能を備えた静的クラスの説明を含む個別のモジュールとして実装されます。 このモジュールは、Bitrix管理パネルからインストールできます。 bitrix / components / greensightフォルダーにはコンポーネントが含まれています。



原則として、WordPressなどの別のプラットフォームにプロジェクトを移植する人はいません。
