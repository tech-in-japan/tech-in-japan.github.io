問題に直面した人の中には、「ああ、でも正規表現を使っている」と思う人もいます。 現在、2つの問題があります。

ジェイミー・ザウィンスキー



ユアン・マは、次のように述べています。 問題の構造全体をプログラムするには多くのコードが必要です。

マスターユアンマー、「プログラミングブック」



プログラミングツールとプログラミングテクニックは、無秩序に進化し続けます。 生き残るのは美しくて独創的ではない場合もありますが、たとえば、他の成功したテクノロジーに統合されている場合など、その分野で非常にうまく機能している人だけです。



この章では、このようなツール-正規表現について説明します。 これは、文字列データのパターンを記述する方法です。 JavaScriptや他の多くの言語とツールに含まれる小さな別個の言語を作成します。



レギュラーは同時に非常に奇妙で非常に便利です。 それらの構文は不可解であり、JavaScriptのプログラミングインターフェイスは扱いにくいです。 しかし、文字列を探索および処理するための強力なツールです。 それらに対処すると、より効果的なプログラマーになります。



正規表現を作成する



Regularはオブジェクトのタイプです。 RegExpコンストラクターを呼び出すか、スラッシュで囲まれた目的のテンプレートを記述することで作成できます。 



var re1 = new RegExp("abc"); var re2 = /abc/;




これらの正規表現はどちらも1つのパターンを表します。文字「a」、文字「b」、文字「c」の順になります。



RegExpコンストラクターを使用する場合、テンプレートは通常の文字列として書き込まれるため、バックスラッシュに関するすべての規則が適用されます。



パターンがスラッシュの間にある2番目のエントリは、バックスラッシュを異なる方法で処理します。 まず、パターンはスラッシュで終わるため、パターンの前に含めるスラッシュの前にバックスラッシュを置く必要があります。 さらに、\ nのような特殊文字の一部ではないバックスラッシュは保存され（行のように無視されません）、パターンの意味を変更します。 疑問符やプラスなどの一部の文字は、正規表現では特別な意味を持ち、そのような文字を見つける必要がある場合は、バックスラッシュを前に付ける必要があります。 



 var eighteenPlus = /eighteen\+/;




どの文字の前にスラッシュを付ける必要があるかを知るには、レギュラーのすべての特殊文字のリストを学習する必要があります。 これは非現実的ですが、疑わしい場合は、文字、数字、スペース以外の文字の前にバックスラッシュを置くだけです。



一致を確認する



レギュラーにはいくつかの方法があります。 最も簡単なのはテストです。 文字列を渡すと、指定されたパターンの出現が文字列に含まれているかどうかを示すブール値が返されます。 



 console.log(/abc/.test("abcde")); // → true console.log(/abc/.test("abxde")); // → false




非特殊文字のみで構成される規則性は、単にこれらの文字のシーケンスです。 abcがチェックしている行のどこかにある場合（最初だけでなく）、testはtrueを返します。



文字セットを探しています



文字列にabcが含まれているかどうかを調べるには、indexOfを使用することもできます。 レギュラーを使用すると、さらに進んでより複雑なパターンを作成できます。



任意の数を見つける必要があるとします。 レギュラーシーズンに文字セットを角かっこに入れると、これは式のこの部分が角かっこ内の文字のいずれかに一致することを意味します。



両方の式は、数字を含む行にあります。 



 console.log(/[0123456789]/.test("in 1992")); // → true console.log(/[0-9]/.test("in 1992")); // → true




角括弧内では、2文字間のダッシュを使用して、Unicodeでシーケンスが指定されている文字の範囲を指定します。 0から9の文字が一列に並んでいる（48から57のコード）ため、[0-9]はすべてをキャプチャし、任意の数字と一致します。



文字のいくつかのグループには、独自の組み込み略語があります。



\ d任意の数字

\ w英数字

\ s空白（スペース、タブ、改行など）

\ Dは数字ではありません

\ Wは英数字ではありません

\ Sは空白ではありません

。 改行以外の任意の文字



したがって、次の式を使用して、2003-01-01 15:20のような日付と時刻の形式を設定できます。 



 var dateTime = /\d\d-\d\d-\d\d\d\d \d\d:\d\d/; console.log(dateTime.test("30-01-2003 15:20")); // → true console.log(dateTime.test("30-jan-2003 15:20")); // → false




ひどいですね。 バックスラッシュが多すぎると、パターンの理解が困難になります。 後で少し改善します。



バックスラッシュは角括弧でも使用できます。 たとえば、[\ d。]は数字またはピリオドを意味します。 角括弧内のドットは特別な意味を失い、単なるドットに変わることに注意してください。 +など、他の特殊文字についても同じことが言えます。



文字セットを反転することができます-つまり、セット内の文字以外の文字を見つける必要があるということです-開始角括弧の直後に^記号を置くことによって。 



 var notBinary = /[^01]/; console.log(notBinary.test("1100100010100110")); // → false console.log(notBinary.test("1100100010200110")); // → true




テンプレートの一部を繰り返します



1つの番号を見つける方法を知っています。 そして、整数を見つける必要がある場合-1桁以上の数字のシーケンス？



レギュラーシーズンで何かの後に+記号を付けると、この要素を複数回繰り返すことができます。 / \ d + /は、1桁以上の数字を意味します。 



 console.log(/'\d+'/.test("'123'")); // → true console.log(/'\d+'/.test("''")); // → false console.log(/'\d*'/.test("'123'")); // → true console.log(/'\d*'/.test("''")); // → true




アスタリスク*の値はほぼ同じですが、パターンをゼロ回表示できます。 何かの後にアスタリスクが付いている場合、その行にテンプレートが存在することを妨げることはありません。ゼロ回だけ存在します。



疑問符は、テンプレートの一部をオプションにします。つまり、ゼロまたは1回出現します。 次の例では、u記号が発生する場合がありますが、パターンは存在しない場合にも一致します。 



 var neighbor = /neighbou?r/; console.log(neighbor.test("neighbour")); // → true console.log(neighbor.test("neighbor")); // → true




パターンが発生する正確な回数を指定するには、中括弧を使用します。 要素の後の{4}は、行に4回出現する必要があることを意味します。 間隔を設定することもできます。{2,4}は、要素が2回以上4回以下を満たす必要があることを意味します。



1桁または2桁の日、月、および時間を許可する別のバージョンの日時形式。 そして彼女はもう少し読みやすいです。 



 var dateTime = /\d{1,2}-\d{1,2}-\d{4} \d{1,2}:\d{2}/; console.log(dateTime.test("30-1-2003 8:45")); // → true




数値の1つを省略することにより、オープンエンドギャップを使用できます。 {、5}は、パターンが0〜5回発生することを意味し、{5、}-5回以上発生することを意味します。



部分式のグループ化



複数の要素で*または+演算子を一度に使用するには、括弧を使用できます。 括弧で囲まれた規則性の部分は、演算子の観点から1つの要素と見なされます。 



 var cartoonCrying = /boo+(hoo+)+/i; console.log(cartoonCrying.test("Boohoooohoohooo")); // → true




最初と2番目のプラスは、booとhooの2番目の文字oにのみ適用されます。 3番目の+はグループ全体に属し（hoo +）、これらのシーケンスの1つ以上を見つけます。



式の最後にある文字iは、通常の大文字と小文字を区別しないため、Bはbと一致します。



マッチとグループ



テスト方法は、定期的にチェックする最も簡単な方法です。 一致が見つかったかどうかのみを報告します。 レギュラーにはexecメソッドもあります。何も見つからなかった場合はnullを返します。それ以外の場合は、一致に関する情報を含むオブジェクトを返します。 



 var match = /\d+/.exec("one two 100"); console.log(match); // → ["100"] console.log(match.index); // → 8




返されるexecオブジェクトには、一致が発生した文字の番号を含むインデックスプロパティがあります。 一般に、オブジェクトは文字列の配列のように見えます。最初の要素は、一致をチェックした文字列です。 この例では、これは探していた一連の数字になります。



文字列には、ほぼ同じ方法で機能する一致メソッドがあります。 



 console.log("one two 100".match(/\d+/)); // → ["100"]




括弧でグループ化された部分式が正規表現に含まれている場合、これらのグループに一致するテキストも配列に表示されます。 最初の要素は常に完全に一致します。 2番目は、最初のグループ（かっこが他の全員よりも先に出会ったグループ）、次に2番目のグループなどと一致した部分です。 



 var quotedText = /'([^']*)'/; console.log(quotedText.exec("she said 'hello'")); // → ["'hello'", "hello"]




グループがまったく見つからない場合（たとえば、後ろに疑問符がある場合）、配列内のその位置にはundefinedが含まれます。 グループが複数回一致する場合、最後の一致のみが配列に含まれます。 



 console.log(/bad(ly)?/.exec("bad")); // → ["bad", undefined] console.log(/(\d)+/.exec("123")); // → ["123", "3"]




グループは、文字列の一部を抽出するのに役立ちます。 文字列に日付があるかどうかを確認するだけでなく、抽出してオブジェクトを表す日付を作成する必要がある場合は、数字のシーケンスを括弧で囲み、execの結果から日付を選択できます。



しかし、手始めに、JavaScriptで日付と時刻を保存する好ましい方法を学ぶ小さな余談。



日付タイプ



JavaScriptには、日付用の標準オブジェクトタイプがあります。 日付と呼ばれます。 newを介して日付オブジェクトを作成するだけの場合、現在の日付とレミーを取得します。 



 console.log(new Date()); // → Sun Nov 09 2014 00:07:57 GMT+0300 (CET)




特定の時間を含むオブジェクトを作成することもできます。 



 console.log(new Date(2015, 9, 21)); // → Wed Oct 21 2015 00:00:00 GMT+0300 (CET) console.log(new Date(2009, 11, 9, 12, 59, 59, 999)); // → Wed Dec 09 2009 12:59:59 GMT+0300 (CET)




JavaScriptは、月番号がゼロから始まり、日番号が1から始まるという規則を使用します。 これはばかげてばかげています。 気をつけて。



最後の4つの引数（時間、分、秒、ミリ秒）はオプションであり、ゼロに等しくない場合。



タイムスタンプは、1970年の初めから経過したミリ秒数として保存されます。 負の数値は1970年まで使用されます（これは、その頃に作成されたUnixの時間契約によるものです）。 日付オブジェクトのgetTimeメソッドは、この数値を返します。 自然に大きいです。 

 console.log(new Date(2013, 11, 19).getTime()); // → 1387407600000 console.log(new Date(1387407600000)); // → Thu Dec 19 2013 00:00:00 GMT+0100 (CET)




Dateコンストラクターに1つの引数を指定すると、ミリ秒数として扱われます。 Dateオブジェクトを作成してgetTimeメソッドを呼び出すか、Date.now関数を呼び出すことにより、ミリ秒の現在の値を取得できます。



Dateオブジェクトには、コンポーネントを取得するためのgetFullYear、getMonth、getDate、getHours、getMinutes、およびgetSecondsメソッドがあります。 また、93や14など、かなり役に立たない2桁のコードを返すgetYearメソッドもあります。



テンプレートの必要な部分を括弧で囲んだので、文字列から直接日付オブジェクトを作成できます。 



 function findDate(string) { var dateTime = /(\d{1,2})-(\d{1,2})-(\d{4})/; var match = dateTime.exec(string); return new Date(Number(match[3]), Number(match[2]) - 1, Number(match[1])); } console.log(findDate("30-1-2003")); // → Thu Jan 30 2003 00:00:00 GMT+0100 (CET)




単語と行の境界線



残念ながら、findDateは文字列「100-1-30000」から無意味な日付00-1-3000も喜んで取得します。 一致は行のどこでも発生する可能性があるため、この場合は2番目の文字で始まり、最後から2番目の文字で終了します。



文字列全体を取得するために一致を強制する必要がある場合は、^および$ラベルを使用します。 ^は行の先頭に一致し、$は末尾に一致します。 したがって、/ ^ \ d + $ /は1桁または複数の数字のみで構成される行に一致し、/ ^！/感嘆符で始まる行に一致し、/ x ^ /はどの行にも一致しません（行の先頭より前にx）。



一方、日付が単語の境界で開始および終了することを確認する必要がある場合は、\ bラベルを使用します。 単語の境界は、行の先頭または末尾、または英数字\ wが一方の側にあり、もう一方の側に英数字ではない行の任意の場所になります。 



 console.log(/cat/.test("concatenate")); // → true console.log(/\bcat\b/.test("concatenate")); // → false




境界ラベルはシンボルではないことに注意してください。 これは単なる制限であり、特定の条件が満たされた場合にのみ一致することを意味します。



選択テンプレート



テキストに数字だけでなく、数字の後に単数形または複数形の豚、牛、または鶏が続くかどうかを調べる必要があるとします。



3つの常連を書いて順番にチェックすることもできますが、もっと良い方法があります。 シンボル| テンプレートの左側と右側の選択肢を示します。 そして、私たちは次のように言うことができます： 



 var animalCount = /\b\d+ (pig|cow|chicken)s?\b/; console.log(animalCount.test("15 pigs")); // → true console.log(animalCount.test("15 pigchickens")); // → false




括弧は、|が適用されるパターンの部分を制限します。これらの演算子の多くを次々に配置して、3つ以上のオプションの選択を示すことができます。



検索エンジン

そのため、フローチャートを通過するときに「3つの豚」の行で定期的に一致するものを確認することは次のようになります。

基本的に、正規表現は次のように機能します。アルゴリズムは行の先頭から開始され、そこで一致するものを見つけようとします。 私たちの場合、そこには単語の境界があるので、最初の長方形を通過します-しかし、そこには数字がありませんので、2番目の長方形でつまずきます。 次に、彼は行の2番目の文字に移動し、そこで一致するものを見つけようとします...など、一致するものが見つかるか、行の終わりに達するまで続きます。



キックバック



規則性/ \ b（[01] + b | \ d + | [\ da-f] h）\ b /は、bが後に続く2進数、サフィックスのない10進数、または16進数（0〜 9または文字a〜h）の後にhが続きます。 対応するチャート：







一致の検索では、文字列にそのような番号がない場合でも、アルゴリズムが上位パス（2進数）に沿って進むことがあります。 たとえば、「103」という行がある場合、番号3に到達して初めてア​​ルゴリズムが間違ったパス上にあることを理解することは明らかです。 一般に、ラインはレギュラーシーズンと一致しますが、このブランチではありません。



その後、アルゴリズムはロールバックします。 分岐点で、彼は現在の位置を記憶します（この場合、これは単語境界の直後の行の先頭です）。選択したパスが機能しない場合は、戻って別のパスを試すことができます。 行「103」の場合、トリプルに出会った後、彼は戻り、10進数の方法を試みます。 これは機能するため、一致するものが見つかります。



アルゴリズムは、完全に一致するものが見つかるとすぐに停止します。 これは、複数のオプションが表示される場合でも、そのうちの1つだけが（レギュラーシーズンで表示される順序で）使用されることを意味します。



+や*などの繰り返し演算子を使用すると、キックバックが発生します。 文字列「abcxe」で/^.*x/を探すと、規則性の一部*行全体を吸収しようとします。 アルゴリズムは、「x」も必要であることを認識します。 行の終わりの後に「x」がないため、アルゴリズムは1文字をロールバックして一致を見つけようとします。 abcxの後、xも存在しないため、サブストリングabcにすでにロールバックされます。 そして、行の後、彼はxを見つけ、位置0から4で成功したマッチについて報告します。



複数のロールバックにつながる定期的なシーズンを書くことができます。 この問題は、パターンがさまざまな方法で入力と一致する場合に発生します。 たとえば、2進数の正規表現を書くときに間違えた場合、誤って/（[01] +）+ b /のようなものを書く可能性があります。







アルゴリズムが、末尾に「b」を含まないゼロと1の長い文字列でそのようなパターンを検索する場合、最初に数値がなくなるまで内部ループを通過します。 その後、彼は最後に「b」がないことに気づくでしょう、彼は1つの位置をロールバックし、外側のループを通過し、再びあきらめ、内側のループに沿ってもう1つの位置をロールバックしようとします...そして彼は両方のループを使用してこの方法で検索を続けます つまり、文字列の各文字の処理量は2倍になります。 数十人のキャラクターでさえ、マッチを見つけるのに非常に長い時間がかかります。



置換方法



行には、文字列の一部を別の文字列で置換できるreplaceメソッドがあります。 



 console.log("".replace("", "")); // →




最初の引数は規則的である場合があります。この場合、文字列内の最初の正規表現が置き換えられます。 「g」（グローバル、ユニバーサル）オプションをレギュラーに追加すると、最初のオカレンスだけでなく、すべてのオカレンスが置き換えられます 



 console.log("Borobudur".replace(/[ou]/, "a")); // → Barobudur console.log("Borobudur".replace(/[ou]/g, "a")); // → Barabadar




「すべて置換」オプションを別の引数またはreplaceAll型の別のメソッドに渡すことは理にかなっています。 しかし、残念ながら、オプションはレギュラーシーズン自体を通過します。



レギュラーシーズンの指定された行にあるグループへのリンクを使用すると、レギュラーのすべてのパワーが明らかになります。 たとえば、「姓、名」の形式で、人の名前を1行に1つずつ含む行があります。 それらを交換してコンマを削除して「姓」を取得する必要がある場合は、次のように記述します。 



 console.log( "Hopper, Grace\nMcCarthy, John\nRitchie, Dennis" .replace(/([\w ]+), ([\w ]+)/g, "$2 $1")); // → Grace Hopper // John McCarthy // Dennis Ritchie

replaceを使用すると、JavaScriptコードからすべてのコメントを削除する関数を簡単に作成できます。 これが最初の試みです。 



 function stripComments(code) { return code.replace(/\/\/.*|\/\*[^]*\*\//g, ""); } console.log(stripComments("1 + /* 2 */3")); // → 1 + 3 console.log(stripComments("x = 10;// ten!")); // → x = 10; console.log(stripComments("1 /* a */+/* b */ 1")); // → 1 1




「or」演算子の前の部分は、2つのスラッシュとそれに続く改行文字以外の任意の数の文字に一致します。 複数行のコメントを削除する部分はより複雑です。 [^]、つまり任意の文字を見つける方法としての空でない文字。ブロックコメントは新しい行で継続され、改行は期間と一致しないため、ピリオドを使用できません。



しかし、前の例の結論は間違っています。なんで？



[^] *部分は、最初にできるだけ多くの文字をキャプチャしようとします。このため、レギュラーシーズンの次の部分で一致が見つからない場合、1つのキャラクターがロールバックされ、再試行されます。この例では、アルゴリズムは文字列全体をキャプチャしようとし、その後ロールバックします。 4文字をロールバックすると、彼は行に* /-を見つけますが、これは私たちが達成したものではありません。コメントを1つだけキャプチャし、行の最後に移動して最後のコメントを見つけたくはありませんでした。



このため、繰り返し演算子（+、*、？、および{}）は貪欲であると言います。つまり、最初にできるだけ多くを取得し、次に戻ります。そのような演算子（+？、*？、??、{}？）の後に質問を置くと、それらは欲張りでないものに変わり、可能な限り小さな出現を見つけ始めます。



そして、それが私たちに必要なものです。アスタリスクに可能な最小数の行文字で一致を検出させることにより、1ブロックのコメントのみを吸収し、それ以上は吸収しません。 



 function stripComments(code) { return code.replace(/\/\/.*|\/\*[^]*?\*\//g, ""); } console.log(stripComments("1 /* a */+/* b */ 1")); // → 1 + 1




貪欲でない演算子の代わりに貪欲な演算子を使用すると、多くのエラーが発生します。再生演算子を使用するときは、常に貪欲でないオプションを最初に考慮してください。



RegExpオブジェクトの動的作成



場合によっては、コードの記述時点で正確なパターンがわからないことがあります。たとえば、テキストでユーザー名を検索し、アンダースコアで囲む必要があります。プログラムの開始後に名前を認識するだけなので、スラッシュエントリは使用できません。



ただし、文字列を作成してRegExpコンストラクターを使用することはできます。以下に例を示します。 



 var name = ""; var text = "     ."; var regexp = new RegExp("\\b(" + name + ")\\b", "gi"); console.log(text.replace(regexp, "_$1_")); // →   __   .




単語の境界を作成するときは、スラッシュを使用した通常の行ではなく通常の行に書き込むため、二重スラッシュを使用する必要があります。RegExpの2番目の引数には、レギュラーのオプションが含まれます。この例では、「gi」、つまり グローバルで大文字と小文字を区別しません。



しかし、名前が「dea + hl [] rd」の場合（ユーザーがカルハッカーの場合）はどうでしょうか。その結果、文字列に一致するものが見つからない無意味なレギュラーシーズンが得られます。



嫌いな文字の前にバックスラッシュを追加できます。\ bまたは\ nは特殊文字であるため、文字の前にバックスラッシュを追加することはできません。ただし、英数字以外の文字の前に問題なくスラッシュを追加できます。 



 var name = "dea+hl[]rd"; var text = " dea+hl[]rd  ."; var escaped = name.replace(/[^\w\s]/g, "\\$&"); var regexp = new RegExp("\\b(" + escaped + ")\\b", "gi"); console.log(text.replace(regexp, "_$1_")); // →  _dea+hl[]rd_  .

LastIndexプロパティ

通常のオブジェクトにはプロパティがあります。それらの1つは、文字列を含むソースです。もう1つはlastIndexで、特定の条件で、エントリの次の検索を開始する場所を制御します。



これらの条件には、グローバルオプションgの存在の必要性、およびexecメソッドを使用して検索を実行する必要があるという事実が含まれます。より合理的な解決策は、単に追加の引数をexecに渡すことを許可することですが、合理性はJavaScriptの通常のインターフェースの基本的な機能ではありません。 



 var pattern = /y/g; pattern.lastIndex = 3; var match = pattern.exec("xyzzy"); console.log(match.index); // → 4 console.log(pattern.lastIndex); // → 5
