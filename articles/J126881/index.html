Dingoo A320のワイヤレスネットワーク

3年目、Dingoo A320は、エミュレートできるもののポケットサイズのエミュレーターの中で最初の人気を保持しています。正当な理由で-低価格、まともなハードウェア、膨大な数のファンがこのコンソールを文字通り世界を征服することができました。 しかし、XXI世紀には1つの重大な欠点があります-ワイヤレスインターフェースの欠如です。 この問題を解決します。



だから、与えられた-DinguxがインストールされたDingoo A320。 ワイヤレスTCP / IPネットワークを可能な限り予算で固定する必要があります。





* wikipedia.orgからのNzeeminの写真



実際、Vogue / Playboyプレーヤー、さまざまなeInkリーダー、およびモバイルLinuxを搭載した他の多くのデバイスに説明されているすべてのことが当てはまります。







理想的なオプションは、Wi-Fiプレフィックスを取得することですが、これを達成するための唯一の合理的な方法-SDIOカードを使用する-は、2つの理由で消えます：まず、ミニSDディンギングで、そのような形式のSDIOワイヤレスを見つけるのは問題があり、第二に、これまでのdinguxはメモリカードでのみ機能し、スロットは常にビジーです。



Bluetoothは残ります。 プロセッサにBTドングルを接続できるUSBホストがありますが、ボード上のどこにもはんだ付けされていないため、このオプションは表示されません。 そして、ここでかなりクレイジーですが、実行可能なSLIP over Bluetoothの考えがアリーナに入ります。



意味を説明させてください。 Dingooでは、ほとんどのデバイスと同様に、製造元がデバッグに使用するシリアルポートがボード上にあります。 dinguxでは、このポートは完全に自由に使用できます。デフォルトでは、ログインしてコマンドを入力できる通常のLinuxコンソールがハングします。 このコンソールを無効にした場合、コンソールのシリアルポートをコンピューターのシリアルポートに接続すると、クラシックヌルモデムが表示されます。 その後、PPPを使用して、セッションを確立し、シリアルポート経由でTCP / IP接続を確立できます。 はい、それはまさに10年前にモデムを介してプロバイダーに電話したことです。 しかし、コンソールを接続するだけで、モデムプールを整理する必要がないため、PPPは冗長です。



PPPの前身はSLIPでした。 PPPとは異なり、SLIPはピアツーピアネットワーク、つまりサーバーとクライアントマシンが分離されていない場合に機能します。 2台のコンピューターを接続し、シリアルポートを同じ速度で構成し、これがネットワーク接続であることをOSに明らかにすれば十分です。 Linuxでは、slattachコマンドがこれに使用され、Windowsでは数回クリックします。 残念ながら、Windows 7では組み込みのSLIPサポートが削除され、その構成には「Windows XPとの互換性モード」が必要です（Windowsでは強力ではないため、その難しさはわかりません）。

気配りのある読者は既にシリアルポートについて話しすぎており、ワイヤレス接続をどのように整理するかについては何も言っていないため、再送の準備をすでにしています。 読者は正しい。今こそ、切り札を明らかにするときだ。







ミート- ワイヤレスBluetooth RS232 TTLトランシーバーモジュール 。 このデバイスは必要なものを正確に実行でき（上記のスキームでBluetoothシリアルポートを整理し、ヌルモデムケーブルを交換します）、たったの$ 6.60（DealExtremeとeBayで安価です）。 この奇跡（およびほとんどすべての中国のデバイス）についてのきちんとしたドキュメントはありませんが、2、3ページのデータシートから、ワイヤをはんだ付けする場所が明確になります。



必要な接点のピン配列は次のとおりです（アンテナを使用してコンポーネントを保持し、左上の接点から反時計回りに番号を付けます）。

1-Tx、転送（はんだをRxセットトップボックスに）

2-Rx、受信（はんだ付けからTxセットトップボックス）

12-VCC、3.3V

13-GND



ディンを慎重に開きます-背面にある4つのゴム製プラグ、その下にある4つのネジ、その後プラスチックカードでラッチをカチッと開き、バッテリーを非常に慎重に取り外し、さらに2つのネジを外して、ボードを引き出します。 そっと画面をこじ開け、横に曲げます。 画面の下には4つの署名済み連絡先があり、そのうちの2つはシリアルポートです。 BTアダプターを次の「スキーム」に従ってはんだ付けし、はんだ付けを可能な限り「平ら」にしようとします。そうしないと、組み立てられたデバイスで、錫の結節が暗い（これが最高の）スポットが現れる画面を押します。

まず、シリアルポートでコンソールを無効にする必要があります。これは、デフォルトでdinguxが起動する場所です。 これを行うには、USB経由でtelnet経由でdingに接続し、次のように入力します



vi /etc/inittab

Dingooを再起動します。 次に、BTアダプタの設定、つまりポート速度とデバイス名を変更する必要があります。 これを行うには、再度telnetを通過し、

Bluetooth接続が確立されるまで、ボードはATコマンドを受け入れます。 1つの機能があります-入力するときは1秒以内に保持する必要があるため、コマンドは手動で入力するのではなく、コピーと貼り付けで送信する必要があります。 入力したコマンドの後の改行も必要ありません。 まず、いつもの



AT







答えがOKであれば、すべてが正常です。 そうでなければ、はんだ付けの欠陥を探します。

ポート速度を57600に設定します



AT+BAUD7







OK57600が返信します。



およびデバイス名

CONFIG_SLIP = yオプションをオンにして、dinguxカーネルを再コンパイルします。 記事の最後に、私が使用しているコンパイル済みカーネルへのリンクがありますが、それは大部分が時代遅れです（たとえば、マウスエミュレーションはありません）。



再度リブートして、コンピューターの構成に進みます。 残念ながら、手元にWindowsマシンはありません。そのため、すべてはLinuxについてのみ説明されています。 まず、新しいデバイスを見つけてペアリングします。 slattachユーティリティ（Debianのnet-toolsパッケージ）をインストールしてから、スクリプトを実行します



#!/bin/sh

# Should be run with root privileges



rfcomm bind rfcomm0 00:11:22:33:44:55

slattach -L -m -s 57600 -p slip /dev/rfcomm0 &

while ! ifconfig sl0

do

sleep 1

done

ifconfig sl0 10.100.1.1 pointopoint 10.100.1.2 up







00：11：22：33：44：55をdingのbluetoothアドレスに置き換えます（Bluetoothマネージャーまたはhcitool scanコマンドで見つけることができます）。



コンパイルされたslattachのバージョンをdingooにコピーし（最後のリンクを参照）、telnetを（まだusbで）実行し、実行します



#!/bin/sh

/boot/slattach /dev/ttyS0 57600

while ! ifconfig sl0

do

sleep 1

done

stty -F /dev/ttyS0 intr undef quit undef rprnt undef flush undef min 1 cread

ifconfig sl0 10.100.1.2 pointopoint 10.100.1.1 up







端末は最初にユーザーとのテキスト対話用に構成されており、バイナリデータを送信する必要があるため、sttyコマンドが必要です。 このスクリプトをファイルに保存し、dinguxメニューからリンクを作成して、USB経由で接続する必要をなくし、ワイヤレス通信を完全にワイヤレスにする必要があります。



それだけです！ コンピューターで実行する

ping 10.100.1.2





受信した応答に関する楽しいメッセージが表示されます。



デバイスの範囲は約5メートルです（直接視認可能）が、BTアダプターには外部アンテナをはんだ付けする場所があり、その使用により範囲が大幅に拡大する可能性があります。



為替レートを上げるには、まずアダプターを115200ボーに切り替え（そしてすべてのスクリプトをそれに応じて編集し）、次に圧縮バージョン（通常のSLIPではなくCSLIP）を使用します。



もちろん、dinguxがユーザーと通信するためのtelnetおよびftpサーバーは、bluetooth経由で利用可能になりました。 しかし、残念なことに、基本的に他のネットワークソフトウェアはなく、Webブラウザもありません。 そのため、ポーティングがマルチプレイヤーに対応していなかったネットワークエンターテイメントのみのゲーム（Doomなど）から。
