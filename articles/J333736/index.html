KLADRからFIASに切り替えて、何も壊さない方法

GNIVTS連邦税務局は、2018年の初めからKLADRが存在しなくなり、ダウンロードできないことを報告しています。



免責事項：

これらの文字セットの意味がわからない場合は、大丈夫です。 以下では、ロシアの住所を扱う実際について説明します。 興味がない場合は、 地形のしゃれについて読んでください。


正しい住所は、顧客を愛する企業に必要です。 現在KLADRディレクトリを使用しているおなじみの銀行、保険、およびオンラインストアが、次に何をすべきかを尋ねます。 そのため、私たちは疲れて、KLADRからFIASへのある文字から別の文字への移行に関する段階的なガイドを作成しました。







アドレス分類子が必要な理由



企業は、手紙を送り、契約書や契約書に記載するために顧客の住所が必要です。 しかし、手紙を送るだけでは十分ではありません。受け取って読むことをお勧めします。 したがって、アドレスは正しく理解可能である必要があります。



彼から手紙を受け取るためにあなたが最も興味を持っている政府機関は何ですか？ そう！ 税。 そのため、ロシア連邦税務局は長年にわたって部門ごとの全ロシアの住所分類子を開発およびサポートしてきました。 また、完全なディレクトリがなくなったため、あらゆる場所で使用されるようになりました。



最初はKLADRがあり、その中に6つのレベルがありました。





地域→地区→市→町→通り→建物と建物のある家


2012年以降、新しい分類子であるFIASが運用されました。 3年前の記事「FIASまたはKLADR：アドレスのディレクトリを選択する」で、主な違いについて書きました。 連邦税務局は、新しい分類器の開発にさらに徹底的に取り組み、 KLADRマイナスのすべてのレーキを考慮に入れようとしました。 興味深いことに、彼らは録音の開始日と終了日を追加し、各家の固定IDを入力しました（変更されないと想定されています）。



両方の分類子はまだ更新されていますが、互換性を維持することはより困難になっています。 昨年、FIASは新しいレベルを追加し始めました。 たとえば、計画構造はすべての国のパートナーシップおよび細区であり、分類子には既に81,000を超えています。その結果、連邦税務局はKLADRのサポートを停止し、2017年末に削除するという論理的な決定を下しました。









FIASおよびKLADRでの住所の保存方法



CLADR構造にアドレスを保存する多くの方法を見てきました。 基本的には、次の2つのオプションがあります。



  1. 分類器の家の一部は範囲として格納されるため、 「住宅1建物3アパート33 」という形式の通り+家の部分に対するKLADRコード
  2. テキスト形式の住所は単一のフィールドであるか、KLADRの構造によって分類されています。


FIASには家の一部がないため、住所を次のように保存することをお勧めします。



  1. 住所（ 詳細な推奨事項 ）を識別するために、通り+家+アパートへのFIASコード。

  2. FIAS形式の1行の住所。 元のアドレスを保持することをお勧めします。 FIASは依然として税務ガイドであり、さらに多くのことがあります。


つまり、移行のタスクは、KLADR形式のコードまたはテキストアドレスをFIASコードに変換することです。





分類コードは次のようになります。



  • CLADER ：SS + RRR + YYY + PPP + UUUU + DDDD;
  • FIAS ：SS + RRR + YYY + PPP + SSSS + UUUU + DDDD（またはZZZZ）+ OOOO。


恐ろしいように思えます。例を考えてみましょう。ニューオッカービルのマイクロディストリクト、クドロボ村、ヴセヴォロジスク地区のレニングラード地域。 アドレスは実際に存在し、ここではYandex.Mapsにあります。 一見すると、分類コードKLADRとFIASは同じように見えます： 470050000550023 。 それをコンポーネントに分解します。

この例からわかるように、単純な状況でも分類器間に誤解があります。 しかし、別の非常に重要なポイントがあります。



分類コードは住所の現在の構造を反映しており、変更できます。 たとえば、Okkervilでは、領域が変更され、コード全体が無効になりますが、もう見つかりません。



これを取り除くために、FIASはFIAS 識別コードも追加しました。 これは、都市、通り、地区などの各オブジェクトのグローバル識別子です。GUID（Globally Unique Identifier）標準に従って形成された文字と数字のセットのように見えます。 FIASの典型的なGUIDは、 f77948dc-7bc8-42cb-979e-2c958d162d63のようになります。



KLADRコード→FIAS識別コード



あなたの住所が通りのKLADRコード+家の一部（「家1建物3アパート44」）として保存されている場合、彼にとってはすべてが比較的簡単です。





ADDROBJテーブルには、すべてのアドレスオブジェクトとその識別子が格納されます



動作するには、xmlまたはdbf形式のFIASアンロードからのADDROBJテーブルが必要です。 すべてのオブジェクト（都市、道路など）を一意のGUIDで行に格納します。

次の列を使用します。



  • コードKLADR（PlainCode）;
  • テキストの名前とタイプ（FormalNameおよびShortName）;
  • オブジェクトの関連性（CurrStatus）;
  • 郵便番号（PostalCode）
  • FIAS自身および親の識別子（AoGuidおよびParentGuid）;
  • レベル（AoLevel）。


例： モスクワ、アレクサンダーソルジェニツィン通り CLADERコード： 77000000000151900



ステップ1. KLADRコードからストリートにコードを抽出します。つまり、最初の15桁を取得します： 770000000001519 00 。





  • KLADRが15桁未満の場合、11を取り、ゼロを追加します。これは和解のコードです： 77000000000 0000 ;
  • 11桁未満-8を使用： 77000000000000000 ;
  • 8未満-5を取る： 77000 000000000000 ;
  • 5未満-2桁の地域コード77 000000000000000を使用します。



ステップ2. PlainCodeフィールドでコードKLADRを探しています。 レコードが1つある場合は、AoGuidフィールドの値を保存して、次の項目をスキップします。 ただし、コード770000000001519によると3つのエントリがあるため、追加のチェックが必要です。



ステップ3.現在のエントリを見つけます。 CurrStatus = 0フィールドの値は、レコードが関連していることを意味します。 それを選択し、AoGuidフィールドの値を保存します。 FIAS識別コードが見つかりました！





  • Alexander Solzhenitsyn、CurrStatus = 2;
  • 共産主義者B、CurrStatus = 1;
  • Alexandra Solzhenitsyn、CurrStatus = 0、AuGuid = f77948dc-7bc8-42cb-979e-2c958d162d63


ボーナスとして 、見つかったAoGuidのテキストアドレスを収集します。





  • PostalCodeを取得-郵便番号： 109004
  • ShortNameおよびFormalName： 109004、ulを追加します。 アレクサンドラ・ソルジェニツィナ
  • オブジェクトがリージョン（フィールドAoLevel = 1）の場合、アドレスは準備完了です。 しかし、この場合、AoLevel = 7がストリートです。 したがって、ParentGuidフィールドで親要素を探し、前の段落を繰り返します。
  • Alexander Solzhenitsyn Streetの場合、親レコードはすぐに地域のレベルを持ちます-これはモスクワです。 ShortNameとFormalNameを保存し、住所を収集します： 109004、モスクワ、st。 アレクサンドラ・ソルジェニツィナ


ただし、さらに簡単に既成のサービスを使用できます。 DaData.ruは、特定の地域、地域、都市、町の住所を求めることができます。 彼は名前（「 Peterhof 」）、コードKLADR（「 7800000800000 」）およびFIAS （「 8f238984-812b-4bb1-850b-49749fb5c56d 」）を理解しています。
