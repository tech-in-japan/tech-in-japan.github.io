TrackChecker-郵送の監視。 1年半の開発

プログラムの最初のバージョンの1つが記事で説明された瞬間から1年半が過ぎました

Gentlemanのオンラインショッパーセット：Shipito + ProfitBid + TrackChecker by hellt

それ以来、 TrackChecker機能（ 代替リンク ）の開発はまだ続いていません。

ユーザーのサポートとアドバイスのおかげで、プログラムは多くの便利な機能を獲得し、多くのバグを取り除きました。

おそらく、プロジェクトの開発のペースは誰かにとって印象的ではないように思えますが、プログラムの開発は私の趣味であり、私は自由な時間にそれを行うことに注意してください。

この短い記事では、プログラムの新機能、この間に追加された機能とバグについて説明します。





また、文学作品よりもコードを書くことに慣れているので、作家としての私の才能について厳密に判断しないようお願いします。



ご存知のように、自然には、私が作成したアプリケーション（MaxPostTracker、TrackMyPackage、およびその他のいくつか）のような多くのアプリケーションはありません。

しかし、残念なことに、UIまたは機能のいずれによっても、一度も私に適したものはありませんでした。

その後、2010年2月にTrackCheckerが作成されました。



また、アイテムを追跡するために集中型オンラインサービスよりもプログラムを使用する利点を説明したいと思います。

gdeposylka、trackitonline、sms-trackなど：

  • すべてのユーザー情報はローカルに保存されます。
  • オンラインサービスのパフォーマンスに依存しません（明日サイトが機能しなくなったらどうなりますか？）。
  • パッケージの履歴の便利な保管/表示（グループ化/ソートなど）。
  • ブラウザを起動する必要がなく、トラフィックを最小限に抑え、バナー広告を表示する必要がありません。
  • 追跡/保存されたトラックの数に制限はありません。


しかし、否定的な点は言及する価値があります：

  • プログラムの機能をダウンロード、インストール、およびマスターする必要があります。
  • Winでのみ動作します（またはWine経由のLinuxで動作します）。
  • トラックのデータベースはユーザーによって保存されます。それにアクセスするには、トラックのデータベースが記録された記憶媒体を用意するか、ドロップボックスまたは同様のサービスを使用する必要があります。




もちろん、すべてのプログラムとサービスの主なタスクは、郵便物の追跡を容易にすることです。

そして、特定の各ユーザーは、自分にとって最も便利な方法を選択します。

最も重要な変更のリスト



  • サポートされるサービスの数が140個を超えました。 これは、メールサイトからの情報を解析する大幅に変更されたモジュールのおかげで可能になりました。 ただし、完璧に制限はありません。近い将来、ゼロから処理する予定です。
  • プログラムの更新を確認します。 サービスの説明の自動更新。
  • プログラムデータを保存するためのパスを設定します。 データを自動保存します。 データファイルをバックアップする機能。
  • 新しいトラックウィザードを追加します。
  • 高度なサービス設定：選択したサービス、およびデフォルトでトラックを追加するときに選択したサービス。
  • トラックを追加するときのサービスのグループとそれに続く選択。
  • トラックへのサービスの大量インストール。
  • トラックが追加されたときに自動的に選択されるグループのサービスのリストを指定する機能。
  • Ctrlキーを押しながらクリックするとトラックを追跡できる機能の「リンク」フィールド。
  • メニュー項目「サービスのトラックを開く...」。 トラックに関連付けられている郵便サービスのサイトにアクセスできます。
  • サービスを選択するメカニズムが再設計されました（設定で、トラックとグループのsvウィンドウで、大量インストールが行われます）。 サービスのリストでのクイック検索と並べ替え
  • CSVへの簡単なエクスポート。
  • トラックのグループのエクスポート/インポート。
  • トラックのスクリーンショット。
  • トラックツリー内のイベント間の日数はカウントされます。
  • 保存されたトラックの単純な統計をカウントします。
  • サービス記述の新しい形式。
  • QRコードを生成して、 Android用のTrackCheckerにトラックを転送します 。
  • トラックツリーの列の順序を変更します。 照合と並べ替えられた列の保存。
  • プログラムスタートアップキーが追加されました。「d」-データファイルdata.xmlへのパスを指定します。「s」-サービス記述のファイルへのパス、「o」-options.xml構成ファイルへのパス。 指示の形式：[-/]キー「パス」例：trackchecker.exe -d "d：\ data \ data1.xml" -o "d：\ data \ options_home.xml"この場合、データはファイルd：\からダウンロードされますdata \ data1.xml、およびdからの設定：\ data \ options_home.xml、サービス記述のファイルはプログラム設定に従ってロードされます。
