text-size-adjustプロパティはクロスブラウザになります

Safari CSS Referenceの読者は確かに、プロパティ「 -webkit-text-size-adjust 」に出くわしました。これは、ディレクトリに従って、携帯電話（iOS）でのテキストの表示を制御し、次の3つの値のいずれかを取ります：



→「 自動 」（携帯電話のテキストは自動的に調整されます）;



→「 なし 」（携帯電話のテキストサイズは変更されません）。



→「 60％ 」（または別の割合）は、携帯電話で必要な相対的なテキストサイズを意味します。



Roger Johanssonは彼のブログ「 456 Berea Street 」でこのプロパティの使用に警告し、このプロパティの値が「なし」であるサイトに出くわし、フォントサイズを増やすことができないという事実に至ったため、完全に放棄するよう促しました。携帯電話だけでなく、通常のコンピューター（ 「Command-Plus」を押して）では 、WebKitベースのブラウザーのいずれでもありません。Safari、Chrome、OmniWeb、iCabのいずれでもありません...



それにもかかわらず、WebKitはモバイルブラウザの唯一の基盤ではありません。 知っているように、モバイルFirefoxもあります。 Firefox開発者はMDN wikiで 、 Firefox 11以降が「 text-size-adjust 」プロパティもサポートすることを発表しました 。これにより、さらにクロスブラウザになります。 （ XaocCPSが 言うように、 「- ms -text-size-adjust」という名前のこのプロパティは Windows Phone 7でもサポートされています。）



Firefoxが値 " none "をどのように認識するか、時間がわかります。 通常の場合でも、この値に対して-prefix-freeを設定するように注意します。そうしないと、WebKitの結果が不快になります。
