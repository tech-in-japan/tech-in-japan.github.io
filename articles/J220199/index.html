C ++ユーザーグループ、ニジニノヴゴロドでの会議

みなさんこんにちは。



先週の土曜日-4月19日、C ++ユーザーグループの次の会議が開催されました。 今回はニジニ・ノヴゴロドで会いました。

82人が会議に参加し、参加者はニジニノヴゴロドからだけでなく、サンクトペテルブルク、モスクワ、リャザン、サラトフ、チェボクサルイ、エカテリンブルク、ノヴゴロドからでした。 登録したのは少し怖いですが、イルクーツクからの参加者は来ませんでした。 私はすべてが彼とうまくいくことを望みます。



会議について簡単に


3月に、ニジニからC ++で開発しているいくつかの会社に手紙を書きました。 彼は会議の開催を手伝うと申し出ましたが、誰も私に答えませんでした。 特にプログラマーがこれらの会社を含めて会議に来たので、私には奇妙に思えます。 まあ。



会議はUNNが提供する施設で開催されました。 聴衆にはスピーカーがいなかったため、スピーカーを聞くのが難しい場合がありました。 レビューから判断すると、これは会議の最大の欠点の1つでした。 さて、次回はこれが起こらないようにしようとします。



会議では6つのレポートが発表されました。

1.デニス・コルマレフ、 Qt。 地雷原で生き残る方法。 ヒントサッパー。

2. Evgeny Krutko、 並列計算技術を実装して数値計算のパフォーマンスを向上させた経験。

3. Dmitry Prokoptsev、 C ++ 11のRリンク。

4. Alexey Kukanov、 C ++の同時実行性：スレッドではなく、アプリケーションを制御します。

5. Stepan Koltsov、 ミューテックス、効果的なメッセージパッシング。

6.アントン・ポタポフ、 C ++コンテナーおよびマルチスレッド：一緒に、または別々に？



タグ#cppと#nnovをtwitterでブロードキャストしようとしましたが、非常に気が散りました。 後で、会議に関する完全なレポートを収集しようとします。



いくつかのニュース。


最後に、 meetingcpp.ruでWordPressを募集し 、すべての発表がそこに投稿されます。 RSSインポートがあります。これで十分だと思います。 レポートはサイトに投稿されており、ビデオは後日公開されます（今週に期待しています）。



以前は、イベントへの登録が十分に重要であるとは考えていませんでしたが、今では気が変わっており、以降のすべてのイベントでは登録が必須になります。 まず第一に、これは企業を協力に引き付けやすくするために行われました。過去の会議に何人の参加者がいたかは確かです。 さらに、参加者の数が増えるにつれて、地理を追跡することはますます興味深いものになります。 さらに、参加の価格についての質問が少なくなることを願っています。



最後の投稿に対するコメントは、コミュニティにとって関心のあるトピックを選択する問題を提起しました。 提供されたレポートから選択するコミュニティにはまだ反対しますが、トピックの決定を支援するようお願いします（調査は添付されています）。



謝辞


UNN、Yandex、Intelの支援に感謝します！ 彼らの支援がなければ、この会議は起こりませんでした。 土曜日の晴れた春に来て知識を共有するのが面倒ではないことについて、スピーカーに感謝します。 ヒントとフィードバックをサポートしてくれた人に感謝します。 来てくれたみんなありがとう！
