新しく到着したサーバーの1日：ハードウェアのテストと復元の方法







この記事では、サーバーモールの内部キッチンと、サーバーのテストと復旧がどのように行われるかについて少しお話したいと思います。 通常の「使用済み」サーバーとRefurbishedの違いを明確に示し、少し「疲れた」鉄を実際に新しいものに変えるプロセスについて説明します。







サーバーモールのプリセールス部門の素晴らしい内部世界を研究するために、私は彼の専門的な機知と経験を共有したエンジニアの一人と話をしました。 Andreiは、新しく到着したIBM System X 3650 M4に夢中になっていたため、ツアーは実際的な偏りがあることが判明しました。







サーバーを販売することにしたとしましょう



これは、組織だけでなく、評価のためにサーバーモール（CM）に連絡することによって個人でも行うことができます。 Habréは購入プロセスに関する詳細な記事をすでに持っているので、次に何が起こるかを説明します。







電話での会話中に得られた情報によると、専門家は購入の妥当性を決定します。通常は肯定的です。 それでも、一部の第2世代のProLiantの買収は有用ではない可能性が高いため、鉄の見通しの初期評価は非常に重要です。 すべてが順調であれば、特別に訓練されたフォワーダーがセラーに来て、サーバーの目視検査を行い、作業の明らかなエラーをチェックし、彼と一緒にアイロンを取ります。 サーバーの購入は、ロシア全土の会社によって実行されます。













検査により、サーバー復旧のコストを大まかに見積もることができます。ケース上の同じ大きなチップがクラッシュによって簡単に引き起こされ、その後マザーボードのPCBのマイクロクラックによるフローティングエラーが発生します。 サーバーは頻繁にはまったくドロップしませんが、非常にきれいにドロップします。 彼自身が、セダンのトランクで3台のDL380車が輸送され、そこから腺の1つがぎこちなく取り除かれたのを目撃しました。 視覚的には、サーバーの耳が落ちて角がくしゃくしゃになりましたが、起動時に冷却システムでエラーが発生し、定期的に再起動しました。







検査中であっても、サーバーの電源がオンになり、自己診断インジケータとコンソールエラーを確認します。 重要なものがなければ、取引が行われ、車は当局にさらに転送されます。







以下で説明する一連のチェックは、最初から行われたわけではありません。スポイラーの下では、MTBFとその生産の負荷に関する情報はほとんどありません。

最初に、当社のエンジニアは、メインコンポーネントの故障に関する稼働時間の統計に興味を持ち、それを交換するだけでよいという事実に苦労しないようにしました。 信頼性の主な指標はMTBF （Mean Time Between Failures）、つまり障害が発生しない時間です。 コンポーネントごとに図が異なり、すべてのコンポーネントの公式データを簡単に取得できます。







ただし、参考のために、ブランドサーバーで鉄が使用されている一部のOEMメーカーのレポートを使用できます。 たとえば、 Intel 520 SSDでは、MTBFは1,200,000時間です。 もちろん、これはディスクが136年続くという意味ではありません。この特性は統計的であり、大規模なバッチをテストするときに表示されるためです。 式AFR = 1-exp（-8760 / MTBF）に従ってMTBFから派生したAFR（年間故障率）インジケータは、理解に便利です。







この例では、1年目のSSDドライブ障害の確率は約0.007、つまり0.7％です。 計算の精度を下げるには、式8760 / MTBFを使用します。 この指標の計算に関するトピックについては、かなり多くの記事がすでに書かれているので、好奇心people盛な人は公開された資料に目を向けることができます。







サーバーのメンテナンス



新しく到着したサーバーはすべて、必須のテストおよびクリーニングサイクルを通過します。 さらに、曲がった取り付け「耳」などの深刻な物理的欠陥が修復されます。







金属の傷や擦り傷など、非常に表面的なものはそのまま残ります。 ちなみに、サーバーシステムの金属は、工場で特別な帯電防止ワニスでコーティングされていますが、復元はそれほど簡単ではありません。 物質自体の組成は正確にはわかっていません-KFCの調味料のように-したがって、静電気保護に対する美学を犠牲にします。













いわゆる「耳」が損傷している場合は、サーバーをラックから簡単に押し出すことができるため、通常は新しい耳と交換されます。 HPプラスチックパーツの場合、電源のヒンジのように、単に新しいものに変更されます。 取り付けレールは再度注文するだけです。 ケース自体に重大な損傷（たとえば、深くて複雑なへこみ）があるため、新しいアセンブリに変更されるだけです。







サーバーモールエンジニアのすべての経験におけるIBMメタルマウントの損傷は、これまで見られませんでした。 どうやら、このメーカーのシステムのよく知られている「破壊不能性」は、小さなものでも現れています。







ちなみに、ケースの失敗の間隔は非常に長いです。

たとえば、メーカーの1社からのMTBFデータの例を次に示します。







  • ケース自体のインジケータは5,000,000時間です。
  • ディスクバスケットとIMPIモジュールは700,000時間動作します。
  • LEDは2,000,000時間設計されています。




数年の間タバコの煙で冷やされていたサーバーを売るという要求があった。 彼はちょうど隣の喫煙室から行われた空気取り入れ口であるサーバールームに立っていました。 タバコ含有製品の燃焼生成物の香りは、すでに患者への道で感じられました。 モデルは関連性があったので、チャンスをとることにしました。 たばこタールの平らな層を洗ったことがありますか？ そして、サーバーモールのエンジニアは洗濯をしました-喫煙をやめさえしました。 確かに、鉄はまだ販売されておらず、国内のニーズに使用されていました。

検査後、エンジニアはハウジングカバーを取り外し、車を始動してファン、電源、およびドライブの音の背景を聞きます。 診断システムにエラーを与えないクーラーもありますが、その音はベアリングの未来を信じません。 このようなクーラーは、新しいクーラーと簡単に交換されます。 Intel冷却システムのMTBFはわずか100,000時間であるため、ファンを新しいものに交換するのが一般的です。













同様に人気のある音は、電源システムのコンデンサーのきしみ音で、最後までモニターで緑色に点灯します。 比較的新しいサーバーは、ソリッドステートコンデンサーを備えた電源を使用しますが、電解セルを備えたモデルは依然として重要であるため、慎重な診断が必要です。







Intel OEMデータによると、最新の電源のMTBFは967,300時間になります 。 ホイッスルや疑わしい誤動作の場合、はんだ付け作業は経済的に実行可能ではなく、将来の買い手にとっては困難であるため、PSU全体が新しいものに変更されます。







光デジタル診断



最新のサーバーのほとんどは、自己診断システムを備えています。 フロントパネルのLEDインジケータ、すべてのコンポーネントのリストとステータスを備えた個別のモジュール、エラーの存在を示すポインタなどがあります。 いずれにせよ、コンポーネントの重大な問題がすぐにわかります。







IBM、HP、およびDellを例として使用した基本的な診断への小さな余談。
  • IBMバージョンはLight Pathと呼ばれ、インジケータと説明を含む引き出しパネルです。








  • ほとんどのサーバーのデルでは、簡単な説明を含む基本的なセットアップとエラー表示にLCDパネルを使用しています。








簡略化されたインジケータもあります。













  • HPEは、 Systems Insight Display LEDの自己診断を提供します。パネルはIBMバージョンに似ています。








インジケータをざっと見ると、標準の診断ツールを使用して長いプログラムチェックが開始されます。







  • IBM Dynamic System Analysis







  • デルのハードウェア診断







  • HPE Insight Diagnostics


これらのプログラムはすべて、ローカルで、またはIMM、DRAC、iLOを使用して起動されます。 診断がサーバーの管理コントローラーに「縫い付けられていない」場合、製造元の診断ディスクから起動します。 完全な診断には2〜3時間かかり、メモリ、プロセッサ、診断コントローラ、ファン、電源、およびディスクコントローラに関する問題のほとんどを検出します。 新しいドライブは販売中にほとんど常にインストールされるため、ハードドライブはプロセスに関与しません。







従来、マザーボードの弱点は電解コンデンサでした。 それらは膨張し、過熱し、爆発し、完全に動作しなくなりました。 最高温度条件では、このような要素のMTTFは8,000時間に達し、数年の運転後に予定外の修理につながる可能性がありました。 そのため、最新のサーバーシステムでは、いくつかのサーバーの「寿命」に十分な固体コンデンサが使用されます。 Intel S1200V3RPMの例のマザーボードの一般的なMTBFはこれを確認し、 371 523時間になります。













慎重なチェックの後、サーバーは完全に「テーブル上の裸のケースとアクセサリー」の状態に分解され、その後、すべてのコンポーネントが徹底的に洗浄され、アルコールで洗浄されます。 アルコールは、マザーボードの導電経路、要素ベース、およびワニスを傷つけないため、ボードをきれいな外観にするために広く使用されています。 オーバーヘッドを回避し、アルコール依存症の対策として、イソプロピルアルコールが使用されます。













マザーボード上のコネクタには細心の注意が払われています。 特に、エンジニアは、1本の損傷した脚でも最も予測不可能な結果を​​引き起こす可能性があるため、曲がったピンがないか虫眼鏡を通してプロセッサソケットを調べます。 PCIスロットとRAMは無視されず、ネットワークポートリンクがチェックされます。 「ケーキの上のチェリー」として、念のためにBIOSバッテリーを交換します。













入浴後、サーバーは倉庫に転送され、そこで内部倉庫ベースのバーコードがすべてのコンポーネントから読み取られます。 次に、鉄はバイヤーの棚でテストログとすべてのコンポーネントのシリアル番号が記載された保証シートとともに待機しています。







そして、このサーバーに注文が来ました



お客様が「現状のまま」の構成を選択し、何も追加したくない場合はまれです。 したがって、注文したアイロンには、新しいディスク、プロセッサ、特定の電力の電源、メモリ、および必要なコントローラが不足しています。 その後、サーバーは販売前の検証のためにテストエンジニアに再び引き渡されます。







使用されたツールのうち、サーバー製造元の組み込み診断ソフトウェアと、外部ドライブからのいくつかのユーティリティ。 事前販売チェックには約10時間かかり、ストレス下で実行されます。







  • プロセッサとメモリは最大容量で動作します。
  • 複数の電源がある場合でも、電源はすべての電力を供給します。
  • 負荷がかかると、ほとんどの欠陥のあるハードドライブが検出されます。
  • サーバーの要素ベース全体が機能するのは、日常の操作では機能しない可能性が高いためです。








ところで、この段階で、電源の「薄い」欠陥が検出されます。 そのため、サーバーモールでのホイッスルチェックの1つに限定されません。 同じ段階で、 フォールトトレラントオプションを使用する見込みがあるにもかかわらず、顧客が1台の電源でサーバーを購入することを決めた場合、無条件で電源を新しい電源に交換することができます 。







新しいハードドライブは、お客様が独自の理由で、開梱して送信するように求められた場合にのみテストされます。







すべてのネットワークインターフェイスを完全にチェックするには、Windows 2012R2に基づいて特別に準備された環境の外部ドライブからマシンを読み込みます。 サーバーはローカルネットワークに接続し、エンジニアは1つの大きなファイルと多くの小さなファイルのコピーを順番に開始します。 パケット損失が1％を超える場合、ネットワークカードは診断および交換の対象となります。







Memtestはさらに、IBMを除くすべてのシステムでメモリをテストします。 実際、IBMマシンでのMemtestチェックは、ほとんどの場合、スロットの1つで存在しないエラーを検出します。 これが技術的特徴です。













サーバーコンポーネントのいずれかで障害が発生すると、すべてのテストが新たに開始されるため、交換されたコンポーネントとの互換性の問題を回避できます。







DellサーバーのRAIDコントローラーで興味深い問題が浮上すると、すべてのテストは成功しましたが、再起動後、BIOSはかなりまれなH710コントローラーのエラーを表示し始めました。 同等の交換品を探すため、サーバーの出荷を1日間遅らせる必要がありましたが、2倍の帯域幅を持つ最新のH330アダプターと交換することで相殺されました。

合計、各サーバーには約16時間かかります。







  • 2-3時間の初期テスト;
  • 清掃と入浴に3時間。
  • 10時間は販売前テストになります。


すべてのテストに合格したハードウェアと一緒に、購入者は、テストログ、サーバーへの指示、有用なリンク、およびこの特定のメーカーの一般的なエラーに関する記事のオフラインバージョンを含むフラッシュドライブを受け取ります。













特別な言及は、送信のためのサーバーの準備に値します。 パッケージングは​​独自に開発され、レビューによると、元の品質を上回っています。 サーバーは、シリカゲル（吸湿剤）を含むフィルムで密封され、発泡ポリエチレンで包まれ、強力な段ボールに梱包されて顧客に送られます。







結論の代わりに



サーバーモールは、上記の方法で復元されたマシンに対して独自の3年間の保証を提供します。 さらに、標準のサービスセットには、数日以内に故障したコンポーネントを交換することと、重大な問題が発生した場合にサーバー全体を完全に交換することが含まれます。 以前の記事のいずれかで、 HP、IBM、およびDellのブランド提供と保証サポートおよびその違いについて詳しく知ることができます。







ちなみに、会社が存在する間、完全な交換は1回だけ必要でした。 グリッチは再現不可能であることが判明し、サーバーモールエンジニアの存在下ではすべてが時計のように機能しました。 ここに、管理者オーラが動作しています！
