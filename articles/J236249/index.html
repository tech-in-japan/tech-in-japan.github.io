記事の冒頭で、気まぐれなニューシャは、新しくインストールしたUbuntuをEFIモードでロードすることを拒否したと書きました。

/ bootパーティションの構造を調査する過程で、その理由を理解し、解決策を見つけました。

実際には、Nyushaはアドレス/EFI/BOOT/bootx64.efiでブートローダーを探しており、Ubuntuはアドレス/ EFI / ubuntu /にそれを配置し、grubx64.efiを呼び出します。

UPD：問題の説明のタイプミスを修正しました。 MithgolとGhoolに感謝

怠zyな解決策：

Live USBから起動し、コンソールを開いて、次を入力します。 

 $ sudo mount /dev/sda1 /mnt $ sudo mkdir /mnt/EFI/BOOT $ sudo cp /mnt/EFI/ubuntu/* /mnt/EFI/BOOT $ sudo mv /mnt/EFI/BOOT/grubx64.efi /mnt/EFI/BOOT/bootx64.efi $ sudo umount /dev/sda1 $ sudo reboot
