Microsoft Azureのネットワークの基本

火の主からのすべての温暖化の春の挨拶！



天気は雲一つありませんが、これは私のクラウドストーリーへの愛と欲求に影響しません。

実際、Microsoft Azureについての話を続けています。そして今日、クラウドでの通信に関する状況、つまり仮想ネットワークの運用の側面に焦点を当てたいと思います。



だから-ネットワーク相互作用の分野でMicrosoft Azureの機能の観点から謎解きプロセスを実行したいすべての人-私は猫をお願いします。







どんなネットワークがあり、何がありますか？



通信にはネットワークが必要です。これは明確で明白です。これでアメリカを発見したことはありません。 私たちの場合、ネットワークは仮想です。 また、原則として、Microsoft Azureでホストされる仮想マシン間の通信に使用されます。

これをアプリケーションの観点から見ると、それらは分散でき、Microsoft Azureが提供する別の仮想マシンまたはサービスに配置される可能性が高いさまざまなコンポーネントを含めることができます。これらはWebサーバーであり、SQL DBMS自体である可能性がありますアプリケーションサーバー。 そして、そのような仮想マシンのセットで表されるサービス自体はスケーラブルでなければなりません-クラウドについて話しているのです。 また、上記の仮想ネットワークインフラストラクチャは、Azure内のVM間の内部通信にのみ使用されるため、内部ネットワークです。 信頼性の高い使い慣れた通信のために必要なことは、AzureでDNSサーバーをホストすることだけです。 これを古い良い方法で行うことができます-VMを展開してその中で適切な役割を引き受けるか、Microsoft Azureインフラストラクチャコンポーネントのレベルでこのサービスを作成することで、より簡単で信頼性の高い方法で行うことができます。







サービスをハイブリッドにする必要がある場合、つまりローカルネットワークインフラストラクチャ（組織全体または特定のデバイスから直接）と対話する必要がある場合があります。 通常、このような接続にはVPNトンネルが使用されます（この場合も使用されます）。最初の場合はS2S-VPN（サイト間VPN）で、2つ目はP2S-VPN（ポイントツーサイトVPN）です。







ネットワーク間VPN接続については、ExpressRouteオプションもここで利用できます。これは、通信事業者が提供するMicrosoft Azureへの直接保証直接アクセスチャネルです。 このサービスはロシア連邦でも利用できます。



エンタープライズレベルの実際のコンピューターネットワークでは、VPNデバイスはWindows Server 2012R2に基づくソフトウェアであるだけでなく（それなしでも可能）、CiscoやJuniperなどのベンダーの特別なハードウェアソリューションに基づいており、Microsoft Azureはダウンロード機能を提供しますそのようなデバイスを構成するためのネットワークのパラメーターを含む特別なスクリプト。







また、ネットワーク構成はXMLファイルで記述できます。このような構成ファイルをMicrosoft Azureにインポートして、ネットワークを作成できます。



さて、Microsoft Azureネットワークスタックの最後のコンポーネントはトラフィックマネージャーです。 このコンポーネントは、ユーザーからMicrosoft Azureにあるサービスへのネットワーク負荷を最適化するために使用されます。 最適化オプションには、パフォーマンス、ラウンドロビン（ラウンドロビンとも呼ばれる）、フェイルオーバーの3つがあります。



さて、ここではすべてが従来のものであり、従来のインフラストラクチャで使用されるモデルと類似しています。 パフォーマンスとは、ユーザーからAzureでホストされるサービスへの広く高速なチャネルを意味します（ただし、ExpressRouteはこの点でより高速で信頼性が高くなります）。



これで、Azureのネットワークキャパシティ管理機能のレビューは終了です。 プラットフォームのネットワーク機能には、超自然的で異常なものは何もありません。 インフラストラクチャの利益のために使用し、問題を解決してください！
