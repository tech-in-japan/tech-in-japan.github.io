Djangoは、そのシンプルさと柔軟性の点で魅力的なフレームワークです。 ただし、多数の単純なサイトを迅速に構築するには、サイトの開発をストリームに配置する必要があります。 コンテンツ管理システムを開発するたびに疲れて、時間の損失で脅迫します。 プログラマーが既製のCMSに目を向けるのは、そのためです。



Drupal、Joomla！、Wordpressなど、PHPに基づくパイプラインWebプログラミングの巨人についてはほとんど誰もが知っていますが、Pythonの世界にはこれほど強力で成熟したツールがないことを誰もが知っているわけではありません。

教育プログラム





PythonでのWebアプリケーションの話は、すべてのCMSとフレームワークの父であるZopeから始めるべきです。 その歴史はシャギー1998年にさかのぼり、フレームワーク（「アプリケーションサーバー」）には独自のデータベースが含まれています。GuidoVan Rossum自身がプロジェクトの作業に参加し、フレームワークの要素とアーキテクチャソリューションは他の多くの大規模プロジェクト（主にZopeコンポーネントアーキテクチャ）で使用されています。



2番目に重要なプロジェクトは、Zopeに基づいたPlone CMS（1999年生まれ）と考えることができます。 Ploneはロシア連邦の少し西で広く知られています。 その周囲には、建築現場の小さな産業（Drupalでスピンするようなもの）さえも形成していました。



サーバーウェブの最後の波





近年、アジャイルWeb構築の3つのイベントが目立っています。MVCが到着し、phpが退屈になり、Rubyが実際に生まれ変わりました。



Ruby on Rails、Djangoは2005年に発表され、一般的なアイデアに焦点を当てました。Web上にMVCを配置する、MVC（またはDjangoで呼ばれるもの）を使用してWebを操作する。 この波で、以前はほとんど知られていないRubyが台頭し、Web開発におけるPythonの普及が加速しました。



そして、もちろん、なぜPythonとRubyがphpを実行するのかを覚えているだけでは価値がありません。



ただし、Pythonは汎用言語です。 さらに興味深いフレームワークがあります（Twistedだけが価値があります）。 物語はより豊かです。 したがって、私は個人的にPython（および、それに応じてDjango）を最初の補助スクリプト言語として選択し、その後、その豊富な可能性がすべて開かれ、主なものとして選択しました。

Python方式の最後の傾向は（必要ではない場合）、virtualenv環境の使用です。 これは、インタープリター自体、標準ツール、必要なライブラリなど、将来のプロジェクトのすべての依存関係がインストールされる「サンドボックス」です。 私たちの場合、DjangoとDjangoCMSです。



仮想環境であるcms-projectディレクトリを作成します。 



 ＃通常の人々がPythonで書いていないこと：
 sudo apt-get install python-setuptools python-dev build-essential python-pip python-virtualenv

Djangoのテンプレートは、cms_tagsが含まれているのが普通です。 メインタグは、プラグインが含まれる場所を示すプレースホルダーです。 その他の便利なタグ-show_menuとパンくずリストを使用すると、ナビゲーションメニューとパンくずリストをテンプレートに含めることができます。 補助要素の外観は、個別のテンプレートで簡単に調整できます。



プレースホルダータグでマークされた場所には、管理インターフェイスからプラグインを入力できます。 フォーマット済みのテキスト、リンク、ファイル、写真、Googleマップ、htmlの一部をページに含めるためのプラグインをすぐに使用できます。
