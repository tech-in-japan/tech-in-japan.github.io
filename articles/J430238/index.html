Kotlinの別のDSLまたはReactからPDFを印刷する方法





Reactで記述されたページを取得して印刷することはできません。ページセパレーター、入力フィールドがあります。 さらに、ReactDomとPDFに変換可能なプレーンHTMLの両方を生成するように、レンダリングを1回作成します。



最も難しいのは、Reactには独自のdslがあり、htmlには独自のdslがあることです。 この問題を解決するには？ 別のものを書いてください！



私はほとんど忘れていました、それはすべてKotlinで書かれているので、これは実際にはKotlin dslに関する記事です。



なぜ独自のウルクハイが必要なのですか？



私のプロジェクトには多くのレポートがあり、それらはすべて印刷可能でなければなりません。 これを行う方法にはいくつかのオプションがあります。



  • 印刷スタイルを試して、必要のないものをすべて隠し、すべてがうまくいくことを望みます。 ボタン、フィルターなどのみがそのまま印刷されます。 また、テーブルが多数ある場合は、それぞれが別々のページにある必要があります。 そして個人的に、サイトから印刷するときに出てくる追加されたリンク、日付などは私を怒らせています
  • PDFをレンダリングできる特殊な反応ライブラリを使用してみてください。 私はこれを見つけました、それはベータ版であり、その中で、通常の反応コンポーネントを再利用できないようです。
  • HTMLをキャンバスに変換し、PDFを作成します。 ただし、このためにはボタンなどのないHTMLが必要です。 後で印刷するには、非表示の要素でレンダリングする必要があります。 しかし、このオプションでは改ページを制御できるとは思えません。


最終的に、ReactDomとHTMLの両方を生成できるコードを書くことにしました。 途中で改ページに関する特別なタグを挿入して、HTMLをバックエンドに送信してPDFを印刷します。



Kotlinには、Reactを操作するためのタイプセーフなdslを提供するReactを操作するためのレイヤーライブラリがあります。 一般的にどのように見えるかは、以前の記事で見ることができます。



JetBrainsは、 HTMLを生成するためのライブラリも作成しました 。 クロスプラットフォーム、つまり JavaとJSの両方で使用できます。 これもdslで、構造が非常に似ています。



ReactDomまたは純粋なHTMLのどちらが必要かによって、ライブラリを切り替える方法を見つける必要があります。

まず、違いが何であるかを理解しましょう。



  1. htmlでは、 style



    とcolSpan



    、Reactのattrネストされたオブジェクトのトップレベルで割り当てられます。
  2. スタイルの塗り方は異なります。 HTMLでこれが文字列としての通常のcssである場合、Reactでは、JSの制限によりフィールド名が標準のcssとわずかに異なるjsオブジェクトです。
  3. Reactバージョンでは、検索に入力を使用し、HTMLでは単にテキストを表示します。 これはすでに問題のステートメントから始まっています。


まあ、最も重要なこと：これらは異なる消費者と異なるAPIを持つ異なるDSLです。 コンパイラーの場合、それらはまったく異なります。 それらを直接交差させることは不可能であるため、ほぼ同じように見えるが、React APIとHTML APIの両方で動作できるレイヤーを記述する必要があります。



スケルトンを組み立てる



とりあえず、1つの空のセルからタブレットを描画します。 



 table { thead { tr { th { } } } }


HTMLツリーと、それを処理する2つの方法があります。 従来のソリューションは、複合パターンと訪問者パターンを実装することです。 訪問者用のインターフェースはありません。 なぜ-それは後で見られます。



メインユニットはParentTagとTagWithParentです。 ParentTagはKotlin apiのHTMLタグによって生成され（HTMLとReact apiの両方で使用されます）、TagWithParentはタグ自体と2つのAPIバリアントの親に挿入する2つの関数を格納します。 



 abstract class ParentTag<T : HTMLTag> { val tags: MutableList<TagWithParent<*, T>> = mutableListOf() //     protected fun RDOMBuilder<T>.withChildren() { ... } //  reactAppender    protected fun T.withChildren() { ... } //  htmlAppender    } class TagWithParent<T, P : HTMLTag>( val tag: T, val htmlAppender: (T, P) -> Unit, val reactAppender: (T, RDOMBuilder<P>) -> Unit )


なぜそんなに多くのジェネリックが必要なのですか？ 問題は、コンパイル時にHTMLのdslが非常に厳しいことです。 Reactでdivからでもどこからでもtdを呼び出すことができる場合、HTMLの場合はtrのコンテキストからのみ呼び出すことができます。 したがって、どこにでもジェネリックの形でコンパイルするためにコンテキストをドラッグする必要があります。



ほとんどのタグはほぼ同じ方法で記述されます。



  1. 2つの訪問メソッドを実装します。 1つはReact用、もう1つはHTML用です。 最終的なレンダリングを担当します。 これらのメソッドは、スタイル、クラスなどを追加します。
  2. タグを親に挿入する拡張機能を作成します。


ここにtheadの例があります 
 class THead : ParentTag<THEAD>() { fun visit(builder: RDOMBuilder<TABLE>) { builder.thead { withChildren() } } fun visit(builder: TABLE) { builder.thead { withChildren() } } } fun Table.thead(block: THead.() -> Unit) { tags += TagWithParent(THead().also(block), THead::visit, THead::visit) }




最後に、訪問者用のインターフェースが使用されなかった理由を説明できます。 問題は、trをtheadとtbodyの両方に挿入できることです。 1つのインターフェイスのフレームワーク内でこれを表現できませんでした。 訪問機能の4つのオーバーロードが発生しました。



回避できない重複の束 
 class Tr( val classes: String? ) : ParentTag<TR>() { fun visit(builder: RDOMBuilder<THEAD>) { builder.tr(classes) { withChildren() } } fun visit(builder: THEAD) { builder.tr(classes) { withChildren() } } fun visit(builder: RDOMBuilder<TBODY>) { builder.tr(classes) { withChildren() } } fun visit(builder: TBODY) { builder.tr(classes) { withChildren() } } }

セルにテキストを追加します。

「+」を使用したフォーカスは非常に簡単です。テキストを含む可能性のあるタグでunaryPlusを再定義するだけです。 



 abstract class TableCell<T : HTMLTag> : ParentTag<T>() { operator fun String.unaryPlus() { ... } }


これにより、tdまたはthのコンテキストで '+'を呼び出すことができ、テキストを含むタグがツリーに追加されます。



皮をむく



ここで、HTMLが異なる場所とapiをリアクションする必要があります。 colSpanとの小さな違いはそれ自体で解決されますが、スタイルの形成の違いはより複雑です。 Reactで誰も知らない場合、スタイルはJSオブジェクトであり、フィールド名にハイフンを使用することはできません。 そのため、代わりにcamelCaseが使用されます。 HTML APIでは、通常のCSSが必要です。 再びこれとその両方が必要です。



自動的にcamelCaseをハイフネーションにし、React APIのようにそのままにしておくこともできますが、常に機能するかどうかはわかりません。 したがって、別のレイヤーを作成しました。

別の鎧を試着する



Reactで入力を描画する方法と、HTMLでテキストを描画する方法を学ぶことは残っています。 この構文を取得したい： 



 react { search(search, onChangeSearch) } html { +(search?:"") }


仕組み：反応関数は、Rreact APIのラムダを受け取り、挿入されたタグを返します。 タグで、中置関数を呼び出し、ラムダをHTML APIに渡すことができます。 中置修飾子を使用すると、htmlをドットなしで呼び出すことができます。 if {} else {}と非常に似ています。 また、if-elseの場合と同様に、html呼び出しはオプションであり、数回役に立ちました。
