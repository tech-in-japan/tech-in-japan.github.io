1日でiOS開発者向けにCIを上げる方法

プロジェクトを組み立てるには、開発者の面倒な操作が必要です。 マネージャーが進行状況について顧客に報告する必要がある場合、開発者は、リポジトリの取り出し、ライブラリのプルアップ、証明書のインストール、組み立て、確認、サーバーへのアセンブリの配置、およびその他の中間ステップに時間を費やします。 しかし、地球上では、自動化されない日常的なアクションがますます少なくなっています。 開発環境の進歩的な部分は継続的インテグレーション（CI）の方法を実践しており、 Live Typing社のiOS部門は、Jenkinsプラットフォームにアセンブリサーバーを展開することで参加することを決定しました。 それ以来、全く異なる生活が始まっています。



結果として得られたもの：

  1. サーバーは構築を開始します。

    1. マスターブランチへのプッシュの場合、ウェブフックにより;
    2. 目的のブランチと追加のスラックチャットでコマンドによって。 パラメータ。
  2. ユニットおよびUIテストを実行します。
  3. 次のメトリックを取得します。

    1. テストによるコードカバレッジ。
    2. コードの行数。
    3. コードの複製。
    4. 循環的コードの複雑さ。
  4. プロジェクトをipaにアーカイブしてから、それをアセンブリサーバー（独自の開発）に送信し、アセンブリリンクをslackに送信します。


今では、アセンブリを実行してそれを誰かに送信することを忘れて、多くの時間と労力を節約できます。 このマニュアルは、この感覚と経験を共有したいという願いの成果です。 しかし、他の多くのガイドのように、私たちのガイドも究極の真実であると主張していません。



開始するには、次の免責事項を読み、ガイドがタスクと互換性があるかどうかを理解してください。

  1. サーバーはiOS開発のニーズに合わせて展開されました。
  2. Jenkins自体を含むhomebrewを介したほとんどの補助プログラムのインストール（迅速な更新と使いやすさ）;
  3. すべてのビルドテストタスクにxcodebuilderを使用します（UIテストを実行できないため、xctoolの使用を拒否しました）。
  4. GitLabをリポジトリとして使用します。
  5. アセンブリの保管には、独自の設計のサーバーを使用します。 アセンブリごとに一意のURLが生成され、ブラウザでモバイルデバイスからリンクを開いて[インストール]をクリックするだけです。エンタープライズアカウントのおかげで、誰でも電話にアプリケーションをインストールできます。 サーバーへのファイルの送信に関連付けられたアクションの特異性のため、この段階はこの記事では説明されていません。
  6. すべてのプロジェクトでCocoaPods依存関係管理システムを使用しています。


マニュアルはかなり面倒であることが判明し、2つの部分に分けることにしました。 このパートでは、Jenkinsの基本的なインストールと構成に焦点を当てます。



必要なもの：

  1. OS XおよびXcodeがインストールされたMac（この場合、OS X 10.11.4を搭載した2011 MacBook Pro）。
  2. 数時間の空き時間。

2.必要なプログラムのインストール



Jenkinsをインストールするには、Homebrewパッケージ管理システムを使用します。 将来的には、コードメトリックの取得に使用する追加パッケージのインストールと更新のプロセスも簡素化します。

  1. Homebrewをインストールする 

      /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
  2. jenkinsをインストールします。 

      brew install jenkins
  3. cocoapods依存関係管理システムをインストールします。 

      sudo gem install -n /usr/local/bin cocoapods


サーバーをシステムの起動時に自動的に起動するには、launchdに対応するタスクの起動を設定する必要があります。 選択肢があります。LaunchAgentsまたはLaunchDaemonを使用します。 LaunchAgentを使用します。 これにより、Jenkinsとのさらなる作業が簡略化されます。 これを理解するには、下の表をご覧ください。

ただし、LaunchAgentの選択により、システムの起動時にログインユーザーが存在しないという問題を解決する必要があります。 これを行うには、自動ログインを構成します。 これを行う方法は1つしかありません。GUIを使用して（システム設定->ユーザーとグループ->ログイン設定->自動ログイン）。 シェルを介してこれを行う方法を誰かが知っている場合は、コメントで登録解除してください！



LaunchAgentsによる起動を設定するには、次の手順を実行します。

  1. デーモンをアンロードします（Jenkinsのインストール時に自動的に作成されました）： 

     sudo launchctl unload /Library/LaunchDaemons/homebrew.mxcl.jenkins.plist
  2. デーモンを削除します。 

     sudo rm /Library/LaunchDaemons/homebrew.mxcl.jenkins.plist
  3. エージェントを作成します。 

     cd /Users/jenkins/Library/LaunchAgents tap homebrew.mxcl.jenkins.plist
  4. vimエディターを使用してエージェントを構成します。 

     vim homebrew.mxcl.jenkins.plist


plistファイルの内容の例： 

 <?xml version="1.0" encoding="UTF-8"?> <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd"> <plist version="1.0"> <dict> <key>Label</key> <string>homebrew.mxcl.jenkins</string> <key>ProgramArguments</key> <array> <string>/usr/bin/java</string> <string>-Dmail.smtp.starttls.enable=true</string> <string>-jar</string> <string>/usr/local/opt/jenkins/libexec/jenkins.war</string> <string>—httpListenAddress=0.0.0.0</string> <string>--httpPort=8080</string> </array> <key>RunAtLoad</key> <true/> <key>UserName</key> <string>jenkins</string> </dict> </plist>


ここでは、値が0.0.0.0のhttpListenAddressフィールドと値が8080のhttpPortフィールドに注意する価値があります。そのため、サーバーは指定されたポートのIPアドレスを「リッスン」します。



思い出させてください：vimエディターでファイルを閉じて保存するには、次のように入力します：wq

インストール後、Jenkinsはデフォルトで127.0.0.1（localhost）で利用可能になります。

外部ネットワークからアクセスするために、ルーターのポートを転送できます。ジェンキンスWebインターフェースの場合は8080、sshアクセスの場合は22です。



3. Jenkinsのプラグインのインストール



Jenkinsサーバー-> Jenkinsの設定->プラグインの管理に移動します。 [利用可能]タブで、次のプラグインを見つけてインストールします。

  1. ロールベースの認可戦略-セキュリティ。 権利を分配してユーザーグループを作成できます。
  2. GitLabプラグインとGitlabフックプラグイン-gitlabで作業するためのプラグイン。
  3. Xcode統合-Xcodeとの統合。
  4. キーチェーンとプロビジョニングプロファイルの管理-プロビジョニングプロファイルの操作を容易にします。


4.基本的なJenkinsのセットアップ



Jenkinsのウェブインターフェースから、[設定]に移動してから[システム構成]セクションに移動します。 次のことに注意してください。

4.1。 Xcode Builderの設定。 Xcodeを標準ディレクトリにインストールした場合、設定を変更する必要はありません。 それ以外の場合は、指定したコンポーネントのパスを指定する必要があります。 LaunchAgentsを使用してサーバーを起動した場合、Xcodeはlogin.keychainにアクセスできます。 ただし、安全にプレイしたい場合は、スクリーンショットに示されているように、このセクションにキーチェーンを追加できます。







これで、GUIまたはlogin.keychainのシェルから必要な証明書をロードでき、Xcodeはビルド中に必要な証明書を自動的にプルアップします。



4.2。 SSHアクセスを構成するには、次の操作を行います。

  1. そうでない場合は、shhキーを生成します（例の説明はここにあります ）。
  2. CVSセクションにキーを追加します。







  3. GitLabにアクセスするには、Jenkins Webインターフェイスのメインページの[資格情報]セクションに移動します。 ここで、ユーザーの秘密キーを追加する必要があります（この場合、JenkinsはGitLabに登録されているユーザーです）。 次の例に示すように、直接挿入するか、パスを指定できます。







  4. GitLab自体の設定で、対応する公開キーを指定する必要があります。







4.3。 セキュリティを設定するには、ロールベースの認証戦略プラグインを使用します。 Jenkins Webインターフェースから設定に移動し、次にロールの管理と割り当てセクションに移動します。 ここで、ユーザーとグループに割り当てられるさまざまなロールを作成し、それらの特定の操作に対する権限を決定できます。 ここでは、誰もが自分の裁量ですべてを行います。 ただし、サーバーへの外部アクセスを構成している場合は、ゲストユーザーのすべての権限を削除することを強くお勧めします。



5.ジョブの作成とプロジェクトアセンブリのセットアップ



5.1。 Jenkins Webインターフェイスのメインページで、[アイテムの作成]を選択します。 次に、「無料構成でタスクを作成する」項目を選択し、プロジェクトの名前を入力します。

5.2。 ジョブ設定ページで最初に行うことは、「ソース管理」タブに移動して、GitLabからロードされるプロジェクトを構成することです。 ここで、プロジェクトのリポジトリアドレスを入力し、サーバーがGitLabにアクセスする資格情報と、プロジェクトのどのブランチを収集する必要があるかを指定する必要があります。







5.3。 次に、「アセンブリ」セクションに進みます。 CocoaPods依存関係管理システムを使用していて、Podファイルをgitに追加しない場合は、ハースのインストールを開始する「シェルコマンドの実行」アセンブリステップを追加する必要があります。 

 #!/bin/bash -l export LANG=UTF-8 pod install


ここでは、毎回ポッドをインストールしないように、インストール条件をラップできます。 例（ファイルが最後の60秒以内に更新された場合、実行...）： 

 if [ $(( $(date +"%s") - $(stat -f %m Podfile) )) -le 60 ]; then pod install fi


5.4。 ビルドステップの追加：Xcode。 CocoaPods依存関係管理システムを使用する場合は、[ターゲット]フィールドに値を指定するのではなく、[高度なXcodeビルドオプション]セクションで実行可能スキーマと.xcworkspaceファイルの名前を指定する必要があります。 最小構成の例を以下のスクリーンショットに示します。









プロジェクトの設定で、実行可能スキームを共有としてマークする必要があることに注意してください（コンテナはワークスペースまたはプロジェクトのいずれかです）。







6.証明書とプロビジョニングプロファイルを使用する



キーチェーンとプロビジョニングプロファイル管理プラグインのおかげで、プロビジョニングプロファイルのインストール作業を大幅に簡素化できます。



プロビジョニングプロファイルの追加：

  1. Jenkins設定に移動します。
  2. リストで[キーチェーンとプロビジョニングプロファイル管理]を選択します。
  3. [ファイルを選択]ボタンをクリックし、プロビジョニングプロファイルを見つけて[アップロード]をクリックします。


必要に応じて、「ビルド環境」セクションのジョブ設定で特定のプロビジョニングプロファイルを指定するには、モバイルプロビジョニングプロファイルフラグを設定し、ロードされたプロファイルのいずれかを選択する必要があります。







次に、Xcode設定で、スクリーンショットのようにカスタムxcodebuild引数を設定する必要があります。







GUIを使用して証明書をlogin.keychainキーチェーンにアップロードできます（証明書をクリックするだけです）が、これは常に可能とは限りません。 したがって、少し複雑なオプションを検討します-sshで追加します：



  1. Macに必要な証明書をダウンロードし、ダブルクリックしてローカルキーチェーンアクセスにインストールします。
  2. キーチェーンアクセスに移動し、必要な証明書を見つけて、キーを.p12形式でエクスポートします。
  3. 証明書をサーバーに転送します。 

     scp certificate.crt jenkins@server:/Users/jenkins
    証明書-証明書名;

    jenkins-ユーザー名;

    サーバー-サーバーアドレス。

    ：/ Users / jenkins-保存されたファイルのパス。

    （パラメーターを使用して、scp -P 20 certificate.crt jenkins @ server：/ Users / jenkinsの形式で目的のポートを指定できます）

  4. キーをサーバーに渡します。 
     scp privatekey.p12 jenkins@server:/Users/jenkins
  5. ssh経由でサーバーに接続します。 

     ssh jenkins@server
  6. キーチェーンへのアクセスを開きます： 

     security unlock-keychain -p password /Users/jenkins/Library/Keychains/login.keychain
  7. 証明書をインストールします。 

     security add-certificates ./certificate.crt
  8. キーをコピーします。 

     security import privatekey.p12 -k /Users/jenkins/Library/Keychains/login.keychain -P password -A


セキュリティコマンドの詳細については、 こちらをご覧ください 。



特定の証明書をジョブに追加する方法の詳細については、対応するプラグインのドキュメントを参照してください。



7.始める前に



これで、Jenkins Webインターフェースのメインページまたはプロジェクト自体のページの[Built Now]ボタンをクリックする準備ができました。 をクリックして、開始したアセンブリのコンソール出力セクションに移動します。 ログでは、エラーを含む多くの有用な情報を見つけることができます。

すべてを正しく行った場合、ログの最後にアセンブリの最後に表示されます：Finished：SUCCESS、およびJenkinsメインページのアセンブリ名の横にある青色のSUCCESSインジケータが点灯します。
