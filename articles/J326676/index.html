独立系開発会社のゲームコンテスト：12の開発者向けヒント





500万のモバイルアプリケーションのうち、ユーザーが偶然あなたのことに気付く可能性があります。 業界とプレイヤーの注目を集めるために、多くの小さなスタジオが、 Casual Connect Indie PrizeやDevGAMM Awardsなどのコンテストに参加しています 。



そのような競争に参加することは、初心者の開発者にとって良い経験です。 フィードバックだけでなく、プレイヤーや上級同僚の注目も集めます。 ただし、主要な競争の最終候補者になることは容易ではなく、勝つことはさらに困難です。 インディー開発者向けのAppodealコンテストで数百のゲームを評価した後、ゲームがファイナルに到達できない主な問題を特定しました。 可能であればこれを避けてください。



プレイヤーに多くの困難なルールを与えすぎないでください。



平均的なプレーヤーの場合、注意の集中度はそれほど高くありません（そして今、裁判官を想像してください-ある日、彼らは非常に短い時間で数十のゲームを評価する必要があります）。 したがって、ゲームの開始前にルールを読むのに1分以上かかる場合、ユーザーを失ってしまいます。



複雑にしないでください-ゲームが進むにつれて、ゲームのルールを徐々に説明してください。











Appodeal Indie Game Blast Off ContestのファイナリストであるThem Homeを持ち込み 、プレイヤーを徐々に紹介し、各レベルに1つのルールを追加します。



プレイヤーに負け犬のように感じさせないでください



そのため、ユーザーはゲームをダウンロードし、ルールを読み、「再生」ボタンをクリックしました。 2秒後、彼は「ゲームが終わった！」という碑文を見ました。 さらに2秒は同じです。 別の試み-何も変わらず、ゲームオーバー！ そして、命は残っていません。 このような障害が発生した後、ユーザーはあまりcontinue辱的であり続けることはありません。



プレイヤーに勝者のように感じさせてください！ バランスの複雑さ：ゲームプレイにある程度の努力をさせますが、過度ではありません。ゲームを続けることをお勧めします。



プレイヤーを待たせないでください



ゲーム開始時のビデオとイントロは素晴らしいアイデアです。 彼らは私たちを最新のものにし、主人公を代表し、第一印象を与えます。 スクリーンセーバーも面白い場合-素晴らしい、それはユーザーを楽しませ、長い間記憶されます。 ただし、30秒以上続く導入部分は悪い考えです。 ユーザーは、ハリウッドの大ヒット作ではなく、ゲームのためにここにいます。



特にゆっくり表示されたり消えたりするテキストでは、長いスクリーンセーバーを避けてください 。 さて、または少なくともそれをスキップするオプションを追加します。







コンテストのファイナリストの1人であるRunaway Duffyには、魅力的なスクリーンセーバーがあります。 そして、あなたはそれをスキップすることができます。



プレーヤーに広告を出さないでください



広告調停に従事している会社がどのようにそれを示すことを思いとどまらせるか見ることは奇妙であるか。 開発者が広告を通じてプロジェクトを収益化するときに気に入っていますが、これは正しく行われなければなりません。 アプリケーションを起動した直後に広告を表示することは最悪のトリックの1つであり、ユーザーにとって非常に迷惑です。



特に全画面動画などの広告を新しいプレーヤーにすぐに表示する誘惑に抵抗してください 。 彼らにゲームを楽しむ機会を与え、後で広告を残す。



もう1つのヒント：広告の頻度について考えます。 ゲームの1ラウンドに平均30秒かかる場合、それぞれの後に30秒のスキップしない動画を含める必要はありません。 10ラウンドに1回-それは良い比率です。 実験しますが、保持指標を追跡します。



迷惑なほどアプリ内購入を提供しないでください



あなたが店に行くとすぐに走り、完全に不必要な商品をすり込んで、彼らにそれを買うように促している売り手に出くわしましたか？ そうすれば、販売がそのように機能しないことを理解できます。 ゲームにアプリ内購入が含まれており、収益化の大部分を占めている場合は、他のゲームの仕組みに注意してください。



まず、ユーザーに過度の圧力をかけると、ユーザーは単に再生を停止します。 第二に、アプリ内購入は新しいプレーヤーでは機能しません-彼らは、彼らがゲームを気に入って、それに投資する準備ができると決めるまでお金を使いません。



新しいプレーヤーに購入を課す代わりに、ゲームがそれらをフックすることを確実にするように働きます 。 何が起こったのかを理解したら、すぐに店を開いてください。







エピソードシミュレーターnovellaは、ストーリーラインに基づいて収益化をインテリジェントに行います。 エピソードで買い物をすると、一度しか利用できない追加のプレイヤーストーリーが明らかになります。 これは、興味深いプロットのねじれを見逃さないように、今すぐ支払うことをお勧めします。



人気のあるゲームの別のコピーをコンテストに送信しないでください



あなたのゲームはユニークなものであってはなりませんが、有名なヒットの完全なコピーであってはなりません。 そのようなゲームが誤って勝ったとしても、一般の人がそれを知ったときに何が起こるか想像してみてください。 誰もが非常に恥ずかしいでしょう。 Candy Crushの成功を繰り返したいと思っているなら、少なくとも予想外の詳細をプロットに追加してください。



ゲームをオリジナルにしてみてください。 人気のストーリーにひねりを加えましょう！



無料のデザイン要素を使用しないでください。



数百のゲームをレビューした後、裁判官が互いに区別することは困難です。 最も記憶に残るものは通常、オリジナルのビジュアルスタイルを持ち、ゲームプレイで似ている2つのプロジェクトの中で、写真がより興味深いものの可能性が高くなります。



無料のパブリックビジュアル要素は、開発の初期段階では良いアイデアですが、長期的に見ればユニークなアートはより多くのメリットをもたらします。 アートがあなたの最強の側面ではない場合、ゲームをきちんとしていて最小限にしてください。 複雑だが設計が不十分な設計は、単純なミニマリズムよりも悪く見えます。



繰り返し要素を制限し、想像力を発揮します。 結局のところ、アーティストを雇う。







AppodealコンテストのファイナリストであるシューティングゲームLoony Quackは、おもしろいアヒルや鎌を持つ死後のアヒルなど、おもしろくて珍しいキャラクターのおかげですぐに思い出されます。



完璧になろう



プレゼンテーションは重要です。 コンテストの申請書を提出する前に、ゲームが最大限に磨かれていることを確認してください（意図的にレトロな外観を与えていない限り）。 いずれにせよ、テキストと説明のタイプミスとスペルをチェックしてください。 ゲームを評価できる高品質のスクリーンショットを準備し、サイトが機能しているかどうか（ある場合）、サイトへのリンクがアクティブかどうかを確認します。



はい、これらのことは明らかなように見えますが、壊れたアーカイブを送信したり、半死のサイトからゲームをダウンロードするためのリンクを提供したりする開発者の数は信じられません。



チュートリアルを忘れないでください



ゲームが非常にシンプルに思えても、最初にチュートリアルを追加し、後で修正する機能を追加します。 これはオプションの要素ですが、プレーヤーの生活を大幅に簡素化し、ユーザーを維持するのに適しています。 さらに、ユーザーはゲームをしばらく延期し、ルールを忘れることができます。 彼が簡単に覚えられるように助けてください。



チュートリアルを追加（短い！）ルールとゲームの目的をすぐに思い出すために。







Appodeal Indie Game Blast Off Contestの優勝者であるColorfill.ioには、ルール、ヒント、便利なヒントが記載されたチュートリアルがあります。 ゲームの開始時にスキップして、しばらくしてから再表示できます。



インターネットに依存しないでください



ゲームを開始するためにインターネットまたはサーバーへの接続が必要な場合は、事前にユーザーに通知してください。 理想的には、インターネットにアクセスせずにゲームをプレイするオプションを用意する必要があります。そうしないと、ユーザーは最初の画面の読み込みでスタックします。



さらに、ゲームをインストールした直後に追加要素のダウンロードを開始するのは、悪い形です。 プレイヤーにゲームを感じてもらい、興味を持たせます。 そして、追加のアイテムをダウンロードすることを申し出ます。



そしてもちろん、ゲームがすべての主要なデバイスモデルで動作することを確認してください。



インターネット接続がダウンしていてもゲームを利用できるようにします。



審査員のプロモーションコードを忘れないでください



ゲームに隠されたコンテンツがあり、ジャッジがそれにアクセスしたい場合、またはゲームの代金を支払う必要がある場合、コンテストジャッジに無料でアクセスできるプロモーションコードを与えることを忘れないでください。



そして同時に、このコードがどのように機能するのか、なぜそれが必要なのか、どのように、そしてなぜ裁判官がそれを使うべきなのかを明確かつ詳細に主催者に説明します。



良い説明は成功の鍵です



ゲームのわかりやすい短い説明は、すでに見た数百のプロジェクトからショートリストのファイナリストを選択する必要があるときに、審査員が要旨を思い出すのに役立ちます。



説明に重要な点、ゲームの興味深い記憶に残る詳細を含めます。



良い例：



「Loony Quackはシンプルなアニメーションアーケードゲームです。 卵を武器として、プレイヤーは適切なタイミングで画面をクリックして、狂ったアヒルの艦隊を倒さなければなりません。 このゲームは、「プレイしやすい、勝ちにくい」という原則に基づいて動作します。基本的なルール、直感的なインターフェイス、ますます複雑化するレベルです。 ゲーム内のすべてのアヒルは、ユニークなアニメーションとサウンドトラックを備えたキャッチーなキャラクターです。 それぞれに名前とストーリーがあります。」



これらの12ポイントに従うことは、競争で100％の勝利を保証するものではありませんが、それらを使用すると、ショートリストに参加する可能性が確実に高まり、したがってプレーヤーと業界への注意が高まります。 頑張って
