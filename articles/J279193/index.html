有望なプログラミング言語について読むべきこと





重要な情報から始めましょう。有望な言語は必ずしも新しいものではありません。 あなたの最愛のPythonやRubyほど古くなく、人気の高い上位10のランク （ GitHut 、 RedMonk 、 TIOBE Index ）のいずれにも属さず、これらの言語の知識は給与の大幅な増加を提供しません（Swiftを除く） 。 しかし、彼らを革命と呼び、選挙人の間で勢いを得ることも不可能です。 もちろん、独自のニッチ、特定の市場シェア（最大2％）、確立されたコミュニティがあります。 「ブラックスワン」のようなイベントが発生しない場合、将来的にはトップ5の月桂樹に侵入できるのは彼らだけです。



近年、言語の人気評価にほとんど変化はありません。一部は低下し、一部は上昇しますが、「ベスト・オブ・ザ・ベスト」というスローガンの下でトップに侵入する人はいません。 言語の新しい領域をキャプチャするプロセスは、ほとんどの場合何年もかかり、この間に必要な開発ベースが形になります。 そのため、レビューで昨夜書かれた排他的な言語や独自の言語はありません。単に彼らの時間がまだ到着していないからです。 しかし、今後数年のうちに、Java（すべて）、JavaScript、PHPなどの揺るぎない柱の人気は低下し、それらの場所は徐々に「新参者」に取って代わられるでしょう。



スイフト



プログラマーChris Luttnerは、2010年以来、誰もが自分のプログラミング言語に密かに取り組んでいます。開発開始から約1年半後、他のプログラマーが関与しましたが、Luttnerが勤務するAppleがプロジェクトに集中するまでにさらに1年かかりました。



2014年、AppleはOOP言語Swiftを導入しました。これは、Objective-Cを置き換える必要があります。これは、Objective-Cで記述されたアプリケーションを新しい言語に翻訳する機能によって促進されます。 主にiOSおよびOS X開発者向けに設計されたSwiftは、Objective-Cよりも安全で高速な言語と見なされています。



今読むもの







最高の言語の本は、Appleの公式出版物、 Swift for Programmersです。 著名な教師とプログラマーのハーヴェイ・デイテルとポール・デイテルによる本は特別なプレゼンテーションを必要としません-彼らは複雑な素材の簡単なプレゼンテーションで有名です。 「Swift for Programmers」には、独自のアプリケーションを作成するための基本を学ぶのに役立つ実際のプログラムの多くの例が含まれています。 この本の更新版は1年前にリリースされました。







2014年に出版された本「 初心者向けのSwift：開発と設計 」では、Swiftの基本について説明するだけでなく、Xcode環境も紹介しています。 この出版物は、少なくとも高水準言語のプログラミング知識が少なくともあり、iOS、OS X、またはwatchOS用のプログラムの作成方法を学びたい人に適しています。



ただし、すべての更新にもかかわらず、本は言語の変更により必然的に陳腐化するでしょう。 これに対処するには、いくつかの方法があります。たとえば、近い将来どの関連言語の本が出版されるかを事前に知ることです。



これから読むもの







Swift 2クックブックには、Swift 2でアプリケーションを作成するのに役立つ50以上のレシピが含まれています。チュートリアルでは、Xcodeのインストールの基本について説明し、Xcodeを使用してコードをSwift 1.2からSwift 2.0にアップグレードする例を示し、Xcodeがコードのテストにどのように役立つかを示します。 Objective-CまたはSwiftを最初に理解しておくと役立ちますが、Swiftの予備知識がなくても、この本に記載されている資料を学習できます。 この本は今年4月にリリースされます。



スカラ



Scalaは、安全で高速なプログラミングのために設計された強力なマルチパラダイムプログラミング言語です。 Scalaは関数型プログラミング機能を完全にサポートしています。 彼はアカデミックな環境の代表であり、そのために過度の複雑さで非難されることもあります。 この言語は2001年から2004年に作成されました。 ヨーロッパで最も権威のある教育機関の1つであるローザンヌ連邦工科大学で学んだ。 Scalaは、コンポーネントソフトウェアの改善された言語サポートの開発を目的とした調査の結果です。



今日、Scalaは多くの大規模プロジェクトで使用されています。たとえば、サーバー側のTwitterコードのほとんどはこの言語で書かれています。 Scalaは、Javaの設計で行われた多くの間違いを考慮に入れているため、Javaを置き換えることができます。



今読むもの







Programming in Scala：A Comprehensive Step-by-Step Guide、2nd Editionの著者であるMartin Oderskyは、Scalaの作成者です。 さらに、彼はJava（Generics）での汎用プログラミングのサポートを開発し、Javaコンパイラーであるjavacの現在のバージョンを作成しました。



イタリア語から翻訳すると、「scala」は「階段」を意味します。 Oderskyの本は、ある意味では登山階段に似ています。基本的な要素から始めて、言語の追加機能で終わるScalaをマスターします。



Scala in Depth （Joshua D. Suereth著）も役立つかもしれません。 もちろん、多くの面ですでに古くなっていますが、パターンとアルゴリズムを学習する場合は、資料の関連性を当てにすることができます。







「 Scala in Action 1st Edition 」-最新の教科書（2013）。 言語を拡張するツールの概要を提供します。 MongoDB 、 SBT 、 Scalaz 、 Squeryl 、 H2 、 jQuery UI 、 ScalaCheck 、 Specs 、 Akka 、 Lift 、DSLについて詳しく説明し、分解します。



これから読むもの







「 Scala High Performance Programming 」は、著者によると、高性能アプリケーションを作成するScalaの能力を調査した最初の本です。 この出版物は、Liftを知っているScala開発者や、Scalaへの切り替えに関心があるがJavaコードのパフォーマンスを放棄したくないJavaプログラマーに役立ちます。



本のリリースは6月に予定されています。







「 Scala Design Patterns 」は、Scalaプログラマ向けのもう1つの本で、知識を活用して実際のアプリケーションを作成するのに役立ちます。これは一種のクイックリファレンスガイドです。 この本では、設計パターンによく知られている業界標準を使用する可能性について説明し、選択したパターンを使用する必要がある時期と、より適切なものを探す時期を示しています。



ルア



Luaは、1993年にリオデジャネイロカトリック大学のスタッフによって開発されたスクリプト言語です。 Luaの機能はJavaScriptに近いですが、より強力で柔軟な設計が特徴です。 多くのゲーム（World of Warcraftを含む）のプログラミング言語として名声を得ていますが、ゲームだけでなく、他のファイルに保存されているデータの使用が必要な場合にも使用できます。



Luaは、世界で最も生産的なスクリプト言語の1つです。 Redis、nginx、Tarantool、OpenResなどのプロジェクトで使用されます。



今読むもの







古典を読んでください。 Luaのプログラミング （第3版）は、言語の作成者の一人であるRoberto Jeruzalimskyによって書かれた本です。 ゲーム開発者にとって、この出版物は参考書になるかもしれません。 Cとの対話に必要なスキルなど、Luaを使用するほぼすべての側面をカバーしています。



Luaを習得する最も簡単な方法の1つは、次のリンクです： Luaを15分で学ぶ 。



これから読むもの







残念ながら、近い将来、興味深い発表はありません。 比較的新しい（他のエディションと比較して）本が2015年に発行されました-“ Lua Game Development Cookbook ”。 この本の初心者のゲーム開発者にとって、単純な2Dゲームの概念を研究してから、最新の3Dプロジェクトに進むと役立ちます。 さらに、Luaのスクリプト機能を使用すると、ゲームを完全に制御できます。



ただし、この本はLuaの初心者向けではありません。カスタムのLuaモジュールを作成するには、C / C ++の知識が必要です。



行く



ロバート・グリースマー、ロブ・パイク、ケン・トンプソンは2007-2009年にGoを作成しました。 この言語は、大規模なソフトウェア開発の問題を解決するために作成されました。 Goは、開発プロセスをより効率的にする必要がありました。 よりアカデミックなプログラミング言語とは異なり、Goは余分なものをすべて削除しようとしました。 言語の簡潔さと素朴さのおかげで、学習は非常に簡単になります。



Goのアプリケーションは多様です。 そのため、VKontakteのプッシュ通知はGo言語で実装されています 。 Goでは、iOSおよびAndroid用のアプリケーションを作成し、独自の暗号通貨を作成することもできます。 一般に、Goを自分でどこで使用するかを決めることができます。特に、英語の知識がなくても学習できるため、Habréのハブには多くの翻訳と豊富な記事があります。



今読むもの







この本の著者の一人であるブライアン・カーニガンは、真のプログラミングの伝説です。 彼はUnixの共著者の一人であり、1978年にDennis Ritchieと共同執筆したThe C Programming Languageの著者であるditroffやcronを含む多くのUnixプログラムの作成者です。



現時点では、 Goプログラミング言語は最も興味深いGo言語の教科書です。 JavaScript、Ruby、Python、Java、またはC ++の少なくとも基本的な知識があれば、この言語を習得できます。 この本は、言語のニュアンス、最も重要なライブラリを網羅し、幅広い言語の使用法を示すGoコードの実際の例を何百も提示しています。



これから読むもの







Go Web Programmingの作者であるSau Sheong Changは、Ruby on Rails Webマッシュアッププロジェクト、Rubyでのインターネットアプリケーションのクローン作成、およびRubyに関するその他の重要な書籍をいくつか執筆しました。 シンガポールのHP Labsの研究所のディレクターと応用研究が、Rubyの後にGoでプロジェクトを書き始めた理由を知ることは、さらに興味深いことです。 Go Webプログラミングは、最新の設計原則を使用してGoでWebアプリケーションを作成する問題に対処します。 リリースは5月31日に予定されています。



さび



Rustは、2006年以降に開発されたマルチパラダイム、機能的、命令型手続き型、オブジェクト指向プログラミング言語です。2009年以来、Mozilla ResearchチームはRustに取り組んでいます。



Rustは、一定レベルの禁欲主義を維持しながら、コンパイル言語（C ++、機能では劣らない）のパフォーマンスとセキュリティを、動的言語（Python）の開発速度と組み合わせます。 大規模でオープンな開発者コミュニティの活動のおかげで、RustはC ++およびJavaの設計で犯した多くの間違いを修正しました。



今読むもの



さびは急速に変化し続けています。 トレーニング資料が数か月遅れると、知識が危険にさらされる可能性があります。 公式ドキュメント（ マニュアル （「The Rust Programming Language」とも呼ばれる）およびその他のドキュメント ）に精通していることが不可欠です。



これから読むもの







GNU EmacsとGNU GuileのマイナーであるProgramming Rustの作者であるJim Blandyは、Rustのユニークな長所について語っています。 この本は、多くの一般的なエラーを回避する明確で費用対効果の高いプログラミングの例を提供します。







Learning Rustは、C＃/ C ++開発者がパフォーマンスとメモリ管理を改善できるようにするもので、この12月にリリースされます。 この本は、大規模でスケーラブルなソフトウェアの開発におけるRustの機能を示し、変数、手順、メモリ管理などを含む言語の基本についても説明しています。 この本を読んだ後、Rustで複雑なアプリケーションを作成できます。



結論の代わりに



パラダイムシフトプロセスは非常に遅いため、お気に入りのプログラミング言語が完全に古くなるまで待たずに、避けられないものの準備をすぐに開始できます。 明日C ++を忘れる人はいませんが、今すぐ前進して新しいことを学ぶ必要があります。 選択した言語の利点は、ドライ理論に限らず、今日のプロジェクトで実際に使用できることです。 最も有望な言語の1つを自分で試して、結果に失望することはできません。
