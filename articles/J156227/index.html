マネージャーと開発者には「バス番号」という用語があります。 これは、プロジェクトがサポートできなくなるためにバスにぶつかる必要がある開発者の数です。 最悪の場合、バス番号は1です。 私たちのタスクは、バス番号を最大化することです。理想的には、それは開発者の数と等しくなければなりません。

開発者が同じスタイルでコードを書くためのガイドラインがあります。 内部リソースの1つで、開発者はコードのフォーマットに関するルールだけでなく、プロジェクトのエコシステムの機能（ファイルの保存場所と保存方法、プロジェクトの編成方法、コードを書くためのいくつかのルール。



プロジェクトで使用される最も重要なルールの1つは、jsDocを使用したコードの必須ドキュメントです（ http://code.google.com/p/jsdoc-toolkit/、https://developers.google.com/closure/compiler/docs / js-for-compiler ）。 jsDocを使用すると大きな利点があります。 まず、コードの可読性が向上します。 第二に、多くのIDEでは、jsDocを使用するプロジェクトでナビゲートするのが非常に便利です。 vimのような単純なテキストエディタを使用して自分の足を撃ちたい人は、jsDoc-toolkitに付属のコンソールユーティリティを使用して、ドキュメントをHTMLファイルにコンパイルできます。 そして最後に、jsDocを正しく作成するために、開発者は頭の中ですべてを整理する必要があります。

クライアントテンプレート


ほとんどのロジックはJavaScriptで実装されているため、クライアントテンプレートを積極的に使用しています。 最初は、underscore.jsライブラリー（ http://underscorejs.org/ ）に組み込まれたテンプレートエンジンを使用しましたが、それが不便であることをすぐに認識しました。 まず、テンプレートを別のファイルに保存したかった。次に、テンプレートをJavaScriptの文字列変数ではなく、vimのようなテキストエディターでのマークアップの作成が実行に似ているようにしたかった。



さらに、ユーザーのマシンのリソースが複雑な文字列変換で無駄にならないように、これらのテンプレートをプリコンパイルしておくといいと思いました。



インフラストラクチャ部門は、テンプレートを作成して支援を行いました。 JST-JavaScript Templatesと名付けました。 主な構文はunderscore.jsから残されていましたが、テンプレートがjst拡張子を持つ外部ファイルに保存され、プロジェクトのビルド段階でも自動的にJavaScript関数にコンパイルされるように作成されました。 コンパイルされたテンプレートは、通常のjsファイルとしてプロジェクトに接続されます。 



 <!-- path/to/template.jst --> <ul> <% elements.forEach(element, function(el, index) { %> <li><%= element.name %></li> <% }) %> </ul>

依存関係とプロジェクトのビルド


各JavaScriptクラスには独自の依存関係システムがあるため、必要なすべてのファイルを手作業で接続することはほとんど不可能であり、何らかの方法でコードを縮小する必要があります。 このような複雑なシステムでは、多数のファイルを作成する必要があるため、効果的なビルドシステムが必要です。 このシステムを作成する際に、インフラストラクチャ部門が再び私たちを助けました。



プロジェクトをビルドするサービスはmediageneratorと呼ばれ、開発者の1人によって作成されました。 このサービスは、特別なタグの使用に基づいてプロジェクト内の依存関係を収集します。



例に戻りましょう。 主にページ用のpage.jsファイル、base.jsおよびsearchform.jsファイルを含めるには、page.js内に依存関係を登録する必要があります。 これらは、jsDocの@requireタグに登録されます。 必要なファイルを接続するには、page.jsファイルに、たとえば@fileoverviewブロックのどこかに2つのタグを追加するだけで十分であることがわかります。 



 /** * @fileoverview Page controller * @author Igor Alexeenko (twitter.com/iamo0) * @require 'base.js' * @require 'searchform.js' */




それだけです デバッグモードでは、2つの<script>タグがページに追加され、必要なファイルが含まれます。 そして、プロダクションモードでは、メディアジェネレーターはこれら3つのファイルすべてを1つに接着して、ミニファイヤに渡します。 もちろん、彼は複雑な依存関係を追跡する方法を知っており、1つのファイルを何度も添付することはありません。



さらに、jst-templatesとSCSSをコンパイルし、写真をキャッシュし、プロジェクトを組み立てるすべての黒い作業を行うのはメディアジェネレーターです。 したがって、フロントエンド開発者は、手元のタスクにのみ集中できます。
