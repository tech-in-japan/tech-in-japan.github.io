VMwareはGPL違反で法廷に持ち込んだ

Software Freedom Conservancyは本日、クリストフ・ヘルウィグとVMware Corporationに対するハンブルグ地方裁判所の訴訟で訴訟の開始を発表しました。 「これは悲しいが、Hellwig and Software Freedom ConservancyがGPLv2ライセンスの条件を順守するようVMwareを説得するための継続的な取り組みにおける次のステップです」とプレスリリースは述べています。



VMwareがESXiハイパーバイザーに含めるLinuxおよびその他の無料プログラムで使用されるのはGPLv2ライセンスです。



ドイツのプログラマーChristoph Hellwigは、Linuxカーネルの主な開発者の1人（昨年2206のコミット ）であり、無料ライセンスで製品のコンプライアンスを達成しようとするLinuxカーネル開発者への法的支援プログラムの最初の参加者の1人です。



Christoph Hellwigは、2007年にVMwareによるGPL違反を公式に報告しています。 2011年、VMwareは、ESXiに同梱されているBusyBoxのバージョンのソースコードの提供を拒否しており、これはBusyBoxライセンス（GPLv2）に違反していることが判明しました。



この問題に関するVMwareとの交渉は2012年初めに開始され、2013年にも継続されましたが、効果はありませんでした。



Hellwigは、2012年の終わりに前述のSoftware Freedom Conservancyプログラムに参加し、VMwareが提供した非準拠のESXiリリースの分析を支援しました。 アメリカの企業によって提示されたこれらの資料の徹底的かつ長期にわたる研究は、別の事実を明らかにしました：現在のESXiハイパーバイザーは、ヘルウィッグが個人的に所有する多くの知的財産権に違反しています。



Hellwigの調査中、組織はVMwareとの交渉を続けました。 残念ながら、2014年に、彼女はGPLv2の条件にもかかわらず、Hellwigや他のLinux開発者のソースコードに基づいて作成されたプロプライエタリソフトウェアの配布を停止することを望みませんでした。 したがって、訴訟でヘルウィッグを支持する以外に選択肢はありませんでした。



Software Freedom Conservancyは、VMwareがHellwigおよび他の開発者のGPLv2コードとvmkernelと呼ばれる独自のコードを混合し、この汎用パッケージを配布したことを強調しています。



Software Freedom ConservancyとHellwigは参加者であるため、トライアルについてコメントしていませんが、適切な場合、ConservancyはこのケースのFAQページを更新します。



Software Freedom ConservancyのBradley M. Kuhn社長は、GPL違反の数は、この分野で20年間働いてきた20年間で大幅に増加したと指摘しました。 「フリーソフトウェアの個々の開発者や愛好家に見せるために、違反者を撃退する必要があります。コピーレフトを保存することは、ソースコードへのより良いアクセス、変更、改善、公開の権利を達成するための優れた戦略です。 これらの行動において、すべての人にコンサーバンシーを支援するようお願いします 。」



Grant Foundationの別のカーネル開発者であり、Linux Foundationの技術諮問委員会の議長も次のように付け加えました。「GPLライセンスはLinux開発の礎です。 誠実なコピーレフトの誠実さの原則は、Linuxの並外れた成功を大きく決定し、Linuxの恩恵を受ける企業の巨大なエコシステムを作成しました。 GPLの共有原則が無視されると、エコシステム全体が損なわれます。



Software Freedom Conservancyは、訴訟を最後の手段と見なし、残りの資金が使い果たされた場合にのみそのようなアクションをサポートします。
