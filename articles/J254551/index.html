第4世代SCADAシステム：AggreGate SCADA / HMI









この記事では、AggreGate IoTプラットフォームに基づいて開発された製品のストーリーを続けます。 AggreGate Network Manager製品の登場から2年後の2010年に、第4世代SCADAシステムであるAggreGate SCADA / HMIプロジェクトを開始しました。

第4世代SCADAとは何ですか？



英語版のウィキペディアの記事でこれについて説明しています。



  1. 第一世代のSCADAは、インターネットが普及する前に設計されたモノリシックシステムです。 現在、このようなシステムは使用されていません。
  2. 第2世代SCADAは、エンタープライズLAN上で動作するように設計されたシステムです。 彼らは、コントローラー、データ収集サーバー、管理サーバー、オペレーターのワークステーション間の通信にIPネットワークを使用することをすでに想定しています。
  3. 第3世代のSCADAアーキテクチャには、多数の個別の生産サイトやリモート監視施設など、地理的に分散したプロセス制御システムの管理および調整機能が含まれています。 SCADAの進化の頂点であった最近まで、第3世代システムには、モバイルブラウザーでHMIインターフェイスを実行する機能、「戦闘」サーバーで直接プロジェクトをリモートで編集する機能、サーバーを停止してプロジェクトファイルをコピーすることなくイノベーションをテストする機能が含まれていました。
  4. 最後に、 第4世代のSCADAは、モノのインターネットに適合させる必要があります。 これは、さらに高度な分散化と統合、つまり、SCADAサーバーとコントローラー間でアルゴリズムの実行ポイントを迅速に移動できることを意味します。SCADAサーバーとコントローラーは、本質的に統一され、同じ実行環境を持っている必要があります。 もう1つの必要な機能は、VPNを使用せずにセルラーおよび衛星ネットワークを介して機能することです。つまり、静的IPアドレスを持たないコントローラーがクラウドで実行されているSCADAサーバーに単独で接続する機能です。




もちろん、すべてのSCADAベンダーは製品を開発しており、製品は世代から世代へと徐々に移行しています。 さらに、古いシステムの開発は、多くの場合、産業オートメーションの変化する世界の概念にほとんど適合しない過去の開発との互換性を維持する必要性によって大きく妨げられます。



IoTプラットフォーム上に構築されたAggreGate SCADA / HMIは、第4世代SCADAシステムに典型的なすべての機能を継承しました。



  • Javaプラットフォーム上に構築されたシステムはLinuxで正常に動作し、組み込みシステムでSCADAカーネルを実行できます。 OEMパートナーの多くは、Raspberry Pi、BeagleBone Black、および同様の低コストマイコンに事前に展開されたシステムを出荷しています。 さらに、ほとんどの場合、IP通信およびシリアルポートだけでなく、コントローラーのその他の周辺機器（ディスクリートおよびアナログ入力など）にもSCADAコアにアクセスする可能性があります。
  • 同じシステムが従来のサーバーに展開され、集中データ収集を提供し、技術プロセスのニーモニックダイアグラムを処理します。 単一のアーキテクチャに基づいて、従来のサーバーは、分散AggreGateアーキテクチャ内のマイクロサーバー（PLC）とデータを交換するためのピアツーピア関係を確立します。
  • このシステムは、すべてのラインのプログラマブルコントローラーおよびTibboモジュールと完全に互換性があります。
  • 模倣図は、通常のPC、WindowsまたはLinuxを実行するタッチパネル、またはWebブラウザーで開くことができます。
  • システムには、「開発環境」や「ランタイム環境」という概念はありません。開発は、アクセス権を区別する役割ベースのシステムを考慮して、将来の戦闘サーバーで直接リモート接続を介して実行されます。 プロジェクト全体またはその一部を複製する方法は多数あります。 参照プロジェクトとレガシー製品を作成するプラットフォームの機能については、別の記事で説明します。
  • AggreGateプラットフォームは、M2M / IoTデバイスとの連携に重点を置いており、サーバーは、独立して接続するコントローラーと完全に連携できます。 私たちの用語では、そのようなコントローラーはエージェントと呼ばれます。




それでも、疑問が残る場合があります。なぜ別のSCADAを作成することにしたのですか？ この分野の世界市場は飽和状態にあり、ロシアには定評のあるメーカーがあります。



実際、AggreGateプラットフォーム上の「アドオン」の形のAggreGate SCADA / HMIは、技術的な観点から、プロセス制御システムに典型的なデータとベクター画像を収集するための単なるドライバーのセットです。 SCADAに必要なすべての機能はAggreGateプラットフォーム自体の一部です-ニーモニックダイアグラム（ウィジェット）のエディター、レポートエディター、イベントとアラームを管理するメカニズム、独自のタグをモデリングするシステム、フェイルセーフクラスター操作テクノロジー、ドライバー開発キット（DDK）を備えたSDK ）など。



したがって、SCADAシステムの作成への投資は、そのようなシステムをゼロから作成するのに比べてわずかでした-生産プロセスと建物の自動化のためのプロジェクトを実装するために、プロセス制御システム（Modbus、OPC、OPC UA、BACnet、DNP3の世界の標準プロトコルのドライバーを開発しましたなど）、数千の動的ベクトル画像を描きました。







SCADAシステムの標準機能に加えて、AggreGateプラットフォームは、この市場向けの多くの珍しい機能を製品に提供します。



  • リングデータベース（RRD）およびNoSQLクラスデータベース（BigData）の統計の保存
  • AggreGate分散アーキテクチャによる無制限の水平および垂直スケーリング
  • データを収集し、IT管理および監視プロトコル（SNMP、FTP、JMX、SSH、WMIなど）とユニバーサルプロトコル（SQL、SOAP、CORBA、LDAPなど）の両方を管理する機能




これらの機能を使用すると、SCADAソリューションの古典ではないさまざまなプロジェクトでシステムを使用できます。 特に、AggreGate SCADA / HMIは、製造会社の輸送車両の遠隔測定、MESシステムの交換、セルタワーの監視、およびデータセンターのエンジニアリングインフラストラクチャの監視（AggreGate Data Center Supervisor製品の一部として）に使用されます。



アーキテクチャと設計コンセプトの点では、AggreGate SCADA / HMIは他のほとんどの製品と類似しています。 典型的なプロジェクトの開発サイクルには以下が含まれます。



  • フェールオーバー構成でのサーバーまたは複数のサーバーの展開
  • 通常のリレーショナルDBMSであるか、Apache Cassandra DBMSシステムに統合されているストレージを接続します。これにより、毎秒数万のタグのストレージが保証されます。
  • コントローラと他のデータソース（外部データベースなど）を接続し、ポーリングタグの期間を設定する
  • サーバーでタグを処理するための自動化されたアルゴリズムの設定-これらは、追加の計算タグ、電子メールおよびSMSアラートを配信するアラーム、さまざまなタスクのスケジュールなどを定義するモデルになります。
  • 模式図とそれらの間のナビゲーションを備えたダッシュボードの開発
  • ユーザーロール、それらのアクセス権、および必要に応じてLDAP / ADを介した外部認証の構成








Linuxで実行されているAggreGateサーバーには、Windowsで実行されているOPCサーバーからデータを収集する機能があります。 このプロセスは、DCOMプロトコルを使用してIPネットワークを介して実行されるため、SCADAサーバーとOPCサーバーを同じコンピューターにインストールする必要はなくなりました。



AggreGate SCADA / HMIには、「プロジェクト」、「開発環境」、「ランタイム環境」の概念はありません。 彼女のコンセプトによれば、1つの「プライマリ」サーバーが生産施設で使用されます。 実装の初期段階では、システムエンジニアはローカルまたはリモートでサーバーに接続して、模式図の作成、PLCアカウントの作成、データウェアハウスの構成などを行うことができます。初期セットアップ段階が完了すると、試運転と通常の運用で同じサーバーが使用されます。ただし、システムを別のサーバーに転送することは可能であり、簡単です。



統合された環境では、操作を中断することなく「戦闘」システムに変更を加えることができます。 これを行うには、次のものが必要です。



  • 1つ以上のシステムコンポーネントの一時的なコピーを作成します（たとえば、図やアラームを模倣する）
  • コピーに変更を加えてテストします
  • 元のコンポーネントを正常に変更されたコピーに置き換えます




SCADAシステムの最も重要なコンポーネントの1つは、ニーモニックエディターです。 AggreGateプラットフォームから継承したウィジェットエディター（GUIビルダー）を使用すると、単純なコンポーネント（ボタン、ラベル、テキストフィールド、リストなど）と複雑なコンポーネント（テーブル）の両方を含む模倣図とヒューマンマシンインターフェイスをすばやく描画して「アニメーション化」できます、多層パネル、タブ付きパネル、図、地理マップ、動的SVG画像、ビデオウィンドウなど）。







他のシステムに典型的なエディターに外見的に似ているにもかかわらず、AggreGateエディターには1つの重要な機能があります。 ビジュアルコンポーネントの古典的な絶対配置に加えて、どのパネルでも、HTMLのテーブルに似たコンポーネントのグリッドレイアウトを使用できます。 さらに、ネストされたパネルが多数ある複雑な形状（通常、マルチレイヤー、ブックマーク、分割パネル）の場合、各パネルは他のパネルに関係なく絶対レイアウトとグリッドレイアウトの両方を使用できます。



グリッドの配置により、ニーモニックダイアグラム、データ入力フォーム、および任意の画面解像度に有機的に適応するダッシュボードを設計できます。 絶対配置の場合、高さの増加に伴うすべてのコンポーネントの比例スケーリングが適応に適用され、ほとんどすべてのフォームおよびダイアログで受け入れられない結果になります。



模倣図は、いわゆるバインディングを使用してアニメーション化されます 。これにより、サーバーイベントとニーモニックイベントに応じて、サーバーオブジェクトのプロパティとビジュアルコンポーネントのプロパティ間でデータをコピーできます。 AggreGate式言語は、数字、文字列、日付と時刻、テーブルなどの処理に関する操作をコピーされたデータに適用するのに役立ちます。



AggreGateによって処理されたデータはすべて、レポートの作成に使用できます。 式エディタとSQLに似たクエリの組み込み言語は、必要なインジケータを抽出するのに役立ち、システム自体が視覚的なプレゼンテーションに最適なテンプレートを作成します。 その後、レポートエディターを使用して、必要に応じて構成できます。



重要なパラメーターについては、オブジェクトの重大な状態が検出されたとき、または特定のイベントチェーンが受信されたときにトリガーされるアラームを設定できます。 このシステムを使用すると、ほとんどすべての方法でアラームに関するアラートを送信できます（ポップアップ、サウンドアラート、電子メールメッセージ、SMSメッセージなど）。 アラームによって自動的にトリガーされるアクションは、自律的にもオペレーターの制御下でも機能します。 アラームモジュールは、検出、ヒステリシス、優先順位付け、確認、エスカレーションなどの典型的なACS TP機能の残りもサポートします。



AggreGate SCADA / HMIは、生産プロセスの自動化、必要なすべての情報のコールセンターへの持ち込み、視覚化の整理、作業情報のデータベースへの保存、およびレポートドキュメントの作成を支援します。 このシステムにより、プロセスの有効性に関する情報を迅速に分析し、その最適化に関する重要な決定を下すことができます。つまり、ソフトウェアクラスManufacturing Execution System（MES）の能力領域を部分的にカバーします。







大企業では、原則として、いくつかの異なるSCADAインストールが同時に動作しています。 それらのそれぞれは、別のグループのオペレーターの管理下で、任意のワークショップでその機能を実行します。 システムは特定の生産チェーンによって論理的に相互接続されているため、相互の統合と、パフォーマンスおよび出力インジケーターのMES / ERPレベルへの自動転送の両方が必要です。 AggreGateエコシステムでは、これは、 分散アーキテクチャを使用してサーバー間で単一のデータモデルの一部を共有することにより実現されます。



多くの場合、SCADAだけでなく、ITインフラストラクチャの管理、自動化、アクセス制御と物理セキュリティ管理、ASKUEなどのシステムを、同じ施設または同じプロジェクト内で異なる組み合わせと異なる構成で使用する必要がある場合があります。 AggreGateに基づいた他のソリューションを使用すると、同じインストール内でこれらの機能を実装でき、多くの場合、1つのサーバーでいくつかのモジュールを組み合わせることもできます。 これは実際にはどこで発生しますか？ たとえば、コアのアクティブなネットワーク機器、気候センサー、UPS、ディーゼル発電機、エアコン、水冷システム、人員のアクセス、従業員の労働時間を監視する必要があるデータセンター。 別の例：輸送ネットワークおよびセクターアンテナの無線中継機器、気候センサー、エアコン、UPS、セキュリティセンサーなどのパラメーターの監視を必要とするセルタワー。 大規模な倉庫では、人員のアクセス、ローダーの移動、換気、照明システムの監視が必要です。 ほとんどすべての大規模施設は、さまざまな管理システムと監視システムの組み合わせの恩恵を受けることができます。
