レイアウトの6つの原則

これらの考慮事項は、レイアウトの品質の評価に関するakellaの投稿に触発されました 。 この投稿とそれに関するかなりの数のコメントを読んだ後、私はページをレイアウトするときに従う基準（「そこにある何かの10のルール」- そのような見出しを言う -これは非常によく、ブログを「ねじる」のに役立ちます）を作成することにしました。



また、HTML + CSS、およびWeb用のプログラミングは、私にとってはちょっとした趣味であることに注意してください。 私は（これまで？）「メガ」プロジェクトを実装しませんでした。 たぶんそれが、この業界での私のすべての仕事がこれらの基準のほとんどすべてを満たした理由です。



したがって、私が従うHTMLレイアウトには6つの単純な原則があります。



1.コードのセマンティクスと「明度」



コードを意味的にオーバーロードしないようにします。 コードとデザインの両方でシンプルさとエレガントさを好みます。 不要なエンティティを使用せずにデザイナーのアイデアを実装することに興味があります。 これが常にうまくいくとは限らないことは明らかであり、 CSS Zen Gardenでさえ、デザイナーの幻想を実現するために特別に予約されたいくつかのdivがありますが、そのようなことは規則よりもむしろ例外であるように思えます。



2.標準



私は標準を維持しようとしている人です。 これは特に私の職業歴に当てはまります。 この点で、私は最も有効なコードを達成しようとします-少なくとも検証エラーを回避しようとします。 最も興味深いのは、多くのサイト（ Firefox + Html Validatorのおかげ）でエラーと警告を見ると、それらのほとんどは開発者の怠のために作成されたと結論付けていることです。



3. CSSとJavascriptを無効にした場合のパフォーマンス



これは重要な原則の1つです。 そして、ここでのポイントは、 lynxなどのブラウザーでサイトを使用できるようにすることだけではありません。 主な理由は、障害のある人がサイトにアクセスできるようにすることです。



4.アクセシビリティ



多くのサイト（IMHO）は、たとえば目の不自由なユーザーによる訪問には絶対に適していません。 そして、インターネットでの生活を少し楽にすることはそれほど難しくありません-ブロックを正しく配置し、写真やリンクに署名する-これはプロのタイプセッターやプログラマーが提供する必要のあるものの中で最も少ないと思います。 ところで、最近、私は視覚障害者がそれをどのように「見る」かを見るためにある種のブラウザをインストールすることを考えています。



5.クロスブラウザ



私は「キツネ」の下でタイプセットしますが、レイアウトはIE、Opera、Safari Winでチェックされます...時々lynxを試してみてください:)



6.テクノロジー



Webの開発方法が好きで、サポートしています。 私はそれに固執し、コードにマイクロフォーマットのようなものを実装しようとします。



PS



もちろん、私は現実主義者であり、すべての規則と指示を無条件に守ることは常に可能とはほど遠いことを理解していますが、私の意見では、すべての専門家は彼がしていることと仕事を適切に行うことを愛すべきであると考えています。 また、「オーク」インターネットの時代は尽きつつあり、その見返りに、仕事を愛する専門家によって作られた「高品質」インターネットが来ると信じています。
