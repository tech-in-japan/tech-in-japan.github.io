YII2からFirebirdデータベースにアクセスする

そのため、ユーザーインターフェイスをDelphi + Firebirdの束からWEBに転送する必要がありました。 プログラム複合体の作業を停止しないために、フォームを徐々に転送することが決定されました。そのためには、Firebird DBMSの作業を保存し、Webアプリケーションから接続する必要がありました。 Webアプリケーションを開発するとき、YII2フレームワークを使用することにしました。 開発段階では、本格的なHTTPサーバー、Denwer、XAMMP、または他のものが正常に機能する必要はありません。 XAMMPを選んだ理由は デンバーでは、PHPバージョンは長い間更新されていません。自分で手作業でやる気はありません。抵抗を最小限に抑えています。 最初は、XAMMP PHPでもYII2でもFirebird DBMSで作業することはできませんが、タンバリンを使用した特定のダンスでは、これを行うことはかなり可能です。 したがって、Windows 7 x64 、XAMMPおよびYII2、Firebird 2.5がすでにインストールされていると仮定します。



パッケージをインストールする方法（自分で適切な修正を入力してください）：



  • Firebird 2.5 x64バージョン （サーバーの種類は関係ありません） C：\ Program Files \ Firebird \ Firebird_2_5 \
  • XAMMPはDにインストールされます：\ xampp \
  • YII2はDにインストールされます：\ xampp \ htdocs \ yii \
  • データベースはパスDにあります：\ WORK \ database \ TBOT.GDB


これで開始できます。



Firebird Support for PHPをインストールします



Firebirdサポートをインストールするには、XAMMPでApacheサーバーを停止する必要があります。 これを行うには、XAMMPコンソールを開き、スタートボタンがApacheサービスの反対側にあることを確認します。 サービスが実行されていない場合は、「停止」をクリックしてください。







構成ファイルPHP D：\ xampp \ php \ php.iniをエディターで開き、その中の行を見つけます。 



;extension=php_pdo_firebird.dll


「;」を削除します 最初に コメントを外して保存します。 DBMSにアクセスするためにYII2が使用するPDOドライバーが有効になっています。 ただし、機能しません。標準のデータベースアクセスライブラリfbclient.dllをコピーする必要があります。 ここでは注意する必要があります Windows x64およびFirebird x64サーバーをインストールしました。XAMMPは32ビットアプリケーションです。 したがって、 fbclient.dllライブラリは、データベースへのアクセスに使用されるクライアント（この場合はXAMMP、またはXAMMPの一部としてのPHP）のバージョン用にコンパイルする必要があります。 ライブラリの32ビットバージョンは、FirebirdインストールディレクトリのWOW64ディレクトリにあります。 ファイルC：\ Program Files \ Firebird \ Firebird_2_5 \ WOW64 \ fbclient.dllを取得し、次のパスにコピーします。



  • PHP作業ディレクトリD：\ xampp \ php \
  • SYSTEM32 Cの場合：\ Windows \ System32 \ 。 理論的には、ここでコピーする必要はありませんが、他のプロジェクトでライブラリを使用します。システム全体でコピーを作成しないように、ここでコピーします。


D：\ xampp \ apache \ bin \へのコピーを勧める情報源もありますが、私の場合は何の影響もありません。 はい、そして、理論的には、影響を与えることができませんでした ApacheサーバーはPHPを実行し、彼自身は追加のアクセスライブラリを必要としません。



また、FirebirdにアクセスするためのPDOライブラリ、ファイルD：\ xampp \ php \ ext \ php_pdo_firebird.dllの可用性を確認することをお勧めします 。



Apacheサーバーを起動し（上の図を参照）、XAMMPコンソールで[開始]をクリックします。 現時点では、FirebirdのPDOドライバーがシステムに表示されていることを確認するのは良いことです。ドライバーが接続されていないことに気付くまで1日を過ごしました。 YII2エラー処理システムは、この問題の弱いヘルパーであることが判明しました。 別の場所で別のエラーを報告しました。 Model Generator giiのテーブルのリストは読み込まれず、その前にFirebirdにアクセスするための拡張機能のインストール中にエラーが発生したため、エラーを未接続のPDOドライバーに関連付けることは明らかではありませんでした。



YII2 Dの作業ディレクトリにtest-fb.phpファイルを作成します ：\ xampp \ htdocs \ yii \ framework 、その中にコードがあります： 



 <?php foreach (PDO::getAvailableDrivers() as $drv) { echo $drv."<br/>"; } ?>


このコードは、PHPに接続されたPDOドライバーをリストします。 ブラウザーを起動し、アドレスバーにlocalhost / yii / test-fb.phpと入力し 、それに応答して、接続されたドライバーのリストを含むページを取得します。

リストにfirebirdドライバーがあることを確認してください。 MySQLおよびSQLiteドライバーは、デフォルトで常に接続されています。



FirebirdにアクセスするためのYII2拡張機能のインストール



YII2でFirebirdに接続するための2つの拡張機能があります。



  • yii2-firebirddb by Sergey Rusakov 。 説明から判断すると、YII1からの拡張の適応です。 欠点は、 安定したアセンブリではなく開発アセンブリであるため、将来、 composer.jsonの 「最小安定性」を変更する必要があり、 Composerを使用してアップグレードするときに他のYII2拡張機能の不安定なバージョンが読み込まれることです。
  • エドガード・ロレイン・メシアスによるyii2-firebird 。 この拡張のソースと同じバグから判断すると、これはSergey Rusakovの yii2-firebirddbであり 、devブランチからstableブランチに転送されます。


両方の拡張機能のインストールを順番に説明します。 yii2-firebirdの 2番目の拡張機能から始めます。 安定した状態です。



yii2-firebirdをインストールする


ファイルD：\ xampp \ htdocs \ yii \ composer.jsonを開き、その中のコードを見つけます。 



 ... "require": { "php": ">=5.4.0", "yiisoft/yii2": ">=2.0.5", "yiisoft/yii2-bootstrap": "*", "yiisoft/yii2-swiftmailer": "*" ...


最後に、コンマ行で区切られた「edgardmessias / yii2-firebird」： "*"を追加します。 



 ... "require": { "php": ">=5.4.0", "yiisoft/yii2": ">=2.0.5", "yiisoft/yii2-bootstrap": "*", "yiisoft/yii2-swiftmailer": "*", "edgardmessias/yii2-firebird": "*" ...


追加された行の後、カンマは次のようにしないでください。 配列の最後に追加され、 Composerの実行時にコンマが存在するとエラーが発生します。 PHP自体では、配列要素のリストの末尾の閉じ括弧の前にコンマを使用できますが、 Composerは使用できません。



そのため、 Composerにyii2-firebird拡張機能をダウンロードする必要があることを指摘しました。 コマンドラインを起動し、現在のディレクトリをD：\ xampp \ htdocs \ yii \にして、2つのコマンドを続けて指定します。



composer update

composer install







Composerの出力のエラーを注意深く調べ、ディレクトリD：\ xampp \ htdocs \ yii \ vendor \ edgardmessias \ yii2-firebird \の存在を確認します。 拡張機能がインストールされました。



データベースへのアクセスを設定します。 yii2-firebirdのドキュメントには、設定を変更する必要があると書かれています。 設定ファイルD：\ xampp \ htdocs \ yii \ config \ web.phpを注意深く調べると、次の行（約40行目）が見つかります。 



 'db' => require(__DIR__ . '/db.php'),


ファイルD：\ xampp \ htdocs \ yii \ config \ db.phpでデータベース接続設定を編集する必要があることが明らかになります。 私たちはそれを開いてフォームに持ってきます： 



 <?php return [ 'class' => 'edgardmessias\db\firebird\Connection', 'dsn' => 'firebird:dbname=localhost:D:/WORK/database/TBOT.GDB;charset=WIN1251', 'username' => 'SYSDBA', 'password' => 'masterkey' ];


いくつかの点に注意を払いたい：



  • 構成の最初の行は、名前空間を持つConnectionクラスを示しています
  • 行'dsn'には firebirdアクセスドライバーが示されており、データベースのエンコードも行'dsn'に示されている必要があります。
  • データベースへのパスはスラッシュを介して指定されますが、Windowsではバックスラッシュを介してパスを指定するのが一般的です。 実際、どのように表示するかに違いはありません。 D：\ xampp \ htdocs \ yii \ vendor \ edgardmessias \ yii2-firebird \ src \ PdoAdapter.phpクラスコンストラクター（行37）にコードがあります： 



     ... public function __construct($dsn, $username, $password, $driver_options = []) { // Windows OS paths with backslashes should be changed $dsn = str_replace("\\", "/", $dsn); ...


    つまり dsn行は常にバックスラッシュを行に置き換えます。
  • バックスラッシュを使用するときは、PHPで文字をエスケープすることを忘れないでください。バックスラッシュを2回繰り返す必要があります。 つまり dsn行は次のようになります。 



     'dsn' => 'firebird:dbname=localhost:D:\\WORK\\database\\TBOT.GDB;charset=WIN1251',


  • ローカルデータベースに接続する場合、 dsnでlocalhostを指定する必要はありません。TCP/ IPを使用して外部サーバーに接続する方法を示しただけです。

ヘルスチェック


拡張機能、ドライバー、およびデータベース接続パラメーターの正しいインストールを確認する最も簡単な方法は、接続することです。 YII2からgiiユーティリティを起動し、 Model Generatorを開きます。 gii httpを開くためのリンク：// localhost / yii / web /？R = gii 、 Model Generatorセクションの[Start]ボタンを選択するか、直接リンクhttp：//localhost/yii/web/index.php？R = giiを使用します/ default / view＆id = model

Model Generatorがエラーなしで開く場合、すべてがインストールされており、正常に動作しています。 Model Generatorは、データベーステーブルのリストを読み込みます。 Firebirdに接続します。 ページの読み込みエラーが発生した場合、ほとんどの場合、データベースにテーブルがなく、テーブルのリストは空で返されます。 少なくとも1つのテーブルを作成して、再試行してください。 また、構成内のデータベースへの正しいパスを確認します（ファイルD：\ xampp \ htdocs \ yii \ config \ db.php ）。



おわりに



  1. 拡張機能の動作に対するアプローチの違いに注意してください。 yii2-firebirddb拡張機能は、標準のYII2名前空間のConnectionクラスで機能しますが、独自のPDOドライバーを使用してFirebirdにアクセスします。 yii2-firebird拡張機能は、標準の接続を独自に拡張します。
  2. 両方の拡張機能はBLOBフィールドでは機能しません。 このバグの説明はhttps://bugs.php.net/bug.php?id=61183にあります 。 これは私にはあまり関係がないので、この問題に対処しませんでした。blobフィールドをvarcharに変換するビューをデータベースに作成することを提案します。 さらに、 varcharは最大32Kbのテキストサイズを許可します。 メディア（写真、ビューなど）の場合、これでは十分ではありませんが、主にテキストデータで作業し、これで十分です。 2番目のオプションはバグの説明を読むことであるため、修正するパッチを適用することをお勧めします。 誰かがこれを行うと、コメントの購読を中止し、記事に変更を加えます。
  3. PHPとYII2を長い間使用しているプログラマーにとって、この資料は不必要に詳細に見えるかもしれません。 たとえば、多くの場合、「構成を変更する」と言うだけで十分です。 ただし、DelphiからPHPおよびYII2への移行を開始する人にとっては、このようなフレーズはほとんどありません。 構成が存在するファイルを見つける必要があります。 拡張機能を更新するコマンドにも同じことが当てはまります。
