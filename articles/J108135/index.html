Opera Mailのちょっとしたコツ：応答テンプレートとフィルターの同期

Operaメールクライアントの2つの設定について説明します。これらの設定は、設定ファイルでのみ見つけることができます。



1.応答テンプレートの先頭に2つのNewlanesを挿入すると便利です（そうでない場合、引用符の先頭にカーソルが表示されます）。 例：



これは、手紙の先頭で返信する場合に便利です。 GmailとYandexがこれを行います。ほとんどの方がより便利だと思います。



2.メールのフィルターの内容をIMAP経由で同期させることができます（追加設定なしで同期されるラベルなど）。 つまり あるオペラのフィルターに手動で文字を追加すると、同じボックスで動作する他のすべてのフィルターの同じフィルターに追加されます。

これは次の場合に便利です。

  • 複数のコンピューターでOperaメールボックスを使用している
  • オペラの複数のバージョンを同時に使用します（安定版およびベータ版）。
  • オペラ設定フォルダから独立したい（何かが起こった場合、すべてをゼロから上げるため）


すべてを行う方法-カットの下で。







どちらの場合も、構成ファイルを手動で編集する必要があります。

まず、メールディレクトリを定義する必要があります。 これを行うには、opera：に移動して、メールディレクトリ項目を確認します。

Operaを閉じて、プロセスから消えるまで待ちます。

メールディレクトリに移動します。



1.応答テンプレートを編集します。

accounts.iniの編集

Reply =で始まる行を探しています

このような行がいくつかあります（アカウントの数+1による）

デフォルトでは、設定はReply = On％：Date：、％f written：になります。

2つの改行を追加するために...すべてが論理的です、私たちは修正します：返信= \ n \ nOn％：日付：、％fが書いた：

フォワードのテンプレートを修正することもできます：Forward = ...、しかし私は個人的には気に入っています。



2.フィルターの同期

index.iniの編集

次の行を探しています：Name = <filter name>

以下は「キーワード=」の行です。 つまり キーワードが空です。 ここに空でないキーワードを記述するとすぐに、フィルターに追加されたメッセージはIMAPサーバーでマークされます。 別のオペラでフィルターが同じキーワードでマークされている場合、同期が発生します。

そのため、すべてのオペラに同じキーワードをフィルターに設定します。 フィルタ名に一致するキーワードを選択しますが、スペースで機能することを保証できません。

キーワードを割り当てた後、メッセージをフィルターに追加すると、オペラはサーバーに接続します。 これは、メールパネルの下部にはっきりと表示されます。

重要：フィルターの編集は、作成後すぐに行うのが最適です。 フィルターに既に文字が含まれている場合、編集後にサーバー上でそれらの文字はマークされません。 この場合、メッセージを別のフィルターに（一時的に）移動してから戻すことができます。

手動で入力されたフィルターのみが意味を持ちます。 自動フィルターはこのように機能します。

興味深い事実：ラベルの内部実装は、フィルターとまったく同じです。
