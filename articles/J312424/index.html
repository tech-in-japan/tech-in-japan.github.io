マルチテックベースステーションは、わずかなクリックでLoRaWANを展開します

Multi-Techは 、数回クリックするだけでLoRaWANネットワークを展開できるエレガントなソリューションを提案しました！ 彼らはどうやってそれをしましたか？





ご存知のように、LoRaテクノロジーは、領土または建物内に分散された多数のリモートデバイスとデータを交換する必要があるアプリケーションに適していますが、そのような各デバイスが比較的少量のデータを生成する場合に限ります。 たとえば、都市の照明制御システム、水/熱/電力メーターからのデータ収集、ビルディングオートメーション、プロセス制御などです。



LoRaWANネットワークのブロードバンド信号とライセンスのない周波数範囲を使用して：

  • 既存の無線ネットワーク/テクノロジーとの互換性
  • 高いノイズ耐性
  • 数万および数十万のデバイスに対応可能
  • 端末デバイスの広いカバレッジ領域と低消費電力を提供します


これは理論上です。 しかし、実際には？



実験により、LoRaテクノロジーのこれらの特性は一般に確認されていますが、最近まで、LoRaWANネットワークのシンプルかつ迅速な展開を可能にする普遍的なソリューションはありませんでした。 Multi-Techのエンジニアはこれを修正することにしました。 彼らは、Conduitユニバーサルゲートウェイ（屋内用）とConduit IP67ベースステーション（屋外バージョン）を作成しました。これらの構成/プログラミング用に、 Node-Redの直感的なグラフィカルインターフェイスを使用できます。

Node-Redは、グラフィック要素をドラッグアンドドロップするだけで、マウスを数回クリックするだけで独自のアプリケーションを作成できることで注目に値します！ Node-Redの概要については、 http：//nodered.org/で5分間のビデオを見ることができます。



多くの機会があり、すべてを語ることは不可能です。 たとえば、LoRaネットワーク経由で受信したメッセージをTCP / UDP経由で電子メールに送信したり、SMSとして電話に送信したり、Twitterで公開したりすることもできます。

上級ユーザーはNodejsアプリケーションを作成し、それらを個別のNode-Redブロックとして使用できます。

Multi-Techゲートウェイとベースステーションは、スタンドアロンのLoRaWANサーバーとして、または外部サーバーと連携して動作できます。 イーサネットまたは3G / LTEを使用して、機器を外部に接続します。



使いやすくするために、Multi-Techでは、 DeviceHQクラウドプラットフォームを無料で使用できます。これにより、Conduitベースステーションとゲートウェイをリモートで構成および管理できます。 そこで、開発者は同僚が公開したアプリケーションの例に慣れるか、独自のアプリケーションを公開できます。 ただし、ゲートウェイはどのクラウドサービスにも結び付けられておらず、自律的に運用できます。



将来、より複雑なアプリケーションを使用する必要がある場合、ユーザーは、Java、Ruby、Perl、Python、C / C ++、PHP、C＃、JavaScript、およびプリインストールされたSQLite（データベース）、Ligttpd（WebサーバーをサポートするmLinuxディストリビューションでゲートウェイまたはベースステーションを再フラッシュできます）、BusyBox（コアユーティリティ）。 そしてもちろん、カスタムmLinuxアセンブリの作成を禁止する人はいません。



PS Node-Redを備えたゲートウェイおよびBSのバージョンFW 1.3からは、C / C ++、PythonまたはNodejsで記述された独自の個別のアプリケーションを使用することもできます。
