WindowsとLinux：基本的な違い




1.ファイルシステム





  • Linuxでは、論理ドライブボリュームには文字が割り当てられていません。 代わりに、それらの1つがルートとして割り当てられ、残りはその中の指定されたフォルダーに接続されます。 すべてのパスは、Cドライブなしでスラッシュで始まります。

  • すべてのシステムファイルはルートファイルシステムにダンプされ、タイプ/目的ごとにディレクトリに分割されます。 相対的に言えば、すべての設定は/などにあり、実行可能ファイルは/ binおよび/ usr / binにあります-そして、このすべてのために、通常のユーザー（管理者ではない）は読み取り/実行アクセスのみを持ち、これは常にではありません（システムサービスに関しては） ）

  • Linuxのファイル拡張子は完全にオプションです。 ファイルが実行可能かどうかは、特別なマーク-Windowsの「非表示」または「アーカイブ」のマークの種類によって決まります。 Linuxの拡張子のない実行可能ファイル-物事の標準！

  • Linuxには、ファイルが隠されているという特別なマークはありません。 代わりに、先頭にドットが付いた名前が使用され、ファイルマネージャーでは、そのようなファイルの表示をオフにすることができます。 つまり、ファイル/home/user/.bashrcは非表示です。 この場合のポイントは、ファイル名の一部です！
  • 通常のユーザーは、通常/ home /％username％にある自分の個人フォルダーにのみフルアクセスできます。 ドライブDと同様：Windowsでは、多くの場合、別のパーティションが/ホームフォルダーに接続されます。 したがって、すべてのユーザーデータは別のパーティション（または物理ハードディスク）に配置されます。

  • すべてのユーザー（非システム）プログラムは、必要に応じて、データまたは設定の一部を保存します。これは、起動元のユーザーのホームフォルダーでのみ実行します。

  • Linuxでの「ファイル」の概念は、少し異なり、より広範です。 いわゆる 「デバイスファイル」。 たとえば、/ dev / sdaは通常ハードドライブです（ただし、フラッシュドライブの場合もあります）。/dev / sda1はこのハードドライブの最初のパーティションです。 したがって、dd if = / dev / sda1 of = / home / user / backupなどのトリッキーな操作が可能です-コマンドは、/ dev / sdaドライブの最初のパーティション全体をユーザーのホームディレクトリのバックアップファイルにバイト単位でコピーします。 いわゆる 「シンボリックリンク」-ファイルマネージャでは通常のファイルのように見えますが、実際には別のファイルを参照し、ディスクスペースを占有しません。 つまり、1つの実行可能ファイルと、それへのシンボリックリンクがさまざまな場所に存在する可能性があります。





2.パッケージマネージャーと「パッケージ」の概念、プログラムのインストール。



  • プログラムは管理者アカウントからのみインストールされます。 インストール中、プログラムに関連するすべてのファイル（Firefoxなど）はルートファイルシステムによって「拡散」されます-すべてのユーザーに共通の設定は/などに、実行可能ファイルは/ usr / binに、アイコンやグラフィックやサウンドなどのさまざまなリソースに移動します- / usr / share / firefoxで。 この状況では、ユーザーは、原則として、何が正確にあるかを知ることができません。 パッケージマネージャーがこれを担当します。 たとえば、「Firefox」パッケージには多数のファイルが含まれています。 パッケージをインストールするとき、パッケージマネージャはファイルシステムでそれらを分解し、削除すると、それに応じて削除します。
  • パッケージマネージャーのもう1つの重要な機能は、パッケージの依存関係を満たすことです。 たとえば、Firefoxを使用するにはlibjpegライブラリが必要です。 これは、インストール中にパッケージマネージャーがlibjpegパッケージを自動的にインストールし、アンインストール時に、他のパッケージで不要な場合は削除されることを意味します。

  • 通常、パッケージマネージャーには、使用可能なすべてのパッケージを含むデータベースがあり、このデータベースに検索ツールがあります。 したがって、Linuxでのプログラムのインストールは非常に簡単です。最初のチームはデータベースキーワードを使用してパッケージの正確な名前を検索し、2番目のチームは設定します。 サイトを登ったり、検索したり、ダウンロードしたりする必要はありません。 SkypeをArchにインストールする必要がある場合は、 pacman -S skypeと入力してEnterキーを押すと、 すぐに Skypeがインストールされます。 Firefoxが必要です-私はpacman -S firefoxを書いています。 などなど。 別のディストリビューションでは、コマンドと構文が異なります。リポジトリのアドレスを指定する必要がある場合があります-原則自体は変更されていません。

  • Windowsのように、ブラウザから何かをダウンロードして実行しようとしないでください！ あなたが何をしているのかを完全に知っている場合のみ-しかし、なぜあなたはこれをすべて読んでいるのですか？）ファイルをダウンロードして実行することは、Linuxの完全に外国の（敵意さえある）イデオロギーの一部です。 プログラムは、パッケージマネージャーを介してインストールする必要があります。 ポイント。
  • 「configure && make && make install」メソッドを使用してプログラムをインストールしないでください。 これが起こるたびに-痛みを伴う死の世界では、十数人の罪のない子猫が死にます。 この一連のコマンドは、ソースからプログラムを収集し、パッケージマネージャーの知識がなくてもファイルシステム全体にファイルを分散させます。 これは、システムを操作する通常のロジックに違反しています。 これをしないでください））
  • おそらくもう1つ追加します。 多くの場合、「管理者として機能しない」という永続的なヒントを見ることができますが、Windowsユーザーには完全に明らかではない理由があります。 実際のところ、コンソールでコマンドを入力すると、タイプミスや偶発的なクリックの危険性が伴います。 フォルダーを削除し、そのフォルダーへのパスを書き始めて、誤ってENTERを傷つける場合、状況は非常に現実的です。 Linuxには、「本当にそんな馬鹿なの？ y / n "-彼はただやる。 したがって、チームをリクルートするルートの下では、非常に注意する必要があります。 Windowsでは、もちろん、そのような問題はありません。




3.グラフィカルユーザー環境





  • ユーザーのグラフィカル環境全体は、アプリケーションプログラムの集まりです。 歴史的に、Windowsユーザーにとって、「デスクトップ」、タスクバー、システムトレイ、ボリュームコントロール、時計とカレンダー、「スタート」メニューなどの概念は、トレイからネットワーク設定へのアクセスがWindowsの不可欠な部分です。 Linuxでは、上記のすべてが個別のプログラムによって実装されます。 さらに、上記のリストの各タスクには、1つのプログラムからはほど遠いものがあります。 ボリュームコントロールとして、Cで記述されたvolumeicon、またはpythonでvolwheelを配置できます。
  • グラフィカル環境の基礎は、次のプログラムです。

    1. Xサーバー、または単に「X」。 ユーザー（入力デバイスから）と基本的なウィンドウ管理からデータを受信し、それらを最小化および最大化するプログラム。 「ネットワーク透過性」を提供するため、「サーバー」と呼ばれます。Linuxの場合、ユーザーがグラフィカルセッションにローカルまたはリモートでログオンする方法に違いはありません。 基本的な機能としての一種のRDP、簡単に言えば。
    2. ウィンドウマネージャー、別名WM。 ウィンドウインターフェイスの要素のレンダリングに従事し、さらに（トリックアウトに応じて）他の多くの機能を提供します。 一部のWMではデスクトップの背景を設定でき、一部のWMでは「システムメニュー」の機能を追加できます。 Xサーバーとともに、最もシンプルな（持っている）ウィンドウマネージャーであるTWMがインストールされる場合があります。 70年代からずっと、神の罪として恐ろしい。
    3. ウィンドウデコレータ-時々ウィンドウデコレーションの機能、デザインテーマを変更する機能は別のプログラムで実行されます
    4. 複合マネージャー-WMの一部として、または別のプログラムとしても発生します。 そのタスクは、インターフェイスのレンダリングをビデオカードにシフトすることです。 技術的には、原理は単純です-レンダリングされた各ウィンドウは、ビデオカードのメモリ内の個別のテクスチャです。 また、テクスチャを処理したり、エフェクトや歪みを追加したり、空間の平面に投影したり、半透明性を変更したり、長年にわたって重ねたりすることができます。
    5. インターフェイス要素：タスクバー、トレイ、ネットワークマネージャー、システムメニュー、デスクトップの壁紙を設定するためのプログラム
    6. 基本的なアプリケーションソフトウェア-ファイルマネージャー、ターミナルエミュレーター（コンソールコマンドを美しい半透明のウィンドウに書き込むことができるように）


  • グラフィカル環境の選択された要素のプリセット「セット」、上記のリストからのプログラムは、「デスクトップ環境」またはDEと呼ばれます。 最も有名なのは、GnomeやKDEのようなDEで、最も重くて脂肪質のDEです。 XFCEとLXDEもあります。 インストールは、多くの場合、いわゆるメタパッケージのインストールによって実装されます-パッケージ自体にはファイルが含まれていませんが、依存関係として、DEを構成するプログラムのセット全体のインストールが必要です：WM、decorator / composite manager、file managerなど
  • 「ピース」から好みに合わせて環境を収集することもできます（多くの場合、合理的です）-WMを個別に選択し、ファイルマネージャーを個別に選択するなど




まとめ




少しの自己教育の後、テキストコンソールグラフィックシステムからの取得は1つのコマンドで行われます。 私の場合、次のように入力します。

pacman -S xf86-video-ati xorg-server openbox tint2窒素lxterminal xcompmgr wicd-gtk volumeicon 。

これは、リストされているすべてのパッケージをインストールするコマンドです。

xf86-video-atiは私のビデオカード用のオープンソースドライバーです

xorg-serverはX

openboxは、システムメニュー（「スタート」など）を備えた軽量のWMです。

tint2は、アプリケーションが最小化されるトレイを備えたタスクバーです。

窒素を使用すると、デスクトップの背景を設定できます

lxterminalは私のお気に入りのターミナルエミュレータです

xcompmgrは最もシンプルな複合マネージャーで、半透明性と影を追加します

wicd-gtkは、トレイでハングするネットワーク接続マネージャーです

volumeicon-ボリュームコントロール



その後、グラフィカル環境からlxterminalを使用して、ブラウザ、ファイルマネージャー、ビデオおよびオーディオコーデック、プレーヤー、libreoffice、gimpなど、生活に必要なものをすべてインストールします。
