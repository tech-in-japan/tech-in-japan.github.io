ZX Spectrumのシステムモニターはどこにありますか？ Riddle PC Duet

Duet PCは、Lianozovsky Electromechanical Plant（LEMZ）が製造したZX Spectrum 48kのロシア製クローンです。 これは私の最初のコンピューターであり、まだ私と一緒です。 若い頃から、プログラミング、マイクロプロセッサアーキテクチャ、およびその上でのデジタル回路の設計の基礎を学び始めました。 しかし、それ以来、システムモニターはどこにあるのかという質問に悩まされています。 結局のところ、それはドキュメントに記載されています。 標準のZX Spectrumでは、システムモニターの存在を思い出せません。 また、システムモニターに関するドキュメントでは、詳細は説明していません。 1990年のROMバージョンには 48kのモニターがあります。 ただし、オンにすると、Duet PCは標準の挨拶の代わりに「（c）1982 sinclair research ltd」という別の挨拶： 「（c）DUET」を表示します。 これは、ROMがまだ変更されていることを意味します。 それとも、ハードウェア監視機能がありますか？ たとえば、クローンOrel BK-08には、シャドウRAM、NMIボタン、MZ80モニターなどの多くの改善点があります。 長い年月を経て、あなたの鉄片の隠された能力を見つけることは非常に興味深いでしょう。



最後に、私はこの数年間、定期的に私を心配させていた質問に対する答えを見つけました。

モニターの次の問題



初めて購入後すぐにモニターについて質問しましたが、答えが見つかりませんでした。 すでに私たちの時代、2016年の初めに、私はフォーラムで調べようとしましたが、再び答えは見つかりませんでした。 質問は忘れられていたでしょうが、偶然スペクトルクローンの販売広告「Nafanya」に出会い、「 ROMにはBASICインタープリターとシステムモニターが含まれています」というドキュメントページの写真がありました。 まあ、私たちはまだそれを理解し、この問題に終止符を打つことにしました







フォーラムの仲間はアドバイスを与えました：

ほとんどの場合、システムモニターは一連の標準ROMルーチンを指します。 試してください：

1.ボードにインストールされているROMを確認します。各8Kが2個あるようです。これは16Kです。追加のROMがない場合は、使用可能な16Kを調べる必要があります。

2. ROMダンプをマージし、標準のSinclairダンプと比較します。通常、クローンの碑文のみが変更されます。 違いによると、何か面白いものがあるかどうかを判断できます。
まあ、それは簡単なことではありません！ 回路を調べると、追加のROMがないことがわかりました。2つの8Kb ROM（標準の16kB BAS ROMを構成しています）。 私の標本の内部の写真から判断すると、私のバージョンはサイト上のスキームと違いはありません（実際、マイナーな改善/変更がありますが、おそらく別の機会に）。 したがって、モニターの存在に関するハードウェア機能は見つかりませんでした。



ROMダンプを比較する



ダンプをマージして元のダンプと比較するには、次の手順を実行する必要があります。



1）PC DuetのROMからデータを取得します。 ROMをボードから取り外してプログラマーを使用することはまったく必要ありません。 48kバージョンのROMが0..16383アドレスのアドレス空間に完全にマップされるという単純な理由によります 。 これは、アドレス空間の最下部です。 プログラムで読み取るか、BASICコマンドSAVE "ROM" CODE 0, 16384



を使用してテープ出力経由で保存できます。



2）PCのユーティリティを使用して、サウンドファイルからTAPファイルにデータを変換します

3）TAP形式からバイナリ形式にデータを抽出する

4）Duetと元のSpectrumの2つのバイナリファイルのバイトを比較するユーティリティを作成または検索します。



ラップトップにはリニア入力がなく、アダプターは、Duet PCの入力に接続するために、リニア出力を介してPCまたはスマートフォンでサウンドを再生する方向にのみはんだ付けされているため、すでに最初の段階でこのアイデアを拒否しました。 これは、現在99％がDuetにデータをロードしているためです。逆の場合も同様です。



したがって、正反対のことをすることにしました。 元のROMを取り出し、テープ入力を介してDuetのRAMにロードします。 次に、メモリセルのデータと異なるセルの情報の出力を調整するプログラムを作成します。 元のROMのバイナリデータは、ほぼすべてのZX Spectrumエミュレーターにあります。 しかし、それらを音声に変換するために、その後テープ入力を介してダウンロードするために、少し手を加える必要がありました。 バイナリファイルからTAPファイルを作成する前に、3〜4つのプログラムを試しました。 その後、簡単になりました-ユーティリティTape2WAVの助けを借りて、サウンドファイルを受け取りました。



ZX Spectrum BASICのテープレコーダーからデータをロードするには、LOADコマンドを使用します。コードがロードされることを示すパラメーターと、それをロードするメモリアドレス、それぞれLOAD "" CODE 40000



は、アドレス40000から元のROMのデータをメモリにロードします。



記憶を調和させるプログラムを書く



ZX Spectrumの学校での経験とプログラムを自分で覚えておく必要があるため、私は喜んで手をこすります！ しかし、ところで、覚えておくべきことは何ですか？ 基本的なことは忘れることができず、さらに簡単です。 各キーストロークでコマンド全体がすぐに導入されるため、コマンドセットを間違えることは困難です。 T9のようなBASIC専用の高速で便利な！ また、エミュレータで48k BASICを使用することは、BASICコマンドが最新のコンピューターで署名されていないという事実のため、単に便利ではないことを言いたいと思います。 48kのような小さなマシンのもう1つのプラス：スイッチを入れるとすぐに作業を開始できます。 開発環境もコンパイラも必要ありません。 作業に必要なものはすべてROMにあります。



ZX Spectrum内の基本が解釈されます。 これにより、プログラムの速度が低下します。 しかし、アセンブラーでこのようなシンプルで速度が重要なプログラムを作成するためには、これはすでにやり過ぎだと思います。 プログラムは5分で書かれます。 そして、それが同じように機能する場合でも、それは私に合っています。 しかし、5分もかからなかったことがわかりました。







スクリーンショットでは、エミュレータに入力したプログラム（ソースコードを含むスクリーンショットを撮る場合のみ）。 Duetでも同じプログラムを入手しました。 スクリーンショットの上に、メモリセルで見つかった違いに関するデータに署名しました。 さて、それほど多くの変更はありません。 次に、これらの変更がどのような影響を与えたかを理解する必要があります。 これを行うには、見つかったアドレスのROMの内容を知る必要があります。 幸いなことに、すべての情報はロシア語の本「Full ZX-Spectrum ROM Disassembler」にあります。
