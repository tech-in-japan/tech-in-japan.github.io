約1年前、私はさまざまなインターンシップに参加した経験についてブログに投稿しました。 この投稿のおかげで、私はリクルーターにもっと目立つようになり、Googleで仕事を得ました。







また、インターンシップについて質問がある学生から多くのメールを受け取り始めました。 そのような手紙を受け取るたびに、私の自我は約2倍になります。 ありがとう

＃2自分のWebサイトを作成する



ここにちょっとした秘密があります。あなたが自分自身を宣伝すればするほど、より多くのリクルーターがあなたに連絡したいと思うでしょう。 独自のWebサイトを作成すると、より目立つようになります。







あなたのウェブサイトは、本質的には、より詳細な形式の履歴書ですが、あなたの個人的なブランドでもあります。 サンプルサイトは次のとおりです。















サイトはいくつかの機能を同時に実行します。







  • 採用担当者が検索エンジンでポートフォリオをより簡単に見つけられるようにします







  • 1ページの履歴書ではできなかったのと同じくらい自分のことを書くことができます。 特に、これは彼らの美的資質と美意識を示す絶好の機会です。







  • 見栄えが良い場合は、写真を追加する必要があります。 誠実で笑顔の顔はそれ自体に気を配ります。


Github Pages、Google App Engine、Wordpress、Weeblyなどのプラットフォームを使用して、無料でWebサイトを作成できます。 ドメイン名は非常に安く、年間約10ドルです。







コーディング能力を実証することに加えて、コードを書くことにあまり慣れていない人々が理解できるような方法でプロジェクトのリストを作成する必要があります。 プロジェクトのブログノートとチュートリアルを作成すると、さらに良い結果が得られます。 あなたのサイトは、人々がそれが役に立つと思うならば、より人気があるでしょう。







あなたのウェブサイトを通してあなたが語る物語（あなたの第一印象を形成する）は非常に重要です。 あなたがすべてを正しくすれば、蜂蜜のための蜂のように、リクルーターはあなたに走りに来ます。







＃3コンピューターサイエンスを学ぶ



あなたが将来何をしたいのかわからない場合は、仕事を選ぶ際に最も柔軟になることができるスキルを選択してください。 数学+コンピューターサイエンス（研究に興味がある場合）または描画+コンピューターサイエンス[二重学位...ウィキへのリンク]（エンターテイメント業界に興味がある場合）を勉強することをお勧めします。







学士号を取得して勉強していた私は、 神経科学を勉強すると思っていました。 これは大きな誤りでした：







  • 神経科学を主要なポイントとして示したため、私の履歴書は無視されました。 最終的には、Googleの採用担当者に電話インタビューでチャンスを与えてくれるように頼み、突破することができました。 その後、「Applied Mathematics + CS」の束に切り替えました。







  • CSでマスターになるには多くの時間がかかります。 大学/学校はこれに適した時間です。







  • 聴衆の神経科学は、実験室の神経科学と同じではありません。 高度な研究は、光遺伝学またはコンピューティングに関するものです（これは、偶然にも、神経科学よりもCS +数学+物理学により近い）。


最後のポイントに加えて、MATLABで神経科学とプログラミングを勉強している学生は、高度な研究問題に対処し、実験データに取り組む必要があることがわかりました。 他の学生は研究室で汚れた作業を行い（解剖し、液体のピペットで収集）、コード分析を他の人に転送しました。







テクノロジーの影響を受ける分野は、神経科学だけではありません。 近い将来、より多くのソフトウェアベースの研究が登場するでしょう。 良くも悪くも、将来的には、科学者、医師、弁護士、すべてプログラマーになるでしょう。







数学がなぜそんなに重要なのですか？ 必要に応じて、複雑な研究​​職で働く機会を提供します。 CS学位を1つしか持たない研究チーム（Google ResearchやMicrosoft Researchなど）に異動することは非常に困難です。







Two Sigmaでインターンシップをしている間に数学でもっと面白いことを勉強することはできましたが、過去の経験でソフトウェア開発者として定義されていたため、量的研究でインターンシップを得ることができません。 私の失敗は、数学にほとんど注意を払わなかったことです。







映画やゲーム業界でプロダクトマネージャーとして働きたい場合、数学を勉強することの利点はそれほど明白ではなく、イラストを勉強する方が良いでしょう。 Pixarでこれに気付きました。多くのテクニカルディレクターは、スクリプトとビジュアルコンポーネントにより大きな貢献をしたいと考えていますが、投稿に「ロック」されていることがわかりました（たとえば、「車で男」、「植物の影で女の子」 「など）







イラストが得意であれば、アートディレクターまたはストーリーアーティストの役割を主張できます。 このスキルを持つ人は、仕事の分野を自由に選択することもできます。イラストレーターは、デザインからコミック、ゲームまで、あらゆる場所で必要です。 イラスト+ CSは強力なスキルセットです。







しかし率直に言って、数学は絵を描くよりも「安全」で、柔軟性があり、収益性が高いです。 また、将来的には賢明な選択ですが、他のスキルはそれほど多くありません（設計、法律、ビジネス）。 したがって、私は絵を描くことは非常に価値があり、趣味としてそれを実践し続ける価値があると信じています。

＃6履歴書評価段階をスキップしてみる



ほとんどのインターンシップの申請の最初の段階は履歴書です。 多数の申請書を読む必要がある採用担当者は、履歴書を6秒間見るのをやめ、次の履歴書に進むか、肯定的な回答を送ります。







6秒！ これは、有名な教育機関の名前、企業の名前、およびあなたが知っているプログラミング言語を見つけるのに十分です。 また、採用担当者は、履歴書がどれほどきれいで美しいかを評価します。 その結果、特に経験の浅い学生を評価する必要がある場合、履歴書の確認は退屈な作業になります。







幸いなことに、この手順をスキップする方法はいくつかあります。







  • 社内に紹介がある場合、採用担当者は履歴書をより慎重に確認します。 あなたの履歴書が嫌に見えない場合、次の段階に進んでいる可能性が高いです。 かつて、ピクサーとツーシグマで紹介を見つけることができたのは幸運でしたが、これはまったく別の話です。







  • あなたがテクノロジーで過小評価されている少数派（URM）（ハイテク企業の数が少ない人：女性、アフリカ系アメリカ人、ハワイ人など-約）の1人である場合、企業はあなたをインタビューに誘い込もうとします。 Grass Hopperのような会議では、実際に履歴書の段階や電話インタビューをスキップし、アップル、Facebook、Google、Pinterestなどの企業とのその場でのホワイトボードインタビューに参加できます。これにより、インターンシップの機会が大幅に増えます。 私のクラスメートは、履歴書の助けを借りてのみ、その場でアップルからインターンシップの招待状を受け取ることができました（インタビューなどを経験することなく）。 そのような会議への参加を後援することについて質問をあなたのトレーニング部門に書いてください。







  • あなたの大学の元卒業生の助けを借りてIT従業員を見つけ、あなたの紹介者になるよう依頼してください。 恥ずかしがらないでください-彼らは最小限の努力を必要としますが、インターンシップで終わる場合は大きな紹介ボーナスを支払います。 最悪のシナリオでは、彼らはあなたの手紙を単に無視しますが、この場合、あなたは何も失うことはありません。


デフォルトでは、履歴書は仕事中でなければなりません。すべてが完璧にレイアウトされ、タイプミスなく明確に書かれています。 会社に提供する履歴書の各コピーを慎重に確認してください。







企業は、あなたのアカデミックビルディングを訪問して、学生とのインタビューを実施することもできます（Yelp、Microsoft、Googleがこれを行います）。 これは非常に便利であり、働きたくない企業とのインタビューを行う価値があります。 当然のことながら、結局のところ、すべての企業の採用担当者は、他の誰よりも実践的なアドバイスを提供できます。







ぜひ、履歴書提出段階を避けてください。 実際、申し出の締め切りが容赦なく迫っている場合、企業は直接あなたをオンサイトインタビューに転送することができます。 履歴書と電話インタビューは、オンサイトインタビューの単なる選択であり、後者ははるかに重要です。 正面玄関から入らないでください 。







＃7電話およびオンサイトインタビュー



再開フェーズの後、あなた次第です。 通常、1つまたは2つの電話インタビューが最初に来て、5時間のオンサイトインタビューが続きます。 電話インタビューは、Google DocまたはEtherpadでコードを書くオンラインインタビューのミニチュアバージョンです。







この段階で最も重要なことは、プログラミングの問題をどれだけうまく解決できるかです。 問題を迅速かつ正確に解決し、通常どおり過剰に動作しない場合、このジョブを取得する可能性があります。







私の経験では、面接の複雑さは会社の給与と選択度に依存します。 私が経験した最も困難なインタビューは、Google Deepmind、DE Shaw、Two Sigma、Quora、Vatic Labsでした（ITの巨人よりもリスクが大きいため、スタートアップのインタビューは通常より厳しいです）。







GoogleとFacebookは複雑さが平均的です。 私は、Pixarのソフトウェア開発者の地位に関するインタビューを受けなかったため、インタビューは表面的で非常に簡単でした。 また、ジェーンストリートでのインタビューは技術的に最も難しいと聞きました（明らかにMIT学生の間で非常に人気があります）。







あなたが必要とする本は、コーディングインタビューをクラックすることだけです。 私が応募したすべての企業の実際的なタスクはほぼ同じ難易度であり、唯一のアドバイスは絶えず改善することです。







DE ShawやJane Streetなどの金融機関は、数学の質問をするのが好きです。 次の3冊の本をお勧めします（難易度の高い順に）：







  • 定量的財務インタビューの実践ガイド

ホワイトボードの面接の準備は、SAT （大学入学のためのアメリカの試験-約）の準備に似ています -同じ時間の損失ですが、非常に重要なので、あなたはまだそれをしなければなりません。 不完全なインタビューシステムから脱却しようとしているスタートアップはいくつかありますが、成功するかどうかはわかりません。
