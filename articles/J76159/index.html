シンクライアント-内容と使用内容（WTWareの例）

コンピューター技術のシンクライアント（英語のシンクライアント）-サーバー上の情報を処理するためのすべてまたはほとんどのタスクを転送する、クライアントサーバーまたはターミナルアーキテクチャを備えたネットワーク内のディスクレスクライアントコンピューター（Wikipedia）。



よりシンプルな場合、シンクライアントは軽いオペレーティングシステムをロードする非コンピューターであり（Linuxは通常使用されますが、レビューでは先験的に取ります）、ターミナルサーバーに接続します。



通常、シンクライアントは、まれに他の理由でハードウェアとソフトウェアを節約するために作成されます。







この記事では、 WTWareの概要を簡単に説明します。WTWareは、シンクライアントの作成専用に設計されたLinuxディストリビューションです。



まず、シンクライアントについてです。



シンクライアントはシステムユニットであり、通常はハードディスクを備えておらず、トンクライアント（以下、単にシンクライアント）のオペレーティングシステムを起動するために必要な最小限のハードウェアセットのみがあります。 電源装置、マウス、キーボード、モニター、ネットワークケーブルがシステム装置に接続されています。 標準セットに加えて、他のデバイスを認識してターミナルサーバーに転送できる場合は、他のデバイスをシンクライアントに接続できます。

シンクライアントネットワーク図は次のようになります。





仕組み：

  1. コンピューターでは、シンクライアントがいずれかのソースから起動します。 ダウンロードソースの主なオプションは、LAN、CD、HDDです。
  2. シンクライアントをロードするプロセス（またはLANバージョンの前）で、コンピューターのネットワークカードにIPアドレスが発行されます。
  3. ダウンロードの最後に、rdesktop経由のシンクライアントは、設定で指定されたターミナルサーバーとのターミナルセッションを解除します。


なぜ機能するのか：

  1. ハードウェアのコストを大幅に削減しました。 企業は1ペンスで古いジャンクを購入できます。その運用に必要なのは、十分なリソースと調整されたシンクライアントを備えたターミナルサーバーだけです。
  2. ソフトウェアのコストが削減されます-デスクトップ用のソフトウェアを購入する必要はなく、ターミナルサーバーのライセンスを取得するだけです（ただし、ターミナルライセンスを購入する必要があります）。
  3. 管理コストが削減されます。 ターミナルサーバーを管理するだけです。 実践が示しているように、シンクライアントは（集中的な努力をしなければ）事実上殺されず、事実上失敗しません。 ただし、システム管理者を変更する場合、たとえば仮想マシンでシンクライアントの動作をエミュレートすることで、システム管理者がこのこと全体を理解する必要があることを理解する必要があります。


ダウンロードの種類：

  1. ネットワーク経由でダウンロードします。 次のように機能します。ローカルネットワークでは、DHCPサーバーとTFTPサーバーを上げる必要があります。 コンピューターには、BootROMを備えたネットワークカード、またはBootROMをエミュレートするネットワークカード用のドライバーが必要です。 ネットワークカードは、ネットワークでDHCPサーバーを探し、必要なすべてのネットワーク設定とTFTPサーバーアドレスを受け取ります。 次に、TFTPサーバーにアクセスし、オペレーティングシステムをロードします。
  2. CD / DVD / Flash / IDEからダウンロード-他のオペレーティングシステムと同様に、ここに標準があります。


WTWareとは何ですか？



WTWareは、シンクライアントの構築専用に設計されたGNU / Linuxディストリビューションです。 Thinstationと呼ばれる一般的なクライアントに基づいています。 主な違いは、ロシアのユーザーに焦点を合わせていることです（Thinstation自体にはキリル文字に問題があります）、さらにあらゆる種類のマイナーな修正です。



WTWareを構成します。



DHCPサーバーとTFTPサーバーの構成については説明しません。すべてが標準です。 DHCPサーバーではTFTPサーバーのアドレスを指定し、TFTPサーバーではブートファイルへのパスとこのファイル自体の名前を指定する必要があることを思い出してください。



また、WTWareの微調整は行いません。 WTWareの公式Webサイトの情報は非常にアクセスしやすく、多くの情報があり、すべてロシア語です。 主な側面のみを示します。



だから。 まず、WTWare WebサイトからThinstationイメージをダウンロードします。 開梱します。

ブートファイルは、PXEプロトコル（ネットワークまたはマザーボードにBootROMが組み込まれている場合）を介してロードされる場合はpxelinux.0、Etherbootブートローダー（BootROMエミュレータを使用する場合）はwtshell.nbiと呼ばれます。



ところで、Etherbootは、ほとんどすべての既存のネットワークカードのファームウェアをリリースするオープンソースプロジェクトです。 Etherbootファームウェアは、BootROMチップまたはネットワークカードのフラッシュメモリに書き込むことができ、フロッピーディスクまたはハードディスクからブートセクタまたはDOSのプログラムとして起動できます。



さらに、LAN経由で起動し、DHCPサーバーとTFTPサーバーを正しく構成していれば、すべてが「そのまま」動作するはずです。 唯一のことは、シンクライアントをまだ構成していないため、ターミナルサーバーが見つからないことです。



別の方法で起動する場合は、 ここを読んで 、興味のある起動方法を選択してください。



構成



繰り返しますが、何百ものパラメーターがあるため、私は構成ファイルのジャングルを掘り下げません。 ここに完全なリストを見ることができます。 主なものについてのみ説明します。



WTWareには、次のタイプの構成ファイルがあります。

  1. all.wtcは、システム全体の構成ファイルです。
  2. list.wtc-リストに従って構成ファイルを含めます。
  3. 個々の構成ファイル。


個々の構成ファイルには、次のタイプがあります。

  1. TerminalName.wtc。 端末名は、それぞれDHCPサーバーによって発行されます。
  2. ma.ka.dr.es.te.rm.wtc。 バインディングは、接続された端末のポピーアドレスに行きます。


構成ファイルには次の優先順位があります。

  1. all.wtc
  2. list.wtcで指定されたファイル
  3. includeを介して接続されたファイル
  4. 個別の構成


システム全体の構成変数：

win2kIP = 10.100.50.1 //ターミナルサーバーアドレス1。

win2kIP2 = 10.100.50.2 //ターミナルサーバーアドレス2。

video = VESA（S）//ユニバーサルドライバー、ほぼすべてのビデオカードで動作

mouse_wheel = on //マウスホイールを有効にする

解像度= 1024x768 //画面解像度

bpp = 32 //色深度



個々のファイルの構成変数：

ユーザー=ユーザー名//ユーザー名

パスワード= user_password //ユーザーパスワード

domain = enterprise_domain //エンタープライズドメイン



個々のファイルで一般ファイルに存在する変数を書き込む場合-より高い優先度を受け取ります。



さらに、プリンター、スキャナーなどの接続されたデバイスは、個々のファイルに登録されます。



最後に、ローカルリソース（フロッピー、DVD、フラッシュ、HDD、サウンド）の接続という別の興味深い機能に言及したいと思いました。 構成では、次のようになります。

フロッピー=オン

cdrom = on

usb1 =オン

サウンド=オン

ディスクは、現在のユーザーセッションでWindowsエクスプローラーの\\ tsclient \ {floppy | cdrom | usbN}で使用できます。



短所：

  1. システムにデバイスのドライバーがない場合、機器の接続に問題がある可能性があります。 松葉杖を使って画像を解析し、そこにドライバーを押し込み、画像を収集できることを知っています。 自分で試したことはありません。
  2. カードにBootROMがない場合、Etherbootファームウェアの選択に問題がある可能性があります（すべてのカードではありません）。


ライセンス：



システム自体は無料ですが、起動画面からWTWareロゴを削除するという非常に興味深い目的のためにライセンスを購入できます。 私が理解しているように、これはアウトソーシングの支援の下でこの製品を大量に導入している企業のために行われました。



シンクライアントを作成するための機器：



WTWare Webサイトでは、シンクライアントを作成するための機器を購入することもできます（ジャンクから収集しないようにするため）。 私はそれ（機器）が魅力のすべての要件を満たしていると言わなければなりません。 いくつかのスクリーンショット：





まあ、それがおそらくすべてです。 端末、DHCP、およびTFTPサーバーが正しく構成されていれば、すべてが機能するはずです。 インターネットには多くのロシア語の文献がありますので、セットアップに問題はないはずです。 そして一般的に、ドキュメンテーションの観点から、私はシステムが本当に好きでした。メーカーのウェブサイトにはほとんどすべてのものがあります。
