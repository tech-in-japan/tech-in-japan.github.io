なぜOpera Uniteなのですか？

多くの人が同じコメントを書いているのに私はここでうんざりしていて、まだ答えが得られなかったので、トピックを書いています。



Opera Uniteは、ifconfigを自宅とするDNSやuPNPを設定できる人や、file_get_contentsがひどい下品な人には向いていません。「ルーター」という言葉でさえ、無礼に怒らせようとしている人向けです。 オペラは、これらの人々に写真をネットワーク自体に投稿することでFlickrに依存しない機会を与えました。小規模な起業家がWordで作成された静的なWebサイト（htmlとして保存）をネットワーク上に置く機会を提供しました。自分用の小さなパーソナルサービスを作成するか、アプリケーションのプロトタイプを作成します。 メッセージングドキュメントなどを簡単に交換できます。



ポイントは、Opera Uniteが初心者向けのツールであることです。

そして、プロにとっては、これは小さな個人用アプリケーション（たとえば、1日5分かかる個人用請求書）を作成するためのオプションです。



これは、オフィスの全員がドキュメントを共有し、他の人に渡すことができるイントラネットシステムです。



このp2p Webでは、構成が必要な深刻な技術（httpd.conf / my.cnf / nginx.conf / vsftpd.conf / ...ほかに必要なもの）に誰も接続していません。



これは、突然誰もnginx / apache / php / mysql / djangoを必要としないことを意味するものではありません-彼らは同じ方法でそれを必要とします。



SketchUpは3DSMAXを廃止しませんでしたが、建築家、ジョイナー、デザイナーに、MaxとMayaの深刻さのために処理できないシンプルな3Dツールを提供しました。 MaxersとMaitsaは去っていませんが、3Dを専門としない多くの人々が3Dに対処できるようになりました。



Opera Unite（常にOpera Nativeを作成）は、SketchUpが3Dに対して行うことを行います！ これに参加できる人々の基盤を拡大します。
