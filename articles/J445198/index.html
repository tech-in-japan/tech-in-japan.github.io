Linux用のCmusコンソールプレーヤー

良い一日。



現時点では、非常に便利なcmusコンソールプレーヤーを使用しています。 これを踏まえて、短いレビューを書きたいと思います。







新しいワークステーションで、ついにLinuxに移行しました。 この点で、仕事関連のニーズに適したソフトウェアを見つける必要がありました。 Linuxには十分なインターフェイスプレーヤーがありますが、それらはすべて幾分積み上げられており、あまり美しくありません（私見は読者の意見と一致しないかもしれません）。 以下の基準が検索パラメーターに含まれていました。



  • シンプルさ
  • 便利さ
  • 低RAM消費
  • サブフォルダーを再生およびスキャンする機能
  • 通常のプレーヤーに固有の関連パン


その過程で、私はcmusに出会いました。 プレイヤー...コンソール...そして、なぜそうではないのか、私は多くの人のために通常のコマンドを考えて実行した： 



sudo apt-get install cmus


最初の実行後、次が表示されます。







プレイリストに音楽を追加するには、コマンドを実行するだけです。 viのように「：」を押して、次のように記述します。 



 add ~/Music


このコマンドの結果、最初のスクリーンショットのようにサブフォルダーとトラックのリストが表示されます。



プログラムをナビゲートするには、矢印キー、タブ（領域を切り替える）、スペース（サブフォルダーを展開する）、削除（リストからトラックを削除する）を使用できます。 また、膨大な数のホットキーがあります。 プレーヤーのウィンドウを変更するには、1〜7の数字キーを使用します。7番目のウィンドウでは、自分用にホットキーを再構成できます。



コマンドの実行により、これに関する詳細を読むことができます： 



 man cmus


再生モードでは、プレーヤーは約10 mbのRAMを「消費」しますが、これは非常に控えめなサイズだと思います。

公式Webサイトでは、いくつかのより快適な機能（オーディオスクロールなど）とアドオン（マルチメディアキーの接続、spotifyまたはyoutubeからの音楽のダウンロード）のサポートを主張しています。



要約すると、このプレーヤーは非常に便利で柔軟であると安全に言えます。



私はそれがすべての人のためではなく、コンソールのオタクだけのためであることに同意します=）



これで私は休暇を取る。
