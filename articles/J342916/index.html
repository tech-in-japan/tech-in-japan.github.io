幻想のない実際のケース







当社は、電子機器の製造および販売を行っています。 世界中の顧客。 毎日、5〜10個の小包が郵便、運送会社によって（オンラインストアの標準により、かなりの量で）送られます。 海外、ロシア。



  1. 日中は、データベースに分類される注文、つまりGoogleシートテーブルが届きます。
  2. 一日の終わりに、小包が完成し、パッケージの住所が記入されます。
  3. 小包は中央郵便（24時間対応）に配信され、送信されます。
  4. 小切手は写真に撮られ、特別な電報チャットに送信されます。
  5. スタッフの誰かがチャットからテーブルへのトラック番号を中断します。
  6. トラック番号はメールで顧客に送信されます。 ロシア語と英語の手紙。


これには過剰なものはないようです。



しかし、これらの小さなルーチン操作の合計時間を計算すると、 毎日約2時間が費やされていることがわかります。



  • パッケージに住所を記入する
  • データベースへのトラック番号の中断
  • トラック番号を確認します（番号が正しく印刷されない場合があります）
  • 異なる言語でクライアントに手紙を送る
  • 他の小包の監視：税関で立ち往生していないか、すでに1週間受信者に郵送されているかなど。

プロセスが形式化されている場合、自動化できます



1C、Bitrixなどのさまざまなシステムを導入しようとしました。 彼らは単純な理由で「ログイン」しませんでした-アプリケーションは長時間ロードされます。 たとえば、300ルーブルの無駄を作るには、アプリケーションがロードされるまで待機し、いくつかのフィールドを突く、タグを置くなどする必要があります。

その結果、費用、タスク、その他の情報は入力されません。



テレグラムチャットを備えたシステムが定着しました。常に手元にあり、すばやくロードでき、クロスプラットフォームです。



結論： Telegramが中小企業に提供する主な利点は、 情報を入力する速度です。 数秒以上かかるすべてのものは、最終的には数回クリックされません 。







さて、情報を作成しましたが、実際には、これらのチャットはすべて、生の情報の単なるダンプです：領収書、費用、購入、連絡先、小切手など



そしてここで、この情報を処理するボットが必要であるという論理的な結論に達しました。主なことは、情報が入力されることです。



トラック番号を認識する



最小タスク：ボットに少なくともチェックからのトラック番号を認識させ、チェックサムをチェックしてチャットに送信し、手動で中断するのではなく、文字をコピーしてテンプレートに貼り付けるだけにします。



問題は、メールの領収書にバーコードまたはQRコードの形式のトラック番号がないことです。



そして、ここでLinux用のABBYY CLI OCRが私たちの助けになります。









このツールの詳細については、別の記事で説明します。設定、言語、バーコードおよびQRコードの認識を備えたABBYYテキスト認識は、Googleよりもはるかに優れています。



彼女に写真を送り、出力で十分に適切な認識されたテキストを取得します。それを解析し、きれいにし、その中のトラック番号を見つけ、エラーをチェックして修正する必要があります（たとえば、番号8は0とみなされます）







さて、小切手からトラック番号を引き出したので、自動的に送信することもできます。唯一の質問は-誰にですか？ このトラック番号を何らかの方法で対応する順序と一致させる必要があります。 phpで小さなサービスを作成し、そこにGoogleシートからデータベースを転送します。







住所フォーム（手で入力する代わりにパッケージに接着されています）、ピッキングリスト、および注文へのリンクを含むQRコードを含む背骨を印刷します。







小包を郵便局に直接送信した後、ルートに添付された小切手をQRコードで撮影し、ボットを落とします。 すべて、その後、すべてが自動的に行われます。



  • 配送サービスのトラック番号が認識されます。
  • チェックサムがチェックされます。
  • 出発タイプと言語が決定
  • 希望する言語のトラック番号を含む手紙がクライアントに送信されます
  • トラック番号がデータベースに入力され、注文ステータスが変更されます
  • 小包の運命は自動的に追跡されます
  • 遅れると、行動を起こすように通知します
  • 到着時に、小包は顧客にそれを受け取るように思い出させます
  • 受け取った後、2週間後、レビューを残すように要求する


このシステムは時間を節約し、日常的なエラーや人為的エラーを排除し、サービスを改善し、最も重要なこととして利益を上げます。
