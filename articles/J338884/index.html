バックアップシステムは、企業情報システムの重要な部分です。 適切な組織により、重要なデータを確実に保護することができ、企業セグメントでは、データの可用性と安全性がビジネスプロセスとビジネスパフォーマンスの継続性の鍵となります。 今日の投稿のトピックは、重複排除機能を備えたバックアップシステムです。 どれか-あなたはすでに推測しましたが、私たちは自分自身に先んじることはありません。









データバックアップシステムの導入により、さまざまな状況で情報を迅速に回復できますが、これらのシステムには多くの欠点があります。 エンタープライズバックアップシステムの従来の問題は次のとおりです。



  • 非効率 ：データボリュームの最大200％が週にコピーされます。

  • それほど信頼性が高くない ：回復不能のリスクが高い（Gartnerによると、10％以上）。

  • 耐災害性の問題 ：テープをリモートストレージに移動すると、テープを失うリスクがあります。





これが、従来のバックアップシステムの仕組みです。 今日、指数関数的なデータ成長の条件で効率的に機能し、規制当局の要件を強化し、SLA要件を厳しくし、バックアップウィンドウを削減する新しいバックアップおよびリカバリソリューションが必要です。





プールベースのレプリケーション：効率を最大化するために、Avamarはグループの優先順位付けとレプリケーションを並行して行います。

データ重複排除テクノロジーとその利点



重複排除は、複製に基づいて、共有の可能性があるデータの単一コピーを作成し、データストレージ容量（IDC）の使用効率を高めるための技術です。 簡単に言えば、重複するデータシーケンスを削除することで、保存されるデータ量を削減します。





重複排除を使用すると、ネットワーク経由で転送して、保存するデータを減らすことができます。



重複排除は、ファイル、ブロック、またはアプリケーションレベルで機能します。 格納されているデータの量をさらに削減するために、重複排除はしばしば圧縮（圧縮）と組み合わされ、このすべてのデータ圧縮を呼び出します。









重複排除アルゴリズムの仕組みとその結果。



一般的な場合、重複排除アルゴリズムは重複（データブロック）を探し、それらを「データポインター」（このデータオブジェクトの正確なコピーへのリンク）に置き換えます。 重複排除は、データを保存および送信する際の不要な作業を最小限に抑えることにより、コストを削減します。

バックアップシステムでは、重複排除は多くの問題の解決に役立ち、次の利点があります。



backupバックアップウィンドウの削減



従来のバックアップでは、大量のデータが転送されるため、バックアップウィンドウへの「適合」は非常に困難な場合があり、不可能な場合もあります。 重複排除により、バックアップするデータの量が減り、選択したウィンドウ内でバックアップできます。



recovery復旧時間を短縮してダウンタイムを削減



テープではなくディスクに保存するバックアップデータを少なくすると、復旧時間が大幅に短縮されます。 ドライブを使用すると、ヘルスチェックやその他のフェールセーフ操作も簡単になります。



storage保管コストの削減



重複排除はバックアップデータを削減し、その結果、ストレージ容量の要件を削減します。



protection保護が必要な企業データの成長の問題に対する部分的な解決策



重複排除を使用すると、増加する保存データの量を効率的にバックアップできます。



securityセキュリティの向上



重複排除とレプリケーションを組み合わせることで、コピーをオフサイトに保存できるため、手動でテープを操作する必要がなくなり、セキュリティが向上します。



branchブランチ構造を持つ組織でのデータ保護の簡素化



リモートブランチオフィスデータには、集中的な保護と復旧が必要です。 集中化されたバックアップを使用すると、重複排除により、大量のバックアップデータをネットワーク経由でデータセンターに転送するプロセスが簡素化されます。

一般に、重複排除は、データ量を増やし、バックアップウィンドウを観察しながら、バックアップインフラストラクチャコストの増加に対処するためのツールとして広く使用されています。



詳細に触れない場合、バックアップシステムでデータ重複排除を使用する主な理由は、バックアップのサイズの削減とストレージ容量の必要性の削減、ネットワークトラフィックの量の削減、バックアップウィンドウ（バックアップの時間）の削減です。 1回の手順で迅速なリカバリを保証するために、毎日フルバックアップを実行する機会があります。



重複排除バックアップシステムは、テープドライブとライブラリ、仮想テープライブラリ、ディスクアレイ、組み込み重複排除を含むストレージシステム、クラウドバックアップなど、さまざまなバックアップストレージオプションを使用できます。





重複排除により、転送および保存されるデータ量が大幅に削減され、ITインフラストラクチャのボトルネックが解消され、ストレージコストが削減されます。



重複排除により、容量要件を平均10〜30倍削減でき、データの回復と取得の速度と信頼性が大幅に向上します。 一意のデータのみをディスクに保存することは、高速災害復旧のために最小限のコストでネットワークを介してリモートサイトに複製できることも意味します。



ディスクスペース要件を最小限に抑えるために、一部のストレージシステムにはデータ重複排除が組み込まれています。オンザフライで（バックアップまたはアーカイブのプロセスで）実行されるため、既に重複排除されたデータがディスクに保存されます。



重複排除の種類とその有効性



重複排除にはさまざまなタイプがあり、それぞれの長所と短所があります。 データが到着するとオンライン重複排除（「オンザフライ」）が実行され、ストレージシステムに書き込まれる前であっても冗長性が排除されます。 これにより、I / Oとストレージの数が減りますが、重複排除の速度は、バックアッププロセスを遅くしないように十分に高くする必要があります。 通常、このような重複排除はデータソースで実行されます。



後処理は可能な限り高速のデータコピーを提供しますが、重複排除の前に一時ストレージに追加のストレージが必要です。 これらの方法が組み合わされることもあります。データ重複率が特定の制限に達するまでオンライン重複排除が行われ、その後、後処理処理に移行します。









ソース重複排除とポストプロセス（術後）重複排除の比較。



このようなトレードオフの理由は、重複排除のコストが高いことです。 コンピューティングリソースが必要であり、システムパフォーマンスの低下を引き起こします。 したがって、各方法を使用する際に避けられないコストを考慮する必要があります。



したがって、2つの主要な重複排除スキームがあります。最初の方法では、データはクライアント側（「ソース」）で処理され、重複排除はストレージデバイスに送信されます。 その他の場合、重複排除はエンドポイントストレージデバイスで発生します。 最初の方法は、ネットワークの負荷を軽減しますが、クライアント、つまりクライアントのコンピューティングリソースに特殊なソフトウェアをインストールする必要があります。 2つ目は、より強力で高価なストレージシステムの使用を伴い、初期データ転送のコストは実際には削減されません。





重複排除は、ソースで実行するか、データウェアハウスに直接実行できます。 これに応じて、計算負荷が分散されます。これは、ブロックサイズが変更されたときにも変更できます。 いずれの場合でも、システムリソースのかなり高い重複排除要件を考慮する必要があります。



重複排除は、データが非常に冗長である場合、および小さな変更が加えられた後にコピーおよび/または保存される場合に最も効果的です。 一般に、ドキュメントファイル、仮想マシン、電子メール、アーカイブなどの非構造化データは、重複排除率が高くなります。 ボリュームはしばしば20〜30倍減少します。 データベースなどの構造化データの重複排除では、係数が低くなります（通常5〜8倍）。





例による重複排除の有効性。



一部のハッシュベースのスキームは、データ前処理を使用して重複排除係数を増やします。 クライアント側でデータを処理する際の可変長ブロックの重複排除により、一意のセグメントのみが保存されるため、バックアップ時間が大幅に短縮されます。 この重複排除方法は、データセットのわずかな変更でもファイル全体のバックアップにつながる場合、従来の固定長セグメントの重複排除よりも効率的です。

Avamarを使用するシナリオ



▍1。 仮想化環境



ゲストOSレベルとVMware vSphereおよびMicrosoft Hyper-Vイメージレベルの両方で仮想マシンをバックアップおよび復元できます。 Avamarは、VMware vCenter、VMware vCloud Director、データ保護のためのVMware vStorage API、vSphere Web Client、Microsoft Hyper-Vと統合します。 また、Avamar Virtual Edition（AVE）の特別なバージョンもあります。これは、仮想環境を保護するための専用のソフトウェアソリューションです。









ストレージシステムおよびソフトウェアエージェントの「データスナップショット」機能を使用したVMwareでのVMバックアップ用Avamarアプリケーション。



VMware環境では、AvamarはVMware Changed Block Tracking（CBT）を使用して、変更されたブロックを追跡します。 これにより、バックアップとリカバリのプロセスが高速化されます。 また、スループットは、複数のプロキシVMにわたるインテリジェントな負荷分散によって強化されます。









VMware仮想環境の保護におけるAvamarの利点。



Avamarは、個々のファイルまたは完全なVMイメージを、元のVM、代替VM、または新しいVMにワンステップで迅速にリカバリします。 VMイメージがData Domainに保存されている場合、vMotionを使用して実稼働環境に移動することにより、仮想マシンへのアクセスをすぐに取得できます。









ゲスト仮想マシンのバックアップ。



ESXiサーバー上の仮想マシンをバックアップする製品であるVMware vSphere Data Protection（VDP）は、既製の仮想モジュール（仮想アプライアンス）として展開されるAvamarソリューションに基づいていることに注意してください。 ソフトウェアはエージェントなしで動作し、CBTを使用し、VM全体ではなく、ブラウザを介して個々のファイルも復元できます。





Dell EMC Avamarは、VMware vSANで実行されている仮想マシンのバックアップとリカバリもサポートしています。



AvamarをvRealize Automation（vRA）およびvCloud Director（vCD）と統合することにより、パブリック、プライベート、ハイブリッドクラウド、およびクラウドソフトウェアのデータ保護サービスを使用できます。 VRA拡張機能vCD Data Protection拡張機能は、バックアップサービスをvCloud Directorに直接統合します。



また、Avamarは、Hyper-VやAzureなど、Microsoftのプライベートおよびサーバーベースのクラウドのデータ保護を提供します。 管理者は、Microsoft System Center Virtual Machine Managerを使用して適切なデータ保護ポリシーを割り当てます。 ポリシーは、仮想マシンのリソースをプロビジョニングするときに適用されます。



ちなみに、Data Domain Cloud Tierは、Azure、Amazon、Virtustream、およびS3プロトコルで動作するデバイスをサポートしています。

Avamarは、イメージベースのバックアップ方式を使用して各VMを保護し、Hyper-VファイルシステムでVMファイルを直接スキャンします。 このようなデータ処理は、エージェントを使用するよりも効率的です。 クラスター共有ボリューム（CSV）により、複数のノードがクラスター内のすべてのディスクにアクセスできます。



▍2。 ネットワークストレージシステムのバックアップ



NDMP（ネットワークデータ管理プロトコル）を使用すると、NASバックアップ時間が短縮されます。 基本的な完全バックアップの長時間の作成をオプトアウトできます。









Avamarでは、Avamar NDMP Acceleratorノードを使用してネットワーク接続ストレージをバックアップおよび復元できます。



Avamar NDMP Acceleratorノードを使用する場合、ゼロレベルバックアップは最初の完全バックアップ中に1回のみ実行されます。 レベル1の増分ダンプのみが適用されるため、バックアップ時間とNASリソースへの悪影響が大幅に削減され、復旧が簡素化され、高速化されます。



ただし、バックアップウィンドウに準拠するために、ファイルとボリュームの数またはサイズに制限はありません。 マルチスレッドのおかげで、Avamarは大規模で水平方向にスケーラブルなNAS（Dell EMC Isilonを含む）で動作し、コピーデータにより高い帯域幅を提供します。



Avamarの高速増分バックアップアーキテクチャはHDFSでも機能し、従来のファイルベースのバックアップに固有のボトルネックを排除します。

Avamarは、バックアップ時間と変更のみの毎日のストレージを削減し、ネットワーク帯域幅を最適化することで迅速に復元とバックアップを行う機能を備えています デスクトップおよびラップトップユーザーは、セルフサービスリカバリ機能を有効にできます。



Avamarデータ重複排除機能は、他の同様のツールよりも高度です。 グローバル重複排除の特異性は、外部デバイス、セッション、データのタイプや数に関係なく、データが1回コピーされ、重複排除プールに1回保存されることです。



たとえば、Veeamは、同じバックアップジョブ内の重複排除によって特徴付けられます。 新しい次のタスクは、重複したデータでも再度コピーされることを意味します。 HPEバックアップソリューションでは、システムに2つの論理デバイスが作成されている場合、これらのデバイス間にグローバルな重複排除はありません。
