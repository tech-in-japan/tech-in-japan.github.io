インサイドアウトのGPRS。 パート1

この一連の記事を使用して、モバイルオペレータのネットワークでのパケットデータ送信の技術について居住コミュニティに伝えたいと思います。 パケット交換（PS）コアネットワークの模式図を見て、さまざまなネットワーク要素間の通信に使用されるプロトコルスタックを見て、モバイルネットワークでパケット伝送を使用できるようにする主な要素の機能を詳しく見ていきます。 具体的には、この記事では現時点で最も一般的なGPRS / EDGEテクノロジーに焦点を当てます。





物語


だから、私たちは、事業者のモバイルネットワークの開発の初めにデータ転送の面で何を持っていました。

いわゆる「カウントダウン」を始めましょう CSD [回線交換データ]。 この技術はGSM規格に登場し、モデム内蔵または加入者のデバイスに接続されたモデムを使用して接続を確立することを可能にしました。一方、基地局送信機の加入者には1つのタイムスロット（TS）

CSDを使用したデータ転送は、通常の音声通話と実質的に違いはありません。 通話時には、あなたは完全にチャネルを占有します。したがって、そのような接続の料金は1分ごとに実行され、モバイルネットワークの開発の夜明けには決して小さくありませんでした。

モバイルネットワークでのデータ伝送の開発における次の段階は、CSDテクノロジーの改善でした-HSCSD（en） [高速CSD]テクノロジーが登場しました。 この技術を使用すると、4 TS +を組み合わせてデータ転送速度を上げることができ、「単純化された」エラー修正方法の使用により、1つのチャネルのスループットが14.4 Kbit / sに増加しました。 したがって、HSCSDの最大スループットは57.6 Kbpsでした。

低い伝送速度と毎分課金にもかかわらず、この技術は、主に最新のデバイスでの使いやすさにより、システム内の少量のデータ、たとえばセキュリティアラーム（メーターの読み取り値、インジケーター）の転送に引き続き人気があります。



パケットデータ伝送技術の出現（2000年/ 2001年にフェーズ1仕様が登場）によってすべてが変化しました。GPRS [General Packet Radio Service]は、データ伝送帯域幅を大幅に増加しました（8 TSが使用される場合、最大伝送速度-171、 2 kbps）、およびパケットスイッチングも使用しました.CSD / HSCSDのチャネルスイッチングとは異なり、基地局でリソースをより効率的に使用できましたが、同時にこの技術はネットワーク構造に追加要素を導入する必要がありました-SGSN GGSN。

基本的に、 EDGEテクノロジー[GSM Evolutionの拡張データレート]は、実際にはGPRSと違いはありません。 既存のネットワークに実装できます。 EDGEの実装の変更は、無線インターフェイスのコーディングスキームの変更、およびネットワーク要素のソフトウェアの変更に関連しています。 EDGEが提供できる最大速度は473.6 kbps（8タイムスロットx 59.2 kbps）です。



したがって、GPRS / EDGE技術は、回線交換技術（CSD / HSCSD）と比較して、エンドユーザーに以下を提供できます。

  • 高速伝送
  • セッションを開く時間の短縮
  • より有利な使用率
  • 毎分ではなく、転送されたデータの量に応じて課金
  • データ転送中にチャネル全体を占有しないでください


モバイルネットワークでのデータ転送の新しい原理の出現は、GSMネットワーク自体のアーキテクチャの変更を必要としたため、ネットワークアーキテクチャを見てみましょう...

GPRSネットワークアーキテクチャのコア要素はSGSNです。 図からわかるように、SGSNはさまざまなインターフェイスによってGSMネットワークアーキテクチャのほとんどの要素に接続されています。 オペレーターのパケットネットワークにおけるSGSNの「衛星」はGGSNです 。これは、オペレーターのIPバックボーンと他のパケットデータネットワーク（PDN）の間の一種のブリッジです。 「通常の」CiscoルーターはGGSNの鉄版として機能しますが、Nokia Siemens Networks（NSN）、Huaweiなどのベンダーとは別のソリューションもあります。 ほとんどの場合、通信事業者のネットワークには同様の要素がいくつかあり、ネットワークの容量とテリトリーの負荷によって決定されます。



機能的にSGSNを使用すると、次のことができます。

  • 加入者にパケットデータを送受信する機能を提供する

    （モバイルインターネット/ WAP / MMS /イントラネット）
  • サブスクライバーの認証と承認
  • オペレーターに請求データを提供する
  • SMS_over_IPを送信します
  • 政府機関へのインターフェースを提供する
  • HLR / MSCの加入者データを監視および更新する、いわゆる モビリティ管理
  • ユーザーセッションを管理する


SGSNをさまざまなネットワーク要素に接続する可能性のあるインターフェイスを下の図に示します。



ここで、すべてのSGSNインターフェースを検討するのではなく、いくつかの重要な点にのみ焦点を当てます。



SGSNに関連付けられているすべてのインターフェイスは、文字「G」で指定されます （ 対応するドットと混同しないでください ）。それらの多くは必須であり、その他は特定の機能を導入し、補助的です。 特に、Gsインターフェイス（ MSCとSGSNの間）を使用すると、GPRSサービスを使用しながら音声サービスを送受信できます。 Gdインターフェイス（SMS-GMSCとSGSNの間）を使用すると、パケットネットワークでSMSメッセージを送信できます*。

*-ところで、SMS送信サービスは、事業者が収入を減らすことを好まない方法の鮮明な例です。 パケットネットワークを介したSMSの送信は、エンドユーザーにとってほぼ2〜3倍安く、オペレーターはこの機能の使用を「拒否」することがよくありますが、同時にGPRSのカバレッジはどこにもありません。 通常、デバイスでは、SMSを送信する機能は、メッセージ->メッセージ設定->テキストメッセージ->パケットデータを使用して設定されます。

上記の2つのインターフェースはオプションですが、オペレーターのネットワークに機能を追加します。 異なるインターフェース間でデータを転送するために使用されるテクノロジーはFR / IP / ATMですが、最近では技術的に最も簡単に実装され、データ交換のための費用効果の高いトランスポートであるIPバックボーンに切り替える傾向があります。



動作原理


つまり、パケットネットワークの原理は次のように説明できます。

1.基地局コントローラ側でのパケット送信のためのリソースの割り当て*

*-これには、音声サービスの優先度が考慮されます。

2.いわゆる加入者端末の識別を含む、加入者認証手順（GPRS接続）の実行 IMEIチェック*

*-オプションです。

3. HLRの加入者の位置情報を更新する

4.ストリーム暗号化キーネゴシエーション

5.加入者の端末デバイスとPSコアネットワーク間の通信を確立します。これは、GPRS / EDGEアーキテクチャの用語ではPDP（パケットデータプロトコル）コンテキストのアクティブ化と呼ばれ、要求されたデータのタイプに依存します-モバイルインターネット/イントラネット/ Wap / MMS / SMS_over_IP

6.パケット送信サービスの使用が終了すると、加入者は切断されます-PDPコンテキストは非アクティブ化されます

7.加入者の端末デバイスがパケットネットワークとの永続的な接続用に設定されていない場合（ほとんどのデバイスでメニュー->設定->デバイス接続->パケットデータ->パケット接続->オンデマンド/パーマネントアクセスで確認できます）、実行される操作、つまり加入者の初期認証と承認の逆、いわゆる GPRSデタッチ。



このセクションについては、記事の第2部でさらに詳しく検討します。 承認プロセス中に転送されるデータ、およびサブスクライバー/ SGSN'a側に保存されるデータを決定します。GPRS/ EDGEアーキテクチャで使用されるいくつかの暗号化アルゴリズムに触れます。



見込み


GPRS / EDGEテクノロジーを改善できる有望なテクノロジーは、直接の「相続人」です。EGPRS2/ Evolved EDGEは、Nokia Siemens Networks（NSN）やNortelなどのベンダーによってサポートされています（テクノロジーは3GPP Rel-7によって標準化されました）。

Evolved EDGEにアップグレードするには、既存のEDGEネットワーク上のソフトウェアをアップグレードするだけで十分です。一方、サプライヤーは、Evolved EDGEテクノロジーがEDGEで行われる方法と比較してスペクトル効率を2倍以上にできると約束しています。

特に、新しい標準への移行後、ユーザーはネットワークから最大1.2 Mbps（ダウンリンク-基地局から加入者への方向）でデータをダウンロードし、最大473 kbpsの速度で基地局（アップリンク）への方向にデータを転送できます。 / s

メーカーによると、GSMテクノロジーのこの進化により、通信事業者は次世代テクノロジー、特にLTE 、およびGSMと次世代モバイルデータ間のサービスの完全な互換性への費用効果の高い移行を実現できます。
