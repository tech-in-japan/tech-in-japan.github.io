Transmissionが新しいリリースをリリースしました-試してみてください！



素晴らしいTransmission 2.21クライアントの新しいバージョンがリリースされました。 さらに、2人は2日間の休憩で2つのバージョンをリリースしました-2月6日-トランスミッション2.20、2月8日-トランスミッション2.21。



そして、変更のリストからわかるように、彼らは無駄にしようとしませんでした：



すべてのプラットフォーム用

  • tarアーカイブ2.20のコンパイルエラーを修正
  • ボリュームがオフラインでもダウンロードしたファイルを記憶する
  • 状況によっては、ファイルの再確認が不要になりました
  • エラー「開いているファイルの数を制限する」が修正されました
  • トレントでの総座っ時間とダウンロード時間の表示
  • ウェブ側の修正
  • IPv6のみのトラッカーのより良いサポート
  • RPCを介して送信セッションを終了する機能を追加しました
  • NAT-PMPおよびUPnPがUDPポートを転送するようになりました
  • dht-0.18のDHTコードを更新
  • 暗号化されたデータの暗号化の改善
  • スクリプトサポートの改善-トレントのダウンロードが終了した瞬間
  • 後者が404ステータスエラーを返す場合のトラッカーポーリング間隔を修正
  • uClibcバージョン0.9.27以降を実行しているプラ​​ットフォームで修正されたチェックサムのバグ
  • Solarisのmemmem（）エラーを修正

  • インターフェースの変更

これらのバージョンのクローズドチケットのリストも利用できます 。



新しいバージョンを試すには、公式ウェブサイトから直接ダウンロードできます 。

さまざまなオペレーティングシステムの標準配信形式に加えて、ソースコードを取得してコンパイルすることもできます。



また、Ubuntu MaverickおよびNattyの場合、PPAを使用できます。 

sudo add-apt-repository ppa:transmissionbt/ppa sudo apt-get update sudo apt-get install transmission-gtk




ところで、更新時に、別のパッケージに属するファイルを、伝送共通パッケージのファイルで上書きできないという問題が発生する場合があります。 これを修正するには、次のコマンドを使用します（目的のバージョンのファイルに置き換えます）。 

 sudo dpkg -i --force-all /var/cache/apt/archives/transmission-common_2.21-0ubuntu1.10.10.1_all.deb
