Undead Carnageの作成者へのインタビュー：Windows Phoneの償還

ゲーム「 Undead Carnage：Redemption for Windows Phone-Artyom Veselovsky」のクリエイターへのインタビューをお届けします。 sashaeveにインタビューしました 。



[Alexander]こんにちは、Artem。 あなた自身、あなたのチーム、そしてUndead Carnageゲームを作成するアイデアがどのように生まれたかについて少し教えてください。



[アルテム]こんにちは！ 私の名前はArtyom Veselovskyです。25歳です。私たちのチームは3人で構成されています。デザイナーのSasha Synyava、プログラマー、レベルデザイナーのOrest Blagodarです。エンジンコアと3Dアニメーションの開発を担当しています。 ゲーム開発は自由時間に行われます。 私たちの一部にとって、これは高校時代から夢です。誰かが後で夢を見るようになりましたが、私たち全員が最初にゲーマーになりました。



別のチームに集まった瞬間から、スマートフォン向けのアクティブなゲームを作りたかったのです。 多くの試行がありました...多くのアイデアがあり（まだありました）、多くのプロジェクトが始まり、概念実証のスタイルで多くのものがありました。





このアイデアは、昨年の夏の初めに生まれました。 その後、電話でゲームの最初の動作ビルドを作成しました。 テストにはHTC HD2を使用しました。 率直に言って、彼女は曲がっており、くて退屈でした。 特に退屈な...ゲームは2Dゾンビアクションと呼ばれていました。 多くの場合、エンジンの一部、バランスの取れたダイナミクスを作り直しましたが、彼女は常に設計アプローチに欠けていました。 3Dゲームを夢見て、開発を停止しました...



彼らは3Dシューティングゲームを作成し、3Dグラフィックスのプログラミングについて多くのことを学び、独自の3Dエンジンを作成し、戦車戦をテーマにしたデモゲームを作成しました。 彼らはそこに人工知能を追加し、電話の性能に依存しました。



その後、どういうわけか、彼らはゲームがカテゴリの1つであったMicrosoftウクライナとノキアからのWindows Phone開発コンテストの発表を見ました。 参加することにしました。 時間はほとんどなく、ゼロから行うことはありませんでした。 そして彼らは、私たちがほとんど機能するゲーム、ツール、アイデアを思いついたことを思い出しました。 彼らは座って、それについて考え、模擬物語を書き、責任を共有し、仕事に取り掛かりました。



最初は、3つすべてがプログラムされましたが、デザインの欠如は非常に顕著であり、ゲームプレイのどのトリックとも重なりませんでした。 そして三亜が登場しました。 Visual Studioを使用して、彼は完全にPhotoshopに切り替えました。 そして、最初の開発の後、ゲームはクールなデザインと雰囲気になりました。 私たちがやったことを見て、今、私たちの最初のゲームはUndead Carnage：Redemptionです。



[Alexander]なぜ償還？



[Artem]償還とは償還を意味します。 ゲームの歴史は、ゲームのイベントの数年前に愛する人を失った男の話を伝えますが、これらの数年の間に起こったことは何も言いません。 はい、そして物語は続編が示唆するような方法で書かれています。 過去の罪のtoneい。 パートはゲーム自体を伝えます。 「過去」に関する部分については、後で説明します。 一般的に、プレイ:)



[Alexander]作成するのにどれくらいの時間とリソースがかかりましたか？



[Artem]開発は常に3人で行われました。 時間がかかりました。 コンテストの作業バージョンを急いで作成しました。 それで、ゲームのデモ版ができました。 彼らは物語の始まりと、ゲームの完全には完成していないデザインを示しました。 絶対的に言えば、ゲーム自体に約5か月かかり、エンジンとツールを開発するのに約1か月かかりました。



[Alexander]ゲームの技術的な側面について教えてください。



[Artem]当初から、彼らはテクノロジーに非常に力を入れていました。 鉄を最大限に活用しました（それは、Windows Phone 7を実行するデバイスの第1世代のみでした）。 彼らは最大限の美しさをしました。 主な側面-これはもちろん、調整されバランスの取れた視差スクロールです。 そのアイデアは、異なるスケールとスクロール速度でグラフィックスのいくつかのレイヤーを描画することです。 これが適切に設定され、設計が正しい場合、単純な2Dゲームは3Dゲームの奥行きと遠近感の錯覚を作成します。 また、このゲームでは、ブルームエフェクトが実装されています。これは、Windows Phone 7での今日の唯一の実装である可能性が最も高いです。 キャラクターが明るい背景に対して色が暗い場合、彼の輪郭も強調表示され、背景でぼやけます。 後に、曲線空間とスローモーションの効果が追加されました。 実際、かなり複雑な描画パイプラインであることが判明しました。 シーン全体を画像に描画し、適切な場所に広げて照明します。 次に、この画像をポリゴンメッシュのテクスチャとして使用します。このメッシュは曲げることができ、それにより、たとえばガラス玉を与える曲率を作成できます。 後に敵に発砲する機能を追加しました。 デモで示した火の玉は、点火により多くの損傷を与えます。 燃えて焦げたゾンビが主人公に向かっていく様子を見るのは美しい:)



[Alexander]そして、誰があなたに音楽を書いていますか？



[Artem]彼ら自身...ギターのパートが録音されました（Sashaの弟であるMaxim Synyavaと、従業員であり友人であるAndrei Lutsukによって演奏されました）。 その後、サーシャはこのすべてをコンピューターで処理しました。 一部のトラックはまったく記録されませんでした。 つまり、これは最も純粋な形の電子的な創造物です。



[Alexander]ゲームを作成するときの最大の技術的問題は何でしたか？



[Artem]最初の問題は、SDKとプラットフォームの制限です。 電話はプログラム可能なシェーダーをサポートしていません。 はるかに小さいサイズではバッファーを使用できます（XNAの到達プロファイルと高解像度プロファイル）。 これにより、プログラマの能力が大幅に制限されます。 マイクロソフトによる奇妙なステップですが、原則として、私は問題が何であるかを推測します。 問題は、携帯電話のハードウェアとXNA 4.0の非互換性にあります。 XNAフレームワーク自体は、DirectX 9.0インターフェイスの単なるラッパーです。 また、Windows Phoneデバイスは、Shader Model 4.0（DirectX 10）を備えたGPU上に構築されています。 Microsoftのスタッフは、シェーダームーブを5つの組み込みのエフェクトで記述することで状況を少し節約しましたが、ほとんどの場合、これで十分ではありません。 Windows Phoneなどの場合は、SDKを最初から作成する必要があります。 ここでは、PS Suite SDKについてSonyに少額の謝礼をします。 すべてが5プラスで行われました。 C＃言語、Monoで動作する高レベルフレームワーク、シェーダー言語としてのNVidia CG、安全でないコードを使用する機能。 スプライトバッチさえありません。 自分で書くために必要なものはすべて、すべてがハードコアで、すべてが美しいです。



2番目の難しさ、そして同時に利点は、コードを特定のハードウェアにターゲットできないことです。 つまり、ケースの90％で第2世代の携帯電話で問題なく動作するものは、第1世代では大幅に遅くなります。 美しさを断ち切る必要がありますが、それは残念です。 しかし、現時点では、第一世代のデバイスはたくさんあるので、値引きすることはできません。



3番目の問題-ウクライナには開発者向けの市場がありません。 また、プログラムを自分でWindows Phoneマーケットプレイスに公開することでもありませんが、AppHubには登録された開発者だけが利用できるものがあるという事実です。 これは学習を困難にします。



4番目の問題は、Microsoft自体の無関心です。 ここでは、Xbox Liveについて説明しています。 Xbox Live for Undead Carnageについてマイクロソフトに書き込みました。 私たちは、ソーシャルゲームの機会を利用するための合意を望んでいました：成果、MSサーバー上のリーダーボード。 興味があれば連絡を取ると言われました。 私たちは連絡を取りませんでした... Facebookで何かを考えます（アラワルとノキアも肯定的な答えを与えませんでした-アレクサンダー） 。



まあ、最も重要なことは、理想に向かって常に努力することです。 しかし、それを達成することは不可能であり、ゲームは絶対にすべてのユーザーに好かれることはありません。 さて、正しいものと、実際には正しいものに合わない新しいクールなアイデアとの間の絶え間ないバランス。 このため、ゲームエンジンのメインモジュールを何度か書き直す必要がありました。 とにかく、これは経験です。



[Alexander] Undead CarnageはMicrosoftからの3つのコンテストに参加しましたが、勝つことができたのは1つだけです。 そして、これは彼女がロシアとアメリカのマーケットプレイスのリーダーだったときのことです。 その理由は何だと思いますか？



[Artem]これはいくつかのことと関係があると思います。 主な側面-ゲームには、対象となる視聴者と年齢の評価があります。 結局のところ、評価やトピックに関係なく、誰もが製品の品質を評価するわけではありません。 つまり、ゲームがうまくできていれば、見た目が美しく、サウンドがあり、筋金入りのゲームプレイを提供しているのに、なぜヒナギクとゾンビの間に線を引きますか？ おそらく、ゲームはモバイルデバイスには複雑すぎます...しかし、これはXboxモバイルプラットフォームです。つまり、ゲームはプラットフォームに似ている必要があります。



[Alexander]デモ版は合計で120,000回以上ダウンロードされました。 これは良い結果だと思いますか？



[Artem]これは「ペンテスト」に関しては素晴らしい結果だと思います（そしてチーム全体が同意します）。 それでも、これは私たちの最初のゲームであり、最後にもたらされました。 誰も資金を提供しておらず、夜間に自宅の99％のケースでゲームを開発し、時々会社に泊まってラップトップでゲームをした経験のないチームにとって、これは素晴らしい結果です。



[Alexander] Windows Phoneの将来について多くの話があります。 Windows Phone 7の展望は、ここと世界のどこにあると思いますか？



[Artem]繰り返しますが、私はチーム全体を代表して話をします。 プラットフォームは繁栄すると信じています。 最も重要な問題は常に人にあります。 彼らは何か新しいものを恐れています。 試すのが怖い。 わずかなマイナスで新しいものを破棄し、チャンスを与えません。 多くの人がWindows Phoneを一目で気に入っています。 そして、私は彼女と同じ知り合いでした。 幸いなことに、私はデバイスを購入しませんでした。 Windows MobileでHTC HD2を使用し、AndroidをインストールしてからWP7をインストールしました。 私はAndroidを理解せず、認識せず、取り壊し、返却しました。 しかし、その滑らかさと安定性を高く評価しました。 数日後、彼はAndroidを破壊し、WP7を返しました。 2010-2011年の冬でした。 そして、彼女はまだそこにいます:)そして、みんなは彼女を好きでした。



現在、会社にはWP7を搭載した10を超えるデバイスがあります。 新しいユーザーはすべてWP7に夢中になります。 昨日、WP7に対応したAndroid搭載携帯を持っている人を見るのはいいことです。 また、WP8は新しい開発ラウンドであり、WP7で定められたアイデアの継続であると考えています。 また、マイクロソフトは大規模な開発者だけでなく小規模な開発者にも感謝し始めると信じています。 これには多くの例があります。 ゲームでも。 WP7でも。 大規模で有名な開発者のゲームは常に優れているとは言えませんが、インディーズプロジェクトよりも悪い場合が非常に多くあります。 独立系開発者に機会が与えられれば、はるかに優れたエコシステムが存在し、プラットフォームの開発も改善されます。



ウクライナでは、これはもう少し複雑です。 繰り返しますが、これはマーケットプレイスであり、通常のモバイルインターネットであり、私たちの欲です。 多くの人は、これがソフトウェアにどのように支払われるかを理解していません...



[Alexander]さて、最後の質問、将来の計画は何ですか？



[Artem]ゲームを作成します:)正確には何ですか？ 時間はわかります。 WP7でゲームの品質基準を設定しようとすることしかできません。



私は個人的に、サシャと私がリヴィウのハッカソンで1日以内に書いた3DゲームであるDemon Smasherを完成させてリリースしたいと考えています（ハッカソンはwp7rocks-Alexanderチームによって実行されました） 。 たくさんのアイデアがありますが、どれも興味深いものです。 多くの開発、多くのコードが書かれており、ユーザーの携帯電話で動作する順番を待っています。



Undead Carnageの開発は多くのことを教えてくれました。 これで、開発中に問題が発生する可能性のある場所、問題の解決方法、さらには問題に対処する方法がわかります。 また、Undead Carnage：Windows 8を搭載したタブレットのリデンプションを移植したいという要望もありました。
