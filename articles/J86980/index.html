SMSの代替パス

ほとんどの携帯電話では、「メッセージ」メニューに移動し、標準設定に加えて「メッセージ設定」、「テキストメッセージ」の順に選択すると、「パケットデータを使用」オプションが表示されますが、残念ながら多くのオペレーターは伝えたがりませんより短いメッセージを送信する可能性について） オペレーターが基本的にパケットネットワークを介してショートメッセージを送信する機能を使用せず、 SMS - GPRS経由のサービスを提供する理由を見てみましょう。





イントロ



上記のオプションは、オペレーターのパケットネットワーク（ PSコアネットワーク ）を介して短いテキストメッセージを送受信する能力に過ぎず、音声チャネル（タイムスロット）に影響を与えずにパケット送信（GPRS / EDGE）専用に割り当てられた無線リソースを使用することを意味します）



一方では、このオプションはオペレーターにとって非常に興味深いはずです。 無線インターフェースを介して同時に送受信されるメッセージの容量を増やし、ネットワークを最適化しますが、パケットネットワークを介して160文字のテキストメッセージを送信するコストは最小限であるため、「実際の」SMSのコストについて疑問が生じます。



モバイルネットワークでのパケットデータ送信の標準によると、加入者がGPRS接続手順を完了した後、つまり オペレーターのネットワークで認証と承認を通過し、まだ​​複数のPDPコンテキストをアクティブにできなかったため、 GPRSを介したSMSの 1つの機能しか使用できません（オペレーターのパケットネットワークでの認証プロセスの詳細については、記事「 GPRS内部から。パート2 」を参照）



ショートメッセージサービス全体を考慮すると、エンドユーザーに提供できるいくつかの重要な機能（手順）を区別できます。

  • モバイル発信SMS（ MO-SMS ）-サブスクライバーの端末（MS）からショートメッセージを送信する機能。
  • モバイル終端SMS（ MT-SMS ） -SM-SCが加入者端末（MS）にショートメッセージを配信する機能。
  • Alert Service Center（ SC ）-この手順では、特定の加入者を「サービス」するSM-SCは、加入者端末（MS）がショートメッセージを受信する準備ができていることをHLR経由で通知します。 この手順は、次の2つの場合に使用されます。

    1. 加入者のモバイル端末（MS）は、この時点まで利用できない場合、加入者がアクティブであることを通知します。
    2. 加入者のモバイル端末（MS）は、端末自体のメモリオーバーフローが原因でSMSメッセージの受信が拒否された後、メッセージを受信するのに十分なメモリが「搭載」されていることを通知します。


ここで、問題の技術面を見て、モバイルオペレータのパケットネットワークを介してこのような機能を提供するために必要な条件を判断してみましょう。



技術的要件



SMSサービスがオペレーターのパケットネットワークを介した送信に非常に簡単に適応できるように、新しいネットワーク要素や新しい送信プロトコルを導入する必要はなく、既存のネットワーク要素間の標準化された接続インターフェイスを実装し、関連するすべてのネットワーク要素。



そのようなサービスを提供するために、 SGSN'a 、加入者の認証と許可を実行するネットワーク要素、および加入者データのバッチ送信のトランスポートリンクは、「きつく」参加する必要があります。



SMS over GPRSオプションがSGSNで利用可能になる条件：

  • 基地局サブシステム（ BSS ）とのGbインターフェースの存在
  • CAMELサービスの相互作用のためのGeインターフェースの存在、すなわち 課金プラットフォームへのインターフェース（GSM-SCF-プリペイド加入者用のSGSNとINプラットフォーム間のインターフェース）



    MO-SMSの場合（メッセージはモバイル端末から送信されます）、SGSNはSCPプラットフォーム（INプラットフォーム）から「許可」を要求して、プリペイド加入者にショートメッセージを送信します。 プリペイドサービスの加入者のアカウントに必要な資金の残高がない場合、INプラットフォームはメッセージの送信を拒否します。 SGSNがSMSメッセージの送信機能を提供する「承認」を受け取った場合、配信が正常に完了すると、SCP（IN）プラットフォームにこのイベントが通知され、メッセージを送信するためにサブスクライバーのアカウントの残高も更新されます。





  • SGSNとHLR間のGrインターフェイスの存在
  • SGSNとEIR間のGfインターフェイス（加入者機器の検証-IMEIチェック手順）
  • SGSNとSMS-GMSC間のGdインターフェイスの存在








さらに、最後の2つのインターフェースはオプションであり、オペレーターの要求に応じて接続されることに注意してください。 SGSN側に必要なインターフェイスが存在することに加えて、サブスクライバーのモバイル端末でショートメッセージを送信する優先方法を再定義する必要があります（最初の段落-設定で「パケットデータを使用」を選択）+もちろん、SGSN自体でSMSメッセージを送信する機能をアクティブにします。



メイントピックから少し脱線し、上記の図について少し説明します。

まず、図から、ネットワーク要素間のシグナリング専用に設計されたインターフェイスと、ネットワーク要素間のシグナリングとデータ転送の両方に設計されたインターフェイスがあることがわかります。 これら2つのタイプのインターフェイスの主な違いは、信号インターフェイスがさまざまなサブシステムとSS7プロトコルのメッセージを「処理」できることです。また、データ転送のインターフェイスでは、これに加えて、ユーザーとのペイロード転送も可能です。 たとえば、SGSNとHLR間のGrインターフェースは、主に加入者プロファイルのサービス情報の受信/変更/更新に使用され、加入者との間のパケットはGnインターフェースを通過します。



次に、GbインターフェイスはSGSNとベースステーションコントローラー（ BSC ）の間のリンクであるという事実に注目したいのですが、実際には、SGSNをBSCの一部であるパケットデータ（PCU）を処理するために設計された特別なボードに接続します。 これらのカードは、ベースステーションコントローラ（BSC）上のパケットデータ転送（GPRS / EDGE）の容量特性です。



第三に、Geインターフェースに注目すると、SGSN'a側とSCP（IN）プラットフォーム側の両方で実際にいくつかのコンポーネントを接続していることがわかります。 これらのコンポーネントは、SS7に関しては特別な論理モジュールです。SGSNの場合はgprsSSF、SCPの場合はgsmSCFであり、指定されたネットワーク要素にプログラムで実装されます。



パケットネットワークを介してSMSメッセージを送受信する機能に「影響」を与える可能性があるいくつかの条件は、音声サービスに関連するHLRのサブスクライバープロファイルのパラメーターです。 SMSメッセージを送信するための「標準」スキームは、音声サービスと密接に関連しています。

  • BAOC-発信通話の禁止
  • BOIC-発信国際電話の禁止（発信国際電話の禁止）
  • BOIH-自国のPLMN国に向けられたものを除く発信国際電話の禁止（加入者のホームネットワークに宛てられたものを除く発信国際電話の禁止）


原則として、音声サービスの禁止とSMSメッセージには区別がありますが、一部のデバイスはこれらの障壁を正しく処理しないため、音声サービスの禁止の存在がGPRS経由でSMSを送信する機能に影響を与える可能性があります。



上記に加えて、GPRS機能を介したSMSの実装を簡素化するために、いわゆる SGSN側のSMSルーティング設定 。これにより、SMSセンターの変更に関して、サブスクライバ側で変更を行う必要なく、メッセージが正しく配信されます。



モバイル終端SMS転送



例として、「標準」のメッセージチャネルを介したショートメッセージ（MT-SMS）の受信が、オペレーターのパケットネットワークを介した同じメッセージの受信と比較して根本的に異なることを見てみましょう。



上記の図からわかるように、「標準」スキームに従ってSMSメッセージを配信する手順は、原則として、MSC / VLRの代わりにメッセージ転送を実装および制御するキーリンクがSGSNであるという事実を除き、パケットネットワークを介してショートメッセージを送信することと同じです。 この状況は、「標準」スキームと比較して、パケットネットワークを介してSMS（MO-SMS）メッセージを送信する場合にも当てはまります。



まとめ



上記の説明全体からわかるように、既存のネットワークでは、オペレーターのパケットネットワーク経由でSMSメッセージを送信するための追加機能の導入は特に難しくありませんが、同時にネットワーク全体を最適化し、サブスクライバーに追加機能を提供できます。



SMS over GPRSサービスに関するすべての情報を要約し、特定します



欠点：

  • 加入者のモバイル端末によるサポートが必要です（原則として、ほとんどのデバイスがこの機能をサポートするため、これはもはや制限ではなくなりました）
  • オペレーターのGPRS / EDGEカバレッジは通常、GSM音声カバレッジよりも「弱い」
  • SMSメッセージをオペレーターに送信するコストを削減します。


と利点：

  • SMS_over_GPRSの送信速度は1分あたり最大30メッセージのレベルで決定されますが、「標準」スキームでは、速度は1分あたり6〜10メッセージのレベルで変化します（ 「Tssss ....！」のみ、それ以外の場合はスパマーが読み取ります:)
  • パケットデータチャネルを介して送信されるメッセージのコストを削減する機能
  • 負荷軽減+オペレータネットワークの最適化


しかし、問題はCISでこの機能が未請求のままであり、オープンのままである理由です...オペレーターはこの機能について知らないか、それを実装したくない、または何か他のものです... :)



UPD。 小さな説明：この機能を完全に確認するには、GPRS接続手順を実行する必要があります（Nokiaでは、オペレーターのパケットネットワークに永続的に接続することを選択できます：「メニュー」->「デバイスの接続」->「パケットデータ」->「パケット接続」->「永続的な「この場合、通常、文字「G」[「E」]が表示されます。これは、オペレーターのパケットネットワークで認証手順に合格したことを象徴しています）
