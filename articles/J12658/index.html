Rar + nnCronで重要な情報をバックアップする

重要なデータをアーカイブするために、非常に興味深く、シンプルで便利なプログラムを長い間使用してきましたが、2年半の作業で単一の障害は発生しませんでした！

このプログラムはnnCronと呼ばれ、xUSSR登録は無料です。 NnCronはwinrarと連携して動作し、いくつかの簡単な操作を実行します。おかげで、重要なフォルダーのアーカイブを6か月間保管しています。 当然、ディスク領域を犠牲にしなければなりませんでしたが、情報はもっと高価です。



winrarとnncronのインストールをスキップし、手順を説明します。

1.クラウンでwinrarを実行するタスクを作成します

2. Winrarは現在のアーカイブを作成し、適切な場所にドロップします

3. nnCronは、アーカイブが保存されているフォルダーをチェックし、特定の日数より古いファイルを削除します



実装。

1. nnCronでタスク（新しいタスク）を作成します。 たとえば、「01。 重要な文書をアーカイブします。」 起動時間（3泊など）、起動の頻度（毎日-毎日）を設定し、2番目のタブ「アクション」で「プログラムを実行またはドキュメントを開く」を選択し、アプリケーションまたはドキュメントパスの行に次のように入力します



c：\ Program Files \ WinRAR \ Rar.exe a -ag_yyyy.mm.dd e：\ _ backups \ www f：\ www



説明：

c：\ Program Files \ WinRAR \ Rar.exe a -ag_yyyy.mm.dd-必要なパラメーターを使用してrarを呼び出します。ここで、

a-ファイルをアーカイブに追加します

-ag_yyyy.mm.dd-現在の日付を次の形式でアーカイブ名に追加します



e：\ _ backups \ www-アーカイブを保存するフォルダー。wwwはアーカイブの将来の名前です。



f：\ www-アーカイブするフォルダー



スタートアップフォルダーに「 c：\ Program Files \ WinRAR \」と入力します



保存します。 したがって、毎日のデータバックアップの準備はできていますが、大量のデータがある場合、時間が経つとディスクがいっぱいになり、アーカイブを保存する場所がなくなるため、2番目のタスクを作成します



2.新しいタスク「02。 古いアーカイブの削除»

タブでは、頻度と開始時刻はまったく同じように構成されていますが、開始時刻までに1時間追加されます。 念のために、突然、アーカイブが長い間詰め込まれています。

2番目のタブ「アクション」で「ディレクトリの消去」を選択し、「パス/マスク」行に入力します



e：\ _バックアップ\ *。*



mask *。*は、このフォルダー内のX日より古いすべてのファイルが削除されることを意味します！


「Age」の行に、アーカイブを保存する日数、たとえば100（日）を書き込みます。

保存します。



運用中に、アーカイブに必要なスペースを自分で決定し、保管日数を変更します



このようにして、ローカルで行ったすべての開発をアーカイブします。

さらにタスクを作成したら、mysqlデータベースをダンプしてアーカイブします。



ちなみに、dos rar.exeは登録を必要としません。nnCronジョブ設定では、タスクの開始方法（最小化ウィンドウ、非表示ウィンドウ、通常ウィンドウ）、タスクの優先度、およびタスクを開始するユーザーを指定できます。
