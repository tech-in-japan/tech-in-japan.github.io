Web Standards Days：永遠のフロントエンド値





会議に出席することは、同僚と会ってコミュニティに参加するチャンスであるだけでなく、専門的な成長の機会でもあります。 状況によってそのようなイベントに参加できる場合は、必ず参加してください！ 私自身のアドバイスに従って、1月28日にWeb Standards Daysを訪れました。 何らかの理由でこの会議に参加できなかった場合は、1日分の録画が既に投稿されているYouTubeに追いつくことができます。 個々のレポートの記録は後で表示されますが、今のところは個人的な印象を共有します。



注目に値する最初のことは、会議が非常に人気があり、印象的な数の参加者を獲得したことです。 登録時にさえ、私は並んでいなければならず、レポート自体の間に、多くの人が立ち上がったスピーカーに耳を傾けました。 このわずかな不便さ、そして私がこのイベントに初めて参加したという事実にもかかわらず、私が自分の場所にいるという感覚は私を離れませんでした。 ランプの雰囲気についてWSDチームに特別な感謝を申し上げます。また、フロントエンドコミュニティの多くの馴染みのある顔を再びご覧いただく機会を設けたいと思います。 そのようなイベントでの偶然の一致とランダムな会議は驚くことをやめませんが、それについては後で詳しく説明します。 私が報告したい主なことは、報告間で追跡されてきた一般的な傾向に関するものです。 私が個人的に聞いた演奏についてのみ話すように、私はすぐに予約をしなければなりません。 それでも、イベントは長い労働週間の後に土曜日に開催され、すべてに十分な力がなかっただけです。



まず、Reactが議論されなかった場合、どのような会議になりますか？ 彼なしでどこに！ すべてのフロントエンドヒップスターは、Sberbankの統合フロントエンドシステムのプロジェクトでこのフレームワークをどのように使用するかについて、Alexandra Vorobyevaのプレゼンテーションを楽しみました。 bem-react-coreパッケージは、興味をそそりました。レポートの著者であるウラジミール・グリネンコは、「すでにあなたの静脈のスムージーです！」



同時に、有名なSVGO Lev Solntsevのメンテナーが正確に指摘したように、フレームワークは行き来し、「永遠の価値」が残っています。 1999年に彼が最初のWebサイトを作成したという事実を考えると、彼はこの問題に信頼することができます。彼は、フロントエンド市場の典型的な傾向を追う十分な時間を持っていました。 では、どのような「永遠の価値」について話しているのでしょうか？ つまり、HTML / CSS / JSデータベースと仕様、そもそも知識が必要です。 これは、LeoとVadim Makeevのレポート「My Vanilla CSS」、メンターとしての経験とアドバイスを共有したSergey Popov、およびWeb仕様のみを使用してブラウザに実装されたゲームについて語ったVsevolod Shmyrovによって言及されました。 このすべての情報からの一般的な結論は、前述のベースの強調がフロントエンド市場ではっきりと見えるということです。 そして、それは私を喜ばせざるを得ません！



説明します。 プリプロセッサ、アセンブラ、ライブラリ、およびフレームワークを研究する中で、多くの人は「永遠の価値」に関する知識がなければ本格的な専門家になることは不可能であるという事実を見失います。 私はこの立場を全面的に支持し、あらゆる種類のコレクターやその他の「魔法」がプロジェクトを改善すべきであるが、それらがなくても機能するという事実について、Vadim Makeevの言葉に加わります。 たとえば、ファイルを縮小するためにプラグインを使用できますが、このアップグレードがなくても、プロジェクトは実行可能なままです。 もちろん、ほとんどの場合、サイトの少なくとも1つの完成したページの形で作業の結果を表示するには、100,500のタスクを実行し、すべてのSassファイルをコンパイルして、すべての「ドライホット」を行う必要があることを理解しています。 しかし、これはそのような状況が正常であることを意味するものではありません。







ちなみに、ファンタジースタイルシート（Vadimの用語）を完全に放棄する必要があるかどうかはわかりません。つまり、この「砂糖」とCSSには存在しないさまざまな発明のすべてです。 時間が経つにつれて、最良の機能と機能が何らかの形で定着し始め、新しい仕様に実装されます。 例はCSS変数です。 同様の傾向は、ECMAScriptでCoffeeScript機能を借用する場合にも見られます。 したがって、常に新しいものを空想して試してみる価値があります。 誰が知っている、突然あなたが思い付くものが仕様になります！



最後に、約束どおり、イベントの雰囲気と人々に戻りたいと思います。 以前の学生と現在の学生、職場や他の会社の同僚、すでに愛されているスピーカー、以前の同様の会議で覚えていたおなじみの顔を見ることができてとても嬉しかったです。 コミュニティの一員であると感じ、多くの思いやりのある人々に囲まれていることは、何とも言えない楽しい感覚です。 さらに、Web Standards Daysでは、オンラインコースで学んだ人たちにも会いました。 ずいぶん前のことで、お互いをライブで見たことはありませんでしたが、お互いを認識しました！ いくつかのフレーズとストーリー、そして私は自分自身と彼らの作品の両方を即座に思い出し、その時代の懐かしさに陥りました。 結局のところ、結果よりも多くの恐怖がありました。 そして今、会議はちょうど最初に、最初の教師を思い出し、お互いにどのように競争したか、過去の面白い話を笑うことを可能にしました。



間違いなく、すべての可能なイベントに参加する必要があります！ このようなイベントに参加しているすべての人に会えてうれしいです。 日付と場所の短いリストは、私があなたを取得し、自分でそこにいるようにすることをお勧めします：



  • ノボシビルスクでのCodeFest 4月1〜2日。
  • piterccs_conf 6月16日、サンクトペテルブルク。


重要なイベントを見逃した場合は、コメントでお知らせください。 あなた自身が1月28日にWSDに参加していた場合は、コメントのレポートに関するあなたの感情や考えも共有してください。 私自身が注意を払っていないことを学ぶのは面白いでしょう。



そして最も重要なことは、フレームワークは行き来するが、永遠の価値は残るということを忘れないでください！
