EclipseとSFTPは問題ありません

長い間、AptanaとEclipseには多くのトラブルを引き起こす問題がありました。つまり、sshアクセスが可能なリモートマシンで正常に動作できないということです。 Aptanaはプロ版でのみsftpを完全にサポートしており、公式にサポートされていないsftpプラグインには多くの問題があります（最も重要なのは、ネットワークがドロップおよび発生すると、プラグインは何も起こらなかったように動作し続けますが、ファイルを保存しません）、Eclipseにはすべての利用可能なプラグインがあります少なくとも1年は更新されていません。



幸いなことに、解決策が存在し、彼の名前はRemote System Explorerです。 RSEは、ファイルへのリモートアクセスの問題を解決するだけでなく、リモートターミナル、ファイル検索、さらにはリモートデバッグも提供します。



インストールは簡単で、Update Managerで「Remote System Explorer End-User Runtime」を見つけてインストールします。 追加のリポジトリを接続する必要はありません。 インストール後、ビューリストに新しいカテゴリ「リモートシステム」が表示されます。最も興味深いのは、既存の接続のリストを表示し、新しい接続を追加できるビュー「リモートシステム」です。



PSこれで、どのリモートフォルダからでも、コンテキストメニューの「リモートプロジェクトの作成」アイテムを使用してプロジェクトを作成できることを忘れないでください。 （これは、接続プロパティでルートパス設定を探すユーザー向けです）
