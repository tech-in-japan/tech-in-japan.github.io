App Storeの新しいセクション-「Best New Game Updates」



多くのモバイルアプリ開発者は、App Storeの「視認性」の悪さについて不満を述べています。 リリース後最初の数か月でゲームが点灯しなかった場合、ストアのメインページにアクセスする機会は日々少なくなっています。 すばらしいプログラムであっても、大量のジャンクアプリケーションや低品質のアプリケーションにdrれることがあります。



2番目のポイントは、すべての企業が初めて高品質の製品を生産できるわけではないということです。 アプリケーションに重大なエラーが発生した場合、再び視聴者を取り戻すことは非常に困難です。



次の状況：最も重要なリソースは時間であるため、多くの人がアプリケーションの機能を部分的に更新します。 そのため、アプリケーションが非常に興味深いものになると、すでに6か月間店頭にあり、数千人しかインストールしておらず、他の人はそれについて学習していません。 一部の開発者は単純にそれをたたき、他の開発者は作成を新たにレイアウトします。 たとえば、KingやSupercellのような大企業は、アップデート後にすでに人気のある製品をAppleと共有することに事前に同意しています。 しかし、誰もがこの機会を利用するわけではなく、誰もが成功できるわけではありません。



Appleは開発者の意見を聞き、「Best New Game Updates」セクションを作成したようです。 私たちの観察によると、すでによく証明されているゲームがあります。 おそらく、Appleはグロスですでに成功を収めている人だけをサポートするツールを作ったのでしょう。 観察します。





インストール数がアプリケーションの重量に依存する方法



Fiksuは、アプリケーションのサイズに応じて、インストールの変換統計を計算しました。 そもそも、統計ではゲームアプリケーションとモチベーションの高い広告のみを考慮することを強調することが重要です。フィクス自身がレポートの最後に脚注でこの情報を掲載しています。 この事実を理解していないと、誤った状況が発生する可能性があります。 したがって、これは、やる気のある広告を使用した場合のインストールへの変換の有効性に関するデータです。



一般に、統計には新しいものは何も表示されません。アプリケーションの重量が大きいほど、インストールへの変換が少なくなります。 0〜20 MBのアプリケーションの最大効率は87.94％です。 ただし、100MBを超えるアプリケーションの場合、変換に大きな違いはありません-76.31％。 やる気のある広告を使用する場合、アプリケーションのサイズがそれほど重要な要素ではないことは重要です。 技術の開発が活発になればなるほど、この差は少なくなります。ユーザーが小型アプリケーションと100 MB以上のアプリケーションの両方をすばやく簡単にインストールできる場合、インストール数の差は滑らかになります。
