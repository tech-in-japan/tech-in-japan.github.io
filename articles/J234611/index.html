時間追跡システムのインストール：自分で試してみてください





背景から始めましょう。 会社では、マネージャーは部下が勤務時間中に何をするかを知りたがっていました。 システム管理者は従業員のコンピューターを監視するように指示されましたが、従業員はこれを不器用に行う必要があり、常に倫理的ではありませんでした。



最後に、管理者の忍耐力が切れ、彼は勤務時間を記録するシステムを確立することを提案しました。 はい、単純ではありませんが、彼女自身が従業員がどれだけの時間を「野郎を蹴る」か、どれだけ「仕事で忙しい」かを分析します。 はい、はい、あなたに同意します、システム管理者は不親切でした。 従業員の生活を台無しにした今、上司は自分のお気に入りのZhezheshechkiとソーシャルネットワークでどれだけの時間を費やしているかを知ることになります。 しかし、これは別の話です。



初期システム要件：



-多数のPC（たとえば、さらに100台を追加する見込みのある200台のマシン）で速度が低下しないようにするため。

-可能な限り自動化されます。

-データが内部サーバーに保存されるようにします（クラウドのリーダーシップは決して好きではありません）。

-マネージャーに長時間使用する方法を教えるのではなく、便利に働くため。

-まあ、「塩とコショウを味わう」、つまり、他の有用な機能は禁止されていません。



「行こう！」（C）


そこで、システムをインストールすることにしました。 Google検索のゴーグルをひっかいて、5人の候補者を見つけました。 最初に検索するのはCrocoTimeとYawareで 、次にOffice MetricとDisciplineはさまざまな成功度でお互いを打ち負かそうとしますが、5番目のシステムであるStakhanovetsは連合への憧れからテストを開始しました。



ややれ



サイトで「無料でYawareを試す」ためには、フォームに記入して登録する必要があります。 スタンドアロンバージョンについては何も見つかりませんでした、テクニカルサポートに書きました。 彼らはすぐに答え、サイトに登録するように頼まれ、マネージャーはダウンロードへのリンクを提供しました。 Yawareのスタンドアロンバージョンである同志の重量は、ほぼ4ギガです。 ポップコーンと忍耐力を蓄えます。



パッケージには、プログラムがインストールされているオペレーティングシステムの手順とイメージが見つかりました。 データ漏えいを防ぐために、テスト中にインターネットをカバーしました。 VMware仮想マシンをインストールし、プログラムを起動し、YawareとともにOSイメージをダウンロードしました。 イメージを開始しようとしたときに、エラーが発生しました。 VMWareをVirtualBoxゲストシステムにインストールできませんでした。



コンピューターにインストールしようとしましたが、数分で判明しました。 Yawareが推奨するフルフィルメントされた仮想マシン設定が起動され、インターフェースに入るためのIPアドレスを取得できませんでした。 サポートに書き込み、あらゆる種類のスクリーンショットを送信しましたが、3時間以内に何も達成できませんでした。



私はNATを介して作業することにし、アドレスが表示され、Webインターフェイスに入りました。 アクティベーションキーを生成するには、リンクをクリックする必要があります。 従順です 次に、サイトのアカウントにログインする必要があります。 数十回の試行の後、再びテクニカルサポートに連絡する必要がありました。 彼はキーを直接送信するように要求しました。少し議論した後、彼らは私のパスワードを変更しました。 彼らが言うように、戦争は戦争であり、昼食...私は食べに行きましたが、車を止めませんでした。



戻ってきて、タブを開いて、Webインターフェースにアドレスを書きましたが、何も得られません。 伝統によれば、私はスクリーンを送るように頼んで技術サポートを書いています。 私たちは1時間考えて、ログを送るように頼みました。 私は指示を読みましたが、これは開始されなかったWebインターフェースを介してのみ実行できることに気付きました。 技術サポートについてひどく考え、親切な言葉を書きました。 30分後、彼らはすべてを破壊し、最初からやり直すことを申し出ました。 この場合、サイトに再度登録し、新しい配布を受け取ります。キーも新しいものです。



暗くなってきました。 明日、それについて考えることにしました。



クロコタイム



Yawareを苦しめた後、彼は最後に2回目の試みを延期しました。 彼は検索エンジンの別のリーダーであるCrocoTimeのインストールに切り替えました。 サイトで3キロメートルのフォームに記入する必要がありました。組織の名前、役職、電話番号、PCの数が必要です。 私は、PCの名前と数を除くすべてについて嘘をつきました（指定された数のトライアルライセンスが発行されたと思われます）。



通常のexeファイルをダウンロードし、標準インストールを開始しました。 ブラウザーの指示ページでシステムが開きます。 最初に3台のコンピューターでシステムを試すことにしたため、統計収集エージェントを手動で取得する方法を選択しました。 5〜6分後にエージェントをインストールした後、統計は2つから取得されました。 1つは、どうやら、眠りに落ちた。 テクニカルサポートに連絡し、問題はマシンへのネットワークアクセスであると説明しました。 行った。



このサイトにシステムをセットアップするためのマニュアルがないのは奇妙ですが、技術サポートはリクエストに応じてpdfを送信しました。 すべてが非常に簡単になり、更新の構成段階で停止しました。 プロキシサーバーを構成する方法はありません。 テクニカルサポートに電話しましたが、プログラムサーバーがインストールされているコンピューターのプロキシサーバーに例外を設定する必要があると言われました。 私は長い間規制を設定しましたが、すべては論理的ですが、時間のコストはかなりのものです。



勤務時間と昼食のスケジュールが分かれていることが好きで、統計は生産的と非生産的として発行されます。 遅刻、欠勤、早期出発を監視します。 システムをサードパーティのソフトウェア製品と統合するためのAPIを提供し、グループポリシーを介してインストールし、Active Directoryから従業員をインポートできます。 システムの機能は、マネージャーのメールへの違反に関するアラートであるため、プログラムに参加するたびに警告が出されます。 上司はそれが好きだった。



7ユーザーの場合、ソフトウェアは無料です。 なんで？ 7人の従業員のマネージャーは追跡しませんか？ リモートワーカーのみの場合。



しつけ



このサイトには再び個人データが必要です（私は一般に、これがそのようなシステムの開発者の「スタイル」であることに気付きました-連絡先なしで何も与えないでください）。 フォームに入力するのが面倒な人には、ページを下にスクロールできます。別のメニューがあり、プロファイルなしですぐにデモが開始されます。



サーバーのバージョンに関する情報が見つかりませんでしたが、テクニカルサポート（番号が無料ではない最初の企業であるため、Skypeを介して通信することにしました）が再び心配でした。 マネージャーは、Teamviewerからプログラムをリモートでインストールすることを提案しましたが、実験の純度のために私は拒否しました。 彼らが配布物を送ることに同意した。



マイナス面：IISを含むシステム用の追加ソフトウェアの束。 その後、Postgres、ASP.NET MVC4パッケージ、最新のWindowsアップデート、.Net Framework 4が必要でした。ディストリビューション全体をダウンロードすると、パッケージの半分が不要になりました。



追加にピッキングに失敗した後。 プログラム、リモートアクセスを介してプログラムをインストールすることを決めた。 インスタレーションで、彼らは約1時間いじり 、 テキーラコーヒーを飲み、この映画を見ました。 プログラムのアクティベーションに問題があり、エラーが発行されました。 ワイヤーのもう一方の端で、彼らはディストリビューションを再コンパイルすることにしましたが、問題は残りました。



その結果、デフォルトでサポートされている10台のマシンでテストすることにしました。 気に入った場合は、テクニカルサポートにより、より多くのPC自体のライセンスがインストールされます。



したがって、10台のPCでテストします。 ユーザーを手動で追加する必要があることは好きではありませんでした。 インターフェイスは使用するには不便であり、統計データは互いにぶつかります。 従業員アカウントの種類を変更できることが気に入っています。管理者権限を任意の従業員に割り当てることができます。 レポートの配布（特定の時点で、従業員または部門の活動に関するレポートが選択した従業員に電子メールで送信されます。設定は構成可能です）およびマネージャーおよび従業員のアクセス権の設定があります。



オフィスメトリック



繰り返しますが、スタンドアロンソリューションが見つかりませんでした。テクニカルサポートに書きました。 プログラムをダウンロードするためのリンクと5ライセンスのキーを送信しました。 通常の実行可能ファイル、パッケージをインストールすると、Firebirdが追加インストールされます。 わずかに一時停止、40秒キーを入力するようすぐに申し出ますが、まだメールが必要です。



追跡モジュールが生成されると、「データ収集モジュールファイルがプログラムフォルダに見つかりませんでした」というメッセージが表示されました。 テクニカルサポートに書き込みました。 フォルダー内のファイルの名前を変更することをお勧めします。 その後、追跡モジュールを取得しましたが、2回目です。 初めてフォルダが空になったとき。



追跡モジュールをコンピューターにインストールすると、統計情報は30分ごとに取得されます（同じCrocoTimeと比較して-非現実的に長い）。 インターフェイスは便利で直感的です。 部門ごとに従業員が並べ替えられており、ユーザーアクティビティをグラフと表の両方で確認できます。 「グラフ表示」ボタンのみが自発的にブロックしますが、その理由は明らかではありません。 遅れと出発を設定できます。 「後の出発による遅延を補うために」クールなダウがあります。



スタハノベッツ



製品には、ホームStakhanovets、クラウド、および単にStakhanovetsの3つのタイプがあります。 サーバーStakhanovetsはオプションです。 通常の実行可能ファイルとしてインストールされます;機能するには、MS SQL Server 2008と同時に（！）MySQLが必要です。 サーバーには、従業員の到着/退出時刻、サイトのリスト、プログラムの使用に関する情報が含まれています。 有益な時間と非生産的な時間に分けられません。 写真、スクリーンショット、通信、テキスト、グラフィック、クリップボード、ウェブカメラからのデータ、そしてマイクからも聞くことができます。 オフィスではなく強制収容所がうまくいきます。



トリックがあります：プログラムは鉄の設定を収集します。 誰かが何かに一致した場合、システムはそれを表示します。 インターフェイスは便利ではありませんが、すぐに機能しました。 問題-データをオンラインで表示できないため、レポートをダウンロードする必要があります。



デブリーフィング


Yaware-アマチュア向けのソフトウェア。 混乱して同じスタンドアロンソリューションをインストールする準備ができている人は、大歓迎です。 2回目にインストールしました。スクリーンショットを撮るので、従業員が「説明」を書くことができます。これは、勤務時間中にコンピューターを使わなかった理由です。 私にとって、これは少し愚かです。実際にお茶を飲んだとしても、私はいつでも会議に出席したと書くことができます。 適切な機能から：遅刻と欠勤を考慮して、時間を有用なものとそうでないものに自動的に分割します。 私は50台のPCでの作業の速度が気に入らなかったので、200台になります-想像するのは怖いです。



スクリーンシステム








CrocoTimeは適切に作成され、すぐにインストールされ、速度は低下しません（後で50台のエージェントにインストールされ、その後200台すべてのコンピューターにレポートの速度は変更されません）。 自動化の要件（GPOによるインストール、Active Directoryを介した従業員のインポート、従業員および部門の生産的および非生産的時間の自動計算）を満たします。 スクリーンショットを撮らず、通信を傍受しません。 軟膏で飛ぶ-プロキシサーバーの問題、長い時間制限設定。
