適切なOSにはセマフォがあります。 FreeRTOSには、「バイナリ」と「カウント可能」の2つのタイプのセマフォがあります。 すべての違いは、カウンタが1の場合、バイナリはカウントの特殊なケースであるということです。さらに、バイナリは高速システムで使用するのは非常に危険です。



通常の変数に対するセマフォの利点は何ですか？



まず、このセマフォの状態が変わるまで、タスクスケジューラにスレッドにリソースを割り当てないように依頼できます。 つまり、ジュースの持ち主は「ジュースが必要ですか？」と売り手を脅かすことはなくなりますが、誰かがジュースを必要とするまで奥の部屋のどこかで眠ります。



次に、タスクスケジューラは必要なすべてのチェックを実行して、セマフォとその値の整合性を維持します。 つまり、異なる速度でジュースを運ぶ1人の売り手に複数のサポナーを立ち上げることが可能であり、互いに絡み合い、必要以上のジュースを持ち込むことはありません。

「グローバル」コードのどこかにセマフォを作成します。 



 xSemaphoreHandle BigMac;

そして、無限ループのコードを次のコードに変更します 



 if(HAL_GPIO_ReadPin(GPIOA,GPIO_PIN_0)==GPIO_PIN_SET) { xSemaphoreGive(BigMac); } osDelay(200);




つまり、ボタン（およびボード上のPA0を対象とする）が押された場合、200ミリ秒ごとに1つのセマフォを発行し、bigmaxを要求します。



そして、各コードの点滅LEDに追加 



 xSemaphoreTake( BigMac, portMAX_DELAY); HAL_GPIO_WritePin(GPIOE,GPIO_PIN_15,GPIO_PIN_RESET); ...




つまり、BigMacセマフォが理解できるまで待機しています（portMAX_DELAY）。 任意の数のティックを設定するか、portTICK_PERIOD_MSマクロを使用して待機するミリ秒数を設定できます。



注意！ コードでは、可読性を高めるためのチェックを導入していません。
