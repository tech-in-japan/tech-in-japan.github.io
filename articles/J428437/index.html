アップルウォレット それは何で、カードをそれに統合する方法

ウォレットはCISで最も人気のあるサービスではないことが一般に受け入れられています。 しかし、すでに2番目のプロジェクトで連続して、顧客はタスクを「ウォレットと統合」に設定します。 したがって、私はこの記事を書いてサービス全体について話し、製品をサービスに統合する方法を示すことにしました。







ウォレットとは何ですか？ 携帯電話にさまざまな種類のカード（チケット、割引カードなど）を保持できるため、製品のユーザーにとって使いやすくなります。 さらに、プッシュ通知を介してマップに関する情報を更新することも可能ですが、これは別の記事のトピックです。 ただし、電話に統合できるカード/チケット/サブスクリプションがある場合は、解決策があります！ 方法-以下をお読みください。







原則として、サーバーはマップの作成を担当します。 アプリケーションは.pkpassファイルの形式でカードを受け取り、ユーザーはアプリケーションを介してカードをウォレットに追加できます。







地図構造



開発者の観点から見たマップとは何ですか？ マップは、拡張子が.pkpassのアーカイブです。 カードの表示と操作に必要なすべてのデータが含まれています。 アーカイブの内容は以下の表にあります。

次のタイプのカードが利用可能です。







  • 搭乗券：飛行機または電車で。 通常、クーポンは1回の旅行で有効です。
  • クーポン：クーポンおよび特別オファー用。
  • イベントチケット： 1つのイベントとシーズン全体の両方で機能します。
  • 割引カード：ポイントカード、割引カード、ギフトカード。
  • 一般表示カード：上記のいずれにも該当しない場合：たとえば、地下鉄またはジムへのパスで旅行するためのカード。


異なるカードの外観を模式的に検討してください。 写真は、上の表に示すように名前を付けるのが最適です。

マップを作成します。 パート2



そのため、写真の準備が整い、pass.jsonが形成され、すべてをまとめる必要があります。 これを行うには、manifest.json（表1を参照）に入力します。ここで、すべての画像とpass.jsonを含める必要があります。 次のようになります。 







 . . . . . . "pass.json" = 303c753abc39aa732ec74643d6db28348fe8a823; "strip.png" = 736d01f84cb73d06e8a9932e43076d68f19461ff; "strip@2x.png" = 468fa7bc93e6b55342b56fda09bdce7c829d7d46; . . . . . .


これ以降、何も変更する必要はありません。SHAが正しくないため、変更の場合はSHAを再生成する必要があります。







次に、開発者のオフィスでパスタイプIDを作成し、証明書を作成する必要があります。 プロビジョニングプロファイルなどを以前に作成した場合、手順は多かれ少なかれ慣れている必要があります。









次に、キーチェーンに移動し、そこからApple Worldwide Developer Relation Certificate（WWDR）を.pemとしてエクスポートします。













そこから、作成されたパスタイプIDを.p12としてエクスポートします。 この時点で、ハウスキーパーは証明書のパスワードの入力を求めます。 この場合、パスワードはオプションです。

それ以降のすべてのアクションは、manifest.json、pass.json、およびpicturesがすでに存在する1つのフォルダーで実行する必要があることに注意してください。







次に、アーカイブに署名する署名を生成する必要があります。 最初に、Pass Type IDとそれにキーを.pemとしてエクスポートします。 







 openssl pkcs12 -in certificate.p12 -clcerts -nokeys -out passcertificate.pem -passin pass: your_password


そして 







 openssl pkcs12 -in certificates.p12 -nocerts -out passkey.pem -passin pass: -passout pass:new_password


これで、署名を生成する準備が整いました。 コマンドにしてみましょう： 







 openssl smime -binary -sign -certfile WWDR.pem -signer passcertificate.pem -inkey passkey.pem -in manifest.json -out signature -outform DER -passin pass:___


したがって、すべての準備が整っています。アーカイブを収集するだけで、次のコマンドを使用します。 







 zip -r nameOfPass.pkpass manifest.json pass.json signature logo.png logo@2x.png logo@3x.png icon.png icon@2x.png icon@3x.png


カードのデータアーカイブ（.pkpass）を含めるすべてのファイルをここにリストする必要があることに注意してください。

その結果、コンピューターで開くことができる.pkpassファイルを取得します。 カードのプレビューが表示されますが、その外観は電話機の外観と異なる場合があります。

これはすべて少し簡単に行うことができます。 Appleは、すべてのSHA計算（ manifest.json

次に、コマンドを実行します。 







 ./signpass -p wallet


ウォレットは、すべてのリソースが配置されているフォルダーの名前です。 出力はwallet.pkpassです。 その内容は、wallet.pkpassを解凍することで表示できます。 







 unzip wallet.pkpass


pkpassの作成がバックエンドに送信される可能性があります。その場合、.p12形式のPass Type IDの証明書とそのパスワードをWWDR開発者に転送する必要があります。







アプリケーション統合



アプリケーションがウォレットにカードを追加できるようにするには、アプリIDでこの機能を有効にし、プロジェクトの機能でこの機能を有効にする必要があります。











これは、ウォレットでの完全な正しい作業に必要です。 そうしないと、ウォレットからカードを読み取ることができなくなり、たとえば、当社のカードが追加されているかどうかを理解できなくなります。 pass.jsonのチームIDはチームIDと一致する必要があります。そうしないと、資格に手動で追加する必要があるため、状況を修正できますが、これは確認しませんでした。













カードを追加する



マップの追加は非常に簡単です。 







 guard let passPath = Bundle.main.path(forResource: "wallet", ofType: "pkpass") else { return } let error: ErrorPointer = ErrorPointer(nilLiteral: ()) guard let passData = NSData(contentsOfFile: passPath) else { return } let pass = PKPass(data: passData as Data, error: error) let passLibrary = PKPassLibrary() passLibrary.addPasses([pass]) { (status) in print(passLibrary.containsPass(pass)) }


ただし、繰り返しますが、多くの場合、.pkpassファイルをサーバーからダウンロードする必要があります。

PassKitは非常に読みやすいエラーを生成するので、何が間違っていたかを簡単に理解できます。







追加されたカードに関する情報の取得



ウォレットで使用可能な、アプリケーションに関連するカードに関する情報を取得するには、PKPassLibraryオブジェクトを有効にする必要があります。 







 let passLibrary = PKPassLibrary() let passes = passLibrary.passes()


したがって、マップを追加するかどうかを理解し、インターフェイスを更新できます。 さらに、PKPassLibraryを使用して、マップを更新および削除できます。 マップはWebサービスを介して更新することもできますが、この記事ではそのようなオプションは考慮しません。







一意性チェック



サービスでは、原則として、カードはアカウントに関連付けられているため、アプリケーションでは、カードが現在のユーザーに属しているかどうかを何らかの方法で判断する必要があります。 これをserialNumber



行うことをお勧めします。 たとえば、ユーザーのserialNumber



番号IDまたはカード番号を設定します。







テスト中



Appleはさまざまなタイプのpkpassの例を提供しています。それらに焦点を当てることができます。

アップルウォレットのサンプル

マップの外観を確認するには、pkpassをプロジェクトに追加します（「マップの追加」を参照）。 追加/削除のプロセスは既に上で説明されています。ウォレットのカードが1つの開発者アカウントで作成され、開発自体が別のアカウントから実行された場合（アウトソーシング会社に関連）、アプリケーションは既に追加されたカードを表示しないことを思い出してください。 問題なくカードを追加できます。

QRコードスキャナーを使用して、バーコードの情報が正しくエンコードされているかどうかを確認できます。 そして、このスキャナーで作業の正確さを確認することは間違いなく必要です。
