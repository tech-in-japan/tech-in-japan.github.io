私たちはしばしばインターネット上でエラーとタイプミスを見ます。 原則として、文盲の（または不注意な）作家を軽heartして尊厳を持って通り過ぎるか、逆に教育活動を開始する熱意を持って（常に、残念ながら敬意を払うわけではありません）通行するのが慣習です。 最良の場合、Orphusがサイトにインストールされている場合、Ctrl + Enterを押します。



これは私がやったことでしたが、ある日、私は「一般的に」またはそのような何かを書いたことに気づいて怖がりました。 フォーラムを読むことの破壊的な効果は、そして実際、読解力に対するHabr自身も明らかです。 そしてその間-今、私たちの貧しい目の運命を緩和することができる多くのものがあります！



さらなる議論の簡潔さをおaびします。 いつものように、時間の問題。 コメントで質問してください。

いくつかの単純なイデオロギーのステップを踏むだけです。



ステップ1. 自動的に修正できるエラーのクラスがあります。 たとえば、GENERALおよびGENERALをGENERALLYに置き換えることは難しくありません。また、ロシア語には-TOTALで終わる動詞がないことに注意してください。 自動的に、HOLDとHOLDの違いを探すことはお勧めしません。

このアプローチは、チャットルームやフォーラムでの汚い言葉に対抗するために、そして非常に破壊的に使用されます。 それ以外の場合、原則として、テキストエディターは、テキストの作成者（ ライター ）が入力したときに、なじみのない単語のみを赤で強調します。 この推論により、スムーズに次のステップに進みます。



ステップ2. 読者には、読む前にテキストを修正する権利があります。 はい、はい、作家のリテラシーや編集者/モデレーターの警戒に頼る必要はありません。 独自のインターネットをよりクリーンで明るくすることができます。



そこで、chas-correctのアイデアが生まれました。これは、多くのエラーを自動的に修正するブラウザーの拡張機能です。 githubのリポジトリ 。 Firefox、Firefox Mobile、Chromiumライクがサポートされています。 （Google Chromeの場合-ディレクトリ内のymatuhinの 非常に近いフォーク ）。

更新2015年10月4日クロムライク用のユーザースクリプトを作成しました。 インストール手順 。



次に、質問と回答の形式で詳細を示します。



Q：なぜchas-correctが必要なのですか？ 最新のブラウザはエラーを赤で強調できます。

A：もちろんです。 入力時に発生する「彼らの」間違いのみがti病に強調されます。 Chas-correctは、それ自体で、読み込まれたページのテキストに元々含まれていた他の人のエラーを読者の目から隠します。



Q：これについて本当に考えたことがありませんか？！

A：私自身も驚いています。 同様のアイデアがありました： 1と2 。 しかし、何らかの理由で彼らは団結しませんでした...



Q：chas-correctはすべてのエラーを修正しますか？

A：もちろん違います！ それだけが、最初に頻繁に発生し、次に、簡単に形式化され、自動的に修正されます。



Q：Google Chrome / Opera /の公式拡張ディレクトリにchas-correctがありますか？

A：いいえ。 私は彼らの検証に煩わされる時間がありません。どういうわけか、いつでも禁止されたり制裁を受けたりすることができるアカウントにもっと高価なドルを投稿することを残念に思います。 さらに、一元化されたインストールを強制し、拡張機能をチェックするというポリシーは疑わしいと思います。 ただし、これは別のホリバーのトピックであり、コメントで繁殖できます。 誰かがそれをcrxにパックしてディレクトリに入れても、もちろん反対しません。



Q：chas-correctは更新されますか？

A：残念ながら、まだです。 かなりの数のエラーが既に彼によって分析および修正されており、エラーディクショナリをさらに増やすには、アルゴリズムをある程度最適化する必要があります。 辞書だけでなくエンジン自体も更新し、公式の拡張リポジトリをバイパスすることを提供する場合、インストーラーはパラノイアを正当化するでしょう（実際、Chromeストアからのインストールの場合。場合によっては）。 将来的には、すべての種類のストーリーをバイパスして、辞書のみの更新を開発する予定です。



Q：拡張機能が機能することを確認する方法は？

A：ここにエラーが表示されない場合-おそらく動作します。 または、これが適切なhabraの例です。



Q：chas-correctが間違った権利を修正したときに、誤報は可能ですか？

A：実質的に除外されています。 理論的には、これは新語、キャラクター名、地名、およびその他の非語彙に直面した場合に可能です。 ただし、拡張機能は間違ったものと間違ったものを修正する場合があります。そのため、「何か」がある場合でも、「何か」は常に「何か」に変わります。



Q：どのように機能しますか？

A：常連です。 ここでは、あごひげを怒って握りしめ、空に手を上げる必要はありません。 ブラウザでのリアルタイムテキストの本格的な分析は、まだほとんど不可能です。 少なくともコンピューターの現在の計算能力。 正規表現の場合でも、大幅な最適化の努力が必要でした。

UPD：専門用語、用語、固有名詞などが豊富であるため、正規表現は辞書よりもエラーを説明する優れたモデルであると考えています。



Q：検証用のテキストを同じYandex-Spellerにリアルタイムで送信してみませんか？

A：第一に、これはかなりの数の誤検知につながります。そのような実験は私の前で行われています、 繰り返します。 第二に、これはすべてのユーザー通信がサードパーティのサービスに送信されることを意味します...それを控えめに言っても、問題のある時代にはあまり良くありません。



Q：chas-correctは、VKontakteインスタントメッセージなどの動的にロードされたコンテンツで動作しますか？

A：タイマーをレイプしないように機能しますが、常に即時ではありません。



Q：エラーデータベースとパフォーマンスインジケータはどこから来たのですか？

A：Habrにはこのような便利なページ構造があります...直接番号付け。 サイクル、wget、パーサーへ、単語へのカット、カウント、そして最も頻繁-Yandex-Spellerを介して。



Q：chas-correctは入力したテキストを修正しますか？

A：拡張機能の作業は、キーボードのキーストローク後約3秒間ブロックされます。その後、入力されたテキストが修正され、カーソルが最初に移動します。 したがって、chas-correctを使用して、読んだものだけでなく、書いたものも修正するようにしたい場合は、メッセージの終わりから送信まで3秒間待機することをルールにしてください。 これは一般的に非常に便利です...



Q：なぜ著者はこれをすべて行うのですか？

A：正直に言うと、私を刺激してくれた素晴らしい茶色の目の女の子がいます。 はい、ほとんどが必要です。 誰かが重宝してくれたらとてもうれしいです。 さらに、これはポートフォリオ内の行（ある種）、Habrへの招待、Voronezh State Universityからの励ましの機会です（この機会を利用して、多くの奨学金についてInformsvyazに感謝します）。 そして、chas-correctは知的な女の子を快適にする良い方法です（読者に注意してください：ちなみにchas-correct + AdBlock-そして彼をフィルハーモニックにドラッグできます）。



Q：これは非識字の問題を解決するものではなく、単に目を閉じるのに役立ちます！

A：しかし、彼らは推測しませんでした。 スペルミスを見ない人は、いわゆる「リーダーリテラシー」を蓄積し、より少ない確率で文章やタイピングの間違いを犯します。 一般的に、あなたは母国語への愛情を植え付ける必要があります。



Q：コードはひどいです！ それをとかし、最適化する必要があります！

A：GPLに基づくコード、パッチは受け入れられていますが、フォークもキャンセルされていません。 ちなみに、コメントアウトされたコードは私の過失によるものではなく、失敗した最適化を10回再発明しないためです。



Q：設定、グラフィカルインターフェースなどはありますか？

A：いいえ。 まだ多くはまだありません...良心だけでは、このアイデアを未公開のままにしておくことはできません。 （スタートアップを書いたVasyaとPetyaについての教科書の話があるはずです。）しかし、Ctrl + Shift + Aを押すと、句読点の周りにスペースが自動的に配置されます。 そして大文字。 少なくともそれは起ころうとします。



Q：プロジェクトの見通しは何ですか、著者は次に何をする予定ですか？

A：時間の経過とともにアルゴリズムを最適化します。 辞書を補充します。 グラフィカルインターフェイス、チェックボックスを添付：修正e / e、アンチマットなど



Q：私はプログラマーです。正規表現を知っており、助けたいです。

A：たとえば、dictionary.jsにはTODOマークがあります。 本当にやるべきことがあります。



Q：私はプログラマーではなく、正規表現を知りませんが、助けたいです。

A：エラーを収集します。 nickkolok@mail.ruに送ってください-ページのアドレスを必ず入力して、単語のつづりを間違えてください。



Q：私はウェブマスターであり、学生がエラーを頻繁に書くサイトを管理しています。 サイトでchas-correctの実績を直接使用することはできますか？

A：本当にしたいのなら、できます。 キャッシュも機能するはずですが、マニフェストのようにスクリプトを接続するだけです。 もちろん、このようなフィルターを介して送信し、既存のデータベースを1回処理する方がはるかに正確です。 Node.jsでは、互換性があるようです。 一般的には、nickkolok @ mail.ruに連絡すればわかります。



もちろん、拡張機能の作業を継続する必要があります。 3つ目のイデオロギーのステップがあります-しかし、まだ行われていません。これは完全に異なる話です。
