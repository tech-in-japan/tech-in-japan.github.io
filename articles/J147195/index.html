Rambler Maps API



私たちはデザイン、スタイル、そしてカードを扱うときの喜びの感覚を構成するすべてのささいなことに多くの注意を払います。 したがって、あなたのプロジェクトで私たちのカードをどのように使用するかという質問がますます繰り返されるようになったとき、私たちはRambler-Card APIを開くことにしました。



アプローチとして、ミニマリストのパラダイム-最大のシンプルさを選択しました。 そのため、たとえば、最初にキーシステムを放棄し、メソッドの数を最小限に抑えました。 APIの操作は非常に簡単ですが、それでも一般ユーザー専用に設計されているという意味ではありません。 経験豊富な開発者も非常に快適で、彼と一緒に働くことに興味があるでしょう。



これで、APIで次のことができるようになりました。

•プラグなどの追加のレイヤーを接続する

•グラフィックオブジェクトを描く（マップ上に鉛筆のように簡単に描くことができます）

•ユーザー情報を使用してタグとバルーンを表示し、アニメーション化する

•住所、興味のある場所を検索する

•カスタムアプリケーションの統合が簡単



API構造は、その場で個別のユーザーアセンブリにアセンブルできるモジュールで構成されます。 このため、サイトには特別なコンストラクターが用意されており、ユーザーは本当に必要なものだけを接続できます。



ユーザーアプリケーション用の特別なインターフェイス（ ウィジェット）を開発し、開発者のタスクを大幅に簡素化しました。 独自のウィジェットを作成して共有したい場合は、それらを送信してください！



モバイルプラットフォームとの互換性にかなりの注意を払いました。 私たちのグラフィックスサブシステムは、たとえばCanvasとSVGの両方をサポートしています。 ジェスチャーもサポートされています。



そのため、ドキュメントとサンプルを含む別のAPIサイトを立ち上げました。 Facebook 、 Vkontakte 、 Twitterでグループを見て、評価し、参加し、議論し、質問し、機能的な部分であなたの願いを表現してください。



そして、これはほんの始まりに過ぎません。
