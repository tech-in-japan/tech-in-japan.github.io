ポートの構成と機能についてもう少し。







スイッチ間接続（ISC）は、グループのスイッチ間のデータ転送を担当します。これは、別個のポートまたは集約された回線のグループのいずれかです。

ピアキープアライブは、パートナーの寿命を監視する責任があり、サービス情報とHelloパケットのみが送信され、トラフィックは送信されません。

分散トランクインターフェイス（DTI）は、仮想スイッチの共有論理回線の一部です。



この技術を実装できる機器のセットは限られているため、その構成については説明しません。十分な詳細なマニュアルを備えた十分なリソースがあります。



DTIポートに接続されたネットワーク上のデバイスの場合、スイッチのペアが1つとして表示されます。この側面により、集約可能な機器を接続できます。

簡単なスイッチを使用して、アップリンクを集約する方法をほぼすべて知っており、カーネルに向けられたポートをいくつか組み合わせるだけです。 ルーターを使用すると、すべてが少し複雑になり、すべてのルーターがLAG（Link Aggregatin Group）に対応できるわけではありません。 特に、Cisco機器の場合、3925ルーターはEtherCnannelでき、1841はできません（追加のスイッチモジュールなし）。 3925では、EtherCnannelの2つのポートとすべてを組み合わせます。 しかし、1841年以来、問題を解決するには、実際には賢くなければならないため、BeckUpインターフェース+ ip unnamberedテクノロジーの束が使用されました。 BackUp Interface + bridg groupをまとめて使用することは可能かと思われますが、実際にはテストされていません。 おそらく、habbschestvoは他のオプションを提供するでしょう。
