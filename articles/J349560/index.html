タスク番号1。 カレンダー

ポケットカレンダーを生成する単一のSQLクエリを記述します。 タスクのパラメーターには、カレンダーの年と、月のマトリックスを形成するための行と列の数が示されます。 パラメータは、次のクエリによって設定されます。 







with param(year, c, r) (…)


ここで、それぞれ、







  • 年-暦年
  • cは、カレンダーマトリックスの列数です。
  • rは、行列の行数です。


月は、カレンダーマトリックスのセルに左から右、上から下の順に配置されます。 各月の数字は、曜日、最初の列の最初の曜日などにあります。 週の初めは、リクエストが起動された時点のデータベースローカリゼーション設定に対応している必要があります。 月の名前もローカライズ設定から取得され、数字の上の中央に表示されます。 隣接する月の数が「互いにくっつかない」ように、月の間にはギャップを残す必要があります。 最初の行は年の中央に行く必要があります。 空行があってはなりません。







たとえば、次のパラメーターを使用します。 







 with param(year, c, r) as (select 2016, 3, 4 from dual)

タスク番号2。 輸血

3リットルと5リットルの容量を持つ2つの容器と、水が入った蛇口があります。 ある容器から別の容器に水を移し（そのうちの1つがいっぱいになるか、空になるまで）、水を注ぎ（完全にのみ）、蛇口から上部まで容器を満たします。 正確に4リットルを測定できるすべての輸血オプションを見つける必要があります。 ループを回避するために、血管の充満状態が繰り返されるオプションは破棄されます。







SQLクエリは、可能なすべての輸血オプションを次の形式で表示する必要があります。各オプションについて、適切なステップで各容器の水量を示す数値のペアの形で輸血チェーンを出力します。 数字はマイナスで区切られ、ペアはコンマとスペースで区切られます。







一般的な方法で問題を解決する必要があります。パラメータは、最初のv1および2番目のv2血管の容量と、輸血の結果として取得する必要があるresの結果を指定します。 条件の例： 







 with param(v1, v2, res) as (select 3, 5, 4 from dual)

タスク番号3。 方法について

頂点とエッジで定義された無向グラフがあります。 エッジの長さと開始ピークと終了ピークの名前を指定します。 1つのSQLクエリでは、最初の頂点から最後の頂点までの最短距離のパスを見つける必要があります。 結果は、2つの値の形式で表示される必要があります。







  1. パス行では、頂点の名前はマイナスでリストされます。
  2. パスの長さ（エッジの長さの合計）。


昇順のパス長でソートされた3つの最短オプションを導出する必要があります。 問題の状態では、頂点のペアとそれらの間の距離が設定されます（グラフは無指向性であり、距離は最初の頂点から2番目へ、およびその逆も2番目から1番目への移動にも有効です）、最短パスを構築するための初期および最終頂点。







グラフ内のエッジの数は合理的に制限されており、50エッジ以下です。

タスク番号4。 電卓

算術式は文字列として指定されます。 式の値を計算するには、1つのSQLクエリが必要です。 この式には、加算、減算、乗算、除算、べき乗、単項マイナス、括弧の演算の符号が含まれています。 単項マイナスは、式の開始時または開き括弧の後にのみ発生します。 数値には小数部分が含まれる場合があり、形式はデータベースの現在の言語設定のNUMBERタイプに完全に対応します。 明確にするために、式にはスペース文字を含めることができます。 式の長さは合理的に制限されています。50オペランド以下、括弧の深さは20以下です。式は常に正しいです。







要求は、NUMBER形式で単一の値を返す必要があります。







サンプルテストデータ： 







 with param(expr) as ( select '(-1 + 5^(1/2) ) / 2' from dual )

タスク番号5。 ラビリンス

ラビリンスは番号付きの行の形式で設定されます。 壁には「＃」、入口点「s」、出口「e」が付いています。 国境を越えることは不可能であり、行番号は順番に進み、すべての行の長さは同じです。 1つのSQLクエリでは、文字 "*"を使用して、入力から出力への最短パス（複数のパスがある場合はそのうちの1つ）を描画する必要があります。







迷路のサイズは制限されているため、迷路を通過する（内部ループなし）オプションの数は10,000を超えません。
