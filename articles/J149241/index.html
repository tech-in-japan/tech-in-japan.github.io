1Cサーバー8.2 + MsSql 2008 +バックアップ

似たような記事はたくさんありますが、私は自分で最初にこれを書き、考えられる問題を説明するメモに焦点を当てました。 この記事が他の人にも役立つことを願っています。

1. 1Cプラットフォームをインストールする

2. MS SQL Server 2008をインストールします。インストール時に、データベースユーザーを設定します。 （SAはどちらですか）。



インストール後、1Cエンタープライズサーバーの管理パネルを開き、空であることを確認します。

サーバーを作成する必要があります。コンソールルートを開く-> Central 1C：Enterprise 8.2サーバー。 マウスの右ボタンでそれをクリックし、新しいアイテムを選択します。 ドロップダウンメニューで、Centralサーバー1C Enterprise 8.2を選択します。 4つのフィールドを持つウィンドウを開く前に：

プロトコル -データの送信に使用されるプロトコル

名前 -サーバーが配置されているネットワーク上のコンピューターの名前

IPポート-サーバーがアクセス可能なポート

説明-説明。 必ずしもではありません。



注：

1Cプラットフォームがコンピューターにインストールされた後、コンピューターの名前が変更された場合、1Cプラットフォームは非常にスマートなプラットフォームであり、インストール中にコンピューター名を特定のファイルに書き込むが、コンピューター名が変更されたときにプラットフォームが書き換えます。 これらのファイルは、RAGENT 1Cサービスが機能するために必要です（Windowsサーバー管理パネルから実行中のサービスにあります）。 これらはすべて、これらのファイルの名前を変更するには、RAGENTサービスを停止する必要があることを示唆しています。 ファイル自体は次の場所にあります。

C：\ Program Files（x86）\ 1cv82 \ srvinfo \ srvribrg

C：\ Program Files（x86）\ 1cv82 \ srvinfo \ reg_1541 \ 1CV8Reg

メモ帳でこれらのファイルを開き、ペンで現在のマシンの過去の名前を編集します。 RAGENTを保存して実行します。



設定に戻る：

フィールドが表示されたウィンドウがいっぱいになったら、[OK]をクリックします。すべてが正しく完了したら、そのマシンの名前のサーバーが表示されます。



など。 サーバーが実行されているので、MYSQLサーバー上にデータベースを作成し、1Cの北に接続する必要があります。 いくつかの方法があります-ここでは最も簡単な方法を説明します：

エンタープライズのサーバー1Cで、サーバー名の横にある[+]をクリックして、新しく作成したサーバーを開き、[情報データベース]項目を右クリックして、[新規作成]-> [情報ベース]を選択します

目の前にウィンドウが開き、次のフィールドがあります。



この名前は、 1Cサーバー上のデータベースの名前です （混乱しないように、多くの場合、多くの場合、データベースフィールドと同じように記述します）

説明-説明

安全な接続 -デフォルトでは無効になっています。 有効にできますが、サーバーの負荷が増加します

データベースサーバー—サーバーが同じサーバー上にある場合は、この方法でかっこ内に（ローカル）を指定します。このサーバー上にない場合は、サーバーIPを指定します

DBMSタイプ -MS SQLタイプを選択

データベース -MS SQLサーバー上のデータベースの名前。 ベースがない場合、チェックボックスの1つでボックスをチェックすると作成されます

データベースサーバーユーザー—インストール中に作成したユーザーを示すか、MS SQLで別のユーザーを作成して権限を割り当て、ここに書き込みます。

データベースサーバーのユーザーパスワード-パスワード

1Cエンタープライズサーバーによるライセンスを許可—はいを選択します

国 - 国を選択します

日付オフセット -0に設定

「不足の場合にベースを作成」チェックボックスは、そうでない場合にベースを作成するためのまさにチェックボックスです

チェックボックス「定期メンテナンスブロックを設定する」-チェックしない



[OK]をクリックすると、サーバーが構成され、[情報ベース]タブに指定した名前の下に情報ベースが表示されます。



次に、バックアップを構成します。 私は何が起こるのか、情報など、海については話しません。 たとえば、この素晴らしい記事をご覧ください 。



バックアップを構成するには、 Microsoft SQL MANAGEMENT STUDIOを開く必要があります。

ログインを入力し、サーバーに接続します。

私たちの前にあるのは管理コンソールです。 オブジェクトエクスプローラーで、 [ 管理 ]タブを開き、その中のメンテナンスプランを確認します。 ここで、必要なBackUPを作成します。 通常どおり、[ メンテナンスプラン ]- > [新しいメンテナンスプラン ]を右クリックします。 メインウィンドウにサブプランタブが表示され、 オブジェクトエクスプローラーの下に、 メンテナンスプランタスクが埋め込まれた別のツール ボックスウィンドウが表示されます。 その中で、 [ データベースタスクのバックアップ ]を2回クリックして選択します。 メインウィンドウに転送されます。 2回クリックすると、ウィンドウが再び表示され、実行するバックアップ、バックアップベース、保存先を選択できるフィールドが表示されます。 設定の最後に、[OK]をクリックします。



注：

バックアップをネットワークフォルダーに保存するとき（ディレクトリ選択ウィンドウにローカルリソースのみが表示されるため、ペンでパスを書き込む必要があります）、アクセス権を監視し、同時に、MySqlサーバーで認証を監視します（認証がアカウントによって設定されていない場合） Windows、ただし内部DBMSユーザーの場合、およびADサーバーが発生している場合、実行を試みるとエラーが発生します。これは、ロケールを除き、内部DBMSおよびADユーザーの代わりにこれを行うためです。 コンピュータ情報。



バックアップのパス、ベース、およびタイプを構成したら、スケジュールを構成する必要があります。 これを行うには、作成したタスクの上のメインウィンドウにSubPlanサインがあります。 プレートの最後（右側）にカレンダーアイコンがあります。 それをクリックすると、スケジュール設定に移動します。 曜日のチェックボックスをマークし、時間を設定することにより、スケジュールを設定します。 SubPlanというフィールドを2回クリックすると、Task-aの名前を変更できます。 すべてを設定したら、[ファイル]-> [ すべて保存]に移動します 。 メンテナンスプランに保存すると、バックアップに指定した名前のタスクが表示されます。



設定の最後に、必ず作品を確認する必要があります。 これを行うには、作成したタスクを右クリックして、Exicuteを実行します。



注：

Exicuteが失敗した場合は、Studioから表示されるエラーを読み、最初にSQL Serverエージェントが実行されているかどうかを確認します。 タスクを実行するのは彼であり、Exicute機能はタスクを彼に向けます。 実行されていない場合、実行は失敗します。 オブジェクトエクスプローラーのStudioでdiエージェントが動作しているかどうかを確認するには、[ SQL Serverエージェント ]タブに移動します。 箇条書きアイコンに十字が付いた赤い円がある場合、エージェントは停止しています。 起動するには、右クリックしてコンテキストメニューの[開始]オプションを選択します。
