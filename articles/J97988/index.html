コンパックアルマダ1530



このラップトップは1997-1998年にリリースされました。 正確に今を確立することはすでに困難です。 彼は2002年に私のところに来て、数年間作業機械として使用されました。 したがって、構成：



CPU：ペンティアム133 Mhz mmx

RAM：16Mb（基本）。 2003年に48 mbに拡大

HDD：1.4 Gbシーゲイトマラソン（故人）

FDD：はい、ただしテストされていません

CDROM：スタブではなく、インストールされていません

ポート：1xCOM、1xLPT、1xVGA、1xPS / 2、2xPCMCIA 32ビット+ポートレプリケーターコネクタ

ディスプレイ：800x600 256色

重量：3.2 kg +ハードドライブ150 g





大学の最初のコースでプログラミングとFIDOに使用しました。 CDROMがないため、ソフトウェアはいくつかの方法でインストールされました。



1）COM-> COMケーブル（ヌルモデム）およびNorton Commanderを使用。 これにより、このような接続を介したファイルの交換が可能になりました。 モデム接続レベルでの速度。

2）LPT-> LPTケーブルの速度は最大200〜300 kbps。 最初のオプションに似ていますが、レースを見つけることはもう少し困難でした。







Windows 98seが最初にインストールされました（com-> comケーブルを使用）。 16人の工作員の作業は非常に快適でした。 48まで拡大した後、それは一般的に素晴らしいものになりました。 2003年に、Debian Woodyがインストールされました。 CDROMがまだないので、インストールは4枚のディスケットを作成し、さらにネットワークを経由しました（まともな3COMネットワークカードを借りる必要がありました）。 長いシャーマニズムがこの鉄の地殻に落ちた後でも、XがDebianですべてうまくいったわけではありません。 そのため、ソフトウェアはコンソールのみであり、すべてが非常に迅速に機能しました。 パスカル用のDosbox、ニュースゲート経由でfidoを読むためのslrn、メール用のmutt、ブラウザとしてのリンク。



しかし、これはすべて物語です。 ネジが死んでいることを発見して、私は長い間、このコンピュータに何かを転送する方法を考えていました。 私は長年FDDを持っていません。CDROMも持っていません。PXEを備えたPCMCIAネットワークカードもなくなりました。







OSの修復とインストール。



多くの人がこの方法を古い鉄の再生に役立つと思うでしょう。

1）VirtualBoxで仮想マシンを作成する

2）OSをインストールします

3）VDI画像をRAW画像に変換する

VBoxManage -q internalcommands converttoraw win98.vdi win98.eng.se.raw

4）次に、このイメージをハードドライブにコピーします

dd if = win98.eng.se.raw of = / dev / sdX bs = 512



ハードドライブとして、古い1G CFカードとIDE-> CFアダプターを使用しました







ビンに古いWiFiアダプターCisco Airtonet 350（802.11b）が見つかりました。 驚いたことに、彼はなんとか私のルーターと友達になりました。
