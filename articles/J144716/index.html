TP-LINKルーターとモバイルデバイスを友だちにした方法

AndroidまたはSymbianを搭載したデバイスを介して接続するときに、WiFiで問題が発生しましたか？ 会った。 確かに、私にとってこれは新しいものではありません-Windows Mobile 5が立っていたPDAで災害が発生していました-WM 6.5で最新のファームウェアにフラッシュする直前にWPA / WPA2で動作することを完全に拒否しましたが、どのアクセスポイントでも動作しませんでした。 ただし、Symbian 9.4を搭載したNokia 5800スマートフォンとAndroid 2.3.1を搭載したAirPad X10タブレットは、WPAまたはWPA2セキュリティを備えたWiFi経由で作業する場合、まだ発売されていません。 昨日まで、TP-LINK TL-WR741Nルーターをインストールする必要がありました。 この奇跡を、L2TPを介した自動接続のルーティングモードで問題なく機能させることができました。 WiFi接続はWPA / WPA2 PSKセキュリティモードとAES暗号化でテストされました。これに対して、Lubuntu 11.10のネットブックでワイヤレス接続が作成されました。これはパイントの半分から上昇し、数秒後にブラウザがGoogleホームページを表示しました。 しかし、後で、Symbianを使用してスマートフォン経由でインターネットにアクセスしようとすると、「サーバー名が間違っています」というエラーが表示されました。 ）、および白（「ネットワークはまだ利用できません」）。



私が最初に思いついたのは運河でした。 チャンネル1を自動的にインストールし、ゾーン内に他のポイントはありませんでしたが、それを変更しました。 次に、モードをデフォルトの「11bgn mixed」からさまざまな11b / 11gバリエーションに変更しようとしました-結果はまったく同じです。 WDSとQSSは無効になりました。ここではすべてが正常です。

ルーターのデータから判断すると、デバイスからパケットを受信します。受信したパケットの数は、「到達」する新しい試行ごとに増加します。 ただし、ルーターから送信されるパケットは常に4です。

IPがリモートサーバーにも到達できなかったために、それは却下されなければならなかったのはDNSの問題であるという仮定がありました。 しかし、リモートサーバーについてはどうですか-WiFi Webインターフェイス自体にアクセスできませんでした。

ルーターのファームウェアを最新バージョンに更新しても何も変わりませんでした。

主にスマートフォンからSymbianを使用して接続をテストしている間、ネットワークセキュリティを無効にしようとしましたが、問題は同じままでした。ポイントに接続しても何も接続されません。 しかし、Androidのオープンネットワークへの接続を最終的に確認することを考えたとき、彼にとって問題は安全であることが判明しました。 その結果、自動WPA / WPA2から特にWPA2モードへの切り替え-問題は最善の方法で解決されました。

ここにそれらの時代があります！

現在、Symbianのみがありました。 彼にとってそれはすでにより困難でした-すべてのワイヤレス設定がすでに検証されているようです...そして、アイデアが私の心を越えました：DHCPをチェックしてください。

判明したように、プールのIPアドレスは100番目から発行されました。 原則として、特に犯罪的なものは何もありませんが、...アドレスプールの開始を192.168.1.2に変更した後、スマートフォンはすぐに私が提案したURLを活発に踏みつけ、最終的にリモートサーバーに接続しました！

これがルーターの魔法です。 そしてデビッド・ブレインはいません。
