RTLを使用したBadaアプリケーションのテスト





この記事は「Samsung Badaプラットフォームでのアプリケーションの開発-パート1」の続きではなく、新しくリリースされたBada SDK 1.0のベータステータスで登場したRemote Testing Labs（RTL）メカニズムを使用したリモートハードウェアでのBadaアプリケーションのテストの簡単な紹介として機能します.0。 すぐに書きます。無料です。 とにかく、さようなら。



翻訳を助ける


はじめに


RTLは、インターネット経由でSamsungデバイスにアプリケーションをインストールおよびテストできるリモートテストラボです。 開発コストを削減し、Samsungデバイスでの互換性と使いやすさを簡単にテストできます。 RTLは、リモートアプリケーションの開発とテストのために、ネットワークに接続されたモバイルデバイスに24時間365日、リアルタイムのオンラインアクセスを提供します。 RTLは、配布前にアプリケーションの機能をテストする最も簡単な方法です（Samsung Apps-およその翻訳。）。



RTLを使用できるのは誰ですか？


RTLは、Bada開発者サイト（http://developer.bada.com）のすべてのメンバーが無料で（ステータスに関係なく）利用できます。 できるだけ多くの人がサービスを利用できるようにするために、次のクレジットシステムを使用して時間を予約します。

  • 各ユーザーは毎日10クレジットを受け取ります。
  • 1つのローンには15分の作業がかかります。
  • サービスを使用するための最小時間は、一度に15分（1クレジット）です。
  • 最大労働時間-1日10時間（40クレジット）;
最大制限が小さすぎて追加のRTL時間を必要とする場合は、サムスンに連絡して何か可能性があるかどうかを確認してください。 サムスンはお客様のリクエストに応えようとします。

既知の制限
  • サウンドはサポートされていません。
  • デフォルトでは、追加のアクセサリはありません（メモリカードがない場合でも、およその翻訳です）。
  • RTLはリモートソリューションです。 したがって、サーバーまでの距離、接続速度、チャネル負荷、ファイアウォールなどのローカルネットワークポリシーなどの変数は、サービスの品質に影響を与える可能性があります。
  • デバイスが常に使用可能であるという保証はありません（切断がある場合があります）。
  • メンテナンスのために、デバイスをRTLから切断する必要がある場合があります。 メンテナンス中、電話は切断されたと見なされます。
  • このサービスはサポートを提供しません。 一般的な質問については、rtl.support [at] samsung.comにメールを送信してください。
  • 画面を横向きモードにすることはできません-追加されました。 翻訳。
  • 他の開発者を尊重し、テストの完了後にアプリケーションをデバイスから削除します。
RTLインストール


そのため、最初にここから新しいBada SDK 1.0.0をダウンロードしてインストールする必要があります 。 上記の記事で、Badaの開発に参加するには、developer.bada.comに無料で登録する必要があることを既に書いています。 彼女なしでは何も機能しません。 SDKをインストールした後、Eclipse用のRTLプラグインをインストールする必要があります。 メニューで[ヘルプ]-> [新しいソフトウェアのインストール]を選択し、[追加]ボタンをクリックします。 「RTL-Install」、場所「 http://rtl.innovator.samsungmobile.com/rtl-update 」を設定した名前。 [OK]をクリックして、RTL Updateの左側にdawを配置します。 次へをクリックします。 さらに説明する必要はありません-したがって、それは明らかです。 その結果、Eclipseを再起動すると、RTLが表示されます。 すぐに設定でユーザーを設定します-RTL-> Configuration RTL Settings。 ユーザー名（ユーザーID）は、developer.bada.comのログイン（メールID）と同じです。 はい、繰り返します。RTLを使用するには、Internet ExplorerがJavaスクリプトを実行でき、Java Web Startが必要です。



RTLを使用する


システムに入ります-RTL-> RTL Serverにログイン（またはツールバーから）。 残念ながら、パスワードは保存できず、毎回入力する必要があります。 3つのタブのあるウィンドウが表示されます-予約（スケジュール）、電話（利用可能な電話のリスト）、ヘルプ。







スケジュールから判断して、モスクワ時間の13:30には 、RTL-emを使用している人はいませんでした 。 どうやら、まだそれについて知ることができた人はほとんどいないようです...韓国と英国にあるテストに利用できるのはWaveだけです（うーん、これ以上Bada電話はありません）。 韓国の携帯電話では、すぐにうまくいきませんでした-彼らはそれが非常に大きなpingだと書きました。 英語で動作しますが、画面のリフレッシュレートは1 fpsに近いです。 ここではゲームをテストできませんが、静的なアプリケーションを試すことはできます。 他の人のアプリケーションが電話で何度か出くわしました（「既知の問題」ヘルプの最後の段落を参照）。
