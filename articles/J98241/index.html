Razor-新しいASP.NETビューエンジン



私のチームが取り組んでいたプロジェクトの1つは、ASP.NETの新しいプレゼンテーションエンジンでした。



ASP.NET MVCは、常に「プレゼンテーションエンジン」の概念をサポートしており、実際には、さまざまなテンプレート構文の選択を実装する交換可能なモジュールです。 今日、ASP.NET MVCの標準プレゼンテーションエンジンは、ASP.NET Webフォームと同様のファイル-.aspx / .ascx / .masterを使用しています。 他の一般的なASP.NET MVCプレゼンテーションエンジン-SparkおよびNHamlがあります。



私たちが取り組んでいる新しいプレゼンテーションエンジンは、テンプレートコードに重点を置いてHTMLコードを生成するために最適化されています。 このエンジンのコード名は「Razor」で、最初のベータ版がまもなく利用可能になります。



開発目標



プロトタイプを開発し、Razorを評価している間に、いくつかの重要な目標を特定しました。

  • コンパクトで表現力豊かで柔軟 ：Razorは、ファイルでの作業に必要な「キーボードの負荷」を軽減し、迅速で柔軟なプログラミングプロセスを提供します。 ほとんどのテンプレート構文と異なり、HTMLでサーバー側のコードブロックを明示的に定義するためにコードの書き込みを中断する必要はありません。 パーサーは、コードに基づいてこれを行うのに十分スマートです。 入力が簡単で楽しいコンパクトで表現力豊かな構文が得られます。
  • 習得が簡単：Razorは習得が容易で、最小限の労力で短時間で生産性を高めることができます。 既存のプログラミング言語とHTMLスキルをすべて使用します。
  • 新しい言語ではありません ：Razorの新しい命令型言語に基づいていないパスを意図的に選択しました。 代わりに、開発者が既存のスキルを使用してC＃/ VB（または別の言語）を操作できるようにし、驚くべきHTML構築プロセスを提供するテンプレートマークアップ構文を提供し、言語の選択はユーザーに任せています。
  • 任意のテキストエディターで動作します。Razorは特定のツールを必要とせず、古いテキストエディターで生産性を発揮できます（メモ帳でも問題なく動作します）。
  • Intellisenseのサポート ：Razorは特定のツールやコードエディターにバインドせずに設計されたという事実にもかかわらず、Visual Studioで優れたサポートを提供します。 Visual Studio 2010およびVisual Web Developer 2010は、すぐにインテリセンスの更新を受け取ります。
  • ユニットテスト ：エンジンの新しい実装は、ビューでユニットテストを実行する機能をサポートします（コントローラーやWebサーバーを必要とせず、ユニットテストプロジェクトに配置できます-特別なアプリケーションドメインは不要です）
この数か月間、このテクノロジを使用してアプリケーションを開発し、.NET以外の開発者のグループを含むさまざまなボランティアでユーザビリティ調査を実施しました。 人々からのフィードバックは素晴らしかった。



選択と柔軟性



ASP.NETの最大の機能は、文字通りコードの一部を置き換えることです。 特定のメカニズムがニーズに合わない場合は、簡単に実装に置き換えることができます。



次のバージョンでは、ASP.NET MVCに新しい[追加]-> [表示]ダイアログボックスが追加され、新しいプレゼンテーションテンプレートファイルの構文を選択できるようになります。 ローカルマシンにインストールされている使用可能なプレゼンテーションエンジンを簡単に選択できます。







Razorは、ASP.NET MVCに直接統合されるため、リストに追加されます。 すべてのヘルパーメソッドとモデルは、Razorと.ASPXの両方のプレゼンテーションエンジンで使用できます。



同じアプリケーションまたはサイト内のさまざまなエンジンを使用して記述されたプレゼンテーションテンプレートを組み合わせて比較する機会があります。 たとえば、.aspxを使用していくつかのビューを作成し、.cshtmlまたは.vbhtml（それぞれC＃およびVBのRazorファイル拡張子）を使用するビューと、SparkまたはNHamlを使用するビューを作成できます。 それだけでなく、別のエンジンで記述された部分ビューを使用して、1つのエンジンでプレゼンテーションテンプレートを作成できます。 あなたはキャプテンであり、必要なものを選択します。

レイアウト/ MasterPage-基本



Webサイトまたはアプリケーションのすべてのページで統一された外観を持つことが非常に重要です。 ASP.NET 2.0は、「ページマスター」の概念を導入しました。これは、.aspxページまたはテンプレートを使用するときにこの結果を達成するのに役立ちます。 Razorは、「レイアウトページ」を使用してこの概念もサポートしています。これにより、サイトテンプレートを定義し、そこからビュー/ページを継承できます。



レイアウトの最も単純な例



以下は、「SiteLayout.cshtml」という名前で保存するレイアウトページの簡単な例です。 静的なHTMLと動的なサーバーコードを含めることができます。 次に、リクエストされたURLの特定のコンテンツ本文を表示する場所に、「RenderBody（）」ヘルパーメソッドへの呼び出しを追加します。







次に、「Home.cshtml」という名前のプレゼンテーションテンプレートを作成します。このテンプレートには、要求されたページの特定の本文を作成するために必要なコードまたはコンテンツのみが含まれ、レイアウトテンプレートに基づいています。







Home.cshtmlファイルのコードで「LayoutPage」プロパティを明示的に設定する方法に注意してください。 これは、このビューのレイアウトとしてSiteLayout.cshtmlテンプレートを使用することを意味します。 Home.cshtmlをプレゼンテーションテンプレートとして指定するか、サイトの標準レイアウトでカスタマイズすることにより、ASP.NET MVCコントローラーでレイアウトファイルを直接設定することもできます（この場合、1つのプロジェクトファイルで定義し、すべてのプレゼンテーションテンプレートが自動的に選択します）



Home.cshtmlをプレゼンテーションテンプレートとして生成すると、レイアウトとサブページのコンテンツが結合され、そのコンテンツがクライアントに送信されます。

ページレイアウトは、独自のコンテンツでビューを変更できるさまざまな「セクション」を内部で定義する機能をサポートしています。 これにより、レイアウトページのコンテンツの断続的なセクションを簡単に変更/入力でき、サイトのレイアウトの柔軟性が向上します。

「Razor」は、コード指向のテンプレートに新しいエレガントなプレゼンテーションエンジンを提供すると考えています。 プログラミングプロセスは高速で楽しいものになります。 構文はコンパクトであり、キーストロークを減らしながら、マークアップとコードの可読性を向上させます。 ASP.NET MVCの次のバージョンでは、エンジンはシステムに統合されます。 アプリケーションの.cshtml / .vbhtmlファイルを削除し、個別のページとして実行することができます。これにより、ASP.NET Webフォームアプリケーションで使用することもできます。
