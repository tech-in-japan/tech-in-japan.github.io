Oracle Storage Cloud Services─エンタープライズデータウェアハウスに必要なものすべて

パブリッククラウドの企業データウェアハウスと関連サービスは、すでにお客様によって十分に検証されています。 データストレージ、アクセス、セキュリティのニーズを満たします。 オラクルを使用すると、このデータの安全で柔軟かつ簡単な使用が、いつでもインターネット関連のIT環境で利用できます。











オラクルは、ストレージコストを削減できるデータウェアハウスを開発および提供しています。また、配置と単一アクセスインターフェイスを最適化することにより、あらゆるレベルで保存されているデータへのアクセス時間を最小限に抑えています。



この記事では、データストレージサービス、このクラスの他のシステムとの違い、ソリューションアーキテクチャの構造、およびOracle Database Cloud Serviceの実際の使用例について説明します。



1. Oracle Storage Cloud Service─オブジェクトストレージ



このソリューションは、構造化データおよび非構造化データを含む、あらゆるタイプのデータセットのストレージを提供します。 データのコピーは、オンプレミスシステムに保存したり、クラウドに移動したり、共有したりできます。



このアプローチにより、データセンターの運用が簡素化され、新しいハードウェアを必要とせずに拡張され、電力消費と冷却システムの要件が減るため、設備投資と運用コストが削減されます。



共通のインフラストラクチャにより、実質的に無限のスケーラビリティが可能になります。 これにより、絶え間ない予測や、新しい容量を獲得するための長いサイクルが不要になります。



OpenStackおよびRESTful APIに基づくシンプルな管理により、統合が簡素化され、他のクラウドプロジェクト用のリソースが解放されます。 従量課金およびサブスクリプションモデルは、長期契約と比較してコストを削減します。



クライアントデータの暗号化は、Oracleデータセンターの追加の暗号化と組み合わせて、マルチレベルの情報保護を提供します。 クライアント側の暗号化は、Oracle Storage Cloud Software ApplianceとJava SDKによって実行されます。 すべてのローミングデータは、SSL（Secure Sockets Layer）レベルで暗号化されます。 データへのユーザーおよび管理者のアクセスレベルは、コンテナレベルで実行されます。



マスターバックアップおよびアーカイブアプリケーションとの統合により、データの展開と復旧が簡単になり、被害が最小限に抑えられます。 最も困難なワークロードの場合は、パブリックインターネットとFastConnectの両方を介して接続できます。



ジオレプリケーションテクノロジーがサポートされています。地理的に離れたデータセンターにデータのコピーを自動作成します。 同時に、データプライバシールールが守られています。保存されたデータは、管理者が指定した地域から移動できません。



開発標準─Open Stack SwiftのRestAPI、Javaアプリケーション（ファイル転送マネージャーおよびJava API）、シェルスクリプト（アップロードCLI）。



ソリューションには「11の9の信頼性」があります。つまり、異なるデバイスで各オブジェクトの複数のコピーがサポートされるため、99.999999999％の信頼性が提供されます。 定期的なデータ整合性は、自己修復ツールによって検証されます。



Oracle Storage Cloud Software Appliance─NFS互換のPOSIXインターフェースを備えた使いやすいクラウドストレージ。 クラウドへのNASゲートウェイです。



2. Oracle Storage Cloud Service─アーカイブストレージ



これらは、業界で最も費用対効果の高いデータウェアハウスです。 エンタープライズレベルのセキュリティと柔軟な拡張性を備えた、頻繁なアクセスを必要としないデータ用に特別に設計されています。



典型的なアプリケーションは、大規模なデータセットとその長期保存用のクラウド内のアーカイブ、および異種マルチメディアコンテンツ、研究結果、文化遺産のデジタルコピーです。



このソリューションの主な利点には、データセンター運用の最大限の簡素化、低消費電力と冷却システムの要件による低資本投資と運用コストが含まれます。 従量制およびサブスクリプションモデルも実践されており、アーカイブストレージの合計料金は、ピア間で最低です（年間12 TB / TB）。



クライアントデータは暗号化された形式で保存され、トラフィックはSSLで暗号化され、役割ベースのアクセスはコンテナレベルで実行されます。 アクセス、開発標準、接続性、信頼性、ジオレプリケーションは、オブジェクトストレージと同じです。



3. Oracle Database Backup Service



データ保護サービスは、RMAN（Oracle Recovery Manager）との直接統合により、Oracle Databaseクライアントの固有のニーズに合わせて設計されています。 したがって、ITプロセスでクラウドベースのデータ保護を活用できます。



バックアップはデータソースで直接暗号化され、確実に送信されてクラウドに保存されます。 データは常に指定された地理的地域に保存されます。 組み込みの冗長性が使用されます-ミラーリングは3つの方法で実行され、データの可用性を常に確保します。 バックアップはRMANから簡単に暗号化されます。 データは常に監視されており、データの劣化を防ぎます。



バックアップは、クラウドに移行する前に圧縮されます。 データウェアハウスの容量は、ボタンをクリックするだけでオンデマンドで変更されます。 データ管理には、指定された基準（保存期間など）に応じたロールベースのアクセス制御とデータの自動削除が含まれます。



4. Oracle Storage Cloud Softwareアプライアンス



このソリューションは、役割ベースの暗号化を提供し、チェックサムをチェックすることでデータの整合性を確保し、ファイルとオブジェクト間の自動変換を実行し、データキャッシングとdockerテクノロジーによりローカルNASレベルでパフォーマンスを発揮します。



このソリューションには、使い慣れた、よく理解されたインターフェイスがあります。 NFSベースのクラウドデータストレージにアプリケーションを接続するだけでなく、POSIXおよびUnix / Linux NFSクライアントとの互換性もあります。 自動データ変換の実装-インターフェースにより、ファイルと同様にデータを操作できますが、データはクラウドにオブジェクトとして保存されます。



情報セキュリティツールは十分に開発されています。 送信前に、データは暗号化されます。 詳細（選択的）を含む、クライアントによる暗号化の可能性。 対称および非対称キーを備えたマルチレベルアーキテクチャは、最大限の情報セキュリティのために使用されます。 ユーザーは、独自のキーを持ち込むか、暗号化の直前にキーを生成できます。



メタデータをクラウドに定期的にバックアップすることでデータの可用性が確保され、ローカルキャッシュレベルでのパフォーマンスはバッファーキャッシュによって提供されます。 キャッシュは、ワークロードに応じて構成できます。 最長時間未使用（LRU）アルゴリズムは、最新のデータをキャッシュし、不要なデータを削除します。



管理には、N-Way Managementテクノロジーが使用されます（N-Way─イーサネットネットワークデバイスとユーザーデータを管理するための通信プロトコル）。 ブラウザまたはコマンドラインインターフェース（コマンドラインインターフェース）に基づいて、管理UIから作業できます。 スクリプトは自動化に使用できます。 単一の中央の場所から複数の展開を提供するREST API。



5. Oracle Public Cloudデータ転送サービス



サービスは、初期データセットをクラウドに高速転送し、パブリックOracleクラウドにデータウェアハウスを形成します。 これは、クラウド内のデータを開始するための最速の方法です。 Storage Appliance Importサービスは、大規模なデータセット（履歴アーカイブ、データレイク、継承データの大規模なデータベース）を確実に転送します。 一度に最大400 TBのデータを移動できます。



同時に、1回限りの「卸売」データ転送のためのより高速なネットワークの構築に費用を支払う必要はありません。 トラフィックはAES-256を使用して暗号化されるため、送信は安全です。 シンプルな標準NFSv3インターフェイスを使用して、オブジェクトをロードしたり、ストレージデータをアーカイブしたりします。 同時に、複数列構造が形成されます。 クラウド内のデータをコピーして、作業ベースのレプリカを作成することもできます。



Oracle Storage Cloud Serviceおよび従来のソリューション



従来のストレージソリューションには、スケーラビリティ、パフォーマンス、および管理に問題がある傾向があります。 Oracle Storage Cloud Serviceはそれらの克服に役立ちます。



1.直接接続されたストレージデバイス（デスクトップまたはラップトップの通常のハードドライブなど）を備えたシステムでは、OSはファイルシステムを介してデータのストレージ、検索、および編成を制御します。



このようなデバイスは、低レイテンシとかなり信頼性の高い長期データストレージを提供します。 ただし、大容量ドライブが必要な場合は、個々のディスクデバイスに分散されるため、スケーリングが難しくなり、パフォーマンスが低下します。



2.ネットワーク接続ストレージ（NAS）デバイスでは、ストレージハードウェアは、アプリケーションが実行されているサーバーから物理的に分離されています。 ストレージデバイスはネットワークドライブとして使用できます。 データの保存、取得、およびシーケンスは、ネットワークファイルシステム（NFS）によって制御されます。 異なるサーバーで実行されるアプリケーションは、データストレージ用にNASを共有します。 ローカルネットワークのストレージリソースの管理は集中的に実行されます。



直接接続されたストレージデバイスと同様に、NASアプリケーションもメインOSに依存していますが、ネットワークファイルシステムにも依存しています。 しかし、そのようなアーキテクチャは限られた地理的領域でしか実装できず、スケーラビリティも限られています。



3.ブロックストレージの原則は、1秒あたりの入出力操作（IOPS）が高いOLTPデータベースなどのアプリケーションで使用されます。 これにより、OSをバイパスしてデータを効果的に保存および復元し、仮想ブロックデバイスと直接やり取りすることができます。



データフラグメントはブロックに保存され、それぞれが独自のアドレスを持ちますが、他のメタデータはありません。 データの保存場所は、アプリケーションが決定します。 対応するブロックアドレスにデータを復元します。



ブロックストレージの原則は、IOPS用にストレージシステムを最適化し、Oracle Compute Cloud ServiceにPOSIX互換のファイルシステムを提供します。 ただし、このアプローチにはスケーラビリティの点で制限があり、メタデータのグラニュレーションはサポートしていません。



4.オブジェクトデータストレージは、大量の非構造化データを格納しながら、パフォーマンス、スケーラビリティ、管理性の最適な組み合わせを提供します。 複数のストレージノード（ノード）は、データがオブジェクトとして保存される単一の共通の水平方向にスケーラブルなプールを形成します（「データプール内のデータドロップ」）。



すべてのオブジェクト保存データはメタデータに関連付けられており、一意の識別子を持っています。 したがって、カスタムメタデータをコンテナとオブジェクトに割り当てることができ、データの検索、分析、管理が非常に容易になります。 アプリケーションは、オブジェクト識別子を使用して、REST API呼び出しを介してデータに直接アクセスします。



オブジェクトストレージは使いやすく、その仕事をうまくこなし、事実上無制限に拡張できます。



上記を要約すると、Oracle Storage Cloud Serviceは、安価で信頼性が高く、安全でスケーラブルなストレージソリューションを提供することに注意してください。 これにより、非構造化データを保存し、いつでもどこからでもアクセスできます。



このアプローチは、データのバックアップとアーカイブ、ファイルの共有、およびログ、センサーから受信したデータ、仮想マシンのイメージなどの大量の非構造化データの保存に最適です。



アーキテクチャ概要



Oracle Storage Cloud Serviceのアーキテクチャは非常に手頃な価格で冗長です。 ユーザーアプリケーション、Java SDK、RESTクライアントなどの外部アクセスに最適です。



オブジェクトがOracle Storage Cloud Serviceに格納されると、データセンターの複数のストレージノードノードを通じてデータが複製されます。 この戦略により、保存されたオブジェクトデータがハードウェア障害に耐えることができます。
