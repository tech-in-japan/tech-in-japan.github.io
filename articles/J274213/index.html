HIVE：クラウドPBXの未来または来年の行き先

Habréの企業ブログは、私たちの意見では、コーディング技術についてコミュニティに伝え、技術的な「機能」を説明するだけでなく、購読者や読者とニュース、考え、気分を共有することも目的としています。 今日、CRMの別のフロントエンドをどのように作成したか、クラウドベースのPBX冷蔵庫、コーヒーメーカー、および電子機器をどのように同時に統合したかを説明するのに長くて退屈なことはありません。 クラウドサービスプロバイダーとして、私たちは次の年を検討し、次の年に行われたことを共有したいと考えています。







最初に注目したいのは、クラウドソリューションへの関心が高まっているという明確な傾向です。 2015年の初めには、仮想エクスチェンジやCRM統合に誰も興味がなく、需要は安定しており、新しいユーザーを定期的に接続して統合ソリューションを実装したとは言えませんが、仮想エクスチェンジはボックス内にのみ存在するべきだと考えました見た目では、中小企業にVoIPのあらゆる利点を提供できます。 同時に、仮想PBX ZEONは最初からほぼ2つの装いで存在していました-ボックスとパブリックSaaSです。 amoCRMとの統合およびFMC IP SIMカードを使用したプロジェクトの立ち上げSIMがクラウドドライブに大幅に追加されました。 統合ソリューションの販売における最初の経験は、傾向を明確に概説しました-クラウドは「支配」し、パブリックSaaSを提供せずにボックスのみを販売し続けます-省略、エラー、誤算。 この非常に波に乗って、売り手の絶え間ない悩みの中で、私たちは完全にクラウドベースのHIVEプロジェクトを立ち上げました。これは2016年には機関車製品になると考えています。



通常のクラウドベースPBXに追加されたいくつかの興味深い「機能」のうち、FMC IP SIMカードとの統合である「Trend 2015」に言及したいと思います。 これが「撃つ」ものであり、将来販売される可能性が高いものです。 IP SIMに基づいて、ビジネスのさまざまなセグメント向けにいくつかのソリューションを作成する予定ですが、ここでは、作成済みのソリューションを簡単に示します。







Pocket in Officeは、携帯電話のクラウドPBXです。 このソリューションを使用するには、ソフトフォンやIP電話を使用する必要はまったくありません。 IP SIMは、最も一般的なオフィスの電話のようにクラウドPBXに直接接続されており、ワイヤーやリダイレクトを使用せずにコールを処理するために必要なすべてのシナリオを構成できます。 携帯電話からの通話はPBX統計に記録され、記録されます。携帯電話には短い番号があります。携帯電話からの発信中に、セル間で通話を転送できます。携帯電話番号ではなく、単一の会社番号です。



似たようなものがすでに私たちの前に市場に存在していましたが、それらの以前のソリューションでは明確な分離がありました。携帯電話は企業のモバイルとオフィスの2つのネットワークのように機能しました。 私たちの場合、この部門は消滅し、企業の携帯電話は最も一般的なオフィスの電話になりました。







完全にモバイルな会社の従業員は、オフィスに常駐したり、クラウドシステムにインターフェイスを持つ必要はありません。 すべてのコールシナリオを一度設定し、IVRをダウンロードし、モバイルに短い番号を割り当てるだけです。 その後、一定の動きをしながら、携帯電話を散らばって作業することができます。 私たちの観点からは、これはオフィスやクラウド交換にお金をかけたくないが、成功のすべての属性を持つ美しい大企業のようになりたいマイクロビジネス向けの優れたソリューションです：美しい着信番号、電子秘書、通話転送、音声通知。



モバイルテクノロジーとCRMシステムと統合されたHIVEクラウドベースPBXの2番目の化身は、IP SIMと多くの着信番号を持つシンプルなクラウドCRMの友情です。ロシア全体で100以上の仮想DIDを持つSIMカードの一種です。 クライアントはスマートフォンとのみ対話し、稼働しているコンピューターとは時折しか対話しないため、Multi IP SIMと呼ぶことにしました。 Multi IP SIMの販売経験は、私たちが要点を示し、機密サイトや多数の着信コールを通じて顧客と積極的に連携するマイクロビジネスの生活を楽にする方法を見つけたことを示しました。







すべてのアナウンスメントからのすべてのコールは1枚のSIMカードに送られ、CRMで修正されます。スマートフォンの着信コールを使用すると、どのアナウンスメントからのアナウンスを確認できます。 通信履歴はCRMに保存されます。これにより、通話統計を収集し、各製品の広告の有効性を評価し、広告会社によるコンバージョンを評価できます。



開発者が開発したモバイル会議サービスをこのような統合とソリューションに追加すると、携帯電話、CRM、クラウドPBXなどの小規模ビジネスを自動化するための優れたツールを手に入れることができます。







携帯電話の従業員を会議に接続できるようになりました。携帯電話の状態（通話中、オフライン、オフ、通話中）を考慮に入れてください。 地下鉄に乗っている人や携帯電話がオフになっている人を自動的にダイヤルアップし、ネットワーク上での出現の事実を分析します。



私たちは統合に多く取り組んでおり、統合はクラウドサービスを開発する方法であり、積極的に追求する必要があることを理解しています。 ほんの数年前、クラウドサブスクリプションはあいまいな「不必要なもの」のセットとして認識されており、相互に疎結合されており、各コンポーネントが厳密に定義された機能を実行するビジネス用の単一のクラウドIPを表していません。 状況は根本的に変化しています-CRMのないクラウドベースのPBXは間違っています;オフィスの電話ネットワークと統合されていないセルラー通信は非常に間違っています。 私たちは来年の統合に取り組み、さらに多くのクラウド、スクリプト、コネクタ、REST API、および主な売り手と売り上げを提供し、開発者の毎晩の集会を収益に変えます。 -商業部門では、着信コールに座っています。



今後の祝日におめでとうございます、私たちは先に進み、独自のプロジェクトを開発し、スキルを向上させたいと考えています。 私たちは皆、非常にダイナミックで豊かな環境、つまりIT技術の分野で働いています。
