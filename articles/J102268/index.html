オラクルの行動はオープンソースキャンプの中性子爆弾です

ご存知のように、OracleはAndroidオペレーティングシステムのライセンスなしでJavaの使用に関連してGoogleを訴えました。 当然、グーグルはこの主張を拒否し、他の人々の知的財産権を侵害しなかったと言っています。 これはすべて非常に正常であり、通常の法的戦いの一部です。 しかし、異常なのは、彼らが言うように、小さな活字で印刷されているOracleの主張のその部分です。 見ると、Oracleの要件とその実際のアクションは、Googleだけでなく、オープンソースコミュニティ全体を深刻に脅かすはずです。



オラクルは、Androidのすべてのコピー（このオペレーティングシステムで販売されているデバイスを含む）、および3倍の損害と法的費用を破棄することを要求しています。 もちろん、これはブラフです。 実際、グーグルはかなり安く支払うことができますが、グーグルにとってこの有利なステップはIT産業にとって非常に危険です。 それがトラップです。



InfoworldのMartin Hellerが昨日、Oracleの主張の分析と、この訴訟の発展の可能性の予測を発表しました。



一見したところ、Googleの状況はかなり悲観的に見えます。



たとえば、この場合のOracleの弁護士の1人はDavid Boiesです。 彼は最近、カリフォルニア州での同性結婚の禁止を解除する訴訟で勝訴したことで知られています。また、Microsoftに対する古い独占禁止法の裁判もあります。 彼はまた、ナップスター事件のRIAAの顧問であり、2004年の米国大統領選挙での投票数を争う聴聞会でアルゴア副大統領を代理しました。 彼が非常に深刻な意図がない限り、誰もデイビッド・ボーイズを雇いません。



Oracleは、7つのソフトウェア特許の侵害を主張しています。



*コンピューターシステムにセキュリティを提供する保護ドメイン（2000）

*リソースへのアクセスの制御（2000）

*クラスファイルの前処理およびパッケージ化の方法と装置（1999）

*マスターランタイムシステムプロセスのメモリスペースクローン作成によるクラスの動的プリロードのシステムと方法（2008）

*コード生成（2003）でデータ参照を解決する方法と装置

*仮想マシン命令とネイティブマシン命令のハイブリッドを使用した関数の解釈（2005）

*静的初期化を実行する方法とシステム（2000）



これらはすべてソフトウェアの特許です。



残念ながら、 ビルスキー事件の最近の判決にもかかわらず、米国最高裁判所はソフトウェア特許を完全に取り消すことを敢えてせず、ソフトウェア技術の特許化の不条理は多くの人に明らかであるが、この問題に関する議論はまだ進行中です。 そして、最高裁判所はこれに向かっているようです。



著作権侵害の申し立てもあります。 しかし、AndroidチームがJava VMの純粋なリバースエンジニアリングの結果としてDalvik VMを作成し、それを証明できる場合、訴訟の開始時にそれらはすべて拒否されると思います。



Phoenix TechnologiesはかつてIBM PC BIOSを使用してこのようなトリックを行うことができたため、Javaの場合にGoogleがこれを証明できる可能性は十分にあります。



致命的なOracle要件

オラクルは、Androidのすべてのコピー（このオペレーティングシステムで販売されているデバイスを含む）、および3倍の損害と法的費用を破棄することを要求しています。 また、トライアルも必要です。



これらは明らかに誇張された要件です。 彼らが言うように、彼らは「もっとポニー」を求めることができました。 北カリフォルニアでの裁判の結果に基づいても、裁判所は上記のすべてを彼らに与えません。 私が言えることは、 東テキサスでも彼らはこれを達成しなかったでしょう。



しかし、Oracleが勝った場合でも、いくつかの要件を満たすことができます。 ただし、この場合、Googleの控訴は完全に提出され、控訴が審査されるまで裁判所は確かに要件の実行を遅らせます。



Googleが控訴に勝った場合、Oracleはその控訴を最高裁判所に提出できます。 ただし、このレベルになると、最高裁判所がすべての特許を完全に取り消すと決定する可能性が高くなります。 この脅威だけで、Oracleがプロセスをその段階に持ち込むのを防ぐのに十分です。



Googleはどのように同意できますか

グーグルは、オラクルとの試用前の合意に達することができるのはかなり安い。 これには、現金での支払いまたはクロスライセンスの2つのオプションがあります。



ただし、そのような合意は、たとえGoogleにとって有益であっても、IT業界に深刻な害を及ぼす可能性があります。 Googleがお金を払うことに同意すると、一連の新しい犠牲者がOracleのために並んでいます。これは、決定の結果によってすでに無防備になります：HTC、Motorola、Samsung、他のスマートフォンメーカー、Verizon Wireless、T-Mobile、他の通信事業者、さらには大企業、 Androidデバイスを使用するユーザー。 これはモバイル業界にとって本当の悪夢かもしれません。



誰もがクロスライセンス契約を締結する方がはるかに良いと思われました。 この場合、ほとんどの場合、Googleは将来のOracleからの申し立てからすべてのライセンシーとAndroidユーザーを保護します。



私の同僚の中には、SunのすべてのオープンソースプロジェクトをOracleの将来の行動から何らかの形で保護するために、 すぐに分岐することを求める人もいます。 しかし、プロジェクトが本当に安全なとき、彼らがそのような段階に問題をもたらすことができるとは思わない。



痛いところを打つ

私が20年のように急進的な場合、この訴訟はオープンソースコミュニティに中性子爆弾を投下することに似ているため、Oracle製品の世界的なボイコットを求めます。



考えてみてください。OracleDBMSユーザーはIBM DB2またはMicrosoft SQL Serverにアップグレードでき、MySQLユーザーはMonty forkまたはPostgresに移行できます。 Oracle FormsおよびOracle Web FormsのユーザーはAlpha Fiveに移行でき、Oracle ERPの顧客はSAP、Microsoft、またはSageから選択できます。
