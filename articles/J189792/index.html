アプローチ



移動と位置決め


キーストロークとリリースキー、位置、マウスクリックを記録するだけでは機能しません。 より正確には、次の理由により、予想したとおりに再生されない場合にのみ記録されます。

  1. キャラクターの初期位置と彼の視線の方向を再現する必要があります。 自動レースの前に反対方向にわずかに曲がると、曲がって走り、遅かれ早かれ壁にぶつかります
  2. 正確な座標と角度を書き留めると、そこにキャラクターを配置することに失敗します。 ブルドーザーのようなボタンに反応し、同じように回転します。 そして、100メートルの1度の誤差でも、 メーター（戸口幅）
  3. ボタンを押して離す時間も理想からはほど遠い。 目を閉じてアパートを歩き回るようなものです。 6歩先、厳密に右側にあるように見えますが、実際には、トイレの代わりに、私たちはトイレにいます。 また、ハンドルを感じずに目を閉じてドアを開けるようにしてください。


これらの理由から、座標を使用して世界に自分自身を配置する必要があります。 同時に、位置を確認し、100ミリ秒ごとに定期的に調整する必要があります。

モジュール


したがって、ボットのハードウェアとソフトウェアの複合体の一部が迫っています。

  1. WoWインターフェース用のアドオンを作成し、キャラクターの位置、方位角、傾きを決定します。 ゲームアドオンのルールは禁止されていません。指定された情報は、アドオン内のゲームAPIから入手できます。 彼はそれをすべて画面に描きます。 アドオン自体はLua言語で記述されています。これは、ゲームの開発者が決定したためです。
  2. アドオンでは、何かを定義したり、描画したりできますが、キャラクターを動かすことはできません。 アドオンでも、ファイルに書き込むことはできません。壁の場所、敵の場所、そしてまだできないことをゲームに尋ねることはできません。 したがって、 外部制御アプリケーションを作成する必要があります 。 私はオリジナルにならず、 AutoItを使用しました。 強く型付けされた言語でのプログラミングに慣れている私にとっては、少しワイルドですが、本当に高速で便利です 。

    • アプリケーションは自分のアクションを記録できるはずです。 この機能レコーダーを呼び出す
    • アプリケーションは記録を再生できる必要があります。 プレイヤーを呼び出す
  3. 記録されたアクションと座標は、行ごとにテキストファイルに保存されます。 これらは手動で編集できますし、編集する必要があります。 松葉杖を追加し、ごみを取り除きます。 これが最も難しいとすぐに言います。




LuaのWoWのアドオン



WoWのアドオンは、 Lua言語のスクリプトです 。 この言語は、完全で柔軟性があり、効果的です。 このシステムでは何もできません。 ゲームですべてを実行できますが、ゲームが提供するAPIのフレームワーク内でのみ可能です。 Blizzardはプレイヤーや起業家がボットを書くことを望んでいないか、むしろ禁止しているため、上記の座標に加えてAPIはほとんど有用なものを提供しません。 もちろん、私たちのニーズに役立ちます。 Luaの説明を含むアドオンをすぐに作成するための優れたガイドは、 ここにあります 。 さて、オプションについて説明します。

スラッシュコマンド


ところで、デバッグについて。 すぐにスラッシュコマンドハンドラーを追加することをお勧めします。 



SLASH_HELLO_WORLD1 = '/helloworld'; function SlashCmdList.HELLO_WORLD(msg, editbox) local facing = GetPlayerFacing(); local pitch = GetUnitPitch("player"); local x, y = GetPlayerMapPosition("player"); print(format("HelloWorld %.2f %.2f %.2f %.2f", x*100, y*100, facing, pitch)); end


これを行うには、 SLASH_IMnの形式の変数を定義します。 NAMEはすべてのアドオンで一意であり、 nは空または1からの序数です。そして、 NAMEという名前の関数をオブジェクトに追加します。 C ++プログラマーにとって、このハンドラー関数をどこにも明示的に登録していないのは奇妙に思えるかもしれません。 はい、変数の名前だけで文字列変数に関連付けます。 しかし、ここにあるのは、Luaの力と魔法です。



これで、WoWコンソール/ helloworldのチームが必要なアクションを実行します。デバッグ情報を表示できるようになります。実際、最も単純なアドオンについては、作成したすべての操作を実行します。



それでは、必要な情報を引き出すWoW APIを紹介しました。



イベントハンドラー


アドオンでGUIを操作するためのシステムは、メッセージとその処理の点でWindowsの動作に似ています。 



 local EventFrame = CreateFrame("Frame") function EventFrame:OnEvent(event, ...) print("HelloWorld:", event) self[event](self, ...) end EventFrame:SetScript("OnEvent", EventFrame.OnEvent) EventFrame:RegisterEvent("PLAYER_LOGIN") function EventFrame:PLAYER_LOGIN() --   end function EventFrame:OnUpdate() --    end EventFrame:SetScript("OnUpdate", EventFrame.OnUpdate)


「 Frame 」タイプのフレームを作成し、コマンドで「 OnEvent 」および「 OnUpdate 」アクションに接続します 



 EventFrame:SetScript("OnEvent", EventFrame.OnEvent) EventFrame:SetScript("OnUpdate", EventFrame.OnUpdate)


OnUpdateハンドラーは各フレームの後に呼び出されます-座標を更新するために必要なもの。 OnEventは、他の必要なイベントで呼び出されます。 それから対応する関数を引き出します： 



 self[event](self, ...)


Luaでさらに便利なのは、この設計を使用すると、フォームの一意のハンドラーを呼び出せることです。 



 function EventFrame:PLAYER_LOGIN()


ここで、 PLAYER_LOGINは、ワールドに入ってすべてのフレームがユーザーインターフェイスを再起動したときに送信されるイベントです。 その他のイベント： PLAYER_LEAVE_COMBAT 、 QUEST_FINISHED 、 PLAYER_EQUIPMENT_CHANGED 、 PLAYER_DEADなど。 完全なリストはここにあります 。 コマンドで処理したいイベントを登録します 



 EventFrame:RegisterEvent("PLAYER_LOGIN")

連載


2つの座標と2つの角度があります。 すべて浮動小数点数です。 アドオン内のピクセルの色のコンポーネントも浮動小数点数ですが、 0.0〜1.0です。 さらに、色成分は1バイトのままであることが知られています。 最初に、各座標を1つのカラーコンポーネントに保存しました。 その結果、その場所での測位精度は地図のサイズの1/255でした。 GPSのようになりました：座標があるように見えますが、ナビゲーターを通る道路でマシン（人なしでコンピューターを制御）で車を運転することはできません。 だから私は2バイトを与えなければなりませんでした。 1つの小数を2バイトで保持する方が良いでしょうか？ 私はそうしました 



 local x1, x2 = math.modf(x*255)


modf関数は、コンマで区切られた数値の整数部と小数部を返します。 ここでは並列割り当てが使用されます-もう1つの便利なLua機能です。



このアプローチでは、座標が格納される2バイトのそれぞれの力を最大限に活用します。 さて、方位角と傾斜角は、それらが間隔0.0-1.0に収まる場合に限り、それほど正確ではありません。 次のようになります。 



 local math = getfenv(0).math function EventFrame:OnUpdate() local facing = GetPlayerFacing(); local pitch = GetUnitPitch("player"); local x, y = GetPlayerMapPosition("player"); local x1, x2 = math.modf(x*255) local y1, y2 = math.modf(y*255) HelloWorld1.texture:SetTexture(x1/255, x2, facing/7) HelloWorld2.texture:SetTexture(y1/255, y2, pitch/4+0.5) end


現在、左上隅で世界中を移動すると、2つの10 x 10の正方形がランダムに色を変えます。
