Huawei Enterprise Server

この夏の初めに、私は深Shenzhenのファーウェイ本社への旅行から戻りました。ここでは、次のファーウェイプリセールスエリートキャンプが開催されました。

今回は、親切な中国人が彼らのサーバーラインを紹介してくれました。 新しいIntel Xeon V3プロセッサのリリースに伴い、大幅な変更が行われました。これについては、この記事で説明します。









ほとんどのベンダーのサーバーラインの更新が、Intelプロセッサの新世代のリリースと同時に行われるのは秘密ではありません。 Huaweiも例外ではなく、Intel Xeon V3に基づく新しいソリューションをリリースしました。 以下では、彼らと知り合い、彼らが現時点で何を提供できるかを理解することを提案します







RHシリーズラックサーバー



かなり多くのレビューがすでにHuaweiラックサーバーに掲載されているため、詳細については説明しません。 私はあなたに興味があるかもしれない少数の事実だけに言及します。 Huawei Enterpriseの公式Webサイトで、すべてのモデルの詳細な仕様をいつでも見つけることができます。







すべてのラックモデルの名前は次のとおりです。RH****（例：RH 1288）



•Rはラックマウントを表します。

•Hは、サーバーにIntelプロセッサーが搭載されていることを意味します。 手紙Iを 90⁰にすると、手紙Hが得られます。 ここにそのような神秘的な東洋の論理があります。

•最初の数字は、サーバーの高さの単位に対応しています。

•2番目の数字は、プロセッサソケットの数に対応しています。

•最後の2桁は、モデルの「レベル」に対応しています。 Huaweiは新しいモデルをリリースし、市場からレビューを収集します。 また、大きなニーズがある場合は、サーバーのパラメーターを多かれ少なかれ「調整」できます。 そのため、プロセッサ、サポートされる最大メモリ数、PCIeスロットの数が異なる、同じモデルのさまざまなグラデーションがあります。

o 85-最初のバージョン。

o 88-古いプロセッサモデルを備えた「再発行」バージョン。

o 85A / 88A-「弱められた」予算バージョン。

o 85H / 88H-拡張バージョン（ハイエンド）。



ただし、ラインナップを慎重に検討すると、矛盾に気付くことができます。RH5885/ RH5885Hモデルの高さは4Uと4ソケットです。 これは、中国文化の4番は「死」という言葉と同様に発音されるため、残念なことです。 したがって、あらゆる方法で中国人はこの数を避けます。

また、これはIT分野だけでなく、どの家の4階のアパートのコストも、他のどの階よりも1桁少なくなります。



Huawei RHサーバーでのV3モデルの出現により、次のことが可能になります。

•Intel Xeon V3プロセッサー。

•ランダムアクセスメモリDDR4。

•I / Oカード16 Gbs FC、56 Gbs IB、FCoE、および10 Gbs銅（RJ-45）イーサネット。

•最大1.6 TbのPCIe SSDカード。

•多くの人々が長い間待ち望んでいたもの-SASバックエンド経由でDASとストレージを接続するためのSAS HBA（LSI SAS2008）の公式サポート。

•最大2つのSATA DOM 32Gbをインストールする機能。



さらに、それほど重要ではない改善点に注目する価値があります。

•プロセッサーの最大温度しきい値を最大45°に上げる。

•iMana（リモート管理ツール）はiBMCに置き換えられました。

•LCDタッチスクリーン診断パネルの出現。







おそらく、より多くのプロラックソリューションには何も言えません。



Xシリーズ高密度ソリューション



Huaweiは、X6000の代わりにX6800 V3を追加することにより、高密度サーバーのラインを再設計しました。 したがって、現在利用できるモデルは2つだけです。

1.X6800-4U、8ノード

2. X8000-44U、80ノード



新しいX6800ソリューションについて詳しく説明します。







ユーザーが使用できるノードには3つのタイプがあります。

1. XH628 V3ストレージノード -高ディスク容量のダブルノード。 12 * 3.5 "ディスク+ 2 * 2.5"ディスクを含むホットスワップバスケットがバスケットとは別にインストールされています。

2. XH622 V3 HPC（アプリケーションアクセラレーションとも呼ばれる）ノードは、2つのフルサイズデュアルスロットPCIe（たとえば、NVIDIA GPUまたはXeon-Phi）のインストールを可能にするHPCのデュアルノードです。 ;

3. XH620 V3 Compute Node-高性能の単一ノード。 レイアウト密度の増加により、8ノードのソリューションには4Uに16個のE5-2600 V3プロセッサーと128個のDDR4スロットがあります。



現在、同じ構成しか使用できないことに注意してください。 つまり 混合ノードタイプの構成は使用できません。



このソリューションは、HPCクラスターまたはパブリッククラウドを構築するプロジェクトで特に興味深いものになります。



Eシリーズブレードソリューション



まあ、あなたは助けることができませんが、Huawei E9000ブレードソリューションに言及してください。







2012年に市場に登場しましたが、その関連性は失われていません。

シャーシ自体は変更されていません。 新世代のプロセッサの出現により、ブレード自体が更新され、Intel Xeon V3プロセッサが取得されました。







16G FC、10GEカッパー、FCoE、56G FDR IBおよび40 GEスイッチのサポートが追加されました。 ただし、コンバージドスイッチの10GE + 8G FCは依然として最も需要が高いものです。

現在の技術革新に加えて、ファーウェイは潜在的なテクノロジーのサポートを発表しました：IB EDRおよび100 GE。



設定を簡単にするために、ここでスイッチとメザニンの対応マトリックスを提供します。







使用されるブレードの種類に関して、このソリューションはすべての潜在的なケースを閉じることができます。

1. CH121 V3-ハーフ2プロセッサーの高密度ブレード。

2. CH140 V3-超高密度のハーフダブルダブル2プロセッサーブレード。

3. CH220 V3-フルサイズのI / Oブレード。

4. CH222 / CH226 V3-ストレージ容量が増加したフルサイズのブレード。

5. CH240 V2 / CH242 V3は、フルサイズのコンピューティング4プロセッサブレードです。



結局のところ、このような「由緒ある」時代にもかかわらず、このソリューションは、その技術的特性に関して他の技術ベンダーとの競争力を維持していることに注目する価値があります。



フュージョンキューブ



そして最後に、私は完全にエキゾチックで未知の解決策を残しました。

時代に遅れずに、Huaweiは競合他社に遅れずについていくことに決め、独自のハイパーコンバージドFusionCubeソリューションをリリースしました。



ハードウェアベースとして、X6800またはE9000サーバーを使用できます。 仮想化環境は、Huawei独自のFusionComputeハイパーバイザー（Xenベース）です。 FusionStorageはディスク容量の仮想化に使用され、FusionManagerは管理ツールとして使用されます。



FusionCube X6800構成では、スケーリングはシャーシレベルで、FusionCube E9000では-ブレードレベルで行われます。



当社の市場におけるこのソリューションの成功と関連性は非常に複雑です。
