招待者-2年間の仕事

Habréに1か月間、あごひげを生やした新興企業と新興企業のレポートがあります-私もバラライカを共有します。



それは、2年前（プラスまたはマイナス2週間）に開始された私のプロジェクトInviter（招待の交換）についてです。 ただし、2009年11月13日は、Habrの尊敬する聴衆とプロジェクトへのリンクを共有したときに、正式に誕生日と見なすことができます。



インサイダー、私はこの2年間にInventorが旅した道について話し、アドバイスを求め、誕生日を祝って最後までメモを読んだほとんどの患者ユーザーにホリデークッキーを配布したいと思います。



技術



Inviterを思いついたとき、HTML、jQuery、およびSmarty [/ irony]でのみプログラムを[皮肉]し、許容できるインターフェースを描画することができました。 したがって、当時のテクノロジーの選択は、PHP / ZendFramework + MySQL + Apacheのように、私にとって明白で簡単でした。 このサイトはどうにかして1年以上も生き続けて開発されましたが、Railsを徹底的に試してみて、震えることなく私の古いコードを見ることができませんでした（ちなみに、 Kirill MokevninはHabréの Railsでの感染の同様の症状を説明しました ）。 新しいアイデアには新しいアーキテクチャが必要である、つまり書き換えることを意味するということを首尾よく語り、数ヶ月間、主な機能を再現し、同時にデザインを更新しました。 その結果、プロジェクトは現在、多数のNginx / Unicorn + Rails 3.1 / MySQL + HAML / SASS / CoffeeScriptに取り組んでいます。 もちろん、コードはまだ理想からはほど遠いものであり、頭が急になるエラーがありますが、とにかく-それらのデバッグと検索は、何倍も速く、より便利になりました。



プロモーション



このプロジェクトが意図的に宣伝されたとは言いません。 口コミと検索エンジンは、視聴者を引き付ける上で主要な役割を果たし、引き続き果たしていますが、プロモーションの残りの方法には一時的な効果がありました。



最後の段落を書いている間、招待者は、それでもHabréではなくLeprochkaで公開されたことを思い出しました。政府によって投稿はすぐに削除されましたが、最初の大量のフィードバックと聴衆の一部がそこに得られました。 しかし、Habréへの投稿後、インターネットへのリンクが他のオンライン出版物のページに表示されるようになりました。 この一連の投稿で最も重要なのは、ランブラーによる「今日のサイト」の招待者の話でした。その後、サーバーは樽の上に完全に置かれました。



一般的に、聴衆には一定の関心があり、それがプロジェクトを継続する強い動機を与えました。 ある時点で、彼らが言うように、サイトは国内のユーザーによってポップアップされ、プラットフォームは安定性が低く、タスクに対応しているため、英語を話す視聴者のためにローカライズして起動する必要があり、そのおかげで出席者が大幅に増加することがわかりました。 実際には、すべてがそれほどバラ色ではありませんでした。 まず、多言語化の可能性が当初考慮されていなかった（それでも喜びである）サービス自体の翻訳プロセスには、予想よりも数倍の時間がかかりました。 第二に、待望の視聴者の成長はせいぜい12パーセントでした。 理由を言うのは難しいです。 最も可能性が高い-サービスのアイデアを西洋の人々に伝えることはできませんでした。 または、この考えは単に彼女にとって興味深いものではありません。 私たちはHackernewsに書き込み、Diggeのレビューを公開し、TechcranchとMashableの編集者を中傷し、あまり知られていない12のリソースにプロジェクトの説明を投稿しました。作業の結果は新規ユーザーの悲惨な割合でした そのため、「欧米の視聴者とのやり取り」の列で、FAILがマークされ、デバイスでこの方向に置かれました。



その時に私に大きなインスピレーションを与えた別の瞬間がありました-私の10代の夢が実現しました：私は雑誌の編集者にサイトについて書いた] [そして彼らはサイトを知っていて、確かにインバーターについて話す番号を印刷する準備をしているとの応答を得ました書かれています。 そして、あなたはだまされていません！ 雑誌の最後のページの定期的なレビューのコラムには、私の作品のスクリーンショットと簡単な説明が記載されたブロックがありました）。 うわー、どれだけの喜び！ シートを引き裂いて壁のフレームに掛けなかったことに驚いた。 おそらく雑誌を後悔した。 しかし、これはすべての歌詞です。 この段落は、ハッカーに関するものではなく、オフライン活動に関するものです。 そして、オフラインでの活動の結果はこうです。紙のジャーナルにメモを掲載した後、西洋の聴衆を引き付けるためのすべての試みよりも多くの人々が来始めました。



最近、ソーシャルネットワークで実験しています。 たとえば、ユーザーが希望する場合、新しい招待のリクエストは、プロフィールへのリンクとともに、Twitter、Vkontakte、Facebookに自動的に送信されます。 このようなクロスポストの有効性は特に大きくありません。3つのネットワークすべてのうち、週に約100人がリンクを利用しています。 登録前はさらに少なくなります。 しかし、これは面白くありません-どのように登録されているかは面白かったです。 少し前まで、指定されたソーシャルサービスを介して新しいユーザーの承認と登録を結び付けました。 3つのソーシャルネットワークのうち、特定の視聴者が使用することを好むものは何ですか？ だから、怠け者の場合は、Twitterで登録し、トークンでサイトを信頼するかどうかを10回考えます。 そして、ユーザーは落ち着いてVKontakteを使用します。 そのような面白い事実。
