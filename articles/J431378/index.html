キューを使用する



Nucleus SEでは、ビルドフェーズ中にキューが定義されます。 アプリケーションは最大16個のキューを持つことができます。 アプリケーションにキューがない場合、キューに関連するデータ構造もサービスコードもアプリケーションに含まれません。



キューとは、単一のADDR要素に十分な大きさのメモリ内の領域のセットであり、複数のタスクが使用できるようにアクセスを制御します。 タスクは、すべての領域がいっぱいになるまでキューにデータを書き込むことができます。 タスクはキューからデータを読み取ることができます。通常、データはFIFO（先入れ先出し）ベースで受信されます。 混雑したキューにデータを書き込んだり、空のキューからデータを読み取ろうとすると、選択したAPI呼び出しパラメーターとNucleus SEの設定によっては、エラーまたはタスクの一時停止が発生する場合があります。



キューとデータリンク



Nucleus SEはデータチャネルをサポートしています。これは、以前の記事（＃5）でも言及されており、以下のいずれかで詳細に説明されます。 キューとチャネルの主な違いは、メッセージのサイズです。 キューには、 ADDRタイプの単一変数で構成されるメッセージが含まれます（通常、これらはポインターです）。 チャンネルには、アプリケーション内の各チャンネルに個別の任意のサイズのメッセージが含まれ、パラメーター設定時に割り当てられます。



キュー設定



キューの数



ほとんどのNucleus SEオブジェクトと同様に、キュー構成は主にnuse_config.hファイルの#defineディレクティブによって制御されます。 主なパラメーターはNUSE_QUEUE_NUMBERで、アプリケーションで構成されているキューの数を決定します。 デフォルト値はゼロ（つまり、アプリケーションにキューがありません）で、最大16の値を取ることができます。誤った値はコンパイル中にエラーになり、 nuse_config_check.hファイル（ nuse_config.cファイルに含まれコンパイルされます）それとともに）、 ＃errorディレクティブをトリガーします。



ゼロ以外の値を選択すると、キューの主要なアクティベータとして機能します。 このパラメーターは、データ構造を定義するときに使用され、そのサイズはその値に依存します（これについては次の記事で詳しく説明します）。 さらに、ゼロ以外の値はAPI設定をアクティブにします。



API呼び出しをアクティブにする



Nucleus SEの各API関数（ユーティリティ呼び出し）には、 nuse_config.hに #defineディレクティブをアクティブ化する機能があります。 キューの場合、これらのディレクティブは次のとおりです。 



NUSE_QUEUE_SEND NUSE_QUEUE_RECEIVE NUSE_QUEUE_JAM NUSE_QUEUE_RESET NUSE_QUEUE_INFORMATION NUSE_QUEUE_COUNT


デフォルトでは、これらはFALSEに設定されているため、すべてのサービス呼び出しが無効になり、それらを実装するコードの組み込みがブロックされます。 アプリケーションでキューを構成するには、必要なAPI呼び出しを選択してTRUEに設定する必要があります。



以下は、 nuse_config.hファイルからのコードスニペットです。 



 #define NUSE_QUEUE_NUMBER 0 /* Number of queues in the system - 0-16 */ /* Service call enablers */ #define NUSE_QUEUE_SEND FALSE #define NUSE_QUEUE_RECEIVE FALSE #define NUSE_QUEUE_JAM FALSE #define NUSE_QUEUE_RESET FALSE #define NUSE_QUEUE_INFORMATION FALSE #define NUSE_QUEUE_COUNT FALSE


キューAPIの機能がアクティブになっているが、アプリケーションにキューがない場合（常に有効になっているNUSE_Queue_Count（）を除く）、コンパイルエラーが表示されます。 アクティブ化されていないAPI呼び出しをコードで使用すると、実装コードがアプリケーションに含まれていないため、レイアウトエラーが発生します。



コールキューイング



Nucleus RTOSは、次の機能を提供する10個のキューサービスコールをサポートしています。



  • メッセージのキューイング。 Nucleus SEはNUSE_Queue_Send（）関数に実装されています。
  • キューからのメッセージを受け入れます。 Nucleus SEはNUSE_Queue_Receive（）関数を実装しています。
  • キューの先頭への投稿。 Nucleus SEはNUSE_Queue_Jam（）を実装しています。
  • 一時停止されているすべてのタスクを解放して、キューを未使用状態に復元する（リセット） Nucleus SEでは、 NUSE_Queue_Reset（）で実装されています。
  • 特定のキューに関する情報を提供します。 Nucleus SEはNUSE_Queue_Information（）に実装されています。
  • アプリケーションで現在構成されているキューの数を返します。 Nucleus SEでは、 NUSE_Queue_Count（）で実装されています。
  • アプリケーションに新しいキューを追加する（キューを作成する）。 Nucleus SEは実装されていません。
  • アプリケーションからキューを削除します。 Nucleus SEは実装されていません。
  • アプリケーション内のすべてのキューへのポインターを返します。 Nucleus SEは実装されていません。
  • キューで中断されているすべてのタスクにメッセージを送信する（ブロードキャスト）。 Nucleus SEは実装されていません。


これらの各オーバーヘッドコールの実装について、以下で詳しく説明します。



キューからの書き込みおよび読み取りのサービス呼び出し



キューで実行される基本的な操作は、書き込み（キューメッセージと呼ばれることもあります）および読み取り（メッセージの受信とも呼ばれます）です。 キューの先頭に書き込むこともできます（ジャミング）。 Nucleus RTOSおよびNucleus SEは、これらの操作のための3つの基本的なAPI呼び出しを提供します。これについては後で説明します。



待ち行列



キューに書き込むNucleus RTOS APIユーティリティコールは非常に柔軟であり、暗黙的に、または操作をすぐに完了できない場合（たとえば、フルキューに書き込む場合）に特定のタイムアウトでタスクを一時停止できます。 Nucleus SEは同じ機能を提供しますが、タスクの一時停止はオプションであり、タイムアウトは実装されていません。



Nucleus RTOSでのメッセージのキューイング



サービスコールのプロトタイプ：



STATUS NU_Send_To_Queue（NU_QUEUE *キュー、VOID *メッセージ、UNSIGNEDサイズ、UNSIGNEDサスペンド）;



パラメータ：



queue-ユーザーが提供するキュー制御ブロックへのポインター。

message-送信されるメッセージへのポインター。

size-メッセージ内のUNSIGNEDデータ要素の数。 キューが可変長のメッセージをサポートしている場合、このパラメーターはメッセージのサイズと同じか、キューでサポートされているメッセージのサイズより小さくなければなりません。 キューが固定サイズのメッセージをサポートする場合、このパラメーターはキューでサポートされるメッセージのサイズと正確に一致する必要があります。

suspend-タスクの一時停止の指定。NU_NO_SUSPENDまたはNU_SUSPENDの値、またはタイムアウト値を取ることができます。



戻り値：



NU_SUCCESS-呼び出しは正常に完了しました。

NU_INVALID_QUEUE-無効なキューポインター。

NU_INVALID_POINTER-メッセージへのNULLポインター（ NULL ）;

NU_INVALID_SIZE-メッセージサイズは、キューでサポートされているメッセージサイズと互換性がありません。

NU_INVALID_SUSPEND-中断はタスクに関係のないスレッドから実行されました。

NU_QUEUE_FULL-キューがいっぱいで、中断が指定されていません。

NU_TIMEOUT-指定されたタイムアウトでタスクが中断された後でもキューはいっぱいです。

NU_QUEUE_DELETED-タスクの中断中にキューが削除されました。

NU_QUEUE_RESET-タスクの中断中にキューがリセットされました。

サービスコールのプロトタイプ：



STATUS NUSE_Queue_Send（NUSE_QUEUEキュー、ADDR *メッセージ、U8サスペンド）;



パラメータ：



queue-キューインデックス（ID）;

message-送信されるメッセージへのポインター;これはADDRタイプの1つの変数です。

suspend-タスクを一時停止するための指定; NUSE_NO_SUSPENDまたはNUSE_SUSPENDの値を取ることができます。



戻り値：



NUSE_SUCCESS-呼び出しは正常に完了しました。

NUSE_INVALID_QUEUE-無効なキューインデックス。

NUSE_INVALID_POINTER-メッセージへのNULLポインター（ NULL ）;

NUSE_INVALID_SUSPEND-タスクに関連付けられていないスレッドからタスクを一時停止する試み、またはAPIサービス呼び出しがタスクをブロックするために無効になっている場合。

NUSE_QUEUE_FULL-キューがいっぱいで、中断が指定されていません。

NUSE_QUEUE_WAS_RESET-タスクの中断中にキューがリセットされました。



Nucleus SEでキューイングを実装する

API関数コードNUSE_Queue_Send（）のバージョン（パラメーターを確認した後）は、タスクロックのサポートがアクティブかどうかに応じて、条件付きコンパイルを使用して選択されます。 両方のオプションを検討します。



タスクのロックが有効になっていない場合、このサービス呼び出しのコードは非常に簡単です。 



 if (NUSE_Queue_Items[queue] == NUSE_Queue_Size[queue]) /* queue full */ { return_value = NUSE_QUEUE_FULL; } else /* queue element available */ { NUSE_Queue_Data[queue][NUSE_Queue_Head[queue]++] = *message; if (NUSE_Queue_Head[queue] == NUSE_Queue_Size[queue]) { NUSE_Queue_Head[queue] = 0; } NUSE_Queue_Items[queue]++; return_value = NUSE_SUCCESS; }


この関数は、キューに空き領域があるかどうかを確認するだけで、 NUSE_Queue_Head []インデックスを使用してメッセージをキューデータ領域に格納します。



タスクのロックが有効になっている場合、コードはより複雑になります。 



 do { if (NUSE_Queue_Items[queue] == NUSE_Queue_Size[queue]) /* queue full */ { if (suspend == NUSE_NO_SUSPEND) { return_value = NUSE_QUEUE_FULL; } else { /* block task */ NUSE_Queue_Blocking_Count[queue]++; NUSE_Suspend_Task(NUSE_Task_Active, (queue << 4) | NUSE_QUEUE_SUSPEND); return_value = NUSE_Task_Blocking_Return[NUSE_Task_Active]; if (return_value != NUSE_SUCCESS) { suspend = NUSE_NO_SUSPEND; } } } else { /* queue element available */ NUSE_Queue_Data[queue][NUSE_Queue_Head[queue]++] = *message; if (NUSE_Queue_Head[queue] == NUSE_Queue_Size[queue]) { NUSE_Queue_Head[queue] = 0; } NUSE_Queue_Items[queue]++; if (NUSE_Queue_Blocking_Count[queue] != 0) { U8 index; /* check whether a task is blocked on this queue */ NUSE_Queue_Blocking_Count[queue]--; for (index=0; index<NUSE_TASK_NUMBER; index++) { if ((LONIB(NUSE_Task_Status[index]) == NUSE_QUEUE_SUSPEND) && (HINIB(NUSE_Task_Status[index]) == queue)) { NUSE_Task_Blocking_Return[index] = NUSE_SUCCESS; NUSE_Wake_Task(index); break; } } } return_value = NUSE_SUCCESS; suspend = NUSE_NO_SUSPEND; } } while (suspend == NUSE_SUSPEND);


いくつかの説明が役立つ場合があります。



コードはdo ... whileループで囲まれています。このループは、タスクのポーズパラメーターがNUSE_SUSPENDの間に実行されます。



キューがいっぱいでサスペンドがNUSE_NO_SUSPENDの場合、API呼び出しはNUSE_QUEUE_FULLで終了します 。 中断パラメーターがNUSE_SUSPENDの場合、タスクは一時停止します。 完了時（タスクの再開時）、戻り値がNUSE_SUCCESSの場合、つまり、メッセージが読み取られたために（キューがリセットされたためではなく）タスクが再開された場合、コードはループの先頭に戻ります。

キューがいっぱいでない場合、提供されたメッセージはキューのデータ領域のNUSE_Queue_Head []インデックスを使用して保存されます。 キューに一時停止されたタスクがあるかどうかを確認します（保留中のメッセージ）。 そのようなタスクがある場合、最初のタスクが再開されます。 中断変数はNUSE_NO_SUSPENDに設定され、API呼び出しは値NUSE_SUCCESSで終了します。



キューからの読み取り



Nucleus RTOS APIユーティリティのキューからの読み取り呼び出しは非常に柔軟で、暗黙的に、または操作をすぐに完了できない場合（空のキューから読み取ろうとする場合など）に特定のタイムアウトでタスクを一時停止できます。 Nucleus SEは同じ機能を提供しますが、タスクの一時停止はオプションであり、タイムアウトは実装されていません。

Nucleus SEでのキューメッセージの実装

タスクロックのサポートがアクティブかどうかに応じて、条件付きコンパイルを使用して、API関数コードバリアントNUSE_Queue_Receive（） （パラメーターのチェック後）が選択されます。 両方のオプションを検討してください。



ロックのサポートが有効になっている場合、このAPI呼び出しのコードは非常に簡単です。 



 if (NUSE_Queue_Items[queue] == 0) /* queue empty */ { return_value = NUSE_QUEUE_EMPTY; } else { /* message available */ *message = NUSE_Queue_Data[queue][NUSE_Queue_Tail[queue]++]; if (NUSE_Queue_Tail[queue] == NUSE_Queue_Size[queue]) { NUSE_Queue_Tail[queue] = 0; } NUSE_Queue_Items[queue]--; return_value = NUSE_SUCCESS; }


この関数は、キューにメッセージがあるかどうかを確認し、 NUSE_Queue_Tail []インデックスを使用してキューからメッセージを取得し、メッセージへのポインターを使用してデータを返します。



タスクのロックが有効になっている場合、コードはより複雑になります。 



 do { if (NUSE_Queue_Items[queue] == 0) /* queue empty */ { if (suspend == NUSE_NO_SUSPEND) { return_value = NUSE_QUEUE_EMPTY; } else { /* block task */ NUSE_Queue_Blocking_Count[queue]++; NUSE_Suspend_Task(NUSE_Task_Active, (queue << 4) | NUSE_QUEUE_SUSPEND); return_value = NUSE_Task_Blocking_Return[NUSE_Task_Active]; if (return_value != NUSE_SUCCESS) { suspend = NUSE_NO_SUSPEND; } } } else { /* message available */ *message = NUSE_Queue_Data[queue][NUSE_Queue_Tail[queue]++]; if (NUSE_Queue_Tail[queue] == NUSE_Queue_Size[queue]) { NUSE_Queue_Tail[queue] = 0; } NUSE_Queue_Items[queue]--; if (NUSE_Queue_Blocking_Count[queue] != 0) { U8 index; /* check whether a task is blocked */ /* on this queue */ NUSE_Queue_Blocking_Count[queue]--; for (index=0; index<NUSE_TASK_NUMBER; index++) { if ((LONIB(NUSE_Task_Status[index]) == NUSE_QUEUE_SUSPEND) && (HINIB(NUSE_Task_Status[index]) == queue)) { NUSE_Task_Blocking_Return[index] = NUSE_SUCCESS; NUSE_Wake_Task(index); break; } } } return_value = NUSE_SUCCESS; suspend = NUSE_NO_SUSPEND; } } while (suspend == NUSE_SUSPEND);


いくつかの説明が役立ちます。



コードはdo ... whileループで囲まれています。このループは、タスクのポーズパラメーターがNUSE_SUSPENDの間に実行されます。



キューが空でサスペンドがNUSE_NO_SUSPENDの場合、API呼び出しはNUSE_QUEUE_EMPTYで終了します 。 中断パラメーターがNUSE_SUSPENDの場合、タスクは一時停止します。 完了時に（つまり、タスクが再開したとき）、戻り値がNUSE_SUCCESSの場合、つまり、メッセージが送信されたために（キューがリセットされたためではなく）タスクが再開された場合、コードはループの先頭に戻ります。



キューにメッセージが含まれている場合、保存されたメッセージはNUSE_Queue_Tail []インデックスを使用して返されます。 このキューに一時停止（送信待ち）タスクがあるかどうかを確認します。 そのようなタスクがある場合、最初のタスクが再開されます。 中断変数はNUSE_NO_SUSPENDに設定され、API呼び出しはコードNUSE_SUCCESSで終了します。



キューの先頭に書き込む

message-送信されるメッセージへのポインター。

size-メッセージ内のUNSIGNEDデータ要素の数。 キューが可変長メッセージをサポートしている場合、このパラメーターはメッセージサイズと同じか、キューでサポートされているメッセージサイズより小さい必要があります。 キューが固定長のメッセージをサポートする場合、このパラメーターはキューでサポートされるメッセージのサイズと正確に一致する必要があります。

戻り値：



NU_SUCCESS-呼び出しは正常に完了しました。

NU_INVALID_QUEUE-無効なキューポインター。

NU_INVALID_SIZE-メッセージサイズは、キューでサポートされているメッセージサイズと互換性がありません。

NU_INVALID_SUSPEND-非タスクフローから一時停止を試みます

NU_QUEUE_FULL-キューがいっぱいで、中断が指定されていません。

NU_TIMEOUT-特定のタイムアウトのためにタスクが中断された後でも、キューはいっぱいです。

NU_QUEUE_DELETED-タスクの中断中にキューが削除されました。

NU_QUEUE_RESET-タスクの中断中にキューがリセットされました。



Nucleus SEのキューの先頭にメッセージを書き込む呼び出し

このAPIコールは、Nucleus RTOS APIのコア機能をサポートしています。



サービスコールのプロトタイプ：



STATUS NUSE_Queue_Jam（NUSE_QUEUEキュー、ADDR *メッセージ、U8サスペンド）;



パラメータ：



queue-キューインデックス（ID）;

message-メッセージへのポインターは、タイプADDRの 1つの変数です。

suspend-タスクの中断の指定 。値NUSE_NO_SUSPENDまたはNUSE_SUSPENDを取ることができます。



戻り値：



NUSE_SUCCESS-呼び出しは正常に完了しました。

NUSE_INVALID_QUEUE-無効なキューインデックス。

NUSE_INVALID_POINTER-メッセージへのNULLポインター（ NULL ）;

NUSE_INVALID_SUSPEND-タスクに関連付けられていないスレッド、またはタスクブロックのサポートが無効になっているスレッドからタスクを中断しようとする試み。

NUSE_QUEUE_FULL-キューがいっぱいで、中断が指定されていません。

NUSE_QUEUE_WAS_RESET-タスクの中断中にキューがリセットされました。
