この記事に関する経験者のコメントを期待して、書かれたものはすべて長い間知られており、新しいことは何も言われていないということを期待して、私はこれがモバイル市場での標準的な手法とプロモーション方法についての抽象的な記事ではないと急いで言います。 これらの方法の使用方法と最終的に得られた結果に関する記事。 これにはすべて、実数とグラフが伴います。 このような情報を共有する準備ができています。 経験のある人でも読んで自分の経験と比較したり、おそらく記事へのコメントで議論したりすることに興味があると思います。



ストーリーは、十分な数の大規模な競合他社（レポート作成とデータ視覚化のための.NETコンポーネント）により開発の主な方向に見込みがないため、プログラマーのチームがまったく異なる方向に進むことを決定したことです-モバイルアプリケーションの開発。 幸いなことに、以前から蓄積された開発経験を使用できるツールがあり、ゼロから始めることはできません。



あなたは、マーケティング部門とモバイルアプリケーションのプロモーションについて言うことはできません。 開発者向けのコンポーネントのプロモーションは、モバイルアプリケーションのマーケティングとは間違いなく異なります。



この方向の主力製品は、Androidデバイス用のデータベースを作成および管理するためのアプリケーションになりました。

最も重要なことから始めましょう-ASO（App Store Optimization）-ストアの最適化。



アプリケーションの名前と説明はプロモーション戦略の非常に重要な部分であり、ストアの検索結果はその品質に依存します。 これは長期的な投資です。外部ソースからのトラフィックは時間の経過とともに減少するため（グラフで確認できます）-これが時間であり、ほとんどのユーザーはストアで直接必要なアプリケーションを探します-これらは2つです。



アプリケーション名



必須アイテムはタイトルのキーワードです。 また、キーワードを先頭に近づけるほど、検索結果のアプリケーションが高くなります。 しかし、現実は驚きをもたらす可能性があることを忘れないでください。 ユーザーがアプリケーションまたはキーワードで類似したものを検索していると仮定します。 しかし、実際には、人々は何か他のものを必要とするか、ストアで検索があなたのアイデアからはほど遠いアプリケーションを提供することがわかります。



そのため、アプリケーションの名前を確認して実行可能性を確認し、適切なユーザー（インストールして購入するユーザー）に焦点を当てることをお勧めします。ストア検索でキーワードを入力し、Googleが検索するアプリケーションを確認します。 被写体に近い場合、すべてが大丈夫です。 いいえ-新しい名前を考え出します。



私たちは、実験を通して私たち自身の経験からこれを見つけました。 私たちのアプリケーションは、Mobi DBからMobiDB Database Managerに移行し、その後MobiDB Database Designerに移行しました。 名前が変更された後、インストールの数が増加しました。これは、アプリケーションがSERPで上昇し、ターゲットオーディエンスによりターゲットを絞ったためです。 名前が示すように、キーワードは「データベース」ですが、名前の最初のバージョンではまったくありませんでした。 Googleは、ストア内のアプリケーションをWebページとしてインデックス化します。アプリケーション名= Webページのタイトル。 名前にキーワードはありません-アプリケーションは検索に表示されません。 だから、私たちと一緒でした。 そのため、名前をMobiDB Database Managerに変更しましたが、これはアプリケーションの本質を正確に伝えていなかったため、あまり良い名前ではありませんでした（完成したデータベースの管理ではなく、データの作成、整理、およびデータベースの維持を目的としています）。 もう一度実験することにしました-MobiDB Database Designerは、私たちにとっては、アプリケーションの本質を非常に明確に伝えています。 私たちの仮定は、インストール数の増加によって確認されました。







約1か月で、インストール数はほぼ10倍に増加しました。 そのため、アプリケーションの名前を賢明に調べてください。理解できないクリエイティブ名は商業的な結果にはなりません。



アプリケーションの説明



最大4000文字の文字数が許可されていますが、Googleはインデックスを作成し、さらに重要なこととして、最初の350文字のみをビジターに表示します（モバイルストアではさらに少ない）。 これらの350文字では、適切なインデックス作成に必要なキーワードの数に合わせて管理し、訪問者が読みやすいようにし、「詳細」ボタンをクリックするか、「インストール」ボタンをクリックするように強制する必要があります。



ストア内のアプリケーションの位置を改善するために、説明を数回（つまり、最初の段落）変更しました。



知っている人はほとんどいませんが、ストア内のアプリケーションの説明をフォーマットして、最も重要なポイントと機会に注意を向けることができます。 Playストアでサポートされているタグのリスト：

http://stackoverflow.com/questions/11071127/google-play-app-description-formatting



説明に注意してください。説明に含まれるキーワードの濃度が高すぎると、スパムと見なされ、ストアからアプリケーションが削除されます（2つのアプリケーションでは、幸いにもすべてを時間通りに修正しました）。 Playストアにはアプリケーションのタグを含むフィールドはありません。これをアプリケーションの説明に追加することはお勧めしません-スパムと見なすこともできます。

アプリケーションのローカライズ



アプリケーションのオーディエンスを増やす良い方法は、ストア内のアプリケーション自体とその説明をローカライズすることです。 この順序で望ましいです。 Googleは、アプリケーション自体をローカライズして、アプリケーションの言語以外の言語で説明を追加することを主張していません。



ユーザーは、自分の言語で説明を見て、アプリケーション自体が同じ言語であることを期待します。 また、アプリケーションをインストールして実行した後、たとえばフランス語ではなく英語で表示される場合の失望は明らかです。 そして、英語を話さないユーザーからの否定的なレビューの出現は明らかです。



この影響を回避するには、まずアプリケーションをローカライズします。 プロフェッショナルサービスのコストは非常に高くなります。 顧客に支援を求めることで、コストを大幅に削減できます。 Androidデバイスのユーザーは非常に反応がよく、アプリケーションの有料版またはAmazonの証明書を喜んでお手伝いします。



プロフェッショナルサービスのサービスとネイティブスピーカーのサービスの両方を使用しました。 ユーザーのローカライズ後、プロのサービスで校正を実行することは依然として優れていますが、これはすでに翻訳を最初から注文するよりもはるかに安価です。

サテライトアプリケーションの作成



プロモーションのもう1つの方法は、直接競合することはないが、まだリンクが含まれているメインアプリケーションをプロモーションする関連アプリケーションを作成することです。



当社の主力アプリケーションは、任意のユーザーデータベースを作成するように設計されています。 それを促進するために、特定のカテゴリのユーザーを対象とした、より専門的なアプリケーションを作成しました。 それらの中で最も成功しているのは、在庫管理とプロジェクト管理です。 データベースを使用する特殊なケース。 アプリケーションは無料ですが、すべての機能が利用できるわけではありません。 たとえば、データベース構造を変更したり、同期を使用したりすることはできません。 このような機会にアクセスするために、フラッグシップアプリケーションを購入することを提案します。



結論 ：このアプローチは機能し、定期的にそのようなアプリケーションから新しい顧客を獲得します。 プラスは、より多くの、より具体的なオーディエンスを対象としていることです。マイナスは、これらのアプリケーションをストアで宣伝して、それらを良好にする必要があることです。



店外プロモーション



ストアの外部では、アプリケーションも昇格する必要がありますが、素晴らしい結果は期待できません。 2014年に実施されたMobileDevHQの調査によると、スマートフォンユーザーの総数のわずか2％が店舗外のアプリケーションを探しています。







しかし、これはあなたがそれをする必要がないという意味ではありません。 Google従業員自身によると、ストア内のページへのリンクの表示も、ストア内のアプリケーションの位置にプラスの影響を及ぼします。 ただし、リンクは正しくなければなりません-少なくとも「ここからアプリケーションをダウンロードできます」だけでなく、キーワードが含まれるアンカーが必要です。



店舗外でのプロモーションのもう1つの重要なポイントは、Googleアナリティクス（ https://developers.google.com/analytics/?hl=en ）または他の同様のサービスを接続して、外部ソースからのコンバージョンを追跡し、努力の効果を評価できるようにすることです。 Googleアナリティクスで外部ソースからのクリックに関する統計を表示するには、リンクが特定のタイプである必要があります。そうでない場合、リンクは直接/なしと見なされます。

レビュー



現在、インターネットには、Androidオペレーティングシステムとそれに関連するすべて（デバイス、アクセサリ、アプリケーション、ウィジェットなど）専用の多くのリソースがあります。 このトピックについて書いている多くのブロガーもいます。 このようなリソースに関する出版物を入手するのは非常に簡単なはずです。 しかし、これはまったく真実ではありません。 公開されるアプリケーションの数は増え続けており、すべての開発者は自分のアプリケーションについて書きたいと思っています。 オンラインリソースのジャーナリストと編集者は、レビューの公開を求めるリクエストの流れにjustれています。 特に無料で公開する場合は、アプリケーションに注意を引くのに多大な労力が必要です。 レビューリクエストを作成するためのヒント：



1.パーソナライズする必要があります。 ジャーナリストに直接書いて、これが大量スパムではないことを明確にしてください。 このジャーナリストによる同様のテーマの記事を参照してください。 彼は本当の専門家であり、彼の意見はあなたにとって非常に興味深いものだと書いてください（誰もがお世辞が大好きです）。

2.アプリケーションがめちゃくちゃ便利なものだけでなく、アプリケーションのレビューがジャーナリスト/ブロガー自身にもたらすメリット（興味深いトピック、より多くの購読者、質の高いコンテンツなど）についても記述してください。

3.ストアからアプリケーションの説明全体をレターに挿入しないでください。 誰も2ページの手紙を読みません。 アプリケーションの目的、競合他社との違い、ストアへのリンク、主要な機能の簡潔なリストを、2、3の文で説明するとよいでしょう。

4.レターの総量を1000文字に制限します。これにより、少なくともメッセージが読まれる可能性を高めることができます。

5.時間をかけてリクエストを読んでくださった読者に感謝します。

有料レビュー



ストアでのレビューは非常に重要です。まず、検索結果のアプリケーションは、評価（ceteris paribus）に従って表示されます：より高い評価-ページの先頭に近い。第二に、上位3つ以上の評価でアプリケーションをインストールしますか？もちろん、これは可能ですが、代替手段がない場合のみです。



したがって、レーティングの不正行為を提供するサービスの外観は理解できますが、それらを使用することを敢えてしませんでした。これはストアのポリシーに対する明らかな矛盾であり、結果としてストアからアプリケーションが100％削除されます。
