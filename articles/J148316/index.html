モニターとキーボードのない物理的にアクセス可能なシステムにDebianをインストールする

長い間、私は手にラップトップしか持っていなかったが、つい最近システムユニットを手に入れた。 モニターとキーボード、言い換えれば、ヘッドレスシステムなし。 したがって、 タスクが発生しました：



Debianを物理的にアクセス可能なマシンにインストールします。これにはモニターやキーボードを接続する能力はありませんが、特定のソフトウェアとハ​​ードウェアリソースのセットがあります。



すなわち：
  1. BIOSセットアップ「エラーなし」または「すべて、キーボードなし」でPOST順序がすでに設定されているターゲットマシン（そうでない場合、手順は少なくとも何かのダウンロードを開始することさえできなくなります*）、最初に1つのsataデバイスから起動し、次に別のデバイスから起動します（ブート順序の他の詳細は重要ではありませんでした）
  2. ラップトップ
  3. ローカルネットワーク（私の場合、ターゲットマシンへの有線接続とワイヤレス-ラップトップを使用したWi-Fiルーター。ただし、詳細は重要ではありません）
  4. インターネットアクセス（安価でできれば高速）
  5. ハーフデッド2.5 "折りたたみ式外部ハードドライブ、内部SATAインターフェイスおよび外部USB


Debianがターゲットマシンの内部ハードドライブにインストールされている場合、タスクは完了したとみなすことができ、後者はユーザーの介入なしでSSHアクセシビリティをオンにします。



ソリューションスキーム



この問題の最適な解決策は、次の一連のアクションでした。

  1. そこにDebianをインストールするのに十分な外付けハードドライブスペースを再活性化する
  2. ラップトップで、Oracle VirtualBox仮想化パッケージのVBoxManageツールを使用してusb接続の外部ハードドライブをvmdkファイルにバインドし、Debian外部ハードドライブにインストールして構成します。
  3. 外付けハードドライブを分解し、最初にダウンロードキューでsataインターフェースを介してターゲットマシンに接続し、マシンを起動してSSH経由でラップトップから接続します
  4. 2.5「ターゲットマシン3.5のネイティブドライブ」からシステムをクローンし、grub2を再構成して、SSH経由でラップトップからターゲットマシンを制御します。


代替案


一般的に、この問題には多くの解決策があります。 提案されたスキームの代わりに、私の代わりに、ほとんどの場合、経験と準備が整った別のユーザーが次の解決策のいずれかを選択できます。

  1. 既存のDebianディストリビューションを変更して、CD / DVDから起動するときにユーザーに質問することなく、ターゲットマシンのハードディスクをパーティション分割するために必要な方法を含め、システムを自動的にインストールおよび設定します。 これを行うには、インストーラーのすべての要求に対する回答を含む特別なpreseed-fileを作成し、すべてが完全に考慮されていることを期待する必要があります。 詳細については、 公式ドキュメントを参照してください。
  2. ラップトップ上のDHCPサーバーを上げる（またはルーターのサービスを使用する）、Webサーバーを使用してpreseedファイルを公開し、tftpサーバーを使用してネットワーク経由で初期pxelinux.0ブートコードをターゲットマシンに転送します-つまり、PXEブートを使用します（ BIOSセットアップでネットワークブートサポートを有効にする必要がある理由）。 これは、私の状況の重要な戦略的利点であるターゲットマシンへの物理的なアクセスを使用しない最も強力なアプローチです。
  3. 適切なlive-cd Linuxディストリビューションを見つけるか、既存のディストリビューションを変更して（たとえば、simple-cddユーティリティなどを使用）、単一の質問なしで起動し、SSHサーバーを自動的に起動してから、Debianディストリビューションを手動でインストールします（このプロセスについては、 ドキュメント ）
  4. ただし、他の方法を使用する場合、ほとんどの場合、説明されている方法の組み合わせになるか、使用可能なリソースを超えます。たとえば、3.5インチディスク用のusb-sataアダプターや、少なくともアダプターとしての些細なことで、 SATAの「パパ」（3.5インチドライブはターゲットマシンのPSUから電力を供給し、2.5インチ外部ドライブからコントローラーを使用してUSB経由でラップトップに接続するか、スリムDVDの代わりに3.5インチドライブをラップトップに直接接続することもできます。 -ドライブ。 それは可能である - しかし、そのようなアダプタがある場合のみ）


もう少し


ここで、私を成功に導いたステップをより詳細に説明し、同様の状況であなたを導くことを望みます。



第一段階


まず、外部ドライブ（容量160 GB、ドライブ全体に1つのパーティション、ext3ファイルシステム）をLinuxがインストールされたラップトップに接続し、後者を自動修正してエラーをチェックします（以下では、＃プレフィックスがルートとして実行されるコマンドを検討します）。また、外付けハードドライブはラップトップによってsdcデバイスとして識別されると想定しています）。 

# fsck -f -y -c /dev/sdc1


-f-エラーが含まれていないとマークされている場合でも、ディスクをチェックします

-y-すべての質問に自動的に「はい」と答えます。 つまり、すべてのエラーを修正するだけです

-c-破損したディスクブロックを検索してマークします。 これらの目的のために、特別なユーティリティbadblocksがありますが、このオプションを使用すると、レポートfsckが自動的に送信されます。

ドライブを指定するのではなく、ドライブ上の特定のパーティションを指定することに注意してください。

私の場合、膨大な数のエラーが原因でチェックが20％で「死亡」したため、ディスクの先頭に20ギガバイトのパーティションを作成するという単純な決定を下しました（Debianをインストールするのに十分です）。 ディスクのパーティションを再分割するには、cfdiskユーティリティを使用すると便利です。 

 # cfdisk /dev/sdc


対話モードでは、必要に応じてディスクを構成するだけです。ディスクの先頭に20 GBのプライマリパーティションを作成し、残りのスペースを空のままにします（また、事前にブート可能としてマークします。プログラムを終了する前に変更をディスクに書き込むことを忘れないでください）。 次に、その上にファイルシステムを作成します。 

 # mkfs.ext3 /dev/sdc1


また、すべてが正常であることを確認します。 

 # fsck -f -y -c /dev/sdc1




第二段階


VirtualBoxが既にインストールされているとします。 次に、ディスクのvmdk-imageを作成します。これにより、物理ディスク上でのすべての操作が実際に表示されます（たとえば、〜/ VirtualBox \ VMs \ desktopなどの使いやすいディレクトリにいると仮定します）。 

 # vboxmanage internalcommands createrawvmdk -filename usb-hdd160.vmdk -rawdisk /dev/sdc


次に、「デスクトップ」と呼ばれる新しい仮想マシンを作成します。 

 # vboxmanage createvm --name desktop --ostype Debian --register


適切な量​​のRAMを割り当てます。 

 # vboxmanage modifyvm desktop --memory 512


ハードドライブとCD / DVDドライブを接続するためのコントローラーを装備しています。 

 # vboxmanage storagectl desktop --name "sata" --add sata


外付けハードドライブを接続します。 

 # vboxmanage storageattach desktop --storagectl "sata" --port 0 --type hdd --medium ./usb-hdd160.vmdk


ドライブを切断するには（これは複数回役に立ちます）、メディアとして「なし」を選択する必要があります。 

 # vboxmanage storageattach desktop --storagectl "sata" --port 0 --type hdd --medium none


ここで、Debianをインストールする理由を考えます。 通常のインターネットチャネルがある場合、「小さなインストールイメージ」を選択しました。オプションで、フルサイズのCDまたはDVDをダウンロードすることもできます。 詳細はこちら 。 また、仮想マシンにインストールするため、i386バージョンに制限する必要があります-VirtualBoxは、ハードウェア仮想化VT-xまたはAMD-Vをサポートするホスト上の64ビットゲストのみをサポートします-これは、最も安価なプロセッサーだけが誇ることができます。 そのため、Debian isoイメージを現在のディレクトリ（debian-6.0.5-i386-netinst.isoなど）に配置します。 次に、ハードドライブを接続するイメージと類似性で、それも接続します。 

 # vboxmanage storageattach desktop --storagectl "sata" --port 1 --type dvddrive --medium ./debian-6.0.5-i386-netinst.iso


車を始動できます： 

 # vboxmanage startvm desktop


（これらのアクションの一部はVirtualBoxグラフィカルシェルからも実行できます。起動するときのみ、デフォルトでは1人のユーザーに対してマシンのみが作成されることを忘れないでください。したがって、rootとしてグラフィカルシェルを実行することもできます。ハードドライブを操作するには特権アクセスが必要なので、sudoを使用してください）

ここで、インストールのすべての段階を実行し、rootとユーザーのパスワードを指定する必要があります。また、インストールテンプレートからSSHサーバーを選択することが非常に重要です。 ディスクの非常に不安定な状態のため、ディスクを分割するとき、既存のパーティションを/としてマウントし、スワップパーティションを選択しなかったため、アクションを最小限に抑えました。ターゲットマシンに既に接続できます。 インストールが完了した後でも、キーボードとモニターからアクセスできる限り、システムで作業し、好みのものを設定し、isoイメージを切断するなどしてすべてが実際にロードされていることを確認できます。



第三段階


3番目の段階で必要なのは、外部ドライブをターゲットマシンに接続し、それをオンにすることだけです。 ルーターはこの段階で助けてくれました。ステータスページでマシンのIPを見て、このアドレスのラップトップからSSHセッションを開いたからです。 

 # ssh root@192.168.0.103


ほとんどの場合、マシンのアドレスは異なります。



第4段階


すべてがうまくいけば、システムを外部ドライブから内部ドライブに転送することができます。 この段階に関しては、それを実装する方法について特に多くの意見があります（完全に正しいとは限らない意見を含む）。 まず、盲目的な仕事を考えると、最初にこのセクションで説明するすべてを仮想マシンのラップトップで実行することは非常に便利です。 この素晴らしいガイドと私がすでに行ったコメントに従って、別の本当に仮想的なハードディスクをデスクトップ仮想マシンに接続し、これがターゲットマシンの内部ディスクであることを考慮して、それを練習することができます。 最終的には、外部ドライブを切断しても、仮想マシンからDebianを起動できるはずです。 状況をさらに「戦闘」に近づけるために、ホストマシン（ラップトップ）からSSH経由で仮想マシンを操作できます。 このため、たとえば、デフォルトで仮想マシンのネットワーク設定（NAT）を残すことができますが、ポートを転送します： 

 # vboxmanage modifyvm desktop --natpf1 "guestssh,tcp,,2222,,22"


（詳細はこちら ）

これで2222への呼び出し（番号は任意ですが、1024を超えることが望ましい-ルートのみがより小さいポートにアクセスできます）ラップトップポートは仮想マシンのポート22（標準SSH）に転送されます。 ホストからセッションを開くことができます： 

 # ssh -p 2222 root@localhost


（ほとんどの場合、localhostはアドレス127.0.0.1につながります）。 次に、ddを使用してディスク全体をコピーしないでください。コピーに時間がかからず、ほとんどの実用的なケースでスムーズに実行されます。 cp、rsyncを使用してファイルシステムをコピーできますが、特別なダンプ/復元ユーティリティ（非常に迅速かつ確実に動作します）が最適であり、ブート順序を復元するために、dd MBRレコードでコピーを削除することはできません-現代のDebianはgrub2を使用ドライブを一意のUUID番号にアドレス指定するのが好きな人は、システム内の外部ハードドライブへのすべてのリンクを残します。

そのため、以下を実行する必要があります（ターゲットマシンは外部ドライブからロードされます。/dev / sda、内部ドライブは/ dev / sdb、 SSHを使用してラップトップから作業しています）。

cfdiskを使用して、同じ方法で（または同じ方法で、コピー先のパーティションに目的のデータが含まれているはずの）内部ディスクを分割します。 

 # cfdisk /dev/sdb


ファイルシステムを作成します。 

 # mkfs.ext3 /dev/sdb1


マウントする： 

 # mount /dev/sdb1 /mnt


そして、外部ドライブのファイルシステムをそこにコピーします。 

 # (dump -0f - /) | ( cd /mnt; restore -rf - )


次に、grub2を構成する必要があります。 まず、コピーしたファイルシステムでchrootを実行します。 

 # mount --bind /dev /mnt/dev # mount --bind /proc /mnt/proc # chroot /mnt


現在、ターゲットマシンの内部ドライブにあるDebian環境に実際にいます。 ハードドライブの構成を更新します（仮想マシンによってゴミが残っています）。 

 # grub-mkdevicemap


-/boot/grub/device.mapファイルに自動的に書き込まれます。 grub2構成を更新します。 

 # update-grub


内蔵ディスクにインストールしましょう： 

 # grub-install /dev/sdb


（ドライブ上のパーティションではなくドライブを指定していることに注意してください）

また、設定を更新する価値があります。 

 # update-grub


次に、chrootをキャンセルしてシステムの電源を切ります。 

 # exit # halt


次に、外部ドライブを切断し、ターゲットマシンを再度起動します。 ラップトップからSSH経由で再度アクセスする必要があります。

これで、スワップパーティションを作成して接続できます。cfdiskを使用して場所を割り当ててから、 

 # mkswap /dev/sdb2 # sync; sync; sync # swapon /dev/sdb2


（同期はシステムバッファをフラッシュします）

おそらくホームセクションなどを開始します （ こちらを参照）。

システムの準備が整いました。
