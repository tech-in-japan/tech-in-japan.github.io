Ubuntu Server 12.04 LTSホームメディアサーバー

こんにちは、Habr！ 「理想的な」ホームネットワークに関する1つの記事の議論の中で、ハードウェアNASまたはLinuxディストリビューションを備えたミニコンピューターの方が良いという論争が生じました。 筆者はハードウェアNASを使用することを提案しました。管理が容易であると思われ、Linuxの知識を必要とせず、実際NASは静かです。 しかし同時に、彼はDLNA TVでサポートしていないビデオを見るために、DLNAをトランスコードするラップトップのスイッチを入れることを提案しました。 控えめに言っても、これは理想的なネットワークにあるべきではないので驚いた。 したがって、ホームネットワークの重要なコンポーネントの1つである集中データウェアハウスのビジョンを提示したいと思います。これはUbuntu Server OSを備えたミニPCに基づいています。



何が必要ですか？

まず、NASには、もちろん、信頼性の高いデータストレージとそれに便利なアクセスが必要です。 まず、信頼性のためにRAIDが必要です。これは、ハードドライブの故障によりホームメディアアーカイブ全体を失うことは、少なくとも愚かだからです。 データにアクセスするには、FTPおよびSambaアクセスを構成する必要があります。 もちろん、誰もがそれぞれのニーズを持っているので、MacOSまたはLinuxを使用する場合は、おそらく他のプロトコル（NFS、AFP）が必要になりますが、私は自分で行った方法で構成を説明します。

スマートテレビからメディアデータにアクセスするには、DLNAサーバーが必要です。 また、ダウンロードの利便性のために、トレントクライアントが必要です。 これはすべて、ウェブインターフェースを介して管理することをお勧めします。



なぜハードウェアNASではないのですか？

メーカーは長い間ユーザーの世話をしており、特に家庭用の既製の箱を長い間製造してきたようです。 しかし、それらには欠点があります：

1）高価です。 あなたは20,000ルーブルよりも安い見つけることはほとんどありません。 4台のハードドライブをAtomプロセッサに接続できるNAS。 安価なものは、通常、2つのデータストリームをダウンロードする（DLNAで映画を見たり、写真などをコピーする）際に同じトレントに十分ではない弱いプロセッサを使用します。 しかし、わずか6,000ルーブルでAtomと4 GBのメモリを搭載したmini-ITXマザーボードに基づいた本格的なミニPCを組み立てることができました！

2）制限されています。 つまり、製造元によって定められた機能のみを提供します。 ファームウェアのコアは大幅に切り捨てられる可能性があるため、その機能を拡張するには、「タンバリンと踊る」ことが通常必要です。 Ubuntuを使用すると、実質的に何でも無制限になります。あらゆる種類のソフトウェアの巨大なリポジトリにより、サーバーから仮想マシンの引き上げまで、あらゆることが可能になります。



なぜFreeNASまたはOpenFilerではありませんか？

お願いします。 まず、ハードウェアNASの欠点のポイント2を参照してください。つまり、これらのディストリビューションの機能を増やすことは非常に問題がありますが、Ubuntuには既に構成されたソフトウェアの巨大なリポジトリがあります。 第二に、これらは巨大なシステム要件です。特に、FreeNAS 8には最低2 GBのRAMが必要であり、OpenFilerの新しいバージョンはx86アーキテクチャ用にリリースされなくなりました。 さらに、FreeNASはどういうわけかスムーズに開発されていません-トレントクライアントとDLNAサーバーを備えたバージョン0.7はかなり前に時代遅れになり、8番目の商用バージョンではまだDLNAを設定できませんでした。システム障害が発生した場合、どのようにデータ復旧を注文しますか？ 難しいです。



Server 12.04 LTSディストリビューションが選択されるのはなぜですか？

LTS（Long Term Support）は、長期間のサポートとアップデートのリリースがあるディストリビューションです。 可能な場合は、一度構成すると、将来何年も静かに動作するサーバーが必要になるため、このバージョンの配布キットを選択することをお勧めします。

理想的には、グラフィカルシェルでリソースを無駄にする必要がないため、サーバーバージョンが明らかに選択されます。 Linuxに精通している場合、またはすでにUbuntuバージョンのデスクトップバージョンを使用している場合は、原則として通常のバージョンのディストリビューションを選択できますが、これは重要ではありません。



始めましょう

インストールは完全に透過的であるため、詳細には説明しません。 ハードドライブの内訳について説明します。











ハードウェアRAIDをサポートしない予算のマザーボードを使用しましたが、実際には、マザーボードに組み込まれたハードウェアRAIDが最良の面を示さないことがあるため、いわゆる「ソフトウェア」RAIDを編成します。 2つの新しいハードドライブがデータストレージに使用されます。 追加のストレージメディアはなかったので、ディスクを2つのセクションに分割します。1つはシステムの1つで、もう1つはデータ用です。 2台のハードディスク上の両方のパーティションはRAID 1に結合されます（仮想マシンでの利便性のためにすべての操作を実行するため、パーティションのサイズが小さいことに注意しないでください）。

最初に、最初のディスクにパーティションテーブルを作成し、2つの部分に分割します。 これらを「RAIDのパーティション」としてマークしますが、これは必須ではありません。







2番目のディスクも同様に分割されます。 次に、「ソフトウェアRAIDの設定」項目を選択します。 「MDデバイスの作成」と言い、2つのディスクの最初のパーティションを選択します。 データのセクションでも同様です。 ちなみに、RAIDは動的に変更および拡張できるため、これまでハードドライブが1台しかなく、2台目のハードドライブを購入する予定がある場合は、お気軽に構成し、購入後に簡単に取り出すことができます。







RAIDを作成したら、使用するためにそれらをマークします。 ext4ファイルシステムを選択し、マウントポイントを割り当てます。ルート（/）としてのシステムパーティション、および任意の場所へのデータパーティション（/ mntフォルダーにマウントすることをお勧めします）。







さらに、RAIDアレイに障害が発生した場合にシステムを起動するかどうかをシステムが通知します。 「いいえ」と答えることをお勧めします。ハードドライブに障害が発生しても気付かないこともあります。システムは1つのドライブで動作し続けますが、2番目のドライブに障害が発生すると、データ復旧会社に持ち込む必要があります。



スワップパーティションを作成しません。最初にファイルを作成でき、次に個人的には必要ないためです。ミニPCに4 GBがインストールされ、メモリ使用量が10％（400 MB ）、ただし通常の状態ではさらに少なくなります（現在は130 MBのみが使用されています）。 仮想マシンをレイズする予定がある場合は必要になる可能性があるため、インストール後にスワップファイルを作成する方法について説明しますが、スワップパーティションを作成するという提案に否定的に対応します。



ファイルをコピーする短いプロセスの後、システムはリポジトリからのデータの更新を開始し、更新のインストール方法を尋ねます。 システム管理が最小限に抑えられているため、自動更新を選択します。 システムは、すぐにどのパッケージをインストールするかを尋ねます。 OpenSSH（リモートコマンドラインが必要）、LAMP（Webインターフェイスに必要）、プリントサーバー（この記事ではプリンターの接続方法については説明しません）、そしてもちろんWindowsマシンからアクセスするためのSambaファイルサーバーを選択しました。



さて、最終段階で、システムはMySQLのパスワードとGRUBのインストール要求を要求します。 再起動-システムがインストールされました！ ログインして、DHCPが割り当てられたIPアドレスを確認します（ifconfigコマンドを使用しても実行できます）。私の場合、アドレス192.168.1.180が発行されました。



すべて、モニターをオフにして、便利な場所でシステムユニットを取り外すことができます。その後、SSHを使用して作業します。 これにはPUTTYを使用します。





構成



1）スワップファイル
まず、スワップファイルの設定方法を説明します。本当に必要な場合は、数行のコマンドですべてが行われます。

ゼロで満たされたファイルを作成します。 
> sudo dd if=/dev/zero of=/swap bs=1M count=2048


スワップとして使用するために準備します。 
 > sudo mkswap /swap


作成したファイルをfstabファイルに追加して、スワップファイルとして使用します。 

 > sudo nano /etc/fstab
      
      

        
        
        
      

    
        
        
        
      
      

        
        
        
      

    
     
 /swap none swap sw 0 0


再起動： 
 > sudo shutdown -r now


2）ソフトウェア更新
すべてのパッケージをすぐに更新します。これは2つのコマンドで実行されます。 
 > sudo apt-get update > sudo apt-get upgrade


3）Webインターフェース
Webインターフェイスを介してシステムを管理するためのwebimパッケージがありますが、残念なことにリポジトリにはないため、準備されたパッケージを手動でダウンロードします。 
 > wget http://prdownloads.sourceforge.net/webadmin/webmin_1.580_all.deb


webimのインストールにはいくつかの依存パッケージが必要になりますが、私の場合はそのようなリストです。他のものを含める必要があるかもしれません。 
 > sudo apt-get install libnet-ssleay-perl libauthen-pam-perl libio-pty-perl apt-show-versions


さて、実際のインストール： 
 > sudo dpkg --install webmin_1.580_all.deb


Webインターフェースにアクセスできるすべて： 
 https://192.168.1.180:10000






4）FTPアクセスを構成する
ftpには、pure-ftpdを使用します（ただし、好みに応じて選択できます-proftpdとvsftpd）

パブリックフォルダーを作成します。 
 > sudo mkdir /mnt/data/public


リポジトリからpure-ftpdをインストールします。 
 > sudo apt-get install pure-ftpd


原則として、システムアカウントで既にログインできますが、これは日常の使用にはまったく適していません。 パブリックフォルダーのみにアクセスできる仮想アカウントを作成しましょう。 
 > sudo pure-pw useradd public -u local -g nogroup -d /mnt/data/public


データベースを更新します。 
 > sudo pure-pw mkdb


仮想ユーザーの使用を有効にしましょう： 
 > sudo ln -s /etc/pure-ftpd/conf/PureDB /etc/pure-ftpd/auth/50pure


サービスを再起動します。 
 > sudo service pure-ftpd restart


5）サンバ
Windowsベースのマシンからサーバーへのアクセスを設定します;さらに、私の家族は個人的に大規模であり、複数のユーザー間で権利を共有する必要があります。 また、Windowsから（プロパティの[セキュリティ]タブを使用して）フォルダへの権限を簡単に編集するために、ACLを使用します。

ドメインがないため、Windowsマシンと同じようにユーザーを作成する必要があります。 
 > sudo useradd -d /home/PaulZi -s /bin/true -g users PaulZi


Windowsと同じパスワードを設定します。 
 > sudo passwd PaulZi


作成したユーザーをSambaに追加します。 
 > sudo smbpasswd -a PaulZi


拡張権利を管理するには、ユーティリティをインストールできます（オプション）。 
 > sudo apt-get install acl > sudo apt-get install attr


sambaがACLを使用するには、POSIX ACLをサポートするファイルシステムが必要です。ext4は非常に適していますが、デフォルトではこのサポートなしでマウントされます。 この機能を有効にするには、「acl」オプションを/ etc / fstabファイルに追加します。 しかしそれ以上に、権利の継承のサポートはWindowsで実装されているため、これはLinuxでも実装されるため、sambaは追加データをどこかに保存する必要があります。 これを行うには、拡張ファイル属性、オプション「user_xattr」を有効にします。 同時に、「noexec」オプションを使用して、データセクション全体でのファイルの実行を禁止します（セキュリティのため）。 
 > sudo nano /etc/fstab
      
      

        
        
        
      

    
        
        
        
      
      

        
        
        
      

    
     
 /dev/md0 /mnt/data ext4 defaults,noexec,acl,user_xattr 0 2


再起動： 
 > sudo shutdown -r now


サンバの設定を編集します（簡潔にするため、変更と追加のみを行います）。 
 > sudo nano /etc/samba/smb.conf
      
      

        
        
        
      

    
        
        
        
      
      

        
        
        
      

    
     
 [global] workgroup = Home netbios name = Server security = user # add settings admin users = PaulZi #       root map acl inherit = yes #   acl store dos attributes = yes #   dos  #   windows : map archive = no map system = no map hidden = no map readonly = no # public share [public] comment = Public path = /mnt/data/public browseable = yes #   read only = no #    guest ok = yes #    inherit permissions = yes #    inherit acls = yes #   windows- inherit owner = yes #    hide unreadable = yes #


サービスを再起動します。 
 > sudo service smbd restart


6）DLNA / UPnP-サーバー
DLNAサーバーとして、minidlnaを選択しました。 MediaTombやServiio（Javaまたはグラフィックライブラリをプルする）のような不要な依存関係をプルしないという単純な理由の1つを選択しました。 ただし、トランスコーディングが必要な場合は、minidlnaではなく、それらのいずれかをインストールすることをお勧めします。

リポジトリからのインストール： 
 > sudo apt-get install minidlna


カスタマイズ： 
 > sudo nano /etc/minidlna.conf
      
      

        
        
        
      

    
        
        
        
      
      

        
        
        
      

    
     
 media_dir=/mnt/data/public friendly_name=Ubuntu


再起動： 
 > sudo service minidlna restart


7）トレント
さて、この記事で取り上げる最後のサービスはトレントクライアントです。 私は、定評のあるWebベースのクライアントとしてTransmissionを使用しています。

インストール： 
 > sudo apt-get install transmission-daemon


サービスを停止します。そうしないと、プロセスの完了後にすべての変更が消去されます。 
 > sudo service transmission-daemon stop


カスタマイズ： 
 > sudo nano /etc/transmission-daemon/settings.json
      
      

        
        
        
      

    
        
        
        
      
      

        
        
        
      

    
     
 "download-dir": "/mnt/data/public/torrents" "rpc-password": "local" "rpc-username": "local" "rpc-whitelist-enabled": false


ここでは、4つの設定を変更します。Webインターフェイスのダウンロードパス、ユーザー名、パスワードを設定し、インターフェイスへのアクセスのホワイトリストを無効にします-すべてのユーザーを許可します。 パスワードは明確な形式で指定し、その後の起動後に暗号化されます。

サービスを開始します。 
 > sudo service transmission-daemon start


Webインターフェースに入り、すべてが正常であることを確認します。 
 http://192.168.1.180:9091/






あとがき

その結果、本格的なホームサーバーが手に入りました。 もちろん、この記事ではサービスの基本設定のみを示しているため、おそらく自分で何かを構成する必要があります。 はい、追加のサービスが必要になる場合がありますが、記事からわかるように、特別な「タンバリンと踊る」ことなく、これらすべてを非常に簡単に行うことができます。Googleで検索するだけです。Ubuntuでのサービスの設定に関する多くの情報があります。
