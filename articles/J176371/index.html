ビジネスアプリケーション1C + Androidの開発

こんにちは。





この記事では、1C + Androidで企業のモバイルアプリケーションを開発するプロセスを大幅に促進する新製品「FBA Toolkit」を紹介します。

「FBAツールキット」は次のとおりです。

-JavaのネイティブIDEでの「ネイティブ」Androidアプリケーションの開発。

-1C：データソース、企業情報システムとしてのEnterprise 8.1（8.2）。



まず第一に、このフレームワークは、Android用のビジネスアプリケーションを開発したいが、新しいプラットフォームを開発するのが困難な1Cプログラマーにとって有用です。 Javaの初期知識は、開発を開始するのに十分です。 次に、クライアントサーバーアプリケーションの実用的なプロトタイプをわずか数時間で作成する方法を示します。



職場の準備



開発プロセスを直接開始する前に、開発者の職場を準備する必要があります。 必要になります：



1.インストールおよび構成：

-Java SDK + Eclipse + Android SDK;

-Webサーバー（IISまたはApache）;

-1C：エンタープライズ8.1（8.2）



2. Eclipseの場合、SVNリポジトリからActionBarSherlockとru_profu1c_fbaの 2つのライブラリプロジェクトをインポートします。



3.「FBA」構成配布キットをインストールし、1Cベースのアプリケーションソリューションと組み合わせます。



プリセットのプロセス全体については、記事で詳しく説明しています。

開発者の職場を準備する

「FBA」構成ディストリビューションのインストール

「FBA」構成とアプリケーションソリューションの組み合わせおよびWebサーバーへの公開



セットアップにはそれほど時間がかからないはずで、おそらく、インストール済みの必要なソフトウェアの一部です。



アプリケーションの説明



企業の状況を定期的にマネージャーに通知するように設計されたモバイルアプリケーション「マネージャーへのレポート」を作成します。 アプリケーションは3つの画面で構成されます。

  1. 「ログイン」-プログラム開始時のユーザー認証。
  2. 「レポートリスト」は、アプリケーションのメイン画面です。 利用可能なレポートのリストが表示され、1Cデータベースのデータを表示および更新できます。
  3. 「プログラム設定」-ここで、ユーザーは許可パラメーターと1Cサーバーのアドレスを設定し、1Cサーバーとの自動交換のスケジュールを設定します。






アプリケーションは、1Cサーバーからデータを受信するための2つのオプションを実装します。

  1. レポート「倉庫内の商品」。 サーバー上で生成され、HTMLドキュメントとしてコンパイルされ、モバイルアプリケーションに単純に表示されます。 最も単純なオプションであるレポートフォームは固定されており、変更できません。
  2. レポート「財務実績」。 ここでは、レポートの初期データがテーブルの形式で取得され、その後、レポートが作成されます。 このオプションのプラス：任意の形式のレポート出力、税込。 グラフおよびチャートの形式で。 選択とソートが可能です。


1Cでのモバイルアプリケーションテンプレートの作成



驚くことではないので、モバイルアプリケーションの開発は1Cから始まります。 ここで、メタデータの構成を決定し、交換スキームを設定し、将来のモバイルアプリケーション用にテンプレートを再生成します。



「Report to the Manager」という名前と識別子SAMPLE_REPORT_HEADを使用して、モバイルアプリケーションディレクトリに新しい要素を作成します。



「メタデータ」リストで「外部データ」グループを強調表示し、ツールバーの「追加」ボタンをクリックします。 図に示すように、新しいテーブルのパラメーターを入力します。







「インジケータ」 -レポートインジケータのテスト表現、行（250）

「値」 -インジケータの数値、数値（15.2）

原則として、列の名前とテーブル自体はロシア語で設定されます。これらに従って、テーブルには組み込みの1C言語を使用してデータが入力されます。 [OK]をクリックすると、テーブルがメタデータツリーに追加されます。モバイルアプリケーションの場合、名前は自動的にラテンアルファベットに変換されます。



したがって、レポート「財務指標」のデータを転送する値の表の構造（1Cの観点から）を準備しました。 2番目のレポート「倉庫の商品」では、データは外部ファイルとして転送されます。



「Exchange Scheme」タブで、「Upload」フラグがチェックされていることを確認します。 「ダウンロード」フラグは重要ではありません。 モバイルアプリケーションからこのテーブルに受信した変更を1Cで処理することはありません。







変更を保存します。 ツールバーの「モバイルアプリケーションテンプレート」ボタンをクリックして、モバイルアプリケーションのパラメーターを設定します







「テンプレートディレクトリ」-生成されたモバイルアプリケーションテンプレートファイルが保存されるディレクトリへのパス。



[基本]タブで、パッケージの名前を指定します。これは一意の識別子である必要があります。 Webサイトがある場合は、それを使用してプレフィックスを生成します。 この例では、 ru.profi1c.samples.report.headを指定しました



「Webサービス」タブにはIPアドレス10.0.2.2が含まれています。デフォルトでは、Androidエミュレーターからアクセスするときのコンピューターのアドレスです。 [Webサービス名]フィールドと[アプリケーションサブディレクトリ]フィールドに、Webサービスの公開時に指定されたデータを入力します。







1C 8.2では、ほとんどの場合、アプリケーションのサブディレクトリに追加のサブディレクトリ/ ws 、つまりdemoFba / wsがあることに注意してください。詳細については、 こちらを参照してください。

[ 作成]をクリックすると、Androidプロジェクトテンプレートが生成されます。

この例では、テーブルにダミーデータが入力されています。実際のアプリケーションでは、ここで任意の数のレポートインジケーターを転送できます。

最初のレポートのデータが整理されると、2番目のレポートの準備が残ります。 同じモジュールfbaSubscribeOn Eventsで、次のコードをPrepare BinaryDataForSendingプロシージャに追加します。

コードについてコメントします。 モバイルアプリケーションが「REPORT_PRODUCTS_IN_STOK」という識別子を持つ任意のデータ（ファイル形式）を要求すると、レポートが1Cで実行され、その結果が一時ファイルに保存されます。 その後、この一時ファイルはバイナリデータとして読み取られ、モバイルアプリケーションに送信されます。 一時ファイルが削除されます。



この場合、htmlファイルが生成されるため、問題ではありません。 標準の1Cメカニズムを使用して、スプレッドシートドキュメントをHTMLとして保存するか、HTMLを手動で生成できます。 出力の例は、構成「FBAデモ」、レポート「倉庫内の商品」で確認できます。
