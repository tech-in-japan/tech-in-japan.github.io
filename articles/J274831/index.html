ハイキングをする普通の人は自然を楽しみます...



Linuxを理解していない人のために、フレームバッファは（一般的なデバイスと同様に）Linuxファイルであり、その中に（理論的に）メモリースナップショットファイルを書き込み、画面に表示できることを説明します。 ハードウェアのスクリーンショットを撮ることもできます。 このファイルを別の場所にコピーしてビデオメモリをダンプし、コピー（または書き込み）して表示します。



これが必要な理由：
なぜこれが必要なのか：たとえば、ターゲットマシンでは、仮想スクリーンへのビデオ出力があり、その解像度は本の解像度（つまり600x800）に等しくなります。 その後、定期的に仮想スクリーンフレームバッファファイルが取得され、SOMポートにリダイレクトされます。その最後に、comコマンドからcatコマンドを使用して、これが本のフレームバッファにリダイレクトされます。



すべてのコマンドはターゲットマシンで実行され、本はUARTを介して、/ dev / ttyUSB0でハングするVCPを介して接続されていることを示します。 コマンドは、たとえばクローネで数分ごとに呼び出されます。



echo "cat / dev / ttyS0> / dev / fb0"> / dev / ttyUSB0



このUARTコマンドは、COMポートファイルの内容をフレームバッファーにリダイレクトするコマンドを送信します。 次に、2番目のコマンドがあります。



猫/ dev / fbn> / dev / ttyUSB0



このコマンドは、fbn仮想画面（nは画面番号）の内容をCOMポートに転送します。 すべてが正常に機能するはずです。ファイルの終わりの文字が到着すると転送が終了します。

 /* showpic.c */ #include <stdio.h> #include <sys/types.h> #include <sys/stat.h> #include <fcntl.h> #include <sys/ioctl.h> #include <sys/mman.h> #include <string.h> #define FBIO_EINK_GET_TEMPERATURE 0x46A1 //Returns temperature in degree Celsius #define FBIO_EINK_DISP_PIC 0x46A2 //Displays picture int main (int argc, char* argv[]) { printf ("Show image\n"); int *fb, *image; int pio_fd = open ( "/dev/fb0", O_RDWR); int f_image = open ( argv[1], O_RDWR); //open file into arg int t= ioctl (pio_fd, FBIO_EINK_GET_TEMPERATURE, NULL); //configure framebuffer fb= mmap(0, 800*600, PROT_WRITE, MAP_SHARED, pio_fd, 0); //map device into memory image= mmap(0, 800*600, PROT_READ, MAP_SHARED, f_image, 0); //load image into memory memcpy(fb,image,800*600); ioctl (pio_fd, FBIO_EINK_DISP_PIC, 0); close(pio_fd); close(f_image); return 0; }

UPD寝ている間に、テキスト入力とコンソールの問題を解決する方法を見つけました。 ファイルがテキスト形式で送信されるのを妨げるものは何もありません。 kinsoliからのデータを受け入れ、それに2つのコマンドを渡すように、ファイルを少し書き換える必要があります。

startは転送の開始で、 stopは終了です。 次に、送信されたファイルをソースコンピューターでバイト単位で読み取り、バイトをニブル（半分のバイト）に分割します。 ニブルは、0から15の値、または0からFの16進数の値を取ります。まあ、文字を0からFに転送します。本の側面で、テキストニブルをバイトに逆変換します。 転送の開始は、開始、停止の終了を意味します。 コンソールのエコー伝送を制御します。

所要時間は2倍になりますが、解決策は最も単純で明白です。
