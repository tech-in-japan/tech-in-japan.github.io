WinC ++に関する噂

インターネット上には、特定のWinC ++に関するうわさがあります。 TolleyはVisual C ++の新しい名前であり、Tolleyはそれ以上のものです。 私はVisual C ++のトピックに興味を持っているため、このトピックに注意を払いました。 5月4日の最近の記事「WinC ++とは何ですか？また、ツールを20億ドルのビジネスにするというマイクロソフトの提案にどのように当てはまりますか？」 ロシア語ではこれまでWinC ++が静かなので、Habraコミュニティにこの記事の翻訳を提供することにしました。 翻訳はあまり良くなかったので、読むのは退屈です。 したがって、多くの人はおそらくオリジナルを知っていることを好むでしょう。



最初のパンケーキはゴツゴツしていると言うので、この記事をarticleさないでください。 この記事で有用かつ最も重要なことは、おそらく、MicrosoftがVisual C ++を埋めることを明確に計画しておらず、むしろこの方向に新しい力を注ぐことです。 C ++は、生きて進化するツールと見なされています。 喜ばれます（少なくとも、:)）。



したがって、Mary-Joe Follyの記事の実際の翻訳は、WinC ++とは何ですか？また、ツールの売り上げを20億ドルに増やしたいというマイクロソフトの願望とどのように関連していますか？







注釈



Microsoftは、WinC ++の今後のリリースに関連して、C ++製品とマーケティング戦略を強化しています。 これについて、私がなんとか発掘したいくつかの情報を以下に示します。



マイクロソフト内のサーバーおよびツール開発部門の再編成によって今週発生した塵が解決し始めたので、質問「誰？」から質問「なぜ？」および「方法？」に移りましょう。



マイクロソフトの当局は、なぜ開発マーケティングと伝道グループを開発部門の長であるソマソマセガーのオフィスに移すことにしたのですか？



「マイクロソフトには、開発環境でのコンプライアンスに大きな問題があります」と、企業の懸念を知っている友人の一人が言いました。 Hewlett-Packardからパブリッククラウドプログラミング戦略 （Java、Ruby、および「オープンソース言語」に焦点を当てている） についてリークされたばかりの情報は、この点を正当化します。 C＃、C ++、およびVisual Basicの言語にはまだ支持者がいますが、Java、Ruby、Python、PHP、Objective-CなどのMicrosoftがサポートしていない言語は自信を持って基盤を保持し 、高速ではないにしても高速で開発します。



Somasegarの再編成に関するSTBへのメールからわかるように、同社はVisual Studioの売上を現在の10億ドル以上から20億ドルに引き上げる努力をしています。 （Visual Studioは、WindowsおよびOfficeスイートに加えて、すでに10億ドルの販売フロンティアを超えている数少ない製品の1つです。）



マイクロソフトは、確立された開発チームに関係なく、どのように収益を上げるつもりですか？



方法の1つは、Visual C ++などの開発者向けの「ネイティブ」ツールの位置を強化することです。 ソマセガーのメールでWinC ++がさりげなく言及されていることに気付きました。 WinC ++がVisual C ++の新しい名前になることが判明しました-このアイデアは、Microsoftの求人広告の1つに「Windows C ++チーム、Visual C ++チームとしても知られている」という明確な説明があることで確認されています。



私にとって興味深いものになりました。WinC++は、実際には古いコンパイラの新しい（そしてWindowsブランドで明確に「シャープ」になった）名前にすぎませんか？



Microsoftでの作業に関する別の発表から、私は、同社がC ++が今後のバージョンのWindows用のアプリケーションを作成するための鍵になるという考えを実際に促進していると結論付けました。 以下は、WinC ++のプロジェクトマネージャーの任命に関する発表からの抜粋です。



「Windowsプラットフォームを開くアプリケーションの新機能に感銘を受けましたか？」 世界中のC ++開発者と協力し、刺激を受けて、最先端のユニークなWindowsソリューションを作成したいですか？ ''



（このようなWindowsへの焦点は、Internet Explorer開発チームの活動を思い起こさせます。InternetExplorer開発チームは、 Windows 7での最適化によりIE 9がどのように改善/高速化されるかをますます強く主張しています 。グループは同じポリシーとIE 10に関連して、メッセージングエンジンの「現在改善されたネイティブHTMLサポート」を指し、それにより、ブラウザーに最適なプラットフォームであるWindows 8を指します。



次の質問は、MicrosoftがC ++製品の名前を変更するだけでなく、Visual Studio 2012と呼ばれる予定のVisual Studioの次のバージョンに新しいC ++機能を実装する以外にやることがありますか？ はいのようです。



以下は、WinC ++に言及しているMicrosoftでの別の求人からの抜粋です。



「開発者が次のバージョンのWindows用のC ++アプリケーションを作成できるようにしたいですか？」 COMの新しいバージョン（コンポーネントオブジェクトモデル、複合オブジェクトのモデル）を詳細に研究し、その機能を誇るコンテンツを作成しますか？ 次のバージョンのWindowsの新しいプログラミングモデルについて書きたいですか？” ...



「開発チームは、大規模で複雑なプロジェクトの作成を目標に、C ++言語の革新と「ネイティブ」コードの開発に積極的に投資しています。 彼女はまた、統合開発環境（IDE）をより広範囲にスケーラブルにし、より使いやすく、そして最も重要なこととして、最新のC ++開発者のニーズに迅速に適応できるようにすることで、C ++開発者が豊富な開発環境を使用できるようにする技術にも取り組んでいます。 これらの投資は、ほとんどすべてのマイクロソフトソフトウェア製品だけでなく、多くの大手独立系ベンダーの製品にも直接影響します。 私たちは、言語の革新やコード生成の分野など、いくつかの分野で非常に野心的で効果的なソリューションを数多く開発し始めています。」



WinC ++に関するマイクロソフトの別の発表は、「Visual Studioに付属するC ++言語とCOMテクノロジの新しい開発、および標準テンプレートライブラリなどの既存のC ++ライブラリの更新」に関するものです。 」



ここに、おそらく私たちのトピックに関連したいくつかのメモがあります：

  • マイクロソフトのChanel9チームは最近、新しいC ++コンテンツを投稿しました 。
  • サーバーおよびツール部門の再編成の一環として、Microsoftは並列プログラミングおよびツール開発チーム（Steve Teixeira、 David Callahan 、Sharokh Mortazaviが率いる）を「WinC ++、PCP（おそらく並列および/または高性能プログラミング？）モーセン（アグセン）のリーダーシップの下で。」 nguageランタイム（共通言語ランタイム）。 （ 2011年2月のChannel9とのインタビューで、彼は「C ++の復活」について話しました 。）
  • 今年の少し前にブログで書いたWindows 8用のMicrosoft Jupiter開発環境は 、XAMLとネイティブコードのアプローチに何らかの関係があるようです。 おそらくC ++は、Jupiterでサポートが計画されていた言語の1つでした。
  • このオペレーティングシステムでC ++ / XAMLサポートが利用できることを考えると、 Windows Embedded Compact 7 （およびこのプラットフォームで実行されるすべての携帯電話とタブレット）との何らかの接続もあるかもしれません。
