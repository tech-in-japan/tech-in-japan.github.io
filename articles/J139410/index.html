MS Windows Server 2003フェールオーバークラスター

はじめに



MSA500 G2を搭載したHP ProLiant DL380 G4パッケージクラスター -これは、Hewlett Packardのターンキーソリューションであり、外部ストレージを備えた2つのサーバーで、クラスターの構築を意味します。 まあ、クラスタはとてもクラスタです、完了したと言いました。 そのようなシステムを作成したことはこれまでになかったため、私たちが犯したすべての間違いを含めて、できる限り詳細にプロセスを説明しようとします。





理論のビット



では、なぜクラスターが必要なのでしょうか？ おおよそ2つのオプションがあります。フォールトトレラントシステムの作成、または負荷分散システムのいずれかです。 私の場合、これはフォールトトレラントシステムです。1台のサーバーが最初に動作し、ソフトウェアまたはハードウェアの障害が発生すると、2台目のサーバーがケースに入り、これらのサーバーにインストールされた特定のサービスがすぐに動作を再開します。 これがすべて行われているサービスの詳細には触れません。何かをpingし、それに関するレポートを作成するプログラムの例にしましょう。



クラスターを構築するには、いくつかの基本的な条件を考慮する必要があります。クラスターセグメントの互換性のある機器-問題はありません。同一のサーバー、互換性のあるソフトウェア、共有ディスクスペースがあります。 また、クラスターセグメントは同じドメインに存在する必要があります。 私たちのシステムはその技術的ネットワークにおいて独立しているため、ドメインも作成する必要があります。 ドメインコントローラとして別のサーバーが必要です。また、システムはフォールトトレラントである必要があるため、2つのドメインコントローラが必要です。 ただし、これらの同じサーバーが存在しないため、クラスターセグメントはドメインコントローラーになります。 Active Directoryサービスは、クラスター化には適していません。まあ、クラスターはクラスターであり、ドメインコントローラーそのものです。



ディスクアレイ



ハードドライブの準備から始めましょう。 HP StorageWorks 500 G2 Modular Smart Arrayディスクアレイには、146ギガバイトのSCSIドライブ9台と34ギガバイトのSCSIディスク2台が搭載されました。 SCSIインターフェイスを介してサーバーに接続され、サーバー自体のRAIDコントローラーの組み込みユーティリティを使用して制御されます。 合計ディスク容量には、146ギガバイトのディスクを使用し、それらを組み合わせてRAIDアレイにします。 可能性のあるオプションのうち、もちろん5番目と比較してRAID 6が選択されました。生産性は低く、原則として私たちにとって重要ではありませんが、信頼性はより高くなります。 また、クラスターの場合、クラスターを監視するための技術セクションであるクォーラムを作成する必要があります。 彼は少なくとも50メガバイトのディスクスペースを必要とするため、クォーラムでは、34ギガバイトの残りの2つのディスクのミラー（ RAID 1 ）を使用します。 2つのSCSIディスクがサーバー自体のボードにインストールされます。これらはRAID 1と同じであり、サーバーオペレーティングシステムがインストールされます。



したがって、3つのRAIDアレイRAID 1と1つのRAID 6を取得し、9つのディスクのうち8つだけが使用され、1つのディスクが冗長であるため、理論的には3つのディスクが故障してもRAIDは機能します。







OSおよびActive Directory



最初にWindows Server 2008をインストールしましたが、後でHP ProLiant DL380 G4はパラレルSCSIを使用し、共有ストレージデバイスの種類としてパラレルSCSIサポートがWindows Server 2008ベースのフェールオーバークラスターで削除されたことがわかりました。WindowsServer 2003とHyper -Vは問題外です。 また、Windows Server 2003 StandardおよびWeb Editonはクラスターをサポートしていないため、Windows 2003 EnterpriseまたはDatacenterを使用する必要があることにも注意してください。



Active Directoryインストールウィザードを使用して、新しいドメインを作成します。











ウィザードのすべての段階が意味を示すのではなく、いくつかの点にのみ注意を払います。 ADセットアップウィザードは、DNSがないことを警告します。 もちろん、Active Directoryを作成する前にDNSサーバーの役割を追加することもできますが、ここで行うことができます。







そのため、Active Directoryを作成する場合、クラスターはドメインアカウントで構成されているため、将来のクラスターのドメイン管理者権限を持つアカウントを追加する必要があります。

アカウントは、Radio.localドメインのRadioserverと呼ばれます。







次に、同じActive Directoryインストールウィザード（dcpromo）を使用して、2番目のサーバーを追加のドメインコントローラーとして接続します。







追加のコントローラーが追加され、Active Directoryコントロールウィンドウにコントローラーが表示されます。







これで、クラスターを作成する準備ができました。



クラスター



クラスターを作成するには、サーバーの準備を完了する必要があります。 フォールトトレラントサーバーの場合、フォールトトレラントネットワークを作成する必要があるクォーラムもあります。 サーバーには2つのネットワークアダプターがあり、1つはクラスターセグメント間の内部通信に使用され、もう1つは外界との通信に使用されます。それぞれを「クラスター」と「ラン」と呼びます。







「クラスター管理」を開始します。アクションは「新しいクラスターの作成」です。 クラスター作成ウィザードが起動します。ネットワーク上の他のデバイスでは、クラスターは別のサーバーのように見えるため、ドメインを指定して一意のクラスター名を割り当てる必要があります。 次に、新しいクラスターの最初のノードとなるコンピューターを指定します。 次は構成分析です。







構成がクラスタリングを満たさない場合、すべてのコメントを削除することが望ましいです。問題の説明はログで詳細に説明されます。 さて、すべてのポイントが満たされている場合、クラスターのIPアドレスを入力すると、ウィザードが将来のクラスターの構成を提供します。 同じウィンドウで、自動的に選択されたクォーラムセクションを変更できます。デフォルトでは、ウィザードは最小のパブリックセクションを選択します。



クラスターの作成が完了すると、2つのクラスターグループと2つのネットワークを持つ1つのノードで構成されるクラスターがあることがわかります。







クラスター化されたグループには、クラスター名、IPアドレス、およびクォーラムというリソースが含まれます。



ここで、最初のノードと同様に、作成したクラスターに既に2番目のノードを追加します。 最後に、クラスターの準備が整いました。その操作性を確認するために、クラスターグループをあるノードから別のノードに手動で移動してみてください。たとえば、パブリックネットワークのネットワークアダプター「Lan」を切断することで、この移行を人為的に引き起こすことができます



さて、クラスターの準備はできましたが、次に何をすべきでしょうか？ 残念ながら、このアプリケーションはクラスタリングをサポートしていません。つまり、仮想マシンで実行する必要があります。



Microsoft Virtual Server 2005 R2



選択された仮想マシン-Microsoft Virtual Server 2005 R2 SP1-Enterprise Edition 。 両方のノードに仮想マシンをインストールしますが、その前にWindowsコンポーネント（「アプリケーションサーバー」）を追加する必要があります。







インストール後、ノード1のクラスターサービスが仮想サーバーをノード2に転送するために使用するスクリプトを作成します。さらに、クラスターサービスが停止すると、すべてのリソースがノード2に転送されます。 「C：\ Stop_cluss_script.cmd」という名前で。 グループポリシー（gpedit.msc）に移動し、[シャットダウン]を選択してスクリプトを追加します。







このスクリプトは両方のノードに追加する必要があります。 次に、「％systemroot％\ Cluster」フォルダ内の両方のノードで、スクリプト「Havm.vbs」を追加します。これにより、ノードが変更されたときに仮想マシンが起動します。







仮想マシンを両方のノードで実行するには、共有ディスク-D（データ）に配置する必要があります。 これを行うには、共有ディスクをクラスターリソースとして追加します。 クラスターの管理に移動し、「新規グループ」操作を選択します。そこで、優先所有者を示します-これらは2つのノードです。 グループをSmartPTTと呼びます。 作成されたグループで「DiskStorage」リソースを作成し、同様に所有者を追加し、ドロップダウンディスクから物理ディスクを選択します。



Virtual Server管理Webサイトを使用して、仮想ネットワークを作成します。 仮想ネットワークアダプターとして、クラスターのパブリックネットワークのアダプター「Lan」に対応するアダプターを選択します。







次に、構成ファイルを共有ディスクに転送する必要があります。これには、ナビゲーションパネルで[仮想ネットワーク]をポイントし、[構成]をクリックして、[すべて表示]をクリックします。 新しく作成したネットワークにマウスを合わせて、「構成の編集」を選択します。











これにより、デフォルトで設定ファイルが保存される場所がわかります。 Virtual Server管理Webサイトで、[戻る]をクリックし、ネットワークにカーソルを合わせて[削除]を選択します。 この手順の目的は、仮想ネットワークの作成を廃止することではなく、.vncファイルがクラスターストレージに移動された場合にVirtual Serverシステム情報をクリアすることです。 この操作をスキップすると、仮想ネットワークは正しく機能しなくなります。



共有ストレージディスクに「SmartPTT」フォルダーを作成します。仮想マシンのファイルがあります。 そして、ClusterNetwork.vncをここに移植します。







次に、Virtual Server管理Webサイトで[仮想ネットワーク]をポイントし、[追加]をクリックして、.vncファイルの新しい場所を指定して、[追加]をクリックします。 次に、仮想マシン名で仮想マシンを作成するときは、名前を入力するだけでなく、仮想マシン構成のストレージパスを入力する必要があります。リポジトリ内のフォルダー-「D：\ SmartPTT \ ... vnc」を指定します。



[マスターステータス]ページで、作成されたマシンのスケッチ。 VMRCサーバーを有効にするように求められたら、ActiveXコントロールを設定してください。 以降のセキュリティダイアログで[OK]をクリックします。 作成したマシンの構成に進み、CD / DVDセクションを選択します。ここで、インストールファイルの入手先を指定します。 次に、マシンの[オン]をクリックすると、標準のOSインストール手順が開始されます。 次に、ノード2で、仮想サーバー管理Webサイトを使用して、仮想サーバーの構成ファイルを手動で指定する必要があります。



次に、クラスター管理に移動し、以前に作成したグループに新しいリソースを作成します。 リソースタイプ「ユニバーサルスクリプト」：







Havm.vbsスクリプトへのパス-％systemroot％\ Cluster \ Havm.vbsを指定し、リソースの依存関係に「DiskStorage」を追加します。共有ディスクが仮想マシンにアクセスできない場合、構成ファイルを取得する場所がないためです。 これでセットアップが完了し、仮想マシンにオペレーティングシステムをインストールし、すべてのサービスを実行しました。



おわりに



私たちのサービスは、たとえば、何かをpingして、それに関するレポートを作成するプログラムです。 仮想マシンの外部でレポートを表示するには、仮想マシンおよび他のすべてのネットワークメンバーがアクセスできるディスクにレポートを保存する必要があります。 この状況を回避する方法として、データストレージであるネットワークドライブが仮想マシンで実行されているOSに追加されました。 ネットワークドライブがクラスターIPアドレスを介して接続されていたため、問題が発生しました。 クラスターリソースの「クラスターIPアドレス」を移動する場合、仮想マシンを含む他のネットワークメンバーは、クラスターがアダプターを変更したことを認識せず、障害が発生したノードの非アクティブアダプターにクラッシュし続けます。 その結果、クラスターIPアドレスは5〜10分間利用できず、その結果、このサービスは常にネットワークドライブにデータを保存できませんでした。
