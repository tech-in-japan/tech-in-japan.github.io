Itseezコンピュータービジョンサマースクール：いかがでしたか

今年、 イッツェーズは、コンピュータビジョンに興味のある学生のために、Interuniversity SchoolとSummer Internshipという2つの夏のイベントを開催しました。 ここでは、過去の両方のイベントについてお話します。ニジニノヴゴロド州立大学のITMM研究所と共同で開催された学校について N.I. ロバチェフスキー、そしてインターンシップ。彼らはイッツェーズの会社に直接行きました。

トレーニングがどのように行われたか、参加者の選択がどのように行われたか、そして一般的にどのように起こったかを共有します。 冬には、別の学生イベントが計画されています-コンピュータビジョンアプリケーションのパフォーマンスを最適化する冬の学校です。 したがって、この記事は、Itseezの将来の教育活動に参加したい人にとって特に興味深いものです。













イベントについて調べる方法は？



もちろん、私たちが準備していた最初の質問の1つは、これら2つのイベントのターゲットオーディエンスが誰で、どのように参加者の間で正確に見たいかを人々にすばやく伝える方法でした。



これはそのような最初のプロジェクトだったので（最後ではありません！）、まず最初に、ニジニノヴゴロド大学（NNSU、HSE、NSTU、NNGASU）の学生と卒業生の参加に限定することにしました。 この情報は、 Itseez VKontakteグループに投稿され、大学や対象学部の公共の場所でポスターやチラシの形で配布されました。











また、何人かの生徒と直接会うことができ、学校の主催者に個人的に質問する機会がありました。 発表が比較的遅いように見えたという事実（学期の終わり、多くはセッションに行った）にもかかわらず、私たちは多くのアプリケーションを受け取りました。 競争は1箇所あたり約5人で、合計約150の応募がありました。



参加者の競争力のある選択



多数のアプリケーションでは、かなり短い期間で参加者を効果的に選択する方法の問題が自然に発生しました。 長時間の議論の後、私たちは、アンケートを100回読むよりも一度見て、話すほうがよいという原則に完全に沿って、対面インタビューが最良の形式であると判断しました。



ItsezのエンジニアとUNNの教師で構成される2つのフル稼働日、3つの委員会が、150人の候補者全員にインタビューしました。 残念ながら、すべての候補者がインタビューで提案されたタスクに完全に対応したわけではありません。 繰り返しますが、コンピュータービジョンの分野で作業するには、数学とコンピューターサイエンスの分野で優れた基礎知識が必要です。 したがって、私たちが尋ねた質問は、主に1〜2年の専門技術のカリキュラムから借りたものです。 数学とプログラミングの2つのグループの質問がありました。 最初の部分では、主に線形代数、解析幾何学、離散数学、確率理論などの古典的なコースの知識をテストしました。 プログラミングでは、質問は主にC / C ++およびOOPにありました。 質問によって応募者は「行き止まり」に陥ることがありますが、正しい判断につながる一連の思考が生み出されることもあります。



決定を下すことは本当に困難でした（学校とインターンシップの場所の数が限られていました）。誰も気分を害することなく、必要な結論を下し、将来的には100％準備が整うことを心から願っています。 道路が引き継ぐため、私たちはあなたを再び待っていることを忘れないでください！ 新しいプログラムに応募するためのリンクが以下にあります。



また、良いニュースは、将来、いわゆる「無料リスナー」のカテゴリを拡大することを決めたことです。 実質的に誰でも、競争を迂回して、講義に参加し、自宅で独立してタスクを実行することができます。 もちろん、原則として、教師とのフルタイムの仕事はより便利ですが、この形式では、学生はより柔軟に参加し、同時にレベルをチェックし、必要に応じて引き上げることができます。



一方で、主催者にとって、学校の実質的な準備の主な時期が来ました：スケジューリング、講義のためのプレゼンテーションの準備、および実践的なクラスのための教材の開発。 すべての条件と参加者を作成するための多くの雑用と同様に、参加者はインターンシップと学校での滞在を通して快適に感じることができました。 それで、何がそれから来たのか。



インターンシップ



インターンシップは主に上級生、学部生、卒業生を対象としています。 主な目標は、「戦闘」ITプロジェクトでのその後の雇用に備えることです。 Itseezの従業員の指導のもと、実際のプロジェクトで2か月（7月から8月）集中的に作業したことにより、コンピュータービジョンの分野の実際のタスクに触れることができました。 今年は合計13人のインターンがいました。









インターン



各研修生は、インターンシップ期間中、正式に雇用されており、初心者の研究開発者の役割を試すことができました。 業界ではよくあることですが、機密性の理由から、プロジェクトの内容に関する詳細情報をここに掲載することはできません。 典型的なプロジェクトは、ある種のコンピュータービジョンアルゴリズムを説明する最新の記事の実装であるとしか言えません。 彼らは、画像上の線やテキストの検出、人体の位置の評価、カメラのキャリブレーション、セグメンテーションタスクなど、多くの基本的なタスクの解決に従事していました。 インターンが作成したデモの例を次に示します。

また、OpenCVライブラリの関数とOpenVX標準を例として使用して、パフォーマンスの最適化に専念した人もいました。 将来の投稿の1つで、これらのタスクの1つについて詳しく説明しようとします。



作業の結果は、Itseez内部セミナーの一部として発表され、メンターや同僚から高く評価されました。 多くの人が会社で面白い仕事のオファーを受けました。 以下では、インターンシップに関する参加者からのフィードバックを読むことができます。



パベル（ロバチェフスキーにちなんで命名されたNNGU、IITMM / VMK、卒業生）

「何よりも、会社は雰囲気が好きでした。 取締役からスタッフまで、会社のすべての従業員は常に友好的で親切です。 小さなプロジェクトがあります。 したがって、プロジェクト内のコミュニケーションに問題はありませんでした。 各チームメンバーは、常に支援や提案を行う準備ができています。 このインターンシップは、ITおよびコンピュータービジョンでのキャリアを始めるのに理想的です。 また、上級生は、研究を中断することなく、産業用コードと複雑なシステムを開発する実践的なスキルを身に付け、若い従業員に必要なIT企業の仕事と組織の原則を理解することができます。







ミハイル（ロバチェフスキーにちなんで命名されたNNSU、IITMM / VMK、6年）

「経験豊富なキュレーターの指導の下で、また理想的な状況で興味深い課題を解決する手段とともに、インターンシップは、仕事を本当に愛する人々を観察するための別の非常に貴重な機会を与えてくれました。 これらの人々は、文字通り、情熱、創造的な精神、そして勤勉に感染しています。 彼らは私がアイデアを形作り、技術的な困難を克服するのを助けてくれたので、彼らにとても感謝しています。 会社の内部の雰囲気は特別です。 セミナーでは、大家族の雰囲気が支配します。 このような環境での高度での労働の組織化は、非常に迅速に進行できます。 貴重な経験と楽しい時間を過ごしてくれたItseezに感謝します。」





ニコライ（ロバチェフスキーにちなんで命名されたNNSU、IITMM / VMK、4年）

「私にとって、これはプログラマーとしての最初の経験ではありませんが、大量の研究作業がここに追加されました。 驚いたことに、私は大学の最初のコースから一見しっかりと忘れられていた数学を思い出しました。新しい経験はアルゴリズムを作成する際の実際の応用であり、実際の画像で実際の結果を見るのは印象的でした。 一般に、コンピュータービジョンは、理論と実践が非常に密接な分野の1つであり、数学的に導き出された計算がすぐに実装および検証されることを嬉しく思います。







ローマン（NNSU、IITMM / VMK、卒業生のロバチェフスキーにちなんで命名）

「私のプロジェクトのスタッフの中で、彼らの真の熱意と決意に惹かれました。 私は有望なプロジェクトに参加し、昨日は素晴らしいと思われた新技術の市場への直接参加者になるチャンスがあります。 グローバルレベルでプロジェクトを行っている会社で経験を積んだり、仕事をしたりしたい場合は、難しいタスクを恐れずに、大学の知識の応用を探してください。





学校



インターンシップとは異なり、 学校は短いイベントです。 2週間、男たちに高いレベルの集中​​力と集中力が必要でした。 22人の参加者と無料のリスナーがいました。









夏の小学生



最初の週の講義と実習の厳しいスケジュールにより 、1分間リラックスすることができませんでした。 最初の週の各日は個別のトピックに専念しました。

  • 月曜日-プログラミングツール：Git、GitHub、CMake、Google Test
  • 火曜日-OpenCVコンピュータービジョンライブラリの紹介
  • 環境-画像内のオブジェクトの検出
  • 木曜日-ビデオ上のオブジェクトの追跡
  • 金曜日-パフォーマンスの最適化への小さな余談


Itseezのエンジニアによる2回の午前の講義では、実用的なタスクの実装に必要な理論的基礎が提供されました。 時々、男たちは練習に十分な時間を割り当てられませんでした（大学のカップル2人）が、疲れているにもかかわらず、ターミナルクラスに残って決定を出しました。



学校の資料はGitHubに投稿され 、誰でもプレゼンテーションを読んだり、実用的なタスクを完了したりできます。 私たちは、資料にできるだけアクセスしやすく、オープンにして、段階的な指示を提供しようとしました。 誰かがエラー、間違いを見つけた場合、または単にジョブの説明を改善し、GitHubでプルリクエストとして送信することを決定した場合、私たちは幸せになります:)私たちの学校の最初の実践的なレッスン！
