GRUB全能、または起動可能なUSBフラッシュドライブの作成

grub自体については、ブータブルフラッシュドライブ、特にディスクを作成するための特別なユーティリティがたくさん書かれていますが、焦点が絞られているように見えます。 たとえば、 Windows 7フラッシュドライブを作成するためだけに、 このトピックのように、さまざまなPEtoUSBがより汎用的です。 一連のユーティリティは、基本的にブートローダーgrub、isolinux（あまり頻繁にBCDWまたはlilo、Win7のように少ない）をインストールし、提供するファイル（Windows7、WindowsXP、Ubuntuなど）をフォルダーにレイアウトします。 ここから、画像への厳密さ（ このトピックに関するコメントで書いたように）。



しかし、その一方で、インターネットにインストールされたドライブ（特にUSB）の大部分はisolinuxとgrubを使用していますが、それらが単純にマージできないのはなぜですか？ 何もない！



特定の配布キット用に調整されたユーティリティを使用せずに、ブート、インストール、およびライブフラッシュドライブをすべて1つで作成する方法を例を挙げて説明します。 また、必要に応じて、再フォーマットせずにUSBフラッシュドライブに他の何かを簡単に追加できます。



（C） そこからの写真



デモンストレーション



Linuxユーザーがgrub-install / dev / sdbコマンドの使用方法を説明する必要はなく、 grubの利点は明らかです。 ddのようなコメントを予想して、この記事は主にWindowsユーザーを対象にしていると言いたいです。

Windowsユーザーの場合、grubの利点を説明しようとします。grubを最初に保持し、次に2番目のブートローダーを確実に保持する必要があると確信できることを願っています（grubを備えたフラッシュドライブは間違いなく必要です）。

まず、MBRを上書きせずに試すことができます。 boot.iniに grubを書くだけで十分です：

GRUB4DOSアーカイブのファイルが必要です。ルートC：\にコピーします。FATである必要はありません。NTFSでも動作します。

default

grldr

grldr.mbr

menu.lst







Windows XPの場合、 c:\grldr="GRUB4DOS"



をboot.iniに追加します 。 それだけです

または、アプリケーションのファイルからGRUB \ install.batを実行します（ install.batファイルはxPUD ©インストーラーから取得され、grubを追加するだけで、何もコピーしません）。



Vistaおよび7のユーザーの場合、インストールはXPよりもやや複雑です-これはbcdeditを使用して行われます。



バックアップする

bcdedit.exe /export "C:\BCD.bak"





bcdedit.exe /create /d "GRUB4DOS" /application bootsector





「{1a5b5afd-0469-110d-9a85-000103005000}」という形式のGUIDを覚えてから、このコマンドのGUIDをこの行に置き換えます



bcdedit.exe /set GUID device boot

bcdedit.exe /set GUID device partition=%SYSTEMDRIVE%

bcdedit.exe /set GUID path \grldr.mbr

bcdedit.exe /displayorder GUID /addlast







または、アプリケーションのファイルからGRUB \ install.batを実行します。



一般に、これでインストールは終了し、すでにリブートしてコマンドラインで遊ぶことができます（grubメニューが表示されたらCを押します）。



たとえば、ドライブからダウンロードを開始できます。

chainloader (fd0)+1

rootnoverify (fd0)

boot







CDドライブから：

cdrom --init

map --hook

chainloader (cd0)

boot







再起動：

reboot







コンピューターの電源をオフにします。

halt







事前に挿入されたフラッシュドライブから起動できます（もちろん、何らかの種類のブートローダーが存在しない限り）。

map (hd1) (hd0)

map (hd0) (hd1)

root (hd1,0)

chainloader +1

boot







そして、Windows 7のインストールを開始できます。そのイメージは、USBフラッシュドライブに解凍されただけです（フラッシュドライブ自体は起動できません）。

root (hd1,0)

chainloader /bootmgr

boot







これをすべてコマンドラインモードから実行したことを思い出させてください。 ただし、1分の1（最後のboot



コマンドのみを削除）、これらのコマンドはmenu.lstにも記述でき、メニューから機能します。



人生の事例：最初のディスクのMBRが点滅し、このBIOSですぐに2番目のディスクからgrubをロードしました。 おなじみのブルースクリーンを見て、MBRは別の実験の後、破損しており、復元する必要があることに気付きました。 私に必要なのは、コマンドラインモード（ C ）に切り替えて入力することだけでした。

root (hd0,0)

chainloader /bootmgr

boot







そして、Windows7の起動が始まりました。起動後、 bootsect / nt60 c：/ mbrを実行しました。これですべてです（！）

セーフモードやリカバリモード、 2、3回の再起動 、およびリカバリディスクはありません（また、ネットブックにCDドライブがありません）。 手順の後、再起動する必要さえありませんでした。Windowsは通常どおり起動しました（一度、このように1週間起動したとき、ロード後にMBRを復元するのを忘れていました）。

また、GRUB4DOSアーカイブには、Windows XPおよびWindows 7 / Vistaの自動検索の例があり、Windowsがインストールされている場所がわからなくても、飛行したMBRで起動できます。



設置



ですから、あなたはインスピレーションを受けて自分でそのようなフラッシュドライブを作りたいと思っていますが、すでに16GBのフラッシュドライブがあり、それをフォーマットすることは非常に望ましくなく、ブートフラッシュドライブを作成するためのほとんどの一般的なユーティリティ（および手順）は事前にフォーマットすることを提供していますので、作成されたフラッシュドライブが正確にFAT32ファイルシステムであることを確認してください。 そして、そこにgrubまたはisolinuxを再度インストールします。



フォーマットせずにgrubをインストールできます（もちろん、既にFAT32がインストールされていて、その大部分はFAT32です）。 Winrubユーティリティ（GUI愛好家向け）があり、 GRUB4DOSアーカイブにはbootlace.com があります 。 WinGrubオプションを検討してください 。 以前は、％programfiles％に自分自身を解凍し、デスクトップ上にショートカットを作成するsfxアーカイブでしたが、これは好きではありませんでした。 自分で解凍してからgrubinst_gui.exeを実行するだけで十分です 。 私が記事を書いて、最新バージョンをダウンロードしたとき、インストーラーは多かれ少なかれ本格的です。 ダウンロード、設定。 または、記事の最後にある添付ファイルを見て、アーカイブごとにアセンブリ用のキットをレイアウトします。 起動し、（サイズに基づいて）USBフラッシュドライブを選択し、パーティションリストでMBRまたはディスク全体を選択します。 インストールをクリックします。 コンソールウィンドウ「 MBR / BSが正常にインストールされました 」が表示され 、どれを閉じるかをクリックした後、Enterを押すように求められます。 「 grubinst：ディスクイメージである必要があります」と書いた場合は、[フロッピーイメージ]ボックスをオンにして、もう一度[インストール]をクリックします。







これで、GRUB4DOSがMBRに登録されました。

その後（可能性はありますが）、GRUB4DOSアーカイブからフラッシュドライブのルートにファイルをコピーします。

default

grldr

grldr.mbr

menu.lst





menu.lstを自由に編集すると同時に、フラッシュドライブをいっぱいにします（いくつかのポイントは既に上記の記事で確認でき、アーカイブ自体には例が含まれています）。



埋める



1. アルキッドライブ

完全な配布には、multiboot.7zアーカイブがあります。grubのサンプルファイルがあり、そこから取得して、ISO自体をルートに解凍します。 i386をminintに名前変更します 。開始しないことを忘れても、 minintが見つからなかったと言います（それほど明白ではないヒントになりました）。

menu.lstに追加

title Alkid Live CD

root (hd0,0)

chainloader (hd0,0)/minint/setuplns.bin







プログラムフォルダで、フラッシュドライブがそれほど大きくない場合は、余分なものを削除できます。 また、ウイルス対策データベースを現在の状態に更新します。



2. HirenのBootCD

ISOを開き、isolinuxブートローダーの設定がHBCDフォルダーにあることを確認します。

HBCDフォルダーをフラッシュドライブのルートに解凍し、 isolinux.cfgを開きます 。

menu label Start BootCD

kernel /HBCD/memdisk

append initrd=/HBCD/boot.gz







grub構文に簡単に変換できます

title Start BootCD

root (hd0,0)

kernel /HBCD/memdisk

initrd /HBCD/boot.gz







menu.lstに追加します

インストールが完了しました。



3.いくつかの小さなユーティリティはすでにインターネット上で準備ができています。 通常、memtest、Ghost、DOS、PQMagicを含むフロッピーディスクイメージ。 それらからの読み込みも難しくありません。 menu.lstに書き込みます 。

title Ghost 11

map --mem /Ghost.ima (fd0)

map --floppies=1

map --hook

chainloader (fd0)+1

rootnoverify (fd0)







私はCDドライブのないEEEPCを持っていますが、突然それを復元することが必要になったとき、外付けドライブをどこで入手するかさえ考えませんでした。 Ghostイメージをdsik DVDから「大型」コンピューターのUSBフラッシュドライブにコピーし、USBフラッシュドライブから起動して復元しました。



4.配布キットを開始します。 Parted Magicのイメージを例に取ります。 USB用のイメージをダウンロードします。 すでにgrubにあり、さまざまなブートオプションの非常に大きなメニューがあります。 それをインストールするには、アーカイブを解凍し、 pmagicフォルダーをルートに配置し、 menu.lstファイルをboot \ grubから\ pmagicにコピーして、 grossに追加します。

title Pagic

configfile /pmagic/menu.lst







このコマンドはpmagicのネイティブmenu.lstからメニューをロードします。何も作成する必要はありません。 \ pmagic \ menu.lstに追加しない限り、次のものに戻るための項目：

title <<main menu

configfile /menu.lst







PMagicの最新バージョンでは、以下を含む多数のユーティリティが追加されました。 memtest、grubリカバリ、ハードウェアアナライザー。 USBフラッシュドライブに「再配置」して、 p。3またはmenu.lstを覗いて、個別に練習することをお勧めします。



5. Live Ubuntuを起動します。 「 ubuntu-9.10-desktop-i386.iso 」 があったので、例を使って見ていきます。 また、ルートに解凍します。 isolinuxフォルダー（後で削除できます）のtext.cfgファイルで、ライブモードの開始方法を見つけ、それに応じてmenu.lstに行を追加します（そこに何があったか、何が起こったのかを比較します）

title Try Ubuntu without any change to your computer

root (hd0,0)

kernel /casper/vmlinuz file=/preseed/ubuntu.seed boot=casper

initrd /casper/initrd.lz







6. Debianのインストール（Ubuntuも同じ方法でインストールします）：

これを行うには、リポジトリからのHD-Mediaファイル（またはグラフィカルインストーラー ）とDVD1 ISOイメージが必要です。 これをすべてルートに追加し、grubに追加します。

title Debian Install

kernel /vmlinuz

init /initrd.gz







インストールを開始すると、ISO DVD1イメージが検出され、ネットワークなしで、CDドライブなしでインストールが続行されます。

おわりに



例が誰かがそれを理解するのを助け、誰かがこのブートローダーに興味を持ち、ブータブルフラッシュドライブを作成するためのユーティリティが笑顔になることを願っています（私は誰かを怒らせたくない、私はすべての開発者を尊敬し、さらに私はそのようなユーティリティで始めました）。 もちろん、grub自体とGRUB4DOSをより詳細に学習することを強くお勧めします。たとえば、後者のロシア語ガイドを次に示します。

別の配布キットまたはユーティリティを追加する前に、フォルダーが一致していないことを確認してください。 Linuxディストリビューションの一部は、器用さとgrubの優れたコマンドを使用して、フォルダーの名前を簡単に変更できます。 特に、pmagic、puppy、xpudのように大きくありません。

一般的に、実験することを恐れず、幸運を祈ります！
