Hyper-Vを開始するための10のヒント

通常、Hyper-V環境のインストールは基本的に複雑ではなく、時間もかかりません。 ただし、そのセットアップでは多くの質問が発生する可能性があるため、細心の注意が必要です。 したがって、本日は、Hyper-Vを適切に構成する方法に関する有用なヒントを提供したいと思います。 アプリケーションの順序は任意です。













1.更新プログラムをインストールしてから、今後Hyper-Vを更新する方法を決定します



インストール方法に関係なく、Hyper-Vサーバー上のコンポーネントを更新する必要があり、その方法を決定する必要があります。 ホストの再起動が必要な更新は、ホストで実行されている仮想マシンを一時停止し、ホストが再起動されるとすぐに作業を再開します。 このオプションが受け入れられない場合、仮想マシンの転送、SCVMMの使用、更新のための時間の割り当てなどの可能性を考慮する必要があります。









2.ドメインを決定する



他のすべてが配置されている同じActive DirectoryドメインにHyper-Vホストを追加するかどうかについては、さまざまな観点があります。 Hyper-Vホスト用に別のドメインを設定するユーザーもいれば、Hyper-Vホストをドメインに入れないユーザーもいます。 リスクを評価し、ドメイン管理機能と考えられる障害シナリオを検討して、何よりも仮想環境の評価に基づいて決定を下します。つまり、普遍的な答えはなく、自分で解決する必要があります。

さらに、この時点で、適切なHyper-Vホスト名を考え出す必要があります。 Hyper-Vホストでの作業中にエラーが発生することはないため、サーバーの命名法で際立っている名前を選択することをお勧めします。 名前を混同し、稼働日の真っConf中にあるメールサーバーとファイルサーバーの仮想マシンでホストを再起動することは、仮想化が導入される状況ではありません。









3.ストレージを構成する



MPIOおよびNVAアダプタードライバーまたはSANソフトウェアが必要な場合は、Hyper-Vの役割を追加する前にこれをすべてインストールしてください（これはWindows Server 2012のインストールスクリプトです）。 仮想マシンを使用するまでに、これらのコンポーネントはすべてすでに動作しているはずです。









4.管理者アカウントを設定する



他のすべてのWindows Serverサーバーと同じ管理プールを使用したいですか？ 次に、段落2を読み直します。









5.ネットワークインターフェイスのフレンドリ名を選択する



仮想環境を正しくセットアップすると、ホストに複数のネットワークインターフェイスが存在します。 それらがすべて「ローカルエリア接続」または「ローカルエリア接続2」と呼ばれる場合、これはあまり便利ではありません。 さまざまな種類のインターフェイスがある場合（たとえば、BroadcomおよびIntelデバイスに基づいている場合）、そのインターフェイスの位置と場所を思い出すことができます。 しかし、あなたの記憶だけに頼る価値はありません。 各インターフェイスに「ホストLAN」、「制御LAN」などのわかりやすい名前を付けます。通信チャネルのタイプを指定することもできます。ケーブルの場合は「K」、複雑なコンポーネントの場合は「C」などです。









6.不要なプロトコルを無効にします



Hyper-Vホストがデータセンターで実行されている場合、Windowsのほとんどのピアツーピアネットワークテクノロジ、特にIPv6はほとんど使用しません。 すべてが非常にシンプルである必要があります。インフラストラクチャを簡素化し、IPv6、データリンクトポロジサービスなど、一般的に使用しないすべてのコンポーネントを無効にする価値があります。













7.必ずWindowsをアクティブ化してください



使用するアクティベーション方法に関係なく、キーマネージメントサーバー（KMS）、マルチプルアクティベーションキー（MAK）、Windows OEM、またはその他、システムのアクティベーションの提案によりWindowsが中断されないようにしてください。コンソールを使用します。 Microsoftの無料ハイパーバイザーであるHyper-V Server 2012を使用する場合、この点はそれほど重要ではありません。









8.リモートおよびローカル管理を構成する



これはMicrosoftの推奨事項の1つであり、必要ですが、多くの場合無視されます。 新しいサーバーマネージャー、PowerShell、およびHyper-Vマネージャーを使用すると、サーバーに直接ログインしなくても、多くの操作をリモートで実行できます。 ただし、自分にとって便利なツールを選択する必要があります。 たとえば、Windows Server Core with Hyper-VまたはHyper-V Server 2012など、単一のコマンドラインで共通コンソールを使用する必要がある場合でも、リモートデスクトップモードでサーバーにログオンすることを好む人もいます。









9.仮想マシンとディスクへのデフォルトパスを設定します



ローカルドライブ上の仮想マシンに出くわすよりも厄介なことは何ですか？ 奇妙なことに、これは管理エラーが原因で発生する場合があります。 仮想ハードドライブと仮想マシンの設定で正しい場所を指定してください。 C：\ドライブを埋めてホストが正常に動作しなくなると、何も期待できません。したがって、上記のオプション（図2のE：\ドライブ）が最適でなくても、ドライブがいっぱいの状況では、なお良いです。













10.フリーランスアクセスを確認する



このヒントは、Hyper-VとvSphereの両方に関連しています。 問題が発生した場合は、システムに接続できることを確認してください。 KVM、Dell DRAC、HP iLOなどのツールが役立ちます。









これらは、仮想環境での経験に基づいた、最も一般的な性質のヒントですが、他にも便利なトリックがたくさんあります。 実際、経験を積む最良の方法は、システムを展開し、インストール後に変更する必要があったものを正確に修正することです。
