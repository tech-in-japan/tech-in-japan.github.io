グラフィックカードを備えた仮想サーバー

2016年10月21日から、 UltraVDSにより、リースされたサーバーにビデオカードを追加できるようになりました。 このような機能は、更新されたHyper-V仮想化環境を含むWindows Server 2016のリリースで可能になりました。











Microsoftの新しいハイパーバイザーであるRemoteFXを使用すると、サーバーバージョンのWindows 2008および2012よりもはるかに広範囲に割り当てられたビデオカードのリソースを管理できます。



Microsoft RemoteFXは、RDP（リモートデスクトップ）プロトコルの視覚体験を向上させるために設計された商標および技術です。



歴史への短い遠足：



RemoteFXは、 Windows Server 2008 R2 SP1のリリースで初めて登場し、次の機能が含まれました。



  • RemoteFX vGPU-GPU仮想化とWindows 7を実行する複数のマシンへの配布。これにより、仮想サーバーは複雑なグラフィックシナリオ（3Dレンダリング、そしてもちろんゲーム）の処理にハードウェアアクセラレーションを使用できました。
  • RemoteFX USB-USBデバイスをサーバーで実行されている仮想マシンにリダイレクトします。
  • RemoteFXコーデックは、ビデオとテキストを再生するときに仮想マシンの処理能力を使用してエンコードする単純なコーデックです。


Windows Server 2012のリリースでは、次のコンポーネントがRemoteFXに追加されました。



  • DirectX 11をサポートするためにRemoteFX vGPUが更新されました。
  • RemoteFX Adaptive Graphics-さまざまな作業環境条件に適応するために学習したリモートマシンから画像を転送します。グラフィックコンテンツの種類、プロセッサパフォーマンス、およびネットワーク帯域幅。
  • RemoteFX Multi-Touch-同時に256のタッチポイントを持つマルチタッチをサポートするタッチスクリーンを介したリモートインタラクションのサポート。


Windows Server 2016では、更新されたRemoteFXメディアストリーミングが、マルチメディアリダイレクトによって以前に完全または部分的に使用されていたすべての種類のビデオコンテンツに完全に置き換わり、以下も表示されました。



  • OpenGL 4.4およびOpenCL 1.1 APIのサポート。
  • サーバーで実行されている各仮想マシンに割り当てられたビデオメモリの高度な管理。


サーバービデオカードをクラウドに接続する新しい機能により、UltraVDSは家庭用または職場用のデスクトップコンピューターに代わる真の選択肢となりました。
