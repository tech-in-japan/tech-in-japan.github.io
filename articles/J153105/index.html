Opera 12.10ベータ-あらゆる好みに対応したAPI





本日、Opera 12.10の最初のベータ版がリリースされました。 通常、私たちは常にブラウザーのユーザーに楽しい驚きを与えようとしますが、今日では開発者にも多くの便利な革新を用意しています。 Operaの拡張機能の開発者とユーザーの両方にこの革新がアピールされることを願っています。



コンテキストメニューAPI



これで、Opera拡張機能の開発者は、ブラウザのコンテキストメニュー（ユーザーには右クリックメニューとも呼ばれます）を変更できるようになります。 新しいAPIのおかげで、ソーシャルネットワーク、オンライン翻訳者、その他の多くの一般的なサービスでの作業を簡素化する新しい機能をコンテキストメニューに追加できるようになりました。



スクリーンショットAPI



スクリーンショットを撮るのは必ずしも簡単ではありません（キーボードの[画面の印刷]ボタンがどこにあるか誰もが知っていますか？）、そして友人とスクリーンショットを共有することはさらに困難です。 APIを使用して拡張機能の開発者がスクリーンショットを作成することにより、Operaユーザーはブラウザウィンドウに表示されているものを友人に簡単かつ迅速に表示できます。



データ読み込みAPI



このAPIにより、Opera拡張機能は独自のCSSとJavaScriptを使用して、Webページの機能とデザインを改善または変更できます。 たとえば、Comic SansファミリーのフォントをHelveticaに置き換えたり、表示するすべてのWebページで高度なJavaScript機能を使用したりできます。



再び基準



これは、APIに関するすべてのニュースではありません。 サポートされるWeb標準の数を増やし、フルスクリーンAPIとページ可視性APIのサポートを追加しました。これにより、開発者はページコンテンツの表示をより適切に制御し、ブラウザーリソースの使用を最適化できます。



Webソケットを使用する開発者にとっては、良いニュースもあります。 Opera 12.10には、開発者向けの拡張機能とエンドユーザー向けの改善された機能を備えたWebソケットの最新の実装のサポートが含まれています。 そして、このサポートはデフォルトで有効になっています。



SPDY



パフォーマンスと速度の改善の長いリストの先頭で、SPDYネットワーク標準をHTTPプロトコルへの追加として提示します。これにより、Webページの読み込みが大幅に高速化されます。 Opera 12.10では、SPDYサポートはデフォルトで有効になっており、この標準をサポートするWebページで動作します。 たとえば、TwitterやGmailなどの大規模なWebサイトは、従来のHTTPプロトコルに加えてSPDY標準をサポートしています。



変更の時間



WindowsとMacの両方の世界で、この秋にオペレーティングシステムの具体的な変更が行われました。 Opera 12.10の新バージョンには、これらのオペレーティングシステムの新機能をより完全に使用できるようにするいくつかの革新が含まれています。



Mac OS X Mountain Lionのリリースにより、多くの新機能がシステムに直接組み込まれました。 Opera 12.10 for Macは、通知センター、Retinaディスプレイ、ソーシャルネットワーク上にコンテンツを公開する組み込み手段などの革新をサポートしています。



間もなくWindows 8が一般公開され、Opera 12.10にはこのOSの一部の機能のサポートが含まれます。 特に、Windows 8クラシックモードのタッチコントロールのサポートを追加し、Windows 7のボーナスとして同様のサポートを追加しました。
