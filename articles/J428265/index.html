WinCEデスクトップにアクセスし、Keysight DSOX1102GオシロスコープでDoomを実行します

Jason Ginのブログの記事「 Tear to pieces 」の翻訳



TL; DR：はい、Keysight 1000 Xシリーズオシロスコープでは、実際にDoomを実行できます！ ただし、これは簡単ではありません。





Keysight DSOX1102G



オシロスコープの武器庫には、自尊心のある電子愛好家が必要です。 要するに、オシロスコープを使用すると、回路内の電気信号の波を調べることができます。デジタルオシロスコープ（DSO）は、アナログオシロスコープまたはマルチメータが認識しない信号のまれなエラーを見つけることができるため、不可欠です。



私の記事のトピックは、競合他社と比較して価格品質比が優れている安価なオシロスコープのKeysight Technologies（旧Agilent）のDSOX1102Gです。 ほとんどのオシロスコープと同様に、 Windows Embedded CE 6.0 （Windows CEまたはWinCEとも呼ばれます）はこのモデルで実行されますが、WinCEのほとんどの場合と同様、インターフェイスはほとんど表示されません。ユーザーインターフェース。



ステージ1：覚醒



2017年の初めにKeysight 1000-Xシリーズが発売されたとき、Hackadayのレビューアーの1人は、これらのオシロスコープにデータを保存する方法がクラッシュやリブートにつながることがあることに気付き、ロードする前に数秒間マウスポインターが画面に表示されることに注意しました。 彼の投稿にはgifがあり、クラッシュを引き起こすファイルを保存しました。アニメーションのフレームの1つで奇妙なことに気付きました。そこで、エラーのある黒い画面のすぐ前にあるWindowsタスクバーを区別できました。 面白い！..







KeysightのScope Monthコンテストのおかげでオシロスコープに勝ちましたが、数か月間、自分で画面が故障するまでは考えませんでした。 私の場合、Windows CEヘッダーがオシロスコープの障害ハンドラーの上に表示されていることがわかりました。 ヘッダーをドラッグするとトレイルが残り、WinCEがハングアップしました。 これは非常にまれにしか発生しなかったため、その後障害が発生したため、ハンドラーにファイルシステムのスキャンとOSの再起動を許可しました。



しかし、私はこれに興味をそそられ、基盤となるWinCEシステムで何が起こっているのかについて多くを知りたいと思いました。 オシロスコープのUSBポートは非​​常にエラーが発生しやすく、フラッシュドライブをポートに挿入するだけでクラッシュする可能性があることがわかりました。 ただし、これは信頼性の低い方法であるため、必要な量の情報を収集するには不十分でした。



それで、WinCEデスクトップへのアクセスの検索を開始しました。



最初は純粋なソフトウェアソリューションを試して、.ksxファームウェアアップデートファイル（実際は.cabアーカイブ）を作成しようとしました。これは、オシロスコーププログラムをシャットダウンしてWindowsエクスプローラーを開くことができました-動作しませんでした。 オシロスコープソフトウェアは、ファイルを開くことができないというエラーメッセージを発行しました。 更新中はオシロスコープソフトウェアがデスクトップに移動しないため、更新ファイルを強制的にダウンロードしても、このようなソリューションは機能しません。 この最初の深刻な障害に直面して、私は一時的に好奇心を捨て、意図した目的でオシロスコープを使用しました。



ステージ2：詳細を調べる



私の好奇心から、オシロスコープがUSBポートを介して3.5インチのフロッピーディスク（または、若者が言うように印刷された保存アイコン）を読み書きできるかどうかを確認することにしました。 Eurekaの電源を入れたときにUSBポートにドライブを置いたままにするとクラッシュしました！確実にクラッシュを引き起こす方法を見つけました。



ここで、残念ながら、2番目の重大な障害が私を待っていました。 この起動エラーは、USBポートに1つのデバイス（ドライブ）が含まれている場合にのみ発生しました。 ドライブを含むUSBハブを使用した場合、エラーは発生しませんでした。 つまり、キーボードとドライブとUSBマウスを非常にすばやく切り替える必要がありました。 起動プロセス中に、できるだけ早くドライブを取り外して、キーボードとタッチパッドの組み合わせをUSBに挿入するのは面倒で面倒でした。 より良いソリューション-ハードウェアソリューションが必要でした。





USB専用のA / Bスイッチ、デバイスの迅速な交換用に私が作成



古いUSBケーブル、デッドUSBハブ、およびDPDTスイッチを使用して、USB A / Bスイッチを作成し、デバイス間の切り替えプロセスを簡素化および高速化しました。 この方法を使用して、タスクバーが画面に表示されている間に、フォールトハンドラーがすべてのラズベリーを壊すまで、WinCE OSとの接触を一瞬確立しようとしました。 Samsung Galaxy S9のスローモーションシューティングの魔法を使用して、WinCEにキーストロークを送信し、システムのスクリーンセーバーでもキーストロークを処理できることを確認できました。 盲目的にキーを押してから、システムに関する情報を取得することができ、その後、オシロスコープソフトウェアに緊急の障害が発生した場合の応答を調査しました。 それから私は第二の障害にぶつかりました。



WinCEとの非常に短い対話の可能性は良好ですが、フォールトハンドラーがシステムを再起動するまで制御できなかったため、役に立たなかった。 ハンドラーはOSに対して高速に保持され、キーボードスワイプやCtrl + Alt + Deleteを使用せずにWinCEに戻ることができました。



ステージ3：足場を探す



繰り返しますが、古いSony Clie PEG-NX73V （2003年からPalmOSでハンドヘルド）をUSBドライブとして使用することに決めたとき、私の定期的な好奇心が重宝しました。 リムーバブルディスクと同じ方法で、ファイルをメモリカードにドラッグアンドドロップできるデータインポート機能がありました。



USBドライブの場合と同様に、PDAを抜かずにオシロスコープの電源を入れると、システムクラッシュが発生しました。 ただし、今回は障害ハンドラーはPDAファイルシステムが破損したファームウェアセクションであると判断し、外部USBフラッシュドライブからファームウェアアップデートをダウンロードすることを提案しました。







この動作は規則的ではなく、時々オシロスコープソフトウェアがロードされ、WinCEから非常に奇妙なウィンドウが現れ、画面に痕跡が残る明るい青色のマウスカーソルが表示されました。 しかし、この奇妙な状態で、私はInfiniiVisionオシロスコープソフトウェアウィンドウを何とか横に引いて、OSから突き出そうとしました。 しかし、オシロスコープソフトウェアは非常に積極的に動作し、ウィンドウの外側をクリックするたびにフォーカスを取り戻しました。 奇妙にペイントされたOSで大騒ぎした後、私はそれを回避することができました。 オシロスコープソフトウェアウィンドウから長時間制御できなかったため、ファイルシステムを表示できませんでしたが、システムプロパティダイアログボックスを呼び出すことができました。このダイアログボックスには、オシロスコープがWindows CE 6.00に基づいており、100 MBのRAMが搭載されています。







それから、EEVブログフォーラムを調べてみることにしました。そこでは、人々がオシロスコープをハッキングして、さらなる可能性を広げようとしています。 そこで、ソフトウェアがUSBフラッシュドライブのルートにあるinfiniiVisionStartupOverride.txtファイルを探しており、それが見つかった場合、そこからオシロスコープソフトウェアをダウンロードしようとすることがわかりました。 そして、どうやら、ファームウェアが実際にフラッシュドライブからソフトウェアをダウンロードしなかったようですが、このプロセスはオシロスコープソフトウェアの起動を中断し、だれも私からOSの制御を奪いませんでした。 その瞬間から、すべてがより興味深いものになりました-フォールトハンドラーがエクスプローラーウィンドウを開き、「*。*」と入力しました。ファイル名フィールドに、オシロスコープのファイルシステムとUSBフラッシュドライブを調べ始めました。 これは、まさにWinCEを制御するために必要なものです。 しかし、別の障害に遭遇しました。60秒後にオシロスコープが再起動したため、オペレーティングシステムを掘る時間があまりありませんでした。





フラッシュドライブからファイルを選択するためのダイアログを含むファームウェアアップデートDSOX1102Gのリクエスト





Windows CEタスクマネージャーは、ファームウェアの回復画面に実行中のプロセスを表示します



Windows CE Task ManagerなどのいくつかのWindows CEツールをコピーすると、フォールトハンドラーがまだ表示されているときに開始された2つの興味深いプロセスに気付きました。recoverInfiniiVision.exeとprocessStartupFolder.exe。 どうやら、最初のものは、オシロスコープソフトウェアのクラッシュ後にWinCEにアクセスできない障害ハンドラーでした。 iTaskMgrを使用して2番目のプロセスを破った（WinCEタスクマネージャーの無料版ではプロセスを強制終了できません）ため、オシロスコープの再起動を防ぐことができ、最初のプロセスを破って、きれいなWinCEデスクトップが表示されました。 残念ながら、タスクバーを復元できなかったため、OSの操作がかなり不便でした。



エクスプローラーを開くためにデスクトップに新しいフォルダーを作成し、ようやくオシロスコープのファイルシステムを調べることができました。 このバージョンのWinCEにはなかったテキストエディターが組み込まれたプログラムTotal Commander / CEは、多くの助けになりました。





Total Commander / CEを使用してファイルシステムを掘り下げます（タスクバーはまだありません）



ステージ4：フルコントロール



さて、スカイブルーのデスクトップにたどり着いたので、WinCEを完全に体験するには、タスクバーを復元する必要がありました。 グーグルで、Stack Overflowに登って、私はこのために小さなプログラムを投げました。 エクスプローラーから開くと、WinCEデスクトップのフルバージョンが手に入りました。 最後に、ベースOSを完全に制御できました！





自由-オシロスコープ上のフルWinCEデスクトップ！



その瞬間から、ファイルシステムをざっと調べて、そこにどんな面白いツールが見つかるかを確認し始めました。 コマンドラインインタープリターは起動したくありませんでしたが、レジストリを調べて、値が0xFFFFFFFFであるキーHKEY_LOCAL_MACHINE \ Drivers \ Console \ OutputToを見つけました。 0に変更することで、コマンドプロンプトをデスクトップに表示できるようになったので、それを行う別の小さなプログラムを作成しました。



すべてがうまくいったので、オシロスコープソフトウェアを停止するために必要なすべてのコマンド、フォルダーの開始ハンドラー、エラーハンドラー、タスクバーを復元し、コマンドプロンプトを開始できるようにするバッチファイルを作成しました。 ただし、PDAはフォールトハンドラーメニューを開く必要があったため、他のユーザーはそのような効果を再現できませんでした。



さらに調べてみると、スプラッシュスクリーンが表示され、パネルのLEDが点滅し始めると、WinCEはデバイスがソフトウェアをドロップしなくてもキーストロークの処理を開始することに気付きました。 win + Uを押すと、オシロスコープが一時停止しました。これにより、[スタート]メニューが開き、[一時停止]オプションが選択されました（その後、OSには電源オフボタンしかなかったため、制御を戻す機会がありませんでした）。 これを念頭に置いて、ファイルの名前を入力しやすいようにファイル名をa.batに変更し、win + R、コマンド\ usb \ a.batを使用して読み込み、Enterキーを押したときに実行しようとしました。 しかし、結果としてオシロスコープはスクリーン上にスクリーンセーバーを表示するだけでしたが、バックグラウンドではWinCEが生きていて、そこで何が起こっているのかがわかりませんでした。 フォールトハンドラーは、OSデスクトップのデモンストレーションに必要なコンポーネントであり、バッチファイルに数行を追加して、フォールトハンドラーを開始してから停止するだけでしたことがわかりました。 これらの最後のタッチを追加することで、USBフラッシュドライブ、マウス、キーボードのみを使用して、デスクトップが表示される直前に（半自動で）オシロスコープを読み込むことができました！



ステージ4：はい、DOOMを実行します！



WinCEにアクセスできるようになったので、ついに「Doomは起動しますか？」という質問に答えることができました。 オシロスコープを起動してから1年半かかりましたが、ついにこのマイルストーンに到達しました。





これはデマではありません。Keysight1000 XオシロスコープでDoomを実行できます！





Doom IIはDSOX1102Gで動作します！ （3フレーム/秒）





320×240ウィンドウでDoomを起動すると、ゲームに十分な1秒あたりのフレームを圧縮できました。 色のパレットを見てください！



次の記事では、この伝説的なビデオゲームをゲーム用ではない鉄片でもう少しプレイします。





320×240、256色の解像度で動作する運命！ オシロスコープで！



ダウンロードするファイル



オシロスコープで同じことを試すために必要なファイルをアップロードしましたが、それをレンガやその他の不快なものに変える責任は負いません。 これはすべてDSOX1102Gでのみテストしましたが、1000 Xシリーズの他のモデルや、ファームウェアリカバリ機能を備えた他のKeysightオシロスコープも同様に機能すると思われます。 オシロスコープのファームウェアは、すべてのWinCEがRAMで実行され、再起動後に保存されないように作成されているため、システムを破壊するOSへのすべての変更は、オシロスコープをブリックに変換しません（ファームウェアのファイルは、エクスプローラーで開くことができないNANDフラッシュドライブのディレクトリにありますが、名前でのみダイヤルできます）。



フラッシュドライブはFATまたはFAT32でフォーマットし、 Scope Liberator zipアーカイブをそのルートに解凍する必要があります。 手順はreadme.txtにあります。
