はじめに



Oracle RACのインストールのすべての手順を実行し、ラップトップまたはデスクトップコンピュータに8 GB以上のRAMがある場合、これはOracle VirtualBoxを使用して完全に実行可能です。 この記事の指示に従ってください。小さなテストデータベースをホストできる稼働中のRACシステムを入手できます。 作成されたシステムは、完成したシステムではありません。 このガイドでは、RACのインストールと使用、およびさまざまな管理手順のテストに慣れることができます。 この記事では、イメージを保存し、イメージからRACを数分で復元する方法についても説明しています。 テストシステムが「落ちた」としても、簡単に復元できます。



この記事では、64ビットバージョンのOracle Linux、バージョン6.3、およびOracle 11gリリース2、バージョン11.2.0.3を使用します。 VirtualBoxを使用すると、同じサーバー上で複数の仮想マシン（VM）を実行できます。これにより、同じマシン上で複数のRACノードを実行できます。 さらに、共有仮想ディスクを使用できます。 完成したシステムには、2つのゲストオペレーティングシステム、2セットのOracle Gridインフラストラクチャ（Clusterware + ASM）、および1つのサーバー上の2つのデータベースインスタンスが含まれます。 必要なディスク容量は約32 GBです。完成したRACのイメージを保存するには、さらに12 GBのディスク容量が必要です。



この記事は、私が最初にティムホールによって書かれた記事「 VirtualBoxを使用したLinux上のOracle Database 11gリリース2 RAC 」に触発され、彼が彼のブログで公開した後に書かれました。 その後、ほぼ完全に改訂され、改訂されました。現在、これらの記事は、ほとんど共通点がありません。



システム要件



  • 8 GBのRAM。
  • 32 GBのハードディスク空き容量。


この記事で説明するすべては、64ビットWindows 7でテストされています。ただし、他のOSおよびハードウェアソリューションでVirtualBoxを使用しても問題はないはずです。 他のOSで成功または問題がある場合はお知らせください。



必要なソフトウェア



次のソフトウェアをダウンロードします。

  1. Oracle Linux
  2. VirtualBox （バージョン4.1.14以降でなければなりません）
  3. Oracle 11gリリース2（11.2）ソフトウェア（64ビット）




仮想マシン（VM）のインストール



この例では、64ビットWindows 7にインストールされたVirtualBoxを使用します。

次に、2つの仮想RACノードを定義する必要があります。 1つの仮想マシンを定義することで時間を節約でき、すべてがインストールされたらクローンを作成します

VirtualBoxを起動し、ツールバーの「新規作成」ボタンをクリックします。 仮想マシンウィザードの最初のページで[次へ]ボタンをクリックします。

名前「Rac1」、OS「Linux」、およびバージョン「Oracle（64ビット）」を入力し、「次へ」ボタンをクリックします。





16 GBのRAMがある場合は、最大3072 MBのメモリを割り当てます。それ以外の場合は、以下のスクリーンショットのように2048 MBを使用し、[次へ]ボタンをクリックします。





「次へ」ボタンをクリックして、デフォルトの選択を受け入れて新しい仮想ハードディスクを作成します。





デフォルトのVDIタイプを受け入れて、仮想ディスク作成ウィザードの[次へ]ボタンをクリックします。





「次へ」ボタンをクリックして、デフォルトの「動的に割り当てられた」オプションを受け入れます。





デフォルトを受け入れ、サイズを「16GB」に設定し、「次へ」ボタンをクリックします。





「新規仮想ディスクの作成」画面で「作成」ボタンをクリックします。





[新しい仮想マシンの作成]画面で[作成]ボタンをクリックします。





「Rac1」VMが左ペインに表示されます。

右側の「ネットワーク」ボタンのリンクをクリックします（青で強調表示）：





「アダプター1」がオンになっており、「ブリッジアダプター」に接続されていることを確認します









次に、タブの「アダプター2」ボタンをクリックします。 「アダプター2」がオンになっており、「内部ネットワーク」に接続されていることを確認します。 次に、「OK」をクリックします。





必要に応じて、[オーディオ]リンクを使用してサウンドカードを無効にできます。 これにより、ディスクスペースやメモリを節約し、オーディオ設定で起こりうる問題を回避できる可能性があります。 また、システムに4個以上のプロセッサコアがある場合、仮想マシンに2個のプロセッサを割り当てることは理にかなっています。 これは、「システム」設定メニューで実行できます。



仮想マシンは、ゲストオペレーティングシステムのインストールを開始できるように構成されています。



ゲストオペレーティングシステムのインストール





Virtual Boxは、インストール中にVM領域内にマウスポインターを保持することに注意してください。 マウスを制御するには、キーボードの右の「Ctrl」キーを押します。

Oracle Linux 6.3 DVD（またはそれ以降）をDVDドライブに挿入し、次の2つのスクリーンショットをスキップします。

DVDがない場合は、ダウンロードする必要があります。 ISOイメージを仮想DVDに入れます。 VirtualBox Manager画面の右ペインで「ストレージ」リンクを選択して、「ストレージ」画面を開きます。 次に、「ストレージツリー」セクションでDVDドライブを選択します。





[属性]セクションで、DVDディスクのアイコンをクリックして、.ISOファイルを選択します。 ファイル名はストレージツリーに表示されるはずです。 [OK]ボタンをクリックします。





ツールバーの「開始」ボタンをクリックして、仮想マシンを開始します。 その結果、コンソールウィンドウにはOracle Linuxブートウィンドウが含まれます。 「既存のシステムのインストールまたはアップグレード」を選択します。





メディアをテストしないでください。 [スキップ]ボタンを選択します。





通常のサーバーの場合と同様に、Oracle Linuxのインストールを続行します。 次の3つの画面で、言語、キーボード、およびプライマリストレージデバイスを選択する必要があります。

「ホスト名」を「rac1.localdomain」に設定し、「ネットワークの構成」ボタンをクリックします。





ネットワーク接続ウィンドウで、「System eth0」インターフェースを選択し、「編集」ボタンをクリックします。





[自動的に接続]ボックスをオンにします。 タブで[IPv4設定]を選択し、[方法]が[自動（DHCP）]に設定されていることを確認します。 「IPv6設定」を選択し、「方法」が「無視」に設定されていることを確認します。 [適用]ボタンをクリックします。





ネットワーク接続ウィンドウを閉じて、次のインストール画面に進みます。 タイムゾーンを選択してください。 rootパスワードを入力：oracle

「すべてのスペースを使用」インストールタイプを選択し、「パーティションレイアウトを確認および変更」チェックボックスを選択します。





lv_swapデバイスのサイズを1500 MBに変更します。 次に、lv_rootのサイズを14380 mbに変更します。 [次へ]をクリックします。





すべての警告とパーティションの作成を確認します。 ブートローダー画面にとどまります。

タイプソフトウェアのインストール画面で、「データベースサーバー」を選択し、「今すぐカスタマイズ」ボックスをオンにします。 [次へ]をクリックします。





画面設定で、[データベース]を選択し、すべてのアイテムのチェックを外し、[デスクトップ]アイテムを選択し、[デスクトップ]および[グラフィカル管理ツール]ボックスをチェックして、[次へ]をクリックしてインストールを完了します。 再起動



仮想マシンが再起動すると、より多くの構成画面が利用可能になります。 Oracleアカウントを作成しないでください。これは後で行います。

おめでとうございます！ Linuxがインストールされました。



インターネットアクセスを確認する





追加のパッケージがインターネットからインストールされるため、インターネットアクセスが必要になります。 ターミナルを開き、インターネット上の任意のWebサイトにpingを試行します。たとえば、次のとおりです。



ping ya.ru







pingが失敗した場合は、「Ifconfig」コマンドを使用して問題を修正し、起動してネットワーク接続（Linuxデスクトップのメインメニュー|システム|設定|ネットワーク接続）を変更します。

ネットワーク接続に変更を加えた場合は、仮想マシンまたはインターフェースを再起動します。

インターフェイスを再起動するには、次の2つのコマンドを使用します。



# ifdown eth0





# ifup eth0







次に、pingを再度確認します。
