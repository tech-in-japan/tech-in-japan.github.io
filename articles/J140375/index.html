Cloudstack 3.0.0



説明


Cloudstackは、データセンターのコンピューティングリソース管理コンソールです。 Zynga、Nokia Research Center、Cloudcentralなどの大企業は、このプラットフォームでクラウドを構築しました。 プロジェクトの開発は、Citrixによって実行されます。 プラットフォームには独自のAPIがあり、プラットフォームを構成して既存のインフラストラクチャと統合することができ、CloudBridge Amazon EC2アダプターを使用して、Amazon APIをCloudstack APIに変換できます。 アダプタでサポートされているコマンドの完全なリストは、 ここにあります 。 主なプラットフォーム機能：

  • ハイパーバイザーに依存しない（KVM、XEN、ESXi、OVM、BareMetal）
  • 役割
  • 仮想ネットワーク
  • リソースプール
  • スナップショットとボリューム
  • 仮想ルーター、ファイアウォール、ロードバランサー
  • ホストメンテナンスを使用したライブマイグレーション


ESXiハイパーバイザーを使用する場合、プラットフォームはvCenter APIを使用します。 したがって、VMware上に構築された既存のインフラストラクチャにプラットフォームを実装するのは簡単です。



費用


最良の部分は、現時点ではCloudstackは無料であり、GNU Public License Version 3の下で配布されているということです。有料サポートに興味がある場合は、開発者にメールで連絡する必要があります。



コミュニティ


プラットフォームのインストール中に、多くの問題が発生する可能性があります。 コミュニティに連絡して、それらを解決することができます。 フォーラムでは多くの問題について議論し、それらの解決に役立ちます。 IRCには質問もできるチャンネルがあります。



ドキュメント


おそらく、すべての製品でかなり重要な要素はドキュメントです。 ユーザーがこのテクノロジを簡単に入力できるかどうかを決定します。 ドキュメントがシンプルで明解であればあるほど、興味のある人がプラットフォームをインストールできる可能性が高くなります。 ある程度のバックグラウンドがあれば、問題なく簡単にプラットフォームをインストールできます。これで十分に理解できます。 より複雑なインストールとアーキテクチャの計画には時間がかかりますが、すべての機能がドキュメントに記載されているわけではありません。 ドキュメントは段階的なスタイルで行われ、プラットフォームの仕組みについては説明しません。 プラットフォームの実際の使用後にのみ、知識の一部があなたに届きます。



体験


基本的なインストールとセットアップ。 VMwareクラスターとkvmクラスターを使用します。 使用したオペレーティングシステムはCentOS 6.2です。 Rightscaleのクラウドを管理コンソールに正常に接続しました。 CentOS 6バージョンを使用している場合、libvirtdの起動時に問題が発生する場合があります。 これらの問題を解決するには、/ etc / cgconfig.confに追加します。



group virt {

cpu {

cpu.shares = 9216;

}

}







その後、/ etc / init.d / cgconfig restartを実行し、libvirtdの実行を試みます。

サイトを介したコンソールにも問題がある可能性があります。 1つの問題は、vncサーバーホストでは127.0.0.1アドレスで実行されることです。 実行を確認するには：



netstat -nlp | grep kvm

tcp 0 0 0.0.0.0:5900 0.0.0.0:* LISTEN 11673/qemu-kvm

tcp 0 0 0.0.0.0:5901 0.0.0.0:* LISTEN 11872/qemu-kvm

tcp 0 0 0.0.0.0:5902 0.0.0.0:* LISTEN 15227/qemu-kvm

tcp 0 0 0.0.0.0:5903 0.0.0.0:* LISTEN 12587/qemu-kvm







このコマンドを実行した後、アドレスが127.0.0.1の場合、/ etc / libvirt / qemu.confファイルの行のコメントを解除する必要があります。



vnc_listen = "0.0.0.0"







そして、仮想マシンを再起動します。



結論


非常に興味深いクラウド管理コンソール。 優れた機能、無料で急速に成長しています。 実際のインフラストラクチャに実装できますが、ESXiハイパーバイザーのみを使用する場合は、vCloud Directorが最適なオプションになります。







Eucalyptus Open Source 2.0.3



説明


Eucalyptusは別のクラウド構築プラットフォームです。 ソニー、プーマ、NASA、トレンドマイクロなどの大企業は、このプラットフォームでプライベートクラウドを構築しました。 Eucalyptusには、有料版と無料版の2つのエディションがあります。 これらのバージョンでは、機能が大きく異なります。 Eucalyptusがユーザーに伝えようとしている主な利点は、そのAPIがAmazon APIと完全に互換性があることです。 つまり、Amazon APIで動作するすべてのスクリプトとプログラムは、Eucalyptusプラットフォーム上に構築されたクラウドに使用できます。 Eucalyptusは、XEN、KVM、ESXiの3つのハイパーバイザーをサポートしています。 後者（ESXi）は、Enterpriseバージョンでのみサポートされています。

主な利点：

  • 役割
  • ハイパーバイザーに依存しない
  • クラスタリングとゾーニング
  • 柔軟なネットワーク管理、セキュリティグループ、トラフィック分離




費用


上で書いたように、無料版と有料版があります。 EucalyptusオープンソースとEcalyptus Enterprise Cloud。 バージョン間の違いはここで見つけることができます。



コミュニティ


この製品のコミュニティを評価できませんでした。 すべてが問題なくインストールされました。



ドキュメント


製品にはインストールドキュメントがありますが、すべての側面が開示されているわけではありません。 人の背景が大きいと想定されます。 仮想化自体の設定はありません。 より複雑な構成をインストールする場合、ドキュメントは役に立たないので、自分の能力だけに頼るべきです。



体験


ドキュメントに従ってインストール、構成します。 CentOS 5.5がクラウドコントローラーのオペレーティングシステムとして使用され、XENハイパーバイザーを備えたCentOS 5.5がホストのオペレーティングシステムとして選択されました。 また、クラウドをRightscaleに接続しました。 管理コンソール自体は存在しないため、euca2toolsを使用して管理するか、RightScaleに接続する必要があります。 ただし、有料版には独自の管理コンソールがあります。 インストールで次の問題が発生しました。

1）いくつかの依存関係を順番にインストールするときに、インストールを拒否しました。 解決策は、依存関係を一度にインストールすることでした。



rpm -Uvh python25-2.5.1-bashton1.x86_64.rpm

error: Failed dependencies:

libpython2.5.so.1.0()(64bit) is needed by python25-2.5.1-bashton1.x86_64







解決策：



rpm -Uvh python25-2.5.1-bashton1.x86_64.rpm python25-devel-2.5.1-bashton1.x86_64.rpm python25-libs-2.5.1-bashton1.x86_64.rpm







2）プラットフォームをインストールした後、コントローラーを起動すると、次のエラーが発生する場合があります。



/etc/init.d/eucalyptus-cc start

Starting Eucalyptus cluster controller:

Enabling IP forwarding(13)Permission denied: make_sock: could not bind to address [::]:8774

(13)Permission denied: make_sock: could not bind to address 0.0.0.0:8774

no listening sockets available, shutting down

Unable to open logs

Failed to start the CC!







問題を解決するには、Selinuxをオフにします。

3）XENの十分な経験がない場合、XENの使用に問題が発生する可能性があります。 XENを/etc/grub/menu.lstにインストールしたら、default = 1-> default = 0を修正して、XENからカーネルを起動する必要があります。

4）CentOSでも、libvirtグループを作成し、そこでユーカリを実行するユーザーを追加する必要があります。



結論


なんとなく湿っています。 無料版は特に積極的に開発されていません。 問題があります。 おそらく商用版の方がはるかに優れているでしょう。 プラットフォームの調査中に、商用製品の試用版をリクエストしましたが、商用版の機能を評価できませんでした。







vCloud Director 1.5



説明


vCloud Director VMwareクラウド構築プラットフォーム。 インフラストラクチャ全体がVMware上に構築されている場合、vCloud Directorを導入することが完璧なソリューションになります。 vCloud Directorを使用すると、真のハイブリッドクラウドを構築できます。 VMware vCloud Connectorを使用すると、パブリッククラウドとプライベートクラウドの間で仮想マシンを移行できます。 主な機能：

  • 仮想データセンター
  • vShield Security Technologies
  • インフラストラクチャサービスカタログ
  • マルチテナント組織
  • セルフサービスポータル
  • VMware vCloud API、Open Virtualization Format、およびコールアウト




費用


有料。 自分に合ったパッケージのおおよその価格を調べるには、地域のマネージャーに連絡する必要があります。



コミュニティ


VMwareはマーケットリーダーであるため、素晴らしいコミュニティを持っています 。 知識ベースの形で優れたサポート。 VMwareから製品を購入した後の有料サポートとサポートについて忘れないでください。



ドキュメント


有料製品には常に優れたドキュメントがあります 。 そして、実践が示しているように、それに従えば、インストールは問題なく通過します。



体験


vCloud Directorをインストールして構成します。 vCloud Director自体をインストールするには、Red Hatが必要です。 vCloud Directorがインストールされないコンポーネントもあります。

-vCenter（クラスターおよびDRSを使用）

-vShield

vCloud DirectorがvCenter APIを使用していると推測するのは難しくありません。 したがって、vCloud Directorを使用してクラウドを構築する場合、すべてのvSphere機能を利用できます。



結論


主な欠点は、それが支払われることですが、VMwareの仮想化をすでに使用している場合、vCloud Directorが理想的なオプションになります。







Openstack 2011.3



説明


Openstack 2011.3は、クラウドを構築するためのオープンソースプラットフォームです。 Openstackプロジェクトには、Nova（Amazon EC2のアナログ）、Swift（Amazon S3のアナログ）、Glance（画像を提供するサービス）の3つの製品が含まれています。 比較では、Novaのみを検討しますが、Swiftに注意を払う価値があります。この製品は独自の方法であり、多くの製品が統合されています（Cloudstack 3）。 現時点では、Novaは適切なレベル2のハイパーバイザーであるKVMとXENを維持しています。 プラットフォームは急速に開発されており、優れた機能を約束します。 Cisco、Dell、NASA、Intel、AMD、Citrix、Rackspace、Rightscaleなどの企業からの巨大なコミュニティとサポートがあります。 Openstackの主なコアは、NASAが開発した星雲でした。 主な機能：

  • 仮想化されたコモディティサーバーリソースを管理する
  • ローカルエリアネットワークを管理する
  • 仮想マシン（VM）イメージ管理
  • セキュリティグループ
  • 役割ベースのアクセス制御
  • プロジェクトと割り当て
  • Webブラウザを介したVNCプロキシ




費用


この製品は完全に無料のオープンソースですが、コミュニティによる製品開発の支援を忘れないでください。



コミュニティ


私の考えでは、Openstackは、私たちが検討しているすべてのプラットフォームの中で、最大かつ最も活発なコミュニティを持っています 。 製品は積極的に開発されているため、多くの問題があり、多くの場合、人々はこれらの問題の解決を助けます。



ドキュメント


Openstackにはドキュメントに関する問題があります 。 製品の急速な開発のため、ドキュメントには更新する時間がありません。 重要なことはあまりないので、 フォーラムやIRCで調べる必要があります。



体験


Nova、keystone、ダッシュボードの基本的なインストールと構成。 仮想化には、KVMを使用しました。



結論


このプラットフォームは無料であり、急速に開発されていますが、それでも未熟です。 実際のインフラストラクチャに実装するには、念頭に置くPython開発者のチームが必要です。 Amazon向けの開発がある場合は、Amazon APIとの互換性が役立ちます。 プロジェクトインキュベーターにはOpenstackダッシュボードがありますが、APIの再設計により、常に100％の機能が提供されるわけではありません。
