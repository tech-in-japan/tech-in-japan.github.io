Linuxでは、マウスとタッチパッドの速度を分離します。

こんにちは、私は幸せなラップトップユーザーです[ブランドを宣伝しないように頼まれました] 。 一般に、素晴らしいマシンであり、最もLinux互換のマシンの1つです。 アメリカでは、プレインストールされたubuntuで販売されているので、鉄にはまったく問題ありません。 すべてが決定的に機能します。



しかし、私は小さな迷惑がありました。 ほとんどの場合、USBに接続されたマウスを使用し、touch歯類を装着する機能がない場合のみタッチパッドを使用します。 そして、小さなゲーミングポンツーンなので、私のマウスの解像度は1600dpiです。 言い換えれば、彼女は過敏です。

もちろん、これは特定の問題ではありません。オペレーティングシステムの設定でマウスポインターの感度を下げることで解決されるためです。 しかし、その後、そのような設定でタッチパッドを使用することはほぼ不可能になるため、トラブルが始まります（画面の隅から隅にカーソルをドラッグするには、指を5回動かす必要があります）。

実際、私の前に、外付けマウスとタッチパッドの感度設定を分離するというタスクが発生しました。







長いグーグルとページングのフォーラムでは解決策が得られませんでしたが、guish touchpad configuratorsは存在しますが、そのようなオプションを提供しません。 ドライバーのドキュメントを読んで、結果として私を救った。



したがって、実際にはすべてが非常に簡単に解決されます。

古き良き/etc/X11/xorg.confを開きます



タッチパッドドライバーがインストールおよび構成されている場合、次のような入力デバイスセクションが必要です。



Section "InputDevice"

Identifier "Synaptics Touchpad"

Driver "synaptics"

Option "SendCoreEvents" "true"

Option "Device" "/dev/psaux"

Option "Protocol" "auto-dev"

Option "SHMConfig" "true"

Option "HorizEdgeScroll" "0"

Option "MinSpeed" "0.7"

Option "MaxSpeed" "1.2"

Option "AccelFactor" "0"

EndSection









構成に含まれていない可能性が最も高い最後の3行は、タッチパッドの感度の原因です。

MinSpeed-指をゆっくりと表面上で動かしたとき。

MaxSpeed-それぞれ、高速で。

AccelFactor-カーソルの加速の度合い。



これらの行を追加するとき、タッチパッドを使用したカーソルの速度（少なくとも私の場合）は、システムの一般的な設定に依存せず、ここから排他的に構成されます。 マウスの速度は昔ながらの方法です。
