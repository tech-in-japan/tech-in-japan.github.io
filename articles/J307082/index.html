S7を作成するための経済的前提条件



最近、企業にとってのパブリッククラウド環境の利点はますます明らかになってきていますが、多くの組織はまだワークロードをクラウドに移行していません。 彼らはパフォーマンス、セキュリティ、および管理を懸念しています。



昨年のIDCによると、プライベートクラウドとパブリッククラウドを含むクラウドITインフラストラクチャへの投資は、世界で21.9％増加して290億ドルでした。2015年の第4四半期には、全体の32.2％を占めました。企業7の消費者のみのコスト：可能な限り最小のITインフラストラクチャで、今日最も人気のあるアプリケーションの最高の効率を確保します。 IDCの予測によると、2020年までに、そのような年間費用のほぼ50％がクラウドソリューションによって占められます。 金銭的には、これは578億ドルになります。



多くのお客様は、高価なシステムから、どこでもプライベートクラウドとハイブリッドクラウドが構築されるx86「ボックス」にすばやく切り替えます。 これらの顧客がSPARCシステムに興味を持ち続けるために、オラクルは市場の下位セグメントに参入し、対応する機能を備えたシステムを導入しました。



このイニシアチブは2016年6月末に会社によって公式に発表されました。SPARCプロセッサとサーバーファミリの新しい外観を提供します。これらは、オラクルでかつてない最低価格で提供されています。 同時に、そのようなシステムは「最も安いもの」ではありません-最低価格は約1万1千ドルです。同時に、コンピューティングコアあたりの価格はx86サーバーの価格に相当します。 Sunの購入後、これはOracleの最も大きな発表です。



アナリストは、新しいシステムをSPARCおよびSolaris OSクライアントの混乱を防ぎ、既存の顧客を維持する方法と考えています。



新しいプロセッサはS7と呼ばれます。 実際、それは2015年10月に発売されたハイエンドM7チップのあまり強力ではないバージョンです。Sonomaと呼ばれていたS7の計画は2015年に概略的に発表され、その最初の製品は6月29日に発表されました10月のOpenWorld 2015。





S7プロセッサチップ（Sonoma）には、多数の高度な技術が組み込まれています。





x86プロセッサと比較して、Oracle S7にはいくつかの利点があります。特に、データ分析、暗号化、セキュリティ、I / Oを加速するための組み込みツールがあります。



S7は、M7と同じ第4世代のSPARCコアに基づいており、同じ20nmプロセスを使用して生産されています。 ただし、M7では32コアではなく8コアしかありません。 M7と同様に、新しいチップには、DBMSとセキュリティ機能を高速化するための「Software on Silicon」（シリコンインソフトウェア）が組み込まれています。



S7には、メモリとI / Oインターフェイスが統合されています。これらは、M7とは別個で、マザーボード上にあります。 これにより、S7サーバーの最終構成のコストが大幅に削減されます。

さらに、今日では、4.27 GHzの周波数で動作するS7プロセッサの1つのバージョンのみが提供されています。 消費者に可能な限り低いコストで、今日の最も人気のあるアプリケーションに最高の効率を提供するように設計されています。



その結果、Oracleは、新しいチップを搭載した2つのスタンドアロンサーバー-S7-2とS7-2L、およびOracle SuperClusterの小型バージョンである「エンジニアリングシステム」MiniCluster S7-2を発表しました。



S7-2サーバーは建設的に1ラックフットプリント（U1）を占有し、最大密度に設計されています。 シングルおよびデュアルプロセッサ構成で販売されています。



S72Lサーバーは、より強力なモデルです。 ラック（U2）の2つの座席を占有し、通常は2つのプロセッサーを備え、情報ストレージシステムの複数の構成を作成できます。 両方のサーバーは、SASおよびNVMeストレージを任意の組み合わせで使用できます。



11,000ドルで、お客様は1つのプロセッサ、64 GBのメモリ、2つの600 GBのハードドライブを備えた構成を受け取ります。 2つのプロセッサと1テラバイトのメモリを使用した最大構成には、約5万ドルかかります。



「大規模な」SPARCシステムは従来ERPやCRMなどのアプリケーションを実行していましたが、S7サーバーはより高度なJava、Hadoop、Sparkワークロードに焦点を当てています。



S7-2は、データベース、ビジネスインテリジェンス、高密度仮想化環境などの集中的なワークロードの「低コストリーダー」です。 3つのPCIe 3.0x8スロット、4つの10G Base-Tイーサネットポート、およびIntegrated Lights Out Management（ILOM）制御ポートがあります。



S7-2Lにも同じ目的があり、「ダブル」S7-2を表します。 6つのPCIe 3.0x8スロットと、同じ4つの10G Base-Tイーサネットポートと1つのILOMポートがあります。 大きな違い-Lモデルは最大27台のハードドライブを使用できます。 128 GBのRAMと24.6 TBのディスクアレイを備えたこのクラスの製品の場合、2万ドル未満の価格は高すぎないようです。





S7シリーズサーバーコンストラクトと関連クラウドサービス



Oracle MiniCluster S7-2



最適化されたOracle SuperClusterハードウェアおよびソフトウェアシステムのセキュリティとパフォーマンスの利点を必要とするミッドレンジソリューションのために、オラクルは新しいOracle MiniCluster S7-2も導入しました。



Engineered System MiniCluster S7-2は、Enterpriseクラスのセキュリティを備えたEngineered Systemsのすべての利点を提供します。 簡素化された管理、すぐに使用可能な起動、高い信頼性、小さなフォームファクタにより、リモートオフィスや小規模オフィス、孤立したワークロードに最適です。 ワークロードが増大している場合、MiniClusterでは、シンプルなフルタイム構成ツールを使用して、数分以内にシステムにプロセッサを追加できます。



データベースアクセラレーションは、プロセッサだけでなく、オールフラッシュストレージの高速入出力によっても実現されます。 対応するセキュリティモード（PCI-DSSまたはCIS）は、ボタンをクリックするだけでアクティブになります。



SuperCluster M7とのアプリケーションおよびデータベースの完全な互換性により、Oracle MiniClusterは組織がハードウェアとソフトウェアのコストを大幅に削減できるようにします。



この新しい複合施設は、アプリケーションやデータベースのマルチテナント統合、リモートオフィス/ブランチ用のコンピューティングプラットフォーム、テスト/開発環境などのソリューションをサポートするために作成されました。



オラクルはMiniClusterの価格を公開していませんが、事前に設定された仮想化および管理ソフトウェアを備えた構成済みのボックスでクライアントにコンピューティング能力、ストレージ、およびネットワークコンポーネントを提供することにより、エンジニアリングシステムの市場を拡大することが期待されています。



MiniClusterとSuperClusterの大きな違いは、インストールと管理のほとんどがここで自動化されることです。 このように、顧客は実際に、より大きなマシンのセキュリティと高可用性レベルを取得でき、Solaris OSの専門家を必要としません。



「大衆向けシリコンソフトウェア」



メインプログラムコードをシリコンに転送したCPUアーキテクチャにこの新しいパラダイムを実装するM7プロセッサは、約2年前にリリースされました。



多くの組織の最大の問題は、購入する新しいサーバーの開始価格が高いことです。 すべてが32コアを必要とするわけではありません。 そのため、OracleはM7を4倍削減しましたが、同時にすべての機能を保持するだけでなく、いくつかの新しい利点も追加しました。



それらの1つは、低消費電力です。 ロジックのほとんどはマザーボードからCPUに直接転送されるため、システムパフォーマンスが向上し、生産コストが削減されます。



興味深いことに、S7プロセッサにはチップに統合されたInfiniBandコントローラーが含まれており、サーバーを高速クラスターに結合します。 しかし、新しいシステムはまだこの機能を使用していないため、よりシンプルなオプションとしてイーサネットを優先しています。 この点に関して、オラクルの推奨事項は次のとおりです。「お客様は、必要に応じてInfiniBandの拡張ボードを使用できます。」



すべてのシステムがSolaris OSを実行しています。 Solarisを使用すると、LDOMハードウェア仮想化技術や関連ソフトウェアなど、クライアントにはLinuxに比べていくつかの利点があります。



Solaris OSは、ロールベースのアクセス、組み込みのコンプライアンスレポート、およびOSの標準的な部分であるエンタープライズオートメーションに基づいた多数のオプションにより、高度なセキュリティを提供します。 多くのコマンドが同一または類似しているため、Linux管理者はSolarisをすばやく習得できます。



S7-安全性、効率性、シンプルさ



そのため、SPARCファミリの新しいメンバーは、4.27 GHzのクロック周波数と、Silicon Secured MemoryおよびData Analytics Acceleratorを含む「Software in Silicon」機能を備えた新しい8コア、64スレッドSPARC S7マイクロプロセッサに基づいて構築されます。



今日、S7システムは、単一のコンピューティングコアに関して業界最高のパフォーマンスを提供し、組織が低コストの大量使用サーバーのコストでSPARCプラットフォーム上であらゆる規模のアプリケーションを実行できるようにします。



組織で使用されるすべてのアプリケーションは、セキュリティ、効率、およびシンプルさを大幅に向上させ、変更することなく新しい企業クラウドサービスとSPARCソリューションで動作します。 新しいSPARCプラットフォームの主な機能は次のとおりです。



安全性 新しいサーバーのS7ファミリは、悪意のある侵入やプログラミングエラーから保護するために設計されたSilicon Secured Memory機能のセキュリティおよび規制へのコンプライアンスを提供します。 暗号化アルゴリズムとハッシュ関数のハードウェアアクセラレーションにより、パフォーマンスを2％未満低下させながら、完全に安全なクラウド環境を実装できます。



さらに、制御されたダウンロード、変更不可能なコンテンツ、不正な変更の防止、セキュリティツールの強制更新、および仲介者のないハードウェアとソフトウェアの信頼できるサプライヤチェーンによって、追加のセキュリティレベルが確保されます。



効率性 新しいサーバーは、待ち時間とコストを削減します。 統合されたデータ分析アクセラレーターは、企業およびクラウドアプリケーションやビッグデータ処理ソリューションで分析クエリを実行するときにパフォーマンスを10倍向上させます。



（カーネルに関して）x86サーバーと比較して、完全に統合されたS7-2およびS7-2Lサーバーは最大100％効率的に動作します。 Javaアプリケーションのパフォーマンスが1.7倍、OLTPデータベースアプリケーションの1.6倍、そして負荷の高い分析クエリとクラウドアプリケーションのスループットが2〜3倍向上します。



単純化 。 最適化されたOracle MiniCluster S7-2ハードウェアおよびソフトウェア複合体は、サーバーよりも高いレベルへの統合を実現します。 これにより、エンタープライズコンピューティングの4つの最も複雑な側面（セキュリティとコンプライアンス、高可用性、更新と管理、パフォーマンスチューニング）が大幅に簡素化されます。 この複合体により、組織は次のことができます。



-デフォルトでシステムを保護し、企業のセキュリティ専門家を不要にします。



-コンプライアンスの監視と監査を自動化して、常に安全なシステム状態を維持します。



-ハードウェアとソフトウェアに組み込まれた高可用性ツールを使用して、サービスの円滑な運用を維持します。



-セキュリティシステムソフトウェアの更新を含め、システムの現在の状態を維持します。-ソリューションスタック全体のサービスパックを簡単に展開できるため、

既存のインフラストラクチャと簡単に統合できるように設計されたこれらの製品には、クラウド用の完全に統合された仮想化および管理ツールが含まれています。
