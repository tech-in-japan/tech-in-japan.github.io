絶滅の危機にある10のIT専門職

死後を求めて、あなたは遠くに行く必要はありません-コンピュータ産業を見てください。 古いシステムは地下室と奥の部屋で生活しています。 Novell NetWareネットワークエンジニアとOS / 2管理者は、ヘルパーを見つけるのがますます難しくなっていると感じています。 しかし、最近まで、これらは非常に有望な技術であり、非常に有望な職業であると思われていました。



ITリクルーターと相談した後、完全に死んではいませんが、間違いなく死に近づいているコンピュータースキル、職業、テクノロジーのリストを作成しました。 彼らの普遍的な人気は永遠に遅れています。



1. COBOL

問題Y2kは、COBOL言語のプログラマーにとって2番目のゴールドラッシュを生み出したかのようです。 2000年には、彼らの能力は金に匹敵する価値がありました。 しかし今、6年半後、この職業を救うチャンスはありません。 この言語は大学でほとんど教えられなくなりました。 同時に、実務家は、全米の数千の企業が、時代遅れではあるが機能しているシステムを処理するためにCOBOLプログラマーを必要としていることを保証しています。 これまで、COBOLはほとんどのコード行が記述されたプログラミング言語と見なされていました。



COBOLコースは、南イリノイ大学で教えられています。 イリノイ州の2大企業は、COBOLでビジネスアプリケーションを使用していますが、このプログラミング言語を知っているコーダーが必要です。



2.非リレーショナルDBMS

1980年代には、データベース管理システムには2つの主要なクラスがありました。IBMのIMSやSAS InstituteのSystem 2000などの階層システムと、CAのIDMSや以前のVAX DBMSであるOracleのDBMSなどのネットワークDBMSです。 現在、これらのビューは両方とも、DB2、Oracle、Microsoft SQL ServerなどのリレーショナルDBMSに取って代わられています。



3.非IPネットワーク

TCP / IPはネットワークの世界の大部分を征服しており、IBM Systems Network Architecture（SNA）ネットワークの専門家に対する需要は事実上ありません。 採用担当者は、このスキルの価値はゼロに近いと主張しています。 SNAの知識があれば、ネットワークエンジニアの給与は1％しか増えません。 多くの銀行、保険会社、その他の企業がSNAネットワークインフラストラクチャに多額の投資を行っていますが、この専門分野の学生は実際にはどこでも訓練を受けていません。



4. cc：メール

80年代に作成されたローカルネットワークのメールプログラムは、その人気のピーク時に2,000万人の聴衆がいました。 ただし、時間の経過とともに、Lotus NotesやMicrosoft Exchangeなどのより洗練されたチームワークシステムに電子メールアプリケーションが統合されたため、cc：Mailの人気は低下しました。 2000年に、このプログラムは市場から完全に削除されました。 今日ではメールクライアントは常に他の企業ソフトウェアと連動しているため、クラスとしては姿を消しました。cc：メールは隔離されたため、消滅する運命にありました。 この製品のサポートロータスは現在、カリフォルニアの会社Global System Servicesを引き継いでいます。



5. ColdFusion

かつて非常に人気があり習得が容易なスクリプトWebプログラミング言語は、90年代半ばにAllaireによって作成されました。Allaireは後にMacromediaに買収され、MacromediaはAdobeに買収されました。 この言語は、Microsoft Active Server Pagesや.Net、Java、Ruby on Rails、Python、PHP、その他のオープンソース言語など、他のWeb開発プラットフォームと競合することはできませんでした。 ColdFusionの知識に対するプレミアムはWebプログラマー市場で大幅に減少している一方で、ColdFusionアプリケーションが競合他社と比較してどの程度信頼性と拡張性があるかについてはまだ議論があります。



6. Cプログラミング

Webが進歩するにつれて、C言語の重要性は低下します。 C ++とC Sharpはまだ生きており、積極的に使用されていますが、基本的なC言語だけの知識では、少なくともいくつかの作業を見つけるのに十分ではありません。



7. PowerBuilder

10年ほど前、PowerBuilderプログラマーは雇用主にホットケーキとして理解されていました。 90年代初期には、それは非常に関連性のある有望なテクノロジーでした。 1994年、Oracleの強力なライバルであるSybaseは、Powersoftからクライアント/サーバー開発システムを購入しました。



現在、PowerBuilder開発者は、ホットな職業のランキングの最下位にいます。 給与は最小限であり、COBOLのプログラマの給与と比較できます。 PowerBuilder製品が進化し続けるにつれて、これはさらに驚くべきことです。今年は、.NETコードを生成できるPowerBuilder 11をリリースする予定です。



8. NetWare認定エンジニア

90年代前半、すべての部隊はNetWareエンジニアの企業証明書を取得しようとしました。 小説がPCサーバー市場の90％を支配していたため、これは驚くことではありません。 状況は非常に急速に変化しました。 わずか数年で誰もがNTに移行したため、それ以来、CNE証明書は雇用主の目から見てその価値を急速に失いつつあります。



9. PCネットワーク管理者

より多くのネットワークがWindowsサーバーに切り替わるにつれて、IBMサーバー（AS / 400など）に精通しているネットワーク管理者の人気は低下しています。



10. OS / 2

OS / 2の大まかな、しかし非常に適切な評価は、「自然の間違い」です。 マイクロソフトとIBMのこの不幸な子供は、1987年に大騒ぎで世界に生まれましたが、すぐに両親は口論し、望まない子供を運命に任せました。 IBMは2005年に公式にOS / 2の配布を停止し、小規模企業Serenity Systems Internationalはシステムの販売と保守に関与しています。 不幸な半軸はすでにeComStationという別の名前で販売されています。
