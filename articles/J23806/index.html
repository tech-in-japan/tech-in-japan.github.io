心から投票する

こんにちは

実際のところ、 Nudnik.ruブログで格付けのコンパイルに関する興味深い投稿を読んだのはそれほど前ではありません。 そして、私は彼に関するコメントを読みました。 そして、「もっと読む」をクリックすると、突然修正されて補足されたこの投稿が表示されます。



—————



最初の投稿の主な（そして唯一の）仮定（これは2番目の説明）です。

すべての投票において、タイプ+ 1 / -1は実際には+ N / -Mである必要があります（N！= M（NはMと等しくない））。




この輝かしい仮説を理解している人はほとんどいなかったため、この記事は執筆されました。



しかし、最初に、一般的な投票のポイントは何ですか。 まず第一に、 もちろん--- ...しかし、とにかく誰かがそれを言うべきです。 投票は選択です。 5人の大統領があなたの前に立ち、それぞれの下にラジオボタンを掛けました。 いずれかを選択してください。



ZhirikとZyugaに同時に投票することは不可能であることは明らかです。 本当にしたい。



たくさんの素材があり、それぞれのクリーチャーに1〜5の評価を与えることができる場合は別の問題になります。たわごとと多かれ少なかれ適切なトップスを作ります。



選択の余地がないため、ユーザーに投票する目的はまったく異なります。 ユーザーは、評価されたすべてのエンティティに5ポイントを置くことができます。 「全員が同時に大統領になりたい」という完全な類似物。 これらの票の価値がゼロであることは明らかであり、その純粋な形ではほとんど発見されませんが、そのように投票する機会があります。



限られたリソース（「1つを選択」）がないため、経済的に（または合理的に）使用する動機もありません。 しかし、少なくともいくつかの動機が必要であるため、感情的なメカニズムが活性化されます。これは、「好き/嫌い」というバイナリでもあります。



「気に入ったが、あまり好きではない」、つまり「5」の代わりに「4」を置くことを区別するためには、「それについてごめんなさい？」という意味もありません。 いいえ、それは残念ではありません。 いいね！ しかし、私たち、そのような良いものが、私たちを怒らせたら...



投票の観点からは、選択として、スケールの離散性が重要です。スケールが大きいほど、精度が高くなります。 最も適切な尺度は100点です（7.5のような投票も100点の尺度であり、ユーザーにとっても10点で理解できることを理解する必要があります）。



人々と特に社会的相互作用の観点から、最も便利なバイナリスケール：くそ/キャンディ。



また、75〜95％の人がスケールをバイナリに縮小します。







したがって、中間的な結論：スケールをどのように呼んだとしても、スケールが「良い」と「悪い」に気づくとすぐに、正確に1つの方法で使用されます。



ディスクリートスケールは、それを「イデオロギー的に正しい」と使用する適切な人々の10％にとってもう少し理にかなっています。 別の副作用：スケールの分割が多いほど、「寛大な」人が5/5の代わりに9.7 / 10を置く可能性が高くなります：「これはそれほど不快ではありません。」



「心で投票」する人々は、各評価の個別の署名でさえも、何にも侵入することはできません。 彼らはまだ「彼らはそう見る」という独自の見解を持っています。



—————



サイトの作成者と訪問者の間に矛盾があります。



良い例：写真は、映画「ゴッドファーザー」のimdb.comでの投票の結果を示しています。これは、上位250位にランクされています。







6％の人が映画を10分の1にしたことがわかります。つまり、「ゴッドファーザー」は信じられないほどの魅惑的な糞だと考えています。



明らかに、これは数学的にもそうではありません：グラフは「7」になり、1〜7の範囲で映画を評価した人々はどこに行ったのかは明らかではありません。 数学的予想によると、さらに何倍もあるはずです。



史上最高の映画の平均スコアはわずか8.7です。



対立は基本的な方法で「解決」されます。「一人、一票」は、従う必要のない素朴な愚か者の原則です。



入り口にある投票は、データのように、そのように評価する必要がある単なる生の統計データです。 さらに、平均スコアの計算は、このデータで実行できる最もばかげたことです。 平均スコアはすぐには機能せず（1つは投票済み）、その後も機能しません。なぜならばかばの割合は一定の値であり、悲しいかな、それらのタグはハングアップしないからです。



Imdbは加重投票平均を使用します。



IMDbは、生データの平均ではなく、加重投票の平均を公開します。 生データにさまざまなフィルターを適用して、映画の本当の意見を伝えるよりも映画の現在のレーティングを変更することに関心のある個人による「投票の詰め込み」の試みを排除し、削減します。



使用する正確な方法は公開されません。 これにより、ポリシーが引き続き有効になります。 結果は、より正確な投票平均です。




簡単に言えば、これは「すべての音声が同じように役立つわけではない」ということだけを意味します。



「すべての投票において、タイプ+ 1 / -1は実際には+ N / -Mである必要があり、N！= M（NはMと等しくない）」に戻ると、これはただ1つのことを意味します。 「引き上げる」（またはその逆）よりも意欲が高い場合、この仮説が当てはまる場合、収集された統計は、愚かな量ではなく、重みを使用することで考慮するのに役立ちます。



つまり、ほとんどすべてのものは上記で説明したものと同じですが、バイナリシステムを参照しています。



問題は、バイナリシステムの正規化がより難しいことです。 条件付きで言えば、100ポイントシステムでは、「1」を率直な遅延として配置した人を安全に考慮することができます。バイナリシステムでは、このための十分なデータがありません。



一方、+ 1 / -1などの「投票」は、「低下」のためのほぼ完璧なツールです。したがって、このように使用する必要があります。 「彼らは私をカルマと親しくしました。」 このような投票のもう1つのプラスは、一人の人の影響が理解可能な定数であり、「対処方法」が明確であることです。 Nポイントスケールでの投票および平均スコアの計算とは異なります。



（「スパーリング」投票システムをこれまで誰も実装していない理由はわかりません。ユーザーには3〜5つのランダムなジョブが与えられますが、仕事の評価ではほぼ同じであり、1つを選択する必要があります）

したがって、理想的な投票システム：



1.バイナリスケールではなく、主に自分自身だけでなく、ごく一部の普通の人にも提供します。

（ユーザーは、5ポイントに「慣れている」と言っていますが、嘘をついているだけです。バイナリに持ち込む方が簡単です。「深刻な」プロジェクトでは、100ポイントを作り、すべてを7.5 / 10にします）。

2.この尺度の解釈をユーザーに強制しません（！）。

3.義務付けられておらず、各投票を均等に考慮すべきではありません。

4.最終評価を計算するためのメカニズムを閉じています。
