検索2.0、どのようなものか

よく知られている2つの引用符を出発点としてみましょう。

  1. 「検索エンジンは世界の2つの新しい不思議の1つになり、ホモサピエンスは情報への無制限の即時アクセスを可能にしました。」イリヤセガロビッチ、「検索エンジンの仕組み」
  2. 「インターネットは大きなダンプのようなものです。すべてがそこにありますが、見つけることは不可能です。」民俗の知恵


現代の検索エンジンで遭遇しなければならない3つの主要な問題があります。



情報を見つけることの主な問題



  1. 情報を検索する代わりに、情報へのリンクを検索します 。

    あなたはまだスニペットに飽きていませんか？ リンクを訪問し、ページ上の情報を検索するのに費やした時間。 はい、同時にバナー、訪問者カウンターなどをねじって所有者を喜ばせますが、RSSフィードの公開に苦しむ人はいませんか？ 100件中90件のケースで、情報を検索し、それが存在する可能性のあるリンクを取得します。 瞬時にアクセスする代わりに、必要な情報は特定の障壁を超えて私たちから遠ざかります。

  2. 広範囲に検索する代わりに、詳細に検索します。

    「インターネット」の要求に応じて、Yandexは6億2000万ページを提供し、この数字は増え続けています。 そして、あなたの対談者があなたの質問「インターネットとは何か」、「私はこの言葉を602百万回聞いた、あなたは再び何に興味がありますか？」 おそらく、私たちの精神的な対話者は、技術的な専門家であればプロトコルについて、そうでなければソーシャルネットワークについて話し始めるでしょう。 いずれにせよ、彼は質問に対して1つの答えを持っています。 たとえ有名なジョークのように、この答えが完全に正確で完全に役に立たない場合でも。

  3. 混合検索結果 。

    カモメを求めてください。 時計、車、鳥などに関する1つのヒープ情報を受け取ります。 互いに混合された単一のリストによって。 はい、一部のシステムでは、関連する単語またはクラスターツリーを近くに引き出すためのti病な試みがありますが、これにより、ローカルなカオスを伴う新しいクエリが発生します。 問題を慎重にセクションに分割して（関連する単語ではなく）、これらのセクションの結果を生成できないのはなぜですか？ 鳥は別であり、機械は別です。そうでなければ、情報への瞬間的なアクセスはその不均一性の下に沈んでいます。


ここでの開発者は、これらの問題に適用できるすべての機会に特別なマントラを持っています-「ユーザー自身が彼が何を望んでいるかわからない」ので、私たちは行動するように強制されます。 すみません、同僚やさりげない通行人に尋ねると、彼はあなたが彼に何を望んでいるか理解していますか？



検索の説明2.0



批判は最も簡単な作業ですが、それでも「明日」の検索エンジンや検索2.0で何を見たいですか？

  1. インターフェース：ウェブチャット、コマンドラインインターフェース（cli）、インスタントメッセンジャー（im）
  2. 原則：検索エンジンとユーザー間の対話
  3. 違い：

    • 準備情報
    • 会話コンテキストのサポート（「ITにかかる費用は？」）


  4. ボーナス：

    • AI-推論を通じて、新しい情報を生成する能力
    • 対話サポート（「こんにちは、元気ですか？」）




そして最後に、記事の冒頭で特定された問題を解決するための私自身の試みについての簡単な説明（重要な注意-以下に説明するサービスはすべてプロトタイプであるため、英語でのみ機能します）。 これは検索2.0ではなく、Googleのアドオンを最適化し、検索結果を再フォーマットします。



検索エクスプローラー



Search Explorerを呼び出してnewisearch.comにある検索結果に自分の注文を表示する最初の試み。 主な機能の中で：

  • コンテキストキーワードの定義
  • テーマグループへのクラスタリング
  • グループ内のブックマークナビゲーション
  • 「I'm lucky」モードでは、検索結果ではなく、選択したクラスターに最も関連する「準備ができた」サイトでナビゲートできます
  • 同音異義語を削除するコンテキストを含む検索モード。 例：クエリ「jaguar @car」では、フォーム「jaguar rides」の結果を検索できます。クエリ「jaguar animal 」-「jaguar running」。

検索概要



この記事で検索を最適化する最後の試みは検索要約（ newisearch.com/sum ）です。検索結果の数を増やす代わりに、検索結果を目に見える数に減らし（オプティマイザーは私を許して）、テーマに分けます。 主な機能の中で：

  • 検索結果をいくつかのクラスターに分割し、各クラスター内にいくつかのスニペットを配置し、それらの間を移動します。
  • 選択したクラスター内で「失敗」する機能（ドリルダウン）-現在のキーワードに基づいて新しい検索を実行します
  • プロジェクトのさらなる開発：右側に、断片の代わりに、サマリを表示します
