Leopardのインストールエクスペリエンス



Mac OS X Leopardは、 Appleコンピューターのユーザーだけでなく、すべてのユーザーにとって画期的なイベントです。 Appleは300を超える革新を約束し、彼らの子孫の革命的な性質を予測しました。 事前に申し上げます-会社はすべての約束を守り、宣言されたものを完全に実装しました。

公式リリースの数日前に、いわゆる「ゴールドマスター」システムイメージが、よく知られているファイル共有サービスに登場しました。これは、販売前にブランクがスタンプされるサービスです。 システムをできるだけ早く入手したい場合は、急流をダウンロードしてください。

速度は望みにはほど遠いものでしたが、10月26日に目が覚めて、ダウンロードが完了したことがわかりました。



小さな余談-友人たちは、私と同じことをすることを決して勧めない。 機会があれば、 Leopardで箱を購入してください。それにより、多くの喜びを味わうことができ、最も重要なことは、ライセンス製品の所有者になります。

ダウンロードした画像は.dmg形式で、私をst迷に陥れました。私のMacBook Proは最初のモデルラインに属し、 DVD-DLを書き込むことができません。 原則として、これは大きな省略ではありません。さらに、低速のラップトップドライブにDVDを書き込むことは、めったに行われず、日常生活で特に必要とされないことの1つだと思います。 「ホームメディアセンター」として機能するコンピューターはWindows XPで実行されるため、 dmgをisoに変換し、通常の2層DVDディスクに書き込む以外に方法がありませんでした。 残念ながら、最も人気のあるdmg2iso Windowsスクリプトは画像の処理を拒否し（明らかに画像サイズが大きすぎる-6.66GB ）、私は重要な操作をしなければなりませんでした。 LANケーブルが802.11gよりも速いため）ネットワークケーブルを介してMacBookProをホームPCに接続し、端末を開いてmacosxhints.comで見つかったコマンドを実行し、それに必要なパラメーター（ PCの入出力ファイル）を示しました。 MacBook Proは、ネットワーク上のdmgイメージを標準のisoに変換しました。

hdiutil convert /path/to/filename.dmg -format UDTO -o /path/to/savefile.iso


このプロセスには約30分かかり、その間に朝食を調理する機会が与えられました。 結果のイメージは、 Nero Expressを使用して16倍速でDVD-DLに書き込まれました 。

イメージの変換に関していくつかの懸念がありました。dmg ブートパーティションはisoのものとは異なり、 Macの起動時と終了時にドライブが駆動しないというレビューを少し前に読みました。 しかし、ディスクをMacBook Proに挿入し、表示された「 Mac OS X Leopardを 再起動してインストールする 」ウィンドウをクリックすると、ブナが再起動しました。 インストールがMac OS X Tigerから呼び出されたという事実により、すべてのデータを保存する必要がありました。 Leopardのインストールが開始されました。 システム言語（予想どおり、ロシア語がシステムレベルで公式にサポートされるようになった）とインストールハードドライブを選択した後、インストールはうまくいきました。 このプロセスには約1時間かかりましたが、これは十分な長さです。 私の場合のインストール時間は、すべてのユーザーデータとパラメーターを保存することで説明できると思います。 プログラム、メールボックス、ドックのアイコン、ドキュメントなど、すべてのデータが保存されています。 したがって、更新の方法に非常に満足しています。

最近、 Radio-Tポッドキャストを聞いたので、尊敬されるbobuk bobukが最初にサードパーティソフトウェアを使用してBackUpを作成し、次にシステムを最初から作成しました。 最も簡単なインストールパスであるTigerの「上」を選択しましたが、失うことはありませんでした。



システムは、作業の速度からユーザーインターフェイスの更新まで、ほとんどの部分で間違いなく満足していました。 私は、 Leopardでの作業の印象に関する一連の小さなメモが続くと思います。
