Windowsインストーラーの学習を開始するには？

Windows Installer Technology（MSI）は、デスクトップおよびサーバーバージョンのWindows用のソフトウェアを配布およびサポートするシステム管理者の世界における事実上の標準です。 MSI形式は、すべての最大の構成管理システム（Microsoft SCCM、CA Unicenter、その他多数）でサポートされており、一部のソフトウェア配布システムでは、唯一のサポートされている形式です。

このテクノロジーの理論的および実用的な経験は、Infopulse Ukraineでの経験の中で得られ、MSIパッケージの作成および企業環境での自動インストール用の既製のMSIパッケージの準備に関連していました。





なぜこれが必要なのですか？



Windowsネットワーク上のアプリケーションの集中インストールに関与している場合、MSI形式の知識と、この形式でパッケージを作成および編集する機能は、展開用のパッケージを準備し、問題を診断するときに非常に役立ちます。 ヨーロッパとアメリカでは、Windows Installerテクノロジが多くの大規模な顧客から需要があります。そのため、MSIの知識は、システム管理者にとって有益です。



勉強を始める前に知っておくべきことは何ですか？



複雑なことは何もありません。 Windowsマシンの小規模な（できれば大規模で階層的な）ネットワークの管理とサポートの経験がある場合は、必要な基本的な質問と概念に既に出くわしている可能性があります。 開始する前に、Windowsファイルシステムの構造、ユーザープロファイルの構造、およびレジストリの構造を理解していること、Windowsにあるプロセスとサービスを知っていること、ユーザーアカウントを管理できること、イベントログの場所を知っていることを確認してください。 MSIパッケージの作成方法を学習するために、開発者の経験は必要ありませんが、もちろん、そのような経験がある方が良いです。 また、SQLクエリを作成できる必要はまったくありませんが、これは便利な場合もあります（結局、すべてのMSIファイルは小さなリレーショナルデータベースです）。 ただし、技術文書を英語で読んで理解できる必要があります。これがないと、ほとんど何も機能しません。 理論を勉強するとき（および実際にパッケージを作成するとき）、少なくとも最初はWindowsインストーラー標準に厳密に従うようにして、InstallShield / InstallScriptなどのユーティリティを避けてください。そうすれば、MSIパッケージを正しく作成する方法を理解できます。



サンドボックスを準備し、練習を開始します





仮想マシンをインストールして構成することから始める必要があります。 学習プロセスでは、多くのことを実験する必要があるため、サンドボックスで行う方が良いでしょう。 無料の製品が必要な場合は、 VirtualBoxまたはVMWare Playerをインストールします（専門的なタスクの場合、機能は十分ではないかもしれませんが、トレーニングには最適です）。 もちろん、仮想マシンでは、Windows（XP以上）を選択してインストールする必要があります。

Sysinternalsのユーティリティの経験がある場合-すばらしい。 そうでない場合は、ギャップを埋めるときです。 少なくとも2つ必要です。インストール中またはその操作中にアプリケーションのファイルとレジストリアクティビティを監視する-Process Monitor 、LocalSystemアカウントに代わってインストールを開始する-PsExec 。 メモ帳などの単純なアプリケーションでこれらのユーティリティを使用する練習（ファイルを開くとき、ファイルを保存するときにメモ帳が機能するリソース、LocalSystemユーザーとしてメモ帳を実行する、ファイルの保存とファイルの関連付けの設定を試す）、これらのスキルは後で役立ちます。



Panda QuickShotまたは他のソフトウェアをインストールして、システムの状態を比較します。 基本機能を理解し、新しいアプリケーションのインストール時またはシステム設定の変更時にシステム内のファイルとレジストリの変更を追跡する方法を学びます（最も簡単なのは、メモ帳に新しいファイルの関連付けを登録し、デスクトップの壁紙を変更し、これらの設定が保存されている場所を追跡することです）。 創造力を身につけ、実験の分野は広大です。



単純なアプリケーション（Orca、InstEd、Firefoxなど）のMSIインストーラーをダウンロードします。 上記のユーティリティを使用して、仮想マシンにパッケージをインストールし、インストールするリソースを調べます。 Windowsインストーラーのコマンドライン（msiexec /？）の使用方法を学び、さまざまなインストールオプションを試し、結果を比較します。 コマンドラインからアプリケーションをアンインストールし、パッケージをサイレントモードでインストールします。

ホストマシンにInstEdまたはOrcaをインストールします。 これらのユーティリティのいずれかを使用して、仮想マシンにインストールしたMSIパッケージを開きます。 MSIパッケージ（ファイル、ディレクトリ、レジストリ）のメインテーブルのレコードを、このパッケージによって仮想マシンにインストールされたリソース（ファイル、フォルダー、レジストリキー）と比較します（Panda QuickShotを使用してスナップショットを比較した結果）。

完成したパッケージを試して、これらのトピックに関するドキュメントの読み取りをサポートします 。時間の経過とともに、パッケージを自分で作成してみることができます 。 この知識は、企業環境での展開の準備時にMSIパッケージに小さな変更を加えるのに十分です（通常のタスクは、パッケージプロパティの設定、レジストリの編集、ファイルとショートカットの削除、プログラムの追加と削除でアプリケーションの[削除]ボタンを無効にすることです）。 パッケージを商用運用する前に、パッケージに変更を加えた後、厳密なテストを行うことを忘れないでください。 テストするときは、 この記事で提供されている推奨事項を必ず使用してください 。

ユーティリティとドキュメントの概要



機能が制限されたMSIファイルのシンプルなエディターを無料で使用できます。ここでは、従来のMicrosoft Orcaとかなり便利なInstEdについて説明します。 これらのエディターを使用すると、完成したMSIファイルに小さな変更を加え、トランスフォームを作成および適用できますが、MSIパッケージをゼロからゼロから作成することは非常に困難です（大きなパッケージの場合はまったく意味がありません）。 しかし、MSIテクノロジを本当に学びたい場合は、InstEdまたはOrcaでいくつかの簡単なパッケージを収集し、Windowsインストーラー標準に準拠しているかどうかを検証するだけです。

セットアップキャプチャ（アプリケーションのインストール前後のシステムの「スナップショット」の比較を使用して、いわゆる「古い形式」からMSI形式にインストールパッケージを作成または変換するための特別なユーティリティ）は、無料のAppDeploy Repackagerです 。または、これらのユーティリティを使用して作成されたより深刻なMSI Generator .MSIパッケージには、MSIエディターを使用して実行できる改良が必要な場合があります。

MSIパッケージを使用したフルサイクルの作業をサポートする本格的な（非常に高価な）IDE-専門的にMSI技術に取り組んでいる場合、これらのプログラムなしでは実行できない可能性があります。 最近まで、FlexeraのAdminStudioとシマンテックのWise Package Studioの 2つの製品がこのセグメントを支配していましたが、最近シマンテックはWise Package Studioの開発の中止を発表し、市場はFlexeraの手中にあります。 MSIとの連携に加えて、AdminStudioでは、Microsoft App-V、VMware Thin App、およびCitrix XenApp形式の仮想パッケージを操作できます。 トライアル目的のために、トライアル21日間バージョンが利用可能です。

それとは別に、 WIX （Windows Installer XML） -XML記述からMSIファイルを作成するためのユーティリティセットに言及する価値があります。 このプロジェクトは、オープンパブリックパブリックライセンスの下でMicrosoftによってリリースされ、コマンドラインインターフェイスをサポートし、インストールパッケージのバージョンの作成を自動化するために使用できます。 Habréには良い説明があります。
