通信省と外務省は、BRICS諸国にルートDNSサーバーの自律システムを作成します。





今年の10月末に、ロシアの安全保障理事会は、通信と外務省に対し、BRICS諸国（ブラジル、ロシア、インド、中国、南アフリカ）でルートDNSサーバーの自律システムを開発および作成するよう指示しました。 これまでのところ、重複したルートドメインネームサーバー（DNS）の自律システムの作成についての議論を開始することについて話し合っています。



この要求に関する情報は、ロシアのプーチン大統領が11月5日に署名した安全保障理事会会議の議事録に含まれているとRBCは報告している 。 現在、グローバルDNSシステムは、すべてのトップレベルドメインに関する情報へのアクセスを提供する13のルートサーバーに基づいています。 しかし、これらのサーバーは欧米諸国によって管理されているため、国内の政治家に深刻な懸念を引き起こしています。



「...ロシアの安全に対する深刻な脅威は、情報空間で攻撃的作戦を実施する西側諸国の能力と、それらを使用する準備ができていることです。 インターネットガバナンスの問題における米国と多くの欧州連合諸国の優位性は依然として残っています」と、安全保障理事会の資料で提案の理由を説明しています。



ロシア当局は、インターネットアドレス空間を管理する独立した国際組織ICANNが米国の管理下にあると繰り返し主張しています。 Igor Schegolevは、ロシアがICANNの意思決定システムの変更を主張していると述べました。



BRICS諸国向けの自律DNSシステムを作成するプロジェクトに精通している専門家は、そのようなプロジェクトを実装する可能性を疑っています。 だから、会社の代表者

RunetのDNSインフラストラクチャをサポートするインターネットテクニカルセンター（TCI）は、インターネットのドメインネームシステムは階層的であり、ルートは1つしか存在できないと述べました。 したがって、グローバルな独立ネットワークの現在の構造内では、情報がルートサーバー間で一点から分岐するため、これを達成することは不可能です。これがIANAです。 したがって、国際管理者から独立したルートドメインネームサーバーのシステムの作成は、既存のインターネットから独立した代替インターネットの作成と同等です」と、TCIの代表は述べています。



Geektimesで報告されたように、2014年に通信省は、外部の脅威に対するRunetの抵抗に関する演習を実施しました。 これらの演習の目的は、アドレス指定システムでの違反の可能性を検証することです。 「演習の条件の下で、domain.ruに関する情報がICANNデータベースから削除されたという事実により、DNSネットワークは不適切に機能しました。 TCI、MSK-IX、およびその他の通信会社は、当然のことながら、モデルネットワークのレベルで国内インターネットセグメントの操作性を維持することになっています」



彼は、ロシアにルートDNSサーバーのミラーがあれば、システムの操作性を実現できると考えています。 たとえば、ICANNがルートサーバーからドメイン情報を削除した場合、このデータはロシアのサーバーに保存されます。 ロシアのインターネット全体がこのサーバーに対して閉じられている場合、すべてが必要に応じて機能します。 このようなスキームは、数年前の演習で解決されました。 確かに、それは緊急事態の問題であり、ロシアの鏡の恒久的な運用の問題ではなかった。



2014年の秋にロシアがトップレベルのサーバー.ruおよび.rfの複製ミラーインフラストラクチャシステムを作成したという事実は、TCIの代表者によって報告されました 。 この問題は現在解決されています。 グローバルDNSを複製する独自の自律インフラストラクチャを作成するプロジェクトに関しては、このアプローチはグローバルネットワークの断片化に徐々に貢献します。 「この方向の動きがロシアでデジタル経済を構築するためのイニシアチブの開発と並行して活性化されているのは奇妙です。 このような動きは相互に排他的です」とPIRセンターコンサルタントのOleg Demidovは言います。



ところで、ロシアのプロジェクトは、ルートDNSサーバーに代わるものを作成することが提案されている唯一のプロジェクトではありません。 以前は、いくつかの組織が同様の構造を開発して実装していました。 Open Root Server Network、OpenNICなどのプロジェクトについて話し合っています。 ルートゾーンの現在の状態をコピーしますが、必要に応じて、トップレベルドメインで独自のアドレススペースを作成できます。



たとえば、検閲に対処するために、OpenNICは独自の.pirateドメインを作成しましたが、このドメインはシステムに代替DNSを登録したユーザーのみが利用できます。 早くも2000年、Internet Engineering Councilは代替ルートDNSを「単一のインターネットを分割する可能性を開く」と批判しました。
