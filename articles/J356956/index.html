絵文字からOS Xのパスワードを作成するとどうなりますか？

好奇心user盛なユーザー、Artiom Dashisnkyは疑問に思いました。OSX Yosemieのパスワードに絵文字を使用することは可能ですか？



それは判明しました-それは可能です。



その後、Artyomには別の問題がありました。 認証画面にはネイティブキーボードのみがあり、絵文字はありません。 パスワードを入力するには？ この質問で、ユーザーはStackExchangeプログラミングの聴衆に目を向けました。



ブートディスクはFileVaultで完全に暗号化されています。



Artyomはここで3つのオプションを見ますが、それらのいずれを実装するかを知りません：



1.パスワードフィールドに絵文字を入力する方法を見つけます。

2.何らかの方法で、別のコンピューターに接続し、そこからパスワードを入力します。

3. FileVaultを使用しているにもかかわらず、パスワードを回復する方法を見つけます。



そのような状況で何をすべきか？



正解はTom Geweckeからです

画面の右上隅にオプション「その他の入力ソース」があります。そこで、 Unicode Hex Inputを選択する必要があります。 その助けを借りて、あなたは絵文字または他の文字を入力し、あなたはそのコードを知っている必要があります。 Character Viewerで見つけるか、インターネットで見つけることができます。 たとえば、飛行機U + 2708✈。



Unicode 16進入力キーボードから文字を入力するには、 Optionキーを押したまま4桁のコードを入力する必要があります。



シンボルがU + 1F600のような5桁のコードに対応する場合、2つの4桁のサロゲートを見つける必要があります。同じCharacter Viewerまたはインターネットのfileformat.infoなどのサイトを調べます。 たとえば、コード1F600は代理D83DおよびDE00に対応します。 この場合、 Optionキーを押しながらD83DDE00と入力します。



そして、管理者アカウントで二度とおかしなことをしようとしないでください。



基本的に言えば、Appleはパスワードの絵文字を禁止することで、この小さな悪夢を防ぐことができたでしょう。 一方、顔文字からパスワードを取得するのは非常に楽しいです。 そして、覚えることは簡単です。



標準のUnicode文字からこのようなパスワードを提供します。



❤❤♞☂☭☢☢



この写真は、雨の中でソビエトチェルノブイリの放射線爆発から逃げているお気に入りの馬としてすぐに表示されます。 これをどうやって忘れますか？ つまり、このようなパスワードは非常に高いエントロピーを持ちながら、いつまでも覚えやすいものです。



情報セキュリティの観点から、おそらく顔文字からのパスワードは、一般の人々に推奨されるプラクティスと呼ばれることさえあります。 エントロピーは、使用する標準パスワード（123456、password123など）よりも明らかに高くなります。



エントロピーが高いにもかかわらず、 Optionキーを押したままそのようなパスワードを入力するのは非常に不便です。 3本の絵文字を入力し、1本の指でコードを入力している間、他の人は長い間ASCIIの40文字のパスワードフレーズを入力しました。
