音声キーボードレイアウトJCUKEN

ほとんどのソビエトのコンピューターでは、JCUKENの音声レイアウトが一般的でした。 当時コンピューターを使い始めた人はそれを見つけて、QWERTYがアメリカのコンピューターの出現に支配される前に実際に研究したと思います。







フォネティックレイアウトの利点は、ベースとなるレイアウトが調査される場合、非常に迅速に調査されることです。 音訳で印刷することも非常に便利です。また、刺激が間違ったレイアウトで印刷を開始すると、abracadabruではなく、読みやすい単語が表示されます。



その結果、私は自分でこのレイアウトを作成する実験を行い、最小限の時間で盲目的に入力することを学びました。



レイアウトを少し変更して、最新のキーボードとの互換性を高めました。さらに、コンマを持ち歩きたくありませんでした。 変更は重要ではなく、文字Xは以前の場所（H）に残され、文字Bはbに転送されます。 セミコロンとコロンはbに移動します。 最初はロシア語でVがVであったバージョンもありましたが、最終的には古典的なバージョン（Wの文字）を支持して拒否しましたが、これはCtrl + Vで不便をもたらします







その前に、私は盲目的に次のパラメーターで入力していました。

-YTsUKENのレイアウトでのロシア語テキストの印刷速度-300〜350文字/分。

-QWERTYレイアウトでの英語テキストの印刷速度は、270〜300文字/分です。



実験では次のことが示されました。

-すぐにJCUKENで80文字/分の速度で印刷できます。

-5日間、このレイアウトで1日30分/時間印刷するようにトレーニングしました。

-印刷の時点で、彼はロシア語の単語を印刷として提示することがありました。

-レッスン後に毎日測定された速度は、1日あたり1分あたり約30文字増加しました。

-5日間の終わりに、速度は毎分230文字に増加しました。これは完全な作業に十分でした。

-これらの5日後、速度の増加はほとんど停止し、エラーの数は減少し始めました。

-戻るためにQWERTYで印刷することはできません。再学習する必要があります（実験を繰り返す場合はもう一度考えてください）。



今、速度の面では、私は徐々に前のQWERTYレベルに戻りつつありますが、まだ完全にエラーを取り除いていないため、一部の文字は以前のように入力されることがあります。 また、最小限の時間でこのレイアウトを学習したいという非常に強い欲求があったことを考慮する必要があります。そうしないと、実験をオフにして以前のスキルを返す必要があります。



Windowsのレイアウトは、 klava.org / layout / xsmit.zipから入手できます。

Ubuntu Linuxにインストールする手順は私のLJにあります

Macintoshバージョンはまだ作成中です。



Klavarogオンラインキーボードシミュレータでトレーニングできます。レイアウトのリストでは、選択した英語のテキストを「jcuken」と呼びます。



この実験の結果、彼は不便であるにもかかわらず、JCUKENにとどまることにしました。 私はラップトップに新しいキーボードレイアウトを貼り付けますが、見た目だけのために、盲目的に入力します（友人をあざ笑うために:)



YTsUKENでブラインドタイピングを十分に習得した人、QWERTYをゼロから学習したいという欲求はありません（これはかなり長い時間です）。タイプライターの打楽器メカニズムがくさびをとらないように作業を遅くするために開発されたすべての人に課されたQWERTYの下で曲がり、トレーニング中に小指が非常に苦しみます。



すでにQWERTYを学習していて、他の人のコンピューターで作業している人は、このレイアウトの学習を控えることをお勧めします。これにより、顕著な印刷速度は得られず、不必要な困難が生じます。 また、YAVERTAの音声レイアウトもあり、最初はすべてのオペレーティングシステムで使用できます。これは、QWERTYに長い間取り組んでおり、YTsUKENをまだ研究していない人に適しています。



ドヴォルザークなど、他のレイアウトで働く人々の意見を知ることは興味深いでしょう。



PS WとVを入れ替えたいという非常に強い欲求に悩まされ、Ctrl + VはCtrl + XとCtrl + Cの隣になり、Wは同様のFと一緒になり、ロシア語Vの代わりに音訳Vを書くことが受け入れられます。
