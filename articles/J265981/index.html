ゼロからプログラム。 「今」は開始するのにふさわしい時です



（ A. P. Ershovによる記事「 プログラミング -2 番目のリテラシー 」のイラスト）



スティーブ・ジョブズは、1行もコードを書いていなかった、とビル・ゲイツは書いた。 Paul Grahamは一流のプログラマーであり、Peter Thielはトレーニングを受けた弁護士で、PayPalの共同設立者でPalantirの所有者です（彼は何かを書いたのですか？）。



#tcehの Nikita truesnowは、 「Hacking into Ruby」コースで、どんな人でも、ゼロからプログラミングする方法を教えてくれると教えてくれました。 私は彼に尋ねました：「あなたは人文科学を扱うことができますか？」そして、私たちは「人文科学」がプログラミング言語を学んだ場合があるかどうか疑問に思いましたか？



ホームレスのプログラマーはすぐに思いつきましたが、アプリケーションが成功した後、彼は「禅僧」であり続けることを好みました。 他の例はありますか？ 誰もがコーディングを学べるインスピレーションを与える例-盲目のプログラマー。 Habréの記事「目を閉じてコードを見る」を読んだとき、自分の言い訳以外に障害はないことに気付きました。



そして、私はあなたのために私の人生から3つの物語を持っています。 たった3つの物語。



History One-レーマー



「コンピューター」の「ラマー」という言葉を読みました 。 そこで定義されたのは、「Lamerは決して、どのように（どのように穏やかに表現するのか？）プログラムすることを知らない文盲の人ではありません。」 真実に触発されて、私は走ってクラスメートにこれについて話しましたが、彼らは超大国を習得するよりもサッカーをすることを好みました。 面白いのは、彼ら全員がプログラミングによってお金を稼いだことですが、私はしませんでした。 1週間後、私はプログラミンググループがあった若い技術者のための地域ステーション（KSUT）にサインアップしました。 そこでは、初心者向けにqbasicが提供されました。 そして、Cで失われた神々



それで、どういうわけか「神」の一人が私をモニターに呼び、「これはできますか？」と尋ねました。 そして、ユーザーがコマンドラインに任意の文字を入力するのを待っているプログラムを見せてから、90度回転させました。 びっくりしました。



おそらく、私は夢中になって、どうにかして、可能なことをすべて学び、最もクールなプログラマーになることを決めました。 充電は何年もの間十分でした。 私はこのケースを忘れて、数年が経ち、それを覚えて、私は20分の基本的な議事録に数十行しか書きませんでした。プログラムは完璧に機能しました。



おわりに パダワンとの個人的な関係は非常に重要です。 おそらく、あなたの単純なプログラムは人の運命を決定し、ライフパスの選択に影響を与えるでしょう。



2番目のストーリー-最初の暗号



アイデアは突然思い付いた。 私は眠りにつくことができないように来て、コーディングをしました。 暗号から、シャーロック・ホームズの人々の踊りしか知りませんでした（これはクラス8または9で、私は6か月または1年間プログラミングに従事していました）。 簡単な分析で簡単に開けないようにコードを書きたいと思いました。



アイデアは次のとおりです。ロシア語と英語の文字、特殊文字の形で、ロシア語のアルファベットの各文字に5文字の配列を割り当てます。 暗号化するとき、それらの1つをランダムに選択します。 その結果、「拡張された」単純な置き換えが行われました。 今ではそれを破る方法が明確になっていますが、それから私にとっては並外れた高さがありました。 3〜4年後にコンピューターセキュリティを入力します。



おわりに 興味深い問題を解決するのは素晴らしいことです。 しかし、自分でタスクを見つけて見つける方がはるかにクールです。



歴史3-BASICの勝利



私は2年間プログラミングに携わってきました。 そして、学校に現れた...コンピュータサイエンス。 何らかの理由で私は弱いグループに送られました。 後でわかったように、私の先生は私の顔が好きではなく、彼女は私が馬鹿だと思った。 私の友人は監督のオフィスに押し入り、地域のプログラミングオリンピアードで賞を受賞したと言いました。 分解が始まりましたが、私はあまり気にしませんでしたので、私は弱いグループにとどまり、そこでExcelで数値を入力するように教えられました。 最初の1週間は、プログラム全体に合格し、落ち着いてコンピュータークラブでDiablo 2をプレイしました。



そして、オリンピックでは面白い出来事がありました。 許可されている言語はC、Pascal、および私はハードコアBASICでした。 出ました。 タスクはexe'shnikの形式で行われ、exe'shnikを偽造する基本をそわそわして見つけました。 プログラムはテストに合格し、テスト自体はサーバーで取得されました。 1つのタスクは純粋に計算でした。 中心の座標と2つの円の半径が与えられました。 円の相互配置をテキスト形式で提供する必要がありました。円は交差し、接触し、一方が他方に重なり、一致し、その他です。



数式をいじるのは面倒で、最近はグラフィックスを扱っていました。 この問題はグラフィカルに解決され、1つの色の円を描いてから、2番目の円に沿ってピクセルごとに「スキャン」しました。 プログラムがモニターなしでサーバー上で動作することはできないという大きな恐怖がありましたが、うまくいきました-10のうち10のテストタスクの解析には高いポイントがありました。 50人が著者として1対1で問題を解決しました。



おわりに 学校は悪です。 結果は素晴らしいです。 ボックスの外側を考えることは楽しいです。 常に回避策があります。

    例えば
    一般的な演習：

    Project Eulerには、500以上の数学の問題（数論、数体系など）が含まれており、どの言語のプログラミングでも解決する必要があります。

    Code Abbeyには、200以上のプログラミングタスクが保存されています。 決定については、125人に証明書が発行され、多くの学生の意欲を高めます。

    RosalindはProject Eulerのような別のサイトで、200を超えるバイオインフォマティクスタスクを選択できます。

    Codingbat.comは、初心者と上級プログラマの両方にJavaとPythonの演習を提供します。

    codegolf.stackexchange.comは、ソフトウェアパズルが公開され、議論されているサイトです。

    Rubyクイズは、さまざまな複雑さの短いプログラムを書くように求められる一連のパズルです。 元のソリューションはRubyで記述されていますが、任意の言語で実装できます。



    プログラミングオリンピックの問題：

    UVa Online Judge-オンライン意思決定システムを備えた、オリンピックのプログラミングからの数百のタスクのコレクション。

    TopCoderには、さまざまな競争から長年にわたって収集されたアルゴリズムに関する複雑なタスクが含まれています。 彼はまた、月に数回、プログラミングオリンピックを開催しています。

    CodeForcesはTopCoderに似ていますが、コンペティションや「仮想コンペティション」などの特別な機能のタスクが多く含まれています。

    TimusはUVAと同じです。 最新の競技会（世界および地域レベル）のタスクが含まれています。

    SPOJ -UVAと同様に、膨大な数のプログラミング言語から選択できます。

    USACOには、アルゴリズムのいくつかのタスクと、それらの解決方法に関するガイダンスが含まれています。



    特定の言語の場合：

    Prolog、Lisp、および同様の言語については、 P-99およびL-99、99個のタスクのセットにアクセスして、これらの言語での有効性を高めてください。

    Python Challenge-パズルとパズルを組み合わせて、Pythonを本格的に使いこなすのに役立ちます。

    4Clojureは、Lispに似た言語であるClojure用に特別に設計されたプログラミング演習のセットです。
