ラップトップグラフィックカードのアップグレード

ここでは、5年前のサムスンR45から Pixel Shader 3.0を搭載していないATI Radeon Express 200Mグラフィックカードを使用してゲーム用のかなり強力なマシンを作成する方法について説明します。プログラミング実験。







サムスンR45があります。

1. ATI Radeon Express 200Mグラフィックスカード

2. 512MBのネイティブメモリ

3.Intel Core 2 Duo 1.6GHz T5500プロセッサー



タスク ：赤ちゃんを最大限に搾り出します。



解決策 ：

予備作業：

1. RAMをアップグレードします。 ビデオカード用に最大256MBが提供される2GBの最大メモリ。

古い2x256を捨てて、2x1GBを購入します。

2. CPUを交換します。 古いCPUを置き換えるのに適したCPUを探しています。これはIntel Core 2 Duo 2.3GHz T7600です。 バス、詰め物、ソケットが一致することが重要です（FSB、TDP、ソケット）。

3. HDDを交換します。 SSD HDD、320GB用のラップトップネジ、4USBアダプタ、および各500GB用の2つのUSBネジを購入します。

これらの3pp-技術の問題、何も面白い。



準備が完了したら、モンスターの作成を開始できます。



アプローチ1. ビデオカードを交換します 。

ネットワークにはビデオカードの交換に関する情報がいくつかあり、基本的に誰もが交換は不可能または非常に難しいと考えています。 私の場合、R45では、ビデオカードはマザーボードにはんだ付けされており、交換はできません。 さらに掘ります。



アプローチ2. 外部ビデオカードを使用します 。

「外部ビデオカード」トピックに関するネットワーク検索では、 ViDockやAMD XGPなどの多くの結果が得られました。 ExpressCardスロットを介してPCI Express x1バスに接続し、データ転送速度を制限します。 しかし、ビデオカードを備えたそのようなユニットのコストは200ドル以上かかります。



このアプローチを停止します。 ただし、価格が高すぎるため、ビデオカードを変更することはできません。

しかし、私たちは私たちが望むものを知っています。 そして、このようにしたい





そして、ビデオは遅くなりませんでした。



さらに、ネットワーク上にPCI-E x1からExpressCardへのアダプターを作成する台湾のオフィスを見つけました。

PE4Hシリーズと呼ばれます。 次のようになります。





今私たちは絶対に自信を持ってEbayに行き、販売のための追加の15ドルの送料で私たちの824ドルのPE4Hショールを見つけます。

www.track-trace.com/postでムーブメントを注文、支払い、フォローします。パッケージは2010年11月26日に離陸し、2010年12月10日にすでに自宅にいました。



キットには、ATXスイッチBP用のボードが含まれています。



メインボードも：



、PCI-Ex1ワイヤはPCI-Ex1、およびPCI-Ex1用のスロットを備えた特別なExpressCardです。

ドライバーディスクがまったくなかったことに注意してください。 後で判明したように、それらはボードには必要ありません。

外観とマニュアルから、ボードは4つのPCI-Ex1インターフェイス経由でも接続でき、外部電源ユニット、USB、および電源ユニットからのミニ4ピンコネクタを介して電力を受け取ることができることがわかります。



ビデオカードの選択 。 データ転送はPCI-E x16ではなく遅いPCI-E x1で行われるため、バスが一番上に詰まっているため、速すぎるビデオカードではパフォーマンスが向上しません。 ただし、ソフトウェア（ゲーム）をダウンロードするときに、必要な最大メモリ（テクスチャ）をすぐにコピーすれば、それを処理できます。 そのため、大量のメモリを搭載した高速の非トップビデオカードが必要です。 選択はPalit GTX 460 2GBで決まりました。



次に、最寄りの店舗で、指定されたビデオカードを購入します。





また、それに電源（4本の6ピン電源ケーブル：このビデオカード用の2本のケーブルと将来のための2本のケーブル）。





スキームに従ってこの全体を接続します。





そして...驚くべきこと-すぐにすべてが見つかりました！





NVidiaの新しいドライバー用にドライバーを更新します（ここでは、ATIとNVidiaのドライバーがどのように友達になるかが興味深いですが、誰にも不満はありませんでした）。















モニターを接続すると、見事に検出されます（1 | 2はラップトップの2番目のDVIスロットです）。







CUDA、DirectXなど -すべてが正常に動作します。



3DMark06での2つのテスト（すべてが面倒で、結果は予測可能です）：







結論-予想どおり、GTX460は（私のラップトップの）必死の解像度ですぐにタスクを処理しますが、コピープロセスはFPSを著しく遅くします。 爆発、煙、シェーダーが再生されるショットでは-すべてがスローダウンなしで発生しますが、カメラがマップ上を移動してリロードされる場所では-非常に大量のテクスチャで時々ジャークします。



ゲームテスト ：

1. Alien Swarmは最大設定ですが、画面は1024x768です（高解像度-多くのテクスチャーなので、あまり配置しません）。 応答性100％。 2〜6 FPSと、送信できないほどの遅延が発生する前に、画面は一度も動きませんでした。

2. NFSMW。 プログラマーは、テクスチャを使用して完全に正しいことを行いました。 最初は、すべてがメモリ内にあり、追加の負荷が最小限に抑えられます。その結果、1920x1080ではそのような美しさは見られません。 遅れはまったくありません。

3.次に、見たものすべてを開始しました。画面解像度はそれほど大きくなく（1400x900まで）すべて正常に機能しました。 または設定を下げますが、HDで行います。

4. HDムービーの動作は異なります。 5秒ごとに、フレームは消え、目立って不安になりました。 理由は再び同じです-この秒でバッファがビデオカードに転送されました。 しばらくして、プレーヤーにバッファリングを設定し、適切なプレーヤーを選択することで決定しました。



最も興味深いのは、同時に機能するビデオカード間でアプリケーションを切り替えるプロセスです。

これを行うには、単に映画のあるウィンドウを右にドラッグします-ATI Radeon 200Mはこれに対処し、ゲーム-左のモニターに（同時に、左側に-エイリアンの群れなど、右側に-フィルム）。



外見上、ドラゴンは次のように見えます（干渉しないように窓辺に移動しました）。



ドラゴンの太い尾にはATX電源ボードがあります。



幸いなことに、ビデオカードにはまだ2つの空きHDMIスロットがあり、ラップトップ自体にはDVIスロットがあります。

合計で最大5台のモニターを構成でき、そのうち1台がデフォルトのラップトップ画面です。



マイナスの点として、大量のデータをコピーする場合、ボード上の安価な台湾の要素の静かな割れを聞くことができます。 クラッキングレベルは31db-ビデオカードのファンの2倍です。 一般的な背景ノイズの中で、家はほとんど感知できません。

順序は次のとおりです。

1.トレイで、usb-mediaのようにアダプターを無効にします。 その結果、ドライバーはすぐにアンロードされ、メインモニターがラップトップ上のモニターになります。

2.ミニATXボードの便利なボタンを使用して、メインボードの電源をオフにします（ワイヤをほどくのではなく）。

3. ExpressCardを取り出します。 Windowsは、削除の確認時に「tylyn」を公開します。

4. OSは約実行されています。
