Bad Toonz 2Dアニメーションパッケージコード

先日、TOONZプロジェクトの作成者であるDigital Videoと日本の出版社DWANGOが、2Dアニメーションを作成するためのソフトウェアであるToonzプロジェクトのDWANGO社による買収に関する契約に署名したことがわかりました。



当事者間で締結された契約条件に基づき、Toonzが開発したプロジェクトであるOpenToonzへのオープンアクセスが公開されます。 また、これらのプログラムのアクティブユーザーであるStudio Ghibliが開発した要素も含まれます。 たとえば、スタジオジブリは「ハウルのウォーキングキャッスル」、「千と千尋の神隠し」、「ポニョフィッシュ」などの絵画を作成しました。 その中には、Futuramaの漫画もあります。これは、OpenToonzのソースコードに関するこの明らかになる記事を書くきっかけになりました。



はじめに



OpenToonzは2Dアニメーションソフトウェアです。 基本は、イタリアの会社Digital Videoによって開発されたToonzプロジェクトです。 Studio Ghibliはこのプログラムを採用して、長年にわたってこのプログラムの使用に成功しています。 アニメーション映画に加えて、このプロジェクトはコンピューターゲーム（Discworld 2およびClaw）にも関与していました。



パッケージの価格はこれまで10,000ドルでしたが、コードの品質には多くの要望があります。 このプロジェクトは、あらゆる静的アナライザーにとって天の恵みです。 OpenToonzのソースコードのサイズはFreeBSDカーネルの約1/10であり、PVS-Studioアナライザーを使用して40以上の重大なエラーが見つかりましたが、さらに多くがあります！



OpenToonzは、Visual Studio 2013でPVS-Studioバージョン6.03静的アナライザーを使用してテストされました。静的アナライザーは、積極的に開発され、C / C ++ / C＃言語とさまざまなビルドシステムをサポートします。 プロジェクトのコンパイル中であっても、コンパイラの警告の数が急速に増え始めたとき、私は警戒していました。 アセンブリの終了時、その数は1211個でした！ つまり ソースコードの品質管理はほとんど無視されていました。 さらに、一部のコンパイラ警告は#pragma warningディレクティブを使用して無効にされましたが、ここでもエラーが発生しました。これについては以下で説明します。 この記事は、プロジェクトで多くの実際のエラーが見つかったという事実によって区別されます。これは通常、C / C ++を学習し始めたばかりの初心者によって作成されます。 メモリエラーとポインターエラーで検出されたアナライザーの警告の説明を開始します。
