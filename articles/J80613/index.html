ロシアのメディアにあなたのスタートアップの製品について書いてほしいですか？ だからあなたの欲望を恐れて！

すべての人に良い一日を！



スタートアップの製品についてロシアのメディアで本当に書きたいのなら、あなたの願いを恐れてください。 その結果、彼らのプロフェッショナリズムの最も深い危機のために、完全に予測不可能になる可能性があります。 しかし、根拠がないように...



Computerra-Onlineの編集者である Sergei Villanovとの私的なやり取りの過程で、彼はTerraウェブサイトで私の製品について書くことを提案しました。 私は同意して、DefenseWallのHIPSバージョンについてではなく、現在ベータ版である完全に新しいパーソナルファイアウォールエディションについて書くように依頼しました。 セルゲイへの手紙の中で、彼は競合他社と比較した私のプログラムの主な利点について簡単に説明し、Andrei Krupinがプログラムの機能におけるあいまいな問題について議論し、記事について議論するよう私に連絡するのを待ち始めました。 代わりに、Andreiは記事を（1週間で）すばやく書き、それをComputers Webサイトにアップロードします。 涙がなければ、 この作品を読むことは不可能です。 この「作成」を詳しく見てみましょう。





1.そこに記載されている「電子ニュースレター」は、セルゲイビジャノフとの私的な通信であり、それを公開する許可を求めたことを覚えていません。

2. 「すべて問題ありませんが、HIPSモジュールは自動的にユーザーに馴染みのあるアプリケーションをブラックリストに追加し、通常の操作を中断し、設定の保存と製品の更新をブロックします 。 このプログラムには「ブラックリスト」はありませんが、ユーザーのコンピューターへの感染の「ゲートウェイ」であるため、信頼できないゾーンに配置されているアプリケーションのリストのみがあります。 DefenseWallは通常の操作に違反することはなく、セキュリティに重要な設定を除くすべての設定が通常通り保存されます。 それ以外の場合、悪意のあるアプリケーションもそれらを「更新」できるため、すべての製品の更新は信頼できるゾーンからのみ実行できます。 しかし、あなたはそれを好きではないでしょう...

3. 「Internet Explorer、Outlook、Opera、Mozilla、FireFox、ICQ、Google Talk、The Bat！ 他にも数十のプログラムがあります。 」 ユーザーのコンピューターが感染しているのは、それらを通してであるという事実。 それとも、おそらく、過去1年間にInternet Explorerの穴を塞いだマイクロソフトの数を思い出してください。 また、記事にはまったく記載されていない何らかの理由で、「有罪」のフラッシュドライブもありました。

4. 「はい、安全な環境でアプリケーションを起動する機能をネットワークフィルターに導入するという考えは非常に疑わしいです。 」 実際、これはDefenseWallの革新的なアプローチの本質です。外部接続から完全に自動化された保護の助けを借りてサンドボックスの脆弱な「信頼できる」ゾーンを保護し、信頼できるプロセスによって作成される「リスニング」ポートをブロックします。 simの場合、ポートを構成する必要はありません（eMuleやuTorrentなど）、および「学習モード」でも同様です。 つまり、DefenseWall Personal。 ファイアウォールは、ねじ込み式HIPSを備えたサージプロテクターではなく、サージプロテクターを備えたサンドボックスHIPS（サンドボックス）であり、不正な着信および発信接続から保護します。

5. 「私たちの意見では、実際的な頭痛以上のものがあります 。 」 すみません、「私たち」は正確に誰ですか？ たとえば、プロのマルウェア対策セキュリティポータルの外観を見ると、DefenseWallは、ゼロデイ脅威（0日）に対する保護のテストで100％の絶対的な結果を示した唯一のアプリケーションです。 そこには、祖母と、安全な環境でアプリケーションを実行する「疑わしい利点」があります。 しかし、Andreiには、AV-ComparativesおよびAnti-Malwareよりもはるかにプロフェッショナルなプロのセキュリティテスターもあります（詳細は以下を参照）。

6. 「5つのタブがありますが、そのうちの1つだけがいくつかのファイアウォール設定用に予約されています。 他の同様のツールが、システム内の現在のネットワーク接続、転送されたトラフィックの量、使用されているポート、およびその他の情報を動的に示す画面を誇っている場合、DefenseWall Personal Firewallはネットワークアクセスルールを持つアプリケーションをリストするだけです 。 まあ、はい、なぜもっとですか？ このプログラムは、「単純な」ユーザーを対象としています。データを含むこれらの「画面」はすべて意味のないゴミです。 Andreyがこの「退屈」を見つけたからといって、競合他社の間違いを繰り返すべきでしょうか？ そして、「類似のツール」はまだ観察されていません-DefenseWallは世界初のパーソナルファイアウォールサンドボックスです。 さて、またはあなたが好きなように、サンドボックスはパーソナルファイアウォールです。

7. 「信頼性、トロイの木馬モジュールをブロックする能力、悪意のあるコードを導入および隠蔽する最も困難な方法に耐える能力を含むファイアウォールの最も多様な指標を決定するMatousecの専門テストでのIlya Rabinovichによれば、DW PFは最大数のポイントを獲得して成功裏に通過しますマルチレベルの評価システムを通じて 。 」 はい、正確に。 しかし、これらのテストをダウンロードします（ところで、そのうち148個があります！）そして、アンドレイのチェックは弱かったです。 次の「真珠」があります：

8. 「しかし、誰でも閲覧できるMatousecレポートでは、DefenseWall Personal Firewallについての言及は1つも見つかりませんでした 。 」 もちろん、彼はそこにいません！ 「プロアクティブセキュリティチャレンジ」セクションの「よくある質問」セクションを開いて、ベータ版がテストに受け入れられておらず、他のバージョンのパーソナルファイアウォールがまだないことを理解するだけで十分です。

9. 「アンチウイルスポータルVirusinfo.infoの読者の1人が、いくつかのテストの失敗で自慢のファイアウォールをキャッチしました 。 」 問題は、どれだけなのかです。 前述のリンクをたどって読みます：Comodo Buffer Overflowバッファーオーバーフローテストは失敗しました（そしてDefenseWallはこのタイプの攻撃から保護されません）、PCFlankは失敗しました（後で発見されたように、信頼ゾーンから起動されたため、テストは信頼されていないから成功しました"）そして、ポートを隠しません（"ステルス "）。 ええ、はい、ポートは隠れていません。これが原則としてセキュリティと、特にサンドボックスの信頼ゾーンの保護にどのように関係するかを説明してくれますか？

10. 「そして、サイトAnti-malware.ruのフォーラム参加者は、悪意のあるアプリケーションがファイアウォールコンポーネントの動作を妨害する状況に遭遇することができました 。 」 これは、通常、Personal FirewallではなくHIPSで作業していたため、通常は大文字のPearlです。 はい、私はこのVirutを運転しました56-危険なことは何もなく、サンドボックスは100％で動作しました。 しかし、なぜアンドレイはテストに苦労するのでしょうか。反論の余地のない証拠として、1か月前にフォーラムから投稿する方が良いでしょう。

11. 「診断ツールの活用に失敗することはありませんでした。AtelierWeb Firewall Testerユーティリティを起動した結果、DW PFはWindowsに組み込まれたファイアウォールの保護レベルよりも劣らないことがわかりました 。 」 これは、直接の自然科学実験で検証された唯一のものであるため、この作品で最も興味深い声明です。 そのため、サイトにアクセスし、信頼できないブラウザーで「Atelier Web Firewall Tester」（ちなみにMatousecに比べてやや劣るテスト）をダウンロードし、インストーラーを解凍して実行します（DefenseWallが「untrusted」属性を継承したことがすぐにわかります）。 インストール後、テストを実行し、簡単に最大10ポイントを獲得します。 明らかに、私たちはすべてを間違って行っており、Andrei Krupinのみが「正しい」結果を得る権利を持っています。
