百万、百万、百万…青い証明書

非常に多くの異なる証明書があり、ごくわずかしかそれらを理解できません。 今日は、最高レベルの証明書の取得とその方法についてお話します。 そして、最も興味深いのは、この認定に合格した人の一人がそれについて教えてくれます。 猫の下を見てください！







著者に発言権を与えます。



こんにちは、Habr！ 私の名前はValery Samoilovで、あるIT会社の主要な開発者です。 私の主な専門分野は、CAD / CAM / PDMおよびAEC / BIMシステムです。 10年以上、私はAutoCAD、SolidWorks、Inventor、Revitシステムでエンジニアおよびアプリケーションプログラマとして働いています。 私はC ++、Qtを知っており、最近C＃でたくさん書いています。



最近、MCSDステータスの試験に合格したので、経験を共有したいと思います。



なぜ



私は座って誰にも触れずにコードを書いた後、突然、Microsoftゴールドステータスの試験に合格するボランティアを発表したことがわかりました。



当社はすでにシルバーコンピテンシーのマイクロソフトパートナーステータスを持っています。 これにより、特定の特権が得られます。開発ソフトウェアの25ライセンス、Azureを使用するためのローン額、その他の便利なものです。 ゴールドステータスにより、最大100ライセンスを取得し、Azureの量を増やすことができます。 ゴールドステータスが堅実に聞こえるという事実は言うまでもありません。 会社はボランティアに試験の費用を支払い、認定された専門家にさまざまなパンを提供します。



通常の開発者に提供する証明書は何ですか？ 証明書は役に立たないと考え、これを合理化する人もいます。 ある意味では、彼らは正しいのです。なぜなら、証明書を持たないプログラマーは、証明書を持たないプログラマーとすべて似ているからです。



私はこの観点を共有しません。 日常業務では、特定の言語または技術のすべての機能を使用するわけではありません。 試験の準備により、使用されているツールをよりよく勉強できるだけでなく、少なくとも基本レベルで新しいツールを学ぶことができます。 証明書自体は、この領域を理解していることを確認します。 さらに、ほとんどの試験は英語で行われます。これは、英語の技術文献のリーディングスキルを拡大および深める良い理由です。 したがって、ロシア語版がある場合でも、英語の試験を選択することをお勧めします。



準備する方法



「証明書を取得した方がいい」という抽象的な考え方から実践に移行するために、試験プログラムに慣れ、自分の強さを評価し、準備計画を立てました。 はい、他の真面目なビジネスのように、計画が決まります。



試験プログラムはマイクロソフトのWebサイトにあります。 プログラムは頻繁に更新されるため、準備の過程で時々レビューすることをお勧めします。そうでない場合は重要なプログラムをスキップします。 これは特にAzure試験に当てはまります。



プログラムを表示するときに、予備試験の日付を設定できます。 トピックの名前は、ナビゲートするのに役立ちます。 私は準備ができていることに気付いたときに、3〜5日で試験の特定の日時を指定しました。



試験の準備のために、平日は1〜2時間、週末は3〜4時間割り当てました。 毎日準備ができたわけではありませんが、私はこれらの指標を目指しました。 計画は期限だけでなく、会計と管理でもあることを忘れないでください。 したがって、試験の各トピックのキーポイントをマークし、進捗状況を追跡します。 カードとリストの便利なシステムのため、計画にはTrelloを使用しました。



試験は3種類の質問で構成されています。



よくある質問。 このタイプの質問は、テストフォームで少なくとも1回試験に合格したすべての人に知られています：タスク、質問、回答オプション。 1つ以上の正しいオプションを選択します。 必要なオプションの数は、質問の説明に示されています。 通常の質問のブロックでは、自由に動き回って答えを変えることができます。



シリアルの問題。 同じ質問が連続して2〜4回繰り返されます。 質問には、条件、問題、および提案された解決策が含まれています。 ソリューションが適切かどうか（はい/いいえ）を答えます。 彼が答えたように、答えられた質問に戻ることは不可能です-彼はそう答えました。



ケーススタディ。 タスク（背景）、ビジネス要件、技術要件、コードセクションの説明が記載されています。 タスクにはいくつかの質問があります（連続的な質問もあります）。 質問は要件のコンテキストに対応します。要件が何をすべきかを直接言うこともあれば、実際にあなたが本当に欲しいものを分析する必要があることもあります。 1つの試験では、原則として、いくつかのケーススタディがあります。 TKと顧客と連携するスキルは、このような問題を解決するのに役立ちます。



こちらの質問の例をご覧ください（試験形式と質問の種類のセクション）。



どこでどのように取る



試験は、 ピアソンVUEテストセンターで、または自宅/職場のコンピューターからリモートで受けることができます。 私はテストセンターでフルタイムで受講することを好みます。 動員します。 あなたはコンピュータに取り残されています-今あなたはそれであるか、それはあなたです。



試験の代金は、テストセンターまたはMicrosoft Webサイトで直接支払うことができます。 マイクロソフトは、割引のある特別オファーを提供するため、より安価です。 たとえば、9月30日まで、「試験+再受験」のオファーは割引価格で有効です。



指定された時間の少なくとも15〜20分前にテストセンターに来ることをお勧めします。 そのため、必要な手続きをすべて急いで行う必要があります。フォームに記入して写真を撮ってください。 そして、主な目標である試験に合格することに集中する時間がまだあります。



試験の前に、パスポートを除く身の回り品を金庫に持ち込む必要があります。ポケットに何も入れないでください。 マーカー付きの用紙のみが与えられ、コンピューター上に座ります。



すべての試験に英語で合格しました。 ロシア語でAzure試験に合格できますが、これは本物のサムライにふさわしいものではありません。 はい、MSDNは英語で役立ちます。機械学習の成功にもかかわらず、ロシア語への機械翻訳は、控えめに言っても、多くの要望を残しています。



証明書を取得することを固く決心しており、そのために試験の準備と合格の困難に耐える準備ができている場合は、欲しがるMCSDに3つの決定的なステップを実行するだけです。 以下に、この方法で行った経験について説明します。



ステップ1. MCP



MCSDへのパスの最初にあるのは、試験70-483 C＃でのプログラミングです。 それに備えて、 試験番号70-483：C＃でのプログラミングを学習し 、ジェフリーリヒターによるC＃経由の優れたCLRの本を読み直しました。 試験は非常に簡単であることが判明しました。 私は彼の背後でC＃で2年間の開発を行っているプログラマーが問題なくそれを渡すと思います。







この試験の主な難点は、言語の全範囲を網羅していることです。 C＃100％を仕事に使う人はほとんどいません。 準備では、知識のギャップと試験番号に特別な注意を払いました。 これは最も人気のある試験の1つです。ネットワークには、準備プロセスの詳細な説明と合格者の個人的な経験が含まれています（たとえば、 こちら ）。 試験には1つのセクションがあり（50の質問がありました）、質問に戻ることができます。 テストの1つの質問に別の質問への回答が含まれている場合は、冗談ではありません。



70-XXXシリーズの最初の試験に合格すると、 Microsoft Certified Professional証明書が発行されます。 多くはこれで立ち止まりますが、これは私に関するものではありません。 彼は引っ張りをつかんだ、ダースではないと言ってはいけない！



ステップ2. MCSA



したがって、最初の試験に合格します。 次は？ 次に、選択を行います。70-357モバイルアプリの開発 （XamarinではなくUWPについてです）または70-486 ASP.NET MVC Webアプリケーションの開発を有効にします 。



Web用に開発しなかったため、70-357を選択しました。UWPは、私がよく知っているデスクトップアプリケーションにずっと近いからです。 さらに、試験プログラムでは、おなじみの略語であるWPFとMVVMに出会いました。







この試験の主な問題は、体系的なトレーニング資料がほとんどないことです（賢明な本はなく、試験番号もありません）。 無料のMSDNに、試験のトピックに関する多数の記事と公式の模擬テスト （有料）が保存されます。 模擬試験を受けて、後悔しませんでした-質問とタスクは、試験で何に直面する必要があるかについての良いアイデアを与えてくれます。 しかし、もちろん、テストからの質問が実際の試験で行われることを期待するべきではありません（おそらくいくつかは似ているでしょう）。



試験自体は、構造的には70-486よりもはるかに複雑です。一連の質問とケーススタディが表示されます。 あなたは彼らのために準備する必要があります。 たとえば、理解することなく、いくつかの質問を見逃しましたが、残りの質問は成功するのに十分でした。



さて、70-486を渡す方が正しいと思います。 70-357行き止まり-次のレベルを閉じるためのUWP試験。



2番目の試験に合格すると、 Microsoft Certified Solutions Associate証明書を受け取ります。 アソシエイトのランクは、あなたが真のアプリケーション開発者であることを認識するために汗をかく必要があることを示唆しているようです。







ステップ3. MCSD



最終試験として、Microsoftは多くのオプションを提供しています。 目が走る。 しかし、よく見るとすぐに、そのうち3つだけが視点を持っていることが明らかになります。



  • 70- 487 Microsoft AzureおよびWebサービスの開発 （試験70-486の自然な継続として）;
  • 70-532 Microsoft Azureソリューションの開発 。
  • 70-535 Microsoft Azureソリューションの設計 。




リストから、私はすぐに70-487（Web開発は私のものではありません）と70-535（早すぎると判断しました）を投げました。 したがって、1つの試験が残っており、そのために準備を始めました。



これまで、クラウドテクノロジーについては漠然と考えていましたが、試験の作成者によると、PowerShellスクリプトを使用して仮想マシンを同様に正常に構成し、コンテナーを展開し、SQLおよびNoSQLデータベースを理解し、もちろんWebアプリケーションを作成する必要があることが判明しました-およびモバイル-Azureを使用。 さて、これは新しいことを学ぶ機会だと思った。 特に、クラウドテクノロジーがすでに完全な成長の中で使用されているため、デスクトップCADおよびCAMシステムの居心地の良い世界から覗き見することが少なくとも時々必要です。



無料のAzureアカウント、 Exam Ref 70-532Developing Microsoft Azure Solutions and Practice Testで武装し、準備を始めました。 ただし、Azureアカウントには200ドルのアカウントが添付されていましたが、30日間の「有効期間」が制限されていました。 非常に便利です。試験トピックを勉強するときに、実験を控えることができないからです。



前回の試験に関するメインのトレーニングツールは、MSDNでした。 もちろん、試験参照と模擬試験は良い助けですが、それらを生産的に使用するには基本的な知識が必要であり、それは私には十分ではありませんでした。 しかし、最も重要なことは、戦闘サーバーですべてをライブですぐに試すことができることです。



準備には約2か月かかりました。 今回は、「私は来た、見た、勝った」というアプローチの代わりに、合理的な注意を払いました。3か月前には考えもしなかったトピックについて試験を受けることは依然として危険です。 Azure試験の場合、1つの試験価格で「試験+再受験+模擬試験」というプロモーションがありました。



そして、注意が必要でした。 初めて、何かがおかしくなりました。13ポイントでは足りませんでした。



失敗した試験の結果では、あなたが最悪であることが判明したトピックを書いています。 私は集まってエラーを分析し、2回目は正しくやった







MCSD証明書は正式に永続的です。 しかし、知識を最新に保つために、試験の1つに合格することで毎年更新できます。
