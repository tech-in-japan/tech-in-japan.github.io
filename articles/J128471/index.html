タブレットでのWindows 8 Developer Preview

Windows 8 Developer Previewをダウンロードするためのアクセスを許可しました。 主な革新の1つは、タブレット用の洗練されたインターフェイスです。 私は自分自身をセットアップし、これがWindows 7と比較して一歩前進するかどうかを確認することにしました。



指定：x86-タブレットOdeon TPC-10-Atom 1.83GHz / 2GB RAM / 32 GB SSD、Windows 7 / Android 3.2（メインOSはWin7、Androidは興味があるかどうかを確認するように設定されているので、その価値があるかどうかをお知らせします）

要件：Windows 8をインストールし、タブレットの観点からユーザビリティとハードウェア要件を正確に評価します。







インストールプロセス


通常のWindowsのようにインストールされますが、注目に値するものは何もありません（ソースをダウンロードし、ブートし、インストールしました）。

タッチスクリーン、wifi、仮想キーボードは、再起動後すぐに機能しました。 実際のキーボードはインストールを開始するためにのみ必要であり、初期セットアップはそれなしですでに行うことができました。 32ギガバイトで十分で、インストール後の空き領域は20ギガバイトでした。





印象


OSがロードを開始してからデスクトップが表示されるまで、ほぼ瞬時にオンになり、約10秒かかります（さらに、POSTを渡すなど、タブレット自体を起動するのに10秒かかります）。 休止状態から40〜50秒、最初からタブレットに7つ-30。



それはかなり速く、おそらく7つよりもさらに速く動作します。 [スタート]ボタンはタブレットのすぐ下に鋭く表示されます。タイルでMetroシェルを呼び出します（装飾的で、2〜3個のボタンが動作し、古い起動を取得できます-まだわかりません）。 7ドライバーが適切であること）。





これまでのところ、目立った新しいものは何も見えないように見えます。残りの場所からは、わずかに修正された7つのスティックが突き出ています。





仮想キーボードを変更し、多少便利になりました-ウィンドウのサイズを変更せずに画面の端に取り付けられるようになり、大きくなりました。 しかし、数字ボタンは、戻ることができるかどうかのメインレイアウトから削除されました-私が理解するまで。







IE10では、2本の指でピンチズームがスムーズに機能します。 7つでは、これはより離散的でした。 テキストを強調するために「マーカー」を追加しました-スクリーンショットの円は、非常に便利です。





加速度計が検出されなかったため、ターンに対する反応の速度を確認しませんでした。 私は7つとの顕著な違いがあるとは思わないが、iPadまたは3番目のアンドロイドの滑らかさは達成できない。



バッテリーの寿命は測定しませんでしたが、Windows 7との差はないと思いますが、このタブレットから4〜5時間以上は絞ることができません。



一般に、8人はタブレットの見通しを持っていますが、まず第一に、すべてがソフトウェアに依存します。 使用したすべてのソフトウェア（firefox、MSO2007、stdu viewer、2gis）がWindows 8で正常に起動したため、私はまだ7つを返しません。



更新： 768未満の垂直解像度では、メトロアプリケーションが起動しないことが判明しました。 そして、私は1024x600を持っています。 外付けモニターを手に取ると、正常に起動します。 そして、それをオフにすると、実行中のアプリケーションは引き続き動作し、通常は画面サイズに合わせてスケーリングされます。 おかげで海賊



更新2：左側のタスクバー-Windows XPでも同様です。 ワイドスクリーンモニターではより便利です。 また、アイコン付きのタスクバーは7です。
