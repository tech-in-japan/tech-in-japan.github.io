店舗向けの「キューブ」：なぜハイパーコンバージェンスが本当に必要なのか、なぜ単なる流行語ではないのか



古いインフラ



それぞれ1万平方を超える面積を持つ8つの大きな店舗があります。 各店舗-ユーザーとワークフローのあるオフィス。 各ポイントにサーバーノードがあります-取引アプリケーション、ファイルサーバー、ドメインコントローラー、その他のサービス。 通信チャネルは非常に薄く、外国企業の標準によって定義されています。 これは、管理アクションと、夜間の1日の累積ベースとの同期に十分です。 データセンターとの同期または非同期のデータベースレプリケーションについては疑問の余地はありません。差分を毎晩送信するモードのみです。 ストリーマーへのバックアップ。 壁には、店の従業員が1日1回カートリッジを交換するという指示がありました。



このような状況で、Simplivitiを導入しました-ロシアでこのクラスのソリューションを導入した最初のプロジェクトの1つです。 要求は「解決策を教えて」という形ではなく、「非常に多くの力があり、そのようなボリュームが必要」という特定のタスクの形で出されました。 さらに、5枚の高価な鉄のセット、または2枚の高価な鉄のセットのいずれかが判明しましたが、これは見慣れないシャーマニックなシンプリヴィティのものです。 2番目を選択しました。 その結果、単一のスペースを備えた単一のインフラストラクチャと、サイト間の交換が非常に遅くなりました。 とても奇妙なことです。



ここで、シャイタンシステムの機能を説明します。 少し先を見据えて-ファッショナブルなハイパーコンバージェンスと主な機能-グローバル重複排除があります。







各ストアには4〜5個の仮想マシンがあります。 それらのいくつかはインフラストラクチャに関連しています：ドメインコントローラー、ファイルサーバー。 パート-取引のための特定のアプリケーション：重量管理、値札とポスターの印刷、分析サービス。 これらすべてのタスクは、2つのノードSimpliVity OmniCube CN-1400の小さなシステムに収まります。



ご覧のとおり、このプロジェクトでは、SimpliVity OmniCubeは2つのノードのクラスターです。 VMwareはクラスターにインストールされ、各サーバーには内部ディスクがあります。 ディスク領域が仮想マシンに割り当てられます。 有用なスペースが単一のストレージシステムに結合され、サーバーに戻されます。 ScaleIOやRAIDIXのようなSDSは、機能のみが他の人のために少し設計されています。 主なことは、サーバーディスクだけで、別のデータストレージシステムを用意する必要がないことです。



ノードは10 GbEネットワークで直接相互接続されているため、店舗にある高価な10ギガビットスイッチは必要ありません。 仮想化ソフトウェア-VMware vSphere。 その結果、「インザボックス」という新しいストアのインフラストラクチャの典型的なセットができました。 新しいストアを開くときに、サーバーハードウェアについて頭を悩ませる必要はありません。SimpliVityキューブを2枚用意するだけです。 異なるストアのSimpliVityクラスターは相互に認識し、一元管理されます。 VMバックアップは1日に1回作成され、2週間保存されます。 1つのバックアップはローカルに保存され、誤って削除されたり論理的に破損した場合にデータを迅速に取得します。 2番目は、近くの店舗のSimpliVityに保存されます。 2番目のクラスターのストレージは、最初のクラスターから独立しています。



これは1つの実装に基づいて非常に高価です（他のソリューションでも同様に安価に実行できます）が、管理者の時間、冷却、エネルギー、サポートのコスト、および将来の拡張と合わせて、これらすべてが従来のスタック全体に置き換わると考えてください/インフラストラクチャ、それは非常にきちんと美しく判明。 実際のところ、ここは「オールインワン」、非常に単純なコンポーネントのセットです。 VMvaraの知識があれば、管理者の半分しか必要ありません。 このソリューションは、任意の数のノードに簡単に拡張できます。 主なことは、巨大なデータベースにはオールフラッシュアレイがあるため、メガファットデータベースがないことです。 主なケースは、ユーザー端末のブランチとファームです。 テストプロセス中、通信チャネルの組み込み最適化とオンザフライの組み込み重複排除のおかげで、このことは悪いチャネルでも非常にうまく機能することが明らかになりました。



その結果、既存の8つの店舗に加えて、店舗は年に数台の店舗を開いています。 これは地下鉄タイプのサイズです。 30人のアクティブユーザー向けのオフィス。 彼らは、オンプレミスアプリケーション用の典型的なITインフラストラクチャソリューションを選択しました。 新しい店を開くことが計画されていましたが、既存の店では古い鉄を変える必要がありました。 ITプロフェッショナルはインフラストラクチャだけに関与しているわけではないため、ソリューションはできるだけシンプルで、一元管理する必要があります。 理想的には、顧客は店舗に複製できる既製の箱を1つ探していて、一般にITインフラストラクチャがあることを忘れていました。



2番目の要件は、ストアデータのバックアップコピーをリモートサイトに保存することです。 通信チャネルを介したバックアップの転送は、長い間判明していたため、意味がありませんでした。 データは異なるジオポイントに保存する必要があり、管理者はこれを非常によく理解しています。



テスト中



ソリューションテストのレポート（実装前の昨年のレポート）は次のとおりです。 患者は2腕のSimpliVity CN-3400でした。







各ノードは、独自のSimpliVityフロントパネルと内部の特別なPCIeカード（OmniStackアクセラレータカード（OAC））を備えたDell R720デュアルユニットラックサーバーです。 各サーバーには、2個のCPU E5-2680v3、24個の16GB DDR3スロットおよび24個のディスク（4個のSSD 400GB、20個のHDD 1TB 7.2k SAS）があります。







フロントパネルの下のフロント部分には、24個のディスクがあります。4個のSSDと20個の2.5インチSASディスクです。 背面には、2つの電源コ​​ネクタ、5つのネットワークポート、1つのOACカードサービスポート、2つの300GB 10K SASシステムHDDがあります。

通常、各サーバーはデータネットワークへの2つの10ギガビット接続、管理ネットワークへの2つの1ギガビット接続により接続され、IPMIサーバー管理コンソール（iDRAC）には1つのインターフェースが必要です。 これは接続オプションの1つにすぎません。 ギガビットネットワークには接続できませんが、クラスターネットワークとVMトラフィックの両方に10GbEを使用します。 ネットワークカードを追加して、いくつかの追加ポートを選択することもできます。 2日間の構成の場合、10ギガビットネットワークを編成する必要はまったくありません。2つのDACケーブルでノードを互いに直接接続するだけです。



電源、ネットワークをケーブル接続し、IPMIコンソールでIPアドレスを設定したら、システムの初期設定とvCenterへのソリューションの統合を続行できます。



この場合、vCenterは専用のWindowsサーバーにインストールされました。 vCenter Server Applianceもサポートされています。



システムを初期化するいくつかの方法をテストすることができました。 これは、ロシアで最初に提供されたキットをテストしたという事実によるものです（テストは昨年行われたことを思い出してください）。 そして、その上にバージョン2.x生成システムがインストールされました。 このバージョンの場合、手順は次のとおりです。 vCenterをインストールするには追加のトレーニングが必要です。新しいsvtuserユーザーを追加し、仮想スイッチを構成する必要があります。 すべての設定は、システムに添付されているパンフレットに記載されています。

既存のインフラストラクチャの準備に関する推奨事項に従って、システムのIPアドレスを設定し、ノードをvCenterに接続し、SimpliVityフェデレーションを作成し、VMwareクラスターを初期化する必要があります。 すべては、通常のUSBフラッシュドライブ上のシステムにバンドルされているシンプルなウィザードを使用して行われます。 セットアップ中に、スプリットブレインの場合にメインノードを選択するアービターがセットアップされます。 vCenterホストと別のサーバーの両方にインストールできます。



システムを現在のバージョン（3.5.x）に更新した後、システムを再度初期化しようとしました。 最新バージョンでは、インストールはDeployment Managerを使用して行われ、彼はvCenterへのアクセスのみを必要とし、残りは自分で行います。

ここでプリセットが終了し、vCenter-SimpliVityフェデレーションタブで、必要なデータストア、バックアップスケジュールを作成し、残りの機能を使用できます。



一般に、システムのセットアップは非常に簡単で、添付の指示に正しく従えば問題はありません。 提供されたシステムを更新する必要がない場合（IPアドレスが事前に準備され、vCenterが構成されている場合）、ラックのマウント、切り替え、初期構成を含むシステムの展開には約30分かかります。

vCenterのSimpliVity Federationセクションでは、パフォーマンス、物理/論理容量、重複排除と圧縮率を監視し、バックアップポリシーを管理できます。







インターフェースでは、バックアップはスナップショットに似ていますが、技術レベルでは重複排除スペースが1つあり、バックアップを作成するときに仮想マシンのフォークがあります。 SQLおよびExchangeとのバックアップ統合モジュールがあります。

重複排除係数は、私たちの意見では、システムが計算するものであるため、プライマリコピーについて完全に有益ではありません。 残りについては、多かれ少なかれ十分です。



スナップショットを作成した後、VM全体またはそこから個別のファイルを復元できます（Windows OSを備えた仮想マシンのみをファイルにきめ細かく復元できます）。 仮想マシンを復元するとき、「既存の仮想マシンを置き換える」または「新しい仮想マシンを作成する」を選択できます。 同時に、任意のバックアップを任意のリモートデータセンターに手動で復元できます。



すべての基本機能は正しく、エラーなしで機能します。 SimpliVityは、vSphereクライアントにシームレスかつ緊密に統合され、この機能またはその機能を見つけることは難しくありません。



性能



パフォーマンスを測定するために、仮想マシンのパラメーターでVMware IO Analyzer 1.6.4を使用しました：VMバージョン：7、vCPU：1.メモリ：2 GB、VD：16 GB。 OS：SUSE Linux Enterprise11。平均して、負荷で7,560 IOPS（8K、50 / 50w / r、NoRandom）、AllRandomで20,352 IOPSを取得しました。 これはすべてキャッシュを過ぎています。



外出先でディスクを引き裂き、相互接続を中断し、ノードを切断するという私たちのお気に入りのクラッシュテストはすべて、システムが正常に生き残り、コンポーネントが返されたときにその状態を正しく復元しました。 すべてのいじめによるシステムパフォーマンスへの影響は、相互接続の切断を除いて最小限でした。 彼はシステムを2倍高速化するのに役立ちました！ （ただし、ノード間の同期レプリケーションは失われています）。



結論：鉄片は興味深いものです。これは、従来のデータセンターアーキテクチャのほとんどすべての要素をラックサーバーとスイッチのセットで置き換えることができるためです。 システムは分散負荷にうまく対処しますが、統合負荷にはあまり適していません。 シングルスレッドアプリケーションや大規模なデータベースはありません。







ストアでの結果。



実装には2日かかり、サイトに1日かかりました。 ソリューションの開発とテストには半年かかりました。 ほとんどの場合、機器を納入してテストしました。 顧客は、「ボックス内のインフラストラクチャ」を約束するさまざまな製品、つまりハイパーコンバージドシステムとモジュラープラットフォームを検討しました。 バックアップのために個別のソフトウェアを必要とせず、すべてのマシンを単一のセンターから見ることができるため、SimpliVityに停止しました。一般的に、コンソールの分散ではなく、高度な重複排除によるスロップチャネルへの非常に良い姿勢が制御されています。 たとえば、220 GBの使用済み容量があるVMの毎日のフルバックアップでは、570 MBが通信チャネル上を飛びます。 インフラストラクチャ全体の完全バックアップは2〜3時間で完了します。 管理者にとっては素晴らしいように思えましたが、実際には機能します。



IT部門は小規模であり、大企業のように専用の仮想化やバックアップの専門家はいません。 したがって、従業員は、請求書の処理から現金デスクの護衛まで、すべてを行う必要があります。 ストレージ、SAN、サーバー、バックアップソフトウェアのすべての傾向とニュアンスを調査するリソースはありません。 その結果、顧客はインフラストラクチャを標準化し、生活を簡素化しました。 各ストアでサーバー、ストレージ、およびバックアップを購入、構成、サポートする代わりに、既成の「キューブ」をそこに置きます。これは、仮想マシンの起動、効果的な保管、およびバックアップの方法を知っています。 ITの1つのタスクが少なくなりました。 これは、ハイパーコンバージェンスの意味です。



結論：2台のサーバーがストアに配置され、バックアップは5分で構成されます。 それだけです
