Android ThingsとRaspberry Pi3のSmart Lock

2016年12月、GoogleはAndroid Thingsの最初の開発者プレビューバージョンのリリースを発表しました 。 それ以来、プロジェクトは大きく変わりました。 プレビュー版のみが引き続き利用可能ですが、すべてのステップでプラットフォームに新機能が追加され、サポートされるデバイスの数が増えています。







現実の世界でIoTデバイスを使用する新しい例が毎日あり、プラットフォーム自体がより魅力的になっています。 Live Typingの私たちは、モノのインターネットの興味深い世界に飛び込んで、私たちの経験について話すことも決めました。 この記事は、Android Thingsについて聞いたことがありますが、試してみるのが怖い人向けです。 また、「スマートロック」をどのように実装し、自社のオフィスで使用するかについても説明します。













アイデアの説明



問題1：当社は、電子バッジシステムとガラスドアを備えたオフィスを借りています。 多くの場合、従業員は自宅でカードを忘れるか、単にカードなしで外に出て、同僚をノックするか電話して戻ってきます。 カードは、オフィスの内外で磁気ロックに適用する必要があります。 内側でカードをロープに縛っただけなら、外側からキーなしでオフィスに入ることは解決したい問題です。







問題番号2：週末には、さまざまな種類の会議がオフィスで開催されます。 出席者の大部分は同僚ではありません。 その数はさまざまですが、どれだけ多くても、常にドアを開けたままにするように、部外者にキーを渡すことはできません。これは私たちの財産にとって安全ではありません。 そのため、特別な「個人用ドアマン」を任命するか、必要以上にドアをサポートする必要があります。







パススルーシステムを切断、削除、またはアップグレードする権利はありませんが、外部からの接続には制限は適用されません。 ドアにサーボドライブを装備し、顔認識が成功した場合、接続されたカードを読み取りセンサーに回転させることにしました。 顔写真はカメラによって提供されます。 したがって、一種のスマートドアロックを取得します。







このようなロックを使用すると、ブラックジャックと幅広い可能性を持つ独自の識別システムを使用できます。 （たとえば、従業員の面白い写真から、社内で遊び心のある使用のためにステッカーを作成できます）。 2番目の問題について話す場合、城では会議の参加者の写真を登録してスキップできます。 すごいですね







さらに、アイデアは付随するニュアンスと質問に囲まれていました。 写真の撮影を開始および終了するタイミングは？ 写真を撮る頻度と長時間 勤務時間外や暗闇の中でシステムを切断する価値はありますか？ システムを視覚化する方法は？ しかし、これについてはさらに個別に。 プロジェクトの基礎として、公式のAndroid ThingsページからDoorbellの例を取り上げました。 元の例は「ベル」と呼ばれますが、システムユーザーに最小限の労力でドアのロックを解除してもらい、見知らぬ人は中に入りませんでした。 したがって、「スマートロック」と呼ぶ方が正しいと考えました。







謝辞



最初は何もありませんでした。 Raspberry自体も、アクセサリも、それらの経験もありません。記事とドキュメントから得られた理論的知識のみです。 Android Thingsで初めてプレイしたのは、 Mobilatoriumのメンバーがオムスク市のシベリアのIT首都で開催したCodeLabでした 。 Google Cloud Visionの代わりに、 Tensor Flowに FindFace実装を実装するプロジェクトをすぐに開始しました。 バックエンドがどのように機能するかに興味がある場合は、優れた記事「Looking for Familiar Faces」を読むことができます。著者は、顔認識を使用するための原理とアルゴリズムについて詳しく説明しています。 そうでない場合は、上記のCodeLabで行ったように、 Google Cloud Vision + Firebase Realtime Databaseバンドルを使用できます。







私たちがオフィスに戻ったとき、従業員のミシャは必要なすべてのコンポーネントと、最近購入したRaspberry Pi3さえも持っていて、そのようなものに夢中になっていることがわかりました。 Arduinoの研究で[夏学期]（ https://vk.com/mobilatorium?w=wall-130802553_81%2Fall ）を費やした男たちのコンポーネントもありました。 提供されたハードウェアに感謝します。







付属品



スマートロックを実装するには、次のものが必要でした。







  • Raspberry Pi 3-1個;
  • MicroSD 8Gb-1個;
  • NoIR Camera V2-1個;
  • ブレッドボード-1個;
  • 赤外線PIRモーションセンサーモジュール-1個;
  • SG90サーボモーター-1個;
  • フォト抵抗器（光センサー）-1個;
  • プッシュボタン-1個;
  • LED-3個;
  • 抵抗器（1k）-3個;
  • 抵抗器（10k）-1個;
  • ピンジャンパー線-多くの部品。
  • 絶縁テープ-1個


すべてのコンポーネントは、中国のサイトで簡単に見つけることができ、安価に注文できます。また、検索しやすいように、特に名前は英語のままにしました。 キットの費用が約100〜125ドルになることは言及する価値があります。 最も高価なコンポーネントは、カメラとRaspberry Pi3自体です。







実装



理解を深めるために、実装の説明を個別のステップに分けます。 スキームを部分的に組み合わせると、任意のステップで画像を復元する方が便利です。 コードは十分ではありません。Android用のアプリケーションを少なくとも1つ書いていれば、問題はないと思います。 開発には使い慣れたAndroid Studioを使用します。 Dagger、RxJava、Retrofit、OkHttp、Timberなどのお気に入りのライブラリとフレームワークを使用することもできます。







始める前に、 Android Thingsの簡単な紹介とRaspberry Pi3のピンに慣れておく必要があります。 また、このピン配列のカラー写真は優れた視覚的ガイドであり、何度も役立ちます。







Raspberry Piは、異なるハードウェアインターフェイスのセットをサポートしています。 ただし、主にGPIO （汎用入出力）およびPWM （パルス幅変調）に関心があります。 これらは、プロジェクトの実装中にボードとセンサーの間の相互作用の主な方法になります。







さまざまな周辺機器のライブラリがすでに作成されており、それらの多くはすぐに使用できます。 したがって、新しいセンサーの統合を開始するときは、まずこのリポジトリとこのリポジトリに慣れてください。 多くのドライバーがここでコンパイルされています。 おそらく、あなたは正しいものを見つけるでしょう。 そうでない場合、GoogleはAndroid Framework Servicesの機能を拡張する特別なユーザードライバーの概念を提供しています 。 ハードウェアで発生していることをフレームワークにリダイレクトし、標準のAndroid APIツールで処理して、独自のドライバーを作成できるようにします。 簡単に言えば、任意のドライバーでの作業は、次の手順に分けることができます。







  • ドライバーオブジェクトを作成する
  • ドライバーを登録します。
  • イベントを購読する
  • 登録解除および登録のキャンセル。

    この記事では、独自のドライバーを実装する例を見ることができます。

ステップ4：カメラ



私たちは、カメラの使用NOIRカメラV2を 。 およそ$ 45あなたは私たちのプロジェクトのために十分な特性が得られます。







  • 最大解像度：8 M（3280×2464）。
  • ビデオフォーマットをサポート：1080（30fpsの）、720P（60fpsの）、640×480P（90fps）を、
  • 消費電力：250ミリアンペア
