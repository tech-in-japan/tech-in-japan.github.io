BlackBerryが優れている理由と、上司がそれを選択している理由

欧米では、BlackBerryはほとんどビジネス向けのスマートフォンと見なされていますが、わが国ではほとんど知られていません。



その理由は、 BlackBerryは「ダイヤラー」であるだけでなく、個人データや企業データの安全なリアルタイム同期のためのサービスとハードウェアの完全な組み合わせだからです。



BlackBerryは、「すぐに使える」高度なセキュリティと、モバイルオペレーターのネットワークに統合された対応するグローバルインフラストラクチャを備えています。



カットの下の詳細。



歴史ツアー



ハードウェアとソフトウェアは、カナダの大手通信会社であるResearch in Motionによって作成されています。 1984年にマイクラザリディスによって設立されました。 最初に、マイクはさまざまな方向で数年間実験し、モバイルデータテクノロジーのみに集中しました。 1999年、最初のBlackBerryが登場しました。これは最初は単なる美しいポケットベルでしたが、その後、現代のビジネススマートフォンになりました。



ロシアでのBlackBerryの歴史は2005年に始まり、サービスの開始に関する最初のアイデアが発表されました。 しかし、法執行機関側の調整不足のため、それらは実施できませんでした。 2008年になって初めて、ビーラインは規制当局の要件と顧客の要望を満たすサービスを開始しました。 現在、ロシア連邦で利用可能な機能を拡張する作業が進行中です。



サービス



現在、サービスには主に2つのタイプがあります。BIS（個人）とBES（企業での使用のみ）です。



BlackBerry Internet Service -BIS-主に個人を対象としたソリューション。 これは、リアルタイムプッシュ同期、トラフィック圧縮、およびプライバシーです。 簡単にするために、一部の企業ユーザーも気に入っています。 標準のメールプロトコルを使用して外部クライアントと連携できる企業メールサーバーと完全に連携します。 Exchangeの場合、OWAインターフェイスを介してBISを統合できます。 GoogleとYahooのメールボックスを接続すると、アドレス帳とカレンダーを同期するための追加機能が表示されます。



Beelineは、2種類のBISサービスを提供します。無制限のメールトラフィックのみが含まれる通常のサービスと、BISブラウジングも含まれる「オールインクルーシブ」（より高価な）です。



Blackberry Enterprise Server -BES-は、企業のメールサーバー、ワイヤレスキャリアネットワーク、およびBlackberryスマートフォン間の集中化された安全な接続のために設計されています。 BlackBerryは主に企業ユーザーと連携するために作成および開発されたものであり、進化の過程でビジネスのニーズに合わせて最適化されていることに注意してください。 BESは、企業情報とアプリケーションへの安全で簡単なアクセスを提供します。 また、IT部門の従業員がBlackberryソリューションを集中管理できる便利な方法で管理できるツールのセットも含まれています。 MS ExchangeおよびIBM Lotus Dominoメールサーバーと統合します。



BESは：

  • ワイヤレスメールの同期、
  • ワイヤレスオーガナイザーの同期（連絡先、カレンダー、タスクなど）、
  • 会社のアドレス帳を検索する
  • 企業データへのアクセス（イントラネット、ファイル共有）、
  • 高度なセキュリティ設定、
  • ITポリシーをスマートフォンにワイヤレスで適用する機能、
  • スマートフォンからさまざまな企業アプリケーション（CRM、SFA、ERPなど）へのリモートアクセスの組織、
  • 簡単なインストールと管理（サーバーの展開はパートナー企業によって行われ、通常約3時間かかります）。


BlackBerry Enterprise Server Express -BESx-BESのすべての重要な利点と主な機能がありますが、インストールが簡単で、システムリソースへの負担が少なくなります（メールサーバーに直接インストールできるという事実まで）。 実際、BESの「軽量」バージョンであるため、BESに統合された一部のサービス（たとえば、ユーザーのスマートフォンで企業のインスタントメッセージングサービスの機能を拡張できるコラボレーションサービス）がなく、利用可能なITポリシーがより少なくなっています。 ただし、BESのフルバージョンとは対照的に、その紛れもない利点は、ユーザーライセンスを購入する必要がないことです。 ところで、私の意見では、BESxは企業クライアントの90％にとって最良の選択です。 SMEセグメントの製品として位置付けられているという事実にもかかわらず、ほとんどの大企業に適しています。



BESおよびBESxは、サーバーソフトウェアの名前です。 サービスとは、クライアントのサイトにサーバーパーツが存在することを意味します。両方のバージョンのBESは、ハードウェアとソフトウェアの複合体の形で提供されます。 資本コストを削減するために、ハードウェアコンポーネントの内容が異なる3種類の複合体が作成されました。 1つ目は、特定の仕様を満たす物理サーバー、特別なハードウェアとソフトウェア、オペレーティングシステム、BESソフトウェアを含むターンキーソリューションです。 2つ目は、無料のサーバー容量を持っているお客様向けのソリューションで、BESソフトウェアと特別なソフトウェアとハ​​ードウェアのみがお客様に提供されます。 3番目は、ITインフラストラクチャが仮想化（Hyper-VまたはVMware）を使用しているお客様向けに設計されています。



BESのタイプと選択した複合施設に応じて、サービスの配信コストは異なります。 したがって、1 + BES-最も高価、3 + BESx-最も経済的です。



Hosted Blackberry Enterprise Service -Hosted BESは、企業インフラストラクチャ内でBESサーバーをインストールおよび管理したくないお客様向けのソリューションです。 サーバーは、Beelineサイトのリモートにあり、IT専門家によって管理されています。 このオプションには、いくつかの特定の利点があります。

  • Blackberryコーポレートソリューションの機能へのフルアクセス。まるで社内に物理的に実装されているかのように、
  • アウトソーシング（IT管理者を割り当てる必要はありません）、
  • 最小コストと実装時間、
  • ホスティングソリューションと企業の既存のインフラストラクチャとの直接統合。

仕組み



BlackBerryは完全なサービスソリューションです。 モバイルオペレータからの特別な通信サービスが利用できない場合、スマートフォンはグローバルBBインフラストラクチャとやり取りできないため、正常に動作できません。 このサービスの存在により、デバイスがプロビジョニングプロセスに参加できるようになります。 これは、特定のサービスに対するBlackberryデバイスのアクセス権を決定するプロセスです。 プロビジョニングプロセス中に、アクティブ化、変更、管理、およびブロックを実行できます。



これは、特定のBlackBerryデバイスのアクティベーション、後続の標準アクティベーション用の特定のパラメーターセットの割り当て、RIM請求システムの情報の更新などのタスクで表現できます。



次の図は、BlackBerry Provisioning Systemノードとそれらの間の相互作用を示しています。 主要なノードは、BlackBerry Infrastructure（BBI）です。これは、デバイスと直接、およびシステム内の他のノードを介してやり取りします。







プロビジョニングの主な目標の1つは、スマートフォンの正常な動作を保証することです。 大ざっぱに言えば、箱から出してすぐに、すべてのBBデバイスは同じです。 ネットワークへの登録、プロビジョニングシステムとの相互作用、および必要なサービスブックのセットを取得した後にのみ、特定の機能セットが提供されます。



サービスブック-デバイスに送信され、特定のサービスまたは機能に関する情報を含む構成ファイル。 たとえば、BlackBerry Internet Browsing Service [BrowserConfig]サービスブックには、アクセスポイント、ホームページ、デフォルトのブックマークの設定など、インターネットブラウザアプリケーションに関する情報が含まれています。



プロビジョニングプロセスは常に次の場合に発生します：デバイスがワイヤレスネットワークに初めて登録され、新しいSIMカードがデバイスにインストールされ、デバイス設定で「今すぐ登録」オプションが選択された場合（パラメーター設定-高度なパラメーター-ノードのルーティングテーブル）、デバイスファームウェアが更新、ロールバックまたは再インストールされた場合。



新しい接続の例を分析しましょう。 ユーザーは、このまたはそのBlackBerryサービスが接続されているデバイスとSIMカードを購入しました。 SIMカードのサービスに接続することにより、デバイスはblackberry.netアクセスポイントを介してパケット接続を構築できます（ほぼすべてのBBスマートフォンでは、デフォルトのアクセスポイントです）。 このアクセスポイントからのすべてのトラフィックは、オペレータによってBBIにルーティングされます。 これと並行して、通信事業者の課金システムからプロバイダーシステムにコマンドが送信され、どの特定の通信サービスがユーザーに接続されたか、およびどのBB機能セットがそれに対応すべきかに関する情報が含まれます。 一方、スマートフォンはすでにblackberry.netを介してアクティブな接続を構築しており、利用可能なサービスのセットに関するリクエストをBBIに送信しています。 プロビジョニングシステムでは、このデバイスはサービスプロバイダーおよび接続されたサービスとともに識別され、Service Book Pushingノードを介して、選択したサービスのタイプに対応する必要な設定パッケージがデバイスに送信されます。



デバイスは、モバイルオペレーターのネットワークの登録アイコン（3Gについての大文字EDGE / GPRSまたは「ベリー」アイコン）の特別な表示によって、BBIへの接続が成功したことを通知します。



インフラ



ここで、会社AのユーザーAから会社BのユーザーBに電子メールメッセージを送信するシナリオを考えてみましょう。このシナリオによれば、BES（BESx）ソリューションの購入時に電子メールが同期されます。 他のソリューションも同様に機能します。







そのため、ユーザーAはデバイスでレターを作成します。 ネットワークにメッセージを送信する前に、デバイスはメッセージを暗号化および圧縮します。 モバイルネットワークオペレーターはこのメッセージをBBIにルーティングし、この特定のデバイスのメンバーシップは特定のBESサーバーによって決定されます。 A社のBESサーバーは、メールメッセージをピックアップし、復号化して解凍し、メールシステムの特権ユーザー（送信者として承認）を介してユーザーAに代わって送信します。 （ユーザーBのメールボックスへのアクセス権を付与されたメールシステムの特権ユーザーを介して）メールボックスからこのメッセージを取得し、暗号化および圧縮して、BBI側に送信します。 そこから、このメッセージはユーザーBのデバイスにプッシュされ、そこで展開および復号化された後、スマートフォンのディスプレイに表示されます。



ロシアの現実



それとは別に、この機能をロシアで実装する際の最も重大な問題について説明したいと思います。 RIMは、製品の長年にわたる開発と開発を通じて、その周辺にある高度に保護されたソリューションについて、当然の評判を築いてきました。 繰り返し、同社は州のセキュリティサービスのレベルでこの決定に関する対決に参加する必要がありました。 ただし、同社は、BlackBerryサービスのセキュリティを疑うように顧客を惹きつけることはありませんでした。 しかし、ロシアでは、この評判がRIMでトリックを演じました。



注意、妄想！ ある国ではもっと、もっと少ないところでは、私たち全員が「ボンネットの下」にあり、疲れ知らずのコントロールをしていることは秘密ではありません。 しかし、この制御は法律を超えたものではありません。多くの規制法があり、通信事業者として私たちが遵守しなければならない要件があります。 このため、製品は世界中で1つの形式で提供されていますが、ロシアでは追加のソリューションが提供されており、国家安全保障サービスによる制御の可能性を提供しています。 このため、特別なソフトウェアとハ​​ードウェアを各サーバーに追加で提供する必要があります。 このため、世界中で無料で配布されているBESxサーバーがここで販売されています。 同じ理由で、デバイスが海外にあるBESに接続される予定の場合、サービスをSIMカードに接続できません。 このため、ロシア連邦の領土に公式に輸入されたスマートフォンは、いくつかの機能などを奪われています。



これだけでなく、多くの人々が州の安全保障機関が彼のメールを読むことができるという懸念を表明する理由も謎のままですが、同時に少数の人々は同じ「制服を着た人々」が電話での会話を聞くことができると心配しています。 第一に、誰も常に何かを読んだり聞いたりすることはありませんが、たとえばテロの脅威が発生した場合にこれを行うための一連の技術的手段しかありません。 第二に、これらの同じ技術的手段は、法律に従ってのみ使用されます。つまり、特別な根拠と許可の存在を意味します（例えば、裁判所の決定）。 第三に、システムは競合するインテリジェンスに対する非常に高いレベルの保護を提供しますが、理論的には、独自の高価なITソリューションによってのみ提供できます（まだ認証が必要であり、制御する機能も提供します-さもなければ、これは記事になります）。



ほとんどの読者の質問を予想して、ロシアでサービスを開始する前に、2つの選択肢がありました。可能な限り行うことと、ソリューションを絶えず改善するか、何もしないことです。 2番目のケースでは、ユーザーは世界中の太陽の下でしっかりとその地位を獲得したソリューションを利用できなかったでしょう。

BlackBerryは、通常設定されている「通常すぐに使える」保護を備えている点で、真空の球形電話よりも優れています。さらに、企業サービスですぐに使用できる「すぐに使える」保護も備えています。 セキュリティに関しては、ターンキーソリューションの分野で最高レベルの機密性の1つを提供します。
