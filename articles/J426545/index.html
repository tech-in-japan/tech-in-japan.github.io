5 BoFセッション



会議は丸2日間行われているため、牛肉の増加を待っています。



  • 適切なAPIデザイン
  • Java 11-行くかどうか？
  • ツーリング（IDE、CIなど）
  • 生産のトラブルシューティング
  • FPからリアクティブへ


突然興味がある場合、BoFは「羽の鳥」の略で、その起源は最初のIETFミーティングからカウントされます。 Java One参加者は、bofsも頻繁に訪れたことを思い出すかもしれません。



BoFは現在、当社の会議では従来の形式です。 全員が参加できる円卓またはディスカッショングループのようなもの。



スピーカーと参加者による区分はありません。全員が平等に参加します。 これは、少なくとも3つの理由で重要です。



  • レポートの形式は、特定のトピックに焦点を当て、注意をそらす機会がないため非常に便利です。 全員が座ってスピーカーを聞いています。 BoFセッションは、すべての参加者間の完全なコミュニケーションに適しており、幅広い関連トピックをカバーできます。
  • 参加者の中には、多くのクールな専門家がいます。 彼らはこの会議で報告をしていませんが、彼らの冷静さはそれを損ないません。 多くの場合、彼らは私たちの他の会議の講演者であるか、有名な企業の主要な地位にいます。
  • 話し手と1時間以上費やしたとしても、全員がディスカッションエリアで質問する時間があるわけではありません。


ここでは、会議の最も舞台裏の部分をつかみ、誰も書いていないことを見つけることができます。







ディスカッションエリア



各レポートの後、スピーカーは専用のディスカッションエリアに移動し、そこでチャットして質問することができます。



正式には、これはレポート間で実行できます。 講演者には義務はありませんが、通常はずっと長く滞在します。たとえば、次のレポートの期間中です。 メインプログラムからレポートをスキップして（チケットを購入した場合でもエントリが残っている）、重要な専門家との集中的なコミュニケーションに費やすことが理にかなっている場合があります。



傍観するのではなく、議論に参加して質問することが非常に重要です。 人気のあるスピーカーは、たいてい大勢の人に囲まれている場合があるため、すぐにスピーカーを「キャッチ」し、ディスカッションエリアに行き、最前列に立ち、気軽に興味深い質問をするのが理にかなっています。



コミュニケーションはジョーカーで行われるトークだけに限定されません。 他の会議やYouTubeなどで特定の著者によるスピーチを見た場合は、これらのトピックについても話し合うことができます。
