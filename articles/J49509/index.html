ネットブック上のWindows XPのインターフェースを最適化します





今、ファッションはいわゆるネットブックに行きました-ラップトップ、サイズが小さく、ハードウェアの詰め物はかなり弱いですが、安いです。 だから私は誘惑に負けて、数ヶ月前にそのようなマシンを購入しました：ASUS eee PC。 このマシンのコンパクトさの欠点の1つは、小さな画面です。 さらに、画面の対角線と物理的解像度の両方が小さく、わずか800 x 480ピクセルです。 このため、多くのプログラムウィンドウは画面に収まりません。作業を最適化するために、システムのグラフィックデザインを調整し始めました。 第一に、なぜこのようなかさばるウィンドウタイトルが必要なのか、第二に、なぜツールバーにそんなに多くの空きスペースを残すのか？ 理由もなく、私は数えて、Windows XP 2の標準の青いデザインの代替品を探し始めました。



1.テーマを置き換えるための準備

Windowsで「左」のテーマを配置できないのは、そのためだけです。このため、OSがピックアップを停止するよう説得する必要があります。 パッチを当てる必要があるのは、1つの小さなシステムライブラリuxtheme.dllだけです。



Windows XP SP3の場合、パッチプログラムは、たとえば次のリンクでダウンロードできます： 1、2 。 またはGoogleを使用します 。 Windows XP SP2の場合： 1 、 Google 。 または、特別なバージョンについては、 こちらをご覧ください 。



2.テーマの置換

これで、任意のテーマを設定できます。 新しいテーマをインストールするには、目的のテーマを含むアーカイブからC：\ WINDOWS \ Resources \ Themesから* .msstylesという名前のフォルダーとファイルをコピーする必要があります。 次に、[デザイン]タブの画面のプロパティで、リストからこのテーマを選択します。



Windows XPのインターフェイスを改善するために、薄いNiteLite XPテーマの黒いバージョンを選択しました。 最適化結果：







画面解像度が800x480ピクセルのみの場合、各ピクセルは高価です:)ここで、アプリケーションウィンドウの高さを8ピクセル 、タスクバーのさまざまな要素の幅を56ピクセル節約しました。 もう悪くない。 次に、ウィンドウフレーム要素のサイズの違いを比較します。







現在の節約は、高さ10 ピクセル、幅23ピクセルです。



お気に入りのMozilla Firefoxブラウザーでパネルのサイズを小さくするために、 Classic Compactテーマをインストールしました。 最適化結果：





もちろん、ウィンドウが最適化されていても、そのような画面解像度に収まらないアプリケーションがあります。 このような場合、 MoveInactiveWinプログラムをインストールしました。これは、Linuxシステムと同様に、Altキーを押したままにして、タイトルをクリックするだけでなく、マウスカーソルでウィンドウをキャプチャして移動できる小さなユーティリティです。
