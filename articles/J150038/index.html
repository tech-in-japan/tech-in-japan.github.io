コンピューターの知識は、年齢と90年代の困難な状況の両方が原因で、OS DOSのp1 90-120mHzで始まり、4メガバイトものRAMがありました。 少し後に、Windowsに直面しておとぎ話のGUIの世界を見ました。 まあ、少し後でさえ、コンピューターにパスワードが表示されました。一度は両親がコンピューターが私に悪い影響を与えたと思っていたからです、私はそれをプレイしませんでしたが、基本的に、私は10-12文字よりも速くテキストを入力することを学びました毎分。 一般に、ここの全員がWindows 95のログイン画面を覚えていると思います。





おそらく、最もおもしろいのは、「キャンセル」を押すことができ、システムが何も起こらなかったかのように機能したという事実でした。 確かに、この問題はすぐに修正されました。パスワードはBIOSの段階で表示されたので、簡単に移動することはできませんでしたが、当時のリセット用のバッテリーについてはまだ知りませんでした-自宅のコンピューターやインターネットさえも事実上文献がありませんでした



少し後に新しいOSが登場しましたが、基本的に98番目の「Windows」は何も変わりませんでした。ただし、ウィンドウタイトルにグラデーションが表示されたことを除きます。





Windows NTでは、一般的なネットワーク機能が登場し、そのウィンドウは次のようになりました。





最初のイノベーション-ドメインエントリのサポートが登場しました。 「キャンセル」ボタンは、コンピューターを使用するのに役立ちません。 Windows 2000でも同じことが起こりました。





一般に、すでに推測しているように、このトピックはログイン方法に影響します。

Windows XPのログイン画面は著しく異なっていました-このシステムには、ユーザーフレンドリーなインターフェイス構築の最初のヒントが含まれていました。





ユーザーアバターが表示され、すべてのユーザーが1枚のシートに表示されました。自宅で使用するためにユーザー名を入力する必要はありませんでした。 企業ユーザーの場合、ctrl + alt + delおよびシステムの「シンプルな」設計により、通常のログオンに戻る手間はありません。

数年後、Windows Vistaは多くの人からあまり愛されていませんでした。ウィンドウはXPのウィンドウとほとんど変わりませんでした。ただし、「縦」が縦ではなく一列に並んでいました。 そして、「7」の入り口は革新をもたらさず、システムの新しい色で塗り直されました。 システムへの侵入の「領域」では、停滞が観察されました。 ログイン、パスワード、ドメイン。

ただし、XP SP2の時代には、一部のラップトップでは、代替のログイン方法が登場しました。これは、さまざまな成功を収めた指紋スキャナーです。 おそらく、これは私が出会ったコンピューターでの最初のバイオメトリック認証方法であり、彼との会議はASUS Nシリーズの「領土」で行われ、そのデジタルインデックスは長年にわたって記憶の裏庭で失われました。 現在、この要素はほとんど見当たりません。明らかに、流行は過ぎ去りました。 しかし、3、4年前、すべての自尊心のあるメーカーには、同様の「洗練」のモデルがありました。

現在、認証システムは、Webカメラを使用した顔認識を通じて対応し始めています。 ASUSにはそのようなシステムがあります-Smart LogOnが呼び出されます。 誰でも公式サイトからダウンロードすることができ 、互換性のあるハードウェアで動作し、夜間にモニターの光の中で顔を認識し、通常はきちんと動作します。





おそらく、このようなプログラムの主な欠点は、だますのは比較的簡単だということです。多かれ少なかれ新鮮な写真をスリップしてログインするだけです。

ただし、写真の表情がトレーニング中にカメラに見せた表情と一致する場合、これは機能します。 トリックは、Facebookの写真（写真を撮って印刷するのが最も簡単な場所）または家族の写真アルバム（弟がラップトップにアクセスするのを恐れている場合）にない表現でプログラムを「トレーニング」することです。両親と一緒に訪問している間）。

たとえば、そのような顔を作る場合：





... Smart LogOnは、ソーシャルネットワークから印刷された通常の写真と混同することはありません。 そして、ラップトップを開きたい人は、システムに入るための別の方法を探す必要があります。 この方法のもう1つの利点は、顔の体操と組み合わせることができることです。 最終的には、1日に少なくとも5回「ログイン」します。つまり、私たちはこれらのエクササイズを無意識に実行する必要があり、通常の生活では「中断」します。 ちなみに、「顔を若くしてデータを保持する」-スタートアップにとっては素晴らしいアイデアだと思います！ :)



もちろん、指紋スキャナーにはさらに多くのトリックが必要でした-それほど前のことではない素晴らしい記事がありましたが、私が覚えている限りでは、「単純な」スキャナーはテープ、水袋、その他のさまざまな「民俗」方法を使用してコストがかかりました。 誰もがラップトップで複雑で高価なモデルを構築することはありません。 そして、指紋自体が人のかなり良い識別子である（そして最も重要なこと-一意である）と考えるなら、ここの問題は明らかに指ではなくスキャナーにあります。



生体認証市場には他に何がありますか？ 網膜認証？ これまでのところ-高価で非常にかさばる。 DNA認証？ さらに高価で、遅く、扱いにくく、信頼性が低く、DNA自体を取得するのはそれほど難しくありません。肉/皮膚/爪または髪の毛で十分です。正しいサンプルをスキャナーに「入れる」ことで問題が発生します。 しかし、虹彩のスキャナーは非常に有望な保護方法ですが、ラップトップでのみ、明らかにそれを待つ価値はありません。 はい、もし誰かがあなたのデータを盗もうとするなら、彼らはラップトップを分解します（神は私を許して）他の誰かの目を拾います。



Webカメラからのログインは改善できますが、そのためには、Kinectサイズのセンサーを使用して、写真が目の前にいるのか、それとも生きている人なのかを把握するか、顔のアニメーションで記録する必要があります。 つまり、写真を表示するだけでは十分ではありません。また、ソフトウェアが感情を記録されたサンプルと比較できるように、笑顔にしたり、ウインクしたり、言語を表示したりする必要があります。 それまでは、そのようなシステムは、人が忙しいときや機密データを保存する必要がないときにシステムへの侵入を加速できますが、ハッキングからシステムを確実に保護することはできません。
