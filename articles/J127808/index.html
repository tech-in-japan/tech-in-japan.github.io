jPCTの開始-Java用の無料の3Dエンジン

jPCTはJava用の3Dエンジンであり、OpenGLおよびソフトウェアレンダリングを使用したり、デスクトップ、インターネット、Android向けの3Dアプリケーションを開発したりできます。 少しゲームを書いて、実際に試してみましょう。





ここで jPCTを入手してください 。 アーカイブを自分の都合の良い場所に解凍します。 その中には、jarファイル、ドキュメント、および必要なネイティブライブラリがあります。 次に、Eclipseを起動して、新しいプロジェクトの作成を選択します。 プロジェクトの名前を入力して、「次へ」をクリックします。 [ライブラリ]タブで、[外部jarの追加]、[jpct.jar、lwjgl.jar、lwjgl_util.jarの追加]の順に選択します。 jpctにはjavadocの場所を指定し、lwjglにはネイティブライブラリの場所を指定します。





マウスがチーズを探して迷路を走るという小さなゲームを作ってみましょう。 おそらく3d jPCTエンジンはObject3D、Camera、Worldのクラスを提供します。 jPCTは3Dモデルのいくつかの形式をサポートしていますが、興味がある場合はドキュメントを参照してください。 3dsモデルは、Loader load3DSクラスの静的メソッドを使用してロードされ、シーン内の3dsオブジェクトの配列が返されます。 3dオブジェクトをロードする前に、テクスチャをロードすることが望ましいです。テクスチャ名が3dsファイルから受け取った情報に対応していれば、テクスチャリングは自動的に行われます。 テクスチャは次のようにロードされます。



final TextureManager texMan = TextureManager.getInstance();

texMan.addTexture( "TextureName.png", new Texture ( "TextureFileName.png" ) );







オブジェクトを作成したら、WorldクラスのaddObjectメソッドを使用してオブジェクトをワールドに追加する必要があります。 プログラムには複数のWorldオブジェクトを含めることができますが、同時に各オブジェクトはそのうちの1つだけに属することができます。 オブジェクトを表示する前に、法線や境界ボックスなどを計算しながらオブジェクトを「構築」する必要があります。これは、WorldクラスのbuildAllObjects（）メソッドを呼び出すことで実行できます。 彼らが言うように、今、世界を創造しました、光があってみましょう！ おそらく既にaddLightメソッドで推測したように、その座標と赤、緑、青の強度を示しながら、光を追加します。 jPCTのすべての座標は、主にSimpleVector（クラスjPCT）の形式で設定されますが、3つの浮動小数点座標を持つ「古い」形式は保持されています。

これをすべて見る方法は、カメラによって決まります。 カメラを移動、回転、オブジェクトに向け、オブジェクトのz軸に平行に向けることができます（整列）。 ところで、jPCTでは、x軸は右に向けられ、z軸は観測者からのものであり、y軸は下にあります（脚の下）。



前述したように、jPCTはソフトウェアレンダリングとOpenGLをサポートしています。 これがフレームバッファの作成方法です：



final FrameBuffer buffer = new FrameBuffer(800, 600, FrameBuffer.SAMPLINGMODE_NORMAL);

buffer.disableRenderer(IRenderer.RENDERER_SOFTWARE);

buffer.enableRenderer(IRenderer.RENDERER_OPENGL);







その後、必要ない場合は、以前の状態に戻します。



buffer.disableRenderer(IRenderer.RENDERER_OPENGL);

buffer.dispose();







バッファにワールドを描画するのは非常に簡単です。



buffer.clear(java.awt.Color.BLUE);

theWorld.renderScene(buffer);

theWorld.draw(buffer);

buffer.update();

buffer.displayGLOnly();







ご想像のとおり、周囲はすべて青になります。



実際のレンダリングに加えて、jpctにはユーザー入力を処理するためのクラスがあります。 キーボード入力の場合、これはKeyMapperクラスです。 新しいKeyMapperオブジェクトを作成するだけで、poll（）を呼び出すことでいつでもキーの状態を取得できます。



衝突検出器もあります。 このゲームでは、マウスが壁にぶつかり、最後にマウスとチーズの衝突によりゲームが終了します。 したがって、マウスオブジェクトについては、setCollisionModeメソッド（Object3D.COLLISION_CHECK_SELF）を呼び出します。 チーズと壁のsetCollisionMode（Object3D.COLLISION_CHECK_OTHERS）。 チーズはまだ衝突を通知するため、ゲームが終了したことがわかります（addCollisionListener（this））。 これで、マウスが移動するたびに、Object3DクラスcheckForCollisionEllipsoidのメソッドを呼び出す必要があります（SimpleVector-移動、SimpleVector-マウスをほぼ記述する楕円体、8-再帰の深さ）。 楕円体に加えて、球体と立方体を使用できます-それぞれ、メソッドcheckForCollisionSpherical（...）、checkForCollision（...）。 これが物理学のjPCTのすべてです。さらに何か必要な場合は、たとえばjBulletを見ることができます。
