ドイツでVPSを選択し、Archlinuxをインストールする

タスクとサイトの選択



目的：安定したドイツのサイトで、安価なトラフィックと月額最大20ユーロの仮想サーバーを選択し、archlinuxをインストールします。







Habréでの検索では、 いくつかの 有用な 記事と、開始可能な別のリソースが提供されました。

その結果、選択は3つの申請者に落ちました：server4you.net、hetzner.de、hosteurope.de。





料金プラン



選択したサイトの料金プランへの直接リンク：

  • server4you
  • hetzner.de
  • hosteurope.de


Hetzner VQ 12-価格は月額12.90ユーロで、19％のVAT（他の国の居住者に返還されます）、インストールは19.90（マイナス19％）です。 テスト期間はありません。 支払いは毎月です。

1000 MBのRAM、40 GBのスペース、1つのコア。

トラフィックは、100 Mb / sの速度で2 TBに制限されていないと言うことができます。

VNCインストール。 コントロールパネルから再起動します。



Server4you PLUS X4-価格は月額18.85ユーロで、インストール料はかかりません。 1、12、24か月分の支払い。 （私は示された6ヶ月を無料で理解していませんでした。）

3072 MBのRAMが保証され、最大6144 MB。 4000 MHzのCPUパワー。 50 GBのディスクスペース。

6 GBのトラフィックが含まれ、1 GBあたり0.19を超えます（1 TBは190ユーロの費用がかかります。攻撃の場合は少し費用がかかります。恐らく私は妄想を抱いており、サーバーは単にネットワークから切断されています）。



Hosteurope L 4.0-12,99ユーロ。 0を設定します。

1 GB RAM、50 GBディスク、パフォーマンス-1つ星。

無制限のトラフィック、2 IP。 ドイツ語のウェブサイト バージョンなし。



それらの質問。 サポートする



日曜日はそれらに書いた。 サポート（server4youとhetzner、当時Hosteuropeはまだ候補ではありませんでした）。



Server4youからの応答は2時間で届きました。

ヘッツナーは20時間後に月曜日の午後に応答しました。



答え



archlinuxのインストールに関する質問に答えたのはhetznerだけです。 Server4youはそれを無視しました。



サーバーがインストールされた状態で別の関税に切り替える可能性についての質問は、必要なものを注文して現在のものをキャンセルすることを申し出た見返りに、両社によって拒否されました。 Server4youは、「長期契約でサーバーを使用する場合は、サーバーの最後まで固執する必要があります」（契約が終了するまで待ちます）。



別のパラグラフであるServer4youは、より低い関税のためのVPNの禁止を示しています。私はこれについても尋ねました。

答えは：

「基本的には、ProおよびPlus vserverのいずれにもvpnサーバーをセットアップできないことを意味します。 ただし、そのオプションはPremiumおよびPlatinumパッケージで有効になっています。



Hetzner proシングルコア：

「仮想化技術としてkvmを使用しています。 これは通常のLinuxプロセスです

ホストシステム。 kvmマシンのターンであれば、2.66 GHzのCPU速度になります。



登録



最終決定はまだ行われていませんが、画像からシステムをインストールする機能を文書化していたhetznerがお気に入りでした。

ホストヨーロッパは、どういうわけか焦点を失い、その後参加しませんでした。



2つのホスティング事業者に登録されています。 ヘッツナーは、銀行カード番号を入力せずに登録を完了することを許可しませんでした。



5分後、彼らは身元を確認するためにクレジットカードとパスポートの写真を送信するリクエストを送信しました。



写真を送信してから5分後に、システムにアクセスするためのユーザー名とパスワードを送信しました。 カバーレターは、サーバーがインストールされるとすぐに通知を送信することを示していました。



そこにRobotと入力すると、サーバーセクションで緑色のボックスとサーバーIPが既にオンになっています。 サーバーの準備ができました。







5時間後、Server4youはサーバーが試用期間の準備ができたという通知を送信しました。



プリセット機能



ブラウザを介して、コンソールに移動し、サーバーを再起動した後、メニューがロードされます。







ディストリビューションはプレインストールの準備ができています





シークレットメニュー





ISOイメージから配布キットをインストールするという主な目標を達成するために、「レスキューメニュー」を起動します。







ホストはinstallimageユーティリティの使用を提案します。

このユーティリティは、インストール済みのtar.gzシステムなどのパッケージ化されたイメージのみを受け入れますが、isoがあります。



完成した画像の内側





既製の画像





ISOイメージから適切なインストール方法を見つける



ダウンロードしたisoイメージは、次の方法でディスクにコピーされます。

dd if = arch.iso of = / dev / sda



コンソールのレイアウトは珍しいことに注意してください。「y」は「z」ボタン、「/」はマイナスボタンなどです。



cfdiskユーティリティを使用して必要なパーティションを作成します。/dev/sda1は削除せず、/ dev / sda2にインストール/起動します。 この方法の欠点は、インストールイメージが占有するディスク上の177メガバイトの損失です。



再起動後、システムインストーラーが読み込まれます。 マニュアルに従ってすべてをします 。
