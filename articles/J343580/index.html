RusGIS-分析とジオデータを使用した複雑なタスクのためのプラットフォーム

今日人気のある膨大な数のシステムは、すべてを見るGoogleから面白いTinderまで、地理データを使用しています。 有名なマッププロバイダーのAPIを使用して、いくつかの基本的なジオファンクションを実装できます。 また、サービスに特別なジオデータが必要な場合はどうなりますか？ 異なるレイヤーの空間情報の束を組み合わせてカスタマイズしますか？ これを分析にリンクしますか？ それとも、人々にあなた自身の地理データを提供したいですか？ 新しいRusGISプラットフォームは、複雑さが増す同様のシナリオを指向しています。









RusGISは、空間データやその他の属性データを収集、保存、処理、分析、視覚化します。 また、分析レポートを使用して基本マップに作成します。



RusGISの主な機能：



  • Webインターフェイスを介したデータモデル（ドメイン記述）の作成。

  • オーバーレイ、サブトラクション、コピー、フィルタリングなどを使用して、外部ファイルまたは既存のレイヤーに基づいて新しいレイヤーを作成します。

  • トポロジ（隣接、境界線の入力、交差点）を考慮して、マップ上の空間データを含むWebインターフェイスを介してデータを編集します。

  • ドラッグアンドドロップをサポートするWebインターフェイスを介したさまざまな形式の空間データのインポート/エクスポート。

  • 外部ジオコーダー（Yandex / OSM）または独自のジオコーダーに基づくバッチフォワード/リバースジオコーディング;

  • データセットおよびREST / SOAPサービスのレベルでの柔軟な統合。

  • 柔軟なアクセス制御。

  • データモデルを管理する機能を含む、サードパーティのソリューションに埋め込むためのAPI。

  • 空間分析ツール（アクセシビリティゾーン、ヒートマップなど）;

  • 分析レポートの組み込みデザイナー。

  • 分離データ編集用のモバイルアプリケーション。



ソリューションのアーキテクチャ



RusGISは、3レベルのアーキテクチャで実装されています。



  • 情報ストレージレベル（DBMSサーバーとファイルストレージ）、

  • アプリケーションサーバー層（アプリケーションビジネスロジック）

  • クライアントアプリケーションのレベル（Webを介したシステムとのユーザーインタラクション）。





Openourceソフトウェアが主なものとして使用されます。



  • データストレージ用のPostgreSQLおよびPostGIS。

  • スタイルに基づいて空間データを視覚化するジオサーバー。

  • Tesseractグラフィック情報認識ソフトウェアと、レポートや電子文書を生成するツールであるJasper Reportsは、Java Enterprise Edition WildFly仕様を満たすサーバーで実行されます。

  • Apache Webサーバー。

  • Apache Directory Server LDAPサーバー。

  • LinuxオペレーティングシステムRedHat 6、Cent OS 6.8。



カスタムツール



それはすべて、ベース基板の選択から始まります。 衛星、OSM、Rosreestr、Yandex、2GIS、Rumap、衛星画像のモザイクなど、独自のベースを使用するか、外部カードを接続できます。









レイヤーの操作は、Photoshopと同様に柔軟に構築されます。 レイヤーは凡例と一緒に自由にオン/オフされ、任意のグループに結合され、その順序と透明度は調整可能です。









マップ上の任意の測定値を使用できます。破線に沿った距離、自由に構築されたポリゴンに沿ったエリア、円のパラメーター、ポイントの座標。 属性または空間検索、およびジオコーディングがあります-リストまたは座標からテリトリーの名前を介して目的のサイトにすばやく移行します。









オブジェクトのエッジまたはノードに隣接する、複雑なジオメトリを持つオブジェクトの作成をサポートします。









RusGisは、分析処理の結果（ヒートマップ、図、テーマの色付け）に基づいてレイヤーを作成できます。 ソースからのデータとともに表示されます。

















データおよび機能へのアクセスは、公開するか、役割ごとに柔軟に構成できます。 アプリケーション、サービス、およびデータモデルの外観は、管理者インターフェイスを介してカスタマイズされます。



使用例



未計上の税項目を検索する



連邦税務サービスのデータベースには、誤って説明されている不動産があります。 これらのベースに実際に存在するオブジェクトの中には、まったく存在しないものがあります。 そして、RusGISはこれらのギャップと不正確さを埋めるのに役立ちます。



システムは、地方自治政府である連邦税務局のRosreestrのデータベースを比較し、矛盾を特定し、タブレット、テキスト、およびさまざまな違反の兆候のあるACSとACSが異なる色で強調表示されているマップを作成します。



さらに、UAVまたは航空機から取得したオルソフォトマップもここに含まれます。 それらをRosreestrおよび地方行政のデータと組み合わせることで、空いている土地区画、悪用の兆候がある区画、実際の国境と文書の間に矛盾がある区画をさらに特定することができます。









道路修理モニタリング



1つのレイヤーを使用して、既存の道路修復計画を入力し、別のデータレイヤーを使用して、計画された活動の実行ステータスを入力します。 実行ステータスが変更されると、マップ上の道路セクションの表示が自動的に変更されます。









レールの安全性の向上



鉄道の負傷者を減らすには、危険な場所を特定し、統計に基づいて事故の考えられる原因を分析する必要があります。 通常、分析のソースデータは、膨大な数のテキストベースのインシデントレポートです。 それらを手動で分析するのは長くて苦痛です。



しかし、解析の助けを借りてこれらのテキストからデータを抽出し、凡例を介してそれらをマップに適用できます。 プロジェクトでは、さまざまなベースカードが基板として使用されます。 衛星画像に基づいてインシデントのマップを重ね合わせ、地上の原因をすぐに確認できます。 たとえば、都市部の鉄道を横断する装備された交差点の欠如。 さまざまな場所でのインシデントの数をすばやく推定するために、イベントの強度の分析に基づいて構築されたヒートマップが使用されます。









大きなデータ配列の認識を向上させるために、規則のクラスタリングが使用されます。









このプロジェクトは、問題のある領域を特定するだけでなく、新しいインシデントへの迅速な対応にも役立ちます。 これを行うために、検察官のオフィスと輸送の総務省の線形部門​​の層がマップに追加されました。 新しいインシデントに関する情報が表示されると、システムは最寄りの輸送検察官、総務省の部門を決定します-残っているのは連絡先の詳細を取得することだけです。 このようなソリューションは、すでにヴォルガ連邦地区で使用されています。



通信ネットワークのインシデント監視



ネットワークマップには、さまざまなタイプの回線と通信ノードが表示されます。 インシデントに関する統計、倉庫内のスペアパーツの可用性と移動のビューがあります。









老朽化した住宅からの移転管理



RusGISはカード図を作成し、1つのプロセスのフレームワーク内でそれらの異なる特性を比較できます。 テーマ別の色分け、ヒストグラム、重量、円グラフを使用して、個々の集落での第三国定住の進捗を評価できます。 マップは、プログラムが実行される集落があり、プログラムをまだ開始していない集落があることを示しています。 さまざまな中間段階があります。















API



RusGISプラットフォームには、外部システムと統合するためのAPIがあります。 主な機能：データの視覚化、モデルの変更、データへのアクセスの制限などを実装します。 もちろん、ドキュメントがあります。 リクエストに応じて提供するAPI。



今後の計画



近い将来、私たちが行うことは次のとおりです。



  • データ分析ツールを改善する

  • ジオデータおよびジオサービスのパートナーサプライヤーの数を増やし、

  • UAV、リモートセンシング、GIS、IoTなどのテクノロジーのインターフェースで新しい製品とサービスを提供する

  • ネットワークトポロジを維持するためのツールを開発します。

  • プラットフォームユーザー向けの便利なデータとアプリケーションのショーケースを作成します。



RusGISはB2BおよびB2Gセグメントに重点を置いているため、Yandex、Googleマップ、およびその他の有名なサービスとは異なり、無料版はありません。 しかし、RusGISの機能は異なり、異なるセグメントのソリューションです。 プラットフォームはRostelecomクラウドで利用可能です。 ライセンスを購入することにより、独自のサーバーにインストールすることもできます。



ジオサービスを開発してジオデータを提供している場合、または彼らの助けを借りてソフトウェアを改善することに興味がある場合は、RusGISが興味深いかもしれません。 システム、デモへのアクセス、および協力に関する質問については、 geodata製品オフィスgeo@rt.ruに連絡してください 。 また、コメントの質問にお答えします。
