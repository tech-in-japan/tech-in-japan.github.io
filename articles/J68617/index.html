XenServer 5.5上のソフトウェアRAID1（ミラーリング）

「偽の」RAID'omを備えたサーバーにCitrix XenServer 5.5をインストールする必要がありました。



ちょうど古典的な状況：BIOSのIntel Matrix Storage Managerは通常RAID1アレイを作成しますが、Linux（XenServerが構築されている）は引き続き2つの独立したSCSIディスクとして認識します：/ dev / sda + / dev / sdb ... ！



おそらく誰かが、ソフトウェアRAID1用のXenServer 5.0の無料構成で説明されているレシピがXenServer 5.5 でも 機能するという事実を理解するでしょう（コメントのIljaによる訂正を見逃さないでください）。



すべての構成は、ssh root @ xenserverまたは物理コンソールで実行できます。



アイデアは簡単です：XenServerを最初のディスクにインストールした後、劣化モードで2番目のディスクにraid1-arrayを作成し、そこに最初のディスクからコンテンツをコピーし、このアレイ（実際には2番目のディスク）から起動し、最初のディスクをそれに接続しますアレイを駆動し、通常の動作モードにします。 これについての詳細は、アイデアの著者のページで読むことができます（ありがとう、Dmitry Komarov！）。



これは「実際のRAIDコントローラ」ではありませんが、目標は達成されています。ディスクの1つが死んだ場合でも、サーバーは動作し続けます（安全に交換する機会を与えてくれます）。
