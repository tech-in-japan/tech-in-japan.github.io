自動販売目標到達プロセス：技術的な実装

今日は、自動ファネルの技術的な実装の問題を提起したいと思います。 そして、いくつかのサービスのバンドルの例にもっと注意を払ってください。 たとえば、CRMとファネル。 この記事では、実践されているいくつかのソリューションを検討することを提案します。







自動販売ファネルとは何ですか？



背景情報でテキストを過負荷にしないために、括弧の外に残すことを提案します。 この用語に慣れていないため、最初からトピックの学習を開始したい場合は、たとえばこの投稿から始めることをお勧めします。 彼らは最近、自動販売ファネルが新しい支店で販売を開始する問題をどのように解決できるかについてのコンセプトも発表しました。 人生における自動漏斗の使用を示しています。







CRM統合



サービスを相互に統合することは、自動マーケティングファネルを開発する際の典型的なタスクです。 場合によっては、プログラミングなしで行うことができます。zapier.comのようなサービスがあります。このサービスは、デフォルトでは互いに統合できない複数のツールをリンクできます。







たとえば、Facebook Lead Adsで連絡先を収集し、すぐにMailchimp、Unisender、またはGetResponseメーリングサービスに転送します。 もう1つのオプションは、Googleフォームを使用して調査を整理し、データを保存し、メールボックスをサブスクリプションサービスに送信することです。









サービスを使用したデータの収集と交換の編成は、人生における新しいアイデアの実装速度を大幅に向上させることができます。 そして、文字通りの意味で、毎日改善してください。







マーケティングの分野の複数の専門家とプロジェクトマネージャーのチームがプロジェクトに取り組むとき、誰もが知っています。 間違いや明らかな改善が発見されたらすぐに、誰でもすぐに修正できます。 標準チェーンがなければ、「プログラマにタスクを設定する->優先順位を割り当てる->アプリケーションを待つ」。 これにより、新しいタスクに集中できる開発者と、自動漏斗のパフォーマンスに即座に影響を与えるチームの両方が満足します。







同様の解決策は、「ここでそれを保存-そこに保存」という直接的なロジックで単純な束を作成する必要がある場合に機能します。 開発者と一緒にロジックを実装する例を見てみましょう。







Bitrix24 +ユニセンダー



クレーターの1つでは、次のロジックが代わりに配置されました。 サブスクライバーが関心のある資料を受け取った後、最初のパーソナライズされたオファーを送信します。 7日間有効である必要があり、その後アクセスできなくなります。







使用されるCRM：Bitrix24。

メールサービス：Unisender。









じょうごの最初の要素







そのような問題をどのように解決し、何を想定する必要がありますか？







  1. 特別なリンクをクリックすると、Bitrix24でリードを自動的に作成する必要があります。
  2. 同時に、存在しない1,000個の電子メールアドレスを置き換えることによって、顧客ベースが単に「スパム」されないように保護することが重要です。
  3. 移行日を修正する必要があります。 オファーが無効になると、彼女から7日かかります。
  4. リンクは一意で、1週間で「燃え尽きる」必要があります。 この場合、リンクをクリックすると、スクリプトは対応するリードの存在を確認する必要があります。 ある場合は日付を確認し、ない場合は新しいリードを作成します。




スターターオファーのスクリプトのロジック







マナオの開発パートナーとともに問題を解決しました。 各アイテムごとに：







1. Bitrix24でLidaを作成します。 メカニズムは簡単です-Unisenderによって送信されたレターのリンクに、2つのパラメーターを挿入します：{{Email}}-受信者の電子メールアドレスと{{SendDate}}-メッセージが送信された日付。 使用できるオプションの完全なリストはこちらです。







これらのパラメーターを使用して、リンクをクリックすると、スクリプトが置かれている文のあるページに移動します。 そのタスクは、APIを介してBitrix24でリードを作成することです。







通知の送信日を送信します。 したがって、関心のある人がすぐにリンクをたどらない場合、たとえば2日後にはすべてが正しく保存されます。 そして、手紙に書かれていたように、彼にはあと5日しか残っていません。









この方法で作成されたリードは、特定の名前とステータスで作成されます。 フィルタリングを簡単にするには







2.スパムから身を守ります。 リンクは次のようになります： http : //site.ru/offer1/?email=ya@ya.ru&date= 08 / 01 / 2017＆ type = hello







明らかに、攻撃者が対応するGET変数でさまざまな電子メールアドレスに対処するリスクがあります。 そして、CRMには無意味な情報がたくさんあります。







保護にはいくつかのオプションがあります。 Unisender APIカウンターチェックを選択しました。 ユーザーが提供されたリンクをクリックすると、Unisenderに返品要求を送信し、私たちの手紙を受け取りたいという意思を確認したユーザーの中に本当にユーザーがいるかどうかを確認します。







3.オファーの日付とタイプを修正します。 提案のタイプと作成日は、Lidaプロパティに記録されます。









各リードは、オファーのタイプと期間のプロパティのペアを自動的に記録します







Bitrix24に送信されるすべての情報は、ロボットだけでなく、生きている人でも使用できます。 多くのバリエーションがあります。







4.リンクは一意で、1週間で「書き込み」される必要があります。







新しいスクリプトを作成する前に、スクリプトは対応するメールでBitrix24 Lidaを検索し、オファーの種類と作成日を確認します。 リードが既に作成されており、7日以上が経過している場合、「申し訳ありませんが、遅れています」というメッセージが表示されます。 そして、オファーは無効になりました。







オファーの有効期限が切れていない場合、スクリプトは特別なオファーのあるページを表示します。







APIサポートを備えたほとんどすべてのCRMに対して同様のソリューションを作成できます。 そして、メーリングリストサービス用。







同様に、目標到達プロセスの「下」では、CRMの変更から逆に進むことができます。 たとえば、ビジネスプロセスの助けを借りて、リードまたはトランザクションのステータスが変更されたときに、ユーザーを別の電子メールセグメントに転送します。 または、適切な会話を実行します。 そして、最初の取引が完了したら-自動販売ファネルの別のブランチに転送します。







Bitrix24 + Unisender + Tilda.cc



この記事の一部と考えられるソリューションの別の例は、ランディングページの編集です。 上記の理由により、デザイナーで特別オファーのランディングページを作成すると便利です。 それは速く、安く、シンプルで美しいです。 ティルダが大好きです。







上記のロジックに従って、提案の関連性を考慮して、Tildaにページを表示すると、非常に素晴らしいでしょう。 そして、ここでいくつかの困難があります。







実際のページアドレスをoffer1.site.ruのようにし、Tilda.ccサーバーに配置します。 ただし、ユーザーがそれについて知らないようにする必要があります。 特別オファーは常に個人的なものであり、個人的なリンクがあり、有効期間が限られていることを思い出させてください。 ファンネルや個人リンクのない人がいつでもページを開くことができる場合、すべての意味が失われます。







結論は、住所を非表示にし、プロポーザルが関連する場合、デザイナーからのページが読み込まれるが、住所の元の形式は保持されることを確認する必要があるということです（ http://site.ru/offer1/?email=ya@ya.ru&date=01.08.2017&type=hello ） そして、これは1つの場合にのみ可能です：ページがホスティングにあるとき。







これを行うために、別の統合を実装しました。 Tilda.ccで。 APIを使用して、ページコードをサーバーにエクスポートします。 [公開]ボタンをクリックすると、Webhookがスクリプトに送信され、ページが変更されたことが報告されます。 TildaからのデータはJSON形式です。 サービスリクエストの数は限られているため、レスポンスをキャッシュします。 プロジェクトを変更すると、Tildaはリクエスト（Webhook）をサーバーに送信し、それを受信するとキャッシュファイルを削除します。 ページを開くと、回答が再キャッシュされ、ユーザーにページが表示されます。







このソリューションは、「ねじれ」自動漏斗のさらなる作業を大幅に促進します。 ある特別なオファーを別の特別なオファーに完全に置き換える、またはその説明を変更する場合、問題は数時間から1日に解決されます。 [公開]ボタンをクリックして、自動システムに新しいページを埋め込むだけです。







おわりに



もちろん、自動ファネルは主にマーケティングです。 そして通常、彼らがマーケティング技術について書くとき、ほとんどの場合、他のチャネルと比較して、「前」と「後」の場合、推論はROIに帰着します。 ただし、自動ファネルはロジック、アルゴリズムであり、（私が見せることができたように）多数の統合でもあります。 したがって、この記事では、少なくとも技術的な実装の最も単純な例を示したいと思いました。







もう一度注目する価値があります。レビュー済みのBitrix24とUnisenderの代わりに、ほぼすべてのCRMまたは配信サービスを使用できます。 主な条件は、APIの可用性です。







また、自動マーケティングファンネルは強力なコンテンツマーケティングです。 多くは、車のファンネルのコンテンツ戦略に依存します。 したがって、次の記事のいずれかをこの問題に充てる予定です。
