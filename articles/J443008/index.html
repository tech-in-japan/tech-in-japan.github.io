あるプログラマーの日への賛辞

こんにちは、Habr！



もう一度、金曜日のお祝いに別のプロジェクトを展開して、突然ミューズを訪れました。



これはコメディアンのナンセンスではありません！

あるプログラマーの日。

（著者のメモ）



まず、これが法律です！

チームを学ぶ 



git clone


次に、 溝に移動しないように

からのコンボ 



 composer install


そして 



 composer update


あなたが「熟練したスコーチャー」でない場合

ショーを見る 



 git branch


以下は、これらは単なる言葉ではありません：

表示されるものを見て 



 git branch


フラグ付き 



 -a


新しい場所を作成する場合 



 git checkout


伝統に従ってテストを行う

偵察を行うことを忘れないでください、

ブランチの命名について

それからあなたの運を試してください

gitブランチという名前を付けることにより

カミングアウトは起こりませんでした：

すぐにチェックアウトすることを忘れないでください

そして、失敗を止めるために

gitブランチが与えるものをもう一度見てください

夜のタスクに座った後

すべて追加： 



 git add . #


主なポイントを明確にします。 



 git commit -m "[ ]"


コミットが曲がらないように

チームをテストする 



 git pull


コードが完全なナンセンスでない場合

サーバーコマンドを待っています 



 git push
...

猛烈なクエストはすぐに終了します。

リポジトリ。 次に、 リクエストをマージします。

明確な良心をもって、コーヒーを注ぐ

コードレビューが完了した場合のみ。

そして、テストに合格したら、

Prodへようこそ。

ここはとても良い日です、

各ステップがOKの場合。



みなさんを仕事に招待してください。
