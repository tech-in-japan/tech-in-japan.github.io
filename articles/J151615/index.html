Windowsストアにアプリを公開します。 パート1-Windowsストアに参加する

Windowsストアアプリケーションストア（Windowsストア）は、史上最大の可能性を秘めた成長する聴衆にアプリケーションを見せたいすべての開発者に門戸を開きました。 Windows 8 Developer Preview、Consumer Preview、Release Previewが利用可能になったとき、これらのディストリビューションはMicrosoft Webサイトから1600万回以上ダウンロードされましたが、そのうち700万回以上はRelease Previewです。 OSの最新バージョンが700万台のコンピューターにインストールされたことを誇りに思う人もいますが、市場に新しいデバイスがない Windows 8の予備リリースだけが既に700万人をインストールしていることがわかります。 ただし、この記事の主な目的は、開発者向けのWindows 8の大きな可能性を確信する必要のない人々です。 Windowsストアでの公開を目的としてWindows 8用のアプリケーションの開発を既に開始している場合、またはそのようなアプリケーションを既に開発している場合は、この記事が役立ちます。



そのため、Windowsストアへの登録と、その後のアプリケーションの公開の基本的なルールと機能は何ですか。



最初のステップ。 登録







実際、多くの人が何ヶ月もの間このステップを踏む機会を待っていました。 まず、次の3つが必要です。

  1. Microsoft ID（Live IDとも呼ばれます）。 まだMicrosoft IDを持っていない場合は、 作成します 。 有効な電子メールを使用してログインするか、 outlook.comなどで新しい電子メールを取得できます。 Microsoft IDプロファイルで有効な携帯電話を指定することを強くお勧めします。 重要なSMSは彼に来ることができます。 また、Microsoft IDの国（このようなパラメーターがあります）は、クレジットカードが発行された国と一致する必要があります。 どのカード？ この次の項目について。
  2. 登録するアカウントに関係なく、有効な銀行カードが必要です。 一部の回線で無料で登録する権利を付与するトークンで登録すると、カードで少量がブロックされます。 これは検証するために必要です。 何らかの理由でカードを使用できない場合は、 仮想銀行カードを登録するための一連のヒントを用意しています。
  3. 最後に、有効なメールアドレスが必要です。これは、Windowsストアチームとのやり取りのメインアドレスとして選択されます。 これは、Microsoft IDが関連付けられているアドレスと同じにすることができます。


トークンといえば。 開発者の3つのカテゴリは、Windowsストアで無料で登録できます。

  1. DreamSparkプログラムを通った学生。
  2. スタートアップ-BizSparkを通じて。
  3. MSDNサブスクライバー （Visual Studio Professional、Test Professional、Premium、Ultimateの各層）。


適切なレベルのMSDNサブスクライバーである場合、[サブスクリプション特典]セクションにそのようなセクションがあります





ここで、Windowsストアの登録コード（トークン）を取得できます。 これは、6〜10個の文字と数字のシーケンスです。



登録を開始します。



https://appdev.microsoft.com/StorePortals/Account/Signup/Startにアクセスし 、上記で説明したMicrosoft IDでログインします。

ところで、Windowsストアでの認証プロセス中に（登録プロセス中だけでなく）、Microsoft IDサブシステムがセキュリティコードを要求する場合があります。 その前に、彼は郵便局またはSMS経由で送られます。 郵送先住所として、Microsoft IDプロファイルの「追加の」住所が選択されます。



このページで追加の住所と電話番号を追加できます 。 初めて認証を行わない場合、「このPCを信頼します」オプションを使用すると、かなり長い間コードを入力する必要がなくなります（Cookieを有効にする必要があります）。



アカウントタイプの選択



認証に成功したら、登録国とアカウントの種類を選択する必要があります。



すでに述べたように、国は実際の居住地と銀行カードの発行国に基づいて選択する必要があります。 法人にとっては、検証手続きを行うために存在を確認する必要があるため、これはさらに重要です。

アカウントの種類により、すべてが簡単になります。 個人-個人（個人アカウント）、会社-法人（組織アカウント）。

マイクロソフトは、組織に代わってアカウントを作成する権限があることを確認する必要があるため、組織アカウントにはもう少し高度なテストが必要です。 これは、デスクトップアプリケーションをストアに送信できる唯一のタイプのアカウントです。 さらに、組織アカウントは、いくつかの追加機能を使用するアプリケーションを送信できます。

  • enterpriseAuthentication 。 Windows資格情報を使用して、企業イントラネットにアクセスします。
  • sharedUserCertificates 。 ソフトウェアおよびハードウェア証明書またはスマートカードを使用して、アプリケーション内のユーザーを識別できます。
  • documentsLibrary 。 ファイルを追加、変更、削除する機能を含む、ユーザードキュメントのライブラリへのアクセスを許可します。


これらの機能の詳細については、MSDNのCapabilityをご覧ください。

次のステップに進みます。



アカウント情報の入力



これは、Windowsストアへの登録で最も重要な手順です。

連絡先の詳細では、明らかな名、姓に加えて、[メール]フィールドが最も重要です。





このアドレスは、Windowsストアチームとのすべてのやり取りでメインアドレスとして使用されます。 迷惑メールを含め、メールを注意深く定期的に確認してください。

すべての連絡先の詳細は、アカウントに関連する問題について連絡するためにのみ使用されます。 たとえば、最初に受け取るメッセージは、登録完了の確認です。 その後、支払いを送金したとき、またはアカウントを修正する必要があるときにメッセージが送信されます。

登録フォームに入力されたデータのほとんどは、追加の説明を必要としませんが、一部のフィールドの内容は明確ではない場合があります。

  • ウェブサイト
  • 出版社の表示名


開発者アカウントにリンクするWebサイトを指定します。 アプリケーションの説明ページで同じデータを入力するまで、クライアントはこのフィールドに入力したデータを見ることができません。

発行者の表示名は、アプリケーションがストアに配置される名前です。 顧客がアプリケーションを購入すると、この名前が表示されます。 発行者の名前は一意である必要があります。アカウントを確認する場合、確認が完了するまでこの名前を変更することはできません。

別のMicrosoft Developer Servicesアカウントで使用した発行元名をバックアップできない場合は、WindowsストアのMicrosoftアカウントの選択の手順を参照してください。 登録を再度開始する必要がある場合があります。 出版社名は、ストアでの商標になるため、責任を持って選択してください。 この名前から、顧客はアプリケーションに関する最初の情報を受け取るため、あなたの個性と製品の性質を反映した名前を選択してください。 選択した発行者名が別のユーザーに属していないことを確認してください。 選択した名前が以前に別のユーザーによって商標として登録されているか、著作権として登録されている場合、アカウントは閉鎖される場合があります。

開発者アカウント情報

開発者アカウント情報には、このアカウントの開発者の連絡先情報が含まれています。 個人アカウントと企業アカウントの両方について、このアカウントに関する通知を受け取る連絡先に関する情報を入力する必要があります。

法人アカウント情報

個人アカウントとは異なり、企業アカウントには追加情報が必要です。

  • VAT ID会社に付加価値税（VAT）コードがある場合は、このフィールドに入力します。


  • Microsoft 承認者（または承認者）またはその請負業者は、 承認者情報セクションにリストされている会社の公式に連絡することにより、企業アカウントを確認します。 承認者情報の正確かつ有効な情報は、承認プロセスの遅延を解消するのに役立ちます。


企業アカウントの場合、登録はマイクロソフトの請負業者であるシマンテックとジオトラストによって実行され、登録後すぐに検証が開始されます。 迷惑メールを含め、メールを注意深く定期的にチェックして、重要な指示を見逃さないようにしてください。 会社の登録を確認する書類のコピー（協会規約、協会メモ、登録証明書）を求められる場合があります。 場合によっては、コピーの公証と通常のメールによる送信が必要になる場合があります。

次のステップに進みます。



開発者契約



Windowsストアアプリの開発者契約に同意する必要があります 。 これにより、次のステップに進みます。



登録コード（トークン）を入力してください



登録コード（トークン）がある場合は、入力します。 コードを入力した結果、登録の支払い額が減少する場合があります。 トークンは1回しか使用できないことに注意してください！

支払い用のデータを入力してください

登録コードを使用したかどうか、所有しているアカウントの種類に関係なく、このステップでは、Windowsストアでの登録の支払いのためにクレジットカード情報を入力する必要があります。 再度、 仮想銀行カードを登録するための一連のヒントを用意していることに注意してください。

入力されたデータの正確性と有効性は、少量をブロックすることで確認されます。 この保留トランザクションは、アカウントの検証と信頼性の検証を目的としてのみ実行されます。 ただし、これにより、アカウントで使用可能なクレジットの量が一時的に減少する場合があります。 しばらくしてからアカウントのこの金額を差し控えます。 上記の金額がアカウントから差し引かれる正確な時間は、特定の金融機関によって異なります。

https://appdev.microsoft.com/StorePortals/en-us/Homeで登録を完了するとすぐに、SMSから送信されたコードまたはクレジットカードから引き落とされた正確な金額を入力する必要があるリンクが表示されます。

次のステップに進みます。



確認



いわゆる注文のデータをもう一度確認し、「購入」をクリックします。 すべてがうまくいけば、「ダッシュボード」（ダッシュボード）に自分自身を見つけるでしょう。

おめでとうございます！ 登録されました！





アプリケーション名の予約



これで、最初のアプリケーションの名前を予約できます。 これは多くの理由で重要なステップです。 それらのほんの一部を次に示します。

  1. Windowsストアには多くのアプリケーションはありませんが、自分で新しい魅力的な名前を選択できます。
  2. 競合他社がこれを行うことができます。
  3. 名前の選択、登録商標、および一般に、アプリケーションをアクセス可能にするつもりである国の法律を尊重して、注意する必要があります。


[ アプリを送信]をクリックします。 開いたアプリケーション公開パネルで、最初のセクション-「 アプリ名 」を選択します。 [アプリ名]フィールドに、アプリケーションの名前を入力します。 これは、潜在的なバイヤーに表示される名前とまったく同じです。 [アプリ名の予約]をクリックします。

[保存]ボタンの上に名前を予約すると、次の碑文が表示されます。



なぜこれが必要なのですか？ アプリケーションインターフェイスをローカライズすることを決定したすべての言語でアプリケーション名を入力するには、追加の名前の予約が必要です。 これは、このセクションで実行できます。

少し先に進むと、ストア内のアプリケーションインターフェイスとその説明だけでなく、名前、さらにはタイルまでもローカライズすることが可能である（そして必要である）と言う価値があります。
