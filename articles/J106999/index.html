BlackBerry Hosting Launch History

同じブランド名のBlackBerryブランドとスマートフォンは、ロシアの若い競合他社であるiPhoneなどのようにロシアではあまり普及していませんが、欧米では、BlackBerryブランドはビジネスおよび一般ユーザーの両方にとってカルトブランドです。

今日は、ブランドの歴史と、Infoboxがどのようにロシアのお客様がBlackBerryテクノロジーを利用しやすくするための最初の一歩を踏み出したかについて少しお話します。



ちょっとした歴史



カナダの企業Research In Motion （RIM）が所有するBlackBerryブランドの歴史は、このブランドのポケットベルとスマートフォンが最初にリリースされた90年代後半に遡ります。 BlackBerryデバイス-便利で信頼性が高く、「職場での永続的な存在」の効果を生み出し、企業クライアントの間で急速に人気を博しました。 しかし、これらのデバイスの主な利点は、暗号化（AES / 3DESアルゴリズムを使用した256ビットキー）と、独自のテクノロジを使用した送信データの圧縮です。これにより、スマートフォン、メールサーバー、その他の企業情報システム間で機密情報の安全な交換を編成できます。 これらの利点により、過去10年間で、BlackBerryスマートフォンは世界中の数千の企業の企業標準となっています。 Fortune 500企業の90％がBlackBerryを使用していることが重要です。



しかし、北米と西ヨーロッパの一部の国を除く多くの国でBlackBerryスマートフォンの普及を制限したのは、暗号化方式であり、特別なサービスに限定されていました。



ただし、数年前、ロシアのモバイルオペレーターMTSおよびBeelineにBlackBerryスマートフォンが登場しました。 しかし、顧客には、Blackberryを接続する1つの方法-独自のメールシステム（Exchange、Lotus）の可用性、 Blackberry Enterprise Server （BES）のライセンスの購入、および組織内での実装-のみが提供されました。 BESライセンスの価格は、約200,000ルーブルとクライアントライセンスだけです。もちろん、同社には既製のITインフラストラクチャ、独自のメールシステム、およびこのプロジェクトを実装するための技術スペシャリストが必要です。 これにより、BlackBerryを大量に使用するにはエントリバーが高すぎます。 同時に、Exchangeホスティングとともに提供されるBlackBerryホスティングは世界中で非常に一般的です。



Exchangeホスティングサービスを提供する一方で、BlackBerryホスティングをいつか開始したかったのです。 初めてBlackberry Enterprise Serverに遭遇したとき、ある大規模な国営企業が、以前に提供したExchange Serverベースの企業メールシステムで既存のBlackberryデバイスを完全に利用する問題を解決するように依頼しました。 そして、MTSの専門家とともに、ホスティングのメールシステムと統合されたBlackBerry Enterprise Server（BES）を展開しました。 その後、RIMとのパートナーシップ契約に署名した後、組織にBlackBerryソリューションを実装しました。



さらに最近では、ロシア最大の携帯電話事業者とBlackBerryハンドセットの小売業者MTSとともに、まったく異なる、はるかに少ないコストで、ホスティングモデルに応じたBlackBerryサービスを提供することができました。 つまり、BlackBerryで使用するボックスをExchangeホスティングプラットフォームに転送し、BESホスティングサーバーと連携します。 その結果、クライアントはExchange 2010およびBlackBerry Hosting Serviceに基づくメールサービスの料金を支払います。また、MTSからの特別なデータプランに接続する必要があります。



BlackBerryホスティングの実装方法







Exchange 2010 Rollup 4 HUB / CASロールのNLBクラスターとメールボックスデータベースのDAG上の既存のホスティングインフラストラクチャに、Windows Server 2008 x64でBES 5 SP2を実行する2つの物理サーバーのフェールオーバークラスターを追加しました。 BlackBerryメールボックスとユーザーの作成は、システム管理者の関与なしにWebSitePanelを使用して管理されます。システム管理者は、 BlackBerry Resource Kitを BESにインストールします。特別なBESadminユーザーは、このバンドルが正しく機能するために、Exchangeで作成されたメールボックスにフルアクセスできます。



デバイスのアクティベーション



ホスティングプラットフォームExchangeおよび対応するBlackBerryユーザーでメールボックスを作成したら、スマートフォンをアクティブにする必要があります。 ユーザーがコーポレートアクティベーションメニューにメールボックスのアドレスとアクティベーションパスワードを入力すると、次のことが発生します。



•スマートフォンはアクティベーション情報をセルラーネットワークに送信し、セルラーネットワークはそれをRIMネットワークにブロードキャストします。

•サーバーのRIM側では、アクティベーションの責任者がアクティベーション要求を識別し、レターに添付されているETP.DATファイルにパックして、アクティベートされたメールボックスに送信します。

•BlackBerry Enterprise Serverエージェントは、ホスティングプラットフォームのExchangeメールボックスに新しいレターを見つけ、アクティベーション要求を認識して削除します。

•レターで受信したファイルのデータを使用して、BlackBerry Enterprise Serverはスマートフォンに接続し、アクティベーション要求の受信を確認します。

•次に、BESはアクティベーションパスワードが正しいことを確認し、正しく指定されている場合、デバイスおよび他の情報をスマートフォンにアクティベートするための情報の送信を開始し、同期プロセスが進行中です。



同期が完了すると、デバイスのアクティベーションが完了し、ユーザーはExchangeと連携するように構成されたスマートフォンの使用を開始できます。 顧客がすべてのメールボックスをすぐにホスティングプラットフォームに転送した場合、それは簡単です。 しかし、多くの場合、それは異なります。



BlackBerryホスティングを使用したドメインボックスの作業の一部



BlackBerry Hosting Serviceを使用する予定の多くのお客様は、通信プロバイダーまたは独自のメールサーバーによってサービスが提供される「通常の」メールボックスと、ホスティングExchangeインフラストラクチャおよびBESサーバーによってサービスが提供されるメールボックスにメールを分割する問題を解決する必要があります。 これは、たとえば、クライアント企業の少数の従業員のみがBlackBerryデバイスを使用する場合に必要であり、経済的またはその他の理由ですべての企業メールサービスをExchangeホスティングプラットフォームに転送することは実用的ではありません。







実際には、この問題を次のように解決します。 ドメインのMXレコードを、BlackBerryデバイスが使用されるメールボックスから変更しないように、メインサーバーですべてのメールをExchangeサーバーで作成された特別なメールボックスにリダイレクトします。 これらは、2つのエイリアスを持つメールボックスです。たとえば、user @ example.ruとuser@zXXX.demoexchange.ruです。ここで、zXXX.demoexchange.ruは、メールサーバーによって議論されるサービスクライアントドメインです。 したがって、このメールボックスには、MXを使用してメールボックスに転送されたメールを受信し、メインのメールボックスの代わりにメールを送信する機能があります。 ただし、SPFレコードがある場合は、送信サーバーのアドレスで補足する必要があります。 提案されたスキームが正しく機能するように、Exchangeサーバーでエイリアスuser@zXXX.demoexchange.ruをこのメールボックスのプライマリとして割り当て、example.ruドメインを「外部」にして、ドメイン内のメール配信に問題がないようにします。 BlackBerryデバイスはuser@zXXX.demoexchange.ruメールボックスでアクティブ化され、アクティブ化後、user @ example.ruはプライマリとして示されます。



BlackBerryの制限



ご存知のように、BlackBerryのメーカーが定めたすべての機能が機能するわけではなく、ロシアでは許可されていません。 最大の禁止事項の1つは、Wi-Fiブロックです。 幸いなことに、この制限は最近解除されました。 それ以外の場合、ホスティングソリューションでは、BlackBerryの最大の可能性と、Microsoft製品に基づいたビジネスクライアント向けの他のサービスのすべての機能を使用しようとしました。 制限の一部はまだ回避されていません。 たとえば、BlackBerryはユーザーの共有カレンダーにアクセスできず、別のユーザーのタスク設定は機能しません。 また、ホスティングプラットフォームでは、共有Active Directoryへのアクセスを提供できないため、現時点では、社内のActive Directoryクライアントとの同期は不可能です。 追加のサードパーティアプリケーションをインストールすると、いくつかの制限を克服できます 。



（BlackBerry所有者間の無料メッセージング用の）ロシアではまだ利用できないBlackBerry Messengerは、他の無料クライアントに置き換えられており、OCSモバイルクライアントを社内のビジネスコミュニケーションに使用できます。 Outlook、SharePointポータル、CRMシステムと統合されたOffice Communications Server 2007 R2に基づくサービスを提供し、BlackBerry所有者間および他のすべての会社従業員間のメッセンジャーとして機能します。



BlackBerryをホストした人



ホスティングモデルに基づいたBlackBerryサービスは、以前はこのソリューションを自社の施設に展開する機会がなかった中小企業が利用できるようになりました。 通常、これは、2〜3人のコミュニケーターを使用したい企業のトップマネジメント、または企業標準としてBlackBerryを使用している西洋企業の代表的なオフィスです。

したがって、組織は、企業情報へのユニバーサルアクセス、メールトラフィックの暗号化と圧縮、およびロシア全土での無制限のインターネットアクセスを備えたアクティブ化されたデバイスをBlackBerryホスティングから受け取ります。



これまでのところ、サービスの価格は欧米の同僚の価格よりもわずかに高くなっています。 しかし、限られた市場では、これは正当化されます。 顧客が増えると、価格は下がります。 いずれにせよ、当社の顧客は、ソリューションの独立した実装と比較して、ホスティングモデルを使用することに大きな違いがあると考えています。
