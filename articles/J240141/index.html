Microsoft Azureクラウドでデータを保存および処理するためのオプション

本日、Azureクラウドのストレージ機能についてお伝えできることを大変嬉しく思います。 データは最も重要なものであり、データはインフラストラクチャの作成対象であり、一般的にITが存在することを知っています。 したがって、データストレージの問題が最も重要です。









Microsoft Azure あなたとあなたのデータの保存方法



まず、Microsoftが独自のサービスのデータとユーザーのデータを保存する方法と、会社が使用し、顧客に提供するオプションを見てみましょう。

















お客様のすべてのニーズを提供することは非常に重要です。そのため、データセンターを継続的に拡張し、容量を増やし、新しいものを作成しています。 今日、私たちは世界中で16の最も近代的で非常に大規模なデータセンターを提供し、何百ものサービスと世界中の何億人もの人々が使用している何千もの顧客のサービスを提供しています。 これにより、Azureサービスは24時間年中無休でサポートされている89の市場で利用できます。

















データについて言えば、中規模企業のデータ量は毎年50〜60％増加していることを理解する必要がありますが、企業内のデータを扱う作業は、主にローカルストレージに合理的に保存できる一種の作業データセットで実行されます。 残りのデータボリュームは、クラウドに配置して処理するのに最も収益性が高く、コストを削減します。クラウドデータストレージのコストは、自社のインフラストラクチャのコストと比較して低いことに留意してください。

















このビジネスはクラウドストレージの利点を長い間認識してきたため、今日では10兆を超えるユーザーオブジェクトがAzureストレージに保存されています。 私たちのストレージは毎年成長しているだけでなく、独立した分析会社が指摘しているように、安価で高速になっています。

















データストレージの重要な問題は、セキュリティ、ストレージの信頼性です。 Azureでは、保存するデータに関係なく、データ項目は少なくとも3回保存されます。 この場合、あなた自身は絶対に安全について考える必要はありません。 データのストレージの絶対的な信頼性を確保するために、データセンター内のデータのレプリケーションとリカバリのタスクを引き受けます。

















データの信頼性について考えると、自然災害の場合など、巨大なデータセンターが無効になる場合でも、さらに進んで信頼性の保証を提供します。 さらに高い信頼性を確保するために、データの3つのコピーをもう一度コピーします。今回は、最初のデータセンターから少なくとも400マイルにある別のデータセンターにコピーします。 これにより、噴火する火山が1つのデータセンターを破壊した場合でも、別のデータセンターで自動的に作成されたレプリカから顧客がデータを利用できるようになります。

















現在、ほとんどの大規模または中規模の企業は、データを保存および処理するための独自のソリューションをすでに持っています。 この場合、容量を継続的に拡張するソリューションが必要になる場合があります。 このようなタスクには、クラウド内の未使用データを自動的にアーカイブし、最も頻繁に使用されるデータのみをローカルに保存することにより、データセンターをAzureクラウドに拡張できる独自のStorSimpleサービスを提供します。 このアプローチにより、企業は今日の機器コストの60〜80％を節約できます。

















データのバックアップコピーを保存するためのクラウドストレージの使用については、既存のソリューションをバックアッププロセスに統合できることが重要です。 SQL ServerデータをAzureクラウドストレージに自動的にバックアップする機能と、クラウドまたは企業サーバーでの簡単なリカバリを提供します。

















データベースのバックアップに加えて、ユーザーおよび企業のファイルとフォルダーをバックアップできることが非常に重要です。 企業インフラストラクチャデータのバックアッププロセスをAzureの信頼できるクラウドストレージに透過的に統合します。









Azureストレージ 大量のデータを安全に保存する



特定のAzureサービスと機能をさらに詳しく見てみましょう。 Azure Storageから始めましょう。

















Azure Storageは、さまざまな種類のデータ用の信頼性の高い単一の分散ストレージです。これにより、各ユーザーは、インフラストラクチャや投資について考える必要なく、ペタバイトのデータを保存できます。









ストレージの成長機会を提供するAzure Storageは、ユーザーの操作なしで信頼性の高いストレージを実現するために、すべてのデータを数回自動的に複製します。









便利で信頼性の高いストレージを持つだけでは、クラウドソリューションを作成するのに十分ではありません。 便利なツールと開発機能のサポートも必要です。 Azureは、すべての一般的な言語、プラットフォーム、開発ツールへの統合により、一流の開発者サポートを提供します。









最後に、データストレージについて言えば、信頼できるストレージの価格について考える必要があります。 そして、ここで、Azureクラウドは、最高の信頼性とパフォーマンスで、市場で最も低い価格を提供しています。 Azureクラウドにデータを保存する方が信頼性が高いだけでなく、安価です！

















Azure Storageは、多くの種類のデータを保存する機能を提供します。 ユーザーは、構造化データをキーと値のセットとして保存できます。









別のオプションは、ワークフローソリューションでの信頼できるメッセージングのためのメッセージキューストアです。









3番目のオプションで最も一般的なのは、テキストファイル、ドキュメント、マルチメディアなど、Azure Storageクラウドにバイナリオブジェクトを保存することです。









最後に、Azure StorageのBLOBストレージ機能の拡張機能は、標準のプロトコルとAPIを介して通常のネットワークファイルストレージとしてストレージにアクセスする機能です。これにより、クラウドについて何も知らない古いアプリケーションを簡単かつ簡単に移行できます。









Azureキャッシュ。 コードからすばやくアクセスするためのキャッシュデータ











データを操作できる次のAzureサービスはAzure Cacheです 。これにより、頻繁に使用されるデータを高速メモリにキャッシュし、再利用できます。









Azure Cacheは、データアクセス、ストレージ、取得の最高速度を提供し、高負荷向けに最適化されています。









Azure Cacheの中心となるのは、人気のあるオープンソースのRedisソリューションです。これにより、多数の既製のツールにアクセスでき、クラウド内のデータを効率的に複製できます。









Azureキャッシュの使用を開始するには非常に簡単です。ポータルでパラメーターを設定するだけで、すぐにストレージへのアクセスアドレスを取得し、アプリケーションで使用を開始できます。 すべてがとても簡単です。









Azureのキャッシュの優れた機能の1つは、ユーザーが意思決定に使用できる既存の作業ロール内でメモリをキャッシュし、効率的に使用できることです。 このソリューションにより、大幅に節約できます。









Azure SQLデータベース。 鉄や管理者なしでリレーショナルデータを保存します



アプリケーションとサービスが動作する最も重要なタイプのデータは、リレーショナルSQLデータベースです。 Azure Cloudでは、このデータを効率的に保存でき、いくつかのアプローチを提供しています。

















Azure SQL Databaseサービスは、データベースをサービスとして取得できるクラウド内の完全に管理されたデータベースサービスです。









同時に、データベースホスティングインフラストラクチャを完全に管理することで、ユーザーが予測可能なパフォーマンス、柔軟なスケーリング、ビジネス継続性、開発者向けの豊富な機会を保証します。









同時に、インフラストラクチャコストをゼロに削減し、データベースサーバーを管理するすべてのタスクを完了します。 サービスによって管理されているため、Azure SQL Databaseは機能を失わず、エンタープライズレベルのDBMSソリューションのままです。









通常のDBAツールと開発機能を必要とするすべてのアーキテクトと開発者にこのクラウドサービスをお勧めします。

















AzureクラウドのSQL Databaseサービスと共に、SQL Serverなどの独自のカスタムデータベースサーバーをホストできます。 2つのアプローチを比較すると、サーバーをホストすると柔軟性が向上し、サーバーを完全に制御できるようになりますが、SQLデータベースを使用すると管理コストをほぼゼロに削減できます。 Azureクラウドの作成では、ユーザーに任意の選択肢を提供したかったのです。 したがって、最適なソリューションを選択できます。









Azure DocumentDB。 スキーマなしで、SQLクエリを使用してNoSQLデータを保存します











最近、一般名NoSQLで、リレーショナルデータではなく構造化データを格納するための多様なソリューションが人気を集め始めています。 このタイプのデータをAzureに保存するために、効果的で迅速なソリューションであるDocumentDBサービスが提供されます。









Azure DocumentDB -JavaScriptおよびJSONデータ形式を考慮して開発および作成されたNoSQLソリューション。

















ただし、DocumentDBはNoSQLソリューションであり、サービスのユニークな機能は、SQLクエリ言語を使用してデータを選択する機能です。 DocumentDBに保存されているデータにはスキーマがまったくなく、サービスのインデックス機能も完全に自動的に機能し、データを最適化してすばやく検索できます。









DocumentDBサービスの速度とアクセスは、最適化されたアーキテクチャとSSDの使用により保証されます。









データストレージツールは、ユーザーと開発者にとって便利なはずです。そのため、DocumentDBは、データへのアクセスのためのオープンな標準と開発者向けの多くのツールを提供します。









Microsoft Azure あらゆるタスクのためのあらゆる好みのための多くのサービス











Microsoft Azureは、クラウドサイト、ソリューション、およびサービスを構築するための豊富なサービスを提供します。 Azureには、既に確認したリストにある主要で最も人気のあるサービスに加えて、データを操作するための特別なサービスがいくつかあります。









たとえば、 Azure HDInsightサービスを使用して、事前に構成された環境とHadoopエコシステム全体をサポートする既製のHadoopクラスターをオンデマンドでクラウドに取得します。 HDInsightを使用して、ビッグデータを効率的に処理します。









新しいAzure Machine Learningサービスを使用して、 機械学習を実装し、予測サービスを構築します。









Azure Searchを使用して、オンデマンドでインデックス作成機能を取得し、情報をすばやく見つけます。









Azure Backupを使用して、データをクラウドに効率的にバックアップします。









最後に、 Azure CDNを使用して、コンテンツ配信ネットワーク上のファイルを世界中のサイトに効率的に配布します。
