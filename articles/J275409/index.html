プロローグ



この記事は、 kulhackerプログラマーになることを決めた人、参加できる場所がまだわからず、労力をあまりかけないと考えている人向けです。



この記事は、経験豊富なプログラマーによる読書にはお勧めしません。 結局のところ、だれかが言うことはありますが、プログラミングに対する見解は異なるかもしれません。

したがって、マイナスにならないようお願いします。 経験豊富なプログラマーが初心者に言いたいことがあるなら、あなたの記事も書いてください。



まあ、あなたがこの職業で自己決定の過程にあるなら-あなたは大歓迎です！



専門用語



教科書によると、プログラミングはプログラミング言語（YP）で命令を記述するプロセスにすぎません。

しかし、プログラムを作成して作成するのは簡単ですか？

プログラムを書くことは可能ですか、それはコンパイラーへの命令（命令）のセットをリストするだけで、すべて準備ができていますか？

コマンドを暗記してプログラムにスタンプを押して、戦利品をloぎましょう！？



プログラミングは、技術的なプロセスではなく、純粋に研究的な創造です。



プログラミングとは、オブジェクト、機能的な依存関係、仮想イメージを抽象化し、考える能力です。



プログラミングは創造的な思考の逃避のプロセスであり、多くの場合、仕事の日や生活様式の範囲を超えています。



プログラムは単なるテキスト行の集まりではないことを理解することが重要です。

これは、本から書かれた一連のコードではありません。 これは複雑なマルチレベル製品です。 これらは、互いに通信するさまざまな動作を持つさまざまな構造の数十および数百のオブジェクトです。 そして、プログラマーは、これらのオブジェクト間のすべての微妙なつながりを理解し、感じる必要があります。



プログラマーがプログラムに取り組むとき、それは彼の頭の中に住んでいるようです。

プログラムはコードに没頭して書かれていることを理解する必要があります。 これは、現在のモジュールのソースコードが脳にロードされ、そこで処理される（デバッグされる）ように見えるときです。

したがって、すでにプログラムを実行するために開始している「正しい」プログラマーは、それがどのように実行されるかを完全に知っています。



つまり、コードを書いてその動作を確認し、実行し、どこかでエラーが発生して落ちたときに非常に驚いたプログラマーをたくさん知っています。 彼らは頭の中でコードを実行できなかったため、このコードがどのように機能するかを理解していませんでした。 まあ、彼らは対応するプログラムがありませんでした 。



プログラミングで最も重要なスキルは、心の中でコードを再現（実行）する方法を学ぶことです。

プログラマーの頭脳は、頭の中でコードのセクションを実行できるはずです。 ループをループし、変数を計算し、出力がどうなるかを大まかに想像します。



あなたがプログラムを感じているという感覚は、プログラマーになるための非常に重要なステップです。

ちなみに、優秀なプログラマーは、開発、研究、設計に90％の時間を費やし、ランダムに生成されたコードのデバッグにこれらの90％の時間を費やすという点で、悪いプログラマーとは異なります。



結論 プログラマーは、コンパイラーに渡す前に頭の中でコードを実行できなければなりません。


苦い真実



原則として、教育機関でプログラマになることを学ぶことはできません。

ほとんどのプログラマーは独学です。 教育は工学知識の基礎を築くことができるだけです。 残りは精神的能力と気質であり、勤勉さと労働力の二乗、そして階乗（！）でさえです。



コンピュータープログラミングの学位を取得したプログラマーとの仕事で、多くのインタビューを行いました。 インタビューの中で、彼らは一生のうちに1つのプログラムしか書いていないことが判明しました。 私はそのような候補者に尋ねましたが、詩人がまだ詩を書いていないのであれば、詩人は自分を詩人と呼ぶことができますか？ 通常、結局のところ、詩のコレクションが出てきて、詩人は詩人自身の認識と地位を受け取ります。 答えは愚かな沈黙と誤解でした。



なる



当社の業務は次のとおりです。 アルゴリズムについて読んだり、言語を学んだり、さまざまな小さなプログラムを書いたりします。 任意のプログラム-自分のため、友人のため、技術を扱うためなど。そして、あなたはこれに多くの時間を費やします。 もう一度明確にします-かなりの時間。 そして、あなたは成功しますか？ あなたは入り、あなたは理解し、あなたは好きで、あなたはこれから内なる感情的な隆起を得るでしょう！ プログラムが計画どおりに動作すると、幸福の波を感じ、エンドルフィンが脳に飛び散ります。 ここにある！ いいね！



その後、プログラマーのような人になります。 そして、その時だけ、 洗っていない古いセーターとあごひげ...



笑いながら笑いますが、プログラマーのひげの長さによって、プロジェクトのどの段階で彼がいるのかがわかります。

結論 プログラマーは、プログラムを書くだけでなく、彼らが働く人でもあります。


言語について。



多くのプログラミング言語がありますが、C ++は常にこのリストから離れています。

なぜC ++なのか？



C ++は、アセンブラーからマクロのコレクションとして登場した最小限の言語です。 むしろ、S。が最初に登場しましたが、それはポイントではなく 、 私は歴史家ではありません 。

なぜ普通のプログラマーがC ++を知っている必要があるのですか？

聖書を知らない司祭のようなものです。 マーケターとして、4Pルールを知らない。 化粧なしのモデルのように。 したがって、C ++は非常に必要な荷物であり、少なくとも1つの棚の頭に横たわる必要があります。



C ++を実際に使用することはできませんが、すべての基本の基礎として、キヤノンとして知っておく必要があります。 ただし、C ++を使用すると非常に危険です。たとえば、ポインタを操作するとき、プログラマが基本を完全に理解していない場合、データセクションでコードを実行するために飛ぶことができます（これは、JAVAでキャンセル）。 また、「自分で足を踏み入れる」こともできます））））C ++の作者であるStraustrup氏はそう思いました 。 C ++などのツールであるため、それを使用してマスターするためには非常に有能である必要があります。



また、Java、PHP、Perl、Rubyなど、一部の現代言語はC ++をベースとして使用したことにも留意してください。

すべての既知のプログラムは、Windows、Linux、MacOS、iOS、Firefox、Kaspersky、Chromeなど、C ++（またはC）のみで記述されています。 さらに、有名なプログラムの多くにはオープンソースがあり、それらからプログラミングを学ぶことができます。



現時点では、ネイティブネイティブプラットフォーム用の最も高速で最適化されたコードを提供するのはC ++のみです。

プログラマーの手順



あなたのプログラミングはどこから始まりますか？

1.アルゴリズム化の概要

ここで、まず第一に、パスカルはまだ良いです。 彼はブロック思考の開発が得意です。 プログラミングの主題を紹介します。 C ++のように怖がらない。 そして、脚全体が残ります。

2.変数とデータ型

3.言語の基本要素

構造、クラス

ループ、ブランチ、選択肢、遷移、関数

4. OOPに慣れる

OOP-これはまさに、 人のプログラムを「良い」プログラムにする機能です。 これは、適切なプログラムのスケルトン（スケルトン）です。

これが初心者向けの屋根です。 これが愚かなプログラマーが走っている理由です。

これが抽象化の始まりです。

理解する必要があるのは1つだけです。OOPは、適切に設計されたプログラムのバックボーンです。

OOPがないと、プログラムモジュール間の論理的で直感的な相互作用を構築するのが難しい場合があります。

5.テクノロジーに関する知識-ファイル、ストリーム、グラフィックス、ソケット、DBMS、Web、XMLを操作する

6.深化-6〜9か月間、プロジェクトに取り組みます。 ここでは、既製のコンポーネントをプルするのではなく、純粋なC ++コードを記述することが重要です。

ただ手を入れることが重要です。



フレームとコンパイラ（IDE）



フレームワークは、大規模なソフトウェアプロジェクトのさまざまなコンポーネントの開発と統合を促進するソフトウェアプラットフォームです。

コンパイラは、作成したコードを実行可能コードに変換するプログラムです。



IDEは、ビジュアルフォームの作成とコードの編集を容易にするグラフィカルエディターエディターです。

最新のプログラミングの問題は、フレームワーク、環境、およびYaPovの過度の多様性です。

すべてを学ぶことはできません。 顧客または雇用会社がロットを決定します。

Visual C ++（Microsoftの環境）、Builder C ++（Embarcadero）、およびQtで学習を開始できます。



Visual C ++は古典です。 便利なビジュアルコンポーネントが必要な場合、洗練されたインターフェイスが必要な場合は、Builder C ++を使用してください。 クロスプラットフォームが必要-Qt C ++を使用してください。



私はDOSでC ++でプログラミングを始めました。 それから（90年代）それ以上はありませんでした（unixはカウントしません）！

これで、C＃.NET、Qt、wxWidgets、C ++ Builder、C ++ VisualStudio for Windowsができました。 そして、Linuxでも同じくらい大胆なバリエーションです。

そして、あなたがJava、Androidの方向に目を向けるなら、それは可能性から唾液を流しているだけです。



ドキュメントを大量に注入する準備をしてください。 プログラマーが言うように-RTFM 。

良いプログラミング規則



1.プログラムの最初のバージョンを書きました！？ それを捨てて、2番目のバージョンの作成を開始します。

2.プログラムを紙の上で開始する-ブロック、リンク、形態、図（UML）を設計する

3.自己文書化変数-bPersonNameValidity

4.コードの60％-エラーおよび例外的な状況の処理。

5.コードを小さなモジュールに分割します。 1つのクラス、1つのモジュールを使用することをお勧めします。



何がありますか？



システム（C）と応用（P）プログラマーへの分離。

C-オペレーティングシステムのデバイス、APIの基本、OSの動作（スレッド、プロセス、割り込み、フック）を知っており、サードパーティのコンポーネントを使用せずにすべてのコードを自分で記述しようとします。 多くの場合、VisualStudioの支持者。 彼らは、大きなグラフィカルインターフェイスを備えたソフトウェアを作成することを好みません。

P-サードパーティのモジュールとコンポーネントを使用してプログラムを作成します。 OSデバイスに入らないでください。 DelphiとBuilderが大好きです。



埋め込みは最もプロフェッショナルなレベルです。 ハードウェアのプログラミング-ハードウェア。 電子機器とアセンブラの知識が必要です。 最高の有料番組セグメント。



Webプログラマー-動的サイトのコードを作成します。 PHPでより頻繁に。 さらに、HTML、CSS、JSなどを知っている必要があります。
