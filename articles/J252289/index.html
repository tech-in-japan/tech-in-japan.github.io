Kotan用のKrita Linuxビルド



最後に、フリーソフトウェアを使用する優れたアーティストであり、Krita FoundationおよびBlender Instituteコミュニティの正規メンバーであるDavid Revoitによる別の記事の翻訳。 マスターは、ソースコードからKritaグラフィックエディターの最新バージョンを構築するための手順を初心者アーティストと共有します。 彼は、このプロセスは実際には一見したほど複雑で複雑ではないと考えています。 すべての手順が美しく示されています。 参加してください！



エントリー



CompitscriptsをGitHubでリリースしてからほぼ1年で、間違いを犯したことに気付きました。 Compilscriptアプローチのアイデア自体が間違っていることが判明したため、 サポートを停止することにしました 。



Compilscriptとは何ですか？

Compilscriptプロジェクトは、ユーザーがKrita（開発者向けの最新バージョン）をビルドおよびインストールし、プログラムの「自動」メンテナンスを実行するのに役立つスクリプトのセットでした。 スクリプトはほとんど成功していません。 私はそれをしながら学んだことを今でも誇りに思っています。



何が悪かったのですか？

Compilscriptがエラーをスローして動作を停止した場合、異常な何かが発生した場合、Compilscriptは動作を停止します...そして、「異常なこと」がLinuxでよくあることなので、ユーザーは理解できないシステムに非常に依存していました。 そして、それが壊れると、ユーザーのワークフロー全体も壊れ、何も修正する方法がなくなります。 これは解決策ではありません。



新しいアプローチ

したがって、 Arch Linuxの最適な方法は自動ツールを提供するのではなく、アーティストが自分で簡単にプロセスを実行し、何が起こっているのかを理解できるドキュメントを作成することであると判断しました。 アーティストは、インストール、更新、および機能しない場合は古いバージョンのコードに戻ることができるはずです。 これはすべて、指示に従って簡単に実行できます。



すべてが示されている

また、コンパイルに関する多数の図も提供しています（ CC-Byライセンスの下で配布されています）。 これらの画像が、他のプロジェクトがよりユーザーフレンドリーになり、簡単な例えで理解しやすくなることを願っています。猫が家を建てているということです。



なんで猫？

インターネット全体がアザラシをよく知っているからです。アザラシを間違えることは不可能です。



料理ディレクトリ





最初に必要なステップ：場所の準備。 ホームディレクトリに約5GBの空き容量が必要です。 開発者が推奨する構造を使用します。



/ home / <ユーザー名> / kde4 /ソースコードのsrc

/ home / <ユーザー名> / kde4 / Kritaをビルドするためにビルド

/ home / <ユーザー名> / kde4 / inst Kritaをインストールする


注： このマニュアルでは、<username>をユーザー名として扱います（例：/ home / deevad / kde4 / src ）



これを行うには、ターミナルを開き、行をコピー（Ctrl + C）してターミナルに貼り付けます（Ctrl + Shift + V）：



mkdir -p〜/ kde4 / src〜/ kde4 / build〜/ kde4 / inst


また、ディストリビューションのパッケージマネージャーで、 calligraおよびkritaパッケージがインストールされていないことを確認してください。 名前による検索を使用し、必要に応じて削除します。



ソースコードを入手してください。





cd （ディレクトリの変更） コマンドとTabキーを使用してフォルダーに移動し、名前を自動補完します。



cd〜/ kde4 / src


ディストリビューションのパッケージマネージャーを使用してgitをインストールします。

次に、 gitにコマンドを入力してソースコードを取得し、ターミナルに〜/ kde4 / srcディレクトリにある次の行を挿入します。



git clone git：//anongit.kde.org/calligra.git




注意！ 現時点（2015年3月）、Qt5への移行に関連して、Kritaの主な開発はcalligra / 2.9ブランチで実行されるため、ソースコードを受け取った直後に、現在のブランチをコマンドで切り替える必要があります



cd〜/ kde4 / src / calligra

git checkout calligra / 2.9




ライブラリと依存関係を取得する





この部分は複雑になる可能性があります。各ディストリビューションは独自の方法でパッケージを管理するため、ライブラリをインストールします。 KritaはCalligraの一部であるため、多数のライブラリに依存しています。 1つのコマンドでインストールできるとは限りません。 一部のディストリビューション（Ubuntu、openSUSE）には、このための基本的なメタパッケージがありますが、他のディストリビューションにはありません。 必要なライブラリを自動的に取得するためのコードについては、 こちらの公式ドキュメントをお読みください。 （ 注： これはWikiページへのリンクです 。 他のユーザーにこのプロセスを容易にするために、編集、追加、変更をためらわないでください 。）



ディストリビューションがリストにない場合は、プランBに進み、パッケージマネージャーでパッケージを1つずつ探します。 それはひどいように聞こえますが、それほど長くはかからないはずです。 （ 注： 一部のディストリビューションでは、これらのライブラリーは原則として<library-name> -devまたは<library-name> -develで命名されています ）。



このリストはあなたに役立ちます：

git make cmake boost boost-build kdepimlibs eigen kdegraphics-okular

libgsf libwpd libwpg libwps pstoedit glew gsl automoc4 boost libkdcraw

libpqxx fftw opengtl lcms2 vc exiv2


まれに、ディストリビューションが古くなっている場合と同様に、ライブラリの古いバージョンしか利用できない場合があります。 これらの依存関係を手動で収集する必要があります。 READMEファイルと検索エンジンは友達です。



構成





cmakeで設定すると、Kritaをビルドするのに十分なライブラリがインストールされているかどうかがチェックされます。 構成中にエラーが発生した場合は、欠落しているライブラリを調べてください。コマンドは、出力についてそれについて書き込みます。 これが、すべてが正常であるかどうかを確認することが重要な理由です。



フォルダー構造についてcmakeに通知し、Kritaのみをビルドするコマンドも提供します。 ただし、最初にcdを実行して、すべてを適切な場所で実行します。



cd〜/ kde4 /ビルド

cmake -DCMAKE_INSTALL_PREFIX = $ HOME / kde4 / inst $ HOME / kde4 / src / calligra -DCMAKE_BUILD_TYPE = RelWithDebInfo -DPRODUCTSET = KRITA


組立





設定レポートを読んだ後、すべてがうまく見えたら、Kritaをビルドします。

/ kde4 / buildフォルダーに残り、 -j < mount >パラメーターを指定してmakeを呼び出します。ここで、<mount>は、プロセッサーでサポートされている並列操作の数に置き換える必要があります（+1が推奨される場合があります）。 たとえば、8つのコアがあるため、-j9を使用します。



make -j9


ヒント：コアの数がわからない場合は、簡単なコマンドで確認できます。



cat / proc / cpuinfo | grepプロセッサ| wc -l


設置





アセンブリがエラーなしで100％に達した場合は、 makeコマンドを使用して、フォルダーにファイルをインストールできます。



-j9をインストールする


パスと環境変数







インストールは完了しましたが、システムはインストールフォルダーの内容をシステム上のアプリケーションセットの一部としてまだ考慮していません。 システムに正しいパスを表示しましょう。 ターミナルで次の行を順番にコピーします。



export KDEDIRS = $ HOME / kde4 / inst：$ KDEDIRS

export PATH = $ HOME / kde4 / inst / bin：$ PATH


残念ながら、これらの環境変数は一定ではないため、システムの電源を切ったり再起動したりすると失われ、ブリッジが崩壊します。 ログインするたびに確認するには、お気に入りのテキストエディターを使用して〜/ .profileファイルの最後に挿入します（一部のディストリビューションでは、プロファイルはxprofileと呼ばれ、home / <username>フォルダーの隠しファイルを確認します）。 その後、Kritaをシステムに登録する必要があります。kbuildsycoca4はターミナルで起動した後にこれを行います：



kbuildsycoca4


最初の打ち上げ





おめでとうございます！ Krita-minute- freshnessを実行するには、 ターミナルで kritaを記述するか、システムメニューを使用します。

Kritaがそこに表示されない場合は、デスクトップ環境のドキュメント「カスタム起動ボタンの作成方法」をお読みください。



更新する





新しい素晴らしい機能の追加について聞いたことがあるか、迷惑なバグの修正について読んで、バージョンを更新したいですか？ 落ち着いて、もう一度gitを呼び出します。 不足しているコード行をフォルダーに追加します。 パッケージ全体をダウンロードする必要はありません。



コードフォルダーに移動（cd）してから、 git pullにコードの更新を依頼します。

cd〜/ kde4 / src / calligra /

git pull




ただし、コードを更新するだけでは十分ではありません。 Kritaの新しいバージョンをテストするには、設定、コンパイル、インストールを繰り返す必要があります：

cd〜/ kde4 /ビルド

cmake -DCMAKE_INSTALL_PREFIX = $ HOME / kde4 / inst $ HOME / kde4 / src / calligra -DCMAKE_BUILD_TYPE = RelWithDebInfo -DPRODUCTSET = KRITA

make -j8

-j8をインストールする


毎日更新する場合、独自の最小限のスクリプトを作成してプロセスを自動化できます。 私は 'KDE paste'サービスを使用して私のものを共有しました 。



回復





開発者向けの最新バージョンがクラッシュしたり、使用できなくなることがあります。 実験的な変更は毎日追加されます。 「過去に戻る」方法がわからない場合、これは生産性を損なう可能性があります（たとえば、お気に入りのブラシが機能しなくなったなど）。 ただし、これを行う方法を知っていれば、 混乱することはありません。前の状態に戻ることができるからです。



コードと時間を旅するには、年表に慣れる必要があります。 ターミナルでは、これにgitログが使用されます。

cd〜/ kde4 / src / calligra /

git log


git logを使用すると、「コミット」と呼ばれるコード変更を追跡できます。 長い識別番号（例：15c1749105a7e7cc56b646a66919c29646f8653a）に興味があります。 git logをスクロールし、ID番号をコピーして終了します（キーボードのQの文字）。 それでは、コードフォルダーに戻ってみましょう。

git checkout 15c1749105a7e7cc56b646a66919c29646f8653a


これで構成、コンパイルが完了し、再び安全になりました。 また、 git履歴を表示するためのグラフィカルインターフェイスを持つgitgまたはgitkパッケージをお勧めします。 オンライン版もこちらから入手できます。



masterという名前のインスタントフレッシュネスコードに再びアップグレードするには、 git checkoutでgitにアクセスして（ pull ）アップデートを取得するようにgitに依頼します。

git checkout master

git pull
