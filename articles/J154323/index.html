仮想化されたアプリケーションオブジェクトの個別リカバリ

最新のバックアップ製品の重要な機能は、ファイル、電子メール、ワークフローシステム内のドキュメント、およびさまざまなアプリケーションのその他のオブジェクトをきめ細かく復元する機能です。 このような完全なロールバックが行われると、システムのすべてのユーザーを「過去に」「キャスト」せざるを得なくなり、「一定期間のデータ損失」に関連する多くのビジネス上の問題が発生します。



システムが完全にロールバックされない場合、このようなきめ細かな回復には2つの方法があります。「 ユニバーサル 」（特定のアプリケーションに依存せず、システム全体をテストラボまたは「サンドボックス」に復元することによって実行されます）および「 特殊化 」（オブジェクトを抽出できる特定のアプリケーション用に設計された） 「サンドボックス」でアプリケーション（すべての依存コンポーネントを含む）を実行する必要なしに、アプリケーションデータのファイルから直接特定のアプリケーションのデータを取得します）。



さらに、これらのアプローチの本質は、Veeam Backup＆Replicationの例によって示されます。VeeamBackup＆Replicationは、特定の場合にユーザーにとって最適なものに応じて、両方のアプローチを使用できる機能を備えています。



ユニバーサルリカバリテクノロジー


Veeam Universal Application Item Recovery

ユニバーサルリカバリメカニズムは、選択した仮想マシンのディスクを一時的な場所にアンパックすることなく、相互依存する仮想マシンのセットをサンドボックス（本番ネットワークから隔離された環境）でバックアップリポジトリから直接起動することに基づいています。



使用シナリオの観点から言えば、作業プロセスの概略は次のとおりです。



  1. アプリケーションオブジェクト（たとえば、誤って削除されたExchangeサーバーのレター）を復元する必要があるユーザーは、U-AIRユニバーサルリカバリウィザードを起動し、関心のあるオブジェクトを復元する要求を送信します。
  2. Backup Enterprise Manager管理者には、新しい復元アプリケーションの登録がメールで通知されます。
  3. 回復のために要求されたデータに対するユーザー権限がチェックされます
  4. ユーザーには自分のアクセス権を確認した結果が通知され、権限がある場合、管理者はSureBackupバックアップの自動テストの技術を使用して仮想マシンを復元するタスクを開始します。 このテクノロジーにより、ディスクを一時的な場所に解凍することなく、バックアップリポジトリから直接仮想マシンを実行できます。
  5. 仮想マシンのセットが復元され、サンドボックスで起動されるとすぐに、ユーザーは目的のアプリケーションを備えた準備済みの仮想マシンへのリンクを含む通知を受け取ります。
  6. 受信したリンクを使用して、ユーザーは関心のあるアプリケーションオブジェクトを個別に復元できます。
  7. リカバリ手順の最後に、仮想マシンは自動または手動で（管理者によって）シャットダウンされます。
単純な回復シナリオが使用され、ユーザーが必要なすべての権限を持っている場合、および製品の推定使用の場合、上記のシナリオでのアプリケーションの操作は実行されない場合があります。



専用の個別リカバリ技術


Microsoft Exchange向け Veeam Explorer

製品は、インストールされている仮想マシンの使用可能なバックアップコピーを使用して、削除されたMicrosoft Exchange Server電子メールを回復します。 あらゆる種類のバックアップをリカバリに使用できます（完全、増分、圧縮、および/または重複排除）。 Veeam Explorer for Microsoft Exchangeを使用すると、仮想マシンのディスクからExchangeサーバー情報データベースを直接開いて、レターを復元するときに管理者が通常実行する一般的な操作を実行できます。Exchangeメールボックスの内容を表示し、ユーザー指定の基準に従って検索し、メッセージをメッセージ形式またはグループで復元できます.pstファイル形式の文字。 製品は、エクスプローラーのようなシンプルなインターフェイスを使用します。



この復旧方法では、相互依存仮想マシンの「ファーム」は「サンドボックス」で開始されません。これは、Exchangeサーバーが原則として開始されないためです。代わりに、バックアップリポジトリから直接、Exchange情報ストレージファイルがある仮想マシンのドライブがマウントされますその後、適切な形式のパーサーを使用して低レベルで動作します。



個々のレターの実際の回復操作に加えて、Veeam Explorer for Microsoft Exchangeでは、さまざまな検索条件と結果をPSTファイルにエクスポートする機能を使用して、コンプライアンス検索（法律の要件に従って検索）を実行できます。



どちらを選ぶか？


ユニバーサルリカバリはほとんどの場合に適用できますが、サンドボックス内の依存サービスを使用して多数の仮想マシンを実行する必要がある場合があります。 たとえば、Exchangeサーバーを復元するには、（少なくとも）ドメインコントローラーと（場合によっては個別にインストールされた）DNSサーバーを備えた仮想マシンを復元する必要があります。 このような「ファーム」を起動するプロセスでは、仮想マシンとサービスの開始を待機するネットワーク遅延が発生する可能性があり、これは復元されたアプリケーションサーバーの操作性に依存します。 サーバーファームを順次起動するプロセス全体は、アプリケーションサーバーを起動しない特殊な回復よりも大幅に時間がかかる場合があります（アプリケーションデータベースファイルで起動するにはオペレーティングシステムのみが必要です）。 ただし、特別に設計されたパーサーまたは公開されているパーサーの数によって決定される、最も一般的なアプリケーションサーバーの限られたリストに対して、通常、特殊な回復が利用できることに注意する必要があります。したがって、常に適用できるとは限りません。



したがって、次の推奨事項を作成できます。


「特別な回復」に製品を使用することをお勧めします 。

  • 相互接続された仮想サーバーのセットで構成される復元されたアプリケーションサーバーの複雑な構成が存在する場合（たとえば、分散Exchange組織）-回復操作の速度の理由から。
  • 必要に応じて、たとえば「法律の要件に従って検索する」など、ユニバーサルリカバリ製品では通常利用できない特殊な操作を実行します。
  • 可能であれば、ユーザードキュメントの復元を管理者に委任します 。 つまり、ユーザーがドキュメントを復元するプロセスでそのすべてのコンテンツに慣れる（潜在的な）権利を管理者に提供する準備ができている場合です。 復旧リクエストの例：「 誤って削除されたレターを今年の第2四半期の財務報告とともに復元してください 。」 このような要求を実行することにより、管理者は、以前にキーワードで選択されていた場合でも、その期間に削除されたすべての文字を表示する必要があることは明らかです。


ユニバーサルリカバリツールを使用することをお勧めします。

  • ユーザーが自分でレターを復元する必要がある場合、管理者に復元中のドキュメントの内容に慣れる権利を与えたくないため、
  • アプリケーションサーバーの構成の複雑さとサイズがそれほど大きくない場合、ユニバーサルリカバリにはそれほど時間はかかりません。
  • （明らかに）必要なアプリケーションサーバー用の特別な回復ツールがない場合 。
