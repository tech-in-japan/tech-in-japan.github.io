「サイトは表示されません」

サイトをまったく表示していないユーザーを支援する記事。



これはまさに、顧客サイトで作業しているときに遭遇した問題です。 さらにテクニカルサポートへの問い合わせがいくつか届いた後、「私のサイトが開かない」ことが明らかになりました。これはインターネットの悪さや不正な手によるものではなく、よりグローバルなものです。



問題の症状は次のとおりです。 コンピューターはインターネットに接続され、HabrahabrとGoogleが正常に開きます。 クライアントサイト-いいえ。 サーバーはpingしません。 奇妙なこと-サイトが開く同じネットワーク上の他のコンピューターから、問題はありません...



「ページを表示できません」というエラーを報告したクライアントはあまり役に立ちませんでした。 私は欠陥のあるコンピューターを探して、それらの問題が何であるかを理解しようとしました。



この場合：

  • 定期的に更新されるウイルス対策がコンピューターにインストールされます。
  • オペレーティングシステムを再インストールすると、問題は解決します（それでも！）


2時間の実験で次の結果が示されました。

  • ping要求は、最も近いネットワークルーターでも表示されません
  • 他のサーバーへのリクエストは非常に成功しています


実際、さらに-技術の問題。 route print



は、ゲートウェイ192.168.0.0



（存在しないゲートウェイ）を通る巨大なルーティングルールが存在する巨大なテーブルを示していることが判明しました。



問題の解決策は非常に簡単です。魔法のコマンドで十分です。



route -f







または同様：



route delete *







このコマンドは、ルーティングテーブルをクリアします。 その後、デフォルトゲートウェイを介したルーティングを追加することにより、インターネットを「修正」するだけです。



route add 0.0.0.0 mask 0.0.0.0 192.168.0.1







当然、ゲートウェイアドレスは異なる場合があります。 または、単に再起動することもできます-通常の時間に気の毒ではないが、脳の時間には気の毒だと感じた場合。



さらなる調査により、クライアントのコンピューターがウイルス（またはワーム？）に感染している可能性が最も高いことがわかりました。Backdoor.Win32.Shiz。* Groupから、多くの情報を取得できませんでした。



しかし、問題が発見され、解決策が見つかった、他の理由。 しかし、疑問は未解決のままでした。そのようなウイルスがまだ必要なのは誰ですか 競合他社のサイトへのアクセスをブロックすることで、サイトを宣伝することを誰かが決めましたか？ それとも、「クリーンなインターネット」を求める闘いですか？ わからない...
