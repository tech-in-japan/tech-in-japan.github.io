Veeam Availability Suite 9.5-Veeam ONEの最新バージョンと無料のNFRキーについて

今日の私の話は、仮想環境とバックアップインフラストラクチャを監視およびレポートするVeeam ONE 9.5ソリューションの新機能、および誰がどのように完全に無料で、つまり最近リリースされたVeeam Availability SuiteのNFRライセンスキーを無料で取得できるかについてです9.5。 詳細については、猫へようこそ。











Veeam ONE 9.5の新機能



おそらく、Veeam ONEに長い間触れていないので、この製品はVMwareおよびHyper-V仮想環境のパフォーマンスと状態、およびVeeam Backup＆Replicationインフラストラクチャを監視でき、アラートの生成、統計パネルでの最新の要約の表示、問題をできるだけ早く解決するための組み込みの知識ベース。 さらに、Veeam ONEは、IT管理者の作業に非常に役立つ既製レポートとカスタムレポートの両方を提供します。リソースの配分を考慮し、最大容量（購入の正当化に役立つ）の達成を予測し、実行できない保護されていないVMの概要を提供します構成機能などによるバックアップ。



Veeam Backup＆ReplicationのようなVeeam ONEの新しいバージョンでは、一部の変更がデータ収集と処理の技術に影響を及ぼし（つまり、「内部」のまま）、レポートと通知に有用なニュースが表示されました。



変更の中で、まず第一に、Windows Server 2016のサポート：



  • すべてのVeeam ONEコンポーネントは、このOSを実行しているマシンにもインストールされます。
  • Veeam ONEは、Microsoft Hyper-V Server 2016でも仮想環境の監視とレポートを行うようになりました


AlwaysOn可用性グループをサポートするSQLサーバー上にVeeam ONEデータベースを作成できるようになり、新しくデプロイされたVeeam ONE WebサイトがHTTPSを使用するようにデフォルトで設定されるようになりました。

さまざまな規模のインフラストラクチャの展開シナリオに対応するデータを収集するためのいくつかのモードがあり、収集するメトリックに応じて構成されます。 最初の2つのモードは、仮想環境とバックアップインフラストラクチャの両方からデータを収集することを目的としています。3番目は、バックアップインフラストラクチャからのみです。



  • 標準展開用に最適化 （ 標準展開に推奨）-100台以下のホストと1500台以下の仮想マシンを含む中小規模の仮想インフラストラクチャに推奨されます。 このモードでは、Veeam ONEはパフォーマンスメトリックだけでなく、インフラストラクチャのトポロジと構成に関するすべてのデータを収集します。 それらに基づいて、レポートが作成され、監視中のアラートで使用されます。 このモードの利点は最大のデータ粒度ですが、欠点もあります-Veeam ONEサーバーの負荷が増大し、データベースサイズが大きくなります。



  • 高度なスケーラビリティ展開向けに最適化（スケーラビリティ向上のために最適化）-大規模なインフラストラクチャ（100台以上のホスト、1500台以上のVM）に推奨。 このモードでは、アラートおよびレポートの操作に必要なパフォーマンスメトリックが収集されます。 このモードでは、同等の高度なデータ粒度はありませんが、Veeam ONEサーバーの負荷とデータベースサイズははるかに小さくなります。



  • バックアップデータのみ -仮想環境の分析を掘り下げることなく、Veeam Backup＆Replicationインフラストラクチャの監視とレポートに集中したい場合にお勧めします。 このモードでは、トポロジ、構成、およびパフォーマンスメトリックを含むデータがVeeam Backup＆Replicationサーバーから収集されます（パフォーマンスメトリックは仮想化サーバーから収集されません）。したがって、レポートと通知ではバックアップ操作に関する情報のみが表示され、データパフォーマンスの観点では、VMware vSphereおよびMicrosoft Hyper-Vオブジェクトはそこに到達しません。 このモードは、最小のサーバー負荷と最小のVeeam ONEデータベースサイズを提供します。

監視とレポート



便利なランサムウェア検出アラームが登場しました 。これにより、プロセッサの使用状況を監視し、ストレージシステムに操作を書き込むことができます。 VMでこれらのカウンターが急激に増加した場合、内部のデータの暗号化に作業がある可能性が高くなります。



いくつかの新しいレポートは、たとえば、 VM構成のチャージバックレポートなど、サービスプロバイダーにとって非常に役立つと思います。特に、現在の構成のどのVMが最高か最低かを示します。 それに基づいて、各クライアントが消費するリソースのコストを計算し、適切な請求書を発行できます。 このレポートは、プライベートクラウドの各部門およびアプリケーション所有者に同様のサービスが提供される企業にも役立ちます。



レポートに含まれるもの：



  • 要約情報 -VMの数、ビジネスビューグループの数、1つのVMの平均コスト、すべてのVMの合計コスト。



  • 上位10の最も高価なVM （10の最も高価なVM ）-割り当てられたリソースに応じたVMコストチャート、それぞれのvCPU、vRAMおよびストレージのコストをカウントします。


コスト分布 -リソースのコストの図。

VM電源ステータス -この図は、オンになっているVMの数とオフになっているVMの数を示しています。

ビジネスビューグループのコスト -管理ビューでビジネスビューの分類を使用してレポートを作成した場合に作成され、カテゴリに従ってVMのコストが表示されます。

詳細 -現在のVM構成のコストに関する詳細データの表。









さらに新しいレポートが提案されています：



  • VMパフォーマンスチャージバック-CPU使用率とメモリ使用率が最高および最低のVMを報告します 。
  • ホスト構成のチャージバック -ハードウェアコストと現在の構成に関して、どのVMが最も高価で最も安価かを報告します 。


また、一般に、レポートを生成する時間が大幅に短縮されたことに注意してください。

今回は無料のNFRキーを配布しますか？



はい、このバージョンにもこのような機会があります。 VMwareおよびHyper-Vプラットフォーム用のVeeam Availability Suite 9.5の2 CPUソケットの場合、1年間、NFRライセンス（商用利用ではない）を取得できます。 これを行うには、 サイトと右側のフォームにアクセスして、あなたのメリットが大切な鍵を受け取る権利を与えていることを確認してください。つまり、あなたは次のタイトルの所有者です。



vExpert、VCP、VCAP、VCO、VCDX、VMUG Leader、VTEC、VMCE、MVP、MCSE、MCSA、MCT、Cisco Champion、PernixProメンバー。



タイトルがリストにない場合-絶望しないで、リストから「 その他」を選択して登録フォームに入力し、 「Get My NFR Key」という碑文が付いた緑色のボタンを押して、指定したアドレスへの手紙を待ちます。



注：たとえば、ブロガーで、自宅のメールしか持っていない場合は、フォームではなくveeam_team@veeam.comに申請書を送信できます。
