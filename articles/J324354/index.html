Project Olympus：マイクロソフト独自のオープンソースサーバーハードウェア





ソーシャルネットワークFacebookは、データセンター用のオープンハードウェアを開発した最初の企業の1つです。 このタイプの機器は、プロプライエタリなサーバーや他のシステムよりも製造、運用、アップグレードが安価であるため、これは会社にとって有益です。 ソーシャルネットワークは、2011年にOpen Compute Projectの作業開始を発表しました。 それ以来、マイクロソフトを含む複数の通信大手がプロジェクトに参加しました。



11月、レドモンド社は、50％に対応する独自のバージョンのサーバー用オープンハードウェアを導入しました。 マイクロソフトは、サードパーティの専門家と愛好家にプロジェクトの完了を依頼しました。 プロジェクト自体はProject Olympus（会社の「クラウド」サーバーとも呼ばれます）と呼ばれていました。 開発者が考えたように、データセンターの新しい機器は単一の標準に準拠し、Microsoftのすべての作業（AzureからOffice 365、Xbox、Bingまで）を確実にします。



企業は、機器をカスタマイズし、特定のタスクを実行するようにカスタマイズするだけでなく、X86やARMなどのさまざまなアーキテクチャと互換性のある単一のプラットフォームを取得したいと考えています。 新しい標準Microsoftのハードウェアは、x86チップ向けのIntelやAMD、ARM向けのQualcommやCaviumなどの企業の生産を説得したいと考えています。 また、ベンダーが新しいプログラムの作業を迅速に開始するほど、プロジェクトの実装が迅速になります。







「国内のニーズに必要なものすべてを生産するスペシャリストのチームではなく、サードパーティのメーカーと交渉し、ハードウェアとソフトウェアのエコシステムを開発し、市場に新しいソリューションを提供するつもりです」



経営陣によると、同社はすでにOpen Computeプロジェクトの一部として開発および運用されているオープンハードウェアのすべての利点を評価しています。 現在、同社が購入した機器の約90％は、ラック、配電システムの要素、およびその他のコンポーネントを含む標準OCPを満たしています。 そのような機器を使用する目的は、サーバー機器の大きく多様な「動物園」に関連する運用コストを削減することです。 10〜20種類のサーバーハードウェアの状態を監視する必要性について話し合っています。それぞれのサーバーハードウェアには、異なる方法の運用と保守が必要です。











標準化は、マイクロソフトだけでなく、他の企業も時間とお金を節約する方法を学ぶのに役立ちました。



Project Olympusは、機器の統合と統合データセンターインフラストラクチャの形成における次のステップです。 重要な側面の1つは、電力システムのローカル特性に関係なく、30 Aまたは50 A、380 V、415または480のいずれであっても、ほぼ同じスキームを使用して世界中のデータセンターを作成できるユニバーサルエネルギー分配システムです。同じ要素を持つ標準ラックは、オズボーン、ロンドン、北京、またはムンバイのデータセンターに設置できます。



現在、Project Olympusはすでに80％完成しているため、Microsoftの担当者は、開発されたハードウェアの特定の要素を既に実稼働に移行できることを望んでいます。 残念ながら、Microsoftはある程度の秘密を保持しているため、この点についてはあまり明確ではありません。







理解できる限り、ベンダーは次のようなプロジェクトへの参加に同意しています。



Qualcomm、Intel、Dell Technologies、Hewlett Packard Enterprise、Advanced Micro Devices、Samsung Electronicsなどの企業。 Project Olympusは、とりわけ、Nvidiaが開発したAIプロジェクトのハードウェアプラットフォームと互換性があります。



興味深いことに、Microsoftは現在、サーバーチップの最大の購入者の1つです。 同社がx86から​​ARMアーキテクチャに焦点を切り替えた場合、サーバー市場におけるIntelのシェア（現在約99％）は大幅に減少する可能性があります。
