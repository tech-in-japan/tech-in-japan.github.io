UX / UIデザインとは実際には何ですか？

デザイナーと開発者の間の多くの誤解。 また、初心者向けのUXおよびUIに関連する愚かな質問も多数あります。 多くの場合、単に人々がUX / UIの概念の本質を知らず、自分が何について話しているのか分からないために、物事は固有の名前ではないと言います。



私は一度終止符を打って、UX / UIデザインが何を意味するかを、シンプルでわかりやすい言語で説明したいと思います。





刃を研ぐためのさまざまなタイプのインターフェース。





旋律計のボタンを押したり、砥石のハンドルをひねったり、スマートフォンの画面をスワイプしたりしても、これらはすべてメカニズムとやり取りするインターフェイスです。 さらに、インターフェイスはデジタルである必要はありません。 「インターフェイス」という言葉自体はごく最近英語から借用されたものであり、デジタル時代にのみ私たちに伝わりました。



UI / UXデザイナーの目標は、ユーザーをインターフェイスの論理的なポイントに導くことです。 ユーザーを目標に到達させます。

UXは ユーザーエクスペリエンス （文字通り「ユーザーエクスペリエンス」）の略です 。 つまり、これは、ユーザーがインターフェイスでの作業から受ける経験/印象です。 彼は目標を達成することに成功しているか、それを行うのがどれほど簡単か難しいか。



UIはユーザーインターフェイス （文字通り「ユーザーインターフェイス」）です。インターフェイスの外観と取得する物理的特性です。 それはあなたの「製品」が何色になるか、人が指でボタンを押すのが便利かどうか、テキストなどが読めるかどうかを決定します...



UX / UIデザインとは、使いやすさが見た目と同じくらい重要なユーザーインターフェイスのデザインです。



UXおよびUIデザインとは、つまり



UX / UIデザイナーの直接的な責任は、たとえば、インターフェイスを介して製品またはサービスを「販売」することです。 UX / UIデザイナーの作業に基づいて、ユーザーは「あるべきかどうか」という決定を下します。好きか嫌いか。 購入するかしないか。



実際、設計者の目標は異なる場合があります。 何かを「売る」必要はありません。 しかし、このテキストを初心者が理解しやすくするために、あまりにも抽象的なフレーズを使用したくはありません。 プレゼンテーションスタイルが90年代のサンプルのプログラミング言語の読者にならないように。


UX / UIデザインは、スマートフォンとWebサイトのみに適用されるわけではありません。 さらに、UX / UIデザイナーの職業は太古から存在していました。 以前はそう呼ばれていなかっただけです。 より正確には、それはまったく呼び出されなかったが、他の職業の一部であった前に。



最初の例は次のとおりです。WilliamSchickardが1623年に弾性計を発明したとき、彼はすでにUX / UIデザイナーでした。







なぜなら、計算の結果を得るために、人がどのタンブラーをどの順番でねじるかを発明したからです。 そして彼は、それらがどの論理的な順序で配置されるかを考えました。 そして一般的に、彼はこれらすべてのペンがどのように見えるかを思いつきました。 彼はマシンと対話するためのインターフェースを作成しました。



別のより古くて原始的な例は、砥石（ホイール）です。 中世初期でも、そのような車輪には多くの種類とメカニズムがありました。



  • 手でねじることができます
  • 他の人が車輪を回す可能性があります
  • または、足でペダルを踏むことで、手がなくてもねじれを解くことができます。
  • その他...


これらはすべて異なる種類のインターフェイスでした。







だから、次の砥石の発明者が考えたとき：



  • 彼は座って自分でペダルを踏みますか
  • または、彼はメカニズムを単純化しますが、手でホイールを回す奴隷を置きます、


その瞬間、彼はUXデザイナーでした 。



そして、石の大きさ、スタンドに使用する木の色、木の棒の固定方法（釘や革のひだ付き）、ハンドルの長さを考えた人はUIデザイナーでした 。



そして、あなたがあなたの剣を研ぐその方法は、 インターフェースと呼ばれるでしょう。



UXとUIの違いは、UXデザイナーが、ユーザーがインターフェイスと対話する方法と、何かを実行するために必要な手順を計画することです。 UIデザイナーは、これらの各ステップの外観を思いつきます。 上記の例からわかるように、UXとUIは非常に密接に関連しているため、概念間の境界線が洗い流されることがあります。 したがって、UXとUIの両方は通常、1人のデザイナーによって処理され、彼の職業は/で記述されます。



最近、UX / UIデザイナーの職業の普及は、デジタルテクノロジーの開発により深く関係しています。 しかし、まさにこの「ブーム」（毎回の仕事の発表で「UX / UI」という用語を見始めたとき）が名前自体に関係しているのは、誰かが最近思いついたものです。


UI / UXデザインは現在、デジタル業界で最も人気のある職業の1つです。 どのくらいの期間需要があるかは、この業界の発展次第です。 そして、明らかに、彼女は勢いを増しているだけです。



UXとUIはトレンドではありません。 技術者は進化しています。 サイトの需要は増加しています。 デジタルアプリケーションはキノコのように見えます。 また、設計および開発ツールは非常に単純化されているため、プログラミングの知識がほとんどない人でも、名刺サイトを「ひざまずく」ことができます。 それはこのサイトがどうにか見えるはずです。 テキストやボタンの抽象的なフレームとしてだけではありません。 ここで、プログラマーはUX / UIデザイナーの助けを必要とします。



WebデザイナーとUX / UIデザイナーへの分割は、インターネットの発展とともに現れました。 時間が経つにつれて、Webサイト専用のインターフェイスを作成するより狭い専門家が必要になりました。

はい。UI/ UXデザインは、Webデザインよりも広く包括的なコンセプトです。



PS一部の人々はUI / UXを書くが、私はUX / UIを書くことを好む。 そしてこれは、ワークフローでは最初にUXが作成され、次にUIが作成されるためです。 しかし、それは問題ではありません-あなたが望むように書いてください。 主なことは、作業プロセス中にこの順序を混同しないことです。 なぜなら、多くの初心者のデザイナーが自分のインターフェースにいくつかのクールなボタンとチップを思い付くようになるからです。 しかし、彼らはユーザーが一歩一歩どのように進むかについては考えていません。
