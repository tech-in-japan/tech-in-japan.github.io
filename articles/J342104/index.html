Clever Geek Handbook

5日間で、シリコンバレーの5社でインタビューを受け、5件の求人を受け取りました。

2017年7月24日から7月28日までの5日間、LinkedIn、Salesforce Einstein、Google、Airbnb、Facebookのインタビューに合格しました。 5社すべてが仕事を提供してくれました。 それは素晴らしい経験であり、努力が報われたことの幸運を理解しているので、それについて書くことにしました。 ここで、インタビューの準備方法、インタビューの方法、企業からの印象についてお話します。











それがすべて始まった方法



私はグルーポンでほぼ3年間働いていました。 これは私の最初の仕事です。 素晴らしい人々と素晴らしいプロジェクトがありました 。 私たちはあらゆる種類の興味深いことを行い、会社内で変更を導入し、そのようなものを公開しました。 しかし、時間が経つにつれて、私は自分の教育のペースが衰え始めた（言い換えれば、スローダウンする）ようになり始めたので、心に十分な食べ物がありませんでした。 さらに、シカゴのソフトウェア開発者と同じように、私はサンフランシスコのベイエリアに惹かれました。有名な会社がたくさんあります。



寿命は短く、職業寿命はさらに短くなります。 妻とすべてについて話し合い、彼女の完全なサポートを確保した後、私は決定的な一歩を踏み出し、人生で初めて仕事を変えることにしました。



準備する



機械学習に関連する職種に興味がありましたが、企業で利用できる求人は名目上およびインタビューの実施方法の点でさまざまでした。 3つでは、機械学習エンジニア（LinkedIn、Google、Facebook）、1つではデータエンジニア（Salesforce）、もう1つではソフトウェア開発者（Airbnb）を探しました。 したがって、機械学習、コード記述、システム設計という3つの分野の知識を一度に更新する必要がありました。



私はフルタイムで働き続けたので、準備のプロセス全体が2、3か月続きました。 これが私がしたことです。



コード



インタビュー中にコードを書くことは、開発者のレベルを完全に評価する最良の方法ではないことに同意しますが、短期間で優秀なプログラマーであるかどうかを判断する方がよいでしょう。 私の意見では、これは避けられない悪であり、目的の場所にたどり着くために通過しなければならないものです。



私は主にLeetcodeとGeeksforgeeksで練習しましたが、HackerrankとLintcodeも優れたリソースです。 数週間、一般的なデータ構造とアルゴリズムを繰り返し、あまり強くない分野を取り上げ、最終的に典型的なタスクの解決に取り掛かりました。 暇な時間がほとんどなかったので、通常は1日に2つのタスクに制限していました。



以下にいくつかの提案を示します。



  1. 可能な限り訓練します。 それなしでは仕方ありません。
  2. しかし、Leetcodeで600個すべてのタスクを解決する代わりに、さまざまなタイプを選択し、それぞれを慎重に理解してください。 合計で、約70のタスクを完了しました-これで十分だと感じました。 私はこのように見ています：70のタスクがあなたに何の利益ももたらさないなら、それはあなたがそれらに正しくアプローチしなかったことを意味し、700も役に立たないでしょう。
  3. 最も難しいものから始めます。 それらの後、他の誰もが簡単に見えるでしょう。
  4. 2時間経ってもまだ地面から降りられない場合は、解決策をご覧ください。 原則として、より多くの時間を費やすことは意味がありません。
  5. タスクを正常に完了した場合でも、答えを確認してください。 そこで提案されたソリューション、特にPythonの単一行のソリューションの妥当性と優雅さにしばしば疑問を抱かなければなりませんでした。
  6. 面接中の説明に問題がないように、あなたが最もよく知っている言語を使用してください。

インタビューはさまざまなトピックに触れる場合がありますが、一般的には、2つの一般的な規則に沿って、タスクにアプローチする方が適切です。



  1. 最初に要件を処理し、次に一般的なスキームの概要を説明してから、詳細を掘り下げます。 あなたが何を望んでいるかを正確に理解するまで、すぐに詳細を把握する必要はありません。
  2. 理想的なシステムは存在しません。 設定したタスクに基づいて、犠牲にするものを選択します。


要約すると、インタビューの前にシステム設計を実践する最良の方法は、座って何らかのシステムを設計することです。 結局のところ、それは仕事で毎日しなければならないことです。 上部をスライドしないでください-使用するツール、フレームワーク、ライブラリを深く研究してください。 たとえば、クライアントからDDLを開始する代わりにHbaseを使用する場合は、一般的なアーキテクチャを理解しようとします。たとえば、読み取りと書き込みの実行方法、HBaseの一貫性の提供方法、完全および不完全な圧縮（マイナー/メジャーコンパクション）が必要な理由、および場所システムはLRUキャッシュとブルームフィルターを使用します。 HBaseとCassandraを比較して、類似点と相違点を見つけることもできます。 この場合、分散ストレージを設計するように求められても、壁に閉じ込められていると感じることはありません。



ブログには、独立したプログラマーのページ、大企業のエンジニアのブログ、オープンソースプロジェクトの公式ドキュメントなど、多くの興味深いものがあります。



ここで最も重要なことは、好奇心を保ち、知識を過大評価しないことです。 接触するすべてのものを吸収するスポンジのようになります。



機械学習



機械学習の分野でのインタビューでは、理論と製品設計という2つの側面を区別できます。



この分野での研究経験がなく、対応するコースを優等で修了していない場合、特別な文献を読むことは害になりません。 統計学習とパターン認識の要素や機械学習のような古典的な作品は優れたオプションですが、より狭いトピックに興味がある場合は、必要なトピックで何かを拾うことができます。



変位と散布のジレンマ、再訓練、勾配降下法、L1 / L2正則化、ベイズ定理、バギング/ブースティング、協調フィルタリング、次元縮小などの基本概念をよく理解してください。 ベイズの定理などの最も一般的な式と、ロジスティック回帰やサポートベクトル法などの一般的なモデルの派生を調べます。 決定木やK-meansクラスタリングなどの単純なモデルを使用します。 履歴書でモデルに言及する場合は、徹底的に研究し、長所と短所について話す準備をしてください。



製品設計に関しては、作成プロセスがどのように展開するかを想像する必要があります。 私は次のように行動しました：



  1. 目標を定義します：予測、推奨、クラスタリング、検索など。
  2. 目的のアルゴリズムを選択します。教師、分類または回帰、一般化線形モデル、決定木またはニューラルネットワークなどがあります。 選択した引数を準備します。
  3. 利用可能なデータに基づいて必要な機能を選択または作成します。
  4. モデルのパフォーマンスを評価するメトリックを選択します。
  5. オプション：開発に提出する前にこのモデルを改善する方法を示します。


ここでは、好奇心を持ち続け、常に新しいことを学ぶことがいかに重要であるかを再度強調したいと思います。 Spark MLlibまたはXGBoostのAPIを使用して達成感を持って「Done！」と言うだけでなく、確率的勾配降下が分散学習に適している理由を見つけ、XGBoostと従来の勾配ブースティングの違いを理解してください。彼の損失関数の特異性、二次導関数を計算する理由、およびそのようなこと。



インタビューはどうでしたか



私は、LinkedInでHRマネージャーの登録を解除し、推奨事項を尋ねることから始めました。 スタースタートアップへの参入に1回失敗した後（これについては後で説明します）、数か月間慎重に準備し、最後に採用担当者の助けを借りて、シリコンバレーの企業へのインタビューを1週間スキャンしました。 私は日曜日にそこに飛び、5日間、世界最高のIT企業の30人以上の専門家とコミュニケーションを取りました。 私は非常に幸運でした-5社すべてが私に空席を提供しました。



電話インタビュー



すべての電話インタビューは標準でした。 期間だけが異なりました。たとえばGoogleなどの誰かと、1時間、他の人（Facebook、Airbnb）と45分間話しました。



ここで資格が前面に出ます。あなたは非常に厳しい時間枠によって制限されており、ほとんどの場合、二度目のチャンスはありません。 その場でタスクの種類を決定し、一般的な用語でソリューションを説明する必要があります。 目標と思考の流れを必ず発音してください。 最初は、これにより少し遅くなるかもしれませんが、面接ではコミュニケーションが最も重要な役割を果たします。 多いほど良い。 既製のソリューションを読もうとしないでください。対話者があなたのトリックを理解する可能性が非常に高いです。



機械学習エンジニアとしての職に応募する場合、この分野から質問されることがあります。 それの準備をしなさい。



時間を最適に管理するために、同じ日に1時間間隔で3つのインタビューをスケジュールしました。 ここでの利点は、良いスタートを切った場合、「フラッディング」効果が機能する可能性があることです。 欠点はそれぞれ、失敗した最初のインタビューが次の結果にも影響する可能性があることです。 したがって、この戦略はすべての人に適しているわけではありません。



いくつかの企業で選択を一度に渡すと、いくつかの利点が得られるという意味で便利です。 LinkedInとFacebookが最初の電話での会話の後に私を個人的な会議に招待したという事実により、AirbnbとSalesforceは2回目のインタビューも行いませんでした。



Googleはさらに驚いた。彼らは電話インタビューの段階をまったくスキップすることができ、翌週に4回のインタビューが予定されていると聞いてすぐに無料の日の予約をした。 Googleからの招待を辞退することは非常に難しいことを理解しました。

一般に、Airbnbのインタビューは他のインタビューよりも難しいことがわかりました。インタビューは長く、タスクは複雑で、セッションは異常な形式です。 そこに行きたいのなら、まず企業文化と価値に精通してください。

Databricksの失敗したジョブインタビュー











サクセスストーリーは常に敗北から始まり、インタビューも例外ではありません。 5月に、これらの企業に立候補する前に、Databricksとのインタビューに失敗しました。



4月に、従業員の1人であるXiangruiから連絡があり、Spark MLlibチームの一員になる機会に興味があるかどうか尋ねました。 私は3つの理由で7番目の天国にいました。1）彼らはSparkでプログラムします。私はScalaが大好きです。 2）Databricksにはクールなエンジニアがいます。 3）現在、Sparkはビッグデータの世界に革命を起こしています。 そのような機会を逃すことは不可能であり、数日後、私は選択に合格し始めました。



バーは非常に高く、プロセス全体に時間がかかります。最初にアンケートに記入し、次に電話でインタビューを行い、次にコードを書く作業を完了し、最後に個人的な会議に行きます。



私はなんとか個人的な会合の舞台に着き、トレジャーアイランドを見下ろすサンフランシスコのダウンタウンにある彼らのオフィスを訪問しました。



非常に高いクラスのスペシャリストが私と話しましたが、同時に苦情もありませんでした。 インタビューの間、私はしばしば自分の能力の限界に達していると感じていました。 あるセッションで大きな方法で穴を開けるまで、すべては多かれ少なかれスムーズに進みました-十分な知識と準備がなく、すべてが完全に失敗しました。 Xiangruiは私にとても親切で、インタビューの後で私が行っていた場所に連れて行ってくれました。 彼と話すのはよかったです。



数日後、私は拒否の手紙を受け取りました。 これは予想されることでしたが、しばらくの間、私はいらだちに打ち勝ちました。 私はDatabricksで仕事を得ることができませんでしたが、彼らが業界に多大な貢献をし続け、より多くを達成し続けることを心から願っています。



おわりに



  1. 人生は短いです。 職業生活はさらに短くなります。 あなたが一歩を踏み出す必要がある瞬間をお見逃しなく。
  2. 面接は面接だけでなく、ネットワーキングをして新しい友達を作る機会でもあります。
  3. 常に好奇心を持ち、何か新しいことを学ぶ準備をしてください。
  4. 条件の議論の結果は、あなたの仕事の満足度を決定します。
  5. 仕事を提供された場合、最低要件を満たします。 最大要件はありません。 改善し続けます。


私の仕事を変えるプロセスは長い間続きました。最初の面接に行った5月から、9月末に申し出を受け入れたときまで、それは簡単ではありませんでした。 誰も私の主な仕事をキャンセルしていないので、準備するのは困難でした。 数週間、私はそのようなスケジュールに従って生活しました。インタビューのタスクを午前1時まで完了してから、8時30分に起床し、1日を会社の作業プロセスに完全に没頭させました。
