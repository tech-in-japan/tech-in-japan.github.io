Windows Serverのライセンスに関する注意事項、またはライセンス監査で会社を破産させない方法

マイクロソフトのライセンスポリシーは、かなり複雑でわかりにくいものです。 ただし、多くはすでにすべての基本的なルールを学び、特定の製品のライセンスを取得する方法を知っています。 しかし問題は、わずかなニュアンスを1つだけ考慮することなく、数桁の間違いを犯す可能性があることです。 そして、そのような間違いは、マイクロソフトからのライセンス監査プロセス中に会社の破産につながる可能性があります。



たとえば、ほとんどすべての会社で使用されているWindows Serverを考えてみましょう。 ほとんどすべてのシステム管理者は、Windows Server StandardのライセンスがWindows Server Datacenterとどのように異なるかを知っています。 バージョン2016で再び登場した機能の違いは省略します。主な違いは、1つのWindows Server Standardライセンスを1つのホスト内の2つの仮想マシンにインストールでき、Datacenterは単一ホスト内の無制限の仮想マシンにインストールできることです。 2番目の重要な要件は、サーバーのすべての物理コア（以前はプロセッサー）のライセンスです。 これは、誰もが知っているように思えます。 しかし、ご存知のように、悪魔は詳細にあります。



実用的な問題を考えてみましょう。



5ノードのHyper-Vクラスターがあります。 各ホストには、20コアの2つのプロセッサがあります。 また、各ホストには、Windows Serverを実行する10台の仮想マシンがあります。 prolicensingがより収益性が高い方法を推定しましょう。



まず、Windows Server Datacenterを計算しましょう



Datacenterを使用する場合、仮想マシンの数に関係なく、ホスト上のすべてのカーネルのライセンスを取得するだけです。 つまり 2 * 20 = 40コアホストライセンス。 Windowsは2つのコアライセンスで販売されているため、20の2Coreホストライセンスを取得します。 5つのホストがあるため、20 * 5 = 100の2コアライセンスを取得します。



コストで：



9EA-00128 WinSvrDCCore 2016 SNGL OLP NL 2Lic CoreLic 40579.94 * 100 =4'057'994。 （400万ルーブル）


それでは、Windows Server Standardを計算しましょう



ホストごとに10個の仮想マシンがあります。 ホストあたり10 VMには、5ライセンスのWindows Serverが必要です。 （1つの標準ライセンスは2つのVMであるため）。 ホストあたり2x20 = 40コアであるため 40コアのライセンスを5回取得する必要があります。 -これは200コアライセンスです。 ただし、Windows Server Standardは2ライセンスで販売されているため、ホストごとに100個の2Coreライセンスが必要です。 合計で5つのホストがあるため、5を掛けます。500の2Coreライセンスを取得します。



コストで：



9EM-00124 WinSvrSTDCore 2016 SNGL OLP NL 2Lic CoreLic 5815.38 * 500 = 2'907'690摩擦 （290万ルーブル）


Windows Server Standardのライセンスを取得すれば、Datacenterと比較して100万ルーブルを節約できるようです。 だから？



いいえ、そうではありません！



私たちは、誰もめったに注意を払わない小さな些細なことを1つ逃しました。 すでに私が言っていることを推測しましたか？



そうでない場合、小さなことは、クラスターがあることを最初に書いたことです。 また、クラスター内では、任意の仮想マシンがホスト間を移動できます。 さらに、リソースの動的な割り当てがない場合でも。 ホストの1つに障害が発生した場合に備えて、別のホストに移動できます。 また、Microsoftのライセンスポリシーに従って、理論的に機能するすべてのホストにすべての仮想マシンのライセンスを付与する必要があります。 監査人にとっては、HAクラスターがあれば十分です。 さらに、このシナリオはドキュメントに明確に記載されています（写真を参照）。









そして今、私たちはシナリオのWindows Server Standardライセンスを「正しく」数え上げます。



10台のマシンの5つのホストはそれぞれ50台の仮想マシンです。 25ライセンスWindowsが必要です。 各仮想マシンは、すべてのホストにライセンスを付与する必要があります。 すべてのホストのすべてのコアに対して、これは200コアです。 つまり 200 * 25 = 5,000コアライセンスを取得します。 まあ、または2500 2Coreライセンス！



コストで：



9EM-00124 WinSvrSTDCore 2016 SNGL OLP NL 2Lic CoreLic 5815.38 * 2500 =14'538'450。 （1450万ルーブル）


1450万ルーブル-290万に対して、最初に計算しました。 5xエラー。 ホストが5つではなく、50または500の場合、災害の規模を想像できます。



つまり この場合、当然、より高価なDatacenterライセンスですべてのホストにライセンスを付与する方がはるかに安くなります（これには400万ルーブルの費用がかかります）。 一見したところ、標準バージョンの方が安いようです。
