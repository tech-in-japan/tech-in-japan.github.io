wordpress Webサイトが1日あたり42,735,587ヒットの負荷に耐えられるように、VPSを5ドル（512MB RAM / 1 CPU）で最適化します

256 MBまたは512 MBのRAMを搭載し、プロセッサパワーの一部のみを搭載したVPSサーバーを購入する場合、MySQL / PHP / Apacheなどのサービスのデフォルト設定を使用するのは非常に悪い考えです。 現在、512 MB RAM / 1 CPUを搭載した最も安い料金プランで3つのサイトを立ち上げました。 完全には定かではありませんが、出席者は1日あたり約5〜1万人です。 次に、512 MBのみを使用してスワップを行わずにLAMPを最適化する方法に関する指示を共有したいと思います。 通常、この設定では256〜378Mbのメモリが使用され、すべてが非常に迅速に機能します。

$ free -m

 $ ps -eo pmem,pcpu,rss,vsize,args | sort -k 1 -r | less

アパッチ


Apacheの最大の問題は、Apacheが使用するRAMの量です。 作業を高速化し、RAMの消費を削減するために、次の方法を検討します。



  • 同時リクエストの処理を減らす
  • モジュールの読み込みが少ない（未使用を無効にする）。
  • ロギングが少なくなります。




実行中の最も少ない子プロセスのみを使用するようにApacheを構成する



Preforkは本当の魔法が起こる場所です。 これは、多くのプロセスを生成するようにApacheに指示する場所です。 デフォルトでは、大量が割り当てられ、サーバーのRAMが消費されます。 apache2.confがあまりにも多くのサーバーを実行するように構成されていないこと、または多くの予備サーバーがあることを確認してください。 以下に例を示します。 



 <IfModule mpm_prefork_module> StartServers 1 MinSpareServers 1 MaxSpareServers 3 MaxClients 10 MaxRequestsPerChild 3000 </IfModule> <IfModule mpm_worker_module> StartServers 1 MinSpareThreads 5 MaxSpareThreads 15 ThreadLimit 25 ThreadsPerChild 5 MaxClients 25 MaxRequestsPerChild 200 </IfModule>


また、「 KeepAliveTimeout 」パラメーターを必ず10または15に調整してください。私の意見では、15秒は小さなページを表示するのに必要な長さよりも長く、ページの表示に長い時間がかかるよりも短くなります。



最も必要なモジュールのみをダウンロードする


デフォルトのApache Webサーバーは、不要なモジュールを大量にロードします。 次のコマンドを使用して、インストールおよび有効化されているモジュールを確認できます。 



 # apache2ctl -M


以下は、Wordpressが機能するために必要なモジュールのリストです。 

 LoadModule dir_module modules/mod_dir.so LoadModule log_config_module modules/mod_log_config.so LoadModule mime_module modules/mod_mime.so LoadModule setenvif_module modules/mod_setenvif.so LoadModule alias_module modules/mod_alias.so LoadModule authz_host_module modules/mod_authz_host.so LoadModule rewrite_module modules/mod_rewrite.so


メモリを節約するために、残りのモジュールをコメント化する必要があります。 または、コマンドラインからモジュールを有効/無効にすることができます。 有効にするには、次のコマンドを使用します。 



 # a2enmod module_name


無効にするには： 



 # a2dismod module_name


操作後、Apache Webサーバーを確実に再起動する必要があります。 



 # service apache2 restart

MySQLサーバーの最適化


少量のRAMを使用するようにMySQLを調整するのは非常に簡単です。

次に、次のタイプのMySQL設定を検討します。



  • オフにできるもの。
  • MySQLサーバーのパラメーターの最適化。
  • サードパーティのMySQL構成カスタマイズツール。




MySQLを最適化するには、 / etc / mysql / my.cnfファイルを編集する必要があります。



オフにする必要があるもの


Mysqlはいくつかのテーブルストレージエンジンを提供します。 そのうちの2つが最も人気があり、 InnoDBとMyISAMです。 それらの主な違い：

  • MyISAMはテーブルレベルのロックを提供します。つまり、情報がテーブルに書き込まれると、テーブル全体がロックされ、その時点で同じテーブルで同時に実行する必要があるレコードがある場合、最初のレコードが正常に追加されるまで待機する必要があります;
  • 一方、 InnoDBは行レベルのロックを提供します。つまり、文字列への書き込みが発生すると、その単一の行のみがロックされます。 残りは記録に使用できます。




テーブルレベルのロックの問題は、非常にビジーなサーバーでのみ顕著です。 通常のWebサイトの場合、MyISAMは安価なサーバーで最高のパフォーマンスを示します。



MyISAMテーブルを使用する場合は、my.cnf構成ファイルに次の行を追加する必要があります。 



 default-storage-engine=MyISAM default-tmp-storage-engine=MyISAM


MyISAMテーブルのみがある場合は、InnoDBエンジンを無効にして、my.cnfに1行だけ追加することでRAMを節約できます。 



 skip-innodb


過去にInnoDBを使用していた場合、以下ですべてのInnoDBテーブルをMyISAMに自動的に変換するスクリプトを提供します。 



 #!/bin/bash MYSQLCMD=mysql for db in `echo show databases | $MYSQLCMD | grep -v Database`; do for table in `echo show tables | $MYSQLCMD $db | grep -v Tables_in_`; do TABLE_TYPE=`echo show create table $table | $MYSQLCMD $db | sed -e's/.*ENGINE=\([[:alnum:]\]\+\)[[:space:]].*/\1/'|grep -v 'Create Table'` if [ $TABLE_TYPE = "InnoDB" ] ; then mysqldump $db $table > $db.$table.sql echo "ALTER TABLE $table ENGINE = MyISAM" | $MYSQLCMD $db fi done done


MySQLサーバーのパラメーターを最適化します


以下は、MySQLサーバーを高速化するために調整できるいくつかのパラメーターです。



キーバッファサイズ


これは、RAMの消費と最適化に必要なパフォーマンスに影響する最も重要なパラメーターの1つです。 MySQLはインデックス化されたすべてのものをキーバッファに配置しようとするため、このオプションは非常に高いパフォーマンスをもたらします。 SQLクエリはRAMから直接送信されます。 キーバッファに設定するサイズを言うことはできません。使用できるRAMの量がわかるのはあなただけだからです。



クエリキャッシュ


同じクエリを連続して2回実行すると、結果がクエリキャッシュに格納されるため、mysqlはクエリを再度実行する必要がなくなります。 パフォーマンスを改善する場合、このオプションは大きなメリットがありますが、メモリ消費が増加します。 したがって、このパラメーターは大きすぎず、小さすぎないように、つまりWebサイトに必要なだけ設定する必要があります。



以下は、クエリキャッシュの動作に影響を与える3つの変数です。 



 query_cache_size query_cache_limit query_cache_type




接続の最大数


これはオプションのパラメーターです。 すでにapacheプロセスの数を制限している場合、すべてがすでに問題ありません。 そうでなく、数千のユーザーを同時に処理する必要がある場合は、このパラメーターの値を増やす必要があります。



テーブルキャッシュ


テーブルにアクセスするたびに、MySQLはテーブル参照をテーブルキャッシュの1つのエントリとしてロードします。 これはテーブルへの並列アクセスごとに行われますが、メモリ使用量ではなく、パフォーマンスにとって非常に重要です。 テーブルキャッシュは常に増やすことができますが、オペレーティングシステムで開いているファイルの数が制限に達するため、そのことに注意してください。 テーブルキャッシュが小さすぎる場合、mysqlはあなたをukeしますが、これは望ましくありません。
