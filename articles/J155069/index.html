Dart向けの最初の安定したSDK

Dartプログラミング言語の開発者は、Dartの最初の誕生日と最初の安定したSDKのリリースという祝日を祝い ます 。 新しいSDKには高速のDart VMが付属しており、 Octaneの一部のテストではV8 Javascriptレンダリングエンジンよりも優れています。



SDK for Windows / Mac / Linuxとともに、DartからJavascriptへのより効率的なトランスレーター、 Dart Editorコードエディター 、 Dartサポート付きのChromiumアセンブリ （Dart VM付きのDartiumブラウザー）がリリースされました。 Googleは、将来、すべてのブラウザでネイティブのDartサポートが表示されることを期待しています。



上記に加えて、本日発表された：

  • すべてのブラウザで透過的に動作するHTMLライブラリ 。
  • Javascriptコードと対話するためのライブラリ。
  • 新しいパブパッケージマネージャー。
  • サーバーI / Oライブラリ 。
  • Dartの仕様を新しい機能で更新しました。


Dart構造ウェブプログラミング言語は、1年前に技術プレビューとしてリリースされました 。 Javascriptの代替/代替として位置付けられており、Javascriptの次のバージョンでは修正できないか、Typescriptのようなスーパーセットをリリースすることでは修正できない「基本的な」欠陥に悩まされています。 Dartは、Javascriptを開発または変換するいくつかの可能な方法の1つとしてGoogleが考えている革新的なオプションです。



Dartは、ライブラリ、デバッガ、コードエディタ、その他のツールを使用して、高性能で複雑なモジュラーWebアプリケーションを開発するのに適しています。



Dartは、3つの主要な目標を念頭に置いて設計されています。



  • パフォーマンス。 Dart仮想マシンには、すべてのEcmaScriptマシンが抱えるパフォーマンスの問題はありません。
  • 開発の利便性。 Javascriptの動的で学習しやすい、コンパイル不要の性質が保持されるため、Webプラットフォームはアマチュアプログラマの絶対的なリーダーになります。
  • コードエディターと追加ツールのサポート。 Dartは、リファクタリングや関数を呼び出す場所の検索などの機能を含む、サポートを必要とする大規模で深刻なプロジェクトで追加のツールを簡単に使用できるように設計されています。 同時に、Dartは効果的なプログラミングのためにコードエディターの必須使用を必要としないため、通常の開発者はテキストエディターでの作業に満足するでしょう。


また、Dartは、このタスクが3つの主要なタスクと競合しない最大限のセキュリティを提供する必要があります。
