イニシアチブについての何か。 継続

最近では、Skolkovoの悲劇的な（ 1、2、3 ）サイトを作り直してプロ版にするというアイデアがハブに登場しました。 この投稿では、私たちが今何をしたのか、何をしたのか、そして将来の私たちの計画は何なのかをお伝えしたいと思います。 そして、私は参加を希望する人々によってすでに提出されたアプリケーションの簡単な分析を行います。



どのようにすべてが始まりましたか？

すべては、estronas ユーザーのコメントから始まりました。
放り出すこともありますが、サイトを正常にしましょう。 Habra-immunityを備えたサーバーを設置すると、ニュースを入力して共有し、最も重要なことは誇りに思うでしょうか？ そして、結局のところ、一日後、イノベーションの中心が絞殺されました！ デアデビルズ）




そしてDGusevによる同様のコメント「「私は準備ができています」のスタイルで多くの答えを引き起こしました：
それは本当にサイトをよりモダンで思慮深いものにすることができます。 たとえば、ii-gorod.ruドメインに配置できますか？！ コミュニティには、わが国の開発者の本当のスキルを示すのに十分な愛国心があると思いますか？ これは、官僚からの深刻な反応でしょう。


翌日、彼らはzaislで私にestronasを書き、Google Docsのドキュメントへのアクセスを許可してくれました。 当初、このドキュメントは有用なツールでした。アイデアを表現して議論することができました。 しかし、時間の経過とともに、参加者の数は大幅に増加しました。 新興企業の支援から退役軍人の支援まで、多様で圧倒的なアイデアは、彼らの背景に反して、 若々しい最大主義についてのコメントがよく記憶されていました。 仕事のために独自のプラットフォームを作成し、独自のフォーラムを作成することを決定した人もいました。もちろん、すべて3日でした。



その結果、自己組織化ツールを使用する試みは失敗しました。 ブラックジャックと売春婦を使ってドキュメントを作成した人もいれば、情報ソースの数は5人に増えました。 停止する時間でした。



コラボレーションの問題解決

状況をどうにかして変えるために、私は自由に仕事へのアプローチをわずかに変えました。それはその後、他のコーディネーターによって承認されました。



最初に全員が自分と自分のアイデアをGoogleドキュメントに入力するだけで登録した場合は、 フォームから登録することをお勧めします。 彼は、イデオロギーの問題を解決できる多くの追加の質問を追加しました（たとえば、登録時に、ユーザーは大きなプロジェクトを行うかどうか、i-gorodをやり直したい場合、経費報告書を発行したい場合など）。



古い方法でのアイデアの議論を完全に放棄しなければなりませんでした。それらのバックアップのみが作成されました。 少し冷静にするために、一時的に話をやめました。 この時点で、ドメインhabrateam.comを登録してGoogle Appsをリンクし、 estronasでhabrateam.ruを登録しました。 次のように、国際版の作業を継続することにしました。 GoogleのDNSレコードを介してドメインを登録すると、ほぼ瞬時に作成および更新され、.ruゾーンでリダイレクトのみを1日待つだけです（ところで、DNSの設定にご協力いただきありがとうございます。ドメインのコントロールパネルはGoogleが必要とするTXTレコードを受け入れません）



私が観察した唯一の対立は、私と、私たちのドメインに関するフォーラムを執persistentに要求していた何人かの人々の間でした。 フォーラムはグループ開発ではなくコミュニケーションのために作成されたものであり、情報源の複製やインフラストラクチャの複雑化は何の役にも立たないと信じて、私は彼らのリクエストを断りました。



これで、コラボレーションに必要なすべてのツールが用意されました。
  • すべての参加者のリスト、タスクのビジョン、スキル、および彼らが私たちに提供できる時間。
  • ドキュメントを公開するためのGoogleドキュメント。
  • 部門（デザイナー、レイアウトデザイナー、コーディネーター）間のコラボレーションのためのGoogleグループ。
  • 投票および一般的な意思決定のためのGoogleモデレーター。
  • 開発者の活動を整理するためのGoogle Code Review（またはGithub）。
  • 権利を配布する便利な方法は、誰もが自分のアカウントとメールをドメインに持っていることです（以前は、権利は全員に順番に渡されていました）。
