Azureクラウドプラットフォームの新しいサービスの発表とパフォーマンスの向上

2014年12月11日に、Azureクラウドプラットフォーム の新機能が導入されました 。これにより 、開発者およびITプロフェッショナル向けの新しいシナリオが開かれ 、生産性がさらに向上し、クラウドソリューションの機能が向上します。







特に、Azureプラットフォームに次の革新が導入されました。



  • プレミアムストレージ 新しい高速クラウドストレージ。
  • RemoteAppリリース 。 任意のモバイルプラットフォームでデスクトップアプリケーションへのアクセスを整理するためのRemoteAppサービスの商用化。
  • SQLデータベースの更新 。 クラウドDBMSサービスの新機能。
  • 新しいメディアサービス機能 。 ライブビデオブロードキャストの組織の商業化ライブメディアストリーミング。
  • Azure Searchの更新 。 Azure Searchのクラウドベースのサービスとしての検索のメジャーアップデート。
  • DocumentDBの新機能 。 クラウド内のJSON向けに最適化されたNoSQLデータベースの新機能、Java開発者向けの新しいツール、
  • Site Recoveryリリース 。 中規模企業やオフィス向けの災害管理サービスを商業運転に導入。


すべての発表を順番に検討し、



プレミアムストレージ



以前に導入されたPremium Storageサービスは、公開テストで利用可能になりました（ 詳細な発表 ）。 Premium Storageは、Azure Storageデータを操作するときに新しいレベルのパフォーマンスを提供し、1つの仮想マシンで50,000 IOPS以上および最大32 TBのスペースを取得する機能を提供します。 同時に、SSDに基づいて構築されたPremium Storageは、1ミリ秒未満の読み取り操作の遅延を保証します。



仮想マシンに接続された各Premium Storageドライブには、次の機能があります。







複数のディスクをアレイに組み合わせて、スループットを向上させることができます。



Azure Storageの新しいパフォーマンスレベルにより、クラウドとオンプレミスサーバーの違いがあいまいになり、SQL Server、Dynamics AX、Dynamics CRM、Exchange Server、MySQL、Oracle Database、IBM DB2、SAP Business Suiteなどの重いソリューションをクラウド環境にデプロイできます。



こちらからPremium Storageのプレビューバージョンにサインアップできます 。 技術的な詳細はここで説明されています 。



RemoteAppリリース



Azure RemoteAppの商用運用のリリースについてはすでに書きました。 Azure RemoteAppは、RDPを介してAzureからWindows Serverベースのアプリケーションを提供します。



ユーザーは、ほぼどこでも、いつでも、さまざまなデバイスからアプリケーションにリモートでアクセスできます。 アプリケーションはAzureで一元的にホストされますが、エンドユーザーデバイスで実行されているかのように機能します。







Microsoft Officeアプリケーションと、クラウドで個別にダウンロードして事前構成できる他のアプリケーションの両方にすることができます。 ハイブリッドシナリオを構築する豊富な機能により、 複雑な分散ハイブリッドソリューションを構築し、ローカルインフラストラクチャ（一部はIaaS）にバックエンドサービスの一部を配置することもでき、アプリケーション配信自体はAzure RemoteAppに基づいて実行されます。



このサービスの主な利点は次のとおりです。



  • どこからでもWindowsアプリケーションを起動
  • ビジネスニーズの変化に迅速に対応
  • 機密企業アプリケーションの保護
  • クラウドおよびハイブリッド展開のサポート


開発チームのブログにアクセスして、サービスクライアントをプラットフォームにダウンロードすると 、サービスの説明ページで追加情報を見つけることができます。



それとは別に、Microsoftの戦略エキスパートであるAlexander ShapovalのAzure RemoteAppを使用して、いつでもどこでもアプリケーションを使用する新しい無料コースを学ぶことをお勧めします


Azure SQL Databaseの更新



Azure SQL Databaseは、重要な機能の更新を受け取りました。 プレビューでリリースされた次世代のクラウドデータベースサービスは、列インデックス、インメモリテクノロジー、CLRプログラマビリティ、T-SQLウィンドウ関数、XMLインデックス、変更追跡など、SQL Server 2014の機能を提供します。



さらに、Azure SQL Databaseを実行中のSQL Server 2014カーネルに移行すると、ソリューションのパフォーマンスが大幅に向上します。 6億行のデータを使用した内部テストでは、新しいSQL Databaseサービスのパフォーマンスが5倍向上しました。 そして、列インデックスとインメモリの機能を100回使用するとき！



Azure SQL Databaseの新しいバージョンの主な機能は次のとおりです。



  • 大規模データベースの管理の改善 。 並列クエリ、テーブルパーティション、オンラインインデックス、大きなインデックスの再構築（2 GBの制限が削除された）、およびその他のデータベース変更コマンドの機能が開発者に利用可能になりました。
  • よりプログラム可能な機能のサポート 。 開発者は、CLRプログラミング関数、T-SQL Windows関数、XMLインデックス、および変更追跡サポートにアクセスできます。
  • 生産性を100倍に向上させます。 インメモリテクノロジーとカラムインデックスのサポートのおかげで、ソリューションのパフォーマンスは最大100倍まで向上します。
  • 改善された検索およびトラブルシューティング機能と監視 。 拡張イベント（XEvent）のサポートと、データベース管理ビュー（DMV）の拡張セットを介した100を超える新しいテーブルビューへのアクセスが追加されました。
  • 新しい関税とS3パフォーマンスレベル 。 Azure SQL Databaseの新しい料金表とパフォーマンスレベル-100 DTUのパフォーマンスを備えたS3ソリューションを導入しました。 詳細

Azure Searchの更新



フルテキストインデックス作成とドキュメント検索をサービスとして提供する新しいAzure Searchサービスには、新しい機能と更新された管理機能があります。 まず、Azure管理ポータルには、Azure Searchインデックスを管理するための新しい機能があります。







ポータルの他の新機能には、インデックス内のドキュメントの数を監視し、インデックスに含まれるフィールドに関する情報を受信する機能が含まれます。 さらに、ポータルはスコアリングプロファイルを作成または変更する機能を追加し、検索結果をニーズに合わせて調整しました（以前はAPIを介してのみ使用可能でした）。







ロシア語を話す開発者にとって素晴らしいニュースは、 Stemingをサポートするロシア語を含む、 Azure Searchサービスによる27の新しい言語のサポートです。



Azure Searchタスクをより適切に管理するために、サービスの作成、スケーリングなどの管理タスクを実行する新しいAzure Search Management REST APIが導入されました。 このリンクで APIの使用例を見つけることができます。



新しいDocumentDB機能



最新のAzure DocumentDB NoSQLデータベースサービス（JSON用に最適化され、SSDで実行）は進化を続け、ポータルを通じて新しい管理機能を提供します。



  • Document Explorerを介してJSONドキュメントを追加する
  • DocumentDB SQLクエリ構文の強調表示。






新機能により、Azure管理ポータルからサービスを簡単に操作できます。 フォーラムを介してサービスの開発者と通信できます。 こちらで Azure DocumentDBサービスを試すことができます 。



DocumentDB用のJava SDK



もう1つの良いニュースは、Java開発者向けの個別のツールキットのリリースです。 Java用のDocumentDB SDKの助けを借りて、開発者はDocumentDBデータベースサポートをJavaプロジェクトに簡単に追加できます。 SDKライブラリはオープンソースです。



Azure Site Recoveryリリース



Azure Site Recoveryサービスは 、展開されたソリューションに耐障害性を提供​​するように設計されており、特にオフィスおよび中小企業向けに設計された新しい機能を導入しました。



新しい機能により、ユーザーはMicrosoft Azureで仮想マシンを複製、保護、および復元できるようになりました。 この更新により、ユーザーはWindows Server 2012 R2とMicrosoft Azureの間の回復タスクにSystem Center Virtual Machine Managerを使用する必要がなくなります。



ユーザーにはいくつかの復旧モードがあります。保護の要件に対してインフラストラクチャをテストするためのフェールオーバーのテスト。 すべてのデータと仮想マシンをAzureクラウドに計画的に移行するための計画的なフェールオーバー、およびローカルデータセンターで障害が発生した後にAzureのインフラストラクチャを復元するための計画外のフェールオーバー。
