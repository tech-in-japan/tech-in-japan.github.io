約束どおり、UMI.CMSパートナーの議論、観察、経験に基づいて書かれた記事の第2部を公開しています。 リスクを最小限に抑え、インターネットプロジェクトの開発の各段階でプロセスを高速化するために、Webスタジオの所有者の実践的な推奨事項とコツをまとめて共有することは興味深いことでした。



最初の部分では、顧客を見つけて契約を締結する段階で「チップ」について話しました。 今日-デザイン、実際の開発（実装）、およびプロジェクトの実施について。



3.デザインの開発と承認



顧客がプロジェクトを受け入れるという点では、多くの場合、設計がボトルネックになります。 顧客が最も積極的な役割を果たし、推奨事項と提案を提供し、さまざまなオプションを提供するように依頼するのは、設計です。 多くの場合、設計の承認は長期の追加につながる可能性があります。 スタジオにとって不利な仕事。 ほとんどの場合、顧客にとっても不利です。



3.1。 個人的なプレゼンテーション

初めて直接デザインを提示することは非常に重要です。 最初の設計オプションを電子メールで送信することはできません。 顧客が別の都市にいて、個人的な会議が不可能な場合は、電話での同意と同時にレイアウトを転送できます。 これはまず、作業を正当化および正当化できるようにするために必要です。 顧客にとって-これも便利です。 実際、ウェブスタジオにとって、デザインは彼女がプロである専門分野の一部です。 Webスタジオは数十、数百のサイトを作成し、顧客はユニットを作成します。そのため、ほとんどの場合、Webスタジオは顧客にとって本当に有益な高品質の製品を提供します。 しかし、彼の作品には、彼の製品の説明を含めるべきです。 準備が整っていないユーザーには明らかなほど複雑です。



ここでの主なポイントは、顧客が自分でデザインの第一印象を与える必要がないことです。 著者の助けを借りてコンパイルする必要があります。



3.2。 顧客の関与

顧客は、次のように設計開発プロセスに参加する必要があります。 彼にとっても、それは創造的なプロセスであり、彼はそれに参加する権利を持っています。 あなたはさらに言うことができます-彼が支払うお金の一部、彼は心理的に彼の創造的な参加のために与えます。 この場合の開発者のタスクは、プロの観点からクライアントのエネルギーを正しい方向に向けることです。



「特別オファーの星の色に注意してください。 私たちにはいくつかの選択肢があります。あなたの参加がなければ、どちらが良いかを決めることはできません。



Webprofy Web Design Studioからの練習

プロジェクトへの顧客の参加は、インターネットプロジェクトの品質を損なうことなく、開発者の損失または統合利益の減少につながらない限り、論理的かつ公正です。 設計を受け入れる際の無限で魅力的なプロセスを止めることは不可能です。 この場合、顧客と開発者の両方が苦しみます。

この段階に終止符を打つことは、会社のレターヘッドに作成された「最終コメントのリスト」に役立ちます。 それは、顧客が設計に対して行うよう要求するすべてのコメントを記述し、それらは彼の署名によってサポートされています。 これらのコメントを満たすために、この文書が契約の付属書として作成されていない場合でも、原則として、顧客も請負業者も設計作業を継続する理由はありません。



4.レイアウト、CMSの実装、プログラミング





4.1。 箱入りCMSの使用

キネティカ社 、バルナウル、遺伝子の実務経験。 アレクセイ・サモイロフ監督：

「ボックス化されたCMSへの移行は、非常に実用的であり、ビジネスプロセスの観点から検討しています。 スタジオの所有者として、私たちは何も失いませんでした。 実際、CMSをインストールするたびにプログラマーに給与を支払い、今では相応のお金がアウトソーシングに使われるようになりました。 言い換えれば、今ではYumisoftプログラマーに給与を支払っていますが、同時にメンテナンス、管理などの費用は一切負担していません。



一方、プログラマーにとっては有益です。 まず、スタジオCMSを開発する必要がなくなりました。新しいモジュールの機能、使いやすさのテスト、ユーザードキュメントなどを記述するための異常な作業が大量に行われているため、特に喜ばれたことはありません。 第二に、最も価値のあるものは、現在販売段階では、「これを実現でき、どのくらい時間がかかるのか」などの質問でマネージャーから引き出されません。 標準のUMI.CMSモジュールはセールスマネージャーに理解できる制限を設定しますが、一方で、クライアント自身はそのような厳密に記述された機能に忠実です-「ボックスマン」のマーケティングポリシーが役立ちます。



ボックス化されたCMSへの移行後、開発者の給与基金は減少しておらず、なお成長トレンドを維持していることは注目に値します。 売上だけでなく、プロジェクト予算の売上も増加しました。今では、企業のWebサイトを2〜3倍高速化できます。 これはどうしてですか？



状況1：

私は自分のCMSで作業し、プログラミングとレイアウトの段階で1000ドルかかります。 たとえば、給与計算基金（プログラマーから管理職まで）の60％は、つまり 私は、ビジネスマンとして、「商品を出荷する」とすぐに400ドルを獲得します。 このステージをアクトで締めくくります。 この場合の「レジから倉庫への道」は非常に長く、プロジェクトに応じて1週間から1か月です。 つまり 危険なお金を1か月間流通させています。



状況2：

私は箱入りのCMSで作業し、プログラミング段階で1000ドルを請求します。 CMSによる割引+開発コストの削減（以前に「標準」の企業サイトが5日間作成された場合、現在は3）400ドルの利益が得られますが、今では少なくとも半分の金額で商品を「出荷」できます-システムを販売できますすぐに行動する！ 良好なフローの場合、私たちは事業の売上高に関してかなりの金額について話している。



そして、状況2はクライアントにとってもより有益です-私自身のCMSを受け取り、行為に署名したので、彼は私からその上でプログラミングを購入せざるを得ません。 彼には他の選択肢はありません。 つまり 彼が受け取る製品はある程度「仮想」です。 しかし、彼が「箱」を受け取ったとき、彼は本当の商品を受け取ります-結局、彼はそれをさらなる仕事のために何百人もの開発者に移すことができます。」



4.2。 主なタイプのインターネットプロジェクト用の標準テンプレートの作成

インターネット代理店Gekos 、CEO Artyom Gerashchenkovの経験。

スタジオが専門とするいくつかのタイプのプロジェクトを収集します。たとえば、サービス部門で働いている会社の名刺サイト、商品カタログのあるサイトなどです。 これらのテンプレートサイトにデザインの観点からの構造、コンテンツの観点からの構造を与え、特定のページ数に典型的な情報を入力します。 このようなテンプレートサイトは、顧客向けに作成された実際のサイトに基づいて作成することができます。

それは何を与えます：

-注文を受けた翌日には、顧客に最初のプロトタイプを見せることができます。

-既製のサイトテンプレートにより、サイト作成のコストを少なくとも50％削減できます。

-ウェブサイト開発が流れます-支出面を維持しながら生産と売上高を増加（生産プロセスは単純化と標準化により加速されます）=収益性の成長。



このプラクティスは、低予算および中予算および大規模なボリュームに関連します。 明らかに、大規模な顧客、高い予算、および個々のインターネットプロジェクトで作業する場合、適用されません。



4.3。 プロジェクトのチームワークにシステムを使用する（タスク、プロジェクト、コラボレーション管理）

原則として、企業にゼロからこのようなシステムを導入すると、従業員は強く拒否され、このツールは無視されます。 ほとんどの場合、従業員の1人に対する示唆的な罰則がこの問題の解決に役立ち、その後、残りはツールの使用を開始します。 原則として、その後、彼らはすでにプロジェクトの内部コミュニケーションの有効性の増加を評価することができ、将来、彼ら自身が会社の新しい従業員の使用の開始者になります。

5.プロジェクトの配信



Webprofy Web Design Studioからの練習

Webにサイトを配置することは、プロジェクトの全額支払いと行為の署名後にのみ行われます。

すべての利点は明らかであり、コメントする必要はないと思います。



匿名のままにしたい会社からの練習

プロジェクトは、ウェブ上に投稿でき、全額の支払いと行為の署名ができるまで続きます。 逆説的に、顧客がプロジェクトを受け入れたくない場合は有益です。 彼は正式に引き渡されておらず、受け入れられていないという事実にもかかわらず、彼はすでにオンラインであり、顧客担当者は彼と協力し始めています。 ある段階で、開発者がプロ​​ジェクトの受け入れと支払いを拒否された理由がわからなくなったとき、彼は顧客にプロジェクトをウェブから「テイクアウト」するよう提案します。 彼は受け入れられません。 そして、開発者は絶対に正しいです-行為が署名されておらず、作品が全額支払われていない場合、製品はありません。 ほとんどの場合、このようなイベントの展開により、顧客は最終的に適切な改善に同意し、その後プロジェクトが完了するか、「現状のまま」（明確に主張を定式化できない場合）を受け入れます。 最終的に、いずれにせよ、彼は彼を満足させるプロジェクトを受け取ります。



ゲコスプラクティス

開発者の主なタスクは、プロジェクト完了後も顧客を維持し、さらなるサポート（広告、SEO、技術サポート、コピーライティング、ユーザビリティ監査など）のためのさまざまなサービスを提供することです。 これを行うには、他のタイプのビジネスに従来の割引システムを使用できます。 たとえば、サイトが会社で開発された場合、広告主がサイトの開発の割引を提供するため、将来的に広告の割引を提供できます。 複数のサービスを使用したお客様には、「卸売」割引を提供できます。



そして最後に、これらのヒントを文字どおりに受け取らないように注意してください。 特定の会社に基づいて、状況を使用するかどうかを決定する必要があります。 しかし、いずれにせよ、そのようなテクニックのアイデアを持っていることは誰も傷つけないと思います。 そしてもちろん、あなたもあなたのプロの「秘密」をコメントで共有すれば素晴らしいでしょう:)
