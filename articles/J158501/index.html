クラウド時代のITの役割を再考する

遅かれ早かれ、IT部門はクラウドテクノロジーの導入に関連して作業スタイルをわずかに変更する必要があります。 いいえ、これは革命的な革命ではありません。すべてのプロセスを制御下に置き、側で制御されるものの最も合理的な使用に切り替えることをやめる必要があります。



私は、インターネット技術と、クラウド、API開発、および成長を続ける相互接続性と相互依存性の時代でそれらが果たす役割について多くの時間を費やしました。 アプリケーションは徐々に中心になり、サーバーはバックグラウンドにフェードインしています。 そして、これらすべてにおいてITはどのような役割を果たしますか？ ITをクラウドに移行するとはどういう意味ですか？



Nick Carrは、著書「Does IT Matter？」で、ITがビジネスに差別化の特定の欠如をもたらす方法について説明しています。 彼の主なポイントは、IT業界がますます予測可能になり、ビジネス製品がますます類似しているということです。 もしそうなら、なぜ各ビジネスには独自のソフトウェア製品があるのですか？



Carrの見解に同意する場合、クラウドテクノロジーへの移行は、IT業界のイベントの最も論理的かつ合理的な開発です。 The Big Switchの本で、Carrは別の興味深い考えをもたらしました。技術の標準化が進むと、人類にとって最も必要なリソースの1つ（水と電気とともに）の状態に近づくと彼は言いました。 それは本当に理にかなっており、今日のクラウド市場で何が起こっているのかを理解するのに役立ちます。



この場合、IT部門は時間の経過とともに完全に消滅し、データ分析および処理サービスを提供するサードパーティのインターネットサービスに取って代わることができますか？ いいえ、そうは思いません。



多かれ少なかれ大規模な組織では、ヘルプデスクと機器管理を提供するだけでなく、ITがはるかに重要な役割を果たすと思います。 この役割を理解するには、組織で作成および使用されているアプリケーションを注意深く調べるだけでなく、ITが現在行っていることは、タスクがクラウド指向の構築である場合とはまったく異なることも理解する必要があります組織モデル。



古き良きIT



もちろん、各組織では独自の方法で作業しているため、一般的にITについて話すことはかなり困難です。 しかし、いくつかの基本的な傾向を強調することはまだ可能です。 最も重要な傾向の1つは、私が「サーバーの中心的役割」、より正確には「インフラストラクチャの中心的役割」と呼ぶものです。



まず、コンピューター技術の時代がどのように始まったかを思い出しましょう。 まず、少なくとも何かをするためには、コンピューターが必要でした。 オペレーティングシステムをコンピューターにインストールする必要があります。これは、実際には、ユーザーがマシンと対話できるようにするインターフェイスです。 また、OSがインストールされたコンピューターがある場合にのみ、特定の問題を解決するためにアプリケーションの助けを借りることができます。



コンピューターは常にこのモデルの不可欠な部分でした-残りの部分は鉄なしでは機能しませんでした。

したがって、企業は、自社のデータセンター、オペレーティングシステム、アプリケーションの提供、データベースなどに多額の費用を費やす必要がありました。 これはすべて、主要なビジネスプロセスを確保するために行われました。これがITの役割でした。



ITおよびクラウドテクノロジー



開発者は、コンピューター技術を使用してビジネス上の問題を最終的に解決する人です。 明らかな理由で、彼らはコンピューター技術とソフトウェアの非常に限られた能力に常に動揺し、インフラストラクチャのサポートは常に困難な作業であり、印象的な時間と材料費を必要としました。



しかし、今では、ゲームのルールさえも変更されていません-ゲーム自体が大幅に変更されています。 一般的なインフラストラクチャは、今では月額わずかな料金で誰でも利用できます。 さらに、インターネット、オープンソースプロジェクト、その他のクラウドを通じて利用できる多くの革新的なソフトウェアツールとサービスを追加すると、クラウドソリューションがIT部門のスタッフにとって、独自のベースで開発するよりも、ある種のアプリケーションを開発する方がはるかに便利である理由が明らかになりますインフラストラクチャ。





Amazon Web Services仮想サーバーを提供します。



開発者は、運用について考えるとき、使用されるインフラストラクチャよりもアプリケーション自体に焦点を合わせます。



それで、それは何をしますか、ITは多くの組織の作業サイクルから徐々に移動していますか？ 「公式に」ではなく、非常に静かですが、それでも、さまざまな分野で活動するさまざまな企業でこれが起こります。 一方で、これはクラウドによって提供される2つの主要なソフトウェア（スケーラブルなWebアプリケーションとデータ収集分析）で発生し、他の多くの領域は単にITの渇望で死にかけています。



IT部門は何をしますか？



この質問に対する答えは、「アプリケーションの中心的役割」が実際にビジネスにとって何を意味するのかを明確に理解することだと思います。 1つのアプリケーションのみを使用するビジネスはありません。 また、特定のアクティビティを1つだけ実行するビジネスはありません。 各企業は、相互に依存する多くのソフトウェアコンポーネント、サービス、およびデータで構成されるアプリケーションシステム全体を使用しています。これらは、企業の安定した発展を確保するために特定の方法で連携する必要があります。



クラウドテクノロジーの時代におけるIT部門の本当の目的は、アプリケーションシステムを調整することです。組織の内部インフラストラクチャ内でクラウドサービスの作業を調整することです。



以下は、新しいアプリケーションの注文に関連してIT業界が答える必要がある重要な質問です。

  1. 企業が識別、承認、認証、データ処理システム、およびその他の最も重要なセキュリティ機能を制御する方法。その機能はさまざまな制御システムを通じて調整されます。
  2. アプリケーションが制御システムを介してやり取りする場合、またはチームに期限を設定する場合に発生する問題を企業が解決する方法。
  3. 既存のシステムのパフォーマンスを改善できる、個々のアプリケーションに依存しないソリューションがありますか。
  4. アイデアやコンポーネントを新しいアプリケーションに統合する方法について必要な推奨事項を提示するのに十分なシステムの知識を持っているのは誰ですか？




したがって、ITの主なタスクは、インフラストラクチャ管理からソフトウェア管理に移行することです。より正確には、開発者が大規模なソフトウェアシステムを管理できるようにすることです。 この「助言」機能には、サポートされている開発者に関連する必要がある特定のツールとサービスのセットが含まれています。 これは、たとえば、アプリケーションとそれらが消費するリソースの関係を視覚化するツール、または管理リテラシーを開発するサービスです。



主なアイデアは、ITがすべてのプロセスを制御下に置き、外部で制御されるものの最も合理的な使用に切り替える試みを放棄する必要があるということです。 本当に可能です。 これは、NetfixとAmazonが今日行っていることです。 その結果、協力、コミュニケーション、測定が成功の重要な要素になりつつあるため、柔軟性と革新性のレベルが大幅に向上しています。



最新のIT組織の最も重要な欠点は、エンタープライズソフトウェアに関しては、集中化された、または少なくとも暗黙的に集中化されたインフラストラクチャです。 開発分野における集中型アプリケーションとセルフサービスへの移行は非常に複雑なプロセスであり、企業の企業文化とスタッフトレーニングのレベルの両方を変更する必要があります。 つまり、プライベートクラウドサービスは、IT業界が取った最も重要なクラウドイニシアチブではありません。



ほとんどのIT組織がこれをすぐに理解するだろうか。
