Hyper-V 2008 R2ニークラスター

こんにちは、habracheloveki。



Hyper-Vの最新の実装を備えたWindows Server 2008 R2の最近のリリースは、何らかの形で誰もが知っていると思います。 通常の2008 Hyper-Vと比較した主な革新は、クラスターノード間で仮想マシンをリアルタイムで転送するためのライブマイグレーション機能のサポートです。 もちろん、この機能を実証するためには、外部データウェアハウスとクラスターが必要です。そして、パントリー内の余分なSANは通常は存在しません。







この記事では、手元のハードウェアに本格的なHyper-V 2008 R2クラスターを実装する方法について説明します。2つのハードウェア仮想化をサポートする3つのワークステーションです。 カットシーンの下で、Hyper-Vのフェールオーバークラスターを組み立てる手順。ニュアンスと機能を考慮し、ライブマイグレーションテクノロジーのビデオデモを行います。





はじめに









ライブ移行テクノロジー自体の簡単なツアー。 クイック移行との主な違いは、メモリをディスクにアンロードせずに仮想マシンをオンザフライで転送することです（これは前の図で示されています）。 さらに、Microsoftによると、マシンの転送はTCPセッションが中断する時間よりも高速であるため、ユーザーが不快感を感じる場合はわずかです。



ハードウェアとソフトウェアの選択



もちろん、テスト用の追加のSANはありません。

私が持っているのは、2台のAquariusワークステーション（C2D 6420、4Gb DDR2、80Gb HDD）とCeleron-D上の1台のステーションです。

もちろん、2008 R2は64ビット版にのみ存在するため、すべてのプロセッサが64ビットをサポートする必要があります。







クラスターを実装するには、Hyper-V Server 2008 R2の無料バージョンを使用します。これは、カーネルモードの修正されたWindows Server 2008 R2で、ハイパーバイザーはデフォルトでオンになっており、集中管理のための比較的便利なコンソールメニューです。 この選択は、ハイパーバイザー以外の役割のサポートにホストマシンのリソースを費やしたくないという事実によるものです。







このハイパーバイザーを使用する場合は、仮想マシンのみにライセンスが付与され、有効にします。 これは、誰かがこの職人技のシステムを生産に導入することを考えているかもしれない場合に備えて言っています。



ストレージとして、私は最も弱いワークステーションを使用し、それぞれに250GBのハードドライブを2台ずつ投入し、オペレーティングシステムを使用してそれらをソフトウェアRAID0に結合します。 即興サーバーをリポジトリーに変換するには、最も安価なオプションiSCSIを使用します。 同じマシンがHyper-Vクラスターのマネージャーとして機能し、同時に小さなドメインのコントローラーになります。



WindowsにはいくつかのiSCSI実装があります。 最も一般的な2つは次のとおりです。

  1. Microsoft iSCSI Software Target -Microsoft OEMパートナーからサーバーを購入する場合、Windows Storage Server 2008にのみ同梱されます。 パブリックドメインでそれを見つけるのはそれほど簡単ではありませんが、誰かがそれを見つけた場合は、試すことができます。 真実は、Storage Serverのみに当てはまりますが、これは私たちのニーズにはあまり適していません。
  2. StarWind iSCSI Software Targetは、ロシア語をサポートするStarWind社（Habréで完全に人気のないブログを持っています ）の製品です。 StarWindには無料バージョンがあり 、ストレージは2TBに制限され、1つのターゲットに対して2つの競合接続があります。 そして、もう確認する必要はありません。




緊急ストレージは本格的なWindows Server 2008 R2で実行されますが、この場合のバージョンは重要ではありませんが、32ビットシステムからクラスターを管理すると問題が発生することが実験的にわかっています。 さらに、2008 R2には新しいフェールオーバークラスタリングコンソールがあり、他のバージョンではリモート管理ツールを更新する必要があります。 マシンがドメインコントローラーの機能も実行することを忘れないでください。



カスタマイズ





ステージ1。 予備


したがって、2つのクリーンで未構成のHyper-V Server 2008 R2と1つのクリーンなWindows Server 2008 R2があります。



クラスターを構成する前に、適切に動作するようにネットワークインフラストラクチャを構成する必要があります。 小さなスイッチ（私の写真のような）でそれらを選択した後、すべてのサーバーをローカルIPアドレスに設定します。 RDPとMMCコンソールの両方を介して、ノードへのリモートアクセスを許可します。



ノードがドメインにある前にこれを行うことをお勧めします。 サーバーがファイアウォールポリシーの変更が不可能であることを呪った状況がありました。 ドメインを上げて、将来のクラスターノードに入ります。



もちろん、 フェールオーバークラスタリング機能をオンにします。



ステージ2。 リポジトリ。




これで、ストレージ構成に直接移動できます。 まず、結果のストライプ（RAID0）を2つの論理ドライブに分割します。1つはクォーラム用（現在はFile Share Witnessと呼ばれます ）、もう1つは仮想マシンを格納するためです。 クォーラムに約15GBを割り当て、その他はすべてデータ用に残しました。 良い意味で-クォーラムの下ではギガバイトで十分ですが、何を知っているかはわかりません。







StarWind iSCSI Targetのインストールに進みます。 メーカーのウェブサイトからダウンロードするには 、登録が必要ですが、これは原則として義務ではありません。 登録後、インストーラーとライセンスキーファイルの直接ダウンロードへのリンクが電子メールに表示されます。 インストールは[次へ]ボタンを数回クリックするだけなので、基本的に複雑なことはありません。 インストール後、少なくともポート3260および3261がWindowsファイアウォールを通過する必要があります。 怠zyな人のために-あなたはそれをオフにすることができます、なぜなら私たちの即興ネットワークには私たちのサーバーが3つしかないからです。







StarWindコンソールを起動し、iSCSIサーバーに接続しようとします。iSCSIサーバーは、謎のアドレス127.0.0.1のコンピューターです。 認証には、デフォルトのログインとパスワードの秘密の組み合わせが必要です。これらは、基本バージョンではそれぞれrootとstarwindのように見えます。 インストールプロセス中にパスワードの設定を求められなかったのはなぜですか、またはインターフェイスにヒントを付けなかったのはなぜですか？







サーバーに接続した後、電子メールで落ちたライセンスファイルを彼に渡す必要があります。 これは、StarWindサーバーコンソールの[構成]タブで行います。 右上隅にあるファイルをダウンロードするリンクを除き、すべてが直感的でシンプルです。 登録後のみ、プログラムによりターゲットを作成できます。







ターゲットを作成するプロセスでは、LUNの役割を果たす事前にタグ付けされたimgファイルを作成します。 目標に従ってセクションにそれらを配置し、競合するiSCSI接続（つまり、クラスタリング）を担当するボックスをチェックすることを忘れないでください。 データキャッシュを設定することもできます。低速のアイロンを使用すると便利です。







その結果、およそ次の図が判明するはずです。 2つのターゲット-1つは定足数用、2つ目はデータ用。 それらをサーバーに接続します。



ステージ3。 リポジトリにしがみついています。




バージョン2008 R2のサーバーには、iSCSIイニシエーターが事前にインストールされています。iSCSIイニシエーターは、即興ストレージからのLUNをローカルディスクとして接続します。 確かに、コンソールHyper-Vサーバーに適切なGUIがないため、コマンドラインまたはPowershellからiscsicplコマンドを使用して呼び出す必要があります。







黄色で強調表示されたフィールドに、ストレージのIPアドレスを入力する必要があります。その後、サーバーに配置されたターゲットを含むポップアップウィンドウが表示されます。 もちろん、ファイアウォールが正しく構成または無効にされている場合。 ターゲットがサーバーに接続されたら、ディスクとしてマウントを開始できます。







[自動設定]をクリックすると、\？\で始まるディスクへの2つの長いパスが空のウィンドウに表示されます。 その後、ディスクの管理に安全に移動して、ディスクに文字を割り当てることができます。 ところで、最初のハイパーバイザーに文字を設定すると、2番目のハイパーバイザーがドライブ名を自動的に選択します。



その結果、iSCSIを介してストレージに接続された2つの共有ディスクを持つ2つのハイパーバイザーを取得しました。 クラスタリングに直接行くときです。



ステージ4。 クラスタリング




2つの既存のハイパーバイザーでフェールオーバークラスターを構成する場合、ウィザード自体がクォーラムとデータ用のファイルパーティションを選択しました。 アルゴリズムを正しく理解していれば、iSCSIドライブが共有リソースとして選択され、最小のものがクォーラムとして選択されました。 または、クォーラムラベルを小さいセクションに付けたので、彼がそれらを選択したのかもしれません... =）







ウィザードとそのようなマザーを使用して2つのハイパーバイザーをクラスターに結合すると、フェールオーバークラスターのコンソールは上の図のようになります。 現時点では、1つのノードのみがすべての作業を実行するフォールトトレラントクラスターがあり、2番目のノードはこの時点でアイドル状態であり、最初のノードが停止するまで待機することに注意してください。 しかし、これは必要ありません。







ここで、クラスターの共有ボリュームが役立ちます。クラスターコンソールの新しいタブは、2008 R2でのみ登場しました。 フォールトトレランスを提供するために仮想マシンを配置できるのは、このストレージ上です。 私たちのタスクは、魔法の[ストレージの追加]ボタンを使用して、通常のクラスターリソースを共有リソースに変えることです。 このタイプのリソースがバージョン2008 R2以降でのみ機能することを警告すると、ディスクが[共有リソース]タブに表示されます。 ディスクはC：\ ClusterStorage \ Volume1にマウントされ、すべての共有データがそこに保存されます。 仮想マシンを含む。 不要な不具合を避けるために、各ノードのハイパーバイザーの設定では、共有ディスク上の場所を使用して、ディスクと仮想マシンのデフォルトパスを設定することをお勧めします。







ステージ5。 仮想マシン。




これで、クラスター管理スナップインから仮想マシンを直接作成できます。 [サービス]タブで、[仮想マシン]を選択し、ノードを選択してマシンを作成します。 どこかから配布キットをマウントし、インストールしてアンマウントします。 ライブマイグレーションを正しく動作させるには、各ノードの外部リソースへの依存関係がないことが必要です。







ビデオは、あるノードから別のノードにメモリを蒸留するプロセスを示しています。 ポータブル仮想マシンには同じWindows Server 2008 R2があり、マシンのメモリは1024mbでした。 すぐに移行の非常に平凡な速度に注目する価値があります。これは、ストレージと100 MBのネットワークの職人による実装に関連しています。 後で、ギガビットネットワークを使用してクラスターを再構築し、パフォーマンスの向上を確認します。
