結論：

  • COMポート経由で何かを送信する必要がある場合は、それを行ってください！ しかし、我慢してください。
  • WinXPでファイルを投げた場合、FloppikはCOMポートよりも高速です（純粋なDOSでは、転送速度が速くなる場合があります...）。
  • LPT経由でデータを転送する場合-Fastwireを強くお勧めします。時間を節約することは価値があります。
  • これら2つのオプションが選択できる場合は、LPTを選択します（上限がありました）。
  • Windows XPでは、DOSモードは完全にエミュレートされていないため、LPTを使用しても速度は悲しくなります。 おそらく何かを設定する必要があります-それを理解する必要があります。
  • 焼き切れたポートは悲しいです:(

  • File MavenとINTERLNKの問題はラップトップにあります。 DOSで起動したときにディスクが表示されない別のユーティリティに出会いましたが、Windowsで起動するとディスクは使用可能です。 これらのユーティリティのテストを繰り返す必要があります。
  • 適切なネットワークカードが見つかりましたが、これまでは構成できませんでした。 Windowsでは、十分な割り込みがありません（IRQ 29 o_Oを要求します）。また、ドライバーが手動で選択されていない場合、サイトドライバーがハードウェアと一致しないことを報告します。DOSでは、ドライバーがカードの存在を判断しますが、設定を取得できません カードの名前はTrendnet TE100-PCBUSRです。 ラップトップBIOSにはほとんど設定がありません。Windows/ DOS用のIBMの構成ユーティリティでは、重要なものを変更することもできません。 問題の分析が続行されます。
  • アイデアは、電話回線なしでモデムとモデムを接続し、電話の麺を直接貼り付けるという発想でした-私が考えて、それがどのように機能するか、ソフトウェアとハ​​ードウェアがどのように動作するか、構成機能と一般的なそのようなソリューションの妥当性を確認するまで。
