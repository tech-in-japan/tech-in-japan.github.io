ワンクリックですべて

ある時点で、時間管理（および同時に退屈）のアイデアに触発され、余分な時間を無駄にしないように日常のタスクを最適化するのが好きであることに気付きました。 当然、1か月を費やして5分かかるタスクを自動化し、1年に1回実行するのは意味がありません。 それでも、私たちが毎日何度も実行するタスクがあり、それらはより速く簡単に完了することができます（それぞれ、より必要なもののために時間を解放します）。 カットの下は私がそれをする方法です





何かを最適化する前に、私は理解しようとします：

1）タスクが十分に定期的に繰り返されるかどうか

2）実装に多くの時間を費やすかどうか

3）このタスクを最適化するのは簡単ですか？



3つの質問すべてに対する答えが「はい」の場合、「カットではなくシャープにする」、つまり最適化する時間があります。



あなたは言う、これはすべて理論であり、あなたは正しいでしょう。 私はあなたに同意し、私たちは練習に移ります。 私の経験から小さな例を説明し、説明したタスクをどのように最適化したかを説明します。 単純な例から複雑な例に移行します



例1.接続

私の場所でインターネットにアクセスするには、VPNを起動する必要があります。 システムに応じて、アクションはわずかに異なりますが、基本的にこれは（Windowsの場合）メニューから選択されます。

「接続」

利用可能な接続のリストから選択します

ユーザー名とパスワードのダイアログで、[接続]をクリックします（以前にデータが入力されていた場合）



ほぼすべてのユーザーがこれを行ったことはありますが、ほとんどのユーザーはこれまでどおりです。 Windows 7の場合、ツールバーから接続を開始できますが、原則は変わりません。

解決策1

a）接続の選択は非常に便利な操作ではないため、クイック起動パネルへのショートカットとして転送し、目立つ場所にドラッグしました

b）接続のプロパティで、「ユーザー名とパスワードを要求する」オプションを削除しました

これで、タスクはワンクリックで完了しました

私にとっては、多くの場合、就業日の80％に使用する一連のプログラムがあります（もちろん、合計で:)）。

これはMS Outlookです

b）すべてのプログラムに「ショートカット」を割り当てた。 Wordの場合、これは「Ctrl + Alt + W」、Excel-「Ctrl + Alt + E」です。 シリーズを続けるのはとても簡単だと思います...



例3.データのアーカイブ

あなたが持っている仕事に関連するすべてのものがバージョン管理システムに保存されることを願っています。 作業中のプロジェクトは、最低限、私の場所に保存されています。 ただし、失いたくない小さな何かに単独で取り組むこともあります。また、これの異なるバージョンへのアクセスも良いでしょう。 ある時点で、Archiveフォルダーを作成し、そこでアプリケーションのさまざまなバージョンをスタックし始めました。 プロセスはスキニーではなく、日常的なものであり、パパを作成してそこに新しいバージョンを配置する必要があることを理解しています。

実際、ワンクリックでこれを行うバッチファイルを作成しました。

ソースフォルダーの内容を取得し、新しいバージョンのサブフォルダーを作成し、そこにソースをコピーします。 すべてが自動的に機能し、ワンクリックで起動します。 ある時点で、ハードドライブ上の場所の質問がアーカイブの追加を強制し、スクリプトは以下のようになりました



SET SOURCE_DIR="D:\tests\ReportBrowser\"

SET APP_NAME=ReportBrowser

SET ARCHIVE_PATH=D:\Archive\



SET ARCHIVATOR="d:\Program Files\7-Zip\7z.exe"

SET PARAMS=a -r

SET EXT=.7z

@ECHO ON



FOR /L %%X IN (300,-1,1) DO IF NOT EXIST %ARCHIVE_PATH%%APP_NAME%\%%X SET PATH_DIR=%ARCHIVE_PATH%%APP_NAME%\%%X

MKDIR %PATH_DIR%\

%ARCHIVATOR% %PARAMS% %PATH_DIR%\%APP_NAME%%EXT% %SOURCE_DIR%*.*

REM XCOPY %SOURCE_DIR%*.* %PATH_DIR%\ /i /f /r /k /y /s











新しいスクリプトを作成するには、前のスクリプトを新しい名前でコピーし、最初の2行の変数を編集します。 そして、クイック起動バーにタグを掛けます。 スクリプトがほとんど使用されなくなった場合、クイック起動から削除します



バックアップスクリプトを含むソースポストは、英語のSimpleScript for Backupにあります。
