なぜあなたの好きなメッセンジャーは死ななければならないのですか



インスタントメッセンジャーの墓地。Skype、Viber、WhatsApp、ハングアウト、ooVoo、Apple iMessage、Telegram、Line、Facebookのメッセンジャー、および近い将来リリースされる予定の数百のインスタントメッセンジャーが必ず含まれます。



インターネット通信の分野の恐ろしい状況、インスタントメッセージング、オーディオビデオコール、およびどのメッセンジャーがまだ適切で正しく安全であるかについての迷惑な議論のツールを開発するという脅迫的な見通しにより、私はこのテキストを書くよう促されました。



近年、メッセンジャー市場での競争はかつてないほど大きくなっています。 スマートフォンで誰もがアクセスできるインターネットにより、メッセンジャーは最も頻繁に使用されるアプリケーションになりました。 怠け者だけが彼のメッセンジャーを書かない。 毎日、コミュニケーション手段に革命をもたらすことを約束する新しいアプリケーションがあります。 Yoアプリケーションのように、1つの単語だけを相互に送信できるようにすることもできます。

各メッセンジャーにはそれぞれの視聴者がいて、お気に入りのサービスを使用するようにキャンペーンを展開しています。 そのため、必要なすべての人にすばやく連絡できるように、さまざまなサービスで多数のアカウントを開始し、多数のアプリケーションをインストールする必要があります。



現在の状況は非常にひどいため、将来的にはコミュニケーションの基本原則を脅かしています。 この記事では、具体的な例を使用して1つの考えを伝えようとします。



_なぜ、このような技術がインスタントメッセージングやオーディオビデオコールなどの人類にとって重要であり、どの企業も独占できない。 テクノロジーの開発を阻害し、通信の自由とセキュリティを脅かす方法。





私はすぐにオープンソースの脳に苦しむことはなく、プロプライエタリな製品に恐怖心はない、私自身はクローズドプログラムとプロトコルを使用していると言わなければなりません。 以下に説明する特定の例は、既存の集中型アーキテクチャの問題を示すためだけに提供されています。 私はこれらの製品や企業をまったく嫌いではなく、逆に、それらは十分に進化できると信じています。 彼らの仕事の既存のモデルだけが死ぬべきです。











今日私たちが知っているように、インターネットはオープンスタンダードのおかげで存在しています。 物理（有線、無線、光チャネルによる信号エンコーディング）からアプリケーションのアプリケーションレベル（HTTP、電子メール）まで、ネットワークインタラクションのすべてのレベルはオープンであり、誰でもアクセスできます。 誰でもウェブサイト、ブラウザ、メールクライアントを作成できます。 これを行うために、誰かに許可を求めたり、特許を購入したり、契約を結ぶ必要はありません。

そのため、インターネットで動作する多くのオペレーティングシステムと、一般的なプロトコルをサポートするさまざまなデバイスやアプリケーションがあります。



メールはOutlookから同じOutlookにのみ送信でき、Gmailには送信できなくなったと想像してください。 また、メールを送信する前に、メールクライアントがどの受信者であるかを確認する必要があります。 または、たとえば、携帯電話からの音声通話は、同じメーカーの電話間でのみ機能します。 つまり、SamsungからのみSamsungを呼び出すことができます。 不条理ですね。



これは、チャットを介して誰かに連絡したり、インターネットを介して音声ビデオ通話を行う必要がある場合の状況です。 オーディエンスのリーチを最大化することを目的とする営利企業がボールを支配します。 したがって、各メッセンジャーは他のメッセンジャーから隔離されており、企業はユーザーのためにjeしている。



問題は、ある時点で、オープンスタンダードがユーザーのニーズに追い付かなくなり、営利企業がより魅力的なソリューションを提供できるようになったことです。 これは、商用メッセンジャーに非常に人気があります。 したがって、たとえば、Skypeのような音声品質のようなダイヤルアップの時代には、競合他社は誰も提供できず、無料でさえも提供できませんでした！ 今日、さまざまな情報源によると、Skypeは国際電話の約40％を占めています。



スマートフォンの大規模な普及により、メッセンジャーの普及の第2波が急増しました。 代替メッセンジャーで終日同じお金でチャットできる場合、1 SMSに多額のお金を払うのは不公平であることに人々は徐々に気づきました。 そのため、かつて大きなSMS市場は収縮し始めました。

ある時点で同じ運命は必然的に電話を待っています。 モバイル事業者はこれを理解し、この瞬間を可能な限り遅らせようとします。 まず、ネットワークでVoIPサービスをブロックし 、次にMTSに完全に変装しようとします 。MTSはSkypeの代替を開始します



状況は著作権所有者の著作権侵害に対する闘争を幾分連想させます。まず、オーディオテープ、VHS、CDでのホームレコーディングの可能性、そして後に新しいDRMの導入によるインターネット上の著作権侵害、 DMCAのような法律の採用。 最終的にほぼ完全な損失に終わった。



インターネットチャンネルのかなりの部分を所有している非常に影響力のある通信会社の一部であるモバイルオペレーターのみを紹介します。 したがって、ほとんどの場合、オペレーターのSIMカードと音声サービスを拒否するプロセスは痛みを伴い、長くなります。

ただし、すでにこの方向へのシフトがあります。 サムスンとアップルによって開発されたe-SIM規格により、SIMカードを挿入することなく、電話で直接オペレーターを選択できます。 その結果、新しいオペレーターに接続するプロセスは、音楽サービスに登録するのが難しくなりません-購入したばかりの電話をオンにし、この地域で利用可能なオペレーターのリストを見て、適切な料金を選択し、クレジットカードで支払い、行ってください。

このような状況では、ユーザーは常に電話番号を保持する必要があります。 その数はiNumであるはずでしたが 、私には思えますが、失敗しました。 おそらく、電話番号の代わりに、お気に入りのメッセンジャーコミュニケーターにIDがあります。 明らかに、この場合、ユーザーはオペレーターからインターネット以外のものを必要としません。 そのため、オペレータサービスの公開リクエストは次のように聞こえます。「インターネットにアクセスして、追加サービスをご利用ください」。 すでに、オペレーターが有料サービス、ホロスコープ、ジョーク、SMSパッケージなどのサブスクリプションを提供している場合、インターネット専用のタブレットでSIMカードを使用しているユーザーの怒りがすでに見られます。将来、この感情はより明確に定式化されるでしょう。

Skypeがクールなのはなぜですか？





今日、Skypeは同クラスで最も先進的なプログラムであり、すべての最も近い競合他社よりも桁違いに優れています。その理由は次のとおりです。



  • 音質。

    私の観察によれば（そして十分に試しました）、他のすべての条件が同じであれば、Skypeは常に最高の音質を持っています。 また、ビットレートだけではありません（ SILKコーデックは優れていますが）、ルーティングは動的に構築されるため、Skypeは遅延を最小限に抑えるため、低品質のインターネットではより良い結果が得られます。 近くにあるSIPもありませんでした。 この時点で、多くの人が反対するかもしれません：「しかし、SIPにはHDコーデックがあります！」、しかし、実際には、それらは常に動作するという警告があります。 これは次の段落です。

    Skypeのエコーキャンセレーションにより、対談者はこれを理解することすらできないにもかかわらず、スピーカーに自分の声を伝えることができます。 Skypeでのみ、スピーカーを大音量でオンにして、エコーのわずかなヒントなしで別の部屋から音声を聞くことができます。

  • 装甲貫通性と生存性

    Skypeは悪意のあるcなプログラムで、高度なファイアウォール、不適切に構成されたNAT、ブロッキングを乗り越えることができます。

    どんな奇妙な状況であっても、インターネットがポート80および443へのtcp要求に制限されている場合でも、Skypeはおそらく動作します。 インターネットアクセスがブロックされる状況をよく観察しましたが、Skypeがまだ機能するのは、インターネットを備えた物理ネットワーク上で隣人を見つけて通過したためです。 他のすべてのプログラムは、このような柔軟性を夢見るだけです。

    このような背景に対して、NATの設定が不十分で、RTPが通過せず、UPnP / NAT-PMPが正しく機能しないというSIPプロバイダーのしつこさは、ばかげているように見えます。

  • 暗号化

    私の通信が傍受や分析から保護されることが重要です。 そして、あらゆる種類のMicrosoft Government Security Programにもかかわらず、特別なサービスの要求に応じて、必要なすべてのデータを関心のある団体に転送できるため、ネットワークレベルでは、新しいDPIシステムが経験したとしてもプロバイダーは私の会話を傍受できないことが確実ですSkypeトラフィックの割り当てが困難です。

    多くのメッセンジャーがプログラムを安全にする機能として提示するのは、ネットワークレベルでの暗号化です。 次に、トラフィックの暗号化がセキュリティと同義ではない理由を調べます。

  • 安定性

    少なくともWindowsとOS Xのデスクトップ上のSkypeは非常に安定しており、クラッシュすることはめったにありません。 これにより、数日間中断することなくアクティブなビデオ通話を維持できます。 同時に、たとえばCPUの負荷が小さい場合、たとえば、プロセッサを負荷し、非常に不安定なWebRTCソリューションから。

  • 機能性と利便性

    巨大な機能：ビデオ通話のグループ化、画面共有、ファイル転送、最後に送信されたメッセージの編集機能（！）、普通の電話への発信。 これらはすべて、便利な顧客としての標準納品です。 プラグインや拡張機能はありません。





なぜSkypeは死ななければならないのですか？





  • 安全でない

    安全でないだけでなく、非常に危険です！

    Skypeは、ローカルのIPアドレスを含め、引き続きIPアドレスを公開します。 このデータから、現在の場所を見つけてディスプレイスメントマップを作成できます。

    パブリックバグトラッカーがありましたが、Skypeが1年以上にわたって脆弱性を修正し、Windows用のリモートDoSクライアントを許可している様子を観察することができました。

    ミュートされたマイクをリモートでロック解除できる脆弱性は 、約6か月間修復されました。

    文字通り、開発者に脆弱性に注意を払うように頼まなければなりませんでした。そのため、プログラムを使用することは不可能でした。



    電子メールが知られているアカウントを盗むことを許可する脆弱性が公開されたとき、私は1日間サポートに到達しようとしましたが、できませんでした。 しかし、 Alexei NavalnyとAnton Nosikのアカウントをハッキングしました。

    ところで、この事件の後、Skypeはメールによる確認なしにアカウント登録ポリシーを変更していません。 登録中に任意のメールを指定することができ、検索中にアカウントが表示されます。これにより、メールでアカウントを検索した場合に検索結果をスパム送信できます。



    この記事の執筆時点（ !!! ）では、他の誰かのSkypeアカウントを永久にブロックして、被害者がそれを使用できないようにする少なくとも2つの脆弱性を知っています。 この場合、攻撃者は被害者のログインを知るだけでよく、メールやパスワードは必要ありません。 繰り返しますが、私はこの問題でこのサポートに到達できず、彼らのフォーラムに書きました 。 問題はまだ修正されていません。

    資格のあるサポートに連絡する方法はありません。すべての手紙は、FAQを引用し、問題が本当に自分の側にあり、開発者に渡さなければならないと信じたくないインド人に包まれています。



    したがって、Skypeは安全なメッセンジャーとして位置付けられており、その転送プロトコルは本当に安全であるという事実にもかかわらず、安全ではないだけでなく、率直に言って危険です。 社内の官僚機構の規模により、実際のセキュリティ上の問題は非常に長い間解決されていますが、新しいバージョンではそれぞれ12個の新しいビデオ絵文字が追加されています。



    結論は次のとおりです。



    安全なトランスポートプロトコルはメッセンジャーを安全にしません



    暗号化の3つのレベル、キーの長さ、軍事レベルのセキュリティ、および実際のセキュリティに関連しない他のナンセンスについての別のマーケティングのナンセンスを読むとき、これを覚えておくことが重要です。



  • 閉鎖と予測不能

    プロトコルが閉じられ、頻繁に変更されるため、代わりのSkypeクライアントを取得して作成することはできません。 したがって、Skypeが機能するデバイスは、会社のみが決定します。 少し前に、自分のデバイスに統合するためのクライアントを開発できるSkypeKit SDKがありました。 これを行うには、いくつかの非公開ドキュメントに署名し、サーバーでskypekitを実行しないことを約束し、個人証明書（この場合は取り消すことができます）を発行する必要があり、開発を開始することができました。 しかし、その後、会社は考えを変え、skypekitのサポートを撤回し、既存の購入済みバージョンをすべてブロックすることにしました。 Skype4ComとSkype for Asteriskでも同じことが起こりました。 したがって、Skypeサポートを製品の1つに統合する場合、明日はSkypeが機能しなくなる可能性があり、すべての開発作業が無駄になります。

    同様の状況は、デスクトップクライアントインターフェイスの場合です。 次の新しいバージョンのリリース後、半画面のビデオの笑顔、ボリュームボタンの欠落など、マーケターの狂った空想を消す方法を見つける必要があります。

    明日、Skypeがどうなるかを予測することはできません。怖いです。





Skypeはメッセンジャーコミュニケーター業界のフラッグシップと見なすことができるため、Skypeに大きな注目を集めています。 それは最も豊富な機能を持ち、競合他社はまだそれから非常に遠いです。 同時に、それは最も古く、それを除去する必要があることを示す明確な症状があります。



Skypeの問題は、何らかの形で、すべての一般的なインスタントメッセンジャーに起因します。 私はそれを使用するのに十分な経験がないので、私はその最も近い競合他社のViberを考慮しません。

なぜTelegramは死ななければならないのですか？



  • 集中管理。

    これは、ここで説明するすべてのサービスの主な問題です。 この議論の深刻さについて誤解することがよくあります。 ある会社があなたのコミュニケーションを完全にコントロールしているのは悪いように思えますか？ 結局のところ、彼らはすべてそこにいる善人であり、お金を要求しません、そしてAPIは便利で、絵文字は美しいです。

    しかし、これは常に独自の利益を追求する会社であることを忘れてはなりません。そして、おそらく、世界の平和のために命を奪って仕事をする仕事はありません。

    すでに、他の営利会社からの圧力の下で、Telegramが議論できないトピックをどのように決定するかを観察することができます。TelegramはAppleの要請で「イスラム国家」に関する78のチャットを削除しました 。

音声の場合、これはSIP + ZRTPであり、テキストの場合、XMPP + OTRです。 しかし、すべての商用サービスを背景に、これらのプロトコルは機能するよりも機能しないことが多いと言えます。

多くの場合、SIPを介してコールする場合、一方向の音声、着信または発信コールの問題、およびすべての商用ソリューションで奪われている問題の大部分を取得できることは、私にとっては当たり前のことです。 このネットワークで使用されているNATの種類、ICEやSTUNの終了、ポートを転送する必要がある場合の転送ポートについては絶対に考えたくありません。 これは嫌です。

既存のプロプライエタリ製品で致命的な問題が発見される前にオープンコミュニケーションツールの重要性が認識され、必要な標準に取り組むためにそのような組織が作成されることを望みます。
