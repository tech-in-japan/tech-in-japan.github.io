GNUstepがMac OS Xとの完全な互換性のための資金を調達

GNUstepのプロジェクトマネージャーであるGregory Casamentoは、 Kickstarterで資金調達キャンペーンを開始し、完全に互換性のあるMac OS X 10.6無料バージョンのCocoa APIを作成しました。







スクリーンショットEtoile 0.4。



habrの聴衆はGNUstepが何であるかを認識していると思いますが、念のため、思い出させていただきます。 GNUstepプロジェクトは15年以上存在し、NeXTによって開発されたOpenStep仕様（および一部はCocoa）と互換性のあるObjective-C言語のクロスプラットフォームフレームワークのセットであり、開発ツールのセットです。 これにより、Unixライクなオペレーティングシステム、およびMicrosoft Windows用のアプリケーションを作成できます。 Objective-Cに加えて、Java、Ruby、およびSchemeで記述できます。これには、対応するバインダーがあります。 デスクトップ環境もユーザーアプリケーションもGNUstepプロジェクト自体の一部ではありません;個々のプロジェクト、 GNUstep Application ProjectおよびEtoileはこれを行います 。



GNUstepは、アプリケーション開発者にとって非常に興味深いものになりました。 たとえば、iOSアプリがAndroid上でビルドおよび実行できるようにするApportableプロジェクトで使用されます。 おそらく多くの人がApportableを使用して移植されたAndroid用のOsmosゲームをプレイしました。 アプリケーションの完全なリストは、 ここにあります 。 ところで、habrにはすでにApportableに関する記事があります。



クラウドファンディングが成功した場合、グレゴリー・カサメントは何を提供しますか？



  • プロジェクトが50,000ドルを獲得した場合、GNUstepはCocoa APIバージョン10.6と完全に互換性があります。 これにより、開発者はMac OS X、GNU / LinuxおよびMS Windows用のクロスプラットフォームアプリケーションを作成できます。
  • 100,000ドルで、動作中のWebKit / WebCoreが前のアイテムに追加されます。
  • 私の意見では、150,000ドルのコレクションを持つオプションが最も興味深いです。 この量を収集できる場合、GNUstepはDarlingプロジェクトとマージされます。




ダーリンはワインのようなものです。 Darlingの目標は、実際のMac OS Xとバイナリ互換で、再コンパイルせずにMacアプリケーションを実行できるGNU / Linuxエミュレーションレイヤーを作成することです。 ダーリンは現在まだ初期段階にあり、エンドユーザー向けではありません。
