大正義のために：ビジネス向けKPIサービス

KPI（主要業績評価指標）の概念は、世界の管理経験が今日基づいている柱の1つです。



いわゆるターゲット管理の手法を使用すると、特定の結果に基づいて「公平に」と言われるように賃金を計算する場合、経営者の規律を最大限に制御し、従業員が完全な献身で働くように動機付けることができます



近年、KPI会計を自動化するための多数のシステムがIT市場に登場しています。 それらに関する情報は、開発者の公式ウェブサイトで公開されています。 デモもダウンロードできます。 それらすべてを研究することは不可能であるため、ここでランダムに選択された6個の可能性について説明します。







システムは、分析と比較のために次の基準を考慮して分析されます。



  • 機能性 システムに組み込まれているサービスだけでなく、それらの実装方法も重要です。 企業がシステムを「展開」してスタッフを適応させることができるまでの時間に依存します。
  • さらに、ロシアのビジネス環境で最も一般的な1C：エンタープライズプラットフォームとの「シームレス」な統合は重要な要素です。2つの完全に異なる会計システムでの並行作業がすべての利点を無効にするためです。 システムをオフィススイートに「接続」することも必要です。これも機能に組み込む必要があります。
  • スケーラビリティ 。 顧客企業は、原則として静止していません。製品の基本的な機能を研究した後しばらくすると、高度な機能が必要になる可能性があります。


「目標管理とKPI」



製品ウェブサイト： http : //www.m-bo.ru/technology_mbo/



これは、Volgasoftが1C：Enterpriseプラットフォームで開発した、人員と会社の有効性を評価するための構成です。



システムでは、オブジェクト間の明確な階層関係を使用して論理的な組織構造を構築し、受信したデータに基づいて「透明な」給与計算が行われるように、会社と従業員のインジケーターのマトリックスを作成できます。



さらに、機能にはビジネスインテリジェンスの幅広い武器が含まれています-ダイナミクスと静的およびさまざまなセクションのインジケータを監視および分析するためのツール、ならびに注文の実行の形成と運用制御のためのモジュール。



すべてのレベルで会社の目標を管理するために、3種類のコントロールパネルがあります。



  • 戦略パネルは、会社の全体的な目標（BSC）を提示します。 ここでは、企業のトップマネージャーと所有者が、会社のすべての分野の全体像を見ることができます。 特定の時点で達成された指標に基づいて、適切な戦略的措置を講じることができます。




  • 戦術パネルでは、責任ゾーンが従業員に分配され（重要な目標でもあります）、主要業績評価指標（KPI）が決定されます。 委託された業務分野の従業員が達​​成した結果に基づいて、ボーナスおよびその他のボーナスが授与されます。
  • 操作パネルは、その名前が示すとおり、企業内のプロセスの運用管理に役立ちます。 このパネルでは、ダイナミクスのプロセスの監視と分析、ビジネス分析の実施、多変量解析が利用できます。


製品は1C：Enterprise 8.2プラットフォーム上で独立したソリューションとして作成されましたが、システム間でデータの相互交換を提供する標準構成「Trade Management」、「Enterprise Accounting」、「Production Enterprise Management」に簡単に統合できます。 XMLパッケージを通じて。



つまり、この決定は宣言されたすべてのパラメーターに対応する必要があります。

2つの「しかし」があります：インターフェイスはツールとパネルで過負荷になりすぎています。 これは1C製品の伝統的な機能ですが。



Fox Manager FM Pro + Control 1.7



製品ウェブサイト： http : //www.fox-manager.com.ua/fox_manager_1_7_update.html



Fox Manager FM Pro + Control 1.7プログラムの主な目的は、ビジネスプロセスのモデリングです。



有効性の監視の可能性について話すと、次のようになります。



  • パフォーマンス指標の形成;
  • パフォーマンス指標の監視。
  • 選択した指標のインタラクティブなグラフとチャートの形成。
  • レポート生成;
  • 達成された結果を考慮したボーナス発生額の計算。


「コントロール」モジュールで、各従業員のパフォーマンスインジケータを入力および編集し、従業員のタイプと体重（優先度）を決定できます。

これらのインジケータは、「コントロール」タブで手動で入力するか、MS Excelからエクスポートすることで入力できます。







入力されたデータに基づいて、プログラムは現在の従業員の有効性を自動的に決定し、その後、パフォーマンススケジュールを作成できます。







得られたデータは、従業員のボーナスの計算に使用されます。



各従業員は自分の業績指標を追跡できます。 そのため、いずれかのインジケーターに遅延がある場合、このインジケーターは赤でマークされます。 インジケーターのダイナミクスが正の場合-それに応じて、緑色で強調表示されます。



プログラムと他のシステムおよびサービスとの統合に関しては 、MS WordおよびExcelアプリケーションのオフィススイートからデータをインポートする機能によってこれまでのところ制限されています。 1C：Enterpriseを含む他の企業システムとの相互作用に関する情報が欠落しています。



システムの機能を更新および拡張しても、すべてがスムーズであるとは限りません。 プログラムを更新するとき、古いデータを新しいバージョンに正しく転送するのは困難です。 たとえば、中間バージョンをバイパスしてバージョン1.0からバージョン1.5に「ジャンプ」することにした場合、一部のデータが失われる可能性があります。 プログラムのバージョンを更新した後でも古いデータベースへのアクセスを維持するには、起動前にデータのバックアップコピーを作成する必要があります。

このセクションに進むと、会社の従業員に関する包括的な情報と、会社と個々の従業員の有効性を示すパラメーターとインジケーターのリストを含む多くのサブセクションが開きます。







特別なサブセクションは、会社とその部門の目標に専念しています。



セクション「募集」は 、応募者との面接に関するデータを入力することを目的としています。

データの場所の構造と形式により、このモジュールを企業の人事部門のヘッド/スペシャリスト向けのツールのセットとして判断することができます。



他のシステムとの統合といえば、アプリケーションをGoogle Appsと同期でき、そこからExcelにデータをエクスポートできます。

KPIモニター



製品のウェブサイト： http : //www.kpi-monitor.ru/



システムの名前は、おそらく、それ自体を物語っています。 Microsoft Silverlightプラットフォームのおかげで、システム開発者はビジネスプロセスの監視と視覚化の優れた機能を実現することができました。



KPI MONITORシステムの機能のリストには以下が含まれます。



  • KPIおよびBSCシステムの作成および編集。
  • グラフとチャートを使用した変化のダイナミクスの視覚化。
  • 企業システムおよびローカルファイルからデータをインポート、カスタム処理および変換。
  • KPIシステムの柔軟な設計、調整アルゴリズム、コントロールパネルの形成、および報告メカニズム。
  • パフォーマンス指標の計算。
  • プロセスの多変量解析、インタラクティブな詳細設定およびドリルダウン分析。
  • アクセス権の柔軟な設定。


プログラムの便利で直感的なインターフェース。 システムコントロールパネルはユーザー画面の左側にあり、作業領域は右側にあります。

コントロールパネルの特定のセクションを見ると、画面の右側に、それぞれに対応する一連のパラメーターと特性が表示されます。







前述のように、KPI MONITORの利点は、ビジネスプロセスのグラフィカルモデルとそのインタラクティブな視覚化を構築する幅広い機能です。

システムに表示される視覚効果には、ビジネスプロセスとインジケーターのアニメーション図、チャート、ヒストグラムが含まれます。 これらのツールはすべて、企業の変化のダイナミクスを明確に示しています。









新しいインジケータは、 「インジケータ」エディタを使用してシステムに入力されます。



システムは、パフォーマンス指標を決定するためのツールを開発しました。



パフォーマンスインジケータには、基本、推定、ターゲット、コンテナの4種類があります。



  • 基本的な指標は、システムに手動で入力されるか、他の会計システムからインポートされた重要な指標を定義しています。
  • 計算されたインジケータは、計算式を使用して変更されたシステムの基本的なインジケータです。
  • 目標は、一定期間に達成された現在の実際の結果です。
  • 「コンテナ」インジケータは、いくつかのインジケータの比率をグループ化および計算するために使用されます。


すべてのインジケーターに特別なレポートフォームがあります。



SMARTタスクモジュールは、ユーザーの運用タスクを設定し、完了期限を考慮して、従業員がどのようにそれらに対処したかを評価することを目的としています。



その柔軟なアーキテクチャにより、システムは最も一般的なデータベースとの相互作用をサポートし、他の企業会計システム（1C：Enterpriseなど）と統合できます。



システムは、xml、csv、Excel、SQLサーバーなどの形式からデータを自由にインポートできます。

一般的に、システムは柔軟性があり、動的な開発とスケーリングが可能であるという印象を与えました。
