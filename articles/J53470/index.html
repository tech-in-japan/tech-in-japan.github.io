クイックブックマークを作成する場合、リンクを短縮する場合など。

興味深いことに気づいたことがあります。 すべてが論理的に思えますが、効果は使用できます。



1.ブラウザで任意のページ、たとえばhttp://example.com/を開きます。

2.次に、アドレスの前に、ドメインhabrahabr.ruなどのアドレスを入力し、疑問符「？」を入力します。

habrahabr.ru?http://example.com/

3.ここで、受信したアドレスをダウンロードすると、ブラウザーに次のページが表示されます： http : //habrahabr.ru?http : //example.com/ 。



つまり、ルーティングが構成されていない場合、 http： //example.comという名前の変数がGETを介してスクリプトhabrahabr.ru/に渡されたという事実について話すことができます。



スクリプトでは、この変数の名前を取得して、必要な処理を実行できます。

-ブックマークに保存

-リンクを短縮してユーザーに短いバージョンを提供します（ b23.ruなどのサービスと同様）

-その他。



どういうわけか、興味のために、私は自分自身をこのような迅速なブックマークサービスにしました。 リンクを解析して保存した後のスクリプトは、受信したアドレスへのリダイレクトを行いましたが、私には画面に変更はありませんでした。

当然、このアイデアを工業的に使用するには、非常にシンプルな太平の短いドメインが必要です。



この機能はすべてのブラウザで機能するわけではありません。 たとえば、スクリプトにアドレスを渡すのではなく、同様のリクエストを持つFFは、デフォルトの検索エンジンを使用してアドレスの検索を開始します。 しかし、疑問符の前にスラッシュを付けると、機能します。
