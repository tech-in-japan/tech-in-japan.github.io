C＃と* nixについて



多くは、nixシステムでC＃を記述する方法に興味がありました。 友達を助けようと思います。 Monoプロジェクトがあります。Wikipediaから次のことがわかります。

Monoは、フリーソフトウェアに基づいた.NETシステムの本格的な実装を作成するプロジェクトです。

Monoには、C＃コンパイラ、mcs、.NETランタイムモノ（JITサポートあり）およびミント（JITサポートなし）、デバッガー、およびADO.NETおよびASP.NETの実装を含む多数のライブラリが含まれます。 このプロジェクトは、.NETプラットフォーム上のGTK +グラフィックシステムのバインディングも開発します。

monoランタイムは、C＃、Visual Basic .NET、Java、Boo、Nemerle、Python、JavaScript、PHP、およびObject Pascalで記述されたモジュールを実行できます（.Net / Mono環境にコンパイラーがある場合）。 C、Ada 2005、およびEiffelのサポートも予定されています。

次のオペレーティングシステムのモノラル実装が存在します：GNU / Linux、Solaris、Mac OS X、Microsoft Windows、およびUnix。


これは、ほとんどすべてのC＃OSで簡単にプログラムできることを示しています。 Mono Projectの最新リリースは2.2です。 Mono Project-ここでは、最新バージョンをダウンロードしたり、ドキュメントを参照したりできます。

さらに良いニュースもあります。 Mono開発チームには、 MonoDevelopという補助プロジェクトがあります。 これは、C＃およびASP.NET用の無料のIDEです。 * nixシステムでの迅速かつ迅速な開発に必要なすべての機能をサポートします。

C＃データ型



ウィキペディア

データ型（データ型という用語もあります）は、プログラミング理論の基本概念です。 データ型は、1）値のセット、2）そのような値に適用できる操作のセット、そしておそらく3）値の格納を実装して操作を実行する方法を定義します。 プログラムが動作するデータは特定のタイプです。




C＃は強く型付けされた言語です。 使用するときは、作成する各オブジェクトのタイプ（整数、浮動小数点数、文字列、ウィンドウ、ボタンなど）を宣言する必要があり、コンパイラーは変数に値のみを割り当てることに関連するエラーを回避するのに役立ちますそれらに一致するタイプ。 オブジェクトの型は、コンパイラにオブジェクトのサイズ（たとえば、int型のオブジェクトがメモリ内で4バイトを占める）とそのプロパティ（たとえば、フォームを表示および非表示にすることができるなど）を伝えます。



それらの目的をよりよく理解するために、例を挙げます。 タイプは同じ種類です。 たとえば、さまざまなリンゴ。 ある品種のリンゴは大きく、鉄を含んでいて赤く、別の品種のリンゴは小さくてすみれ色になることがあります=）



C＃はまた、型を他の2つのカテゴリに分割します： 次元と参照 。



組み込み型

インライン型の変換



あるタイプのオブジェクトは、暗黙的または明示的に別のタイプのオブジェクトに変換できます。 暗黙的な変換は自動的に行われ、コンパイラーはユーザーの代わりにこれを行います。 値を異なる型に「キャスト」すると、明示的な変換が発生します。 暗黙的な変換により、データが失われないようにもなります。 たとえば、short（2バイト）からint（4バイト）に暗黙的にキャストできます。

繰り返しますが、小さな例です。 バケツとカップを用意しましょう。 =）カップからバケツに水を注ぐと「暗黙の変換」が得られ、バケツからカップに入れると「明示的に」のみが得られます。 これはどのように可能ですか？ 残りの水はすべて注ぎ出しています。

値が短くても、intに変換するときに失われることはありません。

  1. short x = 1; int = ; //



  2. short x = 1; int = ; //





逆変換を行うと、もちろん、情報を失う可能性があります。 intの値が32.767より大きい場合、変換中に切り捨てられます。 コンパイラは、intからshortへの暗黙的な変換を実行しません。

  1. 短い x;
  2. int y = 5;
  3. x = y; //コンパイルしません


キャスト演算子を使用して明示的な変換を実行する必要があります。

  1. 短い x;
  2. int yは5です。
  3. x =（ short ）y; //そして今、すべてがOKです

オブジェクトは、いくつかの優れたメソッドを提供します。 Equals（） 、 GetHashCode（） 、 GetType（）およびToString（） これらのメソッドはどのオブジェクトでも使用でき、後で説明します。

タイプ文字列



C＃には独自のタイプ文字列があります。 したがって、コピーやマージなどの操作は即座に発生します。

  1. 文字列 s1 = "Hello、" ;
  2. 文字列 s2 = "Habra" ;
  3. 文字列 s3 = "Habr！" ;
  4. 文字列 s4 = s2 + s1 + s3;
  5. Console .WriteLine（s4）; // =>こんにちは、Habrahabr！


値による型のスタイルで割り当てが行われますが、System.String型は参照による型です。 行に戻ります！

変数



変数は、特定のタイプのオブジェクトのメモリ内の場所です。 上記の例では、xとyは変数です。 変数には、初期化に使用する値を設定できます。または、これらの値をプログラムで変更できます。

変数を作成するには、変数のタイプを指定してから、そのタイプに名前を付ける必要があります。 宣言中に変数を初期化するか、プログラム実行中に変数に新しい値を割り当てることができます。 以下に、最初のケースでは初期化を使用して変数に値を割り当てるサンプルプログラムを示します。2番目のケースでは、「=」演算子を使用して変数に値の割り当てを使用します。

  1. クラス変数
  2. {
  3. 静的 ボイドメイン（）
  4. {
  5. int myInt = 10;
  6. システム Console .WriteLine（ "初期化された変数myInt：" + myInt）;
  7. myInt = 5;
  8. システム Console .WriteLine（ "値を割り当てた後の変数myInt：" + myInt）;
  9. Console .ReadLine（）;
  10. }
  12. }


このプログラムの結果は次のとおりです。

初期化された変数mylnt：10

値を割り当てた後のmyInt：5

行は次のとおりです。

int myInt = 10;

mylnt変数の宣言と初期化を意味します。 行：

myInt = 5;

mylntを5に設定することを意味します。

C＃では値の定義が必要です。つまり、変数は使用する前に初期化する必要があります。 このルールをテストするには、次の例を検討してください。

  1. クラス変数
  2. {
  3. 静的 ボイドメイン（）
  4. {
  5. int myInt;
  6. システム Console .WriteLine（ "初期化されていない変数myInt：" + myInt）;
  7. myInt = 5;
  8. システム Console .WriteLine（ "変数myIntを割り当てた後：" + myInt）;
  10. }
  11. }


この例をコンパイルしようとすると、コンパイラは次のエラーメッセージを表示します。

エラーCS3165：未割り当てのローカル変数 'mylnt'の使用

C＃で初期化されていない変数を使用することはできません！
