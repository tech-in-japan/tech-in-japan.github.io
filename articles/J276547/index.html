Azure Stack TP1をインストールする

猫の下には、Azure Stack TP1のインストールプロセスの簡単な説明があります。

この製品についてはすでに簡単に説明したので、この獣をインストールしてみましょう。





ハードウェア要件：





VHDとして既存のハードウェアにインストールできます。つまり、サーバーの寿命への干渉を最小限に抑え、疲れたときに非VHDから起動できます。 すでに書いたように、必要に応じて、ネストされたVM内にインストールできます（ PSS、VMware Workstation ）（ Hyper-V ）。 2016 TP4、すべてのアップデート、およびKB 3124262のアップデートをインストールする必要があります。

LifeHack！ -Azure Stackディストリビューションをダウンロードする場合、WindowsServer2016Datacenter.vhdxが含まれます。これを使用できます。更新プログラムをインストールする必要はありません。 （LifeHackという言葉を書いたことはありません。まあ、本当にやりたかったです）。

さらに、Azure Stack TP1は認証にAzure Active Directoryを使用するため、テストAzure Active Directoryを作成する必要があります。将来のバージョンでは、ローカルADおよびAD FSのサポートが追加されます。 このビデオでは、Microsoft Azureに登録し、Azure Active Directoryを作成するプロセスを示します。

このディレクトリに、Azure Stackの管理者権限を持つユーザーを作成する必要があります。 そして、一度ログインしてパスワードを変更します。



これで、2016 TP4サーバーと更新プログラム、Azure Active Directory +管理者権限と配布を持つユーザーが必要になります。



1.サーバーで、配布パッケージを解凍すると、次のファイルが含まれます。

DeployAzureStack.ps1-Azure Stackをインストールするスクリプト。

MicrosoftAzureStackPOC.vhdx-Azure Stackバイナリを使用したVHDX

SQLServer2014.vhdx-SQL 2014を使用した仮想マシン

WindowsServer2012R2DatacenterEval.vhd-2012 R2仮想マシン

WindowsServer2016Datacenter.vhdxは、必要なすべての更新を含む同じ2016 TP4です。



2. WindowsServer2016Datacenter.vhdxをコピーして、名前をMicrosoftAzureStackPOCBoot.vhdxに変更します。 このVHDXをマウントし、コマンドを実行します： bcdboot <>:\windows







3.サーバーを再起動し、このVHDから起動し、UTCではなく現地時間で動作するようにBIOSを構成します （そのようなオプションがない場合は、UTC -8を使用します）。 OSでローカルディスクが使用可能であり、「オンライン、RAW」と表示され、使用されていないことを確認します。 OSはドメイン内にあるべきではありません。OSの管理者権限を持つアカウントでログインする必要があります。 インターネットアクセスが必要です（Azure.comへ）。 インストール時に許可されるネットワークインターフェイスカードは1つだけです; Azure Stackに特定のネットワークインターフェイスカードを使用する必要がある場合は、他のすべてを無効にします。



4.管理者としてpowershellを実行し、DeployAzureStack.ps1スクリプトを実行します。 次に、ローカル管理者のパスワードを指定し、Azure Active Directoryにログインする必要があります。 その後、Azure Stackが使用するAzure Active Directoryを選択し、オブジェクト（2ユーザー、Azure Active Directoryの3アプリケーション）の作成に同意し、あらゆる種類のeula（powershell、xplat cli、visual studio）に同意します。



5.「Microsoft Azure Stack POCを展開する準備ができました。 続行しますか？「「y」と答える必要があります。 ;））



6.待機:)インストールが正常に完了したら、サーバーマネージャーで[IEセキュリティ強化の構成]を無効にします。

便利なメモ：

インストーラーログ-「C：\ ProgramData \ Microsoft \ AzureStack \ Logs」

鉄のマシンを「azurestack」と呼ぶことはできません

ステップ119でインストールが失敗する-NATVMは、Azure Active Directoryでの認証のためにインターネットアクセスが必要です

配布パッケージをディスクのルート、短い名前のスペースのないフォルダに解凍することをお勧めします。 そしてそこからスクリプトを実行します。

スクリプト（Invoke-AzureStackDeploymentPrecheck.ps1）を変更する場合、次のように、ローカルディスク（サポートされていない）だけでなく使用できます。

$physicalDisks = Get-PhysicalDisk | Where-Object { $_.CanPool -eq $true -and ($_.BusType -eq 'RAID' -or $_.BusType -eq 'SAS' -or $_.BusType -eq 'SATA' -or $_.BusType -eq 'ISCSI') }





Invoke-AzureStackDeploymentPrecheck.ps1およびPoCFabricSettings.xmlを変更して、32 GBのRAMにAzure Stackをインストールすることもできます。
