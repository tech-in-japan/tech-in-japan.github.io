素早い反応と鋼鉄の神経に関するiOSゲームの物語

4月上旬、仕事を辞めました。 余暇から、iOS用のゲームを書くことにしました。 かわいそうな立方体についてのゲーム。これは、他の幾何学図形によって常に追求されています。 本当のドラマ。 キューブは、衝突することなくできるだけ長く保持する必要があります。 ゲームのプロトタイプは約8時間で作成されました。 合計で、ゲームの開発には3.5か月かかりました。 私がこれまでずっとやってきたことは、以下で見つけることができます。







開発



cocos2d-swiftフレームワークを使用しました。 彼はとても人気があり快適です。 SpriteKitと比較した場合の利点は、7未満のiOSバージョンのサポートです。 このゲームは、iPhoneとiPad用に同時に作成されました。 iOS 6以降。







レコードを共有できるようにするために、UIActivityが使用されます。 デフォルトでVKontakteに共有を追加する方法は、この記事habrahabr.ru/post/214637で詳しく説明されています。



各ラウンドでは、敵はランダムな速度と動きの方向を持ちますが、速度の合計は常に同じであるため、ラウンドごとに複雑さは浮き上がりません。 ラウンド中の敵の速度は徐々に増加しますが、一定の制限があるため、プレーヤーの最大結果は無制限です。



競技場の下のスコアボードには、常にプレーヤーの世界ランクが表示されます。 Game Centerに登録されているプレーヤーに対してのみ表示されます。 結果自体に加えて、Game Centerは結果のハッシュ、日付、コードに縫い付けられた文字列を使用して制御データを送信します。 ハッシュはコンテキスト変数で送信されます。 はい、これは万能薬ではないことを理解していますが、私の意見では、偽の結果の大部分を特定して削除するのに役立ちます。 Game Centerでの偽の結果に対する100％の保護は不可能です。 これはオンラインゲームではなく、サードパーティのデバイスから受信したデータを信頼しています。



貧しいキューブを移動するためのコード： 

- (void)touchMoved:(UITouch *)touch withEvent:(UIEvent *)event { if (_gameStatus == kGameStatusStarted) { CGPoint currTouchLocation = [touch locationInNode:self]; _hero.position = ccp(_hero.position.x + currTouchLocation.x - _lastTouchLocation.x, _hero.position.y + currTouchLocation.y - _lastTouchLocation.y); _lastTouchLocation = currTouchLocation; } }


プレイヤーが画面上で指を動かすと、主人公は競技場を動き回ります。 プレーヤーが画面の任意の部分で指を1センチメートル上に動かすと、立方体も1センチ上昇します。 実際、プレイヤーの動きに可能な限り速く反応するために、キューブは動かされるのではなく、即座にテレポートされます。 指を非常にすばやく動かすことができ、競技場全体を60フレーム/秒で1-2フレームで飛ばすことができます。 ここで、トンネル効果が現れます。 衝突は1秒間に60回しか確認されないため、敵をすり抜けることができ、衝突は発生しません。 私は最も簡単な方法で行った。 すべての動きをシミュレートし、衝突をチェックする頻度を6倍に増やしました。 なぜ6時に？ この係数を試してみたところ、1秒間に360回すべての位置を更新すれば、すべてが正常であることがわかりました。 CPUリソースを無駄にするだけで、さらに頻繁に実行します。 係数が6の場合、CPU負荷は10％だけ増加します。 iOSでは、画面上の指の現在の位置も1秒あたり60回発行されます。タッチスクリーンのポーリングレートに影響を与えることはできません。 また、賢明ではなく、現在と前の中間点を5つだけ見つけます。

音楽と音



このような単純なゲームでは音楽が必要ですか？ 私はそうは思いません。 しかし、私はまだ音楽の伴奏を追加したいと決めました。 数百万の予算でクールなプロジェクトで音楽を聴きました。音楽が好きなら、ゲームの作曲家を探していました。 私は2人の作曲家を選びました。 1つは彼が働いている会社との契約のために拒否し、他のモバイルゲーム用の音楽を書くことはできず、2つ目は（プロ意識ではなく）同意しました。 音楽の費用は約14,500ルーブルです。



私は無料と有料のサウンドストックで音を拾いました。 freesound.orgとpond5.comをお勧めします。 freesound.orgでは、MinecraftまたはFlappy Birdで使用されていたサウンドに偶然出くわすことがあります。 また、pond5.comには、少額のお金で大量の優れた音楽とサウンドがあります。



設定には、音楽用とサウンド用の2つのスイッチがあります。 他のゲームでこのオプションを求めている人をよく見ました。







収益化



広告は収益化に使用されます。 1ドルでオフにすることができます。 ゲーム中にバナーをすぐに拒否しました。 特に古いデバイスでは、画面上の貴重なスペースを占有し、ゲームにブレーキをかけます。



ゲームでは、iAd、AdMob、Chartboostのフルスクリーン広告を使用しています。 私の意見では、これらは現時点で最も収益性の高いネットワークです。 Googleが購入したAdMobは、広告ネットワークだけでなく、AppleのiAdを含む12を超えるサードパーティネットワークもサポートしています。 つまり、AdMobを実装すれば十分であり、AdMobを介して既に他のネットワークを接続しているため、何時間もドキュメントを理解する必要はありません。 残念ながらChartboostはサポートされていません。 接続されたネットワークごとに、Googleからアダプターをダウンロードし、ビルドフェーズ->ライブラリとバイナリをリンクに追加する必要があります。







ローカリゼーション



ゲームは14言語に翻訳されています。 Real Racing 3の言語のリストを取得し、トルコ語を追加しました。 当然、これは最後のステップである必要があります。そうしないと、ゲーム内で何かが変更された場合、すべての翻訳を修正するために苦しめられます。 一般的には、可能であれば、翻訳せずにすべてを明確にすることをお勧めします。 アイコンは私たちのすべてです。



onehourtranslation.comで翻訳。 翻訳者に各行に徹底的なコメントを加え、スクリーンショットとビデオを提供することをお勧めします。そうすれば、すべてがスムーズに進み、ここで何を意味するのかを中国人に長く苦痛なく説明する必要はありません。 翻訳は4,000ルーブルかかりました。



ローカライズ後、小さな問題が表示されます。 スクリーンショットの山を作成する必要があり（私の場合は210）、iTunes Connectでそれらすべてを埋める必要さえあります。 （iPhoneの2つの解像度+ iPadの1つの解像度）* 5つのスクリーンショット* 14の言語= 210のスクリーンショット。 30番目のスクリーンショットで私は殺したいと思う。 UI Screen Shooterは、すべての言語のスクリーンショットを自動的にクリックするのに役立ち、iTC Localized Screenshot Uploaderは、iTunes Connectにスクリーンショットを自動的にアップロードします。 両方のプロジェクトはgithubで利用できます。 スクリーンショットをクリックするための特別なゲームモードを作成しました。このモードでは、すべてが「軌道に乗って」行き、画面を押してスクリーンショットを撮り、待機してからもう一度押すなどのUIスクリーンシューター用の簡単なスクリプトを書きました。 210枚のスクリーンショットには30分かかります。 しかし、主なことは、すべてが完全に自動的に行われるということです。 UIスクリーンシューティングのスクリプト： 



 #import "capture.js" var target = UIATarget.localTarget(); var window = target.frontMostApp().mainWindow(); var model = target.model(); var delayFactor = 1.0; if (model.match(/iPad/)) { delayFactor = 3.2; } target.delay(2.5 * delayFactor); target.tap({x:100, y:200}); target.delay(2.0 * delayFactor); captureLocalizedScreenshot("screen1"); target.tap({x:100, y:200}); target.delay(2.0 * delayFactor); captureLocalizedScreenshot("screen2"); …


iPadのdelayFactor係数は実験的に取得されます。 Retina iPadシミュレーターは私のMacBookで非常に遅く、すべての遅延を増やす必要があるため、私はそれを導入しました。







出版社検索



次の出版社にゲームに関する情報を送信しました：Chillingo、FDG Entertainment、Fingersoft、Wooga、KamaGames、G5 Games、Apps Ministry、Pocket Gems、Renatus、Big Fish Games、Game Insight。 一部は答えさえしなかった。 いくつかの出版社は、テスト版を送信するように依頼しました。



ある出版社は興味を持ち、このゲームはいくつかの国でトップ10の無料アプリに昇格できると述べました。 すべての収入は半分。



ゲームの収益は、日中にゲームに参加するプレイヤーの数に依存します（DAU-Daily Active Users）。 Flappy Birdが1日50,000ドルを稼いだことは誰もが知っています。 ゲームの毎日の視聴者は200万人のプレーヤーでした。 ロシアのトップになった後、ゲームの観客は1日あたり2万人になり、広告から1日あたり300〜400ドルを獲得します。 さらに、アプリケーションに悪いレビュー、魅力のないアイコンまたはスクリーンショットがあり、アプリケーション自体に問題がない場合、有料インストールのストリームが終了すると、アプリケーションは上から飛び出し、パブリッシャーの投資は報われません。



1日、ロシアのApp Storeでトップ10に入ることを保証するために、13〜2万ドルを提供する準備をしてください。 数字は変わる可能性がありますが、それはすべてゲームのバイラル性、つまり、人々がゲームをアドバイスする方法に依存します。 友人からアドバイスを受けた人は無料です。 各ユーザーが1人の新しいユーザーをもたらす場合、アプリケーションは爆発的な成長を期待します。 実際、この指標はほとんど常に1未満です。 トップに到達するためにインストールを購入することは高価な教訓ではありません。 1回のインストールには1.5〜3ドルかかります。 ここでは、ウイルス性は考慮されていませんが、より安価です。 なぜトップに到達するのが目標ですか？ アプリケーションが頂点に達すると、人々がこれらの頂点を見るため、膨大な量の無料インストールが提供されることは明らかだと思います。



タイプ3の大規模で高品質のシェアウェアゲームの1人のプレイヤーは、ゲームの「人生」の全期間にわたって3〜4ドル以上をもたらします。 1人のゲームでゲーム内の購入に数百ドルを簡単に費やすことができる、いわゆる「クジラ」の参加なしでは、無料ゲームでの一人当たりのこのような高い平均収益は達成されません。 広告モデルでは、これは非現実的です。 最後に、出版社の代表者はすべての取引を描き、そのようなリスクを冒さないことにしたと述べました。



興味深い事実



8月18日の夕方から8月23日の夕方まで、Game Centerに送信された結果はすべて、送信後約30分で消えました。 すべてのゲームで。 Appleはこの事実を8月22日の開発者フォーラムでのみ認識しました。 プレイヤーから、彼らの記録がなくなったというフィードバックを受け取りました。 私にとっては、Game Centerからの情報を信頼するのは間違いでした。 この問題のため、一部のプレーヤーの結果はローカルでリセットされています。 8月18日から23日に受信したレコードの主要数が結果テーブルに返されました。 しかし、すべてではありません。
