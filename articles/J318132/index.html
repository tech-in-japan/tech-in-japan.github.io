クラウド価格：AWS vs Azure vs Google vs Cloud4Y

さまざまなプロバイダーのクラウドプラットフォームの価格を比較することは、一見すると思えるほど簡単ではありません。 各プロバイダーには、独自の価格設定アルゴリズムと割引を取得するためのさまざまな条件があります。 さらに、プロバイダーは製品の価格調整を定期的に実行します。 今日は、どの要因の下で最も有利な割引を得ることができるかを調べ、それによりクラウドプラットフォームの購入を節約しようとします。





クラウドを選択してください



このパブリケーションでは、最新世代のAWSプラットフォームm4およびc4の2016年12月1日からの最新の値下げと、最新のAWSr4インスタンスの概要を考慮しています。



クラウド割引の本質



平均して、クラウドを使用する場合の主な費用はコンピューティングリソースであり、総費用の約75％〜80％を占めています。



プラットフォームの使用に対して支払う最終的な価格は、プロバイダーからの割引により大幅に引き下げることができます。 したがって、価格比較に進む前に、特定の各プロバイダーからクラウドインスタンスを購入する際のコストを削減する方法を理解することが重要です。 これが、今日の記事で理解しようとするものです。 すべての価格には付加価値税が含まれていません。



AWS



AWSクラウドプラットフォームの最も有利なオファーは、リザーブドインスタンスRIです 。RIは、それ自体はインスタンスではなく、「オンデマンドインスタンス」の割引です。 この割引により、インスタンスの価格が引き下げられ、特定のパラメーター（ロケーション地域、オペレーティングシステム、およびインスタンスタイプ）が組み込まれます。



割引のサイズは期間によって異なります。 AWSは、1年と3年の標準リザーブドインスタンスを提供し、期間が長いほど、購入のメリットが大きくなります。 購入時に、一部または全額の前払いを選択した場合、割引も引き上げられます。 3年間のコンバーチブルRIオプションがあります。このタイプでは割引は低くなりますが、変更により同等以上のコストのリザーブドインスタンスが作成される場合、リザーブドインスタンスの属性を変更することができます。



したがって、節約量は、タイミング、タイプRI、およびテリトリアルゾーンに応じて24％から75％になります。 仕事に最適なオプションは、前払いなしの年次RIです。このタイプは、さまざまなプロバイダーのプラットフォームの比較分析で使用します。 3年間のRIを購入するときは、この期間中に負荷量が変化する可能性があるため、注意することをお勧めします。



コンバーチブルRIを使用すると、インスタンスの属性を変更し、コストを5〜6％節約できます（表を参照）。ただし、1年ではなく3年間インスタンスに関連付けられることに注意してください。

紺Azure



Azureの使用における節約は、契約Enterpirse c Maykrasoft（ EnterpriseAgreement-EA ）によって決定されます。 Azureを使用する場合の割引のサイズは15％〜45％の範囲であり、計画されているインフラストラクチャコストによって異なります。 プロバイダーのオファーを比較するために、平均30％の割引を使用します。



Googleクラウド



Googleのクラウドサービスは、SustainedUsageDiscount（SUD）（現在の負荷に対してのみ支払う）を節約する簡単な方法を提供します。 SUDは前払いを必要とせず、月末に請求するときに割引が計算されます。 この場合、特定のタイプのインスタンスを使用する実際の時間が考慮されます。 負荷が1か月あたり25％の場合、割引は20％です。 負荷がマイルストーンの50％と70％に達すると、割引はそれぞれ20％増加します。 したがって、100％の負荷の条件下でも、可能な最大の割引は30％になります。

Cloud4y



Cloud4Yは、インスタンスとは少し異なる支払いモデル（「Pay as you go」）を提供し、各リソースを個別に柔軟に構成できます。 このモデルにより、消費時にリソースの支払いが可能になります。 15％〜50％の割引の柔軟なシステムがあります。 最大50％の割引を受けるには、1年間契約を締結する必要があります。 「オンデマンド」と呼ばれるものすべてに利用できる割引率は19％です。 すべての場合において、CPUとRAMの料金は、マシンを使用していない「オン」時間にのみ課金されます-料金は課金されませんが、マシンの電源が入っているかどうかに関係なくストレージを支払う必要があります-料金は常にかかります（ディスクが空の場合、データがディスクにある瞬間まで-無料です）。

価格比較



さまざまな割引オプションを検討したので、上記のプロバイダーのインスタンスの価格とタイプの比較に移りましょう。



注：Google SADを使用する場合の節約額は、クライアントの状況に関係なく、プロバイダーによって個別に計算されます。 AWS RIの価格はパラメーターのセットに依存します。 Cloud4Yは、仮想マシンの特性に応じて価格を設定します。 Azureは契約の条件に基づいて計算されます。



価格を比較する場合、選択に関係するインスタンスのパラメーターが類似しているかどうかを考慮することが重要です。 分析では、ローカルデータストレージを使用する場合と使用しない場合の3種類のインスタンスを使用しました。 合計で、6つの異なる関税が取得されました（表の列1を参照）。 それでも、表からわかるように、詳細な比較は、主に次の理由で非常に困難です。



1.似たタイプのインスタンスにもかかわらず、プロバイダーは異なる量のメモリを提供します。 ほとんどの場合、スプレッドは大きくありません（10％以下）が、highcpuのようなインスタンスの場合、AWSおよびAzureプロバイダーはGoogleの2倍のメモリを提供します。 同等の比較のために、AWS、Azure、およびGoogleに技術的に類似したCloud4Yのインスタンスを思い付きました。



2.ローカルSSDの存在：



  • 各AWSインスタンスタイプ（c3、c4、m3、m4、r3、r4）は、ローカルSSDの有無にかかわらず使用できます。



  • Azureには、すべてのインスタンスタイプに常にローカルSSDが含まれています。 結果として、あなたはあなたの欲求に関係なくそれに対して支払います。



  • Googleは、インスタンスのコストにローカルSSDのコストを含めません。 余分に支払う必要があるからです。 同時に、最小SSDサイズは375 GBです。 もちろん、データベースをディスク（Google Presistent Disk）に保存することもできますが、SSDよりもデータベースへのアクセスの時間制限が厳しくなります。



  • Cloud4Yでは、ローカルSSDの有無にかかわらず料金を選択できます。両方のオプションが表に示されています。









オンデマンドインスタンスの価格比較



提示された6つの料金表のそれぞれについて、「オンデマンド」のインスタンスの1時間ごとの支払いと、RAMのギガバイトごとの1時間ごとの支払いによる内訳を作成しました。







結論：Cloud4Yは、RAMの少ないインスタンスに最低価格を提供し、GoogleはSSDなしでRAMを増やしたインスタンスに最低価格を提供します。SSDのインスタンスでは、375 GBのローカルSSDを追加することにより、Google Cloudの高価格が正当化されます。 AWSは、ローカルSSDと連動して大量のRAMが必要な場合に最も有益です。



割引価格の比較：



価格をより視覚的に比較するために、年間コストを参考にしました。 ほとんどの場合、AWS RIの最小インスタンス期間は1年であるため、Microsoft EA契約は3年です（ただし、来年ごとに期間を決定する可能性があります）。 Cloud4Yは最も柔軟です-最小レンタル期間は数日です。
