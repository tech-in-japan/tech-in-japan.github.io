ASUS WL 500G Premiumの出血

Habréには、ASUSの非常に優れたWL 500Gプレミアムルーターに関する記事が既にいくつかあります。

このデバイスには十分な可能性があります-非常に高速なBroadcom BCM4704プロセッサ、32 MBのRAM、2つのUSBポート（低帯域幅ではありますが）。 802.11gを使用する場合の制限は54MbpsであるWi-Fiモジュールは、私たちを失望させました。 しかし、ラップトップ、ネットブック、コミュニケーター、その他のモバイルデバイスでは十分です。







この充填には、ASUSプログラマーが望んでいたよりも適切な使用が必要でした。 そのため、ネイティブファームウェア（Oleg`a oleg.wl500g.infoのファームウェア）の修正と完全に新しいプロジェクト（DD-WRT www.dd-wrt.com ）の両方が作成されました。

USBインターフェイスを介して外付けハードドライブを接続し、ビッグブラザーをアンロードし、夜間に完全に無音でスリープするトレントクライアント（Transmission、rtorrent）をインストールする可能性は非常に人気がありました。

これらのすべての操作が行われ（これについて別の記事を書くことができます）、ある種のシードボックスが取得され、その任務に非常にうまく対処しました。





...配布された急流の数は150個を超えませんでした。 ディストリビューションが非常に多いため、ルーターの速度が著しく低下し始め、ハードドライブ上のスワップの存在は状況を救いませんでした。 したがって、この素晴らしいハードウェアにRAMを追加することが決定されました。

この理由は、同じ名前のファームウェアの作者であるオレグによって書かれた記事www.wl500g.info/showthread.php?t=7048と、フォーラム参加者によるその後の議論www.wl500g.info/showthread.php?t=7523でした 。

数日間このトピックを読んだ後、私はルーターの損失に自分自身を和解させ（常に最悪の事態に備える必要があるため）、アップグレードに必要なチップの検索に進みました。



この操作には、次の条件に適したチップのみが適していました。

1）333 mhz以上の周波数のDDR 1チップ（チップが333 MhzでCl2の遅延で安定して動作することが望ましい）。

2）チップの内部構成は32M x 16（16ビットで32メガソードとして読み取られる）でなければなりません。

3）チップケース66ピンTSOP



テスト済みのチップとそれらが削除されたメモリモジュールのリストは、すでに言及されているフォーラムwww.wl500g.info/showpost.php?p=122252&postcount=630からも見つけることができます。

数日後、Hynix HYMD532646CP6J-D43マーキングとHynix HY5DU121622CTP-D43チップを搭載した256Mb PC3200 DIMMモジュールが見つかりました。



移植のために使用されました：



  • はんだごてとヘアドライヤー付きのはんだ付けステーションLukey 852D +ファン




  • 電子部品のはんだ付け用フラックスF-2000






  • はんだLC-60






  • はんだ除去テープ（非常に便利なものです。余分なはんだを素早く簡単に除去し、はんだ付け後にブルーパーをきれいにすることができます）






  • イソプロピルアルコール。 はんだ付け後にフラックス残渣を除去するには






  • アルコールの使いやすさのための綿棒
  • 多くの忍耐




そのため、ルーターを分解します。 これを行うには、底のゴム製の脚を外すだけで十分です。 それらの下に4本のネジがあります。 ネジを外し、カバーを取り外します。 ボードは4本のレールに取り付けられており、指でこじ開けるだけで簡単に取り外すことができます。 この前に、Wi-Fi導波管を切断することを忘れないでください。 右下の写真には、金属板があります。 これは、プロセッサとRAMを接続する回路を電磁干渉から保護するカバーです。







ドライバーを使用すると、カバーを簡単に取り外すことができます。







その下には、2つのSamsung RAMチップ（HynixとWinbondのオプションもあります）、プロセッサー、およびファームウェアが格納される8メガバイトのフラッシュメモリがあります。 その後の操作の利便性のために、周囲を囲むリブの近くでワイヤーカッターで切断してクロスピースを取り外すことをお勧めします。



ボードの反対側には、同じシールドのために、長方形のホイルが接着されています。これは慎重に取り外す必要があります。 その下には、電流制限抵抗を備えたアドレスラインのトラックがあります。

メモリを交換する前に、33オームの抵抗をR43の下の空いている場所にはんだ付けする必要があります。 これにより、アドレス行A12が、インストールする128 MBのメモリすべてをアドレス指定できるようになります。 この名称とサイズの抵抗は、チップの提供者になった同じDDRモジュールから、または故障したIDEハードドライブから、または単にラジオ部品店で購入することができます。







そのため、抵抗ははんだ付けされています。



チップの交換を始めましょう。 チップは、耐火性の鉛フリーはんだではんだ付けされています。 私の場合、ステーションインジケータによると、はんだは420度で「泳ぎ」始めました。 チップ端子にフラックスを適用します-少し。 両側の脚を静かに温め、ピンセットでチップを取り外します。

はんだ付けステーションがない場合は、先端が平らな60ワットのはんだごてを使用できます。 この場合、端子の下にねじ山をねじ込み、はんだごてでそれらを温め、はんだが溶けるまでゆっくりと引っ張り、チップの最初の片方のはんだ付けを解除してから、もう一方のはんだ付けを解除する必要があります。







別のチップに対して操作を繰り返します。



次に、テープを使用してはんだを除去し、古いはんだの残りを除去します。 これを行うには、残りのはんだの上にテープを置き、はんだごてでテープを通して温めます。

写真は私のものではありませんが、プロセスは明確です







テープは細い銅の静脈から織られ、フラックスが含浸されており、それ自体にはんだを引き込みます。 はんだで飽和しているサイトは切断され、破棄されます。 その結果、きれいなコンタクトパッドが得られます。







次に、フラックスをそれらに適用し、はんだごての助けを借りてはんだLC-60を適用します。それほど耐火性ではなく、作業がはるかに簡単です。 これにより、古い耐火性はんだを残す場合に発生する可能性のある非はんだが除去されます。 結果は、適切に調整された連絡先になります。

キーに導かれ、ボードにチップを押し付けます。







そして、各脚に薄いはんだごてを流し、はんだをその下で強制的に溶かし、接触領域に押し付けます。 別のチップについても同じ操作です。

すべてが消えたので、綿棒、アルコールを取り、回路基板とチップの脚の間のフラックスの痕跡を拭きます。 途中で、「鼻水」の存在を調べます。 はんだごての不正確な接触、震えている手、その他の地球的および地球外的要因に起因する可能性のある、隣接するチップ脚の望ましくない接続。 また、非飲酒の可能性についても調べます。 出力と回路基板パッドの高品質な接触ではありません。

すべてが正常な場合は、アルコールが乾くまで待ってからデバイスの電源を入れます。 ジョブが正常に完了すると、ルーターがすぐに起動します。「電源」インジケータが点灯し、5〜10秒後に「空気」が点灯します。

デバイスの電源を入れ、電源インジケータがオフで、空気が暗い場合、これはルーターが起動しなかったことを示します。 この理由は、原則として、飲酒（はんだごてで各足をもう一度取り除く）、または「鼻水」（虫眼鏡またはレンズ、テーブルランプを取り、はんだ付けの場所を慎重に調べる）ではなく、余分なものを取り除き、テープとはんだごてを使用することです

ルーターを正常にロードしたら、telnetまたはsshを介してコンソールにアクセスし、128メガバイトすべてのソフトウェアアドレッシングを有効にするコマンドを入力します



nvram set sdram_init=0x0011

nvram set sdram_config=0x0062

nvram commit
