Data ONTAP Cluster Modeにアップグレードする理由

以前の投稿で書いたように、Data ONTAP v8.3.xは、FASシリーズのNetAppストレージシステム向けオペレーティングシステムの最も重要なリリースの1つです。



この記事では、最新バージョンのClustered Data ONTAPで最も重要なNetAppストレージシステムの機能を提供します。 伝統的に、私は車の例を挙げます。テスラ車を持っていると想像してください。ファームウェアを更新し、オートパーキング機能付きのオートパイロットを無料で入手しました。 本当にいい？ したがって、システムをCluster-Modeにアップグレードするための最も重要な引数は、投資を節約し、古いハードウェアで最新の機能を取得する能力です。



  • 外出先でのゼロのオンライン検出（重複排除）は 、データベースやプロビジョニング仮想マシンの場合に非常に役立ちます。
  • FlashPool（およびAFF）システムのオンライン重複排除。SSDドライブの寿命を延ばします。 この機能は8.3.2以降で使用可能です。
  • VMWare vSphere 6にアップグレードすると 、NASとSANの両方でvVOLがサポートされます。
  • QoS-I / O操作の最大しきい値またはファイル、ムーン、バリュム、SVMのMb / sの設定。
  • VMware vSphere 6にも存在するNFS4.1サポート
  • NFSを並列化し、再マウントせずにクライアントからファイル共有へのパスを切り替えることができるPNFSサポートは、RHEL 6.4以降でサポートされています。
  • Win 8およびWin 2012以降のクライアントで動作するSMB（CIFS）3.0のサポート
  • Data ONTAPからSMB 3.0のファイルとセッションを閉じるためのサポート
  • SMB 3.0暗号化のサポート。
  • SMB連続可用性（SMB CA）。切断せずにパスとストレージコントローラーを切り替える機能を提供します。これは、SQL / Hyper-Vにとって非常に重要です。
  • Microsoft SAN / NASを使用する場合のODXを使用すると、特定のパターンでデータブロックを詰まらせるなどの日常的なタスクをアンロードでき、ホストとストレージ間の不要なデータを追跡できません。
  • クラスタの他のノードを含む、アグリゲート全体のボリュームのオンライン移行
  • 他のクラスターノードを含むボリュームによるオンラインムーン移行
  • HAペアのノード間のユニットのオンライン切り替え
  • 異種システムを1つのクラスターに結合する機能。 したがって、アップグレードはデータへのアクセスを停止することなく実行されます。この機能のおかげで、NetAppはクラスターをImmortalと呼びます。 クラスターの更新時に、そのノードはcDOTの異なるバージョンで構成される場合があります。 機会を逃すことはできず、ほとんどの競合他社がまったくクラスタリングを行っていないことは言及できません。まず、ノードの数が非常に限られていること、そして次に、すべてのクラスターノードが同一でなければなりません（均質なクラスター）。
  • ADP StoragePool-キャッシュ（ハイブリッドアグリゲート）の下でSSDをより合理的に分散するためのテクノロジー。 たとえば、SSDが4つしかなく、2、3、または4つのアグリゲートでSSDのキャッシングを利用したい場合。
  • ADPルートデータパーティショニングにより、FAS22XX / 25XX / 2600およびAFF8XXX / A200 / A300 / A700システムの専用ルートユニットを放棄できます。
  • SANのスペース再生 -削除されたブロックをストレージに返します。 SCSI3 UNMAPがない場合、データブロックが月に削除された場合、ボールト自体の薄い月にはこれらのブロックが使用済みとしてマークされ、まだディスクスペースを占有していることを思い出してください。ストレージとホストのフィードバックメカニズム。 スペース再生をサポートするには、ホストがESXi 5.1以上、Win 2012以上、RHEL 6.2以上である必要があります。
  • 適応圧縮 -圧縮データの読み取り速度が向上します。
  • ファイルとムーンのFlexCloneの機能強化 。 これで、ファイルまたはムーンのクローンを削除するポリシーを設定できます（ vVOLなどで役立ちます ）。
  • Active Directoryを使用してストレージ管理者を認証する機能（CIFSライセンスは不要）。
  • Kerberos 5のサポート ：128ビットAESおよび256ビットAES暗号化、IPv6サポート。
  • SVM DRのサポート （ SnapMirrorに基づく）。 つまり SVM全体をバックアップサイトに複製する機能。 バックアップサイトでは、メインサイトとは異なるネットワークアドレス範囲がよく使用されるため、重要な点は、レプリケーション関係のセットアップの段階で新しいネットワークアドレスを事前設定できることです（ ID破棄モード）。 ID破棄機能は、L2ドメインをメインサイトからスペアサイトに拡張するために機器や通信チャネルを購入できない大企業にとっては非常に便利です。 クライアントが新しいネットワークアドレスに切り替えるには、DNSレコードを変更するだけで十分です（簡単なスクリプトで簡単に自動化できます）。 すべてのLIF、ボリューム、LUN設定がリモートサイトに保存されている場合、 ID保存モードもサポートされます。
  • ボリューム全体を復元せずに、SnapVaultバックアップからファイルまたはムーンを復元する機能。
  • ファイルシステムをスキャンするために、ストレージシステムとウイルス対策システムを統合する機能 。 Computer Associates、McAfee、Sophos、Symantec、Trend Micro、Kasperskyがサポートされています。
  • FlashPool / FlashCacheの動作を最適化しました。 圧縮データと大きなブロックをキャッシュできます（以前は、これらのタイプのデータは両方ともキャッシュに落ちませんでした）。

FAS22XXアップデート



まず、10Gbitメザニンアダプターが必要です。 FCメザニンを使用している場合は、10Gbitに置き換えてiSCSIに切り替える必要があります。



第二に、そのような各コントローラーから1つの10GbitポートをCluster-Interconnectのニーズに寄付する準備をします。 これは最低限必要です。 コントローラーから1つまたは2つの10Gbitポートを提供できます。1つの10Gbitポートがクラスター相互接続に割り当てられている場合、同じニーズに別の1Gbitポートを提供することを貪欲にしないでください。 言い換えると、更新によりネットワーク接続が再設計される可能性があります。



すでにアクティブ/パッシブ構成を使用している場合は、ADPルートデータパーティショニングのおかげで、ルートアグリゲートからリリースされた3〜4個のディスクを保存できます。



FAS22XXをcDOTに更新する価値があるのはなぜですか


7-Modeの2つのコントローラーからの4つの10Gbitポートの代わりに、ホストを接続するために、コントローラーからのアップストリームポートにCluster-Modeで2つの10Gbitポートを取得します。

まず、このようなソリューションのフォールトトレランスは、リンクまたはコントローラーに障害が発生した場合のLIFインターフェイスがIPおよびMACアドレスと共に2番目のコントローラーにオンラインで移動するため、影響を受けません。 iSCSIの場合、ホスト側では、リポジトリへのパスがマルチパスによってドライバーレベルで単純に切り替えられます（SAN LIFは移動せず、代わりにマルチパスが機能します）。



第二に、一部のユーザーは合成ロードを実行しようとし、48x SAS 10kディスクで4つのリンクのうち2つを40％以上ロードできず、他の2つはまだアイドル状態でした。 また、通常の動作モードでは、同じユーザーからのリンクの使用率は10％に達しませんでした。 別のお客様は、アクティブ/パッシブ構成のcDOT 8.3.1およびFAS2240 20x900 +4 SSDシステムに住んでいます（つまり、実際には1つの10Gbitリンクのみが使用され、2番目のリンクは緊急時に必要になると受動的に予測します） Cluster-Modeに切り替えた後のリポジトリへのアクセス速度のマイナスの変化。ただし、このリポジトリ上に、小規模ではないすべての仮想インフラストラクチャが存在します。

32XXおよび6XXXシステムの更新



通常、古いシステムを更新しても、ストレージをネットワークに接続する際に変更は発生しません。ボードにクラスターインターコネクト用の10ギガビットポートがあるか、空きPCIスロットに配信できるためです。



32XXシステムでは、10Gbitポートはボード上にありませんが、無料のPCIコネクタに配信できます。



6XXXシステムでは、すでに10Gbitが搭載されており、それらを有効にするか、空きポートがない場合は10Gbgit NICを購入する必要があります。



結論



古いNetAppストレージシステムをアップグレードする機能は、FAS22XXシステムでコントローラーから1つの10ギガビットポートが失われたという事実を考慮しても、投資を節約する方法です。幅広い追加機能により、ゲームは価値があります。 古いシステムでは、変換は原則としてネットワーク接続の変更をまったく伴いません。 他の多くの評判の良いベンダーにとって、アップグレードプロセスは通常、ディスクシェルフを含むハードウェアの完全な交換です。 この点で、NetAppは、最新のファームウェアで古いハードウェアを更新する機能と、古いストレージシステムのすべてのシェルフを接続する機能により、はるかに柔軟性が高く、Low / Mid / High-Endクラス間でディスクシェルフの互換性を区別しません。 不滅のクラスターを使用すると、さまざまなモデルを組み合わせて、垂直および水平に継続的にスケーリングしたり、更新およびアップグレードしたりできます。
