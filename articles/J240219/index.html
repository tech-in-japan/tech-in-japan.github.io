言語が重要な理由



初めは、コンピューターの分野の誕生時に、プログラミング言語はありませんでした。 プログラムは次のようになりました。 



00110001 00000000 00000000 00110001 00000001 00000001 00110011 00000001 00000010 01010001 00001011 00000010 00100010 00000010 00001000 01000011 00000001 00000000 01000001 00000001 00000001 00010000 00000010 00000000 01100010 00000000 00000000




これは、1から10までの数字を追加し、結果を表示するプログラムです（1 + 2 + ... + 10 = 55）。 非常に単純な仮想マシンで実行できます。 最初のコンピューターをプログラムするには、スイッチの大きな配列を目的の位置に設置するか、パンチカードに穴を開けてコンピューターに送り込む必要がありました。 退屈でエラーが発生しやすい手順であることが想像できます。 単純なプログラムでさえも、多くの知性と規律が必要でした。 複雑なプログラムはほとんど考えられませんでした。



もちろん、これらの神秘的なビットパターン（ゼロと1）を手動で入力すると、プログラマはウィザードのように感じることができました。 そして、仕事の満足という意味で何か価値がありました。



指定されたプログラムの各行には、1つの命令が含まれます。 通常の言語では、次のように説明できます。



  1. メモリ0に0を書き込む
  2. メモリ1に1を書き込む
  3. セル1の値をセル2に書き込む
  4. セル2の値から11を引く
  5. セル2の値が0の場合、手順9から続行します。
  6. セル0にセル値1を追加します
  7. セル1に1を追加
  8. ポイント3に進みます。
  9. セル値0を出力




このオプションは、たくさんのビットよりも読みやすいですが、それでもまだあまり便利ではありません。 命令番号とメモリ位置の代わりに名前を使用すると、理解が向上します。 



   'total'  0  'count'  1 [loop]  'compare'  'count'  11  'compare'  'compare'  ,   [end]  'count'  'total'  1  'count'   [loop] [end]  'total'




今では、プログラムの仕組みを理解することはそれほど難しくありません。 対処できますか？ 最初の2行は、2つのメモリ領域に初期値を割り当てます。 totalは計算結果の計算に使用され、countは現在作業中の数を監視します。 「比較」を使用する行は、おそらく最も奇妙な行です。 プログラムは、カウント11がカウント停止と等しくないかどうかを理解する必要があります。 架空のマシンは非常に原始的であるため、変数がゼロに等しいかどうかのチェックのみを実行し、別の行にジャンプするかどうかを決定できます。 したがって、「比較」と呼ばれるメモリ領域を使用してcount-11の値を計算し、その値に基づいて決定を下します。 次の2行は、結果カウンターにカウントを追加し、プログラムがまだ11に達していないと判断するたびにカウントを1ずつ増やします。



同じJavaScriptプログラムを次に示します。 



 var total = 0, count = 1; while (count <= 10) { total += count; count += 1; } console.log(total); // → 55




さらにいくつかの改善。 主なもの-ライン間の遷移を手動で示す必要はありません。 while言語構造は、これを独自に行います。 彼女は、条件が満たされるまで（count <= 10）、つまりcountの値が10以下になるまで、中括弧で囲まれたブロックの計算を続けます。一時的な値を作成してゼロと比較する必要はなくなりました。 それは退屈でした、そしてプログラミング言語の力は退屈な詳細を取り除くのを助けることです。



プログラムの最後に、while操作が完了すると、出力のためにconsole.log操作が結果に適用されます。



そして最後に、便利な範囲および合計操作がある場合、プログラムは次のようになります。これにより、特定の間隔で一連の数値を作成し、設定量を計算します。 



 console.log(sum(range(1, 10))); // → 55




このf話の教訓は、同じプログラムを長いものと短いもの、読みやすいものと読みにくいものの両方で書くことができるということです。 プログラムの最初のバージョンは完全にあいまいで、最後の-ほぼ実際の言語-1から10までの数値の範囲の合計を書き留めました。次の章では、そのような方法を説明します。



優れたプログラミング言語は、プログラマが必要な操作を高レベルでコンピューターに通知するのに役立ちます。 退屈な詳細を省略でき、便利なビルディングブロック（whileおよびconsole.log）を提供し、独自のブロック（合計と範囲）を作成でき、ブロックの結合を簡単にします。



javascriptとは何ですか？



JavaScriptは、Netscape NavigatorブラウザのWebページにプログラムを追加する方法として1995年に導入されました。 それ以来、この言語はすべての主要なグラフィカルブラウザで定着しています。 彼は、ブラウザベースの電子メールクライアント、地図、ソーシャルネットワークなど、最新のWebアプリケーションを表示できるようにしました。 また、従来のサイトでも使用され、双方向性とあらゆる種類の付加機能を保証します。



JavaScriptはJavaと呼ばれる別の言語とはほとんど関係がないことに注意することが重要です。 マーケティング上の理由から、同様の名前が選択されました。 JavaScriptが登場したとき、Java言語は広く宣伝され、人気を博しました。 誰かがこのエンジンにしがみつくのがいいと決めた。 そして今、私たちはこのためにどこにも行きません。



言語がNetscapeを超えた後、そのサポートを主張するさまざまなプログラムが同じように機能するように、言語の作業を説明するドキュメントが作成されました。 これは、ECMA組織の名前でECMAScript標準と呼ばれます。 実際には、ECMAScriptとJavaScriptを同じものとして話すことができます。



多くの人がJavaScriptを批判し、それについて多くの悪いことを言っています。 そして、これの多くは真実です。 初めてJavaScriptでプログラムを記述しなければならなかったとき、私はすぐにうんざりしました-言語は私が書いたほとんどすべてを受け入れましたが、これを意図したとおりに解釈しませんでした。 これは主に、自分が何をしているのかわからなかったという事実によるものでしたが、問題がありました。JavaScriptはリベラルすぎるからです。 それは初心者向けのプログラミングの促進として考えられました。 現実には、システムが問題を報告しないため、プログラム内の問題を見つけるのが難しくなります。



柔軟性には利点があります。 より厳密な言語では不可能な、さまざまなテクニックの余地が残されています。 「モジュール」の章で見るように、言語の欠陥のいくつかを克服するために使用できる場合があります。 彼と本当に学び、一緒に仕事をした後、JavaScriptを愛するようになりました。



JavaScript言語のいくつかのバージョンがすでにリリースされています。 ECMAScript 3は、2000年から2010年までの言語の台頭中に広く普及した支配的なバージョンでした。その時点で、野心的な4番目のバージョンが準備され、言語のいくつかの根本的な改善と拡張が計画されていました。 しかし、政治的な理由により、生きている人気のある言語を変更することは非常に難しく、バージョン4の作業は2008年に中止されました。野心的な第5バージョンは2009年に登場しました。本。



JavaScriptはブラウザのみでサポートされていません。 MongoDBやCouchDBなどのデータベースは、スクリプト言語およびクエリ言語として使用します。 デスクトップとサーバー用のプラットフォームがいくつかありますが、最も有名なものはNode.jsであり、ブラウザー外でプログラミングするための強力な環境を提供します。
