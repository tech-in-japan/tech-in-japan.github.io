Oracle Database 18c XEがリリースされました



起こった！ シャンパンを開いてパーティーを開くことができます。前回のリリースから7年以上が経過し、 XEを搭載した最新のOracle Database 18がついにダウンロード可能になりました。 これまでのところ、Linux x64のみに対応していますが、他のプラットフォームのバージョン、DockerおよびVagrantのイメージも間もなく登場する予定です。

最新ではない人のために、 Oracle Database XEまたはExpress Editionは、最も強力なリレーショナルDBMSの無料版です。 従来、このタイプのエディションには、使用可能な最小機能、使用するリソースに関する特定の制限、および小さな配布サイズがあります。 それでも、このバージョンのDBMSでさえ、このDBMSの機能を習得しているだけの人々、現在すべての制限に満足していて、あなたや私のような普通の愛好家の間で技術を評価したい小さな企業の間で常に非常に人気があります。







XEの新しい18番目のバージョンは、このDBMSを使用する前例のない範囲を開きます。これは、今回Oracleがその戦術を変更し、最小限の機能を提供する代わりに、このエディションに最も高価で機能的なEnterprise Editionのほとんどすべての最も人気のあるオプションが含まれているためです ただし、使用するリソースには制限があります。







なぜ18代ですか？









毎日OracleのDBMSを使用していない読者には、図18の由来を説明します。 実際、バージョン12 cリリース2の後、製造業者はバージョン管理スキームを変更し、DBMSの新しいバージョンをリリースするために1年サイクルに切り替えることを決定しました。 したがって、リリース12.2.0.2はバージョン18とみなされ、将来的にはOracle Databaseの19、20以上のバージョンがリリースされる予定です。 同時に、Express Editionの新しいプロダクトマネージャーは、他のエディションのリリースとほぼ同時に、新しいバージョンのXEを引き続き楽しむことを約束しました。 また年に一度。 唯一の違いは、SEおよびEEとは異なり、XEに対してパッチおよびセキュリティパッチが発行されないことです。 ただし、XEの後続の各メジャーリリースには、前年全体のSEおよびEEのエディションで行われたすべての修正および変更が含まれます。もちろん、喜ぶことはできません。 特に7年間、XEの11番目のバージョンもパッチや修正を受け取らなかったと考えると、コミュニティから大きな需要がありました。







新機能



それでは、XEを備えた Oracle Database 18の新しいバージョンで注目すべき点は何ですか？ 実際、多くのこと。 しかし、このエディションの以前のバージョンとの最も重要な違いは、今回はStandard EditionおよびEnterprise Editionのほとんどすべての最も一般的なオプションが含まれていることです。 それらの最も注目すべきものをより詳細に検討してください。







マルチテナント



12c以降、 Oracle Databaseはいわゆるプラガブル・データベースをサポートしています。ポイントは、わずか数個のSQL * Plusコマンドでデータベースをインスタンスに「接続」および「切断」できることです。 これにより、ホスト間やDBMSバージョン間でも問題なくデータベースを転送できます。 さらに、Oracle Databaseの新しいバージョンとの互換性を約束します。 アップグレードプロセスは、unplug-uninstall-install-plugほど複雑ではありません。 マルチテナントアーキテクチャでは、たとえば、同じホスト上の異なるデータベースでAPEXの異なるバージョンをサポートすることもできます。







インメモリ列ストアと集計



最も強力で高価なOracle Databaseオプションの1つは、インメモリ列ストアです。これにより、データのコピーを列指向形式でメモリーに格納し、透過モードで迅速にアクセスできます。







パーティショニング



パーティショニングは新しくてエキサイティングなものではありませんが、Oracle Databaseでは常にEnterprise Editionのユーザーのみが使用できましたが、競合するDBMSでは、ほとんどの場合、このオプションはすぐに使用できます。 たとえば、 最近リリースされたPostgres 11ハッシュパーティショニングでも追加されました。 現在、この機能は無料版のOracleでも使用できます。







高度な分析とセキュリティ



これは、データマイニング用のツールセット全体です。 インターフェイスデータマイニングSQL、Rプログラミング、Oracle Data Miner UI。 セキュリティに関しては、これはすべて透過的なデータ暗号化に関するものです。







オンラインインデックスリバイド



プロジェクトの1つでこの機会を逃した方法。 これで、インデックスの非ブロック再構築をその場で実行できます。







オンライン表再定義



パッケージDBMS_REDEFINITION

クエリ結果キャッシュとPL / SQL関数結果キャッシュ



パフォーマンスに関して言えば、データをキャッシュせずに実行できないことは秘密ではありません。 SQLクエリ結果とPL / SQL関数のインスタンスレベルのキャッシュが利用可能になり、アプリケーションのパフォーマンスを向上させるために使用できるようになりました。







ビットマップインデックス



特別な種類のインデックス。その使用は通常、分析システムに関連付けられています。 これは、このタイプのインデックスは、ほとんど更新されないテーブルでのみ使用することをお勧めしているためです。 ただし、それらの利点は、これらのインデックスを組み合わせてデータへのアクセスをさらに高速化し、複数のソース（いわゆるビットマップ結合インデックス）の使用にも使用できることです。







Oracle Advanced Compression



XEでのリソース制限の達成を遅らせるオプション。 データベースとインスタンス自体のメモリ構造の両方で透過的なデータ圧縮を生成します。







マテリアライズドビュークエリの書き換え



オプティマイザがSQLクエリを自動的に「書き換え」、マテリアライズドビューを使用してパフォーマンスを改善できるオプション。







Oracle Spatial and Graph



ストレージに関するすべてとジオデータの操作。

制限事項



軟膏のハエはどこにありますか？ このバージョンのDBMSで使用されるリソースの制限。 ここで論理的な増加が認められましたが：







  • 最大データベースサイズは12 GBです （11 g XEでは11 GB）。
  • 最大2 GBのRAM（11 g XEでは1 GBに対して）。
  • 最大2つの CPUスレッド（11 g XEの1つに対して）。
  • 最大3つのプラガブルデータベース（プラガブルデータベースまたはPDB、11 g XEにはそのような機能がありませんでした）。


もちろん、産業用の場合、そのようなパラメーターはばかげているように見えますが、アカデミックな使用、新興企業、および小規模企業の場合、すべてが必要なフレームワークに適合します。 深刻な産業の巨人でさえ技術を評価するだけでなく、これで十分すぎる場合があります。
