部門の負荷の別の計算

部門の負荷計算を読んだ後、同様のプログラムを作成した経験を共有することにしました。 トピックが上記の双子であることが判明する可能性があることをおizeびしますが、プログラムは非常によく似ています。 システムは私が開発したもので、顧客は部門の長でした。



説明



メインウィンドウには、メインデータベーステーブルを編集できるタブが含まれています。 主なもののみを提供します。

「カリキュラム」タブには、部門に来て、さらに配布される主題と時間に関するデータが含まれています。







さらに、[学期の科目]タブの[更新]ボタンを使用すると、生徒に関する入力情報と時間の基準に従って時間数を計算できます。 監視基準は、カリキュラムと同様に外部から部門に届きます。







形成された時間の配分は次のタブで実行されます。講義と練習が異なる教師によって行われる場合、1つの科目を複数の教師に分けることができます。 当初は、この問題が解決されるまで当学部にはないため、いくつかの専門/学部で大規模なストリーミング講義を実施する規定はありませんでした。







別のウィンドウで、先生による時間の統計を頭で見ることができます。 部門は非常に便利です。







現時点では、プログラムは2種類のレポートを作成します。



  • 部門の総負荷
  • トレーニングカード




また、教育タスク「ジョブ」用の特別なカードを作成することもできます。これには、教師に配布されないすべての授業時間が含まれます。



レポートは、Excelを参照しないc ++ライブラリであるxlsxファイルで生成されます。











技術的な詳細



  • DBMS Sqlite
  • GUI Qt
  • C ++言語
  • C ++でxlsxレポートを生成するためのライブラリがここにありましたが、著者は気にしませんでした




サードパーティのライブラリも使用して、QTableViewクラスで階層化されたヘッダーを作成しました。



おわりに



この学年にプログラムが使用されましたが、その改善を希望するお客様がおり、ほとんどの場合、休暇後に実施されます。
