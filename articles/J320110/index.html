Windows Performance Stationまたはコンピューターに効率的に動作するように教えた方法





親愛なる読者の皆さん、こんにちは！ それで私はHabrに行き、ここで私のプロジェクトの話をしたいと思います。 問題の組織の企業秘密を侵害しないように、彼ら自身の経験と仕事の多くの側面をかなり一般的な用語で説明する必要があります。



このストーリーの本質は、Windowsでのコンピューターのパフォーマンスの問題を解決するために見つかった方法を共有し、このソリューションをユーザーごとに完全に普遍的かつ個別にしたアプローチについて話すことです。 コードを記述し、最も効果的なメカニズムを選択するのに約2年の毎日の作業が必要でした。また、さまざまなIT分野の多くの同志からのアドバイスと支援もありました。



ですから、この話は2013年に始まりました。その時、私は粉末とシャンプーを生産する1つの大きな国際組織で働きました。 私の立場はシステム管理者でしたが、よくあることですが、本質的に私は「タイコプログラマー」でした。 作業の本質は、いくつかのEDMSシステムを維持し、VBAでさまざまなマクロを作成し、さまざまな単純なシステムを自動化または統合するためのユーティリティなどでした。



それと同時に、私は組織の緊急問題の1つ、つまり、多くの従業員の新しいラップトップでのさまざまなソフトウェアの不十分な作業を解決していました。 実際、大企業の多くのシステム管理者が遭遇する問題に対処しなければなりませんでした。スタッフのコンピューターでは、お気に入りのMS Office、SAP、1Cなどに加えて、すべてのIT専門家に愛される「mcafee」、「punto switcher」 「、」「メールサテライト」など。 この会社が従業員のコンピューターに対するリベラルな態度を促進し、実際にBYODを生み出したことを明確にする価値があります。



以下を含むパフォーマンスに関する苦情が寄せられています。 そして、私にとっては、Process Lassoに似たパフォーマンス調整用のアプリケーションを探し始めました。 私の選択は、「Bill2のプロセスマネージャー」に落ちました。 当時、私は多くの代替手段を見つけましたが、それらはすべて、ひどく過負荷のインターフェースを備えており、最適化によって解放されるよりも多くのリソースを自分の作業のために消費していました。









当時見つかった「Bill2のプロセスマネージャー」には、システムツールよりも多くの利点がありました。 管理と設定が非常に簡単でした。 実際、タスクマネージャーでできることと同じこと、つまりアプリケーションの基本優先度とカーネルマスクを設定できましたが、マネージャーとは異なり、再起動後にこれらの設定を保存し、作業中のウィンドウに表示されるアプリケーションを区別することができましたデスクまたはバックグラウンドで作業します。



このアプリケーションのおかげで、私は最も差し迫った問題の1つを解決し、現在使用しているアプリケーションと、私にとってより重要なもの-EXCELまたは更新プログラムのバックグラウンドインストールをコンピューターに説明することができました。



しかし、私にとってこのアプリケーションは主な問題を解決したという事実にもかかわらず、多くの欠点がありました。



  1. 構成の連続性がないため、同じ構成のコンピューター間でアプリケーションをコピーするだけでなく、再度構成する必要がありました。 この問題は、設定されたアプリケーションをポータブル実行可能ファイルにパックすることで部分的に解決されました。 （おそらく何らかの形でレジストリからパラメータを取得することは可能だったかもしれませんが、その時点では理解できませんでした）。



  2. RAM制御の欠如、つまりRAM圧縮。 多くの調整機能には、いわゆる「メモリの最適化」がありますが、これは本質的にナンセンスです。 RAMの最適化は、パフォーマンスの点ではまったく意味がありません。 RAMにハングアップしてSWAPに分類されないデータをクリーンアップする必要があったため、RAMに急激な負荷がかかると既に解放され、OSはRAMに必要なものとそうでないものを推測しようとしませんでした。 ちなみに、1年後、この機能はWindows 10に定期的に登場しましたが、RAM内にアーカイブされているため、本質的に問題を完全に解決することはできません。



  3. ハードディスク（またはSSD）への入出力の優先順位の制御の欠如。 通常、WindowsはVistaからこの機能を備えていますが、Win10でのみ積極的に使用されるようになりました。



  4. 「焦点」アプリケーションの定義の欠如、として 1つがコンパイルされ、もう1つがYouTubeを視聴する多数のアプリケーションで作業している場合、ウィンドウがすべてではありません。ウィンドウが他のウィンドウではなく、それにフォーカスしている場合、YouTubeが遅れることはありません。



  5. 新しいインストール済みプログラムにルールを一度に適用するためのアプリケーションを含むフォルダーを指定できない。


実際、Windowsのアーキテクチャを研究して、理想的には1つの「BUT」を提供するだけで、システムのパフォーマンスを理想的に構築できる多くの優れたメカニズムを見つけました。 しかし、ソフトウェアを作成するプログラマーがオペレーティングシステムに、ソフトウェアの機能、目的（ゲーム、映画など）、およびCPU / RAM / HDDにアクセスする優先順位を伝える場合。







人生では、Windowsのソフトウェアアーキテクチャを考慮して、ソフトウェアのごく一部しか書かれていないことが判明しています。 非常に深刻で大規模で、これらのパラメーターを指定せずにソフトウェアを作成し、OSはデフォルトですべてを使用します。



また、アクションのためにイベントが到着するとすぐに、利用可能なリソースを完全に恥知らずに食い尽くすサービス（更新プログラムのダウンロードやインストールなど）は言うまでもありません。 PCで状況を確認する場合は、タスクマネージャーを開き、プロセスのタブに「基本的な優先順位」列を表示できます。



MMCSSはこの状況を少し修正しますが、実際のパフォーマンスにほとんど影響を与えることなく、プロセスにあまりにも緩やかに影響します。









Windowsのマルチタスク処理で最も不快なことは、常に私を悩ませていましたが、リソースを大量に消費する操作中に音楽を聴くことでした。 なぜなら 仕事中、Chromeで音楽を聴き、Excelで大きなテーブルの計算を開始しました。CPUが完全にロードされたときに正確にフリーズするか、このロードのジャンプで絶えず喘鳴するという特性がありました。 この状況は、「Bill2のプロセスマネージャー」を使用してほぼ完全に解決されましたが、うなり声が出ないように手動でChromeに設定する必要があり、音楽が不要になったときでもChromeが常に優先順位が高いことが判明しました。



また、ラップトップの標準によってコンピューターが非常に強力に使用されたことを明確にすることも価値があります。 当時は、Core i5 2430m / 8GB Ram / 256GB SSDでした。



同時に、BYOD会社の設立当初は、職場のラップトップを使用する従業員は、 自宅から、同時に、作業中のPCを自宅のPCとして使用して、映画、ゲームなどを視聴します。









これらのすべての要因により、最終的には独自のアプリケーションを作成するようになりました。これは、利用可能な代替案の最高のものを吸収し、それらの欠点を排除しますが、タスクマネージャーとしてのチューニングのためのインターフェイスではなく、現在のパフォーマンスに適応するための完全自動システムタスク。 実際、PCでBYODに理想的な構造を作成しようとしていました。この構造では、Steam、iTunesなどのプロセスを遅くすることなく作業できます。 再生、企業のソフトウェア、ウイルス対策などの影響を受けずに映画を見る



この問題を解決するために、ITで働いていた友人を何人か引き付ける必要がありました。 最初のアイデアは、私たち自身の便宜のためにアプリケーションを作成し、OSとさまざまなアプリケーションに関する私たち自身の知識を広げることでした。









最初は「Bill2のプロセスマネージャー」の簡単な代替案を作成しましたが、構成インターフェイスはありませんでした。そのため、アプリケーションは事前構成された構成で動作しました。 主な機能は、どのウィンドウを最大化するか、どのプロセスをバックグラウンドで動作させるか、どのアプリケーションに焦点を合わせるかの定義でした。 また、プロセスだけでなく、特定のフォルダーのすべてのプロセスにも優先順位を割り当てるオプションをすぐに追加しました。 このアプリケーションの作成は非常に簡単で、経験のおかげで、メモリリーク、リークハンドラ、クラッシュ、その他のバグを防ぐことがすぐに判明しました。 アプリケーションは、再起動せずに長時間バックグラウンドで動作する必要がありました。 アプリケーションを簡単にし、効率的に動作させるために、標準のOSおよびWinAPI関数のみを使用しました。 当初は、インターフェイスなしでも実行しました。









次のステップは、入力パラメーターを使用してアプリケーションの目的を決定するローカル学習アルゴリズムを作成することでした。 しきい値伝達関数に基づくニューロンを含むニューラルネットワークの数学的モデルが基礎として採用されましたが、その時点では重みは考慮されていませんでした。 このニューラルネットワークは最初に、アプリケーションの動作時間、その名前、構成のルール、およびハードディスク上の場所の基本パラメーターのみを調査しました（たとえば、多くのアプリケーションが常に使用され、バックグラウンドで動作しない場合、それらは「C：/ Games / 、その後、このフォルダから起動されるすべての後続アプリケーションがアクティブに使用される可能性が高く、バックグラウンドで優先度を下げるべきではありません。



このアルゴリズムを完成させるのにほぼ半年かかりました。 その後、分析用の入力パラメーターの数が増加し、サードパーティのアプリケーションに適用される最適なルールが選択されました。



ここでは、従来のタスクディスパッチャーとは異なり、優先度、コア、RAM、およびI / Oを個別に管理するのではなく、これらのパラメーターをさまざまな割合で組み合わされたルールに結合することを明確にする価値があります。 たとえば、タイトルアプリケーションの場合、ルール= FOREGROUND（すべてのコアが利用可能、優先度は平均を上回り、I / O優先度は高く、SWAPのRAMはリセットしません）。









さらに、ウィンドウ位置のパラメーターが学習アルゴリズムに追加されました（アプリケーションにウィンドウが表示されているかどうか、最小化されているか最大化されているか、フルスクリーンに拡大されてフォーカスされている場合、背景などよりもこのプロセスに注意を払う必要があります）。



サービス定義が追加されました 同じプロセスと呼ばれるsvchost.exeサービスプロセスは、まったく異なるタスクを実行します。



おそらく最も難しいのは、異なるプロセスで同じプロセスを区別するようにアプリケーションに教えることでした。 たとえば、多くのブラウザーは、個別のタブ（Chrome、Firefox、Yandex、Operaなど）ごとに個別のプロセスを開始し、サウンドとビデオはホストプロセスを再現します。したがって、20のクロムプロセスから、焦点が合っていない場合、それらのプロセスを増やす必要があります音を再現します。 バックグラウンドで動作する他のプロセスにも同じことが当てはまります。







設定で世界に知られているすべてのプロセスを登録しないために、最終的にアプリケーションは、ユーザーがプロセスで作業する頻度だけでなく、プロセスの実際の目的と作業の重要性も決定することを学びました。



したがって、アプリケーションは、ユーザーが現在作業している内容に応じて、主要なコンピューターノード間で負荷を分散することを学習しました。 さらに、アプリケーションの使用シナリオに応じて、各PCに個別に負荷が分散されます。 ウィンドウで使用されるかどうか、ビデオ/サウンド/ 3Dを再生するかどうか、ユーザーがこのアプリケーションにアクセスする量、このアプリケーションがどのフォルダーにあるか。



アプリケーションはシンプルでユニバーサルであることが判明し、インストールは数回のクリックで行われ、操作に設定は必要ありません。すべてが自動的に適応します。







このすべてのロジックは、システムとの密接な統合なしには存在できなかったため、すべてのメカニズムを実装するには、Windows API関数を最大限に使用する必要がありました。 これにより、計算された電力の最小コストで、深いコア機能にアクセスし、x86-64プロセッサー上のPCとの最大の互換性を維持することができました。



ユーザーが使用しているコンピューターの種類を考慮することも重要でした。 たとえば、バッテリー電源で作業する場合、CPUコアで作業するためのわずかに異なるアルゴリズムが使用されて、負荷が軽減され、電力が節約されます。 電源管理オプションが追加されたため、ノートパソコンやタブレットの電源をオフにすることはできませんが、画面をオフにした後も、スリープモードに近い電力消費で音楽のダウンロードやコピー/アーカイブを続けます。



また、プロセッサがマルチスレッドでどのように機能するかを判断する方法と、コアに負荷を分散する方法を学ぶことも重要でした。



これらのアルゴリズムはすべて、微調整やオーバークロックではなく、OSや他のソフトウェアの機能を制限しない、真に普遍的なソリューションの作成に役立ちました。 実際、このアプリケーションは、経験豊富な管理者が特定のタスクごとに行うように、ユーザーのリクエストを理解し、主要ノードの負荷を管理するようにコンピューターに教えます。
