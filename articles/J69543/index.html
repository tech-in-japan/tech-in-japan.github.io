クラウドの準備はできましたか？ 技術者の74％が3台以上のコンピューターを使用

主にクラウド内の個人データの安全性とセキュリティについて懐疑的であるため、ITワーカーはクラウドの主な敵です。 新しいTechRepublicの調査では、ITプロフェッショナル自身がクラウドを最大限に活用できるユーザーであることが示されています。



9月の第1週に行われた調査では、TechRepublicの回答者の74％が日常業務で3台以上のコンピューターを定期的に使用していると回答しました（下図を参照）。 1台のコンピューターで作業していると答えたのはわずか6％でした。 調査の説明では、TechRepublicのメンバーは、問題の解決に協力している他の人のコンピューターではなく、作業するコンピューターのみを示すように求められました。



以下は、TechRepublicメンバーが複数のコンピューターを使用する方法と理由についてのコメントです。

  • 「同時に動作する6つの異なるシステムがあり、それらのほとんどが仮想システムを実行しています」（ HAL 9000 ）

  • 「いくつかのテストシステム、2つのメインシステムと2つのLinuxシステム、およびデスクトップサーバー（デスクトップベース）があります。 自宅に1台のデスクトップ（実際には3台ですが、ほとんど使用していません）と2台の古いラップトップを持っています。 労働者がいる。」（ ザ・スカムミー・ワン ）
  • 「開発者として、自宅や外出先で作業するためのラップトップ、開発用のデスクトップ、多くの仮想マシンを備えたテストPC、およびサービスと古いアプリケーションを備えた複数のPCを持っています。」（ Nfrost ）
  • 「システム管理者としての日々の仕事のために、WindowsとLinuxを備えたデスクトップを持っています。 自宅にはLinuxデスクトップがあり、17 '' Windowsでポータブル、9 ''ネットブックです。」（ Geromyh ）
  • 「私は、仮想化とデュアルブートの大騒ぎを避けようとし、Windows 2000、Windows XP、Windows 7、およびさまざまなLinux実験に別々のシステムを使用することを好みます。 KVMスイッチを使用してそれらを切り替えます。 また、ラップトップとワークショップコンピューターを使用することもあります。 Windows Home Serverはコンピューターと見なされますか？」（ John3347 ）
  • 「職場で3人、自宅で2人。 昔のグリーンスクリーンでは、12台の端末がありました！」（ Ryoung ）
  • 「デスクトップとラップトップ、さらに特別な注意を必要としないホームサーバー。 ただし、すべてのコンピューターはマルチブートであるため、1週間に10以上のオペレーティングシステム（WindowsおよびLinux）とやり取りできます。」（ Dogcatcher ）
  • 「私のBlackBerryはPCと見なされていますか？」 私は彼とほとんどすべてをやります。」（ テストラダス ）


クラウドコンピューティングにより、これらの技術は、スクリプトや共有フォルダーに煩わされることなく、複数のコンピューターからファイルやアプリケーションに簡単にアクセスできます。 実際、さまざまなブラウザーやコンピューター間でブックマークを同期するXmarksのような小さなクラウドアプリケーションでさえ、あるマシンから別のマシンにジャンプするワーカーにとって非常に価値があります。



したがって、Xmarks、 Google Docs 、 Evernoteなどのクラウドサービスで忙しくなり、重要でないファイルやデータ専用にそれらを使用する技術専門家の数が増える可能性が非常に高いと思います。



ただし、クラウドコンピューティングゲームにより、ITワーカーは、信頼できるエンタープライズデータやアプリケーションにクラウドを使用することに近づくことができますか？ これはまだ難しい仕事です。 TechRepublic Osiyo53メンバーの1人が書いたように 、「開発、材料、アプリケーションなどを維持します。 「クラウドのどこか」？ これはありそうもない。 アイデア、知識、成果が盗まれ、自分のものとして売られたとき、私たちはすでに不快な状況にありました。 一番下の行は、私たちが直接自分自身を制御しないシステムを信頼しないということです。」
