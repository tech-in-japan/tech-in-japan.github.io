WindowsでisoイメージからLinuxをインストールする

この記事は、WindowsマシンからLinuxディストリビューションをハードドライブのisoイメージから直接インストールするのに役立ちます。 この記事は、ネットブックの所有者だけでなく、何らかの理由で通常のインストールディスクを使用できない人にも役立ちます。



Grub4Dosをインストールする



開始するには、Grub4Dosをダウンロードしてください。 執筆時点では、最新の安定版リリースはGrub4Dos 0.4.4でした。



勝利XP / 2000



  • grldrとmenu.lstを、Windowsがインストールされているディスクのルートにアーカイブから展開します。
  • boot.iniを編集します 。 ファイルを保存するには、「読み取り専用」属性を削除する必要がある場合があります。 

      attrib -r boot.ini
  • ファイルの最後に、追加します 

      C：\ grldr = "Grub4Dos"


Vistaに勝つ/ 7



  • grldr 、 grldr.mbrおよびmenu.lstをアーカイブからWindowsがインストールされているディスクのルートに解凍します。
  • 管理者としてコンソールを起動し、実行します 

      bcdedit / create / d "Grub4Dos" /アプリケーションブートセクター
  • 前のコマンドに応答して、作成されたレコードのIDを取得します。これは、次の3つで置き換えます 

      bcdedit / set {id} device partition = C：
 bcdedit / set {id} path \ grldr.mbr
 bcdedit / displayorder {id} / addlast


Grub4Dosの詳細なインストール手順については、 こちらをご覧ください 。



menu.lstの編集



ブートローダーメニューに新しいアイテムを追加する 



タイトルブートインストーラー
フォールバック1
ルート（hd0,1）
カーネル/ vmlinuz vga = normal ramdisk_size = 14972 root = / dev / rd / 0 rw-
 initrd /initrd.gz


ここでは、ディスク番号とパーティション番号をあなたのものに置き換えます。 Ubuntuがインストールされている場合、 vmlinuzおよびinitrd.gzの必要なイメージをダウンロード （ amd64 ）できます 。 その他の場合、それらはディストリビューション自体のiso-imageにあります。 参考のために、私は異なる分布の名前の小さなテーブルを与えます

Ubuntu vmlinuz initrd.gz
Debian vmlinuz initrd.gz
フェドラ vmlinuz initrd.img
マンドリバ vmlinuz all.rdz
openSUSE Linux initrd
Slackware bzImage initrd.img
ジェンツー ジェンツー gentoo.igz


次に、配布パッケージのiso-imageを結果のヒープにコピーし、再起動します。



設置



おそらく、インストールはディストリビューションによって多少異なります。 例としてUbuntuを使用したインストールを見ていきます。



再起動後、ダウンロードマネージャーに新しいアイテムが表示されます。 選ぶ





Grub4Dosブートローダーが表示されます。 「ブートインストーラー」を選択します





インストーラーは言語の選択を促します。 私たちは偉大で力強い





おそらく、インストーラーは独立してイメージを見つけることができないでしょう





この場合、メインメニューから検索を再開し、場所を手動で指定する必要があります



画像を選択すると、いくつかの使い慣れたウィンドウが表示され、インストールが開始されます。



menu.lstを編集します。 オプション＃2



別のmenu.lstも可能です。 



 タイトルブートインストーラー
フォールバック1
 find --set-root / image_name.iso
 map / image_name.iso（0xff）|| マップ--mem / image_name.iso（0xff）
マップ-フック
チェーンローダー（0xff）
 savedefault --wait = 2


ここでは、追加のジェスチャーは必要ありません-おなじみのUbuntuインストーラーがロードされます。





イメージはハードドライブからロードされるため、インストールを開始する前にOSとスワップのセクションを選択することをお勧めします。
