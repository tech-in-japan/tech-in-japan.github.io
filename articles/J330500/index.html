Brocadeイーサネットスイッチ

ストレージスイッチのメーカーとして最も知られているBrocadeは、イーサネットスイッチも製造しています。 このようなスイッチの特定のセット（ICXおよびVDX）が手元にあり、簡単にレビューして、興味深いと思われる点に注意したいと思います。 マーケティングの言語と数字を避けるようにしますが、興味がある人のために、メーカーのウェブサイトへのリンクを示します。 始めましょう:)



ICXスイッチ



これらのスイッチは、エンタープライズキャンパスネットワークで使用するために設計されています。 ICXファミリは、アクセス回線（7150/7250/7450）、集約（7450）、および集約/コア（7750）で構成されています。









ICX 7150は、ICXファミリーに追加された最新の最新製品です。 24および48ポートの標準モデルに加えて、このラインにはコンパクトな12ポートスイッチ（7150-C12P）と802.3bzマルチギガビットイーサネットポート（7150-48ZP）のスイッチがあります。 12/24/48ポートのモデルには受動冷却がありますが、PoEを備えた24/48ポートのモデルは能動冷却を使用しなくても機能します。









ICX 7250-24および48ポートの標準モデル。 主な違いは、8つのアップリンク/スタッキングポートです。









ICX 7450-24および48ポートの標準モデル。 スイッチには、アップリンク/スタッキングポートが組み込まれていませんが、追加のインターフェイスモジュール（1GE、10GE、40GE）用の3つのスロットがあります。 また、このラインには、802.3bzマルチギガビットイーサネットポート（7450-32ZP）と集約スイッチ7450-48F（48x1GE SFP）を備えたスイッチがあります。









ICX 7750-集約/コア。 3つのモデル（48x10BASE-T、48x1 / 10GE SFP / SFP +、26x40GE QSFP）を使用でき、すべて6x40GE QSFPポートへの追加モジュールのインストールをサポートしています。 すべての40GEポートはブレークアウトモードをサポートしています（40GEポートを4つの10GEポートに分割）。



ICXラインのスイッチは、このようなポジショニングのための一連の標準テクノロジーをサポートしていますが、機能の明らかなギャップは認識されていません。 7150/7250/7450シリーズでは、PoE +に加えて、Power over HDBaseT（ポートあたり最大90ワット）をサポートするモデルがあります。



スイッチ管理および監視ソフトウェア-Brocade Network Advisor（WindowsまたはLinux、別のサーバー、または仮想マシン上で実行）。



機能から注意したいこと：



  • スイッチのスタック。 すべてのICXスイッチは、アップリンク/スタッキングポートを介したスタッキングをサポートしています。ライン内でスタックを構築できます（たとえば、7450を備えた7250は構築できません）。 スタック内のスイッチの最大数は12です。スタッキングには、通常の10GEまたは40GEポートが使用され、最大スループットは480ギガビット/秒です。 スタックの接続の最大長は10 km（長距離スタッキング）です。 リングトポロジとリニアトポロジの両方がサポートされています。



  • フォールトトレランスを提供するためのオプションとしてスタッキングを検討していない人のために、7750は代替案-マルチシャーシトランキング（別名MC-LAG）を提供します。 つまり 1組のスイッチがクラスターに組み込まれ、クラスターに接続されたデバイスは1つの論理デバイスとして認識されます。



  • ライセンス。 7150/7250/7450の場合、メインライセンスはポートオンデマンド（10GEポートのライセンス）およびL3機能（ダイナミックルーティングプロトコル、PIM、PBR、VRRPなど）のライセンスです。 7750にはライセンスが1つだけあります-L3-アクティベーションは不要です。



  • スイッチOSイメージに関しては、スイッチイメージとルーターイメージの2つのタイプがあります。 各スイッチには2つのフラッシュ（プライマリとセカンダリ）があり、デフォルトでは両方のタイプのイメージが含まれています。



  • OCイメージとL3ライセンスの組み合わせにより、機能セットに次のオプションが提供されます。レイヤー2（ライセンスなしのスイッチイメージ）、基本レイヤー3（ライセンスなしのルーターイメージ）、フルレイヤー3（ライセンス付きのルーターイメージ）。 たとえば、静的ルーティングが必要な場合は、基本的なレイヤー3で十分です。



  • サポートされている機能は、 機能と標準サポートマトリックスに簡単にリストされています 。 これはモデル文書であるとさえ言えます。



  • Brocadeが仮想ケーブルテストと呼ぶ、時間領域反射率計の機能。



  • 7150は新しいラインであり、データシートの一部には、将来のソフトウェアリリースでサポートされる機能というラベルが付けられています。 7月には、ソフトウェアリリースが計画されています。このリリースでは、とりわけ、スタッキングとL3機能が予定されています。



次に、ICXに基づいて組み立てられるものを見てみましょう。 エンタープライズキャンパス向けの通常の2/3層アーキテクチャ（スタンドアロンスイッチ+マルチシャーシトランキング）に加えて、2つのソリューションが提供されています。



最初に、スタッキングを使用し、オプションで長距離スタッキングを考慮に入れると、次のようになります。









スキームは、コメントを必要としないと思います。



2番目のソリューションはCampus Fabricと呼ばれます。 実際、これはICXスイッチでの802.1BR標準（ブリッジポート拡張）の実装であり、7750は単一制御点（コントロールブリッジ-CB）として機能し、残りのスイッチは外部ラインカード（ポートエクステンダー-PE）として機能します。









そのようなアーキテクチャの利点は何ですか？



  • これは、ポート容量を簡単に増やすことができる分散シャーシです。

  • すべてのリンクがアクティブです。

  • 集中管理プレーンとコントロールプレーン。
  • L3ライセンスは、必要に応じてControl Bridgeでのみ必要であり、L3ライセンスを節約できます。


しかし、物議を醸す点があります：



  • 集中制御プレーンは、次のような欠点と見なすことができます。 ファクトリ全体が1つの障害ドメインです。



  • すべてのトラフィックは、工場の中心（コントロールブリッジ）を通過します。 例外はマルチキャストおよびブロードキャストレプリケーションであり、これらはポートエクステンダーでローカルに実行されます。



  • ポートエクステンダーは、アクセスポートでLAGをサポートしません。



スケーリング/冗長性については、次のことに注意してください。



  • 7750に基づくコントロールブリッジは4（最大）スイッチのスタックであり、推奨されるスタックトポロジはリングです。 少なくとも2つのスイッチを使用します。 Control Bridgeの予約が絶対に必要です。



  • 7450および7250をポートエクステンダーとして使用できます。7150のサポートは後日提供されます。



  • PoDライセンスがまだ必要です。



  • ポートエクステンダーの最大数は36です。スイッチスタックは個別のスイッチとしてカウントされます。



  • ポートエクステンダーをチェーンで接続する機能-最大6台のデバイス。



そのようなソリューションでネットワーク全体を構築する価値があるかどうかは重要ではありませんが、アクセスソリューションとしては、このオプションには生命権があります。



VDXスイッチ



折りたたまれたClosネットワークがどのように見えるかを思い出させてください：









リーフスイッチはサーバー接続を提供し、スパインスイッチはリーフスイッチに冗長な高速接続を提供します。 写真は、このようなネットワークの3ステージ（リーフ-スパイン-リーフ）バージョンを示していますが、ブロケードでは、ドキュメントで5ステージ（リーフ-スパイン-スーパースパイン-スパイン-リーフ）バージョンも考慮しています。



VDXファミリでは、6740と6940の2つの行について説明します。









VDX 6740-48x10GE（10GBASE-TまたはSFP +）+ 4x40GE QSFPポートを備えたリーフスイッチ。









VDX 6940-リーフ/スパインスイッチ。 6940-36Q-36x40GE QSFPポート。 6940-144S-96x10GE SFP +および12x40GE QSFPポート（3つの40GEポートを1つの100GEポートに結合してQSFP28トランシーバーを使用できます）。



これらのスイッチは、データセンターで使用するために配置されており、対応する特性があります。



  • すべての40GEポートはブレークアウトモードをサポートしています（40GEポートを4つの10GEポートに分割）。

  • フロントツーバックおよびバックツーフロントの両方の冷却がサポートされています。

  • 2つの電源。

  • FCoEおよびDCBのサポート。

  • 自動化ツールのサポート、OpenStackおよびVMware製品との統合機能。

  • VXLANサポート。



10GE / 40GE速度のデータセンターネットワークが少し時代遅れであることは否定しませんが、Brocadeの25GE / 100GEスイッチは別のファミリー（SLX）にあり、それらは私たちに届きませんでした。



VDXスイッチの機能の中で、次のことに注意してください。



  • ライセンス。 ライセンスには、ポートオンデマンド（10GEおよび40GEポート用）とFCoEのライセンスの2種類があります。 PoDライセンスは自動的にアクティブなポートにバインドします。 つまり スイッチを箱から取り出してオンにし、ライセンスでアクティブ化されるポートのグループをドキュメントで確認する必要はありません。トランシーバーをインストールするだけです。 ライセンスを無料のライセンスプールに戻すには、CLIでいくつかのコマンドを実行する必要があります。



  • これらのスイッチは、TrillベースのイーサネットファクトリーであるVCS Fabric（最大48台のスイッチ）をサポートしています。 TRILLベース、なぜなら TRILLからはデータプレーンのみが使用され、コントロールプレーンは独自のプレーンを使用します。 事柄はトポロジーにとって気取らず、構成と保守の両方で便利です。



    通常、VDXスイッチは常に論理シャーシモードにあります。 ほとんどすべてのスイッチは、1つのスイッチからのVCSファクトリです。 そして、なぜなら VCSファクトリ内では、構成は自動的に保存されます。VDXスイッチでは、コピーランスタートアップの類似物は見つかりません。



    VCSは基本的な機能ですが、VCSファクトリの使用はオプションであり、無効にできます。



  • 2つのフラッシュですが、ICXとは異なり、フォールトトレランスとソフトウェア更新の利便性のためにあります。 オプションによるOSイメージの分離はここにはありません。



  • サポートされる機能は、 機能と標準のサポートマトリックスに引き続きリストされています 。 この点でドキュメントを作成するアプローチは、ICXと変わりません。



  • なぜなら Brocadeであるため、イーサネットスイッチでいくつかのSANスイッチテクノロジーを使用できることが期待されています。 たとえば、これらはMAPS（高度な監視）およびVCS工場の技術の一部です。



  • 10GEおよび40GE上のすべてのブランドのDAC（銅線直接接続）がアクティブです。



建築



VCSファクトリーに基づいた設計に加えて、BrocadeはもちろんIPファクトリーに基づいた設計を提供します。 これは、L2 / L3境界がリーフスイッチ上にあるClosネットワーク（3ステージまたは5ステージ）であり、リーフスパインとECMPの接続が動的ルーティングプロトコル（通常BGP）を提供します。



普及したeBGPを備えたIPファクトリ（すべてのタイプの接続にeBGPを使用）。









iBGPを使用したIPファクトリ（eBGPはスーパースパイン<->スパインおよびスーパースパイン<->エッジリーフ接続にのみ使用され、iBGPはスパイン<->リーフのDC POD内で使用されます）。









どちらの場合も、サーバーを2つのリーフスイッチに接続する（マルチホーミング）ために、2つのスイッチのVCSファクトリ（vLAGペア）が使用されます。



そして、VXLANを忘れないでください。 BrocadeはデータプレーンとしてVXLANを使用した仮想化をサポートし、EVPNはコントロールプレーンとして使用できます。 そこで、IPファクトリーの設計を取り入れ、EVPN VXLANを追加し、オプションを取得します。

およびiBGPを使用







マルチホーミングでは、IPファクトリと同じメカニズムが使用されます。



Brocadeは、iBGPまたはeBGPの使用を主張していません。両方のオプションは同等です。 Brocadeは、VXLAN EVPNの統合ルーティングとブリッジングの両方のオプション（非対称および対称）をサポートしています。 私が見たこれらのEVPN VXLAN実装のうち、VDXは最も完全に機能するものの1つです。



もちろん、VDX以外にも、Brocadeデータセンター製品は他にも多数ありますが、この記事では取り上げていません。
