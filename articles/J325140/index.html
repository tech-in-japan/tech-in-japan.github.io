ソフトウェア工学研究所は、標準設計コードC ++へのアクセスをオープンしました。

ソフトウェア開発のベストプラクティスを公式化する米国連邦研究センターであるソフトウェア工学研究所（ SEI ）は、2016年にC ++プログラミング標準をリリースしました。 C ++でコードを記述するためのルールについて説明します。これにより、開発者は、脆弱性のない安全で信頼性の高い安全なシステムを作成できます。



標準自体は、少し前にリリースされたCコード標準への追加でした。 そして先週、ソフトウェア工学研究所は、CおよびC ++での安全なシステムの開発に関する貴重な専門知識への無料アクセスを開放しました。



/ Flickr / jeff dlouhy / cc



C ++プログラミング標準は、その内容における数十年の研究を反映しており、C ++言語の機能を考慮した83の新しいルールが含まれています。 また、信頼できるプログラムの作成を支援するために設計されたアルゴリズムについても説明します。



「新しい標準はCの標準の拡張です。たとえば、オブジェクトとコンテナを操作するためのガイドラインを提供します」とCEIのCERTテクニカルマネージャーであるRobert Schiela氏は述べています。 「ラムダ式を操作するための推奨事項も含まれています。」



CEIのCERTのCTOであるマークシャーマンは、次のように付け加えています。「SEI C ++コード記述標準は、安全なソフトウェアを開発するための他の3つのガイドの一部です。 このフォーマットへのアクセスを開いて、学校で使用したり、専門的なマニュアルや仕様などを書いたりするときに使用できるようにします。
