タグ#AndroidDevによるキーパーソン



Androidアプリケーションを開発しているほとんどのプログラマーは、最新のAndroidコミュニティーのニュースを最新にしていないことにしばしば気付きます。 ほとんどすべての人がOSの新しいバージョンの外観について知っている場合、開発者にとって興味深いさまざまな便利なライブラリまたは記事のリリースがしばしば気付かれなくなります。



過去1年間、私はソーシャルの連絡先のリストを作成しました。 Androidコミュニティの最も有名な人格のネットワーク。 これらの人々は定期的に記事を書いたり、ライブラリを公開したり、StackOverflowに応答したりしています。 このリストは、Androidデベロッパーコミュニティのニュースを初めて知った人の1人に役立つので、ここで共有することにしました。



Google開発者


  1. Romain Guy （ Twitter 、 Google+ 、 Github ）-主にUIとアニメーションで動作します。 彼は、Google I / O、Devoxxなどの大規模な会議で頻繁に講演しています。
  2. Reto Meier （ Twitter 、 Google+ ）-GoogleのAndroidデベロッパーアドボケート。 彼は本を書いたり 、会議で講演したり 、アプリケーションのビデオレビューをしたり、StackOverflowに頻繁に反応します。
  3. ローマン・ヌリク （ Twitter 、 Google+ 、 Github 、 GoogleCode ） -Googleの Android Developer Advocate、UIデザイナー。 彼はカンファレンスで講演し、オープンソースでさまざまなカスタムコントロールをリリースすることが非常に多く、 Android Design in Actionを毎週実施しています。
  4. Chet Haase （ Twitter 、 Google+ ）-Androidのグラフィックとアニメーションで動作します。 それ以前は、サンとアドビで働いていました。 Google I / O、Devoxx、およびその他の主要な会議で定期的に講演。
  5. Kirill Grouchnikov （ Twitter 、 Google+ ）-AndroidのUIの作業。 有名なブログPushing pixelの著者。 Androidチームに参加する前は、Swingに取り組んでいました。
  6. ニック・ブッチャー （ Twitter 、 Google+ ）-Android Developer Advocate。 UI / UXに従事。 Android Design in Actionのクリエイターの一人。
  7. Xavier Ducrohet （ Twitter 、 Google+ ）-Android SDK /開発者ツールテクニカルリード。 彼は現在、Gradleに基づくAndroidプロジェクトの新しいビルドシステムに取り組んでいます。
  8. リチャード・ ハイドマン （ Twitter 、 Google+ 、 GoogleCode ） -Android Developer Advocate。
  9. Adam Powell （ Google+ ）はAndroidのエンジニアの1人です。 彼はGoogle I / Oでdokaladyと共演しました。
  10. ダイアン・ハックボーン （ Google+ ）-Androidフレームワークエンジニア。 Fragments APIを開発しました。 Google+で興味深い技術記事を執筆することもあります。


サードパーティ開発者


  1. Jake Wharton （ Twitter 、 Google+ 、 Github ）は、オープンソースAndroidの世界で最も有名な開発者の1人です。 使用される多くのライブラリの作成者（ ActionBarSherlock 、 NineOldAndroids 、 ViewPagerIndicatorなど）。 彼は、非常に広範なGithubプロファイルを持つ企業、Squareで働いています 。
  2. Jesse Wilson （ Twitter 、 Google+ 、 Github ） -Googleで働いていましたが、現在はSquareで働いています。 人気のあるAndroid DI Daggerフレームワークのメイン開発者。
  3. Eric Burce （ Twitter 、 Google+ 、 Github ） はもう 1つのSquare開発者です。 githubでアクティブになり、いくつかのSquareライブラリにコミットします。
  4. Juhani Lehtimaeki （ Twitter 、 Google+ ）-Android UI / UXパターンに関する有名なブログを管理しています。 最近、このテーマに関する本を書きました。
  5. Kevin Sawicki （ Twitter 、 Github ）-Githubで動作します。 Github Androidクライアントおよび他のいくつかの一般的なアプリケーション/ライブラリの作成者。
  6. Cyril Mottier （ Twitter 、 Google+ 、 Github ）は、 ポラリス 、 グリーンドロイドなどを含む、いくつかの有名なオープンソースプロジェクトの著者です。
  7. Mark Murphy （ Twitter 、 Github 、 StackOverflow ）は、 The Busy Coder's Guide to Android Developmentの著者です。 彼はしばしばStackOverflowに答え、Githubに多くの便利なコンポーネントとクラスを投稿しました。
  8. Diego Torres Milano （ Google + 、 Github ）は、 Android Application Testing Guideの著者です。 彼は、Androidアプリケーションのテストについてブログにたくさん書いています。
  9. Steve Kondik （ Twitter 、 Google + 、 Github ）はCyanogenModの創設者であり、主要な開発者です。
  10. Chris Banes （ Twitter 、 Google+ 、 Github ）は、人気のFriendcasterアプリの開発者です。 一般的なカスタムUIコンポーネント（ PullToRefresh 、 PhotoView ）を作成しました。
  11. Alexandre Thomas （ Twitter 、 Github ）は、最近リリースされたAndroidKickstartrプロジェクトの作成者です。
  12. Roberto Tyley （ Twitter 、 Google + 、 Github ） -Android GitクライアントAGitの著者であるGithubで働いていました。
  13. Mark Allison （ Twitter 、 Google+ 、 Github ）は、 Styling Androidブログの著者であり、さまざまなインターフェース要素のスタイルのカスタマイズと変更について書いています。
  14. Lars Vogel （ Twitter 、 Google+ 、 Github ）は、膨大な数の記事と数冊の本の著者です。 StackOverflow以外の検索クエリは、しばしば彼のサイトにつながります。
