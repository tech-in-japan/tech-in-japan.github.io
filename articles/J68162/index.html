Webアプリケーションを開発する際のデフォルトのブラウザーとしてのFirefox

最近尊敬されているakakuninは、「 Firefoxはデフォルトのブラウザーになります 」というトピックを公開しました。このトピックでは、最近リリースされたSpring Source Cloud Foundryサービスのコンソールにアクセスすると表示されるメッセージを引用しました：「使用しているブラウザーはCloud Foundryでサポートされていません。 このブラウザを使用して探索を続けることはできますが、多くの機能とレイアウト要素は正しく機能または表示されません。 Cloud FoundryにはFireFox 3.0+を使用することをお勧めします。」この事実から、彼とコメントの人々はトレンドについて考え始めました。



実際、この傾向の説明は非常に単純で、FireBugとSeleniumの2つの製品名で構成されています。



FireBugは、少なくとも何らかの形で私をAjaxの世界と和解させるものです。 それなしでは、Ajaxアプリケーションの開発を想像することはできません。 はい、新しいOperaにはDragonFlyがありますが、PMSMの操作はより困難です。



Selenium -Webアプリケーション開発者の誰かがそれが何であるかを説明する必要がある場合、私はこれらの人々に心から申し訳ありません:)原則として、適切に組織化された開発プロセスはSeleniumの使用と連動して、ライブテスターなしで行うことができます。 また、アプリケーションがSpringフレームワークで記述されている場合は、 umangiteを使用できます。これは、アプリケーションのSeleniumの構成を作成するライブラリは非常に簡単です。 Linuxでは、SeleniumはFirefoxでのみ動作します（少なくとも最近まで）。 Windowsでは、私が知る限り、FirefoxはSeleniumの親友です。



一般に、Firefoxが意図したとおりにアプリケーションを表示することを確信できる唯一のブラウザであることがわかります。
