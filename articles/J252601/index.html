さまざまなモバイルプラットフォーム上のアプリケーションでファイルを操作する機能

機能にファイル機能があるクロスプラットフォームモバイルアプリケーションを開発する場合、各プラットフォームでのファイルワークプロセスの編成について疑問が生じます。 Edustyの新しいバージョンを開発するときにこの問題に直面しました 。これにより、クラスメートとファイルを共有できます。 この記事では、オペレーティングシステムiOS、Android、Windows Phoneで実行されているアプリケーションでファイルをインポートおよびエクスポートする方法を説明します。









iOS



iOSオペレーティングシステムでは、ファイルシステムの概念はユーザーに対して隠されており、ファイルとの対話はアプリケーション自体によって実行され、アプリケーションディレクトリにあるファイルとのみ実行されます。 ファイルをアプリケーションディレクトリにインポートするには、iTunesファイル共有を使用するか、アプリケーションのファイルタイプを登録する方法がいくつかあります。



iTunesファイル共有を使用している場合、アプリケーションはiTunesの「一般ファイル」セクションに表示され、コンピューターからアプリケーションにファイルを追加できます。 この方法で追加されたファイルは、アプリケーションの/ Documentsディレクトリに分類されます。







アプリケーション自体は、新しいファイル用にこのディレクトリを制御する必要があります。 また、実際には、iTunesファイル共有はユーザーにドキュメントへの直接アクセスを提供するため、このディレクトリ内のファイルの名前を変更したり、いつでも削除したりできることを忘れないでください。



iTunesファイル共有を使用するには、 UIFileSharingEnabled （アプリケーションがiTunesファイル共有をサポートしている）フラグをinfo.plistアプリケーションファイルに追加する必要があります。 



<key>UIFileSharingEnabled</key> <true/>




ファイルの種類を登録する場合、標準の「アプリケーションで開く」ダイアログボックスをクリックすると、アプリケーションが選択リストに表示されてファイルが開きます。







この方法でファイルを開くと、そのコピーが/ Documents / Inboxディレクトリに配置され、アプリケーションはapplication：openURL：sourceApplication：annotation： protocol UIApplicationDelegate methodを呼び出します。 このメソッドでは、開くファイルのURLが渡されます。



/ Documents / Inboxディレクトリ内のファイルは読み取りおよび削除できますが、変更することはできません。 ファイルを変更するには、別のディレクトリ、たとえば/ Documentsに移動する必要があります。



この方法でファイルをインポートするには、 info.plistアプリケーションファイルにCFBundleDocumentTypes （ドキュメントタイプ）キーを追加する必要があります。 その値は配列であり、その各要素は、アプリケーションでサポートされる各タイプのドキュメントを記述するために使用される辞書です。 可能な手がかりとその意味の完全な説明は、ドキュメントにあります： developer.apple.com/library/ios/documentation/General/Reference/InfoPlistKeyReference/Articles/CoreFoundationKeys.html



ファイルの種類を制限しませんでしたが、任意のファイルをインポートできるようにしました。 同時にInfo.plistは次のようになります。 



 <key>CFBundleDocumentTypes</key> <array> <dict> <key>CFBundleTypeName</key> <string>All files</string> <key>CFBundleTypeRole</key> <string>Viewer</string> <key>LSHandlerRank</key> <string>Alternate</string> <key>LSItemContentTypes</key> <array> <string>public.data</string> </array> </dict> </array>




アプリケーションからファイルをエクスポートするには、同じ「アプリケーションで開く」ダイアログボックスが使用され、 UIDocumentInteractionControllerが初期化されます。 ファイルを開くためのアプリケーションのリストをすぐに開いたり、ファイルを開くためのアプリケーションのリストを開いたり、ファイルの印刷、メールでの送信などの標準サービスを開くことができます。 また、組み込みのプレビューアでファイルを開くこともできます。そこから、別のアプリケーションでファイルを開くためのボタンも使用できます。







Android



Androidでは、アプリケーションはプライベートアプリケーションディレクトリ内のファイル以外のファイルにアクセスできます。

ファイルの選択、電子メールの送信、ブラウザでリンクを開くなどの操作を実行するには、インテントが使用されます。

ファイルシステムからファイルを選択するには、 ACTION_GET_CONTENTアクションを使用する必要があります。

setType（）メソッドを使用して、選択可能なファイルタイプを指定できます。 たとえば、指定する場合

setType（“ audio / mp3”） 、ファイルを表示するアプリケーションでは、拡張子が.mp3のファイルのみを表示するか、すべてのファイルが表示されるように“ * / *”を指定します。 その後、 startActivityForResult（）メソッドを呼び出します。このメソッドでは、パラメーターとしてIntent.createChooser（）を渡し、アプリケーション選択ダイアログを作成します。

また、マニフェストファイルで、必要な権限を追加する必要があります。 



 <uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />








ファイルマネージャーでファイルを選択した後、 onActivityResultメソッド（int requestCode、int resultCode、Intent data）を取得します。ここで、 データはファイルのUriです。



サービスからファイルを開くには、 ACTION_VIEWアクションを使用します。 ファイルを開くためのアプリケーションは、 FileOpenクラス（StackOverFlowの拡張部分にあり、わずかにやり直し）を使用して選択され、潜在的な拡張子に従って、開こうとしているファイルの種類を確認します。



Fileopen.class 
 public class FileOpen { public static void openFile(Context context, File url) throws IOException { File file = url; Uri uri = Uri.fromFile(file); Intent intent = new Intent(Intent.ACTION_VIEW); if (url.toString().contains(".doc") || url.toString().contains(".docx") || url.toString().contains(".odt")) { // Word document intent.setDataAndType(uri, "application/msword"); } else if (url.toString().contains(".pdf")) { // PDF file intent.setDataAndType(uri, "application/pdf"); } else if (url.toString().contains(".ppt") || url.toString().contains(".pptx")) { // Powerpoint file intent.setDataAndType(uri, "application/vnd.ms-powerpoint"); } else if (url.toString().contains(".xls") || url.toString().contains(".xlsx")) { // Excel file intent.setDataAndType(uri, "application/vnd.ms-excel"); } else if (url.toString().contains(".zip") || url.toString().contains(".rar")) { // ZIP Files intent.setDataAndType(uri, "application/zip"); } else if (url.toString().contains(".rtf")) { // RTF file intent.setDataAndType(uri, "application/rtf"); } else if (url.toString().contains(".wav") || url.toString().contains(".mp3")) { // WAV audio file intent.setDataAndType(uri, "audio/x-wav"); } else if (url.toString().contains(".gif")) { // GIF file intent.setDataAndType(uri, "image/gif"); } else if (url.toString().contains(".jpg") || url.toString().contains(".jpeg") || url.toString().contains(".png")) { // JPG file intent.setDataAndType(uri, "image/jpeg"); } else if (url.toString().contains(".txt")) { // Text file intent.setDataAndType(uri, "text/plain"); } else if (url.toString().contains(".3gp") || url.toString().contains(".mpg") || url.toString().contains(".mpeg") || url.toString().contains(".mpe") || url.toString().contains(".mp4") || url.toString().contains(".avi")) { // Video files intent.setDataAndType(uri, "video/*"); } else { intent.setDataAndType(uri, "*/*"); } intent.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK); context.startActivity(intent); } }




一致するものを見つけるときは、 setDataAndType（）メソッドを使用して、送信するデータのUriおよびMIMEタイプを指定します。







ファイルに当社が考慮していない形式が含まれている場合、 setDataAndType（）でタイプとして「* / *」 を指定します。 したがって、システムはデバイスにインストールされているすべてのアプリケーションを表示し、ファイルを開く対象を自分で選択できるようにします。



Windows Phone



Windowsストア/ Windows Phoneストアのユニバーサルアプリケーションは、少なくともシステムフォルダーではなく、ファイルシステムをほぼ完全に開きます。 OpenFilePickerクラスを使用すると、デバイスメモリ、外部メディアとデバイス、クラウド（適切なアプリケーションがインストールされている場合）の任意のソースから任意のプラットフォームでファイルを開くことができます。 この点で、これらのデバイスでファイルを開くことに問題はありません。標準クラスでは、ファイル選択メニューから写真を撮ることもできます。これは非常に便利です。



標準のファイル選択ダイアログからファイルを開くには、OpenFilePickerクラスのインスタンスを作成し、いくつかのパラメーターで初期化する必要があります。 

 List<string> .FileTypeFilter
-許可されたファイル拡張子のコレクション（ファイルを選択できるようにするには、エントリ「*」を追加する必要があります）。 

  PickerLocationId .SuggestedStartLocation
-この列挙型により、ダイアログで最初に開く場所を指定できます（例：PickerLocationId.ComputerFolder）。 

 PickerViewMode .ViewMode
-ファイル表示のタイプ（リストまたはグリッド）を選択できます。



Windows 8.1とWindows Phone 8.1のファイル選択ダイアログの呼び出しは、ユニバーサルAPIにも関わらず異なります（ご存じのとおり、100％ユニバーサルではありません）。



Windows 8.1アプリケーションの場合、1つまたは複数のファイルを選択する必要があるかどうかに応じて、 .PickMultipleFilesAsync（）または.PickSingleFileAsync（）メソッドを呼び出す必要があります。 これらのメソッドは、それぞれIReadOnlyList <StorageFile>およびStorageFileを返します（厳密には、Taskを返すため、async / awaitを使用する必要があります）。







Windows Phone 8.1の場合、事態はもう少し複雑です。 同様のメソッドは.PickMultipleFilesAndContinue（）および.PickSingleFileAndContinue（）と呼ばれます。 どちらのメソッドも何も返さない（void）が、ダイアログでファイルを選択した後、 IFileOpenPickerContinuableインターフェイスのメンバーであるContinueFileOpenPicker（）メソッドが呼び出される。 このインターフェイスは標準APIには組み込まれていませんが、 ContinuationManagerクラスで使用できます。 * AndContinue（）メソッドを呼び出した後、作業を​​続行する方法については、 ContinuationManagerクラスをダウンロードしてください 。 そのため、指示に従ってContnuationManagerクラスをプロジェクトに追加した後、 .PickMultipleFilesAsync （）または.PickSingleFileAsync（）メソッドの呼び出し 元のクラスを継承し、 IFileOpenPickerContinuableインターフェイスから継承する必要があります。 次に、 void ContinueFileOpenPicker（FileOpenPickerContinuationEventArgs args）メソッドを実装する必要があります。args.FilesにはStorageFileオブジェクトのコレクションが含まれます。







サービスからファイルを開くには、ファイルをアプリケーションの内部フォルダーのディスクに保存し、 LauncherクラスのLaunchFileAsync（）メソッドで指定します。 その後、システムは、このタイプのファイルのデフォルトのアプリケーションでファイルを開くか、インストール済みのアプリケーションからアプリケーションを選択するか、ストアで検索するかを提案します。
