DD-WRTでIPv6をオンにします

ウェブサイト上の統計によるとtunnelbroker.net IPv4アドレスの枯渇前にプール年半は左。 そして、dd-wrtを実行しているルーターを新しいプロトコルに転送する方法を伝えることにしました。 320でのDir-テストしますが、サポートされている任意の他のAUCHAN、ルータ上で動作するはずDD-WRTを 。







残念ながら、最新バージョン（24 r13064）はIPv6をサポートしていません。 したがって、我々は取るここで、この自家製 、それを縫い付けています。 個人的に、私は（言葉の通常のロシアのアナログ？:)）13064から10070へのダウングレード、および設定がリセットされません。



次に、 Hurricane Electricの Webサイトにアクセスして、登録し、実際のIPv4アドレスでドライブし（通常のトンネルを作成）、設定のリストと新しいIPv6アドレスを取得します。 サーバーのIPv4アドレス、サーバのIPv6アドレスおよびルーテッド/ 64：リスト全体のうち、私たちは3行に興味を持っています。



次に、dd-wrt内の設定に直接進みます。



リンクWebベースのインターフェイスに移動します：

  • セキュリティ/ファイアウォール：何だろう私たちのブローカーたちソー用ブロック匿名WAN要求（pingを実行）からのチェックボックスを外します。
  • 管理/マネジメント：アクティブIPv6とRADVD
  • 次の一節挿入radvdの設定には：

    interface br0

    {

    AdvSendAdvert on;

    prefix _Routed_64

    {

    AdvOnLink on;

    AdvAutonomous on;

    };

    };





    vash_Routed_64あなたがルーテッド/ 64バーに持っているという事実を置き換えます
  • 管理/コマンド ：[ スタートアップを保存 ]をクリックします
  • トップマージンのドライブ：

    ip tunnel add he-ipv6 mode sit remote _Server_IPv4_address ttl 64

    ip link set he-ipv6 up

    ip addr add _Server_IPv6_address dev he-ipv6

    ip route add ::/0 dev he-ipv6

    ip addr add _Routed_64 dev br0







    それぞれの交換：

    vash_Server_IPv4_address - サーバのIPv4アドレス

    vash_Server_IPv6_address - サーバーのIPv6アドレス

    your_Routed_64-ルーテッド/ 64

  • 管理/コマンド ：[ ファイアウォールの保存 ]をクリックします
  • トップマージンのドライブ：

    insmod ip6t_REJECT

    ip6tables -F

    ip6tables -A FORWARD -p tcp -i he-ipv6 -syn -m multiport –dports ftp-data,ftp,ssh,smtp,http,https,ntp,domain -j ACCEPT

    ip6tables -A FORWARD -p tcp -i he-ipv6 -syn -j REJECT -reject-with adm-prohibited

    ip6tables -A FORWARD -p udp -i he-ipv6 -m multiport -dports ntp,domain -j ACCEPT

    ip6tables -A FORWARD -p udp -i he-ipv6 -j REJECT -reject-with adm-prohibited
