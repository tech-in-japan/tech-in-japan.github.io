プログラマーの仕事でソーシャルライクを便利にする方法は？

クールなプログラマーになるには多くのプログラミングが必要です。



しかし、面白くて新しい本を読んで知識を補充することも重要です。 以前はブログやRSSフィードでしたが、多くのソースから情報を取得できます。 しかし、時間が経つにつれて、私は3つの場所から得られるほとんどすべての最も興味深いものであるという結論に達しました： twitter 、 github 、 stackoverflow 。



これらすべてのサイトには共通の機能が1つあります。これは、あなたにとって興味深いと思われるコンテンツが、「いいね！」とマークされるのは簡単です。 Twitterの興味深いブログ記事へのリンク、githubの新しいフレームワーク、またはstackflowの質問を見つけた場合、間違いなく気に入っています。 このようにして、コンテンツの作者に感謝します。 しかし、「いいね」からもっと何かを得ることができますか？



経験豊富なプログラマーは、常に知っています 。 「これはすでに私の前で決定されています 。 」 したがって、適用されたタスクで作業する場合、すでにどこかでそれを読んでいることを理解しています。 しかし、難しい場合があります-見た場所を正確に覚えることは非常に困難です（ブログ投稿、githubの既製のソリューション、stackoverfowの有用な回答など）。 検索エンジンは役立ちますが、常にではありません-キーフレーズは、必要な正確なソースを見つけるのに十分具体的ではない場合があります。



私たちが開発中のこの問題を解決するために設計された製品、 Likeastoreを共有したいと思います。







Likeastoreでは、ソーシャルネットワークを1つのアカウントに接続できます。 その後、彼はすべての「いいね」を収集し、索引付けして、それらを検索する機会を与えます。 「いいね」を収集する初期プロセスは非常に高速であるため、ネットワークに接続するとすぐにdeschboardの最初の結果が表示され、通常はインデックス作成に時間がかかります。



簡単な使用例を挙げましょう。新しいSPA（単一ページアプリケーション）アプリケーションの開発を始めています。 この場合、SEO（検索エンジン最適化）の問題を解決することが重要です。 プラットフォーム、フロントエンドフレームワークなどに応じて、この問題に対する多くのアプローチを見たことを思い出します。 しかし、私が最も気に入った解決策があったことは確かです。 リクエストをyandexで試してみてください- " seo spa server side render"そして、 89000ページの結果を取得します。 私が探しているものもあるかもしれません-しかし、私はすべての検索結果を見るために多くの努力を費やす必要があります。
