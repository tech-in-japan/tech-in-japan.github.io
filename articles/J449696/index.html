こんにちは、Habr！ 私たちは、才能、またはむしろ最も才能のあるオーバークロッカーを探しています-彼らのために、HyperXはオーバークロック実験室OCLab.ruとともに、HyperX Memory OCompetition2019コンテストを開始します。 あなたが初心者のオーバークロッカーであり、CASレイテンシがRASからCASディレイに変わらない場合、それは問題ありません。 私たちの指示を使用すると、競争であなたの運を試すこともできます。 競争の本質は非常に単純です。HyperXMemory OCompetition2019は、KingstonとHyperX RAMのオーバークロック専用です。







競争目標



競争はオーバークロックであるため、参加者はオーバークロックRAMで競います。 周波数の加速における最良の結果は、公称値の割合として比較されます。 タイミング、電圧、パフォーマンスは重要ではありません。主なことは、KingstonまたはHyperXモジュールの周波数の最大増加を絞ることです。

参加方法



良い競争は完全に透明であり、詐欺から保護されるべきであると信じています。 したがって、オーバークロックの結果を検証し、システムのスクリーンショットと写真でバックアップする必要があります。



検証のために、 CPU-Z 1.88とThaiphoon Burner （無料版）をダウンロードしてください。



1.まず最初に、公称値に対するRAMの周波数で最大のオーバークロックを達成します。 最初の結果を急いで送信しないでください-おそらく1日か2日で新しい記録を立てることができるでしょう。

2. CPU-Zの4つのコピーを実行し、プログラムウィンドウを並べて配置し、それぞれで異なるタブを開きます：CPU、メインボード、メモリ、SPD。 それらの隣に、Thaiphoon Burnerウィンドウとはがきの[読み取り]タブを配置します。 5つのウィンドウすべてがデスクトップ上にはっきりと見えるはずです。 一般的なスクリーンショットを撮ってください。これは前提条件です。







3. CPU-Zでオーバークロック結果を検証します。 参加者の申し込みフォームに記入する際にリンクが必要になります。



4.マザーボードとオーバークロックしたKingstonまたはHyperXメモリが写真に見えるように、コンピューターまたはテストベンチの写真を撮ります。



5. リンクの参加者の申請書に記入します。 検証されたオーバークロック結果への参照に加えて、アンケートで記憶の記事を指定する必要があります。 CPU-Zの[SPD]タブにあります-[Part Number]列に示されています。 アンケートでは、システムのスクリーンショットと写真を求めていませんが、賞の場所のいずれかを取る場合、それらは後で必要になります。



また、CPU-ZとThaiphoon Burnerのスクリーンショットとコンピューターの写真をアンケートに添付します。



6. OCLab.ruでのコンテストの結果を待ちます！







ルールと日付



HyperX Memory OCompetition2019は、2019年4月24日から5月24日まで開催され、結果はこの期間中のみ受け入れられます。 結果は、2019年5月27日にOCLab.ru Webサイトに公開されます。



コンテストには、賞品を獲得するために遵守する必要がある規則と条件があります。



  • KingstonまたはHyperXブランドのDDR3またはDDR4メモリをオーバークロックして得られた結果のみが、コンテストに参加することができます。 システム内のモジュールの数は重要ではありません。 他のブランドのメモリ、および第1世代のDDR2とDDRは、競争に関与していません。
  • 有効なXMPプロファイルは、モジュールの定格周波数と見なされます。
  • 任意のプロセッサとマザーボードの使用、タイミング、電圧の変更、冷却の適用が許可されています。
  • 1人の参加者から、1台のPCからの1つの結果が考慮されます。 新しい結果を送信できますが、前の結果はキャンセルされます。
  • ロシアおよびEAEU CU諸国の居住者は、競争に参加できます：ベラルーシ、カザフスタン、アルメニア、またはキルギスタン。
  • 「参加方法」セクションに記載されているすべての条件を遵守する必要があります。RAMを搭載したコンピューターのスクリーンショットと写真がない場合、CPU-Z検証があっても結果を受け入れることはできません。


CPU-Zで結果を検証する方法



オプション1

CPU-Zの[バージョン情報]タブで、[検証]ボタンをクリックします。 開いたウィンドウの[提出]フィールドに、名前と電子メールを入力し、[送信]をクリックします。 検証結果へのリンクを含む手紙がメールに送信されます-これは、参加者のプロファイルで指定する必要があるものです。







オプション2

CPU-Zの[バージョン情報]タブで、[検証]ボタンをクリックします。 開いたウィンドウの[オプション]セクションで、[検証ファイルの保存]をクリックします。 名前（英字のみ）を作成し、ファイルをコンピューターに保存します。 valid.x86.fr/statistics.phpを開き、アカウントを作成して結果をサイトにアップロードします。 結果へのリンクが参加者のプロファイルに必要です。



注意!!!

valid.x86.fr/statistics.php検証サイトにアクセスして結果を登録できない場合は、検証ファイルをicealex.oclab@oclab.ruにメールで送信できます。 手紙はあなたの名前を示さなければなりません。
