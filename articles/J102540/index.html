地理的に分散したデータセンターへのライブ移行

典型的な状況：あなたは、いくつかの地理的に分散したサイト、いわゆるデータセンター（データセンター）を持つ若い開発会社です。 サイトAとサイトBを条件付きで呼び出します。サイトAでは、ビジネスクリティカルなアプリケーションを備えた物理サーバーが展開され、組織全体のユーザーがネットワーク経由でアクセスできます。 サイトAで建物が崩壊した場合、またはサーバーを停止する必要がある場合にビジネスアプリケーションでサーバーのメンテナンスを行う必要がある場合はどうなりますか？



答えは一つです-利益に影響するサービスを提供するのは簡単です。 どうする もちろん、仮想化と地理的に分散したクラスターを使用してITインフラストラクチャを構築することです。 何言ってるの？



最も単純な形式では、地理的に離れた2つの仮想化サーバーと、サーバーからは1つのストレージシステムとして認識される地理的に離れた2つの同期レプリケーションストレージシステムです。







そのため、仮想化プラットフォームはHyper-Vの役割を持つMicrosoft Windows Server 2008 R2です。 Windows Server 2008 R2フェールオーバークラスターコンポーネントに基づいて高可用性クラスターを作成し、クラスターの共有ボリューム機能を有効にしてから、高可用性を備えた仮想サーバーを作成します。 出来上がり-次の利点を提供するソリューションがあります。



1. TCP接続が失われることなく、あるホストから別のホストに仮想サーバーを移動できるライブマイグレーションの機能

2.最初から流れます-必要に応じて、すべての仮想サーバーをサイトBに移動することにより、エンドユーザーにサービスを簡単に提供できます。

3.接続を失うことなくサイト間で仮想サーバーを移動できるため、サイトでの動的な負荷分散

4.サイトAで災害が発生した場合（たとえば、サーバーに水が溢れた場合）、仮想サーバーはサイトBで自動的に起動します

5.フェールオーバークラスターGUI、Microsoft System Center Virtual Machine Manager、またはPowerShellスクリプトを使用したライブマイグレーションの管理のしやすさ。



1つのBUT-異なるサイトのストレージシステム上の仮想サーバーを同じにし、サイト上のLUNを1つにするにはどうすればよいですか？



現在、このタイプのタスク専用に設計されたベンダーのハードウェアおよびソフトウェアソリューションがあります。
