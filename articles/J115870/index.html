Firefox 4.0の新機能





最後に、多くの人に愛されているブラウザの4番目の新バージョンを待ちました。 公式リリースは明日03/22/11に予定されています。 カットの下で、革新の完全なレビュー、慎重に多くの写真、手紙、1つのビデオ







*停止ボタンと更新ボタンを1つに統合

*デフォルトのタブが一番上になりました

*通常のタブは、右クリックコンテキストメニューで「アプリタブ」に変換できます

*ブックマークツールバーはデフォルトでブックマークボタンに置​​き換えられます（必要に応じて戻ることができます）

*パノラマ機能により、開いているすべてのタブの完全な視覚的概要がユーザーに提供され、並べ替えとグループ化が可能になります

*アドオンマネージャーの改善

*既に開いているタブを検索し、スマートアドレスバーを使用して切り替えることができます

*接続ステータスメッセージが小さなオーバーレイ（左下隅）に表示されるようになりました





* HTTPヘッダーのサポート（ " DNT "）-追跡しない







*一部のレンダリング操作には、Windows XPではDirect3D 9、Windows Vistaおよび7ではDirect3D 10、Mac OS XではOpenGLを使用したハードウェアアクセラレーションがあり、Mac OS XではCore Animationも改善されています。

* Windows 7ユーザー向けのDirect2Dハードウェアアクセラレーション（デフォルト）

*すべてのプラットフォームにWebGL標準の完全サポートが含まれていますが、他のハードウェアアクセラレーション機能と同様に、ドライバーを最新バージョンに更新する必要があります（著者の注意：ATIビデオカードを搭載したLinuxドライバーは幸運ではありません。まだ）







* Firefox Syncがデフォルトでインストールされるようになり、セットアップインターフェースが改善されました





*新しい高速のJavaScriptエンジンであるJägerMonkeyが使用

*より効率的なJavaScriptアニメーションを提供するいくつかの実験的APIのサポート

* JavaScriptで値を表す新しい方法。これにより、Firefoxは（グラフィックやアニメーションに使用される）より重いデジタルコードをより効率的に実行できます。







*複雑なアニメーションを処理するための再設計されたアルゴリズム

* HTML5フォームのAPIにより、Webフォームの実装と検証が容易になります

*アドオンと拡張機能の管理者向けの新しいAPIとグラフィカルインターフェイスの変更

*新しいAudio Data APIのサポート

* HD HTML5 WebMビデオ形式のネイティブサポート

* FirefoxにHTML5ビデオ用のバッファが追加されました

*新しいHTML5パーサー

* Firefoxは自動的にオフラインになりません



* OpenTypeタイポグラフィの改善（合字、カーニング、フォントバリエーションのサポート）

* Web開発者はCSSを使用してコンテンツをアニメーション化できます

*拡張機能用のサードパーティインターフェイスであるJS-ctypesで、大幅なAPIの改善が利用可能

*より多くのHTML5フォームコントロールのサポート

* Web開発者は、HTML履歴用の新しいAPIを使用して、ページをリロードせずにURLフィールドを更新できます



*スクロールと応答性の改善

*遅延フレームのおかげで、より応答性の高いページレンダリング

*ブックマークと閲覧履歴に関するコードが再設計され、Firefoxの起動とブックマークの操作の速度にプラスの効果があります。

*起動速度とプロセス分離を向上させるために、XPCOMコンポーネントの登録方法を変更しました

*ページをロードするときの応答性を改善するために非同期にされたリンク履歴クエリ



* Windows、Linux、およびMac OS X上のAdobe Flash、Apple Quicktime、またはMicrosoft Silverlightプラグインのクラッシュ保護

*ページビュー履歴を表示するサイトの機能をブロックするCSSの変更

* HSTSセキュリティプロトコルのサポートにより、サイトはSSL経由でのみロードすることを主張できます。



Firefoxの未来

新しいバージョンのリリースに関するポリシーが改訂されたことは注目に値します。 現在、重要なバージョンはそれほど大規模なものではなく、4か月に1回程度出てくることがよくあります。



Firefox 5では、以下を追加する予定です。

新しいアカウント管理インターフェイス（写真はUI開発ユニットからのサンプルスケッチです）



私にはこれで十分でしょうが（たぶん、4ヶ月半後に誰かが盲目になります！？！）



友達とリンクを交換するためのシンプルなインターフェース（F1サービス統合）

ユーザーインターフェイスにアニメーション効果を実装する

Windows用の64ビットビルドを準備します。



Firefox 6では、Webアプリケーション（ Open Web Appプロジェクト ）の動作を保証する機能の拡張、キャッシュとJavaScriptエンジンのパフォーマンスの向上、Mac OS X 10.7用のアセンブリの準備に焦点を当てます。



Firefox 7：さまざまなブラウザーサブシステムの分離された実行を保証する作業を継続し（ユーザーインターフェイスと異なるタブのコンテンツの処理は異なるプロセスによって並列に処理されます）、 XBL（XMLバインディング言語）のサポートを統合する予定です 。
