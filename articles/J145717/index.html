iOSアプリとEvernoteの統合：最初のステップ





Evernoteの利点の1つは、その遍在性です。 当社のアプリケーションは、ほぼすべての重要なモバイルプラットフォーム、2つの最も人気のあるコンピューターOS、およびその他すべての場合（Webインターフェイス経由）で動作します。 しかし、私たちはそこで止まるつもりはありません。



現在、より多くのモバイルアプリケーションがEvernoteに情報を読み書きする機能をサポートしています。 私たちはそれをとても気に入っており、さらに続けたいと思っています-あなたのノートにアクセスする方法が多いほど、より良いです（いずれにせよ、私たちに思えます）。 開発者として、3000万人以上のユーザーにアクセスし、これらの人々の目でアプリケーションの価値を即座に高め、外部の「デジタルメモリ」と通信する機会を開くことができます。



Evernoteプラットフォーム（または開発全般）の学習を始めたばかりで、アプリケーションをサービスに統合したい開発者向けのマニュアルを定期的に発行することにしました。



この投稿では、EvernoteをテストiOSアプリケーションに統合するための特定の手順について説明します。 Xcodeで新しいプロジェクトを開始し、Evernoteソフトウェア開発キット（SDK）をダウンロードしてインストールし、Cloud APIを介してEvernoteと対話するシンプルなアプリケーションを作成します。



始める前に、将来の独立した仕事に必要なものをリストします。

  1. sandbox.evernote.comのアカウントは、開発者向けのEvernoteサンドボックスサーバーです。 Evernote Cloud APIを介して実行されるアプリケーションをデバッグするために使用されます。 ここでテストアカウントを作成できます。
  2. Evernote APIキー（キーをリクエスト ）。 APIキーは、 コンシューマキーとコンシューマシークレットの2つの部分で構成されます。 （ 注：独自のアプリケーションを開発している場合は、アプリケーションの開発が終了したら、Evernoteメインサーバーでキーを有効にするように依頼する必要があることに注意してください。これはdev.evernote.com/supportのフォームから行えます ）。
  3. 執筆時点でのXcodeの最新バージョンは、Intel Macにインストールされている4.3.2です。


すべて在庫がありますか？ それでは、始めましょう。



このアプリケーションは、現在のユーザーの名前とアカウント内のメモの総数を表示するという2つのことを正確に行います。



*アプリケーションのためにこのアイデアを偶然に選択したことを明確にする価値があり、開始時にこの情報に簡単にアクセスするためのAPIがあるかどうかわかりませんでした*



基礎を築く



コードを書き始めて最初のユーザーに感銘を与える前に、Xcodeプロジェクトを作成し、iOS用のEvernote SDKをインストールする方法を準備する必要があります。



Xcodeプロジェクトの作成


  1. Xcodeを起動し、メニューから「ファイル」>「新規」>「プロジェクト」を使用してプロジェクトを作成します。
  2. プロジェクトテンプレートを指定するように求められたら、左側のメニューの[iOS]で[アプリケーション]を選択し、[シングルビューアプリケーション]をクリックします。 次に「次へ」をクリックします。







  3. 次の画面で、新しいアプリケーションのいくつかの特性を明確にする必要があります。

    • 製品名に「EvernoteCounter」を指定します。
    • 会社識別子は任意です。 この値は、世界中のアプリケーションを識別するために使用されますが、コンピューターよりも先に実行することはないため、重要ではありません。 この例では、「com.somecompany」を指定しました。
    • クラスプレフィックスに「EC」を指定します（「EvernoteCounter」の略）。 この略語は、コードの記述を開始するときにすべてのクラス名に先行します。
    • デバイスファミリで「iPhone」を指定します。
    • 3つのボックスがすべてオフになっていることを確認してください。
    • 完了したら、次のようにフィールドに入力する必要があります。



  4. 最後に、プロジェクトの場所を示すように求められます。 あなたの裁量でそれを選択してください。


Xcodeプロジェクトの準備ができました。 数分間それを脇に置きます。 アプリケーションがEvernoteクラウドAPIとやり取りするには、SDKが必要です。 それを取得して、プロジェクトに追加しましょう。
