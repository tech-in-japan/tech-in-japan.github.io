プラナリウム。 一般的な動員

みなさんこんにちは。 私たちは最近、Planariumを大幅に更新しました。投稿を書いてhabrasocietyの新機能について話さずにはいられなかったのでとてもうれしく思います。



このアップデートのモットーは「普遍的な動員」であり、ご想像のとおり、Planariaはコンピューターだけでなく使用できるようになりました。



左側に表示されているのは、今日のすべてのイノベーションの1/3であるGoogleガジェットです。 これはgmailからのものです。 はい。今では、お気に入りのメールから直接、すばやく便利にあなたの業務を管理できます。 「大きな」Planariaのほぼすべての機能がサポートされています。タスクの転送、フォーマット付きのメモの作成、タスクの完了などです。 これで、メールからまったく出られなくなります。







しかし、ガジェットを使用するには、まだコンピューターが必要であり、それを拒否できると約束しました。 約束は守られました-現在、Planariaにはモバイル版もあります。 モバイルデバイスからplaney.ruにアクセスするだけで、使用を開始できます。 そこはすべてシンプルで、90年代のWebのスタイルです（携帯電話のブラウザーがJSと対立していない場合）。 しかし禁欲主義！=低機能。

モバイル版では、メインサイトと同じように業務を行うことができますが、それほど便利ではありません=）



最後に、3番目の3番目は新しいジャバーボットです。 古いPlanariaボットはあまり機能的ではなかったため、ユーザーから絶えずやり直すように頼まれました。 同時に、ボットを完全に書き直さなければならないことを理解しました。



*技術者向けの詳細がいくつか追加され、特にルビストにとって興味深い*



古いボットはxmpp4r gemを使用して作成されました。 ただその瞬間に代替手段がなかっただけです。 このgemを使用した例をこれまでに見たことがある人なら誰でも、そこには実際には利便性がないことを知っています。 また、ボットに負荷がかかると、彼は本当に飛び出すのが好きだということがわかりました。

RailsEnvy Podcastの次号を聞いた後、私は素晴らしいことを聞きました 。 読み、発火し、2時間でボットを書き換えました。 確かに、泡自体のエラーのために機能しませんでしたが、その著者は非常にクールな男であることが判明し、すべてをすばやく修正しました。 これで、ボットがクラッシュした場合、それはたわごとコード=のせいだけです。したがって、アプリケーション用のジャバーボットを作成する場合は、すべての人にblatherを確認することをお勧めします。



そして今、ボット自体について、ここにそのコマンドのリストがあります：



  • 今日 <ケース名>; <2番目のケースの名前>; ... <n番目のケースの名前>
  • 明日のために <ケース名>; <2番目のケースの名前>; ... <n番目のケースの名前>
  • 1週間 <ケース名>; <2番目のケースの名前>; ... <n番目のケースの名前>
  • 来週 <ケース名>; <2番目のケースの名前>; ... <n番目のケースの名前>
  • カオスへの <ケース名>; <2番目のケースの名前>; ... <n番目のケースの名前>


上記のすべてのチームは、次のアルゴリズムに従って動作します。ケースがない場合は作成され、そのようなケースがある場合は、適切なエリアに転送されます。 また、ビジネスにメモをすぐに追加できます。 ケースの後に括弧で囲んで示すだけです。



  • どうする -今日の予定リスト全体を表示します。
  • なんて明日
  • 一週間は何？
  • 来週は何？
  • 混乱状態にあるのは何ですか？
  • <ケース番号> をアーカイブします 。 <ケース番号>（ここで、ケース番号は、「対処方法」コマンドを使用する場合のケースの番号付けと一致します）
  • サイトへ -承認プロセスをバイパスして、すぐにサイトにアクセスできるリンクを返します。




何も理解していない場合は、ボットに関するビデオの説明をご覧ください。



高解像度で見ることをお勧めしますが。
