Azov Developers Meetupカンファレンス-9月23日、Taganrogにご招待します









Azov Developers Meetup 2017の Taganrogで9月23日を一緒に過ごすことをお勧めします。



午前9時、通りにあるコングレスホテルタガンログに到着します。 Dzerzhinsky、161は開始前にコーヒーを飲み、午後3時にコーヒーを飲み、開発に関する講義を聞きました。 Vue.jsとCQRSについて話し、SQLとNoSQLの共通点を見つけようとし、コードの卓越性と実際の品質保証の限界について話し、顧客とのインタビューの不安を払拭し、設計の質問に答えます。 さらに、テレビクラブのプレイヤーであるミハイルスキプスキーを招待しました。 どこ？ 「いつ？」、2つの「クリスタルフクロウ」の所有者（2010、2016）-彼は、小グループでの意思決定と有効性について話します。



いつものように、講義に加えて、参加者のための活動を準備し、講演者のスピーチのすべてのプレゼンテーションとビデオ録画はイベントのウェブサイトで利用できるようになります。



そして、17時にマーマレードショッピングセンターのステージクラブでアフターパーティーが開催されます。ミハイルは会議参加者のための知的バトルを開催します。 彼のスピーチから論文を実際にチェックしましょう！



レポートの詳細：



顧客とのインタビューの方法

アンドレイ・ホリヤフキン、アルカディアのタガンログ事務所長



サービスIT企業の開発者またはテスターは、遅かれ早かれ、この不思議なイベントに近づきます。新しく、このような興味深く魅力的なプロジェクトにたどり着くには、クライアントの代表者とのインタビューを行う必要があります。 初めてでも怖いことがあります！



レポートでは、こうしたインタビューと興味深いケーススタディを無事に完了するためのベストプラクティスを収集し、まとめました。 開発者とプロジェクトマネージャーの両方にとって興味深いものになります。



Vue.js：新しいフロントエンドフレームワーク

開発者、ミハイル・ゾトフ



フロントエンドの多くのフレームワークの中で、最も人気があり有名なものはAngularとReactですが、新しいVue.js.フレームワークはそれらとうまく競合します。 Michaelは、競合他社に対するVue.jsの利点を検討し、Vue.jsのコンポーネントの内部構造を詳細に説明します。



Erectus TesterからQA sapiensへ

ニキータ・ベルコフスキー、QAエンジニア



ニキータは、さまざまなチームとさまざまな管理アプローチを使用して、さまざまな複雑さのプロジェクトのテストに従事していました。 しかし、あるプロジェクトは他のプロジェクトとは大きく異なりました。 このストーリーは、彼が「弱い」プロジェクトでどのように品質保証とは何かを感じることができたかについてです。



イベントWebアーキテクチャのCQRS原則

ITスペシャリスト、ミハイル・チェルノルツキー



このレポートは、Webバックエンドアーキテクチャの構築パターンの1つであるCQRS（コマンドクエリリクエストの分離）の検討に専念しています。 CQRSとは何か、どのような場合にそれを使用するのが理にかなっているかを説明します。



  • CQRS（Command Query Responsibility Segregation）パターンの起源と目的。
  • 長所と短所;
  • WebシステムアーキテクチャでのCQRSパターンの適用。
  • 関連概念：イベントソーシング、結果整合性、マテリアライズドビューなど




設計の問題

UX-UIデザイナー、ボリスビノクロフ



UX / UIデザイナーとは誰ですか？ Webアプリケーションの設計プロセスで必要なのはなぜですか？ 彼らの助けなしに行うことは可能ですか、それは助けですか？ 彼らは他のデザイナーとどのように違いますか、彼らはアーティストと見なされますか？ ワークフローはどのように配置されていますか？ 良い結果と考えられるものとその理由は？ 創造性と創造性または問題解決？ または、あなた自身の質問がありますか？ 喜んでお答えします。



商用開発の純粋なコード。 完璧に限界はありますか？

Yuri Kovalev、開発者



このレポートでは、大規模な商業プロジェクトを開発およびサポートする際に、ダーティコードを根絶する問題について考察しています。 人生の具体例として有名なスカウトのルールを考えてみましょう。 どのリファクタリングルールが今後繰り返し報われるか、またそれらのために開発プロセスが遅くなるほど重要ではないものを判断してみましょう。 最後に、開発タイムラインを中断することなく、最小限の労力で非常に汚れた複雑なコードをクリアする方法を見てみましょう。



SQLまたはNoSQL？

Vladimir Kalskov、シニア開発者

Oleg Bragin、シニア開発者



2つの異なる-一見したところ-データストレージを整理するアプローチ。 異なる種類の思考を持つ人々が生活し、働く2つの異なる宇宙。 彼らは一緒に来ることができますか？ 何らかのアプローチが未来を支配しますか？ データを保存する最適な方法を見つけてみましょう。



なに？ どこ？ いつ？ チームワークモデルとして

ミハイル・スキプスキー



私たちは通常、テレビゲームを面白いコンテンツと見なし、ゲームはもともとモデルであるという事実についてはほとんど考えません。 「何？」の場合 どこ？ いつ？」-社会的および技術的な意思決定モデル。 これは、ChGKや同様のゲームのプレイ方法を理解したことで、グループの決定方法、管理方法、小さなチームの効果を妨げる問題を理解できることを意味します。
