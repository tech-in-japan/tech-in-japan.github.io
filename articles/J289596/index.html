そして、フランチャイジー自身が自分の仕事をどのように組織するか、1Cの販売と実装に関与する企業でどのような部門構造が形成されるか、どの構造部門が主なものか、どのように働くかについてお話したいと思います。 これらの要因を理解することで、新しいソフトウェアの導入を支援するアーティストを選択する際に何に焦点を当てるか、セールスマネージャーとコミュニケーションをとる際に注意を払うことが重要なこと、プロジェクトの実装に関与する専門家をよりよく理解できます。



フランチャイジー企業の構造1C



1Cフランチャイジー企業に何を期待するかを理解するには、その構造を理解することも価値があります。 フランチャイジーはどのように機能しますか？ 彼らは何でできていますか？ 会社の規模、支店数に関係なく、フランチャイジーは1つのユニットで構成されています。これは、1CフランチャイジーLLCなどの特定の法人です。



そして、内部から見ると、会社は通常次のようになります。



  1. 監督
  2. 営業部。
  3. 実装部門。
  4. テレマーケティング部。
  5. 設計部門（十分な規模の企業のみ）。
  6. 情報部と技術サポート。


このリストでは、人事部、IT部、経理部、および企業には存在するがクライアントが遭遇しない他の部の説明を意図的に超えました。 ここでは、顧客が協力の過程でコミュニケーションを余儀なくされるユニットのみを表示したいと思います。



それでは、会社の各部門が行う仕事の質を見てみましょう。



監督


ディレクター（会社または支店）-これはほとんどの場合マネージャー、マネージャーです。 その結果、この人は1Cについてほとんど何も知りません。 多くの場合、かつてこのビジネスを組織し、構築したのはこの人でしたが、彼は1C、ソフトウェア製品の技術的特徴については事実上何も知らず、これらの問題では部下に完全に依存しています。



ディレクターは最も先進的な営業担当者です。 彼が専門家であるのは販売で、彼の機能は主に困難な状況で妥協点を見つけるのを助けるために減らされます。 たとえば、プロジェクトの実装プロセスでは、問題が発生し、営業部門または実装部門が対応できず、時間が引き出され、顧客が満足しません。 それが監督です-これが最後のリンクであり、「火を消す」のに役立つ別のステップです（クライアントに同意し、彼を安心させ、期限を延長するよう説得するなど）。これらはフランチャイジーのディレクターによって実行される機能です。実際に出くわしました。



ディレクターは従業員の専門能力開発にほとんど関与せず、単に管理者として行動します。



何がそんなに悪いの？


悪いニュースは、専門的な意味での会社が単独で働くことです。 ディレクターはメンターではなく、メンターではなく、単なる管理者です。 この状況はほとんどすべての企業で見られますが、例外は非常にまれであり、10〜15人の小規模企業で最もよく見られます。 この場合、ディレクターはしばしば管理機能と1Cスペシャリストの機能を組み合わせ、プロジェクトに取り組み、資格を失うことなく、常にすべての問題と技術革新について知識を持ち、自分自身を発展させ、会社を発展させます。 しかし、繰り返しますが、これはルールの例外です。 ほとんどの場合、ディレクターは管理者です。



営業部



営業部門は通常どのように働いていますか？ 州では、ほとんどの場合、1Cについてほとんど知らない若い男性（22〜25歳）を見ることができます。 ほとんどの場合、彼らは1Cの販売に関する講座を受講し、一定量の断片的な情報を受け取り、いくつかの用語を覚えていました。 そして、彼らは主に電話に出たり、質問をしたり、顧客の主なニーズを見つけたりすることに従事しています。 会話がうまくいった場合、電話での会話の後、これらの若者はクライアントと会います。



そのような作業組織の何が悪いのでしょうか？


原則として、営業スタッフは1Cについて何も知りません。 それらのレベルはしばしば非常に低いため、構成とプラットフォームを混同する可能性があります！ その結果、そのような不幸な専門家は販売しますが、彼ら自身は何を知りません。



もう1つ問題があります。 フランチャイジー企業は、しばしば自身を1Cとして位置付けます。 ただし、1C自体にはさまざまな製品があります。 今日、300以上、さらには400以上のアイテムがあります（率直に言って、私自身は正確に覚えていません）。 ビューティーサロン、卸売販売および小売販売、ジュエリー販売、石油化学製品、生産およびサービス用の既製の構成があり、ビジネス分野や目的に応じて非常に異なる製品が多数あります。



当然、21歳のときに、1Cセールスコースでトレーニングを受けた若い男性は、それをすべて把握することはできません。 ほとんどの場合、彼はこれらの製品のリストさえ覚えていませんが、それらが互いにどのように異なるかを理解するためには、ある場合には1つの名前を選択するほうがよいのですが、別の場合には完全に異なるレベルの1Cの知識が必要です。



しかし、これらの若者の動機は販売です。 彼らは売らなければなりません。 その結果、顧客は電話をかけ、質問をし、会社の従業員が彼に販売しますが、彼自身は何を知らず、その存在を覚えている製品の選択肢を提供します。実際、非常に短いリストにはその製品がまったく含まれていないことがよくありますクライアントが実際に必要とするもの。



どんなにばかげても、実際には販売は次のようになります。



クライアントからの電話：「こんにちは！ 1Cプログラムを購入したいです。」

マネージャー：「何をしていますか？」

クライアント：「服を着ます。」

この短い対話の後、マネージャーはカタログに行き、衣服に関連する何かを見つけようとします。その後、彼は見つけた製品を買い手に提供します。


結果は何ですか？ クライアントは、貿易管理、衣料品取引、プラス1Cアカウンティングを購入する（そしてしばしば販売する）ことを提案されます。 しかし、実際には、人は注文に応じて製造または調整を行っています。 そして、彼は貿易文書、分析、会計を必要としません。 多くの場合、クライアントのニーズを詳しく調べると、彼は最初から最も一般的な「1Cアカウンティング」を取得したいと思っていたことがわかります。



なぜこれが起こっているのですか？ マネージャー自身は自分が何を売っているのか理解していませんが、できるだけ売る必要があることをよく知っています。 結果は明らかです：クライアントはまったく購入せずに退去するか、この会社と協力して不満を抱き続け、将来、必要に応じてプログラムを更新するか、変更を加えると、代替手段（フランチャイズ会社またはソフトウェア製品）を探します。 さらに、もちろん、顧客の不満は常に会社の評判を汚すものです。



営業部長


この状況から抜け出す方法はもう1つあります。マネージャーまたはクライアントは、営業部長のアドバイスを直接求めます。



通常、どの会社でも、営業部門は普通の従業員とマネージャーで構成されています。 後者の違いは何ですか？ 彼は年長で、経験が豊富で、販売する製品について多くのことを知っています。



さらに、普通の従業員の間で非常に高い離職率があることを理解することが重要です。 多くの場合、従業員はわずか数か月間働くことができ、最小限の経験しか得られず、別の会社に行きます。 このような状況は、1Cフランチャイジーの多くの企業で発生しています。 ほとんどの人は営業の専門家が不足しているため、わずかに訓練された売り手でさえすでに労働市場で評価されており、競合他社はしばしば彼を誘惑しようとします。



営業部長は会社のために長い間働いてきました。会社が設立されたときからでさえ、彼の部下が彼が販売するものについて知っているよりもずっと多くです。 しかし、それでも販売に焦点を当てています。



一般に、1Cに従事している企業では、これは大きな問題です。営業は営業に従事しており、他の部門を離れた人、たとえば、実施や生活プロジェクトから離れた人ではありません。 したがって、営業部門は、契約が成立した後に何が起こるかを単に想像していません。これらの人々は、プロジェクトがどのように進行していて、専門家が後でどのような困難を抱えているかを見たことはありません。 彼らは売れたばかりで、その売上げのパーセンテージを得た、それだけです。 クライアントや他の部門の同僚の問題は、彼らにとって関心のないものです。



営業部門の責任者は、部下よりも少し製品をよく知っている人であり、心理学に精通し、販売経験が豊富で、複雑な顧客であっても共通言語を見つける方法を知っています。 しかし、それにもかかわらず、彼はセールスマンであり、システム自体を知りません。



重要：1Cシステムを知ることは完全に不可能です。 現在存在する構成は、ほぼすべてのアカウンティング、アクティビティのすべての領域をカバーしています。 1人の人がすべての製品を知ることはできません。 しかし、少なくとも、どの構成が何に最適であり、特定の場合にクライアントに提供する方が良いかという一般的な考え方は非常に現実的です。


実施部門



そのため、クライアントはある種の構成で販売され、おそらく正しい構成で販売されていたかもしれません。 しかし、契約は締結され、販売が行われました。 次に何が起こりますか？



次に、実装部門が作業に接続されます。 1Cプログラマー、1Cコンサルタント、実装者、1C専門家、1C実装スタッフなど、非常に異なる名前で呼ばれる技術専門家で構成されています。 いずれにせよ、実装部門の専門家が顧客を訪問して、1Cソフトウェア製品をインストールおよび構成します。



残念ながら、通常、実装者の知識レベルは非常に低いです。 主な理由は、1Cではプログラマーとコンサルタントの区分が非常に貧弱だからです。 ほとんどの場合、人はプログラミングまたはコンサルティングのいずれかをよく知っています。 したがって、彼はプログラムに精通しており、必要なものすべてを構成し、不足しているものを追加することもできますが、サブジェクトエリアに精通していないため、正しくインテリジェントにアドバイスすることはできません。 どちらかが優れたコンサルタントである場合、彼は語り、伝え、助け、説明しますが、プログラムを使用する設定とニュアンスに関しては、率直に「浮かんでいます」。



プログラマーが作業を行い、インストール、構成、および必要に応じてソフトウェア製品を完成させるために、2人の専門家がクライアントと連携するように、実装者の作業を分割するのが正しいでしょう。 コンサルタントは、ソフトウェアの使用を開始し、助け、伝え、見せ、教えてください。



しかし、残念なことに、この問題では、1Cフランチャイジーのほぼすべての作業スキームのように、これらのニュアンスの管理には誤解があります。 無能に関連する問題があります。 頭は1Cでの作業の特徴を理解していないため、従業員は作業を最適化する方法を正確に説明する能力がないか、単に習慣に同意するだけです。 その結果、チェーン全体が多少間違っており、顧客はまず第一にこれに苦しんでいます。



ディスクキャリア


多くの場合非常に多くの経験を持つ技術専門家である実装者自身に加えて、実装部門にはいわゆるディスクキャリアがあります。 これらは通常、1Cについてほとんど何も知らず、ディスクの配布に従事している若い男の子と女の子です。



この作品の本質は次のとおりです。 ある種のアップデートをインストールするには、非常に頻繁にディスク上のクライアントにそれをもたらす必要があります。 行商人が行うのはこの仕事です。 さらに、彼らはこのアップデートをその場でインストールします。 これは数回のクリックで行われ、資格は必要ありません（コンピューターレベルのユーザーレベルを除く）。



これらの従業員は、その機能に加えて、ディスクを持参して更新プログラムをインストールしますが、方法がわからず、わかりません。 彼らは顧客の質問に答えることができず、更新の本質を伝えることも、インストール中に発生する可能性のあるいくつかの問題を外出先で解決することもできません。 ほとんどの場合、これらの人々は発達せず、学習しません。 原則として活動範囲を変更するか、40歳でもディスクを持ってきていくつかのキーを押すことができるユーザーと同じままです。 これらは、私の単なる観測ではなく、実際の観測です。 私が理解しているように、プログラミングを学べる人は行商人に行くことはほとんどありません。



この記事では、1Cプログラマーについては説明しません。これは、これが大きな別個のトピックであるためです。 別の記事でそれらについて話すつもりです。 そして今、私は彼らが、もちろん、状態にあることに言及することにだけ自分自身を制限しますが、クライアントはほとんど彼らと通信しません。


カスタマートレーニング：これはどのように行われますか？


そのため、ソフトウェア製品はクライアントに配信され、インストールおよび構成されました。 次は？ クライアントとその従業員が独立して作業することを学ぶか、新しいソフトウェア製品の作業を習得するのを支援する実装者が割り当てられます。



ここでは、次のニュアンスが非常に重要です。実装者は、このソフトウェア製品の専門家でなければなりません。 その後、彼はクライアント企業の従業員が新しいプログラムに移行するのを支援できるようになります。 しかし、侵入者が選択された構成の専門家ではない場合、クライアントが需要のないエキゾチックなものを販売している場合は特にそうですが、作業の結果は悲惨なものになります。 ほとんどの場合、すぐにクライアントのスタッフが古いソフトウェアで作業を再開し、新しいソリューションは実際には非常に優れていても、ディスクにほこりを集めるでしょう。



クライアントの従業員が独自に理解し、結果がどうなるかを記録して新しいプログラムに記録しようとするようなまれなケースでも、メリットはほとんどありません。 結局、お金は以前に使用されたソリューションの不便な代替品を得るためではなく、新しい機会、新しいレベルの会計自動化、高度な分析、および結果として誰も使用しない他の会計ツールに費やされました。



実装部長


原則として、実装部門の責任者は1Cについて何も知りません。 しかし、これらの人々はほとんどの場合、優れた管理者です。 実装部門の責任者は、実際に1Cを知らず、実装プロセスの進行状況を理解せず、管理し、同時にクライアントと通信して現在の問題を解決しようとします。 したがって、実装部門の長は、常に優れた交渉者であり、非常に優れた管理者です。



さらに、1Cの専門家ではないこれらの人々は、実際、最先端の技術専門家ではありません。先に述べたように、ほとんどの実装者はインストールするものに非常に精通していないからです。



しかし、すべてにもかかわらず、実装部門のリーダーは通常、なんとかして自分の仕事の秩序を維持し、問題を解決します。 コンサルタントを雇うこともあれば、設定の権利を知っているプログラマーやフリーランスの実装者を雇うこともあります。 同時に、彼らはクライアントに同意し、サービスの特定の可視性を維持します。



個人的には、ほとんどの場合、サービスの可用性と顧客の会社への忠実な態度という事実こそが、実装部門の長のメリットだと思います。 通常、このポジションは、困難な状況で解決策を見つけることができ、優れた交渉能力を備えた非常に賢い人々によって占められています。 さらに、原則として、営業部の長は会社が設立された日からほぼ働いており、非常に忠実です。



1つマイナス-実装部門の長はまだプログラムを知らない。 一方、彼の優れた管理者スキルはこの欠点を補っており、このポジションの専門知識はまったく必要ありません。



テレマーケティング部



テレマーケティングとは、今日ではほとんど誰もが知っています。 この作業は、さまざまな企業の連絡先データがさまざまなデータベース、ディレクトリ、カタログ、その他のオープンソースから取得されるという事実に基づいています。 その後、部門の従業員（ほとんどの場合女性）は、潜在的な顧客の連絡先に体系的に電話をかけ、いわゆる「コールドコール」を行います。



この部門の従業員の責任は何ですか：



  1. 電話の関連性を確認してください。
  2. 1Cのどのバージョンがインストールされているか、彼らが会社で一般的に使用しているものを調べます。
  3. 責任者（主任会計士、ディレクターなど）の連絡先の詳細を取得する
  4. 責任者に連絡できる場合は、会社のサービスを提供してください。
  5. 潜在的なクライアントからの関心の場合、「リード」（利害関係者と会社全体の連絡先）は、さらなる作業のために営業部門に転送されます。


この部門の従業員は、ほとんどの場合、1Cについて何も知りません。 彼らの電話はすべて、空白、いわゆるスクリプトに基づいています。 テレマーケティング部門の従業員の仕事の主な目標は、責任者の連絡先の詳細を取得することです。



テレマーケティング部門の長は通常、営業部門です。 そして、私の意見では、この顧客を引き付ける方法は非常に効果的です。 実際には、テレマーケティング部門の本当に高品質で効果的な仕事を何度も観察しました。 もちろん、彼らは顧客を誤解させ、「1C会社」のように見せますが、とにかく結果があります。 さらに、テレマーケティング部門のコストは、コールドコール方式が会社にもたらす収入よりもはるかに低いです。



設計部門



クライアントがかなり大きな希望を持っている場合、実装部門は実装に対応できず、作業のこの部分は設計部門に転送されます。 たとえば、クライアントはソフトウェア製品の詳細な個別設定を必要とするか、洗練（新しいレポートの作成、処理、印刷フォームなど）を必要とします。 原則として、実装部門自体が多くの同様のタスクに対処する必要があります。 しかし、実装者が失敗した場合、またはクライアントのすべての要望を満たすためにかなりの量の複雑な作業が必要であるという結論に達した場合、これらの要望はすべて設計部門に転送されます。



設計部門は何で構成されていますか？



  1. プログラマー1C
  2. コンサルタント1C
