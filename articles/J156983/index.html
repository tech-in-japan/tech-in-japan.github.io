Amazon EC2 / VPCの第2世代の標準インスタンス



Amazon EC2 / VPCで利用可能なサーバーのリストに、新しい名前M3 Extra Large InstanceとM3 Double Extra Large Instanceが登場しました 。 これらは2つの新しいタイプの標準インスタンスです。 なぜ標準なのですか？ すべての標準インスタンスにはメモリとCPUのバランスがあるためです。 ハイメモリやハイCPUタイプなど、いずれの方向にも大きな利点はありません。



さて、AWSはどのような車を提供していますか？ 猫をお願いします。



M3特大インスタンス


  • 15ギブメモリ
  • 13 EC2 Compute Units（3.25 EC2 Compute Unitsを備えた4つの仮想コア）
  • EBSストレージのみ
  • 64ビットプラットフォーム
  • I / Oパフォーマンス：中程度
  • EBS-Optimized Available：いいえ
  • API名：m3.xlarge


M3ダブルエクストララージインスタンス


  • 30ギブメモリ
  • 26 EC2 Compute Units（3.25 EC2 Compute Unitsを備えた8つの仮想コア）
  • EBSストレージのみ
  • 64ビットプラットフォーム
  • I / Oパフォーマンス：高
  • EBS-Optimized Available：いいえ
  • API名：m3.2xlarge


はい、悪い形ではありません。 新しいインスタンスを使用すると、システムアーキテクチャをより柔軟に構築し、より効率的にお金を使うことができます。 これらおよびその他のタイプの詳細については、 Amazon EC2インスタンスタイプをご覧ください。



値下げ



何度も繰り返しますが、AWSはバージニア州とオレゴン州で再び価格を引き下げます。 平均して、価格に19％の値下げが影響を与えました：

  • M1スモールインスタンス
  • M1スモールインスタンス
  • M1ラージインスタンス
  • M1特大インスタンス


個人的に、しかし私の経験では、これらのタイプのインスタノは可能な限り頻繁に使用され、これらの地域は価格カテゴリーで最も魅力的な地域のひとつであり、主要市場である米国に近接しています。 価格の詳細については、 Amazon EC2 Pricingをご覧ください。



AWSがクラウドサービスの機能性だけでなく、価格設定においても市場の主力製品であることを何度も繰り返したいと思います。
