PHPconf 2008で学んだ新機能

数日前、PHPプログラマーに焦点を当てたWeb開発者向けにPHPConf 2008カンファレンスが開催されました。 印象を整理するために、私はこの会議から個人的に取り出した投稿を書くことにしました。 私は究極の真実のふりをしていないので、他の参加者は確かに全く違うものに興味を持っていました。 誰かがピルスナーウルケルビールを試飲することを目標に、誰かが主に傍観者の話をする目的で旅行したことは確かです。 私はそれが好き/嫌いだと書いた。



だから、私たちはポイントに関するすべてのスピーチを検討し始めます（多分私は何かを忘れましたが、それは私が間違いなくそれを本当に好きではなかったことを意味します）。



  • 最初に話すのはDmitry Stogov（Zend）で、PHP 5.3 / 6.0の新機能について話しました。 要するに：

    • 主なもの-5.3は、Unicodeがない場合のみ6.0と異なります。

    • 最も重要な革新の1つは名前空間です。 十分な名前空間の文法が導入されています。

    • その後、ガベージコレクターが表示されます。これは、長期間有効なPHPアプリケーションに非常に役立ちます。

    • 嬉しいことに、短い演算子？：（$ C = $ a？$ A：$ b; $ c = a ?: $ Bと同じ）; ）

    • また、誰かが静的データへの動的アクセスを必要としていた可能性もあります。 動的な名前（$ classname :: method（）、$ classname :: $ prop）でクラスの静的要素にアクセスします。

    • 非常に面白いこと-Java jarに似たpharパッケージ。 確かに、通常のパッカーの実装はまだありませんが、どういうわけか既に動作しています。
  • 2番目はミハイル・クラソフスキーで、データ管理について話しました。 彼はいくつかの一般的な言葉を話した、原則として、有用な何かが皮質下に落ちたかもしれないが、彼はほとんど眠りに落ちた。
  • その後、Dmitry Stogovが再び登場し、PHPの内部構造（Cで）と、同じCでPHPのライブラリを作成する方法について話しました。 非常に興味深いパフォーマンス。 私がそれから学んだ主なこと-PHPの特定の関数が遅い場合、より速い言語で実装を書くことはかなり可能です。
  • Ilya Kantor氏は、ブラウザに十分な量のデータを保存するための既製のライブラリがあると述べました（クッキーのように、より多くのボリュームのみ）。 同様のトピックに関する記事はすでにハブにあります（window.nameプロパティに格納されています）が、イリヤは別の解決策を提案しました-最近登場したブラウザー永続性（DOMのデータストレージ）を使用し、これをサポートしないブラウザーにはデータを保存します

    フラッシュ経由。 それが本当にどこでも動作するのであれば、多くのアプリケーションを見つけることができます。 ライブラリはすでに作成されており、プレゼンテーションにはダウンロードするためのリンクがありました。
  • Alexey BorzovはPostgreSQLについて話しました。 PostgreSQLデータベースについて多くのことを聞きましたが、まだ触れたことはありません。 このパフォーマンスの後、私はまだ彼女に会いたかった。 Alexeyは、PostgreSQLのクエリオプティマイザー、およびクエリをデバッグするための可能な方法について十分に詳しく説明しました。
  • Anton Terekhov（RBC）は、PHPconfの初日に個人的に私にとって最も興味深いパフォーマンスを発揮しました。 彼は、photofile.ruプロジェクトのライフストーリーで、増え続ける高速パッチと、アーキテクチャの完全な変更にどのように対処しようとしたかについて語りました。 Webサーバーとして、nginxへのプロキシを備えたクラスターがそこで使用され、負荷が分散されますが、このような難しいプロジェクトではphpがfast-cgiではなくapacheを介して実行されることに気付きました。
  • Bolshov Victorは、flv-videoの翻訳をサイトで簡単にアレンジする方法を話しました。 トリッキーなことはありません-ビデオの視聴を開始する位置をREQUESTリクエストで送信し、この位置からサーバーからflvファイルを送信します。 また、クライアントでは、ビデオはFlashプレーヤーを受け入れます。Flashプレーヤーのオープンソース実装もプレゼンテーションで名前が付けられました。
  • それからUMIの若い男が話しました。 彼は、XSLTでテンプレートを使用するのがどれほど良いかを語った。 このトピックは非常に興味深く、レポートは非​​常に不明瞭でしたが、それでも有用です。 スピーチの後、ほとんどの質問はスピードに関するものでした。これは、このテクノロジーの障害です。
  • 最後の話では、MySQLの開発者が、オブジェクト間の距離に制限のある地理座標で検索を整理する方法について話しました。 トリッキーなものはありません-2つの式が導き出されました-1つは正確（望ましい半径の円の場合）、もう1つはラフ（2つの半径を持つ辺の場合）、地理座標による距離を表します。 私が注意した主なアイデア-非常にトリッキーな式に依存する何かに基づいて検索する必要がある場合、最初に選択の候補の数をより粗いがはるかに単純な計算式に制限する必要があります（インデックスを使用するように） ）、そしてその後、正確な式を適用します。





2日目は、はるかに豊かで興味深いことがわかりました。



  • 最初のペースは、OOPの基礎を広めたMarcus Burgerの最初のレポートによって設定されました。彼らは明らかでしたが、英語でITトピックに関する講義を聞くことは良い習慣となりました（彼らは同時翻訳のために受信者を配りましたが、実際の翻訳でこれを聞くことはばかげているように見えました）。 その後、より複雑で興味深い部分が始まりましたが、一部の場所では理解できない部分もありました。 彼は、プログラミングパターン、イテレータ、その他のさまざまな賢いことについて話しました。 プレゼンテーションから得た主なアイデア-PHPプログラミングパターンについて読む必要があります。 2つ目：PHPプログラマーでもGoogleにアクセスできます。
  • その後、マイクロソフトの代表者が話しました。 最初のジョーク（Alexey Shulenin）は、無料のSQL Server 2005を使用しても意味がないと確信しました（ただし、私は反対を試みました）。 2番目（Andrei Beshkov）は、新しいスクリプト言語であるPowerShellについて非常に興味深い話をしました。 コンソール用のスクリプトを書くのは、Bashのような魔法のようなものではなく、プログラミングのようになりました。 主な機能は、言語がオブジェクト指向になったことです。 つまり、標準のps要求でも、PowerShellはオブジェクトのコレクションを返します。 最後に、Windowsサーバー管理者が通常コンソールでサーバーを管理できるようになります。 そして、サイズが100 MBのGUIなしでリリースされたWindowsを追加すると、Microsoftに関して私の魂の中で一般に何らかの形で暖かくなります。
  • Andrey Nigmatulinは、非常に興味深いことにphp-fpmについて話しました。 fast-cgi（nginxなど）を簡単に実行できるようにするPHPのパッチ。 また、私はそれについて初めて学びました（管理作業は行いません）が、レポートは非​​常に役立ちます。
  • その後、MySQLから2つのひどいパフォーマンスがありました。 1つ目はMySQLバグレポートからバグのリストを読み取ることに基づいており、2つ目は愚かな質問に答えることに基づいています（次のトピックで質問したKhabrovitesに聞かせてください）。 2番目のパフォーマンスは、プログラマーの1人によって実際に保存されました。プログラマーは、（見たところ）興味深い情報を持ち、将来の開発のための計画を共有した唯一のプログラマーでした。
  • 次のプレゼンテーションは、私にとってスフィンクス技術の発見でした。 彼女については何も聞いたことがありませんでしたが、今ではそれが存在することだけでなく、それが一般的にどのように機能するかを学びました。 Sphinxは、主に全文検索に焦点を当てたデータベースであり、たとえばMySQLよりもはるかに高速です。 さて、スピーカー（Andrey Aksenov、Sphinxの作成者）は単にアニーリングしました。
  • 最後のプレゼンテーションは、XMLベースのSedna 3.0データベースの概要であり、完全にXMLに基づいています。 クリモフ・ユージーンは彼女について非常に興味深い話をしましたが、馬鹿オーガナイザーは彼を止めて、面白い話を聞く代わりに3時間妄想的な賞品を提供し始めました。



    データベース機能-すべてがXML上に構築されています。 ツリーデータを保存するためのグラウンド。 DB2 Xmlがリレーショナルデータベースのデータ型に単純に埋め込まれている場合でも、リレーショナルデータベースの理解にはテーブルはまったくありません。 一般的に、非常に興味深い技術です。
