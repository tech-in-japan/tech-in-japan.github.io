オープンな電子学習プラットフォームMoodleのサーバーへの迅速なインストールと公開

ますます便利な開発ツールの出現により、Webアプリケーションでの作業は誰にとってもよりシンプルで手頃な価格になっています。 たとえば、今日、WebMatrix開発ツールを使用して、数十の人気Webアプリケーション（Drupal、BlogEngine.NET、DotNetNuke、WordPress、Orchard、Joomlaなど）を数分でアップロード、編集、サーバーに公開できます。







この記事では、WebMatrixを最短時間で使用して、人気のあるMoodleオープンソース教育プラットフォームに基づいてプロジェクトを自動的にダウンロード、インストール、公開する方法を説明します。



確かにあなたはすでにMoodleについて聞いています。そうでない場合は、Habréに関するこの記事をよく理解することをお勧めします。





WebMatrixとは何ですか？



WebMatrixは、Webアプリケーションを開発、展開、および保守するための無料の軽量ツールです。 WebMatrixインストールパッケージはわずか8メガバイトで、数秒でインストールされます。 WebMatrixは、IIS Express 7.5サーバー、SQL Server Compact 4.0コンパクトデータベース、Web Deploy 2.0プロジェクト自動展開ツールなどの最新のMicrosoftテクノロジーで動作するように構築されています。





図 1. WebMatrix 2の 開始ウィンドウ



WebMatrixは、Web開発者向けに次の機能を提供します。

  • Webアプリケーションプロジェクトを最初から作成するか、基本的なテンプレートに基づいて作成します。
  • 一般的なWebアプリケーションテンプレートをリモートストレージからダウンロードする
  • テストおよび自動データロギング中のWebアプリケーションへのリクエストのパラメータの組み込み追跡。
  • Webアプリケーションコードの組み込みエディター：構文の強調表示およびその他の標準機能を備えたスクリプト、スタイル、マークアップなど。
  • データベーススキーマの組み込みの本格的なエディター：データベースとテーブルの作成、データベース構造の削除と編集、データベース内のデータの入力、削除、編集。
  • ページを分析し、検索エンジン最適化の問題、レイアウトエラー、壊れたリンク、その他の多くのパラメーターに関するレポートを作成することにより、組み込みのプロジェクトをサポートします。


特定のプロジェクトで作業する場合、WebMatrixは、関連するリソースへのクイックリンク（現在のプロジェクトの管理パネルへの特別なリンクなど）を開発者に提供するように構成できます。 図2では、Moodleベースのプロジェクトで作業しているとき、WebMatrixコントロールパネルにボタンの形でそのようなリンクを見ることができます。 この場合、環境は、Moodle管理パネル、モジュール、プラグインへのアクセスを提供する特定のタイプのプロジェクト専用に拡張されます。 さらに、WebMatrixでダウンロードしたMoodleプロジェクトのメインページには、エディターに統合された便利なリソースへのリンクがあります：公式Webサイト、ニュース、サポート、フォーラム、コミュニティ、ドキュメント、APIなど。





図 2. WebMatrixでMoodleを使用する



WebMatrixを特定のタイプのプロジェクトに拡張および適応させる機能により、このエディターは異なるシステムで作業するために特別になります。この場合、LMS Moodleの特別なエディターです。





図 3. WebMatrix でコードを操作する



WebMatrixの重要な機能は、Webアプリケーションを作成できるだけでなく、ライフサイクル全体でWebアプリケーションを使用できることです。 したがって、WebMatrixは、Webアプリケーションの設計から開発までの段階だけでなく、アプリケーションが実際のプロジェクトとして機能するときにも役立ちます。





図 4. WebMatrix でデータベースを操作する



WebMatrixは、Web開発者向けの他の重要な開発ツールと同様に、Web Platform Installerの自動インストールツールから自動ダウンロードできます。 このMSDNブログ投稿からWeb Platform Installerをダウンロードして使用する方法を学ぶことができます。





図 5.プロジェクトの分析と WebMatrix でのレポートの操作



WebMatrixに関する多くの情報は、公式Webサイトhttp://www.microsoft.com/web/webmatrix/およびhttp://www.asp.net/WebMatrixで見つけることができます 。 ロシア語のWebMatrix開発者コースは、MSDNで見つけることができます。



Moodleを自動的にダウンロードしてインストールします



Moodleプロジェクトは非常に簡単にWebMatrixにアップロードできます。 開発環境を開始し、開始ウィンドウ（図1）で[ Web ギャラリー からのサイト]（ ギャラリーのテンプレートに基づくWeb サイト ）を選択します。



リモートリポジトリで利用可能なウェブアプリケーションテンプレートのリストが表示されたウィンドウで、Moodleを見つけます（図6）。 すばやく検索するには、CMSセクションを選択します。





図 6. WebMatrixで Moodle エンジン を 選択する



[サイト名]フィールドにサイト名を設定し、[OK]をクリックします。 Moodleがコンピュータ上で動作するために必要なコンポーネント、たとえばMySQLデータベースサーバーがない場合、WebMatrixはこれを自動的に検出し、必要なツールのインストールを提案します。 参加しなくても、不足しているコンポーネントのインストールが自動的に行われます。



その後、WebMatrixは、ダウンロード可能なテンプレートのライセンスに精通し、インストールのためにリポジトリ（図7）からダウンロードする必要があるデータのサイズを教えてくれます。





図 7.インストール前の情報ウィンドウ



[同意する]ボタンをクリックして、すべてのコンポーネントの自動ダウンロードとインストールを開始します。 数分後、すべてのコンポーネントがインストールされ、その後、Moodleサイトの初期データを設定する機会が提供されます。 ここでは、サイトのデモデータをインストールする必要があることを示すこともできます。



必要なサイトデータを入力し、[次へ]ボタンをクリックします。 数秒後、Moodleの実行に必要なすべてのコンポーネントがシステムに自動的にインストールおよび設定されたという通知を受け取ります。 さらに、サイトに自動的に設定されたすべてのログインとパスワードを受け取ります（図8）。





図 8.コンポーネントの正常なインストールの通知



[OK]をクリックすると、ロードされたMoodleベースのすぐに使えるプロジェクトを備えたWebMatrix開発環境が表示されます。 メインのWebMatrixウィンドウは、LMS Moodleに関する追加情報（リンク、ドキュメント、フォーラム）を提供するようにカスタマイズされていることに注意してください（図9）。 サイトを開始するために必要なことは、「実行」ボタンをクリックすることだけです。





図 9. WebMatrixのメインウィンドウ



Moodleアプリケーションを起動すると、ブラウザで開き、システムに必要な初期セットアップ手順を実行できます（図10）。









図 10. Moodleの初期セットアップ



Moodleの初期初期化が完了したら、管理パネルに移動してサイトを手動で設定できます（図11）。 承認後、コントロールパネルにアクセスできます。





図 11. Moodleベースのサイトのメインウィンドウ



作業中のMoodleベースのプロジェクトにアクセスするために、インストール段階で入力した管理者ユーザー名とパスワードを使用してサイトにログインします。 ログインとパスワードを入力すると、Moodle管理パネルが表示されます（図12）。





図 12. Moodle管理パネル



コントロールパネルとローカリゼーションMoodle



デフォルトのMoodleベースのプロジェクトは英語のローカライズで動作します。 サイトをロシア語で機能させるには、いくつかの簡単な手順を実行する必要があります。



Webサイトの管理パネルに移動し、パネルで[言語]を選択してから[言語パック]を選択して、追加モジュールをロードする機能に切り替えます（図13）。





図 13.追加モジュールのダウンロードフォーム



利用可能な言語パックのリストで、ロシア語を選択し、「選択した言語パックのインストール」ボタンをクリックします。 しばらくすると、新しい言語パックが最初のリストに追加されます。 その後、[言語設定]セクションに移動し、デフォルトの言語設定でロシア語を設定します（図14）。





図 14. Moodle設定でロシア語をインストールする



[変更を保存]ボタンをクリックします。 これで、サイトのヘッダーのコントロールを使用して、いつでも複数の言語を切り替えることができます（図15）。





図 15.言語インターフェースMoodleの選択



ロシア語を選択してください。 これで、Moodleベースの管理パネルのインターフェース全体がロシア語にローカライズされました（図16）。





図 16.ローカライズされたMoodle管理パネル



サイトをサーバーに公開する



WebMatrixを使用して必要なすべてのコンポーネントをダウンロード、インストール、設定し、Moodleに基づいてサイトを起動、設定したら、インターネット上で利用できるようにサーバーに公開します。



WebMatrixは、サイト、そのファイル、およびデータベースをサーバーに公開するすべての日常的な作業を行い、自動的に実行できることをお勧めします。 サイトを配置し、[公開]ボタンをクリックするサーバーのパラメーターのみを知っている必要があります。



[公開]ボタンをクリックすると、Moodleサイトを展開するサーバー設定を指定するよう求められます（図17）。





図 17.公開用のサーバーデータの入力



インターネットでサイトを公開するには、独自のサーバーを使用するか、WebMatrixを介した簡単な公開をサポートするホスティングプロバイダーのサービスを使用できます。 たとえば、無料サービスhttp://webmatrixhosting.ru/を使用できます 。 このサービスを使用すると、最大60日間の無料ホスティングを利用できます。その後、プロジェクトを有料の料金に移行できます。料金は月額80ルーブルからです。



サーバーのデータを公開用に指定すると、WebMatrixはWebサイトがインターネットで動作するために必要なすべてのファイルとデータベースの公開を開始します。 サーバーにデータを転送する前に、WebMatrixはサーバーに公開されるすべてのファイルのリストを通知します（図18）。





図 18.公開するファイルのリスト



ファイルを選択して、公開したくないファイルを公開または非公開できます。 [続行]をクリックすると、WebMatrixはサーバーへのファイルの公開を開始します。 公開はバックグラウンドで行われるため、環境内でサイトを引き続き使用できることに注意してください。 現在のパブリケーションのステータスはウィンドウの下部に表示されます。これにより、ファイルのパブリッシュ方法を監視し、パブリケーションの完了時期を確認できます（図19）。





図 19.サーバー上のサイトの公開ステータス



正常に公開されたら、インターネット上のサイトで作業を開始できます。



おわりに



ご覧のとおり、WebMatrixを使用すると、サイトテンプレートやMoodleなどの管理システムのダウンロードとインストールが大幅に簡素化されます。 WebMatrixは、サイトが機能するためにコンピューターにインストールするために必要な依存関係を決定するすべての作業を引き受けます。 WebMatrixはこれらの依存関係を自動的にダウンロードしてインストールします。



WebMatrixは、Moodleなどのプロジェクトを操作するための豊富なツールサポートを提供します。 コード、データベース、テストサイトなどのコード品質を編集できます。



WebMatrixはサーバー上での公開を大幅に簡素化します。これにより、ボタンのクリックになります。 したがって、データをサーバーに公開する際の大量のルーチン作業を回避し、時間を節約できます。 これらすべてにより、WebMatrixは、最も訓練を受けていないユーザーでも使用できるツールになります。



ご覧のとおり、WebMatrixを使用してMoodleサイトをダウンロード、インストール、公開するのに数分かかりました。 これにより、WebMatrixは非常に便利なツールになります。
