Lenovoハードウェアを使いこなし、XClarityのニュアンスに驚く







デザート用のシステムソフトウェアに関する一連の記事では、有名な青い巨大なモンスター、Lenovo XClarityとIBM Systems Directorが残っています。







IBMアプリケーションのドキュメントを読むたびに、マルチステージクエストの通過と強い関連があります。 ただし、技術プレゼンテーションの徹底に対する影響を相殺し、IBMの監視モンスターの機能を理解します。







IBM x3650 M4マシンは、次の鉄の購入後にサーバーモールに引き継がれ、驚くほど燃えるような赤色光を備えており、研究の患者として機能します。 もちろん、サーバーにはほとんどの状態を示す独自のLight Pathがありますが、独自の監視アプリケーションでさらに多くの情報を収集できます。







IBM / Lenovoサイトの荒野には、次のハードウェア管理ツールがあります。







  • Lenovo XClarityは、IBMおよびLenovoサーバ​​ー用の中央リソース管理システムです。 マシンの構成と更新、機器のステータスに関する通知の受信、およびリモートでの管理ができます。







  • Lenovo XClarity Integrator for VMware vCenter-VMware vCenterコンソールから直接仮想化ホストを管理するためのツール。







  • Lenovo XClarityモバイルアプリ -IBM / Lenovoマシンを監視、電源管理、再起動するためのモバイルアプリケーション。







  • IBM Systems Directorは、XClarityの前身であり、より控えめな機能セットを備えています。


これらのツールについてさらに詳しく説明し、それらの機能と管理者の潜在的な関心について簡単に説明します。 IBMと長い間仕事をしているのであれば、この記事の大部分は啓示にはなりません。 この場合、コメントでIBMシステム管理ツールのセットを共有していただければ幸いです。







XClarity-レノボのパーソナルモンスター



Lenovo XClarity機器監視および制御システムは、有名なIBM Systems Directorパッケージに取って代わりました。 IBMおよびLenovoのサーバーおよびネットワーク機器に関する情報を自動的に収集し、いくつかの設定を行うことができます。 XClarityは、IMM機能を直接使用して、エージェントなしでネイティブ機器と連携できます。







すぐに重要なニュアンスに注意してください-XClarityは通常IBM / Lenovoサーバ​​ーシステムのみを監視できるため、このソフトウェアは異なるベンダーの機器が混在する環境では面白くありません。







IMMは、Dellサーバー用のiDRACまたはHP用のiLOに類似した、IPMIリモート制御プロトコルの独自の実装です。







次のシステムがサポートされています。







  • IBM System dx360 M4、タイプ7912、7913;







  • IBM System x3300 M4、タイプ7382;







  • IBM System x3500 M4、タイプ7383;







  • 中間では、Cisco Nexusスイッチを使用できます。


XClarityは、監視システムの通常の機能をすべて実行し、問題のより詳細な調査のためにSyslogでの記録をサポートします。 時代の精神では、プッシュ通知を介して監視メッセージを受信するためのモバイルアプリケーションもあります。







IMMコンソールの監視とリモート接続に加えて、システムは、委託されたシステムのいくつかの構成タスクを実行できます。 たとえば、XClarityを使用すると、最大28のオペレーティングシステムを同時にインストールしたり、テンプレートを使用して共通の分母にマシン構成を設定したりできます。 最も大胆で決定的なものとして、大規模なファームウェア更新も利用できます-一度に最大50台のサーバー。







構成テンプレートでは、ストレージおよびネットワークアダプターのパラメーター、必要なファームウェアバージョンを指定し、ロードするオペレーティングシステムを選択して、一般的なパラメーターIMM、BIOS / UEFIを割り当てることができます。







テンプレートに加えて、ポリシーもあります。 彼らの助けを借りて、サーバーシステムの内部IT標準への準拠を確認できます。 XClarity Administratorは、使用されているファームウェアの不一致について警告し、管理者の承認後にそれらを更新します。 テンプレートと一緒に、それはActive Directory for ironのようなものになります。







XClarityは、VMware vSphereおよびMicrosoft Hyper-V向けに事前構築された仮想マシン形式で提供されます。 以下の表では、利用可能な2つのエディション、AdministratorとProの簡単な比較を示します。

ProバージョンはVMware vCenterとの統合をサポートしています。これにより、後者はハードウェア障害について学習し、たとえばマシンを別のホストに移動できます。







vCenter用Hand Extender



vSphere環境では、バランシングメカニズムに基づいて、またはホストが利用できない場合やリポジトリが表示されない場合、マシンを別のホストに移動できます。 もちろん、ハードウェアに関する詳細情報がある場合、このリストは大幅に拡張できます。







XClarityを使用すると、トリガーのリストが次のように補完されます。







  • プロセッサ、キャッシュ、バスコントローラー、または外部ホストインターフェイスの問題。







  • RAMの負荷。







  • 電源サブシステムの問題。







  • コンポーネントの過熱;







  • ドライブ、RAIDコントローラーの問題。







  • ファームウェアが正しくインストールされていません。


これらの状況はすべて、仮想マシンの移行ポリシーによって制御できます。 ポリシー自体は一度設定されると、XClarityはすぐにトラブルを予測します。







マシンを別のホストに自動的に移動するには、vSphereクラスターが完全自動DRS（Distributed Resource Scheduler）モードで動作する必要があります。 さらに、すべての監視サブシステムを適切に動作させるには、Lenovo ESXiアセンブリを使用することをお勧めします。







vCenter統合プラグインは、 XClarityバージョンのProで使用できますが、XClarity Administratorの個別のコンポーネントとして購入することもできます。 そのようなライセンスの費用は、ホストごとに約406.99ドルで、3年間の更新とテクニカルサポートのサブスクリプションが含まれます。







次のパラメーターを使用して、物理マシンと仮想マシンの両方にインストールできます。







  • 構成されたIIMインターフェイス。







  • バージョンESXi 4.1-6.0、およびLenovoからのESXiのより良いビルド。







  • オペレーティングシステム：Windows Server 2008（x64）-2012 R2;







  • 少なくとも4 GBのRAM、8 GB以上。







  • ディスク上に少なくとも20 GB、40 GB以上。







  • 1-2プロセッサー。


XClarity Administratorは、小さなサーバーラックのインフラストラクチャから始めると便利です。 サーバーが大きいほど、効果は強くなります。 残念なことに、オペレーティングシステムのドライバーを更新する方法がわからず、サードパーティの機器を監視できません。







サーバーコントロールパネル



XClarity Mobileモバイルアプリケーションを使用すると、AndroidまたはiOSスマートフォン上のすべてのIBM / Lenovoシステムから情報を受信できます。これは日常業務に役立ちます。













たとえば、作業中の輸送中に、ファームウェア更新がスケジュールどおりにどのように機能したかを確認できます。 また、コンピュータールームの反対側のサーバーから「怠け者」として使用することもできます。スマートフォンのボタンをクリックして、サーバーを再起動するように送信しました。 または、目詰まりしたラックをすばやく検索するために、マシンインジケータライトをオンにします。 一般に、十分なスクリプトを考えることができます。Lenovoがマーケティング資料で教えているとおりに行う必要はまったくありません。







このアプリケーションは、 Android 4.4以降のスマートフォン、またはiOS 8以降のデバイスで使用できます。 動作するには、XClarityを実行しているネットワークへのVPNが必要です。







無料のXClarityの代替



IBM Systems Directorは、ライフサイクル管理システムの「祖父」と見なすことができます。その歴史は1993年にOS / 2システムにまで遡ります。 現在では、物理サーバーシステムおよび仮想サーバーシステム用の無料の監視システムであり、公式にはサポートまたは開発されていません。







Systems Directorは、パッシブ（プラットフォーム）およびアクティブ（共通）エージェント、またはWMI、SNMP、IPMI、またはIMMプロトコルを介して情報を収集します。







データは、次のいずれかのタイプのデータベースに保存されます。







  • IBM DB2







  • オラクル







  • Microsoft SQL Server（Expressを含む）。


最新のインフラストラクチャでは、Windows Server 2012を使用することの微妙な違いにすぐに注目する価値があります。このOSを搭載したマシンは、IBMの推奨事項に従って 、特別に構成する必要があります。













新しいXClarityとの違いについては、次のとおりです。







  • XClarityはESXiおよびHyper-Vハイパーバイザーでのみ機能しますが、IBM Systems DirectorはXenをサポートします。







  • 無料版では、Systems Directorの機能は、IBMサーバーハードウェアの監視、アラート、および更新に限定されています。







  • XClarityには、ハードウェア構成とファームウェアバージョンのポリシーへの準拠を確認するための追加機能が含まれています。







  • IBM Systems Directorは、XClarityとは異なり、ドライバーの更新をサポートします。


IBMが製品をさらに改善することを拒否し、XClarityの機能がより制限されたシステムに切り替えたことは奇妙です。 ただし、「老人」配布キットはリクエストに応じて引き続き利用できます 。







合計



IBM / Lenovoの場合、制御ソフトウェアの過剰な選択の目は実行されないため、簡単な推奨事項を提示できます。







  • ご使用の環境でIBM SystemXマシンが使用されている場合、無料のSystem Directorがファームウェアの監視および管理に最適なツールになります。 このアプリケーションを使用すると、SMTP、WMI、およびIPMIを介してデータを受信できるため、サードパーティの機器の監視に接続できます。







  • 新しいLenovoハードウェアでは、特にVMware vSphereまたはMicrosoft Hyper-Vハイパーバイザーとの統合が必要な場合は、XClarityを使用することをお勧めします。
