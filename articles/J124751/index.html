モバイルゲームを開発するためのマルチプラットフォームエンジンの選択（パート2）

続編の出版をわずかに遅らせたが、最初の部分は十分な力を蓄積することができず、それからバム、そしてある種の見知らぬ人が助けてくれた。



ここで最初の部分を読みます： モバイルゲームを開発するためのマルチプラットフォームエンジンを選択する（パート1）



今回は、導入部分を省略して、すぐに患者に進みます。



cocos2d-x


ウェブサイト： www.cocos2d-x.org

ドキュメント： Wiki 、 API

サポートされるプラットフォーム： iOS、Android、WoPhone。

プログラミング言語： C ++、Lua。

最低価格：無料（MITライセンス）。

デモ版：すべてが無料であるため、ダウンロードして使用します。



Cocos2d-xは、cocos2d-iphoneゲームエンジンのマルチプラットフォームポートです。 C ++プログラミング言語。 Wikiでは、obj-cのコード例とC ++の類似物を見ることができます。



Luaサポートがcocos2d-0.99.5-x-0.8.5に追加されました（エンジンのすべてのクラスとメソッドが移植されたと言われています）。 ただし、Lua Wikiには1ページしかなく、それも空白です。 Luaフォーラムも混雑していません。強大な中国語のトピックが1つだけです。



一般に、ドキュメントとサンプルを含む他のエンジンと比較すると、Cocos2d-xには明らかな問題があります。 一方で、その自由と著者が彼らの祖先（cocos2d-iphone）の開発に遅れをとっていないという事実は喜ぶことはできませんが。



UDK（アンリアル開発キット）


ウェブサイト： www.udk.com

ドキュメント： www.udk.com/documentation

サポートされているプラ​​ットフォーム： iOS、PC、ゲームコンソール。

プログラミング言語： UnrealScript

最低価格：アプリケーションから50,000ドルを獲得するまで無料。この金額を超えるすべての収入については、UDKライセンスに対して25％+ 99ドルのライセンス料を支払う必要があります。

デモ版： www.udk.com/download



UDKは有名なゲームスタジオEpicの作成物で、最近IOSのサポートを開始しました。 このエンジンには、組み込みのビジュアルエディターがあり、通常、あらゆる種類の機能が多数あります。 UDKのゲームは非常に美しく（Infinity Bladeを参照）ですが、これは主にゲーム開発者のメリットとそれに費やされる予算です。



PS最後の投稿へのコメントで、 EntropiouSはUDKでのAndroidのサポートについて言及しましたが、この確認は見つかりませんでした。 確かに、EpicはいつかAndroidのサポートを追加することを約束すると言っておく価値があります。 ですから、この重要なイベントを見るために私たちが生きるあらゆるチャンスがあります。



Deltaエンジン（2012年半ばにリリース予定）


ウェブサイト： deltaengine.net

ドキュメント：開発者は、この点に関してすべて準備ができていることを約束し、エンジンがリリースされるとすぐに、すべてのドキュメントと完成したゲームの例を公開します。

サポートされているプラ​​ットフォーム： iOS、WP7、Android、PC、Mac、Linux、ゲームコンソール。

プログラミング言語： C＃

最低価格：開発者は、コードがオープンであり、PCの開発は完全に無料であることを約束しますが、他のプラットフォームには費用がかかります。 ライセンス 。

デモ版： -



Deltaエンジンはまだリリースされていませんが、有望に見えます。 開発者はあらゆる種類のニシュチャコフの多くを約束します。 プラットフォームのネイティブな機能を使用し、開発のシンプルさで終わる、異なるプラットフォーム用のアプリケーションの単純なアセンブリから始まり（正しく理解できれば、Windows上のiOSのアセンブリのためにリモートで実行できます）。



PSマルチプラットフォームを確保するには、MonoTouch、MonoDroidを使用します。



モアイ


ウェブサイト： getmoai.com/moai

ドキュメント： getmoai.com/docs

サポートされるプラットフォーム： iOS、Android。

プログラミング言語： Lua

最低価格：無料（CPALライセンス）。 さらに、ゲームのクレジットでは、ゲームがMoai SDKで作成されていることを示す必要があります。

デモ版：ダウンロードするには登録が必要です。



モアイはまだ若いですが、開発者の計画はナポレオンのようです。 Moaiは、もう1つのゲームエンジンとしてだけでなく、クライアント（エンジン自体）とサーバーパーツ（Moai Cloud）、および収益化と分析を備えたプラットフォーム全体として考えられています。 一度にすべてを使用する必要はなく、エンジンのみを採用できます。



ゲームの作成にはスクリプト言語Luaが使用され、ゲームのサーバー部分とクライアント部分の両方がLuaで作成されます。 エンジンAPIはまだそれほど広範囲ではありませんが、独立して拡張できます（コードは開いています）。
