Google、GMailでの私の位置はどこですか？ GMailでショートカットがどのように機能するかを正確に知っていますか？



Googleが提供する15ギガバイトの空き領域のうち、私のメールはすでにほぼ12ギガバイトを占めていることに気付き始めました。 そして、この傾向は私を喜ばせません。

一方、私は完全に同期されたThunderbirdをメールクライアントとして使用しています。 つまり すべての手紙をアップロードする必要があります。 したがって、すべての文字とインデックスを含むThunderbirdフォルダーは、わずか3ギガバイトしか使用しません。 物事の論理によると、サイズは多かれ少なかれGMailの占有場所と一致するだけでなく、大きくなるはずです。 Thunderbirdはレターをアーカイブしませんが、そのまま保存し、インデックスを作成して検索を高速化します。

問題は顔にあります！ 私たちは問題の底に到達し始めます。



最初にショートカットに移動しました（はい、GMailの場合、フォルダーではなくショートカットを言うのが正しいです。詳細はこちら ）「すべてのメール」で、50万件を少し超えるメッセージがあることがわかりました。 状況は、私が約100個のショートカットを持っているという事実によって複雑になりました！ また、GMailのショートカットは、Thunderbirdの典型的なフォルダーです。 Thunderbirdで文字の総数をすばやく計算する方法が見つかりませんでした。 しかし、先を見据えて、私はその中に約20万人いると言います。 ここから、ディスク上のスペースが少ない理由が明らかになります。

ただし、同じ質問が残っています。Gmailには、Thunderbirdには表示されないがGMailのある場所を占有する30万のメッセージがありますか？



好奇心mind盛な心+夜寝ないという欲求+実際のタスクでGoに触れたいという欲求から、Goコンパイラを使用し、 GMail APIを研究し、 GMailの裏側を確認する必要があるという決断に至りました。

上で述べたように、約100のショートカットがあります。 通常、メールには1つのラベルがあります。 そして、各ラベルでマークした文字の数と、それらが合計でスペースを占める量を調べたいと思いました。

GMail Webインターフェース（1つまたは別のラベルでマークされた文字のボリューム）でラベルのサイズを見つける方法が見つかりませんでした。

袖をまくり、Goコンパイラをインストールし、DockerでMongoDBコンテナーを選択しました（はい、私はそのような変態です！しかし、これは私のペットプロジェクトであり、私が欲しいものです、特にトレーニング目的で使用します）。

さらに、私はこのプロジェクトを参照します。

すべてのタグをGMailで取得し、 Users.labels：リストデータベースに配置します。 

GMailMessagesSize -importLabels -mongoConnectionString 10.211.55.5 Imported labels: 112


Users.messages：リストボックスにあるすべてのメッセージのIDを削除します。 

 GMailMessagesSize -mongoConnectionString 10.211.55.5 -importMessages Processed 100 messages Processed 200 messages Processed 300 messages ....... Processed 523100 messages Processed 523115 messages


もちろん、すぐには得られませんが、並列化する方法を見つけることができませんでした（APIでは許可されていません）。

これまでのところ、メッセージIDのリストのみがあり、各メッセージのラベルとサイズを知る必要があります。 Users.messages：これのためのgetメソッドがあります。 しかし、リクエストで興味のあるフィールド（internalDate、labelIds、sizeEstimate）を正確に示しているにもかかわらず、すぐにはうまくいきません。

Batching Requestsの実装が見つかりませんでした。

しかし、私はGoに書き込みます。ゴルーチンを使用しないのは罪です。 すぐに言ってやった。 情報をスレッドの数に引き込みます（必要なだけですが、50の制限を設定します）。 インターネットが高速でコンピューターが愚かでない場合、Googleからのリクエストのレート制限にすぐに依存し始めます。 スクリプトを停止して続行することもできますが、単に待つことができます。 制限がトリガーされると、ゴルチンは5秒間スリープし、Googleを苦しめ続けます。 はい、毎回、たとえば2回スリープ時間を増やして、上記の制限を忘れないようにすることができます。 ただし、この場合は、簡単な5秒で十分です。

合計で50万通の手紙を処理しましたが、約3時間で完了したようです。 一般的に、健全な時間。

ところで、検索バーに、特定のラベルが1つしかないメッセージを表示する方法が見つかりませんでした。 つまり たとえば、CATEGORY_PROMOTIONSというラベルがあり、その他のラベルがないメッセージをすべて削除することにしました。 アーカイブにこれらの広告レターは絶対に必要ありません。 ところで、いくつありますか？ 

 GMailMessagesSize -mongoConnectionString 10.211.55.5 -showSizes -l CATEGORY_PROMOTIONS -onlyThisLabel LabelId;Label name;Messages size;Messages count CATEGORY_PROMOTIONS;CATEGORY_PROMOTIONS;1197364170;14618


ギガバイトごとにそれらを蓄積しました。

-onlyThisLabelは重要なオプションであり、この単一ラベルを持つメッセージのみを検索できます。 

 GMailMessagesSize -mongoConnectionString 10.211.55.5 -showSizes -l CATEGORY_PROMOTIONS -l IMPORTANT -onlyThisLabel LabelId;Label name;Messages size;Messages count CATEGORY_PROMOTIONS;CATEGORY_PROMOTIONS;1197364170;14618


はい、まだ1.5ギガバイトの「重要な広告」メッセージがあります:)これは、ギガバイトの重要でない広告に加えて追加されていることに注意してください。

手はすぐに彼の髪をとかし、それをすべて取り除きました！ 

 GMailMessagesSize -mongoConnectionString 10.211.55.5 -deleteMessages -l CATEGORY_PROMOTIONS -l IMPORTANT -onlyThisLabel -procNum 10


実際、手紙は削除されず、バスケットに入れられます。 そこに、30日後に、それらは完全に消えるか、または手動で自分自身をきれいにすることができます。

アプリケーションを機能させるには？
  • このウィザードを使用して、Google Developers Consoleでプロジェクトを作成または選択し、APIを自動的にオンにします。 [続行]、[資格情報に移動]の順にクリックします。
  • ページの上部で、[OAuth同意画面]タブを選択します。 電子メールアドレスを選択し、まだ設定されていない場合は製品名を入力し、[保存]ボタンをクリックします。
  • [認証情報]タブを選択し、[認証情報を追加]ボタンをクリックして[OAuth 2.0クライアントID]を選択します。
  • アプリケーションの種類として[その他]を選択し、「Gmail APIクイックスタート」という名前を入力して、[作成]ボタンをクリックします。
  • [OK]をクリックして、結果のダイアログを閉じます。
  • クライアントIDの右側にある（ダウンロードJSON）ボタンをクリックします。
  • このファイルを作業ディレクトリに移動し、client_secret.jsonという名前に変更します。
