SDHCの問題と解決策



おそらく、多くはEEE PCでSDHCカードを記録する問題に直面していました。 メモリカードが正常に機能する方法について説明します。

キングストンのかなり一般的な商標であるメモリカードがあり、容量は4 GB、クラス6です。100MBを超えるファイルを書き込もうとすると問題が始まりました。 しばらくして、記録エラーに関するメッセージが表示された後、記録インジケータがハングしました。 最初は、カーブカードを考えた後、XPで正常に機能することを思い出しました。 彼は再びフォーラムを探し始めました。 多くのメーカーがこのような問題に直面していることが判明しました。主に、4GB以上の容量を持つSDHCカードです。 フォーラムで、EEE PCでサポートされているメモリカードのリストを見つけました。 私のカードはリストに載っていませんでした。 徹底的な検索の後、私は問題の最初の解決策に出会いました。

  1. コンピューターを再起動します。
  2. 「 F2 」をクリックしてBIOSに入ります。
  3. 「 詳細 」タブを選択します。
  4. 項目「 OSインストール 」が「 終了 」から「 開始 」に変更されます。
  5. 「 F10 」を押して「はい」を選択します。


それは基本的にそれです。 カードをロードして確認し、すべてが正常に書き込まれていることを確認します。 この解決方法は私には向いていませんでした。 結局のところ、ウェブカメラは動作を停止します。 Finish値をStartに変換すると、USBポートの速度が2.0から1.1に変更されますが、内蔵カメラはUSB 1.1では動作しません。 さらに、書き込み速度も数倍遅くなります。 毎日Skypeを使用する必要があります。BIOSを使用して常に切り替える必要はありませんでした。 この方法は、たとえば、EEE PC 4 serf以下のバージョンの場合に役立ちます。 私は解決策を探し続けました。

次のソリューションはもう少し複雑ですが、EEE PCの作業にマイナスはありません。

モジュール「 eee.ko 」が必要になります。 ターミナルを開き、次のように書きます：



wget eeepc-linux.googlecode.com/files/eeepc-linux-0.2.tar.gz

tar -zxvf eeepc-linux-0.2.tar.gz

cd eeepc-linux /モジュール/

sudo make



ファイル「 eee.ko 」が表示されます。 次に、「/ lib/modules/2.6.22-14-generic/kernel 」にコピーする必要があります



sudo cp eee.ko /lib/modules/2.6.22-14-generic/kernel



次に、depmod -aを実行します



sudo depmod -a



「 /etc/modprobe.d/blacklist 」を編集して、そこからコメントアウトする必要があります「 i2c_i801 」



sudo gedit /etc/modprobe.d/blacklist



次の行を見つけます。



ブラックリストi2c_i801



に置き換えます：



#blacklist i2c_i801



「 / etc / modules 」の編集



sudo gedit / etc / modules



最後に、次を追加します。



i2c_i801

ええ



これで、次回の起動時に、 eeeモジュールが自動的にロードされます。 しかし、それだけではありません。

「 /etc/init.d/checkfs.sh 」の編集



sudo gedit /etc/init.d/checkfs.sh



最後に追加します：

#SDHCカードをチェック用に安全にする

modprobe i2c-i801

modprobe eee

echo 70 24 1> / proc / eee / fsb



保存して再起動します。 今本当にすべて。

書き込み速度を確認できます。



メモリカードに行きます



cd /メディア/データ



dataはメモリカードをインストールしたフォルダです。通常、デフォルトはディスクのみです

ファイルを作成する

dd if = / dev / zero of = test bs = 1024 count = 102400



サイズが100mbのファイルが作成されます。 私の場合、私はこの結果を得ました：
