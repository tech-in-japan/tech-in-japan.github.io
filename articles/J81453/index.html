ドメインのゾーンを選択する

地理的ドメインの委任に関する問題に関連して 、サイトにドメインを登録するのが理にかなっているドメインゾーンを選択するための合理的なアプローチについて、少し言いたいと思います。



この資料には啓示は含まれていません。ウェブマスターの開始を目的としており、一般的なドメインゾーンに関する情報を体系化します。







ドメインを登録するときの考慮事項





多かれ少なかれ責任のあるプロジェクトを始めて、そのプロジェクトのドメインを選択する場合、最初に考えることは次のとおりです。 このドメインに誰が属しますか？



ドメインの所有権の問題は、あなたからお金が引き出されるドメインを登録するゾーンに対してのみ意味があります。 次の理由により、すべての無料ドメインはすぐにドロップアウトします

  • 彼らは、原則として、あなたを含む誰にも属していません。
  • いかなる方法でも、そのパフォーマンスを制御することはできません（更新またはホスティングからホスティングへの転送を含む）。
  • ドメインが機能していない場合、誰にも申し立てることはできません。まず、フリーゾーンでドメインを使用するためのルールで明示的に示されていますか。次に、誰にもお金を払わなかった場合にどのような申し立てを行うことができますか？




したがって、ドメインに基づいて深刻で責任のあるインターネットプロジェクトを作成することは強くお勧めしません 。

  • GENERIC （.ORG.RU、.PP.RU、.AC.RU、.EDU.RU、.INT.RU、これらはRosNIIROSによって提供されます ）;
  • GEOGRAPHICAL （.SPB.RU、.MSK.RUなど、Relcom。DS LLC提供）;
  • その他（.narod.ru、.ucoz.ruなど、さまざまな無料ホスティングサービスによって提供されます）。




有料ゾーンでも、そう簡単ではありません。 ドメインの所有権を誰がどのように管理しているかを知ることが重要です。 特に、 .COM 、 .NET 、 .ORG 、 .BIZ 、 .INFO （いわゆるgTLD ）などの広く普及したドメインゾーンを管理するICANN組織は、ロシア連邦の法的分野の管轄外にあり、たとえば、何も知りません。ロシアの法人について。 .COMドメインを登録して、それがあなたが代表する会社または政府機関の財産であることを保証することはできません。 また、たとえば.EUドメインは、EU市民のみに登録することもできますが、EUの居住者であるシェル企業のロシア市民の登録には積極的に提供されます。 ドメインの所有権に自信がなく、信頼できないことは明らかです。



同様の状況は、他の州（いわゆるccTLD ）の国内ドメイン（ .US 、 .TV 、 .WSなど）であり、 IANAと呼ばれる組織によって管理されています。



とりわけ、受け取った苦情（虐待）に応じてgTLDドメインとccTLDドメインが分類的に分割され、作業能力と多くの神経を回復するためにかなりの時間のリソースが必要な場合があります。



.ドメインについて考えるのは時期尚早です。Runetは技術的にそのようなドメインを使用する準備ができていないため、いつ準備ができるかはまだ明確ではありません。



しかし、国内ドメインの中で、 .RUドメインと.SUドメインは有利に際立っています。

IANAが.RUゾーンの管理を委託している国内ドメインの調整センターは、その規則において、 .RUドメインの所有、廃棄、および使用のすべての法的および技術的側面を明確に説明しています。 .SUドメインの場合、インターネット開発基金の文書によって同程度の明瞭さが導入されました。



2009年8月15日に調整センターが行った管理規則の変更後 エンドカスタマーの.RUドメインを登録するための最小コストをキャンセルすると、 .RUドメインは、完全に手頃な価格で、または完全に無料（ホスティングの購入時）で登録できるようになりました。 同時に、 .SUドメインは依然として高価であり、価格を下げる傾向はありません。



将来のドメインのゾーンを選択する際に留意する最後の要素は、 ウェブサイトのプロモーションのプロセスにおける検索エンジンの「態度」です 。 私はアメリカにあなたに事実を伝えません。あなたのプロジェクトがロシアで、ロシアの聴衆のために推進されているなら、 .RUドメインを登録することをheしないでください。

まとめ



ドメインゾーンの「ヒットパレード」の使用の快適性の降順：

  1. ccTLD
  2. .SU
  3. .COM / .NET / .ORGなど
  4. .TW / .US / .TVなど
  5. .ORG.RU / .COM.RU / .EDU.RUなど
  6. .SPB.RU / .MSK.RU / .NNOV.RUなど
  7. .UCOZ.RUなど
  8. .RF *




さらに短くすると、プロジェクトにccTLDドメインを使用する場合、間違いなく問題は発生しません 。
