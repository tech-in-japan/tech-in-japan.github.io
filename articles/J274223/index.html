Android開発者向けの2015年の結果

2015年は終わりました。つまり、在庫を確保する時が来たということです。 昨年Android開発者にとって何が重要で、便利で、興味深いのかは、ブログでご覧ください。









今年は、多くの新しいデバイス、新しいOSバージョン-Android Marshmallow、および最も人気のあるアプリケーションのデザインの多くの変更（Material Designが始まります）を受け取った一般ユーザーだけでなく、それほど重要ではないと感じたAndroid開発者にとっても非常に波乱に富んだものでした作業環境、ライブラリ、および設計パターンの変更。



最後に、材料設計ガイドラインをほぼ完全にサポートするライブラリがリリースされました-Design Support Library。 開発者は、FAB、Navigation Drawer、SnackBar、Collapsing Toolbarなどの要素を実装するための新しいコンポーネントを受け取りました。 最初は、多数のバグがありましたが、現時点では、ライブラリはすでに広く使用できるようにファイナライズされています。 サポートライブラリも更新および追加されました。その完全なリストは、 こちらで確認できます 。



Androidマシュマロ



Androidオペレーティングシステムの新しいバージョンは、インターフェイスの設計に新しいアプローチをもたらしただけでなく、アプリケーションの省エネとアクセス権を大幅に修正しました。 開発者には、SDKの新しいバージョン（API 23）が導入されました。これは、革新とともに多くの問題をもたらしました。

  • リアルタイムの許可。 現在、潜在的な「危険な」アクションについて、開発者はアプリケーションの実行中にユーザーに許可を求める必要があります。 詳細はこちらをご覧ください 。 サポートライブラリの最新バージョンは、権限を操作するための非常に便利なインターフェイスを提供しますが、これは、権利の取得に関連する問題のほんの一部を解決しますが、アプリケーションのロジックには解決しません。 たとえば、ユーザーがアクセス許可を付与していない場合の対処方法に関する論争は依然として続いています。
  • DOZEモード。 Googleチームは、以前は多くの批判を引き起こしていたバッテリーパフォーマンスの最適化に向けて非常に重要な一歩を踏み出しました。 ユーザーは、バッテリー消費の履歴を見ずに、OSを直接非難し、 疑わしい作者からの疑わしいアプリケーションがバッテリーを使い果たしていることに気づきませんでした。 これで、ユーザーはこれについて心配する必要がなくなりました。 居眠りモードとアプリスタンバイの2つの新しいモードの登場により、オペレーティングシステムは、過度に大食いするアプリケーションへの酸素を遮断します。 居眠りモードは、Android Marshmallow OSを実行しているすべてのデバイスが一定の非アクティブ時間を経過した後、充電せずに進むモードです。 2番目の新しいモードはアプリスタンバイです。これは、長期間使用されていないすべてのアプリケーションのデバイスリソースへのアクセスの自動奪取です。 開発者はJobShedulerクラスとSyncAdapterクラスの作業の変更を考慮することが重要です。新しいモードに切り替えると、現在のタスクはすべてキャンセルされ、新しいタスクは目覚めるまで延期されます。 詳細はこちらまたはこちら 。 居眠りモードの興味深い移行スケジュール 。
  • アパッチ API 23でのApacheパッケージのアクセス不能のニュースは良いものになりましたが、以前にアプリケーションで使用していた場合は、拒否するか、個別に接続する必要があります。
  • システム自体を使用したシステムセキュリティ。 画面のロックを解除し、サードパーティアプリケーションのセンサーと連携し、スマートフォンのモデルに関係なく、Google Playストア内で購入し、指紋でサイトを認証することは、Androidの世界で現実になりました。 詳細




Google Playサービス



バージョン8.4への新しいサービスの更新により、開発者は以前のモジュールを使用する可能性を拡大しました。

  • Googleマップ -オフラインで作業する機会。 さらに、Wearデバイスでのカードの実装が利用可能になりました。
  • Gcm 。 プッシュ通知のモジュールが再設計されており、以前にWakefulBroadcastReceiverを使用していた場合は、新しいアーキテクチャに切り替えることをお勧めします 。詳細はこちらをご覧ください 。


多くの新しいモジュールもGoogle Play開発者サービスに追加されました。

  • Google Fit API-さまざまなデバイスセンサーを使用してデータを収集するフィットネスアプリケーションの業界があります。 これで、センサーを操作するための新しいモジュールが簡単になりました。
  • Places API -Google Addressesへのアクセスも登場しました（以前はWeb APIを介してのみアクセスできました）。 ただし、GoogleはまだGoogleマップ（Directions API）でのルート案内のサポートを実装していないことに注意してください。
  • Google Drive、Mobile Vision API、Marshmallow Permissions、Application Measurement、Nearby Messagingなど、他の多くのモジュールをここで見つけることができます。




ライブラリとフレームワーク



レトロフィット2

非常に重要なのは、最も一般的なネットワークライブラリであるRetrofitをバージョン2.0に更新したことです。 これまでのところ、このバージョンはベータ版ですが、開発者が積極的に使用することを妨げるものではありません。 主な変更点：リクエストをCallオブジェクトにカプセル化する（リクエストのキャンセルを含む、より柔軟な作業を整理できるようにする）、ライブラリをコンポーネントに分割する、さまざまなコンバーターとアダプターをサポートする、新しいアノテーションの形式の構文シュガー さらに詳しく-Droidcon での Jake Warthonのパフォーマンス 。



Rxjava

今年は、AndroidでのRxJavaフレームワークの人気が飛躍的に高まったことも特徴です。 このライブラリは、RetrofitおよびRealmなどの他のライブラリと密接に連携します。 RxJavaには、クエリの実行に多くの利点があり、結合、再起動などの可能性があるため、ネットワークの操作などに使用すると便利です。 今日、多くのリソースがこのフレームワークに費やされています： ブログ 、 habrなど。 また、多くの開発者は、リアクティブスタイルで標準機能を慎重に書き直します： RxBinding 、 RxPermissions 、 RxLoader 。



ダガー2

また、人気が高まっているのはDagger 2です。ここでは、Dagger 1との主な違いに言及する価値があります。

  • リフレクションの欠如。これにより、ソースコードの可読性が向上し、生産性が向上しました。
  • POJOと連携するためのAPI。
  • コンパイル段階でのグラフの検証。


ドキュメントとサンプルDagger 2 + MVPバンドル。



コトリン

この言語の約束されたリリースはほぼ行われ、現在のバージョンは1.0 Beta 4です。もちろん、コンパイルに関する多くの苦情を含む問題が発生する場合がありますが、JetBrainsはこれに積極的に取り組んでいます。 そして一般的に、Kotlinはすでに戦闘プロジェクトで使用する準備ができています。 詳細はこちらとこちら 。



確立された慣行



前年がインターフェース設計の点で標準化された場合、今年度はアプリケーションの「内部」を標準化する試みでいっぱいになりました。つまり、かなり普遍的な方法でアプリケーションを開発できるようなパターンを提供するためです。 以前は、Androidアプリケーションの一般的なアーキテクチャについて話すことは困難でしたが、今ではほとんどの企業がすでに確立された開発パターンを使用するためのプラクティスとアプローチを熱心に導入しています。



MVP

これは、最も人気があり、最も物議を醸すパターンです。 人気-ビジネスロジックとインターフェイス要素を正常に分離できるため、テスト中に便利になります。 議論の余地-絶対にすべての開発者がこのパターンの実装方法について独自の意見を持っているからです。 ここでその実装のいくつかを見ることができます。 また、MVPとRxJavaの組み合わせについて説明しているハブに関する優れた記事もあります。



MVVM

WPFで積極的に使用されている別のトレンディなパターン。 彼は、Google-Data Bindingの開発のおかげで、Androidの世界で人気を得ました。 このライブラリはまだ十分に安定していないことに注意する必要があります（ベータステータスがあります）。 ただし、Googleは間違いなくそれを思い起こさせるので、 今勉強する価値があります 。



Android Studio 2.0



今年の終わりは、開発者にとってもう1つの重要なイベントでした。11月に、Android Studio 2.0がプレビューチャンネルのすべての人に利用可能になりました。 明らかに、提示された機能により、この更新が最も重要なものになります。

  • インスタント実行。 多くの開発者が長い間待っていたもの。 これで、数行の変更の後、毎回アプリケーションを再度デプロイする必要がなくなりました。 これまでのところ、Gradle 2.0 alphaを使用している場合にのみ、非常に安定して動作しません（これも非常に安定していません）。 Googleが安定するのを待ちます。これは私たち全員が信じられないほど幸せです。
  • 新しいエミュレータ。 すべての開発者はネイティブエミュレータの速度を知っています。Genymotionを好む人が多いのは驚くことではありません。 確かにGoogleはこのような状況を好まなかったので、Android Studioには、カメラ、GPS 、ブラックジャックなど 、必要なすべての機能を備えた優れたエミュレーターがあります。
  • また、Googleチームはプロジェクトのビルド時間に十分な注意を払いました 。 彼らによると、新しいバージョンでは、プロジェクトは2倍の速さで組み立てられます。 当然、これは喜ばしいことです。

  • モバイル決済。 最後に、Googleはスマートフォンからの支払いシステムのバージョンであるAndroid Payを導入しました。 銀行カードはNFCチップを搭載したスマートフォンに取り付けられており、ユーザーはAndroid Payで支払いが可能な場所でデバイスを使用して支払いを行うことができます。
