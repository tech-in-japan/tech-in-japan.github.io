LyncとAvaya IPOの統合



読者の皆さん、ご挨拶！ 今日、MS Lync Server 2010と外部テレフォニーの共有の問題を強調したいと思います。 名前が示すように、Avaya IPOを例として使用します。 実際、特定のPBXモデルは役割を果たしません。 これは、アスタリスクを含め、インターネット上であまり記述されていない統合についてすべてのもので動作します。 一部の機器では、LyncはDirect SIPを使用して動作できます（たとえば、CUCMを使用）。



このような統合が解決できるニーズの最前線については、長い間説明しません。 それらの多くがあり、それらはプロセスの本質に影響を与えません。 私の場合、これはユーザーが社内電話を使用して、外出中に電話会議に参加できるようにするために行われました。 つまり、この特定のアプリケーションでは、Avayaを補完するのはLyncであり、その逆ではありません。 テレフォニーと通信を提供するのはIPOです。



だから処分：

オフィスにはAvaya IPO406v2（番号5xx）があります。 電話は、アナログ（パナソニックおよびほとんどが安価）、デジタル（ブランドデバイスAvaya）、およびあらゆる種類のハードウェアSIP電話（一般的に動物園）で使用されます。 ネットワーク上にはLync 2010 Standart（4xxx番号）があり、主にリモートの従業員がオフィスと通信するために使用しました。 LyncのEDGEサーバーはインターネットを監視しており、リモートの従業員は外部からインターネットを通過します。



一般的なタスクは、LyncとAvayaが相互に音声を送信できるようになります。



さて...セットアップに直接進みましょう。



Lync Serverから始めましょう。


最初のステップは、仲介サーバーをオンにすることです。 サーバーが標準の場合は、フロントエンドサーバーで直接Collocated Mediation Serverをオンにします。 Lyncのバージョンがエンタープライズの場合、メディエーションロールを持つ別のサーバーをインストールする必要があります。 私たちには標準があるので、それを考慮してください。









この段階では、ゲートウェイのリストは空です（事前に追加されていない場合）。

PSTNゲートウェイを追加する必要があります。

ゲートウェイを追加すると、システムはゲートウェイのIPアドレス（この場合はPBX）を要求します。 通信ポートとプロトコル。 プロトコルTCPおよびポート5060を選択します。

ところで、Lyncサーバーは、OCSとは異なり、1つの仲介サーバーを介して同じ方法で複数のゲートウェイと連携できます。



次に、メディエーションを有効にした場所にゲートウェイを追加します。



その後、トポロジを公開し、仲介サーバーサービスを開始します。 すべてのLyncを再起動する方が理想的ですが、これは必要ありません。



次のステップは、PBXを構成することです。


SIP回線を作成します。

この場所では、少し余談する価値があります。

すべてのユーザーはPBXからLyncへの呼び出しを受信できますが、エンタープライズVoIPが有効になっている（このためにエンタープライズCALが必要です）ユーザーのみが "外部"を呼び出すことができます。

さらに、PBXからLyncユーザーに電話できるように、すべてのユーザーに電話番号を提供する必要があります。 私の場合、内部の数字と視覚的に区別するために4桁にしました。

次のダイヤルプラン。 Lyncコントロールパネルの[音声ルーティング]に移動します。

デフォルトでは、グローバルダイヤルプランがあります。 異なるプランを異なるユーザーに配布する必要がない場合は、編集するだけで済みます。 計画自体（および政治家）には3つのレベルがあります。 グローバル、サイトレベル、およびユーザーレベル。 優先度はカバレッジに反比例します。



正規化ルールを作成します。

具体的には、3つのルールが必要です。

  • PBXへの呼び出し用。 ^（5 \ d {2}）$
  • 外部への呼び出し用。 ^（\ d {11}）$
  • 内部コール用。 ^（4 \ d {3}）$

PSTN使用法を作成します。



PSTNの使用法の一部として、ルートを作成します。 つまり、番号<->ゲートウェイのパターンの形式の接続です。 前述したように、ルーティングには複数のゲートウェイを使用できます。





この手順を3つの方向すべてについて繰り返します。



各段階で、導入されたルールの正確性をテストすることができます。 また、永続的なテストを作成し、セットアップの各段階で、現時点でコールがどの段階を通過するかを確認できます。









実際、その後、サーバーの仲介サービスを再起動すると、すべてが機能します。

LyncでエンタープライズVoIPが有効になっているユーザーには、新しいダイヤルモードと自動転送モードが使用されます。 ユーザー設定で内線番号または携帯電話番号が指定されている場合、連絡先リストから直接呼び出すことができます。



電話はまだ簡単です。 Lyncユーザー番号をダイヤルするだけです。 発信者の名前と番号は正常に送信され、キリル文字も完全に理解されます（もちろん、電話自体が理解できる場合）。
