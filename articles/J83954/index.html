構造化情報ストレージ

組織内の個人情報管理（PIM）実践の状況と呼ばれる興味深いビデオに偶然出会い 、 ドキュメントを読みました（PIMの作成者はこの問題に徹底的に取り組んでいます-研究、テスト、理論、この主題に関する候補者や博士研究さえ擁護しているようです）そして、記事に記載されている情報の保存と情報への迅速なアクセスを整理するためのガイドラインは、日常業務で長い間使用されてきました。 そして、あなたが知っているように、彼らは良い人生からではなく、私のところに来ました。

たまたまPMで作業しているときに、プロジェクトの数が5を下回らず、5〜10の範囲で安定して保持されていました。 プロジェクトの大部分は小規模で、長さは3〜5か月、開発者は2〜4人（さらに、ネットワーク管理者、プロジェクトチームのテスター）でした。 しかし、生産のサポートとメンテナンスがこの富に追加され、重要な手紙の数が1日あたり50〜70を超えたとき、プロジェクト情報の効率的な構造化ストレージ、その検索、計画されたイベントのリマインダーについて考える必要がありました。 以下は、何が起こったのか、どのように...



1.すぐに作成される階層ストレージ-情報が利用可能になる前であっても、それを保存する必要があります。 私にとっては、ディスク上では次のようになります。 



 <名\姓\ニック>
  | __ <会社>
      | __ <プロジェクト>
           | __ <アクティブ>
           |  | __ <プロジェクト1>
           |  | __ <ドキュメント>
           |  | __ <仕様>
           |  | __ <その他のプロジェクト固有のフォルダー>
           | __ <完了>
           | __ <キャンセル済み>  

 <ドキュメント>-プロジェクトのドキュメント（プロジェクト計画、プロジェクトスケジュール、要件、HLD、テスト計画、操作マニュアルなど）
 <仕様>-外部ドキュメント（プロジェクト内で作成されていない-仕様など）

プロジェクトが完了するとすぐに、完全に<Completed>または<Cancelled>に転送されました。


同じ構造がMS Outlook 2007で作成され、プロジェクト活動に関するすべての通信が保存されました。

ファイルの命名については、 プロジェクトのディレクトリとファイルを整理する方法からのアイデアを使用することが可能になると思います



2.動的フォルダー。 MS Outlookでは、指定された基準に基づいてメッセージフロー全体をフィルタリングする検索フォルダーを指定できます。

a）この機能を使用する特別なケースは、Follow up folderです。 MS Outlook 2007では、任意の日付のフォローアップを作成し、日付ごとにグループ化することができます。 したがって、今日のタスクを表示して毎日の作業を開始し、入力メッセージを処理し、将来のフォローアップ日付に従ってそれらを整理し、すぐにそれらを忘れます:)そして、通常10-20タスクを含む空白のシートからの新しい日:)

b）メッセージ構造が明確に定義されている場合（たとえば、指定されたテキストPriorityまたはAreaを含む障害\提案\問い合わせメッセージ）、ルールを使用して、カテゴリ（エリア）または色（優先度）の設定を構成できます。次に、検索フォルダーを使用して検索、グループ化、メッセージを並べ替えます。



PS営業日の終わりに未読の手紙を残さないようにしてください。 「あなたが物をコントロールしなければ、 物があなたをコントロールします！ 3番目のオプションはありません！”単語の代わりに置換する単語-文字、プロジェクト、人生:)
