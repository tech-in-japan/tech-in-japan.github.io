JetBrains C ++ IDE：いつ？

エイプリルフールのジョークが成功した後、C ++ IDEをいつリリースするかを定期的に尋ねられます。 Linuxで書く人は誰もがそのような開発環境を長い間待ち望んでいたことは明らかであり、WindowsやMac OSで利用できる環境に代わるものは誰も傷つけないでしょう。



C ++ IDEは別の製品であり、他のIDEと同様にIntelliJプラットフォームに基づいています。 ReSharperでのC ++サポートは完全に異なるトピックであり、個別に（今日ではありませんが）記述します。







プライベートプレビューのリクエストのクラウドが送信されました。 私たちはこれに心からそして非常に心地よく打たれました。 そして、この投稿を書いています。 プライベートプレビューはまだ勇敢な人に発行される準備ができておらず、同僚に触れる準備ができていますが、その内容はどうなるかをお伝えします。 まもなく、3月に一部のユーザーから最初のレビューを受け取る予定です。



最初の成功した開発者はすでに選択されており、その手にはプライベートプレビューが最初に含まれます。 選択の原則は単純です。プライベートプレビューではうまく機能するはずであり、まだ完全には洗練されていません。 フォームに記入する際、テストにサインアップした人は、自分の興味とプロジェクトの性質を示しました。 IDEで既に機能している機能と関心が一致するものを選択しました。 すぐに彼らは待望のダウンロードリンクを受け取り、彼らにどれほど幸運が微笑んだかを確認します。



ところで、 プライベートプレビューにサインアップするのに遅すぎることはありませんが、 サインアップしたすべての人が最初のプライベートプレビューを受け取ることを保証するものではありません。 誰かがこのビルドをバイパスしても私を責めないでください。



次のプライベートビルドで最も興味深いもののリストを次に示します。



  • CMake + GCC + GDB7。
  • 複数のプラットフォームのサポート：Linux、OS X、Windows（Cygwin、MinGW）。
  • C ++ 11のサポート（ラムダ、文字列リテラル、可変数のパラメーターを持つテンプレート（可変アドインテンプレート）、decltype、autoなどを含む）
  • コード分​​析と多くの多くのクイックフィックス。
  • リファクタリング（抽出メソッド/変数/ typedef /定義/など、署名の変更、名前の変更、安全な削除など）。
  • コード生成とテンプレート。
  • プリプロセッサのサポート（つまり、使い慣れた便利な機能はすべてマクロでも機能します）。




Clang / LLDBサポートは、後続のEAPビルドで表示されます。



そして今、プライベートプレビューを待たずにこの美しさをすべて見たい人は、C ++ IDEがどのように動くかを示すスクリーンキャストを注意深く調べることができます（ちなみに、アニバーサリースクリーンキャストはYouTubeで400番目のJetBrainsTVビデオです）。
