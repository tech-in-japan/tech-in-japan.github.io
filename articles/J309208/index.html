用語を更新しましょう



FMC（Fixed Mobile Convergence、Converged Communications）は、モバイル/モバイル/ GSMテレフォニーと企業の電話交換、単一の番号計画を持つ社内ネットワーク、およびオフィスと携帯電話の共通の通話管理ルールとの統合です。 一般に、ネットワークのコンバージェンスについて言えば、実際には、モバイル通信（セルラーでもあります）、公衆電話ネットワーク（固定電話と呼ばれるもの）、IPテレフォニーなどのさまざまなネットワークのコンバージェンスです。 これは、インフラストラクチャのさまざまな要素を使用して、テクノロジーの交差点でサービスが作成されることを意味します。



残りの用語は、原則として、ほとんどすべての人にすでに知られていますが、念のため、次のことを思い出してください。



  • モバイル通信（セルラー、GSM）-受信がある地域の携帯電話から利用可能なモバイル通信（カバレッジ）。

  • 固定接続-固定端末からアクセスできる有線接続とIPテレフォニーを介した、電話ユーザー間の電話接続。

  • 端末-ユーザーの端末機器（電話、スマートフォン、コンピューターなど）、通信サービスへのアクセスが可能なデバイス（この場合）。

  • 内線番号-外部呼び出しを行うときと呼び出しを転送するためにアクセスするためにユーザーに割り当てられる短い番号（多くの場合、内部呼び出しと一致します）、特定のマネージャーの受話器に乗ることを望む組織番号またはIVRメッセージの直後に発信者によってダイヤルされます;

  • 内部番号-オフィス内の呼び出しのためにユーザーに割り当てられた短い番号（たとえば、会計士は007をダイヤルしてディレクターを呼び出し、ディレクターはシステム管理者を呼び出します666）。



FMCは何を提供しますか？ これについては、以下で詳しく説明しますが、今のところ、誰もが遭遇したいくつかの例があります。



  • 企業ネットワークを介して、内線番号と長い番号の両方を呼び出すことができます。 同時に、すべてのワークコールをプラットフォーム経由でルーティングし、記録および監視できます。

  • オフィス内線と同時にモバイル内線リングを使用できます。 そして、5、8、30、n秒の遅れがある可能性があります。 いいえ、nは多すぎますが、kを取ります。

  • 曜日ごとにコールの受信を設定し、時間、従業員の雇用、優先度のチェックを行うことができます。 あらゆる種類のアルゴリズムが可能です。



簡単に要約すると、特に私たちの場合、これは従業員の携帯電話を企業の電話に組み込むこと、つまり、すべての勤務時間（および非勤務時間）とすべての公式交渉の範囲です。

FMCのロジック



Mobile Employeesサービスが接続されたときに正確に何が起こるかを図で調べてみましょう。







写真から、FMCの概念を簡単に理解できます。



  • クライアントネットワーク-F、固定（オフィス内の固定電話の場合）。

  • Beeline Network-M、モバイル、モバイルオペレーターのネットワーク。

  • F + Mを接続するTelfinジョイントはC、収束です。



PSTN-公衆交換電話網。会社がやり取りする外部番号が含まれます。 現在、モバイルからパブリックネットワークに電話をかける場合、電話はオフィスPBXを経由します。 外部の当事者は会社番号を見ました。 会話が録音されました。 コールデータは、PBXおよびその他の分析システムによって受信されました。



同様のスキームが反対方向に機能し、矢印のみが反対方向に変更されます。 モバイル従業員に到着した外部通話は、発信者の正しいデータを会社のPBXに転送します。



オフィスに固定/固定電話接続がない場合、スキームは変更されますか？ 変わりません。 クライアントのPBXは、Telfin仮想PBX、従来のPBX、アスタリスクなどの一部である場合があります。 本質は同じです：







クライアントが仮想交換と鉄交換を組み合わせたハイブリッドソリューションを使用している場合、FMCスキームも実行可能です。







まとめ 企業の通信ネットワークがどのようなものであっても、設定後、モバイルネットワークからの通話は企業ネットワークを通過できます。 現在、すべての電話は、携帯電話も含め、企業の音声ネットワークの一部となっています。 携帯電話は内線番号を受け取りました。



FMC-SIMから電話を受ける方法



まず、オフィスの電話に関連付けられた内線番号を使用して電話を受ける方法を思い出します。 会社の固定電話番号への着信は、音声メニューに分類されます。 そして、余分なもの（たとえば、200）を追加すると、この呼び出しは「テーブルの上」にいる人に届きます。 デスクの電話が鳴り、人が電話を取り、会話を始めます。 しかし、人が昼食をとったり、出張に行くと、デスクの電話が鳴ります。 すでにこの圧倒的な刺激を感じましたか？



昼食、出張、そして厳しいビジネスの世界でのしばしば寒い病気の休暇は、電話に出ない言い訳になるのをやめたので、やがて彼らは電話転送を接続し始めました。 現在、オフィスの電話が鳴ると、2、3秒後に携帯電話も鳴ります。 人は自分の携帯電話で何を見ますか？ 着信の転送元の会社番号。 その結果、不在着信の場合、誰が電話したかを理解することは困難です。 さらに、コール転送は、モバイルへの発信コールとしてPBXに記録されます（ただし、これは最初は着信コールであり、クライアントとの関係の履歴にも含まれる必要があります）。 結論：混乱、誰が、いつ電話されたのか、電話交換で電話を見つけるのが難しく、CRMの会話にマークがなく、電話の録音がありません。



携帯電話からの通話に関するデータを保存するもう1つの方法は、携帯電話に特別なアプリケーション（ソフトフォン、ソフトフォン）をインストールすることです。 このソフトフォンを電話交換で別の内線番号としてマークします。 それが良くなります：そして、通話が記録され、発信者の初期番号が示されます。 ただし、ソフトフォンはインターネット経由でのみ機能します。接続速度が低下した場合、または人が3G、3G +、4Gネットワ​​ークの外にいる場合、少なくとも音声伝送の品質のため、会話は行われません。



Mobile Employeesサービスを使用するとどうなりますか？ 特別なSIMカードを挿入し、サービスを接続します。 また、以前に携帯電話番号しかなかった場合、今では本格的な内線番号になり、さらに覚えやすいです。 以前は、番号は「911-xxx-xx-xx」でしたが、現在は「200」になり、PBXで個別に割り当てられています。 これらの3つの番号は、固定電話、ソフトフォンにあり、現在は携帯電話にあります。 したがって、モバイルは企業ネットワークの一部になります。 これで、元の加入者番号である通話者を確認できます。 この電話に出られなかった場合は、慣れてきたときに電話をかけ直します。見逃した番号を1回クリックします。 同僚と情報を明確にしたい場合は、モバイルから他の同僚に通話を転送します。 私は電話会議に参加し、会話を記録し、通話の統計情報を保持します。これはすべてモバイルからです。 ほぼ幸せと相対的な調和。



モバイルから追加ダイヤルする方法



携帯電話から3桁をダイヤルすると、PBXは理解し、対応する内線番号を持つ同僚に転送します。 同僚は、抽象的な携帯電話を呼び出しているのではなく、内線から電話をかけていることに気付きます。 はい、ここではモバイルリソースが使用されますが、これはまだ内部PBXコールです（ここでは、手頃な価格でシンプルなコンバージェンス）。 従来、これらはオフィス内の2台の電話であるため、それらに割り当てる番号を割り当てて、それらが決定されるようにしました。 これは内部接続であり、他の電話ネットワークには一切適用されません。私は自分の欲しいものを2箱渡しません。



仕事や個人的な問題の発信



外部（オフィスに関連）、携帯電話、または固定電話の電話番号に発信する場合は、ルールを適用します。 ユーザーの利便性のために、次の規則が適用されます。 アスタリスク（*）を介して電話をかける場合は、職場で電話をかけ、プラス（+）を介して電話をかける場合は、個人的な問題について電話をかけます。 それぞれの場合に何が起こるか見てみましょう。



たとえば、私は顧客に電話しています。 携帯電話のアスタリスクから番号をダイヤルしています：



  1. 電話でモバイルクライアントにダイヤルします：* 8-921-xxx-xx-x

  2. FMC SIMカードを使用した内部ネットワークで、PBXに通話が着信しました。

  3. PBXはコールをPSTNネットワークにルーティングします。

  4. クライアントはオフィス番号/ PBX番号を見ました。



個人的な質問のために携帯電話から電話をかける場合、会社が個人的な会話を記録しないようにします（必要に応じて、聞き取りません）。 プラス記号で始まるアスタリスクなしの通常の番号をダイヤルします。



  1. 友人の番号をダイヤルしています：+ 7-921-xxx-xx-xx。

  2. この例では、コールはBeelineネットワークを介してMegaphoneネットワークに送られます。

  3. 友人は私の通常の携帯電話を見ています。



この呼び出しの分離は偶然ではありません。 外部のパーソナルコールはPBXを経由せず、記録されません。 会社はサードパーティの会話を保存せず、従業員には迫害マニアはいません。 誰のネットワークで通話が行われたかは論理的であるため、課金されます。 +7からの通話は個人通話です。 個人通話の場合、SIMカードの所有者は、モバイルオペレーターのモバイルネットワークを通過するときに支払います（コストは、モバイルオペレーターが選択したパッケージによって異なります）。 アスタリスク（*）で始まるもの-モバイル従業員サービスの一部として行われ、Telfinサービス料金に従って請求されます。



接続方法

当社と携帯電話会社との間で契約が締結され、SIMカードが接続され、ルーティングルールと企業プログラムとの統合が設定されます。 既存のSIMカードでは、セットアップに1日かかります。



クライアントがモバイル従業員を個別に構成する場合、Telfin.Office PBXのインターフェースにモバイル拡張機能があります。







サービスを有効にするには：



1. PBXの「設定」でアイテム「従業員とキュー」を開きます。

2.モバイル専用の拡張機能の設定に移動します。

3.内線番号「モバイル従業員」を選択し、モバイル番号を指定します。

4. [保存]をクリックします。



ユーザーが成功しなかった場合や質問がある場合は、いつでもお手伝いします。 まず、クライアントのセットアップ時間を節約します。 第二に、FMCの実装の利便性を評価し、市場で機能を変更する必要がある場合に調整を行います。



ロシアの厳しい事件



あるクライアントがそのようなリクエストを思いつきました。 彼が働いている地域では、インターネットに固定電話とモバイルの両方で大きな問題があります。 さらに、同社のロイヤルティプログラムの従業員にはスマートフォンが提供されており、個人的なコミュニケーションと仕事上のコミュニケーションの両方の主要な電話として喜んで使用できます。 従業員の半数以上が常に顧客やディーラーに出向いており、顧客からの電話に応答する必要がありますが、固定接続は役に立ちません。 さらに、GSMのソリューションが必要でした。GSMは、サーバー側のCRMと友達になるはずでした。



彼らは従業員を企業の関税でFMC SIMカードに移し、Mobile Employeesサービスを有効にし、その効果を待ち始めました。



  • コミュニケーションのための会社と従業員のコストが減少し、通話コストが明確に分離されました。

  • 呼び出しが便利になり、すべての番号が決定され、呼び出しに関する情報が書き込まれ、データがCRMに送られます。

  • 不在着信と未応答の通話数が削減されました。 テレフォニー全体がより透明になりました。

  • 高速通信によるトランザクション時間の短縮。

  • クライアントとの仕事の質が向上しました-マネージャーは、「品質管理の目的で、すべての通話が記録されます」と通知されました。

  • 録音された電話は、既存のマネージャーや新人にとって優れたトレーニングベースになりました。トレーニングは理論ではなく、実際の交渉で行われます。



徐々に、クライアントはオフィスのコミュニケーションスキームを複雑にしました。たとえば、2人の携帯電話を持ち、一方を朝、もう一方を電話にかけ、夜に別のタイムゾーンのクライアントにボイスメールを電子メールで送信することができます。 しかし、会計と分析に困難は生じませんでした。 まあ、そしてもちろん、クライアントを驚かせた主なものですが、私たちはそれに慣れています：オフィステレフォニー（PBX、通話録音、分割請求書を使用）は、この条件がない場合に正しく機能しました。



楽しいボーナス：注文番号による電話



モバイルは現在、他の機器と同様に、企業ネットワークの本格的な一部です。 そのため、モバイルで新しい機能を利用できます。テレフォニーとCRMを組み合わせます。



例。 顧客は宅配便を待っています。 顧客から会社に電話がかかってくると、着信番号がチェックされます。 この番号がクライアントベースで見つかった場合、指定されたアルゴリズムに従ったコールがすぐに宅配便業者に送信されます。 宅配便は遅れていますが、連絡があり、クライアントと話す準備ができています。



別の興味深いオプションは、CRMから注文番号への直接呼び出しです。 人生では、このように見えます。 従業員には注文番号が手元にあります（ピザの配達、タクシーの呼び出し、郵便の場合）。 彼は携帯電話で注文番号を直接ダイヤルし、「呼び出し」を押します。 PBXはCRMで注文番号と顧客の電話番号を見つけ、従業員を顧客に接続します。







Telfin.Office PBXには、IT製品との既製の統合が15個あります（さらに、サードパーティ開発者による未確認の自己記述型統合が多数あります）。 携帯電話からPBXへの通話のデータを保存し、CRMで顧客のカードにモバイル通話を添付し、従業員からの断片的なデータではなく、顧客との通信の全履歴を確認します。 今では可能になっています。



スキームは注文番号だけでなくCRMでも機能します。



  1. アスタリスクの後にいくつかの番号をダイヤルし、呼び出しを押します。

  2. 携帯電話から当社のTelfinネットワークに接続するリクエストがあります。

  3. ダイヤル規則を知っているPBXは、これが電話番号ではなく、ある種のコード（注文番号、顧客コード、サポートコール、猫の餌付け時間）であることを理解しています。

  4. さらに、PBXはアクションへの信号を送信します：呼び出し、ステータスのマーク、データの送信。



追加されるかもしれないものの原理が変わり、顧客体験が変わったので、今では新しい機能が携帯電話で獲得されています。 しかし、以前は、デスクの電話で注文番号をダイヤルするために変更しましたが、ここでは何も変更されていません。 しかし、携帯電話では、このようなサービスは旅行マネージャー、宅配便業者、タクシー運転手にとって興味深いので、これはより必要です。 私たちは、携帯電話を使用する人間の経験を活用し、既存の電話交換とコンピューター電話の機能にそれを適用しようとします。



そのようなサービスが会社のすべての問題を解決し、売上が増加し、コストが減少し、髪が光沢があり、絹のようになると言うのは魅力的です。 確かな経済的効果は確かに続きますが、現実的です。 このサービスは、まず、企業通信の透明性、利便性、シンプルな展開、およびインターネット接続の速度への独立性を提供します。 さらに、これは従業員がオフィスの外にいる人や遠隔地にいる人にとって価値のあるオファーです。 そして、透明性、コントロール、アクセシビリティは今日の重要なビジネス価値です。 競合他社は眠っていません。あなたは、何も答えられなかったという理由だけで、あなたは戦いなしにクライアントに与えるべきではありません。
