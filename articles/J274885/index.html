データベース



dcoinでは、sqlite、postgres、mysqlをサポートしました。 通常のデスクトップアプリケーションでは、デフォルトでsqliteが選択されます。 余分なものを追加する必要はありません。 dcoinプールを上げる人は、postgresとmysqlのどちらかを選ぶ方が良いでしょう。

golangでSQLデータベースを操作するには、パッケージデータベース/ sqlが責任を負います。 sqlite 、 postgresql 、 mysqlデータベースドライバーを接続する必要があります。



最初はsqliteのみを使用してから、postgresqlとmysqlを接続することにしました。 それほど難しくないことが判明しました。 sql.Open（）でデータベースに接続するためのパラメーターのみを変更し、SQLクエリの構文の違いを考慮する必要がありました。

これがデータベースへの接続の実装です。

データベースの操作にラッパーを使用しなかったのは、 古いバージョンのDcoinからベアsqlクエリを転送する方が便利でした。同じ理由で、データベースから受け取ったデータ型の変換を伴うコードの構成がかなり貧弱です。



私が踏んだ熊手：

1. db.Query（ "sql"）の後にrows.Close（）を呼び出すことを忘れた場合、閉じられていない接続がたくさんあります。

2.エラー処理の前にdeferを呼び出すことはできません。 エラーがある場合、rowはnilになり、rows.Close（）を呼び出すとパニックが発生します。 これはできません： 



rows, err := db.Query("SELECT * FROM table") defer rows.Close() if err != nil { return err }


それは本当です： 

 rows, err := db.Query("SELECT * FROM table") if err != nil { return err } defer rows.Close()

解析ブロック



dcparserパッケージは、ブロックを直接解析し、テーブルにデータを入力します

たとえば、新しいユーザーキー、基本的には新しいDcoinユーザー-new_user.goを登録するトランザクションを考えます。

NewUserInitは、このトランザクションのデータを含む変数を設定します。 それらは2つだけです-public_keyとsign

NewUserFrontは、データのチェックを担当します。 特に、このトランザクションを生成したユーザーがマイナーであるかどうかを確認します。 制限を超えているかどうかを確認します。

NewUserはデータベースにデータを書き込みます

NewUserRollbackFrontは、NewUserFrontメソッドによって行われた変更をロールバックします

NewUserRollbackは、NewUserメソッドによって入力されたデータをロールバックします

70種類のトランザクションのそれぞれに対して、同様の4つの方法があります。

270,000ブロックがどのように解析されるかをちょっと想像してみてください。 70個のメソッドが180個のテーブルと1100個の異なる列にデータを入力します。 そして、私を最も魅了しているのは、RollbackとRollbackFrontを介して、逆の順序で1つのブロックのすべてのデータをロールバックすることです。 このような完全なロールバックは、dcparserの正しい動作をテストするために行われます。 どこかでデータが誤って入力された場合、これはすぐに表示されます。
