未来のWeb 2.0

私は先週シンガポールでWeb 2.0に関するプレゼンテーションを行いました。また、地方政府機関であるInfocomm Development Agencyがシンガポールの新興企業を開発するための戦略を立てるのを手伝いました。 Web 2.0の将来についてよく聞かれましたが、それはバブルなのでしょうか？ 幸いなことに、最近O'Reilly Radarでこのトピックに非常にうまく取り組んでいるので、彼らの前で未来の写真をはっきりと提示できます。 彼女はこんな感じで......



2004：Web 2.0および関連用語の出現。



2006：Time誌によると、Web 2.0に敬意を表して「You」（通常のユーザー）の名前は「Man of the Year」になりました。



2007：私たちは今ここにいます。



2008：Firefox 3.14159がリリースされました（Mozillaのこれらのgezは単純に修正不能であるため、それらの間で真の戦争が始まり、開発者の辞任が続きます-すべては新しいブラウザーをPiFoxと呼ぶことができるかどうかについての議論のためです;しかし、最終的にはPyreFoxで停止します）。 このバージョンでは、 Ajaxアプリケーションのオフラインサポートが表示されます。 人々はこれを「Web 3.0」と呼びたいと思いますが、この用語はセマンティックWebに事前に与えられているため、ブロゴスフィアはWeb 2.86と呼ぶことにしますが、その点はまもなく失われます（純粋主義者の不満のため）、メディアは用語286



2009：セマンティックWeb研究者は、Web上でエンコードされた数学的知識を使用して任意の問題を自動的に解決する演ductive的計算機を開発します。 生徒は喜んで新しいツールを使用して宿題をしますが、電卓では問題の条件をTeXマークアップで表現する必要があります。 新しい形式の唯一のドキュメントは、ロシアの大学のあいまいなドキュメントであり、リーマン空間での情報理論の結果の幾何学的表現に特化しています。 Web 3.0がまだ到着していないことが明らかになります。



2009：ウィジェットの魅力により、Firefox 4はオペレーティングシステムに統合され、ウィジェット用の新しいクロスプラットフォームプラットフォームになります。 まだ頑固に研究を続けているセマンティックWebの開発者への敬意から、「Web 3.0」という用語を残すことを決定します。 ブロガーはこの番号をスキップして、 Web 3.1で停止します 。



2010：セマンティックWeb開発者が新しいXML形式をリリースし、Web 3.0への最後のステップが宣言されました。



2010：Ajax、Flexなどに基づいて構築された洗練されたユーザーインターフェイスの人気が急速に広まったため、Web開発者は標準化の必要性について騒々しくなります。 W3Cコンソーシアムはこの問題を解決することはできませんが、コンセンサスの結果、Firefox 5で実装される最も有名なAjaxセットと、IE 12で数か月後に選択されるAPIとウィジェットが選択されます。インターネットオペレーティングシステム「ティム・オライリー、彼はこれに最適な名前がWeb 95であることを皆に納得させることができるでしょう。



2011：セマンティックネットワークの研究者は、RDFオブジェクトのオンラインデータベースを補充するための子供向けゲームを作成します。 このシステムはWeb 3.0の鍵であると宣言されていましたが、その結果、世界最大のポケモンデータベースを除いては何も起こりません。



2012：Mozillaの隠れたプロジェクトの立ち上げ：汎用性のあるシンクライアント、Webサーバーの立ち上げのみがタスクである、完全にダウンロード可能なLinuxベースのプラットフォーム。 クライアントがGNOMEとKDEのどちらで動作するかをめぐる開発者の戦争は、2つのバージョンのリリースで終了し、最初はプログラムの人気が損なわれます。 ただし、まもなく2つのプロジェクトは団結して、Windowsに対する優位性を実証します。 新聞に「2011年がLinuxデスクトップの年になりました」という見出しが掲載され、Firefox 6に組み込まれたGNODE環境（これが統合グラフィックスサブシステムの名前になります）が世界を征服します。 Web 98はヘッダーに表示されます。



2013：大きな革新なしに長い時間が経ちます。 この期間中、Firefox 6は市場でほぼ完全な支配権を獲得し、そのために特別に作成された強力な悪意のあるソフトウェアの波が殺到します。 Mozillaの開発チームは、穴を埋めて新機能を実装することで引き裂かれ、Firefox 7での作業を予定通りに完了しようとして失敗します。 結果として生じるWebの断片化（IE 13が市場から姿を消し、2012年にMicrosoftがサービス会社に変わって以来、誰もがすでに忘れていた）が混乱につながります。 Mozillaは「No Trouble」と呼ばれる移植性ツールを約束しますが、決してリリースしません。彼らの名誉を称えて、この時代はWeb NTとして歴史に残るでしょう。



2013：セマンティックWeb研究者は、Java 6 Enterprise Editionの新しいRDFデータベース（「レイジングマーモット」）をリリースします。 Web 3.0の時代の到来を記念して、パーティーが開催されます。



2015：MozillaはOperaシリーズを廃止し、特に気に入っているいくつかの残りの機能をFirefoxに統合します。 ユーザーの忠誠心を取り戻すために、Mozillaはデザイナーとインターフェースの専門家に多額のお金を費やします。 これらの問題に対するこのような細心の注意は、 Web XPの出現につながります 。



2020：マルウェアを長年にわたって開発し、戦い続けてきたMozillaは、Firefox 7の機能セットを大幅にダウングレードすることを決定します。 彼らは7番目のバージョンを見逃し、2秒ごとに友人情報を更新するブログ、IM、Twitter、新しいクラックメッセージングシステムのRSSサポートを備えたFirefox Xをリリースし、作業中に黄色のフォントサイズで64ポイント点滅しますバックグラウンド。 クラックシステムは非常に中毒性が高いため、Firefox Xの売り上げが急上昇します（Mozilla Corporationは、Firefox 7のリリースを目指して、長年にわたってほぼ継続するため、Firefox Xに課金することを余儀なくされます）。 Mozillaは、Firefox XとIPOの売り上げから莫大な収入を得て、第三世界の国々での病気、飢hung、軽薄のcombat延と戦うための慈善基金を開設します。 6か月後、クラックユーザー（スクリーンから身を引き裂くことができず、疲れ果てていなかった）の犠牲者の数は、世間の注目を集め、株式市場に影響を与えますが、ジンはすでにボトルから解放されています。 死体が路上に集まるにつれて、世界の知的労働者はますます少なくなります。 人類は狩りと集まりに戻ります。 文明は崩壊し、司祭たちはオフィスや都市から解放された新しい時代の到来を宣言し、人類が丘の緑のフィールドと雲で満たされた空の下に住む世界の到来について話します。 この世界のビジョンに敬意を表して、新世紀はWeb Vistaと呼ばれます 。



2022：最新のセマンティックネットワーク研究者が、RDFで定式化された数独パズルを解くことができるプログラムの作成を発表します。 そして、地球上の最後のラップトップ（2008年の軍事型Pentium）の故障のみがWeb 3.0の発症を防ぎます。
