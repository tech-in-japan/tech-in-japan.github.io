「オープン」ディストリビューション、オープンソース、およびエラスティックについて

Open Distroに関するイベントのコンテキストでは、X-Packソースコードの発見と 記事「The Cloud and Open Source Powder Keg」は、Shey Banon（Elasticの創設者兼CEO）による投稿の翻訳です。







Elasticでは、優れた製品の作成、それらの周りのコミュニティの構築、および製品のユーザーの成功の確保に重点を置いています。



2009年、私は座ってElasticsearchの最初の数行を書き、 そのソースを開きました 。 私は仕事を辞め、製品の作成とそれを中心に構築された素晴らしいコミュニティの支援に2年間投資しました。 2012年にElasticを設立しました 。 ユーザーコミュニティに投資し、その周辺で開発されたオープンソース製品のエコシステムを構築しました。 Apache Luceneに非常に多くの機能を追加したため、スコアを追跡できなくなり、ソリューションの強固な基盤が作成されました。 Rashidが作成したKibana、Jordanが 作成したLogstash 、およびMonicaとTudorが作成したPacketBeatなどを追加しました 。 私たちは製品とその周辺のコミュニティを作成し、ユーザーが最大限の利益を得られるようにすることに焦点を合わせました。 今日、私たちには毎日この膨大なコミットメントに取り組む数百人のElastic開発者がいます。 毎日数十万人のコミュニティメンバーが共通の成功に貢献しています。 そして、私はこれに貢献するために作成した会社を誇りに思っています（プロセス- 約 ）。



私たちは私たちのユーザーに信頼できるレベルを構築しました。 私たちはオープンであるという事実から始め、私たちの活動においてコミュニティとユーザーに正直であり続ける必要があります。 また、これから私たちをそらすものは何もありませんでした。



私たちが始めて以来、私たちは長年FUDを扱ってきました。 成功したものを作成する場合、それは間違いなく起こります。 FUDは主に、そのような動きが何をするのかを恐れる（より）大企業から来ています。 これは自然なプロセスです。 「製品を使用しないでください。これはおもちゃです。」 「彼には少数の開発者しかいません。彼らがバスにぶつかったらどうなりますか？」「彼らは企業が何を望んでいるのかわかりません。」 「それらは本当のXまたはYまたはZではありません（今日の言葉を入力してください）。」 これが私たちに影響を与えたり、考えを占拠したりすることを決して許しませんでした。 これは、私たちと私たちのコミュニティを主な目標からそらすために行われます：ユーザーが大好きな素晴らしい製品とコミュニティを作成することです。 これを実現する場合、ユーザーを失望させ、決して失望させません。



私たちの製品は何度も分岐、再配布、人員不足になっているため、数え切れません。 これは、当社製品の成功と規模の証です。 さまざまなベンダーから、現在はAmazonを含む中国の大企業まで。 常に誤った利他主義や善意によって隠された「理由」が常にありました。 それらのどれも（「理由」を意味する- 約Per。 ）持続しませんでした。 彼ら（Elastic製品に基づくソリューション- 約Per。 ）は、独自のニーズを満たすために作成され、混乱を招き、コミュニティを破壊しました。 ユーザーが好む優れた製品とコミュニティを作成することへのコミットメントとフォーカスが私たちの道を形作り、それがユーザーであるあなたに共鳴しました。 私たちはあなたとの信頼レベル（コミュニケーション）に到達しました。期待される革新と驚くべき協力のレベルです。



オープンソースとそれがもたらす力を信じています。 また、最初から、一部の機能が商用になるとその理由を述べました。 私たちの誠実さが私たちの共通の成功の主な理由の一つだと思います。 接続と実装が簡単になるように、オープンソースコードを作成しました。 当初から、私たちはパスを変更せず、長年にわたってユーザーの信頼を強化し、私たちの言葉とユーザー（ユーザー- 約Per。 ）に忠実であることを強化しました。



私たちの商用コードは、他の人にとっては「インスピレーションの源」であり、さまざまな企業によって盲目的にコピーされ、特定の配布オプションまたはフォークで返されました。 ユーザーが好む優れた製品とコミュニティの作成に引き続き注力しました。 私たちはこれが私たちの注意をそらすことを許しませんでした。



私たちのブランドは何度も使用され、in辱され、情報を盗み、歪められました。 企業は、当社、特にAmazonと協力して働いていると誤って主張しています 。 これにより気が散ることはありませんでした。ユーザーが好む素晴らしい製品とコミュニティを作成し続けました。 焦点を弱めることは会社の敵であり、これが私たちに影響を及ぼすことは決して許しません。 重要なのはユーザーだけで、周りの騒音は重要ではありません。



企業と力を合わせて、コードを公開しました。 ユーザーがAPMに私たちを使用しているのを見たとき、私たちはみんな興奮していました。 APM分野のOpBeatと呼ばれる完全にSaaSの会社と協力しました。OpBeatは私たちにとって大きな商業投資であり、このコードのほとんどはパブリックドメインに投稿されました。これらはすべて無料です。 私たちはユーザーが好む素晴らしい製品とコミュニティの作成に焦点を合わせているため、それは簡単な決定でした。



他の人が閉じたとき、 私たちは開きました 。 同じライセンスの下でオープンソースコードを維持し、企業としてのオープン性を2倍にしました。 より厳格なライセンスの下で既存の商用コードを再ライセンスし、コードを開きました。 私たちは、オープンソースコードと同じレベルのコラボレーションと透明性を、私たちが行うすべてのもので作成しようとしています。 これはユーザーとの多くの議論に対する反発であり、それがあなたに多大な共鳴をもたらしたことをとても嬉しく思います。 それ以来、オープンソースへの投資レベルは増加しているだけであり、明確に定義され、理解できるように共有されているより自由な機能と経験へのコミットメントも増加しています。



私たちの成功を見て、企業が私たちのところに来て、コードで一緒に作業するための特別な作業関係を確立するように頼み、ユーザーよりも高い優先順位を要求したとき、私たちはノーと言いました。 これは何年にもわたって何度も起こりましたが、最近になってようやく、今回はAmazonで行われました。 いくつかは私たちとコミュニティのために参加し 、素晴らしいパートナーになりました。 残念ながら、他の人はこの道をたどりませんでした。 私たちは、製品の開発に貢献する1人の開発者だけでなく、他のすべての人にも関係するというコミットメントを持っています。 プリファレンスはありません。利用可能性のリクエストは拒否します。 私たちの答えは常に一定です。他の人と同じようにプルリクエストを送信します。 品質はそれ自体を物語っています。



これはいくつかの理由で書いています。 まず、私たちはコースに忠実であることを確認するために、何を、なぜ行ったのか、何が成功したのかを考える必要があります。 あなた、私たちのユーザー、私たちのコミュニティ、私たちの会社。 第二に、さまざまな理由で気を散らすことができる他のすべての人にとって、集中力を維持し、正直であることだけが重要なことです。 最後に、ユーザーが楽しめる素晴らしい製品とコミュニティを作り続けたいという私たちの共通の願いを表明すること これが私たちの本当のコースです。



エラスティックは常にゼロです（開発者のように、最初からナンバリングを使用します）。 10年の旅で私たちがすべての皆さん、ユーザー、そして何年も前から歩いてきた最初のコード行から何も変わっていません。 みんな♡
