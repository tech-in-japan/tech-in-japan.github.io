技術的な機能と目的に関するグループがいくつかあります。 サーバーハードウェア用の追加のソフトウェアには、次の2種類があります。
  • Quanta System Manager（QSM）は、直感的なグラフィカルインターフェイスを備えたデータセンター管理システムです。 基本的に、クアンタの鉄管理システム。
  • Quanta Datacenter Managerは、追加機能を備えたQuanta System Managerです。 鉄を制御できるだけでなく、さまざまなセンサーから情報を受け取り、エアコンなどの外部システムの動作を変更できます。 このため、電気のコストを最適化し、冷却などで正しく動作するようになっています。




QuantaGridとQuantaPlexの販売-新しい最新モデル。 ストラトス-古い鉄、ロシアで購入される可能性は低い。

標準ラックサーバー



はい、ラインナップにはそのようなものがあります。 これらは、クライアントのニーズの90％をカバーする鉄のオプションであり、標準的な主力製品です。 サーバーは、1U / 2U / 4Uフォームファクター、ラックマウントバージョンで製造されています。 Intel x86上に構築されています。 メザニンカードやRAIDコントローラーなどのさまざまなコンポーネント、さまざまなディスク構成オプション（3.5または2.5インチドライブ）。 一部のモデルでは、ホットスワップ可能なNVMe PCIe SSDカードをインストールできます。 NVMeは、SSDストレージメディアで動作するように設計された特別なプロトコルであり、多数の最適化と改善が含まれています。 新しいコマンドセットと、最新のマルチコアプロセッサで動作するように最適化されたキューイングメカニズムを使用することにより、SSDのパフォーマンスが向上し、応答時間が最小になります。



多くのサーバーには、フロントパネルドライブに加えて、SSD用に2〜4個の追加スロットがあります。 ディスクはRAIDで組み合わせることができます。 OSやデータのキャッシュに使用すると便利です。 ディスクは非常に薄く（2.5インチドライブよりも2倍薄い）、多くの場合、サーバーの背面に配置されます。 次のようになります。







GPGPUサーバー



これらは同じラックマウントサーバーです。 主な違いは、1つ以上のGPUアクセラレータGPU / Xeon Phiの存在です。 GPUの主な機能はHPCコンピューティングです。 CPUには順次データ処理用に最適化されたいくつかのコアが含まれていますが、GPUは複数のタスクを同時に処理するように設計された数千の小型でエネルギー効率の高いコアで構成されています。 これにはCPUよりも多くのメモリが含まれていますが、それほど高速ではなく、膨大な数の実行ユニットが含まれています。 実際、GPUの作業は、膨大な数の独立したタスクを操作することになります。 したがって、このシリーズのサーバーは次のようなタスクに最適です。

  • HPC（High Performance Computing）/テクニカルコンピューティング（さまざまな数学的並列/分散コンピューティング、もちろん、アプリケーションコードはGPGPUでのデータの計算を考慮して記述する必要があります）;
  • 3Dレンダリング。




マイクロサーバー



このソリューションには、比較的少数のユニットに多数の計算ノードが含まれています。 同時に、ラック内のユニットあたりのCPUおよびメモリの高密度化が達成されます。 3つのユニットに最大24のノードを含めることができ、各ノードにはOS用の2つまたは4つのディスク（サーバーモデルに応じて）を含めることができます。 ディスクはRAIDに結合できます。 次のようになります。

マルチサイトサーバー（ボックス内のクラスター）



マルチサイトサーバーは、1つの2Uエンクロージャ内の2つまたは4つのサーバーです。 高密度の単一電源により、ラック内のスペースと、鉄の電力と冷却のための電力が節約されます。 クラスター構成では、サーバーの前にあるディスクは一度に2つのノードに表示されます。 サーバーは以下に最適です。

  • 2ノードクラスターおよび多数のノードからのクラスターの実装（多くの負荷の高いWebサイトの検索およびコンテキストメカニズム）。
  • 仮想化（VMware、Microsoftなど）;
  • Hadoopのソリューション。
  • 1Uサーバーラックあたり最大密度のコンピューティングリソースが必要なお客様。
  • 省エネ。

一般的な印象



サーバーはしっかりと作られており、アグネイルやぶら下がるワイヤーはありません。 すべての鉄は、論理的で調和がとれており、開けやすく、分解しやすいように見えます。 彼と一緒に仕事できてうれしいです。

各サーバーにはリモート管理コンソールがあります。 別のネットワークポートを介して動作することも、OSからのデータ用の簡易ネットワークポートを介して動作することもできます。 この機能により、ラック内のスイッチング数を削減し、スイッチのネットワークポートを解放できます。 機能面では、コンソールは他のベンダーの同様のコンソールと変わりません。 ブレーキなしでスマートに動作します。 BIOSとIMMは、ブートディスクからではなく、同じコンソールから更新できます。



OSは、起動可能なUniversal Deployment Kitを使用してインストールされます。 サポートされているOSのリストには驚かされました。

  • Windows 2008 R2 \ 2012 \ 2012 R2;
  • Rhel 6.5＆7;
  • Sles 11 SP3;
  • Ubuntu 14.04および14.04.1。




不思議なことに、他のすべてはどこにあるのでしょうか？



実際、サポートされているOS（サーバーにインストールできるOS）のリストははるかに広くなっていますが、OSのインストール後にドライバーを追加してそのままインストールする必要があります。 メーカーのウェブサイトにはLinux / Window / ESX用のドライバーのセクションがあり、そこから必要なものをすべてダウンロードできます。 研究室では、UbuntuとSLESの最新バージョンを使用しましたが、完全に機能しました。



このサイトにはほとんどドキュメントがありません。 実際、データシート以外はありません。 これはいくつかの不便を引き起こします。 追加のSSDをRAIDに統合するには、メーカーに連絡する必要がありました。 この機会を得るには、BIOSに入り、いくつかの設定を変更する必要があることが判明しました。 その後、すべてが明らかです。 興味深いことに、人々はドキュメントなしでこれをどのように見つけますか？

QuantaPlex T21P-4U



これは4ユニットのサーバーで、いくつかの構成で利用できます。

  • ノードと78個のディスク。
  • 2つのノードと70のディスク。 8台のディスクの2番目のノードとトレイは交換可能なコンポーネントであり、各サーバーには35個のディスクがあります。


各ノードには、2つのホットスワップ可能なSSDを追加で提供できます。 ディスクはRaid1で結合されます。 これらのディスクにゲストOSを配置すると便利です。

ディスクコントローラーでは、2つのRAIDグループのみを作成できますが、70個のディスクでは十分ではないため、ソフトウェアレベルでRAIDグループにディスクを収集する必要があります。

サーバーの長さは標準ラックの長さに対応しており、スリーブはすでに突出しています。 したがって、サーバーは深いラックに設置するように設計されています。

QuantaPlex T21P-4Uは、多数のドライブを必要とするあらゆるタスクに使用できます。 可能性があります：

  • アーカイブストレージ;
  • ビデオ監視;
  • ファイルストレージ。
  • SDSおよび分散データストレージ（CEPHまたはEMC ScaleIO）。




QuantaGrid D51PH-1ULH-1U



サーバーは1ユニットを占有します。 前のモデルと同様に、その長さはラックの長さに対応していますが、このサーバーのすべてのホットプラグコンポーネントは、ラックから取り外すことなく変更できます。 したがって、彼はスリーブを必要とせず、サーバーを標準ラックに設置できます。



サーバーには、12台の3.5インチドライブと4台のSSDがあります。 RAIDコントローラは、さまざまなタイプのRAIDグループをサポートしています。 サーバーの前面から3.5インチドライブトレイが延びています。



このサーバーは、SDSおよび分散データストレージ（CEPHまたはEMC ScaleIO）、大規模データの保存および処理（Hadoop）に最適です。



ネットワークスイッチQuantaMesh T3048-IZ1



テストのために、サーバーハードウェアに加えて、ネットワーク1、つまりQuantaMesh T3048-IZ1も受け取りました。 解決策は興味深いものであり、一般的なシステムから際立っていますが、最初に最初のものがあります。

すべてが効率的かつ健全に組み立てられます。 フロントパネルのケーブルホルダーとハンドルに注意してください-些細なことですが、素晴らしいです。

しかし、最も興味深いのはもちろんスイッチの中です。 Intel ONPプラットフォーム（Open Network Platform）に基づいて構築されています-これは、高性能ネットワークプラットフォームとデータセンター向けのオープンなリファレンスデザインであり、SDN（Software Defined Networking）とNFV（Network Functions Virtualization）についての考えが組み込まれています。

これは、鉄レベルのこのスイッチがOpenFlowパケットをサポートし、Open vSwitchで動作する準備ができていることを意味します。

Intel ONPプラットフォーム自体が理由で登場し、Broadcomを抑制する試みであることに注意してください。Broadcomは現在、ネットワークセグメントで素晴らしい気分です。

ソリューションは標準化されているため、Intel ONPスイッチには、IntelイーサネットスイッチFM6000スイッチングマトリックスとx86プロセッサベースのサーバーパーツがあります。 2 GB DDR3および32 GB SSDを搭載したIntelイーサネットスイッチFM6764とIntel Core i3-2115Cは、スイッチで並行して動作します。 この構成は、1μs未満の遅延で1.28 Tb / sまたは9億6,000万パケット/ sのスループットを提供し、48ポートの1 / 10GbE SFP +および4ポートの10 / 40GbE QSFP +を取得します。

ハードウェアについて説明したので、ソフトウェアに移りましょう。 Intelは徹底的に登場し、Wind River Linuxに基づいたIntel ONS（オープンネットワークソフトウェア）のプラットフォーム全体を開発しました。 このオペレーティングシステムは、リアルタイムカーネルと高速化された高度なネットワークインターフェイスを備えているという点で興味深いものです。

ウインドリバー自体は最近Intelに買収され、リアルタイムオペレーティングシステムを独占的に扱っています。その例として、CuriosityローバーOSがあります。したがって、そのようなOSを信頼しない理由はありません。 SDKとAPI、およびプラットフォーム自体用のさまざまなデバッグユーティリティがリリースされました。これにより、自分でOSを変更したり、独自のプロトコルに追加機能を追加したりできます。 これで十分でない場合は、オープンブートローダーがスイッチ自体にインストールされ、x86アーキテクチャのおかげで、他のOSが利用可能になります。
