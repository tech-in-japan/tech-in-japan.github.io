Selectel Storage-クラウドストレージ

何を提供していますか？



Selectel Storageは、OpenStack Swiftクラウドベースのオブジェクトストレージソリューションです。



何のために？



ファイルバックアップの便利でシンプルで安価なストレージと、Webサイトおよびアプリケーション用の静的ファイル（画像、音声、ビデオ、js、css）の配布。



背景



クラウドでの仮想マシンの使用シナリオの分析では、ほとんどの仮想マシンユーザーはフロントエンドとバックエンドから複雑なバンドルをセットアップすることにあまり熱心ではないことが示されました。 したがって、サーバー側では、コンテンツの配信は、発信トラフィックとストレージコストだけでなく、書き込み/読み取り操作、プロセッサ時間、およびRAMオーバーヘッドによっても「評価」されます。



また、多くのクライアントはバックアップ用に個別の仮想マシンを保持します。これらの仮想マシンはバックアップ中にオンになり、その後オフになります。



これらのケースはどちらもリソースの誤用を伴うため、Selectel Storageを提供しています。



特徴



費用


Selectel Storageの使用コストを計算するとき、Selectel Cloudと同じ計算スキームを使用しました。これにより、ユーザーは実際に使用したリソースに対してのみ支払うことができます。 また、着信トラフィックの記録を拒否し、ユーザーはストレージからのデータの保存とダウンロードに対してのみ料金を支払います。



ストレージは、複製されたコピーを除く、アカウントに保存されているデータの総量によって考慮されます。 ネットワーク消費は、総発信トラフィックによって計算されます。



開発されたネットワークインフラストラクチャを備えたデータセンターを使用しているため、ストレージとデータ転送の価格を可能な限り低く抑えることができました。



無制限のサイズ


Selectel Storageを使用すると、必要な量のデータを保存でき、サイトのデータ、個人ファイル、およびさまざまなコンテンツのバックアップコピーを保存するための最適なソリューションを提供できます。



独自ドメイン


また、静的なデータを配布するために使用できる、リポジトリ内のコンテナにドメインをアタッチする機会を提供します。例：static.domain.ru/img.png





信頼性


ダウンロードしたすべてのデータは、すぐに3つの独立したサーバーに複製されます。 3台のサーバーのうち2台に障害が発生しても、データは影響を受けず使用可能になり、ストレージの内部制御システムは、失われたデータのコピーを他の稼働中のサーバーに自動的に配布します。



また、ユーザーAPIは、チェックサムによってダウンロードされたデータの整合性を制御する機能を提供します。



クイックアクセス


ストレージへのアクセスは、それぞれが10 GBitの速度で外部ネットワークに接続している複数の配布プロキシサーバーによって提供されます。



WEBインターフェース


Webベースのインターフェイスを使用すると、ファイルをすばやくアップロードおよびダウンロードし、アクセス権を設定し、ドメインをバインドし、統計を受信できます。











OpenStack Swift API


Selectel Storageは、多くのプログラミング言語（php、python、java、.Net、ruby）での実装例を使用して、RackSpaceとNASAが開発した一般に受け入れられている標準との互換性を提供します。



追加のAPI機能


  • 各オブジェクトのダウンロード数
  • アカウント全体およびコンテナごとのトラフィック統計






使いやすさ


  • 利用可能なソフトウェアでFTPプロトコル経由でアクセス
  • iOSおよびAndroidデバイスからアクセスする機能（サードパーティソフトウェアを使用）
  • ローカルデータをストレージと同期するための幅広いソフトウェア（リストはコントロールパネルで利用可能）
  • FUSEを介してパーティションとして接続する機能
  • スケジュールされたバックアップ自動化機能github.com/selectel/supload
