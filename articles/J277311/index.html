まず、AppleがSunSpiderのような従来のテストレポートよりもオープンなJSBenchレポートでより良い結果を報告するという事実に興味がありました。 JavaScriptの作成者であるBrendan Eichなど、JSBenchの背後には興味深い人々がいますが、従来のテストとは異なり、JSBenchは整数またはそのようなものが決定的な役割を果たすプログラムを作成しません。 代わりに、JSBenchはAmazon、Facebook、Twitterが提供するすべてのものを自動的に取得し、それらすべてからテストを作成します。 （率直に言って）ほとんどの人がFacebookにアクセスするために使用するWebブラウザーを作成している場合、Facebook専用のテストを行うことがどれほど便利か想像できます。 一方、スプレッドシートプログラム、ゲーム、または画像フィルタリングアプリケーションを作成している場合、整数演算とmd5ハッシュを使用した従来のテストは、Facebookの分析コードの速度を観察するよりもはるかに有用であると思われます。



別の重要な事実は、Appleによると、SunSpiderのパフォーマンスを向上させることは、他の何かを向上させることを必ずしも意味しないということです。 Appleの推奨テストを示す同じドキュメントで、Eichと他の人は次のように書いています。



「この図は、SunSpiderによると、Firefoxバージョン3.6のパフォーマンスがバージョン1.5に比べて13倍向上したことを明確に示しています。 それにもかかわらず、Amazonの改善点を見ると、3倍ほど控えめに見えます。 さらに興味深いことに、過去2年間でAmazonの改善は横ばいになりました。 どうやら、Sun Spiderでうまく機能するいくつかの最適化は、Amazonでは効果がありません。


同じドキュメントで、 JavaScript の作成者とMozillaの主要なアーキテクトの1人は、過去2年間にAmazonのJavaScriptサイトのパフォーマンスに何も起こらず、信じられないほどエキサイティングなことは何もなかったことを公然と認めています。 これは、マーケティング担当者が製品を何年も強制することを示唆しています。

しかし、いずれにせよ、オープンな情報に頼って、パフォーマンスの3.8倍の向上に関するAppleの声明は、必ずしもあなたにとって有益なものを意味するとは言えません。 また、AppleのChromeに対する優位性に反論するテストがあった場合、それらを公開することは許可されないと言えます。



このセクションを次の結論で締めくくりましょう。誰かが自分のWebブラウザーが高速であることを示すグラフを持っている場合、これはJS全体が高速になっているという意味ではありません。
