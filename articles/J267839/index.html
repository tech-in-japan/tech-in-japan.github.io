レディアセンブリニュースポータルNewsModxBox





2年前、 MODX Revolutionでのオンラインストアの完成したアセンブリについて書きました。 この間、かなりの数のオンラインストアがShopModxBoxで作成され（それらのほとんどはサードパーティの開発者によって作成されました）、現在、エンジンのインストール数は1か月あたり200〜300ユニットです。 このプロジェクトは現在も開発を続けており、カーネルに新しい便利な機能が追加されています。



ShopModxBoxで注目する主な品質は、高性能、柔軟性、最小限のコードサイズです（ShopModxBoxはMODX Revolutionフレームワークに基づいたソリューションであり、コード自体の部分は文字通り3〜5,000行のphpコード+ Smartyテンプレートです）。



そして先日、新しいアセンブリNewsModxBoxをリリースしました。 ここでの基礎はShopModxBoxの場合と同じです。ロジックのみがニュースポータルおよびメディア専用に調整されています。 アセンブリは小さなニュースポータルではなく、実際の小さなニュースポータルで行われ、多くの点で、電子メディアおよび混合メディアのビジネスロジックの現実に適合しています。

この場合、変数$ paramsの呼び出しで、制限、並べ替え、検索条件などを指定できます。 など 開発プロセスでは、アップロードプロセッサでコード変更の結果を確認するために、毎回ページを再読み込みする必要はありません。 最初に、すべてが管理パネルでデバッグされ、サンプルが適切にそこに行くと、同じ結果が前面に表示されます。 コメントメカニズム、出版物、フィードバックフォーム、その他すべてについても同じことが言えます。



テンプレート化



パターン化はSmartyに基づいており、プロセッサーに非常に適しています。 たとえば、これはニュースの配列を取得し、それらを最終的なHTMLにループする方法です。 



 {$params = [ "limit" => 10, "sort" => "pagetitle", "dir" => "DESC" ]} {processor action="web/resources/articles/getdata" ns="modxsite" params=$params assign=result} {if $result.success && $result.object} {foreach $result.object as $object} <p>{$object.pagetitle}</p> {/foreach} {else} <div class="alert alert-danger">{$result.message|default:"   "}</div> {/if}


もちろん、これは単純な例です（実際には、拡張可能なテンプレート、ブロック、インクルードなどがあります）が、一般的には、そこでどのように行われたかがわかります。 そして、テンプレートとコンソールの呼び出しの類似性に注意してください。 両方のパスは、呼び出されたプロセッサー（この場合、web / resources / articles / getdata）とオプションの$ params配列パラメーターに渡されます。



しかし、Smartyを使用する主な利点は、複数のデザインテンプレート（スキン）を一度に使用できることでしょう。 基本的なテンプレート（アセンブリに付属）があり、最終的なサイトでデザインやロジックを変更する必要がある場合、スキンの完全なコピーを作成して完全に切り替える必要はなく、追加のスキンを示すという原則があります既に変更を加えています。 これにより、独自のスキンの数が減るだけでなく、既に機能しているサイトでアセンブリ自体への変更をロールアップして、新しい機能を取得し、現在のデザインを壊すこともできます。 これを行う方法に関する詳細なマニュアルを次に示します。 そこで、ShopModxBoxの例では、繰り返しますが、NewsModxBoxの原則は同じです。



ダウンロードする場所と方法





方法1.クリーンなMODXにインストールする



注意!!! このパッケージを戦闘サイトにインストールしないでください。 これはサイト全体のスナップショットであり、現在のサイトを上書きします。 純粋なMODXにのみインストールされます。



したがって、インストールプロセスは、MODX Revolutionを既に使用しているユーザーにとって比較的簡単です。 きれいなMODXを入れてください。 次に、パッケージ管理で、アドレスhttp://rest.modxstore.ru/extras/の新しいリポジトリを追加し、そこからNewsModxBoxパッケージをダウンロードしてインストールします。



非常に不快なバグ、まだ受け入れられていないパッチがあるため、MODXバージョン2.4.0および2.4.1をインストールすることはお勧めしません。 2.3.6を置く方が良い



方法2。アーカイブから。



最初の方法は便利ですが、すべてのホスティングサイトが成功するわけではありません。 まず、パッケージの重量は50メートル以上であり、メモリがほとんど割り当てられていない場合は、単に開梱されない場合があります。 第二に、インストールは2秒間続きません。インストールプロセスは、タイムアウト（およびtimeweb.ruで、プロセッサが100％ロードされ、5秒後にスクリプトが終了する）によって中断されるだけです。 この場合、MODXに精通している、または単に数日間Web開発に従事している人には、 完全なダンプからサイトを展開するオプションが適しています。 このサイトはMODXで完全にパックされており、ルートにdump.sqlがあります。 必要なのは、それを解凍し、データベースを作成し、その中にダンプを記入し、サイトの設定を微調整することだけです。



両方のインストール方法は、 ShopModxBoxダウンロードページで詳細に説明されています （ビデオもあります）。



さて、アセンブリを自分でインストールできない、またはしたくないが、内部からそれを確認したい人のために、3番目のオプションがあります-テストサイトのインストールを注文してください。 発行価格は300ルーブルで、サイトは1か月間利用可能です。 それはホスティングにインストールされ、あなたからの不必要な動きを必要としません。 サービス注文フォームはこちらです。

簡単に言うと、システムの長所/短所
