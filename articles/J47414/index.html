1年前、私はLeopard（Mac OS X 10.5）を実行しているMac miniのサーバーをピックアップしました。すべてのまともな人々と同じように、FreeBSDの初歩/スタブを見つけ、finkポート（finkproject.org）をダウンロードしてインストールしました。 1年が経ちました... WiFiアダプターをインフラストラクチャモードにしました...今ではラップトップにインターネットを提供し始めました...トレントを24時間365日ダウンロードし、VNCストリームをLANから外部インターネットにリダイレクトします（自宅から遠く離れたときに同じラップトップからアクセスするために）...繰り返しになりますが、私の愛するSFTPディスケットに500Gのサイズを提供し、jabberのトランスポートを保持します...



そして、それは退屈になりました...

しかし、もしあなたが本物のFreeBSDを使ってそれを置いたらどうでしょうか？ 少年は言った-少年はしましたか？



FreeBSD 7.0 PowerPCイメージを焼き付けました。 Macはディスクを正常に起動しました。 インストールを開始します...おっと...しかし、fdiskはどこにありますか?? fdisk'aはありません。 ラベルエディターがあります。 しかし、彼が見つけ始めたとき、彼はパーティションテーブルで何もすることができません...そして私たちは去ります。 2週間の掘り下げの結果、小さなToDo「MacにFreeBSDをインストールする方法」が誕生しました。これをあなたの注意を喚起します。

AppleのOpen-Firmware（詳細は下記）。



2） 「特別な」オープンファームウェア

すべての進歩的な人類はすでにBIOSシステムを放棄しており、それらを徐々に（理論的には理解しているが、ポピーではすでに実践されているように）Open Firmwareに置き換えています。 セルフテスト機能を備えたハードウェアプリローダー。 ポピーでは、これはOpen Firmwareであることが示されていますが、一連のコマンドに基づいて、独自のOpen Firmwareのままです。 また、このOpen FirmwareはMac HFS +パーティションからの起動のみを理解します。 実際、タンバリンとのそれ以上のダンスもこのOpen Firmwareで発生します。

ここに記述されているもの-en.wikipedia.org/wiki/Open_Firmware 道徳的に死んだBIOSに代わる最新のテクノロジー。 管理コマンドのセットは、 www.firmworks.com / QuickRef.htmlにあります。 しかし、AppleのOpenFirmwareの特徴は、記載されているものと実質的に重複しないことです。 以下に、わずかに変更されたコマンドセットを示します（指定されたリファレンスのほとんどのコマンドは機能しません）。これらのコマンドの動作は、わずかに予測不能です。 非常に多くの場合、同じチームは仕事をやめるか、異なって働きます。 唯一の解決策があります-リブート。

だから、MacのOpenFirmwareへの入り口はAlt + Cmd + O + Fです。短いトレーニングセッションの後、この組み合わせは1つ右で簡単に押されます（左は機能しません-私は=を試しました）

1.ブートローダーの準備（隣接コンピューター上 ）



フラッシュドライブを取り、ローダーとその上にboot.tbxiをコピーします（任意のフォルダー、フラッシュドライブ-持ち運び用のツールとして、これ以上）

どんな種類のファイル...そしてどこで入手できますか？ loader-FreeBSDインストールディスク（\ boot \ loader）にあり、boot.tbxiは同じ場所にありますが、見つからない場合は修正または再作成する必要があります。

これは、次のようなOpen FirmwareのForthブートスクリプトです。



<ブートスクリプト>

「画面」出力

boot hd：???、\ boot \ loader hd：???

</ boot-script>



ブートコマンドに興味があります。 その構文を扱いましょう：

1番目のパラメーターは、ブートローダー（この場合は、FreeBSDのboot2 / EFIブートローダー）の場所です。 この例では、シンボリック置換、hd-ハードディスク（cd-コンパクトディスク）、パーティションのコロン番号（少し低いパーティション番号について）、およびこのパーティション上のブートローダーファイルへの忘れたパスが示されています。 HFS +ファイルシステム。



2番目のパラメーター-ブート可能システムのカーネルの場所。 ここでは簡単です-セクション番号だけです。 パラメーターはファイルローダーの入力に渡されるため、ファイルシステムの形式に関する要件はありません。



必要なパーティション番号を見つける方法は、これらのパーティションを作成するときにそれぞれ見つけます。 そのため、近隣のコンピューターでスクリプトを編集してもそこで終わりません...



2.ハードドライブを準備します。



MAC OS Xインストールディスクが1回必要になるため（問題番号4を参照）、そこから開始します。

2.1ディスクユーティリティを使用してディスクを分割します。



ラベルエディターを使用して1つのセクションを作成し、スライスに分割してみませんか？ 動作しません。 ラベルエディターは、そのために作成されたパーティションのみを使用できますが、パーティションテーブルに干渉することはありません-動作しません。 だからこそ、スライスごとに独自のセクションを作成する方が良いのです...

ディスクのパーティションを作成する前に、パーティションテーブルのタイプを選択する必要があります。 GUID、Apple Partition MapまたはMBR。 PowerPC mac-APM、Intel mac-GUIDを選択する必要があります。



次のセクションを実行しました。

ルートの下-4 Gb

スワップ下-2 Gb

var-2 Gb未満

usrの下-8 Gb

家の下-残っているすべて、すなわち22 Gb。



次のステップで端末でそれらを識別しやすくするために、これらのセクションにroot、swapなどのラベルを設定します。



一般に、あらゆる種類のブルジョアフォーラムでは、グリッチの疑いがあるため、Disk Utilitiesを長時間使用することをお勧めしません。 私はこの事実を否定しません、それはそのようでしたが、私は曲がった手にそれを書き留めました。

Mac OSおよびBSDファイルシステムの互換性について少し説明します。 Disk Utilitiesには4種類のファイルシステムから選択できます-それらはすべて、長くて、ファイル名の大文字と小文字を区別するためのサポート（またはなし）ジャーナリングとサポート（またはなし）を備えた拡張ファイルシステムと呼ばれます。 これは実際には、オプション付きのジャーナリングまたは非ジャーナリングExt3にすぎません。 したがって、これらのセクションの理論と実践の両方をBSDの下から読むことに問題はありません（彼はUFSに変換します）。 しかし、まだ問題があります。 HFS +でブートローダーをロードする必要があり、カーネルはすでにBSDで認識されるパーティションからです。

インストールされたBSDをさらに正常にロードするには、ブートローダーとカーネル自体を分離する必要があります。 これを行うには、ディスク上で既に説明したパーティションにブートローダー 'aのパーティションを作成します。 サイズを0.2 GBと指定しました。Leopardはすぐに1 GBに調整しましたが、パーティションプロセスの後、パーティションのサイズは0.86 GBになりました。 まあ、彼と議論しないでください=）。 さらに、インストール中にBSDはそのパーティションをポピーから読み取れないようにします。そのため、Leopardのインストールディスクは、BSDをハードディスクにインストールする前に一度だけ使用します。 そうしないと、次にLeopardインストールディスクを起動するときに、Mac OSは準備されたすべてのパーティションを不適切な程度に「改善」するため、インストールプロセスを最初からやり直す必要があります。



したがって、次のパーティションテーブルがあります。

ルート（4G）

var（2G）

usr（8G）

ホーム（22G）
