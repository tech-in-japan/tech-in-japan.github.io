知事および販売代理店向けモバイルアプリ

こんにちは、Habr！ Habrは政治のためではありませんが、たまたまEOS（EOS-Electronic Office Systems）の開発が民間企業だけでなく政府機関の間でも人気があるのは偶然です-全国の数千のオフィスとCISで当社のソリューションが使用されています。 今日は、EOSモバイルアプリケーションについてお話ししたいと思います。 職員の間でモバイルビジネスアプリケーションがどのように使用されているかを知ることは、誰にとっても興味深いと思います。







モバイルEOSソリューションのエコシステムには、次の3つの開発が含まれます。



  1. iOSの場合、iEOSアプリ
  2. Windows 8用マネージャーワークステーション
  3. そして、それらの共通の祖先はWindows 7用のマネージャーワークステーションです。


ちょっとした歴史



EOSでのモバイル開発は2007年に始まりました。 しかし、転換点は2010年であり、最初のiPadがリリースされた年です。 それから、モバイルソリューションはエキゾチックではなく、Blackberryのような個々の狭プロファイルの企業の運命ではなく、今日であることも明らかになりました。 ロシアでは、iPadの人気は個人的にメドベージェフ大統領によって促進されました。メドベージェフ大統領はiPadを積極的かつ公的に使用し始め、iPad、つまりAWPのモバイルバージョンの関係者の間でリクエストを作成しました。 そして、2013年にはWindows 8用のバージョンがありました。そして最後に、今年、Android用のバージョンのプレゼンテーションが行われます。







開発プロセスでは、各プラットフォームに固有の多くの問題が発生しました。



iEOS（2010）：



  1. 未知のモバイルプラットフォームとの出会い。 セキュリティ制限。
  2. 新しい設計要件。 指の向き;
  3. 客観的なCに精通している。
  4. 外国店での国内電子製品の流通の問題。 暗号化マシンであり、カード内にアルゴリズムを実装するJaCartaへのオリエンテーション。
  5. 通常のpdf処理ツールの欠如。 標準ツールではサポートされていなかった注釈を挿入する必要がありました。 その結果、PDFで注釈の解析、読み取り、書き込みを行いました。
  6. （モバイルソリューションサーバー）-異なるEDMS（うち3つ）からのドキュメントをさまざまなタイプのクライアントに同期します。 複雑なロジック：同時に到着するバージョンの競合を解決します。 モバイルクライアント向けのタイムゾーンおよびモビリティソリューション（地球上を移動することによるゾーンの変更）。 不安定な通信チャネルで動作します。 さまざまな形式を変換します。


Windows 8 / RTでのマネージャーワークステーション：



  1. Modern UIアプリケーションの開発の開始時には、.NET開発者になじみのある多くのコンポーネントとツールが欠落していました。 特に、UnityおよびEntity Frameworkには欠けていました。
  2. Windowsランタイム8にはPDFラスタライザーがなく、サードパーティのコンポーネントを使用する必要がありましたが、その速度は満足のいくものでした。 その後、Windowsランタイム8.1にラスタライザーが登場したとき、問題はもはや関係ありませんでした。
  3. Windows 8.1で導入されたコントロールを使用すると、Windows 8とWindows 8.1のアプリケーションのバージョンに大きな違いが生じました。
  4. AWPのデスクトップバージョンは、Microsoft Wordとの密接な統合をユーザーに提供しました。 Modern UIアプリケーションとWindows RT用のOfficeバージョンの制限により、AWP RTの統合は一方的であることが判明しました。


現在、EOSモバイルアプリケーションは、ロシアおよびCISの100を超える組織で使用されています。 確かに今この記事を読んでいる人の中には、私たちのソリューションで作業している多くのユーザーがいます。



アプリケーションは異なる時期に開発されたという事実により、それらの機能はわずかに異なりますが、主にそれらは同じです。マネージャーのワークステーションとiEOSは、ドキュメントを操作するように設計されています：レビュー、閲覧、署名、実行制御、イニシアチブ指示。



この場合：



  • Win 8のバージョンでは、ファイルを添付し、ポータブルスキャナーで作業でき、EDSIGNソリューションを使用してドキュメントに署名と印刷スキャンを追加できます（電子署名を視覚化-印刷+署名など）。したがって、Word文書に直接Wordで署名できます。次に、たとえば、Win8マネージャーのAWPで注文を開き、この署名済み文書を添付します。
  • iEOSは、無線リーダーを含むロシアの会社Aladdin R. D.のソリューションを使用した電子デジタル署名をサポートしています。


EOSモバイルソリューションは誰のためですか？



  • 上級管理職および中間管理職の場合：ドキュメントを扱う、署名する、ドキュメントの部下の仕事を委託および管理する必要があるすべての人。
  • 上司のようにタブレットを使用するアシスタントマネージャーの場合。
  • オフィスの外のドキュメントで作業する必要があるライン従業員-たとえば、フィールドエージェント。


あなたが大ボス、中間管理職、または普通の従業員であるかどうかは関係ありません-ドキュメントを扱う場合は、ほとんどの場合、私たちのアプリケーションはあなたのために特別に作成されました。



仕組み



ドキュメントはシステムの中心です。 ドキュメントの主な作業は、本格的なデスクトップオフィス管理システムで実行されます：「ケース」、EOS for SharePoint、eDocLib。 モバイルソリューションは、それらの一種の拡張機能として機能します。これにより、インターネットを使用しなくても、事実上あらゆる環境で論文を使用した運用作業を行うことができます。



従業員は上記のシステムのいずれかで生成されたドキュメントを受け取り、iEOSモバイルアプリケーションまたはマネージャーのワークステーションで開くことができます。 検討、承認、署名、レポート、アーカイブなど、さまざまなステータスのすべての受信ドキュメントが表示されます。

行われた決定に応じて、解決策（順序）がドキュメントに適用されます。 解像度のテキストは、ユーザーが作成するか、システムからロードされるテンプレートのリストから選択することができます。 同時に、ドキュメントを扱う従業員は、日付を決定し、実行責任者を選択し、コントローラーを任命して、回答の通知を受け取ることができます。

プラットフォームの人気



ほとんどの場合、おそらく自分で推測できるように、iOSバージョンは需要があります。 これにはいくつかの理由があります。メドベージェフ大統領が設定したAppleデバイスの流行は消えていません。 しかし、ファッションに加えて、Apple全体がより高度なモバイルインフラストラクチャを備えていることがより重要です。Microsoftのソリューション（Surfaceタブレット、企業のDellタブレットなど）は勢いを増しています。 したがって、現在、iOSおよびWindowsのバージョンのユーザーの分布はほぼ70/30です。 さらに、これらの30％のうち、小さい部分はWindows 8でデバイスを使用します-それらの大部分は、Win 7で動作するマネージャーのワークステーションの最初のバージョンを使用します-つまり すでにデスクトップ上にあります。



Androidはどうですか？ これは別の興味深い話です。 知事や代理人の中にはAndroidを気にしない人もいることは明らかです。彼らはすべて「ipads」で購入されており、それらをすばらしく使用しています。 しかし、多くの場合、従業員は仕事を家に持ち帰ったり、個人のタブレットをオフィスに持ってきたりします。 ここでは、単純な平凡な算術のおかげで、配布はiOSに有利ではありません。Androidデバイスの市場シェアははるかに高いです。 さらに、多くの企業がタブレットを製造しており、タブレットは最初は特にAndroidのビジネスユーザーに焦点を当てています。 一般的に、このOSに対する需要は増大しています。今年はAndroid向けのソリューションをリリースすることで、このOSを満足させようとします。 一般的に、お楽しみに！







会社について



EOS会社は1994年から存在し、200人以上の従業員がいます。 本社はモスクワにあり、遠隔ユニット（コールセンターとテスター）はウラジミール州コブロフにあり、さらにミンスクに追加の開発オフィスがあります。 現在、EOSソリューションはロシアのすべての地域と多くのCIS諸国の両方で使用されています。
