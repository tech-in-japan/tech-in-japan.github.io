世代の異なるIntelプロセッサのパフォーマンスの比較

ほぼ毎年、新世代のIntel Xeon E5中央演算処理装置が市場に登場します。 各世代では、ソケットとプロセスが交互に変わります。 核はますます増えており、熱は徐々に減少しています。 しかし、「新しいアーキテクチャはエンドユーザーに何をもたらすのでしょうか？」という自然な疑問が生じます。



これを行うために、異なる世代の同様のプロセッサーのパフォーマンスをテストすることにしました。 マスセグメントのモデルを比較することにしました：8コアプロセッサ2660、2670、2640V2、2650V2、2630V3、および2620V4。 同様の世代間スプレッドでのテストは完全に公平ではありません。 V2とV3の間には、異なるチップセット、より高い周波数の新世代メモリがあり、最も重要なのは、4世代すべてのモデルに周波数の直接的なピアが存在しないことです。 しかし、いずれにせよ、この研究は、実際のアプリケーションと合成テストでの新しいプロセッサのパフォーマンスがどの程度成長したかを理解するのに役立ちます。



選択されたプロセッサラインには多くの同様のパラメーターがあります 。同じ数のコアとスレッド、20 MB SmartCache、8 GT / s QPI（2640V2を除く）、およびPCI-Eラインの数は40です。



すべてのプロセッサのテストの適切性を評価するために、 PassMarkテスト結果に注目しました 。



結果の概要グラフは次のとおりです。







頻度は大きく異なるため、結果を比較することは完全に正しいとは限りません。 しかし、それにもかかわらず、結論はその場で思い浮かびます：



2620V4と同等のパフォーマンスで1.2660

2.22670は2620V4よりもパフォーマンスが優れています（明らかに周波数のため）

3. 2640V2がサグし、2650V2が全員にヒットします（これも周波数による）



結果を周波数で除算し、1 GHzでパフォーマンス値を取得しました。







ここで、結果はより興味深く明確です。



1. 2660および2670-1世代のフレームワーク内での予期しない実行、2670は全体的なパフォーマンスが非常に高いことのみを正当化します

2. 2640V2および2650V2-非常に奇妙な低い結果であり、2660の結果よりも悪い

3. 2630V3および2620V4-唯一の論理的な成長（明らかに、新しいアーキテクチャのため...）



結果を分析した後、さらにテストする価値がない興味深いモデルの一部を除外することにしました。



1. 2640V2および2650V2-私の意見では、中間世代であり、あまり成功していない-候補から除外

2. 2630V3は優れた結果ですが、同様のパフォーマンスを考えると2620V4よりも不当に高価であり、さらに、これはすでに世代を超えたプロセッサです

3. 2620V4-適切な価格（2630V3と比較）、高性能、そして最も重要なこと-これは、リストにあるハイパースレッディングを備えた最新の8コアプロセッサの唯一のモデルです。

4. 2660および2670-2620V4と比較して優れた結果。 私の意見では、特に興味深いのは、Intel Xeon E5ラインの最初と最後の（現時点で）世代の比較です。 さらに、倉庫には第1世代プロセッサの十分な在庫があるため、この比較は非常に重要です。



2660プロセッサーと2620V4プロセッサーをベースにしたサーバーのコストは、後者を支持しない最大2倍異なる場合があります。したがって、パフォーマンスを比較し、V1プロセッサー上のサーバーを選択すると、新しいサーバーを購入する予算を大幅に削減できます。 しかし、テスト結果の後にこの提案についてお話します。



テストのために、3つのスタンドが組み立てられました。



1.2 x Xeon E5-2660、8 x 8Gb DDR3 ECC REG 1333、SSD Intel Enterprise 150Gb

2.2 xeon E5-2670、8 x 8Gb DDR3 ECC REG 1333、SSD Intel Enterprise 150Gb

3. 2 x Xeon E5-2620V4、8 x 8Gb DDR4 ECC REG 2133、SSD Intel Enterprise 150Gb



PassMark PerformanceTest 9.0



テスト用のプロセッサを選択するとき、合成テストの結果を既に使用していましたが、これらのモデルをより詳細に比較することは興味深いです。 比較はグループで行われました：第1世代と第4世代。







より詳細なテストレポートにより、いくつかの結論を引き出すことができます。



1.数学、含む。 浮動小数点、主に周波数依存。 100 MHzの違いにより、2660は、決済操作、暗号化および圧縮（およびメモリ周波数の大幅な違いにもかかわらず）で2620V4を上回ることができました。

2.新しいアーキテクチャの高度な命令を使用した物理学と計算は、低頻度にもかかわらず、より良く実行されます

3.もちろん、メモリを使用したテストはV4プロセッサに有利に合格しました。この場合、異なる世代のメモリ（DDR4とDDR3）が競合したためです。



それは合成でした。 特殊なベンチマークと実際のアプリケーションが表示するものを見てみましょう。

CINEBENCH R15



CINEBENCHは、プロのアニメーションプログラムMAXON Cinema 4Dで動作するコンピューターのパフォーマンスを評価するためのベンチマークです。







Xeon E5-2670は周波数を抜いて2620V4を打ちました。 しかし、周波数でそれほど目に見えない利点を持っているE5-2660は、第4世代のプロセッサに失われました。 したがって、結論-このソフトウェアは、新しいアーキテクチャに便利な追加機能を使用します（すべてがメモリ内にある可能性はありますが...）が、これは決定的な要因ではありません。



3DS MAX + V-Ray



実際のアプリケーションでレンダリングする際のプロセッサのパフォーマンスを評価するために、3ds Max 2016 + V-ray 3.4 +いくつかの光源、ミラーと透明なマテリアル、および環境マップを含む実際のシーンを束ねました。







結果はCINEBENCHに似ていました。XeonE5-2670は最短のレンダリング時間を示し、2660は2620V4を取得できませんでした。



1C：SQL /ファイル



結論として、1Cのgilevテスト結果を同封します。







ファイルアクセスを使用してデータベースをテストする場合、E5-2620V4プロセッサが自信を持ってリードします。 表は、同じテストの20回の実行の平均値を示しています。 ファイルベースの場合の各スタンドの結果の差は2％以下でした。



SQLデータベースのシングルスレッドテストでは、非常に奇妙な結果が示されました。 2660と2670の異なる周波数、およびDDR3とDDR4の異なる周波数を考えると、違いはわずかであることが判明しました。 SQL設定を最適化する試みがありましたが、結果はそれよりも悪いため、基本設定ですべてのスタンドをテストすることにしました。







マルチスレッドSQLテストの結果は、さらに奇妙で矛盾していることが判明しました。 1スレッドの最大速度（MB /秒）は、以前のシングルスレッドテストのパフォーマンスインデックスと同等でした。



次のパラメーターは（すべてのフローの）最大速度でした-結果はすべてのスタンドでほぼ同じでした。 異なる実行の結果は強く変動したため（+ -5％）-時々、それらは異なる方向にあり、一方の方向と他方の方向の両方に大きなマージンがありました。 マルチスレッドSQLテストの同じ平均結果は、3つの考えにつながりました。



1.この状況は、最適化されていないSQL構成が原因です。

2. SSDがシステムのボトルネックになり、プロセッサがオーバークロックできなくなりました

3.これらのタスクのメモリとプロセッサの頻度にほとんど違いはありません（これはほとんどありません）

結論



いくつかの汎用計算テストの結果は、ほとんどの場合、プロセッサ周波数が世代、アーキテクチャ、さらにはメモリ周波数よりも重要であることを示しました。 もちろん、新しいアーキテクチャのすべての改善点を使用する最新のソフトウェアがあります。 たとえば、次のようなビデオトランスコーディングが実行されることがあります。 AVX2.0の指示を使用しますが、これは特殊なソフトウェアです。ほとんどのサーバーアプリケーションは、コアの数と頻度に関係しています。



もちろん、プロセッサ間にまったく違いがないことは宣言しません。特定のアプリケーションでは、「計画された」新世代への移行には意味がないことに注意してください。
