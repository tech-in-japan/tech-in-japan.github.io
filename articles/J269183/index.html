パート2. Officeストアでのアプリケーションの公開









ストアで公開するプロセスでは、いくつかのタスクに直面します。



  1. Officeストア用のアプリケーションを作成する
  2. https://sellerdashboard.microsoft.com/registrationの （Microsoftアカウントを使用して）Sales Dashboardでアカウントを作成します。
  3. アプリケーションの価格を決定し、必要に応じて、税務プロファイルに関する情報を提供します。
  4. サポートされている1つ以上の言語と国にアプリケーションを追加します。

    1. アカウントがまだ承認されていない場合でも、アプリケーションを追加および保存できます
    2. アカウントが承認されるとストアのアプリケーションを公開し、税金と支払いに関する情報が入力されると有料のアプリケーションを公開する


  5. ストア承認アプリを送信する

    1. ステータスはコントロールパネルで確認でき、変更の必要性がそこに報告されます。
    2. 承認後、アプリケーションは1〜3日後にストアに表示されます。


  6. 指標を追跡し、お金を管理する




ステップ1.アプリケーションを作成する



この記事では、OfficeおよびSharePoint用のアプリケーションの作成方法については説明しません。 これについては、シリーズの一部として公開されている他の資料で詳しく説明しています。 ニュアンスについて話しましょう。



開始方法については、 http：//dev.office.com/getting-startedにアクセスしてください。

アプリケーションを開発するには、たとえばVisual Studio 2015など、さまざまなIDEを選択できます。http://visualstudio.comからOffice Developer Toolsをダウンロードすることを忘れないでください。 同じツールを使用して、アプリケーションパッケージ/マニフェストを作成できます。 アプリケーションマニフェストを公開できるようにするには、次の推奨事項を考慮してください。



  1. アプリケーションをダウンロードするとき、次のように、アプリケーションの名前の選択を慎重に検討してください。

    1. マニフェストの名前は、OfficeまたはSharePointにアプリケーションをインストールした後にユーザーに表示される名前です。
    2. セールスダッシュボードの名前は、ストアでアプリケーションが呼び出される方法です
    3. これら2つの名前の不一致は、ユーザーを困らせる可能性があります
    4. マニフェストの表示名（Officeアプリ）またはタイトル要素（SharePointアプリ）を使用する


  2. Application Storeのアイコンを2つの異なる場所に2回ダウンロードする必要があります
  3. プロバイダーがホストするSharePointストアアプリには、クライアントIDとシークレットにセラーダッシュが必要です

    1. クライアントIDとシークレットにより、リモートアプリケーションコンポーネントはユーザー権限なしでSharePointデータにアクセスできます。
    2. ストアアプリを作成する場合、SharePointを使用してIDとシークレットを作成することはできません
    3. これらのSDで作成されたアイテムは、Visual Studioのプロジェクトのweb.configおよびマニフェストファイルに追加されます。




ステップ2.販売ダッシュボードでのアカウントの作成



Officeストアでアプリケーションまたはアドオンを公開する予定がある場合は、まず販売ダッシュボードでユーザーアカウントまたは組織アカウントを作成する必要があります。 決定事項を販売する場合は、支払いの詳細も追加する必要があります。



注： 会社が既にMicrosoftアカウントでMicrosoftストアに登録されている場合、同じアカウントを使用してSales Dashboardに登録する必要があります。 別のMicrosoftアカウントを使用して、会社をセールスダッシュボードに登録することはできません。 提供される情報は、このアカウントを使用してMicrosoftストアで提供される情報と一致する必要があります。 たとえば、会社の同じ名前と住所を提供する必要があります。



登録する際は、アカウントの種類（個人または組織）を一度選択すると、後で変更することはできません。 新しいアカウントを作成する必要があります。



アカウントには、マーケティングプロファイルも含まれています。 マーケティングプロファイルから、顧客はあなたの体験について、またアプリケーションやアドオンがビジネスタスクに対処するのにどのように役立つかについて学びます。 よく考え抜かれたマーケティングプロファイルは、潜在的な顧客を引き付ける可能性を高めます。 文法、情報の一意性、完全性について忘れないでください。 これにより、登録の承認をすぐに得ることができます。



マイクロソフトの推奨事項を使用することを忘れないでください。 ドキュメントの対応するセクションで見つけることができます（たとえば、 https：//msdn.microsoft.com/RU-RU/library/jj635874.aspxを参照 ）



セールスダッシュボードでアカウントを作成する手順を見てみましょう。



  1. Microsoftアカウントを使用してセールスダッシュボードにログインします。 あなたまたはあなたの会社がMicrosoftアカウントを持っていない場合、 登録をクリックしてください。 会社に関連付けることができるMicrosoftアカウントは1つだけです。

  2. 個人情報を入力してください。 会社のアカウントを作成する場合は、名、姓、連絡先の電子メールフィールドに入力して、会社の管理者に連絡先情報を提供します。







  3. 居住地または会社の法的住所に関する情報を提供します。





  4. [アカウントの種類]セクションで、正しいアイテムが選択されていることを確認し、[次へ]ボタンをクリックします
  5. 次のページで、マーケティングプロファイルを記入します
  6. 表示名を選択する必要があります。 これは、Officeストアで顧客があなたを認識する名前です。 名前はストア全体で一意でなければならないことを忘れないでください。そうしないと、名前は承認されません。







  7. 次に、ロゴを追加する必要があります。 「デフォルトのロゴ」のままにしておくこともできますが、写真からすぐにユーザーに認識してもらいたい場合は、印象的で思い出に残るものを作成してください。 画像サイズは96x96ピクセル、重量-250 kV以下である必要があります。 ダウンロードする前に、ウイルスのロゴを確認することを忘れないでください。







  8. あなたまたはあなたの会社の説明を入力してください。 説明で、クライアントがあなたについて知っておくことが重要であることを示してください。あなたがアプリケーションを選択してビジネス環境にインストールできるように、あなたがプロとして信頼できるようになります。 このような説明を読んだ場合、何に感銘を受けるかを考えてください。 テキストは複雑で、複雑な書式設定は含まれません。 アイテムを追加する場合は、ハイフンまたはタグ[#LI]および[/＃LI]を使用できます。 そして、もう一度、思い出してください、文法をチェックしてください。







  9. 顧客があなたとあなたのアプリケーションについてさらに学ぶことができるウェブサイトを示してください。 サイトの所有者であり、サイトが機能している必要があります。 URLはhttp：//またはhttps：//で始まる必要があります







  10. [承認のために送信]ボタンをクリックします。

    企業アカウントと個人アカウントを作成するプロセスはわずかに異なります。

    プロセスの最後に、ダッシュボードに次の図が表示されます。









現在、申請が検討および承認されるのを待つ必要があります。 どれくらい時間がかかりますか？



1日から2週間。 2週間後もアプリケーションのステータスが「アカウント保留中の承認」状態のままである場合-テクニカルサポートにお問い合わせください。



何がチェックされますか？



フェーズ1.本人確認



  • あなたは誰であると主張していますか？ このステップは、開発者をなりすましから保護し、顧客が信頼できるソースからデータを受け取ることを保証します
  • 加入者は会社と提携しており、アプリケーションの公開を許可されていますか？
  • アドレスは提供されていますか（あなたまたはあなたの会社）？


フェーズ2.情報監査



  • マーケティングプロファイルチェック
  • 名前/会社名。 名前の一意性を確認してください
  • ウェブサイト サイトが存在するかどうか、そしてあなたがそれを所有しているかどうかを確認します
  • 会社の連絡先。 会社に連絡し、有効な連絡先が提供されているかどうかを確認します
  • Microsoftアカウント このアカウントが他のMicrosoftストアで使用された場合、誰によって
  • ロゴ。 サイズ、形式、コンテンツ




アカウントを作成するためのアプリケーションの検証と承認を待っている間に、財務プロファイルに記入できます。



支払いと税金



  1. 「支払いと税金」タブを選択します





  2. 支払い方法を選択します。 たとえば、銀行口座とPayPalのどちらかを選択できる場合。 そうでない場合は、何を受け入れます
  3. 支払い情報を追加します。 銀行のSWIFTコードを知っている必要があります（銀行のWebサイト、詳細、またはオペレーターなどから見つけることができます）。 アカウントの所有者の名前は複雑さを増すことはありません。おそらく、彼のことをよく知っているからです。 IBAN番号は、口座明細書など、またはインターネットバンキングサイトで確認できます。 あなたはあなたの銀行に電話するか、行くことで見つけることができます。 また、他の詳細が必要です（カザフスタンの場合、IINやRNNなど）





  4. 請求先住所を入力します。 すべてがシンプルです。 注意点-ラテン文字のみを使用します。 キリル文字は使用できません。





  5. その後、[次へ]ボタンをクリックして、税務フォームと外国原産地証明書（米国居住者でない場合）に記入します。







  6. すべてが正しく記入されていれば、有料アプリケーションを公開し、それらのお金を受け取ることができます







アカウントを作成するためのアプリケーションの承認には時間がかかるため、コメントを修正して承認を何度も待つ必要がないように、アプリケーションのすべてのフィールドに正しく入力してください。



発言。 アカウントを変更すると、承認プロセスは最初から開始されます。



ステップ3.アプリケーションを追加する





アプリケーションの公開は、非常に簡単な手順です。 Officeストアでの公開用に準備した後、いくつかの決定を行う必要があります。



  • サポートする言語と市場を決定します。 最小-ストア内のローカライズされたメタデータ。 より良いのは、ユーザーインターフェイスをローカライズすることです。 アプリケーションに言語を追加すると、この言語を使用するユーザーをサポートする義務が課せられることを忘れないでください。
  • アプリケーションの価格を決定します。 利用可能なオプションは、無料、ユーザーごとの支払い（1回限りまたは毎月のサブスクリプション）、または試用の可能性がある有料版のみです。 SharePointおよび電子メールアプリケーションには追加のオプションがあります。 試用版のパラメーターは、開発者が完全に決定します。


公開を始める前に準備する価値のあるもの。



  • すべてのテキストエントリの表現

    • アプリケーションのタイトル、タイトル、短い説明と長い説明、すべてが必要に応じてローカライズされています。


  • 有効なメタデータ

    • アプリケーションのバージョン番号、リリース日、1-3アプリケーションがストアで表示されるカテゴリ


  • 正しいアプリケーションロゴと1〜5のスクリーンショット
  • テスターがアプリケーションを正しくテストできるようにするためのメモ（もちろん一般向けではありません）

    • 有効なユーザーデータ、リンク、手順をここに含めてください。


  • ドキュメントとプライバシーポリシーへのリンク
  • オプション：デモビデオおよびカスタムエンドユーザーライセンス契約へのリンク


アプリケーションを公開するプロセスを、いわば写真で見てみましょう。



  1. ダッシュボードに移動し、[新しいアプリケーションを追加]ボタンをクリックします





  2. アプリケーションの種類を選択します。 ストア向けに特定のタイプのアプリケーションまたはアドオンを初めて作成する場合、契約が表示されます。 「市場の状況」セクションで契約を読み、同意する場合はチェックボックスをオンにして、[次へ]または[キャンセル]ボタンをクリックします。





  3. 概要ページで、次を実行します。

    1. [一般情報]セクションで、アプリケーションまたはアドインの種類に応じて、名前アプリケーション、バージョン、リリース日などの情報を指定します。 SharePointアドインの場合、[ バージョン]フィールドで指定された番号は、アップロードされたマニフェストファイルのバージョン番号と一致する必要があります。 [ リリース日]フィールドの値は、アプリケーションまたはアドインのリリース日として表示される日付です。 更新を送信して将来の日付を選択すると、その日付が選択されるまで、Officeストアでアプリケーションまたはアドインが検出されません。 アプリケーションまたはアドインがOfficeストアで利用可能であった場合、それらは更新バージョンのリリース日までユーザーから非表示になります。
    2. 少なくとも1つのカテゴリを選択してください。 Officeストアでリストをフィルター処理することで、顧客がアドインを見つけやすくするために、最大3つのカテゴリを選択できます。 アプリケーションまたはアドインの教育カテゴリを選択した場合、 検証ポリシーのポリシー7.17に従う必要があります。





    3. ロゴセクションで、アプリケーションロゴタイルを選択し、ロゴファイルを送信します。 Officeアドインでは、アドインに関連付けられているロゴは1つだけです。 ローカライズが必要な場合は、ローカライズされたアドオンを各言語のロゴとともに個別に送信する必要があります。





    4. 送信のタイプに応じて、推奨アクションのいずれかを実行します。

      1. Officeアドインの場合-[ アプリケーションパッケージ]セクションで、 [アプリケーションパッケージ]タイルを選択し、アドインのマニフェストファイルを送信します。
      2. SharePointアドインの[ アプリケーションパッケージ]セクションで、 [アプリケーションパッケージ]タイルを選択し、アドインを送信します。


    5. [テストノート]セクションで、アプリケーションまたはアドインを示すリソースまたはビデオへの指示とリンクを入力して、テスターがアプリケーションを承認できるようにします。 たとえば、必要に応じて有効な資格情報またはパスワードを提供できます。 資格情報とパスワードは一般的な用途ではなく、Microsoftのみが使用します。 [テストノート]セクションのデータは承認のみを目的としており、ストアリストには公開されません。 詳細なテストノートを提供すると、アプリケーションの承認に役立ちます。 Webアプリケーションの場合、テスト担当者がアプリケーションを見つけるのに役立つように、テストURLにシングルサインオンURLを提供します。



    6. アプリケーションまたはアドインが暗号化または暗号化を呼び出し、サポート、含む、または使用する場合、チェックボックスを選択すると、 アプリケーションは暗号化または暗号化を呼び出し、サポート、含む、または使用します 。 詳細を確認し、「はい」または「いいえ」を選択します。







      暗号化の詳細については、暗号化を使用するアイテムのEARコントロールを参照してください。

    7. アドインがiOSと互換性がある場合、 iPadのOfficeアドオンディレクトリにこのアドインを含めるを選択します 。 Apple IDフィールドにApple Developer IDを入力します。



      IOS互換のアドオンは、追加の検証要件を満たす必要があります。 詳細については、Officeストアに送信されるアプリケーションとアドインの検証ポリシー（バージョン1.8）を参照してください。



    8. ヘルプドキュメントのセクションで、ヘルプドキュメントのWebサイトへのリンクを入力します。 http：//またはURLを含めます。 さらに、プライバシードキュメントへのリンク、ビデオへのリンク、および修正されたエンドユーザーライセンス契約を提供できます。 ライセンス契約を変更しない場合は、ストアが提供します。





  4. [ 次へ]、 [ ドラフトとして保存]または[ キャンセル]をクリックします
  5. 次のページInformationで、言語設定を選択する必要があります。

    1. [言語の選択]ドロップダウンリストで 、アプリケーションまたはアドインがサポートする言語を選択し、[確認]をクリックします。



    2. 説明セクションで、 アドインの名前、短い説明、および適切な言語の詳細な説明を指定します。 たとえば、前の手順で英語を選択した場合は、アプリケーション名、短い説明、詳細な説明を英語で入力します。



    3. スクリーンショットセクションで、 [ 基本]を選択して、少なくとも1つのスクリーンショットを送信します。 最大5つの画像を送信できます。 各スクリーンショットは、PNG、JPG、JPEG、またはGIF形式で、幅512ピクセル、高さ384ピクセルで、サイズが300 KB以下である必要があります。





  6. [ 次へ]、 [ ドラフトとして保存]または[ キャンセル]をクリックします。
  7. [ アクセスブロック]ページで、アプリケーションまたはアドインが表示および販売されない国および地域が存在する場合、それらを選択します。



    1. [ 特定の国または地域でアプリの購入をブロックする]チェックボックスをオンにします。
    2. [国または地域の選択 ]ボタンを押します。

      1. [ブロックする国と地域を選択してください ]ダイアログボックスで、アプリケーションまたはアドインの購入を禁止する国または地域を選択します。
      2. 必要な国または地域を選択したら 、[国または地域のブロック ]または[ キャンセル]ボタンをクリックします 。 特定の国または地域をブロックすると、その地域のユーザーはアプリケーションまたはアドインを購入して使用できなくなります。







  8. [ 次へ]、 [ ドラフトとして保存]または[ キャンセル]をクリックします。
  9. [ 価格設定]ページで、アドインの価格を入力します。 Webアプリケーションを送信する場合は、デフォルト値を選択したままにしてくださいこのアプリケーションは無料で提供されます（Webアプリケーションは常に無料です）。



    次のオプションのいずれかを選択します。



    1. このアプリは無料です（デフォルト） 。
    2. このアプリケーションは、1回限りの購入を目的としており、 ユーザーあたりの価格を選択します 。
    3. アプリケーションは有料サブスクリプションによって提供され 、 ユーザーごとの価格パラメーターの値を設定します。


    アドインの価格は、そのレベルに応じて設定されます。 言語ごとに価格を設定することはできません。 価格カテゴリのみが指定されています。



    有料または購読済みのアドオンについては、試用版を提供するかどうかを示します。 試用版を提供するには：



    • 試用版の入手可能性の下で、 [私のアプリケーションは試用版として利用可能です]ボックスをチェック
    • [ 試用版の期間]ドロップダウンリストから、試用期間の期間を選択します。
    • [ 試用ユーザーの数]ドロップダウンリストで、試用版を有効にするユーザーの数を指定します。



  10. [ ドラフトとして保存 ]または[ キャンセル]ボタンを押します 。
