Sailfish OSの開発：サンプルのメモ作成アプリケーションを使用した通知の操作

こんにちは この記事は、Sailfish OSモバイルプラットフォーム向けのアプリケーション開発に関する一連の記事の続きです。 今回は、ユーザーがメモを保存、タグ付け、画像、写真、リマインダーを追加し、Evernoteアカウントと同期できるようにするメモを取るアプリケーションに焦点を当てます。



アプリケーションのユーザーインターフェイスの詳細な説明から記事を開始し、 技術的な詳細に進みます。



アプリケーションの説明



アプリケーションのメイン画面は、ユーザーエントリのリストです。リストの各要素には、ノートのタイトル、説明（または説明が完全に収まらない場合はその一部）、および追加された場合の画像が表示されます。 リストアイテムには、メモを削除したり、編集用のダイアログを開いたりできるコンテキストメニューもあります。 リスト項目をクリックすると、このノートに関するすべての情報を表示する画面が開きます。

アプリケーションのユーザーインターフェイスについて詳しく説明しましょう。 アプリケーションのメイン画面は、ユーザーエントリのリストです。リストの各要素には、ノートのタイトル、説明（または説明が完全に収まらない場合はその一部）、および追加された場合の画像が表示されます。 リストアイテムには、メモを削除したり、編集用のダイアログを開いたりできるコンテキストメニューもあります。 リスト項目をクリックすると、このノートに関するすべての情報を表示する画面が開きます。







アプリケーションのメイン画面には、 PullDownMenu要素とPushUpMenu要素も含まれています 。 PullDownMenuには、新しいノートを追加するためのダイアログを開くメニュー項目が1つしかありません。 PushUpMenuには2つの項目があります。1つ目はすべてのタグのリストを含むウィンドウを開き、2つ目は設定ウィンドウです。







メモを追加/編集するためのダイアログには、タグ、タイトル、説明を入力するためのテキストフィールド、および画像、写真、アラームを追加するためのボタンが含まれています。 「画像の追加」ボタンをクリックすると、ダイアログが開き、ユーザーは画面に何かを描いたり書いたりして、この画像をメモに追加できます。 [写真を追加]ボタンを押すと、デバイスのカメラが開き、画面をクリックすると、画像のような写真がメモに追加されます。







リマインダーを追加するためのボタンをクリックするとダイアログが開き、標準コンポーネントのDatePickerDialogおよびTimePickerDialogを使用してリマインダーの日付と時刻を設定できます。









[タグ]画面は、メモに追加されたすべてのタグのリストです。 リスト内のタグをクリックすると、このタグでマークされたレコードのみを含む画面が表示されます。 この画面では、メモを使用したすべての操作（情報の表示、編集、削除、新しいメモの追加）を利用できます。







設定画面には「Evernote経由のログイン」項目が1つ含まれており、認証後に「Evernoteからログアウト」と「データの同期」という2つの項目に変更されます。 1つ目は、Evernoteアカウントからログアウトしてデータ同期を無効にすることを可能にし、2つ目は、同期プロセスを手動で開始することを可能にします。 また、データが変更されると同期が自動的に開始されます。







Nemo QMLプラグインの通知



この記事では、Sailfish OSでの通知の操作に焦点を合わせることにしました。 通知を処理するために、Sailfish SDKはNemo QMLプラグイン通知プラグインを提供します 。 プラグインには2つのクラスが含まれます。



  • QMLコード内に通知を作成するためのQMLクラス通知 。
  • C ++コード内で通知を作成するためのC ++ 通知クラス。


Notificationクラスを使用すると、D-Busを使用してLipstick Notification Managerと通信するために使用できる通知インスタンスを作成できます。 以前の記事の1つで、 D-Busの概要とその操作方法について既に説明しました。 まだ確認していない場合は、必ず確認してください。



通知は、いくつかのパラメーターを使用して生成されます。 主なものをリストします。



  • appIcon-通知自体と共に表示されるアプリケーションアイコンへのパス。
  • appName-通知アイコンに加えて、アプリケーションの名前も表示できます。
  • 概要 -通知パネルに表示される通知タイトル。
  • previewSummary-画面上部の通知バナーに表示される通知タイトル。
  • body-通知の「本文」、通知パネルに表示される説明。
  • previewBody-通知バナーに表示される通知の説明。

  • itemCount -1つのアイテムによって表示される通知の数。 たとえば、 itemCountが4の場合、1つの通知で最大4つの不在着信を表示できます。
  • タイムスタンプ -通知が関連付けられているイベントのタイムスタンプは、通知自体の作成には影響せず、通知が表示される時刻ではありません。
  • remoteActions-プロパティ「name」、「service」、「path」、「iface」、「method」、「displayName」、「icon」、および「arguments」を持つオブジェクトのリストは、作成された通知をクリックして可能なアクションを決定します。 remoteActionsについては、以下で詳しく説明します。


公式ドキュメントのすべての通知パラメーターについて読むことができます。QMLで通知を作成する例を以下に示します。 



Button { Notification { id: notification appName: "Example App" appIcon: "/usr/share/example-app/icon-l-application" summary: "Notification summary" body: "Notification body" previewSummary: "Notification preview summary" previewBody: "Notification preview body" itemCount: 5 timestamp: "2013-02-20 18:21:00" remoteActions: [{ "name": "default", "service": "com.example.service", "path": "/com/example/service", "iface": "com.example.service", "method": "trigger" "arguments": [ "argument 1" ] }] } onClicked: notification.publish() }


説明したボタンをクリックすると、 publish（）メソッドが呼び出されることがコードから理解しやすくなります。 publish（）メソッドは、通知を通知マネージャーに発行し、デバイス画面に表示します。



前述のように、通知に関連付けられたアクションをカスタマイズできます。 それ自体を示唆する例は、通知をクリックしてアプリケーションを開くことです。 通知はD-Busを介して機能するため、最初に必要なことは、独自のD-Busサービスを作成することです。 これを行うには、最初にプロジェクトルートにdbusディレクトリを追加し、次の内容を持つ* .service拡張子のファイルを作成します。 



 [D-BUS Service] Interface=/com/example/service Name=com.example.service Exec=/usr/bin/invoker --type=silica-qt5 --desktop-file=example.desktop -s /usr/bin/example


さらに混乱を避けるために、サービス名（ Nameパラメーター）とファイル自体の名前に同じ名前を使用することをお勧めします。 また、 Execパラメーターはプロジェクトの* .desktopファイルとデバイス上のアプリケーションへのパスを使用するという事実に注意してください。ここでは、「例」の代わりにプロジェクトの名前を使用する必要があります。



次に、* .proファイルにD-Busサービスへのパスを登録する必要があります。 



 ... dbus.files = dbus/com.example.service.service dbus.path = /usr/share/dbus-1/services/ INSTALLS += dbus ...


また、* .specファイルにも。 



 ... %files %{_datadir}/dbus-1/services ...


通知アクションをアプリケーションに関連付けるには、 DBusAdaptorを作成する必要があります。 DBusAdaptor -D-Busサービスと対話する機能を提供するオブジェクト。 



 DBusAdaptor { service: 'com.example.service' iface: 'com.example.service' path: '/com/example/service' xml: ' <interface name="com.example.service">\n' + ' <method name="trigger">\n' + ' <arg name="param" type="s" access="readwrite"/>\n"' + ' </method">\n' + ' </interface>\n' function trigger(param) { console.log('param:', param); __silica_applicationwindow_instance.activate(); } }


サービスおよびifaceプロパティは登録したD-Busサービスの名前であり、 pathプロパティはデバイスファイルシステム内のサービスオブジェクトへのパスです。 特に興味深いのはxmlプロパティです。 サービスの内容、つまり呼び出すことができるメソッドの名前とその引数を記述します。 ここでは、 トリガー（）関数をサービスメソッドとして使用します。このメソッドは、文字列を入力として受け取ってコンソールに表示し、 ApplicationWindowオブジェクトでactivate（）メソッドを呼び出してアプリケーションを開きます。



次に、作成した通知にアクションを関連付ける必要があります。 NotificationクラスのremoteActionsプロパティはこれに役立ちます。 



 Button { Notification { ... remoteActions: [{ "name": "default", "service": "com.example.service", "path": "/com/example/service", "iface": "com.example.service", "method": "trigger" "arguments": [ "argument 1" ] }] } onClicked: notification.publish() }


remoteActionsでは 、D-Busサービスとの通信のためのservice 、 path 、およびifaceパラメーター を記述します。 methodパラメーターはサービスメソッドの名前です。argumentsプロパティでは、メソッドのパラメーターのリストを渡します。 そしてそれだけです。 これで、通知をクリックすると、D-Busサービスメソッドが呼び出され、アプリケーションが開きます。



通知を処理する機能の1つは、アプリケーションが閉じられたときに通知を表示するには、D-Busに登録されたサービスを管理するアクティブなデーモンが必要になることです。 サービスを閉じると登録が解除されるため。 これはSailfish FAQにも書かれています。



アプリケーションで通知を操作する



アプリケーションで通知の処理を実装するために、C ++ Notificationクラスを使用しました。 アプリケーション内の通知はメモのタイトルと説明で構成され、それにリマインダが追加されるため、次のクラスプロパティにのみ関心があります： summary 、 body 、 previewSummaryおよびpreviewBody 。 もちろん、 publish（）メソッドにも興味があります。







通知を管理するために、2つのpublishNotification（）およびremoveNotification（）メソッドを含むNotificationManagerクラスを作成しました。 1つ目はリマインダーを作成して表示するために必要で、2つ目はリマインダーを削除するために必要です。



Notificationクラスは、通知を表示する時間を設定する機能を提供しないことに注意してください; publish（）メソッドは、通知（メソッド）が呼び出された瞬間に通知を表示します。 タイマー（ QTimerクラス）を使用して通知が表示される時間を制御することにより、この問題を解決しました。 また、リマインダー付きのメモが複数ある場合があるため、このようなタイマーもいくつかあるはずです。 したがって、 NotificationManagerクラスでハッシュ（ QHash ）が作成され、そのキーはデータベース内のノートIDで、値はQTimerです。 



 class NotificationManager : public QObject { Q_OBJECT public: explicit NotificationManager(QObject *parent = 0); Q_INVOKABLE void publishNotification(const int noteId, const QString &summary, const QString &body, QDateTime dateTime); Q_INVOKABLE void removeNotification(const int noteId); private: QHash<int, QTimer*> timers; };


publishNotification（）メソッドの実装をさらに詳しく検討しましょう。 引数として、データベース内のIDノート、通知のタイトルと説明、および通知を表示する日時を取ります。



まず、このメソッドは新しいタイマーを作成し、そのタイムアウト（）信号をラムダ関数で記述されたスロットに関連付けます。 ラムダ関数では、通知が作成および構成され、 publish（）メソッドが呼び出されます。 通知が表示された後、タイマーを停止します。 また、 publishNotification（）メソッドは、そのようなレコードIDを持つタイマーが既にハッシュに追加されているかどうかを確認し、追加されている場合はハッシュから削除します。 次に、タイマーを開始し、停止する時間（ミリ秒単位）を設定して、新しいタイマーをハッシュに追加します。 



 void NotificationManager::publishNotification(const int noteId, const QString &summary, const QString &body, QDateTime dateTime) { QTimer *timer = new QTimer(); connect(timer, &QTimer::timeout, [summary, body, timer](){ Notification notification; notification.setSummary(summary); notification.setBody(body); notification.setPreviewSummary(summary); notification.setPreviewBody(body); notification.publish(); timer->stop(); }); if (this->timers.contains(noteId)) removeNotification(noteId); timer->start(QDateTime::currentDateTime().secsTo(dateTime) * 1000); this->timers[noteId] = timer; }


removeNotification（）メソッドははるかに単純に見えます。 パラメーターとして、通知を削除するノートのIDのみを受け入れます。 このメソッドは、指定されたIDを持つタイマーが既にタイマー付きハッシュ内にあることを確認し、もしそうであれば、タイマーを停止してハッシュから削除します。 



 void NotificationManager::removeNotification(const int noteId) { if (this->timers.contains(noteId)) { this->timers.value(noteId)->stop(); this->timers.remove(noteId); } }


おわりに



その結果、画像、タグ、リマインダー付きのメモを保存し、それらをEvernoteと同期できる幅広い機能を備えたアプリケーションが作成されました。 このアプリは、Jolla HarborアプリストアでSailNotesという名前で公開されており、誰でもダウンロードできます。 アプリケーションのソースはGitHubで入手できます 。
