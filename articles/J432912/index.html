Googleでインターンシップを取得する方法

一週間前、 私たちは教育プログラムについて話しましたが、コメントでインターンシップと実際の経験の重要性を指摘しました。 理論的な知識は実践によって強化されなければならないため、これに反対することは不可能です。 この投稿では、学生の夏のインターンシップに関する一連の記事を開きます。男たちがどうやってそこに着くのか、彼らが何をするのか、なぜそれが良いのか。







最初の記事では、インタビューのすべての段階を正常に完了し、Googleでインターンシップを取得する方法を紹介します。

私はHSEのサンクトペテルブルクキャンパスの1年生で、アカデミーオブマシンラーニングの学士課程を卒業しました。 彼は学部課程でスポーツプログラミングに積極的に関与し、さまざまなハッカソンにも参加しました。 後者については、 こちら 、 こちら 、 こちらをご覧ください 。







インターンシップについて



始めるために、Googleのインターンシップが内部からどのように見えるかについて少しお話したいと思います。







Googleに来るすべての研修生はチームに結び付けられています。 これは、社外の人が何も聞いていない内部インフラストラクチャを開発しているチーム、または世界中の何百万人もの人々が使用している製品かもしれません。 そのような製品には、悪名高いYouTube、Googleドキュメントなどが含まれます。 これらのプロジェクトの開発には数十人または数百人の開発者が関与しているため、最終的にはその狭い部分に特化したチームになります。 たとえば、2018年の夏に、Googleドキュメントで作業し、テーブルを操作するための新しい機能を追加しました。







会社のインターンであるため、ホストと呼ばれるマネージャーがいます。 これは定期的なフルタイマーであり、それ自体が製品の開発に従事しています。 何かわからない、解決できない、または問題が発生した場合は、彼に連絡する必要があります。 通常、1対1の毎週の会議が予定されており、プロジェクトの作業の現在の状況について議論したり、まったく無関係なことについてチャットしたりできます。 さらに、ホストは、インターンシップ中に行った作業を評価する人々の1人です。 別のレビューアは、2人目の追加のレビューアによって評価されます。 そしてもちろん、彼らはあなたのためにすべてがうまくいくことに興味があります。







Googleはあなたに予防接種をしますが、確かにそうではありません。何かをする前にデザインドキュメントを書くのは良い習慣です。 知らない人にとっては、dzdokは既存の問題の本質とその解決策の詳細な技術的説明を記載した文書です。 製品全体といずれかの新しい機能の両方について、dizdockを作成できます。 このドキュメントを読んだ後、製品が考案された目的とその実装方法を理解できます。 また、多くの場合、プロジェクトの一部を実装するさまざまな方法を議論するエンジニア間の対話をコメントで見ることができます。 これにより、各決定が行われた理由を十分に理解できます。







インターンシップの特徴は、すばらしい内部開発ツールを使用する必要があることです。その中には多くのGoogleがあります。 Amazon、Nvidia、その他の有名なテクノロジー企業で以前働いていた多くの人々とチャットし、おしゃべりをしたことで、これらのツールはあなたが人生で出会うすべての中で最高のものになる可能性が高いと結論づけることができます。 たとえば、Google Code Searchというツールを使用すると、コードベース全体、コードの各行の変更履歴を表示できるだけでなく、Intellij Ideaなどの現代の開発環境で使用されているコードをナビゲートすることもできます。ただのブラウザ！ 同じ機能に関連するマイナス点は、Googleの外部ではこれらの同じツールを見逃すことです。







グッズに関しては、会社にはクールなオフィスがあり、彼らはよく食事をし、ジム、良い保険、その他の肉焼きを提供します。 ニューヨークオフィスの写真をここにいくつか残しておきます。















オファーを取得する方法？



復習



今こそ、もっと深刻なことについて話をする時です。インターンシップを取得する方法は？







これはGoogleについてではなく、一般的なケースでこれがどのように起こるかについてです。 Googleでの研修生の選択プロセスの機能については、以下に記述します。







企業へのインタビューのプロセスは次のようになります。







  1. インターンシップの申し込み
  2. Hackerrank / TripleByteクイズコンテスト
  3. スクリーニングインタビュー
  4. 最初の技術面接
  5. 2回目の技術面接
  6. オンサイトインタビュー


インターンシップの申し込み



明らかに、それはすべてインターンシップを取得するあなたの欲求から始まります。 これを行うには、会社のWebサイトでアンケートに記入してそれを表現する必要があります。 あなた（またはあなたの友人）がそこで働いている友人を持っているなら、あなたは彼らを乗り越えようとすることができます。 このオプションは、他の学生の膨大なストリームから目立つのに役立つため、推奨されます。 これが不可能な場合は、自分で食事をしてください。







「あなたはとてもクールですが、私たちは他の候補者を選びました」というフォームの内容の手紙をメールで受け取ったとき、あまり動揺しないようにしてください。 そして、ここにあなたのためのヒントがあります：











Hackerrank / TripleByteクイズコンテスト



採用担当者が履歴書を気に入った場合、1〜2週間以内に次のタスクを記載した手紙を受け取ります。 ほとんどの場合、割り当てられた時間内にアルゴリズムの問​​題を解決する必要があるHackerrankのコンテスト、またはアルゴリズム、ソフトウェア開発、低レベルシステムに関するさまざまな質問に答える必要があるTripleByte Quizに参加するよう求められます。 この段階は、選択プロセスの初期フィルターとして機能します。







スクリーニングインタビュー



テストが成功した場合、スクリーニングインタビューが待っています。そこでは、リクルーターとあなたの興味や会社がインターンに提供するプロジェクトについて話します。 興味を示し、以前の経験が会社の期待に応える場合、青信号が与えられます。 私の経験では、これはプロセス全体で最も予測不可能な場所であり、採用担当者に大きく依存しています。







これら3つのテストに成功した場合、主なチャンスはすでに遅れています。 それから、あなたにより依存している技術面接がありました。つまり、あなたは彼らの結果により影響を与えることができます。 そしてこれはいいです！







技術面接



次に、通常、Skypeまたはハングアウトを介して行われる技術面接が行われます。 しかし、追加のソフトウェアのインストールを必要とするよりエキゾチックなサービスが存在する場合があります。 したがって、すべてがコンピューターで機能することを確認してください。







技術面接の形式は、面接する立場に大きく依存します。 Software Engineering Internの位置について話している場合、おそらくいくつかのアルゴリズム上の問題が提供されます。その解決策は、たとえばcoderpad.ioなどのオンラインコードエディターでエンコードする必要があります。 また、オブジェクト指向設計について質問して、ソフトウェア設計にどれだけ慣れているかを確認することもできます。 たとえば、簡単なオンラインストアの設計を依頼される場合があります。 確かに、私はそのような仕事に出会ったことはありません。その解決策によって、このスキルを判断することは本当に可能です。 インタビューの最後に、おそらく質問する機会が与えられます。 これを真剣に受け止めることを強くお勧めします。なぜなら、プロジェクトへの関心を示すことができる質問のおかげで、トピックに関する能力を実証できるからです。 私は通常、潜在的な質問のリストを事前に準備します。







  • プロジェクトの作業はどのように調整されていますか？
  • 最近、どんな難しい仕事を解決しなければなりませんでしたか？
  • 最終製品に対する開発者の貢献は何ですか？
  • なぜこの会社で働きに行くことにしたのですか？


あなたは、将来一緒に働く人と常に面接を受けるわけではありません。 したがって、最新の質問は、会社全体で何が起こっているのかを知ることができます。 たとえば、私にとっては、最終製品に影響を与えることが重要です。







最初の面接が正常に完了すると、2回目の面接が提供されます。 最初から、インタビュアーおよびそれに応じてタスクによって区別されます。 形式は同じままです。 2回目のインタビューに合格した後、3回目のインタビューを提供できます。







オンサイトインタビュー



この時点までに拒否されていない場合、候補者が会社のオフィスでの面接に招待されると、オンサイト面接が待っています。 通常、複数の技術面接と1つの行動面接で構成されます。 行動面接では、プロジェクト、さまざまな状況でどのような決定をしたかなどについてマネージャーと話します。 つまり、インタビュアーはあなたの性格をよりよく理解し、あなたの経験をより詳細に理解しようとしています。 一部の企業では、オンサイトインタビューの代わりに3〜4回の技術面接を実施し、1つの行動のみをリモートで行うことを提案しています。







今ではリクルーターの応答を待つだけです。 すべてが順調に進んだ場合、待望のオファーのある手紙が必ず届きます。 申し出がない場合は、落胆しないでください。 企業は良い候補者を体系的に拒否します。 来年もインターンシップに応募してみてください。







インタビューコーディング



だから、ちょっと...まだインタビューをしていない。 全体のプロセスがどのように見えるかを見つけたので、今は面白くて便利に夏を過ごす機会を逃さないように、インタビューの準備をしなければなりません。







Codeforces 、 Topcoder、 Hackerrankなどのリソースがあります。 これらのサイトでは、多数のアルゴリズムの問​​題を見つけることができ、それらのソリューションを自動検証に送信できます。 それはすべて素晴らしいですが、スズメの発砲のようなものです。 これらのリソースの多くのタスクは、長いソリューション時間のために設計され、高度なアルゴリズムとデータ構造の知識を必要としますが、インタビューのタスクは通常それほど複雑ではなく、5〜20分間設計されます。 したがって、このケースでは、技術面接の準備ツールとして作成されたLeetCodeなどのリソースがはるかに適しています。 さまざまな複雑さの100〜200の問題を解決する場合、高い確率で、インタビュー中に問題が発生することはありません。 まともなものの中には、 Facebook Code Labもあります。この上で、セッションの継続時間（60分など）を選択できます。システムは、解くのに平均1時間もかからない一連のタスクをピックアップします。







さらに多くの人が、Cracking the Coding Interviewを読むことをお勧めします。 私自身はその一部のみを選択的に読みました。 しかし、学校時代に多くのアルゴリズムの問​​題を解決したことは注目に値します。 それにもかかわらず、そのような経験を持っていなかった人は、少なくともこの本を熟読すべきです。







また、あなたの人生で外国企業との技術面接がほとんど行われていない場合は、いくつかのトライアルを経験することをお勧めします。 しかし、多ければ多いほど良いです。 これにより、面接中に自信を持ち、緊張を和らげることができます。 試験面接は、 Prampで手配できます。







行動面接



先ほど述べたように、行動面接では、面接官はあなたの経験についてもっと学び、あなたの性格を理解しようとします。 あなたが優れた開発者であるが、チームで働くことができない場合はどうなりますか？ 多くの人が満足しないのではないかと心配しています。 たとえば、「あなたの弱点は何ですか？」という質問があります。この種の質問に加えて、重要な役割を果たしたプロジェクト、解決しなければならない問題について話し合うように求められます。 技術面接の最初の数分で、これについて尋ねられることもあります。 このようなインタビューの準備方法については、「コーディングインタビューのクラッキング」の章で詳しく説明されています。







グーグル



インターンを選択するプロセスが一般的なケースでどのように見えるか、インタビューの準備方法が明確になったので、Googleのケースでこれがどのように機能するかについて話します。







利用可能なインターンシップのリストはこちらにあります 。 夏のインターンシップに参加する予定がある場合は、9月に応募を開始する必要があります。







就職面接



ここでは、プロセスは少し変わっています。 スクリーニングインタビューと2つの技術面接があります。 それらをよく見せれば、プロジェクト検索の段階に行くことができます。 現在のスキルをすべて示すかなり長いアンケートに記入する必要があります。また、プロジェクトのトピックやインターンシップを希望する場所に関する好みを表明する必要があります。







このフォームに十分な注意を払って記入することが非常に重要です！ プロジェクトで人を探している潜在的なホストは、利用可能なインターンを見て、彼らが好きな候補者との会話を手配します。 場所、キーワード、アンケートのチェックマークで生徒をフィルタリングし、インタビューのポイントで並べ替えることができます。







会話中、インタビュアーは作業対象のプロジェクトについて話し、また候補者の経験についても学びます。 あなたはあなたのホストになる人と通信しているので、作業プロセスが実際にどのように見えるかを知る絶好の機会があります。 面接の後、プロジェクトの印象を記したリクルーターに手紙を書きます。 プロジェクトが気に入ったが、インタビュアーがあなたを気に入った場合、オファーがあなたを待っています。 それ以外の場合は、後続の呼び出しが予想されます。これは2-3-4個であり、おそらくそれ以上ではありません。 あなたがよくインタビューされたとしても、プロジェクト検索段階で単一のチームがあなたを選択しなかった（またはおそらく誰もあなたと話しさえしなかった）場合でも、悲しいことに、あなたは申し出なしで残されることを明確にすることは価値があります。







アメリカかヨーロッパか？



とりわけ、インターンシップが行われる場所を決定する必要があります。 私はアメリカとEMEAの間で選択肢がありました。 そして、ここでいくつかの機能について知ることが重要です。 たとえば、アメリカに入国するのはもっと難しいと感じています。 まず、アルゴリズムの問​​題を解決する必要がある90分間の追加のコンテストと、キャラクターを明らかにしようとする別の15分間の調査を行う必要があります。 第二に、私の経験と私の友人の経験では、チーム検索フェーズに興味を持つ可能性は低くなります。 たとえば、2017年に、私は1つの会話しか行わなかった後、チームは別の候補者を選択しましたが、オファーを受け取りませんでした。 彼らがヨーロッパに給餌している間、4-5のプロジェクトがありました。 2018年、彼らは1月にチームを見つけましたが、かなり遅れています。 彼らはニューヨークで働いていましたが、私は彼らのプロジェクトが好きで、同意しました。







ご覧のとおり、アメリカでは事態はもう少し複雑です。 しかし、私はヨーロッパよりもっとそこに行きたかった。 加えて、米国ではより多くを支払います。

これらの2つのオプションは、現在のプロジェクトを正常に完了した場合に使用できます。 これが最初のインターンシップでない場合、面接なしでフルタイムのポジションを提供することさえあります。
