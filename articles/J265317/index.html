HPとMicrosoft-進歩の共生

HPは従来、非常に幅広く詳細な製品を提供しています。 特に、これはサーバーの生産にも当てはまります。これについては、この記事で説明します。 ユーザーに完璧なソリューションを提供するために、さまざまなHP ProLiantシステムがさまざまなMicrosoftオペレーティングシステムとどのように適合するかを見ていきます。



いわゆる「エントリレベル」レベルのサーバーは、ML110およびDL120モデルで表されます。 「ML」および「DL」という表記は、システムの構成を意味します。ML-タイプタワー、DL-ラック。 番号付けシステムは、モデルのレベルとシステムのメーカーを示します。たとえば、この例に示すように、番号の末尾の「0」はIntelシステムの生産を意味します（システムはAMDに基づいて作成されます-番号の末尾の「5」）。 そのため、エントリーレベルのサーバーはタワー内のML110 / ML115とラック形式のDL120エンクロージャーです。 さらに、製品ラインに沿って移動すると、「中間」クラスML330 / ML350またはDL360 / DL380のタワーおよびラックモジュールに昇格できます。 行の「上部」部分は、DL580などの大きなボックスで表されます。 数が増えると、システム自体も大きくなり、BLまたはブレード環境になります。 これは非常に技術的で高性能なシステムです。



それでも、特定のユーザーに適した製品を選択するには、システム全体だけでなく、個々の製品の提供内容を検討する必要があります。



だから、ML110とDL120。 これらのシステムは両方とも、Microsoft Windows Server 2012 R2 Foundationと優れた組み合わせであり、「私の最初のサーバー」のような環境として知られている素晴らしいエントリーレベルのサーバーを作成します。







問題は、ML110とDL120は、最大8 GBと数テラバイトのディスク容量をサポートするシングルプロセッサIntelシステムであるということです（ML110には4 TBがあります）。 要求に応じて利用可能な冗長電源や冗長ファンなどのオプション、およびECCエラー修正（パリティ）を備えたメモリにより、エントリーレベルのサーバーシステムとモバイルサービスを備えた優れたPCを実現できます。



ML330は、Microsoft Windows Small Business Server 2011と完全に統合されるモデルです。







この組み合わせの結果、効率が最適化された優れたサーバーができました。 ML330には、独自のシステムアーキテクチャが組み込まれています。 これはデュアルプロセッサシステムですが、各プロセッサにはシステムボードに一定数のDIMMスロットがあります。 したがって、ユーザーは、現時点で必要なものを取得し、たとえば、1つのプロセッサーと必要なメモリーを購入します。 さらに、ニーズの拡大に伴い、2番目のプロセッサを購入することもできます。これにより、メモリスロットを追加してパフォーマンスを向上させることもできます。 これらの手段はすべて、コンポーネントの購入コストを削減するのに役立ちます。 システムは、両方のプロセッサーを搭載した8 TBのディスク容量、140 GBを超えるメモリをサポートします。 Inside Controlソフトウェアなど、他の特徴的な機能があります。



次に、Microsoft Windows Server 2012 R2 Standardと完全に組み合わされ、消費電力の面を含めてパフォーマンスと効率の両方で大きな利点を持っているML350およびDL380モデルを検討します。







両方のシステムは、Intelベースのデュアルプロセッサです。 最新のメモリ、最大速度および容量を備えたこれらのシステムは、190 GBを超えるメモリをサポートしています。 また、柔軟性もあります。小型および大型の両方のフォームファクタでハードドライブをインストールできます。 また、Inside Controlソフトウェアは最大のシステムパフォーマンスを実現するのに役立ち、非常に高いワークロードを起動し、最も重要なこととして仮想化することができます。



結論として、HP製品はMicrosoftオペレーティングシステムと良好に相互作用し、これらの製品の範囲は非常に広いと結論付けることができます。 これにより、ユーザーは必要なものを、そして必要なときにそれを選択できます。 これは最も深い理解における普遍性です。
