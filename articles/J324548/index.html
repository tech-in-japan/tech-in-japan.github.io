.NETプログラミング言語の開発戦略

Microsoftで11年以上にわたって.NET言語を開発してきたMads Torgersenは、チームの指針となる原則の説明を共有し、各言語の開発について決定を下しました。 私は床をMadsに渡します。







11年以上にわたり、私はMicrosoftで.NETプログラミング言語を使用してきました。この間、言語を開発するために、開発者の利益を考慮に入れようと試みました。 オープンソース革命のおかげで（.NETプログラミング言語だけでなく、.NET スタック全体に関しても）、開発者との対話は根本的に改善されました。これは、私がより良い選択をするのに役立ちました。 ただし、これらの決定をどのように行ったのかを正確に共有できるとは限りませんでした。 プログラミング言語の開発戦略、つまり各.NET言語の概念とその開発計画について話し合っています。



この記事では、各言語に関する決定を行う際に遵守する原則を詳細に強調したいと思います。 これは一般的な概要であり、開発計画ではありません。



C＃



C＃は何百万人もの人々によって使用されています。


Stack Overflowの開発者への調査によると、C＃は2016年に最も人気のあるプログラミング言語の1つとして認識されました。 Javaと、もちろんJavaScriptだけが人気を集めました（SQLはプログラミング言語とは見なされませんが、これについては議論しません）。 これらの結果は、さまざまな言語コミュニティでのStack Overflowサイトの人気のレベルに影響される可能性がありますが、C＃は間違いなく最も一般的なプログラミング言語の1つです。 Unityエンジンのゲーム、Xamarinのモバイルアプリケーション、ASP.NETのWebアプリケーション、Windowsのビジネスアプリケーション、 Azure上のLinuxの.NET Coreマイクロサービス、およびAmazon Web Services（AWS）プラットフォームなど、さまざまな方向で広く使用されています。 。



また興味深いのは、前回の調査で最も好まれた10の言語とテクノロジーに関するStack Overflowの別の調査で、C＃とPythonだけが含まれていたという事実です。



長い年月を経て、人々はC＃を愛し続けています！ なんで？ この間ずっと、私たちはプラグマティズムと私たち自身のスタイルでそれを開発し、新しい問題を解決し、言語の本質を変えませんでした。 C＃は、強力で効率的で使いやすい言語と見なされています。 互いになく、未来がないため、通常は.NETに関連付けられます。



C＃開発戦略



開発者の増大するニーズに対応し、主導的な地位を維持するために、C＃の開発を続けます。 特別な「精神」を維持しようとしながら、積極的にイノベーションを導入します。 開発者の多様な視聴者を考慮して、私たちは生産性の向上と言語の改善に特に注意を払います。これはほとんどの人にとって有用です。 一貫した一貫性を確保するために、設計上の決定を完全に制御しながら、エコシステムを拡大し、C＃言語の開発方向を決定する上でのその役割を強化し続けます。


C＃の次の各バージョンには大幅な改善が含まれています：C＃2.0、LINQおよびC＃3.0の多数の機能改善、C＃4.0の動的型、C＃5.0の非同期、C＃6.0の小さくても便利な関数のセット。 新しいユースケースに適応するために多くの機能が追加されました。C＃5.0以降、接続されたデバイスとサービス、そのような接続の速度、およびそれらを通過するデータストリームの処理に特別な注意が払われます。 C＃7.0の新しいバージョンも例外ではありません。その主な革新は、タプルとパターンマッチングです。これにより、コード内のデータフローと制御の作業を変換および簡素化できます。



C＃6.0から、言語開発者からのメモをGitHubで共有し始めました。 コミュニティとの対話の中で新しい機能が次第に登場し、Microsoftが "貢献者"の1つとして認識されているという事実が出てくることがあります。



C＃言語開発プロセスの説明はGitHubのdotnet / csharplangセクションで公開されており、議論はcsharplangメーリングリストで行われます 。



ビジュアルベーシック



Visual Basicは、数十万人が使用しています。


それらのほとんどはWinFormsを使用してWindows用のビジネスアプリケーションを作成し、一部はASP.NET Webフォームを使用してWebサイトを作成するのが得意です。 それらのほとんどもC＃を使用しています。 多くの開発者にとって、これは彼らが取り組んでいるさまざまなプロジェクトの言語要件によるものです。 ただし、Visual Basicの基本的なユースケースを超えて、Visual Basicを使用できる場合でも、多くのユーザーはためらうことなくC＃に移行します。 C＃のエコシステム、例、およびコミュニティは、より広いことが多いです。



Visual Studioには興味深い傾向があります。新しい開発者の間でVisual Basicを使用する割合は、すべての開発者の2倍です。 これは、Visual Basicが引き続きプログラミングに精通している高品質でアクセス可能な言語の役割を果たし、プラットフォームと開発の両方に精通している人に適していることを示唆しています。



Stack Overflowの回答者は、Visual Basicに対して否定的な反応を示しました。言語のリストの中で、ユーザーが代替品を見つけることを好む最初の場所になりました。 これは懐疑的な見方をするべきだと思います。 まず第一に、これらの人々には、前進したいと非難できないVB6開発者が含まれる場合があります。 さらに、スタックオーバーフローはVisual Basic開発者の主な生息地ではないため、調査に参加することを決めた場合は、別のプログラミング言語のファンであるスタックオーバーフローを既に訪れている可能性が高いです。 最後に、これらの結果を詳しく見ると、ほとんどのVisual Basic開発者はC＃に切り替えることを好みます。これは、Visual Basicを放棄するのではなく、1つの言語で.NETの開発に集中したいことを示している可能性があります。



上記のすべてを考えると、そのような統計は考えさせるものです。 Visual Basicを使用している人の多くは、言語の遅れを感じたり、言語の未来を疑ったりしているようです。 機会を利用してこの問題に対処しましょう！



Visual Basic開発戦略



Visual Basicをシンプルで手頃な言語としてサポートします。 .NET環境の主要言語の1つであるために必要なすべてを行います。たとえば、C＃で新しい機能を追加するときにAPIが変更された場合、Visual BasicでこのAPIを使用するのは自然に見えるはずです。 多くの開発者がVisual BasicとC＃の両方を使用していることを考えると、クロス言語ツールを使用するという原則を保持します。 Visual Basicおよびユースケースの一般的な領域での更新プログラムのリリースに焦点を当てます。


これは、C＃とVisual Basicが共通のコースをたどった2010年に行った共同開発戦略からの脱却です。 急速に進化するC＃に従うことは、Visual Basicの戦略を失うだけでなく、シンプルさとアクセシビリティに悪影響を及ぼします。



Visual Studio 2015では、Visual Basic 14とC＃6.0は基本的に一緒に開発を続け、NULL句（？。）、NameOfなどの演算子が多く改善されました。 同時に、それらのそれぞれに固有の欠点のいくつかは、両方の言語で削除されました。 たとえば、Visual Basicでは、複数行の文字列リテラル、暗黙的な行継続後のコメントなどが表示されました。 同時に、関数とプロパティは、C＃の式の本体（式の実体を持つメンバー）、およびVisual Basicにはない他の機能に登場しました。



VB 15では、C＃7.0の新機能の一部が導入されます。 タプルは、それ自体で役立つだけでなく、API署名で使用されるため、互換性を提供するためにも役立ちます。 ただし、Visual Basic 15では、is式、out-variable変数、ローカル言語関数など、言語の可読性を損なうなど、より害のある機能はありません。



Visual Basic言語開発プロセスの説明はGitHubのdotnet / vblangセクションで公開されています。fディスカッションはvblangメーリングリストで行われます 。



Visual Basicの開発戦略については、 この記事で詳しく説明します 。



F＃



F＃は何万人もの人々によって使用されています。


急速に発展しており、さらなる成長の見通しを示しています。 普遍的な言語であるため、かなり幅広いアプリケーションがありますが、クラウドとWebサービス、ツールとユーティリティ、分析ワークロード、データ処理での使用が特に重要です。



F＃は、お気に入りのプログラミング言語のランキングで高い位置を占めています 。 このリストにある他のほとんどの言語と比較して、F＃には素晴らしいツールキットがあるという事実にもかかわらず、C＃やVisual Basicよりも経験が少ないです。 多くの革新が言語の進歩に大きく貢献しています。 .NET環境（Microsoftの内外両方）の開発者は、F＃を考慮に入れて習得する必要のあるプログラミング言語と考えています。



F＃には非常に興味のあるコミュニティがあり、特に完全にオープンな言語開発プロセスにより、開発と継続的な改善に非常に重要な役割を果たしています。 これは、オープンソースの.NET言語の中で絶対的なリーダーであり、その開発への重要な貢献はマイクロソフトの外部で行われています。



F＃開発戦略



F＃の開発へのコミュニティの積極的な参加をサポートおよび奨励し、開発者コミュニティのさらなる貢献に必要なインフラストラクチャとツールの作成を継続します。 F＃を市場で最高のツールを備えた関数型プログラミング言語にし、ツールの言語と機能を改善し、コミュニティへの参加の障害を取り除き、C＃とVisual Basicの間のギャップを減らすために問題のある問題を解決します。 C＃言語の新しい機能が利用可能になると、F＃との互換性が確保されます。 F＃は、コミュニティにとって重要なプラットフォームを引き続きターゲットにします。


MLファミリの強力な機能的遺産と.NETプラットフォームとの深い統合に基づいて、F＃は多くの革新的な機能を獲得しました。 タイププロバイダー、プロアクティブパターン、および計算式は、この言語を一歩踏み出して学習する準備ができている人に素晴らしい機会を提供します。 F＃は、あらゆるレベルでの採用の障害を取り除くことを最も必要としています。



F＃4.1には、Roslynエディターワークスペースの抽象化との統合、.NET Coreおよび.NET Standardへのフォーカス、およびコンパイラエラーメッセージの改善により、Visual Studioのツールが大幅に改善されています。 Visual Studioツールキットの最適化の重要な部分（改善されたエラーメッセージを含む）は、F＃コミュニティの積極的な参加の結果です。 将来的には、F＃コミュニティとマイクロソフトの他のスペシャリストの両方と協力して、F＃にクラス最高のツールを提供し、市場で最高のツールを備えた関数型プログラミング言語にする予定です。



Fit言語の説明は、 GitHubとRFC で提供されているリポジトリで見つけることができます 。
