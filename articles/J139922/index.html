Ubuntu ShellをXfceに拡張する

Linux Ubuntuの革新的なUnityデスクトップ環境での作業の問題は誰もが知っています。 ユーザーインターフェイスは大幅な変更と実験を受けており、他のシステムやアセンブリへの次の「移民の波」につながっています。 Ubuntu 10.04 LTS（Long Time Support）のUnity環境での一定期間の作業と、Unityの現実に慣れるのに非常に成功した後、11.10の新しいビルド環境に戻る必要がありました。 ここでは、サポートされるデスクトップテーマの数が減少し、システム設定が何らかの形で途切れてしまったため、驚きが予想されました。 トピックのリストには、閉じるボタンが適切に配置されたウィンドウがなく、Unity環境での存在がどんなに進歩的であっても、他のシェルの検索につながりました。 1年以上前に、安定した肯定的なレビューを備えたXfce 4.8作業環境のバージョンがリリースされたため、Unityの代わりに試してみることは論理的でした。 別の方法として、LXDE環境がテストされ、Xfceに匹敵する明るさで、テーブルを使用した作業の編成において非常によく似ています。



デスクトップを選択した後にLinuxをインストールすると、Ubuntuに基づいて、Unityを使用せずにLubuntuとXubuntuという名前の両方のデスクトップ環境を使用する2つのクローンがあります。 選択の決定が後で行われた場合、判明したとおり、システムの柔軟性により、1つおよび他の代替環境を順番に実行できます。 標準のインストール手順の後、すべてのオプションを試すことができます。 ただし、これはニュースではありませんが、この記事で説明したい主なことは、Xfce環境設定の柔軟性です。これにより、不器用なUnityコントロールスタブがはるかに遅れ、LXDE設定の電源がバイパスされます。 、そしてもちろん、誰かが完全に満足しています。 私にとっては、驚くほど柔軟なXfceの選択を好みました。



どちらの環境も多くの点でWindowsデスクトップに似ていますが、柔軟性とセットアップの容易さには大きな違いがあります。 たとえば、Windows 7でプロセッサとグラフィックスカードの処理能力を高める必要があるAeroインターフェイスは、Xfceでより多くのバリエーションとはるかに少ない労力で実装され、GTKで実装され、UNIXシステムの簡単で高速なシェルとして位置付けられています。



前述のように、さまざまなスタイルにふけることのないハードコアUnityウィンドウを実現する試みとして、 Ubuntu 11.04のWindows 7テーマも使用されました。 11.04でWindowsウィンドウの外観（右側の閉じるボタン）に達すると、トピック11.10で左側の閉じるボタンが表示されますが、残りのウィンドウはかなりきれいになり、エアロ効果があります:)。 確かに、設定に触れると牧歌に違反し、ボタンの外観は典型的なUbuntスタイルになりました。 復元するには、インストールスクリプトを再度実行する必要がありました。 テーマ管理の不便さの合計により、私はこのユーザビリティレスキュープランを放棄し、XfceとLXDEのインストールに進みました。





LXDE （Lightweight X11 Desktop Environment）は、比較的厳格な一連の機能を備えた高速指向の環境です。 ウィンドウマネージャーはOpenbox 、ファイルマネージャーはPCManFM 、プログラムメニューのパネルはLXPanel 、すべてはWindowsに非常に似ていますが、パネルの黒い色を「愛して」おり、多くの設定が好きではありません。 この2つの状況が、この環境での環境の設定に関するさらなる作業の欲求を止めました。 誰かがウィンドウの他の部分にある黒と対照的な白を好むだろうが、私は個人的に黒のパネルを取り除く簡単な方法の欠如を好まなかった。 順番に、より有望な機会がある別の環境がありました。

タスクバーは、Xfceのように高さと幅（ウィンドウの幅の割合）を調整できるため、ウィンドウの水平または垂直の端にある占有スペースを保存してから増やすことができます。



Xfce （「XForms Common Environment」を意味する文字の翻訳不可能な組み合わせ）は、XFormsのない長い歴史を持つアプリケーションであり、コンポーネントのモジュール性の概念の一貫した実装により、設定とコンポーネントのパワーを組み合わせて増加させながら、コードサイズを軽くします。 このセットには、いくつかの便利なプログラムが含まれています。灰色の背景にあるGnomeのターミナルと、いくつかの調整。 Orageカレンダーリマインダー Thunarファイルマネージャー。 オンボードのオンスクリーンキーボード。 Xfconfパラメーターエディター。 Xfce 4.8に関する最近の記事でそれらについて詳しく読んでください。



Xfceカーネルの作成者自身は、そのようなコンポーネントを区別します（環境の説明からコピーされた半分）：



ウィンドウマネージャー （xfwm4）。 画面上のウィンドウの配置を制御します。



パネル （xfce4-panel）。 プログラムを起動するためのボタン、ウィンドウ管理、アプリケーションメニュー、ジョブの切り替えなど。 これは、プログラムの名前またはアイコンからメニュー全体を構築できる、かなり複雑な構造要素のセットです。 Mac / Unity / Win7のボタンのシンプルなラインは2次元に変わりますが、いくつかの無駄な設定が必要です。 最初は2つのパネルが提供されていますが、好きなだけ作成でき、設定は非常に柔軟です。 それらのいずれにも、開いているウィンドウや閉じているウィンドウのボタンなど、通常の「トレイ」要素を配置できます。



デスクトップ （xfdesktop）。 デスクトップの色または背景画像を設定します。 アプリケーションメニュー、デバイスアイコン、最小化されたアプリケーション、起動ボタン、またはフォルダーの内容を表示します。



ファイルマネージャー （thunar）。

セッションマネージャー （xfce4-session）。 起動時にセッションを復元し、Xfce環境からコンピューターをオフにすることができます。 開いているすべてのウィンドウとパネルを元の形式に復元します。



システムのセットアップ （xfce4-settings）。 デスクトップ、モニター、キーボード、およびマウスの外観をカスタマイズします。 速度と明るさの割引なしで、いくつかの設定ウィンドウは理想的には作業環境を構成します。 多くの設定がありますが、少し考えずによく考えた方がましです（Unityのように）。



アプリケーションを検索します （xfce4-appfinder）。 システムにインストールされているアプリケーションをカテゴリ別に表示します。これにより、それらを簡単に見つけて起動できます。 （一方で、更新のチェックと構成はそれに起因する可能性があります。）



ユーティリティとスクリプト （xfce-utils）。 起動時のスクリプト、実行ダイアログ、およびXfceバージョン情報。



構成マネージャー （xfconf）。 D-Busベースの構成ストレージサービス。



Xfceは、 開発者が独自のデスクトップアプリケーションを作成できるようにするいくつかのライブラリを提供する開発プラットフォームでもあります。

さらに、シェルはすばやく最適に調整されました。下部と上部の2つの不透明なパネルが3つの半透明になり、それぞれが不完全な幅であるため、常に全幅のウィンドウに干渉しません。 （スクリーンショットは仮想マシンのウィンドウで取得されるため、ウィンドウのサイズはこのような異常な形状になります）。 左側のパネルには、ブラウザー、テキストエディター（下部）のプログラムセットが手動で入力されていました。 ウィンドウを閉じるためのボタンは、マウス操作から最も遠い左上隅に移動されました（より便利に思えた）。 この構成の柔軟性はどのように達成されますか？







ログアウトメニューもすばらしいです。ボタンは非常に「正方形」になっており、タッチスクリーン上の指やデスクトップ上のマウスなど、あらゆる種類のマニピュレーターに適しています。 また、Unity以前のWindowsXP-2003の小さなWindowsボタンやオプションのリストよりも便利です。

ここに通常の設定をリストしても意味がありません-ユーザーに表示されます。 私たちは、ウィンドウ環境の選択の有効性に興味を持ち、説得する異常なものを示します。 3つの設定ウィンドウを開き、画面にどれだけ収まるか、最も興味深い場所、つまり半透明の設定を表示します。 焦点が合っていないウィンドウは半透明であることがわかります。 これは、ウィンドウマネージャーの[効果]タブで提供されます。 パネルの半透明性-ウィンドウ「パネル」、そしてそこに-色の選択。 [要素]タブで、パネルメニュー項目が選択され、メニュー項目として表示されます（左）。 設定に応じて、メニューも半透明であることがわかります。 また、アクティブウィンドウのタイトルバーも半透明です（ほとんどAeroのように、ぼかしがありません）。







半透明性は必ずしも便利ではありません-開いている多くのウィンドウを操作するときは、通常の外観の複数のウィンドウが必要な場合があります。 しかし、印象的なチューニング機能は、比較的少数のチューニングコンポーネントから組み立てられています。 そして、そのような印象は、設定ウィンドウの各要素から残っています。これは、インターフェースの人間工学が長年にわたって機能していることを示しています。 または少なくとも非常に徹底的に。



設定には、数十種類のウィンドウタイトルがあります。 スクリーンショットに表示されるWindowsのようなキャプションは、スクリーンショットで強調表示されている[スタイル]タブのWallisテーマです。 残りの大きなリストには、多くの奇妙なテーマが含まれています。 すべてを表示するのは難しく、見やすく、正しいものを選択するのも簡単です。 別の優れたトピック選択リストは、外観-スタイルタブにあります。 彼は、ウィンドウ要素の色を担当しています。







LXDE環境は、その背景に異なる印象を与えます。 そのアプローチもバランスが取れており、考え抜かれています。 しかし、思いやりは別の種類のものです。 彼女は、次のように言っています。「それは簡単にする方が良いですが、より複雑でセットアップが難しいよりも便利です。」 したがって、インターフェイスは、Xfceに似た同様に配置された（手動でも）パネルよりも少し鈍く、構成できません。
