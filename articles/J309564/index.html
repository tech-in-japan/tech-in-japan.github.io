一元化されたバックアップポリシー管理

Microsoft Azure開発チームからのもう1つの良いニュースです。 Azure ARMポータルで既にバックアップサービスを使用している読者の1人が、バックアップポリシーの管理が非常に難しいことに気付いているかもしれません。 実際、 Recovery Services Vaultに新しいサーバーを追加するときにのみ、既存のポリシーを選択するか、新しいポリシーを作成する機会が与えられました。









今、すべてが変更されました。 本日、Microsoft Azureチームは 、Azure Resource Managerポータルの別のスナップインでバックアップポリシーを操作する機能を追加しました 。



Recovery Servicesでバックアップポリシーを設定しようとした人は、これは新しいサーバーを追加するときにのみ実行できることを覚えています。









これに関する最も不便な点は、利用可能なすべてのバックアップポリシーをポータルで直接表示することができなかったことです。 PowerShellを使用してこれを実行できることは明らかですが、ポータルにこの機能が存在しないことは依然として奇妙でした。



今、すべてが間違っています。 本日より、 Recovery Services Vaultプロパティでバックアップポリシースナップインを直接開き、すべてのポリシーを確認する機会があります。









ここで、既存のポリシーの設定を適宜変更したり、新しいポリシーを追加したりできます。



Azure Backup の監視およびアラート機能の最近の実装と組み合わせて、このサービスの開発速度でのみ喜ぶことができます。
