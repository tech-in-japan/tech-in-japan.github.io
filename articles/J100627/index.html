Hyper-V v2.1用のLinux ICのベータ版を残しました





最後に、Linuxシステム用のHyper-Vの統合コンポーネントがリリースされました。 このニュースは、仮想環境で動作する異種インフラストラクチャを厳格なガイダンスの下で管理している管理者を喜ばせます。 このリリースでは、ベータ版で約束されている機能が追加され、仮想Linuxシステムでの作業がさらに便利になりました。



このリリースは、システム管理者の国際的な日に私たちの「仮想チーム」からの一種の贈り物であるという事実のためにも記憶されます。 当社のハイパーバイザーは、仮想サーバー時代からエンタープライズLinuxシステムをサポートしており、引き続きサポートし、コラボレーションを継続的に改善していきます。



バージョン2.1のイノベーション：

- 合成デバイスのドライバー ：統合コンポーネントは、合成ネットワークアダプターと合成ディスクサブシステムのサポートが改善されました。

-Hyper-VのFastpathダウンロードのサポート ：VMBusとの直接の相互作用により、ブートディスクはVirtualization Service Client（VSC）を使用してロードを高速化します。

- 時刻の同期 ：仮想マシンのハイパーバイザーのクロックとの組織的なクロック同期。

- 統合シャットダウン ：Linuxを実行している仮想マシンは、Hyper-VマネージャーまたはSystem Center Virtual Machine Managerを使用して正しく「シャットダウン」できます。

- 対称型マルチプロセッシングのサポート：サポートされているLinuxディストリビューションは、1つの仮想マシンで最大4つの仮想プロセッサを使用できます...

- ハートビート ：ハイパーバイザーがゲストLinuxシステムの状態とアクティビティを監視できるようにします。

- プラグインタイムソース ：プラグインクロックモジュール。これにより、仮想マシンの時刻をより正確に同期できます。



従来、すべての機能はRed Hat Enterprise Linux 5.2-5.6およびSUSE Linux Enterprise Server 10 SP3 to 11を使用して保証されています
