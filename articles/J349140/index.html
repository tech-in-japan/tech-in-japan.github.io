第2部では、パイプラインモードで一連の関数を使用できるようにする、関数をさまざまな方法で組み合わせ、最後に構成関数とパイプ関数で終わることについて説明しました。



その投稿では、引数を1つだけとる単純な関数パイプラインを取り上げました。 しかし、複数の引数を取る関数を使用する場合はどうでしょうか？



たとえば、書籍オブジェクトのコレクションがあり、特定の年に出版されたすべての書籍の名前を検索するとします。 Ramda反復関数のみを使用してこの問題を解決しましょう。 



const publishedInYear = (book, year) => book.year === year const titlesForYear = (books, year) => { const selected = filter(book => publishedInYear(book, year), books) return map(book => book.title, selected) }


フィルターとマップを組み合わせてパイプラインにすると良いでしょうが、フィルターとマップは2つの引数を取るため、これを行う方法はわかりません。



また、フィルターで矢印関数を使用する必要がない場合にも役立ちます。 最初にこの問題を解決しましょう。これにより、パイプラインを作成するときに使用できることがわかります。



高階関数



このシリーズの最初の部分では、関数を一流の構造として説明しました。 同様の関数をパラメーターとして他の関数に渡し、他の関数の結果として返すことができます。 前者の多くを行いましたが、まだ後者を適用していません。



他の関数を取得または返す関数は、「高階関数」とも呼ばれます。



上記の例では、矢印関数をフィルターに渡します：book => publishedInYear（book、year）。これを削除するとよいでしょう。 これを行うには、本を受け取り、その本が適切な年に発行された場合にtrueを返す関数が必要です。 ただし、年数を渡して柔軟にする必要もあります。



この問題を解決する方法は、別の関数を返す関数を作成することです。 ここで何が起こっているのかを理解できるように、通常の関数構文でこれを記述しますが、矢印構文を使用した短いバージョンに進みます。 



 //    function publishedInYear(year) { return function(book) { return book.year === year } } //  : const publishedInYear = year => book => book.year === year


これはpublishedInYear関数の新しいバージョンです。矢印関数を除外することでフィルター呼び出しを書き換えることができます。 



 const publishedInYear = year => book => book.year === year const titlesForYear = (books, year) => { const selected = filter(publishedInYear(year), books) return map(book => book.title, selected) }


さて、filterを呼び出すと、publishedInYear（year）がすぐに呼び出され、本を取得する関数を返します。これは、まさにフィルターに必要なものです。



部分的に適用可能な機能



必要に応じて、この方法でいくつかの引数を使用して関数を書き換えることができますが、すべての関数がこのように動作するわけではありません。 また、通常の方法で複数の引数を持つ関数を使用することもできます。



たとえば、特定の年にその本が出版されたことを確認したいだけの他のコードがある場合、publishedInYear（book、2012）のように書きたいのですが、このように書くことはできません。 代わりに、publishedInYear（2012）（book）のように少し書きます。 読みにくく、煩わしいです。



幸いなことに、 Ramdaはこれを支援する2つの関数、 partialおよびpartialRightを提供します。



これらの2つの関数を使用すると、必要な引数が少ない別の関数を呼び出すことができます。 これらは両方とも、残りの引数を取る新しい関数を返し、すべての引数が提供された場合でも元の関数を呼び出します。



partialとpartialRightの違いは、指定した引数が元の関数に必要な左端または右端の引数に置き換えられることです。



元の例に戻って、publishedInYearを書き換える代わりにこれらの関数を使用してみましょう。 年を指定するだけでよく、これが右端の引数なので、partialRightを使用する必要があります。 



 const publishedInYear = (book, year) => book.year === year const titlesForYear = (books, year) => { const selected = filter(partialRight(publishedInYear, [year]), books) return map(book => book.title, selected) }


publishedInYearに（book、year）ではなく（year、book）を指定して記述した場合、partialRightではなくpartialを使用します。



partialとpartialRightに渡す引数は、そのうちの1つだけを渡す場合でも、常に配列内にある必要があることに注意してください。 これを何回忘れて紛らわしいエラーメッセージを受け取ったかは言えません。 



 First argument to _arity must be a non-negative integer no greater than ten


カレー



どこでもpartialとpartialRightを使用する必要があるため、冗長性と疲労が生じます。 しかし、一連の単一引数関数として多くの引数を持つ関数を呼び出す必要性は長い間悪いです。



幸いなことに、Ramdaはカリー化というソリューションを提供します。



カレーは、関数型プログラミングのもう1つの基本概念です。 技術的には、カリー化された関数は常に、先ほど申し上げた一連の単一引数関数です。 純粋な関数型言語では、構文は、いくつかの引数を使用して関数を呼び出す場合と違いがないように見えます。



しかし、RamdaはJavaScriptライブラリであり、JavaScriptには一連の単一引数関数を呼び出すための優れた構文がないため、著者はカリー化の従来の定義をわずかに緩和しました。



Ramdaでは、カリー化された関数は引数のサブセットを使用して呼び出すことができ、残りの引数が受信されるのを待つ新しい関数を返します。 すべての引数でカリー化された関数を呼び出すと、通常の関数呼び出しがトリガーされます。



カリー化された関数は、2つの世界のベストと考えることができます。すべての引数で呼び出すことができ、それらは機能します。 または、引数の一部でそれらを呼び出すことができ、部分的なアプリケーションモードで動作します。



カレーでは関数の呼び出し方法を決定し、次に何を行う必要があるかを決定する必要があるため、この柔軟性はパフォーマンスにわずかな影響を与えることに注意してください。 一般に、複数の場所で部分バインディングを使用する必要があることがわかった場合、関数カリー化を使用します。



カリブリング機能をpublishedInYear関数に適用してみましょう。 カリー化は常にパーシャル関数を使用しているかのように機能し、partialRightのようなバージョンを使用する方法はないことに注意してください。 次に、このトピックについてもう少し説明しますが、ここでは、1年が始まるように、publishedInYearの引数を逆順に変更します。 



 const publishedInYear = curry((year, book) => book.year === year) const titlesForYear = (books, year) => { const selected = filter(publishedInYear(year), books) return map(book => book.title, selected) }


これで、publishedInYearを年に1回だけ呼び出して、本を取得して元の関数を呼び出す関数を取得できます。 ただし、いらいらすることなく、publishedInYear（2012、book）を通常の方法で呼び出すことができます（構文。2つの世界のベスト！



引数の順序



カリー化を機能させるには、引数の順序を逆にする必要があることに注意してください。 これは関数型プログラミングでは非常に一般的であるため、ほぼすべてのRamda関数は、最終的に動作するデータが引数リストの最後になるように記述されています。



最初のパラメーターは、操作の構成と考えることができます。 したがって、publishedInYearの場合、yearパラメーターは構成（何を探していますか？）であり、bookパラメーターはデータ（どこを探していますか）です。



反復関数を使用したこの例は既に見ました。 このプログラミングスタイルが簡単になるため、全員がコレクションを最後の引数として使用しました。



間違った順序の引数



publishedInYear関数の引数の順序を変更しない場合はどうなりますか？ カレーの性質からどのように利益を得ることができますか？



Ramdaはいくつかのオプションを提供します。



フリップ



最初のオプションはフリップです。 Flipは、2つ以上の引数を持つ関数を取り、同じ引数を取る新しい関数を返しますが、最初の2つの引数を交換します。 ほとんどの場合、2つの引数を持つ関数で使用されますが、アプリケーションではより一般的です。



フリップを使用して、publishedInYearの元の引数の順序に戻すことができます。 



 const publishedInYear = curry((book, year) => book.year === year) const titlesForYear = (books, year) => { const selected = filter(flip(publishedInYear)(year), books) return map(book => book.title, selected) }


ほとんどの場合、より便利な引数の順序を使用することをお勧めしますが、たとえば、制御していない関数を使用する必要がある場合、フリップは便利なオプションです。



プレースホルダー



より一般的なオプションは、プレースホルダー引数（__）です。



3つの引数を持つカリー化された関数があり、最初と最後の引数を渡し、将来のために中間の引数を残したい場合はどうなりますか？ 中間の引数にプレースホルダーを使用できます： 



 const threeArgs = curry((a, b, c) => { /* ... */ }) const middleArgumentLater = threeArgs('value for a', __, 'value for c')


また、呼び出しで代替を複数回使用することもできます。 たとえば、中間の引数のみを渡したい場合はどうでしょうか？ 



 const threeArgs = curry((a, b, c) => { /* ... */ }) const middleArgumentOnly = threeArgs(__, 'value for b', __)


必要に応じて、フリップの代わりにプレースホルダースタイルを使用できます。 



 const publishedInYear = curry((book, year) => book.year === year) const titlesForYear = (books, year) => { const selected = filter(publishedInYear(__, year), books) return map(book => book.title, selected) }


このバージョンの方が読みやすいと思いますが、publishedInYearの「反転」バージョンを繰り返し使用する必要がある場合は、flipを使用して追加の関数を定義し、どこでも引き続き使用できます。 今後の投稿でいくつかの例を見るかもしれません。



__は、任意の関数でpartial、partialRightおよびflipが機能する場合にのみ、カリー化された関数で機能することに注意してください。 通常の関数で__を使用する必要がある場合は、これを行う前にいつでもカレーでラップできます。



コンベアをやりましょう



呼び出しを移動して、パイプライン内でフィルター処理とマッピングを行う方法を見てみましょう。 これはコードの現在の状態で、publishedInYearの便利な引数の順序があります。 



 const publishedInYear = curry((year, book) => book.year === year) const titlesForYear = (books, year) => { const selected = filter(publishedInYear(year), books) return map(book => book.title, selected) }


以前の投稿でパイプと作曲について学びましたが、この研究から最大限の利益を得るためには、別の情報を学ぶ必要があります。



最後の情報は次のとおりです。ほぼすべてのRamda関数はデフォルトでカリー化されています。 これにはフィルターとマップが含まれます。 したがって、フィルター（publishedInYear（year））は完全に適合し、map（book => book.title）のように、その後書籍が渡されるのを待つだけの新しい関数を返します。



そして今、パイプラインを書くことができます： 



 const publishedInYear = curry((year, book) => book.year === year) const titlesForYear = (books, year) => pipe( filter(publishedInYear(year)), map(book => book.title) )(books)


一歩前進して、titlesForYearの引数を反転させて、最後にRamdaデータ契約に準拠してみましょう。 関数をカリー化して、後続のパイプラインで使用できるようにすることもできます。 



 const publishedInYear = curry((year, book) => book.year === year) const titlesForYear = curry((year, books) => pipe( filter(publishedInYear(year)), map(book => book.title) )(books) )
