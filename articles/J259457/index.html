jQueryの開始方法

jQuery JavaScriptライブラリは9年前にリリースされ、それ以来、このオープンソースプロジェクトはWeb開発の世界に大きく貢献しています。 jQueryの起源を振り返って見るのは確かに興味深いことです。



2015年4月、jQueryの作成者であるJohn Resig は、 2006年1月の最初のjQueryバージョンを公開しました 。 この投稿では、JohnはjQueryがどのように作成されたかを思い出してコードを補足しました。



ジョンのコメントを見るとわかるいくつかの事実を次に示します。



1. jQueryはもともとJSelectと呼ばれていました。



ただし、 JSelect.comドメイン名が使用され、 JQuery.comは使用されませんでした。 名前の最初の文字は少し後に小文字で書かれ始め、最初は「 jQuery 」でした。



出所



2. jQueryはCreative Commonsの下でライセンスされました。



クリエイティブコモンズ （CC） ライセンスは通常、ストック写真、PSDテンプレート、アイコンセットなどのクリエイティブな作品に関連付けられています。 しかし、CCはオープンソースには適していませんでした。 さらに、この時点でより適切なオープンソースライセンスがすでに存在していました。



さらに、 Creative Commonsのスタッフでさえ、オープンソースソフトウェアのライセンスをMITまたはGPLに基づいて推奨しています。 また、 オープンソースのアイデアを促進する組織であるOpen Source Initiativeは、Creative Commonsライセンスの下でのオープンソースプロジェクトの公開に反対しました。 これらすべてにもかかわらず、2006年にジョンはこのライセンスの下でjQueryの最初のバージョンを公開しました。 JQueryは、 CC Attribution-ShareAlikeライセンスの使用は悪い選択であったことを認めています。 まもなく、プロジェクトはMITのライセンスを変更しました。



出所



3. jQueryはバージョン管理システムなしで作成されました。



今日、 バージョン管理システム （VCS）を使用せずにリリースされる深刻なオープンソースプロジェクトを想像することは不可能です。 しかし、当時、jQueryはプロジェクトにVCSを使用することすら考えていませんでした！ 公平を期して、大規模プロジェクトでもVCSを使用しなかったことに注意してください。 jQueryは、最初の貢献者の出現によってのみバージョン管理システム（SVNでした）に切り替えました。



出所



4. jQueryという名前は別のプロジェクトから「盗まれた」。



「jQuery」という名前は、JavaのSQLライブラリでも使用されていました。 しかし、John Rezigは、Java-jQueryの作成者が彼に連絡したときにのみ、名前の競合について知りました。 ジョンは、この混乱のせいで自分が動揺したことを思い出します。



出所



5. jQueryがニューヨークで公開されました。



JQueryは、2006年1月にニューヨークで開催されたBarCampカンファレンスで公開されました。 BarCampは、インターネットおよびオープンソーステクノロジーの問題について議論したテクノロジー会議です。



出所



6. jQueryは、他のオープンソースプロジェクトの影響を強く受けています。



コメントの中で、ジョンはjQuery開発に最も大きな影響を与えたいくつかのプロジェクトに言及しています。

  • プロトタイプjavascriptフレームワーク
  • moo.fx
  • XPath
  • Behaviour.js


これらすべてのプロジェクトのうち、 Prototypeのみがまだアクティブです。 他のプロジェクトは現在開発されていません。 XPathは1999年にW3Cの推奨ステータスになり、HTMLやCSSとは異なり、それ以降変更されていません。



7.最初のjQueryプラグインはjQuery用のJSONでした。



2006年1月に作成されたプラグインは、JSONデータを操作する簡単な方法を提供しました。 また、プラグインは、プラグインを使用した機能の拡張性のために、jQueryに大きな将来性があることを示しました。 将来、jQueryが成功した理由の1つは、サードパーティ開発者からの多数のプラグインでした。



出所



8. jSLintは、品質管理のためにjQueryで使用されました。



John Rezigは、コードに関する現代の解説で、自分の意見では十分ではなかった場所を指摘しました。 この問題は、JSコードを分析するためのツールであるJSLintを使用してさらに解決されました。



出所



9. jQueryには最初はAjaxサポートがありませんでした。



非同期Webアプリケーションは、2000年代初期から中期に流行していました。 今日、多くのWebアプリケーションは非同期で動作し、誰も驚かないでしょう。 しかし、それは革新的なアイデアであり、多くのWeb開発者が急いでこの列車の時流に乗っていました。 私たちは皆、ページの完全なリロードを避け、必要な部分のみを更新したかったのです。 驚くべきことに、jQueryは最初のバージョンではAjaxをサポートしていませんでした。



出所



10. jQueryコードは、オプションである場合は中括弧を使用しませんでした。



JavaScriptでブレースを必要としない場合でも、ブロック構造にブレースを使用することをお勧めします。 中括弧を使用すると、特にチームで作業している場合にエラーを防ぐことができます。 上記で説明したJSLintは 、中括弧がないことを誤りと見なします。 ただし、jQueryの最初のバージョンでは、Johnはオプションの中括弧を省略することを好みました。 「余分な括弧は本当に好きではありませんでした。このスタイルのコードは私たち全員を苦しめ、将来多くの論理的エラーを引き起こしました。」



したがって、jQueryの最初のバージョンでは、次のコードを見つけることができます。 

if ( !b ) for ( var j in a ) this.style[j] = a[j]; else this.style[a] = b;




中括弧では、このコードは次のように記述する必要があります。 

 if ( !b ) { for ( var j in a ) { this.style[j] = a[j]; } } else { this.style[a] = b; }




出所



著者注 ：かつては、オプションの中括弧の使用を拒否することも好きでした。 そのようなコードは読みやすいと思いました。 さらに、中括弧のないコードは、その時に登場した新しいクールなWebフレームワークの構文に似ていました。



11. 2006 jQuery APIは引き続きサポートされています。



.css（）、. toggle（）、. show（）、および.hide（）などのお気に入りのjQueryメソッドは、最初のバージョンにあり、APIで引き続きサポートされています。 そして、これらの9年間で、古いバージョンとの互換性は壊れていません。 「jQuery 2006を使用するコードを使用して、最新のjQueryをそれに接続しても、それが機能する可能性は十分にあります」とJohn氏は言います。

jQueryはWeb開発を容易にします。



最初はjQueryの構文に惹かれました。 私は彼がいかに簡潔で率直であるかをどのように考えたかを覚えています。

$（）関数のおかげで、DOMツリーの要素の操作には数行しかかかりませんでした。 選択した要素を保存し、オブジェクトをループして、イベントをオブジェクト内の各要素にバインドするためにオブジェクトを開始する必要はありませんでした。 DOM要素を選択し、それらを任意の数のメソッドにバインドするだけです。 そして、ほんの数行しかかかりません。 この機能が最初のバージョンからjQueryにあったとは信じられません。



jQueryが最初に登場したときのことを覚えています。 Prototype + moo.fxを使用したと思います。 それとも道場だったかもしれません。 または、 Prototype + script.aculo.usかもしれません。 または、それらをすべて一緒に使用したかもしれません。 もちろん、ある時点で私はMooToolsに落ち着きました。 当時、私はMooToolsが大好きでした。 私も彼についての本を書きました。 （私の本の2人のバイヤー、 ママとパパにとって、あなたは素晴らしいです。ありがとう。いつも私を支えてくれました。 ）しかし、jQueryを使うと 、すべてがずっと速くなりました。 jQueryは、Protoype、Dojo、GWTと比較して単純な構文と概念を持っているため、経験の少ない開発者を雇うことができたので、JQueryも役に立ちました。



最新のWebプロジェクトの開発は、多くのサードパーティの依存関係を結びつける方向に向かっています。 jQueryはこの傾向を認識しており、多数のサードパーティライブラリでのみ改善されています。 Johnのコメントを読んで、ライブラリの使いやすさがプロジェクトの最初の日からjQuery哲学の基礎であることが明らかになりました。



たとえば、次のようなコメントを見つけることができます。

  • 「非常にシンプルなコードを書く機会を与えたかった。 私はそれについて情熱的でした。」
  • 「私はいつも.wrap（）メソッドが大好きでした。 最初から書くとき、このことは本当に苦痛を引き起こします、そして、もちろん、単一の方法の形でそれを持っていることは非常に便利です。




jQueryは、優れたドキュメントの重要性を強調した数少ないオープンソースプロジェクトの1つでした。 当時、オープンソースプロジェクトには親しみやすいドキュメントはありませんでした;これはオープンソースコードの文化に対応していませんでした。 信じられていました：「あなたはプログラマです。 あなたはそれを理解します。」



2000年代半ばの開発者向け のオープンソースプロジェクト。 そして、彼は開発者の利便性について考えましたか？ ほんと ？ 珍しかった。 ユーザーフレンドリーなjQuery哲学は、私のような多くの通常のWeb開発者を引き付けました。 そして、今日の成功したオープンソースプロジェクト-Bootstrap 、 WordPress 、 Sass 、 Chrome （組み込みの開発者ツールもあります）を見ると、開発者の生活を簡素化するという考えに基づいています。



jQueryは、現代のWebサイトの構築を支援するツール以上のものであり、Web開発の数か月を節約しました。 JQueryの最大の成果は、何百万ものサイトで使用されていることではありません。 これは単なる副産物です。 私にとって、jQueryで最も重要なことは、小さなライブラリ（わずか83 Kb）がオープンソースの文化とWeb開発業界に根本的な変化をもたらしたことです。 ジョンが何を成し遂げたいと思っているのか、まったく考えていません。 しかし、これは彼が控えめなJavaScriptライブラリをBarCampで発表してから9年後のことです。
