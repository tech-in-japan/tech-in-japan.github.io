Zend Framework 2.0.0beta3がリリースされました！

本日、3回目のベータリリースがリリースされました。 開発者によると、製品が成熟するまで各新リリースに新機能が追加され、その後開発が凍結され、リリース候補がリリースされます。 6週間ごとに新しいバージョンをリリースする予定です。



革新と変更のリスト：

  • 構成コンポーネントのリファクタリング
  • すべての設定はZend \ Config \ Reader名前空間に移動されました。
  • Zend \ Config \ Factoryが追加され、構成ファイルから配列を取得しやすくなりました。
  • 既存の構成形式は、追加の構成ファイルのインポートをサポートするようになりました。
  • すべての処理定数は、Zend \ Config \ Processor名前空間に移行されました。


新しいビューレイヤー

  • 新しいサブコンポーネントには、Zend \ View \ Model、Zend \ View \ Resolver、Zend \ View \ Renderer、およびZend \ View \ Strategyが含まれます。
  • Zend_ViewコンポーネントはZend \ View \ Renderer \ PhpRendererに移動され、より柔軟に使用できるようにその責任をサブコンポーネントに分散するように書き直されました。
  • 選択的なテンプレートレンダリングをサポートする新しいZend \ View \ Viewクラス。
  • MVCは、ビュー、エラーページなどの作成など、標準コンポーネントの使用を簡素化します。


書き換えられたDBレイヤー

新しいAgileZenコンポーネント

  • 完全なAgileZen APIサポート
  • framework.zend.com/zf2/boardを使用するように設計


PHP 5.4サポート

  • PHP 5.4のバージョンで作業しているときのすべてのエラーを見つけて修正しました


現在のベータリリースは、Zend \ GData、Zend \ Navigation、Zend \ Session、Zend \ Service \ StrikeIron、Zend \ Service \ Technorati、Zend \ Service \ Twitter、Zend \ Http \ Header \ Accept *、Zend \などのコンポーネントにも影響しましたLdap、Zend \ OAuth。 2番目のベータのリリース以降の200以上のエラーを修正しました。



出所
