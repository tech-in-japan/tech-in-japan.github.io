Dell Active Fabric Manager 2.5：それが必要な理由と最新バージョンの新機能

5月上旬、DellはActive Fabric Manager 2.5の新しいバージョンをリリースしました。 Habréでは、この製品に関するレビューはまだ行われていません。そのため、なぜ必要なのか、新バージョンの特徴は何かを説明します。



Active Fabricとは何ですか

この概念は、仮想化されたデータセンターとプライベートクラウドの問題を解決するために開発されました。 大規模なデータセンターのノード数は非常に急速に増加しています。また、同じレベル（東西）のノード間で送信されるトラフィックの量も増加しています。 アクティブファブリックアプローチは、最適かつ経済的で簡単にスケーラブルな方法でノード間の通信を構築するために開発されました。





この目的のために、DellはDell Active Fabricと呼ばれるネットワーク工場を組織するための一連のスイッチを発売しています。 これらには、1、10、または40 GbEインターフェースを備えた固定構成スイッチ、およびDell PowerEdge M1000eブレードシャーシ用のブレードスイッチが含まれます。 このスイッチセットとアクティブファブリックの概念に従うことにより、データセンターネットワークの従来のアーキテクチャをフラットなものに変えることができます。これにより、管理が大幅に簡素化され、運用コストが削減されます。また、データセンターを迅速かつ簡単に拡張できます。





アクティブファブリックベースのネットワークは、モジュラーシャーシ形式の従来のネットワーク機器に基づくネットワークと比較して、平均で30〜70％経済的であり、運用コストを30％以上削減します（ Gartnerレポート-Enterprise Data Center NetworkのRightsizingを参照） 。



Active Fabric Managerの機能

このアプローチに基づいて、デルはデータファブリックスイッチングネットワークを設計、展開、および管理するActive Fabric Manager（AFM）をリリースしました。

このソフトウェアソリューションにより、これらすべての問題を単一のコンソールで解決できます。 さらに、既製のネットワークアーキテクチャテンプレート（カスタマイズ可能）と高度な自動化により、データセンタースイッチングネットワークの手動設計、管理、構築に関連する問題を解決する時間が86％短縮されます。





したがって、デルでは、自動化された建物およびネットワーク管理機能の導入により、SDN（ソフトウェア定義ネットワーク）アプローチを部分的に実装しています。

AFMソフトウェア製品は、視覚的な環境でネットワークを設計および構成するネットワーク管理者を提供します。これにより、プロセス全体が大幅に簡素化および高速化されます。 ネットワーク設計の段階で、AFMはアクセスレベルのポート数とそのタイプ、工場オプション（レイヤー2またはレイヤー3）を要求し、ネットワークデザインテンプレートのいずれかを選択することを提案します。





AFMには、実際の企業でテストおよび使用されている工場テンプレートが何百も事前にインストールされています。 その中には、レイヤー3のスパインリーフファクトリー、iSCSI、FC、FCoEプロトコルを使用する統合LAN / SAN環境、および仮想リンクトランキング（VLT）テクノロジーを備えたレイヤー2ファクトリーがあります。工場で。





選択したテンプレートに基づいて、AFMは最適なトポロジを自動的に計算し、スイッチポートを相互に接続するためのスキームを提供し（下図を参照）、手動設計中にしばしば発生するエラーと欠点を排除します。





AFMを使用すると、開発されたネットワーク設計に基づいて、数分でネットワークを展開できます。 設計段階が完了すると、AFMはデバイスを自動的に構成し、管理者に微調整のためのツール（使用されるVLAN番号、IPアドレス、ルーティングプロトコル）を提供します。 したがって、管理者は、単一のCLIコマンドを入力しなくても、AFMユーザーインターフェイスを介して基本的なタスクを解決できます。





その後、スイッチをラックに取り付け、ポートを交差させ、スイッチの制御ポートをアクティブファブリックマネージャーソフトウェアで仮想マシンにアクセスできる管理ネットワークに接続し、電源をオンにします。 DHCPおよびTFTPプロトコルを使用して、スイッチは構成ファイルと現在のオペレーティングシステムファイルを自動的に受信し、ロードします。 AFMは、設計の設計とスイッチの現在のポート相互接続性が一致することを確認し、準拠していない場合に管理者に通知します。



AFMソリューションには、ネットワーク操作の毎日の監視、ソフトウェアのアップグレード、個々のスイッチのバックアップと再起動、ネットワーク管理者の注意が必要なネットワーク緊急イベントの追跡などのためのツールが含まれています。

つまり、実際には、AFMには、データセンタースイッチングネットワークの存在のライフサイクルの各段階に対応した完全なツールセットがあります。

同時に、ソフトウェア製品は、データが「進む」パスの外側で機能します。 AFMを備えたサーバーに障害が発生した場合、AFMが管理するネットワークは引き続き機能します。 ただし、人間の介入なしで問題に対処するために、DellはAFMに落下保護を装備するようにしました。

AFM 2.5の新バージョンには、DellのOpenManage Network Manager、OpenStack Neutronプラグイン、VMware vSphere Distributed Switch（VDS）など、多数の高レベルアプリケーションと統合するためのAPIも含まれています。
