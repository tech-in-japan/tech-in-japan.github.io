Visual Studio用の20の無料アドオン

Visual Studio 2010用に約900個のアドオンが既に作成されています。 それらの多くは、日常的なタスクの解決と利便性の追加を容易にします。 以下は、仕事に役立つと思われる20の人気アドオンです。



01：生産性向上ツール





実際に導入する必要のないアドオン。 クイック検索、コードナビゲーションの改善、Solution Navigatorの追加、新しいファイルナビゲーションインターフェイス、新しいアドオンオプションメニュー、参照アセンブリの簡単な検索、1つのコマンドで特定のテンプレートを使用したプロジェクトの作成、自動ブラケット、現在の編集行の強調表示、誤ったタブ、列の行、構文の強調表示への追加の修正-これは、この追加が可能なことの不完全なリストです。

アドオンページ



02：VSCommands 2010





IDEにしっかりと根ざした非常に人気のある別のアドオン。 このアドオンには、画像のプレビュー、ランダムなドラッグアンドドロップの排除、要素のグループ化、ソースファイルの検索、プロジェクトファイルとソリューションの編集など、ソリューションエクスプローラー用の非常に多くの追加機能があります。 さらに、出力ウィンドウの行の色の強調表示、デバッガーとプロジェクトアセンブリの操作の改善に注意する必要があります。

アドオンページ

04：インデントガイド





コードに垂直ルーラーを追加するシンプルだが、効果的なアドオン。 Productivity Power Toolsにも同様の機能がありますが、誰かが別に必要とする場合があります。

アドオンページ





05：JScriptエディター拡張機能





Jscriptコードでの作業を改善するアドオン。 改善点には、ペア括弧の強調表示、折りたたみ領域の自動作成、選択した単語の強調表示、コードのコメントで指定されたパラメーターのインテリジェンス、およびその他の改善が含まれます。

アドオンページ





06：正規表現エディター





正規表現を作成するために不可欠なツール。 優れた機能のうち、RegExp構文の強調表示とペアの括弧、構文のヒント、および正規表現のテストに注意する必要があります。 アドオンはコードエディターに統合されています。

アドオンページ





07：ミニファイアー





会話名Minifierのアドオンを使用すると、一般的なClosureおよびYUI Compressorサービスを使用してjsおよびcssファイルを圧縮できます。

08：devColor





CSSファイルで数値のカラー値を操作するための小さいながらも非常に便利な追加機能。 エディターで適切な色のコードを強調します。

コード内の選択された単語のすべての出現を強調表示するシンプルだが、それでも便利な追加。

13：メインメニューを隠す





目の前のコードのみを表示する場合、このアドオンはVisual Studioのメインメニューを非表示にするのに役立ちます。 Altボタンを押すだけで元に戻すことができます。

コードのメトリックを学習できるアドオン。 この機能がVisual Studioの古いエディションのみに含まれていることは秘密ではありませんが、現在、Professionalエディションの所有者であっても、コードメトリックスパワーツールによって準備されたメトリックスを便利なモードで確認できます。

15：Visual Studio 2010のエラスティックタブストップ





「エラスティックタブストップ」モードを有効にできる非常に興味深いアドオン-複数行の位置に応じたコードブロックの動的な移動。

16：画像の挿入





アドオンを使用すると、コードに画像を直接挿入できます！ コードにイラストを提供できるようになりました。これにより、実装の特定の重要なポイントに関するコメントを改善できます。

17：ウルトラファインド





Visual Studioの標準の検索ダイアログを置き換える。 クラス、メソッドの検索、テンプレートおよび拡張子による検索からのファイルの除外、結果の強調表示ができます。 とても速い。 Windows Searchテクノロジに基づいているという点で興味深い。

18：斜体コメント





このアドオンをインストールした後のコードコメントは、斜体フォントで表示されます。 些細なことのように思えますが、多くの人にとって非常に楽しいものです。

19：WoVSクイック追加リファレンス





キーワードとその外部アセンブリ内の存在に基づいてコードに参照を挿入するアドオン。 とても快適です！

20：CleanProject





送信する前に、コンパイル済みのバイナリ、テスト結果、その他の不要なファイルからプロジェクトをクリーンアップします。 コンパイルされたソリューションのみを送信する必要がある場合、これはかけがえのないものであり、ディレクトリをウォークスルーし、これらのファイルを手動で削除することに関連する手間を排除します。
