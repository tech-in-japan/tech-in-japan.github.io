認知とMISISは、学校のオリンピックを開催しました

こんにちは、Habr !!




NITU "MISiS"とCognitive Technologiesは、9年生から11年生までのプログラミングにおける全ロシア通信オリンピアードの結果をまとめました。 オリンピックは10月20日に開催され、ACM ICPCの規則に従って開催されました。 彼女の目標は、スポーツプログラミングのMISiSチームでのその後の選考のために、新しい才能を探すことでした。 オリンピアードの主催者によると、1871人が参加し、そのうちの191人が問題解決に成功したという参加証書を受け取りました。 そして、最も才能のある33人が2014年1月3日から10日にかけて冬のプログラミングキャンプに招待され、その枠組みでフルタイムのオリンピックツアーが開催されます。 郊外の別荘にあります。 さらに、主催者は宿泊と食事に関連する費用を負担します。 貴重な賞に加えて、受賞者は新しいCognitive TechnologiesとMISiSプログラミングチームに参加する機会を得ます。

/

オリンピアード組織委員会の代表者は、提示されたタスクのレベルは、ある意味で最適であると報告しました。 ちなみに、これはオリンピックでは必ずしもそうではありません。 1つまたは2つのタスクが「デッド」であることが起こります。つまり、誰もそれらを解決できません。 これは誰にとっても良くありません。 私たちの場合、参加者が克服しなかったであろう単一のタスクはありませんでした。 一方、すべての問題を解決できる人は一人もいなかったため、タスクは単純ではありませんでした。 最良の結果は、7つの完了したタスクでした。

その結果、オリンピアードの勝者は、ヤロスラブリ地方のミハイル・イワノフ（リセウム2号、ルイビンスク）とサマラ地方のパベル・グラモヴィッチ（リセウムNo. 67、トリアッティ）でした。 6つの問題が解決された2番目の場所は、ニキータスレズキン（A.A.スルコフにちなんで命名された28番学校）、ルイビンスクが撮影しました。 ウリヤノフスク州ディミトロフグラードのアレクサンダー・マルケロフ（市体育館第13号）は3位だった。

オリンピアードにはロシア全土の都市とベラルーシの小学生が参加し、彼らは勝者の一人になれました。

ヤロスラブリ地域、タンボフ地域、クラスノヤルスク地域、ウリヤノフスク地域、サマラ地域などの地域に注目する価値があります。 これらの地域は最大の活動を示しました。 彼らは最大数の参加者を提示し、問題解決において最高の結果を示しました。



オリンピアードの参加者の決定を確認するために、 ejudge.ru自動チェックシステムが使用されました 。 テストシステムは、プログラミング言語C、C ++、Pascal、Java、Pythonで書かれた決定を下しました。 合計11のタスクが提示されましたが、そのうち2つのタスクは比較的単純でした。 たとえば、リトルヴォロディアは、加算と乗算の方法を学習するだけで、驚くべきことを発見しました。2+ 2 = 2 * 2 = 4ですが、2 + 3 = 5は2 * 3 = 6とは異なります。数学的な偏りのある学校の10年生の生徒である彼は、「2-2」特性の一般化を定式化しました。 このセットの数の合計がそれらの積に等しい場合、彼はN個の自然数のセットを正しいと呼びました。 あなたの仕事は、109を超えない与えられた数の正しいセットを見つけることです。

そして、オリンピアードについて受賞者が書いたものは次のとおりです。「そもそも、私は英語で家庭教師をしていたので、オリンピアードに30分ほど遅れました。 家に着くと、私はすぐにラップトップに急ぎました。 脱ぐことなくオンにしました。 彼は問題を順番に解決し始めました。 （タスク番号は英語のアルファベットの最初の文字に対応していました）。 彼女のトピックは私のお気に入りなので、タスクDの後、すぐに問題Hの状態を読み始めました。 私の意見では、最初の3つのタスクはかなり楽でしたが、残りのことを考えなければなりませんでした。 一般に、すべてのタスクは非常に興味深く、解決可能です。 すでに「完了」ですべてのタスクを実行できましたが、タスクJの代わりに最後の1時間でFの解決を開始したことに気分を害しました。 私はすぐにそれを作る方法を考えました。 結局のところ、それは正しかったのであり、「完了」に関するこのタスクは私の最初の試みから渡されました。



また、グラフの理論、動的計画法、数学、さらには幾何学など、問題のオリンピックで提示されたトピックの非常に広い地理にも注目したいと思います。 オリンピアードの勝者であるミハイル・イワノフは、（主催者への著者のコメントである）あなたに特別な感謝を述べます。



3人の指導者の1人でもあるPavel Gramovichは、このようにオリンピアードについて次のように説明しています。 私にとって最初の4つの問題は非常に簡単でしたが、難易度は急激に高まりました。 タスクは非常に興味深いものでした。 私は7つの問題を解決しましたが、残りは単に十分な時間を持っていませんでした。 これは私のせいです。 私は力を誤って割り当てました。 終了の15分前に最後のタスクに合格しました。 私はC ++でのみ記述します。 タスクの定式化を理解することは困難でしたが、私はそれに対処しました。 次に、主催者がタスクの分析を投稿するのを待ちます。」



すべての受賞者と受賞者を祝福します！ そして、良い結果を出してくれた人たちが私たちの大学に行き、最高レベルのトーナメントでふさわしいパフォーマンスをするスポーツプログラミングチームの仲間入りをしたいと思います。
