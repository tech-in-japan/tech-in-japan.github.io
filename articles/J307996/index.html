Macおよびその他のParallels Desktop 12の機能でバッテリー寿命を最大化する方法

2016年8月、世界的に有名なMac用Parallels Desktop Utilityの12番目のバージョンが登場しました 。 この記事では、開発者がその主な機能について説明します。 技術用語で知りたいことについてのコメントを書いてください。以下の資料であなたの希望を考慮します。



Parallels Desktop for Macは、MacアプリケーションとWindowsアプリケーションを再起動したり切り替えたりすることなく、Mac上でWindowsプログラムを実行するソリューションとして常に推進されてきました。 同時に、これは技術の正確な説明ではないことを認識していました。実際、WindowsではなくMacで実行されていますが、PCのように動作する仮想マシンです。



Parallels Desktopは、PCをエミュレートし、PC上で実行される任意のオペレーティングシステム上で実行できる仮想化ツールです。 Windowsが最も人気があるのは偶然です。 また、仮想マシンはまだ標準のPCよりも機能がいくらか豊富であるため、その制御下にあるWindowsは少し良くなっています。



定期的なシステムメンテナンス



最初に、Windows 10ユーザーの定期的なメンテナンスに関して、生活を楽にしました。 作業の代わりに、Windowsの更新プログラムがインストールされるまで待たなければならない瞬間を好む人はいません。何らかの理由で常に独自の計画があります。 さらに、Windows 10の更新とメンテナンスのプロセスは非常にリソースを消費し、プロセッサー時間と帯域幅の顕著なシェアを消費し、プロセッサー（クーラー）のアクティブな冷却も含みます。 Parallels Desktop 12は、このようなアクションについてユーザーに警告しますが、もちろん、ユーザーの利便性のためにこのプロセスに影響を与えることができるはずです。 ユーザーは、Macで作業していない時間を指定してWindows 10を自動的に維持できます。この場合、スケジューラーは、ユーザーが作業を必要とするときはいつでもWindowsの更新を禁止し、プロセッサー時間を占有せず、パフォーマンスを低下させません。 これは、更新プログラムのインストールだけでなく、更新プログラムのダウンロードにも適用され、パフォーマンス、特にネットワークに影響を与える可能性があります。



Parallels Desktopの以前のバージョンのユーザーは、ユーザーがmacOSで直接作業している間に、明確な理由もなくWindowsが大量のリソースとエネルギーを消費し始めていると不満を述べました。 Windows仮想マシンはそれ自体を自由と見なし、独自の裁量でバックグラウンドプロセスを開始することがわかりました。 メインOSとの統合を追加し、PC仮想マシンはシステムのステータスを独自に決定せず、macOSオペレーティングシステムからステータスを受け取ります。



確かに、重要な更新プログラムがあります。より正確には、Windows Updateが重要と見なす更新プログラムです。 この場合、ユーザーの勤務時間を設定します。この場合、どのような状況でも、重要な更新プログラムであっても、Windowsの更新を許可しません。







同時に、プランナーはさまざまなビジネスシナリオとの互換性があります。 組織で確立されたポリシーに従って、ワークステーションが更新、ソフトウェア配布などのために企業のマネージャーによって管理されている場合、もちろん、私たちは彼を悩ませません。 一般に、Parallels Desktop 12が企業環境で動作する場合、その設定のほとんどは、リモートで作業したり、設定の特定の変更を禁止したり、パスワードで設定を閉じたりできる管理者によって制御されます。



生産性を向上



Parallels Desktop 12の仮想マシンは、グラフィカルインターフェイスなしで、バックグラウンドですぐに使用できるモードにすることができます。これにより、Windowsアプリケーションを通常の約3倍の速度で起動できます。 原則として、アプリケーションの起動に必要な時間は、仮想マシンの起動、この仮想マシンでのWindowsのロード、およびWindowsを使用した選択したアプリケーションの起動に費やされます。 平均的なMacまたはMacBook構成では、このサイクル全体で、たとえば約30秒かかる場合があります。 私たちの新しいオプション「常にバックグラウンドで準備ができています」は、このプロセスを最適化し、それをほぼ瞬時にします。 ユーザーがParallels Desktopを離れると、仮想マシンは一時停止され、すぐに起動して操作を継続するために必要なものはすべてRAMに残ります。







ユーザーがWindowsアプリケーションまたはWindowsプログラムを含む特定のドキュメントを必要とするとすぐに、Parallels Desktop 12はユーザーインターフェイスを表示するだけです。 これはすぐに起こります。



ドキュメントを操作する



システムツールだけでなく、使いやすさを向上させ、オフィスアプリケーションの機能を拡張する方法によって、ドキュメントとアプリケーションの作業の生産性を向上させます。 たとえば、Outlook 2016 for Windowsの機能を誰もがよく知っているため、最近作業中だったドキュメントを電子メールメッセージに添付できます。 Parallels Desktop 12では、最近のドキュメントのリストには、ユーザーがWindowsではなくmacOSで作業したドキュメントも含まれています。







さらに、Windows用のOfficeドキュメントのサポートをMac OSブラウザーに統合しました。Safariだけでなく、ユーザーがインストールしたMacブラウザーでも同様です。 ユーザーが開くブラウザーに十分なMicrosoft Officeオンラインエディターがない場合、「Excel / Word / PowerPointで編集」ボタンをクリックして、Parallels Desktop 12は、Windows用のフル機能のMicrosoft Officeアプリケーションでこのファイルを開くために必要なWindows仮想マシンを「指示」しますMicrosoft OfficeはWindowsにインストールされます）。



寝ないで！



標準のWindowsツールを使用すると、プレゼンテーションの表示中にスクリーンセーバーとエネルギーセーバーが機能するのを防ぐことができますが、macOSは現時点ではこの特別なWindows動作モードについては何も認識せず、最も不適切な瞬間に眠り続けます。 Parallels Desktop 12では、Windowsアプリケーションを構成する簡単な方法を思い付きました。ユーザーは、次のような「ドック」属性にあるアプリケーションアイコンを割り当てることができます。



thisこのアプリケーションを常にフルスクリーンモードで実行します。

thisこのアプリケーションの実行中にMacを「スリープ」状態にしないでください。

thisこのアプリケーションをアクティブにするときは、常に他のウィンドウを非表示にします。

applicationアプリケーションの実行中に、サイレントモードを有効にします。



たとえば、PowerPointでプレゼンテーションを表示する場合、スクリーンセーバーを画面に表示したり、バッテリーを節約するためにコンピューターを突然シャットダウンしたり、リスナーがメール、インターネットインスタントメッセンジャー、システムステータスメッセージのポップアップを表示したりしないようにします。 [邪魔しない]属性を使用すると、プレゼンテーション中にあなたや視聴者の気を散らすものがないことを確認できます。



網膜



AppleがMacコンピューターを搭載し始めて以来、Retinaディスプレイをサポートしてきました。バージョンごとにこのサポートを改善しています。 Parallels Desktop 12では、2つの大きな改善が行われました。 まず、複数のディスプレイで作業する場合、Parallels Desktop 12は各ディスプレイで個別に機能し、Retinaディスプレイで鮮明な画像を、他の外部ディスプレイで正しくスケーリングされた画像を取得できます。 第二に、Windows 10にRetinaディスプレイにインターフェイスの要素を正しく表示するように教えました。 開発者は限られた数の物理ディスプレイでオペレーティングシステムを実行することを計画しており、異なるモニターのインターフェイス要素の表示に固定値を提供するため、Windows 10自体は異なる画面解像度でインターフェイス要素を表示するのにあまり柔軟性がなく、仮想マシンのスケーリングの問題を解決しません。 この問題を解決し、Windowsウィンドウでは、サイズと比率に関係なく、ユーザーは常に標準サイズのWindows 10のテキスト、アイコンを表示します。



省エネ



「システムの定期メンテナンス」と「生産性の向上」のセクションで説明した対策のおかげで、計画外のバックグラウンド操作を実行するシステムが少なくなり、ハードディスクへのアクセスが少なくなるなど、バッテリーでシステムの動作を非常に真剣に最適化しました。仮想マシンを起動すると、バッテリーが長持ちします。







Parallels Desktopで仮想マシンを負荷なしで実行すると、Macの総エネルギーのわずか3％しか消費できないことが実験的に確立されています。 負荷がかかっている状態、つまり 実行中のオフィスアプリケーションでは、Parallels Desktopの次の各バージョンの最適化は前のバージョンと比較して10〜15％です



Sierraの準備ができました！



この秋に正式にリリースされるMacOS Sierraには、Mac上のアプリケーションとファイルのディスク使用量を明確に示すOptimized Storageという機能があります。この機能を統合し、Sierraのネイティブユーティリティから直接仮想マシンが占有するスペースを最適化できます。 仮想マシンにインストールされているオペレーティングシステム、およびそれらを操作する慣習に応じて、最適化は非常に重要です-場合によっては、数十ギガバイトです。







macOS Sierraは、デスクトップとドキュメントをiCloudに保存することを提案しています。 ユーザーに不便をかける可能性があるため、仮想マシンを作成するためのデフォルトのフォルダーを変更しました。 これで、すべての新しい仮想マシンがユーザーの作業フォルダーに作成されます。



専門家



Parallels Desktopユーティリティが必要なユーザー向けに、オフィスのWindowsアプリケーションを実行するだけでなく、さまざまな環境をエミュレートし、ソフトウェアをテストし、複雑なクライアントサーバーシステム用の深刻な技術的ニーズにも対応しています。ProfessionalEditionバージョンをリリースします。 このようなユーザーは通常、多数の仮想マシンを実行しています。たとえば、1つはアクティブなMicrosoftディレクトリを持ち、もう1つはサーバーアプリケーションを管理する端末などです。 Macコンピューターは強力なサーバーシステムではなく、実行中の多数の仮想マシンのサービスが常に得意とは限りません。 そのため、プロのユーザーが仮想マシンで利用可能なリソースをリアルタイムで制限する機会を提供し、互いの作業やmacOSでのユーザーの作業を妨げないようにします。



また、仮想マシンのディープスリープ手順のパフォーマンスを大幅に向上させました（メモリがハードディスクに書き込まれ、CPU、RAM、およびネットワークリソースがMacに解放されるとき）。 このプロセスは60％に加速しました。 多くのゲームはWindowsとXboxでのみ利用可能ですが、Macでは利用できません。 現在、Parallels Desktopを使用すると、これらのゲームを仮想マシンでもプレイできます！



強力なプロフェッショナルツール-共有フォルダー。 この機能により、Windowsを実行している仮想マシンからMac上のファイルを操作できます。 私たちは長い間開発しており、バージョンごとに改善しています。 Parallels Desktop 12では、このツールの生産性が大幅に向上したため、Windowsで開発され、Macに保存されるプロジェクトのコンパイルが25％加速されました。



アクロニスとのパートナーシップのおかげで、ユーザーには、Acronis True Imageクラウドバックアップツールの年間ライセンスが500ギガバイトのスペースで無料で提供され、改良版では、増分バックアップオプションを使用して、操作に必要な時間と帯域幅の両方を大幅に節約します。



Macユーザー



典型的なMacタスクのキーボードショートカットの使用に関するユーザーレビューを分析しました。 スクリーンショットを撮る、画面からビデオを録画する、ファイルをアーカイブする、またはインターネットからビデオを保存するには、これらすべてのアクションに十分なメモリが必要です。 Macで同様のタスクを実行するための数え切れないほどのキーの組み合わせを覚えている人がいるわけではありません。また、いくつかの小さなタスクでも、個別のアプリケーションを検索してインストールする必要があります。 Parallels Desktopでは、最も一般的な機能を実行するための一連のツールをユーザーに提供し、Parallels Toolboxと名付けました。 次の記事では、彼と彼の主なツールについて詳しく説明しますが、現時点では、注目を集めるための小さな絵です。
