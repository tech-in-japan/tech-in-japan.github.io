結論



最初に、キネティックショートストーリーの開発と配信が3つの目標を追求したことを強調する価値があります。

-インタラクティブなストーリーテリングテクノロジーを実行して、Telltale Gamesスタイルの手描きのストーリーテリングの準備をします。

- リセット後の設定（ソリッドサイエンスフィクション、ポスト黙示録）にできるだけ多くの新しいプレーヤーを紹介します。これは現在開発中です。

-実際にAppStoreとSteamの対象読者を比較します。



1.ロシアのAppStoreの視聴者の大多数は、「 トーシンスキーの視聴者 」に属している可能性が最も高い（特に、英語を知らず、学習を望まず、無料で受け取ったアプリケーションにも1の評価を付けます）。 同時に、ロシアのSteamの視聴者は、逆に、貴族のCORE / Khabrahabrovskayaに近く、英語のプロジェクトに対して非常に友好的です。



2.エンコーダー/パブリッシャーのリクエストにより、AppStoreでゲームが配布された国のリストでロシアをブロックしないことにしました。 もちろん、私は視聴者のなじみのない部分からユニットを取得しましたが、同時に、視聴者の一部のために、ロシアの一部のファンからコンテンツへのアクセスを奪うことは公平ではありませんでした。 このソリューションは、製品の2番目の目標セットに基づいていることに注意してください。 キャッシュの貨幣化と最大化が目標であり、ロシア語でローカライズされていない場合、ローカライズ（存在する場合）までロシアでのアプリケーションの可用性をブロックする方が収益性が高くなります。それ以外の場合、ロシア語以外の大多数のAppStoreユーザー（ Tinderでさえ同じ理由でバンドルを選択します）、これはグローバルランキングに影響します。



3. Steamは、開発者にとって最も使いやすいプラットフォームです。 最も広範な十分なゲームオーディエンス（1億2500万人のアクティブユーザー）であるMVPは、現時点で最も便利で広範なマーケティングサポートシステムであるGreenlightの通過によるテストを要求しています。 Git関数についてはまったく話していない。 一般的に、最初のゲームを見ている場合、私のアドバイスはSteamで行うことです。



4. MacおよびLinux上のSteamの販売シェアは非常に高いため、ユニバーサルエンジンを使用する場合は、設計時に、Winだけでなくこれら2つのOSとの互換性をすぐに考慮してください。 これは、たとえば、NPC /キャラクターのボーンの数やシェーダーの記述に適用されます。



5. Steamでの最初のゲームであるGreenlightの平均売上は500〜1000ドルです。 これらの結果は、Steamリリースが初めての経験であり、半年から1年半までゲームの開発に費やした数十人の同僚にとっても同じです。 ただし、後続の各ゲームのROIは増加します。 少なくとも、以前のゲームが好きだったファンの成長によるものです。

NBあなたはコマーシャルなしで開始できますが、統計によると、その可用性は米国でのダウンロード数を平均27％増加させます。 統計と分析については、AppAnnieを接続することをお勧めします。 シンプルで有益でフレンドリー。 AppAnnieをiTunes Connect開発者アカウントにすぐに接続できない場合でもてないでください（これは非常に頻繁に発生し、さまざまな種類のエラーを伴います）-AppAnieサポートチームにエラーを報告するだけで修正できます（平均して、修正には5から20がかかります）日）。



5.ほら！ ゲームを公開しました。 数十のインストール（ゲームに料金がかかる場合）または数百のインストール（ゲームが無料またはフリーミアムの場合）に気づきます。 Appleはこれを「機能」に追加し、ダウンロード/インストールの平均35％の増加を保証します。 しかし、次は何ですか？



6.次のステップは、ゲームを最小実行可能製品（MVP）としてテストすることです。 「生存率テスト」。 いくつかのプレスリリース、プロモーションパックを準備し、ジャーナリスト、レビュアー、およびyoutubeに配布する必要があります。



NB独自のプレスリリースを作成し、iSpreadNews（250ドル）およびprMac（150ドル）経由で送信できます。 また、少し忍耐強く、直接書く必要のある150人のアクティブなAppStoreレビューアのシートを作成します。 Healthy Man Reviewerは、気に入った場合は常にゲームのレビューを公開し、無料で行います。 お金のためにレビューを公開する「Smoker Surveyors」に時間を無駄にしないでください。 原則として、ROMIは0を目指します。MVPをテストすることを目的とするインディー開発者にとって、有料レビューはまったく意味がありません。



7.前の段落で適用されたこれらのマーケティング努力はすべて、平均で約100回以上のダウンロード/設定（ゲームが有料の場合）および約1000回以上のダウンロード/設定（針が無料の場合）、またはこれらの極端なインジケーター（ゲームの場合フリーミアム）。 MVPのテストが完了したら（通常1か月かかります）、「ストリートマジック」を開始します。ユーザーあたりの平均収益（ARPU）と月間コンバージョン率（CR D30）を計算します。



一般的に、ARPUを計算するには、今月の収入をバイヤー/寄付の数で割る必要があります。 また、CRD30を計算するには、今月の購入者数/寄付数をダウンロード数/インストール数で割る必要があります。



たとえば、テストでは、10人のユーザーにゲームを販売して10ドルを獲得しました。 ゲームのダウンロード総数は100ダウンロードでしたが、 したがって、ARPU = 10/10 = 1.0ドル、CRD30 = 10/100 = 0.1です。 完全に有料のゲームの場合、各ジャンプ/インストールが購入に関連付けられているのは論理的です。したがって、CR D30は、インストールから購入への変換ではなく、AppStoreのゲームページへの訪問のインストールへの変換として検討する方が適切です。



NB 2016年初頭、無料/フリーミアムモバイルゲームの平均市場ARPU（平均開発予算が75,000ドル以上のゲーム）は、戦略ジャンルで1.8ドル、アクション/スラッシャーで1.4ドル、ソーシャルギャンブルで0.9ドル、パズルで0.6ドルでした。



8.これで、1人の有料ユーザーの獲得に対して支払うことができる金額-アクションごとのコスト（CPA）がわかりました。これは、完全に支払われたゲームのインストールごとのコスト（CPI）と同じです。 ゲームがフリーミアムの場合、最大許容CPIを計算できます。 そして、この情報を入手して、MVPとしてのゲームの実行可能性と、収益化の価値があるかどうかを判断するのはあなた次第です。 得られた指標と市場指標との比較により、開発を継続するか、調整を変更するか、トラフィックのアクティブな購入を受け入れるか、AppStoreを放棄して作成されたゲームを興味深い人生体験として扱うように説得する必要があります。



積極的に収益化することに決めた場合は、ユーザーあたりの獲得コスト（ACPU）のコストでCPAトラフィックを購入する必要があります。これはARPUよりも低く、差と規模で稼ぐことができます。 トリックは、99％のケースでACPUがARPUより高いことです:)



また、ACPUを削減するには、規模でプレイする必要があります（トラフィック調停の経験も考慮に入れます）。ユーザーを購入するには20,000ドルのマーケティング予算が必要です。



NBアメリカのAppStoreの関連トラフィックを取得する最も有名で信頼できる場所はAdMobです。 そして、もちろん、高品質でターゲットを絞ったトラフィックの購入は、それ自体が芸術であり、ニュアンスに満ち、経験が必要です。
