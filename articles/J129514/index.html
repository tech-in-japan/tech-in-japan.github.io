Tactoom-クローズドベータ

Octaveと私は少し仕事をしてTactoomを作りました 。 彼は現在、クローズドベータ版です。

長い紹介文は書きませんが、すぐに仕事に取り掛かります。



Tactoomは、3つの基本的な「I Want」に基づいて構築されたWebサービスです 。



1.私と同じものに興味がある人と共有したい。

2.興味のあることだけを他の人から学びたい。

3.「精神的に近い」人に私を見つけてもらいたい。



*自動招待はこのページからのみ機能します*



サービスのすべての機能とロジックは、これらの「Want」（UserStories）を提供することを目的としています。



詳細：



1.私と同じものに興味がある人と共有したい。







Tactoomの各ユーザーには、自分の興味（たとえば、 Yuri Bogdanovの興味）のリストがあります。 Tactoomで投稿を公開し、たとえば「コンピューター」と「インターネット」のタグを付けてタグを付けると、この投稿を特定の人々の興味に関連付けることができます。 つまり、あなたのエントリーのこれは、「コンピューター」と「インターネット」に興味がある読者のそれらによって見られます。

Tactoomでは、すべての読者と共有するのではなく、特定の投稿に明らかに興味がある人と共有します。



2.興味のあることだけを他の人から学びたい。







だから、あなたはあなたの興味のリストを持っています。 ニュースフィードの外観はそれらに依存します。

たとえば、興味の対象に「犬」がいない場合、たとえそれを読んでも、ニュースフィードにクラスメートの犬の写真セッションは表示されません。 彼女があなたの共通の関心事の枠組みの中で何かを投稿すると、あなたはそれをあなたのストリームに見ることができます。

そうです。 ニュースでは、読んだ人があなたの興味に関連している場合、その人からの投稿が表示されます。 もちろん、スパマーやIQ <60の幸せな所有者を読むと、スパムが表示されます。 別のケースでは、スパムや「おいしい朝食を食べました、ここに私のサンドイッチの写真があります」などの投稿はほとんどありません。



3.「精神的に近い」人に私を見つけてもらいたい。







精神的にあなたに近い人をどこで見つけることができますか？ ソーシャルネットワークで、職場で、休暇で...？

ですから、世界には、私が話したいことがあり、コミュニケーションをとることに興味がある人がたくさんいると思います。 しかし、何らかの理由で、このような大規模でソーシャルなインターネット上でそれらを見つけることができません。 そして、彼らは私を見つけることができないことを知っています。

Tactoomでは、あなたが人であることを表すあなたの興味を示し、精神的に近い人を見つけます。多くの興味を持っている人はあなたのものと一致します。

Tactoomは、あなたと同じものを大切にしている人を見つけるために作成されました。



だから。

Tactoomでは、必要に応じて多面的で本物の自分になります。 あなたはそのような会社の従業員でも、クラスメートでも、同僚でも、古い知り合いでも、会議の男でもありません。 Tactoomであなたはあなたです。 また、他の人と共有したい場合、「必要ない」などの質問はありません。 ここでは、コミュニケーションとコミュニケーションは「それを必要とする人々のために」の原則に基づいて構築されています。



近日発売予定の機能：

-オンライン通知

-ブログの色設定（テーマ選択）。

-写真/音声のダウンロード

-ブックマークレット



後で起動されるいくつかの機能：

-プライベートメッセージ

-推奨アルゴリズムの改善

-リアルタイム、ページのリロードが少ない

-集団知能に基づく自動調停アルゴリズム

-RESTful API

-ニュースフィード、ブログ、コミュニティのAtom / RSSフィード



また、ユーザーからよく寄せられる2つの重要な質問への回答もここに投稿します。



質問： 「人が「猫」に関する投稿を投稿するのを防ぎ、タグに「プログラミング」と表示して、プログラミングに興味のあるすべての読者がニュースフィードに「猫」をまったく興味のないものにしていることを示します。

回答：まず。 「プログラミング」に関する投稿で「子猫」とマークすると、読者はいますか？

二番目。 すべてのユーザーが2回のクリックで投稿を不適切としてマークできる機能がまもなく提供されます。 メカニズムは単純です。大まかに言えば、100人がユーザーから「不適切」と指摘した場合-一部のタグの禁止。 体系的な「不十分」-Tactoomのaufvidzerzen。

三番目。 後で、集合知能に基づくアルゴリズムが開始されます。 本質的に本質的に異なるタグは、「霧の中に」送られます。



質問： 「ナンセンスな投稿タグを変更するために人が再投稿するのを妨げるものは何ですか？」

回答：投稿のタグ（インタレスト）を変更することにより、ユーザーは自分のページでのみ、読者のみにタグを変更します。



サービスへの登録は招待によるものであり、非常に時間がかかります。



追伸 ちなみに、プロジェクトはNodeJSのYura Bogdanov（ octave ）によって8か月でmongodb / redisによって作成されましたが、近いうちに技術記事が作成される予定です。



UPD：

私たちは2人（JuraとI）います。私たちは会社ではなく、自分ですべてをしているので、物理的にコメントに応答することはできません。

今、私たちの仕事は、便利で便利、高品質、高速のサービスを完全に実現することです。 興味深い人に有効です。
