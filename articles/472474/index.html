Funny cosmetic bug in Google Chrome

I recently discovered a funny bug in the installer of the most popular Google Chrome browser. Despite the fact that the bug is cosmetic - it made me smile.







The bug is that the installer creates two shortcuts on the quick launch panel. Warms the soul, the fact that not only I leave bugs after me. In general, I wonder how it happened? With the release of the new release, they conducted partial testing, but not complete testing? Shouldn't it be checked how the update lays down on top of the new version? Most likely, Google has automatic tests that did not completely cover all outcome options: for example, they checked for the presence of a shortcut, and the fact that it should be in a single copy is not taken into account.



At the time of publication of the article, the bug is relevant.

OS: Windows 7 Home Basic 32bit.



Steps to play:

1) Download the latest official release from google

2) Install the browser (if you already have one, then delete it first)

3) We look at the result



Expectations:

Google Chrome installed and added shortcuts: to the desktop, to the quick launch bar, to the main menu of the Start button.



Result:

Google Chrome installed and added one more shortcut to the quick launch bar.
