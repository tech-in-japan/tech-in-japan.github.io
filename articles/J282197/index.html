Ubuntu 16.04 LTSリリース-スナップ、OpenStack、およびその他の革新。 AMDグラフィックスカードで起こりうる問題







CanonicalはUbuntu 16.04 LTS （コードネームXenial Xerus）をリリースしました。 LTSリリースは以前4年間サポートされていました。 現在のリリースでは、Ubuntu、Kubuntuが5年間、残りの派生ディストリビューションが3年間サポートされます。 LTSリリースには、安定したリリースバージョンを維持しながら、カーネルを新しいバージョンにアップグレードする機能もあります。 これにより、安定性と新しい鉄のサポートとの間に一定のバランスが得られます。

同時に、このディストリビューションの派生物が登場し、異なる基本プログラムセットとグラフィカルシェルを備えています。







  1. Xubuntu（xfce）
  2. Kubuntu（kde）
  3. Lubuntu（lxde）
  4. Ubuntu MATE（メイト）
  5. Ubuntu GNOME（gnome3）


このリリースの主要な更新：









  1. 新しい「スナップ」パッケージ形式。 従来のdebパッケージとともにサポートされます。 主な違いは、依存関係の満足度の欠如です。 パッケージには、必要なものがすべて含まれています。 一方で、これは、各プログラムがメガトンのライブラリとデータをドラッグするWindowsへの道です。 一方、ソフトウェアの新鮮な実験バージョンの配信の便利な形式。
  2. OpenStackをサポートする新しいLXDコンテナー仮想化
  3. ZFSおよびCephFSは、大規模なクラウドインストールをサポートします。
  4. ダッシュのオンライン検索をデフォルトでオフにしました。これは、過去にフリーソフトウェアの多くの支持者を非難していました。
  5. これで、画面の下部にあるドックを移動できます。 これに先立ち、Unityドックは伝統的に左側に釘付けされていました。 物議をかもすイノベーションですが、多分誰かがそれを高く評価するでしょう。
  6. これで、ウィンドウタイトルにメニュー項目を常時表示できるようになりました。 （@ r3r）

AMDグラフィックカードユーザーの問題









Ubuntu開発者は、独自のドライバーであるAMD Catalyst（fglrx）の古いサポートを使用していますが、これはUbuntu 16.04のリリースには含まれません。 ユーザーは、オープンAMDGPU / Radeonドライバーの使用に切り替えることをお勧めします。 AMD WebサイトからAMD Catalystドライバーをインストールすることは、最新リリースがXサーバー1.17と3.19までのLinuxカーネルのみをサポートし、Ubuntu 16.04がXサーバー1.18とLinuxカーネル4.4を使用するため、防止されます。 以前は、ディストリビューションのグラフィックスタックのコンポーネントのバージョンを考慮して、AMDとともにUbuntu用に特別なドライバーアセンブリが準備されていました。

GCNアーキテクチャに基づく最新のAMDグラフィックスカードの完全なサポートを提供することを目的とした、AMDによるオープンAMDGPUドライバーの開発を考えると、独自のAMD Catalystドライバーの提供は、古いAMD GPUシリーズのみをサポートする意味があります。 問題は、古いGPUのサポートがAMD Catalystからすぐに削除されることです-すぐにドライバーの大幅なクリーニングが実行され、その結果、Radeon HD 8000シリーズ（HD 8400以下）、Radeon HD 7000シリーズ（HD 7600以下）のサポートが中止されます）、Radeon HD 6000、Radeon HD 5000、およびGCNアーキテクチャを使用しないその他のカード。







AMDGPUスタックに基づくドライバーで修正される予定のオープンドライバーの問題には、OpenGL 4.5（OpenGL 4.1のオープンドライバー）のサポートの欠如、多くのゲームの起動時のパフォーマンスラグ、OpenCLの不完全なサポート、CrossFireマルチGPUのサポートの欠如が言及されています。
