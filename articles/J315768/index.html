強いモード



Dartのストロングタイピングの問題に比較的多くの注意が払われました。 Dart自体は完全に静的に型付けされた言語ではありませんが、Dartには完全な型サポートと、入力エラーをチェックできる静的アナライザーがあります。 アナライザーモードの1つはストロングモードで、特にこのエラーを「細心の注意を払って」分析します。







Googleは既に製品をStrong Modeに切り替えています。 これにはいくつかの理由があります。 まず第一に、開発者がより多くのサポートされているコードをリファクタリングし、記述しやすくします。 ただし、ストロングモードを使用する同様に重要な理由は、コードツールが型システムに依存できることです。 したがって、たとえば、以下で説明するDart Dev Compiler（DDC）を開発するための新しいコンパイラは、より人間が読めるjsコードを生成するために型システムに依存しています。







このドキュメントでは、ストロングモードとは何か、なぜストロングモードが必要なのか、またはDart Summitのレポートをご覧ください。









Ddc



Dartの開発インフラストラクチャを補完するもう1つの部分は、Dart Dev CompilerまたはDDCで、これもサミットで議論されました。 その名前が示すように、開発モードで使用されるコンパイラです。







現在、Dartでの開発には、ブラウザーでの後続の作業のためにコードをすばやく更新できる仮想マシンがDartiumブラウザーのみであるという事実に関連する特定の不便さがあります。 Chromeを含む他のブラウザーでデバッグを実行する場合、dart2jsコンパイラーが使用されます。 コードが変更されるたびに、（変更されたコードだけでなく）すべてのコードをコンパイルします。 比較的大きなコードベースでは、このアプローチは十分に便利ではないかもしれません。







DDCを使用して開発する場合、アプローチは大きく変わります。 まず、GoogleはDDCでの開発を支持してDartiumのさらなるサポートを放棄する予定です。つまり、開発されたコードは本番コードと同じ環境で実行されます。 最新のブラウザで。 第二に、ストロングモードのおかげで、DDCは型システムを信頼し、視覚的に実質的に元のDartコードと変わらないES6コードを生成できるため、デバッグが大幅に簡素化されます。 さらに、ブラウザーでのDevToolsとの統合を簡素化します。







もう1つの重要な利点は、DDCがコードベース全体ではなく、コードの変更された部分のみを再コンパイルすることです。 さらに、開発をさらに加速するために、 Webpackなどのモジュラーフレームワークとの統合を実行して、たとえばホットリロードを提供できます。







DDCはdart2jsを置き換えるのではなく補完することに注意してください。まもなくすべてのコンパイラ（dart2js、DDC、vm）に1つのコードベースが追加されます 。









JS相互運用



新しい言語の課題の1つは、古いコードと対話することです。 Webの世界では、10億行を超えるコードがjsで記述されていますが、ほとんどの場合、これはDartで書き換えられることはほとんどありません。 そのため、Dartlangチームはjsの世界との相互作用を改善するために多大な努力をしました。







以前は、Dartの「JSON.stringify（a）」の単純な呼び出しは次のようになりました。 







import 'dart:js' as js; var a = { 'value' : 1 }; var str = js.context["JSON"].callMethod("stringify", [a]);


このアプローチでは、タイプをチェックする機能がなく、「jsの直感」だけに依存する必要があります。







新しいエディションでは、この問題の解決策ははるかに簡単に見えます。 







 import "package:js/js.dart"; @JS("JSON.stringify") external String stringify(obj); var a = { 'value' : 1 }; var str = stringify(a);


さらに、jsと対話する前に、Map型のオブジェクトを作成する必要がありました。 







 var jsObj = new JsObject.jsify({'value':1}); jsMethod(jsObj);


もちろん、この状況では、型チェックに依存できませんでした。なぜなら、 Mapのすべてのタイプ（キーと値の両方）は動的でした。







新しいエディションでは、すべてがより透明に見えます。 







 @JS() @anonymous class Options { external int get value; external factory Options({bool value}); } var jsObj = new Options(value: 1); jsMethod(jsObj);


そのため、オブジェクトのフィールドとそのタイプを完全に制御できます。







構文と新しいアプローチの本質について詳しくは、 こちらをご覧ください 。会議のビデオをご覧ください 。









既存の制限 ：古い（import 'dart：js'）スタイルと新しい（import "package：js / js.dart"）スタイルを混在させることはできません。







また、チームはjsコードのラッパーの生成を自動化するために素晴らしい仕事をしました。
