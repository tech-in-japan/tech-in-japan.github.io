プロセスアプローチの例：産業会社のプロジェクト前調査。 BPMNチャートの例





「 コンピューター情報システムとは 」および「ソフトウェアを実装する際のプロセスモデルの構築」の記事で、理論的知識を実践する例を紹介することを約束しました。 以下は、会社の調査の結果に関するレポートとグラフィカルな表記で構成される実際のプラクティスの例であり、提案された作業モデルを明確に示しています。



このようなレポートは、タスクの一部として会社の仕事を研究した結果です。 つまり 文書を作成する前に、会社の経営陣と従業員の調査を実施し、部門間の相互作用の技術的能力と特徴を調査しました。 そして、既存のビジネスプロセスの理解に基づいて、自動化（ソフトウェア製品の実装）の使用を含む作業を最適化するためのドキュメントが作成されます。



活動分野：産業機器の製造および販売。



会社のスタッフ：約50人。



プレビューから画像を見るには、ここで説明されているプロセスのグラフィカルな表現をリンクのBPMN形式で見ることができます。



会社の構造



3つの営業部門が顧客との連携に取り組んでおり、それぞれが組織内で受け入れられている名前で独自の方向性を導きます。



  1. 高さ***
  2. プロム***
  3. 期間***


さらに、Vysの一部門であるテスト部門は同じクライアントと連携するため、クライアントと連携する際に個別に検討されることはありません。



会社の構造には次のものも含まれます。



  • 自社生産
  • マーケティング部
  • 倉庫
  • サービス部門。


ここでは考慮されない販売プロセスに関連しないユニットも同様です。



タスク



同社の調査結果によると、顧客との作業の自動化の低レベルが明らかになりました。



  • 従業員の大量の肉体労働、
  • クライアントとの自動電子通信はありません。
  • 電話を制御することはできません。
  • 潜在的な顧客との仕事の計画と品質管理の可能性はありません。


私たちの専門家は、次の分野を解決することを任されました。



  • 着信リクエスト（リード）による作業の自動化、
  • 標準化されたビジネス販売プロセスの開発、
  • ビジネスプロセスの1C構成の構成、


新しいシステムで作業するスタッフのトレーニング：



  1. 倉庫作業員;
  2. 営業部門;
  3. マーケティング部門。


作業は段階的に実行されます。



  1. ビジネス販売プロセスの開発と実装。
  2. マーケティング部門向けのツールの開発。
  3. 倉庫の自動化。
  4. 営業部門のビジネスプロセス


アプリケーションを配布するとき、どの部門に向けられるべきかを考慮することが重要です。 それ以外の場合、クライアントと連携するビジネスプロセスはまったく同じです。



システムへの登録後、すべてのリクエストは責任者が処理する必要があります。責任者には、リクエストごとに営業部門のソートと任命が含まれます。 この作業は、少なくとも1日2回行う必要があります。 そして可能であれば-それらが利用可能になると。



営業チームの役割



営業部門には2つの主な役割があります。



  1. セールスマネージャー（販売者）;
  2. 営業部長
  3. マネージャーは、特別な困難、ニーズ、標準的な販売スキームからのその他の逸脱がない場合、通常の方法でクライアントと対話する主な作業を実行します。


頭：



  • 着信コールを配信し、
  • マネージャーの仕事を監督する
  • 困難で異常な場合にクライアントと連携します。


モニタリングは、会計システムでレポートを調査することによって実行され、売り手によるタスクの履行の適時性が監視され、売り手の満足度について顧客の選択的な電話調査が行われます。 非標準的な状況またはマネージャーがタスクに明らかに対応していない場合、営業部門の責任者は営業プロセスに直接接続し、問題の状況の原因を特定して解決策を見つけます。



着信チャンネル



アプリケーションは、次のチャネルを介して受信されます。



  • 電話
  • メール
  • 会社のウェブサイト。


この場合、サイトからのすべての情報は自動モードで電子メールを介して送信されます。 したがって、実際には、電話と電子メールの2つのチャネルをCRMシステムに実装する必要があります。



電話での作業には、MIKOシステムが最適です。 各着信コールを修正する必要があります。 そして、電話がデータベースにない場合、つまり 新しい潜在顧客が電話をかけている場合、電話のリードが自動的に作成されます。



次のいずれかのテンプレートに基づいて、電子メールの手紙をCRMシステムにロードする必要があります。



  1. JivoSiteオンラインチャットでの対応。
  2. サイトのフィードバックフォーム。
  3. 潜在的な顧客からの手書きのメール。
  4. 電子メールアドレスをCRMシステムにアップロードするとき、連絡先の住所がデータベースと一致することを確認するために、同様のチェックが実行されます。 また、不明なアドレスからリクエストが検出されると、リードとコールが自動的に生成されます。


リード登録



リードとは、商品やサービスの購入に関心を示したが、まだ顧客になっていない潜在的な顧客です。 私は選択をせず、支払いもしませんでした。



リードの登録は自動的に行われます（電話での通信プロセスで受信した情報を除く）。 カードには、最初の連絡時に受け取った情報（電話番号、電子メール、連絡先の名前、リクエストの本質、および提供されている場合はその他のデータ）が含まれています。



リードは、作成された根拠に基づいてアピールを参照して、タイプ「潜在的なクライアント」を持つ取引先として1Cシステムで自動的に作成されます。



新しいリードの出現に関する情報は、営業部門の責任者に送られます。営業部門の責任者は、情報を調査し、責任ある従業員を手動で任命します。



リード資格（一次処理）

見込み客の資格とは、見込み客データの収集です。連絡先、会社情報、潜在的なクライアントのニーズや希望に関する情報の取得などです。



これは以下に基づいて発生します。



  • 最初の控訴時にリーダーが提供した情報。
  • データベースチェック。


この段階はプライマリです。 これにより、着信要求が基本的にリーダーであるかどうか、または関心が予備的であるか、誤っているかなどを識別できます。 初期処理は、営業部長が任命した従業員によって実行されます。



取り引き



リードのニーズを特定し、協力への関心を確認した後、クライアントとのすべてのさらなる作業が実行されるトランザクションが作成されます。



トランザクションを作成する場合、記入する必要があります：



  1. 顧客のニーズ（彼のすべての願いがここにリストされています）;
  2. 法人に関する情報（名前、支払いの詳細、住所など）
  3. 連絡先情報。


トランザクションの主な段階：



  1. 機器の選択;
  2. 生産;
  3. 在庫からの販売。
  4. インストール（試運転）;
  5. サービス


各段階は、トランザクションのステータスに反映されます。



機器の選択



機器の選択は、会社の現在の能力やその他のニュアンスを考慮して、顧客の要求に基づいています。



クライアントの特性に応じて、協力のための2つのオプションが可能です。



契約が締結された対価の結果に基づいて、商業提案を作成して送信し、請求書が発行されます。 または、クライアントは拒否し、「unfit」のステータスを受け取ります。 後者の場合、マネージャーは売り手の作品の品質をチェックするタスクも受け取ります。



承認段階のない支払い請求書（契約）。



トランザクションは、クライアントのニーズと要件を反映しています。 議論され、交渉可能な希望。



すべての問題について合意した後、作業は次の2つのオプションのいずれかになります。



  1. 倉庫からの出荷。 必要なすべての機器が会社の倉庫ですでに利用可能な場合。
  2. 製造オーダー。 機器の一部を製造する必要がある場合。


検討することが重要です。顧客の注文の商品リストに、倉庫で入手可能な商品の一部と、工場で注文した一部が含まれている場合、これをトランザクションに反映する必要があります。



  1. 倉庫からの商品に対して予備が作成されます。
  2. 製造指図が生産に送信されます。
  3. 生産サイトで注文された商品の場合、サブシステム「生産」を作成する必要があります。サブシステムでは、製品の可用性の程度と条件が表示されます。


在庫からの販売



製品が工場で予約または注文されたかどうかに関係なく、顧客への出荷は倉庫から行われます。 生産は完成品を倉庫に送り、商品は天びんに表示されます。 そうして初めて販売できます。



販売に付随するすべての文書（倉庫、税、出荷請求書などの委託メモ）は、現在のトランザクションに「リンク」する必要があります。



認証



2種類の機器には認証が必要です。



  1. 気候**;
  2. 真似***


認定は特別な組織によって実施され、宣言されたパラメーターへの機器の適合性を検証するために必要です。



他のタイプの機器の場合、このステップは不要です。



サービス



販売と試運転の完了後、顧客にサービスが提供されます。 したがって、管理者が「サービスを提供する」タスクを形成する移行時、および同意を得る場合は「サービス契約を結ぶ」というサービスの義務段階です。



サービスの交渉、支払いの受領、および関連するすべての会計書類の後、取引は終了します。



1Cでの販売のビジネスプロセスの実装



提案されたソリューションを1Cソフトウェアシステムに実装するには、次のツールが必要です。



潜在的な顧客の着信要求を記録する必要がある「アピール」のディレクトリ。 既存のお客様の場合、新しい「アピール」（リード）は作成されません。 既存のカウンターパーティからのリクエストは、クライアントとの通信履歴に追加する必要があり、新しいリクエストに関する通知は、このカウンターパーティとの連携を担当するマネージャーによって受信されます。

クライアントは、デフォルトの「パートナー」と「カウンターパーティ」に加えて、カウンターパーティのディレクトリに「リード」と入力します。



商品、数量、用語、およびその他の重要な情報のリストを含むドキュメント「トランザクション」。 主要な取引パラメータ：



  • 適用範囲;
  • 機器の種類（ポンプ、コンプレッサー）;
  • 処理された材料の種類。


印刷されたフォーム「商用オファー」は、トランザクションで指定されたパラメーターに基づいて自動的に生成される必要があります。



バイヤーの注文-トランザクションで合意された商品のリストに基づいて、ドキュメントが自動的に作成されます。



倉庫オーダ、製造オーダ。 サプライヤへの注文-バイヤーの注文に基づいて自動的に生成されます。



  1. 商品の在庫がある場合、商品の予約とその後の出荷のために「倉庫注文」が作成されます。
  2. 商品を製造する必要がある場合、「製造指図」が作成されます。
  3. 商品の在庫がなく、会社がそれらを製造していない場合、「サプライヤへの注文」が作成されます。
  4. 商品およびサービスの販売に関する文書。 完成した注文の発送と支払いのためのドキュメントのパッケージ。 トランザクションに関連付けられ、それに基づいて形成される必要があります。


営業部門の自動化のためのタスクとソリューションの方法



販売自動化を実装するために取るべきアクション：



1Cシステムの現在のバージョンからデータ転送を伴う「1C Trade UT 14」構成への移行：



  1. 倉庫内の商品の残高。
  2. 商品と顧客のディレクトリ;
  3. 決済残高;
  4. 閉じられていない顧客注文。


上記のソリューションのリストに従って、システム1C UT 11.4を完成させます。



  1. 1Cで通話を登録するためのMIKOモジュールの接続。
  2. UT 11システムの電子メールへの接続。
  3. 営業部長とマネージャーのためにシステムを操作するための指示を行います。
  4. 新しいシステムで作業するためのスタッフのトレーニング（トレーニングの実施）。
  5. システムを稼働させる。
  6. マーケティング部門の自動化
  7. リードと顧客の連絡先情報に基づいて、マーケティング部門が郵送を実施するための電子メールデータベースが自動的に生成されます。


これを行うには、1CメールアドレスからMailChimpソフトウェアへのアップロードを作成する必要があります。



2番目の方向は、電話マーケティングです。 ここで、作業するには、電話と連絡先の氏名を含む最新の顧客データベースが必要です。 したがって、マーケティング部門の要求に応じて定期的に実行されるこのデータの自動アップロードを実装する必要があります。



さらに、得られたデータに基づいて、マーケティング担当者は1人のマネージャーまたは別のマネージャーにタスクを設定できる必要があります-顧客に電話します。



マーケティング部門のタスクと解決方法



自動化を実装するために実行するアクション：



  1. MailChimpへの後続のアップロードのために、選択したセグメントのコンテキストで顧客連絡先情報のアンロードをExcelファイルに書き込みます。
  2. 電話マーケティングを計画する際のマーケティング担当者のその後の快適な作業のために、選択したセグメント（会社、連絡先、電話など）のコンテキストで顧客連絡先情報のアンロードを記述します。
  3. マーケティング担当者向けの職務指示書を作成します。
  4. スタッフトレーニング（トレーニング）を実施します。
  5. 倉庫の自動化


倉庫の効率を改善するには、いくつかの領域で作業を行う必要があります。



  1. バーコードに基づいて自動化を実行します。
  2. 有効期限による計量の自動化：バーコードを使用してシリアル計量を整理します。 このタイプの会計は、特定の賞味期限を持つ商品に必要です。 利用可能になったら販売します。 このアプローチは、期限切れ商品の償却を大幅に削減するのに役立ちます。
  3. 倉庫自動化のタスクと解決方法


タスクを完了するには、次のことが必要です。



  1. バーコードおよびバーコードのスキャン用の機器を購入するには;
  2. 商品の受領のシリーズ（パーティ）を考慮してバーコードを操作するための1C構成をセットアップします。最初に販売されるのは最も古いシリーズの残りです。
  3. 倉庫スタッフを訓練します。


用語集



リードは潜在的な顧客です。 ほとんどの場合、見込み客はマネージャーが記録した着信リクエストまたは関心のように見え、潜在的なクライアントが会社に提示しました。 CRMでは、リードは、製品に関心を示した人の連絡先の詳細の形式で記録されます。また、要求の性質に関する情報、つまり、彼が本当に興味を持っているもの、回答が必要な質問などの形で記録されます。



連絡先は、マネージャーが交渉している人物に関する情報です。 彼の名前、重要な情報、連絡先の詳細。



クライアントは、商品やサービスの購入者（顧客）である組織または個人です。 これには、組織、起業家、個人（エンドユーザー）があります。 クライアントに代わって、1人または複数の人が働くことができ、1つの店舗（倉庫、オフィス、アパート）でも、別の店舗でもかまいません。 しかし同時に、クライアントは1つです。 一般的なガイダンスと資金があります。



トランザクション（潜在的なトランザクション）は、クライアントに送信されるオファーです。 金額、当事者の義務、およびその他の重要な詳細を修正します。 通常、取引を作成するのは、見積を送信するか、契約を修正するために将来の契約の詳細を口頭で話し合うときです。 契約の締結および/または支払いの受領後、取引は完了したとみなされます。



トランザクションのステージ（ステータス）は、トランザクションで正確に何が起こっているのか、交渉はどの段階で、特定の潜在的な注文に現在取り組んでいるマネージャーのどの段階の質問に答えるパラメーターです。 トランザクションの段階では、現在の段階を修正し、このトランザクションの後続作業を計画する必要があります。



1C会計システムは、企業のさまざまなプロセスの会計を自動化するための、ロシア連邦およびその他のソビエト連邦諸国で最も人気のあるソフトウェアです。 1Cには、会計、人員、生産管理、貿易、倉庫などのために設計された多くの構成があります。



構成1C UT 11-ソフトウェア1C「取引管理」の構成の現在のバージョン。



IPテレフォニーは、IPを介したインターネット経由の電話接続です。 IPテレフォニーとは、従来の電話通信機能を提供する通信プロトコル、VoIP機器、ソフトウェア、技術、および方法のセットを意味します：ダイヤル、ダイヤル、双方向音声通信、およびインターネットまたはその他のIPベースのネットワークを介したビデオ通信。
