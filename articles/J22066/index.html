ステップ9.プログラムのインストール

時々、新しいタスクが表示されるか、古い問題を新しい方法で解決する必要があります。その後、追加のソフトウェアが必要になります。 Linux用の新しいプログラムが必要な場合はどうなりますか？ 特定のアルゴリズムを説明しようとします。



1.必要なソリューションの種類を理解する必要があります。 これには検索エンジンを使用できます。 問題のリクエストを作成し、適切なソリューションを選択してください。



2.次に、どのプログラムがこの問題を解決できるかを理解する必要があります。 Linuxには同じプログラムの類似物が多数ありますが、それらの中では、機能、速度、利便性、セキュリティ、無料など、重要な基準に従って最適なものを選択する必要があります。 また、検索エンジンでプログラムを探していますが、問題に対する特定の解決策の要求を既に生成しています。



3.特定のプログラムを選択したら、検索を開始できます。 Synapticを開き、検索フォームを使用して適切なアプリケーションを見つけます。 それとともに、多くの拡張機能や追加モジュールを見つけることができます。 必要なものをすべて入れます-これが最良のオプションです。 リポジトリ内のアプリケーションは通常テストされ、機能します。



アプリケーションがリポジトリで見つからない場合、これを行うには2つの方法があります。リポジトリに存在する類似物を探すか、リポジトリ外でこのアプリケーションを見つけるかです。



1.アプリケーションの公式Webサイトを見つけ、そこに.debパッケージを見つけようとします（Ubuntu Linuxがあります-deb形式に基づくパッケージシステムがあります）。 そのようなパッケージが公式Webサイトにある場合は、ダウンロードしてインストールします。



2. .debパッケージが公式Webサイトにない場合は、検索エンジンで探しています（作成者に加えて、他の人が便宜のためにアプリケーションのdebパッケージを収集できます）。 リクエストは、「xneur deb」または「gimp deb」のようになります。



3.幸運でなく、プログラムが非常にまれで、debパッケージがない場合は、一般的にどのような形で広がるかを調べます。



tar.gzからのアプリケーションのインストール



多くの場合、アプリケーションはtar.gzアーカイブで配布されます。 この形式はUbuntuではあまり便利ではありません。パッケージではなく、ソースコードとコンパイルされたアプリケーションとライブラリの両方が存在できる単なるアーカイブです。



tar.gzからのインストール：



1.アーカイブを別のディレクトリに解凍します。



2.実行可能ファイルがある場合-実行して使用し、ない場合-「コンパイル」セクションをお読みください。



SVNからのアプリケーションのインストール



SVN-Subversion。 これは、アプリケーションのソースコード、特にオープンソースを保存するコードのバージョン管理システムです。



1.アプリケーションのディレクトリを作成します。



2.ディレクトリ内のターミナルを開きます（cd 'path / to / directory'）;



3.ソースコードをダウンロードします。



svn co (SVN-)







4.「コンパイル」セクションを読みます。



CVSからのアプリケーションのインストール



CVS-同時バージョンシステム。 また、コードのバージョン管理システムでもあります。



1.アプリケーションのディレクトリを作成します。



2.ディレクトリ内のターミナルを開きます（cd 'path / to / directory'）;



3.ソースコードをダウンロードします。



cvs -z3 -d (CVS-) co ./







4.「コンパイル」セクションを読みます。



RPMからアプリケーションをインストールする



rpmパッケージはUbuntuにネイティブではありません。 通常どおりにインストールできるエイリアンユーティリティがあります（sudo apt-get install alien）。 その助けを借りて、rpmパッケージをdebパッケージに変換できます。 非常に使いやすい：



alien .rpm







そして、rpmパッケージのあるディレクトリにdebパッケージが表示されます。 そして、簡単にインストールできます。



編集



1.アプリケーションのあるディレクトリでターミナルを開きます（cd 'path / to /ディレクトリ'）;



2.アプリケーション構成に関する情報を確認します。



./configure --help







出力を見て、設定する必要があるパラメーターを決定します。 このコマンドでエラーが発生した場合、コンフィギュレーターはありません。 コンフィギュレーターが存在する場合-構成：



./configure ()







引数なしでこのコマンドを実行できます-標準設定があります。



構成中にエラーが発生する可能性があります-通常はモジュールの不足です。 エラーが発生した場合は、すぐに検索エンジンにアクセスして解決策を探します。



3.アプリケーションをコンパイルします。



make







コンパイル中にエラーが発生する場合があります。 エラーが発生した場合、検索エンジンが再び役立ちます。



コンパイル後、ほとんどの場合、起動して使用できる既製の実行可能ファイルを取得します。



4.インストール。



アプリケーションのインストールが必要な場合は、実行します（管理者権限が必要です-sudoコマンドを忘れないでください）：



make install







このコマンドは、アプリケーションファイルを必要なシステムディレクトリにコピーします。



5.アプリケーションを使用します。



この方法でインストールされたアプリケーションは、アンインストールが難しい場合があります。 したがって、checkinstallプログラムについて読むことをお勧めします。



私の記事に遅れないようにするには、RSSフィードを購読してください。
