nanoCAD 3.5：2011年12月5日リリース

最近、シーズンチケット所有者向けのバージョン3.2のリリースを発表しましたが、1か月後にバージョン3.5をリリースする予定です...なぜですか？



マーケティングの殻がない場合、新しいバージョンは実際にはバージョン3.2（追加の改善-以下を参照）と、nanoCADのアプリケーションを個別に開発できるオープンプログラミングインターフェイス（API）です。 2つ目はこのバージョンの鍵です。nanoCADは、CADアプリケーションを開発するための無料のオープンプラットフォームです。 グローバルCADコ​​ミュニティのユニークな現象-無料のnanoCADに基づいてアプリケーションを開発し、独自の目的で使用することができ、完全に合法です。



これについてもう少し詳しく説明し、CADユーザー向けのアンケート（現在作業中のCADアプリケーションを知りたい）をカットします。



技術開発


現在、nanoCAD 3.0のユーザーライセンスベースは7万6千ジョブであり、ソフトウェア使用率はこの1年で4倍以上増加しています。 巨大さは、最適化、バグ修正（喜んで修正します）、および非標準のハードウェア構成に関する提案とともに、あらゆる側面から膨大な数のリクエストが寄せられるという事実につながります...現在の基本機能は、2D図面の開発に十分です。 したがって、新しいバージョンに含まれています：

  • コンピューターグラフィックスアダプターテストシステム：プログラムの最初の起動時に自動的に起動し、ハードウェアと現在使用されているドライバーの両方を分析します。 そして、分析の結果に応じて、最高のパフォーマンスを実現するためにnanoCADを自動的に構成します。 さらに、新しいVPERFTESTコマンドを実行すると、画面にテスト結果が表示され、ビデオサブシステムに関する情報を別のファイルとして保存できます。
  • 新しく作成されたMTEXTおよびTEXTの高さは、レイアウトスケールに関連付けられました。 デフォルトでは、SPDS（1：100）を設定すると、高さは250単位に等しくなります。 ESKD（1：1）の高さを設定する場合-2.5単位。 また、たとえば1:10などのスケールでノードを作成すると、デフォルトの高さは25になり、作業が大幅に加速されます。

  • 2D設計の最適化：

    • [外部リンクの挿入]ダイアログに自動パンオプションが追加されました;有効にすると、画像は自動的にズームおよびパンされ、挿入されたリンクが表示されてフルスクリーンで表示されます。
    • ドキュメントで使用可能な名前付きビューのリストのコマンドラインに出力を追加しました-[？]オプション付きのVIEWコマンド。
    • 選択時にオブジェクトを追加/削除するロジックを修正-通常のCADと同様
    • SHIFTを押し続けると、ORTOモードが一時的にオン/オフされます。
    • オブジェクトに沿ってブロックを配布する新しいレイアウトコマンド（Drawing \ Point \ Layout）;
    • ユーザー座標系などを制御するための新しいUCSパネル

    • など


一般的に、3.2と比較しても小さな変更が多数あります。新しいバージョンを以前の無料のものと比較すると、nanoCAD 3.5では半年以上で300以上の改善と修正が行われました。 50以上のチームとそのオプションを修正および改善しました。 基本的に、すべてがより速く、より最適化された描画に焦点を当てています。



しかし、バグを修正することは3.5では最も重要なことではありません。 私たちの観点からさらに興味深いのは、nanoCADの下で、どの開発者でもアプリケーションを開発できることです。



オープンソフトウェアインターフェース


一般的に、正直に言うと、小さなアプリケーションをより早く開発できます-COM APIを開いておくことができます（ドキュメントはプログラムがインストールされているフォルダーにあります：％nanocad％\ help \ api）。 ただし、COMはnanoCADで非常に幅広い機能をすべて提供するわけではありません。



バージョン3.5のリリースに伴い、nanoCAD開発者クラブを開設し、そのメンバーは次のタイプのAPIにアクセスできます。

  • NrxGateは、C ++アプリケーションを作成および移行するために設計された古典的なC ++ APIです。
  • Multi CAD APIは、クロスプラットフォームCADアプリケーションを作成するために設計された独自のC ++ APIです。
  • .NET APIは、.NETアプリケーションの開発用に設計された最新のAPIです。


API構造は他のCADプラットフォームのAPIに近いため、アプリケーションを既に開発している場合は、プラットフォームに簡単に移植できます。 そして、この作業を簡単にするために、nanoCADの開発者のコ​​ミュニティを組織してサポートします。 同じコミュニティがAPIのさらなる開発に貢献します-開発はまだ完了していません。 クラブの発表に注意深く従ってください！



今、どうすれば助けられますか？


新しいバージョンを起動する前に、CADユーザーから追加情報を受け取りたいと思います。CADソフトウェアに追加するアプリケーションは何ですか？ 開発者は誰ですか？



アイドル状態の好奇心には興味がありません。近い将来、nanoCADプラットフォームでお気に入りのアプリケーションが見つかる可能性が大いにあります。 アプリケーション開発者に連絡して、既存のCADプラットフォームと互換性のある新しい無料のnanoCADプラットフォームに注意してください。 または、アプリケーションについて教えてください-開発者に連絡します:-)



アプリケーションについて話すには、 Googleプロフィールに記入してください。
