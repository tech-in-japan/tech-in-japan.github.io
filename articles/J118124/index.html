GitHubのマージボタン（マージ）

Pull 2.0リクエストを使用すると、コードの検証とパッチの受信がこれまでになく簡単になります。 このメカニズムはGitHubで広く使用されており、オープンソースプロジェクトで使用したいと思っています。



たとえば、 神の 文書修正プールのリクエストを考えてみましょう。









従来、このプルリクエストのマージには、Gitコマンドラインを使用した多くの手順が必要です 。 しかし、これはそうではありません！



これで、すべてのプルリクエストにマージボタンが含まれるようになりました。









マージの競合が検出された場合、ボタンは手動のマージ命令に置き換えられます。









ボタンを1回クリックすると、プルのリクエストが自動的にマージされて閉じられます。









マージは常にプルリクエストの番号、コード、ヘッダーを含むマージコミット（ git merge --no-ff



）を生成します。









プルリクエストを必ず試してください 。 良いマージ！
