VHDネイティブブートの内側と外側

この記事の目的は、VHDネイティブブートと呼ばれる非常に便利であまり知られていないWindows機能、つまりVHD / VHDx形式の仮想ハードディスクから起動する機能に関する私の経験を説明することです。



バージョン7以降、Windowsには仮想ディスクVHD / VHDx（以下、単にVHDと呼びます）を作成し、「ディスクの管理」グラフィカルインターフェイスとdiskpartコマンドラインユーティリティを介してそれらを接続および切断する機会があります。 さらに、Windowsはそのようなディスクから起動することを学習しましたが、すべては問題ありませんが、この同じネイティブブートは古いバージョンの所有者、つまりPro以降からのみ利用できました。 明らかに、これはマーケティングの制限にすぎません。Windows10の登場により、Anniversary Update（1607）とCreators Update（1703）を確認したため、これ以上の制限はありません。 これはWindows 10 Homeでも機能し、ホストとゲストの両方として機能できます。 以下で、その外観と使用方法について学習します。



長い間、私はいわば、稼働中のコンピューター、つまり内部仮想化に関連して仮想化を使用するというアイデアに興味がありました。 ソフトウェア開発者、セキュリティの専門家、テストの専門家にとって、仮想マシンはどれほど便利で便利なのか。 しかし、このビジネスはまだ自宅/職場のコンピューターとそのオペレーティングシステムのレベルには達していません。 オペレーティングシステムが非常に複雑でデリケートなコンポーネントである場合、ユーザーを無差別に信頼することはできず、オペレーティングシステムが何かに感染または損害を与えようとすることは明らかです。 はい、リカバリポイントから（つまり、シャドウコピーから）バックアップと復元がありますが、これらはすばらしいことです。 ただし、これらは非常にエラーに敏感なコンポーネントであり、保存できない場合があります。さらに、多くのマルウェアはシャドウコピーを削除し、ユーザーに機会を与えません。 「落ちた」または「台無しになった」システムが数分以内に稼働状態に戻るように、コピーと貼り付けのレベルでシンプルで平凡なものが欲しいです。 もちろん、ソリューションをシステム自体に組み込み、単にシステムに組み込むのが理想的です。 Hyper-Vはまだ適切ではありませんが、必要なレベルまでドープできます。 結局のところ、私はマシンのすべての機能、すべての鉄、すべてのパワーを最小限の犠牲者で利用できるようにしたいのです。



システム全体が1つのファイルに収まることを考えると、実際のディスクの代わりに仮想ハードディスクを使用することは正当な被害者のように思えます。このファイルを時々「横に」どこかにコピーするだけで十分です。 結局のところ、1つのファイルをコピーすることは、たとえ大きいファイルであっても、数万よりも明らかに簡単です。 さらに、このようなファイルは、組織にWindowsを展開するために簡単に使用できます。



数種類のコンピューターがある場合は、システムと必要なすべてのソフトウェアをVHDにインストールし、このファイルをすべての同様のコンピューターにコピーするだけで、現場作業が最小限に抑えられます。 Windowsをロードせずに何らかのシェルを使用するとよいでしょう。Windowsにアクセスして、ファイルレベルでVHDを操作、コピー、置換、更新などできる「ハイパーバイザーコンソール」のようなものです。 さらに、Windows自体の構成にはそのようなシェルがあり、Windows Recovery Environment、WinREと呼ばれます。 実際にどのように見えるか見てみましょう。



1. Windowsを最初からVHDにインストールする



このトピックはWebで広く取り上げられており、賢明なガイドが多数あります（記事の最後のリンクを参照してください）。

一般的に、コンピューターがインストールディスクから起動した時点で、魔法の組み合わせShift-F10を押すだけです。 並行して、コマンドラインウィンドウが開きます。そこでは、diskpartを使用して、実際のハードディスク（コンピューター/ディスクが新しい場合）をフォーマットおよびパーティションし、必要なサイズのVHDを作成する必要があります。 簡単にするために、64ビットバージョンとハードドライブをMBRでインストールすることを検討します。



そのため、ハードディスクが破損し、対応するボリュームのVHDsフォルダーが作成されます。今度はdiskpartで、このフォルダーに仮想ハードディスクを作成し、フレンドリ名を付けて添付を実行する必要があります。次の文字が仮想ディスクに割り当てられます。 これで、Windowsインストールウィンドウに戻り、この特定の文字をインストール用に選択できます。 すべて、インストーラーはすべて自分で行います。 含めると、BCDファイルに目的のエントリが追加されます。



私は、bcdeditを使用することはあまりにも残酷な自虐行為のように思えたので、操作のために1つのサードパーティツールを使用することを許可しました。これは、対応する容量のBooticeユーティリティです。 同じインストールディスクにあるとします。 そうでない場合は、将来、「ハイパーバイザー」シェルに「投げる」方法を示します。



そのため、デモンストレーションのために、25 Gbのハードドライブを1台用意します（お気に入りのVirtualboxを使用して表示します）。1つのセクションがあり、仮想ディスクを作成してWindows 10をインストールしたVHDフォルダーがあります。







これが、Booticeのシステムのブートメニューの外観です（セクションBCD、イージーモード）。







ここで25 Gb C：これは、仮想20 Gbを作成し、Windows 10をインストールした「物理」ディスクです。すべて問題ありませんが、管理用のシェルを作成する必要があります。 ご存じのように、WinREは常にWindowsとともにインストールされ、起動の問題が検出された場合に助けになります。 私たちは別の目的のためにそれを必要とします、私はそこにVHDファイルを扱うために行きたいです。 WinREアイテムをブートメニューに追加します。 これを行うには、Booticeでプロフェッショナルモードを使用します。左側のリストの最後のオブジェクトはWindows回復であり、右側ではVHDでその場所を確認できます。







このオブジェクト、またはそれへのリンクをブートメニューリストに追加し、左上隅でWindowsブートマネージャーブランチを選択し、右ペインで[表示順序]アイテムを選択し、ドロップダウンリストからWinREアイテムを追加する必要があります。







これで、イージーモードに戻るとわかるように、Windows回復環境の項目がシステムのブートメニューに表示されます。







再起動して2番目の項目を選択すると、WinREがロードを開始します。ここでは、トラブルシューティング、高度なオプション、コマンドラインの項目のみに関心があります。 これらはすべて、Windowsインストーラーと、広く知られているWindows Preinstallation EnvironmentであるWinREの両方に似ています。 実際には、ここからシェルを使用した作業が開始されますが、どれを選択するかはそれほど重要ではありません。すべてがほぼ同じであるためです。



メインのハードドライブはC：ドライブで、master.vhdはVHDsフォルダーにあり、どこかに簡単にコピーできます。 WinREでは、magicコマンドを使用してネットワークを接続します。 



wpeutil initializenetwork


ネットワークアダプタードライバーが自動的に選択されて起動され、DHCPサーバーからIPアドレスが取得され、ネットワークを操作できます。 Virtualboxでは、次のコマンドでネットワークフォルダーに接続できます。 



 net use z: \\10.0.2.2\d$


そして、そこからすでにシェルで動作するために必要なツールをコピーします。 x64バージョンが選択されているため、WinREで実行されるプログラムはx64である必要があり、サロゲートは開始されません。



Booticeに加えて、Far Manager、7-zipが簡単に追加され、それらを使用することで、すでに何らかの形でより楽しくなっています。 動作するPalemoon Portable Webブラウザーを見つけることさえできました。これを使用すると、Webから必要なコンポーネントを非常に簡単にダウンロードできます。 Cygwin64は完全に機能し、混合環境でssh / rsyncの道を開きます。 さらに、vhdファイルを安全にアーカイブおよびコピーすることができます。 master.vhdに問題がある場合、WinREを起動してネットワークストレージからバックアップコピーを取得し、WinREを終了してシステムを元に戻します。



WinREシェルから直接diskpartまたはBooticeを使用して、新しいVHDディスクを作成し、Windowsインストーラーを実行できます。他のバージョンを追加し、この新しいWindowsを新しいVHDにインストールする場合、OSブートメニューの目的の項目が追加されます。



残っているのは、master.vhdですべてが非常に悪く、WinREシェルを起動できない場合に備えて安全にプレイすることです。これは、このディスクの一部であるためです。 もちろん、これは致命的ではありません。Windowsインストールディスクからいつでも起動してShift-F10を押すことができますが、WinREをホストディスクに配置してそこから起動することができます。 ブートメニューは次のようになります。







2.実行中のコンピューターのVHDにWindowsをインストールする



新しいVHDを作成して接続し、インストーラーを実行して、接続されたドライブに割り当てられた文字を選択することにより、既存のコンピューターにオペレーティングシステムを追加することは問題ではありません。 さらに難しいタスクは、システムの物理ディスクに既にインストールされている現在の構成を仮想ディスクに転送することです。 ここでいくつかのオプションが思い浮かびます。 最初に思い出したのは、Windows Backupを使用することです。これは、システムイメージ作成モードでVHD（vhdx）ファイルを作成するだけだからです。 必要なことは、ブートメニューにそのようなVHDへのリンクを追加し、何が起こるかを確認することだけであるように思われます。 そのため、最初にWindowsを起動したときにエラーが発生し、その後のすべてで、非常に長い間、何かが慎重に読み込まれ、元のシステムのロック画面の写真でウィンドウを垣間見ましたが、再び消えました。 理由はわかりませんが、バックアップから取得したVHDディスクからWindowsをロードできません。 SysinternalsバンドルのDisk2vhdを使用して、別の方法で行かなければならなかった。



すべてが非常に簡単です。物理ディスクのパーティションまたはディスク全体を選択すると、Disk2vhdからVHDファイルが作成されます。







しかし、それから不便が始まります。 結果のVHDは、実際のサイズに関係なく、物理ディスク全体のサイズであると報告します。 つまり、物理ディスクが180 Gbで、サイズが100 Gbの最初のパーティションのみを選択した場合、VHDファイルは約50 Gbでしたが、約180 GBと報告されます。 ここでの問題は、そのようなVHDから起動する場合、Windowsが機能するには180 GBのスペースが必要になることです。 つまり、最適化（デフラグ、sdelete -z）、または圧縮（コンパクトvdisk、縮小）の方法に関係なく、Disk2vhdの元の特性を取り除くことはできません。 複雑な変換に行き、空のVHDを作成し、それをVirtualbox PartedMagicにロードし、変換された空のVHDをそこに入れて、Gparted（およびbootrecを台無しにしたくない場合はClonezilla）を使用して、パーティションを移動する必要がありました。 結果は20 GbのVHDであり、この記事から現在執筆中です。



3.差動VHDを使用する



特に信頼性の低い環境、公共のコンピューター、またはいくつかの危険な実験では、差分、変更された情報、および元のVHDのみを記録する差分VHDディスクを使用すると便利です。 開始するには、VHDドライブ上に動作中のシステムが既に存在している必要があり、その後、差動ドライブでオプションを追加する必要があることは明らかです。 このようなディスクは、diskpartで作成することも、すべて同じBooticeで作成することもできます。 master.vhdをメインディスクとし、[作成]ボタンをクリックして、差分のchild.vhdを作成します。







次に、master.vhdの代わりに差分child.vhdを指定して、BCDのVHDからの起動を担当するアイテムを追加/修正する必要があります。



これを行うには、Booticeでプロフェッショナルモードを使用し、既存のWindows 10アイテムのコピーを作成し（右クリックしてこのエントリを複製します）、Windows 10 Child VHDで新しいものの名前を変更します。 さて、この時点で、vhdファイルの名前を変更してApplicationDeviceとOsDeviceを修正します。







これで目的のアイテムがブートメニューに追加されました。 Windows 10 Child VHDを選択すると、Windowsが起動し、これ以降、すべての変更がchild.vhdに書き込まれます。 ロード時のchild.vhdの下では、実際のサイズが数百倍小さい場合でも、master.vhdで示されているように、つまり20 Gbのスペースが予約されることに注意してください。 時々、マージ手順を実行することは意味があります。つまり、何も失わないように、累積した差分を子からマスターに送信します。 実際には、子ではなくmaster.vhdに基づくマスターまたはWinREで起動すると、マスターと子の間の接続が切断されるため、子を修復する必要がありますが、Booticeはこれを実行できます。







4.起動可能なVHDを使用する場合の推奨物理ディスク構成



次のように物理ディスクをパーティション分割することをお勧めします。



VHDファイルを保存するのに十分な大きさの1つのセクションは、必要なVHDの数に依存します。 Windows x64の最小インストールには20 GBが必要です。ダイナミックディスクを作成できます。つまり、実際にサイズが大きくなるのは、内部になったときだけです。 ただし、動的VHDの読み込み時に、Windowsは指定された最大サイズに応じて領域を予約することを再度強調します。



マイクロソフトでは、実稼働環境で固定サイズのVHDを使用し、動的なVHDをテストのみに使用することをお勧めしますが、動的なVHDでのパフォーマンスの低下はあまり感じませんでした。



2番目のセクションでは、たとえば、異なるVHDから起動し、同じファイルとプログラムで作業する場合、ユーザーデータとポータブルアプリケーションのセットを作成します。 このような区分は、VHDパーティションを完全に非表示にして、エンドユーザーの不当なアクションを避けるためにも役立ちます。



また、選択したドライブとVHDを保存するパーティションを考慮して、diskpart用のこの単純なスクリプトを使用してパーティションを非表示にできます。 



 sel disk 0 sel part 1 set id=17


このセクションは非表示になり、文字は割り当てられませんが、Windowsはこのセクションに保存されているVHDから起動します。 唯一の注意点は、ページファイルの物理ディスク上の場所を選択することです。 システムによって選択され、これがまさに非表示になるセクションである場合、Windowsを起動するたびに、ページファイルを作成する場所を尋ねられます。



そしてチェストを戻すには、diskpartでコマンドを実行するだけで十分です。 



 set id=07
または 

 set id=27
